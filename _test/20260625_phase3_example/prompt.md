# Phase 3 분석 프롬프트 (1편)

너는 NAND 플래시 LDPC ECC 엔지니어다. 첨부된 논문 PDF **1편**을 정독하고,
"이 기법을 **NAND 고전 바이너리 LDPC ECC**(부호설계→디코더→HW)에 떼어다 이식할 수 있는가"를
판단해 아래 표를 채워라. 표준/교과서 수준 재사용뿐이면 이식성 `하`.

규칙:
- 각 카테고리는 **정해진 enum 중 하나**만 (오타 금지 — 필터 키). 숫자 항목은 본문 수치 그대로.
- **본문 근거로만** 채운다. 모르면 `미상`, 디코더 아니면 C섹션 전부 `N/A`. 추측 금지.
- 자유 텍스트는 핵심기여·한계·추가확인 3개뿐, 각 한 줄.
- enum 목록은 `categories.md` 참조.

## 출력 (이 형식 그대로: 2열 표 + JSON)

```
### {id} — {제목} ({연도})

| 카테고리 | 값 |
|---|---|
| 이식성 | 상\|중\|하 |
| NAND관련성 | 직접\|간접\|낮음 |
| 대상 | decoder\|encoder\|both\|code-design\|other |
| 부호종류 | QC-LDPC\|SC-LDPC\|regular\|irregular\|protograph\|기타\|미상 |
| 부호rate | (숫자\|미상) |
| 부호length | (숫자\|미상) |
| 연판정 | hard-1bit\|soft-2~3bit\|soft-4bit+\|무관\|미상 |
| 핵심기여 | (한 줄) |
| HW설계 | O\|X |
| 검증수준 | 이론\|시뮬\|FPGA\|ASIC합성\|실측\|미상 |
| 복잡도 | (gatecount@공정\|미상) |
| 병렬화 | 없음\|부분\|완전병렬\|미상 |
| Throughput | (숫자Gbps\|미상) |
| 메모리라우팅 | 언급\|없음\|미상 |
| 정정능력향상 | O\|X\|N/A |
| 개선영역 | waterfall\|error-floor\|both\|N/A |
| iteration감소 | O\|부분\|X\|N/A |
| latency | 개선\|동등\|악화\|N/A |
| 추천도 | 상\|중\|하 |
| 한계·주의 | (한 줄) |
| 추가확인 | (한 줄) |

> 총평: (한 줄)
```

그 아래 같은 내용을 JSON 블록으로 한 번 더 (키=enum 영문 슬러그, DB 적재용).
