Linear Complexity Computation of Code Distance and Minimum Size of Trapping Sets for LDPC Codes with Bounded Treewidth
Qingqing Peng, Ke Liu, Guiying Yan, Guanghui Wang

arXiv:2509.13040v2 [cs.IT] 30 Jan 2026

Abstract—It is well known that, given b ≥ 0, finding an (a, b)-trapping set with the minimum a in a binary linear code is NP-hard. In this paper, we demonstrate that this problem can be solved with linear complexity with respect to the code length for codes with bounded treewidth. Furthermore, suppose a tree decomposition corresponding to the treewidth of the binary linear code is known. In that case, we also provide a specific algorithm to compute the minimum a and the number of the corresponding (a, b)-trapping sets for a given b with linear complexity. Simulation experiments are presented to verify the correctness of the proposed algorithm.
Index Terms—low-density parity-check codes, tree decomposition, trapping sets, NP-complete.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] have been extensively studied by coding theorists and practitioners over the past decade because of their near-capacity performance when decoded using low-complexity iterative decoders. However, the error floor phenomenon at high signal-to-noise ratios (SNRs) causes the bit error rate (BER) curve to flatten, limiting further performance improvement. To address this issue, researchers have focused on identifying the underlying causes of the error floor. Extensive experimental studies [2]– [7] have shown that structures known as (a, b)-trapping sets are the primary contributors to the error floor, where a represents the number of variable nodes in the structure, and b denotes the number of check nodes with an odd degree. In particular, trapping sets with small values of a and b are the most significant contributors to the error floor.
Identifying the smallest trapping set in an LDPC code is a crucial problem, which can be formulated as follows:
Problem: MINIMUM TRAPPING SET Instance: A Tanner graph G, and an intege b ≥ 0. Question: What is the minimum value of a for which an (a, b)-trapping set exists in graph G? And find an (a, b)trapping set with minimum a.

In particular, when b = 0, finding the minimum size of an (a, 0)-trapping set is equivalent to computing the minimum distance of the LDPC code. Let H be the parity-check matrix corresponding to graph G. If there exists an (a, 0)-trapping set S in G, let x be the vector where the bits corresponding to the variable nodes in S are set to 1, while all other bits are set to 0. It is straightforward to verify that Hx = 0, and the weight of x is a. On the other hand, if x is a weight-a vector satisfying Hx = 0, let S be the set of variable nodes corresponding to the nonzero entries of x. In the induced subgraph of S, each check node is connected to an even number of nodes in S. Therefore, S is an (a, 0)-trapping set.
The study of the MINIMUM TRAPPING SET and its special cases has a long history. As early as 1971, Dominic Welsh [8] called for the development of efficient algorithms for computing the minimum distance of a code. For nearly two decades, this problem remained unsolved and was widely recognized as an open problem [9]–[12]. It was not until 1997 that Alexander Vardy formally proved [11] that computing the minimum distance of a binary linear code is NPhard. The study of more generalized trapping sets originated from Richardson’s semi-analytical analysis in [2], where he demonstrated that trapping sets are the primary cause of the error floor and proposed a method for rapidly estimating the error floor based on trapping sets. Since then, researchers have recognized the importance of identifying trapping sets in LDPC codes and have made considerable efforts to develop efficient methods for detecting dominant trapping sets [13]– [16]. Unfortunately, as proven by McGregor et al. [17]–[19], not only is the problem of finding a minimum-size trapping set NP-hard, but even approximating its size remains NP-hard, regardless of the sparsity of the underlying graph. This implies that, for binary linear codes, no polynomial-time algorithms exist to solve the MINIMUM TRAPPING SET problem in the worst case.
Although the MINIMUM TRAPPING SET problem has been proven to be NP-hard, it continues to attract significant attention. Researchers have been persistently working on designing efficient algorithms to reduce the coefficient of exponential complexity, enabling the identification of the minimum size of trapping sets for codes of practical length (1000 to 10000) [20]–[23]. Several reasons underscore the importance of this problem. Firstly, based on trapping sets, the error floor of LDPC codes can be quickly estimated [2], [3], which is particularly valuable for LDPC codes with error floors too low to be accurately observed through conventional

computer simulations. Secondly, by modifying the decoder to account for the influence of specific trapping set structures, the decoder’s ability to mitigate error-prone patterns can be enhanced, thereby improving the overall error-correcting performance of the code(see, e.g., [24] and [25]). Furthermore, a better understanding of trapping sets can guide the design of LDPC codes by eliminating small trapping sets to achieve a lower error floor (see, e.g., [26]).
The treewidth of a graph is a fundamental concept in graph theory. It indicates how far a graph is from being a tree or forest: the closer the graph is to a forest, the smaller its treewidth. The concept was first introduced by Bertel´e et al. [27] under the name dimension. It was later rediscovered by Halin [28] in 1976 and by Robertson et al. [29] in 1984. Now, treewidth has garnered significant attention due to its pivotal role in addressing classical NP-complete problems (see, for example, [28], [30]–[32]). Notably, many NP-complete problems can be efficiently solved in polynomial time when restricted to graphs with bounded treewidth [33]. One such example is the w-Dominating-Set problem, which can be formulated as follows.
Problem: w-Dominating-Set Instance: Graph G = (V, E), k, r ∈ N. Parameter: treewidth k Question: Let a w-dominating set be a vertex subset S such that for all v ∈ V, either v ∈ S or it has at least w neighbors in S. Decide whether G has a w-dominating set of cardinality at most r. The MINIMUM TRAPPING SET problem, as an NP-hard problem, is also expected to be solvable in polynomial time with bounded treewidth. Our primary contribution is proving that the MINIMUM TRAPPING SET problem can be solved with linear complexity in the code length for LDPC codes with bounded treewidth k. Furthermore, if a tree decomposition of width k for the LDPC code is known, we also provide a specific algorithm to compute the size and the number of the smallest trapping sets for a given b ≥ 0 with linear complexity. The paper is organized as follows. Section II introduces the fundamental concepts of trapping sets in LDPC codes and treewidth. Section III demonstrates that the minimum distance can be computed with linear complexity when the treewidth is bounded. Section IV shows that the minimum size of an (a, b)-trapping set and the corresponding number of such sets for a given b can also be computed with linear complexity under bounded treewidth. Section V provides verification using spatially coupled LDPC codes as an example. Concluding remarks are provided in Section VI.
II. PRELIMINARIES
In this section, we define several important concepts, such as trapping sets and treewidth, as well as some set operations that are essential for Sections III and IV.
A. Trapping Set
Let G = (L ∪ R, E) be a bipartite graph, or Tanner graph, corresponding to the LDPC code, where L is the set of variable

nodes, R is the set of check nodes, and E represents the edges between L and R. For a node u ∈ L ∪ R, its degree is the number of neighbors of u. For a subset S ⊂ L, let ΓG(S) denote the set of neighbors of S in G. The induced subgraph of S ∪ ΓG(S) in the Tanner graph, denoted G[S ∪ ΓG(S)], has S ∪ ΓG(S) as the set of vertices, and the set of edges is given by {(v, c)|v ∈ S, c ∈ ΓG(S), (v, c) ∈ E}. Let ΓoG(S) denote the set of check nodes in ΓG(S) with odd degree in G([S ∪ ΓG(S)]). For notational simplicity, throughout this paper we also use G[S] to denote G[S ∪ ΓG(S)].
Definition 1. For a Tanner graph G = (L ∪ R, E) and a nonempty subset S ⊂ L, we define S as an (a, b) trapping set in G if |S| = a and |ΓoG(S)| = b. The integer a is referred to as the size of the trapping set S.

IV. COMPUTER THE SIZE AND NUMBER OF THE SMALLEST TRAPPING SET WITH b > 0 IN CODES WITH
BOUNDED TREEWIDTH
As a generalization of Section III, this section presents an algorithm to find an (a, b)-trapping set with the minimum a in Tanner graph G, given b > 0, in linear complexity with respect to the code length. Furthermore, we also demonstrate that the number of smallest trapping sets with b > 0 contained in G can be computed in linear complexity with respect to the code length.
First, we provide the generalized form of the algorithm in Theorem 1 to compute the size and the number of the smallest trapping sets with b > 0 in G, along with its proof.
Theorem 4. Let G be a Tanner graph with treewidth tw(G) = k, corresponding to an LDPC code of length n. Then, for a given b > 0, the size and the number of the smallest (a, b) trapping set in G can be determined in time O(n(b + 1)24k).
Proof. A nice tree decomposition of G is given by (T, (Bt)t∈T ), with root r and width k. We adopt a dynamic programming algorithm, similar to that in Theorem 1, to solve the Minimum Trapping Set problem. The algorithm likewise processes the nodes of T in a bottom-up order t1, t2, . . . , tr, such that every child of a node ti appears earlier in the order. The objective is achieved by sequentially updating the function values fti and gti at each node ti ∈ T . Corollary 5. Let G be a Tanner graph with treewidth tw(G) = k, corresponding to an LDPC code of length n. We can identify an (a, b) trapping sets with minimum a in G in time O(n(a + 1)(b + 1)34k).

The GMTS algorithm is a slight modification of the algorithm presented in Theorem 4. It operates on the graph G together with a tree decomposition (T, (Bt)t∈T ), and returns a node u such that, for some ti ∈ T and its child tj, Bti = Btj \ {u}, and the corresponding value fti (v, B, b) reaches a for the first time during the execution of the algorithm.

Corollary 6. Let G be a Tanner graph with pathwidth pw(G) = k, corresponding to an LDPC code of length n. Then, for a given b, the size and the number of the smallest (a, b) trapping set in G can be determined in time O(n(b + 1)2k).

VI. CONCLUSION For any fixed b ≥ 0, we have shown that for LDPC codes with bounded treewidth, the minimum size of an (a, b) trapping set, the number of minimum trapping sets, and a corresponding trapping set itself can all be computed in time linear in the code length.
