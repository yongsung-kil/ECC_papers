Affine Subcode Ensemble Decoding of Linear Block Codes
Jonathan Mandelbaum, Graduate Student Member, IEEE, Paul Bezner, Graduate Student Member, IEEE, Holger Ja¨kel, Member, IEEE, Stephan ten Brink, Fellow, IEEE, Laurent Schmalen, Fellow, IEEE

arXiv:2604.06889v1 [cs.IT] 8 Apr 2026

Abstract—In the short block length regime, ensemble decoding schemes with their inherently parallel structure can improve error correction performance and reduce latency compared to stand-alone suboptimal decoders such as belief propagation (BP). In this work, we introduce affine subcode ensemble decoding (aSCED), which uses an ensemble of decoders operating on linear block codes and both linear and strictly affine subcodes. This generalizes the recently proposed subcode ensemble decoding (SCED), which is restricted to linear subcodes. We derive BP update rules for affine subcodes and show that aSCED simplifies ensemble design compared to SCED, multiple bases BP, and automorphism ensemble decoding. Monte-Carlo simulations of two low-density parity-check codes and two Bose–Chaudhuri– Hocquenghem (BCH) codes demonstrate improved error correction performance of aSCED over competing existing ensemble schemes. Notably, for one BCH code, when combining ensemble design with algorithms for constructing high-performance paritycheck matrices, aSCED achieves near-maximum likelihood performance using only 64 BP decoding paths.
Index Terms—Belief Propagation Decoding, Ensemble Decoding, Universal Codes, Code Agnostic Decoding
I. INTRODUCTION
L OW-density parity-check (LDPC) codes decoded with belief propagation (BP), a low-complexity iterative message passing algorithm, achieve excellent error correction performance in the large block length regime [3]. However, in the short block length regime, the decoding performance of BP becomes unsatisfactory [4], [5]. This hinders the application of LDPC codes in emerging applications such as the Internet of Things (IoT) and ultra-reliable low-latency communication (URLLC) in future wireless systems. In contrast, polar codes, proposed by Arıkan in 2009 [6], paired with successive cancellation (SC) list decoding [7], are well known for their strong performance at short block lengths.
There exist two independent key research directions to enable the use of BP in the short block length regime:
1) Parity-check matrix (PCM) design: Many short codes with large minimum Hamming distance, e.g., Bose– Chaudhuri–Hocquenghem (BCH) codes, exhibit dense

Tanner graphs with short cycles that degrade BP performance. Different approaches to enhance BP performance aim at improving sparsity or eliminating short cycles in the graphical representation of the code [8]–[13]. 2) Ensemble decoding: Employing multiple suboptimal decoding algorithms in parallel—not limited to BP—and selecting the best estimate to improve overall the error correction performance [14]–[16].
Ensemble decoding can improve the performance in the short-block regime, enhancing both latency and frame error rate (FER) [15], [17], [18]. Prior work on ensemble decoding methods achieves diversity by decoding over modified representations of noise realizations or graphs [1], [14]–[24].
Homomorphism-based ensemble decoders, such as automorphism ensemble decoding (AED), generalized AED (GAED), and endomorphism ensemble decoding (EED), leverage known code symmetries to transform the noise representation. If the automorphism group is known and its effect is not absorbed by a symmetry of the decoder, AED can yield performance gains [15]–[17], [19]. Related noise-altering ensemble decoding schemes include noiseaided ensemble decoding [20] and saturated BP [21]. Other methods, such as multiple bases BP (MBBP) and scheduling ensemble decoding (SED), improve performance by decoding over ensembles of distinct graphs or BP variants. MBBP, for instance, employs a set of equivalent PCMs, i.e., structurally different graphs, while SED varies the scheduling order of layered BP updates [16], [18]. However, when considering an arbitrary code, MBBP relies on an NP-complete search for minimum-weight codewords of the dual code, severely limiting its applicability for codes of medium-to-long block lengths [13], [25].
In [1], we introduced subcode ensemble decoding (SCED), an ensemble decoding scheme using different linear subcodes of the code C. PCMs of subcodes are obtained by appending linearly independent rows to the original PCM, thereby forming PCMs for BP-based SCED. It is essential to ensure a linear covering (LC), i.e., that the union of the linear subcodes covers the code C [26]; otherwise, some codewords are not decodable.
For arbitrary codes, SCED simplifies ensemble design. Unlike MBBP, it only requires identifying low-weight linearly independent rows, which is easier than searching for minimum-weight dual codewords. Moreover, unlike AED, SCED does not require any knowledge of the code structure beyond its PCM. For the codes investigated in [1], MonteCarlo simulations confirm that SCED consistently outperforms AED, which itself surpasses other ensemble methods [1], [16].

In this work, we extend SCED by introducing aSCED, which not only decodes on linear subcodes but also operates on suitably defined affine subcodes. Our contributions are:
(i) we provide theoretical results on aSCED showing that the additional use of affine subcodes further simplifies the ensemble design compared to SCED.
(ii) we demonstrate that some decoders in the ensemble employ the same decoding graphs, which potentially allows for hardware reuse.
(iii) we develop an efficient approximation of the ensemble list error rate (LER), which equals the FER when maximum likelihood (ML) decoding errors are negligible.
(iv) we show that the use of subcodes allows us to design better performing PCMs. To demonstrate this, we pick up the second research direction that aims to enhance BP performance by designing high-performance PCMs. In particular, we use a refined version of the algorithm from [9] to construct the so-called structured sparse PCMs (ssPCMs) of subcodes and demonstrate different structural benefits of using subcodes.
(v) we conduct Monte-Carlo simulations for both LDPC and BCH codes, demonstrating improved performance compared to other equal-complexity ensemble decoding schemes. Notably, for the BCH code C1(63, 30), we show that BP-based aSCED using 64 paths of subcode ssPCMs already achieves near-ML performance.
The remainder of this paper is organized as follows. In Sec. II, we briefly review linear codes and BP decoding, before introducing a general framework of ensemble decoding in Sec. III. In Sec. IV, we present aSECD and prove some of its properties. Additional structural benefits enabled by the usage of subcodes are discussed in Sec. V. Numerical results supporting these structural advantages and demonstrating the improved error-correction performance of aSCED are provided in Sec. VI. Finally, Sec. VII concludes the paper.

II. PRELIMINARIES
A. Linear Codes and Subcodes
A linear block code C(n, k) =: C with k ≤ n is a k-dimensional subspace of the vector space Fn over a field F. When clear from the context, the block length n ∈ N and information length k ∈ N are omitted. Every linear code can be defined as the span of the rows of a generator matrix G ∈ Fk×n or by the null space of a PCM H ∈ Fm×n, i.e.,
C := uG : u ∈ Fk = x ∈ Fn : HxT = 0 ,

where rank(G) = k and rank(H) = n − k ≤ m. We denote the code characterized by H as C(H). A PCM with m > rank(H) is called overcomplete. The weight wt(H) := j,i Hji of H is the number of non-zero elements. The dual code of C, denoted by C⊥, is defined by interchanging the generator matrix and the PCM, i.e., H⊥ = G. We refer
to codewords of the dual code simply as dual codewords. Following [1], a linear subcode Cs(n, k′) of C with k′ ≤ k
is a k′-dimensional subspace of C. We define an strictly affine subcode Ca(n, k′) of C by a tuple (Cs(n, k′), xa) as
Ca(n, k′) := {xa + x : x ∈ Cs(n, k′)} =: xa + Cs(n, k′), (1)
where Cs(n, k′) is a linear subcode of C and the affine offset xa satisfies xa ∈ C but xa ∈/ Cs(n, k′) requiring k′ < k. Thus, a strictly affine subcode Ca coincides with a coset within the quotient space C/ Cs. Let Hs denote a PCM of Cs, then we define the affine syndrome sa := HsxaT. Since every codeword x ∈ Ca can be written as x = xa + xs with xs ∈ Cs, all codewords within the strictly affine subcode have the same non-zero affine syndrome, i.e., HsxT = sa ≠ 0 for all x ∈ Ca.
Linear subcodes and strictly affine subcodes are subsets of C, i.e., Cs ⊆s C and Ca ⊂a C, where the subscripts indicate the relation of a linear and a strictly affine subcode to the original code C, respectively. A linear subcode Cs(n, k′) is a proper subcode if k′ < k. In this case, we replace ⊆s by ⊂s. In contrast, a strictly affine subcode does not constitute a subspace of C since it does not contain the all-zero codeword
and therefore is not closed under addition. Throughout this
paper, we exclusively consider linear subcodes and strictly
affine subcodes. For brevity, we omit the term strictly and
refer to the latter simply as affine subcodes. We consider the transmission of a binary codeword x ∈ C
over a binary-input memoryless symmetric-output channel (BiMSC) with output alphabet Y [3, Ch. 4]. At the receiver, the corresponding channel output is denoted by y ∈ Yn and
can be mapped onto the bit-wise log-likelihood ratio (LLR) vector λ := (l(yi))in=1 ∈ Rn with l(yi) given by [3, Def. 4.5].

B. Belief Propagation Decoding

BP is an iterative message passing algorithm operating on the Tanner graph of a code [3]. A Tanner graph, associated with a PCM H of a linear code, is a bipartite graph, consisting of variable nodes (VNs) and check nodes (CNs). A VN vi is associated with the ith code bit, and a CN cj is associated with the jth parity check equation. An edge connects VN vi and CN cj iff Hj,i ̸= 0. III. ENSEMBLE DECODING
A. Basics of Ensemble Decoding
Ensemble decoding schemes aim at improving the performance of a single suboptimal decoding algorithm by employing multiple parallel decoding paths [16]. Fig. 1 illustrates the general structure of ensemble decoding. The LLR vector λ is provided as the input to M parallel paths. Each path, indexed by ℓ ∈ [M ], comprises a preprocessing function Pre (·|τℓ) : Rn → Rn, a decoder Decℓ : Rn → F2n, and a corresponding postprocessing function Post (·|τℓ) : Fn2 → F2n. Hereby, the pre- and postprocessing functions are based on a homomorphism τℓ : C → C, and we denote xτℓ := τℓ(x). The preprocessing function aims at reproducing the effect of τℓ(·) on the transmitted codeword x in the LLR domain. Thus, the ℓth decoder returns an estimate xˆτℓ for xτℓ and the postprocessing identifies xˆℓ such that τℓ(xˆℓ) = xˆτℓ [24].
Typically, an ensemble decoding scheme either

(i) uses M different decoders Decℓ, but omits pre- and postprocessing, i.e., τℓ = id, ℓ ∈ [M ], or
(ii) employs the same decoder across all paths, i.e., Decℓ = Dec, ℓ ∈ [M ], but uses M distinct homomorphisms τℓ with associated pairs of pre- and postprocessing functions (Pre(λ|τℓ), Post(xˆτℓ |τℓ)), ℓ ∈ [M ].

Next, we characterize several ensemble decoding schemes based on how they select distinct decoders and pre- and postprocessing functions to enhance the decoding performance.

B. Multiple Basis Belief Propagation Decoding
MBBP decoding was first proposed by Hehn et al. in [18] and later extended in [14], [22]. In MBBP, the ensemble consists of M parallel BP decoders, each instantiated with a distinct PCM Hℓ, resulting in Decℓ = BP(Hℓ) : Rn → F2n. Note that MBBP employs no pre- or post-processing functions, so it belongs to category (i).
MBBP requires M PCMs suitable for BP decoding, i.e., M matrices ideally composed of minimum-weight dual codewords. In general, for an arbitrary code, finding such matrices is an NP-complete problem [25]. However, for short- to medium-length codes, efficient algorithms for finding lowweight codewords exist, e.g., [28]. In particular, Hehn et al. investigate overcomplete PCM designs and information exchange between parallel BP decoders [14], [18]. More recently, Shen et al. showed that MBBP, in combination with a refined PCM construction, achieves strong error correction for various short non-LDPC codes [12].

C. Homomorphism-based Ensemble Decoding
With the introduction of AED of Reed–Muller and polar codes [17], homomorphism-based ensemble decoding schemes have attracted significant attention in wireless communications, e.g., [29]. Such schemes belong to category (ii), employing the same decoder in each path, i.e., Decℓ = Dec for all ℓ ∈ [M ], while selecting M distinct homomorphisms τℓ.

AED can yield substantial decoding gains compared to stand-alone decoding, provided that the effects of the selected automorphisms are not absorbed by the symmetries of the decoders [15], [17], [30]. In particular, the authors in [15] investigate AED to enhance stand-alone BP decoding of quasicyclic (QC) LDPC codes. Due to the QC construction of their PCM H, and noting that Aut(C) = Aut(C⊥) [31], certain permutation automorphisms are known a priori. It was observed that the AED path using Dec = BP(H) and τℓ is equivalent to BP decoding with τℓ−1(H), where the QC permutation τℓ−1 is permuting columns of H, while both pre- and postprocessing become the identity map id [32]. For QC LDPC codes τℓ−1(H) is typically equivalent to H up to row permutations, such that BP(H) and BP(τℓ−1(H)) yield identical decoding results preventing an ensemble gain. Hence, BP decoding based on H is equivariant with respect to these QC automorphisms. A similar effect can occur for polar codes under SC decoding [17]. Nevertheless, [15] proposes to break the decoder path equivariance for QC LDPC codes by modifying H, e.g., by removing a row, superimposing two rows, or appending a new linearly dependent row.
Recent works generalize the concept of AED. For instance, by dropping the constraint that τ must be a permutation, GAED [24] adopts the more general notion of the automorphism group prevalent in linear algebra. EED further removes the requirement of bijectivity and instead relies on mappings from the set of endomorphisms [23]
End(C) := {τ : C → C, τ linear} .
Both GAED and EED require a preprocessing to mimic the effect of the respective linear map in the LLR domain. This preprocessing causes information loss, which limits the applicability of GAED and EED to arbitrary codes.

D. Linear Subcode Ensemble Decoding

1) Principle Idea: SCED [1] belongs to ensemble decoding schemes of category (ii). In particular, SCED possibly uses proper linear subcodes Cℓ ⊂s C and their respective decoder Decℓ in the parallel decoding paths [1]. Note that for every path ℓ ∈ [M ], there exist codewords x ∈ C such that x ∈/ Cℓ, implying that the ℓth path cannot decode this codeword x. To ensure that all codewords can potentially be decoded, the ensemble of linear subcodes should constitute a LC of C. First, the minimum Hamming distance of an arbitrary subcode Cℓ ⊆ C is lower bounded by the minimum Hamming distance of C. Hence, a subcode Cℓ can eventually correct more errors than the original code C. Second, efficient subcode decoders may exist, which can be leveraged within an ensemble.

Note that for τℓ ∈ End(C), it holds that τℓ(C) ⊆s C [23]. Thus, EED operates over linear subcodes of C while achieving ensemble diversity through distinct endomorphisms. In contrast, SCED explicitly employs distinct linear subcode decoders avoiding the information loss induced by preprocessing.
2) BP-based SCED: In particular, SCED with BP decoding is investigated in [1]. In this setting, each path ℓ ∈ [M ] employs a PCM Hℓ describing a subcode Cℓ, such that Decℓ = BP(Hℓ). Given the PCM H of a binary code C, the PCM Hℓ of a subcode C(Hℓ) ⊆s C can be constructed by appending to H a matrix Mℓ ∈ Fm 2 ℓ×n whose mℓ rows are possibly linearly independent of the rows of H, i.e.,

If ∆ℓ = 0 for all ℓ ∈ [M ], then BP-based SCED reduces to MBBP, highlighting that BP-based SCED naturally generalizes MBBP. Importantly, in contrast to MBBP, SCED allows appending linearly independent rows to create diversity. Thereby, it can avoid the exhaustive search of minimumweight dual codewords, while still providing ensemble gains [1]. Even when employing an exhaustive search for lowweight dual codewords, these appended rows can yield further structural advantage for BP decoding since the search is performed over Cℓ⊥ rather than C⊥, as shown in Sec. V.
For LDPC codes, appending even a single linearly independent row, i.e., mℓ = 1, already yields ensembles with substantial decoding performance gains [1]. This aligns with [33], where Laendner et al. observed that even a single linearly dependent row can disrupt trapping sets, an effect that may also occur when appending linearly independent rows.
However, the design of SCED ensembles still faces certain challenges. For instance, some codewords, e.g., the all-zero codeword, are decodable by a larger number of decoding paths than others [1]. To overcome this imbalance and to further simplify the ensemble design while retaining the advantages of SCED, we seek an approach that operates on subcodes while guaranteeing that every codeword is recoverable by the same number of decoding paths. To this end, we propose aSCED, which extends SCED by decoding over affine subcodes.

IV. AFFINE SUBCODE ENSEMBLE DECODING
A. Idea and Some Properties of aSCED
We propose aSCED for an arbitrary linear code C. The central idea is that aSCED generalizes SCED by allowing affine subcodes, in addition to linear subcodes, each with its respective decoder. Hence, aSCED belongs to category (i) and employs M distinct decoders for subcodes Cℓ that are

either linear or affine. As in SCED, to ensure that all codewords can potentially be decoded, we require that the set {Cℓ ⊆ C : ℓ ∈ [M ]} of linear and affine subcodes satisfies (2). As shown next, the use of affine subcodes yields beneficial properties compared to SCED.

Theorem 1. Let C be a linear binary code. Then, there exists a covering satisfying (2) using M = 2 proper subcodes. This is the minimum number of proper subcodes for a covering and can only be attained if the covering consists of a linear and an affine subcode.
Proof: Since Cs(n, k − 1) is a (k − 1)-dimensional subspace of C, the set of cosets with respect to Cs(n, k − 1), contains exactly two elements due to Lagrange’s Theorem. Since the coset 0 + Cs(n, k − 1) constitutes the neutral element of the quotient space, there exists at least one element xa ∈ C \ Cs(n, k − 1) such that C/Cs(n, k − 1) = {0 + Cs(n, k − 1), xa + Cs(n, k − 1)}. Defining C1 = 0 + Cs(n, k − 1) and C2 = xa + Cs(n, k − 1), we finally obtain a covering C = C1 ∪C2 consisting of a proper linear subcode C1 and a proper affine subcode C2. Obviously, a single proper subcode cannot satisfy (2). Note that neither two proper linear subcodes nor two affine subcodes can satisfy (2). The latter cannot cover the all-zero codeword, and the former fact is proven in [26].
Theorem 1 demonstrates that the smallest number of subcodes needed to satisfy (2) is M = 2, i.e., when allowing affine subcodes, M can be reduced relative to M = 3 when only proper linear subcodes are allowed.
Remark 1. For subcodes Cs with rank deficiency ∆ > 1, , the quotient space C/ Cs(n, k′) contains 2∆ distinct cosets 0 + Cs, xa,1 + Cs, . . . , xa,2∆−1 + Cs. This is the minimum number of subcodes to achieve a covering based on Cs.
Remark 2. The lemma can be extended to non-binary codes over Fq by using a (k − 1)-dimensional linear subspace and q − 1 affine subcodes, i.e., requiring M = q.
We next show that aSCED can match ML decoding when employing sufficiently many paths.
Proposition 1. For a linear binary code C, there exists an aSCED with M = 2k paths whose ML-in-the-list decision coincides with ML decoding.
Proof: Consider the singleton linear subcode Cs = {0}. According to Remark 1, it induces M = 2k singleton affine subcodes Cℓ = xℓ + Cs, ℓ ∈ [M ]. Consider an aSCED ensemble with M paths, where the ℓth path, ℓ ∈ [M ], employs the decoder Decℓ : Rn → Cℓ. Each path outputs its unique codeword, so the candidate list is L = C.
Proposition 1 establishes that an aSCED approaches ML performance as the number of paths increases; the construction above relies on M = 2k paths, which is infeasible for practical codes. Design Principle 1. Let Hs be the PCM of a proper subcode Cs(n, k′) of C, and let ∆ := k − k′ the rank deficiency of its PCM. Following Remark 1, identify 2∆ − 1 distinct cosets xa,i + Cs =: Ca,i. The whole aSCED ensemble possibly consists of multiple aSCED batches.
Given a proper subcode Cs ⊆s C with PCM Hs, the design principle is to construct an aSCED batch, illustrated in Fig. 2 that ensures full coverage of C without degrading the performance of individual decoding paths. Note that aSCED may comprise multiple aSCED batches. In this case, the total number of paths is M = i∈[L] 2∆ℓ , where L denotes the number of aSCED batches and ∆ℓ is the rank deficiency of the ℓth batch.
When following Design Principle 1, every codeword x ∈ C is decodable by L paths. In contrast to SCED [1], where ensuring equal protection for all codewords may degrade the performance of individual decoding paths, Theorem 2 formally establishes that aSCED adhering to Design Principle 1 achieves this without compromising the average error performance of any path.
Theorem 2. Let Hs be the PCM of a proper subcode Cs(n, k′) of C and let Ca = xa + Cs be an arbitrary coset of Cs. When transmitting via a BiMSC, the error probability of the decoder BP(Hs), when averaged over all x ∈ C, is identical to the average error probability of the affine decoder BPsa (Hs).
C. All-zero Assumption
The all-zero codeword assumption accelerates Monte Carlo simulations by avoiding the need for encoding, and simplifies analytical analysis, e.g., in density evolution [3]. However, in general, this assumption is not always feasible for assessing the frame error probability (FEP) of aSCED.
In the following, we show that the all-zero assumption can remain valid for performance evaluation, and allows to reduce the number of simulated paths per aSCED batch to one. To formalize this, consider the transmission of the all-zero codeword 0, yielding the channel output y0. Define the events A := {0 ∈ L} (list success) and B := {arg maxx∈L(11×n − 2x)Tλ = 0} (frame success).

Remark 7. The LEP of an aSCED batch can be (tightly) upper-bounded by P (BP(Hs, y0) ̸= 0), which can be estimated as the LER of Monte-Carlo simulations under the allzero assumption, simulating only the linear subcode path and ignoring the 2∆ − 1 affine paths.

To evaluate the performance of different decoding schemes, we conduct Monte Carlo simulations using a binary-input additive white Gaussian noise (BI-AWGN) channel. We use the notation {aSCED, SCED, MBBP, AED}-M to denote the respective ensemble decoding technique employing M parallel paths. For every simulated SNR point, we collect at least 200 frame errors. VII. CONCLUSION
In this work, we introduce aSCED for binary linear codes. This approach extends the previously proposed SCED by incorporating affine subcodes alongside linear subcodes. To this end, we develop a BP decoder for affine subcodes. Leveraging the fact that an affine subcode is a coset of a linear subcode, we show that the decoding graph structure remains unchanged, only requiring sign flips at specific check nodes.
We further provide theoretical results on aSCED and, in particular, demonstrate that by appending linearly independent rows before search-space generation, we gain a structural advantage over MBBP ensembles. Consequently, subcodes enable the design of high-performance subcode ssPCMs.
Compared to SCED, aSCED offers four key advantages:
1) Simplified design: Fewer PCMs must be designed to achieve the same ensemble size.
2) Uniform protection: The inclusion of affine subcodes ensures that all codewords are equally protected.
3) Improved performance: aSCED consistently outperforms other ensemble methods at equal complexity.
4) Hardware efficiency: All decodings within an aSCED batch share the same graph structure, potentially enabling hardware reuse.
To demonstrate its effectiveness, we evaluate aSCED using two LDPC codes and two BCH codes, achieving significant gains over stand-alone BP and other equal-complexity ensemble decoders. Furthermore, when combining aSCED with ssPCMs, we report ML performance for the (63, 30) BCH code with BP-based decoding using only 64 decoding paths.
