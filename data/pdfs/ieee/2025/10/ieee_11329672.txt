A Reconfigurable LDPC Decoder for NAND Flash Supporting Codeword Lengths up to 4kB
Junyi Wang, Yanling Jiang, Liang Zhang, Humiao Li, Zhixiong Li, Yida Li, Longyang Lin
Southern University of Science and Technology, China
Abstract—This work presents a reconfigurable quasi-cyclic lowdensity parity-check (QC-LDPC) decoder supporting NAND flash memories with codeword lengths up to 4 kB. The proposed design provides cross-standard compatibility while enabling dynamic performance scaling to optimize throughput and power for diverse application scenarios. Implemented in 22 nm CMOS, the core occupies 1.79 mm² and achieves 20.3 Gb/s maximum throughput.
Keywords—LDPC, Reconfigurable, NAND, Dynamic adaptation
I. INTRODUCTION
Low-density parity-check (LDPC) codes, with errorcorrection performance near Shannon’s limit, are widely used in communication and storage systems. Unlike communication standards, NAND flash memories lack a unified ECC specification, and variations in page size, error rate, and other parameters across vendors require frequent decoder redesigns. As shown in Fig. 1, an ideal decoder should adapt its codeword length, rate, and structural parameters across scenarios, including mobile devices that select low-power, balanced, or high-performance modes and SSDs that adjust code rate as NAND ages.
Existing LDPC decoders for NAND flash face limitations in reconfigurability and scalability. [1] supports long codeword lengths up to 4kB but offers no reconfigurability. While [2] and [3] demonstrate excellent performance, [2] is only applicable to the 802.11n/ac standard, and [3] is limited to the 5G NR standard. [4] supports a 4-kB data block but has limited reconfigurability, lacking row/column weight adjustment and dynamic adaptation.
This work presents a reconfigurable QC-LDPC decoder supporting codeword lengths up to 4 kB, with full reconfigurability of codeword length, code rate, row weights, and column weights. This enables compatibility with any NAND flash specification below 4 kB and allows dynamic tuning of error-correction strategies for either low-power or high-throughput operation.

II. PROPOSED LDPC DECODER DESIGN A. Decoder Architecture
Fig. 2 shows the architecture of the proposed reconfigurable QC-LDPC decoder. Upon receiving the parity-check information, including the base-matrix (Hb) elements, M (rows of Hb), N (columns of Hb), and Z (sub-matrix size), the offset memory module stores the base-matrix elements and computes the row and column weights of the expanded parity-check matrix. The resulting parameters are forwarded to the variablenode-update (VNU) and check-node-update (CNU) modules.
The VNU module allocates one VNU unit for each basematrix column, which enables reconfigurability in the number of columns. Each VNU unit supports processing from 1 to 6 data points, determined by the column weight, which provides column-weight reconfigurability. Similarly, the CNU module allocates one CNU unit for each base-matrix row, with each unit supporting from 1 to 128 data points according to the row weight, which enables row-weight reconfigurability. By adding multiplexers before the VNU input ports, and setting the unused input ports to 0 when the actual number of input data is less than the maximum number of ports, the column weight can be made reconfigurable. The same principle applies to CNU.
The functions of each module of the LDPC decoder are as follows: The LLR memory module is responsible for storing the initial LLR values input from external sources, while the VNU module updates the variable nodes, and the CNU module updates the check nodes. The H-RAM module stores only nonzero elements of the parity-check matrix. The permutation network module is responsible for calculating the addresses for reading and writing the CNU to the H-RAM module. The judge module is responsible for making decisions on the decoding result after one round of parity-check matrix updates.

B. Permutation network of H-RAM The maximum supported parity-check matrix is set to
32768×1536, with each element in 16 bits. Since updating the parity-check matrix is a core decoder operation requiring continuous read and write access, a dedicated storage module H-RAM is essential. A direct implementation would require 32768×1536×16 bits of storage, incurring prohibitive area cost. To address this, the folding property of QC-LDPC codes is exploited by storing only the non-zero elements and using the base matrix to map their positions in the expanded parity-check matrix, significantly reducing memory requirements. Because the stored matrix contains only non-zero values and is reconfigurable, direct addressing for the VNU and CNU becomes non-trivial. This challenge is resolved by combining the base-matrix shift values with a bias network in each CNU, ensuring correct memory access during read/write operations. As shown in Fig. 3(b), this scheme reduces the storage requirement to 128×1536×16 bits, which is only 1/256 of the original size.
III. SIMULATION RESULTS As illustrated in Fig. 4, the sub-matrix size Z determines the number of update cycles. The number of variable nodes equals the base-matrix column count multiplied by Z, and the number of check nodes equals the base-matrix row count multiplied by Z. Repeating Z cycles ensures that all updates are completed. IV. CONCLUSION
In conclusion, a reconfigurable LDPC decoder has been developed supporting a maximum codeword size of (32768, 1536). The proposed design offers full reconfigurability across codeword length, code rate, row weight, column weight, and expansion factor. This flexibility enables compatibility with diverse NAND flash memory specifications, eliminating the high redesign workload of conventional decoders, and allows dynamic performance adaptation to a wide range of application scenarios.
