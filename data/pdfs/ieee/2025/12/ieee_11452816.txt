Design and Implementation of CODEC for CCSDS Defined LDPC code for Deep Space Communication

Rajagopalan Alias Sabarinathan P1, Pradeesh Madhavan2, Satish Sharma3, H Ramamurthy4, Lekshmi V5, Sudhakar S6
UR Rao Satellite Centre, Indian Space Research Organisation, Bengaluru,
India 1prs@ursc.gov.in, 2pradeesh@ ursc.gov.in, 3satish@ ursc.gov.in, 4hrmurthy@ ursc.gov.in,
5vlekshmi@ ursc.gov.in, 6sudhakar@ ursc.gov.in

Abstract‚ÄîThis paper presents the hardware implementation of Min-Sum based decoder for CCSDS defined LDPC code for deep space communication link. The implementation includes encoder and decoder. The encoder design involves matrix multiplication of input message vector and the Generator matrix. The decoder is a fully parallel architecture controlled by a Finite State Machine for running the algorithm iteratively. The implementation is done in Kintex Ultra scale FPGA. The encoder output is verified by checking the syndrome. The decoder is evaluated by interfacing with the S-band soft decision digital receiver wherein the data to the receiver is from designed encoder. The coding gain achieved for the decoder as a part of decoder evaluation is 5.2 dB
Keywords‚ÄîCCSDS, LDPC, channel coding gain, BER, Softdecision, Hard decision, BPSK, FPGA.
I. INTRODUCTION
LDPC(Low Density Parity Check codes)are linear ,capacity approaching ,forward error correcting codes introduced by Gallager in 1960 [1] .It is used in various communication protocols like WiFi, WiMAX, DVB-S2 etc. The CCSDS (Consultative Committee for Space Data Systems) has defined a LDPC code for the use with Space Data Link Protocol.
LDPC codes are of different types- Random codes, Quasicyclic codes, Repeat-accumulate codes, Progressive edge growth codes [2]. CCSDS has used Quasi-cyclic method of code generation, this method facilitates reduction in memory & complexity of routing in hardware implementation [15].
There are two types of decoding algorithms - soft decision and hard decision algorithm. Soft decision algorithms use multibit quantized inputs known as Log Likelihood ratio values from the demodulator and process it for deciding upon the received bit [2]. Soft-decision algorithms have better error correction capability in lieu of consuming more resources for hardware implementation with reduction in throughput as well [3]. Hard-decision algorithms depends upon bit quantized input and process it with syndrome checking as the preliminary step [15]. Its decoding performance is not satisfactory when compared to soft-decision decoding in the preview of coding gain[15]. Nevertheless, its throughput and hardware utilization are better than soft-decision algorithms [3].
The soft decision algorithms proposed for LDPC decoding in [2] are Belief propagation [7], Approximate Belief

propagation [6], Min-sum and Normalized min-sum algorithms [8].
The complexity of Belief propagation, Approximate Belief propagation in hardware implementation is intensive as it uses trigonometric computations. Wherein the Min-sum algorithm and the variants are hardware efficient as it uses Max-log approximations applied to the trigonometric calculation involved in Belief propagation algorithm [9]. Hence Min-sum algorithm is the candidate for hardware implementation of LDPC decoder.
The architectural design of LDPC decoders proposed in the literature are fully serial, partially parallel, fully parallel architectures [2].
The fully serial decoder [10] reuses the Check node and variable node multiple times in accordance with the H-matrix dimensions. This design is hardware efficient in lieu of reducing the throughput
In Partially parallel decoder architecture [9] the message update or processing is done sequentially either in Check node unit (represents the row of the H-matrix) or in Variable node unit (represents the column of the H-matrix) and not in both of the nodes (check node and variable node).Hence it is called as partially parallel decoders. The advantage of this design is that it has an advantage of resource efficiency and throughput in combination.
The fully parallel decoder architecture is a straight forward implementation of the Min-sum algorithm. The gain in this construction is, it can give the highest throughput achievable.
The spacecraft communication (uplink) requires maximum possible throughput. Hence fully parallel decoder architecture is considered for this application. However, the logic utilization is high so hardware efficient Check node unit and variable node unit implementation is necessary.
Various designs for hardware efficient decoder are proposed wherein which optimization is concentrated more towards Check node unit (CNU) implementation as it involves more logic. In [11] the second minimum (second lowest among the variable node messages) calculation is approximated by deriving the value from first minimum (lowest among the variable node messages received at check node), it reduces the hardware complexity by not implementing sorting algorithms to find two minimums in the input messages from variable nodes. II. PRELIMINARIES
LDPC code is defined as (ðä, ðá) code where ðä = 128, ðá = 64 so the code defined by CCSDS for deep space link is (128, 64) which is a rate 1ýµó2 code. Further the LDPC code defined here is a systematic, irregular, half-rate, transparent code [4]. A systematic code is a code where codeword starts with message in MSBs followed by the parity check part [15]. An irregular code is a code where in which column or row weights is not same in its H-matrix or an irregular code is a code where information bits & parity bits are intertwined [15]. A half-rate code is advantageous when compared to quarter-rate code as its increased error correcting capability will not compensate the energy loss [1]. Transparent code is a code which is independent of phase reversal of the received signal [15].

B. Overview of the Min-Sum LDPC Decoder architecture
The architecture of the LDPC decoder has the following design blocks and it is depicted in Figure 2

1. Soft data reception and formatting module(SDRFM)

2. Check node unit banks(CNUB)

3. Variable node unit banks(VNUB)

4. Message selection multiplexer(MSM)

5. Channel LLR to Check node unit router(CLCR)

6. Variable Node Unit And Check Node Unit Message

7. Exchange Module (VTCMEM)

8. Syndrome check unit (SCU)

9. Decoder controller(DC)

The SDRFM consist of offset binary to 2s complement converter as the incoming data is in offset binary. The incoming message will have marker to indicate the start of first code word block. So it has to be detected to start framing the block (which is done by code block framing)

The code block framing module combines 128 soft decision data vector (4 bits) which forms a 128- bit soft data blocks to be decoded. Finally, a valid data indication is generated and

forwarded to the decoder controller FSM to start the decoding process.
The Check node unit bank (CNUB) and Variable node unit bank (VNUB) consist of 16 nodes each to have 64 Check node units and 128 Variable node units. The CNUB receives message from MSM. The MSM selects message either from CLCR or from VTCMEM. The CLCR routes the channel LLR to check nodes based on the H-matrix configuration, VTCMEM forwards message from CNUB to VNUB and vice-versa.
The two nodes of bipartite graph are Check node unit (CNU) and Variable node unit (VNU). The CNU finds the 2nd Minimum from the received message from the VNU and forwards the 2nd minimum to node which has sent the 1st minimum and forwards the 1st minimum to all other nodes. In this architecture eight VNUs are connected to single CNU. The VNU adds all the connected CNU messages along with input channel LLR and subtracts the message received if the CNU & VNU matches in identity of the nodes (i.e. if the column & row number matches in the H-matrix).
The syndrome check node unit verifies the validity of the code word generated from the VNU hard decision output. This information is forwarded to decoder controller. The decoder controller controls the valid input message passing and valid output generation. It detects the availability of the message from the SDRFM and indicates the decoder to start decoding and waits for the decoding time to get over. It also checks for the validity of the decoded message by verifying the syndrome check unit. If the syndrome is ‚Äò0‚Äô then it finishes with the single iteration otherwise it continues for next iteration and continues for till the syndrome becomes ‚Äò0‚Äô or till the end of maximum iteration limit. This controller indicates the valid message indicator as well for the output interface to take the message and process or to serialize for further applications.

The TSM module is 2‚Äôs complement to Signed magnitude converter to ease the computation of CNU. The CNU calculates the 2 minimums among the channel LLR or updated message and converts the messages back to 2‚Äôs complement using SMT and forwards it to the VNUs. The hard decision output of the VNUs is used for checking syndrome for finding the validity of the codeword. The CN update is where channel LLR is added and fed back to the multiplexer input leaving out the check node input which matches the index of the processing variable node.

V. CONCLUSION
This paper presents the design of LDPC CODEC. The encoder is implemented with matrix multiplication and the decoder is based on Min-Sum algorithm and the H-matrix used by the decoder is defined by CCSDS for deep space communication link. The architecture is a fully-parallel implementation. The design of CNU is modified in comparison to earlier works considering the tradeoff between hardware

utilization and coding gain requirement. The testbed has been developed for performance evaluation of the LDPC decoder. The input considered for the verification is LDPC encoded PRBS-10 pattern. The coding gain achieved is 5.2 db.
