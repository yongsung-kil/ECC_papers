FPGA-Based Implementation of Single-Cycle High-Throughput LDPC Encoder for 5G New Radio
A. Sharma , G. Purohit , Member, IEEE, and A. R. Asati , Senior Member, IEEE

Abstract—This letter presents a novel architecture for a highthroughput encoder for quasi-cyclic low-density parity-check codes. This low-complexity encoder is specifically tailored for the 5th generation (5G) new radio (NR) standard. To achieve high throughput, we employ an automated approach to design customized encoders for individual base graphs using a Verilog Code Generator (VeCoGen) that we developed in MATLAB. Our proposed architecture improves efficiency by rearranging the wire indices during the hardware design language code generation stage itself, instead of performing shifting operations on the data carried by those wires on the fly. This eliminates the need for RAM (for storing base graph coefficients) and dedicated barrel shifters. The parity bits are computed in parallel, while maintaining optimal performance and gate count through effective exploitation of the base graph’s sparsity. Consequently, the encoder can be designed as a fully combinational circuit between the input and output registers, enabling the generation of the complete codeword within a single clock cycle. This results in exceptionally high throughput, optimal hardware utilization, and minimal number of XOR operations, making our approach highly effective for 5G NR applications.

I. INTRODUCTION
I N THE era of Industry 4.0, 5G plays a crucial role in enabling applications in the biomedical and automotive sectors. Key scenarios include: 1) ultrareliable low-latency communication (uRLLC); 2) massive machinetype Communication (mMTC); and 3) enhanced mobile broadband (eMBB). To support uRLLC, significant increase in data transmission rates is required. LDPC codes, first introduced by Gallager in 1963 [1], are ideal for this, offering near-Shannon-limit error correction [2] and low-complexity encoding, making them key for high-rate data transmission.
At the 2016 3GPP conference [3], LDPC codes were adopted for 5G New Radio (NR) data channels. As these are linear block codes, they are defined by their parity check matrices (PCMs) [1]. However, the size of a PCM can become quite large, depending on the length of the incoming message

To address this, fixed-dimension base graphs, of types BG1 and BG2, were finalized, and the concept of lifting size (Z) was introduced to convert these base graphs into PCMs. Unique base graphs of both types were defined [3] for each of the 51 Z values listed in Table I. Each base graph has six submatrices: a Zero matrix, an Identity matrix, and four variable submatrices (A, B, D, E), as illustrated in Fig. 1.
A base graph for a given Z is a matrix with integer values ranging from −1 to (Z−1). As explained in [3] and [9], the PCM is generated from the base graph (for a given Z) by replacing each element with a Z × Z matrix. The element ‘−1’ becomes a zero matrix, ‘0’ becomes an identity matrix, and ‘i’ transforms into an identity matrix column-shifted to the right in a circular fashion by ‘i’ positions [with i ranging from 1 to (Z-1)]. It follows that the base graph is Z2 times smaller in size than the PCM. The base graph consists of m rows and n columns, and the incoming message vector has (n-m) or Kb blocks, each containing Z bits.

Serial encoding was applied to larger Z values, while row encoding was used for the others. This improved hardware efficiency but reduced throughput for higher Z values. Liao et al. [9] implemented the architecture on an Nvidia TITAN RTX GPU, achieving throughputs of 41.8 GBPS (BG1-Z64) and 62.6 GBPS (BG1-Z96), which are similar to [4] but at a lower cost compared to ASIC implementations. Tian et al. [8] proposed a fully parallel design optimized for the maximum Z value under BG1. For smaller Z values, vacant LSB bits were padded with zeroes before being fed to the barrel shifters and XOR network, missing the opportunity to save on area and power. This results in degraded performance as Z decreases. The design, synthesized on SMIC 28-nm CMOS technology, used 64 Zbit barrel shifters and 27 clock cycles (for BG1) at 800 MHz, achieving a maximum throughput of 257.9 Gb/s. Unlike other references, [8] alone reports power consumption, with a peak dynamic power of 123.5 mW.
A common feature in all these architectures is the heavy use of barrel shifters, RAM storage for base graph coefficients, and multicycle operations to generate the output. This letter proposes a novel approach that eliminates all these components, resulting in significantly higher throughput.

IV. PROPOSED ARCHITECTURE
The proposed architecture employs custom designs tailored to each base graph. We developed Verilog Code Generator (VeCoGen) in MATLAB to produce optimized hardware design language (HDL) code for each base graph. Instead of real-time shifting via barrel shifters, VeCoGen handles shifts for all matrix multiplication operations in (7) and (8) by rearranging wire indices during HDL code generation, passing them to XOR trees (XOR blocks in Fig. 2) for modulo 2 sum. As shown in Fig. 2, this eliminates the need for barrel shifters and RAM for base graph storage. Here, Ci (i ∈ {1, 2, . . . , n}) represents codeword blocks as defined in (2). Each XOR tree is customized to process only nonzero blocks. Fig. 3 shows the count of nonzero blocks in submatrices A and D of the base graph in Fig. 1. Unlike existing designs that invariably XOR kb blocks of Z bits each (1012Z XOR operations for BG1 and 420Z for BG2) during matrix multiplication for AST and DST in (5) and (6), the proposed approach leverages the sparsity of the base graph, where nonzero blocks are significantly fewer and reduces the number of XOR operations to 265Z and 420Z for BG1 and BG2, respectively. This corresponds to a 74% reduction in XOR trees for BG1 type and 65% for BG2 type.

V. FPGA IMPLEMENTATION As the Z value increases, the number of bits entering and exiting the encoder grows. To test the functionality and find the operational frequency, routing to I/O ports becomes necessary. However, the limited number of FPGA I/O ports makes the FPGA implementation for higher Z values challenging. To address this, a new testing strategy is devised using Vivado’s IP integrator to design a testing apparatus. Tcl automation has been created in Xilinx Vivado to generate block designs for various Z values as shown in Fig. 4. For a particular base graph, VeCoGen is used to dump the Verilog code for the blocks “LDPC Encoder,” “LDPC Testing,” and “Control Unit.” Test message vectors are stored in block memory. VII. CONCLUSION
This letter presented a single-cycle, high-throughput encoder for quasi-cyclic low-density parity-check (QC-LDPC) codes in 5G NR. Automations via Tcl and MATLAB-based VeCoGen enable design generation and testing across all Z values for BG1 and BG2. Efficiency is enhanced by limiting XOR operations to nonzero blocks through custom encoder designs for different base graphs. By removing barrel shifters and RAM, the proposed single-cycle architecture surpasses state-of-the-art designs in throughput, making it ideal for 5G eMBB and uRLLC scenarios.
