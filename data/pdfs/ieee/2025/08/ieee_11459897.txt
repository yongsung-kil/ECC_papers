Research on the Application of LDPC Decoding Algorithm Based on Neural Network

Hong Zhang Noncommissioned Officer School The Army Armored Corps Academy
Jilin, China 2277307349@qq.com

Abstract—In this paper, an optimal implementation method of LDPC decoding algorithm under the 1553B bus is introduced to meet the requirements of data transmission speed and quality in in-vehicle and inter-vehicle storage, interconnection and fault diagnosis. Based on the neural network algorithm, combined with the programming language and hardware structure, the parallel structure decoding is adopted, which effectively solves the difficult problems such as the generation of the check bit of the teaching evidence, enhances the communication encryption, and reduces the false baud rate in the transmission process. The comprehensive results show that while occupying less hardware resources, the design results have a smaller false baud rate, and the LDPC code has a higher accuracy rate in the codeword ratio sum product algorithm with a longer transmission code length.
Keywords—LDPC code, sum product algorithm; neural networks, encoder
I. INTRODUCTION
With the development of equipment digitalization, the reliability and confidentiality requirements for data collection, storage, and data transmission and processing between vehicles have attracted much attention [1]. LDPC code does not rely on network decoding, and has the characteristics of low bit error rate, good encryption characteristics and simple coding, which is an important alternative scheme for equipping digital data channel coding." However, LDPC codes are long, there is a lot of data that needs to be stored in decoding and generation, and there is no quantitative standard for incremental redundancy, so the second-generation mobile communication standard is missed [2]. With the development of artificial intelligence, the combination of neural network algorithm and LDPC decoding is improved, and the parallel mode of multiple processors solves the original shortcomings, processes high peak data rate, and can adapt to the data channel transmission requirements of in-vehicle and vehicle interconnection [3].
In this paper, taking the QC-LDPC code as an example, the decoding characteristics are analyzed, the decoding algorithm is optimized by combining the network algorithm, the buffer layer and the convolutional layer are designed to improve the decoding efficiency, and the performance is verified in the 1553B bus network.

A. Encoding Module Design
In order to complete the decoding of LDPC code, in addition to configuring a multi-core operating environment, it is also necessary to build a decoding module in python language, and at the same time have hardware structure support [6]. The hardware structure selection includes graphics processing unit, storage unit and operation unit, etc., here the DSP processor is selected as the hardware structure basis, the processor can process signals in real time, which is more suitable for the needs of LDPC code in the decoding throughput rate, in order to make up for the shortcomings of the limited hardware resources, the Qt framework can provide GUI and tools, create visualization and interactive interfaces, Eigen can provide a linear number library, improve the speed of matrix operation, and the two class libraries have stability and reliability in the field of data processing and application.
Using software and class libraries, the coding scheme is designed through the programming hardware platform, and the core module of the algorithm is the sample training module. As shown in Fig. 3, the sample training is divided into two parts: the front-end and the back-end, which need to cooperate and interact with each other. The main body is implemented by the class, and in the initialization stage, the connection between the signal and the slot needs to be completed, Qt provides the communication between the slot and the signal, completes the processing of specific events, and the Qt object on the front end sends a signal and calls the slot function. Before training, import the configured training set, mainly the input and the training algorithm, the input represents the information to be transmitted by the LDPC code, and the LDPC decoding result is output after the training algorithm, and the system will perform a pre-training check after configuration. Firstly, the sum product algorithm is used to decode the fields contained in the LDPC code in the above fault types, and the variation of the false baud rate with the signal-tonoise ratio (SNR) of the simulated decoding algorithm is shown in Fig. 5. The results show that with the increase of the signal-to-noise ratio, the bit error rate of the parallel algorithm decreases gradually, and the bit error rate of the two algorithms is basically consistent. This shows that the parallel decoding environment can effectively shorten the decoding time and improve the decoding efficiency without affecting the decoding performance. In addition, in terms of the number of operations, when the number of iterations increases, the parallel algorithm can ensure that the input codeword data is processed quickly. Due to the long code length of the 12 unit fault types transmitted, there are some abnormalities in the decoding process, mainly because there is "illegal channel conversion" when the channel establishes communication with the terminal, and the terminal goes offline, and the abnormal performance is "response timeout".

Fig. 5. Comparison between ordinary decoding and sum product decoding in a parallel environment.
The ij and  code word samples in the LDPC code are used for neural network algorithm simulation and training. The simulation results are shown in Fig. 6, and the codewords and final decoding errors at the output end of the buffer layer of the neural network are very small, and the results are very good. Compared with the sum product decoding effect, this algorithm still has a good decoding accuracy for LDPC codes with long codewords, but the effective codewords themselves are longer in the fault transmission of the bus network, so this method has high application value. However, there are more codewords to participate in the training, and when the single processing exceeds 1kbit, the convolutional layer structure will become complex, and it needs to occupy a large amount of memory, in the radio platform composed of the python language design training module combined with the DSP hardware structure, the parallel structure can meet the training requirements of the neural network, and compared with the sum product algorithm, the LDPC decoding method with stronger fault ability to process the code length can complete the data splitting of the fault in the car. After adding soft verdicts at the input end of the interface, for example, the probability judgment of the codewords in the channel is performed first, and the output neuron verifies the probability information at this time, which can reduce the number of iterations and reduce the false baud rate.

V. CONCLUSION
In this paper, through the analysis of LDPC coding definitions and rules, python software programming sample training module and multi-threaded parallel structure of decoding environment are configured, combined with DSP hardware to build a wireless data transmission platform, after studying the traditional LDPC coding structure of sum product decoding calculation, combined with neural network algorithm, the linear block code neuron decoding algorithm of convolutional layer and buffer layer is built, the training rules of ij and  are completed, and the transmission results of 12 communication faults are tested based on the 1553B bus network. Finally, the comprehensive results show that while occupying less hardware resources, the design results have a smaller false baud rate, and the LDPC code has a higher accuracy in the codeword ratio and product algorithm with a long transmission code length, which can be applied to the environment with low power consumption and strong encryption, and can be used to design and realize a large-frequency and low-loss communication method for

vehicle interconnection under the control of artificial intelligence algorithm in the future.
