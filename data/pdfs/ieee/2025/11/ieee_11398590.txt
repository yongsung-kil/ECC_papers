Abstract—With the increase of satellite communication data, 
the modem in the ground application system is required to have a 
higher codec rate and communication capacity. This paper 
proposes a method for implementing an ultra-high speed variable 
block code data encoding and decoding structure, which supports 
serial frame input of correlated ultra-high speed block code data, 
completes ultra-high speed parallel encoding and decoding, and 
achieves independent and stable output of ultra-high speed data 
serial frames. The method proposed in this paper is applicable to 
FPGA based ultra-high speed variable block code data encoding 
and decoding, compatible with various block codes such as LDPC, 
RS, BCH, and multi rate encoding and decoding; Adopting a 
parallel structure processing approach to rapidly improve the 
modem throughput of satellite ground application systems; The 
software calculates the number of parallel loop modules for high-
speed data caching, drives the loading of corresponding FPGA 
bitstreams, quickly reconstructs FPGA encoding and decoding 
structures, flexibly allocates resources, and improves resource 
utilization. 
Keywords—Block Code, Ultra-high speed, Encoding and 
decoding structure, FPGA, Parallel structure 
I. INTRODUCTION  
At present, there has been extensive research and hardware 
implementation on linear block codes[10], especially on FPGA 
platforms. Meanwhile, independent algorithms such as LDPC[1], 
RS[3], and BCH[6] codes have been extensively researched and 
applied. For example, Zhang Zhe[2] et al. proposed a parallel 
decoding method for LDPC codes constructed by m-sequences, 
and Abderrazak Farchan[8] et al. proposed a generalized parallel 
cascaded block code, construction, and iterative decoding based 
on BCH and RS codes. With the increase of satellite 
communication data volume, studying multiple encoding[7] and 
decoding[9] methods one by one to improve modem throughput 
is complex and expensive. It is crucial to find a simple 
structure[4] to implement multiple encoding and decoding of 
modems[5] to improve throughput. In response to the above 
issues, this article proposes a universal ultra-high speed block 
code data encoding and decoding structure, which achieves 
parallel processing of various block code information through a 
variable block code data input method to meet the requirements 
of satellite ultra-high throughput. This structure supports high-
speed input of serial block code data frames with correlation, 
parallel processing of data through strategies such as 
decorrelation, and ultimately outputting independent high-
speed encoded and decoded data frames. Using software 
computation and scheduling, loading corresponding bit streams, 
and achieving fast FPGA reconstruction of various block code 
ultra-high bit rate encoding and decoding structures such as 
LDPC and RS. A simple and practical method has been 
implemented while ensuring error performance and reducing 
computational complexity. 
II. DESIGN OF ULTRA-HIGH SPEED VARIABLE BLOCK CODE 
DATA ENCODING AND DECODING STRUCTURE 
The structure of the ultra-high speed variable block code data 
codec is shown in Figure 1, which consists of six modules: 
indicator generation module, variable block code data decoding 
correlation module, high-speed data parallel cache loop module, 
encoding and decoding parallel acceleration module, encoding 
and decoding parallel data frame processing module, and high-
speed data serial frame output module. This structure supports 
serial input of high-speed block code data frames with 
correlation, and the length and frame header position of the code 
group data can be changed. Using data rate to calculate the 
number of parallel loop modules M for ultra-high speed data 
caching, and through software loading of bit streams, achieve 
fast reconstruction of FPGA ultra-high speed variable block 
code data encoding and decoding structure. 
Indicator generation module: This module receives variable 
block code serial frame data, calculates and generates the first 
frame header indication identifier  and buffer group control 
indication identifier  based on the frame header position flag, and 
provides support for subsequent module operation. 
Variable block code data decorrelation module: This module 
inputs code group data in units of Q bytes in one data clock cycle, 
with variable block code length and the first byte of the frame 
header can be placed at any position in the Q byte. Based on the 
indication identifier generated by the module, decorrelation the 
block code data, frame header address, and frame data address 
input by the module. Thus, the high-speed related serial frames 
are converted into parallel independent frames, providing correct 
input data and input data write enable for the high-speed data 
parallel cache loop module.

III. IMPLEMENTATION METHOD OF ULTRA-HIGH SPEED 
VARIABLE BLOCK CODE DATA ENCODING AND DECODING 
STRUCTURE  
Serial input of ultra-high block code data frames with 
correlation, By using variable block codes to decode related 
strategies, implement parallel and independent data frames 
before encoding and decoding. Encapsulate the encoding and 
decoding module, standardize the interface, and adopt M-

A. Indication identifier generation method 
Indicator generation module provides indicative markers for 
subsequent data processing. Based on the input frame header 
position flag (flg), it calculates and generates the first-frame 
header position indicator (fh_flg), as well as the buffer indicator 
Switch=i%M+1 (where i denotes the i-th frame, and M 
represents the number of parallel loop modules in the high-speed 
data buffer).

3) Parallel data frame processing module after encoding and 
decoding module 
To ensure the independence between subsequent high-speed 
data serial frame output modules and frames, the first bit of the 
encoded and decoded data frame header must always be located 
at the first bit of the Q-byte data. When the frame length after 
encoding and decoding is a non-integer multiple of Q bytes, the 
encoded and decoded data frame needs to be expanded to an 
integer multiple of Q bytes. This module adopts a zero padding 
strategy at the end of the encoded and decoded data frame, 
receives the encoded and decoded information of M-channel 
ultra-high speed parallel independent frame data, and outputs the 
zero padding information of M-channel ultra-high speed parallel 
independent frame data after encoding and decoding: Frame 
header pulse(flgc_addi), enables(vldc_addi), Data(dinc_addi). 
Zero padding for encoded and decoded frame data:   
Taking the output of Q bytes data per clock cycle in the 
encoded and decoded data frame as an example, zero padding 
calculation is performed in bytes. The frame length after 
encoding and decoding is L_code=M * Q+N, where M is the 
rounding of Q byte by L_code and N is the remainder of Q byte 
by L_code. When N=0, L_code is exactly a multiple of Q bytes 
and no zero padding operation is performed; When N!=0, 
L_code is a non-integer multiple of Q bytes, and zero padding 
needs to be performed at the end of the encoded and decoded 
data. The number of zero padding bytes is N.

memory group, Cr=M reads only the buffer data information in 
the output buffer M, thereby achieving high-speed and stable
