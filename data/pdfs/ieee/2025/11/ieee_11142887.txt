Efficient Encoding Algorithm and Architecture for 2-D Quasi-Cyclic LDPC Codes With Applications to 2-D Magnetic Recording

Karthik Bharadwaj and Shayan Srinivasa Garani
Department of Electronic Systems Engineering, Indian Institute of Science, Bengaluru 560012, India
We propose an efficient encoding algorithm and architecture for 2-D quasi-cyclic (QC) low-density parity-check (LDPC) codes. The encoding algorithm is derived based on the null space of the parity-check tensor obtained by tiling random permutation tensors satisfying certain girth constraints toward improved error correction performance. Our contributions are threefold. First, the construction of 2-D LDPC codes is generalized to accommodate random tilings of permutation tensors, providing code design flexibility over prior work based on predefined shifts. We provide the conditions for obtaining girth greater than four and six, useful for deriving the parity-check tensor of the code algorithmically. Second, based on the parity-check tensor, the generator tensor of the 2-D code is derived. We prove that the generator tensor of a 2-D code whose parity-check tensor has the same i-shifts within each block row, regardless of the j -shifts, comprises tiles of circulant tensors, useful for hardware realization. Third, three different hardware architectures that trade off hardware resources with speed and throughput are proposed. Finally, we analyze the performance of the code via simulations. The proposed 2-D codes are capable of correcting random errors and cluster errors by design. Specifically, for a 2-D LDPC code of rate 0.5 and size 50 × 100, the proposed approach with random shifts along with layered decoding achieves a 0.7 dB signal-to-noise ratio (SNR) gain for random errors at a code failure rate (CFR) of 10−5 and a 1.8 dB SNR gain for burst errors, compared to non-layered decoding with predefined shifts.
Index Terms— 2-D quasi-cyclic low-density parity-check (2-D QC-LDPC) codes, circulant block, efficient encoding, error-correcting codes (ECCs), hardware architecture, two-dimensional magnetic recording (TDMR).

I. INTRODUCTION
T HE promise of increased areal densities in multidimensional (multi-D) data storage systems, such as two-dimensional magnetic recording (TDMR) [1], [2], [3], bit-patterned media [4], and (3-D) NAND flash memory [5], has catalyzed significant advancements in error correction strategies to ensure data integrity. These systems exploit multi-D topologies, such as lateral bit arrangements in TDMR and vertically stacked layers in 3-D NAND to achieve higher storage capacities, but introduce complex error profiles, including inter-symbol interference (ISI), data-dependent noise, and spatially correlated burst erasures, which render traditional 1-D error-correcting codes (ECCs), inefficient and inadequate [6]. To mitigate these challenges, algebraic and iterative coding schemes have been developed, offering robust error correction tailored to handle multi-D data.
Algebraic codes, such as Reed–Solomon (RS) [7] and Bose– Chaudhuri–Hocquenghem (BCH) codes, provide deterministic error correction through polynomial-based frameworks, whereas iterative codes, such as low-density parity-check (LDPC) codes [8], leverage message-passing algorithms to achieve near-Shannon-limit performance [9] for both random and burst errors, making them well-suited for data storage channels. Furthermore, efficient hardware implementation of 1-D codes has already made their way into high-speed read

The burst erasure correction capability of 1-D QC-LDPC depends on the zero span of the code, i.e., a burst erasure of length l can be corrected if, for every position j in the codeword, there exists an i th row in H1-D = [hi, j ] such that for some (i, j ) = (a, b), ha,b = 1 and ha,(b+1) = ha,(b+2) = · · · = ha,(b+l−1) = 0. Now, we extend this concept to 2-D codes. Here, the codewords are a 2-D array of bits and can be viewed as a

collection of individual row vectors stacked one below another. Each row vector requires a corresponding parity-check matrix to correct errors in each of its bit positions. This results in parity-check matrices, with their number being equal to the number of row vectors in the 2-D array of bits. Since the row vectors are stacked one below another to form the complete codeword, the corresponding parity-check matrices are also stacked one below another to form a parity-check structure. The structure so obtained is a 3-D tensor. Therefore, a paritycheck tensor for a 2-D codeword is analogous to a parity-check matrix for a 1-D codeword. In the parity-check tensor, each horizontal layer acts as a parity-check equation, just like how each row in the parity-check matrix of a 1-D LDPC code represents a parity-check equation.

A. Prior Works
Matcha et al. [13] constructed native 2-D LDPC codes with 2-D burst erasure correction capability by extending the ideas in the 1-D QC-LDPC code construction. The codeword of a 2-D LDPC code is a 2-D array of bits. In a 2-D LDPC code, the codeword is a 2-D array of bits. The orthogonal arrays to the 2-D LDPC code are also a 2-D array of bits. These orthogonal arrays are stacked one below the other to form a 3-D parity-check tensor, which represents the paritycheck equations. Each layer of this parity-check tensor acts as a single parity-check equation, as described earlier in this section.
The parity-check tensor H2-D for a 2-D LDPC code is obtained by stacking c × h × w 3-D permutation tensors along the i-, j-, and k-axes, as shown in Fig. 2.

It has been established that the 2-D LDPC code constructed using the shifts given in (6) has a burst erasure correction capability of at least p × p. In addition, the following observation has been made regarding the girth of the code. If the dot product between any two layers l1 and l2 of H2-D is less than 2, it implies that there exists no short cycle of length 4 between layers l1 and l2. By proving this condition for all possible combinations of layers l1 and l2 in H2-D, the code is said to have a girth greater than 4. It is established that the code is free of cycles of length 4 if (h − 1)(c/ p − 1) < p and (w − 1)(c/ p − 1) < p.
The constructions outlined above determine the shifts of the permutation tensors based on their positions within the paritycheck tensor, resulting in a fixed structure for the parity-check tensor. Additionally, the codes constructed in [13] and [16] lack a specified girth condition for achieving a code of a certain girth, i.e., free of cycles of length 4 and 6. This necessitates altering the choice of permutation tensor shifts to a generic case. Likewise, in [14], the code is shown to be free of cycles of length 4, but to achieve a larger girth, the shifts must be flexibly chosen driven by certain conditions, algorithmically.

B. Random Tensor Tilings and Girth Properties
We propose a method to construct a parity-check tensor by tiling permutation tensors, each created by applying random shifts to the identity tensor I3-D along the i-, j-, and k-axes. This approach ensures that the resulting code has no cycles of length 4 while allowing flexibility in arranging the permutation tensors to form the parity-check tensor. Additional conditions, discussed later, can also be applied to eliminate cycles of

length 6, useful in practice. The 2-D codes, constructed with random shifts, significantly improve decoding performance. However, deriving the generator tensor using the property HGT = O results in a non-circulant structure, which is a tradeoff for the enhanced performance. This is detailed further in Section III.
We begin by defining key terms used throughout this article. The parity-check tensor H2-D (shown in Fig. 2) consists of c × h ×w permutation tensor blocks, each of size p × p × p. These blocks are indexed by i ∈ {1, . . . , c}, j ∈ {1, . . . , h}, and k ∈ {1, . . . , w}, corresponding to positions along the blocklevel i-, j-, and k-directions, respectively. A block at position (i, j, k) occupies bit indices i′ ∈ {(i − 1) p + 1, . . . , i p }, j′ ∈ {( j −1) p+1, . . . , j p }, and k′ ∈ {(k −1) p+1, . . . , kp }, where i′, j′, and k′ denote the bit-level indices within the tensor. Within each block, the placement of 1s is determined by shifts: the bit-level i-shift, denoted b(i, j, k) ∈ {1, . . . , p}, determines the offset along the bit-level i-axis such that a 1 appears in layer i′ = (i − 1) p + b(i, j, k); the bit-level j-shift, denoted a(i, j, k) ∈ {1, . . . , p}, determines the cyclic shift along the bit-level j-axis; and the bit-level k-shift is fixed by the i- and j-shifts as specified in (5).
A block row is a row of permutation tensors sharing the same block indices (i, k), spanning j ∈ {1, . . . , h}, and a block column is a column of permutation tensors sharing the same block indices ( j, k), spanning i ∈ {1, . . . , c}. A row bit-array is a layer of p × p bits local to a permutation tensor block at position (i, j, k), consisting of bits sharing the same bit-level index i′ ∈ {(i − 1) p + 1, . . . , i p }. A column bit-array is a layer of p × p bits in a permutation tensor block at position (i, j, k), consisting of bits sharing the same bit-level index j′ ∈ {( j − 1) p + 1, . . . , j p }.
A row bit-page, equivalent to a horizontal slice, is a layer of bits in the parity-check tensor consisting of bits sharing the same bit-level index i′ across all blocks (refer to Fig. 4(a) for a visual illustration), and a column bit-page, equivalent to a lateral slice, is a layer of bits consisting of bits sharing the same bit-level index j′ across all blocks (refer to Fig. 4(b) for a visual illustration). These row bit-pages and column bit-pages are rectangular slices of the parity-check tensor perpendicular to the i- and j-axes, respectively. Prior to stating theorems on girth-4 and girth-6 free construction conditions, we clarify when vertical and horizontal connections are formed that can lead to cycles.
Consider the parity-check tensor H2-D of dimensions 8 × 8 × 4, with parameters p = 4, c = h = 2, and w = 1, as depicted in Fig. 5. This tensor is visualized as four permutation blocks, slightly separated for clarity, where each 4 × 4 portion represents a permutation block in the paritycheck tensor. Within these blocks, red blocks indicate bits with value 1 and yellow blocks denote bits with value 0 at their respective positions. Green solid lines highlight vertical connections, while blue solid lines represent horizontal connections in the equivalent H1-D matrix obtained after rastering, both of which can contribute to cycle formation; notably, vertical connections form only when the j-shifts of permutation blocks within a given block column are identical, and not all possible connections are shown to avoid cluttering the figure.
After rastering to obtain H1-D, it can be observed that horizontal connections are always formed without any dependence on the shifts of permutation tensors in a given row block, as the rastering process aligns bits across rows. However, for vertical connections to form, the j-shifts of permutation blocks in a given block column must be the same, as this alignment is necessary for bits to connect across columns in the rastered structure.
Consequently, in the theorems, we primarily focus on avoiding these vertical connections to prevent cycles of length 4 and 6, as controlling the j-shifts provides a mechanism to break such cycles, while horizontal connections are inherently formed regardless of shift constraints.

C. Reiger Bound for 2-D Burst Erasure Correction

The Reiger bound for an (n, k) linear block code (1-D) is given by 2l ≤ n − k, where l is the burst error-correcting ability of the code and n and k are the block and message lengths, respectively. In the following theorem, we state a similar bound for the 2-D burst error correction ability of a 2-D code. A 2-D code is defined by a parity-check tensor, which will be described in Section III-B, consisting of permutation blocks with circulant size p, organized into t block columns, with the tensor having rank r . Here, p represents the size of each permutation block, t indicates the number of block columns in the reformed tensor structure and t = wh, and r corresponds to the rank of the reformed tensor. For such a code, the message size is given by p × (t p − r ), and the codeword size is p×t p. The detailed formulation of the paritycheck tensor, including the precise definitions and derivations of these parameters and dimensions, will be provided in Section III-B.

B. Central Ideas Behind 2-D Encoding
Due to the 3-D parity-check tensor structure of the code, efficient encoding of 2-D LDPC codes requires a fundamentally different look than what is done for 1-D codes. To begin with, we reform the structure of the parity-check tensor as shown in Fig. 8, and this reformation of the parity-check tensor is consistently followed throughout this article. Specifically, the parity-check tensor H2-D, consisting of c × h × w permutation tensor blocks each of size p × p × p, is transformed into a new tensor H2′ -D by rolling it out block column-wise. A block column, indexed by ( j, k) for j ∈ {1, . . . , h} and k ∈ {1, . . . , w}, contains blocks spanning all i ∈ {1, . . . , c}. The reformation process reindexes these block columns into a single dimension t, where t = wh, effectively flattening the h × w block columns into a sequence of t block columns. Each block column ( j, k) is mapped to a new index t′ = (k − 1)h + j, for t′ ∈ {1, . . . , t}, following a column-wise order that processes all j values for each k. This reorganization results in a tensor of dimensions cp × t p × p, where each new block column at index t′ contains c permutation tensor blocks along the i-direction, each of size p × p × p.

IV. HARDWARE ARCHITECTURE FOR 2-D QC-LDPC ENCODING
The encoding method for the full-rank case is the same as the one for deriving the G sub-tensor in the rank-deficient case. To avoid repetition, we will focus on proposing encoding architectures specifically for the rank-deficient case since the full-rank case can be handled similar to how we implement the G sub-tensor part. In Sections IV-A–IV-F, we discuss architectures for codes with a circulant structure throughout, meaning codes with parity-check tensors having the same i-shifts within each block row, regardless of j-shifts. Section IV-G covers the requirements for codes with i-shifts that vary within each block row.
The generator tensor Gq∗c can be interpreted as a composition of two sub-tensors as in (15). Therefore, an encoder corresponding to G∗qc can be implemented with two subencoders. The first sub-encoder corresponds to sub-tensor G and the second sub-encoder corresponds to the sub-tensor Q. To facilitate this approach of encoding, the 2-D data/message array A is divided into two sections A(G) and A(Q). A(G) section consists of the first (t −l) p columns of the 2-D data array and A(Q) section consists of the remaining columns of the array. V(G) is the codeword array corresponding to A(G), generated by the G sub-tensor. Similarly, V(Q) is the codeword array corresponding to A(Q), generated by the sub-tensor Q. The sum of V(G) and V(Q) (modulo-2) gives V, which is the 2-D codeword array corresponding to the input 2-D data array A.

V. 2-D QC-LDPC DECODING ALGORITHMS AND SIMULATION RESULTS
In this section, we present the decoding algorithm for 2-D LDPC codes and evaluate its performance through simulations. The sum-product algorithm (SPA) delivers optimal decoding performance but is computationally demanding due to intricate logarithmic operations at the check node unit (CNU), rendering it impractical for hardware implementation. In contrast, the min-sum algorithm (MSA) simplifies the SPA by substituting complex CNU operations with comparatorbased computations, making it more suitable for hardware applications.
Parallel processing of block rows and block columns requires substantial hardware resources, whereas serial processing of these blocks leads to reduced throughput. Our primary interest lies in the layered MSA (LMSA), which delivers superior decoding performance compared to the NLMSA algorithm [19], despite its lower throughput. The NLMSA algorithm, by processing block rows in parallel, prioritizes higher throughput, making it suitable for speed-critical applications. Furthermore, the random construction proposed in this article, when combined with the LMSA in Section V-C, enhances performance, achieving better decoding outcomes.
A. Parity-Check Tensor and Equivalent Matrix Representation for 2-D LDPC Codes
As outlined in [14], a 2-D LDPC code is characterized by a parity-check tensor H2-D, with dimensions cp × hp × wp, created by combining c × h × w permutation tensors of size p × p × p along three directions with defined shifts. Each horizontal slice of H2-D forms a parity-check equation perpendicular to the code space. The code, spanning hp × wp, translates to a Tanner graph with cp check nodes and hp × wp variable nodes. B. Non-Layered MSA
The NLMSA for decoding LDPC codes, as described in [14], operates on a Tanner graph constructed from a parity-check matrix H, comprising m check nodes (rows) and n variable nodes (columns), with edges corresponding to 1s in H. Initially, variable nodes receive log-likelihood ratios (LLRs) as soft information and forward these to connected check nodes. Each check node calculates the minimum absolute value of messages received from all but one connected variable node and sends this value to that node, satisfying the modulo2 sum. Subsequently, each variable node computes the sum of messages from all but one connected check node, adds its initial LLR, and passes the result to that check node. For hard decisions, variable nodes sum all incoming check node messages with their initial LLR; a positive sum results in a 0 and a negative sum results in a 1, forming a binary string c. The syndrome is computed as s = HcT . If the syndrome is zero, decoding moves to the next frame; otherwise, iterations continue from the check node update until the code is corrected or a maximum iteration limit is reached. In this article, we focus on the layered MSA, which processes check nodes in a sequential, layer-by-layer manner, and we will compare its performance with the non-layered MSA to evaluate their decoding efficiency.
C. Layered MSA
The LMSA for decoding LDPC codes operates on a Tanner graph derived from a parity-check matrix H, which consists of m check nodes (rows) and n variable nodes (columns). Edges in the Tanner graph connect check nodes to variable nodes where H has a “1.” In contrast to the NLMSA, the LMSA organizes check nodes into layers, processing them sequentially to update variable node messages within each layer, enhancing decoding efficiency. The algorithm proceeds as follows.
Step 1 (Initialization): Variable nodes receive soft information in the form of LLRs from the channel. These LLRs are stored as initial a posteriori messages for each variable node Vj , denoted as L(Vj ), and are used to initialize messages sent to connected check nodes.
Step 2 (Check Node Update Within a Layer): Check nodes are grouped into layers, where each layer consists of a subset of check nodes (e.g., a block row of H). For a check node Ci in the current layer and a connected variable node Vj , the algorithm computes the extrinsic message to Vj . This message is the minimum absolute value of the incoming messages from all other variable nodes connected to Ci (i.e., {Vk | k ̸= j}), adjusted to satisfy the modulo-2 sum constraint. The message is sent to Vj .

Step 3 (Variable Node Update Within a Layer): For each variable node Vj connected to a check node Ci in the current layer, the a posteriori message L(Vj ) is updated. This update involves subtracting the previous check-to-variable message from Ci to Vj , adding the new check-to-variable message computed in Step 2, and retaining the initial LLR. The updated L(Vj ) is used to compute new variable-to-check messages for the next layer, calculated as L(Vj ) minus the check-to-variable message for the target check node.
Step 4 (Layer Iteration): The algorithm processes all layers sequentially, repeating Steps 2 and 3 for each layer. After processing all layers, one full iteration is complete. The updated a posteriori messages L(Vj ) reflect contributions from all check nodes.
Step 5 (Hard Decision and Syndrome Check): For each variable node Vj , the a posteriori message L(Vj ) is used to make a hard decision. If L(Vj ) ≥ 0, the decision is 0; otherwise, it is 1. This forms a binary codeword array c. The syndrome is computed as s = HcT . If s = 0, the codeword array is valid, and decoding proceeds to the next frame. Otherwise, the algorithm returns to Step 2 for the next iteration, starting from the first layer, until the syndrome is satisfied or a predefined maximum number of iterations is reached.
This layered approach typically converges faster than the NLMSA due to more frequent updates of variable node messages, and its performance is evaluated in comparison with the NLMSA in Section V-D.

For burst erasure correction, we introduced a 25 × 25 burst error at a random position, along with Gaussian random errors, to mimic real-world challenges in data storage systems. This makes the 2-D LDPC-RTT code more suitable for applications like magnetic storage, where burst errors are common, supporting better data recovery and system performance.
VI. CONCLUSION
Native 2-D LDPC codes are well-suited for 2-D storage channels because they can effectively correct 2-D burst erasures along with providing the needed error correction against random errors. This makes them a good choice for improving data reliability in such systems. We introduced a new construction for a 2-D LDPC code with a size of hp ×wp based on random tilings. This construction ensures no short cycles of length 4 by carefully selecting code parameters and achieves a burst erasure correction capability of at least p × p, abiding by the Reiger bound. Furthermore, we presented an encoding method that works for any 2-D QC-LDPC code, addressing both full-rank and rank-deficient cases, generalizing the codec design for 1-D LDPC codes widely used in practice. We observed that codes with the same i-shifts within each block row, regardless of j-shifts, produce a generator tensor with a circulant structure, whereas those with i-shifts that vary within each block row lack this circulant property.
Additionally, we proposed three different hardware architectures for encoding 2-D QC-LDPC codes. We analyzed their resource usage, encoding speed, and throughput for analyzing the tradeoffs in the design choice. Through simulations, we benchmark the performance improvements of using a layered decoding algorithm compared to the nonlayered one, for decoding 2-D LDPC codes in the presence of random and burst erasures, highlighting its practical value.
