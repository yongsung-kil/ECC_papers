Abstract—A low-complexity QC-LDPC codec for high-speed 50G-PON is implemented on FPGA-based OLT and ONU prototypes. The proposed QC-LDPC achieves 50 Gbps performance with reduced hardware resource utilization. An optical gain of about 8.7 dB and error-free 40 Gbps transmission are demonstrate through real-time testing.
Keywords—50G-PON, LDPC (17280,14592), Normalized Min Sum, Forward Error Correction
I. INTRODUCTION
PON is a representative access network technology that provides broadband services using passive optical components, typically consisting of one OLT and multiple ONUs. Recently, the ITU-T G.9804 working group has standardized single-wavelength 50G-PON as a nextgeneration high-speed PON [1]. Since PON transmits optical signals through passive components, bit errors can occur in the receiving section depending on the optical power level. To address this, forward error correction (FEC) is employed as shown in Fig.1. Legacy systems such as E-PON and G-PON use RS(255,239) coding to achieve an output BER of 10-12 from an input BER of 10-4. Similarly, XGS-PON and 10GEPON utilize RS(248,216) and RS(255,223) codes, respectively, to correct an input BER of 10-3 down to output BER of 10-12. In contrast, beyond 10G-PON systems such as 25G-PON and 50G-PON have adopted low-density parity check (LDPC) coding to achieve of approximately 9 dB using hard-decision decoding and up to 10.3 dB using soft-decision decoding [2].
In 50G-PON, LDPC (17280, 14592) codes was adopted as the FEC scheme, utilizing a parity check matrix (H) of size 3072 x 17664 bits for parity generation and error correction as shown in Fig.2. The H matrix consists of a 12 x 69 array of 256 x 256 submatrices. Each submatrix exhibits a sparse structure, implemented either as a row-wise cyclic shift of an identity matrix denoted by values ≥0 or zero matrix denoted
by -1. The matrix can also be represented as a bipartite factor

graph composed of variable nodes (VNs) and check nodes (CNs), connected via edges [3-4].
In the LDPC encoder, codewords of 17,280 bits are generated by appending 2,688 parity bits computed using the H matrix to 14,592 information bits, followed by the puncturing of 384 bits. LDPC decoders perform probabilistic error correction through iterative message passing, based on the H matrix. Compared to RS FEC, LDPC codes have a more complex structure, which directly impacts chip area and power consumption. Consequently, the large size of the H matrix increases the computational complexity, posing challenges for hardware implementation in 50G-PON systems. Therefore, efficient algorithm selection and hardware architecture design are essential for enabling realtime operation of LDPC codec.
In this paper, we evaluate the BER performance in realtime using FPGA-based 50G-PON OLT and ONU, which incorporate a normalized min-sum (NMS)-based quasi-cyclic (QC) LDPC encoder and decoder. We also achieve an optical gain of approximately 8.7 dB through iterative decoding with reduced hardware resource utilization.
II. IMPLEMENTATION OF LOW-COMPLEXITY QC-LDPC
To implement the common transmission convergence (ComTC) function for 50G-PON on a single FPGA, a 128-bit data bus and a clock frequency of 388.8 MHz are typically used, taking into account timing constraints and resource utilization. Consequently, the LDPC encoder/decoder for 50G-PON processes one codeword every 135 clock cycles. After decoding, the results are stored in the corresponding output buffers and transmitted within 21 clock cycles. This input scheduling operates cyclically, allowing the decoder to process multiple codewords concurrently. The parallel structure enables the QC-LDPC decoder to complete codeword processing within 405 clock cycles, allowing iterative decoding to be performed within this time window. As a result, the proposed architecture maximizes throughput by efficiently overlapping data reception and decoding operations.
Fig.3(b) shows the internal structure of the QC-LDPC decoding submodule, which operates using a 5-stage pipeline architecture. It consists of three main components: Blockwise Cyclic Shift Registers (CCVMs), Variable Node Units (VNUs), and Check Node Units (CNUs), interconnected through fixed wiring networks. The submodule performs matrix computations over a 3,072 × 552-bit space within 5 clock cycles. The VNUs and CNUs are designed with 3-stage and 2-stage pipeline architectures, respectively. The decoding process begins by loading the received codeword into the CCVMs, which cyclically store log-likelihood ratio (LLR) values across parallel registers. These registers enable efficient data shifting and structured access to the VNUs. Each VNU processes incoming messages from the CCVMs and performs variable node computations while also calculating the channel LLRs internally to reduce memory access and improve hardware efficiency. The computed messages are then transferred to the CNUs, which execute parity-check operations and return the results to the VNUs. This messagepassing between VNUs and CNUs is repeated iteratively for LDPC decoding. The use of fixed interconnects between components eliminates the need for complex routing logic and contributes to faster processing. The pipelined architecture of the decoding submodule provides high decoding efficiency and scalability, making it well-suited for real-time 50G-PON

applications. Fig.3(c) illustrates the structured processing architecture of the QC-LDPC decoder, based on the H matrix, which is divided into 32 phases to enable high-throughput parallel decoding. Each phase corresponds to a specific vertical slice of the matrix and is mapped to a set of VNUs and CNUs that operate concurrently during the decoding process. The matrix has a total size of 17,664 columns and 3,072 rows, corresponding to the number of variable and check nodes, respectively. The decoder instantiates T = 552 parallel VNUs, each responsible for a subset of variable nodes, and 3,072 CNUs, one for each row of the H matrix. Given that the H matrix consists of 69 columns, the decoder requires 36 clock cycles to process one complete iteration. This phased and parallel architecture enables efficient scheduling and computation, supporting real-time operation in high-speed 50G-PON systems.
During decoding, each phase activates a specific subset of connections between the VNUs and CNUs, as determined by the non-zero entries in the parity-check matrix (H). These connections are represented by the colored vertical lines, which illustrate how LLR messages are exchanged between VNUs and CNUs. The phased scheduling ensures that no two active connections conflict, enabling conflict-free execution and continuous data flow without stalling. This phased operation not only reduces routing congestion and hardware complexity but also enables efficient pipelined message updates across all 32 phases in parallel.

IV. CONCLUSION
We have successfully demonstrated and verified a 50GPON system integrated with a low-complexity QC-LDPC using FPGA-based OLT and ONU prototypes. The proposed QC-LDPC achieves an optical gain of approximately 8.7 dB at a received power level of -27 dBm, enabling reliable data transmission. In addition, we confirm error-free operation without packet loss at a service rate of 40 Gbps. The results also show that the ComTC for both 50G-PON OLT and ONU can be implemented using relatively low-cost Xilinx Kintex series FPGAs. This indicates that ASIC development is feasible with low hardware resource usage and power consumption. Furthermore, by utilizing a 64-bit data bus and a clock frequency of 777.6 MHz, the system can support up to 22 iterations of LDPC decoding while maintaining efficient resource utilization.
