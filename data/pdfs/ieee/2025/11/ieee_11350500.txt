Abstract—With the increasing throughput in optical fiber
communication networks, Forward Error Correction (FEC)
design plays a crucial role in countering channel noise and
interference. To improve the BER performance, we construct a
class of 28%-overhead HC-GLDPC-CCs with the BCH codes
as component code. In this paper, we investigate the effects of
code length, the number of iterations, and the proportion of
GC nodes on GLDPC codes with software implementation, and
emulate the error floor performance with FPGA platform. As
shown in the FPGA simulation result, the error floor of the
concatenated code with 24 iterations is below 10-15 for an input
BER of 4.51e-2, and the net coding gain(NCG) is 12.28dB.

I. INTRODUCTION
With
the
increasing
throughput
in
optical
fiber
communication networks, FEC design plays a crucial role in
countering
channel
noise
and
interference.
Recently,
generalized low density parity check (GLDPC) codes have
been widely researched for their excellent waterfall and error
floor region performances with low code rate[1-3]. By
replacing a small proportion of single parity check constraint
(SPC) nodes in standard LDPC with generalized constraint
(GC) nodes[4,7] to improve the waterfall region performance,
at the cost of higher FEC code rate, while the convolutional
structure can bring additional performance gain, the hybrid
check generalized LDPC convolutional codes (HC-GLDPC-
CCs) seem to be a good solution for FEC design in high
speed optical fiber communications.
In this paper, we design a class of 28%-overhead HC-
GLDPC-CCs with the Bose-Chaudhuri-Hocquenghem (BCH)
codes as the component code, and the selection of BCH
codes is influenced by the proportion of GC node. Through
software emulations, we investigate the effects of code length,
the number of iterations, and the proportion of GC nodes on
the decoding performance of these HC-GLDPC-CCs. In
addition, we investigate the ultra low error floor performance
of a concatenated code with 30% overhead implemented on a
large FPGA platform. In this configuration, HC-GLDPC-CC
serves as the inner code, while BCH(8000,7896) functions as
the outer code. The FEC algorithm we proposed offers a
pathway to enhance transmission performance in next-
generation
coherent
optical
communication
backbone
scenarios.
This paper is organized as follows:in Section II, the basic
principles of HC-GLDPC-CC encoding scheme is firstly
introduced. In Section III, we continue to introduce the
principles of HC-GLDPC-CC decoding scheme. Section IV
illustrates the performance of HC-GLDPC-CC and the
concatenated code. Finally, we make a brief conclusion on
the key point of the paper in Section V.
II. THE ENCODING STRUCTURE OF HC-GLDPC-CC
The parity-check matrix of a periodically time-varying
HC-GLDPC-CC can be derived from the parity-check matrix
of a BCH code and a LDPC block code. Firstly, the parity
check matrix of LDPC-CCs can be obtained through a
matrix-based unwrapping procedure [5]. And then, in this
matrix, “1” in the first few rows of every 32 rows is replaced
by a column in the BCH parity matrix, and “0” is replaced by
an all-zero column vector. Replacing the first row of each 32
rows in the matrix, we can obtain the 1/32 check matrix of
HC-GLDPC-CC. In the same way, we can obtain the check
matrices of HC-GLDPC-CC when the proportions of GC
node range from 2/32 to 5/32. In summary, when the GC
node proportion is k/32, (k = 1, 2, ..., 5), the first k rows of
each 32 rows of the matrix are replaced, respectively. We
construct a class of 28%-overhead GLDPC-CCs from a
regular (3, 15) QC-LDPC block code with z×z circulant
permutation matrix (CPM), where z = 1024,1536 and 2304,
and a class of BCH codes with error correcting capability t =
1,
which
are
respectively
BCH(15,11),
BCH(30,25),
BCH(45,39), BCH(60,53), and BCH(75,67), corresponding
to k = 1, 2, ..., 5, while the overhead remains unchanged.
Fig. 1. The Tanner graph of the 1/32 HC-GLDPC-CCs.
The linear encoding algorithm described in [5] is utilized,
which is composed of four modules: GC encoding, payload
data cyclic shift, data splicing, and SPC encoding.
A. GC Encoding
For GC nodes, the encoding process is carried out by
multiplying the input data with the expansion matrix, which
can be implemented by first using shift registers and then
multiplying with the BCH parity check matrix.

B. Data Shift and Spicing
These codes are non-systematic, where at some positions
of the codeword, there is a situation where the generalized
check
results
and
information
data
are
spliced.
The
information data needs to be shifted first according to the
cyclic shift value of the current layer, and then the
generalized check is inserted into these data.
C. SPC Encoding
For SPC nodes, the encoding process is accomplished by
multiplying the input data with the circulant permutation
matrices, and this can be implemented through shift registers.
Finally, the input payload, SPC and GC check bits are
concatenated into a legal HC-GLDPC-CC codeword.
III. THE DECODING PROCESS OF HC-GLDPC-CC
In this section, we show the decoding algorithm for
GLDPC-CC. The layered decoding algorithm in [6] is used
in the decoding implementation. The decoder consists of four
modules: relative cyclic shifter, the variable node updating,
the check node updating and the posterior probability
updating. The shifter places the variable nodes from a check
equation in the same position. In the check node updating,
different check nodes adopt different update algorithms [7].
SPC nodes adopt the normalized min-sum(NMS) algorithm,
and GC nodes apply the Chase algorithm. It should be noted
that, the LLR fed to the Chase decoder needs to consist of the
first k rows in each 32 rows of the external information to
form a legal BCH codeword.

A. Chase Decoding Algorithm
The algebraic decoder such as Chase algorithm may be
commonly applied in a low-complexity, soft-input/soft-
output(SISO) errors-and-erasures decoder[8]. The Chase
decoder is typically constituted by four modules: a test
sequences(TSs) generator, a BCH hard decoder, a candidate
codeword selector, and an external information calculator.
The process can be summarized as follows, firstly, selecting
p bits with smallest LLR absolute values as least reliable bits
(LRBs), the test patterns(TPs) containing all combinations of
GF(2) symbols is then constructed at the LRB position, the
test sequences are the results adding modulo-2 the TPs to the
hard decision sequence from the ܳ݊݉. Secondly, calculate
the syndrome of all TSs and flip the error position according
to the syndrome. Then, select two TSs with the smallest
inner products as the optimal codeword and the competitive
codeword, after the codeword deduplication. Eventually, the
extrinsic information is calculated based on the reserved
codewords.

IV. THE SIMULATION RESULTS
In this section, we investigate the effects of code length,
the number of iterations, and the proportion of GC nodes on
GLDPC codes with software implementation, and emulate
the error floor performance with FPGA platform. In order to
decrease the error floor down to 10-15, we add a 32-lane
convolutional
interleaver
and
a
simple
outer
code
BCH(8000,7896), which is capable of correcting 8 or fewer
random errors. The implementation employs 50 pieces of
Xilinx ultrascale-XCVU125 FPGA clocked at 150 MHz,
whose throughput is 24Gbps and 50Gbps for 24 iterations
and 15 iterations respectively.
A. Performance with the software platform
The HC-GLDPC-CC performances with software emula-
tion platform are plotted in Fig. 2 and 3, where the
proportion of GC nodes ranges from 2/32 to 5/32, while the
number of iterations is fixed to 24, and the size of CPM is
fixed to 1024 in Fig. 2; the number of iterations is 16 and 24,
z = 1024, 1536, and 2304, and the proportion of GC nodes is
fixed to 4/32 in Fig. 3.

As shown in Fig. 2, when the proportion of GC nodes
ranges from 2/32 to 4/32, the corresponding output BER
performance is gradually improved, which is 5×10-3, 2×10-4
and 1×10-6 respectively for an input BER of 4.70e-2. The

that of
3/32,
so
4/32
is
the

proportion
for
performance. The remaining parameters of 4/32 are traversed
in Fig. 3. From the emulation results, the output BER
performance is gradually improved when z increases from
1024 to 2304. The corresponding output BER is respectively
1×10-5, 4×10-6, and 3×10-7 for an input BER of 4.65e-2 with
16 iterations. In conclusion, as the size of CPM, and the
number of iterations increasing, the corresponding output
BER performance gradually enhances. Nevertheless, the
performance initially improves and subsequently declines
with the proportion of GC nodes increasing. When the
generalized node ratio is less than 4/32, the error correction
capability of BCH codewords surpasses the average number
of bit errors within the codeword. As this ratio increases, the
more variable nodes assisting SPC node decoding, resulting
in corresponding performance improvements. However,
when the ratio exceeds 4/32, the capability becomes inferior
to the average error bits. In this regime, a higher ratio
amplifies error propagation effects, ultimately leading to
performance degradation. And increasing the number of
iterations improves performance more significantly than
increasing the size of CPM z. Considering the performance
and implementation complexity constraints comprehensively,
we choose the parameter scheme of 4/32, z = 1024, and 24
iterations.

V. CONCLUSIONS
In this paper, we construct a class of HC-GLDPC-CCs by
combining hybrid check and convolutional structure, while
keeping the 28% overhead unchanged, and investigated the
performance with software and FPGA implementation.
Based on the software emulation, and the optimal parameter
is 4/32, z = 1024, and 24 iterations. According to the FPGA
emulation, setting the fixed z = 1024 and the proportion is
4/32, the error floor of 24 iterations is 3×10-12 for an input
BER
of
4.51e-2,
and
concatenated
an
outer
code
BCH(8000,7896) can effectively decrease the error floor to
below 10-15. The NCG of the concatenated code is 12.28dB,
which indicates that it achieves a 0.55 dB gain over the HC-
LDPC in [7] and the gap to Shannon limit is 0.93dB. In
summary, the HC-GLDPC-CC offers a pathway to enhanced
transmission performance in next-generation coherent optical
communication backbone scenarios.
