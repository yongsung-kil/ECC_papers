AbstractÔÇöWith the rapid development of deep learning research and physical layer communication applications, deep learning-based channel coding has gradually become a research hotspot. In this paper, we propose a LDPC decoding network based on neural belief propagation (NBP) decoding. By introducing weights sharing mechanism, the weights vary across different iterations. The number of parameters is reduced, which significantly lowers memory requirements. In addition, we optimize the loss function to better train the model, achieving a lower bit error rate (BER) performance. Experimental results show that the proposed decoder yields significant performance improvements with respect to NBP, and significantly reducing the number of learnable weights.
KeywordsÔÇödeep learning, belief propagation decoding, LDPC decoding
I. INTRODUCTION
LDPC code was first introduced by Gallager in 1962 [1]. Due to wide applicability and performance approaching the Shannon limit, LDPC code has been a main channel coding and one of the research hotspots in the field of channel coding and decoding [2]. The belief propagation (BP) algorithm and min-sum (MS) algorithm are the two main methods for LDPC decoding. The BP algorithm performs well but has high computational complexity when updating check nodes, making it difficult to implement. In contrast, the MS algorithm simplifies the check nodes updating, sacrificing the decoding performance. The offset min-sum (OMS) and normalized min-sum (NMS) offer good decoding performance with low complexity [3].
In recent years, deep learning has been applied to channel decoding and has gradually become a research hotspot. T. Gruber and S. Cammerer studied the fully connected neural network decoder for LDPC codes [4], which learns a large number of codewords to ensure good decoder performance. Channel decoding with deep learning is a binary classification problem, where the number of information bits k determines the number of classes ( 2k ) for classification in the network [5]. As the code length increases, training the neural network will require large codeword dataset [6]. To the end, researchers have attempted to reduce the complexity of decoding by minimizing the number of iterations or operations. Nachmani et al. proposed a feedforward method where the iteration process unfolds by passing messages, with weights introduced at the edges of the Tanner graph [7][8]. Compared to traditional BP decoders, the algorithm achieves similar decoding performance with fewer iterations. This decoding method is commonly referred to as neural belief propagation (NBP), which can be seen as a generalization of the BP decoder, where individual messages are scaled by a single damping factor [9]. In NBP, the weights help mitigate the impact of short cycles by appropriately scaling the messages.

Although NBP improves the traditional BP algorithm, the number of parameters in the NBP LDPC decoding network increases dramatically with the length of the LDPC code, leading to significant memory consumption. Dai et al. proposed a neural MS decoding method for the original LDPC code, where the same parameters are shared by a bundle of edges derived from the same edge type [10]. Eliminating redundant parameters is crucial for reducing complexity. In this paper, we improve the NBP presented in [8] and propose the SNBP-R decoding network. By introducing weights sharing, using the same weights in one iteration, the number of parameters is reduced in a reasonable manner. The parameter settings of SNBP-R is closer to BP than those of NBP.
The rest of this paper is organized as follows: Section Ôůí describes the BP algorithm and the NBP LDPC decoding network. In Section Ôůó, we propose a well-design SNBP-R LDPC decoding network based on NBP. In Section Ôůú, we make a set of experiments with the proposed network. Finally, we conclude in Section ÔůĄ.
II. SYSTEM MODEL AND BACKGROUND
LDPC code is a class of linear block codes characterized by a sparse parity-check matrix H. For (N, k) LDPC code, the codeword consists of N bits, which are made up of k information bits and (N ÔÇô k) parity-check bits. Vector x with the k-bit information is encoded into N-bit codeword u using the formula u = xG, where G is the generator matrix.
The LDPC decoding network we propose is based on the following the BP algorithm and the NBP LDPC decoding network.

B. Neural BP LDPC Decoding Network
The traditional BP decoder iteratively updates messages between VNs and CNs by unrolling the Tanner graph for message passing. One method to mitigate the impact of short cycles on the BP decoding performance of LDPC codes is to introduce weights on each edge of the unrolled Tanner graph. This approach leads to the Neural Belief Propagation (NBP). The NBP LDPC decoding network consists of an input layer, hidden layers, and an output layer. The hidden layers are further divided into CN layers and VN layers. The input layer initializes the channel information, and the output layer performs the decoding decision.

The NBP LDPC decoding network assigns weights to all nodes in each iteration, meaning that each edge in the Tanner graph has a distinct weight. To reduce the complexity and storage requirements of NBP, the weights can be bound. In [11], the weighted relationships are explored both temporally (i.e., across iterations) and spatially (i.e., within a layer, where all edges have the same weight). It is important to note that setting all the weights of the NBP decoder to 1 results in the traditional BP decoding method.

III. THE PROPOSED SHARED NEURAL BP LDPC DECODER
For the NBP LDPC decoding network, each connection edge between CNs and VNs is assigned a distinct weight during each iteration. For long code, using different weights for each iteration can result in a significant computational overhead due to the large number of multiplications required.

Although the neurons in SNBP-R have different function, i.e., Neurons in CN layers, Neurons in VN layers, and Neurons in output layers, the correction factor of SNBP-R is the weight shared by all the edges in each layer. The structure of the SNBP-R LDPC decoding network is unfolded, just as a Tanner graph. Sharing parameters allows weights to be shared between edges of adjacent network layers. This sharing mechanism helps prevent overfitting during network training.

IV. EXPERIMENTAL SETTING AND EVALUATIONS
A. Test Setup
In this section, we train and test the performance of the SNBP-R LDPC decoding network. We select LDPC code with a codeword length of 576 and a code rate of 3/4 according to IEEE 802.16e, and the codeword randomly generated. We use mini-batch gradient descent to train the network. The mini-batch size was set to 100, and each SNR occupies the same proportion in one minibatch. The training data was generated at various signal-to-noise ratios (SNRs) from 1 to 5 dB, which helps improve the robustness of the data-driven neural network. The network was optimized using the adaptive moment estimation (Adam) optimization method and the learning rate of 0.001. It is worth noting that one iteration of the BP algorithm corresponds to one variable node layer and one check node layer in the network.
B. BER Performance with Number of Network Layers
The experiment analyzes the relationship between the number of network layers and BER performance.
As shown in Fig. 3, the BER performance significantly improves with an increase in the number of network layers. When the network reaches 42 layers(20 iterations), there is a significant improvement in performance. However, after that, the performance improvement becomes less noticeable. It is important to note that increase in the number of layers leads to higher computational complexity. Therefore, considering the trade-off, we recommend choosing a network with 42 layers (20 iterations).
C. BER Performance Comparison
In this experiment, we compare the BER performance of the BP, NBP, and SNBP-R LDPC decoder. The training method of the SNBP-R LDPC decoding network conforms to the NBP LDPC decoding network.
As shown in Fig. 4, the BER of the SNBP-R LDPC decoding network with 42 layers (20 iterations) reaches 10-8 at SNR = 5 dB, which is approximately 0.6 dB better than the NBP LDPC decoding network. From the perspective of weight sharing, the decoding structure of the SNBP-R LDPC decoding network is closer to the BP. Comparing the three decoders, we can obtain that the weights in the SNBP-R decoder only change with each iteration, which is equivalent to adding a normalization factor to the BP. Therefore, the structure and weight sharing scheme of the SNBP-R decoder are similar to the BP algorithm, serving as the basis for network unfolding.
D. Generalization of SNBP-R LDPC Decoding Network
The experiment analyzes the generalization of the SNBPR LDPC decoding network. Taking the (576, 432) LDPC code as an example, it is generated by 802.11e Basic Graph A during training, but the testing LDPC code generated by the 802.11e Basic Graph B. The result is shown in Fig. 5, the performance of the proposed network is not very well, for several reasons. In the First, the check matrices generated by Basic Graphs A and B have different structures, which means the connection relationships between check nodes and variable nodes are different. E. Ablation Studies
1) Weight Sharing: After introducing weight sharing based on the NBP, the network performance improves. In Table I, we can obtain that weight sharing significantly reduces the number of weights required by the network, which in turn greatly decreases the memory requirements.
2) Optimization of Loss Function: We can observe an improvement in performance by optimizing loss function. Taking 20 iterations as an example, Fig. 8 shows the training process before and after weight optimization. After using the optimized loss function, the weight training process becomes much smoother, and the loss value converges more quickly.
V. CONCLUSIONS AND FUTURE WORK
In this paper, we have improved the NBP algorithm by introducing weight sharing and optimizing the loss function, leading to the proposed SNBP-R decoding network. It enhances network performance. Simulation results show that, compared to the NBP LDPC decoding network, the proposed network has better BER performance. Additionally, the improved network reduces the number of learning parameters, making it easier to train and implement. In future work, we plan to develop a decoder with relaxed constraints and implement it on FPGA.
