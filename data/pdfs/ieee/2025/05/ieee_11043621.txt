Hardware Implementation of Modified Noisy Gradient Descent Bit-Flipping Decoders
Qingan Li, Wai-man Tam and Francis C. M. Lau Department of Electrical and Electronic Engineering, The Hong Kong Polytechnic University, Hong Kong SAR, China
Email: qingan.li@connect.polyu.hk, wm.tam@polyu.edu.hk, francis-cm.lau@polyu.edu.hk

Abstract— Recently, modified noisy gradient descent bit flipping (MNGDBF) algorithms have been proposed to eliminate the Gaussian random generators required in the original noisy gradient descent bit flipping (NGDBF) algorithm for the decoding of low-density parity-check (LDPC) codes. In this paper, a platform has been established for the hardware implementation of the LDPC encoder, the Gaussian noise channel generator and the decoder using MNGDBF decoding algorithms based on the use of field programmable gate array (FPGA). With 6-bit quantization, results show that the error performance achieved by hardware experiments is similar to that obtained by floating-point computer simulations. Moreover, we propose a new MNGDBF algorithm, in which the flipping condition is determined with the help of the summation of all inversion functions. Experimental results show that the proposed algorithm has superior performance than the original MNGDBF algorithm.
Index Terms— Low-density parity-check (LDPC) codes, bit flipping, field programmable gate array (FPGA).
I. INTRODUCTION
Low-density parity-check (LDPC) codes have been widely studied, the excellent performance of which has been demonstrated with the soft-decision decoding, such as belief propagation (BP) and min-sum (MS) algorithms. However, the trade off for the good performance is the high complexity of the decoder. Hence, hard-decision algorithms such as bitflipping (BF) algorithms have been proposed by making use of simpler computational units [1], [2]. Gradient descent bit flipping (GDBF) algorithm is an effective method to balance the performance and the complexity, which treats the decoding process by maximizing an objective function [3]. The problem of this algorithm is that it is easily trapped in a local maximum. To solve this problem, random binary number has been generated to determine the flipping for the binary symmetric channel (BSC) in probabilistic GDBF (PGDBF) [4], probabilistic parallel bit flipping (PPBF) [5], and improved GDBF (IGDBF) algorithms [6]. Also, in noisy GDBF (NGDBF) algorithm, a Gaussian perturbation signal has been introduced into the inversion function for the additive white Gaussian noise (AWGN) channel [7]. In [8], modified NGDBF (MNGDBF) algorithms have then been proposed to eliminate the Gaussian perturbation generators by transforming the received signals to generate the perturbation signals.
Regarding the hardware implementations, PGDBF [9], [10], PPBF [11] and IGDBF [6] have been performed for BSC channels. However, for an AWGN channel, only an architecture has been provided with consideration of practical concerns including quantization, adaptive threshold, and noise sample generation, for the implementation of the NGDBF algorithm

[7]. In this paper, we explore the implementation of MNGDBF algorithms on FPGA.
In [12], the syndrome-based minimum inversion function is considered to determine the threshold for the flipping. Inspired by this idea, we modify the MNGDBF by making use of the summation of all inversion functions in the flipping condition. The experimental performance of this new MNGDBF has also been examined.
This paper is organized as follows. Section II reviews the MNGDBF algorithms. Section III introduces the hardware implementation of MNGDBF decoding, and Section IV proposes a new MNGDBF algorithm. Section V presents the experimental results. Finally, Section VI gives the conclusions.
II. REVIEW OF MNGDBF ALGORITHMS
Let H represent an m × n parity-check matrix of a binary LDPC code, the elements of which hij ∈ {0, 1} for i = 1, 2, · · · , m and j = 1, 2, · · · , n. Also, define sets N (i) = {j :
hij = 1} and M (j) = {i : hij = 1} to record the locations of 1s in the i-th row and the j-column of H, respectively.
We consider that the bipolar type c˜ of a codeword c is passed through an AWGN channel after binary phase-shift keying (BPSK) modulation. Then the received vector denoted by y = (y1, y2, · · · , yn) can be expressed as y = c˜ + z, where z = (z1, z2, · · · , zn) is a noisy vector. Next, a decision vector x = (x1, x2, · · · , xn) ∈ {−1, +1}n is obtained as the sign of vector y. The bipolar syndrome component for each
row can be evaluated by si = j∈N(i) xj = ±1. If si = +1 ∀i = 1, 2, · · · , m, then x outputs to a valid codeword.
To eliminate the random number generator and reduce the complexity of the decoder, modified NGDBF (MNGDBF) algorithms have been proposed [8]. In the MNGDBF algorithms, qk’s (k = 1, 2, . . . , n) (refer to (2) and (4) in Algorithm 1 and Algorithm 2, respectively) are replaced with the transformed version lk’s of the received signals yk’s. For k = 1, 2, . . . , n, the three transformations are described as follows.
Transformation #1 : if yk > 1, then lk = yk − 2; elseif yk < −1, then lk = yk + 2; else lk = yk.
Transformation #2 : if lk > 0, then lk = lk − 1; else lk = lk + 1.
Transformation #3 : Given two fixed values p1 & p2, if p1 < lk < 1, then lk = lk + p2;

We consider the PEGReg504x1008 code in [14]. Based on the parity-check matrix H, we evaluate the generator matrix G by using Gaussian elimination. Thus, random binary bits can be used as the input to generate the codewords with FPGA hardware circuits.
Next, the process of the Gaussian noise generator for the channel is mainly based on the generation method proposed in [15]. The AWGN samples are produced by using a pre-computed table and a pseudorandom number generator (PRNG) constructed by linear feedback shift registers (LFSRs).

When t = 1, the decision vector x can be evaluated by extracting the first bit (i.e., sign bit) of the received signal vector y as the output, which will also be used as an input to the calculate syndrome module.

Step B calculates the inversion function. The output of this module will determine whether each bit needs to be flipped. This step takes 4 clock cycles. Step C flips the input data based on the inversion function. If the output condition is met after flipping, output the data; otherwise, the output data of this module will be used as the input data for the “calculate syndrome vector” module. This step takes 3 clock cycles. According to such an arrangement, each iteration consisting of Steps A, B and C requires 10 clock cycles. The coded data with 1008 bits emitted by the encoder is first corrupted by the channel noise. Each corrupted signal is then quantized into 6 bits and input to the decoder. Thus y occupies a total of 6048 bits of memory whereas the output of the decoder x has a length of 1008 bits.
Fig. 2 shows the inputs and outputs of the “calculate syndrome vector” module. In this module, the calculation process of the bipolar syndrome is si(t) = j∈N(i) xj(t), where xj ∈ {−1, 1}. In the context of circuit design, the multiplication operation of the bipolar values can be realized by exclusive-or (XOR) operations of the corresponding binary

values. If any one of the output syndromes equals −1, then set the condition of syndrome vector s (Con s) = 1, which means that the iteration will continue. Otherwise, stop the iteration and output the decision vector.

In [12], a syndrome-aided gradient descent bit flipping (SAGDBF) algorithm has been proposed for LDPC decoding [12], which determines the flip condition of each bit based on the minimum inversion function and the syndromes during each iteration. Results show that the algorithm can improve the decoding ability of the GDBF algorithm. Here, we propose a modified algorithm based on MNGDBF by introducing a flip condition function fE(t) determined by the summation of all inversion functions in (8). VI. CONCLUSION
Based on three MNGDBF algorithms, including the original MNGDBF algorithm and its two enhanced versions (MNGDBF+A and MNGDBF+B algorithms), hardware implementations have been performed based on FPGA. With 6-bit quantization, MNGDBF+B has the best performance among three MNGDBF algorithms. Moreover, a new MNGDBF algorithm has been proposed with the aid of the summation of all inversion functions. Experimental results show that the new algorithm slightly outperforms the original MNGDBF algorithm.
