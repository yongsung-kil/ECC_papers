Abstract—This paper presents a high-throughput decoding method for quasi-cyclic low-density parity-check (QC-LDPC) codes, optimized for both CPU and GPU-based Software-Defined Radios (SDRs). The approach leverages a SIMD data alignment architecture that minimizes cache misses and is designed to take full advantage of modern CPUs and GPUs with rich registers and caches. An innovative decoding algorithm is introduced, exploiting the SIMD capabilities of contemporary CPUs to enhance parallelism, while optimizing memory access patterns to align SIMD operations and minimize branching and cache misses on AMD EPYC CPUs (2xEPYC-9754). This results in significant throughput improvements. The method is further extended to GPUs, utilizing their large-scale parallelism through efficient use of registers and local memory. To maximize performance, a memory-efficient check node decoding strategy is employed, ensuring that operations fit within the cache of the Streaming Multiprocessors (SM) to reduce memory access overhead. Simulation results demonstrate the effectiveness of this approach: the CPU-based 8-iteration layered min-sum decoder with early termination achieves coded bit throughputs of up to 72.9 Gbps for 16200-bit codes and 50.9 Gbps for 64800-bit DVB-S2 QC-LDPC codes. On GPUs, the decoder reaches 132 Gbps for short 802.11n Wi-Fi 1944-bit codes and 22.3 Gbps for 5G Base Graph 1 26112bit codes on an NVIDIA RTX 4090. For 6G prototype QC-LDPC codes, the proposed 3-layer design achieves up to 76 Gbps for 1920-bit codes, with surpassing 104 Gbps on the NVIDIA RTX 5090. Compared to the open-source AFF3CT CPU framework, the proposed method offers 5 to 12 times higher throughput across various code lengths. Even when AFF3CT is optimized with MIPP intrinsics, the new decoder provides 3.2 to 5.7 times better throughput. The GPU implementation delivers up to 120 times the speed of previously published results and offers 13 to 40 times improvement over older GPUs like the GTX 1080 Ti and Titan X, scaling effectively with modern GPUs like the RTX 4090. This approach establishes new performance benchmarks for QC-LDPC decoding, positioning it as an ideal solution for next-generation 5G/6G wireless and satellite communications.

The factor graph of the LDPC code, denoted as G and illustrated in Fig. 1, is a Tanner graph comprising two vertex sets. One set encompasses m = n − k factor (check) vertices (nodes) c0, c1, . . . , cm−1, representing m rows of the matrix H. Message-passing soft decoding, commonly known as belief propagation (BP) or sum-product decoding, along with its min-sum approximation, is widely used for error correction in LDPC codes [25], [26]. The decoding process involves iterative updates of messages between variable nodes and check nodes in two alternating steps: variable-to-check node updates (V2C) and check-to-variable (C2V) node updates, Fig. 1. For efficient hardware implementation, the decoder algorithm employs a fixed-point quantization approach, which restricts the range of message values to optimize hardware processing. The quantization limits are set as follows: variable nodes (V Nclipping) are represented using q-bit integers, with V Nclipping = 2q−1 − 1, while check nodes (CNclipping) are represented using p-bit integers, with CNclipping = 2p−1 − 1. III. SIMD SOFT MESSAGES OPTIMIZED DECODING
The proposed decoding architecture utilizes SIMD AVX256/512 instructions to optimize the performance and efficiency of LDPC decoding, specifically for Quasi-Min-Sum (QNMS) algorithms. By incorporating SIMD AVX vector instructions, the architecture enhances efficiency by simplifying the C2V message updates, reducing memory overhead, and boosting processing speed. The use of shared memory and registers helps optimize memory usage, minimize global memory accesses, and reduce memory bandwidth demands. With SIMD instructions integrated into the design, the architecture maximizes parallelism and throughput, perfectly aligning with the requirements of Min-Sum decoding.

• Prefetching and Predicated Operations: The design leverages AVX-256/512 prefetching and predicated operations (e.g., mm512 mask blend) to eliminate branching by applying conditional masks directly in vectorized operations. This approach minimizes control flow disruptions, enhancing decoding efficiency.
• Reordering of CPM: To optimize memory alignment and data locality, circulant permutation matrices are reordered. This preserves the first layer of the circulant permutation, preventing unwanted shifts and enhancing system efficiency.
• 1D VN Loading: The algorithm maps the first dimension of the array to the SIMD vector register width (F = 512/8 = 64 bits for AVX-512). The second dimension size is ⌈Z/F ⌉ = 128/64 = 2, where Z is the expansion factor. Memory alignment is achieved through zero-padding, ensuring that each Variable Node (VN) group becomes a fixed-size block ⌈Z/F ⌉×F , even when Z mod F ≠ 0. The overhead introduced by padding is negligible, as Z is typically large.
This SIMD-based approach significantly improves performance in high-throughput applications like DVB-S2 systems. By efficiently aligning and handling VN data, it reduces memory access latency and boosts throughput. Moreover, the integration of the 2D array architecture with SIMD processing optimizes computational resource utilization, ensuring better scalability and performance in LDPC decoding. This method represents a major advancement in decoding efficiency, making it particularly well-suited for modern, data-intensive communication systems.

The proposed GPU Decoder integrates an efficient forwardbackward (FB) message-passing mechanism with a memoryoptimized processing approach specifically tailored for GPU architectures. This method addresses the challenges of memory bottlenecks, parallelism, and data alignment that are commonly encountered in LDPC decoding. The key achievement of the memory-reduced approach lies in improving the efficiency of CN updates in LDPC decoding by effectively balancing computation and memory usage Alg. 4. The traditional two-minimum sum method for CN updates requires 4dc comparisons. By splitting the CN update into forward, backward, and merging processes, the FB approach reduces the total number of comparisons to 3(dc − 2). This reduction leads to computational savings that are proportional to the degree dc, thus improving efficiency.In the FB approach, rather than storing separate forward (Fj) and backward (Bj) messages, memory space is shared between these messages during the process. This reduces the storage requirement for intermediate messages from 2(dc − 2) to dc − 2, optimizing memory usage and reducing overhead.By leveraging the GPU architecture, CN processing is assigned to thread blocks, and memory allocations are optimized through the use of shared memory and registers. This strategy minimizes memory bandwidth bottlenecks, ensuring high throughput and efficient use of the available computational resources. The proposed GPU Decoder with Memory-Optimized Forward-Backward Processing represents a significant advancement in LDPC decoding, Alg. 5. By integrating memory principles with GPUspecific optimizations, the proposed decoder achieves high throughput, low latency, and efficient memory utilization. This design offers a scalable and robust solution for modern errorcorrection systems, effectively bridging the gap between theoretical performance and practical implementation. The forward and backward message-passing processes are synchronized, ensuring minimal idle time for GPU cores and maximum utilization of available resources.

V. CONCLUSION
This work establishes new performance frontiers for QC-LDPC decoding on modern CPUs and GPUs through hardware-aware optimizations targeting SIMD parallelism, memory hierarchy utilization, and layered message-passing. The proposed architecture with AVX vectorization achieves 72.9 Gbps throughput for DVB-S2 codes (16200 bits) and 50.9 Gbps for 64800-bit codewords on dual AMD EPYC-9754 CPUs—a 5–12× improvement over the AFF3CT framework with MIPP SIMD Wrapper. These results demonstrate that multi-core x86 architectures can deliver FPGA-class throughputs when combined with cache-aligned data structures and predicated SIMD operations eliminating branching penalties. On GPUs, our register-centric forward-backward check node update strategy achieves 66.77 Gbps for Wi-Fi 802.11n codes and 24.41 Gbps for 5G NR BG1 codes on RTX 4090—surpassing prior GPU implementations by 40–120× [16], [18]. Using 802.11n QC-LDPC codes (1944, 972), an RTX 5090 GPU demonstrated a throughput of 104.9 Gbps. Three innovations enable this breakthrough.
