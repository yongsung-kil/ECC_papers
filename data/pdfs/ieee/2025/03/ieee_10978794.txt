AbstractвҖ”We propose in this paper to enhance the 5G lowdensity parity-check (LDPC) codes by transmitting the codewords in a block Markov superposition transmission (BMST) manner, resulting in a class of spatially coupled LDPC codes. We present a generalized extrinsic information transfer (EXIT) chart for performance analysis, showing that the decoding thresholds can be improved by increasing the memory size (coupled width) m. However, for m > 1, the receiver requires a relatively large window size for the sliding window decoding (SWD) algorithm, potentially causing unacceptable decoding latency. To address this issue, we propose an adaptive sliding window decoding (ASWD) algorithm, in which the decoding window size depends on the decoding state of the BMST system. The proposed EXIT chart analysis can also effectively guide the setting of the maximum decoding window in the ASWD algorithm, as well as the setting of the superposition fractions for the BMST-5G-LDPC system. Simulation results show that: 1) the ASWD algorithm can effectively reduce the average decoding window size in the medium to high signal-to-noise ratio (SNR) region, without the performance loss compared to the conventional SWD algorithm; 2) the proposed BMST-5G-LDPC code can achieve about 0.4, 0.5 and 1.7 dB performance gains compared to the 5G LDPC code over the AWGN channel, the fast fading channel, and the quasistatic block fading channel, respectively.
Index TermsвҖ”5G, extrinsic information transfer (EXIT) chart analysis, low-density parity-check (LDPC) codes, spatially coupled LDPC (SC-LDPC) codes.
I. INTRODUCTION
Low-latency immersive communication services, such as immersive extended reality (XR) and holographic communication, are important applications in 6G [1], which present more rigorous requirements on reliability and latency. Due to the constraint of latency, the hybrid automatic repeat request (HARQ) mechanism with multiple transmissions is not effective. Therefore, designing a forward error correction (FEC) code with

Due to their good performance and high decoding throughput, the rate-compatible low-density parity-check (LDPC) codes, have been adopted as the coding scheme for data channels in the 5G standard, referred to as 5G new radio (NR) LDPC codes. One way to improve the performance of LDPC codes is to invoke the spatially coupled LDPC (SC-LDPC) codes [2]вҖ“[4]. However, the hardware implementation of the encoder and decoder for conventional SC-LDPC codes is generally loosely related to that of basic LDPC codes, as the encoder/decoder for block LDPC codes cannot be directly reused in the corresponding SC-LDPC codes. This does not fully meet the downward compatibility requirements for 6G wireless communication.
Alternatively, the block Markov superposition transmission (BMST) [5] is hardware-friendly, as the encoder and decoder can be implemented by reusing the architecture of the basic code. Through the BMST manner, the generator matrix of the basic code is spatially coupled [6], and an extra coding gain can be obtained. Meanwhile, existing studies indicate that the performance of BMST is closely related to the selection of basic codes, such as repetition codes [7], convolutional codes [8], and stronger LDPC codes [6], [9], where the code block (CB)-coupled BMST-LDPC code was proposed in [6] and the transport block (TB)-coupled BMST-LDPC code was proposed in [9]. Both of the above BMST-LDPC codes utilize the memory size (coupled width) of m = 1, where every two adjacent codewords are superimposed, to improve the performance of the LDPC block codes. A straightforward question arises: can we further improve the performance by increasing the memory size m and using the 5G NR LDPC codes? If so, is there any way to keep the decoding latency at the same level as that for m = 1? The answer is positive, as justified by our performance analysis and simulation results. C. Adaptive Window Decoding
For the decoding of BMST codes, a conventional SWD algorithm with window size d is adopted, which can be described as an iterative message passing algorithm over the normal graph [5] containing d layers as shown in Fig. 3. The message-updating rules for these four types of nodes in each layer are summarized below.
вҖў Node = : The connected edges must take the same (binary) value. The message updating rule at this node is the same as that of the variable nodes in an LDPC code.
вҖў Node C : The connected edges must satisfy the constraint of the 5G LDPC code C , where the messages on the halfedges are associated with the information source.
вҖў Node + : The connected edges must satisfy the constraint that the sum is zero over F2. The message updating rule at this node is the same as that of the check nodes in an LDPC code, where the half-edges are calculated from the channel observations.
вҖў Node Si : The connected edges correspond to the input and the output after the operation of a selection matrix with superposition fraction Оұi. This node simply transfers the messages associated with the selected bits between the node = and node + .
