B. Decoding Method
The decoding procedure can be summerized as follows.
1) Decode the received block using Sum-Product decoding. Store decoding candidates afer a fixed numbe of iterrations.
2) Decompress the decoding candidate with the highest number of calculations obtained in Step 1.
3) Using the decompressed data, construct the frequency table.
4) Compare the obtained frequency table with the frequency table embedded in the header of the compressed data. If both frequency tables are equal, the decoding is successful and the procedure stops. Otherwise, abandon the decoding candidate obtained at Step 2 and let the candidate with the second most calculation be the next decoding candidate. Go to Step 3.

B. Computational Time and Memory Usage
In the existing method, Viterbi decoding is used to obtain candidates during decoding. However, Viterbi decoding is computationally expensive and requires substantial memory resources to derive one candidate. The proposed method is expected to reduce the computational time because it uses sparse matrices of LDPC codes. Furthermore, Sum-Product decoding enables parallel computation, which is expected to further reduce processing time.

V. CONCLUSION
This paper has proposed an error recovery method for Static PPM with LDPC code. Since the proposed method uses LDPC codes and Sum-Product decoding, its encoding and decoding times are shorter than those of the existing method, which uses convolutional codes and Viterbi decoding. Future work will focus on improving the robustness of the error recovery capability.
