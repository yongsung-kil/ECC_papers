Channel-Aware Gradient Descent Bit-Flipping Algorithm for LDPC Codes
Woohong Min and Kyeongcheol Yang , Senior Member, IEEE

Abstract— The gradient descent bit-flipping with momentum (GDBF-w/M) and probabilistic GDBF-w/M (PGDBF-w/M) algorithms significantly improve the decoding performance of the bit-flipping (BF) algorithm. In this letter, we propose a channelaware GDBF-w/M algorithm which operates deterministically based on the received values from the additive white Gaussian noise (AWGN) channel. Numerical results show that the proposed algorithm does not only mitigate the error-floor phenomenon of the GDBF-w/M algorithm, but it also has better decoding performance than the PGDBF-w/M algorithm without the need for a random number generator. Furthermore, the complexity of the proposed algorithm is slightly higher than that of the GDBF-w/M algorithm.
Index Terms— LDPC codes, bit-flipping algorithm, decoding algorithms.
I. INTRODUCTION
LOW-DENSITY parity-check (LDPC) codes [1] have capacity-approaching performance under messagepassing (MP) decoding when the code length goes to infinity [2]. Due to their excellent error-correcting capability, they have been adopted as channel codes in the standards for mobile and broadcasting communication systems such as 5G New Radio, IEEE 802.11n/ac/ax, IEEE 802.3an, and DVB-S2.
MP algorithms such as the sum-product (SP) algorithm [3] and the min-sum (MS) algorithm [4] have outstanding decoding performance for long LDPC codes. However, due to their computational complexity, they might not be applied to low power applications including wireless sensor nodes and wearable health-trackers. As an alternative, the bit-flipping (BF) algorithm may be considered in such applications. In fact, it has much lower computational complexity than MP algorithms, since it passes only binary information through the edges between variable nodes (VNs) and check nodes (CNs). Roughly speaking, the BF algorithm computes a flipping function (FF) to measure the reliability of a VN and uses it in determining whether a bit (or a group of bits) is to be flipped or not [1]. The BF algorithm exhibits good performance within a few iterations for very long codes [5], but shows degraded performance for moderate-length codes.
In order to improve the BF algorithm, the FF has been slightly modified in the weighted BF (WBF) [6] and the improved modified WBF (IMWBF) algorithm [7]. Based on

the gradient descent (GD) method [8], Wadayama et al. [9] introduced a new form of the FF and proposed the gradient descent BF (GDBF) algorithm. These algorithms operate iteratively, but compute their FFs without using any information from previous iterations. Savin [10] proposed a GDBF algorithm with momentum (GDBF-w/M algorithm, for short) by noting that the convergence characteristic of the GD technique can be improved through a momentum [11], [12]. A momentum term is contained in the FF of the GDBF-w/M algorithm, which reduces the probability that a flipped bit is flipped again within a certain number of iterations.
The GDBF-w/M algorithm has dramatically better decoding performance than the GDBF algorithm, but still has an error floor over the additive white Gaussian noise (AWGN) channel. In order to mitigate the error-floor problem, the probabilistic GDBF-w/M (PGDBF-w/M) algorithm was proposed in [10]. Since the PGDBF-w/M algorithm utilizes a probabilistic flipping scheme [13], it requires a random number generator (RNG) for hardware implementation. As an alternative, a decoding diversity method was proposed in [14] for a deterministic decoder without the need for a RNG, but it was limited to the binary symmetric channel.
In this letter, we propose a channel-aware GDBF-w/M (CGDBF-w/M) algorithm that adequately adjusts an FF’s scaler based on the received values for BF-based decoders operating deterministically over the AWGN channel. Numerical results show that the proposed algorithm can not only overcome the error floor of the GDBF-w/M algorithm, but also has better decoding performance than the PGDBF-w/M algorithm. In particular, it behaves deterministically without the need for a RNG. The complexity of the proposed algorithm is slightly higher than that of the GDBF-w/M algorithm.
The rest of this letter is organized as follows. In Section II, the BF algorithm and its variants are briefly reviewed. The proposed algorithm is presented in Section III. We present numerical results on the performance and complexity of the proposed algorithm in Section IV. Finally, we give concluding remarks in Section V.

III. THE PROPOSED CGDBF-W/M ALGORITHM
The PGDBF-w/M algorithm has excellent decoding performance thanks to the momentum term and probabilistic flipping, but its hardware implementation becomes complicated because it requires a RNG. To overcome this issue, we propose a CGDBF-w/M algorithm that has better decoding performance than the GDBF-w/M algorithm while operating in a deterministic manner.
A. Observations
Before proceeding with our discussion of the proposed CGDBF algorithm, it is essential to investigate the effect of the momentum term and probabilistic flipping. For this purpose, we consider decoding the dv4R050N1296 code [15], which has length 1296, rate 0.5, and column weight 4. The decoding parameters are the same as those in [10].
In order to look into the effect of the momentum term, we analyze decoding errors in the GDBF and GDBF-w/M algorithms where ℓmax is set to 300. For a given value of Eb/N0, Fig. 1 demonstrates the conditional cumulative distribution function (CDF) of the number of bit errors left in the frame, provided that a frame is incorrectly decoded. Here, Eb denotes the energy per information bit and N0 is the one-sided noise power spectral density. When Eb/N0 is high, the conditional CDF for the GDBF-w/M algorithm converges rapidly to 1 as the number of bit errors, unlike the GDBF algorithm. In other words, the number of bit errors remaining in the incorrectly decoded frame under the GDBF-w/M algorithm is small with high probability. This phenomenon becomes much clearer as Eb/N0 increases.
The effect of probabilistic flipping at early iterations has been observed by examining the change in the number of bit errors under the GDBF-w/M and PGDBF-w/M algorithms. Fig. 2 shows the probability mass function (PMF) of the number of occurrences where the number of bit errors in x(ℓ+1) is larger than that in x(ℓ). IV. NUMERICAL RESULTS
To demonstrate the decoding performance of the proposed algorithm, we consider two LDPC codes: One is the dv4R050N1296 code [15] and the other is the IEEE 802.3an code of length 2048 and dimension 1723 [16], denoted by (2048, 1723) 802.3an code. As a reference, the SP algorithm is also compared with the (P)GDBF and (P)GDBF-w/M algorithms. The BF-based algorithms including the proposed algorithm have ℓmax = 300, while the SP algorithm has ℓmax = 20. Their decoding parameters are the same as those in [10]. In other words, α, δ, d, p are set to 1.8, 1.1, (2, 2, 2, 2, 2, 1, 1), 0.9 for the dv4R050N1296 code and 4.5, 1.2, (3, 3, 2, 1), 0.8 for the (2048, 1723) 802.3an code, respectively.

A. Error Rate Performance
The FER performance of the dv4R050N1296 code is shown in Fig. 3. The proposed CGDBF-w/M algorithm outperforms the GDBF-w/M algorithm by about 0.8 dB at FER = 10−5. Surprisingly, the proposed algorithm has better decoding performance than the PGDBF-w/M algorithm. It means that the error-floor phenomenon can be mitigated just by adaptively splitting α, without using a RNG.
Fig. 4 shows the FER performance of the (2048, 1723) 802.3an code. The superiority of the CGDBF-w/M algorithm over the GDBF and GDBF-w/M algorithms can be similarly observed. In fact, the CGDBF-w/M algorithm has approximately 0.25 dB gain over the GDBF-w/M algorithm at FER = 10−6. It is also checked that the proposed algorithm alleviates the error-floor phenomenon at the region of high Eb/N0. It is noteworthy that the CGDBF-w/M algorithm has better FER performance than the PGDBF-w/M algorithm, despite the fact that its decoding process is deterministic.
The convergence performance of the (2048, 1723) 802.3an code is demonstrated in Fig. 5, when the maximum number of iterations goes to 300. Here, Eb/N0 is set to 4.5 dB.

The momentum term enhances convergence speed, which makes the (P)GDBF-w/M algorithm have lower FER than the (P)GDBF algorithm when the maximum number of iterations increases. On the other hand, probabilistic flipping is not always beneficial. At early iterations, the PGDBF and PGDBF-w/M algorithms perform worse than the GDBF and GDBF-w/M algorithms, respectively. The CGDBF-w/M algorithm, along with the GDBF-w/M algorithm, exhibits the fastest convergence up to about 200 iterations. From iteration ℓth = 250 onward, an oscillation phenomenon is mitigated so that the FER of the CGDBF-w/M algorithm is significantly reduced.

B. Complexity Analysis
Clearly, the FF of the proposed CGDBF-w/M algorithm is distinct from that of the GDBF-w/M algorithm only at the first term when ℓ ≥ ℓth. Like the GDBF-w/M algorithm, the CGDBF-w/M algorithm initially calculates and stores αyn (for all n). At iteration ℓth, αyn is scaled to αnyn for all n. These computations are performed exactly once only for the frames in which decoding proceeds up to ℓth or more iterations.
