Abstract—The possibility of reinterpreting established coding structures is considered. A new modification of traditional cascade and generalized cascade codes is proposed, which, in turn, gives rise to a fundamentally new family of codes and opens up prospects for the development of novel decoding algorithms. Three examples of code constructions with error localization are presented, along with the results of their simulation. Directions for further research on new classes of codes are outlined.
Keywords—error localization; cascade codes; decoding algorithms; noise immunity; code construction
I. INTRODUCTION
With the emergence of pioneering works by Shannon and Hamming [1, 2], the theory of error-correcting codes, which was already beginning to take shape as an independent scientific discipline, has consistently outpaced the capabilities of its practical implementation. Notably, the gap between theory and practice has not always been solely due to delays in the development of hardware components. Often—if not always—it has required viewing the same problem from a different perspective, something only possible with time.
Several illustrative examples can be cited:
• Iterative codes (product codes) were introduced by Elias in the 1950s [3], but it was not until the mid1990s that Claude Berrou proposed turbo codes based on this concept [4]. While Berrou’s version used convolutional codes and Elias’s approach was based on block codes, the core idea of “turbo” applies to both.

• The OFDM scheme, now indispensable in most modern radio-telecommunication standards, was originally proposed by R. Gallager in the 1950s [5] as a means to extend Shannon’s theorem to time-varying channels. Practical implementations, however, began only 30 years later, first in dial-up telephone channels, then in DSL and DVB systems, and eventually in wireless communications.
• LDPC codes, introduced in Gallager’s 1963 doctoral dissertation [6], likewise saw significant practical adoption more than three decades later, despite being technically feasible much earlier.
• Cascade codes, proposed by D. Forney in the 1960s [7], met a similar fate. The concept was later generalized by E. Bloch and V. Zyablov [8], and decades later led to the development of polar codes. The cascade construction was also extended to signalcode constructions (SCCs) by V. Ginzburg [9], G. Ungerboeck [10], and V. Zyablov and S. Portnoy [11].
Given this context, it is not surprising that a fresh perspective on a long-established coding structure can result in fundamentally new properties.
This paper introduces yet another modification of the wellknown cascade and generalized cascade codes, leading to the creation of a fundamentally new family of codes and, consequently, new decoding algorithms.

II. BRIEF DESCRIPTION OF THE CODE CONSTRUCTION
A. General encoding idea The encoding algorithm is described using the following
terminology. Consider a cascade code consisting of a binary code (n₁, k₁, d₁) (Code 1) and a Reed–Solomon code (n₂, k₂, d₂) over the field GF(2ᵏ¹) (Code 2). Additionally, there exists a code (n₂, k₃, d₃) over the field GF(2ⁿ¹⁻ᵏ¹) (Code 3). Code 1 will be referred to as the inner code, Code 2 as the outer code, and Code 3 as the error localization code.
The encoding algorithm proceeds as follows: • k information symbols are arranged into two matrices
of sizes k₁ × k₂ and (n₁ − k₁) × k₃, respectively. Thus, the total number of information symbols is k = k₁ × k₂ + (n₁ − k₁) × k₃, and the codeword length is n = n₁ × n₂. • The k₂ symbols over GF(2ᵏ¹) are encoded using Code 2. • Each symbol of Code 2 is further encoded using the binary Code 1, resulting in a matrix of size n₁ × n₂. • The k₃ symbols over GF(2ⁿ¹⁻ᵏ¹) are encoded using Code 3. • In the non-binary case, the symbols of Code 3 are multiplied by pseudo-random elements from the field GF(2ⁿ¹⁻ᵏ¹), with the sequence of multipliers known to both the transmitter and receiver. • The resulting symbols of Code 3 are then added bitwise to the lower portion of the cascade code matrix. • The final matrix constitutes a codeword of the (n, k) code and is transmitted column-wise through the communication channel. Although the encoding process consists solely of linear operations, it cannot, in general, be represented by a single generator matrix; therefore, the resulting code is not necessarily a linear code. The final code is referred to as an error localization code (ELC). An illustration of the encoding process is provided in Fig. 1.

B. General Decoding Idea
Decoding of a received codeword is performed across several cosets. The number of cosets used in the decoding process is determined by the number of possible erasure positions within a given coset. In this context, Code 3 functions both as an error-localizing code and as a carrier of additional information symbols.
For each candidate codeword within a coset, decoding is first performed using Code 1, followed by decoding with Code 2.
III. CASCADED ERROR LOCALIZATION CODES WITH LONG OUTER CODES
The performance of error localization codes can be improved by employing long outer codes, such as Reed– Solomon codes. In this case, a three-dimensional cascaded construction with error localization is obtained.
To enhance the error-correcting capability of the threedimensional cascaded construction, one can additionally apply decoding of the Reed–Solomon outer code with both error and erasure correction. This is particularly feasible because block erasures can be easily implemented within the binary inner code.
This combination of codes is well-suited for control and telemetry channel applications, as it allows flexible adjustment of the outer code depending on transmission conditions.
A. Description of a (32, 16) Code with Outer Codes of Length 8
Let the outer code (Code 2) be a Reed–Solomon code (8, 5, 4) over GF(8), the inner code (Code 1) be a binary (4, 3, 2) code with parity check, and the error localization code (Code 3) be a binary (8, 1, 8) code.
The encoding process is the same as previously described. Five GF(8) symbols are encoded using the outer code, then converted into binary form and arranged in a matrix. This matrix is column-wise encoded using the binary inner code. Depending on the value of the last information symbol, a codeword from Code 3 is added to the bottom row of the resulting matrix (either inverting the row or leaving it unchanged).
It is important to note that the proposed decoding algorithm for the (32, 16) code achieves performance close to that of maximum-likelihood (ML) decoding for the (32, 15) code, while significantly reducing computational complexity: only 56 erasure patterns are considered instead of an exhaustive search over all 32,786 possible codewords of the cascade code. Simulation results comparing the proposed decoding algorithm with ML decoding are shown in Fig. 2. The same Fig. also presents results for the (32, 15) code using the proposed decoding method.

V. COMPLEXITY ANALYSIS OF THE PROPOSED DECODING ALGORITHM
Let us consider the decoding complexity for Example 1. The decoding process involves 56 executions of the Reed– Solomon (8, 5) code over GF(16). Each decoding step corresponds to solving three linear equations over GF(16), which requires 15 hex-symbols XOR operations. This procedure is repeated across 16 iterations, resulting in a total of approximately 53,760 binary additions. Other operations are negligible in comparison. It should be noted that this is a pessimistic upper-bound estimate and that the complexity can be significantly reduced with practical optimizations.
Now consider the complexity of decoding for Example 2. Here, maximum likelihood (ML) decoding of Hamming (8, 4) codes is added, but the number of Reed–Solomon decodings is significantly reduced. Moreover, since only a single erasure is corrected in each Reed–Solomon decoding, the main computational burden lies in multiple ML decodings of the Hamming (8, 4) code. However, this process is also easily optimized. For a single decoder, the number of such decodings is equal to n2 * 16. Currently, a new algorithm is under development that aims to drastically reduce this number.
VI. DIRECTIONS FOR FURTHER RESEARCH
This work presents initial results on a fundamentally new coding construction and novel decoding approaches, both for the constituent codes and for the construction as a whole. The authors identify the following directions for future investigation:
• Development of optimal decoding techniques for block codes of length approximately 128, achieving near maximum-likelihood performance at low complexity. This would rely on decoding both within the code and within its cosets.
• Achieving better performance with three-dimensional cascade constructions than with conventional (255, 223) Reed–Solomon codes combined with a Viterbi decoder (constraint length k = 7), by enhancing the inner block code and allowing both error and erasure correction in the outer code.
• Extending the proposed approach to joint coding and modulation schemes, to optimal trade-offs between data rate, error resilience, and decoding complexity.

VII. CONCLUSIONS
A new platform for designing decoders for short block codes has been proposed, offering near-maximumlikelihood performance while primarily relying on simple logical operations.
The proposed platform has several key characteristics:
• Specific decoding strategy for the inner code
• List decoding of the Reed–Solomon outer code
• Very short constituent codes with low overall decoder complexity
• Use of new code types with coset decoding and error localization to accelerate encoding
