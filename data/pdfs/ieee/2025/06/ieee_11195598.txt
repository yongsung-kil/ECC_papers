Abstract—We consider the reconstruction of a codeword from multiple noisy copies, each independently corrupted by insertion, deletion, and substitution errors. This problem arises, for example, in DNA data storage. A common code construction uses a concatenated code with an outer linear block code and an inner marker code, decoded via Belief Propagation (BP) and the BahlCocke-Jelinek-Raviv (BCJR) algorithm, respectively. However, the BCJR algorithm scales exponentially with the number of noisy copies, making reconstruction from more than about four copies infeasible. In this paper, we introduce BCJRFormer, a transformer-based neural inner decoder for marker codes. BCJRFormer achieves error rates comparable to the BCJR algorithm for single-message transmissions while scaling only quadratically with the number of noisy copies. This makes BCJRFormer well suited for DNA data storage, where multiple reads of the same DNA sequence are common. To further reduce the bit error rate, we replace the BP outer decoder with a transformer-based decoder. Combined, this results in a performant and efficient endto-end transformer-based pipeline for decoding multiple noisy copies corrupted by insertion, deletion, and substitution errors.
I. INTRODUCTION
DNA data storage is an emerging storage medium that encodes binary data into DNA sequences for high-density, longterm storage. However, the processes of writing, storing, and reading the DNA are error-prone, resulting in multiple noisy copies of the same sequence [1]. Errors can be categorized into substitutions and erasures, and insertions and deletions. While optimal linear codes are known for erasures and substitutions, no such guarantees exist for insertions and deletions.
A common approach to address insertion, deletion and substitution errors is a concatenated coding scheme with an inner and outer encoder/decoder pair. The inner code is often nonlinear, such as marker codes [2] or watermark codes [3], and is typically decoded using the Bahl-Cocke-Jelinek-Raviv (BCJR) algorithm. BCJR achieves maximum a posteriori (MAP) decoding under the assumption of perfect channel state information. The outer code is often a linear block code, such as a low-density parity-check (LDPC) code [4] or a polar code [5], and is decoded using Belief Propagation (BP). BP achieves MAP decoding when the Tanner graph it decodes on has no cycles [6].

To address this, we propose BCJRFormer, a transformer-based architecture for inner decoding of marker codes that achieves error rates close to joint BCJR decoding while scaling only quadratically with the number of noisy copies.
To further reduce bit error rates, we replace the outer BP algorithm with the Error Correcting Code Transformer (ECCT), originally proposed in [8] for binary-input symmetric-output channels. Together, BCJRFormer and ECCT form a twostep decoding approach for LDPC codes concatenated with a marker code, which improves efficiency and reduces bit error rates for correcting insertion, deletion, and substitution errors compared to traditional BP and BCJR decoding.
II. RELATED WORK
Deep learning has been widely explored for decoding in wireless communication systems [9]–[17]. We focus on related work addressing neural decoding for insertion, deletion, and substitution errors, as this is most relevant to our setting.
Several neural decoders have been proposed to replace traditional iterative algorithms such as BCJR or BP [18]–[23]. These approaches can be broadly categorized into model-based and model-free [24].
The authors of [25] propose two model-based recurrent neural network architectures, FBNet and FBGRU, for decoding marker codes under insertion, deletion, and substitution errors. The model input is derived from the BCJR algorithm, similar to how we construct the input for BCJRFormer. FBNet and FBGRU outperform BCJR when channel state information is imperfect.

Their method uses one BiGRU for decoding marker codes and another for decoding codewords encoded with LDPC or convolutional codes.
Model-free neural decoders are not constrained by a fixed architecture but need to learn both the encoding scheme and how to correct errors. The authors of [8] propose the Error Correction Code Transformer (ECCT), which encodes code structure into the model by restricting attention to bits related by parity-check equations. ECCT outperforms existing neural decoders on binary input symmetric output channels.
More recently, the authors of [27] propose CrossMPT to improve ECCT’s decoding performance. CrossMPT uses two cross-attention modules, one for channel noise and one for syndrome information, mimicking the check and variable node updates of the BP algorithm.
The authors of [28] propose DNAformer, an end-to-end retrieval system for DNA data storage. DNAformer uses transformers to reconstruct sequences from multiple noisy reads, followed by a classical coding scheme to correct remaining errors after reconstruction.
III. BACKGROUND
In this section, we provide an overview of the concepts relevant to our work. We first introduce our channel model and discuss concatenated codes, with a focus on the BCJR algorithm, which is used to derive the input to BCJRFormer. We then describe the transformer architecture and the ECCT decoder.
A. Channel Model
We consider transmission over the insertion, deletion, and substitution (IDS) channel for the binary alphabet. The IDS channel is parameterized by the deletion probability pD, the insertion probability pI , and the transmission probability pT = 1−pI −pD. A bit is transmitted correctly with probability 1 − pS and substituted with probability pS. The IDS channel can be generalized to non-binary alphabets, such as the quaternary alphabet used in DNA data storage [3].
B. Concatenated Schemes and Linear Codes
Concatenated codes consist of an inner and an outer encoder/decoder pair. A message m ∈ {0, 1}k is encoded by an outer encoder to xout ∈ {0, 1}nout , which is then encoded by an inner encoder to xin ∈ {0, 1}nin . After transmitting xin, the inner decoder synchronizes the received sequence r ∈ {0, 1}nrec and provides soft information in the form of log-likelihood ratios to the outer decoder. Using this soft information and its parity-check bits, the outer decoder corrects remaining substitution errors.
The outer code is typically a linear block code. A (k, nout) linear block code is defined by a generator matrix G ∈ {0, 1}k×nout and a parity-check matrix H ∈ {0, 1}(nout−k)×nout . The generator matrix G encodes messages m into codewords

xout. The parity-check matrix H defines the parity-check equations for decoding and can be represented as a bipartite Tanner graph [6]. The BP algorithm operates on the Tanner graph and iteratively passes messages between check and variable nodes to reconstruct the original message. We use LDPC codes [29] as our outer code, as they are a common choice.

A transformer consists of multiple stacked multi-head attention layers. Each attention layer contains nh attention heads, where each head performs attention on a reduced hidden dimension of dk/nh. The outputs of all heads are concatenated and passed through a linear layer, typically followed by dimensionality reduction to match the task-specific output size.
The ECCT decoder [8] incorporates code structure by masking out unrelated bit positions in its self-attention mechanism. Let rBPSK ∈ [−1, 1]nout denote the noisy output of a binary phase-shift keying modulated codeword transmitted over a binary-input symmetric-output (BISO) channel. The model input is the concatenation of the magnitude |rBPSK| ∈ Rnout and the syndrome ϕ(syn(bin(rBPSK))) ∈ Rnout−k, where bin(x) = 0.5(1 − sign(x)) binarizes the signal, syn(x) = Hx mod 2 computes the syndrome using the parity-check matrix H, and ϕ(x) = 2x − 1 maps binary values to bipolar form.

B. ECCT as Outer Decoder
We adapt ECCT for use as an outer decoder by transforming its input. We transform the normalized log-likelihood ratio approximation from the inner decoder (after removing marker bits) into bipolar vectors xϕ = ϕ (exp (LLR (xout | r))) ∈ [−1, 1]nout . The input to ECCT is the concatenation of the magnitude part |xϕ| and the syndrome part ϕ syn(bin xϕ .
The inner decoder introduces noise that depends on the specific input codeword. As a result, outer decoding over the IDS channel varies with the codeword, unlike decoding over a BISO channel, where the noise is independent of the input. For example, aligning an all-zero codeword is simpler than aligning a codeword with alternating patterns. Consequently, we cannot train only on the all-zero codeword as in [8]; instead, we train on pseudorandomly generated codewords to capture codeword-dependent noise.

D. Complexity
We determine the complexity of jointly decoding multiple sequences using BCJR and BCJRFormer to justify our deep learning-based approach.
As in Section IV-A, we restrict the drift states of the BCJR decoder at each time step to the range [dmin, dmax]. For M sequences, joint decoding requires tracking δM joint drift states. Each drift state allows up to κ = Imax + 2 transitions, with Imax being the maximum number of insertions per state. Consequently, the total complexity of BCJR decoding is O(nin(κδ)M ), where nin is the input sequence length [7].
In contrast, BCJRFormer only scales quadratically in M . The transformation of the received sequences into the model’s input, as in (6), and the subsequent embedding have a combined complexity of O(M ninδdk). Each transformer block incurs quadratic complexity, scaling as O M 2n2inNdecdk , where Ndec is the number of attention layers [32].
V. EXPERIMENTAL RESULTS
We compare BCJRFormer to the classical BCJR algorithm, which performs maximum likelihood decoding and sets a performance upper bound. We show that BCJRFormer achieves error rates close to BCJR while allowing joint decoding of multiple sequences, and that combining it with an outer transformer decoder outperforms the combination of classical decoders BCJR and BP.
A. Setup
We train all neural decoders with the Adam optimizer [33]. Each BCJRFormer model has a hidden dimension of 96, six attention layers, and eight attention heads per layer, for a total of about 700,000 parameters. We train at a batch size of 256 for 160,000 iterations, and use a cosine decay learning rate schedule that starts at 2.5e−4 and decays to 2.5e−5 after a 20,000 iterations warmup.
All ECCT models have a hidden dimension of 128, eight attention layers with eight heads per layer, for a total of about

1.6 million parameters. We train ECCT at a batch size of 1024

for 120,000 iterations with a constant learning rate of 1e−4.

B. BCJRFormer Single Codeword
We compare BCJRFormer and BCJR for inner decoding of single sequences using a short (96, 48) LDPC code [34] and a longer (204, 102) LDPC code [35]. For the short LDPC code, we insert the marker sequence sm = 001 every Nm = 6 bits, and for the longer LDPC code, every Nm = 7 bits. This gives inner codeword lengths of 144 and 291 with rates of 2/3 and approximately 0.7, respectively. We trained a range of models for both codes over varying pI = pD with pS = 0. The BER results in Fig. 3 show that BCJRFormer closely matches the performance of the BCJR algorithm for both code lengths and across all insertion and deletion rates considered.

C. BCJRFormer Joint Inner Decoding
We evaluate BCJRFormer for jointly decoding M noisy copies of a codeword. Inner decoders are trained for different values of M , with pI = pD varied and pS fixed at 0.012. We compare the resulting error rates to those of standard BCJR decoding.
For M = 3, we evaluate BCJR on only 40,960 samples due to its exponential complexity. As shown in Fig. 4, BCJRFormer achieves error rates that closely match those of BCJR as the number of received copies increases. For M > 3, BCJRFormer continues to attain decreasing BER, while BCJR becomes computationally impractical.

In DNA data storage, the number of received copies Mk varies for each transmitted sequence, making it inefficient to train models on fixed cluster sizes M . We show that a single BCJRFormer model can decode sequences when Mk varies between Mmin and Mmax. We handle varying Mk by padding the concatenated input RninMk×δ×2 with a dedicated token to RninMmax×δ×2 and mask the Mmax − Mk padded columns in (5) and (7). We evaluate the BER of a dynamically trained model, where we sample Mk ∈ [Mmin, Mmax] uniformly at random for each codeword, and compare it to models trained on fixed Mk. As shown in Fig. 5, the dynamic model performs comparably to models with fixed Mk, with only a minimal performance trade-off.
D. ECCT for Outer Decoding
We compare the ECCT decoder with BP using the short (96, 48) LDPC code from Section V-B. We consider two inner

code constructions, a (2, 1, 2) zero-terminated convolutional code (generator polynomials g = (5, 7)8) with a pseudorandom offset to reduce block cyclicity [36], and a marker code with markers 001 inserted every Nm = 6 bits. We train models on the output of an inner BCJR decoder, varying pI = pD while keeping pS = 0.012 fixed. Fig. 6 shows that the ECCT decoder outperforms BP decoding across most channel configurations.
While the comparison uses standard BP, additional messagepassing iterations or extensions such as ensemble BP [37] and weighted BP [13] may reduce the performance gap. We leave such comparisons to future work, as our main focus is on improving inner decoding.
E. End-to-End Transformer Decoding
We show that an end-to-end transformer pipeline with BCJRFormer as the inner decoder and ECCT as the outer decoder outperforms both BCJR and BP.
We reuse the BCJRFormer models from Section V-B and train a separate ECCT outer decoder for each, using the inner decoder outputs corresponding to a channel configuration with pI = pD and a substitution probability of pS = 0.012. Fig. 7 compares various combinations of inner and outer decoders. ECCT consistently achieves lower error rates than BP across all channel conditions considered. In the low-noise regime, combinations using BCJRFormer as the inner decoder significantly outperform those using BCJR, demonstrating that a transformer-based pipeline can outperform iterative decoding in concatenated coding schemes.
VI. CONCLUSION
In this work, we proposed a performant and efficient transformer-based decoding approach for multiple noisy copies of a codeword transmitted over the IDS channel. Future work can investigate jointly training BCJRFormer and ECCT to further reduce error rates, and evaluate how BCJRFormer’s performance scales with compute.
