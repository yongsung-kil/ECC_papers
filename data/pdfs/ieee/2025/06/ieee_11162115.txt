A New Post-Quantum Signature Based on
Punctured QC-LDPC Code With Random Insertion

Abstract—This paper proposes an enhanced version of the CFS signature scheme, leveraging QC-LDPC codes with puncturing and random insertion techniques. The puncturing process is designed based on the selection of minimum Hamming weight codewords, ensuring that the remaining structure of the paritycheck matrix retains sufficient error-correcting capabilities. Random insertion of new rows into the punctured parity-check matrix further enhances security by introducing randomness. The cyclic structure of QC-LDPC codes greatly reduces the key size while the BP decoding algorithm improves the decoding success rate, thereby enhancing the signature generation efficiency. The scheme proposed in this paper reduces the key size to 6144 bytes compared to the original CFS scheme’s 6283256 bytes under the similar parameters setting. Additionally, the average number of signing attempts is reduced from t! to a constant multiple of t. It also greatly enhances security compared to recent improvements, particularly in resisting structural attacks, Stern’s attacks, OTD attacks, and forgery attacks. Similar to the original CFS scheme, the proposed scheme can be proven to satisfy Existential Unforgeability under Chosen Message Attacks (EUF-CMA) security.
Index Terms—Post-Quantum Signature, CFS scheme, QCLDPC codes, Puncture, Random Insertion, EUF-CMA
I. INTRODUCTION
The Courtois-Finiasz-Sendrier (CFS) [1] signature scheme, proposed in 2001, is one of the most prominent code-based digital signature schemes. Its security is based on the hardness of the syndrome decoding problem, which makes it resistant to quantum attacks. However, despite its security advantages, the original CFS scheme has faced several challenges, particularly with regard to the public key size, the signature length, and the overall signing efficiency. Over the years, various improvements and adaptations of the CFS scheme have been proposed to address these limitations, but many of these versions introduced new challenges. Finiasz and Matthieu [2] proposed the Parallel-CFS algorithm aiming to achieve standard security levels with smaller security parameters by utilizing parallelism. However, this comes at the cost of longer signatures and reduced signing efficiency. In

2011, Barreto et al. [3] introduced an improvement based on Quasi-Dyadic codes. This approach reduces the public key size significantly by leveraging the structure of Quasi-Dyadic codes. Unfortunately, the signature generation process became less efficient due to the increased complexity of the code structure. In 2013, Baldi et al. [4] proposed an alternative improvement using Low-Density Generator Matrix (LDGM) codes. However, the signature generation in LDGM-based CFS introduced a correlation between multiple signatures, allowing adversaries to exploit this correlation to forge new signatures. In 2014, Gaborit et al. [5] introduced an innovative approach with the RankSign scheme, which was based on rank distance Low Rank Parity Check (LRPC) codes. But it was later broken by Debris-Alazard et al. [6], exposing vulnerabilities in its structure and leaving it ineffective against more sophisticated attacks. In 2019, Debris-Alazard et al. [7] proposed the Wave signature scheme, which utilizes (U |U + V ) codes and offers security based on the ternary high-error-rate HSD problem and the (U |U + V ) code distinguishing problem. While Wave has not yet been broken, its practical application is limited by the size of the public key and the inefficiency in signature generation. In 2020, Lee et al. [8] introduced MpqsigRM, an improvement based on modified Reed-Muller (RM) codes. This scheme achieved shorter public keys and faster signature generation while maintaining resistance to known attacks. However, the security of MpqsigRM remains a subject of debate, as its resistance to quantum attacks is still uncertain.
This paper introduces a novel enhancement to the CFS signature scheme by leveraging QC-LDPC codes, combined with puncturing and random insertion techniques. QC-LDPC codes leverage a quasi-cyclic structure that enables a sparse yet efficient parity-check matrix, significantly reducing public key size due to the reduced storage requirements for cyclic blocks. Puncturing creates irregularities by removing certain columns without significantly impacting error-correcting capability, while random insertion of additional rows or columns adds complexity to the matrix structure, enhancing resilience against structural attacks. This approach improves the CFS scheme by reducing both key size and signature generation time, while enhancing security against matrix-structure-based attacks.

C. CFS Signature Scheme

A. Linear Block Codes
Let F2 denote the finite field with two elements. A linear block code C is a k-dimensional subspace of Fn2 , where n is the code length and k is the dimension of the code. The vectors in Fn2 are referred to as words, and those in C are specifically called codewords. A generator matrix G for C is
a k × n matrix composed of k linearly independent column
vectors that form a basis for C. The parity-check matrix H of the code C is defined such that GT H = 0. For any codeword c in C, the equation HT c = 0 holds, and the syndrome of any word c is given by HT c.
Definition 1: The support of a codeword c is defined as the
set of indices i for which ci ̸= 0, denoted by supp(c). Definition 2: Given a codeword c and an index set I, the
projection of c onto I, written as projI (c), is a sub-codeword formed by the components of c indexed by I. For a linear code
C, the projection of the code onto I is defined as projI (C) = {projI (c) | c ∈ C}.

D. QC-LDPC Codes And Its Modification

1) QC-LDPC Codes: Low-Density Parity-Check (LDPC) codes are a class of linear block codes characterized by a sparse parity-check matrix H of size (n−k)×n. The defining feature of LDPC codes is the low density of non-zero elements in H, with ’1’s scattered randomly across the matrix. This structure implies that LDPC codes have a very low-density parity-check matrix. In this context, n represents the code length, k is the dimension, and r is the number of check bits, satisfying r = n − k. Quasi-cyclic low-density parity-check (QC-LDPC) codes are a subclass of LDPC codes characterized by a block-cyclic structure in their parity-check matrix. 2) BP Decoding Algorithm: The block-cyclic structure of the parity-check matrix in QC-LDPC codes allows for efficient storage of keys during the signing process, as only the positions and cyclic shifts of each non-zero submatrix need to be stored. And the QC-LDPC codes can apply the Belief Propagation (BP) algorithm to decode the syndrome. The BP algorithm operates by iteratively passing messages between variable nodes and check nodes in the Tanner graph representation of the QC-LDPC codes. The quasi-cyclic structure of QC-LDPC codes ensures that the messages passed during decoding are repeated cyclically, allowing efficient hardware implementations and reducing decoding latency. The algorithm’s complexity remains low, even as code length increases, allowing for rapid and scalable decoding.
3) Puncturing QC-LDPC Codes with Random Insertion: Building upon the method presented in [10] for puncturing Reed-Muller (RM) codes with random insertions to establish a secure RM code-based public key scheme, similar techniques can be applied to QC-LDPC codes. These puncturing and random insertion methods can also be used in a QC-LDPC code-based signature scheme. In the case of QC-LDPC codes, puncturing the generator matrix G is analogous to performing deletions of rows and columns in the parity-check matrix H [11]. Since a signature scheme requires the use of a paritycheck matrix H, we propose to use a modified version of H with certain rows and columns deleted, along with random row insertions. III. THE PROPOSED SCHEME

Our digital signature scheme adheres to the CFS scheme and encompasses three principal algorithms: key generation (Gen), signing (Sign), and verification (Verify). The Gen algorithm initializes the system parameters and key pairs. With the established parameters, a secret key, and a plaintext, the Sign algorithm generates a valid signature for the plaintext. The Verify algorithm then assesses the signature’s validity using the public key, plaintext, and signature, outputting ACCEPT for a valid signature and REJECT otherwise.

A. Key Generation
1) Puncture: Let G denote a k × n generator matrix for a QC-LDPC code C. Start by finding a codeword u in C with minimal weight and determining its support, represented by supp(u). Next, identify a codeword v with minimal weight in the sub-code projsupp(u)(C), which comprises components of C restricted to the support of u. Set p to satisfy w(v) ≤ p ≤ 2 w(v). Randomly select an index set ID such that supp(v) ⊆ ID and |ID| = p. Finally, remove columns in G indexed by ID to form a modified generator matrix, denoted GD.
2) Random Insertion: Define II = {i1, . . . , i|II|} as a set of randomly chosen indices for columns, with each ij satisfying 1 ≤ ij ≤ n − |ID| + |II |. Insert new columns at positions in II into GD, resulting in the updated matrix GDI , where columns at indices in II represent the inserted elements. By choosing |ID| = |II |, the shape of the modified generator matrix is kept consistent with the dimensions of the original matrix G.

The attacker’s reliance on Stern’s algorithm becomes impractical due to the increased computational complexity caused by these random disruptions in the code’s structure.

V. EFFICIENCY
A. Key Size
The public key size calculation for different coding schemes varies according to the code length n and the information bit length k. In the proposed scheme, the public key is finally represented as Hpub = SHDI P , and the private key as (S, H, P, γ, ID, II ). The public key storage requirement is k(n − k) + 2pn and the secret key storage requirement is k + 2kn + 2pn. Compared to the original CFS scheme whose secret key storage is k(n − k) + 2kn, since normally n is large but p is small, the proposed scheme has a significant reduction in the size of the secret key while the public key size maintains the same due to the introduction of a cyclic structure. As shown in Table 1, The proposed scheme utilizing punctured QC-LDPC codes with random insertion achieves an information rate of 0.75 with a markedly reduced key size of 6144 bytes, contrasting with the 20481 bytes needed for schemes using LDPC codes. Furthermore, when contrasted with the original CFS scheme using Goppa codes, which also operate at an information rate of 0.75, the QC-LDPC code’s key size is dramatically smaller than the hefty 6283256 bytes demanded by Goppa codes. These attributes make QC-LDPC codes highly advantageous for applications that demand both high efficiency and low resource consumption, such as digital signatures in Internet of Things (IoT) devices [16].

B. Signing Efficiency
The digital signature σ = (m, e, i) consists of three components: a k-bit vector m, an n-bit vector e, and an integer i, which takes values in the range (0, 1, . . . , 2n−k − 1). In the original CFS framework, achieving a decodable syndrome requires an average of approximately t! decoding attempts due to the sparse distribution of decodable syndromes in the Goppa code’s syndrome space. This high computational demand arises because traditional Goppa codes rely heavily on exhaustive search during decoding. By contrast, when substituting Goppa codes with QC-LDPC codes utilizing BP decoding, the number of decoding attempts significantly decreases. The QC-LDPC code’s sparse and quasi-cyclic structure allows BP decoding to converge faster, achieving high decoding success rates without exhaustive search. Consequently, each generated syndrome has a greater probability of being decodable, which reduces the overall decoding iterations required for signature generation. With BP decoding, the average number of decoding attempts is reduced from t! to a constant multiple of t.

VI. CONCLUSIONS
In this paper, we introduced an improved CFS signature
scheme using punctured QC-LDPC codes with random in-
sertion. By employing the quasi-cyclic structure, we sig-
nificantly reduced public key size. The puncturing process
creates structural irregularities while the addition of random
insertion introduces further complexity, disrupting potential
attack patterns and enhancing the scheme’s robustness against
Stern’s attack and OTD attacks. Moreover, the use of BP
decoding in our QC-LDPC-based scheme improves decoding
efficiency, achieving a faster convergence rate that reduces
signature generation time. This scheme not only mitigates
the inefficiencies and large storage requirements associated
with traditional CFS schemes but also provides enhanced
resilience against attacks aimed at exploiting matrix structure.
With proven EUF-CMA security, this optimized QC-LDPC-
based scheme is well-suited for post-quantum cryptographic
applications requiring efficient digital signatures.
