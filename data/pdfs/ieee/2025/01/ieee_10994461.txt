ABSTRACT Bit-patterned media recording (BPMR) is increasingly recognized as a leading technology for next-generation storage systems due to its potential for achieving extremely high area density. In BPMR, data is stored on discrete magnetic islands arranged in a patterned layout, enabling greater storage capacity than traditional magnetic recording techniques. However, as area density increases to meet growing data demands, the magnetic islands are positioned closer together, leading to significant two-dimensional (2D) interference. This interference complicates signal interpretation, making accurate data recovery more challenging and necessitating advanced detection methods. To address these challenges, sophisticated algorithms such as maximum a posteriori (MAP) detection have shown promise in improving data recovery by accounting for the probabilistic nature of signals and noise. MAP detection can effectively mitigate 2D interference when it incorporates extrinsic information (additional data that enhances the detection process). One way to obtain this extrinsic information is through iterative algorithms that utilize the output of self-MAP detection or decoding. Low-density parity-check (LDPC) codes are commonly employed alongside MAP detection to exploit extrinsic information. However, previous studies have shown that suboptimal parity matrix structures in LDPC decoders degrade performance, particularly when using the sum-product algorithm. In this paper, we propose an optimized approach that integrates MAP detection with LDPC decoding for BPMR systems while refining the parity check matrix structure to enhance performance. Our modifications focus on reducing the number of 1-bits to maintain low density (consistent with LDPC standards) while strategically redistributing them throughout the matrix. This optimized parity matrix structure leads to improved performance compared to previous studies.

In a BPMR system, bit data is first modulated into an electrical signal by the read-and-write head. This signal is then converted into a magnetic signal, which interacts with the magnetic islands on the storage medium. The magnetic poles of these islands ultimately represent binary data as either a 0-bit or a 1-bit [21]. Each magnetic island is isolated by surrounding non-magnetic zones. To achieve higher area density (AD), these data storage islands are positioned closer together, which inevitably increases two-dimensional (2D) interference. This interference manifests as inter-symbol interference (ISI) and inter-track interference (ITI), affecting both the cross-track and down-track directions. As the spacing decreases, signals from neighboring islands overlap, making it more challenging to accurately distinguish stored data points, leading to higher noise levels and error rates [22]. During manufacturing, deviations in island position or size result in two types of media noise: position fluctuation and size jitter [23]. Additionally, when reading signals from magnetic islands, the read head may deviate from the intended data track, causing a track misregistration (TMR) effect. Furthermore, additive white Gaussian noise (AWGN) is introduced to the received signal, contributing to overall noise in BPMR systems [24].
To address these challenges, various methods have been proposed: modifying the structure of the BPMR media, employing error-correcting codes, and designing advanced detection techniques. In terms of structural modifications, one approach involves rearranging the magnetic islands into a staggered BPMR configuration, as shown in Fig. 1. In staggered BPMR, the magnetic islands within a given track are arranged in a staggered manner relative to the tracks above and below. This configuration ensures that each island is influenced by only six neighboring islands, rather than the eight surrounding islands in regular BPMR.
As a result, staggered BPMR significantly reduces interference compared to regular BPMR, leading to improved performance [25]. However, due to the half-bit skewed structure between tracks, staggered BPMR cannot effectively mitigate 2D interference using conventional algorithms. To address this issue, some researchers have proposed a 2D equalizer that converts the received signal into a onedimensional (1D) interference model, allowing the use of conventional 1D detection methods [26], [27].

For error correction, modulation coding can be applied before storing user data in the BPMR system to prevent isolated patterns, as illustrated in Fig. 2. It can be observed that in isolated patterns, the center bit is highly susceptible to interference from surrounding islands, making accurate restoration nearly impossible. Consequently, these patterns significantly degrade the performance of BPMR systems. To mitigate this issue, modulation codes can be designed with a minimum Hamming distance of at least two, ensuring error-correction capability. In [28], the authors proposed a simple 2/3 modulation code with a bit-flipping technique to reduce inter-track interference (ITI) in BPMR systems. Subsequently, Buajong and Warisarn introduced a 3/4 2D modulation code to mitigate ITI in BPMR systems [29]. To accommodate staggered BPMR systems, a rate 5/6 2D modulation code was later presented in [30].
With advancements in computational structures and algorithms, low-density parity-check (LDPC) codes have also been applied to BPMR systems. Since LDPC codes are inherently designed to handle 1D interference, modifications have been made to adapt them for processing 2D signals in BPMR systems [31], [32], [33]. Furthermore, with the rise of deep learning and artificial intelligence (AI), error-correcting codes have been further enhanced by integrating deep learning and machine learning models, significantly improving coding efficiency in BPMR systems [34], [35], [36].
For detection, the interference coefficients from the channel are estimated and supplied to the trellis diagram. The Viterbi algorithm (VA) or maximum a posteriori (MAP) algorithm is then used to detect the original signals and mitigate interference [37], [38]. Initially, partial response

maximum likelihood (PRML) detection was introduced in [39] and [40] to convert 2D interference into 1D interference in BPMR systems. Once the received signal exhibits 1D interference, VA can be directly applied to remove the interference. Subsequently, to enhance the accuracy of PRML, Nabavi and Kumar proposed the two-dimensional generalized partial response (2D-GPR) target [41], which generates adaptive coefficients for the target instead of relying on fixed coefficients, as in PRML. With the 2D-GPR target, 2D channel coefficients can be incorporated into the detection process, necessitating the design and development of a dedicated 2D detector [42], [43], [44], [45]. Among these methods, the serial structure of 2D detection has demonstrated high performance [46]. Furthermore, the use of the MAP algorithm has been shown to significantly improve the bit error rate (BER) performance of serial detection, as demonstrated in [47]. With the increasing adoption of LDPC codes, the combination of MAP detection and LDPC decoding has been proposed for BPMR systems in [31], [48], and [49], achieving high BER performance.
In this paper, we introduce a novel model that integrates serial MAP detection with LDPC coding. The serial MAP detection consists of horizontal and vertical detection stages. The output from the LDPC decoder is utilized to generate extrinsic information, which is then fed back into the vertical detection stage, effectively establishing a feedback loop between the LDPC decoder and serial MAP detection. Additionally, we propose a modified parity-check matrix for LDPC codes to enhance the performance of the joint MAP-LDPC detection and decoding process. In this matrix, the number of 1-bits is reduced to better adhere to the вҖҳвҖҳlow-densityвҖҷвҖҷ principle of LDPC codes. Simultaneously, we introduce a method to evenly distribute the 1-bits throughout the matrix to further optimize performance. Simulation results demonstrate that the proposed model significantly improves BER performance compared to existing approaches.
Below, we summarize the key contributions of this paper.
- We proposed a novel model that integrates serial MAP detection with an LDPC decoder by leveraging extrinsic information from the LDPC decoder.
- We present and analyze the proposed model to demonstrate its effectiveness in mitigating 2D interference.
- We introduce a modified parity-check matrix for LDPC codes to enhance performance.
- Finally, we provide simulation results to validate the proposed model and highlight its performance improvements.

IV. PROPOSED MODEL In this paper, the user data is decoded using the sum-product algorithm, which relies on the parity-check matrix H for decoding. Therefore, we employ Hamming and LDPC code encoders to encode the user data. In this section, we explain and analyze the encoding process and the integration of detection and decoding. Additionally, we design the structure of matrix H to optimize its performance within the proposed model.

From Fig. 6, it is evident that as the parity-check matrix H grows larger, efficiently storing and decoding Hamming codes with high code rates using the syndrome method becomes impractical. To address this issue, the sum-product algorithm can be applied using the H matrix to decode Hamming codewords. This approach eliminates the need for a lookup table and instead employs iterative message passing, making it more suitable for handling high-rate Hamming codes.
The key advantage of using the sum-product algorithm in this scheme is that it eliminates the need for a look-up table. However, since the sum-product algorithm relies on the relationship between parity bits and data bits to decode the received signal, it does not fully leverage the minimum Hamming distance of Hamming codewords. As a result, its error-correction performance is lower than that of the syndrome method. Therefore, to effectively apply the sumproduct algorithm, LDPC codes are the optimal choice for the decoder in our proposed model.

nodes in the Tanner graph. A high-density matrix leads to a more complex graph with many cycles, increasing computational complexity and slowing down convergence in iterative decoding. Thus, by reducing the number of 1-bits, the number of computations (message passing operations) is reduced, making the decoding process more efficient. For evenly distributing 1-bits, an uneven distribution of 1-bits can lead to some variable nodes having too many connections, while others have too few. Highly connected nodes (high-degree nodes) may dominate the decoding process, causing error propagation and poor convergence. A more balanced distribution ensures that information spreads more evenly across the graph, improving the reliability of iterative decoding. In addition, short cycles (especially length-4 cycles) in the Tanner graph degrade decoding performance because they create dependencies between messages passed in belief propagation. A well-designed H matrix with reduced density and better distribution minimizes short cycles, leading to more independent probability updates and improved decoding accuracy. Finally, a lower-density H matrix reduces storage requirements in hardware implementations. Fewer connections in the Tanner graph mean fewer memory accesses and lower power consumption during decoding.

V. SIMULATION RESULTS AND DISSCUSIONS To validate the proposed approach, we implemented the BPMR system by simulating the model shown in Fig. 5. The user data u[k] вҲҲ {0, 1} was generated randomly, with the probability of bit 0 being equal to the probability of bit 1. Subsequently, u[k] was encoded into c[k]. To represent the data stored on the BPMR systemвҖҷs media, the 1D signal c[k] was modulated into the 2D signal a[j, k] вҲҲ {вҲ’1, 1} by processing the data row-by-row. In this process, the magnetic poles were represented as вҖ“1 for bit 0 and +1 for bit 1.
Typically, in the simulation, a[j, k] was configured as a data page of size 1200 Г— 1200 bits. The simulation was divided into two main processes:
1. Parameter Estimation Process: In this phase, one data page a[j, k] was used to collect the signals y[j, k] and d[j, k], and to estimate the parameters of the target G and equalizer F using the method described in Section III. These estimated parameters were then provided to the next process for evaluating the BER performance.
2. BER Performance Evaluation Process: Ten data pages were used to estimate the BER performance. In this phase, the signal y[j, k] was converted to z[j, k]. The horizontal interference in z[j, k] was mitigated using вҖҳвҖҳHorizontal MAP detection,вҖҷвҖҷ resulting in the signal b[j, k], which contained only vertical interference. Subsequently, вҖҳвҖҳVertical MAP detectionвҖҷвҖҷ was applied to b[j, k] to recover the modulated signal aЛҶ[j, k]. The signal aЛҶ[j, k] was then provided to the decoder. In this study, the sum-product algorithm was used to decode the signal from the вҖҳвҖҳVertical MAP detectionвҖҷвҖҷ stage. This decoding process enabled the extraction of extrinsic information, which was fed back into the вҖҳвҖҳVertical MAP detectionвҖҷвҖҷ to enhance detection performance. Finally, the decoderвҖҷs output cЛҶ[k] was tested the threshold to recover the original signal uЛҶ[k]. The recovered signal uЛҶ[k] was compared bit-by-bit with u[k] to calculate the BER performance of the BPMR system.
