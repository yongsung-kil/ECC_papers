2016 IEEE International Symposium on Information Theory

Integrated Parallel Interleaved Concatenation for
Lowering Error Floors of LDPC Codes
Naoaki Kokubun and Hironori Uchikawa
Center for Semiconductor Research & Development Toshiba Corporation, Storage & Electronic Devices Solutions Company
Email: {naoaki.kokubun.hironori.uchikawa}@toshiba.co.jp

Abstract-In order to suppress error floors of low density parity check (LDPC) codes, we study a parallel concatenation scheme with an intermediate single parity check (SPC) code and interleavers . The interleavers of the concatenation play an important role to lower error floors, because it is highly possible that interleaved bits in a trapping set are not to be in error even if the bits in the trapping set are originally in error. For quasi-cyclic LDPC codes, our simulation results confirm that the integrated parallel interleaved concatenation with circulant-size cyclic-shift interleavers improves error-floor performance, and it is better than serial concatenation schemes with BCH codes. If a decoder can afford to accommodate higher-hardware complexity, the decoding performance of our scheme improves so that the error floor can be lowered further.
I. INTRODUCTI ON
The error-floor phenomenon of low density parity check (LDPC) codes is a crucial obstacle in many communication systems. Even if codes have advantages of decoding perfor­ mance, implementation simplicity, or power consumption, the good codes cannot be used if their error floors are high. In order to deal with error floors, there are three directions [l]: concatenation, code design based on specific trapping set infor­ mation, and improved decoding method. Concatenation can be used without detail information on trapping sets. In addition, the impact of lowering error floors by concatenation can be the strongest among them. The drawback of concatenation is to use additional redundancy. If the redundancy is small, the good codes with high error floors can be options, hence we focus on concatenation in this paper.
Serial concatenation is often used for reducing error floors. There are several options to select the outer code. In order to keep the waterfall performance of the inner code, the outer­ code rate should be very high. Hence, BCH codes and Reed­ Solomon codes are used [2], [3], [4]. For another option of the high-rate outer code, rate-l codes (accumulators) are used for the outer codes to enhance error-floor performance [5], [6].
To decrease the additional redundancy due to the concate­ nation, a single parity check (SPC) code is used for the intermediate code between the inner code and the outer code [3], [7], [8], [9]. Erasure decoding of the SPC code is often performed with decoded results of the inner code. Interleavers have not been discussed much in this context. In [7], they are discussed to facilitate implementation of an efficient decoder.
Parallel concatenation is usually used not for reducing error floors, but for improving waterfall performance. Parallel

concatenated Gallager codes are investigated in [10], and multiple parallel concatenation of LDPC codes with column weight 2 is studied in [11]. Since the interleaver does not improve waterfall performance [10], interleavers have not been discussed in this context, either.
In this paper, we present a parallel concatenation of LDPC codes with an intermediate SPC code and interleavers. The interleavers of the concatenation play an important role to lower error floors of LDPC codes. Simulation results confirm that our concatenation improves error-floor performance, and it is better than other concatenation schemes with BCH codes. We also show that the decoding performance can further be improved if a higher-complexity decoder is accepted.

III. INTERLEAVER AND CONS TITUENT CODEW ORDS
A. Simplification of Interleavers
Interleavers are expected to spread distances between neigh­ bor bit positions in order to improve minimum distance in the context of parallel turbo codes and accumulator based LDPC codes [5], [6], [12]. Since waterfall performance is not im­ proved by interleavers in the context of parallel concatenation of LDPC codes [10], they have not been discussed much as far as we investigated.
For our scheme, the spreading distances is not important. The role of interleavers is to shuffle error bits for an escape from trapping sets. In general, bit positions in a trapping set of random LDPC codes are already spread randomly all over the codeword, hence the probability of the escape is independent of the distance between an original bit position and a permuted one. Therefore, interleavers of our scheme for random LDPC codes can be simplified to one-bit cyclic-shift interleaver. On the other hand, for structured LDPC codes, such as quasi­ cyclic (QC) LDPC codes, one-bit cyclic-shift interleaver is not enough for the escape. Since code constraints of QC-LDPC codes do not change over the circulant matrix size, a circulant­ size cyclic-shift interleaver is suitable instead of one-bit cyclic shift interleaver.

IV. SIMUL ATI ON RESULTS AND COMPARIS ON
In the following simulation, we focus on moderate-length high-rate codes which are also studied in [13], because error floors of the codes are likely to be high. The quasi-cyclic irregular repeat accumulate (QC-IRA) code [14] which is a class of QC-LDPC codes, is employed the constituent code in our simulation. The parity-check matrix of the QC-IRA code, constituent code, is 16 x 96 array of 27 x 27 circulant matrices, and the code length, code rate, column weight of user-data part, and girth are 2592 bits, 0.8333, 4, and 6, respectively. The minimum weight of a codeword appeared in our simulation is 12. In the following simulation, we use randomly generated binary sequences as user data, the memoryless BI-AW GNC, and the sum-product algorithm with the maximum iteration of 50 for decoding of LDPC codes. According to the discussion in the previous section, we consider the case of m = 8.

A. Comparison with Different Interleavers
In order to investigate the impact of interleaver design for our scheme, we compare our schemes with no interleaver and four different interleavers: a one-bit cyclic-shift interleaver, a random interleaver, a random circulant-order interleaver, and a circulant-size cyclic-shift interleaver, where the random circulant-order interleaver shuffles the order of circulant matri­ ces randomly with keeping the order of bits in each circulant, and the circulant-size cyclic-shift interleaver just shifts a 27bit of Ui in a cyclic manner in this simulation, because the circulant size of the QC-IRA code is 27.
The simulation result is shown in Fig. 3. We discuss about which type of interleaver is adequate. The performances with no interleaver or a one-bit cyclic-shift interleaver can work,

but are worse than those with a random, a random circulant­ order, or a circulant-size cyclic-shift interleaver. The error­ floor performance of circulant-size cyclic-shift interleaver can be as well as that of a random interleaver which is more complex, and it is better than that of no interleaver. In summary, a circulant-size cyclic-shift interleaver works well and is simple, hence we use it in the subsequent evaluations, and integrated parallel interleaved concatenation with the in­ terleaver is abbreviated to IPIC for simplicity in the following.

B. Comparison with Different Concatenation Schemes
In this section, we compare several concatenation schemes with the IPIC with the algorithm A. One is the non-shared scheme, the others are serial concatenation code (SCC) schemes. From SCC schemes studied in [3], SCC-II and the new SCC with T = 1, called SCC-SPC in this paper, are also evaluated for comparison. The result is shown in Fig. 4.
We explain the SCC schemes in the first. In this simulation, the inner code of the SCC schemes is the same as that of the IPIC. In this simulation, the performance of the SCC-SPC is almost same as that of the SCC-II. Since the encoder of the SCC-SPC is less complex than that of the SCC-II, the SCC­ SPC will be compared in the following discussion.
Let us compare between the IPIC with the algorithm A and the SCC-SPc. For the SCC-SPC, the inner code is decoded by BP decoding with soft-input information, while the outer code is decoded by bounded-distance decoding with hard-input

For the IPIC, BP decoding of both the first and second-stage, i.e., inner and outer, decoding uses soft-input information, hence the decoder for IPIC needs a larger memory than that of the SCC-SPC. However, the decoder of the IPIC does not need different decoders for the first and second-stage decoding, because the encoder and decoder of the constituent code can also be used in the steps A2 and A3 in the algorithm A of the second-stage decoding. Therefore, if you can admit memory overhead of the decoder, the IPIC with the algorithm A is better than the SCC-SPC at least for the rate-0.833 QC­ IRA code used in this simulation.
Finally, we compare the IPIC and the non-shared scheme to investigate a trade-off between waterfall and error-floor performance. The waterfall performance of the non-shared constituent code is about 0.2 dB better than that of the IPIC, which is consistent with the difference of the Shannon limit between them discussed in Section III. The error-floor performance of the IPIC is much better than that of the non­ shared scheme. Therefore, for the cases that the degradation of the waterfall performance is negligible, the IPIC with the algorithm A seems a good candidate to lower error floors.
