High Throughput LDPC Decoder for C-RAN Optical Fronthaul Based on Improved Bit-Flipping
Algorithm

AO LI, VAHID MEGHDADI, JEAN-PIERRE CANCES AND CHRISTELLE AUPETIT-BERTHELEMOT

XLIM, UMR CNRS 7252/University of Limoges, France ao.li@xlim.fr

Abstract – In this paper we demonstrate the potentiality of the integration of low density parity check (LDPC) codes for a full self-seeded optical architecture dedicated to Cloud Radio Access Network (C-RAN). After comparison, Gradient Decent Bit Flipping algorithm (GDBF) has been retained for this application. We show that improvements can be obtained by using multiple-GDBF or mixed (single + multiple or multiple + single)GDBF. We demonstrate that it is possible to be in accordance with the requirements (BER=10-9) after LDPC decoder with an input BER of 10-4. We propose a serial FPGA architecture that permits to achieve throughputs up to 500 Mb/s using a 500MHz FPGA. We give the guidelines to increase the rate by parallelizing the architecture. We obtained, by FPGA circuit simulations, up to 2.5 Gb/s with affordable complexity.
Keywords: LDPC FEC; C-RAN; hard decision; fronthaul; FPGA
I. INTRODUCTION
The architecture of the mobile network is evolving to provide higher speeds and to fit the various multimedia uses increasingly preferred by users who want connections ubiquitous (anywhere, anytime with any device). One of the developments retained by the operators is the C-RAN (Radio Access Network cloud) [1], which proposes to centralize the BBU (Base Band Unit) by moving the antennal sites to the CO (Central Office). In this way, only the RRH (Remote Radio Head) are present on the antennal sites which simplify deployment and maintenance while providing coverage and maximum throughput. A new optical segment, called the front-haul, and the exploitation of the D-RoF (Digital Radio over Fiber) allows this transformation. This link transports the digital radio signal with a high bit rate by using the CPRI protocol (Common Public Radio Interface) [1]. To overcome the expensive solution of using a point-to-point distribution network between the RRH and BBU, WDM (Wavelength Division Multiplexing) is pointed to be an interesting solution. In this context, the use of achromatic transmitter including RSOA (Reflective Semiconductor Optical Amplifier) selfseeded configuration is promising [2] [4]. This type of transmitter, performance and less interference would be the same for all RRH (Radio Remote Head) and allow an assignment to intrinsic wavelength. One future challenge is to define forward error codes (FEC) suitable for the optical front-haul link in order to achieve a throughput of 2.5 Gb / s and 10 Gb/s.
In this typical context, the objective of this paper is to study by simulation the possibilities offered by LDPC (Low Density Parity Check)[5] ,[6] as part of an optical architecture using RSOA self-seeded configuration for the mobile access network. Different algorithms decoding are compared (BF: Bit Flipping, WBF: Weighted Bit Flipping, SGDBF: Single Bit Flipping Gradient Descent). The implementation of the algorithms has been done on FPGA and the LDPC decoder performance has been analyzed.

The first part briefly recalls the main of coding and decoding of LDPC codes with various aforementioned algorithms. LDPC codes performances are presented in the concerned context. Last part shows the FPGA implementation we proposed for the system. A summary of results are given.
In the optical transmission, LDPC codes can be good candidates because they can be exploited with high message coding rate (typically 0.9). Their performance and cost of exploitation are largely determined by the choice of the decoding algorithm, which is iteratively operated by exchanging messages between the individual processing nodes. Among the various decoding algorithms, BP (Belief Propagation) algorithm and approximate MS (Min-Sum), using flexible decisions, offer the best performance on the AWGN channel with binary input [5] [6]. However, the number of necessary arithmetic operations creates limits for high speed telecommunications systems. Parallelizing calculations and processing allow correcting it to some extent. Other solutions consist in using hard decisions, such as BF (Bit Flipping), WBF (Weighted Bit Flipping), the MWBF (Modified Weighted Bit Flipping) [7] [8] and GDBF (Gradient Descent Bit Flipping)[9] [10]. These suboptimal algorithms are finally presented and implemented in this study. Recently a dynamic Weighted Bit-Flipping algorithm is proposed where the weighting is updated at each iteration [11]. However the updating procedure makes this architecture more complex.

In [8] it is demonstrated that noisy GDBF can improve the performance with smoothing output and threshold updated at each iteration. However, the increase of complexity may not be supported for a very high throughput system. This paper shows that improvements can be obtained also by using multiple-GDBF or mixed (single + multiple or multiple + single)GDBF. We tested these two methods in our simulations with high rate codes, 0.7 and 0.9. We observe that the mixedGDBF is preferred to multiple-GDBF or single-GDBF. In the case of multiple-GDBF, if bits are flipped at once, it is possible to generate more errors for the next iteration and sometimes the system becomes unstable (the errors will never be corrected whatever the number of iteration is set and a kind of oscillation is generated). The simulations show an important result: first multiple then single flipping for GDBF performs better for low rate matrix, while doing the reverse (single flipping first and then multiple flipping) is more suitable for high rate code.

V. FPGA IMPLEMENTATION
A great number of architecture have been already proposed for the soft decision algorithm [11][13]. A full parallel implementation of decoders for large LDPC codes has been problematic due to very large amount of resources required. However, since hard decision algorithm requires less message passing between CNU (check node unit) and VNU (variable node unit), the complexity can be reduced. In this section, we concentrate on mixed-GDBF for which we propose architecture.
A. Design procedure A parameterized hardware model of the decoder was proposed and developed using the Hardware Description Language (HDL) and synthesized using Xilinx Synthesis Tool. The block diagram of the LDPC decoder as implemented is shown in Fig. 8.
Fig. 8 . Block diagram of design LDPC decoder
The decoder consists of a global ‘Clock’ and synchronous ‘Reset’ inputs. The maximum number of iteration is determined and prefixed by 20, as it is illustrated in section 1. The ‘Input’ receives the messages from ADC. ‘Load’ defines when decoder starts to receive the code word and ‘Busy’ shows that the decoder has not finished the decoding process. The ‘Decoded data’ can be obtained when the ‘Data Out ready’ signal is asserted.

VI. CONCLUSION
This article shows the potentiality of LDPC FEC on high throughput transmission for optical front-haul in the C-RAN context. Different decoding strategies have been presented for high rate LDPC code as well. In the case of bit flipping algorithm, the effect of two bit sampling was considered and the performance improvement was given. The best threshold has been determined by simulation results to optimize the two-bit coding of analog input signal. For implementation, a realistic architecture based on Xilinx FPGA has been given, where a

bit rate up to 2.5 GB/s can be obtained. The simulation shows that one iteration is sufficient if the number of errors in a packet is less than 5. This corresponds to a BER of about 5e-3, which is completely practical in optical links.
