Iterative LDPC-LDPC Product Code for Bit Patterned Media
Seongkwon Jeong and Jaejin Lee* Soongsil University

Abstract— Bit patterned media implements high density recording of more than 10 Tera-bits per square inch, but burst error can be a serious problem. Burst errors are occurred by media defect and data write failure. However, if burst errors are erasure compensated, the performance of LDPC code can be improved. In this paper, we show that a proposed iterative LDPCLDPC product code performs better than LDPC-LDPC product code when there are only random errors, and both random and burst errors.
Index Terms - Low-density parity check (LDPC) codes, product code, iterative detection, bit patterned media (BPM), burst error.

I. INTRODUCTION
Bit patterned media (BPM) is a high capacity storage system that has attracted a great deal of attention as a next generation data storage method [1]. The present hard disk uses perpendicular magnetic recording (PMR) as a replacement for the longitudinal magnetic recording (LMR) method. However, it is difficult to increase the density of the channels, due to the problem of super-paramagnetism. Therefore, BPM is one of the possible solutions to overcome the recording limitations caused by the problem of super-paramagnetism.
Burst error caused by physical impact on the high recording density storage system affects very long burst error in contrast with a conventional storage system. Thus, it is necessary to have an error correction capability for long burst error [2]. To correct long burst error, various product code has been proposed such as RS-RS, RS-LDPC, and LDPC-LDPC product code, as shown in Fig. 1 [3]-[6].
In this paper, we propose an iterative LDPC-LDPC product code with extrinsic information (EI) based on conventional LDPC-LDPC product code for BPM.
II. PROPOSED ITERATIVE LDPC-LDPC PRODUCT CODE
To improve correcting capability for long burst error, product code has been used. Inner code has minimum number of parities for detecting burst error. The inner code can correct most of random and short burst error. However, long burst error in each row generates decoding failure. For these uncorrected errors, we declare erasures which inform the locations of errors to outer code decoder. For instance, two dimensional data structure is 100×100 (10,000 bits) and burst error of 5×100 (500bits) occurs. Burst error of 5 rows can be detected by inner code decoder. Then, it declares erasure to 5 lines and Log-Likelihood Ratio (LLR) values are changed to 0. Then, outer decoder can correct these errors using the erasure information.

After the received data pass through the soft output Viterbi algorithm (SOVA), LDPC1 decoder receives LLR value of data and parity of rows from SOVA. After iterative decoding of LDPC1 decoder is completed, LDPC2 decoder receives extrinsic information (EI) and erasure information from LDPC1 decoder and LLR value of data and parity of columns from SOVA. After LDPC2 decoder finishes iterative decoding for columns, LDPC2 decoder informs EI to LDPC1 decoder. The erasure information is only sent at the first step and EI is sent between LDPC1 and LDPC2 decoders in iterative process.

If the write head is off track during the write operation, it can fail to write the target track and even accidentally overwrite the data stored on an adjacent track [7]. The performance of proposed method is better than existing method because we use more EI and parity of rows and columns by iteration between LDPC for lows and LDPC for columns.
IV. CONCLUSION
In this paper, we proposed an iterative LDPC-LDPC product code to improve the performance of BPM. The main idea is to use more EI and parities of both rows and columns iteratively. The performance of decreasing the number of iterations of LDPC decoder and using iterative decoding between LDPC1 and LDPC2 with many EI is better than that of increasing the number of iterations of LDPC with one EI. Therefore, the performance of proposed iterative LDPC-LDPC product code is better than LDPC-LDPC product code in random error and burst error.
