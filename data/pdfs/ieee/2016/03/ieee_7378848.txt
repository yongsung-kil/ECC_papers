A Relaxed Min-Sum LDPC Decoder With Simplified Check Nodes
Saied Hemati, Senior Member, IEEE, François Leduc-Primeau, Student Member, IEEE, and Warren J. Gross, Senior Member, IEEE

Abstract—This letter presents a heuristic technique for simplifying the parity-check node operation in a relaxed min-sum iterative decoder. The proposed decoder eliminates the secondminimum computation in check nodes, which allows broadcasting the same output to all neighboring variable nodes to alleviate routing problem in VLSI implementations of low-density parity check (LDPC) decoders. The second-minimum, when required, is emulated by adding an offset to the first-minimum. The proposed relaxed decoder also uses a relaxation factor equal to 0.5 to simplify variable nodes. Simulation results for two LDPC codes show the proposed decoding algorithm with only 4-bit quantization closely matches the performance of floating-point normalized/offset min-sum and sum-product decoders in the waterfall region.
Index Terms—Relaxed iterative decoders, belief propagation, low-density parity check codes, LDPC codes, energy efficiency.
I. INTRODUCTION
L OW-DENSITY parity check (LDPC) codes were first invented in the 1960s [1] but were soon forgotten – mainly due to the projected implementation complexity at the time of invention – until they were re-invented in 1995 [2]. An LDPC code is a block code with sparse parity-check matrix and can be graphically represented by a bipartite graph called a Tanner graph G = (C, V, E), where elements of C, named check nodes, represent the parity-check equations, and elements of V, named variable nodes, represent the variables of the parity-check equations. An edge exists between a variable node and a check node if the variable is an argument of the parity-check equation associated with the check node. LDPC codes can approach the channel capacity for additive white Gaussian noise channels when belief propagation decoding algorithm (also called sum-product algorithm (SPA)) is utilized [3]. The belief propagation decoding algorithm and its improved variants [4]–[6] have found little applications due to their high implementation complexity, which is mainly due to the complexity of the check node computations. The Min-Sum (MS), Offset Min-Sum (OMS), and Normalized Min-Sum (NMS) algorithms [3], [7]–[9] reduce the implementation complexity and have become the decoding algorithms of choice for LDPC decoding.

feasible to broadcast only a single message to all neighboring variable nodes. To do so, each variable node should extract its own distinct incoming message by eliminating the impact that its immediate output to the check node has had on the check node’s output. Broadcasting technique can be regarded as a method for reducing the number of effective edges and wiring congestion, which reduces delay and increases the clock frequency and throughput. It can also reduce the overall capacitive load associated with the wire connections, improving the energy efficiency of the decoder.
As a simple example, let’s assume six variable nodes are located at the following coordinates in an XY plane: (−4, ±3), (0, ±3), and (4, ±3) and a check node is located at the origin. The required wire length per bit, calculated based on the Manhattan distance between nodes, is 34 units for a nonbroadcast c → v (i.e., 3+3+7+7+7+7), while in a broadcast scheme it is only 22 units (i.e., 3+3+4+4+4+4), which is 35% shorter. If the Euclidean distance was relevant in VLSI design, the required wire length for the non-broadcast scheme would be 26 units (i.e., 3+3+5+5+5+5), which is still 8% longer than the broadcast scheme. Furthermore, each check node can be divided into a few smaller sub-check nodes to merge the variable nodes’ outputs locally and send the outcome to a central sub-check node. These nodes can be placed wherever the broadcast check node to variable node (c → v) link is split into two or more links, i.e., (0, 0) and (0, ±3). Thus the wire length associated with a check node’s incoming (v → c) and outgoing (c → v) messages will be identical and 35% shorter than a conventional non-broadcast scheme.
While broadcasting is not theoretically applicable to MS based decoders, there have been some attempts in using broadcasting for these decoders [10]–[12] in order to benefit from the aforementioned implementation advantages. In these algorithms, the first minimum (1st min) is broadcast to adjacent variable nodes and the second minimum (2nd min) is emulated in the variable node. These algorithms achieve low routing complexity and high energy efficiency, but a coding loss of a few tenths of a dB is observed in comparison with floating-point OMS or NMS decoders. In [10], check nodes are simplified to only output the 1st min and the sign bit and in variable nodes, if the magnitude of the sent and received messages are equal, an empirically optimized offset is added to the received message. For a (2048,1723) RS-LDPC code [13], a 4-bit quantized MS decoder with single minimum check node performs roughly 0.85 dB worse than a full precision MS decoder at a BER of 10−6. Using correction in the variable nodes to emulate the 2nd min, the coding loss is reduced to 0.25 dB. In a 2nd min emulated MS decoder when there are multiple winners in a check node, i.e., multiple check node’s incoming messages have equal magnitude and are equal with the 1st min, adjacent variable nodes will mistakenly increase the magnitude (corresponding to their reliability) of the received messages. This scenario can be repeated for many iterations and falsely increase the reliability of a few wrong bits connecting to a check node. In high SNR region and for codes that are susceptible to error floor, this modified MS decoder can cause an early error floor. Nevertheless, error floor problem can often be resolved using post-processing techniques (see [14] and [6] and references therein). To resolve this issue in [12], 2nd min emulation has become contingent on the existence of a single winner in the adjacent check node. In [12], variable weight Min-Sum (vwMS) decoding algorithm was proposed and the correction parameter (weight) in variable nodes is varied in each iteration. The check node finds the global minimum and verifies if two or more winners exist. Furthermore, an extra wire link is required to inform the adjacent variable nodes if there are more than one winner. This feature improves the decoding performance of vwMS algorithm since the 2nd min is emulated exactly when it is required. A simplified version of the algorithm (simplified vwMS or svwMS) partitions each check node to two parts and compares their minimum values to find the global minimum and if partitions have equal minimums, no correction is performed in the adjacent variable nodes. In the waterfall region, floating-point svwMS and vwMS perform very close to NMS with only 0.03 dB loss, however, a 6-bit quantized svwMS has more than 0.2 dB coding loss compared to floating-point NMS. It was also shown that 8-bit quantized svwMS and vwMS can achieve a bit-error-rate (BER) of 10−15 without any error floor when NMS damping factor (α) is equal to 0.75 with some coding loss compared to floating-point NMS. In [15], a split-row algorithm was proposed to alleviate routing complexity in fully parallel LDPC decoders, relying on simplified and partitioned check nodes that do not find a global minimum. The original algorithm had a coding loss of up to 0.7 dB compared to the NMS algorithm. A more elaborate version of the split-row algorithm can perform within 0.1 dB of NMS. The simulated decoder is very power efficient, which is the result of simplified check nodes and routing, at the expense of some coding loss in the waterfall region and possibly an early error floor at low error rates, which has not been investigated.
This letter presents a new low-complexity relaxed Min-Sum (RMS) decoding algorithm with 2nd min emulation in the variable nodes. Simulation results show that the proposed Relaxed Min-Sum with 2nd min emulation (RMS-2ME) decoder with only 4-bit resolution has negligible coding loss in comparison with floating-point NMS and SPA in the waterfall region, which makes it different from more complex decoders in [10]–[12], [15].

III. SIMULATION RESULTS
The (2048,1723) RS-LDPC code [13] and the (1008,504) PEG-LDPC code [16] are used for evaluating the performance of the proposed RMS-2ME algorithm. In all simulations, the maximum number of iterations is set to 100 and random codewords are used. Channel noise is considered to be additive white Gaussian and at least 75 frames in error have been observed for each simulated point. It is important to note that while floating-point SPA (FP-SPA) is provably optimal for cycle-free graphs, it is not optimal for practical codes that have finite length and include many cycles in their graph representation. There are several iterative decoding algorithms that outperform FP-SPA in the waterfall region, including damped belief propagation [4], relaxed belief propagation [5], and dithered belief propagation [6]. Furthermore, FP-SPA with finite resolution and clipped messages suffers from numerical error in its check nodes, which is the consequence of repeatedly using tanh and arctanh in addition to messages that quickly become extremely large in the LLR domain at high SNRs. Numerical instability is not normally seen in Min-Sum variants as check nodes only output one of the inputs. Therefore, it should not be surprising that the introduced RMS-2ME and a few other decoding algorithms in Figures 1 and 2 outperform FP-SPA.
