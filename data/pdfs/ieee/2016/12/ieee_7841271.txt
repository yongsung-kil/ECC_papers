Reliability analysis of memory centric LDPC decoders under probabilistic storage failures

Alexandru Amaricai, Sergiu Nimara, Oana Boncalo University Politehnica Timisoara Timisoara, Romania
Email: alexandru.amaricai, oana.boncalo@cs.upt.ro

Emanuel Popovici University College Cork
Cork, Ireland Email: e.popovici@ucc.ie

Abstract—In this paper, we perform a simulated fault injection reliability assessment of memory centric flooded LDPC decoders affected by probabilistic storage errors. We investigate the error correction capability in terms of Frame Error Rate (FER) of faulty flooded Min-Sum decoder, under Binary Additive White Gaussian Noise (BiAWGN) channel model. We have injected all the memories, as well as only the memories associated to each type of message. The analysis indicates that the LDPC decoders are capable to correct errors affecting the memory blocks within the decoder. Furthermore, a graceful degradation of the error correction capability of the decoder can be observed with the increase in the error probabilities can be observed. The results show that, although the error rate is 3× smaller in the input LLR memory, they have the strongest impact in the overal reliability of the decoder.
I. INTRODUCTION
In the conventional models for communications, errors are assumed to appear only on the data transmission, while the hardware which performs the required signal processing - forward error correction (FEC), modulation/demodulation, MIMO processing, etc - is fault free. However, todays nanometer CMOS technologies add a wide range of reliability issues to semiconductor devices. These reliability issues are augmented by the aggressive supply voltage reduction, till near and subthreshold regimes of operation [1]. In this regions, CMOS logic gates may present a probabilistic behavior. Therefore, the reliability analysis of components such FEC coders and decoders is of major importance.
Today’s FEC in modern communication systems is performed using soft decision iterative decoders, based on Turbo or LDPC codes. These codes have shown improved error correction capability, close to the Shannon limit, for the Binary Additive White Gaussian Noise typical for the wireless communication channels [2][3]. The decoding is performed in an iterative manner, in several decoding iterations using message passing algorithms. In this paper, we evaluate the reliability of a flooded LDPC decoder architecture, which uses memories for message exchange between the processing nodes. Because the memory represents the most important feature of the proposed architecture, we investigate the error correction capability of the LDPC decoder affected by storage errors. The considered fault model is represented by the probabilistic fault, due to the timing violations in a flip-flop based memory. Latch and flip-flop memories are recommended

for usage when very low supply voltages are used, instead of the conventional SRAM [4].
This paper is organized as follows: Section II presents previous work regarding reliability analysis of LDPC decoding; Section III depicts the analyzed LDPC decoder architecture; Section IV is dedicated to the fault map corresponding to the decoder; simulation results are presented in Section V.
II. FAULTY LDPC DECODER ANALYSIS
In recent years, analysis of faulty LDPC decoders has been performed in a significant number of works. This type of fault tolerance evaluation has been performed mainly in the presence of probabilistic errors caused by timing violations, due to the need of increased clock frequency or reduced supply voltage. Analytical analysis has been carried in order to evaluate different LDPC decoding algorithms. The one step majority LDPC decoding has been analyzed in [5]. Density evolution has been derived for hard-decision noisy GallagherB [6]. The erroneous FAID decoders have been evaluated in [7]. MS LDPC decoder analysis for unreliable message storage has been performed in [8]. The error correction capability and the decoding perfomance of MS based (Min-Sum, Offset Min-Sum and Self-Correcting Min-Sum) decoders under probabilistic faults has been performed in [9][10][11]. Fault tolerant decoding algorithm based on probabilistic gradient bit flipping decoder has been proposed in [12]. In these works, the evaluation has been performed on C/Matlab LDPC decoder models, with simplistic probabilistic fault models considered. The work in [13] considers the timing error violations in a fully parallel Offset MS decoder. The probabilistic timing errors are modeled using a deviation model. A simulated fault injection based reliability evaluation has been performed on a partially parallel flooded LDPC decoder in [14]. The considered architecture consisted of processing units, barrel shifters for routing and memory blocks for message storage. The probabilities for each combinational primary output have been derived corresponding to their delay. The work in [15], evaluates the performance of the spatially coupled LDPC decoders under different types of memory faults, including stuckat. The analysis is mainly analytical, and density evolution is determined.

III. MEMORY CENTRIC FLOODED LDPC DECODER
A. Flooded Min Sum Decoding of LDPC Codes
LDPC can be described in terms of sparse bipartite graphs, also known as Tanner graphs. A variable-node and a checknode are connected by an edge if the corresponding entry of the parity-check matrix H is non-zero [2]. The QC-LDPC codes are obtained from a base matrix B [3]; the parity check matrix is obtained by the expansion of each element within the B by a z × z matrix, as follows: the −1 elements are replaced by the z × z 0 matrix, while the non-negative elements are replaced by the permuted identity matrix. The variable node degree - dv - represents the number of non-zero entries in a column of the H matrix, while the check node degree - dc represents the number of non-zero entries in a row. LDPC decoding is performed in an iterative manner, using beliefpropagation algorithms [2]. Decoding relies on exchange of messages between two types of processing units: variablenode units (VNU) associated to the columns in the H, and check-node units (CNU) associated to the rows. Decoding takes several iterations until a codeword is found, or the maximum number of iterations is reached. Regarding the scheduling strategies, we address flooding scheduling, where at each iteration, all CNUs and subsequently all VNUs pass new messages to their neighbors. The most frequently used LDPC decoding algorithm is represented by Min-Sum (MS) and its variants (Offset MS, Normalized MS or Self-Correcting MS). In MS, the VNU performs addition between the check node messages corresponding to the variable node, while at the CNU level, comparisons between the variable node message are performed. Frequent message sizes are between 3 and 6 bits; therefore, the arithmetic operations within the processing units are simple. We rely on the following notations: β - check node messages; α - variable node messages; γ channel input or input Log Likelihood Ratio (LLR).
B. Memory Centric LDPC Decoding Architecture
The straight-forward architecture for an LDPC decoder is represented by the implementation of its corresponding Tanner graph. The decoder would consist of a number of VNUs equal to the number of columns in H matrix and a number of CNUs equal to the number of columns. The main drawback of this architecture is represented by the very large number

of interconnection wires between the processing units, which leads to very high area. In order to reduce the cost of the decoder, serialization is applied. In this case, message storage is required. Therefore, a typical partially parallel flooded architecture would consist of processing units, an interconnection network - which for QC-LDPC is implemented using barrel shifters - and memory blocks. Approaches which eliminate the interconnection network have been proposed, by using memory modules for message passing [16][17][18] - Fig 1. This architecture is composed of the following components:
• Processing units - a number of VNUs equal number of columns in the B matrix, and a number of CNUs equal to the number of rows is employed; each VNU and CNU process their corresponding messages in a parallel manner - for a regular LDPC code with dv = 3 and dc = 6, the VNUs will process 3 β messages each clock cycle, while the CNU will process 6 α messages each clock cycle;
• γ and HD memory blocks - these memory blocks are used to store the input LLR messages and the hard decision bits ; the number of these modules is equal to the number of rows in B; the depth of these memories is equal to z, while the width is γ quant bits for γ memory and 1 bit for HD memory;
• α memory blocks - these memory blocks are used to store α messages; the number of blocks is equal to the number of non-zero entries in the B matrix; their depth is z, while their width is α quant bits; each VNU is connected to dv α block memories - performing writes -, while each CNU will read from dc blocks;
• β memory blocks - these memory blocks are used to store variable node messages; the number of blocks and the depth of each block is the same as for the α message memories, while their width is β quant bits; each VNU is connected to dv α memories - performing reads -, while each CNU will write to a number of dc β memory blocks;
• Control unit - this module generates the appropriate sequence of operations for the decoding process; it will generate the corresponding read and write addresses for the memories, as well as the control signals corresponding to the initiation or termination of decoding phases;
Several throughput optimizations have been performed for this type of architecture, which include overlapped message processing [17], vectorization, folding [18] or multiple codeword processing [16]. With respect to other flooded LDPC decoding architectures, this one presents low logic utilization, due to a small number of processing units and no barrel shifters for routing, while the memory footprint is dominant.
IV. PROBABILISTIC FAULT MAP
We have implemented the proposed architecture for an (3,6) regular LDPC code with 12 rows and 24 columns in the B matrix, and circulant size of 54. The processing is performed by 12 CNUs with 6 α inputs and 6 β outputs, and 24 VNUs with 3 β and 1 γ inputs and 3 α outputs. The messages are quantized on 4 bits. A number of 72 α, 72 β, and 12 γ memory blocks are used. The decoder performs 30 iterations.

We have considered the memory implemented with D flipflops. The failure probabilities have been extracted using SPICE simulations, for flip-flops operating at 0.3 V, implemented in 45 nm technology [1]. For this near threshold supply voltage, flip-flop and latch based memories are used, instead of the conventional SRAM based [4] [19]. Although near and sub-threshold functional SRAM memories can be designed, they require complex read and write access logic. An equal error rate per memory bit is applied to all three memory types. Because the α and the β memories have higher size with respect to the γ memory, the error rate in these memories is significantly higher. We have applied saboteur based simulated fault injection for each of the memory blocks. The saboteur has been applied after each read operation, with error probabilities applied to each read bit.
We have considered error probabilities for clock frequencies of 400 MHz, 450 MHz, and 500 MHz. These corresponds to error rates of 1.25 × 10−3, 2.4 × 10−3 and 4 × 10−3 per clock cycle, per memory bit. V. SIMULATION RESULTS
The simulations have been performed in Modelsim commercial simulator. A C++ transmission channel model has been used. This model consists of: (i) random word generator,(ii) encoder, which performs LDPC encoding to the random word, (iii) channel error model, which implements BiAWGN model, (iv) results processing, which compares the output of the decoder with the output of the encoder. A System Verilog simulation framework has been used in order to interface the injected LDPC RTL model with the C++ transmission channel model. The outputs of the decoder are compared with the output of the encoder. Therefore, we can compute the bit error rate/frame error rate (FER) metrics, which will indicate the error correction capability of the faulty LDPC decoder. VI. CONCLUSIONS This paper performed a simulated fault injection based analysis of memory centric LDPC decoders. We have investigated how the probabilistic failures of the memory blocks in the proposed architecture affect the LDPC decoder error correction capability. Simulation results indicate that graceful degradation in the decoding performance manifests in memory centric LDPC decoders with faulty storage elements. Results presented in this work are consistent with the one obtained in [14], as well as the analytical based reliability analysis [9][7].
