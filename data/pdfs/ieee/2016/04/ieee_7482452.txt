FPGA Architecture of Multi-Codeword LDPC Decoder With Efficient BRAM Utilization

Sergiu Nimara, Oana Boncalo, Alexandru Amaricai, Mircea Popa
Computer Engineering Department University Politehnica Timisoara
Timisoara, Romania sergiu.nimara@student.upt.ro, {alexandru.amaricai, oana.boncalo} @cs.upt.ro

Abstract— Implementation of Quasi-Cyclic (QC) Low Density Parity-Check (LDPC) decoder on FPGA devices has shown great interest in both wireless communication, as well as error correction for Flash memories. This paper presents an FPGA flooded LDPC decoder which uses multiple codeword processing for efficient memory utilization. It is based on a partially parallel implementation, which relies on memory blocks for message passing between the processing units. We obtain efficient memory utilization by packing multiple messages corresponding to multiple codewords into the same Block RAM word. The increase in throughput is linear with the number of processed codewords. The proposed LDPC decoder can process up to 9 codewords in parallel, for 4-bit message quantization, or up to 12 codewords, for 3-bit message quantization, without introducing significant memory overhead.
Keywords—LDPC; FPGA; BRAM; Min-Sum; multi-codeword; Flooded
I. INTRODUCTION
Low Density Parity Check (LDPC) codes represent a class of capacity approaching codes, with increased error correction capability for both Binary Symmetric Channel (BSC) and Additive White Gaussian Noise (AWGN) channel [1]. They are used in a wide range of wireless communication standards [2], and recently in NAND based Flash memories [3]. Due to the high demand for increased throughput, hardware implementations for LDPC decoders represent a natural choice, which are favored by their parallel nature.
This paper proposes to efficiently exploit the FPGAs’ Block RAM (BRAM) memories in partially parallel flooded LDPC decoder architectures by packing messages from different codewords in a single memory entry. Hence, multiple codeword processing allows important throughput gains, by increasing the level of parallelism of the LDPC decoder and processing units, without increasing the memory requirements. We exploit the partially parallel architecture proposed in [11][12][13], which use the memories as means of message passing between the processing units. A drawback of these FPGA implementations is represented by the low hardware usage efficiency (HUE) in the BRAM blocks. We address this problem by storing the extrinsic message associated to multiple codewords in the same BRAM. Thus, we may store messages of 9 codewords for 4-bit message quantization and

12 codewords for 3-bit message quantization with no significant increase in memory resources. FPGA logic resources cost increase is linear for the proposed approach, as the number of processing units is proportional with the number of processed codewords.
This paper is organized as follows. In section II, an introduction to flooded LDPC decoding is presented; related FPGA architectures are described in Section III; the proposed approach is detailed in Section IV; Section V is dedicated to results. The last section brings some concluding remarks.
II. FLOODED MIN-SUM LDPC DECODING
LDPC codes are defined by a sparse parity check matrix ‫ܪ‬ [1]. Based on this parity check matrix, a bi-partite graph, called Tanner graph, can be derived [5]. The Tanner graph contains two types of nodes: variable nodes, corresponding to the columns of the ‫ ܪ‬matrix, and check nodes, corresponding to the rows of the matrix. An edge in the Tanner graph corresponds to a non-zero entry in the parity check matrix. Decoding is performed using message passing algorithms, which rely on the exchange of messages between the variable node units (VNU) and check node units (CNU), in several decoding iterations.
The vast majority of standards use the quasi-cyclic LDPC (QC-LDPC) code, due to their efficient hardware implementation [6]. These codes are a sub-class of LDPC codes, which present highly structured parity check matrix, defined by blocks of circulant matrices. The QC-LDPC is defined by the base matrix ‫ܤ‬, while the ‫ ܪ‬matrix is obtained by expanding the base matrix, with an expansion factor ݉ as follows: െͳ entries are expanded by a square ݉ ‫ ݉ כ‬zero matrix, while non-negative entries are replaced by the identity matrix right shifted with the non-negative value in ‫ܤ‬.
Two types of scheduling can be employed in LDPC decoding: flooded and layered [4]. The flooded scheduling represents the classic convention of decoding, where at each iteration, all the CNUs, and subsequently, all the VNUs, pass the newly computed messages to their neighbours in the Tanner graph.

III. RELATED FLOODED LDPC ARCHITECTURES
The straight-forward architecture for a flooded LDPC decoder is represented by the implementation of the Tanner graph [8][9]. The advantage of this fully parallel approach is its high throughput; however, due to the high overhead in the interconnection network between CNUs and VNUs, it may not be suitable for practical hardware implementations. Serialization is used in order to reduce the resource usage of the LDPC decoders, with penalties in throughput. Due to

serialization, message storage is required. A typical partially parallel flooded architecture [10] for QC-LDPC codes comprises of processing units (VNUs and CNUs), interconnection networks (usually implemented using barrel shifters) and memory units.
An approach presented in [11][12][13] eliminates the interconnection network, by using memory modules for message passing. This architecture presents a number of VNUs equal to the number of columns in the ‫ ܤ‬matrix, and a number of CNUs equal to the number of rows in the base matrix. For each non-zero element in the ‫ ܤ‬matrix, two memory blocks are required: one for storing check-node messages and one for storing variable node messages. For the input LLRs, a number of memory blocks equal to the number of columns in the ‫ܤ‬ matrix is used. Each memory has the word size equal to message quantization – such as 4 bits -, while the memory depth is equal to the size of the circulant matrix ݉. FPGA embedded BRAM memory blocks have typical size of ͵͸ ‫כ‬ ͷͳʹbits for Xilinx RAMB18E primitive or ͹ʹ ‫ כ‬ͷͳʹ bits for the Xilinx RAMB36E primitive. Therefore, this approach has the drawback of having very low memory usage. One way to alleviate this problem is based on using dedicated LDPC codes, with very large expansion factor ݉, such as 256. This way, good usage in depth for FPGA BRAM is obtained. However, this approach has two drawbacks: (i) many wireless communication standards have smaller expansion factors (96 for WiMAX), (ii) it still has low memory usage with respect to the memory word size.
Several optimizations have been proposed in order to improve the memory usage for this architecture. Overlapped message processing has been proposed in order to have a single memory block for storing both the check-node messages and variable-node messages corresponding to the non-negative entries in the ‫ܤ‬matrix [11][12]. Waiting-time minimization algorithms have to be employed for avoiding memory conflicts and read-after-write data hazards. However, these algorithms do not output good solution for all LDPC codes, and thus, this technique may be code dependent.
Further optimizations are represented by vectorization and folding [12]. Vectorization represents the packing of several

This work proposed the processing of multiple codewords in parallel in order to increase the memory usage, as well as the throughput. The messages corresponding to multiple codewords are stored in a single memory word; parallel processing units are employed. By employing multiple codeword processing, we target increase in throughput, without impacting the FPGA memory requirements.
IV. MULTIPLE CODEWORD LDPC FLOODED ARCHITECTURE
In order to increase the BRAM block memory utilization, we propose to store in a BRAM memory word, messages from different codewords. Thus, for a 36-bits BRAM memory width, and a 4-bit message quantization, messages from up to 9 codewords can be stored in the same BRAM block memory.
Processing the messages from the different codewords will be performed by parallel processing units (VNUs and CNUs). Therefore, the number of VNUs will be equal with the number of columns in the base matrix multiplied by the number of processed codewords, while the number of CNUs will be equal with the number of rows in the base matrix multiplied by the number of processed codewords.
Regarding the control component of the decoder – which generates the memory addresses, the read/write enable signals for memory accesses, or the start processing signal for both VNUs and CNUs – a single one is employed for all the codeword processing. The shared control is due to the fact that processing multiple codewords in parallel requires identical sequence of operations from a decoding perspective. The proposed solution relying on multi-codeword is suitable for systems that require adaptable throughput. Codewords can easily be grouped in multiples of 8 bits and memory operations as well as datapath processing can be disabled / enabled as required by the system for these groups. Throughput for the proposed architecture can be doubled if a waiting time minimization algorithm is employed such as [11].
VI. CONCLUSIONS
This paper proposes a novel optimization technique for FPGA based partially parallel flooded LDPC decoders, which rely on memories for message passing. We employ parallel multiple codeword processing in order to increase the BRAM memory usage, as well as the decoding throughput. We pack messages from several codewords in a single memory word. This way, we efficiently use the BRAM memory word. The synthesis results indicated that processing up to 9 codewords for 4-bit messages, and up to 12 codewords for 3-bit messages, results in no significant increase in BRAM memory usage, while increasing the throughput by a factor of 9 or 12 respectively. With respect to other LDPC decoders which use memories for message passing [11][12], we employ one order of magnitude less BRAM blocks per processed codeword. Further optimizations may be obtained by employing waitingtime-minimization algorithms to reduce the memory requirements for extrinsic message storing.
