Fast Converging ADMM-Penalized Algorithm for LDPC Decoding
Imen Debbabi, Bertrand Le Gal, Nadia Khouja, Fethi Tlili, and Christophe Jego

Abstract—The alternate direction method of multipliers (ADMM) approach has been recently considered for LDPC decoding. It has been approved to enhance the error rate performance compared with conventional message passing (MP) techniques in both the waterfall and error floor regions at the cost of a higher computation complexity. In this letter, a formulation of the ADMM decoding algorithm with modified computation scheduling is proposed. It increases the error correction performance of the decoding algorithm and reduces the average computation complexity of the decoding process thanks to a faster convergence. Simulation results show that this modified scheduling speeds up the decoding procedure with regards to the ADMM initial formulation while enhancing the error correction performance. This decoding speedup is further improved when the proposed scheduling is teamed with a recent complexity reduction method detailed in Wei et al. IEEE Commun. Lett., 2015.
Index Terms—LDPC, linear programming, ADMM, convergence, iterative process, flooding scheduling, layered scheduling.
I. INTRODUCTION
T HE linear programming (LP) technique was first proposed as an alternative to iterative low-density parity check decoding (LDPC) belief propagation (BP) algorithm [2]. However, the huge need for memory and the high computation complexity made the LP decoding impractical for real implementations. With the advance of the ADMM technique [3], which is a conventional linear programming optimization approach, the LP decoding of LDPC codes was brought back to the sphere of iterative processing. Indeed, the work in [3] exploits the geometry of the LP decoding constraints and presents an algorithm with a message passing interpretation.
Recent works in [4] and [5] propose more enhanced ADMM algorithms - called penalized l1 and l2 ADMM - to improve the error rate performance. They demonstrate that they can perform better than both the standard LP and BP decoders through adding penalty terms to the objective function of the LP formulation. Besides, the ADMM technique is shown to have less execution time requirements than the traditional Sum-Product LDPC decoding method [5]. Then, the authors of [1] suggest to reduce the complexity of the ADMM l1 penalized decoder by suppressing the useless Euclidean projections. It enables decoding time savings [1] at high SNR values though it increases the memory complexity.
Hitherto, all the previous ADMM LDPC decoders [1], [4]– [7] were based only on the flooding scheduling. This has turned our research for a faster ADMM LDPC decoding approach to exploring other scheduling strategies of traditional LDPC

However, since the nodes’ processing is very different in the ADMM decoding compared to BP approach [8], [9] (e.g. the Euclidean projection [5] versus the Min-Sum approximation), the validity and the advantages of a layered schedule has to be evaluated in terms of error correction performance impact as well as computation and memory complexity reductions. Indeed, all these features have to be improved so as to enable advances of both software and hardware real time implementation of ADMM decoders.
In this letter, an algorithmic formulation of an ADMM horizontal layered LP algorithm is proposed for LDPC decoding. The experimental results for the studied codes show that the proposed layered scheduling outperforms the usual flooding based scheduling. First it is efficient to decrease the average number of decoding iterations while improving the error correction performances and minimizing the memory footprint of about 20%. Second, the layered schedule can be applied to the ADMM LP decoders without any negative impact on their error correction performances like error floors. Third, it reduces the computation complexity of about 40% with regards to the work given in [1]. Associated with the method in [1], the layered formulation provides up to ×2.3 decoding process speed-up with higher correction performance.
The remainder of this letter is organized as follows: Section II summarizes the ADMM algorithm principles for LDPC decoding. Section III compares the flooding scheduling with the layered one applied on the ADMM LDPC decoding. In section IV, some performance results are provided and discussed.

II. PRELIMINARIES
A (dv, dc) LDPC code C of block length N , having M check equations, is described by a parity check matrix H ∈ F2M×N (M ≤ N ) having dv ones at each column and dc ones at each row. It can also be represented by its Tanner graph where the variable nodes (VN) are indexed by I = {1, . . . , N } and the check nodes (CN) are indexed by J = {1, . . . , M}. Each one in the parity check matrix at the position (n,m) corresponds to an edge in the Tanner graph that connects V Nn to C Nm. Let Nv(i) be the index of the set of neighbors of variable node vi and Nc( j) be the index of the set of neighbors of check node c j . The dc j × N binary transfer matrix Tj selects from an N -vector the coordinates corresponding to the dc j neighboring variables of check j. The check polytope Pdc j is the convex hull of all binary vectors of length dc j with an even number of ones. C. Layered Scheduling for ADMM Algorithm

A layered scheduling for the BP decoding of LDPC codes was first introduced in [12]. Inspired by a Horizontal Layered presented in [9], we transformed the computation scheduling of the ADMM iterative LDPC decoder accordingly. Algorithm 2 depicts a formalization of the horizontal layered scheduling for the ADMM l2 penalized method. It merges Kernels 2 and 3 of Algorithm 1 into a single one (Kernel 2 in Algorithm 2) called “CN centric”. In fact, during each iteration, the CN nodes are considered one by one. For each CN, the incoming messages from its neighbor VNs are first computed (Algorithm 2, lines 8 : 10). Then, the outgoing messages towards these neighbors are generated (Algorithm 2, lines 12 : 16). At last, the current check node’s neighbors can update their intermediate sum ti of incoming messages. This last VN update (Algorithm 2, line 19) helps spreading reliable messages to the next CNs. Consequently, the decoding convergence would be speeded-up. The syndrome checking which is used to stop the decoding process once a valid codeword is found, is executed after each decoding iteration.
Theoretically, the horizontal layered ADMM decoder has a higher computation complexity than that of the flooding based formulation when the same amount of decoding iterations is performed. It comes from the fact that for each CN, dc messages from VN are computed on the fly. Unlike the BP LDPC formulation, the ADMM VN processing requires few more computations. In the next section, we show that the amount of decoding iterations required to converge to a valid codeword is much lower for a horizontal layered scheduling. This leads to a meaningful computation saving compared with the flooding based approach and also when compared with complexity optimized flooding decoder in [1]. Moreover, [1] technique can be successfully incorporated in the proposed layered formulation to improve the processing efficiency.
Concerning the memory footprint of the layered based decoder, it is lower than that of the flooding based ones. Indeed, these laters need at least 2 × (N + M) memory elements while the version provided in [1] requires N + 3 × M memory elements. N is the number of VNs and M is the number of exchanged messages. In the horizontal layered formulation, only N + 2 × M elements have to be stored.
To sum up, the horizontal layered formulation enables a perceptible computational and memory complexity reductions compared with the original flooding based scheduling [7], [11]. These improved features make this algorithm more compatible with real time hardware of software implementations.
IV. EXPERIMENTAL RESULTS
In this section, the impact of the scheduling on the error correction performance is illustrated. Then, the convergence speed enhancement is compared with the results of the flooding scheduling for the ADMM l2 penalized decoding. A final comparison is performed in terms of average decoding time.

Fig. 1. FER comparison of the ADMM l2 penalized decoders (F: flooding scheduling, HL: horizontal layered scheduling) for C1 and C2 LDPC codes.
For the flooding scheduling, the original ADMM l2 penalized decoder from [6] was considered. It was slightly modified to handle floating point data format and minimize memory usage and access so as to bring out the flooding scheduling. Besides, the approach of using different parameters for variable nodes with different degrees was included. Hence, parameters α, ρ and μ were chosen according to [7]. Finally, an optimization described in [1] which enables to reduce the number of projections, was also added1. For the Monte Carlo simulations, the two LDPC codes considered in [7] were chosen: 576 × 288 rate-1/2 LDPC code (C1) and 1152 × 288 rate-3/4B LDPC code (C2). Both codes are used in the IEEE 802.16e standard. All the simulations were performed over an additive white Gaussian noise channel with binary phase shift keying.
A. Error Correction Performance
Figure 1 shows the frame error rate performances (FER) of C1 and C2 codes for two ADMM l2 penalized decoders. The first decoder makes use of a flooding scheduling while the second is based on a horizontal layered scheduling. The maximum decoding iteration number is set to 200. It can be seen that the decoding performance of the horizontal layered based decoder is better than the flooding based one for both LDPC codes. Error correction performance improvement reaches 0.05 dB for C2 and 0.25 dB for C1. Hence, the horizontal layered scheduling formulation speeds-up convergence without any error floors.
B. Comparison in Terms of Convergence Speed
In a second set of experiments, the average decoding numbers of iterations were measured for the two computation schedulings when the maximum number of iterations is set to 200. The results exhibited in Figure 2 show that in the waterfall region, a saving ranging from 5% to 35% is reached while the error correction performance is improved. A gain of 30% is still observed in the convergence region. As expected from the analysis given in Subsection III-C, the spreading of reliable messages helps speeding-up the decoding convergence.
In order to emphasize the impact of the convergence speed on the error correction performance, we compared the frame error rate of the layered scheduling to those of the flooding scheduling at SNR = 3.5 dB while varying the maximum number of iterations. V. CONCLUSION
In this letter, an iterative horizontal layered scheduling LDPC decoder based on the penalized ADMM LP approach was detailed then tested on two WiMAX codes. Simulations show that it converges considerably faster than the flooding based scheduling one. It enables the decoding acceleration of up to ×1.9. When associating the complexity reduction approach described in [1] with the horizontal layered scheduling, the decoding accelerating factor reaches up to ×2.3. It can be considered as a promising solution for future hardware designs for real time LDPC decoding.
