Evaluation of the Hardware Complexity of the ADMM approach for LDPC decoding

Imen DEBBABI, Nadia KHOUJA and Fethi TLILI GRESCOM Laboratory, SUPCOM Carthage University, Tunisia.

Bertrand LE GAL and Christophe JEGO IMS Laboratory
Bordeaux Polytechnic Institute, Talence, France.

Abstract—Linear Programming (LP) is a novel technique for LDPC decoding. With the advance of the Alternate Direction Method of Multipliers (ADMM) approach, a significant step towards LP LDPC decoding scalability and optimization is made possible. Yet, this innovative decoding technique has not been implemented in hardware. Its hardware complexity has neither been estimated nor compared with traditional techniques. In this paper, an overview of the ADMM approach and its error correction performances for LDPC decoding is provided. Then, its computation complexity is evaluated to show the hardware feasibility of ADMM-based LDPC decoders. Our analysis is mainly carried at two levels. First, a quantitative complexity analysis is reported and a comparison with traditional LDPC decoders is given. Second, a proposal of a partially parallel architecture is described and its hardware complexity is evaluated then compared with state-of-the-art LDPC decoders.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes are considered among the best error correcting channel codes known today. Invented by Gallager [1] then rediscovered by MacKay [2] and enhanced to irregular LDPC codes by Richardson et. al. [3], they are now used for forward error correction in many digital communication standards like DVB-S2, DVB-S2X, WiMAX, WiFi and WRAN.
The main advantage of LDPC codes is that they can perform close to Shannon limit when coupled with the Sum Product message passing Algorithm (SPA) though they still depend upon intensive calculations. To reduce the complexity issues, other message passing (MP) methods namely Min-Sum (MS) are preferred [4], despite their error correction impact.
Recently published works drew benefit from Linear Programming (LP) approaches, primarily invented by Feldman [5], and proved these techniques to be more efficient in terms of error correction performance for LDPC codes decoding. The work in [5] establishes that the LP decoding outrivals the classic MP decoding for LDPC codes since it has the Maximum Likelihood (ML) certificate property; meaning that if the solution of an LP decoder is integer then it is the ML solution. Besides, MP algorithms are not guaranteed to converge and are subject to error floors while LP decoding has not been observed to encounter the same problem for some practical long block length codes [6].
A more recent research work of Barman et. al. [7] proposes the Alternating direction method of multipliers (ADMM), which is a classic linear programming optimization technique, for the purpose of LP LDPC decoding. The authors exploit

the geometry of the LP decoding constraints and develop a decoding algorithm that has a message passing interpretation. Recent works [8] [9] proposed enhanced ADMM decoders to improve the error rate performance and demonstrate that they can perform better than both standard LP and SPA decoders.
The so far developed on chip LDPC decoders are based on decoding algorithms premised upon the MP techniques and their MS variants thanks to their low hardware complexity. The recently published works that were concerned with Linear Programming (LP) approaches, proved that such techniques are more efficient in terms of error correction performance for LDPC codes. However, the huge need for memory and the high computation complexity make the LP decoding impractical for both hardware and software implementations.
In the current article, a first analysis of the ADMM decoding algorithm complexity compared to commonly-used SPA and MS decoding techniques is presented. The objective is to demonstrate whether this novel approach is likely to be implemented in hardware unlike previously proposed LP decoding techniques based on solvers. In fact, even though the algorithm complexity seems comparable to MP decoding approaches, it is still a challenging task to implement it in hardware due to the complicated check node computation which needs an intricate projection operation.
This paper is organized as follows: Section II summarizes the ADMM algorithm principles for LDPC decoding. In Section III, the ADMM computation and memory complexities are compared to traditional MP techniques. A first hardware complexity evaluation of check nodes and variable nodes processing elements is provided in Section IV. Conclusions and future works are finally detailed.

Kernel 2 and Kernel 3 have the highest computation complexity. They are executed N × q times and (N − K) × q times, respectively. q is the number of decoding iterations. Kernel 4 is where the hard decisions cˆn are obtained. This kernel is executed only once at the end of the decoding process.

B. Traditionnal LDPC decoding algorithm
The SPA [11] is currently the most efficient LDPC decoding algorithm based on the message passing technique, in terms of error correction performance to decode LDPC codes. To simplify its computation complexity and facilitate implementations of both hardware and software LDPC decoders, approximations of the SPA called MS variants were proposed in [12]. The message passing scheduling for CN and VN computations is called the flooding scheduling. It consists in an iterative process in which all the messages from VNs to CNs are computed simultaneously, then the same processing is performed for the messages from CNs to VNs. Details about the flooding based scheduling can be found in [13].

C. Error correction performance
According to previous works, the ADMM decoding approach provides interesting error correction performance improvement even when compared to SPA decoding algorithm as demonstrated in [9]. Moreover, it can provide ML certificate to decoded frames. Examples of the achieved decoding performance are provided in Figure 1. For all the FER curves, the maximum amount of decoding iterations is set to 200, an AWGN channel is used for the two WiMAX standardized LDPC codes. The parameters of the ADMM-l2 penalized decoder are obtained by the parameter generator given in [14].
From Figure 1, it can be seen that the ADMM-l2 penalized decoder performs better than the optimal SPA decoder. For example, at FER=10−5, the performance gain for the ADMMl2 decoder is 0.3 dB for the 1152×288 code. The performance gain for the ADMM-l2 decoder is 0.5 dB for the 576 × 288 code at FER=10−4.
III. COMPLEXITY COMPARISON
A. Computational Complexity
In this section, a complexity evaluation of the ADMM decoding algorithm computation is provided at VN and CN levels. Hopefully, this evaluation is intended to be the fairest.
The numbers in Table I represent the amount of computation of each operation type. Floating point data format is considered by default. Data address computations and indexing are voluntarily ignored. Besides, the computation complexity of MS, SPA algorithms are provided for comparison.
At first sight, these values indicate that the ADMM algorithm’s complexity for VN processing appears slightly higher

than SPA and MS. This is due to the use of multiple multiplications and divisions required to inject the over relaxation parameter ρ. Concerning the CN processing, MS algorithm is much less complex than ADMM algorithm. The comparison between SPA and ADMM seems to be more intricate. In fact, even though SPA needs two transcendental costly functions, ADMM needs a single - though complex - projection operation. Both arctan and projection operations are not atomic which justifies the difficulty to provide fair comparison. However, the key part to solve constraints optimization problems with ADMM is the method used to project a vector of real values on the constrained convex space, known as the fundamental polytope for LP decoding [9]. In [15], Barman et. al. proposed a projection algorithm, which was further improved in [6]. For a given vector, the algorithm sorts first its coordinates in descending order and characterizes two slices whose convex hull contains the projection. Then, the projection of the ordered vector is found through solving a quadratic program. However, the ordering of all coordinates that is required to project any given vector increases the complexity of the algorithm. From a deep analysis of the projection algorithm, that requires many multiplications and data sorting processing, one can conclude that ADMM CN kernel complexity is higher than SPA. Moreover, some parts of the processing do not provide parallel computation opportunity.
To sum up, the ADMM decoding algorithm has a higher complexity than MS and SPA decoding algorithms at CN and VN levels. Consequently, the decoding complexity is higher at decoding iteration level and also at full decoding level when the same number of decoding iterations is executed.
B. Memory Complexity Comparison
Along with the computational metrics, a second hardware complexity metric useful to compare the LDPC decoding approach complexity is the memory footprint required by the decoding processes. For SPA and MS algorithms implemented using the flooding scheduling algorithm, the memory footprint δ provided in terms of elements required by the decoding process is given by δSP A = N +2×m; with N the code length and m the number of exchanged messages between CN and VN nodes. However, in the implemented hardware decoders, this memory cost is higher in order to overcome the decoding latency as in [16], [17]. It is equal to δSP A = 2 × (N + m).
For the ADMM decoder, the memory footprint required by the decoding process is given by δADMM = 2 × (N + m). It is equal to the memory footprint of the implemented MS and SPA decoders. Indeed, as shown in Algorithm 1, N channel values, N CN to VN messages, two message sets named zj(t) and λ(jt) which are composed of m elements, are needed.
Ultimately, in terms of memory complexity, the ADMM decoding algorithm needs the same memory storage elements as the traditional LDPC decoding approaches.

IV. ARCHITECTURAL COMPARISON
The theoretical evaluation presented in Section III shows that the ADMM algorithm has a higher complexity than the traditional SPA and MS approaches. Yet, the ADMM’s error correction performance counterbalances its complexity. In this section, a proposed architecture for an ADMM LDPC decoder is depicted and compared to state-of-the-art MS LDPC decoder. The hardware complexity of both decoders is also evaluated at coarse grain level.
A. Related works on flooding based implementation of LDPC decoders
Several studies were interested in hardware implementing LDPC decoders which can be serial, partially parallel, or fully parallel. Serial architecture’s throughput is usually very low and insufficient for most standard applications [18]. A fully parallel architecture as in [19] can reach maximum throughput, but suffers from huge connection complexity. Besides, fully parallel architectures are usually built on custom silicon, which precludes any prospect of reprogramming.
The partially parallel architecture divides the node operations of Tanner graph over p processing elements, with p < (N + M ). These architectures are studied extensively and provide a good trade off in throughput, complexity and flexibility.
The structure of the architecture in Figure 3 described in [16] is partly parallel, thus only a subset of nodes in the Tanner graph is instantiated as variable node and check node processors. The node processors work in a serial manner which gives the needed flexibility regarding the variable node and check node degrees.
In the two-phase decoder, each VN node contains four RAMs to store the channel information, to manage and store the sum of messages from previous and current iteration. According to [18], the CN processing element can be designed to be serial, with internal pipelining to maximize throughput, or parallel processing all data concurrently. These two design approaches are depicted in [18]. Finally, all messages are exchanged via two Barrel Shifters.

The proposed ADMM decoder block diagram is shown in Figure 4. It consists of p elements for the VN processing and p elements for the CN processing. Each processing element performs the computation associated to multiple nodes. According to [18], partially parallel architectures are studied extensively and provided a good trade off in throughput, complexity and flexibility. Each VN consists of two block RAM as shown in the structure depicted by VN1. The first stores the channel information while the second stores the updated messages computed for CNs for the current iteration. At the kth iteration of the decoding process, the updated messages during the previous iteration from the neighbor check nodes are accumulated. The channel information is then extracted. Subsequently, the result is multiplied by a penalty term then clipped to [0, 1] (Algorithm 1, lines 8 : 10). The final computed message relative to each neighbor check is then fed to the CN processor.
On the other hand, the CN processor consists of two block RAM in which λ and z values are stored. At the kth iteration, the updated values from the iteration k − 1 are drawn from the memory blocks. Then, they are consumed to provide the updated values of z and λ (Algorithm 1, lines 14 : 15). These values replace the old ones once fed back to the memory.
The controller which manages the CN and VN Units as well as the RAMs access is similar to the MS decoder’s controller. However, the Barrel Shifters would be more complex due to the higher data exchange. To sum up, the proposed architecture for the ADMM LDPC decoder shares many similarities with the recently published ones for MS LDPC decoders. However, more messages are exchanged between the CN and VN processor units. As for the memory usage, the total number of memory blocs is the same for both ADMM and MS LDPC decoders. It is important to note that floating point data is manipulated for the ADMM decoder unlike the state of the art MS decoders which are implemented using fixed point data representation.
For performance sake, investigations haven’t been yet oriented towards fixed point data representation for the ADMM

LDPC decoder, unlike related work on traditional LDPC decoding techniques [22]–[24]. In fact, a first study of the fixed point quantization impact on the error rate performance should be carried on. For this reason, only a preliminary floating point evaluation is done. Subsequently, CN and VN processing units are more complex for the ADMM LDPC decoder which is likely to have less throughput than the MS decoder; notwithstanding, its error correction performances and its rapid convergence counterweight this complexity.
C. Synthesis Design estimation
High Level Synthesis (HLS) tools are interesting means to accelerate IP creation by enabling high level specifications to be directly targeted into programmable devices without the need to manually create RTL [25]. In [26], the feasibility of HLS tools for implementing efficient LDPC decoder architectures was exploited. Moreover, in [27], a novel wide-pipeline LDPC decoder approach was proposed for execution on fieldprogrammable gate arrays, using a high-level synthesis tool to reduce the development effort and design validation time. Finally, in [28], an FPGA accelerator for non-binary LDPC decoders was designed through Vivado HLS tool.
Similarly, in this work, the design estimation of CN and VN processing units for the proposed ADMM LDPC decoder is done through the Vivado HLS tool. The target synthesis device is set to Zynq device (Zedboard). Three examples of degree 6 CN and degree 3 VN were synthesized for three different LDPC decoders using SPA, MS and ADMM algorithms. Both SPA and MS implementations are fixed point while the ADMM is a floating point one. In Table II, the logic resource utilization of the target FPGA is shown. The ADMM CN processor consumes 1.7 times more LUTs and 1.4 times more flip flops than the SPA CN processor while their latencies are comparable. As for variable nodes, the ADMM VN processor consumes 1.4 times more LUTs and 1.6 times more flip flops than the SPA VN processor and has twice its latency.

V. CONCLUSION
In this paper, the iterative ADMM penalized algorithm used for LP decoding of LDPC codes is described. When compared to MP-SPA LDPC decoder, for different studied LDPC codes, it provides 0.5 dB improvement in error correction while converging faster in the waterfall region. An analysis of the computation complexity and the memory footprint of the ADMM algorithm is provided and compared with the SPA and MS approaches. Further, an on chip implementation of the ADMM algorithm for LDPC decoding is evaluated. The proposed partially parallel architecture shares many similarities with the state-of-the-art MS LDPC decoders, though the hardware cost is found to be higher. The complexity study of basic blocs hereby presented is a first step towards a fixed point FPGA implementation. These optimistic results are motivating for a future work that will focus on fixed point data representation to reduce hardware and memory complexities.
