Improved LDPC Decoding Algorithms based on Min-Sum Algorithm

Dept. of Electronic and Telecommunication Engineering University of Moratuwa Moratuwa 10400 Sri Lanka

Abstract—Low-Density Parity check (LDPC) codes offer highperformance error correction near the Shannon limit which employs large code lengths and some iterations in the decoding process. The conventional decoding algorithm of LDPC is the Log Likelihood Ratio based Belief Propagation (LLR BP) which is also known as the 'Sum-Product algorithm' which gives the best decoding performance and requires the most computational complexity and implementations with increased hardware complexity. Another simpler variant of this algorithm is used which is known as 'min-sum algorithm' which reduces computational complexity as well as hardware complexity but with reduced accuracy. This paper analyzes the reason min-sum algorithm is more prone to errors when compared to the sumproduct algorithm, and puts forward two improved algorithms which improve the performance of the min-sum algorithm with comparable algorithmic complexity.
Keywords—LDPC, decoding, sum-product, min-sum, hybrid
I. INTRODUCTION
Low-Density Parity Check code is a family of linear block codes discovered by Gallager in 1960s [1]. It falls under the category of modern codes with turbo codes and repeat accumulated codes. For a long time since the discovery, LDPC codes were ignored due to computational power requirement during the decoding process. In 1981, Tanner gave a graphical interpretation of the decoding architecture of LDPC codes but still did not become a part of communication at that time. After about a decade later in 1990s D.J.C.Mackay and M.Neal reinvestigated LDPC codes and found that when large code sizes are used, it can achieve good Bit Error Rates(BER) rates approaching the Shannon limit[2].Since then LDPC has been well spread and much research has been conducted. These have led LPDC codes to be standardized in communication protocols like DVB-S2 and integrated with many other communication methods.
The performance of LDPC codes are achieved due to several factors. Mainly due to the factors in the code properties and factors in the decoding process. LDPC codes consist of paritycheck matrices which contain mostly zeros and only a small number of ones. LDPC decoding process is iterative and is soft decisions based or hard decision based. Whole decoding scheme can be paralleled resulting in very high throughput

reaching in Gbit/sec. The complication of an efficient implementation of such decoder is the interconnect complexity which results from random 1's in the large parity check matrix. Tanner graphs are used to illustrate the parity check matrix properties which consist of check nodes and variable nodes. Interconnections between these nodes depict the parity check matrix properties.
There are several types of LDPC decoding algorithms. Decoding algorithms are classified into hard decision-based decoding algorithms and soft decision based decoding algorithms. Hard decision-based algorithms use binary decisions about each received bit made by the detector and use binary arithmetic operations to decode the code words hence low complexity decoders. Since 1960’s several hard decisions based algorithms have been put forward, and Gallager’s version was named as Bit-Flipping algorithm. Bit flipping algorithm suffers from significant performance loss. Hence, soft decision based algorithms have been put forward. The mainly used soft decision-based algorithm is the Sum-Product algorithm (SPA). It uses Log Likelihood Ratios of input data samples from the channel in the decoding process. The best algorithm available is the Sum-Product Algorithm (SPA) but it needs heavy computational power. Therefore, most commonly used algorithm is the Min-Sum algorithm (MSA) which is an approximation of the SPA, used with a performance loss. MSA is widely used in LDPC hardware decoders. Most of the research has been done to improve the performance of the MSA by modifying the algorithm with various techniques to reach a performance near SPA. In literature we could find various techniques like regression and specialized LDPC parity check construction methods that improve MSA’s performance. But none of them have done theoretical analysis of MSA’s performance degradation. In our study we will theoretically analyze the difference between these two algorithms and proposes two new algorithms named Hybrid Algorithm and Alternative algorithm that improve the performance of the MSA with very little computational complexity. These algorithms can be applied to any kind of LDPC codes without any specialization. Alternative algorithm is an extended version of MSA and Hybrid algorithm is an efficient implementation of

For LDPC codes, it has been observed that intermediate LLR

values of LDPC decoding processes tend to oscillate before

converging. Based on that the authors have divided the

decoding process into two stages. The first stage is similar to

normal routine, but with a limited number of iterations (τ )

which will be determined using simulations for each parity

check matrix. If the decoding process reaches τ then stage two

of the process gets started. It divides the variable nodes into two

groups, and the signs of the two groups are changed in a

different manner than normal procedure. The Algorithm does

not affect the BER performance of the code, but states that it

can decode heavily distorted code words quickly than normal

algorithms. The throughput is slightly increased by this method,

but it has not well discussed in the paper. A similar approach is

taken in [7] to avoid an unnecessary number of iteration usage,

which is known as a self-correction algorithm. MSA algorithm

is modified such that unreliable variable node information is

detected using sign fluctuations, and those bit node values is

erased and set to zero. This prevents unnecessary decoding

iterations as well as improving the BER performance almost up

to 0.1dB. In [8] a memory efficient MSA algorithm has been

proposed. When code-words become large, the memory

handling will be an issue. An LLR value should be expressed

using 4-5bits where thousands of such units are employed. So

each intermediate LLR is expressed using two bits, where the

1st bit is the hard decision on a received bit, and 2nd bit

represents the confidence level of the received bit. The

algorithm is simpler than MSA, and the complexity is also

reduced. Hardware implementation of the design is given in the

paper, but the algorithm does not perform as MSA. Another

customized solution has been put forward in [9] which is called

Block-LDPC where LDPC code construction, encoding

algorithm, and decoding algorithm is unique. Details have been

provided in the paper and consists of complex mathematical

operations. An improvement of BER is shown using

simulations, but complete error correcting process is complex

when compared with other algorithms. Most of the papers have

put forward methods altering the decoding process but have not

discussed the reasons for MSA's performance degradation.

V. HYBRID APPROACH

The difference between MSA and SPA gets severe when more and more LLR’s fall into critical region. So as a remedy we propose three rules to follow during check node calculation.
1. When no LLR falls into the critical region(x< 0.89) and all the LLR falls into safe region, MSA approach is taken.
2. When only a single LLR falls in the critical region, still the MSA approach is taken.
3. When more than one LLR falls in the critical region,
the sum of φ values of those is taken.

By following hybrid approach, we can make sure that the difference between MSA and SPA is minimized since we take the sum of the LLR's fall in the critical region.
