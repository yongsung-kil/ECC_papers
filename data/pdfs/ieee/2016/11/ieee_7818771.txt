Error Floor Reduction for LDPC Codes Using Self-Correction Technique

Evgeny Likhobabin, and Vladimir Vityazev, Member, IEEE

Abstract—In this work, we introduce a very simple technique that improves the performance of the belief propagation (BP) and BP-based decoders in waterfall and error-floor regions. Unlike other techniques known in literature, this technique does not explicitly use any trapping sets elimination algorithm, but it modifies the variable node processing by erasing unreliable messages. However, this modification positively affects on the overall decoder performance, and reduces the error floors. At the same time self-correction have insignificant contribution in total computational complexity of LDPC decoder. Moreover it is shown, that such a modification can be used for any BP-based decoding algorithm.
Keywords—belief propagation, LDPC, self-correction.
I. INTRODUCTION
Low density parity check (LDPC) codes was proposed by R.Gallager [1] in 1963, and later rediscovered by MacKay and Neal [2]. Like turbo codes, LDPC codes can achieve near the Shannon limit error performance [3], and represent a very promising prospect for error control coding. Nowadays they are widely used in many applications, all modern standards such as DVB-T2, DVB-S2, DVB-C2, WiFi, WiMax, IEEE 802.15.3 used this codes.
Gallager also proposed algorithms for decoding LDPC codes. The most popular of them is soft decision based, near optimum belief-propagation (BP) algorithm, also known as sum-product algorithm (SPA), and a hard decision based bit-flip (BF) decoding algorithm. They are iterative decoding algorithms.
The main advantage of BP decoding algorithm is high performance, that allows to achieve near the Shannon limit, and the main drawback in it is complexity, that remains relevant nowadays for applications with limited computational and energy resources. The BF algorithm has a reverse disposition. The main advantage is simplicity and the main drawback is poor performance. Due to high complexity of BP decoding algorithm, many authors try to compile advantages of BP and BF algorithms in one algorithm, and a lot of modifications and new ways of decoding LDPC codes have been proposed from its invention. The most significant ways of LDPC decoding evolution are simplifying BP algorithm with loss in performance, and complicating BF algorithm with gain in performance. The other significant drawback of LDPC codes is the presence of error-floors in BP decoding. It is limit their usage in applications that needs extremely low error rates. The errorfloor can be characterized as an abrupt decrease in the slope of a code’s performance curve. The error-floors were first discovered by MacKay and Postol in [9] for (2640,1320) Margulis code. It was shown that the performance of LDPC codes with message-passing decoding in the low-error-rate region is determined by the multiplicity and structure of the trapping sets [10], rather then minimum distance of the code. We can reduce trapping set influence to the performance of decoder in error-floor region using specially designed algorithms [11], [12] that find and erase nodes in code graph, that belong to trapping sets. This improvement needs additional computational cost on each iteration of decoding, and may be complex enough.
In [8] was shown, that self-correction technique may be used not only for MS algorithm, but also for UMP-APP decoding algorithm [4].
In this paper we proposed to use self-correction (SC) technique from [5] for BP algorithm and it modifications in assumption that it improves performance in error-floor region by erasing unreliable messages, entering into trapping sets.

Any LDPC code is specified by a parity-check matrix which contains mostly zeros and only a small number of ones. A regular (N,K) LDPC code, with block length N and dimension K, has a parity check matrix H of M ≥ N - K rows and N columns with exactly dc ones in each column and dr ones in each row. III. BP-BASED DECODING ALGORITHMS
A. Belief propagation decoding algorithm
Belief propagation (BP) decoding algorithm was proposed by Gallager [1]. It is a suboptimal decoding algorithm, that can achieve maximum-likelihood performance if the Tanner graph [14] of decodable code have no cycles. Unfortunately graphs for finite-length codes contain a large number of cycles. Belief propagation decoding algorithm has high computational complexity as far as utilize transcendental functions for check node processing. So consistent decision is simplify Gallager’s BP decoding algorithm with control of decoding performance degradation.
B. Min-sum decoding algorithm
Min-sum algorithm (MSA) [4] is one of the most popular decoding algorithms for LDPC codes. MSA can be considered as a simplification of operation for log-likelihood ratios (LLR) summation in parity-check node decoder. For two independent binary random variables a1 and a2 with probabilities p(ai = x) = p(xl), x ∈ {0, 1} and LLRs Li = L(ai) = log(p(0l)/p(1l)). It can be shown, that LLR of the binary sum A2 = a1 + a2, defined as:
L(A2) = sign(L1)sign(L2)min(|L1|, |L2|) + g(L1, L2).
For MSA only the first part of this sum computed:
L(A2) = sign(L1)sign(L2)min(|L1|, |L2|).
C. A-min* decoding algorithm
Although MSA has a very attractive complexity, but in some applications error performance degradation in comparison with BP decoding may be unallowable. In such a case a-min* [6] algorithm can be very useful, as a combination of BP and MSA decoding. Algorithm a-min* reduces complexity of check nodes update in comparsion with BP decoder. Consider min-sum approximation
L(A2) = sign(L1)sign(L2)min(|L1|, |L2|) + g(L1, L2) =
= sign(L1)sign(L2)min∗(L1, L2). For MSA decoding operator min* is equal to
min∗(L1, L2) = min(|L1|, |L2|).
And for a-min* algorithm
min∗(L1, L2) = min(|L1|, |L2|) + g(L1, L2).
For a-min* algorithm as is for MSA, in every check node only two outgoing messages computed. The first one is for bit with the lowest reliability, and the second one for all other bits.

VI. CONCLUSION AND FUTURE WORK
Applying of SC to BP and any BP-based algorithms has a sense to be made. Such modification have performance gain both in waterfall and error-floor regions. It is significant that amount of error-floor reduction may vary for different BPbased algorithms. And the best results can be achieved if rough approximations of BP check node processing is used.
