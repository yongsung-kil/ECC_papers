Abstract— NHK (Japan Broadcasting Corporation) is conducting research on the next generation of digital terrestrial broadcasting system to enable large-volume content services such as UHDTV (ultra-high-definition television). In our previous study, we proposed spatially coupled low density parity check (SC-LDPC) codes for broadcasting. The lengths of these codes can be easily extended and are expected to have good performance. If SC-LDPC codes are implemented in hardware, it is necessary to reduce the decoding calculation complexity and the hardware volume. This paper reports a parameter optimization for the decoding process and log likelihood ratio (LLR) quantization. Offset min-sum decoding was used in order to reduce the calculation complexity while maintaining performance, and the offset value was optimized in a computer simulation. The quantization bit depth of LLR was also determined with a computer simulation. The results were incorporated in the hardware implementation of the SC-LDPC codes, and a laboratory experiment was conducted with a prototype modulator and demodulator for broadcasting. The BER performances measured in the laboratory experiment and computer simulation were compared, and the relative deterioration of the hardware implementation was under 0.1 dB.
I. INTRODUCTION
Japan started 4K test broadcasting by satellite in 2014, and will start 8K test broadcasting by satellite in 2016. To enable UHDTV terrestrial broadcasting such as 4K and 8K, NHK is conducting research on large-capacity transmission technology. As part of this development, we have developed large capacity transmission technologies using higher-order modulation (e.g., 1024QAM or 4096QAM) orthogonal frequency division multiplexing (OFDM), dual-polarized multiple-input multiple-output (MIMO), and low-density parity-check (LDPC) codes [1]. LDPC code is a robust error correction code that approaches the Shannon limit, and it has already been incorporated in various broadcasting standards

such as DVB-T2 [2], i.e., the second generation of digital terrestrial standard in Europe, and ARIB STD-B44 [3], i.e., the satellite standard in Japan. We are studying LDPC codes for better performance in order to broadcast large-volume content such as UHDTV.
SC-LDPC code was recently proposed in the previous studies [4]. This sort of code is based on LDPC convolutional code [5] and has a cyclic structure that is composed of a base matrix. The matrix of SC-LDPC code is able to be optimized by optimizing the base matrix. Therefore, it is easier to produce a parity-check matrix that has good performance than to produce a parity-check matrix of conventional LDPC codes. For this reason, the code length of SC-LDPC can be easily extended. Our research focused on this characteristic, and its goal was to design SC-LDPC codes whose length is longer than that used in the conventional standard and to improve the performance of error correction. We previously designed SCLDPC code whose length is about 260,000 bits, four times as long as that used in DVB-T2, and confirmed that it has good performance in a computer simulation [6]. On the other hand, in order to implement very long LDPC code, it is necessary to reduce the calculation complexity and hardware volume. Another previous study reported a verification of SC-LDPC codes for ultra-high-speed optical communications [7]. But there is no previous report on SC-LDPC codes for broadcasting, or longer LDPC codes, about 260,000 bits, than that of conventional codes. Here, we report on the design of SC-LDPC codes about 260,000 bits long and evaluate the characteristics of a hardware implementation intended for broadcasting. In this paper, first, we briefly explain the design of our SCLDPC codes. Second, we explain the decoding method, using the TDMP (Turbo Decoding Message Passing) algorithm [8]

Third, we optimized the parameters of the decoding process. The offset value of each code rate, used in the offset min-sum method, was optimized in a computer simulation. B. LLR quantization

We verified the quantization bit depth of LLR. As in the evaluation in the previous subsection, the quantization bit depth was varied, while the BER at each code rate was measured. The LLR quantization bit depth was from 4 to 8 bits. Fig. 6(a), (b), and (c) show the results for each code. Up to 6 bits, the deterioration of the required C/N is under 0.1 dB. For 8 bits, there is a slight difference between the results with and without quantization. Regarding the hardware implementation, 8 bits equal to 1 byte is a good unit. As a result, the LLR quantization was determined to an 8-bit depth.

IV. CONCLUSION
In this paper, we explained the design of SC-LDPC codes for broadcasting. After that, we briefly explained the decoding process that is a combination of the TDMP algorithm and offset min-sum method. Next, we reported the parameter optimization the decoding process in order to reduce the calculation. We also reported an LLR quantization in order to reduce the hardware complexity. The results of these verifications were incorporated in hardware, and a laboratory experiment was conducted with a prototype modulator and demodulator for broadcasting. We found that the deterioration of the hardware implementation relative to the computer simulation is under 0.1 dB. In the future, we will conduct another laboratory experiment and assess the BER performance of higher-order modulation QAM-OFDM such as 1024QAM and 4096QAM in order to confirm the hardware implementation.
