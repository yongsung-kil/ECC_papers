An Improved Noisy Gradient Descent Bit-Flipping Decoding Algorithm for LDPC Codes

Abstract—Gradient descent bit-flipping (GDBF) algorithm achieves good error performance for decoding low density parity check codes. In the GDBF algorithm, a search point is apt to be trapped in a local optimum, which causes performance degradation of the GDBF algorithm. To overcome this issue, the noisy GDBF algorithm introduces random perturbation into the inversion function of each symbol, which decreases the probability of being trapped in a local optimum. In this paper, we develop the noisy GDBF algorithm by enhancing the threshold adaptation in multi-bit flipping procedure to improve the BER performance without increasing the time complexity.
I. INTRODUCTION
Low-density parity-check (LDPC) codes have been devised by Gallager [1], [2]. For LDPC codes, Gallager presented two basic iterative decoding algorithms, i.e., a belief-propagation (BP) decoding algorithm and a bit-flipping (BF) algorithm. The BP decoding algorithm achieves excellent performance; however, it involves significant decoding complexity. In contrast, although the performance of the BF algorithm is inferior to that of the BP decoding algorithm, the BF algorithm is considerably less complex than the BP decoding algorithm and suitable for hardware implementation. There have been numerous studies on BF algorithms for LDPC codes [3], [4], [5], [6].
Wadayama et al. proposed a gradient descent BF (GDBF) algorithm [7], which resulted in better error performance than BF algorithms. The GDBF algorithm consists of two bit-flipping mode, i.e. single-bit mode and multi-bit mode. In the single-bit mode, the process to find the minimum value from the inverse function values of the entire bits is not parallelizable. To overcome the weakness in parallelized implementation, parallelizable GDBF algorithms such as adap-

tive threshold GDBF (AT-GDBF) [8], and parallel GDBF (PGDBF) [9], have been proposed.
In the GDBF algorithm, a search is likely to be trapped by local optima, which causes decoding failures. In 2014, Sundararajan et al. proposed the noisy GDBF (NGDBF) algorithm [10]. The NGDBF algorithm applies a random perturbation to the inversion function of each symbol, which is capable of being less likely to be trapped in a local optimum and the performance was improved. The NGDBF algorithm introduces methods of threshold adaptation proposed by Ismail et al. [8] to improve the convergence of multi-bit flipping algorithm. It enables parallel implementation. The NGDBF algorithm achieves the further performance improvement. To date, the NGDBF algorithm have the best performance in the GDBF algorithms [10].
In this paper, we develop the threshold adaptation method for the NGDBF algorithm to improve the error performance. To accelerate the threshold adaptation, we introduce an interval in which both bit-flipping and threshold adaptation are executed.
For noise-perturbed decoding alogorithms, it is typically difficult to perform theoretical analysis. It is normal that the effectiveness of these decoding algorithms is evaluated empirically. In this paper, we also evaluate the performance of the proposed decoding algorithm through simulation.

IV. PROPOSED THRESHOLD ADAPTATION METHOD
In this section, we describe the proposed threshold adaptation method, which we apply to the multi-bit NGDBF algorithms, i.e., the MNGDBF algorithm and SMNGDBF algorithm.
In the proposed method, we make the threshold adaptation more likely to increase correct bit-flipping. In the conventional method, if the inverse function value Ej ≤ θj, bitflipping occurs, otherwise if Ej > θj, threshold adaption occurs. As shown in Fig. 1, in the proposed method, we introduce an erasure interval between the threshold values of 0 and 1. During this erasure interval both bit-flipping and threshold adaption occur simultaneously.
