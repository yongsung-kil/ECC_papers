On Iterative Decoding of Short Codes
Nimrod Shacham and Ofer Amrani School of Electrical Engineering Tel Aviv University, Israel

Abstract—This paper is concerned with iterative decoding of short error correcting codes having high density parity check matrices (HDPC Codes). There is a need for reliable graph-based iterative decoding of short codes; for instance this is beneficial in the framework of a newly introduced random version of OFDMA (R-OFDMA) [1]. We first note how the structure of the parity check matrix can alter an iterative decoder’s performance. Specifically we focus on the allowable redundancies in the matrices, where it is possible to benefit from inserting redundant parity checks without altering the code properties. We then employ computation tree analysis and density evolution which both support simulation results showing a performance improvement.
I. INTRODUCTION
A. Linear Block Codes
When defining a Linear Block Code, one can use either it’s Generating Matrix G(k)×n or it’s equivalent Parity Check Matrix, H(n−k)×n. The parity check matrix’s rows constitute parity checks for n-dimensional vector c and provide a simple test to check a codeword’s validity, i.e. c is a valid codeword iff it’s syndrome (s) equals zero:

C. Code Representation and Decoding
1) Tanner Graphs: The Tanner Graph was introduced as an additional means of representing error correcting codes [3]. It’s benefits are based on it being a graphical representation, as opposed to the previously shown algebraic representations. A tanner graph (TG) at it’s core is a bipartite graph depicting the connections between a code’s variables, and parity check equations. Let νi denote the i-th variable node, and κj the j-th parity check node of the tanner graph T. A non-directional edge between nodes νi and κj will be marked as (νi, κj), and will be added to the graph if and only if the corresponding member hji in the parity check matrix H is non-zero.

H uniquely defines C, as the codewords belonging to C form the kernel of the linear mapping made by H. In other words, the code C forms the null space of H: (∀c ∈ C | c·HT = 0). While the parity check matrix does define C⊥ and therefore C, it is not unique in this regard. Similarly to the generator matrix and C, we can use any set of (n−k) independent vectors from the dual code C⊥ to form a valid parity check matrix for C. Furthermore, we can use more than this minimum number of vectors, and still receive the same subspace. This draws us to the main point of our research, employing analytical methods to demonstrate the benefit of using more than the minimal number of parity checks during decoding.
B. LDPC and HDPC Codes
LDPC codes [Gallager,[2]] are defined by the sparsity of their parity check matrices with ’regular’ LDPC codes having a fixed number of ’1’s in each column and row (not necessarily being equal), and the superset family of irregular LDPC codes which have a variable number of ’1’s in each row and column. Alternatively, High Density Parity Check (HDPC) Codes are codes with a high density of ’1’s in their parity check matrix. III. COMPUTATION TREE ANALYSIS
A. Background
In his 1996 dissertation, Wiberg [5] introduced the groundbreaking concept of Computation Trees and Deviations. Similar to trellis detours, deviations can be used to construct a bound on the probability of a decoding error. The Computation Tree (CT) for variable node νi of code C is built by choosing node νi as the root node of a new Ci(G) graph. Then, recursively adding nodes and edges according to the each node’s original local function while ignoring the fact that nodes repeat. Each instance of each variable or check node is treated as an additional node. A configuration on the computation tree which includes an erroneous root node and which doesn’t contain any subset of configurations is called a deviation (e). The set of all deviations on a computation tree will be called the deviation set E, i.e. (∀e ∈ E). For a configuration to be a deviation, the following conditions have to be valid:
1) Each check node in the CT has two or zero ’1’ variable nodes adjacent to it.
2) The deviation includes the root node. 3) No proper and non-empty subset of nodes form a valid
configuration in the CT.
When each configuration (pseudocodeword) is given a global cost (G), it was shown that maximum likelihood (ML) decoding on the computation tree corresponds to finding a valid configuration that minimizes the global cost G(x). Wiberg additionally formulated a necessary condition for bit error, given a computation tree built upon a certain variable node as root. In order for the possibility of a bit to be decoded incorrectly, the following condition must be true:
min G(e) ≤ 0 f or some e ∈ E .
It can be seen that the probability of error in decoding bit s is upper bounded by the probability that a deviation in bit s’s computation tree has a global cost that is less than zero.
B. Upper Bound with Redundant Parity Checks
Wiberg’s necessary condition for error can be seen as a test to see whether an erroneous bit can occur. In order to turn this into a proper upper bound, we describe the following worstcase scenario: For each bit, if there exists any deviation within the computation tree rooted at that bit, such that G(dev) < 0, then count that bit as in error, i.e. run the deviation bound separately on each bit of a received vector. IV. CONCLUSION
It is known that adding redundant (linear dependant) parity checks to the parity check matrix can be beneficial for iterative decoding. This has been analyzed analytically for the BEC Channel but not for the AWGN Channel. In this paper we analyze the theoretical performance of Belief Propagation iterative decoding using two methods, Density Evolution and an upper bound using Wiberg’s concept of deviations. Both agree with simulations, and confirm that an improvement in error performance can be achieved when allowing redundant parity checks to be added to a code’s parity check matrix.
