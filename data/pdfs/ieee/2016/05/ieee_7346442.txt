Low-Complexity First-Two-Minimum-Values Generator for Bit-Serial LDPC Decoding
Jea Hack Lee, Student Member, IEEE, and Myung Hoon Sunwoo, Fellow, IEEE

Abstract—This brief proposes a low-complexity first-twominimum-values generator for a bit-serial scheme. Since the hardware complexity of generators utilizes a significant portion of the min-sum low-density parity-check decoder, a low-complexity generator is crucially important. To reduce hardware complexity, an existing bit-serial generator that finds only one minimum value instead of two has been proposed; however, it can cause bit error rate (BER) degradation. By contrast, the proposed low-complexity bit-serial generator can find the exact first two minimum values and thus can improve the BER performance. Moreover, the proposed generator does not suffer from any throughput loss since its latency is almost the same as that of the existing generator.
Index Terms—Area–time complexity, bit-serial scheme, lowcomplexity design, low-density parity-check (LDPC) decoder, minimum-value generator, min-sum algorithm.
I. INTRODUCTION
L OW-DENSITY parity-check (LDPC) codes [1] have become popular owing to their excellent error-correcting capability and parallel structure. Therefore, LDPC codes have been considered for a wide range of applications, such as error correction codes (ECCs) for flash memory [2], [3], 10GBase-T [4], and DVB-S2 [5].
In the min-sum algorithm [6], the check-node unit (CNU) is used for row operations to produce the outputs. Instead of producing the actual output values, the CNU can significantly reduce the memory space by producing only the two minimum values and signs of all output values. LDPC codes allow a finelevel parallel message-passing decoding [6]. This parallelism can potentially be used to build serial LDPC decoders [7]–[9] with multigigabit per second throughput. The crucial part of the calculation module in LDPC decoders is a generator that finds the two minimum values over the magnitude of all variable node messages [10]. Thus, for high-speed and low-complexity LDPC decoder designs, the first-two-minimum-values generator is the key building block of CNUs [11].
All of the bit-parallel generators proposed in [12]–[14] for finding the first two minimum values are based on tree and radix architectures. Furthermore, the bit-parallel generator [15]

The hardware complexities exponentially increase with an increase in input messages. Therefore, the bit-parallel architectures [12]–[15] may not be suitable for long-code-length LDPC decoders.
To reduce the hardware complexity, several serial schemes have been proposed for fully parallel LDPC decoders [7]–[9]. By using the approximate min-sum algorithm [7], the bitserial generator for the LDPC decoders [7] finds only the first (W = 1) minimum value (min1) and then min1 +1 is used instead of finding the true second minimum value (min2). The LDPC decoder [7] can decrease its area; however, this results in a performance degradation of approximately 0.5 dB at BER = 10−6 [9] for the (2048, 1723) 10GBase-T LDPC code [4] compared with the min-sum algorithm. To design the firsttwo-minimum-values (W = 2) generator using the bit-serial scheme, [8] proposed a unique zero detector (UZD). Assuming that n is the word length, the generator [8] can complete finding the first two minimum values in 2n clock cycles, whereas the bit-serial generator [7] requires n + 1 clock cycles. As a result, the throughput of the LDPC decoder [8] is limited because the CNU has a long check-node processing time (2n clock cycles).
This brief proposes a low-complexity first-two-minimumvalues (W = 2) generator using the bit-serial scheme. The proposed generator takes n clock cycles to find the two minimum values; thus, it can achieve high throughput and occupy a small area as in bit-serial LDPC decoders. In addition, to eliminate the BER performance degradation [7], the proposed bit-serial generator finds the two exact minimum values.
The rest of this brief is organized as follows. Section II describes the existing bit-serial architecture and an example. Section III presents the proposed generator for finding the first two minimum values (W = 2) using the bit-serial scheme. The implementation results are presented in Section IV, followed by the conclusion in Section V.

IV. IMPLEMENTATION RESULTS
Bit-parallel architectures are optimized to boost up the overall throughput, whereas bit-serial architectures focus on low complexity instead of throughput. Hence, comparisons in terms of the delay or area complexity between the bit-parallel generators [12]–[15] and the proposed one do not make sense. Accordingly, we compare the proposed generator with the bitserial generators [7], [8].

The generator [7] requires n clock cycles to find only min1. However, another clock cycle is spent on computing min1 +1. Hence, the latency of the generator [7] is n + 1 clock cycles. The generator using the UZD in [8] finds min1 and min2, while the latency requires 2n clock cycles. However, the proposed generator can find not only min1 but also min2 in n clock cycles.
For a fair comparison, the experimental results shown in [7] and [8] have been reproduced using the IBM 90-nm CMOS technology in which the check node degree is 36. We found that the proposed architecture as well as the other architectures in [7] and [8] can operate at a higher than 1 GHz in the IBM 90-nm CMOS technology based on the synthesis results using Design Compiler. We have limited the clock frequency of the generator as 500 MHz (2 ns) to reduce the power consumption. Moreover, the bit-serial LDPC decoder operating at 500 MHz can achieve approximately 30 Gb/s throughput [7], which is higher than those required by flash memory [2], [3], 10GBase-T [4], and DVB-S2 [5]. At n = 5, the latencies of the existing [7] and proposed generators are five clock cycles. By contrast, the latency of the existing generator using UZD [8] is ten clock cycles; therefore, the long latency [8] may limit the throughput of the LDPC decoder.
The LDPC decoder using [7] degrades the BER performance caused by the approximate min-sum algorithm, whereas the LDPC decoder using the proposed generator can eliminate the BER degradation. Moreover, the latency for the proposed generator is reduced by half compared with that of [8]; thus, the LDPC decoder using the proposed generator can achieve a high throughput rate. In addition, the AT complexity of the proposed generator is cut to approximately half compared with that of the generator [8]. Therefore, the proposed bit-serial generator shows good performance among the existing bitserial generators presented in [7] and [8].
V. CONCLUSION This brief has proposed a low-complexity algorithm to find the first two minimum values using a bit-serial scheme. The existing bit-serial LDPC decoders [7] may suffer from BER performance degradation. To improve the BER performance, the proposed generator cannot only find the two exact minimum vales but also maintain the latency. Hence, the proposed lowcomplexity generator can be quite suitable for long-code-length LDPC decoders, such as ECCs, for flash memory [2], [3], 10GBase-T [4], and DVB-S2 [5].
