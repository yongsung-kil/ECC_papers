Optimum Message Mapping LDPC Decoders derived from the Sum-Product Algorithm
Jan Lewandowsky, Maximilian Stark, Gerhard Bauch Hamburg University of Technology Institute of Communications 21073 Hamburg, Germany
Email: {jan.lewandowsky,maximilian.stark,bauch}@tuhh.de

Abstract—Starting from a discrete density evolution scheme originally introduced by Brian M. Kurkoski et al. which we improved by applying the Information Bottleneck method, we recently presented results on message passing decoders for Low Density Parity Check codes that have much lower complexity than state of the art decoders. In the decoders all node operations are replaced by discrete message mappings of unsigned integers what yields a great complexity reduction. Anyway the decoders perform very close to belief propagation decoding. New included simulation results prove that using a 4 bit integer architecture these decoders loose only 0.1 dB over Eb/N0 in comparison to an exact belief propagation decoder applied to the quantized output of a Gaussian channel. The most important contribution of this paper is the derivation of the message mapping decoders from the sum-product algorithm. Until now these decoders are assumed to not be linked to this algorithm. In order to reveal the hidden connection, we explain the decoding principle of the message mapping decoders in general factor graphs.
I. INTRODUCTION
Modern communication systems are expected to offer transparent high speed access to communication services. Unfortunately inevitable bit errors always degrade the quality of the received data. To overcome this problem very mighty forward error correction schemes such as Low Density Parity Check (LDPC) codes [1] are utilized in state of the art communication systems. LDPC codes offer performance close to the theoretical limits obtained by Claude Elwood Shannon, while iterative decoding algorithms are available that allow decoding with reasonable complexity. A widely known iterative algorithm to decode LDPC codes is belief propagation decoding. Despite the iterative processing that is the key to allow decoding in reasonable time, belief propagation decoders still require high implementation efforts. The check nodes in a belief propagation decoder need to evaluate nonlinear functions of log-likelihood-ratios (LLRs). Since LLRs are arbitrary real values they need a quasi continuous representation in hardware which needs a high number of bits per represented code bit. Furthermore the chip architecture needs to be designed to process real values requiring high resolution data types such as fixed point types or floating point types. Of course also the corresponding arithmetical and logical units need to be implemented to process these high precision data types. This is why the implementation of decoding algorithms for LDPC codes still causes bottleneck situations and the efforts to build low complexity decoders for LDPC codes have a long history

(as e.g. [2]–[4]). Recently progress on discrete LDPC decoders [3], [5], [6] has been made. Discrete LDPC decoders have the potential to overcome hardware constraints, because they pass and calculate functions only on unsigned integers. The implementation of such decoders is efficient, since the number of required bits per represented and processed code bit is small. Such decoders form an intermediate step between soft decision decoders and hard decision decoding algorithms. In [6] we presented a method to construct and implement discrete LDPC decoders that result from a key technique invented by Brian M. Kurkoski et al. [3]. This key technique is called discrete density evolution. We modified the original discrete density evolution scheme [3] by an application of a dedicated Information Bottleneck [7], [8] algorithm. Doing so we were able to construct discrete LDPC decoders with performance just 0.4 dB over Eb/N0 away from the decoding performance of belief propagation decoding [6]. Discrete density evolution yields message mappings for the node operations of discrete LDPC decoders that directly map the incoming discrete messages of the node operation to a discrete output value from a finite alphabet. The key idea is to pick the message which shares the highest amount of mutual information with the bit this message represents. In this paper we interpret the construction algorithm for the discrete node operations and the decoding in general factor graphs [9], [10]. Doing so we are able to point out that discrete density evolution can be derived from the general sum-product algorithm which is a new very interesting result. The considered decoders were explicitly stated to not be derived from the sum-product algorithm in [3], [5]. Instead their construction was interpreted as a mutual information maximizing quantization in [3], [5] and similarly as a clustering approach in [6]. Based on the derivation from the sum-product algorithm in this paper we are able to explain how for the considered decoders the decoding is linked to the sum-product algorithm: We show that for the proposed decoders a variant of the sum-product algorithm collapses to simple integer passing during decoding. Finally we present some new simulation results that prove the performance of the resulting decoders for an other code than presented in [6]. In the next section we briefly recall some basics on LDPC encoded transmission, factor graphs and the sum-product algorithm. Furthermore we recapitulate necessary prerequisites from [3], [5] and [6]. II. PREREQUISITES
A. LDPC encoded transmission over a discrete output channel
All codewords x of an LDPC code fulfill H · x = [0, 0, . . . , 0]T, where H is a sparse binary Nc×Nv parity check matrix. The Nc rows of H represent parity check conditions of the code. Parity checks of LDPC codes are usually visualized in a Tanner graph [10] consisting of check and variable nodes. In this paper we consider regular LDPC codes with fixed check node degree dc and fixed variable node degree dv. The transmitter uses binary phase shift keying (BPSK) to transmit a codeword over a symmetric additive white Gaussian noise (AWGN) channel of variance σn2 . At the receiver a quantizer maps the analogue received samples to unsigned integers ych from the alphabet Ych = {0, 1, . . . , |Ych| − 1} in the symbol clock. The quantizer is designed to minimize the loss of mutual information between the channel input and the quantizer output that results from quantization. Its construction is described in [6] and a globally optimum construction algorithm is presented in [11]. The resulting discrete memoryless channel is characterized by conditional distribution p(ych|x). The integers ych from the quantizer are fed to a discrete message passing decoder from [6] that stops after a maximum of imax iterations or earlier, when all check equations are satisfied. The main difference to state-of-the-art decoders is that the considered discrete LDPC decoders only work on integers from a finite set Ydec = {0, 1, . . . , |Ydec|−1} of decoder messages instead of LLRs. Furthermore all node operations are replaced by message mappings of integers that can be efficiently implemented in a trellis structure presented in [6]. Finally the whole decoder just needs integer arithmetic. Although using different alphabet cardinalities for channel messages and the messages passed in the decoder during iterative decoding is possible, we choose Ydec = Ych in order to avoid a different bitwidth of the representation of both message types in the hardware of the decoder.

In communications factor graphs are often utilized for the problem of efficient marginalization: When the global function Ω(x0, x1, . . . , xM−1) is the joint probability p(x0, x1, . . . , xM−1) of random variables X0, X1, . . . , XM−1 and the corresponding factor graph contains no cycles, a generic algorithm exists that efficiently calculates the marginal distributions p(xj) ∀j up to a scaling factor. In the sum-product algorithm messages μ are passed along the edges of the factor graph of p(x0, x1, . . . , xM−1). In this paper we use the notation from [9] for these messages. A message from variable node xl to factor node ωn is a function of xl and denoted by μxl−→ωn (xl), while a message from factor node ωn to variable node xl is denoted by μωn−→xl (xl) and also is a function of xl. We also use the not-sum notation ∼{.} and the sum-product update rule for factor nodes from [9].
C. Discrete density evolution
Discrete density evolution is a key technique for the construction of discrete LDPC decoders that was first introduced in [3] and then used in [5] and in a modified variant in [6]. Discrete density evolution is used to find information optimal message mappings for the node operations of LDPC decoders. Such a message mapping is a function f : (Ydec)M −→ Ydec which maps incoming messages consolidated in vector y = [y0, y1, . . . , yM−1]T of an arbitrary node in an LDPC decoder to a certain output message f (y) to generate information. The principle is illustrated in the left part of Figure 1. M denotes the number of processed messages for the generation of the output message. The check nodes generate extrinsic information processing M = dc −1 incoming messages, while the variable nodes process one message from the channel and dv − 1 messages from connected check nodes, what causes M = dv for the generation of extrinsic information of the variable nodes. the constitutive structure of the yl defined in equations (1) and (2) this can be understood as choosing a certain p(xl, tl) for a fixed tl (which essentially is a function of xl) in each step and directly passing it via the variable node xl to the next factor γl+1(.). The brilliant consequence resulting in an enourmous complexity reduction is that instead of passing this function in form of real values, as it would be done in the sum-product algorithm, only the integer representant tl needs to be passed for decoding. This sufficiency results from the fact that during discrete density evolution the following partial node operation has been designed based on the corresponding function p(xl, tl). This enables the following partial node operation to calculate its output without ever considering the actual value of p(xl, tl) during decoding. In fact the knowledge on the particular p(xl, tl) was processed offline in the construction of the message mapping gl+1(yl+1) of the recipient factor node γl+1(.) during discrete density evolution. This finally degenerates the decoding problem to recursive look ups of integers at the local factor nodes γl(.) that can be fully implemented in integer arithmetic. In [6] we perform the recursive look up in a trellis diagram. With the new knowledge on the structure of the factor graph in Figure 2 we are now able to explain that this trellis structure is a natural choice from the perspective of factor graphs. By factoring the global functions Γ(.) we introduced auxiliary state variables xl. Doing so we constructed a cycle free factor graph that describes the node operation. In fact such a cycle free factor graph corresponds to a trellis as it is explained in [10].
IV. PERFORMANCE OF DISCRETE LDPC DECODERS
In this section we investigate the performance of discrete LDPC decoders. As in [6] we generate the message mappings of the decoder only for one particular value σn2 and then use the resulting message mappings for all Eb/N0. Message mapping generation then only needs to be performed once and offline. The influence of the particular σn2 used for the generation of the message mappings is investigated. At the receiver we only adapt the channel output quantizer to the actual noise variance of the channel. We apply the dedicated Information Bottleneck algorithm from [6] to construct the classifiers p(tl|yl).

V. CONCLUSION
Our newly found connection between discrete density evolution, the resulting discrete LDPC decoders and the sumproduct algorithm was pointed out. We showed that the update rules for discrete density evolution directly follow from an application of the sum-product update rule in a factor graph. The sum-product algorithm is used to calculate the joint probability p(xl, yl) in discrete density evolution. Afterwards a compressed approximation p(xl, tl) of the joint probability p(xl, yl) is constructed. Since this joint probability p(xl, tl) is processed in the construction of the following partial node operation, during decoding it is enough to pass the integer tl instead of a function represented by real values. As a consequence the decoder can fully be implemented in a low complexity integer architecture. Continuing the performance investigation in [6], we provided new simulation results that prove the very good performance of the decoders and motivate their further investigation. Resumingly, this paper provides a better understanding of discrete LDPC decoders and links them to the sum-product algorithm. Moreover our general derivation might be transferable to other use cases of the sumproduct algorithm beyond the scope of this paper, perhaps offering great complexity reductions for the respective problems.
