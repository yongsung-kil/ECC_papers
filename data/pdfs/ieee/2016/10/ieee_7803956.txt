Abstract—A previous study shows that a low-density paritycheck (LDPC) coded modulation scheme with an extremely sparse parity-check matrix and non-Gray mapping is able to achieve an error-rate performance compatible to that of a conventional Gray-mapping based scheme. This paper presents an associated iterative demodulation and decoding (IDD) receiver architecture that is targeted at flash memory applications. By adopting this extremely spare LDPC code, both the storage and the computational complexity of the decoder are lowered. The complexity of demodulator is also reduced by using the proposed simplified method. Moreover, a two-codeword scheduling approach is used so as to minimize the idles and, hence, a high throughput can be achieved. Compared to the conventional Graybased scheme, the proposed scheme provides a similar error performance and a better hardware efficiency.
Index Terms—LDPC codes, coded modulation, iterative demodulation and decoding (IDD), Flash memory
I. INTRODUCTION
As a result of sophisticated manufacturing processes, flash memory has become a popular storage medium. High-density storage requirement drives the advance of the mainstream techniques, that have shifted from single-level cell (SLC) to multi-level cell (MLC) and triple-level cell (TLC) as shown in Fig. 1. Although more bits can be stored in a single cell, the reliability of the stored data drops as the density of the flash memory increases, meaning that errors occur more frequently. Conventional error correcting schemes, such as BCH codes, can no longer provide a satisfactory error-correcting capability; therefore, low-density parity-check (LDPC) codes are required [1][2].
The SLC, MLC, and TLC techniques can be modeled as 2ASK, 4-ASK, and 8-ASK modulation schemes, respectively. Conventionally, Gray mapping is used, as shown in Fig. 1. The advantage is that in a Gray-mapping based scheme, the decoding results do not need to be returned to the demodulator. However, if iterative demodulation and decoding (IDD) is used, which means that the decoding results can be used to enhance the demodulation performance, the Gray mapping is no longer the only approach available and bit-interleaved coded modulation with iterative decoding (BICM-ID) [3][4][5]

can be considered. In [6], an LDPC coded 8-ASK modulation scheme was designed for flash memory applications, where non-Gray mapping strategies for 8-ASK modulation were considered. It was shown in [6] that the density of the paritycheck matrix (PCM) for the designed LDPC code is only about one third compared to the LDPC code used in a conventional Gray-based scheme. Moreover, the error-rate performance is not degraded. Consequently, it is interesting to investigate the hardware complexity for an IDD receiver based on this nonGray LDPC coded 8-ASK modulation scheme.
In this paper, an IDD receiver is designed based on the LDPC coded 8-ASK scheme presented in [6]. Although a sparse LDPC code is used, an efficient implementation of the IDD receiver is still regarded as a challenge due to the close interaction between the decoder and the demodulator. In addition, since both the decoder and the demodulator are involved in the iterative processing tasks, one of these two function blocks is forced to remain idle while the other is in operation, meaning that a low hardware utilization is introduced. In order to increase the hardware efficiency, a two-codeword architecture is adopted in this work, where the decoder and the demodulator can separately process two different codewords simultaneously. Furthermore, the complexity of the demodulator is also reduced using the proposed simplified method. The proposed IDD receiver is realized using a 90nm CMOS process, and occupies a silicon area of 5.33 mm2, which is equivalent to 1891K gate count, and provides a throughput of 1.1 Gbps at a clock frequency of 166 MHz. Compared to the conventional Gray-based scheme, the proposed IDD scheme can provide a better hardware efficiency based on a similar error-rate performance.

The following is a description of the message exchange between the demodulator and the decoder, where the ith labelling bit, i.e., bi, is considered. The demodulator calculates the a posteriori LLR for bi, i.e., LM,p = ln(P r(bi = 0|y)/P r(bi = 1|y)), using the noise corrupted symbol y and the demodulator a priori LLR, i.e., LM,a, which is obtained from the extrinsic LLR of the LDPC decoder, i.e., LD,e. The extrinsic LLR for the demodulator is calculated according to: LM,e = LM,p LM,a. The LDPC decoder uses the extrinsic LLR , i.e.,LM,e, for the demodulator as the a priori LLR to produce the a posteriori LLR LD,p. Following that, the extrinsic LLR for the decoder is calculated according to: LD,e = LD,p - LD,a. By iteratively exchanging messages between the demodulator and the decoder, the error performance can be enhanced in successive iterations.

B. NMS-based layered LDPC decoding
In the decoding Tanner graph, check node m corresponds to row m of the PCM, and variable node n corresponds to column n. In layered decoding, the PCM is horizontally divided into L layers, and the messages are updated between variable nodes and check nodes layer by layer. Rmn and Qmn denote the check-to-variable (C2V) message from the mth check node to the nth variable node and the variable-to-check (V2C) message from the nth variable node to the mth check node, respectively. The decoding operations are executed iteration by iteration until either the decoded word satisfies all the parity-check equations, or the maximum number of iterations (Imax) is reached. The operations for the i-th iteration, 1 ≤ i ≤ Imax, and the g-th layer, (0 ≤ g ≤ L − 1), are as follows:
1) V2C updating: For each variable node n involved in the gth layer, calculate Qm (i,ng) corresponding to each of its check node neighbours m in the gth layer according to: Q(mi,ng) = Λn - Rm(i−n1,g), where Λn is the a posteriori LLR values which is equal to LD,a in the initial step.
2) C2V updating: For each check node m in the gth layer, compute the Rm (i,ng) corresponding to each of the variable node neighbours n, i.e., n ∈ N (m), according to: Rm (i,ng) = α × Sm(i,ng) × minn ∈N(m)\n(|Qmn |) and Sm(i,ng) = n ∈N(m)\n sgn(Qm (i,ng)), where 0 < α < 1 is a normalization factor.
3) LLR updating: For each variable node n involved in the gth layer, compute the Λn value corresponding to each of its check node neighbours m in the gth layer according to: Λn = Qm (i,ng) + Rm (i,ng).

After completing the processing for the layered LDPC decoding, the LDi ,a value is deducted from the latest LLR LDk ,p and the result is sent back to the LD,e memory block. At the same time, an earlytermination check is executed and the decoding is terminated when the hard decisions by LD,p result in a legal codeword. Fig. 5(b) shows the scheduling between the demodulator and the decoder. Since a data dependency exists between the demodulator and the decoder, the demodulator and the decoder must wait for the other to complete the data updating, resulting in a lower hardware efficiency. To combat this and achieve an enhanced hardware efficiency, two interlaced frames are used in the proposed IDD receiver. When the decoder processes the first frame, the demodulator processes the second frame, as shown in Fig. 5(c). For this two-codeword (two-frame) scheduling technique, the size of the memory blocks for LD,a, LD,e, C2V (R) and the V2C signs, i.e., the shaded parts indicated in Fig. 5(a), needs to be doubled compared to the single-codeword scheduling approach. In the layered LDPC decoder architecture presented in [8], the memory block for the V2C magnitude is used to temporarily store the V2C magnitude values for a single layer during the layered processing and, hence, it is not necessary to double the size of the associated memory in the proposed IDD receiver.

IV. CONCLUSION
We have presented an efficient IDD receiver for LDPC coded TLC applications. The complexity of the implementation is reduced since an LDPC code with an extremely sparse PCM can be used in combination with a low-complexity demodulator. Moreover, the combination of the two-codeword demodulation and the decoding scheduling is able to signifi-

cantly enhance the hardware efficiency. The IDD receiver im-
plemented in a 90 nm CMOS process can provide a throughput
of 1.1 Gb/s at a clock frequency of 166 MHz, with an area of 5.33 mm2.
