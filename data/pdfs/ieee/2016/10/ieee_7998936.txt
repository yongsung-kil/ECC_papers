Reduced Complexity Look-up Table Based 1t-Rotation LDPC Decoder

Abstract 
This paper presents a look-up table based, configurable 
and serial-parallel scheduled 1I-rotation LDPC decoder. 
By using the proposed permutation mapping scheme 
together with an optimized normalized min-sum decoding 
algorithm, the LDPC decoder structure and circuit 
resource 
requirements 
can 
be 
greatly 
reduced. 
Furthermore, the proposed approach is compatible with 
different code lengths, bit widths and permutation vectors 
provided that the parity-check matrix has the 1I-rotation 
structure. Specifically, the proposed architecture in this 
work is implemented with a code length of 1968, a code 
rate of 112, 6-bit quantization and an iteration limit of 10 
on a Xilinx Virtex4 XC4VLS200 FPGA. The synthesis 
results 
demonstrate 
the 
feasibility 
of the 
proposed 
approach to achieve a good BER-SNR performance using 
a simple decoding scheme and an efficient circuit 
implementation. 
1. Introduction 
Low density parity check (LDPC) codes have become 
very important for advanced communications and storage 
systems due to their excellent error correction capabilities. 
While fully random code constructions lead to impressive 
error 
correction 
performance 
[1][2], 
their 
implementations may consume a large amount of circuit 
resources. In recent years, various code construction 
strategies have been developed to simplify the algorithm 
complexity and to decrease the required circuit resources 
[3][4]. 
To obtain a reasonable compromise between error 
correction performance and hardware complexity, codes 
based on certain principles are usually employed, such as 
the QC-LDPC and 1I-rotation LDPC codes [5]. 
One of 
the main advantages of the 1I-rotation structure is the use 
of a single permutation matrix to define the entire code. 
The geometric symmetry in this structure allows very 
significant reductions in the complexity of selecting the 
proper permutation to provide good performance [6]. 
Therefore, 
the 
memory 
space 
and 
circuit 
resource 
requirements of 1I-rotation LDPC codes are small, and 
various optimizations for these codes have been found 
[6] [7]. 
978-1-4673-9719-3/16/$31.00 ©20161EEE 
In this paper, we propose a novel LOPC decoder for 
1I-rotation LOPC codes. The new approach is realized 
based on using look-up tables with a read/write address 
mapping scheme. The absence of a switching network in 
the decoder is the main reason for the reduction in the 
design size. To further decrease the circuit complexity, the 
normalized min-sum decoding algorithm is optimized as 
well. The synthesis results show that the proposed LDPC 
decoder not only achieves a comparable 
error correction 
performance to that described in [6][7], but it also 
consumes a smaller amount of circuit resources compared 
with other LOPC decoder designs. The new approach is 
configurable and it is compatible with various code 
lengths, bit widths and permutation vectors as long as the 
parity-check matrix is in the 1I-rotation style. 
The remainder of this paper is organized as follows: 
Section 2 briefly reviews the basic concept of a 1I-rotation 
LDPC code and its construction. Section 3 illustrates the 
proposed perm mapping strategy in detail. In Section 4, 
the proposed decoder approach is described. Finally, in 
Section 5, our conclusions are given.

This means that the

sequence according to the perm values and their addresses. 
In this case, as shown in Figure I (a), the perm addresses 
are located in the horizontal direction and the perm values 
are in the vertical direction. These two parameters 
construct a coordinate marking each of the elements of the 
matrix 7tA. The perm values are 2, 4, 3 and I based on the 
original perm address order of 1, 2, 3 and 4. The perm 
address having the largest perm value will cause the 
corresponding ROMs address' message element to be 
written into the first location in the RAMs. Therefore, the 
message elements are read out from ROMs one by one in 
their proper order, but are written into the RAMs one by 
one in the order of their corresponding perm addresses' 
perm values, 
which are sorted 
from maximum 
to 
minimum. We refer to this mapping process as write 
address mapping. The perm vector itself indicates the 
order, provided that a type of complement operation is 
performed on the perm values given by (m+ I-perm). 
After the complement operation, the perm vector becomes 
[4+1-2, 4+1-4, 4+1-3, 4+1-1]=[3, 1, 2, 4], and this vector 
indicates where the message elements should be saved. 
Finally, the multiplication operation is completed by a 
simple reordering of the message vector based on the 
perm values (or their complementary values). In other 
words, the 7tA matrix multiplication is implemented using 
forward perm address order, complementary perm values 
and write address mapping.

4.1. Architecture of Proposed LDPC Decoder 
In this section, we will present the decoding strategy for a 
1I-rotation code with a novel look-up table (LUT) based 
perm mapping scheme. The dataflow graph (DFG) of the 
proposed LDPe decoder is shown in Figure 2. 
The received soft symbols are first shifted into 8 
slices of RAMs RI-Rg. Each RAM slice (or block) 
contains 246 6-bit words. Since the parity-check matrix H 
is divided into two parts Hp and Hd, the dataflow in the 
decoder will be processed in two branches. The Hp branch 
is shown in red in Figure 2. Due to its particular structure, 
this branch can be simply processed using normal order 
shifting. Delay cells are used to align the data with the Hd 
processing branch. In the Hd branch, shown in blue, the 
symbols are processed according to the perm mapping 
principle of Table 1. Based on the min-sum algorithm of 
equations (3)-(6), messages are passed back and forth 
between two types of processing units, the check node 
memory update unit and the variable node memory update 
unit.

4.2. Decoding schedule 
As shown in Figure 3, the received data is shifted into 8 
RAMs (246x6 bits in each slice), and then the messages 
will be processed in four layers. 8 RAM slices in each 
layer are processed simultaneously in 246 clock cycles. 
RI-R4 are processed with simple shifting, while R5-R6 
utilize the perm-mapping scheme. 
Since the four layers are generated in sequential time 
slots, the temporary message buffers for the perm 
mapping can be shared and reused. However, since a 
RAM slice cannot be written and read at the same time, a 
ping-pong scheme is adopted. In this way, two RAMs are 
working alternately to handle each layer of messages. The 
output messages from Layer 1 to Layer 4 are then sent to 
the check node update module for further processing. 
The perm de-mapping uses the same scheduling 
scheme and ping-pong operation. The detailed data flow 
and memory resource usage are shown in Figure 4.

5. 
Conclusion 
We have proposed a novel permutation mapping strategy 
for n-rotation LDPC codes that eliminates the need for a 
switching network in the decoder, To verify its feasibility, 
a decoder was realized using a Xilinx Virtex4 FPGA, 
The design uses only a small amount of circuit resources 
and it has a good bit error rate performance, Our approach 
is well-suited for low-power, small area applications such 
as wearable devices and body area networks,
