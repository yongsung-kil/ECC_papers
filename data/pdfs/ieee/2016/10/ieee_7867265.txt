Abstract-it is known that the more reliable LDPC code is the longer code, and it is also known that LDPC decoders with flooded architecture which allow parallel processing can achieve high throughputs. But, just architectures become very complex and require more hardware resources when the code length is large and random, so achieving a high error correction rate on short code lengths is considered a challenge. In this letter, an FPGA design and implementation of a scalable LDPC decoder for short code length are represented. The proposed LDPC decoder is based on the integration between Scaled MSA (SMS) with a Weighted Bit Flipping (WBF) algorithm to allow fast convergence and enhance the error rate, while high throughput is obtained from using flooded scheduling strategy. The results show that the new method can achieve BER reaches to 10-6 and FER reaches to 10-4 at 2.5 dB SNR over BAWGC using BPSK as a modulation scheme. In this letter also, the architecture of this algorithm is exploited to create a scalable design to achieve variable throughputs reaches to multi Gbps versus variable performances. A new scaling method that allows a 76% reduction, and new efficient design for WBF, are introduced.
Keywords-LDPC; MSA ; WBF; Fast Convergence.
I. INTROD UCTION
However LDPC family is introduced by Gallager in 1960, it is ignored due to its complexity at that time until it is rediscovered in 1990' s by McKay [1]. LDPC long codes that are based on Belief Propagation (BP) decoding algorithm such as Sum Product Algorithm (SPA) can achieve near capacity performance [2]. SPA algorithm is based on soft information message passing between variable and check nodes. Although the perfect performance of the SPA, the majority of researches use other algorithms like Min Sum Algorithm (MSA) which has lower complexity than SPA with some performance degradations. Bit Flipping (BF) is a simple LDPC decoding algorithm, where it is based on using hard information instead of soft information. But using hard information resulted in a serious performance loss.
The remarkable contributions that are achieved in this field can be viewed from two perspectives, the first perspective is the complexity reduction methods of SPA algorithm, and the second is the perfonnance enhancement methods ofBF. Actually, in the field of complexity reduction

there are many contributions introduce more simplified algorithms as simplified versions of the SPA algorithm such as MSA [3,4,5,6], scaled MSA, and Offset MSA (OMSA) [7] . Unfortunately, the complexity reduction method of these algorithm resulted in perfonnance degradation. In the field of performance enhancement of BF, many of the contributions such as WBF [8], MWBF [9], IMWBF [10], and others [11 ,12], are achieved. In fact among these algorithms, the higher perfonnance algorithm is the more complex algorithm.
Actually, there are other contributions seek to reduce the computational complexity, and enhance the performance through fast convergence. Some of these methods based on serial decoding [13] and efficient scheduling [14], while others based on deactivation [15] or deleting of high probability error free nodes [16].
However, such methods can effectively compensate the performance degradation resulted from using short codes, and lower complexity LDPC decoders like MSA, it unfortunately sometimes rises the error floor [17] as will be discussed later.
This paper proposes a design and implementation of the integrated WBFSMS LDPC decoder that achieves high throughput based on short code length, and flooding scheduling. Although the proposed WBFSMS algorithm is designed for short code length to suit the flooding strategy, it introduces a high performance, error correction rate thanks to using WBF algorithm as post processing. SMS is used to allow nodes to reach to steady state values and in the same time resulted in speeding up the WBF convergence. At lower SNR WBFSMS is replayed again after enhancing the variable nodes that are matched from previous WBF output and SMS output.
This method of treating nodes is better than node deactivation that suffers from low error floor resulted from freezing error nodes [17]. Although using a large number of iterations, especially at lower SNR high throughput is achieved by fast convergence, and at higher SNR multi Gbps can be obtained. Because of different throughputs that can be achieved, WBFSMS is considered a scalable decoder. The proposed algorithm achieves 1.4 Gbps, BER 10-6, and FER 10- at 2.5 dB, while it also achieves 27 Gbps for SNR 6 dB .

C. Fas! Convergence Decoding
In order to speed up the convergence rate, and reduce the decoding iterations, one of the following methods is used.
• Serial decoding and efficient scheduling method [19, 13,14,20]: that depends on the structure and density distribution function of the parity check matrix; it exploits this structure to build a schedule for updating the variable and check nodes serially.
• Early stopping [21]: by this method the average of the LLR values is calculated and when its updating change rate is very small according to threshold the decoder is forced to stop decoding.
• Node deactivation [15] : this method based on freezing some nodes, which are assumed to be error free after a certain number of iterations. Nodes deactivation is decided upon certain thresholds.
• Node deletion [16,22]: this method is introduced in order to solve some problems resulted from using the node deactivation method. The improvement is obtained by deleting stable nodes to shrink the parity check matrix to reduce the computational complexity. It uses nod reactivation if error bits are wrong deleted.
• Post processing [23]: this method based on using a BF algorithm after the Force Convergence MSA (FC-MSA) to correct errors resulted from using Force Convergence functions. It uses BF when the errors are less than a certain threshold according to syndrome feedback.
• Hybrid Weighted BP (HWBP) [24] : this method depends on using reliability factors obtained from IMWBF to weight the updating nodes to speed up the convergence.
III. PROBLEM FORMULATION
The tradeoff between performance, throughput, and complexity represent the real problem. For example, high performance requires long codes, and complex decoding algorithm as represented in [24], where it uses SPA with IMWBF which require more hardware resources and achieves weak throughputs.
Using of pipelining technique can partially solve long codes' problem, and achieve high throughputs, but it requires also more memory elements, and difficult routing between nodes and it is most probably suits the well structured codes

like QC-LDPC, and maybe not convenient to be used with random parity check matrices.
On the other hand, using of short codes and low complexity decoding algorithm can achieve high throughputs through flooding algorithms, but cannot introduce a real solution for performance problems [4]. Even using of fast convergence method with low complexity like node deactivation methods, can result in high throughput, but it suffers from the low error floor as described in [17], while other fast decoding methods that are based on serial decoding require large codes. So the problem can be summarized in designing practical, low complex algorithm that can achieve high throughput for short codes with high error correction capability, based on fast convergence methods and flooding decoding.
IV. THE PROPOSED ALGORITHM AND RELATED WORKS
WBFSMS represents a practical LDPC decoder that achieves high throughput based on speed convergence and parallel decoding. It based on post processing for increasing the error correction capability, but it differs from the post processing method described in [23], where it uses a few iterations of SMS to give chance for nodes to reach to its steady state values without external forces as recommended in [15] to speed up the convergence of WBF, and to enhance the error correction rate, while the method described in [23] uses BF to correct errors resulted from using force convergence in MSA.
The proposed method also uses early stop decoding as [21], but through syndrome checker to avoid deactivation problems. It has two modes of operation; the first mode is simply a post processing mode or one cycle mode, while the second mode is the cyclic mode which is used to achieve higher error correction rates.
The cyclic mode is simply one cycle repeated n times, where n is the number of cycles required. In cyclic mode before the beginning of the next cycle, the output of the WBF and SMS for the previous cycle is compared and the corresponding LLRs for the matched output of the previous outputs are increased. This method avoids deactivation problems with a little throughput reduction.

V. FPGA IMPLEMENTATION OF THE PROPOSED METHOD
High throughput can be easily achieved through pipelining techniques, when the code is regular or structured such as QC-LDPC. In QC-LDPC, decoding long codes are visible, where the structure of H allows building small modules, and filling the time gapes through pipelining methods. This method of implementation of long codes allows both high throughput and high performance as described in [13].
For other LDPC with random construction parity check matrix even (3 ,6) LDPC code, it is difficult to use long codes even if using generic modules, because the problem embodies in the large mapping size that allow routing between check nodes and variable nodes. High throughput of such codes can be achieved through flooding scheduling, but the hardware size limits the length of the code, and this limitation affects the performance [6].
In the proposed method, flooded scheduling is used and two decoders are used to increase the throughput, and enhance the performance. Although by using flooding technique and two decoders for (512,256) LDPC, it does not require much hardware resources.
Many hardware optimizations are achieved in order to reduce the required hardware without affecting the algorithm throughput or performance.
The main block diagram for one cycle of the proposed method for (3 ,6) LDPC as shown in Fig 1 begins with saving the received code word, then using optimized VNU update modules which treats half of the frame at every clock cycle. The connections or routing map which is wire wraps is selected for every half frame through a multiplexer. The VNU output is connected with CNU memory through the same connection for both half frames, but only the required location saves data according to control signals. The CNU requires only one clock to process one frame . So the total required number of clocks for iteration is 3 clocks.

In order to reduce the hardware and achieve early stop decoding, WBF's syndrome checker is exploited to achieve an early stop decoding for SMS, where WBF is idle during SMS's operation.
The hardware implementation of the WBF is innovative where as mentioned before WBF based on calculating weights that require adders and resulted in much delay, but the proposed WBF is based on only XORs, permutation, and decoder as shown in Fig.2, it costs only one clock to execute iteration. WBF begins with the selection between the new frame and feedback data, it XOR variable nodes according to the parity check matrix to check the syndrome. The next module is used to gather the errors from check nodes for every variable according to H matrix.
The WBF decoder generates three outputs; the first output reflects which variable nodes have 3 errors, while the last output reveals the variable nodes that have only one bit error. According to the weights from the decoder and the syndrome output the last module rises the success interrupt if the syndrome is null, and outputs the decoded data, while in other cases it XOR the input with the highest error vector from the decoder.
CNU function is based on choosing the minimum value of de - 1 variable nodes that share the check node except required variable node itself, so there are de sets every set contains de - 1 elements to be compared. For CNU reduction, there are three methodologies; the first depends on the fact that only two values are required for the de set, the second methodology based Bit Search (BS) from the MSB to LSB, while the third is based on 3 min fmd [3]. BS is very useful if the de is large compared to quantization bits, but for the proposed design it is not effective. The 3 min as described in [3] has many advantages, where its output is ordered and its paths has the same length to avoid critical paths, and also has low hardware, but the problem embodies in its need to de modules for scaling, because there is no indicator for the first or second minimum.

VII. CONCLUSION
This paper proposes a practical implementation of a new fast convergence decoding method for LDPC short code length. The proposed method introduces a real tradeoff between performance and throughput through scalable design, where it achieves 1.4 Gbps, BER 10-6 at SNR =2.5 dB, while it can achieve the same BER, 27 Gbps for SNR > 6 dB. This scalability is obtained from using WBF as post processing to enhance the SMS performance while SMS speeds up the WBF convergence. At lower SNR the high performance can be obtained by repeating the decoding over the proposed algorithm after enhancing the LLR values of the matched nodes between SMS and WBF previous output frame .
