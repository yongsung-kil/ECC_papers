Decoding LDPC Codes with Binary Perturbation

Eun Chong Baek, Hyun Jae Lee, and Sang-Hyo KimвА† College of Information and Communication Engineering
Sungkyunkwan University Suwon, Republic of Korea вА†E-mail: iamshkim@skku.edu

AbstractвАФIn this paper, we provide a simplified perturbation method for LDPC decoding using a binary noise generator with maximal length sequence (m-sequence). The proposed method provides the improved performance of the conventional perturbation-based decoding without performance degradation.
KeywordsвАФ Low-density parity check codes, binary noise, maximal length sequence.

I. INTRODUCTION
Low-density parity check (LDPC) codes, introduced by Gallager [1], are primarily decoded by the belief propagation (BP) decoding algorithm, and have good performances that approaches the Shannon limit [2]. However, when the code is short, the BP decoding may suffer from short cycles existing in the Tanner graph.
In order to improve the decoding performance of short LDPC codes, a number of decoding methods were proposed [3][4][5]. In the proposals, LDPC codes are concatenated with cyclic redundancy check (CRC) codes and several types of perturbation techniques were used. CRC is used to recognize the undetected error in BP decoding. When BP decoding fails, iterative perturbation using Gaussian noise was used to move the starting point of decoding, thereby it can opportunistically avoid weaknesses of the Tanner graph [3][4]. A perturbation method based on bit forcing was later proposed [5].
In this paper, we propose a simplified perturbation using binary noise which can easily be realized by a maximal msequence. Basically, the proposed scheme inherits the decoding procedure of the conventional perturbation based decoding [4]. The proposed binary perturbation has a merit in its simple implementation however maintains the big gain of the conventional perturbation method as compared to BP decoding.

A. BP Decoding
LDPC code is a linear block code defined by sparse parity check matrix пАА where пААпААпАА = 0 holds for a valid codeword. This matrix can be represented by a bipartite graph that is called the Tanner graph. In the bipartite graph, variable nodes and check nodes are connected to each other. In the standard BP algorithm, messages representing likelihoods of the connected variable nodes are transferred through edges forming the graph.
The BP decoding operates as follows. First, the initial log likelihood ratio (LLR) of each variable node for received bit is calculated fed to the decoder. Then check node processing is carried out. Variable nodes transmit the variable node messages in log likelihood ratio (LLR) form - to the connected check nodes. A check node generates its own messages from incoming messages, and then transmits them to the neighbor variable nodes. Variable node processing then runs. The next step is the decoding decision that can check the validity of the temporal codeword estimate by пААпААћВпАА. Two node processings alternates iteratively until we reach a valid codeword or the maximum decoding iteration. If the number of iterations of the BP algorithm reaches the predetermined maximum (say 50) without a valid codeword then decoding failure is declared.
B. Perturbation method with Gaussian noise
Perturbation method [4], was proposed for improving error performance of short LDPC codes. This method repeatedly changes the starting point of decoding by adding Gaussian noise until we find a valid codeword. For each codeword, LDPC codes are concatenated with пАА-bit CRC codes. When the codeword is transmitted to decoder through the channel, this codeword is recorded as a copy of the received signal пААвА≤ = пАА. After that, the codeword is decoded with BP decoding algorithm and the CRC codes of the codeword are checked. If an error is detected in the decoding process, a number of bits that consider smallest absolute value of LLR and check equation satisfaction between neighbor check nodes are selected [4]. Then perturbation is performed by adding Gaussian noise to selected bits.

IV. SIMULATION RESULTS
In this section, we analyze the error performance and the decoding complexity. We also compare the proposed method with the conventional perturbation method [4]. We use three short LDPC codes that have a code length of 192, 288, and 384 bits with code rate 1/2, 2/3, 3/4 in [9]. In this simulation, binary noise with m-sequence of degree 15 is generated by primitive polynomial пААпААпАА + пАА + 1 [10], and 8-bit CRC code is generated by пААпАА + пААпАА + пААпАА + пААпАА + 1 . The number of perturbed bits пАА and binary noise amplitude пАА are empirically optimized. The code with code rate 1/2 and code length 192 has пАА = 48, пАА = 0.4 , the code with code rate 2/3 and code length 288 has пАА = 60, пАА = 0.35, the code with code rate 3/4 and code length 384 has пАА = 72, пАА = 0.3.
We compare the block error rate (BLER) of the BP decoding, perturbation method using Gaussian noise [4], and proposed method as shown in Fig. 3. Note that for perturbation method, a CRC code is concatenated to LDPC codes. In the simulation, the BLERs are observed in Eb/N0 region from 1dB to 4.5dB. In addition, we set the maximum number of iterations of the LDPC decoder as 50 and the maximum number of perturbation rounds as 100.
As shown in Fig. 3, the BLER performances of perturbation method in [4] and the proposed method are almost same, but the BLER performances of two methods using perturbation have improved performances compared to the BLER performance of the conventional BP decoding. The BLER performance comparison shows that the performance gains of each code are about 1.75dB, 1.9dB, and 2.0dB at BLER =10-4 at the code length of 196 bits, 288 bits, 384 bits and the code rate 1/2, 2/3, 3/4.

V. CONCLUSIONS
In this paper, we proposed a simplified perturbation method for LDPC decoding based on binary noise. Binary noise can easily but effectively be generated by m-sequence and its simple LFSR circuit. This proposed method maintains the improved error performance of the conventional perturbation method with lower implementation cost as compared to BP decoding.
