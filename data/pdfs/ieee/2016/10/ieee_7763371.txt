Burst Error Sensing Scheme for Page-Oriented Data

Seongkwon Jeong and Jaejin Lee*
School of Electronic Engineering Soongsil University Seoul, Korea
seongkwon@ssu.ac.kr, zlee@ssu.ac.kr{*corresponding author}

Abstract—Burst error occurs due to various factors and degrades the system performance. In this paper, we propose a burst error sensing scheme for page-oriented data. Using this scheme, the proposed method can sense a two-dimensional burst error location. In addition, it is possible to improve the performance by using erasure decoding on the sensed burst error position.
Keywords— Two-dimensional burst error; sensing; ReedSolomon code; Low-density parity check code
I. INTRODUCTION It is possible to do efficient communication due to the development of error correction code. Since the 1990s, error correction codes approaching the theoretical limit have been proposed and rediscovered such as turbo and low-density parity check (LDPC) code [1]-[3]. Also, it is used more actively in storage systems into two-dimensional (2D) data array. Storage systems using 2D data array are holographic data storage system (HDSS) and bit patterned media recording (BPMR). Therefore, the error also occurs in a two-dimension rather than one-dimension. For example, if the position of the chargecoupled device (CCD) array is not aligned exactly when reading data in HDSS, burst error can occur, and bit error rate (BER) performance is deteriorated. Since the similar problem also happens in BPMR, burst error may occur. For correcting burst errors, several product codes have been proposed in storage systems. Inner code in the product code has a minimum number of parities for detecting long burst error or correcting most of random and short burst errors. However, the long burst errors in each row generate the decoding failure in inner code decoding process. To solve this problem, inner decoder declares erasures which give the location information of errors to the outer code decoder. The outer code generally contains a maximum number of parities for correcting errors. To improve the correcting capability for long burst error, product code has been studied by using RS (Reed-Solomon)-RS, RS-LDPC, and LDPC-LDPC product codes [4]-[7]. A block containing a large number of burst errors will fail to correct the errors if it exceeds the correction capability of the error correction code. Many researches on the burst error problem have been studied. M. N. Kaynak, T. M. Duman, and E. M. Kurtas proposed burst error identification schemes [8]. S. Kim and J. Lee proposed to set the LLR information of channel detector zero using channel state information [9]. To resolve the burst error, 2D interleaving schemes have been proposed. A 2D interleaving scheme for multiple random error

correction codes is proposed by [10], and HDS needs correction of two-dimensional and three-dimensional bursts (clusters) error [11]. Also, optimal 2D interleaving schemes without repetitions have been constructed by [12]. Most of the researches focus on the correction capability itself, but we concentrate on sensing the burst errors, especially, for the pageoriented data in a 2D manner. If we have that information, the error correcting codes can improve their correction capability.
In this paper, we propose a method of sensing 2D burst errors using RS code for page-oriented data. This method is designed to sense the position of the burst error block. After sensing the burst error area by using characteristics of RS code, the error location information is sent to outer error correcting code decoder. For understanding the performance of the proposed burst error sensing scheme, we use an LDPC code as an outer code.

Figure 2 shows flowchart of the proposed burst error sensing algorithm. If the number of errors is less than the correction capability of RS code, RS code can correct error. However, if the number of errors is greater than the correction capability, RS code cannot correct error and it declares the block is an erasure block (burst error is sensed by RS code within the block). Thus, we set log-likelihood ratio (LLR) value of 0 after flagging erasure decoding to the block.
Figure 3 illustrates an example of the burst error situation according to the size of burst error. The burst error (red dotted line) size is 6×6, and the sizes of a block are 4×4 in Fig. 3(a) and 8×8 in Fig. 3(b). When the 6×6 burst error occurs, it determines the dark areas have burst errors if the RS decoder cannot correct errors in the codeword. Thus, if we make the block be small, then the sensed blocks as burst error blocks can be more close to the real burst errors. Consequently, if codeword length of RS code is reduced and correction capability of RS code is increased, it is possible to achieve more accurate burst error sensing at the price of decreasing the code rate.
III. SIMULATION AND RESULTS In the simulation, we simulated 10 pages with an array size of 1260×1260 (1,578,600) bits per page. Figure 4 displays a block diagram of overall simulation procedure. Data is encoded by LDPC code (outer code) and RS code (inner code) in order, and it passes through additive white Gaussian noise (AWGN) and burst error. RS decoder receives and decodes the noisy codeword. If the RS decoder corrects error, then the LDPC code decoder decodes the received data. However, if the RS decoder fails to decode the codeword, burst error sensing scheme senses error location and declares erasure which set LLR value of 0, and the LDPC decoder decodes the data with that information. We define the channel signal-to-noise ratio (SNR) as 10log10(1/σ2) where σ2 is AWGN power. We use the (7, 1) RS code over GF(23) and (3600, 3360) structured LDPC code. The structure of the LDPC code is quasi-cyclic, and the parity check matrix H that has three column weights is created randomly. The decoder of the LDPC code is implemented by a sum-product algorithm with a message-passing algorithm. The maximum iteration number of LDPC decoder is 10. In Fig. 5, we compare the performance of random data, LDPC, and error-sensing algorithm + LDPC (RS-LDPC) when the size of burst errors are 400 (20×20), 1600 (40×40), 3600 (60×60), 6400 (80×80), and 10000 (100×100) in each page. Location of the burst error is random and SNR is 8dB. The LDPC code using error-sensing scheme shows better performance than LDPC code not using this method. Figure 6 shows BER performance according to several burst error. The sizes of four burst errors are 900 (15×15×4), 2500 (25×25×4), 4900 (35×35×4), and 8100 (45×45×4) in each page. Location of the burst error is random and SNR is 8dB. The performance of the LDPC using error sensing scheme is better than that of LDPC only.
IV. CONCLUSION In this paper, we proposed a burst error sensing scheme for page-oriented data. When the number of errors exceeds

correcting capability of RS code, we sense the location of burst error by using the characteristic of RS code. The advantage of the burst error sensing algorithm finds error location and helps to improve the performance.
