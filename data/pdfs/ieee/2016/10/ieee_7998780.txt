Implementation of a High Throughput LDPC Codec in FPGA for QKD System

Cong Hui*, Yonggang Wang and Xiaoming Lu

Department of Modern Physics, University of Science and Technology of China, Hefei 230026, China * Email: huicong@mail.ustc.edu.cn

Abstract-The contribution of this paper is implementing a high throughput LDPC codec in FPGA for quantum key distribution (QKD) system. By software, the throughput of error correction in QKD system via LDPC codec could only reach 1.8Mbps, which is not satisfactory for high speed QKD systems. Thus, it is desirable that LDPC error correction is realized in FPGA to increase the throughput. LDPC codec is implemented according to IEEE802.16e standard. The encoder directly uses a parity-check matrix of lower triangular structure without the construction of generator matrix, so that it effectively reduce the encoding complexity of Richardson encoding algorithm. The decoder uses the BP Min-Sum algorithm to balance performance and complexity. The codec is implemented in an Altera Arria II EP2AGX260 FPGA with a 2304-bit code length and 2/3 code rate. The partly parallel decoding structure achieves a 99.95 Mbps processing throughput on the condition that the maximum number of iterations for each code block is 20. The achievable throughput is much higher than the requirements of current QKD systems.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) encoding/decoding method was introduced early in 1962 by Gallager [1], but draw much attention until the work of Mackay in 1995. Because of the sparseness of the parity-check matrix, the LDPC codec can decode with high efficiency and maintain a linear relationship between the decoding complexity and the code length, in particular for a large code length. Because LDPC codes can achieve an error correction performance approaching to the Shannon limit, it becomes very attractive in channel coding field. In addition to the high performance, its partly parallel decoding algorithm could simplify the hardware implementation. LDPC codes has been widely used in many applications, such as wireless communication systems, deep-space communication systems, storage systems and quantum key distribution systems.
In this paper, we implement the LDPC codec in FPGA for error correction in a high speed QKD system. According to IEEE802.16e standard, Richardson encoding algorithm [2] and BP Min-Sum decoding algorithm [3, 4] are explored for hardware implementation and the LDPC code rate is chosen as 2/3 by the simulation of a MATLAB

program. The LDPC codec is implemented in an Altera Arria II FPGA (EP2AGX260) with a 2304-bit code length. The hardware codec can run at 100 Mhz system clock, the processing throughput is measured as 99.95 Mbps.

B. BP Min-Sum Decoding Algorithm Standard belief propagation (BP) algorithm for LDPC
decoding is considered to be difficult to implement in FPGA due to its complex check nodes update functions. Thus, we choose the BP Min-Sum algorithm for decoding which requires much less hardware resources for the check nodes update functions.
A (N, N-M) LDPC code, c, is the null space of a sparse M×N parity-check matrix, H. It can also be described by Tanner graph. Tanner graph of an LDPC code consists of two set of nodes. Check nodes represent the rows of H and variable nodes represent the columns. In this algorithm, extrinsic messages are passed between check and variable nodes in the form of log-likelihood ratios (LLRs). Let P represent the LLRs of posterior probability of the codeword c; Rim.n represent the message passing from check node m to variable node n in the ith iteration; Qmi ,n represent the message passing from variable node n to check node m in the ith iteration. III. IMPLEMENTATION AND RESULTS

A. Encoder Module There are large numbers of matrix multiplications in the
encoding procedure as shown in Fig. 2. In general, matrix multiplication is complex and requires high resources consumption in FPGA if matrices are ruleless. But the parity-check matrix H in IEEE802.16e standard is special for its sparsity. H is composed of many basic sub-matrices of size 96×96 . Most of the sub-matrices are cyclic-shift identity matrices and the others are null matrices. So that matrix multiplication can be actually optimized as each non-zero sub-matrix multiplying the corresponding subsegment of the codeword, which can be implemented by cyclic-shift of the sub-segments. Then we use the 96-bit barrel shifter structure to realize matrix multiplications conveniently in FPGA. Fig. 4 shows the structure of one row of matrix multiplication in FPGA.

In decoding procedure, the most important step is updating the values of Q, R and P for each row in the paritycheck matrix H. The values of Q, R and P of 96 continuous rows can be updated in parallel because of H is composed of basic sub-matrices of size 96×96 and sub-matrices have the cyclic-shift structure. In FPGA, P-values are stored in 96 dual-port RAMs, Q-values and R-values are updated and stored in 96 updating units. We also design a dedicated ROM to store the messages of H. The system clock of the LDPC codec is 100 Mhz. Encoding process costs only 9 clock cycle and outputs 2304bit codeword at one time. Thus, the maximum throughput of the LDPC encoder can reach 25.6Gbps. As for decoding respect, the maximum throughput can reach 99.95 Mbps due to the iterative calculation in decoding process. This speed is much higher than the requirements of QKD system.
IV. CONCLUSION
We have implemented a high throughput LDPC codec in an Altera Arria II FPGA for QKD system. Richardson encoding algorithm and BP Min-Sum algorithm are successfully adapted for hardware realization. The LDPC codec has been used in our high-speed real-time QKD system, which has a 1 Ghz photon repetition rate [6].
