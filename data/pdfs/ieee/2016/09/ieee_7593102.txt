Improved Minimum Weight, Girth, and ACE Distributions in Ensembles of Short Block Length Irregular LDPC Codes Constructed using PEG and
Cyclic PEG (CPEG) Algorithms

Umar-Faruk Abdu-Aguye, Marcel Adrian Ambroze, and Martin Tomlinson School of Computing, Electronics, and Mathematics Faculty of Science and Environment Plymouth University Plymouth, UK
Umar-Faruk.Abdu-Aguye@plymouth.ac.uk, M.Ambroze@plymouth.ac.uk, and M.Tomlinson@plymouth.ac.uk

Abstract-In this paper we introduce a novel progressive edge-growth (PEG) algorithm, the cyclic PEG (CPEG) algorithm. The CPEG algorithm uses an alternative edge establishment sequence to construct low-density parity-check (LDPC) codes. Irregular LDPC codes constructed using the CPEG algorithm have improved girth and approximate cycle extrinsic message degree (ACE) compared to existing PEG algorithms. We also analyze the minimum codeword weight, minimum stopping set weight, local girth, and local ACE distributions for codes in four very large ensembles of irregular LDPC codes. The code ensembles analyzed were constructed using standard PEG, ACE modified standard PEG, CPEG, and ACE modified CPEG algorithms. Modifications to improve the ACE in PEG LDPC codes, by Xiao and Banihashemi, were implemented in the 'ACE modified' versions of the PEG algorithms. The ACE modified standard PEG algorithm constructed the code ensemble with the highest minimum codeword weight and minimum stopping set weight distributions, and the ACE modified CPEG algorithm constructed the code ensemble with the highest local girth and ACE distributions. Short block length irregular LDPC codes with good degree distributions which have higher minimum weights than have been published for similar LDPC codes were found in the four code ensembles.
Keywords-Approximate cycle extrinsinsic message degree (ACE); edge connections sequence; girth; low-density parity-check (LDPC) codes; minimum distance; minimum stopping set weight; PEG algorithm; Tanner graph.
I. INTRODUCTION
The progressive edge-growth (PEG) algorithm was first proposed by Hu et al. [1], [2]. The algorithm uses graph expansion to efficiently construct low-density parity-check (LDPC) codes with large girth and minimum distance. Following its introduction, several modifications to the PEG algorithm have been proposed in order to optimize one or more parameters of parity-check matrices (or Tanner graphs) to improve encoding or decoding performances.
Xiao and Banihashemi [3] proposed one of the most important PEG algorithm modifications. The modification

significantly lowered the error t100r of short-to-medium length irregular LDPC codes. In the PEG modification, an additional candidate check node selection criterion was incorporated into the algorithm to increase the approximate cycle extrinsic message degree (ACE) in code graphs. A generalized ACE constrained PEG algorithm was subsequently proposed by Vukobratovic and Senk [4]. Despite modifications to the PEG algorithm and other methods of constructing LDPC codes, the error t100r of existing short-to-medium length LDPC codes remains unsatisfactory. LDPC codes are favored over close competitors like turbo codes because they have remarkably good performance when decoded using low complexity iterative message passing algorithms like the belief propagation/sum-product algorithm (BP/SPA).
The minimum distance dmin of a code is the weight of the lowest weight nonzero codeword of the code. The stopping distance Smill of a code is the minimum weight (or size) of a nonempty stopping set of the code. dmin and Smill have significant bearing on the cardinality of decoding errors under iterative message passing algorithms. In additive white Gaussian noise (AWGN) channels, the two main sources of errors in the error t100r region are low weight codewords (which result in undetected errors), and trapping sets [5] (which result in detected errors). In binary erasure channels (BEC), the two main sources of errors in the error t100r region are low weight codewords, and low weight stopping sets (which result in detected errors). Consequently, codes with high minimum weights are preferred.
Krishnan and Shankar [6], as well as other researchers, have shown that determining the stopping distance of LDPC code graphs is an NP-hard problem. Notwithstanding this fact, algorithms for determining the stopping distances for short-to-medium block length LDPC codes exist. Examples are algorithms proposed by Wang et al. [7], and Rosnes and Ytrehus [8]. One of the most efficient of such algorithms was recently proposed by Rosnes et al. [9]. This algorithm implements a complete branch-and-bound tree search in short-to-medium length LDPC code graphs to determine the initial part of the codeword weight spectrum and stopping set

In this paper we propose a PEG algorithm which uses an alternative edge establishment sequence to construct LDPC codes. We refer to the proposed algorithm as the cyclic PEG (CPEG) algorithm. We use the LDPC code weight evaluation algorithm in [9] to determine the minimum weight, i.e. dmin and Smim distributions in ensembles 01' LDPC codes. Minimum weight distributions are used to compare ensembles of codes constructed using standard PEG and CPEG algorithms. Using the PEG modification of [3], the effect of improved ACE in graphs on the minimum weight distributions in ensembles of codes obtained using standard PEG and CPEG constructs are investigated. Additionally, we use the algorithm in [10] to compare global girth, local girth, and ACE distributions in the different PEG code ensembles. We aim to obtain codes with higher dmin and Smin than published for similar length irregular LDPC codes constructed with good degree distributions, like those reported in [9]. Finally, we compare the decoding performances 01' CPEG and standard PEG codes.

III. MINIMUM CODEWORD WEIGHT AND MINIMUM STOPPING SET WEIGHT DISTRIBUTIONS IN PEG, ACE PEG, CPEG, AND
ACE CPEG CODE ENSEMBLES
In this section, we compare the minimum weight, i.e. d min and Smino distributions in four LDPC code ensembles. Two code ensembles were constructed using the standard PEG and CPEG algorithms, and the other two code ensembles were constructed using algorithms derived when the ACE improvement modification of [3] was implemented in the standard PEG and CPEG algorithms. A code ensemble was constructed for each of the following algorithms.
i) PEG, i.e. the standard PEG algorithm, ii) ACE PEG, i.e. the standard PEG algorithm modified

for improved ACE in codes, iii) CPEG, i.e. the cyclic PEG algorithm, and iv) ACE CPEG, i.e. the CPEG algorithm modified for
improved ACE in codes.
The four code ensembles were made up of irregular rate-Y2
codes 01' dimension (n, k) = (512, 256), where n is the code
length and k is the information length. All codes in the four code ensembles were constructed using density evolution (DE) [13], [14] optimized symbol node degree distribution ,.l(x) = 0.34961x2 + 0.39844x3 + 0.09961x4 + 0.04883x5 + 0.1 0352x14• This degree distribution used was obtained from [15]. Random candidate check node selection, as adopted in [I], [2], was used to construct all codes in the four ensembles. The only design parameter which differentiates one code from another within an ensemble is the unique random number seed used for its construction. The random number seed controls the pseudorandom choices of candidate check nodes to which edges are connected during graph construction. Each code ensemble comprised of 6000 LDPC code matrices.
The LDPC code weight evaluation algorithm was used to determine the minimum weight distributions for codes in each 01' the four code ensembles. C. ACE Distributions
The ACE is a measure of the connectivity of cycles in a graph. The ACE of a cycle is obtained by counting the number of edges through which the cycle is connected to the rest of the graph through its symbol nodes [3]. The ACE of a cycle is expressed mathematically as ACE =LiCdi - 2), where di is the degree of the i-th symbol node. The summation is taken over all symbol nodes in the cycle.
We define the following terms in order to compare the ACE distributions in codes: i) the 'local ACE' at a symbol node 5J or a 'symbol node ACE', which is the minimum ACE of the short cycles that pass through the symbol node in a graph, ii) a graph's 'modal symbol node ACE', which is the symbol node ACE value that appears most often in the graph, iii) the 'average ACE' of a graph, which is the sum of all the symbol node ACEs in the graph divided by n, the number of symbol nodes in the graph, and iv) the 'ensemble average ACE' of an ensemble of LDPC codes, which is the sum of the average ACE of all the codes in the ensemble divided by the number of codes in the ensemble [10].

V. CONCLUSION
In this paper we proposed the CPEG algorithm which constructs LDPC code ensembles with higher girth and ACE distributions compared to standard PEG algorithms. ACE PEG and ACE CPEG algorithms resulted in code ensembles with significantly increased fractions of codes with improved minimum codeword weight, minimum stopping set weight, local girth, and local ACE distributions. On the one hand, the ACE PEG algorithm resulted in the code ensemble with the highest dmil1 and Smil1 distributions. On the other hand, the ACE CPEG algorithm resulted in the code ensemble with the highest girth and ACE distributions. Short block length irregular codes which have higher dmil1 and Smil1 than published 1'or similar length irregular LDPC codes constructed with good degree distributions were found in the fOUf code ensembles. Simulation results show that CPEG codes do not lower the known error floor of corresponding standard PEG codes.
