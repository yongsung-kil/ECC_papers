LDPC Decoding Mappings That Maximize Mutual Information
Francisco Javier Cuadros Romero, Student Member, IEEE, and Brian M. Kurkoski, Member, IEEE

Abstract— For low-density parity-check (LDPC) codes widely used in NAND flash memories, the bit-error rate performance is closely tied to the number of bits per message used by the message-passing decoder. This paper describes a technique to generate message-passing decoding mapping functions for LDPC codes using 3 and 4 bits per message. These maps are not derived from belief-propagation decoding or one of its approximations, instead, the maps are based on a channel quantizer that maximizes mutual information. More precisely, the construction technique is a systematic method, which uses an optimal quantizer at each step of density evolution to generate message-passing decoding mappings. Numerical results show, for high-rate codes suitable for flash memories, that 4 bits per message and a few iterations (10–20 iterations) are sufficient to approach full belief-propagation decoding, less than 5–7 bits per message typically needed. The construction technique is flexible, since it can generate maps for arbitrary number of bits per message, and can be applied to arbitrary memoryless channels.
Index Terms— Decoding mapping functions, LDPC codes, quantization.
I. INTRODUCTION
R ECENTLY, scaling improvements have enabled the manufacture of NAND flash memories with increasingly higher storage capacity and lower power consumption. As a result, NAND flash memories are employed in a vast number of information storage systems, including mobile phones and solid-state drives. In NAND flash memories, information bits are stored in cells which can be one bit per cell (single-level cell; SLC), two bits per cell (multi-level cell; MLC), or three bits per cell (TLC); so-called 3D flash further increases densities. With increased density comes increased noise and interference that combine to decrease the uncoded storage reliability and performance of NAND flash memories [2], [3].
Early NAND flash memories employed hard-decision BoseChaudhuri-Hocquenem (BCH) codes to guarantee specified error correction requirements [4]. However, continued technology scaling has resulted in the consideration of more powerful ECC particularly low-density parity-check (LDPC) codes [5]. At the same time, the incursion of LDPC codes into data storage brings the problem of that they cannot guarantee

In NAND flash memories, the latency of reading soft information is proportional to the number of bits per message used for memory sensing, and a target is to reduce the number of bits as much as possible to reduce the latency of the reading process. At the same time, for the quantization of LDPC message-passing decoding algorithms, a reduction of the number of bits per message can lead to a performance penalty [6]–[8]. Indeed, this topic has received substantial attention in both the research and engineering communities, e.g. [10], and past work is summarized below.
In this paper we propose a method to find message-passing decoding mapping functions for regular LDPC codes which can approach belief-propagation (BP) performance using only 4 bits per message. More precisely, the technique is a systematic method which uses an optimal quantizer at each step of density evolution to generate message-passing decoding mappings which maximize mutual information. These maps do not necessarily correspond to elementary mathematical operations, but may be implemented by a lookup table. The resulting message-passing decoding mappings are not quantized versions of BP decoding or min-sum decoding, instead, the proposed maps are based on a quantizer which maximizes mutual information. There exists a strong connection between channel quantization and BP decoding message quantization.
Our approach has both theoretical and practical aspects. The theoretical approach of this work is derived from a strong connection between the problem of classification in statistical learning theory and the problem of optimal quantization of discrete memoryless channels (DMC) in information theory. On the practical side, finite-length results for various regular LDPC code rates show that using 4 bits per message is sufficient to perform close to theoretical limits, achieving full BP decoding performance.
We believe that the proposed decoding maps can lead to a hardware implementation that can meet the requirements of data storage applications. This is due to the fact that the capability of implementing the proposed decoding mappings as lookup tables (LUTs) in a hardware implementation can result in high throughput for regular LDPC decoders. This paper concentrates on regular LDPC codes of high rate, as appealing for data storage applications where its regularity can simplify the hardware implementation and allows a scalable design [12].
Here, past work on quantized LDPC decoders is summarized. In [13] two BP-based decoding algorithms are proposed to reduce the decoding complexity. Due to the complex operations involved in BP decoding to generate the check-to-variable messages, BP decoding is usually implemented using approximations. One of the most common is the so-called BP-based approximation [14] (commonly known as the “min-sum” (MS) approximation [15]).
MS decoding reduces the implementation complexity of the iterative decoding process performing just a few tenths of a decibel inferior to BP performance. In [16] the effects of clipping and quantization on the performance of MS are studied, the best performance is achieved using 6-bits per message. In [17] results were reported using 5, 6 and 7 bits per message with an uniform quantization scheme. From an engineering perspective, error floors of the (2048, 1723) Reed-Solomon based LDPC (RS-LDPC) code and (2209,1978) array-based LDPC code are studied in [18]. In it, 6-bit uniform quantization is employed for a BP decoder using a parallel-serial decoder architecture in a field programmable gate array (FPGA). Offset min-sum using 6 bits with a quasi-uniform quantization over a binary-input additive white Gaussian noise channel (BI-AWGNC) has been proposed in [19].
More recently, in [20] a 3-bit finite alphabet iterative decoder (FAID) was proposed. FAIDs are designed using the knowledge of potentially harmful subgraphs that could be present in a given code. Presented results focus on columnweight-three codes over a binary-symmetric channel (BSC), and in some cases show better performance than BP decoding.
The idea of designing message-passing decoding algorithms using maximization of mutual information has been previously proposed by Lee and Thorpe [11]. They used a nonuniform quantization scheme for a (3, 6) regular LDPC code. Comparing with floating point BP decoding algorithm, 0.2 dB and 0.1 dB gap has been observed using 3 and 4 bits per message respectively, albeit a significant amount of hand-optimization is mentioned and the optimization procedures were not explained in detail. Also using 4 bits per message, Lewandowsky et al. applied the information bottleneck method to derive message mapping decoders [29], where hand optimization over the variance value σ 2 of a symmetric AWGNC was mentioned to achieve the best possible error rate performance of a code with rate R = 0.4.
Thus, compared to past work, the presented method has several advantages. The effectiveness of 3 or 4 bits per message is significant since the aforementioned works considering a BI-AWGNC require 5–7 bits per message to achieve similar error performance. The proposed method can be applied to arbitrary memoryless channels, and is not restricted to the BSC. The proposed method is systematic, in the sense that

given only the channel model and a regular LDPC code, it will produce maps suitable for decoding.
While decoding performance of the proposed messagepassing decoding mappings using 3 bits per message (8 quantization levels) appeared in conference proceedings previously [1], contributions of this paper include the generalization of the construction method to an arbitrary number of quantization levels, and numerical bit-error rate results showing that 4 bits per message closes the gap to BP for finite-length LDPC codes. Additional new results include noise thresholds for the quantized BI-AWGNC, a detailed explanation of the lookup table construction technique, simulations of codes suitable for data storage, and numerical comparisons with FAIDs. In some cases, the proposed techniques have a lower error floor than BP, similar to FAIDs.
The remainder of this paper is organized as follows. In Section II, the optimal quantizer that maximizes mutual information and its application to design decoding mapping functions are described. Section III describes messagepassing decoding considering a decomposition in the nodes, also notation of LDPC codes is presented. In Section IV, density evolution applying the quantizer presented in Subsection II-A is introduced. Section V gives simulation results divided into noise thresholds for the BI-AWGNC and bit-error rate (BER) and word-error rate (WER) results for finite-length regular LDPC codes. Finally, aspects exposed in the paper for the proposed decoding mapping functions are discussed in Section VI.

Throughout this work we shall consider only binary regular LDPC codes with block length N, number of 1’s per column as dv (degree of the variable node) and number of 1’s in each row as dc (degree of the check node). IV. DENSITY EVOLUTION
In this section, we describe an explicit method to compute the decoding mappings functions for a (dv , dc) regular LDPC code. The construction of the mapping functions considers the decomposition presented in Section III, where each node is decomposed into a set of 2-input mapping functions. The quantization algorithm presented in [21], finds an optimal quantizer of a DMC in the sense of maximization of mutual information. In this research, we not only use this algorithm to quantize the channel, we also used it to quantize the conditional probability distributions of the decoder messages, and find decoding mappings as in Section II-B.
The set of decoding mapping functions for each iteration are generated in the context of density evolution [24]. Classical density evolution is restricted to channels with certain symmetry properties. But here, arbitrary and asymmetrical channels are allowed, and the optimized decoding lookup tables, and thus the distributions, may be asymmetrical even if the channel was symmetrical. Wang et al. generalized density evolution to asymmetric channels [25]. They showed that

VI. DISCUSSION
In this paper, message-passing decoding mappings that maximize mutual information are proposed to decode regular LDPC codes. These maps are not derived from BP decoding or one of its approximations, instead, the proposed maps are based on a channel quantizer which maximizes mutual information. Simulation results show that the proposed decoding mapping functions using 4 bits per message can approach BP decoding performance in the waterfall region and can surpass it in the error-floor region, this observed in the BSC as well as in the BI-AWGNC. This is significant for data storage applications where a few bits per message to perform soft-decision decoding are needed to meet the througput requirements without increase the memory latency for the reading process. In addition, finite-length results show that 10 to 20 iterations are enough to achieve BP decoding performance. Because of these characteristics, the proposed decoding algorithm is suitable for high coding rate and low-power requirements for NAND flash memories.
As explained in Section III the message-passing decoding mapping functions suggest to decompose the check nodes and variable nodes into a series of two-input nodes. In each small node a quantizer that maximizes mutual information is located. For the highest noise value where convergence is obtained, the decisions of those quantizers are saved to finally generates the decoding mappings functions. Therefore, the decoding mapping functions change for each iteration , from this, we can say that each mapping function performs a locally optimal quantization that maximizes mutual information.
Having different sets of decoding mapping functions for each iteration can required additional hardware resources, although this issue is strongly related to the synthesizer used for the hardware implementation. One way to reduce the number of mapping functions for hardware implementation is to keep using a set of mapping functions calculated for a certain number of consecutive iterations, in this way, the number of distinct mapping functions can be reduced by applying an optimization algorithm that for example minimizes the energy used in a certain number of iterations as a function of the different number of sets of decoding mapping functions allowed.
