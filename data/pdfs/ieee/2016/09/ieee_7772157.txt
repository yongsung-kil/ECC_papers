Lowering the Error Floor of Short-to-Medium Length LDPC Codes using Optimal Low-Correlated-Edge
Density (OED) PEG Tanner Graphs

Umar-Faruk Abdu-Aguye, Marcel Adrian Ambroze, and Martin Tomlinson
School of Computing, Electronics, and Mathematics Faculty of Science and Environment Plymouth University Plymouth, UK
Umar-Faruk.Abdu-Aguye@plymouth.ac.uk, M.Ambroze@plymouth.ac.uk, and M.Tomlinson@plymouth.ac.uk

Abstract—We propose a simple modification to the progressive edge-growth (PEG) algorithm in order to control the girth of low-density parity-check (LDPC) code Tanner graphs constructed. Using the modified algorithm, the edge densities in short irregular rate-½ LDPC code graphs can be increased without increasing the correlation between edges in graphs by creating cycles of undesired length. Density evolution (DE) optimized symbol node degree distributions used for constructing short-to-medium length codes were further optimized (slightly altered) while maintaining the maximum edge correlation (minimum local girths) that exists in graphs constructed using the unaltered DE optimized distributions. As a result of improved degree sequences (and graph densities) which were derived, short-to-medium length LDPC codes with lower error floors than published for LDPC codes of identical length and rate were found. These slightly denser codes have lower error floor than codes constructed using DE optimized degree distributions in PEG algorithms modified for improved approximate cycle extrinsic message degrees (ACE). Simulation results show that, at a code length of 512, efficient irregular rate-½ PEG codes have significantly lower error floors than a rate-½ IEEE 802.16E (WiMAX) LDPC code of length 576.
Keywords—Approximate cycle extrinsic message degrees (ACE), cycle, edge correlation, edge density, error floor, girth, low-density parity-check (LDPC) codes, minimum distance, PEG algorithm.
I. INTRODUCTION
Low density parity check (LDPC) codes were first introduced, by Gallager, in the early 1960s [1]. Following their rediscovery in the mid 1990s by MacKay and Neal [2], and also by Wiberg [3], there has been a lot of research to improve the performance of practical length LDPC codes. LDPC codes are favored over turbo codes [4], because they have remarkably good performance when decoded using low complexity iterative decoding schemes such as the belief propagation or sum-product algorithm (BP/SPA).
The use of irregular degree distributions for constructing LDPC codes, first proposed by Luby et al. [5], significantly improved their decoding performance. Richardson et al. [6], [7] subsequently optimized irregular degree distributions using

density evolution (DE) techniques. The progressive edge-growth (PEG) algorithm, proposed
by Hu et al. [8], [9], uses graph expansion to efficiently construct arbitrary length LDPC codes with large girth and minimum Hamming distances. The algorithm is very flexible and is easily modified to change the encoding and decoding performance of the parity-check matrices constructed.
To improve the decoding performance of LDPC codes, researchers have modified the PEG algorithm and specified many different criteria for connecting edges between symbol nodes and candidate check nodes during graph construction.
Recently published PEG modifications include: i) for irregular
LDPC codes, the PEG algorithm was modified to maximize the connectivity of newly created cycles to the rest of the graph by selecting only candidate check nodes which maximize the minimum ‘approximate cycle extrinsic message
degree (ACE)’ of new cycles [10], ii) for irregular codes, the
PEG algorithm was modified to select candidate check nodes that prevent connections which lead to small stopping sets
[11], iii) a generalized application of ACE constraints in the
PEG algorithm to further increase the ACE in graphs compared to those obtainable in [10] and lower the error floor
of resultant codes [12], iv) for finite length LDPC codes in
binary erasure channels (BEC); the PEG algorithm was modified to select candidate check node(s) which create the
smallest number of ‘minimal cycle sets’ in graphs [13] and, v)
for lowering the error floor of codes over the additive white Gaussian noise (AWGN) channel; the PEG algorithm was modified to select candidate check nodes that prevent the creation of dominant ‘elementary trapping sets (ETS’s)’ which contain the variable node being added to the graph at each stage of code construction [14].
Linear block codes like LDPC codes which are characterized by cycle-free graphs, have poor decoding performance. Such poor performances manifest not only under iterative decoding schemes but also under maximum likelihood (ML) decoding [15]. Therefore, it is clear that cycles are necessary in the bipartite (Tanner) graph of capacity-approaching LDPC codes [16]. However, under iterative message-passing decoding algorithms such as

BP/SPA, the presence of cycles cause edges (and corresponding symbol nodes) to become interdependent (i.e. correlated) after a few iterations, which significantly reduces decoding efficiency. The shorter a cycle the higher the correlation of its member symbols. Cycles with lengths of 4 (4-cycles) are generally not desired in code graphs. A trade-off exists between edge correlations and edge densities in LDPC code graphs. For any code of length n, the densest graph, which avoids only 4-cycles, would have significantly more edges than the densest graph which avoids 4-cycles and 6-cycles. Most of the best symbol node degree distributions used for constructing irregular LDPC codes, such as those optimized through DE, avoid constructing graphs with local girths of 4, and others avoid local girths of 4 and 6. These degree distributions are used to construct graphs with densities below the highest densities achievable by codes of identical girth.
In this paper, we propose a PEG algorithm modification in order to be able to control the girth of LDPC codes which are constructed. During code construction, the algorithm skips, i.e. forfeits, all edge connections which would have resulted in undesired local girths. The proposed modification is used to construct irregular short-to-medium length codes with increased densities without reduced girths. When combined with ACE improvements of [10], these codes have significantly lower error floor than obtained when only the ACE improvements are made. The rest of this paper is organized as follows. In Section II standard definitions and notations used for describing LDPC codes and Tanner graphs are given. In Section III we present the modified PEG algorithm and describe how it is used to construct irregular LDPC codes with lower error floors. Section IV presents performance simulations for short-to-medium length irregular rate-½ codes over the AWGN channel, the error floor of our codes are compared with the best error floors published for PEG and IEEE 802.16E (WiMAX) LDPC codes using graphical curves. Finally, the paper is concluded in Section V.
II. BACKGROUND
We adopt the same definitions and notations in [8] [9] to describe LDPC codes and the standard PEG algorithm. LDPC codes are linear codes defined by sparse parity check matrices. The dimension of a parity check matrix H is m × n, where m is the number of rows in the parity-check matrix (parity-check equations) and n is the number of columns in the parity-check matrix (the variable/symbol length of the code described by the matrix). A Tanner graph is denoted as (V, E), where V is the set of parity-check and symbol vertices (nodes), such that V = Vs ∪ Vc, where Vs = {sj}, ∀ {0 ≤ j ≤ n – 1}, is the set of symbol nodes and Vc = {ci}, ∀ {0 ≤ i ≤ m – 1}, is the set of parity-check nodes. E is the set of edges or connections between parity-check nodes and symbol nodes. E ⊆ Vs × Vc, with edge (ci, sj) ∈ E if and only if hi,j ≠ 0, where hi,j is an entry of H at the i-th row and the j-th column of the matrix.
A symbol node degree sequence is given by Ds = {ds0, ds1, . . . . , dsn-1}, where dsj is the degree of symbol node sj, {0 ≤ j ≤ n – 1}. Ds is arranged in a nondecreasing order, that is ds0 ≤ ds1 ≤ . . . . ≤ dsn-1. Accordingly, the set of edges E can be partitioned in terms of Vs as E = Es0 ∪ Es1 ∪ . . . ∪ Esn-1, with the entire edges incident on

symbol node sj contained in Esj. The k-th edge incident on sj is denoted as , {0 ≤ k ≤ dsj – 1}.

The girth at a symbol vertex sj (the local girth at sj), denoted by , is the length of the shortest cycle that passes through sj, and the girth of the graph (the global girth), denoted as g, is the length of the shortest cycle in the graph. A stopping set S is a subset of Vs such that all the neighbours of the symbol nodes in S are connected to S at least twice. The size or weight of a stopping set s is defined as the cardinality of S. The minimum (Hamming) distance of a code is the number of 1’s in its lowest weight nonzero codeword.

optimal low-correlated-edge density (OED) PEG codes.

SIMULATION RESULTS In this section we compare the performances of five types of irregular binary LDPC codes: i) a benchmark code constructed using the standard PEG algorithm, ii) OED PEG codes constructed using the MLG (ES) PEG algorithm, iii) ACE PEG codes constructed using the improved ACE modification of the PEG algorithm in [10], iv) OED ACE PEG codes constructed using a combination of the algorithms in ii) and iii), and iv) the IEEE 802.16E (WiMAX) LDPC code construction. Only rate-½ PEG codes with (n, k) dimensions of (512, 256) were considered, note that here k is the number of information bits. A rate-½ (512, 256) standard PEG code was used as a benchmark of good error

floor performance. This code was obtained from [17] and

arguably has lowest error floor published for codes of equal

dimension.

An LDPC code weight evaluation algorithm, by

Rosnes et al. [18], was used to determine the stopping distance

smin (defined as the minimum weight (or size) of a nonempty stopping set) and the Hamming distance/minimum codeword

weight, dmin (which is an upper bound on smin) for all simulated codes. Additionally, an efficient algorithm for determining the

girth and ACE distributions in LDPC code Tanner graphs [19]

was used to determine the girths, ACE spectra, and average

ACE of all codes simulated.

same base-graph. These codes have identical length and rate,

connections made during graph construction. The only

on the same base-graph differ only at the last stages of graph

construction. They have identical dmin and smin but have different densities.

The OED code derivation technique described in this work is not suitable for irregular rate-½ codes with dimensions of more than (1024, 512). At these code lengths, in order for edge connections to be skipped by the MLG (ES) PEG algorithm, the ds(max) in DE optimized degree distributions must be increased to such extents that the densities of codes constructed result in significantly longer decoding times without commensurate improvements in decoding performance.
V. CONCLUSION
The MLG (ES) PEG algorithm controls the girth of codes it constructs by skipping all edge connections which, if connected into the graph, would have resulted in undesired local girths. Consider a DE optimized symbol node degree distribution used for constructing good short-to-medium length PEG LDPC codes of a predetermined girth. OED PEG codes based on this distribution are obtained when the ds(max) in the distribution is increased to extents that when the resultant distributions are used in the MLG (ES) PEG algorithm, with g(min) set at the predetermined girth of the unaltered distribution, edge connections are skipped during graph constructions. Optimum ds(max) values are determined via code performance simulations. Simulation results show that irregular rate-½ (512, 256) OED PEG codes and OED ACE PEG codes have significantly lower error floor
