On the Design of Capacity-Approaching Unit-Memory Spatially Coupled LDPC Codes for Optical Communications
Laurent Schmalen, Detlef Suikat, Vahid Aref, and Detlef Ro¨ sener
Nokia Bell Labs, Lorenzstr. 10, 70435 Stuttgart, Germany, first.last@nokia.com
Abstract We consider unit-memory spatially coupled LDPC codes for optical communications. We analyze the region of wave-like convergence using an FPGA-based windowed decoding emulator. We show that the post-FEC errors occur in long bursts and highlight some design guidelines.

Introduction
Soft decision based forward error correction (FEC) is considered an important means to improve the reach performance of beyond 100 Gbit/s fiber optic communication system. Today, there are two competing classes of codes that allow for soft-decision decoding and that are attractive for implementation in optical receivers at decoding throughputs of 400 Gbit/s and beyond. Besides block Turbo codes (BTCs), low-density parity-check (LDPC) codes are often considered for soft-decision decoding. LDPC codes are theoretically well understood and the decoder is of relatively little complexity, which renders their use attractive for optical communications.
Spatially coupled LDPC codes (sometimes also called LDPC convolutional codes) are the combination of LDPC codes with simple convolutional-like structures. Recently, it has been understood that spatially coupled (SC) LDPC codes exhibit outstanding theoretical2 and practical properties, which makes them attractive for use in many communication systems, and especially in optical communications 1–3. Some coding schemes with high net coding gains have been presented that exhibit low error floors1 and also low decoding latency 4.
As SC-LDPC codes are theoretically capacityachieving, we would like to exploit their full potential to further close the gap to Shannon’s limit and construct coding schemes with high net coding gain (NCG). In this paper, we illustrate such an attempt and show some of the pitfalls that are encountered when constructing capacityapproaching SC-LDPC codes, especially when taking into account practical constraints. We highlight the burst-like error patterns that do not occur in conventional coding schemes and use an FPGA-based emulator platform 1 to assess the performance of SC-LDPC codes.

A small memory μ can be beneficial for optical communications: for a small μ, the rate loss in (2) is minimized (for a given L) and hence the spectral efficiency is increased. Furthermore, a large μ leads to a large required decoding window w, comprising (w + μ)n code bits. Furthermore, the minimum required w depends 5 on μ. A small μ, preferably μ = 1, thus allows us to keep the hardware resources small. Furthermore, as conjectured 1 before, the error correction capabilities improve with small μ and large n. We call codes with μ = 1 unit-memory SC-LDPC codes. In the remainder of this paper, we only consider unit-memory codes (μ = 1).
With μ = 1, the final task is to design matrices H0 and H1. One key parameter is the degree distribution. We denote by dv,i the number of “1”s in column i of the equivalent chunk matrix Hchunk = H0T H1T T . A code is variableregular if dv,i = dv, ∀i ∈ {1, . . . , n}. Kudekar et al. have shown 2 that regular SC-LDPC codes can achieve capacity if dv → ∞, L → ∞, μ → ∞ and an infinite number of iterations can be carried out at the receiver. In the following, we focus on variable-regular SC-LDPC, which can achieve capacity and have implementation advantages.
Unit-Memory Variable-Regular SC-LDPC Code
In order to approach capacity, we should select μ → ∞. However, as highlighted above, we want to keep μ = 1. Fortunately, we can still control the performance by changing the distribution of “1”s in H0 and H1. We can uniformly distribute the “1”s to both matrices (called uniform coupling), but in some cases it is advantageous to unequally distribute these “1”s leading to so-called non-uniform unit-memory SC-LDPC codes 6,7, used also in, e.g., coded modulation 6.
We have to distinguish between two distinct decoding regions in SC-LDPC codes: In the region of wave-like convergence (low SNRs), which is illustrated in Fig. 2, we exploit the fact that due to spatial coupling and boundary conditions 2, the BER inside the decoding window has a characteristic wave-like shape. The fact that the leftmost positions inside the window have a small BER is used to improve the performance of the highly erroneous bits inside the decoding window. Within a decoding window, we carry out I iterations and then shift the window, such that the leftmost chunk of the code contains zero residual errors. We can see that (asymptotically) we can quite closely approach the Shannon limit for dv = 5 by less than 0.1 dB! However, we also see that the critical region (between wave-like decoding and macro-convergence decoding) grows with dv. For codes with dv = 3 and dv = 4, which have been studied earlier 1,3, this critical region is 0.2 to 0.5 dB, and we have shown that even with leftterminated codes (L → ∞) we can achieve significant coding gains. We also show the performance of a dv = 4 unit-memory SC-LDPC code decoded with a hybrid window-pipelined decoder as described in 9. We can see that because of the smaller rate loss, L = 99 is a more effective selection than L = 24. Increasing the window size to w = 13 further delays the burst occurrence probability and thus improves performance. Block LDPC codes are outperformed by all SC-LDPC codes.
Conclusions
In this paper, analyzed capacity-approaching lowcomplexity SC-LDPC codes with windowed decoders. We have illustrated the burstiness of the decoder output which makes conventional BER measurements difficult to interpret. However, unit-memory SC-LDPC still outperform block LDPC codes.
