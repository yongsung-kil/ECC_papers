An Encoder with Speed over 40Gbps for RC LDPC Codes with Rates Up to 0.96∗

HE Zhiyong, ZHAO Qiang, XU Hushan, CUI Wenjuan and LUO Yuxi
(Institute of Modern Physics, Chinese Academy of Sciences, Lanzhou 730000, China)

Abstract — We propose a class of Rate-compatible (RC) Low-density parity-check (LDPC) codes with a very wide range of code rates. To widen the range of rates, we have developed an optimal transmission scheme to push the upper bound of code rates to 0.96. Characterized by a parity check matrix in a dual-diagonal form, the proposed RC LDPC code can be encoded in linear time. Constructed from shifted identity sub-matrices, the proposed codes are particularly well-suited for the high-speed implementation of parallel encoders. Furthermore, the encoder can be implemented efficiently with several left circular shifters and XOR gates. To maximize the encoding speed, we have proposed a q-parallel encoder architecture, where q is the size of each sub-matrix. The implementation results into Field programmable gate array (FPGA) devices indicate that a 72-parallel encoder for the proposed RC LDPC code with a code rate from 0.5 to 0.96 is capable of reaching a speed of 42 Gigabits per second (Gbps) using a clock frequency of 300MHz.
Key words — Low-density parity-check codes (LDPC),
High-speed encoder, Field programmable gate array

I. Introduction
A critical issue in the design of Rate-compatible (RC) Low-density parity-check (LDPC) codes is how to widen the range of code rates, in order to maximize the total channel capacity and provide proper service according to channel conditions (e.g. Refs.[1–7]). Among existing approaches, puncturing provides an effective way to acquire a series of RC codes, where higher-rate daughter codes can be derived from a mother code by puncturing some parity bits. The important challenge in the code design with the puncturing approach is to keep the performances of the mother code and all daughter codes close to the capacity. Because puncturing may modify the degree distribution, which is a more important factor for determining the decoding performance, it is very difficult to obtain good daughter codes with very high code rate.

In this paper, we propose a class of RC LDPC codes with a wide range of code rates. By proposing an optimal transmission scheme and employing a zero-filling decoding algorithm, the approach to obtaining RC codes does not modify the degree distribution of the daughter codes. Thus, the proposed approach pushes the upper bound of code rates to a very high value, such as a rate of 0.96. Since the mother parity-check matrix is constructed from shifted identity matrices, the proposed codes are particularly well-suited for the high-speed implementation of parallel encoders. The implementation results into Field programmable gate array (FPGA) devices indicate that a 72-parallel encoder for the proposed LDPC codes with rates is capable of reaching a speed of 42Gbps using a clock frequency of 300MHz.

1. Code design After adopting a deterministic form for the parity part of the mother matrix, the design of the proposed RC LDPC codes is to construct the systematic part which is composed of a J × K array of sub-matrices. Because each sub-matrix within H is either a null matrix or a shifted identity matrix, the code design consists of the following two steps. The first step is to determine the location of non-null sub-matrices, while the second step is to obtain the shifting coefficients of these non-null sub-matrices. Prior to the mapping of non-null sub-matrices, we need to know the column degrees, i.e. the number of non-null sub-matrices in each block-column. To maximize the coding gain of LDPC codes, the distributions of column degrees should be optimized using simulation methods, such as the method of density evolution proposed in Ref.[8]. It has been observed that the performance in the waterfall region of the error curve can be improved by increasing the maximum column degree or by decreasing the fraction of columns with maximum degree in the parity check matrix[9]. On the other hand, the code having a very good threshold of Signal-to-noise ratio (SNR) in the waterfall region may exhibit a high error floor. Thus, the trade-off between the performances in the waterfall region and in the error region should be considered in the optimization of column degrees. After determining the column degrees, the non-null sub-matrices are located manually according to the following approach: the column degree increases from the lowest degree in the rightmost column to the highest degree in the leftmost column. The second step in code design is to determine the shifting coefficient of each shifted identity matrix, starting from the rightmost columns with the lowest degrees to the leftmost columns with the highest degrees. The shifting coefficient Cm,n in position (m, n) within H is chosen from the restrained set between 0 and q to avoid short cycles and cycles of medium length, where q is the size of each sub-matrix. The encoder was modeled in VHDL (Very high speed integrated circuit hardware description language). IV. Transmission Scheme and Performance Analysis
The performance of a RC LDPC code depends on many issues, such as code design, transmission scheme and decoding technique. The proposed RC LDPC codes can be effectively decoded using the sum-product algorithm or the belief propagation algorithm (e.g. Refs.[12,13]), which performs the summation of all extrinsic messages in each column and the product of all extrinsic messages in each row. Instead of the separate row-column processing in the traditional sum-product algorithm, we use a joint sum-product decoding algorithm in Refs.[14,15] which incorporates the calculation of the extrinsic messages in the column processing into the row processing, i.e. a joint row-column processor is used to process each row from the lowermost row to the uppermost row in each iteration.

To evaluate the performance of the proposed RC LDPC code, we performed simulations with BPSK modulation in an AWGN channel for the RC code which is represented by the mother matrix shown in Fig.1 with q = 72. The iterative decoder with the zero-filling algorithm stops when either a valid codeword is found or 200 decoding iterations are reached.

V. Conclusion
We have proposed a class of RC LDPC codes with a wide range of code rates. In order to maximize the range of code rates, an optimal transmission scheme has been developed to keep the optimal degree distribution unchanged for the mother code and all daughter codes, and thus the optimal scheme guarantees that the performances of all codes are close to the capacity.
