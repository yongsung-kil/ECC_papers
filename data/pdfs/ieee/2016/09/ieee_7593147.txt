Towards Gaussian Capacity, Universality and Short Block Length
Chu10ng Liang, Junjie Ma, and Li Ping Department of Electronic Engineering, City University of Hong Kong, Hong Kong Emails: chuliang@cityu.edu.hk.junjiema2-c@my.cityu.edu.hk.eeliping@cityu.edu.hk

Abstract-Forward error control (FEC) codes based on turbo, low-density parity-check (LDPC) and polar principles provide practical approaches towards channel capacity. There are , how­ ever, remaining challenges. Some of these are listed below.
• How to achieve performance with Gaussian signaling'? • How to design codes that are universally good over different
channel conditions'? • How to reduce code length while maintain good perfor-
mance?
In this paper, we introduce a scheme based on the recent progresses on spatial coupling and compressed sensing. The new scheme offers a potential solution to the above mentioned challenges.
I. INTRODUCTION
Gaussian capacity for an additive white Gaussian noise (AWGN) channel refers to the capacity with Gaussian signal­ ing, which is known to be optimal [1]. There is a gap between Gaussian signaling and the capacity of a discrete constellation (e.g., BPSK). This gap can be noticeable in the high rate regime. Properly designed bit-interleaved coded modulation (BICM) [2] or superposition coded modulation (SCM) [3] can partially close this gap, but these schemes typically require very long block length to ensure good performance.
Spatial coupling (SC) can be used to circumvent a poor fixed-points in an iterative processor [4] [5]. SC involving linear additions (over real or complex fields) has been in­ vestigated for code division multiple access (CDMA) ap­ plications [6] [7]. It is shown that the data coupling (DC) scheme [7] can approach Gaussian capacity at high signal to noise ratio (SNR). The SNR level for the DC scheme with near capacity performance is noticeably higher than the range in practice. The sparse-regression (SR) scheme [8]­ [11], together with SC and compressed sensing (CS), can also approach Gaussian capacity. Similar to DC, SR also works best at high SNR. For low or median SNR, SR requires very high sparsity to ensure good performance, which incurs high decoding complexity. Other schemes related to SC can be found in [12] [13].
In this paper, we study a combined SC and CS (SCCS) scheme. In its simplest form, SCCS can be obtained by re­ placing sparse position modulation in SR with general forward error control (FEC) coding. Our analysis reveals that FEC coding, including sparse position modulation as a special case, holds the key. We show that SCCS can potentially achieve Gaussian capacity for all SNR levels. We will also discuss

a generalized structure of SCCS that offers better simulated performance.
SCCS provides a way for (approximately) universal coding. Here a code is said to be universal if it remains good after random puncturing. Such codes are useful in, e.g., type-II ARQ [1] applications. Conventional binary erasure codes may encounter difficulties here if the code rate after puncturing exceeds 1. Existing high-rate coded modulation methods typ­ ically do not work well after heavy puncturing.
The well-known turbo [14], low-density parity­ check (LDPC) [15] and polar codes [16] all rely on long block length for good performance. This causes difficulty in services with latency constraints. In SCCS, the requirement on block length can be shared by multiple users, which effectively reduces block length per-user. As many wireless systems are multiuser by nature, this approach is practically attractive.
The above mentioned advantages of the SCCS scheme will be verified by both analytical and simulation results.

C. Practical Codes
For a practical code C , 1j;(p) can be generated numerically. In this case, Lemma 4 still holds provided that a locally optimal decoder is used. We observed numerically that c/5 is typically small but it may incur loss.
Fig. 4 shows the state evolution and simulation results for an SCCS scheme using a concatenated zigzag Hadamard (CZH) code (non-systematic and un-punctured) [22] with number of component codes = 2, Hadamard code length = 64, number of information bits (per copy) = 24576 and Rc = 3/64. For SCCS, W = 3 and K = 50.
The analysis in Section II-D is based on the assumption that Aj,k contains lID Gaussian entries. In our simulations, for complexity considerations, each Aj,k is generated using randomly selected rows from an NH X NH Hadamard matrix. The difference between Hadamard and lID Gaussian sensing matrices involving SC has been analyzed in [23]. We observed that for a coded system this difference is typically very small. In simulations, we use the orthogonal AMP (OAMP) algorithm in [24] for its robustness with Hadamard sensing matrices. Fast Hadamard transform (FHT) is used in OAMP with complexity log2 (NH ) per bit. Iteration proceeds until convergence.
Theoretically, CS requires asymptotically large sensing ma­ trices, which incurs high cost even with FHT. Simulations showed that the performance of the above scheme remains almost unchanged for NH > 65536. Therefore, to reduce cost, each copy (coded length = 524288) is partitioned into 8 parts, each of length 65536, that are individually compressed.
Soft output FHT [22] is used for decoding the CZH code. Fig. 4 also shows 1j;(.) for the CZH code, that is clearly not ideal. Nevertheless, the SCCS performance is still excellent. For Res = 1, v falls sharply when p passes the minimum SNR threshold for C CH. For Res = 0.5, v falls slower slightly, as in this case Vo is considerably larger than zero at the fixed point (Po, vo) in Fig. 2. From Fig. 4, state evolution can predict the simulated performance reasonably well. Termination incurs rate loss in SC [4]. For W = 3 and K = 50, the actual rates realized by SCCS in Fig. 4 are 0.4808 and 0.9615 for, respectively, C eH = 0.5 and 1. Therefore, simulation can be slightly better than evolution.

C. Multiuser System with Short Block Length per User
In Seetion II-G, we showed that sees based on a relatively poorljJ(p) can still perform weil, provided that the loss related E / 6" in Fig. 2 is smalI. This property holds for C with a short block length.
We now consider a multiuser sees version of the system in Fig. 5 consisting of K users. We divide the time span into slots, each slot corresponding to a copy. In slot k, user k generates the signals based on its data d k and transmits them over W slots starting from k. The signals from different users are separated by user-specific interleaving, following the interleave division multiple access (IDMA) principle [25]. The linear additions in Fig. 5 are naturally realized in the channel [7].
Fig. 7 shows the performance of this SeeS-IDMA scheme for both K = 16 and 64 using a rate-1/4 non-systematic and un-punctured ZH code with Hadamard code length = 16. The rate of the eZH code = 1/16. Number of information bits (per copy) = 1024. Each copy is partitioned into 4 parts that are individually compressed. To achieve a rate exactly Res = 0.5, for K = 16, each Aj,w is formed by randomly se1ecting 1725 rows in an Hadamard matrix with N H = 4096 so 6" = 0.4211; for K = 64, each Aj,w is formed by randomly se1ecting 1956 rows in an Hadamard matrix with N H = 4096 so 6" = 0.4775. For sees, W = 4.
The sees performance improves with K. This is a common property of se [26], since the overhead due to termination reduces when K increases.

for which users are separated by TDMA. We can see that SCCS outperforms LDPC noticeably. Intuitively, the coding rate of each user in SCCS is 1/8 (after compression) since it occupies in W = 4 slots. This is much lower than the rate (= 1/2) of the LDPC code. Therefore, SCCS might provide better coding gain if cross user interference can be ignored. However, interference does exist in SCCS. It appears that SCCS provides an efficient way for multiuser interference cancelation. This phenomenon was first noted in [28] for LDPC codes with SC but without CS.
CDMA with conventional successive interference cancela­ tion (SIC) cannot help in this case. If a short code is used by each user in CDMA, decoding loss will accumulate in SIC, resulting in large overall loss. Superposition coding [3] also suffers from the same problem.
Incidentally, the block length problem is usually due to the latency constraint. It cannot be solved by, e.g., increasing receiver speed. It is a source problem; the source can only generate a limited number of information bits within a fixed duration in many real-time applications. Clearly, SCCS offers an attractive solution to the latency problem by sharing the constraint on block length in multi-user systems.

IV. CONCLUSIONS
We have shown that an SCCS scheme is universally good for a quite wide range of puncturing ratio, with performance reasonably close to Gaussian capacity. In a multiuser system, the scheme can be implemented with low latency per user.
