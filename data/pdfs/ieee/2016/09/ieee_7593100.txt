Abstract-This paper presents a linear model for the sum­ product decoding of random low-density parity-check (LDPC)
codes, specifically in the high signal-to-noise ratio (SNR) region.
The sum-product algorithm is first approximated as the min-sum algorithm and then a linear model is estimated for the latter. To formulate the model, the dominant trapping sets of the particular code are found. We consider a criterion based on the channel log-likelihood-ratio values to estimate the linear model. The bit error rate performance predicted by the linear model is compared with that obtained through importance sampling. This confirms that the linear model can accurately predict the performance of
LDPe codes in the high SNR region.

II. SUM PRODUCT AND MIN-SUM DECODING

I. INTRODUCTION
It is of utmost importance to evaluate the performance of the low-density parity-check (LDPC) codes in the high signal-to-noise ratio (SNR) region before applying them in communication systems where a very low bit error rate (BER) is required [1], [2]. To get a fair idea about the high SNR performance in less time and gain more insight into the dynamics of the decoding algorithms like the sum-product algorithm (SPA), several authors have considered the linear model approach [3], [4]. It is a well-known fact that in the high SNR region, the BER and the frame error rate (FER) curves of the LDPC codes undergo a decrease in slope. This phenomenon is commonly known as the error floor problem. It is caused by some combinatorial substructures of the parity check matrix like the trapping sets (TS) [1] and absorbing sets (AS) [5].
Several researchers have considered the error floor phe­ nomenon while devising a linear model for the LDPC decoder in the high SNR region. In [3], an analog min-sum algorithm is considered as a piece-wise linear system and the relation between the singularity and the error floor is highlighted. In [4], Schlegel and Zhang have considered a class of dominant ASs of a structured LDPC code and devised a linear dynamic model for the iterative behavior within these sets. Since the code is structured, the behavior of all the ASs within the same class will be the same. So, the authors have studied only one AS. To find the total BER, they have calculated the contri­ bution from one AS and multiplied it with the total number of such ASs. This approach is improved to make the linear analysis more accurate [6]. The same linear model is refined in [7], where the the error floor is attributed to the combined effect of the TSs and the limited numerical precision handling the high-reliability messages.
