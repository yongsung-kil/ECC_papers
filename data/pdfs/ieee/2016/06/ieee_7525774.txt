Self-corrected UMP-APP Decoding of LDPC Codes

V.V. Vityazev, E.A. Likhobabin
Power electronics Ltd. Ryazan, Russia
vityazev. v. v@rsreu.ru, likhobabin.e.a@tor.rsreu.ru

Abstract- In this paper we propose a very simple but powerful self-correction method for the UMP-APP decoding of LDPC codes. Like self-corrected min-sum decoding, our method does not try to correct the check node processing approximation, but it modifies the variable node processing by erasing unreliable messages. Monte-Carlo simulations show that the proposed selfcorrected UMP-APP decoding performs better then common UMP-APP decoding. Estimation of the complexity and average number of iteration for self-corrected UMP-APP algorithm are also shown.
Keywords- error correction, LDPC codes, UMP-APP algorithm, self-correction
I. INTRODUCTION
Low density parity check (LDPC) codes was proposed by Gallager [I] in 1963, and later rediscovered by MacKay and Neal [2]. Like turbo codes, LOPC codes can achieve near the Shannon limit error performance [3], and represent a very promising prospect for error control coding. Nowadays they are widely used in many applications, a lot of modern standards such as OVB-T2, OVB-S2, OVB-C2, WiFi, WiMax, IEEE 802.15.3 used this codes [4, 5].
Any LOPC code is specified by a parity-check matrix which contains mostly zeros and only a small number of ones. A regular (N,K) LOPC code, with block length Nand dimension K, has a parity check matrix H of M ?o N - K rows and N columns with exactly de ones in each column and dr ones in each row. We refer to the M rows of H as check nodes, and to all the elements of a LOPC codeword as variable nodes. Then each check node connected to dr variable nodes and each variable node is checked by de check nodes. Also define N(i) - the set of ones in i'th colunm, M(j) - the set of ones in j'th row of matrix H. N(i)-{j} and M(j)-{i} are the same sets, but withoutj'th and i'th positions respectively.
Gallager also proposed algorithms for decoding LOPC codes. The most popular of them is soft decision based, near optimum belief-propagation (BP) algorithm, also known as sum-product algorithm (SPA), and a hard decision based bitflip (BF) decoding algorithm. They are iterative decoding algorithms.
The main advantage of BP decoding algorithm is high performance, that allows to achieve near the Shannon limit, and the main drawback in it is complexity, that remains relevance nowadays for applications with limited computational and energy resources.

The BF algorithm has a reverse disposition. The main advantage is simplicity and the main drawback is poor performance.
Oue to high complexity of BP decoding algorithm, many authors try to compile advantages of BP and BF algorithms in one algorithm, and a lot of modifications and new ways of decoding LDPC codes have been proposed from its invention. The most significant ways of LOPC decoding evolution are simplifying BP algorithm with loss in performance, and complicating BF algorithm with gain in performance. Nowadays a lot of such modifications are known [6, 7].
[n [8] was proposed to use self-correction as a modification for min-sum decoding algorithm (MS), selfcorrection further studied in [9, 10,11, 12]. [t was shown that using of self-correction for MS algorithm [8] and its modifications [12] can significantly increase its performance. The further simplification of MS algorithm is uniformly most powerful APP based (UMP-APP) decoding algorithm [13] it have computational complexity several times smaller than MS decoding algorithm. Unfortunately, the UMP-APP based decoding algorithm can't achieve very good error performance for LOPC codes [13], so some improvements of this algorithm were proposed to achieve better performance [14].
[n this paper proposed to improve the error performance of the UMP-APP based decoding algorithm using self-correction.
The paper is organized as follows. [n Section II, we give a brief review of self-corrected MS algorithm, and UMP-APP algorithm, and motivation for self-corrected UMP-APP algorithm. Simulation results for proposed algorithm provided in Section III. Finally, the conclusion of the paper is given in Section IV.

III. SIMULATION RESULTS
Research of the performance and convergence were carried out for two LDPe codes. The first is a well-known for a number of publications PEG LOpe code N=1008, M=504 [15], and the second is a quasi-cyclic EG N=8176, M = 7156 LOpe code [4]. In both cases maximum number of iterations is 100.
Simulation results for PEG code are shown on Figures 2 and 3 in comparison with MS and BP decoding. It can be seen, that using self-correction has positive results for UMP-APP algorithm. And also, that improvement in performance and convergence can be achieved by using different variants of unreliable nodes processing. But the best result for selfcorrected UMP-APP algorithm may be obtained when message and reliability of the variable node are erasing during the unreliable node processing. This case is denoted as "sc-mr app" on all figures.

Maximum performance gain for bit error probability 10-5 in comparison with UMP-APP algorithm is about 0_1 dB, but self-corrected UMP-APP is still losing about 0.6 dB to MS and I dB to BP algorithm. Average number of iteration in comparison with UMP-APP algorithm decreased for all signalto-noise ratios up to 3.2 dB and insignificantly increased after.
Using self-corrected algorithms for EG code have the same results, though EG code has higher code rate. Error performance and average number of iterations characteristics for EG code are shown on Figures 4 and 5 respectively. Performance loss for bit error probability 10-5 in comparison with MS and BP decoding is 0.3 and 0.6 dB respectively, which is less than for PEG code.
Average number of iterations for EG code using selfcorrected UMP-APP decoding algorithm is less or equal to the common UMP-APP algorithm.
IV. CONCLUSION AND FUTURE WORK
Analysis of obtained results for PEG and EG codes shows, that one can use self-correction with positive results not only for MS algorithm, but for other message-passing algorithms as well. In this paper we considered only self-corrected UMPAPP decoding algorithm. But it can be assumed, that selfcorrection may be used not only for UMP-APP algorithm, but for its modifications and other message passing algorithms.
