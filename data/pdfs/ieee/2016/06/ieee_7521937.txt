Variable LLR Scaling in LDPC Min-Sum Decoding under Horizontal Shuffled Structure1
Kang Zhao2, Yin Xu2, Dazhi He2, Yunfeng Guan2, Wenjun Zhang2 Cooperative Medianet Innovation Center of Shanghai Jiao Tong University, China

Abstract—LDPC was first proposed in 1962, after that considerable achievements have been made both in decoding algorithms and also in the structure of the decoder. In terms of decoding algorithm, from belief propagation (BP) to min-sum (MS), then normalized MS (NMS) and finally to a more performance oriented variable MS (VMS) algorithm. In terms of the decoder structure, there are mainly two, flooding and shuffled structure (FS and SS), and flooding structure is mainly used in the past and the selection of variable factors in FS VMS has already been studied in the literature. As is known, the horizontal shuffled structure (HSS) are more hardware friendly and now widely deployed, thus to apply VMS algorithm to HSS has an important meaning for the industry. However, there is no effective method for the selection of variable factors in HSS VMS. Based on the study of FS VMS, this paper describes how to apply the generalized mutual information (GMI) based metric to HSS VMS reasonably and proposes the modified GMI based formula. Simulation results on a certain DVB-S2’s LDPC code show that, by using modified formula for HSS VMS, we can obtain 0.13dB gain over HSS NMS and the performance is only 0.1dB away from that of HSS BP in terms of bit error rate (BER) at level 1e-7. And simulation results on a certain DVB-T2’s LDPC code show that, by using modified formula for HSS VMS, we can obtain 0.04dB gain over HSS NMS and the performance is only 0.06dB away from that of HSS BP in terms of bit error rate (BER) at level 1e-7.
Keywords—channel coding; LDPC; FS; GMI; HSS; VMS; variable scaling search

I. INTRODUCTION
In 1962, Gallager first proposed a new linear block code in his Ph.D. thesis, which is called Low Density Parity Check Code (LDPC) [1]. Because of the hardware limit in that time, LDPC was not widely concerned. In 1984, Tanner proposed the bipartite graph presentation of LDPC, which is called Tanner graph [2], a powerful tool for studying LDPC. In 1993, Berrou proposed turbo codes [3] which can be decoded iteratively and approach Shannon limit, and with the practical application of Turbo codes, researchers rediscovered LDPC as it also can be decoded iteratively. And in the middle of 1990s, Mackay and Luby et al found the excellent performance of LDPC which can approximate to the Shannon limit [4][5]. Research on LDPC was then carried out extensively, and considerable progresses has been made in related fields including decoding algorithms and decoder structures which are the main concern of this paper.

In 1999, Richardson proposed the belief propagation (BP) algorithm [6]. BP algorithm introduces the concept of the log likelihood ratio (LLR), thus eliminating unnecessary calculations. But because of the high computation complexity in check nodes’ LLRs’ updating, BP algorithm is hard to be used in practical, it is often used as a criterion for evaluating the performance of later algorithms. After studying the numerical characteristics of LLR computation formula in check nodes’ LLRs’ updating, Fossorier made a linear simplification and proposed the min-sum (MS) algorithm [7]. Although there is certain performance loss in MS compared with BP, MS has a low computation complexity which makes it a practical and widely used algorithm. After this, Fossorier proposed normalized minsum (NMS) algorithm to improve the performance of the MS algorithm by multiplying the MS output with a constant scaling factor in each iteration to correct the overestimation problem which lies in MS algorithm [8]. To progress furthermore, MS algorithm using variable LLR scaling (VMS) [9][10][11] is proposed in which scaling factors are different among different check nodes with different degrees and among different iterations. And it has been verified that VMS can greatly improve the performance of NMS and there is only a narrow gap between VMS and BP. About choosing the best LLR scaling of VMS, in papers [9][10], the authors used an optimal post processing nonlinear function for MS’s output and approximated it via a scaled linear function. And by using the mean square error metric, scaling factors are obtained for row degree regular LDPC. But there are two main problems. Firstly, it can only work with row degree regular LDPC which means scaling factor only changes among iterations. Secondly, scaling factors can only be analyzed based on theoretical tools like extrinsic information transfer (EXIT) chart, which makes scaling factors mismatched for real LLRs. In order to solve these two problems, Yin Xu et al proposed a metric based on generalized mutual information (GMI) which can be used for both regular and irregular LDPC [11]. And base on GMI, scaling factors can be obtained by Monte Carlo training using real LLRs.
Regarding the structure of the decoder, there are mainly two types. One is called flooding structure (FS) and the other is called shuffled structure (SS) [12]. The decoding algorithms and scaling search algorithms mentioned above are all studied based on FS. The main difference between FS

And because of this, SS algorithms converge faster than FS and thus have better performance. More importantly, SS can reduce the hardware resource mostly in storage perspective for LLRs compared with that of FS, which makes SS being widely considered and deployed in current communication systems. And in SS, there are Horizontal-wise SS (HSS) and Vertical-wise SS (VSS) [12]. The rest of the paper will be organized in as follows. In section II, the general principle of FS VMS and HSS VMS will be introduced briefly. Section III presents the GMI based metric for scaling search in FS VMS and explains the underlying inappropriateness (like i.i.d may not be valid and multiple check node degrees exits in one parallel operation) by extending GMI into HSS VMS. Following the logic, in section IV, Quasi-cyclic LDPC is introduced, and based on which the method of using GMI based metric in HSS VMS is broadly studied while the method is applicable for general LDPC codes, in particular the i.i.d assumption is doublechecked and re-defined both in simple QC structure and special QC structure. In addition, scaling search formulas under different parallel degrees are studied to deal with multiple check node degrees issue mentioned in section III. In section V, simulation results based on DVB LDPC codes show the effectiveness of our proposed scaling search method in HSS. Finally section VI concludes this article.

Though GMI based metric using in FS VMS has excellent

performance, it might not be suitable to be used in HSS VMS

directly. There are mainly two reasons:

Firstly, from both (11) and (12), it can be seen that in one

iteration, check nodes which have the same degree share the

same scaling factor. But it might not be the case for HSS

VMS. From the description in section II part C, it is clear

that check nodes which have the same degree may belong to

different parallel operations, in which case, later processed

check nodes’ LLRs will use the updated LLRs of check

nodes processed previously during one iteration. From

iterative decoding algorithms’ view, after the previous LLRs

first being passed from variable nodes to check nodes and

then being passed from check nodes to variable nodes of

latter process operation, the belief level of LLRs rises. And

as belief level rises, scaling factors should be different,

which is a basic principle of VMS scaling selection.

Secondly, GMI metric in FS is under the assumption of

i.i.d, however when changing to HSS case, it requires

reconsideration whether the i.i.d assumption is still valid.

Since in one parallel operation, updated LLRs of the current

row will be used immediately in the calculation of the next

row, in which case the two rows’ LLRs are dependent, so it

is imperative to find out how to implement the i.i.d

assumption properly in HSS.

B. The i.i.d Assumption in HSS

From section III, in FS there is a key i.i.d assumption to guarantee the correctness of the GMI based metric. But, in HSS since in one parallel operation, the updated LLRs of the

current row will be used immediately in the calculation of

the next row, in which case the two rows’ LLRs are

dependent, thus from iterative perspective, i.i.d might not

work out in HSS. However, since FS is the extreme case of HSS when

HSS’s parallel operation degree equals to matrix’s row

number, the i.i.d assumption can still be valid when it is

described as there are no LLR exchanged among rows in one

parallel operation in HSS. For the simple case as is shown in Fig. 2, there is only one

‘1’ in each column of the sub-matrix. It is very clear that

within one parallel operation, each row is processed in

parallel and thus is independently updated without

information exchange among each other. So rows in one

parallel operation meets the i.i.d assumption. For the special case as Fig. 3 showed, there are multiple

‘1’s in the same column of one parallel operation. In HSS
