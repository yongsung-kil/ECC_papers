Hardware Complexity Reduction of LDPC-CC

Decoders Based on Message-Passing Approaches

Hayfa BEN THAMEUR1, Chaima BOUZOUITA1, Nadia KHOUJA1, Bertrand LE GAL2, Fethi TLILI1, Christophe JEGO2
1 SUPCOM, GRESCOM Lab, University of Carthage, Tunisia.
2 Bordeaux INP, CNRS IMS, UMR 5218, Bordeaux University, France.

Abstract—LDPC convolutional codes (LDPC-CC) are a family of error-correcting codes (ECC) used in digital communication systems like the IEEE 1901 standard. High throughput and low complexity hardware architectures were designed for real time systems. In this article we demonstrate that an efficient selection of the message passing (MP) algorithm for LDPC-CC decoding improves the architecture features of related works. In fact, considering the LDPC-CC decoders proposed for the IEEE 1901 standard, we show that an appropriate Min-Sum approximation selection can significantly improve the error correction performance by 0.1 to 0.2 dB in terms of Bit Error Ratio. It can also reduce the hardware complexity by 10% to 20% with no impact on the Bit Error Ratio performance.
I. INTRODUCTION
Introduced in 1999 [1], Low Density Parity Check convolutional codes (LDPC-CC) are the convolutional counterpart of LDPC block codes (LDPC-BC) that were proposed by R. Gallager in 1962 [2] and re-discovered by Neal and MacKay in 1997 [3]. The interest of LDPC-CCs stems from their encoding simplicity, their reduced decoding complexity and their strong potential to support versatile application scenarios based on streaming data transmission and adjustable finite block size transmission [4]–[6]. Thus, LDPC-CCs are considered as a promising candidate for the forward error correction module in the 5G air interface design [7].
The potential of LDPC-CCs has been observed for the SumProduct message passing algorithm (SPA) [8]–[12]. As the SPA has low convergence speed and high hardware complexity due to the costly tanh operations, hardware implementation works have adopted the Min-Sum (MS) approximation [10], [12]–[16] and focused on its sub-optimal variant called Normalized MS (NMS) [17]–[22]. However, to the best of our knowledge, neither a comparison of different MS variants (like Offset MS (OMS), Modified NMS (MNMS)) nor an optimization of the NMS parameter, have been reported yet.
In this paper, we show that switching from one NMS variant with 0.75 normalization parameter to another would improve the error correction performance of the considered LDPCCC decoder (by 0.2 dB for the case of OMS compared with the NMS(0.75)) with no impact on the decoder’s hardware complexity. Hence, the decoding performance of existing works can be notably enhanced without significant hardware complexity increase. Furthermore, the same bit error rate (BER) performance can be achieved with a lower number of decoding iterations Q. For instance, comparable performances

of the NMS (0.75) with Q = 10 can be reached with OMS where Q = 8 or 9. Thus, the amount of processor units can be reduced so that the LDPC-CC decoder’s hardware complexity can be minimized by 10% to 20%. The decoder latency and power consumption would be also reduced.
The remainder of this paper is organized as follows: in section 2, an overview of LDPC-CC decoding is presented. Complexity reduction optimizations proposed in literature are then given. In section III, we review MS like decoding algorithms and evaluate their computational complexity. Section IV reports the main experimentation results.

An (ms, J, K) LDPC-CC is called regular if HT has exactly K ones in each column, starting from the (ms(c − b) + 1)th column, and J ones in each row. In real applications, a periodic (ms, J, K) LDPC-CC is of interest. Its syndrome former HT is said to have a period Ts ∈ Z≥1 if its sub-matrices satisfy HiT (t) = HiT (t+Ts), i = 0...ms, t ∈ Z≥0. If Ts = 1, the code is time-invariant, otherwise for Ts > 1, it is time varying.
A bi-partite graph, called Tanner Graph, is used to represent LDPC-CCs graphically. Regarding the structure of HT , the related Tanner graph is semi-infinite and highly localised. It is composed of nodes and edges. A node can be either a Variable Node (VN) or a Check Node (CN). An edge connects (vi) and (cj) nodes whenever the element h(j,i) in HT is a 1. B. Decoding LDPC-CC
Similar to LDPC-BCs, LDPC-CCs are decoded iteratively using the MP-based algorithms [1]. Conventionally, these decoding algorithms perform CN and VN computations in a flooding scheduling. In fact, this scheduling technique was first adopted for hardware implementations of the Min-Sum approximation [10], [13]–[16], [24]. In these architectures, design efforts have been devoted to the design of a single processing unit and its related storage block as well as CN and VN computation kernels. A processor unit gathers the required arithmetic, memory and interconnect elements able to execute one decoding iteration. In the first studies [14], [17], [25], as shown in Fig. 2, each row in the designed processing unit, called processor, behaves like a FIFO. Its length is at least (ms+1) and its width depends on the node degree and the exchanged messages width. An overview of such processor architecture is depicted in Fig. 4. It can be noticed that each processor requires input messages and generates output messages. Let Q denote the number of processor units in the LDPC-CC decoder. The input messages for processor i can be received either from the channel, if i = 0, or processor (i − 1). The outputs from processor i can be forwarded either to the decision unit (if i = Q − 1) or to the next processor (i+1). As mentioned above, messages related to CNs and VNs can be udpated by a flooding scheduling where the MP-based

algorithm is mainly composed of four computation intensive kernels: Initialization, Check Node Update (CNU), Variable Node Update (VNU) and Hard Decision. Notably, kernels 2 and 3 are performed sequentially. The variable node units are activated only after updating the check node units. Therefore, this scheduling is inefficient because the recently updated information are not taken into account. In sub-section II-C, optimized scheduling techniques are reported.
LDPC-CC decoder architectures can process data in parallel or in sequential way, with regards to iteration and/or node dimension. Commonly, parallel LDPC-CC decoders exhibit high throughput performances but they have a high decoding latency and a high number of FIFOs. Sequential architectures are suitable for LDPC-CCs with large ms. With a single processor, memory and resources consumption can be notably reduced. Even though the decoder throughput is low, in [24] a low-cost low-power serial decoder architecture has been designed for a (2048,3,6) LDPC-CC decoder that achieves up to 9 Mbps throughput with few hardware resources.
Referring to the previously described processor architectures, FIFOs can be mapped to registers (Fig. 2) or memories (Fig. 4). In fact, register-based LDPC-CC decoder design, [13], [14], [17], [25], can support unlimited bandwidth but yields expensive power and area costs. It is not suitable for FPGA targets. Memory-based architectures [8], [12], [16], [18], [20]–[22], [26] were proposed instead, where FIFOs are implemented as RAM-based circular buffers and messages are tracked with pointers.

It has been shown that such a scheduling technique yields a twice faster convergence speed compared with its flooding counterpart due to the use of the most recently updated information. It halves the number of decoding iterations and thus the computational complexity that is necessary to achieve similar BER performances. It reduces the memory complexity while the memory cost of messages becomes dc − 1 instead of dc. However, the memory need is still high. A horizontal layered based scheduling MP-algorithm has been proposed in [19]. Instead, they are computed on the fly during the decoding process [12], [20], [22]. In the current work, the horizontal layered algorithm is selected.
2) Node Level optimizations: Conventionally, sequential processing of CNUs was adopted for designing LDPC-CC decoders [1], [8], [14]. It lowers the hardware cost, yet it limits the decoding throughput. Later, CNU-parallel decoders for both time-invariant [28], [29] and time-varying [10], [26] LDPC-CCs were proposed. Notably, with such node parallelization, the decoding throughput can be increased considerably and the decoding latency can be reduced. For instance, in [10], it has been reported that based on a small parallelization factor (= 8), the node-parallel decoder can achieve 400 Mbps with fc = 100 MHz compared with the sequential FPGA decoder in [8] that can reach 40 Mbps with fc = 120 MHz.

C. Minimizing hardware cost of LDPC-CC decoders

In previous sub-sections, we have shown that changing

the decoding algorithm, or switching from NMS to OMS,

or modifying the normalization factor can improve the error

correction performances of LDPC-CC decoders with a very

limited impact on hardware complexity. However, it is also

possible to benefit from error performance enhancement to

reduce hardware complexity at a same BER performance level.

The hardware complexity of LDPC-CC decoders increases

linearly according to the number of decoding iterations,

denoted as Q. Indeed, in most hardware implementations

the atomic processor architecture is tiled as many times as

required. In few others, the memory elements are duplicated

and the CN units are shared. Even though the memory cost is

highly prevailing, it does not change the acknowledgement.

Thanks to the error correction gain, it becomes possible

to reduce the value of Q while reaching the same BER

performance of the reference LDPC-CC decoder (NMS (0.75)

and Q = 10). This assertion is justified by Fig. 7. For instance,

V. CONCLUSION

In this paper, we have demonstrated that LDPC-CC hard-

ware architectures proposed in literature can be optimized

thanks to a judicious Min-Sum approximation selection. In-

deed, error performance gains varying from 0.1 to 0.2 dB

can be obtained with an insignificant hardware over-cost in

comparison with related works. Moreover, for a similar error

correction performance level, it can be used to minimize the

amount of decoding iteration executed and so the number
