Rapid Prototyping of Multi-Mode QC-LDPC Decoder for 802.11n/ac Standard
Qing Lu, Chiu-Wing Sham, and Francis C. M. Lau
Dept. of Electronic & Information Engineering, The Hong Kong Polytechnic University Hung Hom, Kowloon, Hong Kong
Tel: (852) 2766 6129, Email: qing.v.lu@polyu.edu.hk

Abstract— A multi-mode QC-LDPC decoder is proposed to satisfy the 802.11n/ac WiFi standard. With code-specific design, the overall performance of the decoder is enhanced while ensuring an on-the-fly reconfigurable ability. The proposed architecture has been synthesized using an FPGA for measurements. A stateof-art error rate and implementation complexity are reported. Meanwhile, the throughput has been increased to range from 382 MHz to 1852 MHz.
I. INTRODUCTION
With limit-approaching error performance and a favorable implementation flexibility, low-density parity-check (LDPC) codes [1] have gained tremendous popularity in the communication field. In particular, 802.11n/ac WiFi protocol has standardized 12 QC-LDPC codes, measuring 648, 1296 and 1944 bits by length and 1/2, 2/3, 3/4 and 5/6 by rate. To meet this standard, a multi-mode QC-LDPC decoder that can be reconfigured on the fly is proposed and specially tailored for overall compactness. The projected design has been verified and measured using an FPGA prototype.
II. THE ARCHITECTURE OF PROPOSED DECODER
The proposed architecture is illustrated in Fig. 1. In this decoder, a row-wised layered decoding schedule [2] with 27 parallelism degree is applied to ensure a satisfactory throughput as well as utilization efficiency. Since the 12 codes have various decoding patterns, we manage to reduce the hardware redundancies by subtly taking advantage of their resemblance. Therefore, the memory are partitioned into a fixed number of 24 banks each corresponding to one block column of the paritycheck matrices. Moreover, the a posteriori probability (APP) and check-to-variable (C2V) messages are to be stored so that a least number of ports requires the minimum workload for routing, and more importantly, the memory slots can be continually read and written with the allocation following check-node order and hence can be implemented simply using shift registers. The multiplexing load is bias toward the output side of the memory for further simplicity: although there are totally 12 different codes, the corresponding memory depth actually falls into only 8 cases.
As shown in Fig. 1, the decoding goes by initially infusing the channel messages (CM). After that, the subsequent iterations will begin by subtracting each APP by the corresponding C2V message. The result, variable-to-check (V2C) message, then undergoes the format conversion from 2’s complement to sign-magnitude (S2C) and a truncation before accessing the check-node processor (CNP) with the other 23 V2C messages associated with the same check node. After the CNP,

each updated C2V message is converted back to the 2’c complement format and stored back to the memory. Meanwhile, the APP message is also updated by adding the new C2V messages to corresponding V2C messages. A short-delay parallelrouting structure of the CNP has been implemented where the detailed design method is given in our prior work [3]. A lookup-table (LUT) method directly implements the offset min-sum (OMS) algorithm to approximate each output without necessities in finding the global first and second minimum values [4] and keeping track of the minimum value indexes.
Between successive layers, the messages alignment within each block column is fulfilled by a two-dimensional rotation. The spatial rotation relies on the barrel shifter (BS), while the temporal rotation operates on the shift registers with selffeedback. Given the check-wise order of memory allocation, the rotation occurs only once for each layer by considering the differences between consecutive layers. Totally, there are l + 2 offset scenarios for a code of l layers, considering the original order of the codeword bits.
The patterns of all the 12 codes are archived in the pattern ROM by a terse fashion. That is, regarding each sub-block of the parity-check matrix, 8 bits are assigned to record the corresponding cyclic-shift volume and the variable-check connection status. The former instructs the barrel shifter while the latter filters out the messages from the unassociated variable nodes. In our design using 4-bit quantization, the invalid V2C messages are replaced by the positive maximum quantized value (“0111”). Further, if the variable node is not connected, the corresponding APP message should not be updated but stored back as its original.

The proposed architecture has a comparable error performance with the state of arts. Compared to [5], the hardware is more efficient in terms of throughput, which is 23% higher on average even with a half clock rate. The LUT-based CNP has a higher accuracy than the traditional OMS method so the quantization bits can be saved. Moreover, the recourses dedicated to fulfilling multi-mode compliance is 15552 LUTs such that only 24% overhead has been introduced to the largest decoder of one mode to expand its functionality by 12 times.
IV. CONCLUSION This paper proposes a multi-mode QC-LDPC decoder following the 802.11n/ac standard. Techniques are applied for easy memory structure, minimized multiplexing, and enhanced timing restriction. The design has been synthesized and evaluated using an FPGA implementations showing overall improvement from the state of arts.
