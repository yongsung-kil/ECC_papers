Multi-Pass Joint Viterbi Detector Decoder (MP-JVDD) over AWGN/ISI Channels

Ashish James and Kheong Sann Chan Data Storage Institute (DSI), A*STAR (Agency for Science Technology and Research)

Abstract—In this paper, a new two-stage joint detector/decoder named the multi-pass joint Viterbi detector decoder (MP-JVDD) is proposed as an alternative to the iterative detector-decoder (IDD) and the originally proposed JVDD. The MP-JVDD operates on a trellis with the first stage estimating the minimum metric sequence (MMS) similar to a normal Viterbi algorithm, while the second stage utilizes the estimated MMS to perform a decoding search for the minimum metric legal codeword (MMLC). This is achieved by utilizing the parity check matrix (H) to identify survivor path transitions and illegal survivors in the trellis. The survivor path transitions occur only at nodes in the trellis corresponding to each parity check constraint. The corresponding survivors in the trellis are checked for parity upon reaching the last node of each parity check constraint in turn. The survivors violating the parity checks are discarded and thus only survivors corresponding to legal codewords make it through. In contrast to JVDD, the MP-JVDD considers path transitions of survivors only at the parity check nodes. However, this has to be repeated for every parity check in H. Herein lies the multi-pass nature of the algorithm - one pass per parity check. Each pass has substantially fewer survivors, since possible path transitions are considered for only the parity check nodes. This results in lower memory and computational requirements for the algorithm which has been verified through extensive simulation studies.
Keywords–Joint Detector/Decoder, Iterative Detector-Decoder, ML Detector/Decoder, JVDD, ISI Channel
I. INTRODUCTION Inter-symbol interference is introduced in many types of channels and is a major obstacle in current and future magnetic recording and wireless communication channels. To mitigate the interference and recover the transmitted signals, various detection algorithms have been proposed [1], [2]. Another effective means of tackling this problem is by integrating detection algorithms with channel coding techniques [3]–[6]. One such method that offers significant performance improvement is IDD based on the Turbo principle, i.e., exchanging (extrinsic) information between an inner detector and outer channel decoder [7]–[9]. Despite the success of the aforementioned approach there are several open issues for such an IDD scheme, e.g. severe detection/decoding delay especially for codes with short block lengths [4], [10] or prohibitively high computational complexity associated with IDD systems in general. Furthermore, it has a performance gap with the optimal maximum-likelihood (ML) decoder for any codestructure [11]. ML decoders have been analyzed over different channels such as the additive white Gaussian noise (AWGN) [12], [13], binary symmetric channel (BSC) [13] and binary erasure channel (BEC) [14] among others. Optimal ML decoding, characterized by minimizing the Euclidean distance between the received signal and valid codewords, can be performed on different classes of codes and have been extensively analyzed for two broad classes of codes namely block codes and convolutional codes. With convolutional codes, an efficient trellis

based algorithm known as the Viterbi algorithm (VA) can be employed to perform ML decoding by returning the minimum metric path through the code trellis [15]. However, optimal ML decoding of linear block codes has proven to be an NPhard problem [16], whose complexity grows exponentially with increasing codeword lengths. There have been many research efforts in this direction to develop optimal or suboptimal decoding algorithms for such classes of codes with moderate complexity [17]–[21].
The joint Viterbi detector decoder (JVDD) has been introduced as an alternative detection and decoding scheme that aims to return the MMLC [21]. The JVDD operates on a trellis comprising of two parts - metric thresholding and parity checking. In the metric thresholding section of the trellis, the JVDD executes the normal VA by computing metrics for every possible path to a node. However, unlike the VA, the JVDD retains the minimum metric survivor along with a certain number of competing paths in the trellis limited by a threshold parameter. The survivors with metrics larger than the defined threshold of the minimum metric are discarded for each node. Setting a larger threshold retains more survivors which in turn minimizes the probability of discarding the MMLC, and leads to better performance but with increased computational complexity. The parity checking section of the JVDD reduces the complexity by performing syndrome check on those incoming survivors (retained after the metric thresholding section) to a parity check node. The syndrome check identifies the validity of the received partial codeword by computing the syndrome: cˆHT , where cˆ is the hard decisions of the received data and H is the parity check matrix. A valid codeword has a syndrome weight of 0 and hence those survivors which result in non-zero syndrome correspond to failed parity checks and are discarded from the JVDD trellis.
The threshold parameter in the JVDD trades off performance for complexity. Small thresholds reduce the number of survivors in the JVDD trellis but risk discarding the MMLC that is sought. Hence, the JVDD algorithm is conditionally optimal over a coded AWGN/ISI channel with the condition being sufficiency of computational resources. The MP-JVDD algorithm proposed in this paper targets to reduce the complexity without performance degradation. This is achieved in the MP-JVDD by restricting the search for the MMLC to only those candidate codewords that satisfies each parity check in turn, which requires multiple passes. Similar to the JVDD, the MP-JVDD also performs the search for the MMLC on a trellis but performs detection and decoding in two stages.
The first stage of MP-JVDD processing initializes a list of low-metric survivors. This list initially consists of just the MMS obtained by executing the normal VA that returns the path with the smallest Euclidean distance from the received sequence. However, the estimated MMS might not correspond to a valid codeword and the second stage of processing is

This is achieved by considering each possible path transition evolving from a node that is involved in the parity check constraint and adding the newly generated survivors back to the list. The corresponding surviving paths in the trellis are retained in MP-JVDD and checked for parity upon reaching the last node in the parity check constraint. Those survivors which satisfy the current parity check are retained in the list for subsequent parity checks while those that fail are discarded. The multi-pass nature of the algorithm is apparent, where the second stage involves multiple passes through the trellis corresponding to each parity check constraint (row) of H. In this way, the MP-JVDD obtains survivors that progressively make through each parity check in turn. This reduces the number of survivors in the trellis as the path transitions occur only at nodes involved in the parity check for each pass as opposed to every node for the JVDD. This results in lower memory and computational resources for the algorithm.

A. System Model
At the transmitter, the length k information vector u = [u1u2 · · · uk] is first encoded utilizing a linear block code. A low density parity check (LDPC) encoder operating at rate R is considered which generates the length n codeword c = [c1c2 · · · cn], where the code-rate R = k/n. The corresponding (n, k) code is represented by a parity check matrix H = [hi j]m×n, where m = n − k corresponds to the number of parity checks and n is the number of coded bits. The coded bits are modulated into x = [x1x2 · · · xn] and transmitted over a channel. where γ(y j|c j) is the branch metric. The branch metrics are computed recursively and correspond to weight of trellis transitions across states. The sum in (4) represents the accumulation of the branch metrics along a given path through the trellis (path metric), with every path representing a codeword. The MP-JVDD performs both detection and decoding on the trellis and is described as follows:
1) Computing MMS: The first stage aims at reducing the search complexity by estimating the MMS through the VA. This initializes the list of survivors, populating it with just the MMS. In this stage, the branch metric for all possible transition from each state to every other state is computed and the path metric is recursively updated. The lowest metric path called the survivor path is retained for each state at each time step. For binary codes this process doubles the number of survivors by splitting at each step into two (one corresponding to appending a ‘+’ and the other a ‘−’), and then halves it by discarding the larger metric path. This computation proceeds until the algorithm reaches the end of the trellis (which is typically terminated by returning to the all-zero state) and returns the MMS corresponding to the maximum-likelihood path. Even though, the number of survivors in this stage is equal to the number of states and is much less than the total number of paths in the trellis (which corresponds to lower decoding complexity), the evaluated MMS probably might not correspond to the MMLC. This is handled in MP-JVDD in the second stage processing.
2) Multi-pass with Splitting at Parity Checks: The strategy of the second stage is to estimate the MMLC from the list of survivors that was initialized in the first stage along with those that will be fed back and used on each pass of the algorithm. This is done using a similar technique to the JVDD,

The differentiating idea for the MP-JVDD is to consider all possible path transitions from nodes that are involved in the current parity check constraint. For binary codes, the incoming survivors to a node associated with the parity check constraint is split into two new survivors. The remaining nodes that are not involved in the current parity check have transitions corresponding to the survivor path under consideration. This involves obtaining different survivor paths by computing metrics (branch and path) for each survivor evolving from a node associated with the current parity check constraint. One addendum in the MP-JVDD algorithm is that the splitting of survivor paths will happen only once for a particular parity check node even if it is associated with more than one parity check constraint. This is because the possible survivors emanating from a node has already been decided at its first occurrence in the parity check constraint. To split the survivor path again at the same node could possibly reverse a decision made at the earlier parity check constraint that a particular survivor should not make it through the MP-JVDD trellis.
The MP-JVDD retains all survivor paths emanating from a particular node (involved in the current parity check constraint). However similar to the JVDD algorithm [21], threshold parameter is utilised for the MP-JVDD algorithm to limit the computational complexity and memory requirements. Thus only survivors with metrics within the threshold of the minimum metric are retained for a particular node. In this way, survivors are propagated down the trellis by splitting only at the parity check nodes and then curtailing their numbers via the predetermined threshold until it reaches the last node of the current parity check constraint (which corresponds to the last one of the current row in the parity check matrix). At these particular nodes (similar to the JVDD), all bits required to perform the check are detected and the MP-JVDD performs the parity checks on each incoming survivor by computing the syndrome. The list of survivors are then updated with those survivors that satisfy the parity check and is fed back for the next pass of the algorithm which corresponds to the next parity check constraint. C. Parity check matrices In the second stage of MP-JVDD processing, the possible
survivors emanating from a node are considered for only those that are involved in parity check constraints. Therefore, the parity check matrix has to be designed with each codeword symbol participating in at least one parity check constraint to ensure the algorithm searches for all possible codewords from the MMS obtained in the first stage. In this context, the present paper considers the JVDD class of codes known as the variable-gradient Gaussian distribution linear diagonal (VGGDLD) codes with uniform column weight (wc). In other words, every symbol node (corresponding to codeword symbols) is involved in wc parity checks. JVDD codes were designed to evenly space the parity checking functionality throughout the trellis resulting in fewer number of survivors. This is achieved in VGGDLD codes through a diagonal construction, with the gradient of the diagonal controlled by two independent shift parameters, ρ and τ, where ρ is the offset from the main diagonal to the right at the top left corner and τ is the offset from the main diagonal upwards at the bottom right corner. It is observed that increasing the parameter ρ leads to increased number of survivors in the trellis as the parity checking gets shifted in the trellis, thus the optimum value is set to ρ = 0. However, the parameter τ has to be designed optimally. Too small a value leads to under-protected bits at the end of the codeword, and too large a value reduces the number of parity checks which increases the complexity of the trellis. IV. CONCLUSION
In this paper, a novel, more computationally efficient joint detector and decoder, the MP-JVDD is proposed. The MPJVDD operates on a trellis with two stage decoding structure. The first stage estimates the MMS using the regular Viterbi algorithm. The second stage considers all possible path transitions at parity check nodes and additional computations are performed whenever the bit corresponding to the last one in any row of the parity check matrix H is detected. At these positions, the syndrome for each surviving path in the MP-JVDD trellis is evaluated and those which satisfy the current parity check make it forward to be considered for the subsequent parity checks. This highlights the multi-pass nature of the algorithm with one pass required for each parity check constraint in H. In this way, the MP-JVDD obtains a list of survivors corresponding to legal codewords that progressively satisfy one parity check at a time and involves fewer survivors as only those survivors evolving from each parity check node are retained. This results in lower memory and computational resources for the algorithm. However, the multi-pass nature of the MP-JVDD does end up increasing the algorithm latency. In effect, the MP-JVDD gives us the latency as an additional algorithm parameter to trade-off, on top of the performance

and computational complexity.
