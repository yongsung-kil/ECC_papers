I. 
INTRODUCTION 
Since their rediscovery by Mackay and Neal in 1996 
[1],[8], the Low-Density Parity Check (LDPC) codes have 
been the focus of many researches on error correcting codes 
due to their capacity-achieving error performance. LDPC 
codes not only have remarkable performance, but also have 
low complexity for decoding (due to its sparse parity-check 
matrix [9]). Furthermore, they are suited for implementations 
that make heavy use of parallelism with potential for high­
speed decoding. Therefore LDPC codes are widely used in 
Mobile Communication System, digital television, deep 
space communication and so on. The main decoding 
algorithm of LDPC codes is Belief Propagation (BP) 
algorithm [1],[10]. When decoding, belief (Le. message) is 
propagated along the edges in Tanner Graph (TG) [11]. BP 
algorithm has the best performance but is not suited for 
implementations with plenty of multiply and division 
operations. Then, Logarithmic Belief Propagation (Log-BP) 
algorithm was proposed which converts multiplication into 
addition by using a log-likelihood-ratio (LLR) [2]. To further 
improve the performance of traditional BP decoding 
algorithm, optimization algorithm was introduced into LDPC 
codes. In [3], Genetic Algorithm (GA), a fast search and 
optimization algorithm, was combined with BP algorithm. 
GA has been applied to search good performance LDPC 
codes which are friendly to hardware implementation in [4]. 
A.G. Scandurra et al. applied GA to LDPC decoder which 
improved the performance slightly with much higher 
computing complexity to get rid of estimation of the channel 
information [9]. All these above show great potential of 
optimization algorithm for improving performances of 
LDPC codes. 
Similar to GA, Particle Swarm Optimization (PSO) is 
also an evolutionary computational model which is based on 
swarm intelligence and population which was originally 
attributed to Dr. Kennedy, Dr. Eberhart in 1995 [12]. At first, 
they were trying to simulate the migration and aggregation of 
bird flock when they seek for food, while they subsequently 
found that PSO was a good optimization method. Compared 
to GA, the PSO has much more profound intelligent 
background and could be performed more easily. It has the 
merits of high precision and quick convergence which 
attracts great academic interest and shows its superiority in 
solving practical problems. At present, PSO algorithm is 
attracting more and more attention of researchers, and has 
been widely used in fields like function optimization, neural 
network training, pattern recognition and so on [5]. In 
addition, the study of PSO algorithm also has infiltrated into 
electricity and communications. In [6], PSO algorithm was 
used to design near Shannon limit LDPC codes. PSO was 
also used to optimize the degree distribution pairs in irregular 
LDPC codes [13]. 
In this paper, a modified Log-BP decoding algorithm 
combined with PSO is proposed to improve the performance 
of the traditional Log-BP decoding algorithm. We begin the 
paper in Section II with introduction to the traditional Log­
BP algorithm and then an introduction to basic PSO 
algorithm in Section III. In Section IV we exploit the 
marriage of Log-BP and PSO algorithm. Simulation results 
of BER performance are presented in Section V. And 
conclusion is drawn in Section VI.

BASIC PSO ALGORITHM 
PSO algorithm is swarm intelligence based evolutionary 
computation technique [7]. Similar to other evolution 
algorithms, PSO algorithm adopts a strategy based on 
competition and cooperation between particle swarms to 
realize parallel global random search for optimal solution. 
PSO algorithm randomly initializes a group of particles to 
generate original swarm. Each particle is potential solution 
for the optimization problem and has an adaptive value 
determined by the object function. The particle flies in search 
space with the flying velocity determining the direction and 
step of search. Generally, the particle tracks the best position 
found so far by certain numbers of generations to find the 
optimal solution. In each generation, particles swarm tracks 
two best current positions respectively found by particles 
themselves (pbest) and that of the whole swarm (gbest) [8].

IV. 
PROPOSED ALGORITHM 
PSO can also be regarded as an optimization algorithm 
by message-passing, which makes it possible to combine 
Log-BP with PSO. This paper proposes a way to achieve it. 
Take Fig.I for example, Tanner Graph contains two 
kinds of nodes, the bit nodes also called the variable nodes 
that are shown as circles corresponding to the ten symbols 
(VI, V2, ... , VJO) and the check nodes shown as squares in 
Fig.I. The arrows from bit nodes to check nodes represent 
the messages that bit nodes send to check nodes, such as the 
arrow from VI to C. Conversely, the arrows from check 
nodes to bit nodes represent the messages that check nodes 
send to bit nodes, such as the arrow from CI to VI in Fig.I. 
According to the TG, all of the messages that the n-th bit 
node sends to the m-th check node can be regarded as the 
swarm with each message as a particle. In this way PSO 
algorithm can be introduced into Log-BP decoding algorithm.

Generally, PSO algorithm randomly initializes a group of 
particles to generate original swarm. However, in the 
decoding process of LDPC codes, if the original swarm were 
generated in this way, these particles and channel message 
would have little correlation. Consequently, PSO can only 
converge through lots of iterations. Therefore, we set the 
L41m (cn) got after the first BP iteration as the original 
value. L41m (cn) represents the reliability of this message (its 
absolute value presents the accuracy of decoding, the bigger 
its absolute value is, the bigger the probability that the bit has 
a value of "1 " or "0" is [14]). 
In addition, when all the bit nodes involved in the m-th 
check have least wrong bits and then choose the L45 (cn) as 
local optimum (pbest) when its absolute value is biggest. 
Accordingly, set L41m (cn) as global optimum (gbest) on 
condition that the decoded word has least wrong bits. 
Proceed with PSO algorithm when L41m (cn) is updated 
in each iteration of BP by following rules: 
1) 
Update optimum value: Update hard decision and 
count wrong bits in decoded codeword as criterion for 
optimum value. 
2) 
Update optimum value: Update hard decision and 
count wrong bits in decoded codeword as criterion for 
optimum value. 
3) 
Update pbest value. 
4) 
Update gbest value. 
5) 
Update velocity and position: Generate new swarm 
according (7) and (8). 
6) 
Stopping criterion test: Termination condition is that 
the number of iterations exceeds predefined T max or get a 
valid codeword. 
To sum up, modified Log-BP algorithm combined with 
PSO can be expressed as following. 
1) 
Initialization: Initialization for Log-BP algorithm. 
2) 
Update check nodes. 
3) 
PSG processing: 
a) End decoding process if valid codeword has been 
gotten. 
b) If the number of PSO iterations exceeds predefined 
Tmax but the number of Log-BP iteration is less than 
predefined maximum number, end PSO process and 
continue the Log-BP process; if the number of Log-BP 
iteration is also exceeds predefined maximum number, the 
decoding process ends and c is the final decoded codeword. 
4) 
Back to Log-BP iteration: Update bit nodes and the 
decoding repeats from 2).

CONCLUSION 
An algorithm based on Log-BP algorithm and PSO 
algorithm is presented. Simulation results show that the 
Modified Log-BP decoding algorithm combined with PSO 
has better decoding performance than standard Log-BP 
algorithm, which shows a wide field of application with good 
prospects in LDPC codes decoding where is strict with 
decoding performance and complexity.
