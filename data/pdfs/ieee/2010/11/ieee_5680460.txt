Analysis of Error-Prone Patterns for LDPC Codes

under Belief Propagation Decoding

Abstract—Lowering the error floor of LDPC codes is extremely attractive to some systems, such as deep space communication systems and storage systems, which desire very low error rates. The error floor phenomenon of LDPC codes, which is associated with their message passing decoding algorithms, is mainly caused by some unfavorable combinatorial characteristics (or errorprone patterns) of LDPC codes. In this paper, a mathematical analysis method which enables the identification of some significant features of error-prone patterns is presented with the help of the beliefs passed in their decoders. Based on the analysis, an improved decoder is proposed which can effectively deal with the traversable trapping sets and achieve significantly improved error floor performance compared with current decoders. More importantly, this proposed decoder does not require the information of all the possible trapping sets of an individual LDPC code when correcting the error bits in its trapping sets.
Keywords-LDPC code; error floor; belief propagation (BP) algorithm; iterative decoding; trapping set; error-prone pattern; two-stage decoder; pesudo-cycle.
I. INTRODUCTION
Low-density parity-check (LDPC) codes, first proposed by Gallager [1], have been extensively studied since their rediscovery [2-4] because of their excellent Shannon-limit performance when decoded using low complexity iterative decoding algorithms. Message passing algorithms (MPAs) are a general class of low complexity iterative decoding algorithms. When decoding an LDPC code, the MPA proceeds in the code’s corresponding bipartite graph. If the messages in a message passing decoder are continuous reliable probabilities of variable node values, these probabilities can be called beliefs and this decoding algorithm is referred to as a belief propagation (BP) algorithm [2, 4, 5]. The BP decoding algorithms are a suboptimal class of MPAs. In [6], the BP algorithm was extensively analyzed over a large class of channels with the help of density evolution tools.
With the assumption that the incoming beliefs are independent at every round in the decoding process, the BP decoding algorithm is able to efficiently decode an LDPC code transmitted over a channel below its threshold [6]. However, this assumption cannot always be guaranteed especially when the number of iterations becomes large because of some unfavorable combinatorial characteristics in LDPC codes. These combinatorial characteristics are conventionally called low-weight near code words [7], or trapping sets [8-9] (called stopping sets if codes are over binary erasure channels (BECs)). The existence of such combinatorial characteristics

may lead to the high error floors of LDPC codes [9]. Because the messages passed in the BP decoding algorithm are related to the channels over which codes are transmitted, we need to investigate those combinatorial characteristics of LDPC codes according to the channel type. In [8], those combinatorial characteristics which result in the error floors of LDPC codes over BECs were completely analyzed. But for LDPC codes over other binary symmetric channels (BSCs), this kind of combinatorial characteristics is collected only by using experimental searching strategies and Important Sampling method [9-14]. Exhaustively enumerating all such possible combinatorial characteristics is profoundly unpleasant and often unfeasible. Therefore, a systematic analysis of such combinatorial characteristics is needed in order to analyze the performance of any specific code and lower its error floor. This work will present a mathematical analysis method of identifying some significant features of the possible unfavorable combinatorial characteristics of LDPC codes over binary input AWGN channels (BIAWGNCs).
One important purpose of identifying bad combinatorial characteristics for LDPC codes is to analyze and lower their error floors. Based on the identified bad combinatorial characteristics, many approaches have been explored to improve the error floor performance of LDPC codes [11-22]. However, these approaches either need all the codes’ possible trapping sets or work only for some specific codes. In this work, we will describe a two-stage decoder which can effectively improve the error floor performance of any LDPC codes transmitted over BIAWGNCs.
The rest of this paper is organized as follows. Section II briefly introduces some terms and the BP algorithm. In section III, the features of possible unfavorable combinatorial characteristics for BECs and BIAWGNCs are characterized by analyzing the BP decoding algorithm for LDPC codes. Section IV provides an effective approach for mitigating the impact of these characteristics. Section V provides simulation results. Finally, section VI concludes this work.

B. Stopping Sets and Trapping Sets
When decoding a finite length LDPC code using MPA, its error floor performance may decrease suddenly [9] as the Eb/N0 increases, which is known as an error floor phenomenon. This performance degradation is mainly attributed to some bad combinatorial characteristics of the code. In this work, a general term, error-prone pattern [21], will be used to represent this kind of unfavorable combinatorial properties. One type of such error pattern, stopping sets, is given in Definition 2.1.

Definition 2.1 [Stopping Set] If an LDPC code is transmitted over a BEC, its error-prone pattern can be defined as a set of VNs whose neighbors are connected to this set twice or more in the subgraph induced by this set of VNs. Such an error-prone pattern is called a stopping set [6, 8].
Since there are three possible output values over BECs, i.e. certainty value 0, 1, and erasure value İ, we generally set the messages of BP at each round using this rule [26]: the processed result of a VN will be a certainty if there is at least one incoming certainty message; otherwise, the processed result of this VN will be an erasure value; for a CN, its processed result would be a certainty value iff all incoming messages are certainty; otherwise, its result would be an erasure value. Combining this rule with Definition 2.1, we can explain why the stopping sets can degrade the code’s performance in the error floor region under BP decoding when message bits are transmitted over a BEC. The definition of stopping sets and their roles were well explained in [6] and [8].
If the LDPC codes are transmitted over other binary memoryless channels, such as BSCs and BIAWGNCs, the error-prone patterns cannot be analyzed as easily as stopping sets. By simulating LDPC codes over BSCs and BIAWGNCs, Richardson observed another type of error-prone patterns which dominate the error floors and named them trapping sets [9]. However, Richardson did not give any general analysis method of identifying trapping sets. He only presented some combinatorial characteristics of this kind of VN sets according to his observation. The definition of trapping sets is given in Definition 2.2.
Definition 2.2 [Trapping Set] a trapping set T(v, c) is a set of v VNs that are not successfully decoded in the decoder, and the subgraph induced by T(v, c) has c CNs with odd-degrees.
Based on the studies of [7] and [9], many papers [11-14, 27] were published on the distribution and influence of trapping sets. At the same time, some subtle strategies of mitigating the negative influence of trapping sets and lowering the error floors of LDPC codes [11-13] were stated. However, these studies are mainly based on the observed trapping sets and only work for some special codes. In section III, we will analyze these error-prone patterns for LDPC codes over BECs and BIAWGNCs. Based on the beliefs passed in the BP decoding process, some features of error-prone patterns in LDPC codes will be discussed.
III. ERROR-PRONE PATTERNS UNDER BP DECODING
A. Error-prone patterns for LDPC Codes over BEC
According to Eq. (5) and the discussions in Section II, any output of a CN with degree l is dependent on the product of (l1) hyperbolic tangents of its inputs. Therefore, any output of a CN will be an erasure if its two or more inputs are erasures. Based on this observation, a necessary condition of stopping a decoding process can be obtained: for a set of VNs, each neighbor of this set (or subgraph) has two or more erasure inputs at each round of decoding. From Eq. (5), we can see that the decoding process will fail if each neighbor of a set of VNs with erasure values is connected to this set twice or more. Therefore, under BECs, a set of VNs with the above combinatorial property can be identified as an error-prone

B. Error-prone patterns for LDPC Codes over BIAWGNCs
The analysis of BP decoding over BIAWGNCs is more complicated compared with BECs. In [9], Richardson defined the decoding failure of an LDPC code over BIAWGNCs based on a set of incorrect VNs which is called a trapping set or failure set T(y), i.e., after a certain large number of iterations, the decoding fails if T(y) is not empty. For convenience, we will use T(y) and the subgraph induced by T(y) interchangeably, and message bits and VNs interchangeably.
For convenience, we classify the neighbors of T(y) into two types. One type is the CNs that are only connected to VNs in T(y), and named as in-check-nodes (ICNs). The other is named out-check-nodes (OCNs), which are connected not only to VNs in T(y) but also to VNs outside of T(y). For a T(y), most CNs connected to it are OCNs. For either OCNs or ICNs, if the correspondent check equations are satisfied, they cannot provide meaningful beliefs to their neighbors and are not able to help correct the message bits in T(y). This type of check nodes are called mis-satisfied CNs in [11, 14]. In this paper, only OCNs will be considered when analyzing the beliefs related to T(y) because the ICNs have similar influence on T(y) to OCNs. IV. IMPROVED TWO-STAGE DECODERS
There are few studies on decoder improvements for solving the error floor problem caused by trapping sets. In [14], a postprocessing (PP) approach was proposed by modifying the decoding procedure: the decoder first runs as a conventional BP decoder for certain number of iterations or until successfully terminated; if the decoder gets a non-zero syndrome, a look-up table of known trapping sets will be used to locate the error bits. This PP technique can effectively decrease the negative influence of trapping set under an assumption that a set of unsatisfied CNs has a one-to-one correspondence to a trapping set. However, the application of this approach will be restricted if the cardinality of trapping sets of an LDPC code is large. In [11-12], the authors proposed a bit-pinning technique which erases the bits of each trapping

set of interest before transmission. This bit-pinning approach only works when there is little trapping set overlap after bitpinning. In [12], the authors also put forward two other approaches for improving a decoder. One is called a generalized-LDPC decoder which adds CN processors to deal with the unsatisfied checks in the trapping sets. The other is a bi-mode syndrome-erasure technique which mitigates the negative influence of trapping sets by combining the conventional SPA decoding and BP decoding over BECs. This method is applicable only when the erasure bits do not form any stopping set.
In this paper, a two-stage decoder is proposed. The idea of building a two-stage decoder is based on this observation: for a cycle-free LDPC code with infinite length, it may be successfully decoded using the BP algorithm when transmitted at a rate below its density-evolution threshold, i.e., a wrong message bit can be corrected with any arbitrarily large possibility by the correct beliefs coming from its neighbors when running a BP decoding algorithm; However, if a wrong message bit is included in a short cycle, this bit may not be able to receive correct information after certain number iterations, because its information travels back to itself; this is the reason for requiring the independence assumption for MPA decoding; in real applications, even though there is no cycle, a wrong message bit may still be unable to be corrected because of the existence of some other wrong bits which could send wrong information to this bit after certain number iterations, which resembles a state of having a cycle. In this paper, we say these wrong bits form a pseudo-cycle. Generally, a trapping set is composed of small pseudo-cycles (some small real cycles may also exist in a trapping set). In a trapping set, the shortest pseudo-cycles can be identified by the shortest paths between two unsatisfied CNs because these CNs are able to send wrong information to their neighbors. If a stopping set is traversable, most, if not all, message bits in this trapping set can be located by those shortest paths. The task of a two-stage decoder is to identify most (or all) message bits in a trapping set and further break this trapping set.
In the first stage of a two-stage decoder, a conventional BP decoding algorithm is run like the PP decoder. If an error occurs, the decoder goes to the second stage. In the second stage, the decoder enumerates all the possible shortest paths between any two unsatisfied CNs. After that, the decoder collects all the VNs in these paths and flips the values of these bits. Based on our observations, in the error floor region, most paths are purely composed of trapped bits. Therefore, in the error floor region, the modified codeword after flipping should be a correct codeword or have very few error bits. After rerunning the conventional decoding process using the modified codeword as the decoder’s inputs, the codeword can be correctly decoded with high probability. The exceptional merit of this two-stage decoder is that it does not need to know the format and cardinality of trapping sets when solving the error floor problem. Therefore, this approach can be applied to any specific LDPC code and eliminates the worries of exhaustively enumerating trapping sets. This approach can effectively correct a class of commonly occurring trapping sets, i.e., traversable trapping sets T(v, c) with c no less than 2, by monitoring the beliefs passed in the decoding process.

easily, we can set a marker for each CN in the conventional BP
decoding process. The general picture of setting markers is as
follows: first, set all CNs’ markers to zero; during the decoding
process, if an incoming belief is inconsistent with the value of a
VN, the CN which sends out this belief is an unsatisfied CN
and its marker is set to 1. Based on the markers of CNs, we can
apply the previously mentioned algorithm to any two CNs
sending inconsistent belief. For paths between two unsatisfied
CNs in the subgraph, some of them may not be purely
composed of the VNs in a trapping set, even though this kind
of paths is rare according to our observation. One such example

is shown in Fig. 2. Therefore, if all bits in those paths are flipped, the number of error bits may be greatly decreased (possibly to zero). If the updated codeword is used as inputs to the decoder once more, the codeword can be correctly decoded with high probability. The unsatisfied CNs can also be located by using a syndrome because all unsatisfied check nodes exactly correspond to the nonzero elements in a syndrome.
The space and time costs of this two-stage decoder are dependent on two aspects: the cost of BP algorithm in the first and second stage, and the cost of searching trees in identifying the VNs in a trapping set. After bit-flipping, the same conventional decoder hardware can be used for the second pass. Therefore, the costs of BP decoding processes in the first and second stage are roughly equal. Based on our observations, the cost of building searching trees is usually less than the cost of BP decoding process, and it can be loosely estimated by ݇ ή ሺ‫ݒ‬Ȁ݆ሻௗೌೡ, where k is the number of trees, v and j are the numbers of variable nodes and odd-degree check nodes in the subgraph, ݀௔௩ is the average node degree of subgraph. VI. CONCLUSION
Some features of error-prone patterns for LDPC codes under BP decoding were analyzed based on the beliefs passed in the decoder. With the help of these features, a two-stage decoder is proposed which can effectively mitigate the negative influence of trapping sets on the error floors of LDPC codes without knowing their trapping sets.
