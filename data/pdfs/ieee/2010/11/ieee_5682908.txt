Reduced-complexity decoding of low-density parity check codes based on adaptive convergence

1Jia-Ning Su, 2Zhenghao Lu*
Department of Microelectronics, Soochow University Suzhou, 215006, China *luzhh@suda.edu.cn
Abstract—Low-density parity-check (LDPC) codes have recently been considered as a viable candidate for forward error correction in system level hardware-redundant, fault-tolerant logics. An important factor that influences the choosing of a specific FEC technique in a nano-scale system implementation is its real-time performance, namely its computational complexity. In this paper, we propose a set of rules to decide whether a variable node in a LDPC decoder should update its value in subsequent iterations of the decoding process, or be considered as converged. We show that by carefully choosing the convergence rules for variable nodes, significant reduction of decoding complexity can be achieved with endurable performance loss.
Keywords-LDPC; adaptive convergenc; min-sum decoding
I. INTRODUCTION
Low-density parity-check codes have recently been selected as forward error correction scheme by a number of commercial standards and real-time applications due to their capacityapproaching performance and highly parallel decoding structure. It is well established that long (>106) LDPC codes are very good; however the actual LDPC codes used in standards are always several orders of magnitude shorter. A major concern in practical implementation of LDPC codes is the computation complexity in decoding, which greatly influences the real-time performance and the power consumption.
The most frequently used decoding scheme for LDPC codes is the Log-likelihood Ratio Belief Propagation (LLR-BP) algorithm proposed by Gallager in 1963 [1]. In LLR-BP algorithm, decoding is done by passing LLR values on edges between nodes of the bipartite “Tanner graph” that corresponds to the parity check matrix of the LDPC code. Decoding practices show that the computation complexity is mainly related to the following four factors:
1) The precision of calculations at each node.
2) The number of calculations at each node.
3) The number of active nodes in each iteration.
4) The number of iterations in each decoding.
The first factor concerns the accuracy of calculation per node which is crucial to decoding performance, the problem of obtaining a good trade-off between calculation precision and

III. ADAPTIVE CONVERGENCE DECODING
Adaptive convergence technique can lower the decoding complexity by reducing the number of active nodes in every iteration. The feasibility of this technique is based on the fact that during decoding process, quite a number of variable nodes have a tendency to converge after a small number of iterations and their outgoing LLR values change very little in subsequent iterations, and the LLR values of such nodes needn’t to be updated. The task of adaptive convergence is to identify these inactive nodes to facilitate decoding.

The most direct and simple way to screen out the inactive nodes is to set a threshold Tvar to the decision LLR value of a variable node, so the convergence rules:

B. The refined adaptive convergence rules:
The adaptive convergence with single threshold is always not enough to make a satisfactory trade-off between decoding complexity and performance, so more optimized rules are needed. Here three supplementary convergence conditions are proposed to use together with threshold rule.
Decoding process shows that the LLR values of variable nodes tend to change rapidly during the initial iterations, so it is hard to tell which bits are stable during the initial “turbulent” stages. The number of iterations taken for all the channel information to be propagated throughout the graph at least once

is the diameter of the graph, which is typically 6 for a code of length 1080. So the first additional proposal:

• A stand-by period of 6 should be inserted before normal convergence rules take effect in order to avoid the turbulent period at the start of the decoding.

Usually before a variable node is forced to converge, it tends to get more and more certain as iterations increase, and the unstable variable nodes are unlikely to exhibit such behavior. So when a variable node gets progressively more certain on successive iterations, it can always be safely predicted to converge. So the second supplementary convergence condition:

V. CONCLUSIONS
We present a decoding scheme based on “adaptive convergence” that can effectively reduce the number of active nodes in decoding process. A Threshold based convergence rule combined with a set of refined constrains is proposed to achieve a good tradeoff between complexity and performance.
Simulations show that the refined convergence rules with appropriate parameters can gain a complexity reduction of 35% with a performance loss of 0.2~0.3db at BER = 10-5.
