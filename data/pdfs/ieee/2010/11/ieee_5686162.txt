A Tree-based ML Decoding Algorithm with Adaptive Thresholding
Jing Cai, Martin Tomlinson, Cen Tjhai and Li Yang Fixed and Mobile Communications Research University of Plymouth PL4 8AA, United Kingdom
Email: jing.cai, martin.tomlinson, cen.tjhai, li.yang@plymouth.ac.uk

Abstract‚ÄĒIn this work, we propose a maximum-likelihood (ML) soft-decision decoding algorithm of linear block codes based on an enhanced tree-based search algorithm. The algorithm considers each codeword as a tree branch and explores the subtree corresponding to the more reliable basis (MRB) of received vectors. By making use of the pre-defined ensemble branches and their costs, the branch evaluation process is effectively controlled by adaptive thresholding. This paper details the implementation of the proposed algorithm, compares the performance with the classic ordered statistic decoding (OSD) algorithm, and discusses the general decoding complexity and search effort in comparison to that of the OSD algorithm.

For linear block (n, k) codes, the decoding algorithms with

the use of the soft-decision received vector are generally called

soft-decision decoding, which normally are categorised as

reliability-based decoding algorithms and code structure-based

decoding algorithms. As a classic reliability-based decoding

algorithm, the ordered statistic decoding (OSD) algorithm,

introduced by Fossorier and Lin [1], with the order of i,

decoding algorithm [2], in which, hard-decisions are derived

from the soft decisions using standard bit-by-bit detection, and

it obtains the result by choosing the binary state closest to

the received co-ordinate. The idea of the Dorsch algorithm

later was extended by Tomlinson et al. [3], which involves a

technique to treat (n‚ąík) less reliable soft decisions as erasures

in determining candidate codewords.

In recent years, various tree-based algorithms were proposed

heuristically. By utilising the hierarchical property of tree data

structure, in 2005, Rosnes and Ythehus [4] proposed a precise

but exhaustive tree-based search algorithm on the stopping sets

of turbo codes, which has been known as the ‚Äúexhaustive

tree-search algorithm‚ÄĚ. In 2007, Rosnes and Ythehus [5]

further upgraded the algorithm and employed it to find all

the stopping sets and codewords for low-density parity-check

(LDPC) codes. The upgraded tree-based algorithm improves

the search efficiency by giving a preset limitation on the search

scope. In 2009, Ambroze et al. [6] developed Rosnes algorithm

into a tree-based weight spectrum evaluation algorithm for linear block codes.
In this paper, we propose a novel ML-approaching algorithm which is a combination of the reliability-based decoding algorithm and the code structure-based decoding algorithm. Section II gives a description on the code tree and defines different types of code trees based on their growing directions. In Section III, the branch-evaluation search algorithm based on code trees, abbreviated as the BESCT, is introduced in details. Then, we apply the BESCT decoding algorithm to different linear codes and compare the performance and computational complexity with the OSD algorithm in Section IV. Conclusions and a discussion of future work are given in Section V.
II. TREE-BASED REPRESENTATIONS OF LINEAR BLOCK CODES
The code tree is a graphical code representation by tree structure, which is composed of branches and vertices. The set of branches is written as B = {B0, B1, . . .} and the set of vertices is written as V = {v0, v1, . . . , vn}. The arborescence traits can be represented mathematically by a given specific order, which is controlled by code properties. Suppose the binary linear code C(n, k) has its generator matrix and the parity-check matrix as G and H respectively. Each codeword ci ‚ąą C constitutes a complete branch Bi of its corresponding code tree TC, where the index i is in the range of 0 ‚Č§ i ‚Č§ 2k ‚ąí 1 ‚ąó. Therefore, a complete code tree should contain 2k branches.
A complete branch Bi corresponds to a codeword ci = (z0z1z2...zn‚ąí1), which has n spans. Each span is denoted as bi,j and it has the value of zj. The set of vertices {v0, v1, v2, . . . , vn} can also be considered as the indication of the tree growing direction. At each vertex, the branch diverges into q spans when the applied code is over GF (q). In this work, we only consider binary linear codes and henceforth, at each vertex, there are at most 2 diverged spans. The vertices of v0 and vn represent the ends of the tree. Different to the trellis representation of a code, the code tree does not have to start from the vertex of v0. It can be initialised from any intermediate vertex vj, where j can be arbitrarily chosen from 0 to n. The initial growing vertex is designated as vinitial.

V. CONCLUSION
In this work, we have introduced an efficient tree-based algorithm to maximum likelihood soft-decision decoding by deploying adaptive thresholding. The BESCT algorithm returns a codeword branch which maximises the likelihood of the corresponding received vector. By the definition of bidirectional code tree, the branch evaluation process is constrained within the subtree of more-reliable sequence. The pre-calculated ensemble branches and their costs heuristically indicate the search/growth direction and avoid unnecessary branch explorations. We have compared the performance of the BESCT algorithm to that of the classic OSD algorithm.
