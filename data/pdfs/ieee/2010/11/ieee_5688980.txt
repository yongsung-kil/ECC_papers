An FPGA Design of Low Power LDPC Decoder for High-speed Wireless LAN
Min Hyuk Kim, Tae Doo Park, Chul Seong Kim, Ji Won Jung
Korea Maritime University Korea

Ahstract-This paper proposes three kinds of simplified complexity-reduced algorithms for a low density parity check (LDPC) decoder. First, sequential decoding using a partial group is proposed. Second, an early detection method for reducing the computational complexity is proposed. Finally, in order to support the high-speed systems, we implemented LDPC decoder using three algorithms by N=648, R=1/2. And the decoder runs at a clock speed of IOns. Therefore we implemented the LDPC decoder with the decoding speed of 11OMbps.

I.

INTRODUCTION

IEEE 802.11n, known as the standard for wireless local access network(WLAN) system, employs a low density parity check (LDPC) coding technique as a channel coding scheme. LDPC codes, first introduced by R.G.Gallager decades ago and rediscovered by Spielman, Mackay and many others, can achieve very good performance when decoded with the belief­ propagation(BP) or the sum-product algorithm.[l][2].
In aspect to LDPC decoding process, LDPC codes have an easily parallelizable decoding algorithm, which consists of simple operations such as addition, comparison, and creation of a look-up table. Moreover, the degree of parallelism is 'adjustable', which makes it easy to trade-off both throughput and complexity. The standard recommends that the LDPC coded block size be 1944, 1296 and 648 and the number of iteration be about 8. Conventional method of LDPC decoding in general give rise to a large number of computation operations, mass power consumption, and decoding delay. It is necessary to reduce the iteration numbers and computation operations without performance degradation in order to implement an LDPC decoder with low power consumption. This paper proposes three kinds of simplified complexity­ reduced algorithms. First, sequential decoding with a partial group is proposed. It has the same hardware complexity, and a fewer number of iterations is required at the same level of performance in comparison with a conventional decoder algorithm. The computation of bit node weights and check node weights can seem to be as an approximate projection based on the parity-check matrix; the grouping simply says that this projection can be done in several steps to obtain an approximate answer. Secondly, an early detection method for reducing the computational complexity is proposed. Using a confidence criterion, some bit nodes and check node edges are detected early on during decoding. In this way, because early detected edges are not computed from following iterations, the computational complexity of further processing is reduced.

B. Early Detection Algorithm
Another approach is early detection for reducing the computational complexity. The early detection method is based on the observation that bit nodes and check nodes with a high log-likelihood value can be considered reliable. So detected bit nodes and check nodes are negligible for the next iteration. This means that we don't need to calculate the bit node and check node updates for a detected node from following iterations. Therefore, computational complexity of the next iterations is reduced. In this Section, we describe hardware implementation for our LDPC decoder. To implement an LDPC decoder based on low complexity algorithms, we employ partial-parallel architecture. The top level of block diagram of our LDPC decoder is shown in Fig 5. which is composed by Finite State Machine unit (FSMU), input buffer unit (IBU), received data register (RDR), output buffer unit (OBU), check node update unit (CNUU), log-likelihood ratio memory unit (L-Memory), Bit Node Update unit (BNUU).
A. Buffer Unit
LDPC decoder works simultaneously on consecutive received words. While one received word is being iteratively
decoded, the next received word is buffered in IBU with N * q
bits until decoder has been finished previous received words. RDR buffers received date in order to calculate BNU simultaneously until iterations have finished. The OBU buffers the decoded words for match LDPC input and output clock frequency. The L-memory unit buffer LLRs which are used for LDPC decoding and next iteration.

When the data is received into the decoder, the state is "0001" and the control signal of register unit is activated. After RDR buffers all of received data, the state is changed "0010", the control signals of H matrix index and output of register are activated. When the data corresponding to H matrix index out from register, the enable signal of CNU unit is activated. Furthermore, the control signal of memory unit for storage of CNU output is activated.
When the state is "0100", the control signals for output of memory, LLR unit and BNU unit are activated. And then the state is changed to "0010".
Finally, after the end of iterations, the state is not changed to "0010" from "0100". The state is changed to "1000" from "0100". And then read signal of OBU is activated.

We know that the required number of iterations is almost halved compared with a conventional algorithm maintaining the same computational complexity, and is without performance degradation. Secondly, an early detection method for reducing the computational complexity is proposed.
