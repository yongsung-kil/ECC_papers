Multiple Parallel Concatenation of Low-Density
Parity-Check Codes Based on BIBD
Zhiliang Qin, Kui Cai, and Songhua Zhang Data Storage Institute, Singapore, 117608 E-mail: {qin_zhiliang; cai_kui; zhang_songhua}@dsi.a-star.edu.sg

Abstract—In this paper, we propose a low-complexity multiple-parallel-concatenated (MPC) coding scheme for magnetic recording systems, where a high-rate quasi-cyclic (QC) low-density parity-check (LDPC) code with column weight 2 based on the Balanced Incomplete Block Design (BIBD) construction is used as the component code. Simulation results show that the proposed scheme improves the error floor performance considerably as compared with previous MPC schemes with single-parity-check (MPCSPC) codes and performs comparably to structured LDPC codes over idealized partial-response channels.
Index Terms—Multiple-Parallel-Concatenated (MPC) Code, Balanced Incomplete Block Design (BIBD), Netto Construction
I. INTRODUCTION
Recently, iterative decoding for magnetic recording systems [1] has received considerable attention. After being filtered and equalized to some partial-response targets, the magnetic recording channel appears as a rate-1 inner code to an outer turbo code [2] or a low-density parity-check (LDPC) code [3]. In practical applications, however, the complexity issue of outer coding schemes has to be carefully considered.
For the case of turbo codes, the encoding is much less complex than that of LDPC codes, whereas the decoding of LDPC codes can be performed efficiently based on the sum-product algorithm (SPA) [3]. In [4]-[6], a low-complexity coding scheme that combines advantages of both was proposed, which concatenates in parallel multiple branches of several single-parity-check (SPC) codes. Despite that the Tanner graph of the MPCSPC code has length-4 cycles, simulation results [6] show that it performs very well over magnetic recording channels. However, an error floor is usually visible over additive white Gaussian noise (AWGN) channels [5], which can be attributed to the existence of degree-1 bit nodes in the Tanner graph. The application of the SPA to this particular structure ensures that messages passed from degree-1 parity bit nodes to their neighboring checks are not affected by the decoding process and remain the same as channel input. This results in a high error floor, which does not improve with block length.
In this paper, we present a novel MPC coding scheme where a quasi-cyclic (QC) LDPC code with column weight dv=2 is used to replace the SPC code as the component code, as shown

in Fig. 1. QC-LDPC codes with dv=2 themselves are generally not considered to be good codes in terms of bit-error-rate (BER) performance. However, they are of much practical interest in high-speed applications as their paritycheck matrices have the lowest density of nonzero elements, resulting in the lowest possible decoding complexity. Moreover, their encoders have very low complexity and can be implemented with linear feedback shift registers [7]. The adoption of component codes with dv=2 leads the Tanner graph of the resulting MPC code to avoiding degree-1 bit nodes, which significantly enhances the effectiveness of the SPA at high signal-to-noise ratio (SNR) regions with all bit nodes benefiting from refined messages generated in the iterative decoding process. By modifying various design parameters, the proposed approach can generate high-rate codes with flexible rates and block lengths.
This paper is organized as follows. In Section II, we describe the construction of the proposed MPC codes. In Section III, the encoder and decoder structure of MPC codes are described, respectively. In Section IV, simulation results over AWGN channels and partial-response channels are presented, respectively. Finally, the conclusion is drawn in Section V.
II. MPC CODE CONSTRUCTION
In the following, we first design LDPC codes with column weight dv=3 based on the Netto’s construction of Balanced Incomplete Block Designs (BIBD) [8]-[10]. Afterwards, we apply array decomposition [11] to decompose circulants of the parity-check matrix to derive LDPC codes with dv=2, which will be used to form MPC codes.
1) BIBD Design
A BIBD with parameter (v, c, Ȝ) is defined as an ordered pair (V, B), where V is a set of v objects and B is a collection of n subsets of V with each subset (called blocks) consisting of c objects, such that each object of V is contained in exactly ȡ blocks and every two objects of V is contained in exactly Ȝ blocks.
For a given (v, c, Ȝ)-BIBD, we form a point-block incidence matrix H of size v×n over binary fields GF(2) with rows corresponding to v objects of V and columns corresponding to n blocks of B, in which hi,j=1 if the ith element of V is contained in the jth block of B, and hi,j=0 otherwise. It follows from the properties of BIBD that the incidence matrix has constant row weight ȡ and constant column weight c. For the special case of Ȝ=1, every pair of objects in V appear exactly in one block, which implies that H satisfies the row-column (RC) constraint,

2) MPC Decoding
MPC codes can be viewed as a special type of irregular LDPC codes whose parity-check matrix has size PbMc×(bKc+PbMc). The corresponding Tanner graph has PbMc check nodes whose degree is the same as that of the component LDPC code. The code graph also contains PbMc parity bit nodes of degree 2

(thus avoiding degree-1 nodes as in MPCSPC codes) and bKc data bit nodes of degree 2P, respectively.
To decode MPC codes, we propose a SPA decoding schedule that involves both parallel iterations and serial iterations. For each of b LDPC codes contained in each branch, a standard parallel decoding schedule [3] is carried out where all Mc check nodes are updated simultaneously followed by processing all Nc bit nodes. Once the decoding of a branch is completed, the generated extrinsic information corresponding to bKc data bits is passed to the other branches and used in the decoding as the a priori information. This is equivalent to a serialized branch-by-branch schedule as in decoding MPCSPC codes [4] and multiple turbo codes [12]. All extrinsic information is properly interleaved (or deinterleaved) and multiplexed (or demultiplexed) when exchanged between various branches.

IV. SIMULATION RESULTS
To be applicable to magnetic recording systems, we are mainly interested in high-rate MPC (rate>0.8) codes of block length 4k~5K. Unless specified, the maximum number of outer decoding iterations is set to 6, while the number of inner SPA decoding iterations is 5. The interleaver for each encoding branch of MPC codes is randomly generated.
We first consider MPC codes transmitted over AWGN channels. For clarity, only performance curves at the 6th outer iteration are shown. Fig. 2 shows that the component LDPC code with dv=2 performs worse than the Netto code with dv=3 by 1 dB at the BER of 10-6. However, the MPC code with P=2 based on this weak component code outperforms the Netto code by 1.4 dB. The MPC code with P=3 provides an additional gain of 0.4 dB. For comparison, we also include the performance of a rate-0.900 (4800,4320) MPCSPC code and a rate-0.857 (5040,4320) MPCSPC code, which are composed of P=2 and P=3 branches of 240-block (19,18) SPC codes, respectively, and have parameters close to those of MPC codes. Fig. 2 shows that MPCSPC codes do not perform well over AWGN channels. An error floor starts appearing for the MPCSPC code with P=3 slightly below the BER of 10-4. On the other hand, no error floor is observed for MPC codes with BER simulated down to 10-7.

In the simulation, we perform 5 overall iterations between the BCJR detector and the MPC decoder, each of which involves 6 outer iterations between MPC branches and 5 inner SPA decoding iterations within component LDPC decoders per branch. For clarity, only performance curves at the 5th overall iteration are presented. For comparison, we include the performance of two structured QC-LDPC codes with column weight dv=4 over ME2PR4 channels, which have parameters similar to those of MPC codes and are designed based on extended Reed-Solomon (eRS) codewords [14]. Decoding is performed by using the SPA with 30 iterations per overall iteration. These LDPC codes are known to be among the best codes over partial-response channels [15].

Fig. 5 shows that MPC codes perform slightly better than LDPC codes through the simulated SNR range. Note that the block length of eRS-based LDPC codes is required to be a multiple of the field order over which eRS codes are defined, and hence do not exist for certain system specifications. On the other hand, MPC codes feature several design parameters that can be easily adjusted. By modifying these parameters, such as the number of parallel branches P, the number of component codes per branch b, and the component code parameters (Nc,Kc), MPC codes can be constructed to achieve flexible rates and block lengths.
