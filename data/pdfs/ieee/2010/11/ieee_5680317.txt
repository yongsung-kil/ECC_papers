A Design of Rate-Adaptive LDPC Codes for Distributed Source Coding Using PEG Algorithm
Min Jang, Jin Whan Kang, and Sang-Hyo Kim†
School of Information and Communication Engineering, Sungkyunkwan University Suwon 440-746, Korea
Email: iamshkim@skku.edu

Abstract—We propose a method for constructing rateadaptive low-density parity check (LDPC) codes with large girth by syndrome merging with progressive-edge-growth (PEG) algorithm. Proposed scheme has a different structure from LDPC accumulate (LDPCA) codes. As a result, the new construction method turns the girth up for each specific code, and eliminates error floor.
Keywords-low-density parity check (LDPC) codes; rateadaptive LDPC codes; girth; progressive edge growth (PEG) algorithm; distributed source coding (DSC)
I. INTRODUCTION
In 1973, Slepian and Wolf revealed that two correlated random signals can be source-coded separately at the rate of joint entropy and perfectly reconstructed by joint decoding [1]. The result was later extended to lossy compression by Wyner and Ziv [2]. The source coding theory is called as a more general term ‘Distributed Source Coding (DSC).’
After a number of decades, the practical realizations of distributed source coding began being attempted. Sensor networks and distributed video coding (DVC) are probably the most feasible applications, which are devised for responding to increasing demand on low complexity encoding at low-end devices such as dummy sensors or cheap handhelds. Especially, DVC can explicitly be implemented as a form of practical Wyner-Ziv coding wherein a practical Slepian-Wolf codec is embedded [3].
It has been proven that source coding can be established using channel codes [4]. Practical Wyner-Ziv (or Slepian-Wolf) coding were also realized by employing powerful error correcting codes such as turbo codes [5] or low density parity check (LDPC) codes [6]. For scalable Wyner-Ziv coding, rate-adaptation of employed channel codes is essential. While proper puncturing provides the desired rate directly for turbo codes, the design of rate adaptive LDPC codes is more challenging and has been studied for the usage in HARQ systems of mobile communications [7]. Varodayan, Aaron, and Girod proposed a construction of rate-compatible LDPC accumulated (LDPCA) codes fitting in the framework of Slepian-Wolf coding, that is better approaching the

theoretical bound than turbo codes [8]. The LDPCA codes were adopted in DISCOVER, European project for advanced DVC [9].
The performance of rate adaptive LDPC codes is determined by the set of member codes for serving rates. However, it is not easy to optimize the code set supporting rate-adaptivity. Feng Cen optimized degree distribution of LDPCA codes for specific range of correlation between source and side information [10]. Recently, it is showed that non-binary LDPCA codes significantly outperform the binary LDPCA codes at the price of considerable complexity increase [11]. In that sense, design of good binary rate adaptive LDPC codes for practical Wyner-Ziv coding falls still into our interest.
LDPCA code set is designed in syndrome splitting [12] keeping the size of the stopping sets. However, the LDPCA code design did not take the cycle property into account so that the each member code in the code set has rather small girth which induces error floor. In this paper, we propose a new rate-adaptive LDPC codes designed by syndrome merging with a greedy girth maximization manner, so called progressive-edge-growth (PEG) algorithm [13, 14].
This paper is organized as follows: Section 2 introduces design criteria for fixed-rate LDPC codes and basic concepts of rate-adaptive LDPCA codes. In Section 3, we propose a method to keep the large girth of rateadaptive LDPC codes by syndrome merging with PEG algorithm. In Section 4, we compare the performance of the proposed scheme with that of LDPCA codes of [8] for distributed source coding of two binary correlated sources. Finally, Section 5 concludes this paper.
II. PRELIMINARY
A. LDPC Codes and PEG Algorithm

Bipartite graph is a graph whose vertices can be divided into two disjoint sets, and commonly used for representing constraints holding for codeword bits. Cycle of length 2d in the graph is defined as a closed path constituted by connections among d variable nodes and d check nodes without repeated nodes other than the starting and ending nodes, and girth is the shortest one of the graph. Short cycles contribute to error floor of a given code, and it slows down the convergence speed of decoding. Convergence speed means how fast given codes recover the errors, and affects decoding complexity.
Progressive-edge-growth (PEG) algorithm is a classical method to avoid a short cycle in designing LDPC code [13, 14]. Given the number of variable nodes, the number of check nodes, and the variable node degree sequence, PEG algorithm connects edges from variable nodes to check nodes such that new edge has as small an impact on the girth as possible. For implementing this, spreading tree from a selected variable node may be drawn. Depth of the tree denotes the distance from root node in this tree. To maximize the length of cycle, distance between check nodes connected with the same variable node should be as long as possible. Therefore, a new edge links the root with the leaf of the bottommost depth. The phase lasts until every edge is connected.

B. Rate-adaptive LDPCA Codes
Varodayan, Aaron, and Girod proposed LDPC accumulate (LDPCA) codes supporting rate-adaptivity [8]. This code realizes rate-adaptation function by simple syndrome accumulation.
In the first instance, N-length binary source vector X is multiplied by N-by-N parity check matrix H over GF(2), and syndrome vector S is generated as a result. Each produced element of syndrome vector is sequentially cumulated in order to generate the accumulated syndrome vector A whose element ai is modulo-2 sum from s1 to si. After the accumulation procedure, A is stored in a buffer, and some of the elements are transmitted to the decoder.
At the decoder, received accumulated syndromes are cumulated again, but it is somewhat different from that of the encoding. For example, consider a2, a4, a6, and a8 out of total eight accumulated syndromes are transmitted from the encoder. A. Design of Rate-adaptive LDPC Codes
On the contrary to the LDPCA code design, we begin to design a new rate-adaptive LDPC codes from the highest compression rate 1. First of all, N-by-N mother parity check matrix HN×N is created using PEG algorithm. According to (4), it has a sufficient number of check nodes for the graph to guarantee the girth greater than 4. Consider merging k pairs of check nodes from this graph to generate a code with one compression step lower, and the code set is designed recursively. In order to maximize the girth for designed codes, we can use the PEG algorithm in choosing k pairs of check codes at each step of design. Design phase proceeds as follows:
1. Initialization: Construct the N-by-N mother parity check matrix HN×N for the highest compression rate one.
2. Under the current graph setting corresponding Hi of rate Ri=Mi/N, place k pseudo variable nodes from v1(i) to vk(i) as represented in the Fig. 1-(a). For the j-th pseudo variable node vj(i), select two check nodes cj1(i) and cj2(i) by PEG manner.
(a) Select the first check node cj1(i) having the smallest degree under the current graph condition.
(b) Draw a spreading tree from the pseudo variable node vj(i). Then, choose the candidate from the bottom-most depth for second check node cj2(i).

(c) Whenever we face a multiple choice from the bottom-most depth, we select the candidate having the lowest degree.
(d) If there are still more than two candidates with the same lowest degree, choose the one which have maximum approximate cycle extrinsic message degree (ACE).
(e) When all candidates have common node, the neighbor variable nodes of both cj1(i) and candidates, select the one having the fewest number of common nodes to minimize the number of edge vanishing.
3. Merge the two check nodes cj1(i) and cj2(i) on the current graph setting as represented in Fig. 1-(b).
4. Repeat the Step 2 and 3 for all k pseudo variable nodes in order to build the parity check matrix H(i-1) of rate R(i-1)=M(i-1)/N.
5. Repeat from the Step 2 to Step 4 until the parity check matrix H1 of the lowest rate R1 = M1/N is constructed. After designing, we get the set of parity check matrices {H1, H2, … , HN×N}
In Step 3-(c) and 3-(d), we consider the the degree of check nodes and connectivity whenever we pick out the second check node from several candidates. It makes sense to choose the check node with the smallest degree, because regularity of check nodes’ degree helps the performance be better. Moreover, local cycle including only degree-2 variable nodes has a serious impact on the performance of irregular LDPC codes, so we consider the connectivity such as ACE [14].
In Step 3-(e), we also should keep in mind that the edges of bipartite graph vanish when two check nodes which have common nodes are merged. The lower the compression rate of a code becomes, the more difficult we avoid this event. Since we should make an effort to reduce the number of edge vanishing, the candidate which

Propose scheme and LDPCA codes are also compared under incremental redundancy strategy. Rate-adaptive LDPC codes receive additional syndromes until the syndrome check is satisfied, but the decoder may sometimes find a wrong codewords. In addition, the proposed rateadaptive code lowers the average number of iterations by approximately 10% than LDPCA codes over a wide range of conditional entropy.
V. CONCLUSION
A syndrome merging construction method of rateadaptive LDPC codes has been proposed, and it is certain that this scheme maximizes the girth of the code set of rate-adaptive LDPC codes. Unlike LDPCA codes, new rate-adaptive codes are designed in a top-down manner, and have different encoding and decoding structure. Proposed design tries not to make short cycles for each member code, thereby improving the performance of each code in terms of error floor. As a result, undetectable error rate and speed of convergence are improved without any compression rate loss compared to LDPCA codes for incremental redundancy scheme. Also, compression performance is better than LDPCA codes for nonincremental redundancy strategy.
This paper deals with the codes having only regular variable nodes’ degree distribution. For irregular codes, the N-by-N mother parity check matrix designed by normal or improved PEG algorithm performs badly. Thus, we will research into a way of designing irregular rateadaptive LDPC codes in a syndrome merging manner for further study.
