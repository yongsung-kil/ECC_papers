Cycle Analysis of Time-invariant LDPC Convolutional Codes

Hua Zhou and Norbert Goertz
Institute of Communications and Radio-Frequency Engineering Vienna University of Technology
Gusshausstrasse 25/389, 1040 Wien, Austria Email: {Hua.Zhou, Norbert.Goertz}@nt.tuwien.ac.at

Abstract—Time-invariant low-density parity-check convolutional codes (LDPCccs) can be constructed from a polynomial form of a parity-check matrix that defines quasi-cyclic LDPC block codes based on circulant matrices. Based on this polynomial matrix, we discuss the relationships between the polynomial domain and the time domain parity-check and syndrome former matrices with respect to cycle properties. We present a novel, simple way to describe cycles in the polynomial version of the syndrome former matrix and we exploit this concept in a new cycle counting algorithm.
I. INTRODUCTION
As a counterpart to classical LDPC block codes[1], LDPC convolutional codes (LDPCccs) were first introduced in [2]. Compared with LDPC block codes, they are much more suitable for practical implementation with continuous transmission [3], and they achieve excellent performance. Moreover, the same encoder can be used to obtain coded sequences of variable lengths by choosing different “termination points” of the code [4], similar to conventional convolutional codes; such flexible block size is a very important feature for practical use, e.g., in multimedia transmission.
LDPCccs are defined by a semi-infinite1 syndrome former matrix HT which is the transpose of the time domain paritycheck matrix of the code. In terms of the structure of the syndrome former matrix, several construction methods have been introduced in the literature. Traditionally, as shown in [2] and [10], HT is generated by a four-step procedure from the parity-check matrix of a LDPC block code. Moreover, in [5], [6], [7], structures derived from protographs have been shown to achieve improved performance. Permutation matrices are also used to construct LDPCccs; details can be found in [8].
A popular algebraic method to construct LDPCccs is based on quasi-cyclic (QC) LDPC block codes that are generated by circulant matrices [9]. This type of LDPCccs can be represented in the form of a polynomial parity-check matrix: the polynomial entries of the matrix correspond to the location of a “one” in the first column of each circulant matrix in the parity-check matrix of the associated QC LDPC block code. Simulations in [9] show that LDPCccs generated this way outperform the corresponding QC LDPC block codes as well as randomly generated ones for small-to-medium block size.
1The syndrome former matrix becomes “finite” by termination of the code.

The “performance” of LDPC codes depends on a variety of aspects: encoding and decoding complexity, minimum or free distance, girth, cycle structure, decoding threshold etc. In this paper, we will focus on analyzing the cycle properties of LDPC convolutional codes derived from QC LDPC block codes. The rest of the paper is organized as follows: In Section II we review the structure of QC LDPC block codes together with the generation of a polynomial matrix for the associated LDPC convolutional code. In Section III we discuss how cycles are formed in a polynomial parity-check matrix of LDPCccs. In Section IV, we present an algorithm to track the numbers of cycles with different lengths; this algorithm can also be used to examine the girth of the code.

III. CYCLE PROPERTIES FOR LDPCCCS DESCRIBED BY A POLYNOMIAL PARITY-CHECK MATRIX
It is well known that “short” cycles in the factor graph of LDPC block codes affect the decoding performance. The larger the girth – the shortest cycle in a factor graph – the better the decoding performance will be. A “good” code, however, will have to have cycles in the code graph to achieve good properties such as large minimum distance. Hence, we are interested to analyze cycle properties of LDPC codes. In this section, we will explain how cycles are formed in LDPCccs.
Without loss of generality, we will below consider the transpose of the H matrix, which is called (time-domain) syndrome-former matrix HT . An example is given in Fig. 3: this matrix is the transpose of the matrix in Fig. 2, with the “empty checks” y(0),y(1) being removed as described above.

A horizontal edge in HT corresponds to a connection between two polynomials in the same row of HT (D): such a connection links two checks that use the same code bit, i.e., there is no time-difference (“delay”) between them.
In contrast, a vertical edge in HT will incur a delay as code bits at different time steps are linked by a parity check. As long as we have only weight-one polynomials (or monomials6) in the polynomial matrix of the syndrome former, each check will only connect to one of the bits a,b,c (see Fig. 1) at a particular time instant t. Guess-and-test and hill-climbing algorithms in [13], [12], respectively, design large-girth LDPC codes from circulant permutation matrices.

The algorithms mentioned above are used for LDPC block codes. In this section, we present a new algorithm that examines the cycle properties of a LDPC convolutional code using the polynomial form (3) of the parity-check matrix.
Considering the delay properties in (7) and (8), obviously, we only need to process the differences between any two monomials in the same column of H(D)T , which also explains the equivalence by removing common factors from columns of H(D)T . From this point of view, we state a greedy algorithm to compute the number of cycles with different lengths smaller than a certain given cycle length. Due to the greediness and tracking time for relatively large j and k, this algorithm is suitable to examine cycles of length up to 12. Those cycles are, however, the most important ones with respect to decoding performance, as the latter is dominated by “short” cycles and, due to code construction, 12-cycles can not be avoided anyway [13].
For each element in the polynomial matrix H(D)T , it is assumed that first edge starting from a polynomial entry in a path goes horizontally in H(D)T . Fig. 6 illustrates our algorithm. V. CONCLUSION
In this paper, we have illustrated a new way to analyze the cycle properties of LDPC convolutional codes described by polynomial matrices that are derived from QC LDPC block codes. In terms of the achievable length of cycles to be examined, it depends on the values of j and k considering the search time. The cycle tracking algorithm does not require excessive memory to store data due to the continuously applied “freememory”-manipulations, which are shown in the algorithm description in Section IV. In addition, this algorithm can also be used to calculate cycle properties in a polynomial (nonmonomial) matrix with regular or irregular degree distributions as well as to determine the girth of LDPC convolutional codes.
