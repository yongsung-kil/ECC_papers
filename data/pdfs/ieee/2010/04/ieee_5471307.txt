Abstract
A new LDPC decoding design aimed for a reduced finite word-length hardware implementation is proposed for WiMAX LDPC codes. In this design, normalized BP-based decoding method and layered decoding scheme are employed. The correction factor of the decoding method is chosen to simplify the multiplication. By analyzing the statistical distribution of variable node message, the word-length of bit node messages is reduced by 1 bit. Simulation result shows that though with small word-length, the proposed design has a similar decoding performance compared to a conventional 8 bits word-length decoder.
1. Introduction
Low-density parity check (LDPC) code was proposed by Gallager in 1960 [1]. It was rediscovered by MacKay and Neal in 1996 [2]. Some years later, a LDPC code that can achieve within 0.04 dB of the Shannon limit was designed [3]. Because of the excellent performance, LDPC code has widely adopted by many communication systems, such as digital video broadcasting second generation (DVB-S2), wireless local area network (802.11n) and world interoperability for microwave access (802.16e) [4].
Belief-propagation (BP), or sum-product [2], is the most powerful computation algorithm in LDPC decoding, but it is too complex to be implemented. In recent years, two main approaches called normalized BP-Based algorithm [5] and offset BP-Based algorithm [6] are widely used in hardware implementation. Both of them are based on the so-called min-sum approximation [7]. There is a multiplicative correction factor in the first method and an additive correction factor in the second.

Layered decoding scheme was proposed in [8]. In this decoding scheme both check node and variable node update concurrently. It can reduce the decoding latency by about 50% with no performance loss.
LDPC code in WiMAX is a kind of Quasi-Cyclic (QC) LDPC codes [9]. A QC LDPC code is constructed by a base matrix and many cyclic shifted identity matrixes. The parity check matrix can be expanded in this way: every “1” in base matrix is replaced by a shifted identity matrix and every “0” is replaced by an all zero matrix. This structure is very suitable for hardware implementation. By the way, QC LDPC codes can achieve similar performance to computer generated random codes.
For hardware implementation of LDPC decoder, the finite word-length of the bit node message deeply affects the total area and complexity. Furthermore, it affects the size of memory and the complexity of the routing networks and the node updating operation unit.
In this paper, we propose a decoding design for LDPC codes in WiMAX system. The design uses normalized BP-Based decoding algorithm, layered decoding scheme, 5bits quantization scheme for input message, and 7bits quantization scheme for extrinsic message. Simulation results show that no error floors are detected at FER of 10-5 and BER of 10-6.
The rest of this paper is organized as follows. Section 2 introduces the most famous BP-based decoding method including min-sum, normalized BPbased and offset BP-based algorithm. Section 3 presents the layered decoding scheme, and analyzes how to choose the bit node messages’ finite wordlength in layered decoding scheme. The proposed decoding design and simulation results are given in Section 4. Finally, section 5 concludes this paper.

4. Decoding design and simulation results
Use normalized BP-based decoding algorithm and layered decoding scheme, we have designed a decoding module for WiMAX LDPC codes, and also, AWGN channel and BPSK modulation are assumed. This design supports all rates and all codeword lengths defined in WiMAX. The finite word-length and the

quantization scheme of LLRs and bit node messages are the same as in section 3. We have chosen 0.75 and 0.875 as the correction factors, results are illustrated in figure 2 and figure 3.
As shown in figure 2 and figure 3, if the correction factor is 0.75, the code has a serious performance loss and an error floor at FER of 10-3 (R=1/2 code); if the correction factor is 0.875, the decoding performance is pretty good. In the second case, all the 6 rates don’t have error floors at BER of 10-6 and FER of 10-5.
We also have set the bit node messages’ wordlength as 8 bits. Slight performance differences are detected at low SNR, but no difference detect at high SNR. When we set the word-length as 6 bits, the decoding performance is bad, for example, the R=1/2 code (codeword length is 2304) has an error floor at BER of 10-6.
Compared to [12], with one bit fewer in bit node message (at least 10% of the implementation area will be saved), our decoder has a better performance for R=1/2 code, similar performance for R=2/3A code and R=5/6 code, worse performance for the rest three rates (with less than 0.2dB performance loss).
5. Conclusion
In this paper, a LDPC decoding design aimed for a reduced finite word-length hardware implementation for WiMAX LDPC codes is proposed. The normalized BP-based decoding method and layered decoding scheme are employed in this design. In the decoding method, we choose the correction factor which will simplify the multiplication. The word-length of bit node messages is reduced by 1 bit. Simulation result shows that the proposed design has a good performance.
