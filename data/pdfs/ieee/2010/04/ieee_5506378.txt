Efficient Implementation of Interior Point Decoding Based on Barrier Function for LDPC Codes
T. M. N. Ngatched, Attahiru S. Alfa, and Jun Cai Department of Electrical and Computer Engineering
University of Manitoba Winnipeg, MB R3T 5V6
Canada Email: {ngatched, alfa, jcai}@ee.umanitoba.ca

Abstract—In this paper, an efficient implementation of the interior point algorithm recently proposed by Wadayama [5] for linear programming (LP) decoding of low-density paritycheck (LDPC) codes is presented. The main idea behind the modification is to approximate the barrier function for the fundamental polytope defining the code so that it contains only one linear constraint for each of the parity-check constraints. Simulation results show that the approximations introduced do not result in any performance degradation, while considerably reducing the decoding complexity and latency.
I. INTRODUCTION
Low-density parity-check (LDPC) codes originally introduced by Gallager [1] are becoming the standard for error control in many communication and digital storage systems where high reliability is required. By combining randomness and sparsity, LDPC codes with sufficiently long block length can approach Shannon limit using belief propagation (BP) decoding (also known as sum-product algorithm (SPA)). While the observed error-correcting performance of BP is unparalleled, its theoretical understanding remains incomplete.
Proposed by Feldman et al. [2], linear programming (LP) is an alternative to iterative algorithms for decoding LDPC and turbo-like codes. LP decoding is based on solving a linearprogramming relaxation of the integer program corresponding to the maximum likelihood (ML) decoding problem. Although the practical performance of LP decoding is comparable to message-passing decoding [2], [3], [4] for LDPC codes with short to medium block lengths, LP decoding has some attractive features. LP decoding has the ML certificate property; whenever the result of linear programming decoder is integer, it is provably the maximum likelihood codeword. There are some close connections between LP decoding and standard forms of iterative decoding for LDPC codes. Moreover, LP decoding is amenable to nonasymptotic (finite-length codes) analysis.
An algorithm for LP decoding of LDPC codes was recently introduced by Wadayama [5]. The algorithm, called interior point decoding (IPD), uses a barrier function to transform the constrained problem resulting from the LP relaxation

of ML decoding into an unconstrained problem which can be solved using unconstrained optimization techniques. The barrier function corresponds to the constraints on the fundamental polytope defined based on an LDPC matrix, and is evaluated many times during the decoding process. Unfortunately, the complexity of computing the barrier function increases exponentially with the maximum check-node degree, therefore making this approach impractical even for paritycheck matrices of moderate row weights.
In this paper we propose an efficient implementation of Wadayama’s algorithm. Inspired by a theorem proved in [6] that at most one of the constraints introduced by each check node can be violated, we develop two corollaries which enable us to approximate the barrier function for the fundamental polytope defining the code so that it contains only one linear constraint for each of the parity-check constraints. This leads to an approximate variation of the Newton method used to solve the convex optimization problem. A huge reduction in computational costs is hence obtained, making it possible to apply interior point algorithms with barrier function for the decoding of any LDPC codes, irrespective of its row weight. The performance of the proposed algorithm is demonstrated through simulation, and it is observed that the approximation does not result in any performance degradation, while considerably reducing the decoding complexity and latency.
The rest of this paper is organized as follows. In the next section, background on LDPC codes and linear programming relaxation of maximum-likelihood decoding are provided. A review of the interior point decoding based on the barrier function method, with an emphasis on its implementation, is presented in Section III. In Section IV, an efficient implementation of the barrier function method is introduced. Some simulations results are presented in Section V, and finally concluding remarks are made in Section VI.

In the absence of additional structure, the ML decoding problem (1) requires an exhaustive search over an exponentially large space and is known to be NP-hard [7]. It is straightforward to convert the ML decoding problem into a linear program over the convex hull of all codewords, a polytope known as the codeword polytope [2]. Although linear programming is polynomially solvable in general, for a general binary linear code, the number of constraints required to characterize the codeword polytope, unfortunately, grows in a nonpolynomial manner with blocklength. A natural approach is to relax the linear program by taking only a polynomial set of constraints that approximate the codeword polytope. In [2], Feldman et al. proposed a relaxation using the parity-check matrix H. For each check, the corresponding local codeword polytope is the convex hull of the bit sequences which satisfy the check (local codewords). The relaxed polytope P, also known as the fundamental polytope [4], is obtained by looking at each check independently, and taking the intersection of all the local codeword polytopes.
For LDPC codes, the relaxed polytope P can be described by a linear number of inequalities as follows. III. SOLVING THE LP USING THE INTERIOR POINT METHOD
Although the LP decoding problem (6) is easier to solve than the original MLD problem (1), it is still a computationally difficult problem. Moreover, using general purpose LP solvers can be highly inefficient as they do not take advantage of the particular structure of the optimization problems arising in LP decoding. In this section, we briefly review the interior point method (IPM) with barrier function for LP decoding.

The problem is overcome by a process called centering, where a succession of these modified problems are solved with t increasing (and therefore the accuracy of the approximation) with each centering step (ti+1 = μti). First, the modified problem is solved using some small initial t0 by Newton’s method. The solution of a centering step is used as the starting point of the next centering step. Each execution of the centering step is called an outer iteration, and the Newton iterations or steps executed during the centering step are called inner iterations. In our implementation, for practical reasons, a limit is placed on the number of outer and inner iterations. It should be noted that, although we assume Newton’s method, any method for linearly constrained minimization can be used in the centering step.

B. Choice of Barrier Step Size μ and Starting Value t0
The choice of the parameter μ involves a trade-off in the number of inner and outer iterations required. If μ is too large, then we will have less centering steps (outer iterations), but each step will require more inner iterations before it converges. If μ is too small, then we will require many centering steps. Typical implementations take μ between 10 and 20.
Another important issue is the choice of t0, the starting value of t. Here the trade-off is simple: If t0 is chosen too large, the first outer iteration will require too many iterations. If t0 is chosen too small, the algorithm will require extra outer iterations, and possibly too many inner iterations in the first centering step. We found that values from around 20 or so seem to work well.

E. Initial Search Point
The barrier method requires a strictly feasible starting point x0. It can easily be shown that, if wr ≥ 3, the point x = (1/2, 1/2, . . . , 1/2) satisfies both the box constraints and the parity constraints. It is thus a feasible point, i.e. x ∈ P∗, and can been used as the initial search point.
IV. EFFICIENT IMPLEMENTATION OF THE BARRIER FUNCTION METHOD
The most complex step at each iteration of the IPD is in the Newton iterations. During these steps, a search point x is updated using the Newton method to minimize the objective function ψ (x). During this process, the objective function, and hence the barrier function, is evaluated many times. Any odd-sized subset V of the neighborhood N of each parity check node introduces a unique parity inequality. It can easily be shown, using the binomial theorem, that there are 2wr−1 constraints corresponding to each check node of degree wr. The number of inequalities corresponding to the parity constraints is thus M 2wr−1. Therefore, a brute-force evaluation of the barrier function requires O (N 2wr ) computation time. For parity-check matrices of high row weights, this number can be very large, and even prohibitive.
In addition, the execution of the Newton Method requires the computation of the gradient and the Hessian of the objective function. Computation of the gradient involves the partial derivative of the barrier function and contains M 2wr−1 terms. Likewise, a straightforward evaluation of the second derivative of the barrier function required for the Hessian needs O (N 2wr ) computational time. The computational time of computing the gradient and the Hessian are therefore an exponential function of the row weight wr as well. These

computational costs are also a major obstacle to achieving an efficient implementation of the interior point decoding for LDPC codes of moderate or high row weight. In this section, some approximations are introduced in order to make the interior point decoding computationally tractable and applicable to any LDPC code, irrespective of its row weight.

V. SIMULATION RESULTS
The error performance, in terms of bit-error rate (BER) and frame-error rate (FER) as a function of the rate-normalized signal-to-noise ratio (SNR) per information bit, of the standard IPD and the proposed IPD is evaluated on a number of LDPC codes of various constructions with short to medium block lengths. The maximum number of outer and inner iterations is set to 5 and 20, respectively. However, due to space limitation, we only present a couple of results. For comparison, the performance of the codes using BP is also included. The maximum number of iterations is set equal to 200 for BP. At each SNR value, at least 200 codewords errors are detected for short block length codes, and at least 100 codewords errors for medium block length codes.
Figs. 1 - 4 show a couple of results. The code in Fig. 1 is a randomly generated (3, 6)-regular LDPC code of length 120, dimension 56, and rate 0.4667. The code in Fig. 2 is a randomly generated (3, 10)-regular LDPC code of length 273, dimension 191, and rate 0.7. The code in Fig. 3 is a (3, 10)regular LDPC constructed using the progressive edge growth (PEG) [9] algorithm. It is of length 504, dimension 252, and rate 0.5. The code in Fig. 4 is the (255, 175) Type-I 2-D Euclidean geometry (EG)-LDPC code [10] of rate 0.686. It can be observed that, in all cases, the approximations introduced in the proposed IPD do not cause any performance degradation. On the other hand, compared with the proposed IPD, the running time of the standard IPD was about 5 times, 10 times, and 7 times for the first code, the second code, and the third code, respectively. For the last code, it was impossible to obtain the performance using the standard IPD due to the large row weight of the code (wr = 16). It can also be observed that, at low SNR, BP slightly outperforms the IPD in some cases. However, at medium to high SNR, the situation is reversed with the IPD always outperforming BP in the error-floor (high SNR) region.

VI. CONCLUSION
In this paper we studied an efficient implementation of the interior point method with barrier function for the decoding of LDPC codes.
