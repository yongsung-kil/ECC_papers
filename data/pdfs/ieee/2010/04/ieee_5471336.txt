Alternate Iteration of Shuffled Belief Propagation Decoding

Abstract
Low-Density Parity-Check (LDPC) codes are usually decoded by running an iterative decoding algorithm, such as belief-propagation (BP), over the factor graph of the codes. The standard BP schedule is a parallelizable decoding, using the same pre-update information during updating all the check nodes and bit nodes. In order to improve the convergence speed in terms of number of iterations, several sequential schedules were studied, such as the Layered beliefpropagation (LBP) and the Shuffled belief-propagation (SBP). Correspondingly, for these serial decoding algorithms, the messages were generated by using the latest available information. In this paper, a modified shuffled version of iterative decoding of low-density parity-check codes is present by changing the updating order at bit nodes. Compared with the plain shuffled version, the proposed scheme has the same convergence rate which is twice as the conventional belief propagation, and can reduce complexity without any performance degradation. At last, the simulation result was shown with respect to the LDPC codes in CMMB standard.
1. Introduction
Low-density parity-check codes (LDPC) [1] have been attracted great attentions due to its near Shannon limit performance and parallelizable decoding based on belief propagation (BP) algorithm [2][3]. In each iteration, the BP iterative decoding algorithm simultaneously updates all the variable nodes (with each update using the same set of pre-update data in the last iteration), and then updates all the check nodes (again with each update using the same set of pre-update data in the last iteration), so it requires a large number of iterations to achieve high performance. In order to take advantage of more reliable extrinsic messages updated within the single iteration, some serial schedules were

introduced in LDPC decoding. A horizontal approach under the name of Layered BP (LBP) was addressed in [4] [5]. In LBP decoding, the bit node probabilities and check node probabilities were updated row by row [6], which is a similar approach for low-latency decoding of turbo product codes [7]. Accordingly, a “shuffled” LDPC decoding method which we called Shuffled BP (SBP) was presented as a vertical schedule in [8] [9] [10]. In SBP decoding, bit node probabilities and check node probabilities were updated column by column. These serial schemes can converge twice as fast as standard BP decoding without any increase of decoding complexity per iteration. In this paper, we propose a simplified version of shuffled BP decoding intended to reduce the complexity of SBP at no cost of performance.
This paper is organized as follows: section 2 briefly reviews the standard and shuffled BP decoding of LDPC codes and then carefully explain the simplified SBP decoding. Some simulation results and analyses of computation complexity are presented in section 3. Finally, we give concluding remarks in section 4.
2. Iterative decoding of LDPC codes
A binary (N, K) LDPC codes [1] is a linear block code described by a sparse M × N parity check matrix H, i.e., H contains mostly zeros and only a small number of ones. The parity check matrix H can be represented by a bipartite graph with N variable nodes corresponding to column of the matrix H and M check nodes corresponding to the row of the matrix H.
Following a notation similar to [3], we denote a set of bits that participate in check m by N(m)={n: Hmn=1} and the set of checks in which bit n participates as M(n)={m: Hmn=1} similarly we define N(m)\n as the set N(m) with bit n excluded, and M(n)\m as the set M(n) with check m excluded. 2.3. Modified Shuffled BP for Iterative Decoding of LDPC Codes

Shuffled BP decoding is a bit-based sequential approach and the method described in Section 2.2 is based on a natural increasing order, i.e., the messages at bit nodes are updated according to the order n=1,2,…,N. The larger the value of n, the more independent pieces of information are used to update the message at bit n and the more reliable these messages become. Therefore, as the index n increases, the reliability of the bit decisions increases and the corresponding error rate decreases. Accordingly, the same reasoning applies if shuffled BP decoding is performed in reverse order; hence, if shuffled BP decoding is employed using a bit order starting with bit N and ending with bit 1, the error rate increases with the index n.

3.2. Comparison of Complexity
Compared with shuffled BP decoding algorithm, there is no reduction happened in vertical process, we just compare the complexity of horizontal process. We take the regular (9216, 4608) (3, 6) LDPC codes as an example. Since the second iteration, there is one of six check-to-bit messages value Umn equal to the last iteration’s. Because in this procedure, the same values Vmn computed at the previous iteration were selected. Consequently, these check-to-bit messages will not be updated except the first iteration.
4. Conclusion
In this paper, shuffled BP decoding of LDPC codes is studied. In view of the each iteration, the messages of plain shuffled BP decoding at bit nodes are updated following a natural increasing or decreasing order through, while our scheme updated the messages at bit nodes according to an increasing or decreasing order alternately. Simulation result and theoretic analysis show that our schedule has lower computational complexity than the shuffled BP decoding algorithm without any performance degradation.
