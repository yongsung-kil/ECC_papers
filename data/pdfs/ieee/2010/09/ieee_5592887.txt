Improved Linear Programming Decoding and Bounds on the Minimum Distance of LDPC Codes
David Burshtein and Idan Goldenberg School of Electrical Engineering
Tel-Aviv University, Tel-Aviv 69978, Israel Email: {burstyn, idang}@eng.tau.ac.il

Abstract—We propose a technique for improving LP decoding, based on the merging of check nodes. This technique can be applied to standard as well as generalized LDPC codes. Furthermore, we show how a recently-discovered linear-complexity LP decoder can be used to derive non-trivial lower bounds on the minimum distance of specific LDPC codes, with complexity that exhibits quadratic growth with respect to the block length. This bound can be refined using the check node merging technique. The lower bound on the minimum distance is shown to be an upper bound on the fractional distance of the code.
Index Terms—Linear programming decoding, iterative decoding, maximum likelihood (ML) decoding, low-density paritycheck (LDPC) codes.

This paper is organized as follows. In Section II the linearcomplexity LP decoder [5] is reviewed. Section III describes how improved LP decoding can be achieved via merging check nodes. I. INTRODUCTION
The method of linear programming (LP) decoding of linear codes was proposed by Feldman et al. [1]. Over the past several years, this method stirred a substantial amount of research. One reason for this is that the LP decoder has the ML certificate property [1], i.e., that if the LP decoder succeeds in producing an integral solution vector, then it produces the ML solution.
However, LP decoding is also disadvantaged as compared to iterative decoding algorithms such as belief propagation or min-sum decoding because the decoding complexity can be much larger. In general the LP decoder entails a polynomial, non-linear complexity. Several authors have proposed algorithms for low-complexity LP decoding, e.g. [2], [3], [4]. Vontobel and Koetter [2] have proposed an iterative, GaussSeidel-type algorithm for approximate LP decoding. In [5] the approach of [2] was studied under a newly-proposed scheduling scheme. This study showed that the performance of the LP decoder can be approached with linear computational complexity. It was also shown that the iterative LP decoder can correct a fixed fraction of errors with linear computational complexity. These results were also extended to generalized LDPC (GLDPC) codes. In this paper, we rely on the results of [5] and show how merging check nodes in the graph can tighten the relaxation and improve the LP decoder. We also show how the linear-complexity LP decoder can be used to obtain a lower bound on the minimum distance of a specific code, which is also an upper bound on the fractional distance of the code, with complexity O(N 2), where N is the block length. In what follows, Propositions 1 and 3 will be proven in the full version of this paper.
Proposition 1: Q′(C) ⊆ Q(C). Note that when we merge all the constraint nodes into one constraint node, the resulting polytope, QML(C), is the convex combination of all possible codewords. That is, the set of vertices of this polytope is the set of codewords of C, and the minimization of P(c) over c ∈ QML(C) yields the ML decoded codeword, cˆML. Thus we have QML(C) ⊆ Q′(C) ⊆ Q(C). Note also that for any codeword c we have c ∈ Q(C), c ∈ Q′(C) and c ∈ QML(C). That is, Q′(C) is a finer relaxation of QML(C) than Q(C). Let the error probability Pe(c) of the decoder that uses the initial graph G, given that the transmitted codeword is c, be defined by,
Pe(c) = Pr (∃c˜ ∈ Q(C) : c˜ = c, P(c˜) ≤ P(c))
Similarly, the error probability Pe(c) of the decoder that uses the graph G′, given that the transmitted codeword is c, is defined by,
Pe′(c) = Pr (∃c˜ ∈ Q′(C) : c˜ = c, P(c˜) ≤ P(c))
By Proposition 1 we immediately obtain the following, Proposition 2: Pe′(c) ≤ Pe(c). This proposition is essentially due to the fact that the second LP decoder uses a tighter relaxation of QML(C).

In this manner, we create a set of L1 norms {lcr }cr∈Cr . It is easily seen that the minimum lmr in over this set is a lower bound on the minimum distance of the code, subject to the restriction that the local codeword on r is nonzero. This is due to the fact that
a minimum-distance codeword subject to this restriction must have some local nonzero codeword cr on check node r, and while running Algorithm 1 we optimize over a set containing
this minimum-distance codeword. Since a minimum-distance
codeword must contain a check node, the neighbors of which
represent a nonzero local codeword, we conclude that the minimum lmin minr∈J {lmr in} is a lower bound on the minimum distance of the code.
Recall that we assume parity check degrees which do not depend on N . Consequently, for any check node r, the bound lmr in is obtained by running Algorithm 1 a constant number of times. Repeating this process over all check nodes thus entails a total complexity of O(N 2). V. NUMERICAL RESULTS
Figure 2 shows the lower bound on the minimum distance. The plot shows, for various block lengths, the average bounds over 10 randomly-generated codes, taken from Gallager’s ensemble of (3, 4)-regular codes. The results appearing in the figure include those with and without merging of check nodes. The brackets in the figure indicate the range of results, over the 10 codes, for the case where check nodes are merged. As a reference, we also include results from [1] which depict exact fractional distance, averaged over a sample of 100 codes. We pick check nodes for the merging process as follows. First, a list of short cycles (in this case, up to maximum length of 6) in the graph is generated using the procedure in [6]. Based on this list, in each cycle we merge the check nodes into one new check node. This approach is motivated by Proposition 3, which guarantees an unchanged relaxation if the set of merged constraint nodes is cycle-free.
We note that since the minimum distance is an integer number, any fractional result can be rounded up to the next integer and still be a lower bound on the minimum distance. This fact is not reflected in Figure 2. Finally, since the merged check node results yield tighter relaxations, we can use the unmerged results as upper bounds on the fractional distance.
