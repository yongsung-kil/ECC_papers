graphs. These graphs belong to a general class of bipartite graphs which consists of two classes of nodes. The variable nodes represent code words, which are the columns in PCM, and the check nodes represent parity check equations, which are the rows in PCM.
The Tanner graph has connection between variable node i and check node j if the corresponding bit hij in the PCM is 1, as shown in the example of Fig. 1. The decoding algorithm for LDPC codes is based on the belief propagation algorithm (BPA) [1]–[4]. A suboptimal method proposed in [4] approximates the tanh function of BPA with a min function. A modified version, later proposed in [5] and called the offsetmin-sum algorithm, has degradation of only 0.1 dB. Previously known architectures for the min-sum unit available in the literature either support regular LDPC [6]–[8] with check node of a fixed degree or the ones [9]–[12] that support both irregular and regular have a serial architecture. Though one can stack or multiplex a number of such serial decoders to achieve higher throughput, they do not provide a generic and scalable architecture for LDPC with check node of any degree d. At the same time they do not support all the code rates and all the code lengths of a specified standard for which they are designed for.

II. LDPC DECODING

Decoding of LDPC codes of rate k=n (where k is the number of message bits encoded into n bits codeword with k < n) is performed by a class of sum-product algorithms called two-phase belief propagation. They are also referred to as message-passing algorithms because they exchange messages along the edges of the Tanner graph.
In the BPA algorithm the a posteriori probabilities [13] calculated at the check nodes have a nonlinear dependence. If they are to be represented as fixed-point numbers, then to maintain sufficient enough accuracy, the number of bits required would be large. Size of memory required for traditional lookup-table-based implementation is exponentially large. Such an approach is impractical for codes with large length. A suboptimal version [4], [5] approximates the nonlinear function with an offset-min function. The performance of this suboptimal method is very close to belief propagation algorithm (about 0.1 dB of degradation in comparison to the standard belief-propagation). For fair comparison with other reported architectures, throughput results for the proposed architecture were obtained assuming five iterations, 96 CNPUs, rate 1/2 code, and block length of 2304 bits. However, with an increase in the number of iterations the throughput would reduce, for example, with 15 iterations the throughput would be 364 Mbps. When operating at smaller code lengths, all the 96 CNPUs are not needed and hence can be shutdown to save power. Throughput can be further increased by adding additional pipeline stages but increases the overhead on area and power dissipation. For example, by adding one pipeline stage in the min unit the LDPC decoder can be run at a maximum clock frequency of 200 MHz. However, the area and power dissipation would be 1.3 million gates and 2 W respectively. The energy per bit per iteration of the proposed architecture is 87.10 pJ/bits/iteration. Although [7] has energy per bit per iteration of 7.4 pJ/bits/iteration, it only supports single code rate and a small code length regular LDPC code. Furthermore, the proposed architecture supports any check node degree, without the requirement of any reconfiguration and can accommodate changes to code rate and code length on-the-fly.

Abstract—In ultra-deep submicrometer (UDSM) technologies, the current paradigm of using copper (Cu) interconnects for on-chip global communication is rapidly becoming a serious performance bottleneck. In this paper, we perform a system level evaluation of Carbon Nanotube (CNT) interconnect alternatives that may replace conventional Cu interconnects. Our analysis explores the impact of using CNT global interconnects on the performance and energy consumption of several multi-core chip multiprocessor (CMP) applications. Results from our analysis indicate that with improvements in fabrication technology, CNT-based global interconnects can significantly outperform Cu-based global interconnects.

I. INTRODUCTION
As technology scales into the ultra-deep submicrometer (UDSM) region, poor interconnect scaling has made interconnect design a major bottleneck for emerging multi-core chip multiprocessors (CMPs) that have tens to hundreds of components integrated on a single chip [1]. According to the International Roadmap for Semiconductors (ITRS) [2], global interconnect performance in particular has become one of the semiconductor industry’s topmost challenges. With technology scaling, conventional copper (Cu)-based global interconnects have become increasingly susceptible to electromigration, rising crosstalk coupling noise, and parasitic resistivity, which has reduced reliability and performance [3]. There is therefore a critical need to investigate innovative global interconnect alternatives to Cu.
