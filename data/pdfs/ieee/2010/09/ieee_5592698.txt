Abstract—Polar codes are the first codes to provably achieve capacity on the symmetric binary-input discrete memoryless channel (B-DMC) with low encoding and decoding complexity. The parity check matrix of polar codes is high-density and we show that linear program (LP) decoding fails on the fundamental polytope of the parity check matrix. The recursive structure of the code permits a sparse factor graph representation. We define a new polytope based on the fundamental polytope of the sparse graph representation. This new polytope P is defined in a space of dimension O(N log N ) where N is the block length. We prove that the projection of P in the original space is tighter than the fundamental polytope based on the parity check matrix. The LP decoder over P obtains the ML-certificate property. In the case of the binary erasure channel (BEC), the new LP decoder is equivalent to the belief propagation (BP) decoder operating on the sparse factor graph representation, and hence achieves capacity. Simulation results of SC (successive cancelation) decoding, LP decoding over tightened polytopes, and (ML) maximum likelihood decoding are provided. For channels other than the BEC, we discuss why LP decoding over P with a linear objective function is insufficient.

I. INTRODUCTION

Introduced by Arıkan, polar codes are the first explicit

code sequences that provably achieve capacity on a B-DMC

with low encoding and decoding complexity [1]. Let W be

a B-DMC and let I(W ) denote the mutual information with

uniform distribution on the inputs. If W is symmetric then

I(W ) is the capacity of the channel. The discovery by Arıkan condenses to: (1) Asymptotically in block length N = 2n,

after n polarization steps, it is possible to synthesize N virtual

channels, N I(W ) of them noiseless and N (1−I(W )) of them

completely unreliable. (2) By transmitting bits through the

noiseless virtual channels, under successive cancelation (SC)

decoding (defined in [1]), polar codes achieve capacity. The

block error probability PB of the original polar codes under SC decoding decays stretched-exponentially in N as O(2−Nβ )

Inspite of the good performance guarantees, the finite-length

performance of polar codes under SC decoding is not record

breaking. Arıkan introduced a belief propagation (BP) decoder

on a sparse factor graph representation of polar codes to

compare its performance with Reed-Muller codes [4]. In [5]

BP was used to illustrate the significant performance gains that

can be achieved compared to SC decoding. It was noticed that

for channels other than the BEC, the schedule of the message

update in the BP decoder plays a significant role. The use of overcomplete factor graph representations was shown to improve BP decoding over the BEC. However, such gains were not realized on the binary-input additive white Gaussian noise channel (BAWGNC) [5]. This is due to the sensitivity of the BP decoder to message-passing scheduling. An ML decoder was implemented via the BCJR/Viterbi algorithms on the codeword trellis for short polar codes [6].
The present paper introduces linear program (LP) decoding of Feldman et. al. [7] for polar codes and is of independent interest for decoding high-density parity check codes which have some structure. An advantage of LP decoding is that it does not involve any schedule. Moreover, tightening the polytope (e.g. by using overcomplete representations) can only improve the performance of the decoder. Other benefits include the ML-certificate property, algebraic structure, adaptive LP decoding, facet-guessing improvements, and related (branchand-bound) mixed-integer formulations.

III. A NEW LP POLYTOPE P
The discussion in the previous section suggests that we need a different (better) polytope for the LP decoding to succeed. Here we show how to use O(N log N ) auxiliary variables to define an LP polytope P for polar codes. Our polytope is based on the sparse representation of Reed-Muller codes which was used by Arıkan in [1] and [4] as well as in [5]. This polytope has strong resemblance to the efficient representation of Q¯ used by Feldman et. al. [7] for high density parity check codes which involves O(N 3) auxiliaries. Following this work, Yang et. al. describe a cascade LP using a polytope of O(N α) (1 < α < 2) auxiliary variables [10]. The noted polytopes’ projections are equivalent to Q¯, whereas the projection of the polytope we introduce is provably tighter than Q¯.
To define P, we exploit the sparse factor graph representation of polar codes, shown in Fig. 1 for block length N = 23 [1]. The vectors u, a∗,1, a∗,2, and x are binary and there is a 1-1 mapping between them defined through the generator matrix u = xGN . The circle nodes in the diagram represent a total of N (1 + log N ) variables. The squares in the diagram are check nodes, e.g. u1 ⊕ u5 ⊕ a1,1 = 0. Notice that each check node is only connected to a maximum of three variable nodes.
In order to define an LP relaxation, we use the notation τ (xi) ∈ [0, 1] to denote the real variable associated to the binary xi ∈ {0, 1}. In addition, τ (x) ∈ [0, 1]N is the real vector associated to x ∈ {0, 1}N . In Fig. 1, for all binary nodes, we introduce τ (u), τ (a∗,1), τ (a∗,2), and τ (x). The convex polytope P is defined as the intersection of local minimal polytopes of each check node in the graph. More generally, let a factor graph G contain independent node sets S = {1, . . . |S|} where |S| = N (1 + log N ) for variable nodes and J = {1, . . . , |J |} for check nodes where |J | = N log N . Variable nodes include the u, auxiliaries a∗,∗, and x variables. Edge connectivity is defined via Arıkan’s sparse factor graph of polar codes (see e.g. Fig. 1). The polytope P is then defined as following.

0 ≤ τ (s1) ≤ τ (s2) + τ (s3),
0 ≤ τ (s2) ≤ τ (s3) + τ (s1),
0 ≤ τ (s3) ≤ τ (s1) + τ (s2),
τ (s1) + τ (s2) + τ (s3) ≤ 2.

B. Tightening Polytope P
The polytope P may be tightened systematically in several ways. One method particular to polar codes is to interchange the transformations in the factor graph of Fig. 1 as done in [5] for the BP decoder. The transformation G⊗2 n where n = log N is represented as a∗,1 = uT1, a∗,2 = a∗,1T2, etc., so that x = uT1T2 · · · Tn. There are n! ways to permute the Ti so that the transformation between u and x does not change. Constraints from each redundant factor graph can only strengthen the LP relaxation and improve performance. Another systematic method, called “lift-and-project” introduces higher-order interactions between variables [12, Sec.5.4.2] and does converge to the codeword polytope.

As described in Sec. III-B, let x = uT1T2 · · · Tn. An adjacent factor graph permutation switches Ti with Ti+1. There are only O(n) adjacent permutations (out of n! total). Adding

hyperplane constraints corresponding to these overcomplete

representations improves LP decoding performance (see Fig. 3

conducted with MOSEK LP solvers [13].

For a wider class of channel models like the BSC, polytope

P is insufficient as the block length N increases. Let us

consider the following, which explains the failure of LP for other channels, e.g., BSC. Let ¯i denote the number obtained

by bit reversal of i − 1. Consider any bit ui where i ∈ I. Let us assume that the decoder knows all uj for ¯j ≤ ¯i. Also

assume that we do not know the ujs if ¯j ≥ ¯i (this is not true because some of those indices may belong to I). It is reasonable to assume that the error probability in decoding ui (under the above assumptions) is less than the block error probability of LP decoding. Now, consider the error probability of decoding only the bit ui using LP. For this, it is enough to consider the tree code corresponding to the bit ui (see e.g., Fig. 2). LP on a tree is equivalent to min-sum. However, we only know that the root has low error probability under bit MAP decoding which is equal to BP on that tree. Under minsum decoding we are definitely suboptimal as far as the root is concerned and this results in a large error probability. We note the success of BP decoding on the sparse factor graph which requires specialized scheduling of messages between the u and x [5]. Therefore, the aim of future work is to design algorithms based on optimization (LP or semi-definite programming) which emulate BP closely on a tree. The hope is to achieve competitive performance with a polynomial amount of constraints as shown in the BEC case.
