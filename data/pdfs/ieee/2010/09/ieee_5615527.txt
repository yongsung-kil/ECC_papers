Abstract—The new Digital Video Broadcasting Satellite (DVB-S2) standard is able to provide capacity gains of about 30% over the previous standard by using a powerfull Forward Error Correction (FEC) scheme based on very large LDPC codewords and BCH codes. The implementation of the DVBS2 FEC decoder is a big challenge. The designer must deal with the overall design complexity and the decoding throughput in order to obtain a high decoding performance in terms of bit error rate (BER). We present in detail a complete design flow allowing a better understanding of the algorithm in terms of complexity, performance and its hardware implementation. We focus on complexity-performance trade-offs due to message quantizations and we compare its effects on several algorithm corrections used to check nodes for DVB-S2 decoding. The simulation results show that the best compromise between complexity and performance is obtained for the FOMS algorithm approximation.
Keywords-Forward Error Correction, LDPC, check node approximation, architecture for DVB-S2 decoding.
I. INTRODUCTION
The Digital Video Broadcasting by Satellite (DVB-S2) is the second generation standard [1] which offers the forward link to broadband satellite services. It uses a powerful FEC system based on a inner Low-Density Parity Check code (LDPC) concatenated with a outer BCH code, allowing a theoric Quasi-Error-Free operation at about 0.7 dB to 1 dB from the Shannon limit. In the last years many researches on low complexity decoders for DVB-S2 have been presented. Different aspects have been addressed such as architecture parallelization for DVB-S2 decoding [5]. Some recent works improve the convergence of the DVBS2 decoding by using a new decoding schedule combining a Gauss-Seidel algorithm [13] [8] (also known as layered decoding) with the two-phase message passing algorithm (flooding schedule) [9]. Also in order to decrease the decoder complexity it is necessary to use suboptimal check node approximations as min-sum algorithms [4]. No detailed works address quantization effect analysis on approximation algorithms and complexity-performance trade-offs for DVBS2 decoding. It is due to the very complex nature of LDPC codes used in DVB-S2 and the high decoding performance requirement defined in the standard (FER=10−7), so long simulation time are required to deal with these trade-offs. In the context of this paper the evaluation of the DVB-

S2 approximation algorithms in terms of performance was possible thanks to the use of an especially build hardwareaccelerated Matlab simulation platform. This approach is especially needed for floor-error detections.
This paper is organized as follows. In section II, the LDPC codes used in the DVB-S2 standard and the characteristics of the general DVB-S2 standard are presented. In section III the belief-propagation (BP) algorithm (used for the decoding) and the reduced-complexity algorithms (studied and implemented in this work) are introduced. Section IV details the general parallel architecture, functional units (FUs) which process the reduced-complexity algorithms and allow high decoding throughput. Then, the general development flow used for both algorithmic and implementation studies is presented (Section V). In the last section (VI), implementation complexities, throughputs and decoding performance results (obtained for several reduced-complexity algorithms and data quantizations) are discussed and compared. Finally, the hardware-accelerated Matlab simulation platform (which significantly reduces by 360 the simulation time) is presented.
II. DVB-S2 LDPC CODES
DVB-S2 LDPC codes are linear block codes whose sparse parity check matrix H is characterized by Nldpc columns and Nldpc − Kldpc rows. Kldpc represents the information block length, Nldpc is the resulting codeword length and Nldpc − Kldpc is the redundancy length generated by the encoder. There are two types of codewords of either Nldpc = 64800 or 16200 bits (designated as long and short codewords respectively), moreover 11 code rates (see table I) and 4 modulation schemes (QPSK, 8 PSK, 16 APSK and 32 APSK) are specified. The DVB-S2 LDPC codes can be described by the Tanner graph [12] of figure 1 and the corresponding H matrix of figure 2. The Tanner graph is composed of two types of nodes, bit nodes (BN) and check nodes (CN). A check node j is connected to bit node i only if Hij = 1. The degree of a node represents the number of edges connected to the node. The DVB-S2 LDPC codes are irregular, which means that BN nodes are of varying degree as showed in table I. Bit nodes are divided in two subsets, information nodes (IN) and parity nodes (PN). PN connections to check nodes have

IV. PARALLEL ARCHITECTURE
The large size of DVB-S2 codes, the need to support codeword length and a set of different code rates make any hardware implementation very challenging in terms of complexity, decoding performance and throughput. A direct and fully parallel architecture implementing the Tanner graph structure is obviously impractical. Hopefully the DVB-S2 code is hardware-aware and the periodicity of the H matrix

A. Architecture and node memory mapping
BP algorithm performs node update in a flooding manner, firstly CN nodes and then BN nodes. At each step node processing can be parallelized taking into account the H matrix periodicity. Hence functional units (FU) can group both CN and BN processing with a maximum of M = 360 FUs. Equation 1 shows that one CN is connected only to one IN belonging to a group of 360 successive INs, allowing the parallel processing of 360 INs of a group. All INs within a group have a same connection degree. As showed in table I node degrees are not fixed, so in order to save hardware resources node messages are processed in a serial manner, one per clock cycle (and a maximum of 360 in parallel with 360 FUs). As previously explained equation 1 eases the permutation network architecture as CNs connection addresses within a given IN group are relative to one value (given in the norm for the Wc or W3 messages of the first bit node of each IN group) and q code parameter.
Messages generated by FUs during each half algorithm iteration are stored in RAM. Figure 4 shows the general architecture of the DVB-S2 decoder. The greater part of the architecture is dedicated to messages storage. ”M ess IN mem” denotes memories used to exchange messages between IN and CN nodes. These memories have two ports, one for reading incoming messages and one for writing results coming from FUs after BS shuffling. ”M ess P N mem” memories hold messages shared between PN and CN nodes. ”CH IN mem” and ”CH P N mem” store respectively the information and parity part of channel bits.
In [5] a modulo M parallel architecture is presented. It allows to decrease the number of FUs from 360 to any integer number modulo of 360 (180,90,45). We have applied this architecture folding to decrease the number of FUs to 180. Hence the throughput is divided per a factor of two compared to a full implementation, but decreases the decoder complexity. Node memory layout is unchanged, only the overall control is adapted to match the new processing scheme of node groups.
The fixed zigzag connectivity between CN and PN could be exploited to reduce the memory needed for the exchange of messages and improve decoding performance. So during the sequential processing of CNs we can compute an already updated message for the next CN. This forward updated message is stored in the ”Reg F W ” registers associated to each FU. Backward message updates (right to left) will only be used in the next iteration, so they must be saved in the parity node message memories (M ess P N mem).

B. FU architecture
FUs implement both BN and CN node processing. We chose a serial implementation for FU in order to save resources and ease the overall control. Our FU architecture is highly pipelined using registers. That allows the processing of two different nodes at a same time to increase the decoder throughput. All incoming messages ”M ess IN ” read from ”M ess IN mem” are firstly processed in the input message stream side. Once all input processing have been done on incoming messages (summations and storage in BN mode; sorting, sign summations and sign storage in CN mode), we can process a next node and compute the output messages of the previous node in the output stream side. Figures 5 and 6 sketch architectures of BN and CN node in the case of the FOMS algorithm. Logic for PN update is not shown for the CN mode. For long-frame code rates there is no problem on CN mode pipelining when processing two nodes with different degrees, because CNs have a constant degree and FIFO management is seamless. But for short-frame some code rates have irregulars CNs degree and input pipelining must be stalled for some cycles before processing a CN with a lower degree. Since both modes are used successively synthesizer tools can use logic sharing techniques to save resources.

C. Min-sum corrections and resources-performance tradeoffs
A 6 bit quantization (W llr) for the LLR with 4 bits of precision (W llr f ) and one for the sign is the minimum required bit quantization to achieve the required performance defined by the norm and from the point of view of implementation considerations (IP area). 4 bits of precision for LLRs is needed for good decoding results on all code rates in regards with our correction values.
Two quantizations have been tested for node messages and FU operations. The first one is a 7-bit implementation where messages, FU operations and hard decoding use 7bit data (4 of precision and 1 for the sign). The main drawback of this quantization are the growing of message size in memory and FU resource needs. The advantage is an accurate correction proceeded by FUs in CN mode and a greater message dynamic. The second quantization requires only 7-bit adders FUs in BN mode and for the hard decoding, 6-bit for input and output messages as well as for min-sum correction.
Table II shows implementation results of the DVB-S2 decoder for the four BP approximation algorithms on a Stratix III EP3SE260 FPGA. Only the LDPC part of the decoder is shown and synthesizer optimization is tuned to balance (area-speed). FU 4x means that four FUs are grouped in one VHDL component, allowing a better resource sharing management by the implementation tool (QuartusII). Hence our architecture is composed of 180/4 = 45 FU 4x. VII. CONCLUSION
A full development flow addressing DVB-S2 decoding has been presented. It allows fast and extended experimentations of check node algorithm approximations needed to decrease the decoder complexity. Practical results on decoding performance and complexity tradeoffs have been discussed. The Matlab-based simulation acceleration allows quantization effect study and error floor effect at very low BER for DVBS2 check node algorithm approximations. From our results the best compromise between complexity and performance is clearly the FOMS algorithm approximation.
