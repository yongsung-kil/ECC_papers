Abstract-We propose a novel design of rate-compatible

protograph-based low-density parity-check code families that can

cover a wide range of code rates. In contrast to traditional method

of lifting, our lifting method use a combination of different

circulant matrix sizes for the base protograph and accumulator

to increase the number of achievable rates. We apply this method

� to an existing code family called AR4JA, which can offer variable

rates of R =

by process of extension. The proposed

codes namely mixed-circulant rate-compatible protograph code

(MCRC-PC) can reduce the rate step within the AR4JA family.

We show that the proposed codes can also retain many of the

favorable properties of protograph codes such as a fast encoder

and decoder implementation. A set of length k=1024 MCRC-PC

codes is designed with decoding thresholds obtained from multi­

edge type density evolution, and simulation results show that the

rate compatible code family can achieve good performance under

message passing decoding algorithm.

I. INTRODUCTION
Low-density Parity-check (LOPC) codes were initially pro­ posed by Gallager [1] in early 1960s. Their excellent perfor­ mance with iterative decoding over various channels is over­ shadowed by the complexity of encoding and decoding. After Richardson and Urbanke's landmark work [2]-[5] on these codes, there has been renewed interest in the development of new LOPC codes. One of these improvements is to design LOPC codes with imposed sub-structure, such as multi-edge type LOPC codes in [5]. These sub-structures not only allow for a compact description of LOPC codes, but also aid the development of more efficient algorithms.
Flexible rate is a desired feature in the design of practical error control system. Rate-compatible (RC) codes are a family of codes of different rates that can be decoded conveniently and efficiently using a common hardware implementation. Some previous works on the design of rate compatible LOPC codes are [6]-[9]. In most of the schemes, puncturing has been widely used to construct rate-compatible LOPC codes. In [6], the author proposed that good puncturing patterns exist and that the puncturing can be performed in a rate-compatible fashion with small performance loss. Afterwards, Zheng et al. in [7] proposed a method to design rate-compatible LOPC codes based on progressively increased column weights, the performance loss due to the puncturing can be reduced with no puncturing patterns are required. Moreover, the author in [8] shows that puncturing alone is insufficient for the formation of a sequence of capacity-approaching LOPC codes across a wide range of rates. The author used extending (adding columns

and rows to parity matrix) and puncturing to achieved rate­ compatibility. Expurgation and lengthening techniques in [9] are used to construct rate-compatible protograph [11] based LOPC codes. Expurgation is accomplished by splitting check nodes, equivalent to information shortening, and lengthening is accomplished by attaching extra variable nodes.
An existing rate-compatible protograph code family was proposed in [10] [16] [17] called AR4JA code family. The
code family can achieve code rates of 1/2 and higher, and
good iterative decoding performance for each code. However, the rate step is large between two consecutive code rates. For
instance, the next code rate after rate 1/2 is 2/3. Hence, the
number of achievable code rates is limited. We extend the work in [10] and [16] and present in this paper a new method to design protograph based rate-compatible LOPC (RC-LOPC) code family that is able to increase the number of achievable rates in the AR4JA code family without changing the structure of protograph. However, the method is not suitable for IR­ HARQ applications due to the block size of each code is not constant within the code family. In the next section, a general overview of definition of a protograph is introduced. Followed by a description on how the proposed method achieves more code rates for a fixed dimension length k. Section IV and V discuss the structure of fast encoder and decoder for the new codes, respectively. Finally, simulation results are shown to illustrate the practical performance of our method.

typical protograph is a small bi-partite Tanner graph with only a small number of variable and check nodes. These small pho­ tograph are then 'lifted' to form larger codes by introducing circulant permutations on each edge. This is known as the "copy and permute" operation [11]. The operation consists of
first making N copies of the protograph. As a result, each protograph edge becomes a bundle of N edges, connecting N variable nodes to N check nodes. Then the copies of the
protograph are interconnected by permuting these variable to check pairings within each bundle. After the permutations, the resulting LDPC code is known as the lifted code. The
dimension of the lifted code is N times as large as the code
corresponding to the original protograph with the same rate and the same distribution of variable and check node degrees. Thus, LDPC codes with protograph structure are a subclass of multi-edge type LDPC codes. When the permutation applied to each edge of the protograph is circulant, the resulting lifted code is permissible to high speed decoding.

III. R ATE-COMPATIBLE DESIGN
Our motivation is to implement a rate-compatible approach for protograph-based LDPC codes by modifying the lifting process. The proposed code families that are designed from our method should achieve a smaller rate step (obtain more code rates) and good iterative decoding performance. In the following section, we present the method of how to obtain
more rates when information length k is fixed by using the
protograph from the AR4JA code family as an example.

IV. EN CO DER DESIGN
A fast encoder implementation for protograph codes has been designed by Andrews et al. in [14]. This encoder can be adapted to MCRC-PC with minor modifications. The proposed encoder converts a block circulant parity check matrix into a systematic block circulant generator matrix, which may be implemented efficiently with shift registers. This construction methodology may not be applied directly to MCRC-PC since the circulants are not all of the same size. In order to address

this issue, we perform the encoding in two stages. Firstly, the

message symbols corresponding to the nodes on the accumu­

lator circulant matrices are transformed into a format suitable

for the systematic generator. Then these transformed symbols

are concatenated with the original symbols and multiplied by

a modified generator matrix to create the parity check bits.

This process is illustrated with an example.

V. DECO DER DESIGN
The decoder for MCRC-PC is conceptually the same as the one proposed in [15]. The decoding process for an LDPC code generally involving several thousand variable and check nodes. A typical FPGA or ASIC might not be well matched to the size of the protograph, also it is not possible for the node computation circuit to perform full message updates in a single clock cycle. In practice, it is natural to design a decoder for protograph codes by expanding the protograph into the available size of FPGA and updates message of nodes in a pipelined manner. In [15], a decoder for structured protograph based codes has been proposed by Jones et al. The concept of this decoder is that for LDPC codes with protograph structure, the message passing architecture is partially parallel and partially serial, where inside each copy of protograph, all of the variable (check) nodes can update simultaneously, and that each copy updates serially.

VI. COMPARISON WITH OTHER TYPES OF LDPC CO DES
It is clear that our construction is a form of multi-edge type LDPC code as it consists of a number of edge types that occur with a certain percentage. It is also clear that it is not the same as existing protograph codes or regular LDPC codes. It does not fall in to the protograph family as there is no single protograph which can generate this code from a "copy and permute" operation. It is also not regular, as the edges are not fixed, some edges are "deleted" due to the mismatch in circulant size. Therefore, the only possible existing technique for describing the code is to use a multi-edge type degree distribution. This description is both inefficient and tedious for our codes as there are many edge types in the protographs, which are multiplied by the mismatched circulant sizes. It would be desirable to find a compact description for this code combined with an exact or approximate density evolution algorithm.
The concatenation between the base and accumulator by different size of circulants destroys the property of determin­ istic neighborhood. To illustrate this, consider the situation in
Fig. 3 b) where M :::; N, all of the edges of the accumulator
are connected to check nodes, but some check nodes are not connected to accumulators. If the circulant size for variable node 3 is less than that of other variable nodes, we cannot determine directly from the protograph that whether the check
node A is connected to variable node 3. Thus, the lifted code
will no longer retain the same performance characteristics as the original protograph.

VIII. CONCLUSION
We have proposed a novel lifting scheme for protograh­ based LDPC codes that can provide rate compatibility. The

scheme uses a combination of different circulant sizes during
the traditional lifting process. By doing so, we increase the
number of achievable rates in a protograph code family.
The proposed methodology has been verified with simulation
results by applying the method to the AR4JA code family. The
fast encoder and decoder structures of mixed-circulant rate­
compatible protograph (MCRC-PC) codes are discussed. For
future work, theoretical performance analysis may be required
for this type of codes, in particular, a proper description
in terms of degree distribution is needed. With acceptable
complexity requirements, further designs can be carried out
by considering multiple circulant sizes rather than just two.
