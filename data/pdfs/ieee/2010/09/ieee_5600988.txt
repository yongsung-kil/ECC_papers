Abstract—Group shuffled belief propagation (GSBP) algorithm is a modified algorithm to decode LDPC codes. Compared with belief propagation decoding, this algorithm converges faster. In this paper, we analyze the performance of GSBP decoding by means of Gaussian approximation. Under the assumption of Gaussian density, the updating message means during GSBP decoding are derived. The results show that the proposed algorithm can verify the fast convergence property of GSBP decoding effectively.
Index Terms—Gaussian approximation (GA), Group shuffled belief propagation (GSBP) decoding, Low-density parity-check (LDPC) codes.
I. INTRODUCTION
Low-density parity-check (LDPC) codes have been receiving a lot of attention, mostly due to their near-Shannonlimit error performance when decoded with the iterative belief propagation (BP) algorithm [1]. However, The BP algorithm needs large number of iterations for the decoding process to convergence. Based on BP decoding, Group shuffled belief propagation (GSBP) decoding algorithm is proposed in [2] to speed up the convergence. In this algorithm, the bit nodes (BNs) are divided into groups. These groups are performed sequentially to complete one iteration. In this manner, updates become available at the same iteration, which can achieve a fast convergence.
Several techniques have been proposed in the literature to analyze the performance of iterative decoding, including density evolution (DE), Gaussian approximation (GA), and extrinsic information transfer (EXIT) charts. DE algorithm [3] is performed by tracking the densities of messages which pass during the decoding process. GA algorithm [4] is proposed to simplify DE over a binary input additive white Gaussian noise (biAWGN) channel. It carries out the asymptotic analysis of LDPC decoding by tracking the updating means of Gaussian densities. EXIT charts [5] are based on the mutual information, and this method is performed by plotting two EXIT curves in closed form under a specific signal-to-noise ratio (SNR). The performance of GSBP decoding algorithm is analyzed by means of DE and EXIT charts in [6]. And in [6], the results predicted by DE algorithm are based on the assumption that no two BNs in the same group are connected to the same check node (CN). It will be unsuitable when the number of groups is small.

In this paper, we will use GA algorithm to analyze the performance of GSBP decoding. Assuming the parity check matrices of LDPC codes are randomly constructed, we derive the GA algorithm under GSBP decoding. Using the proposed algorithm, we can prove that GSBP decoding converges faster than standard BP decoding without lowering the error performance.
The paper is organized as follow. GSBP decoding of LDPC codes is introduced briefly in Section II. The derivation of GA algorithm under GSBP decoding is given in Section III. In Section IV, an ensemble of LDPC codes is analyzed by means of GA algorithm and the results are discussed. Finally, conclusions are made in Section V.
II. GSBP DECODING ALGORITHM OF LDPC CODES
A binary (N ,K) LDPC code is described by M × N parity check matrix H. The parity check matrix H can be viewed as a bipartite graph with two kinds of nodes: N BNs corresponding to the columns of H and M CNs corresponding to the rows of H. This codeword is modulated by binary phase shift keying (BPSK), and then transmitted over a biAWGN channel with zero mean and variance σ2. yn (n = 1, ..., N ) is the corresponding received signal.
We denote the set of CNs adjacent to BN n (n = 1, ..., N ) as Cn, and the set of BNs adjacent to CN m (m = 1, ..., M ) as Rm. Cn\m represents the set Cn with CN m excluded. Similarly, Rm\n represents the set Rm with BN n excluded. Let L(qn(lm) ) and L(rm(l)n) be the log-likelihood ratio (LLR) messages which are sent from BN n to CN m, and CN m to BN n at the lth iteration, respectively. L(zn) is the LLR messages from the channel.
In standard BP algorithm, the belief information is updated iteratively and can be implemented in two phases at each iteration. In the first phase, all check-to-bit messages L(rm(l)n) are updated by using bit-to-check messages obtained at the previous iteration L(qn(lm−1)); in the second phase, all bit-tocheck messages L(qn(lm) ) are updated by using the new obtained check-to-bit messages L(rm(l)n).

IV. SIMULATIONS AND ANALYSIS
Fig.1 depicts the bit-error rate (BER) of a (480,960) (3,6) LDPC code [7] with standard BP and GSBP decoding for G=1, 2, 4. G is the number of BN groups. We set the maximal iteration number to 8. It is seen from Fig.1 that at the BER 10−4, GSBP decoding with G=2 performs 0.22dB better than standard BP decoding, and the larger the value of G, the better the error performance.
Then we will use GA algorithm proposed in Section III to analyze the performance of GSBP decoding. For different LDPC codes, the thresholds calculated by GA algorithm for GSBP decoding are equal to that for standard BP decoding. It means that with sufficient large number of iterations, both standard BP and GSBP decoding algorithm achieve the same error performance.
