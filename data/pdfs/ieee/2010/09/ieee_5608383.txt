Lower the Error Floor ofLDPC with Fixed Error Pattern

Chen Huang, Liang Zhou, Hong Wen, Qian Zhao, Fen Xu
Communication Nation Key Laboratory University of Electronic Science and Technology of China

Abstract-This paper presented a FEC scheme with very high performance and high information bit rate. By analyzing the error-floor characteristics of a family of QC (Quasi-Cyclic)­ LDPC codes, we design a concatenated code to eliminate this kind of trapping set which caused fixed error pattern. The simulation results have demonst-rated the performance of the novel scheme is efficient. The error floor that we estimate is below Ie-IS. This approach can be used on the high speed IOOGp/s communication.
Keywords-EG, QC-LDPC, trapping set, error floor, concate­ nated code

I.

INTRODUCTION

The very growing needs for faster and reliable comm­
unication systems have forced many researches to seek ways to attain ultimate limites of reliable communication [1]. Error control using Low-Desity Parity-Check (LDPC) codes is currently the most opromising coding technique to achieve Shannon capacity for a wide range of channels. LDPC codes, discovered in the early 1960s [2] and rediscovered in 1996 [3], have recently attracted much attention due to their capacity-apporoaching performance and low decoding complexity. There have been many contributions to the design and analysis of LDPC codes. Base on the methods of construction, LDPC codes can be classified into two classes: random-like codes, which are generated by computer search and strcured codes, which are constructed based on algebraic and combinatorial tools.

Although LDPC codes has good performance and very close to the Shannon capacity, LDPC codes could not satisfy the development of high speed communications. LDPC codes also have the problem of error-floor. Some applications, such as Optical Communications [4] and Deep
Space Commu-nications, require FEC without error-floor below Ie-IS BER. So how to reduce LDPC error-floor is very importance.

This paper is concerned with analysis of EG-LDPC trap­ ping set and reduction of EG-LDPC error-floor base on conca-tenated code. A binary EG QC-LDPC is constructed. Experi-mental results show that the constructed code perform have error-floor at Ie-IO BER over the AWGN
channel with min-sum iterative decoding. Therefore, concatenated code is used to eliminate the trapping set. The performance of code is very well.

This paper is organized as follows: A method for constru­ cting QC-LDPC codes based on EG is proposed in Section II. Trapping set of LDPC code is analyzed in Section III. Section IV presents the method to eliminate the trapping set. Section V is simulation result. Finally, Section VI concludes the paper.

III. THE TRAPPING SET OF LDPC CODES
LDPC codes adopt BP (Belief Propagation) decode method. This method is an near-optimum algorithm, and have a wonderful performance. However, for sufficiently high SNRs, some of these codes tend to exhibit sudden changes in the slope of the bit-error rate (henceforth, BER) curve. This change in the slope of the BER curve is known as the error-floor of the code. The error floor is mainly caused by trapping sets, stopping sets and absorbing sets. The last two sets can be seen as some forms of trapping sets under special circumstances. Trapping sets also involve the above mentioned two kinds of error.
The Tanner graph of an LDPC code is a bipartite graph with two sets of nodes: variable (bit) nodes and check (constraint) nodes. The check nodes (variable nodes) connected to a variable node (check node) is referred to as its neighbors. The degree of a node is the number of its neighbors. The degree of a node is the number of its

neighbors. The girth g is the length of the shortest cycle in The Tanner graph [4].

The exist of trapping set in iterative decoding is the main reason cause error floor. Error floor is determined by the size and distribution of trapping set in LDPC.
To solve the problem of error floor, there are three methods:
I. Improving BP iterative algorithms
Averaging or factor multipling the conveyed information of check node stop the shock of message of variable node, so increasing the possibility of get out the trapping set and converge to right codewords. But this method is effective to partly trapping sets with the sacrifice of convergence rate.
2. Improving coding method
Preventing mainly concentrate of trapping set from cons­ truct good topological structure, the method is very effective to middle and low bit rate LDPC code, but the design of high bit rate LDPC is very hard.
3. Using concatenated code: LDPC as internal code, alge-bra code as outside code
This method is very effective to variable and high bit rate LDPC. This is the one we used to lower the error floor of LDPC.
We focus on the method of concatenated code. Because the first two kind of method is very hard and inefficient, using concatenated code method could easily to eliminate the error floor. But this method is efficient to LDPC with fixed trapping set pattern. Because the (13299, 11285) LDPC code has the (4,4) trapping set with the fixed pattern which is a cyclic type and it is have high bit LDPC 0.85. So we consider that using a concatenated code to correct this kind of error. The (4,4) trapping set cause the error bit in a frame is 4, this is a small error, so we use a high code rate code to concatenate. Considering (3860, 3824) BCR code, this code code rate is 0.99, and it could correct 3 bit error. It's a shorten BCR.
