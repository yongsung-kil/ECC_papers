Integer Linear Programming Based Optimization of Puncturing Sequences for Quasi-Cyclic Low-Density Parity-Check Codes

Anton Blad, Oscar Gustafsson Electronics Systems, Linkoping University
SE-58I 83 Linkoping, Sweden Email: antonb@isy.liu.se.oscarg@isy.liu.se

Meng Zheng, Zesong Fei RCDCT, Modern Communication Lab, Dept. of E. E.
Beijing Institute of Technology, Beijing, China Email: zmra@bit.edu.cn.feizesong@bit.edu.cn

Abstract-An optimization algorithm for the design of punctur­ ing patterns for low-density parity-check codes is proposed. The algorithm is applied to the base matrix of a quasi-cyclic code, and is expanded for each block size used. Thus, storing puncturing patterns specific to each block size is not required. Using the optimization algorithm, the number of I-step recoverable nodes in the base matrix is maximized. The obtained sequence is then used as a base to obtain longer puncturing sequences by a sequential increase of the allowed recovery delay. The proposed algorithm is compared to one previous greedy algorithm, and shows superior performance for high rates when the heuristics are applied to the base matrix in order to create block size­ independent puncturing patterns.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1], [2] have emerged as an error-correcting scheme for many wired and wireless technologies. Communication standards that have adopted LDPC codes as either mandatory or optional chan­ nel coding schemes include DVB-S2, IEEE 802.lIn, IEEE 802.I6e, IEEE 802.3an, and as the theoretical performance of LDPC codes significantly exceed those of traditional coding schemes, their use is expected to increase.
For both traditional convolutional codes and Turbo codes, puncturing is commonly used to achieve rate-compatible codes [3]. The benefits of rate-compatibility are many. Sharing of the encoder and decoder for different rates is permitted by the fact that higher-rate codes consist of a subset of the bits in the lower-rate codes. Also, rate-compatibility allows incremental redundancy communication through type-II Hybrid ARQ [3], [4] and relaying [5]. However, most of the current LDPC implementations use dedicated codes for the different rates, and only recently has rate-compatibility been investigated for LDPC codes.
The idea of investigating the number of decoder iterations needed for a punctured node to be recovered by its neighbours was first proposed in [6]. A greedy algorithm was proposed, that successively chooses variable nodes that receive likelihood data from their neighbours as early as possible. Later, such nodes have been denoted k-step recoverable (k-SR), for nodes that receive non-null information after k iterations [7]. Since then, several other greedy algorithms have been proposed. In

[8], an algorithm that takes into account the reliability of the information used for reinitializing punctured nodes was suggested. However, it can only be used for parity-check matrices with dual-diagonal structures. In [7] and [9], algo­ rithms maximizing the reinitialization reliability for general parity-check matrix structures were proposed. The algorithm in [7] is essentially a refinement of the one proposed in [6], sequentially puncturing k-SR nodes for increasing values of k while trying to choose nodes with high recovering reliability. In [9], the aims are similar, but instead of minimizing the number of iterations for recovering, the algorithm tries to choose nodes based on the number of unpunctured nodes used in the computation of the recovery information.
The problem of the previous approaches is apparent when scalable block sizes are considered in addition to code rate flexibility, both of which are usually desired in practical use. Scalable block sizes are commonly enabled using quasi­ cyclic LDPC (QC-LDPC) parity-check matrices [10]. QC­ LDPC parity-check matrices are defined as the expansion of a base matrix by replacing each element by a square matrix which is either the zero matrix or a cyclically shifted identity matrix. To retain the scalability of the block size in conjunction with rate-compatibility, there should be a fixed rule of deter­ mining the puncturing sequence from the scale factor and a base puncturing sequence. One such algorithm is provided in [11], where a deterministic and structured puncturing method is presented. However, it is limited to matrices with dual­ diagonal structure, and also provides puncturing sequences with unnecessarily long recovery delays. It is also possible to apply the current greedy algorithms to the base matrix in order to construct a base puncturing sequence. However, there are two problems. First, using greedy algorithms may yield a suboptimal base puncturing sequence, in terms of node recovery delay, and the suboptimality is amplified by the base matrix expansion. Second, for many of the greedy algorithms, random choices are made in several steps, which result in very different recovery delays when applied to the base matrix. Thus, the number of k-SR nodes depends on the random choices made.

II. RATE-COMPATIBLE LDPC CODES
A. LDPC Codes
An LDPC code is defined as a linear block code with a
very sparse parity-check matrix H [I], [2]. Commonly, the
code is visualized using a Tanner graph [12], a bi-partite graph containing a set of variable nodes and a set of check nodes. The variable nodes correspond to the columns of the parity­ check matrix, and the check nodes correspond to the rows. There is an edge between a variable node and a check node if the intersection between the corresponding column and row in the parity-check matrix contains a one. An example of a Tanner graph is shown in Fig. I.
A QC-LDPC code is commonly defined from a base matrix
Hb of size Mb x Nb. The elements of Hb are either -lor non­ negative integers. The parity-check matrix H of scale factor z is then defined from Hb by substituting each element by a square matrix of size z. Thus the size of H is M x N = 2Mb X ZNb. For each -1, the zero matrix is substituted, and for
each non-negative integer, a circularly shifted identity matrix is substituted. For the shifted identity matrices, the shift is defined as a function of the scale factor z and the base matrix element. Several wired and wireless standards, including IEEE 802.lIn, IEEE 802.16e, and IEEE 802.3an, have adopted QC­ LDPC codes.
Decoding of LDPC codes is commonly done using the sum­ product algorithm [13], defined on the Tanner graph of the code. The algorithm iteratively passes soft bit information

B. Rate-compatibility
A class of rate-compatible codes is defined as a class of codes where the codewords in the higher-rate codes can be obtained by removing bits in the codewords of lower-rate codes [3]. Thus the information content is the same in the different codes, but the amount of parity information differs. There are two main methods of defining such classes of codes: puncturing and extension. Using puncturing, a low-rate mother code is designed and the higher-rate codes are then defined by removing bits at fixed positions in the blocks. Using extension, lower-rate codes are defined from a high-rate mother code by adding additional parity bits. With puncturing, encoding can be done using the mother code, and bits are then selectively removed before transmission. At the receiver side, decoding can be done using the parity-check matrix of the mother code, with the log-likelihood ratio of the punctured bits set to zero.

As shown in Fig. 2, a check node is denoted I-step enabled (I-SE) if it involves a singular punctured node and is used to recover that node in the first iteration. Furthermore, a punctured variable node is denoted I-SR if it is connected to a I-SE check node. Generally, a check node is denoted k­ SE if it begins to communicate non-null information in the kth iteration and has been chosen to recover a punctured node. A punctured variable node is denoted k-SR if it is connected to a k-SE check node but to no lower valued SE check node.
It is possible to create puncturing patterns that contain nodes that will never be recovered. Such nodes thus never change value and severely impact the decoding performance. In [7], it was shown that decreasing the recovery time increases the reliability of the recovery information, and it was then conjectured that increasing the number of nodes with fast recovery enhances the performance of the puncturing pattern.

In this section, the ILP-optimized puncturing pattern design algorithm is presented. The primary goal of the algorithm is to optimize the puncturing pattern for fast recovery of the punctured nodes. Given a parity check matrix, it maximizes the number of I-SR nodes. Following that, it maximizes the
number of punctured nodes for increasing values of K, given
the puncturing patterns obtained in earlier iterations and the
constraint that only k-SR nodes with k :s; K are allowed.
As secondary optimization criteria, the algorithm chooses k­ SR variable nodes and k-SE check nodes such that first the sum of the degrees of the k-SR variable nodes is minimized, and then the sum of the degrees of the k-SE check nodes is minimized, as low-degree nodes were reported to give good results in [9].
In the first subsection, the optimization problem with vari­ able definitions, goal function and constraints is described. Also, design cuts to decrease the optimization time are dis­ cussed. In the second subsection, the proposed algorithm for designing the puncturing patterns using the ILP optimization problem is presented.

The following algorithm is proposed for the design of
puncturing sequences for a class of codes based on a base
matrix Hb with different expansion factors z. 1) Initialize the puncturing sequence P = 0 and set the maximum recovery delay K = 1. Set the costs Cp » Cr » Cv » Ce• 2) Set the forced puncturing pattern F = P and solve the optimization problem on Hb• 3) Let q denote the puncturing pattern solution. Order q \p
first by increasing recovery delay and then by increasing
node degree, and append the sequence to p. 4) If a pre-defined maximum recovery delay K has been
reached, quit. Otherwise, increase K and go to Step 2.
The result of the above procedure is an ordered sequence p
defining a puncturing order for the blocks of the base matrix
Hb. For the parity-check matrix H with expansion factor z, the puncturing sequence pz is defined by replacing each element a in p with the sequence za, ... , (z a + 1) -1.

rate-l12 base matrix ( (n, m) = (24, 12)) to create a base

The base puncturing pattern was expanded with factors of 24, 36 and 48 to create puncturing patterns for block lengths of 576, 864 and 1152. To solve the optimization problem, the selP solver [14] has been used. V. CONCLUSIONS
In this paper, an optimization algorithm for the design of rate-compatible puncturing sequences was proposed. The algorithm sequentially maximizes the number of punctured nodes given a maximum recovery delay (maximum k-SR node permitted), for increasing values of k. It is applied to the base matrix of a quasi-cyclic low-density parity-check code, in order to design a base puncturing pattern. Puncturing patterns for the expanded matrices are then obtained from the base puncturing pattern and a scaling factor. Thus, separate design of puncturing sequences for all permitted block sizes is not needed. While the performance of the optimized puncturing sequences is slightly better than the heuristics, the method should be seen primarily as a way to optimize the recovery delay of the punctured nodes.
