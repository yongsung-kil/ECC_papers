Abstract—Recently, the development of Low-Density ParityCheck (LDPC) decoding solutions has been proposed for a vast set of architectures, ranging from dedicated hardware to fully programmable ones (e.g. Cell/B.E. and graphics processing units). In this paper we propose efficient embedded programmable multicore architectures for achieving real-time LDPC decoding. The proposed multicore architectures allow to exploit data parallelism by decoding in parallel multiple codewords on the provided cores, with enough local memory capacity to store all data corresponding to the Tanner graph. Therefore, with this distributed memory and local computing approach, only a single shared bus is required to communicate the codewords. The proposed class of architectures can be prototyped on field programmable gate arrays or implemented on application specific integrated circuits, and it is validated by using the popular Cell processor, which relates very closely with the one here proposed. Finally, we discuss the related art of dedicated and programmable LDPC decoders, and discuss the advantages and disadvantages regarding the proposed solution.
I. INTRODUCTION
Initially proposed in the 1960’s by Robert Gallager [1], LDPC codes exhibit excellent error correction capacity. Due to advances in semiconductor industry and microelectronics over the last decades they recaptured the attention of academia and industry after the invention of Turbo-Codes [2] by Berrou et al. in 1993. Over the last years, LDPC codes have been adopted by modern communication standards, such as Digital Video Broadcasting, Satellite 2 (DVB-S2) [3], the WiMAX standard [4] for local and metropolitan area networks, or optical communications, and are being considered for 4G wireless systems.
Decoders for LDPC codes demand intensive processing. Efficient hardware dedicated architectures and application specific integrated circuits (ASICs) have been designed to develop embedded processors [5], [6] for real time LDPC decoding, with cost and power consumption restrictions. However, since different standards employ different LDPC codes, it is important to design flexible LDPC decoders that can support different applications. More recently, more flexible solutions for designing dedicated LDPC decoders have been proposed by using a high level synthesis (HLS) methodology [7]. Although more flexible, it has to use the program-in chip-out

tool to produce RTL descriptions adapted to the considered coder and its requirements.
With the recent advances in architectures and technology, multicore computers and accelerators became popular [8], [9], namely general purpose multicore personal computers, such as the PlayStation 3’s Cell/B.E. and graphics processing units (GPUs) that can be used for general purpose processing. LDPC decoders have been also recently developed using non-conventional programmable approaches [10], [11] that exploit the increasing computational power of these multicore systems.
This paper addresses the main challenges in the design of multicore architectures suitable for the development of programmable efficient embedded processors for LDPC decoding. Based on the experience of implementing LDPC decoders, mainly on programmable multicores [10], [12], [13], [14], [15] but also designing dedicated hardware decoders, in this paper we propose a programmable architecture that suits the fundamental requirements necessary to perform LDPC decoding with good performance at low cost. The proposed programmable architecture exploits data parallelism at different levels by processing in parallel multiple codewords. Multicores with distributed memory are used to independently decode in parallel distinct codewords, while each core exploits sub-word parallelism, a particular form of Single Instruction Multiple Data (SIMD), for individually decoding several codewords. In the proposed architecture a single shared bus is applied, since there are no dependencies between the several instances processed in parallel, and consequently the requests for communicating and synchronizing are low. The proposed architectures can be easily prototyped in field programmable gate arrays (FPGAs), by using soft-cores [16] and standard buses [17], or by designing ASICs [7].

II. LDPC CODES
A linear binary block code (N, K), with rate = K/N and M = N − K, can be described by a binary H matrix with dimensions M × N . Also, it can be elegantly represented by a Tanner graph [18], as depicted in figure 1. In this type of graph, edges connect two distinct types of nodes:
• Bit Nodes (BN), also called variable nodes (VN) with a BN for each one of the N variables of the linear system of equations, and;
• Check Nodes (CN), also called restriction or test nodes with a CN for each one of the M homogeneous independent linear system of equations represented by H.
Figure 1 shows an example for a binary (8, 4) code with N = 8 BNs and M = 4 CN parity-test equations. Each CN connects to all the BNs which have a contribution in that restriction or equation. For the other type of nodes, each BN corresponding to a ci bit of the codeword, connects to all the CN equations where bit ci participates in. The graph edges connect BNs with CNs, but never nodes of the same type, which defines a bipartite graph. Every element in H where Hmn = 1, represents a connection between BNn and CNm [1]. For regular codes, the number of ones per row/column is constant, while no restriction is imposed to this number in irregular codes. In the example of figure 1 BN0, BN1 and BN2 are updated by CN0 as indicated in the first row of H. From the second until the fourth row, it can be seen that the subsequent BNs are updated by their neighboring CNs.

III. PARALLELISM AND DATA REGULARIZATION FOR LDPC DECODING
Mainly data parallelism, task parallelism, or both data and task parallel approaches can be used to target parallel systems. In this section we identify the parallelism that can be exploited to speedup LDPC decoding and discuss a compact representation for the Tanner graph having in mind the proposal and the design of an efficient embedded multicore architecture for performing LDPC decoding in real time.
A. Parallelism to exploit
Task parallelism can be exploited on LDPC decoding at different granularities. At the finest granularity, one task can be responsible for updating a single edge of the Tanner graph. This approach would be suitable for large number of processors without restrictions accessing the graph in memory. In a more realistic approach, each task can process a node of the Tanner graph. In this case, depending on the scheduling, if a Forward-and-Backward approach [20] is used, redundant memory accesses and extra computations can be saved. In practice, for designing hardware processors, a set of dedicated cores are used to process in parallel a sub-matrix of H, and these submatrices are sequentially processed by these cores. These approaches require synchronization and communication to process all the Tanner graph from the partitions computed in parallel.
Figure 2 shows how task parallelism, herein defined as intracodeword parallelism, allows processing distinct nodes of the graph in parallel by different tasks (ti). It is based on the fact that the Tanner graph can be partitioned into distinct subsets. Each one of those subsets can then be processed by a different task or processing unit, in parallel, where tasks t0 to t3 are used to describe parallelism applied to the horizontal kernel of the Min-Sum algorithm [19], and tasks t0 to t7 represent the parallelism applied to the vertical kernel. To minimize the overheads introduced by redundant communications and to allow the efficient Forward-and-Backward recursion [20] to be adopted, the size of a task should not be set too small under the context of LDPC decoding. Typically, a set of nodes, for example composed by all the BNs associated to a CN for horizontal processing, or by all CNs associated to a BN for vertical processing, is assigned to a task.
Data parallelism can be exploited on LDPC decoding by simultaneously decoding multiple codewords. SIMD is the paradigm that can be used for exploiting fine grained data parallelism, while Single Program Multiple Data (SPMD) can be used to implement multi-codeword LDPC decoding on multiple processing units [12] that have access to the information of the whole Tanner graph. B. Data Regularization
To achieve the goal of obtaining efficient parallel algorithms for software LDPC decoders, we have developed compact data structures to represent the sparse binary H matrices and corresponding Tanner graphs. They adequately allow representing data as streams to suit the parallel processing of the SPA and MSA, which represents a different solution from the conventional Compress Row Storage (CRS) and Compress Column Storage (CCS) formats [21] used to represent sparse matrices in a compact format. In the context of parallel processing, the edges of the Tanner graph are represented using a proper addressing mechanism, which facilitates the simultaneous access to different data elements required by the SPA and MSA algorithms. The compact data structures herein proposed also suit recent parallel subword SIMD like architectures, which impose special restrictions associated with the alignment of data in memory, and simplify the simultaneous decoding of several codewords.
We propose to represent the H matrix that defines the Tanner graph of an LDPC code using two distinct data structures, namely HBN and HCN , which are used for the horizontal and vertical processing steps, described in section II, respectively.

Data structures HBN and HCN depicted in figure 4 illustrate the generic irregular case. It can be seen that they represent addresses (or indices) of elements to update. HBN contains row weights in vector wc and the corresponding addresses of Lrmn data elements to be updated by corresponding Lqnm messages arriving from nodes they connect to. The structure HCN performs the opposite operation: it contains column weights in vector wb and the corresponding addresses of Lqnm elements which have to be updated by incoming Lrmn messages from connected neighboring nodes.
The access to HBN and HCN data structures can be irregular. Multicore architectures provide resources that allow to achieve parallel/low-latency accesses to memory, typically by masking data transfers with computation [10], [22], [23], or by using efficient data alignment/resizing strategies, that facilitate parallel accesses to memory [15]. To better suit these mechanisms, data blocks to be accessed in memory should obey to specific size and address requirements, which may not be directly achievable from HBN and HCN data structures in many cases. In these cases, runtime data realignment mechanisms [15] can be applied to allow data to be accessed more efficiently, since it is not possible to align it at compile time. Also, as the memory access patterns of LDPC codes are random, they do not admit the simultaneous use of aligned memory accesses for both read and write operations of the decoding process. To overcome this difficulty we have developed data mapping transformations which allow new addresses to be contiguously accessed for one of the mentioned memory access types [10].

a memory centric architecture, with each memory module holding HBN and HCN data structures, that is coupled to a core that iteratively computes the Lrmn and Lqnm values. The main characteristics of such distributed memory multicore architectures ideally are:
• a high number of cores, or processing units, which depends of the specifications and configuration, but should individually support SIMD processing;
• the cores do not need to support a large and complex instructions set; in fact, they can be very simple and perform appropriately with a few arithmetic and logic instructions; they should also support instructions for wide memory accesses;
• the use of global main memory, typically slow, should be avoided;
• each core should have a tightly coupled fast local memory; the minimum size of the scratchpad local memory should be carefully considered in order to be able of storing the data structures necessary to support HBN and HCN data structures and also the program;
• the sharing/competition of/for resources (memory and others) should be avoided;
• a bus is required with a minimum bandwidth to support high speed data I/O for multiple codewords in simultaneous;
• hardware techniques, such as DMA controllers, have to be provided to autonomously manage data transfers between adjacent cores/memories.
The cores should include a minimum instruction set, which depends on the Instruction Set Architecture (ISA) of the algorithm adopted for LDPC decoding. For the SPA should be considered, at least, floating-point instructions such as add, mul and div, and load, store and brcond instructions; while for the MSA, the instruction set should accommodate instructions

for fixed-point arithmetic and logic operations, such as abs, min, add, add/sub with saturation, xor and also load, store, and brcond. Ideally, it should also include specialized intrinsic instructions for SIMD computation able of simultaneously performing operations in parallel on 8, 16, or 32-bit operands for wide ALUs (e.g. 128-bit).

V. EXPERIMENTAL RESULTS
The architecture in figure 5 that has several similarities with the Cell/B.E. [22], namely the distributed memory multicore structures built around a common bus. However, the ISA can be simpler, by tailoring it to LDPC decoding, data I/O buses are also simpler and, most importantly, the size of its local scratchpad memory is not limited to 256 KByte as in the Cell. Consequently, due to the existing similarities, the Cell/B.E. platform was adopted for validating and assessing the proposed embedded architecture.
The Cell/B.E. is an heterogeneous processor [22], [24] composed by one main 64-bit PowerPC Processor Element (PPE) that communicates with eight Synergistic Processor Elements (SPEs) each having 256 KByte of scratchpad local memory for data and program, and a 128-bit wide vectorized architecture. Data transfers between the PPE (that accesses main memory) and SPEs are performed by using DMA controllers. The Memory Interface Controller (MIC) controls the access to the main memory and the peak value for its bandwidth nearly reaches 4.2 GByte/s [22].

Differently from shared memory architectures, such as x86 multicores or GPUs, in the Cell/B.E. data is explicitly loaded from main memory into the local memory of a SPE and viceversa, allowing each SPE to efficiently exploit data locality. This type of solutions leads to better results for codes represented by larger matrices H, because the communication time between the CPU and the GPU relatively decreases in comparison with the computation time.

When compared against programmable solutions, the main disadvantages of ASICs are the use of high non-recurring engineering (NRE) costs and inflexibility. Costs associated with NRE have been increasing exponentially along with transistor capacity. Also, ASIC solutions are non-reconfigurable and they can only support a predefined set of LDPC codes. On the other hand, GPUs are flexible and reconfigurable, but have higher power consumption.

VII. CONCLUSIONS

Nowadays the research on new architectures for LDPC decoding is a very active area. This paper addresses the main computational characteristics of LDPC decoding and proposes programmable multicore architectures for developing embedded real-time processing systems. These architectures are designed to exploit data parallelism by decoding multiple codewords in parallel. The architecture adopts a distributed memory approach and a single

A compact representation of the Tanner graph is proposed, which allows it to be stored locally in the memory modules associated with each core. We analyze the main properties of these architectures required to achieve a given throughput, namely the minimum number of cores required, the complexity of the instruction set architecture, and the minimum bus bandwidth. The proposed architecture can be easily prototyped in FPGAs or implemented in ASICs.
