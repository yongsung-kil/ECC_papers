Abstract-As one of the candidates of cellular phone TV standard, the DAB-based CDMB adopts convolutional codes as its channel coding. In this article, we present a new sum-product decoding algorithm, where parity-check matrix is constructed under the circumstance that the sending encoder is unaltered or partially modified. It shows that our method obtain about 1dB gain higher than the traditional Viterbi algorithm in high SNR regime and with the same quantified bits. It is of importance in performance enhancement for the convolutional codes based system in practice.
I. INTRODUCTION
IN 1962, Gallager proposed low-density parity (LDPC) codes [1] for the first time. However, it was not rediscovered by Mackay and others until the end of the century. They found the codes can approach the Shannon limit [2] and be comparable with turbo codes. Therefore, LDPC codes have attracted much attention in the world.
LDPC codes are linear block codes and generally decoded with sum-product (SP) algorithm [3]. When code length tends to be infinite and there is no cycle in tanner graph, the SP algorithm is equivalent to the maximum likelihood algorithm.
As another kind of classic channel code, in 1955, convolutional codes [4] are proposed by Elias and have been used in many communication systems successfully. A convolutional code is a finite state machine, so its decoder is a maximum-likelihood sequence estimator. Generally, convolutional codes are decoded with Viterbi algorithm [5], but for these codes with large code length, the sequential decoding algorithm or the Fano algorithm is usually adopted.
Recently, several solutions are proposed to be the Chinese mobile television standard. Because of using LOPC codes as channel coding, China Mobile Multimedia Broadcasting (CMMB) standard has better performance than the others. Whereas another candidate - China Digital Multimedia Broadcasting (CDMB) solution is based on Digital Audio Broadcasting (DAB) standard [6], therefore, it uses convolutional codes as its channel coding, following the way of the DAB system. So improving the performance of channel coding effectively is necessary. There are some relationships between LDPC codes and convolution codes being addressed recently [7] - [9], but the COMB system has been modified too much.

III. PARITY-CHECK MATRIX BASED ON THE COMB
CONVOLUTIONAL ENCODER

A linear block codeword is composed of the information bits and the check bits, respectively. The OAB convolutional encoder has a constraint length of 7 and the generator polynomial are 133, 171, 145 and 133, respectively, which means the bits of the codeword are all check bits and there is no initial information bit. In view of the first bit and the last one outputed from the OAB convolutional encoder being same, we modifY the first generator polynomials to 100 for the COMB standard. Therefore, every 4 bits outputed from the convolutional encoder, the first bit is the information bit itself and the last three bits are the check bits.

B. Parity-check matrix ofpunctured code
In the COMB system, the punctured convolutional coding is applied based on the mother codeword whose rate is 1/4. The bits of the serial mother codeword are split into consecutive blocks of 128 bits. Each block is divided into four consecutive sub-blocks of 32 bits. All sub-blocks belonging to the same block are punctured using the same puncturing vector. There are 24 puncturing vectors, producing the codeword with the rate from 8/9 to 1/4
In the puncturing process, these bits which are not transmitted are all check bits and they are independent with each other. Therefore, as for the parity-check matrix mentioned above, their corresponding rows should be deleted directly.

C. Optimization of the parity-check matrix

According to above two sections, a parity-check matrix used for decoding a COMB convolutional code can be obtained. However, in this tanner graph, each variable node mapped from the check bit has a degree of I. Correspondingly, in the parity-check matrix, there are many columns with column weight of 1. When decoding, these variable nodes only send initial LLR to their neighbors in each iteration, which degrade the performance of the SP algorithm dramatically. Therefore, it is very important to optimize the degrees of each variable node for improving the decoding performance.

IV. USING THE SP ALGORITHM FOR THE DAB
CONVOLUTIONAL CODES
In the above mentioned CDMB solution, the polynomial for the first check bit is modified from 133 to 100, which results in the information bits for encoding being comprised in outputting sequence from the encoder. After decoded with the SP algorithm, these bits can be extracted as the decoding result directly. However, many solutions based on convolutional codes have been promulgated. It is inconvenient to make further adjustment for their transmitter.
For a convolutional code, vectors for encoding are split into segment of k bits. The generated check bits are not only involved with current segment, but also involved with
preceding N -1 segments (where N is constraint length). In the DAB standard, k and N are I and 7, respectively. In
other words, each information bit is used to generate multiple check bits. Therefore, any check bit can be obtained with other check bits. According to the method mentioned above, the parity-check matrix can be generated. However, the code words do not comprise the original information bits themselves. So, the original information bits used for encoding can not be recovered from the decoded codeword directly. In the other words, the SP algorithm is only used to check the encoded code words of convolutional codes. In order to recover the information bits, the Viterbi algorithm should be adopting to complete the decoding. Thus, it is inevitable to increase the decoding complexity.

VI. CONCLUSION
In this paper, taking convolutional codes used in the CDMB system and the DAB system for example, we propose an approach of decoding convolutional codes with SP algorithm based on the construction of parity-check matrix. The best performance can be obtained with both SP and Viterbi algorithm being employed for the original convolutional codes. But with some convolutional encoder modification, the modified convolutional codes can be decoded with SP algorithm alone. Simulation shows that with the same quantified bits, about 1dB gain is obtained in high SNR regime in comparison with the traditional Viterbi algorithm.
