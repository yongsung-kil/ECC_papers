Modified Progressive Edge-Growth Algorithm based LDPC
Coded-Modulation for 400 Gb/s Optical Transport
Ivan B. Djordjevic1, Hussam G. Batshon1, Lei Xu2, and Ting Wang2
1University of Arizona, Department of Electrical and Computer Engineering, Tucson, AZ 85721, USA; E-mail: ivan@email.arizona.edu 2NEC Laboratories America, Princeton, NJ 08540, USA

Abstract— We propose a polarization-multiplexed iterative polar quantization (IPQ)-based LDPC-coded-modulation scheme suitable for beyond 400 Gb/s serial optical transmission. The component LDPC codes are designed using modified progressive edge-growth (MPEG) algorithm. The proposed scheme employs 32-IPQ-based-modulation.
I. INTRODUCTION
In the recent years, with the rapid growth of data-centric services and the general deployment of broadband access networks, there have been strong demand driving the dense wavelength division multiplexing (DWDM) network upgrade from 10 Gb/s per channel to more spectrally-efficient 40 Gb/s or 100 Gb/s per wavelength channel, and beyond [1]. The 100 Gb/s Ethernet (100GbE) is currently under standardization for both local area networks (LANs) and wide area networks (WANs) [1]. The 400 Gb/s and 1 Tb/s are regarded to be the next steps after 100 Gb/s and have started attracting research community interests.
In order to achieve beyond 400 Gb/s per wavelength optical transmission using commercially available equipment operating at 40 Giga symbols/s (40 GS/s), we propose the use of iterative polarization quantization (IPQ)-based modulation scheme with component LDPC codes designed based on modified progressive edge-growth (MPEG) algorithm.
II. POLMUX IPQ MPEG-LDPC-CODED-MODULATION
The polarization-multiplexed (PolMUX) IPQ-based coded modulation scheme suitable for beyond 400 Gb/s per wavelength optical transmission is shown in Fig. 1. The mx+my (index x (y) corresponds to x-(y-) polarization) independent data streams are encoded using different LDPC codes of code rates Ri=Ki/N (i∈{x,y}), where Kx (Ky) denotes the number of information bits used in the binary LDPC code corresponding to x- (y-) polarization, and N denotes the codeword length. The mx (my) input bit streams from mx (my) different information sources, pass through identical encoders that use LDPC codes with code rate Rx (Ry) designed using a modified PEG algorithm, which will be described later in this section. The outputs of the encoders are then bit-interleaved by an mx×N (my×N) bit-interleaver where the sequences are written row-wise and read column-wise. The mapper maps each mx (my) bits, taken from the interleaver, into a 2mx -ary ( 2my -ary) IPQ signal constellation point based on a lookup table (LUT), which is for M=32 given in TABLE I. The corresponding constellation diagram is shown in Fig. 2, and it is obtained by using the algorithm we described in [2]. Given this general description of proposed coded-

modulation scheme suitable for beyond 400 Gb/s optical

transmission, we turn our attention to the design of LDPC

codes using MPEG algorithm. The original PEG algorithm for

design of (n,k) LDPC codes can be formulated as follows [3]:

given (n-k), n and variable-node degree distribution, design an (n-k)×n parity-check matrix with largest possible girth g. The

MPEG algorithm, suitable for design of LDPC codes for

optical communications, can be formulated as follows: given (n-k) and bit-node degree distribution, design an (n-k)×n parity-check matrix with girth g≥8 and code rate R=k/n≥0.8.

We are turning our attention to the BER performance

evaluation of MPEG algorithm based LDPC codes. The

results of simulations for the AWGN channel model are given

in Fig. 3, where we compare the MPEG-based LDPC codes

against RS codes, concatenated RS codes, turbo-product codes

(TPCs), and QC-LDPC codes. The MPEG-based irregular

LDPC(34665,27734) code outperforms, observed at BER of 10-9, TPC of similar length (BCH(128,113)×BCH(256,239))

by 1 dB, quasi-cyclic (QC)-LDPC(16935,13550) code,

designed as explained in [1] by 0.2 dB. Compared to QC-

LDPC in addition to coding gain improvement, which can be

considered small, MPEG-based LDPC code design can push

down the error floor and offers much more flexibility in

designing of LDPC codes with desired code rate and length.

The MPEG-based irregular LDPC(34665,27734) code

outperforms the concatenation RS(255,239)+RS(255,223) (of

rate 0.82) by 3.45 dB, and RS(255,239) by 4.8 dB, both at BER of 10-9. The net effective coding gain (NECG) for PEGbased irregular LDPC(34665,27734) code at BER of 10-9 is 9.66 dB. Expected NECG at BER of 10-12 is 11.05 dB. By

using longer MPEG-based LDPC(345165,276132) code, the NECG (at BER of 10-12) can be improved to 11.15 dB.

The results of simulations of proposed PolMUX MPEG-

based LDPC-coded IPQ scheme with 400 Gb/s in aggregate data rate are shown in Fig. 4. The NECG at BER of 10-6 is found to be 9.2 dB. The expected NECG at BER of 10-9 is

about 12.4 dB. The proposed scheme, therefore, represents an

excellent candidate for 400G Ethernet technologies.

III. CONCLUSION
We proposed the PolMUX IPQ coded-modulation scheme suitable for beyond 400 Gb/s serial optical transmission with component LDPC codes designed using the MPEG algorithm. The 32-IPQ coded-modulation scheme provides the NECG of 12.4 at BER of 10-9. The MPEG-based LDPC code design provides more flexibility compared to QC code design.
