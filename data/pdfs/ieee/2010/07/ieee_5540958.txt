A Fully-Overlapped Multi-Mode QC-LDPC Decoder Architecture for Mobile WiMAX Applications

Bo Xiang, Dan Bao, Shuangqu Huang, and Xiaoyang Zeng, Member, IEEE State Key Lab. of ASIC and System, Fudan University, Shanghai, China

Abstract—A fully-overlapped multi-mode QC-LDPC decoder architecture, adopting improved TDMP algorithm, is presented in this paper. With symmetrical four-stage pipelining, block column and row permutations, nonzero sub-matrix reordering, sum memory odd-even partition, and read-write bypass, two phases are fully overlapped and each phase scans nonzero submatrices one by one in block row-wise order without access conflicts to sum memories. The sum memories store not only variable node sums but also prior messages. In this case, it saves an additional FIFO of 13 440 bits. The decoder attains 248-287 Mb/s at 150 MHz and 15 iterations.
I. INTRODUCTION
The low-density parity-check (LDPC) code [1] has become one of the most important forward error-correcting (FEC) codes in digital broadcast and communication systems since it was rediscovered [2] that it has remarkable error-correcting performance. Due to linear encoding complexity and flexible partially-parallel decoder architecture, the quasi-cyclic LDPC (QC-LDPC) code [3] is much more practical than the pseudorandom LDPC code [1] and is widely applied in the systems such as WLAN (IEEE 802.11n), WiMAX (IEEE 802.16e), DVB-S2, CMMB, digital terrestrial/television multimedia broadcasting (DTMB) system, magnetic storage system, and maybe next generation mobile communication systems.
The QC-LDPC code is defined by a sparse parity-check matrix H, which is made up of M × N sub-matrices. Each sub-matrix is a b × b either all-zero or circulant permutated square matrix. Here size b is also called expansion factor. The total number of nonzero sub-matrices is t. Therefore, a base matrix Hb with position indexes and offsets of all nonzero sub-matrices can uniquely identify H. An example of QCLDPC codes in the WiMAX system is given in Fig. 1. The Tanner graph is another efficient way to present the code structure, which is shown in Fig. 2, where Wr and Wc denote row weight and column weight, respectively. There are t × b bidirectional interconnection paths between M × b check nodes and N × b variable nodes. Table I lists the parameters (M, N, b, t) of some modes in the WiMAX system.
The iterative decoding algorithm brings much better errorcorrecting performance. Due to high computation complexity of original two-phase message-passing (TPMP) algorithm, many simplified decoding algorithms come forth: 1) min-sum

algorithm [4]; 2) offset-based min-sum algorithm [5]; and 3) normalized min-sum algorithm [6]. The first has the lowest decoding complexity, but suffers about 1.0 dB performance loss. The error-correcting performance of the second is a little worse than the third. The second and the third are much more realizable since it can largely reduce complexity and memory bits for substantial intermediate messages. To speed up the convergence rate further, the turbo-decoding message-passing (TDMP) algorithm is introduced in [7]. In TDMP algorithm, each block row in Hb is treated as a constituent code, and the messages not only propagate between different iterations but also propagate between different constituent codes.
Making tradeoffs between throughput and area, most of the QC-LDPC decoders are implemented with partially-parallel architectures. Generally, there are two normal categories of partially-parallel decoder architectures. For [8], [9], M check node updating (CNU) processors and N variable node updating (VNU) processors work in parallel. They dispense with permutation networks, but need large numbers of small memories corresponding to each nonzero sub-matrix in H. For [10], [11], b CNU processors and b VNU processors run in parallel. They are much more configurable with permutation networks, and the total number of memories can be largely reduced. For the decoder [12] with M × b CNU processors and b VNU processors and the decoder [13] with b CNU processors and N × b VNU processors, large numbers of register arrays instead of memories are required and not area-efficient, especially for large parity-check matrices.
To improve throughput and hardware utilization further, a fully-overlapped multi-mode QC-LDPC decoder architecture is proposed in this paper. It adopts the improved TDMP algorithm. With symmetrical four-stage pipelining, block column and row permutations, nonzero sub-matrix reordering, sum memory odd-even partition, and read-write bypass, each phase scans nonzero sub-matrices one by one in the block row-wise order without read-write access conflicts to sum memories. Two phases are fully overlapped with each other. Section I is the introduction. Section II presents the improved decoding algorithm and decoding schedule. The proposed decoder architecture is presented in Section III in detail. Section IV shows the chip layout and simulation results. Finally, Section V concludes the paper.

II. DECODING ALGORITHM AND DECODING SCHEDULE
A. Decoding Algorithm
For TDMP decoding algorithm, M consecutive block rows in base matrix Hb are treated as M cascade constituent codes (sub-iterations). The updated messages of one constituent code are fed into the next constituent code directly. It requires 50% less iterations to converge at moderate to high SNRs [7] than the original TPMP algorithm.
The soft-input soft-output (SISO) decoding engine is the key function for each constituent code. It accepts the prior messages, which are sums of all extrinsic messages pertaining to the other M-1 constituent codes. The updated messages are stored as extrinsic messages pertaining to current constituent code. At the end of each sub-iteration, the variable node sums, which are fed into the next constituent code, are calculated by adding the updated extrinsic messages to the prior messages.

B. Decoding Schedule
The partially-parallel decoder architectures of the second category update each block row in Hb serially in 2 × Wr clock cycles with b concurrent CNU processors and b concurrent VNU processors. Fig. 3 (a) demonstrates the partial base parity-check matrix of code (2304, 1152) in the WiMAX system. The partially-parallel decoders [7], [11], [13], [16], and [18] perform continuous row-wise scanning with another FIFO, which stores the prior messages pertaining to one block row. In this case, its size is linearly proportional to row weight Wr, which is shown in Fig. 3 (b). For the multi-rate irregular QC-LDPC decoders, the utilization rate of the FIFO is much low for those rows with small row weights. However, the FIFO is unnecessary in our proposed fullyoverlapped decoder architecture as shown in Fig. 3 (c).
Sum memories are partitioned into two independent banks, which store not only variable node sums but also temporary prior messages. With seven-bit quantization, it can save 20 × 96 × 7 = 13 440 memory bits. Sum memory partition is the prerequisite for the proposed fully-overlapped decoder architecture without FIFO. To solve large access conflicts to sum RAMs, the base matrix Hb should be preprocessed with the following three proposed steps: 1) block column permutation; 2) block row permutation; and 3) nonzero submatrix reordering.
Block column permutation makes the number of nonzero sub-matrices in the block odd column approximately equal to the number in the block even column for each block row

To improve decoding throughputs and help solving access conflicts to sum RAMs, the decoder adopts a symmetrical four-stage pipelining. As shown in Fig. 4, the CNU phase is pipelined in three stages: 1) reading variable node sums and check node message components; 2) permutating sums, recovering extrinsic messages, and calculating prior messages; and 3) writing prior messages and updating message components. In the same way, the VNU phase is pipelined in three stages: 1) reading prior messages and recovering updated extrinsic messages; 2) updating variable node sums; and 3) writing updated sums. For the same memory entry in each phase, writing operation occurs two clock cycles later than reading operation. The start stage and the end stage for both phases are read-write accesses to memories. Using twoport sum RAMs, the start stage of the CNU phase are overlapped with the end stage of the VNU phase, and vice versa. The symmetrical pipelining indicates that there are no memory writing conflicts if memory reading conflicts are free.

Distribution denotes the physical distribution (including block column and row indexes) of nonzero sub-matrices, and correlation denotes the number of block columns in which both consecutive block rows have nonzero sub-matrices. To solve reading conflicts to sum RAMs, block columns and block rows of the base matrix Hb should be permutated to feature the following two points firstly:
1) The number of nonzero sub-matrices in the block odd columns approximately equals the number in the block even columns for each block row ( Nodd − Neven = 0 | 1 ); and
2) The sum of all correlations between two consecutive block rows including between the last block row and the first block row is as less as possible.

For the original base matrix in Fig. 5 (a), the matrix after some block column and row permutations is shown in Fig. 5 (b), where the numbers in the brackets are the original positions (block column and row indexes). The mapping information of block column and row permutations is stored in ROMs.
After block column and row permutations, the scanning orders of nonzero sub-matrices for both phases should be rearranged further. Fig. 6 presents the rearranged updating orders for both phases. The upper (white) row corresponds to the CNU phase, and the lower (grey) row corresponds to the VNU phase. Each phase incessantly scans nonzero submatrices one by one in the block row-wise order. However, the updating orders for two phases may be different. Here the integers out of brackets are block column indexes, and the integers in brackets are remainders of block column indexes divided by two. Two remainders at every clock cycle are different with each other (0 and 1), so the sum memories can be easily partitioned to supply two independent ports without read-write access conflicts for the proposed fully-overlapped decoder architecture.

D. Configurable Permutation Network
Two 96-way logarithmic shifters constitute a configurable permutation network ( ≤ 96 ), which is compatible for all 114 modes in the WiMAX system. Two shifters are cascaded together as shown in Fig. 8, where b denotes the expansion factor. Two parameters offset and expansion factor cut the input sequence of length 96 into three pieces A, B, and C. The original sequence gets through the first shifter with an offset of length A, and then gets through the second shifter with an offset of length C. The length A is less or equal to the expansion factor b ( b ≤ 96 ), so the first logarithmic shifter needs seven cascaded stages. However, the length C in the WiMAX system is equal to 4 × j, where j is an integer from 0 to 18. Therefore, the second logarithmic shifter just needs five stages to shift length C. In this case, the decoder can totally save about 14.3% ( 2 / (2 × 7) ) hardware resources for
permutation network. The final output sequence is assembled with part B of the first shifted sequence and part A of the second shifted sequence.

Compared with the float TPMP/TDMP algorithms, seven-bit quantization brings about 0.4 dB performance loss. The matrix permutation and nonzero sub-matrix reordering has no effects on performance. Overall, improved TDMP algorithm at 15 iterations satisfies the performance requirements of the WiMAX system.
The proposed fully-overlapped decoder is synthesized and designed physically in SMIC 0.13 μm CMOS technology. Fig. 10 shows its layout view, including 72 input/output (I/O) pins with 48 signal pins and 24 power/ground (P/G) pins and 44 memories. The memory numbers of each type are shown in brackets. All memories are placed around the chip to balance the power dissipation. The core area is about 1.57 × 1.57 mm2, and the chip area is about 2.03 × 2.03 mm2. The chip physical utilization is about 80%. Table III summarizes different QC-LDPC decoders. The proposed decoder is compatible for all 114 modes in the WiMAX system, and also configurable for any other QC-LDPC codes (M, N, b, t). It only has 410 000 equivalent logic gates including memories, and takes 78-90 clock cycles per iteration. Therefore, the proposed decoder attains 248-287 Mb/s at 150 MHz and 15 iterations, and consumes 282 mW with power efficiency of 66 pJ per bit per iteration by post-layout simulation.

V. CONCLUSION
This paper proposes a fully-overlapped multi-mode QCLDPC decoder architecture for the WiMAX system, adopting improved TDMP decoding algorithm. With five techniques of symmetrical four-stage pipelining, block column and row permutations, nonzero sub-matrix reordering, sum memory odd-even partition, and read-write bypass, two phases are fully overlapped and each phase scans nonzero sub-matrices one by one in the block row-wise order without access conflicts to sum RAMs, which store not only variable node sums but also prior messages.
