Abstract - This paper is a continuation of the work in [IS] 
wherein a novel code-construction method was proposed for 
constructing (3, k) regular LDPC codes. In this paper, the (3, k) 
construction technique is generalized to construct (5, k) regular 
code. The basic design of the proposed code-construction utilizes 
deterministic base matrices for information part and parity part 
in three stages of code-construction to achieve no rank-deficiency 
of H, no girth of 4-cycles, non-singular parity part (Hpar) and low 
encoding complexity. A typical performance of proposed (5, k) 
regular LDPC codes has been shown. 
Keywords: low density parity check (LDPC); regular LDPC; 
rank-deficiency of H; low encoding complexity; non-singular parity 
part.

INTRODUCTION 
A code-construction in low density parity check (LDPC) 
codes begins with techniques to build its parity check matrix 
(fl). If H has constant number of } ones in each column, 
constant number of k ones in each row and zeros elsewhere, 
this H is recognized as (j, k) regular LDPC codes. The 
invention of regular LDPC codes was done by Gallager in 
1960's. In Gallager's regular LDPC codes, there are at least 
(j - 1) rows of H which are linearly dependent [1]. Other 
structured construction of regular LDPC codes that also has 
some dependent rows in H is algebraic

codes as one example of algebraic codes have at least (j -1) 
dependent rows in H [5], [14]. A full-rank of H in the design 
of regular LDPC codes is important to get an encoded code. 
Moreover, encoding complexity of regular LDPC code is in 
the order of 0 (N)-) [1]. QC LDPC codes solve high encoding 
complexity in regular LDPC codes since they have sufficient 
structure to allow simple encoding with simple shift registers 
[5], [8]. Unfortunately, this type of encoding is only useful for 
QC and cyclic codes.

A typical performance of proposed (5, k) regular LDPC 
codes is given in Fig. 8 and Fig. 9. Since there are no other 
published results of (5, k) regular LDPC codes with the 
same code length and code rate, the proposed codes are 
compared only with Shannon limit - a yardstick used as a 
reference for a coded system at a given rate R for BER of 
10.6 [1 7]. The proposed (5, k) regul ar LDPC codes in this 
paper are encoded by H using Richardson- Urbanke method 
[16]. Examples of (5, 16) and (5, 1 7) regular LDPC code are 
taken since the proposed H in this code rate gives non­
singular parity part and avoids girth of 4-cycles using the 
above methodology without an extra algorithm. Based on 
BER result of Fig.8, the proposed (5, 16) regular LDPC 
code performs 2.38 dB from Shannon limit at BER 10.6 
when code rate R = 0.69 and code length N 
= 3840. 
Encoding complexity of the proposed (5, 16) regular LDPC 
code is in the order of 0 (1.104 N). 
Meanwhile in Fig. 9, the proposed (5, 1 7) regular LDPC 
code performs 2.33 dB from Shannon limit at BER 10.6 
when code rate R = 0. 706 and code length N 
= 4080. 
Encoding complexity of the proposed (5, 1 7) regular LDPC 
code is in the order of 0 (1.098 N). In the value of SNR of 
2.9 dB, the proposed (5, 1 7) regular LDPC code is not able 
to outperform theoretical un-coded BPSK s ince the coding 
threshold for this code is 3 dB. This is a common situation 
in error correcting codes [1 7]. Therefore, in order to get a 
better performance for coded system, we should apply an 
SNR above its coding threshold.

construction method to construct (5, k) regular LDPC codes
