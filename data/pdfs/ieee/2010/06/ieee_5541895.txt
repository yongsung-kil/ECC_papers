Construction of Quasi Cyclic LDPC with ACE constraint 
Jilong Li, Na Di, Ming Yang, Zhiyong Wu 
Wireless Technology Institute 
Academy of Broadcasting Science 
Beijing, P.R.China 
e-mail: [lijilong.dina.yangming.wuzhiyong]@abs.ac.cn 
Abstract-In this paper, LDPC code structure and decoder 
implementation is reviewed. The effect of graph connectivity 
on LDPC performance is analyzed. 
The metric of ACE 
(approximate cycle EMD) in code construction avoids small 
trapping sets by emphasizing the connectivity. A construction 
algorithm with ACE constraint is proposed which maximize 
the girth and stopping sets in a best effort way. Simulation 
results justify the efficiency of the algorithm. 
Keywords: QC-LDPC ;ACE; stopping set; 
I. 
INTRODUCTION 
Low-Density Parity Check (LDPC) codes can achieve 
good performance. Using LDPC codes with iterative 
decoding algorithm, it is possible to reach very close to the 
Shannon limit. In fact, it was shown that LDPC codes can 
reach within 0.0045 dB of the Shannon limit (for very long 
block lengths).[l] 
However, delay, memory size, processing complexity 
and other requirements limit the maximum LDPC code 
length in practical systems. Thus short and medium length 
irregular LDPC code construction is currently an attractive 
research area. Density evolution can be exploited to derive 
optimal node distribution with the aim of minimizing the 
SNR threshold. But the short or medium length LDPC codes, 
randomly constructed with density evolution, lead to rather 
undesirable performance. Another design criterion for LDPC 
is girth (shortest cycle length) conditioning, due to the fact 
that small cycles will degrade the performance of BP 
decoding. However, for short block length codes, girth 
conditioning will be particularly difficult due to the average 
cycle length being only logarithmic with the block length. 
The other means of improving LDPC codes' performance 
is to design LDPC code graphs free of small stopping set that 
are proved to introduce problems for the iterative BP decoder 
on binary erasure channel. And similar effect is proved on 
additive white gaussian noise channels.[2]

reviews LDPC code structure and decoder implementation,

stopping set, EMD (extrinsic message degree) and ACE, 
then explains their influence on code performance. Section 4 
presents the design algorithm of QC-LDPC based on ACE. 
Section 5 gives simulation results. And paper is concluded in 
Section 6.

II. 
INTRODUCTION OF QC-LDPC 
Random LDPC codes provide excellent coding gains, but 
decoding complexity is too high for practical implementation 
due to the random structure of their parity-check matrices. 
QC-LDPC codes have implementation advantage over 
random LDPC codes. The QC-LDPC code has parity check 
matrix that is composed of cyclically shifted identity 
matrices. Each cyclically shifted identity matrix is made 
from identity matrix, which cyclically shift right certain steps 
according to its offset. Structured connections between check 
and variable nodes can be processed in parallel mode, and 
generally reduce hardware complexity. The memory address 
generation can be simplified and the amount of memory 
required is compressed. The cost of encoder and decoder is 
reduced. 
There are three ways by which LDPC decoder can be 
implemented: serial decoder, full parallel decoder and semi­
parallel decoder. Serial decoders require less hardware but 
deliver low decoding throughputs. In full parallel decoders, 
row and column processors are directly connected to each 
other according to the Tanner graph of the corresponding 
parity check matrix. Full parallel decoders can provide very 
high throughputs, but require the most hardware functional 
units. Semi-parallel decoders perform row and column 
operations partially in parallel and deliver higher throughput 
than serial decoders. To balance the trade-off between area 
and time, the best strategy is semi-parallel design. QC-LDPC 
codes 
are 
well 
suited 
for 
semi-parallel 
decoder 
implementations, and can deliver higher throughput and 
reduce processing delay.

III. 
LDPC CODE CONSTRUCTION ANALYSIS 
There are several aspects that constraint the performance 
of the LDPC codes. First, longer code length provides better 
performance. Second, degree distribution of node needs to be 
optimized. Third, short cycle in the code should be reduced. 
Finally, connectivity of the tanner graph should be enlarged. 
The code length is restricted with the application, and the 
node degree is optimized with density evolution. In the 
following, the rest two aspects are analyzed. 
LDPC 
codes 
are 
decoded 
with 
message 
passing 
algorithms (MPA) which are iterative algorithms. At each 
round of the iterative algorithms, messages are passed from 
message nodes to check nodes and from check nodes back to 
message nodes. The MP A assumes that the messages passed 
are statistically independent throughout the decoding process. 
The appearance of short cycles makes this assumption 
cannot stand up. The messages will loop back on themselves 
in short cycles. So in LDPC code construction, cycle is 
designed as long as possible. The progressive-edge-growth 
(PEG) 
is one of the 
most powerful algorithms 
for 
constructing LDPC codes with large girth [6]. This algorithm 
maximizes the girth of the Tanner graph at variable nodes 
with a greedy design. For LDPC codes of short or medium 
length, it is difficult to exclude all short cycles, so the 
performance of PEG algorithm is decreased. 
Conventional girth conditioning techniques increase the 
girth of LDPC codes. We notice that not only the girth but 
also the connectivity of cycles plays important roles in error 
performance. Longer cycles with poor graph connectivity 
may be more harmful for code performance than short cycles 
with good graph connectivity. Incorrectly received variable 
node in a cycle with poor connectivity cannot be revised by 
messages from its connected check variables, thus wrong 
messages will be spread in MP A and code cannot be 
correctly decoded. This is how the nodes with poor 
connectivity, which forms trapping sets, decrease the 
performance. Trapping set is a structural property of tanner 
graphs, and is composed of a set of variable nodes of which 
every check node neighbor is connected to this set at least 
twice.[3] Researches show that the performance of LDPC 
codes, especially for short or medium length codes, depends 
on size and distribution of trapping sets. 
Since every stopping set possesses a cycle, it is possible 
to influence the size of the stopping sets by influencing 
cycles. In the traditional approach by removing all short 
cycles the small stopping sets can be removed. However, for 
short or medium length LDPC codes this approach is not 
effective enough due to limit of code length. So the obtained 
girths are short and small stopping sets are left. To improve 
performance, we should find an effective way to remove 
small stopping sets. 
According to the connectivity of cycles, we can suppress 
small stopping sets by improving the edge expanding 
properties of small cycles. Based on this observation, an 
appropriate metric, named the extrinsic message degree 
(EMD) of a cycle, is proposed to measure the level of 
connectivity of a cycle with the rest of the graph via its 
variable nodes. A set of variable nodes with large EMD 
require additional concatenation of nodes. High EMD by 
virtue of a level of graph connectivity statistically surpasses 
cycles with few variable nodes which form small stopping 
sets. 
However, it is not an easy task to find the EMD of cycles 
in the code graph, since it requires additional steps to 
determine if the edge is within other cycles in the stopping 
sets. If this difference is neglected and both kinds of edges 
are taken into account in the cycle metric, the simplified 
version of the EMD metric, named the approximate cycle 
EMD (ACE) metric, is obtained. ACE is an upper bound on 
EMD. To simplify the algorithm to suppress small stopping 
set, ACE instead of EMD can be used. Low-ACE cycles 
have less connection with the rest of the graph. These poorly 
connected sub-graphs are vulnerable to channel noise 
because they do not have sufficient impinging message owe 
to correct errors. We thus target the elimination of small 
cycles with low ACE.[2] 
IV. 
CONSTRUCTION BASED ON ACE 
We present a construction algorithm for QC-LDPC codes 
by 
given 
arbitrary 
parameter. 
The 
model 
matrix 
is 
constructed and its nonzero entry follows optimal degree 
evolution. For each nonzero entry of the model matrix, girth 
condition and variable node connectivity are checked and 
shift-value is updated by some condition. Based on the 
criterion of the previous section, our construction procedure 
for QC-LDPC codes is summarized as follows: 
Step 1) Determine the parameter m, n, p for desired code 
rate and code length. 
Step 2) Use density evolution to generate optimal degree 
distribution for desired code rate and modify the distribution 
to size m, n. 
Step 3) Determine the position and shift value of nonzero 
entry of the model matrix. The according bit node is 
connected to all possible parity node, and shift value is 
selected within [0, p -1]. The position of the nonzero entry 
and the shift value is selected to maximize girth and ACE.

Step 4) Add the optimal shift value to the selected 
position in the model matrix and repeat step 3), step 4) until 
all nonzero elements are determined. 
For given degree distribution of a Tanner graph, the 
optimal connection and shift value is searched in step 3. The 
corresponding girth and connectivity value is calculated, then 
compared with the previous shift value. If two conditions are 
satisfied, shift value is updated by current value. First 
condition is that the current cycle is larger than previous 
cycle. Second condition is that the current connectivity is not 
smaller than previous connectivity. 
The way by which the updating condition is used is 
described detailly in the folowing. If the current shift value's 
girth and ACE are both greater than these of the previous', 
update the previous shift value with the current shift value. If 
the current girth is smaller than the previous girth, the 
previous shift value is reserved. If the girth is same with the 
previous girth, then compare the sum of girth in current 
tanner graph, and shift value is updated if the current sum is 
lager than the pervious sum. 
Since this method consider not only the cycle of sub-block 
which will be added but also the cycle of sub-block which 
already inserted in graph, resultant parity check matrix have 
large average local girth. By consider all possible shift value 
and connection, this method ensures the girth and ACE are 
maximized. Since all sub-block are circulant permutation 
matrices or the zero matrix, the length of cycle of the bits in 
the same block are all same. So the algorithm's complexity is 
not high, for girth and ACE are calculated in model matrix 
whose size is p2 smaller than parity matrix. 
V. 
SIMULATION AND ANALYSIS 
We generated two short-length irregular LDPC codes 
(576, 288) and (lO56, 528) with PEG and ACE methods 
respectively using the same degree distributions, and the 
code rates both are 1/2. The circulant size p is 24 and 44 
respectively for the two codes. In the computer simulation, 
we assume that the LDPC codes are modulated by BPSK and 
transmitted over an A WGN channel. The iterative BP 
algorithm which allows a maximum number of 20 decoding 
iterations is used for decoding. 
The simulation results are shown in Fig, 1 and 2. It can 
be seen the error floor is lowered for (576, 288) codes with 
the ACE method in Fig. 1. And in Fig. 2 the waterfall region 
is steep with the ACE method for (lO56, 528) codes and 
target BER can be achieved with lower EblNo. In the two 
figures, it can be seen that 0.2dB decrease can be achieved at 
high EbNo region. 
We also generated two 2/3 code rate LDPC codes (576, 
384) and (lO56, 704) with PEG and ACE methods. The 
simulation condition is the same with the previous one. In 
Fig.3 and FigA, we can hardly see the error floor, but better 
performance is observed in high EbNo region with the ACE 
method. About a 0.2dB decrease is achieved in Fig.3 when 
BER is lO·5 and more than O.ldB decrease is achieved in 
FigA when BER is lO·6. 
From simulation results of the codes generated with PEG 
and ACE methods, we concludes that the designed QC­
LDPC codes using the proposed approach can exhibit better 
BER performance in both error floor and waterfall regions 
than LDPC codes generated with PEG method. 
VI. 
CONCLUSION 
In this paper, 
we discussed the effect of graph 
connectivity on LDPC performance and proposed an ACE 
constraint algorithm to construct QC-LDPC codes. By 
maximizing the ACE property of short cycles, small stopping 
sets can be suppressed. In the proposed algorithm, the small 
stopping set is removed by ACE and girth constraint, and 
local girth is enlarged by sum of girth constraint. These 
constraints ensure the error performance. Simulation results 
show an improvement in the error floor region and waterfall 
region of QC- LDPC codes over codes generated with PEG 
method.
