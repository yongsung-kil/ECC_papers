Abstract—Based on the particular structure of parity check matrix (PCM) of LDPC codes in WiMAX and WiFi wireless communication standards, a new early stopping criterion (SC) is proposed to save the unnecessary decoding iterations. By using the proposed SC, decoding process will be stopped if all the information bits in a code word are corrected even if there are still some errors in the redundant (parity) part. Simulation shows that our proposed SC outperforms previous ones on decoding speed evaluated by average iteration number (AIN) with no bit error rate (BER) performance loss.
I. INTRODUCTION
Recently, Low-Density Parity-Check (LDPC) code [1] as an error correcting code is adopted by many communication standards such as DVB-S2 , WiMAX (IEEE 802.16e) [2] and WiFi (IEEE 802.11n) [3] standards because of its nearcapacity error correction performance and potential for highthroughput decoding using iterative message-passing algorithm. Conventionally, the decoding process will be stopped if a pre-defined maximum iteration number is achieved. To compensate its low adaptability to variable communication channel conditions and speed up the decoding, early stopping criterion (SC) is invented. During the iterative decoding process if a defined SC is met, the decoding will be stopped thus to save the unnecessary decoding iterations.
The most straightforward SC is a satisfaction of all paritycheck equations (SPE) [4, 5] defined by the rows of a PCM of a certain LDPC code. SPE based SC can be met if and only if all the bits in a code word are correct, which means that it will not introduce any BER performance loss.
Another popularly used SC is called hard decision aided (HDA) SC which was originally invented for Turbo code decoding [6] and applied in LDPC code recently. For HDA SC, if the hard decisions made in two successive decoding iterations do not change, the decoding will be stopped. However, HDA SC does introduce some BER performance loss since no changes of hard decisions made in two successive decoding iterations does not guarantee a correct code word. To overcome this demerit, an additional threshold for magnitudes of a posterior probability (APP) is added to

make more restrictions, leading a BER performance improvement. [7]
In this paper, firstly, the convergence of information bits and redundant bits in a code word is discussed. Secondly, we exploit the particular structure of LDPC PCMs defined in WiMAX and WiFi standards. Finally, based on the previous discussion, a new early stopping criterion is proposed. For our proposed SC, if all the information bits in an LDPC code word are verified to be correct, the decoding will be stopped immediately even if there are still some error bits in redundant (parity) part of it. Simulation shows that our proposed SC outperforms previous ones on decoding speed, evaluated by average iteration number (AIN), with no degradation in bit error rate (BER) performance.
The organization of this paper is as follows. Section II introduces PCM of LDPC code defined in WiMAX and WiFi standards. In section III, an observation on convergence speed is discussed. Our proposed SC is presented in Section IV. After the simulation results and comparisons in Section V, Section VI concludes the paper.
II. PARITY CHECK MATRIX IN WIMAX AND WIFI
An LDPC code is defined by an M×N parity check matrix (PCM) where N and M equal to the code length and the number of redundant bits in a code word, respectively.
LDPC codes in WiMax and WiFi standards are quasicyclic (QC) codes, the PCM of which is an expansion of a c× d base matrix which is expanded by z×z sub-matrices where z is also called an expansion factor. The sub-matrices consist of cyclically shifted identity matrix and all-zero matrix. The base matrix is composed of nonnegative integers and “-1” (“-” for WiFi) where the nonnegative integer represents a z × z cyclically shifted identity matrix and “-1” represents a z×z allzero matrix. For the cyclically shifted identity matrix, the number of circulant shift equals to its corresponding nonnegative integer. The code length N equals to z×d, where d is the number of columns in a base matrix. An expansion example of z=4 and d=3 is available in Fig. 1. For all base

III. OBERSERVATION ON CONVERGENCE SPEED
In this section, an observation on convergence speed of information bits and redundant bits in a code word of WiMAX and WiFi is shown and discussed. Based on the observation we propose an idea to reduce the unnecessary decoding iterations.
LDPC code can be decoded using iterative messagepassing algorithm. With the check and variable messages being passed and updated between check nodes and variable nodes in a decoder, a posterior probabilities (APP) messages, based on which the final hard decisions are made, converge to be more and more reliable. Simulation results revealed that the bigger the weights of columns in PCMs, the faster the convergence of the APP messages corresponding to those columns.
For LDPC PCMs defined in WiMAX and WiFi standards, except z columns, e.g. z columns expanded by the column 12 in base matrix shown in Fig. 2, the weights of all the columns in redundant part of the matrix are two (marked red in Fig. 2) which is the smallest number among all. This particular feature leads to an interesting phenomenon that averagely, APP messages of information bits in a code word converge to a reliable value much faster than redundant bits which is shown in Fig. 3.
As shown in the figure, curves for average magnitude of APP values in log likelihood ratio (LLR) form over iteration numbers observed at different channel conditions are drawn.

More importantly, because the information bits converge much faster, they will be corrected earlier during the iterative decoding. Therefore, the code words which are incorrectly decoded due to the lack of decoding iterations always contain more errors in redundant part than information part. To demonstrate this, a simulation was run where a rate-0.5 length-2304 code in WiMAX was used. The incorrectly decoded code words are labeled into two types where code word of Type I has errors in information part or first z redundant bits and the errors in Type II code word are all among the rightmost M-z redundant bits. The proportions of two kinds of error codes at different channel conditions are depicted in Fig. 4.

By exploiting the particular structure of PCMs in WiMAX and WiFi standards, a mechanism is devised to find the Type II code word for a decoder. Based on that, a new early SC is proposed in the end of this section.

According to the statement, SAV of Type II code word contains only even numbers which can be easily verified by hardware. An example of z=4, R=5/6 and n=4 is given in Fig. 6. For this example, the SAV equals to [0, 4, 0, 2]T.
Reversely, if SAV contains only even numbers, the corresponding code word is of Type II for almost all the circumstances. This is because an error among information or first z redundant bits will at least affect three parity-check equations corresponding to three rows in random positions of a PCM, making the SAV disorderly. Other than strict proof, we run a simulation test where a length-2304 rate-0.5 code in WiMAX is used to show it. Among 7 million code words with SAVs containing only even numbers, only one code word is not of Type II which means the false detection rate is smaller than 10-7. Therefore, it is reasonable and effective to claim a Type II code word if SAV contains only even numbers. Based on the discussion above, now we propose our new early stopping criterion as follows.
For each decoding iteration, calculate the syndrome accumulation vector (SAV). If SAV contains only even numbers, stop the decoding.

VI. CONCLUSIONS
In this paper, we proposed a new early stopping criterion (SC) for decoding the LDPC codes in WiMax and WiFi standards. By using the proposed SC, the decoder will be stopped immediately if all the information bits in the code word are correct. By saving the unnecessary decoding iterations for correcting redundant-bit errors, our SC reduces the average iteration number (AIN) by up to 11% without loss of BER performance.
