I. INTRODUCTION
I T is known that the required signal-to-noise ratio (SNR) of error correcting codes to achieve very low bit error rates approaches the Shannon limit as the code lengths tend to infinity if the code rate is lower than the channel capacity. Unfortunately, since the complexity of encoders and decoders grows rapidly as the code length increases, long code lengths have not been affordable. For example, 32 k-bit codes can outperform 4 k-bit codes at the price of increased complexity for a fixed code rate. Recently, data storage industry has been moving toward 4 k-byte (32 k-bit) sectors [1] in order to take advantage of powerful longer error correcting codes by replacing conventional 512 byte (4 k-bit) sectors.
Meanwhile, many researchers have investigated low-density parity-check (LDPC) codes in magnetic recording channels in order to obtain coding gains over conventional Reed-Solomon (RS) codes with hard-decision decoding. Most of the research papers were focused on 4 k-bit LDPC codes, which can fit the conventional 4 k-bit sectors. Recently, LDPC codes for 32 k-bit sectors have been investigated in a few papers.
Chang and Cruz [2] investigated performances of nonbinary LDPC codes for 32 k-bit sectors. In their work, it was shown that 32 k-bit nonbinary LDPC codes show better performance than 4 k-bit nonbinary LDPC codes for 32 k-bit sectors in magnetic recording channels with transition jitter noise. In addition, they showed that the nonbinary codes outperform 32 k-bit binary LDPC codes. The coding gains over the binary LDPC code were relatively small in a high channel bit density regime whereas the coding gains increased at low channel bit densities. These results show very useful performance-complexity tradeoffs because the number of operations for 32 k-bit binary LDPC codes are the smallest among those three types of codes, 4 k-bit non-

the potential of 32 k-bit LDPC codes by illustrating available

A. Number of Operations in Decoders
Table I shows the complexity of LDPC decoding as the number of operations. In general, the number of overall operations is proportional to the number of ones in the parity check matrix of an LDPC code, or equivalently the number of edges in the graph representation. One of the desirable properties of LDPC codes is that the number of ones in the parity check matrix grows only polynomially in the code length, not exponentially. For example, if the density of matrix is maintained, the number of operations per bit per iteration grows linearly because the number of ones in parity check matrices grows as the square of the code length. We can see that the number of operations per bit per iteration is proportional to the column weight. The number of overall operations will be proportional to the number of iterations as well. Meanwhile, the cost of hardware implementation can depend on regularity and circulant size of the parity check matrix.
In Table I, as long as the column weights are the same, we can see that the number of operations per bit per iteration is the same for both the 4 k-bit code and the 32 k-bit code. However, designs of higher column weight 32 k-bit codes that potentially have better performance are available. For example, while 4 k-bit column-weight-9 codes have only girth 4, 32 k-bit LDPC codes can be designed such that those codes have column weight 9 or higher with girth 6. Available designs of 32 k-bit LDPC codes will be discussed in Section IV.

A subtle difference is that the relative positions of the 4 k-bit LDPC code and the 32 k-bit RS code was slightly changed.

Table II shows available designs for (36864, 32768) LDPC codes using the PEG algorithm with quasicyclic constraints. A circulant size, a desired girth, a column weight distribution, and the maximum number of trials are the input parameters to the PEG algorithm. Here the circulant size 512, for example, denotes the size of a 512 512 circulant matrix. A single column weight, rather than a distribution, was used for a regular column weight whereas row weights are not guaranteed to be regular. If the algorithm is successful within the maximum number of trials, a parity check matrix whose girth is not smaller than the desired girth is obtained. Otherwise, the algorithm stops with failure. Since the PEG algorithm is a randomized algorithm, the outputs from different executions can be different.
For low complexity implementation, large circulant size is desirable since quasicyclic codes can reduce the memory requirement for storing the parity check matrix by a factor of the circulant size. However, large circulant size limits the degree of freedom in design. For better performance, larger column weight is also desirable for large minimum distance of the code. However, larger column weight not only increases the implementation complexity but also limits the girth.

V. CONCLUSION
For the new 32 k-bit sector size, the coding gain of a 32 k-bit LDPC code over a 4 k-bit LDPC code, a 32 k-bit RS code, and a 4 k-bit RS code in magnetic recording channels is investigated. The decoding complexity of 32 k-bit LDPC codes and 4 k-bit LDPC codes is also discussed. It is also shown that 32 k-bit LDPC codes converge more slowly and need larger number of iterations than the 4 k-bit LDPC codes. In order to take advantage of the full potential of the 32 k-bit binary LDPC codes, higher column weight codes should be investigated, i.e., the 32 k-bit code in this work can still be improved. To illustrate the potential, quasicyclic LDPC codes with column weights up to 13 with girth 6 is designed, and such design is not available in 4 k-bit LDPC codes.
