Abstract� A new design approach of the LDPC decoder based on FPGA Impulse C Programming is proposed. The latest technique of Impulse C programming is used to implement hardware circuit, which is more efficient than the traditional HDL method. The decoder for a family of (3,6) LDPC Codes with 0.5 code rate and the block size of 2500 bits is implemented on Xilinx Virtex2 XC2V2000. When the maximum iteration time is 10 and the frequency is clocked at 50MHz, the throughput is 10Mbps.
Key words- Low-Density Parity-Check Codes; decoder; Field Programmable Gate Array; Impulse C programming;
I. INTRODUCTION
Low-cost, high-performance computing using highly parallel platforms is not a new concept. FPGA provide a large amount of highly parallel, configurable hardware resources. At the same time, LDPC code is a kind of high­ performance correct codes because it is closest to the Shannon limit. And it is suitable to be implemented using FPGA because of the low complexity and the parallelism of the decoding algorithm.
The features of Impulse C for expressing parallelism at a system level are similar to features found in other C-based languages for hardware and mixed hardware/software design. Impulse C can be used to describe a wide variety of functions that are appropriate for compiling to FPGA hardware. Impulse C maintains compatibility with standard C programming environments. In fact, the Impulse C programming model[l] is a communicating sequential process model intended for streams-oriented, mixed hardware/software applications. And the Impulse C tools allow designers to develop C language algorithm. Compiled by C language into the existing design flow, designers can quickly create and evaluate different hardware-software partitioning strategies, and establish a working prototype.
The goal of Impulse C is to allow the C language to be used to describe one or more units of processing and connect these units to form a complete parallel application that may reside entirely in hardware, as low-level logic

mapped to an FPGA, or that can be spread across hardware and software resources. This multiprocessing, parallel approach is highly appropriate for FPGA-based embedded systems.
Therefore, a new and convenient idea to design LDPC decoder on FPGA Programming in C is proposed in this paper, which is based on the latest programming technology from Impulse C to hardware implementation and suitable to the highly parallel FPGA platform.
II. THE DESIGN OF THE DECODER IN IMPULSE C
Decoding algorithm of the channel coding is an important factor to determine the coding performance and the application prospects. The complexity of the decoding algorithm determines the future of the coding. The relation between the decoding complexity and the code length of LDPC codes is linear. So the decoder with long packet length can be achieved without considering the problems of the decoding computational complexity.
Although several improved decoding algorithms for LDPC codes have been proposed in order to adapt to different situations, but the key of the algorithm is still the transmission of information between the variable nodes and the check nodes. In order to achieve a better balance between the decoding performance and complexity, and taking into account the hardware implementation, the MSA (Min Sum Algorithm) [2-5] is chosen in this design. The decoding algorithm flow is shown in Figure 1. Consequently the complex calculations and look-up tables are avoided. So, the complexity can be significantly reduced, the decoding performance can become better, and this algorithm may be suited to be implemented too.

A function library and related compiler and debugging tools are provided by Impulse C platform. Impulse C library can manage lots of multiple independent processes, which can be connected through data streams, signals and optional shared storage resources[6].Using the Impulse C library function, Impulse C language can be used to describe the highly parallel applications and has the minimal complexity in software programming. These library functions are used in the data communication between the flow and signals and assign the processes to the programming platform for the actual hardware resources.
The communications and synchronization between multiple processes is accomplished primarily through the data stream in this design. For example, when reading the decoding data, the CO PIPELINE instruction is used, because the pipelining technology allows parallel execution of multiple iterations of loops. When a loop that inputs a value and outputs a result, each iteration is converted to a pipeline, the resulting hardware may not produce any output until after it has received some number of input values, because multiple iterations are being executed at the same time. Suppose that N=1, without pipelining, the body of this loop requires two cycles. This example requires 200 cycles to process 100 inputs. Now, suppose that pipelining were used, this example results in a pipeline with two stages. However, after the first data value is read, stage one immediately starts reading the next input in parallel with the computation and output of the sum using the first input. This example now requires only 101 cycles to process 100 inputs.

THE IMPLEMENT OF THE DECODER ON FPGA Although Gallager proposed the LDPC decoding methods and a rough hardware circuit diagram, LDPC codes was not been focused and spread due to the limited technological level and the hardware conditions at that time. Therefore, FPGA provide a good solution now. Moreover, the Impulse C approach focuses on the mapping of algorithms to mixed FPGNprocessor systems, with the goal of creating hardware implementations of processes that optionally communicate via streams, signals and memories. The Impulse C tools include a software-to­ hardware compiler that converts individual Impulse C processes to functionally equivalent hardware descriptions and that generates the necessary process-to-process interface logic. The Loop unrolling compiler step performs a corresponding expansion of loops into equivalent parallel statements. The process from software to hardware compiler is the key to rapid development. It ensures that the applications program can be compiled directly into FPGA. A set of synthesizable and simulatable HDL files are generated finally, which includes a VHDL top-level module and a bottom module, and a number of related hardware and software interface source files. The speed and the hardware complexity are two important factors to measure the performance of the decoder. Higher decoding speed often means a greater consumption of resources. How to obtain both a faster decoding speed and a smaller hardware area? Finding the balance between the coding rate and the resource consumption must be taken into account during the design process.
Compared with those decoders in traditional hardware description language HD07-91, the number of logic cells is reduced, and the hardware scale is small relatively in this design. It is owing to the effective "stream programming" approach provided by Streams-C compiler in Impulse C libraries, and the parallel decoding architecture using pipelined structures in the chip. At the same time, some system does not require a high decoding rate, such as the third-generation mobile communication system, the highest rate of decoding of it is lOMbps. Thus the throughput of the chip in this design is able to meet the needs of the actual project and has the lower share of resource utilization.
IV. CONCLUSION
A kind of LDPC decoding hardware algorithms oriented data flow through the Impulse C language in the mixed hardware/software platforms is implemented in this paper. The advantage of FPGA design with Impulse C programming is obvious, and a fast and new idea is provided for software engineers to develop embedded hardware systems from C to FPGA. The next step of the study will be how to increase the parallelism and optimize the Impulse C codes in order to achieve low-cost high­ performance computing using highly parallel platform.
