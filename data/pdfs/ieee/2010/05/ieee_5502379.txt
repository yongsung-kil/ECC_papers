Hybrid Decoding of LDPC Codes Based on Interior Point Method
T. M. N. Ngatched, Attahiru S. Alfa, and Jun Cai Department of Electrical and Computer Engineering
University of Manitoba Winnipeg, MB R3T 5V6
Canada Email: {ngatched, alfa, jcai}@ee.umanitoba.ca

Abstract—In this paper, a hybrid decoding algorithm for finite-geometry low-density parity-check (FG-LDPC) codes is proposed. The algorithm is based on the interior point method with barrier function introduced by Wadayama [6]. First, an efficient implementation of Wadayama’s algorithm is presented. The main idea behind the modification is to approximate the barrier function for the fundamental polytope defining the code so that it contains only one linear constraint for each of the parity-check constraints. A two-stage hybrid decoding which combines the interior point decoding (IPD) and a low-complexity decoding algorithm for FG-LDPC codes is then proposed. In the first stage, the interior point decoding is used to generate a search point. If the first stage decoding fails, the decoding is continued by the low-complexity algorithm that is initialized by the result of the IPD. Compared with a conventional iterative messagepassing (IMP) decoder, the proposed hybrid algorithm achieves better bit-error rate (BER) and frame-error rate (FER) for small block lengths at medium to high signal-to-noise ratio (SNR).
I. INTRODUCTION
Standard decoders for low-density parity-check (LDPC) codes [1] are based on iterative message-passing (IMP) algorithms such as iterative decoding based on belief propagation (IDBP) [2]. While their observed error-correcting performance is unparalleled, the theoretical understanding of IMP decoders remains incomplete. Proposed by Feldman et al. [3], linear programming (LP) is an alternative to iterative algorithms for decoding LDPC and turbo-like codes. The LP decoder is based on a LP relaxation of the original maximum-likelihood (ML) decoding problem, and utilizes a set of constraints which contain all valid codewords of a given code and a linear objective function. Although the practical performance of LP decoding is comparable to message-passing decoding [3], [4], a significant advantage is its relative amenability to nonasymptotic (finite-length codes) analysis. Moreover, there turn out to be a number of important theoretical connections between the LP decoding and standard forms of iterative decoding [5]. In addition, contrary to standard IMP algorithms, LP decoding has the ML certificate property. The design of LP solvers for LP decoding of LDPC codes using interior-point methods has recently been investigated in [6] and [7], respectively. In

particular, the approach of Wadayama [6] called interior point method with barrier function yields encouraging results.
Finite geometry (FG) LDPC codes [8] have been attracting great interest by virtue of the fact that they are algebraically generated and allow effective, low-cost, hardware encoder implementations. However, compared to other classical LDPC codes, decoding FG-LDPC codes using the standard interior point algorithm of [6] is practically impossible, due to the heavy row weights of their parity check matrix.
In order to make Wadayama’s algorithm computationally tractable and applicable to FG-LDPC codes, in this paper, an efficient implementation of the algorithm is first proposed. The main idea behind the modification is to approximate the log-barrier function for the fundamental polytope defining the code so that it contains only one linear constraint for each of the parity-check constraints. This leads to an approximate variation of the Newton method used to solve the convex optimization problem. A huge reduction in computational costs is hence obtained. A two-stage hybrid decoding which combines the interior point decoding (IPD) and a low-complexity decoding algorithm for FG-LDPC codes is then proposed. In the first stage, the IPD is used to generate a search point. If the first stage decoding fails, the decoding is continued by the low-complexity algorithm that is initialized by the result of the IPD. The process is repeated iteratively until either a codeword is found, or the user-predefined maximum number of outer iteration is reached. The performance of the proposed algorithms is demonstrated through simulation, and it is observed that the approximation does not result in any performance degradation, while considerably reducing the decoding complexity and latency. It is also observed that the hybrid decoding outperforms the IDBP for small block lengths, making it a promising candidate for decoding short FG-LDPC codes.

II. LP RELAXATION OF ML DECODING
A. LDPC Codes
A binary LDPC code is described by its sparse binary paritycheck matrix H. For an (N, K) LDPC code, H has N columns and M ≥ N − K rows. For a regular LDPC code, H has a constant column weight wc and a constant row weight wr. The set of bits that participate in check m is denoted as N (m) {n : Hmn = 1}. Similarly, the set of checks in which bit n participates is denoted as M(n) {m : Hmn = 1}. For the high rate FG-LDPC codes of interest, M = N implying that there exist many redundant check sums in H.

Moreover, using general purpose LP solvers can be highly inefficient as they do not take advantage of the particular structure of the optimization problems arising in LP decoding. IV. EFFICIENT IMPLEMENTATION OF THE BARRIER FUNCTION METHOD FOR FG-LDPC CODES
The heart of the decoding algorithm is in the Newton iterations. During these steps, a search point x is updated using the Newton method to minimize the objective function ψ (x).

During this process, the objective function, and hence the barrier function, is evaluated many times. Any odd-sized subset V of the neighborhood N of each parity check node introduces a unique parity inequality. Using the binomial theorem, it can easily be shown that there are 2wr−1 constraints corresponding to each check node of degree wr. The number of inequalities (4) is thus M 2wr−1. Therefore, a brute-force evaluation of the barrier function requires O (N 2wr ) computation time. Even for parity-check matrices of moderate row weights, this number can be very large. This becomes more significant and prohibitive for FG-LDPC codes due to the heavy row weight of their parity check matrices.
Moreover, the execution of the Newton Method requires the computation of the gradient and the Hessian of the objective function. Computation of the gradient involves the partial derivative of the barrier function and contains M 2wr−1 terms. Likewise, a straightforward evaluation of the second derivative of the barrier function required for the Hessian needs O (N 2wr ) computational time. These computational costs are also a major obstacle to achieving an efficient implementation of the interior point decoding for FG-LDPC codes. In this section, some approximations are introduced in order to make the interior point decoding computationally tractable and applicable to FG-LDPC codes.

V. HYBRID DECODING OF FG-LDPC CODES BASED ON INTERIOR POINT DECODING

In the IPD process, it is possible that the current search point

approaches close enough to a codeword vertex. In such a case,

it is advantageous to use a low-complexity decoding algorithm

to complete the decoding instead of waiting for the search

point to converge to a codeword vertex, which in general

requires a much longer computational time. In this Section,

we propose a two-stage hybrid decoding which combines the

IPD described previously and a low-complexity decoding al-

gorithm for FG-LDPC codes. Many low-complexity decoding

algorithms for FG-LDPC codes have been proposed in the

literature. Among these algorithms, the multiple bit flipping

(MBF) of [11] offers a good tradeoff between performance

and complexity.

The hybrid algorithm works as follows. In the first stage, the

IPD is used to generate a search point. A binary quantization

of the search point is obtained, and the syndrome of the

resulting vector is calculated. If the first stage decoding fails,

the log-likelihood ratio of each coordinate of the search point

is calculated, and the decoding is continued by the MBF

algorithm. The process is repeated iteratively until either a

A. Accuracy of the Approximation
Due to the prohibitive complexity of the standard (without approximation) interior point decoding algorithm for FGLDPC codes, it was impossible to obtain simulation results even for a small block length as 255. To have an insight on the effect of the approximation, a randomly generated (3,10)-regular LDPC code of length 273, dimension 191 and rate 0.7, where the cycles of length four were removed from the Tanner graph of the code, was used. Fig. 1 depicts the error performance of the code with the standard interior point, the proposed interior point, and the IDBP algorithms. It can be seen that the proposed IPD achieves almost the identical performance with the standard IPD. Even for this LDPC code of moderate row weight (wr = 10), the running time of the standard IPD algorithm was about 10 times that of the proposed IPD algorithm. Compared with the IDBP algorithm, it can be observed that the proposed algorithm achieves better performance in the high SNR region.
B. Performance of the Proposed Interior Point Decoding
The performance of the interior point decoding algorithm for the (255, 175) code is shown in Fig. 2 for different outer iteration steps. As expected, the performance improves with increasing number of outer iteration. There is, however, a diminishing return as the number of outer iteration increases. It can be observed that, there is no significant improvement after the second outer iteration. Fig. 3 depicts the average number of outer iterations during the decoding of the same code. We see that, at high SNR, it takes, on average, less than two outer iterations to achieve the decoding.
C. Performance of Hybrid Iterative Decoding
Fig. 4 shows the performance of the hybrid decoding algorithm for the (255, 175) code for different outer iteration steps. The performance also improves with increasing number of outer iteration with a diminishing return. As in the case of interior point decoding, there is no significant improvement after the second outer iteration. It should also be noted that, due to the MBF decoding, the gain from the first outer iteration to the second outer iteration is much less than in the case of interior point decoding. The average number of outer iterations during the decoding is shown in Fig. 5. It can be seen that the hybrid decoding requires, on average, less outer iterations. At high SNR, it takes exactly one outer iteration to achieve the decoding. Since, an iteration of the interior point decoding

algorithm is much more complex than the MBF algorithm, a considerable amount of computation is thus saved using the hybrid algorithm. The performances of the IDBP, interior point decoding, and hybrid decoding algorithms are compared in Fig. 6. It can be observed that, at low SNR, the IDBP slightly outperforms both the interior point and the hybrid algorithms. However, at medium to high SNR, the situation is reversed with the interior point and the hybrid outperforming the IDBP. At the BER of 10−6, the hybrid algorithm outperforms the IDBP by about 0.2 dB.
VII. CONCLUSION
In this paper we studied efficient implementations of interior point method with barrier function for the decoding of FGLDPC codes. Key to our approach is the approximation of the barrier function for the fundamental polytope defining the code so that it contains only one linear constraint for each of the parity-check constraints. This approximation, together with the subsequent approximation of the gradient and the Hessian, make the interior point method with barrier function computationally tractable and applicable for the decoding of FG-LDPC codes. A two-stage hybrid decoding which combines the IPD and a low-complexity decoding algorithm for FG-LDPC codes is proposed. The performance of the proposed algorithms are demonstrated through simulation, and it is observed that the approximation does not result in any performance degradation, while considerably reducing the decoding complexity and latency. It is also observed that the hybrid decoding outperforms the IDBP for small block lengths.
