A Generic Conflict-Free Architecture for Decoding LDPC Codes using Perfect Difference Networks.
Balaji Janakiram, M.Girish Chandra, B.S. Adiga, Harihara S.G and Balamuralidhar P Innovation Labs, TATA Consultancy Services, Bangalore, INDIA.

Abstract‚ÄĒThe flurry of activities in the area of Low Density Parity Check (LDPC) codes from last one decade have led to different flavors of codes and the corresponding decoders. The decoder involves computing elements and an interconnection mechanism, usually tailor made for a particular code. This paper explores a generic reconfigurable architecture for decoding different LDPC codes based on the Perfect Difference Networks (PDNs). The architecture is conflict-free in terms of the message data access by the nodes and can also be utilized in large factorgraph based processing.
I. INTRODUCTION
Low Density Parity Check (LDPC) codes have emerged as very powerful error control codes with performance approaching capacity limits. These codes are finding utility in various applications and standards; examples being the DVBS2 standard (2nd Generation Digital Video Broadcasting via Satellite) and various storage applications.
LDPC codes are linear block codes originally proposed by Gallager in the early 1960s [1]. Their parity check matrix is sparse having low density of ‚Äėone‚Äôentries. LDPC codes can be either regular or irregular. In the regular case, they have uniform column and row weights in the parity check matrix. Over the years, many different LDPC codes (both regular and irregular) have been developed [2], exhibiting different properties in terms of the way of encoding, the encoding complexity, the way of decoding, the decoding complexity, error performance (both bit and frame error rates), etc.
The popular LDPC decoding algorithm is the Sum-Product Algorithm (SPA), also referred to as the Belief Propagation algorithm. The decoding operation can be viewed as repeated (iterated) message passing operating on the Tanner Graph (which is a bipartite graph) representing the parity check matrix. The nodes of the graph are variable and check nodes. Each of these nodes is a processing element and performs particular function computation to generate the requisite information for message passing. In each iteration, messages from each check node are passed to the adjacent variable nodes and similarly from each variable node to its adjacent check nodes. The decoding performance is achieved through repeated iterations of the message passing along the edges of the graph, with some stopping criterion. In a nutshell, a decoder architecture of an LDPC code consist of computing nodes and an interconnect network to facilitate the communication among them. Further, the decoding LDPC codes is communication intensive as messages are exchanged frequently between variable nodes and check nodes. Needless to say, there can be

conflicts among the computing nodes during the process of message exchange, either utilizing their internal memory or through external memory. This necessitates an efficient interconnection network between the computing nodes.
It is to be noted that in general, the decoder architecture is code specific and is designed with a view to speed up communication between nodes by reducing or avoiding conflicts. Having a generic architecture, which is conflict free, for decoding different LDPC codes can definitely be interesting and challenging proposition. In this paper, an attempt is made to arrive at a generic conflict-free architecture, using Perfect Difference Networks (PDNs). These were originally invented while trying to achieve low-latency, high bandwidth, energy-efficiency and robustness in networks for parallel and distributed computing [3], [4].
The paper discusses in detail a strategy to map the decoder of any arbitrary LDPC code onto a PDN which is easily scalable and reconfigurable. To the best of our knowledge, this is not yet explored elsewhere. In this direction, it is organized as follows. In Section II, the necessary background on PDNs is provided. In Section III we arrive at the proposed generic architecture and elaborate on the decoding using the same. The paper concludes with some discussion presented in Section IV.
II. PERFECT DIFFERENCE NETWORKS
Given a set of nodes, the ideal situation would be to have each node connected to every other node, thus allowing one-hop communication between any pair of nodes. This connectivity is referred to as the complete-graph connectivity; the complete graph is of diameter 1. Physically, however, complete-graph connectivity is difficult to provide for large systems that are of practical interest, due to both the high cost of nodes with many communication channels and lack of scalability for system growth. The next best strategy is to arrive at good diameter-2 interconnection structures leading to more economical networks. Perfect Difference Networks (PDNs) offer one solution in an asymptotically optimal manner as defined in [3],[4].
PDNs are based on the mathematical notion of perfect difference sets (PDS), which were first discussed by J. Singer, formulating them in terms of points and lines in a finite projective plane [5]. These sets have (őī + 1) elements, where őī = pt, p is prime and t is a positive integer. The elements of PDS are such that any integer from {1, 2, 3, . . . , őī2 + őī} can be written in a unique way by taking the modulo difference

B. SPA decoding using PDN
The LDPC decoder architecture essentially has a shuffle network which caters for the exchange of soft information between variable nodes and check nodes. The LDPC decoders for cyclic codes can involve a very simple shuffle network (inter-connecting architecture)[7]. But it can become really be cumbersome for a non-cyclic LDPC code [6], [8]. In order to account for both cyclic and non-cyclic LDPC codes (regular or irregular), i.e. to have a universal shuffle network, we have designed an elegant PDN based inter-connection architecture as shown in Fig. 2, which efficiently manages the connectivity among the nodes without any clashes. It is to be noted that there are already few flexible LDPC decoder architectures which can cater for broad variety of LDPC codes [9]. But the PDN based architecture can be utilized to decode any arbitrary

LDPC code and the architecture itself is obtained through a
systematic mathematical framework. Let őľ = {őľ(k); 1 ‚Č§ k ‚Č§ N } be the set of N nodes in the
PDN where each node őľ(k) can be viewed as a processor. For a LDPC code having n variable nodes {v1, v2, . . . , vn} and m check nodes {c1, c2, . . . , cm}, if a processor őľ(k) is mapped to at most one variable node and one check node, it is called
one-to-one mapping.

IV. DISCUSSION AND CONCLUSIONS
After presenting how the decoding of LDPC codes can be carried out using PDNs, it is important to examine how the suggested architecture is generic and can be reconfigured to decode different LDPC codes. This can be achieved by the schematic depicted in Fig. 4. The necessary messages (LLRs) are appropriately stored in the memory banks for conflict-free decoding. The PDN of N nodes can be on a dedicated hardware. The scheduler which is programmed with code specific information is responsible for assigning the variable and/or check node processing tasks to the nodes. A simple strategy is to have a round robin scheduling scheme where variable and/or check nodes can be assigned in a circular fashion. To increase the overall decoder efficiency, code specific scheduling schemes can be designed based on a heuristic approach. In many practical systems, LDPC codes are often designed keeping the decoder architecture in mind (decoder first design). This generally results in reduced error performance. With the proposed decoding scheme, it is possible to focus solely on the codes that achieve very good error performance. The decoding is still assured to be straightforward and elegant with the PDN based architecture.
It is to be noted that the Tanner Graph is a special case of Factor Graph, which graphically represent the factorization of functions. Factor graphs provide a powerful and generic framework to solve many inference (both estimation and detection) problems [10]. The PDN based architecture presented in the paper can be an elegant computing framework to solve many of these problems. It is worth noting that in a factor
