An Effective Puncturing Scheme for RateCompatible LDPC Codes

He-Guang Su, Ming-Qiu Wang, Shu-Tao Xia
Graduate School at Shenzhen, Tsinghua University Guangdong 518055, P. R. China Email: xiast@sz.tsinghua.edu.cn

Abstract - We consider the problem of finding good puncturing patterns for rate-compatible LDPC codes over additive white Gaussian noise (AWGN) channels. When studying the recovered process of punctured nodes, Ha et al. maximized the number of lower k-SR nodes and obtained a so-called grouping and sorting puncturing scheme, where only single survived check node was guaranteed. In this paper, we investigate the effect of multiple survived check nodes and afford theoretical analysis to them. Based on the analysis, we propose an effective puncturing scheme called MSCN, which maximizes the number of survived check nodes. The MSCN scheme is quite different from the grouping and sorting scheme in the rules of finding nodes and update. By simulations, the MSCN scheme is shown to be superior to the grouping and sorting scheme over AWGN channels at low rates for rate-compatible LDPC codes.
Keywords - Low-density parity-check (LDPC) codes, ratecompatible, Tanner graph, puncturing.
I. INTRODUCTION
In wireless communication systems, the channels are usually time-varying. Error correcting codes are required to be flexible in terms of their rates in order to maximize the throughput of the channels. Rate-compatibility is adopted to achieve this flexibility, using only one pair of encoder and decoder. Puncturing is a practical and effective method to attain rate-compatibility. Though puncturing is convenient to realize rate-compatible codes, it has the shortcoming of performance loss. For a low-density parity-check (LDPC) code, its Tanner graph consists of variable nodes and check nodes. A puncturing pattern is the selection of punctured variable nodes. Finding good puncturing patterns is of great importance to reduce the performance loss.
Lots of work has been done to find good puncturing patterns. Ha et al. [1] show that good puncturing patterns exist and the puncturing can be performed in a rate compatible fashion. Moreover, it is possible that a single LDPC code punctured in a rate-compatible way remains good across a range of rates [2]. However, the block length is assumed to be infinite in their results. Some work with assumption of finite block length can be found in [3,4]. Puncturing in [3,4] are performed randomly, in which some important bits may be lost and corresponding punctured nodes will thus be unrecoverable, especially at high code rates. To overcome this problem, Ha et al. [5] studied the recovered process of punctured nodes when decoding. They adopted the idea of maximizing the number of

lower k-SR nodes and propose algorithms for puncturing: the so-called grouping and sorting scheme. Grouping algorithm takes the variable nodes into different sets called G1, G2,…,GK, where Gi’s are sets of i-SR nodes. With the grouping algorithm, the number of nodes in lower sets has the priority to be maximized. First, the size of G1 is maximized, and then the size of G2 is maximized and so on. In each set, the order of nodes to be punctured is determined by sorting algorithm. Though the grouping and sorting scheme outperforms random puncturing schemes, it guarantees only single survived check node.
We investigate the role of multiple survived check nodes and propose a scheme named MSCN for finding puncturing pattern accordingly. Simulations show that the proposed scheme is superior to the grouping and sorting scheme at low rates over AWGN channels. The rest of the paper is organized as follows. Section II briefly introduces some basic notations and density evolution with Gaussian approximation. In section III we make an analysis of multiple survived check nodes, explain our basic puncturing ideas and propose an effect puncturing scheme for finite-length LDPC codes. Section IV shows the simulation results of the proposed puncturing scheme. Finally, conclusions are provided in section V.
II. PRELIMINARIES
A. Basic Notations
An LDPC code is defined by a sparse parity-check matrix H of size m*n. For a regular LDPC code, all the degrees of the variable nodes are the same. Let dv be the variable node’s degree, i.e. the number of ‘1’ in each column is dv. For a node V, we define N(V) as neighbors of V. Note that if V is a variable node, then nodes in N(V) are all check nodes and vice versa.
Define A\B by{node i : i ∈ A, i ∉ B} .
A punctured variable node can’t get message from a channel. It is said to be recovered if it receives none-zero LLR message from its neighbors. In [5], punctured node V is called one step recoverable (1-SR) if it has at least one neighbor C whose all other neighbors except V are unpunctured. This check node C is referred to be a survived check node (SCN). Generally, a punctured variable node V is called k step recoverable (k-SR) if V has at least one neighbor C, such that N(C)\{V} contains at least one (kˉ1)-SR node and the others are m-SR, where 0İmİkˉ1. B. Basic Puncturing Idea and A Simple Example
It is shown by the analysis above the LLR of a punctured variable node with more survived check nodes will evolve faster. Simultaneously, it will result in a better performance. Based on the analysis, our new algorithm adopts the idea of maximizing the number of survived check nodes. Initially when a variable node V is punctured, we turn N(V) into V’s survived check nodes. N(N(V))\{V} should not be punctured accordingly, since one check node can only recover one punctured variable node. As puncturing proceeds, the number of punctured nodes grows but new punctured node will destroy survived check nodes of previous punctured nodes inevitably, which will lead to performance loss. To reduce this loss, our new algorithm controls the decrease of survived check nodes’ number strictly. The new variable node we select to be punctured should have the minimal impact on the previous punctured nodes. Namely, the decrease in the number of survived check nodes of previous punctured nodes, due to the new punctured variable node, should be minimized. Hence the new punctured variable node must have the minimal number of neighbors that are previous punctured nodes’ survived check nodes.
If a punctured variable node V has only one survived check node C which carries state (l, 1), this check node C’s neighbors except the punctured one V should not be punctured any more, otherwise the previous punctured node V couldn’t be recovered. C’s neighbors except V should be appended to G0, where G0 is a set of variable nodes which are not punctured during puncturing. The punctured nodes and the nodes in G0 are called fixed nodes. Our finding of puncturing pattern is finished when all the variable nodes are fixed.
Next, we will make a brief comparison below between our puncturing algorithm and the grouping algorithm in [5]. The grouping algorithm can be summarized as follows:
1) Finding one pair (V*, C*), where V* and C* are an l-SR variable node and a survived check node of it. V* is chosen
with minimal neighboring candidate check nodes to maximize

the size of lower l-SR groups and C* is chosen with minimal
neighboring non-grouped variable nodes so as to reduce error
probability. 2) Update. The nodes in N(C*)\{V*} are affiliated into G0
to ensure C* can recover V*. Meanwhile, the nodes in N(V*)\{C*} are eliminated from candidate check nodes and
they may become survived check nodes of higher l-SR nodes.
3) Decision. If all the punctured nodes have been found,
the finding of nodes is finished. Otherwise we continue to
find the next punctured node. And our puncturing algorithm can be summarized as
follows:
1) Finding a node V* as l-SR variable node. V* should be
chosen with minimal neighboring survived check nodes of previous punctured nodes such that the damage by V* can be minimized. Almost all the nodes in N(V*) are V*’s survived
check nodes. 2) Update. N(N(V*)) must be exclude from the candidate
variable nodes to ensure V* has survived check nodes as
many as possible. The nodes in N(Partner(j)\{j}),where jęN(V*), are also excluded so that the decrease in the number of survived check nodes of precious punctured nodes is
minimized.
3) Decision. If all the variable nodes are fixed, we finish
the finding of punctured nodes, otherwise continue to find the
next node. Grouping algorithm maximizes the number of lower k-SR
nodes, while our proposed algorithm maximizes the number of survived check nodes. They are quite different in the rules of finding nodes and update.

New punctured nodes may destroy survived check nodes of previous punctured nodes. By using check node state, we can see the situation of survived check nodes clearly. With the help of the information from check node states, controlling the change of survived check nodes is feasible, which can’t happen in [5]. Generally, in order to ensure good performance, the number of survived check nodes of a previous punctured node can only be decreased by 1 when a new variable node is punctured. In other words, the variable p in a check node state can only be decreased by 1 each time. So the state of a check node changes from (1, dv) to (1, dvˉ1), (1, dvˉ2) … (1, 2), (2, dv), (2, dv ˉ1)… step by step in general situation.
Processing State (l, p) in the algorithm we proposed indicates that we will take the neighbors of check nodes with state (l, p) to be candidates for puncturing. The punctured nodes will be selected from these candidates.
In step 0, all variable nodes are candidates for puncturing, and all check nodes are initialized with state (0, 0). nowPS is the check node state we process now, that is, we will take the neighbors of check nodes with state nowPS to be punctured. Candidate is updated with Candidate \ N (N (V)) to make sure punctured node V has the most number of survived check nodes, i.e. dv survived check nodes, all the neighbors of V are its survived check nodes. So the states of N (V) change to be (1, dv). All the punctured nodes obtained in this step are 1-SR and have dv survived check nodes respectively.
In step 1.0, prePS are the states we process previously. We look for node V with the largest |N(V)ģ{j:State(j)ęprePS}|. The number of links between this node and check nodes carrying state nowPS is the least, i.e. dvˉ |N(V)ģ{j:State(j)ę prePS}|, since V has dv neighbors. Thus damage by new punctured node V is minimized. The punctured nodes obtained in this step are k-SR, while that obtained in later steps are (k+1) -SR.
After step 1.2, most of check nodes’ states are altered to be (k, dvˉ1) from (k, dv ). In step 1.3, the main states of check nodes are (k, d), where 1<d<dv. In update 1, N(Partner(j)\{j}, j ę A) are excluded from Candidate and Candidate*. The exclusion ensure that the node previously recovered by check node j has a slow decrease in the number of survived check nodes, reducing only by 1. To guarantee this new punctured node has the most number of survived check nodes, we exclude N({j:jęB}) from Candidate and Candidate*. In update 1 Ĺ, if State(j').p =1, the variable node V recovered by check node j' has only 1 survived check node, i.e., j'. As a survived check node, j' couldn’t be destroyed any more, otherwise, V couldn’t be recovered in the k-th iteration. Therefore N(j') should be assigned to be fixed nodes. In update 2 ĸ, all V’s survived check nodes connect to k-SR variable nodes, so that V’s recoverable level is k+1.

How to determine the order of selected nodes for puncturing? In each loop of the proposed algorithm, the selected nodes form a group respectively. It is important that nodes in the previous group should be punctured before that in the latter group. Within each group, if the nodes are selected randomly, the order to be punctured between them can be arbitrary, otherwise the nodes should be punctured in the order they are selected.
IV. SIMULATIONS
In this section, we simulate the performance of the proposed algorithm (MSCN), comparing with that of grouping and sorting algorithms. The mother code we use is 1/2 rate (3, 6)-regular LDPC code of which block length is 1024. It is constructed by Progressive Edge-Growth (PEG) algorithm in [7]. The bit error rates (BER) and frame error rates (FER) of the punctured code are presented in Fig. 3 and Fig. 4 respectively. In the figures, 0.6-Grouping/Sorting means that the code is punctured applying algorithms of grouping and sorting at rate 0.6. Similarly, 0.6-Proposed Algorithm means that the code is punctured applying our proposed algorithm at rate 0.6.
As is shown in both figures below, our proposed algorithm performs better than the previous algorithms at lower rates. In section III, our proposed algorithm ensures that each punctured variable node has survived check nodes as many as possible. With the analysis, it is clearly more survived check nodes ensure lower error probability. Therefore our algorithm shows better performance than the previous algorithms. At higher rates, new punctured nodes destroy previous punctured nodes’ survived check nodes. The number of survived check nodes of a previous punctured node gets smaller and smaller, decreasing to 1 in the worst case. Accordingly, the performances of the proposed algorithm are very close to that of previous algorithms at higher rates. In Fig. 3, the proposed algorithm provide nearly 0.15, 0.2db SNR gains compared with grouping and sorting algorithms at rate 0.6, 0.65 respectively at the BER of 10-5.

V. CONCLUSIONS
In this paper, we investigated the effect of multiple survived check nodes to obtain good puncturing patterns of rate-compatible LDPC codes. Theoretical analysis showed that multiple survived check nodes can lower error probability. Based on the analysis, in order to find good puncturing patterns, an effective puncturing scheme was proposed to maximize the number of survived check nodes. The proposed scheme is quite different from previous schemes in the rules of finding nodes

and update. By simulations, it was shown that the proposed scheme outperformed previous schemes at certain cases over AWGN channels.
