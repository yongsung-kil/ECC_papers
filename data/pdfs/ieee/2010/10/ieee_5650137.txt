Abstract
Quasi-cyclic (QC) low-density parity-check (LDPC) codes have advantages over other types of LDPC codes due to their cyclic shifting property. In this paper, a partially parallel QC-LDPC decoder with efficient memory design is proposed. To improve the memory utilization, the structures of efficient chunk and predictor are realized. The efficient chunk can store parts of check-to-variable messages. This design can effectively reduce the memory requirement. Furthermore, the predictor verifies variable-to-check messages and updates efficient chunks instantly. With less check-to-variable messages and variable-to-check messages stored in the memory, this decoding architecture only deposits few messages in several efficient chunks, such that the cost of memory is less. Eventually, the proposed approach dramatically reduces memory requirement about 65% compared to the traditional method while maintaining the same low-cost.
1. Introduction
Being first introduced by Gallager in 1962 [1] and rediscovered by Mackay recently, Low-density parity-check (LDPC) codes are a group of linear block codes specified by very sparse parity-check matrix H. Their performances are very close to Shannon limit and asymptotically superior to turbo codes [2]. LDPC codes usually can be applied to many high data rate applications, such as magnetic storages and wireless communications due to their excellent error correcting performance. However, the typical sparse, random, and large parity-check matrixes used to construct LDPC codes, impose great design challenges about the data routing and large amount of memory requirement on hardware implementations.
Quasi-Cyclic LDPC (QC-LDPC) codes [3], one of LDPC codes class, suit for the decoder implementation because of the regularity in the parity-check matrix. The QC-LDPC codes with code-length less than 10000 bits can achieve comparable performance to the randomly constructed ones while the their construction is well structured [4]. Besides, partially parallel QC-LDPC decoder has been studied widely to improve the throughput of decoding in recent years [5-7].
Basically, the message passing algorithm (MPA) [8] is adopted to decode the LDPC codes. Then, the variable-to-

check and check-to-variable messages are iteratively calculated and exchanged along the edges in the corresponding Tanner graph until stopping criteria are satisfied. The QC-LDPC decoder, consisting of the Check Node Function Unit (CNFU) and the Variable Node Function Unit (VNFU), stores all the check-to-variable and variable-to-check messages in separate memory block. However, this intuitive method is uneconomical for the hardware design. Accordingly, a QC-LDPC decoder with the efficient memory utilization has been proposed in [6]. With this optimization, all check-to-variable messages corresponding to one row in H matrix can be stored as a compressed form. Indeed, this efficient architecture realizes the column summation of check-to-variable messages in the VNFU. Thus, this strategy can save memory about 37%, compared to the traditional scheme. Chen and Hocevar [7] also introduced a kind of parallel decoder architecture. However, the irregular property of the parity check matrix is rarely discussed for the parallel requirement because that considerably adds to complexity in the MPA.
In this paper, we will store check-to-variable messages corresponding to one row in H matrix as a simplified memory form, names as efficient chunk, with a modified Min-sum algorithm (MSA) to achieve the memory efficiency. Furthermore, a proposed predictor will verify variable-to-check messages and update efficient chunks instantly. Without variable-to-check messages stored in the memory, the memory complexity can be reduced dramatically. The structure of this paper is as follows. Section II reviews the QC-LDPC codes construction and related decoding algorithms. The memory optimization policy and modified MSA are presented in Section III. Section IV shows the complexity analysis of the simplified memory structure. Finally, concluding remarks are revealed in Section V.
2. QC-LDPC codes
QC-LDPC codes can easily satisfy the requirement of parallelization for the decoder design with MPA. This strength appears with the block structure in the parity check matrix. In other words, the connections between variable nodes and check nodes are grouped with certain constraints. The cooperation between the code construction and MPA is essential to the effect of parallelization.

The construction of QC-LDPC codes would be focused on the corresponding parity check matrix H. To give a realistic cases, this parity check matrix H can consist of several cyclic shift matrices of identity matrices. For (j, k) QC-LDPC codes, the generation would be illustrated as follows.
1) Construct the all-one block matrix H' which size is jГ—k.
2) Determine offsets with (b(s-1)Г—a(t-1)) modulo L, where the pair (s, t) denotes the location of each sub-matrix with 1 в‰¤ s в‰¤ j, 1 в‰¤ t в‰¤ k , and a, b are multiplicative orders of k, j, respectively.
3) Replace each element in H' with cyclically shifting LГ—L identity matrix with a given offset, where L is a prime number and j, k are prime factors of L-1 and obtain the parity check matrix H.

In Fig. 1, two smallest magnitudes of variable-to-check messages, min1 and min2, will be chosen to update check-tovariable messages. All check-to-variable messages are updated with the min1 and the sign bit except the message with min1. This left message would be updated with min2 and the sign bit. Therefore, two methods of memory reduction from these observations are discussed. Firstly, all the check-to-variable messages which have been updated in a check node can be compacted and stored in an efficient chunk which includes the two smallest magnitude (min1 and min2), the smallest magnitude index (min1 index), and all the sign bits of variableto-check messages, as shown in Fig. 2. "0" in the sign bit indicates the positive sign of the magnitude, and "1" for the negative. In addition, an additional unit records the binary-sum "вЉ•" of the sign bits in the variable-to-check messages. Next,
each updated variable-to-check message can be used to update the efficient chunk in next iteration immediately in Fig. 3. Accordingly, the storing of all variable-to-check messages is unnecessary. We declare an efficient chunk as a simplified data structure for storing check-to-variable messages with respect to these two behaviors. Thus, the memory requirement is reduced dramatically. Besides, due to the regularity of edge number in QC-LDPC, the flexibility is not an issue to be considered with this efficient chunk.

C. Proposed QC-LDPC Decoder
Fig. 4 represents the architecture of QC-LDPC decoder. In (J, K) regular QC-LDPC codes, there are J block rows. Due to the property of regular QC-LDPC codes, each block row occupies one memory bank and no access conflict among this rows. Efficient chunks with updated check-to-variable messages are located in memory1. Check-to-variable messages for the next iteration are updated by the predictor and stored in memory2 in the form of the efficient chunk. In an iteration, memory1 and memory2 will exchange, such that the next update process can access these memories in different function unit if the iteration is in progress. Intrinsic messages are stored in Intrinsic Message (IM). In the memory access process, the circular buffer can fulfill memory access in every memory bank, because QC-LDPC codes have the cyclic shifting feature. This regularity does not rely on lots of look-up tables.

4. Analysis
In the simple case with the configuration (2, 3, 7) in the proposed scheme, storage requirements for the check-tovariable message memory module, the intrinsic memory module, and the decoded messages are 448, 126, and 42, respectively. Thus the total requirement is 616 bits as shown in Table I. In the traditional architecture, the memory requirements for the extrinsic module, intrinsic module, and the decoded messages are 504, 126, and 42, respectively. While the parallel architecture [6] is taken account, the memory requirement is increased as 840 bits; 420 bits for the R-memory module, 252 bits for the S-memory module, 126 bits for the Imemory module and 42 bits for the decoded messages. Table I shows the evaluation of memory quantity for the traditional decoder, the proposed decoder and the reference decoder [6] and proposed decoder. The total memory requirement includes memory of storing intrinsic messages, check-to-variable messages and variable-to-check messages. Note that the improvement is more obvious in the case of (4, 36, 128), i.e. the memory requirement is dramatically less than [6], about 56%. With this summary, the memory consumption is reduced by this novel parallel design.

5. Conclusions
In our paper, a partially parallel QC-LDPC decoder is proposed. All check-to-variable messages in the same block row are compacted in the efficient chunk and memory efficiency is achieved. In column-major decoding process, our decoder exactly regains each check-to-variable message from the efficient chunk by the recover unit. When each variable-tocheck message is updated, the predictor updates the efficient chunk immediately instead of storing it. The experimental results show that only 35% of traditional storage requirement is required based on (4, 36, 128) QC-LDPC codes.
