The Design of Rate-Compatible Protograph LDPC Codes

†Department of Electrical Engineering, The University of Texas at Dallas, Richardson, TX 75083-0688 USA ‡Jet Propulsion Laboratory, California Institute of Technology, Pasadena, CA 91109-8099 USA Email: nvanthuy@utdallas.edu, aria@utdallas.edu, Dariush.Divsalar@jpl.nasa.gov

Abstract— This paper presents a simple yet powerful method

for designing embedded rate-compatible families of LDPC codes.

Rate compatible codes are essential for many communication

applications, e.g. hybrid automatic repeat request (HARQ) sys-

tems, and their design is nontrivial due to the difficulty of

simultaneously guaranteeing the quality of several related codes.

Puncturing can be used to generate rate-compatible LDPC

codes, but it produces a gap to capacity that, in practice, often

significantly exceeds the gap of the mother code. We propose an

alternative method based on successively extending a high-rate

protograph. The resulting codes not only inherit the advantages of

protograph codes, namely low encoding complexity and efficient

decoding algorithms, but also cover a wide range of rates and

have very good performance with thresholds that are all within

I. INTRODUCTION
The most common way of generating rate-compatible family of codes is puncturing, i.e., starting with a low-rate mother code and then selectively discarding some of the coded bits to arrive at higher rate codes [1], [2], [3]. This approach is simple but is not free of problems [4]: Specifically, (1) the mother code is optimally designed for low rates, so higher-rate punctured codes have a wider gap to capacity; (2) the optimal low rate code structure and puncturing patterns are designed separately, which is suboptimal. Even though it has been shown that puncturing can theoretically achieve the same gap to capacity as the mother code, in existing codes puncturing has increased the gap significantly [5], [1].
In this paper, we design a rate-compatible family of LDPC codes using code extension based on protographs [6]. Code extension starts with a high-rate code (a daughter code), then lower rate codes are obtained by lengthening and expurgating the parity check matrix of the daughter code [4], [7], [8], [9]. A protograph code is an LDPC code that can be constructed from a small base matrix with a few elements. While [4], [7] are mainly designed for rate-compatible irregular LDPC codes with highly optimized framework and unstructured design, rate-compatible protograph-based codes can achieve very good thresholds with low encoder complexity as well as fast decoding [10], [11]. Several protograph-based ratecompatible codes have been produced[12], [10], [13] whose design starts with a good protograph followed by puncturing or node splitting. These methods do not guarantee to cover all possible connections in a graph structure, thus they result in higher decoding thresholds.

In our work, we use a structure involving a pre-accumulator and a punctured node, in a manner similar to [10]. However, the code structure in [10] by itself is not sufficient for our purposes, since a rate-compatible family (in the usual sense) must have the same information block size, otherwise it is not useful for, e.g., HARQ applications. To create a code that satisfies this property, we use code extension, starting with a high-rate code. The idea of code extension for designing rate compatible codes is not entirely new and has appeared in [4] in the context of irregular LDPC codes, but its application to protographs is novel and the quality of the codes that have been produced in this way is superior to other known results.
We should also mention [8] which produced rate-compatible protograph codes based on node-splitting and attaching additional accumulators, but these codes are only available for rates higher than 1/2 and furthermore their gap to capacity is higher than the codes reported in this paper.
To summarize, the main contributions of the paper are as follows. This paper gives a simple method to design a ratecompatible coding scheme for adaptive coding and HARQ applications based on protographs with a wide range of rates. Due to limited space, we only give examples of ratecompatible codes with rates from 0.375 to 2/3 but the method is general. These codes have thresholds within 0.15 dB of capacity and a linear minimum distance property [10]. As a by-product of our main results, we also produce several highrate protograph codes that do not have uniform information size. The rates of these codes range from 1/2 to 5/6 and have thresholds within 0.14 dB of capacity. Although our protograph designs are constrained within a nested structure, the resulting protograph codes still perform close to capacity and have the lowest iterative decoding thresholds among structured codes with linear minimum distance property [10] reported in the literature.
II. PROTOGRAPH CODES FOR THE AWGN CHANNEL
A. Protograph Codes
A protograph [6] is a Tanner graph with a relatively small number of nodes. A protograph code (an equivalent LDPC code) is a larger derived graph constructed by applying a “copy-and-permutation” operation on a protograph. The protograph code is the graph of a code N (copies) times as large as the code corresponding to the protograph, with the same rate and the same distribution of variable and check node degrees.

B. Computation of Iterative Decoding Thresholds
An iterative decoding threshold of a protograph is the minimum channel quality that supports reliable iterative decoding of asymptotically large LDPC codes built from the protograph. Thresholds of LDPC code ensembles can be computed by using density evolution (DE) [14] and extrinsic information transfer (EXIT) chart [15] techniques. However, the general EXIT chart cannot be applied to compute the thresholds of protograph codes due to its inability of computing thresholds of graphs that have degree-1 variable, punctured variable nodes or protographs with same degree distributions and different thresholds [16]. To solve these problems, Liva and Chiani [16] proposed the PEXIT method that is simple and allows to compute thresholds of protographs even more accurate than the method of reciprocal channel approximation (RCA) used in [10]. For example, thresholds of the AR4JA family of protograph codes [10] plotted in Figure 2 are shown in Table I. For comparison purposes, the performance of protographs using RCA [10] is also given.
The PEXIT method will be used throughout this paper.
C. Design of Improved Protograph Codes
Although protographs have a simple structure, it is not easy to find a protograph that performs close to its capacity limit and whose minimum distance grows linearly with codeword length [10]. The difficulty is due in part to the

parallel edges and punctured nodes that are often required for high-performance protographs. In order to handle all possible connections in a graph structure, an obvious solution is to use advanced optimization techniques such as simulated annealing [6] as well as differential evolution [16], [13] to search for a good protograph. However, to our knowledge, most of good protographs reported in the literature were only obtained by applying a node splitting or pre-coding technique from a given LDPC code (see [10] and reference therein).

In this section, we propose a simple approach to search for a good protograph. We concentrate on graphs that possess good protograph properties that likely have a low decoding threshold and linear minimum distance. Such graphs should have degree1 variable nodes [17], [18], a fraction of high-degree and degree-2 nodes [14] as well as linear minimum property [10]. Even though searching for graphs with these constraints is suboptimal, we will show that resulting protograph codes are very good, in fact better than existing protographs in terms of decoding threshold. IV. DESIGN OF RATE-COMPATIBLE PROTOGRAPH CODES FOR HARQ APPLICATIONS
The rate embedded codes designed in the previous section are not suitable for HARQ applications since their information block-length is not the same. There are some works on this topic using irregular LDPC codes as well as structured codes such as [4], [7], [8], [13], [9], [1], [2]. In this section, we use a similar approach, with some modifications, to design another family of embedded rate-compatible protograph codes which is suitable for HARQ applications and also perform close to their capacity limits.
The new rate-compatible code structure is constructed by extending a protograph of a higher rate code. This idea is described in Figure 4. In this figure, a rate 1/3 code is constructed from extending a protograph of a rate-1/2 code. A rate-1/4 is again extended from the rate 1/3 code. This structure forms a nested family of rate-compatible codes. This idea was used before by Jacobsen in [4] in order to design rate-compatible codes for irregular LDPC codes.

V. CONCLUSION
This paper presents a simple approach to construct ratecompatible LDPC codes based on protographs which perform within 0.15 dB of their capacity limits. The rate-compatible structure has the advantage of low encoding complexity and efficient decoding algorithm with one hardware implementation for all members of the code family.
