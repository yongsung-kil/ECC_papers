Abstract—An iterative decoding algorithm of rate-compatible punctured low-density parity-check (RCP-LDPC) codes of high coding rates is developed. This algorithm performs a predetermined recovering process of punctured bits sums at the beginning of each iteration of the standard belief-propagation (BP) decoding algorithm. By propagating messages of two punctured bits sum, this algorithm can recover much more punctured bits than the standard BP decoding algorithm. It is shown that the proposed algorithm is applicable for RCP-LDPC codes of higher coding rates with little increase of decoding complexity.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] proposed by R. G. Gallager in 1962 have been forgotten for a long time, and recently their performance is re-focused [2]. The beliefpropagation (BP) decoding algorithm is well-known as an iterative decoding algorithm for LDPC codes [1], [2]. In this paper, we call this decoding algorithm the standard BP decoding algorithm. The standard BP decoding algorithm calculates a posterior probability for each received symbol and attains performance close to the Shannon limit when LDPC codes with a large code length are used. This algorithm can be regarded as a message passing algorithm on a Tanner graph, and it processes all bits in parallel at each iteration.
For channels under time-varying environments, the codes are required to be flexible with respect to the coding rates depending on the channel states. It is desired that codes can handle with several code rates by using only a single encoderdecoder pair. It can be realized by puncturing codeword bits of a low rate code according to some pre-determined order so that we can obtain codes of higher coding rates [6], [7]. We call an original LDPC code of a low rate the mother code, and punctured codes are called rate-compatible punctured LDPC (RCP-LDPC) codes.
The recovering process of the punctured bits is important for RCP-LDPC codes under the BP decoding. A lot of studies have tried to construct a class of RCP-LDPC codes which are recoverable at high coding rates [6], [7], [11]. For constructing RCP-LDPC codes, two approaches have been considered: (1) constructing structured codes such as extended irregular repeat-accumulate codes [12] or protograph LDPC codes [11], (2) determining the location and the ordering of the punctured bits [7]. However these codes often exhibit error floor around

their mother coding rates. The recovering process is similar to the BP decoding over the binary erasure channels (BEC) if punctured bits and non-punctured bits are regarded as erased bits and non-erased bits, respectively. Therefore it is claimed in [8] that randomly constructed irregular LDPC codes with a high iterative threshold in the density evolution [4] over the BEC is desirable. Since these works assume the standard BP decoding algorithm, it is impossible to recover all the punctured bits when these bits form a stopping set [5]. If high rate codes are required and a large number of bits are punctured, it tends to be difficult to recover all of these bits. On the other hand, an improved BP decoding algorithm over the BEC has been developed [13]. By applying this algorithm to a recovering process, it may recover more punctured bits than the standard BP decoding algorithm, and high rate codes can be obtained.
In this paper, we device a new BP decoding algorithm by enhancing performance of a recovering process for the punctured bits. The recovering process, which is determined in advance, propagates messages of two punctured bits sum at the beginning of each iteration in the standard BP decoding algorithm. The proposed decoding algorithm can recover much more punctured bits than the standard BP algorithm. We show that the proposed algorithm is applicable for RCP-LDPC codes of higher coding rates with a little increase of decoding complexity.

B. Standard BP Decoding Algorithm
The standard BP decoding algorithm [1], [2] simultaneously
processes all codeword bits by calculating their a posterior
probability. Since the decoder receives no information from the channel for n ∈ P, we assume Pr(yn|xn = 0) = Pr(yn|xn = 1) = 0.5. For a parity-check matrix H, let N (m) and M(n) be the index sets of row m and column n such that Hmn = 1, respectively, i.e., N (m) = {n : Hmn = 1} for m = 1, 2, . . . , M, and M (n) = {m : Hmn = 1} for n = 1, 2, . . . , N .
The algorithm iteratively updates two messages Vm(in) (a) and Um(i)n(a) for a ∈ {0, 1} at each iteration i. The quantity Vm(in) (a) expresses the probability that a codeword bit xn is a, given by the information obtained via all the checks in M(n) \ {m}. The quantity Um(i)n(a) expresses the probability that a check m is satisfied if a codeword bit xn is a and the sum of other bits xn ∈ N (m) \ {n} are a. C. Recovering Process of Punctured Bits with BP Decoding
For a punctured bit n ∈ P and for m = 1, 2, . . . , M , the decoder will set both messages Vm(0n)(0) and Vm(0n)(1) to 0.5 at the initialization of the algorithm. For a pair (m, n) such that Hmn = 1, if there exists n , n ∈ N (m) \ {n}, in m such that Vm(in−1)(0) = Vm(in−1)(1), the standard BP decoding algorithm produces Um(i)n(a) of equal quantities for a ∈ {0, 1} and we cannot estimate the punctured bits to be either 0 or 1. We call a punctured bit n is recovered if two messages Um(i)n(0) and Um(i)n(1) are updated to biased values, i.e. Um(i)n(0) = Um(i)n(1), in Step 1-a, regardless of whether the values of these messages are correct or not [7]. This updating rule given in Eq. (1) tells us that punctured bit n will be recovered if all the incoming messages in check m, Vm(in) (a), from bits n ∈ M(n)\{n} are biased. We call check m whose punctured bits are recovered in iteration i the survived check in iteration i. In step 2, if punctured bit n receives biased messages Um(i)n(a), a ∈ {0, 1} from at least one of its checks m ∈ M(n), it is said to be recovered. As the decoding proceeds, all punctured bits have to be recovered using the information from the non-punctured bits.
This recovering process is similar to the BP decoding algorithm over the binary erasure channels if we regard punctured bits and non-punctured bits as erased bits and non-erased bits, respectively. So it is impossible to recover all the punctured bits when these bits form a stopping set [5]. In other words, it is clear that the standard BP decoding fails to recover all the punctured bits if there exists no survived checks at some step of the algorithm. If high rate codes are required, it is difficult to recover all the punctured bits since the number of punctured bits becomes large.

III. PROPOSED DECODING ALGORITHM
From the discussion in the previous section, it is clear that when the BP decoding algorithm fails to recover the punctured bits, all the checks have either no punctured bits or at least two punctured bits in N (m) for m = 1, 2, . . . , M . We call the check m containing two punctured bits in N (m) the bivalent check. By using bivalent checks to propagate the messages of two punctured bit sums, we can provide a new decoding algorithm which enhances the recovering performance of the standard BP decoding algorithm.

C. Design of Recovering Process
In this subsection, we give a systematic method for determining recovering process which enables us to recover the punctured bits by the proposed decoding algorithm. To determine this process according to an example in Subsection III-A, we first search loops in a submatrix of H indexed by the punctured bit positions. All of these loops must satisfy the following conditions in some step of the algorithm:
• All but one of rows in the loop contain two punctured bits.
• Only one of rows in the loop contains punctured bits greater than or equal to three.
For example in Subsection III-A, a set of rows 1, 2, 3 and a set of punctured bits 4, 5, 6 form a loop of length 6 satisfies the above condition, since rows 1 and 2 are bivalent checks and a row 3 contains three punctured bits. Moreover a set of rows 1, 2, 4 and a set of punctured bits 4, 5, 6 also form a loop of length 6 and satisfies the condition, since a row 4 contains four punctured bits. So the recovering process consists of two rules which can recover punctured bits 3 and 4. Moreover we implement these two rules until all the punctured bits in rows 3 and 4 are recovered and do not need to implement afterward. The punctured bits 3 and 4 will be recovered in the BP iteration 3. Therefore these two rules are implemented in the interval of BP iteration 1 to 3 and they are valid in this interval. This behavior of recovering is the same as that of erasure decoding algorithm in [13] which is an improved BP based decoding algorithm by performing additional procedures. The designing method produces a decoding rules by labeling each check, bit, and non-zero element. By employing the decoding rules produced in the labeling method, we can recover much more punctured bits than the BP decoding algorithm.

V. CONCLUSION AND FURTHER WORKS
We developed a new BP decoding algorithm by enhancing performance of the recovering process for the punctured bits. The new recovering process propagates joint messages of two punctured bits sum, and the proposed algorithm can recover much more punctured bits than the standard BP algorithm. From the simulation results, we showed that the proposed decoding algorithm is applicable for high rate RCP-LDPC codes. The gain of coding rate employing our decoding algorithm is about 0.005 – 0.015. Additional time and space complexities of our algorithm are small especially for moderate length of codes.
