Abstract—This paper presents a class of high-rate codes called low-density parity-check accumulate (LDPCA) codes. The code design is the serial concatenation of an LDPC outer code and an accumulator with an interleaver. The iterative decoding for the LDPCA code design has complexity linear to the code length. When using regular LDPC codes with column weight 2, the proposed codes have low encoding complexity and are advantageous for hardware implementation. Simulation results show that the regular LDPCA codes have the same error performance with the regular LDPC codes at the waterfall region and outperform product accumulate codes at the error floor region. The investigation on weight distributions proves that regular LDPCA codes have the asymptotic minimum distance proportional to the code length. In addition, iterative decoding thresholds under density evolution are obtained with a Gaussian approximation.

It is shown that PA codes have the best threshold, but the threshold difference between PA and LDPCA codes becomes very small at very high code rates. Compared with PA codes, the regular LDPCA codes have a larger asymptotic minimum distance proportional to the code length. We found that PA codes have a very small asymptotic minimum distance, which explains the error floor of PA codes shown in [14] and our simulation results.

I. INTRODUCTION

II. STRUCTURE OF THE LDPCA CODES

Ever since low-density parity-check (LDPC) codes [1] were rediscovered by Mackay [2] in the 1990s, many codes have been applied by various applications and standards and have attracted a great deal of research effort. Numerous works and papers are focused on encoding. One of most successful works is on cyclic LDPC codes [3], which can be encoded with a single shift-register based on the generator polynomial. The other codes with encoding efficiency include quasicyclic LDPC codes [8]. The structures of the parity-check matrices for these LDPC codes give encoding advantage in hardware implementation. In this paper, we propose a class of unstructured high-rate codes. The design model of the proposed codes is the serial concatenation of an LDPC outer code with an accumulator as the inner code through a random interleaver as shown in Fig. 1. A regular LDPC code with column weight 2 can be chosen as the outer code. Its encoding can be performed with linear or near-linear complexity by the method in [6]. Besides, we can use the serial concatenation of an accumulator and a series of single-parity-check (SPC) codes through an interleaver. The resultant outer code is equivalent to a near-regular LDPC matrix for which almost all columns have weight 2. Therefore we call the proposed codes LDPC accumulate (LDPCA) codes. Over the binary-input AWGN channel, LDPCA codes can be decoded iteratively with low complexity. It should be noted that product accumulate (PA) codes [13] were proposed with a similar idea. However, the outer code of PA codes is equivalent to a systematic lowdensity generator-matrix (LDGM) code [19] other than an

As depicted on the left side of Fig. 1, the LDPCA code design is composed of a rate-1 accumulator of the form 1/(1 + D) and an LDPC outer code concatenated through a pseudo-random interleaver. The code graph is shown on the right side of Fig. 1. The upper part is the subgraph of the accumulator, and the lower part is the Tanner graph [4] of the LDPC outer code. There is an interleaver in the middle, which is required only to avoid short cycles in the graph. If the variable nodes of the outer code are ordered properly, the interleaver can be excluded. There are two kinds of nodes in the graph representing two kinds of constraints on the bit values of the connecting edges, called variable and check nodes respectively. Variable nodes represent repetition codes such that all connecting edges should have the same binary value. Check nodes represent SPC codes such that the binary values of all connecting edges should add up to zero in modulo 2. The graph structure of an accumulator is formed by a zigzag of edges connecting weight-2 check nodes on the input side and weight-2 variable nodes on the output side. For the outer code, suppose that the regular LDPC codes have the column weight γ and row weight ρ. The weight-γ columns of the parity-check matrix put constraints equivalent to (γ, 1) repetition codes, and the weight-ρ rows put constraints equivalent to (ρ, ρ − 1) SPC codes. The edges connect these two kinds of constraint nodes in a random order to form a bipartite graph. In simulation results, we see that LDPCA codes with LDPC outer codes of γ = 2 have both of good decoding thresholds and low error floor.

IV. SIMULATION RESULTS
Since the goal of the LDPCA code design is a high-rate coding scheme with efficient encoding, the simulations are focused on the high rate codes with code lengths N > 4K. In the following simulations of Example 2 to 5, the BCJR-SPA decoding is performed with 100 turbo iterations and 1 inner iteration. For LDPC codes, the number of iterations for the SPA decoding is set to be 100. To optimize the performance of the NMS decoding, we set c = 0.9 for the inner decoder, and c = 0.83 and 0.87 for the outer decoder of LDPCA and PA codes, respectively.
Example 2: (Medium code length) Fig. 4 compares the BER and frame (block) error rate (FER) performance of the rate-3/4 LDPCA codes with that of the LDPC code. The LDPC code is a (4096, 3073) masked EG-Gallager code with γ = 4 and ρ = 16, taken from [5]. The base matrix is a 16 × 64 array of 64×64 permutation matrices constructed by EG(2,26), and the masking matrix is a 1 × 4 array of 16 × 16 circulant matrices with weight 4. There are two kinds of LDPCA codes in this figure. The first is with a QC-LDPC outer code with γ = 2 and ρ = 8. The second is with an SPCA outer code of (4, 3) SPC codes. The code length is N = 4096. The LDPCA codes have advantage by less than 0.1 dB in BER over the LDPC code.

We see in the last section that LDPCA codes have advantage in the threshold by 0.52 dB over LDPC codes at code rate 3/4. Although not shown in the figure, the LDPCA codes show no error floor at BER above 10−9. Fig. 5 shows the BER and FER of the LDPC and LDPCA codes at rate 7/8. The LDPC code is a (4096, 3585) masked EG-Gallager codes with γ = 4 and ρ = 32. The base matrix is an 8 × 64 array of 64 × 64 permutation matrices constructed from EG(2,26), and the masking matrix is a 1 × 8 array of 8 × 8 circulant matrices with weight 4. The LDPCA code has an SPCA outer code of (8, 7) SPC codes. The code length is also N = 4096. At high SNR, the LDPC codes with algebraic construction have better BER than the LDPCA codes, but the error floor of the LDPCA codes is still less obvious.
Example 3: (Long code length) Fig. 6 compares the error rates of several LDPCA and LDPC codes with longer code lengths. The (8192, 7171) and (16384, 15363) near-regular QC-LDPC codes are taken from [8], with γ = 4, and ρ = 32 and 64 respectively. Their parity-check matrices are the subarrays of a 256 × 256 array of 256 × 256 circulant permutation and zero matrices constructed by GF(257) [8]. The (8192, 7168), and (16384, 15360) LDPCA codes have the SPCA outer codes of (8, 7) and (16, 15) SPC codes, respectively. We see that the LDPCA codes have slightly better performance than the LDPC codes without an obvious error floor. Although the gain is insignificant, the LDPCA codes have a simple code design with encoding advantage.
Example 4: (Comparison with PA codes) Fig. 4 and Fig. 5 also show the error performance of PA-I codes. At code rate 3/4, the PA-I code with N = 4096 has 0.1 dB gain in BER over the LDPCA code at the water fall region. However, due to the small minimum distance, they have a very high error floor at BER = 10−7. The rate-7/8 PA-I code with N = 4096 also shows poor error floor performance with little advantage in the water fall region.

V. CONCLUDING REMARKS
We have introduced a class of high-rate regular codes with a simple code design and encoding advantage. It has been proved that LDPCA codes have the asymptotic minimum distance growing linearly with the code length. Although the threshold of LDPCA codes is higher than that of PA codes, the difference shrinks when the code rate is near 1. In simulation results we saw that LDPCA codes perform as well as LDPC codes without an obvious error floor.
