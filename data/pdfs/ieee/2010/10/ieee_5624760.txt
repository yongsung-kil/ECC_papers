Applying Transparent Lossless Data Compression to Improve the Feasibility of Using Advanced Error Correction Codes in Solid-State Drives
Ningde Xie, Guiqiang Dong, and Tong Zhang ECSE Department, Rensselaer Polytechnic Institute, USA

Abstract—As advanced error correction codes (ECC) which support soft decoding is suggested to use in the future NAND flash memories, the requirement of sensing and transferring multi-bit soft information from the flash cells to controller will incur a large read access latency. This paper proposes to exploit the lossless compressibility of files in LDPC coded NAND flash memories to reduce such latency overhead, other than saving storage space as in conventional practice. The key idea is to apply run-time lossless data compression to enable an opportunistic use of a stronger LDPC code with more coding redundancy, and trade such opportunistic extra error correction capability to allow more coarser-grained memory sensing and hence lead to less read response speed overhead without sacrificing the overall performance. Since the basic operation of NAND flash is typically realized in the unit of page (e.g., 4KB user data per page in the current NAND flash), we only apply this strategy to each individual page independently in order to be completely transparent to the firmware, operating systems and users. This paper quantitatively studies the effectiveness of this design strategy in 2bits/cell NAND flash memories. Results in the case study show that with this design strategy, up to 95.24% on-chip memory sensing latency reduction and 66.67% flash-tocontroller data transfer latency can be achieved respectively.
Index Terms—NAND flash, lossless compression, error correction code, LDPC.
I. INTRODUCTION
As the fastest growing segments in semiconductor industry, NAND flash memory is being used in ever-diverse applications to realize high-capacity, high-performance and non-volatile solid-state data storage. The continuous growth of NAND flash memory storage density has been mainly driven by aggressive technology scaling (e.g., NAND flash memory using 25nm technology has been recently reported by Intel and Micron [1]). Besides technology scaling, multi-level per cell (MLC) storage technique has been widely used to further improve NAND flash memory storage density. In current design practice, most MLC NAND flash memories store 2 bits per cell, and 3 and even 4 bits per cell NAND flash memories have been recently reported in the open literature [2]–[6]. Such continuous technology scaling and aggressive use of MLC storage have significantly reduced the bit cost and increased storage capacity per chip (e.g., above 40% reduction of bit cost per year [7] and the capacity per chip has been doubling every year since 2001). These trends, along with the advantages in terms of low power consumption, small size and no mechanical part, make solid-state drive (SSD) based

on NAND flash memory become a viable alternative to the traditional hard disk drives.
As the industry is pushing the technology scaling and use of MLC technique to their limits, raw storage reliability of NAND flash memory has been inevitably degrading, which demands an increasingly powerful error correction capability. In current design practice, BCH codes are being widely used. Besides employing stronger BCH codes, there have been a growing interest in the industry to explore other more advanced error correction codes (ECC) [8], e.g., low-density parity-check (LDPC) codes and soft-decision Reed-Solomon (RS) codes. However, since almost all the advanced ECCs require soft-decision input for their decoding, a straightforward use of those advanced ECCs tends to significantly degrade the overall SSD read response speed from two perspectives: (i) NAND flash memory sensing latency is linearly proportional to the sensing granularity, and (ii) the bus for flashto-controller data transfer has to handle a much heavier load, leading to much longer data transfer latency. Since the read response time is one of the most important metrics in SSD [9], how to most practically apply advanced ECCs in future SSD is a nontrivial issue.
This paper proposes to apply transparent lossless data compression to reduce the read latency overhead induced by using advanced ECCs in SSD. The rationale of this work is very intuitive and can be briefly described as follows. In NAND flash memory, data storage is realized in the unit of page, and each page stores a certain number of user data (typically ranging from 512 byte to 4K byte) and the associated ECC coding redundancy. If one page of user data can be losslessly compressed to a certain extent, more storage space will become available to store ECC coding redundancy, which makes it possible to use a stronger ECC with more-thanenough error correction capability for the present page. For advanced ECCs demanding soft-decision memory sensing, a stronger ECC on average requires less soft-decision precision (i.e., a relatively coarser-grained memory sensing) than a weaker ECC, given the same target decoding failure rate. Therefore, intra-page lossless data compression can enable an opportunistic use of stronger-than-normal-case ECCs, which can allow memory sensing precision reduction and hence lead to less overhead on SSD read response speed. II. BACKGROUND

A. Basics of NAND Flash Memory

Each NAND flash memory cell is a floating gate transistor whose threshold voltage can be configured (or programmed) by injecting certain amount of charges into the floating gate. The continuous growth of NAND flash memory storage density has been mainly driven by technology scaling and MLC technique, i.e., to store more than 1 bit in each memory cell (or floating-gate MOS transistor) by programming the cell threshold voltage into one of l > 2 voltage windows. Because of its obvious storage density advantage, MLC NAND flash memory has been increasingly dominating flash memory market. In current design practice, most MLC NAND flash memories store 2 bits per cell.
Before one flash memory cell can be programmed, it must be erased (i.e., its threshold voltage is set to the lowest voltage window). A tight threshold voltage control is typically realized by using program-and-verify approach with a stair case program voltage Vpp [10]. Because NAND flash memory should sweep the entire operational voltage window using this program-and-verify procedure, the NAND flash memory write latency tends to be inversely proportional to the program step voltage ΔVpp. Ideally, threshold voltage distributions of different storage states should be sufficiently far away from each other to ensure a high raw storage reliability. In practice, due to various effects such as background pattern dependency, noises, and cell-to-cell interference [11], the threshold voltage distributions may be very close to each other or even overlap, leading to non-negligible raw bit error rates. In the following, we present an MLC cell threshold voltage distribution model that will be used for quantitative performance evaluation and comparison in this work. The erase state tends to have a wide Gaussian-like distribution [12], i.e., the probability density function (PDF) of the threshold voltage distribution can be approximated as

B. SSD Read Access Latency

An SSD contains one controller and multiple NAND flash chips. SSD read access latency includes the on-chip NAND flash sensing latency, flash-to-controller data transfer latency, and the latency of ECC decoding on the controller. The on-chip NAND flash sensing is carried out in a fully serial manner, hence its latency is linearly proportional to the memory cell sensing levels. The on-chip hard-decision sensing latency of 2bits/cell NAND flash memories is typically few tens of μs, e.g., 20μs. Therefore, to support a soft-decision n-bit per cell memory sensing in 2bits/cell NAND flash memory, the on-chip memory sensing latency will increase by (2n − 1)/(22 − 1) times, e.g., from 20μs to 207μs when n equals to 6. Note that for the page-based read operation in NAND, reference voltage has to be moved to all possible boundaries instead of carrying out binary tree search which can only be used if sensing one cell.
In current design practice, most NAND flash memory chips support an 8-bit I/O bus with clock frequency of up-to 100MHz. Assuming the NAND flash chip connects with the SSD controller through a 100MHz 8-bit I/O bus, it will take at least 41μs to transfer one 4KB page from NAND flash chip to controller in the case of hard-decision memory sensing. Similarly, in order to support a soft-decision n-bit per cell memory sensing in 2bits/cell NAND flash memory, the flashto-controller data transfer latency will increase by n/2 times, e.g., from 41μs to 123μs when n equals to 6. The latency of ECC decoding on the controller depends on the achievable decoding throughput, e.g., assume the decoder can achieve a throughput of 4Gbps, the decoding latency can be around 16μs when the page size is 4KB. The above discussions clearly show that the support of fine-grained soft-decision memory

III. PROPOSED DESIGN STRATEGY
One of the most prevailing real-life applications of data compression [13] is to increase the effective storage capacity of mass data storage media such as conventional hard disk drives and SSD. However, compared with lossy data compression being universally used in multimedia file storage, lossless data compression is much less popular for two main reasons: (i) a compressed file may not be easily modified and managed on the storage media, and (ii) since the lossless data compression ratio1 can rarely be much better than 2:1 in contrast to lossy data compression ratio of over hundreds, there is less interest and incentive in using lossless data compression to save storage space. As a result, except multimedia data files, many files stored on mass data storage media today are not compressed at all although they are losslessly compressible, such as web pages, text files, and software and system files, etc. This observation directly motivates the basic idea of this work, as illustrated in Fig. 2: We can apply runtime transparent lossless data compression to opportunistically enable the use of a stronger ECC. Its stronger-than-necessary error correction capability can be traded to allow coarsergrained memory sensing, which can directly improve SSD read response speed. We note that lossless decompression only involves simple table lookup and can be done on-the-fly, hence it does not induce any noticeable latency overhead on SSD read operations.
Since compression ratio of lossless data compression heavily depends on the data length, it can be very difficult to achieve a large compression ratio such as 2:1 within each individual page. Fortunately, as demonstrated in the following case study, the compression ratio may not necessarily be a concern in practice. Different from the conventional use of lossless data compression that aims to improve effective storage capacity and hence should achieve a compression ratio of close or even larger than 2:1, a very small compression ratio can provide enough extra storage space for a sufficiently stronger ECC. For example, a compression ratio of 1.11:1 can reduce the ECC code rate from 19/20 to 6/7, which represents a very large coding gain and hence can potentially enable a coarser-grained memory sensing. Therefore, this can drastically simplify the implementation of intra-page transparent lossless data compression. Moreover, current NAND flash memories have reasonably large page size, e.g., most NAND flash memories use 4K-byte user data per page. Such relatively large page sizes make it more feasible to achieve a small compression ratio within each page at reasonable implementation cost.
1In this work, compression ratio is defined as the ratio of the data length before compression over the length after compression, hence it should be greater than one.

Let Rn denote the code rate of the normal ECC used by SSDs without using any intra-page transparent lossless data compression, i.e., the same rate-Rn ECC is used to protect all the pages of uncompressed original user data as in current design practice. If one page of user data can be compressed with a compression ratio of rc, we could accordingly reduce the code rate from Rn to Rn/rc, leading to more redundant space with the same codeword length. As a result, we can leverage such run-time opportunistically extra redundancy to reduce the NAND flash memory sensing precision while maintaining the same overall storage system reliability. Ideally, we may expect that the code rate (and hence error correction capability) can be gracefully adjusted with a very fine granularity in order to maximize the potential benefit. However, such ideal scenarios with fine-grained configurability and adaptability may largely complicate the overall system design and incur non-negligible implementation overhead. Therefore, in this work, we consider the simplest and most practical scenario where only two different ECC code rates are allowed. Let Rn and Rs (where Rn > Rs) denote these two code rates, where Rn is the rate of the normal ECC when NAND flash memories do not use any data compression and Rs is the rate of a stronger ECC enabled by opportunistic run-time lossless data compression. In particular, we have
• If rc ≤ rt, then we use the normal rate-Rn code to protect the uncompressed user data. Accordingly, NAND flash memory chip will use a full-precision memory sensing when this page is being read.
• If rc > rt, then we use the powerful rate-Rs code to protect the compressed user data. Accordingly, NAND flash memory chip will use a reduced-precision memory sensing when this page is being read.
Let qf and qr (where qf > qr) represent the memory sensing quantization bit number in full-precision and reducedprecision memory sensing, respectively. In practice, designers may have to carry out extensive representative simulations to evaluate this involved trade-off and select the optimal rt leading to the maximal average SSD read response speed. V. CONCLUSIONS
This paper demonstrates the potential of using lossless data compression to reduce the bus load in the LDPC coded NAND flash systems. This work has a two-fold motivation: (i) lossless data compression is not being widely used today in mass data

storage systems such as NAND flash memories, leaving many losslessly compressible files are stored uncompressed, (ii) the basic read/write unit in NAND flash is equal-sized pages, and modern NAND flashes employ increasingly powerful ECCs to protect each page data to ensure the storage integrity, while a stronger ECC needs more storage space to store coding redundancy. The underlying idea of this work is to apply run-time intra-page lossless data compression to enable an opportunistic use of a stronger LDPC code without incurring any storage space overhead, and trade such more-than-enough error correction capability for a coarsely sensing of soft information. To evaluate and demonstrate this design strategy, we carried out a case study and the results show that with this design strategy, up to 95.24% on-chip memory sensing latency reduction and 66.67% flash-to-controller data transfer latency can be achieved respectively.
