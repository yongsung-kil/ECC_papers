Abstract
Since high-density flash memory has high error rate, strong error control is necessary for the solid-state drive (SSD). The number of erasure cycles of each memory cell is limited, where the cell should be erased before writing. Wear-leveling is used for leveling the erasure cycles in a flash memory. Since the existing wear-leveling is executed in a chip, it is not effective if write operations are concentrated into specified chips.
This paper proposes wear-leveling and error control method by using redundant flash memories in order to improve reliability and lifetime of the SSD. In the proposed method, error control is usually executed, and wear-leveling among the chips is executed when the bias in the erasure cycles is large. The execution frequency of wear-leveling is adjusted considering deterioration of the cell. Evaluations of bit error rate and lifetime show that the proposed method has high reliability and durability
Keywords
solid-state drive (SSD); flash memory; wear-leveling; error control code; ifetime prolongation;
I. INTRODUCTION
Storage capacity of solid-state drive (SSD) which has a large number of flash memories is rapidly increasing in recent years; and therefore, the SSD can replace a hard disk drive (HDD). The SSD is mainly used in mobile devices because of fast random access, low power consumption, and high shock resistance [1]. Also, power consumption of the large scale storage servers can be greatly reduced using the SSD. On the other hand, since high-density flash memory tends to have errors within its data, we are concerned that the reliability of the SSD which has a large number of flash memories decreases. In order to improve the reliability of the SSD, a two-level error control code and a three-level error control code which have more strong error control capability than the conventional error control code have been proposed [2]. Also, flash memory cells have a limited number of erasure cycles. If many writes concentrate into the same part, the memory cells in the part are broken and become unable to store the data [3]. Wear-leveling which levels the erasure cycles in a flash memory and prolongs lifetime of the memory is used in a flash memory [4]. That is, the conventional wear-leveling is the one among memory cells in a chip. However, the cells are broken and lifetime of the memory is shortened if write operations concentrate into a specific memory chip.
This paper proposes wear-leveling and error control method by using some flash memories in the SSD. In the proposed method, error control is usually executed, and wear-leveling among the chips is executed when the bias in the erasure cycles is large in order to improve reliability and lifetime of the SSD. The proposed wear-leveling is the one among memory chips in a SSD.
This paper includes 7 sections. Section 2 describes the performance and structure of the SSD and Section 3 describes the structure of a flash memory, wear leveling, and the factor of errors. A two-level error control code is illustrated in Section 4. Sections 5 and 6 shows the proposed method and its evaluation, respectively. This paer is conluded by Section 7.
II. SSD MEMORY CHARACTERISTICS
The SSD is a storage which has multiple flash memories. Since the SSD excels in power consumption, shock resistance, and access speed, it replaces the HDD in mobile devices recently. The performance comparisons between SSD and HDD are listed in Table I. The general structure of the SSD is shown in Figure 1. The SSD has a host interface which constitutes various physical interfaces, such as SATA, Fibre Channel, PCI Express, and USB, in order to communicate with the exterior. CPU mainly executes the reading and writing to the flash memories, and wear-leveling. Also, RAM stores the information of wear-leveling and address mapping, and they are periodically backed up into the flash memories. The flash controller controls addressing, writing, erasure, and reading. The data are written into each flash memory in parallel by multiple channels, where the channel means a transmission way which connects the flash controller and each flash memory.

III. FLASH MEMORY A. Structure and characteristic
NAND flash memory is generally used for SSD, and it has a page structure as shown in Figure 2. Each page has a storage capacity of (k + r) × b bits; the information part is k × b bits, and the check part for error control is r × b bits. The storage area of NAND flash memory is constituted of a large number of pages. The writing and reading of data are executed by the page, and the erasure of data is executed by the block which is constituted of a large number of pages [8]. Let the number of blocks and the number of pages within a block be S and T respectively. For example, 4Gb flash memory has the parameters of k = 2048, r = 64, b = 8, S = 4096, and T = 64 [6].
Also, since flash memory isn’t fully bit-alterable, the update data are written after erasing the data when the data are updated. The value of flash memory cell is determined by the quantity of electric charge which is stored within a floating gate. Single level cell (SLC) flash memory has 2 charge levels according to the quantity of electric charge. On the other hand, multi level cell (MLC) flash memory has multi charge levels, such as four, eight, etc. The concept of SLC and MLC is shown in Figure 3.
The number of erasure cycles is limited for each memory cell. Therefore, if updating or erasing the data is repeated, the cell deteriorates gradually. Since the deteriorated cell has a high bit error rate (BER), it is hard to store the correct data. SLC can endure 100K erasure cycles and MLC can endure 10K erasure cycles [9].
B. Wear-leveling In the flash memory, if many writes concentrate into the same block, the block is broken earlier than other blocks. The
solution for this problem is wear-leveling which is executed in the flash memory. The general method of wear-leveling is as follows: The data which are updated frequently are moved to the block which has small erasure cycles. On the other hand, the data which are hardly updated are moved to the block which has large erasure cycles [10]-[12]. Since the erasure cycles are leveled by using wear-leveling, it becomes possible to prolong the lifetime of flash memory. However, it is hard to level the erasure cycles perfectly because it is difficult to catch the characteristics of the actual write data and the conventional wear-leveling is the one among memory cells in a chip.

C. Factor of errors High-density flash memory tends to have errors within its data easily, and the error factors are as follows. • The quantity of electric charge of the adjacent cells is disrupted by reading and writing to a selection cell, and disturbance errors are caused [6]. • Soft error probability will increase when deep-submicron design is employed in chip fabrication [13]. • Read errors are caused by narrowed interval of charge levels in MLC [14].
In order to correct these errors, the error control code is applied in each page. Also, in order to improve reliability further in the SSD, a two-level error control code which constitutes error control code within a page and among the pages has been proposed [2].
IV. TWO-LEVEL ERROR CONTROL CODE In this method, the SSD has an array of N flash memories and one of them is used for storing parity-check bits [2]. Single erasure correcting parity-check code is constituted among the pages and random bit error correcting code is applied within each page. The coding procedure is shown below. 1) The input data D are divided into N − 1 segments as D = (D0, D1, · · · , DN−2), where Di(0 ≤ i ≤ N − 2) is a
binary b × k matrix. 2) Generate the parity-check segment as DN−1 = D0 ⊕ D1 ⊕ · · · ⊕ DN−2 , where DN−1 is a binary b × k matrix and
⊕ means matrix addition over GF (2). 3) For i ∈ Z0N−1 = {0, 1, · · · , N − 1}, encode Di by the random bit error correcting code and b × (k + r) matrix W = i
is generated. 4) For i ∈ Z0N−1, write Wi into the i-th flash memory. The above-mentioned coding method is shown in Figure 4. Even if errors cannot be corrected within the page and can be only detected, one page is restored by using parity-check code among the pages.

V. PROPOSED METHOD This section propose a wear-levelig among chips in a SSD. The conventional wear-levellig methods are the ones among memory cells in a chip. The proposed method uses check data among chips for wear-levelig or error control flexibly. The SSD has an array of N flash memories, and one of them is used for wear-leveling among the chipss and error correcting among the pages. Since error correcting code among the pages requires having at least one page erasure correction, the two-level error control coding method is used in the proposed method. Wear-leveling among the chips is as follows: First, each block-erasure cycles is compared before the data are written into flash memories. Second, the data aren’t written into a flash memory whose block has large erasure cycles if the bias in erasure cycles is large. In the case of reading out, the data which was not written into is restored using the one page erasure correction capability. In this case, although error control capability falls, erasure cycles can be leveled in the whole of the SSD. The write-in algorithm of the proposed method is obtained from the two-level error control coding procedure. The algorithm is shown below. 1) The input data D are divided into N − 1 segments as D = (D0, D1, · · · , DN−2), Di(0 ≤ i ≤ N − 2) is a binary
b × k matrix. 2) Generate the parity-check segment as DN−1 = D0 ⊕ D1 ⊕ · · · ⊕ DN−2 , where DN−1 is a binary b × k matrix and
⊕ means matrix addition over GF (2). 3) For i ∈ Z0N−1 = {0, 1, · · · , N − 1}, encode Di by the random bit error correcting code and b × (k + r) matrix W = i
is generated. 4) Read out the erasure cycles of write-in blocks in each flash memory. If the difference between the maximum value
of them and the minimum value of them exceeds α, let the number of the flash memory which has a maximum block-erasure cycles be j-th, then go to 5. Otherwise go to 6. 5) For i ∈ Z0N−1 - { j }, write Wi into the i-th flash memory. 6) For i ∈ Z0N−1, write Wi into the i-th flash memory. The above-mentioned algorithm is shown in Figure 5. The value α which judges wear-leveling execution is enlarged according to increase of erasure cycles. This is for reducing the rate of wear-leveling execution according to degradation of a flash memory cell, and heightening the error control capability.
