EFFICIENT ENCODING FOR HARDWARE IMPLEMENTATION OF IRA LDPC ON 802.16 STANDARD
Trio Adiono, Agi Prasetiadi, Anugrah Salbiyono
School of Electrical Engineering and Informatics Institut Teknologi Bandung, Indonesia

ABSTRACT
We propose an encoding scheme for quasi-cyclic low-density par­ ity check (QC-LDPC) suitable for LDPC codes in IEEE 802.l6e. We developed accumulated recursive formula to calculate parity bits, so we can encode the message in real time. Basic idea from our method is, use first parity vector recursively to produce another parity vector efficiently. While parity vector is chosen as an output, it also saved in another register for being used in next operation. An encoder architecture is proposed and implemented to verify the results. Later, we discuss the hardware simulation results.
1. INTRODUCTION
Low-density parity check (LDPC) [ 1] codes has received tremen­ dous attention from channel coding communities recently and se­ lected as a channel coding in several commercial standards, such as IEEE 802.l6e [2], IEEE 802. 1 1n [3], and DVB-S2 [4]. By care­ fully designing an LDPC codes, error correcting capabilities from LDPC can achieve near Shannon limit by a fraction of dB [5]. De­ spite its powerful performance, LDPC has a major issue in encod­ ing process [6]. Encoding of LDPC codes involve matrix multi­ plication that has quadratic complexity in code length [7]. Several methods has been presented to overcome this problem.
Lin et al. [8] suggest using parity bit prediction and correc­ tion to break up the data dependency within the encoding pro­ cess. Myung et al. [9] proposed an efficient encoding by break-up parity-check matrix and dividing encoding process in four steps. Yoon et al. [ 10] generates arbitrary parity bits, then given the parity bits for the first sub-block and exploiting the dual-diagonal struc­ ture, all parity bits are found through correction.
In this paper, we modified approach in [8] by recursively cal­ culating parity bit from the starting parity vector. By placing all message bit in one place, we can calculate the parity value by ex­ panding each recursive formula into specified XOR function. The advantage of our proposed scheme is, while starting parity vector is being chosen as the output, it also saved into another register for next operation. We will illustrate the encoding process by us­ ing codeword of 2304 bits and rate 2/3 B from [3]. Later, we will present and discuss the synthesis results.
This paper is outlined as follows. Section II explains the basic idea about the IRA LDPC encoding algorithm. Section III shows the LDPC implementation on hardware. And section IV notes the critical path and area. Finally, section V brings the final conclusion of this paper.

S. CONCLUSION
1. We can calculte parity on IRA LDPC encoder based on vec­ tor method fast and efficiently instead of using real matrix storage system.
2. Straight calculation of the first parity block ro doesn't give any significant difference of performance in each expan­ sion or rate. In other words, without having to make a pre­ dictional parity, we still can get good performance on the encoder.
3. The area that used to implement this design is growth lin­ early comparing to the number of expansion.
