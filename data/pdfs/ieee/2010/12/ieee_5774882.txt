Design of Turbo Decoder Based on Min-Sum Decoding Algorithm of LDPC Code
Pengjun Wang, Fanglong Yi
Institute of Circuits and Systems, Ningbo University Ningbo 315211, China
wangpengjun@nbu.edu.cn flyxiangshan@163.com

Abstract—Turbo code has been used by multiple communication standards due to its performance near Shannon limit. However, its decoding algorithm is complicated. Through the research on Turbo code, a novel design of Turbo decoder is presented in this paper. Turbo code is decoded by the Min-Sum decoding algorithm of LDPC code, which is a low complicated decoding algorithm. Moreover, messages are stored compressively, and the check-to-variable messages and the variable-to-check messages are stored alternately. This design reduced the required memory significantly. Finally, the decoder is simulated by ModelSim SE6.0 and the simulation results show that the decoder has high decoding speed.
Keywords—Decoder, Turbo, LDPC, Min-Sum
I. INTRODUCTION
Communication is intended to send the information from source to sink efficiently, reliably and in time. But a variety of noise and interference introduced by the channel deteriorate the reliability of communication. Under the interference of noise, how to send the information efficiently and reliably is a key issue of communication systems. Therefore, researchers have done a thorough research on channel coding. A lot of great coding techniques have been invented, such as Gray code, Hamming code, cyclic code and BCH code. However, their performance is limited, which are difficult to achieve the goal of near Shannon limit [1]. In 1993, C.Berrou presented a novel coding scheme — Turbo Code, which is based on convolutional code and concatenated code [2]. Turbo code is a kind of error control coding techniques which has strong error-correcting capability and attracts much attention due to its performance near Shannon limit. So Turbo code is widely used in mobile satellite communication, deep space communication, digital video broadcasting, UMTS/3GPP, etc. Furthermore, it has been become the error-correcting code standard of the third-generation mobile communication systems.
Nowadays, the most sophisticated and efficient decoding algorithms of Turbo code are Bahl Cocke Jelinek Raviv (BCJR) algorithm, Maximum a posteriori (MAP) algorithm [3], Log_MAP [4], Max_Log_MAP [5], Soft-Output Viterbi Algorithm (SOVA) [6], etc. Although they all have excellent decoding performance, they are not suitable to decode by

hardware because of their high computational complexity, severe time delay and low throughput.
Through the research on Turbo code, it is found that the check matrix is very sparse [7] and Turbo code is a special case of Low Density Parity Check [8] (LDPC) code. Therefore, a novel design of Turbo decoder is presented in this paper with low-density feature. Turbo code is decoded by Min-Sum decoding algorithm of LDPC code, which is a low complicated decoding algorithm. Moreover, messages are stored compressively, and the check-to-variable messages and the variable-to-check messages are stored alternately. So the required memories are reduced greatly. The simulation results show that the decoder has high decoding speed.
The rest of the paper is organized as follows. Section II introduces the check matrix of Turbo code and Min-Sum decoding algorithm of LDPC. Section III shows the proposed decoder structure. Section IV presents the performance and analysis, and conclusion is drawn in Section V.
II. DECODING ALGORITHM
Min-Sum decoding algorithm of LDPC code is a low complicated decoding algorithm, which is a Message Passing Algorithm (MPA) based on Tanner graph [9]. Tanner graph establishes the corresponding relationship between check matrix H and bipartite graph. There are two types of nodes in Tanner graph, called check nodes and variable nodes (or bit nodes). VV={v1, v2, …, vn} denotes the set of variable nodes, the number of variable nodes is equal to the number of H columns. VC={c1, c2, …, cm} denotes the set of check nodes, the number of check nodes is equal to the number of H rows. A check node j is connected to a variable node i if Hji=1. One such graph of a regular (10,3,6) LDPC code is shown in Fig. 1.

IV. SIMULATION AND ANALYSIS
Turbo code is decoded by Min-Sum decoding algorithm of LDPC code in Additive White Gaussian Noise (AWGN) channel, its rate is 1/3, the length of interleaver is 512, and the component code is RSC code whose generator polynomial is g=(1,(1+D+D3)/(1+D2+D3)). The simulation results are shown in Fig. 6. Fig. 6 shows that the performance of Min-Sum decoding algorithm is worse than the performance of traditional Turbo decoding algorithms, because of the presence of short cycle in check matrix of Turbo code and short block length. And the performance under different number of iterations is different. With the increasing of the number of decoding iterations, the performance of Min-Sum decoding algorithm is closer to the performance of SOVA decoding. Moreover, the performance of 20 iterations is near to the performance of 25 iterations, and the performance of 15 iterations is poor. In order to get higher throughput and better performance, the maximum number of iterations is set to 20.

V. CONCLUSIONS By using the convenience of Min-Sum algorithm for LDPC code and coupled with the advantages of Turbo code and LDPC code, a Turbo decoder based on Min-Sum decoding algorithm of LDPC code is designed in this paper. The decoder reduces greatly decoding complexity, has a high

throughput and is favorable to hardware implementation, though the performance declined slightly. The decoder was simulated by ModelSim SE6.0 and the throughput can achieved up to 60.7 Mbps at 100MHz. And the decoder was synthetized by Synopsys in SMIC 0.18 μm, totally 24.5K logic gates and 75K memory bits are cost.
