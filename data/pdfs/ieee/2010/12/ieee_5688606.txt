Low-Complexity High-Performance LDPC Decoder for CMMB

Taiqiu Tan, Yubai Li, Chang Wu
DSP Lib, School of Communication and Information Engineering, University of Electronic Science and Technology of China Chengdu, China tantaiqiu@yahoo.cn

Abstract—A low-complexity semi-parallel low density parity check (LDPC) decoder for China Mobile Multimedia Broadcasting (CMMB) system is designed based on the structure of LDPC codes. The designed decoder utilizes turbo decoding message passing algorithm. A new memory compressing strategy for message updating was proposed, which can reduce more than 50% memories. By using them appropriately, two different code rates can reuse memories without increasing any hardware resource. The decoder is implemented on FPGA of Altera Stratix II. The throughput of the LDPC decoder is up to 40Mbps under CMMB work point, which can fulfill the request of real time decoding.
Keywords-CMMB; LDPC; message passing; storage; memory compressing
I. INTRODUCTION
Low-Density Parity Check (LDPC) [1] codes and turbo codes are among the best codes that operate near Shannon limit [2]. When compared to the decoding of turbo codes, LDPC decoders require simpler computational processing and they are more suitable for parallelization, low implementation complexity, and low latency. Although it has many advantages, researchers didn’t pay more attention to it until Mackay and Neal rediscover it [3]. LDPC codes are considered for error correction coding in next generation digital video broadcasting (DVB-S2), digital television terrestrial multimedia broadcasting (DTMB) and china mobile multimedia broadcasting (CMMB), which was proposed in 2006 [4].
There are three architectures for LDPC decoder, fully-serial, fully-parallel and semi-parallel [5]. Fully-serial decoder is the simplest one, but its throughput is low, can not fulfill real-time system request. Fully-parallel decoder has high decoding speed, but the decoder designs suffered from complex interconnect issues. As the codeword length increase, the hardware complexity will increase greatly. For instance, a fully-parallel decoder with codeword length of 1024 bits, which decoding speed can be up to 1Gbps, but it also consumes more than 1.7M gates [6]. Semi-parallel structure is a compromise between the architectures before, and it can be design flexibly according to the degree of parallelism.
LDPC code in CMMB is a kind of High-Structured (HS)
code. In this paper, based on this special structure of H matrix,

a low-complexity LDPC decoder for CMMB was designed. To improve decoding efficiency, memory compressing strategy is used for message updating. So that LDPC decoder can be realized with very low complexity. Whole design was synthesized and routed on Stratix II EP2S180 of Altera Corporation. Highest throughput achieves 40Mbps.
The rest of the paper is organized as following: Section II presents the LDPC decoding algorithm based on Turbo decoding message passing (TDMP) mechanism and offset minsum algorithm. The low-complexity high-performance decoder architecture is proposed in Section III. Implementation results as well as discussion is given in Section IV. Section V concludes the paper.
II. DECODING ALGORITHM OF LDPC
A binary (N , K ) LDPC code is a linear block code of
codeword length N and information block length K that can be described by a sparse M × N parity check matrix where M denotes the number of parity check nodes. LDPC codes
can be decoded by the Gallager’s iterative two-phase message passing algorithm (TPMP) which involves check-node update and variable-node update as two phase schedule. Check-node
updates will calculate ϕ (x) function, so there are various
approximate algorithms, and widely used are sum of products (SP), min-sum (MS), and Jacobian based BCJR. Mansour and Shanbhag [7,8] introduced the concept of turbo decoding message passing (TDMP), which is sometimes also called as layered decoding. TDMP algorithm updates check-node and variable-node at the same time, so the iteration number can be reduced. TDMP offers 2x throughput and significant memory advantages when compared to TPMP.
In TDMP, the array LDPC is divided into s block rows,
and each block row has d rows. After the check-node
processing is finished for one block row, the messages are immediately used to update the variable nodes, whose results are then provided for processing the next block row of check nodes. TPMP process all check nodes first and then compute
the variable node messages. Each decode iteration has s
number of sub-iterations in TDMP. In the beginning of the decoding process, variable messages are initialized as channel values and they are used to process the check nodes of the first

A. Memory Resource Analyse
Compared to TPMP algorithm, our decoder can save 57.3% memory. There are two reasons: The first one is that we just need store 9216 variable node messages, but TPMP will store 9216h3 variable node messages. The other reason is that we can compress check node messages by memory compressing strategy. For each row, we just need store the first minimum value, the second minimum value and the index of the first minimum value. But TPMP will store all check node messages, that will be 4608h6 messages. B. Throughput
TDMP is able to reduce the number of iterations required by up to 50% without performance degradation when

compared to the standard message passing algorithm, because it update check node and variable node at the same time.
With extensive simulation, it can fulfill the system request
when we set MAX _ it to 7 in CMMB standard work point.
But in fact, we set it to 8 in case some unexpected situation. The system clock is 100MHz, input need 288 clock cycles, output need 144 clock cycles. Once iteration will update 4608/32 (or 2304/16 for rate 0.75) sub-matrix, and each submatrix need 19 clock cycles. So the system throughput is:
9216/(288+144+4608/32h19h8)h100 = 41.2Mbps (6)
The system limit bandwidth is: 138240 h 4/25ms=22.2 Mbps, the symbol number of a time slot is 138240, a symbol maps to 4 bits in 16QAM modulate mode, a time slot is 25ms. The throughput is almost twice limit bandwidth, so our decoder can meet the requirements completely.
V. CONCLUSION
In this paper we focus on the LDPC decoder for CMMB. Based on the turbo decoding message passing mechanism, a low-complexity high-performance LDPC decoder is designed; and memory compressing strategy is adopted to improve efficiency. Whole design is synthesized on Stratix II EP2S180 device. Compared to TPMP, the decoder can save 57.3% memory resource and the maximum throughput is up to 40Mbps.
