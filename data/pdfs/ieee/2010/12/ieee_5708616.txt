Low Complexity Decoding Algorithm of QC-LDPC Code

Fanglong Yi, Pengjun Wang

Institute of Circuits and Systems, Ningbo University Ningbo 315211, China
e-mail: flyxiangshan@163.com, wangpengjun@nbu.edu.cn

Abstract—Through the research on the decoding algorithm of low-density parity-check (LDPC) code, a low complexity decoding algorithm suitable for quasicyclic LDPC (QC-LDPC) code is proposed in this paper. The algorithm adopts layered decoding (LD), and the sub-minimum is replaced by the sum of an optimal correction factor and the minimum during the process of the check nodes update. Therefore, it is not required to find the sub-minimum and the computational complexity reduces significantly. Matlab simulation results show that the decoding performance of this algorithm is very near to min-sum (MS) algorithm, and more superior than single min-sum (SMS) algorithm.
Keywords- LDPC; decoding algorithm; layered decoding; optimal correction factor
I. INTRODUCTION
Communication is intended to send the information from source to sink efficiently, reliably and in time. But a variety of noise and interference introduced by the channel deteriorate the reliability of communication. Under the interference of noise, how to send the information efficiently and reliably is a key issue of communication systems. It is importance for communication systems to have adequate means for detection and correction of errors in the information received over communication channels. Therefore, researchers have done a thorough research on channel coding and a lot of great coding techniques have been invented, such as Gray code, Hamming code, Cyclic code and BCH code. However, their performances are limited and difficult to achieve the goal of near Shannon limit. Low-density parity-check (LDPC) code was firstly invented by Gallager in 1962 [1], but it was ignored for about thirty years due to backward technology at those times. In 1996, Mackay and Neal rediscovered LDPC code and showed its excellent performance and extremely practical value [2]. Nowadays, LDPC code has engaged a great deal of attention and been considered for many communication standards such as the next generation digital video broadcasting (DVB-S2), IEEE 802.16e and IEEE 802.11n [3].
Quasi-cyclic LDPC (QC-LDPC) code is a special kind of LDPC code, whose generator matrices and parity-check matrices have a character of quasi-cyclic. QC-LDPC code has excellent performance and is well suited for hardware implementation owing to its regular parity-check matrix [4]. Moreover, the QC-LDPC code can be encoded in the way of shifting to reduce encoding complexity. In addition, the

quasi-cyclic parity-check matrices can reduce the routing complexity of the decoder. Thus, QC-LDPC code is being widely used in communication standards [5].
So far, researchers have proposed many decoding algorithms of LDPC code. Out of all these algorithms, best results are achieved with belief propagation (BP) decoding [2], but it is very difficult to implement by hardware. Through the research on the decoding algorithms, a low complexity decoding algorithm of QC-LDPC is proposed in this paper. The computational complexity of check nodes update reduces significantly, and is lower than min-sum (MS) algorithm [6], but its performance is very close to the MS algorithm.
The rest of this paper is organized as follows. Section II summarizes many decoding algorithms of LDPC code. Section III proposes a novel and low complexity decoding algorithm. Section IV gives the computational complexity and the performance. Finally, section V concludes the paper.

II. DECODING ALGORITHM OF LDPC CODE

LDPC code is a linear block code defined by a very sparse m×n parity-check matrix H. On the other hand, Tanner graph establishes the corresponding relationship between parity-check matrix H and bipartite graph [7], which directly reflects the structure of parity-check matrix H. There are two types of nodes in Tanner graph, called
check nodes and variable nodes (or bit nodes). VV={v1, v2, …, vn} denotes the set of variable nodes, and the number of variable nodes is equal to the number of H columns. VC={c1, c2, …, cm} denotes the set of check nodes, and the number of check nodes is equal to the number of H rows. A check node j is connected to a variable node i if Hji=1. Heo explained the effect of normalization factor by the theory of density evolution [11]. V. CONCLUSION
On the premise of great performance, MSNMS algorithm is proposed, which is a low complexity decoding algorithm and is suitable for QC-LDPC code. During the process of check nodes update, the optimal correction factor adds the minimum to replace the sub-minimum to reduce the complexity. Finally, the Matlab simulation results show that the algorithm has good performance.
