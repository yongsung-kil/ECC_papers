Abstract-In this paper, a multi-rate Low Oensity Parity 
Check 
(LOPC) 
decoder 
for China 
Mobile 
Multimedia 
Broadcasting (CMMB) is proposed. The architecture of 
proposed decoder only utilizes a parallel with 2 for 112 rate 
and 1 for 3/4 rate can meet the throughput requirements for 
the CMMB standard. Various techniques are employed to 
significantly reduce memory and logic resources, including a 
special quantization scheme, solution for memory collision, 
the reuse properties of layered decoding, compressing of 
check-to-variable messages and the layered algorithm, which 
lead to a reduction of memory and logic resources by a half. 
I. INTRODUCTION 
Low-density parity check (LOPC) code is a kind of 
forward error correction code (FEC) which has a very sparse 
parity check matrix. It shows excellent performance and can 
approach the Shannon limit. It was first discovered by 
Gallager in I960s [1]. However, it made a little impact 
because 
of 
the 
computational 
complexity 
and 
storage 
requirements. 
As 
the 
development 
of 
modern 
VLSI 
technology, it attracted much attention due to its high bit error 
rate performance and potential. 
Oeveloped and specified by the State Administration of 
Radio, Film, and Television of China [3], China Mobile 
Multimedia 
Broadcasting 
(CMMB) 
is 
one 
of 
the new 
standards for 4G mobile communication. CMMB adopts a 112 
and a 3/4 code rate LOPC codes for FEC. To implement a 
multi-rate decoding, in this paper, we adopt an architecture 
which 
is 
capable 
of 
handling 
two 
H-matrices. 
This 
architecture allows a less parallel and a faster convergence. 
We will detail a particular mechanism to avoid data collision. 
Moreover, Quasi-Cyclic LOPC (QC LOPC) codes are well 
suited for hardware implementation because of the regularity 
in their parity check matrices. Here, several classes of 
QC-LOPC codes [4-5] have been proposed that can achieve 
comparable 
performance with equivalent random LOPC 
codes. However, LOPC codes in CMMB are a kind of HS 
(High-Structured) codes, but they are not QC-LOPC codes. 
Thus, we try to convert the HS (High-Structured) codes to 
QC-LOPC codes. We have implemented the decoder by 
adapting the converted LOPC codes. 
In the paper [6], by using layered algorithm can not store 
the variable nodes messages, which lead to a large reduction 
of memory. For the CMMB LOPC decoder, the total memory 
used can be reduced by 37.5%. In the paper [7], by using a 
compressed form to store check-to-variable messages, which 
results in the total memory storage can be further reduced by 
28%. In our article, by observing a large number of simulation 
and analysis, we come to a result that the log likelihood ratio 
posteriori probability messages (LLR) with check-to-variable 
messages (R) have a relation: LLR [i] ::o4*R[i]U]. By using 
this relationship, we use six bitwide to represent the LLR 
messages, four bitwidth to represent the check-to-variable 
messages. So we can reduce the quantization performance 
loss, while also reduce check-to-variable messages memory 
(R-MEM). By adopted the techniques mentioned above, the 
total memory usage can be reduced 55.6% for 1 / 2 rate of 
CMMM LOPC decoder and 60% for 3 / 4 rate of CMMM 
LOpe decoder. 
The structure of this paper is organized as follows. In 
Section II, background about CMMB will be presented. 
Section 
III presents a partially parallel LOPC decoder 
architecture, 
solution for memory 
collision and various 
optimizations to further reduce the hardware complexity. 
FPGA implementation results are shown in section IV. 
Section N. The conclusions are drawn in Section V.

III. PROPOSED ARCHITECTURE 
A. Overall decoder architecture 
We adopt the Layered algorithm (LA) which reduces the 
number of iterations into a half. The min-sum algorithm is 
integrated with LA to reduce the decoding complexity. Based 
on 
the 
fixed-point 
analysis, 
we 
choose 
4 
bit-width 
quantization, 6 bits fixed-point representation as the bit-width 
for the log likelihood ratio (LLR) messages and 4 bits for 
check-to-variable messages. The proposed LOPC decoder has 
a parallel with 2 for 112 rate and 1 for 3/4 rate and maximum 
iteration number is set to 10. Fig. 1 shows the performance of 
fixed-point simulation of CMMB LOPC codes. Fig.2 shows 
the multi-rate LOPC decoder architecture for CMMB system. 
It consists of the following portions. 
I) R-MEM is used to store the check-to-variable messages R, 
which is composed of four element, I) the smallest magnitude, 
2) the second smallest magnitude, 3) the location index of the 
smallest magnitude, 4) the signs of all R messages. The 
required memory to store the check-to-variable messages can 
be reduced from 4xl2x2304xlbits to 30x2304x2bits. Totally, 
37.5% memory can be reduced by using this method. To 
recover the individual R message from their compressed form, 
the data distributor is introduced. 
3) LLR-MEM .The Othto 35th RAM are used to store the LLR 
messages corresrondin to a block column of matrix H3 and 
H4.The The 37t to 39 
RAM are respectively used to store 
duplications of the 6th ,9th , 13th, 32nd RAM. The 36th RAM is 
used to store duplication of the 22nd RAM when the decoder 
work under 112 rate mode and store duplication of the 6th 
RAM when the decoder work under 3/4 rate mode. 
4) 
The 
distributor 
is 
introduced 
to 
convert 
the 
check-to-variable messages corresponding to a row of H 
matrix into their true values. 
5) CNU is used to computing the check-to-variable messages.

B. Solution for memory collision 
The check matrix for 112 rate have one sub-matrix at row 
9 column 23 of the base matrix whose weight is 2 instead of 1. 
For the check matrix for 3/4 rate, sub-matrix at row 1 column 
7, row 2 column 14, row 4 column 33 , row 5 column 3 of the 
base matrix whose weight is 2 instead of 1. This means we 
have one memory to read and write two times at one clock for 
the 112 rate and four memory to read and write two times at 
one clock for the 3/4 rate. If we do not solve the memory 
collision, it will greatly reduce the performance or throughput 
of LDPC decoder. In our 
architecture, we utilize four 
memories to store duplications of the ram which have to read 
and write two times at one clock. When the decoder has to 
read and write two times at one clock from one memory, 
another can read and write from the duplication memory. This 
leads to a lower parallel to reach the throughput requirements. 
To illustrate the solution of memory collision, we take 
the 22nd RAM for example. The 22nd RAM has to be updated 
three times at one iteration because its column weight is 3. 
The first and second updating are at the same clock that has to 
read and write at the same memory. Fig.3 shows the steps 
how to resolve the contention. At the first and second 
updating, one data read from RAM22 and another from 
RAM36 and the data are write back to other side after 
processing. The newer data are stored in the memory that the 
elements are gray color. At the third updating, the decoder 
read data under the direction of the arrow and write back to 
both ram memory. When finished the whole iteration, both 
ram memory are the newest data.

C. Reuse eNU for 112 and 3/4 rate 
The critical task of CNU is to find the two smallest 
magnitudes from all input data and identifY the relative 
position of the input data with the smallest magnitude. In this 
paper [6], an optimized 2 x 2 Pseudo Rank Order Filter 
(PROF) and an efficient with eight inputs CNU based on the 
PROF is presented. We present the architecture of CNU with 
six inputs for 112 rate is shown in Fig.4. 
The C&S unit is used to compare two input data. If the 
larger magnitude is in the upper position, the comparison 
result indicated by 1, otherwise, it is o. In the same time, the 
larger magnitude is placed at upper output position. Input data 
are 
variable-to-check 
messages 
and 
represented 
in 
sign-magnitude format. On the output side, the smallest 
magnitude 
and second 
smallest magnitude, 
the relative 
position of the smallest magnitude, denoted as mI, m2, index, 
respectively. 
The architecture of CNU with twelve inputs for 3/4 rate 
consists of two CNU with six inputs for 112 rate, shown in 
Fig.5. This property simplifies interconnects and reduces logic 
resources. Since the 112 and 3/4 rate LDPC decoder use the 
CNU module together.

IV. FPGA IMPLEMENTATION RESULTS 
We implement a multi-rate CMMB LDPC decoder in 
Altera device AtratixII-EP2S 180F 1 020C3. Implementation 
results show that the decoder can operate at a maximum 
frequency 
of 
123MHZ 
and 
achieve 
24Mbps 
decoding 
throughput 
under 
10 
iterations 
and 
60MHZ 
working 
frequency, which meet the throughput requirements for the 
CMMB standard. Table II shows the decoder implementation 
results compared with other CMMB LDPC decoders. From 
the table, we can see that the proposed decoder can greatly 
reduce memory and logic resources. 
V. CONCLUSION 
We present a resource efficient multi-rate LPPC decoder 
of CMMB system. Our work offers several advantages when 
compared to the other CMMB LDPC decoder in terms of 
significant reduction in logic, memory and interconnect. Our 
contribution is in using the reuse properties of CNU, READ 
and WRITE NETWORK and R-memory and devising a new 
LDPC decoder architecture using permuted H matrix to offer 
significant additional benefits.
