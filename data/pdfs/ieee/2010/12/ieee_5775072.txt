POWER EFFICIENT COLUMN OPERATION-BASED MESSAGE-PASSING SCHEDULE FOR REGULAR LDPC DECODER

Eun Ji Kim, Myung Hun Lee and Myung Hoon Sunwoo

School of Electrical and Computer Engineering, Ajou University San 5, Wonchun-Dong, Yeungtong-Gu Suwon, 443-749 Korea Tel : 82-31-219-2390 Fax : 82-31-212-9531 e-mail : sunwoo@ajou.ac.kr

ABSTRACT
This paper proposes a power efficient column operation-based message passing schedule for an LDPC decoder. Redundant memory accesses and column operations are removed by using the improved column operation-based message passing schedule. Therefore, the number of memory accesses can be significantly reduced, and thus, power consumption can be also reduced. As a result, the experimental results show that the proposed LDPC decoder can save about 27% power consumption and reduce 25% column operations compared with existing designs with the same error correcting performance.
Index Terms — iterative decoding, message-passing schedule, forward error correction, LDPC, IEEE 802.11n standards
1. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were proposed by Gallager in the early 1960s [1], however, they were forgotten until the 1990s. LDPC codes were rediscovered by MacKay and Neal in 1996 [2]. LDPC codes achieving close to the Shannon limit performance among the forward error correction (FEC) systems [2] have been adopted by the advanced standards, such as Digital Video Broadcasting – Satellite – Second Generation (DVBS2) [3], WLAN (IEEE 802.11n) [4] and WiMAX (IEEE 802.16e).
Recently, the message-passing algorithm that can perform row and column operations independently for LDPC codes is very suitable for LDPC decoder hardware implementation. There are two kinds of parallel LDPC decoders for the message-passing algorithm, i.e., full parallel decoders [5],[6] and partial parallel decoders [7],[8],[9]. The partial parallel decoders occupy less hardware area and show better correcting performance than the full parallel decoders.

To achieve high speed decoding, some partial parallel decoders utilize concurrent schedule which performs one row and one column operation simultaneously. However, the concurrent schedule efficiency between the row and column operations decreases. To improve the messagepassing efficiency, the accelerated message-passing schedule in [10] was proposed for regular LDPC codes. The algorithm performs one row operation, and multiple column operations using concurrent messages updated by the row operations.
This paper proposes a new efficient column operationbased message passing schedule for regular LDPC codes. We modify the original schedule for regular LDPC codes in [10] and reduce the unnecessary computation of column operations for the LDPC decoder.
This paper is organized as follows. Section 2 explains the existing message passing algorithms for the partial parallel regular LDPC decoders in [10]. Section 3 describes the proposed message passing schedule for regular LDPC decoders. Section 4 presents performance evaluations. Finally, Section 5 concludes the paper.
2. Existing Message Passing Algorithms for LDPC Decoders
The concurrent schedule performs the row and column operations in parallel. The row and column operations are performed independently. This schedule reduces a single iterative decoding delay and makes the controller very simple. However, the message passing efficiency between the row and column operations decreases and the memory size is doubled compared to the standard message-passing schedule.
To simplify the addressing and control logics and to improve the message passing efficiency, a high efficiency message passing algorithm was proposed in [10]. In the conventional message-passing schedule for regular LDPC codes, during the row operation in the row operation module, the column operation module concurrently perform

3. PROPOSED MESSAGE PASSING ALGORITHM
3.1. Proposed High Efficiency Message Passing Algorithms
We propose an improved high efficiency message passing algorithm. Fig. 2 illustrates the proposed high efficiency message passing schedule flow. The proposed message passing schedule performs i1, i2, i3-th row operations. i1, i2, i3-th row operations update the messages α‘(i1(q)), α‘(i2(q)), α‘(i3(q)) (Step1). The concurrent row operations do not start from the first row in sequence within each sub block like as the conventional message passing schedule. Such a parity check matrix which called quasi cyclic parity check matrix could be obtained by rightshifting the identity matrix. Therefore, each starting point of i1, i2, i3-th row operations could be selected to perform the jq -th column operation at the same column position. Secondly, one column operation is performed at the same column positions of the updated messages α‘(i1(q)), α‘(i2(q)), α‘(i3(q)) (Step2). Therefore, the one column operation for the jq-th column is performed using each one of messages α‘. The jqth column operation updates the message β‘(jq(1)), β‘(jq(2)), and β‘(jq(3)) (Step3).
Fig. 2. Proposed high efficiency message passing schedule.
If we still use the schedule in [10], three column operations are need to execute in sequence (jq1, jq2, jq3) after every row operation. However, the concurrent column
operations generate (3-1) × 3 updated messages β‘. In other
words, six updating operations of β messages are unnecessary.
Since the updating count of β messages increases with the increasing of column weights, more redundancies of β message updating exist in the columns with higher weights. Therefore, if the β messages are updated only when the β values are utilized by the following row operations, the unnecessary updating operations can be efficiently saved. The proposed schedule for column operation reduction is shown in Fig. 3. The upper half of Fig. 3 shows the row operations and the lower half of that shows the column operations. The proposed row and column operations are composed of 7 and 5 stages, respectively. The row operation starts by reading message β and setting the messages in registers of Check Functional Unit (CFU) which performs the row operation (stage 1 and 2). Six messages β(x, y1), ···, β(x, y6) are fed to CFU for decoding a (3,6)-regular LDPC codeword. To avoid reading and writing the message β at the same clock cycle, we partition the read and setting procedure into two stages.
After that, CFU computes the absolute value |β| (stage 3), and determines the first and second minimum values

(stage 4). CFU compares the absolute values once in each cycle. After determining the first and second minimum values, CFU selects one of the two (stage 5), and then add a sign (plus or minus) for each message α which is given to the column operation (stage 6). The conventional messagepassing schedule in [10] performs the three column operations immediately after the row operations are performed. However, the proposed high efficiency message passing algorithm performs only one column operation immediately after the row operations are performed. In other words, the two grey column operations are unnecessary to update β messages. The column operation module is composed of 5 stages. The column operation starts by reading message α and setting the messages in registers of Bit Functional Unit (BFU) which performs column operations (stage 1 and 2). Then CFU computes ∑α (stage 3) and adds the initial message λ (stage 4). Therefore, the unnecessary updating column operations can be efficiently saved by utilizing the proposed message passing algorithm.
4. PERFORMANCE EVALUATION RESULTS
We have performed the MATLAB simulation to verify the proposed high efficiency message passing schedule. We apply the proposed message passing algorithm and the existing message passing algorithms [10]. We have verified and compared the proposed message passing algorithm and the existing message passing algorithms to regular LDPC code in the WLAN (IEEE 802.11n) standard. In this paper, the code length = 3,072bit (code rate 1/2) have been chosen for performance evaluations. Assume that the encoded data bits are modulated by BPSK and transmitted through the AWGN channel. The number of iterations for LDPC decoding is up to 5 and 10. 5. CONCLUSIONS
This paper proposed the improved high efficiency message passing schedule for the regular LDPC decoder. The proposed schedule efficiently reduces the memory access frequency and redundant column operations. According to the experimental results and comparisons with the existing work, the proposed decoder provides about 27% power consumption saving and reduces 25% column operations with the same error correcting performance.
