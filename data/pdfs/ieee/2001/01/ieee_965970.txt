Abstract - We consider in this paper the possibil- ity of a cyclic or quasi-cyclic LDPC code as the com- plete error control system in a magnetic recording chan- nel. Thus, we compare the performance of selected LDPC codes with two Reed-Solomon (RS) code schemes on an EPR4 channel, with particular emphasis on their perfor- mance in noise bursts (induced by thermal asperities). We nd some very promising initial results in that the LDPC codes are very robust against large noise burst (128 bits long), and are superior to the RS schemes examined in the measurable error rate region. The extent to which they are superior depends on the particular LDPC scheme in- volved, and the results provide motivation for further in- vestigation in this area.

In the current paper, we re-examine the LDPC code argument in light of the following. First, the complex de- coder problem has been largely solved in [7], [8] where LDPC codes are designed using old-fashioned textbook techniques based on nite geometry and algebra. The re- sulting LDPC codes fall into the cyclic and quasi-cyclic classes of block codes and lend themselves to simple en- coder implementation via shift-register circuits. (More re- cently, a related LDPC code design technique using com- binatorics has been proposed [9].) Second, the reason that LDPC codes produce a large number of errors when it fails to decode (or, rarely, when it decodes to an incorrect codeword) is because the distance between codewords is so large; that is, because a (good) LDPC code has a large minimum distance. This should not be considered to be a drawback, although it does appear that way in view of the outer RS code's inability to correct LDPC-uncorrectable error patterns. Thus, we will examine the performance of an error-control coding system consisting only of a long LDPC code, with the outer RS code removed.

Of course, the role of an outer RS code is not only to \clean up" residual errors left by the inner code, but to correct error bursts due to thermal asperities. Thus, the paper will focus largely on the ability of each of the LDPC codes we consider to perform in the presence of noise bursts. Some work in this area has been consid- ered in [4] and [10], but the results here improve upon those reported earlier, owing to improved channel-state- information passing as described below. The ecacy of LDPC codes in noise bursts was recognized by Tanner 20 year ago [11], although he gave no performance results and did not consider partial response signaling. Some work on the performance on LDPC codes in bursts in fading chan- nels has been reported in [12].

In this section, we briey touch on code properties nec- essary for eective performance on burst noise channels. The case of RS codes is well known: they are eec- tive against bursts since they are symbol-error correct- ing codes with the maximal minimum distance, and their performance may be further improved via interleaving. As for LDPC codes, their ecacy in noise bursts can be explained via a Tanner graph representation of the code and the associated message-passing decoding algorithm [13], [6], [4].
Consider Fig. 2 which depicts a section of a Tanner graph for some LDPC code together with an indication of which code bits are aected by a noise burst. Con- sider now the decoding of the ith bit, ci, which has been exposed to the burst, so that its LLR, L(ci), has been ini- tially set to zero via CSI. Now suppose, as depicted in the gure, cr, cs, and ct are involved in the same parity check equation as ci; but have not been exposed to the burst. Assuming that the background noise is suciently low, we may expect the LLR values for cr, cs, and ct to have the correct signs, at least after a few decoding iterations. Thus, the LLR values for these bits can be used to obtain the value of the bit ci via message passing from nodes cr, cs, and ct to node ci (through node fj). One can also argue in a similar fashion that if two or more bits in a check equation have been exposed to a burst, then there is no such guarantee that these bits can be determined via message passing.
Thus, an LDPC code is eective against noise bursts of length  provided that, in any span of  consecutive code bits in each codeword, no two of the  code bits are involved in the same parity-check equation. In terms of the code's parity-check matrix H, this means that any  consecutive bits in any row of H may contain only a single 1. This certainly guarantees the eectiveness of the code, but it is probably too tight of a constraint. All that may be necessary is that, for each of the  aected bits, there exists at least one row (one parity-check equation) that does not involve the other  1 bits.

The results presented have demonstrated that LDPC codes are very eective against noise bursts and, in fact, superior to RS codes in the regime simulated. This fact makes the possibility of an error control system based solely on LDPC code very promising. The results also open up new problems, namely, determining the level (or existence) of an error rate oor (although, for codes whose minimum distance is on the order of 65, a oor will not be a problem). Another problem is the design of higher rate geometric and algebraic LDPC codes and determin- ing whether they are eective in the scenario studied in this paper.
