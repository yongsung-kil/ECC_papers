zontal circuit paths. These values will be represented in two

equivalent forms, lambda and delta. Lambda values are posi-

tive numbers that vary between 0.00 and the maximum allowed

storage value while delta values vary between -1.00 and +1.00.

A. Decoding regular LDPC Codes
One advantage of using regular LDPC codes is the regular patterns that can be arranged in a distributed processing decoder. For example, in the π-rotation pattern each vertical and horizontal product is composed of a small number of terms. This fact is utilized to evenly distribute storage and signal ow in the decoding circuit layout.
The decoding process requires both lambda and delta multiplications. Since the delta multiplication requires a full quadrant multiplication, we design the circuit to multiplex the delta multiplication between rows of the matrix. The lambda multiplications occur continuously.

Delta values which are input to and output from the SILO's are processed sequentially with a special delta multiplication circuit shown in Fig. 8. Using an input reference oscillator (which also serves as the system clock), mixers are arranged to provide all six horizontal calculations of a single check row of the matrix. Values from SILO's associated with a row of the H matrix are fed through the delta multiplication to produce the delta products for the entire row. The results are stored in the appropriate SILO location through the return path (labeled DO 1 through DO 6). A single iteration is complete after all rows are processed. VII. CONCLUSIONS
In this paper we introduce the π-rotation low density parity check codes. The π-rotation pattern is important because it reduces the randomness associated with conventional LDPC codes and allows the complete description of the code with a single permutation vector. This code is easily encoded with an efcient encoding circuit and the symmetries of the pattern suggest a compact decoding design. Since π-rotation codes are completely dened by a single permutation vector, the identication of each code is naturally compressed. The performance of πrotation codes is as good or better than randomly generated regular-LDPC codes. These features make this code attractive for implementation in practical communication products.
