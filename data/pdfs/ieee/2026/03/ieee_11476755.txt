Abstract‚ÄîQuasi-cyclic low-density parity-check (QC-LDPC) decoders are indispensable in high-throughput storage and communication systems. This paper proposes a lookup-table (LUT)-based Reconstruction‚ÄìComputation‚ÄìQuantization (RCQ) decoder tailored for triple-level cell (TLC) NAND flash memory and implemented on a Xilinx Zynq UltraScale+ FPGA using high-level synthesis (HLS). By replacing costly arithmetic operations with pre-computed reconstruction and quantization tables, the RCQ architecture markedly reduces routing congestion while preserving decoding performance. We detail the algorithmic formulation, describe the QC-LDPC code, and explain table construction via density evolution coupled with importancesampling-based error-floor optimization. The hardware design, synthesis results, and a comprehensive throughput analysis are also presented. Compared with state-of-the-art FPGA soft decoders for QC-LDPC codes, our solution‚Äîemploying (3, 6)-bit quantization‚Äîdelivers roughly twice the decoding throughput (5.7 Gb/s at an average of four iterations) with comparable resource utilization using LUT-based simplified routing. In addition, it achieves a raw bit-error-rate close (gain of 3.3%) relative to the Codelucida QC-LDPC code (code rate Rcode=0.89, length N=32768). These results demonstrate that LUT-optimized RCQ decoding offers an efficient and high-performance solution for flash memory applications.

I. INTRODUCTION
Quasi-Cyclic (QC) LDPC codes have become the de-facto forward error correction (FEC) scheme for modern solid state drives (SSDs), wireless standards, and satellite links due to their excellent waterfall performance and hardwarefriendly parallel structure [2]. In Field-Programmable Gate Array (FPGA) implementations, LDPC decoders can deliver multi-gigabit per second throughput while maintaining low power consumption.
A primary bottleneck in conventional beliefpropagation decoders is the high-precision arithmetic required for log-likelihood ratio (LLR) updates. The Reconstruction‚ÄìComputation‚ÄìQuantization (RCQ) paradigm [1], [3] addresses this challenge by mapping LLRs to a small integer alphabet using two pre-computed tables:
‚Ä¢ Reconstruction table (R): Expands quantized messages to higher precision for check-node updates
‚Ä¢ Quantization table (Q): Compresses high-precision messages after variable-node updates

Since both tables are indexed by layer identifiers in the layered decoder architecture, memory footprint scales linearly with the number of layers rather than the full parity-check matrix size. Building upon our earlier work [3], this paper contributes:
‚Ä¢ LUT-based table construction optimized for TLC NAND flash memory;
‚Ä¢ Fully pipelined HLS architecture with efficient memory organization, using HLS optimization directives (#pragma HLS PIPELINE II=1) [4];
‚Ä¢ Comprehensive FPGA synthesis study on Xilinx Zynq UltraScale+;
‚Ä¢ Throughput analysis demonstrating 5.7 Gb/s performance.
The remainder of this paper is organized as follows. Section II introduces QC-LDPC notation. Section III details the RCQ layered algorithm. Section IV describes densityevolution-based table construction. Section V presents the hardware architecture. Section VI reports synthesis results. Section VII provides simulation results, and Section VIII concludes the work.

IV. OPTIMIZATION OF QUANTIZATION AND RECONSTRUCTION TABLES
The optimization of the reconstruction (R) and quantization (Q) tables employs density evolution (DE) on a binary-input AWGN channel to achieve a specified bit-error-rate (BER) target [1], [3], [5]. The DE process tracks discrete probability distributions through decoder iterations rather than direct work on continuous LLR values. A key distinction from probability

density functions (PDFs) is that LLRs are signed quantities and can be negative. Fine-Tuning Optimization: The DE-optimized tables provide near-optimal initial values during early iterations (before ‚åäg/4‚åã iterations where g is the girth). However, as cycles and trapping sets emerge in later layers, two fine-tuning strategies are employed:
‚Ä¢ Mutual Information-Based Optimization: Maximizes mutual information between high-resolution and quantized decoders using Monte Carlo simulations to estimate output PDFs [6]‚Äì[8].
‚Ä¢ Error-Floor Fine-Tuning: Employs importance sampling techniques to simulate rare error events contributing to the error floor, adjusting tables for enhanced performance at target SNR [10], [11].
Both methods address key challenges in quantized decoder design: reducing the error floor while optimizing mutual information retention.

All units are instantiated several times to exploit QC structure (CPM) parallelism. High-throughput decoder architectures require careful pipeline design to balance throughput and latency [16]. Pipeline initiation interval is 2 for both VNU and CNU loops. The Control Unit manages the iteration, performs syndrome check. VNU and CNU are executed in a loop through all rows in protograph.
B. Memory Organization
The design utilizes on-chip BRAM as follows:
1) Channel LLR memory: Stores N = 32, 776 input LLRs (6 bits each)
2) Message memories: Dual-port RAM banks for VN-toCN and CN-to-VN messages (3 bits each)
3) Table ROMs: R and Q tables stored in BRAM (76 blocks total, ‚âà4% of device)
AXI master ports stream data in bursts of 32,776 bits.

All critical paths meet the 280 MHz clock constraint and work with such frequency in burn mode AXI-interface, with timing diagram for the pipelined RCQ represented on Fig. 6. The reconstruction R(‚Ñì, ¬∑) occurs at the end of the CNU stage before the updated Œ≤ is written back to memory.
With early termination reducing average iterations to 4, effective throughput reaches ‚àº7 Gb/s raw (information throughput >5 Gb/s at code rate 0.882). The LDPC decoder does

not consume any DSP slices. All arithmetic is performed with LUT-based logic and block-RAM look-up tables, so the implementation relies solely on LUTs, flip-flops and BRAM. This purely LUT/BRAM architecture is especially attractive for low-cost FPGA families (e.g., Artix-7, Spartan-7) where DSP resources are limited or absent. This characteristic, combined with low hardware consumption, makes the implementation of RCQ decoders particularly promising for low-cost FPGA families.
VII. SIMULATION RESULTS
Monte-Carlo simulations conducted over additive white Gaussian noise (AWGN) channels from the TLC-channel [17] with gray code demonstrate that lookup tables optimized by density evolution and importance sampling attain a frame-error-rate (FER) within 0.1 dB of that achieved by (4,9) layered Normalize Min-Sum decoders when the target bit-error-rate (BER) is 10‚àí17. The simulation environment adopts the following settings: a (3, 6)-bit quantization scheme, a maximum of 24 decoding iterations, and a quasi-cyclic LDPC (QC-LDPC) code derived from a protograph size 16 √ó 136 with a lifting factor Z = 241, which yields an effective code rate R ‚âÉ 0.882.
For reference, the performance of the Codelucida QC-LDPC implementation is plotted using only two data points supplied by the FPGA IP-core report [18]. Figure 7 presents the resulting BER/FER curves for both decoders. Despite this limited comparison set, our RCQ decoder delivers a raw bit-error-rate improvement of 3.3% relative to the Codelucida code (code rate Rcode = 0.89, length N = 32768). This gain, together with the near-optimal FER proximity, underscores the efficacy of DE and Importance sampling optimized tables in preserving decoding performance while reducing hardware complexity compare to normalize min-sum decoder.
VIII. CONCLUSION
This paper presented a LUT-optimized RCQ layered decoder implemented on Xilinx Zynq UltraScale+ FPGA using HLS. By replacing arithmetic operations with precomputed reconstruction and quantization tables, the architecture achieves 5.7 Gb/s raw throughput (average four iterations) while consuming only 19% of LUTs and 4% of BRAM resources. The design scales efficiently with code rate and circulant size due to linear table memory growth with layer count. Subsequent research will focus on optimizing reconstruction and quantization tables through deep neural networks, developing adaptive mechanisms for diverse channel conditions, and implementing mixed-precision check-node processing. These directions build upon established node-degree-based weightsharing schemes for neural LDPC decoders, which have proven effective in substantially reducing model parameters while preserving decoding performance [22]‚Äì[24].
