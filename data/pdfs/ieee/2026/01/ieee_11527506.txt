Abstract—This paper presents a new 1KB open Forward Error Correction (oFEC)-like encoding and decoding architecture based on extended BCH (576, 545) codes specifically designed for NAND flash memory. The architecture adopts a hard decoding approach to reduce implementation cost and latency. Furthermore, the proposed smashed structure preserves the oFEC-style property while requiring substantially less memory, thereby achieving high error correction performance. Implemented in the TSMC 40nm process, the extended BCH decoder of the proposed architecture achieves a throughput of 25.71 Gbps at an operating frequency of 357 MHz, with a core area of 0.027 mm2 and energy efficiency of 0.16 pJ/bit.
Index Terms—error correcting codes, oFEC codes, NAND flash memory.
I. INTRODUCTION
The error-correcting capability of Bose-ChaudhuriHocquenghem (BCH) codes in NAND flash memory is limited by the storage space for parity bits. Although Low-Density Parity Check (LDPC) codes offer stronger error correction, they rely on accurate soft information for effective decoding. Retrieving soft information from NAND flash is difficult and introduces significant hardware complexity and latency, which degrades system performance.
Open ROADM [1] recently introduced open Forward Error Correction (oFEC), an architecture designed for highthroughput optical communications. While oFEC achieves strong error correction with soft decoding, its high latency and power consumption make it unsuitable for NAND flash applications. To address these issues, an oFEC-like architecture optimized for hard decoding is proposed. This architecture incorporates a smashed structure that significantly reduces memory usage, while maintaining comparable error correction performance.
This paper introduces the background of oFEC in Section II. Sections III and IV present the proposed oFEC-like architecture and hardware implementation. Experimental results

are discussed in Section V, and conclusions are provided in Section VI.
II. OFEC ENCODING AND DECODING PROCEDURE
As illustrated in Fig. 1 (a), the oFEC encoder is composed of multiple basic units, where each unit is a B × B block with B = 16. The entire encoder is organized into R = 22 row blocks and C = 8 column blocks, resulting in a total of N = B × C columns. This arrangement is referred to as a Tile structure, with a coordinate system (R, C, r, c) used to describe the position of each bit within the encoder. Here, R and C represent the row and column block indices, while r and c specify the row and column indices within each unit block. Both r and c range from 0 to 15. Additionally, guard bits occupy 2G row blocks, where G is typically set to 2.
The oFEC encoding process is based on extended BCH (256, 239) codes and involves splitting the codewords into front bits and back bits based on an index k, where k ranges from 0 to 2N − 1. When k < N , the corresponding bits are referred to as front bits, while bits with k ≥ N are referred to as back bits. The encoding process begins by reading front bits from the upper portion of the guard bits. Specifically, 32 sets of 128-bit front bits are read from the tile structure. These front bits are then combined with 32 sets of 111-bit input messages and encoded into 32 extended BCH (256, 239) codewords. Finally, the encoded back bits are written back to the lower portion of the guard bits. The interleaving method used for reading and writing these bits can be determined from the equations:

III. PROPOSED 1KB OFEC-LIKE SMASHED DECODER

A. Code Constructions

Fig. 2 (a) illustrates the initial oFEC-like architecture designed for 1KB NAND flash applications,which is constructed to preserve the same interleaving scheme as oFEC. In this architecture, the unit block size B is increased from 16 to 32, and the number of column blocks C is adjusted from 8 to 9. Additionally, the guard bits are removed (G = 0), as they do not impact ECC performance and their removal helps reduce memory usage. As a result, the total number of row blocks R is set to 20, with the output process modified to produce one row block at a time. Using the same interleaving method, 32 sets of 288 front bits are obtained and combined with 32 sets of 257-bit input messages. These are then encoded into 32 extended BCH (576, 545; 3) codes. The total code length is 9216 bits, with an information length of 8224 bits, resulting in a code rate of 0.892.
To further optimize memory usage, a new architecture called the ”Smashed Structure” was proposed. This approach involves removing blank row blocks, effectively compressing the original structure shown in Fig. 2 (a) to only 10 row blocks, as illustrated in Fig. 2 (b). The Chien search in the proposed design uses the StraightForward architecture [7] with a parallel factor of P = 115. As illustrated in Fig. 8, the architecture checks 115 roots per cycle by directly substituting the roots into error locator polynomial. The search completes within 5 cycles for all 575 roots. The extended BCH decoder consists of four stages to complete the decoding process: Syndrome Computation (SC), Key Equation Solver (KES), Chien Search (CS), and parity check. The syndrome computation is performed by directly multiplying the received codeword r by the parity-check matrix H, as described in equation (4). This approach leverages full parallel processing, allowing the syndrome to be computed within a single cycle, thereby significantly reducing latency.

VI. CONCLUSIONS
This paper presented a 1KB oFEC-like decoding architecture optimized for NAND flash memory. The architecture utilizes HIHO decoding across all tiles to eliminate the latency and hardware complexity associated with retrieving soft values and performing soft decoding. To enhance resource efficiency, a smashed structure was introduced, reducing memory usage by compressing unused row blocks while maintaining comparable error correction performance. Experimental results demonstrated negligible differences in ECC performance between the smashed and original structures across various tile configurations. Compared to conventional BCH and LDPC decoding techniques, the proposed architecture achieved a significant improvement in BLER performance, reaching 4×10−5 at SNR = 5.4 dB—orders of magnitude better than LDPC MATE-GDBF and UP-GDBF decoders. The decoder also attained a throughput of 28.12 Gbps in the TSMC 40nm process. These results position the proposed oFEC-like architecture as a competitive solution for high-performance NAND flash memory applications.
