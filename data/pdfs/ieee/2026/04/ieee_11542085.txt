Abstract—Low density Parity Check codes are extensively utilized in modern communication systems because they offer error correction performance that approaches the Shannon limit. This work presents the design RTL implementation synthesis & analysis of a 24,12 LDPC Encoder to Decoder Architecture Using Min Sum decoding algorithm. The system includes an LDPC encoder error signal to channel Log Likelihood Ratio LLR generator and iterative LDPC decoder suitable for ASIC and FPGA platforms. The complete design is implemented in synthesizable Verilog and evaluated using Cadence Genus for timing, power and area analysis. Results show that the decoder successfully recovers all transmitted Wi-Fi test patterns using single bit and double bit errors while achieving positive slack in Static Timing Analysis and maintaining low dynamic power consumption. The architecture is efficient for low power wireless applications including Wi-Fi and 5G error correcting systems.
Index Terms—LDPC, Min-Sum Algorithm, ASIC, FPGA, Error Correction, Verilog HDL, Static Timing Analysis, Power Estimation.
I. INTRODUCTION
A synthesizable Verilog HDL architecture was developed for all blocks including encoder channel model, VNU, CNU and syndrome checker. The system was validated through behavioral simulation FPGA synthesis and ASIC implementation targeting standard CMOS technology. FPGA based testing validated correct recovery under single bit and double bit error patterns and confirmed reliable decoding for WiFi style binary data sequences. Further ASIC implementation optimization was performed by executing logic design synthesis static, timing checks and simulation at the gate level power and area estimation. The ASIC results showed positive timing slack reduced area utilization and favorable power characteristics confirming that proposed architecture suitable for integration into high-speed communication receivers [8]. The project demonstrates a complete work flow from RTL modeling to FPGA prototype and ASIC ready design and it shows the feasibility of implementing LDPC decoding with high efficiency using the Min Sum approach. The results demonstrate that this architecture is very well suited for real time use in applications such as Wi-Fi modems IOT radios and low power embedded communication systems [4].

II. RELATED WORKS
A. Explanation of the (24,12) LDPC Code
The LDPC code used in this work encodes every 12 information bits into a 24-bit codeword. This is a short block LDPC structure specifically useful for simulation demonstrations due to the fact that all fundamental LDPC characteristics are preserved while keeping the computational complexity feasible on FPGA and ASIC. The code is specified by fixed the generator matrix G and parity check matrix H that decide the linear relations among bits[9].
While the implemented (24,12) code serves as a highly effective platform for validating the complete end to end encoder and iterative decoding pipeline practical Wi-Fi and 5G standards utilize significantly larger LDPC matrices. The proposed architecture is designed to accommodate this scaling due to its highly modular check node unit and variable node unit subsystems as well as its inherently parallel FPGA mapping. The syndrome based decoding mechanism evaluates the H · cT = 0 condition dynamically making the
early termination process highly scalable without requiring full codeword comparisons [6].
B. Generator Matrix, Parity-Check Matrix and LLR Generation
The LDPC encoding in this work uses a structured (24,12) code where a 12-bit message is converted into a 24-bit codeword using a generator matrix G. This matrix specifies the XOR relationships required to generate parity bits from the information bits enabling a fully combinational encoder with low latency small area, and simulation efficiency. Each row of G corresponds to one parity equation implemented using XOR logic. The complementary paritycheck matrix H defines the constraints for iterative decoding. For the (24,12) LDPC code H is a sparse 12×24 matrix in which rows correspond to check node units and columns to variable node units in the tanner graph. Decoding verifies the received codeword using the syndrome condition H · cT = 0 a
zero syndrome indicates a valid codeword. The sparsity of H reduces interconnections and computational complexity enabling efficient message pass- ing. III. LDPC CODE STRUCTURE AND ALGORITHMS A. CNU and VNU subsystem architecture
Check Node Unit implements the Min Sum algorithms sign XOR and minimum magnitude operations [10]. Each CNU is feed messages from its connected VNUs computes two smallest magnitudes and propagates an updated reliability value back to all connected variable nodes. The Variable Node Unit accumulates the incoming CNU messages along with the initial LLR to estimate the final bit decision reference figure shown below in fig.1. This accumulation is then followed by subtraction to form extrinsic information for outgoing messages. Both CNU and VNU are fully pipelined in the ASIC implementation using registers to break critical paths and achieve timing closure. The resulting simulation structure is highly modular with 12 CNU and 24 VNU interconnected using the parity check matrix structure [13].

B. LDPC Decoding Algorithms Min-Sum vs Sum-Product and Normalized Min-Sum:
To justify the selection of baseline Min Sum algorithm it is essential to compare its performance and hardware tradeoffs against sum product Algorithm and normalized Min Sum algorithm. The sum product algorithm offers optimal error correction performance by utilizing precise probabilistic likelihood values. Computing these values requires highly complex mathematical operations including multiplication, division and exponential or logarithmic calculations. SPA incurs a very high hardware cost drastically increases silicon area and is rated only moderate for ASIC suitability.
To overcome these implementation barriers the Min Sum Algorithm approximates the SPA by replacing all complex floating-point computations with simple minimum magnitude and sign propagation operations. This strategic simplification results in substantial savings in hardware cost and dynamic power consumption while introducing only minimal loss in error correction performance [1]. Min Sum algorithm drops the hardware cost to low and achieves excellent suitability for both FPGA and ASIC platforms.
While the Normalized Min Sum algorithm is recognized in literature as a method to close the accuracy gap with SPA often utilized a long side non uniform quantization scheme

it typically requires an additional scaling factor or multiplier step. Because a primary objective of this 12×24 architecture is to eliminate the need for multipliers and normalization circuits entirely to ensure the lowest possible dynamic power and area overhead the baseline fixed magnitude Min Sum approach was selected as the optimal tradeoff for low power wireless and IoT applications [6]. IV. QUANTIZED LLR MAPPING
In soft-decision LDPC decoding Log-Likelihood Ratios represent the reliability of received bits and are typically computed using probabilistic expressions involving logarithmic and exponential operations. Conventional LLR computation requires multipliers, dividers and floating-point arithmetic units which significantly increase hardware complexity silicon area, power consumption and critical path delay. For FPGA and ASIC implementations targeting low-power wireless systems such computational overhead is undesirable[6].
To reduce implementation complexity this work adopts a fixed magnitude signed LLR quantization scheme. Instead of calculating precise probabilistic LLR values the received binary bits are mapped to constant signed integers as follows:
• Received bit ‘0’ → +8
• Received bit ‘1’ → 8 This representation preserves the essential properties required by the Min-Sum decoding algorithm:
• The sign indicates the bit decision direction, and
• The magnitude represents confidence level.

B. Re-Encode & parity check
Simulation utilizes the commonly used re-encode & Compare method to ensure correctness. After the decoder produces 12 estimated information bits these bits are re-encoded using the systems generator matrix G form noise free 24-bit codeword. This newly generated the codeword is then compared to the decoders reconstructed version of the 24b i t code-word fig.3.If both match exactly the decoder is said to have successfully converged. If any mismatch occurs the decoder encountered either too much channel noise or not enough iterations. This technique is especially effective in simulation since it does not rely on knowing the actual transmitted bits rather it checks self-consistency of the decoding process. X. CONCLUSION
This work presented a complete FPGA and ASIC implementation of a 12×24 parity check matrix based LDPC encoder to decoder architecture using the Min Sum algorithm. A fully synthesizable RTL design was developed in Verilog covering the encoder channel model quantized LLR generator iterative decoder with CNU/VNU subsystems and syndrome checker. The structured generator matrix and meager parity check matrix enabled low complexity hardware realization with efficient message passing and reduced interconnect overhead. FPGA validation confirmed correct decoding of Wi- Fi style data under no error, single-bit and double-bit error conditions demonstrating reliable convergence of the Min Sum algorithm. ASIC synthesis using Cadence Genus achieved positive timing slack compact area utilization and low power consumption verifying the suitability of the architecture for high speed and low power communication receivers. The use of fixed magnitude quantized LLRs eliminated complex arithmetic units further reducing hardware cost while maintaining decoding performance.
