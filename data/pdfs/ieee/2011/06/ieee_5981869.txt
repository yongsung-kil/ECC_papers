ABSTRACT
This paper presents the systematic methodology of error correction scheme using low-density parity check (LDPC) codes to improve the reliability and endurance of multi-level cell (MLC) non-volatile memories. Using our realistic error model, the LDPC architecture with the scheme of non-uniform reference voltages (NURV) is proposed to trade off among error correction capability, area, and throughput, which can improve the bit-error-rate significantly.

1. INTRODUCTION
In recent years non-volatile memories, especially flash devices, have been adopted extensively in many consumer products, e.g., digital cameras, MP3 players, cellular phones, and solid-state disks (SSDs), for their data and code storage. However, flash memory also faces severe reliability and endurance challenges with the advance of its manufacturing technology for the multi-level cell (MLC) devices that store multiple bits in one cell with multiple threshold voltages (Vth) to increase the storage capacity. The more levels there are, the smaller the operation margin is. Programming disturbance, read disturbance and retention issues have been shown to be the major causes that alter the Vth of flash cell unexpectedly [1]. Error occurs when the disturbed Vth are accessed, degrading the reliability.

Error correction codes (ECC) have therefore played an indispensable role to improve the reliability and endurance of MLC nonvolatile memories. Linear block codes, e.g., the Bose-ChaudhuriHocquenghem (BCH) codes, are widely used due to the inherent nature of block-wise access for NAND-type flash memories [2–4].
Reports from industry recently have showed that the manufacturing technology of larger and denser flash devices has led to the continuous degradation of reliability and endurance [5, 6]. Even with more and more spare cells as the parity bits, traditional ECC schemes are becoming infeasible to cope with the extremely high raw bit error rate (BER) of advanced flash memories.
Low-density parity-check (LDPC) codes, which have been widely adopted in communication applications, are gaining more and more attention as the superior error correction ability. Several ECC schemes using LDPC algorithm have been proposed for flash systems. In [7], a nonbinary LDPC code has been presented to provide a better error-correction performance. The data in one MLC cell are represented as a nonbinary codeword symbol for the LDPC algorithm. However, for modern MLC NAND-Flash, data bits in one MLC cell are commonly arranged to separate memory pages to reduce the program/read latency. The nonbinary LDPC scheme therefore may suffer from the access performance. In [8], a concept has been given to use multiple reference voltages to obtain likelihood information of the data readout, where the multiple reference voltages are widely used in original programming and erasing procedures.
In this paper, we propose an ECC methodology to design a binaryLDPC code and its hardware architecture for the reliability improvement of flash memories, with the following contributions:
• The error model of MLC memories and its simulation framework is constructed and evaluated with the consideration of the realistic Vth distribution. Based on the simulation framework, the trade-off among design parameters becomes systematic and efficient.
• The non-uniform reference voltages (NURV) scheme is proposed to provide the soft information for MLCs, which can optimize the number of reference voltages and the error correction performance, and thereafter, improve the design complexity and throughput for the data readout.
• Flash-specific architecture of LDPC codes is designed and analyzed to provide the significant improvement on the error correction performance over the BCH codes with competitive latency and area overhead.
Based on the methodology, our area-efficient (9180, 8364)-LDPC code and its CODEC (coder and decoder) architecture for an industrial 43nm 2-bits/cell flash memory has been evaluated. The LDPC code improves the BER from about 10−2 of uncorrected codewords to less than 10−11 after error correction. 2. ERROR MODELING AND ANALYSIS
Fig. 1 shows the typical NAND-flash memory system, which consists of the flash module and flash controller. Operations (e.g. programming, erasing, and reading) of the flash module are controlled by commands issued from the MCU (micro-controller) in the flash controller via the I/O Ctrl (I/O controller). Data is programmed and read in pages to and from the flash array via the Data Buffer. A page usually has 4K to 8K bytes for modern flash memories, which consists of the user area and spare area. The former stores the user data, and the latter is for system-level management such as firmware, wear-leveling and ECC. User data is encoded before programming into the flash array and decoded after reading to correct errors by the ECC module.

2.2 Reliability Analysis with High Raw-BER
BCH codes nowadays are designed to correct 8 to 24 errors within a 512-byte or 1K-byte data. But larger data block with more parity bits is implemented in the advanced flash memory [6]. The test case in this work is an industrial flash device of 43nm technology. The user area of one page consists of 8K bytes, and the spare area has 992 bytes, respectively. The page is divided into 8 partitions. Moreover, 160 out of 992 bytes in the spare area are reserved for system-level management and also encoded. Let one page be divided into eight data blocks. For each data block, the length of protected messages is 8352 (8192 + 160) bits, and the maximum codeword length is 9184 bits, i.e., at most 9184 − 8352 = 832 bits for parity bits in each data block.
An experiment is applied to the LoPage by using our error model with 3.3 ≥ α ≥ 2.5, i.e., the high raw-BERs are between 1.3 × 10−2 and 2.6 × 10−3. Errors are assumed to be uncorrelated in the experiment. Let t be the number of correctable errors of BCH codes and data block be 1K bytes. Fig. 4 shows the corrected BERs with various t’s, i.e., t =24, 48, 72 and 96 of 1.2 × 108 codewords. According to [5], the acceptable application error rate of NANDtype flash is between 10−11 and 10−16 or even lower. It shows that BCH codes with t =24, 48 and 72 are insufficient. Only with t = 96, corrected BER can be smaller than 10−11 when α = 2.7 (with the raw-BER of 4.7 × 10−3). However, the BCH code with t = 96 requires 14 × 96 = 1344 parity bits, which is not affordable by this specific flash device. Therefore, LDPC scheme is considered for such kind of flash memories with high raw-BER.

3.2 Non-Uniform Reference Voltages (NURV)
From statistical point of view, errors could occur when the value of Vth is close to ORVs (VM , VL, VR), and therefore ARVs near ORVs can help estimate the Vth more precisely. On the contrary, when the real Vth is far away from ORVs, it is more likely that the data readout is correct. Therefore, ARVs far away from ORVs provide less information for the correction. We propose the NonUniform Reference Voltages (NURV) scheme to arrange the ARVs efficiently.
In the first step, a straightforward arrangement of uniformly distributed reference voltages is applied. Let ν be the maximum allowable distance between two adjacent reference voltages. For a middle region, η (= (Vr − Vl)/ν − 1) ARVs are uniformly inserted, where Vr and Vl are the right and the left ORVs of the region. For the leftmost and rightmost regions, η/2 ARVs with the same distance are arranged. The value of ν decides the number of ARVs to be used to quantize the value of Vth and the performance of decoding as well. Initially the ν is derived to meet the error correction performance by proper simulations. Note that regions are usually of different sizes, so ARVs may not be exactly uniformly added. In addition, a practical ν has to be chosen considering the hardware complexity, e.g., the granularity of reference voltages, control overhead and storage requirement, etc.
Fig. 5 shows an example of a uniform reference scheme created by the first step with ν = 0.2. Four ARVs are added in the middle regions (R2 and R0); two ARVs for the leftmost region (R3); and two for the rightmost region (R1), respectively. All Vth’s outside the two bounds (V8 and V7) will be modeled by fixed values.
In the second step, we partition all ARVs into different sets according to their distance to the closest ORV. In this example, we have two sets, S1={V1∼V6}, S2={V7∼V12}, where Vj ∈ Si denotes that the distance, Δ, between Vj and the closest ORV is (i × ν) < Δ ≤ (i + 1) × ν.
In the third step, let S be an empty set initially. We incrementally add one set into S at a time. Namely, S = S ∪ {Si, min(Si+1), max(Si+1)}, where Si has the smallest distance i within all sets not in S, and min(Si+1) (max(Si+1)) is the minimal (maximum) ARV in Si+1 for the left bound (right bound). Simulation is performed to verify the decoding performance with present S. Repeat the 3rd step until the decoding performance meets the requirement.

3.3 Flash-Specific LDPC Code
Based on the model proposed in Sec. 3.1, the LDPC codes can be designed and evaluated efficiently. An LDPC code can be described by a sparse parity checking matrix Hm×n. The construction of H (number of 1’s and locations of them) decides the error correction performance, latency, and hardware overhead as well. Instead of finding an optimized H, an LDPC code for flash memories is proposed. First, the regular LDPC code is adopted to reduce the hardware complexity. Second, quasi-cyclic (QC) LDPC codes with the progressive edge growth (PEG) [9] are used to minimize the storage for the matrix, i.e., H is constructed from sub-matrices Hi,j, also called the circulant, for 1 ≤ i ≤ c and 1 ≤ j ≤ t, where bc = n and bt = m. Once the first row of Hi,j is being identified, the remaining parts can be regenerated simply by right-rotating the first row. And third, larger circulant is preferred for reducing both the memory usage and hardware complexity further.
However, the variable degree will limit the size of circulant, i.e., the larger the circulant the smaller the variable degree. In addition, the codeword length will be less flexible for a larger circulant. For our design, the variable degree for a non-QC PEG code as in [10] is evaluated first, and then the circulant size is decided.

• Subtract: The prior message for the ith row is ρ = [ρ1, ρ2, . . . , ρt] = γ(Ii) − λi.
• Decode: Decode ρ by a soft-input soft-output function (SISO). Let Λ = [Λ1, Λ2, . . . , Λt]. • Update: The new extrinsic message for the ith row is updated by λi ← Λ, and the sum of all extrinsic information is updated by γ(Ii) ← ρ + Λ.
The decoding process is executed iteratively until all errors are corrected or the maximum iteration count is reached.
The architecture of the proposed decoder is shown in Fig. 9. The Decode Controller generates the addresses and commands to each memory interface by the information from the checking matrix H. The extrinsic messages are located in λ-memory. The Subtract and Decode steps are performed by k message passing units MPUs, whereas k rows can be updated in parallel for the extrinsic messages. The summation of all extrinsic information is stored in the γ-memory and partitioned into memory banks denoted by bank(t,j) for decoding k consecutive rows in parallel, where t = {1, . . . , 45} and j = {1, . . . , k}.
To further reduce hardware complexity, we let k be a factor of the circulant size (i.e. 204) to avoid redundancy in the memory blocks and to reduce the complexity of the interface. The data γi for 1 ≤ i ≤ 9180 is stored in the bank(t,j), where (t−1)×204+j = i. Each bank(t,j) provides 45 × k γi’s for decoding k different rows in parallel. There is no data conflict because any k consecutive γi’s required for the decoding are stored in different banks. Sec. 5 will show that the proposed decoder is efficient with k =1 to 4.

A simulation model has been constructed by setting σ = 0.05 to model the 2-bits/cell MLC flash memory with the deviation factor α ranging from 3.8 to 2.8. In our CODEC, the number of bits to represent the integer, fraction, and sign-bit for γi are 6, 1, and 1, respectively; and those for δi are 4, 1, and 1, respectively. The scaling factor β is assigned as 0.75. And the maximum decoding iteration is 4.
5.1 Analysis of Error Correction Ability
The BERs of the proposed ECC scheme are compared with results of the uniform-read-voltage scheme, LDPC code with hard information, and BCH code.
Fig. 10(a) shows the HiPage BERs before and after decoding with the deviation factor α=3.8 to 2.8 for 1.2×108 codewords. The curve labeled NURV-17 is the result for 3 ORVs and 14 ARVs, where 14 ARVs are arranged by the proposed NURV scheme. The curve labeled URV-28 is the result for 3 ORVs and 25 ARVs, where the 25 ARVs are uniformly distributed. For the NURV-17, the BER of HiPage is less than 10−11 when α is smaller than 3.2. As compared with the URV-28, the proposed NURV scheme significantly reduces the number of ARVs and still provides acceptable correction performance.
The curve of 3 RVs indicates the result with 3 ORVs (i.e., VL, VM and VH ), which represents the LDPC with hard information. The comparison shows that the decoding with hard information is not sufficient when the raw-BER reaches 10−2.
The curve labeled Ordinary is the result with the ordinary soft information of Vth in real number representation, which indicates the performance of error correction without quantization errors. The decoding result of LoPage with the same configuration is shown in Fig. 10(b), where the BER can be less than 10−11 when α is smaller than 2.8 for the NURV-17.
The LDPC code utilizes 816 spare bits. With the specific amount of spare bits, BCH codes can only correct up to 59 errors. Fig. 10 also shows the simulation results of (9178, 8352, 59) BCH CODEC labeled BCH59. As compared with the NURV-17, the BER of BCH59 is about 107 times worse when α is smaller than 3.2 for the HiPage and smaller than 2.8 for the LoPage, respectively. Note that the evaluation result for distinct flash device can be a lot different. However, with our simulation framework, the area, latency and error capability can be easily traded off and optimized under the proposed CODEC architecture, once given the target flash model.

In this paper, we have presented a methodology to optimize the ECC scheme for realistic non-volatile MLC flash memories. The evaluation shows that with our NURV scheme, the proposed (9180, 8364) QC-PEG LDPC code with soft information provides the BER as 107 times smaller than the BCH code does using the same amount of parity bits for an industrial 2-bits/cell flash memory. The comparison with BCH CODEC justifies that our architecture is cost efficient for advanced non-volatile memories, under the trade-off among error correction capability, area overhead and throughput.
