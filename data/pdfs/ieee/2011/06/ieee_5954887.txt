Multi-rate LDPC Codes for DTV Transmission by 
Known-Bits Padding 
Liang Gongl, Bo Liul, Yin XUI, Bo Rong2, Yiyan Wu2, Gilles Gagnon2, Lin Guil, and Wenjun Zhangl

2 Communications Research Centre Canada 
Abstract-Modern DTV broadcasting systems, such as DVB 
and the Chinese DTMB system, usually have multiple code rates 
for forward error correction to accommodate a variety of channel 
conditions. In these systems, each code rate corresponds to a 
special set of code words, which put a limit on rate granularity 
and thus cannot achieve seamless code rate adaptation. To 
solve this problem, we propose a known-bits padding scheme to 
generate multi·code-rate LDPC codes of arbitrary granularity. 
In our scheme, we take a standardized LDPC code of specific 
rate as mother code, then pad known-bits according to the 
targeted code rate and given shortening optimization algorithm. 
Simulation study shows that our proposed scheme can obtain 
satisfying performance in terms of code rate flexibility and 
decoding threshold. 
Index Terms-DTV, Channel Coding, LDPC 
I. INTRODUCTION 
Originally invented in the early 1960's, Low-density parity­
check (LDPC) codes have experienced an amazing comeback 
in the last few years and remain one of the most attractive 
topics in coding theory today. LDPC codes can provide 
near-capacity performance with fast encoding and decoding 
algorithms, and thus are widely used in the applications where 
reliable and highly efficient information transfer is required. 
For example, Quasi-Cyclic LDPC (QC-LDPC), a class of 
LDPC codes, has been widely adopted by the most advanced 
wireless communication systems, including DVB-S2ff2 [1,2], 
Chinese DTMB [3], IEEE 802.16e [4], 802.lIn [5], and etc .. 
Modern DTV broadcasting systems, such as DVB and 
Chinese DTMB, usually provide multiple code rates for dif­
ferent channel conditions. In these systems, each code rate 
corresponds to a special set of code words optimized or sub­
optimized at that rate. However, these codes have a limit on 
rate granularity and thus cannot achieve seamless code rate 
adaptation. 
To solve this problem, we propose a known-bits padding 
scheme to generate multi-code-rate LDPC codes of arbitrary 
granularity. In particular, we take a standardized LPDC code 
from Chinese DTMB as mother code, and then generate the 
code of targeted rate by padding known-bits to the codewords. 
The known-bits padding scheme is also referred to as code 
shortening in LDPC codes design. It can be traced back to 
the shortened BCH and RS codes. For LDPC codes, this 
topic was first discussed in [6], aiming to enlarge the girth of 
array codes (a kind of QC-LDPC codes). Then, it was further 
investigated from different perspectives, such as H-ARQ [7] 
and multiplying code rates [8]. 
Padding known-bits is to delete columns in the parity 
check matrix or remove variable nodes in the bipartite graph, 
corresponding to the position of the known-bits. In our work, 
we first define the row-weight sum (RWS) for a given column 
in H as the weight sum of rows that has a 1 in that column. We 
then develop a criterion, namely maximum row-weight sum 
(MRWS), which may be constrained by a candidate column set 
Vp. It is worth noting that MRWS is equivalent to the approach 
of extrinsic-sum in [8] if Vp includes only the variable nodes of 
lowest degree. In this paper, we denote the MRWS on variable 
nodes with the lowest degree as MRWS(A), and MRWS 
with no candidate column set as MRWS(B). In addition, we 
propose another criterion that selects the columns resulting 
in the matrix of minimum row weight variance. We denote 
this criterion as MRWSV, which aims to make the row degree 
of residual matrix as regular as possible. Our simulation study 
shows that the proposed MRWS(B) and MRWSV criteria have 
clear advantage over MRWS(A) criterion. 
The rest of the paper is organized as follows. We first 
introduce the known-bits padding for LDPC codes in Section 
II. We then develop the column deleting algorithm for known­
bits padding in Section III. In Section IV, we evaluate the 
performance of our proposed known-bits padding scheme 
through simulation study. In the end, Section V concludes the 
paper. 
II. KNOWN-BITS PADDING FOR LDPC CODES 
The word "padding" comes from the perspective of data 
transmission. It is also known as "shortening" from the per­
spective of channel coding. This topic can be traced back to 
the shortened BCH and RS codes, especially the latter which 
can be shortened almost at will. For LDPC codes, only a few 
studies were published recently [6-8]. 
The shortening of parity bits is often called "puncturing", 
which is used in the scenario where the quality of transmission 
is controlled. As the parity bits depends on those information 
bits, puncturing will change the probability density of mes­
sages input to the decoder and the analysis has to resort to 
density evolution (DE). For the shortening of information bits, 
extrinsic information transfer (EXIT) chart can be applied, 
since these bits are independent from each other and changes 
lie only in code rate, code length, and degree distribution. 
Take the (8, 4) LDPC code defined by Equation (1) as an 
example, where the code has a length of 8 with 4 information 
bits and 4 parity bits.

To this step, it is clear that padding known bits is equivalent 
to to eliminating the correspondent columns in parity-check 
matrix or eliminating the correspondent variable nodes with 
the edges incident to them in bipartite graph as shown in 
Fig. l(b). Another fact shown by this example is that the 
elimination of columns in H changes the code rate from 
1 - 4/8 = 1/2 to 1 - 4/6 = 1/3, the code length from 
8 to 6, and the code from a (2, 4) regular one to a column­
regular row-irregular one with column degree 2 and row degree 
(2, 3, 4). 
From this example, we can reach the following conclusions 
on "information padding". 
1) Padding information is equivalent to deleting columns 
in the information part of the parity-check matrix. 
2) Padding will result in the decrease of code rate and 
length. That is, padding m bits to a (n, k) code will 
achieve a (n - m, k - m) code. 
3) Decoding will be performed on the residual bipartite 
graph, which is correspondent column-reduced matrix. 
4) Padding will change the degree distribution of variable 
and check node. 
Therefore, given the number of padding bits, the pattern 
of padding needs to be optimized regarding the performance 
of residual code. In other words, the degree distributions of 
shortened code need to be optimized for a low decoding 
threshold. 
Padding information not only changes the structure of 
parity-check matrix, but also causes the encoding matrix 
mutated. The systematic encoding matrix, denoted by G, is 
of size k by n and has the information part of a k by k square 
matrix and the parity check part of a k by (n - k) matrix. The 
rows of G and H span the n-dimensional vector space; the 
sub-spaces spanned by the G and H are orthogonal to each 
other, i.e. GHT = o. We demonstrate the correspondence of 
G and H in Fig. 2. As shown in the figure, padding m known 
bits in the codeword is equivalent to deleting m columns 
in H and both m columns and m rows in G. Suppose the 
deleted columns are indexed by Cde! = {Ci' i = 1, ... , m} , 
the columns and rows deleted from G are indexed by the same 
set. The above is a general case of encoding. In case of (I)RA 
codes, there is no need for the G, and the encoding can be 
done using the column deleted H matrix.

III. COLUMN DELETING ALGORITHMS

As aforementioned, padding the known information is 
equivalent to deleting columns in the parity check matrix, 
i.e., the variable nodes in bipartite graph that correspond to 
the information bits. Two issues must be taken into account 
when implementing the deletion. The first is the candidate 
columns set Vp and the second is the criterion of deletion. As 
the columns in H and variable nodes in H's bipartite graph 
are equivalent, we will not distinguish them in the following. 
A. The MRWC Algorithm 
In [8], the authors propose an algorithm of deleting the 
columns in H. They choose Vp to be the one that contains the 
columns with the least column weight in the residual matrix 
and proposes a criterion that selects those variable nodes with 
the largest "extrinsic-sum" to delete. The so-called extrinsic­
sum is defined by

In the algorithm, the columns are deleted one by one. In 
each step, first pick the variable nodes with the lowest degree 
as candidates; then, among the candidates, select those with 
the largest extrinsic-sum; finally pick the one that results in 
the lowest decoding threshold to delete. 
Since all candidate columns share the same degree, com­
puting the the so-called "extrinsic-sum" of each variable node 
is equivalent to computing the degree sum of check nodes 
connecting to the variable node. In terms of H, it is equivalent 
to compute the weight sum of rows that has a 1 in the 
correspondent column. With this interpretation, we denote this 
algorithm as Maximum Row-Weight Sum Criterion(A), or 
MRWC(A). 
B. Extrinsic-Sum versus Row-Weight Sum 
By the definition, extrinsic-sum of a variable node is in 
fact the number of leaf nodes by expanding the computation 
tree by one level at the variable node. Thus, if expanding the 
computation tree further, more variable nodes will appear or 
re-appear in the tree and all the these nodes can be consid­
ered to provide extrinsic information to the root node when 
performing the message passing decoding. In this respect, the 
definition of "extrinsic-sum" is not quite proper. 
However, from another perspective, the weight of a row, or 
the degree of a check node, indicates the number of unknown 
solutions or the level of uncertainty of the underlying equation. 
It is clear that the larger the degree of the check node is, the 
less reliable the output extrinsic information will be. Then, for 
a variable node, the sum of the degrees of all the check nodes 
connecting to it, to some extent, implies the uncertainty that 
the variable brings to the whole equation set defined by H. In 
other words, for columns of the same weight, deleting the one 
that has a larger row-weight sum can reduce more uncertainty 
in the equation set than those with smaller row-weight sums. 
Therefore, we take the row-weight sum, instead of 
"extrinsic-sum", as an important factor for the LDPC code 
shortening or information padding. Also, in this sense, the 
MRWC(A) can be unified with our proposed algorithm and 
criterion in the following section. 
C. Our Proposed Algorithms 
For LDPC codes, the ultimate parameter that characterizes 
its error-correcting capability is the decoding threshold. As 
is shown in [9], the threshold is determined by the degree­
distribution pair. Thus, the shortening of LDPC code is, in fact, 
to optimize the distribution pair to obtain the lowest possible 
decoding threshold under the constraints of the node degree 
profiles of mother code. From this consideration, we proposed 
a decoding threshold based LDPC shortening algorithm as 
listed in Algorithm 1. Unlike in [8], since the target is to 
optimize degree distribution pair, the set of candidate variable 
nodes includes those nodes corresponding to the information 
bits of all degrees instead of the lowest degrees. 
In Algorithm 1, the set of candidate variable nodes, Vinfo, 
are divided into subsets by their degrees, {Vi1fo}' Np are the 
set of integer partitions of np, in which the integers are the 
number of variable nodes of different degrees to be deleted in 
the bipartite graph G. For each deleting number combination 
N;, an optimal deleting pattern, V2, is obtained by running 
Step 2 to 8 and saved as well as its correspondent threshold 
Ti. Finally, among all the saved patterns, the pattern resulted

Indeed, involving variables of all degrees will lead to huge 
amount of possible deleting patterns for each N;. Thus, 
additional criteria are introduced in Step 5 to speed up the 
algorithm. As aforementioned, the maximum row-weight sum 
criterion can be adopted here and we will denote it as algo­
rithm MRWC(B). Note that the difference between MRWC(A) 
and MRWC(B) is in the choice of the candidate set. Moreover, 
we propose another criterion, denoted by MRWVC, which 
selects the variable node resulted in the matrix of the minimum 
row weight variance. This criterion, in other words, is to make 
the row degree of the residual matrix as regular as possible.

On one hand, theoretically, irregular row degree means some 
of the rows may have degree much larger than the average row 
degree while some have much lower degrees. It is good to have 
low degree check nodes since they emit much more reliable 
extrinsic messages than average; however, at the same time, 
those large-degree check nodes will emit much less reliable 
messages. This irregularity could lead to error floor when 
SNR is high. On the other hand, in practice, the decoder is 
implemented from the row perspective. Regular row implies 
that all the row process can be done in the same number 
of clocks with the degree of the rows. If there are rows of 
different degrees, the clock will depend on largest row degree, 
and this will harm the decoder throughput.

A. Asymptotic Performance 
In the following, the asymptotic performance of different 
LDPC shortening algorithms is evaluated by EXIT chart [10]. 
We use the Chinese DTMB LDPC codes as the example to 
implement information padding. There are three code rates, 
0.4,0.6 and 0.8. The column degrees are (3, 4, 11) for rate 0.4 
and 0.8 code, and (3, 4, 7, 11) for rate 0.6. Since the parity­
check matrix is quasi-cyclic, it can be considered as replacing 
the non-zeros entries of a low dimension matrix with 127 x 127 
circular matrices and the zero entries with 127 x 127 zero 
matrices. The low dimension matrix is called base matrix or 
proto-matrix. The dimension of the base matrices of DTMB 
LDPC codes are 35 x 59, 23 x 59 and 11 x 59, respectively. 
The shortening is done on the base matrices and each code 
rate is shortened to a practical rate, i.e., 0.4 to 0.33 , 0.6 to 
0.5 and 0.8 to 0.75. Therefore, 6 , 13 and 15 columns need 
to be deleted respectively for each base matrix.

V. CONCLUSIONS 
Modern DTV broadcasting systems have to adapt to a 
variety of channel conditions. Correspondingly, this paper 
proposes a new scheme of known-bits padding to generate 
multi-code-rate LDPC codes of unlimited granularity. With our 
scheme, a backward compatible LDPC code of desired rate can 
be easily attained from a mother code using certain shortening 
optimization algorithm. We apply our proposed scheme to the 
broadcasting system where the available code rates are limited, 
such as the Chinese DTMB. Numerical results demonstrate 
that our proposed scheme can obtain both code rate flexibility 
and decoding threshold optimization.
