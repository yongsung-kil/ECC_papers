In-Page Management of Error Correction Code for MLC Flash Storages

Abstract-Memory manufacturers have recently advanced
silicon technology to implement the multi-level cell technique
onto NAND flash for the reduction of per-bit device cost.
However, this technical improvement has introduced an
additional problem of reliability and/or durability degradation,
leading to the inevitable use of error detection and correction
techniques. To increase the number of correctable error bit in recent flash memories, ECC techniques tend to use longer code
bits. As the silicon technology of NAND device evolves, such
growing code bits for a user data page could overflow its
corresponding spare area in later devices. In this paper, we
propose a novel management mechanism of excessively long
error correction codes using user data area. The proposed
mechanism is capable of providing error correction capability
for highly error-prone NAND devices by efficiently managing long ECC codes only with negligible performance degradation.
I. INTRODUCTION
Since NAND flash memory has better random read/write performance than hard drives, it is used to build a storage device for server systems which usually handle many small 110 requests from users working simultaneously. Recently, NAND flash memory has been widely used to form solid state drives (SSDs) and mobile device storages because of its low power consumption and large capacity. Although the price of NAND flash memory is still higher than that of a hard disk drive, the NAND flash device market is expected to rapidly grow because of these technical advantages.
NAND flash device manufacturers have recently advanced silicon technology and adopted the multi-level cell (MLC) technique to reduce device cost. However, these efforts have caused the degradation in reliability and durability, leading to the wide use of error detection and correction techniques. In particular, the NAND flash controller used in storage units for highly reliable server needs to use a powerful mechanism to ensure demanded level of storage durability and reliability.
The NAND flash memory controller often uses an error correction code (ECC) to handle bit errors occurring due to inter-cell interference and durability reduction resulting from the aging of data blocks. The ECC code bits are generated during data transfer time and stored in the spare area with the data page during the page program time. There is no extra overhead for storing the ECC code because large-block NAND flash memory devices write both into the data area and into the spare area using a program operation (requiring the latency of tpROO)' The ECC code bits are decoded when the data are read out and returned to the host with the error­ corrected data during the page read time. The NAND flash memory controller normally uses a BCH code algorithm as an ECC. The BCH code algorithm requires a 13-bit ECC code bits to correct a I-bit error in a 512-byte sector. While the size of the spare area in the NAND flash memory increases linearly, the number of required bits for correcting bit errors in the NAND flash memory tends to increase exponentially. As shown in Table I, the BCH code algorithm becomes unsuitable for the most recent NAND flash devices due to the limited capacity of the NAND spare area.
The recent advent of 3-bit MLC technique and 2x or less nm silicon technology have led to a significant increase in errors per sector, which necessitates even more ECC code bits that may in turn overflow the spare area preserved for storing ECC codes. To cope with this challenge, we need an efficient ECC algorithm as well as a new mechanism for storing ECC code bits in flash memory devices. Recently, as an effort to enhance error correction capability, a low-density parity check (LDPC) [1] scheme has been introduced to flash memory storage systems, but the length of code words required to correct errors is inversely proportional to the time budget for processing the LDPC algorithm [2]. In other words, if the error detection and correction should be done with little time overhead, the code word would be long enough to require more space in spare area, as in BCH.
In this paper, we propose a management scheme of error correction code using data area in flash memory. If not properly managed, this approach may incur the waste of user data space as well as the degradation of access performance. The proposed mechanism is capable of providing error correction capability for highly error-prone NAND devices by efficiently managing long ECC codes only with negligible performance degradation.

The main idea of this paper is straight forward: we preserve some space in data area by storing less number of sectors than a page could hold and use the space to store longer (or stronger) BCH code bits for each sector. Both sectors and their long BCH code bits are stored in the same page. Therefore, there is no extra delay in fetching BCH code bits. Now it is possible to correct errors in a sector without any delay while the controller performs ECC decoding using BCH code bits. The management cost of error correction codes in IP-ECM is much smaller than that of the dual-BCH scheme, presented in Section 2, which stores longer (or stronger) BCH code bits in ECC blocks. The dual-BCH scheme requires additional storage space and management cost. This scheme reduces the overheads due to error correction process by first using short BCH code bits. In other words, only when there are bit errors that cannot be fixed using the short code bits, the stronger error correction process starts to work, which consequently incurs longer recovery latency when there are many bit errors. On the other hand, IP-ECM can encode and decode longer (or stronger) ECC code bits during read/write operations without additional overhead because it stores the BCH code bits in the corresponding data page.
B. IP-ECM with O-CFLRU
Like hard disk drives, NAND flash-based storage devices use a volatile memory buffer to achieve high performance. When the host system requests an 110 access to storage, the storage controller temporarily stores the write data into the buffer area or tries to read the requested data from the buffer to the host. The use of buffer memory contributes to hide a potential long latency incurred during 110 accesses and to aggregate requests to the same block (or page) in order to avoid high housekeeping overheads.
One of such buffer management algorithms is Clean-first LRU (CFLRU) [7-9], an LRU-based buffer management algorithm for the storage systems with NAND flash memory devices. Due to the unique "erase-before-write" characteristics of NAND flash memory, the buffer management algorithm victimizes clean pages (or blocks) in buffer when the buffer runs out of empty slots.
In this paper, we have investigated the effect of using a data buffer and its management scheme to IP-ECM. We assume the CFLRU algorithm is used with a given size of buffer space. In our implementation of CFLRU, called 0CFLRU, we allocate or flush buffers in page, that is, the sector requests to the same page are grouped together regardless of the age and type of requests. Another variation is to separate incoming requests into small random requests and large sequential requests based on request size. As illustrated in Fig. 5, if the sector count of an 110 request is higher than a fixed threshold (e.g. 32), the storage controller bypasses the request and stores the data into the NAND flash memory area directly. Fig. 8 represents the normalized NAND flash operation latencies of C-ECM and IP-ECM. For experimental purposes, a trace-driven SSD simulator was developed as a part of this research. The experiments were performed using a trace extracted from the TPC-C benchmark tool and transferred into the SSD simulator. The storage space for BCH code bits was obtained by removing one or five sectors per clustered page. As shown in Fig. 8, when the clustered page size is 16 KB, the difference in normalized NAND flash operation latency between C-ECM and IP-ECM is at most 5% when one sector is used for additional ECC storage. The number increases but remains at 12% when five sectors are used. And, when the size of the clustered page is 128 KB, there is little difference in performance.
This performance observation is also true for the configurations with the O-CFLRU buffer management scheme and varying buffer size. When the buffer management is applied, the flash operation latency is reduced by about 30% for both C-ECM and IP-ECM. However, when the buffer is involved in data transfer, the performance difference between C-ECM and IP-ECM decreases to around 6%. As previously stated, the buffer is useful to aggregate multiple sector requests heading toward the same page to make one NAND page request. So, it is possible that IP-ECM may perform two page operations for the request which otherwise can be processed using a page operation in C-ECM. However, from the simulation results, we observed that aggregated requests are much smaller than a page size in IP-ECM for most cases.

V. CONCLUSION
In this paper, we proposed an in-page error correction code management scheme necessary to provide an enhanced ECC capability. The proposed scheme, called IP-ECM, contributes to the enhancement of storage reliability and durability by efficiently managing long ECC code bits in user data area.
Even when a buffer management increases the possibility of performing multi-page operations, otherwise processed using one page operation, the simulation results reveal that there is only negligible degradation due to this effect. Considering that the bit error rate increases as the semiconductor technology evolves, the proposed scheme could be used to enhance overall storage reliability.
