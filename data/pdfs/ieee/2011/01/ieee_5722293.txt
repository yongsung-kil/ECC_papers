A 4.32 mm2 170mW LDPC Decoder in 0.13μm CMOS 
for WiMax/Wi-Fi Applications 
Dan Bao, Chuan Wu, Yan Ying, Yun Chen*, and Xiao Yang Zeng 
State Key Lab. of ASIC and System, Fudan University, Shanghai, 201203, China  
{081021040, 081021039, 082021017, chenyun, xyzeng}@fudan.edu.cn 
Abstract - An energy-efficient programmable LDPC decoder is 
proposed for WiMax and Wi-Fi applications. The proposed 
decoder is designed with overlapped processing units, flexible 
message passing network and medium-grain partitioned 
memories to achieve flexibility, area reduction, and energy 
efficiency. The decoder can be programmed by host processor 
with several special-purpose micro-instructions. Thus, various 
operation modes can be reconfigured. Fabricated in SMIC 
0.13μm 1P8M CMOS process, the chip occupies 4.32 mm2 with 
core area 2.97 mm2, and consumes 170mW with a throughput of 
302Mb/s when operating at 145MHz and 1.2V.  
I Introduction  
LDPC codes have received great interest in recent years, 
and are incorporated as forward error correction (FEC) device 
in various broadcasting and wireless (BW) standards, such as 
DVB-S2/C2/T2, CMMB, IEEE 802.11n (Wi-Fi) [1], and IEEE 
802.16e (WiMax) [2]. In applications, two or more BW 
standards are merged into a single terminal to provide diverse 
services, e.g. WiMax and Wi-Fi can be integrated to provide 
mobile communication services and local area network 
services [3]. By the fact that FEC decoders, especially LDPC 
decoders, occupy large chip area in the BW systems, 
programmability is desired in the decoder design, which can 
lead to a compact terminal.  
This paper proposes a flexible LDPC decoder architecture 
with low power consumption and chip area. The architecture 
supports various code lengths, rates, and code matrices. The 
decoder covers all 126 modes of WiMax and Wi-Fi LDPC 
codes. The decoder can operate as an FEC accelerator 
controlled by host processor. 
II. QC-LDPC Codes and Decoding Algorithm 
Quasi-cyclic (QC) LDPC codes are well studied for their 
inherent regularity to facilitate decoder design. The parity 
check matrix of (n, k) QC-LDPC codes can be split into 
multiple p×p blocks defined by permutation size s. Thus, the 
parity check matrix has mb block rows (block layers) and nb
block columns, where mb=(n-k)/p, and nb=n/p, and can be 
expressed by a mb×nb base matrix with s as its elements. 
Due to high complexity, original belief propagation (BP) 
algorithm has been rarely used for LDPC decoder 
implementation. Simplified BP-based algorithm with slight 
performance loss, such as MS, is suitable for decoder design 
[4]. Layered scheduling can accelerate decoding convergence 
speed by a factor of 2 [5]. LDPC decoders based on min-sum 
and layered scheduling can attain small area, large throughput, 
and low power dissipation [8]. This paper adopts Modified 
MS algorithm [9] and layered scheduling with optimized 
overlapping to achieve low complexity, throughput boosting, 
and power reduction. 
III. Proposed Architecture and Design Techniques 
Fig. 1 shows the proposed decoder architecture, and it is 
composed of the following building blocks: 1) a control 
interface, i.e. a serial-to-parallel interface (SPI), which can be 
accessed by host processor; 2) code matrix memory (CMM) 
that stores the base matrix information; 3) a control block 
(CTRL) that generates all control signals for the other blocks; 
4) an input/output buffer for initialization of channel LLR and 
sending the hard decisions; 5) two medium-grain partitioned 
memory groups for posterior messages (EMs) and extrinsic 
messages (EMs), i.e. PMM and EMM, respectively; 6) 
scalable processing units (PUs) with overlapping; and 7) a 
message exchange network supports various code lengths and 
permutation size [9].  
The PUs consists of min-operation (MOP) and sum-operat- 
ion (SOP), which perform check node update and bit node 
update, respectively, and are fully overlapped between two 
subsequent block layers. To attain small chip area, the 
min-operation is folded by register feed-back (serial scanning), 
and sum-operation is executed with the same rhythm. The 
2-cycle pipeline stalls in [9], which lead to partly overlapping, 
are removed by reordering the process order. Thus, clock 
cycles required for MOP and SOP are the same, which enables 
seamless overlapping of the two operations. 
The PMM module is optimized with two methods. Based 
on the layered decoding, each layer operation has to read and 
write row-weight (dc) posterior messages.  Investigation to 
the parity-check matrix shows that many of the non-zero 
entries of two block layers have the same location, which 
indicates that access to PMM can be reduced by directly 
passing the message to the next layer without written back [9].

Searching an optimal layer concatenation order can lead to an 
optimized layered-decoding with minimum PMM access 
number. And the other optimization comes from various 
operation modes of the decoder. When the decoder operates at 
shorter code length or higher code rate, part of the memory 
capacity is never accessed, which can be in a standby mode. In 
the proposed decoder, PMM is medium-grain partitioned into 
two groups with 8 banks, respectively. The other memories are 
also partitioned, including the input/output register files with 
12 banks and the EMM with 34 banks. The partition brings 
energy saving with slight area overhead (5%). 
IV. Test Results 
The decoder is fabricated in SMIC 0.13μm 1P8M CMOS 
technology. Its die photo is shown in Fig. 2, and the chip area 
is 4.32mm2 with a 2.9725mm2 core (including power/ground 
ring area of 0.1725mm2).  Full-loading operating at 145MHz 
and 1.2V, the chip is tested with a power consumption of 
170mW and an energy efficiency of 52 pJ/bit/iter. 
Fig. 3 shows the error correction performance of the 
decoder chip in an AWGN channel for 2304-bit WiMax codes. 
The maximum iteration number (max.iter.) can be tuned by 
host processor, according to different throughput requirement. 
Table I summarizes the comparison of this work with 
recently reported WiMax decoder chips. The core area of the 
decoder is reduced by 30%, compared with [6]. The 
throughput advances the decoders [6]-[8] by 2-3 times. The 
energy efficiency (EE) is improved by 50%-75%, compared 
using normalized energy efficiency (NEE, normalized to the 
same process and code parameters, i.e. 2304-bit rate-1/2).

V. Summary and Conclusions 
An energy-efficient flexible LDPC decoder is proposed for 
WiMax and Wi-Fi applications. The decoder can be 
reconfigured by host processor, and support various code 
lengths, rates, and code matrices. Based on overlapped layered 
scheduling and optimized architectures, the decoder achieves 
small area, high throughput, and high energy efficiency.
