A Sorting-based Architecture of Finding the First Two Minimum Values for LDPC Decoding
Qian Xie#1, Zhixiang Chen, Xiao Peng, Satoshi Goto
Graduate School of Information, Production and Systems, Waseda University Kitakyushu, Fukuoka, Japan
1xieqian@akane.waseda.jp

Abstract—This paper presents an efficient architecture of finding the first two minimum values for row operation in LDPC decoding. Given a set of numbers X, efficient algorithm and its corresponding hardware implementation for finding the first minimum value, min_1st, second minimum value, min_2nd and the position of min_1st are greatly needed in LDPC decoder design. The design is based on sorting-based approach proposed in[10]. Compared to the conventional architecture, our architecture performs better in both speed and area. An extension method is also presented to apply the proposed architecture when the number of inputs is an any positive integer.
Keywords— LDPC codes, LDPC decoder, min-sum algoritm, minimum finder, sorting-based architecture.
I. INTRODUCTION
A LDPC code is a linear block code defined by a sparse parity-check matrix H that contains mostly zeros and only a small number of ones, that is, it has a low-density of ones. Recently, Low-Density Parity-Check (LDPC) code [1] as an error correcting code is adopted by many communication standards such as DVB-S2 , WiMax (IEEE 802.16e) and WPAN (IEEE 802.15.3c) [2] standards because of its nearcapacity error correction performance and potential for highthroughput decoding. Among a variety of decoding algorithms, belief propagation (BP) algorithm achieves a good decoding performance. For the standard BP algorithm in log-likelihood ratio (LLR) domain, a lot of logarithmic and multiplicative computations are required for the check node computation. The min-sum (MS) algorithm, which is an alternative method, selects the first minimum message and the second minimum message among several passed messages from the variable nodes. The MS algorithm can significantly reduce the hardware complexity of the BP algorithm at the cost of performance degradations, where complex computations at the check nodes can be implemented with simple comparison operations. Recently, the modified MS algorithms such as normalized MS or offset MS using additional correction factors have been preferred for many practical applications since they offer comparable decoding performance over the BP algorithm for regular LDPC codes[3][4]. In[5][6], the improved normalized and offset MS algorithms are exhibit for irregular LDPC codes with small performance degradations.
Each iteration in MS algorithm is mainly composed of two phases: row operation and column operation. In row operation, a unit (minimum finder) to find the first two minimum values and the position of the first minimum value is needed. For IEEE 802.15.3c standard, 21 min finders are needed if one

layer composed of 21 rows in the PCM are processed together. In larger codes, the number of minimum finders may be even larger. Thus, low cost minimum finder is greatly needed in LDPC decoder design.
Given a set of number X={x0, x1, …, xn-1}, find the first minimum value (min_1st), second minimum value (min_2nd) and the position of first minimum value (pos). Min_1st can be selected by (n-1) comparisons, while the number of comparisons for finding Min_2nd is different according to different algorithms. Generally, some algorithms and their implementations find min_2nd by finding the minimum value excluding min_1st, e.g. [9] , which needs (n-2) comparisons. [7] and [8] describe a sorting-based minimum comparison selection algorithm to find the tth largest of n elements. This algorithm can find min_2nd with just ⎡⎢log2 n⎥⎤ −1 comparisons.
In [10], sorting-based algorithm (XS approach), tree structure-based algorithm (TS approach) and their hardware implementations are proposed. However, their XS approach performs worse in area although it uses fewer comparators than TS approach. We propose another architecture based on their XS approach which can reduce area. Theoretically our proposed sorting-based architecture can achieve better performance in area than TS approach but worse in speed.
The organization of this paper is as follows. In section Ⅱ the minimum comparison selection algorithm and the conventional XS architecture in [10] are presented. Section Ⅲ describes our proposed sorting-based architecture in detail and shows synthesis result. A method to extend our architecture to any positive number n is presented in Section Ⅳ. Section Ⅴ concludes the paper.

IV. ARCHITECTURE OF PROPOSED SORTING-BASED APPROACH FOR ANY POSITIVEINTEGER N
Considering actual applications, n cannot always be represented as n = 2k, where k is a positive integer. In [10], authors propose an extension method of their TS approach for any positive integer n. Their TS approach consists of 2-mVGs and connection units (CN). CN is constructed with 3 mVUs and 1 MUX_w. 2-mVGs only exist in stage0 and the indexG module is not used for finding min_2nd. Their method focuses on arranging CNs and changing architecture of some CNs in detail. Our XS approach is mainly consists of 2-mVGs so that we focus on arranging 2-mVGs. Since the architecture of XS approach is different from TS, we propose our extension method which seems more suitable for XS approach.
