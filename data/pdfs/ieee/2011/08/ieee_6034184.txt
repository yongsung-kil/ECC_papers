Partially-Regular LDPC Codes with Linear Encoding Complexity and Improved Thresholds

Dmitri K. Zigangirov‡, Kamil Sh. Zigangirov‡, Daniel J. Costello, Jr.†,
‡Institute for Problems of Information Transmission, Moscow, B.Karetny per. 19, Russia †Department of Electrical Engineering, University of Notre Dame, Notre Dame, IN 46556, U.S.A.
E-mail: { zig@iitp.ru, kamil@eit.lth.se, costello.2@nd.edu}

Abstract—We consider an ensemble of systematic lowdensity parity-check (LDPC) codes of length N with linear encoding complexity, i.e., with complexity O(N ). We call these codes partially-regular, since they can be considered as modifications of regular LDPC codes. Further, their iterative decoding thresholds on the binary erasure channel (BEC) are found to be significantly better than the thresholds of the corresponding regular LDPC codes.

To analyze the iterative decoding performance of partially-regular LDPC codes, we calculate their thresholds on the binary erasure channel (BEC). We find that the partially-regular codes have significally better thresholds than the corresponding regular LDPC codes. In the following, we analyze in detail partially-regular (6M, 3, 6) LDPC codes and formulate the results for partially-regular (8M, 4, 8) LDPC codes.

I. INTRODUCTION

II. PARTIALLY-REGULAR LDPC CODES

The problem of efficient encoding of LDPC codes has been considered in several papers, e.g., [1]– [4]. In this paper, we introduce a new class of partiallyregular LDPC codes obtained by modifying regular (KM, J, K) LDPC codes with parity-check matrices constructed from permutation matrices of size M [5], where N = KM is the code length. The new partiallyregular LDPC codes are systematic, they have linear encoding complexity, i.e., complexity of O(N ), and they have a slightly irregular structure. For example, the Tanner graph of partially-regular (6M, 3, 6) LDPC codes has 2/3 of the variable (symbol) nodes of degree 3, 1/3 of the variable nodes of degree 2, 2/3 of the check nodes of degree 6, and 1/3 of the check nodes of degree 4, i.e., the average degree of the variable nodes is 2.66 and the average degree of the check nodes is 5.33. Similarly, partially-regular (8M, 4, 8) LDPC codes have an average variable node degree of 3.75 and an average check node degree of 6.75. (We consider only modifications of regular (6M, 3, 6) and (8M, 4, 8) LDPC codes with design rate Rc = 1/2 in this paper, although the generalization to arbitrary rates is straightforward.)

III. COMMUNICATION OVER A BEC USING PARTIALLY-REGULAR LDPC CODES
Now consider communicating over a BEC with erasure probability . We assume that partially-regular (6M, 3, 6) LDPC codes are used and that the girth of the Tanner graph is g = O(log M ), such that in decoding we have 0 = (g − 4)/4 independent iterations2. Suppose that the received sequence is r = (r(1), r(2), r(3), r(4), r(5), ˜r(6)), where again ˜r(6)) denotes the fact that the last symbol in the 6th subblock is deleted. The symbols of this sequence are drawn from the ternary alphabet {0, 1, ∆}, where ∆ is the erasure symbol. We assume that the iterative message-passing algorithm [7] is used for decoding.
In the case of partially-regular (6M, 3, 6) LDPC codes, the Tanner graph has two subsets of variable nodes. The first subset is a group of nodes corresponding to symbols that are included in three parity-check equations, i.e., nodes having degree three. These are the nodes from subblocks v(1), v(2), v(3), and v(4). In the sequel, we say that these variable nodes are from

The check nodes of the Tanner graph can also be split into two subsets. The corresponding equations depend on the entries of submatrices P(11), P(12), P(13), and P(14) of (1). In the sequel, we say that these check nodes are from group 1. The second subset includes the check nodes connected to variable nodes corresponding to symbols in all 6 subblocks of a code word. These check nodes have degree 6. In the sequel, we say that these check nodes are from group 2.
The iterative message-passing decoding algorithm operates as follows. Assume that, at the beginning of the decoding process, the symbols of the received sequence r are associated with the corresponding variable nodes in the code word v. After each iteration, the decoder reconstructs some erased symbols and then uses the modified received sequence r for the next iteration.
In the first half of the first iteration, all variable nodes send messages to their connected check nodes. The message is the value of the corresponding received symbol , i.e., 0, 1, or the erasure symbol ∆. In the second half of the first iteration, all check nodes send messages to their connected variable nodes. The message that a check node sends to a given connected variable node depends on the messages that the check node received from all its other connected variable nodes in the first half of the iteration. If at least one of these messages was the erasure symbol ∆, then the check node also sends an erasure symbol to the given variable node. If a check node received no erasure symbols from these other nodes, i.e., all these received symbols were either 0 or 1, the check node sends the modulo-2 sum of these symbols to the given variable node.
In the followings iterations, variable nodes corresponding to nonerased symbols retain these symbols independently of the messages received from their con-

nected check nodes and continue to pass the same messages. If a variable node corresponding to an erased symbol receives from at least one connected check node a 0 or 1 symbol, it corrects the erasure, resulting in a modified received sequence r , and on all following iterations it passes this symbol to its connected check nodes. Decoding continues until all symbol nodes have 0 or 1 symbols, i.e., the modified received sequence r does not contain any erased symbols, or until the sequence r becomes a stopping set.
IV. BEC THRESHOLDS FOR PARTIALLY-REGULAR LDPC CODES
Our goal is to calculate the thresholds thresh of partially-regular LDPC codes for a BEC with erasure probability . By definition, if ≤ thresh, partiallyregular LDPC codes exist with block length N = KM such that the iterative message-passing decoding block error probability Perr of the code goes to zero as N → ∞.
We assume that the decoder operates on a cyclefree graph, e.g., partially-regular (6M, 3, 6) LDPC codes have girth g = O(log N ) → ∞ as N = 6M → ∞, and the number of decoding iterations does not exceed 0 = (g − 4)/4 .

V. CONCLUSION
In this paper, we have introduced a new class of LDPC codes with linear encoding complexity. The new codes are modifications of well-known regular LDPC codes and are therefore called partially-regular LDPC codes.
