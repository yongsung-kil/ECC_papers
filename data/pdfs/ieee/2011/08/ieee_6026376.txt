Abstract- This paper presents a partially parallel LDPC decoder 
architecture for QC-LDPC codes. In particular, we introduce a 
check node processing element which is 3-parallel, adjustable to 
irregular 
inputs 
and 
easily 
expandable. 
Furthermore, 
our 
decoder is applicable to multi-rate system by simply writing 
additional data to internal RAM. In another aspect of our work, 
we can reduce the check-bit message memory significantly by 
efficient method. Implementation results show that the proposed 
architecture can support the data rate of 360Mbps in FPGA. 
Keywords- LDPC, Architecture, Quasi-cyclic, FPGA 
I. 
INTRODUCTION 
The wired or wireless communication technology has been 
rapidly progressed as semiconductor technology has been 
improved. Nevertheless, need for more high speed transmission 
technology has been gradually increased. In wireless network, 
data rate over several hundred mega bits/s is about to be 
utilized. To support stable data transmission, strong forward 
error correcting code like Turbo code or LOPC code is needed. 
LOPC code has advantage on the parallelism, so it is more 
suitable for high speed transmission. But the difficulties of 
implementation arise when the LOPC architecture should 
support high speed with low power. 
There have been many studies on LOPC decoder 
architecture. They are highly dependent on code generation 
format. Random LOPC code has better performance but it is 
difficult to implement. So, structural LOPC code has been 
progressed. The structural LOPC codes that partition H matrix 
into pseudo-permutation matrices successfully reduce the 
memory of the decoder and attains high throughput[l]. Another 
approach to effective LOPC decoder has been developed from 
decoding algorithm. The belief propagation algorithm is 
typically used for decoding LOPC codes. Recently, many 
notable works show that layer based belief propagation 
algorithm can reduce the number of iteration to about a half of 
that of the belief propagation algorithm[4]. It is a very 
important breakthrough to make high speed architecture. 
In this paper, we will review the representative techniques 
for high throughput LOPC decoder in code design and 
decoding algorithm. They are Quasi-cyclic LOPC (QC-LOPC) 
code and layered decoding respectively. Using these base

The idea of layered decoding is to divide all check nodes

matrix. In the layered decoding, computing (5)-(7) at each layer

iteration of decoding algorithm is composed of m sub­
iterations. 
It is well known that layered decoding can reduce the 
number of iteration to a about a half. Because the throughput of 
LDPC decoder is proportional to the number of iteration, the 
layered decoding is important to our decoder architecture.

III. 
PROPOSED LDPC DECODER 
A. Layer based parallel LDPC decoder 
Fully parallel LDPC decoder which has check node blocks 
for all check nodes obtains high throughput but consumes large 
area to implement. Therefore, we have to consider partially 
parallel architecture. The representative one was provided 
along with layered decoding algorithm [4]. It has z parallel 
check node blocks, where z is the size of sub-matrix, and 
incoming messages to check node are entered one by one. We 
change the check node block to 3-parallel architecture, which 
leads to increase the throughput 3 times. Figure 2 shows the 
proposed LDPC decoder architecture. The encoded word 
length is 1200 bits and the size of sub-matrix is 30x30, which 
corresponds to the WiMedia LDPC codes.

In figure 2, the check node block has 30 check node 
processing element, which computes 3 message values at a 
time. Using this check node PE, the message between memory 
and other blocks is exchanged by 3-parallel signal. 
Three memory banks (MB1, MB2 and MB3) store the bit­
check messages and these 3 MBs divide 40 columns of H 
matrix into 3 blocks. The MB 1 recodes 14 columns and the 
MB2 and MB3 recodes 13 columns. So, the address of each 
MB is controlled by 4 bits. The addresses of MBs are written 
parallel in the execution order RAM and they are read by time 
order. The execution order defmes the time order of bit-check 
messages, entering the check nodes, during an iteration. It has 
to be modified when the code rate changes, so the execution 
orders for all rates have to be stored. The execution order is 
also variable by the selection of the scheduling method, so it is 
needed to record the order using RAM. 
The multiplexer at the data input side is for the selection of 
external log likelihood ratio (LLR) at initial time of iteration. 
The shifter block is composed of 3 Barallel shifter[9]. The shift 
RAM outputs the shift value according to execution order. The 
shift value is the difference between the former shift value and 
the current shift value for the same sub-matrix. The R RAM 
block stores the check-bit node messages (Rmn) of all check 
node PEs as many as execution order.

V. 
CONCLUSION 
In this paper, we have proposed the LDPC decoder 
architecture for multi-rate quasi-cyclic LDPC code. Proposed 
architecture attains throughput enhancement by using 3 parallel 
check node processing element. This element can be adapted 
easily to irregular check node degree. Flexibility of proposed 
decoder makes user to support multi-rate by changing the 
content of internal RAM. We also applied the memory efficient 
method for check-bit message RAM, which reduces the 
memory size to 48% of the usual method. After hardware 
implementation in a FPGA, we have specified the proposed 
architecture could support 360Mbps at maximum.
