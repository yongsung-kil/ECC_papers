Abstract—In this work we address the design of degree distributions (DD) of low-density parity-check (LDPC) codes for the tree-expectation propagation (TEP) decoder. The optimization problem to find distributions to maximize the TEP decoding threshold for a fixed-rate code can not be analytically solved. We derive a simplified optimization problem that can be easily solved since it is based in the analytic expressions of the peeling decoder. Two kinds of solutions are obtained from this problem: we either design LDPC ensembles for which the BP threshold equals the MAP threshold or we get LDPC ensembles for which the TEP threshold outperforms the BP threshold, even achieving the MAP capacity in some cases. Hence, we proved that there exist ensembles for which the MAP solution can be obtained with linear complexity even though the BP threshold does not achieve the MAP threshold.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are well known capacity-approaching (c.a.) linear codes. Originally proposed by Gallager in the early sixties [1], a linear time practical decoding algorithm, the belief propagation (BP) decoder, was not introduced until the mid-nineties [2]. Remarkable works on the analysis of LDPC codes under BP decoding for discrete memoryless channels (DMCs) can be found in [3], [4], [5] and the references therein.
The BP limiting performance when the code-length goes to infinity for a given LDPC code is analyzed in [6], [7] for the binary erasure channel (BEC) to obtain the so-called threshold [5], which defines maximum decodable fraction of errors, is obtained. To compute the BP threshold, denoted hereafter as BP , density evolution (DE) [3] and extrinsic information transfer (EXIT) charts [5] are presented as suitable tools. The design of c.a. LDPC codes is then intimately linked with the decoding algorithm. Under BP decoding, c.a. LDPC ensembles are found by optimizing the BP-DE solution [7], [8]. The BP optimization naturally sets up a very high fraction of degree two variables and it has a negatively effect in the finite-length performance of these codes [5], [6]. Indeed, for those ensembles for which BP equals the MAP threshold, MAP , the LDPC ensemble presents an irreducible error floor since all codes in the ensemble asymptotically have a very low minimum distance [9], [10]. Hence, these codes are not very interesting from the practical point of view.
In this paper, we focus on the design of c.a. codes for BEC and for the TEP decoder. The TEP decoder was first introduced

in [11] as a decoding algorithm for LDPC codes over DMCs. The TEP decoder is able to improve the BP solution thanks to a more accurate estimation of the posterior probability for each bit. Furthermore, for the BEC, the TEP decoder, as well as the BP decoder, has linear complexity with the code length. Hence, better performance can be obtained at linear cost. For finite-length regular and irregular LDPC codes, the TEP decoder significantly improves the system performance, specially in the so-called waterfall region [12].
The TEP decoder limiting performance for BEC is analyzed in detail in [11], where it is shown that it achieves higher thresholds than the BP decoder for the same LDPC ensemble. The TEP threshold is a nonlinear function of the LDPC distribution parameters and, unfortunately, we do not even an have analytic expression of it [11]. We derive tractable methods to improve the TEP threshold. The proposed approximations to this nonlinear problem are based on the known behavior of the peeling decoder (PD) for LDPC decoding [7]. The PD is an alternative formulation to the BP decoder in the BEC. Indeed, the TEP decoder is formulated as an improved PD. We optimize the LDPC graph so that when we are above the BP threshold, the PD residual graph optimizes the performance of the TEP decoder. We show that this optimization suits the TEP decoder, with thresholds much higher under TEP decoding than under BP decoding. Besides, we provide some examples for which the TEP decoder achieves the MAP threshold.
The rest of the paper is organized as follows. In Section II, we shortly review the BP decoder for BEC and the LDPC ensemble optimization based on the DE solution for the BP. Section III is devoted to present the TEP decoder for the BEC channel. In Section IV we analyze the threshold achieved for some LDPC codes and propose some schemes to optimize it. In Section V, we provide some particular results. We end with some conclusions in Section VI.
II. BP THRESHOLD MAXIMIZATION
The BP algorithm exhibits an alternative and simple formulation for the BEC, in which the non-erased variable nodes are removed from the graph in each iteration [7]. The BP, under this interpretation, is referred to as the peeling decoder (PD) [3] and it is easily described using the Tanner graph of the LDPC code [5]. III. THE TEP DECODER.
The TEP decoder can be seen as an extension of the PD [11], in which we also process the degree two check nodes. Assuming again that the all-zero codeword is transmitted1 and given the LDPC Tanner graph once we have removed all the non-erased variables, in every iteration the TEP randomly performs one of the two following steps:
1) Remove a check node of degree one along with the variable it is connected to. This is the PD basic iteration.
2) Remove a check node of degree two. A check node of degree two informs us that both variables are identical. This process is sketched in Fig. 1. The variable V1 heirs the connections of V2 (solid lines) in Fig. 1(b) and the check P1 and the variable V2 are removed.

In [11], it is proved that the TEP solution does not depend on the scheduling, i.e., the order in which checks nodes of degree one and two are removed. In particular, we use the stage scheduling (s.s.) for TEP: under s.s., the TEP decoder sequentially removes all the check nodes of degree one and, then, all the check nodes of degree two. Note that, under s.s., we first run the PD and then the TEP removes all the check nodes of degree two. If check nodes of degree one have been created after this stage, the process is restarted.

VI. CONCLUSIONS
In this work we have addressed the design of LDPC ensembles for the TEP decoder. The TEP decoder has been proposed to improve the BP decoder for LDPC decoding for both the finite-length case and the asymptotic case. The TEP decoder, as well as the BP, has linear complexity with the codelength. We exploit the known behavior of the TEP decoder for arbitrarily large codes to find ensembles for which we appreciably improve the BP threshold. We show that the optimal design can be significantly simplified using the analytic expressions of the peeling decoder. We have provided LDPC ensembles for which the TEP decoder considerably

improves the BP capacity, achieving the MAP solution in some of them. We show that the set of LDPC ensembles for which we can achieve the MAP solution at linear cost is not restricted to those ensembles where the BP threshold equals the MAP threshold.
