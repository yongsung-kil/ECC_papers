Asymptotic estimation of error fraction corrected by binary LDPC code
Pavel Rybin∗, Victor Zyablov† Institute for Information Transmission Problems
Russian Academy of Sciences Moscow, Russia
Email: {prybin∗, zyablov†}@iitp.ru

Abstract—This paper considers new lower bound on fraction of guaranteed corrected errors while decoding the same binary low-density parity-check (LDPC) codes with constituent single parity-check (SPC) and Hamming codes using the same iterative low-complex hard-decision algorithm as in previous works of V. Zyablov and M. Pinsker in 1975 and V. Zyablov, R. Johannesson and M. Loncar in 2009. The number of decoding iterations, required to correct the errors, is a logarithmic function of the code length. The fraction of guaranteed correctable errors computed numerically for various choices of LDPC code parameters with constituent SPC and Hamming codes shows that proposed lower bound gives the better results than previously known best lower bounds obtained by V. Zyablov and M. Pinsker in 1975 for Gallager’s LDPC codes and A. Barg and A. Mazumrad for Hamming code-based LDPC (H-LDPC) codes in 2011. Some of obtained numerical results are represented at the end of the paper to demonstrate these improvements.
I. INTRODUCTION
The error-correcting capabilities of Gallager’s LDPC codes [1] for the binary symmetric channel (BSC) were studied in [2], where it was shown that such Gallager’s LDPC codes exist that capable of correcting a portion of errors that grows linearly with the code length n, with decoding complexity O (n log n). Then in [3] an analytical lower bound was obtained by combinatorial methods. But the numerical results for this lower bound gives smaller values in most cases than for lower bound from [2]. That’s why we don’t consider the lower bound from [3] in this work. The result similar to [2] was proven in [4], [5] and [6] for expander codes.
Hamming code-based LDPC (H-LDPC) codes were first studied in [7]. Distance properties and iterative soft-decision decoding of the H-LDPC codes were further investigated in [8] and [9]. In [10], it was shown that an ensemble of HLDPC codes contains codes with a minimum distance that asymptotically almost meets the Varshamov-Gilbert bound. Then result similar to [2] was obtained in [11] for H-LDPC codes using the generalized methods from [2]. After that in [12] the lower bound for codes on graph was obtained. For H-LDPC codes proposed lower bound gives the better results than in [11].
In this paper, we consider the same Gallager’s LDPC codes and H-LDPC codes and corresponding iterative low-complex hard-decision algorithm as in [2] and [11] respectively. We obtain new lower bound on fraction of guaranteed corrected errors using the methods developed in [2] and generalized in

[11]. The main difference is in our approach. We consider the edges with special properties of corresponding Tanner graph [13] instead of number of constituent codes with unsatisfied checks as it was done in [2] and [11]. Numerical computation for various choices of LDPC code parameters with constituent SPC and Hamming codes at the end of this paper shows that our lower bound gives better results than previously known best lower bound obtained in [2] and [12].

The generalized LDPC code with constituent SPC code we will call Gallager’s LDPC code. The parity-check matrix H of Gallager’s LDPC code has n0 ones in each row and ℓ ones in each column. Such matrix specifies an (ℓ, n0)-regular LDPC code. The generalized LDPC code with constituent Hamming code we will denote as H-LDPC code. The parity-check matrix of an H-LDPC code is, in general, irregular.
The generalized LDPC codes from ensemble EH0 (n0, ℓ, b) contain ℓb constituent codes (b in each layer) with parity-check matrix H0 and have length n = bn0 . Such codes can be represented by a bipartite Tanner graph [13] G = (V1 : V2, E) with n = bn0 variable nodes V1 , and ℓb constraint nodes V2, as illustrated in Fig. 1. Each constraint node comprises m0 parity-check constraints specifies by the rows of corresponding parity-check matrix. If a codesymbol is checked by a constituent code, there is a edge from E connecting the corresponding variable node from V1 and the constraint node from V2 . According to the construction of LDPC codes, each codesymbol is checked by exactly one constituent code in each layer. The graph is regular, with the variable-node degree equal to ℓ, and constraint-node degree equal to n0.
Consider communication over a BSC using generalized LDPC codes with hard-decision decoding. Let v be the transmitted codeword and e be the error pattern. Then the received sequence is given by r = v + e. The weight of the error sequence is W = ∥e∥ and fraction of erroneous symbols is ω = W/n. For code length n → ∞, the fraction of erroneous symbols ω converges in probability to the crossover probability of BSC.

III. DECODING ALGORITHM
The decoding algorithm is similar to the algorithm from [2] and [11]. The main idea is that the weight of generalized syndrome decreases on each iteration of decoding algorithm. We just formulate it in the terms of correcting edges: on each

352

iteration we invert one symbol after another that corresponds to the variable node that is incident with more than ℓ/2 correcting edges. It is clear that inverting of such symbols reduces generalized syndrome weight, because more than ℓ/2 constituent code becomes satisfied and we shouldn’t take into consideration the rest less part of constraints.
Consider an iterative hard-decision decoding algorithm A, whose decoding iteration i, i = 1, 2, ..., imax, consists of the following two steps:
1) consider the symbols of tentative sequence r(i), where r(1) is received sequence r, one after another:
• determine correcting edges that are incident with the current symbol;
• invert current symbol if it is incident with more than ℓ/2 correcting edges (update the sequence);
• proceed to the next symbol of the current sequence; 2) consider the updated sequence r(i) obtained on the
previous step:
• if syndrome of LDPC code for the updated sequence becomes zero (i.e. there is no constituent code with unsatisfied checks), algorithm returns updated (corrected) sequence r(i), sets successful decoding flag and stops execution;
• otherwise if generalized syndrome weight for updated sequence decreases, algorithm proceeds to the next iteration i + 1 with sequence r(i+1) that is equal to the updated sequence r(i);
• otherwise algorithm sets decoding failure flag and stops execution.
It is important to note that according to the description of algorithm A new errors can be added to the tentative sequence while decoding. So, algorithm A doesn’t guarantee that only erroneous symbols are corrected, new one may be added.
If received sequence contains error pattern with weight W , then it is more convenient for us to consider the number of correcting edges as fraction of W ℓ edges, i.e. γW ℓ, where 0 ≤ γ ≤ 1 is a fraction of correcting edges.
Let introduce some useful notations:
• γ0 is such fraction of correcting edges that it is guaranteed that algorithm A inverts at least one symbol on iteration if number of correcting edges is more than γ0W ℓ;
• Wγ0 = ωγn is such number of erroneous symbols in received sequence that such LDPC code in ensemble EH0 (n0, ℓ, b) exists for which number of correcting edges is more than γ0W ℓ for any error pattern with weight W ≤ Wγ0 in corresponding Tanner graph;
• α0 is such fraction of error pattern weight Wγ0 that if initial number of erroneous symbol in received sequence is less than α0Wγ0 then it is guaranteed that error pattern weight of tentative sequence is less than Wγ0 wile decoding, in other words, it is guaranteed that received sequence will be corrected.
Because the algorithm is similar to the algorithms in [2] and [11] it is easy to show the following:

Lemma 1: If the fraction of correcting edges γ > γ0 + ε, where ε > 0, and initial number of errors W ≤ ⌊α0Wγ0 ⌋, then algorithm A corrects all errors with number of operation O (n log n).

IV. MAIN RESULT

In the previous section we described algorithm A and formulated such requirements to the LDPC code and error pattern weight that it is guaranteed that algorithm A corrects this error patten with complexity O (n log n). The following questions arise: does such LDPC code exist in ensemble EH0 (n0, ℓ, b) and what is the weight of correctable error pattern? Main result gives us the positive answer on the first question and the method to find the fraction of the correctable error pattern weight for the second question.

The proof of this theorem mostly builds upon the methods developed in [2] and generalized in [11]. The main difference is in our approach. We consider the number of correcting edges ∥EC∥ in corresponding Tanner graph instead of the weight of generalized syndrome S of corresponding LDPC code as it was done in [2] and [11].
The equation (1) for the given γ gives us such value ωγ that for any W < ⌊ωγn⌋ it is guaranteed that ∥EC ∥ > γW ℓ. If fraction γ of correcting edges is selected so that algorithm A guarantees that it finds the variable node that is incident to more than ℓ/2 correcting edges, i.e. γ > γ0, then it is guaranteed that algorithm A will find the symbol to invert for any error pattern with weight W < ⌊ωγn⌋. But as it was mentioned above algorithm A is able to introduce new errors in tentative sequence while decoding, and at some moment the error pattern weight may exceed ⌊ωγn⌋. In this case we are not able to guarantee that the algorithm A finds symbol to invert. The equation (2) for the given ωγ gives us the fraction α0 of error pattern weight ⌊ωγn⌋, that for initial error pattern weight W0 < ⌊α0ωγn⌋ it is guaranteed that error pattern weight of tentative sequence never exceeds ⌊ωγn⌋ while decoding.
We will show how to select the fraction of correcting edges γ0 so that algorithm A guarantees to find such symbol that being inverted reduces the generalized syndrome weight for Gallager’s LDPC and H-LDPC codes in the following sections.

VI. CONCLUSION

Considering the correcting edges of Tanner graph corre-

sponding to binary LDPC code we have improved lower bound

on fraction of guaranteed corrected errors while decoding with

algorithm A. This is justified by the fact that this approach

let us formulate more accurately the condition when such

symbol for algorithm A exists that being inverted reduces the

generalized syndrome weight. Proposed approach also let us

consider more combination of errors many of which satisfy the

imposed conditions. These advantages of described approach

in general lead to the significant increase of lower bound on

fraction of guaranteed corrected errors while decoding the

binary LDPC code using iterative low-complex hard-decision

algorithm A.
