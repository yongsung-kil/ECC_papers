LDPC Decoder with Modified LLR for Bit Patterned 
Media with Write Errors 
Warang rat Wiriya, Watid Phakphisut, and Pornchai Supnithi 
Faculty of Eng ineering, King Mongkut's Institute of Technolog y Ladkrabang, Bangkok 10520, Thailand 
s3611420 @kmitl.ac.th, s4610134@kmit1.ac.th, ksupornc@kmitl.ac.th 
Abstract- Write-in error is a crucial issue that needs to be 
tackled in bit patterned media (BPM) recording. Previous works 
apply the low-density parity-check (LDPC) codes to improve the 
system performance and the write margin. However, LDPC 
codes are not designed for the write error channel (WE C). 
Therefore, the log-likelihood ratio (LLR) is modified at the 
LDPC decoder by incorporating the information write error 
probability (P) for the performance improvement. In practice, 
such probability is not precisely known, therefore, in this work, 
we investigate the effects of the mismatch of expected write error 
probability and the actual one in the BPM system. The results 
show that the performance of LDPC decoding is worse when p 
mismatches with the write error probability of the WEe. 
Keywords-Bit-patterned media; write-in error; low-density 
parity-check codes; modified log-likelihood ratio; 
I. 
INTRODUCTION 
Bit patterned media (BPM) recording is a promising option 
for future high density perpendicular recording. However, a 
crucial issue in the writing process known as written-in errors 
needs to be tackled. Due to the head field gradient, the 
inherent fluctuation such as the media switching field 
distribution (SFD), the demagnetization field from adjacent 
dots and the dot position deviation [ 1], the previously written 
bit and the target bit may be erroneous causing substitution 
errors that bring about the worse performance and the 
constricted write margin. Therefore, low-density parity-check 
(LDPC) codes that is currently the popular error-correction 
code for the magnetic recording systems has been applied, for 
example, the previous works use the LDPC codes to correct 
the erroneous bits and improve the write margin in the write 
error channel with substitution errors, (WEC) [1, 2]. However, 
the LDPC codes are not designed for WEC, so the LDPC 
codes cannot maintain an acceptable system performance. 
Therefore, in [3], the performance improvement of LDPC 
codes by using the modified the LLR on the binary systematic 
channel (BSC) cascaded with the additive white Gaussian 
noise (A WGN) channel, the cascaded BSC-A WGN channel, 
has been proposed. In this work, we investigate the use of the 
modified LLR to improve the performance of high-rate LDPC 
codes and the write margin on the cascaded WEC-A WGN 
channel. In addition, the effects of the modified LLR density 
to the performance of LDPC decoding is explored in details. 
For example, the performance of LDPC decoder when using 
the modified LLR density at various iterations and the bit error 
rate performance when p mismatches PWEC. Herein, we

THE CHANNEL MODEL 
A. 
Write error channel 
The recording concepts of BPM are that the head field must 
be large enough to saturate the target dot that is being written 
and, simultaneously, the field must not re-switch polarity of 
the previously written dot, small enough to avoid re­
magnetization. However, in practice, the ideal recording 
cannot always be done successfully due to the effect of the 
head field gradient, the media SFD, the demagnetization field 
from adjacent dots and the dot position deviation casing the 
probability of occurrence of the written-in errors in the 
previously written dot and the target dot [1, 4]. Fig. l(a) and 
(b) show the write error scheme. From the figures, normally, 
the main pole of the writing head is located above the dots and 
its trailing edge is at the center of the target dot (dot#3), 
however, due to the write head offset and the dot position 
deviation causing the error of the write head position, the 
trailing edge of the write head mismatches at the center of

IV. 
SIMULATION RESULTS AND DISCUSSIONS 
In this work, the simulation results are separated into two 
parts. Firstly, the write error rate performance is simulated 
from (1) and (2) in Section II. We assume an areal recording 
density of 3 Tbits/in2 with the bit density of 2300 kBPI and the 
track density of 1300 kBPI. The dot diameter, the dot pitch 
and the track pitch are fixed to 7, 11 and 19.5 nm, 
respectively. We also assume that magnetic field at the target 
track is not oscillated by the magnetization on the neighboring 
tracks because the read head has the side shield, and the cross­
track width of read head equals the track pitch [1]. 
Secondly, for LDPC codes, we use the belief propagation 
(BP) algorithm decoding on the high-rate-code binary random 
LDPC code [6]. This code is constructed using the 
progressive-edge-growth (PEG) algorithm [7, 8]. The Tanner 
graph has the degree of the variable nodes, the column weight 
of 3, while the degrees of the check nodes vary from 26-29. 
The girth of Tanner graphs is 6. The codeword block size is 
4608 and the code rate is 8/9. 
A. 
Write error rate perfomance 
Fig. 4 shows the write error rate performance when the 
write head position shifts to left or right at the time. In this 
paper, the parameters Hmax = 12 kOe, He = 9 kOe, dHldx = 450 
Oe/nm, Oposition = 0.03, /)J{c = 0.05 X He. Hd = 0.01 X He [I]. If 
we focus our attention on the left-hand side of the graph, the 
results show that the write error rates decrease from left to 
right as the write head moves away from the previous dot 
because the previously written bit is less affected from head 
field gradient. On the other hand, if we focus attention on the 
right hand side of the graph, the write error occurs on the dot 
that is being written when the write head moves away from 
target dot, shifting to the right. Therefore, the write error rates 
increase again. From this simulation, the PWEC that will be 
used in the next simulation, is obtained from the write error 
rate when the write head shifts to each position. For example, 
the write head position at -2 nm is obtained the write error rate 
10-2• In addition, the arrow shows a range where the write 
head can move and keep the writing error rate performance is 
10-5, write margin is about 1.8 nm. 
B. 
The performance improvement of LDPC decoder using 
the modified LLR 
In this section, we shall explore the performance of decoder 
when the decoder using the modified LLR for the channel that 
has the write error event occurrence. In this simulation, the 
user data sequences are encoded and decoded with LDPC 
codes that has been explained. The codeword is recorded into 
the channel with PWEC of 0.001 and 0.005. Fig. 5 shows the 
performance of decoder for different values of PWEC. Here, the 
bit error rate (BER) versus signal-to-noise ratio (SNR) in 
terms of EJNo = V(2ifR), where Eb is the bit energy, No is a 
one-sided power spectral density of A WGN, if is the variance 
of A WGN and R is the code rate. For each SNR, the 
simulation is run until about 1000 erroneous bits are obtained. 
And for all curves, the number of decoding iterations is 20. 
From Fig. 5, the star line and the triangle line show the 
performance of the decoder with using the conventional LLR,

uses the modified LLR density yields a better performance.

exhibits the same performance levels as the LDPC decoder 
using the conventional LLR. At the BER of 10-5, the SNR gain 
of the LDPC using modified LLR over the LDPC using 
conventional LLR is about 1.65 dB. 
To explore the effect of the modified LLR for LDPC 
decoding, we run the simulation up to 50 iterations. Fig. 6 
shows the performance of the decoder using the modified LLR 
compared with the conventional LLR at various iterations with 
SNR and PWEC fixed at 5 dB and 0.001, respectively. The 
decoder using the modified LLR density is superior to the 
conventional LLR at all iterations. For the BER of 3.037 X 10-
4, the modified LLR requires 4 iterations as opposed to 15 
iterations for the conventional LLR. 
Fig. 7 shows the BER performance when the write error 
probability P mismatches PWEC of the channel. We fix PWEC at 
0.002, 0.0015 and 0.001, represented by cross, circle and 
triangle, then vary P from 0 to 0.01. The results show that the 
BER performance of LDPC decoding is worse if P values do 
not correspond with PWEC, yielding the better performance 
when P values match PWEC. However, the P values can vary by 
about 20% to get the near performance levels at fixed SNR of 
4.4 dB.
