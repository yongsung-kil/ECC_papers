A Technique to Improve the Performance of Fixed-Point TDMP Decoding of QC-LDPC Codes in the Presence of SNR Estimation Error
JaWone A. Kennedy and Daniel L. Noneaker (corresponding author)
Holcombe Department of Electrical and Computer Engineering 305 Fluor Daniel Engineering Innovation Building Clemson University, Clemson, SC 29634

ABSTRACT
Most high-throughput, fixed-point processors offer at least two options for arithmetic operations: 8-bit arithmetic, and 16-bit arithmetic. The lower resolution provides higher computational throughput at the cost of poorer performance in many applications. We investigate the effect of the resolution of saturating, fixed-point arithmetic on the performance of the turbo-decoding message-passing algorithm with quasi-cyclic low-density parity-check codes. We consider limits on the magnitude of extrinsic updates as a means to mitigate the effect of posterior-value saturation on the decoder’s performance. We show that a fixed limit on updates only partially overcomes the greater effect of saturation in 8-bit operations, whereas a limit that depends on the degree of the variable node results in performance almost as good as what is possible with 16-bit operations.
INTRODUCTION
Quasi-cyclic (QC) low-density parity-check (LDPC) codes [1] are used for forward error correction in many wireless communication systems, including Wi-Max (802.16) [2] and DVB-S2 [3]. One option for achieving the high decoder throughput required by these systems is the use of a specialized digital signal processor (DSP) that is optimized for a high level of single-instruction, multiple-data (SIMD) parallelism (exemplified by the stream processor [4]) . Such processors provide the high-level-language programmability of general-purpose DSPs but with much better performance for algorithms suited to SIMD parallelism.
A SIMD-optimized DSP designed for embedded systems provides an efficient trade-off between SIMD parallelism and power consumption by supporting only fixedpoint arithmetic. The fixed-point processor may provide two or more data-resolution modes operating on packeddata operands so that lower-resolution operation results in greater computational throughput. (For example, the peak computational throughput with 8-bit operations is approximately twice the peak throughput with 16-bit operations

in most SIMD processors.) The performance of a decoder for LDPC codes can be degraded due to low resolution in the processor’s fixed-point saturating arithmetic, however [5]. The choice of the fixed-point resolution thus represents a trade-off between decoder performance and computation time per iteration in decoding an LDPC code word for a given decoding algorithm [5]. It motivates the search for a decoding algorithm that permits the processor to operate at the lowest possible resolution without sacrificing significant decoder performance as measured by the probability of code-word error and the average number of iterations per code word.
The turbo-decoding message-passing (TDMP) algorithm [6] is a (merged schedule) row-wise, layered-decoding belief-propagation algorithm [7] that is readily adapted to differing levels of partially parallel computation based on the level of data parallelism available in the processor. It achieves performance comparable to the sum-product algorithm (SPA) with about one-half the average number of iterations. The TDMP algorithm also requires less memory than the SPA [6], and it permits simpler data management than the SPA in processor architectures organized for high levels of SIMD parallelism [5]. The performance of the TDMP algorithm is more sensitive than the SPA to the resolution of fixed-point saturating arithmetic, however [5]. It is shown in [5] that the imposition of a predetermined limit on the maximum magnitude of any extrinsic update improves the performance of the offset-min-sum TDMP algorithm for LDPC codes if low-resolution, fixed-point processing is used.
Offset-min-sum updates [8] do not require an estimate of the signal-to-noise ratio (SNR) of the received signal. In contrast, the max-quartet update of [6] can be used to achieve better performance if the TDMP algorithm is provided with an accurate estimate of the SNR. In this paper, we investigate the use of limits on the magnitude of an extrinsic update for the max-quartet-update TDMP algorithm employing fixed-point saturating arithmetic. We show that properly chosen limits on update magnitudes result in link performance with 8-bit arithmetic that is nearly

THE CONSTRAINED-UPDATE TDMP ALGORITHM
In this section we describe the TDMP algorithm and a novel variant of the algorithm. We consider the use of the algorithm in a system using antipodal binary modulation with transmission over an additive white Gaussian noise channel with double-sided power spectral density N0/2. Coherent demodulation with perfect synchronization is assumed. Each iteration of the TDMP algorithm is applied to an M x N parity-check matrix H of an LDPC code, with updates of posterior values for variable nodes performed in a block-sequential manner. Concurrent updates use message passing based on the parity-check equations corresponding to blocks of rows of H [9]. Extrinsic messages that are generated from decoding earlier row blocks are used as input prior messages for updates of the posterior values for variable nodes participating in later row blocks. The rowwise update schedule thus differs from the SPA in which all check-nodes updates occur before the updates of the posterior values are performed for any variable nodes in an iteration. We consider posterior updates in the TDMP algorithm using the max-quartet function [6], which utilizes an estimate of Es/N0 in the soft-input soft-output function described below. The max-quartet TDMP algorithm results in performance that is more than 0.1 dB better than the performance of the offset-min-sum TDMP algorithm at lowto-moderate signal-to-noise ratios.

DECODER PERFORMANCE WITH LOW-RESOLUTION FIXED-POINT ARITHMETIC
In this section we compare the performance of the standard max-quartet TDMP algorithm with the fixed-constraint and variable-constraint modifications of the decoding algorithm using signed, fixed-point saturating arithmetic for both 8-bit and 16-bit arithmetic operations. With the standard max-quartet TDMP algorithm using high-resolution arithmetic, the maximum number of iterations per code word must be at least 20 in order to achieve most of the potential performance with most QC-LDPC codes of interest. Thus we consider the performance of each algorithm using a maximum of 20 iterations per code word. Uniform quantization with a quantization interval, q, is used to convert real-valued priors into a fixed-point representation matching the resolution of the decoder operations; the quantized priors are used in the decoding algorithm. For the results addressed in this section, the decoder is assumed to have a perfect estimate of the signal-to-noise ratio of the received signal. The value of CQ in the max-quartet function of equation (1) must be an integer multiple of the quantization interval if fixed-point arithmetic is used in the decoder.
The performance of the standard max-quartet TDMP decoding algorithm is illustrated in Fig. 1, which shows the probability of block (code word) error as a function of the signal-to-noise-ratio, Eb/N0, for a system using the rate1/2, block-length-1536 WiMax code that has a parity-check matrix constructed from permutation sub-matrices of size 64 [2]. The performance is shown for the algorithm using (high resolution) floating-point arithmetic; signed, 16-bit fixedpoint arithmetic; and signed, 8-bit fixed-point arithmetic. Performance is shown for 16-bit decoding with two combinations of the constant CQ in the max-quartet function and the quantization interval q: CQ = 0.625 with q = 0.125, and CQ = 0.5 with q = 0.25. (The optimal performance with 16-bit arithmetic is achieved using CQ = 0.625 with q = 0.125.) The performance of 8-bit decoding is shown only for CQ = 0.5 with q = 0.25, which result in the best performance with 8-bit decoding under some conditions.
The decoder performance achievable with 16-bit fixedpoint arithmetic using CQ = 0.625 and q = 0.125 is close to that obtained with high-resolution floating-point arithmetic using CQ = 0.625; in fact, the performance with 16-bit arithmetic is slightly better as seen in Fig. 1. (The floatingpoint algorithm is not the maximum-likelihood decoder; hence, it is possible for mild quantization to improve its

performance slightly in some instances.) It is also seen that substitution of the sub-optimal decoder parameters CQ = 0.5 and q = 0.25 for 16-bit decoding results in a loss in performance of no more than 0.05 dB for a probability of block error of 10−4 or less. In contrast, the performance of the algorithm with 8-bit fixed-point arithmetic is poor even with the value of CQ and the quantization width optimized for 8-bit decoding, as seen in in Fig. 1. The system achieves a probability of block error of 10−4 at a signal-to-noise ratio of 2.2 dB if 16-bit fixed-point arithmetic is used with a quantization width of 0.25 and CQ = 0.5. If the resolution of the fixed-point arithmetic is reduced to eight bits, however, the probability of block error exhibits an error floor of slightly less than 10−2. If the algorithm is used instead with the rate-1/2 DVB-S2 code of block length 16,200 [3], 16-bit fixed-point arithmetic with CQ = 0.5 and q = 0.25 results in a probability of block error of 10−4 at a signal-to-noise ratio of 1.52 dB, whereas 8-bit arithmetic results in an error floor greater than 10−1. The frequent occurrence of saturation in the 8-bit operations has a catastrophic effect on the performance of the decoder for both codes.
The offset-min-sum TDMP algorithm exhibits a similar degradation in performance due to 8-bit fixed-point operations that is illustrated above for the max-quartet TDMP algorithm. It is shown in [5] that the degradation in performance of the offset-min-sum TDMP algorithm can be overcome by imposing a maximum on the magnitude of any one extrinsic update. That is, the fixed-constraint TDMP algorithm is effective in mitigating the effects of 8-bit arithmetic if the algorithm uses the offset-min-sum update. It results in performance essentially equal to that achieved with 16-bit arithmetic.
The use of a fixed constraint on the extrinsic-update magnitude can also be used to improve the performance with 8-bit fixed-point arithmetic if the TDMP algorithm uses the max-quartet update. Its benefit is limited by the severe tradeoff between arithmetic saturation and loss of sample resolution in SNR-dependent updates such as the max-quartet function, however. This is illustrated in Fig. 2 for the WiMax code and the max-quartet TDMP algorithm using CQ = 0.5 and q = 0.25. An overly severe constraint on the update magnitude inhibits the convergence of the algorithm and results in a high error floor. If the magnitude of any update is limited to 2.5 (i.e., j = 2.5 for all j in step 5 of the algorithm), for example, the probability of block error exhibits an error floor of 0.7 (which is not shown in the figure). A limit of 5.0 results in an error floor of 7 × 10−3, which is only slightly better than the performance of 8-bit decoding with no update constraint. As the limit

The gap in performance between 16-bit max-quartet TDMP decoding and 8-bit max-quartet TDMP decoding with the optimal fixed constraint can be reduced by employing a limit on the magnitude of the extrinsic update of the posterior for a code symbol that depends on the weight of the corresponding column in the parity-check matrix. (Again, 16-bit decoding is optimized if no limits are applied to the updates.) For example, the parity-check matrix used in decoding the WiMax code has column weights of two, three, and six. The curve labeled ”8-bit (variable constraint)” in Fig. 2 shows the 8-bit performance of the variable-constraint TDMP algorithm with limits of 15.0, 10.0, and 5.0 for columns of the lowest, intermediate, and highest weights, respectively. The resulting probability of block error is essentially the same as that resulting from optimal 16-bit decoding. Results for 8-bit decoding with the variable-constraint TDMP algorithm are shown in Fig. 3 for the DVB-S2 algorithm, and once again, the performance is indiscernible from the performance with optimal 16-bit

decoding. The use of variable constraints can thus improve the performance of the optimal fixed constraint by up to 0.68 dB for the WiMax code and up to 0.15 dB for the DVB-S2 code.
The introduction of update constraints has a negligible effect on the execution time per iteration and the memory utilization of the TDMP algorithm. Furthermore for a given code and SNR, the average number of iterations per code word differs negligibly among the 8-bit and 16-bit decoders using no update constraints, the optimal fixed update constraint, and the optimal variable update constraints. Thus for SIMD processors used 8-bit and 16-bit packed-data modes, the optimal variable-constraint 8-bit decoder requires only about half the execution time (or, equivalently, it achieves approximately twice the information throughput) of the optimal 16-bit decoder with a negligible cost in the probability of error.

CONCLUSION
The effect of arithmetic saturation on low-resolution, fixed-point TDMP decoding of a QC-LDPC code can be mitigated by imposing a constraint on the maximum magnitude of any single extrinsic update of a variable node (code symbol) posterior value. If the max-quartet update function is used, the best performance is obtained only if the constraint depends on the degree of the variable node corresponding to the updated posterior value. For two important examples of a QC-LDPC code, optimized variable-constraint, max-quartet TDMP decoding using 8-bit arithmetic results in comparable performance to optimized max-quartet TDMP using 16-bit arithmetic: for a given signal-to-noise ratio, they result in the same probability of code-word error and they require approximately the same average number of iterations per code word. Consequently, the use of optimized variable update constraints permits 8bit decoding to achieve the same link performance as 16-bit decoding while requiring only about one-half the execution time per code word on most fixed-point SIMD processors.
