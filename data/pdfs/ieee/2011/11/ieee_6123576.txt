Abstract—Structured quasi-cyclic low-density parity-check (QCLDPC) code is a part of many emerging wireless communication standards, such as WiMAX, WiFi and WPAN. This paper presents a high parallel decoder architecture for the QC-LDPC codes and the corresponding decoder ASIC for WiMAX system. Through utilizing the proposed fully parallel layered scheduling architecture, the decoder chip saves 22.2% memory bits and takes 24~48 clock cycles per iteration for different code rates. It occupies 3.36 mm2 in SMIC 65nm CMOS, and realizes 1Gbps (1056Mbps) throughput at 1.2V, 110MHz and 10 iterations with the power 115mW and power efficiency 10.9pJ/bit/iteration. The energy/bit/iteration reduces 63.6% in normalized comparison with the state-of-art publication.
I. INTRODUCTION
Structured QC-LDPC codes have become the ECC (error correcting code) part of many emerging wireless communication standards, among which WiMAX is one of the approved standards for the fourth-generation mobile communication system (4G). To be used in handheld terminals, the QC-LDPC decoder for WiMAX system needs to deliver high throughput with low power dissipation.
With the purpose of best tradeoff between the hardware complexity and decoding performance, the normalized minsum algorithm [1] or offset min-sum algorithm [2] with the layered scheduling (also called TDMP) [3] has shown its significance for QC-LDPC codes. In the normalized min-sum algorithm, the complicated computation of Gallager function is replaced with simple minimum value finding operation at the cost of decoding performance, which is recovered by the normalized factor. The layered scheduling achieves about two times faster convergence speed than the conventional flooding scheduling (also called TPMP).
Generally the iterative decoding process of normalized min-sum layered algorithm is carrying out layer by layer. Within each layer, different designs give different scheduling architectures. The architecture in [4] processes the nonzero sub-blocks block by block. As a result, the clock cycle number of one iteration is approximately equal to the nonzero submatrix number of parity check matrix (PCM), which is 76 to 88 according to WiMAX standard. Based on this approach, the design in [5] uses two sets of processing units to improve the parallelism and processes two nonzero sub-blocks in one

clock cycle. However, it does not bring double parallelism because of the data conflicting problem. By utilizing the submatrix reordering and complex bypass controlling scheme, it partially solved this problem, which reduces the number of clock cycles for each iteration to 48-54.
Compared with the previous designs, this work changes the block by block scheduling of layered algorithm, which achieves higher parallel architecture. The key schemes of this architecture contain three aspects:
1) It is full parallel in each layer, which is named as full parallel layered decoding. All messages within one layer are calculated and updated simultaneously. In order to avoid the interconnection problem in the placement and routing, the arithmetic units and permutation network are designed as 1-bit form. With 6-bit quantization, the number of clock cycles for each layer is 6, and each iteration needs 24 to 72 clock cycles for different code rates; 2) The parallelism is improved furthermore by dedicated PCM reordering and the two-layer concurrent processing for low code rates. The clock cycles are finally reduced to 24~48; 3) All the messages are stored in registers, not plenty of small and inefficient memory banks. The power increase from using registers is offset by memory bits saving and clock gating. This architecture eliminates the FIFO which is necessary in the previous designs. The variable node (VN) messages and the a-posterior probability (APP) messages share the same storing cells. Thus, it saves at least 22.2% memory bits than the previous works. The clock gating could reduce 27% power due to large amount of registers.
The rest of this paper is organized as follows. Section II introduces the architecture of the proposed decoder. Section III presents the critical modules in this design, including the bit layered decoding engine, matrix reordering module and permutation network. Section IV describes the implementation results and comparisons with previous works. Finally, section V draws the conclusion of this paper.

The 96 processing units (PUs) are the kernel of the decoder, which is used to process the variable node and check node operations for one layer by updating messages from LSB to MSB serially, including the subtractor for Eq. (1), the comparator for Eq. (2) and the adder for Eq. (3). Permutation network (PN) contains 42 one-bit 96×96 cyclic shifters, in which PN A is used to shift 21 blocks of messages according to the base matrix (no shift needed for the rightmost 3 block columns) and PN B is designed to shift overflow and hard decision bit. For the two-layer concurrent processing, the cross-bar switch array (CBSA) in PN is implemented to divide 24 blocks of messages into 2 groups. Since APP and VN messages share the same storage registers in the PUs, a LUT is used to indicate the types of messages.

In layered decoding algorithm, the check node (CN) messages are simplified to the minimum value of last iteration. Thus, the CN messages register array needs only store the magnitude, position and sign bit of the value. Six two-port register files (RFs) which can store 10 code frames form the input buffer for chip testing.
B. Quantization In the proposed architecture, the quantization is a dominate
factor which determines not only the memory size but also the number of clock cycles for each iteration. As Figure 2 shows, the bit error rate (BER) curve of layered scheduling converges much faster than the flooding scheduling under the floating point computation. However, the 5-bit quantization of layered decoding brings about 0.8dB performance loss at the 10-5 BER, which is even worse than the flooding. Conversely, the 6-bit and 7-bit quantization leads to around 0.2dB and 0.1dB loss respectively. Thus, the 6-bit quantization is adopted for best trade-off between decoding performance and hardware cost.

C. Early Termination In the decoding process, it takes different iterations to
correct the code words. The early termination (ET) scheme is used to stop the decoding process when the code word has already been corrected. The natural approach of ET is calculating all the parity check equations. In this design, the ET control unit terminates the decoding based on the stopping criterion which was proposed in our previous work [6]. The basic idea of this criterion is errors in the information part of code word are corrected faster than the errors in the parity part. When all the information bits are correct, the decoding process can be stopped. Figure 2 shows the average iteration number (AIN) of the proposed ET scheme. Compared with the all parity check correcting situation, it reduces 1 AIN on average.

Based on the 6-bit quantization, each layer processing occupies 6 clock cycles. As shown in the timing flow chart, in the first 5 clock cycles of current layer, least significant 5 bits of messages are updated and hard decisions are transmitted to ET unit for analyzing via PN B. At the 6th clock cycle, PN B is used to transmit the APP bit updater generated overflow signals. Based on them and the overflow signals generated by APP/CN REG bit updaters, the VN messages are corrected. Then the VN messages are converted from two’s complement to sign-magnitude form by the 2CSM unit and fed into the min-sorter.
The min-sorter employs the simplest tree structure to find the first and second minimum values among VN messages with 2 clock cycles. After the previous clock cycle for the first minimum value and its position, it finds the second minimum value by setting the first minimum value as the maximum value (111112). Compared with the one-clock design, this structure could reduce the critical path and save about 150k gate count. Moreover, this design would not increase the clock cycles per layer. The first minimum value is found at the last clock cycle of current layer, and the second minimum value finding is arranged to the next layer.

V. CONCLUSIONS
This paper introduces the proposed fully parallel layered scheduling architecture and presents the QC-LDPC decoder chip for WiMAX standard, which achieves higher parallelism than previous designs. By utilizing the two-layer concurrent processing scheme with minor hardware cost, it takes 24-48 clock cycles per iteration. In order to avoid the interconnection problem, the permutation network and arithmetic units are designed as 1-bit form. The memory bits are reduced to a great extent. High parallelism and reasonable hardware cost makes this design realizes low power, which presents significant improvement of power efficiency than the state-of-art work.
