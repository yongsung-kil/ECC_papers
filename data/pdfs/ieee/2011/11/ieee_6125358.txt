2011 8th International Symposium on Wireless Communication Systems, Aachen

Construction of SeIRA QC-LDPC Codes with Low
Error Floor
Wenyan Zhang, Minjian Zhao, Jie Zhong, Jie Wu, Xuanxuan Lv Institute of Information and Communication Engineering Zhejiang University, Hangzhou, China zhangwenyan@zju.edu.cn

Abstract—We investigate the construction method of structured extended irregular repeated-accumulate quasi-cyclic low-density parity-check (SeIRA QC-LDPC) codes with short to moderate length. A new metric is proposed that jointly considers both circle length and circle connectivity, which aims to improve the error correcting performance in the error floor region. Meanwhileˈ iterative searching method is proposed that increase the possibility of finding out good solution based on the new metric. At last, we apply our construct method to design codes with different length by expanding them from the same mother matrix. Simulation results show that FER of the codes constructed by this method outperform the corresponding codes in the 802.16 standard expanded from the same mother matrix by 0.15~0.4dB at FER around 105 .
Index Terms—constructing, iterative searching, girth, ACE, SeIRA

I.

INTRODUCTION

Low-density parity-check codes were first proposed by Gallager in 1962 [1] and rediscovered by MacKay in 1996 [2]. They have shown performance close to Shannon capacity when decoding with sum-product algorithm.
Random constructed LDPC codes have shown good performance. However, they need a large amount of resource to save the information of the parity-check matrix and don’t have effective way for encoding. SeIRA QC-LDPC [3] codes are a class of fast encoding codes with a good structure for the hardware implement.
As girth (shortest circle length) plays an important role in the performance of short to moderate length LDPC codes, many works have been done for it. M.Fossorier investigated circle condition for the QC-LDPC [4]. S.Myung studied unavoidable circles in the matrix and gave a method to construct the matrix [5]. Their work aims to remove the circles with short length as many as possible. However, it is impossible to remove the entire short circles in the matrix for the short to moderate length codes. Furthermore, short circles are not equally harmful to the performance. In addition, the circle connectivity also has a great effect on the performance, especially in the error floor region. T.Tian took a new metric named extrinsic message degree (EMD) into account which reduced the error floor significantly [6]. W.Sulek indicated a new strategy that takes both the length and the connectivity of circles into account [7].
In this paper, a new metric is proposed that jointly considers the length and the connectivity of circles connected to the

variable nodes. The approximate circle extrinsic message degree (ACE) is used to depict the circle connectivity for simplicity. The new metric indicates the decoding failure probability of a given circle with the same ACE and length. A construct algorithm based on this new metric is proposed to design SeIRA QC-LDPC codes. The construct method assumes that the mother matrix is given, and then it searches the shift numbers for the entries in the shift matrix. Iterative searching method is applied to increase the possibility of finding out a good solution.
This paper is organized as follows. In Section II, a brief introduction of SeIRA QC-LDPC codes is given. In Section III, the circle properties of QC-LDPC codes are presented, including the circle condition, the circle connectivity and the new metric proposed for code construction. Section IV gives out the construction method in details. Construction results and simulation compared with the codes in 802.16 standard are shown in Section V. Finally, the conclusion is given in Section VI.

III.

CIRCLE PROPERTY

For LDPC codes with short to moderate length, there is a great chance for the existence of small circles in the matrix, which is harmful to the performance. In addition, the connectivity of circles also plays an important role in the performance, especially in the error floor region. In this section firstly, circle condition of QC-LDPC codes is introduced. Secondly, stopping set and trapping set are also introduced to illustrate the connectivity of the circles. Thirdly, a new metric based on the trapping set is proposed which aims to improve the performance in the error floor region. At the end of this section, the circle statistics method is introduced.

S.Myung proved that if there are f overlaps between a length- 2l circle and a length- 2k circle in the mother matrix, there exist the circles of length 2(2l  2k  f ) in the expanded matrix no matter what values are selected for the entries in the shift matrix [5]. Since this class of circles can not be avoided when the mother matrix is given, we do not take it into account in this paper.

B. Circle Connectivity
The stopping set [7] denoted as Sv is a set of v variable
nodes that form a subgraph without check nodes of degree one in the neighbourhood. It has been known as the set inducing to the failure of decoding in binary erasure channel (BEC). The trapping set [7] denoted as Tvc is a set of v variable nodes that forms a subgraph with c odd degree check nodes in the neighbourhood. Small stopping set or trapping set induce to the high error floor which should be avoided. Fig.1 shows the example of stopping set and trapping set.
The extrinsic check node of a variable-node set is a check node that is connected to this set with odd degree. The EMD of a variable-node set is the number of extrinsic check nodes of this set. According to the definition, there is no extrinsic check node connected to the stopping set. Consequently, the EMD of a stopping set is zero. The variable-node set with high EMD leads to the large stopping set because more nodes need to add to form the set. Hence small circle with low EMD should be avoided during the construction of the matrix. Because it is difficult to find accurate EMD of the graph, the ACE [6] is used to evaluate the circle connectivity. ACE is defined with respect to variable nodes in a given circle. D. Circle Statistics Method In order to find the circles in the mother matrix, we span the
graph of the mother matrix into a tree. In the PEG algorithm a minimum spanning tree is built up to find the shortest circle in the graph. However, it is insufficient to find all circles in the graph. Denote the nodes in layer l by Ql . Ql1 is the set of nodes connected to the entries in Ql . Q0 is the first variable node connected to the graph. If there are the same nodes in the set Ql and their parents in the tree are not the same, a circle of length- 2l exists in the mother matrix.
The variable node with low degree is less reliable than the node with high degree. And the short circles composed of low degree nodes contribute to a high error floor because of small trapping set. For a small mother matrix, more and more circles are formed with the variable node connecting to the graph. Therefore the low degree variable nodes should give a priority during the construction.
As shown previously, by expanding the graph to the layer L t we can get entire circles smaller than 2Lt in the mother matrix. However, the size of expanding tree will grow by

geometric ratio with the increase of the number of layers. The size of tree becomes unacceptable when L t is large. In consequence, appropriate layer should be given to trade off the completeness of the circle statistics and the size of the tree. IV.

CONSTRUCTION

This paper only investigates the case when a mother matrix is given, and it aims to get an expanded matrix with good performance in the error floor region. The degree distribution can be obtained by the density evolution [9], and the mother matrix can be obtained by the PEG algorithm [8]. When the mother matrix is obtained, our algorithm processes by progressively adding the variable node to the shift matrix, and searching the shift numbers for the entries connected to the current variable node. In the algorithm we consider the shift matrix as a solution of a set of equations built up according to the circle condition. For building up the equations, we only count the circles connected to the current variable node added into the contemporary graph. Firstly we find circles with length smaller than 2lt in the mother matrix and secondly build up corresponding equations for them. At last, we search for the solution according to the new metric proposed in Section III.

V.

SIMULATIONS

In this section, several codes are constructed based on the algorithms described in the former section. The algorithm can be applied to design the code with any code length and rate. In order to compare the performance with the codes in the 802.16 standard, we use the same mother matrix that is applied in the standard. We design the codes with rate 1/2 and length from 576 to 2304 for example. The size of the mother matrix is 12 u 24 and the size of the cyclic identity matrix s is calculated according to the length of the code. The modulation is BPSK, and sum-product algorithm is applied to simulate the performance of the matrix designed on the AWGN channel. The maximum iteration times is set to 50, and all the code words are randomly generated for the simulation. For
certain Eb / N0 , the simulation is terminated when 100 block
errors occur.

VI.

CONCLUSION

In this paper, we considered the construction of the SeIRA QC–LDPC codes. A new metric was proposed to jointly consider the length and the connectivity of the circle in the matrix. Iterative search method based on the new metric was applied to the construction of several codes. The optimized construct codes may not be the best codes in the BER sense, but it offers a better FER performance. Furthermore we investigated the value of the parameter chosen in the metric. It was shown that the performance varied a little with different values. Simulation results showed that the proposed schemes outperform the 802.16 standard codes by 0.15~0.4dB.
