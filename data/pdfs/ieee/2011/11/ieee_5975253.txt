Delayed Stochastic Decoding of LDPC Codes
Ali Naderi, Shie Mannor, Senior Member, IEEE, Mohamad Sawan, Fellow, IEEE, and Warren J. Gross, Senior Member, IEEE

Abstract—A new stochastic decoding algorithm, called Delayed

Stochastic (DS) decoding, is introduced to implement low-density-

parity-check (LDPC) decoders. The delayed stochastic decoding

uses an alternative method to track probability values, which re-

sults in reduction of hardware complexity and memory require-

ment of the stochastic decoders. It is therefore suitable for fully-

parallel implementation of long LDPC codes with applications in

optical communications. Two decoders are implemented using the

DS algorithm for medium (2048, 1723) and long (32768, 26624)

LDPC codes. The decoders occupy 3.93-mm and 56.5-mm sil-

icon area using 90-nm CMOS technology and provide maximum

core throughputs of 172.4 and 477.7 Gb/s at

5.5 and 4.8 dB,

respectively.

Index Terms—Delayed stochastic decoding, iterative decoding, low-density parity-check code, stochastic decoding.

I. INTRODUCTION
T HE current growth of IP-based traffic in broadband access networks, provider networks, and optical transport networks requires doubling the Internet bandwidth every 18 months [1]. Such applications demand a throughput as high as 100 Gb/s. Forward error correction (FEC) strategies are essential in such high speed communications to enable the receivers to fix noisy packets. Many implementations for the required FEC decoders have been investigated to produce high throughput solutions [2]–[5]. Fully parallel implementations of a FEC decoder improves throughput, but increases hardware complexity as well. On the other hand, FEC decoding using long codes is required to improve coding performance, which exacerbates the hardware complexity. Therefore, it is of great interest to design a hardware-efficient algorithm for fully-parallel implementation of long codes to achieve higher throughput and coding gain.
Low-density-parity-check codes, first introduced by Gallagher in 1962, offer near Shannon limit performance for many communication channels under message passing algorithms [6].

Recently, it has been demonstrated that stochastic decoding algorithms can be used for implementing hardware-efficient LDPC decoders [7]–[10]. Stochastic decoders work in the probability domain and the messages are represented by means of Bernoulli sequences. The complex mathematical operations in the probability domain can be simplified to bitwise operations in the stochastic decoders, alleviating hardware requirements. Hence, stochastic decoding offers an affordable solution for fully parallel implementation of medium and long codes required in many standards such as IEEE 802.3an (10GBASE-T) [11], the IEEE 802.3ba [12], the IEEE 802.16 (WiMAX) [13], the IEEE 802.11 (WiFi) [14], and the ITU G.975.1 [15].
Iterative decoding using stochastic representation of information for LDPC codes was introduced in [16]; however, it was functional only for short-length codes. To scale the algorithm to longer-length codes, edge memory (EM), tracking-forecast memory (TFM), and majority-based tracking forecast memory (MTFM) modules were added [8]–[10]. These modules work as filters that extract information of the bit-streams in processing nodes, and help in the convergence of the decoder. Near optimum decoding performance has been reported for these improved stochastic decoders. This paper proposes an alternative algorithm, that trades off decoding performance for further reduction in hardware complexity, and is suitable for implementation of long codes. The algorithm, called Delayed Stochastic Decoding, is based on the states of the processing nodes and intelligently performing the parity check operations. In this paper we describe the delayed stochastic algorithm and then show implementation results of high throughput (over 100 Gb/s) fully-parallel decoders for medium (2048, 1723) and long (32768, 26624) LDPC codes.
The rest of this paper is organized as follows. Dynamics of stochastic decoding are discussed in Section II. The architecture of the proposed delayed stochastic decoding algorithm is presented in Section III, and performance results are given in Section IV. The hardware complexity and comparison with existing work are presented in Section V, and some conclusions are drawn in Section VI.

A. Structure of Check Nodes

When a check node is in the regular state, all of its inputs are

regenerative bits. The outputs of the check node are therefore

calculated based on (2). In a delayed state, one or more inputs

of a check node are not regenerative bits. In this case, the check

node cannot decide the incoming bits, and if a variable node

has reached a regenerative bit, that bit could be correct. There-

fore, a delayed check node returns the received regenerative bits

without any change to the variable nodes reporting agreement

states. Those variable nodes reporting disagreement states are

replied to with a random bit from the check node. Recall from Fig. 6. Implementation of (a) a degree-2 subnode, (b) a degree-3 subnode, and (1), when one of the inputs of a variable node is random with a (c) a degree-6 subnode.

D. Decoding Termination Criteria
The DS decoder terminates if the number of iterations3 reaches a maximum, or if a codeword is found. The decoder employs the syndrome checking method to detect convergence to a codeword. Syndrome checking is performed at every iteration by XORing the output bits of the variable nodes at each check node. If all syndrome values are zero, the codeword is correct and the decoding terminates.

IV. DECODING PERFORMANCE

To verify the functionality and the decoding performance of

the DS decoder, we present simulation results for a (2048,1723)

Reed–Solomon based LDPC code, which has been selected for

the IEEE 802.3an standard for 10 Gigabit Ethernet [11]. We also

use a (32768, 26803) LDPC code, which has been generated

with a similar algorithm [24]. Both codes are regular with vari-

able nodes and check nodes of degree 6 and 32, respectively.

The bit-error-rate (BER) performance of the DS decoder is com-

pared with a few recent decoders in Fig. 9. The DS decoder

uses channel inputs with 5-bit quantization and has a maximum

latency of 600 clocks. The decoding performance of the DS

decoder outperforms the modified min-sum algorithm (MSA) in [25] and [26] and improved modified weighted bit-flipping (IMWBF) in [27] by 0.35 and 1 dB, respectively. The DS de-

VI. CONCLUSION

REFERENCES

In this paper, we discussed the dynamics of stochastic decoding, which are based on a filter in the variable nodes. The delayed stochastic decoding algorithm was presented for the implementation of LDPC decoders, which is based on the states of the nodes and delays in the processing of parity check nodes. By employing the nodes’ states, it simplifies variable nodes and reduces memory requirements for stochastic decoders. The implemented decoders present high throughput and area-efficiency, which make it suitable for high speed communications.
