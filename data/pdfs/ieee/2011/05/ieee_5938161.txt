Memory Efficient Layered Decoder Design with Early Termination for LDPC Codes

Jiangpeng Li, Guanghui He, Hexi Hou, Zhejun Zhang, Jun Ma
School of Microelectronics Shanghai Jiao Tong University
Shanghai 200240, China Email: lijiangpeng@ sjtu.edu.cn

Abstract—Layered structure is widely used in the design of LowDensity Parity-Check (LDPC) code decoders due to its fast convergence speed. However, correct checking process is difficult to implement in layered decoder, which results in unnecessary iterations. In this paper, an early termination strategy is presented for layered LDPC decoder to avoid redundant number of iterations. This approach makes use of the comparison between current log-likelyhood ratios (LLRs) and updated LLRs of all variable nodes to determine termination criteria of iterations. Furthermore, a non-uniform quantization scheme and an extrinsic messages memory optimization scheme are developed for memory savings. Based on these proposed methods, an LDPC decoder for the Chinese digital mobile TV applications is implemented using a SMIC 130nm CMOS process. The decoder consumes only 171 Kbits memory while achieving 267Mbps for code rate 1/2, and 401Mbps for code rate 3/4.
Index Terms － Early termination, memory efficient, layered LDPC decoder
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were proposed by Gallager in 1962 [1] and later rediscovered by MacKay in 1996 [2]. They have attracted a lot of interest due to their excellent error control capacities. Recently, LDPC codes have been adopted in many wireless communication standards, such as IEEE802.16e, IEEE 802.11n, and DVB-S2 etc.
The sum-product algorithm and the min-sum algorithm are two well-known iterative decoding algorithms for LDPC codes [3]. Many works focused on the hardware implementtations of the LDPC decoder and various kinds of architecttures (e.g. fully parallel, partially parallel, and serial) have been proposed. However, the convergence speed of these methods is slow, which is troublesome in high throughput applications. Recently, layered decoder [4] has been proposed to address this problem. One iteration in layered decoder is divided into several sub-iterations to accelerate convergence speed. The throughput is increased since less number of iterations is needed at the same bit error rate (BER).
However, correct checking process is difficult to imple-

ment in layered decoder. The iterative process will not stop until the maximum number of iterations is reached, which causes unnecessary calculations that lead to the decreased decoding throughput and the increased power consumption. Therefore, an efficient early termination strategy for layered decoder is necessary. According to instantaneous correct checking, Blad et al. [5] and Ueng et al. [6] proposed some strategies about stopping decoding some variable nodes or subiterations, but the error floor phenomena is always observed [6], which decreases LDPC decoder performance significantly.
In this paper, an early termination strategy is proposed for a memory efficient partially parallel LDPC layered decoder. The strategy avoids unnecessary iterations by introducing low computation complexity. Moreover, the method eliminates the error floor which appeared in previous reported methods [5] [6]. In addition, a novel non-uniform quantization scheme and a structure for extrinsic (EX) messages memory are presented to save memory. Based on the above proposed methods, an LDPC decoder is designed and implemented to achieve considerable high throughput with low memory consumption.
The rest of this paper is organized as follows. Section II introduces the novel early termination strategy. Section III describes the memory efficient structure of layered LDPC decoder. The implementation results and conclusions are presented in section IV and section V, respectively.

B. Early termination stategy in layered LDPC decoder
Correct checking process (HC checking) is the common method of early termination. However, in layered decoder, HC checking is implemented at the expense of decreased throughput and/or more hardware complexity, so it is rarely used. In order to achieve the correct checking, some early termination strategies by stopping decoding some subiterations or variable nodes according to sub-iteration HC checking are proposed [5] [6]. However, the hard decisions of LLR messages for variable nodes may change during layered decoding, even if they satisfy HC checking of sub-iteration occasionally, because all the updated LLR messages are not stable until the entire variable nodes are correctly decoded. Therefore, these strategies may cause misjudgments and decoding BER error floor occurs [6].

To achieve a reliable early termination strategy, this paper presents a method of comparing the signs between current LLR and updated LLR of all variable nodes. After every subiteration calculation, if the signs of updated LLR are the same as the signs of current LLR for variable nodes of this updating process, termination flag is added one, otherwise, termination flag is set to be zero. The iteration will be terminated when the termination flag is equal to the serial degree φ, which is the ratio of the PCM row number over the parallel degree. In this strategy, the number of one bit comparison operations is simply the product of parallel degree and row weight. Therefore, the strategy will not increase much hardware complexity when implemented simultaneously with layered decoding. C. Simulation results
To verify our method, a (9216, 4608) LDPC code in China Multimedia Mobile Broadcasting (CMMB) standard is chosen for example. The quantity of simulation data is 5×109 bits. As shown in Figure 1, the proposed early termination strategy is reliable because there is no error decoded bit when the signalto-noise ratio (SNR) is higher than 1.9dB, while the BER curve of the strategy presented in [5] has the error floor phenomenon. Meanwhile, the same performance of our strategy and the strategy of HC checking in layered is kept. This shows the reliability of the proposed method from another aspect. III. MEMORY EFFICIENT DECODER ARCHITECTURE FOR LAYAERD LDPC CODES

A. Non-uniform quantization scheme

The word length of the soft information directly determines the size of memory, which stores the LLR and EX messages in LDPC decoder. Therefore, the word length reduction with acceptable performance loss can decrease the hardware area which includes memory banks and the computation logic. As shown in Figure 3, the performance of (7:3) uniform quantized layered LDPC decoder in CMMB standard is close to that of the floating point, while (6:3) uniform quantization scheme suffers from serious performance degradation. We presented a 6-bit non-uniform quantization scheme without significant performance loss to reduce word length. As shown in Table I, where s denotes the sign bit, if the magnitude of message |y| is less than 1.75, the message is quantized with uniform quantization with the quantization step 2-3. B. Structure of extrinsic messages storage
For pipeline requirement, the number of EX messages memory is usually equal to the parallelism of check nodes in layered decoder [7]. If the parallel degree is set to be 18, there are 108 dual-port 256x6 memories used for EX messages [7] in CMMB standard, since the row weight is 6. Because the EX messages in a row can be compressed into one mixed message [8], the number of EX messages memory can be decreased to be the same as parallel degree while maintaining the memory depth. A row of EX memory contains a row of EX messages in PCM by their compressed message form. 18 degree of parallelism means 18 rows in PCM are decoded at the same time. Therefore, if the number of EX memory is equal to the parallel degree, one sub-iteration process which calculates 18 rows can be done in one cycle. Furthermore, the EX memory can be designed using single-port memory as long as those 18 rows EX messages are placed to 18 memories, respectively. IV. IMPLEMENTATION RESULTS
To evaluate the performance of the proposed architecture with the early termination strategy, we designed a layered LDPC decoder for CMMB standard. The decoder supports both code rate 1/2 and 3/4 with the same code length 9216. The overall architecture of the decoder is shown in Figure 4, where AG means address generator. The degree of parallelism is 18, so there are 256 sub-iterations in one iteration. To reduce hardware area, the memory which stores LLR message is designed in single-port memory. Therefore, 2 clock cycles are needed to complete one sub-iteration decoding which contains reading data and writing data through memory. We further designed 2-stage pipeline in CTV/VTC calculation module for the tradeoff of the decoder throughput and the critical path length, which is shown in Figure 5. The overall decoder is in 6-stage pipeline structure. The clock cycles required to decode a block codeword LDPC codes are 256×2×N_ite+6+1.As shown in section II, the average number of iterations is 8.4021 when applying the proposed early termination strategy, the total clock cycles are 4309. Thus, the throughput of the decoder is 535Mbps. Calculated by the information bits, the effective throughput is 267Mbps for the code rate 1/2 and 401Mbps for the code rate 3/4.The memory consumption is 171 Kbits. Table II shows the main parameters of our LDPC decoder with some other LDPC decoders design for CMMB standard. The proposed decoder achieves about 21% and 23% memory reduction compared with that in [7] and [10], while still maintain a much higher throughput. Compared with the decoder designed in [8], we have about 44% hardware area reduction if calculated in the same technology, while the memory consumption and the throughput are nearly the same.
