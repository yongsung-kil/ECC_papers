A Methodology based on Transportation Problem Modeling for Designing Parallel Interleaver Architectures
Awais SANI, Philippe COUSSY, Cyrille CHAVET, Eric MARTIN. Lab-STICC, Université de Bretagne-Sud, Lorient

Abstract- For high-data-rate applications, turbo-like iterative decoders are implemented with parallel hardware architecture. However, to achieve high throughput, concurrent accesses to each memory bank has to be performed without any conflict. The consideration applies to the two main classes of turbo-like codes: Low Density Parity Check (LDPC) and Turbo-Codes. In this paper, we present an original approach based on Transportation problem modeling which finds conflict free memory mapping for every type of turbo codes and which optimizes the resulting interleaving architecture.

1. INTRODUCTION
In telecommunications and multimedia domains, continuously increasing customer services require severe performance to implement the new communication standards such as DVB-S2 and DVB-T2, GSM, WiFi, WiMAX. Due to their powerful errorcorrecting properties (near-Shannon-limit), in LDPC [1] or turbolike codes in their parallel or serially concatenated versions [2], are being currently used in most digital communication systems for channel coding, demodulation, equalization and synchronization.
The decoding of turbo codes is carried out by an iterative algorithm in which decoders processed frames iteratively in order to estimate the corresponding data. However, iterative decoders suffer from high latency and parallel decoder architecture becomes the only feasible solution in high throughput applications. In parallel architecture, several processing elements PE (decoders) are connected with memory elements (registers, RAM) through interconnection network composed of steering components (barrel shifter, multiplexers) as shown in Figure 1. The communication network interleaves the frames exchanged by the PEs according to a predefined rule named interleaving law or permutation law. Interleaving law is a permutation law, also referred as ɉ, that scrambles data to break up neighbourhood-relations [3]. Interleaving law varies from one communication standard to another and it is important factor in turbo-codes performance.
Due to the aforementioned interleaving constraint, the implementation of parallel hardware architecture requires complex interconnection network supporting the intensive interleaved memory access. The design of communication network becomes more and more challenging with the increase in the number of data, the number of iterations and the parallelism. Moreover, parallel implementation suffers from memory accesses collision problem in which more than one PE concurrently accesses the same memory bank to read or write data. This collision problem arises because there are two different schedules of the reading/writing operations. In one schedule, decoder accesses the data for reading/writing in natural order while in the other it access the data for reading/writing in interleaved order. Indeed, while designing architecture for parallel turbo-decoder, interleaving is considered as a limiting factor concerning architectural cost and the overall system performance.

2. PROBLEM FORMULATION
To explain the problem, we consider a set of D data elements {d1, d2,…, dD} and a set of P processing elements {PE1, PE2,…, PEP} which process these D data elements first in natural order and then in interleaved order in T time instances {t1, t2,…, tT}, where T = 2D/P. In order to store these D data elements and to achieve parallel processing of data for high throughput, a set of B memory banks {b1, b2,…, bB}, where B = P, is used. All the memory banks have the same size R which is equal to R = D/P.

Mapping problem Store D data elements in B memory banks in such a manner that P
processing elements can access B memory banks in parallel at each natural and interleaved order time instance without any conflict.

We introduce two matrices, one is related to natural order access and the other is to interleaved order access. Each matrix has P rows, related to the processing elements, and T/2 columns, related to the time instances. Data elements in each row are processed by the processing element connected with this row. Similarly, the P data elements in each column need to be accessed in parallel by P processing elements for parallel decoding architecture. 3. DEFINITIONS
A graph G = (V, E) is a set of nodes V, and a set of edges E. If v,w ∈ V then an edge (v,w) ∈ E is incident to v and to w, and vertices v and w are said adjacent. A subgraph of G is a graph whose vertices and edges are in G. A graph G = (V1 ∪ V2, E) is bipartite, if V1 and V2 divide the vertices set so that each edge is incident to a vertex in V1 and a vertex in V2 i.e. V1 ∩ V2 = ∅. The degree of vertex v is the number of edges incident to v. A graph is deg-regular if all vertices have the same degree deg. A graph is semi regular, if either all the vertices in V1 or all the vertices in V2 have the same degree. A path P is a sequence of edges (v1, v2), (v2, v3),..., (vn-1, vn). The ends of P are vertices v1 and vn. If v1 ≠ vn, P is open; otherwise P is closed. A cycle is a closed (simple) path, with no repeated vertices other than the

In semi regular bipartite graph G = (V1 ∪ V2, E), all the vertices in V1 or V2 have the same degree.

Definition 1: A semi 2-factor is defined as a 2-regular subgraph in G with 2Y vertices where every node is incident with exactly two

edges and where Y= Min(|V1|, |V2|).

Definition 2: a Turbo Bipartite Graph TBG is a bipartite graph = (T∪ D, E) in which vertex set T represents all the time instances

and vertex set D represents all the data elements used in the

computation. An edge (t, d) is incident to the data element vertex d

and to the time instance vertex t if d needs to be processed at t (i.e.

data d will be read and next written at time t). TBG has the

following two distinct properties:

1- The number of accesses to data or processing elements at any

time instance is always equal (the number of PEs is equal to

the number of memory banks) which implies that

corresponding bipartite graph is always semi regular and

each time node has same degree ft = P. 2- Each data element is accessed only two times: one in natural

order access and the other in interleaved order access. This

implies that all the data nodes in the bipartite graph have the

same degree, fd = 2.

Corollary 3: every Turbo Bipartite Graph with ft = 2k or ft = 2k + 1, where k is an integer, contains k disjoint semi 2-factors.
Since the graph is semi regular on both vertex sets T and D with fd = 2, we always find semi 2-factor in the Step II when considering TBG. Proof: we first join the two edges connected with each data node and then remove all the data nodes to form regular graph G1 = (T, E1) as shown in Figure 3.b. In this graph, |E1| = |D| i.e., each edge in G1 corresponds to two edges or a data node in G. Since G1 is regular, theorem 1 and 2 give us the proof that 2-factor always exists in G1 which implies that semi 2-factor of 2Y vertices where |T| = Y always exists in G.
Every 2-factor is a collection of cycles that spans all vertices of the graph going from 1 cycle with 2Y vertices up to Y cycles 2 vertices.
Additionally, each cycle ci in G1 is even which means ci contains even number of edges or time nodes because vertex set T is divided into two partitions i.e. natural or interleaved orders.
An edge coloring of G is an assignment of a color to each edge in G. An edge chromatic number, χ`(G), is the fewest number of

colors necessary to color each edge of a graph so that no two edges incident to the same vertex have the same color. In [13], the authors proved that if the maximum vertex degree of a bipartite graph is f then, χ`(G) = f.
Edges of every even cycle can be assigned with two colors which implies that edges in ci and every 2-factor in G1 can be colored with two colors. This results in the following lemma. Lemma 1: all the data nodes in semi 2-factors of turbo bipartite graph can be assigned with two memory banks.

4. PROPOSED APPROACH
The proposed algorithm is divided into two steps. In the first step a bipartite graph is constructed based on two data access matrices. The second step uses an original algorithm that relies on transportation problem modeling to successively find semi 2-factors in the bipartite graph. The proposed algorithm also assigns two colors only to each semi 2-factor i.e. binds data to memory banks. Furthermore, the resulting memory mapping will respect a user defined target interconnection network, each time interleaving law allows.

Step I: a turbo bipartite graph TBG = (T∪ D, E) is constructed based on natural and interleaved data access matrices (e.g. Figure 3).

Step II: TBG is divided into k semi 2-factors as explained in corollary 3 by using a simple approach based on transportation problem modeling [14].

Transportation Problem Let us consider a set of I producers where producer i has a supply
of ai units of a particular item. In addition, there are J consumers where consumer j requires zj units of item. We assume that ai, zj > 0. For each link from producer i to consumer j, lij, let oij be the cost to transport a single item on lij and xij be the capacity of lij i.e., the maximum number of items that can be shipped at the same time along lij. The transportation problem is to minimize the cost of shipping item from
producers to consumers so that each consumer j fulfills its demand zj and every producer i operates within its capacity ai.
The network model of the transportation problem for 3-produer and 3-consumer is shown in Figure 4.a. This network is a bipartite graph since the nodes can be divided into two parts i.e., producers and
consumers. The matrix model of the transportation problem is shown in Figure 4.b. Each cell Mij, corresponding to the producer i and consumer j, contains oij and xij of the route lij. If the route lij does not exist between producer i and consumer j, then the corresponding cell Mij is kept empty. We will use later equivalently Mij and lij.

To find semi 2-factor, we consider our mapping problem for turbo codes as transportation problem by considering all the data nodes as producers and all the time nodes as consumers. The route lij exists between data node di and time node tj if data di is accessed at tj. The capacity xij of lij is kept 1 since a data is accessed only once at a given time tj. In order to find semi 2-factor: (1) the demand of each consumer is kept to 2 and (2) each producer either provides 2 items (i.e. one item for the natural access and one item for the interleaved access) or is not included in the current semi 2-factor (i.e. each producer must work at its full capacity). The cost oij of lij is kept 1 since the cost is not taken into account in the current version. It will be used when we will consider the complexity of the network architecture.

At this point, the algorithm tests whether all the consumers fulfill their demands. If not, the algorithm starts constructing another cycle c2. For this, our algorithm selects consumer whose demand is still unfulfilled and which has at least one deleted route. Using this deleted route, the algorithms selects the route whose selection respects the targeted steering rule and assigns a bank ba to this route (selection procedure for steering rule based on barrel shifter is presented in next section). After the assignment of ba, the algorithm repeats the same process used for the construction of c1 to complete c2. When the algorithm finds that demands of all the consumers are fulfilled then it declares that sfcur is constructed. In that case, the algorithm tests whether k semi 2-factors are constructed. If not, the algorithm removes sfcur from transportation matrix, initializes all consumers with demand of 2 and starts constructing sfnext from remaining matrix using the process described above until k semi 2-factors are constructed. 6. RELATED WORKS

Currently three classes of approaches to design parallel decoder architecture exist to tackle the collision problem:
• Design dedicated interleaving law in order to avoid collision problems [4], [5]
• Use extra memory elements and control logic in the communication network in order to remove conflicts [6], [7], [8]
• Find a memory mapping to provide conflict free concurrent access to all the memory banks [10], [11]
In the first category of decoder implementation, specific interleaver law is designed in order to get rid of collision problem. In [4], the authors explain collision-free permutation thanks to the combination of a spatial and a temporal permutation. However, this approach is unable to handle multimode architecture. In [5], the authors describe a design rule to obtain collision-free interleaver that preserves this property even when pruned. The authors propose an incremental algorithm that generates interleavers by adding new elements in successive steps, to a small initial permutation. However, the resulting architectures in all these approaches may not be standard compliant.
In the second approach of decoder implementation, collision problem is tackled either through the addition of extra memory elements and/or complex interconnection network. In [6], in case of collision, conflicting information is stored in the communication network until the targeted sub-block can access it. In [7], an advanced heterogeneous communication networks (based on Butterfly and Benes topologies) are proposed in order to handle on-chip communication in multiprocessor parallel turbo decoder. These networks avoid the conflict if and only if all the paths have a different destination which is unfortunately not the case in turbodecoding and the authors have to use queues to store conflicting

information. In [8], Binary de Bruijn network is employed for providing flexible on-chip network for LDPC decoder. Concurrent accesses to the same memory bank are avoided through dedicated routing algorithm which deflects one of the conflicted packets at the router. The flexibility in these complex interconnection networks is paid through additional hardware, increased power consumption and decoding latency.
In the last class, methodologies for solving collision problem are proposed to map the data in different memory banks for conflict free concurrent accesses. The most common approach is to prepare conflict graph in which a node represents a data and two nodes are connected if and only if the associated data are accessed at the same time. Node coloring approach can then be used to solve the mapping problem: each color corresponds to one memory bank. However, node coloring is NPcomplete problem as shown in [9]. In [10], the authors proposed a heuristics to find conflict free memory mapping in turbo decoder. Contrary to the literature belief, the authors have proven that for every code, conflict free memory mapping always exists to tackle collision problem. However, the proposed approach is based on a simulatedannealing algorithm, so the user cannot predict when the algorithm will end. In [11], another heuristic is proposed which finds conflict free memory mapping while optimizing the storage element and interconnection network. However, all these heuristics fail to remove the computational complexity of the problem.
7. CONCLUSION
In this paper, we have presented an original approach based on transportation problem modeling in order to design parallel decoder architecture for turbo-like codes. The approach allows finding conflict free memory mapping for concurrent access to the memory banks for every type of codes. The approach can also generate memory mapping which respect a target interconnection network, if the interleaving law allows. In the future, cost of the interconnection network will be considered for designing reduced size and lower hardware cost architecture.
