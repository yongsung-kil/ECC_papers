QC-LDPC Decoding Architecture based on Stride Scheduling

Bongjin Kim and In-Cheol Park Dept. of Electrical Engineering, Korea Advanced Institute of Science and Technology (KAIST)
Daejeon 305-701, Republic of Korea bjkim.ics@gmail.com, icpark@kaist.edu

Abstract—In this paper, an area-efficient decoder architecture is proposed for the quasi-cyclic low-density parity check (QCLDPC) codes specified in the WiMAX 802.16e standard. In order to achieve low area and maximize hardware utilization, the decoder utilizes 4 decoding function units, which is the greatest common divisor of the expansion factors. Furthermore, the decoder adopts a novel scheduling scheme, named as stride scheduling, to remove the conventional flexible permutation network and also minimize the number of memory accesses. The synthesized decoder costs 49K of logic gates and 54,144 bits of memory, while maintaining the throughput over the requirement of the WiMAX.
I. INTRODUCTION
Low-density parity-check (LDPC) codes introduced in the early 1960s by R. Gallager [1] have attracted much attention both in academy and industry, because of excellent error correcting capability near the Shannon limit, low iterative decoding complexity, and inherent high parallelism. Owing to these beauty, they have been adopted in a number of wireless communication standards such as IEEE 802.11n (Wireless Local Area Network, WLAN), IEEE 802.16e (Wireless Metropolitan Area Network, WiMAX) [2], and Digital Video Broadcasting (DVB-S2).
As the straightforward fully parallel architecture [3] is hard to realize due to the complex interconnection, the partiallyparallel architecture has been considered as a tradeoff. Furthermore, the architecture-aware codes such as quasicyclic LDPC (QC-LDPC) codes make the partially-parallel decoders more suitable. The parity check matrix (PCM) of a QC-LDPC code is composed of a number of submatrices each of which is a cyclically shifted identity matrix or a zero matrix. In this structure, all the rows within a layer, or a block row which is as large as the submatrix size, are independent of one another so that they can be processed concurrently without causing any conflicts. The partially parallel decoders take this advantage in their implementation.
The QC-LDPC defined in the WLAN or WiMAX standards supports variable codes of different lengths and different code-rates. For a code rate, the code length can be enlarged by using a large expansion factor, where the

expansion factor indicates the submatrix size in the PCM. Therefore, the multi-mode operation should be provided in the decoders designed for these wireless standards. In order to support multiple decoding cases while keeping the throughput high, a flexible permuter network is generally required to distribute data to a number of processor units installed [4][5]. As a result, the multi-mode decoder consumes large area and power, which is undesirable in portable applications.
In this paper, we present an area-efficient multi-mode decoding architecture for the QC-LDPC codes defined in the WiMAX standard. The proposed decoder is also based on the partially parallel architecture. To reduce hardware complexity without violating the throughput requirement, the number of decoding function units (DFUs) is minimized and they are processed in a pipelined manner. In addition, check nodes are processed in a non-sequential order instead of the conventional sequential order. The non-sequential processing of check nodes, called stride scheduling, makes it possible to replace the complex flexible permuter network with simple, small-sized cyclic shifters.
II. PREVIOUS WORKS
The number of DFUs employed in the conventional partially parallel decoder [4] is as many as the largest expansion factor in order to cope with multi-mode decoding operations, which is 96 for the WiMAX standard. A unified architecture that can perform both the layered decoding [6] and the two-phase message passing [1] has been proposed for the WiMAX QC-LDPC codes in [5]. Alike the decoder in [4], this architecture requires 96 variable node blocks and 96 check node blocks, leading to large area consumption. Furthermore, both the decoders include flexible permutation networks to support various input size and shift values. Two logarithmic barrel shifters are integrated in [5], one for the forward permutation and the other for the backward permutation, while [4] reduces the permuting networks to one Benes network by excluding the de-permuter. As mentioned before, all the DFUs are working simultaneously, which means that the APP memory should supply all the data required in the parallel processing. Suppose that we process a layer containing a submatrix shown in Fig. 3(a), whose shift amount is 13. The APP memory stores data sequentially as illustrated in Fig. 3(b), where 4 data are stored in an entry of the APP memory for the parallel processing. In the sequential scheduling, we need to read the data whose indices are 13, 14, 15, and 16. As the data are split into two entries in the APP memory, however, it is not possible to access the data in a cycle. This drawback can degrade the decoding throughput significantly.

To resolve the data access problem, we propose a new

scheduling called stride scheduling. As shown in Fig. 4(a), the

check nodes of a layer are partitioned into p subsets each of

which equally contains rows. A DFU is responsible for

the process of a subset. More specifically, the 0-th subset is processed by DFU0, the 1st subset by DFU1, and so on. As the number of DFUs is decided to the GCD of the expansion

factor, every DFU serially processes the same number of rows

within a layer, which means that the indices of any two

adjacent check nodes being processed at any moment have an

equal interval or stride, which is always . This is the

reason why the proposed scheduling is called stride scheduling.

the data to be accessed at the same time into one entry. The

data being processed at the same time can be considered as a

group. There are groups in a layer, and each group can be

indexed by the modular value of the row indices. All the data

in a group have the same remainder if the row index is divided

corresponding to rows 0, 6, 12 and 18. Fig. 4(b) shows such a

There is another factor to be considered in the practical implementation. The pipelined processing architecture shown in Fig. 2 accesses the APP memory two times, one for reading and the other for writing. As the two accesses are overlapped in the pipelined processing, two access ports are usually required in the APP memory. A straightforward method to implement the APP memory is to use the two-port memory. However, the two-port memory is associated with much higher cost than the single port memory does. To reduce the overhead, a single port memory is adopted but its entry size is enlarged to include two data sets. In case of 4 DFUs, for example, an entry of the APP memory is enlarged to store 8 data by merging two adjacent entries into one, as shown in Fig. 5. Two data sets are read in a cycle, and two data sets are written in the next cycle, which means that the read and write accesses are interleaved. As only one data set can be processed in a cycle, the other data set is temporarily stored into a buffer and fetched at the next cycle, as depicted in Fig. 5, where two buffers, a read buffer and a write buffer, are inserted to support the interleaved access. The proposed memory structure resolves the memory access conflict without using the two-port memory.
Since the two data sets contained in the same entry are processed independently, the size of the permuter can also be reduced to 4, which is the size of a group. Although [11] uses a small number of processors, the decoder is viable only for specific circumstances, and it is not efficient for multimode operations. A low-area multi-mode decoder [8] presented for the WiMAX standard uses a large number of processing units and requires large-sized cyclic shifters. As the APP data required for the decoding can exist in two separate entries, the data are retrieved by invoking two read accesses using a double-rate clock. The proposed stride scheduling removes this overhead completely.

V. CONCLUSION
In this paper, an area-efficient QC-LDPC decoding architecture has been proposed for the IEEE WiMAX standard. To minimize the decoder area while meeting the throughput specification, the number of decoding function units is set to 4. As a result, the flexible permuter network is replaced with simple cyclic shifters. In addition, the stride scheduling has been newly proposed to process the rows in a layer non-

sequentially, which is effective in further reducing the size of the cyclic shifters and the number of memory accesses. The implementation results show that the proposed architecture achieves the smallest area, and the throughputs obtained for all the code rates and code lengths are far above the requirement of the WiMAX standard.
