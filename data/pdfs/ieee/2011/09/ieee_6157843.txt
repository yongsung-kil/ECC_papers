Infeasible Primal-Dual Interior-Point Algorithm for Linear-Programming Decoding of LDPC Codes

Xuye Wang
School of Information Science and Engineering Shandong University, Jinan, 250100, P.R.China
wangxuye419@163.com
AbstractÔÇöThe linear programming (LP) decoding of LowDensity Parity-Check (LDPC) codes is widely concerned for its maximum likelihood (ML) features. It is that the ML decoding rule for LDPC codes can be relaxed to LP optimization problem. Therefore, this paper focus on an efficient algorithm called infeasible primal-dual interior-point (IPDIP) to solve the LP problem. In each iteration, the IPDIP algorithm obtains the predictor and corrector steps by solving the Karush-KuhnTucker (KKT) equation twice. The predictor term is used to responsible for optimal solution and the corrector term keeps the current iteration away from the boundary of the feasible region. Furthermore, a modification of the centering parameter is developed to accelerate the convergence speed for the IPDIP algorithm. Simulation results of LP decoding demonstrate that the proposed IPDIP algorithm achieves beautiful bit error rate (BER) performance and good global convergence properties with less iteration number and time than other algorithms which only solve the KKT equation once by Newton method or use the not modified centering parameter.
Keywords-LDPC codes; LP decoding; IPDIP algorithm; predictor step; corrector step; centering parameter
I. INTRODUCTION
LP decoding of LDPC codes was proposed by Feldman et al. [1] as an approximation to ML decoding. An important conclusion of [1] is that the ML decoding problem can be relaxed to a LP optimization problem, and for the LDPC codes, the number of LP problemÔÇÖs constraints increases exponentially with the code length. Then it becomes more significant to research an efficient algorithm to solve the largescale LP problem.
At present, it has been proved that the IPDIP algorithm is a powerful tool for solving very large-scale LP problem [2]. Not only because itÔÇÖs computational efficiency, but also because it is considerably simpler than the simplex algorithm for largescale LP.
The main purpose of this paper is to describe the implementation of IPDIP algorithm for LP decoding of LDPC codes and to present the decoding results. Concretely, we explore the relaxed LP decoding for LDPC codes and use the IPDIP algorithm to solve the LP problem.
The rest of this paper is organized as follows. In section II, the LP model of LDPC decoding is constructed. In section III, we give a more explicit presentation for the LP decoding. The NewtonÔÇÖs method, predictor-corrector (P-C) method and IPDIP algorithm are described at full length in section IV,

VII. RESULTS
In the simulations, a random LDPC code with the parity check matrix H with 288 rows and 576 columns, is sent over an AWGN channel after BPSK modulation.
Figure 1 shows the needed iteration number of IPDIP algorithm and P-C method for different SNR but same process to find step size and stopping criteria. The iteration number takes the average value of numerous decoding results. The figure suggests that IPDIP algorithm needs less iteration number and less time than P-C method.
Figure 2 shows the MCP and CP curve for increasing iteration number when SNR is 2dB. From the figure, we can know that MCP reduced more quickly than CP when the iteration near the end stage.
In Figure 3, the values of objective function results were displayed for every iteration at SNR=2 dB. It is clear that the curve of IPDIP algorithm is faster to reach the minimum than other curves. The figure makes a conclusion again that the IPDIP algorithm needs less iteration number and time than NewtonÔÇÖs method and P-C method.
In Figure 5 we present the BER performance results of the IPDIP and min-sum (MS) decoding algorithm. It can be observed that the IPDIP algorithm achieves better BER performance.

VIII. CONCLUSION
The paper has constructed the framework of primal-dual infeasible-interior-point algorithm. Triumphantly and of course, we obtain the preliminary results for LP decoding by using the infeasible primal-dual interior-point algorithm. However, more work should be done to further the research of the algorithm and exploit the implementation in detail.
