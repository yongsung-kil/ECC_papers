I. INTRODUCTION
Cooperative communication has attracted considerable attention for increasing the reliability of systems as a virtual multiple-input multiple-output (MIMO) system. In cooperative communication, users share their resources to help othersŌĆÖ transmission while transmitting their own information, which forms a virtual MIMO system. Therefore cooperative communication achieves cooperative diversity without employing multi-antennas which increase the complexity of the transceiver at the terminal. Van Der Meulen proposes the threeterminal model which is the foundation of cooperative communication in 1971 [1]. Substantial advances of cooperative communication were made by Cover and El Gamal in 1979 [2].
Amplify-and-forward (AF) [3], decode-and-forward (DF) [4] and coded cooperation (CC) [5] are three most popular protocols of cooperative communication. In the AF scheme, the relay node (R) can be regarded as a simple amplifier. In the DF scheme, R re-encodes the information and forwards when it decodes successfully. And in the CC scheme, R re-encodes the information by a new channel code to obtain extra coding gain.
LDPC codes, first discovered by Gallager in 1962 [6] and rediscovered by Mackay in 1990s [7] [8], have caught much attention of channel coding field due to its near-capacity performance. Recently, LDPC codes are used as channel coding in coded cooperative communication system. This paper focuses on the structure of parity check matrix to construct LDPC codes for coded cooperation.

II. SYSTEM MODEL

We consider a classical coded cooperative communication system consisting of a source (S), a relay (R), and a destination (D). As illustrated in Fig.1, the transmission of data packets consists of two phases. In the first phase, S broadcasts its packet xS to R and D simultaneously. In the second phase, if xS can be decoded successfully in R, R will encode the information from S and relay new parity bits to D, otherwise R will do nothing.

We consider two LDPC codes C1 and C2 which are used in S and R respectively with parameters (K1, N1) and (K2, N2), where Ki denotes the code dimension and Ni denotes the code length. S encodes the sequence of information bits I whose length is K1 using parity check matrix H1. C1 and C2 are assumed to be systematic for simplicity. Thus C1ŌĆÖs codeword is (I, P1), where P1 is the sequence of parity bits whose length is N1 ŌłÆ K1. The codeword (I, P1) is broadcast to R and D. If R decodes the received signal successfully, R encodes using the parity check matrix H2 and relays the sequence of new parity bits P2 whose length is N2 ŌłÆ K2. Then D combines the signals from S and R, the combined codeword is (I, P1, P2). And D can decode the combined codeword by using the parity check matrix H which is obtained by the combination of H1 and H2. Fig.2 shows H.
However, H has several weaknesses when it is combined by two independent parity check matrices H1 and H2. The low density of the non-zero entries in the parity check matrix is very important for the performance of LDPC codes. Constructing codes with lower rates, the parity check matrices are always sparser than those with higher rates. But H may not be sparse because the combination of two LDPC codes with higher rates. Besides, cycle is another important factor for the performance of LDPC codes. Though many cycles can be avoided for codes with lower rates, H has all the cycles in H1 and H2. For achieving a good performance, coded cooperative communication system needs successful decoding in both R and D. In other words, system needs two matrices H1 and H both have good performance. In order to solve the above weaknesses, this paper presents two schemes to joint design of H1 and H.
III. CODED COOPERATIVE CODING USING PUNCTURED LDPC CODES
In this section, a coded cooperative coding scheme which uses rate compatible LDPC codes based on puncturing parity bits is presented. This scheme joint designs of matrices H1 and H.

In this scheme, S uses rate compatible LDPC codes. S encodes and obtains codeword (I, P ), where P denotes the sequence of parity bits. Then S punctures part parity bits to obtain codeword (I, P1) and broadcasts the codeword to R and D. If the received packet is successfully decoded, R forwards P2 the sequence of parity bits which are punctured in S (P2 = {x|x Ōłł P x Ōłł P1}). Thus, if coded cooperation works, D receives codeword (I, P ), otherwise D receives codeword (I, P1).
This scheme ensures the good performance of the parity check matrix H which is low density and avoids the interference of cycles. On the other hand, suitable puncturing scheme is used to ensure successful decoding in R and D when coded cooperation fails. For instance, we can choose different puncturing schemes for different block-type LDPC (B-LDPC) codes with different rates [9].
Using rate compatible LDPC codes in coded cooperation is a simple and efficient scheme. This scheme ensures coded cooperation system to achieve good performance with low complexity when the number of the bits relayed by R is suitable according to P . If coded cooperation needs R to forward many bits which are close or even more than the length of P , system canŌĆÖt obtain an acceptable performance or even canŌĆÖt do it. So section IV presents a joint design scheme of H1 and H without the limitation of the number of the relay bits.

C. The construction of QC-LDPC codes based on the multiplicative inverses in finite fields
We construct QC-LDPC codes based on the multiplicative inverses in finite fields [11] to avoid the appearance of cycles and obtain H which is similar with the model described above.

[12] presents a base matrix (W ) which is expressed by (4). [12] proves that the parity check matrix which is obtained based on W can avoid cycles of length 4 and it has a girth at least 6.
According to (4), W is a circulant matrix because of the cycle of finite fields. In W , each row is the right cyclic-shift of the row above. If one element of W is ╬▒0, there will be a diagonal after each element of W is replaced with its circulant permutation matrix, because the circulant permutation matrix of ╬▒0 is identity matrix and W is a circulant matrix. When the differences between the indexes of elements keep unchanged in W , the parity check matrix based on W will not have cycles of length 4. Therefore, W can be multiplied by the multiplicative inverse of one element to make W has ╬▒0 in every row. By this way, we can decide the position of the diagonal. Then replace the submatrices which are above the diagonal with zero matrices to obtain a lower triangular matrix with quasi-cyclic structure.

V. NUMERICAL RESULTS
In this section, simulation results are presented. Fig.5 illustrates the bit error rate (BER) performance of the rate compatible LDPC codes based on puncturing parity bits in coded cooperation. We use quarter phase shift keying (QPSK) modulation and AWGN channel. The signal to noise ratio (SNR) of S-R link is 4.5dB and the SNR of R-D link is 5dB. The SNR of S-D link is shown on the horizontal xaxis. The mother code used in S is IEEE 802.16e (576,288) LDPC code. According to Fig.5, the LDPC code with rate 0.6 outperforms the LDPC code with rate 0.75 by about 4.5dB at BER=0.001, while the LDPC code with rate 0.75 outperforms the LDPC code with rate 0.9 by about 1.5dB. With the rate increasing, S punctures more parity check bits which

increases BER and reduces the possibility of cooperation. Thus, if the bits which are relayed by R is a small part of C, rate compatible LDPC codes based on puncturing parity bits is a good choice for coded cooperation because the system performance is acceptable with low complexity. Otherwise the scheme presented in section IV is a valid construction of QCLDPC codes for coded cooperation.
Fig.6 shows the BER performance of the (3024, 1323) QC-LDPC code which is constructed in section IV in coded cooperation. C1 which is broadcasted by S is a (1512, 1323) codeword and R forwards 1512 bits as extra parity check bits if cooperation works. Fig.6 also includes the BER performance of (1512, 1323) random Mackey LDPC code in the DF scheme. We assume the SNR of S-D link which is shown on the horizontal x-axis is the same with the SNR of R-D link and the SNR of S-R link is high which ensures successfully decoding in R. The simulation uses AWGN channel. According to Fig.6, the QC-LDPC code outperforms the random Mackay LDPC code with DF scheme.

VI. CONCLUSIONS
In this paper, we have proposed two methods for constructing LDPC Codes which are used in coded cooperative communication. The first method is using rate compatible LDPC codes based on puncturing parity bits. When the bits which are relayed by R is a small part of C, this scheme performs well with low complexity. The second method is using QC-LDPC codes based on the multiplicative inverses in finite fields. The QC-LDPC codes outperforms Mackay LDPC codes in DF scheme.
