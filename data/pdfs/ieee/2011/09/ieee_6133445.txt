Abstract— High-rate LDPC codes are increasingly used in transmission systems. Puncturing is a common scheme which produces a code with higher code rate starting from a base code with lower rate. This paper discusses a new method to derive a puncturing pattern, given the parity matrix of the base code. The method allows the extraction of a puncturing pattern which is optimized in terms of error floor performance and satisfies some constraints which results in a significant reduction of HW complexity with respect to other solutions. Simulation and HW measurements results on a punctured 19/20 code derived from a rate 15/16 LDPC code are reported.
Keywords – Low-density parity-check codes, puncturing.
I. INTRODUCTION
Puncturing of LDPC codes has received much attention in the technical literature. While some papers have focused on fundamental properties of LDPC punctured code ensembles and capacity results, see e.g. [1], other papers have focused on the design of puncturing distributions, considering also rate compatible distributions and focusing on performance at the threshold with respect to the optimal code of the same rate [2]. A few papers, see [3] have focused on the problem of finding optimal puncturing patterns for a given parent code and a set of target code rates. Most papers limit the analysis to threshold performance, using EXIT charts as their analysis tool, and present simulation results which fall short of showing the behavior of the punctured code at high SNR.
It is however well known that LDPC codes are affected by the so-called error floor [4] effect, which results in a degradation of the BER curve slope at high SNR. In some applications requiring very low BER at high SNR the errorfloor phenomenon can be a serious limit. In this paper we describe a novel method to design good puncturing patterns for high rate codes, starting from a given parent code and a target code rate, which minimize the error-floor. In addition we provide an extension to the method which reduces the complexity of implementing in HW the puncturing scheme. As an example, we present an application of the proposed method to a high code rate LDPC code used in a microwave transmission point-to-point radio link.
The paper is organized as follows: section 2 discusses the high rate puncturing method and its relationship to existing methods. Section 3 deals with simulation results, section 4 presents the high SNR results obtained using an FPGA-based

emulation platform. In the conclusions we summarize the main results and contributions.
II. HIGH RATE PUNCTURING
A. Reference Algorithm
In [3] Vellambi at al. present a rate-compatible puncturing scheme. Their scheme uses the idea that the degradation in performance is reduced by selecting a puncturing pattern wherein the punctured bits are far apart from each other in the Tanner graph of the code. This choice ensures that no small cycles are introduced when the code is punctured.
The proposed puncturing scheme in [3] takes in a paritycheck matrix H of a LDPC code and returns the set of indices to be punctured (I DEX). To compute I DEX, the algorithm proceeds as follows. Initially, the algorithm starts with an empty I DEX. At each time the algorithm selects a bit node uniformly at random form the set of available indices (REMAI ) and appends the selected vertex to I DEX. It adds all the neighbors of the check nodes connected to selected node to EIBOR and again deleted all elements of EIBOR from REMAI . In this way, the algorithm guarantees that bit nodes that will be selected in future step are at least a distance of four away from previously selected bit nodes. Of course, as the algorithm proceeds REMAI gets smaller and EIBOR gets bigger until finally REMAI becomes empty. At this point,
EIBOR is re-initialized and those check equations the involve the bit nodes in I DEX are deleted from H. Similarly, all bit nodes that do not participate in the remaining equations are also deleted to obtain the smaller matrix H’ (and its corresponding Tanner Graph G’). The algorithm is then re-run with H’ as input and the whole process is repeated until H’ = Ø.
The puncturing scheme presented in [3] is easy to implement and allows room for fine-tuning depending on the parent code to be implemented.
B. ew Algortihm
The proposed new puncturing algorithm extends the algorithm defined in [3] to obtain a good puncturing pattern that can reach higher code rates by relaxing some of the problem conditions. The new pattern is designed so that homogeneous clusters of consecutive bits are punctured. The main effect is twofold: more degrees of freedom in the choice of the puncturing pattern are allowed and the HW complexity of implementation is reduced, since the number of puncturing

In the first phase, the new algorithm first finds the set of punctured bits indices using the method proposed in [3], then enumerates all clusters which contain at least one punctured bits. In the second phase cluster selection is made: clusters with a greater number of punctured bits are picked first, the remaining clusters needed to reach the target code rate are selected so that the number of affected parity checks is minimized.
The algorithm steps are the following:
1. Find the set of punctured bits indices, I DEX, with Algorithm presented in [3];
2. Enumerate all punctured clusters, and collect the total number of clusters, ( _CLUSTER), the bit positions (I DEX_BITS) and the number of punctured bits (N_BITS) for each cluster;
3. If number of clusters with at least NT_BITS punctured bit is less than a preset threshold NT_THRESH, or if the number of clusters is less than the minimum number of cluster needed for the target code rate repeat steps 1 and 2;
4. Select the clusters with N_BITS greater than the preset threshold and add them to the punctured clusters set. If the number of clusters is equal to the one needed for the target code rate (NC_TARGET) exit;
5. Select the remaining punctured clusters with a number of punctured bits less than the preset threshold by minimizing the number of parity check involved, and add them to the punctured cluster set until the number of clusters is equal to the one needed for the target code rate.
Note that the algorithm can be applied to LDPC codes of any block length and code rate. In general, for short block lengths and high code rates the cluster size must be chosen so that the target punctured code rate can be well approximated, taking into account that a too large cluster size limits the algorithm degrees of freedom in selecting the best pattern.
C. Application Example
In the following we describe a worked example. We start from a given LDPC code with code rate 15/16, set the dimension of the cluster to 8 bits, i.e. one byte, and select 19/20 as the target code rate after puncturing.
The parent code is a regular LDPC code with quasi-cyclic structure, code rate 15/16, message length 7665 and coded block length of 8176 bits. The code parity matrix is free from 4-cycles and it is built with a single row of 16 circulants of size 511 and weight 4. This code is described in [5]. This code has been designed for microwave transmission applications.
In order to increase the code rate to 19/20 we need to puncture 88 parity bits out of 511. The 511 parity bits can be grouped in 64 clusters, where each cluster corresponds to 1 byte, with the exception of the last one, which has only 7 valid

bits. We number the bit indices as follows, where the square brackets indicate the byte boundaries:

Step 2 of the algorithm is based on the above indices structure. In fact, the algorithm uses this index to count the number of punctured bits ( UMBER_BITS) for every byte and, thus, to count the number of bytes with the punctured bits ( UMBER_BYTES).
In this example we set the NT_BITS = 2 and NT_THRESH = 9. Therefore the first two steps are repeated until the number of clusters = bytes more than 2 punctured bits is greater than 9. It has been observed in simulation that, in order, to obtain a good puncturing scheme, it is necessary that the number of bytes with two punctured bits must be minimized. This condition, as illustrated by the results in the section 3, is necessary but it is not sufficient, as we will show later.
Since we need to reach a code rate of 19/20, and the number of bytes to be punctured is at least 11, i.e. NC_TARGET set to 11. Depending on the random extraction of INDEX, in step 4 we might be faced with two different cases:
1. If the number of bytes with more than 2 punctured bits is greater or equal to 11, the puncturing scheme is composed by first 11 bytes founded in the steps 1 and 2, picked in order of decreasing number of punctured bits;
2. If the number of bytes with more than 2 punctured bits is less than 11, the puncturing scheme is composed by all these bytes and the remaining M clusters are from the set with 2 punctured bits.
In this case, since the choice can be made from a large set of bytes with 2 punctured bits, the selection of the M bytes is performed as follows:
1. Find the first two bytes with the greatest number of parity checks in common and add them to the punctured clusters set;
2. Continue with step 1 until all M additional clusters have been added to the punctured clusters set.
This selection method to select is based on the observation that decoding would better converge if the number of parity checks with punctured bits is minimized, since this reduces the number of bits which in the decoding process are affected by punctured positions. This heuristic observation will be validated through simulations in the next section.

V. CONCLUSION
Starting from the puncturing scheme presented in [3] we have proposed a new method which offers more flexibility and a reduced HW implementation complexity. The proposed algorithm produces a good puncturing scheme that involves clusters of bits. This solution allows a reduced set of puncturing positions to be stored. The algorithm takes and elaborates the set of punctured bits indices and returns the new set of cluster indices to be punctured. Simulations and measurements results show that to obtain a good puncturing pattern the clusters should be selected so that the number of punctured bits in the set is maximized and the number of parity checks affected by punctured bits is minimized. The resulting pattern show good performance in the convergence as well as in the error floor region. The example discussed show an application of the proposed method to the design of a high code rate FEC for a microwave transmission system.
