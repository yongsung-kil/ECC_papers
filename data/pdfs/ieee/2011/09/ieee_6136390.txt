High Rate (3, k) Regular LDPC Encoder Architecture

Silvia Anggraeni, Fawnizu Azmadi Hussin and Varun Jeoti
Electrical and Electronics Engineering Department Universiti Teknologi PETRONAS
Bandar Seri Iskandar, 31750 Tronoh, Perak, Malaysia silvia31804@yahoo.com, fawnizu@petronas.com.my and varun_jeoti@petronas.com.my

Abstract—This paper highlights recent developments in low density parity check (LDPC) encoder. There are some parameters applied in LDPC encoder such as type of LDPC codes, code length, code rate and encoding method. We emphasize that no attempts have been made for the implementation of (3, k) regular LDPC encoder with high code rate (R ≥ 0.875) and few works on flexible LDPC encoder which accommodates various code rates and code lengths. Therefore, this paper proposes a high rate (3, k) regular LDPC encoder architecture which is suitable for high code rate (R ≥ 0.875) applications. Division of workloads between stages is built based on the number of non-zero elements in the parity check matrix (H).
Keywords-component; low density parity check (LDPC); encoder; encoding complexity; code rate; field programmable gate array (FPGA).
I. INTRODUCTION
In order to achieve a reliable communication where retransmission of information bits is not conveniently implemented and there is no reverse channel to inform of an error, it is necessary to include some redundant or parity bits. These parity bits are utilized to detect and to correct an error during data transmission through a channel. This technique of recovering transmission errors is termed as forward error correction (FEC) scheme with error correcting code (ECC). The capability of detecting and correcting the errors in FEC scheme is mainly determined by the type of ECC applied in the system. Among several ECC’s in the literature, low density parity check (LDPC) codes have become quite popular owing to the fact that they exhibit performance that is closest to the Shannon’s limit.
LDPC codes were invented by Gallager in 1960’s [1, 2]. Gallager’s work exploited two novel ideas which are iterative decoding and code-construction. Gallager’s work received no attention for a long time since that time it was nearly impossible to implement the proposed ideas. LDPC codes were re-discovered and brought back in 1996 independently by MacKay [3, 4], Neal [3], Wiberg [5], M.Sipser [6] and D.A. Spielman [6].
LDPC codes are derived from a pre-designed parity check matrix (H) that consists of extremely low density of 1s. Since the sparse H is generally used in the decoding part, a low decoding complexity is achieved. Unfortunately, encoding complexity of LDPC codes may not be as low as the decoding part.

The common method of encoding in LDPC codes is converting matrix H into a generator matrix (G). If the matrix G is dense, the encoding complexity will be quadratic in the code length denoted by O(N2). If the matrix H can be optimized, there is a possibility that the encoding complexity is linear using this method. Since there is a chance that we get a dense matrix G from the chosen H, some researchers chose to avoid this method. They preferred to utilize the sparseness of H to be used in encoding like Richardson and Urbanke [7] and Neal [8]. Encoding method proposed by Richardson and Urbanke utilizes an approximate lower triangular form of H which is divided into small sub-matrices to get almost linear encoding complexity. Another method introduced by Neal applies lower and upper (LU) triangular matrix decomposition. However, it is not easy to find a good sparse matrix with lower and upper triangular sub-matrices for H.
This paper proposes a high rate (3, k) regular LDPC encoder architecture which is suitable for R ≥ 0.875.This paper also explores the recent developments in LDPC encoders. In order to understand these works better, there are some important parameters utilized in LDPC encoder that need to be mentioned. These parameters are type of LDPC codes, code length, code rate and encoding method. The rest of this paper is organized as follows. Section II discusses important parameters of an LDPC encoder. We then, describe the proposed regular LDPC encoder in section III. We show the architecture of LDPC encoder and a simulation result in section IV. The last section is our conclusion.
II. IMPORTANT PARAMETERS IN LDPC ENCODER
The main purpose of an encoder is to add some parity bits to the information bits and to combine both to generate a codeword (combination of information and parity bits). This section describes some important parameters applied in LDPC encoder such as type of LDPC codes, code length, code rate and encoding method.
A. Type of LDPC Code
If H has constant number of j ones in each column, constant number of k ones in each row and zeros elsewhere, this H is denoted as (j, k) regular LDPC codes. If the regularity of H is not satisfied, this H is known as irregular LDPC code.

III. DEVELOPMENT IN LDPC ENCODER
This section describes the recent developments in LDPC encoder. Some of the works target a specific application like DVB-S2 [9], 802.16E worldwide interoperability for microwave access (WiMAX) [10, 11, 12, 13], Chinese digital multimedia broadcasting for terrestrial and handheld (DMBTH) [14] and Chinese Mobile Multimedia Broadcasting standard (CMMB) [15, 16, 17, 18]. Meanwhile, other works proposed a specific type of LDPC, code rate, code length and encoding method [19, 20, 21, 22, 23].
In the beginning, the implementation of LDPC encoder overcomes the obstacle of encoding complexity by applying a specific structure of H to get linear complexity. Examples of the specific structure of H are applied in DVB-S2 standard [9] and 802.16E - WiMAX standard [10]. Most of these codes are irregular LDPC codes.
European Telecommunication Standards Institute (ETSI) for 2nd Generation of Digital Video Broadcasting (DVB-S2) in 2004 [9] implemented irregular LDPC encoder-decoder for DVB-S2 standard that can be applied for various code rates and two code lengths (N = 64800 and N= 16200 bits). Variable code rates are R = ¼, 1/3, ½, 3/5, 2/3, ¾, 4/5, 5/6, 8/9 and 9/10. The highest code rate is R = 0.9. IEEE 802.16E Standard for WiMAX in 2005 [10] implemented an irregular quasi cyclic (QC) LDPC encoder-decoder which can be applied for various

This encoder does not support many values of R and N. The encoder requires more memory than the work in (20) since it uses back substitution. The highest code rate is R = 0.83. Hemesh Yasotharan and Anthony Chan Carusone [13] proposed FPGA of irregular LDPC encoder using Altera Stratix EP1S80F1508C6 for synthesis and implementation. Encoding method uses back substitution to get G from H which requires more memory (3,911,680 bits) than (20). The highest code rate is R = 0.83.
LDPC encoder which is aimed for DMB-TH is given by Xiaojun Zhang et al. in 2009 [14]. This work proposed QC LDPC encoder architecture based on application specific instruction set processor (ASIP) and uses generator matrix for encoding. The ASIP encoder supports 3 different code rates just by different program for the same code length N = 7493. This encoder is also tested by FPGA using XC2V6000. The highest code rate is R = 0.8.
Some works of LDPC encoder which are used for CMMB are Wang Peng and Chen Yong-en in 2008 [15], Xiangran Sun et al. in 2010 [16], Xiangran Sun et al. in 2010 [17] and Zhibin Zeng in 2011 [18].
Wang Peng and Chen Yong-en [15] proposed FPGA regular QC LDPC encoder, Altera Stratix II EP2S90, encoding with LU decomposition of Neal method using two different code rate (R = ½ and R = ¾). This work proposed another method of memory organization for storing and performing computation on sparse matrix. Xiangran Sun et al [16] proposed FPGA of regular QC LDPC encoder using Altera Stratix II EP1S180F102014, encoding method of RichardsonUrbanke using two different code rate (R = ½ and R = ¾). This work proposes a dynamic programming algorithm to reduce encoding complexity. Xiangran Sun et al [17] proposed FPGA of regular QC LDPC encoder using Altera Stratix II EP1S180F102014, encoding method of Richardson-Urbanke using two different code rate (R = ½ and R = ¾). This work proposes a backtracking algorithm to reduce encoding complexity. This works proposed architecture of encoder which is optimized for R = ½. Zhibin Zeng [18] proposed FPGA of regular QC LDPC encoder using Altera Stratix II EP1S180F102014 and encoding based on Neal method using

LU decomposition using two different code rate (R = ½ and R = ¾). This work proposes a dynamic programming algorithm to get approximate triangulation of H as a substitution of Gauss Elimination. The works in [15], [16] and [17] proposed architecture of encoder which is optimized for R = ½.The highest code rate for all of these works is R = 0.75.
Some works of LDPC encoder which target specific type of LDPC, code rate, code length and encoding method are SuChang Chae and Yun-Ok Park in 2003 [19], Dong U. Lee et al. in 2004 [20], Jia-ning Su et al. in 2005 [21], Zhiyong He et al. in 2006 [22], Jianwen Zhang et al. in 2008 [23].
Su-Chang Chae and Yun-Ok Park [19] proposed FPGA implementation of (3, 6) regular encoder with code rate R = ½, code length N = 1800 bit and encoding using LU decomposition of Neal method. The frequency achieved is 30.726 MHz and the throughput is 15 Mbps. This encoder does not support flexible code rate and code length. Dong U. Lee et al. [20] proposed an FPGA-based irregular encoder on Xilinx Virtex-II XCV4000-6 utilizing the encoding method of Richardson-Urbanke. This works proposed an architecture of the encoder which is optimized for R = ½. They proposed an efficient memory organization for storing and performing computation on sparse matrix at the first time. The highest code rate is R = 2/3. This encoder is fixed for one value of R and N even though it was tested for many values of code rate R and code length N.
Jia-ning Su et al. [21] proposed an FPGA implementation of (3, 6) regular LDPC encoder with N = 1536 bits, R = 1/2 and an encoding using LU decomposition of Neal method. This encoder does not support flexible code rate and code length. Zhiyong He et al. [22] proposed an FPGA encoder for (4, k) regular LDPC using Xilinx Virtex-II with a frequency of 100 MHz. This work proposed a construction of H in a triangular plus dual-diagonal form which is classified as quasi-cyclic (QC) LDPC code with specific encoding method. This encoder supports flexible code rate and code length. This encoder gives very high throughput which is 12.8 Gbps for R = ½ and N = 2048. The highest code rate is R = 0.83. Jianwen Zhang et al. [23] proposed FPGA of non-binary 4-ary regular LDPC code using Xilinx Spartan 3S1500, R = 0.437, N = 2040 and encoding method of Richardson-Urbanke. This works also proposes encoder architecture non-binary 4-ary with R = 0.437. The result of inverse sub-matrix T is sparser than greedy algorithm as a classical algorithm in Richardson-Urbanke method. This encoder does not support flexible code rate and code length. The result of gap value g is higher (g = 299) than greedy algorithm (g = 53) for N = 2040.
IV. PROPOSED REGULAR LDPC ENCODER
Based on the above information, we emphasize that no attempts have been made for the implementation of (3, k) regular LDPC encoder with high code rate (R ≥ 0.875) and few works on flexible LDPC encoder which accommodates various code rates and code lengths. Therefore, we propose (3, k) regular LDPC encoder architecture which is suitable for code rate R ≥ 0.875.

In order to build an expected encoder, it is required to get parity check matrix H with specific characteristics. The required characteristics are having a good bit error rate (BER) performance in high code rate R ≥ 0.875, low encoding complexity and no rank deficiency of H which is fulfilled by novel code-construction in [24].
After selecting the matrix H, we need to choose the encoding method for the proposed encoder. In this paper, we select the encoding method of Richardson-Urbanke based on [7]. Therefore, we need to solve (8) and (9) to get the parity bits.
The basic framework of proposed encoder shown in Fig. 1 consists of two steps: software and hardware. The software part prepares the sub-matrices of H (A, B, C, D, T and E) which is needed in the hardware encoding while the hardware part yields the encoded information bit.

V. RESULTS AND DISCUSSION
In order to show that the selected code-construction in [24] has a good BER performance in high code rate R = 0.875, we show a simulation result in terms of BER performance which is given by Fig. 2.
The proposed (3, 24) regular LDPC code in Fig. 2 is compared only with Shannon limit since there are no other published results of (3, k) regular LDPC codes with the same code length and code rate. Shannon limit is defined as the theoretical limit on minimum SNR required for a coded system at a given R to achieve arbitrarily small probability of error [25]. Fig. 2 shows BER performance of the proposed (3, 24) regular LDPC code with N = 9360 and R = 0.875. It is shown that the proposed (3, 24) regular LDPC code could achieve 1.12 dB from Shannon limit at a BER 10-6.

The purpose of dividing the encoding process into different stages is to balance the execution time and the workloads among stages. Division of workloads between stages is built based on the number of non-zero elements in sub-matrices H which is shown in Table I.
In this table, we take an example of code length N = 6000 and code rate R = 0.875. Column 1 of Table I shows the size of sub-matrices of H while column 2 describes the number of non-zero elements in each sub-matrix H.

Column 3 of Table I gives the density of 1s in each submatrix H while column 4 presents the number of non-zero elements in sub-matrices of H used in the encoding process based on (8) and (9). Column 4 of Table I shows that not all sub-matrices are used in the encoding process. The number of non-zero elements in sub-matrices of H influences the number of operations and the number of clock cycles.

The first stage writes the information bits to the buffer. Stage 2 performs only operation matrix A and writes to the buffer. We do not do any other operations since the number of non-zero matrix A is 15120 which is large.
In stage 3, forward substitution is performed on matrix T and the results are written to the buffer. In this stage, no other operations are performed since the number of non-zero matrix T is 2100. In stage 4, the parity p1 and parity p2 are computed. In the last stage, the received codeword is permuted to generate the valid codeword.

VI. CONCLUSION
We have presented the recent developments in LDPC encoders. Some of the works target a specific application while other works propose a specific type of LDPC code, code rate, code length and encoding method.
In this paper, we have shown an overview of the proposed encoder architecture. Division of workloads between stages is built based on the number of non-zero elements in the parity check matrix (H). This architecture is suitable for high code rate (R ≥ 0.875) applications that have more information bits than the parity bits.
