VLSI DESIGN OF AREA-EFFICIENT MEMORY ACCESS ARCHITECTURES FOR QUASI-CYCLIC LDPC CODES
Ming-Der Shieh, Shih-Hao Fang, Shing-Chung Tang, and Der-Wei Yang
Department of Electrical Engineering National Cheng Kung University, Tainan 701, Taiwan
E-mail: shiehm@mail.ncku.edu.tw

ABSTRACT This paper proposes an area-efficient memory
access architecture that merges small memory blocks into memory groups to relax the effect of peripherals in small memory blocks. An efficient algorithm is also presented to handle the additional delay elements. The proposed LDPC decoder has the lowest area complexity among related studies.
I. INTRODUCTION
Low-density parity-check (LDPC) codes, first introduced by Gallager in 1962 [1], are linear block codes with very sparse parity check matrices. For LDPC decoder designs, fully parallel architectures [2] and partially parallel architectures [3]-[5] are two common choices for VLSI implementations. Although the fully parallel architecture can obtain a higher throughput, it may have complex interconnection between the check-node units (CNUs) and variable-node units (VNUs). Recently, most LDPC decoders are realized using partially parallel architectures, which may use specific optimization techniques. For example, overlapped message passing algorithms [3] were proposed for quasi-cyclic LDPC (QC-LDPC) codes to increase the hardware utilization of CNUs and VNUs by choosing good column starting indices (CSI) and row starting indices (RSI). Kang [4] presented techniques to reduce the number of messages to be exchanged between CNUs and VNUs, thereby reducing the interconnection complexity. For practical applications such as the IEEE 802.16e system, the multi-mode decoder design for various codeword lengths have been investigated [5]-[7].
For the partially parallel architecture, internal memory is required to store the extrinsic messages. To increase the parallelism of data access, the internal memory is usually divided into a set of small memory blocks; however, doing so may increase the required memory space. Based on

this observation, this paper presents an efficient method of selecting small memory blocks and then combining them into a larger one to save memory area. An efficient algorithm is used to deal with the memory access conflicts that result from memory merging management.
The rest of this paper is organized as follows. Section II presents the overview of LDPC codes. Then, the proposed area-efficient memory access architecture is described in Section III. Section IV presents an efficient memory merging algorithm. Section V shows the experiment results of the proposed design. Finally, conclusions are given in Section VI.
II. OVERVIEW OF LDPC CODES AND DECODERS
QC-LDPC codes are widely used because their regular code structures are very suitable for hardware implementation. The parity-check matrix H of a (J,K)-regular QC-LDPC code includes J block rows, K block columns, and J×K circulant matrix Ij,k, where 0 ≤ j ≤ J-1 and 0 ≤ k ≤ K-1. Each circulant matrix Ij,k is a Z×Z cyclically shifted identity matrix with an offset or shift value Sj,k. A J×K base matrix Hbase, which defines all the offset values Sj,k, can also be defined, where 0 ≤ Sj,k ≤ Z-1. After that, a parity-check matrix H with a dimension of M×N, where M=J×Z and N=K×Z, can then be defined.
From the H matrix for QC-LDPC, the corresponding partially parallel architecture of QCLDPC decoding, as presented in [3], is illustrated in Fig. 1. It consists of J CNUs and K VNUs together with J×K memory blocks Mj,k, 0 ≤ j ≤ J-1 and 0 ≤ k ≤ K-1, used to store the extrinsic messages. Since the parity-check matrix of QC-LDPC codes is divided into a set of circulant matrices, a straightforward addressing scheme is to map the ith row to the ith location of Mj,k. III. PROPOSED PARTIALLY PARALLEL ARCHITECTURE WITH COMBINED MEMORY BLOCKS A. Block Row Merging Scheme

Assume that the J×K memory blocks are used to store the extrinsic messages and that each Mj,k is associated with Ij,k. A straightforward merging scheme is to combine all the distinct blocks in a block row. Such an arrangement is referred to as the block row merging scheme. Let Gi denote the ith memory group produced by merging a set of small memory blocks. Applying the block row merging scheme, exactly J memory groups are obtained, with each group Gi consisting of all Mi,k for 0≤k≤K-1. Since K extrinsic messages in a row share the same memory address, all K messages are accessed at the same time.
According to the check-node operations, one can see that the K messages accessed from the same row can be immediately processed by a CNU. In contrast, in the variable-node phase, because all the required messages in a column of H are stored in different memory groups that may not be accessed at the same time, buffer units are needed during the operation of VNUs.
The number of required buffers is determined by the life time of messages when they are read from the memory and then consumed in later time stages. A longer life time implies a large number of additional buffers, which in turn decreases the throughput rate in the variable-node phase. As a result, selecting appropriate starting addresses for the memory groups in the variable-node phase is very crucial for reducing the overhead in the buffer requirement.

IV. MEMORY BLOCK SELECTION ALGORITHM
The memory blocks in a memory group selected by the proposed memory block selection (MBS) algorithm may contain circulant matrices from different block rows and the number of selected blocks in a group is not restricted to a fixed number. As a result, the maximum relative
block delay dR(Gi ) of memory group Gi can be greatly decreased. The steps of the proposed MBS algorithm are summarized as follows.
Step 1: Given a set of RSIs and CSIs, calculate the Ω matrix based on the RSIs and CSIs.
Step 2: Sort all the elements in the Ω matrix and store the sorted elements in the vector ω.
Step 3: Set the maximal allowable FIFO length as L and let the index of the current memory group be i=0.
Step 4: Memory block selection: 4a) Choose the smallest SAv(Mj,k) in ω and denote it as S0. 4b) Include all the memory blocks Mj,k with SAv(Mj,k) satisfying S0 ≤ SAv(Mj,k) ≤ S0+L in memory group Gi, and store the selected SAv(Mj,k) in the vector ωi. 4c) Remove all the selected SAv(Mj,k) in Step 4b from ω. 4d) If ω is empty, go to Step 5. Otherwise, let i=i+1 and go back to Step 4a.
Step 5: Memory group refinement: Let max(Gi) and min(Gi) denote as the maximum and minimum values of SAv(Mj,k) in ωi, respectively. Let |G| denote the

number of memory groups and |Gi| be the number of memory blocks in Gi. Define a cost function N(Gi, Gi+1) as:

N(Gi,Gi+1)= dR(Gi ) ×|Gi|+ dR(Gi +1) ×|Gi+1|. (10)
The value of N(Gi, Gi+1) indicates the total number of required delay elements associated with Gi and Gi+1. The sub-steps in Step 5 are as follows. 5a) Initialize i=0. 5b) Let N0 = N(Gi, Gi+1). Compute Nd = N(Di, Di+1), where Di contains the memory blocks in Gi excluding the memory block(s) with max(Gi), i.e., Di = Gi – {Mj,k | SAv(Mj,k) = max(Gi)}, and Di+1 contains the memory blocks in Gi+1 plus the memory block(s) with max(Gi), i.e., Di+1 = Gi+1 ∪ { Mj,k | SAv(Mj,k) = max(Gi)}. 5c) Compute Nu = N(Ui, Ui+1), where Ui = Gi ∪ { Mj,k | SAv(Mj,k) = min(Gi+1)} and Ui+1 = Gi+1 – { Mj,k | SAv(Mj,k) = min(Gi+1)}. 5d) Let Nmin be the minimum value of N0, Nd, and Nu.
(5d1) If Nmin=Nd and dR(Di +1) ≤ L , move the memory block(s) with SAv(Mj,k) = max(Gi) from Gi to Gi+1, move max(Gi) from ωi to ωi+1, let i=i-1 if i≠0, and go back to Step 5b. Otherwise, go to Step 5d2.
(5d2) If Nmin=Nu and dR(Ui ) ≤ L , move the memory block(s) with SAv(Mj,k) = min(Gi+1) from Gi+1 to Gi, let i=i-1 if i≠0, move min(Gi+1) from ωi+1 to ωi, and go back to Step 5b. Otherwise, go to Step 5d3.
(5d3) If i≠|G|-2, let i=i+1 and go back to Step 5b. Otherwise, go to Step 6.
Step 6: Output Ω, Gi, and ωi for 0 ≤ i ≤ |G|-1.

During memory block selection, memory blocks are sorted according to their computed starting addresses and then the sorted blocks are partitioned into groups subject to the constraint L. For memory group refinement, the number of required delay elements is reduced by migrating memory blocks between adjacent groups without violating the constraint L. Therefore, the maximum

relative block delay dR(Gi ) of memory group Gi can be greatly decreased based on the MBS algorithm.

A comparison of the proposed LDPC decoder design with related designs is shown in Table 1. Although the specifications of the designs in Table 1 are very different, the proposed design is expected to have the lowest area complexity when implemented in the same specifications due to the great reductions in gate count and area. In particular, when compared with the design in [5], which has similar specifications as ours, the proposed decoder can achieve about 35% reduction in gate count. Moreover, from the normalized power efficiency (NPE) defined in [8], one can show that the proposed design is more efficient than that in [5]. As a result, the proposed merged memory access architecture also leads to a power-efficient solution. Since additional FIFOs are inserted in our design, the VNU takes (Z+2L) instead of Z cycles to complete the operations in the variable-node phase. For Z=96 and L=4, this results in only 2L/(2Z) = 4% degradation in the overall throughput performance.
V. CONCLUSION
An area-efficient memory access architecture
was presented for QC-LDPC decoders. Smaller

memory blocks are merged into memory groups to
reduce the impact of peripherals inside the
memory. The conflict of multiple message
accesses that results from merging memory blocks
is successfully resolved by adding buffers. An
efficient memory block selection algorithm was
also proposed to minimize the required buffer size.
Experimental results show that the proposed
decoder has a significant reduction in area and is
also power-efficient as compared to related studies.
