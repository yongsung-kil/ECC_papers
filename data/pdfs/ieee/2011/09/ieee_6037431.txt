Abstract—This paper presents a bottom up approach for implementing high throughput, scalable, layered LDPC decoding architecture for multi-standard applications. A generic implementation of fully parallel check node along with a block level Channel Memory organization scheme are two elements of novelty of this work. The proposed decoder IP core is synthesizable for all codes defined by WiMAX (WiFi) standards. Synthesis results are presented based on 130 nm standard cell ASIC technology.
Index Terms—Low Density Parity Check codes, Min Sum, Layered Decoding, Tree way approach, Flexible architectures
I. INTRODUCTION
Present research in the field of Forward Error Correction (FEC) is aimed at finding the best possible error correcting codes which could allow high throughput decoding with efficient VLSI implementation meeting area, power and speed constraints. Low Density Parity Check (LDPC) Codes [1], a class of linear block codes have gained huge attention in wireless communication domain. Near Shannon limit error correcting capabilities, low error floor, affordable complexity and intrinsic parallelism make LDPC codes an eligible candidate for a number of wireless standards [3]. IEEE 802.16e (WiMAX) [2] is becoming a key technical standard for fixed, portable and mobile data networks supporting up to 70 Mbps. IEEE 802.11n (WiFi) [23] standardizes the physical layer interface of WLAN, capable of reaching minimum data rate up to 100 Mbps. Both these standards feature LDPC coding as an optional channel decoding scheme for forward error correction. However, VLSI implementation of LDPC decoders is a challenging task. Initial designs [4] reported severe layout congestion problems due to pseudo-random message exchange between the computing kernels. But structured LDPC codes greatly simplified the interconnect complexity [24]. Code-decoder co-design is the most suitable approach to ease the implementation. This leads to adoption of architecture-aware LDPC codes by almost all upcoming standards.
Traditional parallel architectures provide maximum throughput but result in huge complexity [4], partially (block-level) parallel architectures have sufficient throughput and affordable complexity but limited to specific structured codes only [5], [6], [10], while fully serial architectures have minimum complexity and throughput. State of the art mainly relies on multiprocessor implementations to achieve flexible, high throughput iterative decoding. In [10], a highly flexible LDPC decoder able to

process all specified codes has been proposed, but it consisted of serial check node and interconnection network, which limits the achievable throughput to large extent.
Realizing high throughput decoders (supporting data rates up to few hundred Mbps) either asks for massive parallelism or high clock frequency, resulting in significant area and power overhead. However, parallelism at check node level is an essential step which can bring significant increase in throughput with affordable complexity. In [8] and [9] authors have presented the first such effort using binary tree, however for a fixed check node degree architectures. Given the multitude of check node degrees to be supported by a multi-standard LDPC decoder, in this work the “ Tree-way ” approach is generalized up to check node degree dc = 32, which is sufficient to support a large number of LDPC codes for next generation standards. In addition, a modular Channel Memory organization is presented which supports the parallel access for proposed check node and provides flexibility in terms of code rates, codeword sizes and parallelisms. The proposed architecture is exploited to design a fully parameterized LDPC decoder IP core for WiMAX and WiFi standards.
II. BASICS OF LDPC DECODING
A binary LDPC code is represented by a sparse parity check matrix H with dimensions M × N such that each element hmn {0, 1}. N is the length of code word and M is the number of parity bits. Each row Hi(1 ≤ i ≤ M ) introduces one parity check constraint on input data vector x = {x1, x2, · · · , xn} i.e.
Hi.xT = 0 mod 2
The complete H matrix can best be described by a Tanner graph [13], a graphical representation of the associations between code bits and parity checks. Each row of H matrix corresponds to a Check Node (CN) while each column corresponds to a Variable Node (VN) in Tanner graph. An (N, K) LDPC code is mapped on Tanner Graph such that there are N Variable (or bit) Nodes (VNs) and M = N − K Check Nodes (CNs). An edge eji on the Tanner Graph connects a V Nj with CNi only if the corresponding element hij is a ’1’ in H. An LDPC code is called regular if the node degree (i.e. the number of incoming edges) is constant for all nodes, otherwise irregular. Irregular LDPC codes have better communication performance than regular LDPC codes.

The entire H matrix is composed of specific pattern of blocks with different cyclic shifts, which allows structured decoding and reduces decoder implementation complexity.

IV. CHECK NODE PARALLELISM : A TREE-WAY APPROACH
In Min Sum decoding, out of all LLRs of a CN, only two magnitudes are of interest i.e. minimum and the second minimum. The state of the art for many applications relies on serial check nodes incorporating on the fly calculation of running minimum, second minimum and sign product [16]. The CN has read and write latency of dc clock cycles which prevents the standardized codes from being decoded with layered schedule. The interconnection network is also serial providing one input to all check nodes per clock cycle. Thus, decreasing the check node and interconnection network latencies is mandatory to realize high throughput LDPC decoders. In [8] and [9] authors have proposed such parallel implementation using binary tree, however the architectures were limited to supporting fixed check node degree of 6 and 8 respectively.
Incorporation of generic parallel processing inside the check node for different check node degrees along with related novel mechanism for dataflow control are some of the key contributions of this work whereby a “Tree-way” check node receives all variable to check node messages in parallel and writes back the updated extrinsic information simultaneously to all connected variable nodes. This significantly decreases the number of clock cycles required for check node update and helps in achieving high throughput with smaller parallelism. Fig. 1a is a particular example of this scheme for dc = 8. Each VN i.e. (I1, I2, · · · , I8) is represented as a leaf node and tree is traversed performing min calculation at branch nodes until all VN extrinsic values are derived at the root nodes (e1, e2, · · · , e8). V. TREE WAY PROCESSING ELEMENT ARCHITECTURE
Figure 2a shows the generic data path for proposed “ Treeway ” processing element (magnitude). The αn are the incoming LLR values, while βn are the updated outgoing LLRs values where (n = 1, 2, · · · , dc) and dc is the maximum check node degree of a given irregular H matrix. The Input Reg consists of dc parallel registers to store αn values which are inputs to the DVC and EC(1,2) stages. For compare select unit CSi, {i = 1, 2, · · · , N }, the first input is either the intrinsic value A[x], {x = 1, 3, · · · , dc − 1} or the direct feed back FBD[i] of previous stage. The second input is selected from five possible options i.e. the intrinsic value A[y], {y = 2, 4, · · · , dc}, the feed back FBP[i] of previous stage after permutation by ACS network, the intrinsic values contained in EC1[i] and EC2[i] vectors and infinity. The output of each compare select unit is stored in its corresponding SM memory. There are N = dc/2 CS units and SM memories and size of each memory is equal to number of stages (or Ncc) which depends on dc. The inputs to ACS network come either directly from CS unit, or from SM memory. ACS network is implemented to allow for all possible permutations for a particular degree as given in Table II. An N ×N barrel shifter supports the implementation of all possible permutations for a degree dc. For retrieval of outputs stored in SM memory from previous stages, a simple mechanism is formulated. Output values at each stage are assigned a binary label e.g. label `10´represents DVC stage output, while labels `100´and `1000´represent first two MSC stage outputs and so on. For a given stage, the address to be accessed in SM memory

depends on dc value and can be derived using a relatively simple control based on binary representation of the value dc − 2. For example for dc = 16, corresponding binary representation of dc − 2 = 1110 i.e. 1000 + 100 + 10, thus address for memory access during the shuffle stage corresponds to values at labels of `100´and `10´. To process an Irregular LDPC code the check node is synthesized for maximum degree dc supporting all CN degrees for that code less than and equal to maximum. For example, WiMax 1/2 rate LDPC has minimum and maximum dc of 5 and 7 respectively. So the proposed check node is synthesized for dc = 8 for check node architecture uniformity, supporting multiple trees for dc = 5 and dc = 7. During EC1 and EC2 stages, the inputs to CS1 change with degree. The block `SEL´consists of pipelined multiplexer which assigns the required input values to CS1 during EC1 and EC2 stages.
The state of art parallel check node implementation pre-
sented in [8], [9], [17] are not optimal in terms of complexity, achievable frequency and scalability. In [8], a parallel check
node architecture has been reported which has a complexity of dc(1 + 0.5(log2 dc − 1)) comparators. The authors in [17] reported a parallel check node based on divided group
comparison technique. The reported check node has a huge complexity and critical path timing of three adders and three multiplexers for dc = 12. The overall design could achieve a throughput of 86 Mbps at 125 MHz occupying 4.94 mm2 area on chip using 130 nm technology. In [9], the authors presented a parallel check node incorporating the value reuse property
of Min Sum algorithm. The check node has a complexity of dc + log2(dc) − 2 comparators. The decoder has been designed for non standardized, rate 0.5 (3,6) code.
Proposed generic “Tree-way” processing element is not only
capable of achieving high clock frequency but also results in considerably small area on chip. The proposed check node has a complexity of dc/2 comparators. The sign product block is also based on tree way approach for parallel product of sign

VI. PROPOSED LAYERED DECODING CHECK NODE WITH “ TREE-WAY ” PROCESSING ELEMENT
Layered decoding [11] is compatible with the parallelization of architecture with a limitation that in order to achieve collision free decoding, the H matrix is divided in groups such that within each group the maximum column weight must be equal to one [24]. Key idea behind layered decoding is that there is no processing inside variable node. It just acts as a Channel Memory unit to store the soft output estimate of previous subiteration. A layered data path architecture has been reported in [6] . The check node is serial with latency equal to dc and consists of a FIFO, message RAM and Check node Functional Unit (CFU). To ensure correct information input to CFU, the corresponding edge messages are immediately subtracted from the message RAM and result is passed to CFU.After processing, the same locations in the message RAMs are updated by newly calculated extrinsic messages. VII. BLOCK LEVEL MEMORY ORGANIZATION
As discussed earlier, the state of the art for min sum decoding of LDPC consists of serial check node. The interconnection network is also serial providing one out of dc inputs to all P check nodes per clock cycle, where P is called parallelism. Therefore, dc clock cycles are required before all incoming messages are received by P check nodes in a sub-iteration. To fully exploit the inherent advantage of proposed check node, a fully parallel network is desired which could move dc messages to all P check nodes with minimum number of clock cycles.
In this work a generalized memory architecture has been presented to support parallel access for proposed “ Tree-Way ” check node. The objective is to break a single memory bank to a number of parallel memory banks and store Nb groups (circulants) of VNs, such that all circulants required for the same subiteration are accessed with minimum memory access conflicts. The procedure to obtain the data dependencies between the circulants accessed in same sub-iteration is as follows;
1) For i = 1 to Nb repeat 2) Select a column Ci and highlight all rows corresponding
to non negative entries of column Ci 3) Check all other columns in highlighted rows.

VIII. SUB BLOCK LEVEL MEMORY ORGANIZATION TO
ACHIEVE SCALABILITY
A full mode IP core for WiMAX and WiFi applications must support
• 6 code rates for WiMAX and 4 code rates for WiFi. • 19 codeword sizes (576-2304) for WiMAX and 3 code-
word sizes (648-1944) for WiFi. • Arbitrary values of parallelism (P) to achieve variable
throughput.
To support all expansion factors and parallelisms, enormous flexibility is required for interconnection network. One solution could be to make the width of each memory bank equal to maximum expansion factor (Zmax) for a particular code and use a fully connected (Zmax × Zmax) network able to support all expansion factors and parallelisms less than Zmax. But this is expensive from implementation point of view, since smaller parallelisms yield hardware resources idle most of the time.
Scalable sub block parallel architecture requires assignment of variable nodes in memory such that given any value of Z and P, all P variable nodes are stored in the same memory word irrespective of the circulant shift. This is done by scheduling the execution of parity equations within a sub-matrix such that given an expansion factor Z and parallelism P be an integer factor of Z i.e. Z mod P = 0, two consecutive variable nodes within a same memory word are at a distance of Z/P . This is explained by an example, suppose Z = 32, then all values of P satisfying Z mod P = 0 are 2, 4, 8, 16 and 32. Considering

Rows 1, 9, 17 and 25 are processed in 1st set and require VNs 18, 26, 2 and 10 which are stored at memory location 2 and obtained after rotating it left by 3. X. COMPARISON WITH OTHER RELATED WORKS
Table VII compares the proposed work with some already published decoders. The Table shows for each decoder the main characteristics: implementation technology, supported codes and decoding modes (combination of code lengths and code rates), achieved throughput, quantization, internal parallelism and occupied area. However comparison to other similar implementations of LDPC code decoders is difficult because of the differences in design choices e.g. implementation technology, finite precision arithmetic. To simplify the comparison in Table VII, area of each decoder has been scaled up to 130 nm process with a scaling factor of 2 and 4 respectively for 90 nm and 65 nm processes. A parameter called throughput to area ratio (TAR) [18] defined as TAR=Throughput/Area has also been included in the Table to evaluate the area efficiency of the designed decoder. On the contrary clock frequency has not been scaled as a universal conversion method between different technologies is not available. A new metric named Decoding Efficiency (DE) given as DE = (Throughput × Iter.)/Frequency has been defined to have a better comparison. DE gives the number of decoded bits per clock cycle per iteration. As shown in Table VII, the proposed decoder architecture achieves a better TAR compared to the implementations on WiFi codes. It also outperforms solutions in [20] [26] [6] in terms of TAR when considering WiMAX codes. Only decoder in [19] gives a better TAR than the architecture proposed in this work, however the proposed solution provides a higher throughput and a better DE. In addition, sufficient throughput is achieved for both WiMAX and WiFi standards at the cost of an occupied area which is smaller than required for most of alternative solutions.
XI. CONCLUSION
LDPC codes have been adopted in a number of nextgeneration wireless standards for forward error correction. Implementing a flexible architecture while satisfying area, speed and power metrics is still a challenging task. In this paper we presented to best of our knowledge first implementation of flexible LDPC decoder based on bottom up parallel approach. Fully scalable architecture for LDPC decoding based on proposed generic architecture of parallel check node realization has been presented supporting different codes of WiMAX and WiFi standards.
