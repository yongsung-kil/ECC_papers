Abstract—A min-sum unit with multi-normalized-factors which is used in LDPC decoder is proposed. This min-sum unit uses a multiplier instead of LUT, and this multiplier nearly has the same precision as the LUT. Synthesis results show that, if four normalized factors are needed, the equivalent gate count of the 5 bits multiplier is 27% less than the LUT, and the equivalent gate count of min-sum unit using this multiplier is 3% less than using the LUT. A fix-point LDPC decoding C-module of 1944 bits 802.11n LDPC codes is also designed and the simulation results show that the decoder using the proposed min-sum unit has a comparable decoding performance to the one using LUT in minsum unit.
Keywords-LDPC codes; nomalized min-sum; multiplier
I. INTRODUCTION
Low-density parity check (LDPC) codes, which were first introduced by Gallager in 1960 [1], were neglected for decades because they were too complicated to be implemented. In 1996 LDPC codes were rediscovered by MacKay and Neal [2]. Some years later, a LDPC code that can achieve within 0.04 dB of the Shannon limit was designed [3]. Because of the excellent performance and the development of integrated circuit technology, LDPC codes had widely adopted by many communication systems, such as digital video broadcasting second generation (DVB-S2), wireless local area network (802.11n) and world interoperability for microwave access (802.16e) [4].
Layered decoding [5] is a very efficient iterating scheme, in this scheme, the parity check matrix is viewed as horizontal layers and each layer can represent a component code. In each sub-iteration, both check node and variable node update concurrently. It can reduce the decoding latency by about 50% with no performance loss.
Belief-propagation (BP), or sum-product [2], is the most powerful computation algorithm in LDPC decoding, but in hardware implementation, it will result in huge complexity. Min-sum algorithm [6], with only compare operation in check node process, greatly simplifies the decoding process with acceptable decoding performance loss. Moreover, BP-based algorithm [7-9], which adds normalized factor or offset factor

in the min-sum algorithm, can improve the decoding performance of Min-sum algorithm. The decoding performance of both normalized min-sum and offset min-sum algorithms are analyzed in [10].
For hardware implementation with finite word length, normalized min-sum can be implemented with a look-up table, or as a register shift for normalization factors like 0.5, 0.25, or 0.125 [10]. However, in most wireless communication systems, the channel code (LDPC code) has many code rates and code word length, this will leads to different normalization factors. But when a look-up table is fixed, the normalization factor is fixed, more normalization factors mean more look-up tables, and that will increase the complexity of the LDPC decoder. If just implement the normalized min-sum as a register shift, the decoding performance will be badly affected.
In this paper, we propose a min-sum unit with multinormalized-factors use conventional multiplier [11]. This unit can provide more accurate normalization factors and thus better decoding performance than a register shift and it has lower complexity than a look-up table with a comparable decoding performance.
The rest of this paper is organized as follows. Section II introduces the min-sum and BP-based decoding algorithm and the multiplier. In Section III, the proposed architecture is demonstrated. In Section IV, the simulation and implementation results of the proposed architecture and some analysis are given. Conclusions are presented in Section V.

III. THE PROPOSED NORMALIZED MIN-SUM UNIT
In normalized min-sum algorithm, the normalized factor always between 0.5 and 1, in normalized APP (a posteriori probability) based algorithm, the normalized factor may smaller than 0.5. In this paper, we just discuss the factor in normalized min-sum algorithm.
In 802.11n, LDPC codes have 4 different rates (1/2, 2/3, 3/4, 5/6) and three different word lengths (648 bits, 1296 bits, 1944 bits). Although the normalized factor should vary with different rates and different word lengths to achieve the optimum performance, it is always kept a constant for the sake of simplicity. In this paper, we discuss normalized factors varying with different code rates of the same code word length.
Considering the decoding performance and the implementation complexity, we choose (6:3) as the quantization scheme, of which ‘6’ means the finite word length is 6, and ‘3’ means 3 bits are used for the fractional part of the value to be quantized, thus the left 3 bits are used for the integral part of the value (1 bit for the sign and 2 bits for the magnitude). The decoding performance of this quantization scheme is shown in fig. 6, part IV. So a 5 bit multiplier is needed in the proposed min-sum unit.
The architecture of a 5 bits parallel multiplier is shown in fig. 3. The normalized factor is a fraction number between 0.5 and 1, so the result is smaller than the multiplicand, and the adder’s output is also a 5 bits data. The normalized factor used here is in binary representation. For example, 0.90625 will be expressed as ‘11101’.
The normalized factors of 1944 bits LDPC codes of 802.11n are derived from simulation in [12], and the results are summarized in table I. In the multiplier shown in fig. 3, the approximation values of the four normalized factors are 0.875, 0.8125, 0.75 and 0.75. The operation of the multiplier can be expressed as:
A0.87 ≈ A0.875 = (A >> 1) + (A >> 2) + (A >> 3)
= A - (A >> 3)
A0.8 ≈ A0.8125 = (A >> 1) + (A >> 2) + (A >> 4)
= A - (A >> 3) - (A >> 4)
A0.77/ 0.74 ≈ A0.75 = (A >> 1) + (A >> 2) = A - (A >> 2)

V. CONCLUSION
In this paper, we proposed a min-sum unit with multinormalized-factors which is used in LDPC decoder. A multiplier is used in the min-sum unit. Synthesis results show that the equivalent gate count of the 5 bits multiplier is 27% less than LUT, and the equivalent gate count of min-sum unit using the multiplier is 3% less than using the LUT. We also designed a fix-point LDPC decoding C-module of 1944 bits LDPC code in IEEE 802.11n, and the simulation results show that the decoder using the proposed min-sum or LUT min-sum unit has nearly the same decoding performance.
