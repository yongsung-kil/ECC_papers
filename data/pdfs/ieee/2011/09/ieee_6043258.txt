An Area-Efficient LDPC Decoder For Multi-Standard With Conflict Resolution
Changsheng Zhou, Yunlong Ge, Xubin Chen, Yun Chen and Xiaoyang Zeng State Key Laboratory of ASIC and System, Fudan University No. 825, Zhangheng Road, Shanghai, China
Email: {09212020074,09212020013,09210720054,chenyun,xyzeng}@fudan.edu.cn

Abstract—This paper presents an area efficient decoder architecture that supports both perfectly structured and not perfectly structured LDPC codes. To verify our architecture, an areaefficient LDPC decoder that supports both China Multimedia Mobile Broadcasting (CMMB) and Digital Terrestrial/ Television Multimedia Broadcasting (DTMB) standards is developed. A solution is proposed to avoid memory access conflict problem caused by TDMP algorithm. The main timing schedule is arranged carefully to handle the operations of our solution while avoiding much additional hardware consumption. We also optimize the extrinsic message storing strategy to reduce the memory bits needed. Besides the extrinsic message recover and the accumulate operation are merged together. Based on SMIC 0.13 um standard CMOS process, the core area of the decoder is only 4.75 mm2 and the maximum operating clock frequency is 200 MHz. With 5 iterations, the estimated average power consumption is 48.4 mW at 25 MHz for CMMB and 130.9 mW at 50 MHz for DTMB with 1.2V supply.
Index Terms—LDPC decoder, TDMP, CMMB, DTMB, Conflict Resolution
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes [1], for its high performance, are wildly used in digital broadcast and communication standards. Usually people need different devices to support different standards. However it’s too costly and inconvenient, so one current for VLSI design is that hardware should be reconfigurable to meet different demands in different situation such as [2], [3]. In some occasion such as electronic product for cars, people may want to receive both CMMB [4] and DTMB [5] signals, in this case hardware should be very flexible to support these standards. However there are two kinds of LDPC codes in different standards: 1) Perfectly structured LDPC codes such as QC-LDPC,AA-LDPC codes and so on that exist in DTMB, IEEE 802.16e and other standards.2)Not perfectly structured LDPC codes such as HSLDPC and IRA-LDPC codes that exit in CMMB, DVB-S2 and other standards. In this paper we will present an area-Efficient LDPC decoder that supports both of them.
Distinguished by the flow of message updating operation, there are two major decoding algorithms: 1) The original Two-Phase Message-Passing (TPMP) algorithm, introduced by Gallager [1], passes messages between check nodes and variable nodes alternately. 2) The Turbo-Decoding MessagePassing (TDMP) algorithm, introduced by Mansour [6], also known as the layered decoding algorithm. The extrinsic messages from variable node to check node are used immediately

during each layer in variable node operation. Generally TDMP algorithm has faster convergence and only needs about half the number of iterations with the same performance than TPMP. Furthermore, as explained in [6], TDMP has memory advantage. All these advantages lead to less hardware and power consumption.
Although TDMP is more efficient, but it will come across memory access conflict problem when decoding not perfectly structured LDPC codes. These codes have Double Diagonal Sub Matrix (DDSM) in their parity check matrices. To solve this problem, [7] first introduces group splitting through memory mapping for DVB-S2. Then [8] proposes a solution based on the reordering of the matrices using split process. But [8] can only reduce the number of DDSMs and the level of parallelism is also reduced. [9] uses additional memory addresses to store the two intermediate posteriori messages from DDSMs, then decoder get the correct posteriori message by sub the old posteriori message from sum of those two new posteriori messages in next process. So [9] will need both additional memories and reading clocks. Besides [9] may also need additional accumulators to finish the computation. Our previous work [10] generates an additional message as one of the prior message to calculate posteriori message in DDSMs. It also needs additional accumulators. Based on [9] and [10], this paper will present a solution with normal decoding process and only little additional hardware.
Traditionally, extrinsic messages of the same layer would be read/wrote in one clock and different layer has one unique address. The size of extrinsic messages per layer needed to store depends on the row weight. But row weight and the number of layers differ largely between codes in different rates and standards. If using the traditional extrinsic message storing strategy, the depth and width of memory needed will differ largely in different cases. Commonly the max value will play the decisive role. However it’s too expensive both in hardware and power consumption. In this paper we will optimize the extrinsic message storing strategy.
We don’t do extrinsic message recover separately as commonly do. Extrinsic message recover does nothing but translates a value in true form to its two’s complement. This translation equals to change add or sub operation to each other in the accumulator. Besides the translation may introduce add operation that will need much hardware. In the decoder proposed, we will do the extrinsic messages recover and

The H matrix can be represented by a bipartite Tanner graph [11]. Each column in H matrix represents a variable node and each row represents a check node. Most elements in H matrix are zero but each nonzero element defines a connection between the corresponding variable node and check node in Tanner graph, as shown in Fig.1 (b).

C. Proposed solution

The performance degradation will be unacceptable when there are many DDSMs. This makes TDMP less attractive when decoding not perfectly structured LDPC codes. To solve this problem, [7] first introduces splitting process that bases on reordering of the matrices. Then [8] splits the matrices of DVB-T2 designed with a parallelism of 360 to matrices with a parallelism of 360/S, where S is the number of splits. But [8] can only reduce the number of DDSMs and the level of parallelism is also reduced. [9] uses additional memory addresses to store the two intermediate posteriori messages from DDSMs. When a third processor needs the correct posterior message, the processor will read out the two posteriori messages and carries out an additional computation (12). So [9] will need both additional memories and reading clocks to get the correct posteriori message. Besides [9] may also need additional accumulators to finish the computation (12).

IV. VLSI IMPLEMENT
For tradeoff between area and throughput, the partially parallel architecture is widely used in LDPC decoders. A certain number (usually the number is the parallelism, P) of variable and check nodes are grouped into a cluster for parallel processing. Besides the partially parallel architecture has very high flexibility which is important in VLSI design for multi-standards. In this paper we also adopt the partially parallel architecture and some optimizations are taken to save both hardware and power consumption. The detail of our architecture will be shown in this section.

D. VLSI architecture
As shown in Fig.7, the proposed architecture has about six parts: 1) Central controller; 2) Configurable cyclic-shift network that supports both 127-way and 128-way permutation; 3) 128 Recover and accumulator array that updates prior and posterior messages; 4) 128 NMS array that calculate extrinsic messages; 5) Memories; 6) Parity check and output block that does the parity check work while decoder is iterating and outputs the hard decision bits at last.
There are five memory banks: Two banks, each is made up of eight memories with depth 72 and width 128, are used to store posterior messages. This enables the decoder to receive channel intrinsic messages while decoding. 72 is the max number of layer and 128 is the max parallelism in H matrices in CMMB and DTMB. As the max row weight is 27, eight memories with depth 26 and width 128 is used to store prior messages. As described in Section IV-B, extrinsic messages occupy a bank of eight memories with depth 112 and width 128. The last bank is in the parity check and output block. This bank is composed of three memories with depth 72 and width 128 each. One memory is used to store the hard decision bits from last iteration that used to do the parity check work. The second one stores the hard decision bits from current iteration while the last one stores the output bits. In this way the decoder can do parity check, iteration and output operation at the same time. Besides all the memory used is single-port memory. No two-port memory is used because it’s about 1.5 times larger and dissipates more power.
