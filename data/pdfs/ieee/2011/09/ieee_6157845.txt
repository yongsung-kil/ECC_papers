Performance and processing time enhancement of LDPC decoder using stopping node modified Sum
product algorithm

Nguyen Thi Dieu Linh, Gang Wang, Abdelhafidh Barkana, Georgia Rugumira The School of Electronics and Information Engineering, Communication Research Center,

Abstract: In this paper, a new method based on Sum Product algorithm (SPA) using stopping node to reduce computing complexity on Low Density Parity Check Codes (LDPC) decoding algorithms is presented. We show how to make early decisions to reduce calculation of the next iteration. Simulation results show that performance of iterative decoding for LDPC codes using stopping node algorithm can reduce up to 10 times computational complexity with maintaining quality level.
Key words: Low Density Parity Check Codes, SP algorithm, stopping nodes, iterative decoding.
I. INTRODUCTION
Capacity increasing over Digital Video Broadcasting – Satellite (DVB-S)[1] under the same transmission conditions by 30% is one of the benefits of using LDPC (Low Density Parity Check) when incorporated with adaptive coding modulation (ACM), and variable coding modulation (VCM) techniques. It is optimized by higher order modulation schemes for satellite signals, such as the amplitude phase shift keying (APSK). The applications of LDPC codes are not limited to Digital Video Broadcasting Satellite second Generation (DVB –S2)[1], they are also used in digital communication and adopted by advanced standards such as 10 Gigabit Ethernet(10 GBASE-T)[2],[3], WLAN, WIMAX.
LDPC codes are a class of linear block codes achieving close to the Shannon limit performance among the forward error correction (FEC) systems [4] having the limit quality of channel capacity. Originally invented by Gallager in 1962[5] but due to the complexity of Decoding Algorithms that requires large volume of calculations and because of the technology that was not mature enough for their practical implementation, they were forgotten until the 1990s. codes were reintroduced by Mackay and Neal in 1996 [7]. Then with the development of computing technology in processing and data storage, the problem of LDPC codes had been solved by an iterative decoding algorithm as Message Passing Algorithm (MPA) or so called Belief Propagation Algorithm, that presents high BER (Bit Error Ratio) performance in which the calculations is done in probability domain. To reduce computational complexity by decreasing the number of iterations and achieving better BER performance, Sum-Product Algorithm (SPA) was introduced to present highest BER performance. LDPC codes were efficiently decoded with SPA in log domain using an iterative message passing algorithms.
In order to reduce processing time and complexity many enhancement of SPA algorithms was proposed. In “Sleepping Node Decoder” method presented by Richard Bresnan [8], reached nodes can make early decisions on the next iteration to reduce the number of calculations. Reference [9] and [10] shows the Elimination Node method to make node stops according to Log likelihood. In addition, modifications using Early Stopping (ES) based on Cross Entropy (CE) coefficient [11] or based on Hard Decision Aided (HDA)[12] and follow Sign - Change - Ratio (SCR) for Turbo codes has also been proposed to be applied to LDPC decoder.
In this paper, a modified SPA algorithm for decoding LDPC codes by including Stopping Node Algorithm (SN) is proposed. This method significantly speeds up decoding process, simplify the decoding schema and reduce the numbers of iterations.
This paper is organized as follows: Section II introduces the SP algorithms. Section III describes the proposed SN algorithm. Section IV presents simulation results. Finally, Section V concludes the paper.

III. STOPPING NODE ALGORITHM

With a certain code, the quality achieved depends on the number of iterations performed in the decoding process. When the number of iterations increases, the decoding processing time and the quality increase. But this relationship is not linear. When performing building code in random or structure mode it can not avoid the appearance of finite cycles. Means that when decoding on the graph, nodes intermessages transmission depends on the whole time taken by message depending nodes, but updating information at any node depends on the node itself.
Resulting quality graph of the code will be in regression then error floor appears. So when building codes we avoid the case of short cycles, especially in case of 4 cycles. And we should reduce decoding time and complexity because the gain of coding is not significantly reduced.
The decoding quality of LDPC iterative decoding algorithms having finite length is limited by the short cycles on bilateral Tanner graph. In iterative decoding algorithms, when transmitting massages from bit node to check node and contrary, often error handling appear within short period. Using stopping node method can improve the quality of LDPC decoder iteration by removing the effect of short cycles.
Fig 1 describes the use of stop node to remove the effect of short cycles. In fig 1 the shortest cycle of the code is 4 because of nodes v3, c2, v5 and c4. At certain iterations, if v5 is the stop node, the shortest cycle of the code will increase to 6 by the set of nodes v1, c3, v4, c4, v3 and c1.
Firstly, we set a group of bit nodes as V(s)=[vi(s)] then the hard decision of codeword in the Jth iterations is C(J)=(c1(J),c2(J),…cn(J)) that correspond respectively to bit nodes (v1, v2,...vn).
Definition: The Stopping nodes (SN) are a collection of bit nodes V(s)=[vi(s)] when sign of LLR (Log Likelihood Ratio) corresponding bit codes remain unchanged.
Considering regression decoding algorithms, we determine the SN node by comparing the sign of c1(J) under iterations.

IV. SIMULATION RESULTS

Effectiveness evaluation of the decoding LDPC algorithm using stopping node and stopping early methods can be done through four parameters: Bit error rate, iterations average number, SN rate (the number of SN at the last iteration of the bit node), and running time simulation at different values of bit energy rate on the noise power spectral density Eb/N0.
Assuming that the modulation and demodulation is ideal BPSK, so AWGN channel average is 0 and variance is V2=N0/2 (N0 is the power spectrum density bilateral). LDPC code rate encoding is set by 1/2 for a block size equal to 504.252 as introduced in [13]. According to experience the value of L can be selected to L=7 or L=9. The maximum number of iterations of LDPC decoding algorithm was set to Jmax = 50.
Mathworks MATLAB software installed in a computer equipped with Intel(R) Core(TM) 2 Duo CPU T5670@ 1,80GHz and 1GB physical memory was been used to run simulations of the SPA, SPA-SN algorithms.

V. CONCLUSION
An enhanced and more efficient algorithm for decoding LDPC codes using stopping node method has been proposed. Comparing performances simulated shows that computation time is significantly reduced, nearly 10 times than conventional algorithms, with maintaining processing quality where losses are insignificant and acceptable. The proposed algorithm can save power, reduce transmission delay and processing cost.
