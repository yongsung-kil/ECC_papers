A Programmable IP Core for LDPC Decoder Based on ASIP
Jun Oengl*, Bing Li2 , Lintao Liu3,Rui Chen2 1, Sichuan Institute of Solid-state Circuits, Chongqing, 400060, China 2, School of integrated circuits, Southeast University, Nanjing, 210096, China 3, Science and Technology on Analog Integrated Circuit Laboratory, Chongqing, 400060, China

Abstract: This paper proposes a programmable soft IP core of a LOPC decoder based on ASIP (application­ specific instruction set processor) which can support multi-mode specified in the IEEES02.11n standard. With the presented specific microinstructions based on ASIP architecture, the decoder can process all the codes for the IEEES02.11n standard in a programmable approach, effectively, and due to the proposed 6-stage pipeline, the decoder performance is improved greatly. To verify the design, the IP has been integrated into a embedded processor system based on Xilinx EOK on a Xilinx Virtex5 FPGA component. Finally, the Logic synthesis on O.IS!1ill CMOS technology from UMC reveals a maximum clock frequency of 203MHz and a total area of 3.94mm2, and the corresponding power consumption is below 326.49mW.
I . Introduction
Among all of the known error correcting codes,
low-density parity-check (LOPC) codes, first introduced
by Dr. Gallager in 1960s [I], own the best performance.
Unfortunately, due to the limited VLSI technology, until
it was rediscovered by Dr. Mackay [2] after three decades,
LOPC codes were widely researched and adopted by
several industrial standards for its much closer to the
Shannon limit. With the dramatic advancement of VLSI
technology, the implementation of LOPC codes as the
forward error correcting schemes in the advanced
wireless communication systems is possible. To cope
with the implementation challenges caused by irregular
interconnect, implementations oriented structural LOPC
codes such as quasi-cycle (QC)-LOPC codes start to be
researched by people. Consequently, the LOPC codes
have been adopted by several standards such as IEEE
S02.lIn (WLAN), IEEE S02.16e (WiMAX), OVB-S2.
However, there still remain many challenges including
multi-mode, high complexity, routing congestion, and

high power consumption. So this paper proposes an architecture based on application-specific instruction-set processor (ASIP) to implement the programmable LOPC decoder to counteract the above issues.
The remainder of the paper is organized as follows. Section II describes the LOPC codes and decoding algorithm. In Section III, the proposed architecture based on ASIP and specific microinstructions for IEEES02.11n are extracted. IP verification based on embedded system is dedicates in Section IV. Implementation results are drawn in Section V. Eventually, conclusions are given in Section VI.
II. LOPC codes and decoding algorithm A. Low-density parity-check codes
Low-density Parity-check (LOPC) codes are one kind of the linear block codes, which is defined by a sparse parity check matrix H of M x N. Figure l.(a) shows that the parity check matrix H with M=4 check nodes and N=S variable nodes, the code rate R=(N­ M)/N=4/S= II2. The parity check matrix H can be visualized by a Tanner graph or a factor graph [3] as illustrated in Figure 1.(b), which shows the graphical representation of the associations between code bits and parity checks. Each column of the matrix H can be mapped to a variable node unit (VNU) and each row of H corresponds to a check node unit (CNU), which represents one code bit and a parity check respectively. The edges connect variable nodes and check nodes according to the non-zero elements of IL which represent the mutual information transmission between VNUs and CNUs.

B. Block LDPC codes of IEEE802.11n There are 12 such parity check matrices defined in

These sub-matrices are either cyclic-permutation of the identity matrix or null sub-matrices. The cyclic-permutation matrix Pi is obtained from the Z x Z identity matrix by cyclically shifting the columns to the right by i elements. C. Decoding algorithm A classical decoding algorithm for LDPC codes is
belief propagation (BP) algorithm, which uses iterative massage passing algorithm to update the probability information between variable nodes and check nodes. To reduce computational complexity, the Min-Sum (MS) algorithm approximates the numerically accurate BP algorithm with simple arithmetic operations. Compared with BP algorithm, Layered belief propagation (LBP) algorithm shown in work [4] can reduce the number of iterations required for convergence to more than 50%, at the same time, which also can increase the throughput up

to two times and save memory space significantly. That's because that after the check-node updating is finished for one block row in LBP algorithm, the messages are immediately used to update the variable nodes, whose results are then provided for processing the next row of check nodes. B. Implementation of ASIP architecture

The proposed architecture based on ASIP [5] has

very high flexibility, which can implement different

LDPC codes decoding with different microinstruction

programs. To shorten the executing period of decoder, a

6-stage pipeline, such as fetch, decoding, read/shift,

check node process, variable node update, and inverter

shift/write back, is adopted by the ASIP architecture as

shown in figure 4. The first two pipeline stages are

FETCHand DECODE. Namely, an instruction is fetched

from the program memory and then decoded, which

generates three parameters such as control signals,

address, and shift value. In the R&S (read & shift) stage,

the APP values of the variable nodes in the channel

memory (APP Memory) are addressed. At the same time, this stage is responsible to shift the APP values according to the cyclically shift value generated by the stage of DECODE to realize the correctly mapping to check nodes. The check nodes are updated in the CPE stage. See equation (1), the message from Rcv Memory of the previous iteration needs to be subtracted from the APP value. The check node process unit includes a minimum search as required for the Min-Sum algorithm, see equation (2). The following stage of VPE calculates the new APP values and then the APP values are saturated to 8 bits, as equation (3). After that the new APP values are shifted inverse to the original order then written back to the channel memory (APP Memory) in the stage of ISH&WB.

VI. Conclusion In this paper we presented a novel decoder IP architecture based on ASIP. Due to a novel approach of controlling the decoding flow by microinstruction and a 6-stage pipeline adopted, the design not only simplifies the implementation for decoder but also realizes the programmability and flexibility. Compared with other works, it improves the performance greatly . The proposed IP can decode different mode of LDPC codes through programming the parity-check matrix in IEEE 802.lln
standard.
