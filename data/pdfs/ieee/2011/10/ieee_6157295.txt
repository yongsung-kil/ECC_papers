High efficiency and low power 
multi-rate LDPC decoder design for CMMB 
Jiang xiaobo 
Ii hongyuan 
(School of Electronic and linfonnation Engineering, South China University of Technology, Guangzhou,Guangdong 510641, Chain) 
Abstract: High efficiency and low power LDPC decoder for CMMB that supports two rates has been designed in the 
paper. Layered min-sum algorithm and memory compression technology are adopted to reduce the usage of memory; 
backup memory method is applied to solve memory read/write conflict existing in CMMB LDPC code at a low cost of 
memory resource; operational unit multiplexing which can process 112 bit rate and 3/4 bit rate simultaneously is used so 
that the resource consumption of operational unit is reduced. The LDPC decoder designed in this paper is synthesized in 
the SMCO.18um process. The synthesized result has indicated that the area of designed CMMB LDPC decoder is 
7.6mm2 and its power consumption is 132.8mW. 
Key words: low density parity check (LDPC); China mobile multimedia broadcasting (CMMB); decoder; FPGA 
1. 
Introduction 
The LDPC codes are adopted by more and more 
communication 
standards 
due 
to 
their 
superior 
error-correcting 
performance. 
The 
China 
Mobile 
Multimedia Broadcasting (CMMB) issued by the State 
Administration of Radio Film and TV also adopted 
LDPC codes as channel encoding and decoding scheme. 
The code length of LDPC codes adopted by CMMB is 
relatively long (9216 digits) and the non-zero elements 
of check sub-matrix are in random distribution which 
results in that the decoder needs to use a large number 
of 
memory 
resources 
thus 
causing 
the 
memory 
read/write conflicts and large power consumption. 
These characters put big challenge for the LDPC 
decoder design. 
The CMMB LDPC decoder uses relatively large 
memory which results in large power consumption and 
area of decoder. The literature [1,2,3,4,5,6,7] has made 
hardware realization to LDPC decoder with the method 
of layered algorithm which do not store the node 
information 
of 
intermediate 
variables 
and 
the 
convergence rate is twice of the min-sum algorithm thus 
saving the memory resource to a large extent. The 
literature [8,9] adopt the method of checking memory 
optimization to reduce the design of LDPC decoder. 
The decoding of LDPC codes is an information 
transfer algorithm involving a large amount of memory 
read/write operation. The randomness of check matrix 
has caused complex memory architecture design, serious 
memory read/write conflicts, increased decoding time 
and the uncertainty of decoding time. For instance, the 
layered algorithm is applied to DVB-S2 LDPC decoder 
and the solution is to reduce the degree of parallelism or 
make reasonable arrangement of update order or wait for 
a timepiece to solve the conflict [10,11]. The above 
solutions have increased the complexity of decoder 
design and also caused the uncertainty of decoding time. 
These kinds of conflicts exist in CMMB LDPC and 
there is another problem of data conflict in CMMB 
LDPC[12] 
since 
some 
check 
sub-matrices 
are 
superimposed by unit matrix and its shift matrix, that is, 
the column weight of this sub-matrix is 2. This means 
that four times of read/write operation need to be 
completed with in a timepiece so as to solve the problem 
of read/write conflicts. 
LDPC 
decoding 
is 
computation 
intensive. 
Its 
complexity is more than one order compared with turbo 
code. The most computation intensive part is the update 
algorithm of check node information. How to realize this 
computational process at the cost of the least resources is 
one of challenges of 
LDPC decoder design. The 
literature [13] has provided a classical update unit of 
check node calculating 6 input data with the method of 
making direct comparison of the two input values. The 
check modules of this kind of hardware structure will 
take up a large number of logic and register resources.

The literature [8,14] have offered an optimal information 
update algorithm which only calculate the minimum 
value, the second minimum value and its corresponding 
position of input values. Moreover, the literature [8,14] 
have also given 2 x2 pseudo rank order filter (PROF) 
and check node update module of 8 input data based on 
PROF. 
Due to the low consumption requirement of CMMB 
LDPC, this paper has adopted the method of modified 
layered min-sum algorithm; the decoder designed in the 
paper has further optimized the quantized method and 
reduced the usage of memory which only takes up 
201kbits. The paper proposed a novel memory backup 
architecture composed by two dual-port RAMs to deal 
with the problem of read/write conflicts generated by 
sub-matrix whose column weight is 2. This paper has 
also provided datapath supporting two code rates and has 
gained favorable results. In the end, the paper has 
designed a multi-rate LDPC decoder conforming to 
CMMB standard which has been synthesized in the 
SMCO.18um 
process. 
The 
synthesized 
result 
has 
indicated that the area of designed CMMB LDPC 
decoder is 7.6mm2 and its power consumption is 
132.8mW. 
The structure of this paper is as follows: the second 
part introduces the decoder parameters setting; the third 
part introduces the decoder design; the fourth part 
provides FPGA verification and DC synthesis result; and 
the fmal part draws a conclusion.

3.2 Memory module design 
figure (d) 
The traditional memory method of check node 
information is that every memory cell stores one piece of 
check information. In this design, in order to save 
memory resources, check node information applies the 
memory method of optimized compression bit width data. 
Because the CNU module inputs the minimum value and 
the second minimum value of input information every 
time, each RAM address only needs to store the absolute 
value of the minimum value, the absolute value of the 
second minimum value, the most significant bit of input 
information and the address code of minimum value. 
37.5% 
memory 
has 
been 
saved 
by adopting 
this 
optimized memory method. The check memory module 
of this architecture is made up of a dual-port RAM that is 
2304 in depth and 30bits in width. 
There are 
mainly two 
functions for posterior 
information memory module: the first is to receive the 
soft information of external input in the initialization 
process; the second is to store the increasingly updated 
posterior information in the decoding iteration process. 
But there is a great challenge because the sub-matrices in 
row 9 and column 23 of 112 code rate check matrix are 
superimposed by unit matrix and its shift matrix, that is, 
the column weight of this sub-matrix is 2. To maintain 
continuous operations while decoding, two data must be 
read from posterior memory unit in one coIck for this 
sub-matrix and two data must also be written in the 
posterior memory unit. Therefore a dual-read-port and 
dual-write-port memory is needed to support it; however, 
hardware realization does not hope this kind of memory. 
The same situation occurs in 3/4 code rate check 
matrices for which there are 3 sub-matrices whose 
column weights are 2. 
Therefore, the paper proposed a novel memory 
division structure made up of two dual-port RAMs to 
deal with sub-matrices whose column weight is 2. A 
sub-matrix memory structure whose column weight is 2 
is shown in Figure (e) and its working process is as 
follows: 
Step 1: the two RAMs on the left and right are initialized

3.3 Check node update module 
In the iteration process of modified LDPC layered 
min-sum algorithm, the part with the largest computation 
amount and the greatest complexity is the update 
calculation of check node information [281
. Each check 
node is related to k information nodes therefore the 
update of every check node needs to calculate k 
correlation values as shown in Formula (1), 
It is found that the output values are only the 
minimum value and the second minimum value of all 
input values, that is, when the input value is the 
minimum input value, it will output the second minimum 
value correspondingly; the corresponding output values 
of other input values exclusive of this input value are 
minimum value. If the minimum value, the second 
minimum value and its corresponding position are 
searched in the input data, then all output data can be 
determined. 
Literature [8,14] have provided an optimized 2 x2 
pseudo rank order filter (PROF) and check node update 
module of 8 inputs based on PROF. The function of this 
module can search the minimum value, the second 
minimum value and its corresponding position of the 8 
input data. 
The function of PROF unit is to solve the 
minimum value and the second minimum value; if the 
minimum value is one of the two data input above, then 
the result is 1, or it will be 0 as shown in Figure (t). 
Based on this design, this paper has devised a 1 12 code 
rate check node update module (CNU) unit of 6 input 
data as shown in Figure (g). The comparator and selector 
(C& S) unit is used to compare which of the two input 
data is larger and sequence them. If the larger input value 
is on top, then the output result is 1, or it will be O. 
Moreover, the values still larger are placed at the upper 
port of output position. The fmal output ports of module 
are the minimum value, the second minimum value and 
its relative position and are indicated by ml, m2 and 
index respectively. 
In order to meet the requirement of CMMB LDPC 
decoder, the designed decoder must support two code 
rates simultaneously. From this paper, two II2 code rate 
CNU units of 6 input data plus a PROF and some 
comparators and selectors can compose CNU units 
supporting both II2 code rate and 3/4 code rate as shown 
in Figue (h).
