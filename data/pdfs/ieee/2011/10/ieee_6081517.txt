Abstract—Remarkably low error rates can be obtained using
low-density parity-check (LDPC) codes and their use is now
an increasingly common option in communications standards.
The classic decoders based on belief propagation offer very high
performance but have high implementation complexity. Therefore
over the last decade simpler architectures such as the bit flipping
class of algorithms have been researched. Recently, a number of
adaptive threshold algorithms have been proposed in which at
each iteration a bit is either flipped if it is below an inversion
threshold, or the threshold is lowered. By choosing certain values
of the scaling factor, the scaling can be efficiently implemented
using a small number of shift and add operations. This work
looks at how the scaling factor affects the convergence and error
performance, and investigates an algorithm that switches between
two scaling values based on the syndrome sum. The dual scaling
method maintained the error performance compared to that with
a single scaling but the iteration count was significantly reduced.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were first proposed
by Gallager in 1963 [1] and rediscovered in the mid-1990s by
Mackay and Neal [2]. Near Shannon capacity limits can be
achieved using the soft decision message passing algorithm
or belief propagation algorithm (BPA) [3]. However this
algorithm features very high computational complexity.
A bit flipping algorithm (BFA) has much lower complexity
and provides for a more practical mobile solution when device
complexity and battery power are limited. For the hard deci-
sion BFA relatively simple bit operations are computed and
although there are usually more iterations in a BFA compared
to a BPA, the complexity per iteration is considerably reduced
and the overall complexity is much lower.
This paper is organized as follows. The low complexity
bit-flipping algorithms are discussed in Section II. A review
of the current adaptive threshold techniques and a proposed
dual scaling method are described in Section III. Performance
results are discussed in Section IV and finally conclusions are
drawn in Section V.

III. ADAPTIVE THRESHOLD BIT FLIPPING
Adaptive threshold techniques have recently been pro-
posed for both the BFA [7] and gradient descent algorithm
(GDA) [8] [9].
A. Threshold technique for high column weight codes
With application to the BFA, Cho defined a ’flipping gain’
as the benefit obtained from flipping erroneous bits minus the
loss from flipping correct bits [7]. A fixed number of errors
were then injected into a data frame and the threshold range 𝑝
that provided a flipping gain greater than one, together with an
optimum value for 𝑝was then computed for the regular LDPC
code (4161, 3341). It was shown that choosing the threshold
was critical to achieve fast convergence and an SNR gain of
0.23dB was made over a fixed threshold method when the
number of iterations was limited to 20. However, the code used
featured a large column weight (sixty-five) and the method
cannot be directly applied to codes with a low column weight
(e.g. three) that are investigated in this work.

from the decoder that has the minimum objective function
are passed on to the next iteration. While the complexity is
increased, the large step-size decoder is required for a short

V. CONCLUSION
This paper has investigated the error and convergence per-
formances of an adaptive threshold bit-flipping algorithm. The
performance was shown to depend on the threshold adjustment
step-size and a simple procedure to update the threshold at
each iteration using bit shifting was used. A dual scaling
algorithm was proposed which switches between a coarse
and fine step-size depending on the objective function. The
dual threshold 𝜃= 0.625 / 0.97 decoder performed almost
identically to the single one with fine step-size 𝜃=0.97 but the
iteration count was reduced from 58 to 8 at 6 dB 𝐸𝑏/𝑁0.
Reductions in iteration count were also achieved particularly
at low SNR by halting further processing once a plateau had
been reached in an objective function.
