Reduced-Routing Complexity Decoder for High-Rate QC-LDPC Codes

Yong Niu, Zhenyu Xiao, Depeng Jin, Li Su, and Lieguang Zeng State Key Laboratory on Microwave and Digital Communications
Tsinghua National Laboratory for Information Science and Technology Department of Electronic Engineering, Tsinghua University, Beijing 100084, China Email: niuyongthu@gmail.com, xiaozy06@mails.tsinghua.edu.cn, {jindp, lisu, zenglg}@tsinghua.edu.cn

Abstract—This paper presents a high-throughput and rout- &1 ing complexity reduced decoder for Quasi-Cyclic Low-Density Parity-Check (QC-LDPC) codes in the high rate wireless personal area network applications (WPAN). Our selected code is the rate1/2 QC-LDPC code in IEEE P802.11ad/D1.0. A new decoder architecture is proposed. The message paths between variable nodes and check nodes are constructed by a fixed wire network. Compared to the fully parallel architecture, routing complexity of the new architecture is reduced greatly. The decoder is implemented using FPGA, and a data rate of 1.21Gbps can be 91 achieved in the air.
Index Terms—LDPC decoder, QC-LDPC, WPAN.

I. INTRODUCTION

Low density parity check (LDPC) codes, first proposed by Gallager [1] in 1962, have attracted much attention because of their excellent error correcting performance, inherently parallelism and high throughput potentials. Millimeter wave (mmWave) Wireless Personal Area Networks (WPANs) described by the IEEE 802.11.ad Working Group are considering QC-LDPC codes as the preferred choice for forward error correction (FEC).
The most typical decoding algorithm is the belief propagation algorithm (BP), which is also known as Sum-Product algorithm. Prior literature has demonstrated that LDPC codes with BP decoding provide performance very close to the Shannon limit [2]. Although BP decoding provides excellent performance, it is too complex for hardware implementation. A tradeoff between hardware complexity and error-correction performance can be achieved with another well-known decoding algorithm called Min-Sum algorithm (MS), as shown in [3] and [4].
With the increasing demand for high-data-rate wireless application, an overlapped decoding architecture and a layered decoding architecture [5] become popular for high-throughput LDPC decoding using MS. However, the message paths between variable nodes and check nodes are complex, which usually leads to routing congestion. Besides, the overlapped decoding architecture can only be applied to the codes that have non-overlapped layers. In this paper, a new architecture

II. QC-LDPC CODES FOR HIGH RATE WPAN APPLICATIONS
QC-LDPC codes are described by c × t base matrix Hb with c = M/Z and t = N/Z, where M is the number of parity check equations, N is the code length and Z is the sub-matrix size. In this paper, we exemplify a 672-bit, rate1/2 QC-LDPC code in IEEE P802.11ad/D1.0 [7]. Its Hb is shown in Table I. The 336 × 672 parity check matrix H is extended from 8 × 16 base matrix Hb in which the submatrix size Z is set to 42. An element of 0 in the base matrix Hb can be expanded as a 42 × 42 identity matrix. A blank element can be expanded as a 42 × 42 all zero matrix. An element of a non-zero integer in the base matrix Hb can be expanded as a 42 × 42 permutation matrix representing right cyclic shifts of identity matrix with the number of shift given by the value of the element. The parity check matrix of an LDPC code can be illustrated graphically using a tanner graph [8] shown in Fig. 1. Each bit is represented by a variable node (a circle in the figure and referred to as a VN), and each parity constraint is represented by a check node (a square in the figure and referred to as a CN). An edge exists between a variable node i and a check node j if and only if H(j, i) = 1, and no nodes of the same type can directly connect to each other. Variable and check nodes that share an edge are called neighbors.

V. PROPOSED QC-LDPC DECODER ARCHITECTURE
A. Overall Architecture of the Decoder
The overall architecture of the decoder is shown in Fig. 3. The decoder consists of fourteen Decoding modules, Input control module, Output control module and parallel to serial conversion module (P/S). The received fourteen LLRs are given to the decoder parallelly. Each frame of LLRs is given a number from 1 to 14 with a period of 14, which is controlled into the decoding module that has the same number. For example, frame 1 goes into decoding module 1. By doing this, the number of clocks reserved for each decoding module to decode is 672. The fourteen decoding modules are all the same, and the maximum number of iterations is six. The decoded word of each decoding module is then controlled into the P/S module by Output control module. Finally, the output port outputs fourteen decoded bits parallelly.
B. Decoding Schedule
In each decoding module, the step of CN updating is divided into two halves. In the first half, the first and second minima of the magnitudes and the product of the signs of the received messages from neighboring VNs are calculated. In the second half, the messages from every CN to neighboring VNs are obtained using the results of the first half. C. Data Flow of Decoding Module
The dataflow of each decoding module is shown in Fig. 6. Prior LLRs RAM stores the prior LLR of each VN. NPU is the node processing unit, which is used to update VNs and complete the second half of CN updating. CNFU is used to complete the first half of CN updating. V2C RAM stores the messages from VNs to CNs. Fixed Wire Network constructs the message paths between VNs and neighboring CNs. ORC is a block of registers and stores the results of CNFU for this iteration. NRC has the same structure with ORC and stores the results of CNFU for next iteration. The process of decoding goes as follows.
Firstly, prior LLRs are read from Prior LLRs RAM and loaded into the V2C RAM by NPU to complete the initialization. At the same time, the first half of CN updating is finished and results are stored in NRC. Secondly, the content of NRC is copied to ORC. With the messages from V2C RAM, the prior LLRs from Prior LLRs RAM and the results of CNFU from ORC, VN updating and second half of CN updating for this iteration are finished in NPU to complete an iteration. At the same time, the first half of CN updating for next iteration is completed by CNFU and the results are stored in NRC. Thirdly, the second step is iterated until the maximum number of iterations is reached. All the operations above are divided into 42 parts, and each part is responsible for sixteen VNs.

E. Fixed Wire Network
In this design, a fixed wire network is constructed between VNs and CNs. The CNs which correspond to the rows of a submatrix are called a CN group (CNG). Each VN in a VNG has a neighbor in a CNG, if the submatrix which the VNG and the CNG correspond to is not all zero matrix. The message paths between VNs and CNs of the first submatrix are illustrated in the left half of Fig. 9.
In Fig. 9, a circle stands for a VN and processes the messages from neighboring CNs. A square stands for a CN and processes the messages from neighboring VNs. The line between a VN and a CN suggests they are neighbors. The right half of the figure shows the method we use to construct the message paths. F. CNFU
CNFU consists of sixteen submodules and each submodule is responsible for a VN. Each submodule consists of four ECNFUs, which is in charge of a neighboring CN of the VN. The structure of ECNFU is shown in Fig. 10, where new v2c 1[3 : 0] is the message to the CN. Its first three bits are the magnitude and last bit is the sign. The first and second minima and the product of signs are from NRC, and the newly updated results will be written back to NRC. When all of VNs are processed, every CN has obtained the first and second minima of magnitudes and product of signs of messages from its neighborhood for next iteration.

VII. CONCLUSIONS
In this paper, a new architecture for QC-LDPC codes in high rate WPAN application is proposed. The routing complexity of the decoder is reduced by constructing the message paths using a fixed wire network. Finally, the implement results show that the architecture has high throughput, low interconnect complexity and good decoding performance. The proposed architecture is expected to be incorporated in next-generation high rate WPAN applications.
