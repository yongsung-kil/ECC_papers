A HIGH PARALLEL MACRO BLOCK LEVEL LAYERED LDPC DECODING ARCHITECTURE BASED ON DEDICATED MATRIX REORDERING
Qian Xie, Qian He, Xiao Peng, Ying Cui, Zhixiang Chen, Dajiang Zhou, Satoshi Goto
The Graduate School of Information, Production and Systems, Waseda University Kitakyushu, Fukuoka, Japan
xieqian@akane.waseda.jp

ABSTRACT
This paper presents a high parallel macro block level layered LDPC decoder architecture for the quasi-cyclic low-density parity-check (QC-LDPC) codes with various code rates and code lengths. LDPC codes defined in WiMAX standard with 6 code rates and 19 code lengths are chosen as the demonstration of this architecture. Based on the proposed dedicated matrix reordering strategy, this decoder costs 1224 clock cycles per iteration for different code rates. Compared with the state-of-art work, this decoder reduces total memory bits to a great extent and achieves 2x~4.3x higher parallelism with 1.2x hardware cost. The synthesis result proves the low power potential of this architecture.
Index Terms— LDPC decoder, matrix reordering, high parallel, low power
1. INTRODUCTION
Low-density parity-check (LDPC) codes [1] is receiving tremendous attention and playing an important role in modern error correcting coding (ECC) area because of its near-capacity performance and potential for high-throughput decoding. In the application of LDPC codes, the parallel property with relatively low hardware complexity makes the structured quasi-cyclic LDPC (QC-LDPC) codes be adopted by many wireless communication standards, such as IEEE 802.16e (WiMAX), IEEE 802.11n (WLAN) and IEEE 802.15.3c (WPAN).
Among a variety of decoding algorithms, the conventional two-phase message-passing (TPMP) by Gallager [1], which involves iterative check node updating and variable node updating, achieves near maximum likelihood (ML) decoding performance. However, due to high computation complexity of TPMP algorithm, min-sum (MS) algorithm [2] comes forth as an alternative method, which is very suitable for hardware implementation. Furthermore, the modified MS algorithms such as normalized MS and offset MS using additional correction operations was proposed for practical

applications since they could offer comparable decoding performance with TPMP algorithm. In addition, Mansour in [3] introduced the concept of turbo-decoding messagepassing (TDMP), which is also called layered message passing algorithm, for the architecture-aware LDPC codes. Compared to the TPMP, TDMP can reduce the number of iteration up to 50% without performance degradation.
Currently, improving throughput and lowering power are both of the two trends for the QC-LDPC decoders in the wireless communication systems. In order to realize best tradeoff between throughput and hardware overhead, most of the decoders are implemented with partially-parallel architectures. Unlike previous designs in [4]-[6], this work explores possibility of much higher parallelism of layered decoding process. In the proposed macro block (MB) level layered decoding architecture, 8 sub-blocks which forms one macro block within one layer are processed simultaneously. With the purpose of avoiding the data conflicting problem and improving hardware utilization efficiency, the dedicated parity check matrix reordering strategy is proposed to reduce the average number of clock cycles required in each iteration with no error correcting performance loss. In this scheme, the matrix reordering algorithm is developed for not only LDPC codes of WiMAX standard but also the codes in other standards. Thus, the proposed architecture and matrix reordering strategy are flexible for different QC-LDPC codes defined in various applications. In implementation of this architecture, the synthesis result shows that it achieves 2x~4.x parallelism with 1.2x hardware cost than the state-ofart work. Due to the high parallelism, it greatly reduces the power with much lower operating frequency at the same throughput.
The rest of this paper is organized as follows. Section 2 introduces the QC-LDPC codes in WiMAX standard and our proposed macro block level layered architecture and decoding process. Section 3 presents the matrix reordering strategy and the supporting algorithm in detail. Comparison of simulation results is shown in Section 4. Section 5 concludes the paper.

2.1. QC-LDPC codes in WiMAX

The parity check matrix (PCM) of the QC-LDPC code is expanded from the base matrix Hb. with the process of replacing each non-negative entry in the base matrix with zby-z cyclically shifted identity matrix and each -1 with z-byz zero matrix, where z is called the expansion factor. In WiMAX standard, 6 base matrices for 6 kinds of code rates (1/2, 2/3A, 2/3B, 3/4A, 3/4B, 5/6) are specified, while the expanded factor varies from 24 to 96 with incremental 4 for 19 code lengths. Recently, many works are implemented based on the multirate QC-LDPC decoder architecture which was proposed in [4]. In these works, one z-by-z non-zero sub-block, as the radix of processing engine (PE), is processed each clock cycle. In the same way, a radix-2 PE based architecture was proposed in [5]. In this architecture, two non-zero subblocks are processed simultaneously by two sets of radix-1 PEs. However, the throughput is not doubled because it did not completely solve the data conflicting problem. Moreover, complicated controlling scheme was introduced because in most case the two sub-blocks are not successive in the base matrix.
To further improve the parallelism, the macro-block (MB) level layered decoding architecture is proposed which is capable of supporting all the six code rates and 19 code lengths of LDPC codes in WiMAX standard. Block diagram of proposed decoder architecture is given in Figure 2. Unlike previous works, one macro-block (MB), which consists of 8 successive sub-blocks in horizontal as shown in Figure 1, is processed by the macro-block processing engine (MBPE) every clock cycle.

Decoding process starts from initializing the registers for storing APP messages. APP messages in one non-zero MB are then sent through the permutation network (PN) into the MBPE array. PN contains 7 6-bit 96 u 96 Banyan network based rotators [7] to cyclically shift 8 messages according to the entry of reordered base matrix. Since each layer contains several non-zero MBs, the decoding operation of an entire layer is divided into VAR message updating (VMU) phase and APP message updating (AMU) phase. APP message permutation, VAR message updating and minimum value generation are included in VMU phase, while generating sign of new check (CHK) messages and APP message updating are included in AMU phase. Thus, VMU phase of current layer and AMU of previous layer are overlapped together to improve the throughput.
Due to the large word width and small word number required in our high parallel architecture, all messages are stored in registers. To overcome the shortcoming of increasing power dissipation, the flip-flop based standard cell based memories (SCMs) [8] is utilized for storing check messages and sign. Since VMU phase and AMU phase of every two neighboring layers are overlapped together, two sets of CHK messages needs to be read each clock cycle, which is not supported by the flip-flop based SCMs. As shown in Figure 3, by adding 13 u 96 registers, TMP CHK REGs array is accessed instead of CHK SCM in AMU phase, which makes our architecture suitable to use flip-flop based SCMs.
3. DEDICATED MATRIX REORDERING
3.1. Proposed matrix reordering strategy
For high code rate (rate > 2/3) LDPC codes, the proposed MB level decoding architecture is very efficient due to the big row weight in base matrix. However, for low rate codes, the situation seems not good enough because many zero subblocks exist in non-zero MBs. Processing these zero subblocks worsens the parallelism and finally slows down the decoding. In order to improve the parallelism, reduce cycle

number per iteration and reduce hardware cost, a base matrix reordering strategy is proposed.

3.2. Matrix reordering algorithm

In Figure 4, step 1 plays the most important role because all the following operations are based on the result of this step. In our design, we equally divide the 24 columns of the original parity check matrix into 3 sub-matrices, Ms1, Ms2, and Ms3. Thus, each row of the sub-matrices is a macro block. For each division, supposing Ms1 contains k1 all zero rows where k1 is a non-negative integer. Similarly, Ms2 and

Ms3 contains k2 and k3 zero MBs. The number of zero MBs n is equal to the summation of k1, k2 and k3. The target of this step is to find one of the best divisions which can achieve max (n). Since no classical algorithm can be used to solve this problem and brute force method would cost too much time, an optimal matrix reordering algorithm is developed. For the simplicity of description and practical processing, the conventional matrix is firstly converted to 01 matrix by converting -1 to 0 and converting non-negative entries to 1.

The key idea of our proposed algorithm is to estimate all possible best divisions and test whether it can be achieved. Estimating the group C which is the range of n is the first step. For a certain value in C, integer cutting is introduced to estimate all combinations of {k1, k2, k3}. Since k positions of all columns in one sub-matrix are 0, these columns share the same pattern. Therefore, patterns of three sub-matrices construct a pattern set. For a certain

By exploiting matrices in WiMAX, a property was found that for the best division if the ith row of one sub-matrix is a zero MB, ith rows of other two sub-matrices must be non-zero MBs. Thanks to this property, by applying these three steps one by one, all possible best divisions can be estimated.
Figure 6 shows how our proposed supporting algorithm works. For the clarity of explanation, a 6-by-9 matrix is shown as an example, supposing the size of macro block is 1-by-3. For a certain matrix, max (k1, k2, k3) is a constant integer no matter how it is divided. Greedy division finds one division in greedy way based on this fact. The submatrix which contains max (k1, k2, k3) zero MBs is found first, and another sub-matrix is selected in the similar way from the rest columns. The function of greedy division is to determine upper-bound and lower-bound of C. Besides, the number of combinations can also be reduced with the help of max (k1, k2, k3) in integer cutting step as shown in Figure 6.
As shown in Figure 6, max (k1, k2, k3) is 2; upperbound of C is 6; lower-bound of C is 4. Thus, there’s only one combination of {k1, k2, k3} for integer 6 because at least one of {k1, k2, k3} would exceed 2 in other combinations. Among all pattern sets for integer cutting {2, 2, 2}, one is achievable after check all columns. Finally, one of the best divisions is found and the number of zero MBs is 6.

5. CONCLUSION
In this paper, a macro-block level layered LDPC decoder architecture is proposed. Choosing QC-LDPC codes defined in WiMAX as demonstration, this decoder costs 12-24 clock cycles per iteration for different code rates base on the proposed matrix reordering strategy. Compared with stateof-art work, this decoder largely reduces total memory bits and achieves 2x~4.3x higher parallelism with 1.2x hardware cost. The synthesis result proves the low power property of this architecture.
