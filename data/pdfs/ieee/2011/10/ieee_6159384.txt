Design EG-LDPC Codes for Soft Error Mitigation in Memory

Xiao Li Yi, Zhu Ming, Zhang Yan Jing
Microelectronics Center Harbin Institute of Technology
Harbin, China xiaoly@hit.edu.cn; zhumingbaby@sina.com
Abstract—As the feature sizes of integrated circuits decreasing, single event transient (SET) in combinational circuits can not been ignored any longer. In this paper, a novel fault-secure scheme for memory has been proposed by studying the structural features of Euclidean Geometry-Low Density Parity Check (EGLDPC) codes. The proposed fault-secure scheme can tolerate transient faults both in the storage cell and in the encoder and decoder, using the parallel majority decoding and the feedback loop structure. In order to improve the decoding speed, an algorithm is presented, which can reduce the majority decoding of EG-LDPC codes into two steps. Furthermore, the proposed scheme can suit ordinary data width (e.g., 2n bits) in memory. Finally, the correcting capability and the reliability of the proposed scheme are analyzed. The experiment results reveal that the Mean Time to Failure (MTTF) of the proposed scheme is 419%, 104% and 118% compared with that of Hamming code, Matrix code and Reed-Muller code, respectively.
Keywords-memory; soft error; EG-LDPC codes; fault-secure memory
I. INTRODUCTION
In today’s System-on-Chip (SoC), memories occupy more than 60% of the chip area. As technology scales and the supply voltage decreases, memories are becoming more prone to soft errors which are caused by cosmic radiation and radioactive decay of semiconductor material. These soft errors have some negative effects on the stored information of memories, including single event upset (SEU) and multiple bit upset (MBU). Furthermore, when feature sizes have been reduced into deep submicron (< 0.18 μm), combinational circuits in memories have become more susceptible to radiation induced single event transient (SET). Thus, memory failure is one of the major causes of reliability problems in modern integrated circuits.
Generally, error correction codes (ECC) are effective approaches to mitigate SEU and MBU in memories. Hamming codes are capable of correcting one error and detecting all possible double errors. However, they may not provide adequate protection against MBU. Reed-Muller (RM) codes [1] can deal with multiple errors in memories. But their encoder and decoder are more complex, which cause remarkable hardware overhead. Matrix Codes [2] divide memory word into matrix form. Then, horizontal error codes are added in each row while vertical error codes are added in each column. They

can correct multiple errors with low hardware overhead. However, Matrix Codes cannot deal with SET in memories. Triple modular redundancy (TMR) can be used to mitigate SET in encoder and decoder of memory. The drawbacks of this scheme are multiple area, latency and power consumption overheads.
Nowadays, EG-LDPC codes have been widely applied in communication and storage medium fields. EG-LDPC codes have the characteristic of high coding gain and low undetectable probability. Furthermore, EG-LDPC codes can be decoded by majority logic decoder which can avoid iteration decoding and reduce the correction latency [3]. The authors in [4] exploit EG-LDPC codes to guarantee the fault-secure property of memory which can tolerate MBU in storage cell and SET in encoding and decoding circuits. However, it is targeted at the primal code length in EG-LDPC codes rather than the ordinary data width (e.g., 2n bits) in memory. So the ordinary data bit in memory cannot be directly transmitted using the proposed scheme in [4].
In this paper, a novel fault-secure scheme for memory is proposed, which can suit ordinary data width (e.g., 2n bits) in memory. By using the parallel majority decoding and the feedback loop structure of detector circuits, the proposed faultsecure scheme can tolerate transient faults both in the storage cell and in the encoder and decoder. Furthermore, an improved majority decoding algorithm is presented, which can reduce the majority decoding of EG-LDPC codes into two steps.
This paper is divided into the following sections. In section 2, the general EG-LDPC codes are introduced. In section 3, a novel fault-secure scheme for memory is proposed. Its encoder and decoder are also presented. In section 4, the reliability of the proposed scheme and the circuit performances are analyzed. Finally, some conclusions are presented in section 5.
II. GENERAL EG-LDPC CODES
The construction of EG-LDPC codes bases on the lines and points of the corresponding finite geometries [3]. EG-LDPC codes have a limited number of 1’s in each row and column of the matrix. This limit guarantees low complexity in their associated encoders and decoders, which provides the benefits of fast speed and light weight. EG-LDPC codes are Euclidean Geometry with n points and m lines. It is shown to have the following fundamental structural properties [3]:

The matrix can be converted to a systematic generator matrix using element row operation. In a basic (n, k) EG-LDPC codes, u=(u0, u1,…, uk-1) is the k-bits information vector that will be encoded into an n-bits codeword, v=(v0,v1,…,vn-1). The encoding performs the following vector-matrix multiplication:

v =u⋅G

(2)

where G is a k×n generator matrix. Matrix G can be obtained by converting the parity check matrix H. The encoder can be implemented through XOR logic from equation (2).
EG-LDPC codes can be decoded by majority logic decoder. This method consists of two parts: 1) Generating a specific set of linear sums of the received vector bits. 2) Finding the majority value of the computed linear sums. The majority value indicates the correctness of the code-bit under consideration; if the majority value is 1, the bit is inverted. Otherwise it keeps unchanged. The core of the majority logic decoding is generating γ parity-check sums from the appropriate rows of the parity-check matrix.

III. NOVEL FAULT-SECURE SCHEME FOR MEMORY
In the proposed scheme, when transient faults affect storage cells or encoder, they can be corrected by decoder if the correct-capability of EG-LDPC codes is larger than the number of errors. When transient faults affect decoder, the most of errors can be mitigated by majority logic circuits. Even if a few errors propagate to detector or detector-self suffers from SET, the detector can detect these errors, and then the data is interrupted and repeated. The proposed fault-secure scheme is shown in Fig.1.

Figure 1. The proposed fault-secure memory scheme.
Firstly, K-bits data is fed to the encoder which is implemented by generator matrix G. Secondly, the decoder is implemented by correction matrix W which can be obtained from parity check matrix H according to the improved majority decoding algorithm. Then, the detector is implemented by syndrome S. It can guarantee the fault-secure property of memory. Finally, the corrected data is exported from the decoder.
A. Designing EG-LDPC codes Encoder
The proposed fault-secure scheme can suit ordinary data width (e.g., 2n bits) in memory. Its encoder can be implemented by XOR logic from equation (2). To explain the proposed scheme, we take a (31, 16) EG-LDPC code as an example. The (31, 16) EG-LDPC code has the following properties: 1) 2-bits random errors can be corrected. 2) 4-bits random errors can be detect. 3) it has fault-secure capability. The generator matrix G of EG-LDPC codes can be mapped into specific Galois field (GF) and each generator matrix G is related with specific generator vectors. The (31, 16) EG-LDPC code corresponds to Galois field GF( 2 5 ). Through equation α 31 = 1, six pair of conjugate elements and primitive polynomial 1 + x2 + x5 can be obtained from GF( 2 5 ). For example, the minimal polynomial of element α 1 in GF( 2 5 ) can be expressed by φ(x) = 1+ α 2 + α 5 . The other minimal polynomials can be obtained by a similar analysis process. According to the characteristic of EG-LDPC codes [3], the roots of the generator polynomial in (31, 16) EG-LDPC code can be expressed by α1, α2, α3, α4, α5, α6, α8, α9, α10, α12, α16, α17, α18, α20 and α24. Thus, the generator polynomial of (31, 16) EG-LDPC code can be obtained from the minimal polynomial of each elements, which can be given by

B. Designing EG-LDPC codes Decoder
Let e=(e0, e1,…, en-1) be the error vector and r=(r0, r1,…, rn1) be the receipt vector. After data is exported from the encoder, the equation r = v + e will be satisfied. It is assumed that w is the row vector of parity check matrix and parity check-sums A is the vector product between r and w. The equation (4) can be obtained.

A = w0e0 + w1e1 + ...... + wn−1en−1

(4)

In equation (4), if the coefficient wi=1, the error position ei can be determined by the parity check-sums A. After the row vector of parity check matrix H performs the element row operation, J orthogonal vectors (i.e., w1, w2,…, wj) can be obtained, where J is the minimum distance of codes. These orthogonal vectors satisfy that the column vector αi is all “1” and the other column vectors have only one “1” or all “0”. Then, J orthogonal vectors make up of correction matrix W. The majority logic decoder can be implemented by correction matrix W, which can correct J/2 and detect J-1 errors.
For the different codeword of EG-LDPC codes, the decoder may carry out multi-step majority logic decoding. They will bring tremendous area, power and latency overheads, which is difficult to apply in high speed memories. We offer an improved majority-logic decoding algorithm in this section. This algorithm can limit majority-logic decoding into two steps, which can effectively reduce the overheads of memory. The improved algorithm is described as the following:
(1) Reading in the parity check matrix H n×(n−k ) .

(2) Generating matrix Z which column vector αi and αn-1 are “1”: any M (M ≤ (n − k) / 2 ) row vector in matrix H performs the element row operation to satisfy αi=1 and αn-1=1, where M + 1 ≤ i ≤ n − 2 .
(3) Generating matrix set Di which is orthogonal on αi and αn-1: we make Z matrix cycle J/2 times to obtain the row vectors which satisfy α0~αi-1 and αi+1~αn-2 having only one “1” or all “0”.Then, these row vectors make up of Di. For example, D1J 2×n =(ZiZj…ZkZm)T and D2J 2×n =(ZiZj …ZkZl )T .

V. CONCLUSION
This paper proposes a new fault-secure scheme using EGLDPC codes to assure the reliability of memory in presence of MBU. EG-LDPC codes are designed to suit ordinary bit width in memories. The proposed scheme can tolerate multiple errors in both the storage cells and the encoding and decoding circuits. Furthermore, the proposed majority decoding algorithm can effectively reduce decoding step and hardware overhead. Experiment analysis shows that MTTF of the proposed scheme is 18.9% and 4.5% higher than Matrix codes and RM codes, respectively.
