AN ENCODING SCHEME AND ENCODER ARCHITECTURE FOR RATE-COMPATIBLE QC-LDPC CODES
Ahmed Mahdi, Nikos Kanistras and Vassilis Paliouras∗
Electrical and Computer Engineering Department University of Patras 26500 Patras, Greece
{mahdi, kanistras, paliuras}@ece.upatras.gr

ABSTRACT
We consider the problem of rate-compatible (RC)-encoder and RC-puncturing of LDPC codes. The proposed encoder is based on a modification of MacKay encoding scheme. The introduced modification enables the application of MacKay scheme for quasi-cyclic (QC) LDPC codes combined with a proposed matrix puncturing scheme based on an also proposed parity-check matrix construction to achieve code–rate compatibility. The proposed encoding scheme and VLSI encoder architecture address the problem of encoding complexity, since about 80% of MacKay encoding algorithm complexity is linearly depended on LDPC check node degree. The proposed matrix puncturing scheme can produce good BER performance especially for high puncturing rates, where only a few parity check symbols are transmitted. A comparison with prior art in puncturing is offered, which shows superior performance of the proposed scheme, in terms of coding gain without any hardware cost.
Index Terms— LDPC encoding, quasi-cyclic LDPC, ratecompatible code, hardware architecture.
1. INTRODUCTION
Low-density parity-check (LDPC) [1] codes are considered good candidates for current and next-generation forward error control in high-throughput optical, wireless and recording applications. Their excellent performance and parallelizable decoder make them appropriate for technologies such as DVB-S2, IEEE802.16e [2], and IEEE802.11n [3]. First and foremost, there is a need to reduce complexity without sacrificing performance. Second, for applications such as wireless LAN, the system throughput depends upon the channel conditions and hence the code needs to have the ability to operate at different rates. Third, while the LDPC decoder can operate in linear time, it may be hard to perform low-complexity encoding of these codes. While the encoding time of LDPC codes can be reduced substantially using the techniques presented in [4] at long code word lengths, their techniques may be hard to apply at short code word lengths. The other option is to resort to QCLDPC constructions that can be encoded by shift registers [5] or any permutation network.

Rate adaptability [6] can be realized with several pairs of encoders and decoders for the desired code rates, which is sometimes unacceptable due to complexity. It can also be realized by puncturing a low-rate code (called mother code), resulting in only one encoder and decoder for each mother code. It is assumed that the decoder knows the locations of punctured coded symbols. If the punctured parity bits in a lower rate code are also punctured in a higher rate code, the rate adaptive channel code is called rate compatible [7][8].
A puncturing method based on the degree distributions of irregular low-density parity-check (LDPC) codes, has been presented [7][9][10]. There exist good puncturing distributions that approach channel capacity, over a broad range of code rates, but they concern infinite codeword lengths. These distributions are ratecompatible also. However, in practice, small codeword lengths (of a few thousand bits) are interesting. Rate-compatible puncturing schemes that have been proposed in the past [8][11], choose arbitrarily punctured bits, which may lead to large performance losses, especially when significant amounts of parity bits are punctured. Some subsets of punctured bits may compose stopping sets [12]. In such a case recovering these bits may be infeasible.
A puncturing scheme that provides rate-compatible LDPC code, without violating the degree distribution profiles of mother codes and is based on lower-triangular parity-check matrix codes, has been recently proposed [13]. This scheme is outperformed by the proposed on [14] scheme, where punctured symbols are selected in a systematic way. The basic idea that guides the selection of bits to be puctured is that it is more likely to recover a punctured bit, in case that it has more neighboring (check) nodes and these check nodes have as much as possible reliable (unpunctured) neighboring (variable) nodes. This selection rule provides rate-compatibility and minimization of the performance loss due to the puncturing.
Vellambi and Faramarz in [15] proposed a novel puncturing scheme different from [14] in the following way. It neither uses the information regarding the degree of the surviving (i.e., not removed) check nodes, nor does it search for optimal nodes during the grouping algorithm. Vellambi and Faramanz classify their puncturing scheme as a partitioning scheme that divides the set of bit nodes into groups. However, during the selection of nodes at any stage of the proposed puncturing scheme, the order in which the nodes are selected is unimportant. It is only important that they puncture the bits obtained from the first round before puncturing those of the next round. Therefore, another difference in the approach of [14] and that of [15] is the lack of a sorting algorithm in their scheme. We compare our work with this puncturing scheme.
In this paper, we propose a new matrix puncturing scheme for forward error correction systems, targeting QC-LDPC codes.

Conventional puncturing schemes do not transmit appropriately selected codeword bits, in order to increase the code rate. Decoder assumes these bits unknown and their corresponding LLR is set to zero. This effect introduces additional noise to the received LLR sequence. The more the punctured bits are, the more the noise added due to this effect. In case of high code rates and low channel noise, puncturing noise becomes dominant. This kind of puncturing may result to trapping sets activation under iterative decoding and consequently to decoding performance degradation. Therefore conventional puncturing schemes are inappropriate for major increase of code rate, especially when the error correction system is designed for low channel noise implementations. The proposed matrix puncturing scheme, overcomes this obstacle, by appropriately changing the code structure, in order to support higher code rates, without skipping codeword bits in transmission.
The remainder of the paper is organized as follows. Section 2 reviews an existing encoding method based on forward substitution. Section 3 introduces a modification of this method that renders is appropriate for QC-LDPC and the proposed matrix puncturing scheme that exploits this encoding method, based on a proposed parity-check matrix construction. The implementation of the modified encoding method and its complexity are presented in Sections 4 and 5 respectively. Finally, in Section 6, experimental results indicate the decoding gain achieved by the proposed puncturing scheme, while conclusions are discussed in Section 7.

2. PRELIMINARIES

LDPC codes are linear block codes specified by a sparse parity

check matrix, H. There are two types of LDPC codes: regular and

irregular. Regular LDPC codes are defined by parity check matrices

that have a fixed number of non-zero entries in each row and column;

irregular LDPC codes are defined by parity check matrices having a

variable number of non-zero entries in each row and column.

Parity check matrix H is of size m × n, where m is the number

of parity check bits in the code and n is the length of the code word.

Matrix H is expanded from a base matrix, Hb. The size of Hb is mb × nb, where mb = m/z, nb = n/z and z, integer. Matrix Hb is binary. To define the extension, a matrix Hbc is used, where each zero of Hb corresponds to an −1 element, while each ace of Hb corresponds to a circular–shift factor in Hbc, namely an integer SF ∈ [0, z − 1]. A parity check matrix is obtained by replacing

each −1 in Hbc with a z × z zero matrix, while each nonnegative element is replaced by a z × z permutation matrix. The permutation

matrix is an identity matrix that has been circularly right shifted by

the associated value specified in the corresponding element of Hbc.

Such codes are called QC-LDPC codes. It is noted that the extension

of the base matrix can be done in more than one extension step,

each by a factor zi. In this case, the size of Hb is mb × nb where mb = m/(z1 × z2 × · · · × zV ), nb = n/(z1 × z2 × · · · × zV ) and zi, i = 1, 2, . . . , V , integer [16], [17].

In systematic LDPC codes, the parity check matrix H can be

partitioned into two matrices, H1 and H2 such that H = [H1 H2].

Matrix H1 has size m × k, k = n − m, and corresponds to the

systematic bits. Matrix H2 has size m × m and corresponds to the

3. MODIFIED MACKAY ENCODING SCHEME AND MATRIX CONSTRUCTION
The encoding algorithm proposed by MacKay is based on forward substitution as a first step to compute the first (m − sc) parity symbols P1. This requires that the corresponding sub-matrix H2 or A in Fig. 1 is ALT and the diagonal contains ones only. The above algorithm is applicable for general LDPC codes that satisfy the allones diagonal constraint. However general QC-LDPC codes do not fall into this category as the corresponding H matrix is composed of circulant shifted sub-matrices, and as a result, the diagonal may not have only ones along it. This paper proposes an encoding method that modifies MacKay encoding algorithm in order to accommodate QC-LDPC codes also.

3.1. Modified encoding algorithm
Forward substitution in case of MacKay encoding, is based on two following constraints (henceforth called MacKay encoding constraints),
- all bits bi, i < k are known, when bit bk is computed,
- there is at least one check equation of the code, where bit bk participates, while no bit bi, i > k participates on this particular equation.
In case that sub-matrix A is ALT and its diagonal is composed of aces only, then (4) satisfy the required constraints. Otherwise, the order in which parity bits are computed must be appropriately permuted to satisfy them.

Since the permutation z × z sub-matrix of matrix H that corresponds to the element (i, i + k) of the diagonal of matrix Hb, has only one ace per row, it is ensured that MacKay encoding constraints are simultaneously satisfied for all of the parity bits that correspond to this sub-matrix. The main advantage of this encoding method is that the generator matrix G of the code is not used for encoding. Instead of this, parity check matrix H and the inverse of a sub-matrix of it (C−1) is used. Since parity check matrix of an LDPC code is much less dense than G and if C−1 is small comparing to H, then memory requirements for H and C−1 storage are much less than that for G. Moreover the encoding complexity is reduced, since less operations are required.

3.3. Proposed Parity-Check Matrix Construction

The error correction capabilities of a code under iterative decoding,

relates to characteristics of its corresponding Tanner graph. The

existence of short cycles on the code graph degrades its decoding

performance [21], while code distance relates to the minimum

number of linearly depended columns of its parity check matrix [22].

Although removing columns/rows from the parity check matrix

H of the mother code, cannot insert new cycles to the derivative

graph, it is possible to affect the distance spectrum of the code and

consequently the decoding performance of the derived punctured

code Hl. Consequently, additional concern is required to preserve

good decoding performance of the derivative punctured codes. In

order to achieve this goal, a code construction method is proposed

here, that preserves the minimum distance of the codes that result by

the proposed matrix puncturing.

4. PREPROCESSING AND HARDWARE IMPLEMENTATION

Encoding for QC-LDPC codes can be done achieving linear complexity since operations (multiplication/addition) are executed exploiting the structure of the sparse parity check matrix. The basic operation performed by each of the three stages of the encoder core is row vector-matrix multiplication (rVMM). As A and B are sparse and composed of monomial circulant shifted identity sub-matrices, we divide the row vector into sub-row vectors of size (1 × z), and multiply it with the corresponding nonzero sub-matrix as shown in Fig. 3. Multiplication is executed by shifting the elements of sub-row vector by the shifting factor of the corresponding z × z non zero sub-matrix. This can be done using a logarithmic Barrel Shifter, shift register or any permutation network. 6. DECODING PERFORMANCE AND COMPARISONS
The decoding performance for various code rates is demonstrated by providing BER vs. noise plots. It is shown the proposed puncturing scheme outperforms prior art [15], for a (2016, 1008) mother code that has been constructed according to the method described in Section 3.2. Sub-matrix C of the mother code parity check matrix, is of size 252 × 252. According to (11), the maximum supported code rate by using matrix puncturing, is 0.8. A message-passing algorithm, namely normalized Min-Sum [27], has been used for decoding with a normalization factor of 0.75. Up to ten decoding iterations have been executed by the decoding algorithm, while coded data have been transmitted through an AWGN channel, using BPSK modulation. The simulation results for four code rates, namely 0.6, 0.7, 0.75 and 0.8 are presented in Fig. 6. The decoding gain of the proposed puncturing scheme, is more than 0.8dB, 2dB and 3dB over prior art, in case of code rate 0.6, 0.7 and 0.75 respectively. In case of code rate 0.8 prior art presents error floor, while the distance of the proposed scheme from the mother code rate is less than 0.8dB. Therefore the proposed puncturing scheme shows improved error-correcting performance.

7. CONCLUSIONS
In this work, a novel effective puncturing scheme is proposed for QC-LDPC codes. The effectiveness of the proposed method concerns the encoding complexity and the decoding performance for a variety of supported code rates. In case of proposed scheme, the decoding performance degradation, due to puncturing, is much less than already existing puncturing schemes. The only disadvantage of the proposed scheme is that the constraints imposed on the parity check matrix structure may degrade the decoding performance of the code. This means that the decoding performance of the mother code rate for the proposed puncturing scheme, may be worse than that of an unstructured code of the same code rate. However, the degradation due to puncturing for higher code rates is much less regarding conventional puncturing schemes.
Mother code parity check matrix is constructed in MacKay’s form. While for each code rate we puncture the parity check matrix by removing a certain number of columns and rows, mother code must be efficiently structured to provide good BER performance, especially for high code rates, where a significant number of columns and rows are punctured. While the proposed puncturing scheme is based on a particular H matrix construction, the constraints imposed on H allow for a degree of randomness. The experimental results reveal that the proposed puncturing scheme can efficiently accommodate a substantial range of code rates with minimal impact on the encoder complexity, substantially outperforming current solutions. Further improvement is currently sought by exploiting the degree of flexibility in the construction of H to further improve error correction performance at the mother code rate.
