LDPC Decoding for CMMB Utilizing OpenMP and CUDA Parallelization

Joo-Yul Park
Electronics Computer Engineering Hanyang University Seoul, Korea
jooyul.park@gmail.com

Ki-Seok Chung
Electronics Computer Engineering Hanyang University Seoul, Korea
kchung@hanyang.ac.kr

Abstract— As the 4G mobile communication systems require high transmission rate with reliability, the demand for efficient error correcting code increases. In this paper, a novel LDPC (Low Density Parity Check) decoding method is introduced. We address a parallel software implementation of LDPC decoding for CMMB (China Multimedia Mobile Broadcasting) standard. LDPC codes for CMMB employ a regular H-matrix which has the fixed row and column weights. While effectively utilizing the regularity of the Hmatrix, we process information on H-matrices for multiple code rates using OpenMP pragmas on a multi-core processor and execute the decoding algorithm in parallel using CUDA (Compute Unified Device Architecture) on a GPU (Graphics Processing Unit). We evaluated the performance of the proposed implementation with respect to two different code rates, and verified that the proposed implementation satisfies the bandwidth requirement for CMMB.
Keyword : LDPC, Decoder, Parallel Processing, CUDA, Graphic Processing Unit, CMMB

I.

INTRODUCTION

The LDPC (Low Density Parity Check) code is one of the linear block codes and it was originally devised by Gallager in the 1960’s [1]. However, the code was impossible to implement in hardware in those days. Therefore, people did not pay much attention to the code. About 30 years later, thanks to the development of communication and integrated circuit design technologies, the excellence of the LDPC code was rediscovered by Mackay and Neal [2-5]. Recently S.Y. Chung and T.J. Richardson showed that the LDPC code can approach to the Shannon limit within 0.0045dB [6]. Iterative LDPC decoding schemes based on the Sum-Product algorithm [7] can be fully parallelized, leading to high-speed decoding [8]. Due to these reasons, the LDPC coding is widely regarded as a very attractive coding technique for the high speed 4G wireless communication.

LDPC codes are used in many standards and they support multiple data rates for a standard. Thus, it is very challenging to design a decoder which supports various standards and multiple data rates. Recently, SDR (Software Defined Radio) baseband processing has emerged as a promising technology to provide a cost-effective and flexible alternative to hardware implementation by implementing a wide variety of wireless protocols in software. However, normally the bandwidth

requirement by physical layer processing is very high. Thus, high processing power is required. To achieve high processing power, multi-core processors are often employed in modern embedded communication devices [9,10]. Also, GPU (Graphics Processing Unit) is often adopted to achieve high computational power [11]. Wide deployment of multi-core processors and rapid advances in GPU performance lead to active studies in designing LDPC decoders using GPUs [1216]. They reported that GPU could reduce the decoding time dramatically.
In this paper, we extend this parallelization further in such a way that various standards and code rates can be supported seamlessly. We propose a design which employs both CPU and GPU. To support various code rates, the host multi-core CPU reads the H-matrix and using OpenMP [17] it generates address patterns which are effective when the GPU executes the LDPC decoding in parallel. The LDPC decoding algorithm is written in CUDA (Compute Unified Device Architecture) [18] which is a parallel computing architecture developed by NVIDIA, and the CUDA program is executed by the GPU.
The rest of this paper is organized as follows. In Section II, we review LDPC codes for CMMB (China Multimedia Mobile Broadcasting) and a parallelization technique using CUDA. In Section III, we present the memory structure for CUDA, an address generation method for LDPC decoding and existing parallelization techniques. In Section IV, the proposed software implementation and performance evaluation results will be presented. Section V will conclude this paper.
II. BACKGROUND
A. Parallelization of LDPC decoding
An LDPC decoding algorithm is capable of correcting errors by repeatedly computing and exchanging messages. A good overview of the LDPC decoding is presented in [19]. The amount of computation depends on the size of the H-matrix. In case of relatively new standards such as DVB-T2 [20] and CMMB [21], the size of the H-matrix is huge. Due to its huge size, both decoding complexity and decoding time increase. Therefore, it is crucial to distribute the computation load evenly to multiple cores, and parallelize the computation efficiently. The LDPC decoding consists of four types of operations: initialization, check node update, bit node update,

Since the parity check matrix H for CMMB standard is a kind of HS(High-Structured)-LDPC, approaches for QC(Quasi-Cyclic)-LDPC which is used for DVB-S2 standard cannot be used for CMMB. Especially, LDPC codes for the CMMB standard are regular, which means the H-matrix has the fixed column and row weights. The H-matrix for the 1/2 code rate has patterns that are shifted by 36 bits for every 18 rows. In case of the 3/4 code rate, patterns are shifted by 36 bits for every 9 rows [23].

III. PROPOSED LDPC DECODER
As aforementioned, as the size of the H-matrix increases, the amount of computation grows rapidly. Hence, satisfactory performance is hard to come by in software implementation. On the other hand, it may take long design time to implement in hardware when the decoder supports various standards and data rates. Therefore we propose a novel parallel software implementation of LDPC decoding algorithms based on OpenMP and CUDA programming. We shall show that the proposed implementation is a cost-effective and flexible LDPC decoder which satisfies the throughput requirement for various H-matrices and multiple code rates. First, we will show the overall software structure, and then we will explain parallelization techniques that we propose.
A. Architecture of the proposed LDPC decoder Figure 2 shows the overall architecture. The overall
structure of the proposed LDPC decoder is as follows. We assume that the target platform consists of a single host multicore processor which will run C codes with OpenMP pragmas and a GPU which can run CUDA codes. To support multiple standards and data rates, multiple H-matrices are stored as files. The host CPU reads the H-matrix for a given standard and a SNR constraint. The host CPU then generates an address table which contains the addresses of data processed in parallel by

When signals are received, the host CPU delivers them to the GPU. The GPU executes the proposed LDPC decoding software in parallel. Upon completion of the decoding, decoded bits are transferred to the host CPU. A CUDA API called “CUDA Copy” is used to exchange data between the host and the GPU. The copy overhead may be significant. Thus, it is very crucial to minimize the copy overhead. It should be noted that in our implementation this copy operation takes place only for generated address transfers, received signal transfers, decoded bit transfers, and configuration (standard, code rates, etc.) changes. Therefore, the overhead due to copy operations is not significant in our implementation.
B. 1-dimensional address generation for parallelization
We will explain how we generate addresses for CUDA parallelization using the H-matrix in Figure 3. The H-matrix is stored in a file as a 2-dimensional array which contains bit node positions which are necessary for the check node update operation. The first table in Figure 4 shows an example. Since the position of the LLR value of Bit Node 1 for Check Node 0 and that for Check Node 1 are different, the bit node order is determined by reading the H-matrix information. We minimize the execution time for this operation by parallelizing the operation using OpenMP. If we use an Intel Quad-Core processor, the following algorithm with four threads may be used.
The position of an LLR value is stored in the form of (x,y) where x is the position of a bit node and y indicates that it is the (y+1)th 1 (0ydegree-1) of the same bit node. To make it more convenient to parallelize the execution and reduce the memory access times, this (x,y) information is rearranged as a 1dimentional array as shown in the bottom of Figure 4. The position of the LLR value for (x,y) in the 1-dimensional array which is the address for check node computation is easily computed as follows:

where Wb is the degree of bit nodes. By using this method, when CUDA parallelizes the decoding process, the position of LLR values is obtained by reading a memory instead of computing a new address. Therefore, the execution time is improved. Figure 5 shows the positions of check nodes which are necessary for bit node update operations. C. Parallel LDPC decoding by GPU LDPC decoding consists of four parts as shown in Figure 6.
The first part is that received LLR values are copied into the location of 1’s in the H-matrix. Then, check node update operations and bit node update operations are carried out. Lastly, parity check operations are conducted.
The first initialization step is carried out with pre-generated Zaddr. When the number of signals received is the same as the number of bit nodes, each received value is copied into the position indicated by Zaddr. This task for the H-matrix in Figure 3 can be processed in parallel using 8 threads if we use the GPU in Figure 1.
Second, a check node update operation is conducted after generating as many threads as the number of check nodes. Each thread sequentially reads values from the memory as many times as the degree of check nodes from the memory, and updates the value and stores it back to the same location as the value was read from.
Third, the bit node update is conducted after generating as many threads as the number of bit nodes. The stored data in memory are arranged in such a way that a check node update operation can be effectively carried out. Therefore, for bit node updates, each thread reads as many values as the degree of bit nodes using Zaddr. Using the input values, bit node updates and determination of a decode bit are conducted. Updated values are stored back to the same location as they were read from.
Last, parity check operations are carried out in parallel for each check node. A parity check operation is supposed to check that all the checking results are 0. The addresses in Laddr are used for parity check operations. When an address from Laddr is divided by the degree of the bit node, we can obtain the position of decoded bits for parity check operations.

Performance evaluation results of the proposed LDPC decoding implementation are presented in this section. Hmatrices for the CMMB standard which is currently used in DMB (Digital Multimedia Broadcasting) in China are used for performance evaluation. The length of the codeword in the CMMB standard is 9216 bits and two code rates of 1/2 and 3/4 are supported [21].
The target platform is composed of Intel i5 750 (a Quad-Core CPU with 2.6GHz) as the host CPU, 4GB of DDR3 RAM, Windows XP Service Pack 3 (32bit). MS Visual Studio 2005 was used as the C compiler. The GPU consists of NVIDIA GT 8800, 512 MB of memory, and CUDA v2.3.
To evaluate the performance of the proposed implementation, we compared the performances (1) when no parallelization technique is applied, (2) when only parallelization utilizing OpenMP is applied and (3) when both OpenMP and CUDA were utilized. Reported times are the latencies to take decoding 10,000 frames for each SNR value.
From Table II, we see that when both OpenMP and CUDA are utilized, we achieved a speedup of 22 over the case where no parallelization is applied. When the iteration count increases with low SNR values, the speedup becomes bigger. The effective SNR should be greater than 1.5dB for CMMB. Hence the iteration count is typically greater than 20. When we iterate 20 times, the decoding performance is 2.046Mbps which satisfies CMMB standard.
V. CONCLUSION
In this paper, we address a parallel software implementation of LDPC decoding algorithms. In the proposed implementation we use both a multi-core processor and a GPU to achieve both flexibility and high performance. Experimental results show that the parallel software implementation of LDPC algorithms satisfies the performance requirement for CMMB.
