Area efficient LDPC decoder design for paraUel layered decoding

Yuan Yao, Fan Ye, and Junyan Ren

State Key Laboratory of ASIC and System, Fudan University, 826 Zhangheng Road, Shanghai 201203, China Email: fanye@fudan.edu.cn

Abstract-An area efficient LDPC decoder hardware design for parallel layered decoding algorithm is proposed. Shift register chain is used to reduce the chip area. Puncturing technique is employed to produce arbitrary rate between 112 and I. This design is implemented based on rate-112 LDPC in 802.l6e with 65nm CMOS. The decoder achieves a throughput of 1.2 Gb/s at 10 iterations with an area of 1.14mm2 and support any rate between 112 and 1.

Index Terms-LDPC decoder, quasi-cyclic codes, parallel layered decoding algorithm, shift register chain, 802.16e

The details can be referred to [2] . In LDA the parity

check matrix of LDPC code is divided into several

horizontal layers. In each iteration, the matrix are

processed layer by layer. Equations (1), (2) and (3) are

performed in CNU and VNU with each layer, which leads

to a long critical path. LDA reduce the number of iterations

by sending A to process a layer until this value has finished

previous layer's decoding and been updated. If pipeline

registers are inserted into the critical path, A may be sent to

process a layer before it has finished previous layer's

decoding. It worsens the BER performance. So the critical

path is difficult to be split by pipeline registers. The clock

speed and throughput are limited by it. Parallel layered

decoding algorithm (PLDA) proposed in [3] and [4] for

quasi cyclic (QC) LDPC decoder design process layers in

parallel. PLDA combines CNU and VUN into a single

check node. During each iteration, the matrix is processed

part by part. Check nodes read A from storage units and

update A by performing equations (1), (2) and (3). Then check nodes send new_A back to storage units. The check node is the critical path. In PLDA several pipeline registers can be inserted into the check node, while A will not be sent to a check node before it has finished previous iteration step and been updated. The clock speed and throughput are improved without worsening BER.
In [3] and [4] , a decoder hardware design for PLDA is also proposed. In this design, route paths between check nodes and RAMs where A is stored are fixed and simplified. But in LDA route paths between CNUs and VNUs are complicated. So in this design chip area can be reduced. This decoder hardware design's implementation result is compared with several other decoders for the same LDPC code in [3] and [4] . The comparison results prove that this decoder hardware design achieves a high throughput with a smaller area. But this decoder hardware design for PLDA still has problems. In this design, each A is stored repetitively in several RAM units. Compared with LDA, several-fold of RAMs is required to store A. The read/write control circuits of these RAMs and the circuits to export A which have finished decoding are also complicated. A new decoder hardware design for PLDA is proposed in this paper. Route paths between check nodes and storage units where A is stored are also fixed and simplified. But each A is stored once in registers, while decoder hardware design for PLDA in [3] and [4] stores it repetitively in several RAM units. This modification significantly reduces the chip area for PLDA's hardware design.

of the submatrix is 96x96. This matrix contains twelve

layers and each layer has 24 submatrixes. A non-zero

submatrix denoted by a marked value S is obtained by

circularly shifting an identity matrix by S towards right.

In PLDA, CNU and VUN are combined into a single

check node. The matrix shown in figure 1 is divided into

96 parts. The first part is the first row of each layer. The

second part is the second row of each layer and so on. Each

iteration is divided into 96 steps. Each step processes one

part of the matrix. The 96 parts of the matrix are processed

one by one from the first part to the 96th part. Each clock

cycle a new step of the 96 steps in an iteration is started. In

each step, check nodes read A from storage units and

update A by performing equations (1), (2) and (3). Then

check nodes send new_A back to storage units. Each part of

the matrix has 12 rows. So 12 check nodes are used.

As mentioned in section 1, LDA reduce the number of

iterations by sending A to process a layer until this value

has finished previous layer's decoding and been updated.

So it is difficult to insert pipeline registers into the critical

path. In PLDA the original LDPC matrix is modified. So

several pipeline registers can be inserted into check node,

while A will not be sent to a check node before it has

finished previous iteration step and been updated. The

modified matrix for the rate-II2 LDPC matrix in 802.I6e

can be referred to [3] and is shown in Figure 2. In each

layer, offset are added to the marked value of the non-zero

submatrixes. In other words, in each layer these non-zero

submatrixes circularly shift towards right by the offset

value. The offset values are carefully selected. In the same

column between any two layers, the difference of marked

values of the non-zero submatrixes is at least 5. So in

PLDA if a A is sent to a check node, it will not be sent to

other check nodes within 5 steps of the 96 steps in an

iteration. So four sets of pipeline registers can be inserted

into the check node, while A will not be sent to a check

node before it has finished previous iteration step and been

updated. The clock speed and throughput are improved.

3. Proposed decoder hardware design for PLDA Figure 3 shows the proposed decoder hardware design
for PLDA. The offset-modified LDPC code shown in figure 2 is used to construct the decoder. Values of A is stored in shift register chains. Two chains are used to form a Ping-Pong structure. When one chain is receiving data and exporting the data finished decoding, the other one

performs the decoding procedure. Each chain contains 2304 units to store 2304 values of A contained in each frame. If M bits are used to represent a A, M registers are required to construct one unit. While performing the decoding process, the chain is divided into 24 subchains corresponding to 24 columns of submatrixes in the matrix. Each subchain contains 96 values of A corresponding to one column of submatrixes in the matrix. In a step of the 96 steps in an iteration, each check node reads 6 or 7 values of A from 6 or 7 subchains. The check node updates A and sends new A back to these subchains.
Figure 3. Proposed decoder hardware design for PLDA Figure 4 shows a subchain with a corresponding submatrix. Each clock cycle some values of A, whose locations in subchains are denoted by a certain part of the 96 parts of the matrix, are sent to check nodes to begin a step of the 96 steps in an iteration. In two consecutive steps, continuous rows of each layer are processed. All the non-zero submatrix are obtained by circularly shifting an identity matrix towards right. So compared with previous step, in current step the locations from which A are sent circularly shift towards right by 1 unit in the subchains. Each subchain contains 96 values of A stored in 96 units. In proposed decoder hardware design, all subchains circularly shift towards left by one unit every clock cycle. So in subchains, locations from which A are sent are fixed.

In proposed decoder hardware design, the route paths between check nodes and subchains are fixed and simplified. But in LDA route paths between CNUs and VNUs are complicated. So chip area can be reduced. These fixed and simplified route paths also shorten the critical path causing by performing equations (1), (2), (3) and improve the clock speed further.
Combining CNU and VUN into a single check node in LDA, PLDA requires fewer check nodes and more clock cycles than LDA to perform an iteration with the offset modified LDPC code shown in figure 2. But in proposed decoder hardware design for PLDA with this LDPC code, clock speed is improved significantly by simplified route paths and inserting pipeline registers into critical path. Chip area is reduced greatly by using fewer check nodes

and simplified route paths. So proposed decoder hardware design for PLDA with this LDPC code is area efficient.
In [3] and [4]'s decoder hardware design for PLDA, each A is stored repetitively in several RAM units. The read/write control circuits of these RAMs and the circuits to export A which have finished decoding are also complicated. In the chip's layout view of [3], TSMC 65-nm and the offset-modified LDPC code shown in figure 2 are used. Six bits are used to represent A and r. With an initial logic density of 0.7, the area to place and route the RAMs where A is stored, the read/write control circuits of these RAMs, the circuits to export A and 12 check nodes is around 1.2 mm2 with 8 metal layers. These circuits' function are realized by two shift register chains and 12 check nodes in hardware design for PLDA proposed in this paper. Six bits are used to represent A and r. Using TSMC 65-nm and the offset-modified LDPC shown in figure 2, the area to place and route two shift register chains and 12 check nodes is 0.44 mm2 with 8 metal layers and an initial logic density of 0.7. So proposed decoder hardware design greatly reduces the chip area for PLDA's hardware design.
4. Puncturing technique In section 3, proposed decoder hardware design support
rate-l12 LDPC code. By using puncturing technique, any rate between 112 and 1 can be supported. The message is encoded by the offset-modified LDPC code shown in figure 2. Then some of the parity bits are punctured to produce rates above 112. Before decoding the encoded and punctured message by proposed decoder hardware design, the punctured parity bits will be patched to be O. A puncturing scheme, which generates arbitrary rate between 112 and 1 and uses offset modified LDPC shown in figure 2 as mother code, is proposed in [3]. To test BER of this puncturing scheme, rates 3/5, 2/3, 3/4 and 5/6 are generated using this puncturing scheme and the offset modified LDPC code shown in figure 2 is used as a mother code. These four rates and their mother code are decoded by proposed decoder hardware design at lO iterations. LDPC codes rate at 2/3, 3/4 and 5/6 are selected from 802.16e. At each selected rate, selected LDPC code length is equal to the punctured LDPC code with the same rate. These three selected codes and the original rate-l12 LDPC code in 802.16e are decoded by LDA at 10 iterations. All these floating point simulation results are shown in Figure 6. It shows that BER of proposed decoder hardware design is comparable with LDA. BER of punctured code is close to the selected code with about 0.1 dB loss. BER of proposed decoder hardware design with fixed point is also shown in Figure 6. Six bits are used to represent A and r. 5. Implementation results and conclusions Proposed decoder hardware design for PLDA based on
the offset-modified LDPC code shown in figure 2 is implemented in TSMC 65-nm with 8 metal layers. The puncturing scheme mentioned in section 4 is used. The puncturing recover circuit, which patches the punctured parity bits to be 0, is implemented together with this design. Fixed point scheme mentioned in section 4 is employed.

Core area for placing and routing is 1.14 mm2 with the initial logic density of 0.7, including about 0.25--0.3 mm2 to place and route the puncturing recover circuit. The maximum frequency is 1 GHz, which corresponds to a throughput of 1.2 Gb/s for all code rates at 10 iterations. The power consumption, which is estimated by Synopsis Prime Power, is 613 mW with a IGHz clock speed. This implementation is compared with several other LDPC decoders for 802.16e as listed in Table 1. All the decoders are scaled to 65-nm technology. The metric called the throughput to area ratio (TAR) is calculated to show how much throughput a decoder can achieve per area unit. Both decoder hardware designs proposed in this paper and [3] use the offset-modified LDPC shown in figure 2. They are based on PLDA and use the same fixed point scheme and puncturing scheme. So they have the same BER. But compared with [3], proposed decoder hardware design in this paper reduces the chip area by 40% and the power by 30% with a comparable throughput in TSMC 65nm.Table 1 shows that this proposed decoder hardware design achieves a higher throughput using a smaller area and support any rate between 112 and 1.
