Abstractâ€”This paper proposes a novel quasi-cyclic low-density parity-check convolutional code, and a two-stage construction algorithm with modified progressive edge growth (PEG) method is provided. We propose both encoder and decoder implementation architecture for this code. The quasi-cyclic form provides the parallelism for encoder and decoder, which can increase the throughput and decrease the delay significantly. The proposed modified min-sum decoding algorithm can speed up the process of convergence and reduce the hardware complexity. We also designed a GPU based simulation platform to speed up about 200 times against CPU to verify the code performance. Simulation results show the proposed code can get 0.5âˆ¼1dB coding gain and lower error floor compared with the LDPC codes in WiMAX standard with the same code length, while the decoder only has 20 iterators.
I. INTRODUCTION
Low-density parity-check block code (LDPC-BC) is first proposed by Gallager [1], which with soft decoding algorithms on Tanner Graph [2] can achieve outstanding capacity and approach Shannon limit over noisy channels at moderate decoding complexity, thus becomes a hot topic in channel coding. Density evolution [3], regarded as one performance analysis tool for LDPC-BC, can offer degree distribution of the row and column weight in parity-check matrix. The progressive edge-growth (PEG) algorithm [4] is a girth-aware construction method that tries to avoid cycles in Tanner Graph. QuasiCyclic LDPC-BC (QC-LDPC-BC) [5] [6] [7] can be applied feasibly due to simple parallel codeword structure. While, the large code length LDPC-BC owns, which may cause relatively great transmission delay for the whole communication system, limits the application for real-time service.
To overcome the shortage of LDPC-BC, LDPC convolutional code (LDPC-CC) [8] is put forward, which can be regarded as the combination of the LDPC-BC parity-check matrix and traditional convolutional code. LDPC-CC owns better decoding performance than LDPC-BC [9] under the assumption of the density evolution, and inherits the realtime encoding and decoding characteristic from convolutional code, so LDPC-CC should have wide application field based on its unique features. While the present decoding algorithm demanding too many iterators [8] causes the increase of output delay and implementation complexity. Besides, the related references [10] [9] neglect the feasibility of encoder and decoder when constructing LDPC-CC codeword, which leads to low net throughput and high hardware cost for the whole

system. In this paper, a novel quasi-cyclic low-density parity-check
convolutional code (QC-LDPC-CC) codeword construction algorithm, where quasi-cyclic is induced as parallel structure into LDPC-CC, is proposed and we also put forward corresponding encoder and decoder architecture as well as a simulation platform based on Graphic Processing Unit (GPU). The QC-LDPC-CC codeword constructor can provide good codes for any code rate R = b/c(b < c), while usual LDPC code rate is R = (c âˆ’ 1)/c. Our construction algorithm takes both the codeword performance and the hardware feasibility into consideration during encoding and decoding. The encoder architecture has very low hardware cost and the decoder can get quick convergence, and the whole system can achieve much more transmission efficiency due to the parallel structure induced. The BER performance of the generated codewords, especially the error floor, is evaluated by the high-speed GPU based simulation platform, whose real-time net throughput can reach 200 times of that on the CPU based platform.
The remainder of this paper is organized as follows. Section II presents the background of our research. Sections III, IV, V introduce the QC-LDPC-CC codeword construction algorithm, the encoder and decoder architecture, respectively. Section VI shows the GPU based simulation platform. Simulation results are provided in section VII and conclusion is drawn in section VIII.

B. The BP Algorithm and Effect of Cycle in LDPC-BC
The BP algorithm for LDPC-BC is a general soft decoding scheme for codes described by Tanner Graph, which is a time-scale iteration method. It can be viewed as the process of iterative message exchange between bit nodes and check nodes. For each iteration, each bit node or check node collects the messages passed from its neighborhood, updates its own message and passes the updated message back to its neighborhood. In this section, we propose the LDPC-CC codeword construction algorithm that takes both performance and feasibility into consideration during the construction of the H matrix of the LDPC-CC, and our improved algorithm is based on PEG which is designed for LDPC-BC originally. We first formulate this problem and then discuss the details of the algorithm.

We now deal with the LDPC-CC codeword construction problem. Given the constraint length خ½s, code rate R, the period Ts, and perhaps other constraints such as fixed degree

â€¢ High performance, which means the code should have high coding gain and good BER performance, including early water-fall region and low error floor. This is strongly related to large girth, few trapping sets and etc.

Traditional construction methods just focus on performance of the code and neglect the consideration of implementation, leading to low throughput and efficiency, which motivates us to find a joint optimized construction algorithm concerning both performance and feasibility for LDPC-CC.
B. The two-stage construction algorithm
In this part, we combine a novel algorithm based on PEG and quasi-cyclic codeword structure together to solve the joint optimized LDPC-CC codeword construction problem.
PEG is a traditional girth-aware construction method that tries to avoid cycles in LDPC-BC Tanner Graph to get good codeword performance, but it does not fit LDPC-CC Tanner Graph and now we should note the main differences between CC and BC:
â€¢ Unlimited expansion. Because the parity-check matrix of LDPC-CC is periodic and unlimited, the Tanner Graph is also unlimited, and the check nodes and bit nodes even connect to each other over period.
â€¢ Unique relation area. Each check node has unique relation area for the bit nodes it covers, as shown within bold squares in Fig. 1(a).
So in our algorithm, we should modify the PEG method to get a new solution suitable for LDPC-CC.
To achieve high parallelism degree in the process of encoding and decoding, we induce quasi-cyclic structure, which is characterized as p أ— p identity circulant matrix or zero matrix, into our LDPC-CC parity-check matrix and here p is called parallelism degree or expansion factor. Each Hi(t) in H becomes (c âˆ’ b) آ· p أ— c آ· p matrix and Hi(t) can be represented by the base form Hi(t)b with the size (c âˆ’ b) أ— c. Accordingly, the base form of H is Hb, whose elements satisfy âˆ’1 â‰¤ Hbj,i < p. The elements Hjb,i = âˆ’1 are expanded to zero matrices, while Hjb,i â‰¥ 0 are expanded to identity circulant matrices with permutation factors Hjb,i. One node in base Tanner Graph of Hb can be regarded as p nodes in H, and one edge with weight Hjb,i represents p interleaved edges in H.

VI. THE GPU-BASED SIMULATION PLATFORM
In this section, we introduce the implementation of highspeed QC-LDPC-CC simulation platform based on compute unified device architecture (CUDA) supported by GPUs. With the GPU-based simulation platform, we can get the BER performance, such as the bit error floor, of the generated LDPC-CC codewords much more quickly than usual CPU provides.
CUDA [13] is NVIDIAâ€™s parallel computing architecture. It enables dramatic increases in computing performance by executing multiple parallel independent and cooperated threads on GPU, thus is particularly suitable for the Monte Carlo model and this paper applies CUDA to accelerate the QCLDPC-CC simulation.
