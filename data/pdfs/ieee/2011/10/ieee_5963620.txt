Effective Informed Dynamic Scheduling for
Belief Propagation Decoding of LDPC Codes
Yi Gong, Xingcheng Liu, Member, IEEE, Weicai Ye, and Guojun Han
Abstract—The simultaneous flooding scheduling is popular
for Low-Density Parity-Check (LDPC) Belief Propagation (BP)
decoding. Non-simultaneous sequential scheduling is superior to
the flooding scheduling, and asynchronous dynamic scheduling
has better FER performance than the sequential scheduling.
However, all strategies encounter the trouble of locating the
error variable node. This paper proposes an informed dynamic
scheduling strategy, which utilizes the instability of the variable
node and the residual of the variable-to-check message to
locate the message to be updated first. The informed dynamic
scheduling overcomes the trapping sets effectively. This paper
also designs an informed dynamic scheduling strategy with
adaptivity to pass more messages in parallel, which effectively
postpones the influence of cycles in the Tanner graph. In some
sense, the strategy lengthens cycles. Simulation results show that
the two informed dynamic scheduling strategies outperform other
algorithms.
Index Terms—Belief propagation, low-density parity-check
codes, message-passing, residual belief propagation.
I. INTRODUCTION
L
OW-DENSITY Parity-Check (LDPC) codes, proposed by
Gallager [1], [2], appear as a class of block codes with
sparse parity-check matrices containing only a very small
number of non-zero entries. They are often represented in
graphical form by a Tanner graph or a bipartite factor graph
consisting of the variable nodes and the check nodes.
The Belief Propagation (BP) algorithm is the dominant
decoding algorithm for the LDPC codes. In general, the flood-
ing scheduling strategy for the LDPC BP decoding updates
all check nodes or variable nodes simultaneously. Several
sequential scheduling strategies such as sequential check-
node updates [3]–[6], sequential variable-node updates [7]–
[9], sequential message updates [10], [11], were addressed. It
has been proved that all sequential scheduling strategies have
the similar performance results and converge twice as fast as
the flooding scheduling at no cost [12]. Given the similarities,

Recently, The informed dynamic scheduling named as the
Residual Belief Propagation (RBP) was employed for general
sequential message passing [13]. Then the Informed Dynamic
Scheduling (IDS) [14] was specifically applied to the BP de-
coding, such as the Node-Wise RBP (NW RBP) [15], the Ap-
proximate Node-wise Scheduling (ANS) [16] and the LDPC
decoder [17]. Since then other dynamic scheduling strategies
were introduced such as the Efficient Dynamic Scheduling
for LBP (EDS-LBP) [18] and the Variable-to-Check Residual
Belief Propagation (VC-RBP) [19]. The dynamic scheduling
strategies speed up the BP convergence at the cost of the
increased complexity [15], and have better performance than
sequential scheduling algorithms according to the simulation
results [15]–[19].
The dynamic scheduling strategies stated above always face
the difficulty in looking for a better sequence of messages to be
updated first. An appropriate updating sequence of messages is
very helpful to accelerate the convergence of the BP decoding.
This paper generates the sequence of messages to be updated
first with the help of a decision. To facilitate the description of
our informed dynamic scheduling, here we give the Unstable
Decision first:
Unstable Decision In the LDPC BP decoding, if the Log-
Likelihood Ratio (LLR) sign of a variable node is in reverse
before and after an update, then this variable node is unstable.
This paper proposes the Informed Variable-to-Check Resid-
ual Belief Propagation (IVC RBP) algorithm which employs
a new informed dynamic scheduling strategy. The informed
dynamic scheduling utilizes not only the unstable variable
node but also the residual of variable-to-check message to
locate the variable-to-check message to be updated first (The
corresponding variable-to-check messages are generated and
used to calculate residuals at the same time in the VC-RBP).
The location strategy is named as dual decisions. The intuition
behind the unstable decision is that the unstable variable node
means its binary decision will be changed, which needs more
information to correct itself. Therefore, updating the unstable
variable node with the largest residual first will increase the
convergence speed of the BP decoding. In the new strategy,
check-to-variable message generations are divided into two
parts. One part is used to update the variable-to-check message

II. BP AND NW RBP DECODING
A. BP Decoding for LDPC Codes
In general, the BP exchanges messages along the edges of a
graph. The messages are generated and propagated from each
node to its neighbors based on its incoming messages. Let m
denote the vector of all messages in the graph and mk denote
the kth message in the vector m. fk(m) is the function which
generates mk from m.

The BP decoding is an iterative algorithm which propa-
gates the messages until a stopping rule is satisfied. In the
flooding scheduling, the simultaneous update of all the check-

update of all the variable-to-check messages mv→c in one

sign of the LLR. The decoding algorithm ceases either when
all parity check equations are satisfied with the decoded bits
or the maximum number of iterations is reached.
B. NW RBP Decoding for LDCP Codes

expressed as the difference of the message values before and
after an update. The residual of a message mk is given by
r(mk) = ∥fk(m) −mk∥. In the LLR BP decoding, all the
messages are real number. That is, the message space is
the real numbers space and it is one dimensional. Thus the
residuals are the absolute values of the difference between the
message values before and after an update.

A. IVC RBP Decoding for LDPC Codes
In the LLR BP decoding, the unstable variable node tends
to be in error in binary decision, and it will be stable (the
LLR sign of the variable node before an update is same as
the one after an update) as the BP decoding converges. Thus,
the instability of a variable node means that the algorithm has
not converged yet. For all of the residual-based algorithms,
there is an important law that the larger the residual, the
further this part of the graph tends to converge. Therefore, first
updating and propagating the outgoing message of the unstable
variable node, with the largest variable-to-check residual,
should accelerate the convergence, which is justified later in
our experiments.

ual outgoing message, it implies that the messages incoming
to vi have the same biggest residual and the lowest reliability.

the most possibility to make the variable node unstable and
slow the convergence speed. Therefore, the check nodes with
lower reliability need more information than the ones with

B. AIVC RBP Decoding for LDPC Codes
In all dynamic scheduling algorithms including the IVC
RBP, only the largest residual is utilized. Although the largest
residual shows that this part of the graph is far from conver-
gence, it does not mean that the message with the smaller
residual has converged. In addition, we know that cycles in
a Tanner graph has influence on the exact LLR because the
cycles break the independence between the extrinsic informa-
tion and the original a priori bit probability. In other words,
the decoding error correction performance deteriorates due to
cycles in a Tanner graph. Thus it is desirable to propagate more
messages synchronously in order to postpone the influence
caused by cycles.
The AIVC RBP algorithm is proposed to make the utmost
of all none-zero residuals generated in each IVC RBP update.
It is a parallel algorithm derived from the IVC RBP with adap-
tivity. In the AIVC RBP, The valid messages are propagated in
parallel in order that more messages have been passed before
the occurrence of the further correlation between the extrinsic
information and the original a priori bit probability caused by
cycles.
In the AIVC RBP, we need two accessorial queues, Q1
storing all none-zero r(mvi→cj ) with unstable variable node
vi and Q2 storing all none-zero r(mvi→cj ) with stable variable
node vi. Available r(mvi→cj ) selected from both Q1 and Q2
are stored in the queue Q. All of the check nodes cj in Q are
non-overlapped. That is to say, each variable node connected
to the check node cj cannot be the neighbor of any other check
nodes. Generating Q adaptively is the key to this algorithm.
In order to make the generating procedure easily understood,
it is described separately as an algorithm. Let ℕdenote the
set of all variable nodes and 𝕄denote the set of all check

IV. COMPLEXITY AND PARALLEL DECODING
A. Complexity Per Iteration
Assume that, 𝑑𝑣and 𝑑𝑐represent any variable node degree
and any check node degree respectively (𝑑𝑣and 𝑑𝑐are fixed
numbers when the LDPC codes are regular and they are
average numbers when the LDPC codes are irregular), e is
the number of all edges in the Tanner graph, p is the number
of all elements in 𝑄and e = dv ⋅N = dc ⋅M is valid for
regular and irregular LDPC codes.
If we call the procedure from line 4 to line 20 in Algorithm
2 an update, then there are e updates in one IVC RBP iteration.
We observe that the IVC RBP only generates and propagates
one variable-to-check message in an update. Thus there are
e variable-to-check message generations and propagations in
one IVC RBP iteration. We use the traditional stopping rules
like other algorithms, decoded bits satisfying all the parity-
check equations at the end of an iteration or the number of
iterations reaching the maximum.

by the rule that either the variable-to-check or the check-

corresponding ones of the flooding BP in one iteration.

and propagating more messages. The complexity of Algorithm
3 grows with blocklength of the LDPC codes increasing.
Generating and propagating more messages increase the time
complexity when the AIVC RBP decodes in serial. However,
the AIVC RBP can be performed in parallel. Therefore, as
shown in Tables III and IV, there will be no extra time
complexity of the message computations when the AIVC RBP

B. Parallel Decoding of AIVC RBP
In order to decode efficiently, several processors are re-
quired in the LDPC BP decoding more and more. Parallel
computation attracts researcher’s attention due to the great
computing efficiency. So the AIVC RBP is designed to decode
in parallel. We try to find the balance between complexity and
performance by setting the different values of p.
According to Algorithm 3, the AIVC RBP can be performed
in parallel because the message generations and propagations
induced by each residual are non-overlapped. Similarly, we
assume that there are p elements in Q as in Section III.
Therefore, once Q has been generated, the message gener-
ations and propagations induced by the p residuals of Q can
be processed independently on p processors in parallel. As
shown in Tables III and IV, there is no extra time complexity
of message generations and propagations compared with the
IVC RBP in one iteration.

VI. CONCLUSIONS
This paper proposes a new informed dynamic scheduling,
the IVC RBP algorithm, for the LDPC BP decoding. The
new informed dynamic scheduling strategy includes the dual
decisions and two parts of the check-to-variable message
generations. In the dual decisions, unstable decision locates
the error variable nodes, and the largest residual finds the
variable-to-check message of the error variable node. So up-
dating the variable-to-check message which has been selected
speeds up the BP convergence. In two parts of the check-to-
variable message generations, one is propagated to update the
corresponding variable-to-check message and make it more
reliable, and the other with more reliability is propagated to
calculate the residuals. This computation mechanism can han-
dle “trapping sets" effectively. Simulation results demonstrate
that the IVC RBP has better performance than other dynamic
scheduling algorithms in a few iterations while suffering from
an increase in complexity.
This paper also designs the AIVC RBP, which is a parallel
algorithm with adaptivity derived from the IVC RBP. So the
AIVC RBP inherits the merits of the IVC RBP. Furthermore,
the AIVC RBP generates and propagates non-overlapped mes-
sages in parallel. In spite of the extra complexity of generating
Q, passing more messages synchronously decreases time
complexity greatly in the AIVC RBP. So the AIVC RBP
had better decode in parallel in view of the complexity. In
particular, the AIVC RBP postpones the influence of cycles
in the Tanner graph by passing non-overlapped messages in
parallel. In some sense, the AIVC RBP lengthens cycles. We
set different values of p in the simulations to find the balance
between complexity and performance. The simulation results
show that the AIVC RBP is superior to the IVC RBP when p
is large enough with code rate 1
2 and outperforms the IVC RBP
with code rate 3
4 provided that they have the same message
computations. So we conjecture that when p is larger than the
girth of LDPC codes which has low code rate, the bigger p
is, the better performance the AIVC RBP has.
The IVC RBP and the AIVC RBP proposed in this paper
may extend to other domains, where the iterative BP is
employed.
