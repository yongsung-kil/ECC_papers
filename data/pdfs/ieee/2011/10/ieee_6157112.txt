Abstract
Leakage power is growing comparable to dynamic power dissipation as a result of technology trends, and thus it has become an important issue in low-power circuit design. As a popular technique for standby power reduction, power gating is applied to high-parallel LDPC decoder for WiMAX standard. The clustered-block processing engine (CBPE) array are divided into 9 power domains, and they are switched on or off according to different code lengths of LDPC code defmed in WiMAX standard. As CBPE array occupies about 70% of the decoder system, the dedicated power gating strategy is very effective in shorter code length case since more power domains can be switched off. At shortest code length, power gating design brings about 55% power reduction compared to that of longest code length.

2. QC-LDPC code and LDPC decoding algorithm
2.1 QC-LDPC codes in WiMAX

1. Introduction
Leakage power dissipation is expected to grow by a factor of five every chip generation [1]. As a result of technology trends, it is becoming a serious problem for low-power circuit design. For 65nm CMOS technology, leakage power grows approaching dynamic power. Power gating is an effective strategy used for reducing leakage power [2]. By cutting off VDD from a circuit block when the block is not switching, the overall leakage power can be reduced. However time delay will be increased by the controlling of the power gated modes, which should be considered when designing power gating circuits. In this paper, power gating technique is applied to a high-parallel Low-Density Parity-Check (LDPC) decoder for WiMAX standard to reduce leakage power at short code length. In recent years, the demand for efficient and reliable electronic communication and storage system has been increasing. Among Error correcting codes, LDPC codes are considered as one of the best candidates for next-generation forward error control due to the excellent Shannon-limit-approaching performance and reasonable decoding complexity. LDPC codes have been adopted in various communication standards such as DVB-S2 (Digital Video Broadcasting­ Satellite-Second Generation), WiMAX (IEEE 802.16e), WiFi (IEEE 802.lIn) and so on. In communication standards, usually variable code lengths are defmed to provide proper service and quality of service for each user according to channel conditions. 2.2 LDPC decoding algorithm LOPC decoding algorithms can be mainly divided into hard-decision and soft-decision decoding. The latter one, firstly proposed by Gallager [4], is usually preferred because it achieves much better decoding performance than the former one. In this algorithm, messages are passed along the edges of the Tanner graph [5] between the check nodes and variable nodes. In order to make tradeoff between decoding performance and hardware complexity, offset min-sum simplification is introduced to TPMP algorithm [6]. On the other hand, based on the constructing procedure, in the PCM of QC-LOPC codes, every z rows expanded from the same row of the basic matrix can be regarded as one layer.
Because the column weight of each layer is 1, the
log-likelihood ratio (LLR) form messages could update layer by layer in the iterative decoding, which can improve the decoding convergence time by a factor of two and hence double the throughput than the TPMP algorithm. 3.2 Power gating design
3.2.1 Power gating design basics As a popular method for leakage power reduction, the basic idea of power gating is to create a power-off mode in corresponding to the normal active mode for the chip. The chip should properly switch between the two modes with minimum performance impact. In this design, we adopted the coarse-grain power gating method in which a shared switching network is used to switch off a circuit block composed of several gates. In Fig. 3, the block diagram of coarse-grain power gating is depicted. The power-gated block connects to the power supply through a sleep transistor controlled by an enable signal. The outputs of the power-gated block are fed to a set of isolation cells to produce stable output signals under the control of the enable signal.

3.2.2 Power gating design for LDPe decoder In this work, we designed and implemented a coarse­ grain power gating strategy. As introduced in section 2.1, the WiMAX QC-LDPC code has 19 different expanding factors, ranging from 24 to 96, resulting in 19 different code lengths. In the proposed architecture, 96 CBPEs are instantiated to support variable code lengths. However, except when the expanding factor is the maximum one, some CBPEs can actually be switched off. The optimal power gating strategy is to create 18 power domains (PD) corresponding to 18 expanding factors except 96. However, too much PDs bring a lot of problems in the APR (Auto placement and routing) process during backend design. In our design, we implemented a sub-optimal power gating strategy. The number of PDs is reduced to 9 by combining every two PDs in the optimal strategy into one.

In this paper, a power gating strategy is proposed and implemented for a high-parallel LDPC decoder for WiMAX standard. The variable code length of WiMAX LDPC code provides the motivation for power gating application. In high-parallel LDPC decoder architecture, 96 CBPEs are instantiated to support variable code length, but are not necessarily in work mode in short code length case. The power gating design brings 55% power reduction at the shortest cod length compared to that at the longest code length.
