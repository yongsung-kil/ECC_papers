Implementation of Soft-Decision Forward Error Correction for lOOG Digital Coherent System

Kiyoshi Onohara, Yoshikuni Miyata, Kenya Sugihara, Takashi Sugihara, Kazuo Kubo, Hideo Yoshida, Kazuumi Koguchi, and Takashi Mizuochi
Information Technology R&D Center, Mitsubishi Electric Corporation, 5-1-1 Ofuna, Kamakura, Kanagawa 247-8501, Japan Tel: +81-467-41-2443, Fax: +81-467-41-2519, E-mail: Onohara.Kiyoshi@eb.MitsubishiElectric.co.jp

Abstract We discuss implementation and performance evaluation of LDPC(4608,4080) for 100Gb/s throughput by hardware emulator. We expect that an NCG of the LOPC code concatenated with enhanced FEC is 10.8 dB at a BER of 10-15•
1. Introduction Transmitting high-speed throughput such as 100Gb/s and beyond in optical communication systems, we come up against optical signal-to-noise ratio (OSNR) and waveform distortion. The demand to address this issue is the key to accelerate the development of stronger forward error corrections (FECs). Recently, several trials aimed at exceeding a net coding gain (NCG) of 10 dB have been initiated. Chan et al. proposed a continuously interleaved Bose-Chaudhuri-Hocquenghem (BCH) code having an NCO of 9.35 dB at a BER of 10-15 with 7% redundancy [1]. This is considered to be one of the best performing hard-decision (HD) FECs for optical communications. On the other hand, soft-decision (SD) FEC is a promising way to improve the error correction capability effectively. Thanks to the advent of digital signal processing (DSP) LSIs and high performance analog-to-digital converters (ADC) such as sampling rate of 560s/s, signal bandwidth of 150Hz, and resolution of 6 bits or more for digital coherent systems [2], SD-FEC becomes much easier to incorporate into the receiver hardware. In response to this trend, triple-concatenated SD-FEC using low-density parity-check (LDPC) codes was proposed. The parameters of LDPC code were (4608, 4080) and an NCO was achieved 10.8 dB at a BER of 10-15 with 20.5% redundancy [3]. An

LOPC(18353,15296) code with an NCG of 11.218 dB at a BER of 10-15 with 20% redundancy was presented [4]. A longer codeword brings with it increased NCG and also prevents any undesired error flaring floor. In this paper, we discuss triple-concatenated SO-FEC scheme for 100 Gb/s transport systems. LOPC codes are introduced as good candidates for strong SO-FEC. We also discuss FEC frame structure and optical transponder for a 100 Gb/s digital coherent system. In addition, we evaluate error correcting performance by implementing the entire 100 Gb/s throughput of LOPC code on an FPGA based hardware emulator.

In the conventional combination the residual error after decoding at an inner code (Fig. l(a)) is normally less than 10-4 to correct properly at an outer code including RS(992, 956) code (Fig. l(b)) [5]. 3. Implementation and Hardware Emulation Fig. 2 illustrates an OTU4V frame structure and associated transponder configuration for a 100 Gb/s digital coherent system. OTU4V frame consists of payload, 7% HO-FEC, and 13% SO-FEC. The transponder consists of an OTU4 framer LSI, a coherent ASIC and an optical front-end (Opt. FE). The OTU4 framer LSI transforms the 100GE signal to and from the OTU4 framed format, and usually incorporates an HO-FEC. The coherent ASIC includes an AOC/OSP, q-bit soft decision (typically q=2, 3), an interleaver, a deinterleaver, and LOPC decoder (OEC). We evaluated the error correcting performance by implementing the entire 100 Gb/s throughput of LDPC(4608, 4080) code on an FPGA based hardware emulator (Mentor Graphics, Veloce). Hardware emulations enable us to save time drastically due to higher speed processing than software simulations and evaluate phenomena that slowly change the signal quality. The clock frequency was 556 kHz. We evaluated 204,800,000 codewords in 7.4 hours. Fig. 3 shows the FEC performance where the vertical axis is post-FEC BER and the horizontal axis is pre-FEC Q represented in dB. A blue solid line shows the results of an LDPC code, and a red solid line shows those of triple-concatenated SD-FEC by software simulation we have reported [3]. Red circles show the results of an LDPC code by hardware emulation. It is clearly shown that the results of LDPC code by hardware emulation are consistent with

4. Conclusions We have discussed the implementation of an SO-FEC for 100 Gb/s transport systems. In addition, we have verified error correcting performance by implementing the entire 100 Gb/s throughput of LDPC(4608, 4080) code on an FPGA based hardware emulator. It has been clearly shown that the results by hardware emulation were consistent with those by software simulation. We expect that an NCG of the LDPC code concatenated with EFEC was 10.8 dB at a BER of 10-15.
