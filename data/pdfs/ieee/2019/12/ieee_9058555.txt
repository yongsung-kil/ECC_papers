Abstract—The Enhanced quasi-maximum likelihood (EQML) decoder has better decoding performance than the traditional belief propagation (BP) decoder, but its decoding speed is greatly reduced due to its complicated computational structure. Therefore, an efficient graphics processing unit (GPU) implementation structure for EQML decoder of low-density parity-check (LDPC) code is proposed. Intra-codeword parallel and inter-codeword parallel decoding schemes on GPU are adopted instead of serial decoding on CPU. Several optimization methods have also been employed to enhance the decoder throughput, such as compressed storing and Kernel thread arrangement scheme, which improve the decoding speed without degrading the decoding performance of the decoder. Experiment results show that our GPU based implementation structure which is about two orders of magnitude faster than that of CPU implementation.
Keywords-LDPC code; GPU; EQML decoder; parallel decoding; BP decoder
I. INTRODUCTION
Low Density Parity Check (LDPC) [1] is a linear block code that is known for its performance very close to the Shannon limit [2]. The LDPC code has the following advantages: The check matrix is flexible in construction, small in delay, low in density, and low in decoding complexity, and the decoding mode can be implemented in parallel operation, thus having high throughput and highspeed decoding potential. In fact, LDPC codes are used in many industrial protocols, such as DVB-S2, DVB-T2, WiFi (802.11n), WiMAX (802.16e), and fifth-generation (5G) wireless systems.
Lower transmission delays and higher throughput are required in the 5G mobile communication system standard. Therefore, to achieve higher throughput requirements, an Application Specific Integrated Circuit (ASIC) [3] and a Field Programmable Gate Array (FPGA) [4] are proposed for the LDPC decoder. However, these solutions have the disadvantage of high design cost and single function. GPU have been widely used in recent years for high computing power. It can execute multiple threads at the same time, and

the peak performance can reach tera-flops per second. GPUbased solutions are cheaper, more scalable, and more flexible than ASIC or FPGA solutions. [5-7] implemented some LDPC decoders on the GPU and achieved good results. Most of these works are used in the traditional Belief-propagation (BP) decoding algorithm's simplified algorithm which is Min-Sum algorithm (MSA), large-scale multi-codeword parallel decoding, and even multi-GPU implementation to achieve higher throughput. There is a lack of specific applications for some specific LDPC decoders.
Recently, in order to improve the bit error rate (BER) performance of LDPC decoder, some researchers have proposed to introduce a reprocessing after the failure of the traditional BP decoder. The reprocessing is not to decode the codeword once, but to select some nodes for the input codeword to be saturated according to node selection rules, and to perform BP decoding on the saturated codeword sequence, which can greatly improve the error correction performance of the decoder [8-9]. The Enhanced QuasiMaximum Likelihood (EQML) [9] proposed a more accurate node selection method and a more reasonable decoding stop rule based on [8]. But Due to the inherent structure of the EQML decoder, the decoding time of the decoder far exceeds the traditional BP decoder.
To improve the decoding time of the EQML decoder, we propose an implementation structure for EQML decoder on GPU. Some optimization schemes are proposed, compressed storage parity check matrix of irregular LDPC code, re-order the traditional BP decoding algorithm, multi-codeword parallel decoding is performed, constant memory and shared memory are used to access. A parallel implementation scheme for EQML decoder for LDPC is designed to reduce the decoding time while maintaining the error correction performance of the decoder.

A. Compressed Storage H Matrix

B. Graphics Processing Unit Implementation of Enhanced Quasi-Maximum Likelihood Decoding Algorithm
There is a highly irregular interconnect structure between CN and VN. Therefore, the EQML decoder available on GPU cannot simultaneously read and write continuous data blocks in memory, resulting in long access time. Because the most time-consuming GPU operation is accessing unordered memory, that is, accessing non-contiguous memory locations, multiple transactions are required to perform unordered memory reads or writes, and all kernel threads must be stopped [5]. In contrast, coalesced memory addressing, which accesses contiguous memory locations, can be performed in a single transaction and allows concurrent threads to execute, which reduces kernel runtime. Furthermore, uncoalesced memory writes are more time consuming than uncoalesced memory reads [10]. Therefore, memory access mode should be reasonably arranged for EQML decoder on GPU. The BP decoding algorithm are reordered to avoid uncoalesced memory writes and to use as many threads as possible for parallelism calculations.
Fig. 4 shows the re-order BP decoding algorithm flow chart. A single codeword consisting of four multi-threadeds Kernels. Each Kernel consists of different parallel-level threads. For example, the VN update in Kernel 1 arranges T threads to calculation, where T is the maximum number of connections between the edges in Fig. 2, and each thread computes an edge in the Tanner graph. V. CONCLUSION
The 5G mobile communication system requires higher data transmission rates, more connections, and a better user experience while ensuring low cost and security, reliability, and stability of transmission. 5G needs to guarantee higher reliability communication. The EQML decoder can eliminate a large number of pseudo-codewords that cause the BP decoder, providing more reliable communication. The EQML decoder provides superior decoding performance at the expense of decoding time. Therefore, to improve the decoding rate of the EQML decoder, this paper implements the EQML decoder on GPU. Compressing the irregular LDPC codes, reorder the BP decoding algorithm to reduce the decoding delay caused by uncombined memory access, a multi-codeword parallel decoding scheme in reprocessing stage, this paper demonstrates the performance of EQML decoder based on GPU with 802.16e WiMax LDPC code as an example. The experimental results show that the algorithm implemented in parallel on the GPU has a higher speed than the serial implementation on the CPU. The maximum speedup ratio can reach about 70 at low SNR.
