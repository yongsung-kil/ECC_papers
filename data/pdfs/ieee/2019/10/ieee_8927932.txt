Tabu-List Noisy Gradient Descent Bit Flipping Decoding of LDPC Codes
Li Zhang, Nan Liu, Zhiwen Pan, Xiaohu You National Mobile Communications Research Laboratory, Southeast University, Nanjing 210096, China
Email:{220170901, nanliu, pzw, xhyu}@seu.edu.cn

Abstract—The Noisy Gradient Descent Bit Flipping (NGDBF) algorithm surpasses the previous Gradient Descent Bit Flipping (GDBF) and other Bit Flipping (BF) algorithms for decoding Low-Density Parity-Check (LDPC) codes. However, the NGDBF decoder sometimes cannot escape from the local maxima. To further enhance the performance of the NGDBF algorithm, an improved NGDBF algorithm called Tabu-List Noisy Gradient Descent Bit Flipping (TNGDBF) is proposed in this paper. A tabu-list is employed to help the NGDBF decoder escape from the local maxima. It is shown in the simulation results that the TNGDBF algorithm gains about 0.2 dB performance compared with NGDBF at the cost of a small amount of complexity increase.
Index Terms—GDBF, bit flipping, LDPC, tabu-list, iterative decoding.
I. INTRODUCTION
A considerable research attention of Low-Density ParityCheck (LDPC) codes [1] has been gained in the past years. Various kinds of decoding algorithms for LDPC codes provide a good tradeoff between performance and complexity. The soft decision algorithms, referred to as Belief Propagation (BP) algorithm [1], Min-Sum (MS) algorithm [2], and Offset MinSum (OMS) algorithm [3], have a wonderful performance on the binary input additive white Gaussian noise (AWGN) channel, but at the cost of high computation complexity. Comparatively, the hard decision algorithms, referred to as Bit-Flipping (BF) algorithms, are easier for hardware implementation and have a lower complexity.
Many variants of BF algorithms which employ soft information have been investigated to improve the performance, such as Weighted Bit Flipping (WBF) algorithm [4] and Modified Weighted Bit Flipping (MWBF) algorithm [5]. Recently, the Gradient Descent Bit Flipping (GDBF) algorithms [6] are proposed. The GDBF algorithms develop a new inversion function derived from a gradient descent formula. The GDBF algorithms outperform the previous BF algorithms, but they tend to be trapped in local maxima and converge on an erroneous codeword. To overcome the drawback of GDBF algorithms, the Noisy Gradient Descent Bit Flipping (NGDBF) algorithms [7] are proposed to improve the performance. To make the decoder get out of the local maxima, every symbol metric employs a random perturbation at each iteration. Unfortunately, the NGDBF algorithms cannot always escape from the local maxima.

In [8], the type of gradient descent bit flipping algorithm is adapted from the AWGN channel to the binary symmetric channel (BSC). The Probabilistic Gradient Descent Bit Flipping (PGDBF) algorithm only uses hard information to decode. In [9], the Tabu-List Random-Penalty Gradient Descent Bit Flipping (TRGDBF) algorithm, which uses a random-penalty and a tabu-list to make the decoder escape from the trapping sets and accelerate convergence, is proposed for the BSC.
In this paper, we propose an improved NGDBF algorithm called Tabu-List Noisy Gradient Descent Bit Flipping (TNGDBF) algorithm on the AWGN channel. Inspired by [9], a tabu-list is used in each iteration to avoid repeated bit flipping between two successive iterations. Tabu-list and random perturbation work together to prevent the decoder form being trapped. Simulation results show tabu-list can help the TNGDBF decoder escape from the local maxima. We also provide the optimal values of noise scale parameter as a function of the SNR value. Only a small amount of computation complexity is needed to gain about 0.2 dB performance compared with the NGDBF algorithm.
II. TABU-LIST NOISY GRADIENT DESCENT BIT FLIPPING DECODING
A. System Model
Let H denote a binary parity check matrix with m × n dimensions and n > m ≥ 1. We consider the set of LDPC codes C which is represented by C {c ∈ F2n : Hc = 0}. In the present paper, we define the set of bipolar codes Cˆ {(1 − 2c1), (1 − 2c2), ..., (1 − 2cn)} corresponding to C. Evidently, C is mapped to Cˆ from binary (0, 1) to bipolar (+1, −1). Assume that the codes are transmitted over a binary input AWGN channel and the transmission is defined by y = ˆc + z, where ˆc ∈ Cˆ and z is a white Gaussian noise vector. Each element of z is an independent and identically distributed Gaussian random variable with zero-mean and variance N0/2, where N0 is the noise spectral density.
The hard decision vector x ∈ {+1, −1}n is obtained by xk = sign(yk) for k = 1, 2, ..., n. Let T be the maximum iteration number which is designed by the algorithm. The vector at the t-th iteration is represented by x(t), where t ∈ [1, T ]. In each iteration, one or more bits may be flipped according to different algorithms. In this paper, we only consider the single BF algorithms [4], [5].

C. Tabu-List Noisy Gradient Descent Bit Flipping Algorithm
To further improve the performance of NGDBF algorithm and keep the low complexity, we add a tabu-list to the NGDBF algorithm and propose a new algorithm called Tabu-List Noisy Gradient Descent Bit Flipping (TNGDBF) algorithm. The TNGDBF algorithm can be formally stated as follows:
Algorithm 2 Single TNGDBF (S-TNGDBF) Algorithm 1: Initialization: Set T = 0, L0 = 0 and Ek0 = xkyk for k = 1, 2, ..., n. 2: Compute bipolar syndrome si = j∈N(i) xj for i = 1, 2, ..., m. If si = +1 for all i ∈ [1, m], output x and stop; Otherwise, T = T + 1. 3: Calculate inversion function at each VN using (3). 4: Find the bit position given by xk for k = arg mink∈{1,2,...,n}\LT−1 Ek, and then flip and record the symbol: xk = −xk, LT = k, where LT is used to record the flipped position in the T -th iteration. 5: If the number of iterations do not reach the maximum number of iterations T , repeat steps 2 to 4 till a valid codeword is detected; otherwise output x and exit.
As discussed above, once the decoder is trapped in the local maxima, it will repeatedly flip some bits of the codeword in a fixed order and can hardly escape from the local maxima. The increase of the objective function is stopped by the local maxima and the decoder will oscillate. In the TNGDBF algorithm, the tabu-list is used to record the flipped bit position in each iteration and prevent the same bit from being flipped between any two adjacent iterations. This can increase the ability of the decoder to jump out of the local maxima with only a little increase in complexity.
As is shown in Step 3, we use the same inverse function as that in the NGDBF algorithm. The optimal noise scale parameter of the TNGDBF algorithm can be obtained by simulation. In Step 4, we add the bit which is flipped in the current iteration to the tabu-list and remove it in the next iteration. Fig. 2 shows the value of f (x) in (1) evaluated as a function of iterations at Eb/N0 = 5.5 dB. All variants of the GDBF algorithms are designed to find the global maximum of the objective function f (x) quickly and accurately. By flipping particular bit iteratively, every class of GDBF algorithm attempts to maximize the f (x). Hence, we can observe the objective function value of each iteration for convergence analysis. The transmitted codeword is the ML solution for any ML-decodable problem, and the global maximum of objective function is obtained by introducing the transmitted codeword in (1). As is shown in Fig. 2, the value of objective function gradually increases with the number of iterations, and the global maximum is fmax ≈ 1534.84. After 25 iterations, different algorithms enter different states. The rise of GDBF, which is indicated by the lower triangle line, stops and the f (x) flattens out, and NGDBF, which is indicated by the asterisk line, is the same, but the fluctuation of NGDBF is greater because of the random perturbation. Obviously, the random perturbation tries to help NGDBF get out of the local maximum, but it fails in this example. Note that tabu-list makes TGDBF and TNGDBF escape from the local maximum effectively.
Fig. 3 shows the BER performance for NGDBF and TNGDBF algorithms on the PEGReg504 × 1008 LDPC code with different maximum iterations. At BER = 10−6, the TNGDBF algorithm outperforms the NGDBF algorithm by 0.2, 0.2 and 0.25 dB with 100, 200 and 300 iterations, respectively. Notably, the TNGDBF algorithm with 200 iterations has the same performance as the NGDBF algorithm with 300 iterations at BER = 10−6, which means that the tabu-list can efficiently accelerate the convergence.

IV. CONCLUSION In this paper, an improved iterative decoding algorithm, denoted as TNGDBF, is proposed based on the NGDBF algorithm. In TNGDBF, a tabu-list is introduced to help the NGDBF decoder escape from the local maxima. With a small amount of complexity increase, the proposed TNGDBF algorithm has about 0.2 dB performance gains compared to the NGDBF algorithm. We also study the relationship between the noise scale parameter and performance. While we only consider the single GDBF algorithms in this paper, the multiple GDBF algorithms [6], [7] are worth looking into as part of the future work.
