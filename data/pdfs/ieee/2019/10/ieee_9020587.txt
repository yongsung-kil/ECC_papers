A QC-LDPC code can be constructed from an M × N base graph (BG) matrix B [7]. Each element bij of the matrix B corresponds to a Z ×Z cyclic permutation matrix (CPM) Pbij .
If each element in B is replaced by the corresponding CPM, a m × n parity check matrix H can be obtained, where the parity length m = M Z, the code block length n = N Z and the information length k = n − m = (N − M )Z. The BG
matrix is usually designed first, and then lifted with the socalled lifting factor Z to obtain the final parity check matrix
with compatible information blocklength size.
Unlike conventional QC-LDPC codes, the BG matrix of an
RL-QC-LDPC code ensemble has raptor-like structure, which
is inherently suitable for rate-compatibility [1]. Two base graphs with raptor-like structure, BG1 with N = 68, M = 46 and BG2 with N = 52, M = 42, were adopted by 3GPP for the channel coding [2]. The lifting factor Z of BG1 and BG2 ranges from 2 to 384. Thus, BG1 is targeted for larger blocklengths (40 ≤ k ≤ 8448) and higher rates (1/3 ≤ R ≤ 8/9), whereas BG2 is targeted for smaller blocklengths (20 ≤ k ≤ 3840) and lower rates (1/5 ≤ R ≤ 2/3).

B. Decoding Algorithm

Besides the parity check matrix, the Tanner graph is also
used to describe an LDPC code, which is a bipartite graph with n variable nodes (VN) on one side and m check nodes (CN) on the other side. If Hxy = 1, there is an edge connects CNx and VNy. A typical LDPC decoding algorithm can usually be represented as an iterative message propagation process on
the graph, by exchanging messages along the edges. There
are two issues determine the performance and computational
complexity during the message propagation: the computation
approach and the message propagation scheduling.
For the computation approach, the belief-propagation (BP)
algorithm, also knwon as the sum-product algorithm (SPA),
shows near optimal BER performance. To avoid multiplica-
tions, a BP algorithm in the Log-Likelihood Ratio (LLR) domain (LLR-BP) is widely used alternatively [8]. Besides the conventional flooding scheduling, layered-based scheduling is also attractive as the message propagation scheduling for the LDPC decoder. In layered-based scheduling, the parity check matrix are looked as a layered graph that can be sequentially processed, and the decoder iteration is then split into sub-layer iterations. In particular, horizontal-layered scheduling was presented in [11]. From an algorithmic point of view, horizontal-layered scheduling has some advantages compared with the flooding one. Firstly, the decoding iteration number required to achieve similar correction performance is reduced by half. In addition, the horizontal-layered algorithm reduces by approximately 40 percent of the memory footprint (MF) [12]. As in [11] and [4], we employ horizontal-layered scheduling in this work.

To study the parallelism of decoders based on different instruction sets, we measured the parallelism Qˆ for Z ranged from 2 to 384 in 5G NR [2]. As shown in Fig. 2, with the increase of maximum parallelism Q, the actual parallelism Qˆ increases accordingly. When the value of Z is large, the parallelism Qˆ is close to Q. If Z < Q, the parallelism decreases significantly, and the intra-CB parallelization scheme cannot bring sufficient throughput gain to the system. In this case, the intraCB parallelization scheme can be combined with inter-CB parallelization proposed in [4] to achieve higher parallelism, and the decoder can further improve the throughput without increasing the latency.
In this implementation, AVX-512 is utilized as the instruction set. On one hand, according to Fig. 2, the LDPC decoder based on AVX-512 can achieve a higher degree of parallelism. On the other hand, SSE and AVX2 instruction sets do not support loading and storing byte-type data with masks due to memory read-write limitations. AVX-512 provides more flexible load and store instructions, which can be utilized to improve decoding performance.

In this paper, a software implementation architecture with SIMD acceleration for horizontal-layered LDPC decoding of 5G LDPC decoders is presented. The application of AVX-512 instruction set effectively improves the degree of parallelism, and therefore the capacity of x86 processors can be fully exploited. Inaddition, operation cost is reducedby exploiting new features of AVX-512. Currently, the proposed decoder implementationfortenlayereddecodingiterationscanachieve 42 to 272 Mbps throughput with a single core, and 5 Gbps throughput with 18-core Intel Xeon CPU. The focus of this paperistoimprovethroughputwhilemaintaininglowlatency. It is foreseeable that the LDPC decoder with lower energy consumption and high performance can also be implemented by GPP.
