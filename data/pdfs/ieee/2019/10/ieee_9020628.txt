In general, an LDPC code (n, j, k) has a (n−k)×n [10] parity-check matrix with j 1s in the column and k 1s in the row. An LDPC code corresponds to a specific check matrix H, which is sparse for the reason that j, k<n and can describe the code in mathematical form.
Each row and column of H represents a check equation and a codeword bit, respectively. Such a matrix can be presented in the form of a graph, i.e., a Tanner graph. For any LDPC codes, the Tanner graph T = (V, C, E) consists of n variable nodes denoted by V and n − k check nodes denoted by C [11]. Therefore, the check node and variable node in the Tanner graph are connected if and only if the element in H is 1.

III. CHANNEL-BLIND BP WITH DEEP LEARNING AND DICTIONARY LEARNING

A. LDPC Pattern

Before decoding, we need to determine the interconnection relationship between the internal nodes of the LDPC codes, that is, the pattern of the template. In order to ensure excellent performance, the template must satisfy the following requirements:
1) The size of the template needs to be suitable, not too large or too small. Because an oversized template will suffer from huge computational complexity, and a template that is too small will lose the information between the internal nodes.
2) The relationships between internal nodes should be reflected in the template as much as possible to minimize the information loss [12].
For LDPC code (n, k), the generator matrix G can be transform to [I P ] through Gaussian elimination linear transformation. Thus, the parity-check matrix H can be rewritten as follows:

B. Pre-Filtering with Dictionary Learning

A channel-blind decoding refers to the problem of decoding without evaluating the channel condition, which is useful in most decoding-related applications. Under the influence of noise, a pixel is declared to be noisy if the gray level of it changes. Therefore, pre-filtering is very essential for image denoising. In the case of Rayleigh channel, an image denoising technique called dictionary learning is applied in LDPC prefiltering.

Generally, the training process of CNN contains the following two parts:
1) Establishment of a network model; 2) Feature learning from the data set [9].

1) Residual Learning: Residual learning [8] of CNN is the solution for the problem that the training accuracy gradually decreases as the network depth increases [9]. Let us consider H(x) as a mapping function with x denoting the inputs of the first layer, which is difficult to map H(x) = x. Thus, we adopt residual function R(x) := H(x) − x rather than expecting stacked layers to approximate H(x).
2) Network Architecture: The input of CNN is y = x + n, which is composed of the original image x and the noise n from channel or environment. According to the residual learning, we train the mapping R(y) ≈ n, then we can represent the input as x = y − R(y) according to the residual mapping.
