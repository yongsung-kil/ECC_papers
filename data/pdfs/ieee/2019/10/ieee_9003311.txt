On the Hard-Decision Multi-Threshold Decoding of Binary and Non-Binary LDPC Codes

Abstract—In this paper, we investigate the effect of single and multiple thresholds approaches applied to iterative harddecision decoders for binary and non-binary low-density paritycheck (LDPC) codes in terms of error correction and managed to optimize the performance of decoding algorithms for these codes.
I. INTRODUCTION
Davey and MacKay were first to describe non-binary LDPC codes over GF (q), q > 2 and its decoding techniques, as an extension of binary LDPC codes, firstly introduced by R.G. Gallager in [1], and to obtain that such codes show a tremendous improvement in terms of error correction properties [2].
Titanic work [3]–[5] was done to improve the original Belief Propagation algorithm, proposed in [2], with respect to complexity and error correction ability. But, in the case of high-throughput optic communication, which requires the high rate and big codeword length, the given soft-decoding algorithms are significantly complex to be embedded into hardware. Therefore, in this scenario, hard-decision decoders must be observed, which considered having low complexity, without significant loss of error correction properties.
Kovalev in [6] made a theoretical estimation on error rate for multiple-threshold majority decoder for binary LDPC codes. Frolov and Zyablov in [7], following the ideas from [6], extended majority threshold decoding scheme for non-binary LDPC. In the papers [8], [9], the main result was theoretical proof of the fact, that threshold decoding improves decoding schemes in terms of error-correcting capabilities. It was also shown in [10], that for a fraction of guaranteed correctable errors for a non-binary LDPC code in a channel with q-ary errors (symbol errors), it significantly exceeds the fraction of guaranteed correctable errors with a binary LDPC code in a channel with binary errors (binary errors). Therefore, the following question arises of the practical study of these codes in various channels.
In this paper, we make an effort to deduct practically the effect of different thresholds in comparison to single thresholds and to compare in this decoding schemes the performance of multiple threshold decoding algorithms for non-binary LDPC

code, proposed in [9], as the extension of majority decoding algorithm, proposed in [7] and binary ones.

II. LDPC CODE
Binary LDPC code C defined by its parity check matrix M × N , denoted as H:

H = [hi,j], hi,j ∈ {0, 1} Every codeword c ∈ C satisfies the following equation:

cH = 0,

Where addition and multiplication are supposed to be bit-wise operations.
Due to the fact, that q-ary LDPC are supposed to be extension of binary LDPC codes, non-binary LDPC code over finite field GF (q), q > 2 is described by its parity check matrix H. For code C of length N parity check matrix is represented as:

H = [hi,j], hi,j ∈ GF (q),
where i = 1, . . . , M , j = 1, . . . , N . Every codeword c ∈ C, satisfies (1) over finite field,
where the multiplication and addition (denoted as , respectively) is processed over finite field GF (q).
Non-binary LDPC code is called regular if it has constant row and column weight among all columns and rows of the matrix. Otherwise, code is called irregular. This classification comes from similar terms for the binary case.

IV. DECODING ALGORITHMS
In this subsection, we denote the main terms in case of syndrome decoding, which uses the sequence of different thresholds to be applied to the received from channel sequence. After data transmission, we receive the sequence of q-ary symbols, the received sequence denoted as r, which corresponds to one of the codewords with the noise added (denoted as η). So the following value is called syndrome and denoted as:

S = rH = (c + η)H
If syndrome’s weight equals zero, it is easy to see from (1), that r - codeword. So the main goal of every decoding technique is to decrease the weight of syndrome at every iteration step. Among them is the Threshold Decoding algorithm, which is described below.
Let us shortly describe the multi-threshold decoding algorithm, proposed in [9]. The single-threshold decoding algorithm assumed to be an internal function of a multi-threshold one. The main idea of multi-threshold decoding is to start decoding with the symbols that are incorrect with high probability (are included in the big number of unsatisfied checks) and have a decision that is correct with high probability (the big number of unsatisfied checks with the same decision for this symbol).

Algorithm 2 Single-Threshold algorithm
Data: parity-check matrix H = [hi,j] of size M ×N , received sequence y, threshold θ
Result: estimated codeword ˆc define Z(i) - non-zero entries in i-th column of matrix H; syndrome S = yH = [s1, . . . , sM ]; initialize b = 1; while b = 0 do
b = 0; for all 1 ≤ i ≤ N do
for j ∈ Z(i) do mj→i ← −h−j,i1sj Lmax = max(L) - maximal subset of equal non-zero messages, vector counter of non-zero messages defined as L; initialize a = |Lmax|; initialize m = argmax(L) - index of Lmax if a − z > θ then cˆi ← cˆi + m; update S = ˆcH; b = 1; end
end end end
In these algorithms, addition and multiplication are defined for the exact type of field (see Preliminaries) or, in the binary case, bit-wise logic operation.

VI. CONCLUSION
From our results, we can conclude that multiple threshold decoding positively effects on error correction performance of the LDPC code for every q. Also, we obtain the fact that for the increasing weight (> 5) we obtain the best performance on Q = 16, but for a more sparse matrix, the binary case is still the best one. Moreover, we show that in terms of input BER binary LDPC code is more efficient, but in terms of SER, non-binary LDPC code dominates.
The following research will be devoted to the threshold number optimization with respect to error correction performance of LDPC code.
