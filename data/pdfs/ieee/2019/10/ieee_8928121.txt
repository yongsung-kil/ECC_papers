Algorithm for LDPC Codes

Abstract—Alternating Direction Method of Multipliers (ADMM) is an effective method for Linear Programming (LP) decoding of Low-Density Parity-Check (LDPC) codes. In the ADMMbased LDPC decoding algorithm, Euclidean projection on the check polytope is the key step. Current Euclidean projection algorithms in general require either sorting or iterative operations. However, when decoding LDPC codes with high coding rate, the sorting operation brings a high level of computation complexity. Moreover, although the current iterative projection algorithms do not involve the complex operation of sorting, its convergence speed is slow. In this paper, based on trend extrapolation, a reduced-complexity iterative check polytope projection algorithm is proposed. In the proposed algorithm, the target projection point can be found based on the variation trend of the geometric position of the projection points obtained from previous iterations. Therefore, it ensures fast convergence and does not sacrifice on bit error rate (BER) performance. The simulation results confirm that the proposed algorithm can effectively reduce the decoding complexity, especially for LDPC codes with high coding rate.
Index Terms—Alternating Direction Method of Multipliers (ADMM), Linear Programming (LP) decoding, Low-Density Parity-Check (LDPC), Euclidean projection, check polytope.
I. INTRODUCTION
L OW-Density Parity-Check (LDPC) codes [1] have many decoding algorithms, most of which are essentially message iterative decoding algorithms based on Tanner graphs. The main decoding algorithms include belief propagation (BP) decoding algorithm [2] and linear programming (LP) decoding algorithm [3]. The LP decoding algorithm regards decoding as a linear programming problem and completes decoding by solving LP problem. The bit-error-rate (BER) performance of LP decoding is close to the Maximum Likelihood (ML) decoding. Therefore, LP algorithm has become a hot field of decoding algorithm research.

Recently, the Alternating Direction Method of Multipliers (ADMM) was proposed for the solution of LP decoding problem of LDPC codes by Barman et al.[4]. ADMM is widely used in solving large-scale distributed convex optimization problems, which is combined with the convergence of the multiplier method under weak conditions and the decomposition and the dual decomposability of dual ascent [5]. Because ADMM guarantees strong convergence, simple scheduling and strong robustness, it perfectly fits in LP decoding. However, compared with BP decoding algorithm, the ADMM-based decoding algorithm in general has high implementation complexity, mainly due to the complexity of Euclidean projection on the feasible region. Therefore, in order to reduce the complexity of ADMM-based decoding algorithm, a great deal of efforts have been made to reduce the complexity of projection procedures. In [6], Barman et al. proposed an efficient projection algorithm. In this algorithm, check polytope can be expressed as the convex hull of two “slices”, which is the convex hull of the same weight vector. Projecting vector onto the check polytope is equivalent to projecting vector onto the convex hull of two adjacent slices. Yet it contains two sorting operations, which still cause high complexity, especially when decoding LDPC codes with high coding rate. To reduce the number of sorting operations, Zhang and Siegel [7] proposed another projection algorithm, which involves only one sorting operation. In this algorithm, the cut search algorithm [8] has been used to find the facet of the polytope in which the projection point lies and then project vector onto it. In [9], Wei et al. also proposed to reduce the decoding complexity by reducing the number of iterations and Euclidean projections, rather than reducing the complexity of projection algorithm. More recently, in order to avoid the high complexity of sorting operation, Wei and Banihashemi [10] proposed to use an iterative structure for projection onto the check polytope. However, the convergence speed of the current iterative projection algorithms is limited because of the geometric structure of the check polytope. Therefore, when the number of iterations required is rather high, the algorithm with iterative nature has a high complexity.
In this paper, we propose a reduced-complexity iterative projection algorithm with the adoption of trend extrapolation. The basic idea of trend extrapolation is to use a model to

A. LP Decoding Method
Consider a binary LDPC code C and a parity check matrix H of size m × n. Each row of the parity check matrix corresponds to a check node, indexed by J ={1, . . . , m}. Each column of the parity check matrix corresponds to a variable node, indexed by I ={1, . . . , n}. Let Nj = {i ∈ I : Hij = 1} represent the set of all variable nodes connected to the check node j and at the same time let Ni = {j ∈ J : Hji = 1} represent the set of all check nodes connected to the variable node i. dj denotes the total number of elements contained in Nj and di is the degree of variable node i.

In the proposed algorithm, we use a constant μ to determine when to use the method of trend extrapolation to find the convergent point. If δ is less than μ, we can find the projection point along the trajectory of u in the iteration. Fig. 2 shows the process of the proposed projection algorithm. At iteration 4, because δ4 is less than μ, we can find the target projection point z along the trajectory from u3 to u4 and get it in one step.
It is worth mentioning that the value of μ should be set carefully. Although setting the value of μ to be a high value can accelerate convergence, it may bring poor BER performance. In Fig. 3, if δ3 is less than μ, the projection point z deviates from the exact value z. Therefore, the value of μ should be optimized based on multiple simulation results.

V. CONCLUSION
In this paper, we propose a reduced-complexity iterative check polytope projection algorithm to reduce the computational complexity of ADMM decoding method of LDPC codes. The proposed algorithm uses the method of trend extrapolation to find the target projection point based on the information from previous iterations, which can greatly reduce the number of iterations. Therefore, compared to existing projection algorithms, it greatly accelerates the convergence speed of iteration, especially for high-rate LDPC codes.
