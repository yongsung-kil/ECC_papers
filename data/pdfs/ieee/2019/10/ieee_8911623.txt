Abstract ‚Äî A sectional degree match approximate Min-sum decoding algorithm for LDPC is proposed in this paper. The degradation factor from BP to MS is analyzed and a proposal to offset it, is explained, with expansion and optimization based on principles of hardware implementation. The decoding algorithm summarizes three correction approximate algorithms considering different LLR values, matrix degrees and rules of hardware implementation. Simulation is following LDPC NR 3GPP 38.212 release and the comparison results demonstrate that the proposed sectional degree match approximate Min-sum decoding algorithm has a better BER and less iteration time. The hardware implementation of this algorithm is strived from Min-Sum decoder and the extra cost is minor besides the memory.

Keywords‚ÄîLDPC, Min-Sum, LDEC, check nodes, approximate
I. INTRODUCTION
LDPC (Low density parity check) codes were proposed by R.Gallager in 1963 [1], and has gained a lot of interest because of its outstanding error-correction performance. There are lots of decoding algorithms proposed with different advantages in actual implementation. The MS (Min-sum) algorithm [2] is the most used decoding algorithm in modem SoC design due to its reusability between NR and LTE decoder modules and straightforward hardware structure. It leads to a balance between error-correction performance and decoding complexity, which is very attractive in hardware implementation. The belief propagation (BP) algorithm [3] provides the powerful tool for iterative decoding of LDPC codes. The MS approximation using so-called BP-based approximation is well known for hardware implementation because it can greatly reduce complexity with the cost of a degradation in decoding performance. This has led to the development of many balanced variants of the MS algorithm [4]‚Äì[8] that nonetheless deliver near-optimum decoding performance.
S. L. Howard proposed degree-matched check node decoding for regular and irregular LDPCs [9]. An asymptotic expression for the correction factor at a check node of any degree is derived in this paper, and used to develop a simple approximation method to the correction factor, matched to the check node degree. Simulation results are presented for each check node approximation when decoding a regular and an irregular LDPC, which can achieve sum-product decoding performance.
N. Maammar proposed a layered offset min-sum algorithm for the decoding of LDPC codes in [10], which utilizes an adaptive offset factor to improve the accuracy of the soft information transferred during the iterative decoding process,

This paper investigates the relationship between BP algorithm and MS algorithm and finds the reason of degradation. IV. RESULTS AND DISCUSSION
The hardware of proposed decoder is successfully implemented with static timing improvement method [15]-[17] in Verilog. The hardware architecture is fully verified at RTL level and synthesized with Intel 14nm technology lib.
The simulation were carried out for LDPC NR 3GPP 38.212 release. The LDPC decoding algorithm max iteration numbers are all set to 50. The code length are all set to 3824 and the code rate set to 0.75 and 0.5 to simulate the LDPC BG1 and BG2‚Äôs different performance. The SNR is set from -10dB to 5dB and at each dB it runs for 10000 times to get an average value for BER comparison among BP, Min-sum, degree match decoder from paper [9], layered off-set from paper [10], proposed decoding algorithm. All the algorithms are simulated as fixpoint decoder besides BP to compare in a realistic hardware needs. The BP is used as the benchmark. The BER result of code rate 0.75 is shown in Fig. 7 and code rate 0.5 in Fig. 8.

when SNR is less than 0, the fix-point operation can be simplified in hardware implementation but it does not bring any improvement to BER performance. When SNR is greater than 0, different decoders are showing different performance. The MS decoder has the worst performance in BER. The layered off-set from paper [10] provided some improvement in BER but still the offset value is based on soft bits and no section degreed, so the performance is better than MS but not good enough. The degree match decoder from paper [9] provided the sectional method but didn‚Äôt consider the different section‚Äôs requirements, so the speed of convergence is worse than BP algorithm. The proposed decoder uses fix-point operation and section degree match approximate method, considering different requirements of decoder in different SNR environment and its BER performance is the closest one to the benchmark among all the decoders, especially when SNR is close to 0 section. This is because the newly introduced sectional degree matched approximate correction focuses on the LLR range from ‚Äú0‚Äù to ‚Äú1‚Äù, which is the most frequent situation in Gaussian noise environment.

That‚Äôs because the noise situation is smoother than with BG1 and degree match is more suitable. The proposed decoder already considered the worst case and designed to have different sections‚Äô solutions for different environments. So the performance of BER is still better than other decoders.

V. CONCLUSION
A sectional degree match approximate Min-sum decoding algorithm for LDPC is proposed in this paper. The decoding algorithm uses the check nodes degree as reference to optimize the calculation and finds three simplified approximation correction algorithms for different LLR value sections to realize this proposal. The proposed algorithm can be easily implemented in hardware. It is verified at RTL level and synthesized with Intel 14nm technology lib and the results show the proposed decoder has better performance in BER and speed of decoding, with minor hardware cost.
The future work will focus on the hardware implementation optimization.
