Improving the Burst Error Tolerance of Irregular LDPC by Optimizing the Parity-Check Matrix Column Arrangement
for 50G-PON Upstream Transmission

Mingwei Yang1, Linlin Li2, Xiang Liu2, Frank Effenberger2, and Ivan B. Djordjevic1
1. Department of Electrical and Computer Engineering, University of Arizona, Tucson, AZ 85721, USA 2. Futurewei Technologies, Bridgewater, NJ 08807, USA mingweiyang@email.arizona.edu; linlin.li@futurewei.com

Abstract: We demonstrate a novel technique to improve the burst-error-correction performance of the irregular LDPC code used for 50G-PON upstream transmission by optimally rearranging the parity-check matrix columns. FPGA-based real-time measurements show over 0.2 dB gains.
OCIS codes: (060.4250) Networks; (060.2330) Fiber optics and optical communication.

1. Introduction
Forward error correction (FEC) with high coding gain is needed to achieve PR30 (29dB) link budget in 50G-PON. Owing to its high performance, irregular low-density parity check (LDPC) FEC has been actively studied for nextgeneration passive optical networks (PONs), such as 50G-PON being standardized by both ITU-T and IEEE [1-4]. With hard decision (HD), a bit error ratio (BER) threshold of about ~10-2 can be achieved at a rate of 0.849 [1-2]. In our previous study, we demonstrated that soft decision (SD) can offer a performance gain of about 1.3 dB [3]. It was also found that the performance of the SD-LDPC is well maintained for non-uniform input bit error distributions where the errors are concentrated at the beginning of each codeword [4,5]. Here, we show that by optimally rearranging the parity-check matrix columns, the burst mode performance can be noticeably improved. This simple design improvement is obtained without affecting the complexity or latency of the LDPC encoder and decoder, making it beneficial for 50G-PON burst-mode upstream transmission.

The bit log-likelihood ratio (LLR) values calculated from the channel are stored in the LLR memory (the Lv module in the LLR min-sum processor). In the LLR min-sum processor, the variable node processor (VNP) performs the sum operation, while the check node processor (CNP) performs the finding minimum values. The output of CNP updates a posteriori probability (APP) memory (the Lcv module in the LLR min-sum processor) which is used in the next iteration. To address the low latency requirement, we limit the maximum iteration number to 15. The loop state controller is to control the operation of the VNP-CNP and indicates the layer index to the AddrGenerator module that is used to generate the cyclic address in each block layer-by-layer. The output of the sum module is sent to the error counter module to check the post-FEC BER value. By using compressor module and expand module, a 32-degree binary tree is employed to process a 69 blocks parallelly with maximum column weight of 24. The process in LLR min-sum processor takes only 13 clock cycles in FPGA. The real-time processing speed is about 0.1 Gb/s. To achieve an output BER of 10-12 with about 500 output errors counted, the real-time measurement needs to run for about 30 days. The power and resource utilization of the platform are depicted in Fig. 1(d) and Fig. 1(e) respectively.

An example of the input BER distribution with the same average input BER and different attenuation values are depicted at Fig. 2(a). These three case studies correspond to attenuations of 0dB, 2dB, and 3dB and all provide the average input BER of 0.027. The rearranged matrix HtypeB(1:69)=[HtypeA(18:57), HtypeA(1:17), HtypeA(58:69)] is depicted in Fig. 3(b). The original 12 columns of parity checks are unchanged to make the decoding complexity unaffected. Figure 3(c) shows another H-matrix (HtypeC) rearranged in the opposite way where HtypeC(1:69)=[HtypeA(58:69), HtypeA(1:17), HtypeA(18:57)], to explore the worst-case burst-mode performance.
3. Experimental Results and Discussion
Based on FPGA measurement, the error correcting performance of HtypeA, HtypeB, and HtypeC are depicted in Figs. 4 (a), (b), and (c), respectively. For HtypeA for the output BER of 10-9, the required input BERs are 2.6×10-2 (Q=5.77dB), 2.5×10-2 (Q=5.85dB), and 2.42×10-2 (Q=5.91dB), for ΔSNR of 0dB, 2dB, and 3dB, respectively. The above result indicates that the SD-LDPC performance is degraded by ~0.13 dB when ΔSNR is increased from 0dB to 3dB, indicating potential burst-mode penalty. For HtypeB for the output BER of 10-9, the required input BERs are 2.6×10-2 (Q=5.77dB), 2.66×10-2 (Q=5.73dB), and 2.72×10-2 (Q=5.68dB), for ΔSNR of 0dB, 2dB, and 3dB, respectively. No error floor observed before 10-12 at ΔSNR of 3dB. Remarkably, the BER performances with nonuniform input BER distribution are better than the performance with uniform input BER. The SD-LDPC performance is improved by ~0.09 dB when ΔSNR is increased from 0dB to 3dB, indicating improved tolerance to burst errors. At the same condition of at ΔSNR=3dB, the HtypeB provides a performance gain of ~0.22 dB over the HtypeA. As discussed before, the first 1/8 variable nodes provide less belief information while the following 7/8 variable nodes provide more belief information. Note that for regular LDPC codes, the variable nodes provide same belief information, which is different from the irregular LDPC codes. The HtypeC performs the worst in the presence of the burst errors, further confirming the reason behind the dependence of the burst-error-correction performance upon the parity-check matrix column arrangement for irregular LDPC.

4. Conclusion

Through real-time measurements, we have found that the burst-error correction performance of the LDPC code that is currently under study by ITU-T’s 50G-PON group can be noticeably improved by properly rearranging the columns of its parity-check matrix such that the beginning parity-check matrix columns have the smallest weight (of 3). This simple design improvement can be obtained without affecting the complexity or latency of the LDPC encoder and decoder, making it beneficial for 50G-PON burst-mode upstream transmission for next-generation optical access networks.
