On the error floor of the code-candidate for CCSDS standard

Abstract—In this paper we propose a simulation system for LDPC codes implemented on a field-programmable gate array device. The presented system includes an LDPC encoder, Gaussian noise generator and an LDPC decoder. An LDPC encoder and decoder are fully flexible and support different constructions of LDPC codes. It makes possible to obtain performance curves of a large number of LDPC codes and analyse their properties. We have found that the LDPC code proposed in [1] has an error floor higher than the LDPC code proposed by the experimental specification CCSDS 231.1-O-1. We have evaluated the performance of 2250 (128,256) LDPC codes with our simulation system and have found an LDPC code that outperforms the LDPC code from the experimental specification CCSDS 231.1-O-1.
Index Terms—Low-density parity check codes(LDPC), fieldprogrammable gate array(FPGA), bit-error rate (BER), experimental specification CCSDS.
I. INTRODUCTION
The problem of design and development of effective communication systems for small satellites (nanosatellites) is more and more relevant. The CCSDS Experimental Specification 231.1-O-1 [2] has been developed to deal with that issue. There is a demand for the development of communication systems that reliably achieve a bit error probability equal to 1e12 or less. Low-density parity-check (LDPC) codes [3] are a class of linear block codes that are known to provide excellent decoding performance. That’s why they were chosen for the control communication channel. However, the LDPC codes have an error floor problem what means that the BER performance curve does not decreases obviously with the increase of the SNR. A lot of works were dedicated to the study of the error floor phenomenon ([4], [5], [6], [7],[8]). Unfortunately, the proposed methods can’t estimate the upper bound for the error floor region, and therefore can’t guarantee achieving the required probability. Thus, to date, the only method of validation for achieving the required decoding error probability is simulation. Using software simulation to calculate the low error probabilities (1e-12 or less) is inappropriate due to the extremely long simulation time required. For that reason it is necessary to develop a flexible hardware implementation of a simulation system on FPGA. There are different iterative

decoding algorithms that are mainly classified as in hard decision decoding and soft decision decoding. Hard decision decoder can be easily implemented on FPGA but has poor performance. Soft-decision decoding gives enhanced performance in decoding procedure of LDPC codes [9], [10]. The Sum-Product algorithm is a soft decision message-passing algorithm. In practical hardware implementations the MinSum algorithm [11] is usually preferred to the Sum-Product one because it requires a simpler architecture and reduces the implementation complexity of the decoder. In the our simulation system we have implemented normalized min-sum algorithm [12]. Our goal was to design a flexible LDPC decoder, that supports different types of codes and enable to test different codes without redesigning the decoder. To study is the ”error floor” phenomena of LDPC codes it is necessary to use a hardware Gaussian noise generator with high generation rate and high Gaussian tail accuracy. For our research we use a GNG hardware implementation proposed in [13].
II. SIMULATION SYSTEM
We have implemented the simulation system depicted in the Fig.1. It includes flow control unit, statistics unit, LDPC encoder and decoder, BPSK and Gaissian noise generator.
A. Flow control unit
That unit determines an LDPC code that we are going to simulate. It writes a generator matrix G and a word a to the LDPC encoder unit and a parity-check matrix H to the LDPC decoder unit. It is also regulates Gaussian noise power.
B. LDPC encoder
The sequence of the bits a1, . . . , ak and the generator matrix G from the flow control unit are mapped to the bit c1, . . . , cn in the LDPC encoder by multiplying message bits with generator matrix, i.e c = aG. The LDPC encoder implementation on FPGA is simple since it requires only XOR structures that performs modulo-2 operations.

C. Gaussian noise generator
The codeword c1, . . . , cn is modulated using binary phaseshift-keying (BPSK) to x1, . . . , xn (xi ∈ {−1, 1}). The sequence x1, . . . , xn is added with white Gaussian noise z1, . . . , zn. The additive white Gaussian noise (AWGN) channel is an important block in the design of communication systems. The Gaussian Noise Generator (GNG,[13]) core generates white Gaussian noise of standard normal distribution, which can be used to measure BER to extremely low BER levels (≈ 10−15). The period of generated noise sequence is about (2176). The core uses a 64-bit combined Tausworthe generator [14], [15] and an approximation of the inverse normal cumulative distribution function, which obtains a PDF that is Gaussian to up to 9.1σ. Noise is quantized to 16 bits with 4 bits of integer, 11 bits of fraction and 1 bit for sign.
D. LDPC decoder
Let H be the parity check matrix of the LDPC code with n columns and r rows. The NMSA is based on passing messages in the Tanner graph [16] with two set of nodes. Let Vj and Ci be the following sets:
Vj = {i, Hi,j = 1},
Ci = {j, Hi,j = 1}
The sequence x1, . . . , xn is added with white Gaussian noise and the resulting sequence r1, . . . , rn goes to the decoder input. The proposed decoder uses r Check Nodes Units (CNUs) and n Variable Nodes Units (VNUs) that is for each check node in the Tanner Graph there is a CNU, and for each variable node there is a VNU. Every VNU and CNU is implemented separately in hardware what allows to reach the highest speed possible among all types of decoders. E. Statistical unit
Statistical unit compares the outputs of the LDPC encoder and the LDPC decoder and calculates the number of bit errors. When the number of errors achieves a fixed level, the statistical unit sends the number of errors and the number of transmitted bits to the flow control unit. The control flow unit stores the received information and then sets up a new value of Gaussian noise power.
III. NUMERICAL RESULTS

The operator ⊕ indicates modulo-2 addition.
In the [1] we have proposed new LDPC code with better energy efficiency than one from the experimental specification CCSDS 231.1-O-1 under the same decoding algorithm. By means of software simulation we have shown the proposed LDPC code has the better decoding threshold. By means of hardware simulation it has been shown that this LDPC code has the worse decoding performance in the error floor region. Then, we have simulated 2250 (128,256) LDPC codes and have found an LDPC code that outperforms the CCSDS LDPC code the whole simulated SNR range. IV. CONCLUSION
We have implemented the hardware simulation system for LDPC codes. That system is intended for quick modeling of LDPC codes constructed with different methods. Our system has a hardware-based Gaussian noise generator that is an important component for exploring LDPC code behavior at very low bit error rates. That system is a tool for searching LDPC codes with good properties and for developing of new methods for LDPC code construction.
