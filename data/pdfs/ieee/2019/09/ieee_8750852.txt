Automatic Verification of GCD Constraint for Construction of Girth-Eight QC-LDPC Codes
Guohua Zhang , Yi Fang , Member, IEEE, and Yuanhua Liu

Abstract— As a general method to yield constructions for quasi-cyclic (QC) low-density parity-check (LDPC) codes with girth eight, the greatest-common-divisor (GCD) framework heavily relies on verifying a type of inequalities, referred to as GCD constraints. An algorithm is developed in this letter to automatically verify GCD constraints without conducting any manual analysis, by bounding from above the GCD of a fixed integer and an integer in the form of linear function. As an application of the algorithm, a set of novel constructions based on GCD framework is proposed. From these new constructions, four novel bounds on the size of circulant permutation matrices (CPMs) are formulated, such that girth-eight QC-LDPC codes always exist for any CPM size greater than or equal to the bounds. The new bound for column weight of 6 slightly improves the existing best one, and those for column weights from 7 to 9 significantly strengthen the state-of-the-art ones by decreasing from essentially a cubic or biquadratic power of row weight to a quadratic power of row weight.
Index Terms— Circulant permutation matrix (CPM), girth, greatest common divisor (GCD), low-density parity-check (LDPC) codes, quasi-cyclic (QC).
I. INTRODUCTION
A (J, K)-REGULAR low-density parity-check (LDPC) code is defined as the null space of a sparse parity-check matrix (PCM), in which each column (resp. row) has weight J (resp. K). A PCM composed of circulant permutation matrices (CPMs) yields a quasi-cyclic (QC) LDPC code [1]. Recently, some explicit constructions (without computer search) [2]–[6] and the greatest-common-divisor (GCD) framework [2], [5] in particular, have attracted much attention, since they can yield short QC-LDPC codes with girth exceeding six (i.e., at least eight) which in turn can be more easily utilized as basic modules to obtain moderate/long QC-LDPC codes with nondecreasing girth.
Although the GCD framework is an efficient method to construct QC-LDPC codes with girth at least eight, it relies on a complicated and tedious analysis procedure of verifying the GCD constraint [2], [5]. For a QC-LDPC code with column weight J, the number of GCD constraints (denoted by ngcd)

Moreover, for J ≥ 5, one often has to distinguish different types of K [5] to obtain girth-eight QC-LDPC codes under the GCD framework, and hence the number of GCD constraints to be verified increases several times. For example, for J = 7, even if the number of types of K is optimistically assumed as 3, ngcd = 3 × C(J, 3) has become a somewhat prohibitive quantity (105) in the sense that the manual analysis as in [2], [5] is labor intensive and thus the result tends to be highly unreliable. LDPC codes with a large column weight generally have bad thresholds according to density evolution, and the decoding complexity for them is relatively high. Nevertheless, they can be served as a good starting point to construct LDPC codes with small column weight by utilizing some girth-preserving approaches, such as column splitting (CS) [7].
In this letter, a novel algorithm is devised to completely avoid the manual analysis of verifying the GCD constraint, for the most common scenarios of GCD framework. As a significant application of the proposed algorithm, a set of new methods for J = 6 ∼ 9 is discovered, from which four slightly or enormously improved bounds on CPM size are obtained.
II. PRELIMINARIES
We consider a type of exponent matrices with a special structure, E(a0, · · · , aJ−1) being a J × K exponent matrix defined by [2] E(a0, · · · , aJ−1)(i, r) = ai · r, where 0 ≤ i ≤ J − 1, 0 ≤ r ≤ K − 1, and {a0, a1, · · · , aJ−1} are J integers satisfying 0 ≤ a0 < a1 < · · · < aJ−1. The PCM is obtained from the exponent matrix by replacing each entry ai · r with a P × P CPM where the nonzero element ‘1’ in the zeroth row is located in the mod(ai · r, P )-th column.
The recently proposed GCD framework provides a very flexible way to construct QC-LDPC codes with girth eight.
Lemma 1 [2]: Let i, j and k be three different row indexes (0 ≤ i < j < k ≤ J − 1). If (ak − ai)/gcd(ak − ai, aj − ai) ≥ K for all triples (ai, aj, ak) (referred to as GCD constraint), then E(a0, · · · , aJ−1) corresponds to a QC-LDPC code with girth at least eight for any P ≥ (aJ−1 − a0)(K − 1) + 1.
Actually, as noted in [5], the GCD framework yields QC-LDPC codes with girth exactly eight.
In the following, we focus on E(a0, · · · , aJ−1), where ai (0 ≤ i ≤ J − 1) takes a special form, i.e., ai = αiK + βi (αi and βi being nonnegative integers). This enables us to conceive an algorithm capable of automatically verifying the GCD constraint without conducting any manual analysis.

The new algorithm relies on bounding from above the GCD of a fixed integer c and an integer in the form of linear function ax + b, where a, b and c (b < c if c = 0) are three known nonnegative integers but x is an unknown integer. There are two cases, for which gcd(ax + b, c) can be exactly obtained without the knowledge of x. (case 1): If c = 0, then gcd(ax + b, c) = ax + b; (case 2): If c = 0, c|b and c|a, then gcd(ax + b, c) = c. For the general case (c = 0), as gcd(ax + b, c) cannot be exactly calculated when x is unknown, an upper bound of gcd(ax+b, c) as tight as possible is mostly preferred. Specifically, we have (case 3): if c = 0, then a parameter gupp can be calculated from the triple (a, b, c) via Algorithm 1.
In Algorithm 1, gcd(u, v), f actor(u) and union(U, V ) stand for functions which return the greatest common divisor of u and v, a row vector containing all the prime factors (including repeated factors) of u, and the combined data from vectors U and V with no repetition, respectively. The operation sum(U == v) is defined as the number of entries in the vector U equaling the integer v.
Lemma 2: gupp yielded by Algorithm 1 satisfies gupp ≥ gcd(ax + b, c), regardless of x.
Proof : The main idea of Algorithm 1 relies on two observations: (1) an upper bound on gcd(ax + b, c) can be obtained by deleting some properly selected factors from c; (2) if a prime p occurs s1 and s2 (0 ≤ s2 < s1) times in the factorizations of gcd(a, c) and gcd(b, c), respectively, then p occurs s2 times in the factorization of gcd(ax + b, c). In other words, the other s3 − s2 factors (all being p) should be deleted from the factorization of c to yield a tighter upper bound on gcd(ax + b, c), where s3 (s3 ≥ s2) is the number of occurrences of p in the factorization of c. From Table V, we observe that: (i) the new bound for J = 6 slightly improves the existing best one [5]; and (ii) except the case of J = 7 and K = 7 where the existing bound and the new one yield the same lower bound (i.e., 331), the new bounds for J = 7 ∼ 9 significantly improve the state-of-the-art ones [6], [5] by decreasing from essentially a cubic/biquadratic power of K to a quadratic power of K.
Remark 2: For J = 6, girth-eight QC-LDPC codes with much smaller CPM sizes can be obtained by some search-based approaches, such as [8]. For example, for K = 7, the CPM size by Theorem 1 is 139 whereas the one in [8] is 70; for K = 8, the CPM size by Theorem 1 is 176 whereas the one in [8] is 95. Besides, in [9], a technique is proposed to reduce the search space in obtaining QC-LDPC codes with the smallest CPM size. Nevertheless, our constructions for J = 6 (and J = 7 ∼ 9) are explicit in the sense that they can be expressed by simple formulas and can be applied to any K.
V. APPLYING COLUMN SPLITTING TO SHORT GIRTH-EIGHT QC-LDPC CODES WITH LARGE COLUMN WEIGHTS
It is well known that column splitting (CS) [7], as a general girth-preserving skill, can be utilized to generate a longer target code with a smaller column weight from a shorter base code with a larger column weight. For the scenario of QC-LDPC codes, CS is obviously only applicable to the base QC-LDPC code with column weight ≥ 6 so as to guarantee a target QC-LDPC code with column weight ≥ 3. To the best of our knowledge, CS has not yet been employed to the base QC-LDPC code with girth eight, due to the difficulty of constructing a short girth-eight base QC-LDPC code with a large J (J ≥ 6). Obviously, each construction in Section IV yields such a base code for CS method. In the following, two examples are introduced to illustrate the application of short QC-LDPC codes with girth eight and a large J in CS method.

VI. CONCLUSION
By bounding from above the GCD of a fixed integer and an integer in the form of linear function, a novel algorithm has been put forward to avoid the tedious and intricate verification of GCD constraint for the most common scenarios of GCD framework. As an application of the algorithm, a set of constructions for girth-eight QC-LDPC codes with large column weights has been proposed. From these methods, four novel bounds on CPM size have been formulated, where the latter three bounds greatly improve the state-of-the-art ones by decreasing from a cubic/biquadratic power of row weight to a quadratic one of row weight.
