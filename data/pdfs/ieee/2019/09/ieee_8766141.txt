An LDPC-Coded SCMA Receiver With Multi-User Iterative Detection and Decoding
Wei-Cheng Sun , Yu-Chieh Su, Yeong-Luh Ueng , Senior Member, IEEE, and Chia-Hsiang Yang , Senior Member, IEEE

Abstract‚ÄĒ This paper presents the first low-complexity realization of an LDPC-code sparse code multiple access (SCMA) receiver with a high-throughput LDPC decoder and a multi-mode SCMA detector. The minimum mean-square error with parallel interference cancellation (MMSE-PIC) algorithm is adopted in the SCMA detection. The modified user-node operations in the MMSE-PIC-based message-passing detector improve the convergence rate in error performance. The proposed receiver also supports multi-user iterative detection and decoding (MU-IDD) to improve the error rate performance. The proposed receiver supports both 4√ó6 and 8√ó12 SCMA systems. The proposed MU LDPC decoder has a 57.1% lower hardware complexity than the direct-mapped design that is achieved through hardware sharing and memory access scheduling. Designed in a 40-nm CMOS technology, the SCMA receiver integrates 10.9M logic gates in an area of 3.382 √ó 3.382 mm2. The proposed design achieves a gross throughput of 1.198 Gb/s and 599 Mb/s for 8 √ó 12 and 4 √ó 6 SCMA systems, respectively, under a practical situation. It dissipates 813 mW at a clock frequency of 300 MHz from a 0.9-V supply.
Index Terms‚ÄĒ Sparse code multiple access (SCMA), low-density parity-check (LDPC), iterative detection-anddecoding (IDD), fifth-generation (5G) communication, CMOS digital integrated circuits.

In the 4G system, orthogonal frequencydivision multiple access (OFDMA), which assigns each user to subsets of sub-carriers, is a key technology for avoiding multiple access interference. However, the spectral efficiency of OFDMA is limited by the carrier spacing in order to maintain the orthogonality between sub-carriers.
In order to meet 5G requirements, non-orthogonal multiple access (NOMA) has emerged as a promising solution for improving spectral efficiency [3]. NOMA serves more users via the same communication resource by allowing acceptable multiple access interference. Two promising NOMA schemes, including power-domain NOMA [4] and sparse code multiple access (SCMA) [5], [6], are currently being considered. The power-domain NOMA technology utilizes superposition coding to transmit signals. In contrast, SCMA is a codebookbased NOMA technology, in which information bits from different users are mapped to pre-defined sparse codewords and then transmitted over time-frequency dimensions called ‚Äėtones‚Äô. The SCMA can achieve a higher sum rate (bps/Hz) than the power-domain NOMA, but at the expense of higher complexity [7]. In [8], a maximum a posterior (MAP)-based message passing (MP) algorithm was proposed for SCMA detection. However, the algorithm requires excessive computational complexity and no feasible hardware mapping has been presented in the open literature, even for an SCMA system that has four frequency bands and six users (a 4 √ó 6 SCMA system).
An SCMA detector can be concatenated with an outer channel decoder to enhance the transmission reliability. Channel codes (including polar code, non-binary low-density paritycheck (LDPC) code [9], and binary LDPC code [10]) can be used in the modern communication systems. Since the binary LDPC code can achieve better error-rate performance and is hardware-friendly for high-throughput realization when the code length is moderate, it has been adopted in many communication and storage applications [11]‚Äď[15]. Therefore, the binary LDPC code is used as the outer channel coding scheme for the SCMA system in this work. In addition, an LDPC-code SCMA receiver can further improve the error performance through multi-user (MU) iterative detection and decoding (MU-IDD) [8]. However, there are several design issues in the hardware implementation for such an MU-IDD receiver. For example, designing an efficient interface between the LDPC decoder and the SCMA detector to achieve efficient message exchange is challenging, meaning that both the algorithm and the architecture need to be jointly

For wireless communications, computational complexity and power consumption are key issues. The lower computational complexity translates into lower power dissipation. The MAP-based MP algorithm proposed in [8] has a prohibitively high computational complexity, especially when the size of SCMA codebook increases. In this work, hardwarefeasible minimum mean-square error with parallel interference cancellation (MMSE-PIC) detection [16] is considered instead. However, the error performance of the direct-mapped MMSE-PIC-based MP detection algorithm is not satisfactory. A global MMSE-PIC detection scheme and an MP detection scheme with modified user node operations are proposed to improve the error performance. An LDPC decoder architecture designed to efficiently process multiple data streams for SCMA systems is proposed. The interface between the SCMA detector and the LDPC decoder is optimized to reduce the processing latency. With its reconfigurable architecture, the proposed receiver is sufficiently flexible to support both 4 √ó 6 and 8 √ó 12 LDPC-coded SCMA systems. The contributions of this work include:
‚ÄĘ The first VLSI architecture for an LDPC-coded SCMA receiver that integrates an MMSE-PIC based SCMA detector and an MU LDPC decoder.
‚ÄĘ Two feasible MMSE-PIC based SCMA detection schemes, including global detection and MP detection. For MP detection, the user-node operations are devised such that the convergence rate in error performance can be increased.
‚ÄĘ An IDD scheme that achieves a better error performance compared to the non-IDD MAP-based MP detection scheme, despite the lower complexity.
‚ÄĘ A multi-mode processing unit (PU) for CORDIC-based QR decomposition (QRD) that achieves a reduction

in complexity of 59.2% compared to the conventional design. ‚ÄĘ An MU LDPC decoder architecture that can efficiently process multiple data streams for both 4 √ó 6 and 8 √ó 12 SCMA systems with minimized hardware overhead. ‚ÄĘ An IDD interface that can efficiently exchange soft information between the SCMA detector and the MU LDPC decoder with 100% hardware utilization.
As a proof of concept, the LDPC-coded SCMA receiver is implemented using 40nm CMOS technology, supports both 4√ó6 and 8√ó12 SCMA systems. To the best of our knowledge, this is the first application-specific integrated circuit (ASIC) solution for LDPC-coded SCMA systems.
The remainder of this paper is organized as follows. Section II provides an overview of SCMA. Section III introduces the global MMSE-PIC algorithm. The proposed MMSE-based MP algorithm for SCMA detection is also described. Section IV presents the proposed SCMA detector and the LDPC-coded SCMA receiver with MU-IDD. Section V describes the performance of the proposed LDPC-coded SCMA receiver. Finally, Section VI concludes this paper.

C. LDPC Decoding
The Min-Sum-algorithm [18] is popular for LDPC decoding and is also a graph-based decoding method, where the operation nodes can be partitioned into variable nodes (VNs) and check nodes (CNs). The layered LDPC decoding technique [19], where the CN and VN operations are performed layer by layer, is widely used since it can achieve a high convergence speed. Consequently, the layered Min-Sum-algorithm decoding approach is adopted in this paper. At the beginning of the LDPC decoding, L DA,,bj,(i), an a priori LLR for the i -th LDPC code bit for user j , is set to the corresponding i -th VN by using the soft information provided through SCMA detection. Since the LDPC decoding operations for all users are identical, the user index j is not labeled for the Q and R values for simplicity. Let Qti,m denote the variable-to-check (V2C) message from the i -th VN to the m-th CN, and let Rit,m denote the check-to-variable (C2V) message from the m-th CN to the i -th VN in the t-th LDPC iteration. In addition, let IC [i ] denote the set of CNs connected to the i -th VN and let IV [m] denote the set of VNs connected to the m-th CN. In the final iteration, the LDPC decoder outputs L DP,,bj,(i), which denotes the a posteriori LLR for the i -th LDPC code bit for user j . When the layered scheduling is applied to quasi-cyclic (QC) LDPC codes, the CNs are divided into Mb groups and each CN group consists of ő≥ CNs, where ő≥ is the block size for the QC LDPC code. A. System Architecture
Fig. 7 depicts the proposed architecture for the LDPCcoded SCMA IDD receiver. The system includes a multi-mode SCMA detector, and a MU LDPC decoder that consists of four LDPC processing elements (PEs). The Gram matrix HHH and the received vector HHy are first fetched from an input buffer. Then, the stored HHH and HHy are sent to the SCMA detector that calculates the preliminary LLR values for all user bits, and the results are sent to the LLR memory block. After that, the LDPC decoder accesses the LLR values and performs LDPC decoding. The proposed receiver iteratively performs SCMA detection and LDPC decoding. The hard outputs are generated after the IDD is complete.

The study presented in [25] proposes a two-codeword scheduling technique that is able to achieve a 100% hardware efficiency, but at the expense of a doubling of the LLR memory. If twocodeword scheduling is applied to the 8 √ó 12 SCMA receiver, 24 LDPC LLR memory blocks are required, resulting in a tremendous increase in complexity. As a result, two-codeword scheduling is not considered.
When layered decoding is applied to the length-1944 rate-0.5 WiFi LDPC code, only a fraction of the LLR memory blocks is accessed by the LDPC PEs. Based on this observation, the proposed memory access procedure is that those LLR memory blocks not being used in the current LDPC layered processing can be accessed by the SCMA detector and, hence, the detector and the decoder are able to simultaneously access the LLR memory blocks. Compared to the two-codeword scheduling, the proposed memory access only requires 12 LLR memory blocks.
Since the LDPC decoder and the SCMA detector simultaneously read from and write to the LLR memory block at different addresses, two-port memory blocks are used in this study. If the LLR memory blocks are divided into two banks, as shown in Fig. 14(a), the SCMA detector reads the

LLR values from bank 1 for each user and writes the LLR values to bank 2 for each user. Meanwhile, LDPC PE 1 reads the LLR values for user 2 from bank 2 and writes the LLR values for user 1 to bank 2. The SCMA detector and LDPC PE 1 simultaneously write the LLR values for user 1 to bank 2 and, consequently, this memory arrangement encounters collision issues. This straightforward memory arrangement has a collision rate of 32% during the IDD processing and, hence, the hardware efficiency is low.
To prevent data collision during the IDD processing, the LLR memory blocks are divided into three banks, as shown in Fig. 14(b). When the SCMA detector writes the LLR values for each user to bank 2, the LDPC PE 1 writes the LLR values for user 1 to bank 3 and, hence, the collision can be totally avoided. The operations for the other LDPC PEs can be performed in a similar manner. Consequently, the proposed memory access technique can achieve a hardware efficiency of 100%.
Fig. 15 shows the reduction in terms of gate count for the proposed MU LDPC decoder and the proposed memory access. The baseline design is implemented by using the direct-mapped MU LDPC decoder architecture combined with two-codeword scheduling. The MU LDPC decoder achieves a reduction in terms of gate count of 28.6%. In addition, the proposed memory access technique achieves a reduction in terms of gate count of 40%. Consequently, an overall reduction in terms of gate count of 57.1% is achieved for the proposed MU LDPC decoder architecture and the memory access. A high throughput can still be achieved with a minimized hardware cost since the message can be updated simultaneously at both the decoder and the detector, thereby eliminating the idle time caused by data dependency.
