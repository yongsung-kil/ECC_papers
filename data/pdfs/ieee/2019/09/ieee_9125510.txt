Abstract

Employing FPGA emulations, we investigate effects of code length, girth, decoding buffer size on LDPC convolutional codes.
We construct a 26.3% concatenated LDPC-convolutional and BCH code that approaches within 0.79 dB of the Shannon limit, a new record at 1015 BER with hardware-verification.

1 Introduction
Low-density parity-check convolutional codes (LDPC-CC) have been proposed for optical fiber communications for strong waterfall performance [14]. The complex convolutional structure, however, makes it challenging to analytically predict the error-floor behavior of these codes at ultra-low post-forward-error-correction (FEC) bit error rate (BER). Due to the lack of effective theoretical and numerical means, it becomes essential to develop a high throughput emulation platform to assist the code design and optimization through a series of ultra-low BER emulations.
In this paper we implement LDPC-CC schemes on a FPGA platform with a throughput up to 150 Gbps, and optimize the code performance from several aspects including code length, girth, convolutional buffer size, interleaving schemes, and concatenated outer code strength. By performing a series of progressive FPGA emulations, we construct a 26.3%- overhead concatenated LDPC-CC and Bose-Chaudhuri- Hocquenghem (BCH) code that achieves a record performance, a pre-FEC BER threshold of 4.19102 for a post-FEC BER of 1015. The FEC performance is within 0.79 dB of the Shannon limit at 1015 post-FEC BER, which is by far the closest point achieved with hardware verification at this level of post-FEC BER for optical fiber communications.

The received data are decoded in zdc/dv-symbol blocks from top to bottom through the buffer. For a single decoding sweep through the buffer, the number of decoding iterations for a data block equals to the buffer size B. The LDPC-CC performance can be improved with multiple decoding buffer sweeps, which comes with a price of higher complexity and latency. In the decoding implementation, a revised min-sum-algorithm is adopted for the layered decoding [7].

5 Conclusion
Employing a high-throughput FPGA emulation platform, we implement and evaluate a series of LDPC-CC based FEC schemes targeting at a post-FEC BER of 1015. With FPGA emulations we show that for the LDPC-CCs, 1) larger girth does not necessarily improve FEC performance in either the waterfall or the error-floor region; 2) increasing code length is not an efficient way to improve the code performance; 3) the decoding buffer size has a significant impact on the waterfall performance, but it has little impact on the error floors. We show that investigation on LDPC-CC error characteristics is important to design an effective and efficient concatenation scheme for error floor suppression, which is made possible by utilizing a high-throughput FPGA platform. We construct a 26.3%-overhead concatenated LDPC-CC and BCH scheme that assures a post-FEC BER of 1015 at a pre-FEC BER threshold of 4.19102. The FEC performance approaches the Shannon limit within 0.79 dB, which is by far the closest hardware-verified point achieved at 1015 BER in the optical fiber communication field.
