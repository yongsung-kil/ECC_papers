Abstract—Flash memory has been developed with bit density improvement, technology scaling, and 3D stacking. With this trend, its reliability has been degraded significantly. Error correction code, low density parity code (LDPC), which has strong error correction capability, has been employed to solve this issue. However, one of the critical issues of LDPC is that it would introduce a long decoding latency on devices with low reliability. In this case, tail latency would happen, which will significantly impact the quality of service (QoS). In this work, a set of smart refresh schemes is proposed to optimize the tail latency. The basic idea of the work is to refresh data when the accessed data has a long decoding latency. Two smart refresh schemes are proposed for this work: The first refresh scheme is designed to refresh long access latency data when it is accessed several times for access performance optimization; The second refresh scheme is designed to periodical detecting data with extremely long access latency and refreshing them for tail latency optimization. Experiment results show that the proposed schemes are able to significantly improve the tail latency and access performance with little overhead.
I. INTRODUCTION
NAND flash memory has been widely deployed as secondary storage for its many advantages, such as high performance, low cost, and light form factors [1][2]. During the last decades, flash memory has been developed with several schemes: First, the bit density is increased from 1 bit per cell, to the most recent 4 bit per cell [3][4]. Second, the technology size is reduced to the most recent 10nm [5]. Third, 3D NAND flash is designed to stack storage cell from 24 layers to the most recent 128 layers [6][2][3]. In prediction, the number of layers will hit more than 256 in the next few years [7]. However, one of the critical challenges with the development is the degraded reliability of flash memory [8]. In addition, the reliability is impacted by many factors, such as process variation [9][10], retention time [11], program/erase cycles [12] and temperature [13]. From the past decade, BCH (Bose, Ray-Chaudhuri and Hocquenghem), which is a constant error correction code (ECC), used to be adopted to solve the reliability issue [14]. Recently, LDPC (low density parity code), which is a high cost ECC with strong error correction capability, has been the default ECC scheme for the state-ofthe-art flash memory based storage devices [15]. One of the distinct characteristics of LDPC is that it is a probability based ECC, which needs a long decoding latency when the device has a low reliability [16][17][18]. In this case, it would induce flash memory to be low performance devices and sacrifice the quality of service (QoS). This paper will propose approaches

to improve the tail latency for QoS optimization on LDPC based storage systems.
The reason that LDPC needs a long decoding latency is as follows: LDPC is a probability based ECC. When the reliability is low, the data will be retried multiple times to acquire an accurate input information for decoding [15][17]. In order to solve this issue, several approaches have been proposed to improve the access performance [16][17][18][15][19]. These approaches can be classified into two types: optimize the error correction capability of LDPC [19][15][20] and improve the reliability of flash memory [16][17][18]. For the first type, most of them proposed to retry the data as accurate as possible with well selected sensing voltages during retry [19]. However, with the reliability degradation, the access performance will become worse and the long access latency will happen. For the second type, most of them proposed to refresh the data by in-place charging [14] or out-of-place updates [18][11]. With this approach, the data will be recovered to a fresh state with low bit error rates. One of the critical challenges for the refresh scheme is its cost, which not only introduces additional read and write cost, but also takes in wearing to the memory. Most of the proposed refresh schemes are proposed to only refresh the data which has high benefit to the overall performance or to the overall lifetime [18][11]. For example, Cai et al. [14] proposed to refresh data when it is approaching the error correction capability of ECC to improve lifetime. Du et al. [18] proposed to refresh data with both long access latency and high access frequency for read performance improvement. However, this work was not able to avoid the long access latency to the low access frequency data, which will introduce tail latency. This work will also exploit refresh scheme for LDPC based flash memory. Differently, we propose to use refresh to avoid the long access latency and improve the QoS of flash memory.
In this work, a smart refresh framework is proposed to improve the tail latency of LDPC based flash memory. The basic idea of the framework is to refresh data before it needs a long access latency. In order to achieve this objective, we need to define the long access latency. For example, if the latency will introduce significant performance reduction, it may introduce tail latency. What’s more, if the latency is extremely long, tail latency will happen. Based on these two understandings, the proposed refresh scheme includes two parts: The first part is to refresh data with a long access latency and high access frequency. The second part is to refresh data

• Presented evaluation for the proposed framework, showing encouraging performance improvement with little overhead.

II. BACKGROUND
A. Reliability Characteristics of Flash Memory
Flash memory is designed with floating gate (FG) or charge trapping (CT) based memory cell [3]. FG has been widely employed in planar NAND flash memory [1] and CT has been widely employed in 3D NAND flash memory [4]. Both FG and CT are designed to store data based on the amount of electronics stored in the cell. To store more data bits in a cell, the number of stored electronics should be differentiated into multiple levels. Based on the number of bits per cell, flash memory can be grouped into SLC (single level cell - 1 bit in a cell with 2 voltage levels) , MLC (multiple level cell - 2 bits in a cell with 4 voltage levels) , TLC (triple level cell - 3 bits in a cell with 8 voltage levels) [2], and QLC (quota level cell - 4 bits in a cell with 16 voltage levels) [4]. In addition to add more bits in a cell, the memory cell size is reduced to improve the density. In this case, the number of electronics stored in a cell is reduced. During the last decade, 3D stacking NAND flash memory is developed to improve the memory density by stacking memory cell vertically [6][2][3]. With

these approaches, the state-of-the-art NAND flash memory density has approaching several tera bits per chip and its cost is approaching the hard disk drives.
Even with the significant improvement, the reliability of NAND flash memory has been degraded especially for the high density memory. The critical issue of FG or CT based memory cell is that they are prone to errors [15]. As above stated, the data in the memory cell is represented by the amount of electronics. Different number of electronics will induce different sensed voltages. If there are some electronics leaking from the cell or charging to the cell, the data may be wrong. Take Figure 1 as an example, which shows a voltage distribution for a TLC flash cell. For the cell, there are 8 voltage ranges for different data and 7 reference voltages are used to differentiate these data. The above simple model can be impacted by many factors in real life [12][8][11]. For example, retention time and temperature can introduce electronic leakage, and program/erase (P/E) cycling, read disturb, write disturb and cell-to-cell interference can introduce electronic charging. In this case, the voltage distribution will cross the reference voltage and overlap with their neighbors. As shown in the figure, the green line represents the electronic leakage and the red line represents the electronic charging. For retention time, it represents the time that the data can be stored reliable in the memory cell. During retention time, the electronics can be leaked from the cell, which has been identified as one of the dominate leakage factors. This is especially true for 3D NAND flash, due to the characteristic of CT [3]. Previously, there are many schemes proposed to optimize the errors during retention time [11][17]. Other critical impact factors include P/E cycling and disturb, which can introduce cell charging [12][8]. All of these impact factors will widen the voltage distributions and introduce errors.

The above issues can be amplified with the bit density improvement, especially with increasing the number of bits per cell and cell size scaling. As shown in Figure 1, the maximal threshold voltage of cell is fixed due to the size of the cell. With 8 voltage states, the noise margin among these voltage levels are compressed. In this case, once there are a little number of electronics leaked or charged to the cell, the state will be covered with its neighbors. If there are 4 bits per cell for QLC, the noise margin will be further compressed and the reliability will be further degraded. From this discussion, the critical issue for NAND flash development is to solve its reliability issue.

LDPC uses iterative belief-propagation to decode the input information. With multiple iteration, if the decoding failed, the input information should be retried from the memory. As shown in Figure 2, there are checking nodes and variable nodes in the graph. By transferring messages between these two types of nodes with multiple iteration, if the variable nodes pass the decoding process, it succeed. Otherwise, the input information is retried based on the predefined voltage levels. Figure 3 shows the retry voltage distribution. As shown in the figure, there are 7 voltage levels between two states. With adding more retry voltage, the input information can be more accurate. Then, the decoding process can be succussed with higher probability. However, with several times of retry, the decoding latency will be significantly increased. At most, the latency can be increased by 21 times, as shown in previous work [21].

IV. SMART REFRESH FOR TAIL LATENCY OPTIMIZATION
A. Overview
Figure 6 shows the organization of the proposed refresh framework. Based on the above observations, the proposed framework includes two refresh schemes. As shown in figure, the two refresh schemes are implemented in the controller: one for access performance optimization and one for tail latency optimization. Both these two refresh schemes are designed by acquiring the number of retries from LDPC to determine the refresh operation. During data decoding, once the data is successfully decoded, the number of retries is sent to the refresh framework. For the threshold based refresh scheme, if the retry number is higher than a threshold and the data is read for several times, it is refreshed. Otherwise, if the retry number is higher than another threshold, the data is refreshed immediately. For the periodic refresh scheme, it is designed to periodically detect data which has an extremely long access latency, and refresh them. In this follow, the above two schemes will be discussed in details.

1) Determine the first threshold, R1: The R1 threshold is critical on the performance improvement. If R1 is set with a small retry number, then most of data will be accessed with a high reliability and then the access performance will be improved. However, there are two problems for a small R1. First, if R1 is set too small, data will be refreshed frequently. As discussed in the background section, with the development of flash memory, reliability is degraded especially due to the charge leakage during retention time and P/E cycling. In this case, it is easy that data needs soft decoding with several retries. The frequent refresh not only introduces additional read and write operations to the devices, but also introduces wearing, which would be further worse the reliability. However, if R1 is set too large, data with long access latency would significantly impact performance, especially for the frequently accessed data. As shown in the motivation section, R1 can be set to 3 since it has good performance. In order to achieve both performance optimization and cost minimization, R1 is carefully selected and studied in the experiment.
Another critical parameter used during R1 is T , which represents the access frequency of the data. T also has important impact to the cost and performance. A large T means a small cost and a small T means better performance. Many previous work proposed to identify the hot access data with some complex data structures, such as bloom filter, least recent used algorithm and so on. Differently, this work adopts a simple counter to determine it based on one of previous works from Li et al. [16]. They found that once a data is accessed 2 or more times, the data will be accessed with read only in the future. In this work, we set T to 2 for simplify.
2) Determine the second threshold, R2: Different from the first retry threshold, the second threshold R2 is set for tail latency optimization. This is because if the data is decoded

with a large number of retries, it will not only need a long access latency, but also if it is accessed next time, longer access latency will happen. This type of data needs to be refreshed immediately without checking its frequency. Different from R1, the selection of R2 is more sensitive. This is because many data may be accessed only one time. If it is refreshed due to the one access, there will be only cost without benefit. In addition, since the first threshold R1 has been designed for performance optimization, R2 should be set only for extremely long access latency. In this work, R2 is set with 5 retries due to its effects on the tail latency. In the experiment, R2 will be studied to show its characteristics.
Based on above design, the LDPC will not only improve performance, but also have improved tail latency. However, there is still a problem not solved for the threshold based refresh scheme. Since R2 is set for the case when data is detected with an extremely long access latency, which may introduce tail latency. If these data are the dominate part, the tail latency will still be there. In order to solve this issue, a second refresh scheme, periodic refresh, is proposed in the next section.
C. Periodic Refresh
The design of periodic refresh is to detect data with extreme long latency and refresh them on demand. The basic process is periodically read all data in the storage device. If the data is higher than a limit, it is refreshed. The limit is set same to R2, as presented in above section. From the basic process, we can find that there are at least two problems: First, periodic refresh needs to read all the data in the flash device. This is high cost especially when the device is busy. Second, if data with extremely long latency is never accessed, there will be no benefit but only cost.
In order to solve above issues, several optimization schemes are proposed. First, for the detecting process, we propose to only read the first page of a flash block. For the state-of-theart flash memory, there are always hundreds to thousands of pages in a block. Example, for the current 3D NAND flash memory [4][2], the number of pages are 1536. If the detected access latency for the first page is smaller than the predefined threshold, the whole block will be skipped. Otherwise, all the valid pages in the block are candidates for refresh. With this approach, the detecting cost can be significantly reduced. For the second issue on no benefit page refreshing, we propose a benefit supported refresh scheme. The basic idea is to record the benefit from last periodic refresh. If there are benefit, the current periodic refresh will be processed. Otherwise, the data will be leaved for future accessed. Then, if it is accessed, it will be refreshed by the threshold based refresh scheme. However, we believe that this kind of data should be small and have little impact to the tail latency.
In addition to above discussion, refresh operation is similar to the garbage collection (GC). Both of them is designed to read data and update them to a new place. With this characteristic, GC and refresh can be combined for cost minimization. For example, if data in a page is collected multiple times, the accumulated errors can be large. In this case, it should be refreshed based on the proposed scheme.

However, since we did not consider GC in this paper, we will leave it as our future work.
D. Implementation and Overhead Analysis
In this section, we discuss the implementation of the proposed refresh framework. As shown in Figure 6, there are two components implemented in the flash controller. For the threshold based refresh scheme, the number of retries need to be acquired for each access. In addition, each page needs to maintain an access counter, F . To maintain the counter, we need to extend the mapping table of flash translation layer (FTL). In this work, we assume page mapping as the default mapping scheme. Then, we use 2 bits as the counter for each entry, which can record 4 accesses to the page. Note that in order to avoid data with multiple accesses during a long time identified as frequent data, the counter is cleaned during refresh or update. For the periodic refresh scheme, all the first pages of all flash blocks are periodically read and decoded by LDPC. If the first page retry number is smaller than R2, the whole block is skipped. Otherwise, the benefit bit is checked. The benefit bit is recorded during last access to the page which is maintained in the mapping table, too.
There are two types of overhead in this design: firmware and storage overhead. First, the firmware overhead mainly comes from the comparison between thresholds and the acquired retry number, which is simple compare operation with negligible overhead. Second, the storage overhead includes the storage of access counter using during threshold based refresh and benefit bit used during periodic refresh. For implementation, there are two bits for access counter and one bit for benefit bit maintained in the mapping entry. Since benefit bit represents that data is accessed from last access, the access counter can be used as benefit bit. Therefore, the storage overhead is negligible overhead with extending the mapping entry with two bits.

This is reasonable based on the workload analysis from previous work [16].
• Threshold: this scheme is evaluated to show the advantage of the proposed threshold based refresh scheme in Section IV-B. In default, R1 is set to 3 and R2 is set to 5. Initially, data is set with 7 sensing levels. Once it is refreshed, its sensing level is set to 3.

B. Results Analysis
1) Tail Latency Optimization: Figure 8 shows the CDF of read latency of the eight workloads for 95 – 100th percentiles.

As shown in the figure, the proposed scheme is able to significantly improve the tail latency. Four conclusions can be made from the results. First, same to the motivation section, traditional scheme has significant long tail latency for all workloads. Compared with the 95th percentile with 99th, the latency is increased by 2-5 times. Second, the most recent work, Du et al. did have benefit on improving the tail latency. For the 95th percentile, the latency is improved by 1.25-9.6 times. Third, compared with Du et al., threshold based scheme is generally slight better. This is because the threshold scheme is able to minimize the long access latency at the first time access. Finally, the proposed scheme is significant better than other schemes in most cases. For example, for STG 0, the tail latency is further reduced by 1.6 times. This is because the periodic scheme is able to refresh data with long access latency. However, the results also show that all refresh schemes have similar effects for several workloads, such as HM 1, RSRCH, WDEV 0 and USR 0. This is because most of accesses in these workloads are hot. In this case, simple refresh is able to significant improve the tail latency. In conclusion, the proposed scheme is able to significantly improve the tail latency of LDPC based storage systems.
2) Read Performance Optimization: Figure 9 shows the normalized read response time for the evaluated 4 schemes. First, the results show that Du et al. is able to significantly improve the average read performance by 43%. Second, the threshold scheme is always better than Du et al. due to that it is able to reduce the first long access latency. Third, the proposed refresh scheme is able to further reduce the read response time by 15.4% compared with Du et al.. This is because there are many long access data for several workloads, such as HM 0, PROJ 0, PRXY 0 and STG 0. However, if most of the accesses are hot, them all the refresh schemes have similar effects.

VI. RELATED WORK
This work proposes to optimize the tail latency induced by the long tail latency from LDPC. It means that with the multiple retries during LDPC decoding, the access latency will be significantly extended. Then, the related work can be classified into two groups: LDPC optimization and tail latency optimization.
Since LDPC has significant impact on the access latency of flash memory, there are a large amount of LDPC optimization schemes [16][15][17][18][19][20]. These work can be classified into two groups: optimize the reliability of flash to improve LDPC and optimize LDPC to improve the reliability. For the first group, the straightforward scheme is to reduce the raw bit error rates (RBER) of flash memory [18][17]. For the dominate errors induced from retention time, several refresh schemes have been proposed in previous work [18][14][11][9]. For example, Cai et al. [14] proposed to remove the errors from flash memory when the employed ECC did not able to correct the errors. Then, the lifetime of flash memory can be improved. Du et al. [18] proposed to refresh the data with both long latency and high access frequency to improve the access performance. Shi et al. [11] proposed to use fix refresh scheme for data with predefined retention time. Some other schemes are also proposed to reduce RBER, such as using fine-grained program scheme [16]. Then, during the reading process, the RBER will be much smaller. However, none of these work are able to optimize the tail latency of flash memory. This is because the retry will happen for LDPC even for above schemes. For example, for the refresh scheme, if the data are not frequently accessed, it will need a long latency once it is accessed. For the second group, there are a large number of work proposed to combine the reliability characteristics of flash memory with the decoding process of LDPC to

optimize the performance [20][15][17]. For example, Wu et al. [20] proposed to exploit the write interference, retention time to optimize the decoding process of LDPC. Du et al. [17] proposed to carefully select the sensing voltage for LDPC. Li et al. [19] proposed to smartly place the sensing voltage of LDPC. All of these work are able to significantly reduce the decoding latency of LDPC by improving the quality of input information. Then, the total access latency will be reduced. However, for the data with low reliability, especially for the state-of-the-art flash memory with QLC and 3D stacking, the long access latency cannot be avoided.
Tail latency is a hot topic in computer system. Recently, there are several works proposed to optimize tail latency of flash memory. The most representative work is from Yan et al. [24]. They found that garbage collection (GC) in flash memory will introduce tail latency. Then, they proposed a set of schemes, including recomputing, round-robin GC and so on, to optimize the tail latency. Some other work also proposed to improve the tail latency from the view of access fairness [25][26]. However, none of these works took the reliability into consideration. The most correlated work is from Kim et al. [26]. They found that the reliability issue in the solid state drives (SSD) may introduce quality of service (QoS) issue. They proposed a holistic approach to selectively employ different types of reliability optimization schemes for QoS optimization. Another correlated work is from Cui et al. [27]. They proposed a latency variation aware scheduling scheme to optimize performance. However, none of these works focus on the tail latency induced from LDPC. In this work, we adopt refresh scheme for tail latency optimization.
VII. SUMMARY AND CONCLUSIONS
LDPC, as one of the strong error correction capability ECC, has been the default configuration for the state-ofthe-art flash memory. However, one problem of LDPC is high decoding cost when the reliability is low, which would introduce extremely long access latency to the flash system. In order to solve this issue, this paper proposed a smart refresh framework, which includes two refresh schemes: the threshold based refresh scheme is designed to improve performance and the periodic refresh scheme is designed to optimize tail latency. The proposed framework is implemented in an popular simulator. Experiment with several real workloads shows that the proposed schemes not only significantly improve the tail latency, but also have benefit on average read response time improvement.
