LDPC Code Design for Delayed Bit-Interleaved
Coded Modulation

†School of Electrical Engineering & Telecommunications, University of New South Wales, Sydney, Australia

Abstract—This paper proposes a method to design low-density
parity-check (LDPC) codes for delayed bit-interleaved coded mod-
ulation (DBICM). In the method, the code variable node (VN)
degree distributions and the assignments of VNs with different
degrees to DBICM subchannels are optimized via two cascaded
differential evolution (DE) steps. In each step, to optimize VN
degree distribution or channel assignment, a parity-check matrix
is constructed, and the associated decoding threshold is calculated
for each element in a generation. In constructing a parity-check
matrix for each channel assignment, we propose a constraint PEG-
like code construction method. Protograph-EXIT is employed to
calculate the decoding threshold for each parity-check matrix. We
apply the proposed method to construct irregular binary LDPC
codes for both 16-QAM DBICM and BICM schemes. Simulation
results demonstrate that the optimized LDPC codes are within 1
dB from the associated capacity limit at a bit error rate (BER) of
10−6. Besides, the LDPC coded DBICM achieves an SNR gain of
0.5 dB to 0.1 dB over BICM counterparts at a code rate ranges
from 0.25 to 0.5.
Index Terms—Delayed bit-interleaved coded modulation, bit-
interleaved coded modulation, low-density parity-check codes.
I. INTRODUCTION
Bit-interleaved coded modulation (BICM) [1] is a pragmatic
approach to combine modulation with channel coding for digital
transmissions. It is shown that the capacity of BICM schemes
with Gray labeling is close to the constellation capacity for
AWGN channels [2]. Recently, a new BICM, namely, delayed-
BICM (DBICM), was proposed in [3]. In DBICM, certain bits
from multiple codewords are mapped onto a symbol, which cre-
ates correlations between codewords. These correlations can be
explored by the receiver to improve the decoding performance.
In [3] and [4], a performance gain of 0.4 dB has been shown
in half 16-QAM constellation for binary LDPC coded DBICM
over LDPC coded BICM. Though LDPC coded DBICM has
considerable performance gains over the BICM counterparts,
to the best of our knowledge, there is no investigation for
constructing optimized LDPC codes for DBICM in the literature.
To design LDPC codes for DBICM, unequal error protection
(UEP) originated from different modulation levels should be
taken into account, which is similar to the LDPC code design for
BICM. In general, this UEP property can be considered and an-
alyzed by using the density evolution for multi-edge type (MET)
LDPC codes [5]. However, it leads to a very high complexity [6].
To simplify the LDPC code design for BICM, MET EXIT chart
was developed in [7] and [8] for regular and irregular binary
LDPC codes. In the MET EXIT chart, bit channels are classified
into reliable and unreliable channels according to the bit channel
capacities. The edges in the Tanner graph of an LDPC code are
also classified into reliable and unreliable edges based on the
channel assignment to the connected VN. If an edge connects
to a VN which is assigned to an unreliable channel, this edge
is called an unreliable edge. Otherwise, it is a reliable edge.
This kind of classification of edges simplifies the MET EXIT
chart analysis. However, the reliability of the edge can also be
affected by the degree of VN that is connected to the edge. But
this has not been considered by the MET EXIT chart analysis
in [8]. Therefore, how to improve the accuracy of the decoding
threshold analysis for irregular binary LDPC coded DBICM and
BICM considering the reliability of both the subchannels and the
VNs is an interesting research problem.
In this paper, we propose an LDPC code design method
for DBICM which generalizes the VN channel assignment in
[7] [8] and employs protograph-EXIT (PEXIT) chart [9] to
obtain the decoding threshold of a given code. In particular,
the bit channels are classified into several types instead of
only reliable and unreliable channels. In each type, the bit
channels have exactly the same bit-channel capacity. This finest
channel type classification accompanied with PEXIT chart leads
to accurate decoding threshold analysis and results in better-
optimized LDPC codes in terms of reducing the decoding
threshold. 1 In our proposed method, VN degree distributions
and the channel assignments for VNs with different degrees
are optimized via two cascaded differential evolution (DE)
[10] steps. In each step, to optimize VN degree distribution
or channel assignment, a parity-check matrix is constructed,
and the associated decoding threshold is calculated for each
element in a generation. The VN degree distribution or channel
assignment with a better decoding threshold compared to that
of the previous generation is selected as the parents for the
next generation. In constructing a parity-check matrix for each
channel assignment, we propose a constraint PEG-like algorithm
to satisfy the given channel assignment. The proposed method
is used to optimize and construct irregular LDPC codes for both
16-QAM DBICM and BICM. Numerical results demonstrate
that this method is effective in constructing near-capacity LDPC
codes for DBICM and BICM.

After a 1-to-m serial to parallel module, m sub-sequences xt(i),
i ∈{0, ..., m −1} are obtained. For simplicity, n is assumed
to be dividable by m. Next, each subsequence xt(i) is delayed
by Ti time slots to generate xt−Ti(i), i ∈{0, ..., m −1}. Note
that a time slot here represents n/m symbol periods. At last,
m bits in the same symbol period, one from each xt−Ti(i), are
mapped to a symbol st via the mapper, which is transmitted
over an AWGN channel.
The receiver obtains noisy signals rt, which are fed to the
demapper. The demapper calculates the log-likelihood-ratios
(LLRs) for each bit in the m bit-channels based on the received
signal rt and the a priori information of delayed bits in previous
codeword(s), i.e. LLR(ˆct−Ti) for Ti = 0. The calculated LLRs
are denoted by ˆyt−Ti, i ∈{0, ..., m−1}. Then, ˆyt−Ti are sent to
an “inverse bit delay module” so that the LLRs for all the bits in
xt, denoted by ˆyt(i), are assembled. After an m-to-1 parallel to
serial module and de-interleaver, the resultant LLR sequence for
ct, denoted by ˆct, is decoded to obtain the estimated information
sequence ˆut. Note that the a posterior information about delayed
bits generated by the decoder is interleaved again and sent to
the demapper to help in decoding the undelayed bits of the next
codeword, which is denoted by LLR(ˆct−Ti).
The most important feature which distinct DBICM from
BICM lies in the delay and inverse delay module, as presented
in Fig. 1. In Fig. 2, we demonstrate the difference between
16-QAM BICM and DBICM by showing how xt−Ti, i ∈
{0, ..., m −1}, are fed to the mapper, after the delay. Here,
T1 =T3 =0 and T2 =T4 =1 are considered for DBICM. It can
be seen that in BICM, each codeword is transmitted in a separate
time slot, no dependence between multiple codewords. On the
other hand, in DBICM two consecutive codewords are coupled
with each other via mapping the bits from two codewords to
the same signal. This coupling effect can be exploited by the
receiver to improve the decoding performance. In particular, the
decoded information about the delayed bits in each codeword
is feedback to the demapper to facilitate the demodulation of
the undelayed bits in the next codeword, which intuitively will
improve the decoding performance of the next codeword. Note
that it is possible to further improve the decoding performance
by iterative detection and decoding, which will not be discussed
in this paper.

B. Constrained PEG-like Code Construction Algorithm
With a proper designed VN distribution λ, the conventional
PEG algorithm can be used to construct LDPC codes with a
large global girth in Tanner graph. However, in constructing
LDPC codes for DBICM and BICM, not only the VN distribu-
tion λ and girth matters, but the channel assignment matrix P
should also be satisfied to minimize the decoding threshold of
LDPC coded DBICM and BICM. To achieve this, we propose
a constraint PEG-like algorithm for constructing LDPC codes
that satisfy P and maximize the local girth simultaneously.
The idea of the proposed constraint PEG-like code construc-
tion algorithm is as follows. For a given code length n, we
connect the VNs and the bit channels by using the continuous
bit mapping, i. e., the i-th VN is connected to the bit channel
with index i−1 mod m. Then, each VN is assigned a degree such
that λ and P are satisfied simultaneously. At last, a PEG-like
algorithm is conducted to connect all the VNs and check nodes
(CNs). In this way, P is imposed as a constraint in constructing
LDPC codes for DBICM and BICM.
To implement the proposed algorithm, we define a VN degree
sequence q which is generated from P to list the degrees for
all VNs. In particular, for a given code length n, q is of length
n and its i-th element qi represents the degree of the i-th VN
vi. Then for each VN vi, i ∈{1, ..., n}, it is connected to qi
check nodes (CNs) in a one-by-one manner. In connecting to
each CN, the local girth of vi is maximized. This resembles the
PEG-like code construction algorithm. In the code construction,
we consider a concentrate CN degree dc for all CNs.

Gray-labeled 16-QAM DBICM and BICM both have two
types of bit channels according to the bit channel capacities.
Therefore, the channel assignment matrix P has two rows. The
first row of P, p1, is associated with the bit channels with
a higher bit channel capacity and the second row of P, p2,
is associated with the bit channels with a lower bit channel
capacity. The optimized P with code rate R = 1/4, 2/5 and
1/2, are listed in Table I.
Fig. 5 shows the BER simulation results for the LDPC
codes constructed based on P in Table I using the proposed
constrained PEG-like algorithm. At a code length of 100, 000,
the constructed DBICM codes all approach their capacity limit
within 1 dB. The designed BICM codes also approach their
BICM capacity limit within 1 dB. Besides, the LDPC coded
DBICM has about 0.5, 0.2 and 0.1 dB performance gain over
their BICM counterparts at an information rate of 1, 1.6 and 2
bits/symbol, respectively. Also, note that the gain grows larger
as the code rate decreases. Note that the BER simulation results
are consistent with the capacity analysis results.
VI. CONCLUSION
In this paper, we consider the design of LDPC coded DBICM.
Code optimization algorithm and a constrained PEG-like algo-
rithm for irregular binary LDPC codes are proposed. We show
that the designed LDPC coded DBICM approaches its capacity
limit within 1 dB for various rates. Furthermore, it is demon-
strated that the designed LDPC coded DBICM outperforms their
BICM counterparts over AWGN channels, by similar values as
the associated capacity gain of the DBICM over BICM.
