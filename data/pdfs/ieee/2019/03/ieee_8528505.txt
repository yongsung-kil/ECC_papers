A (21150, 19050) GC-LDPC Decoder for NAND Flash Applications
Yen-Chin Liao , Chien Lin, Hsie-Chia Chang, and Shu Lin, Life Fellow, IEEE

Abstract— In this paper, a (21150, 19050) globally-coupled lowdensity parity check (GC-LDPC) code designed for NAND flash memories is presented. The proposed LDPC code comprises three disjoint subcodes which can be decoded independently. This highly structural parity check matrix contributes to efficient decoder implementation and flexible decoding flow control. Moreover, a two-phase local/global decoding procedure optimized for the proposed GC-LDPC code is introduced. Scenarios of collaborative decoding that leverages the special code structures are discussed. In the proposed decoder architecture, the pipelined processing elements with scheduling are employed to reduce the critical path and decoding latency as well. Implemented in UMC 65 nm process, the post-layout simulation shows a maximum decoding throughput of 4.32 Gb/s with the chip area 3.376 mm2.
Index Terms— Globally-coupled LDPC codes, two-phase local/global iterative decoding, emerging memory ECC.
I. INTRODUCTION
O VER the past 10 years, NAND flash memory has become the primary storage media in many mobile devices and consumer electronic products. The advances in process and technology drive the NAND flash memories’ capacity to grow rapidly. Due to the shrinkage of memory cells, multi-levelcell representation, e.g. the MLC (2bit), TLC (3bit) or even QLC (4bit), and advanced memory architectures such as the 3D NAND [1]–[5], the stored information are even prone to errors while information density grows. The manufacturers are urged to seek more powerful error correction codes (ECC) [6] to solve the reliability issues. BCH codes [7], [8] are widely applied in NAND flash memories for their definite error correction capabilities. A t-error correcting BCH code can correct up to t erroneous bits. However, as the raw bit error rate keeps increasing, designing BCH decoders capable of correcting more errors becomes challenging. Longer block-length with larger t is a design trend, nevertheless, the implementation complexity and cost may become unmanageable.

Low-density parity check code [6], [9], [10] is an alternative ECC solution for emerging memories. The probabilistic decoding algorithm requires multi-bit representation may introduce longer memory read latency. Although the error correction capability is indefinite, LDPC codes are shown to be capable of correcting more errors with high probabilities by the more complicated iterative message-passing decoding algorithms. Compared to BCH codes, LDPC codes can be a more efficient option as the ECC block length increases [3], [11], [12].
LDPC codes were first invented by Gallager in the early 1960’s [9]. Although being explained by Tanner in the 1980’s [13], LDPC codes are nearly forgotten for more than 30 years until the rediscovery by Mackay and Neal in the late 90’s [10] and then shown to be a capacity approaching code [14]. Vast results from code constructions, performance analysis [15], to encoder/decoder architectural designs and hardware implementations [16] were extensively brought out by numerous researchers over the past two decades. Several wired and wireless communication standards have adopted LDPC codes as the forward error correction schemes due to the promising performance.
Compared with any wired or wireless communication systems, there are distinct considerations and challenges in designing LDPC codes for NAND flash memories. A different design philosophy is required from code construction to the encoder/decoder implementation. Considering the efficiency, the code rates, which is defined as the ratio of information length to the ECC block length, is usually high. Typically LDPC codes designed for NAND flash memories have code rates higher than 0.8. In order to comply with the confined basic data read/write block size (e.g. the page size), and to preserve superior error correction capabilities for such high code rates, large block lengths from 1K-Byte, 2K-Byte, 4K-Byte or even longer, are necessary. For instance, in [17] and [18], very high code rate (0.95) and large block size (for 8K-Byte information) LDPC code construction and implementations have been introduced.
The effectiveness and efficiency of LDPC decoders highly depend on the parity check matrix designs. The challenges of implementing a high-throughput decoder for LDPC codes with high-code-rate and long-block-length were addressed in [19]. An LDPC code with special parity check matrix structure designed for their specific decoding algorithm was introduced to overcome these issues. Globally-coupled LDPC (GC-LDPC) codes [20]–[22] especially have great potentials in emerging memory applications. A GC-LDPC code

The additional globally-connecting part in the parity check matrix joints these smaller codes, which are referred to as local codes, to provide the superior error correction capability of a largeblock-length code. It is possible to leverage the merits of both short and long block length LDPC codes in GC-LDPC decoder implementation. Moreover, due to the flexible and structural parity check matrix form, a GC-LDPC code can be designed in a way that a single decoder implemented for one of the small block-lengthed code to be used for decoding the entire large block-lengthed LDPC code.
The idea of collaborative decoding was also presented in the two-level LDPC code proposed in IEEE P1890 draft [11]. T 1K-Byte LDPC codes are encoded and decoded independently as the level-1 LDPC codes. A fraction of each of the level-1 LDPC codes is selected to compose a level-2 LDPC code, which is identical to the level-1 1K-Byte LDPC code. The level-2 decoding proceeds only when any of the level-1 decoders fail to correct the errors. Only the information selected by the level-2 encoder has a second chance for error correction. The similar two-level decoding is applicable to GC-LDPC codes. However, a GC-LDPC code differs from the draft in that all the information are encoded jointly with the smaller subcodes. All the error bits have a second chance to be corrected by the level-2 decoding
The main contributions of this work are a (21150, 19050) GC-LDPC code designed for NAND flash memories; a local-global two-phase decoding procedure; a high throughput decoder in compliance with the proposed decoding algorithm to prove the feasibilities of the design concepts. II. GLOBALLY-COUPLED LOW-DENSITY PARITY CHECK (GC-LDPC) CODES
Globally-Coupled Low-Density Parity check codes (GC-LDPC) were first introduced in 2016 [23]. A GC-LDPC code is composed of several smaller-block-lengthed LDPC codes. The highly structural code construction facilitates efficient implementation while maintaining compelling error correcting capabilities of large-block-length codes. In this section, fundamentals of LDPC codes will be reviewed, and the construction of GC-LDPC codes will be introduced in subsequent.

Considering the decoding speed and complexity, CNs and VNs are often updated block-wise in a partial parallel decoder architecture. That is, only a portion of the messages are updated simultaneously. The iterative decoding algorithms mentioned above have a flooding-type updating schedule such that all the CN updates in (5) or (9) take the input messages updated by the VNs from the previous iteration. Literature has shown improvements in decoding convergence if a CNU or VNU takes the latest updated messages as the input messages [26], [27].
Column shuffle decoding [26] partitions the VNs into blocks. Only the neighboring nodes connecting to one VN block will perform CN updates before the subsequent VN block proceeds. Shuffle decoding allows the subsequent blocks to process with the latest information. Column shuffle is especially more efficient in implementation when the row degrees are large [28]. Block-wise scheduling can be performed in a row-based fashion as well. Row-based layered scheduling [27], or called row shuffle or layer decoding, partitions the CNs into blocks. When all the CN blocks complete the message updating in one iteration, the VNs are updated more than once, which also leads to faster convergence.

III. GC-LDPC CODES DESIGNED FOR NAND FLASH MEMORIES
GC-LDPC codes designed for NAND flash applications are addressed in this section. The target code rate is 0.9, and the block length is longer than 2K-Byte. Code constructions by algebraic methods are introduced. The (21150, 19050) GC-LDPC code construction will be given by the subsequent examples. To leverage the structural parity check matrix of GC-LDPC codes, a local-global two-phase decoding procedure is introduced. It can be further simplified to a two-level feed forward decoding flow.

B. Local-Global Two-Phase Decoding To leverage the locally disjoint and globally coupled parity
check matrix structure, the decoding can be arranged in a hierarchically local-global two-phase manner. The local phase and the global phase are associated with the local codes and the globally-connecting part, respectively. In the local decoding phase, all the local codes are decoded independently. Several local decoding iterations are performed and the message updating is confined between the local VNs and the local CNs. The information of other local nodes can only be obtained via global decoding. The decoding switches between the local phase and the global phase until the convergence conditions are met. The concept of local-global two-phase decoding was first mentioned in [23] with various usage scenarios.

In the following, possible realizations and the corresponding decoding flow control will be presented. For conciseness, two-phase decoding will be used to refer to the local-global two-phase decoding hereinafter. The two-phase decoding is highly-flexible in the decoding flow control by configuring the iteration number of local and global phase. In some latency-sensitive applications, decoding may terminate as soon as all the local codes’ parity check equations are satisfied.
Fig. 7 illustrates one possible two-phase decoding procedure. IL and IG are the iteration number of local decoding and global decoding, respectively. I is the iteration number of complete decoding. The predefined maximum decoding iteration number for local and global phases are TL and TG, respectively. In layer decoding, this two-phase decoding flow can be realized by constraining the check nodes to be updated in different decoding iteration. A maximum of i tr ×(TL +TG) iterations can be performed. For every TL + TG iterations, only the local check nodes are updated in first TL iterations, and the global check nodes are updated only in the subsequent TG iterations. Early termination conditions are examined at the end of each iteration.
TL and TG are treated as design parameters. In our design, they are derived empirically. We can observe the BER of the local decoding under certain signal-to-noise ratio (SNR). Fig. 6 shows that the local code’s BER goes down quickly at the first few iterations. The decoding convergence slows down with the iteration number. The analysis implies that the decoder should switch to the global phase before the local decoder’s BER starts to saturate. Thus we choose TL = 3 in our design. Then several combinations of TG ’s are tried, and TG = 1 or TG = 2 is found to deliver effective decoding results within i tr = 5 complete iterations.
Moreover, if the bit error rate is non-uniform among these local codes, the local codes corresponding to lower raw bit error rates will converge faster, and may help the other local codes converge by the information exchanged in the global phase. C. Two-Level Decoding
The above mentioned local-global two-phase decoding can be simplified in a feed-forward approach. Similar to the two-level decoding introduced in [11], all the local decoders perform independent decoding first. Only when any of the local decoders fail in correcting the errors the decoding is switched to the second-level: a complete decoding based on HGC . Fig. 8 illustrates the two-level decoding flow. Fig. 9 is the simulation results of 2-level layer decoding for the above-mentioned GC-LDPC-3 code. At the beginning of decoding, each of the 3 local codes is decoded independently. The maximum iterations allowed for these local decoding is TL and TL = 3, 6, 10 are compared. Then the decoder performs decoding based on the complete parity check matrix HGC , this is referred to as the second-level or level-2 decoding. The maximum number of iteration for level-2 decoding, TG , is 10 in our simulation. The figure shows a significant gap between the local decoding and the complete GC-LDPC code decoding because local codes have smaller block lengths (7050-bit) and higher code rates (0.9149). It is perceived that level-2 decoding significantly reduces the gap.

IV. IMPLEMENTATION OF TWO-PHASE LAYER DECODING One big challenge of implementing long LDPC codes
decoders is the large storage requirement. In our proposed (21150, 19050) LDPC code, the a posteriori LLR require 21150 × 6 = 126900 bits of storage elements if the messages are quantized in 6 bits. Furthermore, assuming that the messages stored for one check node are the first-min’s magnitude (3-bit), the second-min’s magnitude (3-bit), the sign (1-bit), and the index of the first-min (6-bit), there will be 2100 ×(3 + 3 + 1 + 6) = 27300 bits required to store all the check nodes’ information. In addition to the large storage requirement, the data bandwidth grows with the decoder parallelism. Although the 3 local codes in the proposed two-phase decoding can be decoded in parallel, the inevitably high memory bandwidth and the hardware routing complexity for the message passing decoding algorithm may offset the throughput increased by higher parallelism.
In this section, the architectural designs and the implementation of the proposed (21150, 19050) GC-LDPC codes with local-global two-phase layer decoding will be presented. A row-scheduled layer decoder architecture is introduced. The layer size is the CPM size, which is 150. The proposed design aims to provide a high throughput decoder of long block length LDPC code with manageable complexity. It has been mentioned in [19] that layer decoding may suffer from the long critical path when the parity check matrix has large row weights. Because of the (almost) identical local part, it is possible to apply hardware sharing, pipelining, and scheduling among the local decodings to achieve high hardware utilization and decoding throughput.

C. Implementation Results

The two-phase layered decoding for the (21150, 19050) GC-LDPC code is implemented in UMC 65nm process. Fig. 15 is the chip place-and-route layout. TABLE II summarizes the chip implementation results based on the post-layout simulation. The decoder is designed for NAND flash applications where the latency resulted from acquiring the soft information is long. A finer resolution, i.e. more input quantization bits, requires multiple-read by adjusting different sensing voltages, and may lead to unfordable longer read latency. Thus the input messages are quantized to at most two bits. The internal quantization is 4-bit and 6-bit for the CNUs’ messages and VNUs’ messages, respectively. The decoder operates in two input modes. The maximum decoding iteration is confined to 6. Each iteration comprises 3 local and 2 global iterations in the two-phase decoding procedure. For the purpose of power saving and higher decoding throughput, early termination is implemented where the decoding process can terminate as soon as the decoder generates a valid codeword. The decoder can achieve 4.32 Gb/s decoding throughput with an average of 1.8 decoding iteration under clock frequency at 322.6 MHz.

V. CONCLUSION
In this paper, a (21150, 19050) GC-LDPC codes designed for NAND flash memories is presented. A two-phase decoding procedure and its simplified two-level decoding that leverage the structural parity check matrix are proposed to fully utilize the high-parallelism parity check matrix structures. A decoder conforming with the proposed decoding algorithm is implemented in UMC 65nm process and proves the design concept.
