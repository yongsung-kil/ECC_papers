FPGA-Based Real-Time Soft-Decision LDPC Performance Verification for 50G-PON

Mingwei Yang1, Linlin Li2, Xiang Liu2, and Ivan B. Djordjevic1
1. Department of Electrical and Computer Engineering, University of Arizona, Tucson, AZ 85721, USA 2. Futurewei Technologies, Huawei R&D, Bridgewater, NJ 08807, USA mingweiyang@email.arizona.edu

Abstract: We have studied the combined use of the mother LDPC code from ongoing industrial standards and soft-decision to further improve the performance of the FEC. Through experimental measurements based on a real-time FPGA platform, we found that this approach offers ~1.3 dB more gross coding gain.
OCIS codes: (060.4250) Networks; (060.2330) Fiber optics and optical communication.

1. Introduction Forward error correction (FEC) with high coding gain is needed to achieve PR30 (29dB) link budget in 50G-PON. IEEE’s 802.3ca 100G-EPON Task Force has chosen a high-performance hard-decision (HD) low-density parity check (LDPC) FEC with a bit error ratio (BER) threshold of about 102 and a rate of 0.849 [1-4]. A previous study analyzed the use of soft-decision (SD) for the same LDPC encoder that has been specified by the Task Force [5]. To achieve even better coding gain, here we analyze the performance of the mother code of the LDPC reported in [1-4], which offers a higher coding gain with a slightly reduced code rate down to 0.826, by using real-time measurements.

where I(p) is a circulant permutation matrix of size b×b with the first row obtained by cyclic shifts of pi,j. For given values of row number γ, column number ρ, block size b, the [zi,j] matrix is a γρ matrix over GF(2). The LDPC parity-check mother matrix we analyze in this paper has the parameters γ=12, ρ=69, and b=256; and it is illustrated in Fig. 1, which is proposed in references [1-4]. Correspondingly, the codeword length is 25669 bits or 17664 bits, the payload length is 25657 bits or 14592 bits, and the parity-check word length is 1269 bits or 3072 bits. The resulting code rate is 57/69 or 0.826. The LDPC decoder is implemented in a real-time platform based on a fieldprogrammable gate array (FPGA). The real-time processing speed is about 0.1 Gbps. To achieve an output BER of 1012 with about 200 output errors counted, the real-time measurement needs to run for about 10 days.

The architecture of the PFGA-based LDPC emulator is depicted at Fig. 2. It consists of the encoder module, additive white Gaussian noise (AWGN) channel module, and the decoder module. The pseudorandom binary sequence (PRBS) is generated inside the encoder module as the information bits. The parallel parity-check sequence can be completed in ρbγ clock cycles by employing the shift-register-adder-accumulator (SRAA) and parallel-toserial (P2S) converter multiplexer. The bit log-likelihood ratio (LLR) values calculated from the channel are stored in the LLR memory (the Lv module in the LLR min-sum processor). In the LLR minsum processor, the variable node processor (VNP) performs the sum operation, while the check node processor (CNP) performs the finding minimum values. The output of CNP will update a posteriori probability (APP) memory (the Lcv module in the LLR min-sum processor), and the updated information will be used in the next iteration. To address the low latency requirement, we limit the maximum iteration number to 15. The loop state controller is a state machine to control the operation of the VNP-CNP and indicate the layer index to the AddrGenerator module that is used to generate the cyclic address in each block layer by layer. The output of the sum module will be sent to the error counter module to check the post-FEC BER value. The processing latency and resource consumption are mainly contributed from the minimum finding part. The minimum finding module employs a 2n degree binary search tree to find out the value and location of the first minimum value and second minimum value. To check the complete post-BER in each layer, all blocks in one layer are required to be checked by sum part. However, not all of them are necessary for the minimum part in irregular LDPC codes structure. Therefore, in our design, the LLR and APP memories are read and updated from all blocks in each layer. The sum part will utilize these complete LLR and APP values, while minimum part will compress them at the input port and expand them at output port based on the value of zi,j. By this way, a 32 degree binary tree is employed to process a 69 blocks parallelly with maximum column weight of 24. The process in LLR min-sum processor cost only 13 clock cycles in FPGA.

3. Implementation and Experimental results
The resource utilization of the implementation is summarized in Fig. 3. Figure 4 shows the experimentally measured BER performance of SD-LDPC in comparison with HD-LDPC. The simulated BER performance of the HD-LDPC chosen by the IEEE 802.3ca [2] is also plotted here as a reference. This reference HD-LDPC code employs the same mother code but has 512 parity bits punctured and 200 payload bits shortened, resulting in a codeword length of 16952 bits, a payload length of 14392 bits, and a rate of 0.849. For an output BER of 1012, the rate-0.849 reference HD-LDPC code [2], the rate-0.826 HD-LDPC, and the rate-0.826 SD-LDPC require the input BER to be 1.1×102, 1.25×102, and 2.4×102, respectively. At the same code rate of 0.826, the SD decoder is 1.1 dB better than the HD decoder in terms of net coding gain (NCG). This is consistent with the theoretical NCG difference between SD and HD for a rate of around 0.8 [6].
Compared to the reference HD-LDPC with 0.849 rate [2], the SD-LDPC with 0.826 rate provides ~1.3 dB more gross coding gain at an output BER of 1012. This additional coding gain is expected to translate into ~0.9 dB lower received optical power [7], and thus better link budget.

4. Conclusion
We have studied the combined use of the mother LDPC code described in [2-4] and SD decoding to further improve the performance of the FEC. Through experimental measurements based on a real-time FPGA platform, we found that this approach offers ~1.3 dB more gross coding gain over that being specified in the IEEE 802.3ca 25G-EPON standards. With this improved SD-LDPC performance, it is possible to support PR30 (29dB) link budget for 50GPON.
