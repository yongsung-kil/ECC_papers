Linear Permutation Polynomial Codes
Ryoichiro Yoshida and Kenta Kasai Tokyo Institute of Technology
Email: yoshida.r.ag@m.titech.ac.jp, kenta@ict.e.titech.ac.jp

Abstract—Quasi-cyclic low-density parity-check (QC-LDPC) codes are one of the most important code classes of LDPC codes. They have two drawbacks: lack of randomness and limited girth lead to a degraded decoding performance in the waterfall and error floor regions, respectively. To tackle these problems, we present a new class of LDPC codes, named linear permutation polynomial (LPP) codes, whose parity-check matrix consists of permutation matrices based on LPPs. The girth of regular QCLDPC codes is upper bounded by 12, while LPP codes break this limit. We demonstrate that LPP codes have error performance almost equivalent to random ones.
I. Introduction
Low-density parity-check (LDPC) codes [1] have been shown to have a remarkable performance close to the Shannon limit over additive white Gaussian noise (AWGN) channels with iterative decoding based on belief propagation such as the sum-product algorithm [2].
Based on the methods of construction, LDPC codes can be classified into two general categories: random constructions, and algebraic constructions. For long code lengths, random constructions of LDPC codes closely approach the Shannon limit under the assumption of no cycles [3]. On the other hand, for medium code length, algebraic constructions can outperform random constructions [4]. Another advantage of algebraic constructions of LDPC codes is that they need less amount of memory than random ones to store their paritycheck matrices.
In algebraic constructions of LDPC codes, there has been considerable insterest in the design of quasi-cyclic low-density parity-check (QC-LDPC) codes, whose parity-check matrices are composed of blocks of circulant permutation matrices (CPMs). The required memory for storing their parity-check matrices can be reduced by a factor 1/m, when m × m CPMs are employed. Furthermore, these codes are quasi-cyclic; hence their encoding can be implemented in linear time with shift registers [5, pp. 256-261].
Regardless of the many advantages of QC-LDPC codes, QCLDPC codes have two problems. The first problem is that they have low randomness because, for QC-LDPC codes over F2, each m×m CPM in their parity-check matrices is characterized by an element of a residue class ring Zm = {0, 1, . . . , m − 1}. The second is that their theoretical upper bound of girth is 12 [6].
In this paper, to tackle these problems, we present a new class of LDPC codes based on linear permutation polynomials (LPPs). In Section II, We define a general class of codes including the proposed codes defined in the next section. Then, we reveal a condition for them to have a cycle. In Section III,

we define LPP codes as proposed codes and show properties of their girth. Furthermore, in Section IV, we present a random construction for LPP codes and show that we constructed LPP codes with girth larger than 12. Finally, in Section V, we demonstrate that LPP codes have an error performance almost equivalent to random LDPC codes.

C. Condition for (A, B)-edges not to form a cycle
Theorem 4. A linear polynomial R(x) = ax + b over Zm permutes Zm iff a and m are coprime.
Proof. (Necessity): Suppose that R(x) permutes Zm. There exists x such that ax + b = b + 1, which implies ax = 1. ax = 1 means that a has the reciprocal x. Therefore, from Lem. 1, a and m are coprime.
(Sufficiency): From Thm. 2, R(x) has the inverse function if a and m are coprime. Futhermore, R(x) permutes Zm if R(x) has the inverse function. Therefore, R(x) permutes Zm if a and m are coprime.
Lemma 3. Let a ∈ Z and q|m. Then,
a 0 (mod q) ⇒ a 0 (mod m).

Proof. We will prove the contraposition. Let a = 0 (mod m) and m = cq. There exists b ∈ Z such that a = bm, and then a = bcq. Therefore, a = 0 (mod q), which completes the

Theorem 6 implies that if m is a prime, H has girth larger than 4 only if all permutation matrices in H are CPMs. In what follows, we concentrate on the case with m which is not a prime. This allows (I, J, m) matrices to have girth larger than 4.
A lower bound of m with which (I, J, m) matrices have girth larger than 4 is given by the following theorem.
Theorem 7. If H has girth larger than 4, then m ≥ J.
Proof. We prove the contraposition, i.e., if m < J, there exists a cycle of length 4 in H. First, let us pick a block row index i 0 and block column indices j1 j2. Consider 4 edges in Pi, j1, P0, j1, P0, j2, Pi, j2 . From bi, j1, bi, j2 ∈ {0, . . . , m − 1} and m < J, we see that there exists at least one pair of permutation matrices Pi, j1 and Pi, j2 such that bi, j1 = bi, j2 . Next, pick 4 edges P0, P1, P2, P3 such that P0 = Pi, j1, P1 = P0, j1, P2 = P0, j2, P3 = Pi, j2 . These edges are (A, B)-edges with
A := a0 − a1a2−1a3 B := a3a2−1(b1 − b2) + (b3 − b0).
In Def. 2, we assumed that P1 = P2 are CPMs. Finally, we obtain
B = b3 − b0 = bi, j2 − bi, j1 = 0.

V. Simulation Results
We demonstrate decoding error performance of LPP codes and other LDPC codes in Fig. 1 and 2.
We constructed a (3, 4, 906) LPP code with N = 3624, K = 910, g = 10. This code is generated based on the construction in the previous section. For comparison purposes, we picked a (3, 4)-regular QC-LDPC code with the same parameters N, K and g. This code is generated based on a random construction proposed in [6]. In addition, we added a random LDPC code with N = 3640, K = 910, g = 10, column weight 3, row weight 4. From Fig. 1, we observe that the QC-LDPC code has no waterfall region up to the Eb/N0 of 3.5 dB. On the other hand, the LPP code outperform the random code in the waterfall region. Furthermore, the LPP code has no error floor down to the WER of 10−6.
We constructed a (3, 6, 762) LPP code with N = 4572, K = 2288, g = 12. This code is generated based on the construction in the previous section. For comparison purposes, we picked a (3, 6)-regular QC-LDPC code with the same parameters N, K and g. This code is generated based on a random construction proposed in [6]. In addition, we added a random LDPC code with N = 4572, K = 2286, g = 10, column weight 3, row weight 6, and a (3, 6)-regular QC-LDPC code with N = 4572, K = 2307, g = 8 in [8, Example 12.12]. From Fig. 2, we observe that the QC-LDPC code generated based on a random construction proposed in [6] has no waterfall region below the WER of 10−4. On the other hand, the LPP code and the random LDPC code, and the QC-LDPC code in [8, Example 12.12] have almost equivalent error performance in the waterfall region. Furthermore, the LPP code has no error floor down to the WER of 10−5.
VI. Conclusion
In this paper, we present LPP codes to tackle two problems of regular QC-LDPC codes, and show that LPP codes can have girth larger than 12 based on a random construction. Furthermore, simulation results show that LPP codes have almost equivalent error performance to that of random LDPC codes and a regular QC-LDPC code in [8, Example 12.12].
