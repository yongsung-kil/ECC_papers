Abstract— A check nodes correction approximate Min-sum decoding algorithm for LDPC is proposed in this paper. The degradation factor of BP to MS is found to do expansion and optimization based on hardware working manner. The decoding algorithm summarized a correction approximate algorithm based on different SNR situations and hardware implementation. The simulation is based on LDPC NR 3GPP 38.212 release and the comparison results showed the proposed check nodes correction approximate Min-sum decoding algorithm can have a better BER and iteration time performance. The hardware of this proposed algorithm is based on Min-sum decoder and the extra cost is only a shifter and an adder besides the memory.

Keywords—LDPC, Min-Sum, LDEC, check nodes, correction
I. INTRODUCTION LDPC (Low density parity check) codes was proposed by R.Gallager in 1963 [1], and has gained a lot of interest because of its outstanding error-correction performance. There are lots of decoding algorithms which demonstrate different advantages for decoder implementation. The MS (Min-sum) algorithm [2] is the most used decoding algorithm in modem SoC design because of the reusability between NR and LTE decoder modules and the less complexity of hardware implementation. It allows to achieve attractive tradeoff between error performance and decoding complexity. The belief propagation (BP) algorithm [3] provides a powerful tool for iterative decoding of LDPC codes. The MS approximation which using the so-called BP-based approximation is well known for hardware implementation because of it can greatly reduce the implementation complexity with the cost of a degradation in decoding performance. This has led to the development of many balanced variants of the MS algorithm [4]–[8] that nonetheless deliver near-optimum decoding performance. S. L. Howard proposed degree-matched check node decoding for regular and irregular LDPCs [9]. An asymptotic expression for the correction factor at a check node of any degree is derived in this paper, and used to develop a simple approximations to the correction factor, matched to the check node degree. Simulation results are presented for each check node approximation when decoding a regular and an irregular LDPC, which can achieve sum-product decoding performance. Yin Xu proposed a variable LLR scaling in min-sum decoding [10]. That give a generalized mutual information (GMI) of LLRs as the criterion to search for the scaling factors for different check node degrees in every iteration in a 1-D thus low-complexity manner. This paper investigates the relationship between BP algorithm and MS algorithm and find the reason of degradation. IV. RESULTS AND DISCUSSION The hardware is implemented with Intel 14nm technology based on the timing optimization method introduced by [11-13] to give a better implementation. It is successfully implemented in Verilog and fully verified at RTL level and synthesized with Intel 14nm technology lib. The simulation were carried out for LDPC NR 3GPP 38.212 release. The LDPC decoding algorithm max iteration numbers are all set to 50. And the SNR is troughed from 0.1dB to 1.5dB each for 10000 times to get average value for BER comparison between MS, [9], [10], proposed decoding algorithm. The BER result is showed in table 1 and iteration times’ comparison in table 2. The table 1 presents the comparison results of BER in different SNR for Min-sum decoder, [9] proposed decoder, [10]’s decoder and the proposed check nodes correction approximate Min-sum decoder. From the result it is clearly to find that the BER is reduced from 55% to 67.4% compared to the classic Min-sum decoder. 10% to 49.7% reduction compared to the [9] and [10]’s decoder especially when SNR is in a bad situation. That’s because the proposed decoded using correction approximate algorithm updated the degradation part between the Min-sum decoding and BP decoding. When SNR is big enough the proposed decoder used just a constant value which is a register in hardware to do the fixing in a small hardware cost. So the BER reduction is not observably but the hardware cost will be more acceptable. The table 2 showed the iteration times comparison results between 4 decoders as table 1. From the table 2 we can see, the proposed decoder can reduce the iteration average time from 19.3% to 33.2% compare to Min-sum decoder and 8.9% to 26.2% compare to the [9] and [10]’s decoder. V. CONCLUSION A check nodes correction approximate Min-sum decoding algorithm for LDPC is proposed in this paper. The decoding algorithm used the check nodes degree as reference to optimize the calculation and find a simplified approximation correction algorithm to realize this proposal. The proposed algorithm can easily implemented in hardware. It is verified at RTL level and synthesized with Intel 14nm technology lib and the results showed the proposed decoder can have a better BER and hardware speed performance. The future work will focus on the correction missed section’s optimization.
