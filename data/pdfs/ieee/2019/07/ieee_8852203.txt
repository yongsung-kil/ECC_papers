Guessing the Code: Learning Encoding Mappings Using the Back Propagation Algorithm
Amrutha Machireddy and Shayan Srinivasa Garani Department of Electronic Systems Engineering Indian Institute of Science Bengaluru, 560012, India Email: {amrutha, shayangs}@iisc.ac.in

Abstract—Error correction codes such as low density parity check (LDPC) codes are popularly used to enhance the performance of digital communication systems. The current decoding framework relies on exchanging beliefs over a Tanner graph, which the encoder and decoder are aware of. However, this information may not be available readily, for example in covert communication. The main idea of this paper is to build a neural network to learn the encoder mappings in the absence of knowledge of the Tanner graph. We propose a scheme to learn the mappings using the back propagation algorithm. We investigate into the choice of different cost functions and the number of hidden neurons for learning the encoding function. The proposed scheme is capable of learning the parity check equations over a binary field towards identifying the validity of a codeword. Simulation results over synthetic data show that our algorithm is indeed capable of learning the encoder mappings and identifying the parity check equations.
Index Terms—Error correction codes, linear block codes, low density parity check codes, back propagation algorithm.

I. INTRODUCTION
Error correction codes (ECC) are inevitably used in all digital communication systems for protection against channel errors, in turn improving the efficiency of the system. ECCs were first developed by Hamming to demonstrate almost error free communication. The message to be transmitted is encoded into a codeword by adding redundant bits. At the receiver end, the received codeword is decoded to obtain the message. This is as illustrated in Figure 1. It must be noted that the encoder and decoder have the knowledge of the underlying code.

There are two types of ECCs: convolutional codes and block codes. In convolutional codes, the output of the encoder not only depends on the current input but also the previous inputs. These codes are used in applications when the input data is serial. Viterbi algorithm [1] is most popularly used for

decoding these codes. Block codes on the other hand encode data in blocks and do not depend on the past inputs.
In the early nineties, neural networks have been explored towards ECC decoding. For the convolutional codes, the recurrent neural networks are used to implement the Viterbi algorithm [2]–[4]. In the case of block codes, Bruck and Blaum [5] compare the maximum-likelihood decoding of the code to be equivalent to finding the global maximum of a function using neural networks. Tseng and Wu [6] explore the possibility of using higher order perceptrons, i.e., by using higher order combinations of the input while training the network for decoding error-correcting codes. In [7], the authors construct the neural network such that the parity network is present in the first layer and a linear perceptron in the next layer. However, the information regarding the parity network is not available always. Nachmani [8] proposed a method of generalizing the standard belief propagation algorithm by training the neural network to learn the weights given to the edges of the Tanner graph. In [9], recurrent neural network architecture is used for sparse Tanner graph representations of the code. In all of the above cases, the knowledge of the parity check equations is assumed in some form. However there might be situations where the encoder information is missing or incomplete, for example in covert communication.
Consider an application, such as, tapping into the information exchange between two systems that encode the data before transmission, such as in satellite communication. At the receiver end, if only certain encoding mappings are known, is it possible to learn the entire encoding scheme? Once the encoding scheme is learnt, the underlying parity check equations can be extracted. Further, the validity of a given codeword can also be established. Motivated by such practical considerations, we propose a scheme to extract the relationship between the message bits and code bits by learning the mappings using the well known back propagation algorithm.
This paper is organized as follows. In Section II, we describe the necessary background on ECC. In Section III, we discuss the proposed scheme for learning the encoding mappings and discuss the influence of hidden neurons. In Section IV, we perform experiments to validate our approach. For example, consider a binary input message of length 2, m = [x1 x2] where x1, x2 ∈ {0, 1} i.e., m ∈ {11, 10, 01, 00}. Encoding is done by performing XOR operation on the message bits to get the codewords c = [p1 x1 x2], where p1 = x1 ⊕ x2, i.e., c ∈ {011, 110, 101, 000}. For the given message block m, the following are invalid codewords: {111, 010, 001, 100} ∈/ C. In general, it should be noted that out of 2n points in the code space, only 2k are valid codewords. A code is said to be systematic if the message block is present as such in the codeword. For example, consider the message block 11 and the corresponding codeword 011. We observe that the message block is present in the codeword and therefore the codeword is systematic. If c1 + c2 ∈ C for all c1, c2 ∈ C, then the binary code is said to be a linear block code. For example, with c1 = 110 and c2 = 101, we see that c1 + c2 = 011 ∈ C. For every linear block code, there exists a generator matrix G, parity check matrix H and syndrome S. Upon completion of training, the neural network represents the mapping P : m → p. Now, by considering binary input vectors of form ei with 1 in the ith position and 0 elsewhere, one can identify the form of parity check equations used to
encode the inputs at the source. The idea behind this approach
is as follows, if a given attribute is present in the parity check equation of a check node, the corresponding output will be 1.
In this way, the influence of each attribute in the message bit is observed at the output. For example, with k = 3, let the parity check equations be p1 = x1 ⊕ x2 and p2 = x2 ⊕ x3. For the inputs m = {[1 0 0]T, [0 1 0]T, [0 0 1]T} fed to the trained network, the outputs y1 = [1 0], y2 = [1 1] and y3 = [0 1] are observed respectively. V. CONCLUSIONS AND FUTURE WORK
We proposed an approach to learn the mapping between the message bits and the parity check bits using the well known back propagation algorithm. The influence of the number of hidden neurons on the performance of the network is observed. Experimentally, we validate our approach by learning the mapping and extracting the parity check equations. We also show the ability of our approach to ascertain the validity of a codeword for varying number of hidden neurons and parity check bits. Future scope of this work would be to learn the mapping in the presence of noise in the given data and extend this work analytically towards computing information storing capacity of the neural network.
