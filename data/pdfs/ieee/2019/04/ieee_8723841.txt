Abstract: This paper presents a soft implementation of the joint approach for irregular Low Density Parity Check (LDPC) encoding and decoding. The principal idea is to construct excellent irregular LDPC codes focus to two constraints that make certain the effective LDPC encoder and decoder implementations. This work propose a new model for memory loading of the LDPC parity-check matrix aiming to use only the Level one (L1) of cache memory, to take advantage of its high speed latency.
Keywords—LDPC, parity check matrix, cache memory
I. INTRODUCTION
Low Density Parity Check (LDPC) codes still attract much attention because of their excellent error-correcting performance. LDPC codes were introduced by Gallager [1] in 1962 and re-discovered by David Mckay [2] in 1996. LDPC codes are used in many standards such as Digital Video Broadcasting–Satellite-Second Generation DVB-S2 [3], and in IEEE WiMax 802.16e [4].
LDPC encoding is a simple matrix operation using a generator matrix G. However, large and sparse matrix G poses implementation problem.
LDPC decoding process uses iterative Belief Propagation (BP) algorithms wherein the preliminary estimations of the bits are improved via a non-stop exchange of messages among decoder computation nodes. Out of the various BP decoding algorithms, the min-sum algorithm (MSA) [5, 6], which is the most common choice for hardware implementation. However, large and sparse parity check matrix H using in the decoding process increases the hardware implementation complexity.

Large sizes of H matrix are used in DVBS2, 64800 columns [3], and in the WiMax standard, between 576 à 2304 columns [4], in order to increase the bit rate.
Several recent works [7][8], have proposed a materiel design for memory loading H matrix, aiming to increase the memory required for presenting the parity-check matrix.
In this work, by using C language, we propose a software implementation of LDPC encoder/decoder, with a new memory loading parity-check matrix aiming to use only the Level one (L1) of cache memory, to take advantage of its reduce latency.

A. Systimatic LDPC encoding

LDPC codes are a linear block codes. Encoding K bits message to N bits code words, refers to a Generator matrix, G, of size K by N. The code rate, R, of the LDPC code is described through the following equation

R = K/N

(1)

A c codeword may be formed from a message, u, using the following formula:

c=uoG

(2)

B. Parity check matrix H :

A binary LDPC decoding use a parity check matrix H as:

{c ∈ {0,1}n | H o ct = 0 },

(5)

If the parity check matrix contains the same number of ones per column (noted dv), and the same number of ones per row (noted dc), the code is called a regular LDPC code. Otherwise, the code is called irregular code. In this paper we are interested to irregular code for their good convergence [9] in term of Bit Error Ratio (BER) related to Signal Noise Ratio (SNR).

H matrix is commonly represented with a Tanner graph, which includes N variable nodes (VNs) and M check nodes (CMs). The n VN is connected to the m CM if Hmn = 1.

V. CONCLUSION

The proposed model for memory loading the H matrix is optimal and minimizes the search for variables outside the L1 cache memory. However, as perspective we proposed to mix this method with hard implementation of Vn, Cm, and Switch tables. Also we proposed to use this model for parallel implementation.
