EXIT Chart Analysis of Serial Window Decoding of SC-LDPC codes
Sirawit Khittiwitchayakul, Watid Phakphisut, and Pornchai Supnithi
Faculty of Engineering, King Mongkut's Institute of Technology Ladkrabang, Bangkok 10520, Thailand. Email: {57601155, watid.ph, pornchai.su}@kmitl.ac.th

Abstract
Spatially coupled low-density parity-check (SCLDPC) codes can be decoded by a sliding window which can provide the lower complexity and latency than that of underlying LDPC block code (LDPCBC) decoding. In this work, we propose to use an Extrinsic Information Transfer (EXIT) chart as an analysis tool for the window decoding of SC-LDPC code. Herein, the EXIT chart analysis is applied for both window decoding strategies like flooding and serial strategies. The analysis results show that the window decoding with serial strategy can converge faster than that of flooding strategy. The required numbers of decoding iteration can be reduced greatly without the performance degradations.
Keywords: SC-LDPC codes, window decoding, flooding strategy, serial strategy, EXIT chart
1. Introduction
Among the modern coding theory, the spatially coupled low-density parity-check (SC-LDPC) codes [1] are emerged to achieve the maximum a-posteriori (MAP) threshold of underlying LDPC block code (LDPC-BC) [2]. Furthermore, since the SC-LDPC code can be decoded by sliding the window along the bit sequence [3], the decoding complexity and latency are lower than that of LDPC-BC. Therefore, the SC-LDPC codes become the promising candidate for the applications requiring the excellent performance and acceptable complexity.
Traditionally, the window decoder of SC-LDPC code relies on the iterative message passing algorithm with flooding strategy [4], whereby all variable and check nodes within the window decoder are updated simultaneously at each decoding iteration. In recent work [5], the window decoding with serial strategy is introduced. In this case, before each variable node is updated, the information of variable nodes which are recent updated are requested [6]. Some simulation results in [5] reveal that the window decoding of SC-LDPC code with serial strategy can converge faster than that of flooding strategy.
In the channel coding theory, we can evaluate the theoretical performance of iterative LDPC decoding by an Extrinsic Information Transfer (EXIT) chart as proposed in [7], and [8]. Previously, the EXIT chart

is employed to analyze the performance of LDPC-BC decoding with serial strategy [6]. In this work, our analysis is motivated by the fact that the previous works focus only on the usability of EXIT chart for LDPC-BC decoding with serial strategy. We then propose to use the EXIT chart for the sliding window decoding of SC-LDPC code with flooding and serial strategies. Our analysis results reveal that the window decoding with serial strategy can converge faster than that of the flooding strategy. Moreover, we show that the numbers of decoding iteration can be reduced by 45% without the significant loss in performances.
In the rest of article, we briefly review the SCLDPC codes and its window decoding in Section 2. We describe how to use the EXIT chart analysis for window decoding in Section 3. The analysis results are presented in Section 4, and then the conclusion is expressed in Section 5.

2. Protograph-based SC-LDPC codes

2.1. Definitions and notations

Given that the large information bits at transmitter side is divided equally into L information blocks, and then each block is encoded separately to obtain the codeword of LDPC-BC, v = [v1, . . ., vt, . . ., vL]. Each code block of LDPC-BC can be characterized by the protogrpah [9] correspond to the base matrix B = [B1, . . ., Bt, . . ., BL]. The main characteristic of SC-LDPC code which differ from LDPC-BC is that the code blocks (codeword) at the different times are interconnected. To achieve the coupled code block, the edge spreading procedure [10] will be applied whereby the edges of variable node at time t are distributed to the check nodes at time t + i, i = 0, 1. ., mcc. The memory mcc is defined as the largest distance between a pair of coupled code blocks. The example of edge spreading procedure is shown in Fig. 1.

2.2. Sliding window decoding of SC-LDPC code
In general, the SC-LDPC code can be decoded by using the iterative message passing algorithm applied across the total length of consecutive code blocks. However, with the convolutional structure of SCLDPC code emerged in (1), the SC-LDPC code can be decoded by sliding a window along the variable node sequence [3]. The window size (W) should be selected carefully by satisfying the condition W ≥ mcc + 1 [11]. During the window decoding at the position t, the received code block y = [yt, yt+1, . . ., yt+W-1] are obtained as the input of window decoder. The soft information in term of log-likelihood ratio (LLR) values are propagated between the variable nodes and connected check nodes within the window until the maximum number of decoding iteration IMAX is reached. Then, the window decoder will be shifted to operates the next decoding position t + 1. Thus, only the received code block yt will be decided at each decoding position t, where we termed it the “target nodes.” Note that the window decoder can be shifted more than one position to obtain the multiple outputs as proposed in our previous work [11]. Traditionally, the iterative message (LLR) passing with flooding strategy [4] is commonly applied for the window decoding. During the lth iteration, all the variable nodes within the window will propagate the LLR values to all connected check nodes. Similarly, all the check nodes will produce the LLR values, and then propagate to all connected variable nodes. In recent work [5], the window decoding with serial (also called as shuffled) strategy is introduced. Before each variable node will produce the LLR value, the LLR values from its connected check nodes will be requested along the edges connected to the variable node. The simulation results [5] reveal that the BER performances of serial strategy are approximately the same as the flooding strategy by requiring the smaller decoding iteration numbers. To confirm those results, the EXIT chart analysis can be used for tracking the mutual information (MI) variation of LLR values as proposed in the next Section.

5. Conclusion

In this work, the P-EXIT analysis is applied for SC-LDPC code with sliding window decoding. The theoretical performances of window decoding with flooding and serial strategies are examined. Our analysis results reveal that the window decoding with serial strategies provide smaller decoding threshold than those of flooding strategies in any window size. Specifically, the numbers of decoding iteration can be reduced by 45% when the serial strategy is applied.
