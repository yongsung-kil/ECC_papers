Abstract—The approaches to decoding block-permutation low-density parity-check codes in channels with memory are considered. The modification of windowed bit-flipping algorithm for bursty channels is proposed, which is guaranteed to correct bursts within the correcting capabilities of the code. The method of determining the position of bursts is proposed. The simulation results in a memory channel with a limit on the burst length are presented.
Keywords—low-density parity-check codes, channels with memory, bursts, bit-flipping.
I. INTRODUCTION The task of correcting errors in the transmission, processing and storage of information is an integral part of modern information and communication systems. To solve this problem, error-correction coding is used. An important place among modern error correction codes is occupied by codes with a low density of parity checks (LDPC, low-density parity-check) [1-4]. These codes are used in many modern telecommunication standards and communication systems [810]. Traditionally, errors in a channel are assumed to occur independently, but in most real-world communication channels, errors are grouped into packets called bursts. In such a situation, traditional decoding algorithms do not provide the required error probability, which leads to the problem of constructing an algorithm for effective correction of grouped errors.
The paper is organized as follows: Section 2 describes the basic concepts such as the block-permutation construction of LDPC-codes and the classical decoding algorithm. In Section 3 we describe the procedure for determining the length of correctable bursts. Section 4 presents the windowed modification of the bit-flipping algorithm. The simulation results are shown in Section 5. Section 6 summarizes the results.
II. BASIC CONCEPTS Low-density parity-check codes were proposed by R. Gallager in the early 60-s of the last century [1,2]. The parity check matrix of LDPC-codes has a sparsity property, that is, the number of non-zero positions is small compared to the total number of its elements. To set the code, a graph is also used, for which the check matrix is the incident matrix

(Tanner graph). As a rule, the minimum distance of LDPCcode is small, but when using symbol-wise iterative decoders, they provide a high correcting ability [3,4].

This matrix structure allows to obtain a quasi-cyclic code for which effective coding and decoding procedures have been developed [4,7]. The presence of short cycles in the Tanner graph has a negative impact on the work of iterative decoders, so when constructing code, the degrees of permutation matrices are chosen so that in the corresponding graph there are no cycles of length 4. Often, to improve the structure of the code, permutation blocks are interspersed with zero blocks, which leads to a decrease in the number of cycles in the corresponding Tanner graph and, consequently, an improvement in the work of iterative decoders.

IV. ERROR BURSTS DECODING ALGORITHM The bit-flipping algorithm from Section 2 is insensitive to error grouping and can be applied to correct error bursts. However, there may be a large number of erroneous positions

inside the packet, and the classical algorithm may not correct them. Error grouping can be taken into account if we prohibit the inversion of positions that are separated by a certain distance. Thus, a windowed version of the bit flipping algorithm may be considered. Let the decoder consistently consider all possible locations of bursts and try to correct errors in the selected positions. Then, at each decoding iteration, Step 3 of the algorithm (see Section 2) is performed for the positions of the current window of length b of vectors f and y.

The windowed algorithm can be improved for the blockpermutation construction of LDPC-codes. Consider a packet of errors eb , the corresponding part of the matrix Hb and the
syndrome S. If the burst length b does not exceed the size of the matrix block m, then in each submatrix under consideration a non-zero position of the syndrome corresponds to a row in which there is one, two, or zero units (see Figure 1). The latter case means that the current submatrix Hb cannot match the correct position of the error burst, and one can move the decoding window one position further. In the case when the non-zero position of the syndrome corresponds to a row of the Hb containing one unit, then the position of the column containing this unit corresponds to an error, and an attempt is taken to correct it: the found column is subtracted from the syndrome and is not considered further for the current window. Fig. 1, a shows an example of such a window.

After performing the described procedure for analyzing non-zero positions of the syndrome (for the current window), it may turn out that all errors are corrected (all positions of the syndrome will become zero). Otherwise, the situation presented in Fig. 1, b appears, that is, two non-zero positions of the Hb submatrix correspond to each unit of the syndrome. In this case, we continue decoding inside the window using the standard bit-flipping algorithm.

V. SIMULATION RESULTS
For the experiments, a random block-permutation matrix consisting of 4× 8 blocks of size m = 31, was chosen, in which there are no cycles of length 4. This defines the (248, 127)-code at the rate of R ≈ 0.5. Using the procedure for determining the length of the correctable burst described in section 3, it was found that the selected code can correct any error bursts of a length not exceeding b = m −1 = 30.
In the MatLab programming environment, the classical and windowed variants of the bit-flipping algorithm were implemented, as well as the preprocessing algorithm proposed in section 4. During the simulation, an error burst not exceeding b = 30 was applied to the random position of the codeword. The probability of error inside the burst was 0.5.
Fig. 2 shows the dependence of the error probability per frame on the maximum number of iterations of the decoding algorithm for different window sizes in the bit-flipping algorithm. The window sizes were chosen as multiples of the block size.
It can be noted that the error probability curve for the bitflipping algorithm without using a window does not fall below 0.5. When using windows of size 3m and 4m, the probability value gradually stabilizes at a level not lower than 10−3. If the window size is close to the burst length, the probability of error decreases and reaches 10−7 with a limit of 15 iterations.

VII. CONCLUSION To correct the error bursts, the bit flipping algorithm can be used. However, it does not provide an acceptable probability of error. Therefore, the paper discusses the restriction of inverted positions within the window size, and also proposed a modification of the decoding algorithm taking into account the design of block-permutation low-density codes. The experiments did not reveal the error bursts that are not corrected by the proposed decoding algorithm (within the

burst-correction capability of the code). It is assumed that this algorithm is guaranteed to correct any bursts within the correcting ability of the block-permutation LDPC-code. Analytical evidence of this hypothesis is the subject of further research.
