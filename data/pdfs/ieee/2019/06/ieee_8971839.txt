Abstract—This paper presents the mathematical representation and the numerical results of the Gradient Bit-Flipping Algorithm (GBF) that is derived from the Bit-Flipping Algorithm (BFA). The experimental simulation results for the proposed decoder of the Low-Density Parity-Check (LDPC) codes of ATSC 3.0 demonstrate an enhancement of up to 3.55 dB in error correction performance without an increase in the decoding complexity. The proposed decoder has the ability to decode an erroneous LDPC message in fewer iterations than the traditional BFA due to the employment of the values of the Log-Likelihood Ratio (LLR) as an input, unlike the BFA that uses binary values ignoring the channel estimation output.
Keywords—ATSC 3.0, channel coding, LDPC decoder, bitflipping, gradient bit-flipping.
I. INTRODUCTION
Since the introduction of the Low-Density Parity-Check (LDPC) codes by Gallager, they became one of the most used Forward Error Correction codes (FEC), due to their near Shannon limit error correction performance [1], [2]. Therefore, they are used in many Digital Television Broadcasting (DTB) standards such as the Digital Video Broadcasting-SatelliteSecond Generation (DVB-S2), the Digital Video BroadcastingTerrestrial-Second Generation (DVB-T2) and ATSC 3.0 from the Advanced Television Systems Committee (ATSC) [3]–[5].
In recent years, the focus was mainly set on reducing the complexity of the optimal LDPC decoding algorithm, the Sum-Product Algorithm (SPA) devised by [6]. The decoding complexity was still high for many successful decoding methods introduced in the literature such as the Min-Sum Algorithm (MSA), the Offset Min-Sum Algorithm (OMSA), the Normalized Min-Aum Algorithm (NMSA), and the Variable Correction Algorithm (VCMS) [7]–[10].
Although the LDPC codes demonstrate a high error correction performance, they require a code word that is longer than 24000 bits, making the decoding process computationally expensive [11]. This forces the implementation of the decoders to be done in hardware instead of software.
The ATSC 3.0 standard adopts the Bit-Interleaved Coded Modulation (BICM). The BICM uses two FEC layers, an LDPC code in the inner layer and a Bose-ChaudhuriHocquenghem (BCH) code in the outer one [12]. The BCH can be replaced with the Cyclic Redundancy Check (CRC) or

A. The Bit-Flipping Algorithm (BFA)
One way to have a low-complexity LDPC decoder is to use the Bit-Flipping Algorithm (BFA) proposed by Gallager [1].
The BFA is the simplest decoding method for LDPC codes. It calculates the values of the parity-check nodes Cj based on the matrix H and 1, then counts the number of unsatisfied check nodes that are connected to a certain variable node Vi using 2, and determine whether the value of this variable node should be flipped or not, based on the condition 3, where F lipVi is the number of unsatisfied check nodes and X is a decision threshold. The process is repeated until all paritycheck nodes are satisfied or a maximum number of iterations is reached [1].

II. THE PROPOSED GRADIENT BIT-FLIPPING ALGORITHM (GBF)

While the classical BFA ignores the values of the input Log-Likelihood Ratio (LLR), the LLRi, and only consider their signals in order to decide whether to flip the current variable node bit; the proposed GBF takes both the signal of the incoming LLRi and its value into consideration and adjust them based on the number of the satisfied check nodes and the unsatisfied ones to produce the output LLRo.

In GBF, just like in the SPA, the probability of each bit being a logical ZERO or a logical ONE is represented by the value of its variable node combined with its signal.

IV. CONCLUSION
In this paper, we proposed a fully functional LDPC decoder for ATSC 3.0 as shown by the mathematical analysis and the experimental results. The proposed GBF is a derivation of the BFA, and it makes use of the LLR information provided by the channel estimator in order to provide a practical software implementation of an LDPC decoder for ATSC 3.0.
