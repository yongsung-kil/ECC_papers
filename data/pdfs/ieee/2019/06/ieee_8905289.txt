Abstract—Low-density parity-check (LDPC) codes, usually decoded using the informed dynamic scheduling (IDS) algorithms, are well applicable to the 5th-generation communication systems (5G). However, the IDS algorithm is greedy because of the update-relayed trend. The updaterelayed trend leads to the unreasonable allocation of decoding resources, which exerts a huge impact on the decoding performance. To solve this greedy problem, we proposed the decoding algorithm based on random select of check nodes with a predefined update range (RSPUR). By decoding with a narrow update range, the RSPUR algorithm effectively suppresses the update-relayed trend and allocates the decoding resources in a more rational way. Simulation results indicate that the proposed algorithm achieves excellent error-correction performance and high throughput compared to the previous IDS algorithms.
Keywords-low-density parity-check codes; informed dynamic scheduling; error-correction performance; throughput
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] have attracted increasing attention since they were reintroduced in 1995 [2]. LDPC codes are well applicable to the 5thgeneration communication systems (5G) for their nearcapacity performance achieved by decoding with the belief propagation (BP) algorithm [3]. The BP algorithm, where all the variable-to-check (V2C) and the check-to-variable (C2V), messages are propagated in parallel, is also called flooding or parallel scheduling. To accelerate the convergence speed, the C2V or V2C messages can be updated in sequential order, so that the newly updated information can be used to propagate and generate extrinsic messages in the same decoding iteration. Those algorithms [4]-[6] are called standard sequential scheduling (SSS) algorithms and converge twice as fast as the flooding [7].
For the SSS algorithms, the error-correction performance is limited by the fixed messages update order. If the update order can be dynamically arranged according to the decoding state, the error-correction performance would be further improved. This idea leads to the investigations of the informed dynamic scheduling (IDS) strategies [7] [8]. The residual belief propagation (RBP) algorithm [7], which is the primitive IDS algorithm, preferentially updates the C2V

message associated with the maximum C2V residual. Although this residual-based IDS algorithm achieves excellent error-correction performance, it suffers from poor convergence performance because of greediness. Some variable nodes occupy too many computational resources for updates, while the other are seldom updated.
To reduce the impact of greediness, a number of IDS algorithms [9]-[11] are proposed. The lazy queue residual decoding (LQRD) algorithm [9] simultaneously updates all the C2V messages whose residuals are greater than a dynamic threshold. The quota-based RBP (Q-RBP) algorithm [10] evenly distributes the update quotas to each edge and allocates the computational resources more fairly. The oscillating variable nodes based RBP (OV-RBP) algorithm [11] prioritizes the message update according to the reliability of the variable nodes. By increasing the decoding complexity, those IDS algorithms achieve appealing error-correction performance.
Although the improvements of the IDS algorithms mentioned above are encouraging, the greedy problem still exists. The greediness of the IDS algorithms is caused by the update-relayed trend investigated in this paper. The updaterelayed trend refers to such a phenomenon that the edges, whose residuals are recalculated with the updated messages, are more likely to be updated in the following decoding process. If the update-relayed trend propagates along the edges connected in cycles, it will cause those edges to be updated multiple times. As a result, a small number of edges take up too many opportunities for updates, exerting a huge impact on the error-correction performance.
In this paper, we present the decoding algorithm based on random select of check nodes with a predefined update range (RSPUR). Contrary to the previous IDS algorithms where all the edges are available to be updated, the proposed RSPUR algorithm decodes with a narrow update range which consists of a few edges. Since the number of forming cycles is reduced in a small update range, the propagation of the update-relayed trend can be significantly suppressed. Moreover, the decoding complexity is substantially reduced as the update range is narrowed. Simulation results show that the proposed RSPUR algorithm achieves excellent errorcorrection and throughput performance compared to the previous IDS algorithms.

III. DECODING ALGORITHM BASED ON RANDOM SELECT OF CHECK NODES WITH A PREDEFINED RANGE
In the previous IDS algorithms, the edge to be updated is selected from all the edges. Thus, all the edges are contained in the update range. Meanwhile, all the cycles formed by all the edges are also contained in the update range. Tens of thousands of cycles within the update range make the

update-relayed trend easy to spread, greatly increasing the risk that the decoder would get trapped into the greedy state. If the update range can be narrowed, the number of cycles which are contained in the update range will also be reduced, and the update-relayed trend will be effectively suppressed. To achieve this goal, we proposed the decoding algorithm based on random select of check nodes with a predefined range (RSPUR).
In the RSPUR algorithm, the edge to be updated is selected from a small update range containing several check nodes. The maximum C2V residual is searched from all the residuals which are generated by all the check nodes in the update range. Each time the maximum C2V residual is found, the check nodes within the update range would be re-selected. The check nodes are selected into the update range in a random manner. Thus, all the check nodes have the same opportunity to participate in the decoding process. To improve the decoding performance, a suitable update range is necessary to be set for the RSPUR algorithm. However, it is not easy to set the update range because the decoding performance is affected by many factors. A simple approach to determine the update scope is through testing. The sender can transmit some test frames to the receiver. For example, the sender transmits a certain number of full-zero code words to the receiver. Thus, the information of the test frames is available to the receiver. By comparing the decoding performance in different update ranges, the optimal update range of the RSPUR algorithm can be obtained.

is defined as the number of erroneous C2V updates divided by the total number of C2V updates. Thus, the lower the update error rate, the better the decoding performance. Fig. 4 shows the bit error rate (BER) vs. update range for the same LDPC code over the same channel. As shown in Fig. 3 and 4, when the update range is set from 1 to 4 check nodes, the update error rate declines significantly while the BER performance of the RSPUR algorithm is improved effectively. When the update range is larger than 4 check nodes, the update error rate barely changes. Meanwhile, the BER performance of the RSPUR algorithm reaches the best. When the update range is larger than 18 check nodes, the decoder gets trapped into the greedy state and the BER performance is poor. It is reasonable to set a small update range because the number of searching operations can be substantially reduced. Therefore, the update range is set to 4 check nodes. Seemingly, the receiver needs to employ additional decoding resources to get the test results. In fact, once the optimal update range is obtained, there is no need to reset as long as the channel conditions remain unchanged. Therefore, the consumption of the additional decoding resources is acceptable. Through a large number of simulations, we find that for different LDPC codes, the optimal update range is usually very small. For example, for the rate-1/2 length-576 WiMAX [12] LDPC code, the optimal update range is 4 check nodes, and for the rate-1/2 length-2304 WiMAX LDPC code, the optimal update range is 8 check nodes.

IV. PERFORMANCE EVALUATION
In this section, the simulation results are provided to investigate the decoding performances of the proposed RSPUR algorithm, and compare them with those of the previous IDS algorithms, including the LLR-BP [1], CSBP [6], RBP [7], NW-RBP [7], LQRD [9], Q-RBP [10] and OVRBP [11]. The simulations are performed over the AWGN channel with the modulation BPSK. The main configuration in simulations is Intel(R) Xeon(R) CPU E5-2630 and the maximum decoding iterations is set to 5.

V. CONCLUSION When the update-relayed trend propagates along the edges connected in cycles, the IDS algorithms easily get trapped into the greedy state. The irrational allocation of decoding resources causes great damage to the decoding performance. To inhibit the update-relayed trend, the RSPUR algorithm is proposed. By decoding with a narrow

update range, the RSPUR algorithm suppresses the updaterelayed trend and achieves excellent decoding performance. Moreover, by significantly reducing the decoding complexity, the proposed algorithm outperforms the previous IDS algorithms in terms of throughput. Simulation results show that the RSPUR algorithm is acceptable, considering the error-correction performance and throughput performance.
