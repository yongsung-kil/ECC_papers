Reliability Enhancement for Multi-level Cell NAND Flash Memory Using Error Asymmetry

Abstract—This paper presents the jointly use of constraint code (CC) and error correction codes (ECC) for the reliability enhancement in Multi-level cell NAND flash memories. In the proposed system, the constraint code helps transform the user data distribution, adapting to the asymmetry in error behavior of MLC NAND flash memories and the ECC corrects more errors thanks to the prior information from the data distribution. The compatibility of CC and ECC is analyzed, and the information loss is shown to be negligible, especially for the use in MLC NAND flash memories. Simulation under practical MLC NAND flash error model has shown that the proposed scheme can improve remarkably output error rate and reduce read latency in these memories.
I. INTRODUCTION
Multi-level cell (MLC) memories store multiple bits per cell and increase the storage capacity [1] significantly. Thanks to the advantages of storage density and cost, these memories now dominate the global flash memory market [2], e.g, they are replacing the single-level cell (SLC) memory in storage applications. However, the reliability of high-density MLC NAND flash memories is decreasing rapidly due to smaller cell sizes of multi-level cell technologies. The uncorrectable bit error rate (BER) increases significantly, for example, Tanakamaru et al. in [3] showed that the error rate of a 2x-nm MLC NAND flash chip can reach to 10−2.
In a NAND flash memory, all read and write operations are performed at the granularity of a page, which is typically between 8 and 16 kB in size. The storage data is organized by the so-called upper page and lower page as the read/write data units. The error models in MLC NAND flash have been studied in several works on the open literature, and one of the key properties is the error asymmetry [4], [5]. The asymmetry of bit errors in MLC flash chips signifies the fact that the error happens dependently to the stored data, i.e., the error appears to data bit 1 differently to data bit 0 as a function of the memory usage status (aging, operating condition...). The error asymmetry also appears in the level of data page where the error in the upper page is different from that of the lower page. Veeresh Taranalli et al. showed in their imperial work that the error in MLC follows the 2Beta-Binomial distribution (2-BBM). The 2-BBM error model

was also reported to provide accurate frame error rate (FER) performance of ECC under program/erase (P/E) cycling [5], [6]. Several studies have been conducted to enhance the reliability/endurance of the flash memories by exploiting the bit errors asymmetry. In [4], two BCH codes with different capabilities for the upper pages and lower pages have been proposed to achieve better results than using a single ECC code. The data shaping techniques have been introduced to reduce flash cell wear by converting unconstrained data to an encoded sequence which has a prescribed distribution [7]– [9]. The usual case of data shaping is the concatenation of optimal lossless compression with optimal endurance coding to minimizes the total wear cost. In another hand, the more powerful ECCs have been considered to replace the traditional use of BCH code in correcting the error in the memory. In [5], [10]–[12], the Low-Density Parity-Check (LDPC) and the Polar codes have been successfully applied in many highdensity storage chips lying on their superior error-correction performance with reasonably low decoding complexity at high code-rate. Besides, constrained codes (CC) are known as sets of finite-length codewords that avoid occurrences of certain patterns entirely, can reduce cell wear in flash memories by using the levels that are more likely to damage the cells less often [13]. Recently, the use of ECC together/jointly with constellation shaping technics for a high order of modulation for optical communications has been proven to be an efficient way to improve the transmission rate. The constellation shaping techniques modify the constellation points to a pre-defined distribution that maximizes the transmission rate, approaching to the channel capacity (more details can be found [14] in and the references therein).
This paper presents a jointly use of constraint code and error correction codes for enhancing the reliability/endurance of MLC NAND flash memories. We show in a study case that this concatenating system results in a negligible information loss when being used in MLC NAND flash memory. We also show in this case that with information calculated on the priorknown distribution, the error correction of the LDPC and Polar decoders are improved significantly. III. PROPOSED METHOD
A. Jointly use CC and ECC in MLC NAND flash memory

II. BIT ERRORS ASYMMETRY OF MLC NAND FLASH MEMORY
A. Bit Errors Characteristic
Several works have been studied on the asymmetry of bit errors in MLC flash memories, such as in [4]–[6]. Fig.1 shows the bit error asymmetry of one tested MLC NAND flash memory (based on the model from [5]). It can be seen that the bit-error transitions 1 → 0 over P/E cycling are significantly different compared to that of 0 → 1. The discrepancy is larger as the number of P/E increased. It is also seen that this asymmetry in error appears more significant in the lower page than the upper page of an MLC NAND flash. We denote the probability of 1 → 0 as q and that of 0 → 1 as p.
B. 2-BBM Error Model
We use the 2-Beta-Binomial distribution (2-BBM) empirical model characterized in [5], [6] for our design and simulations. The 2-BBM model is constructed from a binary asymmetric channel (BAC) model with transition probabilities p and q are the random values of a Beta distribution. In the error model of MLC NAND flash memory, the values of p and q are controlled by four parameters a, b, and c, d respectively. The empirical parameters a, b, and c, d are the functions of the P/E and are different on the types of MLC NAND flash memory. In this paper, we implemented the 2-BBM model in Algorithm 1 for two different vendor A, B memories by parameters a, b, c, d given by authors of [5], [6]. More details of the model can be found in the references above.

We propose the joint use of the constraint code and ECC, as in figure 2, where the constraint code (CC) is serially concatenated to the ECC. For the memory writing step, the uniform-distributed data is separated into data parts for the upper page and lower page. These data parts are then transformed to a target distribution before being encoded by the ECC encoder and stored in the corresponding pages. In the memory reading, the stored data is error-corrected first by ECC decoder before being converted back by the CC decoder, recovering the uniform-distributed data parts. The user data is then recovered by merging these page data parts. The similar property of this system to the one in [14] is that the CC is used before the ECCs in the writing process and after the ECCs in the reading process. Also, the ECCs used are systematic codes where the input data is embedded in the encoded output. The CC is compatibly used to the ECC in the system, especially when it is applied to the MLC NAND flash memory in the writing and reading processes, is because of the two folds. Firstly, the data with distribution produced by the CC encoder will be preserved when stored in the memory due to the systematic ECC encoder. Although the bit distribution in the whole encoded codewords is slightly changed because of the added parity bits, this change is small when using in the MLC NAND flash memory due to the high-rate requirement in the storage applications. Secondly, due to the deterministic operations of the constrained codes [15], it is vulnerable to add more errors when working directly on the data with incorrect bits read from MLC NAND flash memory. The proposed system is designed to have the CC decoder working on the corrected data by the ECC decoder. By this technique, the error rate at the system output apparently depends only on the

The motivation of the proposed technique is on the asym-

metry nature of flash memory, where the error may appear

less on bits 0 (or 1) of the stored data. The CC in the system

is then used to adapt the distribution (or ratio) of bits 0 (or

respectively, bits 1) to such a preferable one. The advantage

in term of the memory endurance can be explained through

the definition of cell-wear, in which the flash chip lifetime

can be significantly improved by transforming data into bit

patterns which cause minimal wear (see more in [7]). We focus

on the advantage of the error rate of the MLC NAND flash

output. Indeed, the adaptation of bit distribution (or simply

the ratio of bits 0 and 1 in the data) in the stored data

apparently helps reduce the error occurrences. Furthermore,

the prior information provided by the known data distribution

part. We present a simple design of the proposed system where

the ECC codes are the MLC NAND flash LDPC and/or Polar

codes, while the constraint code is a fixed weight code. The

It encodes a k bits information word into a n-bit codeword

in which the hamming weight of this codeword is m. The

fixed weight property is on the fact that the output weight for

all possibilities of the k bits information is fixed to m. The

fixed weight constraint code can be implemented using the

algorithm in [16].

IV. NUMERICAL RESULTS
We implemented the 2-BBM model based on two flash memory chips which have feature sizes 1X-nm from vendorA, and 2Y-nm from vendor-B for error observation [5], [6]. To evaluate the efficiency of using CC to error-correction performance of ECC codes for flash memories. We applied a (N = 8192, k = 7683) regular quasi-cyclic LDPC (QCLDPC) code with dc = 64 and dv = 4, where dc and dv refer to the check node and variable node degrees, respectively. The parity check matrix of the QC-LDPC code is constructed using a size of 128 ×128 circulant permutation matrices, and the design rate is specified as 0.9375. Maximum iteration is limited by 50 to make a fair comparison with the previous work [5]. For Polar codes in our simulation, we use systematic and non-systematic Polar codes (N = 8192, k = 7373), at code-rate 0.9 which is optimized for binary symmetric channel (BSC) (p = 0.001) [5]. The reason for this range is derived from our CC code-rate loss analysis in Section III-B; according to this analysis, the ≤0.1 rate-loss range should be focused in simulation. We confirm the FER performances of QC-LDPC and Polar codes with random data are equivalent to Bernoulli 50% distributed data in all cases. Consequently, ϕ = 50% is selected as the reference point in simulation of FER and average iteration number of QC-LDPC for both memory chips (Fig.4,6). Fig.6(a), 6(b) show the improvement of average iteration number that the QC-LDPC decoder needs to correct defective codewords. These results indicate that the iterative LDPC decoder get faster convergence when using CC. However, we have recorded the inefficiency of using CC for upper pages of both memory vendors. Specifically, Fig.4(a) and Fig.5(a) show the FER performance of QC-LDPC and Polar codes in upper pages of vendor A. It can be noticed that increasing 1’s (ϕ1 increases) using CC only improves slightly the error-correction performance of the LDPC code, and no improvement for the Polar code. We also get the same results when applying the proposed method with upper pages of vendor B memory (not shown due to lack of space). This can be explained by the probability p is comparable with q in P/E range of 2000:8000 with upper pages (implied from Fig.1). Consequently, we consider applying the CC only for lower-pages writing and reading.
