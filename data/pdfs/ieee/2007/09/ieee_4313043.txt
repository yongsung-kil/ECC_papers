Abstract— In this paper, we propose a unified decoding algorithm for linear codes, named doubly extended sum-product algorithm (DESPA). The DESPA is described as a belief propagation algorithm over a generalized normal graph that represents the code based on a partitioned parity-check matrix. In one extreme case, the DESPA minimizes the frame-error-rate; while in another extreme case, the DESPA minimizes the bit-error-rate. In practice, the DESPA can be implemented to make trade-offs between decoding complexity and decoding performance.
I. INTRODUCTION
Since the invention of turbo codes [1] and the rediscovery of low-density parity-check (LDPC) codes [2], iterative decoding algorithms have been extensively investigated. Most iterative decoding algorithms can be considered as instances of belief propagation algorithms defined over certain graphs [3][4][5][6]. It is no doubt that graphical representations of codes have been a popular and convenient way to describe iterative decoding algorithms.
In this paper, we propose a unified decoding algorithm for linear codes that is derived as an iterative belief propagation algorithm on normal graphs. Firstly, the parity-check matrix is partitioned into sub-blocks. Secondly, a normal graph is built based on the partitioned parity-check matrix. Finally, a natural associated decoding algorithm over the normal graph is derived. Equivalently, the normal graph corresponding to the partitioned parity-check matrix can be constructed from the normal graph corresponding to the original parity-check matrix by merging nodes. Hence we named the proposed algorithm doubly extended sum-product algorithm (DESPA). The DESPA has many potential applications to q-ary LDPC codes, LDPC coded modulations, etc. Simulation results show that the DESPA provides a way to trade off the decoding complexity against the decoding performance.
The rest of this paper is organized as follows. In Section II, the normal graph corresponding to a partitioned parity-check matrix is illustrated. An iterative message processing/passing algorithm associated with the normal graph is then derived and analyzed. In Sections III and IV, we discuss the applications of the proposed algorithm. Simulation results are provided in Section V. Section VI concludes the paper.

Remarks
1) If we choose N = 1, the proposed algorithm minimizes the frame-error-rate (FER), as the maximum likelihood sequence decoding algorithm does.
2) If N = n and R = 1, the proposed algorithm minimizes the bit-error-rate (BER), as the BCJR algorithm does [8].
3) Intuitively, for given nj, performance can be improved by increasing ri (hence decreasing R).
4) Increasing nj and/or ri may not always cause complexity increase. This is concluded from that increasing nj and/or ri decreases N and/or R and that increasing nj and/or ri (very likely) reduces the number of iterations. In fact, it would be impossible to count the total required computations for a given partitions without simulations.

with modulations. The first approach is called LDPC-BICM,

which applies an LDPC code to bit-interleaved coded modula-

B. Q-ary Low-Density Parity-Check Codes
We take as an example one of the algebraic q-ary LDPC codes constructed by Lin et al. in [14][15]. The parity-check matrix Hq is a 60 × 225 matrix over F24 with row weight 15 and two different column weights 3 and 4. The null space of Hq defines a 16-ary [225, 173] quasi-cyclic LDPC code. We first transform the q-ary matrix Hq into a binary matrix Hb of size 240 × 900 by replacing each entry in Hq by its binary-matrix representation. Accordingly, we have a binary code [900, 692] with the parity-check matrix Hb. The BER performance and the FER performance of this code over AWGN channels with BPSK signalling are shown in Fig. 6 and Fig. 7, respectively. For (ri, nj) = (4, 4), the proposed algorithm is exactly the same as the algorithm in [14][15].
VI. CONCLUSIONS
In this paper, we have proposed a unified decoding algorithm for binary linear codes based on partitioned paritycheck matrices, called doubly extended sum-product algo-

rithm (DESPA). The DESPA allow us to make trade-offs between complexity and performance and hence has many potential applications. In principle, the DESPA can approach the optimal algorithms by increasing ri and/or nj. But the practical limits on ri and nj imply that the DESPA is only attractive for codes having low-density parity-check matrices and short codes, for example, binary cyclic codes with length n < 128. It should be pointed out that the parity-check matrix may be first transformed into an equivalent form by permutations and then partitioned in a proper way.
