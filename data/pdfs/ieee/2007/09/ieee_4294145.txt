Complete Enumeration of Stopping Sets of Full-Rank Parity-Check Matrices of Hamming Codes
Khaled A. S. Abdel-Ghaffar, Member, IEEE, and Jos H. Weber, Senior Member, IEEE

Abstract—Stopping sets, and in particular their numbers and sizes, play an important role in determining the performance of iterative decoders of linear codes over binary erasure channels. In the 2004 Shannon Lecture, McEliece presented an expression for the number of stopping sets of size three for a full-rank parity-check matrix of the Hamming code. In this correspondence, we derive an expression for the number of stopping sets of any given size for the same parity-check matrix.
Index Terms—Hamming code, linear code, parity-check matrix, stopping set enumerator, weight enumerator.

The notion of stopping sets is important in the context of iterative decoders (using H -based Tanner graphs), in particular for low-density parity-check codes [3]. For example, on the binary erasure channel, an iterative decoder will not lead to successful decoding if and only if the set of erased positions contains a nonempty stopping set. Hence, the minimum (nonempty) stopping set size s and the cardinality Ss are performance indicators for iterative decoding, like the minimum distance d and the number of minimum-weight codewords for maximum-likelihood decoding. On the BCJR Algorithm for Rate-Distortion Source Coding
John B. Anderson, Fellow, IEEE, Tomas Eriksson, Member, IEEE, and Norbert Goertz, Senior Member, IEEE
Abstract—The Bahl–Cocke–Jelinek–Raviv (BCJR) algorithm is an important channel decoding method. We extend it to trellis rate-distortion data compression. Beginning from source coding principles, the derivation of the algorithm avoids channel coding or soft output ideas. The encoder does not use entropy coding; equiprobable reproducer letters are emphasized since these maximize entropy. The BCJR method is demonstrated by tests of a tail-biting variant. It performs much better than the ordinary Viterbi algorithm for short and medium blocks. However, the improvement stems from tail biting; the role of the BCJR is to achieve tail biting in a relatively simple way. Some issues that arise with tail biting are explored. It is shown that there is an optimal trellis state size for each block length.
Index Terms—Data compression, rate-distortion theory, tail biting, trellis coding.
I. INTRODUCTION
Consider a continuous-alphabet independent-letter data source x,
Y with reconstruction values y in alphabet and distortion measure 1 1 d( ; ). The problem of trellis source coding consists of finding a path
through the trellis codeword structure whose corresponding reproducer
values y represent a sequence x with minimum distortion. For the
mean square error (MSE) measure, the well-known Viterbi algorithm (VA) delivers such a trellis path (see [1]).
An ordinary VA begins at an agreed-upon trellis state, and here originates a problem. There is a start-up phase during which the trellis is not fully developed and the average distortion is much worse. The loss can
be accepted or the best starting state out of MS states can be selected and sent with an extra  = log2 MS bits. Either way, the encoder per-
formance moves considerably further from the rate-distortion limit. For shift-register implemented codes, it is clear that the start-up phase lasts
at least  stages, but it is much longer in practice. For example, a code with  = 3 requires 15–20 trellis stages. An illustration of a Gaussian
source digitized at rate 1 bit/sample is given in Section V.
