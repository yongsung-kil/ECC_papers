ABSTRACT
In this paper, we propose a construction method of rate-compatible block-type low-density paritycheck (RC B-LDPC) codes using extending and puncturing, which works well over a wide range of code rates by decreasing the amount of puncturing nodes to mitigate the performance degradation at high rate range. RC B-LDPC code constructed by the proposed scheme to achieve the target code rates from 1/3 to 4/5 results in good frame error rate (FER) performance, even better than the welldesigned rate-1/3 code in the small number of iterations (say, < 20).
I. INTRODUCTION
Low-density parity-check (LDPC) codes have been the main research topic because of the capacity approaching performance and the feasible decoding complexity [1]. However, one drawback of LDPC codes is that the encoding complexity is quadratic in the code length, which results in slow encoding. To overcome this shortcoming, block-type LDPC (B-LDPC) codes, which are fast encodable and memory efficient, were proposed and adopted in IEEE 802.16e standard [2].
Rate compatibility always has been considered in the design of practical systems such as hybrid automatic repeat request (HARQ) because the retransmission based on incremental redundancy is possible and only one encoder/decoder pair is necessary. For this purpose, rate-compatible puncturing of LDPC codes has been studied in [3], and another approach using both extending and puncturing is attempted in [4] and [5].
As the puncturing proportion increases, the performance of punctured LDPC codes becomes worse than that of unpunctured LDPC codes when the iteration number is not sufficient (say, < 20) because punctured LDPC codes are decoded by giving 0 LLR values to the punctured nodes, which slows the decoding convergence speed. To solve this problem, RC B-LDPC codes are constructed using both extending and puncturing. This approach

decreases the amount of puncturing to achieve the target rates and mitigates the performance degradation at high rate range. Another advantage of extending is the low decoding complexity. While, for the puncturing only, all parity bits are generated at the encoder whether they will be used or not, the extending generates parity bits only as needed, which avoids unnecessary computations.
In this paper, we propose new extending scheme which enables linear-time encoding and improves the performance by considering the degree distributions and the dependency between the information and the extended parity part, as discussed in [4], [5]. It is shown that the proposed scheme also provides faster and less complex encoder and decoder, compared with the punctured case. Simulation results confirm that the proposed scheme outperforms the puncturing scheme, especially when the number of iterations is small.

B. Puncturing Puncturing has been widely used in error correcting codes to achieve the rate compatibility [6]-[8]. For LDPC codes, the performance becomes better as the check node degree distribution becomes more concentrated. Therefore, the number of distinct check node degrees should be as small as possible after puncturing [9].
One construction method of the rate-compatible puncturing patterns is that the parity bits to be punctured are selected such that the remaining unpunctured parity bits are located as regularly as possible. Through proper puncturing, higher rate codes are obtained from a lowest rate mother code. Although puncturing provides a viable solution to produce RC B-LDPC codes, it is efficient only when the amount of puncturing is not so large.
C. Extending Extending builds low rate codes from a high rate mother code by adding more parity bits. These additional parity bits are generated such that the extended code provides sufficiently good performance at lower rate.

Note that we put all-zero matrix below the parity section to reduce the complexity. Also, the additional parity bits can be generated independently of the previous parity bits and the decoding can be performed independently.

A. Construction of Extended Blocks We construct the extended blocks by considering the following criteria. The concept of these criteria is well known in many literatures such as [1].
Criterion 1 : The number of distinct check node degrees of extended blocks should be as small as possible.

Criterion 2 : Circulants should be added to have the dependency between the information and the extended parity part.
Criterion 3 : Circulants should be added at some columns of high Hamming weight in a mother code to increase the maximum variable node degree.
In [10], the extending uses one column of Hamming weight 3 to enable linear-time encoding and avoid degree-one variable nodes. However, it is known that degree one variable nodes might be allowed if they are used in a controlled way [11]. Since the proportion of degree two variable nodes is large enough in the proposed scheme, one column of weight 1 is used at each extended block without performance loss. We propose the following construction procedures.
Construction Procedures of the Extended Blocks
(1) In the additional columns, locate a degree-one column and the remaining degree-two columns in a diagonal form.
(2) Determine the check node degrees for the extended block from Criterion 1.
(3) Locate circulants in the extended information section in a diagonal form to provide the dependency between information bits and the additional parity bits from Criterion 2. This method gives the dependency by using the minimum number of circulants. If necessary, add more circulants in the circular shifted diagonal pattern according to the check node degree obtained in (2).

IV. SIMULATION RESULTS
For the simulation, we use the three extended base matrices in Fig. 3 for the rate-1/2 12 by 24 model matrix [2] as a mother code and rate-1/3 16 by 24 model matrix [13] as a mother code for puncturing only. In the proposed scheme, to cover the code rates from 1/3 to 4/5, extending is used when the target code rate is lower than 1/2 and puncturing when it is higher than 1/2. We apply only puncturing scheme to the rate-1/3 mother B-LDPC code to achieve the rates from 1/3 to 4/5.
For the simulation, BPSK modulation, AWGN channel, 528 information bits, and belief propagation algorithm are used. Noiseless feedback channel and infinite buffer size at both transmitter and receiver are also assumed.
Fig. 4 compares the FER performance for various iteration numbers, where P and E denote the puncturing only scheme and extending and puncturing scheme, respectively. Fig. 4 confirms that the proposed scheme outperforms the puncturing only scheme, especially when the number of iterations is small.
In particular, Case 1 not only has the lowest complexity but also shows the best performance when the number of iterations is large. Though there exist three all-zero columns when code rate is 2/5 in Case 1, the small number of short cycles and the properly adjusted degree distributions compensate this defect. Case 3 does not perform well as the other cases due to the large number of short cycles generated by putting more circulants in the extended information section.
Simulation results show that the performance gain increases as the code rate becomes higher, which shows that the proposed scheme is suitable for ARQ system where the performance at high rate range is crucial.
V. CONCLUSION
We proposed the systematic construction method of RC B-LDPC code using extending. Simulation results show that the proposed scheme is better than the puncturing only case, and constructs RC B-LDPC codes suitable for a variety of packet data applications. By using the proposed scheme, various RC B-LDPC codes can be obtained and we can choose the best one suitable for the target application.
