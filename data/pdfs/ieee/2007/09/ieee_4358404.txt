22nd IEEE International Symposium on Defect and Fault Tolerance in VLSI Systems

Reconstruction of Erasure Correcting Codes for Dependable Distributed Storage System without Spare Disks
Haruhiko Kaneko and Eiji Fujiwara
Graduate School of Information Science and Engineering, Tokyo Institute of Technology hkaneko@fuji.cs.titech.ac.jp fujiwara@cs.titech.ac.jp
Abstract
Demand is increasing for large-scale dependable storage systems for use in data-intensive servers, such as database and multimedia servers. Conventional dependable disk arrays utilize erasure correcting code to recover lost data stored on failed disks, and require two types of redundant disks: check disks to store check bits, and spare disks to replace failed disks. To reduce the number of redundant disks, this paper proposes a distributed storage system that does not require spare disks. Specifically, the proposed system replaces a failed disk with a check disk rather than with a spare disk. Erasure correction capability of the system gradually degrades with accumulation of failed disks. This paper proposes a reconstruction method for the parity-check matrix of low-density triple-erasure correcting code. Evaluation of the mean time to data loss (MTTDL) shows that, for a storage system with 120 information disks, the proposed storage system with 21 check disks gives an MTTDL of 7.9 × 104 years, while a conventional system with 26 redundant disks gives an MTTDL of 5.8 × 101 years, where the mean time to failure of each disk is 0.5 × 106 years and the maintenance interval of each system is 5,376 hours (i.e., 32 weeks).
1: Introduction
Dependable large-scale storage systems are essential components of data-intensive computer systems such as database servers, large-scale grid computers, and multimedia servers. A redundant array of independent disks (RAID) [1] is widely used as a dependable file storage system. For example, RAID-5 efficiently improves storage reliability by employing single-erasure correcting parity-check code to recover from single-disk failures. Although single-erasure correcting code is sufficient for small- to medium-sized storage systems, it is insufficient for large-scale storage systems with large numbers of disks because the probability of simultaneous multiple-disk failure is non-negligible. Hence, some classes of doubleerasure correcting codes have been proposed for large-scale storage systems, such as ReedSolomon code used in RAID-6, EVENODD code [2], and low density maximum distance separable (MDS) code with minimum distance three [3]. Low-density MDS codes also have been proposed for correcting triple-disk failures [4], and t-disk failures, where t ≥ 4 [5][6].
To quantitatively evaluate erasure correcting codes for large-scale storage systems, Hellerstein et al. have proposed the following four performance metrics [7]:
• Mean time to data loss (MTTDL). • Check disk overhead: the ratio of the number of check disks to the number of infor-
mation disks. • Update penalty: the average number of check disks that should be updated when an
information disk is updated.

In general, conventional erasure correcting codes have a large group size, which means that the disk recovery process invokes many read accesses to non-failed disks. A small group size is preferable, especially for large disk arrays, because large-scale storage systems should operate normally even when some disks are in the recovery process. Low-density tripleerasure correcting codes with small group sizes have been proposed for large-scale storage systems [7].
Unlike conventional storage systems with a centralized controller, such as a RAID controller, distributed storage systems are networked collections of autonomous disk drives. The decentralized structure enables flexible system design and improved data transfer rates. To recover from simultaneous triple-disk failures in a distributed storage system, low-density triple-erasure correcting codes and distributed encoding/decoding algorithms have been proposed [8][9]. It has been shown that these low-density triple-erasure correcting codes give sufficient MTTDL for large storage systems, such as a system with 120 information disks. Also, the codes have a low update penalty and a small group size, which efficiently prevents network congestion. On the other hand, a weakness of the low-density triple-erasure correcting code is that the check disk overhead is higher than that of other conventional triple-erasure correcting codes.
Note that the redundant storage system usually requires spare disks for replacing failed disks, which has not been explicitly discussed in the previous research on erasure correction coding for this type of storage systems. Hence, a redundant storage system has two types of redundant disks: check disks to store check bits and spare disks to replace failed disks. In contrast, this paper proposes a storage system that does not require spare disks. The proposed system replaces failed disks with check disks, and hence the erasure correction capability of the system gradually degrades with accumulation of disk failures. This paper presents a reconstruction method for the parity-check matrix of low-density triple-erasure correcting codes having small group size.
This paper is organized as follows: Section 2 provides preliminaries for erasure correction coding in storage systems. Section 3 proposes a dependable storage system without spare disks. Section 4 presents the parity-check matrix reconstruction method for the proposed storage system. Section 5 evaluates the MTTDL of the proposed storage system, and Section 6 concludes the paper.

This paper employs the following systematic parity-check matrix:

H = PM×K IM ,
where PM×K is an M × K matrix, and IM is the M × M identity matrix. A codeword is expressed as c = (d0, d1, . . . , dK−1, p0, p1, . . . , pM−1), where di, i ∈ {0, 1, . . . , K − 1}, is an information bit, and pj, j ∈ {0, 1, . . . , M − 1}, is a check bit. We consider a storage system with K information disks, D0, D1, . . . , DK−1, and M check disks, P0, P1, . . . , PM−1. The i-th information bit di corresponds to a bit in Di, and the j-th check bit pj corresponds to a bit in Pj. Figure 1 illustrates the relation between a systematic parity-check matrix and disk data.
Hamming code [9] is a linear block code of distance d = 3 defined by an M × N paritycheck matrix H whose column vectors are all distinct and non-zero, where N = 2M − 1. Hamming code is an optimum double-erasure correcting code satisfying Eq.(1) with equality. Figure 2(a) shows an example of H for the Hamming code with M = 4 and N = 15. Odd-weight-column (OWC) code [9] is a linear block code of distance d = 4 defined by an M × N parity-check matrix H whose column vectors are all distinct and are of odd weight, where N = 2M−1. OWC code is an optimum triple-erasure correcting code satisfying Eq.(2) with equality. Figure 2(b) gives an example of H for the OWC code with M = 5 and N = 16.
Low-density triple-erasure correcting codes [8][9], such as Steiner code and additive-
3 code, are defined by a low-density parity-check matrix whose column vectors in the
information part have a constant weight of three. Figure 2(c) shows the parity-check matrix of Steiner code with M = 9 and N = 21, while Fig. 2(d) gives that of additive-3 code with M = 9 and N = 19. These low-density triple-erasure correcting codes can be encoded and decoded using the distributed algorithm proposed in [8].

3: Dependable storage system without spare disks

Figure 3(a) shows a conventional disk storage system with K information disks, M check disks, and L spare disks. The spare disks are used only for replacing failed disks, and are not used if no disk failure occurs. The check disks are used only for storing check bits of an erasure correcting code, and cannot be used for replacing failed disks. Unlike conventional storage systems, which clearly distinguish the functions of check and spare disks, here we propose a storage system in which failed disks can be replaced with check disks. As shown in Fig. 3(b), the proposed storage system has K information disks and M check disks, but no spare disks. When no disk failure occurs in the storage system, all check disks are used for storing check bits. If disk failures occur, selected check disks are used to replace the failed disks, and lost data on the failed disks are restored to the check disks using erasure correcting code. Specifically, when s disk failures occur in the system, s check disks are used to replace the failed disks, and the remaining M − s check disks are used for storing check bits. Erasure correction capability of the system lowers when the number s of failed disks increases. The proposed storage system requires reconstruction of the parity-check matrix because the number of check disks changes dynamically depending on the number of failed disks. The matrix reconstruction will be given in Section 4.

4: Reconstruction of the parity-check matrix

We consider here the system where s failed disks have been replaced, and therefore s rows and s columns have been deleted from the original matrix HKM . Now, we start dealing with an (M − s) × (K + M − s) systematic parity-check matrix of erasure correcting code

expressed as follows:

HKM−s = P(M−s)×K IM−s ,

where P(M−s)×K is an (M − s) × K matrix, and IM−s is the (M − s) × (M − s) identity
matrix. The storage system proposed in Section 3 requires a series of M parity-check matrices, HKM , HM K −1, . . . , HM K −s, . . . , HK2 , H1K , to recover from disk failures, where K is the number of information disks, M the total number of check disks, and s the number of

check disks used as spare disks.

As shown in the example in Section 3, matrix HKM−s, where s ∈ {1, 2, . . . , M − 1}, is generated by modifying HKM , that is, matrix HM K −s is recursively derived from HM K −s+1 as described in the following procedure.
(1) Generate the (M − s) × (K + M − s) matrix
HM K −s = h0 h1 . . . hK+M−s−1
by deleting the rightmost column vector and the bottom row vector from HKM−s+1, where hi is the i-th column vector in HM K −s. (2) According to the upper bounds on code length N = K + M − s of the distance-3 and distance-4 codes shown in Theorems 3 and 4, modify HM K −s to generate
HKM−s = h0 h1 . . . hK+M−s−1
which defines the code with the largest minimum distance for a given M, s, and K, where hi is the i-th column vector in HKM−s. The generation method is given as follows:
– If (K + M − s) ≤ 2M−s−1, then HKM−s is generated as a parity-check matrix of triple-erasure correcting OWC code with minimum distance d = 4. That is, if hi, i ∈ {0, 1, . . . , K + M − s − 1}, has odd weight, then hi = hi; otherwise, some elements in hi are inverted so as to generate an odd-weight column vector hi distinct from all the other column vectors in HM K −s.
– If 2M−s−1 < (K +M −s) ≤ 2M−s −1, then HM K −s is generated as a parity-check matrix of double-erasure correcting Hamming code with minimum distance d = 3. That is, if hi, i ∈ {0, 1, . . . , K+M−s−1}, is distinct from all the other vectors in HM K −s, then hi = hi; otherwise, some elements in hi are inverted so as to generate a column vector hi distinct from all the other column vectors in HKM−s.
– If 2M−s −1 ≤ (K +M −s), then HM K −s is generated as a single-erasure correcting code with minimum distance d = 2. That is, if hi, i ∈ {0, 1, . . . , K + M − s − 1}, is a non-zero column vector, then hi = hi; otherwise, some elements in hi are inverted so as to generate a non-zero column vector hi.
Because the inversion of matrix elements in generating HM K −s from HM K −s induces check disk updates, the number of inversions should be minimized. Derivation of the optimal inversion method involving the minimum number of check disk updates is left for future work. 6: Conclusion
This paper has demonstrated a dependable distributed storage system with no spare disks, and a parity-check matrix reconstruction method suitable for the proposed storage system. Unlike conventional storage systems having information, check, and spare disks, the proposed system does not require spare disks; instead of using spare disks, erased data on failed disks are restored to check disks. Since the number of check disks in the proposed system decreases when disk failure occurs, the erasure correcting code should be dynamically modified to maintain sufficient erasure correction capability. The code modification is accomplished by a simple reconstruction method for the parity-check matrix of the low-density triple-erasure correcting code. Evaluation of the MTTDL using an absorbing Markov chain has shown that the proposed system with K = 120 information disks and M = 21 check disks has longer MTTDL than the conventional system with K = 120 information disks, M = 21 check disks, and L = 5 spare disks. For example, for a long maintenance interval of 5, 376 hours, the MTTDL of the proposed system is 7.9 × 104 years, whereas that of the conventional system is 5.8 × 101 years, where MTTF of each disk is 0.5 × 106 hours. This result indicates that the proposed system is more cost-effective and more reliable than the conventional system.
In future, we will evaluate the throughput, latency, and repair time of the proposed distributed storage system from a practical viewpoint.
