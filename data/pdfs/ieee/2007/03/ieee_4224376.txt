A Novel Scheme for Type-II Hybrid ARQ Protocols using LDPC Codes

Liang Chen, Xiumin Shi†, Shijun Yan, Ziyu Wu, Wenjun Zhang, Yunfeng Guan
Institute of Image Communication and Information Processing Shanghai Jiao Tong University, Shanghai, China
†Department of Electronic Engineering, Beijing Institute of Technology, Beijing, China

Abstract—In this paper, we propose a novel type-II hybrid ARQ scheme using low density parity check (LDPC) codes. The proposed approach combines low hardware overhead with good decoding performance by using an efficient decoder operating at a much higher rate with a much smaller size parity check matrix. Our scheme makes parts of the previously received data gradually improved until successful decoding of the entire original codeword. We also present a novel efficient framework for constructing rate-compatible LDPC codes applied in our hybrid ARQ scheme. The progressive edge growth (PEG) construction method with zigzag pattern results in linear-time encoding.
Keywords—hybrid ARQ protocols, low density parity check (LDPC) codes, rate compatible, decoding, progressive edge growth (PEG) construction
I. INTRODUCTION
Applications of forward error correction (FEC) coding usually consist of the selection of fixed rate coding schemes that are well suited to the channel characteristics and the acceptable error rates for the data to be transmitted. In some cases, however, a more flexible scheme may be desirable to accommodate different error protection requirements, or channels with unknown or time-varying parameters. In such cases, automatic repeat-request (ARQ) protocols have been successfully employed to adapt to changes in the transmission medium. In particular, hybrid ARQ/FEC protocols have been established which exploit both the predictable performance of FEC codes and the rate flexibility of ARQ protocols [1]-[5]. To lower the average transmitted amount of redundancy information, type-II hybrid ARQ protocols have been introduced. For the initial transmission, only a small amount of parity information is transmitted with the actual data. If decoding fails, additional parity information is transmitted incrementally to form lower rate codes. To this end, rate compatible (RC) codes are often used. These allow the construction of a high rate code from a low rate code by puncturing parts of the parity information. Various FEC codes, like convolutional codes [1], turbo codes [2], and LDPC codes [3] have been used in this scheme.
In [3], Li and Narayanan show that puncturing alone is insufficient for the formation of a sequence of well-performing LDPC codes across a wide range of rates. The problem with

puncturing occurs at higher rates where a large percentage of punctured bits paralyze the iterative soft-decision decoder. To overcome this, in addition to puncturing, Li and Narayanan used extending (adding columns and rows to the code’s parity check matrix to produce lower-rate RC-LDPC codes) to create a sequence of rate-compatible codes from a single regular LDPC mother code. In [4], Yazdani and Banihashemi improve the method of extending to construct lower-rate LDPC codes based on an irregular LDPC mother code optimized for a single rate, instead of the regular one in [3].
The proposed hybrid ARQ schemes have certain disadvantages for mobile applications. In an RC-scheme the decoder hardware is not efficiently used, because it always operates at the lowest code rate, requiring a large parity check matrix. Unfortunately, with growing matrix dimensions the chip area dedicated to the decoder also increases. Another issue associated to the matrix size is power consumption. Typically, larger circuits will also consume more power due to the increased activity within the decoder module. Further, in deep sub-micron designs the leakage power becomes an important issue. Leakage power is proportional to the chip size and hence grows with the matrix size. As a result, for mobile applications, the LDPC matrix should be chosen as small as possible.
In this work we present a new scheme for type-II hybrid ARQ protocols with a novel efficient framework for constructing rate-compatible LDPC codes, which combines the advantages of the previous schemes:
• The decoder architecture always operates on the same low complexity decoding hardware. Thus, energy consumption and circuit size can be reduced.
• If the initial redundancy is not sufficient, only additional parity packets rather than entire packets must be retransmitted. As a result, the throughput degrades gracefully with a rising noise level.
A detailed description of the presented scheme is given in Section III.

An LDPC code is said to be irregular if its parity check
matrix has multiple column weights and/or multiple row weights.

The performance of an LDPC code with iterative decoding depends on a number of code structural properties. One such property is the girth of the code which is defined as the length of the shortest cycle in the Tanner graph of the code. For a code to perform well with iterative decoding, its Tanner graph must not contain too many short cycles. The shortest cycles that affect code performance the most are cycles of length 4. Therefore in code construction, cycles of length 4 must be avoided, which the column-row constraint ensures. Many experimental results show that the error-floor of an LDPC code with iterative decoding very much depends on the minimum distance of the code and hence it is necessary to keep the minimum distance of an LDPC code reasonably large. For an irregular code with iterative decoding, its error performance also depends on the degree distributions of the variable nodes and check nodes of its Tanner graph [7].

LDPC codes belong to the class of linear block codes. They can be described by a generator matrix G besides a parity check matrix H. An information word i = {i1, i2, ..., ik} with k symbols is mapped to a codeword c = {c1, c2, ..., cn} with n symbols by multiplying it with the k × n generator matrix G, i.e. c = i · G. In our case the code bits c are transmitted over an additive white Gaussian noise (AWGN) channel as the symbols x = {x1, x2, ..., xn | xl = 1 − 2 ·cl }. The received channel symbols y = x + n(t), where n(t) is AWGN, are demodulated to a received codeword r = {r1, r2, ..., rn | rl = ½ (1-sign(yl))}. r is a valid codeword if the syndrome vector z = {z1, z2, ..., zm}, which is determined by the multiplication of r with the m × n parity check matrix H, is zero, i.e. r · HT = z = 0.
If an error has occurred during transmission, z is non-zero. In this case, the codeword c' = {c1', c2', ..., cn' | cl' ∈ {0, 1}} that is closest to r and that fulfills c' · HT = 0 must be determined. To this end, the belief propagation (Log-BP) algorithm [6] is used. The algorithm is initialized with the channel reliability information Lch (cl | yl) obtained from the received channel values y. III. HYBRID ARQ SCHEMES USING RC-LDPC CODES

A. ARQ System Using RC-LDPC Codes
A typical hybrid ARQ system uses both error correction codes (ECC) and error detection codes (EDC, such as a cyclic redundancy check (CRC)). After the transmitted codeword is decoded by ECC, it is examined by EDC. If the decoding is judged in error, an NACK is sent as a retransmission request. When LDPC codes are used in an ARQ protocol, their strong error detection ability enables them to act as both ECC and EDC. This eliminates the need for another EDC, and thereby reduces the overhead. Whether the error detection capability of LDPC codes can match that of a CRC is not largely discussed in this paper. Nevertheless we assume that the error detection capability provided by the LDPC code is sufficient for the application at hand.

The type-II hybrid ARQ system under investigation uses rate compatible codes along with code-combining and packetcombining to ensure the transmission reliability and to maximize the throughput. A packet is first transmitted using the highest rate code. If it is not judged correctly decoded, an NACK is fed-back and a new set of parity bits is provided by the transmitter (incremental redundancy). This new set of parity bits, combined with all previous transmissions, is treated as a codeword of a lower rate code in the family (code combining) which provides stronger error correction capability. This procedure continues until all supplemental parity bits are used up, and then the procedure restarts with another “initial transmission”. When a new copy of the same coded bits (either information or parity bits) are received, old copies are not discarded, but are combined with the new ones to facilitate decoding (packet combining). In general, packet combining is done by averaging the soft decision values from the multiple copies, and on AWGN channels, this is equivalent to maximum-likelihood (ML) diversity combining. B. Novel framework for constructing RC-LDPC codes
In the previous type-II hybrid ARQ schemes, the decoder always operates at the lowest code rate, requiring a quite large parity check matrix which results in more chip area, more power consumption and more leakage power in the hardware implementation, but there is a main advantage that all higherrate RC codes constructed by parity puncturing can be decoded by the single decoder. Here, to the contrary, we propose a framework for constructing RC-LDPC codes that lower-rate codes can be always decoded by a decoder operating on a basic, high-rate LDPC code.
We can attain the end by adding new rows to the original parity check matrix to form new matrices corresponding to lower-rate codes. The new matrices H' enlarge the mother matrix H longitudinally and retain the whole original rows as the part of new matrices which correspond to check equations. Therefore, any codeword r' at lower rates which satisfy
r'H' T = 0 must naturally satisfy
r'HT = 0 so that r' can be decoded by the decoder designed for the higher-rate code, which corresponds a smaller parity check matrix.
In order to implement a simple encoding scheme, we employ LDPC codes constructed based on PEG method [9] with zigzag pattern as the mother code. We construct Tanner graphs with large girth by progressively establishing connections between variable and check nodes in an edge-byedge manner. The resulting LDPC codes constructed with the PEG algorithm has elegant properties with respect to both the girth and the minimum distance.
When new rows are added, the number of columns of the various parity check matrices associated with the diverse code rates never changes, which means that the set of variable nodes in Tanner graph keeps invariable, but some columns corresponding to data bits of a codeword of the previous code will become corresponding to parity bits after adding rows, which means that new check nodes in Tanner graph are supplemented. We connect new edges to these new check nodes with zigzag pattern by the previous approach as well.
The PEG algorithm presented in [9] is applied to variable nodes. Here we extend its application to check nodes. The theorems on stability condition in [7, 8] imply that the selection of a uniform (or nearly uniform) check node degree results in good threshold performance. It is easy to calculate the mean of the row weights of the original matrix and then round off to a nearest integer. We employ this integer as the weight of all newly added rows to construct new matrices. Since the degree of new check nodes is known, it is easy to apply a modified PEG algorithm to these check nodes, which establishes the connections between the set of newly added check nodes and updated data subset whose elements correspond to the data bits of an LDPC codeword in the set of variable nodes.

form (the rightmost column has weight 2) and the new rows are added on the top of the previous matrix. This structure makes all the RC codes in the sequence linear-time encodable [9, 10]. Moreover, using PEG construction for both the mother code and newly generated RC codes improves the girth distribution [9] of the Tanner graphs, and thus the performance of a sequence of RC-LDPC codes.
To expand the range of applied rates, we can also delete rows from the lowermost rows to achieve higher rates, thus keeping the matrix structure for linear-time encoding as well. However, the higher-rate codes corresponding to the parity check matrices produced by deleting rows may not be decoded by the decoder of the mother code. We adjust the mother matrix from approximate lower triangular form (the rightmost column has weight 2) to normal triangular form (the rightmost column has weight 1, which makes the variable degree distribution slightly deviate from the former, but in practice, this has a negligible effect on performance [4]). In this case, it is easy to find that deleting the lowermost rows is equivalent to implement parity puncturing. Like puncturing, the decoder can perform the decoding algorithm as in an original case.

C. Novel Scheme for Type-II Hybrid ARQ Protocols
In our novel type-II hybrid ARQ scheme based on our construction of RC-LDPC codes, a packet is first transmitted using the highest rate code. If it is not judged correctly decoded, an NACK is fed-back. A set of the parity bits which are equivalently punctured in the preceding case at higher rate are retransmitted. This set of parity bits replaces the counterpart in the preceding codeword. The receiver then decodes the updated codeword, which is more likely successful due to the lower code rate. IV. CONCLUSION
In this paper we propose a novel scheme for type-II hybrid ARQ protocols with an efficient framework for constructing RC-LDPC codes for this scheme. The encoding of designed RC-LDPC codes can perform in a linear complexity while the designed decoder architecture always operates on the same low complexity decoding hardware at a higher rate. Thus, energy consumption and circuit size can be reduced. In our novel typeII hybrid ARQ scheme, we gradually improve the BER of specific data parts. As a result, the average BER of the entire codeword is also lowered so that finally the preceding parity information is sufficient for successful decoding of the entire codeword. In addition, the throughput degrades gracefully with a rising noise level due to gradual incremental redundancy. To achieve high throughput, it is also crucial to choose a strong code for the scheme.
