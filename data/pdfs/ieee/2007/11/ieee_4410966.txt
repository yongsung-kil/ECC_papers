Error Floor Estimation of Long LDPC Codes on Partial Response Channels

Xinde Hu, B.V.K. Vijaya Kumar, Zongwang Li, and Richard Barndt E-mail: xindeh@andrew.cmu.edu, kumar@ece.cmu.edu zli@exaltcom.com richard.barndt@st.com

Abstract—The presence of error floor in low density parity check (LDPC) codes is of great concern for potential applications of LDPC codes to data storage channels, which require the error correcting code (ECC) to maintain the near-capacity error correcting performance at frame error rate as low as 10−12. In order to investigate the error floor of LDPC codes under partial response channels used in data storage systems, we propose a new estimation method combining analytical tools and simulation, based on the concept of trapping sets. The definition of trapping sets is based on the dominant error patterns observed in the decoding process. The goal is to accurately estimate the error rate in the error floor region for certain types of LDPC codes under the partial response channel and further extend the frame error rate down to 10−14 or lower. Towards this goal, we first use field programmable gate array (FPGA) hardware simulation to find the trapping sets that cause the decoding failure in the error floor region. For each trapping set, we extract the parameters which are key to the decoding failure rate caused by this trapping set. Then we use a much simpler in situ hardware simulation with these parameters to obtain the conditional decoding failure rate. By considering all the trapping sets we find, we obtain the overall frame error rate in the error floor region. The estimation results for a length-4623 QC-LDPC code under the EPR4 channel are within 0.3 dB of the direct simulation results. In addition, this method allows us to estimate the frame error rate of a LDPC code down to 10−14 or lower.
I. INTRODUCTION
The near Shannon-limit error correcting performance of LDPC codes [1][2] has led to interest in applying them in data storage systems. However, there are open issues about the LDPC codes. LDPC codes exhibit “waterfall” performance curves as the error rates drop very fast after certain SNR threshold. After the “waterfall” region, the slope of the curve often decreases dramatically, often referred to as the error floor (the bottom of the “waterfall”). In [3], MacKay and Postol discovered in simulation a “weakness” in the Margulis construction of a regular (3, 6) Gallager code that gives rise to a relatively high error floor. For data storage applications, the error correcting codes are desired to maintain the nearcapacity performance even at bit error rates as low as 10−1210−15. Thus, the error floor behavior of LDPC codes is a big concern for data storage applications.
Several research efforts have been devoted to estimating the error floor of LDPC codes for different channels. For

Certain subsets of the variable nodes in the Tanner graph (called stopping sets) decide the final undetermined variable nodes (errors) under iterative decoding. For channels with Gaussian noise, the error floor analysis is much more complex. Certain subsets of the Tanner graph, called trapping sets, are believed to be the primary reason for error floor [5] in additive white Gaussian noise (AWGN) channel. In [5], Richardson uses the concept of trapping sets (definition based on the dominant error patterns) to estimate the error floor. Using this method, the estimation results come within 0.1 dB of the Monte-Carlo simulation result. Also based on trapping sets, Milenkovic et al. [6] developed an analytical method using large deviations theory and statistical methods, which describes certain structures and distribution of trapping sets for particular LDPC codes under AWGN channel. For partial response (PR) channels used in data storage systems, the behavior of error floor remains an open issue. In [7], Sun et al. simulated an LDPC codec on PR channel using FPGA hardware to take advantage of the simulation speed. The hardware simulation is able to achieve BER of 10−12 and frame error rate (FER) of 10−8 for a length-4608 LDPC code. The initial part of the error floor can be observed using the Monte-Carlo simulation. However, even with the fast FPGA simulation, we can not observe error floors at much lower BER/FER regions. In addition, for LDPC codes with relatively low error floor (10−10 BER or lower), the direct simulation method might not be able to accurately estimate the initial part of the error floor.
In this paper, we propose a new method to estimate the error floor of LDPC coded PR channel using a combination of analysis and FPGA simulation. The method is based on the concept of trapping sets. II. TRAPPING SETS OF LDPC CODED PR CHANNEL
A. LDPC Coded PR Channel
Partial response channels are widely used in data storage systems to increase the recording density by controlling the inter-symbol interference (ISI) to a desired target response known as the partial response target. A channel detector (typically Viterbi detector) is needed to calculate soft information of each variable node for the iterative soft decoding employed by the LDPC decoder. The LDPC code decoding process is based on the log-likelihood ratio (LLR) obtained from the soft output Viterbi detector. Different PR targets (e.g., EPR4: [1 1 −1 −1], PR2: [1 2 1]) are used for magnetic recording channels. In this effort, we assume that the channel is perfectly equalized to the targeted PR response from the channel. Thus the channel output is equivalent to ideal PR channel response of the coded binary input sequence plus additive white Gaussian noise (AWGN). The channel model is shown in Fig. 2.
Under PR channel, different types of LDPC codes have shown excellent performance [7]. Quasi-cyclic (QC) LDPC codes, defined by a parity check matrix H matrix consisting of circulant matrices, can be implemented with very efficient encoding and decoding structures [8]. In this effort, we will focus our attention on the QC-LDPC codes. Depending on the

B. Trapping Sets
The error floor of LDPC codes was initially discovered by MacKay et.al. [3], who observed that for a certain type of Gallager code, the slope of the error rate curve decreases dramatically after certain SNR level in AWGN channel. The source of the problem was identified as “near codeword”. In more recent work, Richardson [5] introduced a similar concept of trapping sets to analyze the error floor behavior. For a given channel, the trapping set is defined based on the behavior of the decoder.
Definition [Trapping Sets]: A trapping set of a code, given a representation and a decoding algorithm, is a set of variable nodes which form a fixed point of the decoder when in error.
The concept of trapping sets can be used with any decoding algorithm. Under maximum likelihood (ML) decoding, since only legal codewords are the fixed points in the ML decoder, the trapping sets are precisely all the legal codewords. For iterative decoding algorithm (e.g., sum product algorithm and its variants) for LDPC codes under AWGN channel or PR channel, the trapping sets can be more complicated. During the iterative decoding process, we define the hard decision at each variable node after each iteration as its status. For a particular codeword C sent, the status after lth iteration (Cˆ l) is determined by the signs of LLRs. For the variable nodes in the same trapping set, when all are in error, the iteration process will not be able to alter the status on any of these variable nodes. All legal codewords are still trapping sets under the iterative decoding since all check equations will be satisfied if a legal codeword is reached even in error. But the trapping sets are not necessarily codewords. Certain trapping sets will lead to violations on several parity check equations, although further decoder iterations would not change the status on any of the variable nodes within this trapping sets. Under channels with AWGN, during the “waterfall” region (SNR is less than the starting point of error floor), a decoding failure can be caused by a variety of factors, such as converging to a wrong codeword, decoding oscillation, as well as the trapping sets. But as the error floor starts, the decoding failures are almost exclusively caused by the trapping sets [5]. For Margulis’s regular (3,6) LDPC code, 20 classes of trapping sets were identified and these dominant error patterns account for more than 99% of error events that occur. Most of these trapping sets involve fewer than 15 variable nodes out of a length-2640 codeword [5].

In general, the structure can be identified as a subset of the variable nodes having very small number of odd-degree neighbors (connecting check nodes). When every bit in this subset is in error, all the even-degree neighbors will be propagating positive information on the status of this subset in the future iterations, since the parity check equation is obviously satisfied. Clearly, this part of the belief propagation is misleading since it confirms the wrong status. Although the odd-degree neighbors will give negative feedback, the likelihood ratio is typically not strong enough to overturn the decision. Figure 3 shows a very simple example of this structure. In this case, once the trapping set [V0 V1 V2 V3] is decoded as [1 1 1 1], check nodes C0, C1, C2, C4,and C5 will be confirming the wrong status on this trapping set, while the negative feedbacks from C3 and C6 are not strong enough to overturn the results.
Although both definitions can lead to the finding of key trapping sets [5][10], it is not known if both definitions lead to the same group of trapping sets for a particular LDPC code. In [11], trapping sets are identified based on several typical structures in the bipartite graph and the estimation results of error floor are close to the simulation results. In [5], the trapping sets are identified based on the fixed point errors found in simulation. Those results are also very close to direct simulation results. In this effort, we define trapping sets based on fixed points in the iterative decoding process, and a direct FPGA simulation is used find trapping sets based on the fixed point errors.
C. Finding the trapping sets
For the ideal PR channel with additive noise, the trapping sets can be found through direct simulation. Through the iterative decoding process, we define the error symptom (T ) as the subset of variables whose status are erroneous after a large number (e.g., 60) of iterations.

When the simulation process reaches a fixed point, the error symptom contains exactly the variable nodes that comprise a trapping set. Therefore, the symptoms of error event during the error floor region (high SNR) can directly lead us to trapping sets. Based on this fact, we use the FPGA hardware to simulate the decoding process at high SNR level. The symptom of an error event is recorded as the variable nodes in a trapping set. In the FPGA simulation process, additional 10 iterations are performed to confirm that the decoder would not leave the identified trapping set (error event symptom persists).
This search process will lead us to a list of trapping sets contained in the LDPC code under the PR channel. The completeness of the list relies on the confidence interval of the simulation process. Given the fact that the FPGA simulation can identify large number of decoding failure events caused by trapping sets even at high SNR level, we can assume that the list of trapping sets is near complete.

Since the decoding failure at high SNR is a rare event, we can assume that no decoding failure is caused by two or more trapping sets simultaneously. Thus, the overall decoding failure rate is estimated by the sum of error rates induced by individual trapping sets.

Including all trapping sets found, we calculated the final FER estimate for this QC-LDPC code. The results are shown in Figure 7. The final FER estimation results, as well as the intermediate results during the iteration process of the LDPC decoder were compared to the direct simulation result. During the early stages of the iteration process, the errors are mostly due to insufficient message passing. After only 5 iterations, the FER estimate using trapping sets is about 2 dB away from the simulation result. As the iteration process continues, the FER estimate is closer to the simulation result as a larger percentage of the decoding failure is caused by trapping sets in the error floor region. After 60 iterations, the estimation result in error floor region is within 0.3 dB of the simulation result. In addition, the FER estimation can be extended to 10−14 with reasonable amount of simulation time (given channel throughput of 100Mbps, the total simulation time is less than 4 hours).
