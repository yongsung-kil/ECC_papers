Programmable LDPC Decoder for Wireless Network

Hye-Jin Ryu and Jong Yeol Lee Division of Electronics and Information Engineering
Chonbuk National University, Jeonju, Korea {yoo7jin, jong}@chonbuk.ac.kr

Abstract
Low density parity check (LDPC) code is adopted to high quality satellite broadcast standard in Europe and having been given attention to channel coding of mobile communication due to its high performance. Partially parallel decoding architecture uses less size of the chip. In this paper, we combined the partially parallel architecture with Benes network and implemented LDPC decoder which reduces size of interconnection and supports variable parity-check matrices. The proposed architecture supports multiple code rates and moderate codeword lengths
1. Introduction
LDPC code and related repeat decoding algorithm was introduced by Gallerger [1] in 1962, but it couldn’t be customized because of low error-rate, low computer technology and it was not easy to test performance of a code that has adaptive block-length. But, it has been brought into relief by many engineers, Mackay, Neal [2] and Spielman [3] since 1995. According to their studies, an LDPC code using sum-product algorithm is known that it has the performance that is close to Shannon’s limit.
Some standards such as IEEE 802.11 [4], IEEE 802.16 [5], DVB-S2 [6] adopted the LDPC codes as channel codes since LDPC codes can achieve low error rate demanded by standards and their error floor occurs at a much lower BER. LDPC codes also have the strong point that their decoding is based on sumproduct algorithm whose implementation can exploit parallel processing. In order to comply with the standards which allow multiple code rates an LDPC decoder should support various code rates.
In this paper, we propose a partially parallel LDPC decoder architecture with broadcasting that can reduce the area of interconnection using Benes network and support multiple code rates and variable check matrix.

The rest of this paper is organized as follows. Section II briefly introduces the LDPC code and the sum-product decoding algorithm. Section III proposes a LDPC decoding architecture to the partially parallel architecture and Benes network algorithm. Section IV shows the ASIC implementation results and performance comparison with related work and Section V concludes the paper.

3. Proposed LDPC Decoding Architecture

3.1. Benes Network [7]

We need a switch block that can change the interconnection between variable nodes and check nodes by control signals to implement a LDPC decoder supporting variable parity-check matrices. If the switch block is implemented by a general crossbar switch in Figure 4(a), the complexity of the circuit would be increased as N² where N is the number of inputs to the switch box. But if you use the Benes network switch depicted in Figure 4-(b). The complexity is
N / 2 × (2 log2 N −1) which means that it can
decrease the area of the circuit. By using switch block takes more delay time than
using fixed interconnection to compensate it, we have to use Broadcasting method. Furthermore, we have to reduce the number of level of the Benes network by reducing the number of input messages.

3.2. Broadcasting [8]
When the length of interconnection wires becomes longer, it increases the delay time, and then, affect the transmission speed of data. Check nodes and variable nodes are connected by fixed interconnection, the area of interconnect wires are substantially increased by (2nγ ). To solve this problem, we have to organize decoder such as Figure 6-(a) by substituting the check node and variable node in Figure 6-(b), (c). If we implement the circuit in this way, the area of wire is decreased to 2n.
3.3. Partially Parallel Architecture [9]
We exploit partially parallel architecture to reduce the area of the LDPC decoder. By using the demultiplexer and multiplexer at the front and back of the switch block to time-share the variable and check nodes, we can reduce the number of variable nodes and check nodes.
Due to the reduced number of the variable and check nodes, the number of the levels in the Benes network, total chip area, and the delay of wire are decreased. It can also support various code rates and parity-check matrices by using input message select signals. The following code rates are supported.

5. Summary and Conclusion
In this paper we propose a LDPC architecture that supports multiple code rates and parity-check matrices. The proposed LDPC decoder is based on a partially parallel architecture with broadcasting and exploits Benes network to support various code rates and parity-check matrices.
