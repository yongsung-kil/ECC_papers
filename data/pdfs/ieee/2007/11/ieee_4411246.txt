Abstract—A novel decoder architecture is presented for quasicyclic low-density parity-check (QC-LDPC) codes. The architecture implements a min-sum algorithm and uses feedback shift-registers to store all the messages exchanged within the algorithm. This shift-register-based min-sum (SRMS) decoder does not require complex interconnections between processors and registers and is amenable to a simple, high throughput hardware implementation. A codec with the SRMS decoder for high-rate codes of length 4095 bits has been fabricated with a 0.15 µm, 1.5 V CMOS technology. The decoder has a gate count of 693 K gates and achieves a throughput of 155.52 Mbps (STM-1) using high-order QAM schemes. Measurement results show that a microwave radio system employing this LDPC codec exhibits good error performance.
I. INTRODUCTION
Recently, much research effort has been devoted to hardware implementations of iterative decoding algorithms for LDPC codes (cf. [1], [6]–[8]). Implementations for high-throughput iterative decoder will require a highly parallel architecture in which a number of bit-to-check (or check-to-bit) messages are computed concurrently. Such parallel architectures, however, often result in high implementation costs due to the need for a large number of memories or due to complex interconnections between processors and memories. Further, as coding-rates increase, check-node processing will become complex since the parity-check matrices will become dense and their row weights will increase. These are crucial issues for implementing high throughput, practical decoders for high-rate LDPC codes.
In this paper, we present a min-sum decoder architecture for high-rate QC-LDPC codes. The decoder employs two sets of feedback shift-registers (FSRs), one for holding bitto-check messages, and the other for check-to-bit messages. The messages are simplified so that the register requirements can be reduced to an amount proportional to the number of nodes in a Tanner graph and so that the registers can be interconnected via a simple network. This shift-registerbased min-sum (SRMS) decoder architecture is suitable for implementation with standard CMOS gate array technology.
We have fabricated an SRMS decoder for a (4095, 3687) QC-LDPC code [4] with a 0.15 µm, 1.5 V CMOS technology. The parity-check matrix of the code is a semi-regular 504 × 4095 matrix with column weights 7 and 8. The row weight of the matrix is 64, which is much greater than

those of the matrices discussed in other papers (cf. [1], [6]– [8]). While it incurs an increase in decoding complexity, the code selected exhibits a good error-floor performance. The decoder has a gate count of 693 K gates and achieves a throughput of 155.52 Mbps using 64-QAM. The measured power consumption is 730 mW when the decoder runs at a clock frequency of 60.2 MHz. We show measured BER performance for a microwave radio system in which our LDPC codec has been used. No significant error-floor has been observed up to a BER of 10−11. The decoder can be used to decode codes with parameters (4095, 3687 + 42i), where 0 i 4. With the SRMS decoder, these codes will be well-suited for use in an error-correction scheme in SDH microwave radio systems transmitting STM-1 data streams, where a high rate code will be employed in conjunction with a high-order modulation, such as 64 or 128-QAM, in order to achieve an extremely low BER and a high throughput, while still satisfying transmit power and bandwidth constraints.

An SRMS decoder simply consists of n BNPUs and r CNPUs. As noted above, these processing units have two operation modes: update mode and output mode. In the update mode, the contents of the FSRs of the processing units are updated with inputs, and in the output mode, the contents of the FSRs are shifted out in order one by one and are, simultaneously, initialized. When BNPUs operate in the output mode, CNPUs operate in the update mode. The contents of the BNPU FSRs that are shifted out are directly input to the CNPUs, and the contents of the CNPU FSRs are updated with them. Similarly, when CNPUs operate in the output mode, BNPUs operate in the update mode. Since each mode takes m units of time to complete its processing, 2m units of time are required for one decoding iteration.
V. HARDWARE IMPLEMENTATION
Applying the above decoder architecture, we have fabricated an LDPC codec LSI using 0.15µm, 1.5 V CMOS technology. The codec employs the (4095, 3687) and (4095, 3855) codes in Section II. The decoder circuit (and also the encoder circuit) for the (4095, 3855) code can be completely shared with that for the (4095, 3687) code. The decoder contains sixty-five BNPUs and eight CNPUs. The bit-width of each LLR from the channel has been set to 5 bits. Each message from BNPUs to CNPUs has been set to 6 bits, and thus each BNPU contains sixty-three 7-bit registers. Here, the normalization constant (cf. [2]) has been set to 2. Each CNPU contains 63 registers, each of which stores values Lt(s), M(ts), v(ts), qt(s). Since Lt(s) and M(ts) are absolute values of 6-bit messages from BNPUs, each of these can be represented by 5 bits. Recall that vt(s) is a pair of integers (u, w) which indicates some bit-node and is used in BNPUs for selecting either Lt(s) or M(ts). In order to reduce the hardware complexity, we have used only the least significant bit of the second component of the integer pair v(ts) rather than all bits required to represent it. The decoder has been designed to run at a clock frequency of 60.2 MHz in order to achieve 155.52 Mbps (STM-1), but if necessary, the same architecture would allow for the design of a decoder capable of running at a clock frequency high enough to achieve 311.04 Mbps (STM-2). The measured power consumption is 730 mW when the decoder runs at a clock frequency of 60.2 MHz, when the post-decoder BER is 10−3, and when the maximum number of iterations has been set to 10. The combination of the (4095, 3687) code and the SRMS decoder on average requires only a small number of iterations to converge. In fact, the performance gap between 10 and 100 iterations is less than 0.2 dB.
The codec would be well-suited for SDH radio systems transmitting STM-1 data streams, for which a high codingrate FEC would be employed in conjunction with a highorder modulation (64 or 128-QAM, for instance). Let us consider an SDH radio system employing the (4095, 3687) codec with Gray-labeled 64-QAM. In order to achieve a throughput of 155.52 Mbps, we have set the symbol frequency fs to 30.1 MHz. The maximum number of decoding iterations Imax is limited by the clock frequency applied to the decoder. When the decoder runs at a clock frequency of double the symbol frequency (2 · fs), the decoding process should be completed within 1365 (= 2 · 4095/6) clocks in order to avoid a significant decoding delay. Thus, in this case, Imax should not be set to greater than 10 since at least 126 clocks are required for one iteration. Next, let us consider the case in which the (4095, 3855) codec is employed together with quasiGray labeled 128-QAM. In order to achieve a throughput of 155.52 Mbps, we have set the symbol frequency fs to 24.4 MHz and the clock frequency of the decoder to 2 · fs. In this case, we set Imax to 8 in order to avoid a significant delay.
Fig. 3 shows measured BER performance for an SDH radio system in which our LDPC codec has been used in the above manner. Along with the measured results, it also indicates BER performance as evaluated through computer simulations in which the demodulator structure has been taken into account. The measured BERs closely match the results from the simulations up to a BER around 10−8.
VI. CONCLUSIONS
In this paper, we have presented encoder and decoder architectures for a class of QC-LDPC codes. The architecture enable us to design a high-throughput min-sum decoder with a relatively small gate count, and it can be applied to the design of a decoder for a wide class of LDPC codes including array codes [3].
