CONSTRUCTION OF REGULAR AND IRREGULAR LDPC CODES
USING (n, {kl, k2}, 1) BLOCK DESIGN

Siddarama R. Patil, Sant S. Pathak

Department of Electrical and Electronics Communication Engineering, IIT Kharagpur, India

ABSTRACT

In this paper, we construct regular and irregular LDPC codes using (n,{k1,k2),1)block design. The problem of
designing regular and irregular structured LDPC codes that have good overall error performance for wide range of code rates and block sizes with attractive storage requirements is attempted. The codes constructed by this method have the girth at least 6 and they perform well with the sum-product iterative decoding algorithm. Further we analyze the constructed codes based on number of short cycles of the corresponding Tanner graph. The analysis shows that codes having small number of short cycles and a cycle structure which is not overly regular in their Tanner graph perform better. The presented codes are well structured and unlike random codes can lend themselves to a very low-complexity encoder and decoder implementation.

Index Terms- Block Design, LDPC, sum-product algorithm, Tanner graph, girth.

1. INTRODUCTION

Low Density Parity Check (LDPC) codes were discovered

by Gallager [1] in 1962 and have recently been

rediscovered [2] [3]. An LDPC code is specified in terms

of a sparse N xM parity check matrixH . An (N, k, j)

regular LDPC code is a code of length N having H with

j l's in each row and k l's in each column. A Tanner

graph displays the relationship between codeword bits and

parity-checks and used to describe LDPC codes. Each of

the N code bits and M parity-checks in H are

represented by a node in the graph. A graph edge joins a

code bit node to the nodes of the parity checks that include

it. The degree of a variable node is the number of check

equations that it participates in. Similarly, degree of a

check node is the number of variable nodes which take part

in that particular check. If all variable (check) nodes have

the same degree, then it is a regular LDPC code. If the

degrees are different, then it is irregular. The irregularity is typically specified using two polynomials called bit node

and check node degree profiles, respectively. That is

A cycle in the Tanner graph is a sequence of connected code bits and check sums which start and end at the same node in the graph and contain no other nodes more than once. The length of the cycle is simply the number of edges it contains .
Despite the excellent error-correcting properties of some known pseudo-random LDPC codes, complexity issues tend to dominate system architecture and design considerations. The high complexity of pseudo-random LDPC codes is a direct consequence of the fact that for these codes a large amount of information is necessary to specify position of the non-zero elements in the paritycheck matrix. The complexity drawbacks of pseudorandom LDPC codes can be overcome by using structured LDPC codes. Several classes of structured LDPC codes are known, including those based on combinatorial designs [5] [6], finite geometries [7] and mutually orthogonal Latin squares [8]. All these constructions gives regular LDPC codes and they are not using the good properties of irregular codes.
The LDPC codes have created lot of interest in graphical models (Tanner graphs) of codes. It is widely accepted that good LDPC code designs have graphical models without too many short cycles[9]. Large girth, however, is not enough to ensure a good graphical model. As shown in [10], the performance of two LDPC codes with the same girth but a different number of short cycles can be significantly different. The regularity of the cycle structure of a graph (i.e. how random the graph appears) also affects graphical code model quality. For example, the introduction of irregularity to LDPC designs [4] is known to improve performance. Due to the dependence of the performance of a code on the properties of the associated graphical model, characterizing the cycle structure of a graphical model is of great help in finding out good codes.
Very less consideration has been given to the design of structured irregular codes. In our work we tried to combine good properties of both irregular LDPC codes and structured LDPC codes. The aim of this paper is to design regular and irregular structured LDPC codes based on
(n, {kl, k2}, 1) block design which are generalizations of
balanced incomplete block designs (BIBD's) discussed by G. C. Yang in [11]. Further we characterize the cycle structure of the proposed codes graphical model and

2. BALANCED INCOMPLETE BLOCK DESIGN

In the following, we briefly explain the concept of BIBD's, which were introduced by Bose using the method of symmetrically repeated differences (SRD) [ 12]. To describe LDPC codes constructed using BIBD's called as BIBD-LDPC codes based on SRD, we follow Ammar's
work [13]. Let X={xl,x2...xn} be a set of n -objects. A
BIBD of X, is a collection of b subsets of X denoted
B1, B2,..., Bb, called blocks, such that the following
conditions are satisfied: 1) each object appears in exactly r of the blocks; 2) every two objects appear together in maximum 2 of the b blocks; and 3) the number of objects in each block, defined as k, is small compared to the total number n of the objects in X. The notation BJI3D (n, k, A) specify a BIBD on n points, with block size
k and index 2. The property 2 for A =1 ensures there are no cycles of length 4 in the Tanner graph.
A BIBD can be completely represented by its nx b incidence matrix, Q, with entries in GF(2). The rows of
Q corresponds to the n objects in X; the columns of Q
corresponds to the b blocks of the design. The entry in the
ith row and the jth column of Q is '1' if the ith object is
contained in the jth block. Otherwise the entry is '0'. It
follows from the structural properties of BIBD that the incidence matrix, Q, has constant row weight, r, and
constant column weight, k, and the maximum number of common 'I's between any tow columns is one.

Definition: A circulant matrix is defined as a square

matrix where each row is a right-cyclic shift of the row

above it and the first row is the right-cyclic shift of the last

row; also each column is a downward cyclic shift of the

column on its left, where the first column is the downward

cyclic shift of the last column.

6. CONCLUSION
In this paper the design of regular and irregular LDPC codes using (n,{4,3},1)block design has been discussed. The proposed codes are analyzed by characterizing the cycle structure of the corresponding bipartite graphs. It is found that the code which is having small number of short cycles and a cycle structure which is not overly regular in their Tanner graphs perform better. The work motivates to find the best codes from a family of codes with same girth in their Tanner graph.
