Abstract— In this paper, a novel extrinsic information transfer (EXIT) analysis is presented for protograph-based and multiedge type low-density parity-check (LDPC) codes. A protograph defines a subset of an LDPCC ensemble (identified by the degree distributions of the bipartite graph), introducing further constraints about the edge connections. For many codes belonging to this class, the conventional approach based on EXIT charts cannot be applied. The proposed EXIT analysis takes into account edge connections, permitting the decoding convergence evaluation for protograph-based LDPC codes, allowing the design of highly-structured capacity approaching LDPC codes.
I. INTRODUCTION
Among the density evolution (DE) techniques introduced for the decoding convergence analysis of iterative decoders, extrinsic information transfer (EXIT) charts [1] gained large attention thanks to their simplicity and their accuracy in performance prediction. EXIT charts have been used in [2] to design low-density parity-check (LDPC) codes [3] for modulation and detection. Nevertheless, the EXIT charts generally cannot be applied to the study of multi-edge type (MET) [4] and protograph-based [5] LDPC codes.
In this paper, we present an EXIT analysis for protographbased LDPC codes (PEXIT in the following). The analysis is focused on the additive white gaussian noise (AWGN) channel and on the binary erasure channel (BEC). Moreover, the presented results can be easily extended to various kinds of channels. The proposed approach permits a simple decoding convergence evaluation for many LDPC code classes based on protographs, providing interesting insights on the behavior of different variable node types in a protograph-based bipartite graph [6]. The algorithm presented in this paper provides a simple tool to analyze many MET ensembles amenable of protograph representation with a remarkable reduction in computational complexity relative to the DE analysis proposed in [4]. The presented algorithm can be therefore easily combined to differential evolution [7] tools (with cost function to be optimized given by the protograph ensemble threshold) to design highly-structured capacity approaching LDPC codes.

A. EXIT charts issues for protograph-based LDPC codes

A protograph [5] [6] is a Tanner graph with a relatively small number of nodes. A protograph G = (V, C, E) consists of a set of N variable nodes V, a set of M check nodes C, and a set of edges E. Each edge ei,j ∈ E connects a variable node Vj ∈ V to a check node Ci ∈ C. Multiple parallel edges are permitted. A larger graph can be obtained by a copyand-permute procedure: the protograph is copied q times, and then the edges of the individual replicas are permuted among the q replicas. The derived graph will consist of n = N q variable nodes and m = M q check nodes. A protograph can be described by a base matrix B of size M ×N . The element bi,j of B represents the number of edges connecting the variable node Vj to the check node Ci.
Protographs impose a structure on the derived graph, which facilitates the design of fast decoders and efficient encoders, as well as a refined control on the derived graph edge connections. Consider an LDPCC ensemble Cλ,ρ defined by the degree distribution pair λ(x), ρ(x) [8]. A protograph G with these degree distribution defines the sub-ensemble of Cλ,ρ comprising the LDPC codes with the edge connections imposed by the protograph structure. This refined definition permits the introduction of punctured variable nodes and of degree-1 variable nodes in a bipartite graph. The use of these node classes have been largely neglected in the LDPCC design because their presence in a random bipartite graph usually affects the decoding convergence. Nevertheless a controlled amount of these nodes in an LDPCC bipartite graph can be exploited to design codes with excellent performance in both the error floor and in the waterfall regions [9]. Such a control can be achieved by careful protograph selection.
The usual EXIT analysis described by (1)-(5) cannot be exploited in the threshold evaluation for many codes based on protographs. In the following, we will introduce some examples to discuss three different problems that prevent the usage of conventional EXIT analysis for protograph-based codes.
1) Degree-1 variable nodes: for a degree-1 variable node, the output message carries the same information of the channel observation associated to the node. Therefore, IE(1v) = Ich. The presence of a fraction of degree-1 variable nodes in an LDPCC bipartite graph implies that there is a term in the summation in (5) of the form λ1 · Ich. Since Ich is lower than 1 for any binary-input noisy channel, the average MI of the variable nodes cannot reach the maximum value IEv = 1. Consider

III. PROTOGRAPH-BASED EXIT ANALYSIS (PEXIT)
The EXIT analysis proposed in this paper introduces the tracking of the MI on the edges of the protograph, eliminating the averages in (5). The issue related to degree-1 variable nodes is solved by evaluating the MI between the a posteriori probability estimation produced by each variable node and the corresponding codeword bit. In the following, a MI propagation model for a protograph is introduced. Finally, the overall protograph EXIT analysis will be described.

A. Mutual information propagation model

IEv|s = IEv|s IAv|1, . . . , IAv|s−1, IAv|s+1, . . . , IAv|dv , Ich . (6)

The MI between the a posteriori probability log-likelihood ratio L(j) evaluated by Vj and the associated codeword bit will depend on the MI of all the incoming messages of Vj.
Once the above-mentioned functions have been obtained for a generic channel, and once the channel nuisance parameter has been fixed, the MI evolution along the edges of the graph can be computed iteratively, recalling that the MI on an edge connecting Vj and Ci, at the output of the variable node, is the a priori MI for Ci, i.e., IEv(i, j) = IAc(i, j). Similarly, IEc(i, j) = IAv(i, j). At each iteration a set of IAP P (j), with i = 0 . . . N − 1, is produced. The convergence is declared if each IAP P (j) reaches 1 as the iteration number tends to infinity.
Note that a similar density evolution approach was suggested in [5]. However, we remark that the proposed EXIT analysis provides a much simpler approach to the protograph threshold evaluation respect to an actual DE approach. The saving in complexity is due to the nature of the EXIT analysis, which tracks a single parameter (the MI) rather than whole probability distribution functions (as the actual DE does).

IV. ANALYSIS OF SOME CODE CLASSES
Quasi-cyclic (QC) LDPC codes, regular LDPC codes, repeat-accumulate (RA) and RA-like codes [9], [10] present simple protograph structures. ARA codes possess all the characteristics mentioned in Section II, i.e., large fractions of degree-1 and punctured nodes. Therefore, the algorithm tests will be focused on ARA codes, in both the low and the high rate regimes. The presented algorithm has been tested on ARA code ensembles [9], [11]. Some of these codes possess low code rates, comprised between 1/2 and 1/8. In Table I, the actual AWGN thresholds for AR3A ensembles are listed and compared to the values computed by the proposed protograph EXIT approach. The accuracy of the approximated thresholds on the AWGN channel is evaluated by the gap ( EXIT ) between the actual thresholds and the thresholds computed by the protograph EXIT approach. Finally, the iterative decoding thresholds for some multiedge type LDPC codes have been evaluated. The protograph of a rate-1/2 concatenated tree code (CTC) [12] has been obtained through its MET description of [4]. The EXIT analysis of the CTC protograph provides an AWGN iterative decoding threshold at (Eb/N0)∗ = 0.651 dB, while the actual one is (Eb/N0)∗ = 0.677 dB. A rate-1/2 MET LDPCC presented in [4] has been tested too. The EXIT analysis a rate-1/2 MET LDPCC [4] provides a threshold at (Eb/N0)∗ = 0.319 dB, while the actual one is (Eb/N0)∗ = 0.309 dB. A rate-1/10 multi-edge type LDPC code [4] has been estimated through the proposed approach. In [11], a protograph which approximates the edge distributions of the above-mentioned rate-1/10 MET code is presented. The actual threshold of the protograph is at −1.028 dB, while the EXIT estimated one is at −1.021 dB.
V. CONCLUDING REMARKS
In this paper we presented a novel EXIT analysis for protograph-based LDPC codes. We showed that the conventional EXIT approach used for LDPCC ensembles defined by degree distributions cannot be adopted for protograph-based and MET LDPC codes. The PEXIT analysis takes into account the protograph structure of the code, allowing an accurate and simple prediction of the protograph threshold. The analysis of many code classes demonstrates the accuracy of the approach, which provides precise threshold estimates compared to the ones obtained through the more computationally-expensive approach of [4]. The algorithm exhibits remarkable precisions in both the high and low code rate regimes.
