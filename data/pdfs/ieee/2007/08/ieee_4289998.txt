Parallel Weighted Bit-Flipping Decoding
Xiaofu Wu, Chunming Zhao, and Xiaohu You

Abstract— A parallel weighted bit-flipping (PWBF) decoding algorithm for low-density parity-check (LDPC) codes is proposed. Compared to the best known serial weighted bit-flipping decoding, the PWBF decoding converges significantly faster but with little performance penalty. For decoding of finite-geometry LDPC codes, we demonstrate through examples that the proposed PWBF decoding converges in about 5 iterations with performance very close to that of the standard belief-propagation decoding.
Index Terms— Belief-propagation (BP), bit-flipping (BF) decoding, low-density parity-check (LDPC) codes.
I. INTRODUCTION
I TERATIVE decoding of low-density parity-check (LDPC) codes has received much attention in recent years ( [1]– [5], and references therein). The standard belief propagation (BP) algorithm can achieve excellent performance but often with heavy implementation complexity [1]. Bit-flipping (BF) decoding is simple, but it often incurs considerable performance loss compared to BP decoding [1], [5]. To bridge the performance gap between BF and BP decodings, weighted BF (WBF) decoding and its variants were proposed [1]– [4], among which, the improved modified-WBF (IMWBF) decoding performs extremely well [4].
Like all the other WBF algorithms, the IMWBF algorithm flips only one bit in each iteration. This sequential bit-flipping strategy often results in slow convergence. In practice, fast convergence is very important for decoding algorithms, as it often means a high throughput. In this paper, we propose a parallel weighted bit-flipping (PWBF) decoding algorithm, which flips multiple bits in each iteration and converges rapidly.
Let C be a (N, K) LDPC code of block length N and dimension K, which has a parity-check matrix H = [hm,n] of M rows, and N columns. Let Rc = K/N denote its code rate. For a regular (dv; dc) LDPC code, we means that the parity-check matrix has exactly dc “ones” in each row and dv “ones” in each column. II. PARALLEL WEIGHTED BIT-FLIPPING DECODING
A. Motivation
The standard WBF algorithm [1] and its variants [2]–[4] suffer from slow convergence, mainly due to their serial flipping strategy.
To accelerate the rate of convergence, multiple bits should be allowed for flipping in each iteration. One may borrow the idea from [5] and allow the algorithm to flip a set of bits, say F , for which En(α), n ∈ F is larger than a predetermined threshold. However, simulations show that this direct multiplebit WBF (MBWBF) scheme does not work well. The main shortcoming of this scheme is that it may flip multiple bits subject to a common unsatisfied constraint.
For a given unsatisfied check node, the constraint becomes satisfied if one of the variables subject to it is flipped. However, this is not true if one flips even number of variables subject to a common constraint. In particular, the unsatisfied constraint remains if one flips two variables subject to this common constraint.
We propose a method to partially remedy this main shortcoming. The idea is to avoid flipping even number of variables subject to an unsatisfied constraint. Currently, exploiting the full potential of this idea is still not available, mainly due to implementation complexity. Instead, we propose a simple mechanism to just flip one variable subject to an unsatisfied constraint.
