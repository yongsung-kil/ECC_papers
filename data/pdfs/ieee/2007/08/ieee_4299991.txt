Alternative Approximation of Check Node Algorithm for DVB-S2 LDPC Decoder

O.Eljamaly
Toumaz, Technology Limited Abingdon, United Kingdom e-mail: omar.eljamaly@toumaz.com

Dr P.Sweeney
Centre for Communication Systems Research University of Surrey, Guilford
Surrey GU21 7XH, United Kingdom

Abstract— In this paper, a new approximation of the check node rule algorithm is proposed for updating the check nodes messages in LDPC decoding. The new technique evaluates the check node min-sum magnitude algorithm using the Taylor series and results in reduced computational complexity and hardware implementation. The architecture of a check node processing unit, implementing the new alternative approximation of the check node, is also presented. The proposed method can achieve the same BER performance with a reduction of ~20% hardware complexity in the LDPE decoder. The simulation results obtained shows that the packet error rate performance of 10−7 , which is the DVB-S2 requirement for MPEG packets transmitted, can be achieved with this new method.
Keywords-component; LDPC codes, computational complexity, Min-Sum, check nodes algorithm, alternative approximation.
I. INTRODUCTION
Recently, the digital video broadcasting standard has employed the Low Density Parity Check (LDPC) coding technique to their transmission system as a powerful channel coding scheme that essentially achieves superior channel capacity performance when compared to other Forward Error Correction (FEC) techniques such as Turbo Codes. The DVBS2 channel encoding encompasses three components: an outer systematic Bose-Chaudhuri-Hcquenghem (BCH) code [2], an inner LDPC code, and a block bit interleaver [1]. The BCH block codes are used to remove the possible error floor produced by LDPC undetected errors.
Different decoder structures have been proposed in the literature. Those involve serial, parallel or semi-parallel architecture implementations [3], [4] and [5]. There are different issues to be considered in the hardware implementation such as edge memory requirements and check node processing unit complexity [3]. Apart from decoder implementations of Regular Belief Propagation algorithm, various approximations such min-sum, lambda-min have been proposed in order to reduce the check node computations and edge memory requirements [3]. In [8] the author addressed two new techniques to solve the interconnect problem in the check node processing unit. These techniques reduced the total amount of check-to-variable interconnect wires and also reduced the maximum wire length.

It can be stated that the hardware requirement complexity for the parallel/serial LDPC decoder lies in the implementations of the check node update. From the literature it can be observed that min-sum approximation check node update implementations are still quite complex in terms of area-efficiency.

The work in this paper proposes a new approximation to the check node passing algorithm that reduces the computational complexity and the silicon area. In addition there is no reduction in BER performance for this implementation.

II. DVB-S2 LDPC ENCODER
LDPC codes are linear block codes defined by a sparse binary matrix (parity check matrix) H. A generic word v with size N is a valid codeword if and only if the following condition is met:

H.v = 0

(1)

A column in H is associated to a bit of the codeword and each row corresponds to a parity check. A non-zero element in a row means that the corresponding bit contributes to this parity check. The code can best be described by a Tanner graph, a graphical representation of the associations between variable nodes, and check nodes, with edges connecting them. The number of edges on each node is called the node degree. If the node degree is identical for all variable nodes, the corresponding parity check matrix is called regular, otherwise it’s irregular.

V. SIMULATION RESULTS
In this section, the simulation results of two know solutions [9], [10] is compared with the results obtained in this work. The results obtained are for Bit Error Rate (BER) and Packet Error Rate (PER) error performance. All the results are obtained by a mean of computer simulations using a DVB-S2 communication system modeled in C.
These results illustrate the performance of the DVBS2 LDPC code using different modulation schemes (QPSK, 8PSK, 16 APSK and 32APSK) for a block size of N=64,800. These results are for a binary AWGN channel. However, since the error rate requirements of DVB-S2 are rather stringent (10−7 packet error rate), an outer BCH code with the same block length as an LDPC frame and an error correction capability of up to 12 bits is employed. This reduces any additional errors at the output of the LDPC decoder and improves the overall performance (mainly be reducing the error floor). This technique is usually implemented in a typical DVB-S2 system.
The first simulated case validates the DVB-S2 LDPC code without using BCH. Figure 4 displays the BER performance versus the signal-to-noise ratio Es / No of the simulated LDPC Code without BCH. This simulations is for a block size of N=64,800 and uses the code rate ½ QPSK. In addition, the plot includes previous published data [9] labeled ‘1/2 BPSK reference’. In this case, up to 100 iterations are executed and 500 frames are transmitted, each frame is 64800 bits. The measured error performance is close to 2.10−7 for ½ QPSK for Es / No ≈ 0.8dB . From the figure this result is marginally better than the results published in [9]. The Author in [9] used the hyperbolic tangent function (Tanh) function approximation for their simulation. Both of the results plotted in figure 4 meet the DVB-S2 requirement.

VI. CONCLUSION
In this paper, the new approximation of the check node algorithm is proposed. This approximation reduces the computational complexity and hardware implementation of the LDPC check node processing unit. In addition, no reduction in BER performance is demonstrated. Simulations show that PER error performance of 10-7 can be achieved with the new approximation algorithm implemented for the DVB-S2 system.
