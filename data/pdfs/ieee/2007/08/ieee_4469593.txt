Abstract-This paper presents a flexible decoder architecture
which supports twelve combinations of code lengths-576, 1152,
1728, 2304 bits and code rates-1/2, 2/3, 3/4 for block-serial
irregular LDPC codes based on the IEEE 802.16e standard [1].
Approximate-Min Scheme is used to increase memory efficiency
during message processing. At least 68.4% extrinsic message
memory is saved and this reduction increases with the code rate.
A prototype of the LDPC decoder has been implemented and
tested on an Ateral FPGA.
Keywords-LDPC; Block-Serial; Multi-Rate; Multi-Length; A-Min;
Memory Efficient; Programmable; IEEE802.16e
I. INTRODUCTION
Much attention is being given to Low-Density ParityCheck (LDPC) codes because of their capability to achieve near Shannon limit performance [2]. Block structured irregular LDPC codes even have been proposed as a forward error correction scheme in IEEE802.16e standard [1]. Although, the LDPC decoding algorithm is easy to be parallelized, its need for large hardware resources is still a significant problem especially for mobile devices. With strict area and power constraints, a block-serial implementation that processes each message serially with less hardware requirements is more attractive. However, serial implementation typically increases the decoding latency, thereby limiting the throughput.
To make up the loss of throughput that is caused by serial decoding, a layered shuffled belief propagation algorithm (LSBPA) for block-structured LDPC codes can be employed, leading to a faster convergence speed and less memory requirement [3, 4].
Min-sum algorithm is widely used to reduce the LDPC decoder hardware complexity [5]. However, simulation shows that it suffers from a 0.1dB loss of bit error rate (BER) performance. As a substitute, C. Jones and E. Valle showed in [6] that their Approximate-Min (A-min) algorithm can also reduce hardware resources for message processing without any noticeable degradation in BER performance. Researches in this paper show that at least 68.4% extrinsic message memory can be saved and this reduction increases with the code rate.

Most papers that discuss serial LDPC decoder architecture are based on a fixed code rate and code length [5, 7]. In this paper, we present a highly flexible serial LDPC decoder that supports multiple code lengths (576, 1152 and 1728, 2304 bits) and multiple code rates (1/2, 2/3 and 3/4) based on the IEEE 802.16e standard [1]. Furthermore, LSBPA that converges almost twice as fast as the standard belief propagation is employed. A modified version of A-min algorithm is simulated and used to improve memory efficiency. A prototype of LDPC decoder on FPGA is later introduced in the final section.
II. LOW-DENSITY PARITY-CHECK CODES
In this section, block-structured irregular LDPC codes in IEEE802.16e [1] are introduced. A brief review of LSBPA [4] is then provided to facilitate the architecture discussion. At last, a three-stage pipeline is given to illustrate the serial decoding process.
A. Block LDPC codes in IEEE802. 16e
An LDPC code is a linear error correction code specified by a very sparse parity check matrix (PCM) HM N' where
nonzero entries are randomly placed. For M check bits and a code length of N bits, the code rate isR=(N-M)/N. The
process of belief propagation decoding algorithm is usually represented by Tanner graph in which a variable node corresponds to each PCM column and a check node corresponds to each PCM row. There is an edge between each pair of nodes if corresponding entry equals to 1. When Decoding, Log-likelihood ratios (LLRs) are transferred as messages along those edges.
Recent hardware designs of LDPC decoder are mostly based on block-structured PCMs, in which the z x z sub-blocks are either a shifted identity matrix or a zero matrix. In this paper, LDPC decoder is designed to support irregular block-structured PCMs proposed for the IEEE802.16e wireless standard [1].

IV. PROGRAMMABLE DECODER ARCHECTURE
In this section, a programmable and flexible architecture of LDPC decoder is given which supports all the parameters below:

* Supports block-structured irregular LDPC codes.

* Supports multiple code rates and multiple code lengths without recompilation or re-synthesis. Code Rate R=1/2, 2/3 and 3/4, code length N=576, 1152, 1728 and 2304 bits.

* Supports multiple maximum numbers of iterations. MaxIter=10, 15 and 25.

In Hard Decision block, signs of LLR messages will be sent to X-Ors in check node order. From the beginning of a new iteration, if successive M parity check equations are satisfied, iterations will be ended. Otherwise, it will go on until the maximum number of iterations is reached. Its-end signal from the Hard Decision block tells the controller that iterative decoding of this frame can be ended and it is time to output the valid code.

VI. CONCLUSIONS
In this paper, a novel memory efficient architecture for multi-rate and multi-length decoder that supports LDPC codes in [1] is presented. Layered shuffled belief propagation algorithm [4] is employed to accelerate the convergence speed. Tradeoff between hardware resources and BER performance is carefully taken and A-min algorithm instead of classical Min-sum algorithm is used to increase memory efficiency. Simulation shows that A-min has no noticeable loss of BER performance when using LDPC codes based on [1]. The highly flexible serial decoder architecture in this paper supports all combinations of code rate 1/2, 2/3 and 3/4, code length 576, 1152, 1728, 2304 bits, and maximum number of iterations 10, 15 and 25. An obvious advantage of this

programmable architecture is that rewriting parameters in EPROMs will easily enable the decoder to support code rates and code lengths other than the default ones. A prototype of the LDPC decoder on Ateral FPGA is implemented and the result is given.
