3.2-Gb/s 1024-b Rate-1/2 LDPC Decoder Chip Using a Flooding-Type Update-Schedule Algorithm

Naoya Onizawa, Tomokazu Ikeda and Takahiro Hanyu Research Institute of Electrical Communication, Tohoku University, Sendai, Japan
Email: onizawa, tomokazu, hanyu @ngc.riec.tohoku.ac.jp

Vincent C. Gaudet Department of Electrical and Computer Engineering, University of Alberta, Edmonton, Alberta, Canada
Email: vgaudet@ece.ualberta.ca

Abstract— This paper presents a high-speed low-density paritycheck (LDPC) decoder chip using a new decoding algorithm, called a flooding-type update-schedule algorithm. Since node computations are performed using partially updated messages in the proposed algorithm, because of the good similarity among time-consecutive messages, data-transmission bottleneck between nodes for node computation is greatly reduced. Moreover, longer wires between nodes are appropriately divided into several subwires by inserting flip-flops so that system clock frequency for the LDPC decoding scheme can be much increased while maintaining the same BER as a conventional algorithm using fully updated messages. In fact, a throughput of 3.2Gb/s in a 1024-b LDPC decoder chip under 90nm CMOS technology is attained with the sufficient BER.

I. INTRODUCTION
Due to superior error correction performance, low-density parity-check (LDPC) codes [1] [2] have been proposed for numerous recent data communications standards such as DVBS2, IEEE 802.16, and 10GBASE-T. The LDPC codes are typically decoded using a sum-product decoding algorithm [3] or one of variants which are operated by passing posteriori probability messages along edges between computation nodes, such as variable and check nodes shown in Fig. 1. The messages which are calculated in each node are sent to the other nodes, and then are updated using the other messages. This procedure is repeated for a fixed number of iterations, or until a satisfactory solution is obtained. Decoding is performed iteratively. Thus, the decoding throughput is defined as

Since bitparallel decoder architectures directly map the computations onto processing elements and the edges onto interconnect network, a large code can be simultaneously decoded, while a frequency becomes very low because of interconnect complexity. In the proposed scheduling algorithm, messages of the longer nets are transmitted between the computation nodes by a few steps. Since node computations are updated by using the most recent arrived messages, the computation delay is not limited by the longer nets. Also, since time consecutive messages are similar in the LDPC decoding, the same BER result can be obtained as the conventional algorithm with a little increase of the iteration cycles. Moreover, since the wire lengths are estimated by doing a place-and-route previously, the longer nets can be appropriately divided in the chip implementation, which enables to reduce the worst-case delay. As a result, the proposed LDPC decoder achieves a throughput of 3.2Gb/s under a 90nm CMOS technology.

Fig.3 shows a conventional and the proposed scheduling algorithm. In the conventional algorithm, the node computations are performed by using all of updated messages including those of the longer nets, such as B(k) of Fig.3 (a). Since the data-transfer delay is dominant in the bit-parallel LDPC decoder, the worst-case delay is restricted by the wiring delay of the longer nets, which results in difficulty of increasing a clock frequency [5]. By inserting delay in the nets, the messages are transmitted between computation nodes by a few cycles, which enables to reduce the worst-case wiring delay, while a large number of iteration cycles is required.
On the other hand, in the proposed scheduling algorithm, the messages of only the longer nets are transmitted between computation nodes by a few cycles. After portion of the arrived messages are updated, node computations are performed by using the most recent arrived messages. Thus, the computation delay is not restricted by the worst-case wiring delay.
Also, since time-consecutive messages, such as B(k-1) and B(k) of Fig. 3(b), are similar in the LDPC decoding, the same BER result can be obtained as the conventional algorithm with a little increase of the iteration cycles. This partially data-update schedule is a flooding-type update schedule. Fig. 4 shows the simulated BER results with min-sum decoding algorithm [9] using BPSK modulation on an AWGN channel. The codes are length-1024 regular (3,6) rate-1/2 LDPC codes with 4-bit log-likelihood-ratio messages with one sign bit and three magnitude bits. The proposed scheme achieves the same coding gain with the twice number of decoding cycles compared with the conventional scheme (w/o delay), while the four-times number of cycles is required in the conventional scheme (w/ delay) .

V. CONCLUSION
A new 90nm CMOS 1024-b, rate-1/2 LDPC decoder using a flooding-type update-schedule algorithm has been simulated that achieves a coded throughput of 3.2 Gb/s with the same area as the conventional LDPC decoder. By inserting the flip-flops into the longer nets, the messages are transmitted between computation nodes by a few cycles, which enables to reduce the worst delay. Also, because of the similarity of time consecutive messages in the LDPC algorithm, using a flooding-type update-schedule algorithm, where node computations are performed by using partially updated messages, the same BER result can be obtained as the conventional algorithm with a little increase of the iteration cycles.
As a future prospect, it is important to measure power, BER and in/out signals using a test chip which has been fabricated, and to investigate performance of more bit LDPC decoder.
