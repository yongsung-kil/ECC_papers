Performance of Optical CDMA Systems with Low Density Parity Check Codes
S. Sahuguede, A. Julien-Vergonjanne, J.P. Cances XLIM DPT-C²S² UMR CNRS 6172 -ENSIL, 16 rue d’Atlantis, 87068 LIMOGES, FRANCE
s_sahuguede@ensil.unilim.fr, anne@ensil.unilim.fr

Abstract
Incoherent Optical Code Division Multiple Access (OCDMA) is a potential issue for future high speed optical networks. To reduce the number of errors at the detection due in particular to Multiple Access Interference (MAI), we add to the OCDMA link a block of efficient error correction codes named Low Density Parity Check codes (LDPC). The LDPC decoding algorithm, Belief Propagation (BP), usually employed to correct errors due to Gaussian noise, is reconsidered in order to take into account the MAI binomial distribution. The performance of an OCDMA link using 1 Dimensional and 2 Dimensional code sequences along with several LDPC codes is investigated. The performance is enhanced not only in an ideal case with only MAI, but also in a noisy case with MAI and thermal noise. With such error correction coding scheme, the constraints over OCDMA parameters and the signal to noise ratio are greatly relaxed.
1. Introduction
The Code Division Multiple Access (CDMA) method has recently appear as a future access technique for optical networks [1-2]. Actually, it has been shown that Optical CDMA (OCDMA) could efficiently share the optical bandwidth, by using flexible user allocation. Besides, this access technique provides varying data rates to several users.
However, there are some drawbacks with OCDMA, limiting the performance [3]. In incoherent OCDMA systems, signal processing is based on optical power and thus spreading code sequences are unipolar. This unipolarity enforces the amount of Multiple Access Interference (MAI) which results as a significant limitation in such system.
To enhance the system performance as regards to the MAI and the spreading code length, many studies have been developed around coding methods. The first

investigated codes were the 1 Dimensional (1-D) time domain spreading codes called Optical Orthogonal Codes (OOC) [1]. In order to reduce the MAI and thus to support many simultaneous users for a given quality of service (i.e. a targeted Bit Error Rate (BER)), the spreading code length has to be high, and the available data rate is reduced. Nowadays, the most promising code sequences are 2 Dimensional (2-D) codes constructed both in time and frequency domain [3-4]. Actually, thanks to this additional degree of freedom, the temporal length can be reduced faced to 1-D code length in order to reach the same performance.
Another way to mitigate MAI is to improve receiver schemes [1], [5-8]. The simplest techniques are based on the use of hard limiter devices at the reception end [1], [5] which permits removing some interference patterns. The most efficient techniques to significantly reduce interference impact consist in using multi-user receivers [6-8]. However, their implementation requires a high complexity, increasing with the number of users and thus is more complicated for practical networks.
To avoid complexity, another solution to improve performance lies in the use of Forward Error Correction (FEC) schemes. Channel codes such as turbo codes, Reed Solomon (RS), and convolutional codes have been investigated for optical CDMA systems [9-14]. However, due to the coding rate, the use of FEC requires extra bandwidth expansion to the OCDMA system and thus reduces spectral efficiency. To achieve good performance with low redundant codes, we investigate Low Density Parity Check (LDPC) [15] codes. It has recently been shown that such codes are a viable choice for high-speed optical transmission [16]. To our knowledge, there is no published work on the LDPC decoding for optical CDMA channel.
We consider in this paper LDPC constructed by a deterministic method, called Balanced Incomplete Block Design (BIBD) [17]. This block code

The classical decoding algorithm for LDPC, Belief Propagation (BP) [18], is used to correct errors due to Gaussian noise. The first contribution of our paper is to adapt its process in order to correct errors due to MAI, which repartition is binomial [1].
By optimizing the decoding algorithm, we apply the LDPC to an OCDMA transmission using 1-D OOC and 2-D spreading codes named Multi Wavelength Optical Orthogonal Codes (MWOOC) [4]. We show that the BER is significantly improved not only in a noiseless case with only MAI, but also in presence of thermal or Gaussian noise. By studying the tradeoff between the LDPC redundancy and the error correction performance, our second contribution is to show that the LDPC along with an OCDMA link constitute a suitable solution to achieve good performance. Moreover, we highlight that employing 2-D spreading codes with LDPC can efficiently be used for high speed optical network supporting a high number of users, while maintaining a good quality of service.

In section III, the transmission link with LDPC codes is described and we develop the principles applied to optimize the decoding process for OCDMA. Section IV presents the analysis of error correction coding performance, for 1-D and 2-D code sequences, first in a noiseless case, then in Gaussian noisy case before concluding in section V.
2. OCDMA system
We consider an incoherent Optical CDMA system where each user emits equiprobable binary data (0,1).
For each active user, the OCDMA encoder performs the multiplication of the emitted data by the user code (Fig.1). Then, data are summed before being transmitted upon the optical channel. After an ideal opto-electrical conversion, we consider a Conventional Correlation Receiver (CCR) [1]. A decision over the value of the emitted data is then taken.
The Optical Orthogonal Codes (OOC) [1] are the first studied 1-Dimensional (1-D) temporal code sequences. The code sequences are unipolar and defined by (F ,W , λa , λc ) . F is the temporal sequence length i.e. the number of temporal subdivisions of the bit, called chips. W is the weight i.e. the number of chips set to ‘1’.The auto and cross-correlation constraints λa and λc are set to ‘1’ in order to limit the interference amount. With these codes, and assuming an equiprobable transmission, the probability

x is the datum emitted by the desired user and equals

When considering only MAI and for a threshold

rises to an error, whereas with x=1, no error can

appear.

Another important limitation in OCDMA systems

is the thermal noise due to the components, in

particular to the photo detector which performs the

opto-electrical conversion. The decision variable is

then expressed by:

Z =W⋅x+I +n.

where n is an Additive White Gaussian Noise

As thermal noise is a Gaussian one, n takes positive

or negative values. Therefore, chip amplitude can be

reduced and the decision variable Z lowered. In this

case, errors can happen not only on ‘0’ emitted data as

in the noiseless case, but also on ‘1’ data, affected by a

negative noise contribution.

3. Application of LDPC on OCDMA Link

In order to reduce errors due to MAI and thermal noise, we investigate a transmission with block coding error correction. The error correction code is defined
by its parity check matrix H (M×N ) and its generator
matrix G(K×N) satisfying the condition: G ⋅ H T = 0 . (5)
At the emission, for each active user, the coding block is placed before the (1-D) or (2-D) OCDMA encoder (Fig. 1). Each vector x=[x0 x1 … xK-1] of K bits is multiplied by G(K×N) to constitute a coded block
c=[c0 c1 … cN-1] of length N>K. The available data rate is multiplied by the code rate R=K/N<1, and thus is decreased by the addition of (N-K) redundant bits.
The block received by the LDPC decoder, Z=[Z0 Z1 …ZN-1] can be written as the sum of the emitted block c plus an eventual term of error e. At the decoder, it is then multiplied by HT in order to detect error presence: Z ⋅ H T = (c + e) ⋅ H T = x ⋅ G ⋅ H T + e ⋅ H T = e ⋅ H T (6)
If there is no error, e=0, the result of (6) is null. Otherwise, there is at least one error in the received word, which has to be localized and corrected.
For LDPC code, H is sparsely constructed in order to simplify the decoding process. In this paper, we consider that the code construction is based on a deterministic method called Balanced Incomplete Block Design (BIBD) [17]. The advantages of this construction are not only that the parity matrix H is sparse, but also that H is constructed from vector permutations. For example, the LDPC defined by (N=2715, K=2535), has a parity check matrix H of dimension (181x2715). H contains 2715 columns which are in fact 15 vectors of lengths 181, each one permuted 181 times. Therefore, the decoding process is easier to implement. At the reception end, we consider a “soft” detection. Data are not compared to the threshold after the correlation as for the CCR. The LDPC decoder receives the exact “soft” values of the decision variable Z=[Z0 Z1 … ZN-1] (Fig. 1).

4. Performance analysis
We have first simulated an ideal case without considering thermal noise. The BER results reported on Fig. 3 are given as the function of user number, for the OOC (F=534, W=3), a CCR and error correction performed with three different LDPC parameters (N, K): (N=2715, K=2535), (2172, 1992), and (905, 725) of rates respectively equal to R=0.93, R=0.92 and R=0.8. In order to appreciate the performance of error correction, we have also made simulations with the Reed Solomon codes RS (255, 239) having a code rate similar to the LDPC (2715, 2535) one (i.e. R=0.93). Moreover, for this RS code defined on GF(8), the signal after the encoder is a block of length 255x8=2040 bits, which is comparable to the 2715 bits resulting at the output of the LDPC encoder.
First, it can be pointed out that the two error correcting codes of rate R=0.93 do not have the same performance: the LDPC permits reaching lower BER than RS. Moreover, there is a value of BER for which the error correction has no effect: e.g. on Fig. 3, a BER>10-2 (u ≥ 70) , can not be improved. Conversely,
the error correction is more efficient as the BER without correction is low. For example, with LDPC (2715, 2535), the BER is divided by 10 for u=50 users and by 103 for u=35 users.
Moreover, as expected, as the rate is lowered (i.e. as the redundancy is high), as the LDPC is efficient to correct errors due to MAI. Actually, for a given number of users, the performance is all the more enhanced that R is low. For example, for u=50 users, the BER results are reported on table I as a function of R, and confirm that the BER decreases as R decreases.
We have also reported on Fig.3 the theoretical BER without error correction for the three OOCs leading to the same BER for u=50 users as for the OOC (F=534, W=3) with the three LDPC used. This study in a noiseless case shows that error correction coding over an OCDMA transmission relaxes the constraints over the OOC sequence parameters. More particularly LDPC has high correction performance and permits significantly reducing the temporal code length and consequently increasing the available data rate.
We have also investigated the impact of LDPC with 2-D codes namely MWOOC. Because the loss in data rate is not significant compared to the correction efficiency, the results presented here are obtained with the LDPC (905,725) of rate 0.8.
The used MWOOC is defined by (FxL=67x8, W=L=8), which means that there are 8 wavelengths and that the temporal length is 67 with a weight equal to 8. The BERs with error correction coding and without coding are presented on figure 4.
We remark that the BER is enhanced even for a high number of users, and takes very small values for less than 50 users. In this example, if we consider an electrical bandwidth of 10GHz, the available data rate per user is around 120Mbit/s. This shows that a suitable data rate for current applications in optical networks can be provided for a high number of users while maintaining a good quality of service.

First of all, it is observed that, for each configuration (transmission without error correction, with LDPC or with RS), when the SNR increases, the BER tends to the one in a noiseless case. In addition, the SNR level under which the error correction has an effect is lower for LDPC (SNR1=12 dB) than for RS (SNR1’=14dB). This means that the LDPC thermal noise plus MAI robustness is better than RS one. 8. Conclusion
We have studied in this paper the performance of error correction coding using LDPC over an OCDMA transmission. By studying the probability density distribution of the noise variable, we have optimized the BP decoding algorithm.
We have shown that the BER is improved as well in a noiseless case with only MAI as in a noisy scheme. By comparing LDPC performance with RS one, we pointed out the interest of using LDPC which provide better BER while having an easier decoding. The code sequence parameters and the SNR are significantly relaxed thanks to correction coding. Moreover, using 2-D codes and a CCR along with LDPC permits providing a high data rate for a high number of users while reaching very low BER.
Thus, using error correction coding with LDPC over an OCDMA link is a potential issue for reducing the errors due to MAI plus Gaussian noise with low complexity reception. Thus, this is an efficient way for providing high data rates for future high speed optical networks.
