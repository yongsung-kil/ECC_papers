Construction of Irregular LDPC Codes Based on Balanced Incomplete Block Designs
Siddarama R. Patil1, Sant S. Pathak2 Department of Electrical and Electronics Communication Engineering, Indian Institute of Technology, Kharagpur, India
pdapatil@yahoo.com1 , ssp@ece.iitkgp.ernet.in2

Abstract – In this paper, we present a method to design irregular structured Low-Density Parity-Check (LDPC) codes based on Balanced Incomplete Block Designs (BIBD). The problem of designing irregular structured LDPC codes that have good overall error performance for a wide range of code rates and block sizes with attractive storage requirements is attempted. The proposed codes are well structured and unlike random codes can lend themselves to a very low-complexity implementation. The codes constructed by this method have the girth at least 6 and they perform well with the sum-product iterative decoding algorithm. Furthermore, the codes are compared with known random LDPC codes, in order to assess their relative achievable performance. The proposed LDPC codes demonstrate performance that is better than an average random LDPC code of similar block length, code rate and parity check matrix sparsity.

I INTRODUCTION

Low Density Parity Check Codes were discovered by Gallager [1] in 1962 and have recently been rediscovered [2] [3]. It has been shown that these codes achieve a remarkable performance with iterative decoding that is very close to the Shannon limit. Consequently, these codes have become strong competitor to turbo codes for error control in many communication and digital storage systems where high reliability is required.

An LDPC code is specified in terms of a sparse N × M parity check matrix H . An (N, k, j) regular LDPC code is
a code of length N having H with j 1’s in each row and
k 1’s in each column. A Tanner graph displays the relationship between codeword bits and parity-checks and used to describe LDPC codes. Each of the N code bits and M parity-checks in H are represented by a node in the graph. A graph edge joins a code bit node to the nodes of the parity checks that include it. The degree of a variable node is the number of check equations that it participates in. Similarly, degree of a check node is the number of variable nodes which take part in that particular check. If all variable (check) nodes have the same degree, then it is a regular LDPC code. If the degrees are different, then it is irregular. A cycle in the Tanner graph is a sequence of connected code bits and check sums which start and end at the same node in the graph and contain no other nodes more than once. The length of the cycle is simply the number of edges it contains. It is known that the existence of short cycles in the Tanner graph prevents an exact error-probability analysis of iterative decoding procedures. Without 4cycles in the graph of codes, the iterative sum-product decoding algorithm can be performed well. The maximum overlap between any two columns in the matrix H to be one confirms the absence of 4-cycles.
Despite the excellent error-correcting properties of some known pseudo-random LDPC codes, complexity issues tend to dominate system architecture and design considerations. The high complexity of pseudo-random LDPC codes is a direct consequence of the fact that for these codes a large amount of information is necessary to specify position of the non-zero elements in the paritycheck matrix The complexity drawbacks of pseudorandom LDPC codes can be overcome by using structured LDPC codes. Several classes of structured LDPC codes are known, including those based on combinatorial designs [5] [6], finite geometries [7] and mutually orthogonal Latin squares [8]. Less consideration has been given to the design of structured irregular codes. In our work we tried to combine good properties of both irregular LDPC codes and structured LDPC codes. The aim of this paper is to design irregular structured LDPC codes based on BIBD that have good overall error performance for a wide range of code rates and block sizes with attractive storage requirements. Our new codes have the advantage of reduced storage requirements and low implementation complexity.

BIBD of X , is a collection of b subsets of X denoted B1, B2,…, Bb , called blocks, such that the following conditions are satisfied: 1) each object appears in exactly ρ of the blocks; 2) every two objects appear together in maximum λ of the b blocks; and 3) the number of objects in each block, defined as w , is small compared to the total number n of the objects in X . The notation BIBD (n, w, λ) specify a BIBD on n points, with block size w and index λ . The property 2 for λ = 1 ensures there are no cycles of length 4 in the Tanner graph.
A BIBD can be completely represented by its n × b incidence matrix, Q , with entries in GF(2). The rows of Q corresponds to the n objects in X ; the columns of Q
corresponds to the b blocks of the design. The entry in the ith row and the jth column of Q is ‘1’ if the ith object is
contained in the jth block. Otherwise the entry is ‘0’. It follows from the structural properties of BIBD that the incidence matrix, Q , has constant row weight, ρ , and constant column weight, w , and the maximum number of common ‘1’s between any tow columns is one.

V SIMULATION RESULTS
In this section, performance of the proposed codes are presented and compared with performance results of similar random Mackay codes [2] and referred in this paper as random codes. For our simulation results, all codes are decoded iteratively using the sum-product algorithm and the channel is chosen to be additive white Gaussian noise (AWGN). Bit error rate (BER) curves are presented based on maximum 100 iterations of the sum-product algorithm.

We also designed a (218, 3, 6) BIBD-LDPC code and (542, 3, 6) BIBD-LDPC code as per the procedure given in [10] from two circulants of (109, 3,1) BIBD and (271, 3,1) BIBD respectively and compared the performance with the proposed codes. The two computer generated (MacKay) random codes considered for the comparison are (504, 3, 6) LDPC and (204, 3, 6) LDPC .
The performance of 541 block length, ½ rate, irregular LDPC code (code1) is shown in Fig. 1. At an Eb/No of 2.5 dB the BER is 2.7899 ×10−4 . We see that the proposed code (code1) outperforms the (504, 3, 6) random code and (542, 3, 6) BIBD-LDPC code. At the same Eb/No of 2.5 dB the random code BER is 7.1329 ×10−4 and that of (542, 3, 6) BIBD-LDPC code is 4.3390 ×10−4 . Similar performance is observed for the 211 block length, ½ code rate irregular LDPC code (code2 and code3) and their performance is shown in Fig. 2.
Intuitively, the reason for small block length proposed irregular LDPC codes based on BIBD showing better performance than the randomly constructed codes is that for small block length random codes it is difficult to avoid cycles of length 4. But irrespective of block length, the BIBD codes will guarantee that the Tanner graph is free from cycles of length 4 and hence the girth 6.
CONCLUSIONS
In this paper the design of irregular LDPC codes using BIBD has been discussed. The rows of corresponding H matrix are generated by cyclic shift of the BIBD base block. The advantage lies in the fact that we need not store the entire H matrix for decoder action and hence the decoder memory requirement and implementation complexity reduces significantly. The designs provide codes with various lengths and rates with a guarantee of no cycles of length 4 in their Tanner graphs.
