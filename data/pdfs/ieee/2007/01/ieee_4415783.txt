This paper presents a design of VLSI architecture for the FEC (Forward Error Correction) decoding system supporting the Chinese National Terrestrial Digital TV Broadcasting Standard (GB20600-2006). The architecture of the FEC decoder includes a time-deinterleaver; a Symbol-to-Bit LLR calculator, a LDPC decoder, a BCH decoder, and a de-randomizer. The FEC decoder supports 5 modulation modes (64QAM, 32QAM, 16QAM, 4QAM and 4QAM-NR), 3 rates of LDPC codes (0.4, 06, and 0.8) and 3 kinds of time-interleaver lengths. The implementation has been both prototyped in a FPGA board and silicon-proved by a single-chip DTMB demodulator and FEC ASIC chip. For 64qam, rate 0.6 modes, the FEC decoder can achieve the threshold of visibility (TOV) at 14.2 dB.
1. Introduction
In August 2006, China has announced its own National Terrestrial Digital TV Broadcasting Standard (GB206002006), which is also called DTMB. This standard supports both multi-carrier (MC) and single-carrier modulation (SC) scheme; In MC, it uses the advanced signal processing capabilities of TDS-OFDM (Time Domain Synchronized “C” Orthogonal Frequency Division Multiplexing), which combines time and frequency domain digital signal processing in the wireless broadband time-varying channel. It utilizes the latest FEC technology – LDPC codes. By doing so, the system offers many advantages. It enables superior performance for traditional TV broadcasting and mobile digital TV services in a number of application environments, including indoor, fixed, portable, and mobile. It delivers fast system synchronization and precise channel estimation. And, it efficiently uses frequency spectrum and delivers excellent Single Frequency Network (SFN) performance.

3. 1 Time-deinterleaver
For TDS-OFDM system, the time-interleaver is used to increase the resilience to spurious noise. DTMB system does time-interleaving by using convolutional interleaving scheme with interleave width (branches) B = 52, and interleave depth (delay buffers) M = 240 or M = 720. The total delay of the interleave/de-interleave pair can be calculated from M u (B – 1) u B. For our design, we have implemented the time-deinterleaver with 3 different modes. Mode 1: M = 48, B = 52. Mode 2: M=240, B = 52; Mode 3: M=720, B = 52. So the total time delays introduced by Time interleaver/de-interleaver pair are 127296, 636480, 1909440 symbol clock cycles corresponding to the 3 different modes. Mode 1 (B = 52, M = 48) is added for easy simulation and debugging purposes.

3.2 Symbol-to-Bit LLR Calculator
For reliability-based soft-decision decoding algorithms developed recently, like turbo-decoding, LDPC decoding, etc., the inputs to the decoder generally consist of log likelihood ratios (LLRs) determined by the channel statistics. For the most common modulation/demodulation system we use today, there is generally multiple modulation methods exist simultaneously. So an effective FEC decoder should be capable of decoding channel-corrupted signals for multiple kinds of modulation at the same time, but reduce the hardware cost to a minimum.
In DTMB system, in each signal frame, a known PN sequence is inserted as the frame header. We can use it as the channel sounding signal. At the receiver side, by convolving the received PN sounding sequence with the known PN, we can derive the channel response function H(f), which is defined as the Channel State Information (CSI).
In the presented FEC decoder, when generating the LDPC bit log-likelihood-ratio (LLR) inputs from the channel-corrupted symbols, we have worked out an algorithm which incorporates CSI information during the derivation process for different bits and different modulation schemes, like 16QAM, 64QAM; we have also worked out a hardware implementation structure which only involves addition and subtraction calculations, and incorporates a lot of resource-sharing during the computation process. As a result, the complication of the bit LLR computation circuits are tremendously reduced, and the performance of FEC decoder are greatly improved for multi-path channel.
3.3 LDPC Decoder
There are 3 kinds of LDPC codes used in DTMB standard. They are LDPC (7493, 3048), LDPC (7493,4512), and LDPC (7493, 6016), which correspond to the 3 coding rates, which are 0.4, o.6 and 0.8. Their corresponding parity check matrixes H are all of the Quasi-Cyclic structure, with number of circulant columns as 59; number of circulant rows as 35 (for 0.4), 23 (for 0.6) and 11 (for 0.8); the circulant size is 127.

The whole decoding process consists of a number of iterations. Each iteration consists of a row process called check node update, and a column process called variable node update, which are accomplished by CNU unit and VNU unit respectively. The decoding process stops either when it reaches the specified maximum number of iterations or when harddecision of all rows of symbols are error-free.
The LDPC codes are often represented by a bipartite graph made up of two families of nodes, bit nodes and check nodes, which correspond to the column and row nodes to the code’s H-Matrix respectively. To simplify the mathematical computation and reduce hardware cost, a Log-domain message-passing algorithm, called Sum Product Algorithm (SPA), is generally selected for hardware implementation. 3.4 BCH Decoder
The BCH code used in DTMB standard is BCH(762, 752). After LDPC decoder, we feed the hard-decision bits into a BCH decoder, which can detect 2 bits, corrects 1 bit of error . The outputs are combined into bytes and sent to De-randomizer.
3.5 De-randomizer
Since the data in the transmitter has been randomized using a pseudo-random (PN) sequence before BCH encoder, a de-randomizer must be used right after the BCH decoder. It is realized by using a 15-bits register

5. Conclusion
This paper introduces a FEC decoder used in Chinese National Terrestrial Digital TV Broadcasting Standard (GB20600-22006), which is also called DTMB. It describes the detail hardware architecture of most of the consisting blocks, and also presents some of its SNR performance. The whole FEC decoder has been prototyped in FPGAs, and also realized in 0.13um ASIC chip.
