1 INTRODUCTION
DISK arrays [5], such as redundant arrays of inexpensive disks (RAID), have been widely used to improve the performance of storage systems for a decade. Disk failures are fatal to disk arrays so that they make use of erasure-correcting codes to tolerate disk failures. Most of the existing RAID architectures, e.g., RAID-3 and RAID-5, use a simple parity scheme that can recover one disk failure, but now, more powerful schemes are desirable to tolerate more than one disk failure. One of the best-known erasure-correcting codes is Reed-Solomon (RS) codes. RS codes are maximum distance separable (MDS) (see Section 2 for a definition), but they need finite field operations which require a hardware change of the existing RAID architectures. Another promising class of erasure-correcting codes is array codes [2]. Array codes can be encoded and decoded by using parity hardware with which standard RAID architectures are equipped so that they are very suitable for disk arrays. Several classes of array codes have been proposed so far [2]. For example, the EVENODD code [1] is designed to tolerate double disk failures in RAID architectures. Array codes are MDS and have lowcomplexity encoding and decoding procedures based on exclusive-OR (XOR) operations. This is an advantage over RS codes. In addition, array codes have such sparse parity-check and generator matrices that the number of XOR operations needed for encoding and decoding is very small.
In this paper, we present a new class of low-density MDS array codes for tolerating double disk failures in RAID architectures. The proposed code is very similar to the EVENODD code, but has a sparser parity-check matrix than the EVENODD code. This property leads to faster encoding and decoding procedures for the proposed code. When we prepared the revised version of the paper, we were aware of the recent paper [6]. The independent Row-Oblique Parity [6] is essentially the same as the proposed MDS array code, so our paper loses its originality, but gives a simple proof of the MDS property of the code, a little simpler erasure decoding algorithm with a simple proof of its correctness, and a simple error decoding algorithm. So, we believe that our paper still has its own value.

In this section, we review the definition and a property of MDS codes. Let q be a prime power and let IFq denote a finite field of size q. For a positive integer m, we denote by IFqm an m-dimensional vector space over IFq. A code C over IFqm is said to be IFq-linear if C is a vector space over IFq. Note that every linear code over IFqm can be considered as an IFq-linear code over IFqm in the natural way, but the converse is not true.

Definition 1. Codes that meet the Singleton bound are called maximum distance separable (MDS) codes.
Following the notation of [3], we characterize any IFq-linear MDS code C of length n over IFmq by a pair of integers ½n; k, where k ¼ logqm jCj, which [4] calls the rate of C. Note that km is the dimension of C as a linear space over IFq. The redundancy of C is given by r ¼ n À k and the minimum distance equals r þ 1. An IFq-linear MDS code of length n over IFqm and redundancy r is defined by an mr Â mn parity-check matrix over IFq. We want to construct IFq-linear MDS codes over IFmq with sparse parity-check matrices because the sparseness leads to low-complexity encoding and decoding procedures for the codes. For the sparseness of IFq-linear MDS codes over IFmq , we have the following [3, Proposition 3.4 (a)]: Theorem 1. Let C be an IFq-linear ½n; k ¼ n À r MDS code over IFqm
defined by an mr Â mn parity-check matrix H. Then, there are at least k þ 1 nonzero entries in each row of H.
A trivial IFq-linear MDS code over IFqm meeting the above lower bound is the single parity-check (SPC) code over IFmq , which is the so-called simple parity scheme in the terminology of disk arrays.

3.2 Erasure Decoding Algorithm
Using the geometric description of the proposed code, we can give an efficient erasure decoding algorithm for the proposed code. The erasure decoding algorithm for the code is shown in Fig. 3. In the main case of the algorithm, we recover two erased columns, u and v, 0 u < v < p. Let s ¼ v À u and l ¼ 1. In Step (a) of the first loop, we use the diagonal parity-check (4) passing the ðp À 1; vÞ-entry apÀ1;v, which is zero, and determine the ðhÀ1 þ sip; uÞ-entry ahÀ1þsip;u. Next, in Step (b), we use the horizontal parity-check (3) passing ahÀ1þsip;u to determine the ðhÀ1 þ sip; vÞ-entry ahÀ1þsip;v. We increment l and return to Step (a), in which we use the diagonal parity-check (4) passing the ðhÀ1 þ sip; vÞ-entry ahÀ1þsip;v to determine the ðhÀ1 þ 2sip; uÞ-entry ahÀ1þ2sip;u. We proceed to Step (b) again. We continue these procedures until l reaches hÀv=sip.

4 CONCLUSION
In this paper, we have presented a new class of low-density MDS array codes for tolerating double disk failures in RAID architectures. The proposed code has lower complexity encoding, erasure decoding, and error decoding procedures than the EVENODD code has.
In a sequel paper, we will present an extension of the proposed code to a more general case.
