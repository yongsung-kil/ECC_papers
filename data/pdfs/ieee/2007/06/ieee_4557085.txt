ISIT2007, Nice, France, June 24 - June 29, 2007

On the error correction of regular LDPC codes using the flipping algorithm
David Burshtein' School of Electrical Engineering, Tel-Aviv University, Tel-Aviv 69978, Israel

Abstract- We apply the iterative bit flipping algorithm to the standard regular low-density parity-check (LDPC) code ensemble. In the past it was shown, for a typical code in the ensemble with left degree at least five and block length sufficiently large, that this algorithm can correct all error patterns with some linear (in the block length) number of errors. We extend this result to the case where the left degree is at least four. For the case where the left degree is larger than four, we obtain an improvement of several orders of magnitude compared to the existing results on the fraction of worst case errors that can be corrected. We also show how our results can be improved when we consider random errors (as opposed to worst case errors) produced by the channel.
I. INTRODUCTION
Gallager [1] suggested a simple iterative hard decoding algorithm, instead of his soft decoding (belief propagation) algorithm for decoding LDPC codes. The proposed algorithm flips in sequence each bit which has more unsatisfied parity check bits than satisfied ones. The bit flipping algorithm was used by Zyablov and Pinsker [2] to analyze the performance of LDPC codes under practical iterative decoding. It was shown that asymptotically, almost all codes in the regular code ensemble with left degree at least five, can correct a linear fraction of worst case errors. That is, a typical code can correct an arbitrary error pattern, whose error rate is smaller than some fixed positive number, in any transmitted codeword. The total required computational complexity is O(N log N) where N is the block length. This conclusion was also obtained by Sipser and Spielman [3] using expander graph arguments. In [4] we have shown that expander graph arguments can also be applied to message passing algorithms. For the linear programming (LP) decoder, it was also shown [5], [6] that a linear number of errors can be corrected, however although the computational complexity of the LP decoder is polynomial in the block length, it may be larger than O(N log N).
In this paper we apply the bit flipping algorithm to the standard regular LDPC code ensemble, and extend the results of [2] and [3] to the case where the left degree is at least four. For the case where the left degree is larger than four, we obtain an improvement of several orders of magnitude on the fraction of worst case errors that can be corrected, compared to the existing results. We also show how our results can be further improved when we consider random errors (as opposed to worst case errors) produced by the channel.

II. THE ERROR CORRECTION OF (c > 4, d)-REGULAR CODES
Consider the standard (c, d)-regular LDPC bipartite graphbased ensemble. We denote by R = 1-c/d the design rate of the ensemble in bits per channel use, and let N denote the code length. The total number of edges in the graph is Nc = (1 -R)Nd. In this paper we analyze the bit flipping hard decoding algorithm (parallel version) which is defined as follows.
Algorithm 1: Bit flipping decoding algorithm [3]. Iterate the following: For each variable node, count the number of unsatisfied check nodes among its neighbors. Then set all variable nodes in parallel using the following rules. Flip each variable node with more than c/2 unsatisfied parity check neighbors. Set the value of each variable node with exactly c/2 unsatisfied parity check neighbors to 0 (1) with probability 1/2. Notes:
1) When parallel edges connect some pair of variable and check nodes, we count the vote of the parity check node (satisfied or not) according to the multiplicity of the parallel edges (each edge has one vote).
2) We have slightly changed the original definition of the flipping algorithm when there are exactly c/2 unsatisfied parity check neighbors. This case is possible only if c is even. In fact, in the proofs, this modification will only be needed for the case where c= 4.
Following [3] we say that a variable node is bad in the beginning of some iteration if its value is different from the true codeword bit value. Otherwise, we say that the variable node is good. We say that a variable node is an erasure (or erased) after the iteration if exactly c/2 of its parity check neighbors are unsatisfied. Thus, about half the erased bits at the end of the iteration will be set to bad by Algorithm 1. The other half will be set to good.

III. THE ERROR CORRECTION OF (4, d)-REGULAR CODES
We now extend the results of the previous section to the case where the left degree is four, and show that almost all codes in the (4, d)-regular LDPC code ensemble can correct a linear number of worst case errors. This case is important, since under a fixed rate constraint (which implies some fixed ratio of the left and right degrees), the performance of regular LDPC codes under iterative decoding typically improves when the right and left degrees decrease. Note that under maximum likelihood decoding the opposite is true.

required to correct all the errors is O(log N) so that the total computational complexity is O(N log N). Unlike Theorem 1,

However it asserts success for almost all these sequences (i.e., asymptotically success is guaranteed with probability that approaches
one). Thus, for a code drawn at random from the (c, d)-regular LDPC code ensemble with d > c > 4, we can assert the following (similar to [10] that made this assertion for d > c > 5). Once some algorithm (e.g. belief propagation) has been used to reduce the bit error rate below some threshold, we can switch to the flipping algorithm and correct all the remaining errors with probability that approaches one asymptotically.
The proof is provided in [7] and partly uses ideas from the proof of Theorem 1, with the following important differences. In Theorem 2 and in its proof we consider two probability spaces. The first concerns the ensemble of codes and the second concerns the sequence of random bits that the algorithm uses in order to determine the state of erased variables. While in the proof of Theorem 1 we consider only four variable sets,
B12, B3, 912 and 93, in the proof of Theorem 2 we consider

six variable sets, B1, B2, B33, i9, 2 and 93. In the proof of
Theorem 2 we consider two phases in the operation of the
algorithm. In the first one, the number of errors is larger than p log N (any p > 0 can be used here). In this phase we prove w.p. 1 -o(1), that in O (log N) consecutive iterations
the algorithm will correct at least some fixed fraction of the
errors in each iteration. In the second phase, the number of errors is less than p log N (here p > 0 needs to be sufficiently small), and as shown, for almost all codes, each iteration this number cannot increase, and with probability at least 1/2 it decreases, until all the errors have been corrected.
When the left degree is three, it is not true any longer that the flipping algorithm can correct a linear number of worst case errors in almost all codes in the ensemble. It can be shown that a non-negligible fraction of the codes will have parallel edges. Now, if the bad set consists of a single variable node, which is connected by two parallel edges to some parity check node, the algorithm will not be able to correct this error, since
two out of the three edges of the bad variable are connected to a satisfied parity check node. Thus, in this case the algorithm will not be able to correct even a single worst case error.
IV. CONCLUSION
The improvements that we obtained compared to previous analysis techniques [2], [3] were realized by considering simultaneously the set of variables that are bad before and
after the iteration, and those that are good before and bad after the iteration (i.e. analyzing simultaneously the two corresponding graphical events). Another source of improvement
was obtained by deriving better bounds for the first iteration, since then the errors are located at a random position.
