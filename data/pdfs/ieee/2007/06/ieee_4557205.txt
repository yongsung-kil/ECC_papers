Abstract- The generalized iterative decoding concept offers attractive performance versus complexity trade-off opportunities in the spectrum between traditional iterative decoding and optimal decoding for linear block codes over the binary erasure channel. In each iteration, a system of equations is solved. The maximum number of equations to be solved in one iteration is called the order of the decoder. In case the order is just one, the generalized iterative decoder reduces to the traditional iterative decoder. On the other hand, if the order is set to the redundancy of the codes, the generalized iterative decoder gives the same performance as the optimal decoder. Varying the order between these two extremes allows for a better match to the system specifications. In this paper, we consider aspects regarding the implementation of generalized iterative decoding and we determine the minimum order (as a function of the girth) that can potentially lead to improvement over traditional iterative decoding.
I. INTRODUCTION
Iterative decoding techniques, especially when applied to low-density parity-check (LDPC) codes, have attracted a great attention recently. In these techniques, decoding is based on a Tanner graph determined by a parity-check matrix of the code, which does not necessarily have full rank. It is well known that the performance of iterative decoding algorithms in case of binary erasure channels depends on the sizes of the stopping sets associated with the Tanner graph representing the code [3]. Several interesting results on stopping sets associated with Tanner graphs of given girths are given in, e.g., [8], [10]. General results pertaining to ensembles of LDPC codes are provided in, e.g., [3], [7]. There are more specific results for classes of codes represented by particular Tanner graphs, see, e.g., [4], [14]. The relationship between the stopping sets on one hand and the underlying code representation, rather than the code itself, on the other hand, is studied in, e.g., [5], [6], [12], [13]. Algorithms to continue the decoding procedure in case a nonempty stopping set is reached are presented in [11].
Since linear algebra is often used in studying stopping sets, parity-check matrices may be more convenient than the equivalent Tanner graphs for code representation. Let C be a
binary linear [n, k, d] block code, where n, k, and d denote the

The iterative decoder of order p attempts solving a system of no more than p equations, involving an equal number of unknowns, at each iteration. Solving this system is equivalent to inverting a square invertible matrix of size at most p x p. In case p = 1, the algorithm reduces to the iterative decoding algorithm described earlier. In fact, stopping sets and deadend sets of order p, for p = 1, are precisely stopping sets and dead-end sets as described previously in Section I.

IV. IMPLEMENTATION OF GENERALIZED ITERATIVE DECODING
The generalized iterative decoding concept can be implemented in several ways. Recall that in each successful step, the iterative decoder of order p solves a system of t equations in t unknowns for some t < p. The complexity of retrieving an unknown is the ratio of the complexity of solving such system divided by the number t of unknowns. This ratio is an increasing function of t. Therefore, for efficient implementation, the

decoder should solve the smallest possible system of equations

if there is more than one choice. Notice that the probability

of unsuccessful decoding for an iterative decoder of order p,

which depends only on the channel probability of erasure, c,

and the dead-end set enumerator of order p, D(P)x(), does

not depend on such choices as long as the decoder attempts

all possibilities before declaring a decoding failure.

Therefore, we suggest an implementation that attempts to

solve for t unknowns starting with t 1. If this is not possible,

the decoder attempts to solve for t 3 unknowns, provided

that p > 3. If this is not possible, the decoder increments t

by one and attempts to solve for t unknowns until the value

of t reaches p. If, at any step, t equations can be solved to

determine t unknowns, the erasure set is updated and the value

of t is reset to 1 again. On the other hand, if t reaches the value

p and there is no system of p equations that can be solved in

p unknowns, then the decoder declares a failure. The erasure

set at this point is a stopping set of order p.

If at a certain stage in the decoding process the erasure set

is S and systems of t equations are under consideration, then

VI. CONCLUSIONS
In this paper, we have reviewed the generalized iterative decoding procedure for linear binary block codes over erasure

channels and some of its basic properties. Further, it was
argued that simple implementations of generalized iterative
decoders are possible. Finally, it was shown that the minimum
iterative decoding order that can potentially lead to improve-
ment over traditional iterative decoding on a Tanner graph with
girth g equals L3g/4]. For example, this means that for LDPC
codes with girth six, (i) an iterative decoder of order three
performs the same as a traditional iterative decoder, but (ii) an
iterative decoder of order four or more may give (considerable)
improvements.
