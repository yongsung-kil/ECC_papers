An Algorithm to Find All Small-Size Stopping Sets
of Low-Density Parity-Check Matrices
Eirik Rosnes and 0yvind Ytrehus
Dept. of Informatics, University of Bergen, N-5020 Bergen, Norway Email: { eirik, oyvind }@ii.uib.no

Abstract- In this work, we introduce an efficient algorithm to find all stopping sets of size less than some threshold of a fixed low-density parity-check (LDPC) matrix. The solution is inspired by the algorithm proposed by Rosnes and Ytrehus in 2005 to find an exhaustive list of all small-size turbo stopping sets in a turbo code. The efficiency of the proposed algorithm is demonstrated by several numerical examples. For instance, we have applied the algorithm to the well-known (3, 5)-regular (155,64) Tanner code and found all stopping sets of size at most 18 in about one minute on a standard desktop computer. Also, we have verified that the minimum stopping set size of the
(4896, 2474) Ramanujan-Margulis code is indeed 24, and that
the corresponding multiplicity is exactly 204.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were originally introduced by Gallager [1] and later rediscovered in the mid 1990's by MacKay and Neal [2]. These codes have attracted a great amount of interest during the last decade due to their capacity-approaching capabilities and low-complexity message-passing decoding algorithms [3]. There are several ways to construct competitive LDPC codes, both random constructions, e.g., [4], [5] and algebraic constructions, e.g., [6], [7] have been proposed.
The performance of LDPC codes under iterative messagepassing algorithms is dominated by certain structural properties of the code's graphical representation, on which these decoding algorithms operate. For the binary erasure channel (BEC), the error-prone structures are called stopping sets and were introduced by Di et al. in [8]. For other channels, it is much more difficult even to give a precise definition of errorprone patterns. Recently, several analysis techniques have been proposed, which has lead to the definitions of near-codewords [9], trapping sets [10], and pseudo-codewords [11].
The problem of computing the minimum distance of an arbitrary binary linear code was conjectured to be NP-hard, i.e., the existence of a deterministic polynomial-time algorithm for this problem implies the existence of such an algorithm for any problem in NP, by Berlekamp et al. in [12]. This conjecture was settled in an affirmative way by Vardy in [13]. Recently, the problem of computing the minimum stopping set size for an arbitrary Tanner graph (with or without a sparsity constraint) was also proved to be NP-hard [14]. Finally, Wang et al. [15] extended this result to k-out trapping sets. The inherent intractability of computing the minimum distance/stopping set size of a fixed LDPC matrix has resulted in several approximate algorithms [16]-[19]. For instance, in

[16], Hu et al. introduced the nearest nonzero codeword search (NNCS) approach to approximate the minimum distance of an LDPC code. The method uses reliability-based list decoding based on the algorithm from [20] with controlled noise. Both error-impulse noise and bit-reversing noise are considered.
The fact that the problem of exhausting all low-weight error-prone patterns is NP-hard does not prevent the existence of practical enumeration algorithms for LDPC matrices with small parameters. In [15], Wang et al. proposed the first such algorithm to find all low-weight error-prone patterns for a fixed LDPC matrix. The error-prone patterns considered in [15] are codewords, stopping sets, and k-out trapping sets.
In this work, we introduce an efficient algorithm to find all stopping sets of size less than some threshold of an arbitrary fixed LDPC matrix. The algorithm is different from, and much more efficient than, the algorithm presented in [15]. This will become evident in the numerical results section. The proposed algorithm is inspired by the algorithm in [21], [22], which enumerates small-size turbo stopping sets.

Extended iterative decoding on the Tanner graph, whose input is a binary length-n vector where some of the positions are erased, is formulated as follows. An integer label is associated with each check node with initial value equal to the number of 1-constrained variable nodes connected to the check node. In addition, all check nodes are marked "unfinished". For all unfinished check nodes do the following. a) If the values of all but one of the variable nodes connected to the check node are known, and its label is zero or one, set the missing variable bit to the XOR of the other variable nodes, increase the label of all check nodes connected to the missing variable node with the value of the missing variable node, and mark it as a finished check node. b) If all the variable nodes connected to a check node are known, then if its integer label is different from one, mark the check node "finished", otherwise mark it "contradicted". The procedure is done sequentially, i.e, one check node at a time, and continues until all check nodes are marked as finished, there is at least one check node marked as contradicted, or the algorithm cannot continue further.
Note that extended iterative decoding is equivalent to iterative belief-propagation (BP) decoding on the BEC, as described in [23], if the condition of having a label of zero or one in a) is removed. In this case no check node will be marked "contradicted", since the BEC does not introduce errors. Also, when searching for codewords only, the condition of having a label of zero or one in a) in extended iterative decoding has to be removed, and in b) a check node is marked "contradicted" when its integer value is odd and all variable nodes connected to it are known. Finally, we need to relax the definition of an active row in the sense that a row is active if and only if the row weight is odd.

A. The (3, 5) -regular (155, 64) Tanner code from [25]
In [25], Tanner et al. constructed a particularly nice (3, 5)regular LDPC code of length 155, dimension 64, and minimum distance 20. The underlying Tanner graph has girth 8 and its relatively large minimum distance of 20 [25] makes the code an excellent candidate for iterative decoding. According to the literature, the minimum stopping set size is between 13 and 18, since a stopping set of size 18 has been located by MonteCarlo simulation [15], and it has been verified that there are no stopping sets of size at most 12 by the algorithm from [15]. Here, we have found all stopping sets of size at most 22

using the algorithm in Section III and exploiting the transitive automorphism group of SH. The minimum stopping set size is 18 and the corresponding multiplicity is 465. In fact, we found all stopping sets of size at most 18 in about one minute on a standard desktop computer, which illustrates the efficiency of the proposed algorithm compared to the algorithm from [15].
B. The (3,6)-regular (504,252) MacKay code from [26]
This code is taken from the online database of sparse graph codes from MacKay's website [26]. The minimum stopping set size was estimated with high confidence to be 16 in [18]. Indeed, the minimum stopping set size is 16. Furthermore, we have found all stopping sets of size at most 20.
C. The (3,6)-regular (504,252) quadratic permutation polynomial (QPP) based code from [7]
In [7], Takeshita proposed a novel algebraic construction of LDPC codes using QPPs. The construction gives a quasi-cyclic SH, and we have exploited this property in the algorithm in Section III, as outlined in Section IV-A. For this particular code, the minimum distance is previously unknown, but a codeword of weight 22 has been reported in [7]. This codeword was found using the NNCS approach from [16]. Here, we have found all stopping sets of size at most 22. The minimum stopping set size turns out to be 18, and all stopping sets of size 18 correspond to codewords.
D. The almost (3,6)-regular (504, 252) progressive edgegrowth (PEG) code from [26]
This code is a PEG constructed almost (3, 6)-regular LDPC code. In particular, all the left vertices have degree 3 and all the right vertices, except 6, have degree 6. The code can be found in the online database of sparse graph codes from MacKay's website [26], but was designed by Hu using the PEG algorithm from [29]. The minimum stopping set size is 19, and the corresponding multiplicity is 2.
E. The (3,6)-regular (672,336) Margulis code from [27]
It is known that this code has a codeword of weight 16 [30]. In [15], Wang et al. verified that there are no stopping sets of size at most 13. Thus, according to the literature, the minimum stopping set size is between 14 and 16. Here, we have found all stopping sets of size at most 22 using the algorithm in Section III and exploiting the automorphism group of SH. Note that the minimum stopping set size is 16 and that all stopping sets of size 16 correspond to codewords. Also, there are no other stopping sets of size at most 22. In fact, all stopping sets of size at most 16 were found in less than one minute on a standard desktop computer, which illustrates the efficiency of the proposed algorithm compared to the algorithm from [15].
