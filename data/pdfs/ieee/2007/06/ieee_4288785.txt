AbstractвҖ” The construction of good, finite-length, LDPC codes is currently an attractive research area. Reducing attention to the Binary Erasure Channel (BEC), this problem translates into the problem of finding elements of selected (irregular) LDPC code ensemble with the size of the minimal stopping set being maximized. Faced with the lack of analytical solution to this problem, simple but powerful heuristic design algorithm, an ACE constrained design algorithm, was recently introduced. Building upon the ACE metric associated with every cycle in the code graph, we introduce the ACE spectrum of LDPC code as a useful measure of success in selection of the LDPC code from selected ensemble. Using ACE spectrum, we further generalize ACE constrained design, making it more flexible and efficient.
I. INTRODUCTION
The construction of short and medium length LDPC codes is currently an attractive research area. Many practical systems are constrained by delay, memory, processing or similar requirements that translate into relatively small maximum tolerable code lengths. Therefore, an asymptotic behavior predicted by Density Evolution (DE) [1] analysis cannot be fully taken into account, as it is known that the short-length LDPC codes designed using only DE guidelines behave poorly. Instead, one has to analyze finite-length behavior under iterative BeliefPropagation (BP) decoding to design LDPC code graphs that behave well. In other words, the task is to construct the LDPC code graphs that are free of the subgraphs that are proved to present a problem for the iterative BP decoder.
Our LDPC code design and simulation results presented assume transmission over the AWGN (Additive White Gaussian Noise) channel. Beside the fact that there is a lot of work done recently in the area of finite-length behavior of iterative BP decoding of the LDPC codes over the AWGN channel, for the sake of simplicity we use results that address finite-length behavior over the BEC for our LDPC code design purposes. It turns out that designing good LDPC codes for the BEC produces good LDPC codes for the AWGN channel as well, making this design methodology attractive in a wider sense.
Finite-length behavior of LDPC codes decoded iteratively over the BEC is fully determined by the subgraphs of the LDPC code graph called stopping sets [2]. Small stopping sets deteriorate the performance of LDPC codes, particularly in the error-floor region. There is no explicit construction that

optimizes LDPC code graphs with respect to the minimal stopping set size and this presents one of the most challenging open problems in the LDPC code design. However, recently proposed ACE constrained algorithm [3] heuristically produces LDPC code graphs with significantly larger minimal stopping sets compared to random constructions.
This paper builds upon the ACE constrained LDPC code design. Following the introductory section, Section 2 briefly explaines the ACE metric associated with every cycle in the LDPC code graph and the idea behind the ACE constrained design proposed in [3]. Based on the ACE metric, Section 3 introduces the ACE spectrum of LDPC code, which can be calculated efficiently for any LDPC code, and can provide useful insights into the LDPC code graph structure and performance behavior. The same section gives examples of the ACE spectra of selected LDPC code constructions, accompanied with their Bit Error Rate (BER) performance. In Section 4 we introduce generalized version of the ACE constrained LDPC code design, based on the ACE Spectrum, and provide simulation results that justify the generalized ACE constrained approach. Some interesting, yet unsolved, questions concerning the ACE spectrum and generalized ACE design are presented in the last, concluding Section.
II. CYCLE ACE METRIC
The design of good LDPC code graph assumes maximizing the size of the minimal stopping set. Tanner graph, or LDPC code graph G(H), is derived from the LDPC code parity check matrix H in a usual way, where variable (or left) nodes correspond to columns, check (or right) nodes correspond to rows, and edges correspond to ones in the parity check matrix. We will start with the definition of a stopping set [2].
Definition 2.1: A set of variable nodes of the LDPC code graph G(H), such that every check node neighbor of this set is connected to this set at least twice, is called a stopping set. (However, under the term stopping set we will assume a subgraph of the original LDPC code graph that is induced from variable nodes in a stopping set.)

In this paper, our approach to the LDPC code design is based solely on the ACE values of the cycles in the code graph. In general, pushing the ACE spectrum components to a larger values, particularly in the region of short cycles, produces codes with lower error floors. However, sometimes there are exceptions, and even with the good ACE spectrum we lack good error floor behavior. The reason for this behavior is the occurence of the low-weight near-codewords [9] (trapping sets [10]) in the code graph. For a nice treatment of the additional steps in the code design suggested to prevent the occurence of the bad near-codewords, please refer to [11] and [12].
V. CONCLUSIONS AND FUTURE WORK
In this paper, we introduce the ACE spectrum analysis that provides useful insights into the code graph structure and quick mechanisms to estimate the code performance behavior, without performing lenghty BER performance simulations. Obtaining the expanded ACE spectrum of reasonably large depth of short to medium length LDPC codes is of several orders less time-consuming process than the BER estimation process.
Apart from its diagnosis purpose, we used the ACE spectrum to generalize the existing ACE constrained design. Instead of allowing only flat ACE spectrum, which can be potentially very limiting, in the generalized ACE constrained design we allow arbitrary shape of the ACE spectrum, dealing with each ACE spectrum component independently.
Our ongoing work is concerned with the following three problems. First one is posed in the previous Section, namely, for irregular (О», ПҒ)-LDPC ensemble, order the (l, О·) parameter pairs of the cycles according to potential damage they can produce as part of the code graph. This could help us to shape

the ACE spectrum appropriately. The second problem is to introduce an appropriate order relation in the set of ACE spectra, and to answer the following question: for a given irregular (О», ПҒ)-LDPC ensemble of length n, what is the maximum (or the set of maximal) possible ACE spectrum value for this ensemble? The second question is immediately followed by the third and the most practical question: how (by which design method) can we design LDPC codes with extremal ACE spectrum properties? In parallel with this work, we have made some progress in giving answer to the last question. Basically, we can use the generalized ACE design approach as in the Example 4.1 (the approach of [3]), i.e. to randomly select each variable node neighbors repeatedly until we satisfy given ACE spectrum condition. However, by increasing the values of ACE spectrum components this quickly becomes impractically long design process, particularly for the largest degree variable nodes. Instead, in [13] we modified the well known PEG algorithm [4] and, continuing the work presented in [14] that combines PEG and ACE constrained design, we obtained LDPC code graphs with substantially better ACE properties.
