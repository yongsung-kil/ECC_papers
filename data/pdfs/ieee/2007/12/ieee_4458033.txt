Abstract- In this work, a (576,288) LDPC decoder for 802.16e standard is presented. This design is a partially parallel architecture based on a new optimally reordered decoding scheme. Besides, the proposed architecture handles two different code words at a time to achieve 100% utilization rate of both CNU and BNU. As a result, high throughput and low hardware complexity are achieved. In chip implementation, the proposed design achieves a data rate of
1.45Gb/s with 10 iterations and 7 quantization bits, at the cost of
881K gates, based on UMC 0.18p:m process technology.
Keywords- LDPC code, Decoder, Architecture

I. INTRODUCTION
Low-density parity check (LDPC) code was introduced in 1962 [1], which can achieve performance close to Shannon bound. As such, LDPC has been adopted by many state-ofthe-arts communication systems, such as the recent DVB-S2, DMB-TH and 802.16e systems. It is a kind of binary linear block code whose parity check matrix is sparse which has much fewer l's than a common matrix. A sparse parity check matrix facilitates simple decoding algorithms and lowcomplexity decoder designs.
Check matrix of a LDPC code is often represented by a bipartite graph, called Tanner graph [2], which is composed of n bit nodes Vj, j=1 to n, and m check nodes ci, i=1 to m. Those bit nodes and check nodes are connected by edges defined by nonzero entries of the decoding check matrix H. Tanner graph shows a clear picture of all the information exchange links in a decoding process. Fig. 1 shows an example
with 4 check nodes and 8 bit nodes. The number of "1" in each colunm of H determines the number of edges for each bit node connected to check nodes, and the number of "1" in each row of H determines the connections from each check node to bit nodes.
In a Tanner graph, a check node and a bit node need to execute their own check node equation (CNE) and bit node equation (BNE).

Fig. 1. Tanner graph of a parity check matrix. Although LDPC codes are efficient, huge computation is required particularly for the decoding processes. As a result, in realization, serial architectures are not suitable for LDPC decoder designs. To meet the decoding throughput requirement, a fully parallel LDPC decoder [3] is proposed. In the architecture, each CNE and BNE are realized by their own check node unit (CNU) and bit node unit (BNU), respectively. However, its large area and long on-chip interconnected wires present implementation challenges in placement and routing. To reduce this effect, a partially parallel architecture [4] was introduced, where a few parallel CNUs and BNUs are implemented and shared by all the CNEs and BNEs, respectively. However, its large input multiplexers still cost a lot of chip area. In this paper, an efficient design is proposed, which schedules the decoding processes in an optimal way so that it eliminates multiplexers and further reduces the placement and routing problems.
