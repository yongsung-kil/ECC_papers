Architecture of the Modified Block-Type LowDensity Parity-Check Code Decoding Design

Kuang-Hao Lin, Robert C. Chang, Member, IEEE, Sheng-Dong Wu
Department of Electrical Engineering National Chung Hsing University Taichung, Taiwan 89364306@ee.nchu.edu.tw

AbstractвҖ”In this paper, a combined Low-Density ParityCheck (LDPC) code decoding design method, called modified Block-Type LDPC (B-LDPC), for realistic LDPC coding system architectures is presented. The B-LDPC code, which is a special class of quasi-cyclic LDPC (QC-LDPC), has an efficient encoding algorithm owing to the simple structure of their parity-check matrices. A proposed distribution of irregular parity-check matrix for the modified B-LDPC is developed so that we can obtain an area-efficient decoder design, good error correction performance, and achievable architecture implementation. The modified B-LDPC code decoding utilizes the iterative min-sum algorithm (MSA) and its decoding architecture design employs the non-overlapping operations of bit node unit (BNU) and check node unit (CNU). Different block matrix sizes for parity-check matrix can be adopted so that the modified B-LDPC code decoding improves the throughput without obvious performance degradation.
KeywordsвҖ”Low-Density Parity-Check (LDPC), decoder, encoder, hardware architecture.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes are linear block codes which were first discovered by Gallager [1] and they have better block error rate performance. More recently, with an increase in algebraic and systematic construction approaches to the study of LDPC code based on finite geometries, we have seen a shift in patterns of research. Because the usage of high-throughput data rate increase, reliability requirements for error correction have become critical. LDPC codes technology have been widely used for the digital communication due to their high bit error rate (BER) and frame error rate (FER) performances.
GallagerвҖҷs theory [1] and Richardson et al. [2] offered a sounder theoretical basis for noisy channel coding theorem. Their codes can be decoded in various ways ranging from low to high decoding complexity and from reasonably good to very good error performance. Furthermore, these codes are either cyclic or quasi-cyclic (QC). The majority of research in hardware implementation has focused on QC. Myung et al. [3] provided an excellent review of the methods, results and instructional issues related to QC LDPC encoding and decoding. A special class of QC-LDPC codes, called BlockLDPC, was presented for practical LDPC coding system implementations [3, 4]. The main idea is to construct LDPC

codes subject to certain hardware-oriented constructs that ensure the effective encoder and decoder hardware implementations. The related works reported of decoding in the literature can be classified into two major categories of sum-product algorithm (SPA) or min-sum algorithm (MSA). The readers are referred to good tutorials in [5] and [6] for an introduction to the above decoding algorithm approaches and their applications. Howard et al. [7] has suggested the benefit of soft-bit decoding analysis in LDPC.
In this paper, a special class of QC-LDPC codes, called the modified B-LDPC codes, is presented. They are linearly encodable and have better memory efficiency as compared with the conventional LDPC codes. The modified B-LDPC code is defined with an proposed irregular parity-check matrix to have almost lower triangular form. This restriction on the structure of its parity-check matrix guarantees a linear encoding complexity but, in general, it also results in some loss of performance. We will determine if the use of various block matrix size would improve the throughput of digital communication and keep error correction performance. The decoder architecture in the receiver is also developed. Simulation results show the performances of modified BLDPC decoder with different coderates and the word-length requirement of the architecture.
The paper is organized as follows. Section II discusses the modified B-LDPC codes and the decoding construction with various block matrix sizes. In section III, we present the efficient decoding architecture. We verify their performance by simulation in Section IV. Conclusions are drawn in Section V.

A. Encoding of the Modified B-LDPC codes
The method presented in this paper is developed from QCLDPC, and we want to find a well-suited parity check matrix in encoding the same as decoding. QC-LDPC has been constructed and can encode efficiently in early years. But in hardware realization, QC-LDPC is not easily accomplished. Therefore, we propose a method that developed the parity check matrix so that the hardware can be achieved easier with lower cost.

The block matrix size has a great influence on the memory access time because a block matrix is related to a memory in this architecture. Take the block matrix size 96Г—96 for example, the total operation time from soft-demapping to BNU and eight iterations is 96+192Г—8=1632 cycles. III. HARDWARE ARCHITECTURE
Figure 2 shows the architecture of the modified B-LDPC decoder design that comprises the bit node processing units (BNUs), check node processing units (CNUs), and register files (RFs) components. Here we use N BNU to sum LLR data of node and the result is stored to RFs. N is denoted as n/L, where n is the number of codeword length and L is the number of block matrix size. The output data of BNUs are stored into M two-port RFs through the counter to calculate address, where M is defined as the number of non-zero block of parity check matrix H. The K CNUs was adopted to determine minimum value of information from memory (RFs) output. The K is expressed as NГ—(1пјҚR) and R is the coderate. In the
first load, the LLR data from soft-demapping transported to BNU, and directly sent to RFs via BNUs component. In which, the BNU Connect determined the connection between BNUs and RFs block according to block matrix address of every column in parity check matrix. Similarly, the CNU Connect is determined according to block matrix address of every row in parity check matrix. Finally, hard decision (H.D.) results from BNUs are computed by BNUs and CNUs components with eight iterations respectively.

The CNU structure is shown in Figure 4. The input data consist of two parts: sign bit and magnitude. We determine the sign bits from the six input data by using XOR operation. In other words, the sign bits of the output data (out1~out6) will be changed or kept when there are odd or even negative input data, respectively.
For the magnitude part, the min and 2nd min will be obtained by comparing six input data with CMP-6 and exchange the value with minimum control function block: if the input is min, the output is 2nd min, otherwise the output is min.

The architecture of BNU is shown in Fig. 3. The goal of this BNU architecture is to add the other input data in addition to itself, i.e. the output data out1=in2+in3+LLRN, where the input data (in1~in3) is mapping to output data (out1~out3).The LLR data is transformed as 2вҖҷs complement for operation purpose due to its sign-magnitude format. The addition results will be transformed again to sign-magnitude before sending out data (out1~out3).
In order to assure the precision after adder operation, we use an 8-bit adder. Before output, we use the Clipping to truncate the output word-length to 6 bits that avoids overflow in the next computing. Finally, the iteration results are the sum of all input data and then output to the H.D. port.

IV. SIMULATION RESULTS
System simulations were accomplished to analyze the performance of the modified B-LDPC decoding design obtained by our construction procedure over an AWGN channel. They were encoded efficiently according to the encoding procedure in Section II.
Figures 5 and 6 show the simulated FER versus signal-tonoise ratio (SNR) and BER versus SNR, respectively. For the purpose of comparison, we construct 200 sets for 2304-bit, block matrix size p=12, and different coderates with 1/2, 2/3, and 5/6. From both figures 5 and 6, the coderate 1/2 performance for the same FER and BER show a SNR loss about 1dB compared to the coderate 2/3. In addition, the coderate 2/3 performance for the same FER and BER show a SNR loss about 2dB compared to the coderate 5/6. The coderate 1/2 has best performance, but it will lead to throughput degradation.
The different sizes of block matrix for 12Г—12, 48Г—48, and 96Г—96 are simulated to present their performances for different hardware complexity, which are shown in Fig. 7. We can find that their performances have not much difference from each other. The analysis of fix-point is given in Fig. 8 to determine word-length requirement for architecture implementation. The word-length with 8-bit is enough for hardware implementation.

V. CONCLUSIONS
In this paper, an area efficient modified B-LDPC code decoding suited for MSA with irregular parity-check matrix H for high-throughput is proposed. The LDPC codes decoder architecture is constructed that ensures the effective encoder and decoder hardware implementation. The analysis results are performed by computer simulation in different coderates. In different block matrix sizes, the 12Г—12 block matrix size has higher data rate than the others and has no obvious performance degradation.
