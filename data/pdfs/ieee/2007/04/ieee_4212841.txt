On the Addition of an Input Buffer to an Iterative Decoder for LDPC Codes

Massimo Rovini Dept. of Information Engineering
University of Pisa via G. Caruso 16, 56122, Pisa, Italy Email: massimo.rovini@iet.unipi.it

Alfonso Martinez Dept. of Electrical Engineering Technische Universiteit Eindhoven 5600MB Den Dolech 2, Eindhoven, The Netherlands Email: alfonso.martinez@ieee.org

Abstract— This paper describes the application of a new hardware architecture to the design of a decoder for Low-Density Parity-Check (LDPC) codes. Thanks to the systematic use of the built-in stopping rule in the decoder, the decoder runs the minimum number of iterations on each packet of received data. The addition of a small buffer on the decoder input allows the exploitation of the variations in the decoding time of different packets, in a spirit similar to statistical multiplexing of the data flow. Analysis and simulations are presented using the decoder for the next generation satellite Digital Video Broadcasting (DVBS2) as a case study, to show that the throughput may be doubled with only two extra buffer locations, at almost no cost in chip area.
I. INTRODUCTION
Interest in the implementation of high-throughput LDPC decoders has exploded in the latest years, and can be traced back to the expected ever-larger demand for high-rate data services. As a representative example of this trend, the new satellite broadcasting standard (DVB-S2) [1] makes use of the most power-efficient channel codes currently known, lowdensity parity-check (LPDC) codes [2]. Specifically, it consists of the serial concatenation of an outer Bose-ChaudhuriHochquenghem (BCH) code with an inner LDPC code. The latter is an Irregular Repeat-Accumulate (IRA) code [3], blockwise structured with size Q = 360. This is exploited to implement partially parallel decoder architectures, in which the parallel update of up to Q nodes is performed, in an singleinstruction multiple-data (SIMD) way.
For LDPC codes, decoding speeds as high as 1 Gb/s have been reported [4]. Note however, that this result is typically achieved with short blocks (1024 bits in [4]), effectively trading off increased speed with reduced performance. To any extent, standard hardware designs fall prey to the worst-case situation, where the resources are determined by the maximum number of iterations required.
A different route, attacking the problem from an architectural point of view, would be preferable. As previously suggested in [5], significant gains are possible if the hardware is able to exploit the variations in the required number of iterations, on a packet by packet basis. This is achieved with the introduction of an input buffer, which stores the incoming data and smooths out the differences in decoding time. As we shall see, the advantage of statistically multiplexing the

decoder operations is twofold: given a chip complexity, more packets are decoded per time unit or, dually, for a given gain in throughput the decoder cost is lower. This also allows scaling down the architectural parallelism as for the decoder presented in [6], but, thanks to the proposed technique, the throughput can be maintained unscaled.
In the proposed approach the use of a (any) stopping criterion is crucial to drive the statistical multiplexing of the incoming packets. Although the use of stopping rules is not new, its application to increase the throughput is innovative, since current state–of–the–art designs exploit it only to save unnecessary power.
II. DECODER STATISTICAL MODEL
We assume throughout that data arrive at the decoder input in ‘frames’ which correspond to the LDPC code. Frames will also be referred to as ‘packets’ with no difference of meaning, and flow at a constant rate Γ.
LDPC decoding consists of repeated applications of the maximum ‘a posteriori’ (MAP) algorithm, also referred to as ‘message-passing’ or ‘belief propagation’ [7], as reliability messages are exchanged between variable and check nodes. To simplify the implementation, elaborations are typically in the log domain, and reliability messages have the form of log-likelihood ratios (LLR); in line with that, data arriving in the input packets are ‘a priori’ LLR. Then, the decoding unit1 carries out a number of iterations Nit on each incoming packet.
Several choices are viable for the algorithm schedule. In [8], a DVB-S2 decoder architecture is described, which performs layered decoding [9] over the nodes in the bi-diagonal, systematic part of the parity-check matrix, with savings in iterations and, although less, memory. Other works implement the traditional flooding (or concurrent) schedule [10]. Also the type of elaborations on check nodes may vary from the straightforward implementation with look-up-table (LUT) to several low-complexity approximations [11], [12].

B. Complexity Reduction
It may be possible to exploit the input buffer as a means of reducing the chip complexity, for a fixed throughput. Fig. 3 depicts the complexity, in mm2, of a traditional design, and of the buffered D/G/1/1 + L decoder as a function of the throughput gain γ. In the traditional approach, throughput is increased through parallelization or replication of functional units, and the related curve is a straight line4; on the contrary, the buffered decoder complexity is given by (6). This yields the remarkable saving in the chip area of about 18.75 mm2 (41.2%) when γ = 2 for the selected case-study.
Note that curves diverge when the throughput gain tends to γ∞, as an infinite buffer should be placed. Therefore, at the selected operating points, only some of the configurations can reach γ = 3. Again, results from (4) exhibit a good match with the measured values.
C. Coverage of Different SNRs
Once the system has been dimensioned for a specific triplet modulation/rate/signal–to–noise ratio, variations in the latter may result in different gains. Figure 4 reports the analysis with different SNR for code rate 2/3 and 8PSK. The system was initially dimensioned for SNR = 6.45 dB with Nit, max = 113 to get PF = 10−6; it was γ = 2 when L = 2. FER and blocking probability PL are drawn in Fig. 4 along with the theoretical and simulated gain γ(2), keeping the same Nit, max = 113 and 2 input buffer locations. Above 6.45 dB, we only required PL < 10−8, the system being designed to get PF = 10−6. Below 6.45 dB the gain γ > 2 is guaranteed up to PF = 10−3, which proofs the robustness of the design to variations of SNR.

In the case of structured LDPC codes, this means that smaller hardware parallelisms can be selected for the implementation, as described [6], but the same throughput (or the required one) can be achieved with the proper size of the input buffer. As a refinement of (6), the complexity of the buffered decoder is:

in which K < 1 is the scaling down factor, so that K · Q functional units are implemented in the decoder; if K = 1/γ(L), then the same throughput of the unscaled design is achieved. Note that in (7) only the functional units are scaled down by K, while the input and the message memory (SIO and Smsg, respectively) are not; the same holds for other constant terms added up into the area of the control logic SCL.
The saving w.r.t. the unscaled design is given by ∆S(L) = (1 − 1/γ(L)) · Sproc + L · SIO, and Fig. 5 shows the normalized gain in complexity (percentage decrease) for several simulated and theoretic cases, as a function of the hardware parallelism factor K = 1/γ. Results show that about 8% of complexity can be saved if L = 1 or L = 2.
Note that the scaling factor K must be always rounded as to have an integer number of functional units K · Q.
IV. CONCLUSIONS
In this paper we applied the concept of statistical multiplexing to the design of a high-throughput architecture for iterative decoders using an extra input buffer and an early stop criterion. Applied to the design of a new generation satellite Digital Video Broadcasting (DVB-S2) decoder, this architecture doubles the decoder throughput with only two buffer locations, roughly saving 40% of the complexity of a traditional design. However, this technique is of wider applicability, and promises a new efficient architectural design for any iterative decoder, especially for very complex iterative algorithms, which do not suit to traditional techniques such as parallelization.

Reverting this approach, input packets can be also statistically multiplexed not to increase the decoding throughput, but dually, to reduce the implementation complexity for a given throughput. Thus, in the case of structured LDPC codes, a smaller hardware parallelism can be selected for the implementation, while the same throughput can be kept with the proper size of the input buffer. In this sense, the design of the LDPC code and the performance of the related decoder are made more independent.
Then, in the design of a real modem, this analysis should be extended to all the supported code rates, and the worst-case (maximum) value of L selected for implementation, although a small spread of its values is expected, as suggested by Fig. 1 for the DVB-S2 case-study.
The choice of Es/N0 also affects the results. However, this technique is very robust to variations of Es/N0, and once the system is dimensioned for a certain operating point (Lˆ, Nˆit,max), the same gain is offered over a wider set of conditions.
Finally, additional care must be taken with those systems featuring the acknowledgement of reception within a preset time, as the extra latency of the input buffer may prevent the applicability of the proposed technique.
