Approximately Lower Triangular Ensembles of LDPC Codes With Linear Encoding Complexity
Shay Freundlich, David Burshtein, Senior Member, IEEE, and Simon Litsyn, Senior Member, IEEE

Abstract—The complexity of brute-force encoding of low-density paritycheck (LDPC) codes is proportional to the square value of the block length. Richardson and Urbanke have proposed efficient encoding algorithms for LDPC codes. These algorithms permute the parity-check matrix of the code iteratively, such that it becomes approximately lower triangular. We propose a new approach for efficient encoding of LDPC codes in which we modify the code ensemble to force an approximate lower triangular structure, thus eliminating the need to apply the algorithms of Richardson and Urbanke in this ensemble. We prove that the new ensemble has the same asymptotic threshold as the corresponding standard ensemble. The new ensemble can be used for linear time encoding of an arbitrary code profile. Computer simulations confirm that the performances of the standard and new ensembles are also very similar when using finite length codes.
Index Terms—Code ensembles, encoding algorithms, iterative decoding, low-density parity-check (LDPC) codes.

I. INTRODUCTION

Low-density parity-check (LDPC) codes were originally proposed

by Gallager [1], and have lately attracted a great deal of interest. Typi-

cally, the maximum transmission rate at which it is possible to transmit

information reliably, using these codes under iterative decoding, is very

close to channel capacity [2], [3]. One disadvantage of LDPC codes,

compared to other iteratively decoded codes such as turbo codes [4] is

their encoding complexity. As any linear code, they can be encoded

N 2 k using a generating matrix, i.e., an

matrix, containing as its

k k columns independent codewords (a basis for the code space). Each

bit information word can then be multiplied by the generating matrix
N to produce an –bit codeword. The complexity of such an encoding O N scheme is quadratic in the block length ( ( 2)) compared to linear

in turbo codes. This difference becomes a significant problem when

dealing with long block lengths.

One way to approach this problem is the cascade code that was

proposed in [5], [6] for transmission over the binary erasure channel

(BEC). However, the cascade structure results in performance loss

compared to a standard LDPC code with the same block length.

Another concatenation scheme for linear time encoding that uses

expander codes was introduced by Spielman [7]. This scheme also

results in performance loss compared to a standard LDPC code.

For some families of LDPC codes it was shown that linear encoding

can be realized. LDPC codes based on finite geometries [8] can be en-

coded in linear time by using feedback shift registers. High-rate LDPC

codes based on finite geometries were shown to perform very well

under iterative decoding. LDPC codes with a parity-check matrix that

has a constant column weight of two can also be encoded in linear

time [9].

some loss of performance. Furthermore, standard tools for optimizing

the code profile [2], [3] cannot be used in this construction. Instead,

the profile was chosen intuitively, so that it lent itself to interesting ex-

periments. The performance of the proposed codes was tested using

computer simulations. It was demonstrated that one can choose lin-

can perform almost as well as other codes from the same profile, but it was shown for specific profiles, and no general tools were developed to construct optimal linearly encodable codes. Lower triangular structures for linear encoding complexity were also proposed in [13]. In [14], it was proposed to impose a lower triangular structure on the parity-check matrix and incorporate it into the progressive edge growth algorithm.
In this work, we introduce an ALT-LDPC ensemble, which places less rigid constraints upon the code, but still forces it to have an ALT form, so that linear time encoding can be realized. Given an arbitrary row and column weight distribution, we show how it can be modified to impose the ALT constraint.
In [11], one picks a code at random from the standard ensemble, and then uses successive row and column permutations, to obtain a code with an approximately lower triangular structure with some gap value, whose expected value is determined by the code profile. The method in [11] does not guarantee linear time encoding for an arbitrary code profile. Our approach is, for any given code profile, to restrict attention only to codes with approximately lower triangular structure that have some prespecified gap value. We prove that for any gap value, the asymptotic performance of the new ensemble is the same as the per-
p formance of the standard ensemble. Hence, by choosing the gap suffi-
ciently small (O( N )) we can guarantee linear encoding complexity
and the same asymptotic performance that the standard ensemble has. Furthermore, computer simulations confirm that as long as the gap is not too small, the finite length performance of codes in our linear complexity encoding ensemble is the same as the performance of the stan-
p dard ensemble (in our simulations this was achieved by setting the gap
to G = N ). There is a tradeoff between code performance, that in-
creases for increasing gap, and encoding complexity, and our approach provides flexibility for the chosen gap, since we are not forced to choose the gap of the standard ensemble that results by applying the preprocessing algorithms of [11]. Furthermore, the gap of [11] is sometimes too large so that the encoding complexity is not linear. In addition, our approach saves the need to employ the preprocessing algorithms of [9], since our ensemble is already forced to have the approximately lower triangular structure.
This work is organized as follows. In Section II, we present some background and notations. In Section III, we define the ALT ensemble and analyze its performance. In Section IV, we compare the standard and ALT ensembles at finite block lengths using computer simulations. Section V concludes the correspondence.

Considering all the figures (Figs. 3–11) it can be seen for all ensembles, that the graphs sometimes fluctuate at the low BER region. This is due to statistical instability of the error estimates in this region. However, in the LT ensemble, the graph can fluctuate even at a relatively higher BER (see in particular Fig. 11). We believe that this is a consequence of the fact that in the LT ensemble there is higher variability between codes in the ensemble. This variability is due to a relatively high probability that the drawn code will have small cycles associated with variables with low indices, which results in a relatively bad code. In Fig. 11 we consider a block length of N = 100000. For this block length we used only 1000 different codes in the averaging process. Therefore the fluctuations are even higher.

V. CONCLUSION
We introduced the ALT ensemble, which offers practical linear time encoding of LDPC codes. We proved that any standard irregular (; ) LDPC code ensemble can be converted to the ALT ensemble, thus providing linear time encoding, such that the asymptotic threshold of the ALT ensemble is identical to the asymptotic threshold of the corresponding standard ensemble. Thus, one can design a good degree profile using the known optimization procedures, and then use this profile in the ALT ensemble. The ALT ensemble has a built-in tradeoff between the gap size (i.e., the encoding complexity) and the performance, for any given block length.
By the results in [11], a typical code in the standard ensemble also belongs to the ALT ensemble (after the appropriate row and column permutations). Therefore, the two ensembles are not far from each other.

The difference is that in the standard ensemble the expected value of the gap is a function of the code profile, while in the ALT ensemble the
p gap is a prespecified value. Simulation results confirm that the perfor-
mance of the ALT ensemble with a gap of G = N is very similar to the performance of the standard ensemble.
