Reduced Complexity and Improved Performance for Short Regular LDPC Codes Based on Select Updating Schedule

Jianquan Liu, Youyun Xu, Yueming Cai Institute of Communications Engineering, PLAUST, Nanjing 210007, P. R. China
jianquanliu@gmail.com, yyxu@vip.sina.com, caiym@vip.sina.com

Abstract—In this paper, we present a modified BP decoding algorithm based on select updating schedule by analyzing the properties of message-passing through cycles, we also proposed a concrete scheme to obtain the select updating schedule, the simulations prove that we could improve performance and reduce complexity at the same time for short regular LDPC codes which have a small difference between the maximal and minimal single bit node girths by select updating schedule compared with the conventional BP decoding algorithm.
Keywords-LDPC codes; belief propagation; select updating schedule; single bit node girth; grouped iteration
I. INTRODUCTION
Low density parity check (LDPC) codes [1] have been attracted great attention due to its near Shannon limit performance and parallelizable decoding based on belief propagation (BP) algorithm [2], [3]. The BP iterative decoding algorithms for LDPC codes can be naturally described on the Tanner graph (TG) [2] of the code. Depending on the decoding algorithm and the type of messages, certain computations are performed at the bit nodes and the check nodes of the TG, and the algorithm is executed by exchanging messages between check nodes and bit nodes through the edges of the graph, in both directions and iteratively. When there are no cycles in the LDPC parity check matrix, the BP decoding achieves maximum a posterior probability (MAP) decoding [4]. In order to reduce the complexity, the BP decoding algorithm is

Depending on the type of messages and the corresponding operations performed in bit and check nodes, MS are also named max-product and maxsum. Although MS is much simpler to implement, the performance of it is about one decibel inferior to that of BP. On the other hand, for LDPC codes of short length, it is difficult to exclude all the cycles, so the BP decoding is the approximate MAP decoding. Some magnitudes of a posterior LLRs will oscillate owing to cycles [6]. When the oscillation happens, the reliability of each bit changes iteration by iteration, and it prevents the convergence of decoding. Even if we increase the number of decoding iterations, the decoding cannot converge to the LDPC codeword satisfying the LDPC parity check matrix. The oscillation becomes the dominant error factor in the high Eb N0 region [7]. To reduce effects of oscillating ex-LLRs on decoding, for oscillating ex-LLRs, Reference [8] add the previous ex-LLR to the current ex-LLR and regard the result of addition as the new ex-LLR. Since it regarded that ex-LLR as oscillating when the sign of the ex-LLR in the current iteration differs from that in the previous iteration, the range of the new ex-LLR is small, and the convergence of decoding will need large numbers of iterations. In [9], a probabilistic updating schedule algorithm (BP-PU) is also proposed to avoid oscillating of LLR values and improve performance. We can know that it is based on some kinds of LDPC codes which have much difference between the largest and the smallest girths in the TG, but for regular codes of short length, the difference

in Fig. 3, and cycle of length 4 is included by cycles of length 6 and 8. After initialization, every single iteration includes two updating computation. The first is check nodes and bit nodes followed. So messages passed from the bit nodes 1 or 2 return only need 3 iterations, while messages passed from the bit nodes 5, 6 or 7 return need 4 iterations at least. Because cycle of length 4 is composed of bit nodes 3, 4 and check nodes 2, 3, bit node 3 or 4 has return after twice iterations, and then the reliability of check 2 and 3 is reduced, they will affect all the bit nodes connected to them. If we stop updating bit nodes 3 and 4 after once iteration, they will not affect the rest of bit nodes updating. Due to the same cause, bit nodes 1 and 2 will stop updating after twice iterations, and bit nodes 5, 6 and 7 will stop updating after 3 iterations. The rest bit nodes continue updating until all bit nodes stopped, and then restart a new grouped iteration.

B. Select Updating Schedule
Select updating schedule include two kinds of bit node location at least. We can obtain them as follows.
1) Search the cycle distribution of length girth and girth+2 or more.
2) Compute the number and percent of cycles of length girth and girth+2 or more, like Fig. 1 or 2.
3) Select bit node location 1 if the number of cycles of length girth is zero.
4) Select bit node location 2 from the bit node location 1 according to a certain number, which obtained by Step 2.
5) Continue selecting if another cycle distribution available.
Where girth is the code girth (CG).
C. Proposed Decoding Algorithm
To reduce the unreliable messages passed from some bit nodes, we modified the conventional BP bit node process followed.
1) Divide the maximal number of iteration to
Maxiter ( girth 2 + 2) groups, and generate grouped
iteration. 2) Update bit nodes from 1 to N if the number of iteration
is less than girth 2 , where N is the code length. 3) Update bit nodes according to the bit node location 1 at
iteration girth 2 . 4) Update bit nodes according to the bit node location 2 at
iteration girth 2 +1 . 5) If another bit node location available, continue
updating the bit nodes according to it at iteration girth 2 + 2 , otherwise, updating all bit nodes.

6) Restart the next grouped iteration.

IV. SIMULATION RESULTS
Simulations are performed on a (576, 1152) TS-LDPC code and (1225, 1715) MF-LDPC code. The cycle distribution is given in Figs. 1 and 2. In our simulations, BPSK-modulated codewords are transmitted through AWGN channel and the received vectors are decoded simultaneously using three decoding algorithm. One is conventional BP algorithm, others are probabilistic updating (BP-PU) schedule algorithm and select updating (BP-SU) schedule algorithm. The decoding is stopped when a codeword is reached or at the 50th iteration. The results are obtained with at least 1000 bit errors for each recoded point.
For TS code and MF code, we observe that at the SNR 2 and 2.5 respectively, bit node location 1 of two codes are not changed. It is because that MF-code has more cycles of length girth + 2 than TS-code. When number of bit node location 2 is about 400, not only bit node updating is reduced, but also the performance is improved.

V. CONCLUTIONS
We discussed how to reduce complexity and improve performance for short regular LDPC codes. We analyzed the properties of message-passing through cycles and proposed a modified BP algorithm based on select updating schedule.
Our work also shows that BP-PU algorithm is not available for short regular codes which not have a large difference between the maximal and minimal SG. The results of this work confirm that we could improve performance and reduce complexity at the same time for these codes by select updating schedule.
