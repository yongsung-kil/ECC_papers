ABSTRACT

In this paper we proposed a fully-parallel irregular LDPC decoder which uses only registers to store the temporary intrinsic messages. Our decoder adopts a simplified min-sum algorithm to reduce the hardware implementation complexity and area, and due to the factor modification we achieve a negligible performance loss compared with the general min-sum algorithm. Considering reducing the power consumption, we also propose a power-saved strategy according to which the message evolution will halt as the parity-check condition is satisfied. This strategy will save us higher than 50% power under good channel condition. The synthesis result in 0.18ptm CMOS technology shows our decoder for (648,540) irregular LDPC code
achieves high throughput (1 Gbps) with 9.Ons latency.

III. DESIGN FEATURES

In this Section we will present the features of our design. For our
fully-parallel (648,540) irregular LDPC decoder, we apply a
simplified min-sum algorithm with factor modification to reduce the
hardware complexity and area; also we proposed a power-saved X
schedule to relieve the power consumption.

1. Simplified Min-Sum algorithm withfactor modification

Compared to the original BP algorithm, the general MS algorithm (2) has reduced a lot of hardware complexity, but can result a very approximate value. Usually, by the MS algorithm the minimum value
and the second minimum value among all the input P are required to
be computed, then the bit-check message a can output through the check update rule. This results high hardware complexity and large area when we construct fully-parallel decoder. The fully-parallel regular LDPC decoder in Ref. [8] has proposed a simplified min-sum algorithm, which only requires single minimum value of all inputs. This update rule significantly reduces the hardware consumption and achieves a good approximation with full-precision on high-rate quasi Reed-Solomon LDPC code. However, when we apply it on the structured LDPC codes proposed in WLAN (802.1 In) protocol, the
performance loss is still significant.

IV. HARDWARE ARCHITECTURE
This section illustrates the architecture of our proposed irregular LDPC decoder. There are five module included in our decoder.
Initial Input Module will store the original input message X for the first check node operation and each variable node operation. Parity Check Module receives tentative decision value of each node, and
multiply them by check matrix to detect whether the decoding is
successful, then decide the hardware state, as presented in last section. Also this module will output tentative decision value after final iteration. Controller Module takes charge of controlling the hardware operation and collaboration.
