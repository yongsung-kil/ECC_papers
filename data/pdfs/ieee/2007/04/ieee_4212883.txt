Low Complexity LDPC Decoding Techniques with Adaptive Selection of Edges

Kwangho Shin and Jungwoo Lee
School of Electrical Engineering Seoul National University Seoul, Korea
hoyarius@wmspl.snu.ac.kr, junglee@ee.snu.ac.kr

Abstract— In this paper, we propose low complexity LDPC decoding algorithms with variable number of edges. The main idea of this paper is to use the bit nodes selectively instead of using all the connected bit nodes. The proposed algorithms choose the bit node selectively when calculating the check node update equation. First, we study the relationship between the number of bit nodes for updating check node and the performance of the BP decoding algorithm. Two selective algorithms are proposed accordingly. The first algorithm (Algorithm 1) does not use the bit nodes whose values are highly reliable when calculating the check node update equation. In the second algorithm (Algorithm 2), we increase the number of bit nodes to update check node each time we have a decoding failure. These two algorithms have lower complexity with small performance degradation compared to the LLR-BP based decoding algorithm. These algorithms reduce the computational complexity at high SNR more than they do at low SNR.
Keywords- LDPC codes, BP decoding alorithm, egde selection, reduced-complexity decoding
I. INTRODUCTION
Low-density parity check (LDPC) code was first proposed by Gallager in 1961 [1], and rediscovered by MacKay and Neal [2][3]. LDPC codes can achieve good performance when decoded by the BP decoding algorithm, but the computational complexity is rather high. It is practically important to simplify the iterative decoding algorithm to speed-up the computation, and reduce the complexity. However simplified algorithms often cause performance degradation, and many alternative ways have been investigated to overcome the loss. The min-sum algorithm is the simplest method to avoid calculating the hyper tangent function in the belief propagation (BP) algorithm [4]. However the min-sum algorithm can not approach the performance of the BP decoding algorithm. In [4], the normalization and the off-set techniques are introduced, and the performance of them is much better than that of the min-sum algorithm. More details of theses algorithms are discussed in [5][6]. In [7], an algorithm which updates the only unreliable nodes is proposed. In that approach, the complexity is reduced by not updating reliable nodes. In [8], two complexity reduction algorithms are proposed. One is based on sequential decoding. It has the same hardware complexity as the standard BP decoding algorithm, but can finish decoding faster. The other is called the early detection algorithm. II. BACKGROUND
A. Low-Density Parity Check Codes
LDPC codes are specified by the parity check matrix which contains mostly zeros and a small number of ones. LDPC codes can be classified by two groups. If H is sparse and has the constant number of ones in column or row, the codes are called regular LDPC code. If not, they are called irregular codes. It is well known that the irregular LDPC codes can achieve better performance than the regular LDPC codes. LDPC codes can be represented by a Tanner graph [9]. Tanner graph is a bipartite graph which has two different node types. If we have a M × N parity check matrix H, the corresponding Tanner graph has m check nodes and n bit nodes. Fig. 1 is an example for 3× 7 parity check matrix H. The m rows and n columns of H correspond to the m check nodes and n bit nodes, respectively. A check node ci is connected to a bit node bj if the element hij of H is a 1. For example, b1 is connected to c1 because element h11 is a 1. A Tanner graph is useful to understand the decoding process based on the BP decoding algorithm.

At the first iteration, the LLR-BP decoding algorithm uses the only information based on the channel outputs, and assigns L(qij) to L(ci). Each bit node then sends the LLR value to their connected check nodes. Check nodes process the received LLR values and send back a message to connected bit nodes. Each bit node estimates a codeword using the channel information and the messages from the neighbor check nodes. If the product of estimated codeword and the transpose of the parity check matrix is zero, the algorithm terminates because decoding process is successful. If the multiplication value is not zero, the above operations repeat until the iteration number equals the maximum number of iteration or the decoding succeeds. A decoding failure is declared when the number of iterations exceeds the maximum iteration number. III. PROPOSED ALGORITHMS
Let us first discuss motivations for our proposed algorithms. We consider the check node update that sends a message from a check node to a bit node. The message can be obtained by calculating (2). If the multiplication of all the connected bit node messages has minus sign, it means that the bit node which gets this message will be 1 because a minus sign means 1 in the bipolar mapping. The magnitude of each bit node means the reliability. When the bit node probabilities for 0 and 1 are the same, the tangent value of LLR is zero. On the contrary, if the channel is good, LLR values tend to have high magnitude and their tangent values are close to 1. In this case, if the tangent value is nearly one, that LLR value can be considered very reliable, we can ignore this value because it does not influence (2) much. But the sign of the message should not be ignored because the all signs of the bit nodes are needed to determine the outgoing check message. The min-sum algorithm is the extreme example because the minimum bit node among the connected bit nodes is used and the others are regarded as 1. Thus the min-sum algorithm can avoid calculating the hyper tangent functions and the inverse hyper tangent functions, and it leads to the reduction of complexity. But, as stated in the introduction, the performance degradation is severe compared to the standard LLR-BP decoding algorithm.

In our algorithms, we still need to compute the hyper tangent function, but do not use all the connected bit nodes. We propose two different algorithms which use different adaptation strategies.
A. Algorithm 1
The basic idea of algorithm 1 is simple. As previously stated, algorithm 1 does not use the reliable bit nodes which send high LLR values. The easiest way to implement this criterion is to use a threshold and discard the bit nodes whose LLR values are higher than the threshold. If all the connected bit nodes are higher than the threshold, the minimum LLR value will be used to compute (2) as in the min-sum algorithm. But algorithm 1 doesn’t ignore the signs of the discarded bit nodes. Every signs of the neighbor bit nodes is necessary. When the channel impairment is large, that is, the SNR is low, many bit nodes participate in updating the check node because most bit nodes have low LLR values. At high SNR, on the contrary, a large number of bit nodes will be dropped because almost all the bit nodes have high LLR values. The threshold is the most important parameter to determine the performance of this algorithm. This algorithm is similar to the early detection algorithm in [8].
However, Algorithm 1 updates every check nodes and bit nodes at each iteration and chooses the bit nodes to calculate (2) selectively whereas the early detection method does not update the nodes which are considered reliable Algorithm 1 takes the following steps. B. Algorithm 2
The algorithm 2 increases the number of bit node to update the check node at each iteration incrementally. At the first iteration, the only one bit node is used as in the min-sum algorithm. If the parity check fails at first iteration, each check node uses the two smallest bit nodes to update (2) in the next iteration. If parity check failure occurs again, the three smallest bit nodes are used to update (2) at the third iteration. This is repeated until the decoding operation finishes. If it can not decode successfully with the all connected bit nodes are used, the algorithm becomes the same as the standard BP decoding algorithm. The signs of the neighbor bit nodes should not be ignored likewise algorithm 1.
The BP decoding algorithm can find the right codeword with a few iterations at high SNR. Therefore Algorithm 2 is more advantageous to reduce computation complexity at high SNR than at low SNR. The algorithm can be implemented as follows.

It seems that Algorithm 2 outperforms Algorithm 1. But it is not true because Algorithm 1 can behave the same as Algorithm 2 if we set the threshold to 1. At low SNR, it takes more number of iterations, and it is difficult to lower the computational complexity although the BER performance can be as good as the BP decoding algorithm. V. CONCLUSION
In this paper, we studied the relationship between the performance and the number of bit nodes to calculate the check node update equation. We then proposed two algorithms which use the edges selectively to reduce the complexity of the BP decoding algorithm. We showed that both algorithms have lower complexity, and their performance loss is negligible in simulations. It was observed that both algorithms can reduce computational work at high SNR than at low SNR.
