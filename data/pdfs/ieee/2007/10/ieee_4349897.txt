Abstract— Array low density parity check (LDPC) codes are high-rate codes that can achieve good error rate performance in additive white Gaussian noise (AWGN) channels. However, array LDPC codes do not support arbitrary code lengths, because the code length of an array LDPC code with good error rate performance is limited to a multiple of a prime number. This paper proposes Size Compatible (SC)-array LDPC codes; they achieve good error rate performance while supporting arbitrary code lengths. We conduct computer simulations to evaluate the block error rate (BLER) performance of SC-array LDPC codes in AWGN channels. We also evaluate the corresponding performance of SC-array LDPC coded multiband OFDM systems for data transmission rates over 1 Gbps in the UWB multipath channel model CM 3. We show that if the submatrix size is a nonprime number, SC-array LDPC codes achieve better error rate performance than the conventional array LDPC codes in AWGN channels. We also show that the SC-array LDPC codes achieve better error rate performance than the punctured array LDPC codes in AWGN channels. We also show that if the submatrix size is a prime number, SC-array LDPC codes achieve the same error rate performance as conventional array LDPC codes in AWGN channels. Moreover, we show that for multiband OFDM systems with data transmission rates of over 1 Gbps, SC-array LDPC codes achieve better error rate performance than conventional array LDPC codes.
I. INTRODUCTION
Low-density parity-check (LDPC) codes proposed by Gallager in 1962, have attracted much attention given their good performance [1]. An LDPC code is specified by sparse paritycheck matrix H that is represented by a Tanner graph, with that expresses the relationship between codeword bits and parity checks [2]. The iterative sum-product algorithm (SPA), one LDPC decoding algorithm, converges to the optimal solution provided that its Tanner graph is free from short cycles.
One high rate LDPC code, the array LDPC code with code rate of 0.915, achieves good bit error rate (BER) and block error rate (BLER) performances in AWGN channels [3]. Array LDPC code is specified by three parameters j, k, L ( j, k ≤ L). Parameters j, k, and L represent the row and column weights of the LDPC code, and the submatrix size, respectively. To construct an array LDPC code with good error rate performance, the submatrix size L has to be prime number to eliminate the possibility of 4-cycle. The code length of an array LDPC code is a multiple of submatrix size. Thus, it is difficult to construct

an array LDPC code with good error rate performance that supports arbitrary code lengths. Therefore, when we use an array LDPC code in a communication system (e.g. multiband OFDM system), we have to add dummy bits such that the communication system matches the code length of the array LDPC code used. However, since the dummy bits contain no information data, the transmission rate is degraded. One way to realize arbitrary code lengths is puncturing. However, puncturing deteriorates the error rate performance of the mother LDPC code.
In this paper, we propose the Size-Compatible (SC)-array LDPC code; it supports arbitrary code lengths because the submatrix size of the SC-array LDPC code can be a nonprime number. We evaluate the error rate performance of an SC-array LDPC code for the code rates of R = 11/12, 14/15 in an AWGN channel. We also evaluate the error rate performances of an SC-array LDPC coded multiband orthogonal frequency division multiplexing (OFDM) system with 16QAM in an UWB channel model 3 (CM3) at the transmission rates of 1158, 1192 Mbps (code rates of R = 11/12, 14/15, respectively). Note that multiband OFDM is an ultra-wideband (UWB) systems that divides the available spectrum into multiple bands [4][5]. We show that if the submatrix size is a nonprime number, the SC-array LDPC codes achieve better error rate performance than the conventional array LDPC codes and punctured LDPC code in AWGN channels. We also show that if the submatrix size is a prime number, the SC-array LDPC codes achieve the same error rate performance as the conventional array LDPC codes in AWGN channels. We also show that the SC-array LDPC codes achieve better error rate performance than the punctured array LDPC codes in AWGN channels. Moreover, we show that for multiband OFDM systems with data transmission rates of over 1 Gbps, the SC-array LDPC codes achieve better error rate performance than the conventional array LDPC codes.

B. Issue of Array LDPC Code
If the submatrix size is non-prime, many 4-cycles are generated. Note that the 4-cycles make their own information loop back to on themselves, which degrades the performance of LDPC codes since decoding is based on the belief propagation (BP) algorithm.
Figs. 1 and 2 show examples of the pattern diagrams for prime (submatrix size = 11) and non-prime numbers (submatrix size = 12). The vertical list on the pattern diagrams shows the row numbers and the horizontal list shows the column numbers. The entries are the permutation number of the submatrix. The permutation number is decided by its row and column numbers where it is located. The existence of two identical permutation numbers indicates two submatrices with the same construction. For any arbitrary pair of rows, the same permutation submatrix can be present at more than two shared columns due to the cyclic permutations if the submatrix is nonprime; these represent the existence of 4-cycles.

B. LDPC Coded High Data Rate (HDR) Multiband OFDM System
OFDM is one of the UWB systems that divides the available spectrum into multiple bands. We exploit multiband OFDM systems to show the effectiveness of SC-array LDPC codes. We show the HDR multiband OFDM system in Fig. 7. In the transmitter, binary information data are encoded using the LDPC encoder. The coded data are mapped to the Gray16QAM constellation. The complex Gray-16QAM data are then passed through a serial-to-parallel (S/P) converter and modulated by using inverse Fast Fourier Transform (IFFT). The data are spread over both time and frequency domains. The system can hence take full advantage of the time and frequency diversity of the channel.
At the receiver, FFT is performed on the received data. The received data are then converted by the parallel-to-serial (P/S) converter and passed through the Gray-16QAM demapper. Finally, the received data are decoded using the LDPC decoder.
