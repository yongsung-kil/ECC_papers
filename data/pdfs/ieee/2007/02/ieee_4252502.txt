1. INTRODUCTION
Low-density parity-check codes [1] are known to perform close to the Shannon limit as the block length increases. Under a constant phase offset channel, blind and turbo phase estimators provide a good estimate with a phase ambiguity of 2j for the LDPC decoder, where M represents the size of the PSK constellation. Phase ambiguity can be resolved either by having an inner differential encoder, by having a number of pilot symbols or with the expectation-maximization (EM) algorithm [2]. While LDPC with inner differential encoding/decoding results in a Eb/No loss [3], pilot symbols reduce the data rate. On the other hand, EM algorithm provides a good phase ambiguity estimate under constant phase offset conditions. For channels with frequency offset, factor graph approaches are known to give good performance [4]. However they require the use of pilot symbols and more computations. In practical systems, there is usually a residual frequency offset on the received symbols which is small relative to the symbol rate. The LDPC decoder fails to provide a correct decoding under these circumstances. Sub-block decoding [5] was shown to be a good, practical technique to resolve this problem. Here, the codeword is split in to many sub-blocks. A blind phase estimator and/or turbo phase estimator are used to provide phase estimates for each sub-block and the sub-blocks of received symbols are corrected by that estimate. However phase ambiguity needs to be resolved for every sub-block,

3. LOCALIZED DECODING Given a check node with an odd degree, the parity check will be satisified provided there exists an even number of variables with value '1'. If the variables are rotated by 7 radians, then only an odd number of variables will have '1' and hence the parity check will not be satisifed. This fundamental parity
check criteria as shown in Fig. 2 motivates the following argument. In sub-block decoding every sub-block is provided with an ambiguous phase estimate from the blind estimator of w radians. As the ambiguous phase estimate is applied over the entire sub-block, all the variables present within a sub-block are phase ambiguous. Let us assume that one edge from an odd number of received variables present within a sub-block are connected to a particular check node ck, where k is the check node index. Let us call this check node as 'local check node' ak where the term 'local' implies that the variables participating in this check are present within a single sub-block. Assuming a noiseless case, the parity check at the local check node ak will
be satisifed provided the variables connected to the local check node have a correct phase estimate. If inverted, the check will not be satisfied.

5. PROBABILITY OF PAR ERROR
The error probability Pe due to the above-mentioned phase ambiguity resolver depends on the SNR and Cmax-sb. As the algorithm involves the averaging over the number of local check nodes present in a sub-block, Pe due to PAR will be decreased with an increased number of local check nodes.
However we are unable to increase the Cmax_sb beyond a
certain extent for a given sub-block size and hence a trade-off
between the Cmax_sb and the Pe due to the phase ambiguity
resolver has to be achieved. As the local check node degree decreases, Pe also decreases.
This can be seen clearly, when we realize the check node
calculations with the mmn-product approximation [8]. Fig. 4 shows simulation results to illustrate the decrease in Re as the
number of LCN increases at an Eb/No of 2.5 dB with check
node degree 5.

6. SIMULATION RESULTS
For the code used in this simulation, the maximum number of local check nodes available is equal to 211.Simulation results in Fig. 5 show the performance of the LDPC code with the phase ambiguity resolver at low SNR's. They tackle small frequency offsets with minimal loss of around 0.10.3 dB from the coherent performance. For the short LDPC code considered, it is enough to have 1 or 2 sub-blocks for
|fo rs < 0.0003. Increasing the number of local check nodes within a sub-block and decreasing the local check node degree
will aid the decoder to tackle more frequency offset.
7. CONCLUSIONS
This paper presents a new technique using localized check
nodes to improve phase and frequency synchronisation for LDPC codes. The method is computationally efficient and
does not require additional pilot symbols. The approach can be
extended to irregular LDPC codes with local check nodes having an odd degree. This approach is currently being extended to bandwidth efficient constellations with hybrid blind-turbo synchronisation techniques.
