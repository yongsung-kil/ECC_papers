1. lntroduction
The Integrated Interleaving Error Correcting scheme was introduced by M. Hassner et. al. [I]. This scheme allows us to use redundancies of different levels through all data blocks. That is, we can distribute a redundancy of the error correcting capability of a {strong) code over data blocks which are encoded with different redundancies.
Suppose that we use two codes Ci and C2, where the error comcting capability of C2 is stronger than that of CI, and that we use 2 data blocks B1 and E2 for interleaving. If we use a decoding scheme in [ I ] we can correct errors even for the following case: one of B1 and B2 is correctly decoded as a code word in CI,another block cannot be decoded as a code word in C1 but the sum of received words can be decoded as a code word in C2. If we use our decoding scheme described below, there are correctable patterns even if none of B1 and B2 is decoded as a code word in CI.
