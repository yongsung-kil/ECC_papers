A Comparison of Adaptive Belief Propagation and the Best Graph Algorithm for the Decoding of Linear Block Codes
Aditi Kothiyal and Oscar Y. Takeshita Dept. of Electrical and Computer Engg.
The Ohio State University Columbus, OH 43210 USA Email: {kothiyaa, oyt}@ece.osu.edu

Abstract— In this paper, two iterative message passing algorithms based on belief propagation proposed for the decoding of any binary linear block code are compared. The innovation of these algorithms lies in the fact that the structure of the graph of the parity check matrix is adaptively modified to render it suitable to belief propagation decoding. The modification is done such that the least reliable bits received from the channel are leaves of the graph. These algorithms perform favorably when compared with existing hard and iterative soft decision decoding algorithms in terms of error rate while maintaining a complexity polynomial in block length. It was found that while these two algorithms are conceptually similar, their performance in terms of word error rate (as a function of signal-to-noise ratio), decoding time (average number of iterations) and the updated bit reliabilities are very different.
I. INTRODUCTION
In the last decade, there has been a great interest in the study of low density parity check (LDPC) codes and it is now known that these codes can achieve the Shannon-limit error performance for asymptotically long block lengths [1]– [4]. Belief propagation (BP) provides very good performance for the decoding of LDPC codes defined over sparse matrices. However, for codes of practical block lengths BP decoding does not achieve the optimal error performance [5].
Good traditional codes such as BCH and Reed-Solomon (RS) codes are defined in a way that does not readily permit a sparse parity check matrix representation. So BP decoding has not been extensively attempted over this large class of codes. This is because their parity check matrices are of high density and have many cycles, which causes error propagation in BP. Therefore the bit reliabilities do not converge to the a posteriori probabilities which would be the case if the graph were loop-free.
Recently however, an adaptive belief propagation (ABP) algorithm has been proposed in [6] wherein the graph of the code is adaptively modified at each iteration so that the bit reliabilities from BP can converge in the direction of the correct codeword. This decoding algorithm provides an improvement in word error rate (WER) performance over several soft decision decoding algorithms while maintaining a complexity polynomial in the block length. In this paper,

we propose an algorithm called the Best Graph Algorithm (BGA) based on the same idea of modifying the graph of the code at each iteration to make it suitable to BP. The difference between ABP and BGA lies in the method by which the bit reliabilities are updated after the graph has been modified.
The rest of the paper is organized as follows. In Section II, we give a description of each algorithm and highlight the differences in the LLR update equations of the two. In Section III we observe how the differences in the log likelihood ratio (LLR) update equation translate to differences in bit reliability trajectories, WER performance and decoding time (in terms of number of iterations). In Section IV we discuss the reasons for this performance and finally, in Section V we give some concluding remarks.
II. DESCRIPTION OF THE ALGORITHMS
First we describe the similarity of the two algorithms. We denote vectors by bold face capitals and matrices by caligraphic capitals. Below is the first step of each iteration of both algorithms.
Consider a codeword C = [c1, c2, · · · , cN ] of a binary linear block code C(N, K). Assume that the bits are BPSK modulated with 0 → +1 and 1 → −1 and transmitted over an AWGN channel. At the receiver we characterize the reliability of the received vector by its LLR denoted as L = [L1, L2, · · · , LN ].
The algorithm proceeds as follows. The magnitude of the LLR, |L| is first sorted. Let {i1, i2, · · · , iN } denote the indices of the bits corresponding to sorting the LLR in ascending order. Therefore, the it1h bit is the least reliabile (LR) and iNth bit is the most reliabile. We first reduce the i1th column of the parity check matrix of the code, H to the form [100 · · · 0]T by performing row operations. Then we proceed further and attempt to reduce the i2th column to the form [0100 · · · 0]T . If this is not possible, we continue down the list and try to reduce the i3th column to the form [010 · · · 0]T . Thus we proceed down the list {i1, i2, · · · , iN } and reduce columns to be of weight one. Since H is of full-rank, it is possible to reduce exactly (N − K) columns to be of weight 1, though these (N − K) may not be the LR bits.

A. Adaptive Belief Propagation
Given a certain L and H, the process of reducing H can be represented as a function Φ(H, |L|). During the jth iteration, the parity check matrix is reduced to the desired form based on the LLR vector Lj (initially L0 is determined from the channel output). In other words, Hj = Φ(H, |Lj|). In the second step, the sum-product algorithm (SPA) [8] is implemented on the vector Lj to produce extrinsic information Ljext and update the reliabilities to Lj+1. Define Ψ(H, |L|) as a function that represents one iteration of the SPA algorithm which produces extrinsic information Lext with the input LLR, L and the parity check matrix H. In the jth iteration, we begin with the original H and reduce it based on the LLR vector obtained in the previous iteration namely, Lj−1 (as in ABP, at iteration 1, we start with L0 determined from the channel output). BP [3], [4] is then performed on this new matrix. However, there is an important difference between BGA and BP which we describe here. III. DIFFERENCES BETWEEN ABP AND BGA
A. Trajectories
Observing equations (2),(3),(4) and (5) we notice that the BGA algorithm is similar to the conventional BP, modified to include the fact that the graph of the code may change in every iteration. BGA, similar to BP, is designed to update the LLR such that the channel LLR of each bit converges to the a posteriori LLR of the bit as the iterations progress. This would happen if the graph were acyclic [9]. However since the modification applied to H does not ensure that the graph is loop-free (only that error-propagation is restricted), we can not be assured of obtaining the a posteriori LLR of the bits.

Then V Cgpja is saved by the algorithm so that this information can be sent from VN p to CN a at some future iteration k > j when p and a are connected again. Thus we are not losing any information available in the graph. Our simulation results suggest that this fact improves the performance and helps in convergence.
However there is a disadvantage of saving the information in the graph, which is the creation of “pseudo loops”. We illustrate this idea with an example. Consider a section of a graph of a code at iterations 1 and 2 of BGA shown in Figure 9 (VN are indicated by circles and CN by squares). The corresponding matrices for this section of the graph are given in Table I. It is easy to see that after iterations 1 and 2, a “pseudo-loop” is created between CN a, b and VN p, q, r. So, if r, a and p are connected in the same configuration in some iteration in the future, the message passed from r to a and then to p is V Cgr2a = L0r + L0q + Lp0 and this amounts to positive feedback and error propagation which may render the graph unstable. Therefore, we have to investigate ways to break up these “pseudo-loops”.
A probabilistic bit flipping algorithm has been proposed in [10] wherein if the number of unsatisfied check sums that the bit is connected to suggest that the bit should be flipped, it is flipped with a probability p ≤ 1. In a graph with cycles, it has been shown that this algorithm improves the performance over BP. Similarly in BGA, in order to prevent error propagation we have to probabilistically choose whether to use L0p or V Cgpl a, l ≤ (j − 1) at some iteration j. This probability will depend on the probability of creating a “pseudo-loop” by sending V Cgpl a.
V. CONCLUSIONS
ABP and BGA are both iterative soft decision decoding algorithms for linear block codes based on the idea of modifying the graph of a code and then updating the reliabilities at each iteration. The respective LLR update steps differ and this is the reason for a vast difference in the WER performance, decoding complexity and LLR trajectories. While BGA has better WER performance, ABP has the advantage of monotonic and faster

convergence. Further, ABP with ”modification 2” of [6] seems

to be a good option to obtain improved WER performance.

Therefore it is worthwhile to consider a modification to BGA

similar to “modification 2” of ABP which may yield further

WER and decoding complexity improvement.
