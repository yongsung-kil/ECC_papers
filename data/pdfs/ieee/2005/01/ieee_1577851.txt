Design of Low Density Generator Matrix Codes for Continuous Phase Modulation

Abstract— We investigate the low density generator matrix (LDGM) codes for continuous phase modulation (CPM). The nonsystematic version of LDGM codes is used in the scheme. The overall system has linear encoding complexity due to the low complexity of the LDGM codes. A property of the check node degree of the LDGM code is shown. We use the EXIT chart/function to optimize the LDGM codes. The EXIT function of CPM with a fixed SNR (Signal-to-Noise-Ratio) is shown. We derive the union bound to analyze the error floor performance. Design approaches for lowering error floors are suggested from the analysis process. Numerical results show that this scheme converges earlier (lower SNR) than best found serially concatenated CPM (SCCPM) for iterative decoding while maintaining comparable error floors.

I. INTRODUCTION
Continuous phase modulation (CPM) is an excellent digital modulation scheme which has constant-envelope, and is bandwidth and energy efficient [1]. In order to obtain a further improvement in energy efficiency, CPM can be combined with an external code. With the same principle as a serially concatenated code (SCC) [3], serially concatenated CPM (SCCPM) with iterative decoding was proposed and showed a significant improvement in bit error rate (BER) performance [5], [6].
On the other hand, as a special class of low density parity check (LDPC) codes [4] or as a generalization of concatenated single parity check (SPC) codes [7], low-density generator matrix (LDGM) codes have attracted much research interest [10], [11]. One of the main advantages of LDGM codes over standard LDPC codes is that the encoding complexity is linear in the code length.
Here we will investigate LDGM codes for CPM. Nonsystematic LDGM codes are used as the external code for CPM. Since CPM also has encoding complexity linear in the code length [1], the proposed system has linear encoding complexity.
II. SYSTEM DESCRIPTION
The schematic of LDGM codes for CPM is shown in Fig. 1. On the transmitter side, the outer LDGM code consists of repetition encoders with different degrees as the variable node (VN), a random edge interleaver (length T ) and SPC encoders as the check node (CN). The SPC encoder only outputs the

parity bits. The output codewords from SPC codes are serially input to CPM. If the alphabet size of CPM is larger than 2, a mapper is used to transfer bits into symbols. The LDGM code used here is nonsystematic. The main reason for this is that the systematic bits are unavailable at the receiver side due to the built-in coding property of CPM [1]. Moreover, as we will show later, the decoding schedule of the nonsystematic code is simpler when it is combined with CPM. The interleaver between the CN and the VN is produced randomly but the duplicated edge (two edges connecting the same CN and VN) is avoided. The transmitted signals are disturbed by additive white Gaussian noise (AWGN). We note that there is some similarity between the structures of the proposed scheme and the concatenated Zigzag codes [9] if we see the differential phase modulation process of CPM [1] as a Zigzag encoding process.

III. CHECK NODE DEGREE PROPERTY

From (2), we can see that for the CNs, a necessary condition for an edge to produce a meaningful (non-zero) output Lvalue is that all other edges connecting to this node have non-zero input L−values. Yet since we use nonsystematic LDGM codes, all the channel information in the VNs comes from the CNs. Moreover, since we have to initialize the Lvalues for all edges to zeros before decoding (we assume that the bit values have equal probabilities for being 0 and 1 ), part of the edges connecting to the CNs that has degree (the number of input edges) larger than 1 can not be decoded during the first iteration. Decoding has to rely on these degree1 CNs until nonzero L-values propagate to other CNs from the VNs connecting to the degree-1 CNs. This approach is called doping [12]. The appropriate interleaver rule must ensure nonzero L-values propagating to every CN. B. Design Approaches
Based on the union bound analysis process, we suggest the following design approaches for lowering the error floor.
1) From the bound analysis, the error events from the lowest degree repetition codes will dominate the union bound. Thus, its ratio should not be large.
2) Since the error event of low degree VNs have dominated affects on the system error floor. Their connections (through a common CN) will lead to a serious system performance degrading. Thus, we should use the high degree VN as the neighbor (VNs connecting to the same CN) of the low degree VN, i.e., avoid the low degree VNs connecting to the same CN.
3) For MSK (or other CPM with a modulation index 1/2), the odd input weight (odd input 1s) is not an effective error event. Thus, we should use more odd degree variable nodes. Especially, we should try to use the degree profile that has minimum odd degree VN.
