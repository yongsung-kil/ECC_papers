LDPC Code for Reduced Routing Decoder

Euncheol Kim
Electrical Engineering Texas A&M University Bryan, TX 77801, U.S.A.
eckim@ee.tamu.edu

Gwan S. Choi
Electrical Engineering Texas A&M University College Station, TX 77840, U.S.A
gchoi@ee.tamu.edu

Abstract—A design approach that reduces the routing complexity in a VLSI implementation of Low-Density ParityCheck (LDPC) decoder is presented. An LDPC code is a linear-block code for forward error correction, attributed by a sparse parity-check matrix. Iterative decoding of this code is shown to yield Bit Error Rate (BER) performance approaching Shannon Limit. However, implementation of decoder for this code is difficult due to the routing requirements of its massive number of data-flow structures in decoding logic. We present a routing approach for a parallel LDPC decoder implementation by 1) analyzing the physical routability limitations and 2) designing the code parameters to limit the interconnect lengths to a bounded region. The approach does not compromise the BER performance, and yet achieves a much higher throughput resulting from significantly reduced wires lengths.
I. INTRODUCTION
With the current trend toward deep sub-micron devices and higher integration complexity, interconnect issues are becoming increasingly significant in determining system performance, power, reliability, and cost. To ensure timing closure of designs, interconnect issues must be considered as early as possible in the design flow. Especially for the VLSI designs that consist of dense interconnect networks, the need of early consideration of routability is imperative. LDPC decoder implementation is such an example that brings forth complex routing problems that are difficult to address using existing CAD tools. The routing problems in the LDPC decoder arise mainly from the large amount of buses that connect computational function units and memories in the LDPC decoder.
Several research studies have been presented about the interconnect issues in LDPC decoder implementation. A prototype parallel LDPC decoder with a code-length of 1024-b is implemented in [1]. A special placement and routing software is developed to insert and place buffers by enforcing Manhattan routing geometry. With increase of code-length, this approach poses difficult implementation challenges due to the complex interconnects resulting from the random connectivity between the functional circuit units. A partly parallel decoder [8][9][10] implementation tries to reduce the routing complexity by considering design of

structured LDPC codes having regularity in its physical features. The routing complexity is reduced and the scalability is improved at the cost of reducing the degree of parallelism. However, implementation solutions for fully parallel LDPC decoder designs that promise a very high throughput have not yet emerged due to the difficulty in addressing the routing requirements of its massive number of data-flow structures.
This paper proposes a novel approach for resolving the routing problem specific to parallel LDPC decoder design by introducing the routing issues during the architectural planning. First, a methodology specific to the code construction and physical layout is developed. Then tradeoffs are made between the identifying attributes to yield manageable locality in routing. The result is a streamlined approach for synthesizing VLSI circuits that optimizes circuit placement and routing. The proposed routing approach offers much needed performance improvement solution at the layout-design level.
The paper is organized as follows: In Section II, the background of LDPC code is described briefly together with decoding algorithm. In Section III, various LDPC decoder architectures are explored to demonstrate the code construction procedure for check-node partitioned fully parallel implementation. The resulting parity-check matrix is analyzed with different design constraints. Bit error rate simulation is performed and the results are shown. Section IV is conclusion.
II. BACKGROUND This section describes the background and the physical design issues associated with LDPC decoder implementation. We explore the close relationship between the structure of parity check matrix and the physical layout. LDPC codes belong to a class of linear error-correcting block codes. As the name suggests the codes are defined by sparse parity check matrix that consists mostly of 0’s. The coding gain of this code is known to approach very close to that of Shannon Limit as the code length increases [2][3]. First introduced by Gallager in 1962 [4], the parity check H

The blocklength of this code is n and it is same as the number of columns in the H matrix. Suppose that the number of data bits before the channel encoding is k, then the number of rows of this H matrix is n – k. The rate of this code is defined as k/n = 1 – j / ρ. The set of codewords, x ∈ C, consists of all one-dimensional row vectors that spans the null space of a parity check H matrix; H · xT = 0, ∀ x ∈ C.
LDPC codes can also be represented using a bipartite graph shown in Fig.1. One set of nodes (check nodes) corresponds to parity check constraints and the other set of nodes (variable nodes) corresponds to the data symbols.
The decoding of LDPC codes is based on the iterative message-passing algorithm, also known as belief propagation algorithm [3][4][5]. It is a probabilistic decoding which iteratively computes extrinsic probability values associated with each variable node using disjoint parity-check equations that the variable participates in.
The decoding procedure at each iteration stage includes two distinct probability computation steps. One is a Check Node Update (CNU) and the other is Variable node Update (VNU). In the CNU, each row of the parity matrix is checked to verify that parity check constraints are satisfied. In the VNU, from the extrinsic probabilities computed from the CNU step, the probability will be updated by sum up the other probabilities from the rest of the rows and the a priori probabilities from the channel output.
Even though this efficient iterative decoding algorithm yields very good coding gain like that of turbo codes [3][6], practical implementation and use of LDPC codes has been limited due to the excessive hardware resource requirements and complex routing problems.
There exist several implementation approaches for LDPC decoder. A serial architecture is presented in [7] where one bit of information is processed at a time. The serial architecture benefits from lower hardware resource requirement since each bit processing reuses the same hardware. However, the throughput of this architecture is limited since a number of iterations are necessary to decode a

In contrast to the serial architecture, a fully parallel architecture presented in [1] processes a block of code at a time. Direct implementation of this structure is to construct hardware components by instantiating variable nodes and check nodes in the graph with appropriate circuit components. Although mapping of the circuit components to the nodes may be straightforward, physical implementation is still difficult because of the density and randomized connectivity between the nodes.
III. PARITY MATRIX AND PHYSICAL DESIGN In this section, we describe the properties of parity check H matrix mentioned earlier. The matrix is analyzed with respect to the connectivity issues that arise when an optimized routing solution at the layout-level implementation is sought. We propose a scheme wherein the bipartite graph is partitioned and the resulting LDPC code is scrutinized through simulation to study the impact of BER performance.
A. Partitioning Bipartite Graph The objective of this partitioning is to simplify the
connectivity requirements between the check nodes and variable nodes in the graph. A design approach that explores this problem is presented in [1]. The problem is again pointed out in [8] that the lengths of interconnect wires and the area of the decoder increase proportional to n1/2, when a layout is planned with uniform distribution of circuit components.
The bipartite graph can directly be implemented into hardware decoder by replacing variable and check nodes with corresponding circuit components. The interconnects between the functional circuit units can be mapped directly form the edges between the nodes in the bipartite graph. This is referred to as direct implementation of LDPC decoder. The direct layout implementation is typically preferred since the native graph structure is already a highly regular form and the timing of the parallel decoder architecture can easily be balanced in such setting.
The complexity of direct implementation increases as the block size of the code increases. The number of circuit components and interconnections is linearly proportional to

Furthermore, the randomized code construction exacerbates the routing problem. With arbitrary position of ‘1’s in the parity matrix, the resulting layout architecture must provide flexible wiring solution between CNUs and VNUs.
We present a novel architecture that confines the interconnect wire length within a bounded region regardless the block size. In the parity check matrix, the position of ‘1’s determines the connectivity between CNUs and VNUs. Subsequently clustered positioning of ones can lead to more localized routing solution in the decoder layout. However, localized ‘1’s in a matrix usually tends to create many short cycles and decreases code performance. The key observation made in this paper is the fact that there exists a conflicting H matrix configuration issues: Randomization of placement of 1’s in the matrix increases the performance. Localization of 1’s in the matrix can lead to a partitioned layout where connectivity between CNUs and VNUs are limited to smaller regions in the layout.
Fig. 2 illustrates matrix that has localized ones. Part (a) is the matrix and part (b) is the resulting bipartite graph and part (c) is the physical layout. In order to analyze the effect of localizing ‘1’s in a matrix, a metric “Overlap Length (OL)” is introduced. OL is defined as the number of columns in the parity check matrix that has ‘1’s positioned in more than one subgroup of rows. In Fig. 2(a), rows are divided into two subgroups of C1, C2 and columns into three subgroups of Va, Vb, Vc. The shaded area in the matrix denotes the area that can be filled with ‘1’s. The blank area is filled with only ‘0’s. With the constraints imposed on this parity matrix, the locations of ‘1’s are localized. All the 1’s in the column subgroup Va only appears at the row C1. The 1’s in the column subgroup Vb appears both at C1 and C2. Vc column subgroup has entry of 1’s only at C2. Fig. 2(b) shows the bipartite graph obtained from the parity matrix Fig. 2(a). The graph shows that only the variable nodes in Vb have edges from both C1 and C2. In Fig. 2(c), an example of physical layout is shown that can easily be obtained from the bipartite graph. B. Simulation and Analysis In the previous section we have introduced OL, which
constraints the construction of the parity check matrix H for localized routing. To examine the effect of OL values on the coding gain, BER performance of a suite of codes is investigated through simulation. This suite of codes consists of 9 sets of codes. 12 different codes are generated for each set and all the codes in the same set have same OL values.

If the matrix is divided into more subparts, most of the 1’s will be positioned along the diagonal area of the parity matrix. Of course, in order to maintain the required code performance, the OL values should be carefully designed and empirically validated.
IV. CONCLUSION An LDPC code construction approach is presented to obtain simplified routing architecture. The approach uses code design to minimize the wire routing distance in a parallel design, and thereby reducing the average parasitic loading associated with interconnects in the layout. The effect of partitioning geometry in the parity matrix is analyzed through the BER simulation. With block size of 2040, the connectivity between the partitioned sub-matrices can go down to 0.2 while achieving BER performance comparable to that of direct implementation. The worst-case routing distance is reduced by a factor of 1/2. This approach

can be extended further to achieve hierarchically localized routing architecture. The impact of this approach is that a layout implementation becomes tractable for a very long code-length decoder with high throughput.
