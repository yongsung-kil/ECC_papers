Abstract— Most structured codes use the group-and-permute design approach in which rows and columns are divided into groups of the same size. Row/column groups and individual rowcolumn are connected by some permutation. A general groupand-permute decoder requires an interconnection network for group-to-group communication. In this paper we suggest the use of self-routed Benes networks based on the communication pattern of LDPC codes. Bipartite edge coloring is used to schedule messages such that there are no switch network output conflicts. The looping routing algorithm for Benes networks is used to compute switching codes. Both scheduling information and switching codes are pre-computed and stored for each design code since they do not change in the lifetime of the code.

I. INTRODUCTION
Group-and-permute structured Low-Density ParityCheck(LDPC) codes are the most implemented LDPC codes in literature because of their reduced communication complexity. Group-and-permute codes are constructed by dividing rows and columns into groups of the same size. Groups are then interconnected randomly or by some predetermined permutation. Each row group is connected to k column groups and each column group is connected to j row groups, where k,j are row and column weights respectively. Since the group-sizes are the same the connections ensures that columns/rows in a group do not share the same row/column. This property simplifies addressing and more importantly it ensures that messages for rows/columns in a group are not stored in the same memory locations, hence avoiding memory conflicts. The grouping also simplifies communication and reduces cost as there are less connections to be made compared to when individual columns/rows are connected. Figure 1 shows a random connection of column and row groups of a code with 8 row and 16 column groups. The edge connections are shown in fig. 2 with column weight of 3 and row weight of 6.
Group-and-permute codes could be designed in many different ways depending on the methods of grouping and permutation. Examples of group-and-permute code designs and their implementations are found in [1][2][3][4][5][6][7].

It is therefore important to have good permutations that give larger girths. However, they also have to be easily implementable. Most designs use circular/shift permutations to interconnect rows/coumns between groups. In some designs the number of row and column groups is fixed to j and k. This eliminates the need to find an optimum group-to-group interconnection. However, in [4] it is reported that fixing the number of groups to j and k limits the maximum girth attainable with shifts. Circular permutations have a maximum girth of 12[8].
In most of the group-and-permute designs examples, the implementations were based on the permutations used. That is, the addressing, interconnection switches, and memory designs reflect the code design. We have observed that groupand-permute codes have the same structure and therefore could be executed by a generic architecture. In [6] and [7] architectures for group-and-permute codes were presented. Though the code design appears to be different from others it has the group-and-permute structure. In [6] a hardwired group-to-group interconnect resembling fig.2 structure was implemented for the code design. Though the row/column group permutation could be changed, the group-to-group communication and the column/row weights are fixed. In [7] a vector approach is suggested, where each group is mapped onto a vector slice of processing elements. The second architecture processes groups of nodes in parallel and has flexibility in group interconnections. However, a flexible switching network is needed for vector slice communication. The flexibility of the switching network allows the processor to run a wide variety of code for different applications. This is important especially in fast changing application requirements such as in wireless communications. The authors in [7] leave the choice of the switching network to the decoder designer. In this paper we suggest the use of self-routed Benes network for LDPC decoders based on the LDPC communication pattern. This paper is organized as follows. Section II looks at group-and-permute LDPC communication. In section III we look at Benes networks and their adaptation to LDPC communcation and finally section IV concludes.
II. GROUP-AND-PERMUTE LDPC CODES
COMMUNICATION
LDPC decoders use multiprocessing elements(bit nodes and check nodes) to speedup decoding. They perform singleinstruction multiple-data(SIMD) type computations. Computations are done iteratively with bit and check nodes(fig. 2) exchanging messages between each iteration. The communication structure between processing nodes therefore affects the flexibility in the design of the code. Restricting the flexibility of the network will restrict the code design and probably adversely its performance.We would like to design a flexible network that allows all possible node permutations. In [6]

simulated annealing was used to find the best group-to-group interconnection for a given node permutation. Full flexibility in the switching network will allow the designer to search for the best group-to-group interconnection for a given node permutation. Since nodes operate in SIMD manner all the messages are produced at the same time from one set of nodes. The number of nodes will be larger than the network capacity and several nodes have common destinations as can be seen in fig. 2. Each bit node produces the number of destinations equal to its column weight(j) and each check node produces the number of destinations equal to row weight(k). The number of bit node messages is j ∗ n whereas the number of check node destinations is m, where n and m is the number column and row groups respectively. Similarly the number of check node messages is k ∗m which is greater than the number of bit node destinations (n). In fig.1 a code is divided into 8 row and 16 column groups. The groups are then randomly connected as shown. Assuming messages from one node to another are send in one register, the number of messages produced at a time is 48, yet there are only 16 or 8 destinations ranging from 0 to 15 or 0 to 7. Therefore some messages must be buffered before transmission. To avoid output collision, messages going to the same destination must be scheduled at different times. Since the code is a bipartite graph, an edge coloring algorithm is used to divide the messages into k groups such that messages in each group do not share a destination. A bipartite graph could be edge-colored using the number of colors equal to the highest edge degree, which is the highest check node degree in LDPC codes[9]. Since the group connections do not change through the lifetime of the code, the scheduling needs to be computed once and stored.
III. INTERCONNECTION NETWORK FOR LDPC
COMMUNICATION
Interconnection networks provide communication between processors or processors and memory in multiprocessors. Network performance significantly affects the overall cost and performance of the multiprocessor because processors may spend more time communicating than doing computations. It is therefore important to choice a network that best suits the applications run on the multiprocessor. Ideally, a completely connected network of processors could execute with a minimum time delay and communications overhead. However, this is considered impractical, since a completely connected network of N nodes requires O(N 2) connections, rendering the structure inefficient and expensive for large N. Multistage networks reduce the number of cross-points by increasing the number of stages. However, the increase in the number of stages increases delay and complexity in routing. There is a wide variety of multistage interconnection networks varying on topology, routing, switching strategies, flow control, delay and cost. Multistage networks include banyan and Benes networks. Banyan networks have efficient routing control but are blocking. The probability of blocking in banyan networks could be reduced by buffering techniques and provid-

The following is a description of the Benes network and how it can be applied to group-and-permute LDPC codes communication.
A. Benes Network
The N-dimensional Benes network BN consists of two back-to-back butterflies as shown in fig. 3. It is the least costly rearrangeable network with O(N log2N ) 2x2 switches. An important feature of rearrangeable networks is that they route all possible input-output permutations by rearranging the switches settings of the network. If all permutations are known before scheduling all the routes are computed before transmission. There are N/2 possible paths connecting a pair of nodes in a N xN Benes network. Each of the N/2 possible paths passes through different 2x2 switching elements in the center stages. This makes Benes networks fault-tolerant. Routing in Benes network could be done using the looping algorithm [10]. To realize a permutation (inputi, outputi, i = 1...N ) in an N xN Benes network, for each connection (inputi, outputi) determine weather it goes through the upper or lower N/2 Benes subnetwork. Repeat the process recursively for each of the switching and subnetworks. The looping algorithm takes O(N logN ) steps to set all the switches or O(log2N ) on a parallel machine [11].

Various permutation methods are discussed in the examples cited in section I. The number of row groups is limited to S = N/2, which is equal to the number of check nodes. The number of column groups is assumed to go up to 2S, hence there are 2S bit vector slices (PEs in fig.4). This is based on the fact that most codes have a rate of at least 1/2. However, the number of bit nodes/columns groups can be larger than 2S without changing the architecture or functionality of the processor. The size of the switching network is S = N/2. This is because the number of messages with distinct destinations in a group is S, the number of check nodes and row groups. Therefore, having a network larger than S will be a wasteful. Since the number of bit vector nodes or column groups is larger than S, bit node messages need to be multiplexed/demultiplexed to/from FIFO buffers. Messages from processing nodes are scheduled using the bipartite edge coloring algorithm to avoid output conflicts. The switching codes enable self-routing where a circuit-switched connection can be established by the hardware as it propagates through the network, based on the routing bits. The routing bits are part of the message. The switching elements must have a logic to determine the next direction of path based on the ith bit, where i is the stage number. The input is routed to the upper output if the bit is 0 and if its 1 it is routed to the lower output line. A conflict occurs if the control bits for both the top and bottom inputs of any switch are identical. Self-routing requires less complex routing control. The path delay is the circuit delays of all the switches on the path. Messages could be pipelined from inputs to output to further reduce the delay.
IV. CONCLUSIONS
Most structured codes fall under the group-and-permute family of codes. A generic vector architecture has been suggested for group-and-permute codes. We suggest a selfrouted Benes network for communication between vector slices. Because of the communication pattern of the code does not change in its life time the scheduling and switching codes information does not change. This allows fast scheduling and transmission of data across the network. The Benes network costs more than a banyan network, however, the overall cost of Benes is less compared to what it will cost for a non-blocking banyan type network and it has less delay(number of stages).
