Partially-Parallel LDPC Decoder Based on High-Efficiency Message-Passing Algorithm
Kazunori Shimizu†, Tatsuyuki Ishikawa†, Nozomu Togawa††, Takeshi Ikenaga† and Satoshi Goto† † Graduate School of Information, Production and Systems, Waseda University †† Dept. of Computer Science, Waseda University 2–7 Hibikino, Wakamatsuku, Kitakyushu-shi, 808–0135 Japan shimizu@aoni.waseda.jp

Abstract
This paper proposes a partially-parallel LDPC decoder based on a high-efficiency message-passing algorithm. Our proposed partially-parallel LDPC decoder performs the column operations for bit nodes in conjunction with the row operations for check nodes. Bit functional unit with pipeline architecture in our LDPC decoder allows us to perform column operations for every bit node connected to each of check nodes which are updated by the row operations in parallel. Our proposed LDPC decoder improves the timing when the column operations are performed, accordingly it improves the message-passing efficiency within the limited number of iterations for decoding. We implemented the proposed partially-parallel LDPC decoder on an FPGA, and simulated its decoding performance. Practical simulation shows that our proposed LDPC decoder reduces the number of iterations for decoding, and it improves the bit error performance with a small hardware overhead.
1 Introduction
Low-Density Parity-Check (LDPC) code has been proposed by Gallager in the early 1960’s, and it has been rediscovered by MacKay and Neal in 1996. LDPC codes have been demonstrated to achieve information rates very close to the Shannon limit, and inherent parallelism of the message-passing algorithm for LDPC codes makes it very suitable for hardware implementation [1], [2].
In the last few years some work has been done on designing architecture for LDPC decoding in Refs. [5]–[9]. All of the LDPC decoders are composed of check functional unit (CFU) for row operations and Bit functional unit (BFU) for column operations. They are connected with each other according to the Tanner graph which is composed of bit nodes and check nodes. The message-passing algorithm

exchanges the messages between the check nodes and bit nodes by performing the row operations and column operations iteratively.
The LDPC decoders can be implemented by using the following methods: 1) All of the intermediate messages are quantized, and the messages are obtained from fixed-point computations in the row and column operations. 2) MinSum algorithm which is an approximation of the row operation can be applied to reduce the hardware cost for CFUs. 3) Particular LDPC parity check matrix partitioning into subblocks for the partially-parallel LDPC decoder should be structured in order to enable re-usability of the CFUs and BFUs. The methods 1) and 2) reduce the computational accuracy of the LDPC decoding. The method 3) restricts the compatibility of optimal LDPC codes constructed more randomly [3]. Accordingly, the LDPC decoder using these methods degrades the convergence time of the iterative decoding and the bit error performance of LDPC codes.
From this point of view, we propose a partially-parallel LDPC decoder based on a high-efficiency message-passing algorithm in order to improve the LDPC decoding performance, which is characterized as: (i) Our partiallyparallel LDPC decoder performs the column operations for bit nodes in conjunction with the row operations for check nodes. (ii) Our BFU with pipeline architecture and improved message-passing algorithm enable the decoder to perform column operations for every bit node connected to each of check nodes which are updated by the row operations in parallel.
Our proposed LDPC decoder improves the timing when the column operations are performed, and it increases the number of message-passings from the bit nodes to check nodes within the limited number of iterations for decoding. We implemented the proposed partially-parallel LDPC decoder on an FPGA, and the bit error performance of the decoder was simulated. 2 Low-Density Parity-Check Codes

The LDPC codes can be defined by an M × N parity-

check matrix H, where M represents the number of LDPC

check bits and N represents the number of LDPC codeword

bits. The parity check matrix for LDPC codes is very sparse

which means that the density of ones in the matrix is very

low. The parity check matrix is represented by a bipartite

graph called Tanner graph as shown in Fig.1. There are two

types of nodes in the Tanner graph, called bit nodes and

check nodes. Each of check nodes cm, m = 1, · · · , M is connected to bit node bn, n = 1, · · · , N , where elements of each row in the matrix H are one.

LDPC codes can be decoded iteratively using message-

passing algorithm [4]. Each iteration of message-passing al-

gorithm is composed of two phases. Phase 1 called row op-

eration updates messages in all check nodes, and sends the

messages (αmn) to bit nodes which are connected to each of check nodes. Phase 2 called column operation updates

messages in all bit nodes, and sends the messages (βmn) to check nodes which are connected to each of bit nodes.

3 Partially-Parallel LDPC Decoder Based on High-Efficiency Message-Passing Algorithm
In this section, we propose a novel high-efficiency message-passing algorithm and its hardware architecture for the partially-parallel LDPC decoder.
3.1 High-Efficiency Message-Passing Algorithm
The message-passing algorithm for the partially-parallel LDPC decoder proposed in Refs. [7]–[9] performs the Phase 1 (row operations) and the Phase 2 (column operations) partially in parallel. For the partially-parallel LDPC decoder, the parity check matrix has to be structured in or-

The matrix is composed of 3 × 6 = 18 subblocks, and each sub-block is b × b square matrix. The b × b square matrix is defined by shifting each row of the identity matrix Ib×b to the right. Therefore, the b × b square matrix has only a one in each row. The diagonal line in each sub-block in Fig.3 represents a one in the sub-block. The parity check matrix which is composed of wc × wr subblocks defines a (wc, wr)-regular LDPC code. The length of the (wc, wr)-regular LDPC codeword is N = wr ×b, and the length of the parity bits is M = wc × b. The partiallyparallel LDPC decoder defined by the above parity check matrix has wc CFUs to perform wc row operations corresponding to wc row blocks in parallel, and it has also wr BFUs to perform wr column operations corresponding to wr column blocks in parallel. In each sub-block, each of b row and column operations are performed sequentially.
The schedule of the message-passing algorithm as shown in Refs. [7]–[9] performs the wc row operations starting from the top of each row block, and they also perform the wr column operations starting from the left of each column block. Accordingly, the wc row operations update the messages α, where the elements of each row block in the parity check matrix is one, and the wr column operations update the messages β, where the elements of each column block in the parity check matrix is one. The decoder performs the row and column operations independently. Accordingly, the timing when the column operation is performed with the latest messages α updated by the row operation is different among sub-blocks in the parity check matrix. This is because the shift value of identity matrix Ib×b for each subblock is different from that for the other sub-blocks.
From this point of view, we propose a high-efficiency message-passing algorithm improving the timing when each of column operations is performed with the latest mes-

sages α updated by the row operations. The high-efficiency message-passing algorithm is characterized as follows:
(i) The column operations for the bit nodes are performed in conjunction with the row operations for the check nodes.
(ii) The column operations are performed for every bit node connected to each of check nodes which are updated by the row operations in parallel.
This means that after the wc row operations are performed in parallel, our message-passing algorithm performs wc × wr column operations for every bit node connected to each of check nodes which are performed by the row operations. For example, Our algorithm for a (3,6)-regular LDPC codeword is shown in Fig.4.
The number of row operations by our improved message-passing algorithm is same as that of row operations by the message-passing algorithm shown in Refs. [7]– [9]. However, the number of column operations by our message-passing algorithm increases wc times compared to that of column operations by the message-passing algorithm shown in Refs. [7]–[9]. In addition, the wc × wr column operations are performed immediately after the wc row operations. Thus, the wc × wr column operations can be performed with the latest messages α as soon as possible after the wc row operations. Accordingly, our message-passing algorithm improves the efficiency of message-passings, which are performed by the row operations and column operations.
3.2 Hardware Architecture
Main modules of the partially-parallel LDPC decoder are row operation modules and column operation modules. Our row operation module and column operation module are shown in Fig. 5 and Fig. 6 respectively. We design the partially-parallel LDPC decoder which is composed of three row operation modules and six column operation modules for a (3, 6)-regular LDPC codeword according to the parity check matrix shown in Fig.3.

Each of row operation modules has the six memory banks for messages β which are updated by the six column operations. Each of memory banks for messages β is composed of b words when each of sub-blocks in the parity check matrix is b × b square matrix. Each background memory banks in the row operation module stores the column addresses corresponding to the positions of one in each sub-block.

3.2.2 Column Operation Module
Each of our column operation modules receives three sets of the message α and corresponding column address from the three row operation modules. The addressing unit in the column operation module stores three column addresses, and the column operation module performs three column operations with Eq.(2) for the corresponding column addresses sequentially. In the column operation module shown in Fig.6, the BFU outputs three messages β. Three messages β are obtained from the following three sets of additions, and the BFU in our column operation module performs three sets of additions in parallel.

Each of column operation modules has the three memory banks for the messages α which are updated by the three row operations. The background memory banks is for storing the row addresses corresponding to the positions of one in each sub-block.
Fig.7 shows the timing diagram of the row and column operation. The upper half of Fig.7 shows that our row operation module computes |β| and determine the first and second minimum values |β| in parallel by the pipeline architecture. In addition, the i1, i2, i3-th row operations are performed in parallel. In order to achieve high operation frequency of the decoder, the CFU in the row operation module compares the |β| once in each cycle. Therefore, it takes 7 clock cycles to determine the first and second minimum values. Accordingly, our row operation module with the pipeline architecture takes 10 clock cycles to perform a single row operation as shown in Fig.7. The lower half of Fig.7 shows that each of the column operation modules performs three column operations in conjunction with the row operations. The BFU with the pipeline architecture takes 6 clock cycles in total to perform three column operations. The number of the clock cycles for three column operations is less than that for a single row operation. Therefore, our partially-parallel LDPC decoder does not degrade the decoding throughput when the row operations and column operations are performed concurrently. 3.3 Implementation Results
We designed the partially-parallel LDPC decoder based on a high-efficiency message-passing algorithm. In our partially-parallel LDPC decoder, we apply the parity check matrix shown in Fig.3 to the decoder, and each of the subblocks is defined by 256 × 256 square matrix. Thus, our partially-parallel LDPC decoder can operate for a (3, 6)regular LDPC codeword whose code length is up to 256 × 6 = 1536 [bits]. All of the intermediate messages are quantized, since the messages are obtained from fixed-point computations in the row operations and column operations. We defined the quantization bits to be 8 bits which are divided into one sign bit, four integer bits and three fractional bits. Our partially-parallel LDPC decoder is composed of the three row operation modules, six column operation modules, output and parity check module, memory module for log likelihood ratio (LLR) values λ, and controller. Every memory bank for the row operations, column operations, LLR values λ, and tentative code bits and parity check bits are configured to be dual-port memory.
We implemented the partially-parallel LDPC decoder on an FPGA provided by Xilinx. The FPGA device is a Vertex II xc2v6000-5ff1152 which has 33,792 Slices and 144 block RAMs, and each block RAM amount is 18 [Kbit]. We carry out synthesis and implementation using ISE Ver.6.2 provided by Xilinx. We compare the partiallyparallel LDPC decoder based on our message-passing algorithm to the partially-parallel LDPC decoder based on the message-passing algorithm shown in Refs. [7]–[9] in terms of the numbers of occupied Slices, Slice Flip Flops, 4-input Look Up Tables (LUTs) and Block RAMs, and delay of each module. Refs. [7], [8] apply the dual memories architecture to the decoders. The dual memories architecture has each of memories for the row operation module and column operation module. This architecture enables the decoder to

compute the row operations and column operations concurrently. Ref. [8] does not apply the Min-Sum algorithm to the row operation module. The normal message-passing algorithm which has larger number of gates than the Min-Sum algorithm. Ref. [9] applies the scaling method to the MinSum algorithm, which improves the approximation of MinSum algorithm. We design each of the partially-parallel LDPC decoders which has dual memories architecture. The normal message-passing algorithm and the scaling method do not be applied to the row operation module in order to evaluate the practical overhead of our partially-parallel LDPC decoder compared to the main part of the partiallyparallel LDPC decoder based on the message-passing algorithm shown in Refs. [7]–[9].
The synthesis and implementation results are shown in Table 1. Particular overhead units are involved in the column operation module. The synthesis results show that the number of occupied Slices for our column operation module increases by about 32% compared to that for the column operation module based on Refs. [7]–[9]. The controller controls the write enable signals for every memory bank. The three memory banks in our column operation module are written by the row operation module three times as many as those of the column operation module based on Refs. [7]–[9]. Therefore, the controller of ours is slightly different from that based on Refs. [7]–[9], and the synthesis results are also shown. The implementation results (i.e. Place & Route results) show that the number of occupied Slices for our partially-parallel LDPC decoder increases by about 13% compared to that for the decoder based on Refs. [7]–[9], for details, the number of occupied Slice Flip Flops increases by about 25% compared to that for the decoder based on Refs. [7]–[9], and the number of occupied 4-input LUTs increases by about 5% compared to the decoder based on Refs. [7]–[9]. The number of block RAMs for our partially-parallel LDPC decoder is same as that for the decoder based on Refs. [7]–[9]. 5 Conclusion

In this paper, we propose a partially-parallel LDPC decoder improving the message-passing algorithm. Our pro-

posed LDPC decoder improves the timing when the column operations are performed, and it increases the number of the message-passings within the limited number of iterations for decoding. We implemented the proposed partiallyparallel LDPC decoder on an FPGA, and simulated its decoding performance. Implementation result shows that the particular overhead of our decoder is just addressing unit in each column operation module, which increases by only 13% compared to the decoder based of Refs. [7]–[9]. Practical simulations show that our partially-parallel LDPC decoder reduces the average number of iterations for decoding by up to 45% compared to that of the decoder based of Refs. [7]–[9], and it also improves the bit error performance.
