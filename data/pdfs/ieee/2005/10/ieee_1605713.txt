LDPC codes with protograph structure can be considered as subclass of multi-edge type codes, i.e. there might be many protograph based LDPC codes with the same multi-edge type description. RA, IRA, and ARA codes, with suitable definitions of their interleavers, all have simple protograph representations and thus are amenable to both high-speed encoding and decoding. In [19] further extensions of RA, IRA, and ARA codes, all constructed from simple loop-free encoding modules were provided. These extensions provide greater flexibility to construct codes with lower decoding thresholds. However for certain applications low error floor performance is required.
II. LDPC CODED MODULATION
LDPC coded modulation schemes in this paper are constructed by mapping the output of a binary LDPC with protograph structure to high-level modulation either directly by Variable Degree Matched Mapping (VDMM) which will be discussed shortly, or through a channel interleaver i.e. bit-interleaved coded modulation (BICM) approach. The binary protograph based LDPC will be designed such that the asymptotic minimum distance of the code grows linearly with the block size. To do so we need to compute the asymptotic ensemble weight enumerators for the candidate protograph based LDPC codes to guarantee that the minimum distances grow linearly with the block size of the code. We believe that this property will ensure a low error floor performance. For mapping of the output of LDPC code to the signal constellation, we use distance preserving labeling such as independent Gray labeling. III. ASYMPTOTIC ENSEMBLE WEIGHT ENUMERATORS
Analysis of ensemble average weight enumerators is required to design an LDPC code such that the minimum distance of the code grows linearly with block size (similar to regular LDPC codes of variable degree at least 3 [21]). As with irregular ensembles, our constructions are sensitive to the proportion of degree-2 variable nodes. A code with too few such nodes tends to have an iterative decoding threshold that is far from the capacity threshold. A code with too many such nodes tends to not exhibit a minimum distance that grows linearly in block length. Our constructions are based on projected graph, or protograph. To design such structured LDPC codes with minimum distance that linearly grows with the block size, we need to compute the weight enumerators for protograph based LDPC code ensembles. Such computational tools have been only derived for unstructured irregular LDPC ensembles [21], [24], [25], [26], [27]. Recently a method to compute the asymptotic weight distribution for protograph LDPC codes has been proposed [20]. IV. PRECODED LDPC CODES

LDPC codes with protograph structure can be precoded with an accumulator for better performance. Precoding places a degree 1 variable node between a constraint node and a higher degree variable (forming an accumulator) which is then optionally erased. Precoding often lowers the iterative decoding threshold of a given protograph without altering its rate [17]. An example of a family of precoded regular LDPC codes is shown in Fig. 5. As shown in the table, precoding gain decreases as code rate increases. However, we note that in general iterative decoding thresholds for very high code rate regular LDPC codes are already satisfactory.

As shown in [17], Accumulate Repeat Accumulate (ARA) Codes have reasonable thresholds. However their asymptotic ensemble minimum distance does not grow with n. An RA code with an accumulator precoder is called an Accumulate-Repeat-Accumulate (ARA) code [17].

VIII. CONCLUSION
In this paper we have introduced a new structured LDPC codes that exhibit good threshold performance as well as a minimum distance that increases with blocklength. This work has in addition demonstrated the value of precoding as a technique for reducing the iterative decoding threshold while maintaining rate.
