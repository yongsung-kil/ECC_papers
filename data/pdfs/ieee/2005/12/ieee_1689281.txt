Effect of Check Node Processing on the Performance of Message Passing Algorithm in the Context of
LDPC Decoding for DVB-S2
M.Girish Chandra, Harihara S.G, B.S. Adiga, P. Balamuralidhar, P.S. Subramanian

Abstract- Low Density Parity Check (LDPC) codes have become very popular in recent times due to their capacity approaching performance and are considered in the next generation digital video broadcasting DVB-S2 standard. This paper presents few results and observations obtained when some of the existing decoding algorithms are studied in the context of the standardized codes. These results with further consolidation will facilitate to arrive at the right algorithm for decoder design.
I. INTRODUCTION
LDPC codes are linear block codes originally proposed by Gallager in the early 1960s [1]. Their parity check matrix is sparse having low density of one (1) entries. The original codes were regular codes (having uniform column and row weight in the parity check matrix). Recently, these codes have emerged as competitors for turbo codes, with capacity approaching performance. In [2] as well as [3] it is shown that carefully constructed long irregular (number of l's in each row or column is not constant) LDPC codes approach the Shannon capacity by a fraction of a decibel.
Similar to turbo codes, the good performance of LDPC codes is achieved with a proper choice of code and decoding signal processing. The popular LDPC decoding algorithm is the Belief Propagation algorithm (also referred to as Sum-Product algorithm). This can be viewed as a message passing algorithm operating on the Tanner graph [4], which is a bipartite graph representing the parity check matrix, and consisting of variable nodes and check (or constraint) nodes. The algorithm starts with initialization and in each iteration message passing occurs from each check node to all adjacent variable nodes (first half of iteration) and then in the second half, from each variable node to its adjacent check nodes. The decoding performance is achieved through repeated iterations of the message passing along the edges of the graph, with some stopping criterion [4],[5].
The popularity of LDPC codes from the past couple of years led into the proposals for utilizing the code for various

applications and standards (both for wireless and wired). Also, vigorous research and efforts are put into VLSI/ASIC/FPGA realizations of the decoders and encoders [5],[6],[7]. Concentrating on the decoders, one can see the proposals of different structures on various platforms. There are different issues to be considered in the realization like, serial, parallel or semi-parallel architectures, edge memory (memory to store messages passed along the edges of Tanner graph) requirement, processing unit complexity (for check node processing), and so on [6].
Apart from the implementation of regular Sum-Product algorithm, various approximate versions are proposed in the direction of reducing the check node computations and/or edge memory requirement [6],[7],[8]. Some of these algorithms are (A) The Min-Sum, including its compensated version Compensated Min-Sum [4],[5] (B) A- Min and
Compensated A - Min [6] (C) A Posteriori Probability (APP)-
based; APP-variable and APP-check [6] (D) A - Min* [8].
Very recently, another algorithm, soft-bit decoding algorithm for regular codes is proposed in [9].
In this paper, we present few results obtained during the process of LDPC decoder design for DVB-S2 [10],[11],[12]. DVB-S2 is the latest digital video broadcasting standard, where S stands for satellite and 2 for second generation. Since satellite downlink is a power limited channel and further, error rate requirements of DVB-S2 are rather stringent (the packet error rate of 10-7 is the requirement for the MPEG packets transmitted), a powerful error control coding in the combination of outer BCH code and inner LDPC code has been suggested after closely examining several candidates in terms of performance and estimated ASIC size [11]. The outer BCH code clean up additional errors (up to 12 errors) and will improve the overall performance (mainly reducing the error floor) [12],[11].
Even though some results are available in the literature regarding the performance of the mentioned approximate algorithms for short codes, it is important to see how these algorithms perform in the case of standardized DVB-S2 LDPC

II. STANDARDIZED DVB-S2 LDPC CODE

DVB-S2 fixes the length of the encoder output and there are two forward error correction (FEC) frames defined- normal frame of length 64800 and short frame of length 16200. Normal frames can be encoded in eleven different rates and short frames in ten [10],[11],[12].

In order to facilitate systematic encoding and produce irregular LDPC that are especially suitable for high code rates, DVB-S2 uses a class of LDPC codes called extended irregular repeat accumulate (eIRA) codes [11], [4]. With irregular codes improved performance is possible, since variable nodes with higher degrees (degree is number of adjacent nodes) collect more information from their adjacent check nodes and they get corrected first after a small number of iterations. They then help other variable nodes to get corrected through iterative decoding, similar to "wave effect" [11]. When all variable nodes have the same degrees, as in regular codes, this wave effect is not present and all variable nodes can get stuck during the decoding process [11]. The minimum memory required for LDPC decoders implementing the regular BP algorithm is: the memory required to save the intrinsic information and the memory required to save the check-to-variable messages (or equivalently the variable-to-check messages). For example, for DVB-S2 normal frame rate 2/3 code has 215999 edges, which is 3.33 times 64800. Reducing this edge memory is thus important. An algorithm in between Regular Sum-Product and Min-Sum algorithms for check node processing is the A - Min algorithm [6], which takes into account the A input values which have the minimum magnitude. Here, instead of two values as in the case of Min-Sum, A +1 magnitudes need to be saved. A detailed discussion of the algorithm is available in Chapter 4 of [6]. But, the computation involves nonlinear function F(x) and hence LUT.
As far as A -Min* algorithm is concerned, the check node update rule is approximated for all the incoming messages except for the one with the smallest magnitude for which there
is no approximation. Similar to Min-Sum algorithm, A-Min computes only two outgoing magnitudes at each check node, but instead of minimum value as default, LLR of all the variables is the default value [8]. The computation here
also requires 1(x) if it is implemented in the form given in [6].

corresponding LUT). It is demonstrated in [8] that the A-Min algorithm suffers little or no performance loss and
compared to Min-Sum, it is essentially "self tuning".
It is to be noted in all the three variations summarized, only check-node processing changes and the variable-node processing is the same as the regular Sum-Product algorithm.

V. CONCLUSIONS
The paper presented different check-node processing strategies and it is observed that the Min-Sum algorithm could be utilized for the DVB-S2 LDPC decoder, with little performance degradation. This can be taken care of with the outer BCH code, built into the overall coding scheme. The algorithm has the associated advantages of edge-memory reduction and the less complex check-node processing, which are really useful for the decoder of DVB-S2 LDPC code, as the latter is of long length. Further studies are being carried out for fixed-point implementation as well as consolidation of results for different rates. The examinations are also being considered for A -Min and A - Min * algorithms, to finally arrive at a proper decoder architecture.
