On the EXIT Chart Analysis of Low-Density Parity-Check Codes
Sreenivas Rao Kollu and Hamid Jafarkhani Center for Pervasive Communications and Computing Department of Electrical Engineering and Computer Science
University of California, Irvine Email: {skollu, hamidj}@uci.edu

Abstract‚Äî The extrinsic information transfer (EXIT) charts are used to analyze the sum-product decoding algorithm of lowdensity parity-check (LDPC) codes. The codes are optimized by performing curve fitting on EXIT charts. This technique is developed for binary erasure channels based on area property and is extended to other channels. We show that on channels where the area property is not valid, the curve fitting approach is a sub-optimal method of optimizing for the best convergence threshold. On Gaussian channels, the messages from variable nodes to check nodes are approximated by a symmetric Gaussian distribution. We prove that the check node detector mutual information transfer function derived using this approximation is an upper bound. So the EXIT chart analysis predicts an optimistic value for the convergence threshold. We give an empirical lower bound for the check node detector mutual information transfer function. For few irregular LDPC codes designed for AWGN channel using density evolution, we have examined the accuracy of convergence thresholds predicted by EXIT charts using these bounds for the check node transfer function. The analysis of EXIT chart technique presented in this paper can be extended to other channels for which the distribution of channel messages can be approximated by a symmetric Gaussian/Gaussian mixture distribution.
Keywords: Low-density parity-check codes, mutual information, density evolution, Gaussian approximation,
EXIT chart, convergence threshold
I. INTRODUCTION
The low-density parity-check (LDPC) codes introduced by Gallager [1] and rediscovered by MacKay and Neal [2], [3] are shown to achieve near shannon limit capacity on additive white Gaussian noise (AWGN) and erasure channels [4], [5]. The degrees of the checks and the variables affect the convergence behavior of the iterative decoder and thus affect the performance of the code [6]. Hence there has been considerable interest in the design of good degree sequences for LDPC codes for various channels.
Richardson et al. introduced a technique called density evolution to analyze the sum-product decoding algorithm and design good LDPC codes [7]. Density evolution tracks the probability density functions of the messages exchanged between check nodes and variable nodes in the decoder. Density evolution is computationally intensive and hence using a Gaussian approximation (GA), low-complexity approximate analysis methods [8], [9], that track a single parameter instead of the density functions of the messages have been developed.

The mean and the mutual information of the messages exchanged in the decoder are respectively tracked in GA based on mean [8] and EXIT chart [9] techniques. In this paper we analyze the EXIT chart technique for LDPC codes.
In the cases of density evolution and GA based on mean techniques, the optimization criterion is to maximize the noise threshold for convergence of the iterative decoder. But in the case of EXIT charts, a different approach called curve fitting is taken. We show that the curve fitting approach is an indirect and sub-optimal way of optimizing for the best convergence threshold and that better performing codes can be designed if optimized for the best convergence threshold. Thus the effectiveness of the optimization technique is dependent on accurately predicting the convergence threshold of a code with given check and variable node degree profiles. We investigate the accuracy of the convergence thresholds predicted using EXIT charts. We consider binary input AWGN channel for the analysis but the analysis presented in this paper is valid for bi-partite/tri-partite decoding [10], [11] of LDPC codes on channels where the apriori information from the channel is either symmetric Gaussian or can be approximated by a symmetric Gaussian mixture distribution. The examples for this type of channels are AWGN channel with high order constellation modulation schemes [12], fast fading MIMO channels [13] and ISI channels [14].
In EXIT chart analysis, the pdf of messages from variable nodes to check nodes is approximated by a single Gaussian distribution [15], [13]. We show that this approximation leads to prediction of a convergence threshold better than the actual value and a code does not converge at the predicted convergence threshold. We give a lower bound for the check node detector EXIT function.
The notation used in this paper is explained in Section II. Section III introduces EXIT charts and analyzes the curve fitting approach for code optimization. The accuracy of EXIT charts for regular and irregular codes is discussed in Sections IV and V respectively.

III. EXIT CHART

The EXIT chart is invented by Brink as an engineering tool to analyze the convergence behavior of the iterative decoding of concatenated codes [9]. By treating the LDPC decoder as concatenation of variable node and check node detectors, the convergence behavior of the decoder can be analyzed. For a discussion on EXIT charts and the computation of EXIT functions of check and node variables, we refer to [9] and [13], [15] respectively. The extrinsic information transfer function is computed by modelling the apriori messages by a symmetric Gaussian distribution.

A. Convergence threshold
The convergence threshold of a code is defined as the maximum noise standard deviation for which the probability of error tends to zero as the number of iterations increases. The EXIT chart predicts the convergence of the decoder if the EXIT function of outer/variable node decoder is above the inverse EXIT function of inner/check node decoder [9]. For an LDPC code with degree profiles (Œª, œÅ), the convergence threshold predicted by EXIT chart is the maximum noise standard deviation at which the EXIT function of the variable node detector, Tv lies above the inverse of check node detector EXIT curve, Tc‚àí1. This may not be equal to the convergence threshold of the code as the transfer functions are only approximations. We will refer to this convergence threshold as predicted threshold to avoid confusion with the actual convergence threshold.
B. Curve fitting approach
In curve fitting approach, the optimization is done by matching the transfer functions, Tv and Tc‚àí1 at a specific noise standard deviation œÉ0 while satisfying the convergence criterion. This method is designed for BEC channel based on area property proved for that particular channel [17]. According to the area property, the area between the transfer curves is proportional to the rate loss and hence the transfer curves have to be matched to minimize the rate loss. This optimization technique has been extended to other channels like AWGN though the area property does not hold. In this section we explain the reason why the curve fitting design results in good performing codes even on channels for which the area property is not satisfied. The function Tv is a decreasing function of noise standard deviation while Tc‚àí1 is roughly independent of noise power. Let us assume that the degree profiles of the code are chosen by closely matching the curves at a particular noise standard deviation, œÉ0 and the convergence criterion is met at noise standard deviation œÉ0. When the noise standard deviation is increased above œÉ0, the Tv curve will move downwards and cut Tc‚àí1 and thus when the curves are closely matched at œÉ0, the code does not converge for a noise standard deviation greater than œÉ0. Thus curve matching at a particular noise standard deviation œÉ0 results in a code with a predicted convergence threshold close to œÉ0. If œÉ0 is chosen close to the optimum convergence threshold value for the given design parameters, the curve matching will result in good performing LDPC codes.
Thus the curve fitting approach is an indirect method to optimize for the best predicted convergence threshold. Better codes can be designed by optimizing for the best predicted threshold. We designed two codes with the same design parameters using two different approaches, one by curve fitting at noise standard devation 0.9441 and the other with the optimization technique to maximize the predicted convergence threshold. VI. CONCLUSIONS
We analyzed the curve fitting approach of designing LDPC codes for channels for which the area property is not satisfied. We showed that the optimization technique method based solely on convergence criterion gives better results. For an AWGN channel, we have shown that the check node EXIT function, derived by approximating the apriori messages to the check node detector by a symmetric Gaussian distribution, is an upper bound. So the convergence threshold predicted using EXIT chart is optimistic and an LDPC code does not converge at the predicted convergence threshold. We derived the relation between the transfer functions of single-parity check code and

its dual code, repetitive code. We used this relation to find an
empirical lower bound for the check node EXIT curve. For
a few example irregular codes considered in this work, the
convergence thresholds predicted by using the lower bound
for the check node EXIT function are within 0.1 dB of the
actual values.
