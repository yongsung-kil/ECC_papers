Abstract— This paper presents methods for efficient encoding of quasi-cyclic LDPC codes. Based on these methods, encoding of quasi-cyclic LDPC codes can be implemented using simple shiftregisters with complexity linearly proportional to the number of parity-check bits of a code for serial encoding and to the length of a code for parallel encoding. Various encoding circuits are devised and they provide a range of trade-offs between encoding complexity and speed.
I. Introduction
Quasi-cyclic (QC) low-density parity-check (LDPC) codes form an important subclass of LDPC codes. These codes have encoding advantage over the other types of LDPC codes. Also well designed QC-LDPC codes perform as well as computergenerated random (or random-like) LDPC codes in terms of bit-error performance, block-error performance, error-floor, and rate of iterative decoding convergence, collectively [1]-[5]. This paper addresses the issue of efficient encoding of QCLDPC codes. A QC-LDPC code is given by the null space of a parity-check matrix that is an array of sparse circulants of the same size [1], [6]. For systematic encoding, it is necessary to find the generator matrix of the code in systematic form. This paper presents two methods for finding the generator matrix of a QC-LDPC code in a systematic-circulant form from its parity-check matrix given in circulant form. Based on this systematic-circulant form of a generator matrix, several types of encoding circuits using simple shift-registers are devised. It is shown that the encoding complexity of a QC-LDPC code is linearly proportional to the number of parity-check bits of the code for serial encoding and to the length of the code for highspeed parallel encoding. This is a substantial improvement over the quadratic complexity involved in general type of LDPC codes [7].
II. QC-LDPC Codes
A circulant is a square matrix in which each row is the cyclic shift (one place to the right) of the row above it and the first row is the cyclic shift of the last row. For such a circulant, each column is the downward cyclic shift of the column on its left and the first column is the cyclic shift of the last column. The row and column weights of a circulant are the same, say w. For simplicity, we say that the circulant has weight w. A

If all the parity sections are generated in parallel at the same time, a circuit that completes encoding in b clock cycles can be implemented. In this case c(t − c) feedback shiftregisters are needed. To start the encoding, all the first columns hi,j’s of the c(t − c) circulants in Gqc are loaded into the feedback shift-registers in one clock cycle and the first bits, p1,1, p2,1, . . . , pc,1, of all the c parity sections are formed. This implementation requires a total of c(t − c)b flip-flops, c((t − c)b − 1) XOR-gates and c(t − c)b AND-gates. Other encoding circuits with encoding speeds between cb(or (t−c)b) and b cycles obviously can be implemented. This offers a
wide range of trade-offs between encoding speed and encoding
complexity.

encoder can be implemented in the same way as described

for Case-I. The second sub-encoder can be implemented as a

conventional encoder for a linear block code [10]. If cb − r

is small, the complexity of the second sub-encoder is small.

If the circulants in Hqc have very small weights, say 1 or 2, cb − r is either zero or small. This is in general the

case for the QC-LDPC codes constructed in [1]. Therefore,

the encoder complexity is dominated by the first sub-encoder

whose complexity is linearly proportional to cb or tb. With

the above encoding, the output codeword is not completely

systematic. Only the first (t − c)b bits are identical to the

information bits.

VI. Conclusion
In this paper, we have presented three methods for finding the generator matrix in SC-form of a QC-LDPC code whose parity-check matrix is given as an array of circulants. Several efficient encoding implementations have been given. These implementations require a complexity linearly proportional to the number of parity-check bits of the code for serial encoding and to the code length for high-speed parallel encoding. Efficient encoding of a very special class of irregular QCLDPC codes, called irregular partitioned permutation (IPP) LDPC codes was also proposed in [13]. This encoding method is not as general and efficient as the methods presented in this paper, which apply to all QC-LDPC codes, regular or irregular. Furthermore, the parity-check matrix of an IPP-LDPC code contains many columns of weights 2 and 3 that in general, make the code to have a high error-floor.
