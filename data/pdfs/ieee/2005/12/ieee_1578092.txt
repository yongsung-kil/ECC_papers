Optimal Overlapped Message Passing Decoding for Quasi-Cyclic Low-Density Parity-Check Codes
Yongmei Dai and Zhiyuan Yan
Department of Electrical and Computer Engineering Lehigh University, PA 18015, USA E-mail: {yod304, yan}@lehigh.edu

Abstract— In this paper, we first present a general scheduling model for the overlapped message passing (OMP) decoding of LDPC codes, and show that maximizing the throughput amounts to minimizing the intra- and inter-iteration waiting times. Then we focus on maximizing the throughput for OMP decoders of quasi-cyclic (QC) LDPC codes using a partly parallel hardware structure, which can be modelled as a data dependency and hardware constrained optimization problem. Finally, we propose a scheduling scheme, which gives the optimal solution to the optimization problem under certain conditions. In comparison to previously proposed scheduling schemes, our new scheduling scheme leads to both higher hardware utilization efficiency (HUE) and greater throughput gain.
Index Terms— low-density parity-check (LDPC) codes, quasicyclic (QC) codes, overlapped message passing, throughput.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1, 2] have been receiving growing attention due to their near-capacity performances when decoded iteratively. However, random code structure, often assumed in the theoretical studies of LDPC codes, poses great challenges to efficient hardware implementation. Complex interconnect network with global routing is required for fully parallel decoder architectures [3]. Large memory and complex address generation mechanisms are needed for serial decoder architectures [4]. To overcome these problems, LDPC codes with structural regularities that are conducive to efficient hardware implementation are desired. Quasi-cyclic (QC) LDPC codes1 [5–7] are therefore of great practical interest.
An LDPC code can be characterized by a Tanner graph [9], wherein the codeword bits and the parity check equations are represented as variable and check nodes, respectively. LDPC codes can be efficiently decoded using the multi-iteration message passing algorithm (see, e.g., [10]). Each iteration of the message passing decoding consists of two sequential steps: check node updates followed by variable node updates. Thus, two types — intra- and inter-iteration – of data dependencies exist since the variable (check) node updates of any iteration utilize the results of the check (variable) node updates of the same (previous) iteration. Hence, check node function units (CNFUs) and variable node function units (VNFUs)—the

hardware that implements the check and variable node updates, respectively— often work one after another. This leads to low hardware utilization efficiency (HUE) and low throughput. The HUE can be improved by folding the check and variable node updates [11], but this approach does not increase the throughput. One approach to improve both the throughput and HUE is to allow the CNFUs and VNFUs to operate alternately on two different blocks of data at the same time. Both throughput and HUE are doubled at the cost of doubling the memory requirement [4, 5]. Recently, an overlapped message passing (OMP) decoding algorithm, where the check and variable node updates are (partially) overlapped, has been proposed for QC LDPC codes [8] and LDPC codes in general [12]. Exploiting the concurrency between check and variable node updates, the OMP decoding improves both throughput and HUE while using the same memory and introducing no error performance degradation.
The technical contributions of this paper are as follows. Firstly, we present a general scheduling model for the OMP decoding of LDPC codes, and show that maximizing the throughput amounts to minimizing the intra- and interiteration waiting times. Secondly, we show that maximizing the throughput for OMP decoders of QC LDPC codes based on the same hardware structure as that in [8] can be modelled as a joint optimization problem subject to both the data dependency and the hardware constraints. Thirdly, we propose a scheduling scheme, which first finds the minimal intra-iteration waiting time and then determines the corresponding minimal interiteration waiting time. In comparison to previously proposed scheduling schemes, our new scheduling scheme leads to both higher HUE and greater throughput gain.

In this paper, we assume the same hardware structure as that in [8], which consists of j k-input CNFUs, k j-input VNFUs, and j × k dual-port memory banks. Each memory bank corresponds to one Ixs,t , and has m memory units, labelled with addresses {0, 1, 2, · · · , m − 1}. Each 1 in Ixs,t involves with both a check node and a variable node, and hence the m memory units alternately store the extrinsic log likelihood ratio (LLR) information of the m check and m variable nodes corresponding to Ixs,t . After the check node updates, the LLR of the check node that is related to the 1 in the i-th (0 ≤ i ≤ m − 1) row of Ixs,t is stored in the i-th memory unit. Thus, memory address i is also called the row address of the 1 in the i-th row. The same memory unit
also stores the LLR of the variable node that is related to the same 1 after the variable node updates. Since the 1 is in the (xs,t + i mod m)-th column, the address (xs,t + i mod m) is also called the column address of this single 1. Note this single 1 in Ixs,t corresponds to one row address and one column address, between which there is an offset of xs,t. Each CNFU operates on the data corresponding to one block row of H, i.e., [Ibs , Ibsa, · · · , Ibsak−1 ], and updates the memory units with the same row address in the k memory banks in
the block row, one row per clock cycle.
