Reduced complexity, FPGA implementation of quasi-cyclic LDPC decoder

Christian Spagnol, William Marnane, Emanuel Popovici

Abstract— This paper describes an FPGA implementation of a decoder for a particular family of low density parity check (LDPC) codes, the quasi-cyclic LDPC codes. The structure of a quasi-cyclic code is well known and allows us to reduce the complexity of the interconnections between bit nodes and check nodes. The decoder has a semi-parallel architecture and it takes full advantage of the structure of the code and the hardware resources present in an FPGA. We achieve simple memory controller resulting in an efficient use of the memory. The decoder is implemented based on the parameters that characterize a quasi-cyclic LDPC code. This makes it easily adaptable for a class of quasi-cyclic codes. We evaluate the performance of our codes and present some FPGA design trade-off.
I. INTRODUCTION
Introduced by Gallager in 1963 [1] Low Density Parity Check Codes were neglected until the work of MacKay [2] in 1995; these codes yield high performance on the binary symmetric channel (BSC) as well as on the additive white Gaussian noise (AWGN) channel, and can outperform the Turbo codes in many applications. The algorithm used for the decoding is called belief propagation, known also as the sum-product algorithm, it uses a graphical representation of the code, the Tanner Graph. Unfortunately the hardware implementation of this algorithm involves making a number of design choices that have a trade off between the BER performance and the complexity. One of the dominant problems of any implementation of a LDPC decoder depends on the level of messages passing between variable nodes and check nodes. Every edge of the Tanner graph represents a connection that must be implemented in some way, leading to the classical area versus time design problem. By choosing a code with some degree of regularity this problem can be alleviated. One group of codes that possess this quality is the quasi-cyclic codes family. These codes were first introduced by Townsend in [3] and have seen recent interest both for the mathematic characteristics [4] and for the possibility of efficient encoding[5]. This paper presents an FPGA implementation of a decoder that achieves simplified interconnections and memory control in comparison with standard decoders, and has a parameterization that makes it easy to generalize for a large family of quasi-cyclic codes.

II. LOW DENSITY PARITY CHECK CODES
LDPC codes are linear block codes and so they are defined as the null space of a parity check matrix H. Their characteristic is the sparseness of this matrix. For block codes a vector c is a codeword if (and only if ) it satisfies: cHT = 0
LDPC codes can be represented by a Tanner Graph, see figure 1, that is drawn from the matrix H.This is drawn from the matrix by mapping the rows of the matrix to the check nodes and the columns to the bit nodes. The ’1’s of the matrix form the edges of the graph. This representation of the code allows us to understand the meaning of the belief propagation algorithm as a special case of the message passing algorithm[2]. If any check node computes the value rij(b) as the probability of the check i been satisfied given that the value of node j is b, and any bit node computes the value qij(b) as the probability of that bit to be b given the event that all the check nodes involving bit i except check j are satisfied, then all the messages relative to the bit nodes must be computed and have to be passed to the check nodes they are connected to before these can compute their messages; and vice versa. The belief propagation algorithm starts initializing the bit nodes with values calculated from the data received, keeps updating alternatively check nodes and bit nodes, and sends the messages calculated between the nodes until a codeword is found or other exit criterions are matched. It is demonstrated that under the hypothesis of no cycle in the graph this algorithm converges to the correct codeword, and that even if this hypothesis is not achieved this algorithm performs well.

IV. DECODER ARCHITECTURES
A. General architectures
The computations of the messages for the check nodes and bit nodes carried out in the belief propagation algorithm are independent from each other for each iteration, suggesting that a parallel architecture can be implemented. As mentioned earlier, the Tanner graph associated with the parity check matrix of the code dictates the complexity of the connections network in between nodes. In a full parallel architecture, the number and complexity of the interconnections results in implementations where up to 80% of the area is dominated by the routing. The problem can be partially mitigated through reducing the parallelism (eventually reaching a complete serial architecture) and using memory elements to store the intermediate messages passed along the edges of the Tanner graph. This solution reduces complexity of the routing but introduces new complexity in the memory control and in the routing between memories and node elements. Another drawback of this

solution is the reduction of the throughput of the decoder because every iterations of the belief propagation algorithm has now to be carried on in more than a single clock cycle. This can be alleviated by pipelining several blocks.
B. Quasi-cyclic codes architectures
A possible way to reduce the overall complexity of the decoder is to find a good parity check matrix that constitutes a good trade off between performance of the code and complexity of the hardware. The quasi-cyclic codes are some of the best candidates due to the structure of the parity check matrix and the good performances they have displayed. As seen in section III a quasi-cyclic parity check matrix structure is represented by two parameters m and α. Following from this, two types of architectures can be thought of from a quasi-cyclic code. The first is based on the parameter α and the second on m. Since m is much bigger than α, one architecture is close to a serial implementation and the other is close to a full parallel one. Both of these architectures have advantages and disadvantages. This paper deals mainly with the α-based architecture in comparison with a semi-parallel architecture. In particular the architecture here implemented is called α2 architecture because both the number of bit nodes and check nodes instantiated depend on the parameter α.
V. α2 ARCHITECTURE
Considering a (3,6) LDPC code, the problem that afflicts a serial implementation of the decoder is the structure of the address control logic. This is because the complexity of the interconnections in the Tanner graph is reflected in the randomness of the accesses in memory. If a (3,6) regular LDPC code, described by the matrix (1), is considered then full advantage of the structure can be taken by to instantiating a number of check nodes equal to α and a number of bit nodes equal to 2α. It will be shown how the α2 architecture reduces the complex memory control to a small number of adders and multiplexes, and allows full advantage to be taken of some features typical of the FPGA memory. Every one of the α r mem block contains six m x 4 memories and each one of the six messages coming from a check node module is stored in a different memory. As in the q mem block the ADDR bus is the output of the 0 to m−1 counter and the writing/reading phases alternate every m clocks. During the writing phase, which corresponds with the update r messages phase, the messages coming from the bit node module are stored in the same positions of the six memories. Considering the reading phase, the first memory gives the message related to the first bit node in Hi1, the second memory the one related to the second bit node in the same matrix, so both these messages are connected with the same bit node module. The third input of this bit node module is connected to a third memory of the r mem block, identified by the position of the identity submatrix for that column. It is important to note that the decision to represent the circular matrix with the parameters {e(1), s} instead of {e(1), e(2)} does not achieve any reduction in the number of components, but it simplifies the interconnections. This is because, with this solution, any bit node module (or check node module) always refers to the first element of the binomial as the first message to compute, even if the shifting has made it the second (see row 5 or column 4 of matrix (2) ) and independently from the position of the relative identity submatrix.

The possibility of accessing the memory in configurations that vary from 1-bit wide to 16-bit wide allows the design to store the 4-bit wide messages without any waste of resources. For values of m < 512 it is possible to take full advantage of the dual port architecture of the memory and implement two memories in the same port allowing other parameters of the code to grow. The construction of the architecture of the decoder is completely parameterized with the parameters of the codes allowing the architecture to be used for a large family of codes.
VII. CONCLUSIONS
In this contribution a novel architecture for a quasi-cyclic LDPC decoder has been proposed. The architecture takes full advantage of the structure of parity check matrix of quasi-cyclic codes and translates it to a simplified memory control that allows the reduction of the area and of the power consumed by the FPGA implementation of the decoder.
