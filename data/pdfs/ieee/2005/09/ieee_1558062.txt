Abstract— In this paper, we introduce a new approach to puncturing irregular Low-Density Parity-Check (LDPC) mother codes. The proposed greedy approach constructs a single degree puncture sequence that identifies the variable-node degrees to puncture in a nested fashion. The construction is done one degree at a time choosing the variable-degree that requires the least amount of SNR for newly punctured ensemble. This approach takes into account the finite length of a particular LDPC code realization by adjusting the puncture probabilities using the finite granularity of each variable-degree available for puncturing given a codeword length. This greedy approach results in a single degree puncture sequence of encapsulating subsets for all possible code rates. Subsequently, a single variable-node sequence can be easily constructed from this degree sequence. In AWGN, the SNR gap performance of these rate-compatible LDPC codes varies little for a wide range of effective code rates with respect to the rate-distorted Shannon limit using binary source signalling.
I. INTRODUCTION
In [1] and [2], Gallager introduced Low-Density-ParityCheck (LDPC) error correcting codes. For nearly 30 years, research on LDPC codes remained dormant until their rediscovery that directly followed the discovery of Parallel Concatenated Convolutional Codes (PCCC) (a.k.a. “Turbo Codes”). Since then, research activity in the area of LDPC codes has resurfaced with much enthusiasm because of new code constructions approaching Shannon Capacity performance and rivalling benchmarks set by Turbo codes [3], [4], [5].
Often when describing LDPC codes, the distinction of regular and irregular LDPC codes surfaces. In [2], Gallager introduced regular (m, k) LDPC codes where each column of the parity-check matrix H (corresponding to a variablenode, i.e. codeword element) has exactly m non-zero elements and each row of H (corresponds to a check-node, i.e. paritycheck equation) has exactly k non-zero elements. On the other hand, irregular LDPC codes allow for heterogeneous non-zero element counts in both the rows and columns of H and been shown in [3], [4], [5] to significantly outperform the regular LDPC code constructions.
In [6] and [7], the authors introduced an approach to determine the degree puncture probabilities (for irregular LDPC codes) based upon an extended version of the Gaussian Approximation (GA) that accounts for puncturing. Their approach uses algorithms based in linear programming to maximize the fraction of total codeword elements punctured given a SNR threshold constraint for the asymptotic ensemble.

For different code rates, this approach may results in multiple puncture sequences that cannot be arranged in such a way to form single puncture sequence of encapsulating subsets as the effective code rate increases. As a result, the rate-compatible irregular LDPC codes derived in this manner do not lend themselves well for application in Hybrid-ARQ systems.
Hence, in this paper, we introduce an approach that constructs a single degree puncture sequence for all possible code rates of a given irregular LDPC mother code. The approach is greedy in SNR and takes into account the finite length of the mother code in the puncture sequence’s construction. In the following sections, we briefly introduce LDPC codes and a new approach to making them rate-compatible using a single puncture sequence.

The number of edges connected to any particular node determines its degree corresponding to the number of ones in the respective column or row of H.

B. Algorithm for Greedy Degree Puncture Construction Using the Gaussian Approximation BER Expression
The following describes a greedy method of constructing the variable-degree puncture sequence that can be converted to a specific variable-node sequence for any given LDPC realization of (λ(x), ρ(x)).
1) For each variable-degree available for puncturing, calculate the required mean input Log-Likelihood-Ratio (LLR) values, mu0 , that achieve the design criteria (a target BER within a finite number of iterations) on the ensemble of codes using the Gaussian Approximation BER expression for punctured codes, BERG(kA) .
2) Select the variable-degree, j, within the design criteria for puncturing that requires the smallest mean input LLR value and append the degree to the puncturing sequence.
3) Taking into account a specific code length and the finite number of variable-nodes of each degree, adjust the puncturing probability for the punctured variable-degree selected in Step 2), πj(0) (for an example see Fig. 2 and discussion in Section IV-A).
4) Return to Step 1) and repeat until the puncturing sequence length corresponds to the Binary Erasure Channel (BEC) threshold [8] for random errors (or if you like until you have a code rate of one). If the fraction of punctured variable-nodes reaches or is beyond the BEC threshold, stop.

We used the design criterion of a target BER of 10−6 within 50 iterations. Note that for different design criteria the greedy approach yields different puncture sequences. Note the discontinuities in π4(0). These occur because for N = 3255 there are only three variable-nodes of degree 4. When j = 4 in Step 2), π4(0) is adjusted according to the allowable granularity (i.e. π4(0) ∈ {0, 1/3, 2/3, 1}) and thus take into account the finite length of the codeword. These discontinuities in puncture probabilities highlight the discrete nature of the problem for LDPC code realizations of practical codeword lengths not taken into account previously.
Because for each code rate we use the beginning subset of a single puncturing degree sequence (i.e. nested puncture sequences), the puncturing probabilities monotonically increase as the code rate increases. For the work presented in this document, the variable-nodes in the node puncture sequence were chosen as the first available in the codeword corresponding to their degree in the order of the degree puncture sequence. Other node selection choices can also be made using the degree sequence, and further work can be done to optimize the node selection for the fastest recovery of punctured nodes.
Fig. 3 and Fig. 4 are BER and Codeword Error Rate (CER) simulations results for the punctured LDPC mother code of length N = 3255 using a maximum of 100 iterations in the sum-product decoder. All simulations ran until 300 codeword errors or until 50000 code words which ever occurred first. Fig. 5 plot the SNR gap with respect to the binary signalling rate-distorted Shannon capacity (measured in Eb/No dB) at BER=10−3. It can be seen in Fig. 5 that for a wide range of code rates the punctured code’s performance measured by the SNR gap remains varies little with respect to rate-distorted Shannon Capacity (within 0.5 dB for code rates up around 0.75).
The choice of mother code can play a factor in both code ensemble thresholds and error floors. Random constructions tend to have error floors where as other constructions may have lower error floors. Empirically, we have observed that puncturing tends to lowers the error floor (as seen in Fig. 3 and Fig. 4), and in constructions where error floors are not seen in the mother code, puncturing does not appear create error floors.

V. CONCLUSIONS
In summary, we have presented a greedy approach to constructing variable-degree puncture sequences for irregular LDPC codes that adjusts the puncture probabilities according to the discrete nature of finite length codes. The approach performs well over a wide range of effective code rates with improved error floors as the code rate increases. Although different design criteria results in different error performance, this greedy puncturing approach for LDPC appears to work for mother code realizations of different code rates.
