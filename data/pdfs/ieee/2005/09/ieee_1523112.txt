Versatile Architectures for Decoding a Class of LDPC Codes
Andrew Byrne ∗ Emanuel M. Popovici∗ Michael O’Sullivan †

Abstract — This paper presents a construction for low and high rate Low-Density Parity Check (LDPC) codes, their performance and efficient hardware implementation. The problem with decoding for LDPC codes is the linear increase in resource requirements as the size of the parity check matrix increases. This results in a number of issues with regard to practical implementation. These issues include interconnect routing, memory size and parallelism. A construction for low complexity, variable rate LDPC code will be introduced and an architecture that takes advantage of certain properties of this construction is proposed. A versatile LDPC decoding architecture is then evaluated on FPGA.

1 Introduction

LDPC codes were originally discovered by Gallager[1] in the 1960s but were too complex to implement. Since their rediscovery in 1995 by Mackay and Neal[2], there has been renewed interest in the codes because their bit error rate performance approaches the Shannon limit[3].
LDPC codes have an advantage over Turbo codes in that they require an order of magnitude less arithmetic computations than Turbo decoders. Furthermore, decoding algorithms for LDPC codes also have the benefit of being inherently parallel.
This property has allowed for the implementation of very fast parallel decoders. Ultimately though, this leads to very complex, expensive circuits in terms of interconnect and routing between the bit and check nodes. Serial architectures on the other hand are very simple in terms of interconnect but have very high memory requirements. A balance can be found by introducing a certain amount of parallelism to the serial approach by dividing the decoding process over several serial decoders working in parallel[4].
This paper presents some particular cases of a construction by O’Sullivan[5] for variable rate, low complexity LDPC codes with large girth. In this construction, the parity check matrix is composed of blocks of square matrices. Each square matrix is a sum of permutation matrices which are defined

2 Overview of LDPC Decoding

LDPC codes are a special class linear block codes whose parity check matrix (H) has a low density of ones. The sparseness of the 1’s in H makes them much easier to implement.
To visualize the Message-Passing algorithm[6] used for decoding we consider a H matrix and represent it as a bi-partite graph called the Tanner graph (Figure 1). Each column in H is represented by a bit node in the graph and each row is represented by a check node. In this case, H is a 4 × 8 matrix. Each 1 in H corresponds to an edge between a check and bit node in the graph. The bit and check nodes represent computational blocks and the edges of the graph indicate how messages are passed.

LDPC decoders implement a message passing algorithm which specifies the computation of messages and their communication between bit and check nodes as defined by the edges in the graph. One iteration of the decoding algorithm consists of a round of message passing from each bit node to all connected check nodes, followed by a round of message passing from each check node to all connected bit nodes. The decoding of a codeword is achieved through successive iterations of this message passing until the message is decoded or some upper time limit has been reached.
Although the computational complexity for calculating the check-to-bit and bit-to-check messages is low, the sparse structure of the Tanner graph leads to complex interconnect in parallel architectures, or excessive memory requirements in serial architectures. We aim to tackle these restrictions by presenting a versatile serial architecture.

The main idea of this paper is to exploit this construction in a versatile architecture and to use the simple addressing introduced by these modulo solutions.

Using this method, the nodes of the bipartite graph can be doubly indexed, the row nodes by (r, n) for r = 1..R and n = 1..N , the column nodes by (s, n) for s = 1..S and n = 1..N . The edges of the graph are described simply by the parameters used to determine the matrices Ars. For example, if each Ars is a circulant defined by a shift brs, then the set of RS values brs completely describes the RN × SN matrix H. Row node (r, m) is connected to S column nodes, namely for each s = 1..S, (s, n) where n ≡ m+brs mod N . The situation for affine permutations is similar. In fact one can even make each Ars a sum of affine permutations and still take advantage of these methods.
In [7, 8, 9, 10] each of the Ars is a circulant matrix. Fan [7] models the choice of shift for the circulants on Reed-Solomon codes. Tanner Sridhara and Fuja [9] use a prime N for the dimension of the circulants and use the multiplicative group of Z/N to choose the shifts for the circulants. Fossorier [10] used computer search to choose the shifts for the circulants and achieve girth of 6 and 8. The use of affine permutation matrices was introduced by Bond, Hui and Schmidt [11]. Methods for ensuring that the bipartite graph of H has large girth when affine permutations are used is presented in [5, 12].

The message passing algorithm (discussed in Section II) is inherently parallel, therefore a parallel architecture can process all the bit and check nodes simultaneously. This is because there is no dependency between any of the check nodes Qn for n = 1, 2, .., N nor between any of the bit nodes Rm for m = 1, 2, , M . In fact, one can directly map the bit and check computational blocks onto the bit and check nodes in the Tanner graph, using the edges as the interconnect.
A fully parallel implementation while efficient from a power and speed point of view is inefficient in terms of area due to interconnect between the processing elements. The number of computational blocks required is a one to one relationship with the number of nodes in the Tanner graph. For a large H matrix, the resource demands and complexity for a hardware implementation will become excessive.

4.2 Serial Decoder
A serial implementation (Figure 3)dramatically reduces the complexity of the interconnect and the total area of the design. This architecture only processes one bit at a time and therefore the latency is directly related to the length of the code. In a serial decoder one bit node computational block is repeatedly used to process the entire codeword. This will make the decoding process very slow and will also require a lot of memory to store all the messages. One way to increase the speed of the decoder is to distribute the process across a number of computational blocks. The check and bit node messages are stored temporarily in memory. The addressing of memory must consider the message flow between the check and bit nodes.

4.3 Modulo N arithmetic

Modulo operation as presented in Section 3 is costly when implemented in hardware. Let us consider a construction for an LDPC code from Section 3 where N = 2k + 1. Thus the equation defining the matrix becomes pm + q ≡ n mod (2k + 1). The reason for choosing N of this type is that we can calculate the modulus very quickly and efficiently in hardware. The advantage is that we can determine the locations of all the ’1’s in the H matrix very fast and with a minimum memory cost. The dissadvantage is that there are only few primes of this type. However, the number is sufficient to research good variable rate LDPC codes.

4.4 Versatile Decoding Architecture
In both serial and parallel architectures the issue of concern was storing the H matrix so that it could be used for the decoding process. For a 1542×771 matrix this will require an excessive amount of memory particularly when you consider that less than half of the entries are needed for decoding (only the ’1’s are needed). We looked at the construction of the code for N = 2k +1 so that the properties discussed in Section 4.3 could be exploited. By using the relations described in Section 4.3 we can efficiently locate every instance of a ’1’ in the matrix without storing the matrix in memory. This is achieved by comparing the modulus results for am and n. Every match corresponds to a ’1’ in the H matrix. In this way an efficient addressing mechanism can be implemented as shown in Figure 4. In this architecture we present a serial model, but the addressing can be extended to accommodate the parallel decoder discussed in Section 4.1.
