A Degree-Matched Check Node Approximation for LDPC Decoding

Sheryl L. Howard, Christian Schlegel and Vincent C. Gaudet Dept. of Electrical & Computer Engineering University of Alberta Edmonton, AB Canada T6G 2V4 Email: sheryl,schlegel,vgaudet@ece.ualberta.ca

Abstract— This paper examines ways to recoup the performance loss incurred when using the min-sum approximation instead of the exact sum-product algorithm for decoding low-density parity check codes (LDPCs). Approximations to the correction factor exactly expressing the difference between these two decoding algorithms exist for degree 3 check nodes, and can be applied to higher degree nodes by subdividing them into component degree 3 nodes. However, this results in replication of the approximation. An asymptotic expression for the correction factor at a check node of any degree is derived in this paper, and used to develop two simple approximations to the correction factor, matched to the check node degree. One has very low complexity, and both only need be applied once per check node extrinsic message. Simulation results are presented for each check node approximation when decoding a regular and an irregular LDPC. Both degree-matched check node approximations achieve sum-product decoding performance.
I. INTRODUCTION
Low-density parity-check codes (LDPCs) [1] are capacityapproaching codes which are iteratively decoded by means of belief propagation [2], also known as sum-product decoding, or by its suboptimal approximation, the min-sum algorithm [3]. Exact sum-product LDPC check node decoding is quite complex. The min-sum approximation is easily implemented but results in performance losses near 0.5-1 dB [4], [5].
A correction factor expressing the difference between minsum and sum-product decoding at the check node was found for degree 3 check nodes [6]. This correction factor may be extended to higher degree check nodes by subdividing the check node into degree 3 nodes. A single-step approximation to this degree 3 correction factor presented in [7] and [4] as the modified min-sum algorithm offered near-sum-product results.
The disadvantage of subdividing higher degree check nodes into degree 3 nodes is that the correction factor logic must be implemented multiple times for each extrinsic message, instead of once. This replication motivates the development of a correction factor approximation matched to the check node degree dc. A degree-matched check node approximation would only be applied once per extrinsic message at each check node, reducing implementation complexity.
This paper is organized in the following manner. Section II discusses sum-product decoding at the check nodes and the min-sum approximation. A correction factor between sumproduct decoding and min-sum decoding for degree 3 check nodes is examined, and existing approximations to that correction factor, known as modified min-sum algorithms, are

discussed in Section III. In Section IV, a maximal value for the general correction factor for any degree check node is derived after examination of the general expression for check node decoding. Based on this maximal correction factor value, two degree-matched check node approximations incorporating a degree-matched correction factor are presented. One of these approximations has very low complexity. Simulations for both degree-matched check node approximations are presented in Section V, and compared with the sum-product, min-sum and modified min-sum algorithm performance. Section VI concludes this paper.

The one-step degree-matched check node approximation is simpler than either the two-step degree-matched approximation or the degree 3 check node approximations of Section III, as it eliminates the constraint on the distance between the first and second minimum LLR magnitudes. However, this constraint ensures that the approximation is not applied when the actual correction factor is zero, and its elimination means the approximation may be applied when it is not needed, decreasing the extrinsic output below that of sum-product decoding. However, the reduced complexity of the one-step approximation, with only a single threshold, is significant, justifying its consideration.

V. SIMULATIONS
Simulation results are presented for the five check node decoding algorithms described in this paper. Results are shown for both a rate 0.5 length 1024 regular (3,6) and a rate 0.5 length 1000 irregular LDPC. BPSK transmission of the allzeros codeword over an AWGN channel with noise variance N0/2 was used, and each noisy received codeword was decoded by all five check node decoding algorithms; variable node decoding used sum-product decoding for all five algorithms.
Decoding performance results are shown for bit error rate (BER) vs signal-to-noise ratio (SNR). The SNR in dB is defined as SNR=10 log10(Eb/N0), where Eb = Es/R. Each simulation point counted 100 block errors. A maximum of 64 decoding iterations was used for decoding each transmitted codeword. All simulations use floating point precision.
The modified min-sum algorithm utilizes the degree 3 correction factor approximation of equation 7. The constraint that |λmin| > 0.5 for correction factor use is added to prevent signswitching. This correction factor is applied only once at each check node, rather than dc −2 times to get a final higher degree value. Implemented in this way, the degree 3 approximation has the same complexity as the degree-matched check node approximation; however, some degradation of performance will result due to its application at check nodes of degree > 3.

A (3,6) regular LDPC has all variable nodes of degree 3 and all check nodes of degree 6. Sum-product decoding is exact, and optimal, on cycle-free graphs. However, a finite-length block code when decoded iteratively will have cycles which interfere with the sum-product decoding process, degrading its performance, and introducing sub-optimality. Slight performance improvement is possible for sum-product decoding on graphs with cycles.
The one-step degree-matched approximation shows slight performance loss compared to the two-step approximation, but still achieves sum-product-decoding results, with minimal complexity. The modified-min-sum algorithm loses 0.1 dB over sum-product decoding when applied only once at each check node. The min-sum algorithm costs 0.4 dB in performance.

VI. CONCLUSIONS
In summary, we examined several LDPC check node decoding approximations to recoup some of the loss of the minsum approximation, yet keep its simplicity. The degree 3 check node approximation presented in [7] and [4] provides good performance at higher degree check nodes, but must be applied dc − 2 times at each check node. When applied only once per node, this approximation sees some loss.
The degree-matched check node approximation was developed in this paper to provide an approximation tailored

to the individual check node degree. This approximation is thus applied only once at each check node. Two versions of
this approximation were presented, a two-step degree-matched
approximation, and a simpler one-step approximation without the additional constraint of checking the difference between
the two minimum magnitude input LLRs. Both degree-matched
approximations provide sum-product decoding results, with slight improvement for the two-step approximation. The one-
step approximation, however, is much simpler even than the degree 3 approximations, and recoups nearly all the loss of the
min-sum algorithm for minimal additional cost in complexity.
