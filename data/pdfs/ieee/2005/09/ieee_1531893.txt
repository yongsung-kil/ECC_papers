I. INTRODUCTION

The powerful error-correcting properties of turbo-like codes

make them an attractive choice for all kinds of communication

systems. However, in high data rate applications, there is a

problem connected with the generally high latency of the

iterative decoder. The obvious solution for a low-latency

decoder is a highly parallelized structure. Provided a certain

degree of parallelism, one of the most important problems

to solve becomes the handling of memory access. In fact, if

more concurrent accesses are attempted at the same time, the

collision must be solved by serializing the colliding accesses,

and thus spoiling parallelism.

The iterative decoder is composed by two constituent de-
coders exchanging information about a set of Ä log-likelihood

ratios (LLRs) through several iterations. For LDPCs, the two

constituent decoders correspond to the variable node (VN) and

check node (CN) decoders. Suppose each constituent decoder
is constituted by È processors working in parallel. Every processor is in charge for the update of a subset of Å Ä È

LLRs. Thus, a given processor first reads the current LLR

values, then elaborates on it and finally writes the updated LLR

value in the same memory cell it read it from. We will suppose

that the scheduling of reading operations and of writing of the

corresponding updates coincide, thus allowing us to consider

reading and writing as equivalent from the point of the view
of the memory access. So, for Å consecutive time instants,

each processor attempts an access to a memory cell, where

the current value of an LLR is stored.

We consider the information flowing left to right from one

constituent decoder to the other, since the information flow

from right to left is obtainable by simply reverting the blocks.

We call left (right) spatial clustering the assignment of the

LLRs to the processors on the left (right) side, and left (right)

temporal clustering the assignment of the LLRs to the time

instants on the left (right) side. We order the LLRs such that

III. A DUAL PROBLEM: SCHEDULING FUNCTIONS

The problem could then be formulated as an instance of graph coloring. A dual problem can arise while facing the implementation, i.e., set the mapping function to be equal to the left spatial clustering and find the scheduling function that allows for collision-free implementations. This problem turns out to have exactly the same structure of the one solved in [3] and, as a consequence, it is still a graph coloring problem, which can be solved for example by making use of the annealing procedure.
