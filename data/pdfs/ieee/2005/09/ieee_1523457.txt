AbstractвҖ” We investigate a new class of codes which is in some senses a hybrid between LDPC codes and turbo-codes. We show that when the parameters of this class are well chosen they have very good iterative decoding performances and at the same time a minimum distance which is typically linear in the code length.
I. INTRODUCTION
It is now well known that several different coding schemes such as turbo-codes, low-density parity check codes, repeataccumulate codes or product codes are able to operate with very low error probability after iterative decoding at rates extremely close to the Shannon limit. However, it seems that in all cases, this comes at the cost of having a poor minimum distance, which itself gives a rather high error floor. This turns out to be quite apparent in the schemes which are the closest to the Shannon limit (either irregular LDPC codes [6] or irregular turbo-codes [3]). There are code families which are asymptotically good (in the sense of having minimum distance linear in the length of the code) which can be successfully iteratively decoded, for instance LDPC codes with a small enough fraction of edges of left degree 2 [7], [8] or Tanner codes built from local codes of distance greater than 2 [4], but for all the schemes which have been proposed up to now, they are only able to operate with vanishing error probability after iterative decoding at rates, which are much further from the Shannon limit than the aforementioned constructions.
This raises the issue whether or not asymptotically good codes can approach the capacity with low complexity iterative decoding. Our aim in this article is to explore code families other than LDPC codes in order to have asymptotically good codes which could approach the Shannon capacity. For this purpose, we will study a class of codes which not only contains the class of LDPC codes (and more generally Tanner codes) and a code family related to turbo-codes. This class of codes is constructed in such a way as to allow irregular variable node degrees in order to permit further optimization of the family.
We will define this family in Section II. Since we are interested in the members of the family which are asymptotically good it remains to find conditions which ensure such a property. This has be partially done in [9]. What we have done here, is to explore a particular code family which is proposed in [9] which we call Tail-biting Trellis Low-Density Parity

Check codes (TLDPC) which combines several interesting features when the parameters of the member are well chosen, namely: - a very simple structure which yields low-complexity iterative decoding, - minimum distance linear in the length of the code, - good performances after iterative decoding which beat standard turbo-codes for code lengths of several thousand for instance.
We present examples of members of this class of 1/2 and 1/3 rate where this fact is striking.
Our codes are defined by using two ingredients : a base code and a bipartite graph. To avoid small weight codewords in our codes we have chosen the bipartite structure in such a way that a certain graph, what we call the graph of codewords of weight 2 contains only cycles of linear size. We will see in Section III why this fact is crucial. We have computed the average spectrum of our codes. Note that this average is taken over all bipartite graphs of a given degree distribution and satisfying an additional constraint which ensures cycles of linear size in the graph of codewords of weight 2 . We use this calculation to show that when the parameters of our family are suitably chosen, almost all codes in this family are asymptotically good.

III. THE GRAPH OF CODEWORDS OF WEIGHT 2
An analysis of the number of codewords of a given constant weight w over the ensemble of n length codes obtained from the previous construction shows that their average aw is of order в„Ұ (1). This behaviour comes from cycles of small size in a certain graph derived from the bits of degree 2 and codewords of weight 2 in the base code : the graph of codewords of weight 2 . It is possible to obtain a lower average of order o (1) if we restrict to the ensemble of codes for which this graph of codewords of weight 2 has no small cycles. If one insists upon having a code family where almost all members are asymptotically good, it is crucial that the average number of codewords of constant size be of order o(1).
In order to define this graph we first define Definition 1: Two bits xi and xj of the base code are equivalent if and only if the base code contains a codeword of weight 2 with these bits equal to 1 and the others to 0. The ensembles of equivalent bits of the base code are called clusters.
Definition 2: The graph of codewords of weight 2 is the graph G = (V , E ) where V is the ensemble of clusters defined above and E is the ensemble of edges eij between two different clusters vi and vj if and only if there exist two bits xk and xl of the base code belonging to the clusters vi and vj respectively which join the same degree-2 variable node.
It is straightforward to check that : Fact 1: A length l cycle in the graph of codewords of weight 2 induces a weight l codeword in the code.

The length of the smallest cycle in G is an upper bound on the code minimum distance. An important condition to respect while constructing asymptotically good codes is to avoid the cycles with length l greater than a fraction Оҙ of the code length n (l вүҘ Оҙn) in the graph of codewords of weight 2 .
We notice that this graph of codewords of weight 2 is nothing but a slight generalization of a graph which has been considered in [8] in the case of LDPC code ensembles.

For family A we have chosen the bipartite graph so as to avoid cycles of sub-linear size in the graph of codewords of weight 2 in the following way. We first choose a random cyclic permutation of length m/2 (which is the number of sections in the trellis defined previously) to rearrange the sections in a cyclic fashion. We view this cyclic permutation as a length m/2 cycle, the vertices being associated to the sections of the trellis. We then choose in a uniform way О»Лң2n edges of this cycle (which will correspond to the choices made for degree

In other words if we choose an edge connecting a section i to a section j, there is a degree 2 bit connecting to these two sections of the base code in the bipartite graph defining our code. The exact choice of the position of the bit of degree 2 in the section (either the first or the second one) does not matter, and is chosen uniformly at random. This gives the structure of the bipartite graph for degree 2 bits. For the other degrees, the positions are just chosen at random at the places left in the sections of the base code. This choice ensures that the graph of codewords of weight 2 is a length n cycle in the case О»2 = 1 and a forest in the case О»2 < 1.
For family B it is not always possible to avoid cycles of sub-linear size for any О»2 value. This is actually impossible if О»2 > 2/3. We have chosen О»2 = 1/2 in this case, because it is close to 2/3 and it is possible to choose the structure of the graph of codewords of weight 2 in rather simple way so as to simplify the analysis of the average spectrum. More precisely, we choose the length of the base code to be a multiple of 24. Therefore m/3, which is the number of sections, is a multiple of 8. We choose randomly 3m/24 = m/8 sections of the base code and arrange them in a cyclic way as for the previous family. We view this cycle again as a graph. We then choose m/8 vertices of V randomly that are going to be of degree 2 and match them with the edges of the cycle. Each edge ij matched with a vertex x of V (i and j are the indices of the sections in the base code) of this cycle defines two edges of the bipartite graph G leaving x and going to one bit of section i and one bit of section j. Next, we choose m/24 remaining sections at random which we will be matched each with 3 remaining sections. Say for instance, we match section i with section j,k and l. Each matching of this kind defines 6 other edges of the base code incident to degree 2 vertices. We associate to j,k,l, each to another degree 2 vertex in V , say x, y and z respectively. Let a, a + 1, a + 2 be the positions of the base code which belong to section i. Then there will be edges between x and a, x and a random position in section i, y and a + 1, y and a random position in section j, z and a + 2 and z and a random position in section j. This defines all the edges of left degree 2 in G. We then choose the remaining edges so as to respect the degree distribution. Note that the structure of the graph of codewords of weight 2 is a cycle of length m/8, m/24 isolated points and the union of m/24 3-stars.

VII. CONCLUSION
We have presented a new code family in this article which we call tail-biting LDPC codes (which are in a sense a tailbiting version of LDPC codes). The point of this study was to try to approach the capacity with codes with minimum distance

linear in the length and an iterative decoding algorithm of low complexity. The class of codes constructed at rates 1/2 and 1/3 displays for instance better performances than standard parallel turbo-codes already at lengths of several thousand.
We wish to point out that we could allow in the class of codes constructed in this article bits of degree 1. If we did this we would have to change the definition of the graph of codewords of weight 2 in order to take into account codewords of the base code which contain at most 2 вҖң1вҖқs at positions of degree вүҘ 2, and any number of вҖң1вҖқs at positions of degree 1. The study of such codes would be more involved and requires for instance a very careful choice of the positions of the bits of degree 1 and of the structure of the bipartite graph G. This is beyond the scope of this paper. But we notice that a preliminary study on the exit-charts strongly suggests that by allowing bits of degree 1 one is able to get extremely close to capacity.
