Construction of Low Density Parity Check Codes Using Quadratic Congruential Sequences
Wenming Liu Guangxi Zhu Yongqiang Deng
Department of Electronics and Information Engineering Huazhong University of Science & Technology Wuhan, China
liuwenming1977@hotmail.com gxzhu@mail.hust.edu.cn tsnower@263.net

Abstract—In this paper a method using quadratic congruential sequences to design regular LDPC code is proposed. The main advantage of this method is that the structure of the encoder can be generated using a determined recursion, rather than having to store the encoder structure graph in memory, which facilitates hardware implementation. Simulation results show that these codes provide almost the same performance of a constrained pseudorandom construction that explicitly avoids cycles of length less than or equal to four.
Keywords- quadratic congruential sequences, regular matrix, cycles of length four, shortened sequence.
I. INTRODUCTION
During the past decade a number of substantial improvements have been made in error control coding. Beginning with the introduction of turbo codes in 1993, followed by renewed interest in low-density parity check (LDPC) codes, performance on the additive white Gaussian noise channel is very close to capacity. The exact structure of the parity-check matrix needs to be stored at the receiver and, since the graphs are constructed at random, the memory required to store the structure of the graph using lookup tables may be prohibitive in practical applications. One area of focus is reducing the randomness in the construction of the parity check matrix, H [1]. Therefore, it is very desirable to be able to generate the graph structure algebraically, such that these lookup tables can be avoided.
In this paper, we report an advancement to this technique by defining H with some integers we call a key. From the key we construct a single permutation vector generated using a recursion as the defining core of H. Using the key, the parity check matrix can be generated on the fly in the decoder and encoder rather than having to store the graph connections in memory, which facilitates hardware implementation of the decoder.
The paper is organized as follows. In section Ⅱ, we discuss the graph structure of these codes very briefly. In section Ⅲ, we discuss a particular algebraic construction and give an algorithm to produce a shortened sequence of length S that is not power of 2. In section Ⅴ, we show simulation results to

confirm that these designs provide good bit error rate performance.

The parity check matrix of a regular (N , K ) LDPC code is sparse, with λ ones in each column and ρ = λ /(1− K / N ) ones in each row. A (λ, ρ) regular LDPC code has a graphical representation in which each variable node is connected to λ edges and each check nodes to ρ edges. We first associate with each check node ρ connections, called check connections, and with each bit node λ connections, called bit connections, as shown in Fig.1 for λ = 3 and ρ = 6 . Therefore, there are a total of M = kλ bit and check connections. Let the p th bit connection on the left be connected to the Ap th check connection on the right. A randomly constructed LDPC code is formed by randomly connecting the bit connections on the left with the check connections on the right or, equivalently, when the sequence A0 , A1,", Ap ,", AM −1 is randomly generated sequence. It should be noted that p th bit connection on the left

It is important to note this relationship between bit (or check) connections and actual bit (or check) nodes, since both are used in this paper.
