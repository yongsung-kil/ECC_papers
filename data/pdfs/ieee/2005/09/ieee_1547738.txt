For rates higher than 1/2 the process proposed consists in removing some rows of the original matrix to reach rates greater than 1/2, as in Figure 2-c.
The number of memories used to store information bits depends on the parallelism desired. If a high parallelism is desired, the number required to implement a BIBD with high rate (the highest parallelism is obtained using a memory for any column of the top matrix) can be prohibitive. For example, for a bit degree of 13 the system would require 157 memories by using a kind of BIBD called projective, whose main parameters
are n2+n+1, n+1, 1.
In order to solve this problem, BIBD is cloned and put side by side with itself. The only difference between both parts is the way their pseudo-random primitive generators are designed. The primitive generators of the BIBD on the right are exactly the same as their equivalents on the left, but the init value (explained in the next section) is changed depending on the row and column. This ensures that the new structure is still lengthfour cycle free with the bit degree desired and high rate. An example is shown in Figure 2-d, where any of the cloned matrices are the same as the one in Figure 2-c, once the shadowed elements are removed.

A. TOP Level Architecture
As previously mentioned, in order to avoid length-four cycles with a matrix of minimum dimension for a given bit degree, the approach presented makes use of BIBDs (Balanced Incomplete Block Designs) [11]. BIBDs combine a number of points to form groups. They are defined by (at least) three parameters called v, r, 1, where v is the total number of points, r is the number of blocks which a point is in (i.e. the bit degree in LDPC terms) and A defines the number of blocks in which each pair of points are together. Translating these parameters to the matrix field, v represents the number of columns, r is the number of Is per column and l=1 will determine the absence of length-four cycles: a value of 1 ensures that any pair of 1 s will be in only one row and therefore there are no length-four cycles.
The BIBD based matrices chosen are rate 1/2 matrices, so, for rate 1/2 the construction is immediate, turning the BIBD into a matrix. For example, Figure 2-a shows a BIBD(7,3, 1).

B. BOTTOMLevel Architecture

The proposed method is based on primitive interleavers (we will call them primitive generators) [8]. These interleavers have the desirable property of being very simple to design, fast and with low area cost and, from the point of view of BER performance, they perform only slightly worse than a completely random one in this kind of systems. III. BER PERFORMANCE
The results obtained in the original eIRA paper [6] will be taken as a point of reference. For rate 0.5 example of [6], the frame size used (4018, 2009) is approximately the same as the one reported there (4000, 2000). The reason for the slight frame size difference is the use of a prime N and a BIBD (49, 7, 1). In this particular case, N was set to 41 as this is the prime value that provides the frame length closest to the desired: 41 x 49 = 2009.
We began using the same check and bit degrees as the original eIRA because they have been demonstrated to be optimal using differential evolution. The proposed bit degree for
HI matrix was 58% of information bits with degree 3 and 42%
with degree 7. On the other hand roots of primitive generators were randomly elected. Results can be seen in Figure 3 labeled as eIRA BIBD (58%w3, 42%w7, rr) where rr means random roots. Performance is clearly improved electing roots following previously mentioned criteria, based on separating widely the ones on columns and pair of columns. The use of these distributed roots and its BER performance is labeled as eIRA BIBD (58%w3, O%w4, 42%w7, dr), where dr means distributed roots, in Figure 3.

There are three parameters that define the pseudo-random sequence, the root which is the difference between one value and the next or previous one, N which is the maximum value, and the total number of values where the sequence is complete and the init_value which is the initial value io. How these parameters will be chosen is important in the BER performance of the system and will be explained next.
The proposed design suggests an N which is prime. This fact allows the use of as many different roots as N-1 (a valid root is a root that forms complete N sequences). Moreover, N is chosen to satisfy: framesize = N * column _ number . There are
enough prime numbers, sufficiently widely distributed so that any frame size can be approximated.
The idea of having several roots increments the randomness of the whole parity check matrix because despite using the same generator the sequences obtained are different. On the other hand, following recommendations in [6][12], in order to reduce low-weight codewords and near codewords, for lower degree variables, which are mainly responsible of error floors, roots are selected so that ones of columns and pairs of columns are widely separated. Moreover, chosen root is also used as init_value in order to begin in a different number for any submatrix. Roots selected with mentioned methodology will be called distributed roots in next section of results.

IV. CONCLUSIONS
In this paper a new architecture for designing eIRAs is described. A design based on hierarchical matrices is proposed, combining the deterministic structure of BIBD (x, y, 1) designs (avoiding length-four cycles) with the good BER properties of pseudo-random constructions. A study of the influence of the primitive generators is presented, defining a design method based on using optimal roots and modifying the degree of only a few systematic bits. With this, BER performance is very similar to original eIRA proposal but with a more suitable topology for implementation.
