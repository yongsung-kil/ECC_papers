An (8158,7136) Low-Density Parity-Check Encoder

L. Miles, J. Gambles, G. Maki, W. Ryan and S. Whitaker Center for Advanced Microelectronics and Biomolecular Research University of Idaho, 721 Lochsa Street # 8, Post Falls, Idaho 83854
Electrical and Computer Engineering University of Arizona, 1230 E. Speedway, Bldg. 104, Tucson, AZ 85721

Abstract-Low-density parity-check codes achieve coding performance which approaches the Shannon limit. Based on a novel method for deriving regular quasi-cyclic sub-codes, a radiation tolerant encoder was implemented in 0.25 CMOS. Use of generator polynomial reconstruction, partial product multiplication and functional sharing in the parity register results in a highly efficient design. Only 1,492 flip flops along with a programmable 21-bit look-ahead scheme are used to achieve a 1 Gb/s data throughput. A comparable two-stage encoder requires 8,176 flip flops.

A low-density parity-check code (LDPC) is a linear block

code for which the parity-check matrix has a low density

of 1’s. The sparseness of an LDPC code’s matrix makes it

amenable to a near-optimal decoding procedure called belief

propagation. LDPC codes were first introduced by Gallager

[1] in 1962, but remained largely unexplored for over three

decades. They were re-invented by MacKay [2] and others

in the mid-1990’s at which time they were shown capable of

operation near the celebrated Shannon limit on many standard

channel models.

Virtually all LDPC code design procedures involve specifi-

cation of the matrix after which an encoding procedure

must be devised. A generator matrix may be obtained

from via Gauss-Jordan elimination, but the resulting en-

coding operation via

is still too complex for long

codes. To circumvent this, a technique in which encoding

is performed directly from has been proposed in [3].

Alternatively, cyclic and quasi-cyclic (QC) LDPC codes have

been proposed which lend themselves to lower-complexity

shift-register based encoder implementations [4], [5]. The low-

complexity advantage derives from the fact that the matrix

is composed of circulant submatrices. The code whose encoder

implementation is described herein falls in the QC class.

The radiation tolerant [6], 1 Gb/s encoding chip presented

in this paper actually implements two low-density parity-check

codes. The first is a (4088,3360) code (4K) which is shortened

from a (4095,3367) cyclic code. The second is a quasi-cyclic

(8158,7136) code (8K). Also included on the chip is a CCSDS

[7] standard randomizer as well as a CCSDS standard attached

synchronization marker (ASM) section. The architecture for

the 8K parity generator and the derivation of the sub-code

which enables this efficient architecture will be described in

detail.

IV. ENCODER ARCHITECTURE

An overall block diagram for the chip is shown in Figure 1. Message data is input to the encoder block 16 bits at a time on the data input bus (DI). Data to be encoded is framed by the encode enable (EE) signal. The (8158,7136) code is one of two codes on this chip and is selected by holding the select 8K (S8) input high. The message data and parity output by the encoder block will be randomized if randomize enable (RE) is asserted. An optional internal frame marker will precede the message data if the marker enable (ME) is asserted. Data

Fig. 1. Block diagram for the combined 4K and 8K encoder.
can pass through the chip unencoded using a bypass mode if bypass enable (BE) is asserted. Message data output on the data output bus (DO) is framed by output enable (OE). Valid marker (VM) will flag, if correct, an external frame marker added during a bypass operation.
A 1 Gb/s bit rate can be sustained using a 64 MHz system clock. 446 sixteen-bit data words are input to the chip and passed out of the chip with nine clock cycles of latency when no frame marker is inserted. When the two-word frame marker is prepended, data has an eleven clock cycle latency. 64 sixteen-bit words follow, of which 63 are all parity bits. The final 16-bit output word contains the last 14 bits of parity with 0’s appended.

A 4-bit circulant matrix counter (CMC[3:0]) maintains a count of which 511-bit block of message data is present and provides the ROM address to select the appropriate initial row coefficients. The 28 generator polynomials could then, in turn, be created by right-shifting the initial coefficients in the polynomial generator register. This would require, however, 1,022 flip flops to generate and store the two generator polynomials. By observing that the same mathematics can result from simply left-shifting the parity register before applying the next 21 bits of data to the MAC module, the two 511-bit generator polynomial registers can be eliminated.
A general multiply-accumulate block was used rather than a dedicated look-ahead block [9] to allow arbitrary generator polynomials to be programmed by the coefficient ROM. The MAC operates on the message data n=21 bits at a time. A straight forward design of the MAC would result in a linear delay path of n+1=22 levels. Since propagation through the MAC was the critical timing path for the 8K encoder block, a program was written to transform the logic into a tree form whose logic depth is given by the next integer greater than
. This reduces the delay path to 6 levels of logic, greatly improving the speed of the chip.

VI. ARCHITECTURAL COMPARISONS
The elimination of and functional sharing of flip flops results in a very efficient design. There are a total of 2,230 flip flops used in this design, 728 of which are in the parity register for the 4K code. An additional 10 flip flops are in the randomizer and marker logic. This leaves 1,492 flip flops for the 8K encoder which includes all pipe registers into and out of the chip as well as all control and state machine flip flops.
Four encoding schemes for quasi-cyclic LDPC codes were compared in [10]. Two of these schemes were high throughput encoders requiring 511 clock cycles to encode the message bits for an (8176,7154) LDPC code. These two compare directly in throughput to the architecture described in this paper. The shift-register-adder-accumulator (SRAA) encoder in [10] requires 28,616 flip flops and the two-stage encoder requires 8,176 flip flops for the encoding. These numbers exclude control and pipe registers. The architecture presented in this paper utilizes less than 5.2% of the flip flops required by the SRAA architecture and less than 18.3% of the flip flops required by the two-stage encoder to achieve similar throughput.
The slowest architecture presented in [10] is an SRAA encoder which requires 7,154 clock cycles, 1/14 of the faster rate, to encode the message bits. The advantage of the slowest architecture is a reduction to just 2,044 flip flops. The architecture presented in this paper uses less than 73% of the flip flips required by the smallest architecture of [10] while achieving 14 times the data rate. Radiation Tolerant (RT) chip operation is achieved through

the application of Radiation Hardness By Design (RHBD)

techniques. These include Single Event Upset (SEU) im-

mune flip flop designs utilizing data storage redundancy with

conflict-free fault detection and correction capability inter-

nal to each memory cell [12]. To enable the ‘single event’

assumption in design of these cells, nodes in the flip flop

cells, which if simultaneously upset would latch the effect of

the upset, were separated by large enough spacings to avoid

this potential problem. The separation between cells was also

increased to insure that the minimum spacing required to avoid

simultaneous upset was maintained between cells and rows.

Elimination of the capture of Single Event Transients

(SETs) occurring in the combinational logic that may arrive

at a flip flop input coincident with the storage triggering clock

edge is achieved through the combination of an RT design

characteristic providing redundant fault tolerant inputs to the

flip flop and a delay element connected between the combina-

tional logic and one of the redundant inputs. With this design a

SET dissipates at the non-delayed input before it arrives at the

delayed input, thus insuring that the transient is not presented

to both redundant fault tolerant inputs simultaneously and the

effect of the transient is not stored [13], [14], [15].

The RT library cells were drawn with continuous guard

bars through the middle of the cells. Filler cells and end cap

cells were used to insure a continuous guard ring around and

through the rows of cells. This guard ring eliminates Single

Event Latch-up (SEL) [16].
