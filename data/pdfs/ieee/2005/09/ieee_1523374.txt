Abstract— In this paper, we apply two improved min-sum algorithms, the normalized and offset min-sum algorithms, to the decoding of irregular LDPC codes. We show that the behavior of the two algorithms in decoding irregular LDPC codes is different from that in decoding regular LDPC codes, due to the existence of bit nodes of degree two. We analyze and give explanations to the difference, and propose approaches to improve the performance of the two algorithms.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] can be decoded with a variety of algorithms. The best known decoding algorithm was first devised by Gallager, but can be unified with many other algorithms for decoding, detection or inference into a general algorithm called belief propagation (BP). A detailed description of BP algorithms for decoding and inference, operating on a factor graph, can be found in [2]. We can simplify the check node processing of the BP algorithm to obtain the “min-sum” algorithm [4], which is the same as the algorithm described in [3], or the “maxproduct” algorithm described in [5]. The min-sum algorithm greatly reduces the decoding complexity for implementation, but may cause degradation in performance. Compared to the BP algorithm, although the min-sum algorithm has little degradation in performance for short code lengths (< 200), the degradation can vary from several tenths of dB to one dB for long code lengths. For a regular (3, 6) LDPC code over additive white Gaussian noise (AWGN) channel, the penalty due to the simplification can be as big as 0.6 dB in terms of signal-to-noise (SNR) ratio.
In [6], [7], [8], two improved versions of the min-sum algorithm are proposed. By means of normalization or offset, the improved min-sum algorithms can reclaim the SNR penalty due to the simplified check node processing in minsum decoding with a very limited increase of complexity. Density evolution algorithms are derived in [7] to optimize the normalization factor and the offset value associated with an ensemble of LDPC codes. Besides their low complexity, the improved min-sum algorithms have some other advantages over the BP algorithm. First, for the AWGN channel, the BP algorithm needs to know the channel state information, while the normalized and offset min-sum algorithms need not. In

addition, the normalized and offset min-sum algorithms are less subject to error floors [8], even with uniform quantization and finite quantization bits, say 5 or 6. For a (3,6) regular LDPC code of length 8000 and rate 1/2, it has been shown that the performance of the improved min-sum algorithms is about 0.05 to 0.1 dB away from the performance of the BP algorithm. For short LDPC codes, the improved min-sum algorithms can perform even better than the BP algorithm at high SNR values.
With deliberately selected degree distributions, irregular LDPC codes demonstrate a better waterfall region than regular ones. In this paper, we extend the improved min-sum algorithms for the purpose of decoding irregular LDPC codes. We show via simulation that the normalized min-sum algorithm has poor block error performance in decoding irregular codes, and often delivers error blocks with only 1 or 2 uncorrected error bits. The offset min-sum algorithm performs much better than the normalized min-sum algorithm, but still can output uncorrected codewords of low weights. We give analysis of these behaviors, and propose approaches to improve the performance of the offset and normalized min-sum algorithms when decoding irregular LDPC codes.

III. PERFORMANCE OF IMPROVED MIN-SUM DECODING FOR IRREGULAR LDPC CODES
Degree distributions for good irregular LDPC codes can be optimized with density evolution or some other methods. Generally, a good irregular LDPC code has a large portion of variable nodes of degree two, which contributes to possible small minimum distance of irregular LDPC codes. However, low minimum distance is often not the only obstacle to achieving low error-floors with short block length irregular codes under iterative decoding. The degree distribution polynomials of the code are λ(x) = 0.22276x + 0.19042x2 + 0.08605x5 + 0.16084x6+0.33993x19 with respect to the variable nodes and ρ(x) = 0.25x7 + 0.75x8 with respect to the check nodes. We are interested in examining the performance of different decoding techniques both in transition and error floor regions. The error floor associated with this code is in fact due to low weight trapping set events rather than low weight codeword failures. For simplicity, we use one scaling factor or offset value for all check nodes in the normalization min-sum algorithm and the offset min-sum algorithm, respectively. In our case, we set α = 1.3 and β = 0.2 based on density evolution results. From Figure 2, we can see that the min-sum algorithm has about 1.1 dB performance gap compared to the BP algorithm. Although the normalized min-sum algorithm improves the biterror performance, the block error performance is very bad. Fortunately, the offset min-sum algorithm still performs well and it is only about 0.15 dB away from the BP algorithm in the waterfall region.

We can see that most uncorrected blocks contain less than 10 error bits. For the offset min-sum algorithm, there are also many error blocks with a small number of error bits, although they are much less frequent than the normalized minsum algorithm. Second, we observe that code bits of degree 2 or 3 are more likely to contain uncorrected bits than nodes with higher degree. For example, for the normalized min-sum algorithm at SN R = 1.6 dB, about 94.5% bit errors occur at bit positions of degree 2 and 5.4% occur at bit positions of degree 3. Finally, for both normalized and offset min-sum algorithms, we observe that the decoding can reach a steady state. In other words, after a certain number of iterations, the messages as well as the number of bit errors remain unchanged. This behavior is different from that observed in decoding regular LDPC codes with the two improved min-sum algorithms, in which the messages usually do not converge to fixed values.

In reality, the situation is much more complicated. The isolated subgraph may have more bit nodes involved, and may not be a cycle. Figure 4(b) depicts a real case identified in decoding the (4608, 2304) irregular code with the offset min-sum algorithm. In the steady state, messages entering the subgraph are all large positive numbers and have no effect on the messages passing around in the subgraph. For the top bit node, the two incoming messages are 0.06 and 0.21 in the steady state, which can not provide enough a posteriori information along with the received value to make a correct hard decision.
The isolated subgraph in the Tanner graph of an irregular code accounts for error blocks of low weight. When the update of messages in the subgraph cannot be helped by outside messages, there may be some small number of uncorrected bits in the subgraph. This can also be observed in decoding irregular LDPC codes with BP algorithm or min-sum algorithm. In addition, since bit nodes of degree 2 or 3 are more prone to get involved in isolated subgraphs, they are more possible to be in error.

If the incoming messages to Figure 4(a) or (b) are very large, then the normalized and offset min-sum decoding in fact produce underestimated messages. Of course, on average, normalization or offsetting improves the “quality” of the messages by the min-sum algorithm. However, with very large incoming messages, the min-sum algorithm is equivalent to the BP algorithm in check node processing. Therefore, both normalization and offset will make updated messages in the subgraph underestimated.
Some measures can be taken to improve the performance of the normalized or offset min-sum algorithms in irregular

LDPC decoding. In the case of the offset min-sum algorithm, if the decoding fails to obtain a legitimate codeword after 200 iterations we can set the offset to zero and run for additional 50 iterations. This technique often breaks the steady state by allowing the magnitude of the underestimated messages to increase. A similar method can be applied to the normalized min-sum algorithm. However, we have found that a better improvement of the normalized min-sum algorithm is to normalization by 1.0 (no normalization) for messages to bit nodes of degree 2, by 2.0 for messages to bit nodes of degree 20, and by 1.3 for all other messages. Figure 5 shows the block error performance of the improved offset min-sum and normalized min-sum algorithms. The improved normalized min-sum algorithm provides much better block error performance than the original normalized min-sum algorithm. The improved offset min-sum algorithm lowers the error floor of the block error rate. It performs even slightly better than BP decoding in the error floor region with the same maximum iteration numbers. This is not surprising, however, since the BP decoding is not optimum for LDPC codes of finite length.

IV. CONCLUSIONS
In this paper, we have demonstrated that normalized and offset min-sum algorithms may induce fixed states in isolated subgraphs due to underestimation of message values that are passed to degree-2 bit nodes. The steady-state subgraphs dramatically degrade the block error performance of the normalized min-sum algorithm, and also have some negative effect on the offset min-sum algorithm. We proposed measures to improve improve the error floors of the normalized and offset min-sum algorithms.
