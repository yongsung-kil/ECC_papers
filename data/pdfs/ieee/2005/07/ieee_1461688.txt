Decreasing the Complexity of LDPC Iterative Decoders
Gabriella Bosco, Member, IEEE, Guido Montorsi, Senior Member, IEEE, and Sergio Benedetto, Fellow, IEEE

Abstract— In this letter we analyze the performance of a lowdensity parity-check iterative decoder which makes use of an additional buffer at its input, whose function is to decrease the overall complexity of the decoding circuit. We propose a semianalytical technique that permits the evaluation of the optimum buffer length for each analyzed code.

I. INTRODUCTION
I T WAS recently shown that Low-Density Parity-Check (LDPC) codes, first discovered by Gallager [1] in the early 1960s, can achieve a remarkable performance with iterative decoding that is very close to the Shannon limit [2], [3]. Consequently, these codes have become strong competitors to turbo codes [4] for error control in many communication and digital storage systems where high reliability is required.
All iterative decoding algorithms for LDPC codes can be viewed as an instance of the belief propagation algorithm working on the bipartite graph describing the code [5], where messages are exchanged between nodes connected by an edge. The computational capability of the decoder is usually matched to the maximum required number of iterations, which for LDPC codes is quite large. To save energy, the iterative decoding process can be stopped whenever the decoded sequence is a valid codeword. A drawback of this method is the unused processing capability between the stopped iteration and the start of the iterations of the next block. This inefficiency can be solved by the use of an additional buffer at the input of the iterative decoder [6], whose task is to support some additional iterations on demand.
In this letter we present a complete analysis of the performance of LDPC iterative decoders employing an additional buffer at its input, resorting in a semi-analytical technique to optimize the receiver buffer length.

The available number of iterations nB at decoding step n (i.e., when decoding the n-th received codeword) depends on the level of emptiness of the buffer. The minimum and maximum number of available iterations are respectively equal to nmin = N · T /TD and nmax = (NB − N ) · T /TD . The decoding process ends when either a codeword is found (by syndrome evaluation) or the maximum number of iterations is reached. We define the state of the buffer at decoding step n as σn ∈ {Snmin , ..., Snmax }, where Si = {nB = i}. A fundamental property is that σn only depends on the state of the buffer at step n-1, so that the time evolution of the states of the buffer can be treated as a discrete-time Markov chain [7].

V. CONCLUSIONS
In this letter we presented a semi-analytical technique to evaluate the performance of LDPC iterative decoders employing an additional buffer at its input, which permits the optimum receiver buffer length to be obtained for a given code. The technique and the analysis presented in this letter can be easily extended to turbo code iterative decoders.
