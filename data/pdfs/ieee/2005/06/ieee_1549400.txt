Abstract- An algebraic method for constructing structured regular and irregular LDPC codes with girth at least six is presented. The parity check matrices of the proposed codes are arrays of permutation matrices obtained by decomposing Euclidean geometries. Masking is applied to the resultant parity check matrices to yield codes with various degree distributions. The proposed codes perform well over AWGN channels and binary erasure channels when decoded iteratively.
I. INTRODUCTION
Low-density parity-check (LDPC) codes, combined with iterative decoding, are widely recognized as an effective coding scheme to combat noise in a wide variety of channels. Various methods for constructing LDPC codes have been proposed. Based on these methods of construction, LDPC codes can be divided into two categories: (1) random (or random-like) codes; and (2) structured codes. Although random LDPC codes generally perform better than structured codes in the waterfall region, they usually suffer from error-floor. Furthermore, random LDPC codes lack sufficient structure to allow for simple encoding.
In this paper, we propose a construction method for structured LDPC codes based on Euclidean geometries. Unlike previous constructions of LDPC codes which are typically designed specifically for good performance over either the AWGN channel or the the binary erasure channel, our codes are designed to perform well for the AWGN and/or the binary erasure channels. This is due to the fact that their structure allows us to lower bound their girths, minimum distances, and minimum stopping set sizes in a way that guarantees good performance for both classes of channels.
II. LDPC CODES
A binary regular LDPC code [1] is defined as the null space of a sparse parity check matrix H over GF(2) with the following structural properties: (1) each row has constant weight p; (2) each column has constant weight -y; (3) no two rows (or two columns) have more than one 1-component in common; and (4) both -y and p are small compared to the code length. The matrix H is said to be (-y, p)-regular. Property (3) is referred to as the column-row (RC) constraint. The RCconstraint ensures that: (1) the Tanner graph [2] of the code is free of cycles of length 4; and (2) the minimum distance of the code is at least -y + 1 [3]. An LDPC code is said to

be irregular if its parity check matrix has multiple column weights and/or multiple row weights.
The performance of an LDPC code with iterative decoding depends on a number of code structural properties. One such property is the girth of the code which is defined as the length
of the shortest cycle in the code's Tanner graph. For a code to perform well with iterative decoding, its Tanner graph must not contain too many short cycles. The shortest cycles that affect code performance the most are cycles of length 4. Therefore in code construction, cycles of length 4 must be avoided. Many experimental results show that the error-floor of an LDPC code with iterative decoding very much depends on the minimum
distance of the code and hence it is necessary to keep the minimum distance of an LDPC code reasonably large. For an irregular code with iterative decoding, its error performance
also depends on the degree distributions of the variable- and check-nodes of its Tanner graph [4], [5].
Let V be a set of variable-nodes in the Tanner graph T of an LDPC code and S be a set of check-nodes in T that are adjacent to the variable-nodes in V, i.e., each check-node in S is connected to at least one variable-node in V. A set V of variable-nodes is called a stopping set of the Tanner graph T if each check-node in S is connected to at least two variablenodes in V. The Tanner graph T of an LDPC code may contain stopping sets of various sizes. A stopping set of minimum size, which may not be unique, is called a minimal stopping set. The performance of an LDPC code over the binary erasure channel (BEC) with iterative decoding depends on the stopping sets of its Tanner graph [6]. For good performance, the Tanner graph should not contain stopping sets of small size. It has been proved in [7] that the sizes of a minimal stopping set of Tanner graphs with girths 4, 6, and 8 of an LDPC code with constant
column weight ay are at least 2, -y + 1, and 2a, respectively. In particular, for a (-y, p)-regular LDPC code whose parity check
matrix satisfies the RC-constraint, no stopping set has size a or less. Therefore, for such code, attaining large minimum distance and large size of a minimal stopping set is guaranteed by keeping the column weight of its parity check matrix large.
