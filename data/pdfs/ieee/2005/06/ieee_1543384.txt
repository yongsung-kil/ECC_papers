A New LDPC Decoding Algorithm Aided by Segmented CRCs for Erasure Channels
Yeong-Hyeon Kwon, Mi-Kyung Oh, Student Member, IEEE,and Dong-Jo Park, Member, IEEE

Abstract— This letter introduces a new low-density paritycheck (LDPC) decoding algorithm that exploits the cyclic redundancy check (CRC) information of data segments. By using the error detection property of CRC, we can successively decode data segments in the codeword corrupted by magnetic recording channels. The key idea is that variable nodes corresponding to data segment with correct checksum are fixed to deterministic log likelihood ratio (LLR) values (or probabilities) during the LDPC iterative decoding. In this way, the performance regarding the decoding speed and codeword error rate can be improved without significant modification of the LDPC decoding structure. Moreover, the CRC information is also used for an early stopping criterion of LDPC decoding. Simulation results verify our claims.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are one type of linear block codes described by a very sparse parity check matrix (PCM), which was proposed by Gallager [1] and rediscovered by Mackay [2]. It was shown that the operation limit of the LDPC codes approaches to the Shannon limit as the density of 1’s in the PCM decreases [2]. Moreover, the LDPC codes show better performance than Reed-Solomon (RS) codes for short burst erasures that are frequent in the magnetic recording channels (MRCs) [4]. However, as the error bits in the codeword increase, it takes much time to declare successful decoding in the LDPC decoder [3], [8].
In order to reduce the decoding speed and enhance error performance, we exploit the special frame format, which has multiple cyclic redundancy checks (CRCs) [5], [6]. The multiple CRCs are used as a concatenated coding component for the magnetic channel [5] and error detection mechanism of packet fragments in the partial retransmission scheme [6]. In addition, the CRC code is used for the early termination method of codeword decoding loop [9]. In this letter, the multiple CRCs are used in the LDPC decoding to generate large message during the decoding iteration, where the large message is applied for the segments with correct CRC value. The proposed decoding method shows better convergence speed and better error performance if the undetected error probability of the employed CRC is sufficiently low.
II. FORMULATION AND IDEA DESCRIPTION
We consider the frame format introduced in [5], [6], where multiple CRCs are used for different purposes. In addition to these applications, the multiple CRCs can be used as reliability decision measure in the LDPC decoding by checking CRC values of segments, if the LDPC is employed as an error correcting code (ECC).

Fig. 1. Single data segment (upper) and multiple (Nf ) data segments and CRCs (lower)
Consider a data packet to be transmitted through a MRC, which is sequentially encoded by the CRC and the systematic LDPC encoder. Let x be the data packet of length Nx, c the LDPC codeword of length Nc, and s the systematic part of length Ns in c. It is convenient to specify a code rate Ns/Nc LDPC code with an (Nc − Ns) × Nc parity check matrix H using the Tanner graph representation. The Tanner graph is a bipartite graph, where the nodes on the bottom are associated with the variable nodes, and the nodes on the top are associated with the check nodes [3].
Figure 1 depicts the implementation of multiple CRCs in an encoded codeword in comparison to that of the single CRC construction. We note that for the purpose of detecting the packet error, the CRC code should be placed in the systematic part of the packet. By using Nf CRCs of length NCRC , the length of s is given by Ns := Nx + Nf NCRC . We will consider only Nf > 1 cases. However, we should note that the code rate of large Nf becomes low, while achieving the other advantages.
The multiple CRCs are used in two ways for our decoding algorithm. The first one is to terminate the LDPC decoding loop as in [9]. The second one is to detect the correct fragments of systematic part in a codeword during the LDPC decoding. If a segment is decided as correct one by the corresponding CRC, the segment is temporally assumed to be correct in spite of the undetected error probability of the CRC code. Then, the variable nodes of the correct segments give large message to their neighbors, which makes the convergence speed fast. If the undetected error of the used CRC is sufficiently small, we can treat ‘temporally correct’ as ‘permanently correct’, which can save the additional computation power by avoiding the calculation of the corresponding segments.

B. CRC selection problem

The proposed decoding algorithm exploits the multiple CRCs, which are available inside the received codeword. However, the performance of the decoding algorithm strictly depends on the error detection capability of the CRCs. Therefore, the selection of efficient CRC is very important. In general, the performance of CRC codes is represented by the undetected error probability Pud(N, p),

where Nc is the length of codeword, p is the bit error probability, Ai is the weight distribution of generator polynomial G(x) and R is the order of G(x). As we can see in (6), the performance depends on the codeword length and bit error probability. Therefore, if Nc, p and R are given, we can theoretically find out the optimal CRC code.
However, the search of optimal CRC code is not easy due to the tremendous code space. To avoid the full space search, a search method with a polynomial form of G(x) = (1+x)p(x), p(x) is a primitive polynomial, was proposed [7]. Although the results in [7] present efficient CRC codes, we should note that they may not give the optimal performance for specific applications. Until now, the only way to find out the optimal CRC for a specific environment is to evaluate exhaustively all the error and polynomial combinations. Because of the computational complexity, we will not persist on the search of optimal CRC code.

IV. SIMULATION RESULTS
To verify the performance and decoding speed of the proposed LDPC-CRC decoding, we conducted simulations regarding the codeword error rate (CER) , the number of required iterations, and the effect of burst erasures. Xiao’s progressive-edge growth method is used to generate the LDPC PCMs, where the generated irregular PCMs has a fixed column weight (=3) [10]. For the magnetic recording channel, we employed the erasure channel model (EPR4-equalized MRC corrupted by both random noise and erasures), described in [4]. For simplicity, we only consider the full erasure case of the model. And we denote the erasure density ρ, defined as a ratio: (number of erased bits)/(number of codeword bits). When ρ is specified, then a single erasure burst is generated at random position within the codeword, where the length of the burst is uniformly distributed in [10, ρ]. The efficient CRCs for the multiple CRCs are obtained from the Table 1 in [7].
Figure 2 and 3 shows the performance of the proposed scheme, where the code rate is maintained the same for comparison. The conventional IBP uses a single 32 bit CRC, while the proposed scheme is simulated with two(four) 16(8) bit CRCs. V. CONCLUSION
By exploiting the structure of the LDPC decoder and the error detection capability of the CRC codes, we have proposed a new decoding algorithm named LDPC-CRC decoding. Our algorithm shows the faster decoding speed and the better error performance for MRC with erasures, while the conventional algorithm requires more time for the better performance.
