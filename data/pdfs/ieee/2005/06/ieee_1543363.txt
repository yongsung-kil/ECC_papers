Low Complexity Stopping Criterion for LDPC Code Decoders

Frank Kienle, Norbert Wehn Microelectronic System Design Research Group, University of Kaiserslautern
Erwin-Schroedinger-Strasse, 67663 Kaiserslautern, Germany Email: {kienle, wehn}@eit.uni-kl.de

Abstract— Low-Density Parity-Check (LDPC) codes are one of the most powerful codes known today. They are decoded iteratively by a message passing algorithm. For this an inherent stopping criterion exists for decodable blocks based on parity checks. For undecodable blocks typically a fixed number of iterations are carried out before the decoding procedure is terminated, which is a waste of energy and time.
In this paper we present a stopping criterion which detects undecodable blocks in an early stage of the decoding process. The novel stopping criterion works for floating and fixed point implementations. It greatly reduces the average number of required iterations without any loss in communications performance.

I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes [1] are considered for future telecommunication standards due to their excellent error correction performance. They are already part of the new DVB-S2 standard [2]
The decoding of LDPC codes is an iterative process, e.g. for the DVB-S2 standard up to 40 iterations are required to gain the desired communication performance. Standard LDPC code decoder implementations assume a fixed number of iterations. In [3] even a maximum number of 64 decoding iterations are performed. For decodable blocks the inherent stopping criterion of the check node sum is a perfect stopping criterion. But for undecodable blocks, where the full number of iterations is processed, a lot of energy and processing time is wasted. The design metrics latency and energy consumption increase linearly with the number of decoding iterations. Therefore it is highly desirable to stop the decoding for undecodable blocks at an early stage of the decoding process. We propose to use an iteration control criterion which combines the inherent criterion with an additional stopping criterion. The iteration control determines whether a data block is already successfully decoded or not decodable at all. It is computed based on information available during decoding. At the end of each iteration, the decoder performs a check on the condition for stopping. This is related to our stopping criterion for TurboCodes which was presented in [4]. Detecting undecodable data blocks early in the decoding process would allow to reduce the average number of iterations over all channel characteristics. This is of great interest especially when an outer ARQ scheme is applied.

II. LDPC CODES
A LDPC code is a linear block code defined by its sparse M × N parity check matrix H. It contains j ones per column and k ones per row, called row and column weight respectively. A (j,k)-regular LDPC code has row and column weights of uniform weight, otherwise the code is called irregular.
A parity check code can be represented by a bipartite graph (Tanner graph), see Figure 1. The check nodes (CN) correspond to parity constraints and variable nodes (VN) represent the data symbols of the codeword. An edge in the graph corresponds to a one in the parity check matrix. The degree distribution gives the fraction of nodes for a certain degree, e.g. for the variable nodes {V N2 = 0.4, V N3 = 0.3, ...} states that 40% of the VNs have degree 2 and so on.
LDPC codes are decoded using a message passing algorithm [1] which exchanges soft-information iteratively between variable and check nodes. The code performance mainly depends on the randomness of the graph, the codeword size, and the code rate (R).
The updates of the nodes can be done in a canonical scheduling. In the first step all variable nodes in the second step all check nodes are updated. The processing of the nodes in each step is independent, and thus can be parallelized.
The exchanged messages are assumed to be log-likelihood ratios (LLR). The update of the messages at the variable nodes can be splitted into two parts. In a first step a current estimate of the decoded bit is calculated:

For the iteration control to work we have to distinguish between decodable and undecodable blocks. A. Theory
The criterion for stopping iterations of undecodable blocks is supported by the theory of [5]. In that work, Divsalar et al. applied the method of density evolution to analyze TurboCodes and give insightful answers to some open questions in the field of iterative decoding of parallel and serial concatenated convolutional codes (PCCC, SCCC). Density evolution was originally proposed by Richardson and Urbanke [6] for the asymptotic analysis of iterative decoding.
The variable nodes and the check nodes can be seen as two separate soft-in soft-out (SISO) decoders which passes information between them. This message exchange can be visualized by using EXIT charts [7] which tracks the mutual information at the input of the decoders. To monitor this information is two complex for a decoder harware realization. Therefore, we focus only on the variable node processing due to its simple message update.
For infinite blocksizes the VN outputs Λ (Equation 1) are modeled as Gaussian distributed LLRs, with the density function f (Λ) for symbol +1 and f (−Λ) for -1. If the consistency assumption Λ = ln(f (Λ)/f (−Λ)) is used, a Gaussian distribution with mean µ and variance σ2 fulfills the property 2µ = σ2. The LLR distribution is therefore completely determined by its mean value µ.
We interprete the variable node decoding as a SNR transformation [5], where the VNs output are each associated with

IV. RESULTS
The stopping criterion works for floating and fixed point representation. We focus on quantized LLR values since the stopping criterion is more important for hardware realization due to energy and time savings.
In this paper we use 6 bit message quantization for all LLRs with sign magnitude representation. This quantization scheme results in a communication loss of ∼ 0.1db compared to floating point representation [8]. The two codes used for our demonstration have code rates R = 0.8 and R = 0.5 with a codeword length of 2000 bits and 3200 bits respectively. A maximum number of 40 iterations are carried out.
The R = 0.8 LDPC code has a VN degree distribution {V N2 = 0.2, V N4 = 0.8} the CNs have all degree 18. The results for this code is shown in Figure 3. It shows the frame error rate (FER) and the average number of iterations over the signal to noise ratio Eb/N0 for different stopping criteria. The so called ’Genie’ stopping criterion is a unrealizable criterion which uses foreknowledge of the transmitted bits. It stops the decoding process immediately if the correct codeword is found or, in the case of an undecodable block, it does not start the decoding process at all (0 iterations). It is used as reference.
The parity check stopping criterion terminates the decoding process if all parity checks are fulfilled. It can be seen that this stopping criterion is a perfect stopping criterion in terms of communications performance. The only difference to a Genie stopping criterion at high SNR (Eb/N0 = 4dB) is that the PC

condition is fulfilled a couple iterations later then the genie
condition.
The VNR stopping criterion has to be suppressed after the V N R passes the V N Roff . For a 6 bit message quantization (1 bit fractional part) using a sign magnitude representation, the dynamic range is (25 − 1) = 31. The waterfall region lies at (Eb/N0)W R ∼ 3.5dB. The M B evaluates with Equation 7 to 8.95. This corresponds to a fixed-point value of M B = 18. In Figure 3 three different M B thresholds are shown, where the VNR stopping criterion is switched off.
The communications performance shows no degradation for M B = 16 (half quantization range). Impressive is the average number of only 3 iterations instead of 40 iterations for a low SNR (Eb/N0 = 2.5dB) were decoding is not possible at all.
For M B = 20 the average number of iterations is over the entire SNR range less then 12 iterations, which is very close
to the Genie stopping criterion The degradation in communications performance for M B = 20 is just 0.1dB. This greatly reduces the latency problem of the iterative decoding. In

V. CONCLUSIONS
LDPC codes are candidates for future communication systems. The iterative decoding process is for undecodable blocks a waste of energy. We presented a stopping criterion which detects undecadable blocks without any loss in communications performance. To detect undecodable blocks early in the decoding process is of great interest especially when an outer ARQ scheme is applied.
