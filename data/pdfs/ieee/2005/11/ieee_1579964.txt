Tiling parity-check matrix for Reduced Complexity High Throughput Low-Density Parity-Check Decoders Anand Selvarathinam, Gwan Choi
Department of Electrical Engineering, Texas A&M University, College Station, TX 77843-3128 {manivann,gchoi}@ee.tamu.edu

Abstract An approach for reducing hardware
complexity of LDPC decoders is presented in this paper. Low-density parity-check (LDPC) codes have a sparse parity-check matrix (H matrix). In LDPC decoder, the H matrix is stored in memory and contains information about the parity check constraints. The approach presented in this paper constructs several sub-matrices (pseudo random patterns) that are repeatedly used to form the H matrix. The merits of this approach on the decoder architecture are twofold. First, the switch logic associated with data forwarding in and out of the memory blocks, or alternately the routing of bit nodes to check nodes is simplified. Second, this approach reduces information stored in the design about the H matrix. Thus, the hardware complexity of the decoder is significantly reduced with an added advantage of increased throughput. LDPC code performance simulation results show that the proposed approach does not compromise the Bit error rate performance (BER) compared to that of ideal/optimal H matrix for same code length (N = 2040) and rate.
Keywords LDPC Decoder, VLSI, BER/FER, H matrix, Tiling, Patterns
1. Introduction Low-density parity check codes provide
error correction performance near the Shannon limit [1] when decoded iteratively. Several approaches have been presented in the recent past to construct LDPC codes that provide high BER performance. The performance of LDPC codes depends on the construction of the H matrix. Random constructions and irregular LDPC codes have been found to have better code performance. However, random constructions and irregular codes are difficult to implement in hardware. Hence constraints are imposed on the H matrix to permit reasonable hardware implementation cost and higher throughput. In [5], the rows of H matrix are divided into three groups so that all three edges of a column are into different row groups. This facilitates parallel

processing of the three edges of each bit node in one clock cycle in the serial architecture. In [2], the constraint that successive columns do not overlap in row position is used to process M bits simultaneously (throughput enhanced by a factor of M compared to serial architecture). In [3], several identity matrix patterns are used to construct the H matrix for interconnect optimization. These constraints do not significantly degrade the performance of the LDPC code. This paper presents additional constraints (in addition to [4], [5]) that permit tiling on H matrix for high throughput LDPC architecture [2] to reduce the hardware cost.
Besides error correction, Tiling approach addresses key issues, namely latency and hardware implementation costs in LDPC decoder implementation. The tiling approach for the H matrix has the following advantages a) Low hardware cost and memory – The tiling
approach reduces the hardware complexity of the decoder. This approach reduces information stored in the design about the H matrix by one-third. Further, the H matrix information is embedded into the hardware components reducing decoder memory hardware. b) High Throughput – By simplifying switch design that routes bit nodes to check nodes, decoder can be clocked at higher rate. Thus the decoder throughput is increased. In general, there are several novel methods proposed to eliminate storage of H matrix or reduce design complexity by creating regularity in H matrix. 1) Cyclic shifts of the identity matrix are
used to construct H matrix [3] [13]. 2) At high rates, anti pasch affine
geometries [16] are used to construct H matrices. We can present the tiling approach on anyone of the above platforms that partition and arrange codes for reduced switching. In this paper, tiling is exemplified using random H matrix. The H matrix is constructed using the tiling approach (in addition to existing constraints [2]) by repeating random patterns for a code length

2. Background In
the original work by Gallager [4], two constraints are introduced into the random construction of the H matrix (also a bipartite graph) as follows

i)

Each column has j (>=3) 1’s.

ii)

Each row has k (> j) 1’s.

The value j=3 is shown as the optimum value [4] and we consider a ½ rate code and therefore k=6. For hardware implementation, the following two constraints have been introduced in [5].

iii) First, second and third 1s in a column are in three different groups. The H matrix is constructed such that each of the three 1s along a column is in different row groups. A unique memory block in the serial architecture processes each row group. Therefore, three different memory blocks process the three edges of a bit node simultaneously.

iv) 1s do not occur in consecutive columns

in the same row. If 1s occur in

consecutive columns, memory has to

read and write for the same location in

the current clock cycle, which is not

feasible in the implementation

discussed in [5].

Scalable architecture proposed in [2]

processes M bit nodes in parallel compared to

the serial architecture that processes one bit node

at a time. To achieve this, scalable architecture

proposed in [2] has combinational logic scaled

by a factor of M and memory partitioned by a

factor of M, where M is the parallelization factor

of the architecture. In [2], the H matrix is

constructed to support this parallelism by

introducing the following additional constraint.

v)

The columns are grouped in to blocks of

M columns, where the ith block

contains columns (i-1)M+1 to iM. In

each block, the first 1s in M consecutive

columns do not overlap in row position.

Similarly, the second and third 1s do

not overlap in row position. This

ensures that M bit nodes access M

different memory blocks and there is no

memory access conflict.

Tiling approach is built on top of this parallel (and scaleable) architecture constraint to reduce hardware complexity and increase throughput.

3. Tiling Approach

Tiling approach generates patterns that

satisfy the scalable architecture constraint and

are placed repeatedly to form the H matrix. The

entire H matrix consists of a few repeated

patterns. Therefore, the decoder architecture does

not require storage of complete H matrix, but

only the patterns used to generate the H matrix.

This leads to a huge reduction in hardware

complexity as the H matrix requires significant

number of gates. In [3], the author considered a

pattern in the H matrix that has a single one in

each row and each column. Cyclic shifts of an

Identity matrix generate several such patterns.

The patterns are used in [3] to construct the H

matrix for interconnect optimization. The

number of patterns that can be generated in our

approach is higher than the patterns generated in

[3] because [3] uses only cyclic shifts of the

identity matrix. In this approach, any random

permutation of 1s in the identity matrix is

considered. In contrast to [3], patterns are also

repeated to store fewer patterns or sub-matrices.

We consider a 1020-by-2040 matrix.

The block length N = 2040 and is a rate ½ code.

The maximum parallelization [2] possible for a

(3,6) code with three 1s along a column is given

by parallelization factor M = N/6 (340). A

pattern is a block of size M by M. There are 18

tiles in the H matrix for M = 340. The 18 tiles

are constructed using only five different patterns.

The placement of the five patterns satisfies the

following constraints for high BER performance.

(i)

If two patterns are placed next to each

other along a column block, the two

patterns are not placed in another

column block in the same manner.

Pattern 2 is placed next to pattern 1 in

the first column block. If pattern 1 is

placed along the same row block in

another column block, pattern 2 is not

placed close to it. This constraint

eliminates cycles of length 4 among the

patterns themselves.

(ii) If there are l patterns and m tile

positions, each pattern is repeated a

minimum of [m/l] for fairness.

(iii) The patterns are placed as randomly as

possible to have high BER performance.

The scalable architecture [2] scales the iteration stage by scaling the combinational logic circuits and partitioning the memory into M sub blocks. M bit nodes are processed in parallel and the memory access conflicts are eliminated by carefully constructing the H matrix as discussed in the previous section. Figure.7 shows the memory architecture of the iteration stage [2]. To route the M bit nodes to appropriate memory blocks, switch logic is introduced whose complexity increases with M, the parallelization factor. A second switch logic is required at the output of the memory block to re-order the data that confers with the order of occurrence of the incoming bit node LLRs. For example, 340 8-to1 MUXs implement a switch logic for M = 340. Each MUX needs different select lines and needs appropriate logic to be generated for select lines. The proposed tiling approach reduces the switch complexity by sharing select lines between the multiplexers. For detailed analysis of the serial and scalable architecture, please refer [2], [5].
5. Tiling approach: Architecture enhancements
There are several benefits due to H matrix tiling on the decoder architecture. There is significant reduction in hardware complexity as H matrix is no longer stored in ROM. The patterns that generate H matrix are implemented using 8-to-1 MUX’s, one for each check node. There a total of 340 8-to-1 MUXs for M = 340

and 170 16-to-1 MUXs shown in Figure.8. The number of inputs to the MUX is equal to the number of patterns. Therefore 8-to-1 MUX and 16-to-1 MUX are custom designed as 6-to-1 MUX and 10-to-1 MUX respectively. Further, the select lines are shared between all the M MUXs. This sharing of select lines between the different multiplexers simplifies the switch logic hardware. It results in higher clock speed (five fold improvement for M = 340) and less gate complexity compared to the scalable architecture as shown in the table below.

6. Conclusion Tiling for H matrix has been presented to reduce hardware cost associated with H matrix storage. The tiling approach has been applied on top of constrained parallelization (parallelization factor M). The H matrix constructions for M = N/6, N/12 with fewer patterns has been shown to reduce hardware requirements for H matrix by reducing the number of entries that need to be stored to one-third. Further, the H matrix is built into switch logic as MUX patterns that need to be selected with every clock cycle. This simplifies switch logic. The simulation results show that there is no degradation of BER performance for M=N/6, N/12 with fewer patterns. Hence, the tiling approach can be applied in the hardware implementation of the decoder to reduce hardware cost especially that of memory cost and also improve the throughput of the decoder.
