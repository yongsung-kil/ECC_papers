AbstractŌĆö Low-density parity-check codes have recently received extensive attention as a forward error correction scheme in a wide area of applications. The decoding algorithm is inherently parallelizable, allowing communication at high speeds. One of the main disadvantages, however, is large memory requirements for interim storing of decoding data. In this paper, we investigate the performance of a hybrid decoding algorithm, using an approximating early decision algorithm and a regular probability propagation algorithm. When the early decision algorithm fails, the block is re-decoded using a probability propagation decoder. As almost all errors are detectable, the error correction performance of the hybrid algorithm is negligibly detoriated. However, simulations still achieve a 32% decrease of memory accesses.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were invented by Robert G. Gallager [1] in the early 60ŌĆÖs. However, despite excellent performance and a practical decoding algorithm, they were largely neglected for over 30 years until they were rediscovered by MacKay and Neal [2] in the mid 90ŌĆÖs. Since then, LDPC codes have been found to have performances comparable to, and in some cases exceeding, those of Turbo codes.
The main problem in coding theory is to build a practical and efficient decoder. The decoding algorithm for LDPC codes is inherently parallelizable, and thus suitable for communication at high speeds. One of the main disadvantages, however, is large memory requirements for implementations that are not fully parallel. A typical code with a length of 6000 bits might require up to a million memory accesses for the decoding of a single word. As 95% of the power is consumed in the memories [3], reducing the amount of memory accesses becomes an important concern, especially in low-power hand-held devices.
II. LOW-DENSITY PARITY-CHECK CODES
LDPC codes are defined as the null space of a very sparse parity-check matrix H. H has dimensions M ├Ś N , and the set of valid code words are defined by the relation Hx = 0 . Thus code words are of length N, and each code word contain (assuming H is full-rank) M parity symbols. An LDPC code is typically represented by a bipartite graph between N variable nodes and M check nodes, called a Tanner graph. An LDPC code is regular if all variable nodes have a constant degree of j, and all check nodes have a degree of k.

We distinguish between two failure modes of the decoder: decoder failure and decoder error. Decoder failure is when the decoder fails to find a valid decoding, and reports this fact. Decoder error, on the other hand, is when the decoder arrives at a valid code word that differs from the one that was sent. Hence, decoder failures are detectable, whereas decoder errors are not.

V. HYBRID DECODER
As shown in [9], early decision decoding using a constant threshold t = t0 exhibits an error floor as the signal strength increases. To avoid this, a hybrid threshold - sum product decoder may be used. The threshold decoder, like the regular sum-product decoder, very seldom converges to an erroneous code word. Decoder errors are thus unusual. As decoder failures are detectable, the failed code words can be resubmitted to a more relaxed threshold decoder, or a regular sum-product decoder. Thus, the bit error rate can be significantly reduced. However, as decoder failures still are rare, the computational overhead imposed is negligible.
The hybrid decoder first uses the early decision algorithm for Ih iterations, stopping if a valid code word is found. If Ih is reached, decoding is restarted using the sum-product algorithm, running for I iterations. Generally, good results are achieved if Ih is chosen between 1.5 and 2 times the average number of iterations required for a sum-product decoder. Choosing a higher value causes the average number of iterations to increase because of the large number of iterations required when the early decision decoder fails.
For channels with varying noise conditions, the performance of the hybrid decoder suffers. As seen in Fig. 3, for low SNR:s almost all words must be redecoded by the sum-product decoder, causing the performance of the hybrid decoder to drop far below the performance of the sum-product decoder alone. To mitigate this problem, we propose a modification to the hybrid decoder to use only the sum-product decoder for a small number of iterations whenever the early decision decoder fails. Assuming that channel conditions vary slowly over consecutive frames, as in a fading channel, the modification causes the hybrid decoder to perform close to a sum-product decoder for low SNR:s, while retaining a high level of communication reduction for higher SNR:s. VII. IMPLEMENTATION
Implementing the threshold decoder requires an additional memory storing the decisions of the bits. Thus the required memory is increased by N bits. We discuss an architecture using k variable node processing elements (VPE), and j check node processing elements (CPE), but the architecture can be further parallelized by careful code construction.
The check-to-variable and variable-to-check messages are stored in memories integrated in the VPEs. For each VPE a second memory is added that stores the decision values for the variables handled by that VPE. Thus, the processing element can be completely disabled whenever the decision bit is set for a variable.
The VPEs also compute the pseudo-posterior likelihoods, on which the decisions are based. Whenever the likelihood is above the threshold, the decision bit is set to 1.
The CPEs are connected directly to the VPEs through an interconnection network realizing the structure of the code. Each VPE connects to j CPEs, and typically six bits are used to represent the messages. An additional bit must be added to convey the state of the variable. If the variable is decided, only the value of the bit must be communicated, and soft information can be omitted. Also, the corresponding logic in the CPE can be disabled. Thus, two lines are driven for a variable-to-check message from a variable in its decided state, and no lines are driven for the returning check-to-variable message.
Several algorithms construct quasi-cyclic LDPC codes, where the parity check matrix consists of concatenated cyclically shifted identity matrices. This structure assures that the processing elements will handle adjacent variables in successive computations. Single-bit memory accesses of decision values may thus easily be avoided.
Assuming a wordlength of six bits, the VPEs require 6j bits to be read and written, and the CPEs require 6k reads and writes,

for an ordinary sum-product decoder. Assuming the worst case, an undecided variable will then require 6j+1 reads and writes of a threshold decoder VPE, and a decided variable will require just one memory read, for reading the decision value. The CPEs, on the other hand, require decision values for all the involved k bits, using 7 reads and 6 writes for each undecided variable, and 2 reads for each decided variable.

VIII. CONCLUSIONS
In this paper, we have investigated the performance of a hybrid early decision-probability propagation decoding algorithm. Almost all the early decision algorithmŌĆÖs failed decodings are detectable. Thus, by resubmitting the failed blocks to an ordinary probability propagation decoder, a significant reduction of memory accesses can be achieved, while the error correction performance suffers negligibly. In this paper, we achieve a communication reduction of 40%, with an estimated reduction of the total amount of memory accesses by 32%.
