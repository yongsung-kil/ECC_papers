An Improvement of the PEG Algorithm for
LDPC Codes in the Waterfall Region
G. Richter

Abstract- There exist many different algorithms to construct good Low-Density Parity-Check (LDPC) codes. To maximize the girth of the bipartite graph, which represents the LDPC code, the Progressive Edge-Growth (PEG) algorithm was introduced. This construction methods leads for irregular LDPC codes to a lower error floor, but also to a performance loss in the waterfall region compared with randomly constructed LDPC codes. In this paper, we describe a modification of the PEG algorithm that improves the performance in the waterfall region. Furthermore, this modified PEG algorithm yields codes with lower error floors and a slightly better performance in the waterfall region than randomly constructed LDPC codes.
I. INTRODUCTION
Since the invention of Turbo Code [1] a lot of research was done for iterative decoding schemes. In 1996, MacKay and Neal [2] and also Wiberg [3] rediscovered LDPC codes that were originally invented by Gallager already in 1963 [4]. They showed that LDPC codes could nearly reach the Shannon limit, when they are decoded by an iterative decoding algorithm, the so-called belief propagation or sumproduct algorithm. Because of their good performance, their comparably low decoding complexity, and their easy implementation in hardware circuits, they became serious competitors to Turbo Codes.
Gallager considered only regular LDPC codes, i.e., codes that are represented by a sparse parity-check matrix with a constant number of ones in each row and in each column. It is shown by simulations [5] that the performance of LDPC codes in the waterfall region can be improved by using irregular LDPC codes. To find good irregular LDPC codes a powerful tool, called density evolution is introduced in [6]. In [7] a code is determined that can perform within 0.0045 dB of the Shannon limit for a transmission over an Additive White Gaussian Noise (AWGN) channel.
These codes are designed assuming an infinite blocklength and an infinite minimum distance. However, many randomly constructed irregular LDPC codes, which show a good performance in the waterfall region, have a poor minimum distance and thus also a relatively high error floor. Hence, many different construction methods are introduced to lower the error floor, e.g., the ProgressiveEdge Growth (PEG) algorithm [8] or a randomized algorithm that tries to maximize the Approximate Cycle Extrinsic message degree (ACE) [9]. Unfortunately, these construction methods lead to a performance loss in the waterfall region compared to randomly constructed LDPC codes. In this paper, we describe a modification of the

IV. SIMULATION RESULTS
In this section, we show simulation results for transmissions over the AWGN channel and over the Binary Erasure Channel (BEC). Therefore, we use three different LDPC codes of length n = 10000, Rate R = 1/2, and a variable node degree distribution A(x) = 0.4999X2 + 0.3294X3 + 0. 1707x8. The desired check node degree distribution is p(x) = 0.9514X7 +0.0486X8. The first code is constructed by connecting all degree-2 variable nodes in a zigzag manner to avoid cycles between degree-2 variable nodes. The other edges are selected randomly by avoiding cycles of length four. The second code is constructed with the original PEG algorithm and the third code is constructed with the modified PEG algorithm presented in this paper. For all three codes the degree-2 check node distribution is p(x, 2) = 0.0004x1 + 9996X2. V. CONCLUSIONS
In this paper, we modified the PEG algorithm for the construction of LDPC codes. Therefore, we introduced the degree-s check node distributions, which determines the fraction of check nodes connected to a certain number of variable nodes with degree T. We showed by simulation and by threshold calculations with a generalized density evolution that this modification lowers the error rates in the waterfall region for the AWGN channel as well as for

the BEC. Furthermore, the modified PEG algorithm results
in LDPC codes with lower error floors and a slightly better
performance in the waterfall region compared to randomly
constructed LDPC codes. Unfortunately, the LDPC code
constructed with the modified PEG algorithm that was
used for the simulations shows a higher error floor for a
transmission over the AWGN channel than the LDPC code
constructed with the original PEG algorithm. This error
floor was caused by small trapping sets. Contrariwise, the
LDPC code constructed with the modified PEG algorithm
shows the lowest FERs for all erasure probabilities for a
transmission over the BEC.
