A Shared Hard Decisions Storing in Partially Parallel FPGA-based QC-LDPC Decoder

Tianjiao Xie1,2, Ruijia Yuan2 and Jianhua Zhang2 1. Dept. of Electrical Information Northwest Polytechnical University
2. China Academy of Space Technology (Xi’an) China Email: xiexietianjiao@163.com

Abstract—A strategy of hard decisions sharing intrinsic and extrinsic memory banks for partially parallel Quasi-Cyclic (QC) LDPC decoder is proposed in this paper. The proposed method not need extra memory banks to store hard decisions, could reduce the total number of memory by 33% compared with the decoder in reference [6] which is significantly fewer memory banks than the other published FPGA implementations decoders in reference [4-8], while maintaining the same hardware requirements at the same throughput level.
Keywords—QC-LDPC, LDPC decoder, memory banks, FPGA.
I. INTRODUCTION
Recently, LDPC codes have attracted a lot of attention due to their near Shannon limit performance and inherently parallelizable decoding scheme. Based on their construction methods, LDPC codes can be roughly classified into two categories: random LDPC codes and structured LDPC codes. For large code lengths, it is with high probability that random LDPC codes perform very well. However, they have some deficiencies in terms of encoding and decoding. On the other hand, for small-to-moderate lengths, well-designed structured LDPC codes can perform as well as or even better than random LDPC codes. The inherent structural properties lend themselves to friendly implementations. Well-known constructions of structured LDPC codes are QC-LDPC codes [1-3], which can not only enables the partially parallel decoding architecture but also achieve a good tradeoff between hardware utilization efficiency (HUE) [6] and decoding throughput. Therefore, a wide array of the latest communication and storage systems have chosen QC-LDPC codes for forward error correction in applications including digital video broadcasting (DVB-S2), 10 Gigabit Ethernet (10GBASE-T), broadband wireless access (WiMax), wireless local area network (WiFi), space communications, and magnetic storage in hard disk drives. Moreover, many investigators studied the design of partially parallel QCLDPC decoders in FPGA’s, for example in reference [4-8]. However, these approaches need additional memory banks to store the hard decisions. In general, hard decisions memory banks occupy large ratio of the total number of the decoder memory banks.
In this paper, we proposed a strategy of hard decisions sharing intrinsic and extrinsic memory banks for any partially parallel QC-LDPC decoder. The proposed method neither needs extra memory banks to store hard decisions nor employ registers to store the hard decisions. II. ARCHITECTURE PROPOSED
A. Hardware Architecture
Let us consider a (j, k) regular QC-LDPC, whose parity check matrix is formed by an array of j × k shifted cyclically Identity-matrix of size L × L. Our partially parallel decoder design reuse index L. In other words, j rows (k columns) from different block rows (columns) are updated simultaneously in the variable node update (check node update) phase. All rows (columns) of each block are updated serially in L clock cycles. For processors, the decoder requires k variable node processor units V N Ut with j + 1 − input, j check node processor units CN Us with k − input and j parity equation check units (P CUs) with k − input. For memories, j × k memory banks Ms,t(0 ≤ s ≤ j − 1, 0 ≤ t ≤ k − 1) are used for storing extrinsic information exchanging between V N Ut and CN Us and n memory banks Ft(0 ≤ t ≤ n − 1) are adopted for intrinsic information from channel. Both memory banks Ms,t and Ft are dual-port Block RAM. Each memory bank contains L storage units, each unit of the wordlength Q for the memory banks Ms,t and Ft depends on quantization schemes; here Q denotes Q-bit quantization. There is one modulo-L counter associated with each memory to generate the corresponding memory address.
Unlike those partially parallel decoders architecture in [4-8], which need a lot of memory banks to store hard decisions, The proposed shared hard decisions storing partly parallel decoder architecture could save those hard decisions memory banks. Fig.1 illustrates our decoder architecture for (j, k) regular QC-LDPC codes. In Fig.1 VNU, CNU and PCU include V N U1, V N U2, . . . , V N Un, CN U1, CN U2, . . . , CN Um and P CU1, P CU2, . . . , P CUm, respectively. RAM f and RAM m denote Ft and Ms,t, respectively.
B. Algorithm description
LDPC codes are commonly decoded with an iterative process between the two decoding phases to obtain the optimal BER performance [2]. For iterative decoding, the decoder repeats iteratively until an assigned maximum iteration number is reached. Similarly, Our decoding process can be carried out as follows.

Initialization:
Initialization iteration number I = 1, the maximum number of iteration are initialized to the values MAX ITER. Store the received one frame intrinsic information to memory Ft, and initialize all memory Ms,t to zeros. The data are stored column-wise in the memory Ft and row-wise in memory Ms,t. The VNU start first here since all memory banks Ms,t = 0. Alternatively, the CNU start first if we initialize Ms,t = Ft.
Variable node update:
V N U1, V N U2, . . . , V N Ut, . . . , V N Uk work concurrently. In Ith iteration, the (I − 1)th updated information are simultaneously read from all the memory Ms,t by k V N Ut(0 ≤ t ≤ k − 1), the tth V N Ut read memories in tth column, M1,t, . . . , Mj,t and Ft. Each memory bank is read one data per clock cycle. So in each clock cycle there are j + 1 − input (including j extrinsic information and one intrinsic information) comes into V N Ut simultaneously. The j + 1 outputs of V N Ut are the sign of j + 1 inputs summation (one 1bit hard decision) and subtracting according j inputs from the summation of the j + 1 inputs (j Q-bits extrinsic information), respectively. Then we combine 1-bit hard decision (as high bit) with Q bits extrinsic information (as others bits) into j (Q+1)bits information, and as the Ith updated information are written back to the same address as read operations in the order of L − xs,t, . . . , L − 1, 0, . . . , L − xs,t−1 since all the j memories in one block column have different offset xs,t values thus different addresses for the variable node update. And 1 bit hard decision combine into Q bits intrinsic information as the Ith updated hard decisions are written back to the same address as read Ft operations in the order of (0, . . . , L−1). Consequently, in one iteration, totally L clock cycles are required to complete the updating process of all the k × L columns. After variable node update, setting iteration number I = I + 1.
Check node update and parity equation check:
CN U1, CN U2, . . . , CN Us, . . . , CN Uj run concurrently with P CU1, P CU2, . . . , P CUs, . . . , P CUj. In Ith iteration, Each Ms,1, . . . , Ms,k is read one data by CN Us and P CUs (0 ≤ s ≤ j − 1) per clock cycle. So each clock cycle there are k − input (Q + 1)-bits information comes into

CN Us and P CUs simultaneously. Then we divided the hard decision bit from the extrinsic bits with the same order as VNU combination, and send into CN Us and P CUs, respectively. The k Q-bit outputs of CNU as the Ith updated extrinsic information are written back to the same address in the order of (0, . . . , L − 1) for all the k.Ms,t memories in a block row, all the k memories in one block row share the same memory address. The outputs of P CUs is the sth syndrome vector, we can totally get j syndrome vectors, if all the syndrome vectors are zero, it indicates the parity check is satisfied, and then stop iteration and output hard decisions from memory Ft, otherwise, continue to the next step until the maximum number of iterations is reached. In one iteration totally L clock cycles are required to complete the updating process of all the rows.
From the decoding process outlined above, 2L clock cycles are required to complete the decoding for one frame.
III. STORAGE CONSUMPTION
All the QC-LDPC decoders in reference [4-8] adopt additional memory banks to store hard decisions, which cause waste of large amount memory banks. The above algorithm neither needs extra memory banks to store hard decisions, nor employ registers to store the hard decisions.
Obviously, check node update run concurrently with parity equation check because when variable node update generate the extrinsic information for CNU, they also generate the hard decisions for PCU. It needs j × k memory banks for extrinsic information. In term of this hard decisions sharing intrinsic and extrinsic memory banks method, each memory bank for Ms,t and Ft requires L × (Q + 1) bits and without extra memory bank for hard decisions. Form memory Ms,t, hard decisions and extrinsic information can be accessed simultaneously. Therefore, PCU share the same addresses with CNU, i.e., no extra address generation units are specifically required for PCU. The additional advantage of this approach is that only one set of addressing mechanisms is shared for both CNU and PCU.
The QC-LDPC decoder in reference [6] uses significantly fewer memory banks than the other published FPGA implementations [4-8], which totally need k + j × k + j/2 × k memory banks, but the proposed method only need k + j × k memory banks, could reduce the total number of memory by (100 × r)% compare with [6], here,

IV. IMPLEMENT RESULTS

In order to make a fair comparison to the decoder in reference [6], we have implemented our decoder using the same (3, 6) regular QC-LDPC code with L = 256, the same nonOMP (overlapped message-passing) [6] decoder architecture, the same Normalized Min-Sum decoding Algorithm (NMSA) with the same 8-bits quantization scheme, the same Xilinx Vertex II xc2v6000-5ff1152 FPGA which has 33792 slices and 144 block RAMs (each block RAM is 18K bits).

For reducing the critical path delay and improving the throughput of the decoder, the 8-stage pipeline scheme is adopted while 6-stage pipeline is used in the decoder of reference [6], The synthesis and implementation was carried out using Xilinx ISE Version 10.1. In Table I, we compare the number of occupied slices, flip flops, 4-input look-up tables (LUTs), and block RAMs as well as critical path delays, average iteration numbers, and estimated decoding throughput of our decoder architectures with that decoder from reference [6]. Each memory bank for intrinsic or extrinsic information requires L × (Q + L) = 256 × 9 < 18k bits, so each Block-RAM is configured as a dual-port 2k × 9-bit Block RAM. As illustrated in Table I, our decoder only require 24 memory banks: j × k = 18 and k = 6 memory banks for extrinsic information and intrinsic information respectively. In comparison to the decoder of reference [6], our decoders require significantly fewer memory banks.

Furthermore, in our storage approach, only one set of addressing mechanisms is shared for both CNU and PCU. While the decoder in reference [6] generates addresses respectively for the two processors. Therefore, comparing to the decoder in reference [6], our decoder occupies fewer logic resources, as shown in table I.

In FPGA, data can be stored in three different ways: block RAM, register memory, and distributed RAM. So there is another logic (slice) resources method to store hard decisions (Use k-L bits registers or distributed RAM to store the hard

decisions), which can also reduce the total number of memory by 33% compare with reference [6]. We implemented the decoder using k-L bits registers at the same FPGA as the decoder in reference [6], however, it leads to very high slice resources cost. So our method is the optimal method to share the hard decisions storage of QC-LDPC partially parallel decoders.
V. CONCLUSION
Based on the QC-LDPC distinguishing structural feature, the high throughput decoders for long codes can be achieved by using the partially parallel architecture. However, due to the dual characteristics including the reuse of resources and parallel processing, a significant memory resources are needed for partially parallel the QC-LDPC decoder. A method of hard decisions sharing intrinsic and extrinsic memory banks for partially parallel QC-LDPC decoder is proposed, which could efficiently reduce the BRAM resources. Based on Xilinx Vertex II xc2v6000-5ff1152 FPGA, the proposed decoder could reduce the total number of memory by 33% compared with the decoder in reference [6]. Moreover, a higher system clock frequency (i.e., decoding throughput) and less logic resources can be implemented through the address mechanisms sharing strategy. It is worthy to be mentioned that the designed memory sharing decoder could be applicable for various communication and storage systems.
