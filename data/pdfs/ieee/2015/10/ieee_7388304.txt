A Parallel-Routing Network for Reliability
Inferences of Single-Parity-Check Decoder

Abstract—The computation of the reliability inferences among the variables of a single-parity-check (SPC) code is a common challenge to the implementation of channel decoders. Applicable to a variety of computational mechanisms using disparate kernels, a parallel-routing network has been developed, which is, compared to the state-of-art structure, exploring the best of its parallel nature for an improved timing performance. Furthermore, we assure that the proposed structure has no degradation in neither computation accuracy nor hardware complexity. With this structure, the LUT-based method becomes the optimal solution as a whole to implement an SPC decoder and other decoders containing it, like, for example, a low-density parity-check (LDPC) decoder. The improvement of the proposed design has been verified by a field-programmable gate array (FPGA), showing a 186% increase in clock rate for a 32-degree SPC decoder.
I. INTRODUCTION
Although a single-parity-check (SPC) code itself is rarely applied in practices, it plays a non-negligible role in forwarderror-correction (FEC) field because it appears in plenty of more advanced channel codes in use. For example, low-density parity-check (LDPC) codes [1] can be viewed as built up by multiple SPC codes. In one SPC code, a vector of independent bits are restrained to satisfy the collective parity value of them. This mechanism is generally utilized to infer the reliability of each bit and hence to assist the decoding algorithm. For a near-optimal SPC decoder, the computation of such inference includes non-linear operations namely multiplication, tanh, and tanh−1, which forms a challenge to the purpose-specific implementation because all the precision, complexity and timing performances have to be considered and balanced.
An accurate method to this problem is to convert the multiplication into log-domain addition [2]. Given a certain quantization scheme, this solution is able to perfectly avoid computation errors while, however, suffers from a significant complexity due to incurring mapping functions. On the other hand, another complexity-reduced technique, named “minsum” (MS) [3], is widely applied [4], [5], [6]. By approximating the inference result with the minimum reliability of the inputs, it simply consists of an array of comparators. Unfortunately, the drawback is a decreasing precision with higher

check-node degree. Established on it, the offset MS algorithm can make a considerable improvement with correspondingly higher complexity.
Comparing to these two methods, we prefer and have proposed a method based on look-up tables (LUTs) to facilitate logic minimization [7]. It is noticed that the MS architecture is a special case of using LUTs so this method also has as a low complexity as MS but offers a much higher precision. For an overall SPC decoder, the total number of applied LUTs are minimized using a cascaded structure. However, this structure is substantially inferior to the above two methods in timing performance because it does not allow for a parallel routing as they do and this is just the preponderant reason why it is less practical.
In this paper, we shall reveal that there is indeed an inherent parallel nature in an SPC decoder no matter which type of computational kernel is applied. With the proposed structure, the LUT-based method is efficiently ameliorated to achieve a latency of O(logN ) without introducing redundancy or compromising accuracy. Therefore it is demonstrated to be optimal for implementing a balanced SPC decoder and further applicable to other decoders. The implementation using an FPGA shows that for a 32-bit SPC decoder, the maximum clock rate has been improved by 186% compared to the traditional cascaded-routing network. Since the kernels are not qualified in this structure, the proposed network is also expected to serve some other applications that can be abstracted to have the same problem.

III. REVIEWING TRADITIONAL STRUCTURE
There exists a traditional structure for the SPC decoder that minimizes the requirement of “ ” to be only 3(N − 2) [8]. Nevertheless, the cost of such an economical layout is a serious delay. This method defines the forward-going and backwardgoing partial-sum results with F and B that satisfy: F1 = I1 and BN = IN ; Fl = Fl−1 Il; and Bl = Bl+1 Il. Then O can be computed as O1 = B2, ON = FN−1 and for i = 2, 3, . . . , N − 1, Oi = Fi−1 Bi+1. Each step of building F, B and O takes up N − 2 times of “ ” operations, so the number of the computation kernels is 3(N − 2). Notably, the constructions of F and B each has introduced a cascaded structure of N − 2 stages and thus the latency of this structure is (N − 2)τ , where τ refers to the delay of each kernel. When N is large, the delay shall be linearly extended, deteriorating the decoder by limiting its operational clock rate. To address this deficiency, a symmetric-tree structure is proposed to fulfill a parallel-routing network for the SPC decoder.

If 2n−1 < N < 2n, the design procedure will be going as: we first posit N = 2n to construct Un and Vn and splice them together; then we eliminate those functions and connections that are associated to the fictitious inputs/outputs so that no kernels and wires are superfluous.
To gain an intuitive understanding of the proposed structure, consider an example with N = 14 where accordingly n = 4 and thus U4 and V4 are constructed as illustrated in Fig. 5. A perceptual observation is that the kernels in each sub-network are organized to resemble an incomplete binary-tree structure, and the trees in U and V can be roughly viewed symmetric. Moreover, the trees are balanced so that the maximum hight is minimized for the fewest levels of delay. In terminology of a tree structure, each node in U represents the aggregation of its sub-trees into one group and each path in V sequentially associate each node of U with one child of its sibling node. Next, the nodes and connections in U4 having a track to I15 or I16 can be eliminated under such rules: when both inputs of a node are nonexistent, the node and its output is also nonexistent; when one input is concrete, it will be directly output and the node will be eliminated. Finally, V4 is managed with reference to U4 in a way that we eliminate the node if it has a mirror node whose parent node has been canceled out and the output from its own parent node will directly pass through.
For an N -to-N network, the network U consists of N − 2 “ ” operations and the network V requires double. Therefore the total kernel usage is 3(N − 2), the same as the traditional structure. However, in contrast to the latency of the traditional structure that is linearly depending on N , the propagation time of this network is only 2 ( log2 N − 1) τ , a logarithmic relation with N .

It is observed that with an increasing N the traditional structure has an operational clock frequency limited by a sharply dropping upper bound, while the maximum clock rate of the proposed one declines slowly. Furthermore, these two decoders are applied in the decoding of an quasi-cyclic LDPC (QC-LDPC) code [9] to examine their decoding precisions. Theoretically since we only change the computation order, the results should not be affected neither. This assumption can be also verified by the overlapping BER results illustrated in Fig. 6. However, a subtle discrepancy still exists between the two curves showing that the outputs of the two structures may be different. This is because the messages are quantized and each LUT incurs some amount of quantization error that will be propagated through the subsequent LUTs. Since the proposed structure has less processing stages, the gross quantization error is reduced and the accuracy improved.
Summing the above observations up, the proposed structure of parallel-routing network is capable to significantly suppress the latency while it keeps the complexity and precision unaffected. By comprehensively considering and comparing with other methods [7], using LUTs placed by the proposed structure is generally the optimal method to implement an SPC decoder.
