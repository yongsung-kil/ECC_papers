A Low-Latency Algorithm for Stochastic Decoding of LDPC Codes
Kuo-Lun Huang†, Vincent C. Gaudet‡, and Masoud Salehi†
†Department of Electrical and Computer Engineering, Northeastern University, Boston, Massachusetts ‡Department of Electrical and Computer Engineering, University of Waterloo, Waterloo, Ontario †huang.ku@husky.neu.edu, ‡vcgaudet@uwaterloo.ca, †salehi@ece.neu.edu

Abstract—We introduce a low-latency stochastic decoding algorithm, called conditional stochastic decoding, to implement iterative Low-Density Parity-Check (LDPC) decoders. The conditional stochastic decoder, which utilizes reliable messages in the decoding process and the channel receiving probability to generate stochastic streams, improves error rate performance and decreases decoding latency. Compared to conventional stochastic decoders, the proposed algorithm achieves more than 20-percent reduction in the decoding latency for the (1056,528) LDPC code from the WiMAX standard. We also show that the proposed algorithm, due to its lower frame error rate, is a suitable choice to be used in the Hybrid Automatic Repeat reQuest (HARQ) systems.
Index Terms—Iterative decoding, low-density parity-check (LDPC) codes, stochastic computation, stochastic decoding.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes are a class of error-correcting codes that provide near-capacity performance [1]. An LDPC code is characterized by its sparse parity-check matrix, H, which has a low density of non-zero elements. In [2], Tanner showed that LDPC codes can be effectively represented by a bipartite graph, now known as the Tanner graph. The Tanner graph consists of two distinct groups of nodes: variable nodes and check nodes. For a rate Rc = k/n LDPC code, the Tanner graph contains n variable nodes, representing codeword bits, and n − k check nodes that represent the code’s parity-check equations. These nodes are connected by edges corresponding to the non-zero elements in H.
LDPC decoding is performed iteratively using a form of the belief propagation principle known as the Sum-Product Algorithm (SPA) [3], or its less-complex approximation, the Min-Sum Algorithm (MSA) [4]. Both algorithms iteratively compute a posteriori probability messages that are passed between the nodes in the Tanner graph. The performance of LDPC codes and the error-correcting ability of these iterative decoding methods improve as the codeword length n gets longer. However, the complexity of the interconnection network between nodes also grows higher as n increases, and routing congestion leads to high chip area consumption and low logic utilization [5].
In [6], Gaudet and Rapley demonstrated that stochastic computation can be used as a hardware-efficient alternative

to implement iterative decoders. Later, Tehrani et al. refined the algorithm and presented a fully parallel stochastic decoder [7] for longer practical codes. All messages in the stochastic decoding process are represented by Bernoulli sequences. Using this representation, all operations at the decoder are converted into bit-wise operations, which can be implemented by simple logic arrangements [8]. Furthermore, the inherently serial messages used in stochastic decoders only require a single wire to transmit between nodes. This type of transmission helps to alleviate the routing congestion problem. Because of its efficiency in hardware implementation, stochastic decoding has been used to decode the (1056,528) LDPC code employed in the WiMAX standard [7], the (2048,1723) LDPC code used in the 10GBASE-T standard [9], the (672,588) LDPC code in the WPAN standard [10], and many other error-correcting codes [11][12][13][14].
One crucial factor to ensure the proper stochastic computations is randomness in the stochastic streams [15]. In stochastic decoding, stochastic streams are normally generated by employing a stochastic number generator, which includes a random number generator and a comparator. At the initial stage of stochastic decoding, the messages received from the channel are transformed to stochastic bits by the stochastic number generator and loaded into each variable node one bit per clock cycle. The bits generated by this conventional transformation are merely based on the received channel messages, which remain fixed over the decoding process.
This paper proposes a new stochastic number generating approach to be used in stochastic LDPC decoding. The algorithm, called Conditional Stochastic Decoding (CSD), utilizes not only the received channel messages but also the reliable messages from the check nodes during the decoding process to determine the value of the next channel incoming bit at the variable node. Compared to the existing stochastic decoders, the CSD achieves lower error rate performance and less latency in decoding LDPC codes. The latency reduction of the CSD can be more than 20-percent of the decoding latency of the existing stochastic decoders. Furthermore, the low Frame Error Rate (FER) performance of the CSD makes it suitable to be used as the decoding algorithm in Hybrid Automatic Repeat reQuest (HARQ) schemes, which is a combination of Forward Error Correction (FEC) and Automatic Repeat reQuest (ARQ) error-control method. II. STOCHASTIC DECODING OF LDPC CODES
In stochastic decoding, binary streams are used to represent the iteratively transmitted probabilities between nodes. Each bit of the stream is equal to 1 with the probability equal to the value of the information to be sent. The transformation from a probability p to bit stream involves a uniformly distributed random number R, which varies each clock cycle. If the probability p is greater than the random number R, the stochastic number generator outputs a 1, and a 0 otherwise. Therefore, the probability of a 1 appearing at the output of the stochastic number generator at each clock cycle is equal to p. Since each bit in the stream is generated independently, the position of 1s is not fixed and different stochastic streams are possible to represent a given probability. For instance, (1, 1, 0, 1), (0, 1, 1, 1), and (1, 0, 0, 1, 1, 1, 1, 1) are possible representations of a probability p = 0.75.
The decoding round of stochastic decoding is called a Decoding Cycle (DC) [7], which includes the exchange of one bit on each edge of the Tanner graph. One DC does not directly correspond to an iteration in the SPA or the MSA. However, the node operations in a stochastic decoder are similar to those in the probability-domain version of the sumproduct decoding, but are processed in bit-serial fashion. Fig. 1 shows the basic stochastic computational blocks used for a degree-3 variable node and a degree-3 check node [6]. Higherdegree nodes can be constructed by sub-graphs containing only degree-3 nodes. Hence, we use a degree-3 node to demonstrate the node operations in stochastic decoders. From Fig. 1(a), we note that if the incoming bits of a variable node are not in agreement, the output bit of the node will be the same as the previous one. While a variable node is stuck in a hold state like this, the decoder could face decoding failure due to the lack of randomness in the streams. In order to overcome this problem, moving average filter methods, e.g., Edge Memory (EM) [16] or Tracking Forecast Memory (TFM) [17], were introduced to increase the efficiency of variable node operation in stochastic decoding. Implementations of both EM and TFM require memories at each variable node to store reliable probabilities to ensure proper node operations in stochastic decoders. When the variable node is stuck at a hold state, the outgoing bit is randomly selected from the memory, which is only updated when the variable node is not in a hold state. This mechanism introduces the desired randomness in bit streams and helps to improve the performance of stochastic decoder.

III. CONDITIONAL STOCHASTIC DECODING
It is known that the performance of stochastic decoders is affected by the bit patterns of stochastic streams, where streams even with the same encoded probability but different representations could result in different error rate performance. An auxiliary method that can generate stochastic bits adaptively is essential to the performance of stochastic decoders. The proposed algorithm, the CSD, is a method for generating stochastic bits adaptively during the decoding process. In this approach, the bits are generated not only based on the channel receiving probability but also the extrinsic messages from the connected check nodes.
Except for the transformation of the channel receiving messages to stochastic streams, the decoding process in the CSD algorithm remains similar to the conventional stochastic decoding algorithm. In the CSD, when all the connected check nodes of variable node vi are satisfied, the next channel incoming bit at vi is set to the value of the majority of bits received from its connected check nodes. When at least one of the check nodes connected to vi is not satisfied, the channel incoming bit at vi will be generated based on the channel receiving probability pi by employing the conventional stochastic number generator. Therefore, when all the extrinsic bits are from satisfied check nodes, the variable node vi will use these relatively reliable bits, instead of a randomly

In the CSD, only when all the connected check nodes are satisfied, the extrinsic messages from those nodes are used to determine the next channel incoming bit at the variable node. Variable nodes with higher degree tend to have more reliable bit values because for these nodes more parity-check equations have been satisfied; however, the occurrence of the proposed bit generating scheme will become fewer as the variable node’s degree increases.
We know that the extrinsic bit received from a satisfied check node will be the same value as the one sent out from vi on the same edge of Tanner graph. This outgoing bit from variable node vi has two possible sources: the channel incoming bit when there is an agreement among the bits received at vi, or the bit selected from memory, e.g., EM or TFM, if vi is in a hold state. Hence, the extrinsic bit from the satisfied check node is a convex combination of the channel incoming bit and the bits from memories. It indicates that the bit generating approach in the CSD utilizes both the channel receiving probability pi and the probabilities stored in the memories at vi to provide reliable progress during decoding process.
Furthermore, the stochastic bit generating approach in the CSD lowers the weight of pi that could diminish the influence of highly corrupted channel receiving values. These factors make the stochastic bits generated by the CSD algorithm more reliable than the bits transformed merely based on pi.

The degrees of variable node and check node in the irregular (1056,528) code are dv = {2, 3, 6} and dc = {6, 7}, respectively. We adopt the type-I HARQ scheme [21], where a codeword is transmitted initially and if the receiver fails to decode the codeword, a retransmission is requested by sending a Negative-Acknowledgement (NACK) message back to the transmitter. Upon receiving a NACK, the transmitter sends the same codeword again. When an Acknowledgement (ACK) message is received or the maximum number of retransmissions is achieved, the next codeword is transmitted. This socalled stop-and-wait method is the simplest ARQ scheme.
We also employ the Chase Combining (CC) [22] method in the HARQ system. Multiple received packets are com-

bined based on the maximum ratio combining principle to improve the quality of detection through gains in the SNR. Unless a packet is detected correctly in the first transmission, the retransmitted packets, after reception, are combined with previously received packets before being sent to the decoder.
In our simulations, we studied an HARQ system with two maximum transmissions, i.e., N = 2. Extension of the results to higher values of N is straightforward. We set the total number of DCs for the HARQ system to a maximum of 300 and used two different distributions of the 300 DCs among the first and second decoding attempts: first we set equal maximum 150 DCs, which we denote as 150/150 DCs, and then 200 and 100 DCs available for each decoding attempt, i.e., 200/100 DCs in the HARQ system. We compare the BER performance and decoding latency of the HARQ system using the CSD in both decoding attempts with a system using the EM-based stochastic decoder.

VI. CONCLUSION
We propose a conditional stochastic decoding algorithm, which makes use of the reliable messages during decoding process to determine the channel incoming bits for stochastic LDPC decoders. This approach not only diminishes the influence of corrupting channel receiving values, but also gives lower error rate performance and more than 20-percent latency reduction in stochastic LDPC decoding. We also study the performance improvement of the HARQ system adopting the proposed decoding algorithm. The improved performance and hardware efficient design make the proposed stochastic decoding algorithm suitable for high-throughput communications systems.
