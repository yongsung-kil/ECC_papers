A Low-Complexity Decoding Algorithm for
Concatenated Tree Codes
Daesung Kim, and Jeongseok Ha
Department of Electrical Engineering
Korea Advanced Institute of Science and Technology (KAIST)
Daejeon, 305 – 701, Korea
Email: gods5614@kaist.ac.kr, jsha@kaist.edu
Abstract—This work considers a modified belief-propagation
algorithm with two way scheduling for concatenated tree (CT)
codes in which tree codes are employed as constituent codes.
It will be shown that constituent tree codes can be efficiently
decoded with the modified algorithm. Thus, by repeatedly using
the modified BP algorithm for the constituent tree codes, CT
codes can be decoded in an efficient way. Moreover, we show
that the modified algorithm is especially suited for low complexity
hardware implementation. Finally, a simple analysis technique for
error-correcting performances of tree codes will be presented.
I. INTRODUCTION
Low-density parity check (LDPC) codes have been well
known for their capacity approaching performance [5], and
widely employed in various areas, e.g. wireless/wired com-
munications, high density storages, and etc. In spite of such
growing popularity of LDPC codes, high decoding complexity
has been regarded as a major obstacle. To resolve such
technical challenge, quasi-cyclic (QC) LDPC codes have been
studied [1], which allows decoders to be implemented with
low hardware complexity. Besides the complexity issue, there
are other practical issues associated with LDPC codes such
as inflexible structure in rate and length compatibilities and
no analytic way to evaluate performances in the low-error rate
regime.
In this work as a practical realization of LDPC codes, we
consider concatenated-tree (CT) codes which are a kind of ca-
pacity approaching codes proposed by Li Ping et al in [7]. CT
codes are constructed by concatenating cycle free tree codes in
a parallel fashion. The decoding algorithm in [7] is performed
in such a way that decodings for individual constituent tree
codes are carried out, and the decoding results are exchanged
across the constituent codes as extrinsic information. The
decodings and information exchanges are conducted iteratively
until either a valid codeword is found or the number of
iterations reaches a preset value. CT codes have a few unique
features: 1) maximum likelihood (ML) decoding can be carried
out for each constituent tree codes with the belief propagation
(BP) decoding [4], and 2) CT codes are flexible in the sense
that their rates and lengths can be adaptively adjusted. Thus,
the CT codes are a favorable choice as error-correcting codes
(ECCs) for developing practical error-control systems.
The pivotal part of the decoding algorithm for CT code
is the decoding for the constituent tee codes which in turn
determines the overall complexity. Thus, to develop a low-
complexity decoder for CT codes, a decoding algorithm for
the constituent tree codes must be carefully devised. The CT
codes in [7] has their constituent tree codes with a certain
structure. It was shown that by utilizing the structure, BP
decoding with a two-way scheduling [3] efficiently provides
the ML decoding results. However, the authors in [7] neither
discussed a specific decoding algorithm for constituent tree
codes nor demonstrated any error-correcting performances of
the tree codes.
In this paper, we propose a low complexity decoding
algorithm for constituent tree codes and error-rate performance
of the tree code with the proposed algorithm will be analyzed.
Since the tree codes can be regarded as a kind of LDPC
codes, they can be decoded with the traditional sum-product
algorithm. However, by taking advantage of a structural prop-
erty, i.e. most of bit nodes have degree one, it is possible to
greatly simplify the decoding algorithm of the constituent tree
codes. For the degree one bit nodes in tree codes, there are
only outgoing messages and thus no bit node message updates
are required for the degree one nodes. The low-complexity
decoding process can be further simplified [8] when the QC
structure is applied to the constituent tree codes [2].
The rest of this paper is organized as follows. In Section II,
the structures of concatenated tree codes and constituent tree
codes will be concisely summarized. In Section III, we will
specify the decoding algorithm with two way scheduling of
constituent tree codes. In Section IV, a simple analysis tech-
nique for estimating the error rate performance of constituent
tree codes in the low error-rate regime will be presented.
Finally, in Section V, we will conclude this work.

Following the notations in [6], [7], each constituent tree code
can be parameterized by a tuple (I, J′, J′′) where I indicates
the number of sub-trees in which there are J′ (J′′, resp.)
message bits connected to the recursive (non-recursive, resp.)
check node. Thus, the total number of message bits in a tree
code becomes (J′ + J′′)I, and the code rate of CT code is
(J′ + J′′)/(J′ + J′′ + M).
It is easily understood from Fig. 1 that each constituent
tree code is linear time encodable. That is, from the left
end to the right end, each parity bit including punctured bit
can be generated immediately as neighboring message bits
come into the encoder. Thus, CT codes are also linear time
encodable. Meanwhile, for decoding, a few different decoding
strategies are possible. Since the CT code shown in Fig. 1
can be expressed with a low-density parity-check matrix, we
may directly apply the BP algorithm to the decoding for the
CT codes. As another possible decoding strategy, we illustrate
the decoding algorithm introduced in [7] in Fig. 2 where Dec-
m denotes a decoder for the m-th constituent tree code, and
each constituent code is decoded one by one in a sequential
manner, and the decoding for each tree code produces extrinsic
information [7] for decoding the next constituent codes. The
sequential decoding process is iterated for decoding CT codes.
While the general decoding strategy was introduced in [7],
there was no detailed description of the low complexity
decoding algorithm for constituent tree codes. So, for the
purpose of filling the gap, we present a modified BP algorithm
with two-way scheduling for decoding constituent tree codes
in the next section.

V. CONCLUSION
In this paper, we present a low complexity decoding algo-
rithm for tree codes. It can be used as constituent decoder for
CT codes. In addition, we propose a simple error rate analysis
technique for individual tree code. Based on the proposed
algorithm and analysis technique, researches on developing
practical CT codes will be followed.
