Design of Low Density Parity Check Decoder Using Min-Sum Algorithm

Sachin S. Bhojane, Monica V. Mankar, Gajendra M. Asutkar, Pravin K. Dakhole

Abstract- Low density parity check (LDPC) code has received more attention due to their excellent error correcting performance capabilities. An LDPC code can be decoded using iterative method like the sum-product algorithm and the Min­ Sum algorithm based on its Tanner graph .In this paper, fully parallel architecture has been designed for LDPC decoder using Min-Sum algorithm. This decoder modeled in Verilog synthesized and performed place and route for design using Xilinx 13.1.
Index Terms- fully parallel, Low density parity check (LDPC), Min-Sum algorithm.
I. INTRODUCTION
In modern communication systems, purpose is to transmit the encoded information bits to the receiver correctly by a medium without introducing noise to the data. The Low­ density parity-check (LDPC) codes are fIrst introduced in 1962 by Gallager at MIT during PhD which has received signifIcant attention due to close error performance to Shannons Limit [1]. Thus the codes have been considered by many recent communication standards such as 10Giga-bit Ethernet [2], digital video broadcasting (DVB-S2) [3], [4], WiMAX (802.16e) , Wi-Fi and WPANs [5], [6].
LDPC decoder can be designed using three types of architectures. Serial architecture requires less hardware but deliver low decoding throughputs. In Semi-parallel architecture, row and column operations are performed partially in parallel and deliver higher throughput as compared to serial architecture. II. CONSTRUCTION OF LDPC CODE
LDPC is a linear error correcting code specifIed by a very sparse M x N parity check matrix, of which very few elements are 1's and others are O's. Column weight (we) is the number of one's per column and row weight Wr is the number of one's per row. The (n,k) LDPC code means k bit message can be encoded to be n bit codeword. The parity check matrix H of LDPC consists of (n-k) rows and n columns, and the code rate is kin. If (z=zO.....zn-l) is a codeword, it satisfIes parity check equation as shown in (1).

Graphical representation of LDPC code is known as Tanner graph. "Fig. 1" shows an example of (8, 4) LDPC code parity check matrix and its Tanner graph. Tanner graph consists of two set of nodes: check nodes and variable nodes. Rows and columns of parity check matrix H can be mapped to check node units and variable node units. An edge between check node and variable node is drawn only if there is '1' in the parity check matrix. The connection represents that the mutual infonnation can be transmitted among check nodes (i) and variable nodes (j), which is the foundation of message passing algorithm [9]-[11].

III. DECODING ALGORITHM
LDPC codes can be iteratively decoded in different ways and that decoding depends on the complexity and error performance requirements. Sum-Product (SP) algorithm and Min-Sum algorithm are soft decision optimum decoding algorithms. These algorithms are widely used in LDPC decoders and are known as standard decoders. These SP and Min-Sum algorithm perform row and column operations iteratively using two types of messages: check node message a and variable node message p.
