Construction of Block-LDPC Codes based on Quadratic Permutation Polynomials
Wu Guan and Liping Liang

Abstract: A new block low-density parity-check (Block-LDPC) code based on quadratic permutation polynomials (QPPs) is proposed. The parity-check matrix of the Block-LDPC code is composed of a group of permutation submatrices that correspond to QPPs. The scheme provides a large range of implementable LDPC codes. Indeed, the most popular quasi-cyclic LDPC (QC-LDPC) codes are just a subset of this scheme. Simulation results indicate that the proposed scheme can offer similar error performance and implementation complexity as the popular QC-LDPC codes.
Index Terms: Low-density parity-check (LDPC) codes, quadratic permutation polynomials (QPPs), quasi-cyclic (QC).
I. INTRODUCTION
USING iterative decoding, low-density parity-check (LDPC) codes can approach the capacity of various scenes at relatively low complexity [1]â€“[3]. Since their rediscovery in the 1990s, there have been significant improvements to the design and analysis of LDPC codes, such as optimizing the degree distributions according to a Gaussian approximation algorithm in an asymptotic sense [4], selecting large girth bipartite graphs through a progressive edge-growth (PEG) algorithm [5], or searching large minimum distance parity-check matrices via finite geometries [6].
Recently, LDPC codes were selected by the digital video broadcast (DVB) standard [7] and the worldwide interoperability for microwave access (WiMAX) standard [8]. Moreover, they are being considered for use in various real-life applications such as magnetic storage, 10 Gb Ethernet, and high-throughput wireless local area networks (WLANs). LDPC codes are being extensively used in various areas [9]â€“[12]. However, these schemes generally use quasi-cyclic LDPC (QC-LDPC) codes, whose submatrices must be the identity matrix or its cyclic shifters.
On the other hand, a greater number of LDPC codes with explicit algebraic/combinatorial construction have been reported with polynomial-based matrices instead of identity matrices. In [13], algebraic LDPC codes from finite classical generalized polygons are derived. This provides a method to construct codes with large girth and minimum distance, but allows little freedom to choose code lengths and code rates. C. Code Construction
The parity-check matrix of the Block-LDPC code is composed of sets of permutation matrices. The column indexes of the permutations are defined by QPPs in (17), with coefficients خ±i,j, خ²i,j, خ³i,j, د„ , and خ´. Coefficients خ±i,j are with constraints in (15). To find proper خ±i,j is essentially to construct a QC-LDPC code with the constraints in (14). Researchers have pay much attention to this field, and approaches for construction QC-LDPC codes are proposed in [24]â€“[27]. Adopting these methods, near Shannon capacity QC-LDPC codes can be designed, and coefficients خ±i,j can be picked out from the designed codes. On the other hand, the other coefficients are defined in (20). These are multiples of s or z. They can be easily calculated.
Once we obtain the coefficients for each permutation, we can caculate the column index Ci,j(x) for each permutation. Ci,j (x) is a QPP and can generate a permuted element sequence as Ci,j (0), Ci,j(1), آ· آ· آ·, Ci,j (p âˆ’ 1). Then, each p أ— p permutation Pi,j, will have â€œ1â€‌ in (0, Ci,j(0), (1, Ci,j (1), آ· آ· آ·, (p âˆ’ 1, Ci,j(p âˆ’ 1)) positions and â€œ0â€‌ in the other positions, where (x, y) denotes the row and the column index of submatrix Pi,j. In this way, based on the m أ— n distinct QPPs, we can get m أ— n permutation submatrices, and construct the parity-check matrix H = Pi,j for the Block-LDPC code. The code can avoid short cycles. Thus, it has a high performance.

The difference of implementation complexity between QPPbased Block-LDPC codes and classical QC-LDPC codes is due to the different permutations.
For a QPP-based Block-LDPC code, the permutation is obtained as (6). We can generate the permutation index Ci,j (0), Ci,j(1), آ· آ· آ·, Ci,j(p âˆ’ 1) as follows:

On the other hand, for a classical QC-LDPC code, the permutation is generated using (21), and there is only one mod-p addition. Thus, for each node, there is one additional mod-p addition in the implementation of permutation in the QPP-based Block-LDPC code than in the classical QC-LDPC code [13]. This results in only a small percentage change in the sum-product calculations.
Thus, due to the permutation structure of the Block-LDPC code, its implementation is essentially equivalent to that of a QC-LDPC code, which is the most suitable LDPC code for implementation. Thus, this code can be characterized as having low complexity.

V. CONCLUSION
In this paper, we proposed a new Block-LDPC code. This code utilizes a specific form of permutations that can be defined by QPPs. The scheme provides a large range of implementable LDPC codes, where the most popular QC-LDPC codes are just a subset of this scheme. Numerical results demonstrated that the Block-LDPC codes constructed by the proposed method offer good error correcting performance over a wide range. Moreover, the proposed scheme can be implemented with reasonable complexity in an efficient manner, making it suitable for high-speed communications.
