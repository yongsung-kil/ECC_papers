Abstract— Many classes of high-performance Low-density parity check codes are based on parity check matrices composed of permutation sub matrices. The emulation-simulation framework further allows the algorithm and implementation to be iteratively redefined to improve the error floor performance of message passing decoder. Log-Like hood-Ratio (LLR) based Belief-Propagation (BP) algorithm is presented for Low Density Parity Check codes. Numerically accurate representation of check node update computation used in LLR-BP decoding is described. The implementation of Sum-Product algorithm (SPA) within Low Density Parity Check Code (LDPC) decoder is described in this paper and the correction term is used to improve the decoding performance of min-sum algorithm (MSA). Quantization and log-tanh function approximation in sum-product decoder strongly affect which absorbing set dominates in error floor region. For LDPC decoder, bit error rate (BER) decreases with increase in the signal to noise ratio.
Keywords- Low-density parity check (LDPC) codes, message-passing decoding, error floor, iterative decoder,bit error rate.
I. INTRODUCTION
Low-density parity-check (LDPC) codes have been demonstrated to perform very close to the Shannon limit when decoded iteratively using a message-passing algorithm. LDPC codes are decoded using subclasses of message passing algorithm. Low Density Parity check codes are considered for the use in wide range of application. The LDPC codes requires very low-bit error rate (BER). The system required low bit error rates for two reasons, one is that Decoder required either long interconnects or long memory band width which makes system difficult to map onto Very Large Scale of integration and hence implementation of decoder is very complex. And another reason is that the current lack of adequate theory of analytically predicts the performance of the code past some moderate values of BER [2]. The experimental way to study the LDPC codes is via simulation. The work on LDPC codes has focused on floating point arithmetic. Low density parity check codes are evaluated using some message passing algorithm. This paper explores practical LDPC decoder design issues using simulation based

approach. In this paper sum-product and min-sum algorithm is proposed for reducing low-bit error rate. Sum-product algorithm has best decoding performance. Min-sum algorithm is introduced to reduce the complexity of sum-product algorithm [3]. The Min-sum algorithm simplifies horizontal process of sum-product algorithm [5]. The message-passing algorithm is revived in next section. LDPC codes suffer from Error Floor phenomenon. At low bit error rate the curve changes its slop and is called the error floor. In LDPC decoder, error floors are determined by small structure within the code that are specific to the selected graphical description of code. Error floor region is characterized by more gradual decrease in error rate as channel quality improves and this region appears at higher channel quality [6].

A. Decoding Algorithm And Aprroximation. Low density parity check codes are define by M×N parity
check matrix H. The matrix is sparse matrix. In M×N matrix, N represents number of bits in the codes and M represents number of parity checks. In LDPC codes, H matrix can be graphically illustrated by factor graph where each bit is represented by variable node (shown by circle) and each check is represented by factor node (shown as square ). The messages are passed between variable node and check node. An edge exit between variable node and check node if and only if H (j, i) = 1. Decoding Algorithm utilizes both sign and reliability information and it is called soft messaging. For example, assume binary phase shift keying (BPSK) modulation and additive white Gaussian noise (AWGN) channel. For transmission over the channel, the binary bits {0, 1} are represented by {1, -1}. In this the resulting soft information represents each received bit. In the process of message passing variable nodes are first initialized based on channel output. In the first step of decoding, check nodes receives the initial message from the nearer variable node and returns the information to variable nodes. The initialized first step is given by prior Log like hood ratio (LLR) and this is given in equation (1) given below.
