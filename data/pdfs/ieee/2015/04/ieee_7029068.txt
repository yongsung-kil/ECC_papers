Quasi-Cyclic Representation and Vector Representation of RS-LDPC Codes
Haiyang Liu, Qin Huang, Gang Deng, and Jie Chen

AbstractвҖ”RS-LDPC codes, constructed based on the codewords of Reed-Solomon (RS) codes with two information symbols, are an important class of LDPC codes. In this paper, we present two representations, namely, quasi-cyclic (QC) representation and vector representation, for RS-LDPC codes. Under the first representation, most part of the parity-check matrix of a full-length RS-LDPC code consists of circulant permutation matrices and zero matrices. As a result, the class of codes can enjoy the advantages in hardware implementation as QC-LDPC codes. In addition, the base matrix under the QC representation of an RS-LDPC code can be explicitly given such that the rank of its parity-check matrix can be analyzed combinatorially. Under the second representation, each permutation matrix in the parity-check matrix of an RS-LDPC code is defined by a nonbinary vector, whose entries are a permutation of entries in the field from which the RS code is constructed. Then, the вҖңaffine invarianceвҖқ property is proved for full-length RS-LDPC codes, which can facilitate the structural analysis of the codes.
Index TermsвҖ”RS-LDPC codes, Galois Fourier transform, QC codes, QC representation, vector representation.
I. INTRODUCTION
I N the recent years, low-density parity-check (LDPC) codes have been widely studied due to their remarkable errorcorrection capabilities under iterative decoding algorithms [1]вҖ“ [5]. Quasi-cyclic (QC)-LDPC codes [6]вҖ“[14] are an important class of LDPC codes within the family of algebraic LDPC codes. The quasi-cyclic structure of these codes can dramatically simplify their encoding and decoding implementations [8], [9]. As a result, most LDPC codes adopted in communication and storage systems are quasi-cyclic. Besides, the quasi-cyclic structure also allows us to theoretically analyze the properties of this class of codes [10]вҖ“[12]. One of the most important developments in the recent study of QC-LDPC codes (in fact, any QC codes) is the introduction of matrix transformation via Galois Fourier Transform (GFT) [12]. Furthermore, an efficient algorithm for encoding QC-LDPC codes was proposed based on the GFT [13], [14], which has much lower

Apart from QC codes, there are some other algebraic methods for the constructions of LDPC codes. Based on the codewords of Reed-Solomon (RS) codes with two information symbols, a class of regular LDPC codes, called RS-LDPC codes, can be constructed [15]. Thanks to redundant rows in their parity-check matrices, RS-LDPC codes perform very well and converge fast under iterative decoding, which make them very suitable for communication and storage systems. In 2006, IEEE 802.3an 10G Base-T Ethernet standard [16] adopted a (2048,1723) RS-LDPC code as its forward error-correction technique.
However, the parity-check matrix of an RS-LDPC code consists of permutation matrices rather than circulant matrices, which makes it difficult to implement their encoders and decoders. In fact, the (2048,1723) RS-LDPC code in 10G Base-T is one of the few non-QC standard codes adopted in communication systems. The silicon consumption of the (2048,1723) standard code, as well as its power consumption, have been tough problems in the development of 10G Base-T Ethernet chips. Besides, it is hard to analyze RS-LDPC codes in a theoretical way, and existing works on the code structure analysis were conducted with the aid of computers [17], [18]. To facilitate hardware implementation and theoretical analysis of RS-LDPC codes, we derive two representations, namely, QC representation and vector representation, for RS-LDPC codes in this paper.
First, we show that the parity-check matrix (PCM) of a full-length RS-LDPC code always contains a QC submatrix through proper row and column permutations. As a consequence, the hardware implementation of RS-LDPC codes can enjoy the advantages as QC-LDPC codes. In addition, the base matrix of QC submatrix under the QC representation of an RS-LDPC code can be explicitly given, thus the rank of its PCM can be analyzed combinatorially. We also extend QC representation and its applications to non-full-length RS-LDPC codes.
Second, we represent each permutation matrix in the PCM of an RS-LDPC code by a nonbinary vector, whose entries are a permutation of entries of the field from which the RS code is constructed. Consequently, the PCM of an RS-LDPC code can be described by a compressed matrix, named vector representation. Based on this representation, we prove the вҖңaffine invarianceвҖқ property for a full-length RS-LDPC code. In other words, the structural analysis, e.g., minimum-weight codewords, of a full-length RS-LDPC code is tantamount to the analysis of the structure containing any given position.

Next we would like to show that the above QC representation
can facilitate the implementation of encoding and decoding of
RS-LDPC codes.
First, we propose to encode a full-length RS-LDPC code C(Оі, q) based on its QC representation. Since HЛң (Оі, q) can be obtained from H(Оі, q) after row and column permutations, the code given by the null space of HЛң (Оі, q) is equivalent to C(Оі, q). With a slight abuse of notation, we use C(Оі, q) to represent the code given by the null space of HЛң (Оі, q) in the following
discussions. It follows from (15) that HЛң (Оі, q) contains a QC submatrix
HЛң QC(Оі, q), whose columns correspond to the last q2 вҲ’ 1 bits of a codeword in C(Оі, q). As a result, the 0-th bit can be easily calculated if all q2 вҲ’ 1 bits are known. Then, the encoding
process can be divided into two steps:
1) Encode the last q2 вҲ’ 1 bits using message vector and HЛң QC(Оі, q) by either shift register method or GFT method for QC codes.
2) Calculate the 0-th bit using one parity-check equation corresponding to any row in HЛң t .
It is well known that the encoding complexity of a general
linear code is proportional to the square of its code length, while
that of a QC code is only linearly proportional to the code length
[8]. As a result, the QC representation of RS-LDPC codes can
significantly reduce their encoding complexity compared with
traditional encoding methods [20], [21]. Second, let us consider the decoding of C(Оі, ПҒ). Thanks to the
QC representation, the decoder of C(Оі, ПҒ) can be implemented
in a configurable manner [9] with simple permutation networks
as a QC-LDPC code. It results in much less wire routing cost
and shorter critical path.
