A Hybrid ARQ Scheme Using LDPC Codes with Stochastic Decoding
Kuo-Lun HuangÔÇá, Vincent C. GaudetÔÇí, and Masoud SalehiÔÇá
ÔÇáDepartment of Electrical and Computer Engineering, Northeastern University, Boston, Massachusetts ÔÇíDepartment of Electrical and Computer Engineering, University of Waterloo, Waterloo, Ontario huang.ku@husky.neu.edu, vcgaudet@uwaterloo.ca, salehi@ece.neu.edu

AbstractÔÇöStochastic decoding is a hardware and energyefficient approach to implement iterative decoders. We use this decoding method in an Automatic Repeat reQuest (ARQ) scheme to improve the performance of Low-Density Parity-Check (LDPC) decoders. We show that, aside from the simplicity in hardware implementations, stochastic decoding results in improved error rate performance and requires fewer retransmissions than other iterative decoding algorithms when used in a Hybrid Automatic Repeat reQuest (HARQ) system. We present an analysis of the error rate improvement and decoding latency of the HARQ scheme using stochastic decoding. We also provide a criterion to efficiently allocate the available decoding cycles to the decoding attempts in the HARQ system in order to achieve lower error rates and less decoding latency.
Index TermsÔÇöAutomatic repeat request (ARQ), iterative decoding, low-density parity-check (LDPC) codes, stochastic computation, stochastic decoding.
I. INTRODUCTION
The ever increasing demand for high-speed wireless communications in our daily lives has resulted in development of new techniques for error-control coding that have become an integral part of emerging standards. For instance, in LongTerm Evolution (LTE) [1] and WiMAX (IEEE 802.16) [2] standards, Hybrid Automatic Repeat reQuest (HARQ) [3] is introduced to enhance service reliability.
The HARQ scheme is a combination of Forward Error Correction (FEC) and Automatic Repeat reQuest (ARQ) errorcontrol method. We adopt the type-I HARQ scheme [4], where a codeword is transmitted initially and if the receiver fails to decode the codeword, a retransmission is requested by sending a Negative-Acknowledgement (NACK) message back to the transmitter. Upon receiving a NACK, the transmitter sends the same codeword again. After an Acknowledgement (ACK) message is received, or the maximum number of retransmissions is achieved, the next codeword is transmitted. This stop-and-wait method is the simplest ARQ scheme; however, it is less efficient than more complicated schemes due to the time-out delay between the transmissions. If the application requires higher channel throughput, other ARQ types, such as ÔÇ£go-back-NÔÇØ and ÔÇ£selective repeatÔÇØ are more suitable.
We consider Low-Density Parity-Check (LDPC) codes [5], which provide near-capacity performance [6], as the FEC code

in the HARQ scheme. LDPC decoding is usually performed iteratively using the well-known Sum-Product Algorithm (SPA). The SPA iteratively computes a posteriori probability messages. The SPA, however, requires a fairly high computational load and may suffer from routing congestion in hardware implementations. To alleviate these problems, stochastic computation was proposed as an alternative to the SPA [7][8]. In stochastic computation, stochastic bit streams are randomly generated to represent the probability values. By using stochastic representation, messages in iterative decoders are computed and transmitted serially, i.e., one stochastic bit for each iteration known as a Decoding Cycle (DC) [8]. All computations are carried out by bit-wise operations and can be constructed by simple logic circuits. Due to the efficiency in hardware implementation, stochastic decoding has been used to decode a (16,8) LDPC code [7], a (1024,512) LDPC code [9], the (1056,528) LDPC code employed in the WiMAX standard [8], the (2048,1723) LDPC code used in the 10GBASE-T standard [10], and a growing list of other types of FEC codes [11]-[14].
In this work, we present the advantages of using stochastic decoding instead of other iterative decoding algorithms in an HARQ scheme. We analyze and compare the error rate performance of the SPA, the Min-Sum Algorithm (MSA), and the stochastic decoding algorithm to show that the stochastic decoding algorithm is the most suitable choice to be used in an HARQ scheme. In addition, we present the error rate performance and decoding latency of HARQ systems using these algorithms.
The rest of this paper is organized as follows. Section II presents the system model of the HARQ scheme using stochastic decoding. Performance analysis and comparison with other iterative decoding algorithms are provided in Section III. Simulation results of an HARQ system using stochastic LDPC decoding are given in Section IV, where we discuss the results and the performance of HARQ systems with different distribution of available DCs. Finally, Section V gives the conclusions.

V. CONCLUSIONS
In this paper, an HARQ scheme based on using stochastic LDPC decoding is presented. We explore the advantages of using stochastic decoding compared to other iterative decoding algorithms in an HARQ system. In addition to the simplicity in implementations, we show that the stochastic decoding algorithm has lower FER for having similar BERs. Thus, the expected retransmission number and error rate of the HARQ system are lower while using stochastic decoding algorithm instead of the SPA or the MSA. Simulation results prove that employing the stochastic LDPC decoder in the HARQ system can provide more than 3.5-dB BER improvement with comparable decoding latency. We also present a criterion for optimal distributing of DCs among decoding attempts for stochastic decoding in the HARQ scheme to achieve lower error rates and less decoding latency.
