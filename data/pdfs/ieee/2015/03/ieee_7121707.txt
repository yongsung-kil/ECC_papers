Spatially-Coupled LDPC Protograph Codes for Universal Phase Slip-Tolerant Differential Decoding

Laurent Schmalen(1), Stephan ten Brink(2), and Andreas Leven(1)
(1) Bell Laboratories, Alcatel-Lucent, Lorenzstr. 10, Stuttgart, Germany (2) Institute of Telecommunications, University of Stuttgart, Pfaffenwaldring 47, Stuttgart, Germany
Laurent.Schmalen@alcatel-lucent.com
Abstract: We show how a single spatially coupled LDPC code constructed from an optimized protograph with optimized windowed decoding outperforms different conventional LDPC codes in iterative differential decoding with significantly lower error floors and identical complexity.
OCIS codes: (060.0060) Fiber optics and optical communications, (060.1660) Coherent communications.
1. Introduction In coherent receivers for high-speed optical communications, usually feed-forward phase recovery algorithms exploiting the rotational invariance of the constellation are used. Due to the necessary phase unwrapping algorithm in these estimators, phase slips (also referred to as cycle slips) occur. The probability that a phase slip occurs depends among others on the non-linear phase noise introduced by the optical transmission link. If the phase slip is not recovered, error propagation will occur at the receiver. Several approaches to mitigate phase slips have been proposed where the most common is differential coding, which renders a phase slip into a single error event. In order to alleviate the differential coding penalty, iterative decoding between an FEC decoder and a differential decoder is beneficial [1–3].
Special care must however been taken for adapting the soft-input soft-output differential decoder to the channel model at hand. In [1], it was shown that with a standard differential decoder, large error floors may occur with a residual bit error rate depending on the phase slip’s probability of occurrence. In [2], the error floor was combated by activating the differential decoder only in some well-selected iterations. Although this solution—denoted Hybrid Turbo Differential Decoding (HTDD)—is able to lower the error floor, a hard decision outer code is still mandatory to guarantee sufficiently low residual bit error rates. Another possibility is to modify the differential decoder to take into account the phase slip channel model, as proposed in [3].
In this paper, we first show by an EXtrinsic Information Transfer (EXIT) chart analysis why the conventional iterative differential decoder fails in the presence of phase slips. We show how a modified differential decoder based on an extended trellis representation may lead to an improved performance. In the second part of this paper, we employ this modified differential decoder in an iterative windowed decoder with spatially coupled (SC) LDPC codes. We show how to generate good, universal (i.e., channel agnostic) SC LDPC codes that yield superior performance under windowed decoding. We employ optimized codes constructed from small prototype graphs (protographs) [6].

with decoder setup 3 almost shows the same performance than LDPC Code (2) with advantages at low error rates (we

did not observe any error floor). The proposed SC code thus has the advantage of universality [7]: A single SC code

is powerful in all transmission scenarios requiring only a single channel agnostic transmitter and a receiver selecting

the best decoding setup depending on Pslip. The transmission scheme can thus be kept simple and only a single code

needs to be implemented (instead of a multitude of codes as would be needed with conventional LDPC codes).

5. Conclusions
We have shown that a single spatially coupled LDPC code based on a protograph in the context of iterative differential decoding can outperform a multitude of LDPC codes each optimized for different channel conditions at identical decoding complexity. Only minor variations of the decoder are necessary, leading to lightweight transceivers.
