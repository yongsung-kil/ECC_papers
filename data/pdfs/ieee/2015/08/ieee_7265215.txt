Iterative Reliability-Based Modified Majority-Logic Decoding for Structured Binary LDPC Codes
Haiqiang Chen, Lingshan Luo, Youming Sun, Xiangcheng Li, Haibin Wan, Liping Luo, and Tuanfa Qin

Abstract: In this paper, we present an iterative reliability-based modified majority-logic decoding algorithm for two classes of structured low-density parity-check codes. Different from the conventional modified one-step majority-logic decoding algorithms, we design a turbo-like iterative strategy to recover the performance degradation caused by the simply flipping operation. The main computational loads of the presented algorithm include only binary logic and integer operations, resulting in low decoding complexity. Furthermore, by introducing the iterative set, a very small proportion (less than 6%) of variable nodes are involved in the reliability updating process, which can further reduce the computational complexity. Simulation results show that, combined with the factor correction technique and a well-designed non-uniform quantization scheme, the presented algorithm can achieve a significant performance improvement and a fast decoding speed, even with very small quantization levels (3–4 bits resolution). The presented algorithm provides a candidate for trade-offs between performance and complexity.
Index Terms: Iterative decoding, low-density parity-check (LDPC) code, majority-logic decoding, reliability-based, structured LDPC codes.
I. INTRODUCTION
THERE exist two commonly used construction methods for low-density parity-check (LDPC) codes: 1) Random construction aided by computer searching [1], [2] and 2) systematic construction based on algebraic structures [3], [4]. The second algebraically constructed method generally leads to the structured LDPC codes, which show some advantages as follows. First, most of the structured LDPC codes are cyclic or quasicyclic and can be encoded in linear time with the simple feedback shift registers [5]. Second, the structured LDPC codes per-

Finally, the structured LDPC codes usually have large row/column weights, which implies that they are majority-logic decodable LDPC codes and thus can be decoded by the majority-logic based decoding algorithms.
Decoding methods for LDPC codes are usually categorized into three classes: 1) Hard-decision decoding; 2) probabilitybased decoding; and 3) reliability-based decoding (also known as hybrid decoding). From an implementation point of view, hard decision decoding algorithms, such as the one-step majority-logic decoding (OSMLGD) algorithm [3] and the bitflipping (BF) decoding algorithm [6], have the lowest complexity. However, the lowest complexity often results in serious performance degradation. The probability-based iterative decoding method, such as the well-known sum-product algorithm (SPA) [7], generally has the best bit-error-rate (BER) performance and the fastest decoding convergence speed, but it requires very high computational complexity.
Among these three decoding methods, the reliability-based decoding algorithms can make efficient trade-offs between decoding performance and computational complexity. The well-known reliability-based algorithms are the modified onestep majority-logic decoding (MOSMLGD) algorithm, the weighted bit flipping (WBF) algorithm ,and their improved versions [3], [8], [9]. Huang et al. proposed a reliability-based iterative majority-logic decoding (RBI-MLGD) algorithm [10] and a weighted iterative soft-reliability-based majority-logic decoding (WISRB-MLGD) [11], which perform well with very low decoding complexity. Chen and Zhang et al. extended Huang’s work in [12]–[14].
For the (M)OSMLGD algorithms, no iterative strategies are involved (only one iteration is performed). This may cause serious performance degradation, since no further intrinsic information (or reliability measure) can be used for the decoder. In this paper, we present an iterative reliability-based modified majority-logic decoding algorithm for structured LDPC codes. Instead of directly performing the bit-flipping operation, we design a turbo-like iterative strategy. The integer reliability measure of the code bit can be iteratively updated in the decoding process. The main computational loads of the presented algorithm include only binary logic and integer operations, resulting in low decoding complexity. Furthermore, we have introduced the iterative set associated with the weighted check-sum reliability. Only those variable nodes that satisfy such iterative set will perform the iterative operation. This is quite different from most of the existing reliability-based decoding algorithms and can further reduce the decoding complexity. B. System Model Encoding: Let u = (u0, u1, · · ·, uk−1) ∈ Fk2 be an in-
formation sequence to be transmitted. This information sequence u is encoded by a binary LDPC encoder into a codeword c = (c0, c1, · · ·, cn−1) ∈ F2n.
Mapping: Assume that the simple BPSK modulation is employed in this paper. The codeword c is transformed into a bipolar signal vector x = (x0, x1, · · ·, xn−1), where xj = 2cj − 1.
Channel: The signal vector x is transmitted through an additive white Gaussian noise (AWGN) channel. The received vector is denoted by y = (y0, y1, · · ·, yn−1), where yj = xj + nj and nj is a white Gaussian noise sample.
Quantizing/Decoding: The receiver attempts to recover the transmitted data u based on the received signal y. For some application scenarios (such as to guarantee the integer operation in the iterative decoding process), the real received symbols are quantized into integer values before passing to the decoder.

IV. THE ITERATIVE RELIABILITY-BASED MODIFIED MAJORITY-LOGIC DECODING ALGORITHM
It is shown that the OSMLGD and its modified version have no iterative strategies (only one iteration is performed). This may cause serious performance degradation, since no further intrinsic information (or reliability measure) can be used for decoding the received signals. In this section, we present an iterative reliability-based modified majority-logic decoding algorithm. Instead of directly performing the bit-flipping operation, we design a turbo-like iterative strategy. The integer reliability messages can be iteratively updated in the decoding procedure. Combined with the factor correction technique and a welldesigned non-uniform quantization scheme [14], the presented algorithm shows a significant performance improvement and a fast decoding convergence rate, even with very small quantization levels.

C. Iterative Decoding Strategy at Variable Nodes

Unlike the (M)OSMLGD algorithms which directly flip the
hard-decision bits that fall into the flipping set, we introduce an iterative strategy scheme in this subsection. In the kth it-
eration, the reliability measure associated to the variable node Vj is updated by the newly updated reliability pj(k) from the weighted check-sum indicating vector p(k). This updating strat-
egy resembles the so-called turbo-principle rule, which is com-
monly used in the existing iterative LDPC/Turbo decoding algorithms [10], [12]. Let Rj(k+1) be the updated reliability to be used in the (k + 1)th iteration, then the updating rule can be
defined as follows:

Rj(k+1) = Rj(0) + sign(Rj(k))[αp(jk)]

(13)

for j ∈ Jpresented, where α is a pre-defined scaling factor and [x] denotes the nearest integer to the real value x.
The iterative strategy above is performed only for the code bits falling into the set Jpresented. In this case, pj(k) always takes negative value, indicating the current code bit zj(k) is unreliable. Thus, the reliability Rj(k+1) is designed to move away from the current reliability value Rj(k) (in an opposite direction) while the channel reliability Rj(0) remains unchanged.
Note: The processing at check nodes of the presented algo-
rithm is equivalent to that of the RBI-MSD algorithm in [12],
and the decoding complexity can be reduced by introducing the
iterative set at variable nodes with a slight performance degra-
dation. Hence, the presented algorithm can provide a candidate
for trade-off between performance and complexity.

D. Summary of the Algorithm
Now we can summarize the presented iterative reliabilitybased modified majority-logic decoding algorithm as follows.
The iterative reliability-based modified majority logic de-
coding algorithm: • Input: The received vector y, quantization parameters r, b, the
scaling factor α and the maximum iteration number Imax; • Initialization: Quantize the vector y into q according to (9);
set the loop counter k = 0; for 0 ≤ j ≤ n − 1, set Rj(0) = qj; • Iterations: While k < Imax, do the following steps:
1. For 0 ≤ j ≤ n − 1, make hard-decision zj(k) = 1 if Rj(k) ≥ 0 and zj(k) = 0 otherwise.
2. For 0 ≤ i ≤ m − 1, compute the syndrome s(ik); if s(k) = (s0(k), s(1k), · · ·, s(mk−) 1) = 0, exit the iteration.
3. Compute the weighted check-sum indicating vector p(k)
according to (11).

4. Compute the iterative set Jpresented according to (12). 5. Update the reliability Rj(k+1) of the jth variable Vj accord-
ing to (13), where j ∈ Jpresented. 6. Increment k by 1. • Output: output z(k) as the decoded codeword. Remarks: • Different from the conventional hard-decision decoding algorithms, we do not perform the flipping operation at the variable node. Instead, we iteratively update the newly reliability and let it shift to a designated direction; • In the iterative process, only those variable nodes that belong to the iterative set are involved. This is quite different from most of the existing reliability-based decoding algorithms and can further reduce the decoding complexity.
V. PERFORMANCES AND COMPLEXITIES
A. The Computational Complexity
The computational complexity of the presented algorithm can be analyzed as follows. Step 1 requires n XOR operations (EXCLUSIVE-OR, binary operation) to obtain the harddecision vector. To compute the syndrome vector in Step 2, we need mρ−m XOR operations. In Step 3, we need: 1) mρ(ρ−2) integer comparisons to compute the weighted reliability (see (10)); 2) nγ−n integer additions to compute the weighted checksum indicating vector (see (11)). Step 4 requires n XOR operations to compute the iterative set (see (12)). In Step 5, to update the reliability, we need n′ integer additions and n′ real multiplications to scale the weighted reliability and n′ XOR operations to test the sign of the current reliability (see (13)), where n′ is the order of the set Jpresented. Therefore, to carry out one iteration of the presented algorithm, a total of 2n + (ρ − 1)m + n′ binary operations, nγ − n + n′ integer additions, mρ(ρ − 2) integer comparisons and n′ real multiplications are required. The computational complexities are summarized in Table I. For comparison, we have also included the complexity of the RBI-MLGD algorithm presented in Huang et al.’s work [10].
It is shown that for the presented algorithm quite a part of decoding operations are introduced by the variable nodes, including the n real multiplication operation. However, we emphasize that only a small proportion (about 5%, see Table II in Example 3) of variable nodes are involved in the reliability updating process, since the order of the iterative set may drastically reduce with the iterations.

VI. CONCLUSION
In this paper, we have presented an iterative reliabilitybased modified majority-logic decoding algorithm for the finitegeometry LDPC codes and LDPC codes constructed based on finite fields that have heavy row/column weights. With the turbo-like iterative strategy, the reliability messages can be iteratively updated in the decoding process. The main computational loads of the presented algorithm include only binary logic and integer operations, resulting in low decoding complexity. To

furthermore reduce the computational complexity, we have introduced the iterative set. Only those variable nodes satisfying such iterative set will perform the iterative operation. It is shown by simulations that only a small proportion (less than 6%) of variable nodes are involved in the reliability updating process. This is an attractive result for reduced complexity iterative decoding. Combined with the factor correction technique and a well-designed non-uniform quantization scheme, the presented algorithm can achieve a significant performance improvement and a fast decoding speed, even with very small quantization levels (3–4 bits resolution). Finally, we point out that the presented algorithm is only applicable to structured LDPC codes with heavy column weights. When decoding random-like LDPC codes with relatively small column weight, the presented algorithm exhibits serious performance degradation.
