Approximate Algorithms for Identifying Minima on Min-Sum LDPC Decoders and Their Hardware Implementation
Ioannis Tsatsaragkos and Vassilis Paliouras, Member, IEEE

Abstract—This brief introduces algorithms and corresponding circuits that identify minimum values among a set of incoming messages. The problem of finding two minima in a set of messages is approximated by the different problem of finding the minimum of all messages in the set and the second minimum among a subset of the messages. This approximation is here shown to be suitable for hardware low-density parity-check decoders that implement a min-sum (MS) decoding algorithm and its variations. The introduced approximation simplifies the operation performed in a check-node processor and leads to hardware reduction. The proposed schemes outperform other state-of-the-art simplified MS architectures, approaching the error-corrective performance of the normalized MS decoding algorithm.
Index Terms—low-density parity-check (LDPC) codes, min-sum (MS) decoding algorithm, two minima computation.
I. INTRODUCTION
L OW-DENSITY parity-check (LDPC) codes have been adopted in a variety of wireless communications standards, such as WiMAX and WiFi, mainly due to their excellent error-corrective performance and inherent parallelism. An LDPC code is a linear error correction code that is fully defined by a sparse binary parity-check matrix H [1]. Matrix H can be represented in a graphical form by a bipartite graph, which is called a Tanner graph [2]. This graph is composed of two sets of nodes, i.e., the check nodes and the variable nodes. Although a sum-product (SP) decoding algorithm is considered near optimal [3], LDPC decoder implementations must take into account considerations such as the size and latency. Thus, in an effort to reduce the complexity, hardware decoders commonly use the area-efficient approximations of the SP algorithm, such as a min-sum (MS) algorithm [4] and its variations (e.g., normalized MS (NMS) and offset MS (OMS) [5]). The complexity is reduced by the MS algorithm at the cost of error correction performance degradation. By introducing an additional normalization, or offset, operation at check nodes, the resulting NMS and OMS algorithms significantly improve the performance of the MS algorithm, with a negligible increment in the hardware complexity.

An operation that is typically performed within the check nodes of the LDPC decoder that employs the MS algorithm involves identifying the two minima from a greater number of input messages. For a large number of inputs, the complexity of this operation increases dramatically. Existing methods tend to require circuitry having complexity that is relatively high in terms of hardware requirements, latency, and power dissipation.
Several authors have proposed the approximations of the two minima calculation. Darabiha et al. introduced a simplified MS algorithm that only computes the absolute minimum value [6]. The second minimum is estimated, adding a corrective constant to the minimum value, which depends on the number of inputs that share the same minimum magnitude. This single-minimum MS (smMS) algorithm achieves significant reduction in the hardware complexity at the cost of bit-error rate (BER) performance degradation. Wang et al. improved the performance of the smMS algorithm by applying an effective modification factor (mfMS) on the absolute minimum value [7]. This approach has been used by Zhang et al. to design a flexible multigigabit shift-LDPC decoder [8]. Angarita et al. analyzed the behavior of the two minima on each iteration of the MS decoding algorithm [9]. Motivated by the observation that these values and their difference increase in every new iteration, they proposed the use of variable iteration-dependent correction factors. The introduced variable-weight algorithm (vwMS) exhibits better error correction performance than the smMS and mfMS algorithms. Moreover, Angarita et al. [9] proposed a simplified vwMS algorithm, which is called svwMS, that reduces the high computational cost required to determine if more than one input shares the same minimum value.
We present a class of algorithms, which is called exM in, that reduces the hardware required for the computation of the two minima. An exMin algorithm identifies a first value that is the minimum value of the input messages and a second value that is an estimation of the second minimum. Thus, exMin approximates the problem of finding the two minima by a simpler problem, the solution of which requires substantially fewer comparisons. The implementations of exMin achieve BER performance similar to the conventional NMS algorithm. For some decoding scenarios, the application of a reduction factor to the estimated second minimum value is shown to improve the performance of the exMin approximation.

B. rExMin-n Algorithm
The exMin-n algorithm identifies a min∼2 value that approximates the second minimum of the input messages, and it is always greater than or equal to the true second minimum value. In addition to proposing exMin-n, we also propose a variant of it, which we call rExMin-n, where the resulting min∼2 value is properly reduced by adding a negative factor r. Simulations have shown that the application of this reduction factor on the min∼2 value may reduce the BER degradation, which is caused by the exMin-n approximation.
The sum of r and min∼2 may be lower than or equal to the true minimum value min1. When r + min∼2 ≤ min1, the rExMin-n algorithm returns min∼2 as its second minimum estimation; otherwise, the algorithm returns r + min∼2. The rExMin-n algorithm comprises the following steps.

The smMS, mfMS, and vwMS algorithms substitute the

computation of the two minima with a computation of the min-

imum (min1) and an estimation of the second minimum value. The complexity of the minimum identification, for a set S0 of p values, is p − 1 comparators and p − 1 multiplexers. These

algorithms require an exhaustive search to determine if more

than one absolute minimum value exists. The hardware cost is

affordable for the bit-serial check-node architectures presented

in [6] and [7]; nevertheless, it is not acceptable for parallel

check-node operations, where this search should be completed

in a single clock cycle. In order to reduce the complexity, the

svwMS algorithm divides the input messages into two groups,

and the minimum values of these groups are compared [9].

If these values are equal, the second minimum is considered

equal to min1. Otherwise, a variable weight factor w is added to min1 in order to make an estimation of the second minimum.

This operation requires one comparator, one multiplexer, and

one adder. Additionally, an svwMS implementation with m

different weight factors wi requires m − 1 2-to-1 multiplexers

for the selection of w.

IV. CONCLUSION
The implementations of the proposed schemes substantially reduce the complexity of an MS check-node processor at the cost of a negligible degradation in the correction capability of a decoder. Simulations suggest that the adoption of exMin computation units in MS LDPC decoders can achieve a satisfactory coding gain, outperforming the state of the art for simplified MS algorithms, particularly for small maximum NOIs. By selecting an appropriate number of approximation levels, we succeed to effectively explore a tradeoff between the BER performance and the complexity.
