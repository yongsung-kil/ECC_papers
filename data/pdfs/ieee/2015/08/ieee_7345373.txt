A New Solution Based on Multi-Rate LDPC for Flash Memory to Reduce ECC Redundancy
Shigui Qi1,2,Dan Feng 1 ,Nan Su1,Wenguo Liu1 and Jingning Liu1 1 Wuhan National Lab for Optoelectronics
School of Computer Science and Technology Huazhong University of Science and Technology
Wuhan, China 2 School of Information Engineering, Xuchang University Email: qisg, dfeng, sunan2010@hust.edu.cn, liuwenguo hust, j.n.liu@163.com

Abstract—Low-density parity-check (LDPC) code can provide powerful error correcting performance for NAND flash memory. Different LDPC code rate has different error correcting performance. Moreover, the raw bit error rate of flash memory is very low in the early lifetime. This will generate ECC redundancy that the error correcting performance of LDPC cannot be completely released. We propose a new Switch LDPC (S-LDPC) algorithm based on Multi-Rate LDPC code to reduce ECC redundancy and meet different error correcting requirement in the different periods of flash memory. S-LDPC algorithm can achieve optimal tradeoff among error correcting performance, decoding energy consumption and read performance. The extensive experiments show that S-LDPC algorithm can improve the average read response time of flash memory 25%-54% without reducing the reliability of flash memory. We further demonstrate that LDPC code with code rate 0.96 can save about 40% decoding energy consumption than LDPC code with code rate 0.7.
Keywords-flash memory; ECC redundancy; LDPC; multi rate; energy consumption

II. BACKGROUND
NAND flash memory contains many word-lines (WL) and bit-lines (BL) which can be seen as an array. The intersection of a word-line and a bit-line is a cell. A cell is a floating gate transistor that can be programmed by injecting a certain amount of charges into the floating gate. A MLC NAND flash memory cell stores two bits which are respectively mapped to two pages: least significant bit (LSB) page and most significant bit (MSB) page. All cells on the same word-line constitute a page. A block contains many pages. Before a flash memory cell is programmed, it must be erased. The erased unit is block in flash memory. NAND flash memory usually use ECC to meet the data reliability requirement.
There are two noises induced by P/E cycles: Random telegraph noise (RTN) and Cell-to-Cell Interference (CCI). Data retention noise is another noise which can make charge in the floating gate leak with the increase of data retention time. According to the paper [4], [5], we have created a flash memory noise model [6]. This work is based on this flash memory noise model.
In this work, we adopt QC-LDPC code to correct errors in flash memory. The parity-check matrix of QC-LDPC code is M×N sub-matrix. Each sub-matrix is a z×z cyclic-shifted identity matrix or zero-matrix. Since the parity check matrices of Multi-rate QC-LDPC have similar structures, their decoding processes are similar. III. PROPOSED ALGORITHM
The raw bit error rate of NAND flash memory is increasing gradually with P/E cycles, data retention time and other interfering factors. During the early lifetime of NAND flash memory, flash memory has few errors. Normally, memory manufactures must use strong ECC to meet the reliability requirement of flash memory in the worst case. There is ECC redundancy that the error correcting capability of ECC is under-utilized over the entire flash memory lifetime as shown in Fig. 1(a).
LDPC code is widely used in many areas due to its powerful error correcting performance. LDPC codes with different code rates have different error correcting performance [7] [8]. Conventional research only focuses on the way that apply one LDPC code rate to correct errors of flash memory. If only one strong LDPC code is used in flash memory, its error correcting capability will not be completely utilized especially in the early lifetime.
A. S-LDPC Algorithm Design
According to the error correcting performance difference of different LDPC code rates, we propose S-LDPC algorithm based on Multi-Rate LDPC code to correct errors of flash memory for the first time. The processing unit of S-LDPC is block. The basic idea is that we firstly apply weak ECC for flash memory. When weak ECC fails correcting errors, we replace weak ECC with strong ECC to correct errors as shown in Fig. 1(b).
We set LDPC code with low code rate as strong ECC and set LDPC code with high code rate as weak ECC. For each block, we use a flag Block-ECC to indicate which ECC is used in the block. We initialize Block-ECC to 0. It means that weak ECC is used at the first time. Otherwise, we set Block-ECC to 1 when strong ECC is applied.
The S-LDPC algorithm is illustrated in Fig. 2. When the block is implementing strong ECC to correct errors, we stop switching between weak ECC and strong ECC until the block is erased. Because the erased block becomes new blank storage area and the noises of the block become few. We set a threshold to avoid frequent switching between strong ECC and weak ECC especially in the later lifetime of flash memory. The threshold is set according to the actual situation of flash

We implement multi-rate LDPC code with QC-LDPC parity check matrix. In this work, we use two LDPC code rates to evaluate the S-LDPC algorithm. Firstly, we encode raw user data with two LDPC codes: code rate 0.7 and code rate 0.96. LDPC code is systematic code that raw user data and parity bits can be separated from each other. Therefore, we can generate parity bits of two LDPC code rates simultaneously. For the decoding of S-LDPC algorithm, we can use different LDPC parity bits to generate different LDPC codewords. Then the codeword is decoded by the corresponding LDPC decoder according to the S-LDPC algorithm.
S-LDPC algorithm will increase more additional storage space of parity bits than that of only using one LDPC code rate. Fortunately, the increase cost of storage space is small. For example, the parity bits of LDPC code rate 0.96 is only 10% that of LDPC code rate 0.7.
B. Performance of S-LDPC
1) Program performance: The time of programming data consists of LDPC encoding time and the time of transferring LDPC codeword into flash memory. From the definition of LDPC code rate, the codeword length of low code rate is bigger than that of high code rate. Therefore, the programming time of low code rate is longer than that of high code rate. When we carry out the encoding of S-LDPC algorithm, the codeword consists of raw user data and the parity bits of two LDPC codes. Therefore, the programming time of S-LDPC algorithm is longer than that of one LDPC code rate.
2) Read performance: The read time is mainly LDPC decoding time. The time of LDPC decoding includes three parts: soft-decision information sensing time, transferring softdecision information time and LDPC decoding time. Especially, LDPC decoding needs soft-decision information per bit of codeword. The longer the codeword length is, the more time soft-decision information sensing has. Therefore, the softdecision information sensing time of low LDPC code rate is longer than that of high LDPC code rate. Soft-decision information transferring time of low LDPC code rate is also longer than that of high code rate. In addition, the LDPC decoding time is related to the iteration number.
C. Decoding Energy

Addressing time is 5 clocks. The accessing time is related to threshold voltage sensing levels and the data transferring time from NAND flash memory to the register. Average iteration per page is obtained by the total iteration over the page number as shown in Table I. We find that average iteration per page is decreasing with the increase of SNR point. It means that LDPC decoding can converge quickly at high SNR point. On the other hand, the average iteration per page of LDPC code with high code rate is less than that of LDPC code with low code rate at the same SNR point. This means that LDPC code with high code rate finishes the decoding process quicker than LDPC code with low code rate. Therefore, LDPC code with high code rate will save more decoding energy.
At different SNR points, LDPC code rate 0.7 has better error correcting performance than LDPC code rate 0.8 and LDPC code rate 0.96 as shown in Fig. 4 and Fig. 5. With the increase of SNR point (it means that noises are decreasing), the error correcting performance of LDPC code becomes more powerful. For instance, as shown in Fig. 4, the error correcting performance of LDPC code rate 0.7 attains a gain about 1 dB compared to the LDPC code rate 0.8 for LSB page at error rate 4×10−4. The gap between LDPC code rate 0.7 and LDPC code rate 0.96 is also about 2 dB at error rate 2.5×10−4 for MSB page as shown in Fig. 5. With the decreasing of SNR point (it means the noises are increasing), the error correcting performance gap between low LDPC code rate and high LDPC code rate is increasing. This means that the error correcting capability of low LDPC code rate is more powerful than that of high LDPC code rate when noises are increasing.
B. Program and Read Performance of S-LDPC Algorithm
To evaluate the proposed technique quantitatively through trace-driven simulations, we adopt SSD module [10] in DiskSim [11] with five real workload traces including Finacial1 (72% writing), Finacial2 (14% writing), Postmark (17% writing), WebSearch1 (0% writing) and WebSearch2 (0% writ-

ing). We also use a synthetic workload which includes 50% writing requests. Since these traces are from HDDs and HDDs are much slower than SSD, we have shorten the timestamps of traces 1000 times to simulate real SSD environment. In this work, the simulator SSD has 8 flash chips. Each flash chip has 2 dies which share some control signals and an 8-bit bus. One die contains 4 planes and one plane has 2048 blocks. One block has 64 2KB pages. We set one page reading speed as 20.5 MB/s and set one page writing speed as 3 MB/s [13]. One block erase latency is 1.5 ms. Based on open NAND flash interface (ONFI) version 2.1 [14], we set the bus frequency of flash memory as 100 MB/s. For the purpose of performance comparison, we select LDPC code rate 0.7 as the baseline.
1) Read Performance: We set LDPC code rate 0.96 as weak ECC and set LDPC code rate 0.7 as strong ECC. Firstly, we carry out LDPC code with code rate 0.96 to correct errors. When LDPC code rate 0.96 cannot correct errors, we use LDPC code rate 0.7 instead of LDPC code rate 0.96 since the former has stronger error correcting capability.
Read Performance only using weak ECC. We firstly compare the read performance between LDPC code rate 0.7 and LDPC code rate 0.96. It means that flash memory only uses weak ECC (LDPC code rate 0.96) in the S-LDPC algorithm. For each trace, the read response time is normalized to the read response time of LDPC code rate 0.7. We can see that the read performance of LDPC code rate 0.96 is better than the baseline as shown in Fig. 6(a). For example, the read response time of LDPC code rate 0.96 is reduced by 54% than the baseline for Finacial2. The read response time of LDPC code rate 0.96 is reduced from 52% (for WebSearch1) to 24% (for Synthetic) compared with the baseline.
Read Performance combining weak ECC with strong ECC. When weak ECC cannot correct errors, flash memory will use strong ECC according to S-LDPC algorithm. In this work, we consider two cases that combine weak ECC with strong ECC. The first case is that 5% of the blocks use strong ECC and 95% of the blocks use weak ECC. Another case is that 20% of the blocks use strong ECC and 80% of the blocks use weak ECC. The results demonstrate that the read response time is increasing when strong ECC is used. For the first ECC combination case, the read response time is reduced by 25% than the baseline for Finacial2 as shown in Fig. 6(b). However, the read response time of only using LDPC code rate 0.96 is reduced by 54% than the baseline for Finacial2 as shown

On the other hand, the decoding energy consumption of high LDPC code rate is lower than that of low LDPC code rate. Therefore S-LDPC algorithm can achieve a tradeoff between error correcting performance and decoding energy consumption during different lifetime of flash memory.
V. CONCLUSION
In this work, we propose a new S-LDPC algorithm based on Multi-rate LDPC code according to the error occurring characteristic of flash memory. In the early lifetime of flash memory, we use LDPC code with high code rate to correct errors. LDPC code with high code rate can meet the reliability requirement of flash memory and saves more decoding energy. At the same time, it also improves the read performance of flash memory. During the later lifetime of flash memory, with

the increase of error rate, high LDPC code rate is replaced
with low LDPC code rate. The latter has more powerful
error correcting performance. However, it also consumes more
decoding energy. Therefore, S-LDPC algorithm can get a
optimal tradeoff in terms of error correcting performance, read
performance and decoding energy consumption.
