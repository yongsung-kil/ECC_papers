Abstract- Low density parity check codes (LDPC) are appearing in an increasing number of applications which shows a performance close to the Shannon limit. The near Shannon limit error correction capability has lead LDPC codes to become the coding technique of choice in many communications and storage systems since their introduction. LDPC have good error correcting performance which enables efficient and reliable communication. Quasi-cyclic LDPC codes known as a subclass of LDPC codes are used whose parity check matrices consists of circulant permutation matrices. This character of QC-LDPC codes opens the door for the multi-mode. In the present work partially parallel architecture for single mode and multi-mode quasi-cyclic low density parity check (QC-LDPC) decoders have been designed using modified belief propagation algorithm. QC­ LDPC codes require less memory as compared to LDPC codes and resource occupied by the single mode decoder is only a little more than the multi-mode decoder. This decoder is modeled using Verilog software, synthesized and performed place and route for the design using Xilinx ISE 13.2.
Keywords- LDPC, Quasi-cyclic, Error correction, belief propagation.
I. INTRODUCTION
Low density parity check codes are a special type of error correcting codes known for their excellent error correction performance and high throughput. LDPC codes were first introduced by Robert Gallager and then rediscovered by MacKay in 90's. Since then LDPC codes are appearing in increasing number of applications such as magnetic disc recording, deep space communication etc. LDPC codes have turned out to be most suitable for implementation in modem CMOS technologies. A low density parity check code is a linear block code for which the parity check matrix has a low density of 1'so LDPC codes are designed starting from the parity check matrix. LDPC codes have simple graphical representation based on Tanner graph which consists of two set of nodes i.e check nodes and variable nodes that leads to accurate analysis of performance. The rows and columns of PCM can be transformed in to check nodes and variable nodes of tanner graph [1]. LDPC codes are of two types: Regular and

Irregular codes. In regular codes no. of '1's in each row or column is constant where as in case of irregular codes no. of '1's in each row or column is not constant. Irregular codes generally outperform regular LDPC codes for higher code lengths. Construction of LDPC codes can be done by random and structured method. In this work we have adopted structured irregular LDPC codes known as Quasi-cyclic Low density parity check codes. The block structured parity check matrices of QC-LDPC codes consists of arrays of circulants in which each row is the cyclic shift of the row above it, and the first row is the cyclic shift of the last row [1]. Because of this character of QC-LDPC codes a design method for multimode QC-LDPC decoder is proposed. LDPC codes are decoded using a subclass of message passing algorithms. The various message passing decoding algorithms are Sum Product Algorithm (SPA), Min Sum algorithm (MSA) and Bit Flipping algorithm. Sum Product Algorithm is also known as Belief Propagation (BP) Algorithm [3]. Efficient algorithms are used for row and column swapping so that complexity is reduced. In the present work LDPC decoding is done by BP algorithm. The main essence of BP algorithm is that the probabilities of bit nodes being '1' are exchanged between connected check nodes and variable nodes during each iteration cycle [2].

A. LDPC Decoding
LDPC codes are decoded using a subclass of message passing algorithms. The various message passing decoding algorithms are Sum Product Algorithm (SPA), Min Sum algorithm (MSA) and Bit Flipping algorithm. Sum Product Algorithm is also known as Belief Propagation (BP) Algorithm [3]. Efficient algorithms are used for row and column swapping so that complexity is reduced. In the present work LDPC decoding is done by BP algorithm. The main essence of BP algorithm is that the probabilities of bit nodes being '1' are exchanged between connected check nodes and variable nodes during each iteration cycle [2].
1) BeliefPropagation and modification Belief propagation algorithm has two phases. In the first phase, the variable nodes compute updated information which is sent to adjacent check nodes. In the second phase, the check nodes compute updated information based on the new messages from the variable nodes. This update information is then sent back to adjacent variable nodes and the process is repeated over and over again. The benefits of BP algorithm are high precision message passing and excellent error correction performance. "Fig. 3." shows the basic flow of LDPC decoding. After receiving the corrupted information from an AWGN channel A, the algorithm begins by processing it and then iteratively corrects the received data. First, all check node inputs are initialized to '0' and then a check node update step (i.e. row processing) is done to produce L (rj,) messages.

III. LDPC DECODER ARCHITECTURE
LDPC decoder architecture can be classified in to three categories: fully parallel, serial and partly parallel. LDPC decoder design has a tradeoff between hardware complexity and decoding speed. In fully parallel architecture direct mapping of tanner graph to hardware is done. All node operations are directly realized in hardware and connected through dedicated links. Its main disadvantage is huge connection complexity. Serial architecture consists of one CN, one VN and one memory. The VN are updated one at a

Single mode decoder comprises of two main blocks: VNU Block and CNU Block. The decoder also consists of 'Clock' and 'Reset' inputs. The maximum permissible number of iterations is determined by the value supplied at the ITER input. This can be set at a value in the range 0-11.
In order to get higher code length for code rate R=3/8 we require base matrix Hb of size 5x8 Then this matrix will be expanded to a bigger matrix by a factor known as expansion factor p=9. IV. SIMULATION RESULTS The decoder is described using hardware description language called Verilog and synthesized using Xilinx webpack 9.l ISE. The performance of whole model is check using Virtex7 family, XC7V2000T device and 2flg1925 package Table I. displays resources occupied by single mode and multimode QC-LDPC decoder which shows that hardware resources required by multimode decoder is little .
V. CONCLUSION Based on the quasi-cyclic character of QCLDPC codes, a design method for single mode as well as multimode QC­ LDPC decoder is used. The node memory is skillfully organized and can be expediently addressed by a simple control unit to save memory consumption. The results show that the resource occupied by the multi-mode decoder is only a little more than that of maximal resource occupied by the single mode decoder and decoder can support different code rates with the throughput higher than lOOMbps with iteration number fixed to 12 and the clock frequency is set to 370MHz.
