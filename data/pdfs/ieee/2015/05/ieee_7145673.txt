Abstract—On dispersing different base matrices combined with masking, different kinds of quasi-cyclic low-density paritycheck (QC-LDPC) codes are constructed, which may achieve good error performances. Two new algebraic constructions of regular QC-LDPC codes are presented in this paper. The two constructions are based on two base matrices by dispersing and masking. The two base matrices are based on the generators of the cyclic group of a prime field. Results show that the two classes of codes presented in this paper perform well with iterative decoding over AWGN channel and have advantages over MacKay codes in some aspects of code performances. We find that one of the constructed codes converges very fast with iterative decoding, which is a critical property in ultra-high throughput communication systems.
Keywords—Array dispersion; base matrix; circulant permutation matrix (CPM); LDPC codes; quasi-cyclic codes
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] are a class of capacity-approaching codes, which gradually become the most dominant codes in applications requiring error control coding. A LDPC code is uniquely determined by a sparse parity-check matrix H. The parity-check matrix H follows the following constraint in almost all constructions of LDPC codes: any two rows in H should have at most one place where they have nonzero elements in common, which is referred to as the rowcolumn (RC)-constraint [2]. The constraint guarantees that the associate Tanner graph [3] of the code given by the null space of H has no cycles of length 4 [4], which are the most harmful cycles in iterative decoding. The constraint also guarantees that the minimum distance of the code given by the null space of H, which has a minimum column weight of γmin, is at least γmin + 1 [5]. If H has constant column weight γ and row weight ρ, respectively, the null space of H gives a (γ, ρ)regular LDPC code [1]; otherwise, the null space of H gives an irregular LDPC code. If H is an array of circulants of the same size, the null space of H gives a QC-LDPC code [6]–[9].
Quasi-cyclic (QC)-LDPC codes have some advantages in hardware implementations. A QC-LDPC code can be encoded with cyclic shift registers, having a linear complexity in encoding implementation . A QC-LDPC code can also be decoded in a partially parallel architecture , which provides a great trade-off between implementation complexity and decoding throughput.

Masking [7] is a powerful technique in code construction, which can also be adopted to the following two constructions for new classes of LDPC codes. Example 1: In this example, we construct two relatively long codes to show their Shannon limit-approaching capacities. We choose p = 523 and take GF(523) as the code construction field. Based on (5) and (6), we know that there are 168 generators in the cyclic group of GF(523). Based on the generators, (7) and (8), a 169×169 RC-constrained array H5(12)3 of 523 × 523 CPMs over GF(2) is constructed. By taking 4 row blocks and 31 column blocks from H5(12)3, a 2092 × 16213 parity check matrix H5(12)3(4, 31) is obtained. The null space of H(512)3(4, 31) gives a regular binary (16213, 14124) QC-LDPC code with length 16213 and rate 0.8712. The error performance of the code decoded with 50 maximum iterations of the SPA is shown in Fig. 1. For comparison, the error performances of a (16213, 14124) QC-LDPC code and a (16383, 14253) MacKay code decoded with 50 maximum iterations of the SPA are also shown in Fig. 1. The QC-LDPC code used for comparison is constructed with the approach stated in 11.6 of [2]. We can see that the performance curves of the proposed QC-LDPC code and the MacKay code are very close to each other. The two curves intersect at the BER of about 5 × 10−7. The proposed QC-LDPC code has more advantages in lower region. We can also see that the WER curve of the MacKay code begins to change slope at the WER of about 10−4, while the WER curve of the QC-LDPC code drops rapidly without a visible error oor throughout the simulation region. The two QC-LDPC codes are both constructed on the prime field of GF(523). Compared to the old QC-LDPC code, the proposed one has a much better performance. We can conclude that the restriction on the base matrix does lead to a better long code.
By taking 4 row blocks and 61 column blocks from H(512)3, a 2092 × 31903 parity check matrix H5(12)3(4, 61) is obtained. The null space of H(512)3(4, 61) gives a regular binary (31903, 29814) QC-LDPC code with length 31903 and rate 0.9345. The error performance of the code decoded with 50 maximum iterations of the SPA is shown in Fig. 2. We can see that the performance curve is very close to the Shannon limit. At the BER of 10−6, the code performs 0.665 dB from the Shannon limit. The performance curve drops rapidly and shows no error floor throughout the simulation region.

We see that at the BER of 10−6, the code performs 2.05 dB from the Shannon limit, and the code performance curve drops rapidly from begin to end.
V. CONCLUSION
In the paper, we have constructed two classes of binary QC-LDPC codes based on prime fields. The constructions are achieved by dispersing base matrices into arrays of CPMs and masking. This approach results in sparse parity-check matrices, whose associated Tanner graphs have no cycles of length 4. Two new base matrices have been constructed based on the generators of the cyclic group of a prime field. Results show that codes constructed based on these methods perform well with iterative decoding over AWGN channel. The codes have more advantages over MacKay codes in low BER region and show no error floor in all the simulations. Most impressively, a code constructed in this paper converges very fast and has a great potential to be used in very high throughput communication systems.
