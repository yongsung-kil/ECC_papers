Stopping Set Elimination by Parity-Check Matrix Extension via Integer Linear Programming
Hossein Falsafain and Sayyed Rasoul Mousavi

Abstract—Error-rate floor phenomenon is known to be a serious impediment to the use of low-density parity-check (LDPC) codes for some practical applications that demand high data reliability. In the case of binary erasure channels (BECs), certain error-prone patterns, known as stopping sets, are proven to cause this performance degradation. A possible approach to diminish this drawback over BECs is to eliminate stopping sets by parity-check matrix extension. Given a parity-check matrix H, and a list L of its stopping sets, we present an integer linear programming (ILP) formulation to find a parity-check equation which eliminates the maximum number of stopping sets in L. One of the distinguishing advantages of the proposed scheme is its flexibility for modifications such as: limiting the weight of the new parity-check row, making the new row redundant or linearly independent, 4-cycle avoidance, and taking into account the sizes of stopping sets. Armed with these adjustments, the method can provide good performance improvements, as evidenced by simulation results. Furthermore, for a given ∈ N, by extending the basic formulation, we provide an ILP formulation for finding a set of size of parity-check equations which can best eliminate the stopping sets in L, among all such sets.
Index Terms—Binary erasure channel, error floor, integer linear programming, LDPC code, stopping set.
I. INTRODUCTION
I T IS a well-known fact that the performance of a low-density parity-check (LDPC) code under an iterative message passing decoding algorithm depends on the structural properties of the Tanner graph representing the code. Certain vertex configurations, known as error-prone substructures (EPSs), can lead the decoder to fail by converging to a non-codeword state, which can be the major cause of the troublesome phenomenon of error floor. For the case of binary erasure channels (BECs), the EPSs are called stopping sets and were introduced by Di et al. in [1]. These problematic configurations that lead to decoder failure also include trapping sets [2], near-codewords [3], pseudocodewords [4], instantons [5], and (fully) absorbing sets [6].

One successful recent approach being employed to alleviate the error floor problem in BECs is to remove the stopping sets (specially small ones) by extending the parity-check matrix by new rows, see, e.g., [7]–[12]. It has been demonstrated in [11] that the performance gap between maximum-likelihood and iterative decoding is in fact because of the stopping sets of size strictly less than the minimum distance. Parity-check matrix extension, by redundant [7], [10]–[12] and non-redundant [7]–[9] rows, is seen as a remedy to reduce the number of small-size stopping sets. When the new rows are redundant, this enhancement procedure is known as redundant extension [12]. We say that a parity-check row r eliminates (removes) a stopping set S, if |supp(r) ∩ S| = 1, where supp(r) denotes the support of r, i.e., the set of coordinates at which r is nonzero. Given a paritycheck matrix H ∈ Fm 2 ×n and a list L of its stopping sets, the problem of finding the smallest number of parity-check rows that eliminate all the stopping sets in L is shown to be N Phard [7]. A greedy approach to the problem is to find a paritycheck row that eliminates the highest number of stopping sets in L and to add it to H. Then, this step is repeated until all the stopping sets in L are eliminated. However, the problem of finding a parity-check row that eliminates the maximum number of stopping sets in L is also N P-hard [7].
Regardless of how cleverly the parity-check matrix H corresponding to a code is constructed, adding redundant rows to it cannot result in a new parity-check matrix with stopping distance greater than the minimum distance of the code [11]. However, it was shown in [11] that this upper bound can always be attained by adding a sufficient number of linearly dependant rows to H. In [10], Rosnes and Ytrehus presented a greedy algorithm dubbed GSSR (greedy stopping set removal) to find a small number of dependant parity-check rows to add to the original parity-check matrix to remove all stopping sets of size less than the minimum distance. In [7], Fainzilber et al. proposed a method for finding low-weight redundant rows that eliminate a reasonably large number of stopping sets in a given list of stopping sets of a parity-check matrix H, which brings into play a low-density generator matrix of the null-space of H. They also presented a greedy inexact algorithm for finding independent parity-check rows that do so. In [8], Jiao et al. presented a simple heuristic algorithm for eliminating small stopping sets in irregular LDPC codes, by introducing new independent check nodes into the original Tanner graph. To construct these new nodes, the algorithm makes use of the frequency of each variable node in a given list of stopping sets. Although ILP is itself N P-hard in general, it has been studied very extensively. There is a vast and growing arsenal of powerful heuristics and exact algorithms for the problem which has been improved radically in the last decades. Thus, the presented formulation allows for the use of various robust, industrialstrength ILP solvers for eliminating stopping sets. Besides, the major advantage of the new method is its amenability to adjustments, which include: limiting the weight of the new paritycheck matrix row (for preserving the sparsity of H), making the new row redundant or linearly independent with respect to the rows of H, 4-cycle avoidance, and taking into account the size of stopping sets (for example, based on the argument provided in [9]).
Another superiority of our approach over the previous ones is that, to the best of our knowledge, none of the previous methods, which are only heuristics, can yield the row that eliminates the maximum number of stopping sets in L, but they merely provide a row that hopefully eliminates a reasonably large number of stopping sets in L. On the other hand, in the case of the present method, both exact and heuristic methods can be applied for solving the ILP formulation to obtain, respectively, a “best” row (i.e., a row which can eliminate the highest number of stopping sets in L) or a reasonably good (but not necessarily a best) one. Finally, for a given ∈ N, by extending the basic model, we provide an ILP model for finding a size- set of parity-check equations which, among all such sets, eliminates the maximum number of the stopping sets in L. This formulation can be employed for finding the smallest set of parity-check constraints that eliminate a given set of stopping sets L. This is the first time that one can optimally tackle this problem.
The remainder of this paper is organized as follows. In Section II, after recalling some basic notations, definitions, and facts that will be used throughout the paper, we introduce our ILP-based stopping set elimination (SSE) method. Section III presents some possible adjustments to the basic formulation, which allow for more flexibility in the application of the model. In Section IV, we provide some comparative simulation results illustrating the effectiveness of the proposed SSE approach. Finally, Section V presents the conclusions.

A check node ci ∈ C is connected to a variable node vj ∈ V if the (i, j)-entry of H is equal to 1. A stopping set is a subset of V such that all of its neighbours are connected to it at least twice. The size of the smallest nonempty stopping set of H is referred to as its stopping distance.
Our stopping set elimination method works on a prespecified list L of stopping sets. The list L, which ordinarily contains small stopping sets of size less than some threshold, can be constructed, for example, using methods given in [10], [13], [14]. Our ILP-based approach returns a row that removes the highest number of stopping sets in L. The new row may or may not be in the dual (orthogonal) code C⊥ (i.e., the new row can or cannot be expressed as a linear combination of the rows of H). Adding a redundant row leaves the original code unchanged, and therefore there isn’t any rate penalty, but this is appropriate only when the stopping distance is strictly less than the minimum distance, which is a trivial upper bound on the stopping distance of a linear code (because the support set of any codeword is a stopping set). Adding a non-redundant row (i.e., linearly independent of the rows of H) changes the original code and causes a slight loss of code rate. Such a row, not only may improve the stopping distance, but also may improve the minimum distance of the code.

We conclude this section by a remark on the computational time required for solving the resulting ILP instances. Typically, the major factor which influences the running time of solving an ILP instance is its number of variables. In fact, ILP is N P-hard and using the algorithms available, the solution time grows, in the worst case, exponentially with the number of variables. However, off-the-shelf state-of-the-art ILP solvers (e.g., CPLEX [24]) can cope with instances involving several thousands of variables. Given a parity-check matrix H ∈ Fm 2 ×n and a list L of its stopping sets, the formulation (ILP1) needs 3|L| + m + n binary variables and n general integer variables. Moreover, for a given ∈ N, the formulation (ILP2) needs
(3|L| + m + n) + |L| binary variables and n general integer variables. Thus, the number of variables of a resulting ILP instance, and consequently the computational time required to solve it, depends on the dimensions of the parity-check matrix H, the size of the list L, and, in the case of the formulation (ILP2), the number of rows to be added to H. As a final remark in this section, it should be pointed out that, the formulation (ILP2) yields a best set of size of parity-check equations for eliminating the stopping sets in L, but the formulation (ILP1) yields a best single row which does so. This means that, while providing near-optimal results, employing (ILP1)
times successively may or may not give a best size- set of parity-check equations. However, when the size of the set L, the dimensions of the parity-check matrix H, or the number of rows

to be added to H are relatively large, successive application of (ILP1) is computationally more affordable than using (ILP2).
IV. SIMULATION RESULTS
The following examples demonstrate the strength and versatility of the proposed SSE method. The resulting ILP instances can be solved optimally using available solvers such as CPLEX and GUROBI [25]. It is worth emphasizing that, enriching the basic formulations (ILP1) and (ILP2) by the valid inequalities described in Remark 1 can notably strengthen their LP relaxations. Therefore, due to this tightening effect, all of the ILP models employed in the following examples have been enhanced by such cutting planes.
Due to the intractable nature of the ILP problem, for long LDPC codes, obtaining an optimal solution of the formulation (ILP1) or (ILP2) in reasonable computational time may be difficult. A common practice for dealing with this situation is to set a time-out threshold for the solver. If the solver terminates prematurely due to exceeding the time-limit, then the best incumbent (already found) solution can be considered as a good near-optimal solution of the problem. The quality of this solution depends on the problem size and the time-limit used. It should be noted that, for long LDPC codes, obtaining an exhaustive, or at least sufficiently comprehensive, list of smallsize stopping sets may also be very time-consuming [10], [13], [14]. This means that for such codes it may take a long time to obtain a list L of small-size stopping sets.

V. CONCLUSION

REFERENCES

In this article, we proposed a novel stopping set elimination technique which is based on the formulation of the problem as an integer linear program. The presented method can yield a parity-check that eliminates the highest number of stopping sets from a predetermined list of stopping sets, as opposed to the previous methods that just yield a reasonably good solution, without any guarantee of the optimality.
