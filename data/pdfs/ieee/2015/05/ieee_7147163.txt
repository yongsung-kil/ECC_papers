Abstract—The article summarizes a comparison of numerous irregular low-density parity-check codes using a set of metrics such as girth, number of short cycles in Tanner graph and extrinsic message degree (EMD). Based on this these metrics, we analyze how to decrease error floor for such codes with no negative effect to threshold.
Keywords—Irregular low density parity check (LDPC) codes, error floor, progressive-edge-growth (PEG) algorithm, stopping and trapping sets, extrinsic message degree (EMD), threshold.
I. INTRODUCTION
Every engineered communication system ranging from satellite communication to hard disk drivers to Ethernet must operate under noisy conditions. The key to reliable communication and storage is to add an appropriate amount of redundancy to make the system reliable. Error-correcting coding is one of the most effective ways to increase the safety of information systems to noise. Nowadays low-density parity-check codes (LDPC) [1] and turbo codes [2] have attracted considerable attention due to their capacity approaching performance. Moreover, irregular LDPC codes occupy the most advantageous position among iteratively decoded codes because of low-complexity decoding algorithms [3], which operate on a graphical representation of a code known as Tanner graph. However, practical performance of finite-length irregular LDPC is limited due to error-rate floor phenomenon associated with iterative decoders. This problem arises because of the sub optimality of BP like decoding algorithms on loopy graphs [3]. For this reason, to alleviate the effect of error floor, it is natural to consider three different approaches: find a Tanner graph on which a given decoding algorithm is least susceptible to failures; based on error floor analysis on a given Tanner graph, adjust a given decoding algorithm to achieve better error floor performance; create a serial concatenation of an outer high rate code with the LDPC code to lower the error floor. The first approach, when possible, is arguably more fundamental: once a good code is identified, its practical performance can be seamlessly improved with decoder-based solutions whereas

II. THEORETICAL ASPECTS OF LDPC CODES
As their name suggests, LDPC codes are block codes with parity check matrixes (H) with M rows and N columns that contain only a very small number of non-zero matrixes. The sparseness of H is essential for an iterative decoding complexity that increases only linearly with code length.
Aside from the requirement that H be sparse, an LDPC code itself is no different from any other block code. Generally, LDPC codes are designed by constructing a suitable sparse parity-check matrix first and then determining and encoder for the code afterwards.
Often LDPC parity-check matrixes are represented in graphical form by Tanner graph (Fig. 1), which consists of two sets of nodes: N nodes for the codeword bits (called variable nodes - VN), M nodes for the parity-check equations (called check nodes - CN) and E edges a bit to a check node if that bit is included in the corresponding parity-check equation. The code may alternatively be defined through the [K, N] generator matrix G: the code is defined as the set of all length N binaty vectors c which may be written as c = u·G for some binary (information) vector u of length K. The rate of the code (R) is equal to K/N and represents the fraction of the encoded bit stream occupied by information bits.
All LDPC codes may be separated into regular and irregular ensembles. An LDPC code parity-check matrix is called (ds, dc) – regular if each VN is contained in a fixed number ds of parity checks and each parity-check equation contains a fixed number dc of code bits. In an irregular LDPC

Characteristics of LDPC code in conjunction with the selected algorithm and the number of decoding iterations have a direct impact on the decoder’s energy gain. At the same time, the effectiveness of hardware and software implementation is based on the structure of the parity-check and generator matrixes.
First LDPC code construction algorithms were computerbased [3]. One of such methods proposes that the columns of H are added one column at a time from left to right. The weight of each column is chosen so as to obtain the correct VN degree distribution and the locations of the non-zero entries in each column are chosen randomly from those rows that are not yet full. If at any point there are rows with more positions unfilled than there are columns remaining to be added, the CN degree distributions for H will not be completely correct. It was shown [7] that randomly realized finite-length irregular LDPC codes with block sizes of order of 104 approach their density evolution threshold closely (within 0,8 dB) at rate 1/2 , outperforming their regular counterparts [3] by about 0,6 dB. In contrast, regular codes usually enjoy very low error floors. Regularity determines the potential effectiveness of the code ensemble while structure of individual parity-check matrix defines convenience of practical implementation and properties of this code.
III. PROPERTIES AND STRUCTURE OF IRREGULAR LPDC CODE
For any instance of the ensemble of the irregular LDPC code there are two general characteristics minimum code distance (d0) and girth of the Tanner graph. The girth influences the achievable bit error rate (BER) in the waterfall region because short cycles prevent the decoder from gleaning independent parity-check information. Therefore, the higher the girth, the faster the iteration-aided BER improvement. At the same time removing cycle larger than 6 from Tanner graph tend to concentrate the number cycles at the next larger size – which can actually decrease the code’s performance slightly. It is noted that whilst having a large d0 is always beneficial, the performance of the LDPC codes in the error floor region is largely dependent on the presence of stopping and trapping sets. We further notice that stopping sets characterize the performance of the code on erasure channels, whereas trapping sets play an analogous role for noisy channels, such as additive white Gaussian noise (AWGN).

Fig. 2. An example of irregular parity-check matrix for R=1/2 and code length N=1066.
The progressive edge-growth (PEG) algorithm [5] is probably the most famous and widely adopted algorithm to construct unstructured finite-length LDPC codes with large girth. The PEG algorithm constructs the Tanner graph in an edge-by-edge fashion. In the beginning, the graph is composed only of the VNs and CNs, where each VN is equipped with a number of sockets equal to its target degree and no edges are present. Then, VN sockets are processed in order and, for each of them, an edge is placed in the graph toward an appropriately chosen CN. The rationale behind the PEG algorithm is to tackle the problem of increasing the girth of Tanner graph by maximizing the local girth of a VN whenever a new edge is drawn from this VN toward the CN set.

IV. EMD METRIC OF SHORT CYCLES IN TANNER GRAPH
Not all short cycles are equally harmful. That’s why standard girth conditioning [5] severely constrains code structure by removing all cycles shorter than a specified length even through many of these can do little harm. Efficient way to suppress small stopping and trapping sets is to ensure cycles have enough neighbors that provide useful message flows. Consider the set of VN - Cd in a bipartite graph which has d elements, and one or more cycles are formed between this set and its neighboring constant set. Note that the maximum cycle length that is possible in Cd is 2d. An extrinsic constraint node of a VN set is a constraint node that is singly connected to this set. The extrinsic message degree (EMD) of a variable node set is the number of extrinsic constraint nodes of this variable node set. The EMD of a stopping set is zero. VI. CONCLUDING REMARKS
Several irregular LDPC code ensembles for coding rate R=1/2 were analyzed. We determined the evaluation of set of parameters, such as the girth of the Tanner graph, the number of cycles of length from g0 until g0+4 and there EMD sizes. We have made several assumptions about the relationship between graph structures that effect error floor and η metric sizes for different cycle lengths. Our results show an improvement in the error floor region of irregular LDPC code by several orders of magnitude in BER.
