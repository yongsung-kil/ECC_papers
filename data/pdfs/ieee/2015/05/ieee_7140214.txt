An Iterative Bit Flipping based Decoding Algorithm for LDPC Codes

Abstract—In this paper a new decoding algorithm for low density parity check (LDPC) codes based on bit flipping (BF) is proposed. In the proposed algorithm, the BF criterion benefits the data correlation between the decoded and received signal in current and previous stages. The proposed algorithm also degrades the decoding probability of one code word to another one in the same codebook. Simulation results confirm our theoretical formulation and show a good enhancement in code error rate and bit error rate of the proposed decoding algorithm compared to the conventional ones.
Keywords—bit flipping, LDPC, iterative decoding.
I. Introduction
LDPC codes achieve a good performance and have been the research interest for many researchers [1]. Decoding algorithms of these codes are grouped in three categories: hard decision, soft decision and hybrid methods.
Among the main decoding algorithms for LDPC codes, one step majority logic [2] and bit flipping use hard decision. Weighted bit flipping (WBF) algorithm [3] uses hybrid decision and belief propagation (BP) [4] is the best known decoding algorithm that utilizes soft decision.
Different methods are proposed to reduce the computational complexity in BP algorithm, however, hardware implementation is still a serious challenge. Therefore, considering both the computational complexity and performance, the best candidate in practical scenarios is WBF algorithm. A great variety of improved WBF algorithms are proposed in literature [5], [6], [7], [8], but none of them analyzes the theoretic bounds. Among all of the proposed methods, channelindependent weighted bit flipping (CIWBF) [9], implementation efficient reliability ratio based bit flipping (IERRWBF) [10] and gradient descent weighted bit flipping (GDWBF) [11] show the great performance in case of Bit Error Rate (BER) and therefore they are chosen to be compared with our proposed recursive weighted bit flipping algorithm (RECWBF). Our proposed RECWBF algorithm considers the correlation between the received and decoded vectors and uses the effective parameters in updating flip function.
Regular LDPC (N, K) dp, dc is a linear block code that maps a K bit message to an N bit code. dp is the weight of each row and dc is the number of bits with value 1 in each column. II. Proposed RECWBF Algorithm
The main idea of our proposed algorithm is to use effective values in BF functions. We propose a new perspective in iterative decoding algorithms. In each iteration we use the correlation between the received and decoded vectors. Our proposed algorithm uses the effective parameters in updating flip function. Existing algorithms only consider the syndrome of previous stage in flipping functions. The idea can be seen as an update process for coefficients in LMS and RLS algorithms.
In LMS algorithm, the error criterion is calculated in each update, therefore, the impulse like errors can destructively influence the convergence. In RLS algorithm, the sum of calculated errors is considered so that one step error may not influence the whole procedure and the result is better in term of convergence.
In our proposed algorithm we use vkjy j as a factor that leads to error. In each iteration we use the calculated vkjy j in previous stage, therefore, no sudden jump happens in the algorithm. The sudden jump is a model for the bits with high probability of flipping. This would improve the performance of decoding algorithm.
When the decoded vector is close to the real transmitted vector, the effect of vkjy j may degrade in our criterion. It is like the shortened steps in RLS algorithm while approaching the convergence. We also consider that using iteration we may gain a better codeword that is a fair outcome. Therefore, we add a forgetting factor to vkjy j to emphasis on the main weightening function.

A. Computational complexity
We choose λ and γ as large as possible as long as they satisfy these conditions: 0 < λ, γ < 1. This choice makes store process easier and increase the speed of algorithm convergence.
The computational complexity of our proposed algorithm is increased compared with those of earlier ones. It is also not necessary to store the vk in each stage. In our proposed RECWBF algorithm, only one addition and two multiplications are added compared to GDWBF algorithm. Comparing with WBF, two additions and three multiplications are added. This small amount of computational complexity leads to a great improvement in the performance of decoding process. In our proposed RECWBF algorithm, the final code word is gained in less number of iterations, therefore, the delay process decreases as another improvement.
III. Simulation Results
Performance evaluation is performed on two different codes. PEGReg 504 × 1008 and Gallager (191 273) [12]. For each code the maximum number of iterations is set to Kmax = 100 and each simulation is performed 10000 times. The forgetting factors are set to λ = 0.9, γ = 0.95.
The proposed method is compared with GDBF, IERRWBF, CIWBF and classic WBF. In Fig. 1 the BER versus SNR is plotted for PEGReg code. In SNR values less than 3.5 dB, 1.5 dB improvements are obvious. For SNR values more than 3.5 dB we have at least 2dB gain. It is shown that in SNR values more than 4.5dB, the performance improvement in comparison with WBF algorithm is more than 7dB.
Code error rate (CER) for PEGReg code is shown in Fig. 2. Improvements are as similar to Fig. 1. It is shown that the waterfall region for proposed algorithm (RECWBF) has better slope and starts earlier, compared with conventional methods.

IV. Conclusion
In this paper a new iterative WBF based decoding algorithm (RECWBF) for LDPC was proposed. In our proposed method, data correlation between the decoded and received signal in current and previous stage is considered. Therefore, we apply the flipping function of previous stage in the new stage of decoding, considering some forgetting factor. Simulation results confirm that the performance improvement is considerable. We also simplified the proposed algorithm so that the number of iterations and consequently the overall delay of decoding process decreases.
