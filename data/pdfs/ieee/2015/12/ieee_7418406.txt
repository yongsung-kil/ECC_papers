A 6.16Gb/s 4.7pJ/bit/iteration LDPC decoder for
IEEE 802.11ad standard in 40nm LP-CMOS
Hiroyuki Motozuka, Naoya Yosoku, Takenori Sakamoto, Takayuki Tsukizawa, Naganori Shirakata and Koji Takinami
Panasonic Corporation Yokohama, Japan
motozuka.hiroyuki@ jp.panasonic.com

Abstract— This paper presents an LDPC decoder employing a column-parallel architecture that enables lowpower and high-speed operation suitable for the 802.11ad standard. As compared to the conventional row-parallel architecture, the proposed architecture reduces the required memory size by 60% and also minimizes the number of pipeline stages for high throughput operation. Fabricated in 40nm LP CMOS technology, the prototype achieves high energy efficiency of 4.7pJ/bit/iteration for 6.16Gb/s while supporting all the modulation and coding schemes (MCS0 to MCS12) required for the 802.11ad single-carrier (SC) modulation.
Keywords— LDPC decoder; parallel algorithms; pipelining; multi-stage variable shifters
I. INTRODUCTION
The IEEE 802.11ad standard has been developed for multi-Gigabit 60GHz wireless solutions. The standard offers a much higher throughput than existing 2.4GHz/5GHz wireless LAN and is expected to be supported by future mobile terminals [1]. However, this requires aggressive power reduction not only for analog RF but also for baseband signal processing in which a low-density paritycheck (LDPC) decoder is one of the most power-hungry building blocks. Nowadays, a row-parallel architecture has been widely used for LDPC decoders for achieving high throughput operation required for the 802.11ad [2-4]. However, it suffers from significant power consumption due to the need for large memory and pipeline registers. The prior work [2] employs a non-refresh embedded DRAM (eDRAM) to save memory power, but it requires additional cost as well as higher voltage supply and dedicated physical design for each process node. One paper [3] proposes voltage scaling using fully-depleted silicon on insulator (FDSOI) technology. However, to be practical in low cost consumer mobile applications, it is highly desirable to adopt a standard digital low-power (LP) CMOS process, but this process suffers from limited operation speed and thus requires further power reduction techniques including architectural innovation.
This paper presents an LDPC decoder employing a column-parallel architecture that enables low-power and high-speed operation suitable for the 802.11ad standard. To

II. PARALLEL DECODER ARCHITECTURES
In the 802.11ad standard, four parity-check matrices are defined for different coding rates. Each matrix is constructed based on quasi-cyclic LDPC (QC-LDPC) code, which consists of Z x Z submatrices. Each submatrix is a cyclic shift of an identity matrix or a zero matrix where Z=42. The parity check matrix size is mZ x nZ, where m and n are the number of block columns and block rows, and the number of columns (m) is 16 for all the code rates and the numbers of rows (n) for the four code rates are 3, 4, 6 and 8 respectively. To decode LDPC codes, the min-sum algorithm is widely used for VLSI implementation [2-6]. In the algorithm, variable node (VN) processing and check node (CN) processing are performed iteratively. For the parity check matrix size of mZ x nZ, mZ VN processing and nZ CN processing are performed in each iteration. Each of the VN processing has inherent parallelism and mZ VNs can be computed concurrently. Similarly, nZ CNs can be computed concurrently using inherent parallelism. To achieve high throughput with relatively low circuit complexity, partially parallel LDPC decoders have been proposed [2-5]. In the partially parallel decoder, the parity check matrix is divided into chunks and messages in one chunk are processed concurrently and multiple chunks are processed by pipelined VN and CN processors. Each chunk consists of one or more block rows in row-parallel architecture, while each chunk consists of one or more block columns in column-parallel architecture. The conventional reconfigurable LDPC decoders for the 802.11ad LDPC codes employ row-parallel architecture, used mZ VN processors and Z CN processors [2,3]. To realize reconfigurable decoding for four codes in the 802.11ad, the decoder in [2] uses reconfigurable CN circuits which are switchable between 8-input and 16-input

III. ARCHITECTURE
Fig. 1 shows the overall architecture of the proposed LDPC decoder. It consists of three decoder cores, each of which is operating at 220MHz to achieve 6.16Gb/s total throughput. Each decoder core employs a column-parallel architecture for reducing power consumption in memory which dominates the total power of the decoder. In the proposed column-parallel architecture the check nodes (CN) are fully parallelized with 8Z CN processors, while the variable nodes (VN) are partially parallelized with 2Z processors. The CN and VN processors are connected by interconnect circuits, which consist of variable to check (V2C) shifters, check to variable (C2V) shifters and routers. For power saving, fine-grained and coarse-grained clock gating methods are implemented. The coarse gating is based on an early termination scheme, which turns off the core if the parity check result indicates no errors before reaching maximum iterations. This scheme saves the power consumption of LDPC decoder up to 55%. Besides this, the fine gating inactivates some of the pipeline registers depending on the LDPC matrix structure. In stage 1 and 3, the clocks for pipeline registers are only activated in accordance with the number of the non-zero submatrices in processed chunk. IV. COLUMN-PARALLEL ARCHITECTURE FOR 802.11AD
Fig. 2 shows the proposed column-parallel algorithm. It segments the parity-check matrix into eight chunks. The chunk size depends on the code rate (ex. 8Z x 2Z for R=1/2, 4Z x 2Z for R=3/4, where R denotes code rate) and the positions of non-zero submatrices are different among chunks. In the proposed architecture, each chunk is split into left and right halves, and map each layer data to variable-tocheck (V2C) shifters as illustrated in the right of the Fig. 2. The layer is the set of one or multiple row blocks in which the column weight is one at most. The mapping to check-tovariable (C2V) shifters is determined in the same way. In this way, it can support all the LDPC matrices specified in the 802.11ad by using only eight shifters for each chunk. With proposed mapping, routers are also simplified. C2R router in stage 2 consists of 1-to-2 de-multiplexors since one layer is mapped to two rows. For example, V2C1 (for layer 3) is connected to CN1 (for row index 7) and CN3 (for row index 5). Similarly, the R2C router in stage 3 consists of 2-to-1 multiplexors. Thus, the critical paths of routers are negligibly short in our architecture.
Fig. 3 shows VN and CN architectures. They are based on min-sum algorithm. The 2-input CN has shorter critical path than the 16-input CN, which is used in row-parallel architecture. This also enables high speed decoding.
Fig. 4 compares data path structures for the proposed column-parallel architecture and the conventional rowparallel one. In the proposed architecture, the posterior and V2C messages require only sign bits while the row-parallel one requires soft bits to perform recursive VN calculation. The C2V memory size is also reduced since it holds the minimum and second minimum values of C2V messages instead of all C2V messages. Memory requirements are summarized in TABLE I. The proposed architecture saves 60% of the total memory as compared to the conventional one [2] without reduced marginalization. The memory size reduction has a significant impact on minimizing the access bandwidth thus reducing power consumption.
Fig. 5 shows timing chart of the proposed decoder. It takes ten clock cycles per iteration for ordinary codes. The number of cycles is independent of code rates. In the proposed architecture, the shortened code which is used in PHY header and for MCS0 and MCS1 in the 802.11ad can be processed by simply reducing the number of chunks. This reduces the decoding delay of a PHY header by half and thus reduces the packet processing delay at the receiver.

V. REDUCED-COMPLEXITY VARIABLE SHIFTERS
The variable cyclic shifters used in the V2C and C2V interconnections are also critical for high-speed and reconfigurable decoding. The conventional decoders use Zinput barrel shifters, each of which has log2Z 2-to-1 multiplexers on its critical path [6]. The conventional barrel shifter is delay critical and it requires one pipeline stage even for only shifters and routers. In the proposed decoder, the V2C and C2V shifters are custom designed for the 802.11ad LDPC codes. Fig. 6 compares the conventional and the proposed multi-stage variable shifter. As an example, it shows one of the eight V2C shifters (V2C1), which performs cyclic shift of the V2C message on the layer 3 of the matrix in Fig. 2. To support four different code rates, the V2C1 needs to handle 12 cases illustrated in the lower right of the Fig. 6. These shift values are neatly decomposed into the combinations of two numbers of modulo-42 additions. By cascading two MUX-based cyclic shifters as shown in the upper right of figure, the required 12 cases are realized. An exhaustive search has been made to find the best decomposition such that the number of fixed cyclic shifters is minimized. The proposed variable shifter saves 42% circuit size. The critical path delay is reduced by 33%, where we assume the delay of N-input multiplexor is proportional to ceiling(log2 N).
VI. EXPERIMENTAL RESULTS
A prototype has been fabricated in a 40nm LP CMOS process. It incorporates the 802.11ad physical layer (PHY) including the proposed LDPC decoder, the equalizer, the ADC/DAC, as well as MAC, USB interfaces, 2.4GHz/5GHz WLAN PHY, etc. Fig. 7 shows the measured BER performance and the time domain behavior of the total power consumption. To achieve the required BER performance, the log-likelihood ratios (LLRs) are quantized to five bits and the maximum number of iterations is set to 7. VII. CONCLUSIONS
This paper presented a low-power LDPC decoder suitable for the 802.11ad standard. By employing the column parallel architecture with novel data partitioning scheme, the proposed architecture reduces the required memory size by 60%. The proposed multi-stage variable shifters suitable for decoding of QC-LDPC can minimize the number of pipeline stages for high throughput operation. Fabricated in 40nm LP CMOS technology, the prototype achieves high energy efficiency of 4.7pJ/bit/iteration for 6.16Gb/s.
