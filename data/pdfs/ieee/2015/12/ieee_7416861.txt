Improved Controlled Start-up Stochastic LDPC Decoder for Efficient FPGA-based Implementation
Ghania Zerari and Abderrezak Guessoum
LATSI Laboratory, Department of Electronics University of Blida Blida, Algeria
ghania.zerari@gmail.com

Abstract—This paper introduces a new and an improved controlled start-up stochastic (CSS) architecture of Low-Density Parity-Check (LDPC) decoding, to implement fully parallel FPGA-based decoders. The developed architecture uses a new variable nodes structure with larger internal memory lengths, to improve the convergence, without significant additional field programmable gate array (FPGA) resource. To validate the advantage of the proposed approach, a medium (1024, 512) and short (200, 100) codes are implemented.
The results of Xilinx Virtex-6 VLX240T FPGA shoes that the variable node internal memories lengths can be increased from 2bit, used in CSS and Delayed Stochastic (DS) decoders, to 64-bit without addition resource.
Keywords—improved controlled start-up stochastic decoding; stochastic decoding; low-density parity-check code; field programmable gate array.
I. INTRODUCTION
In recent years, the need for increasing the communication systems capacity, for optical and wireless networks, has attracted the employment of Low-Density Parity-Check (LDPC) decoding. The LDPC codes, initially presented by Gallagher in 1962 [1], are used for repairing transmission errors, introduced by the channel effects. These codes are a powerful linear error-correcting codes, that provided a close Shannon capacity decoding performance [2]. These advantages justify their adoption by different digital communication standards such as IEEE 802.16e (WiMAX) [3], the IEEE 802.3an (10GBASE-T) [4], the IEEE 802.3ba [5], the DVB-S2 [6] and the IEEE 802.11 (WiFi) [7] .
Various implementations for LDPC decoders have been investigated to reach high throughput results. It has been clearly shown that the fully parallel decoding solutions performed the advanced throughput, nevertheless they increased the hardware complexity. In order to overcome this disadvantage the LDPC iterative decoding using stochastic technique was initially proposed by Gaudet and Rapley [8]. The current stochastic decoding algorithm confirmed their adaptability for fully parallel decoding approach [9]-[14]. To

get around the Latch-up (locking) situations, the edge memories (EMs) method [9] was introduced. Concerning the important silicon area demand of ASIC EMs-based stochastic decoders, tracking-forecast memory (TFM) and majority-based tracking forecast memory (MTFM) are proposed [11],[13]. Furthermore, a 2-bit internal-memories-based stochastic decoders called Delayed Stochastic (DS) decoder and Controlled Start-up Stochastic (CSS) decoder are presented in [14] and [15] respectively.
This paper introduces a new and powerful stochastic architecture, to improve the convergence in addition to decrease the latency of the decoding process. The proposed architecture improves the Controlled Start-up Stochastic variant using a new variable nodes structure with larger internal memory lengths, to improve the convergence, without significant additional field programmable gate array (FPGA) resource. To validate the advantage of the proposed approach, a medium (1024, 512) and short (200, 100) codes are implemented on Xilinx Virtex-6 VLX240T field programmable gate array (FPGA) device, from 2-bit to 64-bit internal memories lengths. The paper is organized as follows. In Section II, we provide an overview of the LDPC stochastic decoding. In section III, the improved architecture of the controlled start-up stochastic LDPC decoding is introduced. We demonstrate that variable node internal memories lengths can be increased from 2-bit, used in CSS and Delayed Stochastic (DS) decoders, to 64-bit without addition resource. Results of some FPGA implementations are presented in section IV, and finally a conclusion is given in section V.
II. STOCHASTIC LDPC DECODERS
Conventional LDPC fully parallel decoder uses fixed-point operands to represent the probabilities, exchanged between the Variable Nodes (VNs) and the Check Nodes (CNs) of the factor graph. To encode k information bits, an (n, k) LDPC code uses n encoded bits, where n > k. LDPC decoder can be represented by a factor graph which uses n VNs and (n-k) CNs. A dv degree VN has (dv+1) ports, one of which get the channel probability and the other dv are connected to different CNs, by bidirectional ports.

III. NEW CONTROLLED START-UP STOCHASTIC DECODER

We focus mainly on CSS variant for efficient FPGA implementation. The CSS LDPC decoder achieves decisions by reaching the correct codes or the maximum number of iterations. The converted Bernoulli sequences are used as a variable node input. All CSS decoding variable nodes are initialized by one bit coded probability channel. During the disagreement state, the proposed architecture generate a new bit based on a random bit selection from the VN internal memory (IM). The IM length is increased up to 64 bits.
Based on (6) and (7), the hardware implementation of the new improved decoding approach does not require extra hardware complexity, for FPGA devices. Moreover, the projected technique computes the received probability without any additional decoding cycle.

Fig. 3 represents the main structure of the improved controlled start-up stochastic variable node. Similar to the DS and CSS design, the Decoding Cycle (DC) matches to one iteration for the proposed LDPC stochastic decoding.
The Pch(k-1) signal is transmitted to the VN output DFF during the first process cycle. After the first iteration and until the last one, the multiplexer sends the variable node processor output bit to the VN output DFF. In this way and identically to the CSS process, the majority of variable nodes outputs start with a right bit and detour the random stochastic initialization.
IV. IMPLEMENTATION AND RESULTS
It has been proved that the increasing of the VN internal memories lengths increases the LDPC stochastic decoding converges [9]-[10]. However, adding additional memory capacity set in, mainly, an extra hardware complexity and resources.
The main target of the proposed improvement is to enlarge the FPGA-based CSS LDPC VN memory, without supplementary FPGA resources. To confirm the improvement of the new design, a medium (1024, 512) and short (200, 100) LDPC codes are implemented on Xilinx Virtex-6 VLX240T field programmable gate array (FPGA) device, with 2-bit to 64-bit.
Fig. 4 shows the implementation of the new degree-3 Variable Sub-node of the proposed (1024, 512) and (200, 100) LDPC codes. The degree-3 Variable Node is composed by 3 degree-3 sub-node. The majority state and the random address signals are connected to all sub-nodes. One of the 3 degree-3 sub-node input is connected to probability signal by a comparator. The other two sub-node input are connected to the check node output. The three S signals are combined to produce the majority state signal. The FPGA implementation of VN Internal Memory is achieved by using the SLICEs FPGA distributed RAM. The implemented CNs use the same structure adopted by the DS and the CSS decoders.

V. CONCLUSION
In this work, we presented a new fully parallel stochastic LDPC decoding approach as an improved version of LDPC CSS version, which can exceed the CSS and the Delayed Stochastic (DS) performances. The amelioration is achieved by enlarging the variable node internal memory size. A Virtex-6 Xilinx FPGA implementation proves the advantages of the proposed method. We showed that the implementation of a medium (1024, 512) and short (200, 100) LDPC regular codes, with 64-bit VN internal memory, do not request extra FPGA resources compared to 2-bit based VN.
