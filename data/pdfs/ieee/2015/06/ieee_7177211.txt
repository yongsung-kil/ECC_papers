High Throughput LDPC Code and Decoder Design for HINOC 2.0 Systems

Yijin Zhao1, Yin Xu1, Kang Zhao1, Dazhi He1, Wenjun Zhang1, Hongbin Li2, Jingfei Cui3

1Shanghai Jiao Tong University, China 2Beijing University, China

AbstractвҖ”This paper presents a high-throughput structured low density parity check (LDPC) code and an optimal parallel decoder architecture with advanced Ping-Pong RAMs for the HINOC 2.0 systems. An additional iteration scheme is proposed to further improve decoding performance by taking full use of time intervals of the discontinuous streams, an important feature in HINOC 2.0 systems. The proposed LDPC code proves to be of better performance than the codes adopted in the current standards (802.16e). The proposed decoder is implemented and synthesized on FPGA Stratix V, with the maximum working frequency of 145MHz. The total resource cost is reduced by 23%, which is the cost of one decoder core, with slightly increased complexity in control logics, compared with the traditional PingPong RAM based parallel architecture. The resulting performance is 2Gbps which is enough to support HINOC 2.0 systems.
KeywordsвҖ”Low density parity check (LDPC) codes, Channel coding, Future technologies and services of broadcasting, High performance network over coax (HINOC), Parallel high throughput LDPC decoder
I. INTRODUCTION
Triple-Play services, developed by Next Generation Broadcasting (NGB), have recently received much attention in China. It aims to provide fast Internet access, high-definition television (HDTV) and telephone service in one physical network. With the development of optical fiber communication, the bandwidth in the backbone network is large enough to support high-speed multi-media services. Existing technology like Fiber to the home (FTTH) [1], has already installed optical fibers to provide unprecedented high-speed Internet access in some big cities of China. However, implementing FTTH still remains some problems. For example, FTTH on a large scale will be costly in installation and maintenance. In order to solve these problems, the technology of high performance network over coax (HINOC) was proposed in China. It aims to provide multiple services by taking good use of the already existing coaxial cable network, which is widely deployed among the vast population in China, thus making HINOC a costperformance amiable and promising solution in the future access network.
HINOC 1.0 standard [2] was published as the industry standard in China in August 2012, and HINOC 2.0 standardization is an evolving version which is currently ongoing. HINOC 2.0 supports wider bandwidth, higher data rate and greater configurability than its predecessor HINOC 1.0. In

order to fulfill the physical layer demand, advanced forward error correction (FEC) coding like low density parity check (LDPC) code[3][4] is aggressively adopted. And due to its higher quality-of-service (QoS) requirements, the LDPC code should possess special quality as follows, lower error floor, better error correction performance; and on the other hand, higher demand for hardware implementation: such as higher throughput, processing speed, lower latency should be satisfied. The corresponding LDPC requirements are shown in Table 1 which is discussed extensively among and provided by the technique committee of HINOC 2.0.

There are many existing communication standards that adopt certain LDPC codes and parallel decoders to meet the demands for various applications. For example, the digital television broadcast standard DVB-S2[5] and DVB-T2[6] adopt some long-length LDPC codes with good performance, IEEE802.11ac[7] and IEEE 802.16e[8] have proposed some short-length LDPC codes. And In the work of [9], the authors have provided a parallel LDPC decoder for structured LDPC codes which provides a solution to trade off the complexity and throughput. However, none of these standards are meant to support up to 1Gpbs high-throughput data rates, neither for these LDPC codes and decoder structures.
In this work, based on structured irregular repeataccumulate (S-IRA) LDPC codes, we first design a structured LDPC code with required code length, code rate and a suitable paralleled factor for high throughput realization, and the proposed LDPC code proves to be of better performance than the LDPC codes adopted in the current standards (802.16e). Moreover, to fulfill the 1Gpbs throughput, we propose a high throughput LDPC decoder which is designed for the proposed LDPC code. Finally, to take full use of the time interval of discontinuous streams which is an important feature in HINOC 2.0 system, an вҖңadditional iteration schemeвҖқ is proposed to improve the decoding performance without reducing the throughput.

II. PROPOSED LDPC CODE
As required by HINOC 2.0 standard, the candidate LDPC codes should have code rate 9/10 and code length 1920, the high code rate is mainly to make the system efficient in data transmission, and the relatively short code length is mainly due to the factor that HINOC 2.0 has a two way communication system, among which the content of uplink may consist of VOD Demand signaling or other short message based signaling, thus if the length of LDPC code is too long, it will be costly and inefficient.

With these two requirements in mind, basically we choose to design a LDPC code based on the Irregular Repeat Accumulate structure (S-IRA), which is efficient in fast encoding and decoding.

Considering that the likelihood-ratios (LLRs) of a whole LDPC block is necessary during the decoding period, RAMs are adopted to store the input LLRs, so that LLRs can be read out and delivered to corresponding decoder cores. Ping-Pong RAM is a common scheme applied in LDPC decoder architecture. Traditionally, Ping-Pong RAM contains two separate RAMs, in which LLRs streams are stored in alternative manner, when one of the two RAMs is full of LLRs, it should be read out in parallel to the decoder core. For further improvement, we propose an advanced PingPong RAM based parallel architecture, as shown in Fig.1(c), which consists of three key components: decoder cores, Ping-

Pong RAMs and extrinsic wrapper. Decoder core is a single logic unit used to decode one block codes via a number of iterations, each of which includes row operation module and column operation module, which are performed by turns and finally helps the decoder core converge to decoding results. The Ping-Pong RAMs store input LLRs, which will be read out and delivered to corresponding decoder cores after arbitrary RAM is full. The extrinsic wrapper controls read and write mechanism of Ping-Pong RAMs and master the logical process and timing of parallel decoder cores.
Its working mechanism is as follows: firstly, the coming input LLRs are written into advanced Ping-Pong RAMs according to address increasing order. After Ping-Pong RAM 1 is full, the data is immediately read out and delivered to decoder core 1, and at the same time, Ping-Pong RAM 2 keeps being written with newly coming LLRs. After Ping-Pong RAM 2 is full, its data is read out and delivered to decoder core 2. It continues like this, and when Ping-Pong RAM 5 is full, its data is sent to decoder core 1, at this time, decoder core 1 has just finished decoding of previous block. After that, Ping-Pong RAM 1 is written over by newly coming LLRs. As a result, when Ping-Pong RAM R is full for the x times, it will deliver its data to corresponding decoder core D = (R+x+2)%4+1, and thus the 4 decoder cores can work in parallel without data hazard.
Compared with the traditional Ping-Pong RAM based architecture as shown in Fig. 1(a), our proposal is able to prolong the decoding time period for each decoder with acceptable resource cost, which enables reasonable decoding iteration meanwhile fulfill the 1Gpbs high throughput. And also, compared with traditional Pong-Pong RAM based parallel architecture as shown in Fig. 1(b), our proposal can save resource cost of one LDPC decoder core, while slightly increase the cost in regard to logic control.
Moreover, we propose an additional iteration scheme to further enhance the LDPC decoding performance, as shown in Fig.2.
In HINOC 2.0 standard (different from the traditional oneway broadcasting system), which is a two-way communication system, the input LLRs are not necessarily delivered continuously. Regarding this feature, we propose an additional iteration scheme to allow decoder cores to take full use of time intervals of the discontinuous LLRs, so that additional iteration can be done during the intervals to further improve the LDPC decoding performance.

V. CONCLUSION
This paper proposes a high-throughput LDPC code and decoder design for HINOC 2.0 systems. The designed structured LDPC code proves to be of better performance than the LDPC codes adopted in the current standards (802.16e). The designed decoder increases throughput by the proposed advanced Ping-Pong RAM based parallel architecture. Also, an вҖңadditional iteration schemeвҖқ is applied to increase the iteration by taking full use of the time interval of discontinuous streams, further improving decoding performance without reducing the throughput. The proposed approach could be also applied to a more general setting like large networks [10][11]. This work is implemented and synthesized on FPGA Stratix V, with the maximum working frequency of 145MHz. Therefore, a high performance of 2 Gbps is achieved, which makes this design applicable to HINOC 2.0 systems.
