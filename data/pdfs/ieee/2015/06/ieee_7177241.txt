Lowering Error Floors by Removing Dominant Trapping Sets of Low-Density Parity-Check Codes for Broadcasting Systems
Kyung-Joong Kim, Seho Myung, and Hongsil Jeong

AbstractвАФIn this paper, we propose a method for avoiding high error floors of low-density parity-check (LDPC) codes for broadcasting systems. Harmful trapping sets which may introduce high error floors are investigated and removed by using the proposed method. Applying this method, the error floor of a DVB-S2 LDPC code is dramatically lowered by modifying only two numbers in its sequence representation.
Key WordsвАФChannel coding, Next generation of broadcasting systems.
I. INTRODUCTION
LOW-DESITY parity-check (LDPC) codes, first introduced by Gallager in 1962 [1], achieve the performance near to the Shannon limit by iterative decoding [2]. They are extensively employed in communication and broadcasting system standards such as DVB-S2 [3], DVB-T2 [4], IEEE 802.16e [5], and IEEE 802.11n [6] because of their capacity-approaching performance and low decoding complexity.
Sometimes, one can observe error floor phenomenon in the performance curve of an LDPC code which means that the slope of performance curve is dramatically decreased as signal-to-noise ratio (SNR) increased. Hence, large amount of signal power is required for successful transmission where an error floor is observed at relatively high error rate region. Low error floor is essential for the system with low target error rate such as next generation of broadcasting systems.
One among the LDPC codes adopted in the DVB-S2 system standard (in short, DVB-S2 LDPC codes) has severely high error floor. In this paper, we investigate the error floor of the DVB-S2 LDPC code and propose a method for lowering the error floor. Only two numbers in the sequence representation which defines the DVB-S2 LDPC code are modified by the proposed method for lowering the error floor. The proposed method is not restricted to a specific LDPC code but can be applied to any LDPC codes which have the sequence representation such as DVB-S2 LDPC codes.
The remainder of this paper is organized as follows. Section

II introduces dominant trapping sets, and in Section III, a method for lowering error floors is proposed by removing dominant trapping sets in DVB-S2 LDPC codes. Finally, conclusions are drawn in Section IV.
II. FINDING DOMINANT TRAPPING SETS
An LDPC code is defined by a bipartite graph consists of variable/check nodes and corresponding edges. For the convenience of notation we call a trapping set in a bipartite graph as a tapping set of the corresponding LDPC code. An (a, b) trapping set can be defined as a set of a variable nodes which have b check nodes of odd degree in their induced subgraph. The most harmful trapping sets are called dominant trapping sets. It is well known that trapping sets with relatively small values of a and b are more harmful in error floor region than the others [7]. Generally, dominant trapping sets are composed by one or several cycles which consist of variable nodes of low degree and check nodes connected to the variable nodes. Some examples for (a, 0) and (a, 1) trapping sets are drawn in Fig. 1. In this figure, a circle represents a variable nodes and a square with cross represents a check nodes, respectively.

B. Trapping Sets with Two Variable Nodes of Degree 3
In this subsection, a method to find trapping set structures shown in Fig. 1(c) is briefly described. Let v1 and v2 be a variable node of degree 3 connected to a set of edges {e1, e2, e3} and {e4, e5, e6}, respectively. Then, for a given variable node v1, we trace and record the paths of length up to 2a-4 starting from e1, e2, and e3, respectively. For simplicity, any path passing a variable node of degree 3 may be discarded. We can find a trapping set which have a structure shown in Fig. 1(c) if there exist a variable node v2 such that satisfying the following conditions:

1) There is a path P1 from e1 to e4; 2) There is a path P2 from e2 to e5; 3) There is a path P3 from e3 to e6; 4) Total number of variable nodes contained in paths P1, P2, and P3 is a-2.
C. Trapping Sets with Three Variable Nodes of Degree 3
Trapping sets which have structures shown in Fig. 1(d) can be detected using a similar way described in the previous subsection. The only difference is that one variable node of degree 3 in paths P1, P2, or P3 is allowed when it is not allowed for the structure shown in Fig. 1(c). That is, any path passing more than one variable node of degree 3 may be discarded for simplicity. We can find a trapping set which have a structure shown in Fig. 1(d) if there exist a variable node v2 such that satisfying the following conditions:
1) There is a path P1 from e1 to e4; 2) There is a path P2 from e2 to e5; 3) There is a path P3 from e3 to e6; 4) Total number of variable nodes contained in paths P1, P2, and P3 is a-2; 5) Total number of variable nodes of degree 3 contained in paths P1, P2, and P3 is 1.

Block index j is corresponding to the j-th sequence in the sequence representation of the DVB-S2 LDPC codes [3]. Therefore, the (9, 0) trapping sets may be removed by modifying a number selected among 6 numbers in the 23-th or 68-th sequence in the sequence representation. IV. CONCLUSION In this paper, we proposed a method for lowering error floors by removing dominant trapping sets from the DVB-S2 LDPC code of length 64800 and rate 2/3. It can be applied to any LDPC codes which have high error floors. As a design tool of channel codes for the next generation of broadcasting systems, it can be a good candidate since relatively low target error rates are required in those systems.
