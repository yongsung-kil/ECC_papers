Abstract—In this paper, we apply an efficient scaling strategy on layered scaled min-sum LDPC decoder. In addition, we propose a joint optimization strategy for the quantization and scaling parameters of layered scaled min-sum LDPC decoder. The study of our optimization results, for DVB-S2 LDPC codes with different constellation sizes and code rates, shows that each constellation size code rate pair has different optimal scaling and quantization parameters. In order to maximize the achievable performance, we propose using the different scaling and quantization parameters for each constellation size code rate pair. The simulation results show the performance improvement of separately using optimal scaling parameters or optimal quantization parameters, and the overall performance enhancement of using both optimal scaling and quantization parameters.
I. INTRODUCTION
Low Density Parity Check (LDPC) codes were introduced by Gallager [1] as linear block codes, which use sparse parity check matrix. The near Shannon capacity performance of LDPC codes is reported in [2]. Layered implementation [3] of structured codes [4] decreases encoding and decoding complexity of LDPC codes. Structured codes are used in modern standards such as DVB-S2 [5].
Practical hardware implementation of LDPC decoders requires applying quantization, which has a server effect on the LDPC decoders performance [6]–[9]. In [6], Discrete Density Evolution (DDE) is used in studying quantization effect on the performance of sum-product belief propagation algorithm. In [7], DDE is used to optimize different quantization steps for variable node and check node output messages of sumproduct decoding algorithm. The work in [8] used the distribution of the input Log-Likelihood Ratio (LLR) to design the uniform quantization step of scaled min-sum LDPC decoder. Non-uniform quantization is used with LDPC decoders to achieve lower error floor [9], but with higher complexity. Most quantization design techniques have been mainly concerned with BPSK modulation [6]–[9]. However, the distribution of QAM de-mapper output is different from BPSK de-modulator output. To deal with this problem, optimization of the uniform quantization step for each used constellation size is necessary.
In our previous work [10], we proposed a fast converging algorithm named, Generalized Simplified Variable Scaled (GSVS) min-sum decoding. GSVS-min-sum is a modified version of our previously developed SVS-min-sum decoding [11], where in the GSVS-min-sum the scaling factor increases exponentially with iterations, starting by an initial scaling

factor α0 and ends at unity. As opposed to starting the scaling factor sequence by 0.5 in the SVS-min-sum [11]. In [10], we applied the GSVS-min-sum on flooding implementation of LDPC decoder with floating point simulation to test its improved error rate and converging performance.
Our contribution in this paper can be stated as follows: We develop a GSVS-min-sum LDPC decoding for the layered implementation, In addition, we use finite precision, i.e. quantized practical implementation. We propose using the DDE to jointly optimize the quantization step of uniform quantizer δq, initial scaling factor α0 and the scaling factor updating step S (for each constellation size and code rate). We use the proposed optimization strategy with LDPC codes of DVB-S2 standard [5] (with BPSK and 256-QAM). Since the optimal quantization step of uniform quantizer significantly differs with the used constellation size specially for low rate codes, we propose to use an improved quantization step that changes with the constellation size and code rate to reduce the loss of applying quantization on the floating point LDPC decoder.
The rest of the paper is organized as follows: Section II presents the necessary background on LDPC decoding and quantization. Section III presents the proposed optimization strategy of scaling and quantization of LDPC decoder. The simulation environment and results are displayed and discussed in Section IV. Finally, the paper is concluded in section V.
II. BACKGROUND ON LDPC DECODING AND
QUANTIZATION
An (n, k) LDPC code is a binary code characterized by a sparse parity check matrix H ∈ Fm 2 ×n where m = n − k. It can be represented by a Tanner graph, which contains variable nodes with indices j ∈ {1..n} and check nodes with indices i ∈ {1..m} . We denote the set of variable nodes connected to a certain check node i as V {i}. Furthermore, the set V {i}/j denotes the set of variable nodes connected to check node i excluding j. Similarly, the set of check nodes connected to a certain variable node j is denoted by C{j}. C{j}/i denotes the set of check nodes connected to the variable node j excluding i.

B. Layered implementation of LDPC decoder
In flooding implementation of LDPC decoder, horizontal step is applied on all check nodes concurrently, followed by applying the vertical update step on all variable nodes concurrently. Consequently, Eq. (1) uses the messages of previous iteration for all check nodes. As shown in [3], if partial parallel implementation (layered implementation) is used, i.e.

the updated check nodes messages of the current iteration are used in the horizontal step of remaining check nodes, the number of required iterations decreases to 50%. In this paper, we develop a GSVS-min-sum for layered implementation of LDPC decoder to further improve the convergence speed and the output error rate of the decoder.
C. Quantization of LDPC decoder
Representing belief propagation messages in finite precision (LDPC quantization) is essential for practical implementation. Many articles concentrate on LDPC quantization [6]– [9]. For specific number of bits per message, quantization step design can severely degrade the LDPC decoding performance; large quantization step decreases the resolution of channel LLR Uch representation and increases the probability of erasure ( probability of Uch = 0). Consequently, the error probability increases especially in the waterfall region and decreases the convergence rate. On the other extreme, small quantization step decreases the representation region (saturation level), this low saturation level prevents high reliable messages from propagation through the Tanner graph and solve the trapping sets which is the reason of high error floor in LDPC decoding [9]. This trade-off, between waterfall performance and error floor, increases the importance of uniform quantization step design to obtain the best waterfall performance with the lowest error floor, and this is one of our targets in this paper.
Non-uniform quantization [9] uses relatively small quantization step for small LLR values to improve the waterfall region performance, and uses large quantization steps for large LLR values to decrease the error floor. Although non-uniform quantization successfully solves the finite precision problems, it produces additional complexity in mathematical operations like addition in (3) and (4). To avoid this additional complexity, we adopt uniform quantization with optimized quantization step.
In [8], different quantization step (δq) was used for each code rate to overcome the channel LLR saturation problem. Calculation method in [8] is used to calculate δq[8] in Table II for DVB-S2 long codes with different rates. Section IV shows the good performance of calculated δq[8] for BPSK case, which is not the case for higher constellation size.
III. PROPOSED SCALING AND QUANTIZATION STEP
DESIGN
In this paper, we apply GSVS-min-sum LDPC decoding on layered implementation with finite precision, in addition to, jointly optimize GSVS-min-sum parameters (α0, S) with the uniform quantization step δq. Consequentially for each constellation size and code rate, an optimal quantization step and optimal GSVS-min-sum parameters are used in LDPC decoding to achieve the maximum decoding performance. Different GSVS-min-sum parameters can simply implemented by the circuits in [10], and a simple implementation of different quantization step can be achieved by the procedure in [8].

B. Proposed scaling and quantization parameters Optimization
We propose to use a modified version of DDE with Nelder-Mead [13] optimization method to jointly optimize the quantization step, initial scaling factor of GSVS-min-sum and the updating step of GSVS-min-sum (δq, α0, S). The used DDE (with our modifications described in III-B1 and III-B2) is used to evaluate the performance of a set of parameters, by calculating (Eb/N0)min, which is the minimum Eb/N0 that can achieve a pre-specified Bit Error Rate (BER) using this set of parameters. The pre-specified BER controls the acceptable error floor, which can be corrected by an outer code, e.g. a BCH code is used in DVB-S2 as an outer code to eliminate the error floor [5]. Nelder-Mead optimization method is used to solve the 3-dimension optimization problem of finding the set of parameters (δq, α0, S)opt. that minimizes (Eb/N0)min (similar to GSVS-min-sum parameters optimization in [10]), so we get the optimal set of (δq, α0, S) for any LDPC code with any used constellation size. Jointly optimization of these three variables is used because it achieves better results than optimizing them separately.

DDE evaluates the ability of an LDPC decoder to successfully decode received codewords with given noise variance. This is done by tracing the Probability Mass Function (PMF) of finite precision messages passed between check and variable nodes. DDE is used in [14] to obtain the optimal quantization step and scaling factor of scaled min-sum LDPC decoder for BPSK modulation using flooding implementation. We use the same DDE as in [14] after modifying it to layered implementation, and changing the PMF of channel LLR so we can use it with QAM signalling.

1) Modifying the channel LLR PMF to M-QAM over an AWGN channel: In DDE with BPSK [14], all-zero code-word is assumed because the Probability Density Function (PDF) of the received signal if logical one is transmitted is similar to logical zero transmitting case. This similarity does not exist with higher order constellations [15]. In addition to this, BPSK is similar to 4-QAM which is a special case of general M-QAM with M = 4. Therefore, we have to replace the BPSK in DDE with M-QAM.

To use M-QAM in DDE instead of BPSK, we use a similar procedure to [15], where authors modified the definition of bit’s LLR to be: “LLR of receiving the same bit value as was transmitted” instead of “LLR of receiving 0”. This is equivalent to replacing Uch by Uc+h, where Uc+hj = Uchj × (1 − 2Vj ) where Vj is the transmitted bit number j. So Uc+hj will be positive if and only if Uchj has the same sign as given by Vj (correctly received). Consequently the average PMF of Uc+h , over all possible bit positions into M-QAM constellation point mapping and over logical 0 and logical 1 transmission cases, is a valid indicator of correct decoding and represents all possible cases of M-QAM transmission. IV. SIMULATION ENVIRONMENT AND RESULTS
In our simulation, we use the LDPC codes of DVB-S2 with long codeword (n = 64800). To show the effect of constellation size and code rate on the optimal values of quantization step and GSVS-min-sum parameters, we calculate them for different coding rates with BPSK and 256-QAM in Table II. For comparison, Table II also shows the quantization step calculated by the technique in [8] δq[8] which is used for both BPSK and 256-QAM. Table II shows that:
1) The quantization step dramatically decreases with coding rate increasing, for our proposed method with BPSK δqopt.−BP SK and with 256-QAM δqopt.−256−QAM and for method in [8] δq[8]. The reason of this inverse proportionality is that, higher coding rate means lower ability in correcting error bits and erasure bits (Uch = 0), this leads to requiring lower erasure probability and higher resolution, in other words lower quantization step.

2) The rate between the optimal quantization step of BPSK δqopt.−BP SK and of 256-QAM δqopt.−256QAM illustrates the big difference between them, which decreases with the code rate, this supports the requirement of using different quantization step for each constellation size specially for low rate codes. Different quantization step for each constellation size can be simply implemented by the same technique used in [8] for different quantization step with different code rates.
3) The optimal GSVS-min-sum parameters change with coding rate and constellation size, this leads to using different scaling parameters for each constellation size code rate pair. This is simply implemented by using the circuits in [10].
4) There is a small difference between δqopt.−BP SK and δq[8] for all code rates (except 1/4), this small difference explains the good performance of δq[8] with BPSK as shown in Fig. 1.
5) For low rate codes, there is a large difference between δqopt.−256QAM and δq[8], this explains the poor performance of δq[8] with 256-QAM as shown in Fig. 2.
To verify the performance of our proposed optimization technique, we simulate the layer decoding performance of DVB-S2 LDPC long code (n = 64800) with rates 1/2, using the maximum parallelism size P = 360, and using 5 bits to represent the belief propagation messages, with at most 20 iterations and using the conflict resolution in [17]. The quantization and GSVS-min-sum parameters are shown in Table II as bold format. The simulation results of the BER (represented in dashed lines) and the Word Error Rate (WER) (represented in solid lines) for BPSK and 256QAM systems are shown in Figs.1 and 2, respectively. The simulation results are calculated after simulation of 106 packets or failing in decoding 20 packets. In Figs. 1 and 2:
1) The lines with marks (circle, square or diamond) represent the performance of the proposed GSVS-min-sum decoding algorithm with different quantization schemes. On the other hand, lines without marks show the performance of scaled min-sum decoding algorithm with constant scaling factor for all iterations ατ = 0.75 (used as the scaling factor in most literatures).
2) Floating point simulation results are shown to illustrate the scaling factor effect (without quantization). For both BPSK and 256-QAM, it is clear that GSVS-min-sum ( represented with diamond marked blue line) has better BER and WER performance than constant scaling factor ( represented with blue unmarked line). The reason of this advantage is that ατ = 0.75 is far from the optimal scaling factor for each iteration.
3) To illustrate the effect of quantization step only (without scaling), the simulation results of GSVS-min-sum decoding algorithm with the proposed optimal quantization step (and quantization step of [8]) are shown in black square marked (and red circle marked) lines. For 256-QAM, the proposed quantization step has a much better performance than quantization step of [8] in both BER and WER for both waterfall and error floor regions. On the other hand, BPSK performance shows that the proposed quantization step has superior performance for large Eb/N0 range and has lower error floor. V. CONCLUSION
In this paper, we have proposed a fast converging scaled Min-Sum layered LDPC decoding implementation. The performance of the floating point proposed GSVS-min-sum is superior to the regular Min-Sum with modest complexity added. In general, quantization increases the error floor and negatively affects the convergence of the actual LDPC implementation. In order to combat this shortcoming and optimize the achievable

performance, we jointly optimize the GSVS-min-sum parameters with the step size of the uniform quantization (for each code rate and constellation size). Simulation results confirm the enhanced performance of the joint optimization.
