Abstract —In this paper, an efficient anytime code derived
from protograph-based low-density parity-check convolutional
(P-LDPCC) code is proposed. Density evolution technique is
applied to demonstrate the anytime properties of the P-LDPCC
code over the binary erasure channel (BEC). Through simulation,
the finite-length performance is evaluated and the effects of code
design parameters are observed. The delay exponent is evaluated
using the density evolution functions and simulations of finite-
length codes. The simulation results show that P-LDPCC code
outperforms existing LDPCC anytime codes.
I. INTRODUCTION
Anytime coding has been shown to be an essential strategy for delay-sensitive applications, e.g., real-time communication over noisy channels [1]. It has also been shown to be necessary and sufficient for tracking and controlling unstable processes [2], [3]. In anytime coding, information symbols trickle into the encoder in real time and the encoder encodes the symbols causally. In anytime decoding, at each time instant, the decoder recovers the current as well as previously information symbols. The important requirement of anytime coding is that the decoding error decays exponentially with delay [4].
Two different practical anytime code designs based on LDPCC codes were presented in [5] and [6]. The systematic LDPCC (S-LDPCC) code [5] is easy to construct due to its fixed code structure and low encoder complexity. However, the finite-length performance of this code exhibits high error floor where the probability of error does not decay with the decoding delay. The other anytime code proposed in [6] was constructed based on spatially coupled LDPC (SC-LDPC) code. In the finite-length regime, SC-LDPC anytime codes offer much lower error floors than S-LDPCC codes. A drawback, however, is that the structure of SC-LDPC anytime codes is fully random, and consequently, these codes have poorer finite delay performance and a more complicated encoder compared to S-LDPCC codes.
The aforementioned observation suggests that introducing randomness into S-LDPCC code will lower its error floor while maintaining the finite-delay performance gain in comparison to SC-LDPC codes. This work develops an improved protograph-based LDPCC (P-LDPCC) anytime code by modifying the S-LDPCC code. We show that the P-LDPCC code

asymptotically achieves the desired anytime property. Our P-LDPCC code can be theoretically analyzed and provides a flexible trade-offs between performance and code design parameters. The contributions of this paper are summarised as follows:
• We develop an improved anytime reliable code that is an extension of S-LDPCC codes.
• Through density evolution analysis, we can both observe the anytime properties of the P-LDPCC code and evaluate the delay exponent of these codes.
• We show by simulations over the BEC that finite-length P-LDPCC codes perform better than existing anytime codes [6]. We also show how the performance of PLDPCC codes vary with different design parameters.
The remainder of the paper is organized as follows. Section II introduces a formal description of anytime transmission. A brief description about the anytime code proposed in [5] is given in Section III. In Section IV, we present the P-LDPCC code construction in detail. The finite-length performance and the performance comparison with other anytime codes are given in Section V. The asymptotic analysis and delay exponent evaluation of P-LDPCC codes is detailed in Section VI. Finally, this work is concluded in Section VII.

• S-LDPCC codes can be obtained from P-LDPCC codes by setting w = 1. Under the setting that w = 1, the delay exponent in (15) reduces to the delay exponent of S-LDPCC codes presented in [5].

• Estimates of the delay exponent can also be obtained using the slopes of bit erasure probability curves from the finite-length simulations of section V, Table I shows that this estimate also matches well with (15).

VII. CONCLUSION
In this paper, a novel protograph-based LDPCC code is proposed for point-to-point anytime transmission setup. This code asymptotically achieves the anytime characteristic – the decoding error probability asymptotically decays with the increase in delay. Finite-length versions of this code provide a more flexible way to compromise the expected anytime property with desired decoding performance.
