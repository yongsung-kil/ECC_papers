Abstract—The finite-length performance of multi-edge spatially coupled low-density parity-check (SC-LDPC) codes over the binary erasure channel (BEC) is analyzed. Existing scaling laws are extended to arbitrary protograph base matrices that include puncturing patterns and multiple edges between nodes. A regular protograph-based SC-LDPC construction based on the (4, 8)-regular LDPC block code works well in the waterfall region compared to more involved rate-1/2 structures proposed to improve the threshold to minimum distance trade-off. Scaling laws are also used for code design and to estimate the block length of a given SC-LDPC code ensemble to match the performance of some other code. Estimates on the performance degradation are developed if the chain length varies.
Index Terms—finite-length performance, spatially-coupled LDPC codes
I. INTRODUCTION
Spatially coupled low-density parity-check (SC-LDPC) codes achieve capacity over binary-input memoryless symmetric (BMS) channels under belief propagation (BP) decoding [1]. They are constructed by coupling a chain of L LDPC codes of length M bits each with appropriate boundary conditions. For large L, SC-LDPC code ensembles exhibit a BP threshold close to the maximum-a-posteriori (MAP) threshold of the uncoupled LDPC code ensemble [1], [2]. Besides, it is known that SC-LDPC code ensembles can be desigend with a linear growth of the minimum distance that is inherited from the uncoupled LDPC code ensemble [3].
Multiple families of SC-LDPC codes have been proposed and analyzed in the past years. In [1], the authors consider a family of codes generated by coupling a series of (l, r)-regular LDPC codes using a random pattern, which is convenient for asymptotic analysis via density evolution (DE). Alternatively, we can construct SC-LDPC codes with a predefined coupling structure using protographs [4], defining multi-edge type (MET) ensembles. The inherent structure in protograph-based SC-LDPC code ensembles can improve minimum distance properties [3]. Note that comparisons between different SCLDPC code ensembles in the literature are typically based on asymptotic arguments, i.e. threshold and minimum distance growth.

In this paper, we compare some well-known protographbased SC-LDPC code constructions using finite-length scaling arguments. Recent results have addressed finite-length SCLDPC codes. In [5], the authors propose a scaling law for the binary erasure channel (BEC) and a random SC-LDPC code ensemble. This analysis was extended in [6] to a class of protograph-based SC-LDPC code ensembles, where it was shown that the protograph-structure improves the finite-length properties of the SC-LDPC code ensemble compared to random SC-LDPC codes.
We extend the analysis of finite-length protograph-based SC-LDPC codes in [6] to structures based on multiple edges between nodes and puncturing patterns. For instance, our analysis applies to SC-LDPC codes based on accumulate-repeatby-4-jagged-accumulate (TAR4JA) structures analyzed in [3] or repeat-accumulate (RA) structures in [7]. First, we show that their finite-length performance can be estimated by means of an scaling law. Apart from small differences, no particular SC-LDPC code family clearly outperforms the others in terms of asymptotic performance (threshold). However, we show that indeed these construnction present very different finite-length scaling properties. Beyond simulation-based comparisons, we provide tools to predict these results analytically over the BEC using the scaling parameters computed for each ensemble, opening the door for fast practical SC-LDPC code design.
We also use scaling laws to estimate the block length of a given SC-LDPC code ensemble to match the performance of another code, and to estimate how the performance degrades if the chain length varies.

CONSTRUCTIONS AND THRESHOLDS Consider SC-LDPC codes constructed from protographs [4] which are used as templates to construct a larger code: the protograph is copied several times before the edges between the same type of sockets are permuted. This copy-and-permute operation is called lifting and avoids small girths which often harm iterative decoding. In the following, we review four relevant constructions: the standard (l, r, L)P construction [6], a spatially coupled RA (SC-RA) construction [7] and a spatially coupled TAR4JA (SC-TAR4JA) construction [3].

BEC TRANSMISSION AND PEELING DECODING The finite-length performance of LDPC codes in the waterfall region is often analyzed for the BEC, e.g. uncoupled (l, r)-regular codes in [8] and random SC-LDPC codes in [5]. We use peeling decoding (PD), which is equivalent to BP [9]. The PD is initialized by removing all VNs whose value was received through the channel and their edges. At each step, the PD removes one degree-one (deg-1) CN and its connected VN, as well as all edges connected to these nodes. The finite-length analysis studies the statistical evolution of the residual graph as a function of time. It suffices to analyze the evolution of the degree distribution (DD) of the residual graph [9]. Estimating the error probability involves computing the probability that the process representing the fraction of deg-1 CNs in the residual graph reaches zero before all VNs have been determined.

This encourages further investigation on capacity-approaching SCRA ensembles since they promise both good asymptotic and finite-length properties.

In Section IV-A, we used (11) to predict the lifting factor M of a given SCLDPC code to match the performance of another code. We now predict the performance degradation depending on an increasing chain length L with (11). In the low-error rate regime, (13) suggests that the performance scales linearly with L. This is verified by simulation as shown in Fig. 8 for both the SC-RA ensemble with q = a = 4 and the SC-TARJ4A ensembles for L = 100, L = 150 and L = 200.
