Abstract— Review of self-correction applied for min-sum based decoding algorithms considered. Complexity, performance, and average number of iteration for each algorithm are shown.
Keywords- error correction, LDPC decoding, min-sum algorithm, a-min* algorithm, a-min*-min algorithm, selfcorrection
I. INTRODUCTION
Low density parity check (LDPC) codes was proposed by R.Gallager [1] in 1963, and later rediscovered by MacKay and Neal [2]. Like turbo codes, LDPC codes can achieve near Shannon limit error performance [3], [4], and represent a very promising prospect for error control coding. Nowadays they are widely used in many applications, all modern standards such as DVB-T2, DVB-S2, DVB-C2, WiFi, WiMax, IEEE 802.15.3 used this codes [5][6].
Any LDPC code is specified by a parity-check matrix which contains mostly zeros and only a small number of ones. A regular (N,K) LDPC code, with block length N and dimension K, has a parity check matrix H of M ı N - K rows and N columns with exactly dc ones in each column and dr ones in each row. We refer to the M rows of H as checks, and to all the elements of an LDPC codeword as bits. Then each check contains dr bits and each bit is checked by dc checks.
Gallager also proposed algorithms for decoding LDPC codes, the most popular of them is soft decision based, near optimum belief-propagation (BP) algorithm, also known as sum-product algorithm (SPA), and a hard decision based bitflip (BF) decoding algorithm. They are iterative decoding algorithms.
The main advantage of BP decoding algorithm is it high performance, that allows to achieve near Shannon limit, and the main drawback is it complexity, that remains relevance nowadays for applications with limited computational and energy resources.
The BF algorithm has a reverse disposition. The main advantage is simplicity and the main drawback poor performance.
Due to high complexity of BP decoding algorithm, many authors try to compile advantages of BP and BF algorithms in

one algorithm, and a lot of modifications and new ways of decoding LDPC codes were proposed from its invention. The most significant ways of LDPC decoding evolution are to simplify BP algorithm with loss in performance, and to complicate BF algorithm with gain in performance. Nowadays a lot of such modifications are known [7], [8], [9].
In [10] was proposed to use self-correction as a modification for min-sum algorithm (MSA), self-correction further studied in [11], [12]. It was shown that using of selfcorrection for min-sum algorithm can significantly increase its performance. In this paper using of self-correction in cooperation with other modifications of min-sum algorithm was investigated. For our research we used a-min* [13] and amin*-min [14] decoding algorithms.
The paper is organized as follows. In Section II, we give a brief review of self-corrected min-sum algorithm, a-min* algorithm and a-min*-min algorithm, and motivation for selfcorrected a-min* and a-min*-min algorithms. Simulation results for proposed algorithms provided in Section III. Finally, a conclusion is given in Section IV.

II. MIN-SUM BASED DECODING ALGORITHMS

A. Self-corrected min-sum algorithm

Self-corrected (SC) min-sum decoding detects unreliable information simply by the sign fluctuation of variable node messages. Precisely, any variable node message changing its sign between two consecutive iterations is erased, meaning that any such a fluctuating message is set to zero.

C. A-min*-min algorithm
A-min*-min algorithm is a further simplification of a-min* decoding algorithm. It doesn’t use (5) for computing outgoing message in check node update for all other bits except of least reliable bit. Expression (5) replaced by simple min operation.

D. Self-corrected min-sum modifications
Since variable node update is for a-min* and a–min*-min algorithms is the same as for min-sum algorithm, so we can use self-correction technique for this algorithms in the same way.

III. SIMULATION RESULTS
Performance and convergence research were investigated for two LDPC codes. First is a well-known for a number of publications PEG LDPC code N=1008, M=504 [15], and the second is a quasi-cyclic EG N=8176, M = 7156 LDPC code [5]. In both cases maximum number of iterations is 100.
Simulation results for PEG code were shown on Figures 1 and 2. Using self-correction has positive results for both considered algorithms. Furthermore self-corrected a-min*-min algorithm have better performance than a-min* algorithm.
Using self-correction for a-min* algorithm improve its performance almost for 0.1 dB.
Average number of iteration insignificantly increased for both algorithms.
Using self-corrected algorithms for EG code also has positive results, but not so significant than for PEG code, that can be explained higher code rate of EG code.
Unlike PEG code average number of iterations is almost the same as for a-min* algorithms, or slightly less as for a-min*min algorithm.

IV. CONCLUSION AND FUTURE WORK Analyses of obtained results for PEG and EG codes show, that one can use self-correction with positive results not only for MSA algorithm, but for its modifications as well. In this paper we considered two MSA modifications a-min* and amin*-min algorithms, it can be assumed, that for other MSA modifications same improvement can be achieved, but for detailed results further researches are needed.
