Abstract- SER is increasing for every IC process generation.
Radiation induced soft errors are major concern in semiconductor memories due to technology scaling, higher integration densities and lower operating voltages. Nowadays memory cells are protected by using error correction codes. Among various multiple error correction codes ML decodable codes are suitable for memory applications due to their capability to detect a large number of errors but it requires large decoding time. In this paper a special type of low density parity check (LDPC) codes, which belongs to the family of majority logic decoding called Euclidean Geometry Low Density Parity Check (EG-LDPC) codes which detects the error in less cycle time so the decoding time is greatly reduced and also the memory accessing time also get reduced. The recent paper deals only with error detection in memories but the present paper focus on both error detection and correction by modified implementation of majority gate. The simulation results are compared with the existing techniques.
Index terms- Soft error rate (SER), majority logic (ML), Euclidean geometry- low density parity check (EG-LDPC), memory.
I. INTRODUCTION
In most electronic and computer applications memory plays a major component as a storing device. The data bits are stored in the memories as a small packet of charge. The stored data may be modified, when it is affected by noise. A soft error is a type of error where a signal or datum is wrong. Soft errors may be caused by radiations, electromagnetic interference or electrical noise. The error is 'soft' because the device itself is not permanently damaged by the radiation. This paper focus on soft errors only due to radiations. These soft errors happen at a rate, called the soft error rate (SER) that can be predicted

as a function of the memory density, the memory technology, and the geographic location of the memory system. In earlier days memories are affected by radiation only in aircraft applications, sea level applications and systems used at high altitudes. Later terrestrial level applications are also affected by radiations.
With continuing technology scaling, circuits become more sensitive to soft errors. The three major radiation sources are due to alpha particles, high-energy cosmic rays, and neutron­ induced boron fission. Alpha particles originate from radioactive impurities in chip and package materials. Alphas induce soft errors by generating charges in the silicon device. Cosmic rays, predominantly neutrons, indirectly generate charges by colliding with nuclei within the chip. The products of such collisions are particles that are capable of ionizing silicon atoms. The cosmic ray component of the SER has become more important as increased material purity has reduced the SER due to alpha particles from radioactive contaminants. The third source boron fission, occurs when a low-energy (thermal) neutron hits a lOB nucleus, which then breaks up into an alpha and a lithium recoil. This source gives a significant contribution if specific materials, in particular boron phosphor-silicate glass (BPSG), have been used in the fabrication of the chip. Soft errors occur in SRAM and DRAM devices, but not in ferroelectric RAMs (FRAMs), magnetic RAMs (MRAMs), or flash memories. In DRAM, error rates were widely expected to increase as devices became smaller instead, small scale DRAMs, demonstrate a much better error resistance. One reason for this is that their smaller size allows less charge collection and another reason is that cell size has scaled faster than storage capacitance. Alpha particle contamination has been virtually eliminated in DRAM

To prevent soft errors from causing data corruption, memories are typically protected with error correction codes (ECCs).The most commonly used error detecting technique is Triple Modular Redundancy (TMR). The concept of TMR can be employed to ensure very high soft-error reliability in logic circuits. In this technique, three identical copies of a circuit compute on the same data in parallel and outputs are fed into majority voting logic, returning the value that occurred in at least two of three cases. It requires large area and complexity overhead are the drawbacks of this method. Therefore the ECC became the best way to mitigate soft errors in memory. The most commonly used ECCs are Single Error Correction (SEC) codes and Double Error Correction (DEC) codes. Their main advantages are that they require few additional bits per word and the decoding process also simple. The drawback of this method is detect only single bit flips and have significant power consumption penalty. The commonly used multi error correction codes are Reed Solomon (RS) and Bose Chaudhuri­ Hocquenghem (BCH) are not suitable for this task due to complex decoding algorithm. Among various ECCs cyclic block codes are identified as good choice due to their property of being majority logic (ML) decodable codes, and it also have low decoding complexity. Difference-Set Cyclic Codes (DSCCs) is a type of low-density parity check (LDPC) codes, it is very simple to implement and has low complexity but requires a large decoding time. This paper explores the idea of using a special type of low­ density parity check (LDPC) codes called Euclidean Geometry Low Density Parity Check (EG-LDPC) codes, it detects the error words in less cycle so the memory accessing time is greatly reduced because most of the words are error free. Also, errors affecting more than five bit flips also detected with a probability very close to one. The recent paper focus on only error detection, but the current paper focus on both error detection and correction by a modified majority logic. The extensive simulation results shows that error detection and correction for different code word size using EG-LDPC codes.

11. EXISTING MLD SOLUTIONS

Majority logic decoding (MLD) is efficient one compared with other techniques because it has low complexity and the decoder itself acts as a detector so the additional circuit requirement for detector is reduced. The different type of MLD is discussed as follows.

A. Plain ML Decoder

The ML decoder is a simple and powerful decoder because it has capability to correct multiple bit flips depends upon the number of parity check equations. In each iteration single code word bit only participates in one parity check equation, except the very first bit which contributes to all equations, so the majority result of these parity check equations decides the correctness of the current bit under decoding. The extra component added in the MLD to fmd the fault bits in the code word by calculating the syndrome, so that only faulty code words are decoded. C. MLD with DSCCs
The recent paper use MLD implementation using Difference Set Cyclic Codes (DSCCs) which belongs to the family of Low Density Parity Check (LDPC) codes. It is obtained by serial implementation of majority logic decoding of DSCCs­ LDPC codes. The idea used in this method is to use the first iterations of majority logic decoding to detect if the word being decoded contains errors. If there are no errors, then decoding can be stopped without completing the remaining iterations, therefore greatly reducing the decoding time. If the size of the code word increases then the number of iterations also increased to decode the code word due to majority logic decoding is implemented in serial manner.
Ill. ABOUT EG- LDPC
Low Density Parity Check (LDPC) codes are class of linear block codes. Here the "Iow density " refers to the characteristics of the parity check matrix which contains few 1's compared with number of O's. The LDPC codes are represented either in matrices or in graphical form. An LDPC code parity-check matrix is called regular, it contains fixed number of rows and fixed number of columns. An LDPC code parity-check matrix is called irregular, it contains random number of rows and columns. The other method representation of LDPC is graphical representation. The graphical representation is obtained by using tanner graphs. Tanner graphs are bipartite graphs. That means that the nodes

Among various LDPC codes here we use a special type of codes is called Euclidean Geometry Low Density Parity Check (EG-LDPC) codes which belongs to the family of LDPC. EG-LDPC is effective one compared with other techniques because it is one step majority logic decodable codes and also cyclic block codes. The finite geometry technique is used to derive the error correction technique. The structure of this code is based on the Euclidean Geometry over a Galois field. The finite geometry G with 'n' points and 'j' lines have some structural properties they are given as follows (1) every line consists of points; (2) any two points are connected by one and only one line; (3) every point is intersected by lines (i.e., every point lies on lines); and (4) two lines are either parallel (i.e., they have no point in common) or they intersect at one and only one point. To construct a cyclic EG LDPC code based on EG(m,2s), we form the parity check matrix HEG whose columns are all of the 2ms - 1 non-origin points in EG(m,2s), and whose rows are the incidence vectors of all of the JO lines in EG(m, 2s) that do not contain the origin. The codes used in this paper all have m = 2, so JO = N = 22s-1, which means that the parity check matrix that we use has an equal number of rows and columns (some of the rows are redundant). The EG-LDPC codes are used here in order to reduce the decoding time, and to increase the memory performance.

VI. EFFICIENT EG-LDPC DECODER
Decoder circuit is used to decode the data stored in the memory. Among various decoders EG-LDPC is considered as efficient one compared with other decoding techniques due to EG-LDPC codes are one-step majority logic decodable (MLD) codes. There is no additional requirement of detector circuit, the decoder itself used as a decoder circuit so the additional cost for detector circuit is reduced. The Different Set Cyclic Codes (DSCCs) are a sub group of LDPC codes which are serially implemented in majority logic decoding, it has advantage of detect errors in first iterations of majority logic decoding but it has a disadvntage that the decoding time increases when the size of the code get increased so, it impacts the memory performance of the system. In order to over come this drawback the proposed method use a EG-LDPC decoder. The EG-LDPC decoder detects the word having errors in first three iterations and most of the words in the memory will be error free so the decoding time will get greatly reduced. The operation of the majority logic decoding is depends on the number of 1's and O's of majority logic circuit. The xor matrix is evluated for the first three cycles of the decoding process and when all the outputs of the xor matrix are '0' then the code word will be considered as error free. If the result of the xor matrix have atleast single ' l' in any of three cycles. The efficient EG-LDPC detector detects the errors in first thee iterations depends on the result of the majority logic output. VII. EG-LDPC CORRECTOR The efficient method for error correction is achieved by using one step majority logic. EG-LDPC is a special type of LDPC code used for efficient errorcorrection. The error correction is depends on the result of the parity check sums, if all the parity check sums are 'O's then there is no need for error correction. The majority gate has atleast single 'l' then the bit undergoes correction process. The correction process is implemented by using cyclic shift register and a comparator circuit. The last bit of the code word in the cyclic shift register get inverted if the result of the xor matrix has '1's in the majority logic output, otherwise it is kept unchanged. The structure of serial one step majority logic correctorfor 15 bit code word using EG-LDPC code is shown in figure 7.

Figure 7. Serial one step majority logic corrector structure for (15, 7) EG-LDPC code.
The correction is achieved by using some procedures are given as follows Generate y parity-check sums by computing the inner product of the received vector and the appropriate rows of parity-check matrix. The y check sums are fed into a majority gate. The output of the majority gate corrects the bit by en-linverting the value of en-lif the output of majority gate is "1 ". The serial majority corrector takes n cycles to correct an erroneous code word. To correct each code word, the received encoded vector is cyclic shifted and fed into to the XOR gates as shown in figure 7. If the fault rate is low then the serial corrector performs in effective manner, because most of the words in the memory are error free. For high error rates, the corrector is used more frequently and its latency can impact the system performance. Therefore we can implement a parallel one-step majority corrector which is essentially n copies of the single one-step majority-logic corrector is used to correct the errors. The majority gate gives the majority of binary values present at the output of the majority logic either may be O's or 1'So The sorting technique is used to sort the majority of binary values present at the output. The comparator circuit is used to compare the decoded output and the encoder output so the correction is performed. The comparator circuit used to compare the majority and minority values present at the majority gate. The efficient EG-LDPC technique is used to detect more than five bit flips in the code word. The decoding time is greatly reduced by using the EG­ LDPC codes so the memory accessing time also decreased.
