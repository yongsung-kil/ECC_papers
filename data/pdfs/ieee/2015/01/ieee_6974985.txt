A Low-Complexity LDPC Coding Scheme for Channels With Phase Slips
Laurent Schmalen, Member, IEEE
(Invited Paper)

Abstract—We propose low-complexity coding schemes that are resilient against sporadic phase slip events. These schemes are based on a new phase-slip transparent construction of LDPC codes, called block symmetric codes and on outer differential coding. The slip transparent LDPC codes correct single bit error events but are transparent to phase slips. The phase slips, which correspond to runs of bit errors, are corrected by the outer coding part of the receiver. We present practical binary and nonbinary code constructions for multiple modulation formats, and evaluate the proposed scheme by means of simulation examples.
Index Terms—Error correction codes, iterative decoding, lowdensity parity-check codes, phase slips.
I. INTRODUCTION
I N coherent receivers for high-speed optical communications, usually feed-forward phase recovery algorithms are employed [2]. These algorithms exploit the rotational invariance of the constellation for estimating the phase offset. However, due to the necessary phase unwrapping algorithm in the estimator, a phenomenon called phase slip (sometimes referred to as cycle slip) occurs. The probability that a phase slip occurs depends among others on the non-linear phase noise introduced by the optical transmission link.
If the phase slips are not recovered, error propagation will occur at the receiver and all data following a phase slip cannot be properly corrected by the forward error correction decoder (FEC). Several approaches to mitigate phase slips have been proposed whereof the most common is differential coding, which renders a phase slip into a single error event. In order to alleviate the penalty from differential coding, iterative decoding between a FEC decoder and a differential decoder can be beneficial [3]. This solution leads, however, to an increased receiver complexity, as several executions of a soft-input soft-output differential decoder (e.g., based on the BCJR algorithm [4]) are necessary.
In a recent publication [5], a simple multi-level coding scheme has been presented, where the differential decoder is not placed in front of the modulator but in front of the systematic multi-level FEC encoder. That system has the major disadvantage that it is not robust against phase slips but can only correct a phase offset

III. NEW SLIP-RESILIENT SCHEME FOR V = 2
As in [5], we swap the positions of LDPC encoder/decoder and differential encoder/decoder and place the differential decoder not in front of the modulator but in front of a systematic FEC encoder [1]. In contrast to the scheme of [5], we use a code that can tolerate phase slips occurring at any position inside the codeword. We first describe this new scheme for constellations with V = 2-fold rotational invariance and then extend it to constellations with V = 4 in Section IV.
We seek for a code that does not only allow for a phase slip at the boundaries of each codeword [5], but that additionally tolerates several phase slips within one codeword. The idea is to use a systematic LDPC code that is slip transparent (ST), i.e., a code which only corrects errors due to noise but leaves the phase slips in the systematic portion of the codeword (systematic means that the information word is present verbatim in the codeword and parity bits are appended). The residual phase slips will then be corrected by an outer (hard decision) differential decoder and the remaining errors are corrected by the outer algebraic cleanup code. This system is shown in Fig. 4. For reasons that will become clear below, we denote the ST LDPC code as BS-LDPC code. We start by describing the scheme for the BPSK modulation format.
In order for the scheme of Fig. 4 to be phase slip tolerant, we need an LDPC code that is systematic. Whenever a phase slip occurs within the codeword, the decoder should not aim to correct the phase slip but should only correct the single bit errors and leave the phase slip to be corrected by the outer differential decoder. If a phase slip occurs at time instant t, all symbols at t, t + 1, t + 2, . . . have inverted signs, which means that their bit value is inverted. We therefore seek for a code that is invariant to block-wise bit inversions. We first assume that phase slips may only occur at time instants t = t · S + 1, with t ∈ {0, 1, . . .} and S integer. For example, we assume that a first phase slip occurs at time t1 = S + 1 and a second phase slips occurs at time t2 = 2S + 1. This means that S consecutive bits at positions (S + 1, . . . , 2S) in the codeword are flipped. If we sub-divide the LDPC codeword of length n into n sub-blocks of length S, for every codeword

x ∈ CLDPC , an inversion of the second sub-block must also yield another valid codeword. In general, any combination of bit-inverted sub-blocks of a valid codeword must also be a valid codeword.
In analogy to the class of symmetric codes [12], which is defined such that the inversion of a complete codeword is another codeword, we call the proposed generalized class of codes blocksymmetric (BS) codes.
Definition 2 (BS Code): A BS code of block size S is a code C of length n = n S where each codeword x ∈ C can be subdivided into n blocks xi = (xi,1 , xi,2 , . . . , xi,S ) of S bits each, i.e., x = (x1 , x2 , . . . , xn ) and where the inversion of a single or more blocks leads again to a valid codeword (∈ C). This means that
x˜ = (x˜1 , x˜2 , . . . , x˜n ) ∈ C, where x˜i ∈ {xi, x¯i}
with x¯i = (1 ⊕ xi,1 , 1 ⊕ xi,2 , . . . , 1 ⊕ xi,S ) where “⊕” denotes the eXclusive OR operation.
If the BS condition is fulfilled and the code is systematic, the decoder will correct the transmission errors and leave the phase slip uncorrected, which can then be removed by the differential decoder. As each phase slip results necessarily in a bit error after differential decoding, an additional outer code is needed to cleanup these bit errors. We may easily relax the unrealistic constraint that phase slips occur at positions t S + 1. The decoder will then force the phase slip to occur at these boundaries, affecting, however, the total budget of correctable errors by the code. At a first glance it is tempting to reduce S, however, reducing S will degrade the error correction properties of the code, as:

With an outer differential decoder, a phase ambiguity of a whole codeword can be resolved. We propose to use a single ST code as in the previous sections that is able to tolerate phase slips at arbitrary positions. The proposed system model is shown in Fig. 5. As above, we exchange the positions of the inner FEC code and the differential code. We then have to design an ST FEC code fulfilling the property that:
exchanging the position of bits at even and add positions in a run of bits and inverting one of the exchanged bits should also be a valid codeword. Again, in order to derive an ST coding scheme, we first assume that phase slips can only occur at positions 1, 1 + S, 1 + 2S, . . ., i.e., at multiples of a sub-block size S. A phase slip thus affects at least S consecutive QPSK symbols and we need to design a code that is agnostic to the event that in a run of at least 2S two neighboring bits swap positions and additionally one is inverted. We call such a code QST code. The interleaver Π in Fig. 5 is crucial in the construction and prepares the data for the class of codes that we define below. As in Definition 2, we sub-divide each codeword into n blocks of S bits each, i.e., x = (x1 , x2 , . . . , xn ). The interleaver Π converts a bit stream u = (u1, u2 , . . . , uk ) into u˜ = (u1 , u3 , . . . , u2S −1 , u2 , u4 , . . . , u2S , u2S +1 ,. . ., u4S −1 , u2S +2 , . . . , u4S , . . .) = (u˜1 , u˜2 , . . . , u˜k ), i.e., separates neighboring bits into adjacent blocks where the sub-blocks u˜2i−1 contain bits from the odd positions in u and the sub-blocks u˜2i contain bits from the even positions in u. The interleaver Π is defined in the same way as the interleaver Π with the exception that it operates on a block of n coded bits instead of on a block of k information bits. Due to the deinterleaving Π−1 at the transmitter, this means that each QPSK symbol is constructed using bits b1 and b2 where bit b1 is from a code sub-block x2i−1 and bit b2 from the neighboring sub-block x2i. In order for the code to be ST, we need a code that is agnostic to both the inversion of groups of bits (as in Definition 2) and the exchange of neighboring blocks. Such a code has to be designed according to the following definition.
