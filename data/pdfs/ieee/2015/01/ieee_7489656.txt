Abstract—This paper presents a new class of Quasi-Cyclic One Step Majority logic codes of 1/2 rate constructed from perfect difference set. Theses codes can be encoded with low complexity, and perform very well when decoded with the Iterative threshold decoding algorithm. Much of these codes is a subfamily of the LDPC codes and can be decoded using belief propagation algorithm. A comparison between our results and those for LDPC code in terms of BER performance are presented.
Index Terms—Iterative threshold decoding; Quasi-Cyclic codes; OSMLD codes; Majority logic decoding
I. INTRODUCTION
One of the simplest decoder is Majority logic decoder (MLD). It is a simple and powerful decoder, able to correct several errors depending on the number of parity check equations. MLD codes are suitable for storage, transmission applications and recently for fault memory detection [1] due to their ability to detect and correct a large number of errors. Nevertheless, to find constructive families of those codes has been recognized to be a hard job.
The first MLD algorithm was elaborated by Reed [2], for decoding the Reed-Muller codes [3], it’s a hard in hard out (HIHO) decoder, and the second one which is developed by Massey [4] is called the threshold decoder, a soft in hard out (SIHO) decoder. Finally, the third which is called the iterative threshold decoder (ITD) and developed by Belkasmi and al [5] is a soft in soft out (SISO) decoder, it’s based on the soft out of Massey decoder.
For its higher capability to detect errors among error correcting codes, the quasi cyclic Low density parity check (QCLDPC) codes are the most preferred codes in the last years, we can find application in large number of telecommunication system like DVB-S2, LTE , WiMax and also in storage system. QC-LDPC codes have been selected to increase the performance of the decoders to detect and correct large number of errors. Combining QC-LDPC codes with MLD algorithm, can provide a good performance with low complexity.
In this paper, we present a new adaptation of ITD algorithm for decoding QC-OSMLD codes. The codes presented are based on Singer perfect difference set. These codes were presented first by Townsend and Weldon [6]. The majority of these codes are a part of LDPC codes, which allow us to

decode them with LDPC decoder like Belief propagation (BP), sum-product.
The organization of the paper is as follows. In Section 2, we start with a description of the QC-OSMLD codes, then we describe the construction method. In Section 3 we define the encoding and the decoding algorithm. Section 4 is dedicated to simulation results and analysis of the ITD algorithm for decoding the constructed codes.

II. QUASI-CYCLIC OSMLD CODES

A. Perfect Difference Set

A difference set of order S and modulo m ≥ S(S - 1)+1 is defined as a collection of S integer specified from the set {0,1,...,m-1} such that no two of the S(S - 1) ordered differences modulo m are identical. If m=S(S - 1)+1, then for any non-zero integer n < m there is exactly one pair of elements in the difference set such that their difference is congruent to n modulo m. Such a set is called a perfect difference set.
Singer [7] has demonstrated how to construct such sets when S = pu + 1 and p is prime. Points and lines in the projective geometry PG(m - 1, q) form a difference set of parameters [q,m]. III. ENCODING AND DECODING OF QC-OSMLD CODES A. Encoding QC-OSMLD Codes

In the case of QC codes, the encoding can be achieved with simple shift registers, while the complexity is linear. Because the quasi-cyclic code is not in systematic form, an

additional k-stage register is required by this encoder to store the information symbols of the next block until encoding is completed. This difficulty can be avoided by using an equivalent systematic code. In our case the codes constructed are in systematic form.

It consists of cyclic shift register, XOR matrix, majority gate and XOR for correcting the codeword bit under decoding.

Iterative decoding process (see Figure 1) can be described as follows:
In the first iteration, the decoder only uses the channel output as input, and generates extrinsic information for each symbol. In subsequent iterations, a combination of extrinsic information and channel output is used as input.

As shown in Figure 1, the soft input and the soft output of the qth decoder is achieved through the following equations:

R(q) = R + α(q)LE(q)

Where R(q) represent lines (or columns) of the received data, LE(q) is the extrinsic information computed by the previous component decoder. In our procedure we use a fixed value 1/J for the parameter α(q) and this for all iterations. The value chosen for α(q) reacts as an average of all J estimators which contribute in the computation of LEj .

V. CONCLUSION
In this paper, we have investigated the construction of a class of QC-OSMLD codes, and also the performances of decoding these codes with the ITD algorithm. The simulations results shows that the constructed codes perform well when decoded with ITD algorithm. Simulation results shows that we are at 0.2 dB from the LDPC code which is applied to WiMax. The results obtained join those presented in [4,5,7] to prove that the ITD algorithm has good compromise between performance and complexity. As perspective, we propose to extend the construction to cover QC-OSMLD codes with rate different than 1/2, and to study their performances when decoded with ITD algorithm.
