Spatially Coupled Soft-Decision Error Correction for Future Lightwave Systems
Laurent Schmalen, Vahid Aref, Junho Cho, Detlef Suikat, Detlef Ro¨sener, and Andreas Leven
(Invited Paper)

Abstract—In this paper, we discuss and present some recent advances in the field of error correcting codes and discuss their applicability for lightwave transmission systems. We introduce several classes of spatially coupled codes and discuss several design options for spatially coupled codes and show how rapidly decodable codes can be constructed by careful selection of the degree distribution. We confirm the good performance of some spatially coupled codes at very low bit error rates using an FPGA-based simulation. Finally, we compare all proposed schemes and show how spatially coupled Low-Density Parity-Check (LDPC) codes outperform conventional LDPC and polar codes with similar receiver complexity and memory requirements.
Index Terms—Error correction codes, iterative decoding, lowdensity parity-check codes, spatial coupling.
I. INTRODUCTION
S INCE the introduction of Forward Error Correction (FEC) in optical communications, coding technology has evolved significantly. Modern high-speed optical communication systems require high-performing FEC engines that support throughputs of 100 Gbit/s or multiples thereof, that have low power consumption, that realize Net Coding Gains (NCGs) close to the theoretical limits at a target Bit Error Rate (BER) of 10−15 , and that are preferably adapted to the peculiarities of the optical channel.
Especially with the advent of coherent transmission schemes and the utilization of high resolution Analog-to-Digital Converters (ADCs), soft-decision decoding—which uses the samples computed by the Digital Signal Processor (DSP) and the statistics thereof—has become an attractive means of reliably increasing the transmission reach of lightwave systems. Currently, there are two popular classes of codes for soft-decision decoding that are attractive for implementation in optical receivers at decoding throughputs of 100 Gbit/s and above: Low-

II. LDPC AND SPATIALLY COUPLED LDPC CODES
In this section, we first introduce the general class of LDPC codes. Later, we introduce the specialized class of SC-LDPC codes, which have some appealing properties that makes them interesting for future optical communication systems.

A. LDPC Codes

An LDPC code is a relatively general concept and is defined

by the null space of a sparse parity-check matrix H of size

dim H = M × N and the code thus contains all the binary

code words x of length N such that HxT = 0, i.e., CLDPC = {x ∈ {0; 1}N : HxT = 0}. Sparse means that the number of

zeros in the parity-check matrix H is large compared with the

number of ones. Practical codes are built using parity-check

matrices that have a fraction of ones well below 1%.

Each row of H is denoted by check equation or check node,

while each column of H is usually denoted by variable or

variable node. We now introduce the notion of degrees. We say

that the variable degree (or variable node degree) of a code is

regular with degree dv if the number of “1”s in each column is constant and amounts dv . We say that the check degree (or

check node degree) of a code is regular with degree dc if the number of “1”s in each row of H is constant and amounts

to dc . A more general class of LDPC codes is the class of

irregular LDPC codes. These codes have the property that the

number of “1”s in each column and/or row is not constant but

may vary. The irregularity is then usually characterized by the

degree profile of the parity-check matrix. The degree profile

indicates the fraction of columns/rows of a certain degree. More

precisely, av,i represents the fraction of columns with i “1”s

LDPC codes, developed in the 1960 s by Gallager in his land-

mark Ph.D. thesis [13], form an important class of codes in

optical communications [14]. LDPC codes with soft-decision

decoding are currently being deployed in systems operating at

100 Gbit/s utilizing 16 [15] or 12 [16] decoding iterations, where

the latter also includes iterative differential decoding. Follow-

ing their invention, LDPC codes were not further investigated

for a long time due to the perceived complexity of long codes.

With the discovery of turbo codes in 1993 [17] and the sudden

interest in iteratively decodable codes, LDPC codes were rediscovered soon afterwards [18], [19] and rapidly found their way into various communication standards, such as, e.g., DVB-S2 [20] and 10 G Ethernet (IEEE 802.3) [21]. Note that there is no commonly accepted design rule for LDPC codes. The goal of the LDPC code designer is to find a code that yields high coding gains and which possesses some structure facilitating the implementation of the encoder and decoder. We point the interested reader to numerous articles published on this topic, e.g., [22], [23], [5, Ch. 7] and references therein.
Modern high-performance FEC systems are sometimes constructed using a soft-decision LDPC inner code which reduces the BER to a level of 10−3 to 10−6 and a hard-decision outer cleanup code which pushes the system BER to levels below 10−12 [24], [25]. An outer cleanup code is used to combat a phenomenon called error floor: above a certain signal-to-noise ratio, the BER does not drop rapidly anymore but follows a curve with a small slope [26], [27]. The implementation of a coding system with an outer cleanup code requires a thorough understanding of the LDPC code and a properly designed interleaver between the LDPC and the outer code for avoiding that the errors at the output of the LDPC decoder—which typically occur in clusters—cause uncorrectable blocks after outer decoding. Recently, there has been an interest to avoid the use of an outer cleanup code and to use only soft-decision LDPC codes with very low error floors. With increasing computational resources, it is now also feasible to evaluate very low target BERs of LDPC codes and optimize the codes to have very low error floors below the system’s target BER [28], [29].
Compared to hard-decision product codes, the internal data flow of an LDPC decoder may be increased by more than an order of magnitude [12]. This results also from the fact that the internal soft-decision messages are not binary, but q-ary. The internal dataflow may be reduced by several techniques, e.g., the use of rapidly converging layered decoding architecture [30]. If min-sum decoding is used, the number of bits per message may be reduced even further, as only two q-ary messages, dc + 1 binary messages and one log2 dc -ary message (i.e., 2q + dc + 1 + log2 dc bits) are needed per check node, compared to qdc bits per check node in a straight forward implementation [31]. It has been shown in several publications [15], [32], [16], that the implementation of LDPC decoders at 100 Gbit/s is feasible nowadays and it depends on the application if the additional implementational effort is justified by the increased NCGs.

In order to construct codes with large enough minimum distances (which directly relates to the error correction capabilities of the code), we have two possibilities:
1) We may maximize the size of the sub-matrices Hi, i.e., m, which has a quadratic influence on (2). In order to keep the complexity of the so-constructed code small, we restrict ourselves to small values of the syndrome former memory μ with either μ = 1 or μ = 2. 2) On the other hand, we may maximize the syndrome former memory μ and constrain the size of m (and thus n) to be small enough to realize efficient windowed decoders. We call such codes strongly coupled codes.
Of course, we may also realize codes with both large μ and large m. However, such codes require large decoding windows and are thus of limited practical interest. Such a small minimum distance will naturally lead to low-weight code words and thus an unacceptably high error floor. For this reason, we do not use B directly as a code, but rather as a protograph. This means that we replace each non-zero entry in B by an S × S circulant matrix P si,j , where P is the circulant permutation matrix with the entries PS,1 = P , +1 = 1 for 1 ≤ < S, and “0” otherwise (i.e., the identity matrix cyclically right-shifted by one position).
This lifting procedure allows to construct codes that potentially have a larger minimum distance and that can be efficiently decoded with a partially parallel layered decoder [30]. In order to construct powerful codes, we need to carefully select the shift values si,j , associated to the j’th column of the sub-protograph Bi. This construction does however not help in increasing the girth, as there are unavoidable cycles of length 6 that cannot be resolved [47], regardless of the chosen values of si,j . We may however extend the construction and use time-varying shift values si,j (t). As the protograph is kept time-invariant, the hardware design can be kept simple with a fixed wiring structure. The time-varying shift values only require programmable barrel shifter circuits in the decoder. In order to keep the complexity low, we employ periodic shifting, i.e., si,j (t) = si,j (t + T ) and aim at selecting the smallest period T such that unavoidable cycles of length ≤ g − 2 are eliminated, where g is the target girth of the final code. The shift values si,j (t) are obtained by enumerating all the cycles in the protograph and setting up an inequation for each cycle [48]. We solve the resulting system of inequations modulo S by a heuristic computer search.

decoding algorithm [30] and uses a scaled-minsum check node

computation rule.

The windowed decoder that is implemented can be sub-

divided into three steps. In the first step, a new sub-block of

n quantized LLRs is received from the random number gen-

erator and put into the vacant position of the decoder’s LLR

memory. In the second step, the actual decoding takes place

and the decoder carries out I layered decoding iterations on the

portion including wm rows of the parity-check matrix. The ef-

fective number of iterations, i.e., the number of updates of each

sub-block thus corresponds to w · I. For simplicity, we mainly

set I = 1 such that the window length corresponds roughly to

the number of iterations in a conventional decoder. The out-

put of the LDPC decoder is connected to the BER evaluation

unit, which counts the bit errors and reports the error positions.

A single Virtex-7 Field-Programmable Gate Array (FPGA) is

able to emulate a block LDPC code with I = 12 at a rate of

about 5 Gbit/s and an SC-LDPC code with μ = 2, w = 13 and

I = 1 at approximately 1.5 Gbit/s. This lower rate is mainly

due to increased memory requirements of the windowed de-

coder, compared with the block decoder, which only requires

a single a posteriori memory of size N , while the windowed

decoder requires wn memory blocks. Roughly 25 000 LUTs are

required to implement a single windowed decoder on the Xilinx

Virtex 7 FPGA. Note that in an ASIC implementation with an

iteration-unrolled decoder, these numbers will be approximately

equal.

The situation is somewhat reversed for the left-terminated SC codes. SC Code A, which has been optimized for convergence speed, shows a very early start of the waterfall, however, the slope of the BER curve starts to decrease slowly, such that the performance at low BERs is relatively bad and is expected to cross the BER curve of block code A. The situation is reversed for SC Code B, that exhibits a late start of the waterfall curve, which then drops exceptionally fast. We did not observe any error patterns for the next Eb/N0 point (3.08 dB) of the curve after simulation of 7.9 · 1014 bits, so by extrapolating the BER curve, we may conjecture that SC Code B has an NCGs of 11.91 dB at a post-FEC BER of 10−15 . As a comparison between block codes and convolutional-like codes is difficult, we constrain the codes and decoders to process always an approximately identical number of bits, leading to equal decoding latencies. VI. CONCLUSION
In this paper, we have highlighted Spatially Coupled (SC)LDPC codes as potential candidates for future lightwave transmission systems. We have introduced two classes of SCLDPC codes: strongly coupled and weakly coupled codes. For each class, we have given a construction rule and shown simulation results of the performance. For the class of weakly coupled codes, which seem to be more promising in optical communications due to their lower error floors, we have optimized a degree distribution and shown by means of an FPGA-based simulation that very low error rates can be obtained. Finally, we have compared the SC-LDPC codes with conventional LDPC codes and Polar codes and have shown that these codes can be outperformed with windowed decoders that have the same memory and computational complexity requirements.
