Abstract—This paper presents the mixed-signal circuit implementation of reduced complexity algorithms for decoding low-density parity check (LDPC) codes. Based on modified differential decoding using binary message passing (MDD-BMP), binary addition using discrete-time digital circuits is replaced by continuous-time analog-current summation. Potential degradation due to the mismatch between current sources, P/N strength mismatch and inverter-threshold mismatch is considered in behavioural simulation and shown to be tolerable. Area estimates suggest a reduction from 0.27 mm2 to 0.11 mm2 for the FG(273, 191) code. Finally, transistor level simulation of the FG(273, 191) code using TSMC 65 nm technology shows an efficiency of 0.56 pJ/bit.
Keywords—analog circuits; binary messages; energy efficient; iterative decoding; LDPC; VLSI
I. INTRODUCTION
Low-density parity check (LDPC) codes are state-of-the-art capacity-approaching error-correcting codes [1][2]. They have attracted much recent interest not only because of their excellent error-correcting capability, but also because of the suitability of their message-passing decoding algorithm for parallel high-throughput implementation. However, fullyparallel hardware architectures present challenges for efficient implementation due to complex processing hardware and routing congestion [3].
One approach is to use reduced-complexity binary message-passing algorithms such as modified differential decoding using binary message passing (MDD-BMP) [4] and its improved variant, known as IDB [5]. These algorithms reduce the processing and wiring complexity compared to conventional LDPC decoding algorithms such as offset minsum [6], and are very energy efficient [5]. MDD-BMP and IDB can also be used as a component of a hybrid “gear-shifting” decoder, which has the same error-correcting performance as state-of-the-art offset-min-sum decoders, but with reduced energy consumption [7].
In this paper, we explore mixed-signal implementation of the architecture described in [5] with the aim of reduced hardware complexity and energy-efficiency. We describe the analog-domain versions of MDD-BMP and IDB in Section II. Section III explores the effect of mismatch on the decoding

B. MDD-BMP in Analog
In the approach proposed in this paper, check-node operations remain as binary sign multiplication implemented with XOR gates. However, the complexity of the required circuits in the variable nodes can be reduced if the operation in (2) is ported to analog, as depicted in Fig. 2. The variable-node memory is replaced by a capacitor, possibly using only parasitic wiring capacitance. The currents flowing in or out of the inverters are the scaled check-node messages that are added together by connecting the inverter outputs together. Iterationbased accumulation in the digital implementation is replaced by continuous-time integration of current flowing into the capacitor, C. The sign of Mv is determined by comparing the voltage across the capacitor to the threshold of an inverter chain.

III. EFFECT OF MISMATCH ON ANALOG DECODING
The analog implementation of MDD-BMP is potentially degraded by four phenomena:
a) Finite output resistance of inverters. Output currents depend on the capacitor voltage. iinv = f(vC).
b) Systematic offset between PMOS and NMOS transitors in inverters causing differences in sinking and sourcing currents. f(vC) ≠ - f(0).
c) Random mismatch between inverters feeding variable nodes.
d) Mismatch between inverters used for sign within variable nodes relative to those used in check nodes.
Each of these phenomena was modeled and included in a discrete-time, continuous-valued model and simulated using Python using an FG(273, 191) code. The methodology for modeling each of these and their effect on decoding performance is discussed.
a) Finite output resistance of inverters. The current flowing from the variable node inverters into the integration capacitor C depends on the binary input of the inverter and the voltage across the capacitor. For an input voltage of 0 V and a capacitor voltage near VDD/2, the inverter’s NMOS transistor is off and the PMOS transistor is in saturation. For larger capacitor voltages, the PMOS transistor will enter triode, reducing the output current. The current output of an inverter is summarized in Fig. 3. This shows the normalized current output of the inverter for each of two possible inputs to the inverter. To avoid using complex transistor models in behavioural simulations a single 4th order polynomial mapping VN state to message weight was used. This was found to have close agreement with the simulated transistor behaviour. IDS vs VGS behaviour need not be included in behavioural models since the input to each inverter is a binary-valued signal. The use of this model had no discernible effect on decoding performance.
b) Systemic skew between PMOS/NMOS transistors. At design time, the ratio of PMOS and NMOS transistor widths can be selected so that the inverter currents are equal

Instead, we propose to use the capacitance of a long wire as the storage element in the variable node, meaning that it is the Mv state that is broadcast. The effects of b), c) and d) are shown separately in the first three curves with the x-axis representing normalized difference between PMOS/NMOS currents, relative standard deviation in current mismatch, and standard deviation in inverter thresholds in LLR, respectively. The line denoted by “Expected mismatch” shows BER as a function of P/N skew when inverters have 5% relative standard deviation in current mismatch and threshold mismatch with a standard deviation of 0.26. These values correspond to the anticipated level of mismatch in the chosen technology. The horizontal line denoted by “Quantized digital” shows the BER achieved in [5] when LLR values are quantized using 6 bits. There is only slight BER performance degradation with the “Expected” level of non-idealities. With ideal or almost-ideal behaviour, the BER performance of the mixed-signal implementation is better than the digital version since there is no quantization error in the mixed-signal implementation.

V. CONCLUSION
A mixed-signal implementation of MDD-BMP was presented. By replacing the variable node’s discretetime/discrete-valued summation and accumulation with continuous-time analog processing, the circuit area and energy per bit can be reduced. Behavioural simulations show that the anticipated variability in the analog behaviour of the required circuits does not degrade decoding performance. Initial transistor-level simulations point to smaller area and lower energy per bit than the digital implementation of the algorithm.

The capacitors are set using ideal voltage sources connected through pass-transistors. During this phase, the inverters that drive the capacitors are disconnected. During this interval we can see the range of initial LLR values. At 7 ns, the voltage sources are disconnected and the inverters are connected. The transition of bit errors across the VDD/2 threshold along with the check-node messages in Fig. 6 indicates the correction of bit errors. 0.7 ns was a typical decoding time.
The average current of 76 mA from a 1 V supply was computed when 2 ns was used to program variable node memories and complete decoding. This corresponds to a throughput of 500 x 106 codewords per second x 273 bits per codeword = 137 Gb/s. This corresponds to an energy efficiency of 0.56 pJ/bit. This power dissipation was simulated for a variable node capacitance of 300 fF. The potential for further energy reduction by reducing the accumulation capacitance is being explored.
