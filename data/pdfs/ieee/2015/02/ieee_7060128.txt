Abstract --- Irregular Low-Density Parity-Check (LDPC) codes exhibit a performance extremely close to the Shannon capacity formula. In this paper, we presented a method of developing structured irregular LDPC codes from structured regular LDPC codes. The design is based on the expansion of the column-weight two LDPC codes. We used our proposed technique of developing a column-weight two structured regular LDPC codes based on Gray code representations and adapted the expansion method proposed in literature to convert regular codes into irregular ones, and it is noticed that there is an improvement in obtained code rates using the proposed technique. It is also found that encoding process is simpler and requires less memory.
Key Words–LDPC code, Regular and Irregular LDPC codes, Column-weight two LDPC codes, parity-check matrix, code rate
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were proposed by Gallager in the early 1960s [1]. LDPC code, specified by sparse parity-check matrix ‘H’, is said to be regular if its column and row weights are constant throughout the columns and rows respectively, otherwise, it is said to be irregular. The irregular codes outperform the regular ones. Luby et al. [2] formally showed that properly constructed irregular LDPC codes can approach capacity more closely than regular ones. The tradeoff between threshold signal-to-noise ratio (SNR) and error-floor bit-error rate (BER) has been observed for both turbo codes [3], [4] and LDPC codes [5]. At block size, rate1/2 regular LDPC codes with variable-node degree larger than two [6] enjoy very low error floors; however, their threshold SNR is 0.6 dB worse than their irregular counterparts [7].
Some of the methods for generating regular and irregular LDPC codes are discussed in [8], [9]. The performance of the LDPC codes depends on the column weight ‘wc’, of its paritycheck matrix. Column weight of a parity-check matrix is the number of ones in each of its column. The large column weight codes show better performance in terms of BER. The codes designed for higher column weights wc ≥ 3 have complex designs and hard to implement. On the other side, the codes designed for column weight wc = 2 are less complex to implement. These types of codes can be directly constructed from Regular Graphs [10], [11]. This type of construction is given by Malema [12]. The incidence matrix of the graphs can be used as the parity-check matrix for the LDPC Codes. These types of codes have an advantage of high girth but simultaneously having a limitation of fixed column weight 2. The approximate cycle extrinsic message degree constrained design (ACE) algorithm [13] to construct irregular LDPC codes that have a specific ACE property for short cycles, while maintaining the density-evolution-prescribed degree

distribution along with the effect of graph connectivity on error floors is introduced.
In [14], a construction method has been proposed to design an irregular LDPC codes by increasing the column weight of the parity-check matrix and its concatenation with an Identity matrix. The proposed extension method converts the low column weight regular LDPC codes into an irregular one. The proposed construction also simplifies the encoder design and requires less memory for encoding. We presented a novel method for construction of structured regular LDPC codes with girth twelve using Gray code representations in [15], discussed briefly in section II. In this paper, we adapted the technique of LDPC code expansion proposed in [15], with a small modification and noticed that the code rates improved.
The paper is divided into six sections. The introduction is given in section I. The proposed construction method is briefed in section II. Section III describes the method of generating irregular LDPC codes from regular column-weight two LDPC codes, presented in [15], by using our method proposed in[14], with a small modification. Section IV presents the encoding method of these codes. The comparison of obtained code rates between the standard methods provided in [14], and the proposed method is tabulated in section V. Section VI concludes the paper, followed by references.
II. PROPOSED CONSTRUCTION ALGORITHM
In [14], a method of generating H-matrix of irregular LDPC codes from H-matrix of regular column-weight two LDPC codes is presented. The required regular column-weight two H-matrix is generated from standard methods such as the unextended LDPC code based on Hoffman-singleton graph, a typical Gallager code and High rate Gallager code, and a typical LDPC code generated from Progressive Edge Growth. We proposed a novel method for constructing regular columnweight two LDPC codes, with flexibility in length, density, code rate and girth, using Gray code representations in [15]. The regular column-weight two LDPC code generated by algorithm in [15], is used to generate an irregular LDPC code using the expansion method described in [14]. Our previous work of construction of structured regular column-weight two LDPC codes using Gray code representation is briefed in few steps below:

The number of bits used to represent each decimal number into its corresponding Gray code value, provides the flexibility in code length. The decimal numbers selected are such that, when converted into their Gray code representations, they have more number of 0’s compared to number of 1’s, so that the generated H-matrix is sparse. The proposed algorithm gives the systematic way of selecting the decimal numbers using one of the design equations. It also gives the structured way of constructing the sub-matrices as explained above. Also, it provides flexibility in code length, as described below.

C. Proposed method of code expansion:
Another method of code expansion, keeping the regularity of the graph unchanged, is to include equal number of zeros after each decimal value in the selected set. For example, if a single zero is included, after each integer, the code length doubles. Two zeros increase the code length by three times and so on. Thus ‘p’ number of zeros increases the code length by (p+1) times its original length.

III. PROPOSED CONSTRUCTION METHOD
In [14], the construction method of irregular LDPC codes from column-weight two regular LDPC codes is proposed. The extension method presented in the paper provides the method of converting low column-weight regular LDPC codes into higher column-weight irregular LDPC codes. Columnweight two LDPC codes have simpler design and easy to implement. Typical codes available in literature such as Hoffman-singleton graph based LDPC codes, a typical Gallagar code and High rate Gallagar code, and LDPC code generated from Progressive Edge Growth, are converted into irregular codes and extended to different levels of expansions in[14]. Their code rates with different levels of expansions are compared with their corresponding unextended codes and are tabulated.
In this paper, we adapted the method of expansion proposed in [14], with a small modification. The structured regular column-weight two LDPC matrix, obtained from our previous algorithm, is converted into irregular one. By performing expansions in different levels, we noticed that the code rates are improved.
