Non Iterative LDPC Decoding By Syndrome Generation Using Artificial Neural Network

Dinesh Kumar, Alkesh Agrawal, and Gopal Singh Phartiyal
Department of Electronics and Communication Enginnering, SRMU,Lucknow, Uttar Pradesh, India
Idinesh.srmuI2@gmail.com, 2alkesh.ec@srmu.ac.in, 3gsp2188@gmail.com

Abstract - Low density parity-check code (LDPC) is an error correcting code used in noisy communication channel (e.g. AWGN) to reduce the probability of error in information. By using LDPC codes, this probability can be made comparatively small, so that the data transmission rate can be as close to Shannon's limit. The decoding of Low Density Parity Check (LDPC) codes by iterative process of belief propagation gives challenges for designers looking for real time performance in communication systems. This thesis work proposes the use of Artificial Neural Networks (ANN) to replace belief propagation to approach closer to Shannon's limit more closer than other traditional decoding methods. This thesis is intended to design a new methodology to decode LDPC codes in Non-iterative manner with the help of ANN and Look Up Table (LUT). This work is at initial stage and will be extended for better performance.
Keywords - Artificial Neural Networks; LDPC codes; Belief Propagation Algorithm; Shannon's Limit; A WGN Channel;
[NTRODUCTION
Low density parity check codes are a special type of error correcting codes that is known for their good decoding performance and high throughput. Low density parity check codes were first introduced by Robert Gallager in 1962 [1, 2]. His work remained neglected for decades because of its high computational complexity for hardware implementation in that time and lack of good decoding techniques. R. Tanner did an exceptional work by introducing Tanner Graph to represent LDPC codes in [98 [ [3] [n 1993, LDPC codes were re­ discovered by D. MacKay and M. Neal [4], and showed that the empirical performance can approach to Shannon limit [5] as well as Turbo codes or even better. The work of MacKay was extended by Luby [6] to show that irregular LDPC codes are capable of exceeding the performance of regular LDPC codes, and he also introduced approaches to design irregular codes. Richardson and Urbanke [7] extended that work to soft decision message passing decoding and introduced a method called density evolution.
The idea of using Artificial Neural Network for error correction is not new. Ja-Ling Wu, Yuen-Hsien Tseng and Yuh-Ming Hunag [ 8] proposed the use of ANN for error

correction by training the neural network for all 2k number of input patterns (k = no. of message bits) and that too with a special type of neural network designed called Polynomial Neural Network. This method works efficiently but only for small codes as long codes will need more memory in computer and simultaneously the decoding neural network size and complexity will increase.A. R. Karami, M. Ahmadian Attari and H. Tavakoli [9] the neural network complexity was decreased by employing Multilayer Perceptron, a feed forward class of neural network. But still for a long code 2k number of training patterns (k = number of message bits) are needed to train the network that increases the size. For a message of bits 210 number of training patterns are needed that increases the size as well as high memory is required to save all patterns. Constantin Anton, Laurentiu [onescu, [on Tutanescu, Alin Mazare and Gheorghe Serban [10] proposed Hopfield Network type Associate Memory which is a recurrent neural
network. This method reduces the network size but still due to
its recurrent property (feedback) it takes times to be stable and hence cannot be used in high speed LDPC codes.

This project work proposes a different method by using ANN to make the decoding process non-iterative. By inspection it has been seen that for error in two or more bits in a code word the syndrome generated by parity equations is the Ex-OR operation of same columns defined by the positions of bits having error. [n this method the syndrome is generated by a well trained neural network and the error is detected and corrected using Look Up Table (LUT) defined for each syndrome. The error correcting capability depends on the number of parity bits used. The full implementation is explained in later sessions.

A. Representation ofLDPC Codes
LDPC codes are represented by a bipartite graph called Tanner Graph [3], the term bipartite refers to a set of nodes partitioned into two parts in such a manner that all edges have a vertex in the first part and another one in the second, and no edges connect nodes within the same part. The two parts are called check nodes representing rows, and variable nodes representing columns of the LDPC parity check matrix. We have tested this methodology for (12, 6) codes where 12 is number of code words bits and 6 is number of message bits hence number of parity bits are 12 - 6 = 6. For generating the code-words we have used and irregular parity check matrix as shown in Fig. 2. The encoding is done in usual manner by first

Syndrome is generated using multilayer perceptron (or RBFN) [12]. The size of the MLP depends on parity check equations. If the H matrix is regular type then the number of inputs in MLP would be the number of Is in rows. If the H matrix is irregular then the number of neural networks would the number of unique number of 1s in all rows. For instance in our case we have 4 rows with 4 1s and 2 rows with 5 1s. Hence we used 2 MLPs, one with 4 inputs and other with 5 inputs. Now we have to train each network with 4 and 5 bits patterns that solves the problem of training network for all code words that takes large size of neural network as the number of bits in code words increase.

For example if the errors are in bit positions 3 and 5 then the syndrome generated is the Ex-OR of columns 3 and 5 of H matrix. Now the error correcting capability of this decoder depends on the number of parity bits. If the number of parity bits are 6 (as in our case) then total syndromes generated will be 26 i.e. 64. Hence to correct single bit error number of syndromes would be 12C1 i.e. 12. To correct 2 bit errors number of syndromes would be 12C2 i.e. 66. Now total syndromes to correct errors up to 2 bits would be 66 + 12 = 78. Since there are total 64 syndromes possible using 6 parity bits the extra 14 syndromes are the copies of some of the syndromes in the syndrome vector. Now if we increase the number of parity bits to 12 the total number of syndromes that can be generated would be 212 i.e. 4096 and total number of bits that can be corrected would be 18C1 + 18C2 +18C3 +18C4 = 4047 and 4096 - 4047 = 49 syndromes for errors in 5 bits.
The number of output neurons in MLP is taken only one. The MLP is trained in such a manner that it gives output '0' for input patterns with even parity and '1' for odd parity. The code-words received are first arranged in the shape of H matrix so as to follow parity equations. Then the inputs are applied to MLPs according to the number of terms in each equation. To generate the full syndrome from both the MLPs the results are concatenated serially in accordance with the inputs applied from the corresponding rows of H matrix.

B. Look Up Table Generation
The syndromes for each number of error bits is calculated in advance and stored in look up table. The storage pattern of the LUT is divided into two parts the left part has syndrome calculated in advance and right part has the position of the columns for which the syndrome is calculated. Since each position of each column of H matrix corresponds to position of the code word bits. The left most column corresponds to MSB of the code word and right most column corresponds to LSB of the code word and similarly for other columns and bits. CONCLUSIONS

We have proposed a new method of decoding for LOPC codes

based on neural networks. In this decoder there is no need to

calculate probabilities. It performs a good result with

decreased decoding complexity. It is a simple prototype neural

decoder and we are working to improve its performance. We

are trying to remove the problem of generation of syndromes

for error in 2 or more bits that are same for 1 bit. We will be

trying to generate syndrome in such a manner that it

automatically detect and correct the error without using LUT

so that the latency created by comparing the syndrome can be

removed.
