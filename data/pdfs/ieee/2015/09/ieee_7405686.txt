(7, 4)-Hamming Generalized LDPC Code for Rayleigh Fading Channel

Yangwen Yu 1, Yanan Han 1, Lijun Zhang1*, Lee Lung Cheng 2
1 School of Electr. and Inform. Eng., Beijing Jiaotong Univ, Beijing, 100044, China 2 Dept. of Electronic Engineering, City University of Hong Kong, Hong Kong, China
ljzhang@bjtu.edu.cn; itacheng@cityu.edu.hk

Abstract—To get a generalized version of LDPC code, we construct (7, 4)-Hamming GLDPC code with (7, 4) Hamming component code in this paper, and give the number of short cycles of this GLDPC code as well as the code weight distribution. Via MLD, the negative effects of short cycles limited in the Hamming block can be eliminated. Simulation results show that, up to 2 dB coding gain can be obtained for GLDPC code, with faster convergence rate, in comparison with the counterpart standard code, at BER of 10-7 in Rayleigh fading channel. Even compared with the standard LDPC code which has no 4-cycles, the better behaviors for GLDPC code, with still faster convergence rate, are achieved in several limits. These results may enhance the implementation of this GLDPC code in real world.
Keywords-Rayleigh fading channel; (7, 4)-Hamming GLDPC code; 4-cycle; iterative decoding

I.

INTRODUCTION

Low-density parity-check (LDPC) codes were introduced by Gallager [1] and rediscovered by MacKay and Neal [2]. These codes are shown to achieve closer Shannon theoretical limit capacity on some kind of memoryless symmetric channels such as erasure channels, additive white Gaussian noise (AWGN), etc. Based on Pearl’s belief propagation algorithm, both efficiency and performance of the LDPC codes’ decoder have been greatly improved in the past years [2]. However, the decoders of these codes are confined to a serial decoder of single parity check (SPC) codes and repetition (REP) codes. In order to further improve the efficiency of this kind of decoders, other combination of codes is feasible [3, 4]. In other words, a consideration for the generalized version of LDPC is significant.

Tanner was the first one to propose this idea of generalized low-density parity-check (GLDPC) codes [5]. The GLDPC codes, where the component codes are Hamming codes which can correct single error, are used instead of SPC codes, were originally proposed by M. Lentmaier in 1999 [6]. SPC codes just detect single errors. By contrast, Hamming codes can detect two errors or correct single error. Hence, as the constraints of check nodes for LDPC codes, Hamming codes would have more rigorous constraints than SPC codes evidently. A lower bound of the minimum Hamming distance for this kind of GLDPC codes was given [6], and there existed an example where the Hamming GLDPC codes outperform

Using density evolution (DE) or its expanded theoretical technique i.e. extrinsic information transfer (EXIT) chart, the superiority of Hamming GLDPC codes compared with standard LDPC codes in binary erasure channel (BEC) would be predicted feasibly and these theoretical analysis which we have shown in our previous work.
It is well known that, the majority of existing researches on GLDPC codes are theoretical analysis [7, 8]. In these analysis, just like DE [9] or EXIT chart, the length of code has to be infinite and there are also some other ideal assumptions. Under these ideal conditions, good asymptotic performance of this kind of codes can be obtained. Nevertheless, the idea of GLDPC codes has not been used in practical application. Although there exists some simulation results of this kind of GLDPC codes for which the component codes are Hamming codes on AWGN channel [10, 11], the more complex fading channel model, which is closer to the actual communication channel, has not been considered. In addition to additive white Gaussian noise, there exists multiplicative noise and the environment of simulation is more realistic. Rayleigh fading channel, which bridges the gap between ideal theoretical analysis and practical applications, is more complex than AWGN channel. Hence, it is significant to study the performance of GLDPC codes in Rayleigh fading channel for promoting these channel codes’ practical applications.
In addition, the constructions of these kinds of Hamming GLDPC codes have not been systematized. With the consideration of complexity, (7, 4) Hamming code is used as the component codes of check nodes in our work. Certainly, other kinds of Hamming codes can be used as the component codes. But this kind of Hamming code has a splendid application history. For (7, 4) Hamming code, the complexity and efficiency are all balanceable, so this code is suitable for the actual hardware implementations. If longer Hamming codes are used, the iterative decoder would become involved. Therefore in our construction of Hamming GLDPC codes, (7, 4) Hamming code is a good choice. For instance, here we use PEG method to construct an initialized standard LDPC code for designing (7, 4)-Hamming GLDPC code. Of course, there are various methods besides PEG to construct standard LDPC codes, but it is not the key point of this paper.

II. (7, 4)-HAMMING GLDPC CODES
As to this kind of GLDPC codes, the variable nodes still remain the constraints of REP codes. Hamming codes would be used as the constraints for check nodes instead of SPC codes. The method adopted to construct Hamming-GLDPC codes is to replace the positions of nonzero elements of each row in a standard LDPC code’s check matrix H, with a Hamming check matrix which has the same size as the number of nonzero elements in each row of the initialized standard LDPC code.
Here, (7, 4) Hamming code is considered to construct the Hamming-GLDPC code. As an effective method, PEG is used to build a standard LDPC code. Firstly, we construct a standard LDPC check matrix, which has 42 rows and 147 columns. There are 7 nonzero elements each row which can be replaced by the check matrix of (7, 4) Hamming code. These rows replaced by Hamming code can constitute a plenty of Hamming blocks. The zero element in each row is translated into an all-zero column vector, which has consistent size with the rows of our (7, 4) Hamming check matrix. Hence, a Hamming-GLDPC code matrix has been constructed, which has 42u 3 rows and 147 columns. The null space of the matrix gives a (147, 21) Hamming GLDPC code.
We adopt the way of defining standard LDPC to denote our (7, 4)-Hamming GLDPC code. In an (N, M, K, n, m, k) Hamming-GLDPC code, the parameter N stands for the length of this Hamming-GLDPC code, which also represents the number of columns in the corresponding check matrix; the parameter M stands for the number of rows in matrix; the parameter K stands for the number of information bits. Similarly, n, m and k refer to the corresponding component parameters of Hamming code. Notice that M E  m , where E
is an integer. Hence, our (7, 4)-Hamming GLDPC code would be marked as (147, 126, 21, 7, 3, 4) Hamming-GLDPC code.
For the (147, 126, 21, 7, 3, 4) Hamming-GLDPC code, the check matrix of this code can also be regarded as an ordinary check matrix for a standard LDPC code. Using this check matrix, all kinds of decoding algorithms for standard LDPC code can also be implemented. In the forthcoming section IV, the performance of this standard LDPC code with the same matrix as our (147, 126, 21, 7, 3, 4) Hamming-GLDPC code would be illustrated for comparison.

III. DECODING ALGORITHM OF (7, 4)-HAMMING GLDPC CODE FOR RAYLEIGH FADING CHANNEL
Rayleigh fading channel is more complex than other common binary symmetrical channels such as BEC or AWGN channel. Rayleigh fading is a statistical model which assumes that the magnitude of a signal that has passed through such a communications channel will vary randomly, or fade, obeying a Rayleigh distribution. Rayleigh fading channel model is viewed as a reasonable model for tropospheric and ionospheric signal propagation as well as the effect of heavily built-up urban environments on radio signals [13, 14] .Rayleigh fading is most applicable when there is no dominant propagation along a line of sight between the transmitter and receiver. So compared with symmetrical channels, Rayleigh fading channel is more pragmatic in mobile wireless communications.

V. CONCLUSION
GLDPC codes with (7, 4) Hamming code as component code has been designed. The short cycles and code weight distribution of the GLDPC code are provided. The minimum Hamming distance for this code is 35 and most of codewords have big weights. Simulation results show that (7, 4)Hamming GLDPC code outperforms the counterpart standard LDPC. Even compared with standard LDPC code without 4 cycles, the Hamming-GLDPC code still has powerful behavior within iteration of 5 or less. These properties would propel the pace of the practical application for our (7, 4)-Hamming GLDPC code.

The computational complexity of GLDPC decoding algorithm displayed in this paper is much higher than that of LDPC decoding algorithm. In the future work, the generalized effective decoding algorithms will be devised to improve the GLDPC decoding efficiency.
Moreover, it is an interesting result of our research that the construction of component codes would be effective to resolve the decoding blemish of standard LDPC caused by the short cycles which in the component block. This is worth in-depth studying.
