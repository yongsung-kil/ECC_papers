Efficient Stochastic List Successive Cancellation Decoder for Polar Codes
Xiao Liang1, Chuan Zhang1, Menghui Xu1, Shunqing Zhang2, and Xiaohu You1 1National Mobile Communications Research Laboratory, Southeast University, Nanjing, China 2Intel Collaborative Research Institutes on Mobile Networking and Computing, Intel Labs, Shanghai, China

Abstract‚ÄĒRepresenting continuous values by streams of random binary bits, stochastic decoding has shown advantages in both hardware efficiency and fault tolerance, therefore has been widely adopted by iterative decoding of error correction codes such as low-density parity-check (LDPC) codes and so on. Recently, polar codes, the first codes that can provably achieve the capacity of symmetric binary-input discrete memoryless channels (B-DMCs), have drawn a lot of attentions from both academia and industry. Although, polar codes with list successive cancellation (SC) decoding can outperform several best-known LDPC codes even within high error-rate regions, the linearly increasing hardware complexity makes its efficient implementation difficult. To this end, the stochastic list SC polar decoding algorithm is proposed in this paper to provide a good tradeoff between performance and complexity. In order to increase the decoding performance of stochastic list SC polar decoder, doubling probability approach is presented. The corresponding hardware architecture is also given. The approximate doubling approach is employed to facilitate the efficient implementation. Implementation results have shown that the proposed stochastic list SC polar decoder can achieve a good trade-off between performance and complexity.
Index Terms‚ÄĒPolar code, stochastic decoding, list SC decoder, low latency.
I. INTRODUCTION
Proposed by ArńĪkan‚Äôs breakthrough paper [1], polar codes have been proved to be the first class of channel codes that can provably achieve the capacity of symmetric binaryinput discrete memoryless channels (B-DMCs). Due to its low computational complexity of O(N log N ) and FFT-like structure, successive cancellation (SC) algorithm has become one of the most favorable polar decoding algorithms. However, it is admitted that the decoding performance of SC polar decoder still suffers from a significant degradation compared to that of the maximum likelihood (ML) decoder [2]. In order to narrow the performance gap introduced by the suboptimality of conventional SC decoder, the so-called list SC polar decoding algorithm is therefore proposed in [2, 3]. Simulation results have shown that list SC polar decoder can outperform several best-known LDPC codes even within high error-rate regions.
The main disadvantage of list SC polar decoder is that its complexity increases linearly with the list size L. In other words, in order to achieve better performance, the list SC polar decoder must suffer from prohibitive complexity cost, especially when L is considerable large. Furthermore,

for certain highly noisy application scenarios, existing list SC polar decoders are not robust enough against noise. In order to address above issues, the stochastic list SC polar decoding algorithm is proposed in this paper. To the best of our knowledge, this is the first literature which incorporates stochastic decoding with list polar decoder. Benefitting from the advantages of stochastic decoding, the stochastic list SC decoder is expected to be a paradigm that achieves good performance, complexity reduction and fault tolerance. However, directly applying stochastic decoding to list SC decoder will result in unacceptable performance degradation. In this paper, doubling probability approach and its approximated version are introduced to improve the decoding performance. The corresponding hardware architecture is also proposed, which shows advantages in both performance and complexity.
The remainder of this paper is organized as follows. Section II re-explains the stochastic list SC decoding algorithm with L = 1, which is first proposed by [4]. Section III presents the stochastic list SC decoding with L = 2. The implementation of stochastic list SC decoder is proposed in Section IV. Comparison and conclusion are given in Section V.

III. STOCHASTIC LIST SC DECODING WITH L=2
In this section, the stochastic list SC decoding algorithm with L ‚Č• 2 is proposed. Since the proposed algorithm is general, the stochastic list SC decoding with L = 2 is employed as a running example. Other cases with L > 2 can be simply derived in similar fashion.
A. Deterministic List SC Decoding
List SC decoding algorithm was first proposed by [2] and [3], independently. It can be considered as a generalization of the SC decoding algorithm proposed by ArńĪkan in [1]. The basic idea of list SC decoding algorithm is similar to that of K-best MIMO detection. For conventional SC decoding, only the most likely code bit is picked up at each decoding stage. Whereas the list SC decoder always keeps a list of L code bits at each step, and outputs the most likely codeword when the final step is finished. In other words, list SC decoding improves the performance at the penalty of linearly increasing hardware. For better understanding of the list SC decoding algorithm, Fig. 3 illustrates the decoding steps of list SC algorithm with L = 2.

Since the complexity of list SC decoding is O(LN log N ) and N ‚Č• 210 for real-life applications, large L will certainly result in prohibitive complexity which hinders the efficient implementation of list SC decoder. One possible solution is to implement the algorithm with stochastic decoding, which can greatly reduce the complexity of elementary processing nodes compared to the deterministic one.

C. Numerical Simulation and Analysis
In order to properly evaluate the proposed stochastic list SC decoder. Numerical simulation is carried out for both 1/2rate (64, 32) code and 1/2-rate (256, 128) code. The scaling factor őĪ = 0.5, bit stream length l = 1024. It is expected that the stochastic list SC decoder with L = 2 will outperform the deterministic SC decoder. Unfortunately, simulation results show the opposite. For deterministic list SC decoder, larger L always brings better performance. However, for the proposed stochastic list SC decoder, decoder with larger L does not show any advantages but even worse decoder performance. This performance degradation becomes more prominent when the code length L increases.
