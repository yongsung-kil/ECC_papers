Research on Performance of Visible Light Communication Based on LDPC Code

Yishuo Chen, Yanyong Su
Communication Research Center, Harbin Institute of Technology Harbin, China
chenyishuo@hit.edu.cn, suyanyong@hit.edu.cn

Dong Xue, Chao Dong
Communication Research Center, Harbin Institute of Technology Harbin, China
xuedongadtx@163.com, dongchao@hit.edu.cn

Abstract—Visible light communication (VLC) technology is a new high-speed wireless access technology, finding a appropriate channel coding scheme is one of its main research directions. This paper proposed a new scheme to construct quasi-cyclic lowdensity parity-check (QC-LDPC) code. It is proved by the simulation result that the QC-LDPC code can help to improve the BER performance of the system because there is no girth four in its parity-check matrix. Then this paper studied the decoding algorithms of QC-LDPC code and proposed an improved decoding algorithm which can decrease the complexity of decoding process by introducing the multiplicative factor. The simulation results show that the QC-LDPC code is a suitable channel code for VLC, and the improved MS decoding algorithm reduces the complexity to a large extent on the basis of ensuring the BER performance of the VLC system.
Keywords—VLC; channel model; LDPC code; improved algorithm
I. INTRODUCTION
White light emitting diode (LED) has many advantages such as rapid response speed and high sensitivity, and can be used as emitters, by which signals can be modulated on visible light. Therefore, white LED can be chosen for high-speed data communication [1]. Visible light communication (VLC) is a new wireless access technology based on the white LED technology [2]. In order to reduce the effect of interference and ensure the reliability of VLC system, the application of channel code techniques is essential [3]. Currently, VLC system mainly uses the RS code as channel code to improve system reliability. This paper introduced LDPC code into VLC system and provided theoretical basis for the future applications of VLC technology.
II. FUNDAMENTAL OF VLC
A. VLC System Model
As the traditional RF communication system, a VLC system includes a transmitter, VLC channel and a receiver, as shown in Fig. 1. At the sending end, after being preprocessed, data information is disposed by Channel Coding module and OOK Modulation module successively, then the electrical signal is loaded into the DC source of drive circuit in order to modulate the intensity of light which is emitted from the LEDs.

Fig. 1. VLC system model

Thus, the electrical signal is converted into an optical signal which contains information of the input. After that, modulated optical signal is sent into the atmosphere optical channel for transmission. In this process, the optical signal is interfered by background light and other light sources, and is also affected by inter-shot noise, thermal noise and inter-symbolinterference. At the receiving end, the optical signal is first converted into an electrical signal by the PD Receiver, and then processed by the Amplifier, OOK Demodulation module and Channel Decoding module. Thus, original information is recovered eventually.

C. Improved MS Decoding Algorithm
Through the theory analysis above, it is obvious that LLR BP algorithm shows a good decoding performance; however, this algorithm cannot be integrated into hardware easily due to its large computation. On the contrast, MS algorithm owes a smaller computation at the expense of decoding performance, so it is not suitable for the systems with high reliability requirement. Thus, it is necessary to find a new decoding algorithm which can achieve an efficient tradeoff between computational complexity and decoding performance.

By comparing (11) and (14), it can be seen that the outputs of MS algorithm show same polarity as that of LLR BP algorithm when the information nodes are all the same, while their amplitude are larger than the latter. This indicates that MS algorithm overestimates the signal amplitudes, which results in the decrease of the reliability. In addition, the correlation between information nodes when the code length is short also has a harmful affection on decoding process. In conclusion, in order to degrade the amplitudes of the outputs of information nodes and solve the problems brought by the correlation between information nodes, this paper introduced a multiplicative factor into the updating formula of check nodes so that the decoding performance can be improved. V. SIMULATION RESULTS AND ANALYSIS
The above illustrated VLC system model and VLC channel model, and constructed an improved MS decoding algorithm. Next, simulations of VLC system based on LDPC code using the above two models at MATLAB environment are presented. The specific settings of simulation parameters are listed in Table I.

A. Girth Four Check of QC-LDPC Code
Firstly, check whether the parity-check matrix H of QCLDPC code has short girths or not, especially girth four. The code rate of the QC-LDPC code simulated in this paper is 0.8, specifically, a 2 , b 7 , q 123 , x1 x2 x3 0 , row
number of H is 615, and column number is 3075. Simulation result is illustrated in Fig. 4. As shown in Fig. 4, the elements on principal diagonal are ‘3’ or ‘5’, and all the other elements are ‘0’. This indicates that the QC-LDPC code constructed in this paper does not contain girth four, which ensures the Bit Error Rate (BER) performance of the system.

B. Effect of Iiterations on LDPC Code Performance
The number of iterations is a key factor, which affects the performance of decoding significantly. This paper made simulations respectively with iterations value 5, 10, 20, 30 and 50, and the simulation results are shown in Fig.5. As can be seen from Fig. 5, the system performs worst when iterations value 5. With the increase of iterations, the BER performance is continuously improved. When iterations reach 30, the BER performance of the system becomes stable, and cannot be improved dramatically by increasing iterations. Therefore, from the viewpoints of BER performance and decoding complexity, 10 is selected as the value of iterations in the following simulation process.

C. Effect of Multiplicative Factor on LDPC Code Performance
According to the analysis above, the value of the multiplicative factor P of improved decoding algorithm will determine the decoding performance. P is introduced to reduce the amplitudes of the outputs of check nodes, so its value should be between 0 and 1. When simulating, let P equals different value between 0 and 1 by setting reasonable steps under the conditions of different Signal Noise Ratio (SNR) environments in order to determine the optimal value of P . Therefore, the decoding performance of improved MS algorithm can achieve its best. The simulation results are shown in Fig. 6 and Fig. 7.
Fig. 6 illustrates the situations system BERs change with P under different SNR. Here, iterations are 10, and the BERs
are 3.0dB , 3.3dB and 3.6dB . It can be seen starkly from Fig. 6 that P has a great influence on system BER. The system BER declines at first and then gradually increases with P increasing from 0.5 to 0.8, and achieves its best when P values around 0.65. Fig. 7 displays the relationship between system BER and channel SNR when P values different. It can be concluded from Fig. 7 that the systems with different P perform almost the same when the channel SNR is low; but with the increase of the channel SNR, the influence of P begins to show and performances of the systems with different P varies. It is also obvious that the systems perform best when P values around 0.65.

D. LDPC Code Decoding Algorithms Comparison
After determining the optimal values of iterations and multiplicative factor P , this paper compared performances of three LDPC code decoding algorithms, and also made a comparison between the decoding performances of LDPC code and the RS code using in IEEE802.15.7 standard[7] whose code rate is also 0.8. The simulation result is shown in Fig. 8.
As in Fig. 8, the BERs of three LDPC code decoding algorithms are much lower than that of RS code, particularly, coding gain can even reach 2dB when SNR is high. Among the three decoding algorithms of LDPC code, MS algorithm performs worst and cannot meet the requirements of the systems that need good BER performance. LLR BP algorithm obtains a coding gain of 0.4dB when compared with MS algorithm. Improved MS algorithm performs worse than LLR BP algorithm slightly when SNR is low, but better than the latter when SNR is more than 4dB.

VI. CONCLUSION
This paper introduced QC-LDPC code into VLC system. Firstly, a kind of QC-LDPC code which is suitable for VLC was proposed. It is proved by the simulation results that there are no short girths such as girth four in the parity-check matrix of the QC-LDPC code, which ensures the BER performance of the system. Next, this paper analyzed LLR BP decoding algorithm and MS decoding algorithm, and proposed the improved MS decoding algorithm based on these two algorithms, which is simulated by MATLAB software at last. Simulation results indicate that, the BER performance of the QC-LDPC code is far superior to that of the widely used RS code. We must say that the algorithm complexity is increased, however, with the development of microprocessor technology, the QC-LDPC code will become a suitable channel code for VLC system.
