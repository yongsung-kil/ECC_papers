Updating Conflict Solution for Pipelined Layered LDPC Decoder

Zijing Wu, Kaixiong Su
College of Physics and Information Engineering Fuzhou University Fuzhou, China wuzj87@163.com

AbstractвҖ”Due to the overlap of nonzero sub-matrices in the successive layers of check matrix, the pipeline process might introduce data updating conflict in pipelined layered LDPC decoder. A solution to solve this problem by adjusting the decoding order of layers in check matrix and nonzero submatrices in the same layer is proposed in this paper. Furthermore the corresponding fast algorithm is given. In term of hardware implementation, this method which can be achieved simply by changing the order of the corresponding data in the ROMs will not increase any extra hardware overhead. Experimental results show that due to fewer idle clocks even zero idle clock need to be inserted into decoding pipeline when using this solution, the decoding rate is improved effectively. More importantly, the method will not degrade the decoding performance for LDPC codes.
Index TermsвҖ”LDPC codes, layered decoder, updating conflict, traveling salesman problem.
I. INTRODUCTION
Low Density Parity Check (LDPC) codes [1] have been widely used in various modern communication systems due to their remarkable error correction performance and simplicity in hardware implementation. Currently, the ideal decoding method for LDPC codes is the modified Min Sum decoding algorithm [2] under Turbo-Decoding Message-Passing (TDMP) mechanism [3]. In order to attain high speed decoding, pipelined layered LDPC decoder [4-6] is becoming a hot spot of research. But as a result of the overlap of nonzero submatrices in the successive layers of check matrix, the pipeline process might introduce data updating conflict in this kind of decoders, which becomes a major bottleneck limiting its throughput.
Many scholars have explored this problem and put forward a variety of solutions. The total number of overlapping nonzero sub-matrices was reduced by reordering the layers of check matrix, but the problem about the consecutive overlap of nonzero sub-matrices did not be discussed [7]. And some scholars did it by dividing the check matrix into smaller quasicyclic matrices and reordering the new layers, but this method reduced the number of parallel decoding and the throughput of decoder [8]. To avoid conflict, some researchers modified the decoding algorithm, but there was decoding performance degradation [9]. Other researchers constructed new basic

In conclusion, there is no ideal solution to solve the problems above. So we analyze two kinds of data updating conflicts and put forward a solution which not only can avoid this conflict but also can reduce the decoding delay in this paper. Firstly, we need obtain the optimal sequencing of layers, in which the maximum number of consecutive overlap of nonzero sub-matrices and the sum of overlaps between adjacent layers are all minimized. Then the optimal scheduling problem is boiled down to Traveling Salesman Problem (TSP), and Simulated Annealing (SA) is used to get the optimal sequencing of layers quickly. Finally, the processing order of nonzero sub-matrices in the same layer is adjusted to maximize the minimum difference of the processing order between the overlapping nonzero sub-matrices in adjacent layers. By this way, the updating conflict can be avoid furthest, and the fewest idle clocks need to be inserted. And a fast order adjustment algorithm is proposed also. This solution can reduce the decoding delay effectively and provide a higher decoder throughput without increasing any extra hardware overhead and degrading the decoding performance.

Let Д° be the time between the finish of reading one layer of L(qi) and the beginning of updating the L(qi), here Д° is 2. It can be seen from this figure that if the nonzero sub-matrices of any adjacent layers are in the same column, the update of the two layers all involve the same message L(qi). IV. UPDATING CONFLICT SOLUTION
A. Solution
To solve the updating conflict problem and improve the decoding rate, a new solution which minimizes the total number of the idle clocks inserted for a single iteration is proposed in this paper. It can be seen from the above analysis that there would be no updating conflict problem, if no overlapping nonzero sub-matrix exists in the dependent layers. So a transformation matrix in which no overlapping nonzero sub-matrix exists between adjacent layers may be obtained by simple elementary transformation in the check matrix, namely reordering the layers.
Of course, not all check matrices can be transformed into a non-overlap matrix by adjusting the order of layers. According to Eq. 4, the greater the difference between the processing orders of the two overlapping nonzero sub-matrices in the dependent layers the fewer the idle clocks need to be inserted. It doesnвҖҷt even need to be inserted any idle clock. According to the horizontal update function in Min Sum decoding algorithm as shown in Eq. 2, changing the processing order of the message L(qij) in the same layer has no effect on the updating result. So we can make the difference as greater as possible by adjusting the processing order of nonzero sub-matrices in the same layer. The maximum value of the minimum difference is limited by the maximum number of consecutive overlap of nonzero sub-matrices and the row weight. Row weight limits the greatest processing order, and the consecutive overlap makes the order of nonzero sub-matrix in each layer greater than that in previous layer. The formula for the possible maximum value of the minimum difference is given in Eq. 5.

B. Fast Algorithm
The optimal sequencing of layers can be obtained by using the вҖңbrute forceвҖқ technique. It will try all possible orders and find the best result of them. If there are M layers in the check matrix, the number of computations needed will be M!. When M is large, the amount of computation and the time of computation are staggering, even it is unable to realize. We restate this problem of optimal sequence searching into the famous Traveling Salesman Problem (TSP). All the ways to solve the TSP can be used to solve the problem in the paper.
The TSP asks the following question: Given a list of cities and the distances between each pair of cities, what is the shortest possible route that visits each city exactly once and returns to the origin city? The two problems can be correspond to each other as follow: a layer in the check matrix is considered a city, marked vi ,i вҲҲ[1, M ] . The order of the layer
is considered the visit route. The order starting from any layer vi to every other layer exactly once and returning to the origin layer is described by the vector V. The number of the overlapping nonzero sub-matrices between the ith layer and the jth layer is considered the distance between two cities, marked eij. So the problem in this paper can be expressed as finding a vector V which minimizes the cost. The cost in Step 1 is the sum of eij of all adjacent layers in that order. And the cost in Step 2 is the weighted average of the sum of eij of all adjacent layers and the maximum number of consecutive overlap in that order. Although the rules sound simple, it is an NP-hard problem. At present, the major ways to solve the TSP are ant algorithm, genetic algorithm, simulated annealing method, etc.
The simulated annealing algorithm [11] is adopted to reorder the layers in this paper. The flow diagram of the algorithm is given in Fig. 5. The main parameters in the algorithm were set as follows. Set the initial temperature to 1000, the number of inner loops to 500, the drop in temperature per iteration to 5%, and the lowest temperature to 0.001. The results in the next section show that the simulated annealing algorithm can achieve very good result and tremendously reduce the search time.

We include this matrix as an example of how to implement the solution of avoiding updating conflict in decoder. For the decoder shown in Fig. 2, the basic information about the nonzero sub-matrices is stored in the L(qi)_addr ROM and the Shift ROM in order, as shown in Fig. 6. And with nonzero submatrix for the unit, reading from and writing to memories of decoder and updating operations all run in this order. What change when using the updating conflict solution proposed in this paper are only the layer order for decoding and the processing order of the nonzero sub-matrices in the same layer. So the solution can be accomplished simply by storing the information about the nonzero sub-matrices in the new processing order, as shown in Fig. 6. The column position of the nonzero sub-matrix is stored in the L(qi)_addr ROM and the corresponding cyclic shift value information is stored in the Shift ROM. The implementation of the solution will increase only matrix preprocessing without adding any extra hardware overhead. Besides, the orders of read and write access and updating operations are still the same to each other. As no need to introduce extra memory for read and write order and controller, it has advantages of simple control and lower cost of hardware resource compared to other decoder [12].

VI. CONCLUSIONS
The updating conflict problem in pipelined layered LDPC decoder is analyzed and a solution which greatest reduced the decoding delay is proposed and the corresponding fast reorder algorithm is given. The solution is implemented in the 24 LDPC codes defined in DTMB and DVB-S2 standards and only three of them need to be inserted 1~3 idle clocks into layers, others need no idle clock. The solution is of general usage and can effectively improve the decoding rate without increasing any extra hardware overhead and degrading the decoding performance.
