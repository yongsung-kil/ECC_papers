High-Throughput LDPC Decoder on Low-Power Embedded Processors
Bertrand Le Gal and Christophe Jego

Abstract—Real-time efficient implementations of LDPC decoders have long been considered exclusively reachable using dedicated hardware architectures. Attempts to implement LDPC decoders on CPU and GPU devices have lead to high power consumptions as well as high processing latencies that are incompatible with most embedded and mobile transmission systems. In this letter, we propose ARM-based decoders that go from 50 to 100 Mbps while executing 10 layered-decoding iterations. We hereby demonstrate that efficient LDPC decoders can be implemented on a low-power programmable architecture. The proposed decoders are competitive with recent GPU related works. Therefore, software LDPC decoders constitute a response to software defined radio constraints.
Index Terms—ARM processor, LDPC, SIMD, multi-core.
I. INTRODUCTION
F ORWARD error correction (FEC) techniques are used in most embedded digital systems for controlling errors in data transmissions. The Low Density Parity Check (LDPC) code family has been adopted as a part of a large set of digital communication standards. In fact, high computational complexities associated with high throughput requirements limit the deployment of new radio communication systems such as Software Defined Radio (SDR) and Cognitive Radio (CR). Dedicated hardware architectures implemented on FPGA or ASIC technologies appear to be the solution to perform error correction while respecting throughput constraints and low-power consumption. However, these technologies do not offer the necessary flexibility levels required by SDR and CR systems. Designers have recently focused on software implementations of LDPC decoders on multi/many-core devices [1]–[6]. Thanks to the large computing capacity of such devices, software LDPC decoders are now able to meet the throughputs required by radio communication standards. Yet the power consumption of the devices is incompatible with most of the embedded systems [4]. To solve this issue, an ARM-based software decoder has thoroughly been explained in [3]. Besides, a mixed software hardware based implementation was proposed in [4] but both of them are found to be inefficient as (i) the first one has low throughput performance; and (ii) the second one has a high power consumption since a Stratix-5 FPGA and a host ×86 CPU are required. Another general concern about the multi/many-core decoders is the latency as for instance it can exceed one of second [5], [6].
In the present letter, we provide details about LDPC decoders on low-power processors. These implementations demonstrate that a software decoder can meet SDR and CR system con-

straints without leading to high power consumption. This letter is organized as follows: Section II briefly introduces the LDPC code family and its decoding algorithms. Then the proposed decoder implementation on ARM processor targets is described in Section III. Finally, a performance analysis and some comparisons with other ARM/×86 CPU and GPU implementations are conducted in Section IV.
II. LDPC CODES AND THEIR DECODING PROCESSES
Works [1]–[6] focused on mapping LDPC decoders on multi/ many-core devices using inter-frame and/or intra-frame parallelism levels. The objective was to reach real-time decoding throughputs. Most of these works were based on the standard Two-Phase Message Passing (TPMP) schedule described in [1], [5]. The Sum-Product is the most efficient message computation technique but one that is too complex for real-time implementations due to its transcendental operations. Moreover, it requires a reliable estimation of the noise standard deviation σ . In practice, Min-Sum (MS) based variants are preferred by designers [7]. Further enhancement of the decoding process can be obtained by modifying the computation schedule. In fact, the decoding convergence speed can be doubled through horizontal layered-based decoding schedule like in [8], [9]. This layered schedule reduces the computational complexity and provides some memory footprint saving. Its major limitation is the irregular memory accesses and the irregular computation parallelism. These properties have prevented layered decoding from being implemented over most many/multi-core targets. Indeed, in comparison with a standard schedule that is composed of two sequential loop kernels with no data dependencies, a layered version is composed of a single loop kernel [6], [8] with some data dependencies between consecutive loop iterations. In [6], the GPU decoder achieves high throughputs but its latency that goes beyond seconds makes it suitable for simulation purpose only.
An efficient ARM-based implementation of the horizontal layered-based decoding schedule is described in the next section. It is based on the parallel decoding of a low set of frames that allows reaching high throughput while maintaining low-latency. A layered decoding algorithm is associated to a Normalized Min-Sum (NMS) approximation [7], [10] which improves the correction performances compared with the MS algorithm [1]–[4]. In this work, data is represented on an 8-bit format. The fixed-point format management (width extension and saturation) is based on results given in [11].

It can be approximated by (F × C) × (m + N) bytes with m the number of messages in the decoding process and C the number of cores. Since L2 cache memory is shared between the processor cores, it impacts the throughput for frame lengths higher than 104 bits. Indeed, the amount of cache memory available in targeted ARM processors is insufficient. Compared with ARM-15, ARM-A9 processors have less cache memory and lower clock frequency. Experiments show that ARM-A9 throughputs are about ×3.5 lower than those of ARM-15; meanwhile, its power consumption is about 4× lower.

V. CONCLUSION
A novel approach for software LDPC decoding on lowpower processors is presented in this letter. The software decoders, available in open-source, are evaluated on ARMbased platforms. Experimental results demonstrate that the proposed decoders are efficient in terms of throughput and power consumption. For instance, ARM A15 decoders can achieve throughputs ranging from 60 to 140 Mbps when 10 layered decoding iterations are executed. Their high-throughputs, lowpower consumptions and low-latencies make them attractive for SDR and CR radio communication systems.
