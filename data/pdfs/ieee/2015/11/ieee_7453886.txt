Hybrid Construction of LDPC Codes of Code Length within 2,000 with (14, 8) Extended Hamming Code

Huan Ma, Yangwen Yu, and Lijun Zhang School of Electr. and Inform. Eng, Beijing Jiaotong Univ.
Beijing, 100044, China Email: ljzhang@bjtu.edu.cn

Abstract—Based on a (14, 8) extended Hamming code constructed via column splitting of the (7, 4) Hamming code, a method for constructing LDPC codes of code length within 2,000 and of column weight 3 is presented. By using the (14, 8) code and hybrid construction, it is effortless to generate an LDPC code with code rate generally higher than 0.9 by mapping the this code to component of the parity-check matrix. Simulation results show that compared with hybrid construction of Gallager codes and (14, 8) code, that of PEG codes and (14, 8) code performs better and the capacity of it is only a difference of 1.5 dB around to the Shannon limit with sum-product algorithm decoding.
Keywords–hybrid construction; (14, 8) extended Hamming code; column splitting; Gallager codes; PEG algorithm

I. INTRODUCTION
Low-density parity-check (LDPC) codes, proposed by Gallager [1] in 1962, which has become one of intensely studied areas in coding, have the capacity-approaching performance and parallel decoding architecture. However, for a long time, LDPC codes didn’t attract much attention until 1993, in which period Turbo codes [2] were put forward. Since then, LDPC codes have received intensive research once again. Due to the work of Mackay [3] [4], LDPC codes saw a new era. One noteworthy theory , proposed by Tanner, is the graphical representation of LDPC codes, now called Tanner graph [5]. In addition, in terms of application, LDPC codes have been used in the area of multiple-access channel [6], IEEE 802.11n system [7] [8] and optical system [9], etc. Nevertheless, in this paper, focus is on the theory presented in [10].
LDPC code is a member of block linear codes with a paritycheck matrix whose number of 1 is small and its construction is mainly about how to get the parity-check matrix. It [10] proposes the hybrid construction of combining an algebraic method with a random method. Base on this observation, a (14, 8) extended Hamming code, with excellent merit of almost the same performance either applying sum-product algorithm (SPA) or maximum likelihood decoding (MLD) [11], is introduced to conduct the hybrid construction. These hybrid codes not only have high code rate but include a few 4-cycles.

C. Algorithm analysis
The graphical representation of a binary linear block code is called Tanner graph [5], which is actually a graphical representation of its parity-check matrix. The row and column constraint (RC-constraint) means no two rows (or two columns) have more than one 1-component in common in a parity-check matrix, which implies it is impossible to exist four edges between every two variable nodes and every two check nodes in the Tanner graph.
For the codes with parity-check matrix of small column weight, their iterative decoding performance over AWGN (Additive white Gaussian noise) is likely to suffer to negative influence, which is the precisely the reason to introduce unit matrix in the end of the hybrid code obtained in the third step of the algorithm.
The numbers of 4-, 6- and 8-cycles in all matrices used in this paper are listed in TABLE I.
From TABLE I, it can be seen that the PHs have much less 4- and 8-cycles than GHs which benefits from the PEG algorithm devoting to maximize the girth. Although PHs have 4-cycles, which result from the unit matrix coming later, it is obvious that, for PHs with different code length, the number of 4-cycles has little difference.

III. A TYPICAL COMBINATION
According to algorithm description above, apparently, the matching between the base matrix H1 and the (14, 8) ECH should be guaranteed, that is, the column weight of base matrix must be 7. By replacing 1-component in every column of base matrix with each corresponding row of (14, 8) EHC, three sets of hybrid codes rate can be constructed.
A. Gallager codes
The original definition of an LDPC code proposed by Gallager involves that the m×n parity-check matrix HP of the code has γ m ones in each column and r n ones in each row. The method to construct LDPC codes is depicted as follows. The matrix HP has the form HP=[H1,H2,...,Hγ], where, the submatrix Ha (1≤a≤γ), has the following structure. For any integers μ and ρ, both greater than 1, Ha is a matrix of μ×μρ whose row weight is ρ, while, column weight is 1. Moreover, for ith (1≤i≤μ) row, the elements for columns from iρ−(ρ−1) to iρ are 1. The other submatrices are obtained by column permutation of H1 [1].
B. Progressive-edge-growth (PEG) algorithm
The PEG algorithm is a graph-based method for constructing LDPC codes with large girth proposed in [12]. Given two sets that include one set of n variable nodes and the other of m check nodes, a set of rules and a given variable node degree sequence. In the beginning, two sets of nodes are independent of each other (There is no edge connecting the nodes of two sets). Then an edge-selection procedure is proceeded. In this process, only one edge is added to the graph at a time and it will not terminate until the number of edges added to a variable node is up to its specified degree. The procedure of edge-selection is designed to maximize the local girth of each variable node.
This progressive addition of edges to a variable node is called edge-growth, which starts from the variable nodes of the smallest degree and end with that of the largest degree. The variable nodes with low degree are more error prone, hence, edge placement starts out conducting on them. The first edge attached to the variable node is connected with the check node with the lowest degree for current state of the graph and the

IV. SIMULATION RESULTS
The simulations are conducted to set the the maximum iterations of 200 over AWGN channel with BPSK modulation. Three sets of base matrices corresponding to the H1 in construction algorithm with sizes of 105×105, 119×119, 126×126, respectively, which fill the bill for similar code rate of any set. They are listed in TABLE II. Furthermore, base matrices nust have column weight of 7 in order to make the utmost of (14,8) EHC whose number of row is also 7. Three sets of hybrid LDPC codes can be obtained with the combination rule. Some parameters of these codes are also listed in TABLE II.

Fig.2 shows performance comparison for GHs and PHs. Overall, the curves of GHs are on top of PHs, both for BER and FER. Moreover, while the signal noise ratio (SNR) being smaller than 4 dB, the curves are nearly overlapping, indicating similar performance, yet, the gap between curves of two kinds of codes becomes increasing large while SNR exceeds 4 dB. In addition, the capacity of PHs is only a difference of 1.5 dB around to the Shannon limit.
From Fig.3a, although, the average iterations of PHs are no better than GHs at start, yet from Fig.3b, it is evident that the contrary is the case while SNR reaches some value. Additionally, PHs appear to be little different from each other.
V. CONCLUSION
The construction of hybrid LDPC codes of code length less than 2,000 with (14, 8) EHC is proposed. It can obtain a large number of LDPC codes with code rate higher than 0.9 through this method. Simulation results show that the capacity of PHs is very close to Shannon limit within 1.5 dB. By contrast, the GHs performs much worse, which probably results from 4-cycles.
