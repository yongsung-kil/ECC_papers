Trapping Sets in Stochastic LDPC Decoders
Kuo-Lun Huang†, Vincent C. Gaudet‡, and Masoud Salehi†
†Department of Electrical and Computer Engineering, Northeastern University, Boston, Massachusetts ‡Department of Electrical and Computer Engineering, University of Waterloo, Waterloo, Ontario huang.ku@husky.neu.edu, vcgaudet@uwaterloo.ca, salehi@ece.neu.edu

Abstract—Stochastic decoding is a hardware and energyefficient approach to implement iterative Low-Density ParityCheck (LDPC) decoders. However, stochastic decoding often has a performance loss compared to the sum-product algorithm and suffers from additional trapping sets, which limit the performance of LDPC codes in the error floor region. In this paper, we investigate the error characteristics of stochastic LDPC decoding. We also report stochastic-decoding-specific trapping sets for the (1056,528) LDPC code from the WiMAX standard. This observation has a potential to guide code structure designs that lower the error floor in stochastic decoding.
Index Terms—Iterative processing, low-density parity-check (LDPC) codes, stochastic computation, stochastic decoding, trapping sets.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes are a class of error-correcting codes that provide near-capacity performance [1]. Because of their outstanding performance, LDPC codes have been adopted in several digital communication standards, such as 10GBASE-T (IEEE 802.3an) [2], WPAN (IEEE 802.15.3c) [3], WiFi (IEEE 802.11) [4], WiMAX (IEEE 802.16) [5], and Digital Video Broadcast (DVB-S2) satellite communications [6].
The key characteristic of an LDPC code is its parity-check matrix, H, that has a low density of non-zero elements. In [7], Tanner showed that LDPC codes can be represented effectively by a bipartite graph, now known as the Tanner graph. The Tanner graph consists of two distinct groups of nodes: variable nodes and check nodes. For a rate Rc = k/n LDPC code, the Tanner graph contains n variable nodes that representing codeword bits, and n−k check nodes that represent the code’s parity-check equations.
LDPC decoding is usually done iteratively using a form of belief propagation principle known as the Sum-Product Algorithm (SPA) [8], or its less-complex approximation, the Min-Sum Algorithm (MSA) [9]. Both algorithms iteratively compute a posteriori probability messages that are passed between nodes in the Tanner graph. The performance of LDPC codes and the error-correcting ability of these decoding methods improve as the codeword length n gets longer. However, when implementing an LDPC decoder, more chip area is required for a longer code. The complexity of the interconnection network between nodes also grows higher as n increases. Routing congestion leads to high chip area consumption and low logic utilization [10].

In order to decrease the computational complexity at nodes and alleviate the routing congestion problem, stochastic computation was first proposed by Gaudet and Rapley to implement LDPC decoders [11], and later Tehrani et al. refined the algorithm and presented a fully parallel stochastic decoder for longer practical codes [12]. Stochastic computation is introduced as a method to replace the probabilities transmitted in the decoding process by streams of stochastic bits using Bernoulli sequences. Using this representation, all the operations on probabilities are converted to bit-wise operations, which can be implemented by simple logic arrangements [13]. Furthermore, the inherently serial messages used in stochastic decoders only require a single wire to transmit between nodes.
However, stochastic decoder is like other iterative decoders that tend to suffer from a phenomenon called the error floor. An error floor is an abrupt flattening in the error rate performance at high SNR values. The error floor is attributed to certain sub-graphs of an LDPC code’s factor graph. The code structures that lead to decoding failure must be identified to alleviate the error floor problem. Depending on the decoding algorithm and the channel model, these sub-graphs have different names. For the Binary Erasure Channel (BEC), the problematic structures are known as stopping sets [14]. For the Binary Symmetric Channel (BSC) and the Additive White Gaussian Noise (AWGN) channel, the structures that cause error floors are called trapping sets [15], near-codewords [16] or pseudo-codewords [17]. In this paper, we call them trapping sets for simplicity.
The performance of a code depends on the configurations embedded in its factor graph, including short cycles and trapping sets. A judicious choice of a code graph realization is crucial to the performance of iterative decoders; hence with the knowledge of the trapping sets, we can design a code graph that selectively avoids these structures to lower the error floor. In addition to guide the code structure designs, the knowledge of trapping set is also required in many proposed methods for the iterative decoding algorithm to lower its error floor [18][19][20]. For example, after identifying the subset of variable nodes with erroneous outputs, i.e., the trapping set, the post-processing algorithm in [19] performs the process of modifying the variable nodes inside the trapping set and running further iterations to eliminate the errors. It needs to be noted that, even for the same code, different decoding algorithms could have different error characteristics. In order to have insight of the trapping sets in stochastic decoders, we

II. STOCHASTIC DECODING OF LDPC CODES
In stochastic decoders, the messages passed along the edges between variable and check nodes are represented by streams of random symbols. For the binary case, each bit of the stream is equal to 1 with the value of the soft information to be encoded. The transmitted probability is represented by the appearance of 1s in the stream. For instance, a bit stream containing 75 percent of 1s and 25 percent of 0s indicates a probability p = 0.75, which means that the probability of observing a 1 at an arbitrary position in the stream is p. Stochastic bit is normally generated by employing a stochastic number generator, where the probability p is compared to a uniformly distributed random number R, which varies each clock cycle. If the probability p is greater than the random number R, the stochastic number generator outputs a 1, and a 0 otherwise. The relationship between a probability value and a stochastic stream is not unique; since stochastic streams are randomly generated, different stochastic streams are possible to represent a given probability. For instance, (1, 1, 0, 1), (0, 1, 1, 1), and (1, 0, 0, 1, 1, 1, 1, 1) are possible representations of a probability p = 0.75.
An advantage of stochastic decoding is its inherently low hardware utilization. All the computations and operations at nodes in the SPA can be transformed into bit-wise stochastic operations. At each clock cycle, there is only one bit being computed for and transmitted over each edge. With this process, stochastic decoding reduces the number of physical wires required to exchange messages, and therefore alleviates routing congestion. Because of its efficiency in hardware implementation, stochastic computation has been used to decode the (1056,528) LDPC code employed in the WiMAX standard [12], the (2048,1723) LDPC code used in the 10GBASE-T standard [21], the (672,588) LDPC code in the WPAN standard [22], and many other error-correcting codes [23][24][25].
The decoding round of stochastic decoding is called a Decoding Cycle (DC) [12], which includes the exchange of one bit for each edge of the Tanner graph. One DC does not directly correspond to an iteration in the SPA or the MSA. However, the node operations in stochastic decoder are similar to that in the probability-domain version of the sum-product decoding, but are processed in bit-serial fashion. The decoding process of stochastic decoders is operated as follows.
Step 1. Initialization. Each variable node employs stochastic number generator to transform the channel receiving probability to a stochastic bit and sends the one-bit message to its connected check nodes.

Step 2. Check Node Update. Check node computes the one-bit message by Fig. 1(b) using the incoming bits from all the other edges, and then sends it back to the connected variable node.

Step 3. Codeword Test. After receiving the bits from the neighboring check nodes, each variable node determines the code bit by Fig. 1(a) with all the receiving bits. The decoding process is terminated if a valid codeword is decoded.

Step 4. Variable Node Update. The one-bit message from variable node to its connected check node through the edge is computed by Fig. 1(a) using the channel incoming bit and the extrinsic bits from all the other edges.
The algorithm returns to Step 2 until the maximum number of DCs is exceeded.

III. GRAPHICAL STRUCTURE THAT LEADS TO DECODING FAILURE
The existence of short cycles within the factor graph sometimes creates sub-graphs that are harmful to iterative decoding. These subsets of variable nodes and their neighboring check nodes form a structure that results in an error floor at high SNR values. At the initial stage of iterative decoding, variable nodes send the messages with less reliability to neighboring check

After certain decoding rounds, the external variable nodes start adjusting the incorrect values toward the correct values. However, the variable nodes inside the sub-graph may have significantly biased to the incorrect values, because these variable nodes have low-reliability channel values and there are very few neighboring check nodes capable of detecting the errors within the sub-graph. The unreliable messages remain uncorrected until the end of the decoding process in this sub-graph. This structure results in decoding failure for iterative decoders.
The subset of variable nodes which eventually outputs incorrect code bits after a large number of decoding rounds is defined as the trapping set. Richardson in [15] introduced the notion of trapping sets in the following way. Let us assume the all-zero codeword is transmitted over a binary channel. For a given received sequence y, let Cil(y) ∈ {0, 1} denote the i-th output bit at the l-th round of decoding y sequence. The i-th bit is eventually correct if there exists L such that for all l ≥ L, Cil(y) = 0, for all i. The sub-graph contains variable nodes that eventually output incorrect bit values is denoted as T (y), and we say that T (y) is a trapping set. From the code structure perspective, if there are α variable nodes in the trapping set with exactly β check nodes connected to the subset of variable nodes with odd-degree connection and an arbitrary number of check nodes connected to the trapping set with even-degree connection, the trapping set is denoted by T (α, β). While all the check nodes in the trapping set are with degree-1 or degree-2 connections, the trapping set is known as an elementary set [26].
The size of trapping set can be expanded from the smaller size trapping sets. The small trapping set is more harmful than the large size trapping set, because it has higher probability to occur, especially when β α. In addition, the structure of trapping set also affects the performance of iterative decoders. Dolecek et al. in [27] have demonstrated that absorbing sets, which are defined as a subset of trapping sets whose variable nodes connected to more even-degree check nodes than odddegree check nodes, dominate the error floor performance and shown that most absorbing sets appear to be elementary sets.
Although stochastic decoding has advantages in terms of hardware implementation, it suffers from the errors that do not exist in other iterative decoding algorithms, such as the SPA and the MSA. The Bit Error Rate (BER) curves of an LDPC code decoded by the SPA, the MSA, and stochastic decoding algorithm are shown in Fig. 2. Comparing with the sumproduct decoding, the stochastic decoding has a performance loss of approximately 0.5 dB and a higher error floor which is due to the existence of erroneous outputs that are specific to stochastic decoding. Furthermore, we observe that the errors in these iterative decoding methods are characterized as a failure to converge to a valid codeword, and thus the errors are detectable. These detectable errors are the most common decoding failure in belief propagation decoding of LDPC codes [20].

V. CONCLUSION
In this paper, we show the error characteristics of stochastic decoding and the factors lead to decoding failure in stochastic LDPC decoders. We report stochastic-decoding-specific trapping sets, which do not occur for other iterative decoding algorithms even when decoding the same noise sequence. The trapping set found in the (1056,528) LDPC code from the WiMAX standard is an elementary absorbing T (10, 1) set. The knowledge of the stochastic-decoding-specific trapping sets not only helps to provide the insight of the trapping errors, but also gives guidelines to design a code graph having lower error floor in stochastic decoding.

It needs to be mentioned that stochastic decoding is a nondeterministic process, in some trials, it succeeds in decoding the same noise sequence that results in decoding failure in other trials. The error of stochastic decoding is not assured even decoding the same noise sequence. In cases where a decoding failure occurs, the stochastic decoding algorithm typically generates the same error locations generated by the other decoding algorithms, plus some extra errors. The stochastic-decoding-specific trapping errors are quite rare. We also observe that the variable nodes with lower degrees are prone to be in the trapping set. For example, the degree2 variable nodes in this length-1056 LDPC code are more probable to have erroneous outputs.
