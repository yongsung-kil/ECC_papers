Hamming-GLDPC Codes in BEC and AWGN Channel

Yangwen Yu, Ya’nan Han,Lijun Zhang School of Electr. and Inform. Eng., Beijing Jiaotong Univ
Beijing, 100044, China Email: ljzhang@bjtu.edu.cn

Abstract—Hamming codes are used as component codes to construct generalized low-density parity-check (GLDPC) codes. In BEC channel, the convergence behavior of the HammingGLDPC codes is theoretically analyzed with EXIT chart. The duality property of Hamming codes and simplex codes helps to get the EXIT chart of Hamming code with the bridge of the EXIT of simplex codes. The analysis shows that the advantage of convergence for Hamming-GLDPC codes is obvious over the duplicate standard LDPC codes. In AWGN channel, simulation results of the GLDPC code with Hamming (7, 4) code as component code also indicate the powerful decoding performance of Hamming-GLDPC codes. Up to 1.5 dB coding gain can be obtained for GLDPC code in comparison with the counterpart standard LDPC code at BER of 10-3.
Keywords—Hamming-GLDPC code; EXIT chart; simplex code; dual property
I. INTRODUCTION
Low-density parity-check (LDPC) codes introduced by Gallager[1] are excellent error correcting codes[2], which are arousing interest in the field of optical communication and dense data storage. Generalized low-density parity-check (GLDPC)codes are the generalized versions of LDPC codes. Via the Tanner graph, it indicates that the check nodes and variable nodes of GLDPC codes are both more general than what they are in standard LDPC codes[3]. Thus, the decoders of GLDPC codes are more flexible. Without the restraint of decoders, the performances of GLDPC codes are worth looking forward to. It is significant to take into consideration for the GLDPC codes.
Compared with single parity check (SPC) codes, Hamming codes have better error correcting performance and better ability to resist random and burst errors[4]. At the same time, as a kind of linear block codes, the structures of Hamming codes are relatively simple. On account of more correcting constraints and lower decoding consumptions, Hamming codes are considered as wise choices to be the component codes of GLDPC code[5]. In this kind of GLDPC codes, i.e., HammingGLDPC codes, the check nodes represent Hamming code constraints instead of SPC code constraints, called Hamming check nodes (HCNs). In fact, the variable nodes can also be more complex codes instead of repetition (REP) codes, but such doubly generalized LDPC codes will not be considered in this paper.
To verify the prediction that Hamming-GLDPC codes will have more outstanding performances than the counterpart standard LDPC codes, both binary erasure channel (BEC) and additive white Gaussian noise (AWGN) channel are used[6, 7].

II. CONSTRUCTION OF HAMMING-GLDPC CODES
In an (N, M, K, n, m, k) Hamming-GLDPC code, the parameter N stands for the length of this Hamming-GLDPC code, which also represents the number of columns in the corresponding check matrix, the parameter M stands for the number of rows in matrix, the parameter K stands for the number of information bits. In the same way, n, m and k refer to the corresponding component parameters of the Hamming code. Notice that M = p·m, where p is an integer here. Meanwhile, both K and k depend on the ranks of Hamming-GLDPC code and the Hamming component code check matrices[4]. To construct an (N, M, K, n, m, k) Hamming-GLDPC code, firstly, we need construct a standard LDPC check matrix which has N columns and M/m rows. Meanwhile, it is necessary to ensure the number of nonzero elements in each row is n. Then replace these n nonzero elements by n different columns of Hamming component code’s check matrix successively. Finally, convert other all zero elements to the zero columnvector, whose length is matched to the number of Hamming matrix’s rows, namely, m. Thus, a new (N, M, K, n, m, k) Hamming-GLDPC code is constructed.
For the sake of intensifying the digestion of the relationship between a Hamming-GLDPC code and its relevant initialized standard LDPC code, the description of this relationship by Tanner graph[8, 9] is essential. As depicted in Fig. 1, circles represent VNs as usual, while the square nodes are no longer SPC code constraints, but Hamming code constraints. PEG algorithm is used for constructing a (147, 42) standard LDPC matrix. Certainly, there are a lot of methods which is similar to PEG, so it is not necessary to choose a special method to construct such matrix deliberately in this paper. Then using a (7, 4) Hamming code as component code, a (147, 126, 21, 7, 3, 4) Hamming-GLDPC code would be constructed.

III. EXIT CHART OF HAMMING-GLDPC CODE IN BEC
In BEC, the simulation of standard LDPC code, limited to the actual application, is unengaging for many engineers. However, due to the plain channel model and simple parameters of BEC, numerous theoretical contributions were born in this kind of channels[3−6]. EXIT chart is a typical example, which is a good theoretical method to analyze the performances of iterative decoders. Therefore, this method is employed to portray the iterative decoding behavior of Hamming-GLDPC codes here.

A. EXIT Chart
As a graphical improvement of density evolution (DE), EXIT chart is an effective tool for estimating the convergence behavior of iterative processors[5]. It provides some visualization regarding the dynamics and convergence properties of an iteratively decoded code such as LDPC code and turbo code. EXIT stands for extrinsic mutual information transfer. The extrinsic information probabilities, generated from the decoder, depend on the priori probabilities going into the decoder and the posteriori probabilities coming out of the decoder. Using the knowledge of mutual information and entropy, we define two parameters, i.e., IA and IE. The symbol IA is called the average priori mutual information going into the decoder, and IE is called the average extrinsic mutual information coming out of the decoder. An EXIT chart plots IE as a function of IA.
For an iterative decoder of standard LDPC code, which has two component decoders, the extrinsic information can be exchanged between the variable nodes decoder (VND) and check nodes decoder (CND). Hence, the value IE of VND called IEv is approximately equivalent to the value IA of CND called IAc. Likewise, IEc and IAv are approximately equivalent. The similar relationship between the VND and Hamming check nodes decoder (HCND) are used to investigate the decoding performance of the Hamming-GLDPC code. So the curve of VND and the inverse one of HCND can be plotted in the same diagram.

IV. HAMMING-GLDPC CODE IN AWGN CHANNEL
Due to the complex EXIT function of Hamming code, a theoretical analysis of the decoding performance for Hamming-GLDPC code in AWGN channel is quite involved. Decoding simulation for this kind of code is a feasible method to estimate the performance of the iterative decoding for Hamming-GLDPC code. SPA is used to decode the Standard LDPC code, in which the check nodes decoders are restrained with SPC codes, while for Hamming-GLDPC code which have the same matrix with standard LDPC code, the check nodes decoders of MLD are restrained with the Hamming component check blocks described in section II.
The performance of this (147, 21) Hamming-GLDPC code, which has been constructed above, with different maximum iterations are shown in Fig. 5. V. CONCLUSION
We have constructed one kind of GLDPC codes with Hamming component codes. With EXIT chart and simulation, it can be shown that the Hamming-GLDPC code outperforms the counterpart standard LDPC code in terms of decoding convergence and BER performance, both in BEC channel and AWGN channel. The decoding complexity of the code with the shortest Hamming (7, 4) code is reasonably feasible, which enhances the practice of the code.
In future work, the generalized effective decoding algorithms will be devised to propel the pace of the practical application of the proposed GLDPC codes. Furthermore, other complex codes, which have more outstanding decoding performances, can be considered as the check nodes constraints instead of SPC codes.
