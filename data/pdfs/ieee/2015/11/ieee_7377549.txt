LDPC (512,480) Genetic Design as Alternative to CRC in Implementation of
AODV Routing Protocol Stack

Radek Salom, Jan Broulim

Abstract—This paper presents a design of LDPC (Low Density Parity Check) code. Algorithm deals with code optimization based on Tanner graph mutations. The LDPC code was designed to replace CRC (Cyclic Redundancy Check) code in the application of a tiny AODV routing protocol stack. Own AODV routing protocol stack was previously designed especially for wireless sensor networks with constrained resources and it is intended for usage in wireless mesh network formed by Intelligent firemen suits. Link layer of target platform uses 64B long packet with 4B for LDPC parity. The code was designed to provide the most optimal detection and correction capability with the lowest possible parity bites.
Keywords—LDPC, AODV, CRC, Error correction, Error detection, Parity-check, Sensor wireless networks.
I. INTRODUCTION
T HE popularity of LDPC (Low Density Parity Check) codes, firstly introduced by Gallager in [1], has grown significantly after MacKay’s work [2] in 1997. LDPC codes are represented by a parity-check matrix, which can be projected into a bipartite graph called the Tanner graph. Softdecoding algorithms use this graph. If the column and row weights (node degrees in the graph) in the appropriate paritycheck matrix of an LDPC code are constant, the code is regular. This idea was considered by Gallager. If this condition is not met, the code is irregular. Several construction algorithms were studied and very good performance of irregular codes was shown. These codes beat Turbo codes at error correction capability; see [3]. Many correcting codes in different applications tend to be replaced by LDPC codes because of their very good correction performance.
The motivation of this work was to design an optimal LDPC code for the error detection and correction intended for a practical application within built wireless network. The sensor wireless network utilities self-designed tiny AODV routing protocol stack and it is operated in a mesh network topology. Network is used for communication among intelligent firemen suits. III. AODV ROUTING PROTOCOL
The AODV routing protocol is a representative of a dynamic distance vector routing protocol. Distance vector routing is an adaptive routing mechanism from the perspective of routing protocols. Dynamic routing algorithms search the shortest path for the current topology. The RFC 3561 [6] describes the AODV operation principle and gives recommendations for an implementation.
The AODV utilizes the mesh network topology. The communication paradigm of the mesh topology is decentralized. It means that each node can directly communicate with any other node within its radio range. Fig. 1 shows principle of route discovery within AODV routing protocol, where the node 1 searches route to the node 8. The node 1 sends route request (RREQ) packet, which is broadcasted through the whole wireless network. After route request packet reaches the destination node or a node which has information about valid route to the destination, a route reply packet (RREP) is sent back to the originator.

VI. DATA PACKET STRUCTURE
Fig. 4 shows a structure of the implemented data packet. The packet length is 64B and it is constant. Packet structure is presented in levels of the ISO OSI model. The physical layer (PHY) is given by used RF interface, the link layer (MAC) contains 4B bytes of designed LDPC code. The network layer (NWK) was implemented together with the AODV protocol stack.

We propose a genetically designed LDPC code based on algorithm [8] which works with Tanner graph mutations as a replacement of Cyclic Redundancy Check. The LDPC code was designed for 512 code bits and 32 parity bits (code efficiency is 0.9375). This solution extends detection performance of CRC using the double number of parity bits and furthermore allows correction capability.
The design was performed by genetic optimization up to the 1000th generation. The code was optimized for single and double bit correction with uniformly distributed errors. The evaluation function for optimization was calculated in 2 independent experiments for 1000 codewords and the average value of uncorrectable errors was taken for the fitness value in the optimization process. The single-bit and double-bit uncorrected error rates were calculated and weighted with greater value for single-bit errors.
Let uc(n) be the number of uncorrectable frames with nbit errors determined with uniformly distributed errors in 1000 codewords. The evaluation function (f itness) was calculated by the following process:
uc(n)= uc(n) |experiment1 +uc(n) |experiment2 , 2
fitness=2uc(1)+uc(2).

Designed LDPC code is intended for the wireless network formed by Intelligent firemen suits. Such wireless network is intended for an operation in the very extreme conditions, with very high probability of interferences and data corruption. Usage of efficient error detection with possibility of correction is needed in these conditions. Also due to limited packet length (64B in this case application) the low redundancy is required (4B). Designed LDPC code fits best with its parameters for this application.
Future work will be focused on the optimization on the convergence performance of the genetic optimization and also the decoding performance of LDPC codes. Adaptive decoding techniques can be used for increasing the correction capabilities.
