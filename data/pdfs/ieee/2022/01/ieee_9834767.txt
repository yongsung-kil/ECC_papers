Look-Ahead Bit-Flipping Decoding of MDPC Code

Haruhiko Kaneko School of Computing, Tokyo Institute of Technology
Tokyo, Japan hkaneko@ieee.org

Abstract—This paper presents a modified bit-flipping (BF) decoding algorithm for moderate-density parity-check codes using a property of the number of unsatisfied parity checks (NUPCs) in uncorrected error positions. The proposed BF algorithm selects candidate flipping positions using the current NUPC, and then selects one flipping position based on a sum of NUPCs in suspicious positions after tentative flipping of each candidate position. The block error rate (BLER) is evaluated by computer simulation, and the results show that the presented BF algorithm gives lower BLERs compared to existing decoding algorithms.
Index Terms—QC-MDPC McEliece PKC, bit-flipping decoding, number of unsatisfied parity-checks, block error rate, postquantum cryptography
I. INTRODUCTION
Code-based public-key cryptography (PKC) [1] is one of the promising candidate of the post-quantum cryptography (PQC) [2] along with the lattice-based PKCs. There is high confidence that the original McEliece PKC based on Goppa code is secure [3] since it resisted against all attacks over decades [4]. The security of code-based PKCs is usually measured by the computational cost for information set decoding (ISD) [5] or improved efficiency ISD, such as Stern [6] and BJMM [7] algorithms. Thus it is important to construct efficient decoding algorithm under security constraints of the parity-check matrix. Many variants of code-based PKCs have been proposed using linear block codes [8] other than Goppa code, such as low-density parity-check (LDPC) and moderate-density parity-check (MDPC) codes [9]–[15], and polar code [16]–[18].
One of the major drawbacks of classic code-based PKC is a large key size of private and public keys. Quasi-cyclic (QC) MDPC McEliece [19] reduces the key size significantly by using cyclic structure of parity-check and generator matrices. Many variants of the QC-MDPC McEliece were proposed using, for example, nonbinary MDPC code over prime field [20], MDPC code with real-number error vector [21] (it is broken by soft Stern algorithm [22]), combination with LDPC code [23] (broken by key recovery attack [24]), protograph MDPC code [25], and irregular MDPC code [26]. A class of weak keys in QC-MDPC McEliece PKC was specified in [27].
Decoding of QC-MDPC code usually uses iterative algorithms with non-zero failure probability, such as bit-flipping (BF) algorithm, and thus it is important to lower the decoding failure probability. The failure probability of MDPC code is analyzed in [28], and bound on the error rate of BF decoding is analyzed in [29]. Some modifications to the original BF algorithm were proposed, for example, parity-

check matrix extension method [30], backflipping [31], blackgray-flip (BGF) [32], and weighted BF [33].
Beside the decoding failure probability, the QC-MDPC McEliece PKC also requires decoding algorithms which are resistant to reaction and timing attacks. For example, key recovery GJS attack [34] uses the relation between the error pattern spectrum in ciphertext and decoding failure probability. Resilience of some decoding algorithms against GJS attack was analyzed in [35]. An improvement of the GJS attack was proposed in [36], and also the GJS attack was extended to key recovery timing attack [37]. Beside decoding algorithms, a key generation method to prevent the GJS attack was proposed based on monomial codes [38].
To reduce the decoding failure probability of MDPC code, this paper proposes a modification of BF decoding algorithm for MDPC code, namely look-ahead BF (LABF) algorithm. The LABF is based on a property of the number of unsatisfied parity-checks (NUPCs) in uncorrected error positions. To further reduce the decoding failure probability, this also presents a simple re-execution method of the LABF with an appropriately altered ciphertext. The number of iterations of the LABF is not constant, and thus the LABF is not resistant to the reaction and timing attacks. This problem remains to be addressed in the future.
The rest of this paper is organized as follows. Section II introduces the QC-MDPC McEliece PKC [19]. Section III presents some properties of the NUPC used in the LABF algorithm. Sections IV and V propose the LABF algorithm and the LABF algorithm with re-execution, respectively. Section VI evaluates the block error rate (BLER) by computer simulation, and Section VII concludes the paper. This paper uses the following notations. The set of integers from a to b is expressed as [a, b] = [a, a+1, . . . , b], where a ≤ b. The binary field is expressed by F2, and binary row vector of length n is expressed as x = (x0, x1, . . . , xn−1) ∈ Fn2 . The Hamming weight of x is denoted by w(x), the transpose of x by x⊺, and the zero vector of length r by or.

IV. LOOK-AHEAD BF DECODING ALGORITHM
A. Concept
The existing BF algorithms flip the ith bit ci of ciphertext c if the NUPC αi is relatively large. Criteria for selecting the flipping position i are, for example, (i) selecting one flipping position by i = arg max{αj}, or (ii) selecting a set of multiple
j∈[0,n−1]
flipping positions as

I = i ∈ [0, n − 1] αi ≥ max {αj} − δ ,
j∈[0,n−1]
where δ ≥ 0 is a threshold value. To avoid miscorrections in the BF algorithm, the LABF
algorithm shown in this section employs a look-ahead tiebreaking rule for selecting one flipping position i from I when |I| ≥ 2, where δ = 0. From the discussions in Section III, the expected NUPC in uncorrected-error positions (i.e., positions in {j ∈ [0, n − 1] | ej = 1, i ≠ j}) will have the following properties: (a) if the selected flipping position i ∈ I is an actual error position (i.e. ei = 1), then the expected NUPC increases by the flipping, while (b) if the selected position i is an error-free position (i.e. ei = 0), then the expected NUPC decreases by the flipping. Hence, the NUPC in the uncorrected positions {j ∈ [0, n − 1] | ej = 1, i ̸= j} after the flipping will be useful to determine whether the selected flipping position i is correct or not. Figure 2 schematically illustrates the above property, where (a) is the NUPC before flipping, (b) is the NUPC after flipping of an error position (ei = 1), and (c) is that of an error-free position (ei = 0).

V. RE-EXECUTION OF LABF ALGORITHM WITH MODIFIED
CIPHERTEXTS
A. Concept This section presents a re-execution of the LABF algorithm
when Algorithm 2 fails to decode c = mG + e. In the re-

execution process, additional rounds of the LABF algorithm are performed with slightly modified ciphertext c′ = mG + e + e′, where e′ is a vector with a small weight ρ ≪ t. The nonzero positions in e′ = (e0′ , e1′ , . . . , en′ −1) are determined by the maximum NUPC vector α˜ = (α˜0, α˜1, . . . , α˜n−1) observed while the decoding of c, where α˜j is the maximum NUPC of position j observed throughout the decoding iterations. It is presumed that position j has an error with high probability if the value of α˜j is large. Hence, the set J = j e′j = 1 of nonzero positions in e′ is determined as a randomly selected subset of J ′ ⊂ [0, n − 1], where J ′ is the set of indices of the λ largest values in α˜, and |J | = ρ ≤ λ = |J ′|.
Figure 5 shows a numerical example by simulation for p = 4801, n0 = 2, wc = 45, and t ∈ {85, 90, 95, 100}. Here, the empirical error probability Pr(ej = 1|α˜j) is evaluated for each value of α˜j ∈ [0, wc]. The result shows that the error probability strongly depends on the value of α˜j, and ciphertext bit cj with a large α˜j has high error probability, e.g., the error probability is higher than 0.9 for α˜j ≥ 33.
B. Algorithm
The LABF algorithm presented in Algorithm 1 is extended to Algorithm 3, which re-executes the LABF when decoding failure occurs. The algorithm has a double-loop structure, where the outer loop simply decodes the same word c at most Iout times. Note that the decoding results may vary depending on the non-deterministic selection of flipping position ˜i in line 16 of Algorithm 2. The inner loop flips ρ bits in c using the maximum NUPC α˜ generated in line 6 of Algorithm 2. That is, the inner loop decodes c′ = c + e′ = ψflλp,ρ(c, α˜ ). Here, function ψflλp,ρ(c, α˜ ) defined by Algorithm 4 randomly selects a set J of ρ flipping positions from the candidate set J ′ of λ(≥ ρ) positions which correspond to the positions of the largest values in α˜ . Function ψsel(J ′, ρ) defined in Algorithm 4 randomly selects a subset J ⊆ J ′, where |J | = ρ ≤ λ = |J ′|. When the loop parameters are set to Iout = Iin = 1, Algorithm 3 is equivalent to Algorithm 1.

VII. CONCLUSION
This paper presented a look-ahead BF (LABF) algorithm, which is a modified BF decoding algorithm for MDPC code in which flipping positions are determined according to the relation between the NUPC in uncorrected error positions and the number of errors existing in the word under decoding. This also showed a re-execution of the proposed LABF algorithm based on the maximum NUPC observed in a failed decoding process. The BLER is evaluated by computer simulation, and the results show that the LABF gives the lowest BLER among the evaluated decoding algorithms.
The following points remain to be addressed in the future: theoretical optimization of decoding parameters (τ0, Iin, Iout, λ, and ρ), estimation of BLER in very low error rate region (e.g., 2−128 to 2−256 [32]), and reduction of the decoding complexity. The LABF will not be resistant to the reaction and timing attacks [22], [37], and thus countermeasures for the attacks will be required.
