A High-Efficiency Segmented Reconfigurable Cyclic Shifter for 5G QC-LDPC Decoder

Hing-Mo Lam , Silin Lu, Hezi Qiu, Min Zhang , Senior Member, IEEE, Hailong Jiao , Member, IEEE, and Shengdong Zhang , Senior Member, IEEE

Abstract— A reconfigurable cyclic shifter is a key element of a QC-LDPC decoder, which is crucial for 5G communication systems. If a traditional reconfigurable cyclic shifter can only shift one input of variable size at a time, a traditional QC-LDPC decoder can only decode one codeword at a time as well. Part of the circuitry of the traditional QC-LDPC decoder inevitably stays in idle during the decoding process if the length (or the lifting parameter) of a codeword is not the maximum, resulting in low hardware efficiency. A segmented reconfigurable cyclic shifter is proposed in this paper, which can be divided into multiple segments of different sizes. Each segment can perform a cyclic shift of an input of different sizes and of different shift values independently. Furthermore, a methodology is proposed to upgrade any state-of-the-art QC-LDPC decoder to a segmented QC-LDPC decoder, by using the proposed segmented shifter. The upgraded segmented QC-LDPC decoder is able to parallelly decode multiple codewords (or inputs) of different lengths at a time. A test chip of the proposed segmented QC-LDPC decoder with the proposed segmented reconfigurable cyclic shifter has been fabricated in a 0.18-µm CMOS technology to demonstrate the feature of parallelly decoding multiple codewords. The performance analysis shows that when the number of small codewords is increased from 0 to 100000 per second, the throughput of the traditional QC-LDPC decoder drops from 844.80 Mbps to 4.40 Mbps, while the QC-LDPC decoder with the proposed segmented shifter only slightly drops to 814.01 Mbps. By applying the segmented QC-LDPC decoder in 5G base stations, the base stations are enabled to support more low-traffic users.
Index Terms— 5G, codeword-parallel QC-LDPC decoder, edgeparallel QC-LDPC decoder, segmented cyclic shifter, segmented QC-LDPC decoder.
I. INTRODUCTION
Q UASI-CYCLIC Low-Density Parity-Check (QC-LDPC) code has been adapted as a part of standard in various wireless communication systems, such as Wi-Fi [1], [2], WiMAX [3], and 5G [4]. All these wireless communication systems require QC-LDPC decoder to decode a codeword of variable length. The length of the codeword is linearly proportional to the lifting parameter (Z) of QC-LDPC code.

A decoding process of QC-LDPC decoder is carried out by iteratively passing a bundle of cyclic or anti-cyclic shifted messages between check node cluster and variable node cluster. The bundle of messages includes the number of messages equal to the lifting parameter (Z). In Wi-Fi standards, there are three selectable values (27, 54, and 81) for the lifting parameter. In 5G standard, there are 51 selectable values of the lifting parameter (Z) between 2 and 384.
A reconfigurable cyclic shifter is a key element of a QC-LDPC decoder, which is crucial for 5G communication systems. In 5G standard, a QC-LDPC decoder is required to decode a variable-length codeword with a lifting parameter from 2 to 384. The size of a codeword is linearly proportional to the value of the lifting parameter. The traditional QC-LDPC decoder design has two issues. The first issue is that the traditional QC-LDPC decoder can only decode a codeword of variable length at a time because a traditional reconfigurable cyclic shifter is typically adopted, which can only perform a cyclic shift of an input of variable size at a time. Therefore, part of the traditional QC-LDPC decoder stays in idle during the decoding process, degrading the hardware efficiency. The second issue is that the decoding latency of a codeword is independent of the length of a codeword (or the value of lifting parameter), but depends on the number of the non-negative entries in the selected Base-Graph (BG) matrix. Two issues together cause a serious impact on the throughput of a base station (e.g. 5G base station), especially when the base station receives too many small codewords, because small codeword contains fewer information bits but consumes the same amount of latency for decoding.
The multi-way cyclic shift network is proposed in [5] which can only support Wi-Fi standards because only three choices of lifting parameter, 27, 54, and 81, are supported. The design in [6] can support 5G standard. 51 different selections of lifting parameter between 2 and 384 are supported. Both designs in [5] and [6] are constructed based on the bene network. The disadvantage of this approach is that the delay is usually larger than other solutions because there are more logic levels between input and output. The controller design is also complicated because the control signals for a huge number of switch elements need to be generated in real time. Solutions in [7] and [8] are built from a barrel shifter. Two shifters are employed. One shifts the input data to the right, while the other one shifts to the left. Then a combiner combines the outputs of the two shifters to form cyclic shifted output of input data with variable data size. However, they can only perform a cyclic shift of one input data of variable size at a time. Part of hardware stays in idle if the input data size

However, the data size and shift value must be the same for the different input data. A fully parallel circular-shift rotation network is proposed in [22], which is based on generic barrel shifter (GBS) core block. The aodd-GBS core block can perform cyclic shift of any input data size smaller than or equal to aodd. The value of aodd is the greatest odd divider (g.o.d.) of a set of predefined input data sizes. The issue of the design in [22] is that only a limited set of predefined input data sizes is supported, not any size within the limitation of the hardware. The control signal generation for the trading stages of the design in [22] is also complicated, which requires ROM (read only memory) to pre-store the control signals. Furthermore, another set of a bypass Banyan network and MUX-stage to merge outputs of the original or the bypass network is introduced to perform cyclic shift of predefined set of input data size by carefully computing the control signals of all switches. The design in [23] can only support data size of 27, 54, and 81 for Wi-Fi standard, and 24 to 96 for WiMAX standards. [24] further extends the design in [23] to support 5G standard which requires to handle 51 input data sizes between 2 to 384. The shifter designs in [22]–[24] all have an issue that they can only parallelly handle multiple inputs of predefined limited set of data sizes.
In this paper, a segmented reconfigurable cyclic shifter is proposed [21]. Unlike the traditional designs [9], [22]–[24] that can only parallelly perform cyclic shift of limited set of predefined data sizes, the proposed design can parallelly perform cyclic shift of multiple data of any different sizes. The control signal generation of the proposed design is relatively easy, without need to pre-store the control signals in ROM. Furthermore, the proposed design offers an option to tradeoff the degree of parallelism with the number of control signals. The proposed shifter can be divided into multiple segments of different sizes by separating the control signals of each path and then re-group those paths (or control signals) together to form the final segments.
The previous works [9], [22]–[24] only discussed the design of cyclic shifters. None of them have unveiled how to apply their cyclic shifters to a QC-LDPC decoder to parallelly decode multiple codewords. In this paper, a methodology is presented to apply the proposed segmented reconfigurable cyclic shifter to upgrade a traditional QC-LDPC decoder to a segmented QC-LDPC decoder. The proposed segmented QC-LDPC decoder can be further categorized into a codeword-parallel QC-LDPC decoder and edge-parallel QC-LDPC decoder. The codeword-parallel QC-LDPC decoder can parallelly decode multiple codewords to prohibit received small codewords from affecting the overall throughput of QC-LDPC decoder. The edge-parallel QC-LDPC decoder can parallelly process multiple bundles of edges of a Base Graph matrix (or QC-LDPC code) in the Tanner Graph presentation to reduce the overall decoding latency of a codeword. The decoding efficiency of the segmented QC-LDPC decoder is thereby enhanced significantly. The reduction of decoding

latency is important for V2X (Vehicle-to-every) applications. As a key element of 5G communication systems, the proposed codeword-parallel QC-LDPC decoder can facilitate 5G base stations to support more low-traffic users. The contributions of this work are as follows.
• A segmented reconfigurable cyclic shifter is proposed which can be divided into multiple segments of different sizes and perform cyclic shift of multiple input data in parallel.
• A methodology of upgrading a traditional QC-LDPC decoder to a segmented QC-LDPC decoder is proposed to parallelly decode multiple codewords for 5G base station to support more users or process multiple bundles of edges to reduce the decoding latency.
• An Early Termination module for the codeword-parallel QC-LDPC decoder is presented. The module can independently detect which segments have triggered the early termination condition, and then freeze the decoded results, allowing the other segments to continue the next iteration of the decoding process.
This paper is organized as follows. In Section II, both the traditional reconfigurable cyclic shifter and the proposed segmented reconfigurable cyclic shifter are introduced. In Section III, a methodology is proposed to apply the proposed shifter to upgrade a traditional QC-LDPC decoder to a segmented QC-LDPC decoder. The Tanner Graph presentation, the “slice” concept of QC-LDPC decoder, and the proposed segmented QC-LDPC decoder are presented in Section III. The comparison results and test chip measurement results are shown in Section IV. The paper is finally concluded in Section V.
II. RECONFIGURABLE CYCLIC SHIFTERS
In this section, the traditional reconfigurable cyclic shifter is introduced first, followed by the proposed segmented reconfigurable cyclic shifter [21]. Through the “path” concept, the difference between the traditional and the proposed designs is explained.

An example of a traditional reconfigurable cyclic shifter is shown in Fig. 1. The shifter can support a maximum input data size (ZMAX) up to 8. B. The Proposed Segmented Reconfigurable Cyclic Shifter
The “path” concept of a logarithmic shifter is introduced in this section, followed by the design details of two versions of the proposed segmented reconfigurable cyclic shifters through examples.
1) The “Path” Concept of a Logarithmic Shifter: The realization of the proposed segmented reconfigurable cyclic shifter is based on the “path” concept of a logarithmic shifter. The traditional design cannot divide a logarithmic shifter into multiple segments due to the lack of the “path” concept. The traditional design applies the same set of control signals to all paths of the shifter. The proposed design separates the control signal of each path, and then re-groups the consecutive paths to form a segment by applying the same set of control signals to all paths in the same segment. Therefore, different segments can perform different shift values of different input sizes independently. There is an example of a “path” in Fig. 2b, which is Path 7 of the right logarithmic shifter in Fig. 2a. The set of control signals of Path 7 are the control signals for Stages 1, 2, and 3 of Path 7, which are SL_1, SL_2, and SL_3, respectively, in Fig. 2b.
2) Base Version of Segmented Reconfigurable Cyclic Shifter: The example of the proposed segmented reconfigurable cyclic shifter is illustrated in Figs. 3 and 4, which is an extension of the example in Figs. 1 and 2. Recall the example in Fig. 1 and Fig. 2a, the input data size Z = 5, which is less than the maximum supported input data size ZMAX = 8. Therefore, a portion of the cyclic shifter stays in idle during the process of cyclic shift operation. For example, Paths 5, 6, and 7 of the left and right shifters are idle. Input X[7:5] and output Y[7:5] are thereby ignored. These idle paths now have been re-activated by feeding the second input data IN2[2:0] in Fig. 3. The size of the second input data (Z2) is 3, which needs to be performed cyclic shift two positions to the right (CSR2[2:0] = 2).

III. SEGMENTED QC-LDPC DECODER
In this section, a methodology is presented to apply the proposed segmented cyclic shifter to a traditional QC-LDPC decoder to upgrade to a segmented QC-LDPC decoder. Note that the proposed segmented cyclic shifter can be applied to any state-of-the-art QC-LDPC decoder to upgrade to a segmented QC-LDPC decoder. This upgrading does not reduce the maximum throughput that can be achieved by the state-of-the-art QC-LDPC decoders, but provides the decoders an additional option to parallelly decode multiple codewords (codeword-parallel QC-LDPC decoder) or parallelly process multiple bundles of edges of a Tanner Graph (edge-parallel QC-LDPC decoder). The codeword-parallel QC-LDPC decoder can prohibit small codewords from affecting the overall throughput of a QC-LDPC decoder. It is important for 5G base stations to support more low-traffic users. The edge-parallel QC-LDPC decoder can reduce the overall decoding latency of a small codeword. The reduction of decoding latency is important for V2X (Vehicle-to-every) applications.

A. Tanner Graph Presentation and Decoding Algorithm
The Tanner Graph presentation of Base Graph matrix (or QC-LDPC code) is critical for a digital circuit designer to understand the decoding process of a QC-LDPC decoder. The Tanner Graph is derived from a Base-Graph (BG) matrix and a lifting parameter (Z). A small example of 2 × 4 BG matrix and its corresponding Tanner Graph is illustrated in Fig. 6. Although the matrix size in the example is only 2 × 4, the size can be easily extended to 42 × 52 or 46 × 68 of 5G standard without loss of generality, while the decoding process is exactly the same regardless of the size of the BG matrix. Bi,j is an entry of the ith row and the jth column of the BG matrix. If the value of the lifting parameter (Z) is 1, the row and column of the BG matrix represent a Check Node (CNi) and a Variable Node (VNj) of a Tanner Graph, respectively. The non-negative entry (Bi,j) of the BG matrix represents there is an edge connecting the Check Node (CNi) and Variable Node (VNj). If the lifting parameter (Z) is larger than 1, the row and column of the BG matrix represent Check Node cluster (CNCi) and Variable Node cluster (VNCj) of a Tanner Graph, respectively. The number of edges in the bundle of edges is equal to the lifting parameter (Z). During the decoding process, each edge carries a message, a bundle of edges carries Z messages. B. The “Slice” Concept of QC-LDPC Decoder
The Tanner Graph presentation of Base Graph matrix (or QC-LDPC code or QC-LDPC decoder) with lifting parameter Z = 1 is shown in Fig. 7a, which can be viewed as a slice. In this case, the message does not need to be cyclic or anti-cyclic shifted when the message is passed between CNi and VNj along the edge. The QC-LDPC decoder, with lifting parameter (Z) larger than 1, can be viewed as a stack of Z slices. The case of a QC-LDPC decoder with lifting parameter Z = 384 is shown in Fig. 7c, which can be viewed as a stack of 384 slices. In this case, the Z messages need to be cyclic or anti-cyclic shifted when passing back and forth between CNCi and VNCj. The cyclic or anti-cyclic shift value is the value of the corresponding entry (Bi,j) of the BG matrix. For example, in Fig. 7b, the bundle of messages passing along edges connecting CNC1 and VNC1 is cyclic or anti-cyclic shifted by the value of B1,1 = 1. Namely, the 384 messages are cyclic-shifted by one position to the left when passing from VNC1 to CNC1, and anti-cyclic-shifted by one position to the right when passing from CNC1 to VNC1 by following the bundle of edges. The cyclic or anti-cyclic shift of the bundle of edges (or the bundle of messages) is performed by a reconfigurable cyclic shifter.
In 5G standard, there are totally 51 different selectable values of lifting parameter (Z) from 2 to 384. The traditional “fully-parallel” QC-LDPC decoder prepares maximum 384 slices to process 384 messages once at a time to maximize the throughput. The traditional “partially-parallel” QC-LDPC decoder prepares slices less than 384, then may take multiple times to process a bundle of messages if the number of slices is less than the lifting parameter (Z).
The front view and side view of the stack of 384 slices are shown in Fig. 7c and Fig. 7d, respectively. The front view (Fig. 7c) does not draw the reconfigurable cyclic shifter. The cyclic or anti-cyclic shift of a bundle of edges is equivalent to shuffle the corresponding messages among Z slices. That also means an edge can pass from one slice to another slice. Furthermore, the message that propagates along the edge also passes from one slice to another slice. Besides the operation of cyclic or anti-cyclic shift that is involved to shuffle the messages among different slices, for all the other operations of the decoding process, the message is processed within the slice to which the message belongs.
From the side view of QC-LDPC decoder in Fig. 7d, if a traditional reconfigurable cyclic shifter is used and the lifting parameter (Z) is less than 384, there are only Z slices in active while (384 - Z) number of slices in idle

C. The Proposed Segmented QC-LDPC Decoder
The “slice” concept in the previous section shows that, except for the operation of the cyclic shift, all the other operations of the decoding process are performed within the same slice. The proposed segmented reconfigurable cyclic shifter in Section II can be divided into multiple segments of different sizes. Each segment can perform cyclic shift independently with different shift values. By applying the proposed segmented shifter, the 384 slices of a traditional QC-LDPC decoder can be divided accordingly and upgraded to a segmented QC-LDPC decoder. Each segment of the segmented QC-LDPC decoder can process one bundle of edges (or process one bundle of messages). Below are two applications of the segmented QC-LDPC decoder, i.e., codeword-parallel QC-LDPC decoder and edge-parallel QC-LDPC decoder.
1) Codeword-Parallel QC-LDPC Decoder: An example of the codeword-parallel QC-LDPC decoder is illustrated in Fig. 8. This example parallelly decodes two small codewords. In Fig. 8, two Tanner Graphs of QC-LDPC decoder of two codewords are shown. They are Codeword 1 and Codeword 2 with lifting parameter Z_1 and Z_2, respectively.

In the middle of Fig. 8, the side view of the segmented QC-LDPC decoder is shown. The 384 slices of the segmented QC-LDPC decoder is divided into two segments, Segment 1 and Segment 2 of size Z_1 and Z_2, respectively. The rest un-used slices just stay in idle. As shown in Fig. 8, Segments 1 and 2 can simultaneously process two bundles of edges from Tanner Graphs of Codewords 1 and 2.
The 5G standard supports two different sizes of BG matrices, 42×52 and 54×68. Each size supports eight BG matrices and 51 selectable values of lifting parameter. The index of eight BG matrices and their corresponding selectable lifting parameters are shown in Table I. For example, the second row in the table represents BG matrix index 0 and the BG matrix supports lifting parameter 2, 4, …, 256. The segmented QC-LDPC decoder in Fig. 8 can be divided into more segments to parallelly decode more codewords as long as the sum of lifting parameters of all codewords is smaller than or equal to 384.
The hardware overhead is minimized if the multiple codewords are limited to be from the same BG matrix. Take the example of the matrix size 46 × 68, BG index 0, the supported lifting parameters of which are 2, 4, …, 256. If two codewords select lifting parameters from the above supported lifting parameters of the BG index 0, all operations of the decoding process of two codewords can be fully synchronized. They can complete processing the Min-stage and Sum-stage at the same time. The hardware overhead is the design of the controller of the segmented reconfigurable cyclic shifter to generate the different control signals of different segments.

The design of Early Termination module for the codewordparallel QC-LDPC decoder is required to independently detect which segment triggers the early termination condition and freeze its decoded output. The decoding process must be terminated once the decoded result is identified as a valid codeword, further iteration of decoding may alter the decoding result and cause decoding failure. Thus, the Early Termination module for the codeword-parallel QC-LDPC decoder is required to be able to detect which codeword(s) (or which segment(s)) have finished decoding, then freeze the decoded output of those segment(s) and let the other unfinished codeword(s) go on the next iteration of decoding.

V. CONCLUSION
The capability of a QC-LDPC decoder to decode multiple small codewords in parallel is crucial for 5G base stations to support more low-traffic users. A segmented reconfigurable cyclic shifter is proposed in this paper, which is realized by separating the control signals of different paths of the logarithmic shifter and then recombining those paths (or control signals) together to form one segment. The proposed shifter can be applied in any state-of-the-art QC-LDPC decoder to upgrade to a segmented QC-LDPC decoder to either parallelly decode multiple codewords or to parallelly process multiple bundles of edges of a Tanner Graph of a Base-Graph matrix. A test chip of the codeword-parallel QC-LDPC decoder, which employs the sectioned version of the proposed segmented reconfigurable cyclic shifter, is fabricated in a 0.18-µm CMOS technology and can successfully decode multiple codewords in parallel. The proposed high-efficiency segmented reconfigurable cyclic shifter, together with the proposed segmented QC-LDPC decoder, is therefore an attractive option for various applications in 5G base stations.
