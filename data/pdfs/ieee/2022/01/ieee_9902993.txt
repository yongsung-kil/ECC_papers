ABSTRACT An error floor phenomenon, decoding performance, and throughput are three major concerns for LDPC decoders in NAND Flash applications. With a penalty method and an active iteration mechanism, we present a Unified Penalty Gradient Descent Bit Flipping (UP-GDBF) decoding algorithm, which not only possesses error-floor free property but also improves convergence speed in decoding performance. To fulfill the high-throughput requirement while maintaining reliable error correction capability, we propose an energy-based backtracking scheme to reduce 40% latency with a negligible 0.8% area overhead. Implemented in TSMC 16nm process, the proposed 4KB LDPC decoder can achieve a throughput of 19.3 Gbps with 0.120 mm2 area to satisfy ONFI 5.0 throughput requirement. Compared to existing approaches, our decoder architecture provides superior data rate and decoding performance in both 1KB and 4KB LDPC codes.

I. INTRODUCTION
Q UASI-CYCLIC Low-Density Parity Check (QC-LDPC) code has been widely used as a mainstream LDPC code [1] in NAND flash applications. In order to cover a wide range of aging NAND channel conditions, a multidecoder error correction subsystem becomes a heterogeneous solution to balance the trade-off among decoding performance, throughput, power, and area. Among numerous decoding algorithms, min-sum decoders [2], [3] provide excellent decoding performance at the expense of hardware complexity and power consumption. On the contrary, bitflipping decoders [4], [5] feature lightweight computation by sacrificing decoding capability.
Over the years, the performance of bit-flipping decoding has been improved in several studies [6], [7], [8], [9], [10], [11], [12], [13]. Gradient Descent Bit Flipping (GDBF) [6] firstly interpreted the decoding procedure as a maximum likelihood optimization problem, and it showed a significant performance gain and inspired many following works. Probability GDBF (PGDBF) [7] introduced

randomness in the decoding procedure so the decoding process could escape from local maximum traps. Tabu-list aided PGDBF (T-PGDBF) [8] adopted the tabu-search idea to avoid repeated flipping in the decoding process and was well-improved in [9], [10], [11]. The recent counter random GDBF (CRGDBF) [12] and PGDBF with Momentum (PGDBF w/M) [13] added the flipping histories to improve the performance further.
Flipping decision rules categorize the bit-flipping decoding into global searching [7], [8], [9], [10], [11], [12], [13] and non-global searching [4], [5] decoding. Since globalsearching decoding uses the maximum value to determine flipping bits instead of the estimated threshold, it generally provides better decoding performance at the cost of hardware complexity and decoding latency. Although the global-decoding algorithms are competitive in small code length QC-LDPC code, little is known about their effectiveness in large code length QC-LDPC. Moreover, the existing studies for bit-flipping decoder design in NAND flash applications are limited to non-global searching decoding.

II. BACKGROUND An (N, K) QC-LDPC code in circular permutation matrices (CPM) size Z is defined by a parity check matrix (H) of size M × N, where N, K, and M represent the length of the codeword, information, and syndrome, respectively. Due to quasi-cyclic properties, the matrix is viewed as a GN × GM array of Z × Z sub-matrices, where GN = (N/Z) and GM = (M/Z) are the numbers of column and row blocks. As visualized in a bipartite graph, each element (hj,i) in H matrix represents the existence of edge connecting between the j th check node (cj) in rows and the i th variable nodes (vi) in columns. The number of non-zero elements in a column is called column degree (dc).
NAND vendors nowadays have been stacking more layers of memory cells over 3D NAND; the decreasing lifetime makes 4KB QC-LDPC code a popular choice for its stronger error correction capability. In this work, we adopt an (N = 36864, K = 33028, M = 3840) QC-LDPC code with Z = 256 and dv = 5 as our design target. Since bit flipping is a hard-decision decoding method, it is more precise to adopt the fixed error bit count (EBC) channel (i.e., all testing frames contain the same error bit count in each channel condition but with different error locations) to compare decoding performance in frame error rate (FER).
The penalty method is a way to transform the original constrained optimization problems into the unconstrained optimization functions. This paper utilizes this method in both decoding analysis and algorithm improvement. The

becomes an affordable choice in decoder design. Our 4KB QC-LDPC code is divided into 144 column blocks, and each column block represents processing 256 bits in parallel. Based on this QC-LDPC code structure, it takes 144 cycles to search for the maximum energy value and additional 144 cycles to accomplish flipping and syndrome updates.
The decoder is partitioned into four stages of pipeline architecture in Fig. 5. The first stage is the instruction fetch for sub-module configurations. The second stage is data loading, which separates the memory macro’s access time from the original critical path to boost operating frequency. The third stage consists of the energy computation unit cluster (ECU cluster), value collapse unit, energy-based backtracking unit, and flipping indicator unit. They generate the energy values and flipping vector. The last stage is partitioned to update the global maximum value in the maximum finding phase and update syndromes in the flipping phase (the energy-based backtracking macro will be detailed at the end of this section).
A. MEMORIES AND BARREL SHIFTERS To fully exploit the structure of the QC-LDPC code, the memory macros’ bandwidth equals the CPM size, and the number of entries is the number of block columns. Our design uses a ROM to store the configuration parameters for each column (e.g., The address for syndrome row block selection and the shifting value for barrel shifters). Oneport and two-port register files are deployed for the initial codeword and the estimated codeword to generate correlation penalties. The flipping penalties are also stored in the twoport register file. There are two sets of barrel shifters. The C2V barrel shifters align the selected syndromes to the ECU cluster, and the V2C barrel shifters align the flipping vector to the selected syndrome.

B. RANDOM PENALTY GENERATION UNIT The random penalty generation unit in Fig. 6 consists of a 256-bit iteration-dependent random state, a 16-bitwise block interleaver, and a configurable barrel shifter. Before active iteration, the current random penalty is zero to diminish erroneous randomness effect as mentioned in (5). Considering the randomness’ consistency between the flipping phase and the maximum energy finding phase, we make the random penalty bits depend on the random state (Lock Mode) and the shifting offset for each column block. Moreover, the oneto-one mapping property guarantees the number of ones to be the same as the random state in the initial setting. At the end of the flipping phase, the generated random penalty bits would be collected (Trap Mode) as the new random state for the next iteration.

E. ENERGY-BASED BACKTRACKING UNIT A visualization of the flipping process over a frame under the EBC=120 condition is shown in Fig. 8. Due to the maximum energy value Emax dependency (line 11 in Algorithm 3) in each iteration, the decoder needs to access all column blocks twice - one for the maximum finding phase and the other for the flipping phase. Since there is plenty of redundant access (blue cell) in the flipping phase, it requires a mechanism to save those idle cycles and boost the decoder throughput.
The energy-based backtracking scheme is proposed to leave the breadcrumb along the decoding trail in the maximum finding phase so that this information could navigate the flipping process to access those red cells only. As illustrated in Fig. 9, the energy-based backtracking unit consists of two priority decoders, eight classification entries, backtracking memory, and the related data path control. In the maximum finding phase, the decoder would iterate 144 column blocks. One of the priority decoders would generate the local maximum (Local Max) as a label to categorize the current column index into a specific entry. The original content of the selected entry would be pushed into the energy-based backtracking memory to build up the connection between the column indices sharing the same energy values. In the

At the beginning of the maximum finding phase, eight classification entries are initialized to −1, which denotes the termination flag in the flipping phase. As termination flag (-1) is encountered, the decoder ends the flipping state and proceeds to the syndrome check state for early termination.
In the transition from the maximum finding phase to the flipping phase, two pipeline bubbles (NOP) are inserted in the decoder scheduling to avoid the data hazard in the pipeline architecture, as illustrated in Fig. 11. Although there are three-cycle overhead in pipeline bubbles (NOP) and termination (-1), the proposed method can avoid all unnecessary access in the flipping phase and reduce overall decoding cycles. Fig. 12 plots the average cycle count for each channel condition. It suggests that the proposed method can achieve more than 42% cycle reduction, equivalent to 1.72 times throughput enhancement, above 110 fixed-error-bit channel conditions.

Because the LDPC code’s error correction capability is different in different code lengths, 1KB and 4KB UPGDBF decoders are implemented in a 90nm process to make a fair comparison. In 1KB LDPC code, the proposed UPGDBF provides a 1.23 higher data rate and 152 times more decoding performance than the modified gradient descent bit-flipping (MGDBF) decoder [4] in the EBC=40 channel condition.
Regarding the 4KB LDPC code, since there is no partial parallel architecture implementation in global searching bit-flipping decoders, a 4KB PGDBF w/M decoder is implemented without a backtracking mechanism for detailed comparison. In the EBC=110 conditions, the global searching decoders are beyond 104 times more reliable than the non-global searching decoder. This reliability gain gives the global searching methods an advantage in NAND flash applications. Without a global searching process, the multi-bit flipping decoder (MBF) [5] directly flips the candidate bits with energy values above a predefined threshold. Thus, the MBF decoder provides a higher data rate than the PGDBF w/M decoder.
Among all 4KB bit-flipping decoders, the proposed 4KB QC-LDPC decoder provides the highest data rates for two reasons. Firstly, the UP-GDBF decoding algorithm presents a higher convergence speed than all the state-of-the-art global searching decoding algorithms. In EBC=110 conditions, the UP-GDBF reduces nearly 20% from the PGDBF w/M’s ten iterations. Secondly, the energy-based backtracking scheme can provide an additional 1.72 times data rate enhancement in the decoder. As a result, the proposed UP-GDBF decoder outperforms the others in data rates and decoding performance under the same channel conditions.

VI. CONCLUSION
In this paper, we apply the penalty method to analyze
the error-floor phenomenon and propose a Unified Penalty
Gradient Descend Bit Flipping decoding algorithm with
10% - 20% convergence improvement. The corresponding
decoder architecture not only possesses the advantage of
global searching algorithms in decoding performance but
also overcomes the throughput bottleneck with the energy-
based backtracking scheme. Implemented in TSMC’s 16nm
process, the proposed 4KB UPGDBF decoder can achieve
a throughput of 19.3 Gbps to satisfy the requirement of the
ONFI 5.0 interface. The proposed algorithm and backtrack-
ing mechanism result in significant enhancements that allow
our decoder to provide a higher data rate and more than two
order performance gain in both 1KB and 4KB LDPC codes.

HSIE-CHIA CHANG (Senior Member, IEEE) received the B.S., M.S., and Ph.D. degrees from the Electronics Engineering Department, National Chiao Tung University (currently, National Yang Ming Chiao Tung University), Hsinchu, Taiwan, in 1995, 1997, and 2002, respectively.

He has authored a book, over 120 IEEE journal/conference papers, and over 70 U.S./China/Taiwan patents. His research interests include algorithms and VLSI architectures in signal processing, in particular, error control codes and cryptosystems. Dr. Chang was a recipient of the Outstanding Youth Electrical Engineer Award from the Chinese Institute of Electrical Engineering in 2010, and the Outstanding Youth Researcher Award from the Taiwan IC Design Society in 2011. He has served as the Deputy Director General with the Chip Implementation Center, Taiwan, from 2014 to 2017.
