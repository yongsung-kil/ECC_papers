Abstract—In this paper, we propose a new construction of product codes, where the whole information array is protected row-by-row by a low-density parity-check (LDPC) code while only a portion of the information array is protected column-bycolumn by an algebraic code. The most distinguished feature of the proposed product code is that, thanks to the freeride coding technique, the additional column check bits are transmitted implicitly rather than explicitly. The constructed codes are referred to as implicit partial product-LDPC codes, which have the same rates as the row component LDPC codes. The decoding algorithm can be divided into four stages, including decoding of the free-ride codes, first-round decoding of the row codes, decoding of the column codes, and second-round decoding of the row codes by exploiting the messages associated with those successfully decoded columns. To predict the extremely low error rate of the doubly-protected (by both the row code and the column code) information bits, we derive an approximate upper bound. The simulation results show that, with a (3, 6)-regular LDPC code of length 1024 as the component code, the proposed product code can lower the word error rate (WER) from 10−2 down to 10−6 at the SNR around 2 dB. The numerical results also show that the doubly-protected information bits are more reliable, which can have a bit error rate (BER) down to 10−15 at SNR around 2.6 dB as implied by the presented approximate upper bound.
Index Terms—Free-ride codes, LDPC codes, product codes.
I. INTRODUCTION
In this paper, we focus on constructing low-density paritycheck (LDPC) codes for the transport block (TB)-based transmission scheme adopted in both LTE and 5G. The TB, i.e., MAC layer information block, is segmented into multiple code blocks (CBs) for channel coding at the physical layer, where each CB is independently encoded into a coded block (codeword) and multiple coded blocks are grouped into a transmitted block. This mechanism can minimize the hybrid automatic repeat request (HARQ) feedback overhead but may lead to a waste of transmission power and bandwidth since a single erroneous CB will cause the retransmission of the whole TB in LTE [1] or the whole CB group (CBG) in 5G [2].
A direct way to improve the resource utilization efficiency is to improve the error-correction performance. This can be achieved by taking the advantage of the natural mechanism of the TB-based transmission scheme. Specifically, the errorcorrection performance can be improved by introducing extra

Product codes, proposed by Elias in 1954 [3], are a classical method to improve the performance by introducing constraints across the row codewords. Classical product codes are obtained by placing the information bits in an information array and then encoded first by row codes and then by column codes, or vice versa. Due to the reasonable complexity and high performance, product codes are currently adopted in various communication standards, e.g., the IEEE 802.16 [4], the IEEE 802.20 [5] and the IEEE-1901 [6]. Additionally, the product codes taking LDPC codes as the row codes and high rate algebraic block codes as the column codes were proposed in [7]–[10], where the algebraic codes are introduced to improve the performance of the LDPC codes. However, the additional checks generated by the column codes will bring a notable rate reduction as compared with the row component codes. In addition, due to the additional checks, the transmitted frame structure need to be adjusted, causing inconvenience for the practical design.
In this paper, we present the implicit partial product-LDPC codes for the TB-based transmission protocol, where the whole information array (i.e., TB) is protected by a row component LDPC code while only partial data of the information array is further protected by a column component algebraic code. The partial column check bits are transmitted implicitly rather than explicitly by using the free-ride coding [11], [12], which requires neither extra transmission power nor extra bandwidth. Distinguished from the conventional product codes, thanks to the free-ride coding, the presented product-LDPC codes have the same rates as the row component LDPC codes. For encoding, the coded block length and the transmitted block length keep unchanged as compared with the conventional TB-based transmission scheme. For decoding, we present a four-stage decoding algorithm. At the first stage, we perform the freeride decoding to recover the column checks and then remove the effects of the (estimated) column checks on the received vector. At the second stage, we perform the first-round row decoding (LDPC decoding). At the third stage, if necessary, we perform the column decoding (algebraic decoding) to recover residual errors in doubly-protected (by both the row code and the column code) information bits. Additionally, the messages associated with those successfully decoded columns are updated. At the fourth stage, we perform the secondround row decoding for the unrecovered CBs using the updated

II. PROBLEM STATEMENT AND RELEVANT TECHNOLOGIES
A. Problem Statement
Let F2 = {0, 1} be the binary field. Let C [n, k] be an LDPC code of length n and dimension k, whose paritycheck matrix is denoted by H. For the simple LDPC coded transmission link, upon receiving the noisy sequence, an iterative belief propagation (BP) decoding algorithm, say, the sum-product algorithm (SPA) [13] can be performed to recover the information data. Obviously and not surprisingly, if a few coded bits of length kℓ are told by a genie to the decoder, the overall decoding performance of the LDPC coded link can be definitely improved. This is confirmed by the following simulations.
Example 1: We take a rate-1/2 (3, 6)-regular LDPC code of length 1024 for the simple LDPC coded transmission. Codewords are transmitted using binary phase-shift keying (BPSK) modulation over additive white Gaussian noise (AWGN) channels. At the receiver, the SPA with a maximum iteration number of Imax = 50 is employed for the decoding. The WER performance with kℓ known bits is shown in Fig. 1, where we observe that the overall performance of the simple LDPC coded link can be improved with the known bits. We also observe that the LDPC coded transmission with more known bits can be more reliable, as is consistent with our intuition.
An interesting question arises. Is it possible to find a “genie” who tells some known bits without any increase in transmission power and bandwidth. This can be partially achieved by applying the free-ride coding to product codes.

B. Conventional Product Codes
Let C [n, k] be the row code with length n and dimension k and C1[n1, k1] be the column code with length n1 and dimension k1. A codeword of a conventional product code is formatted as an array of size n1 × n, where each row is a codeword of the row code C , and each column is a codeword of the column code C1. See Fig. 2 for reference.
The minimum distance of a product code is greater than those of the component codes, indicating a stronger capability for error correction. However, the column checks and the checks on checks will bring a notable rate reduction of k/n − (kk1)/(nn1) as compared with the row component codes. In addition, due to the additional checks, the transmitted frame structure need to be adjusted, causing inconvenience for the practical design. Concretely, for a given TB, the transmitted block length becomes longer if the conventional product codes are used.
C. Free-Ride Codes
The free-ride codes were proposed in [11], [12], where a few extra bits can be transmitted over an existing LDPC coded transmission link but without any cost of extra transmission energy or extra bandwidth. The basic idea is to transmit the extra bits over the original coded link in a superposition (XOR) manner.
Formally, let u ∈ Fk2 be the payload data need to be transmitted. Let Cp[n, k] be an LDPC code, referred to as payload code, whose parity-check matrix is denoted as H. Without extra bits, u is encoded by the encoding algorithm of Cp[n, k], resulting in an LDPC codeword v ∈ Fn2 such that vHT = 0. Then the LDPC codeword v is transmitted over a binary-input output-symmetric (BIOS) memoryless channel, resulting in a received sequence y. At the receiver, upon receiving y, the decoding algorithm of the LDPC code is performed to recover the payload data u.
Now we assume that, in addition to the payload data, a few extra bits of length ℓ ≪ k need to be transmitted. To achieve this, one approach is to build an extra coded link for the extra bits, which requires extra bandwidth. The other approach is to construct a new code [n, k+ℓ], which may cause inconvenience due to the complete reconstruction of the coded

III. IMPLICIT PARTIAL PRODUCT-LDPC CODES
As illustrated in Sec. II-A, the overall performance of the LDPC code can be improved with the help of the known bits. For the TB-based transmission scheme, we can use the product-like construction to improve the performance, where the messages associated with those successfully decoded columns can be viewed as nearly “known” messages. These nearly “known” messages in turn improve the overall performance.
More specifically, as illustrated in Fig. 5, we construct the implicit partial product codes, where the row code is an LDPC code and the column code is a high rate algebraic code, say, BCH code. Partial column information sequences are selected for column encoding, resulting in a few column checks. To avoid the notable rate reduction caused by the additional checks, the generated column checks are transmitted implicitly rather than explicitly using the free-ride coding along with the original LDPC coded blocks. The encoding and decoding of the presented product-LDPC codes are describe as follows.

A. Encoding of Implicit Partial Product-LDPC Codes

V. CONCLUSIONS
This paper has presented the implicit partial product-LDPC codes for the TB-based transmission, where the partial column parity-check bits are transmitted implicitly rather than explicitly using the free-ride coding. More distinguishedly, the presented product-LDPC codes share the same rates with the row component LDPC codes, avoiding the rate reduction caused by the additional column checks for the conventional product codes. For encoding, the lengths of the coded block and the transmitted block keep unchanged as compared with the conventional TB-based transmission scheme. For decoding, we have presented an efficient four-stage decoding algorithm, which has a similar decoding complexity to the benchmark component LDPC codes in the high SNR region of interest.
