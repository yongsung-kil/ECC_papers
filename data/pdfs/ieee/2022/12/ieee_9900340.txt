Neural Min-Sum Decoding for Generalized LDPC Codes
Hee-Youl Kwak , Member, IEEE, Jae-Won Kim , Member, IEEE, Yongjune Kim , Member, IEEE, Sang-Hyo Kim , Member, IEEE, and Jong-Seon No , Fellow, IEEE

Abstract— In this letter, we investigate the min-sum (MS) and neural MS (NMS) decoding algorithms for generalized low-density parity-check (GLDPC) codes. Although the MS decoder is much simpler than the a posteriori probability (APP) decoder commonly used for GLDPC codes, the MS decoder has not been considered mainly due to its inferior decoding performance. However, we show that the performance can be improved by i) employing the NMS decoding algorithm and ii) optimizing the component parity check matrix (PCM). For the four representative short GLDPC codes in the literature, experimental results show that the NMS decoding performance with the optimized component PCM significantly outperforms the MS decoding performance and even outperforms the APP decoding performance for some cases.
Index Terms— Generalized low-density parity-check (GLDPC) code, min-sum (MS) decoding, neural min-sum (NMS) decoding.
I. INTRODUCTION
G ENERALIZED low-density parity-check (GLDPC) codes incorporate generalized constraint (GC) nodes defined by arbitrary component codes as well as single parity check (SPC) nodes to protect variable nodes (VNs) [1]. The conventional message passing decoding scheme for GLDPC codes, called a posteriori probability (APP) decoding, operates a set of APP processor for each node [2]. Whereas the APP operation for SPC nodes is well approximated by the simple min operation of the min-sum (MS) decoding algorithm, the APP processor for GC nodes requires a high computational complexity which is exponentially proportional to the code dimension of the component code [2]. Even though bit-wise maximum a posteriori (MAP) algorithms such as the BCJR algorithm [3] are adopted for GC nodes, the decoding com-

Instead, one can directly utilize the low complexity MS decoding algorithm for GLDPC codes by expanding each GC node into multiple SPC nodes based on the parity check matrix (PCM) of the component code [6]. However, the MS decoding performance, in general, is worse than the APP decoding performance because the MS decoder does not perform well over dense graphs with expanded SPC nodes. In other words, the state-of-the art decoding method for GLDPC codes is the APP decoding in terms of decoding performance but its complexity is much higher than that of the MS decoding.
To improve the performance while retaining low complexity decoding, we first employ the neural MS (NMS) decoding algorithm for GLDPC codes in this letter. The NMS decoding algorithm introduces multiple weights on messages that are optimized by the machine learning techniques. Learning techniques have successfully improved message passing decoding for dense linear codes [7], [8], [9] and short LDPC codes [10], and the code construction for LDPC codes [11]. Similar to the previous results, we show that the NMS decoding algorithm for short GLDPC codes overcomes the performance degradation due to dense component codes and enables to attain high decoding performance without the complex APP processor. Fig. 1 compares the conventional APP decoding scheme with the NMS decoding scheme considered in this letter. Along with the reduced decoding complexity, the NMS decoding scheme has an advantage over the APP decoding scheme such that all check nodes are updated by a single operation set consisting of the min operation and weighting, which is an attractive advantage for hardware implementation.
In addition, we optimize the m × n PCM of the component code that determines how to expand each GC node into m SPC nodes. Given the class of the component code, one can reformulate the component PCM without changing the GLDPC code by choosing m rows from the overcomplete PCM [9], [12] with redundant rows. II. CONSTRUCTION AND DECODING FOR GLDPC CODES
A. Protograph-Based GLDPC Codes
In this letter, we use lowercase letters x, bold lowercase letters x, and bold capital letters X to denote scalars, vectors, and matrices, respectively. In addition, xi indicates the ith element in vector x and [X]i,j indicates the (i, j)th entry in matrix X. We assume transmission over the additive white Gaussian noise (AWGN) channel.
The construction of protograph-based GLDPC codes [6] requires:
• Underlying base matrix B of size p × q; • Selection of d ≤ p GC rows; • Component PCM set S = {C(1), . . . , C(d)}, where C(t)
is the PCM of size m(t) × n(t) for the tth GC row; • Permutation π(t) of length n(t) for the tth GC row.
With lifting factor z, the PCM H of size M × N pz × qz is obtained from the base matrix B. Suppose that the first d rows in B are selected for GC rows. It means that each row in the tth set of z rows of H is constrained by the component code defined by the column permuted matrix of C(t) for 1 ≤ t ≤ d. Specifically, the component PCM is applied after permuting the columns of C(t) by π(t). Then, ν d/p fraction of rows in H is protected by GC constraints and the remaining rows are protected by the SPC constraint. In the protograph level, there are d GC nodes and p − d SPC nodes.
Compared to the PCM H consisting of GC and SPC rows together, we can expand H vertically by replacing the jth 1 in the (i + (t − 1)z)th row with the ξj(t)th column of C(t) for 1 ≤ j ≤ n(t), 1 ≤ i ≤ z, and 1 ≤ t ≤ d. In the protograph level, the tth GC node is considered to be expanded into m(t) SPC nodes for 1 ≤ t ≤ d. The total number of SPC nodes in the protograph becomes m p − d + t m(t). The expanded PCM, denoted by H, has size M × N mz × qz. The design rate R is accordingly R = 1 − M /N . Note that, in general, previous works on GLDPC codes [4], [5], [6], [15] assign the so-called standard PCM Cstd of size mstd × n to all GC rows, i.e., S = Sstd {Cstd, . . . , Cstd} for all t. The standard PCMs of various linear codes can be found in [13]. However, we add superscript t on C(t) to represent the irregular case of non-uniform component PCMs, which will be introduced in Section III-C.

C. Overcomplete Component PCM
As the PCM of a linear code is not unique, the code can be decoded by the message passing scheme over different PCMs. For an (n, k) linear code, a PCM is called a overcomplete PCM if the number of rows (constraints) is larger than n − k. Redundant rows can be generated by a linear combination of existing rows or by collecting dual codewords [8], [12].
For GLDPC codes, the MS and NMS decoders can also be performed over different component PCM sets. Since the component PCM set S determines the expanded PCM H, the MS and NMS performances are significantly affected by the choice of S. Let ci be the ith row of the standard PCM Cstd of size mstd × n represented in Section II. We consider the full overcomplete PCM Cfull of size mfull × n containing all mfull ≥ mstd minimum weight rows [12]. For example, Cfull of HM(6, 3) is given using Cstd = [c1; c2; c3] in (1) as
Cfull = [c1; c2; c3; c1 + c2 + c3],
where mstd = 3, mfull = 4, and the last row is a linear combination of {c1, c2, c3}. The other combinations do not have the minimum weight.

D. Optimizing Component PCM Set
Since the standard component PCMs are not designed for the MS and NMS decoding, they are not optimal in terms of the MS and NMS performances. In order to attain a performance improvement without increasing the decoding complexity, we first consider the optimization problem of selecting only mstd rows from Cfull. Our goal is to optimize the component PCM set S = [C(1), . . . , C(d)]. For each component PCM C(t) in S, we assign a 1 × mstd vector v(t) to represent C(t) in terms of row indices of Cfull. Then, optimizing S is translated to optimize a d × mstd matrix V = [v(1); . . . ; v(d)], where 1 ≤ [V]i,j ≤ mfull. Alg. 1 is the optimization algorithm for V based on the differential evolution [11], [14] for which randomly generated matrices evolve through the mutation, crossover, and selection steps. In the selection step, Alg. 1 calculates the frame error rate (FER) of MS decoding, denoted by FERMS, and updates the instance if FERMS is improved. Since the selection is based on the MS performance, the output S(0) of Alg. 1 has the best performance among the instances explored in Alg. 1. We set the parameters by Np = 200, G = 300, F = 0.5, pc = 0.8 and FERMS is measured at Eb/N0 = 5dB.
Next, we include additional redundant rows which do not exist in S(0) in a greedy way. Let S(r) be the component PCM set after adding r redundant rows. We evaluate FERMS for all possible cases of S(r) together with an arbitrary additional row. Then, we choose the additional row c resulting in the minimum FERMS and update S(r) to S(r+1) = S(r) ⊕c, which indicates S(r) together with the additional row c. Fig. 2 shows the flow

Note that, as shown in Fig. 3(a), the APP, MS, and NMS decoding algorithms are worse than the optimal MAP performance because they operate on distributed processors resulting in suboptimal performance. In addition, among message passing algorithms, the APP decoder is not the best because the APP decoding algorithm is derived by the asymptotic analysis which does not guarantee the optimal performance for the finite-length codes. Rather, the NMS decoding is trained on target codes using actual decoding performance and then attains a faster falling slope than the APP performance.
For G2 in Fig. 3(b), the component code is HM(15, 11) whose PCM is denser than the PCM of HM(6, 3). Thus, a larger gap between the APP performance and the MS performance of Sstd is observed compared to the case of G1. However, the NMS performance of the optimized PCM set S(ropt=4) nearly achieves the APP performance at FER 10−6. The maximum number of iterations is normalized to  = 7.64. In addition, Fig. 3(b) shows that the MS performance for Sfull is worse than that for Sstd even though the expanded PCM by Sfull activates much more CNs. This is because the negative effect of short cycles overwhelms the positive effect of additional redundant CNs. On the contrary, the optimized PCM set S(ropt) achieves the point that the positive effect suppresses the negative effect. The similar result is observed for G3 in Fig. 3(c), which demonstrates that the improvement by the NMS performance of S(ropt=0) is also valid for the irregular underlying base matrix. Finally, for G4 employing RM(32, 16) as a component code, the MS performance is much far away from the APP performance (about 5dB away at FER 10−4) due to the high density of the component PCM. The gap is reduced significantly by about 3.5dB for the NMS decoding with the optimized component PCM set S(ropt=25).
V. CONCLUSION
In this letter, we investigated the MS and NMS decoding schemes for GLDPC codes. Significant performance gains were achieved by using the NMS decoding algorithm and optimizing the component PCM sets, compared to the conventional

MS decoding. The performance improvement is verified for
various well known GLDPC codes. Specifically, for GLDPC
codes using simple Hamming component codes, the APP
decoding performance can be achieved by the simple NMS
decoder. It is practically beneficial because it means that the
complex APP processors can be replaced by the simple min
operations and weighting operations.
