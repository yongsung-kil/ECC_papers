SIMULATION OF DYNAMIC WEIGHTED MS DECODING ALGORITHM ON LDPC FOR 5G
WANG QIANCHUN1,2, JONATHAN M.CABALLERO2
1School of Electronic Engineering, Huainan Normal University, Huainan, Anhui Province, China 2dept. Graduate Program, Technological University of the Philippines, Manila, Philippines E-MAIL: wangqc1982@126.com, jonathan_caballero@tup.edu.ph

Abstract: According to the characteristics of high-speed, low delay,
large-capacity data transmission and various scenarios in 5G mobile communication, an improved decoding algorithm based on LDPC code for 5G is proposed. The algorithm adopts dynamic weighting decoding method to dynamically weight the information. In this way, the iteration can be accelerated when the information is transmitted, to reduce the number of iterations and improve the gain at the same time. From the point of view that 5G channel coding needs to support more bit rates, the simulation results show that compared with the traditional decoding algorithm, the performance of the improved MS algorithm is improved by about 0.4dB, and the dynamic weighting method improves the decoding performance and reduces the number of decoding iterations.

1. Introduction
Currently, the 5GNR (new radio) standard stipulates that the peak data download rate reaches 20gbps, the end-toend connection data delay is less than 1ms, and the user data delay in the urllc scenario is 0.5ms [1]. The 5g standard has defined three application scenarios: enhanced mobile broadband (eMBB), Dalian connected Internet of things (mMTC) and ultra-high reliability and ultra-low delay communication (URLLC). EmbB corresponds to the bandwidth of extensive traffic mobile services such as 3D Ultra HD video.
The encoding efficiency of LDPC code is close to the Shannon limit, and has low encoding and decoding complexity, which has low-performance requirements and power consumption. Therefore, LDPC coding and decoding is more suitable for 5g high-speed, low delay, large capacity data transmission and various scenarios. 5G mobile communication system is required to provide higher data transmission rate, serve more connections and obtain better user experience on the premise of ensuring low cost and

transmission security, reliability and stability [2]. 5G needs to ensure more reliable communication. The
air interface block error rate of let for general data must be 10% for initial transmission. This means that in 100000 code blocks, only one error can be made by the channel decoder, and at most one code block cannot be corrected. If the same bit rate is used, the data bits will be destroyed, and the spectrum resources will be wasted.
There are two kinds of decoding algorithm. Hard decision algorithm usually adopts bit flipping algorithm. Soft decision decoding takes Belief Propagation (BP) as the basic idea, and transfers information iteratively between different nodes, obtaining better performance. Soft decision needs much computation and is hard to be implemented on hardware due to its high complexity. To compromise the complexity and performance of soft decision algorithm, many scholars devote themselves to it.
Chen et al. analyzed the differences between MS and LLR-BP algorithms, proposed NMS and OMS algorithms, and gave two ways to calculate the normalization factor for NMS algorithm [3]. Compared with MS, NMS algorithm has only one more multiplication operation in the iterative process, which greatly reduces the implementation complexity. Darabiha et al. proposed the single minimum MS (sm MS) algorithm in 2006 [4]. Subsequently, Angarita improved sm MS and proposed the simplified variable weight MS (svw MS) algorithm by analyzing the rule that the minimum value changes with the number of iterations, thereby improving the decoding performance and convergence speed [5].
Some scholars also improve the throughput and error correction capability of LDPC by adjusting the decoding iteration steps. The first is the dynamic scheduling algorithm [6], which updates the most unreliable node first through various rules during iteration to prevent unreliable nodes from assimilating reliable nodes. Xie et al. solved the greedy nature of the original dynamic scheduling by predefining the check nodes of the update range, thus obtaining higher

Dynamic scheduling needs to complete the remaining decoding process according to various rules in the iterative process, which improves the complexity of the decoding algorithm. The other is hierarchical scheduling, which is slightly different from the flooding scheduling proposed by Kschischang. Hierarchical scheduling gives priority to updating the information values of some check nodes according to the partitioning rules, and these nodes immediately transfer the updated information values to the corresponding variable nodes [9, 10]. In this scheduling mode, nodes with lower levels can use the updated information value, thus improving the convergence rate and decoding performance.
Theoretical research shows that when the code length tends to infinity, the required SNR threshold is only 0.0045db [11] from the Shannon limit for reliable communication over additive white Gaussian noise (AWGN) channels using irregular LDPC codes with bit error rate of 1/2. The improved MS decoding algorithm proposed in this paper can break through the LDPC code and provide a broader choice space and basis for designing 5G mobile communication[12]. Compared with traditional channel coding, LDPC code has strong practical value in other bandlimited systems. This paper simulates the improved MS algorithm from the perspective of different code length and different code rate, and puts forward the application idea of the improved MS decoding algorithm of LDPC code in 5G system. On the premise of meeting the lower transmission delay and higher throughput of 5G system, this method can improve the bit error performance, further reduce the decoding complexity, improve the decoding rate, and is more suitable for the implementation of hardware platform. Simulation results verify the effectiveness of this scheme.

2.2. Algorithm Principle

The optimization of LDPC decoding algorithm is mainly aimed at the update of check nodes. When the verification node is updated, the LLR information is mainly updated. The LLR information reflects the measure of probability value to a certain extent [14]. For example, when p(0)=0.8, P(1)=0.2 or P(1)=0.8, P(0)=0.2, ABS (log (P(0)/p(1)) =0.9; When the probabilities of 0 and 1 are both 0.5, ABS (log(P(0)/p(1)) =0.
Therefore, the larger the LLR is, the greater the probability difference between 0 and 1 is. The more reliable this LLR is and the easier it is to converge. We need this value more. Therefore, for this value, we need to increase its gain. On the contrary, the smaller the probability difference between 0 and 1, the smaller the corresponding LLR value, and the harder it is to converge. We need to reduce the impact of this value in the decoding process. In the bit error rate performance simulation of the algorithm, the adopted modulation mode is BPSK modulation, the channels are set as AWGN channels, and the code length is expressed by the number of binary bits.
LU decomposition method is adopted for coding. The key of LU decomposition method is to optimize LU decomposition as much as possible, so as to reduce the storage of matrices L and U. When the check matrix H is in the form of lower triangle or upper triangle, the coding can be completed by direct forward iteration or backward
iteration without LU decomposition of H 2 . When the
check matrix H is in double diagonal or quasi double diagonal form, iterative coding can still be carried out directly. If the form of upper triangle or lower triangle and the check matrix of quasi double diagonal form are used, the complexity of coding is linear. Therefore, in order to reduce the complexity of coding, the linear structure is generally used when constructing matrix H.
Progress Edge Growth (PEG) algorithm is used for matrix construction. Peg algorithm increases the edges between symbol nodes and check nodes one by one while keeping the ring length of Tanner graph as large as possible. The execution process of PEG algorithm is to first determine the number of symbol nodes n, the number of verification nodes m and the degree distribution of symbol nodes. Then start to select the edges. When placing new edges, make sure that there are no 4 rings in the Tanner graph as far as possible; Then place the next edge, and verify and judge each time, until the edge is finished, which also meets the requirements of symbol node degree distribution.
