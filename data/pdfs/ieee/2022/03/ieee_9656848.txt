Stopping-Set Assisted Reinforced Belief Propagation for Decoding Short LDPC Codes

Asif Ali Zamzami , Kuan-Chuan Chao , and Shih-Chun Lin , Senior Member, IEEE

Abstract— The low-density parity-check (LDPC) code is chosen to be a channel coding scheme for ultra-reliable and lowlatency communications (URLLC) in 5G. However, short LDPC compromises the belief propagation (BP) decoder because of the stopping set (SS) in the decoding graph. We propose to perform different node operations according to the SS. A dynamic SS selection according to the received sequence is further applied. In the length 308 5G LDPC, our decoders outperform prior work, improving BP and collaborative BP at block error probability (BLER) of 3 × 10−5 by around 0.176 and 0.092 dB respectively, with almost the same number of iterations.
Index Terms— 5G, stopping set, short LDPC codes.
I. INTRODUCTION
T HE low-density parity-check (LDPC) code is one of the recently approved channel coding schemes in 5G new radio (NR) [1]. In addition to the application in the enhanced mobile broadband (eMBB), 3GPP also expects LDPC code to be used in the ultra-reliable and low-latency communication (URLLC) [2], where the block error probability (BLER) and the latency are required to be less than 10−5 and less than 1 ms respectively [3]. Unlike in eMBB, the short-latency requirement in URLLC is challenging, which urges a new design. While the low latency can be achieved by the transmission of short blocklength LDPC code (less than 1000 code blocklength), the standard iterative message passing decoder, i.e., belief propagation (BP) decoder, may result in high BLER. For short LDPC code, the problem of BLER arises out of the error-prone substructure, such as stopping sets (SS) [4] or trapping sets (TS) [5], in the decoding graph.
Among the known error-prone substructures in the decoding graph, the SS has been investigated in substantial research [4]–[6]. The BP algorithm [7] performs decoding by iteratively passing the log-likelihood ratio (LLR) in the Tanner graph, which consists of a layer of variable node decoders (VND)s and the other layer of check node decoders (CND)s. Rather than being corrected by the VNDs outside the SS, the erroneous LLRs will nevertheless remain in the VNDs inside the SS during iterations. Hence, the short cycles cannot be prevented, leading to BP failure.
Similar decoding failure can also be observed in a Gaussian channel, and numerous studies have proposed different algorithms to solve the decoding problem for short LDPC code in the Gaussian channel [5], [8], [9]. The collaborative belief propagation (CBP) [5] improves the BLER of BP as follows. First, the BP is executed. If it fails, the CBP decoder will change the decoding graph according to the SS or TS and perform extra iterations. The erroneous decoding then has an opportunity to be corrected due to the additional iterations. Another strategy to deal with the short LDPC-related decoding problem stems from deep learning [8], which applies different weights to different edges in the Tanner graph so as to improve BP. How to calculate the proper weight is trained by pregenerated noisy received sequences and corresponding correct codewords. These two ideas are further combined in [9], where the decoding graph and the weight both change during iterations.
In this letter, we propose to carefully apply LLR reinforcement according to an SS in order to improve BP. It is well known that the reinforced belief propagation (RBP) algorithm [10] can be applied to the case where BP fails, such as a source coding problem where a graph code is used [11]. However, instead of applying LLR reinforcement to all VNDs [10], [11], we apply it only to those outside the SS because the VNDs inside the SS have a higher chance of producing erroneous LLRs. Note that the SS is a subset of VNDs if the neighboring CNDs are connected at least twice to the VNDs in the subset [4], [6]. According to this definition, the SS is solely determined by the Tanner graph (or parity check matrix) of the LDPC codes and is irrelevant to the received sequence. Moreover, the performance of the aforementioned decoder is further improved by a dynamic SS selection according to the received sequences. Our decoders demonstrate improvement on BP and CBP [5] at BLER of 3 × 10−5 by around 0.176 and 0.092 dB respectively, with almost the same number of iterations as 100 and reasonable memory increase. The length 308 and 616 short LDPC codes in 5G [2] are used in these simulations.
The rest of this letter comes as follows. In Section II, we introduce the background of LDPC code decoding in a Gaussian channel. The proposed two decoders and the complexity comparison are presented in Section III and Section IV respectively. Section V shows the simulation results, and finally, Section VI concludes the letter.

II. BACKGROUND OF ITERATIVE LDPC CODE DECODING
The LDPC code is defined by an m×n parity-check matrix, which can be equivalently described by a bipartite graph called as Tanner graph [12]. For the LDPC code decoding graph, there are one layer of VND nodes V = {v1, . . . , vn} and another layer of CND nodes C = {c1, . . . , cm}, with edges connecting these nodes. A CND cj belongs to the neighbor of VND vi if they are connected to an edge, denoted by cj ∈ N (vi). A stopping set S is a collection of VNDs defined as follows. For any S ⊆ V, S is a stopping set if the neighboring CNDs are connected at least twice to VNDs in the stopping set [4]. The approach introduced in [6] can be utilized to identify the stopping sets in the LDPC code.
The iterative LDPC code decoding process is briefly reviewed here. Assume a Gaussian channel with the received sequence y1, . . . , yn following yi = xi +zi, where xi is the ith coded symbol, noise zi ∼ CN (0, σ2) is independent complex Gaussian with zero-mean and variance σ2. The decoder will use yi to compute an initial channel LLR 2yi/σ2 for VND vi [7]. For each iteration, LLR messages are first updated at the VND layer and passed to the CND layer, and then further updated at the CND layer and passed back to the VND layer. The former is called the VND-to-CND operation, while the latter is the CND-to-VND operation in this letter. To be more specific, the conventional BP algorithm is adopted to decode the codewords with blocklength n where the “sum” rule is adopted for the LLR updation at the VND-to-CND operation and the “product” rule is adopted for the CND-toVND operation. The decoding process will be terminated when the maximum iteration number is reached, and the decoder will make a decision on the transmitted codewords based on the final LLRs. The details are referred to [7].

Then the drawback of SS-RBP1 is that the stopping set S is not chosen according to the received noisy sequence {y1, . . . , yn}. IV. COMPLEXITY COMPARISON
The two proposed algorithms, CBP, and neural-network (NN) decoders [8], [9], have the same run-time complexity order as standard BP, that is O(n). The main difference lies in the memory or storage requirement. First, the BP algorithm [7] requires (m+n)·R memory to store the real numbers for LLR messages, and m · n · N memory to store positive integers for edges of the Tanner graph, where R and N denote the memory need for a real number and positive integer respectively. In addition to BP’s storage, CBP [5] has the lowest storage since it only requires the storage of the stopping set |S| · N in the modified Tanner graph. Our SS-RBP1 requires additional memory to store LLR messages of the VNDs outside the stopping set in the previous iteration (n − |S|) · R. Further, in SS-RBP2, all stopping sets in the list S are needed to be stored (|S|·|S|)·N. On the other hand, the NN decoder with relaxation method [8] requires storing the weights for each edge, m·n·R, and the n·R LLR messages of VND from the previous iteration for different SNRs. In [9], the NN decoder D1 requires to store m · n · R and m · R for weights in VND-to-CND and CND-toVND operations, and n · R weights for channel messages; for different SNRs. Furthermore, SNR improvements for (128,64) LDPC code per memory requirement are 5.37×10−6/R for the NN decoder [9] and 1.7 × 10−3/R for our SS-RBP2 assuming four sets of NN weights for different SNRs.
V. SIMULATION RESULTS
In the simulation, we adopt the 5G LDPC codes with Tanner graphs generated via base graph (BG)2 [2]. Note that two kinds of LDPC codes are adopted in 5G NR, i.e., with BG1 and BG2; and it is suggested by 3GPP that BG2 LDPC code suits for code blocklength n ≤ 3824 with code rate smaller than 0.67. Since URLLC typically requires n < 1000, we choose QPSK modulations and BG2 LDPC codes with n = 308 and n = 616. To achieve a very low error probability required in URLLC, typically the SNR operation regime is relatively high for the chosen LDPC code. Therefore, the Gaussian channel behaves more like an erasure channel, where the channel is either in a good or a bad state. The stopping set may contribute more to the decoding error in the Gaussian channel at a high SNR regime [13], [14].
Five stopping sets are identified, as detailed in [15]. We will compare proposed decoders with CBP in [5] due to its reasonable memory size, which is briefly reviewed here. First, standard BP (main decoder in [5]) will run on the original Tanner graph until 100 iterations. Once unsatisfied CNDs are detected, BP will be run for 25 iterations on a graph (sub-decoder in [5]) modified as follows. In the chosen stopping set, one adjacent and unsatisfied CND is randomly removed for each VND and the edges connected to it are disabled. Finally, the original graph will be used and another 25 BP iterations will be executed on it.

VI. CONCLUSION
Short LDPC code is considered as a critical research topic in terms of the short-latency decoding in 5G URLLC. We propose schemes to improve the standard BP decoding by utilizing stopping sets and LLR reinforcements. With BG2 LPDCs under lengths 308 and 616, our decoders improve BLERs of previous BP and CBP decoders with almost the same number of iterations.
