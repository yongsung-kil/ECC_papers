Abstract: This paper presents an area efficient implementation 
of short block length and low rate Quasi cyclic (QC)-Low Density 
parity check code (LDPC) encoder for binary input additive white 
Gaussian noise channel (BIAWGN) targeting the Ultra-reliable 
low-latency communication (URLLC) application. The stringent 
prerequisites like ultra-high reliability and lesser latency have 
made URLLC applications most critical feature of the 5G wireless 
communication. In the proposed architecture the dual diagonal 
encoding is used to generate the codeword. The primary building 
block in the dual diagonal encoding is the barrel shifter, which 
performs the cyclic shift operations. Thus, it occupies larger part 
of area next to the memory blocks. This works aims at drastically 
reducing the number of barrel shifters using hardware sharing 
technique. The implementation of the encoder for short length 
code is done in Xilinx Virtex-7 FPGA platform. The simulation 
results show that the proposed architecture can reduce the number 
of multiplexers in the encoder architecture by 37%, thus 
considerably reducing the area compared to the existing 
architectures.  
Keywords— LDPC, Encoder, Barrel shifters, dual diagonal, 
5G new radio, URLLC, very large scale integrated (VLSI) 
architecture 
I. 
INTRODUCTION  
Considered as the most critical component of cellular 
communication systems, channel coding is used for 
correcting the errors in transmission inflicted by interference 
and noise. 5G New Radio (NR) was developed by 3GPP (3rd 
Generation Partnership Project) for a stronger and more 
capable 5th generation (5G) mobile network. With the advent 
of 5G NR it is possible to deliver smooth and more reliable 
services from Internet of things (IoT) to smart technologies 
on a larger scale. 5G NR is categorized into eMBB (Enhanced 
Mobile Broadband), URLLC and mMTC (Massive Machine 
Type Communication) as stipulated by 3GPP R15. Being 
chosen as a powerful candidate among several error 
correcting codes for eMBB data owing to its better 
performance at higher data rate, the Low-density parity-check 
(LDPC) codes are extensively used for improving reliability 
of modern communication systems. 
URLLC services, like telesurgery, industrial automation and 
so on, requires high reliability, with block error rate (BLER) 
in order of (1-10-9) and end-to-end latency lower than 1ms 
[1]. Thus, a delay sensitive application of 5G NR, URLLC 
application entails two contradictory requirements to be 
satisfied, such as ultra-low latency and greater reliability [2]. 
Short block length codes significantly reduce the network 
latency, owing to its shortened transmit time interval (TTI) 
[3]. Using short codes would in-turn decrease the coding 
gain, thus affecting the reliability. The BLER and the 
decoding latency is critical while designing the decoder. As 
of encoder implementation is concerned the most important 
criteria to be considered is area, power consumption and 
speed. 
However, there exist a challenge in the design of high speed 
and area efficient encoders with an adequate degree of 
reconfigurability to meet the dynamic demands of recent 
standards. This works aims at creating an area efficient 
encoder for 5G NR URLLC applications.   
This paper is briefly organized as follows. Section II 
highlights the conventional encoding architectures for LDPC 
codes. Section III discusses the steps in encoding algorithm 
for QC-LDPC codes. Section IV explains the construction of 
design equation for Base graph 2 targeted for low code rate 
and short block- length codes. Section V details the proposed 
area efficient architecture for the LDPC encoding. Section VI 
presents the implementation results and compares the 
proposed architecture with the existing. Section VII 
concludes the work. 
 
Major contribution of this work: Design equations for short 
length and low code rate QC-LDPC codes are generated for 
subsequent implementation. A novel, area efficient QC-
LDPC codes using hardware sharing technique, suitable for 
URLLC application is implemented in Xilinx. 
 
II. 
LDPC ENCODING ARCHITECTURES 
        The LDPC encoding process should satisfy the 
following parity check equation, as in: 
                                   HCT = 0T                                   (1) 
Where H is the parity check matrix and the codeword C, 
which comprise the message bits d and the parity bits P. This 
section covers the types of encoding techniques available in 
literature. 
A.  Gauss Elimination Method 
The Gauss elimination method of encoding uses 
generator matrix obtained from the parity check matrix to 
generate the codeword [4]. The generation of the codeword is 
done by multiplying the generator matrix G with the message 
bits(d) given by 
                                 C = Gd 
                                   (2)

Eventhough Gaussian method is a direct form of 
encoding, it suffers from memory overhead of storing the 
non-sparse generator matrix, thus increasing the memory 
utilization requirement. The implementation complexity here 
is O(N2), number of bits in the codeword is denoted as N. 
B. Richardson–Urbanke (RU) method 
The RU encoding of LDPC code is performed by 
converting the parity check matrix into lower triangular 
matrix, followed by the back substitution, thus maintaining 
the spareness of the parity check matrix [5]. Even though RU 
method overcomes the increased memory requirement 
observed in Gauss elimination, it suffers from long critical 
paths. RU method also requires to calculate the inverse matrix 
which would increase the complexity and memory storage. 
C. Dual Diagonal encoding 
The 3GPP R15 proposed two base graphs (BG), denoted 
as BG1 and BG2 for channel coding in 5G NR. BG1 is used 
for high date rate and long block length codes (500≤K≤8448). 
BG2 is targeted for low data rate and short to moderate block 
length codes (40 ≤ K ≤ 2560). Both the base graphs have a 
double diagonal structure which simplifies the computation of 
the parity check bits. The parity bits are generated using 
simple cyclic shift operations [6]. 
This works aims at encoding of the QC-LDPC codes 
taking advantage of the dual diagonal structure of the BG2, 
to produce the codewords. The dual diagonal encoding uses 
simple cyclic shift operations, thus reducing the complexity 
of implementation compared to traditional methods of 
encoding Gauss elimination method and Richardson–
Urbanke (RU) encoding scheme.

V. 
PROPOSED AREA EFFICIENT ARCHITECTURE OF 
LDPC ENCODER    
 
        This work aims at implementing an area efficient QC-
LDPC encoder for short length codes. The barrel   shifters are 
the primary processing element in encoder performing the 
cyclic shift operations. It occupies considerable area in an 
encoder architecture. The proposed architecture emphasizes 
on reducing the area by sharing of hardware blocks. The 
important blocks of the architecture are as follows: 
Controller: It provides the control signals like clock (clk), 
reset(rst), chip select (cs) to enable the memory blocks, select 
and data_select to the multiplexers and the demultiplexer 
which controls the hardware sharing. 
Shift Registers: The serial to parallel shift registers act as 
input buffers to transfer the received message into the next 
stage. 
Barrel Shifters: It performs the critical operation of cyclic 
shifting the input block of data in accordance to the data 
provided by the cyclic shift controller. The number of barrel 
shifter in this work is drastically reduced to half as required 
in [6] targeted for BG1, by using the hardware sharing 
technique. The size of barrel shifter is same as the lifting size 
(Z). 
Multiplexer stage: In this paper, the multiplexer stage is 
introduced between the input shift register stage and the 
barrel shifter stage to reduce the number of barrel shifters 
needed for the cyclic shifting operation. The barrel shifters 
comprise of multiplexers such that for N -bit logarithmic 
barrel shifter, the number of multiplexers required to 
construct the barrel shifter will be N x logN. For example, if 
the size of barrel shifter is N=16, then the number of 2:1 
multiplexer required to construct the 16-bit barrel shifter will 
be, 16 x log2 16= 64.   
Therefore, instead of using equal number of barrel shifters as 
that of the input block, the number has been reduced to half 
by using the multiplexer stage, thus diminishing the 
requirement of 2:1 multiplexer considerably, thus improving 
the area efficiency. 
Memory:  ROM memory module is used for storing the shift 
coefficients of the matrix A, C1 and C2, comprising of three

RESULTS AND DISCUSSIONS 
        The comparison between gaussian method, RU method 
and dual diagonal encoding is shown in the Table.3. The 
hardware components used for implementing the encoding 
architectures and the memory utilized has been compared. It 
is evident from the comparison that the dual diagonal

compared to the other techniques. The number of barrel 
shifters used has considerably reduced compared to the

considerable reduction in the number of multiplexers using 
the hardware sharing technique.   
For short code length LDPC codes proposed in this work, it 
is observed that the number of barrel shifter is considerably 
reduced, thus reducing the area consumption. It is observed 
that the area reduction does not affect the number of clock 
cycles in which the codeword is generated. Thus, speed is not 
compromised.

VII. 
CONCLUSION 
    In this paper, a novel-area efficient architecture for short-
block length codes (BG2) of QC-LDPC has been proposed. 
Short codes are suitable for ultra-low latency applications, 
which is one among the category of 5G NR. A multiplexer 
stage is introduced between the input stage and the cyclic 
shifter stage to reduce the number of N-bit barrel shifters, 
which in turn will reduce the number of multiplexers required 
to perform the cyclic shift operation in the overall encoder 
architecture. It is observed that with the introduction of the 
multiplexer stage a 37% reduction in multiplexers is observed 
compared to the existing architecture.
