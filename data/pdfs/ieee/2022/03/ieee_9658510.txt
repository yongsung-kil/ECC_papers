Hard Reliability-Based Ordered Statistic Decoding and Its Application to McEliece Public Key Cryptosystem
Shuyan Yu, Graduate Student Member, IEEE, and Qin Huang , Senior Member, IEEE

Abstract— This letter constructs the hard reliability from the syndromes to decode low/moderate-density parity-check codes in an ordered statistic way. For regular codes with our proposed decoding algorithm, the error-correction capability can be determined by the column weight and the maximum column intersection of their parity-check matrices. Then, an upper bound is derived to verify their decoding performance in ultralow region, which may be required by some applications, e.g., McEliece public key cryptosystem and optical communications. It allows us to reduce the size of McEliece public keys with strong security levels due to its good error-correction capability.
Index Terms— McEliece public key cryptosystem, LDPC codes, hard reliability, ordered statistic decoding.

I. INTRODUCTION

T HE McEliece public key cryptosystem (PKC) [1] is regarded as secure in post-quantum cryptography, whose

security relies on the hardness of the problem of decoding

a linear code. Since the original PKC based on Goppa

codes has very large public keys, the quasi-cyclic (QC)

codes, e.g., QC low-density parity-check (LDPC) codes [2]

and QC moderate-density parity-check (MDPC) codes [3],

are introduced to significantly reduce the key size. Goppa

codes admit bounded distance decoders with guaranteed error-

correction capabilities. Instead, LDPC and MDPC codes

use efficient iterative decoders. However, such decoders are

characterized by some non null frame error rate (FER), which

is hard to estimate. As a consequence, it is hard to design

PKC based on these codes with strong security levels, which

requires very low FER in decryption to resist the feasible reaction attacks [4]–[6], e.g., 2−80, 2−128 and 2−256.

In [7], it shows that the error-correction capability under the

bit flipping (BF) decoding algorithm depends on the column

weight v of the parity-check matrix and the girth g in the

Tanner graph. In the case of g = 6, its error-correction

This letter proposes a hard reliability based ordered statistic decoding (HR-OSD) algorithm in terms of syndromes. It consists of three steps: 1) compute the hard reliability measure [9] according to nonzero parity-checks; 2) divide the received bits into a set of reliable bits and a set of unreliable bits; 3) re-encode the reliable bits to obtain the decoded codeword. We analyze the error-correction capability in terms of the maximum column intersection and the column weight v. For a regular LDPC code with g ≥ 6, the error-correction capability in our approach is v if reencoding correctly. Then, an upper bound on FER is given to determine the decoding performance of the regular codes in ultra-low FER region, which may be required by some applications, e.g., McEliece public key cryptosystem and optical communications. It promises us to design the public keys with the required FER to achieve strong security levels for the McEliece PKCs in [2], [8], [10].
The analysis of an upper bound and simulation results verify that our proposed algorithm has lower FER than the parallel BF decoding algorithm. Thus, shorter codes can be used to reach FER required by strong security levels. As a result, it allows us to design the smaller public keys with the same security levels as keys provided by the parallel BF decoding algorithm.

III. HARD RELIABILITY BASED ORDERED STATISTIC DECODING ALGORITHM
In this letter, we propose to decode regular codes in an ordered statistic way. It proves that its error-correction capability can be twice that of the parallel BF decoding algorithm. In other words, it will promise us to design smaller public keys. Moreover, an upper bound on FER is derived from its error-correction capability.

A. Algorithm Description

In the McEliece PKC, the received vectors do not have any
soft information. In other words, bits in a received vector r can not be ordered in terms of the magnitude of soft
information [15]. Therefore, we compute the hard reliability
measure according to nonzero parity-checks. Then, we can
decode the received vector by re-encoding as OSD [16] and
Prange’s algorithm [17]. Let s = r · HT = (s0, s1, · · · , sn−k−1) be the syndrome

IV. SIMULATION AND DESIGN
In this section, we present the simulation results of the HROSD algorithm. Then, we compare the size and security levels of public keys designed by the HR-OSD algorithm and the parallel BF decoding algorithm.

A. Simulation Results
We consider a (3, 6)-regular LDPC code C1 constructed by the progressive edge growth (PEG) algorithm [20] with code length n = 1200. Figure 1 shows that for C1, the HR-OSD algorithm outperforms the parallel BF decoding algorithm with single iteration and 10 iterations, respectively. It also verifies the validity of the upper bound on FER given by Theorem 3 in Fig. 1. Also, we consider a (45, 90)-regular MDPC code C2 with code length n = 9602, which can resist the efficient attack for QC-LDPC code-based PKC [21], [22]. Figure 1 shows that for C2, the HR-OSD algorithm outperforms the parallel BF decoding algorithm.
