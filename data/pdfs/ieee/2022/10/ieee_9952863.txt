Sneak Path Aware Bit-Flipping Algorithm for ReRAM Crossbar Array

Myungin Kim School of Electrical Engineering
KAIST Daejeon, Korea kmi9212@kaist.ac.kr

Jeongseok Ha School of Electrical Engineering
KAIST Daejeon, Korea jsha@kaist.edu

Abstract—Resistive random access memory (ReRAM) has unique features such as fast data processing speed, non-volatility and high density and thus has been gaining popularity in a wider range storage applications. Meanwhile, ReRAM suffers from read errors mainly due to the variations of cell resistance and the sneak path problem. There have been efforts to cope with the read errors using error-correcting codes such as low-density parity-check (LDPC) codes and Bose–Chaudhuri–Hocquenghem (BCH) codes. In this paper, we propose a simple bit-flipping (BF) decoding algorithm for correcting read errors in ReRAM due to the sneak paths and show that the proposed algorithm efficiently resolves the sneak path problem at reduced complexity. In particular, we assume that ReRAM provides certain side information about the reliability of retrieved bits, and the proposed algorithm utilizes the side information in the decoding. Error-rate performance of the proposed algorithm is compared with that of an existing BF algorithm, which shows a clear performance improvement with the proposed one.
Index Terms—ReRAM, sneak path, LDPC codes, BF algorithm
I. INTRODUCTION
Resistive random access memory (ReRAM) has been finding its applications in a wider range of applications such as main memory, cache, and neuromorphic computing due to its unparalleled features: high read/write speed, high storage density, and low power consumption. ReRAM stores information bit in the form of memory cell resistance which is controlled by applying voltages to metal electrodes in the cell. In particular, ReRAM stores bit values of ‘0’ and ‘1’ in a memory cell by changing the resistance of the cell to a predetermined high and low resistance states, respectively [1]. A ReRAM device is made up of a volume of memory cells which are placed in the device with the crossbar array structure [2]. In a crossbar array, each memory cell is located at the intersection of a pair of word and bit lines.
To store a bit value into a specific memory cell in a crossbar array, the resistance of the cell is modulated by the the voltages applied to the word line and bit line. Meanwhile, the bit value stored in the cell is retrieved by sensing the current due to the voltage drop between the word line and the bit line. During the data retrieval, the other cells located in either the word line or the bit line sharing with the target cell also experience certain voltage drop, which leads to unwanted current flows through the other cells with the low resistance states [3]. The

paths through which the parasitic current moves are called the sneak paths. The parasitic current flows reduce the resistance of the target cell. When the target cell is in the high resistance state representing bit value ‘0’, the reduced resistance makes it harder to distinguish the state from the low resistance state representing bit value ‘1’. Thus, the sneak paths result in read errors, i.e., the bit value of ‘0’ will be erroneously retrieved as ‘1’.
There have been efforts to cope with the read error due to the sneak paths using error-correcting codes [4], [5]. Song et al. in [4] proposed a coding scheme in which user data are encoded into a codeword of low-density parity-check (LDPC) code, and the coded bits are stored across multiple arrays in each of which sneak paths have independent statistical characteristics. Li et al. [5] proposed a two-step across-array bit allocation scheme which also distributes coded bits of LDPC codeword to multiple arrays aiming to reduce the correlation across the coded bits.
For high-speed applications like storage devices, lowcomplexity decoding is especially desirable, which is often suited with bit-flipping (BF) algorithms since they can be implemented with simple integer additions and multiplications [6]. For instance, the BF algorithm in [6] introduces the flipping function for each coded bit as the sum of syndromes with which reliability of the coded bit is measured. The algorithm flips coded bits with the maximum flipping function value during decoding iterations. Due to the light decoding complexity, BF algorithms have been often adopted in storage applications using memory devices [7].
The existing BF algorithms [6]–[8] assumes symmetric channels, i.e., an equal error rate for coded bits of ‘0’ and ‘1’. However, ReRAM has a higher error rate for the bit value of ‘0’ due to the sneak paths [4], and thus the channel model for ReRAM is by nature asymmetric. In addition, the stored bits of ‘0’ can be categorized into two group in terms of their reliabilities, namely, the affected and unaffected ones by the sneak paths. Thus, in total, ReRAM can be modeled as a noisy channel with three different channel outputs: ‘1’, weak ‘0’ and strong ‘0’.

II. CHANNEL MODEL
The data stored in a ReRAM memory array are represented as an M × N matrix X = [xi,j]M×N where xi,j ∈ {0, 1} which is illustrated as a crossbar in Fig. 1. It is known that the resistance values of memory cells representing the stored data follow the log-normal distribution whose parameters depend on the process variations and the thermal noise [9]. In addition, a sneak path occurs at a memory cell in the high resistance state, i.e., representing bit value of ‘0’, called zero-cell for short, depending on the data pattern. IV. SIMULATION RESULTS
To evaluate error correcting performance of the proposed algorithm, Monte-Carlo simulations are carried out for a 32 × 32 crossbar array with a (1024, 896) regular LDPC code constructed with the quasi-cyclic progressive edge growth (QC-PEG) algorithm [13]. The LDPC code has its variable and check node degrees of 4 and 32, respectively. The nominal resistance values for R0, R1, and R0′ are set to 1000Ω, 100Ω, and 200Ω, respectively. For a range of the standard deviation, σ, between 14 to 24, we evaluate and compare the error-rate performances of the existing [6] and proposed BF algorithms. The threshold Rth value for deciding ‘0’ or ‘1’ is set to 157Ω which is the same value as in Fig. 2. Meanwhile, the threshold Rt′h to construct the set B0 is determined to 600Ω which is illustrated as the blue line in Fig. 2.

V. CONCLUSION In this paper, we propose a BF decoding algorithm which takes the interference induced by sneak paths into account to improve the error-rate performance. In particular, the proposed algorithm assumes that the outputs from ReRAM are categorized into the ones with/without being affected by the sneak paths by introducing an additional threshold. The side information is utilized in the proposed decoding algorithm. This work demonstrates, by performing Monte-Carlo simulations, that the proposed algorithm with the side information provide a significant performance improvement in error rate.
