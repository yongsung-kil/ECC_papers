Low Power Decoder Architecture of Product Code for Storage Controller

Sumin Kim, Byungmin Ahn, Bohwan Jun, Mankeun Seo, Hongrak Son, and Yong Ho Song
Memory Business Samsung Electronics Hwaseong, Republic of Korea

Abstract— Error correction codes (ECCs) are essential for correcting the errors caused by the physical characteristics of 3D NAND flash in flash solution products. In this paper, we introduce low power decoder architecture for product code. Precisely, by using Hamming product code and Chase-Pyndiah algorithm, this work takes not only the same correction capability and area as low-density parity-check decoder, but also less power with relatively simple operation for decoding. Only the log likelihood ratios of error-suspicious bits are gathered in SRAM for the next iteration of decoding, so that the power consumption and area of SRAM read/write decrease. Compared with the conventional ECC decoder with the same correction capability and area, our decoder has achieved an average power reduction of 22.4%.
Keywords; Error correction code, Hamming product code, Chase-Pyndiah algorithm
I. INTRODUCTION
Storage solution products based on NAND flash have superior power efficiency and stability compared to magnetic hard disks. Moreover, thanks to its high density and speed characteristics, NAND flash has been widely used in numerous fields such as machine learning and big data processing that require enormous storage space. Nevertheless, due to the physical characteristics of NAND flash that stores data in floating gates, which is prone to errors, an error correction code (ECC) decoder is essential to correct the corrupted data.
As the number of layers increases in 3D NAND flash memory, the bit error rate (BER) also increases, so powerful ECC such as a low-density parity-check (LDPC) code has been exploited because of high error correction capability [1]. However, power consumption also increases to correct more errors [2]. This work addresses a decoder with Hamming product code [3] for less power and same error correction capability compared to LDPC decoder. Hamming product code has structural simplicity of linear block code as well as high error correction capability of turbo code, which is widely used along with LDPC and has good performance close to Shannon limit [4]. In order to achieve such capability, Pyndiah applied the iterative decoding method capable of soft input/soft output to the linear block code based on the Chase algorithm [5].
A decoder inside a storage controller is responsible for reading and writing data from NAND flash. In this paper, we propose a novel decoder architecture, which is suitable to read only the error suspicious bits necessary for the current iteration

and write the other error suspicious bits for the next iteration, in order to reduce power compared to the conventional decoder even with the same area and correction capability. Through experiments, our proposed decoder shows 22.4% lower power compared with the latest decoder.

III. PROPOSED ARCHITECTURE
Chase-Pyndiah algorithm needs to control log likelihood ratios (LLRs) of the suspicious bits for iterative decoding [7]. This paper focuses on the operation of reading the LLRs received from the previous iteration and writing the LLRs transmitted to the next iteration. The LLRs are stored in the SRAM of the decoder, not the register, because all LLRs are not required at every moment of one iteration, but only LLRs associated with decoding, which is sequentially processed in the iteration, are required.
Fig. 2 shows the proposed architecture implemented in the SRAM. Each of memory banks are physically separated. R0, R1, ⋯ , R5 and W0, W1, ⋯ , W5 are 6 LLRs to read and write in one iteration, respectively. A0, A1, ⋯ , A11 are 12 LLRs to read or write in one iteration for exceptional case.
The purpose for separating the read and write position is as follows. In a row decoding operation, the decoder reads the LLR from the SRAM according to the key index of the row iteration, and writes the LLR to the SRAM as the key index of the column iteration required for decoding in the next iteration. In the column decoding operation, the decoder sequentially performs functions in a horizontal direction from left to right in Fig. 1, and as a result, key indexes connected to a row codeword are spread to various column codewords. This means that reading from SRAM is sequential, but writing to SRAM is difficult to predict. Since two operations occur at the same time, it is necessary to separate the read position and the write position in order to cope with the unpredictable movement of the write operation.
The SRAM limits the number of LLRs corresponding to each key, which is 6 in this paper. Nonetheless, exceptional processing is needed in case of exceeding the number of LLRs. Since the extension operation is rare, it is designed to provide more space for LLRs in order to secure error correction capability. The read and write positions are separated from in the static allocations, but the read and write positions are common in the extension allocations.

IV. EXPERIMENTAL RESULT
Table 1 shows the comparison of average power between conventional and proposed decoder. Conventional decoder has same area and correction capability with proposed decoder. The unit of result is ratio in order to show the effect of the proposed architecture. The power consumptions are measured for analyzing application specific integrated circuit power consumption at the same operating frequency. The average power is the power consumed by the one codeword with a fixed number of errors and random messages in an environment where input/output throughput is not disturbed. The proposed architecture can improve power efficiency up to 22.4% compared to the latest architecture.
V. CONCLUSION
In this paper, we propose low power decoder architecture of Hamming product code. In particular, it focuses on optimizing the flow of decoding LLRs in hardware. The proposed decoder has reduced the power compared to the latest decoder by 22.4% when the other conditions are the same, which are area, correction ability and operating throughput. Through the result, this paper verifies that the proposed architecture is appropriate to adopt it in storage solution products using NAND flash memory. In future work, we plan to make more competitive ECC decoders by increasing error correcting capability.
