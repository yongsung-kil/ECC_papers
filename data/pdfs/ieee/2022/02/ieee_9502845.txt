A Low-Complexity Endurance Modulation for Flash Memory
Amr Ismail and Magnus Sandell , Senior Member, IEEE

Abstract—The repeated program/erase (P/E) cycles wear out flash memory cells which limits the device’s lifetime. The larger the injected/erased charge, the faster the wear out rate and thus there is interest in reducing the percentage of cells programmed to high voltage levels. Conventional data shaping techniques often introduce a nonnegligible overhead as well as incurring additional complexity to encode the input data into shaped codewords and decode the read data. We propose a novel lossless yet simple endurance modulation technique that relies solely on the errorcorrection code (ECC) to retrieve the original data. This idea is motivated by the fact that ECCs are usually designed to correct channel errors towards the end of the device’s lifetime and hence at early age, the ECC can be jointly used for channel error correction and endurance demodulation. The proposed scheme consists of data shaping by flipping bits in a controlled manner and simple max-log demodulation to recover those flipped bits. The simulation results show that the proposed scheme does not undermine the device’s ability to recover the original data if the shaping gain is adjusted as the device ages. Once the channel errors exceed a prescribed threshold, the controller disables the endurance modulation and operates in normal mode.
Index Terms—Flash memory, data shaping, soft detection.
I. BACKGROUND
T HE STORAGE unit (or equivalently a cell) in flash memory is a floating gate (FG) transistor where data is stored in the form of trapped charge in its FG. The flash memory die is organized into blocks where the cells sharing the same drain electrode constitute the bitline (BL) and the cells sharing the same gate constitute the wordline (WL). Each WL consists of tens of thousands of cells and is organised into log2(L) logical pages (or simply pages) where L is the number of threshold voltage levels a cell has. The logical pages are separately encoded to enable reading them individually. A WL to be programmed is first activated and progressively programmed via incremental step-pulse program (ISPP) [1]; the electrons are incrementally injected into the floating gate and sensed until the threshold voltage exceeds the target value. As ISPP only increases electric charges, the whole block should be erased prior to subsequent programming. The repeated program/erase (P/E) cycles wear out the flash memory cells which

A. Prior Art
Several techniques have been proposed to shape the input data to reduce the flash memory cells wear out rate. The authors in [2] proposed adaptive endurance coding (AEC) which employs enumerative coding to map unconstrained data into codewords with a desired voltage distribution. To overcome the resulting data expansion, the authors assumed that the original data is compressible and that applying lossless compression prior to endurance coding can compensate for the expansion factor and keep the data size fixed. To that end, a data-dependent wear out model has been introduced; it associates a cost ci to voltage level i with the objective of maximizing the endurance gain for a fixed rate R. The low RBER at the early age of a device allows a portion of the parity bits to be used for data shaping. Once the device ages and the RBER increases, the controller then switches to normal mode where no shaping is applied. A significant improvement of device endurance can be theoretically achieved without rate loss, however the endurance modulation/demodulation still relies on LUTs and hence suffers from the same aforementioned problems.
A clever technique is direct data shaping [5], where data is shaped without prior compression to compensate for the size expansion. The input data is split into blocks, within each block the most frequent strings, e.g., 4 bits, are mapped into other strings with least number of programmed cells. The mapping dictionary is updated throughout the whole codeword and can be dynamically retrieved upon reading. This scheme does not entail rate loss and a significant shaping gain is achievable especially for compressible input sequences. However, for noncompressible input, e.g., encrypted data, it provides limited gain. Furthermore, the backward retrieval of the mapping dictionaries at the reading side incurs additional complexity as it needs to be applied many times.
B. Contribution
We propose a novel lossless yet simple endurance modulation technique that relies solely on the ECC to retrieve the original data. The main motivation behind this work is the same as in [4] with the difference that instead of using a portion of the parity bits to accommodate for the data expansion due to shaping, we use the parity bits for joint error correction and endurance demodulation. The controller in the proposed scheme flips every ns-th zero bit in each logical page prior to programming. Upon reading, the controller estimates the location of the possibly flipped bits based on the hard estimate and assigns the soft information to those bits accordingly. The soft information is then fed to the ECC soft decoder to retrieve the original data. The computational complexity for modulation/demodulation is hence negligible but the average number of soft decoding iterations is expected to be higher w.r.t. the case of no endurance modulation. In terms of storage overhead, there is no need for a LUT and only a single parameter, i.e., ns needs to be stored for each logical page and block. Moreover, by dynamically adjusting the shaping parameter ns, there is no performance degradation. As the device ages, the full error correction capability of the ECC will be needed to handle the increasing number of errors induced by the channel. Hence, before reaching a prescribed RBER threshold, the memory controller disables the endurance modulation and operates in normal mode.

III. PRACTICAL CONSIDERATIONS
A crucial parameter for our endurance modulation is ns which determines the percentage of flipped bits prior to programming. On one hand, choosing ns too large will ensure correct retrieval of the original data to the detriment of a small endurance gain. On the other hand, choosing ns too small yields a large endurance gain at the expense of an increased read failure rate. Therefore, ns must be carefully chosen to provide a tangible endurance gain without compromising the flash memory reliability.
Additionally, if our endurance modulation is applied over the whole codeword, a single channel error can incur many errors in the subsequent bits which might cause a read failure. Therefore, the endurance modulation must be applied

As the device ages, the ECC may fail to retrieve the original data, hence before reaching a certain RBER threshold, the memory controller disables the endurance modulation and operate in normal mode. The prescribed RBER can be checked after each successful read; the controller compares the successfully decoded data (e.g., by passing the CRC) and the read data to estimate the channel RBER.
Our endurance modulation scheme does not flip all the bits with equal probability, in fact some bits may not be flipped at all, e.g., the first ns − 1 bits of each chunk, leading to the different cells being subject to different levels of stress. Wear leveling can be achieved by interleaving the modulated data prior to programming. The interleaver may be a simple circular shift register that moves by one position with each write. Upon reading, a deinterleaver is applied prior to the endurance demodulation, which yields the complete system depicted in Figure 2.
Finally, we have so far assumed an SLC flash that stores a single bit per cell but the proposed scheme can be generalized to cover denser cells, e.g., MLC (2 bits/cell), TLC (3 bits/cell) and QLC (4 bits/cell) as well. The separate encoding/decoding of the logical pages suggests treating each logical page independently when applying our endurance modulation. Assume a TLC flash memory with balanced Gray labelling as depicted in Figure 3; we have only three variables, namely, the shaping parameter for each page so not every probability distribution can be realised. Hence, a suboptimal solution can be derived as follows, let the probability of ‘0’ for the lower, middle and upper pages be given by α, β and γ , respectively. The probability of each voltage level can be derived accordingly, for instance, the voltage levels ER, A and B in Figure 3 will appear with probability (1 − α)(1 − β)γ , (1 − α)(1 − β)(1 − γ ) and α(1 − β)(1 − γ ), respectively. The probabilities α, β and γ can be chosen to optimise the endurance gain defined in (1) and the shaping parameter for each page can be then deduced.
IV. RESULTS
We studied the theoretical limits of endurance modulation for TLC flash memory with balanced Gray labelling (cf. Figure 3) through density evolution analysis [10]. The shaping parameter is varied for each P/E cycles count and the smallest

value of ns such that the residual BER is below 10−6 is the optimal choice denoted by n∗s . The assumed regular LDPC code has rate 8/9 with the check node and variable node degrees given by 3 and 27 respectively. The choice of binary LDPC code is motivated by the study in [11] and references therein. As the logical pages are separately encoded/decoded, the density evolution analysis is carried out for each logical page independently. Our simulation model for flash memory cells takes into consideration several sources of distortion, namely parasitic coupling, inaccurate programming, random telegraph and retention noises.
1) Programming inaccuracy and parasitic coupling: The programming of the cells is done through multiple program/verify rounds where a charge of Vp is added to all cells that have not reached their target level. The induced vertical and horizontal parasitic coupling is calculated by multiplying the added charge by the coupling coefficients and adding the resulting value to the victim WL. This is repeated until all cells on the WL have reached their target levels. It is worth noting that in our simulation we program the WL in two steps; a tentative step consisting of programming the victim WLi to a target voltages vi that is slightly smaller than the nominal values vi = vi − δ. Next, the aggressor WLi+1 is tentatively programmed to vi+1 = vi+1 − δ. A final step; the victim WLi is programmed to its intended voltages vi taking into account the vertical ICI from its direct aggressor WLi+1 (in the tentative round) followed by programming the aggressor WLi+1 to its target voltages vi+1 and adding the resulting ICI to the victim WLi.
2) Random telegraph noise (RTN): It is well known that the read voltage drifts between subsequent reads of the cells, this effect can be accurately modeled as an additive Laplace distributed noise [12].
3) Retention noise (RET): The wear out effect on the reliability of data retrieval due to P/E cycles as well as ageing is modeled by an additive Gaussian distributed noise [13].

V. CONCLUSION
We tackled the problem of low-complexity data shaping to improve the endurance of flash memory. A lossless yet simple endurance modulation/demodulation scheme has been proposed, whereby the employed ECC is used for joint error correction and demodulation. The proposed scheme can be executed by the controller using the current hardware architecture of the NAND flash system. No LUT is needed for shaping and the modulation/demodulation computational complexity is negligible. The endurance gain will depend on the correction capability of the employed ECC; a stronger code yields higher endurance gain. For practical applications, the employed ECC must be optimised for both error correction and endurance demodulation which is beyond the scope of this brief.

Note that the reading latency is dominated by the sensing process [16] which is expected to become even more pronounced in next generation 3D QLC flash memory. Moreover, flash memory from major vendors supports an advanced command called CACHE READ that improves the reading performance by pipelining consecutive read requests regardless of the physical locations of the pages to be read. This is achieved by using an additional cache in the NAND flash chip to store the sensed data while transferring the previously-sensed data to the SSD controller for ECC decoding, hence further reducing the impact of ECC decoding latency on the overall reading performance.
