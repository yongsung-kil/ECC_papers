Low-Complexity Dynamic Single-Minimum Min-Sum Algorithm and Hardware Implementation
for LDPC Codes

Qinyuan Zhang, Suwen Song, and Zhongfeng Wang School of Electronic Science and Engineering, Nanjing University, Nanjing, China
Email: zhangqinyuanzqy@163.com, suwsong@sina.com, zfwang@nju.edu.cn

Abstract—As a type of low-complexity decoding algorithm for low-density parity-check (LDPC) codes, the single-minimum min-sum (smMS) algorithm avoids finding the second minimum, while estimates it by adding a fixed value to the minimum instead. However, the inaccurate estimation of the sub-minimum results in obvious performance degradation. In this work, we propose an improved smMS algorithm, which adds a dynamic value to the minimum based on a special variable that can be easily computed and largely represents the convergence degree of iterative decoding. This new algorithm is thus called dynamic smMS (dsmMS) algorithm. In comparison to the standard normalized min-sum (NMS) algorithm, the performance gap for LDPC code (672,588) is narrowed from 0.55 dB of the smMS to 0.12 dB of the dsmMS. We also present a partially parallel decoding architecture for the dsmMS algorithm, and implement it under 55nm CMOS technology with an area of 0.21 mm2. Furthermore, compared with the traditional NMS decoder, the proposed design can reduce the area of the total decoder by 22%.
Index Terms—Low-density parity-check (LDPC) codes, lowcomplexity, single-minimum min-sum (smMS) algorithm, VLSI implementation.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are a class of linear block codes known for their good performance and low decoding complexity. They were first introduced by Gallager [1] in 1962. Many communication protocols have adopted LDPC codes, such as IEEE802.16e [2], IEEE802.3an [3], and IEEE802.15.3c [4]. In addition, LDPC codes have been widely used in optical fiber communication [5] and 5G communication [6]. Although the belief propagation (BP) [7] algorithm has the best decoding performance among multiple decoding algorithms, it is unrealistic in practical applications because many complex operations are involved, such as exponentiation, logarithm, and division. The min-sum (MS) [8] algorithm was proposed to simplify the check node unit (CNU) and hence reduce the decoding complexity of the BP algorithm. It only includes basic operations like comparison and addition, which reduces the computational complexity but causes noticeable performance degradation. Two variants of the MS algorithm, the normalized min-sum (NMS) and offset min-sum (OMS) algorithms, were proposed in [9] to reduce the performance loss by introducing a correction factor. In MS algorithm and its variants, the core process in CNU is to find the minimum and the second minimum of multiple absolute values. To

reduce the complexity, the single-minimum min-sum (smMS) [10] algorithm uses the minimum plus a fixed value to calculate the second minimum. But this algorithm sacrifices certain decoding performance in exchange for lowering the implementation complexity. To further improve the decoding performance, [11] proposed the variable weight MS (vwMS) algorithm in which the value is related to iterations. [11] divides the number of decoding iterations into several sections, and each section adopts a different fixed value, determined by exhaustive searching. However, in an iteration range, a fixed value is still used regardless of the convergence degree. There is still an obvious performance gap between the vwMS and the NMS.
In this paper, an improved smMS algorithm called dynamic smMS (dsmMS) algorithm is proposed, which focuses on approximating the second minimum with the sum of a dynamic value ∆d and the minimum. To estimate the value more accurately, an easily-obtained variable η is introduced, i .e., the number of non-matching bits [12] between the signs of the external messages and a-posteriori-probability (APP) values, which could well reflect the convergence degree of iterative decoding. A codeword eigenfunction f (x) needs to be pre-constructed between η and ∆d, i .e., ∆d = f (η). Applying the dynamic value, the proposed dsmMS algorithm could offer better performance than the smMS algorithm and the vwMS algorithm. For the considered codes, the dsmMS can get close to the NMS decoding within 0.1 dB. Finally, we develop a partially parallel decoding architecture for the dsmMS algorithm. Synthesis results demonstrate that about 22% of the area could be saved in comparison to the NMS decoder under 55nm CMOS technology.
The rest of the paper is organized as follows. Section II gives the notations and the preliminaries for the MS algorithm and the conventional smMS algorithm. In Section III, the proposed dsmMS algorithm is described in detail. Simulation results and performance comparisons are presented in Section IV. Section V illustrates the hardware architecture including implementation results. Finally, conclusions are drawn in Section VI.

Quasi-cyclic (QC)-LDPC codes [13] are of great interest for low-complexity hardware implementations due to their regular structures. The H matrix of a QC-LDPC code can be divided into Mb × Nb square submatrices of size z × z. Each sub-matrix is either a zero matrix or a z × z cyclically shifted identity matrix.
There are three types of decoder architectures: fully-parallel, serial, and partially parallel [14]. In a fully-parallel architecture, all rows and columns of H are handled at the same time. Because one iteration only takes one cycle, this decoder delivers the highest throughput. Conversely, there is only one CNU and one variable node unit (VNU) in the serial architecture. And the check-to-variable (c2v) and variable-tocheck (v2c) messages are all stored in memories. Partially parallel decoders offer a balance between throughput and area. Numerous CNUs and VNUs are used in a partially parallel architecture to execute row or column blocks of H in a timemultiplexed mode.

III. THE PROPOSED DECODING ALGORITHM
For the sake of description, denote the first and second minimum magnitudes of the input v2c messages in one check node by min1 and min2, respectively. To improve the performance of the smMS algorithm and maintain the advantage of low complexity, the dynamic smMS (dsmMS) decoding algorithm with flooding schedule is proposed. In the modified CNUs, we find min1 and estimate min2 through a concise method, where a dynamic value ∆d is got based on a special variable that can be easily computed and largely represents the convergence degree of iterative decoding. The idea of introducing a variable as a tool to judgment the convergence degree originates from [12] in the turbo decoding.
The relationship between APP values and external messages is shown in Eq. (8), in which these two variables tend to share the same signs as the decoding converges. Therefore, the number of non-matching bits between the signs of APP values and external messages, denoted as η, can be considered as a tool to reflect the convergence degree of iterative decoding. Considering the actual difference ∆ between min1 and min2 is also strongly correlated with the decoding convergence, we propose to construct a functional relationship between the number of non-matching bits η and the actual difference ∆ to simulate min2 with high accuracy. This functional relationship is called the codeword eigenfunction.

IV. SIMULATION RESULTS
The performance of the NMS, smMS, vwMS, and the proposed dsmMS is evaluated on the above three codes. The error-correction performance curves are shown in Fig. 2. These algorithms are simulated over the additive white gaussian noise (AWGN) channel with zero mean and variance σ2 using the binary phase shift keying (BPSK) signaling. The maximum number of iterations is set to 30. The scaling factor α is always 0.75. For fair comparison, parameters in the vwMS and smMS are set as in [11]. The decoding will be terminated when 50 failure decoding codes are collected. Only those parameters (i .e., Eq. (9)) that yield the best performance are selected for comparisons. For code (672,588) and code (672,504), the smMS algorithm exhibits noticeable performance degradation away from the NMS at bit-error-rate (BER) < 10−6. Compared with the smMS, the proposed dsmMS can lower the error floor and significantly improve the decoding performance, more than 0.4 dB for these two codes. The dsmMS algorithm leads to about 0.1 dB performance gain in comparison to the smMS for code (2048,1723). For all three codes, the proposed dsmMS outperforms the smMS and the vwMS, close to the NMS within 0.1 dB.

V. HARDWARE ARCHITECTURE
A. Top-Level Architecture
The overall block diagram of the dsmMS decoder is shown in Fig. 5, including VNUs, CNUs, LLR Memory, Compressor, Decompressor, two Barrel Shifts, and two Message Register Arrays. Compressor combines min1, the index of min1, and the product of all signs of the c2v messages into compressed information. Decompressor has the opposite function, that is, it recovers these messages from the compressed information.
The decoder uses a partially parallel architecture [14], which processes one column block of H per cycle. In this case, each decoding iteration takes Nb cycles. Assume that the H matrix of QC-LDPC code consists of Mb × Nb cyclic permutation matrices of dimension z × z. The row weight is denoted as dc

and the column weight is dv. Because multiple sub-matrices of one column block are processed simultaneously, dvz CNUs and dvz VNUs are needed. Nv in Fig. 5 is equal to dvz.
The decoding process is briefly described as follows: At the beginning of the decoding, old message registers are initialized as 0, and current message registers are set as the maximum values. By shifting and decompressing the information read from Old Message Register Array, the c2v messages can be recovered. Next, VNUs execute Eqs. (4-6). The main operation in CNUs is to find the input v2c messages of the least magnitude, and these minimum values are finally stored in the Current Message Register Array after being compressed and shifted. At last, after all column blocks in one iteration have been processed, the messages are passed from the Current Message Register Array to the Old Message Register Array.
B. Hardware Resources Analysis
The reduction in hardware resources of the proposed decoder is analyzed in this subsection. QC-LDPC code (672,588) is used as an experimental code. In the original NMS decoder, the compressed information contains min1, min2, the index of min1, and the product of all signs of the c2v messages. min1 and min2 are both required, inducing more complex check node processing units, more storage due to min2, and more congested routing. C. Synthesis Results
Our hardware design is described by Verilog language and synthesized using 55nm CMOS standard cell library. To be fair, we re-implemented the standard NMS and smMS decoders, and synthesized them under the same environment. The synthesis results for the LDPC code (672,588) are shown in Table I. Compared with the NMS algorithm, the dsmMS decoder saves 22% of the area with a small increase in delay, and it is very close to the results of smMS decoder. It is observed from Table II that the area reduction of the dsmMS decoder mainly comes from the Barrel Shifters. The essential reason for this phenomenon is the reduction in the bit width of the compressed information. The area of the Decompressor increases from 0.008 mm2 to 0.014 mm2, which has a weak effect on the overall decoder. The area of the CNUs also confirms the analysis in Section V-B.
VI. CONCLUSION
In this paper, we propose a new algorithm called dynamic single-minimum min-sum (dsmMS) algorithm for LDPC decoding. The computation complexity is reduced by approximating the sub-minimum with the sum of a dynamic value and the minimum. Compared with the smMS, using dynamic value ∆d instead of fixed value ∆fix, the proposed dsmMS can significantly improve the decoding performance.
