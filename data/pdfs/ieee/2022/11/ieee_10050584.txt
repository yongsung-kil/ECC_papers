I. INTRODUCTION 
LDPC was first proposed by Dr Gallager in 1962 and 
rediscovered by Mackey and Neal in 1996 [1]. LDPC has 
attracted considerable attention due to its excellent error 
correction capability in BP(Belief-Propagation) decoding 
algorithm. BP decoding algorithm consists of two main 
processes: horizontal update and vertical update [2]. The 
horizontal update completes the parallel update of the 
message from the check node to the variable node (C2V), 
and the module responsible for this update process is called 
the check node update unit (CNU), while the vertical update 
completes the parallel update of the message from the 
variable node to the check node (V2C), and the module 
responsible for this update process is called the variable 
node update unit (VNU). Based on the two-phase message 
passing (TPMP) mechanism proposed by Dr Gallager [3], 
the advantage of this decoding mechanism is that the CNU 
and VNU can take advantage of a completely new passing 
message, but the disadvantage is that the hardware 
utilization is low at 50% [4]. To address this problem, the 
literature [5] proposes Overlapped-NMSA [5]. The 
proposed algorithm lays the theoretical foundation for the 
design of the high-speed LDPC decoder in this paper. 
A storage addressing scheme which is more suitable for 
high parallelism decoder is proposed in this paper.. 
Compared with the storage addressing scheme used in the 
literature, this storage addressing scheme not only avoids 
address access conflicts, but is also simpler and more 
efficient. The next step in this paper is to optimize the 
design of the overall architecture of the decoder around this 
storage addressing scheme, further increasing the decoding 
rate.

III. DECODING ALGORITHM
The probabilistic BP decoding algorithm is one of the 
main decoding algorithms for LDPC, but it requires a large 
number of multiplication operations and is difficult to 
implement through hardware circuits. The Log Likely Rate 
(LLR) BP decoding algorithm replaces the multiplication 
operation in the probabilistic BP decoding algorithm with 
an addition operation, reducing the complexity of the 
operation, but the hardware circuit is still very complex to 
implement due to the presence of logarithmic operations. 
The Min-Sum Algorithm (MSA) is a further simplification 
of the LLR BP decoding algorithm, which requires only 
addition and comparison operations. NMSA is a correction 
of the MSA to bring its error correction performance closer 
to that of the LLR BP decoding algorithm. Although NMSA 
has low operation complexity and is very suitable for 
hardware circuit implementation, it has more invalid 
waiting time because it is based on TPMP decoding 
mechanism The literature [5] addresses this problem by 
proposing 
Overlapped-NMSA. 
Application 
of 
this 
algorithm starts by dividing H into blocks. For the 
horizontal update process, H is uniformly divided according 
to rows; for the vertical update process, H is uniformly 
divided according to columns, and the number of rows and 
columns contained in the smallest submatrix should be the 
same, set as K, so that the horizontal and vertical updates 
can complete the iteration process in the same time, as 
shown in Fig.3.

IV.  DECODER PARAMETER 
 
Fig. 5. Influence of quantization bits on decoding performance. 
In this paper, simulation experiments are carried out on 
the MATLAB simulation platform for the three quantization 
methods, where the left side of the quantization method 
represents the number of bits quantized by the decoder for 
the initial channel information, and the right side represents 
the number of bits quantized by the decoder for the 
intermediate passing information, which is due to the fact 
that in equation (5) C2V needs to be multiplied by a 
correction factor less than 1. Therefore, in order to ensure 
the quantization accuracy of the decoder, the quantization 
bits needs to be added by one to represent the fractional bit. 
As shown in Fig.5, the bit error rate decreases as the 
number of quantization bits increases. (7, 8) has a smaller 
gain compared to (6, 7), while (8, 9) has a larger gain 
compared to (6, 7), but also implies more hardware resource 
consumption. On balance, (6, 7) is the quantization method 
of choice in this paper. 
 
Fig. 6. Influence of maximum iterations on decoding performance. 
The maximum iterations is a key parameter that affects 
the performance of the LDPC decoder. In general, the 
higher the iterations, the better the decoding performance, 
but also the higher the decoding latency. Therefore, in order 
to balance the decoding performance and decoding latency, 
it is necessary to choose a suitable maximum iterations. As 
shown in Fig. 6, it can be seen that a maximum iterations of 
15 is a significant improvement over a maximum iterations 
of 10, while a maximum iterations of 20 is a relatively small 
improvement over a maximum iterations of 15. Therefore, 
the maximum iterations is 15 in order to obtain a high 
decoding rate with a guaranteed decoding performance.

After obtaining the above RA0 and OA0, this paper 
writes them as constants in the CNU (VNU) address 
generation unit, which is responsible for generating the read 
address(RA) of the RAM in MEM_V2C (MEM_C2V) and 
the offset address(OA) of the data sequence adjustment unit. 
When the read enable is low, the iteration process has not 
started, the RA and the OA of the RAM are reset to the RA0 
and the OA0 respectively. When the read enable is high, the 
iteration process starts, the RA is incremented by one as the 
rising edge of the system clock arrives, and when the RA 
reaches 72, the next clock returns to 0. The OA is 
incremented by one when the RA returns to 0 from 72, and 
remains unchanged for the rest of the time. The data 
sequence adjustment unit is essentially a barrel shift register, 
according to the value of the OA, the input data is circularly 
shifted to the left and then output, such as OA = 1, the input 
data will be circularly shifted 0 bits to the left; OA = 2, tthe 
input data will be circularly shifted 7 bits to the left; OA = 3, 
the input data will be circularly shifted 14 bits to the left; 
⋯⋯, the output data will be sent to the VNU and CNU for 
subsequent calculation and processing.

catenated in a specific order and then written to the

The throughput rate is an important indicator of the 
decoder and represents how fast the decoding rate is. It can

VIII. CONCLUSION 
 This paper proposes a storage addressing scheme 
suitable for high parallelism decoders, and completes the 
design of the architecture of the decoder around this storage 
addressing scheme. After the design of the decoder is 
completed, the decoder can run at a maximum clock 
frequency of about 210MHz, with a throughput rate of 
1.1Gbps, which can meet the needs of most high-speed 
communication systems.
