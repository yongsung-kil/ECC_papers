Design of Protograph LDPC Codes Using Resolvable Block Designs for Block Fading Channel
Jaewha Kim , Member, IEEE, Chanki Kim , Member, IEEE, Hosung Park , Member, IEEE, and Jong-Seon No , Fellow, IEEE

Abstract—In this letter, we propose new protograph low-density parity-check (LDPC) codes of high code rates using resolvable block designs (RBDs) for block fading (BF) channels. To analyze the performance of the proposed LDPC codes, we derive an upper bound on bit error rate (BER) using the novel method, called the gamma evolution. Finally, we numerically show that the frame error rate (FER) of the proposed LDPC codes has a slope approaching the channel outage probability.

Construction of large girth LDPC codes is also an important issue since it generally guarantees better performance for finite length performance. Especially in [11]–[13], it was shown that random and quasi-cyclic (QC) LDPC codes with large girths can be constructed using combinatorial designs. Furthermore, large girth construction of root LDPC codes was proven to be effective for BF channels [8], [14]. Likewise, the motiva-

B LOCK fading (BF) channel has been considered as a simple but effective channel model that captures the fundamental characteristics of current wireless communication systems. In [1], the necessity of low-density parity-check (LDPC) codes for 5G NR standards with robustness from fading channels has been pointed out. The construction of LDPC codes for the BF channel requires new structures to obtain the theoretical performance limit of the BF channel. In terms of LDPC codes for BF channel, root LDPC codes and generalized root protograph (GRP) LDPC codes have been extensively studied previously [2]–[9]. Also, spatially coupled LDPC codes were proposed for obtaining an arbitrary diversity order from the convolutional structure [10].

A. Channel Model
In this letter, a BF channel with L blocks is considered. First, a binary information vector of K bits is encoded into N bits using (N,K) binary linear code with code rate r = K/N, where the codeword is denoted by c = (c1, c2, . . ., cN ), ci ∈ {0, 1}. Then, the encoded bits undergo binary phase shift keying (BPSK) modulation, which is converted into x = (x1, x2, . . ., xN ), xi ∈ {±1} and experiences BF channel with block length T = N/L, where L|N . Here, fading gain of each block is αi , i ∈ [L], where the expectation value of αi2 is one.

A. Design Motivation: Resolvable Block Designs

We first introduce some definitions on the existing block

designs that are used to construct the proposed protograph

LDPC codes in this letter. First, the definition of balanced

incomplete block design (BIBD) is given as follows.

Definition 1 (BIBD [18]): A BIBD is a pair (V, B), where

V is a v-set and B is a collection of b k-subsets of V

(blocks) such that each element of V is contained in exactly

μ blocks and any 2-subset of V is contained in exactly λ

blocks. The numbers v, b, μ, k, and λ are parameters of

the BIBD.

In this letter, we only focus on the resolvable block designs.

The resolvable BIBD (RBIBD) is defined as follows.

Definition 2 (RBIBD [18]): A BIBD is called as an RBIBD

if there exists a partition R of its set of blocks B into parallel

classes, each of which in turn partitions the set V; R is a called

as a resolution.

For k = 2, an RBIBD can be constructed from the columns

of a balanced tournament design of size n, denoted by BTD(n),

making an RBIBD with λ = 1 and k = 2 [18]. BTD(n) is

defined as follows.

Definition 3 (Balanced Tournament Design [18]): A bal-

anced tournament design of order n, BTD(n), defined on a

2n-set V is an arrangement of the

2n 2

distinct unordered pairs

of the elements of V into an n × (2n − 1) array such that

i) Every element of V is contained in precisely one cell of

each column, and

ii) Every element of V is contained in at most two cells of

any row.

Each column of the BTD(n) forms a resolution, thus making

an incident matrix with 2n − 1 resolutions. For k = 3, Steiner

triple system with order v, denoted as STS(v), refers to a BIBD

with k = 3, λ = 1. An RBIBD with k = 3 is constructed using

resolvable Steiner triple system, also known as Kirkman triple

system. It is known that Kirkman triple system KTS(v) exists

when v ≡ 3(mod 6). Another resolvable design with k = 3 is

constructed from a configuration, which is defined as follows.

Definition 4 (Configuration [18]): A configuration (vμ, bk ) is an incidence structure of v points and b lines such that: i)

each line contains k points, ii) each point lies on μ lines, and

iii) two different points are connected by at most one line.

Using the result in [12], a configuration can be constructed

by deleting a row and μ = (v − 1)/(k − 1) = (v − 1)/2

columns incident to the deleted row in the STS(v). Likewise,

a resolvable configuration can be made by deleting exactly

one column from each resolution of KTS(v). Thus far, some
