Cell-wise encoding and decoding for TLC flash memories
Daniel Nicolas Bailon∗, Sergo Shavgulidze†, Ju¨rgen Freudenberger∗
∗Institute for System Dynamics HTWG Konstanz, University of Applied Sciences, Germany
Email: {dnicolas,jfreuden}@htwg-konstanz.de Web: www.isd.htwg-konstanz.de
†Faculty of Informatics and Control Systems, Georgian Technical University, Georgia Email: sshavgulidze@gncc.ge

Abstract—Automotive computing applications like AI databases, ADAS, and advanced infotainment systems have a huge need for persistent memory. This trend requires NAND flash memories designed for extreme automotive environments. However, the error probability of NAND flash memories has increased in recent years due to higher memory density and production tolerances. Hence, strong error correction coding is needed to meet automotive storage requirements.Many errors can be corrected by soft decoding algorithms. However, soft decoding is very resource-intensive and should be avoided when possible. NAND flash memories are organized in pages, and the error correction codes are usually encoded page-wise to reduce the latency of random reads. This page-wise encoding does not reach the maximum achievable capacity. Reading soft information increases the channel capacity but at the cost of higher latency and power consumption. In this work, we consider cell-wise encoding, which also increases the capacity compared to page-wise encoding. We analyze the cell-wise processing of data in triple-level cell (TLC) NAND flash and show the performance gain when using Low-Density ParityCheck (LDPC) codes. In addition, we investigate a coding approach with page-wise encoding and cell-wise reading.
I. INTRODUCTION
Portable storage media are indispensable in today’s world. The ability to store and reproduce huge amounts of data quickly and reliable is becoming increasingly important [1]. NAND flash memory is the most common technology to store data on consumer, IoT, medical devices, and automotive computing [2], [3]. These memories have a high processing speed, low energy consumption, and no moving mechanical parts.
In NAND flash memory, data is stored as an electrical charge on floating gate transistors, also called flash cells. The number of bits stored per cell changes for different technologies. With multi-level cell (MLC), triple-level cell (TLC), and quadruple-level cell (QLC) storage solutions, 2, 3, and 4 bits can be stored per cell, respectively. While MLC memory was widely used a few years ago, TLC NAND flash is now mainly used as a storage medium, and QLC is already produced. However, the reliability of these flash cells is limited, and data integrity can only be maintained by error correction coding [4].

For MLC NAND flash, it was sufficient to use harddecision decoding based on Bose-Chaudhuri-Hocquenghem (BCH) codes to provide sufficient reliability [5], [6]. Today, LDPC codes are used for flash systems with hard and softinput decoding [7], [8], [9], [10], where soft-input decoding improves the error correction performance [11], [8].
The individual bits of the cells are assigned to so-called pages, whereby many pages result in one block, and several blocks form a plane. The readout is usually done page-wise, and the data erase procedure is done block-wise. The data readout is performed by applying different threshold voltages to the flash cells. A page-wise data read also requires a pagewise encoding of the error correction code. This page-wise readout enables fast random access because fewer threshold voltages are needed compared to a complete reading procedure that determines all bits stored in one cell. For soft-input decoding, additional voltages are applied to obtain reliable information. These threshold voltages increase the read latency. Moreover, soft-input decoding methods are often more complex than hard-input decoding. For these reasons, hardinput decoding methods are attractive.
Recently, different approaches for cell-wise read and joint decoding for MLC NAND flash were introduced [12], [13]. In [12], it is shown that joint decoding of the two MLC pages improves the achievable information capacity compared to page-wise decoding. Furthermore, a progressive read method was considered to read additional soft data. Reading the entire cell data and the subsequent joint decoding improves the decoding performance. In [13], a paired page readout technique is presented for MLC flash. It shows that by reading the data of the entire cell, the additional information increases the performance in worst-case scenarios. This procedure reduces the read latency compared to soft readout. In [14], [15], the mutual information of TLC flash was analyzed and shown that page-wise read operations imply a performance loss regarding the achievable cell capacity.
This work investigates and applies methods for cell-wise processing of the entire cell data on TLC flash memory. We use LDPC codes and a soft-input decoder. Our research focuses on a comparison of page-wise and cell-wise processing. De-

II. PRELIMINARIES
NAND flash memories are organized such that many flash cells correspond to one page, many pages correspond to one block, and a flash memory consists of many blocks [16]. It is possible to read, write and erase the information stored in the cells. The physical organization of the NAND flash memory allows a page-wise read and write and a block-wises erase. NAND flash memories are constructed such that the spare area used for the error correction and meta-data has a defined size, and each page typically has the same spare area [16]. Consequently, using page-wise encoding implies that each page uses the same error correcting code.
The threshold voltage of a cell is determined by the charge level of the floating gate. A voltage distribution models the threshold voltages across different cells. In Fig.1, we see idealized voltage distributions of the charge states of a TLC flash memory, where the conditional voltage distributions are modeled as Gaussian distributions with equal variance. This model is equivalent to a transmission with an 8-ASK modulation over an additive white Gaussian noise (AWGN) channel.
During the readout process of the flash cell, several reference voltages ri are applied to distinguish the different charge states Si. The reference voltage is defined by the distribution of the charge states Si and Si+1. Each charge state represents binary digits. In the case of a TLC, a state represents three bits, which are assigned to different pages. Such that the first digit is assigned to the most significant bit (MSB) page, the second to the center significant bit (CSB) page, and the last to the least significant bit (LSB) page. Typically Gray codes are used for the labeling of the charge states. With Gray codes, two adjacent binary combinations differ only by one digit. III. CAPACITY CONSIDERATION

Several physical effects impact the reliability of NAND flash cells, e.g., charge loss over time, program erase cycling, inter-cell interference, and temperature-related effects [17], [18]. The use of error-correcting codes is essential to achieve data integrity. The code rate R = k/n, where k is the code dimension, and n is the code length, is limited by the achievable channel capacity such that R < C. In the case of flash memory, the channel input X is the charge state of the flash cell, and the channel output Y is obtained by applying several reference voltages ri. However, only a subset of the possible reference voltages may be applied. IV. PROPOSED CODING SCHEME

The multilevel coding approach requires different code rates

for the different pages. Furthermore, it requires processing

of the first coding stages when a lower level should be

decoded, i.e., joint processing of all pages at the decoder.

An alternative approach is a bit-interleaved coded modulation

(BICM), where all bits are encoded with the same code, and

bit-wise interleaving is applied at the encoder output [29], [30].

BICM has been developed for fading channels but has turned

out to be capable of approaching the capacity of high-SNR

AWGN channels [31]. In the context of error correction for

flash memories, BICM was mainly considered for MLC [30],

[32]. We apply this method to TLC flash memories.

The BICM approach requires an appropriate soft-decision

metric as an input to the decoder. For TLC memories, the soft-

decision metric enables soft-input decoding without reading

at additional reference voltages. The demapper exploits the

different error probabilities of the different charge states.

charge states. We partition the code C into two subsets for the demapping of the ith bit. The set Ci(0) ⊂ C is the set of binary vectors that have a zero in the ith position. Likewise, the codewords of Ci(1) ⊂ C have a one in the ith position.

V. PERFORMANCE ANALYSIS
In this section, we present simulation results for a TLC model to compare the performance of cell-wise and pagewise processing, focusing on hard decision decoding. First, we consider a code for 1kB payload data and code rate R = 0.83, i.e., an LDPC code with dimension k = 8192 and code length n = 9870. These code parameters are proposed in the IEEE standard [10].
Fig. 5 shows the page-wise decoding and joint processing for Gray code 1. In this case, the worst-case page is the CSB page. Note that as for the mutual information, the residual word error rates (WER) of the LSB and MSB page are equal. Using the hard-decision decoding, the joint processing is even better than the best-case page-wise decoding. This gain results from the additional information obtained by the joint LLR calculation.
In addition, results with soft-decision decoding with one soft-bit are shown in Fig. 2 for the CSB page. The threshold voltages for the soft-input are chosen to maximize the mutual information. The decoding performance of the soft-input decoding exceeds the hard decoding.
Next, we consider a code for 4kB payload data and code rate R = 0.85, i.e., with k = 32768 and n = 38556. This code length is common for flash memory devices [34]. Fig. 6 contains the results for this code. The results are comparable to the results in Fig. 6, which shows that increasing the code length achieves a gain in the achievable code rate.

VI. CONCLUSION
In this paper, we have investigated cell-wise and pagewise processing for TLC NAND flash memories. The joint processing increases the mutual information and improves hard-input error correction performance with LDPC codes.
The BICM approach requires that all pages associated with a cell must be read together. This may increase the read latency because more read voltages must be applied. However, additional voltages for soft-reading can possibly be omitted. Moreover, the proposed processing can also be used with pagewise encoding, where the information of the other pages is only used when the page-wise hard-input decoding fails.
