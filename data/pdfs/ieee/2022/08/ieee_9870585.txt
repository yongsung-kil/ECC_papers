Abstract—In this paper, we propose a cubic Turbo Product code (TPC) with the Bose-Chaudhuri-Hocquenghem (BCH) code and the single parity-check (SPC) code as components, as well as an improved turbo-oriented adaptive belief propagation decoding based on the partial layered and row-column weight scheduling (PL-RC-TAB). The proposed coding scheme has comparable encoding and decoding complexities with the two dimensional (2D) TPC based on the classical turbo-oriented adaptive belief propagation (TAB) decoding, but can achieve faster decoding convergence rate and improved error correction performance.
Index Terms—adaptive belief propagation (ABP), BoseChaudhuri-Hocquenghem (BCH) codes, single parity-check (SPC) codes, Turbo Product code (TPC).
I. INTRODUCTION
The Product code was first proposed by Elias in 1954 [1]. Limited by the error correction performance of the adopted hard decision decoding, the Product code was not practical at that time. With the emergence of the Turbo code [2] and the associated iterative decoding algorithm, the Product code came back to people’s sight. In 1994, Pyndiah applied the iterative decoding method of Turbo code to the block Product code, named Turbo Product code (TPC), and achieved good error correction performance [3]. Due to the excellent properties of reduced decoding latency, lower decoding complexity, good anti-interference and anti-burst error capabilities, the Product code has been widely applied in a variety of communication systems (e.g. optical communications [4], satellite communications [5], etc.) and diverse communication standards (e.g. IEEE 802.16 [6], IEEE-1901 [7], etc.).

Mahran et al. first proposed an adaptive decoding algorithm for TPCs, which realized the adaptive adjustment of unreliable bits and reduced the decoding complexity by setting the received signal threshold related to the signal-to-noise ratio (SNR) [9]. Dang et al. proposed an adaptive decoding algorithm that does not require the SNR and adjusts parameters according to the statistical characteristics of received code words, thus reducing the decoding complexity [10]. The hybrid decoding algorithm proposed by Al-Dweik firstly adopts the soft decision to reduce the bit error rate (BER), and then uses the hard decision to correct residual errors [11]. N. Le et al. proposed a distancebased decoding (DBD) algorithm, which has better error correction performance than Chase-Pyndiah algorithm without searching the competing code words [12].
In 2006, Jiang introduced an adaptive belief propagation (ABP) algorithm for the soft decoding of Reed-Solomon (RS) code, which iteratively and adaptively sparsifies the highdensity parity-check (HDPC) matrix of RS code to mitigate the effect of short cycles [13]. In 2009, Je´go further investigated a simplified version of ABP algorithm for TPCs, called turbooriented adaptive belief propagation (TAB) algorithm. The complexity of TAB algorithm is significantly reduced by moving the matrix adaptation step outside of the BP iteration loops. A reduced-complexity version of the TAB algorithm was also proposed to offer a trade-off between the performance and the decoding complexity. Simulation results of TAB algorithm show that the belief propagation based on adaptive paritycheck matrices is a practical alternative to the Chase-Pyndiah algorithm for TPCs [14]. Recently, an improved perturbed ABP decoding algorithm was proposed in [15] for HDPC codes, including TPCs.

B. TAB decoding of TPC
The turbo decoding of Product code involves sequentially decoding rows and columns using a SISO decoding algorithm for several iterations. In each decoding process, the bitreliability is computed as L(i+1)′ = Lch + α(i)ω(i), where Lch is the initial reliability of each bit in the received vector expressed in terms of the log-likelihood ratios (LLR) observed from the channel; ω(i) is the extrinsic information obtained by subtracting the a prior information L(i) (i.e., L(i−1)′ ) from the a posteriori information f (i); and α(i) is a scaling factor that reduces the effect of the extrinsic information in the previous decoding iterations.
The TAB algorithm is a simplified version of the standard ABP algorithm [13] for the turbo decoding of Product codes [14]. The TAB algorithm has the following main steps.
• Step 1: Rearrange the received bit-reliability vector L(i)′ of i -th iteration from the least reliable one to the most reliable one.
• Step 2: Implement the parity-check matrix adaptation with Gaussian elimination (GE) outside of the BP iterations, where the identity part of the parity-check matrix corresponds to the least reliable ones of L(i)′ .
• Step 3: Implement the standard BP algorithm to generate the a posteriori information f (i) with few iterations (named as local iterations, and usually set as 3 to 5).
• Step 4: Compute the bit-reliability vector of i +1-th iteration, i.e., L(i+1)′ , and implement the turbo process (named as global iterations). No damping coefficient [13] is needed for the TAB algorithm. Instead, the reduction of the extrinsic information effect is done during the bitreliability computation.

A. Encoding Structure of BCH-SPC Cubic TPC

The proposed cubic TPC is constructed based on the 2D

TPCs in the X direction and Y direction. In order to avoid

increasing the encoding complexity on the third dimension,

the cross parity-check technique is adopted in the Z direction.

The cross parity-check method is also called horizontal and

vertical parity-check. It arranges the information codes into

a matrix, and checks them simultaneously in both vertical

and horizontal directions according to the single parity-check

principle. The cross parity-check method can correct one bit

error by detecting the error position according to the fact that

neither a row nor a column meets the supervision relation.

Measurement shows that the cross parity-check method can

reduce the bit error rate by 2 to 4 orders of magnitude, which

is widely used in communications, storage systems and some

computer external devices, etc.

B. Decoding Structure of BCH-SPC Cubic TPC

For the proposed cubic TPC, the 2D BCH TPCs in page 1 to page n3 − 1 are sequentially decoded with the proposed PL-RC-TAB algorithm, then the cross parity-check is implemented to further correct the still existed error bits. In the proposed PL-RC-TAB decoding algorithm, the partial layered and row-column weight scheduling strategies are adopted to enhance the decoding convergence rate, so as to reduce the SISO decoding complexity. Moreover, the partial updating strategy can further improve the error correction performance.
1) Layered scheduling: The authors have studied in the early work [15] that, the layered scheduling is suitable for the ABP decoding of HDPC codes (e.g. BCH code and RS code), which has the fastest convergence rate and similar error correction performance, compared with the flooding and the shuffle scheduling strategies. The reason is that the parity-check matrix is transformed into a systematic matrix after GE in each ABP iteration. In the identity sub-matrix

part, a variable node (VN) has only one neighboring check node (CN). Due to the column-by-column iteration pattern of shuffled scheduling and the VN to CN (V2C) update in the flooding scheduling, there is no V2C update in the identity sub-matrix part. On the other hand, in each row of the layered scheduling iteration, all the V2C and check to variable (C2V) messages of the neighboring VNs can be updated. Therefore, the layered scheduling is adopted in the proposed decoding algorithm.
2) Row-column weight scheduling: The authors in [16] discussed the column weight based scheduling of shuffled BP decoding for irregular low-density parity-check (LDPC) codes, which can noticeably increase the convergence speed without introducing additional complexity or error rate degradation. Motivated by this weight based scheduling strategy, we also rearrange the updating order after GE and before the BP iteration. Considering the layered scheduling adopts the layerby-layer iteration pattern, we first rearrange the updating sequence based on the high-to-low row weight ordering. If some rows have the same row weight, then adjust their sequence again based on the high-to-low column weight ordering. The basic principle of the row-column weight scheduling is that updating the nodes with more connections first, then the useful information can be propagated more quickly, thus leading to a faster decoding convergence rate.
3) Partial updating strategy: For TPCs, some edges of short cycles in the HDPC component codes may be skipped by partial updating, which can avoid the error propagation in the local and global iterations during the TPC decoding, leading to a reduced decoding complexity and improved error performance. In this section, we benchmark the error correction performance and the convergence rate of the proposed BCHSPC cubic TPC coding scheme against its component 2D BCH TPCs with classical Chase-Pyndiah algorithm [3] and the TAB algorithm [14]. For TAB and the proposed PL-RCTAB algorithms, the maximum global iterations and the local iterations are set as iglocal = 8 and ilocal = 5, respectively. For the proposed cubic TPC, the code word length in the Z direction is set as n3 = 20.
Fig. 3 shows the BER performance comparison with (32, 26)2 BCH TPC, which has the code rate of 0.66 and the error power of t = 1. For the proposed (32, 26)2 BCHSPC cubic TPC, the code rate is 0.63, with a rate loss of 0.03, and the error correction power is t = 2. The decoding gains at the BER of 10−6 is about 0.45dB compared with the Chase-Pyndiah algorithm. Fig. 4 shows the BER performance comparison with (32, 21)2 BCH TPC, which has the code rate of 0.43 and the error power of t = 2. For the proposed (32, 21)2 BCH-SPC cubic TPC, the code rate is 0.41, with rate loss of 0.02, and the error correction power is t = 3. The decoding gains at the BER of 10−6 is about 0.3dB compared with the TAB algorithm. Accordingly, it can be concluded that the proposed coding scheme have better error performance than the 2D BCH TPC with a certain rate loss.
Fig. 5 and Fig. 6 show the error correction performance and the convergence rate comparisons with (16, 11)2 BCH TPC, which has the code rate of 0.47 and the error correction power of t = 1. The proposed (16, 11)2 BCH-SPC TPC with the code rate of 0.45, and has about 0.6 dB decoding gains at the BER of 10−6. Moreover, the proposed code can also achieve a noticeable reduction of average iteration loops, especially in the low SNR region. The average convergence rate gains in the SNR region of [0, 4]dB is about 48%.

V. CONLUSIONS
In this paper, we have introduced a cubic TPC by making
the cross parity-check on several 2D TPCs in the third dimen-
sion, which would not increase the encoding complexity but
one more bit of error correction power. The rate loss of the
proposed cubic TPC compared with the 2D TPC components is small enough to be ignored if the codeword length n3 in the Z direction is properly set. Moreover, the proposed PL-RC-
TAB decoding can accelerate the convergence rate and further
improve the error correction performance compared with the
prior-art SISO decoding algorithms for TPCs.
