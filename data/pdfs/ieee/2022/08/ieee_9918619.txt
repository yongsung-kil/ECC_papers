Performance Evaluation of Burst Error Correction by LDPC Coding 
and Iterative Decoding System in Magnetic Tape Drive Using a BaFe 
Magnetic Tape  
 
Yasuaki Nakamura1, Taisei Akamatsu1, Madoka Nishikawa1, and Yoshishiro Okamoto1 
 
1Graduate School of Science and Engineering, Ehime University, Matsuyama, Japan,  
{nakamura, akamatsu, nishikawa, okamoto}@rec.ee.ehime-u.ac.jp 
 
The tolerance for burst error of the interleaved low-density parity-check (LDPC) code is evaluated with the cross-interleaved Reed–
Solomon code (CIRC) in the magnetic tape drive using a BaFe magnetic tape. We evaluate the interleaved LDPC coding and iterative 
decoding system with burst error detector (LDPC-BD) system by required signal-to-noise ratio (SNR) to achieve "No errors" for the 
burst-like deterioration such as media defects to the CIRC with erasure error correction (CIRC-EC) system by computer simulation. 
The results clarify that the LDPC-BD system tolerates the longer burst error than the CIRC-EC system. In addition, they also clarified 
that the LDPC-BD system achieves "No errors" with the 3.0-4.5 dB lower SNR than the CIRC-EC system. 
 
Index Terms—Magnetic tape drive, Burst error correction, LDPC code, Iterative decoding. 
 
I. INTRODUCTION 
HE TAPE SYSTEMS provide high reliability, a long media 
lifetime, low power consumption, and low total cost of 
ownership, making them well suited for the cost-effective long-
term retention of data for the backup [1]. To continue increasing 
the cartridge capacities of magnetic tapes, studies on the high-
density recording have been conducted. Above all, the magnetic 
tapes using low-cost barium ferrite (BaFe) magnetic material 
are attracting attention as tapes that achieve a 123 Gbits/in2 
recording areal density [1]. Furthermore, to meet the future 
needs for high-reliability information storage, it is vital to be 
tolerant of burst errors of tape systems.   
In this research, we evaluate the tolerances for burst error of 
interleaved low-density parity-check (LDPC) code [2] with the 
conventional cross-interleaved Reed–Solomon (RS) code 
(CIRC) [3],[4] in magnetic tape drive using a BaFe magnetic 
tape. 
II. EVALUATION SYSTEMS 
Figure 1 shows the block diagram of 6-way interleaved 
LDPC coding and iterative decoding system with a burst 
detector (LDPC-BD).  In the figure, the input data sequence is 
encoded with 202,175 bits/unit. In the 6-way interleaved LDPC 
encoder, the input data sequence is divided into 33,696 bits by 
a 6-way bit inter-leaver, and the regular-(3, 17) LDPC code 
shall be applied to every 33,696 bits. Here, the coding rate of 
the LDPC-BD system is set to 0.824 which is equivalent to that 
of the CIRC with erasure error correction (CIRS-EC) system [4] 
for a conventional system. Finally, the recording sequence is 
output through the 6-way bit de-interleaver. Interleaving 
prevents the burst error from concentrating on one LDPC code 
set, giving us almost full error correction capability of the 
LDPC code. The recording sequence is recorded on the BaFe 
magnetic tape model [5]. We assume that a Lburst channel bit 
length of burst-like signal degradation occurs due to the media 
defect with the amplitude fluctuation which is detected by the 
burst detector composed of a reproducing waveform, an 
equalizer output, an a posteriori probability (APP) decoder 
output, and so on [6], [7].  It is also assumed that media noise 
is due to jitter in which the magnetization transition point 
fluctuates with a standard deviation of 10% concerning the 
channel bit interval 𝑇𝑇𝑐𝑐 , in addition, additive white Gaussian 
noise (AWGN) is superimposed as the system noise. When the 
standard deviation of AWGN fell in bandwidth equals to a 
channel bit rate 𝑓𝑓𝑐𝑐= 1/𝑇𝑇𝑐𝑐  is expressed as 𝜎𝜎𝑠𝑠 , the signal-to-
noise ratio (SNR) for the system noise is defined as SNR𝑆𝑆=
20 log10(𝐴𝐴/𝜎𝜎𝑠𝑠) [dB], where 𝐴𝐴  is the peak value of the 
reproducing waveform from an isolated magnetic transition. 
The high-frequency noise at the reading point is suppressed by 
the low-pass filter (LPF) with a cutoff frequency 𝑥𝑥ℎ= 0.4 
normalized by 𝑓𝑓𝑐𝑐. The output of LPF is equalized by passing 
through the finite impulse response (FIR) filter with 𝑁𝑁𝑡𝑡= 15 
taps, whose transfer characteristic is defined so that an isolated 
channel bit response is equalized by the LPF and the FIR filter 
to the extended partial response [8] class-4 (EPR4) target [1]. 
The EPR4 channel output is fed to an APP decoder, then the 
log-likelihood ratio (LLR) of decoding reliability for the EPR4 
channel is calculated. The LLR sequence from the APP decoder 
is input to the burst detector and the sum-product (SP) decoder

Figure 2 shows the block diagram of the CIRS-EC system. 
In the figure, the input data sequence is encoded with 25,272 
bytes/unit over Galois field GF (28) under 8 bits as 1 byte [4]. 
Figure 3 shows the data set by the C1 and C2 CIRC encoders. 
The C1 CIRC encoder divides the input data sequence into 
every 468 bytes by a 54-way byte interleaver shown in Fig. 3, 
and a 6-byte error correctable parity C1 shall be applied to the 
row direction. In addition, a 5-byte error correctable parity C2 
shall be applied to each 54-byte. Finally, the 64-way byte 
interleaver converts the recording sequence into the input data 
sequence order. Here, the total coding rate of the CIRC-EC 
system is 0.823 [4]. The EPR4 channel output is fed to a Viterbi 
detector for EPR4 and a burst detector. A burst detector is the 
same as the LDPC-BD system and outputs the burst information. 
The erasure correction and the normal error correction are 
performed by C2 and C1 CIEC decoders. In this research, the 
iterative decoding is performed between C2 and C1 CIEC

III. PERFORMANCE EVALUATION AND CONCLUSIONS 
Figure 4 shows the relationship between Lburst and required 
SNRS when the systems achieve "No errors". The iteration 
parameters of 𝑖𝑖𝑐𝑐𝑐𝑐𝑐𝑐𝑐𝑐 , 𝑖𝑖𝑠𝑠𝑠𝑠 , and 𝑖𝑖𝑔𝑔𝑔𝑔𝑔𝑔𝑔𝑔𝑔𝑔𝑔𝑔  are set to 4, 10, and 10, 
respectively. The symbols of open and filled blue squares and 
open and filled red circles show the performances of the CIRC 
system without EC, the CRIC-EC system, the LDPC system 
without burst detector, and the LDPC-BD system, respectively. 
As can be seen from the figure, the LDPC-BD system has a 
tolerance for a longer burst error compared with the CIRC-EC 
system. In addition, it is also clarified that the LDPC-BD 
system achieves "No errors" with the 3.0-4.5 dB lower SNR 
than the CIRC-EC system.
