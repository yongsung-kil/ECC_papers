Cycle-Free Windows of SC-LDPC Codes
Emily McMillon and Christine A. Kelley Department of Mathematics, University of NebraskaŌĆōLincoln, Lincoln, Nebraska 68588

AbstractŌĆöIn this paper, we define a window code to be the portion of an SC-LDPC code seen by a single iteration of a windowed decoder. We consider the design of SC-LDPC codes for windowed decoding via optimization of the window code. In particular, because iterative decoding is optimal on codes with cycle-free graph representations, we ask fundamental questions about the construction and parameters of cycle-free window codes. Further, we show that it is possible to have an SC-LDPC code with cycles and with cycle-free window codes.
I. Introduction
Spatially-coupled low-density parity check (SC-LDPC) codes are a special class of LDPC codes originally introduced by Felstro┬©m and Zigangirov in [1]. These codes exhibit a phenomenon called threshold saturation [2], meaning that SCLDPC codes converge quickly under iterative decoding to relatively error-free solutions.
Due in large part to their highly repetitive structures, SCLDPC codes are amenable to windowed decoding, which is a process by which small portions of a code are decoded sequentially. This sequential decoding is particularly conducive to applications such as wireless communication and audio and video streaming [3], [4]. While there are exceptions, such as [5] that examines decoding thresholds for different window parameters, most existing work has focused on optimizing the parameters of the entire SC-LDPC code. Since at any given time step, only a small window of the code is active, in this paper, we instead optimize the portion of the code seen by the windowed decoder at a given time step, which we call the window code.
There are many different aspects one could ŌĆ£optimizeŌĆØ with respect to the window code of an SC-LDPC code, including distance, rate, degree distribution, and cycle structure. While we plan to examine each of these aspects in the future, this work will focus primarily on cycle structure. In particular, because iterative decoding is optimal on codes with cyclefree graph representations [6], we ask fundamental questions about the construction and parameters of cycle-free window codes. Iterative decoding on a cycle-free graph is optimal in that it avoids combinatorial structures such as stopping sets and absorbing sets that are known to cause iterative decoder failure on certain channels [7], [8]. Cycle-free codes do not see much practical use, as they have poor distance properties [9]. However, because window codes are small pieces of a larger code, they present an opportunity to mix the best of both worlds: optimal iterative decoding and good rate/distance parameters.
This paper is organized as follows. In Section II, we provide notation and background on SC-LDPC codes. In Section III,

we provide the definition and motivation for window codes. Section IV contains results on parameters of cycle-free window codes. Section V concludes the paper.
II. Preliminaries
In this section, we introduce relevant notation and terminology. Let C be a binary [n, k] linear code with block length n, dimension k, and minimum Hamming distance denoted by dmin(C). If C is represented by parity check matrix H, its associated Tanner graph is defined as the bipartite graph G = (V, W; E), where the vertices in part V correspond to the columns of H, and the vertices in W correspond to the rows of H. These are referred to as variable and check nodes, respectively. There is an edge between vi Ōłł V and w j Ōłł W if and only if (H) j,i = 1. When H is sparse, C is a low-density parity check (LDPC) code.
SC-LDPC codes may be constructed by coupling L copies of a base Tanner graph, where L is called the coupling length [4]. Viewing these copies as being in positions 0, 1, . . . , L ŌłÆ 1, this coupling is done by a process termed ŌĆ£edge spreading,ŌĆØ whereby edges connected to variable nodes at position i are spread to check nodes at positions i, i + 1, . . . , i + m such that the variable node degrees of the base graph are maintained and any edge is connected to some copy of the same check node to which it was incident in the base graph. Here, m is called the coupling width. This edge spreading is done so that the way in which edges are spread at position 0 is replicated at positions 1, 2, . . . , L ŌłÆ 1. If additional check nodes are introduced at the end of the coupled graph to complete the connections, the code is said to be terminated. Alternatively, allowing the edges at the end of the coupled graph to wrap around and connect to the beginning of the graph results in a tailbiting code. The coupled graph is often taken as the protograph of an SC-LDPC code.
Remark II.1. Alternatively, SC-LDPC codes may be constructed from array-based codes and making use of a cutting vector [10]. It has been shown that many common SC-LDPC code constructions can be obtained from an algebraic graph lift with appropriate constraints on the base graph [11].

III. Window codes
In this section we review windowed decoding and introduce the notion of window code.
Definition III.1. A windowed decoder runs on a small window of nodes and slides left to right along the received bits as it decodes. The window length, denoted W, is defined to be W consecutive positions of constraint nodes and all of their adjacent variable nodes [12]. For i Ōłł {0, . . . , LŌłÆ1}, the window at position i, denoted Wi, consists of the check nodes in positions i through i+W ŌłÆ1, their adjacent variable nodes, and the edges between them. The set of edges in a given window is called a window configuration of the windowed decoder.

Due to the repetitive structure of SC-LDPC codes, in practice, almost all window codes of an SC-LDPC code will be identical. Hence, in optimizing the window code structure of an SC-LDPC code, the majority of our attention can be focused on a single window code, which we call the typical window code, defined next.
Definition III.5. Let C be an SC-LDPC code of length L, window length W, and memory m. Then the window codes at positions m through L ŌłÆ W are identical. We call this window code the typical window code of C and denote it W(C), or, when C is understood, we simply denote it W. Further, we define the window code distance to be the minimum distance of the typical window code, and denote it by dW, i.e. dW =

In a future work, we plan to compare distance properties of a window code to those of the SC-LDPC code. We will also provide a general construction of SC-LDPC codes with cycle-free window codes and analyze their performance to determine if the iterative decoding performance gained by having cycle-free window codes is worth the rate and distance tradeoffs. Finally, we will consider the feasibility of constraining check/variable node degrees within a cycle-free window.
