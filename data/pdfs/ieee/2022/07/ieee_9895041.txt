Abstract—The Belief-propagation (BP) algorithm is a powerful decoder for LDPC codes. However, the BP algorithm requires the high complexity of the check node operation. Therefore, in the previous works, a lookup table (LUT)-based decoder has been proposed for LDPC codes. In this work, we propose the LUT design for protograph LDPC codes. Our LUT design can support the parallel edge, variable node with degree-1 and punctured variable node. As a result, the coding losses of 2-, 3- and 4-bit quantization of LUT decoder are 1.6, 0.5 and 0.1 dB compared to BP algorithm.
Keywords—protograph LDPC code, channel quantizer, LUT-based decoder, protograph density evolution
I. INTRODUCTION
Low-density parity-check (LDPC) codes invented by Gallager [1] are wildly used in many applications such as data storage technologies, wireless fidelity (Wi-Fi) and 5G mobile communications. The protograph [2] is a blueprint for constructing LDPC codes. To obtain the most powerful performance of the LDPC codes, the belief-propagation (BP) algorithm [3] is generally used as the LDPC decoder. However, the BP algorithm requires high complexity of the check node operation. Thus, the lookup table (LUT)-based LDPC decoder is introduced to reduce the complexity and memory usage. In 2008, Kurkoski et al [4] proposed the finite alphabet decoder for regular LDPC codes. The LUTs are designed by maximizing the mutual information. In 2015, Meidlinger et al [5] designed the LUTs of variable

In 2018, Lewandowsky et al [7, 8] proposed the LUTs decoder for regular and irregular LDPC code. The channel quantizer and LUTs are designed by the information bottleneck method (IBM). In [9], the LUTbased decoder is designed for 5G LDPC code. However, the base matrix, which can be represented by a protograph, is converted to the degree distribution pairs. Then, the LUT operations are designed for the degree distribution pairs instead of the protograph. Therefore, in this work, we will design the LUT-based decoder for the protograph LDPC codes. In our design, the parallel edge, variable node with degree-1 and punctured variable node are allowed. We will use the density evolution (DE) to design the channel quantizer and LUT-based decoder by minimizing the theoretical error performance. Note that the DE was used to design LUT-decoder for the conventional LDPC code [10]. In this work, we consider AR4JA used in CCSDS [11]. The results show that the LUT-based decoder with 4-bit quantization can approach the BP decoder with the full channel message. Moreover, we will show that the LUTs of the punctured node can be eliminated.

B. Design of LUT Operation

To design the channel quantizer and LUT decoder, Firstly, the quantized LLRs are computed by (10) while its integer LLRs are paired. Secondly, the output LLR of the node operation is quantized by (11) with an optimal quantization boundary. The output LLRs are mapped to the integer for each quantization interval. Then, the input and output mapping are converted to the integer LUT. In this work, we will use the error probability computed from (19) as a cost function. The quantization boundary can be classified into 3 groups: channel quantizer, check node quantizer and variable node quantizer. C. Variable Node with Degree-1 and Punctured Variable Node
In protograph code, the parallel edge, variable node with degree-1 and punctured variable node are allowed. The parallel edge refers to more than one edge connected

to the same variable node or check node; the variable node with degree-1 is the variable node which has one edge; the punctured variable node is the untransmitted variable node. Here, these properties can reduce the number of LUTs. For the parallel edge, the outgoing LLR from check node or variable node can use the same LUTs. For the variable node with degree-1, the LLR of channel output can be passed to the check node without LUTs operation at the variable node. For the punctured variable node, the number of LUTs of check nodes and the variable node will be decreased. For example, for the check node, if there is at least one message from the punctured variable node, the output LLR of the check node is always 0. Thus, this process does not require any LUT operation. For the variable node, if there is at least one message from the punctured check node (the output LLR of check node is 0), the output LLR of variable node is not changed. Thus, the LUT operation is not necessary.
V. SIMULATIONS AND RESULTS
In this work, we use AR4JA applied in CCSDS [11] with a code rate of 0.5. The data length is 1,024 bits, the lifting factor z is 256. The parity check matrix H is obtained from [11]. The codeword is transmitted over the Gaussian channel with zero mean and variance σ2. Let Eb be an average bit energy; The signal-to-noise ratio (SNR) is calculated by SN R(dB) = 10 log10(Eb/N0) where N0 = 2σ2. For the channel quantization and LUTbased decoder, we use the 2-, 3- and 4-bit quantizer and the decoder use 10 iterations.
Fig. 1 shows the BER performance of the LUT-based decoder. At BER = 10−4, the coding losses between BP decoder and 2-, 3-, 4-bit LUT-based decoder are equal to 1.6, 0.5 and 0.1 dB.
VI. CONCLUSIONS
In this work, we propose the design of a channel quantizer and LUT-based decoder for the protograph LDPC code. The 2-, 3- and 4-bit quantizer and LUTbased decoder are designed by minimizing the theoretical error performance. As a result, a 4-bit quantizer is sufficient. Moreover, we explain that the LUTs of the punctured node can be eliminated.
