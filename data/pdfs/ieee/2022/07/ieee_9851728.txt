Abstract—In recent years, many communication standards have adopted the LDPC (Low-Density Parity Check) code due to its excellent error correction performance, but it also induces some problems. For example, the use of 5G in mobile phones dramatically reduces battery life. Therefore, it is particularly significant for LDPC code to design a decoding algorithm that does not increase hardware implementation complexity but has superior error correction performance and iteration speed faster than traditional decoding algorithms.
Inspired by several network structures in artificial intelligence, bias is used to improve the performance and robustness of the network. In order to enhance decoding algorithm robustness and accuracy, a method is shown by changing the offset in OMSA to the bias and combining NMSA in the near-zero range in this paper. It significantly avoids the log-likelihood ratio information loss when less than the offset value and diminishes too quickly or sluggishly. A Segmented Bias/Normalize Min-Sum(SB/NMS) LDPC decoding algorithm is proposed with a layered structure. This algorithm can avoid the impact of the problems mentioned above and significantly improve the performance and iteration speed of the decoding process.
Compared with the BLER (resp. BER) performance the traditional BP, NMS and OMS algorithm achieves a gain of 0.04, 0.11, and 0.15dB (resp. 0.035dB, 0.06, and 0.13dB). Meanwhile, the minimum SNR requirements of BLER=0.001 can be reduced by 0.02 dB compared with the BP algorithm. The parameters of SN/NMSA can be pre-processed on the computer, so there will be no additional increase in the overhead in resources and get better performance while getting faster iteration speed, while greatly guaranteeing the reliability and effectiveness of data transmission.

I. INTRODUCTION
Nowadays, the original analog communication system evolved into the 5G mobile communication system. We can see that the communication data transmission rate is getting faster, and the latency is lower than before. These advancements are brought about by constantly increasing the transmit signal frequency and the number of antennas, and other technology. However, higher and higher frequency means that the attenuation of the signal is also growing. It is crucial to choose

a suitable channel coding strategy to provide a more stable communication quality.
In 1948, Shannon published a paper that systematically proved channel capacity limit, thus presenting a new chapter in channel coding. However, Shannon only proved coding exiting that allows information to pass through a channel without causing bit errors when speed is lower than the channel capacity but without mentioning a specific coding scheme. Afterward, various coding schemes have sprung up, such as Hamming code, Reed-Solomen code, and convolutional code. The linear block code with very few 1s included in the check matrix, proposed by Gallager and named LDPC code, is mainly discussed in this paper.
In addition, Professor Gallager theoretically proved and proposed the performance of LDPC code can achieve Shannon Limit and iterative decoding algorithms [1]. However, because of the hardware level of that period, the high computation complexity made it difficult to apply in particular applications, so the LDPC code was rarely studied. With the revolution in the coding field caused by Turbo code, scientists have gradually realized that the coding scheme of iterative decoding based on belief propagation is a practical scheme approaching the Shannon limit. In addition, the hardware conditions that limit LDPC code have been changed with the improvement of integrated circuit technology. Now, LDPC code is used by DVB-S2, 3GPP 5G NR(New Radio), Satellite Digital Television Broadcasting Standard (2nd generation), and IEEE 802.11e due to those advantages such as low error rate and linear increase in decoding complexity with code length.
In the past 20 years, people have continuously explored various aspects such as discovering and constructing betterperforming LDPC code, optimizing LDPC encoding and decoding algorithms, and encoding and decoding algorithms implementing. In [2], an extremely sparse and random check matrix is constructed to obtain the error correction performance approaching the Shannon limit. In [3], checked the edge nodes to construct LDPC code without short loops, which achieves a performance improvement of 0.25dB compared with traditional LDPC code. In [4], used the LU(lowerupper) decomposition algorithm to improve the LDPC coding

The principal difficulty of the LDPC code is that the algorithm proposed by Professor Gallager is practically too complex to be applied. In order to decrease the cost of hardware implementation, a simplified BP decoding algorithm is often used. Nevertheless, reducing the computational complexity leads to the degradation of the error correction performance, so it is imperative to design a simplified algorithm that can reduce the complexity while causing as little performance loss as possible. Standard simplified BP algorithms including MSA and its simplified algorithms NMSA and OMSA avoid overestimating the BP algorithm [8]. The OMSA algorithm estimates all near-zero values to zeros, which leads to performance degradation, and the constant attenuation factor is harmful to decoding performance in NMSA. Therefore, this paper proposes a Segmented Bias/Normalize decoding strategy with a layered structure, which can avoid the problems mentioned above and significantly improve the error correction performance of LDPC code. In [9], it proved that a layered structure decoder will increase the iteration speed to double.

II. PRELIMINARIES
A. LDPC Codes
The check matrix H of the LDPC code is sparse, and it is always used to represent the LDPC code. Assume the size of the check matrix with M rows and N columns. The number of 1s in each row and column is called the row weight wr and column weight wc, respectively. LDPC can be divided into regular and irregular LDPC codes by whether wr in each row and wc in each column are equal. The LDPC code proposed by Gallager randomly selects 1 when constructing the check matrix but ensures that the number of 1s in each row and column is consistent, so it is a randomly regular LDPC code. The design of LDPC code means to design its check matrix H. Because the LDPC check matrix has a significant influence on its error correction performance [11], in order to obtain better error correction performance, the design of LDPC code is fundamental.

III. SEGMENTED BIAS/NORMALIZE DECODING ALGORITHM AND OPTIMIZE
A. Enlightenment
Regardless of the type of neural network, from the simplest perceptron model to the complex CNN (convolutional neural network), an indispensable part is the bias unit. The existence of the bias unit facilitates better fitting of data and improves the robustness and accuracy of the model. Therefore, the decoding algorithm is inspired by bias to fit the data better and obtain better error correction performance.
There is a problem with zeroing the near-zeros belief in OMSA, causing the loss of information. So it inevitably leads to the deterioration of the decoding performance. In order to avoid this phenomenon, the NMSA algorithm is used near the value of 0. For the NMSA algorithm, there is a problem that the reduction speed is too slow or too fast under certain circumstances. Therefore, this article uses the BMSA algorithm in some parts and the B/NMSA(Bias and Normalize Min-Sum) algorithm in the transition section between BMSA and NMSA. The selection of separation points is based on the characteristics of tanh(x) and arctanh(x) functions. Since the value of tanh(x) can be approximately x when it is less than 0.2 and 1 when greater than 5, NMSA is used in these two intervals. Select 0.8 to 2 as the interval of the BMSA algorithm, and in order to ensure the overall continuity, B/NMSA is used in the transition interval.

C. Optimization Based on Genetic Algorithm
For the proposed SB/NMS algorithm, selecting its three parameters will significantly affect the algorithm’s performance. However, it is difficult to describe or approximate the impact of relevant parameters on the algorithm’s performance by any determined polynomial, which is an NP-hard problem and cannot calculate the gradient. Generally, such NP-hard problems can be optimized by swarm intelligence algorithms like PSO (particle swarm optimization algorithm) and GA (genetic algorithm). For the SB/NMS algorithm proposed in this paper, Assuming globally optimal αopt, βopt, and γopt make the performance best. It is considered that these three parameters have some characteristics to make the algorithm obtain the best performance, and the GA principle is to select outstanding genes to optimize the parameter by evolutionary theory. These genes are integrated through select, crossover, and mutation operations to obtain the optimal solution, consistent with some of the above characteristics.
According to the research of [13], to make GA find the global optimal solution, the strategy of retaining the elite of each generation must be adopted. Therefore, GA used in this paper is to retain the elite strategy. In addition, the selection of fitness function designed for GA will significantly affect the optimization result. Due to the particularity of LDPC decoding algorithm evaluation, the algorithm’s performance can only be analyzed and calculated by the Monte Carlo algorithm to balance the accuracy of evaluation fitness and calculation time. Moreover, an excellent LDPC decoding algorithm can perform better in medium and high SNR. In this paper, the Monte Carlo algorithm is used in IEEE 802.16e standard, and Eb/N0 = 2.6dB, the BLER corresponding to α, β, and γ is calculated as the standard to evaluate the error correction performance of the algorithm. It can avoid the limitation of iteration times and maximum population due to the extended evaluation time to improve the algorithm’s robustness. V. CONCLUSION
The Segmented Bias/Normalize Min-Sum algorithm proposed in this paper can adequately balance the advantages and disadvantages of the NMSA and OMSA. The improvement in BLER is comprehensive. Compared with the MSA algorithm, the gain of at least 0.25dB is undoubtedly reduced the link budget. In terms of the SNR required to achieve BLER=10−3, its performance is better than BP.
In summary, the SB/NMS algorithm has achieved remarkable performance enhancements in BER, BLER, and iterative efficiency. In addition, in terms of computational complexity, only one judgment step is added before each normalization operation, and the increase in computational complexity is negligible. Whether for resource-constrained or high-throughput scenarios, the SB/NMS algorithm can play a significant role. Subsequent work can be researched on the hardware implementation of the algorithm.
