A Design of Layered Decoding for QC-LDPC Codes Based on Reciprocal Channel Approximation

Abstract—This paper presents an analytically designed layered decoding algorithm for quasi-cyclic low-density parity-check (QC-LDPC) codes in the 5th Generation (5G) New Radio (NR) mobile communication system. First, a layered density evolution (DE) based on reciprocal channel approximation (RCA) is newly developed to accurately reflect the layered decoding operation in the decoder design. Using this technique, a single nested sequence is optimized in a greedy way to configure the processing order for any given number of layers. The error-correction performance is improved by up to 0.15 dB, while the number of iterations is reduced by about one simultaneously.
Index Terms—LDPC code, layered decoding, density evolution, reciprocal channel approximation, nested scheduling
I. INTRODUCTION
Low-density parity-check (LDPC) codes are a class of errorcorrection codes of which performance closely approaches the theoretical bound as the code length increases [1]. A belief-propagation (BP) algorithm is appropriate for a parallel implementation to achieve high decoding throughput. Due to these excellent characteristics, LDPC codes are used in many commercial systems. Recently, the 3rd Generation Partnership Project (3GPP) adopted LDPC codes in the 5th Generation (5G) New Radio (NR) standard [2].
In this paper, we design a novel layered decoding for the 5G NR LDPC coding system. In layered decoding [3], a given parity-check matrix (PCM) is divided into submatrices called layers, and decoding is performed in a layer-by-layer manner. Within a single iteration, updated information (e.g., log-likelihood ratio (LLR)) obtained from the previous layers is immediately reflected for subsequent layers, so the convergence speed is improved. In particular, the layered decoding is suitable for quasi-cyclic LDPC (QC-LDPC) codes including NR LDPC codes, since the decoding operation in each layer

can be fully parallelized by a proper decoder configuration for this type of code. For this reason, QC-LDPC coding systems commonly employ layered decoding, and the NR LDPC codes were also standardized to have an appropriate code structure under layered operation.
The order of processing layers should be carefully designed to achieve better performance and convergence, and an important work on analysis of sequential decoding was presented in [4]. Studies on how to design layer schedules are existing in the literature, and these works are categorized into dynamic scheduling and fixed scheduling. As an example of the former category, an informed dynamic scheduling (IDS) strategy was proposed in [5] to customize the decoding schedule for each received codeword. On the other hand, a maximum mutual information increase (M2I2) based algorithm was presented in [6] as a fixed scheduling method. More recently, for the 5G NR LDPC coding system, dynamic and fixed scheduling strategies were introduced in [7], and static scheduling methods based on check node degrees were proposed in [8]. This paper focuses on fixed scheduling of layered decoding.
Our goal in this paper is to capture the layered operation for QC-LDPC codes more analytically. We develop a new numerical analysis technique, layered reciprocal channel approximation (RCA) based density evolution (DE), in short, layered RCA-DE. The original RCA method was introduced in [9] as a useful low-complexity technique to analyze LDPC code ensembles, and this work was extended to protograph ensembles (including QC-LDPC code ensembles) in [10]–[16]. All these conventional techniques based RCA-DE for protograph ensembles are established under (full-parallel) floodingscheduled BP decoding. In this work, we devise a new version of RCA-DE by taking into account the sequential operation of layered decoding, thereby making it possible to evaluate protograph ensembles according to the applied layered decoder scheduling.

design a practical decoder for the 5G NR system. Specifically, we propose a nested scheduling framework, which enables us to derive optimized scheduling sequences of various lengths from a single mother sequence. In the NR LDPC coding system, the number of operating layers varies with the applied code rate, and a layer sequence needs to be specified for each given number of layers. Instead of separately optimizing a layer sequence of each case, we design only a single mother sequence in a nested manner to support all code rates of interest. In this design using the layered RCA-DE, we set a limited maximum number of iterations to be actually applied to the decoder. Thus, the proposed technique is exploited to assess the convergence speed of decoding rather than the asymptotic performance. Modified PCMs whose rows are permuted according to the optimized mother sequence can be fixed in the practical implementation, thereby not bringing any costs to separately configuring the layer scheduling depending on the applied code rate.
II. PRELIMINARIES
A. Notations and Definitions
Throughout this paper, we use the following notations:
• Calligraphic letters (e.g., A) are used to denote sets. • Given a set of indexed elements A = ∪k∈K{ak}, we
use A to denote its index set K. For example, A = {3, 4, 7, 8} for A = {a3, a4, a7, a8}. • Given A and B, we write A\B to denote the relative complement of A with respect to B. • Given A and b ∈ R, let b + A {a + b | a ∈ A}. • We use N, Z, and R to denote the sets of natural numbers, integers, and real numbers, respectively. • We write F2 to denote the binary field. • For a non-negative integer n, we write Zn to denote the set of consecutive integers from 0 to n − 1, that is, Zn {0, 1, . . . , n − 1}. • Boldface lowercase letters (e.g., a) and boldface uppercase letters (e.g., A) are used to denote vectors and matrices, respectively.
We use zero-based numbering, where the initial element of a vector and a sequence is assigned to the index 0.
B. QC-LDPC Codes and Protograph Ensembles
Consider an (N, K) LDPC code, where N and K are the code length and dimension, respectively. The number of constraints (parity bits) is denoted by M = N − K. This code is defined by a PCM H ∈ F2M×N such that Hx = 0 holds for any valid codeword vector x ∈ F2N . We write G(V, C, E) to denote its corresponding Tanner graph, where V, C, E are the set of variable nodes, check nodes, and edges, respectively. In this graph representation, we have |V| = N and |C| = M .
QC-LDPC codes belong to a subclass of structured LDPC codes constructed by applying copy-and-permute operations, called lifting, to a small prototype of a graph. The small graph is referred to as a protograph (or a base graph) [17], and its corresponding matrix is generally called a base matrix. The number of protograph copies used to construct the desired full

graph is denoted by the lifting size Z ∈ N. Letting m = M/Z and n = N/Z, we use Hp ∈ Fm 2 ×n to denote a base matrix exploited for constructing H ∈ F2mZ×nZ by lifting Z times. The protograph corresponding to Hp is denoted by Gp, and Hp and Gp are interchangeably used in context.
The protograph-based construction framework has advantages in code design and analysis. The asymptotic performance of a QC-LDPC code is represented by its protograph, so one can analyze the small protograph instead of directly investigating the full PCM, significantly simplifying the code analysis and design. For example, RCA-DE has been studied as an efficient tool to analyze protograph ensembles [10]–[15], and the analysis results well present the average characteristics of the corresponding QC-LDPC code ensemble.
The NR LDPC coding system [2] also employs QC-LDPC codes, and it flexibly supports various code lengths and code rates. Two protographs labeled base graph 1 (BG1) and base graph 2 (BG2) are specified, and they are designed to be concatenated coding schemes of precoding and single paritycheck (SPC) extension in order to support fine rate flexibility. Based on each protograph, eight mother PCMs with different lifting sizes are constructed. By applying modulo-lifting and puncturing to one of the mother PCMs, QC-LDPC codes of any desired length and rate in the system are constructed.
Any PCM in the NR LDPC coding system consists of several sub-matrices, and details of the code structures are given in [2, Clause 5.3.2]. Note that the first npZ information bits (systematic bits) are punctured from the mother codeword, where np = 2 in the NR system. Considering the npZ punctured information bits, we use N = min(N, E/Z ·Z +npZ) to denote the modified code length, where E is the ratematching output length (i.e., the number of actually transmitted bits after puncturing). Then, the desired PCM is constructed by the following steps:
1) Parameter selection: Given the code dimension K and the target code rate R = K/E, a protograph Gp is determined by selecting one of BG1 or BG2. Then, the lifting size Z is determined such that the constructed code derived from the protograph handles K input bits as tightly as possible.
2) Protograph construction: For N < N , the protograph Gp may be modified for efficient encoding and decoding with lower complexity. Let p = (N − N )/Z, and then, the last p variable nodes, p check nodes, and associating edges are removed from Gp thanks to the SPC extension. We write Gp to denote the modified protograph, and it has n = n − p variable nodes and m = m − p check nodes.
3) PCM construction: Based on Z, lifting is applied to Gp to construct H ∈ Fm 2 Z×n Z . In this lifting operation, a modulo-based technique [18] is applied to one of the mother PCMs.

A reciprocal energy function for the BI-AWGNC, denoted by R : R → R, is then given by R(s) = C−1 (1 − C(s)). This function R is an involution, i.e., R(s) = R−1(s), and thus, the SNR value and its reciprocal are transformed by r = R(s) and s = R(r), respectively. In the RCA-DE operation, variable nodes and check nodes generate messages corresponding to the SNR (s) and the reciprocal (r), respectively. The SNR (s) is transformed by R when passed from variable nodes to check nodes, while its reciprocal (r) is also transformed by R when delivered from check nodes to variable nodes.
Capturing the behavior of the layered BP decoding for NR LDPC codes, the AP-SNR si is set to the input s for nonpunctured variable nodes vi for i ∈ Zn \Znp , and all C-to-V reciprocal values are initialized with 0. Then, sub-iteration is sequentially performed for each check node according to the processing order Φm . Since the AP-SNR si is obtained by the sum of all incoming messages, a V-to-C SNR message pi→j is calculated by the difference between the AP-SNR si and the previous C-to-V SNR message qj→i. As shown in line 7 of Algorithm 1, the V-to-C messages are transformed by the function R, and the selective sum of them is re-transformed to generate an intermediate message t. Then, t is used to update the corresponding AP-SNR si and the C-to-V SNR message qj→i.
After sequential processing of all check nodes, the asymptotic decoding convergence is identified by using the updated AP-SNR values. If the minimum AP-SNR value within the systematic part (i.e., vi for i ∈ Zk) is greater than a predetermined threshold δ, then we regard that the layered decoder leads to successful decoding for the given protograph Gp and the BI-AWGNC with the input SNR s. The threshold δ is set to 20 (13 dB) in this paper, which is an acceptable value numerically found using discretized full-DE as a benchmark.

IV. NUMERICAL RESULTS
In this section, we evaluate the practical error-correction performance achieved by the proposed layer scheduling. The details of the evaluation assumptions are listed in Table I. For each combination of coding schemes given in the table, we obtain BLER curves of the conventional scheme with the natural order, two existing schemes in [7], [8], and the nested scheduling scheme proposed in the previous section. A total of 57 codes (3 rates × 19 lengths) listed in Table I are evaluated for each of the four scheduling schemes. In order to effectively display the performance for such a number of coding schemes, we find the Es/N0 required to achieve 1% BLER for each simulation, where this 1% BLER is regarded as an important requirement of the 3GPP NR enhanced mobile broadband (eMBB) [20]. The use of this performance metric instead of full BLER curves simplifies the comparison. In addition, we find the average number of iterations for 1% BLER.
Fig. 2 shows the BLER performance and the average number of iterations for QC-LDPC codes derived from BG1. The proposed scheme consistently brings benefits in a wide range of code dimensions and rates over the other three schemes. In particular, compared to the unscheduled natural order, the proposed layered decoding with nested scheduling achieves up to 0.1442 dB SNR gains for 1% BLER, while simultaneously reducing the average number of iterations by a maximum of 0.9808. For low code rates, the length of the layer sequence m is long, and there is a higher potential for improvement compared to the cases with high code rates. Therefore, the

lower the code rate is, the greater the performance gain is. In addition, the performance gains observed in Fig. 2 are
well matched to the numerical analysis results estimated by layered RCA-DE in Fig. 1. The expected gains in terms of the iterative decoding thresholds are about from 0.1 to 0.2 dB, and a similar level of gains is achieved in the BLER evaluation.
V. CONCLUSION
In this paper, we presented an efficient layered decoding algorithm for the 3GPP NR LDPC coding system. We first developed a new layered RCA-DE to evaluate and model the layered decoding operation for QC-LDPC code ensembles. Using this numerical analysis tool, a nested layer sequence was designed in a greedy manner. The new scheduling leads to performance gains in a wide range of code rates and length, as well as it also reduces the average number of iterations. These improvements are remarkable in that they are achieved just by rearranging the order of processing layers. Also, we observed similar levels of performance improvement when other implementation-friendly decoding algorithms (e.g., fixed-point offset min-sum) were employed, although the results are not presented in this paper due to the page limitation. We expect that no additional costs are introduced in practical hardware and software implementation since the LDPC decoder can be realized by preconfiguring row-permuted PCMs according to the single optimized nested sequence. The layered RCA-DE technique in this paper can be improved by using the SNR configuration method given in [16] together to design layered decoding for general BICM systems.
