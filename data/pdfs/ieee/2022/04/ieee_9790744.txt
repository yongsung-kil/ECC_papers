High throughput FPGA implementation of Min-Sum LDPC Decoder Architecture for Wireless
Communication Standards
Pavel Nikishkin, Ruslan Goriushkin, Nikita Vinogradov, Evgeny Likhobabin and Vladimir Vityazev, Member, IEEE
Department of Telecommunications and foundations of radio engineering Ryazan State Radio Engineering University, RSREU Ryazan, Russia
nikishkin.p.b@tor.rsreu.ru, gorushkin.r.s@tor.rsreu.ru, likhobabin.e.a@tor.rsreu.ru, vityazev.v.v@rsreu.ru

Abstract—This paper presents a min-sum decoder design for Quasi-Cyclic (QC) Low-Density Parity-Check (LDPC) codes. The design is supported various LDPC Parity-Check matrices including the WiMAX (IEEE 802.16e) and the WiFi (IEEE 802.11n) standards matrices. New techniques such as pipelining of the decoding architecture core are proposed. These core calculate variable-to-check (VTC) and new check-to-variable (CTV) messages and also update estimate of posterior probabilities (APPs). The parallel multicore decoding architecture implies a prior shift of values based on the LDPC matrix and simultaneous calculation of values for the core. Proposed decoder is implemented on the Zynq-7000 Mini-ITX Evaluation Board (XC7Z100-2FFG900).
Index Terms—Communication systems, Quasi-Cyclic codes, Low-Density Parity-Check codes, error correction codes, parallel decoder architecture, iterative decoding, min-sum (MS) algorithm, field-programmable gate array, FPGA.
I. INTRODUCTION
The Low-density parity-check (LDPC) codes was proposed by Gallager in 1963 [1]. MacKay and Neal [2] gave LDPC codes new life in the year 1999. Currently, there is a high interest in LDPC codes due to their excellent error correcting capability compared by turbo codes. [3]. For error probability level of 10−6 it is possible to get gain in the signal to noise ratio due to the LDPC coding up to 3 dB. The use of these codes is also effective in counteracting wideband impulse noise, as well as signal interference. These codes are widely used by communication standards such as IEEE 802.11n/ad/ay/ax (WiFi) [4], IEEE 802.16e (WiMAX) [5], and ETSI 5G [6].
In this paper we provide the decoder implementation for QC-LDPC codes, which meets the flexibility to work with various code rates and lengths. The proposed LDPC decoder can operate with matrices similar to IEEE 802.11 and IEEE 802.16e.
The result of implementing architecture on the FPGA part of the Zynq-7000 Mini-ITX Evaluation Board (XC7Z1002FFG900) are provided.

II. MIN-SUM DECODING ALGORITHM

Low-Density Parity-Check (LDPC) codes are a class of linear block codes. The min-sum algorithm (MSA) is a simplified Belief propagation algorithm (BP) built by reducing complexity of the unit responsible for check node updates (CNU). Compared to the BP algorithm, the Min-Sum algorithm does not provide accurate APP values, but only gives an approximate estimate. This allows for a gain in computational costs, but leads to a decrease in decoding quality. The name these codes comes from the characteristic of their parity-check matrix which contains only a few 1’s in comparison to the amount of 0’s. A regular (N, K) LDPC code is represented as check matrix H which includes M ≥ N − K rows and N columns. The M rows of H are defined as check nodes, and all the elements of a LDPC codeword as variable nodes.

III. LDPC DECODER ARCHITECTURE
The general structure of the proposed LDPC decoder is shown in Figure 1 [12].
The soft decisions from the demodulator are used as input to the decoder. The input serial bitstream is converted into a parallel and is written to the distributed memory. For ease of

parallel reading and writing, the APP memory is divided into K = N/z blocks, where (N ) - length of the codeword and (z) - size of the parity check matrix circulant.
When reading data from the memory block, the APP value blocks are shifted according to the initial shifts specified by the check matrix. The reverse shift operation on the APP value blocks is performed after processing on the cores before writing them to the memory block.
The Iterations counter block makes a decision on completion of the decoding process and output of the result.
The Operating mode selection block is used to switch the operating mode to ”decoding” or ”result output” and to organize output of the decoding result.
The VTC calculation block is used to calculate the variableto-check messages based on APPs from the APP block.
The Min/submin calculation block receives dr signed VTC values including the minimum and subminimal values at the input. The dr value is determined by the check matrix. The subminimal value is obtained at the output corresponding to the input with the minimum modulo value. The minimum value is obtained at the other outputs. All outputs are signed and have the sign obtained from the Signs XORing block. The VTC input is also passed to the output. The described algorithm is shown in Figure 2.
To calculate the updated CTV values, the input of the CTV and APP calculation block requires the minimum VTC value, the product of the VTC signs and the sign for the current value. The updated APPs are obtained by summing the VTCs and CTVs.
The decoder architecture was described in detail in the [12].

A. Independent layers processing
The row-layered decoding approach is used in this work. In row-layered decoding the parity check matrix of the LDPC code is partitioned into multiple layers (rows). The message updating is performed layer by layer. The decoding process occurs using a parity check matrix. The initial matrices are presented in the IEEE 802.16e (WiMAX) [5] and the IEEE 802.11n (WiFi) [4] standards. According to the structure of the check matrix of the 802.16e standard, to start processing a new layer it is necessary to wait until the end of the calculation of the previous one. It was noticed that the LDPC check matrix has layers that are independent of each other. Therefore, the rows of the matrix can be rearranged in such a way that each layer can be independent of the previous and next one. So for a matrix from the 802.16e standard the following layer reading order is suitable: 1-3-5-9-7-12-8-6-11-2-4-10. The resulting matrix is shown in the Figure 3. The result of rearranging layers does not affect to the bit error rate (BER). Using independent layers makes it possible to nearly double the decoder’s throughput, since the calculation of a new layer can be started without waiting for the previous one to be ready.
B. Pipeling of decoding architecture core
The architecture was optimized to use the pipeline structure of the LDPC decoder core. This allowed to increase the throughput and made it possible to receive data to to the decoder core on every clock cycle. The LDPC decoder core calculate variable-to-check (VTC) and new check-tovariable (CTV) messages and also update posterior probabilities (APPs). In the LDPC decoder architecture presented in [12], the updated APPs and CTV values are calculated in parallel for one data block. The main problem of this approach was the suspension of the kernel until the end of the calculation in the minimum/subminimum block. VI. CONCLUSION
A min-sum architecture of a QC-LDPC decoder was proposed herein. The parallel decoding architecture was implemented using the Zynq-7000 Mini-ITX Evaluation Board (XC7Z100-2FFG900).
The maximum throughput is achieved when the codeword size is 2304 at the frequency of 180 MHz. The results of comparing the proposed decoder with similar ones are shown in Table I. The Table I shows that in contrast to [8] the proposed architecture allows to increase the throughput by increasing the codeword size.
