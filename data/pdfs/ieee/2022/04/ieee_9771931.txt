AbstractвҖ”In this paper, we present a new construction of the globally-coupled LDPC (GC-LDPC) codes, referred to as implicit GC-LDPC codes, where the additional global parity-check bits are transmitted using the free-ride coding. The presented GCLDPC codes have the same rates as the component LDPC codes, avoiding the rate reduction caused by the global parity checks for the conventional GC-LDPC codes. Moreover, the encoders of the component LDPC codes are reusable in the presented GC-LDPC codes, a distinguished feature as compared with the conventional GC-LDPC codes. The simulation results show that the proposed implicit GC-LDPC codes can improve the performance of the component LDPC codes, yielding extra coding gain of up to 0.7 dB (with (3, 6)-regular LDPC codes as the component codes) and 0.5 dB (with IEEE 802.16e LDPC codes as the component codes).
Index TermsвҖ”Free-ride codes, globally-coupled LDPC (GCLDPC) codes, LDPC codes.
I. INTRODUCTION
Low-density parity-check (LDPC) block codes were invented by Gallager in the early 1960s [1]. Due to the good error-correction performance and high decoding throughput, LDPC codes have been adopted for data channels in both 5G enhanced mobile broadband (eMBB) scenario [2] and ultrareliable low-latency communications (URLLC) scenario [3].
This paper focuses on constructing LDPC codes for the transport block (TB) based transmission protocol adopted in both LTE and 5G, where a TB consists of multiple finite-length code blocks (CBs) and each CB is independently encoded and decoded. Evidently, both the TB-level hybrid automatic repeat request (HARQ) retransmission in LTE [4] and CB group (CBG)-level HARQ retransmission in 5G [2] may lead to a waste of transmission power and spectral efficiency since a single erroneous CB will cause the retransmission of the whole TB or the whole CBG. A straightforward way to improve the resource utilization efficiency is to improve the word error rate (WER) performance of a CB or the frame error rate (FER) performance of a TB. This can be achieved by introducing extra constraints across multiple CBs within a TB.
A well-known approach to improve the error-correction performance is spatial coupling. Specifically, L independent LDPC codes are coupled into a single chain, resulting in the

spatially-coupled LDPC (SC-LDPC) codes [5], [6]. The SCLDPC codes can improve the performance of the underlying LDPC block codes due to the threshold saturation, where the belief propagation (BP) decoding performance of the SCLDPC codes can approach the maximum a posteriori (MAP) decoding performance of the underlying LDPC codes [6], [7]. However, the encoders and decoders of the SC-LDPC codes are usually loosely related to those of the underlying LDPC block codes. This somehow brings inconvenience for real design and practical implementation1.
Another promising coupling approach is global coupling, delivering globally-coupled LDPC (GC-LDPC) codes [10], where L independent LDPC block codes are coupled by the additional globally-connecting check nodes. As a result, the code length can be scaled up (implying potential performance improvement) based on the component LDPC codes, avoiding construction of a completely new longer LDPC code. Since then, the Reed-Solomon based construction [11], the array dispersion construction [12] and the cyclic structure construction [13] for the GC-LDPC codes have been proposed. Recently, a generalized construction for GC-LDPC codes was proposed, where several independent local graphs are coupled by a global coupling graph [14]. A remarkable feature of the GC-LDPC codes is that the decoders of the component LDPC codes are reusable [12]. That is, the decoders of the GC-LDPC codes are compatible with those of the component LDPC codes. However, the encoders of the component LDPC codes can not be directly reused in the conventional GC-LDPC codes since the juxtaposition of multiple codewords of the component LDPC codes is usually not a codeword of the GCLDPC code due to the existence of the global parity checks. Also notice that the rate loss caused by the additional global parity checks can not be neglected for the finite coupling length L.
In this paper, we use the free-ride coding to construct implicit GC-LDPC codes for the TB-based transmission protocol. The basic idea is to transmit the global parity-check bits as extra bits by using free-ride codes [15], [16], which requires neither extra transmission power nor extra bandwidth. The presented GC-LDPC codes share the same code rates with the component LDPC codes, avoiding the rate reduction in the conventional GC-LDPC codes. More importantly, the

For decoding, we present an efficient two-stage decoding algorithm, which has a similar complexity to the component LDPC codes. At the first stage, we perform the free-ride decoding to recover the global checks and then remove the effects of the (estimated) global checks on the received vector. At the second stage, we perform the global decoding with a low-complexity scheduling. The simulation results show that the proposed GC-LDPC codes can improve the performance of the component LDPC codes, yielding an extra coding gain of up to 0.7 dB with (3, 6)-regular LDPC component codes and 0.5 dB with IEEE 802.16e LDPC component codes, but without any code rate reduction.

A. Problem Statement
Let F2 = {0, 1} be the binary field. Let C [n, k] be an LDPC block code of length n and dimension k, whose parity-check matrix is denoted by H. For the simple LDPC coded transmission link, after receiving the noisy sequence, an iterative BP decoding algorithm, say, the sum-product algorithm (SPA) [1] can be performed to recover the information. V. CONCLUSIONS
This paper has presented the implicit GC-LDPC codes for the TB-based transmission, where the global parity-check bits are transmitted as the extra bits using the free-ride coding. The presented GC-LDPC codes share the same rates with the component codes. For encoding, we have presented a parallel encoding algorithm, where the encoders of the component LDPC codes can be directly reused in the presented GC-LDPC codes. For decoding, we have presented an efficient two-stage decoding algorithm, which has a similar decoding complexity to the component LDPC codes.
