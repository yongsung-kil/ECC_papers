Layered Bit-Flipping Algorithms for Decoding LDPC Codes

Abstract—At present, various communication and storage systems have put forward high requirements on both the complexity and performance of low-density parity-check (LDPC) decoders. Bit flipping decoders are a good candidate for these applications. This letter proposes to utilize the column layered scheduling to speed up the convergence and improve the error correcting performance of existing bit flipping algorithms. As the information storage bit flipping (ISBF) algorithm generally performs the best, we investigate the layered ISBF (LISBF) algorithm in detail, including developing two efficient techniques for optimizing the computation of flipping thresholds: forward-backward computation and twomax computation. Simulation results on a (3, 6) LDPC code show that, compared to the ISBF algorithm, the LISBF algorithm can reduce the frame error rate by up to an order and reduce the average number of iterations by around 40%.
Keywords—Low-density parity-check codes, bit flipping, flipping threshold, layered decoding
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] have been widely used in storage and communication systems. The decoding algorithms of LDPC codes can be divided into two categories: softdecision decoding and hard-decision decoding. Soft-decision decoding of LDPC codes, such as the belief propagation (BP) algorithm [1], min-sum (MS) algorithm [2], and offset minsum (OMS) algorithm [3], have very good error correcting performance, but at the cost of high computational complexity. On the other hand, hard-decision decoding algorithms of LDPC codes that only transmit and calculate binary messages, such as the bit-flipping (BF) [4] algorithm, have lower computational complexity but worse error correcting performance.
Up to now, a lot of efforts have been paid to improve the error correcting performance of BF algorithms. The weighted BF (WBF) algorithm [5], improved BF (IBF) algorithm [6], gradient descent bit-flipping (GDBF) algorithm [7], and modified GDBF (MGDBF) [8] algorithm mainly focus on additive white Gaussian noise (AWGN) channels, where soft information is available for enhancing the decoding performance of BF algorithms. However, using soft information certainly results in higher complexity than using only hard information. Concerning that, the main scope of this letter is limited to the BF algorithms based on hard information. More specifically, we consider the BF decoding of LDPC codes under binary symmetric channels (BSCs). In the following, we introduce some related works under this scenario.

In [9], the GDBF algorithm and the probabilistic GDBF (PGDBF) algorithm were proposed for decoding LDPC codes using only hard information under BSCs. Both algorithms convert the decoding problem into a maximum likelihood problem and solve it through gradient descent. In each decoding iteration, the bit energy values are first calculated, and then the maximum value of the energy values is calculated and used as the flipping threshold. The GDBF algorithm flips all bits whose energy values are equal to the flipping threshold, while the PGDBF algorithm flips those bits with a certain probability. As a result, the PGDBF algorithm has a stronger ability to overcome the trapping set than the GDBF algorithm. With the increase in the number of iterations, the PGDBF algorithm has obvious error correcting performance gains compared to the GDBF algorithm. However, these two algorithms can only serially calculate the energy values and the flipping threshold, which leads to a long critical path.
To address the above problem of the long critical path, the probabilistic parallel BF (PPBF) algorithm that does not need to calculate the flipping threshold was proposed in 2018 [10]. After computing the bit energy values, each bit is flipped with a probability proportional to its energy value in the PPBF algorithm. However, the PPBF algorithm does not significantly improve the error correcting performance. Moreover, it needs to optimize many parameters (flipping probabilities), which brings inconvenience to practical applications.
In 2019, a tabu-list random-penalty gradient descent bitflipping (TRGDBF) algorithm was proposed [11]. One optimization technique used in the TRGDBF algorithm is that a random penalty-term is added into the energy value, where the penalty-term equals 0 or 1 with a certain probability. As a result, the bits originally with the second-largest energy value have an opportunity to be flipped, which helps the algorithm escape from the trapping sets. The other optimization technique is to restrict the flipping of the bits which are flipped in the previous iteration by employing a tabu-list. The TRGDBF algorithm can have better error correcting performance than the PGDBF and PPBF algorithms at a large number of iterations. Unfortunately, it also suffers from the long critical path problem caused by serially calculating the bit energy values and the flipping threshold.
Recently, the information storage bit-flipping (ISBF) algorithm [12] was proposed. In the ISBF algorithm, the flipping threshold is calculated based on the energy values from the previous iteration. As a result, the computation of bit energy values and the flipping threshold can be performed in parallel

Moreover, it proposes a nonuniform flipping rule: If the number of unsatisfied check nodes (CNs) connected to the i-th variable node (VN) is greater than or equal to the flipping threshold, flip the i-th bit with probability p0; Else if the energy value of the i-th VN is greater than or equal to the flipping threshold, flip the i-th bit with probability p1, where p1 ≤ p0; Otherwise, do not flip the i-th bit. The ISBF algorithm can generally achieve the best error correcting performance. However, the ISBF algorithm converges slowly.
In the literature, it is common to speed up the convergence of LDPC decoding algorithms based on (column) layered scheduling. For example, in [13], a column layered scheduling was first proposed to speed up the convergence for the BP algorithm. In [14], an efficient two-min scheme was proposed for the first time to calculate the minimum amplitude value of the min-sum algorithm for column layered decoding. The idea of column layered scheduling is to divide all bits into several layers (groups), and in each layer, the updating of messages is processed in parallel, but the process across layers remains serial, thereby accelerating the transmission of information and realizing the multiplexing of hardware. Moreover, in practical applications, it may be impossible to process all bits in parallel due to the long code length and the limitation of hardware resources. Layered decoding can be a perfect solution in this situation by reusing hardware resources. Therefore, it is meaningful to investigate the BF algorithms with layered scheduling. To the best of our knowledge, the column layered scheduling has never been applied to BF algorithms in the literature.
In this work, we propose the idea of (column) layered BF decoding of LDPC codes using only hard information. Considering that the ISBF algorithm exhibits the best performance among all BF algorithms, we study the layered ISBF (LISBF) algorithm in detail. To optimize the calculation of the flipping threshold involved in the LISBF algorithm, we further develop two efficient techniques: forward-backward computation and two-max computation, where the latter is motivated by the twomin scheme in [14]. Both techniques have complexity O(1) and latency O(1) per layer. The simulation results on a (3, 6) LDPC code show that compared to other BF algorithms, the proposed LISBF algorithm and ALISBF algorithm (LISBF algorithm with two-max computation) not only offer the best performance but also provide the (near) smallest average number of iterations.

III. PROPOSED LAYERED BF DECODER
Motivated by [13], we consider the column layered scheduling. Suppose L is the desired number of layers. The layered BF algorithm divides all VNs into L layers (sets), namely V1, V2, . . . , VL, which satisfy the following constraints:
V1 ∪ V2 ∪ · · · ∪ VL = [N ], Vi ∩ Vj = ∅, ∀i, j ∈ [L], i ̸= j.

The general framework of the proposed layered BF decoding is that the VNs within each layer are handled in parallel, and the VNs between different layers are handled in serial. For each layer, the bits’ (VNs’) energies are computed first. Some bits are then flipped based on certain strategies (for example, when a bit’s energy value is no smaller than some flipping threshold T ). Finally, the syndromes are updated due to the bit flipping operations, and the decoding process moves to the next layer.
As reported in [12], the ISBF algorithm tends to have the best error correcting performance among various classical BF decoding algorithms. Meanwhile, we note that the ISBF algorithm can shorten the critical path. Therefore, we focus on applying the layered scheduling to the ISBF algorithm, resulting in a layered ISBF (LISBF) decoding algorithm. The proposed LISBF algorithm is shown in Algorithm 1.
In Algorithm 1, Imax represents the maximum allowed number of decoding iterations. Tl, l ∈ [L] represents the maximum energy value of the l-th layer, which we call layer threshold. T represents the bit flipping threshold, which equals the maximum value of all layer thresholds. We remark that for the l-th layer in the t-th iteration of Algorithm 1, the computation of T in line 10 and the computation of Si, Ei, ∀i ∈ Vl can be implemented in parallel. This actually follows the idea of the ISBF algorithm to shorten the critical path. In lines 14-20, Algorithm 1 computes the flipping flag fi in the same way as the ISBF algorithm, where B(·) is the Bernoulli distribution with a certain probability. As can be seen from lines 21–25, if fi ≠ 0, the i-th bit is flipped and both energies and syndromes are updated. At the end of the decoding process of the l-th layer, Tl is updated, as shown in line 27 of Algorithm 1. This update is necessary for the computation of T in line 10 of the next layer.
We remark that the computation of T in line 10 of Algorithm 1 is critical to the decoding latency. At this point, this computation has complexity O(L) and latency at least O(log2 L) per layer if a binary tree computation structure is used. Fig. 1 shows the frame error rates (FERs) of different BF algorithms for decoding the dv3R050N1296 code. Same as in [12], we set p0 = 0.75 and p1 = 0.45 for the ISBF, LISBF, and ALISBF algorithms. From Figure 1, we can see that the LISBF and ALISBF algorithms more or less perform the same, even if the ALISBF algorithm may not accurately compute the flipping threshold T from layer thresholds. This is actually reasonable, since the two-max approximate computation involved in the ALISBF algorithm generally has sufficiently high accuracy, and a layer threshold also may not be the accurate maximum energy value of the bits in the corresponding layer due to the effect of bit flipping and syndrome update. Moreover, the proposed LISBF and ALISBF algorithms exhibit the best FER performance among all BF algorithms. For example, they reduce the FER of the ISBF algorithm by up to an order, while the ISBF algorithm has the lowest FER among the conventional BF algorithms.
Fig. 2 shows the average number of iterations for different BF algorithms. The LISBF and ALISBF algorithms almost perform

the same, and they both reduce the average number of iterations of the ISBF algorithm by around 40%. On the other hand, when the crossover probability α is greater than or equal to 0.03, the LISBF and ALISBF algorithms have the smallest average number of iterations. As the crossover probability decreases, the average number of iterations of the GDBF algorithm becomes the smallest, but the GDBF algorithm has much worse FER performance compared to the LISBF and ALISBF algorithms as can be seen from Fig. 1.
V. CONCLUSION
In this work, we applied the column layered scheduling to improve the error correcting performance and throughput of hard-decision BF decoders. We first developed a LISBF algorithm based on the ISBF algorithm with layered scheduling. Then, we optimized the computation of the flipping threshold involved in the LISBF algorithm by proposing two efficient techniques: forward-backward computation and two-max computation. Simulation results on a dv3R050N1296 code showed that, compared to the conventional BF decoders, the proposed LISBF algorithm and ALISBF algorithm (LISBF algorithm with two-max computation) have the lowest FER and the (near) smallest average number of iterations.
