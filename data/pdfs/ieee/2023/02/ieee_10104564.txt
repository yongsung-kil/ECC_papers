Ensemble Belief Propagation Decoding for Short Linear Block Codes
Kira Kraft, Matthias Herrmann, Oliver Griebel, Norbert Wehn Technische Universita¨t Kaiserslautern
{kraft, herrmann, griebel, wehn}@eit.uni-kl.de

Abstract—The upcoming communication standard 6G is expected to specify high demands on the efficiency of current communication systems, especially on the latency and error correction performance of the decoding unit. A well-investigated algorithm that allows a low-latency implementation is Belief Propagation (BP) decoding on the Tanner graph, which is the state-of-theart decoding algorithm for Low-Density Parity-Check (LDPC) codes. In this paper, we present a methodology to apply this BP algorithm to non-LDPC codes while achieving a good error correction performance. In addition, we propose a method to generate a diverse ensemble of parity-check matrices for short linear block codes to enable the use of Multiple-Bases Belief Propagation (MBBP), an extension to BP, further improving the error correction performance of LDPC and non-LDPC codes. At the example of LTE Turbo codes, our results show that this approach is capable of achieving the error-correcting performance of the Turbo code’s state-of-the-art max-Log MAP algorithm, while being a universal approach for decoding any kind of short linear block code.
Index Terms—Forward Error Correction, Belief Propagation, Universal Decoder
I. INTRODUCTION
Belief Propagation (BP) [1] is a widely adopted concept in the decoding of forward error correcting codes. As a messagepassing algorithm, BP relies on the iterative exchange of messages (“beliefs”) in a graphical model, which typically varies among different code classes, for example, the component code’s Trellis for Turbo codes [2], the Factor Graph for ReedMuller (RM) codes [3] or Polar codes [4], or the Tanner graph for Low-Density Parity-Check (LDPC) codes [5], [6].
Today, LDPC codes are the preferred code class in high data rate and high performance applications like 5G-NR, Wi-Fi, etc. The reasons for this dominance are capacity approaching code performance for large code block sizes [6] and the availability of highly efficient and high-throughput and lowlatency decoding hardware. The latter is a result of two important factors: i) low-complexity node update rules, especially with the introduction of the min-sum approximation and ii) inherent decoding parallelism on the underlying Tanner graph (in contrast to, e. g., Turbo codes) [7].
It is expected that the sixth generation mobile system standard (6G) will specify even more stringent requirements, especially with respect to latency and error-correcting performance compared to 5G. While decoding latency is not a major concern in LDPC coded transmission systems due to the aforementioned parallelism, the structural latency, that is determined by the code block size, gains importance.

Therefore small code block sizes play an important role in the standardization of 6G. However, for short LDPC codes performance of BP decoding is significantly impaired as a result of the relatively dense matrix and the presence of short cycles in the Tanner graph. The same problem occurs in the decoding of code classes other than LDPC codes, in the following referred to as non-LDPC codes. It is therefore an important question how this effect can be mitigated to enable low-latency BP decoding of short LDPC codes and non-LDPC codes with acceptable error-correcting performance. Several works already tried to apply Tanner-graph-based BP decoding to non-LDPC codes: The authors of [8] propose to adapt the parity-check matrix in every iteration of the BP algorithm, which results in a large computational overhead, while the authors of [9] introduce weights on the edges of the Tanner graph (obtained by a neural network) that can help to mitigate the effects of short cycles in a single BP decoding instance.
Another approach to apply BP decoding on the Tanner graph for non-LDPC codes is Multiple-Bases Belief Propagation (MBBP) [10], [11]. Since different parity-check matrices of the same code can lead to different Tanner graphs, the performance of the Tanner BP algorithm is tied to the paritycheck matrix used to identify the code. The idea of MBBP is therefore to use an ensemble of different parity-check matrices for decoding with multiple instances of a Tanner BP algorithm in parallel. Thereby, a certain decoding diversity is introduces, which can mitigate the effects of short cycles. The main challenge in this approach lies in finding a set of suitable diverse parity-check matrices of the same code. In original MBBP, the approach is restricted to cyclic codes and the different matrices are obtained by cyclic shifts of minimal codewords of the (also cyclic) dual code. Thus, MBBP is still restricted to certain code classes and cannot be used for all kinds of codes as finding the different parity-check matrices is not obvious.
The recently proposed Automorphism Ensemble Decoding (AED) [12] can be seen as a generalization of MBBP. Instead of using multiple parity-check matrices for decoding, the idea is to permute the decoder input according to the code’s automorphisms and thereby generating multiple decoding instances. Basically, instead of relying on the algorithm to perform differently for different representations of the paritycheck matrix, it now relies on a certain decoder asymmetry, as the decoding algorithm performs differently for different inputs (though with the same error weights). II. BACKGROUND
A. Belief Propagation
Belief Propagation (BP) decoding is a commonly used decoding technique for LDPC codes. It is an iterative messagepassing algorithm working on the code’s Tanner graph [13]. The Tanner graph representation of a code is directly derived from the parity-check matrix H. In detail, each column of H defines an instance of a so-called Variable Node (VN), and each row of H defines an instance of a so-called Check Node (CN). Each one-entry in H induces an edge between the corresponding variable and check node. Figure 1 shows the Tanner graph of a given H matrix with 3 CNs and 4 VNs.
The main reason for BP decoding not being used for codes with dense parity-check matrices is the high number of small cycles inside the Tanner graph. The highlighted parts in Figure 1a and Figure 1b show a 4-cycle in graph and matrix

B. Multiple-Bases Belief Propagation
The idea of Multiple-Bases Belief Propagation (MBBP) proposed in [10], [11] is depicted in Figure 2. It requires different representations of the code’s parity-check matrix H, H1, . . . , Hl, l ∈ N, from which l corresponding BP decoding instances are created that all decode the same input y. Every decoding instance that converged to a valid codeword (xˆi ∈ C) processes its output to a list, from which the most likely codeword xˆ according to a Maximum-Likelihood (ML) criterion is chosen. While the idea of MBBP is quite simple, its crucial point lies in the generation/selection of the different matrices Hi. First, each single BP decoding instance has to perform well. Therefore, suitable matrix representations Hi have to be found. The requirements for matrix representations suitable for BP decoding are described in Section II-D. Second, the ensemble of BP decoding instances has to be diverse, meaning that they complement each other well regarding their ability to correctly decode certain error patterns.
In [10], [11], MBBP decoding is therefore only applied to short cyclic codes, where the different parity-check matrices are obtained by cyclic shifts of minimal dual codewords. C. Automorphism Ensemble Decoding
Automorphism Ensemble Decoding (AED) [12] has come up in recent years and can be seen as a generalization of MBBP decoding. The general idea is depicted in Figure 3. While in MBBP decoding, several different BP decoding instances are run in parallel, in AED, the same decoding instances are run with different inputs obtained by permutations πi from the code’s automorphism group. As in MBBP decoding, among all valid codewords found, the most likely one according to an ML criterion is chosen as output.
While the main effort in MBBP decoding lies in generating different suitable matrices, the main effort of AED lies in finding the automorphism group of the used code. So far, the automorphism groups of Reed-Muller codes [12], Polar codes [14]–[16] and of some quasi-cyclic LDPC codes [17] have been identified and can be used for AED. Some codes, however, do not have a non-trivial automorphism group [18]. Among these codes are LTE Turbo codes or non-structured LDPC codes.
D. Matrix Representations Suitable for BP Decoding
To efficiently apply (MB)BP decoding to non-structured codes, a suitable representation of their parity-check matrices needs to be found.
In [19], the authors identified the following features for matrices to work well for BP decoding:
• The number of ones in each row should be small (small check node degree).
• The number of ones in each column should be large (large variable node degree).
• The number of small cycles should be small (ideally no four-cycles in the corresponding Tanner graph).
One way to alter a given parity-check matrix was proposed in [20]. As the set of codewords C is defined as the kernel of the parity-check matrix, i. e.,
C = {c ∈ {0, 1}n | H · c ≡ 0 mod 2},
kernel-preserving operations can be performed in terms of elementary row operations, that are commonly used in Gaussian elimination. In general, they consist of swapping rows,

multiplying rows with non-zero scalars, and adding multiples of rows to other rows. Swapping rows does not affect the performance of the BP decoding algorithm for a flooding schedule, as the connections between the CNs and VNs remain the same. Since one is the only non-zero scalar, multiplications with scalars do not alter the matrix. Therefore, the only kernelpreserving operation that can be performed is the addition of a set of rows to another row. By only using kernelpreserving operations, the authors of [20] propose an algorithm to minimize ones in a given parity-check matrix, thereby also reducing the number of short cycles in the corresponding Tanner graph.
Another approach to break up short cycles was proposed by the authors of [19] as well as the author of [21] by inserting additional VNs and CNs in the corresponding Tanner graph. The results are very promising for Binary Erasure Channels (BEC), but not for Additive White Gaussian Noise (AWGN) channels.
III. METHODOLOGY
In this section, we will first recall the method from [20] and use it to obtain a suitable parity-check matrix representation for BP decoding for any given code. In addition, we extend the method to generate a diverse set of parity-check matrices that complement each other when combined in MBBP decoding, thus enabling the use of MBBP decoding for all kinds of linear block codes.
A. Minimizing Ones in the Matrix
To obtain sparsified parity-check matrices, we apply the row-wise integer programming-based heuristic presented in [20]. It was originally proposed to reduce the runtime of ML decoding, however, we will show that it can also be applied to obtain a matrix representation suitable for BP decoding, as reducing ones in a given matrix inherently reduces the number of short cycles.
The minimization procedure from [20] is recalled in Algorithm 1. It successively traverses the rows of a given paritycheck matrix H and searches for a subset S of rows that can be added to the current row to minimize the number of ones in the current row. Searching for the set S is done solving an integer linear program (ILP) (see Line 3) with a fixed time limit tmax, i. e., the best-found set S is output when this time limit is reached.

B. Obtaining a Set of Matrices
Using Algorithm 1 we are able to find a sparse representation of a dense parity-check matrix. For applying MBBP decoding we however need a diverse set of suitable paritycheck matrices. In a first step, we therefore apply Algorithm 1 in different orders of the rows to obtain different matrices. Since Algorithm 1 is only a heuristic algorithm (as there is the time limit tmax for solving the row-wise ILPs), its output is not exact and depends on the input. With a different order in which the rows are processed, we therefore obtain different outputs. The procedure is depicted in Algorithm 2.
After calling Algorithm 2, N matrices are obtained that are unique up to row permutations, as permuting rows results in the same performance in BP decoding (see Section II-D).
In a second step, out of these N matrices, we want to select a set of l matrices to be decoded in MBBP decoding. The goal is to not only select “good” matrices under BP decoding, but to select several matrices that complement each other best. This problem can be viewed as a Maximum Coverage Problem [22]: Given a collection of sets Si and a natural number l, the goal is to select l sets such that the number of elements in the union of those l sets is maximized. In our case, this translates to the following: For obtaining the sets Si, we performed test simulations (1000 frames at an SNR of 2 dB) for every decoding instance i ∈ {0, . . . , N − 1} corresponding to the N different matrices. The sets Si then represent the correctly decoded frames for this decoding instance, so Si ⊆ {0, . . . , 999}. Out of those N sets, the goal is to select l sets such that the number of correctly decoded frames in MBBP decoding is maximized. As the maximum coverage problem is an NP-hard problem, we use a straightforward heuristic that successively adds the set to the output, which complements the previously chosen sets most [22]. The procedure is formalized in Algorithm 3.

C. LDPC Codes
In Figure 6 the FER performance of the (128, 64) CCSDS LDPC code [23], [25] under MBBP decoding for different l is depicted. As the code’s parity check matrix already has maximum sparsity, applying Algorithm 2 will always output the same matrix. We therefore first randomly densified the original matrix by performing random row operations and then applied Algorithm 2 to obtain different matrices for the (128, 64) CCSDS code that contain up to 15% more oneentries than the original matrix. As most of the frames were already correctly decoded using the original matrix, we only considered the additional decoder instances if the original one failed. It can be seen that the FER improvement under MBBP decoding is around 0.5 dB at an FER of 10−3. Note that the automorphism group for non-structured LDPC codes is in general unknown. Especially in the case of this CCSDS LDPC code, AED cannot be applied.
V. CONCLUSION AND FUTURE WORK
In this work, we presented an approach for the generation of a diverse set of parity-check matrix representations for any given code, eventually enabling the application of MBBP decoding as a universal decoding concept. For LTE Turbo codes, we demonstrated that the presented approach achieves similar FER performance compared to the state-of-the-art max-Log MAP decoding algorithm. In addition, we showcased that our approach can also be used to improve the errorcorrecting performance for short LDPC codes over that of conventional BP decoding.
For future work, combining our approach with a weighted BP decoding algorithm (as proposed in [9]) is of great interest. The introduction of edge weights could not only improve the convergence behavior of the individual decoder (see Figure 5) but also the diversity among the set of decoders operating on the matrices obtained by our approach. This could also result in a crucial performance improvement for applying MBBP decoding for short LDPC codes.
