Minimum Repair Bandwidth LDPC Codes for Distributed Storage Systems
Massoud Pourmandi , Member, IEEE, Ali Emre Pusane , Senior Member, IEEE, Suayb S. Arslan , Senior Member, IEEE, and Elif Haytaoglu , Member, IEEE

Abstract— In distributed storage systems (DSS), an optimal code design must meet the requirements of efficient local data regeneration in addition to reliable data retention. Recently, lowdensity parity-check (LDPC) codes have been proposed as a promising candidate that can secure high data rates as well as low repair bandwidth while maintaining low complexity in data reconstruction. The main objective of this study is to optimize the repair bandwidth characteristics of LDPC code families for a DSS application while meeting the data reliability requirements. First, a data access scenario in which nodes contact other available nodes randomly to download data is examined. Later, a minimum-bandwidth protocol is considered in which nodes make their selections based on the degree numbers of check nodes. Through formulating optimization problems for both protocols, a fundamental trade-off between the decoding threshold and the repair bandwidth is established for a given code rate. Finally, conclusions are confirmed by numerical results showing that irregular constructions have a large potential for establishing optimized LDPC code families for DSS applications.
Index Terms— Distributed storage, LDPC, data regeneration, BEC, repair, differential evolution (DE).
I. INTRODUCTION
I T HAS recently become commonplace to find technologies that deliver various services to their end-users through Internet, such as music and video streaming. Since these technologies require access to vast amounts of data, storage systems must be optimally configured. Since partial data loss is abundant in modern data centers [1], centralized storage systems are progressively replaced with distributed storage systems (DSS). Even though traditional erasure coding schemes such as Reed-Solomon (RS) codes [2] achieve the optimal storage overhead for a given data reliability target, they fail to meet low repair bandwidths. Such weaknesses eventually led to the birth of regenerating codes, a (minimum storage regeneration (MSR)) code family that maintains the same level of reliability as traditional erasure codes while providing low repair bandwidth [3]. However, the decoding complexity of classical regenerating codes based on algebraic constructions

In [5], LDPC codes are shown to be advantageous for fault tolerance in cellular networks, particularly when the cost of downloading a symbol from a base station is not excessive compared to other local network nodes. Additionally in [6], LDPC codes are compared with available coding schemes to determine their suitability for DSSs. In [7], it is demonstrated that LDPC codes in computing clusters require less redundancy for the same fault tolerance level compared to RS codes and replication. Also, it is clearly shown that different configurations of LDPC codes use different amounts of system resources [8]. In [9], in addition to well-known asymptotic performance, authors presented that the use of finite-length LDPC codes is also a viable option for storage systems. In [10], LDPC codes of relatively large blocklengths are compared with RS codes and locally repairable codes (LRC) in terms of recoverability and repair complexity. It is argued that the data reconstruction capability of LDPC codes could be increased without affecting the repair bandwidth performance, as opposed to algebraic code constructions [10]. In [4], a characterization of the average repair bandwidth of LDPC code families is performed. Their work results in a constant check node degree that yields the minimum repair bandwidth. Contrary to [4], this letter characterizes the entire trade-off space of minimum average repair bandwidth and maximum level of recoverability (decoding threshold) for a given code rate. While the tradeoff points exist for codes with regular check nodes, there is still an element of ambiguity for codes with irregular check nodes. Thus, this study attempts to characterize the trade-off between repairability and recoverability for any LDPC code family. To achieve this goal, a repair protocol encompassing the work in [4] is first considered. Then, an optimized repair strategy (named as ideal repair) is used, inspired by the cost of guessing framework [11].

In Section IV, the optimization problem associated with the proposed code construction is provided. Finally, the numerical results are presented in Section V before the letter is concluded in Section VI.

IV. CODE CONSTRUCTION
This section describes an optimization technique to find LDPC code ensembles with low repair bandwidth and high data reliability properties. For the random access repair protocol, assuming γ¯m(riannd) as a fixed constant, ρ(x) is explicitly derived using Theorem 1. However, for the ideal repair protocol, it is necessary to determine the check node degree distribution in addition to the variable node degree distribution.
Since the differential evolution (DE) is proven to be useful in designing storage-efficient LDPC codes [15], [16], a variant of the DE algorithm is employed in this work to search for a repair-efficient code design. In DE, one of the objectives is preserved as the main objective function, while the rest are constrained. Then constraints are handled by a constraint handling method in that the constraint g(x) ≥ g0 is forced to be satisfied by adding the related cost, i.e., max(1 − g(x)/g0, 0), to the main objective function using a multiplier [17]. In the case of random access repair protocol, the main cost function is formulated as 1 − t(λ, ρ) where t(λ, ρ) is the target decoding threshold. Whereas in the ideal repair protocol, the
