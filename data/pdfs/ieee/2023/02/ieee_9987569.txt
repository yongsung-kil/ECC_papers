Design of Implicit Partial Product-LDPC Codes and Low Complexity Decoding Algorithm
Yinchu Wang , Qianfan Wang , and Xiao Ma , Member, IEEE

Abstract— In this letter, we are concerned with a recently proposed class of product codes, called implicit partial product low-density parity-check (IP-LDPC) codes, which consist of LDPC codes for rows and Bose-Chaudhuri-Hocquenghem (BCH) codes for columns. Distinguished from the conventional product codes, only partial columns are doubly-protected and the column checks are transmitted implicitly rather than explicitly. For any given LDPC code, we present three heuristic rules to design an IP-LDPC code with the same code rate, showing how to select the number of doubly-protected columns and the BCH code (for columns). The stringent constraint is that the product of the number of doubly-protected columns and the length of syndrome per column must be less than or equal to the product of the number of the rows and the length of extra bits that can be carried reliably and implicitly per row (LDPC codeword). We also propose a simple bit-flipping algorithm for the column decoding in the case when the number of unsuccessfully decoded rows is relatively small, which occurs often especially when the iterative decoding is implemented. The simulation results show that IP-LDPC codes can lower down the word error rate (WER) of a (3,6)-regular LDPC code with length 1024 from 10−2 to 10−6 at the SNR around 2.0 dB but without any code rate loss.
Index Terms— Bit-flipping, free-ride coding, implicit transmission, LDPC codes, product codes.
I. INTRODUCTION
C ONVENTIONAL product codes were proposed by Elias [1], where an information array is protected not only row-by-row but also column-by-column. Due to the inherent parallel structures of their encoding and decoding algorithms, the product codes have been adopted in many standards, such as satellite communication systems [2] and data storage devices [3]. Due to the near-capacity error correcting performance, low-density parity-check (LDPC) codes, proposed by Gallager in early 1960s [4], are applied to product codes as the component codes. For instance, a class of product codes, proposed in [5], consist of two LDPC codes as the row and column codes. Since the algebraic codes have the superior performance in resisting burst errors, they can be used in constructing product codes. For instance, the LDPCReed Solomon (LDPC-RS) product codes were employed for

To reduce complexity in moderate and high SNR regions, a class of rate-compatible LDPC-RS codes, proposed in [7], employs a hybrid iterative scheme for decoding. To improve the error-correction performance, several approaches have been proposed, such as introducing the interleavers [8] and using the post-processing technique [9]. For the conventional product codes, the total code rate is less than those of the component codes, which is not preferable in the bandwidthlimited regime. In contrast, a class of partial product codes without rate loss was proposed in [10] and [11], where the partial columns check bits are transmitted implicitly along with the row component LDPC codes by using the free-ride coding.
In this letter, we focus on the above-mentioned implicit partial product low-density parity-check (IP-LDPC) codes and discuss their constructions in details. Aided by simulations, we present three rules for design, which show how to choose properly the number of doubly-protected columns and the Bose–Chaudhuri–Hocquenghem (BCH) code for columns. In addition, we propose to embed a simple bit-flipping algorithm into the iterative decoding as the column decoding algorithm, which has a low complexity in the case of few unsuccessfully decoded rows. With the bit-flipping algorithm, the CRC codes [12] instead of the BCH codes can be used for the column coding to introduce more flexibility. The simulation results show that the IP-LDPC codes can reduce the word error rate (WER) performance of a (3,6)-regular LDPC code of length 1024 from 10−2 to 10−6 at the SNR around 2.0 dB.
II. IMPLICIT PARTIAL PRODUCT-LDPC CODES
Similar to the conventional product codes, the construction of IP-LDPC codes involves two component codes, one for rows and the other for columns. The row code is a binary LDPC code C [n, k] of dimension k and of length n, characterized by its parity-check matrix H, while the column code is a BCH code C1[n1, k1], characterized by its generator polynomial g(x) of degree r = n1 − k1 over the binary field F2. A decoding success for the LDPC codes means that a valid codeword is found by the iterative decoding algorithms, while that for the BCH codes means that a valid codeword is found by either the conventional hard-decision decoding algorithms or the bit-flipping algorithm presented in this letter.

B. Free-Ride Transmission of Column Checks

Now we briefly describe the free-ride coding [13], which is
critical to the construction of IP-LDPC codes. The underlying
principle of free-ride coding is to superimpose extra bits onto
payload LDPC codewords, as illustrated in Fig. 2. Denote by s collectively all the syndromes sj, 0 ≤ j < ks. Without loss of generality, we assume that s can be partitioned into n1 segments of equal length  = rks/n1 (if necessary, zeros can be appended to s), which are denoted as f (i), 0 ≤ i < n1. In this letter, we employ Reed-Muller (RM)-based free-ride codes. Let GRM be the generator matrix of an RM code. The encoding algorithm of the IP-LDPC codes is presented in Algorithm 1.
Algorithm 1 The Encoding Algorithm
Input: Information array u of size n1 × k. output: Transmitted codeword c of length nn1.
1) Row Encoding: For 0 ≤ i < n1, the i-th row u(i) is encoded into an LDPC codeword v(i) of C [n, k], resulting in v = (v(0), v(1), . . . , v(n1−1)).
2) Column Encoding: For 0 ≤ j < ns, by treating the j-th column uj of the array u as a polynomial, compute the j-th syndrome sj as sj(x) = uj(x) mod g(x), resulting in the column checks s = (s0, s1, . . . , sks−1), where g(x) is the generator polynomial of the BCH code.
3) Free-ride Encoding: The column checks s are grouped into n1 subblocks of equal length  = rks/n1 (appending zeros if necessary), which are denoted as f (i), 0 ≤ i < n1. Each subblock f (i) is encoded into a free-ride codeword of Cf [n, ] by g(i) = f (i)Gf , resulting in g = (g(1), g(2), . . . , g(n1−1)). Then the transmitted codeword c is computed as c = v ⊕ g.

III. PERFORMANCE ANALYSIS AND DESIGN EXAMPLES
The design begins with an LDPC code. In this letter, we take as an example a (3,6)-regular LDPC code C [1024, 512]. We need answer how to choose ks and the BCH codes.

A. The Choice of ks
Since the first ks columns of u are doubly protected, they may be assumed to be known as the iteration in Algorithm 3 increases. From another point of view, the product code can not have better performance than the LDPC code with ks known bits at the decoder. The following example presents the performance of the LDPC code with ks known bits.
Example 1: We consider an LDPC codeword from C [1024, 512] that is transmitted with BPSK modulation over an AWGN channel. For simplicity, we assume that the first ks coded bits are told by a genie to the decoder. The simulation results with different ks are shown in Fig. 3, where we observe that the performance can be improved by increasing the number of known bits, as expected.
We have the following rule. Rule 1: Choose properly ks such that the performance lower bound is satisfactory. For instance, at SNR = 2.0 dB, to achieve the WER of 10−4, we need choose ks ≥ 100.
B. The Choice of BCH Codes
To validate the assumption that the doubly-protected bits are almost known at the decoder, we need the following rule.
Rule 2: Choose a BCH code C1[n1, k1, dmin], where dmin is the minimum Hamming distance of the BCH code, such that almost all the residual errors after the row decoding can be corrected. That is, dmin/n1 ≈ WER, where WER is associated with the row code.
For example, from Fig. 3, we see that the WER without known bits is about 10−2 at SNR = 2.0 dB. So we need choose a BCH code such that dmin/n1 ≈ 10−2.
Finally, the critical point is whether or not the column syndromes can be reliably transmitted along with the LDPC codewords. To answer this question, we have simulated the free-ride codes on the LDPC code, as shown in Fig. 4.

IV. CONCLUSION
Through analysis and examples, this letter has presented three heuristic rules to the design of the IP-LDPC codes. This letter has also provided a bit-flipping algorithm for column decoding, which has a low complexity especially in the case of few unsuccessfully decoded rows.
