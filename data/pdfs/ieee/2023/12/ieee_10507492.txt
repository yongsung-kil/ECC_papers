Abstract—In this paper, we propose an improved Syndrome Bit Flipping (SBF) decoding algorithm that utilizes narrowed flipping set. The narrowed flipping set is prone to contain all error bit positions with high probability. To construct the narrowed flipping set, part of its elements are selected according to channel information, and the rest are constructed by random selections. The narrowed flipping set mitigates possible error propagation by avoiding some non-error bits being unnessary flipped, thereby further improving the decoding performance of the original SBF algorithm. The simulation results show that the proposed algorithm can achieve about 1.2 dB bit error rate gain compared with the original SBF algorithm, with a modest increase in computational complexity.
Index Terms—Low-density parity-check codes, syndrome bit flipping, narrowed flipping set, error propagation
I. INTRODUCTION
As an exemplary class of channel coding techniques, lowdensity parity-check (LDPC) code has shown excellent performance in low SNR regions, where the bit-error rate (BER) performance can exponentially decrease with the increase in SNR. However, in high SNR regions, the BER often ceases to decrease exponentially after reaching approximately 10−7, manifesting a phenomenon known as the error floor [1]. To achieve enhanced decoding performance of LDPC codes, various decoding algorithms have been extensively studied. The two primary categories are the Sum-Product Algorithm (SPA) [2] and Bit Flipping (BF). SPA is sometimes referred to as belief propagation (BP) decoding algorithm, it is based on soft decision, and achieves excellent BER performance. However, it also demands substantial computational resources in its calculation process.
The BF algorithms have lower algorithmic and hardware complexity compared to SPA algorithm, while the trade-off is that the BER performance is far from optimal. Over the years, various BF algorithms were proposed for making a balance between BER performance and decoding complexity, among which the Gradient Descent Bit Flipping (GDBF) algorithm is a landmark BF algorithm [3]. The performance of the GDBF algorithm outperforms most previously proposed BF algorithms. Moreover, a variant of the GDBF algorithm, known as NGDBF algorithm [4], further improves the BER performance

due to the incorporation of noise perturbations into the energy function. Based on the GDBF and NGDBF algorithms, several enhanced algorithms [5–7] have been proposed, some of which are close to the BP algorithm in terms of BER performance.
Recently, a novel algorithm named Syndrome Bit Flipping (SBF) was proposed in [8], which is similar to Mulit-Threshold Bit Flipping (MTBF) presented in [9]. SBF exhibits a concise design in both algorithmic structure and hardware architecture. Consequently, SBF decoder is suitable to serve as a postprocessing decoder for other algorithms. During the iterative process of the SBF algorithm, bits with energy functions greater than threshold sequence value are flipped. However, when the values within the threshold sequence are very small, such flipping may lead to the bit-flipping of some non-error bits, causing error propagation. Consequently, this undermines the error-correcting performance of the SBF algorithm.
To handle this problem and further improve the performance of SBF algorithm, narrowed flipping set are employed in the SBF iteration. The narrowed flipping set successfully avoids some non-error bits being unnessary flipped, resulting in a considerable BER gain. The primary contributions of this paper are summarized as follows: 1. In the SBF algorithm, the narrowed flipping set is introduced to constraint the flipping set found by the original SBF algorithm. 2. Determining the optimal method for selecting narrowed flipping set.
II. PRELIMINARIES
A. Notation
We denote the LDPC binary m × n parity-check matrix as H, where n > m ≥ 1 and {m, n} ∈ Z+. Accordingly, the binary code associated with H is defined by C ≜ {c ∈ F2n : Hc = 0}, where F2 represents the binary Galois field. The bipolar codewords Cˆ corresponding to C are generated by Cˆ ≜ {(1 − 2c1, 1 − 2c2, . . . , 1 − 2cn) : c ∈ C}, i.e. Cˆ ≜ cˆ ∈ {−1, +1}n. In Additive White Gaussian Noise (AWGN) channel transmission, the received vector at the receiver is defined as ⃗y = cˆ + ⃗z, where ⃗z is a random variable following N (0, σ2), and σ2 is the noise variance. We then we define the bipolar decision vector xˆk = sign(yk), k = 1, 2, . . . , n, and the codeword vector ⃗x = (1 − xˆ)/2. Thus, the syndrome is computed as ⃗s = H⃗x mod 2.

III. SBF WITH NARROWED FLIPPING SET A. Error Propagation in SBF

In the SBF algorithm, the flipping set is constructed by SSBF = {n ∈ {1 . . . N }; Evn > θk(ℓ)}. However, when the threshold sequence value is small, such as when θ(ℓ) = 0, and there is only one wrong ’1’ variable node v¯ in the codeword, the existence of v¯i results in adjacent check nodes c = {cj}j∈M(i) being non-satisfied checks. Consequently the

variable nodes connected with v = {vj}j∈N (c)\v¯ have E = 1. According to the condition E > θ(ℓ), both the v with E = 1 and v¯ are flipped, which causing the error propagation. It is clear that flipping v is unnessary. This phenomenon leads to the question of how to effectively constrain the flipping set found by SBF algorithm to reduce the number of non-error bits being unnessary flipped.
The preceding analysis of the effects of error propagation caused by unnessary bit flipping instructs us to construct a flipping set S that includes the positions of all error bit positions Serr and another n non-error bits, and we use S to constrain the flipping set SSBF. Essentially, the original SBF algorithm can be viewed as a circumstance where S contains all the bits in the codeword, then as n decreases, the flipping set SSBF becomes increasingly constrained by S. We use IEEE 802.3an LDPC code [10] with code length N = 2048 and code rate R = 0.8413 as an illustration, Fig. 1 shows that constraning SSBF resulting in a notable BER gain compared to the SBF algorithm, and BER gain increases as n decreases. Thus constraning SSBF is a viable strategy to optimize the SBF algorithm.

B. SBF Decoding Algorithm with Narrowed Flipping Set
Inspired by the above observation, we can construct a narrowed flipping set S to constrain the SSBF. It is essential for S to satisify the condition Serr ⊆ S, as neglecting any one of error bit positions would lead to the decoding failure. Furthermore, the shorter the length of S, the better performance SBF constrain by S will be.
Firstly, the channel information is used to identify the majority of error bits. Bits with lowest magnitude of |⃗y| are considered as unreliable bits. We select the top T1 unreliable bits to constitute the subset S1 as first part of S. The SBF decoding with narrowed flipping set, denotes as NFS-SBF, is described in Algorithm 1. The decoder parameters include T1, T2 and γ (maximum number of search trials for S). The algorithm is divided into two stages. In the first stage, SBF is implemented. If SBF decoding does not converge to the correct codeword, the second stage employs the narrowed flipping set S to narrow the SSBF found by the SBF algorithm. To reduce the complexity, only one single threshold sequence is used in the second stage, as a successful SBF decoding process usually end at first threshold sequence.

IV. RESULTS AND DISCUSSION
A. BER Performance Comparison
The simulation results are based on LDPC codes (code 1: IEEE 802.3an LDPC codes, code 2: PEGReg504×1008 code from Mackay’s LDPC database [11]). In our simulations, we always consider AWGN channels and employ BPSK modulation, and maximum iteration number is set at 200 for SBF. The threshold sequences for code 1 is the same as [8], while the threshold sequences for code 2 can be found in the Appendix. (T1, T2) = (300, 400) is applied on code 1 and (T1, T2) = (400, 200) is applied on code 2. Meanwhile, M-GDBF and S-NGDBF are included as benchmarks. The parameters for code 1 are M-GDBF with θ = −3.8 and SNGDBF with w = 1/6, η = 0.9, for code 2 are M-GDBF with θ = −0.6 and S-NGDBF with w = 0.75, η = 0.95.
In Fig. 4 , we can observe that NFS-SBF with γ = 10 already leads to a gain of more than one order of magnitude in terms of BER at Eb/N0 = 5.5 dB. At the target BER of 10−6, around 0.7 dB gain is achieved by NFS-SBF decoding algorithm compared to the original SBF decoding algorithm. Also The BER performance of NFS-SBF with γ = 10 overall surpasses that of M-GDBF, while NFS-SBF with γ = 100 underperforms S-NGDBF by approximately 0.25 dB. For code 2, the BER gain is still evident as shown in Fig. 5. Specifically, at BER = 10−6, the NFS-SBF algorithm with γ = 50 achieves a 1.2 dB gain compared to the original SBF algorithm, while it is inferior to the S-NGDBF by 0.3 dB.

C. More Heuristics
Moreover, we discovered that the mechanism of narrowed flipping set is not only applicable to SBF, but also to other multi-bit flipping algorithm based on threshold comparisons, such as MTBF, M-GDBF and M-NGDBF. These algorithms share the same characteristic that during the multi-bit flipping processes, some non-error bits are flipped together with error bits. And varying degrees of BER performance gains can be observed when the NFS mechanism is applied to these algorithms. The NFS mechanism is a cross-validation mechanism - the flipping set is first identified by multi-bit flipping algorithm, then narrowed flipping set filters out some nonerror bits that do not need to flipped. Thereby this mechanism constrains the route of bit flipping, and enhances the decoding performance. However, one limitation of NFS is that at lower SNR, we cannot narrow down the flipping set by selecting the top-n unreliable bit positions, as the distribution of error bit positions in the LDPC code is chaotic under this circumstance. If there exists a method more effective than using channel information to select unreliable bit positions, we believe that the decoding performance of NFS could be further improved.
V. CONCLUSION
In this paper, we proposed an improve SBF algorithm based on narrowed flipping set. Narrowed flipping set is found based on channel information and random selection, and it narrowed the range for bit-flipping in SBF, therefore suppresses error propagation effectively. The NFS-SBF decoding algorithm has been compared with the SBF algorithm and some other

advanced BF algorithms. The simulation results show that our algorithm outperforms the SBF algorithm, with a modest complexity increasing.
