A High Flexibility Decoding Structure with High Hardware Utilization Efficiency for QC-LDPC
Zhenliang Ye†, Xiongfei Zhai†, Jinghong Mo†, and Guojun Han† †School of Information Engineering, Guangdong University of Technology, Guangzhou 510006, China
Email: leon1621@163.com, zhaixiongfei@gdut.edu.cn, 1393044175@qq.com, gjhan@gdut.edu.cn

Abstract—Due to the excellent performance of the quasi-cyclic low-density parity-check (QC-LDPC) code, it has attracted a lot of attention in communications and storage recently. To achieve high flexibility, some flexible 5G LDPC decoders are reported. However, in the existing work, the resources are underutilized for short codes, which yields the poor processing rate and resource utilization. In this paper, we propose a highly flexible decoding structure with high hardware utilization efficiency (HUE) that can be applied to multiple various code lengths simultaneously. Besides, a dynamic planning method for node reordering within layers is developed, which can perfectly solve the memory access conflict problem. The experimental results show that the proposed structure can achieve significant improvement in hardware utilization compared with the existing works.
Index Terms—Quasi-cyclic low-density parity-check (QCLDPC) code, flexible, node reordering, memory access conflict.
I. INTRODUCTION
Among the linear block codes, one of the impressive codes which can approach the Shannon limit is the low-density parity check (LDPC) code. The decoder of LDPC code generally adopts an iterative algorithm based on node confidence update [1], which is an effective way to approach the Shannon limit based on confidence propagation. As a special type of LDPC code, the quasi-cyclic LDPC (QC-LDPC) code can be implemented in parallel and then the throughput can be increased linearly by introducing a small amount of additional hardware [2]. Due to the above advantages, QC-LDPC codes have gained popularity in the areas of communication and storage. These applications require that the QC-LDPC decoder can be applied to various codeword length, where the codeword length is linearly proportional to the lifting parameter (Z).
There are 51 value of Z (varies from 2 to 384) and two base graphs to be selected in 5G standards. To achieve high flexibility, related works are reported in the literature [3]–[6]. However, in the existing works, the resources are underutilized for short codes, which yields the poor processing rate and resource utilization. There are two approaches to address the above issue in field programmable gate array (FPGA). The first one utilizes a generic decoding architecture with a parameterized design, allowing the rapid synthesis of different circuits based on different parameters. However, each circuit can be only applied to a specific code length. The second one is to design a reconfigurable decoder. In [7], the proposed architecture supports the parallel decoding of codewords with the same length to improve the resource utilization. However, the simultaneous and parallel decoding of codewords with different lengths is not considered, which means the resource utilization can be further reduced when Z is from 192 to 384.
Motivated by the above analysis, we propose a highly flexible decoding architecture with high hardware utilization efficiency (HUE) in this work. The main contributions are summarized as follows.

1) Inspired by the reconfigurable permutation network [8], we propose a reconfigurable decoding structure for QC-LDPC to simultaneously decode multiple codewords with various lengths, which improves the HUE and saves processing power.
2) An efficient scheduling algorithm is developed in the pipeline structure, which can effectively reduce the number of idle clock cycles and improve the latency performance. Besides, the convergence is improved by using our proposed algorithm based on node reordering.
The remainder of this paper is organized as follows. Section II introduces the structural features of QC-LDPC code for the 5G standard and describes the layered decoding algorithm based on normalized min-sum. Section III details the decoder structure proposed in this paper and the improved scheduling algorithm. Section IV presents the results of the experiments. Finally, Section V draws the conclusion.
II. 5G LDPC CODE STRUCTURE AND LAYERED DECODE ALGORITHM
This section introduces the 5G LDPC code and the layered decoding algorithm. Besides, the traditional decoding structure is analyzed.
A. 5G LDPC Code
QC-LDPC code is used in 5G standards, whose parity check matrix (PCM) can be represented by a base graph and a corresponding lifting parameter Z. Each non-zero element in the base graph can be expanded into an identity matrix with a cyclic shift of Z × Z. The typical structure of PCM is shown in Fig.1, where A is a 4×kb high-density matrix corresponding to information bits, B is a 4×4 dual diagonal matrix corresponding to the check bits, O is a 4 × (mb − 4) all-zero matrix, D is a (mb −4)×(kb +4) sparse matrix, and E is a (mb −4)×(mb −4) unit matrix. [A, B] forms the core of this PCM, denoted by Hcore. The PCMs in the 5G standard can be obtained by extending Hcore. Besides, 5G also provides a rate-matching scheme to meet the demand for flexible code lengths and rates.
Considering the diversity of 5G scenarios, two base graphs, BG1 and BG2, are proposed to enable various code lengths and rates. Table I shows the basic parameters of the two base graphs. As we can see from the table, the node degree distribution of the two matrices are highly uneven, and the orthogonality between the consecutive rows is poor, which can lead to memory access conflict problems in the implementation. Therefore, we need a reliable data scheduling mechanism to deal with this issue.
B. Layered Decode Algorithm
A traditional iterative decoding algorithm for QC-LDPC code is based on a fully parallel flooding update strategy, which can achieve high throughput in the implementation at the cost of high resource utilization. In order to achieve a balance between decoding speed and hardware utilization, q turbodecoding message-passing algorithm has been proposed in [9].

Its basic idea is to layer a check matrix by rows and update the node message of each layer in sequence. The updated node message of the previous layer can be used immediately to update the next layer. The layered decoding algorithm converges much faster than the one based on the flooding update strategy. In general, each Z row is viewed as one layer in QC-LDPC code, leading to perfect orthogonality among the layers due to the quasi-cyclic structure. Hence, it is possible to decode Z nodes in parallel.

C. Layered Decoding Structure
The traditional layered decoding structure is shown in Fig. 2. The global control unit (GCU) provides the indexes of the variable and check nodes, as determined by the PCM. The variable nodes process units (VNU) and check nodes process units (CNU) perform the updates of variable nodes and check nodes. Besides, the LLRs are stored in the memory banks which always contains up-to-date values.
However, there are two flaws in the traditional layered decoding structure, which is shown as follows.
F law 1: 5G LDPC code has a well-designed structure that supports high parallelism but is only suitable for long codewords with the highest size. For short codewords, the processing unit is not fully utilized, leading to a significant decrease in decoding speed.
F law 2: To improve the throughput performance, the above decoding structure is usually implemented by exploiting the pipeline design in practice, leading to the problems in memory access.

A. Proposed Architecture
At the beginning, some decoding parameters, including lifting parameter and parity check matrix information, need to be input into the FPGA. The proposed structure is shown in Fig. 3, which includes a dual-port memory bank that can store LLRs of 68 words of 9 × Zmax bits. In order to decode multiple codewords simultaneously when Z < Zmax, different channel LLRs need to be concatenated before decoding. An input/output interface unit writes the LLRs into the memory banks and reads the decoded bits if they are decoded correctly, or the maximum number of iterations is reached. The GCU provides the node address, offset permutation values, and control signals. As the key unit, the GCU exchanges information with all units. The check unit performs parity check.
To achieve high flexibility, the permutation network is essential for implementing a reconfigurable decoder, which can rotate Z outputs of the memory banks at each clock cycle with Z < Zmax. Combining with the parity check module, a reconfigurable structure is constructed, which will be discussed in Section III-B.
These Z rotated LLRs are then processed by a nodeprocessing unit (NPU), which aims to improve throughput. The

The key points to implementing a reconfigurable QC-LDPC decoder is the reconfigurable implementation of cyclic shift and verification functions. Recently, a new shifter network was proposed in [8] and [10], suitable for multi-codeword parallelism. The proposed shifter can be divided into multiple segments of different sizes by separating the control signals of each path and then re-group those paths (or control signals) together to form the final segments [8]. In order to construct a suitable shift register for our architecture, we modify the reconfigurable register in [8] as follows.
1) The offset permutation technique [11] is exploited to avoid rotating the LLRs back to their original position, which improves the throughput.
2) When Zmax = 384, it needs a 9-stage path, which requires more clock cycles than a traditional cyclic shift register. The performance of throughput is degraded since more idle clock cycles are introduced when the conflicts occur. To address this issue, we divide the entire programmable cyclic shifter into four stages for pipelining. Besides, the operation of calculating the modulus of the input circular shift coefficients is moved forward in the processing pipeline.
For a better understanding, an example is provided in Fig. 5 when Zmax = 12 with different input data sizes. Unlike conventional designs that include the control signals of all paths of the shifter into one set, the proposed structure separates the control signals of each path and includes control signals of all path within the same segment into one set. As shown in Fig. 5, the rotation value [3:0] (RV [3:0]) is the control signal of the right logarithmic shifter, which controls stages 1, 2, 3, and 4 of the path, respectively. If the control signal of a stage is’ 0 ’, the output of that stage selects the input from the same path (dashed line). The network can be divided into two sections each of which can work separately without being affected by the other part. In this case, the size of each input data must be equal to or smaller than 6. Each section can perform a cyclic shift right independently according to the rotation value. When the input data size Zn > 6, the separate networks can be recombined and perform cyclic shift right with the same rotation value. Note that the shifter can only simultaneously perform a cyclic shift for two sets of inputs at most.
In 5G standard, the input data size of the cyclic shifter is selected from 51 values varying from 2 to 384. In order to reduce hardware complexity, the 384 paths of the shifter network are divided into 6 segments. Hence, the decoder can only simultaneously decode for six inputs at most, and the sum of the lifting parameters of all codewords cannot be greater than 384.

As we can see, the throughput can be improved if the decoding algorithm converges fast (i.e., t is small). Hence, we design a checking module to terminate the current decoding early and decode the next codeword if the parity check passes. It is required that the check module for the multi-codeword parallel QC-LDPC decoder needs to identify which codeword triggers the termination condition in advance. Besides, when multiple codewords with different code lengths are decoded, the decoded bits must be frozen if they are identified to be

valid. These bits will not be output until the all codewords are identified. Hence, the following inter-connected blocks are essential.
1) A ping-pong mechanism is adopted to pre-cache the sign bit of LLRs, which allows the processing of next iteration without a break while performing parity check.
2) A shifter unit is exploited to shift the sign bits of LLRs. It consumes much fewer resource than reconfigurable permutation networks.
3) An early termination identification unit is needed to perform XOR operation on the shifted sign bits according to BG. The final result will be all 0 if the parity check passes. In order to detect which codewords have been decoded, the XOR operation is performed independently according to the segments.
4) A buffer unit is required to freeze the decoded output of segment according to the result of early termination identification unit. And let other codewords continue the next iteration of decoding.

C. Dynamic Planning
Pipeline architecture is needed to satisfy the requirements of a high-speed decoder. Nevertheless, partially parallel pipeline design inevitably brings memory read and write conflicts. For example, the updated LLRs of the previous layer have not been input to the memory yet when the current updating layer tries to read the memory. To avoid the memory conflict effectively, we design a node dynamic planning method. To facilitate the node dynamic planning method, we define the node as a shared node if two consecutive layers share this node. The algorithm can be implemented in the programmable logic (PL) of an FPGA by exploiting a finite-state machine. To avoid the conflicts in data scheduling, idle cycles need to be added. The number of idle cycles is related to the position of the nonzero nodes of the check matrices. However, the introduced idle cycles bring latency and degrade throughput.
Compared with traditional scheduling methods, dynamic scheduling mechanisms allow the decoder to dynamically optimize data processing in real-time based on the characteristics of the parity check matrix. Although the control unit of the address conflict is also proposed in [7], it requires a huge amount of idle clock cycles to wait for the previous conflicting nodes to update. In contrast, our proposed dynamic programming algorithm can skip the conflicting node and update the next non-conflicting node first. Besides, if it is a shared node, the node is updated preferentially in the second stage of NPU. Thus the convergence speed can be further improved. Fig. 6 shows the timing diagram of different methods for dealing with the conflict problem of shared nodes. Here only the processes of reading and writing node messages are presented. As we can see, the number of idle cycles is reduced from 10 to 3 by exploiting the dynamic planning method, which can enhance the throughput significantly.

IV. EXPERIMENT RESULTS
In this section, we propose the experiment results of our proposed structure as compared with the existing works. For a fair comparison, the throughput is normalized by Tnorm = T ×t.
Table II compares the resource utilization and the normalized throughput of our proposed structure and the existing works. Quantization level (9,9,6) represents the initial LLR quantization of 9 bits, the variable node message V2C quantization of 9 bits, and the check node message C2V quantization of 6 bits. HUE represents the normalized throughput per unit resource. We can see that the existing works’ structures can achieve higher normalized throughput than our proposed one when Z = Zmax. However, for smaller Z such as 192, Table III show that the average throughput of the proposed decoder still remains 23.713 Gbps, while the average throughput of the decoder in [3] drops to 16.250 Gbps.
Besides, the proposed structure has better HUE performance as compared with the existing work in [7]. For the maximum lifting parameter, the average throughput per kLUTs of our work is higher than those in [7] with 25.14% improvement. And the average throughput per BRAMs is improved by 165.5%. According to the previous analysis, our architectures can reuse resources when Z is not the maximum. Both the [3] and [7] decoder are not competitive in the average throughput and HUE performance. Hence, our design can serve as a promising candidate for flexible QC-LDPC decoder with highly efficient resource utilization.
V. CONCLUSION
In this paper, to improve the flexibility and HUE, we proposed a reconfigurable multi-codeword parallel decoding architecture. Our proposed structure can be applied to multiple codewords with various code lengths simultaneously by exploiting reconfigurable permutation network and checking module. By jointly exploiting the techniques of reordering, ping-pong mechanism, and dynamic planning, our proposed structure can better address the issue of memory conflicts with improved throughput performance. Experiment results show that our proposed architecture provides more hardware efficiency than the current works at the cost of a small amount of additional memory if the code length (or the lifting parameter) is not the maximum.
