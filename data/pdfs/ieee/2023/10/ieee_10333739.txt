On a Unified Deep Neural Network Decoding Architecture
Dmitry Artemasov, Kirill Andreev, Alexey Frolov Center for Next Generation Wireless and IoT Skolkovo Institute of Science and Technology Moscow, Russia
{dmitry.artemasov, k.andreev, al.frolov}@skoltech.ru

Abstract—In modern communication systems, multiple types of error-correcting codes can be utilized for different transmission scenarios. Therefore, the receiver should include the decoder compatible with multiple codes used in the transmission scheme, which results in the increase of the resources required for its implementation. In this paper, we investigate the possibility of training a single syndrome-based DNN decoder to solve the problem of unified decoding. We observe, that the syndromebased approach allows to extend the unified decoding capabilities to the codes with considerably larger lengths in comparison to the initially described by Wang et al. (2018) method. Through numerical experiments, we show that the model trained for decoding a pair of moderate lengths codes (BCH and CRC-Aided Polar) achieves performance results comparable with classical decoding solutions, while sharing the same architecture and the set of trainable weights. We note, that the unification of a syndrome-based DNN decoder does not lead to large performance degradation, in comparison to the decoder trained on a single code. The approach described in the paper is promising in terms of reducing the hardware resources required to implement the decoder.
Index Terms—channel decoding, deep learning, deep neural networks, unified architecture
I. INTRODUCTION
In recent years, there has been a significant increase in the application of machine learning (ML) algorithms, particularly DNNs, across various fields such as image processing, video analysis, and natural language processing. These applications primarily deal with natural signals. In this paper, we focus on the application of ML methods, specifically DNNs, for the channel decoding problem. It is important to note that the decoding problem can be seen as a classification problem, where the channel output needs to be classified into one of the codewords. The main challenge in this problem is the exponentially large number of classes.
The application of deep neural networks (DNN) for the decoding task is widely considered in the literature. It has already been shown that DNN-based decoders can outperform classical ones. The first papers appeared around three decades ago [1], [2], but at that time the DNN-based decoding approaches were limited by lack of computational resources. Later, in

2017 Gruber et al. [3] identified the model-free direction of the general-purpose DNN application for the decoding task. The intention was to utilize a fully connected neural network (FCNN) to estimate the transmitted information word from the noisy codeword observation. Such an approach suffered from the “curse of dimensionality”. To achieve the performance of maximum a posteriori (MAP) decoding, the neural network (NN) needs to be trained on the entire codebook. However, the codebook size grows exponentially with the number of information bits, making it impractical to train the NN on all codewords for real-world applications. Nevertheless, there is hope that the NN can infer the code structure by observing a limited number of codewords. It is worth noticing that practical codes are linear and can be described by a basis, which means that the basis vectors alone are sufficient for comprehending the code structure. The main discovery of [3] is that while a fully connected NN fails to learn the code structure, it can only generalize the codewords it has encountered during the training procedure. Therefore, this approach is only suitable for very short codes.
Another class of NN decoding methods is neural belief propagation (BP). The main idea of the neural BP approaches is to augment the classical BP algorithm variable and check node update functions with the trainable weights [4]–[6] or to train the graph neural network to learn a generalized message passing algorithm itself [7]. This type of DNN-based decoding methods depend on the type of the code and are widely applied for low-density parity-check (LDPC) decoding due to its sparse nature.
A different model-free approach was proposed in 2018 by Bennatan et al. [8]. It is used to address the aforementioned problem of the “curse of dimensionality”. The idea of the syndrome-based approach is to decouple the transmitted codeword from the channel-induced error and force the NN to estimate the latter from the reliability vector and the syndrome. This approach has the potential to eliminate the need for codeword simulation during training, thereby overcoming the issue of overfitting. The syndrome-based approach is modelfree, which implies that it can be adapted to various NN architectures. The choice of an optimal one is an open question, see [9], [10] for more details.

The simplest way to solve this problem is to equip the receiver with separate decoders for each of the codes utilized in the system. Clearly, this approach is not optimal as it leads to a huge receiver complexity. In this paper we are searching for a solution that (a) can support multiple codes types; (b) has a complexity much smaller in comparison to the trivial solution with separate decoders.
Let us mention the papers related to the question of unified decoder design. In [11] the authors propose a joint decoding algorithm based on the reordering BP algorithm basic operations to the unified add-comparison-add pattern for 5G compatible Polar–LDPC codes. In [12] the authors use the versatility of the BP algorithm to construct a reconfigurable decoding algorithm for Polar–LDPC concatenated codes with shared Tanner graphs. However, this approach depends on a meticulous design of the decoding structure, making it difficult to generalize to different code rates. The possibility of machine learning methods application to solve the problem of a unified decoding algorithm design appears promising due to the primary goal of ML methods – data generalization. In [13] the authors propose a unified deep learning-based decoder for LDPC and polar codes. As the proof of concept authors managed to decode [16, 8] polar and LDPC codes with a fully connected NN (so the authors utilize the architecture from [3]) in the unified scheme and achieved a performance similar to the BP algorithm. As noticed above, such an approach is applicable for very short codes only.
In this paper, we focus on the syndrome-based DNN decoding approach proposed in [8], which is robust to overfitting and is promising for the unified decoding architecture task.
Our contribution is as follows. We start with a syndromebased approach [8] and apply it to the unified decoding task. We consider two scenarios: i) the code type is unknown to the decoder; ii) the code type is passed to the decoder via the indicator flag, as proposed in [13]. We demonstrate the performance of the syndrome-based unified decoding on pairs of systematic [64, 45] BCH – [64, 45] CRC-Aided (CA)Polar and [64, 45] BCH – [64, 21] CA-Polar codes. Unified syndrome-based DNN decoder performance is compared to the results obtained with classical decoders (Chase, SCL) and with syndrome-based DNN decoder trained on a single code. We show that the proposed unified decoding scheme allows achieving comparable to the classical decoding methods performance results while sharing the same architecture and set of trainable weights for multiple codes decoding. We find the described approach prospective for moderate-length codes.
Note that in this study, we do not consider LDPC codes since the described method is designed for short and moderate code lengths. Polar and BCH codes provide better performance in this regime and better fit our main goal – to demonstrate the performance of the unified architecture on some real-world error-correcting codes. In further research, we plan to consider more types of codes and enlarge the code length to make LDPC examples reasonable.

C. DNN model architecture
In accordance with the syndrome-based approach, the role of the DNN, denoted by F, is to estimate the noise induced by the channel. The question of the optimal NN architecture for syndrome-based decoding is an open one. We highlight such works as [8]–[10]. For the experiment setup in this paper we consider Stacked-GRU architecture, as in [8]. This architecture was selected due to its good decoding performance and relatively low amount of time required for training.
Stacked-GRU architecture belongs to the class of Recurrent Neural Network (RNN) models [17]. It is widely applied to Natural Language Processing (NLP) tasks. Stacked-GRU architecture is composed of interconnected Gated Recurrent Unit (GRU) cells [18]. GRU cells are bundled in two dimensions: layers and timesteps. We denote the number of layers by L and the number of timesteps by T . Each GRU cell receives the input from the preceding cell in the layers stacking dimension and the input from the preceding cell in the timesteps stacking dimension as the hidden state, which describes its recurrent structure.

III. UNIFIED DNN-BASED DECODING ARCHITECTURE
For a unified architecture, we need to obtain a decoding function generalized for the codes used in the transmission system. In what follows we consider two scenarios. In the first case, the DNN-based decoder is provided with no information on the code type used for the transmission. In the second case, the code type is provided to NN through the indicator section I, as proposed in [13]. The notion of the code type used for the message transmission is known to the receiver is assumed to be valid by the reason that in modern cellular communication systems modulation and coding scheme (MCS) is reported to the receiver [19]. We presume that in the general case the code indicator I can take the integer value corresponding to the enumerator of the code used in the transmission system or the bit value for the scenario in which only two types of code are utilized.
For the Stacked GRU architecture, we suggest passing a code type indicator to the input of the first layer of GRU cells and to the fully connected layer, as depicted in Fig.2. Recalling the notations from section II-C, the model input vector for the scenario with known code type used for the transmission is redefined as d = [|γ|, ˜s, I]. IV. SIMULATION SETUP
We evaluate the syndrome-based unified DNN decoder performance on decoding pairs of BCH and CA-Polar codes. CA-Polar codes are used for the control channel in the 5G communication systems [20]. For the first pair, we select systematic codes with the same code rate: [64, 45] BCH and [64, 45] CA-Polar with 11 CRC bits. For the second pair, we select codes with different core rates: [64, 45] BCH and [64, 21] CA-Polar with 11 CRC bits. For unified model training on the second pair of codes, the [64, 45] BCH syndrome is extended with a zero vector of length 24, as described in section III.
For the purpose of noise estimation in the syndrome-based DNN decoding approach we use the Stakced-GRU model with 4 layers and 5 timesteps. The hidden size of each GRU cell is set to 5n = 320. Stacked GRU model is followed by a single FC layer.
