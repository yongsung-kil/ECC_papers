Abstract—This paper proposes a novel iterative decoding method - Gradient Descent Symbol Update - for real number parity based error correction codes, as well as its corresponding hardware architecture. The decoding process is based on the gradient descent optimization technique, as well as binary maximum likelihood error correction decoding. We present the error correction performance and the convergence rate of the gradient descent symbol update for a parity based BCH (26,16) code. Furthermore, FPGA implementation results for the corresponding decoder architecture are depicted.
Index Terms—real number codes, error-correction codes, iterative decoding, gradient-descent optimization
I. INTRODUCTION
Real number codes, also called analog codes, have been proposed by Marshall in [1] as a method to implement error correction in digital signal processors that employ standard arithmetic operations. Applications for these include: simultaneous source coding and impulse nose cancellation robust image transmission over realistic satellite channels [2], algorithmic-based fault tolerance (ABFT) for linear signal processing[3], such as Fourier transforms [4], Discrete Cosine transform [5], or digital filters [6], and distributed lossy source coding over digital communication channel [2].
A key component when using real number code as an error correction mechanism is represented by the decoding process. The simplest method is to use table based error correction. It has been employed for Fast Fourier Transforms [4] or digital filters [6], protected by the means of a parity based (7,4) Hamming code. This type of decoding is suitable for codes that can correct single errors. Increasing the error correction capability – correction of multiple errors – , the size of the tables increases exponentially. In order to tackle multiple errors, iterative methods for real number codes have been proposed, including the use of the Kalman filtering in order to estimate the noise vector in [7], [8], or using turbo decoding principle in [9].
In this paper, we propose a novel iterative decoding method, and its corresponding hardware architecture, that employ maximum likelihood decoding and gradient descent optimization. We have developed the method for small size (with codeword lengths in the order of tens of symbols) parity check equation based error correction codes. The proposed method

The paper is organized as follows: notations and the iterative gradient descent optimization method are described in Section II, Section III presents a case-study for a (26,16) parity based Bose-Chaudhuri-Hocquenghem (BCH) code [10], as well as the hardware architecture developed for it, while the last section is dedicated to the concluding remarks.
II. GRADIENT DESCENT CORRECTION
A. Notations
This work relies on parity based linear codes that are defined by a parity matrix H [11], and that can also be viewed in graph form, called Tanner graph [12]. For the binary case a code word can be represented by a vector x of size equal to the number of columns of H, denoted hereafter with N , having the elements of which belong to 0, 1, satisfying the parity equations, that is to say Hx = (0, 0, ..., 0)T . The (·)T denotes the transpose of a vector or a matrix. The result of the s = Hx operation is referred hereafter as syndrome computation, and the syndrome vector s. A code word x is obtained from an information word a, of size M , by the equation x = aG where G is the code generator matrix satisfying the equation HGT = 0. The elements of a code word are symbols. Code words are subject to noise from the channel. Let the vector y = (y1, ..., yN )T denote the channel output that is fed to the decoding unit. In the process of error correction, the maximum likelihood (ML) decoding searches for the code word x = (x1, ..., xN )T having the maximum correlation with the vector y. It is custom to regard the binary symbols in the case of hard decision decoding as bipolar, and represented by +1, −1, the bit 0 being conventionally associated with the symbol +1 and the bit 1 with the symbol −1.

The implementation use-case builds on the BCH code (26,16) [10], with the columns associated to the source symbols – C – described in Table I. The parity check matrix has the form CI10, where I10 represents the identity 10 × 10 matrix. The row degrees associated to this code range from 5 to 8, while the column degrees are 4 and 5 for the first 16 columns, and 1 for the last 10 columns. The number of syndrome vector inputs for each energy block is equal to the column degree value for each symbol value. The number of inputs of the global maximum finder block is equal to the number of symbols, e.g. 26 for the considered error correction code.

The error correction capability for the proposed method has been analyzed using Matlab floating-point models. In order to develop the hardware architecture, fixed point conversion is a must. For this purpose we used Matlab Fixed Point Converter Toolbox. This step provides a systematic way to determine the fixed point quantization. Furthermore, the fixed point converted from Matlab has been used in order to generate the DPI-C SystemVerilog functions used for the verification of the hardware architecture. For the hardware implementation, we used 16-bit quantization, with 10 bits dedicated to the fractional part.
The correction computation unit – Figure 4 – updates a symbol per iteration. At each iteration, energy is computed for each symbol, using the input codeword and the syndrome element. The energies are all computed in parallel for all symbols, according to (2). For each symbol an adder tree sumup the satisfied (+1) and unsatisfied (-1) parity checks for all 26 symbols. Note that the correction is performed if the sign agree value is greater in magnitude than a given threshold. Otherwise, the next update candidate is considered. The symbol update block computes the update value: the sign according to (4), and the magnitude using a comparator tree, which selects the minimum absolute value of all syndrome elements to which a symbol contributes.
Following the update, the parity checks are re-evaluated, consequence of the syndrome vector updates. The correction operation stops when the syndrome is close to zero – smaller than a threshold value – or the maximum allowed number of iterations is reached.

IV. CONCLUSION
In this paper we proposed an iterative method for decoding real number error correction codes, based on the gradient descent optimization techniques for nonlinear problems. Inspired from gradient descent bit-flipping [13], the proposed decoding algorithm computes energy values for all codeword symbols based on the syndrome values, selects the symbol to be updated as the the one with the highest energy, and determines the correction direction – sign of the update – and the magnitude. The method has been validated for a short parity based BCH (26,16) code having 16 useful symbol data, and 10 redundant symbol data. For this use-case, we evaluated using Matlab simulation, the error correction performance, as well as the average number of iterations. Furthermore, we propose a hardware architecture for the proposed decoding method, for which we present FPGA implementation results.
Next steps will require the analysis and extension to large parity based error correction codes (hundreds/thousands of symbols in the codeword). It is well understood that (i) small scale codes are less efficient redundancy wise (i.e. ratio between useful data symbols and redundant data symbols increases significantly with the size of the code-word); thus, we expect that by scaling up the code length, to obtain better error correction performance for higher rates; (ii) maximum likelihood decoding gradient methods, such as gradient descent bit-flip, have been designed for low-density parity check codes, a subclass of linear codes having sparse parity check matrices;
