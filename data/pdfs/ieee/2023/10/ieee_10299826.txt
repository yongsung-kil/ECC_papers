Abstract—This paper presents a memory efficient design for a Low-Density Parity-Check (LDPC) decoder. The design is based on a combination of two techniques that consists of (i) the proposed Enhanced single minimum Min-Sum (EsmMS) algorithm for check-node update processing, which requires only the first minimum value among variable-to-check input messages and its index instead of two minima, and (ii) the memory split method, which adapts to the varying check-node degree of 5G LDPC codes. The implementation results on Xilinx Kintex UltraScale+ (xcku5p) FPGA show that the proposed decoder requires 37% less storage space than MS-based decoders, provides a decoding throughput of 2.63 Gbps, while yielding a decoding gain up to 0.32 dB at the BER of 10−8 compared to the Min-Sum decoder.
Index Terms—CN processing unit, FPGA, Low-Density ParityCheck, Min-Sum, QC-LDPC.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were proposed by R. Gallager in 1962 [1], but they had been long neglected due to significant hardware demands and complex decoding. After the development of Turbo-codes [2], they were rediscovered by MacKay in 1992 [3]. These codes have gained significant attention because of their outstanding performance, lower error floor, and simple decoder design. These excellent characteristics made them suitable for a variety of applications. Actually, they are now used by several standards, including IEEE 802.16e WiMAX [4], DVB-S2 [5], IEEE 802.11 [6], ATSC 3.0 [7] and 5G New Radio (NR) [8].
The process of LDPC decoding requires repeated calculations the extrinsic messages between two node groups known as Variable-Nodes (VNs) and Check-Nodes (CNs). This task is referred to as Message-Passing (MP) decoding algorithm. At the beginning, the LDPC decoder utilizes the Belief Propagation (BP) algorithm (also known as the Sum-Product (SP) algorithm), which is based on the MP algorithm. Although the BP method delivers outstanding decoding performance close to the Shannon limit [9], it consumes significant hardware resources for implementation. The MS algorithm was introduced to simplify the computation complexity by requiring only comparison and addition operations [10]. Unfortunately,

it has been found that the MS decoder suffers a degradation in the error correction performance.
To further improve the decoding performance, several versions of the MS-based decoding algorithms such as the Normalized Min-Sum (NMS) and the Offset Min-Sum (OMS) were suggested in [11], [12]. These algorithms had a common technique that modified the check-node update processing. Although these MS-based algorithms provided better decoding performance, they introduced a higher hardware complexity than the original MS algorithm. One of the methods to enhance the hardware usage efficiency is to exploit an approximation calculation for check-node update processing. More precisely, instead of computing two minimum values among input variable-to-check messages, only the first minimum value is calculated. This approach can lead to significant savings in hardware resources. For instance, according to [13], a new approach, namely single minimum Min-Sum (smMS), was proposed. This involves incorporating a weight parameter into the first minimum value to calculate the second minimum. This method seems to be simple, but it results in an error-floor early. The smMS algorithm’s error correction performance can be further improved with the variable weight smMS (vwsmMS) algorithm, which was proposed in [14]. Instead of using a fixed coefficient as in [13], the authors in [14] proposed to use correction factors that vary corresponding to the number of iterations. Although it improved the decoding performance in the water-fall region, determining the weight parameters was quite complex. In practice, almost LDPC applications employ a version of LDPC code, named as Quasi-Cyclic LDPC (QCLDPC) [15]. Such a structure allows great parallelism in decoder implementations, which is essential for providing the high throughput demanded by modern communications. It should be noted that the 5G QC-LDPC codes exhibit significant irregularities [16]–[18]. Specifically, in the BG1 base matrix, the CN degree varies from 3 to 19, and only 4 out of the 46 rows have a degree of 19. Due to the high check node degree of 5G LDPC codes, the decoder operating frequency will be limited due to the large increase in decoder complexity. The author in [19] presented an efficient reordered layered scheduling based on the number of punctured edges

In this work, an efficient 5G LDPC decoder architecture is introduced. II. PRELIMINARIES
A. The brief concepts of LDPC codes
Let’s consider an LDPC code with the codeword length and the information bits are N and K, respectively. An LDPC code can be described by using either a sparse parity-check matrix (PCM) H of size M × N where M = N − K or a Tanner graph that introduced by Tanner [22]. In the Tanner graph representation, a codeword is divided into two groups of nodes: Variable-Nodes (VNs) and Check-Nodes (CNs). More precisely, VNs correspond to the columns and CNs correspond to the rows of the PCM H. Let’s use the notation H to represent the Tanner graph of the code. Then, H(m) represents the set of VNs connected to CN m and H(n) represents the set of CNs connected to VN n. The ith check-node (CN) is connected to the jth variable-node (VN) by an edge if the entry H(i, j) (i = 1, · · · , M ; j = 1, · · · , N ) of the parity-check matrix H equals 1. The degree of a variable-node (checknode) is the total number of adjacent edges connecting to it. The variable-node and the check-node degrees are denoted by dv and dc, respectively. An LDPC code is regular when all variable and check nodes have the same degree. Otherwise, it is irregular. It is worth noting that most LDPC codes are irregular in practical applications.
Let c = (c1, c2, ..., cN ) denote a codeword. For the sake of simplicity, we assume that the codeword c is modulated by Binary Phase Shift Keying (BPSK). Next, the modulated information x = (x1, x2, · · · , xN ) where xn = 1 − 2cn (n ∈ [1, N ]) is passed through an Additive White Gaussian Noise (AWGN) channel. At the receiver, the received signal can be given by y = (y1, y2, · · · , yN ) where yn = xn + zn and zn is an independent Gaussian random variable with zero mean and noise variance σ2.
As mentioned before, the specific type of LDPC code used in practical applications is Quasi-Cyclic LDPC (QC-LDPC).

QC-LDPC codes typically include a base matrix B with entries bi,j where i and j are the row and column indices of the entry. The base matrix B is extended by a circulant matrix of size Z × Z (where Z is the expansion factor) for each entry to form the PCM H. The extended matrices satisfy the following convention: bi,j = −1 is replaced by an all-zero matrix; bi,j ≥ 0 is used in place with a circulant matrix, which is formed by right-shifting the identity matrix bi,j positions.
In order to offer rate-compatible and scalable data transmission, in 5G two QC-LDPC base matrices BG1 and BG2 are employed [8]. These matrices have the same structure. BG1 is compatible with long codeword and high code rates, while BG2 is suitable for shorter codeword and lower code rates. In this work, we focus on the BG1 matrix. The structure of the BG1 is depicted in Fig.1. As we can see that the BG1 matrix is divided into four parts: the core, which consists of the submatrices A and B, and the extensions, which include the E, O and I (O is the all-zero matrix and I is the identity matrix). There are three types of base matrix columns: information columns, core parity columns, and extension parity columns. The 5G LDPC codes are extremely irregular [8]. The distribution of check node degrees for BG1 and BG2 is listed in Table I. It can be seen that the check-node degree of BG1 varies from 3 to 19. In particular, only 4 rows have a degree of 19, while the check-node degree is less than 11 in the remaining rows.

decoder architecture. We implement a 5G LDPC code using

The overall architecture of the layered LDPC decoder is shown in Fig. 2. Several main functional blocks are described below.
To store the information, the decoder uses two memory blocks: AP Memory Block (AP-MB) and Check-Node Memory Block (CN-MB). In AP-MB, the decoder input signal γn or the γ˜n messages is stored depending on the decoding processing. Note that the decoder input from the channel is stored only in the initialization step. Thus, MUX is used to select between the priori information γn and a posteriori information update γ˜n messages. In CN-MB, the CTV messages are stored for the next layer processing. The variable-nodes (VNs) and check-nodes (CNs) processing, as described in Eqs. (3) and (4b), are carried out by Variable-Node Units (VNUs) and Check-Node Units (CNUs), respectively. The APB executes the AP-update processing, which is given in Eq. (5). It should be noticed that the varible-to-check (VTC) messages before feeding to the CNU need to saturate from q˜-bits to q-bits by SAT blocks. In this design, the CNU output messages are stored into CN-MB in compressed format for memory saving purposes. Therefore, the DECOM blocks use to convert from compressed to uncompressed messages.
The connection units are the last set of blocks. They consist of the Read Permutation Units (Read-PUs), which are responsible for arranging the signals in a way that allows the VNUs or CNUs to process them correctly. The Read-Shifter Blocks (R-SBs) have the capability to organize input signals in any order at the output, according to the non-negative elements of the BG1 matrix. In comparison to the ReadPermutation Units (Read-PUs) and the Read-Shifter Blocks (R-SBs), the Write-Permutation Units (Write-PUs) and WriteShifter Blocks (W-SBs) perform a reverse function. Finally, to ensure that the decoder can work properly, it’s essential to implement the controller block. This includes executing blocks

III. THE PROPOSED CNU ARCHITECTURE AND CHECK-TO-VARIABLE MEMORY FOR 5G LDPC CODES

A. The proposed Enhanced single minimum Min-Sum (EsmMS) algorithm

Inspired by the smMS algorithm [13], we propose a modified algorithm that aims to provide further decoding gain, as well as memory efficiency. For the Check-Nodes (CNs) processing, we apply the same idea as [13]. It means the CNU just finds the accurate value for first minimum (i.e., min1) and its index (i.e., index min1), while the second minimum (i.e., min2) is approximated based on the min1 value and modified factor ω (also called as a weight parameter) as given in Eq. Although the smMS algorithm could achieve high hardware implementation efficiency, the error-correction capacity was significantly decreased (error-floor appeared quite early). To improve the decoding performance, we propose to apply another offset factor to the Variable-Nodes (VNs) processing as follows:

αm,n = γ˜n − sgn(βm,n) · max(|βm,n| − δ, 0) (7) where δ > 0 is the offset factor.

The decoder which applies (6) and (7) will be called as the Enhanced single minimum MS (EsmMS) algorithm. In the proposed EsmMS algorithm, the optimization of ω and δ factors is obtained by simulation methods under the particular LDPC code. Without losing generality, ω is optimized after δ is set to 0.5. The best ω value in this scenario is received. The δ factor is then adjusted through simulation using the value of ω determined in the previous stage to reach an expected target Bit-Error-Rate (BER) threshold. In our case study, the optimal values for the EsmMS algorithm are found by (ω, δ = 0.25, 0.75) for BG1 matrix with the codeword length N of 8832, the code rate of 1/2 and the target BER of 10−8.
B. The proposed CNU architecture
Figure 3 depicts CNU architectures of the conventional MS and the proposed EsmMS 5G LDPC decoders. As previously mentioned, the 5G LDPC code is irregular, which means that the number of CNU inputs is implemented to be equal to the maximum check-node degree, represented as dcmax. In this work, dcmax equals 19. Hence, the CNU in both decoders has dcmax = 19 inputs. The “set” signal of 2-to-1 multiplexers is used to deactivate the unnecessary inputs in case the checknode degree is less than dcmax.

The Min & Index Finder component inside the proposed CNU is used to find the smallest values (i.e., min1) and its index (i.e., index min1) among all the VTC messages. This is done by using the Tree Structure (TS) architecture as described in [23]. The CNU output message signs can be determined by utilizing XOR the input message signs. The CNU block generates output messages in a compressed format, known as compressed β-messages, each of them consists of min1 of (q − 1)-bits, index min1 of ⌈log2(dcmax)⌉bits and signs of dcmax-bits. Therefore, the size of each βmessage is (q − 1) + ⌈log2(dcmax)⌉ + dcmax (bits). It’s worth noting that the size of each conventional β-message will be 2 · (q − 1) + ⌈log2(dcmax)⌉ + dcmax (bits).

As shown in Table I, it should be emphasized that the dcmax value of 19 is only utilized in the first four layers, while the remaining layers operate with a check-node degree is less than or equal to 10. As a result, in case we implement the same width for all CN-MBs, i.e., corresponding to dcmax = 19 for all CNUs (see Eq. (10)), this will lead to a substantial waste of memory resources. This motivates us to propose the memory split method, which is flexible with the change of check-node degree. More precisely, the original CN-MB is divided into two sub-memory blocks, named as CN-MB1 and CN-MB2. The CTV messages are stored in CN-MB1 only for the first four layers (i.e., L1 = 4), corresponding to checknode degree of dc = dcmax1 = 19, while the rest of the layers (i.e., L2 = 20) are stored in CN-MB2, which correspond to check-node degree dc ≤ dcmax2 = 10. V. CONCLUSION
This paper introduced an effective technique for optimizing memory usage in 5G LDPC decoders. This technique was designed to take into account the unique characteristics of 5G LDPC codes, where the degree of the check-node varies greatly. To prevent unnecessary waste of memory, the stored memory is divided into two sub-memories based on the checknode degree values. Moreover, the check-node processing in the proposed algorithm only involves calculating the first minimum value and its index instead of the two minima. These resulting messages are stored in a compressed format, which also helps to save on storage space. As a result, the CN-MB size of the proposed decoder is lower than that of the MS and MS-based decoders up to 37%.
