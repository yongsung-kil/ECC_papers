Trapping and Absorbing Set Enumerators for Irregular Generalized Low-Density Parity-Check Code Ensembles
Emna Ben Yacoub , Graduate Student Member, IEEE

Abstract— The definitions of (elementary) trapping and (fully) absorbing sets are extended to generalized low-density paritycheck (GLDPC) codes. In particular, the fully absorbing sets are stable under the bit flipping algorithm. The finite-length (elementary) trapping and (fully) absorbing set enumerators for unstructured irregular GLDPC code ensembles are derived using generating functions. An efficient method to evaluate the asymptotic trapping and (fully) absorbing set distributions is presented, which requires solving a system of equations. Using this method, the asymptotic distribution of trapping and (fully) absorbing sets are evaluated for some example GLDPC code ensembles. Experimental results show that the proposed definitions yield graph structures that are harmful for bit flipping decoders. The generating function approach is also used to derive the finitelength and asymptotic trapping and (elementary) absorbing set enumerators for non-binary irregular LDPC code ensembles.
Index Terms— Trapping sets, absorbing sets, irregular generalized LDPC codes, asymptotic enumeration, bit flipping, nonbinary LDPC.
I. INTRODUCTION
LDPC codes exhibit excellent performance under iterative decoding, but they may have high error floors. This phenomenon can be explained by the presence of special graphical structures in the code Tanner graphs. Depending on the channel model, these harmful configurations are called stopping [1], trapping [2], [3] and (fully) absorbing sets [4], [5]. In fact, these structures may cause decoding failures under iterative decoding [6], [7], [8]. Hence, they play a fundamental role in the error floor performance of lowdensity parity-check (LDPC) codes. Notably, the errors in a fully absorbing set (FAS) cannot be corrected by the bit flipping algorithm. A more general class of LDPC codes called generalized low-density parity-check (GLDPC) codes was introduced by Tanner in [9]. The codes associated to the constraint nodes (CNs) of a GLDPC code are more general than single parity-check (SPC) codes, i.e., they can be any linear block code that is referred to as a component code.

Examples of GLDPC codes are given in [10], [11], [12], and [13]. The authors of [14] proposed to construct GLDPC codes by replacing SPC nodes involved in certain trapping sets (TSs) [3] by a linear block code that may eliminate the TSs.
The analysis of TSs and absorbing sets (ASs) of LDPC codes has been addressed from combinatorial [8], [15], [16], [17], [18], numerical [3], [19] and code ensemble perspectives [4], [20], [21], [22]. In particular, the asymptotic TS and AS enumerators for irregular LDPC ensembles were derived in [20], [21], [22], [23]. The asymptotic distribution of TSs and ASs for protograph-based LDPC ensembles was developed in [24], [25], and [26].
The problem of TSs and ASs in LDPC code ensembles has been addressed in a few works. In [20], asymptotic enumerators for TSs, elementary trapping sets (ETSs) and stopping sets (SSs) have been obtained for irregular LDPC code ensembles. The approach was extended in [4] and [22] to address ASs for regular and irregular LDPC code ensembles, respectively. The elegant method employed in [4] and [20] relies on random matrix enumeration techniques to study the distribution of the row/column weights in sub-matrices of the code parity-check matrix, and on large deviation theory to derive the asymptotic enumerators. In [8], an algebraic method is used to analyze the multiplicity of minimal FASs in array LDPC codes, while [23] introduces a combinatorial asymptotic analysis of several graphical structures (including ETSs, ASs and elementary absorbing sets (EASs)) of constant size with respect to the block length. In the context of GLDPC code ensembles, [25] provides both a finite-length and an asymptotic enumeration of codeword weights, SSs and pseudo-weights for protograph-based GLDPC code ensembles, whereas the TS enumeration problem is addressed only for protograph-based LDPC codes.
The understanding of TS/AS phenomena affecting GLDPC code decoders is much less mature than in the LDPC code case. In the past few years some attention has been devoted to the analysis of the SS distribution of GLDPC code ensembles. In particular, the definition of a SS was extended to GLDPC codes in [27], while the authors of [28] derived the asymptotic distributions of generalized SSs for unstructured GLDPC and doubly-generalized low-density parity-check (D-GLDPC) code ensembles. For TSs and (fully) ASs, a general definition is yet to be agreed upon. The work in [29] addresses the failures of some classes of hard-decision GLDPC decoders in terms of fixed sets (i.e., sets of corrupt variable nodes (VNs)

To the best of our knowledge, the technique pioneered in [4] and [20] for the TS/AS analysis of LDPC codes cannot be trivially extended to the analysis of GLDPC codes. For this reason, we adopt a generating function approach that has been used to address the distance spectrum and the stopping set enumeration analysis of LDPC and GLDPC codes (see, e.g., [28], [31], [32], [33]).
In this work, we propose new definitions of a TS, AS and FAS for GLDPC codes. The proposed definitions stem from an analogy with the definitions employed in the LDPC code case, and they are based on a reference PBF decoder. We consider the PBF algorithm since it is a low-complexity algorithm suitable for high-throughput applications. In particular, hard decision decoders for GLDPC-like codes (e.g., product codes and staircase codes) with bounded distance decoding at the CNs is currently considered as a baseline approach in the context of very high speed fiber-optic communications, as discussed, for instance, in [34] and [35]. We start from a broad definition of TSs, which includes sets that may be resolved by the PBF algorithm. The definition is then sharpened, yielding a simple definition of ASs and FASs. The latter sets cannot be corrected by the PBF algorithm. For the case where all CNs are SPC codes, we obtain the definitions of trapping, (fully) absorbing sets of binary LDPC [2], [3], [4]. For non-binary LDPC codes, the definitions of trapping and absorbing sets were introduced in [36], [37],and [38]. The authors of [37] and [38] derived the asymptotic distribution of non-binary EASs for regular code ensembles. The method in [37] uses techniques from graph theory on the VN graph representation of EASs, where the VNs are the vertices and the degreetwo CNs connecting the VNs are represented by edges. The approach relies on the fact that the topological configuration of non-binary EASs is also a binary EAS and the edge labels should satisfy some constraints. Therefore, by computing the fraction of weight assignments that lead to a non-binary EAS, we can obtain the normalized logarithmic asymptotic distribution of non-binary EASs by using the normalized logarithmic asymptotic distribution of binary EASs. Since for non-binary trapping/absorbing sets, the underlying configuration may or may not be a binary trapping/absorbing sets, this method cannot be applied for these sets. In Appendix E, we use generating functions to provide a simple derivation of the distribution of trapping and (elementary) absorbing sets for non-binary irregular LDPC code ensembles trying to create a bridge with the derivations for the GLDPC code case.

B. Parallel Bit Flipping Algorithm
We review a hard decision decoding algorithm for GLDPC codes. To this end, we recall first the BDD. Consider a linear code P with minimum distance dmin. BDD can correct all error patterns of Hamming weight t = ⌊(dmin − 1)/2⌋ or less. For a received sequence z, the decoded vector cˆ is given by

where dH(c, z) is the Hamming distance between c and z. The PBF decoding [29] is similar to the algorithm proposed
in [39] and [40] for expander codes. It is closely related to the one introduced in [35] for the decoding of a class of spatiallycoupled GLDPC codes, and to the one used in [27] to decode GLDPC codes based on Reed-Solomon and Bose-ChaudhuriHocquengham (BCH) component codes, and it tightly follows the formulation of bit flipping decoding of GLDPC codes outlined in [9] and [29].
For the PBF decoder, we consider transmission of the all-zero codeword over a binary symmetric channel (BSC). We denote by cˆ(vℓ) the estimate of the codeword bit associated with the VN v at the ℓ-th iteration. Initially, the estimate of a VN is its channel observation, i.e., cˆ(v0) = mvch. At the ℓ-th iteration, each VN sends its estimate cˆ(vℓ) to its neighboring CNs. Each super CN performs BDD on the received messages from the VNs and sends then flip messages to the VNs whose values differ from the decoded vector. Special care is needed for SPC component codes: Here, we follow the policy of flipping the component code decoder input at the output whenever the SPC constraint is not satisfied, i.e., an unsatisfied single parity check node sends flip messages to all its neighboring VNs. For a VN v of degree dv, if it receives strictly more than dv/2 flip messages, it flips its estimate, i.e., cˆ(vℓ+1) = 1 − cˆ(vℓ). A VN is called correct if the corresponding estimate is zero and it is called corrupt if it is one. The VNs and CNs exchange messages iteratively until a maximum number of iterations is reached or a valid codeword is found.

C. Trapping and (Fully) Absorbing Sets
We assign the value 1 to each VN in a set I and 0 to the VNs outside I. We denote by N (I) the set of the neighboring CNs of I. Further, we denote by U(I) the set of unsatisfied CNs in N (I) and S(I) the set of satisfied CNs in N (I). A CN in N (I) is satisfied if it recognizes a valid local codeword when the edges connected to I are assigned the value 1 and the other edges the value 0. In the following, we propose definitions of TSs and (fully) ASs for GLDPC codes that generalize the ones of LDPC codes [4], [8], [20].
Definition 1: An (a, b) TS Ta,b is a set I of a VNs such that U(I) contains b CNs. It was observed in [3] that the error floor performance is dominated by small TSs where CNs are connected to only one or two VNs. These sets were referred to as ETS. Analogously, we propose a definition of ETSs for GLDPC codes.
Definition 2: An ETS TaE,b is a TS where ∀τ ∈ {1, 2, . . . , nc} each CN of type τ in S(I) is connected to dmin,τ VNs in I and each CN in U (I) is connected to exactly one VN in I. An illustration of a (3, 1) ETS of a simple GLDPC code is given in Fig. 1. We have Λ = [0, 6/7, 1/7], P = [2/3, 1/3], the CNs c1, c3, of type 1, are (4, 3) SPC codes and c2 is a (7, 4) Hamming code. We assign the value 1 to the set I = {v3, v5, v7}. Note that c(1) = [0, 0, 1, 0] is not a valid codeword of the SPC code. However, c(2) = [0, 0, 1, 0, 1, 0, 1] and c(3) = [0, 1, 0, 1] are valid codewords of the Hamming and SPC codes, respectively. Thus, U(I) = {c1} and S(I) = {c2, c3}. Moreover, c3, which is of type 1, is connected to dmin,1 = 2 VNs in I and c2, of type 2, is connected to dmin,2 = 3 VNs in I. The unsatisfied CN c1 is connected to one VN in I. Since |I| = 3 and |U(I)| = 1, the ETS has size (3, 1).

The connection between (E)TS and decoding failures is hard
to analyze, in general, and it requires (often) resorting to
numerical methods to obtain accurate performance predictions
(see, e.g., [3], [19], [20]). The merit of the definitions above
is to extend in a natural way the ones that are commonly
accepted for LDPC codes, and to yield to the following stricter
definitions of AS and FAS. The latter, in particular, provides
a rigorous description of combinatorial structures that lead the
PBF to fail. Definition 3: An (a, b) AS Aa,b is a TS with the additional
property that for each VN v ∈ I, we have n(vf) < dv/2. Definition 4: An (a, b) FAS Fa,b is a TS with the additional
property that for each VN v in the Tanner graph n(vf) < dv/2. Remark 1: Consider a set I of corrupt VNs, where each
VN v in the Tanner graph receives nv(f) < dv/2 flip messages. Since for the PBF algorithm, each VN flips its estimate only if nv(f) > dv/2, no VN will flip its estimate. Thus, according to Definition 4, the VNs in a FAS cannot be corrected by the
PBF algorithm. Moreover, an EAS AEa,b (elementary fully absorbing set
(EFAS) FaE,b) is an AS (FAS) where ∀τ ∈ {1, 2 . . . , nc} each CN of type τ in S(I) is connected to dmin,τ VNs in I and each CN in U(I) is connected to exactly one VN in I.

VI. EXPERIMENTAL RESULTS
A. GLDPC Codes
In this section, we present experimental results to check the effect of the trapping and (fully) absorbing sets on the performance of GLDPC codes. We consider a (155, 80) regular GLDPC code with VN degree 3 and the (31, 26) Hamming code as component code. We transmit the all-zero codeword over a binary-input additive white Gaussian noise (biAWGN) channel. We perform a hard decision on the received sequence, thereby the biAWGN channel is converted to a BSC, and apply the PBF algorithm. We set the maximum number of iterations ℓmax = 20.
A first set of results deals with the error profiles obtained with the PBF algorithm. Since the all-zero codeword is transmitted, we say that a VN is corrupt after decoding if its corresponding final estimate is one. For each channel realization leading to a decoding failure, we check if the subgraph containing the corrupted VNs and their neighboring CNs is a trapping or (fully) absorbing set. In this case, we determine its size. We collected 1000 error frames at signal-to-noise ratios (SNRs) Eb/N0 ∈ {8.6, 9, 9.4}. Table II shows the obtained error profiles, which are converging errors, i.e., the VN estimates remain the same in the last few iterations. All the errors provided in Table II are FASs and thus trapping and absorbing sets. This confirms the stability of FAS under the parallel bit flipping decoder. In this example, all decoding failures are caused by FASs.
