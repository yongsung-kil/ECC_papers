Abstract‚ÄĒBelief propagation (BP) decoding of polar codes, which can be executed in parallel, is suitable for low-latency applications. In this paper, we first give a sparse parity-check matrix representation for polar codes, based on which several strategies for performance improvement are proposed. We then devise a scheme of which the performance can approach the maximum-likelihood lower bound and that of successive cancellation list (SCL) decoding. For cyclic redundancy check (CRC)concatenated polar codes, a parity-check matrix representation is also presented so that joint decoding between the CRC code and polar code is possible. An improved CRC-aided BP decoder is hence proposed with performance close to that of the state-ofthe-art CRC-aided SCL decoder.
I. INTRODUCTION
Polar codes [1] can be proved to asymptotically achieve the capacity for symmetric binary-input discrete memoryless channels. Successive cancellation (SC) decoding was presented in [1] to prove the capacity-achieving property, but the finite-length performance is not as expected caused by decision error propagation. To deal with the problem, successive cancellation list (SCL) decoding [2] and cyclic redundancy check (CRC)-aided SCL (CA-SCL) decoding [3] were later proposed. CA-SCL decoding represents by far the best decoding performance of polar codes. However, the main drawback of SC/SCL-type decoding is the long decoding latency due to the serial nature.
Belief propagation (BP) is another type of decoding algorithms for polar codes, which can be executed in parallel, suitable for low-latency-demanding applications. Nonetheless, the performance of the conventional BP algorithm in [4] is worse than that of SCL decoding. There have been quite a number of approaches proposed to improve the performance of BP decoding of polar codes. Belief propagation list (BPL) decoding was proposed in [5], where list decoding was made possible by using permuted factor graphs introduced in [6] and further studied in [7]. A similar approach was also proposed in [8]. The performance of the BPL decoder in [5] was shown to be close to that of the SCL decoder. In [9], BP decoding was conducted over the Tanner graphs for polar codes similar to that for low-density parity-check (LDPC) codes [10] instead of the conventional factor graphs; yet the performance is slightly

inferior to that of the conventional BP decoding. This approach was further improved in [11] with dynamic scheduling, but the performance is still worse than that of SCL decoding. CRC-aided belief propagation list (CABPL) decoding was proposed in [12], where error correction of the CRC code was considered for the CRC-concatenated polar code of length 128; the scheme was further enhanced in [13] for codes of lengths 128 and 256. For code length 1024, the performance of the enhanced BP decoder in [14] is not close to that of the state-ofthe-art CA-SCL decoder. Therefore, there is still much room for performance improvement of BP decoding of polar codes.
In this paper, we first give a sparse parity-check matrix representation for polar codes, based on which BP decoding scheduling polices can be considered. One specific BP decoding scheduling is then identified that is equivalent to the conventional BP decoding over the factor graph. We further propose several strategies and devise a scheme of which the performance can approach the maximum-likelihood (ML) lower bound and that of SCL decoding. As for CRCconcatenated polar codes, a parity-check matrix representation is also presented so that joint decoding between the CRC code and polar code is possible. We hence propose an improved CRC-aided BP decoder with performance close to that of the CA-SCL decoder.
The remainder of the paper is organized as follows. Section II reviews the encoding and conventional BP decoding of polar codes. In Section III, we describe our BP decoding scheme for polar codes. Then we proceed to present the improved BP decoder for CRC-concatenated polar codes. Section IV provides simulation results and Section V concludes the paper.

1) Message Sharing: We consider using the CRC code for error correction not just for error detection. The joint BP decoding follows the rule that the CRC part is updated immediately after the right propagation and before the left propagation. There are two main reasons. One is that after right propagation, the frozen bit messages are passed and thus the CRC part may take less wrong frozen bit locations. The other is that the error rate of the CRC code is much higher than that of the polar code, so we make the CRC part updated fewer times.
However, two categories of 4-cycles are observed. The first is the 4-cycles inside the CRC part of Hňú where two check nodes of őďi are involved in each cycle. The second category is the 4-cycles generated after concatenation of the polar code and CRC code where one check node of the CRC code and one check node of the polar code are involved, which causes the severest effect at the first and last stages since the updating rule results in the immediate bounced-back messages at those stages. To deal with the first category of 4-cycles, we adopt the method in [14], where every time only one check node of őďi is updated. Regarding the second category of 4-cycles, we disregard őď1 and őďn+1 for the CRC code during message update.

For CRC-concatenated polar codes, from Fig. 3, the proposed decoder outperforms the CABPL decoding in [12] of list size l = 32 with only 8 lists and the performance can approach both the ML lower bound and that of SCL decoding in [2] of list size L = 32 with only 16 lists for code length 128. From Fig. 4 for code length 256, the proposed decoder has similar performance to the CABPL decoding in [13] under the same list size. For code length 1024 in Fig. 5, the performance of the proposed decoder with list size l = 32 is close to that of the state-of-the-art CASCL decoding in [3] of list size L = 8. As shown in Fig. 6, the average number of iterations for the proposed decoders are much less than 200 and can be further reduced to 10 or below when the signal-to-noise ratio increases. Moreover, enlarging the list size would not increase the average number of iterations since the decoders can be parallelly implemented.

V. CONCLUSION In this paper, we propose two improved BP decoders for polar codes and CRC-concatenated polar codes. The performance of the improved BP decoder for polar codes approaches not only the ML bound but also that of the SCL decoder of list size 32 with only 10 lists without sacrifice of the parallel structure. The improved BP decoder for CRC-concatenated polar codes has the performance close to the CA-SCL decoder with list size 8 for code length 1024 and outperforms other BP decoders considered. Both improved decoders are made possible by considering the sparse parity-check matrix representations which provide flexible message update strategies and additional decoding capabilities.
