Quasi-Cyclic LDPC Codes With Girth at Least Eight Based on Disjoint Difference Sets
Guohua Zhang , Mengdi Ni, Yulin Hu , Senior Member, IEEE, and Yang Yang

Abstract— A novel relationship between disjoint difference sets (DDS) and QC-LDPC codes without cycles of length up to six is established. Based on DDS with two sets, a class of columnweight-four QC-LDPC codes is constructed with girth at least eight. In the literature there are several explicit or semi-random methods which can yield column-weight-four QC-LDPC codes without 4-cycles and 6-cycles. Compared with these existing methods, the new DDS-based method is able to offer QC-LDPC codes with desirable decoding performance and flexible circulant sizes, and at the same time able to provide a large number of nonequivalent candidate codes. Moreover, by employing DDS with only one set, a class of column-weight-three QC-LDPC codes is constructed with short lengths and with girth at least eight.
Index Terms— Cycle, disjoint difference sets (DDS), lowdensity parity-check (LDPC) code, quasi-cyclic (QC).
I. INTRODUCTION
A S AN important class of low-density parity-check (LDPC) codes, quasi-cyclic (QC) LDPC codes have attracted great attention from both academic research and engineering practice, because they exhibit good decoding performance and facilitate hardware implementations [1], [2], [3], [4]. It is generally believed that eliminating short cycles in the Tanner graphs of such codes is an effective way to improve performance [1], [3], [5], [6], [7], [8]. Efforts along this direction can be broadly divided into two categories: (semi-) random methods based on search and explicit methods via formulas. By taking full advantage of a couple of special objects in combinatorics, the process of constructing QC-LDPC codes without small cycles can be achieved without search or only with a small amount of search. For QC-LDPC codes without 4-cycles, the concepts of a difference set [9], a difference family [10] and disjoint difference sets (DDS) [11], [12] have found application in building codes with flexible column weights. As for QC-LDPC codes without 4-cycles and 6-cycles, a Sidon sequence has been utilized to construct column-weight-three codes [13] and column-weightfour codes [1]. In this letter, by exploiting DDS we propose a novel construction for generating column-weight-four codes without cycles of length up to six, which remarkably extends

The greatest-common-divisor (GCD) method [7] and ZSW method [14] (we follow this name adopted in [1] for consistency) are two typical explicit schemes to construct column-weight-four codes without 4-cycles and 6cycles. Compared with them, the new method is able to offer codes with significantly better performance. In comparison to the recently presented symmetrical (SYM) method [15] which can yield column-weight-four codes in a semi-random fashion, the new construction is able to produce codes with almost identical performance, and at the same time has the benefit of generating a large number of nonequivalent codes simply by permuting the elements in one set of DDS. II. PRELIMINARIES
A (J, L)-regular LDPC code is the null space of a sparse parity-check matrix (PCM) with J ones in each column and L ones in each row. The PCM of a QC-LDPC code is an array of circulants with size P × P . The circulant considered in this letter is a binary P × P matrix defined by an integer z (referred to as an exponent), where the first row is composed of zeros except the z (mod P )-th position, and each other row is obtained by cyclically shifting its previous row to the right by one position. As a result, the PCM of a (J, L)-regular QC-LDPC code can be uniquely determined by an exponent matrix E with size J ×L and a circulant size P . Cycles within the Tanner graph of a QC-LDPC code can be detected by using the corresponding exponent matrix and circulant size. Cycles of length n are denoted by n-cycles, and girth is the length of the shortest cycles.
Let di be a set consisting of k integers, {di(0), di(1), · · · , di(k − 1)}, where 1 ≤ i ≤ t. A (v, k, t)-DDS [11] is a collection of t sets {d1, d2, · · · , dt} such that all the tk(k − 1) differences {di(r) − di(s)|1 ≤ i ≤ t, 0 ≤ r = s ≤ k − 1} are different modulo v. The concept of DDS is closely related to difference triangle sets (DTS) [16]. To be specific, they are almost the

DDS can be algebraically constructed [11], [16] or randomly generated [17]. According to the definition of DDS, the following property is obvious.
Property 1: (i) all elements within each set of DDS are distinct (modulo v); (ii) all differences among elements within a set are distinct (modulo v); (iii) the difference between any two elements in one set of DDS, is not identical (modulo v) to that between any two elements in another set.
As there are in total of tk(k − 1) distinct non-zero differences in DDS, v must be larger than this value.
Property 2: For a (v, k, t)-DDS, v ≥ tk(k − 1) + 1. It should be pointed out that this bound is tight for k = 5 and t = 2, as d1 = {0, 2, 9, 30, 38} and d2 = {0, 16, 22, 26, 40} form a (41, 5, 2)-DDS. Let x and y be two integers. Let gcd(α, v) be the greatest common divisor between two integers α and v. Obviously, if x = y (mod v) and gcd(α, v) = 1, then αx = αy (mod v). Therefore, the following property is self-evident. Property 3: Suppose that {d1, d2, · · · , dt} is a (v, k, t)-DDS. Let α be an integer satisfying gcd(α, v) = 1, and let [x1, x2, · · · , xt] be an arbitrary vector of integers. Then, {αd1 + x1, αd2 + x2, · · · , αdt + xt} is also a (v, k, t)-DDS. Property 2 and Property 3 are very useful in searching for DDS with a small v for given values of k and t. When a (v, k, t)-DDS is used to construct QC-LDPC codes in this letter, the circulant size equals v and the row weight equals k.
III. QC-LDPC CODES FROM DDS: CONSTRUCTION I
In this letter, we focus on DDS with t = 2, as our first new construction is based on such DDS. Before expounding the novel method, we first present two properties on DDS, which are beneficial for building new codes with flexible parameters.
A. New Properties of DDS
Property 4: If {d1, d2, · · · , dt} is a (v, k, t)-DDS, then it is also a (v , k, t)-DDS, where v > 2max{max(di) − min(di)|1 ≤ i ≤ t}.
Proof: Since {d1, d2, · · · , dt} is a (v, k, t)-DDS, all the differences {di(r) − di(s)|1 ≤ i ≤ t, 0 ≤ r = s ≤ k − 1} are different modulo v. Therefore, all the differences are distinct (with no modulo operation). Let m = max{max(di) − min(di)|1 ≤ i ≤ t}. Because −m ≤ di(r) − di(s) ≤ m, all the differences di(r) − di(s) are distinct modulo v , as long as v ≥ 2m + 1.
Example 1: Set d1 = {0, 6, 11, 18, 28, 37, 62, 70} and d2 = {0, 1, 4, 24, 40, 54, 67, 69}, which are two sets used for DTS in [18]. It is readily verify that {d1, d2} turns out to be a (141, 8, 2)-DDS. Owing to Property 4, it is also a (v , 8, 2)-DDS for each v ≥ 141. Four novel QC-LDPC codes without 4-cycles and 6-cycles are compared (in Section IV-C) via the (v , 8, 2)-DDS for v = 150, 180, 210 and 240.
For a Sidon sequence, the notation Zv means that the addition of any two elements in a Sidon sequence is defined as their addition modulo v.

Property 5: Let s1 and s2 be two length-k subsets of a Sidon sequence over Zv such that they have at most one common element. Then s1 and s2 form a (v, k, t = 2)-DDS.
Proof: Take any two elements from s1, denoted by a and b (a = b (mod v)), and take any two elements from s2, denoted by c and d (c = d (mod v)). If a−b = c−d (mod v), then a+d = c+b (mod v). Since (i) a, b, c and d are entries of a Sidon sequence, (ii) a = b (mod v) and (iii) c = d (mod v), it follows that a = c (mod v) and b = d (mod v). It is impossible, because s1 and s2 have at most one common element modulo v. As a result, a − b = c − d (mod v).
It should be pointed out that according the definitions of DDS and a Sidon sequence [13], it is easy to prove that DDS containing only one set is equivalent to a Sidon sequence.

B. New QC-LDPC Codes Based on DDS Our new construction is based on a (P, L, 2)-DDS, which is
composed of two sets, d1 and d2. Define each element in the exponent matrix E by e(0, r) = 0, e(1, r) = d1(r), e(2, r) = d2(r) and e(3, r) = d1(r) + d2(r) for 0 ≤ r ≤ L − 1.
Theorem 1: The exponent matrix E corresponds to a (4, L)-regular QC-LDPC code whose Tanner graph is free of 4-cycles and 6-cycles for the circulant size P .
Proof: First, we consider 4-cycles. Obviously, 4-cycles exist in two rows and two columns. Denote the two rows by i-th and j-th rows, and denote the two columns by r-th and s-th columns. Then such cycles can be represented by (i, j; r, s)-4-cycles. There are six cases.
Case (i) (0, 1; r, s)-4-cycles (Fig. 1(a)): If such cycles exist, then e(0, r) − e(1, r) + e(1, s) − e(0, s) = 0 (mod P ) holds. That is to say, 0 − d1(r) + d1(s) − 0 = 0 (mod P ), which reduces to d1(r) = d1(s) (mod P ). Owing to Property 1(i), it is impossible. Case (ii) (0, 2; r, s)-4-cycles: The proof for the nonexistence of this case is similar

Case (vi) (2, 3; r, s)-4-cycles: The proof for the nonexistence of this case is similar to Case (v).
Next, consider 6-cycles. Obviously, 6-cycles only exist in three rows and three columns. Denote the three rows by i-th, j-th and k-th rows, and denote the three columns by r-th, s-th and t-th columns. Then such cycles can be represented by (i, j, k; r, s, t)-6-cycles. There are four cases.
Case (i) (0, 1, 2; r, s, t)-6-cycles (Fig. 1(b)): If there are such cycles, then e(0, s) − e(1, s) + e(1, r) − e(2, r) + e(2, t) − e(0, t) = 0 (mod P ). It follows that e(1, r) − e(1, s) = e(2, r) − e(2, t) (mod P ), which is d1(r) − d1(s) = d2(r) − d2(t) (mod P ); however, it is impossible due to Property 1(iii). Case (ii) (0, 1, 3; r, s, t)-6-cycles: If such cycles occur, then e(0, s) − e(1, s) + e(1, r) − e(3, r) + e(3, t) − e(0, t) = 0 (mod P ), which reduces to e(1, t) − e(1, s) = e(2, r) − e(2, t) (mod P ). It is equivalent to d1(t) − d1(s) = d2(r) − d2(t) (mod P ), which is impossible due to Property 1(iii). Case (iii) (0, 2, 3; r, s, t)-6-cycles: The proof of the nonexistence is similar to Case (ii). Case (iv) (1, 2, 3; r, s, t)-6-cycles: If there are such cycles, then e(1, s) − e(2, s) + e(2, r) − e(3, r) + e(3, t) − e(1, t) = 0 (mod P ), which reduces to e(1, s) − e(1, r) = e(2, s) − e(2, t) (mod P ). It follows that d1(s) − d1(r) = d2(s) − d2(t) (mod P ); however, it cannot hold true due to Property 1(iii).

but Property 1(ii) does not play a role. Nevertheless,

Remark 1: Our new construction includes two special cases.
If the DDS is obtained via Property 5 where the number
of common elements is zero, the novel construction reduces
to Sidon method [1]. If the number of common elements
is one in Property 5, the proposed construction becomes a
minor modification of Sidon method (denoted by ‘Sidon-M’
for short). The new construction in the setting where DDS
is not necessarily generated from Property 5 is referred to as
DDS method for clarity.
Remark 2: DDS method offers more flexible circulant sizes
compared with Sidon method and Sidon-M method. To yield a (4, L)-regular QC-LDPC code, Sidon method needs a Sidon

sequence with 2L elements, while Sidon-M method only requires a Sidon sequence with 2L − 1 elements owing to Property 5. Therefore, Sidon-M method can choose a noticeably smaller circulant size than Sidon method. Furthermore, due to Property 2, the circulant size for DDS method should be greater than 2L(L − 1), which is a much smaller threshold than those of Sidon method (2L(2L−1)) and Sidon-M method ((2L − 1)(2L − 2)). For example, when L = 10, the circulant sizes are required to be larger than 380, 342 and 180 for Sidon method, Sidon-M method and DDS method, respectively.
IV. QC-LDPC CODES FROM DDS: CONSTRUCTION II
In this section, we propose our second new construction. This method is based on a (P, L, 1)-DDS, which includes only one set, d1. Let a, b and c be three integers. Define the exponent matrix E by [a, b, c]T · d1. Clearly, E corresponds to a (3, L)-regular QC-LDPC code. Given d1 and P , girth of the resultant code is determined by the triple [a, b, c]. If the triple [a, b, c] enables E to yield a code with girth at least g, then the triple is called g-feasible for brevity.
Theorem 2: (i) Let x be an integer. If [a, b, c] is g-feasible, then so is [a+x, b+x, c+x]. (ii) Let α be an integer such that gcd(α, P ) = 1. If [a, b, c] is g-feasible, then so is [αa, αb, αc]. (iii) [a, b, c] = [0, 1, 2] is 8-feasible.
Proof: According to cycle-governing equations for QC-LDPC codes, (i) and (ii) can be readily proved. The proof of (iii) is similar to that of Theorem 1 and hence omitted.
Corollary 1: (i) [0, 1, P − 1] and [0, P − 2, P − 1] are 8-feasible. (ii) If P is odd, then [0, 1, (P + 1)/2] and [0, (P −1)/2, P −1] are 8-feasible. (iii) If [0, 1, c] is g-feasible, then so is [0, 1, P + 1 − c].
Proof: We first prove Corollary 1(i). By setting x = −1 and [a, b, c] = [0, 1, 2] in Theorem 2(i), we have that [−1, 0, 1] is 8-feasible, which suggests that [0, 1, P − 1] is 8-feasible via interchanging the three entries. Similarly, by setting α = −1 and [a, b, c] = [0, 1, 2] in Theorem 2(ii), we have that [0, −1, −2] is 8-feasible, which implies that [0, P −2, P −1] is 8-feasible via interchanging the three entries. Corollary 1(ii) can be similarly proved as Corollary 1(i). Finally, by setting x = −1 in Theorem 2(i) and then setting α = −1 in Theorem 2(ii), Corollary 1(iii) is obvious.
For a given triple [a, b, c], if another triple [a , b , c ] can be obtained via Theorem 2(i), Theorem 2(ii) or combination of them, then the code based on [a, b, c] is equivalent to the code based on [a , b , c ]. Therefore, utilizing Theorem 2, we only need to evaluate the performance of a few codes with nonequivalent triples, as illustrated in Section V (D).

VI. CONCLUSION Based on DDS with two sets, a new construction is proposed for column-weight-four QC-LDPC codes without 4-cycles

and 6-cycles. The proposed method includes the existing
Sidon method as a special case while being able to offer
much smaller circulant sizes than Sidon method. Compared
with several other explicit or semi-random methods, the
new scheme features desirable decoding performance, flexible
circulant sizes, and a large number of candidate exponent
matrices for a given circulant size. Besides, another novel
construction is proposed for column-weight-three QC-LDPC
codes without 4-cycles and 6-cycles, by which short codes are
generated with better performance compared with the existing
counterpart.
