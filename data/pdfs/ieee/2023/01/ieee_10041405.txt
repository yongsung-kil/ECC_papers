III. DEMO DESCRIPTION
The objective of this demo is to show the decoding performance gains in Block-Error-Rate (BLER), decoding energy per bit, and average latency per codeword when noise recycling is applied versus the direct decoding of channels. To show the decoding visually, the encoded input data in the form of a multimedia message with noise is decoded for different codebooks, modulation order, SNR, and strength of correlations. The input image is encoded using the selected codebook. The codewords are interleaved and then modulated using the selected modulation scheme. Correlated noise is added to the modulated symbols. The correlated noise is modeled as GaussMarkov with a temporal correlation coefficient given by ρ. The de-interleaver reverses the effect of interleaving by storing the received symbols in rows but using columns as the output. The output of the de-interleaver is demodulated and fed to the GRAND chip for FEC. The output of the decoder is modulated again and subtracted from the channel’s input to get an accurate continuous noise estimate that the channel has gone through. Fig. 1 shows the block diagram for interleaving and obtaining a noise estimate from the first channel. The process of improving the decoding performance of adjacent channels by applying noise recycling is shown in Fig. 2. This example shows the noise recycling when the first channel is fixed as the lead channel. For the dynamic selection of lead channels, all the columns of de-interleaver are passed through the first iteration of decoding using the GRAND chip, where the abandonment threshold is set to 1-bit. If a channel has more than 1-bit error, the noise guess is terminated, and the channel is marked as abandoned. Channels that pass the 0- or 1-bit error search are chosen as the lead channels from which noise estimate is obtained. These noise estimates from each lead channel are applied to its neighboring abandoned channels on both sides, which improves the SNR, as shown in Fig. 3. These channels are then fully decoded, which results in performance improvement. The measurement setup of the demo is shown in Fig. 4. The continuous-time signal processing is done in the host PC using MATLAB. The demodulated data is sent to the GRAND chip through the FPGA board that is connected to the host PC. An interactive Graphics User Interface (GUI) is used to select the settings and to display the results as shown in Fig. 6.

IV. RESULTS
The input image with correlated noise is first decoded directly without applying the noise recycling. It is then decoded using noise recycling with dynamic selection of lead channels. The decoded image using the two methods is shown in the GUI to visually show the difference along with decoding performance numbers such as BLER, energy, and latency. The simulated BLER for Cyclic Redundancy Check (CRC) (127,113) under different SNR, correlation coefficient, and order of modulation is shown in the Fig 5. This graph shows that as the correlation coefficient increases, decoding gains of greater than 2 dB can be obtained irrespective of the modulation used.
