Efficient FPGA-based LDPC Encoder Implementation for Optical Communication
Systems
Yu Liu,1 Wenhua Gu,1* Zhen Mei,1 Daying Sun1
1Nanjing University of Science and Technology, Nanjing, China *guwenhua@njust.edu.cn

Abstract—The low-density-parity-code (LDPC) code has been suggested as a forward error correction code in the ITU-T G975.1 protocol for optical communications. However, the traditional LDPC encoding method causes high latency or excessive resource usage, which may degrade the encoder’s throughput and performance. In this paper, we propose an efficient encoder implementation of the LDPC code in the ITUT G975.1 protocol based on field programmable gate arrays (FPGA). Specifically, the Richardson-Urbanke (RU) algorithm is modified by incorporating the structural characteristics of the parity check matrix that corresponds to the ITU-T G975.1 protocol, along with specific preprocessing techniques. This approach not only simplifies the encoding process but also makes it more suitable for hardware implementation. Our implementation significantly reduces resource consumption and encoding delay compared with existing methods, achieving a throughput of 10Gbps rate.
Keywords—LDPC, encoder, 10G optical communication, hardware acceleration, FPGA implementation
I. INTRODUCTION
Low-density-parity-check (LDPC) code is a linear code that uses a sparse parity matrix to achieve good error correction performance. It was first proposed by Gallager in 1962 [1]. In 1999, Mackay rediscovered the LDPC code [2], which can approach the Shannon capacity limit. On the other hand, field programmable gate arrays (FPGAs) are widely used in communication systems due to their flexibility and reconfigurability. The Richardson-Urbanke (RU) algorithm is a classic and universal LDPC encoding algorithm [3]. The encoding can be implemented using a hybrid encoding algorithm based on FPGA [4]. However, due to the dense nature of the generator matrix, it requires a lot of storage resources. In addition, triangular matrices have an iterative process, and when the matrix is large, the delay is also high. A pulse code modulation (PCM) two-part method was proposed to avoid the calculation of triangular matrices and reduces latency [5], but it still required a large number of storage resources. To further improve the performance, it is necessary to study the characteristics of specific LDPC codes in the protocol and identify the most suitable encoding method.
In this paper, we design an efficient encoder for LDPC codes of a 10Gbps optical communication system based on the ITU-T G975.1protocol [6]. The proposed method is based on the RU algorithm by taking the structure of the parity check matrix in the protocol into account.

B. FPGA implementation of the encoder
In our FPGA implementation, the encoder processes 64 bits of data per cycle and caches the input data using a firstin-first-out (FIFO), which is also used for output data. The encoder also maps the input information bits, sorts them into the required format for the encoder, and caches the mapped data using a FIFO. Additionally, the encoder performs bitwidth conversion for encoding. After mapping the input information bits, the submatrix A,C and the mapped information are multiplied, and an accumulation iteration is performed. The iteration result is output after the calculation is completed. The coding controller calculates the address and accesses the inverse matrix storage module to retrieve the data of the inverse matrix. The retrieved data is then multiplied with the previous iteration to derive a portion of the check bits.

Furthermore, to improve the throughput and reduce the delay, our implementation uses a pipelining method that inserts registers to hold intermediate results after multiplying the sparse matrix by the vector. Additionally, registers are inserted to hold the data by one more level after reading out the inverse matrix from ROM. When multiplying and accumulating the data in ROM with the vector, multiple lookup tables (LUTs) are cascaded in the computation, resulting in long combinatorial logics. To address this, registers are inserted to hold the multiplication result and receive the final computation result through a shift register. This reduces the length of the critical path and improves the timing performance.

In [4], the RU algorithm was also used for the FPGA implementation of LDPC encoding. Our results show that the logic and storage resources used in our implementation are

lower than those used in [4]. This is due to the greedy matching used in [4], which loses the structural features of H and causes more consumption of logic and storage resources.
In [5], the PCM two-step method was proposed. We combined the structure of H in the ITU-T G975.1 protocol and implemented it based on FPGAs. The sparse matrix multiplication vector consumes few logic resources and lower latency due to the use of the structure of H. However, for the second part of the dense matrix multiplication, the required storage space for the inverse matrix, which is a square matrix of dimension 2045, is larger, and the corresponding circuitry required for computation is also increased. Although a 10Gbps rate is achieved, more storage and logic resources are required compared to the method proposed in this paper.
Our proposed encoding method takes advantage of the structure of H, and only row transformations are performed during preprocessing. This greatly simplifies the computation, and maintains the systematic nature of the coding. In FPGA implementation, our encoder can realize sparse matrix and vector multiplication with fewer logic resources using the operator circuit structure proposed above. In terms of storage resources, compared with the PCM two-step method, the dimension of the inverse matrix to be stored in our method is 1752, which significantly reduces storage resource usage. Additionally, the latency of our encoder is lower, meeting high throughput requirements.
B. Conclusion
In this paper, we designed a high-throughput encoder and coding method based on the RU algorithm for structured LDPC codes in the ITU-T G975.1 protocol. We have utilized the structural characteristics of the parity check matrix to perform fast encoding, achieving a 10Gbps throughput. Compared with existing methods, our LDPC encoder implementation, designed based on the ITU-T G975.1 protocol, is more advantageous than the implementation methods in [4] and [5], in terms of the number of logic resources, storage resource consumption, and throughputs.
