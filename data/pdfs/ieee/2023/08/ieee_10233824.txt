B. QC-LDPC codes
An LDPC code is defined as the null space of a sparse parity-check matrix (PCM). If a PCM is an array of P ├Ś P circulants, a type of square matrix in which each row is just the right-shifted version of its previous row, then the corresponding LDPC code is referred to as a QC-LDPC code. Therefore, a QC-LDPC code can be completely described by P and the so-called exponent matrix in which each entry or each tuple of entries denote the position(s) of the nonzero element(s) in the first row of the corresponding circulant. In this paper, a circulant is either a circulant permutation matrix or an all-zero circulant. In such a scenario, the exponent matrix is composed of integers from 0 to P ŌłÆ 1 and possibly Ōł× (associated with the all-zero circulant). Girth is the smallest length of cycles in the Tanner graph (Tg) of an LDPC code. Small girth is generally recognized as one of the main reasons that damage performance of LDPC codes. For a QC-LDPC code, its girth can be directly determined by the associated exponent matrix and the circulant size P . We use the notation T g(E, P ) to denote the Tanner graph of a QC-LDPC code generated by the exponent matrix E and the circulant size P .

IV. SEARCH ALGORITHMS
Although the existing and new methods in Section III can provide (v, k, 2)-DDSs, the values of v are usually not small enough. This section presents several simple search methods for obtaining (v, k, 2)-DDSs with reasonably small v.
The 2k entries within a (v, k, 2)-DDS can be determined by search algorithms based on certain criteria. The algorithm declares success if it can find all 2k entries, otherwise it declares failure. Three main algorithms are investigated in this section: greedy algorithm, backtracking algorithm and hillclimbing algorithm.

For the columns indexed by even numbers, the 2 entries are generated in the top-to-bottom order, and for columns indexed by odd numbers, the 2 entries are generated in the bottom-totop order.

D. Multiplied-Backtracking algorithm
A variant of the backtracking algorithm is executed only for the first row of a (v, k, 2)-DDS, while the second row is just some multiple of the first row. Such a modified algorithm obviously speeds up the algorithm considerably, but risks missing feasible solutions. In Table II, four DDSs marked ŌĆ£MBŌĆØ are generated by this algorithm.
E. Hill-climbing algorithm
The execution steps of our hill-climbing algorithm are as follows. (1) Randomly generate a 2 ├Ś k matrix. (2) For each entry, adjust its value (keeping the values of other entries unchanged) so that the number of distinct differences is maximized; if the number attains 2k(kŌłÆ1), exit the algorithm. In order to maximize the number of distinct differences for a given matrix randomly generated in step (1), step (2) can be executed for a couple of rounds. Further, in order for the algorithm to obtain a globally optimal solution, step (1) can also be executed in a large number of rounds. In Table II, the two DDSs marked ŌĆ£HŌĆØ are generated by this algorithm.
The above algorithms are not quite efficient for large values of k; for this case, fast search algorithms need to be developed by combining other ideas, such as tabu search [18]. VII. CONCLUSION
Two new types of DDS-based QC-LDPC codes are put forward, which contain all-zero circulants and possess girth

at least eight and girth at least ten, respectively. Compared
with existing methods, new codes based on a DDS not only
exhibit good performance, but also have flexible circulant
sizes. Moreover, a novel result on existence for (v, k, 2)-DDSs
and two new deterministic constructions for (v, k, 2)-DDSs are
proposed. Two issues deserve to be studied in the near future.
The first is how to search for DDSs with small values of v for
large values of k; the second is how to construct other types of
QC-LDPC codes (for example, QC-LDPC codes without small
trapping sets and type-II QC-LDPC codes) using a DDS.
