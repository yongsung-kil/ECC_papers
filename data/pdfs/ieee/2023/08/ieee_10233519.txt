Deep space communication is defined by the International Telecommunication Union (ITU) as a distance equal to or greater than 2 million kilometers [1]. Although deep space optical communications have sufficient bandwidth, and can adopt modulation methods such as pulse position modulation (PPM) [2], the information bits transmit over such an ultra-long distance incurs significant propagation delays. For instance, the maximum propagation delay between Earth-toMars is 22.294 minutes, which is unfavorable for reliable retransmissions [3].
Moreover, the deep space optical communication channel is not constant and change continuously due to the various types of cosmic noise exist, such as planetary noise, background thermal radiation noise, etc. [4]–[7]. The authors in [16] propose a SWD concatenates the soft-decision decoding (SDD) algorithms for the staircase BCH codes. To achieve more flexible code length, the authors in [18] propose a staircase low density parity check (LDPC) codes with sequentially decoding. Moreover, the authors in [17] propose an SDD based SWD algorithm for staircase LDPC codes using a bit-flipping scheme with two bits. However, the rate-adaptive staircase LDPC codes are still lacking of appropriate SDD to improve the bit error rate (BER) performance.

II. SYSTEM MODEL
The staircase LDPC codes with SOSD-SWD algorithm deep space optical communication system is shown in Fig. 1. The memory A in the encoder side stores the re-encoded information bits of the rate-adaptive staircase LDPC code, then the encoded block bits map on a PPM symbol and transmits on the deep space optical communication channel. To enable efficient parallel encoding, the number of LDPC encoders equals the width of the staircase block. The encoded bits are used to update memory A for subsequent information bits encoding and sent to the PPM modulator simultaneously. At the decoder side, the proposed SOSD-SWD algorithm is employed, the SWD first outputs the information values as a priori values to the SOSD in current window, then the SOSD outputs the extrinsic information values back to the SWD in the predetermined iterations. To enable parallel decoding, the number of SOSDs also equals the width of the staircase codeblock. The extrinsic information values are used to update the original a priori values in memory B. After the decoding iterations complete, the decoder outputs the recovered information bits or claim the decoding failure.

III. SOSD-SWD ALGORITHM

Staircase codes are typically decoded via a SWD algorithm. In HDD SWD algorithm, the decoding begins from the last group of staircase codes to the first in current window, where a group of staircase codes refers to two adjacent staircase code blocks. In contrast, the SDD algorithm usually decodes from the first group of staircase codes and onwards, and the soft-decoding iteration consists of two parts: the inner decoding iteration of the component codes and the outer iteration process of the sliding window.
In previous SWD algorithm, the size of the re-encoded matrix is the same as that of the staircase block. Thus, both the front and back blocks of a group of staircase codes can be fully decoded during decoding. Typically, the target block in the sliding window is the first block. However, the reencoded matrix is typically smaller than a code block in the rate-adaptive staircase codes. Consequently, the first block in the current sliding window cannot be fully decoded, and the target block in the sliding window is modified to the second block as illustrated in Fig. 5. Here, the window size w = 4, and only the re-encoded part of the first block in the current window participates in the decoding process.
In our proposed SOSD-SWD algorithm, the LDPC decoder uses SOSD, which is a soft-output variant of the OSD [19]. The OSD can obtain an estimated codeword by performing reliability-based sorting, Gaussian elimination, and reencoding on the input [20], [21], and the SOSD can output the soft information of each bit. The input of SOSD consists of two parts, namely the intrinsic information value and the a priori value, and the difference between the output soft information of the decoder and the input soft information is the extrinsic information value.
In the SDD SWD algorithm, there are mainly three information exchange from the extrinsic information value to a priori value. Firstly, when the target block changes, the sliding window moves backward, and the information is passed from the duplicate block of the previous window. Secondly, within the current window, during one outer iteration, information is passed between adjacent groups of staircase blocks. Finally, between two outer iterations, information exchange occurs during the initialization of the a priori value.
