LIAD: A Method for Extending the Effective Time of 3-D TLC NAND Flash Hard Decision
Xiaolei Yu , Jing He, Qianhui Li, Bo Zhang, Xianliang Wang, Liu Yang, Tianchun Ye, Qi Wang, and Zongliang Huo

Abstract—Triple-level cell NAND flash memory is widely used today due to its higher storage density and capacity. However, with the increase in the storage density, lower reliability results in more read times for flash memory and significantly reduces the read performance. In order to avoid unnecessary read operations, this article proposes a hard decision–soft decoding method called location information-assisted decoding (LIAD) method, which determines the additional information required for decoding by mutual information, and then transmits the required information to correct the log-likelihood ratio (LLR). Different from the conventional LLR correction algorithm, this method does not require additional read operations and correct data. Only using sensing results, our method can reduce uncorrectable error bit rate (UBER) by up to 99%, and the system read latency under SSDsim (Hu et al. 2011) simulation can be reduced by up to 53%.
Index Terms—3-D triple-level cell (TLC) NAND flash memory, bit error rate (BER), low-density parity check code (LDPC), mutual information, read latency.
I. INTRODUCTION
N OWADAYS, people cannot live without a variety of storage devices. From the initial 3.5-inch floppy disks to high-speed solid-state drives, storage capacity and speed have always been the most crucial indicators of storage devices, especially for data centers and commercial servers. The original NAND flash memory debuted as a single-level cell (SLC) having only one nonzero state. Later, available equipment uses multiple voltage levels, known as the multilevel cell (MLC).

Recently, with the emergence of the triple-level cell (TLC), the problem of errors caused by further narrowing the distribution state within a certain threshold voltage range has become more serious, making the conventional decode method incur more read operations, which leads to unbearable read latency. So, this article wants to address a key question: can we get more information from the hard decision?
The answer is yes. Recently, many researchers have used communication principles in flash memory to study similar problems. Wang et al. [2] assumed an independent and identically distributed (i.i.d.) Gaussian threshold distribution for each level of MLC NAND flash memory and proposed the flash memory N-M channel model. By calculating the mutual information of the flash memory channel under different read voltages, a method of taking the maximum mutual information as the read voltage is proposed. However, the threshold distribution in TLC NAND Flash memory does not satisfy i.i.d. We observe that as the threshold voltage of the flash memory increases, the variance of its distribution state will increase. Therefore, using this method to calculate mutual information is not accurate. Aslam et al. [3] optimized the threshold voltage model, but still uses the N-M channel model proposed in [2], which brings two problems. First, the N-M channel model simultaneously considers the mutual information of N distributed states (four for MLC, eight for TLC) and M threshold voltage quantization regions of the flash memory. With the increase of N and M, the computational complexity of mutual information increases exponentially. Second, in the decoding stage, different logical pages will be decoded separately, considering the mutual information of all distributed states will causing errors (for example, to decode the lower page of MLC, we only need to distinguish the E and P1 states and the P2 and P3 states. At this time, the error between the E and P1 distribution states does not affect the decoding result, so it is wrong to calculate the mutual information between the E and P1 distribution states). Due to the unique physical device characteristics of 3-D TLC NAND flash memory, Wu et al. [4] proposed to use the error mode in TLC to correct the decoding log-likelihood ratio (LLR), but the correction parameters only use statistical laws and lack mathematical guidance. Moreover, the correction needs to know the correct data of the other two logical pages in a physical page, which reduces the algorithm’s robustness and increases the read latency under random read requests.

II. BACKGROUND
In this section, we first introduce some basic concepts in NAND flash memory, and in the following sections, we give a more comprehensive introduction to the knowledge involved in the algorithm.
Compared to SLC and MLC, TLC NAND flash memory stores more information bits in one cell, which results in a smaller margin between the two threshold distribution states, making it easier to generate errors. Like MLC NAND flash memory, TLC NAND flash memory usually uses Gray Code when mapping information bits to voltages in TLC cells so that adjacent levels are only different in one bit. Fig. 1 shows an example of 3 bit/cell TLC NAND flash memory. Voltage levels E, S1, S2, S3, S4, S5, S6, and S7 represent information bits “111,” “110,” “100,” “000,” “010,” “011,” “001,” “101,” and “100,” respectively. This article uses physical pages to represent a combination of the lower page, middle page, and upper page. The most significant bit (MSB), center significant bit (CSB), and least significant bit (LSB) in the physical page correspond to the upper, middle, and lower logic page, respectively.

Fig. 2. Illustration of TLC NAND flash memory page read operation.
Retention time refers to the time that data is stored in flash memory, and effective retention time refers to the time that data is stored in flash memory to an unbearable UBER. Obviously, the effective retention time is different under different lifetime noise and decoding algorithm. A decoding algorithm with stronger error correction capability means a longer effective retention time.
A. Read Operation of 3-D TLC NAND Flash Memory
As more than one bit is stored in a single cell, the read operations of TLC NAND flash memory are also complicated. As shown in Fig. 1, Because sense-amplifier comparator provides at most one bit of the information about the threshold at a time, if the cell voltage is on the left of the reading voltage, it contains logic “1”; otherwise, it contains logic “0.” This operation is called sensing. So, in TLC NAND flash memory, reading one page requires at least two sensing operations. As shown in Fig. 2, when reading the lower page, the flash cell needs to be sensed twice by changing the read voltage to V1 and V5, and the sensing result are expressed as D1 and D2, respectively. The final data for the lower page is then represented as D1 ⊕ D2. The read of the upper page is similar, except that the reading voltages need to be set to V3 and V7. The final data is represented as D6 ⊕ D7. For the middle page, the flash cell is sensed three times by changing the read voltages to V2, V4, and V6, representing the results of D3, D4, and D5, respectively. The final data for the middle page is represented as D3 ⊕ D4 ⊕ D5.
The read requests sent to the flash memory can be divided into two types: 1) sequential read requests and 2) random read requests [6]. Sequential read request means that the read operation is performed in the order from the lower page to the upper page. This brings more available device characterization information to the decoding algorithm, so most LLR correction algorithms (LCAs) only take effect on such read requests. Random read request means that the logical page to be read by the read operation is irregular and arbitrarily specified. This

B. Basics of LDPC Code
LDPC code is known as the capacity proximity code for AWGN channels, defined by a sparse M × N parity-check matrix. This matrix can be represented as a Tanner graph with N variable nodes and M check nodes. We can use multiple algorithms, such as the PEG algorithm [7] and the ACE algorithm [8] to generate LDPC codes. Because storage systems usually require a bit error rate (BER) lower than 10−12, and the storage channel is different from the AWGN channel, there has been a lot of research on how to use LDPC in flash memory including [9], [10], [11], [12], [13]. At present, QC-LDPC is mainly used in flash memory controllers.
Error correction is realized by belief propagation algorithm [14], iteratively calculating the decode messages between variable nodes and check nodes. LDPC decoding schemes can be grouped into two types: 1) hard decision and 2) soft decision. Hard decision uses a single read voltage to distinguish between two adjacent storage states. Of course, we can use multiple read operations between two adjacent distinguish states to improve the decoding performance, multiple read operations can provide more accurate LLR, which is called soft decision. The process of decoding using hard decision information is called hard decoding, and using soft decision information is called soft decoding. Hard decoding is faster than soft decoding, but with less error correction capability [15]. However, the NAND flash memory read latency is proportional to sensing accuracy, soft decision will result in longer read latency. Therefore, extending the effective retention time of hard decision is necessary.
The state-of-the-art LCAs are divided into two types: one is use additional reads to get more accurate decoding LLR. Ouyang et al. [16] analyzed the position of the optimal read voltage under nonuniform quantization in detail. This kind of method has a complete mathematical analysis and theoretical basis, but inevitably requires more read times, which has an impact on the system performance. another is to use the characteristics of NAND flash memory devices to optimize the decoding LLR. Wu et al. [4] proposed the error mode aware LDPC (EMAL) algorithm. It uses the numerical correlation characteristics of flash memory cell errors to identify different error modes, and then corrects the decoding LLR according to the empirical formula. It guesses the numerical reliability of the high RBER logic page based on the combination of the values in the two logic pages with the low RBER. But this kind of method needs to read three sequential logic pages at the same time, and the lower RBER logic page needs to be decoded correctly. So, the effect is not so good in practical applications.
C. Introduction to SSDSim
SSDSim is a widely used accurate and multitiered SSD simulator. It obtains the waiting time, processing time, response

time of each request, total erasure count, buffer hit count, and other detailed information by using the I/O trace file provided by the solid-state storage initiative workload I/O capture program (SSSI WIOCP) [17]. Most of the LCAs have a certain system overhead. For example, the algorithm proposed by Zhang et al. [18]. uses the flash pair bit error (PBE) feature to correct the decode LLR, which requires sequential reading of flash logical pages. The algorithm proposed by Kong et al. [19]. uses the flash cell to cell interference to correct the decoding LLR, which requires sequential reading of flash physical pages. These constraints will more or less affect system performance. However, very few papers use the system performance to measure the effectiveness of the LCA. This article uses flash memory chips from Yangtze Memory Technologies Company Ltd. (YMTC) to collect data sufficient for system simulation. Therefore, this article will use SSDSim to verify the algorithm’s effectiveness.
III. 3-D TLC NAND FLASH MEMORY 2-M CHANNEL MODEL
As we mentioned above, the N-M flash memory channel model has some errors, so this article proposes a 2-M flash memory channel model to obtain accurate mutual information. According to the mutual information results, we can divide the flash memory channels into three categories, that is, the algorithm proposed in this article does not need to be use (corresponding to the early retention time, the RBER is low), the algorithm proposed in this article can be used (corresponding to the middle retention time, the RBER is high), the algorithm proposed in this article cannot be used (corresponding to the end of retention time, the RBER is too high). If we decide to use the algorithm proposed in this article, we will determine the decoding information we need through the 2-M model. It should be pointed out that the algorithm proposed in this article extend the effective retention time without increasing read times, so it is suitable for the stage of high RBER. It is necessary to increase the number of reads for too high RBER.
We use the lower page of TLC NAND flash memory as an example and give the channel models of its hard decision and soft decision, respectively. We consider four additional read voltages (six in total) in the soft decision and quantize the threshold voltage into seven different regions. A. LIAD Principle
We use the decoding flowchart shown in Fig. 4 to illustrate the difference between this algorithm and the conventional LLR correct algorithm. First, we save the threshold voltage model in the flash memory controller, which can get the approximate threshold voltage distribution according to P/E and retention time. When a read request comes, we determine

the sensing data needed for decoding. This is done by calculating the channel capacity using the 2-M channel model mentioned in the previous section. Shannon’s law [22] tells us that the maximum code rate that can be transmitted on the channel is less or equal to the channel capacity. Second, We transmit logical page data and the sensing data required for decoding at the appropriate time. Using sensing data, we can optimize the logic page decoding LLR. If we still fail to decode we start the conventional soft decoding process. The impact of different read requests is: under random read requests, we can only use limited sensing results. B. LIAD Decoding Algorithm for Random Read Request
In this section, we introduce the decoding algorithm under random read requests. The results show that it only needs to send data at most two times to get all the information needed by the method.
In the random read process, as shown in Fig. 5, taking the lower page as an example, in the hard decision, because the results of two sensing operations are NXORed, we actually lose the position of logic 1, that is, the decoder cannot know whether the logic 1 is in the E state or the P5, P6, and P7 states. By collecting experimental data, we found that the transition probability of the flash memory cell in the Region1 is completely different from the transition probability in Region3. This indicates that the decoding performance will be improved if the location information of E state can be known. Especially, after a certain retention time, because the cell voltage shifts to the left, Region1 basically no longer has 0 to 1 errors, while Region3 will have more 0 to 1 errors.

C. LIAD Decoding Algorithm for Sequential Read Request In the sequential read process, since the logical page will be
read continuously, we can use the sensing results of other logical pages to assist the decoding. The sensing information of lower, middle and upper pages can assist each other in correct decoding LLR. So, we can divide the flash threshold voltage into up to eight regions. We take lower page decoding using middle page sensing results as an example. The lower page and middle page perform a total of five sensing operations, and the threshold voltage of the flash memory can be divided

into six regions at most. According to the coding principle, six regions need at least 3-bit representation, so we need to transmit data three times.
As shown in Fig. 7, the figure on the left is a channel model without middle page sensing results, the figure in the middle is a channel model with one middle page sensing result, and the figure on the right is a channel model with two middle page sensing results. Of course, under the premise of sufficient mutual information, we can use any combination of sensing results to improve the efficiency of flash IO utilization.
Fig. 8 shows how the lower page uses the sensing results of the middle page to assist in decoding. Flash memory first transfers the lower page data (D8), and then transfers the middle page sensing result (D4) and the middle page data (D9). So, we can distinguish six different regions by combining the data D8, D4, and D9. The data combination “111” means that the data is in region one, “011” means that the data is in region two and so on. Decoding LLR is optimized according to different regions. We can also get six different LLR regions by using the data D2. This is because there will be redundant coding in six regions represented by 3 bits. So, we can choose the free time of the system to transmit data.

A. Experimental Conditions
We use 3-D charge trap NAND flash memories fabricated with 64 stacked word-lines (WLs) by YMTC to evaluate the effectiveness of the LIAD method in improving LDPC decoding performance and reducing read latency. We use the ACE algorithm [23] and the stopping set check algorithm [24] in the NAND flash memory storage system to construct a quasicyclic LDPC (QC-LDPC) [25] with the code length of 16 383 and the code rate of 0.87 according to the degree distribution. The experimental data collection uses an FPGA-based testing platform and the windows-based applications connect to the testing platform through an Ethernet port.
The experimental process described below is mainly divided into two stages: writing data to TLC NAND flash memory and reading the data to detect error codes.
In the write stage, six types of P/E cycles of 0, 1K, 2K, 3K, 4K, and 5K were selected for the experiment. We perform P/E on different blocks to make them reach the corresponding P/E times, and then we generate address sequences and random numbers to program the entire block. High-temperature acceleration was used to accelerate the retention time of flash memory. The experimental temperature used in this article is 125 ◦C and the Arrhenius Law [26], [27] is used to convert the data retention time at 125 ◦C to data retention time at 55 ◦C.
In the read stage, in order to reduce the influence of process variation, we read the pages with numbers 2, 8, 14, 20, . . . , 380, which are in different word lines. To reduce the impact of various noises [28] on read operations, we read each page twice and take the average. The reading method we use is scanning reading, reading the entire page every 25 mV from −1.6 to 6 V to get the accurate voltage of each cell. Due to the unique structure of 3-D charge trap NAND flash memory, multiple readings will not have much influence on the threshold voltage, but it will help the experimental results more accurately.
In order to evaluate the read latency of the system, according to the fabricate’s manual, the flash read time (3-D TLC NAND flash memory multiplane sensing once operation time) was defined as tR, the flash data out time, the time to transmit the data in the page buffer, as a function related to the DQS period and the number of bytes to transmit Tdataout = TDQS × Ndata ÷ 2, and the delay of each decoding iteration of LDPC is set to 1 μs.

C. Discussion
The experimental results found that when the RBER is relatively moderate, the LIAD method can effectively improve the LDPC decoding performance and reduce the read latency. When random read requests come, there is less information available, and the LIAD method tries to find a way to improve system performance without additional read operations. When continuous read requests come, the LIAD method can use more effective information, so the performance is better under sequential read operations. Fortunately, the previous work of [6] showed that sequential read operations accounted for more in NAND flash memory. In general, this article explored the soft information hidden in the hard decision, and tried its best to make the flash memory have better performance without increasing the number of read times.
VI. CONCLUSION
In this article, we proposed the LIAD method, which uses the 2-M channel model to evaluate the flash memory channel and combines the page buffer operation of the 3-D TLC NAND flash memory to improve decoding performance and reduce read latency. The LIAD method improved the performance of flash memory random read and sequential read operations without adding additional read operations. Specifically, transmit the data in the page buffer to reasonably infer the correct bit value of the page to be decoded, thereby reducing the UBER of the flash memory page and decreasing the number

of decoding iterations. The experimental results showed that
the method significantly improves the decoding performance
