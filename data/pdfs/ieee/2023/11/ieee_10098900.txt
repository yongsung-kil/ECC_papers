Interleaved LDPC Decoding Scheme Improves 3-D TLC NAND Flash Memory System Performance
Xiaolei Yu , Jing He, Bo Zhang, Xianliang Wang, Qianhui Li, Qi Wang, Zongliang Huo , and Tianchun Ye , Senior Member, IEEE

Abstract—Although NAND flash memory does a lot of work in effectively using error correcting code (ECC) to reduce uncorrectable bit error rate (UBER). However, if the frame error rate (FER) is not reduced, the lower UBER cannot effectively reduce the read latency of the flash memory system. This phenomenon is especially evident at the end of the flash memory lifetime, where conventional methods significantly reduce the UBER but not to zero, and the remaining error bits are still evenly distributed throughout the flash memory page, resulting in a significant increase in read latency. In this article, an interleaved LDPC decoding scheme is proposed. By re-evaluating the flash memory channel during the decoding process, the codewords in the flash memory page are corrected frame by frame, and the problem of high FER is solved at the end of the flash memory lifetime. Compared with the conventional algorithm, the proposed method can reduce the FER by up to 34%, reduce the average decoding iterations by 63.4%, and reduce the read latency by up to 65%.
Index Terms—3-D triple-level cell (TLC) NAND flash memory, frame error rate (FER), LDPC, read latency.
I. INTRODUCTION
F LASH memory is the product of the mutual promotion of research and industrialization. On the one hand, it significantly increased the capacity and reduced the unit price at the cost of reliability. On the other hand, lower unit prices have allowed flash memory systems to adopt more methods to improve reliability. The most common method is to improve the error correction performance of the error correcting code (ECC) module in the NAND flash memory controller. Currently, 3-bit/cell triple-level cell (TLC) NAND flash memory has been widely commercialized. However, test results show that the error of 2Xnm TLC NAND flash memory is increased by over 10 times compared to the 2-bit/cell multilevel cell (MLC) NAND flash memory [1]. This

Retention errors are caused by charge leakage over time after the flash memory cell is programmed and have been found to be one of the primary flash errors [3]. As a result, numerous articles are dedicated to reducing retention errors to improve the reliability of flash memory. One way to improve the reliability of NAND flash memory is to apply strategies at the system level. “Read Retry” [4] uses multiple read operations to search for an optimal read reference voltage to reduce uncorrectable bit error rate (UBER) and improve reliability. Furthermore, “flash correct-and-refresh” [5] is also an effective strategy. It can correct and rewrite the data in the flash memory page to clear errors that occurred before the refresh. However, they all require a lot of additional read and program operations, which reduces the system’s efficiency and degrades the flash memory cell. Another solution is to use ECC with stronger error correction capability to meet the reliability requirements of the flash memory. LDPC with good error correction performance [6], [7] is considered one of the ECC options for the NAND flash memory system.
The process of LDPC error correction is the propagation of reliability information, and the input of the decoder is decoding information (i.e., probability information or log-likelihood ratio (LLR) information) used to characterize the reliability of bits [8]. The accuracy of the decoding information and the maximum number of decoding iterations determines the error correction capability of the decoder. Because the number of decoding iterations affects the decoding time, the maximum number of decoding iterations cannot be set too large. Therefore, most research focuses on improving the accuracy of the decoding information in a limited number of decoding iterations.
Currently, two main methods exist to improve the accuracy. One is to obtain more accurate decoding information by adding additional read operations [9], [10]. However, this method has the same problem as the system-level method. The increased number of read operations greatly increases the read latency of the flash memory system and reduces the read performance. The second is to optimize the decoding information by using the numerical correlation characteristic exhibited by retention error [11], [12]. At the same time, the LLR optimization scheme proposed in [12] needs to use the correct data from the lower and middle pages to optimize the decoding information of the upper page, which undoubtedly increases the limitation of the algorithm. Their LLR correction strategy is done individually for the entire flash memory block, not for the codewords. Therefore, it is impossible to distinguish the differences between the codewords, including the differences in the word lines where the codewords are located and the differences in the codewords being affected by other logical page data.
This article proposes the interleaved LDPC decoding (ILD) algorithm to improve the above shortcomings. The algorithm can accurately reflect word line differences and optimize the decoding information of any logical page without increasing the read operations and storage overhead. More importantly, it can significantly reduce FER. The algorithm is divided into horizontal and vertical decoding information optimization. The horizontal optimization uses the correctly decoded codewords in the flash memory logic page to obtain more accurate raw bit error rate (RBER) information, thereby solving the problem that the RBER of different word lines is quite different. Vertical optimization uses the information provided by other logical pages to optimize decoding information further. Three logical pages are corrected in the order of RBER from high to low (from upper page to lower page). Alternately using the horizontal and vertical decoding information optimization, we corrected the error codewords in the flash memory page frame by frame. This makes the algorithm proposed in this article still perform well at the end of the flash memory lifetime.

II. BACKGROUND KNOWLEDGE
A. Basic of 3-D TLC NAND Flash
A 3-D TLC NAND flash memory block with a vertical structure consists of many layers shown in Fig. 1, each layer containing many storage cells. The bit line and word line connect cells to form memory arrays. The 3-D TLC NAND flash memory stores three bits in each cell with eight states. When programming, the three bits of each cell are mapped to three different logical pages, namely, the lower, middle, and upper pages [14]. At present, there are two types of 3-D NAND flash cells in the industry, floating gate (FG), and charge trap (CT). The main difference is that FG NAND flash uses a FG made of doped polysilicon to store electrons instead of the silicon nitride membrane used in CT. CT-based 3-D TLC NAND flash technology is widely adopted by Samsung, Toshiba, and Hynix because of its scalability and performance outperforming FG-based 3-D TLC NAND flash memory [15]. However, whether it is CT or FG NAND flash memory, as the number of flash memory stacks increases, the unique array structure will cause significant variation from word line to word line [16]. Therefore, we need a more flexible decoding algorithm to cope with increasingly complex word line characteristics.

B. Basic of LDPC in NAND Flash
In this section, we first introduce the application of LDPC in NAND and then introduce how they work, including obtaining the initial decoding information and correcting errors in the decoding process.
1) Application of LDPC Code in NAND: LDPC codes have been widely used in NAND flash memory due to their excellent error correction capability. The parity check matrix is an important parameter in constructing LDPC codes. However, the conventional LDPC parity check matrix is irregular, with problems, such as high storage cost, difficult reading, and high coding complexity. Therefore, quasi-cyclic low-density paritycheck codes (QC-LDPCs) are generally used in NAND, and its parity check matrix can be divided into multiple square matrices of equal size. Each square matrix is a cyclic shift matrix or an all-zero matrix of the identity matrix, which is very convenient for the storage and addressing of the memory. Thereby the coding and decoding complexity of the LDPC code is significantly reduced.
2) LDPC Decoding: Before starting LDPC decoding, the initial decoding information obtained from flash memory is considered LLR information [17]. If the cell threshold voltage is in the inductive level region [vi, vj], the LLR information for the lower, middle, and upper pages can be calculated by using the following expression:

After getting the initial decoding information, we use the parity check matrix for decoding. The matrix contains only 0s and 1s, and its rows and columns correspond to the check nodes and variable nodes, respectively. We use the parity check matrix shown in Fig. 2 to illustrate how decoding is performed. A row of the check matrix represents a check equation, and the position of the value “1” in a row represents the variable node checked by the check equation. When the check equation is 0, it means that the variable nodes verified by the check equation are all correct, and if the check equation is “1,” it means an error in the variable node verified by the check equation. We can finally determine the wrong variable node position through multiple check equations, thereby correcting the wrong bit.

We use the blue box to represent the entire codeword, the black box to represent the bits in the codeword, and the red padding to represent the error bits. The second and third lines represent the decoded data. We use the green box to represent the entire codeword. C. Related Work
Research on improving the decoding performance of LDPC in flash memory controllers can be divided into two categories. First, use flash read operations to obtain a more accurate initial decoding LLR. Second, use characteristics of the flash memory cell to correct decoding LLR.
For the first type of work, Li et al. [18] proposed using the workload’s access characteristics to reallocate the read and write precision. But as the number of flash memory stack layers increases, the characteristics of different word lines are different. The flash characteristics cannot be adapted according to the workload type alone. Chen et al. [19] proposed an LDPC decoding scheme with nonuniform reference voltages, significantly reducing the UBER of MLC NAND flash memory. However, using this method in TLC NAND flash memory requires adding more read operations than in MLC NAND flash memory, which introduces unacceptable system latency.
For the second type of work, Tokutomi and Takeuchi [11] solved the problem of LDPC’s poor ability to decode burst errors by recording the error position into a table, but this requires more additional redundant bits to record the error position, resulting in an inevitable loss of capacity. The REAL [20] algorithm incorporates the numerical correlation characteristics of retention errors into the LDPC decoding process. According to the error mode of flash memory retention error, the valid data of the same flash memory physical page is used to correct the decoding LLR and obtain better decoding ability. EMAL [12] algorithm goes one step further. The error modes caused by various noises in the flash memory are globally counted. Then, more accurate decoding information is given from a statistical point of view, and the algorithm’s reliability is verified in TLC NAND flash memory for the first time. However, the above method

assumes no difference between flash word lines and uses uniform additional information to correct decoding LLR. At the same time, they need correct data from other logical pages when correcting and decoding information, which is very limited in TLC NAND flash memory.
This article proposes a new decoding strategy for flash memory. We consider the word line characteristics and error mods of flash memory separately and then impose different decoding information on each codeword. The read latency of flash memory is reduced without increasing the number of read operations and redundant bits.
III. CODEWORD CHARACTERISTICS IN 3-D TLC NAND FLASH
In this section, we introduce the horizontal and vertical characteristics of NAND flash memory. We then explain how to use these characteristics to optimize LLR at the decoding stage.
A. Horizontal Characteristics—Word Line Difference
As mentioned before, different word lines of 3-D TLC NAND flash memory have different characteristics, which is mainly reflected in their different RBER. Since a flash memory page contains multiple codewords, these are on the same physical page and thus have similar RBER. Based on this phenomenon, we use the correct codewords to correct the decoding LLR of the incorrect codewords.
Extracting the horizontal characteristics of flash memory requires the correct codeword in the logical page, so we first count the proportion of 3-D TLC NAND flash memory logical pages containing the correct codeword under different conditions. As shown in Fig. 4, we calculated the percentage of correctly decoded codewords on different word lines, where the y-axis represents the percentage of decoded correct codewords, and the x-axis represents the retention time. Error bars represent differences in the 64 word lines of the flash memory. According to the statistical information, even under 3K P/E, most logical pages still have codewords that can be decoded correctly, so the algorithm proposed in this article can be widely applied to the whole lifetime of flash memory. At the same time, we found that the error bar of the proportion of correctly decoded codewords in the word line is relatively large, proving that different word lines of the flash memory have significant differences.
As shown in Fig. 5, five codewords on a logical page are taken as an example. Each square represents a codeword, and the green true mark in the square indicates that the codeword is decoded correctly, and the red wrong mark indicates that the codeword is decoded incorrectly. With the correctly decoded codeword, we can get the raw 0 to 1 and 1 to 0 bit error rate of the logical page. Using RBER, we can get a more accurate decoding LLR of the logical page. When we decode the lower page, since the codeword numbers 1, 3, 4, and 5 are decoded successfully, we can get the position of the error bit by XOR the data before and after decoding. By counting whether the original data of the error bit is 0 or 1, we can get the raw 0 to 1 and 1 to 0 errors of the logical page and then redecode codeword 2. B. Vertical Characteristics—Error Mode This section studies the vertical characteristics of flash
memory, which means that we need to use all logical pages in the physical flash memory page and simultaneously get more accurate decoding information. We will discuss how to use the correct and incorrect bits on the logical page.
There have been several papers analyzing the error modes of TLC NAND flash. Paper [12] analyzes the data pattern of highreliability pages (lower and middle page) of flash memory that causes errors in low-reliability pages (upper page) and uses high-reliability page data to optimize the decoding LLR of low-reliability pages. This article will divide the vertical characteristics of flash memory into two cases. The first case is

that all three logical page codewords are decoded correctly at the same codeword position. As shown in Fig. 5, the codeword number 4 of the three logical pages decoded correctly. The second case is that three logical pages cannot be decoded correctly at the same codeword position. As shown in Fig. 6, at codeword position 1, the upper page is decoded incorrectly, and at codeword position 2, the lower page is decoded incorrectly.
For case one, by decoding the codeword where all three logical pages are decoded correctly, we can get the RBER of all threshold distribution states of the flash memory. Specifically, we count the state the codeword was in when it was read, and since the codeword was decoded correctly, we can get the state it was in when it was programmed. The transition probability of each threshold distribution state can be calculated from this. As shown in Fig. 7, the distribution state S5 should be “1” when programming on the lower page. We can get the transition probability of the lower page from S5 to S4 by counting how many S5 cell voltages are shifted to S4 when reading. Similarly, when the lower page cell is “0” and the middle page cell is “1,” the upper page has a high probability of remaining correct, as the light blue line in Fig. 9(c) indicates, because going from S1 to S4 or S4 to S1 is difficult. When we know that the bit on the lower or middle page is wrong, we can also determine that the upper page has a certain probability of remaining correct. As shown in the red line box, when an error

occurs on the lower page, the most likely error is generated from S0 to S1, S1 to S0, S4 to S5, and S5 to S4. The upper page data corresponding to these errors are always unchanged. That is, the upper page S0 and S1 are equal to “1,” S4 and S5 are equal to “0.” If an error occurs on the middle page, the situation is similar. So unless a cross-state error, such as S0 to S3 or S5 to S2 is generated, there will be no error on the upper page where the error is generated on the lower or middle page.

When the cell of the middle and upper pages is “0,” the error generated by the lower page is a high probability error. A. Overview of ILD Algorithm
As mentioned earlier, the algorithm proposed in this article corrects the incorrectly decoded codewords in flash memory frame by frame. Fig. 10 shows the difference between the ILD algorithm and the conventional LDPC decoding process. The conventional LDPC decoding algorithm obtains the initial LLR information from the flash memory channel and then relies on the iterative updating of variable nodes and check nodes to obtain the final decoded output. However, the initial decoding LLR of flash memory channels affected by noise is often not accurate enough to achieve good decoding performance.
Fig. 10(b) is the LDPC decoding process proposed in this article. When logical page decoding fails, we first use the horizontal characteristic to optimize the decoding LLR. There are two main reasons. One is because the horizontal optimization only needs the current logical page data instead of three logical page data, so we can perform LLR correction immediately after the logical page data is read; the second is because we hope to have more correct codewords to provide us with

Vertical optimization utilizes the data of three logical pages of flash memory. Vertical optimization starts in the order of RBER from high to low (from upper page to lower page). This is because when optimizing the current logical page, we need to use the correctly decoded codewords of other logical pages. When performing vertical optimization for the first time, the upper page has the highest RBER while the lower and middle pages have lower RBER, so we optimize the upper page first. After the upper page is optimized, its UBER decreases, and the middle page becomes the page with the highest UBER, so we use the lower page and the optimized upper page to correct the decoding information of the middle page. D. Discussion
ILD algorithm can improve the LDPC decoding performance and reduce the system read latency at any time in the flash memory life time, but the performance is limited under random read requests. Compared with the conventional algorithm, it can greatly reduce the number of read times. VI. CONCLUSION
To the best of our knowledge, this article is the first to improve the decoding performance by exploiting the local correctness of the flash memory to optimize decoding information on 3-D TLC NAND flash memory. In this article, a new LDPC decoding scheme is proposed. We corrected the LLR information of LDPC decoding by utilizing partial decoding results to improve decoding performance and reduce the number of decoding iterations. The ILD algorithm does not need to increase the number of reads and storage space so that it can reduce the system read latency. It has been experimentally verified that it can work well in most flash memory lifetimes. The simulation results show that this scheme significantly improves the decoding performance and reduces the system read latency.
