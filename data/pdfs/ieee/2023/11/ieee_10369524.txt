Abstract—A stepped low density parity check (LDPC) codes punching algorithm based on multiple check matrices is proposed. The proposed scheme is more flexible to realize the adaptive rate under Gaussian white channel. In a 100Gbit/s optical communication simulation system, the proposed algorithm has about 0.49dB gain after 1000km of transmission, compared to the general punching algorithm.
Keywords—low density parity check codes, punching, optical fiber channel
I. INTRODUCTION
With the rapid development of the Internet of Things (IoT), cloud computing and a variety of new services in recent years, the transmission capacity and reliability of optical communication systems have also ushered in corresponding challenges[1-2]. Channel coding techniques add a certain number of redundant bits at the source so that they can be used to detect the exact location of the error and correct it, which enhances the immunity of the data to interference during transmission and improves the reliability of the system. The LDPC codes, as a linear packet code with excellent performance, has a strong error correction capability[3-4]. The IEEE 802.16e protocol specifies LDPC codes as one of the channel coding schemes in order to meet the requirements of low bit error rate (BER) data transmission [5]. The code rate of the error-correcting code needs to be flexibly adapted to the current channel state, i.e., rate-compatible LDPC (RCLDPC)codes. There are several ways to realize RC-LDPC codes. Multiple code rates can be realized by punching,

In this paper, a stepped LDPC codes punching algorithm based on multiple check matrices is suggested. Based on the base matrix of LDPC codes under the IEEE 802.16e standard, higher rates are obtained by deterministically punching the respective check matrices at different rate nodes. Simulation results show that our proposed punching algorithm outperforms the common random punching scheme on gaussian white channel. Moreover, it has a gain of 0.49dB after 1000km transmission on an optical coherent system.

In our proposed scheme, the multiple check matrices are constructed using the above rules, which are the check matrices of rates 1/2,2/3,3/4, respectively.

The principle of the proposed scheme is shown in Fig. 1. We choose (1/2), r (2/3), r! (3/4) as rate nodes. At the transmitter side, a check matrix with different rate nodes is first selected for punching according to the target code rate. The size of the data bit block is 8100, encoded by the LDPC encoder with the value of the rate node. During the encoding phase, the size of the parity check matrix is determined by the different rate nodes. Puncturing operation is the critical step in our theory. Firstly, the effective row weights of the parity matrix are calculated, sorted and labeled according to the row weight values. When punching bits, preference is given to rows with small effective row weights. If the same row weight occurs, it is randomly selected. When the target code rate cannot be achieved, the row with the smallest row weight is selected among the rows with the highest row weight until the desired rate is met. As the punching proceed, the parity check bits are updated, thus obtaining the target coding rate and frame structure. In the general puncturing operation redundant bits can be punctured either randomly or deterministically by select only one master code. From Fig.1, it can be seen that when the rate node is 1/2, the final code rate obtained by punching according to the method of the proposed scheme is 3/5. When the rate node is 2/3, the final code rate obtained is 7/10. When the rate node is 3/4, the final code rate obtained is 4/5. This is our proposed stepped LDPC punching algorithm.

The pseudo-random binary sequences (PRBS) is generated in MATLAB. The data encoded by the LDPC encoder is QPSK mapped and transmitted to the MachZehnder Modulator (MZM) for modulation. The 1550 nm light source is divided into two separate paths into two MZMs, which generate optical signals through the interaction of the MZMs. The transmitted signal is attenuated by a variable optical attenuator (VOA) to reduce the signal power. And recirculating loop consists of a single mode fiber (SMF) and an erbium doped fiber amplifier (EDFA). We used G.654e fiber with a dispersion of 21.2 ps / nm / km for a single span length of 100 km. After transmission through the optical fiber, the optical signal is filtered by a band-pass filter and enters the coherent receiver, at which time the signal is converted from the optical domain to the electrical domain. The offline operation of the digital signal processing (DSP) is done on MATLAB. The DSP module includes down sampling, clock recovery, digital back propagation (DBP) and phase recovery using blind phase synchronization (BPS).

IV. CONCLUSION
A novel stepped LDPC punching algorithm based on multiple check matrices is proposed and a simulation platform for 100 Gbit/s coherent optical communication system is established. The simulation results show that in Gaussian white channel, compared with the random punching scheme using base matrix, our proposed punching theory has about 0.36dB-0.42dB gain in HD-FEC threshold, and the punching code rate can be improved to 4/5. Also, when the punching rate is 3/5, after 1000km fiber channel transmission, our proposed scheme has about 0.49dB gain. Overall, the results can prove that our proposed stepped LDPC punching algorithm based on multiple check matrices has some advantages in longer distance optical communication systems and the system flexibility is improved.
