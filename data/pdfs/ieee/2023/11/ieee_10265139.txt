ALCod: Adaptive LDPC Coding for 3-D NAND Flash Memory Using Inter-Layer RBER Variation
Meng Zhang , Xiaoyi Zhang, Fei Wu , Member, IEEE, Kai Tao, Feng Zhu, Shu Li , Yahui Zhao, and Changsheng Xie , Member, IEEE

Abstract‚ÄĒThree-dimensional (3D) NAND flash memory has been frequently utilized in consumer electronics as a popular storage device. However, data reliability has become an important problem to be solved. Low-density parity-check (LDPC) codes with superior error correction capability are commonly used in 3D NAND flash memory to ensure data reliability. Unfortunately, high raw bit error rate (RBER) induced by retention time and program/erase (P/E) cycles leads to increased the number of decoding iterations, failing to correct bit errors. Consequently, data reliability cannot be well ensured by using conventional LDPC coding. Moreover, the number of decoding iterations between layers fluctuates greatly due to process variation, which leads to a large difference in decoding latency. To reduce and shorten the gap of inter-layer decoding iterations, this paper proposes ALCod: an adaptive LDPC coding scheme for 3D triple-level cell (TLC) NAND flash memory by exploiting the inter-layer RBER variation. Specifically, this paper first conducts a preliminary experiment, which shows that RBER and the number of decoding iterations between layers and pages have a great difference. And, high RBER induces decreased decoding performance, thus introducing more decoding iterations and time consumption. Then, inspired by these findings, ALCod adaptively performs LDPC coding operations according to the RBER variation induced by retention time and P/E cycles. For pages and layers with higher RBER, by using ALCod, the original bit sequence is split into two parts on average and encoded separately. During decoding, known bits of 0 information are used to improve the initial decoding information (i.e., loglikelihood ratio (LLR) information per bit). To help with the decoding of the codeword‚Äôs unknown information, the LLR amplitude of known bits of 0 is increased. The known information and unknown information participate in the same check equation, which, according to the LDPC decoding principle, can provide useful LLR information to speed up decoding update

I. INTRODUCTION
N AND flash memory has been widely utilized in consumer electronics as the main storage device, such as smartphones and tablet computers (i.e., ubiquitous in our lives) [31], [32]. It has great benefits of small size, low power consumption, high performance, huge capacity, and high storage density [33], [34]. Particularly, three-dimensional (3D) NAND flash memory has a large capacity and high storage density, taking use of the multi-layer structure and multiple bits per cell, which lowers the cost per bit in consumer devices [1], [35]. Due to the several benefits listed above, it is currently the subject of considerable interest in the consumer electronics sector. However, inter-layer interference, retention, and program/erase (P/E) errors have made data reliability a major issue [2]. To ensure data reliability, low-density paritycheck (LDPC) codes as mainstream error correction codes (ECC) are commonly used in 3D flash storage systems [3]. LDPC decoding used in flash memories includes hard sensing decoding (HSD) and soft sensing decoding (SFSD) which have different error correction capabilities. For short retention time and a small number of P/E cycles, the raw bit error rate (RBER) is less, and HSD can satisfy the error correction requirement. Unfortunately, as retention time and P/E cycles gradually increase, the growing RBER exceeds the error correction capability of HSD, leading to decoding failure and more decoding iterations. As a result, data reliability cannot be well guaranteed by using traditional coding approaches, leading to data loss. Especially, for 3D NAND flash, the number of decoding iterations between layers fluctuates significantly due to process variation, introducing a large gap of decoding latency. Subsequently, SFSD with stronger error correction capability needs to be leveraged. More interestingly, although SFSD can improve data reliability, read levels are significantly increased due to fine-grained memory sensing by applying multi-read voltages [4]. Consequently, it is necessary to design more efficient coding schemes to avoid the use of SFSD.
To take full advantage of HSD and reduce decoding latency, this paper proposes ALCod: an adaptive LDPC coding scheme

Specifically, aiming at pages and layers with higher RBER, ALCod divides the raw information sequence into two groups on average and encoded separately. For pages and layers with lower RBER, the normal coding is performed. In the process of decoding, if decoding the page with higher RBER, ALCod adjusts the initial decoding information (i.e., loglikelihood ratio (LLR) information per bit) based on the known bits of 0. Conversely, if decoding the page with a lower RBER, a conventional layered decoding algorithm is launched by assigning LLR with the same magnitude.
For HSD, the hard sensing result only involves bits 0 and 1. With traditional LDPC coding, the initial LLR information is obtained by mapping 0 to 1 and 1 to ‚Äď1, only changing in sign but not magnitude, which results in poor decoding performance. The hard sensing result for ALCod is split into two sequences, each of which contains a portion of known bits of 0. The initial LLR information of known bits of 0 is adjusted to raise the magnitude, which has relatively high accuracy by changing both the sign and the magnitude. ALCod can improve LDPC HSD performance and data reliability. The effectiveness of ALCod is verified by simulation experiments. Results show that the decoding success ratio is improved. Decoding iterations, latency, decoding layers, BER, and uncorrectable bit error rate (UBER) are reduced.

II. BACKGROUND AND RELATED WORK
A. Basics of Flash Memory
According to the number of storage layers, flash memory is divided into two-dimensional (2D) and 3D stacked flash memories. 2D flash blocks with the plane structure are composed of word lines (WL) and bit lines (BL), as shown in Fig. 1 (a). 3D stack flash blocks with the three-dimensional structure consist of multiple storage layers, and each layer contains multiple WLs, as shown in Fig. 1 (b). The source selective line (SSL) and drain selective line (DSL) in 2D and 3D flash memories control data write, read, and erase operations. Moreover, 2D flash cells are the main type of floating gate (FG). 3D flash cells are the main types of FG and charge trap (CT). The distinct difference is that FG and CT cells use polysilicon (i.e., a conductor) and silicon nitride film (i.e., an insulator) as the storage material, respectively.
According to the number of bits stored per cell, 2D and 3D flash memories are split into the types of single-level cell (SLC), multi-level cell (MLC), triple-level cell (TLC), and quad-level cell (QLC). Each cell of SLC, MLC, TLC, and QLC stores 1 bit with 2 states, 2 bits with 4 states, 3 bits with 8 states, and 4 bits with 16 states, respectively. For 3D

C. Related Work
There are many previous works proposed to optimize LDPC codes for flash memory. Optimization of RBER is beneficial to improve LDPC decoding performance. Aslam et al. [5] studied the read reference voltage adjustment and data recovery scheme that can reduce the RBER. Papandreou et al. [10] explored the bit error characteristics of 3D QLC flash memory and proposed a calibration strategy of read voltage based on page recombination to lessen the RBER. Li et al. [11] used the reserved information to adjust and optimize the read reference voltage of 3D QLC, but this method can only approximate the optimal read reference voltage through the pre-stored information, which will inevitably lead to the readretry operation. Guo et al. [8] proposed the Flexlevel scheme to reduce the RBER by reducing the data storage state and expanding the noise tolerance area. When data are written into a flash memory cell, different storage states are affected by

programming interference. According to the access characteristics of data, the storage state is remapped [13] to reduce the RBER. In addition, data caching [6] also can reduce the RBER. The decoding delay can be significantly reduced by optimizing the LLR information. Cui et al. [14] optimized the LLR information according to the influence of P/E periods and retention noise on the threshold voltage distribution. Sandell and Ismail [17] used the machine learning algorithm to calculate the LLR information. Although this method can optimize the LLR information, it increases additional computational complexity and overhead. In the process of acquiring LLR information, the read reference voltage needs to be applied several times, which increases the detection delay. Optimizing the use of read reference voltage can reduce the detection delay, thus improving the LDPC read performance. Li et al. [18] proposed a new reading reference voltage placement method to reduce detection delay based on the asymmetric error characteristics of flash memory cells. Du et al. [4] took advantage of the reliability difference of the 3D flash storage layer to record the initial read reference voltage applied in each layer to reduce the detection overhead. Yong and Chang [19] proposed an error dilution scheme to reduce detection delay by using the feature of RBER between different layers of 3D flash memory. Sun et al. [9] observed that intracell unbalanced bit error probability and data dependence exist in planar MLC NAND flash memory. By utilizing the observation, [9] adopted data interleaving to balance intracell bit errors of planar MLC NAND flash memory. Unlike this method, we divide the original information into two parts equally for encoding and decoding based on the variation of RBER induced by 3D TLC NAND flash storage layer, page type, retention time, and P/E cycles. Besides, [9] proposed data dependence aware LDPC decoding to enhance decoding throughput. Different with this approach, we fill in some bits of 0 as known information in the segmented sequence. Additionally, we leverage the known bits of 0 information with an improved LLR amplitude to aid in the decoding of the unknown bit information in the codeword during decoding. As a result, the error correction capability of LDPC codes with HSD is dramatically improved and the decoding iterations are drastically decreased.
Previous works mainly focused on optimizing the read reference voltage to reduce the RBER and LLR information. Compared with these prior works, this paper does not consider optimizing read reference voltage to improve LLR and reduce RBER, and will not introduce additional read operations. In addition, this paper mainly aims to improve the error correction capability of LDPC hard decoding and avoid soft decoding with high read overhead effectively, especially for 3D NAND flash memories. According to the variation of RBER, the proposed ALCod can adaptively execute LDPC coding operations. The implementation is simple and LDPC decoding performance can be significantly enhanced. More importantly, only performing HSD can reduce bit errors and improve data reliability. More interestingly, this paper exploits known bits of 0 information in ALCod to facilitate decoding for unknown information, further upgrading LDPC error correction capability with HSD.

III. MOTIVATION We test 3D TLC NAND flash memory chips with floating gate structure using a real FPGA hardware platform and the chip information is shown in Table I. During the experiment, commands to program, read, and erase blocks with a given physical address are issued. Randomly generated data can be written to the selected page while completing the program operation. When the read operation is conducted, the raw data are read out to count RBER. Direct access to flash memory media is possible. The testing platform includes the main controller, the peripheral component interface express (PCIe) interface, and eight sockets for attaching eight flash chips. The firmware on the testing board primarily provides certain basic operating interfaces to the host, such as block erasure, page program, and page read. These interfaces can be used by the host to perform the necessary operations on the flash memory chip. In the retention experiment, we use high-temperature baking to reduce real testing time. The raw data are retrieved in three steps: 1) select physical blocks with a specific amount of P/E cycles; 2) bake the chip at a high temperature (i.e., 85‚ó¶C) once the data have been written into the physical blocks; and 3) collect the data under a certain number of P/E cycles and retention periods. By the real platform experiments, we first investigate the variation of the inter-layer RBER under different retention time and P/E cycles. We then study the influence of the inter-layer RBER variation on conventional LDPC decoding iterations, UBER, and BER. With the increase of retention time and P/E cycles for 3D NAND flash memory, RBER increases gradually. More interestingly, RBER shows a larger difference between storage layers due to process variation. Besides, it also demonstrates a larger gap between pages. Consequently, LDPC codes are exploited to reduce bit errors and thus improve data reliability, displaying a larger gap for decoding iterations between layers and pages. Especially, for some layers and pages, decoding iterations become extremely high due to much higher RBER, getting to the maximum value and thus failing to ensure data reliability. This paper concentrates on improving LDPC decoding performance and therefore cutting down the decoding iterations and latency. IV. THE PROPOSED ALCOD SCHEME
This section presents the proposed ALCod scheme for 3D NAND flash memory in detail.
A. ALCod Overview
The overview of ALCod is illustrated in Fig. 5 including encoding and decoding. In the process of encoding, the RBER is considered to select the appropriate encoding involving Norm LDPC and segmentation (Sege) LDPC encoding. When decoding, Norm and Sege decoding are adaptively implemented according to the variation of RBER.
The raw bit information sequence with length N is divided into two bit information sequences of equal length by ALCod when RBER is much higher (i.e., beyond the threshold). The two bit sequences are each filled with N/2 bits of known information, as shown in Fig. 5. Utilizing LDPC codes, two full bit information sequences of length N are encoded separately. To enable error correction during decoding, more bit errors are dispersed throughout the two codewords. Compared to traditional LDPC error correction techniques, the pressure of error correction is greatly reduced. Moreover, the bit errors in a single codeword fall within the range of LDPC codes with HSD error correction. And, the filled N/2 known bits of 0 are fully utilized to aid in the decoding of the information pertaining to the unknown bit. Due to the fact that known information and unknown information are both a part of the same check equation, it can offer useful LLR information to speed up decoding update for unknown information. B. ALCod Encoding
When performing ALCod encoding for raw information bits with length N, the RBER is judged by exploiting the variations of page types, 3D storage layers, P/E cycles, and retention time. Before being used, 3D NAND flash chips are randomly sampled and rigorously tested by manufacturers to determine their reliability and performance. A crucial criterion for measuring reliability is the RBER, which is examined and tested across various page types, 3D storage layers, P/E cycles, and retention times. The specific RBER value (e.g., 6.5 √ó 10‚ąí3) can be recorded for a certain page type, 3D storage layer, P/E cycles, and retention period. This is due to the fact that before running ALCod, it is required to ascertain whether RBER exceeds the cutoff point (i.e., 6.5 √ó 10‚ąí3 described in Section III) based on page type, 3D storage layer, P/E cycles, and retention period. If the RBER is less than the threshold (thre), the Norm LDPC encoding is executed to generate redundant bits (RB). Information bits together with RB make up a complete codeword. It is then written to the flash page. If the RBER is larger than thre, Sege LDPC encoding is launched. Especially, raw information bits with length N are split evenly into two pieces with length N/2. When encoding, two complete sequences of information with

Fig. 6. An example for explaining the Sege LDPC encoding process.
length N are produced by the complement of 0 with length N/2, as shown in Fig. 5. Two new information sequences are encoded in parallel, generating the corresponding RB and codewords. When programming, two codewords are combined to store in the 3D NAND flash page. We take an example to clearly explain the Sege LDPC encoding process, as shown in Fig. 6. Assume that raw bit information is 110101011001 with length 12. It is evenly divided into two parts, each filled with six zeros, to form an information sequence of length 12. As a result, an original information sequence is divided into two information sequences 110101000000 and 000000011001, which are Sege LDPC encoded, thus generating the corresponding RB. Two code words are combined into a complete sequence of information written to the same flash page.
In the beginning, according to Fig. 3, retention time is short and P/E cycles are less, RBER is much lower, all pages are encoded by the normal LDPC process. However, with the increase of storage time and P/E cycles, some pages become weak pages with higher RBER and need to trigger ALCod coding to ensure reliability. ALCod is activated based on whether the RBER exceeds the threshold. Based on the workload access characteristics, we can determine how long the data will be saved once it is encoded [4]. The storage duration for hot read data is usually short. Cold read data is typically kept on hand for a long period (e.g., six months) [7]. When this period has passed, data refresh procedures are performed to lower the error rate. However, premature refresh triggers can shorten flash memory lifetime and increase write amplification [4]. The controller periodically keeps track of the retention time, P/E cycles, storage layer, and page type information (i.e., based on address mapping information) to assess the health state of the flash memory. The information can be used to the ALCod coding design.

A. Evaluation Methodology
ALCod is evaluated via computer simulations, which mainly involve data encoding, adding noise, and decoding. First, randomly generated data of 2KB are encoded by using Norm LDPC codes to produce the codeword. We then add Gaussian noise to the codeword. As a result, because of interference from noise, some bits are error-prone. The RBER is calculated by comparing it with the original information. LLR information per bit in the interfered codeword can be gained by mapping bit 0 to 1 and mapping bit 1 to ‚ąí1. Once obtaining the initial LLR information, the LDPC decoding is activated by using the layered min-sum algorithm, which is widely used in NAND flash memory [15]. When the decoding is complete, the number of decoding iterations, the decoding delay, BER, and UBER are counted under a given RBER.
To prove the validity of ALCod, we encode the randomly generated data of 2KB by exploiting the proposed Sege LDPC coding. The same signal-to-noise ratio is added to the generated codeword to compare Norm and Sege LDPC coding error correction capability at the same RBER. LLR information is optimized by using known bits of 0 information, as described in Section IV-C. Specifically, bit 0 in the unknown information is mapped to 1, bit 1 is mapped to ‚ąí1, and known bits of 0 information is mapped to 5 and 10. The LLR mapping for the known bits of 0 information is related to the decoding decision rule given in Section II-B. The decision rule is defined as follows. If the LLR of the variable node Tj in the formula (3) is less than 0, the decoding result is judged as bit 1. Otherwise, the decoding result is judged as bit 0. Therefore, the LLR mapping for the known bits of 0 information corresponds to the decoding decision rule. Known bits of 0 information is fully utilized to speed up the decoding of unknown information. The LLR information amplitude value is adjusted. By using ALCod, LLR not only has the changes of symbol but also the changes of amplitude value, which can greatly improve the decoding error correction capability. The decoding algorithm is also the layered min-sum algorithm. We compare the number of decoding iterations, decoding delay, BER, and UBER under the same RBER. We use the Gaussian error model to simulate the RBER variation for verifying the effectiveness of ALCod, which has been proved that flash channel approximates Gaussian channel [1], [9], [14], [30].

D. Overhead Analysis
Buffer overhead analysis: Usually, a sequence of length N is encoded by LDPC codes with code rate of N/(N + R). R redundant bits are generated to form a codeword of length N + R. After that, it is stored in the buffer, then transferred and written to the 3D TLC NAND flash page. For ALCod, a sequence of length N is split two N-bit sequences denoted N1 and N2 with N/2 known bits of 0, respectively. After performing ALCod encoding with code rate of N/(N +R), N/2 known bits of 0 do not need to be stored in the buffer. The total length stored in the buffer is N + 2R. After the sequence of length N+2R is written to the 3D TLC NAND flash page, it is cleared from the buffer. It needs additional space to store R redundant bits. Assume that the size of the buffer is 8GB = 236 bit, the LDPC code rate is 8/9 used in the paper, the sequence of length N = 2KB is encoded. R = 2048 redundant bits are generated. The ratio of the additional buffer space to the total buffer space is 2048/236 = 1/225. Therefore, the buffer overhead is negligible.
For 3D TLC NAND flash memory, assume that the maximum retention time is 6-month and the maximum P/E cycle is 5K. We only record the information of page type, 3D storage layer, retention time, and P/E cycle when RBER gets to the threshold (i.e., 6.5 √ó 10‚ąí3). For one 3D TLC block, only some layers and pages reach the RBER threshold when retention time is less than or equal to 6-month and the P/E cycle is less than or equal to 5K. For example, when retention time is less than or equal to 3-month and the P/E cycle is less

than or equal to 5K, the EP RBER of 6 layers (i.e., 2nd, 3rd, 7th, 8th, and 9th layer) can reach the threshold, as shown in Fig. 3 (c), which is record in the buffer. The UP RBER of only 2nd layer can reach the threshold. When retention time is less than or equal to 6-month and the P/E cycle is less than or equal to 5K, the LP RBER of 3 layers (i.e., 2nd, 3rd, and 63rd layer) can reach the threshold, as shown in Fig. 3 (d). The UP RBER of 36 layers (i.e., from 2nd to 23rd layer and from 54th to 63rd layer) can reach the threshold, as shown in Fig. 3 (e). The EP RBER of 33 layers (i.e., from 2nd to 25th layer, from 33rd to 35th layer, from 56th to 58th layer, 42nd layer, 62nd layer, and 63rd layer) can gets to the threshold, as shown in Fig. 3 (f). When limiting the maximum retention time and P/E cycle, the RBER in other cases does not exceed the threshold and does not need to be recorded. It only needs to record the information that may exceed the threshold. When encoding and decoding, it just needs to look up the table to see if the RBER exceeds the threshold. The retention time is recorded using 3 bits, the page type is recorded using 3 bits, the number of layer is recorded using 7 bits, the RBER threshold is recorded using 7 bits, and the P/E cycle is recoded using 13 bits.
For one block, when retention time is less than or equal to 3-month and the P/E cycle is less than or equal to 5K, only the EP RBER of 6 layers, UP RBER of 2nd layer, the corresponding retention time, the P/E cycle, layer, and page type are required to store, as explained above. The needed extra buffer space is 6 √ó (6 + 3 + 13 + 7 + 7) + (6 + 3 + 13 + 7 + 7) = 252 bits. Similarly, when retention time is less than or equal to 6-month and the P/E cycle is less than or equal to 5K, only the LP RBER of 3 layers, UP RBER of 36 layers, EP RBER of 33 layers, the corresponding retention time, the P/E cycle, layer, and page type are required to store, as explained above. The needed extra buffer space is (3 + 36 + 33) √ó (6 + 3 + 13 + 7 + 7) = 2592 bits. For one block, there are 2844 (i.e., 252 + 2592) bits required to store. For one chip with 8 √ó 504 blocks, the extra buffer space needed is 2844 √ó 8 √ó 504 bits (the chip information is given in Table I), which is about 1.37MB. The buffer overhead is acceptable for gaining the benefits of reliability and latency.
Storage overhead analysis: The storage cost of ALCod depends on the RBER. In the early stage of 3D TLC NAND flash memory (i.e., less P/E cycles and shorter retention time), the RBER is lower and thus it is not required to launch ALCod. Norm LDPC coding can ensure data reliability, without introducing additional overhead. In the late stage of 3D TLC NAND flash memory, the RBER is much higher, which may be beyond the error correction capability of Norm LDPC coding. As a result, Sege LDPC coding needs to be activated. The storage space is sacrificed for obtaining the reliability benefit. Raw information is split to encode and thus it needs to store one more piece of redundant information.

The available space for user data is reduced for storing additional ECC information. The sacrificed codeword space of 2KB for one page is used to store additional ECC. The number of ECC codewords will not decrease on every page. The number of ECC codewords for pages with a higher raw bit error rate (RBER) that are only required to start ALCod is reduced. For 3D TLC NAND flash memory, RBER of each page varies greatly in different storage layers after a certain retention time and P/E cycles, as shown in Fig. 3 in the text. Not every page needs to sacrifice 2KB of user space to store additional ECC. The proposed ALCod scheme is only used for some pages with higher RBER (i.e., exceeds a certain threshold of explained in the text in Section III). When RBER is below a specific threshold, only normal LDPC coding needs to be enabled to ensure reliability with no storage overhead.
Next, we analyze the storage overhead when some pages with higher RBER ((i.e., exceeds a certain threshold)) are required to launch ALCod. When the retention time is 3 months and the P/E cycle is 5K, as shown in Fig. 3 (c) in the text, for EP, RBERs of 6 layers (i.e., 2nd3rd, 7th, 8th, and 9th layer) exceed the threshold. ALCod needs to be started to correct bit errors of EP with 6 layers. The extra space that is sacrificed is 2KB √ó 6 = 12KB. For UP, the RBER of only one layer (i.e., 2nd layer) exceeds the threshold. ALCod needs to be launched to correct bit errors of UP with 1 layer. The extra space that is sacrificed is 2KB √ó 1 = 2KB. For LP, the RBER of all layers is below the threshold, ALCod does not need to be started, and no additional overhead is introduced. A block has 2304 pages in it. The total block size is 41832KB (i.e., 2304√ó18592B). Thereby, after the above calculation, the additional storage space sacrificed is 12KB + 2KB = 14KB,

which takes up 0.03% (i.e., (12 + 2)/41832 √ó 100%) of a block storage space. When retention time is 6 months and the P/E cycle is 5K, as shown in Fig. 3 (d), for EP, RBERs of 33 layers (i.e., from 2nd to 25th layer, from 33rd to 35th layer, from 56th to 58th layer, 42nd layer, 62nd layer, and 63th layer) exceed the threshold. ALCod needs to be started to correct bit errors of EP with 33 layers. The extra space that is sacrificed is 2KB √ó 33 = 66KB. For UP, RBERs of 36 layers (i.e., from 2nd to 23rd layer and from 54th to 63rd layer) exceed the threshold. ALCod needs to be launched to correct bit errors of UP with 36 layers. The extra space that is sacrificed is 2KB √ó 36 = 72KB. For LP, RBERs of 3 layers exceed the threshold. ALCod needs to be launched to correct bit errors of LP with 3 layers. The extra space that is sacrificed is 2KB√ó3 = 6KB. Therefore, when retention time is 6 months and the P/E cycle is 5K, for EP, UP, and LP, the total amount of extra space sacrificed is 66KB + 72KB + 6KB = 144KB, which takes up about 0.34% (i.e., 144/41832 √ó 100%) of a block storage space.
To reduce access overhead, we put additional ECC information in the same data page along with the information bits. If the read request is 2KB, either sequential or random reads, it does not need to read data across pages. This is because both the 2KB user data and the 0.5KB ECC parity are stored in the same pages. For sequential read requests of 4KB, the data in each page are read sequentially. The proposed ALCod does not affect performance, having obvious advantages. Sequential read requests have been found to be useful for enhancing flash system performance in earlier work [37]. For random read requests of 4KB, one page may include 7 pieces of 2KB, it needs to read an additional of 2KB across other pages to compose a 4KB of data. In this case, partial page reads (i.e., 4KB read) can be used to minimize the performance penalty. It takes an extra partial page read time (i.e., about 45us) to get an extra 2KB.
When 3D TLC NAND flash memory experiences heavy wearing and long time storage, RBER becomes higher and reliability cannot be guaranteed by using the traditional error correction method. Data loss may occur, which is considered as an important issue that must be guaranteed. Compared with the traditional error correction method, the ALCod scheme can significantly improve the reliability. However, some buffer and storage space as well as some performance may be sacrificed for obtaining the reliability improvement. After the above analysis, there is a partial performance penalty for only 4KB random read requests. For 2KB sequential or random read requests, the proposed ALCod does not incur a performance penalty. After buffer overhead analysis in Section V-D, the buffer overhead is minimal and acceptable for the reliability improvement. Moreover, for the space overhead, after the above analysis, when retention time is 3 months and the P/E cycle is 5K, the additional storage space sacrificed takes up 0.03% of a block storage space. When retention time is 6 months and the P/E cycle is 5K, the additional storage space sacrificed takes up 0.34% of a block storage space. Very little storage overhead and performance penalty can be exchanged for a significant reliability improvement.

VI. CONCLUSION
This paper proposes ALCod for 3D TLC NAND flash memory to reduce Delter, Delayer, decoding time, BER, and UBER by enhancing LDPC error correction capability with HSD at higher RBER. To efficiently improve LDPC decoding performance and thus ensure data reliability, ALCod implements adaptive LDPC coding operations based on the variation of RBER. When the RBER is higher, ALCod launches Sege LDPC coding. Conversely, ALCod activates Norm LDPC coding. As a result, decoding latency is greatly decreased and decoding performance is significantly improved. To further upgrade error correction capability, ALCod exploits known bits of 0 information to optimize LLR, facilitating decoding

convergence for unknown information. Simulation results
show ALCod can greatly decrease Delter, Delayer, decoding
time, BER, and UBER.
