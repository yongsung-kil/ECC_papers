LVLDPC: Intra-Layer Variation Aware LDPC
Coding for 3D TLC NAND Flash Memory
Lanlan Cui†, Meng Zhang∗‡ and Fei Wu∗§ †School of Computer Science and Engineering, XI’AN University of Technology, Xi’an, China ‡ School of Computer Science and Technology, Huazhong University of Science and Technology, Wuhan, China § Wuhan National Laboratory for Optoelectronics, Huazhong University of Science and Technology, Wuhan, China
∗Corresponding author: Meng Zhang, Fei Wu, {zgmeng, wufei}@hust.edu.cn cuilanlan@xaut.edu.cn, {zgmeng, wufei}@hust.edu.cn

Abstract—NAND flash memory employs high code rate lowdensity parity-check (LDPC) code to minimize redundant data. High code rates reduce redundancy but compromise error correction compared to medium/low code rates. Raw bit error rate (RBER) varies among the storage layers for 3D triple-level cell (TLC) NAND flash memory, which causes the number of using read retry to increase. Repeatedly initiating read retry seriously increases the decoding latency and decreases the performance of the 3D TLC NAND flash memory. To alleviate this problem, this article proposes Intra-Layer Variation aware LDPC coding, called LVLDPC. The LVLDPC scheme categorizes RBER into distinct levels. Then, we select LDPC codes with appropriate error correction capabilities to decode data with varying levels of RBER. Through this scheme, we don’t need to start read-retry when RBER < 1.56 × 10−2. The iteration number is reduced by 67% in total. This scheme only causes 1.15% space overhead, which is negligible.
Index Terms—3D TLC NAND flash, LDPC, RBER, code rate
I. INTRODUCTION
NAND flash has been widely used in various consumer products such as mobile phones, cameras, and computers, to store data [1]. With constant technology scaling, more and more bits are stored in one NAND flash cell, resulting in a sharp increase in raw bit error rate (RBER). Error correction code (ECC) can effectively improve the data reliability, thus extending the NAND flash lifetime. Low-density parity-check (LDPC) codes [2] as the popular ECC, are commonly used in 3D triple-level cell (TLC) NAND flash due to their better performance than the traditional Bose-Chanudhuri-Hocquenghem (BCH) codes.
For 3D TLC NAND flash memory, the generation of interlayer interference makes RBER more elusive. Unlike errors caused by program/erase (P/E) cycles and retention time, inter-layer interference does not have strong regularity but appears randomly under different circumstances. The code rate selection for LDPC should be based on RBER. Interlayer interference introduces significant complexity into the RBER, exhibiting severe fluctuation across various storage layers, characterized by notable disparities in RBER. Consequently, the application of a uniform code rate LDPC decoding

scheme poses a challenge in effectively accommodating these variations in RBER. It is important to choose the code rate that matches RBER. For LDPC, error correction capability and redundant space are two contradictory factors that are difficult to satisfy simultaneously. In general, to store more effective information in TLC NAND flash, the code rate of LDPC is high, such as 0.9. The management module of the TLC NAND flash memory chip implements the read retry scheme. When the RBER is high, the LDPC code may not be able to fully correct errors, triggering the activation of the read retry scheme. This results in re-reading and decoding data stored in TLC NAND flash memory. If the decoding is still incorrect, this process will continue until either correct decoding is achieved or the maximum number of read retry is reached. Although the redundancy space of high code rate LDPC is small, the error correction capability of LDPC is not enough, which leads to an increase in the number of using read retry when facing high RBER. Once read-retry is activated multiple times, the decoding latency is seriously increased and affects the read performance of the TLC NAND flash memory.
To solve the above issue, much research was proposed. Some studies were based on the design of LDPC codes [3]. Another type was about decoding algorithms. LAE-FTL was proposed in [4], which adopts a lifetime-adaptive ECC scheme, using weak ECC in the early stage and strong ECC in the later stage to improve the performance and ensure storage reliability. Literature [5] used the clustering algorithm to classify the bit error rates of different pages on the same word line of the 3D TLC NAND flash memory into different categories. When performing error correction, LDPC codes of different code rates can be selected to save storage overhead. Wu et. al [6] developed the LDPC decoding strategy to optimize the loglikelihood ratio (LLR) information based on the error modes of 3D TLC NAND flash. However, these researches do not consider the differences between different layers of 3D TLC NAND flash memory.
Motivated by the above analysis, this article proposes an Intra-Layer Variation aware LDPC coding scheme, called LVLDPC. First, we test RBER under various P/E cycles, retention time, and inter-layer interference. And RBER can be divided into different levels. Then, we select the LDPC code

II. BACKGROUND
A. 3D NAND Flash
Each TLC NAND flash cell represents three bits of information, namely the Least Significant Bit (LSB), the Center Significant Bit (CSB), and the Most Significant Bit (MSB). 3D NAND flash stacks multiple layers vertically to overcome the capacity limitations of 2D NAND flash. This architecture allows for higher density without sacrificing data integrity, thus enabling more storage capacity. While the 3D stacking technology also poses several challenges. Firstly, it introduces greater complexity to the NAND flash memory structure and increases channel noise diversity, thereby posing a heightened threat to data reliability. Secondly, due to the unique production process characteristics of 3D NAND flash memory, there exist variations in the physical structure of storage units across different layers. These variations manifest as significant differences in the RBER among layers, which is a key metric for evaluating data reliability in NAND flash memories.
Despite the increased storage capacity, data errors are becoming more prevalent. As the threshold voltage distribution of each state shifts and widens, raw bit errors arise during read operations. The article primarily investigates three prevalent factors that contribute to data storage errors in 3D TLC NAND flash, namely P/E cycle, retention time, and inter-layer interference. The increasing number of P/E cycles causes electrons to be trapped in the tunnel oxide. The accumulation of trapped electrons results in wear-out, which in turn leads to errors in the data stored in 3D TLC NAND flash. Retention errors are another major source, resulting from charge leakage over time after a 3D TLC NAND flash cell is programmed. The inter-layer interference is unique to the 3D stacking structure. This type of error is caused by the interference between adjacent layers during the programming process. If the pattern of variation in errors cannot be tracked, numerous data errors will occur during reading and programming operations.
B. LDPC Decoding
LDPC decoding includes hard-decision decoding and softdecision decoding. The two ways complement each other to meet the error correction needs of NAND flash. To ensure the data reliability of 3D TLC NAND flash, the read retry scheme is implemented. When a data error occurs during the read process, the hard-decision decoding procedure is initiated first. Hard-decision decoding involves applying a single read reference voltage during data retrieval. This approach is straightforward to implement as it relies solely on symbol features. However, its error correction performance is relatively limited. Another advantage of hard-decision decoding is that the decoding convergence speed is fast and the decoding latency is small. When RBER is high enough to exceed the

ability of hard-decision decoding, the soft-decision decoding starts.
If the hard-decision decoding succeeds, the decoding process ends, and the result is outputted. Conversely, if the error persists after hard-decision decoding, the soft-decision decoding method is activated. Soft-decision is realized by applying multiple read reference voltages to read data and decoding multiple times. Soft-decision decoding has a good ability to correct errors, but adopting soft-decision decoding results in significant read latency. If hard-decision decoding can be improved to enhance its error correction capability, the threshold for initiating soft-decision decoding can be raised, thereby postponing the activation of soft-decision decoding. Therefore, the overall latency is reduced directly.
III. THE PROPOSED SCHEME
1) Test Model: As described in the motivation section, we test RBER under different P/E cycles, retention time, and layer numbers for three types of pages. The retention time can be reduced by baking the chip where the data is written at a high temperature. The test is based on a real FPGA platform. RBER is obtained by randomly writing data and reading the disturbed data.
In addition, in order to select an appropriate code rate conversion scheme, we need to know the error correction performance of LDPC with different code rates. Here, the error correction capability of LDPC code refers to the highest RBER that errors of all frames can be corrected. We use 20000 frames to test the correctable RBER upper limit for different code rates. Use the LDPC with a special code rate for decoding, as the RBER decreases, the decoding error rate becomes lower and lower. When no errors occur in all frames, the RBER is the error correction limit we are looking for. This means that when RBER is less than this value, all errors can be corrected.
We show the error correction capabilities (written as ”L”) of different code rates in Fig 1. We take the position of FER equal to 1 × 10−4 as the division standard to measure that the data is no longer wrong after decoding, that is when the decoding result is less than 1 × 10−4, we believe that the data is no wrong. It can be seen that the RBER is divided into six parts, and each part corresponds to a code rate interval. The five code rates are 0.9, 0.85, 0.8, 0.75, 0.7, from left to right. These code rates are selected because they can correct most of the errors that occur in the current test, and it is meaningless to further reduce the code rate. In Fig 1. it is observed that when the LDPC code rate is set to 0.85, the limit of RBER that can be corrected is approximately 0.01. That is to say, for RBER values below 0.01, the LDPC code with 0.85 code rate is capable of correcting all errors incurred.
2) Code Rate Selection: Based on the statistical analysis we can determine the comprehensive range of RBER values. Subsequently, within this range, we categorize the RBER by the error correction capabilities of LDPC codes possessing varying code rates. The meaning of the shorten algorithm is to reduce the length of information bits in the codeword, thereby reducing the code rate, that is the length of the codeword involved in transmission is shortened accordingly. Assuming the original code length is Sn, the length of the information bit is Sk, and the length that shorten technique needs to handle is Sx , then code rate after shorten operation is (Sk − Sx)/(Sn − Sx). Specifically, the shorten operation involves modifying a portion of the message input to the decoder by setting it to 0 before decoding. This approach serves to limit the propagation of information, thus effectively reducing the code rate. By leveraging this method, we only require the storage of the H matrix corresponding to a code rate of 0.9. Notably, in comparison to a single-rate scheme, our approach does not incur any additional overhead.

IV. EXPERIMENTAL RESULTS AND ANALYSIS
We employed the layer min-sum algorithm and set the maximum iteration number to 30. We selected the (18656, 16748) Quasi-Cyclic LDPC (QC-LDPC) code, renowned for its low implementation complexity. Additionally, a 6 bit quantizer is utilized. We validate the proposed scheme’s effectiveness under various P/E cycles (1000, 3000, 5000) and 6-month retention time, comparing it to the traditional scheme [7] with a fixed code rate of 0.9.
Utilizing LVLDPC, we select code rate with varying error correction capabilities for decoding across distinct RBER intervals. Notably, a single encoder/decoder suffices to manage all code rates, as the multi-code rate capability is achieved through shortening techniques. Fig. 3 presents a comparative analysis of the utilization of read retry. We employ LDPC with a single code rate as the baseline, which is called original scheme. As observed in the figure, when the RBER is less than 1.4 × 10−2, both schemes exhibit similar performance. However, as the RBER increases, the number of voltages required for both schemes escalates, yet the original scheme experiences a more rapid increase. Specifically, after the RBER surpasses 1.56×10−2, the original scheme necessitates the utilization of more than nine voltages. This study does not consider more voltage numbers due to the significant implementation latency involved. In contrast, the LVLDPC is capable of utilizing only three voltages for error correction, even under RBER exceeding 1.56×10−2. Through LVLDPC, when the RBER is below 2.51 × 10−2, we achieve a reduction in the number of starting read retry from over nine to two. Overall, LVLDPC effectively decreases the number of voltages used by 67% across the entire range.
When the RBER is elevated, LVLDPC generates redundant information, thereby occupying valuable information space. The evaluation of this overhead comprises two distinct components. First, we estimate the RBER that is uncorrectable

using traditional methods. Subsequently, we compute the storage space sacrificed as a result of this proportion. The total overhead is determined by multiplying the percentage of wasted space for each code rate by the percentage of code rate utilization across all decoding instances. Through rigorous calculations, we ascertain that in the current context, our increased overhead is (0.9 − 0.85) × 0.078 + (0.9 − 0.8) × 0.067 + (0.9 − 0.75) × 0.0054 + (0.9 − 0.7) × 0.0006 = 1.15%, which can be ignored.
V. CONCLUSION
In 3D TLC NAND flash memory, RBER becomes more complicated due to inter-layer interference. This paper proposes an Intra-Layer Variation aware multi code rate LDPC scheme for 3D TLC NAND flash. According to the correction ability of the LDPC code, the LDPC with stronger error correction ability is used when the RBER is high. Simulation shows that this scheme can reduce the inconsistency of the number of decoding iteration among different layers, and reduce the overall number of iterations by 67% with 1.15% overhead.
