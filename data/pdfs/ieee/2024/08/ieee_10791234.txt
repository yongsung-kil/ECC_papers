Evaluations of Distinct Bitonic Sorters for ORBGRAND

Szu-Hao Huang Department of Electrical Engineering
Yuan Ze University Taoyuan City, Taiwan 320, R.O.C.

Cheng-Hung (Dixson) Lin Department of Electrical Engineering
Yuan Ze University Taoyuan City, Taiwan 320, R.O.C
chlin@saturn.yzu.edu.tw.

Abstract—Guessing Random Additive Noise Decoding (GRAND) is a recently proposed decoding algorithm for forward error correction codes in fifth-generation (5G) consumer communications. Order Reliability Bits GRAND (ORBGRAND) was proposed to utilize received channel values and sort all of them to get more probable error patterns. Therefore, choosing an efficient algorithm to sort the channel values significantly influences the hardware design of the ORBGRAND. In this study, different bitonic sorters are evaluated in terms of frame error rate, synthesized silicon area, and power consumption. To achieve a low-cost ORBGRAND, the half-output bitonic sorter with segmentation not only reduces the number of the compare and swap unit, but also reduces number of the pipeline register. The evaluation results show that the half-output bitonic sorter with two segmentation can achieve 38% area reduction and 39% power reduction, respectively.
Keywords—5G, GRAND, Polar Code, Sorter
I. INTRODUCTION
Maximum Likelihood (ML) decoding is a decoding algorithm used in communication systems. It is a common technique for decoding. It is based on the principle of selecting the message that is most likely to be transmitted. In ML decoding, it considers all possible messages that could have been transmitted and calculates the probability of each one. The receiver then selects the message with the highest probability of the received signal and decodes it accordingly. ML decoding only considers the possibility of the received message, regardless of the encoding algorithm used by the transmitter. So, it can decode all types of linear block codes. It can be considered the universal coding for error correction code.
Polar code was published in 2009 [1]. It proposed a method called channel polarization to achieve the symmetric capacity of binary input discrete memoryless channel (BDMC) for channel coding. The 3rd Generation Partnership Project (3GPP) has standardized the use of polar codes for error correction in fifth-generation (5G) consumer communications, specifically for enhanced Mobile Broadband (eMBB) applications[2], [3] and low code rate for ultra-reliable low-latency communication (URLLC) [4], [5]. The URLLC has strict requirements for low latency and high reliability and has been applied to several applications, such as autopilot, intelligent grid, virtual reality, and more. The

The GRAND algorithm has been mentioned in [15] in detail. The GRAND has been proven to be able to reach or nearly match the performance of ML decoding [6]. The GRAND algorithm differs from other decoding algorithms, such as Polar codes and Low-Density Parity Check (LDPC) codes. While Polar codes [1] and LDPC [14] codes use encoding to encode the message to the codeword and employ the redundant message to protect the message, the GRAND operates distinctly. Instead of encoding the message, it focuses on guessing errors and generating the corresponding error pattern to correct the codeword. After generating an error pattern, the GRAND algorithm checks whether the correct codeword is a member of the codebook, as shown in Fig. 1. If it is a member of the codebook, it will be sent out, completing the decoding process. Since the GRAND algorithm does not require encoding, it can decode any linear block code. There are two types of decoding methods within the GRAND algorithm, distinguished by the method of error pattern generation: hard detection GRAND and soft detection GRAND. In hard detection GRAND, it assumes an equal likelihood of errors occurring for all bits. It tries all possible error patterns, ranging from the Hamming weight (HW) of one up to the Hamming weight of the code length, which also considers all possible bit errors. GRAND abandon (GRANDAB) sets a threshold to limit the maximum number of Hamming weights. Based on the method used to generate error patterns with channel values, there are various types of soft detection GRAND, such as Soft GRAND (SGRAND) [7], Order Reliability Bits GRAND (ORBGRAND) [8], List GRAND (LGRAND) [9], and so on. No matter what type of soft detection GRAND, all methods require an algorithm to sort the channel values and get a stack of indexes that channel values from lowest to highest. Subsequently, the H matrix column values is transported simultaneously.

II. ORBGRAND
An (N, K) arbitrary linear code can only be decoded by ORBGRAND if there is a codebook of the corresponding linear code. The N represents the codelength, and K represents the number of information bits. ORBGRAND uses Logistic Weight (LW) and HW to generate the error pattern. LW is represented as (1) and (2). The z represents the error pattern sorted according to the LLR from the minimum to the maximum.

IV. BITONIC SORTER
A. Original Method
A parallel sorter network is a type of sorting network that has parallel inputs and parallel outputs. It is used to get an order of the input that is from the minimum to the maximum. A sorter network is composed of a network of Compare and Swap (C&S) unit. Each C&S unit has two inputs and two outputs. Fig. 3 (b) shows the two types of the C&S unit. The blue block sorts the larger values to the bottom and the smaller values to the top. The green block sorts the larger values to the top and the smaller values to the bottom. There are lots of parallel sorting algorithms, such as bitonic sorter, Odd-Even sorter, and so on. For ORBGRAND, most of the hardware design used the bitonic sorter. A 16-input and 16output bitonic sorter is shown in Fig. 3 (a).

According to [11] and [12], the number of pipeline registers in the bitonic sorter and the segmentbitonic sorter are 7 and 5, respectively.
TABLE II shows the synthesis result of the different methods of the bitonic sorter. The method of segmentation shows superior performance in terms of area and power reduction compared to the half-output method. Not only does it decrease the number of nets, but it also reduces the number of pipeline registers. Specifically, the segmentation method and the half-output method yield area reductions of 14% and 24%, respectively, along with power consumption reductions of 15% and 16%, respectively. Furthermore, employing both segmentation and half-output techniques in the bitonic sorter results in a combined area reduction of 38% and a power reduction of 39%.
VI. CONCLUSION
To reduce the area, the segmentation and half-output method are used to the bitonic sorter. This study presents a comparative analysis of the two methods of the bitonic sorter for ORBGRAND to decode the CA-Polar. Both methods are designed to reduce the power consumption and area. The synthesis results show that both of the two methods can achieve significant reductions in power consumption and area. The area reduction of 38% and power reduction of 39%, respectively. The bitonic sorter that implements the two methods achieves superior performance compared to the bitonic sorter that implements only one method.
