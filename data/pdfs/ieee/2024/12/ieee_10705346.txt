Dual-Diagonal QC-LDPC Codes Free of Small Cycles From GCD Framework via Negative Coefficients
Guohua Zhang , Mengjuan Lou , Aijing Sun , and Yi Fang , Senior Member, IEEE

Abstract— A novel class of dual-diagonal (DD) quasi-cyclic (QC) low-density parity-check (LDPC) codes with girth eight is proposed. For the exponent matrix of such a DD-QCLDPC code, the left sub-matrix is explicitly generated via new formulae which are motivated by a search algorithm based on the greatest-common-divisor (GCD) framework and a new strategy allowing negative coefficients. The right sub-matrix (DD matrix) is designed by modifying the existing DD matrix while preserving the simple encoding property directly through the parity-check matrix. Performance simulations indicate that the proposed DD-QC-LDPC codes significantly outperform several existing counterparts with or without DD structure. Unlike existing formulae which involve only non-negative coefficients, the new formulae motivated by proposed search algorithm allow for negative coefficients, which lead to significantly improved lower bounds on circulant sizes for column weights from five to seven. As a result, the new DD-QC-LDPC codes not only enable simple encoding and low description complexity, but also possess consecutive and reasonably small circulant sizes, flexible sizes for exponent matrices as well as good decoding performance.
Index Terms— Dual-diagonal, girth, greatest-common-divisor (GCD), low-density parity-check (LDPC) code, quasi-cyclic (QC).
I. INTRODUCTION
LONG low-density parity-check (LDPC) codes can exhibit performance close to the Shannon limit by utilizing iterative decoding. Particularly, quasi-cyclic (QC) LDPC codes [1], [2], [3], [4] possess an additional feature of low encoding/decoding complexity. As a category of QC-LDPC code important for both theoretical analysis and practical standards, dual-diagonal (DD) QC-LDPC codes (DD codes for short) [5], [6], [7], [8], [9], [10] can further reduce encoding complexity by allowing simple computations directly on parity-check matrices. On the premise that no or little searching is required, how to generate DD codes with flexible lengths and flexible rates as well as good performance, is an interesting and challenging problem. Obviously, for a DD code uniquely determined by a J′×L′ exponent matrix and a circulant size P , its length N is P · L′ and the designed rate R is (L′ − J′)/L′. In addition, it is commonly believed that eliminating hazard structures (such as short cycles) caused by exponent matrices

According to the search complexity, the methods of constructing DD codes may fall into two categories: highcomplexity methods [5], [9] and low-complexity methods [6], [8], [10]. All of these methods can produce well-performing codes under some parameters, but we only focus on lowcomplexity methods here. The MDS-code-based method [6] can offer DD codes without any search, but the circulant sizes are limited to prime powers minus one and only 4-cycles are avoided. The approach in [8] allows the left sub-matrix of the exponent matrix to be explicitly obtained by the greatest-common-divisor (GCD) framework [14], but the right sub-matrix needs searching for the value at a single particular position to ensure that the girth is eight. The scheme in [10], also designed from the GCD framework, can provide girtheight DD codes without any search, but the number of rows for exponent matrices is limited to seven. By introducing both modified right sub-matrices and negative coefficients for the GCD framework, a new method is proposed which combines the advantages of the aforementioned three methods, explicitly producing well-performing girth-eight DD codes with both flexible circulant sizes and flexible sizes for exponent matrices.
II. PRELIMINARY
A. QC-LDPC Codes and Exponent Matrices
A binary (J, L) LDPC code is defined as the null space of a binary sparse parity-check matrix (PCM) H, where each row has L “1” elements and each column has J “1” elements. The PCM of a QC-LDPC code is composed of a J′ × L′ array of P × P circulants. Only weight-0 or weight-1 circulants are considered here. A weight-0 circulant is an allzero P × P matrix, which is associated with the exponent ∞ (by convention). A weight-1 circulant is also called a circulant permutation matrix (CPM), which is associated with an exponent e (being an integer) and is obtained by cyclicly shifting each row of an identity matrix of size P × P to the right by e times. Therefore, the PCM of a QC-LDPC code can be compactly represented by the circulant size P and an exponent matrix E of size J′ × L′, where J′ ≥ J and L′ ≥ L. The length of shortest cycles in the Tanner graph corresponding to E and the circulant size P , is the girth of a QC-LDPC code. Consequently, a QC-LDPC code with a target girth can be obtained by directly designing a suitable exponent matrix E for a given circulant size.

B. GCD Framework
The GCD framework [14], [15], [16], [17], [18] can efficiently generate girth-eight QC-LDPC codes. If an ascending sequence S2 = (s0, s1, · · · , sJ−1) satisfies the inequalities (sc − sa)/gcd(sc − sa, sb − sa) ≥ L for all triples (sa, sb, sc), where 0 ≤ a < b < c ≤ J − 1, the sequence S2 is said to meet the GCD constraint [14]. Then the resultant exponent matrix E = S2T · S1 = [s0, s1, · · · , sJ−1]T · [0, 1, · · · , L − 1] corresponds to a QC-LDPC code with girth exactly eight for any circulant size P ≥ (sJ−1 − s0)(L − 1) + 1 [15].
The following properties are self-evident for the sequence S2 that meets the GCD constraint. They are helpful for determining whether two exponent matrices possess identical cycle distribution.
Property 1: The two exponent matrices defined by S2′ = (s0, s1, · · · , sJ−1) and S2′′ = (s0 −s0, s1 −s0, · · · , sJ−1 −s0), respectively, have identical cycle distribution.
Property 2: The two exponent matrices defined by S2′ = (s0, s1, · · · , sJ−1) and S′2′ = (sJ−1 − sJ−1, sJ−1 − sJ−2, · · · , sJ−1 − s0), respectively, have identical cycle distribution.
Property 1 and 2 are separately denoted by “S” and “R”.
C. DD Codes
Generally, the exponent matrix Edd of a DD code is composed of the left sub-matrix (El) and the right sub-matrix (Er), that is, Edd = [El, Er], where Er is a square matrix (named DD matrix) with a roughly dual-diagonal structure [5], [8], [10]. In this letter, El is uniformly explicitly generated via the GCD framework, while Er are obtained by two different schemes (one existing and one new). For the existing scheme [8], the right sub-matrix is an all-∞ square matrix, except that er(s, s) = er(s, mod(s + 1, J′)) = 0 (0 ≤ s ≤ J′ − 1) and er(2, 0) = t, where t is an integer randomly found such that the whole matrix Edd corresponds to a code free of 4cycles and 6-cycles. The resultant right sub-matrix is denoted by Er(0). For the new scheme, the right sub-matrix is generated by first setting the two elements (er(2, 0) and er(J′ − 1, 0)) in Er(0) to ∞ and then diagonally flipping the resulting matrix. The right sub-matrix generated by this scheme is denoted by E(r1). The formal definition of E(r1) is given in Section V. It is easy to see from an elementary analysis that both schemes allow for a simple encoding process directly through the PCM.
III. SEARCH ALGORITHM BASED ON GCD CONSTRAINT AND NEGATIVE COEFFICIENTS:GNC
This section presents a search algorithm based on the GCD constraint and Negative Coefficients (abbreviated as GNC algorithm) which plays a crucial role in revealing explicit formulae for constructing the left sub-matrices of DD codes.
A. GNC
The most important feature of GNC algorithm is the adoption of negative coefficients when searching for the sequence S2 = (s0, s1, · · · , sJ−1). To the best of our knowledge, such a setting has not been considered in the literature. To be specific, it has been well recognized that if si is expressed in the form

αi · L + βi (where 0 ≤ i ≤ J − 1, 0 ≤ αi and 0 ≤ βi) then it is likely to attain certain formulae, which can explicitly define S2. In contrast to the usual practice (0 ≤ βi), the coefficient βi in the proposed GNC algorithm can be negative. The change to the range of βi is by no means a trivial modification. As will be seen in Section IV, this makes it possible to find small sJ−1 and hence obtain smaller lower bounds ((sJ−1−s0)(L−1)+1) on circulant sizes than the state-of-the-art ones.

It works by starting from the minimum of αi, βi and adding one each time, until it finds the tuple (s0, s1, · · · , sJ−1) which is increasing and satisfies the GCD constraint for each L within a certain consecutive range. If a smaller sJ−1 is found, then the algorithm updates the coefficient matrix Coe = (α0, α1, · · · , αJ−1; β0, β1, · · · , βJ−1). B. Reducing Search Space of GNC
The GNC algorithm employs (J − 1) pairs of f or loops, where each f or loop involving α needs to search a maximum of th + 1 values, and each f or loop involving β needs to search a maximum of 2th + 1 values. Therefore, the search complexity basically scales as (2th2)J−1. With the aid of the following two properties of GCD constraint, the algorithm can be noticeably accelerated by restricting search space.

D. Comparisons of New and Existing Lower Bounds

So far, for large column weights (J ≥ 5), the methods suitable for an arbitrary row weight and providing small lower bounds on consecutive circulant sizes, are all based on the GCD framework or its special cases. Specifically, for J = 5 ∼ 7, the early results regarding the lower bounds are L2(L − 1) + 1 [11], (L2 + 1)(L − 1) + 1 [11] and (L2 + L)(L − 1) + 1 [15], respectively; the current bounds have been remarkably improved to (3L + 1)(L − 1) + 1 [18], (4L + 2)(L − 1) + 1 [18] and (7L + 3)(L − 1) + 1 [10]. The bounds due to Theorems 1 ∼ 3, by contrast, are (3L−1)(L−1)+1, (4L−2)(L−1)+1 and (7L−4)(L−1)+1, respectively. Obviously, the new bounds improve the existing ones, providing more flexibility for circulant sizes.
It should be pointed out that, for small values of L, certain well-designed search methods (such as [19], [20]) can find even smaller circulant sizes which ensure girth-eight property, but they generally require high search complexity. Moreover, if parameters (P , J and/or L) are changed, these methods must be re-executed from scratch and it is not known in advance whether they will succeed in seeking out qualified exponent matrices. In contrast, the proposed constructions based on explicit formulae offer exponent matrices suitable for an arbitrary circulant size (not less than lower bounds) and an arbitrary row weight, serving as a good basic module to attain high-performance long codes, for example, the DD codes as shown in Section V.

V. NEW DD CODES

The new exponent matrices designed in Section IV possess
reasonably large values of column weight J, arbitrary values of
row weight L and small lower bounds on circulant sizes. While
exponent matrices can themselves be used directly to construct
QC-LDPC codes (without a DD structure), we consider ways
to combine them with a novel DD matrix to yield DD codes.
Based on GCD framework, a class of girth-eight DD codes
has been proposed in [8]. The exponent matrices for such codes can be expressed as Edd0 = [El, Er(0)]. The left matrix El is simply generated by deleting the first column of an exponent matrix E provided by GCD framework; the value of er(2, 0) = t within Er(0), however, relies upon computer search to guarantee girth eight for the overall exponent matrix Edd0, which is usually not convenient.
The problem can be solved by adopting the following new DD matrix as the right sub-matrix. The novel DD matrix, E(r1), is a J′ × J′ all-∞ matrix, except that e(s, s) = 0 for 0 ≤ s ≤ J′ − 1 and e(s + 1, s) = 0 for 0 ≤ s ≤ J′ − 2. The resultant
exponent matrices for new DD codes are denoted by

Edd1 = [El, E(r1)],

(3)

where El is the matrix obtained by removing the first column of the exponent matrix explicitly defined in Section IV.
Since El in Eq. (3) includes an sub-matrix [0, si]T · [1, 2, 3], there exist 8-cycles regardless of the circulant size. Thanks to the following theorem, it is easily deduced that the proposed DD codes have girth eight.
Theorem 4: Let E be an arbitrary exponent matrix with the first column being all zeros. Let El be the matrix obtained by deleting the first column of E. If E corresponds to a QC-LDPC code without 4-cycles and 6-cycles for the circulant size P , then [El, E(r1)] yields a DD code without such cycles for P .
Proof : Firstly, there are no 4-cycles or 6-cycles in the left matrix El, since El is a sub-matrix of E. Secondly, the right matrix E(r1) is a DD matrix, and hence it is impossible to appear 4-cycles and 6-cycles within Er(1). Therefore, 4-cycles or 6-cycles are only possible across El and E(r1). However, such 4-cycles imply the existence of 4-cycles cross the first column and another column of the matrix E = [0, El]. Similarly, such 6-cycles imply the existence of 4-cycles cross the first column and another column of the matrix E or the existence of 6-cycles cross the first column and other two columns of the matrix E. Therefore, such cycles are impossible. Consequently, [El, Er(1)] corresponds to a DD code without 4-cycles and 6-cycles for the circulant size P . □
In a word, Edd1 in Eq. (3) can yield new girth-eight DD codes. As will be seen in the next section, the proposed DD codes noticeably outperform the existing counterparts.
VI. PERFORMANCE SIMULATIONS
In this section, the new girth-eight DD codes are compared with existing codes in terms of bit-error rate (BER) and block-error rate (BLER). They are simulated with the setting of AWGN channel, BPSK modulation, and sum-product algorithm (SPA) decoding with a maximum of 50 iterations.

The left sub-matrix El is generated by removing the first column of E1. Therefore, the target exponent matrix Edd1 is expressed by [El, E(r1)], which is a J ′ × L′ exponent matrix (J′ = 5 and L′ = (L − 1) + J = 10). Two counterparts are used for comparison. The first one is an existing DD code with Edd0 = [El, E(r0)], t = 6 and P = 71; the second one is the (5, 10) QC-LDPC code (without DD structure) based on symmetrical construction [19] (named Sym. code) with P = 99 which is the smallest circulant size available. The performance is shown in Fig. 1. It is observed that the new DD code perform much better than the longer Sym. code and the existing DD code with the same length. All the three codes have a girth of eight, but cycle numeration shows that the numbers of 8-cycles for the new DD code, existing DD code and Sym. code are 274 × 71, 349 × 71 and 2688 × 99, respectively. Obviously, the new DD code contains the fewest number of 8-cycles, which may explain to some extent why it has the best performance.
