An Embedded Module of Enhanced Turbo Product Code Algorithm
Jianjun Luo , Senior Member, IEEE, Yifan Shen, Boming Huang , Michael Etzkorn , Graduate Student Member, IEEE, Hongqiang Chen, and Cong Yu

Abstract—Low density parity check code (LDPC) is the most popular error correction code (ECC) for current NAND flash memory controllers. Next generation flash memory, such as

triple-level cell (TLC) and quad-level cell (QLC) with a higher bit error rate, bring up the demand for superior ECC algorithms providing excellent performance and acceptable hardware overhead. This letter proposes turbo product code (TPC) as an alternative to LDPC by exploring embedding of a TPC engine into a solid-state drive (SSD) controller architecture. The implementation of this TPC engine uses 2-D error coding and applies the Bose–Chaudhuri–Hocquenghem (BCH) code in each dimension. The algorithm is further improved by flipping

the dedicated bit matrix when the basic TPC decoding algorithm terminates with uncorrectable conditions. This bit-flipping enhanced TPC (BFE-TPC) module is finally integrated into a nonvolatile memory express (NVMe) SSD controller driving eight flash memory channels. This BFE-TPC module illustrates the potential of 2D-TPCs as a replacement for LDPCs due to its high throughput and low hardware overhead.

I. INTRODUCTION
T HE SOLID-STATE drive (SSD) industry has continuously grown for years, benefitting from NAND flash’s high density and low prices. The demand for increasing memory capacity facilitates emerging device architectures to store more bits in one cell, which inevitably causes a rapid increase in the raw bit error rate (RBER) [1]. Fig. 1 shows a diagram of a nonvolatile memory express (NVMe) SSD architecture. The error correction code (ECC) module is integral to maintaining the data integrity of SSDs. With more bits stored in a single memory cell, RBER increases because the NAND flash requires more accurate read threshold voltages. Bose–Chaudhuri–Hocquenghem Codes (BCH) cannot satisfy the requirements of triple-level cell (TLC) or quad-level

storage systems. Researchers are looking for potential ECCs with higher efficiency for NAND flash controllers [3], [4], [5]. Turbo product code (TPC) [6] is one candidate owing to its high coding gain and flexibility in code construction.
This letter proposes a high-throughput, cost-effective TPC instead of LDPC for an embedded ECC engine in an SSD controller. A bit-flipping enhanced TPC (BFE-TPC) decoding algorithm is proposed to improve error-correction capability. The TPC module integrates an embedded processor to manage the bit-flipping operation. This BFE-TPC module is integrated into an NVMe SSD controller driving eight flash memory channels and prototyped using an FPGA. Results indicate these codes capably reduce the uncorrectable bit error rate (UBER) values to reach industry requirements of SSDs.

algorithm, assuming a t1-error-correcting code applied for each row, and t2-error-correcting code applied for each column, as long as the number of errors in the codeword is no more than
ti bits, the error can be corrected. Each iteration includes one column scanning operation and one row scanning operation.

C. Decoding Algorithm Enhanced by Bit-Flipping

In the early stages of the lifespan of NAND flash, the
RBER is low, and errors are correctable using the traditional
algorithm. As the RBER increases, there are a large number
of error bits in USM, requiring the bit-flipping enhanced
algorithm. Note that, every row (or column) has more than t1 (or t2) bit errors in USM, but not all the bits are uncorrectable. When U1 < 2(t1+1), on performing a bit-flipping operation of the USM, a majority of error bits are corrected, creating a new, smaller subset of error bits less than (t1 +1) allowing for error correction. The situation is the same when U2 < 2(t2 + 1). Therefore, the decoding algorithm is enhanced by two steps
as follows.
1) Flipping the USM bits if the USM scale is less than 2(t1 + 1) by 2(t2 + 1).
2) Running traditional HIHO decoding algorithms again in
all rows and columns.

III. DESIGN OF THE ARCHITECTURE
A. Encoder Module
Fig. 5 presents the architecture of the BFE-TPC encoder. The proposed encoder fully utilizes the data input with the associated row encoder and column encoders in each cycle, thereby maximizing the utilization of hardware resources.
1) Input and Output Data Buffer: This is because n1 is not a power of two, input and output buffers are used to connect to an external data bus. When entering the input data buffer, the data is shifted by a shift register to match the width of rows. The output data buffer execute the symmetric operation.
2) Row Encoder and Column Encoders: The encoder employs a row-wise input strategy, where in each clock cycle, the row data is simultaneously fed in sequence to both the column and row encoder. The row encoder is a parallel engine using binary matrix multiplication logic, and the column encoders are serial engines using polynomial division logic. When the final row of data enters into the column encoders, the computed column parity bits are ready and able to be fed back instantly to the row encoder.
B. Decoder Module
Fig. 6 shows a diagram of a BFE-TPC decoder architecture. 1) Decoder Engine Pool: In the decoding module, the
inverse-free Berlekamp–Massey algorithm is applied to achieve efficient complexity reduction [13]. There are a total of four decoding engines in this design in order to satisfy the bandwidth requirement of the SSD controller. The fact this 2D-TPC uses a single type of component code for both rows and columns allows for further reduction of hardware resources by reusing a single pool of decoding engines. 2) Transpose Memories: Transpose memories play a crucial role in improving the decoder bandwidth. Using a separate embedded processor from the controller’s main CPU frees up the bandwidth of the main CPU to focus on the flash translation layer (FTL) tasks.
IV. EXPERIMENTAL RESULTS
A. Error Correction Capability Client SSDs typically require UBER less than 10−15. The
error correction capability of the BFE-TPC decoder is shown in Fig. 7. The figure shows (4 kB, 2 × 2) and (8 kB, 2 × 3) reach the SSDs UBER target of 10−15. In this letter, a (315 × 246, 288 × 228, and 7 × 5) TPC code is constructed for NAND flash memory, where t1 = 3 and t2 = 2. The experimental results indicate that this implementation reaches the SSD UBER requirement of 10−15 when RBER = 10−2.

V. CONCLUSION
In this letter, a 2-D matrix ECC is proposed to replace LDPCs for NAND flash memory applications. The BFE-TPC decoding algorithm effectively improves the overall error correction ability of existing TPC decoding algorithms while also reducing hardware resource overhead. This proposed BFE-TPC is validated on FPGA with a (315 × 246, 288 × 228, and 7 × 5) TPC configuration. The experimental results indicate that UBER is able to reach 10−15 in an SSD module built with typical TLC flash memory chips which have RBER up to 10−2. With the growing size of NAND flash pages of up to 16 kB and investigation of larger mapping sizes for FTL, SSD controllers also benefit from ECCs with easy to design 8 and 16 kB codewords. The BFE-TPC module proposed benefits from an easy-to-design, large codeword. The architecture achieves both low area-size and high throughput, making such a scheme a promising candidate for the future of SSD controllers in specialized computing data centers with high amounts of sequential operations. Future work on larger mapping sizes for FTL will make transitioning from LDPCs to 2D-TPCs natural and more desired.

B. Hardware Resources
The hardware implementation of the proposed TPC encoder and decoder is prototyped with Xilinx Vivado 2022.1 on a Xilinx VU19P FPGA. Table I summarizes the hardware utilization of the proposed encoder and decoder. When running at a clock frequency of 250 MHz, the proposed encoder and decoder achieves a throughput of 29.21 and 17.96 Gb/s.
When targeted to a 28 nm process, the design close timing at 500 MHz according to the synthesis timing report. Table II shows the area comparison result. Compared with 2 kB LDPC, the proposed BFE-TPC implementation running under maximum clock frequency saves 42% area. This design’s high area efficiency arises from the reuse of the same short BCH code for both row and column direction as well as taking advantage of running multidecoders running simultaneously, reducing the number of cycles in one iteration.
C. Analysis of Latency
Table III shows the read latency for BCH, LDPC [16], and BFE-TPC decoding. For the BFE-TPC decoding, in the early stages of the NAND flash lifecycle, RBER is low allowing error bits to be corrected in two iterations. In the later stages, the maximum iteration count is set to ten total iterations, six for regular scanning and four for enhanced scanning after bitflipping. BCH, LDPC-hard, and BFE-TPC decodings achieve short read latency, but BCH and LDPC-hard decodings cannot satisfy high RBER applications such as 3D-TLC.
