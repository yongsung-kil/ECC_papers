Design and Implementation of Partially Parallel LDPC Decoder for Low Earth Orbit Micro-Nano
Satellites Data Transmission

Chong Huang1, Zhi Hou2,*, Peng Li3,Shishan Qi1
IEEE APCAP 2024 Nanjing University of Science & Technology,Schol of Electronic and Option Engineering,Nanjing,210000,China1
Shanghai Dianji University,Shanghai,200000,China2,* Shanghai Aerospace Electronic Technology Institute,Shanghai,200000,China3

Abstract- This paper proposes a LDPC decoder for Low Orbit Micro-Nano Satellites Date Transmision. By analyzing the Hmatrix characteristic, each cyclic sub matrix of the H-matrix is mapped to a Ramdom Access Memory(RAM). The address and data lines of the RAM are fully utilized to realize that RAM can be represented as both an H-matrix and an information storage matrix. The results on FPGA platform show that when the Eb/N0 is 4dB, the throughput of the decoder reaches 145Mbps, which improves the margin of data transmission links and meets the demand for data transmission. The decoder has completed in orbit verification on the experiment verification satellites.
I. INTRODUCTION
The main error correcting code used in communication satellites is the 7/8 bit rate LDPC (8176,7154) code recommended by the CCSDS 131.1-O-2[1]standard for low orbit satellite communication. Partially parallel decoders take advantage of the inherent architecture of structured LDPC codes, providing a trade-off between low complexity and parallel processing.
In this paper, a decoder is designed using hierarchical Normalized Minimum Sum(NMS) algorithm[2]. According to the extrinsic information storage matrix characteristics and the parallelism requirements of the variable node units(VNUs) and check node units(CNUs) modules, different arrangements of the H-matrix are designed. The address lines of the RAM have been used to represent the non-zero element locations of the Hmatrix. As a result, the same RAM can be represented as both an H-matrix and an extrinsic information storage matrix, which greatly saves the consumption of resources.
II. PARTIALLY PARALLEL LDPC DECODER
The LDPC decoder can be simply divided into: input storage module, VNUs module, CNUs module, and decoding judgment module. It is crucial to effectively store the data generated by these modules during the decoding process.
The initial intrinsic information consists of 8176 floating-

point data points. According to the array composition of the code matrix, these floating-point data points can be divided into 16 blocks, with a total of 511 data points per block, as shown in Fig. 1(c). A. VNUs/CNUs-oriented QC-LDPC matrix
The parity check matrix H is a sparse matrix with a size of 1022×8176. The matrix consists of 32 cyclic sub matrixes of 511×511, which form a 2×16 array. For each cyclic sub matrix, there are only two non-zero elements in each row and column, and the position of the non-zero elements in the first row is known. The remaining row elements are obtained by cyclically shift right the data by 1 bit in the above corresponding row. As a result, all row can be obtained. Split each sub cyclic matrix and use two dual-port RAM for storage mapping. For 32 sub matrices, 64 RAM is required for storage.
For the NMS algorithm, the calculation of VNUs and CNUs is only related to the position of non-zero elements in the check matrix and the data at that position. In this paper, we propose to use the address line of the RAM to represent the non-zero elements of the H-matrix, and the data line to represent the extrinsic information during the iteration process. As shown in Fig. 1(a) and (b), based on the characteristics of VNUs and CNUs in the NMS algorithm iteration process, RAM can be fully utilized to realize the parallelism of computing.

C. CNUs Module

The CNUs module has a horizontal operation process

characteristic that simultaneously processes 32 data in each

row of the H-matrix. Processing by parallel full comparators,

shift add multipliers, and symbol selection processing, data is

ready for to Decoding Judge Module. The row parallelism of

the CNUs module is designed to be 2, and the column

parallelism is designed to be 32.

D. Decoding Judge Module

The decoding judge module consists of three parts, including

decision, verification, and output section. The decoding

decision outputs 8176 "0" or "1" data, which are stored in 16

RAM. The verification equation of the H-matrix contains 32

verification data, but 16 RAM can only verify 16 data at a time.

So the 32 verifications data is separated into two verification

16 data. As shown in Fig. 1(d), the first validation was

performed on 16 Rm, and the second validation was performed on 16 Rm'. Rm and Rm' are the same memory, which can be represented as the first RAM and the second RAM in the sub

loop matrix according to their addresses.

IV. CONCLUSION
This paper proposed a partially parallel QC-LDPC decoder for Low Earth Orbit Micro-Nano Satellites. According to the blocking characteristics of the H-matrix and the data storage characteristics of the extrinsic information matrix, different parallelism are used to calculate the extrinsic information at different modules. Through comprehensive analysis of the results, the decoder resource consumption is acceptable, and the throughput reaches 145Mbps, which meets the requirements of satellite ground data transmission indicators.
