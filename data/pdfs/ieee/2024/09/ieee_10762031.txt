Parallel optimization design of LDPC decoder with variable normalization factor

1st Kaidi Liu* School of Microelectronics Hefei University of Technology
Hefei, China 1923201656@qq.com

2nd Ziwei Guo School of Microelectronics Hefei University of Technology
Hefei, China 524881906@qq.com

Abstract—In order to solve the problem of poor decoding performance of the fixed normalization factor in the normalized minimum sum algorithm, the article proposes a variable normalization factor LDPC decoder, which improves the decoding performance compared to the fixed normalization factor. 0.2dB. At the same time, in order to solve the problems of low throughput and high storage resource usage of some parallel LDPC decoders, the parallelism of the system is increased by adopting the intra-layer parallel structure, and the consumption of storage resources is saved by time-division multiplexing of node data cache units. Experimental results show that compared with the traditional partially parallel structure, the decoder designed in this paper uses 13% more LUT resources, reduces storage resource consumption by 38%, and increases the decoding throughput by 14.5M.
Keywords- Improved decoding algorithm; Partially parallel structure; LDPC decoder; Variable normalization factor
I. INTRODUCTION
LDPC code is a linear block code proposed by reference [1] in 1963. Limited by the technical level at that time, the hardware implementation of LDPC code was too complicated and did not receive much attention [2].
In 1996, reference [3] found that LDPC code has decoding performance close to the Shannon limit. Combining with Gallager's research, a belief propagation (BP) decoding algorithm based on probability measurement was proposed. Since the BP algorithm uses the hyperbolic tangent operation, it is difficult to implement in hardware. Therefore, in 1999, reference [4] proposed the minimum sum algorithm (MSA), which approximates the hyperbolic tangent operation to the minimum value operation and the symbol operation, reducing the computational complexity of the BP algorithm. However, the MSA algorithm reduces the decoding performance of LDPC due to the use of approximate calculations. In 2002, reference [5] proposed the normalized minimum sum algorithm (NMSA) to compensate for the performance loss of the MSA algorithm by adding a correction factor to improve the algorithm performance. Selecting an appropriate correction factor can make the NMSA algorithm close to the performance of the BP algorithm [6]. Reference [7] proposed a normalization factor that varies with the number of iterations. Compared with a fixed normalization factor, this method converges faster.

There are three main structures for decoder hardware implementation: fully serial[8], fully parallel[9] and partially parallel[10].In 2020, Boutillon E.[11] designed a new barrel register for the 5G LDPC decoder by adopting an additional layer expansion method, which reduced the number of multiplexers by about 40% compared with the original solution. In 2021, Kang Jing[12] increased the decoder throughput and avoided memory access conflicts by adopting parallel operations and using optimized data aligners, achieving a throughput of 1.02Gbps at a 250M clock. In 2022, E.A.Likhobabin[13] designed a QC-LDPC decoder suitable for all code rates of the DVB-S2X standard. The designed decoder uses a layered structure to achieve a throughput of 500Mbps at a 250M clock.Reference [14] uses a traditional partially parallel structure to design an LDPC decoder. The designed decoder has low resource occupancy, but the low parallelism leads to low throughput, which is not suitable for high throughput scenarios. Reference [15] uses a fully parallel structure to achieve a throughput of 4.7 Gbp/s twice, but its LUT resource consumption is as high as 170,000, and only supports one code rate. Reference [16] proposes a decoder that supports full-rate 5G LDPC. It uses a partially parallel structure with intra-layer parallelism and inter-layer seriality. Compared with the traditional partially parallel structure, it greatly improves the system throughput and occupies fewer resources than the fully parallel structure decoder. However, the storage overhead is large due to the separate storage of updated row and column data.
In summary, in order to solve the problems of low throughput and high storage resource consumption of traditional partially parallel decoding, this paper adopts the intra-layer parallel structure to improve the throughput, and time-division multiplexing the memory to reduce the system's consumption of storage resources. At the same time, the normalization factor in the NMSA algorithm is improved to reduce the number of decoding iterations.

B. Analysis of LDPC decoding algorithm
LDPC decoding algorithms are mainly divided into two categories: one is the hard decision decoding algorithm, such as the bit-flipping (BF) algorithm, which has simple decoding hardware implementation but poor performance and is rarely used in engineering. The other is the soft decision decoding algorithm, such as BP and MSA. The soft decision decoding algorithm uses channel information in decoding calculation and has better performance than the hard decision decoding algorithm. C. Normalization factor selection
In the NMSA algorithm, the normalization factor changes with the signal-to-noise ratio and the number of iterations. It has no specific value and is difficult to implement in hardware. In practice, for the convenience of implementation, a fixed value is often used. For example, the decoding effect is not ideal. Therefore, reference [7] proposes a variable decoding algorithm, which has a faster decoding convergence speed than the fixed one. In reference [7], the normalization factor is 1/2 in the first iteration. Experiments show that when the normalization factor is 1/2 during iteration, the decoding performance is poor. Therefore, this paper draws on the method of reference [7], combines the actual environment of this system and the integer quantization method of NMSA, and sets the decoder to use a variable normalization factor as shown in formula (7), where n represents the number of iterations.

III. LDPC DECODING DEVICE HARDWARE STRUCTURE
A. Decoder overall structure
This thesis adopts QC-LDPC code. The check matrix H has 16×32 sub-matrices with block size b=64, that is, the check matrix scale is 1 024 ×2 048. The traditional partially parallel decoder structure is shown in Figure 3a. The variable node data and the check node data are stored separately, which requires large storage resources. At the same time, 16 parallel check node processing units (CNU) and 32 parallel variable node processing units (VNU) are used for calculation. As shown in Figure 3b, the decoder structure designed in this paper uses the same number of CNUs and VNUs as the block size b. Compared with the traditional partially parallel

structure, the CNU parallelism is increased by 16 times and the VNU parallelism is increased by 8 times.

The designed decoder combines the check node data cache unit and the variable node data cache unit shown in Figure 3b, saving 50% of the storage resources required for this module compared with the reference [16].
B. Node data cache module
The check matrix used in this paper has a row weight of 8 for each row and a column weight of 4 for each column. The quantization bit width of each data is 8 bits. In order to meet the system's requirements for parallelism, the node data cache module uses 64 memories to cache node data, and each memory has 16 rows. Each row of data is all non-zero numbers in the corresponding row of the check matrix, so the size of each storage unit is（8≥ 8） ≥ 16.
As shown in Figure 4, the node data cache unit is introduced by taking the H matrix of size 4≥ 8 as an example. The 4 sub-matrices of size 3≥ 3 in the upper right corner of the H matrix are enlarged. The parts marked with 0 in the 4 submatrices are all 0 matrices and do not need to be stored in the storage unit. For example: in the upper right sub-matrix, only the diagonal elements are not 0, so only the data in the diagonal needs to be stored in the storage unit.
Each row of the H matrix has 8 elements that need to be stored in the same address of the same memory. Therefore, according to the above rules, these 8 elements need to be stored in different bits of the same address in column order.

V(k)2c_c1~V(k)2c_cm indicates that there are m information nodes in each row of the check matrix, and m is the row weight of the matrix. When processing the check node, the input data is first divided into two paths: one path extracts its sign bit information, and then the sign bit is XORed to obtain the sign bit s1(k)~sm(k) of each data after the update. The other path takes the absolute value, and then the minimum and second minimum values in the absolute value and the position of the minimum value are found through the comparator to determine the minimum value except the current data, and then the minimum/second minimum value is normalized. The normalization method is: Substitute the number of iterations into formula (7) to determine the normalization factor The multiplication of the minimum/minimum value and is obtained by shift addition, which saves resource occupation and system delay compared with using a multiplier. Finally, the output data is judged as the minimum/sub-minimum value or the minimum/sub-minimum value complement according to the sign bits s1(k)~sm(k), and the updated check nodes C(k+1)2v_c1~C(k+1)2v_cm. are obtained.

V. CONCLUSIONS This thesis proposes an improved structure to solve the problems of low throughput and high storage resource usage of the traditional partially parallel decoder structure. At the same time, a variable normalization factor algorithm is proposed to solve the problem of poor decoding performance of fixed normalization factors, which reduces the bit error rate in low signal-to-noise ratio environments. The designed decoder reduces storage resource consumption by 38%, and the throughput is increased from 391 Mb/s to 402.5 Mb/s.
