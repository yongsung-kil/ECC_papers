On Construction of Low-Density Parity-Check Codes for Ultra-Reliable and Low Latency Communications

Linqi Zou , Hao Yan, Jie Dong , Yong Li , Member, IEEE, Pingping Chen , Member, IEEE, and Francis C. M. Lau , Fellow, IEEE

Abstract— Low-density parity-check (LDPC) codes with protograph-based raptor-like (PBRL) structure have been chosen as the data channel coding scheme for the fifth-generation (5G) enhanced mobile broad band (eMBB) services. However, these 5G LDPC codes are not optimized for the scenarios of Ultra Reliable and Low Latency Communications (URLLC) and massive Machine Type Communications (mMTC) in which small transport block sizes are usually used. In this paper, a new method is proposed to construct PBRL LDPC codes for URLLC and mMTC. This method incorporates degreedistribution optimization, base matrix derived from parity-check matrix of quadratic residue codes, masking technique and modified progressive-edge-growth (PEG) algorithm. Simulation results show that the proposed PBRL LDPC codes outperform 5G LDPC short codes in terms of error-correcting performance.

Generally, polar codes [2], Bose-Chaudhuri-Hocquenghem (BCH) codes, and low-density parity-check (LDPC) codes [3] are common candidate channel coding technologies suitable for URLLC. As one of the most promising candidates for URLLC, polar codes provide good error correction performance with short block lengths. However, the decoders of polar codes, such as successive cancellation (SC) and successive cancellation list (SCL) decoders, require huge decoding latency. BCH codes usually have large minimum Hamming distances and can obtain a good error correction performance. But the encoding of BCH codes relies on their code structures, which makes it difficult to select their parameters (e.g., code length) flexibly.

Many good LDPC codes have error floors at high signalto-noise ratios because using sub-optimal decoder and their own structure, which limits their applications for URLLC. It was shown that Reed-Solomon based quasi-cyclic LDPC (RS-QC-LDPC) codes constructed based on Reed Solomon codes have excellent performance [11]. Recently, Li et al. In this paper, we propose a mechanism to design PBRL LDPC codes by treating a code as the concatenation of a core high-rate LDPC code and a LDGM code. We extend the QR-QC-LDPC codes shown in [12] as the core part (we will refer to it as core code hereafter). Moreover, we modify the masking technique in [13] and make use of the extrinsic message degree (EMD) concept [14] to optimize this core code and determine filler columns by considering the trapping sets. Furthermore, we introduce the modified progressive-edgegrowth (PEG) algorithm to optimize the overall 5G LDPC code design. The proposed short LDPC codes have a similar structure as the standard 5G LDPC codes, and can perform better in terms of error rates.

III. PROPOSED DESIGN STRATEGY FOR PBRL LDPC CODES

We denote the (s, t)-th entry in BQR by Bs,t, i.e., BQR = [Bs,t]s∈[1,(n−1)/2],t∈[0,n−1]. Then a QC parity-check matrix can be constructed by replacing each Bs,t in BQR with an n×n circulant permutation matrix (CPM) having a shift value of Bs,t, where n is the size of CPM.
Theorem 4: In each row of BQR, all elements are different. Further, each row is a different permutation of [0, 1, 2, · · · , n − 1].
Proof: Since j ∈ [1, n − 1], and t ∈ [0, n − 1], both j and t are elements on GF (n). Also since Bs,t = (js × t) mod n, so Bs,t is the result of multiplication on GF (n) by js and t. For any t1, t2 ∈ [0, n − 1], t1 ̸= t2, it is obvious that Bs,t1 ̸= Bs,t2 . So each row of BQR is a permutation of [0, 1, 2, · · · , n − 1].
Theorem 5: In each column of BQR (except for the first column, i.e., all zero column ), all elements are different.
Proof: When t = 0, Bs,0 = (js × 0) mod n = 0. When t ̸= 0, if the assumption does not hold, then for a given column t, there exist two same elements, that is, there exist two rows j1, j2, satisfies (j1 × t) mod n = (j2 × t) mod n, which is equal to (j1 − j2) × t ≡ 0 mod n. Since n is a prime number, there must be n|(j1 − j2) or n|t, however, it is clear that both are less than n, so the above assumption does not hold and therefore the theorem holds.
By choosing sub-matrices of BQR as base matrices, other QR-QC-LDPC codes of different lengths and rates can be constructed. Let 1 ≤ g ≤ m, 1 ≤ h ≤ n and g < h. A sub-matrix formed by selecting g rows and h columns of BQR represents the base matrix of a regular QR-QCLDPC code of length qh and rate larger than or equal to (h − g)/h. The results in Fig.1 illustrate that QR-QC-LDPC codes with the same size performs better than RS-QCLDPC [11] codes for some parameters. Because of the good performance of QR-QC-LDPC codes and the low number of short cycles, we decided to design PBRL codes based on it.

The main issue in the design of this PBRL-LDPC code is to construct B1 and B2, but meanwhile the 3GPP protocol specifies that we need to not only construct the parity check matrices of LDPC codes, but also determine the corresponding punctured and filled columns [22]. Therefore, our design strategy consists of four main steps, namely (i) optimize degree distribution of HPL; (ii) construct B1; (iii) determine filler bits; (iv) determine punctured bits and construct B2. 2) Masking: Next, we attempt to optimize the base matrix B˜ a×b by masking, i.e., setting some of the entries wi,j to −1 such that the corresponding CPM becomes the all-zero matrix. Moreover, the target variable-node degree distribution and check-node degree distribution of the masked base matrix are derived from Eqs. (9) and (10).
We recall that the extrinsic message degree (EMD) of a cycle is defined as the number of check nodes singly connected to the variable nodes in the cycles [14]. For a base matrix of size a × b, we define ϵi(,2jl,ψ) as the number of cycles of length 2l that (i) pass through the (i, j)-th entry and (ii) have an EMD metric of ψ. Moreover, cycles with small EMD metrics are more detrimental to the error-correcting performance and hence should be eliminated. Our proposed masking algorithm is shown as Algorithm 1. The algorithm aims to eliminate the largest number of short cycles and cycles with a small EMD metric ψ. After the masking process, the masked base matrix is obtained and denoted as B1. C. Determine Filler Bits
A (x,y) trapping set A in a Tanner graph is a subset of the set of variable nodes, such that (i) the number of variable nodes in A is x, and (ii) the number of check nodes with odd degree neighboring to the elements in A is y. Trapping sets, especially the small trapping sets often deteriorate the decoding performance. So in this step, we will select the columns with the largest number of trapping sets in B1 to be the filler columns, so that the remaining part has the best performance. In general, the smaller the trapping set is, the greater the impact on the performance of the matrix is. In this step, we use the algorithm in [18] to search and count the number of (κ, 0) trapping sets in B1 where κ = 2, 3, . . . , 6. Then we choose the columns that can eliminate the largest number of (κ, 0) trapping sets as the filler columns. If there exists two or more alternative options that can eliminate the

same number of trapping sets, then we consider the (κ, 1)

trapping sets in B1 where κ = 2, 3, . . . , 6. We choose the

columns that can eliminate the same number of (κ, 1) trapping

sets as the filler columns. Again, if there are two or more filler

columns that eliminate the largest number of (κ, 1) trapping

sets, next we consider the (κ, 2) trapping sets in B1, and so

on. Eventually we can decide on the only one set of filler

columns. Then we need to put these columns in the positions

specified in the protocol. This means that B1 needs to perform

a column permutation. We denote the column permutation

The complexity of the process is assessed, starting with the construction of B1, which involves three primary parts: submatrix selection, masking, and determination of the filler columns based on trapping sets. The complexity of selecting a submatrix is the highest, determined by the number of all possible submatrices (i.e., Cnb × C(an−1)/2). Fortunately, the complexity of calculating the number of short cycles for each submatrix can be simplified significantly according to Eq. (11). The masking operation has a very low complexity, requiring only the computation of the number of short cycles and its EMD for each element of the base matrix. Thus, the complexity is directly related to the total number of elements (i.e., a × b). For determining the filler columns, it is only necessary to compute the entire trapping sets of B1, resulting in a very low complexity.
Moving on to the construction of B2, the running time of improved QC-PEG algorithm is not long. Additionally, the times of running improved QC-PEG algorithm is determined by the all possible permutations (i.e., Cbb−c). As a result, the complexity of this stage is multiple of Cbb−c and is the most time-consuming part of the entire process. To decrease the time of this part, several approaches can be considered. The first is to directly run the improved QC-PEG algorithm to obtain B2, employing the fast method of constructing B2 as mentioned earlier, followed by determining the position of the punctured columns according to the number of short cycles or other properties. The second approach is reduing the search space by determining potential punctured columns in advance based on the degree distribution or adding other restrictions. However, these methods may compromise the performance of the constructed parity check matrix, yet they enable the construction of a larger base matrix. V. CONCLUSION
In this paper, we propose a mechanism to design PBRL LDPC codes by treating a code as the concatenation of a core high-rate LDPC code and an LDGM code. The whole construction starts with an optimized degree distribution pair, and combines the optimized construction of QR-QC-LDPC codes, masking techniques and a modified PEG algorithm. The proposed LDPC codes have almost the same structure as 5G LDPC codes and outperform the latter in four examples with short block lengths. For future work, we will consider codes with other lengths and rates.
