AbstractвЂ”In this paper, the implementation of a QC-LDPC 
encoder using AR4JA photograph LDPC is presented. This design 
utilizes a block-cyclic generator matrix from a quasi-cyclic parity 
check matrix to reduce encoder complexity. Additionally, an 
increase in speed of 22.87 % and a reduction in resource 
utilization of 14.28 % is also achieved through this method. 
The synthesis report of the utilization summary and delay 
from Vivado is shown and compared with an efficient QC- LDPC 
encoder using the Richardson Urbanke algorithm. FPGA 
implementation is also performed for the proposed design. 
Index TermsвЂ”Quasi-cyclic LDPC codes, AR4JA Protograph, 
RU algorithm 
 
I. INTRODUCTION 
Low-density parity-check (LDPC) codes are a class of error- 
correcting codes, they achieved significant attention because of 
their capacity-approaching performance and their efficient 
decoding algorithms [1].LDPC was introduced by Gallager 
in the 1960s and then rediscovered by MacKay and Neal 
in the 1990s, due to its computational limitations they were 
initially overlooked, but have since evolved into a fundamental 
component of modern communication systems including 5G 
and satellite communications [2]. 
 Gaussian source coding to CCSDS near-Earth standards 
the flexibility and robustness of LDPC codes are demonstrated 
in various applications. These codes exceptional flexibility and 
robustness made them ideal for many applications such as 
satellite communications. In [3],[4], a novel encoder 
architecture with CCSDS-recommended standards focusing on 
low resource utilization are given and both of them are 
implemented and verified it on FPGA. In both papers a 
preprocessing method is used for the special size of generator 
matrices. It is used for near-earth communication. Here we 
focus on Deep-space applications.

Protograph-based QC-LDPC codes gained attention because 
of their flexibility and performance in various communication 
systems.  
 In [7] highlights the critical advancements in protograph-
based LDPC code design which are discussed in recent studies 
such as the elimination of short cycles with chords from the 
Tanner graph. Further developments in protograph-based LDPC 
codes are explored in [8]. A method for constructing double 
protograph LDPC codes is proposed here. This double 
protograph structure provides a systematic way to design LDPC 
codes.  
For constructing generalized protograph LDPC codes a novel 
cycle-concentrating Progressive Edge Growth (PEG) algorithm 
is introduced in [9]. For modern communication systems, 
including satellite communications and 5G networks, the 
innovations in protograph-based QC-LDPC code design have 
far-reaching implications. They are not only efficient in terms of 
hardware implementation but also highly effective in correcting 
errors across a wide range of applications. 
Protograph- based LDPC codes with circulated generate 
matrices are used here. From a quasi-cyclic parity check matrix, 
block- cyclic generator matrices of non-singular protograph-

 LDPC codes are characterized by their sparse parity-check

II. BACKGROUND 
A. Protograph LDPC Codes 
LDPC codes are classified as regular, protograph codes. A 
protograph consists of a set of check nodes, a set of variable 
nodes, and a set of edges that connect a variable node to a 
check node. 
In Fig. 1 an example protograph is shown. From node 1 
to a there is a connection that is called an edge so in the 
corresponding parity check matrix 1 is placed. Similarly, all 
of them are followed. 
Fig. 1. An example protograph 
 
In Fig. 2 copied protograph is shown. The example proto- 
graph is copied three times. Here the expansion factor is taken 
as three times. So each of the variable nodes and check nodes 
is copied 3 times. However, the problem in Fig. 2 is the overall 
graph consists of three disconnected subgraphs. 
Fig. 2. An Example photograph copied three times 
 
A connected subgraph called a derived graph is shown in Fig 
3. In Fig. 3 the endpoints of 3 copies of each edge in 
the 
photograph are rearranged among 3 copies of check and variable 
node. After rearranging the endpoints of the edges, the 
subgraphs are now interconnected. 
 
Fig. 3. Derived protograph 
 
The protograph code construction boils down to the con- 
struction of the submatrices. One of the types is quasi-cyclic 
construction. In this type of construction, each row is a cyclic 
shift of the previous row and the whole matrix is subdivided 
into many submatrices that are themselves cyclic shifts of each 
other. We can get an invertible block-cyclic generator matrix 
from these quasi-cyclic parity check matrices. 
 
B. AR4JA Protograph 
The codes used here are AR4JA (Accumulate-repeat-4- 
Jagged) codes. These codes are most suitable for deep-space 
applications.AR4JA photograph consists of nodes called check 
and variable. The node вЂ™eвЂ™ is a punctured variable node. Quasi-
cyclic construction added with permutation is applied to this 
AR4JA photograph to get the resultant one. The Ar4ja 
photograph is given below in Fig. 4. In the figure given below, 
node 1,2,3 denotes the check node, nodes a,b,c,d, and e mean the 
variable node, and node вЂ™eвЂ™ means the punctured or 
untransmitted node. From node 1 to nodes a,b, and d, no edges 
are connected, which is why 0 is in the corresponding matrix, but 
from node 1 to node e,d, the edges are connected. In this way, 
matrix construction is done.it is a half-rate photograph. Here the 
expansion factor is 4, which is N=4 so each bit is replaced by 
NГ—N submatrices.

IV. PROPOSED CONSTRUCTION 
A. Matrix Construction 
 From the H matrix, which is quasi-cyclic, a block- 
cyclic generator matrix is built. Here, the AR4JA    
parity check matrix is used. 
Here, the selected code rate is 1/2. For ARJA codes, the 
supported code rates are KM/KM+2M or K/K+2, where K 
= 2. Here K=2 and M=4 is taken. Here, the first 8 
columns of H are denoted as Q and the last 12 columns 
are taken as P. P is the 3M Г— 3M, so it is a (12Г—12) matrix, 
and Q is the 3M Г— MK so it is a (12Г—8) matrix. 
Compute P inverse and multiply it with Q, so the Generator 
matrix is constructed by the concatenation of I and W, 
where I is an MK Г— MK matrix, which means here 8 Г— 8 
matrix, and W means it is the multiplication of both the Q 
and P inverses. Here W is an MKГ— 3M matrix, which 
means W is an 8 Г— 12 matrix For compact representation I 
am taking G as W only. An information bit of length KM 
is taken here, which means 8 bit length. 
B. Working of proposed Encoder 
To implement the LDPC Encoder, as per the architecture, We 
need feedback shift registers and memory. Message bits are 
Encoded first, taking the required number of feedback shift 
registers. Generator matrices are stored here. Here information 
bits are taken, which is of length 8 bits, which are passed 1 at a 
time and are multiplied with each row of the generator matrix 
and stored in a temporary register. Here there are 12 registers, 
the first 12 bits are multiplied by one bit of information bits and 
stored in the temporary register another row also does the 
same operation, and the second row is Exored with the previous 
row and the process continues until the last row is and 
multiplied with the last information bits and Exored with the 
previous row and output are stored. so the output codeword 
obtained is 000000001111. The methodology used for 
constructing the Encoder is given in Fig.8.

CONCLUSION 
An implementation of a QC LDPC encoder is given here. 
This design utilizes a block-cyclic generator matrix from a 
quasi-cyclic parity check matrix using AR4JA photograph 
LDPC codes given here. Since it takes fewer steps for mul- 
tiplication, it saves a lot of computation time and memory. 
Compared with an efficient QC-LDPC encoder using the RU 
algorithm an increase in speed of 22.87 % and a reduction in 
resource utilization of 14.28 % are also achieved through this 
method.
