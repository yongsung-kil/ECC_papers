Abstract—Low-density parity-check (LDPC) codes have been
selected as the channel coding scheme for 5G NR standard,
which have excellent performance approaching the Shannon
limit and are easy to implement in hardware. The decoding
algorithm and scheduling scheme of LDPC codes have a
significant impact on performance. The row degree (RD) and the
number of punctured bits (PBs) are widely used to determine the
update order of layers. In this letter, we used Matlab to simulate
the error performance of the 5G LDPC codes. We completed the
bit error rate and frame error rate simulations of six different
decoding algorithms, including flooding LLR-BP, flooding NMS,
layered LLR-BP, layered NMS, the lowest degree and least-
punctured (LD-LP) layered LLR-BP and LD-LP layered NMS
methods, for the 5G LDPC code with a base graph of size [46 ,
68], code rate 1/3, and code length of 1584.We further compared
the performance of LD-LP layered decoding and LP-LD layered
decoding. Moreover, we simulated the error performance of the
core part of the LDPC code, which consists of four layers. We
evaluated the effect of the update order of the four layers on the
error performance of the code, and concluded that some update
orders might give slightly better error performance than others.
Keywords—LDPC codes, 5G NR, layered iterative decoding,
scheduling scheme.
I. INTRODUCTION
In the context of 5G becoming a hot topic in the world,
LDPC codes stand out with their performance approaching the
Shannon limit. The essence of LDPC code is a channel
encoding, which aims to ensure transmission accuracy and
reliability, and improve transmission speed. The 5G NR
standard stipulates that the Quasi-cyclic low-density party-
check (QC-LDPC) codes are used as 5G channel codes [1].
The performance of LDPC codes is largely determined by
decoding. The decoding methods of LDPC include soft
decision decoding and hard decision decoding. The soft
decision decoding algorithm studied in this paper is essentially
an iterative algorithm that decodes code bits by exchanging
messages between variable nodes (VNs) and check nodes (CNs)
in each iteration. For the log-likelihood ratio belief propagation
(LLR-BP) decoding algorithm [2], the information passed
between CNs and VNs is the logarithm of probability, while
the normalized min sum (NMS) decoding algorithm simplifies
the algorithm when updating CNs to make it more suitable for
hardware implementation [3], which means sacrificing some
performance for computational simplification.
In addition, the order of updates also has an impact on
performance [4]. Unlike flooding decoding, the check nodes of
layered decoding updates the variable nodes layer by layer,
while the former performs simultaneously. Generally speaking,
layered decoding has better performance and can often achieve
better results with fewer iterations. This is because layered
decoding can fully utilize the latest information of the variable
nodes in the previous layer, making convergence faster and
more sufficient, while flooding decoding requires one iteration
to utilize the latest information of other layers. In most
previous schedules, the update order of layers is the order of
rows, from the first row to the last row. Then, due to the
characteristic of the parity-check matrix (PCM) that utilizes
node participation in the check equation, different update
schedules obviously have different impacts on performance.
The use of schedule that is more conducive to spreading and
updating posterior probability information can improve the
convergence speed of layered decoding and thereby reduce the
error rate.
Due to the fact that the 5G NR matrix can cover different
code lengths and rates, the first 2z bits of the codeword will be
punctured after encoding and will not be transmitted. The row
degree (RD) and the number of punctured bits (PBs) are widely
used to determine the update order of layers. Several static
layered scheduling schemes have been proposed. The least-
punctured and highest-degree (LP-HD) schedule will prioritize
updating the layers with the least punctured bits [5]. For layers
with the same number of PBs, priority is given to updating the
layer with the highest row degree. One edge-punctured and
lowest-degree (OE-LD) schedule assigns the highest priority to
the layer with one PB and the second highest priority to the
layer with the lowest RD [6]. On this basis, a novel base graph
based static scheduling (BGSS) method using three different
priority rules to determine the update order of layers has also
been proposed [7]. The first two priorities are taken from the
schedule above, least-punctured (LP) and lowest-degree (LD)
respectively, which can be abbreviated as LP-LD. For layers
with the same number of PBs and the same RD, the layer

A. 5G NR LDPC Codes
In the 5G NR standard, in order to meet the needs of
different communication scenarios, LDPC codes must be able
to flexibly support different code lengths and rates. In order to
meet the requirements, the standard divides the PCM of 5G NR
LDPC code as shown in Fig. 1.

A QC-LDPC code is composed of base gragh (BG) and
corresponding expansion factor z. The generation rule is as
follows: the nonnegative elements in BG are replaced with a
cyclic shift matrix of size z × z , and the “-1” in BG is replaced
with the all-0 matrix of z × z. The value of the nonnegative
integers in BG determines the shift value of the cyclic shift
matrix. If it is 0, no shift is required, and it is the identity
matrix. This value determines how many bits the cyclic shift
matrix shifts to the right.

B. Layered Decoding of LDPC Codes
When using layered decoding, the check matrix is divided
into multiple layers from rows, each layer containing a portion
of the rows of the original matrix. During decoding, the system
only starts updating the next layer after updating one layer. The
completion of the last layer update means the completion of
one iteration. After the check nodes in each layer are updated,
the update results of this layer are passed to the variable nodes.
After receiving the update result of the check nodes, the
variable nodes will complete the update operation of its own
node. The update of the check nodes in the next layer will be
affected by the information transmitted by the variable nodes
updated in the previous layers. The information updated by the
previous variable nodes can be immediately used in subsequent
layers without waiting for the next iteration, which makes the
convergence speed much faster than flooding schedule.

III. SCHEDULING SCHEME OF 5G NR LDPC CODES
In 5G NR LDPC codes, the LLR of the first 2z bits is 0 due
to being punctured. If the check node is connected to two
punctured variable nodes, according to equation (4), in the first
iteration, the result of the multiplication will all be 0. This is
also the same for layered NMS decoding algorithm,in which
the minimum and sub minimum values will be zero. Therefore,
the a posterior probability (APP) messages of all variable
nodes connected to this check node will be 0. If the check node
is only connected to one punctured variable node (PVN), all
updates to nodes except for that punctured node will be
invalidated.Therefore, the APP messages of this punctured
VNs can be updated and is no longer zero, while the APP
message of the other VNs related to the CN cannot be updated.
If the check node is not connected to PVN, then all the VN are
not zero and normal operations can be performed. Given this
observation, the LP schedule was proposed to quickly recover
APP messages for PVNs, thereby promoting updates to APP
messages for other variable node. The reason for using LD
update is that a CN with fewer VNs connected should have a
relatively large result after multiplication, so the APP message
update amplitude is larger.
Currently, the BGSS method has the best performance.
However, as shown in [16], it is the LD rule, not the LP, that
can provide maximum error performance improvement. Due to
the fact that most layers in 5G NR LDPC codes only have one
PB. So we reconsider the priority of LP and LD rules and use
LD-LP schedule. If the APP message of the punctured bits
converges well in the first iteration, LD-LP can maximize the
advantages of the LD rule in the remaining iterations.
Compared to LP-LD, LD-LP has a slower convergence speed
for messages from PBs.
The work in this paper has conducted multiple simulations
on various
decoding methods, including flooding schedule
based LLR-BP (which referred to as BP later) and NMS
(which referred to as MS later) decoding algorithms, as well as
layered BP and MS in normal order. In addition, this paper
made a separate comparison between LD-LP and LP-LD
schedule. It was confirmed that the effect of LD-LP is similar
to LP-LD, but slightly better than the latter. In addition, this
paper also used Hcore for layered decoding simulation, tested all
24 layered sequences, and ultimately identified the best
performing scheduling schemes in the core matrix and
validated the effectiveness of LP schedule.
IV. SIMULATION RESULTS AND ANALYSIS
The simulation is based on MATLAB. The simulation uses
the QC-LDPC code of the 5G NR standard, with a BG of size
46 × 68, and expansion factor z is 24. The channel model is
an additive White Gaussian Noise (AWGN) channel. The
information bits are randomly generated by the source and then
encoded by the channel coding. The encoded sequence will be
modulated by Binary Phase Shift Keying (BPSK) and then
enter the AWGN channel. The signal received from the
channel is decoded using different methods after BPSK
demodulation, and the decoding result will be used for bit error
rate calculation.

V. CONCLUSION
The content of this paper mainly focuses on the layered
decoding of LDPC codes, exploring the impact of different
schedule
on
decoding
performance.
Experiments
were
conducted
on
different
update
sequences,
and
different
decoding algorithms were compared to validate the theory.
When the maximum number of iterations is 10, the result has
proven that the performance of layered decoding is better than
that of flooding decoding, and the performance of LLR-BP
decoding algorithm is indeed better than that of NMS decoding
algorithm. For LD-LP schedule, the lowest-degree (LD) is used
as the first priority parameter that has a significant impact on
the update order, and the second priority is the least-punctured-
bits (LP). When using the LLR-BP algorithm with a maximum
iteration of 10, the performance of LD-LP schedule is better
than sequential schedule decoding. And the performance of
LD-LP decoding is similar to LP-LD schedule. The results
have shown that LD-LP schedule is indeed the top two
priorities with the best performance in 5G NR LDPC codes. In
addition, LP-LD scheduling also has similar performance.The
experiment also simulated the error performance of the core
part of the LDPC code, which consists of four layers.We
evaluated the effect of the update order of the four layers on
the error performance of the code, and concluded that some
update orders (2314 order is the best) might give slightly better
error performance than others.
