AbstractвЂ”Channel coding is a key technology for improving the reliability of satellite communication data transmission links. The consultative committee for space data systems (CCSDS) recommends a set of low-density parity-check (LDPC) codes suitable for deep space communication. To meet the different needs of satellite communication, a high-throughput encoder based on recursive convolutional encoder (RCE) circuit is designed, which can achieve all LDPC code encodings recommended in the CCSDS standard for deep space communication. It can reduce hardware usage and improve intersatellite availability for transplanting.
KeywordsвЂ”low-density parity-check (LDPC) code, encoder, consultative committee for space data systems (CCSDS), recursive convolutional encoder (RCE), satellite
I. INTRODUCTION
Low-density parity-check (LDPC) code is a linear block code proposed by Dr. Gallager[1] in 1961. In recent years, a plethora of research have shown that LDPC codes have excellent performance close to the Shannon limit and have extensive application potential. The consultative committee for space data systems (CCSDS) has also recommended it as a channel encoding method for deep space communication[2].
LDPC code is a relatively special type of linear block code, whose check matrix has sparsity, with only a small number of "1" and most of them being "0". LDPC code is defined as the null space of a sparse matrix H with m rows and n columns, H is called the check matrix of LDPC code, and the ratio of the row weight p, column weight y, and code rate is far less than 1; Any two rows (columns) can only have a maximum of one 1 at the same position.
Compared to Turbo codes, which are also close to the Shannon limit, LDPC codes have the advantages of simpler decoding algorithm implementation, construction of arbitrary bit rates, and lower error levels.It is widely used in mobile and fixed wireless communication, satellite digital video, and magnetic/optical storage.
Because LDPC codes belong to linear block codes, their encoding process generally adopt the general encoding method of linear block codes, where the corresponding codeword

sequence is obtained from the information sequence based on the code generator matrix. Although the parity-check matrix of LDPC codes is very sparse, its generator matrix is not sparse, which makes its encoding complexity often proportional to the square of its code length. Its encoding complexity is relatively high, becoming an obstacle to its application.
In recent years, a large amount of research has focused on improving encoding speed, reducing encoding complexity, and saving encoding resources. Richardson and Urbanke proposed an approximate lower triangular encoding algorithm [3] based on traditional encoding algorithms, which preprocesses the original parity-check matrix to effectively control the encoding complexity to close to linear complexity. To further reduce coding complexity, scholars have begun to study structured coding methods, which require the code itself to have a special structural characteristics. A large amount of research has been conducted on low-complexity QC-LDPC codes. Li[4] proposed three classic encoder architectures based on shift-register-adderaccumulator (SRAA).
In the coding research for QC-LDPC codes, various scholars have made improvements to the basic methods based on the characteristics of different codes. In the study of LDPC code encoding under the CCSDS standard, Yao et al. designed a lowcomplexity parallel encoding method for LDPC codes in accordance with the CCSDS standard[5]. Reference [6] proposes a fast encoding architecture based on the generator matrix, which utilizes the QC nature of the generator matrix to achieve parallel operation of the parity-check sequence, effectively improving the encoding speed, but requiring a significant amount of hardware resources to cache the generator matrix information. Reference [7] proposes an LDPC code encoding architecture suitable for multi-rate. This architecture generates a submatrix with a QC structure based on a paritycheck matrix, and then performs a cyclic shift operation on the first row of the submatrix information, effectively reducing the hardware resources required to store matrix information and reducing the implementation complexity of the encoder. This method can also be applied to multi-rate coding in the CCSDS standard and has strong application value.

III. HARDWARE IMPLEMENTATION OF ENCODER
A. RCE Structure
The encoded message can be calculated by mGqc , where
Gqc is a quasi-cyclic matrix. Therefore, it can be implemented using n serial linear feedback shift registers with a total length of 4n / M . Before starting the operation, the binary pattern of the first row of the shift registers is stored in the top small box of the module. After opening the switch, the encoder inputs the information bits into the registers and performs update and shift operations. After completing one round of information processing, the next row of cyclic patterns is updated, and the switch is operated to read out the contents of the registers as the parity portion of the codeword.
We have adjusted the RCE structure to ensure coding flexibility while reducing complexity. RCE computing units can be cascaded, and after cascading, each RCE unit has the same input. The last bits of each RCE module are collected together to form a bus, which is then multiplexed to allocate one bit to the first position of each RCE unit. This method allows for different shift register lengths by configuring the system differently.
According to the CCSDS standard, when the information length is 1024, the code rates of 1/2, 2/3, and 4/5 correspond to M matrix sizes of 512, 256, and 128, respectively. When the information block length is 4096, the M matrix sizes are 2048, 1024, and 512, respectively. When the information block length is 16384, the M matrix sizes are 2048, 4096, and 8192, respectively. The greatest common divisor of these numbers is

B. System Structure The important modules of the encoder include: control
module, generator matrix storage module, check bit calculation module, and output module. The function of the control module is for the encoder to utilize matrix-stored ROM storage read addresses and to read matrix information based on the specific timing operated by the RCE module. The enable signals generated by the ROM are used to determine the code rate of

the current encoding process and select the corresponding encoding matrix.
The main function of the output module is to correctly and orderly output information bits and check bits. Firstly, output the frame synchronization flag. Secondly, output the system bits of the encoded data. Lastly, output the parity-check. At the same time, the state machine synchronizes with the input FIFO flag to determine whether the encoder is in operation.

V. CONCLUSION
This article designs a high-throughput encoder based on RCE circuit, which further improves the throughput of the encoder by improving the RCE structure. This encoder can

encode all LDPC codes recommended in the CCSDS standard. It can reduce hardware usage and improve the availability of inter-satellite communication. The maximum operating frequency of the encoder is 170.64Mhz and the throughput is 70120Mb/s. The research results indicate that it has certain practical value, but there are still limitations in this article. In future research, the structure of the encoder can be optimized to reduce resource consumption.
