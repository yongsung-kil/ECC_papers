An FPGA-based QC-LDPC encoder design

1st Yunling Yang College of Electronic Science National University of Defense Technology
Changsha, China ylyang@nudt.edu.cn

2nd Nan Li* College of Electronic Science National University of Defense Technology
Changsha, China linan@nudt.edu.cn

Abstract—Low-density parity-check (LDPC) codes have been widely used to deal with noise interference and improve the reliability of the system due to their excellent error correction performance. At the same time, with the increase of storage density in digital storage devices, LDPC codes are also applied to the error checking and correcting (ECC) module to solve the situation of random flipping of stored data bits. In this paper, the progressive edge growing (PEG) algorithm is used to construct a high-rate Quasi Cyclic-LDPC (QC-LDPC) code suitable for the ECC module of the storage device, and the generator matrix is analyzed, and finally a high-efficiency encoder is designed based on the FPGA to quickly generate the parity-check bits in the storage device. The encoder in this study can achieve a throughput of 106.4Mbps at a 50MHz clock.
Keywords-encoding algorithm, progressive edge growing algorithm, quasi-cyclic low-density parity-check (QC-LDPC) encoder.
I. INTRODUCTION
Information in digital systems is represented as sequences of "0" and "1" of different bit lengths, which are stored, transmitted, and processed. With the development and popularization of digital information systems, more and more environments with strong noise and strong interference also need to be equipped with corresponding digital equipment. The influence of noise may cause any bit of the internal signal of the system to be confused, and 0 will be recognized as 1 or 1 as 0, which will affect the accuracy of information reception. Based on Shannon's law, coding error correction technology can significantly enhance the anti-interference ability of digital information in the system and improve the reliability of the system. As one of the forward error correction codes, the Lowdensity parity-check (LDPC) code has been widely concerned for its excellent error correction characteristics[2] since it was first proposed by Dr. Gallager in 1963. Compared with generalpurpose processors, the integrated LPDC encoder designed in this paper can generate LDPC encoding sequences in a faster way in the digital system, allowing the system to achieve realtime processing.

B. Structure of QC-LDPC
Quasi-Cyclic LDPC (QC-LDPC) code is a type of structured LDPC code. Different from random LDPC codes, QC-LDPC codes are widely used because their quasi-cyclic characteristics can reduce the hardware resource overhead in the coding process. The sparse parity-check matrix of the QC-LDPC code contains a circular sq uare matrix. In these squares, the elements in each row are the result of the previous row of elements being shifted one place to the right in a cycle, and the first row is the last row to be shifted one place to the right to obtain [4]. Due to the rightshift characteristic of the cyclic square, after obtaining the elements in the first row and the dimensionality of the square, the entire cyclic matrix can be recovered by continuously circulating the right shift. The first row of the circular square is also called the generative vector.
The Hqc in (4) is a sparse parity-check matrix of QC-LDPC codes containing a × b submatrices, which dimensionality is Z. The sub-matrix hij can be a Z × Z cyclic square matrix, or an allzero matrix. The code length of the QC-LDPC code defined by the Hqc is n=bZ, and the number of check equations is m=aZ.

C. Construction of QC-LDPC
There are three common methods for the construction of QCLDPC codes: the parity-check matrix with a large minimum code distance based on finite geometry, the LDPC codes based on the principle of balanced incomplete block design, and the parity-check matrix based on the permutation matrix. The method selected in this paper is based on the quasi-cyclic expansion of the progressive edge growth (PEG) algorithm, which belongs to the permutation matrix construction method, and the construction process is divided into several steps[1]:

III. THE DESIGN OF THE ENCODER
A. Overall structure of decoder
The design idea of the encoder is like the encoding algorithm above, which is to calculate the check nodes by extending the first line element of the submatrix. The overall encoder architecture is shown in Fig.3, where the thick black thread represents the transmission of data, and the thin orange one represents the transmission of control signals. The control_encode module contains a finite state machine (FSM) to control the whole coding process, and four Encode_cell coding units are instantiated in the Parallel_encode module. The four columns of submatrices in the generator matrix are encoded simultaneously by using four parallel channels to improve the coding efficiency Parity_out module will shift the output and count after receiving the signal that the encoding operation is completed. done_out signal is given when all the check bits are output, representing the end of the entire encoding process.

According to the overall architecture diagram, when the encoding starts, the encoder receives 8-bit information bits from the outside, passes them to the separate Encode_cell module through the parallel encoding module for each clock cycle. C. Matrix mapping in memory
Based on the above encoding principle, after obtaining the first row of the cyclic submatrix in the generation matrix, the shift can be used for periodic extension to obtain the information of the entire matrix. To save on-chip memory resources, the design of the generator matrix memory unit in this section also follows this principle. In Fig. 6, the memory unit that generates the matrix will be instantiated as four ROMs, each with a depth of 32 and a width of 256bit. Each data in the ROM represents the first row of data in a cyclic submatrix, and the depth of the ROM corresponds to a column of 32 cyclic submatrices in the tile matrix. The advantage of instantiating four ROMs at the same time is that the 8-bit data entered by each clock cycle can be simultaneously XOR arithmetic on 1024 bits wide (four channels) to obtain the intermediate value, which improves the parallelism in encoding[5].

D. Computing unit in decoder
The encode_cell module is the core module in the encoding process. It starts by instantiating 256 xor_cell, corresponding to 256 columns in one submatrix, and the output of each column is the median of a check node. At the heart of the encode_cell is its shift propagation operation. If the counter in the parallel_encode module reaches 32, the new first-line matrix data will be extracted and stored in the buffer, if it is not 32, the data in the buffer will be shifted to the right by 8 bits, corresponding to the 8-bit input data that has been processed. After updating the data in the buffer, you need to shift the data to expand. The xor_cell module is an arithmetic unit in the encoding process, which uses combinatorial logic to complete the vector point multiplication operation of 8-bit input data and 8-bit matrix column information. Since the matrix multiplication in the LDPC encoding process is based on modulo 2 calculations, the point multiplication between vectors can be replaced by bitwise and operation. And based on the addition and XOR operations of module 2 are the same, XOR is performed on the bits and 8bit data after the operation. IV. ANALYSIS OF EXPERIMENTAL RESULTS
The encoder designed in this paper is based on Xilinx's Zynq UltraScale+ MPSoC family development platform, with the chip number XCZU7EV-2FFVC1156I. Under the co-simulation verification of FPGA and Modelsim, the sequence to be encoded is delivered to the LDPC_encode module and MATLAB at the same time .
The simulation results are saved in an output.txt file, and the output results are compared with the results obtained in MATLAB after encoding. As shown in the white box in Fig. 9, after the 8192 bits of data input is completed, the 1024-bit check sequence will be output after 4 clock cycles. V. CONCLUSIONS
In this paper, based on the encoding principle of LDPC codes and the acceleration idea of parallel real-time processing, the encoder of high bit rate QC-LDPC code constructed by PEG

algorithm is designed. And the results of comparative verification are used to verify the correctness of the code sequence after encoder. The integrated LDPC encoder allows for a reliable encoding function unit to be built quickly in the system , with a much faster encoding speed than on a generalpurpose processor.
