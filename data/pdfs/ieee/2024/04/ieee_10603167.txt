Abstract—In order to serve as a physical channel encoding 
solution for the 5G new radio (NR) data channel, and to meet the 
demand for quasi-cyclic low-density parity-check (QC-LDPC) 
codes, this paper proposes an efficient parallel encoder with a 
compatible structure. The design of this encoder is based on a 
forward substitution efficient encoding algorithm, which is 
integrated into a configurable circuit structure, allowing it to 
adapt flexibly to different base graphs of 5G-LDPC codes. 
Furthermore, the performance of this encoder can be enhanced 
through the utilization of a genetic algorithm (GA) optimization 
approach, aimed at achieving higher throughput or improved 
error correction (EC) capability. The optimized encoder not only 
provides 
high 
throughput, 
but 
also 
achieves 
optimal 
HUE(hardware usage efficiency).

I. INTRODUCTION 
Low-density parity-check (LDPC) codes were adopted as 
the encoding scheme for 5G new radio (NR) data channels at 
the 3GPP conference in 2016 [1]. Compared to Turbo codes, 
LDPC codes offer excellent error correction performance and 
the potential for highly parallel decoding [2]. Currently, 
researchers are focused on developing low-complexity LDPC 
encoding methods, which is a hot topic in LDPC application 
research. Due to LDPC codes approaching the Shannon limit in 
terms of performance, if a direct encoding method is used, the 
data storage and computational complexity of the encoder 
grow quadratically with the code length. To solve this problem, 
a direct encoding method based on the LU(Lower-Upper) 
algorithm was presented in [3], which eliminates the need to 
convert the 𝑯 matrix to the 𝑮 matrix. Specifically, the 𝑯 
matrix is decomposed into an information matrix 𝑯𝒔 and a 
parity-check matrix 𝑯𝑷. However, this method requires an LU 
decomposition of the 𝑯𝑷 matrix and 𝒂 left multiplication of 𝑯 
with a permutation matrix 𝒂. Determining the permutation 
matrix 𝒂 poses a challenge, thus making the encoding 
algorithm relatively complex. In addition, when the 𝑯 matrix is 
singular, it is impossible to perform LU decomposition, which 
leads to the incompleteness of the LU algorithm. Moreover, the 
LU algorithm is also not suitable for hardware implementation 
[4]. To address this issue, a simplified algorithm called 
RU(Richarson-Unbanke) algorithm was presented in [5]. This 
method transforms the sparse parity-check matrix 𝑯 into an 
approximate lower triangular form and achieves fast encoding 
by fast computing check bits. Many researchers have 
implemented RU encoding algorithm and its different 
improved versions on FPGA [6][7]. The disadvantage of the 
RU algorithm is that its column permutation may lead to non-
systematic codewords, which reduces the decoding efficiency 
at the receiving end. Reference  [8][9] adopts a hybrid method, 
where part of the check bits are calculated using the generator 
matrix, while the other part is calculated using the RU method. 
Due to its special code structure, 5G NR LDPC codes can be 
encoded using only forward substitution. Reference [9] 
proposes an efficient LDPC encoding algorithm, which 
quantizes the arrangement information of each submatrix and 
stores it in a ROM, greatly reducing the required storage space. 
This paper presents an efficient parallel LDPC encoding 
scheme based on forward substitution for 5G NR LDPC codes. 
The designed encoder can be configured at runtime to encode 
all LDPC codes covered by the 5G NR standard. The use of 
genetic algorithm optimization significantly improves the 
encoder's throughput and reduced latency, resulting in 
improved hardware utilization compared to other QC-LDPC 
encoding architectures. 
The remaining content of this paper is as follows: Section II 
provides a brief analysis of the characteristics of 5G LDPC 
codes. Section III introduces a highly parallel LDPC encoding 
algorithm compatible with 5G LDPC codes. Section IV 
presents a partially parallel architecture that is efficient and 
compatible with 5G NR LDPC encoders. Section V presents 
experimental results and comparative analysis. Finally, in 
Section VI, we summarize our work and draw conclusions. 
II. CHARACTERISTICS OF 5G NR LDPC CODES 
A.  Advantages of 5G NR LDPC 
To meet the needs of 5G communication, 5G new radio 
technology adopts various new transmission technologies, such 
as non-orthogonal multiple access (NOMA), massive MIMO 
and new channel coding techniques.

Compared with the 4G mobile communication system, the 
5G mobile communication system employs a pair of new 
channel coding techniques for the data channel and control 
channel respectively. Specifically, LDPC codes replace Turbo 
codes for the data channel, while polar codes replace cyclic 
codes for the control channel. 
LDPC codes were initially presented by Dr. Gallager, but 
due to hardware limitations at that time, they did not gain 
widespread recognition. It was not until the mid-1990s when 
hardware technology rapidly advanced that LDPC codes 
regained broad recognition in both the academic and industrial 
communities.  
Compared with the Turbo codes in 4G LTE network, 5G 
NR LDPC codes have the following advantages : 
• Improved area throughput efficiency and higher peak 
throughput. 
• Low decoding complexity and short decoding delay due 
to highly parallelized implementation, with the 
advantages being more pronounced at high code rates. 
• Superior decoding performance with an error floor 
Frame Error Rate (FER) close to or below 10^(-5) for 
all code lengths and rates. 
These advantages of NR LDPC codes are particularly 
suitable for the ultra-high throughput (20Gb/s downstream, 
10Gb/s) and URLLC requirements of 5G networks.

C. The development of LDPC encoding 
Currently, numerous researchers both domestically and 
internationally have conducted extensive studies on the 
implementation of low-complexity LDPC code encoders. 
Reference [10] proposes a parallel encoder based on recursive 
convolution under the CCSDS standard, achieving ideal 
throughput performance while maintaining resource utilization 
at a low level. Reference [11] fully utilizes the cyclic properties 
of QC-LDPC parity check matrices and the characteristics of 
equal row weights to propose an LDPC encoder based on 
block-row-cycle under the CMMB standard. Li et al. presented 
three efficient hardware encoding structures for QC-LDPC [12]: 
the first is a serial encoding structure called shift-register 
adder-accumulator (SRAA), which is not suitable for high-
speed encoding requirements; the second is a parallel encoding 
structure of SRAA, but it has a higher frame delay and requires 
a larger proportion of registers; the third is a two-stage 
encoding structure, known as TWO-STAGE, but it requires the 
parity check matrix to be full rank. 
Reference [13] introduced an encoding algorithm based on 
forward substitution. Experimental results indicate that the 
efficiency of this approach surpasses direct encoding and RU 
encoding methods by several folds, enabling rapid encoding. 
However, this method is only applicable to the BG1 base graph. 
Building upon this approach, this paper optimized a design for 
a partially parallel 5G LDPC encoder based on FPGA dynamic 
configuration, making it compatible with all base graphs.

IV. EFFICIENT PARALLEL 5G NR LDPC ENCODER WITH 
COMPATIBLE ARCHITECTURE 
The encoding algorithm described in Section III can be 
implemented with different degrees of parallelism. If a fully 
parallel implementation is adopted, it would require extremely 
high hardware resource utilization. However, considering 
flexibility as one of the main requirements in this paper, as 
shown in Fig.2, a partially parallel encoder architecture is 
presented for efficient encoding. This section describes the 
structure and operation of the partially parallel encoder. 
Fig. 2. The partially parallel encoder architecture for 5G NR QC-LDPC codes. 
The partially parallel encoder architecture designed in this 
paper for 5G NR LDPC codes accommodates different base 
matrices based on their sizes, which range from 2 to 384. In the 
figure, cyclic shift (CS) represents the cyclic shift operation, 
where the shift size is equal to the size of the base matrix. The 
input buffer is implemented using several serial-input-parallel-
output shift registers, which are used to store the received data 
by the encoder. The output buffer is used to store the encoded 
codewords. When the base matrix size is 192 <  𝑍 ≤ 384, every 
clock 
cycle, 
the 
encoder 
sequentially 
processes 
one 
CPM(circulant permutation matrices) per adjacent rows from 
the base matrix. It reads an information bit group from the 
input buffer with a maximum length of 384. The read 
information bit group is then used to calculate the λ vector 
using the formula described in section III (all GF(2) additions 
are implemented through XOR operations), when the base 
matrix size is 96 < 𝑍 ≤ 192, every clock cycle, the encoder 
sequentially processes up to two CPMs per two adjacent rows 
from the base matrix. In this case, the maximum length of the 
information bit group read from the input buffer is 192, when 
the base matrix size is 2 < 𝑍 ≤ 96, every clock cycle, the 
encoder sequentially processes up to four CPMs per four 
adjacent rows from the base matrix. The input buffer contains 
four identical information bit groups that undergo independent 
CS for calculation of four independent GF(2) sum operations. 
During the calculation of the λ vector, the information bits are 
also passed to the output codeword generator, which writes 
data to the output buffer. 
For different base matrix sizes, the parallelism of λ vector 
calculation varies. The Merge/Select λ module in Fig. 2 
arranges λ vector data to facilitate the core parity bit 
calculation.  
After the λ vector is computed, the encoder continues 
processing the next row in the base matrix in the same manner. 
During this process, the core parity bits are computed in 
parallel. The computed core parity bits are returned to the CS 
module for the calculation of other parity bits. Based on the 
CPM size, one of the four sets of core parity bits is selected, 
and the selected set is replicated in the same way as the 
information bits in the input buffer. Finally, the information bit 
group and the core parity bits are transferred to the CS module. 
The output of the GF(2) sum module consists of other parity 
bits, which are passed to the output codeword generator. The 
entire encoder architecture is designed as a pipelined structure 
to improve encoding efficiency.

C. The results of Throughput and Hardware Resource 
Utilization 
The encoder presented in this paper is implemented on the 
Altera Cyclone IV E development board using the 
EP4CE10F17C8N chip, achieving a maximum clock frequency 
of 𝑓𝑚𝑎𝑥= 402MHz. Fig.3 displays the throughput results for 
all sizes of 𝐵𝐺1 and 𝐵𝐺2, where it can be seen that the peak 
throughput of 𝐵𝐺1 reaches 17.68Gb/s and the peak throughput 
of 𝐵𝐺2  reaches 14.84Gb/s. The partially parallel encoder 
presented in this paper shows significant improvement in 
throughput compared to serial processing, especially for basic 
matrix sizes up to 192 or smaller.

with existing encoders, three additional 5G NR LDPC 
encoders are reproduced, including: (1) the fully serial

of hardware resources.

encoder, which is based on the forward substitution algorithm.
