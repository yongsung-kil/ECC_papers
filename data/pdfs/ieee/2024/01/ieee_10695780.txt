I. INTRODUCTION
A DISTRIBUTED storage system typically introduces redundancy into data storage to ensure fault tolerance and uphold storage reliability. Erasure codes are widely used in distributed storage system, which significantly achieve high reliability than replication at the same storage overhead [1]. One of the best-known and widely adopted erasure codes is Reed-Solomon (RS) codes [2], which satisfy the maximum distance separable (MDS) property. However, RS codes rely on the arithmetic over a larger finite field. Another promising class of erasure codes is array codes [3], where only exclusive OR (XOR) operations are involved in encoding and decoding procedures. A (k + r, k) MDS array code [3], [4] encodes k information packets of l bits into additional r parity packets of the same size, such that any k out of k + r packets can retrieve all kl information bits of k information packets. The number of bits stored in each packet, l, is called sub-packetization level.

Among non-trivial MDS array codes, the case r = 2 is always the first to deal with, such as EVENODD codes [5], [6], RDP codes [7], X-code [8], P-code [9], Short code [10], Liber8Tion code [11] and EVENODD+ codes [12], and it serves as a foundation for the cases r > 2. Moreover, (k + 2, k) MDS codes are also widely adopted in storage systems that utilize RAID (Redundant Array of Independent Disks) technology, particularly in RAID-6 implementations, which offer enhanced data protection. As extension of (k + 2, k) array codes, (k + r, k) array codes for r > 2 have also been extensively studied, such as Rabin-like codes [13], Cauchy codes [14], Blaum-Roth (BR) codes [15], [16], extended EVENODD codes [17], extended RDP codes [18], STAR+ codes [19], extended EVENODD+ codes [20] and so on, which can correct any r reasures.
