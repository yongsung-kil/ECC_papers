Lightweight Error Correction for In-Storage Acceleration of Large Language Model Inference
Jinwoo Jeong1, Byungmin Ahn2, Dongmin Shin2, Jungwook Choi1† 1Department of Electronic Engineering, Hanyang University, Seoul, Korea
2Samsung Electronics, Hwaseong, Korea

Abstract—As large language models (LLMs) expand their sizes, conventional GPU-based LLM inference systems face memory bandwidth and capacity limitations. An LLM inference accelerator using NAND flash storage has been proposed to overcome these challenges. However, this necessitates a significant expansion of flash channels to ensure adequate bandwidth for inference, subsequently escalating error correction code (ECC) costs. This paper examines the impact of flash memory errors on LLM inference accuracy and explores the possibility of lightweight ECC by leveraging LLM’s inherent error resilience. We analyze the impact of 1) high-order bit indices masking for FP32 LLM parameters, 2) clipping, and 3) a dependency by parameter type of error robustness, and show that a combination of them can reduce ECC bandwidth by up to 9.38%.
Index Terms—large language model, error correction code, NAND flash errors
I. INTRODUCTION
Efficiently inference of large language models (LLMs) has become vital in natural language processing tasks like text summarization, understanding, and generation. Transformerbased generative models, with their extensive parameters (e.g., 175B for GPT-3 and 530B for MT-NLG), demand significant memory capacity and bandwidth. These demands challenge the efficiency of GPU implementations: a single GPU often does not have enough DRAM capacity to accommodate these complete models and multi-GPU inference introduces considerable data communication overheads.
NAND flash-based in-storage processing is gaining traction as it addresses both capacity and bandwidth requirements. Previous research [1], [2] extends flash channels to 128 or more to ensure adequate bandwidth for retrieving parameters from flash memory. However, this extension poses a serious problem for storage systems. In particular, sophisticated error correction code (ECC) engines are required to ensure data reliability in NAND Flash-based devices, and the hardware cost of the flash channel controller increases proportionally with the bandwidth to ensure the integrity of all data.

To reduce the ECC overhead in bandwidth-enhanced flash storage systems, a viable strategy is to leverage the error patterns inherent to specific applications. Traditionally, ECC is structured to offset the most severe errors arising from imperfections and failures in flash-memory devices. Yet, if certain applications, like LLMs, naturally are tolerant to errors, this resilience can be harnessed to streamline ECC, thereby decreasing the associated hardware implementation costs. In particular, LLM uses an over-parameterization strategy that increases the number of trainable parameters to improve LLM’s accuracy. The important point is that over-parameterization allows the network to achieve a certain level of error resilience [3], which presents many opportunities for ECC.
In this study, we assess how flash memory errors affect the accuracy of LLM inference. Our goal is to understand if the intrinsic error tolerance of LLMs can be used to reduce the need for extensive error correction. By doing so, we aim to simplify the hardware demands of ECC, a critical component for SSDs.
II. BACKGROUND AND RELATED WORK
A. Reliability for NAND Flash Memory
The cause of errors in flash memory is essentially charge leakage due to wear on flash cells. The wear of flash cells becomes more severe as program/erase (P/E) cycles accumulate, and the raw bit error rate (RBER) increases exponentially. To ensure the reliability of flash-based systems, ECC must keep the uncorrectable bit error rate (UBER) below 10−15. When RBER is low, the cost of keeping UBER low is not significant, but as RBER increases, it becomes more difficult to achieve UBER requirements. High-performance low-density parity check (LDPC) code commonly used in modern SSDs can maintain reliability until RBER reaches 10−2 when performing corrections using soft information [4]. When RBER exceeds the threshold, a data recovery mechanism is triggered [5]. However, such a data recovery technique significantly degrades read performance, making it challenging for SSDs to ensure QoS. Therefore, the corresponding RBER can be assumed to be the maximum BER that can occur in a flashbased memory system.
B. Parameter Configuration of LLMs
Most of the modern LLMs are based on Transformer structures. Transformers represent associations between tokens with high accuracy through a mechanism called self-attention.

Additionally, multi-head attention divides an embedding vector by the number of heads and performs attention operations on each sub-vector. Parameters of Transformers consist of an embedding table, normalization, and linear transformation weights such as attention or feed-forward networks. Normalization parameters are responsible for properly scaling input vectors to avoid underfitting/overfitting. Parameters of attention and feed-forward networks are used to perform simple linear transformations.
C. Related Work
DNN inference accuracy is dependent on where the bit errors occur. [6], [7] mitigate the ECC requirement by polarizing the reliability of MSBs and LSBs in the weights. In MLC or TLC NAND flash memory, LSB pages and MSB pages have different reliability, so error-sensitive MSBs of weight are stored in LSB pages, and error-tolerant LSBs of weight are stored in MSB pages in NAND flash. Furthermore, the LSBs are modified so that the MSBs are stored in the least errorprone state. While the LSBs are stored in flash memory with an artificial error added, the MSBs benefit from being stored in SLC mode. Applying polarization techniques to MLCs [6] shows that they can withstand more than 16K P/E cycles without ECC.
In addition, most weights in DNN models tend to map within a certain range. To capitalize on this property, [8] stores data in an 8-bit integer format by changing it to a sign-magnitude representation. This way, the top two bits, excluding the sign, become fixed values with a high probability and can be treated as known values to maintain high inference accuracy without protecting them with ECC.
Existing research proposes techniques to minimize the loss of weights for bit errors in flash memory. As a result, the tolerable error margin is set naively or minimally. However, LLM employs a strategy to improve performance with overparameterization. The number of parameters is expected to continue to grow, and the resulting error robustness scales with it. To design cost-effective ECC schemes, we need to quantify the tolerable error bandwidth and correlate it with the number of parameters.
III. METHODOLOGY
A. Flash Error Condition
To ensure the integrity of the data that affects inference accuracy, we set an RBER as 10−2 based on modern SSDs. As discussed in Sec. II-A, ECC for NAND flash memory can satisfy reliability requirements when using soft-decision LDPC until an RBER reaches 10−2. By admitting errors under these harsh conditions, we can validate error robustness that can be leveraged over the lifetime of the flash memory.

B. Factors Affecting LLM Accuracy
Since bit errors occur randomly by nature, it is difficult to get reliable results if they are not controlled. Therefore, we will analyze the factors that can cause large deviations in LLM accuracy and explain how to control the corresponding errors.

As expected, errors injected at high-order bit indices cause accuracy collapse even at low BERs. However, as the order of the bit index in which errors are injected decreases, the number of errors that can be tolerated increases, and the perplexity tends to decrease. This suggests that there is a certain level of error capacity that LLM can tolerate. Therefore, we apply a bit index masking that incrementally expands the number of error-masked bits starting from the MSB. By clipping all parameters so that they fall between 0.0078125 and 2, the upper 5 bits of the exponent are fixed at a frequency of 100%. However, since the parameters being clipped are a small number of outliers which has a relatively large magnitude, accuracy can be compromised. And there may even be serious side effects if errors are injected, so validation is required.
3) Parameter Diversity: As mentioned in Sec. II-B, there are different kinds of parameters in LLM, and different kinds of parameters have different sensitivities to errors. According to [9], an embedding layer requires mixed-precision quantization because the contribution of different vocabularies in the table varies greatly. On the other hand, weights for linear transformation are relatively loss-robust, allowing for more aggressive quantization [10], [11]. These differences in robustness by parameter type can be similarly seen when NAND flash errors are injected. Identifying and separately protecting error-sensitive parameter types can increase the robustness of the overall model.
We experiment in three steps to understand the impact of each factor on the error robustness of LLM. First, we find the minimum masking level by using incrementally expansion of the bit index masking region. Second, we repeat the first step to the clipped model to check for the effects of clipping. Third, we apply a layer error mask to three types of parameters, namely embedding, normalization, and linear transformation, to understand the error sensitivity of each parameter. Finally, we put them all together to find a combination that minimizes the ECC bandwidth.

Layer masking to account for parameter diversity improves error robustness for all models. As shown in Table IV, GPT2S/M can reduce the minimum masking level to 8 through embedding layer masking, and GPT2-L/XL can improve perplexity through embedding, and normalization masking. Linear transformation weight accounts for most of the model parameters. Therefore, the number of errors that can be reduced by layer masking is large, but the perplexity improvement is too small. Considering that the cost of layer masking is ECC bandwidth, the reasonable targets for layer masking are the embedding layer and normalization parameters.

V. CONCLUSION
This research has shown that the hardware cost of NAND flash-based inference systems can be reduced by reducing ECC bandwidth by leveraging the error robustness of LLMs. The error robustness of the LLM increases with the depth of the layer, and the tolerable error range also increases. Among the parameters that make up the LLM, the embedding layer is the most sensitive to errors, but the impact becomes less and less as the model gets deeper. In addition, applying clipping techniques can effectively reduce the ECC bandwidth. In conclusion, for the GPT-XL model, protecting only 9.38% of the bandwidth is sufficient to maintain high inference accuracy until 16K P/E cycles of MLC NAND flash memory.
