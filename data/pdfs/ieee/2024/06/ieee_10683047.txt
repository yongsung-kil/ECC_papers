I. INTRODUCTION
Turbo, LDPC and Polar codes are three popular Shannon’s capacity-approaching codes, particularly at large block lengths and have found their usage in several wireless and wired communication systems. In the context of the 5th generation new radio (5GNR) of mobile cellular systems, the Polar and LDPC codes have been selected for error protection of the control and data channels, respectively. Both these codes are linear block codes. The LDPC code is defined by a parity check matrix, while polar codes are based on channel polarization theory [1]. LDPC codes, despite being a capacityapproaching code, face challenges of high encoding complexity and error floor especially at high signal to noise

ratios [2]. In contrast, polar codes have low encoding complexity and do not suffer from error floor [3] but exhibit sub-optimal BLER at smaller to medium block lengths (16 to 1024 bits) due to incomplete polarization of some information-bit channels [4]. Furthermore, the list decoding of Polar codes exhibits high computational complexity as the block length increases, so scalability with increasing block length is an issue for Polar codes.
One solution to these problems is to employ concatenated coding schemes, a concept introduced by Forney in 1965, combining inner and outer codes [5]. Since then, there has been a plethora of research on concatenated coding schemes, including the concatenation of polar codes as an inner code while extended Hamming codes / LDPC codes / BCH codes / Convolutional codes / Reed-Solomon codes have been used as the outer code in a serial concatenation [6-8]. Recently, from the literature [9], an idea borrowed from serial Turbo codes introduces an interleaver between serial concatenation of the polar code and BCH code where polar code is used as inner code while BCH code was deployed as an outer code. Analytical and simulated results confirm that such a serial concatenated scheme outperforms (in terms of BLER performance) a standalone polar-coded scheme under identical conditions with SCL decoding [9]. Similarly, in reference [10], the systematic polar codes in parallel concatenation with the convolutional codes scheme outperform standalone polar codes with BP decoding. Likewise, in [11], a turbo-like polar coding scheme known as parallel concatenated systematic polar codes (PCSPC) utilizes the BP algorithm as one of its decoders. Remarkably, PCSPC with a code length of 192 bits, achieves a nearly identical Bit Error Rate (BER) performance compared to standalone polar code (with BP decoding) having a code length of 256 bits. To mitigate the decoding complexity of PCSPC, an alternative soft output decoder, namely Soft Successive Cancellation (SCAN), was employed [12]. Later on, to enhance the BER performance, soft successive cancellation list decoding (SSCL) was introduced as component decoding of the PCSPC scheme [13]. Notably, this PCSPC scheme surpasses (in terms of BER performance) previous PCSPC schemes that relied on BP/SCAN decoding [13].
The turbo-style schemes employing polar codes, specifically those utilizing soft-input/soft-output SISO iterative decoding like traditional BP or SSCL are excessively

Importantly, these schemes exhibit poor BLER performance compared to state-of-the-art polar code employing CRC-aided listed decoding (CA-SCL) which raises a question mark as to the practical usage of such concatenated schemes. Our research in this paper answers this important question, “can the turbo-like concatenated polar coded schemes (with soft-input / soft-output i.e., SISO decoding) outperform the existing state of the art standalone polar codes with CA-SCL decoding (which is non-SISO list decoding”. In our proposed scheme, we change the decoder of turbo-polar codes from traditional SCAN or SSCL to lowdensity parity check (LDPC) like BP decoding, which is based on the sparse factor graph of polar codes. Details of this lowdensity parity-check (LDPC)-like decoding are well documented in literature [14]. We will show in the results section (Section V) that our proposed Turbo Polar code (parallel concatenated systematic polar codes with an interleaver and SISO iterative decoding with LDPC like BP decoding) can indeed outperform classical standalone polar codes as the block length increases (more precisely larger than 3072 bits). As another contribution of this paper, we extend the application of Turbo principle from Turbo-Polar codes to a mix of both polar and LDPC codes (i.e. one encoder being polar code and the other being LDPC code both concatenated in parallel separated by an interleaver) resulting in TurboLDPC-Polar architecture, and then to Turbo-LDPC architecture (where both constituent parallel concatenated encoders are LDPC codes) and analyze their BLER and complexity performances. Our proposed parallel concatenated schemes demonstrate enhanced performance, with TurboLDPC-Polar and Turbo-LDPC outperforming the state-of-theart CA-SCL (L = 32) Polar decoder at higher block lengths (> 3072 bits). This superiority is evident in both BLER and computational complexity metrics.

B. Belief propagation soft output decoding on the sparse factor graph of a polar code

Since Turbo Polar code requires SISO decoding, we need to use a decoding method that can provide soft output information. For this purpose, we aim to use BP decoding of LDPC codes since it is known that a factor graph can be made from a polar code [14]. However, the factor graph formed from a polar code may not be sparse in which case BP decoding is not very beneficial. However, if the resulting factor graph of Polar code is further processed and converted into a sparse factor graph then applying LDPC style BP decoding on such a sparse factor graph is expected to provide good decoding performance. The way this is done is explained below.
We take the polar code's generator matrix and convert it into its corresponding parity check matrix 9 of the polar code. However, the obtained parity check matrix 9 is usually very dense, leading to decoding errors if the traditional BP is applied. Therefore, to reduce its sparsity (as dense graph contains a lot of short cycles), a pruning algorithm is applied to redundant check and parity nodes that translates it to the sparse matrix. The complete pruning algorithm is well documented in reference [14]. Hence, in this way, polar codes can be effectively decoded by using the traditional BP decoding of LDPC codes over the sparse graph, enabling soft input and soft output decoding of the polar code. This process of converting to the sparse matrix 9 significantly reduces the size of the 9 matrix, leading to reduced computation complexity without sacrificing the error-rate performance [14].

B. Complexity Results
We also investigated the computational complexity comparison in terms of the floating-point operations of the proposed concatenated schemes i.e., Turbo-Polar, TurboLDPC-Polar and Turbo-LDPC with the state-of-the-art CASCL decoding of standalone Polar codes with a list size of 16 and 32 bits, respectively. It can be seen from Fig. 13 that at a smaller block length such as 192 bits, the Turbo-polar scheme has less complexity than polar coded scheme decoded with list size of 32. However, once the list reduces from 32 to 16 bits the Polar code decoded with CA-SCL (L = 16) has the least complexity among the all the compared schemes. However, once the block size increases from the 192 to 6144 bits, it can be observed from Fig. 13 to Fig. 17 that the complexity of Turbo-LDPC codes is less as compared to all the concatenated schemes but slightly more than that of CA-SCL decoding with list size of 16. Nevertheless, at the block length of 6144 bits, the Turbo-LPDC scheme has even less computational complexity as compared to standalone Polar code decoded with CA-SCL (L = 16) decoding. Therefore, it can be inferred that that the proposed turbo concatenated schemes offer almost the same complexity at the lower block lengths and less complexity at relatively larger block lengths while at the same time enjoying superior BLER performance especially at larger block lengths.

VI. CONCLUSIONS
In this paper, we introduced three parallel concatenated coded schemes, namely Turbo-Polar, Turbo-LDPC-Polar and Turbo-LDPC schemes and investigated their performance (both BLER and complexity measured in terms of floatingpoint operations) using the traditional Belief Propagation (BP) algorithm. Our findings revealed that the Turbo-LDPC-Polar

and Turbo-LDPC schemes exhibit superior BLER performance as compared to the standalone polar cod scheme decoded with CA-SCL (L = 32) when operating on relatively large block sizes exceeding 3072 bits. Importantly, these enhancements in BLER performance are achieved while maintaining significantly reduced computational decoding complexity compared to the Polar code decoded with CA-SCL (L = 32). Furthermore, the Turbo-LDPC scheme surpasses the BLER performance of the standalone 5GNR LDPC scheme, offering a notable coding gain of approximately 0.4 dB and about 1 dB over AWGN and correlated Rayleigh fading channels respectively, in various block lengths. We will further analyze and optimize the proposed code design using extrinsic information transfer chart (EXIT) chart and density evolution techniques as the next step in our research. This research is of great significance as it provides valuable insights into the selection of channel coding schemes for the evolving landscape of 6G wireless communication systems.
