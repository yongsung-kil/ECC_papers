Abstract—Given the rapid advancement of quantum computing technology, the McEliece cryptosystem, rooted in coding theory, stands out as one of the few cryptographic systems resilient against quantum attacks. Nevertheless, the system’s extensive public key length and low transmission rate have hindered its practical utility. To overcome these challenges, we propose a methodology to implement the McEliece cryptosystem employing quasi-cyclic low-density parity-check (QC-LDPC) codes, thereby diminishing the public key size albeit at the expense of heightened decryption complexity. Furthermore, we introduce a rowlayered normalized min-sum low-density parity-check decoding algorithm aimed at minimizing system complexity and optimizing hardware resource utilization.
Index Terms—QC-LDPC codes, McEliece cryptosystem
I. INTRODUCTION
With the evolution of society, there has been a rising demand for data protection. In the digital age, an increasing volume of data, including sensitive information like credit card details, personal privacy, and classified military intelligence, is transmitted over public channels. Given the vulnerability of signals transmitted over public channels to interception by third parties, it is imperative to safeguard the sensitive information contained within these signals from unauthorized access.
In cryptography, data security methodologies typically involve integrating encryption and decryption modules into communication systems to enhance security. However, this incorporation increases system complexity. Encryption and decryption processes, central to cryptography, serve three primary purposes: compression, confidentiality, and error correction. Error-correction coding ensures reliable information transmission, necessitating both confidentiality and error-correction coding to uphold communication reliability and security. Exploring the feasibility of jointly designing these modules to streamline systems is a key research area.
In the dynamic field of information security, investigating the integration of McEliece cryptographic systems with lowdensity parity-check (LDPC) codes is crucial. Developing

communication systems that excel in error correction and confidentiality is paramount. LDPC codes are renowned for their efficiency in error correction, while McEliece encryption is rooted in error-correction codes. Enhancing information security in satellite communication systems is critical, emphasizing the importance of safeguarding data and communication channel integrity.
In 2000, the integration of LDPC codes into the McEliece public key cryptosystem was pioneered with the aim of curtailing the length of the system’s public key, leveraging the sparse characteristics inherent in LDPC codes [1]. However, the study noted that for ensuring robust security, a non-sparse matrix had to be combined with the check matrix, thereby undermining its effectiveness in key length reduction. Despite the inability to achieve significant key length reduction in this instance, the endeavor indirectly underscored the potential applicability of LDPC codes within the framework of the McEliece public key cryptosystem.
Subsequently, the integration of quasi-cyclic low-density parity-check (QC-LDPC) codes into the construction of the McEliece public key cryptosystem was proposed in [2]. By harnessing the quasi-cyclic structure inherent in matrices, this approach notably diminished the key length, thereby partially mitigating the inherent limitations of the McEliece public key cryptosystem. Reference [3] utilized the concept of difference families to fabricate QC-LDPC codes and conducted a comprehensive performance analysis of the system. While the system achieved a reduced key length, initial designs fell short in defending against certain attacks. Consequently, the refinement of the McEliece public key cryptosystem, which involved eschewing the selection of permutation-equivalent codewords to bolster its security posture, was later proposed in [4] [5]. Despite the key length reduction facilitated by selecting quasi-cyclic codewords, this approach presented a trade-off in terms of system work factor, which could be mitigated by augmenting the code length.

Shooshtari demonstrated that the utilization of irregular LDPC codes can enhance security performance without necessitating an increase in key length [6]. Reference [7] introduced a novel codeword design approach by adjusting the degree distribution of LDPC codes to bolster their secrecy capabilities, contended that LDPC codes intended for secrecy should prioritize secrecy performance over error correction capabilities, thus advocating for the adoption of moderate density parity-check (MDPC) codes with modified density. Analytical findings suggest that, under equivalent secrecy levels, the adoption of quasi-cyclic moderate density parity-check (QC-MDPC) codes yields a shorter key length in comparison to QC-LDPC codes [8].
For the McEliece public key cryptosystem based on QCLDPC codes, extensive research has been conducted on various attack methods. While certain flaws in the system’s structure have been identified, none of these attack methods have succeeded in fully compromising the system. For instance, attacks targeting the system when the quasi-cyclic code has an even cycle length have been discussed, but such attacks can be mitigated by choosing prime numbers [9]. These issues can be entirely circumvented during the scheme design process. Similar to the conventional McEliece public key cryptosystem, the LDPC-based McEliece public key cryptosystem can undergo transformation into a symmetric key cryptosystem. A scheme outlined in reference [10] proposes such a transformation, asserting that unlike previous schemes, this symmetric key cryptosystem does not compromise security level or error correction capability.
In this work, we consider that although the key matrix for the McEliece cryptosystem based on QC-LDPC codes is a reversible matrix, it is not sparse, resulting in high implementation complexity. To mitigate the overall complexity, we opt for a layered QC-LDPC algorithm. In comparison to traditional decoding algorithms for the McEliece cryptosystem, the posterior probability converges faster, reducing the number of iterations and lowering hardware complexity.

A. RN Attack

RN attack is an effective method against McEliece symmetric key scheme. The error vectors used in the McEliece symmetric key scheme limit their hamming weight:

where d represents the minimum distance between codewords of the (n, k) linear block code, thus the value of t/n is generally small. RN attack demonstrates that in such scenarios, this symmetric key scheme is unable to withstand chosenplaintext attacks (CPA). A chosen-plaintext attack refers to the attacker’s ability to obtain a certain number of ciphertextplaintext pairs to attack the cryptographic system. RN attack is a type of chosen-plaintext attack, specifically represented as:
• Select two plaintext groups, M1 and M2, each differing by only 1 bit, assuming the difference occurs at the i-th bit.
• Compute C1 − C2, where C1 and C2 are the corresponding ciphertexts generated from plaintexts M1 and M2 using an encryption scheme:

C1 − C2 = gi′ + (Z1 − Z2),

(10)

where gi represents the i-th column of the encoding matrix G′, Z1 and Z2 represent noise or errors introduced during the encryption process. • Since the hamming weight of Z1 − Z2 is at most 2t, if t/n takes a relatively small value, then the vast majority of values for C1 −C2 will be the same as gi. This process can be repeated, and an estimate of G′ can be obtained through a large number of votes.
Error vectors are generated using a pseudo-random number generator. This is partly due to the complexity of implementing a hardware random number generator, which generates numbers close to true random numbers through physical processes. Another reason is that during the decryption process, error vectors identical to those generated during encryption need

to be produced. Taking all these factors into consideration,

using a pseudo-random number generator that meets certain

requirements to generate error vectors is a good choice.

Additionally, to ensure security, the probability of 0s and 1s

occurring in the error vector should be roughly equal.

There are multiple algorithms for generating pseudo-random

sequences, but considering implementation complexity, a lin-

ear feedback shift register (LFSR) [14] is used here to generate

the error vector. Although LFSR can produce pseudo-random

sequences with long periods, there still exist some attacks

against LFSR. Therefore, in the system design, the error vector

ep does not directly correspond to the output of LFSR. ep can

be represented as:

ep = s · H.

(11)

In this expression, s represents the pseudo-random sequence output by the LFSR, H represents the parity-check matrix of

the LDPC code, and ep is the error vector in the cryptographic system. s is a pseudo-random sequence of length m, where m = n − k, with n being the coded length and k being the

message length. Therefore, the seed length provided to the

LFSR should be at most m, and the seed provided to the

LFSR cannot be all zeros.

B. Sruik-Tilburg Attack
The Sruik-Tilburg Attack is an effective method against the McEliece symmetric key scheme proposed in [15]. The RaoNam scheme is a symmetric key cryptosystem with a high transmission rate. In this scheme, error vectors are generated using predefined error patterns, with an average hamming weight of approximately half the code length. The Rao-Nam scheme could withstand RN attacks when using (n, k) linear block codes, with a computational complexity of at least O n2k . However, the plaintext selection attack (Sruik-Tilburg Attack) demonstrates that the Rao-Nam scheme is insecure for specific code lengths, revealing three vulnerabilities.
The first vulnerability of the Rao-Nam scheme is the limited number of available error vectors for the recommended parameters. If the number of error vectors is N , then all the error vectors can be obtained with only a few calculations, approximately O(N log N ). Therefore, under the recommended parameters, the RN scheme is not secure enough.

The second point is that the Rao-Nam scheme uses a permutation matrix P, which only rearranges the bits in a specific order without changing their values. Once the set of all error vectors is obtained, the information of the permutation matrix P will be leaked. For this design example, the generation method of error vectors is different, making this type of attack ineffective. The third point is that the encryption process is linear, which according to Struik, may not provide sufficient security. As mentioned in the RN attack, it is possible to use large table voting analysis. Struik suggests the use of non-linear transformations for security, but adopting non-linear transformations may introduce other issues.
In the case of this design example, instead of using nonlinear transformations, it relies on a longer code length and a large number of error vectors with a hamming weight approximately half the code length to ensure the security of the cryptographic system. In conclusion, this design example is capable of countering the struik-tilburg attack.

V. CONCLUSIONS
We have proposed a enhanced QC-LDPC code-based McEliece cryptosystem utilizing the row-layer normalized min-sum decoding algorithm in this letter. This decoding algorithm is of great value because we can use it to more promptly utilize the updated information and produce more reliable information for use by adjacent nodes in the next layer, enabling belief updates to occur more rapidly. Simulation

results show that using the row-layer decoding algorithm
leads to faster convergence of posterior probabilities can
help to asignificantly improve hardware utilization, reduce the
number of iterations, and lower the complexity of hardware
implementation over the conventional McEliece cryptographic
system. For future work, we plan to focus on improving coding
efficiency and conducting in-depth analysis of security, among
other aspects, to further advance the research.
