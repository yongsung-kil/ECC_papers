Abstract—The research work explores the implementation of an LDPC decoder utilizing a barrel shifter architecture tailored for processing 128-bit data. LDPC codes are recognized for their efficacy in error correction across various communication systems, necessitating efficient decoding mechanisms. Integration of a barrel shifter into the decoder architecture facilitates essential shifting operations required for LDPC decoding, thereby enhancing throughput and minimizing latency. The study delves into the design considerations for the barrel shifterbased implementation and elucidates the LDPC decoding algorithm. Through simulation, the performance of the proposed decoder is evaluated in terms of throughput, latency, and resource utilization. The experimental results validate the feasibility and effectiveness of the LDPC decoder with a barrel shifter architecture for processing 128-bit data, underscoring its potential for integrationinto high-speed communication systems.
Index Terms—LDPC decoder, Barrel shifter, Error correction, Belief propagation algorithm, Log-Likelihood Ratios, Wireless Communication.
I. INTRODUCTION
The ”Implementation of LDPC Decoder using Barrel Shifter” project is a prime example of error correction coding progress, meticulously developed to satisfy the needs of modern data storage and transmission systems. Low-density parity-check (LDPC) codes have become more well-known in the field of error correction due to their exceptional qualities. However, optimizing the shifting operations in LDPC decoders designed for LDPC codes continues to pose a considerable issue, resulting in inferior decoding performance. The need to solve this pressing issue and improve LDPC decoder efficiency is what drives this effort. One important way to deal with the complexities of LDPC decoding algorithms is to introduce a specialized extended barrel-shifter. With its precise design, the extended barrel-shifter enhances the versatility of LDPC decoders by providing a flexible instrument for error correction within the framework of LDPC codes. The overall objective is to substantially enhance error correction procedures, which ensure dependable, high-throughput transmission and

effective data storage. The expanded barrel-shifter is essential for increasing decoding speed, scalability, and compatibility with different LDPC code parameters because it optimizes shifting processes. This adaptability is essential for fulfilling the changing requirements of various applications and contemporary communication standards. The code can be deployedin settings with limited hardware since it is designed for the Xilinx Vivado platform and built using the Verilog hardware description language. The driving force behind this initiativeis rooted in dedication to technological advancement. Due to the development of advanced communication systems and the constant increase in data volume, there is a growing demand for robust error-correction tools that can also be adjusted to different coding situations. By addressing the challenges related to LDPC decoding, the primary goal of this research is to increase the LDPC decoders’ speed, reliability, and adaptability in real-world communication and data storagesettings. When decoding 128-bit data with LDPC, estimated bits are iteratively refined using received data and parity checks; this process usually takes many iterations. In a variety of applications, including satellite communication, wireless communication, and storage systems, where dependable data transfer and storage are essential for flawless functioning and data integrity, this approach ensures accurate error fixing.

Fig. 3. Shifting Process of a Barrel Shifter
The following formulas can be used to implement rotations to the left and right in a barrel shifter. Left Rotation: When rotating to the left by k positions, the output data out consists of:
• The k least essential bits of the input data in, denoted by data in[k − 1 : 0]
• The remaining bits from the (N−1)th bit to the kth bit, denoted by data in[N − 1 : k] The justification is that data in[k −1 : 0] represents the k least essential bits in the input, while data in[N −1 : k] represents the remaining bits of the input. In the output data data out, the k least significant bits are filled with the k least significant bits of the input data, while the remaining bits are filled with the (N − K) most significant bits of the input data.
Right Rotation: When rotating to the right by k positions, the output data-out consists of the (N − K) least significant bits of the input data, denoted by data in[N —k − 1 : 0], and the remaining bits of the input data, from the (N− 1)th bit to the (N—k)th bit, denoted by data in[N− 1 : N− k]. The justification for this is that data in[N —k − 1 : 0] represents the (N −k) least significant bits of the input data, while data in[N — 1 : N − k]represents the remaining bits of the input. In the output data data out, the (N−k) least significant bits are filled with the (N − k) least significant bits of the input data, while the remaining bits are filled with the k most significant bits of the input data. In comparing a 128-bit conventional shifter to a 128-bit barrel shifter, the conventional shifter exhibits higher hardware complexity and resource usage due to its sequential shifting nature, while the barrel shifter offers lower hardware complex-

ity and resource usage through parallel shifting. Additionally, the conventional shifter results in higher latency for shifting 128-bit data, whereas the barrel shifter achieves lower latency and higher throughput. Lastly, while the conventional shifter is limited to basic left and right shifts by a fixed number of positions, the barrel shifter provides greater flexibility, supporting various shift operations, including left, right, and rotate, by any number of positions. Nevertheless, it provides notably enhanced flexibility, throughput, and overall efficiency thanks to its capability to execute variable shifts. This characteristic renders the 128-bit barrel shifter more appropriate for tasks that demand dynamic data manipulation and accuracy, like LDPC decoding with 128-bit data streams. Consequently, the 128-bit barrel shifter is typically regarded as the superior option for these specific situations.
