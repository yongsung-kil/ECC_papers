Mixed Integer Linear Programming Method For Energy-based Model Trapping Sets Enumerating

Abstract—The paper introduces an efficient method for identifying trapping sets in codes on the graphs, employing a mixed linear programming approach. The method guarantees a comprehensive search, essential for nonlinear channels in communication systems, energy-based models, natural language processing DNN, metric learning and other intricate dynamic systems. An in-depth analysis of symmetry and asymmetry properties of dynamic systems, represented by trapping sets TS(a, 0)–codewords and TS(a, b)-pseudocodewords respectively, offers a thorough understanding of their dynamics. Implemented in Python, the proposed method is publicly available on GitHub and supports both the Community Edition version, with restrictions on the number of conditions, and the Commercial Edition CPLEX. The technique involves solving a mixed integer linear programming problem using a predefined list of variable nodes participating in the shortest (short) cycles with small Extrinsic Message Degree values within the code graph. These cycles, akin to topological invariants, represent multidimensional voids formed by code and pseudo-code words. Consequently, the method can be viewed as an approach for constructing topological complexes and calculating topological invariants. The method was applied to search for trapping sets in LDPC codes, utilizing the mathematical linear programming package IBM CPLEX Optimization Studio version 22.1.0.0 The computations ran on a 16-core AMD Ryzen 3950X processor with 128GB KF3200 DDR4 RAM, utilizing 32 threads. In the Margulis code (2640, 1320), the proposed method identified the trapping set TS(6,6) in just 0.29 seconds. Notably, compared to the Velasquez-Subramani method, the proposed method achieved a speedup of 15082 times. Leveraging its high speed and thorough search capabilities, the method successfully identified trapping sets TS(62,16) and TS(52,14) for the first time in the Margulis code (4896, 2474). Finally, employing the proposed method with CPLEX Community Edition version 22.1.1.0, we discovered trapping sets, specifically T S(102, 2) and T S(108, 4), within the Mackay LDPC code (408, 204). These findings necessitated a brute-force trapping set search, demanding more than 10101 operations.
Keywords—Codes on the Graphs; LDPC-codes; Trapping Sets; IBM CPLEX; Symmetry; Asymmetry; Dynamic System; Topology Invariant; energy-based models; EBM; DNN
I. INTRODUCTION
The utilization of codes on graphs has significantly ad-
vanced various systems, including quantum source coding
[1, 2], quantum computing [2, 3], information storage across
different media types (magnetic, solid-state, optical), and var-
ious communication technologies such as wireless, optical,
and satellite communications [4-7]. Additionally, it has found

applications in energy-based models (EBMs) operating in discrete spaces [8, 9].
Graph-based codes employing soft decoding techniques for error correction offer substantial energy gains. In powerlimited scenarios, gains range from 1.5 to 2.5 dB, increasing to over 4 dB in band-limited situations, considering the benefits from working with modulation constellations during message transmission (or playback) through noisy channels.
Soft decoding methods leverage soft metrics, such as probabilities or log-likelihoods, derived from the received message using a code parity check matrix. Maximum likelihood decoding methods, which identify the most likely codeword, offer optimal noise immunity. However, their practical application is constrained by the exponential increase in decoding complexity as the code length grows. In real-world scenarios like wireless and optical communications, codes with lengths reaching tens of thousands or even hundreds of thousands of bits are commonplace.
Addressing the challenge of exponential complexity growth involves employing message passing method (Belief Propagation) and their approximations. The effectiveness of these methods is hindered by asymmetric structures in the code graph, such as Stopping Sets for a binary erase channel, Trapping Sets (TS) for an AWGN channel, and Trapping Sets for a Rayleigh channel with fast (selective) fading, [10].
Energy-Based Models (EBMs) offer flexibility and applicability to various data types and structures. They can capture intricate, high-order interactions between variables, making them valuable tools for modeling data dependencies. EBMs exhibit robustness to adversarial attacks by increasing code distance symmetry in the energy landscape (Trapping set (a,0) energy ground states). Additionally, they serve as generative models, capable of generating new data samples resembling observed data, beneficial in tasks like image and text generation. Despite these advantages, training EBMs poses challenges due to the intractability of the partition function and potential convergence issues due T S(a, b) loss function landscape, [11].
Improving codes on the graph for enhanced reliability in computing systems, data storage, machine learning, Deep Neural Networks and communication systems necessitates the identification and neutralization of asymmetric structures in the code graph. Locating such subgraph structures is an NP-

Two approaches exist for finding trapping sets. The first employs the Monte Carlo method with a biased probability estimate using Importance Sampling, relying on a channel sampling. This approach, presented in papers [12]–[17], demonstrates high performance but depends on decoder parameters and channel characteristics, with a finite probability of missing trapping sets, [22]–[24]. The second approach, explored in this article, focuses on the development of a decoder-independent method for searching asymmetric structures of subgraphs of trapping sets using linear programming, [18]–[21]. This linear programming method offers trapping sets, independent of decoder parameters and channel characteristics, albeit with lower computational complexity compared to existing approaches.

II. CODE ON THE GRAPHS. TRAPPING SETS

Codes on graphs serve as mathematical graphical models

designed for representing and manipulating factorized forms

of probability distributions. They find particular significance

in various applications, including forward-error correction,

source coding, cryptography, Bayesian networks, Markov ran-

dom fields, and hidden Markov models. The graphical repre-

sentation of dependencies in these codes plays a crucial role

in understanding and solving practical problems. The structure

of these codes is defined by a factor graph G= (V ∪ C, E),

comprising n vertices of variables V and (n-k) factor vertices

(check nodes) C, where k represents the code dimension. The

edges E connect the vertices in the graph.

The connections between the vertices are encapsulated

in the matrix H, referred to as the parity-check matrix in

forward error correction. The matrix’s rows define equations

over a finite field GF (q), q = 2, given by the expression

Factor functions take a subset of variable nodes as input and

(E) connect the vertices of variables to the vertices of factors

based on the variables’ roles as arguments for specific factor

functions. Figure 1 illustrates an example of a factor graph,

A factor graph provides a clear and concise representation of

complex probabilistic models, facilitating logical inferences,

optimization, training, and error correction within a specific

model or channel. The graph’s structure reflects conditional

independence relationships between variables in a factored

probability distribution, crucial for probabilistic reasoning.

A specific category of graph codes, extensively employed

in forward-correcting coding, is noise-correcting linear codes.

Notably, within this category, LDPC codes are essential. These
codes exhibit a low density of non-zero symbols in their
parity matrix, enabling soft decision decoding using Belief
Propagation (BP) with relatively low complexity. The BP
method demonstrates significantly greater correction power
compared to hard decision decoding methods. The factor graph of the LDPC code, denoted as G and
illustrated in Fig. 1, is a Tanner graph comprising two vertex sets. One set encompasses m = n − k factor (check) vertices (nodes) c0, c1, . . . , cm−1, representing m rows of the matrix H. The second set comprises n variable (code) vertices {v0, v1, . . . , vn−1}, corresponding to the n columns of the matrix H. The decoding efficiency using the BP method is hindered by the existence of cycles in the factor graph, forming trapping sets (TS). Trapping sets, denoted as TS(a, b), are subgraphs composed of a variable nodes, where b of them are incident to check nodes with odd degrees [12], [25]. Trapping
sets lead to decoding errors in the BP method. Examples of
trapping sets are depicted in Fig. 2.

III. LINEAR PROGRAMMING TRAPPING SETS SEARCH

In their work [20], Velazquez and Subramani proposed a

method for identifying trapping sets in a factor graph (G =

(V ∪ C, E)) using integer linear programming.

One drawback of the method lies in its exponential com-

putational complexity growth with an increasing parameter a

of the trapping set. A complete search necessitates analyzing

n a

points in the solution search space.

To mitigate this, we propose modifications to the trapping

set search method using linear programming. This modifica-

tion leverages the presence of code vertices in the shortest

cycles of the graph. By incorporating many code vertices

cyclevns, this method, particularly in the early stages of the linear solver (presolver), efficiently eliminates a maximal

number of infeasible conditions.

V. CONCLUSION
The article introduces a method for searching trapping sets based on mixed integer linear programming, utilizing an a priori list of code vertices. The Python implementation of the proposed method is available on GitHub [33]. The method is efficient and ensures a comprehensive search. In the Margulis code (2640, 1320), the proposed method successfully identified the trapping set TS(6,6) within a time frame of 0.29 seconds. The method’s speedup compared to the Velazquez-Subramani method is an impressive 15082 times. A key distinction between the proposed trapping set search method and the Cole significance sampling method lies in the completeness of the resulting set of trapping sets, attributed to the independence of the linear programming method from decoder parameters. The high speed and completeness of the search enabled the discovery of trapping sets TS(62,16) and TS(52,14) for the first time in the Margulis code (4896, 2474). We discovered trapping sets, specifically T S(102, 2) and T S(108, 4), within the Mackay LDPC code (408, 204). These findings necessitated a brute-force trapping set search, demanding more than 10101 operations.
