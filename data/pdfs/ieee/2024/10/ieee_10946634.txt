Abstract—To meet the demand for quasi-cyclic low-density parity-check (QC-LDPC) codes, this paper proposes an encoder architecture specifically designed for QC-LDPC coding, compatible with the majority of QC-LDPC codes. The encoder is designed based on a forward substitution(FS) efficient coding algorithm, implemented in a partially parallel structure for efficient encoding,with the optimized encoder significantly improving hardware usage efficiency (HUE).
Index Terms—hardware usage efficiency (HUE), low-density parity-check (LDPC) code, quasi-cyclic (QC)
I. INTRODUCTION
Low-density parity-check codes (LDPC), proposed by Dr. Gallager in 1961 [1], are a type of linear block code. Recent research has demonstrated that LDPC codes exhibit excellent performance close to the Shannon limit, making them highly promising for various applications. LDPC codes are a specific type of linear block codes characterized by their sparse check matrix, where the matrix contains a minimal number of “1”s and the majority of entries are “0”s. LDPC code is defined as the null space of a sparse matrix (H), which have m rows and n columns. The matrix (H) is known as the check matrix of the LDPC codes. The code’s parameters include row weight (ρ), column weight (γ), and code rate.Additionally, any two rows (or columns) in (H) can have at most one “1” in the same position.
However, the complexity of LDPC encoding is fundamentally proportional to the square of the block length. To address this issue, researchers introduced a special type of LDPC code known as Quasi-cyclic (QC)-LDPC codes, which feature a parity check matrix composed of circulant permutation matrices. The encoding complexity of QC-LDPC codes can be effectively reduced, leveraging the cyclic shift properties of the generator and parity check matrices. There has been extensive research on low-complexity QC-LDPC encoders [2–8].
QC-LDPC codes are a particular type of LDPC code whose parity check matrix is built used simple shift registers and combinatorial logic. These codes boast very low error floors and fast iterative convergence, with the advantage of cyclic symmetry aiding in the implementation of encoders. Architectures for QC-LDPC encoders primarily include dual-

diagonal, Lower-Upper (LU) and Richardson-Urbanke (RU) decomposition approaches.
An approach based on LU decomposition was proposed in article [9] for direct encoding, avoiding the conversion from the H matrix to the G. Specifically, the H matrix is decomposed into an information matrix Hs and a parity check matrix Hp. However, this method requires a LU decomposition of the Hp matrix and a left multiplication permutation matrix A. The challenge of determining the permutation matrix A makes the encoding algorithm relatively complex. Additionally, when the H matrix is singular, LU decomposition cannot be performed, leading to an incomplete LU algorithm, which is not suitable for hardware implementation [10].
To address this issue, a simplified algorithm, the RU algorithm, was introduced in [11]. This method converts the sparse parity check matrix H into an approximately lower triangular form and enables fast encoding by rapidly computing the parity bits. Many researchers have implemented the RU encoding algorithm and its various improved versions on FPGA [12]. The downside of the RU algorithm is that its column permutation may result in non-systematic codewords, reducing decoding efficiency at the receiver. A hybrid approach is adopted in [13], where part of the parity bits is computed used the generator matrix, while the rest uses the RU method.
This paper proposes a partially parallel encoder architecture for QC-LDPC codes based on forward substitution (FS). The designed encoder is configurable at runtime, suitable for most QC-LDPC codes. This structure optimization significantly boosts the encoder’s throughput, reduces latency, and increases hardware utilization compared to other QC-LDPC encoder architectures.
The remainder of this paper is organized as follow: Section II briefly analyzes the characteristics of QC-LDPC codes. Section III introduces a FS-based QC-LDPC encoding algorithm. Section IV presents an efficient partially parallel architecture. Finally, in Section V, we summarize the work and draw conclusions.

Each entry in the matrix E is referred to as a shift value. It should be noted that the parity check matrix H in equation (1) can be constructed by expanding the mM × nM exponent matrix H. Rather than seeking a generator matrix from H, a LDPC code may be directly encoded utilizing the parity check matrix, which is transformed into a lower triangular form followed by back substitution. This method, referred to as RU encoding, was introduced by Richardson and Urbanke, presenting a linear time encoding technique tailored for parity check matrices with sparsity. Its core concept hinges on matrix transformations executed through row and column permutations solely, thereby reformulating the parity check matrix H into a sparse matrix variant. This approach outcompetes the G matrix multiplication method in complexity reduction significantly. The RU algorithm comprises dual phases: a preparatory phase and the encoding phase itself.
Primarily, during the preparatory phase, the parity check matrix H is morphed into an approximate lower triangular (ALT) form, as depicted in Fig. 1. The parity check matrix H is represented by a m × n matrix, where the code’s block length is denoted by n and m signifies the parity check equations’ quantity. The current drawback of the FS encoding method lies in the need for a large amount of storage space to save the codeword information and the requirement to process it in a certain order, which limits parallelism. However, FS encoding is primarily applied to structured code. In the architecture described in this paper, the memory demand is mainly concentrated on the positions of the CPMs and the corresponding cyclic shift values within the basic graph matrix, without the need to store the entire PCM, which alleviates the storage pressure.
IV. THE PARTIALLY PARALLEL ENCODER PROPOSED The encoding algorithm described in Section III can be implemented with varying degrees of parallelism. A fully parallel implementation would necessitate an extremely high level of hardware resource utilization. However, considering flexibility as a primary requirement, as depicted in Fig. 2, a partially parallel encoder architecture is presented for efficient encoding. This section details the structure and operation of the partially parallel encoder.
Fig. 2. Proposed QC LDPC encoder architecture.
In Fig. 2, the information bits are stored in a dual-port input buffer memory. For enhanced efficiency, the input buffer can accommodate the information bits of two adjacent codewords. Half of its memory is used for reading during the current codeword’s encoding, while the other half stores the information bits for the next codeword, effectively reducing the demand for encoding clock cycles and increasing the encoding speed.
The novelty lies in that the entire hardware architecture is centered around the Cyclic Shift (CS) module, tasked with cyclically shifting the cyclic shift matrix, where the size of the submatrices equals the size of the expansion factor in the submatrix. The input buffer employs multiple serial-in parallelout shift registers to store the data received by the encoder. The output buffer stores the encoded codewords. When the submatrix size is M , the encoder processes one CPM serially from a row of the submatrix per clock cycle. Then, the vector (all GF(2) additions realized through XOR operations) is calculated using the formula described in the previous section; when the submatrix size is M , the encoder processes up to two CPMs sequentially from every two adjacent rows of the submatrix per clock cycle. V. CONCLUSIONS
A novel architecture for QC-LDPC encoding is proposed in this paper. The designed encoder is suitable for QC-LDPC codes where the parity check matrix can be converted into diagonal form. The architecture is based on a serial encoding schedule, processing one CPM of the PCM per clock cycle. However, for codes with shorter codewords, it innovatively reuses existing hardware resources to handle multiple CPMs

per clock cycle. This is achieved through the innovative design of the cyclic shift network, which can be configured to operate as a single shifter for long input data vectors or as multiple independent CSs for shorter inputs. The proposed CS architecture is also applicable to other hardware designs for both encoding and decoding LDPC codes when flexibility is a requirement.
