Abstract—In modern communication systems Low-Density Parity Check (LDPC) codes are widely used. It is because of their performance under iterative decoding, hardware-friendly structure and well-known construction methods. For one to construct good LDPC code Density Evolution (DE) can be used, it estimates the decoding threshold, that is it estimates performance of the code. LDPC codes can be further extended to Generalized LDPC (GLDPC) codes, where simple SPC code is replaced with some component code. With more complex structure common DE is no longer correct. This letter presents the upper bound for decoding error and, therefore, an upper bound of a iterative threshold (in dB). It will be shown that the Gaussian approximation gives an upper bound for a decoding error. As one of the results it means that EXIT-charts analysis also gives an upper bound for an iterative threshold for a GLDPC code.
Index Terms—Density Evolution, LDPC codes, GLDPC codes, iterative threshold
I. INTRODUCTION
L OW-Density Parity-Check (LDPC) codes are widely used in communication systems, including fiber optic communication lines (FOCL), Wi-Fi, and 5G. LDPC codes are implemented in several modern communication standards such as IEEE 802.3ca, IEEE 802.3an, DVB-S2, WiMAX. It is high performance, low hardware complexity and very well-known methods of code construction make LDPC codes so popular. The most popular method to construct high performance LDPC codes is Density Evolution (DE) [1], [2].
LDPC codes can be further extended to Generalized LDPC (GLDPC) codes [3]–[5]. Instead of Single Parity Check (SPC) code for each row of a parity-check matrix some component code is used (Hamming, BCH, etc). GLDPC codes usually have good minimum distance properties [3], [4], compared with LDPC codes, and they generally have faster convergence speed, and they also have better error floor behavior. At the same time, because of component code, there is no trivial way to consider DE for GLDPC codes. Soft Input-Soft Output (SISO) decoder for a component code can be represented by very complex mathematical function and hence there is no clear way how to calculate the probability density function (pdf) for extrinsics information.
In the existing studies there are several ways how to estimate the performance of GLDPC code. For hard input the performance can be estimated for miss-correction-free iterative

Bounded Distance Decoding (iBDD), that is hard decoding of component code either gives correct codeword or returns the input codeword [6], [7]. Despite decoding algorithm being non-realistic for Binary Symmetric Channel (BSC), proposed in these two articles DEs predicts the behaviour of GLDPC codes with hard input, that is if one code is better for decoder without miss-corrections, then it will be better for iBDD with miss-corrections. For soft input there exists an article on Monte Carlo DE, where in order to estimate the distribution of extrinsics information simulation is used [8]. In this article Chase-Pyndiah decoding algorithm was modified so that the distributions in question will be independent. Again, DE itself doesn’t approximate the performance of Chase-Pyndiah decoding algorithm, however it estimates the behaviour of performance. The idea of Monte Carlo DE was expanded and used in [9]. Here, the distribution of extrinsic information is approximated by the mixture of Gaussian distributions. With this method the authors were able to construct a code, which is closer to a Shannon limit. Also, one of the possible ways to construct GLDPC codes is to consider EXIT-charts [10], [11]. Similar to all of the previous methods EXIT-charts analysis allows to estimate the behaviour of code ensemble.
In this letter we consider only Max-log-MAP decoder as SISO for component code. The presented results stays true for other SISO decoders which satisfy the Lipschitz condition. The main contribution of the following article is the analytical upper bound of decoding error for irregular GLPDC code. Additionally, the upper bound obtained with Gaussian approximation. It means that Gaussian approximation and EXIT-charts show the upper bound for an iterative threshold and estimate the behaviour of code ensemble.
The rest of the paper is organized as follows: Section II introduce the main notations and gives some background. Section III introduces two main Theorems which gives the Gaussian upper bound for a decoding error for GLDPC code. Section IV concludes the paper.

IV. CONCLUSION
In this letter we propose an analytical upper bound of decoding error of GLDPC decoder with Max-log-MAP for a component code. The main result is that the decoding error can be upper bounded by a Gaussian distribution function. It means that the tails of the output of the GLDPC decoder is lighter than the Gaussian tails. This results allow to use the Gaussian approximation and the EXIT-charts to estimate the behaviour of ensemble of codes. Also, it is known that Gaussian approximation gives the upper bound for iterative threshold (in dB), that is the real iterative threshold is not worse that the iterative threshold from EXIT-charts/Gaussian approximation.
