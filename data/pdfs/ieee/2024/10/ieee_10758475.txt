Burst Detection and Correction for Gilbert Codes and its QC-LDPC Extensions

Alina M. Veresova HSE University
St. Petersburg, Russia amveresova@gmail.com

Andrei A. Ovchinnikov HSE University
St. Petersburg, Russia a.ovchinnikov@hse.ru

Abstract—The article discusses the issue of determining the location of an error burst based on the structure of a blockpermutation low-density parity-check code. Quasi-cyclic lowdensity parity-check codes with a small number of blocks are considered, making it possible to correct the longest possible bursts within this construction. Through experimentation, the maximum, minimum, and average number of attempts was estimated when determining the start position of the error burst. The features of determining the location of a packet in blockpermutation low-density parity-check codes are revealed. The proposed method is based on using majority decoding techniques and allows reducing the average number of attempts to locate a burst from hundreds to units for the codes under consideration. Using the packet detection method, which is based on the analysis of the structure of the parity-check matrix, algorithms for decoding Gilbert and extended Gilbert codes have been proposed, ensuring the correction of bursts within their error-correction capabilities. The process of decoding extended Gilbert codes utilizes list decoding in a supercode, with a criterion for selecting the error vector based on the syndrome.
Index Terms—Error bursts, burst location, burst decoding, low-density parity-check codes, channels with memory.
I. INTRODUCTION
Error correction methods that do not rely on retransmission for error correction on the channel represent the most significant tool for ensuring reliable data transmission. The specific characteristics of the physical communication medium and information processing procedures at the receiving end lead to the fact that the probability of distortion of a single transmitted symbol is not only dependent on the transmitted signal and channel noise, but also on the state of the communication system, (including the transmitter, channel, and receiver), which can change gradually. In such a scenario, we are discussing the existence of channel memory.
In classical coding theory, channels with independent errors are typically considered, and interleaving is used to mitigate the memory effect. However, from an information theory perspective, accounting for the memory of a channel can increase achievable transmission rates [1]. Therefore, it is essential to explore coding techniques that take into account the presence of memory in a channel by using appropriate channel models. In these models, errors are typically described using bursts, and in many cases, the location of the bursts is known. The task then is to correct erasure bursts [2], [3]. A more challenging decoding task arises when the position of a burst is unknown

and the burst contains errors rather than erasures. Information about the location of an error packet is essential for decoding algorithms that use a sliding window [4].
An important modern class of error-correcting codes are low-density parity-check (LDPC) codes [5], [6]. It is known that these codes can also be used to correct error bursts [7]– [9]. The article [10] considers the problem of determining the location of a burst in a channel with two states, each described by a different Gaussian noise level. In this case, a soft channel output is available to the decoder, which can be used to determine the position of the burst.
This article discusses a discrete channel in which additional information about the reliability of accepted symbols is not available.
The article is organized as follows. The section II describes the error model used in the study. The section III describes the code constructions used. The section IV contains a method for determining the position of a burst, and in the section V decoders for Gilbert codes and their generalization are proposed. The section VI concludes the article.
II. BURST CHANNEL MODEL
The classical model for describing a discrete channel with memory is the Gilbert-Elliott model. This model is described by a Markov chain with two states: “bad” (B) and “good” (G). In the “bad” state, there is a high probability of error. In the “good” state, errors occur rarely. To specify the model, it is necessary to set error probabilities ηB and ηG, as well as transition probabilities b and g between states. Special cases of this model are also often used: the Gilbert model, where ηG = 0, and ηB is typically chosen to be 0.5, and the Markov model, where ηG = 0, and ηB = 1.
When using channel models with two states, error bursts are considered to be segments of the error vector that correspond to being in a bad state, that are sections with a higher frequency of non-zero elements. In coding theory, a simpler definition of a packet with a length b is often used. This is an error vector where the first non-zero element is b positions away from the last one. It should be noted that for errorcorrecting codes, a parameter bmax can be introduced. This parameter is called the correcting capability for correcting single error bursts, and it means that any error packet of length b ≤ bmax can be decoded correctly (it corresponds

III. ERROR BURST CORRECTION USING LDPC CODES.
In modern communication standards, low-density parity check codes are widely used to ensure the reliability of data transmission. These constructions have a good ability to correct independent errors using fairly simple decoding procedures. Due to their sparse structure, they are insensitive to error grouping, and can be used to correct small packets using probabilistic decoding without special approaches. However, when large packets are formed, the decoding error probability significantly increases.
Often, to build an LDPC code, a block-permutation construction of a parity-check matrix of the form (1) is used, consisting of (m × m) blocks Cij, which are the degrees of the cyclic permutation matrices.

Such codes are quasi-cyclic and are referred to in the literature as QC-LDPC codes. We will denote the code determined by (1) as (γ × ρ) code and as (γ × ρ × m) code when the value of m is taken into account. Telecommunication standards use constructions with a large number of blocks to correct independent errors, while blocks are interspersed with zero blocks in order to increase the minimum code distance and improve the properties of iterative decoding. When correcting error bursts, an important characteristic of the code is the maximum length bmax of the burst that this code can correct. It is known that for a construction of the form (1) the length bmax of the corrected burst does not exceed the block size m, and in many cases bmax = m − 1.
Thus, to improve the burst correction capability bmax given fixed codelength n = mρ and code rate R = 1 − γ/ρ, one should increase the block size m and therefore decrease the number of blocks γ and ρ. This leads to observation that unlike LDPC designs for channels without memory, to increase the length of the corrected packet, it is necessary to use a construction with a small number of large blocks.
E. Gilbert in [12] proposed a class of codes for correcting error bursts. These codes can be described as (2 × ρ) QCLDPC codes with γ = 2 and a special selection of blocks:

IV. DETERMINING THE BURST POSITION FOR A RANDOM BLOCK-PERMUTATION CODE
To decode single error bursts, a sliding window decoder with a window length of b can be used, sequentially considering possible burst locations and making decoding attempts. The underlying algorithm of single attempt depends on particular decoding algorithm and is not considered in this paper. It is assumed that if error burst is covered by current window, decoding will succeed, and decoding failure will be detected otherwise. One of such decoders is proposed in [4]. If we assume that the position of the beginning of a burst of length b is random and uniformly distributed, then on average, the decoder will use (n − b + 1)/2 decoding attempts. However, the complexity of this operation can be significantly reduced by estimating the location of the error burst in advance. To do this, we propose the following technique.
Consider the vector f = s ⊛ H, where s = yHT = eHT is the syndrome of the received vector y or its corresponding error vector e (throughout the paper all matrix operations are performed in binary arithmetic except ⊛ which is performed over the real number field). The vector f is used for majority decoding and indicates the number of failed parity checks in each column of the parity-check matrix. It can be expected that the number of columns with a large number of non-zero elements in common with the syndrome (that is, with large values at corresponding positions of f ) will be greater in the area corresponding to the correct location of the burst. We will characterize such areas of length b by elements of vector z = (z1, . . . , zn−b+1), which shows the sum of the elements

Further, when determining the location of a burst, positions where the value of z is greater are considered to be the most likely. The order in which possible burst locations are considered can be set by ranking them according to the values of z.
To improve the estimation of the burst location, we can use heuristics similar to those used in majority decoding. Generally, columns of the parity-check matrix corresponding to large values of f are considered less reliable, while columns with small values of f are more reliable. When calculating the vector z, we will only take into account unreliable positions, that is, those where the value of f exceeds a certain threshold. Since the majority decoding threshold is t = γ/2, let’s use this value as our cutoff threshold and only apply the procedure to elements of f that exceed t = γ/2.
When evaluating the quality of the proposed method, we will consider the number of attempts T to determine the position of the burst before finding the correct position. Since T is a random value, we will consider the minimum, maximum and average number of attempts of E[T ]. In addition, we will compare the obtained value with the average number of guesses or sequential considerations of possible burst positions from 1 to (n − b + 1), which, as mentioned above, is (n − b + 1)/2.

V. DETECTING AND DECODING ERROR BURSTS USING GILBERT CODES
A. Gilbert Code
The method of determining the location of an error burst, described in the previous section, can be applied to the Gilbert code (2). Experiments show that, when using a cutoff threshold of t = 1, the location of the burst can be determined with probability 1. Let us analyze this statistical result by considering the construction of a parity-check matrix of the code.
Consider the submatrix Hb of the matrix (2), consisting of columns HGilb corresponding to the location of the error burst.

If generalized Gilbert code is used, to select the correct codeword we can exploit the third stripe of the parity-check matrix. The decoding procedure can then be carried out in two stages:
1) Decoding in supercode. At this stage, only the two upper strips of the parity-check matrix are used for decoding. Decoding occurs similarly to the procedure for the Gilbert code, but with a packet length corresponding to the correcting capability of the extended code, that is, bmax = m − 1. If the conditions (5) are met for several fb, let’s create a list of error vectors above the code for each possible burst location, then perform step 2.
2) Resolution of uncertainty. For each found error vector in the supercode, the syndrome s in the subcode is calculated (using the full parity-check matrix) and the decision is made in favor of the error vector that gives the zero syndrome.
Note that when decoding an extended Gilbert code, only one error burst of length b ≤ bmax corresponds to a specific syndrome. Inverting the characters of the received word according to this error burst gives both the codeword of the original code and the supercode’s codeword. The conditions (5) must be satisfied for the supercoded codeword, so the corresponding error vector is included in the list of vectors at step 1. Therefore, this procedure guarantees decoding of extended Gilbert codes within their correcting capability.
VI. CONCLUSION
The article discusses the problem of determining the position of a single error burst based on the features of the block-permutation structure of LDPC codes. Experiments were conducted to estimate the number of attempts when finding a burst for 4 × 8 QC-LDPC code with a block size of m = 127, showing a decrease in the average number of attempts from 445 to 2.5.
Based on the proposed method and analysis of the structure of Gilbert code parity-check matrix, an algorithm for decoding Gilbert codes is proposed. The algorithm guarantees the correction of error packets within the correcting capability. For

extended Gilbert codes, a decoding algorithm is proposed that
includes a Gilbert code decoder as a decoder in the supercode.
This algorithm allows you to correct all bursts up to length m − 1 for extended Gilbert codes.
