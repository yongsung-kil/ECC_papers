An Efficient Hardware LDPC Encoder Based on Partial Parallel Structure for CCSDS
1st Zhaohui Wang, 2nd Xin Hao, 3rd Changxing Lin∗, 4th Qiuyu Wu Microsystem and Terahertz Research Center, China Academy of Engineering Physics
Institute of Electronic Engineering, China Academy of Engineering Physics Email: {wangzhaohui, haoxin, linchangxing, wuqiuyu}@mtrc.ac.cn

Abstract — This paper proposes a partial parallel structure for the Low Density Parity Check (LDPC) encoder, the design is capable for (5120, 4096) LDPC code for Consultative Committee for Space Data Systems (CCSDS) standard. A low complexity clclic shifter are designed for reducing the hardware cost due to the Quasi-Cyclic nature of the generator matrix. The generator matrix was divided into systematic part and parity check part, the code in the systematic part is the same as the input code, the parity check part divied into 32×16 submatrices, each submatrix is quasi-cyclic with the size 128×64. After 128 clocks, the encoded process finished, and 8Gbps throughput is achieved at 200MHz clock frequency with acceptable resource consumption.
Keywords — Partial Parallel, LDPC Encoder, Quasi-Cyclic, CCSDS
I. INTRODUCTION
Low density parity check (LDPC) code is a type of linear block code. There are very few non-zero elements in the parity check matrix H, thus it is also called sparse matrix. This code is first proposed by R.G. Gallager in 1960 [1], and in 1996 MacKay showed the long LDPC codes can achieve near Shannon-limit performance [2]. Although the LDPC codes have good error performance, but the implementation of LDPC code is not an easy work. To reduce the complexity of the implementation, a kind of LDPC code was invented, the parity check matrix consists of small cyclic submatrices, the code generated by this matrix is called quasi-cyclic LDPC (QCLDPC) code [3]. Thus various orgnizations adopted QC-LDPC code as the standard LDPC code in their standard, one of the most commonly used LDPC codes is provided by Consultative Committee for Space Data Systems (CCSDS).
Lots of researches had been made to propose low complexity and hardware programmable encoders in literature [4]- [13]. In [4] a novel generator matrix for LDPC encoder based on China digital radio architecture was proposed, In [5] a parallel implementation based on a series of recursive convolutional encoders for CCSDS standard was proposed, In [6] the author proposed a encoder based on block-row-cycle structure for CMMB standard, In [7] fully serial encoder and mixed encoder was proposed to the IEEE 802.22 Wireless Regional Area Network (WRAN) standard, In [8] and LDPC encoder implemented on GPU was proposed for DVB-S2 standard, In [9] and [13] a special structure of multi-giagonal parity matrice was proposed to construct the LDPC encoder for IEEE 802.11n, IEEE 806.12e standard and Chinese digital

terrestrial/television multimedia broadcasting standard, In [10] the author proposed a high-speed encoding and decoding method for CCSDS-recommended (8176, 7154) LDPC code, In [11] an efficient multi-rate encoder for WIMAX was proposed, In [12] a LDPC encoder based on linked list queue is proposed.
In this paper, we present an efficient hardware LDPC encoder based on partial parallel structure for CCSDS standard, this paper is arranged as follows: Section II presents an overview the details of QC-LDPC codes in CCSDS standard, Section III gives the proposed architecture and the FPGA implementation of the partial parallel LDPC encoder, Section IV presents simulation and synthesis results, and Section V includes the conclusion of paper.
II. QC-LDPC CODE IN CCSDS STANDARD
CCSDS standard defines nine punctured LDPC codes with three information block lengths (1024, 4096 and 16384), and three coding rates (1/2, 2/3 and 4/5) [14]. The parity check matrix and generator matrix of QC-LDPC code in CCSDS standard coonsists of square submatrices of the same size over the Galois field 2, every submatrix is an all-zero square matrix or circulant permutation matrix. The first columns of k0 × (n0 − k0) blocks submatrix can be stored in several cyclic shift registers, and the encode process can perform parallelly, then shift serially, this kind of structure is called partial parallel structure. To increase the throughput of the encoder, the degree of parallelism can be enhanced, the larger of the degree, the faster of the encoder throughput.
B. FPGA Implementation
This paper choose CCSDS (5120,4096) LDPC code to implement the LDPC encoder, each of the submatrix has the size 128 × 128, the base generator matrix has the size 32 × 40, the corresponding parity check parts of the generator matrix has the size 32 × 8. The degree of parallelism is set as 16 to enhance the throughput, i.e. divide every submatrix to two submatrices which has the size 128 × 64 each. The 1st and 65th column of the generator matrix are stored in the cyclic shift registers, each block of codeword can be obtained by circulating cyclic shift register 64 times, finally combine all the blocks of codeword together as the parity check codeword. The architecture of the encoder is illustrated by Fig. 2.
The detail of the algorithm is:
1) Fill all the cyclic shift registers the 1st and 65th column of the generator matrix.
2) Divide the information bits into 32 groups of 128bit binary vector.
3) Do ”and” operation between each group of information bits and corresponding cyclic shift register, obtain 32 × 16 group of data, each data has size 128bit.

4) Do ”xor” operation of every group of data accumulately, then get the 1bit data as the final codeword data and store in 16 different cyclic shift registers.
5) Shift the registers in step 1) to right, then get the next column of the generator matrix, and redo step 2) 64 times, then go to step 5).
6) Combine all the data in the 16 cyclic shift registers orderly, output the parity check final codeword.
