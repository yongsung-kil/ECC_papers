Structured Concatenation of Protograph LDPC Codes and Markers for Insertion/Deletion Channels
Ryo Shibata, Gou Hosoya, and Hiroyuki Yashima Dept. of Information and Computer Technology, Faculty of Engineering,
Tokyo University of Science, Tokyo 125-8585, Japan

Abstract—In this paper, the structured concatenated coding of protograph LDPC codes and markers for insertion and deletion channels are proposed, where the joint detector/decoder can deal with the differences in estimation accuracy of these errors. Conventional concatenated coding of LDPC codes and markers does not address the differences. The proposed approach with optimized protographs can exploit those differences and thus improve the performance of the joint detection/decoding. Iterative decoding thresholds and simulation results are provided to support the advantages of the proposed concatenated coding scheme.

I. INTRODUCTION
Low-density parity-check (LDPC) codes were first proposed by Gallager in [1]. LDPC codes are represented by a bipartite graph and decoded by the iterative decoding (message-passing) algorithm on the graph. In recent years, a new class of LDPC codes was introduced by Thrope [2], called protograph LDPC codes. Protograph LDPC codes not only have the excellent error performance but also possess simple graph structure, which allows practical and efficient decoder implementation in hardware [3], [4].
For future high-density magnetic recording systems, bitpatterned media (BPM) has attracted considerable interest. In BPM, mis-synchronization of the write clock caused by timing jitter occurs and is, therefore, one of the major problems in BPM recording systems [5]. This mis-synchronization results in insertion, deletion, and substitution (IDS) errors [6].
For IDS error correction, the concatenation of unstructured LDPC codes and markers has been investigated in [7], [8]. The marker is a predetermined bit string inserted into an LDPC codeword at regular intervals, and they are utilized to recover synchronization impaired by insertions and deletions. A receiver consists of a MAP detector and an LDPC decoder, and they exchange extrinsic log-likelihood ratio (LLR) messages iteratively. The MAP detector is utilized to estimate insertion and deletion errors. With the advantage of the markers, the estimation accuracy becomes higher as the bit positions are closer to markers, which results in the differences in the LLR message distributions among each distance from markers. However, it is difficult for the conventional concatenated coding to deal with the difference because unstructured LDPC codes are specified only by the degree-distribution pair.
In this paper, we utilize protograph LDPC codes to tackle the differences. A protograph is specified by the variable node types, the check node types, and the types of edges connecting them. We first propose a structured concatenated coding of

protograph LDPC codes and markers so that the distances from markers are associated with variable node types. Namely, the edge-connection property is imposed on each distance. We then optimize protographs under the structured concatenated coding. The optimized protographs allow the LDPC decoder to exploit the differences caused by the MAP detection and provide beneficial extrinsic information feedbacks to the MAP detector. Iterative decoding thresholds and simulation results exhibit that the structured concatenation with optimized protographs outperforms an optimized irregular LDPC code concatenated with markers [8] and a protograph LDPC code optimized for AWGN channels [3]. Moreover, the proposed approach can be more practical and useful for the IDS channels not only because of the excellent error performance but also because of the linear minimum distance growth property of the proposed protograph codes.
II. RELATED WORK
A. IDS Channel
For integers i and j such that i ≤ j, a set of consecutive integers {i, ..., j} is denoted by [i, j] in this paper. The probabilistic IDS channel model is expressed as a hidden Markov model. Let x = (x1, x2, ..., xn) denote the transmitted sequence of length n, where xt ∈ {0, 1}. Let r = (r1, r2, ..., rn′ ) denote the received sequence of length n′, where rt ∈ {0, 1}. A channel state at time t is denoted by st, where st ∈ [−D, +D], D ∈ N, and the initial channel state s0 = 0. The state st is the event that both the first t-bit of x is transmitted and the first (t + st)-bit of r is received. The difference between the transmitted and received lengths n and n′ gives the final channel state sn = n′ − n.

B. Concatenation of LDPC Codes and Markers
Figure 1 shows the system model of a concatenated coding
of LDPC codes and markers for the IDS channels. Let CE be a binary LDPC code of length NL and rate RL. Let c denote a codeword in CE excluding puncturing bits, and the length of c is denoted by N (≤ NL). Let CS be a set of markers of length nS, where CS ⊂ {0, 1}nS . The number of inserted markers is denoted by ν, where ν = N/d (d is a divisor of N ), and the
marker coding rate RM = d/(d + nS). The procedure of the concatenated coding is as follows. The
information bits are first encoded by CE. Then, the permuted sequence c˜ is obtained by applying a permutation π : [1, N ] → [1, N ] on the codeword c, i.e., c˜ = (cπ−1(1), ..., cπ−1(N)). Conventionally, a random permutation or no permutation is used. Next, c˜ is divided into ν sequences c˜1, c˜2, ..., c˜ν of each length d, where c˜i = (c˜(i−1)d+1, ..., c˜(i−1)d+d). The kth marker ckS is selected from CS and inserted between c˜k−1 and c˜k. Consequently, the transmitted sequence x is formed by x = (c˜1, c1S, c˜2, c2S, ..., c˜ν , cνS). The length of x is n = N + νnS, and the concatenated coding rate is given by R = RLRM. The parameters of markers are set to nS = 2 and CS = {01} throughout this paper.

III. STRUCTURED CONCATENATION OF PROTOGRAPH LDPC CODES AND MARKERS
The accuracy of the MAP detection differs depending on the distance between a codeword bit and its nearest marker (we call this distance MK-distance), resulting in the difference in the LLR message distributions among each MK-distance. The j-th bit of c˜i has the MK-distance of k = j if j ≤ ⌈d/2⌉, else k = d + 1 − j. Figure 2 shows the LLR message distributions for each MK-distance k when Pins = Pdel = 0.05, Psubs = 0, and d = 12. It is observed that the estimation of insertion and deletion errors gradually becomes more difficult as the MK-distance increases. Nevertheless, the conventional joint detection/decoding proceeds without considering the MKdistances because unstructured LDPC codes are defined only by the degree distribution pair. We utilize the protograph LDPC codes to manipulate the differences in the LLR distributions. The proposed concatenation scheme, called structured concatenation, associates the MK-distance with the VN type of a protograph by modifying the permutation π, enabling us to consider the difference in the distributions. Furthermore, optimized protographs for the structured concatenation would possess properties allowing the LDPC decoder to control successfully the differences caused by the MAP detection and to provide beneficial extrinsic information to the MAP detector.
A. Structured Concatenation

Generally, assuming that there are np VNs V1, V2, ..., Vnp and mp CNs C1, C2, ..., Cmp in the protograph. The number of transmitted VNs is denoted by τ , and Vτ+1, ..., Vnp are punctured nodes. The code rate is given by RL = (np−mp)/τ . The protograph in Fig. 3 is a rate-1/2, and there are np = 3 VNs, mp = 2 CNs, and np − τ = 1 punctured VN. A protograph LDPC code is constructed by applying a “copy-and-permute” operation on a protograph, a process is known as lifting. This operation consists of making z copies of the protograph, and then a large graph is obtained by permuting variable-to-check pairs (edges) corresponding to each of the edge types of the protograph. Through the process, a protograph LDPC code with length NL = znp (the transmitted codeword length is N = zτ ) is obtained. The entire transmitted codeword can be divided into τ blocks according to different VNs Vi = (c(i−1)z+1, ..., czi), i ∈ [1, τ ], in the protograph.
Now, we consider the permutation πSC to associate the MKdistance with VN types. It is assumed that d is a multiple of 2. With πSC, we can sequentially allocate higher reliable bit positions (i.e., lower MK-distances) to VNs from V1 toward Vτ . Figure 4 shows the bit mapping between c and c˜, representing the structured concatenation (SC). The SC is actualized in the following way. Firstly, we gather j-th bit (j = 1, ..., d) from the sequences c˜1, ..., c˜ν and make a sequence wj = (c˜j, ..., c˜(i−1)d+j, ..., c˜(ν−1)d+j). Then we create a merged sequence θi = (wi, wd+1−i) concerning the MK-distance i = 1, ..., d/2. Finally, a sequence Θ = (θ1, θ2, ..., θd/2), whose elements are ascending order of the MK distance, is obtained. The SC can be realized with the permutation πSC reflecting the mapping between a transmitted codeword c of a protograph LDPC code and reordered sequence Θ. Figure 4 also shows an example of the bit mapping of the SC when d = 6, τ = 4, and N = 12. The type of VN corresponding to one MK distance are equivalent if and only if τ = d/2. In the case τ > d/2, some adjacent VN types correspond to one MK-distance.

B. Protograph Design for the SC
By the SC coding, the correspondence between the MKdistance and VN types of a protograph is characterized. It is necessary to optimize the structure of a protograph under the SC coding to utilize the differences caused by the MKdistance.
1) Properties of a Good Protograph: We concentrate on graphs that possess good protograph properties [3] consisting of both the excellent decoding threshold and the linear minimum distance growth. The combination of one or more degree-1 VN, one punctured VN, and several degree-2 VNs are likely to produce a good protograph [3]. However, too many degree-2 VNs do not preserve the linear minimum distance growth property. To preserve this property, the subgraph where degree-1 VNs and the corresponding CNs are removed from the protograph needs to meet the following criteria: there are neither parallel edges nor cycles among degree-2 VNs in the subgraph [3], [4].
2) Design Method: Due to the search space grows as a function of the size of the proto-matrix, it is not feasible to find an optimal matrix of large size with the exhaustive search. Hence, we employ differential evolution (DE) [11], known as one of the metaheuristic optimization techniques, to find approximate solutions. Differential evolution is a method that optimizes parameters by iteratively trying to improve a candidate solution concerning a given cost function. We use the decoding threshold given by the density evolution [9], [2], [10] as a cost function. Our optimization method is analogous to [12], while there is a slight difference in mutation and crossover parameters because we utilize the self-adaptive DE [13]. Executing the self-adaptive DE with S = 10mpnp and G = 500, we obtain two proto matrices called SC-ID and SC-IDS protographs, which are optimized when Psubs = 0 and Psubs = 0.01, respectively, as shown in Fig. 6. The position of each VN in those proto-matrices plays an important role in the SC, so that replacing columns changes the code performance over the IDS channels. It is interesting that the degree-1 VN is punctured in our optimized protographs, while it is known that the puncturing of highest degree VN leads to good decoding thresholds in AWGN channels [3]. This would be because the puncturing of the high-degree VN is not beneficial to the extrinsic information feedback from the LDPC decoder.
IV. NUMERICAL RESULTS
In this section, we compare the decoding thresholds and simulation results between the SC coding and the conventional unstructured concatenation (USC) coding where the codeword is not permuted. The code rates are all RL = 0.5 and the marker interval d = 10 is used. The IDS channel with Pins = Pdel is assumed. The following existing codes are used in the comparison: an irregular LDPC code optimized for the USC coding [8] (whose degree-distribution pair is ρ(x) = x5 and λ(x) = 0.1524x + 0.769x2 + 0.0776x96) and a protograph LDPC code with the linear minimum distance property optimized for AWGN channels, called AR4JA code [3]. We also use the USC-ID protograph code based on the protograph (mp = 4, np = 7, τ = 6) optimized for the USC coding when Psubs = 0 by using the self-adaptive DE.

B. Simulation
Figures 7 and 8 show the bit error rate (BER) and frame error rate (FER) curves as Pins+Pdel varies in the IDS channels of Psubs = 0 and Psubs = 0.01. The lengths of the codes are 5000 ≤ NL ≤ 5005.
In Figs. 7 and 8, the SC with the SC-ID and SC-IDS protograph codes, respectively, yield superb BER performances to the other codes with the USC. It is observed that there are large performance differences between the SCs and the USCs at BERs at 10−5 compared with the differences in their thresholds. Further, contrary to the thresholds, the irregular code results in worse performance than the SC-IDS protograph code in Psubs = 0. The same applies to Fig. 8. According to [9], irregular LDPC codes with large dv are not necessarily useful in simulations with short length codes even though they have excellent thresholds. Besides, for the USC coding, the performances of actual codes from their ensembles do not directly follow the decoding thresholds because the density evolution with the USC coding averages the MK-distances, i.e., the effect of the differences in the LLR distributions is ignored. The SC coding with optimized protographs can behave well not only in the asymptotic regime (i.e., under the assumptions of infinite length codes) but also in nonasymptotic regime (i.e., with the short length codes desired in practical applications).
V. CONCLUSION
A structured concatenated coding of a protograph LDPC code and markers, considering the MK-distance, has been proposed. Furthermore, we propose protograph codes optimized for the structured concatenation, namely, SC-ID and SC-IDS protographs. The decoding thresholds and simulation results have verified that the structured concatenated coding

with the optimized protographs outperforms the conventional concatenated coding with the optimized irregular code as well as the protograph code optimized for AWGN channels.
In our future work, we consider the designing of highrate protograph codes for the proposed method. Moreover, we reveal the relationship among protograph structures and decoding performances.
