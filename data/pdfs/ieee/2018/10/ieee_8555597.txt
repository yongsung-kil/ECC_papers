A Hybrid Decoding Algorithm for Low-Rate LDPC codes in 5G
Kaifei Sun, Ming Jiang National Mobile Communications Research Laboratory, Southeast University, Nanjing, China
Email: {kfsun, jiang ming}@seu.edu.cn

Abstract—A hybrid decoding algorithm for low-density paritycheck codes is presented. The proposed algorithm applies different updating schemes, such as the normalized min-sum (NMS) simplification and linear approximation, to the check-node based on its degree. Meanwhile, to eliminate the dependence on the channel variance estimation, the proposed algorithm adopts a multiplicative factor to initialize the channel input and a fixed linear functions for check-node updating. From the iterative thresholds and decoding simulations, our proposed algorithm can be shown to achieve improved performance (much closer to that of the belief propagation decoding) at the expense of a slight increase in complexity to NMS algorithm.
Index Terms—LDPC codes, hybrid decoding, low rate LDPC codes, iterative decoding

I. INTRODUCTION
Low-density parity-check (LDPC) codes first proposed by Robert. G. Gallager in the early 1963 [1] and later rediscovered by D. J. C. Mackay in 1995 [2] are good channel codes that closely approach the channel capacity. Due to the sparsity of the parity-check matrices, the LDPC codes can achieve the desired performance by iterative decoding with linear complexity. In the past decades, the quasi-cyclic (QC) LDPC codes with block-parallel architectures have been progressively used in various wireless communications systems, such as WiFi and WiMAX. As a consequence, the LDPC codes are regarded as one of the most promising coding techniques for 5G new radio (NR) and finally been accepted by 3GPP as the channel coding scheme in the eMBB scenario [3] [4].
The LDPC coding scheme in 5G NR [4] can be described by two base-graphs (BG1 and BG2) with similar structures, where the BG1 with mb = 46 rows and nb = 68 columns targeted for larger lengths and higher rates is shown in Fig. 1. Each non-zero (colored box) or zero element (blank box) in the BG1 is lifted by a permutation matrix with the size of Z ∗ Z and the information length can be set to kb ∗ Z = (nb − mb) ∗ Z, 22 ≤ Z ≤ 384. This BG1 consists of mainly four parts, the core matrix Hcore, the extension matrix Hext, an identity matrix I and an all-zero matrix O. The matrices Hcore and Hext respectively represent the highest rate code and the low-rate extension. The dual-diagonal with ”a-0-a” parity structure in the Hcore and the identity matrix for low-rate coding both satisfy the rate-compatible restriction, which can well support the incremental redundancy HARQ. In the BG1, the left most two columns are corresponding to the 2 ∗ Z state bits, which are punctured before transmission. The weights of these columns usually are sufficiently large and greater than

Furthermore, unlike the conventional irregular LDPC codes with uniform row weights, there is significant difference in the row weights of the LDPC codes for 5G, where the row weights of the Hcore are more than ten and the lowest row weight is only three in the Hext and the I.
The LDPC codes are generally decoded by the messagepassing algorithms such as the belief propagation (BP) algorithm, which iteratively exchanges the messages through the edges between variable nodes (columns) and check nodes (rows). The BP algorithm achieves near optimal decoding performance, but suffers from high computational complexity. For the traditional regular or irregular LDPC codes, many efficient algorithms [5–9] have been developed and provide a better trade-off between performance and complexity by using the min-sum (MS) approximation. Among these decoding schemes, the normalized min-sum (NMS) algorithm [5] is the simplest one for hardware implementation, where there is only an additional multiplicative factor and no requirement of variance estimation. While, it is quite difficult to optimize the normalized factors for the codes in 5G NR, since the difference of row weights is particularly large. As a result, there is a gnificant performance gap between the BP and NMS algorithms for the low-rate LDPC codes in 5G.

II. THE HYBRID DECODING SCHEME

We assume that the lengths of the coded block and the information bits are denoted by N and K, respectively. Then, the LDPC code can be defined by a sparse M ∗ N paritycheck matrix H = [hm,n]M∗N . In the matrix, the n-th column (variable-node) and the m-th row (check-node) are denoted by vn and cm, respectively, where ρm represents the degree of the check node cm. A codeword w = {w1, ..., wN } of the LDPC code is transmitted using binary phase shift keying
(BPSK) modulation, which maps the coded bits into a bipolar sequence x = {x1, ..., xN }, according to xn = 1 − 2wn, n ∈ [1, N ]. After passing through an additive white Gaussian noise
(AWGN) channel, the received signals can be written as y = {y1, ..., yN } according to yn = xn + zn, n ∈ [1, N ], where zn is an independent Gaussian random variable with zero mean and variance σ2 = N0/2.
The set of variable nodes that participate in the check node cm is defined as B(m) = {n|hm,n = 1} and B(m)\n denotes the set B(m) with the variable node vn excluded. Similarly, A(n) is the set of check nodes in which bit vn involves. Lnm and Lmn stand for the LLRs (logarithm likelihood ratio) which are sent from the variable node vn to the check node cm and from the check node cm to the variable node vn, respectively. In the BP and NMS algorithms, we denote the current iteration and the maximum number of iterations by τ and Γ, respectively.

III. COMPLEXITY AND PERFORMANCE ANALYSIS
A simple threshold evaluation for the structured LDPC codes can be carried out by the protograph-based EXIT (PEXIT) analysis [10]. It provides interesting insights on the connections between the different variable and check nodes in protograph, for example the BG1 and BG2 of LDPC codes in 5G NR, rather than just the degree distributions. For a given LDPC code, the PEXIT analysis can calculate the lowest convergence Eb/N0, the threshold value, for different decoding algorithms. The lower threshold indicates the better performance, so we can analyze and compare different algorithms by using PEXIT.

Similar as the decoding simulation, the normalized factor of the NMS and H-MIN algorithms are all set to α = 0.7, and the initialized factor and degree threshold of the H-MIN decoding are set to β = 0.8 and D = 10, respectively. Let’s focus on the threshold results of PEXIT with maximum number of iterations Γ ∈ [50, 1000], which is large enough for the low-rate LDPC codes. The threshold gap between the BP and H-MIN algorithms is no more than 0.2 dB, while the threshold of NMS algorithm is almost 1 dB higher than that of BP decoding. From the numerical results, we can see that the conventional MS simplification causes considerable performance loss for the low-rate codes.
One important reason that the NMS algorithm suffers from such a server loss in the threshold compared to the BP decoding is that there exists a large portion of low-degree checknodes in the Hext of BG1 or BG2. Usually, the performance of NMS algorithm with a normalized factor can be very close to that of the BP decoding, when the row weights become very large especially in the Hcore. On the other hand, the MS simplification is not well matched to the exact check-node updating of the BP decoding with only a few input LLRs from the variable nodes. Considering the huge difference of row weights between Hcore and Hext, we apply the linear approximation for the low-degree check-node updating, which can help compensate for the performance degradation induced by the MS simplification. Meanwhile, the linear approximation is only applied in the low-degree check-node, ρm <= D, m ∈ [1, M ], so the overall decoding complexity of the H-MIN algorithm can be effectively controlled by the predefined threshold D.
By carefully selecting the predefined threshold of degree D, we can systematically trade off complexity and performance in the H-MIN decoding. For example, a lower threshold D should be applied in order to reduce the complexity, where more check-nodes are updated by the MS simplification and only a few check-nodes with low degrees are approximated by linear functions. V. CONCLUSION
In this paper, a hybrid decoding algorithm is proposed for the LDPC codes in 5G NR. Simulation results show that our proposed algorithm has only a slight increase in complexity with respect to the NMS algorithm and an improved performance much closer to the BP decoding, especially for the low rate codes. It provides a better trade-off between decoding performance and complexity. Also, the layered-based scheduling which allows the updated information to be used more quickly than the flooding decoder can efficiently applied for the proposed decoding algorithm presented in this paper. Both the numerical results of PEXIT analysis and decoding simulations show that the H-MIN decoding has a noticeable performance gain over the conventional NMS algorithm for the low rate LDPC codes in 5G.
