Abstract—In this paper, an iterative soft-decision (SD) decoding algorithm for cyclic codes based on extended parity-check equations is developed. The algorithm does not necessarily utilize the algebraic properties of the code, but operates on transforming the systematic parity-check matrix using the soft reliability information matrix obtained from the received vector. Results show a significant performance gain when compared with the hard decision Berlekamp-Massey (B-M) and belief propagation (BP) algorithms, but present a similar bit error rate (BER) performance when compared to the adaptive belief propagation (ABP) algorithm. An important feature of the decoder is that it functions within a practical decoding time complexity, and can be generally implemented for the class of linear block codes.
Index Terms—Soft-decision decoding, Cyclic codes, Extended parity-check equations, Berlekamp-Massey Algorithm, Belief Propagation Algorithm, Adaptive Belief Propagation Algorithm.
I. INTRODUCTION
Cyclic codes are significant for error control in communication systems due to their algebraic structure, which makes encoding and decoding more efficient. Algebraically, cyclic codes of length n in a vector space Fqn, corresponds to the study of ideals in the residue class ring Rn = Fq[x]/(xn − 1), and the study of ideals in Rn centers on finding the irreducible factors of X n − 1 over GF (q).
For a class of binary cyclic code, traditional hard decision decoders are used to correct up to t = (dmin − 1)/2 errors, where dmin is the minimum distance of the code. However, this decoding method yields loss of information that reduces performance, hence a need for efficient decoder. Soft-decision (SD) decoding utilizes channel measures to correct errors beyond half the minimum distance of a code. A major SD decoder is the belief propagation (BP) based iterative decoding that emerged in [1], [2]. The BP algorithm is practically used to decode low-density parity-check (LDPC) codes and requires that the parity-check matrix is sparse, does not contain short cycles in the related Tanner graph, and has regular or optimized irregular row and column distributions [3].
As demonstrated by [4], the (n = 2m − 1, k, dmin ) primitive BCH codes with rate R ≥ 1/2 and 3 ≤ m ≤ 8 cannot have

Tanner graph free of length 4-cycles. Due to the relationship that exists between performance of the BP algorithm and the Tanner graph properties of codes, different suitable approaches of BP decoding for other classes of short linear block codes was proposed in [5]–[10], which focuses on reducing the number of cycles of length four in the graphical representation of the codes.
A more general approach of implementing BP algorithm for RS codes and any linear block code is given in [9] and [10]. This technique is known as the adaptive belief propagation (ABP) and are based on the adaptive procedure of the paritycheck matrix at each iteration. The bit reliabilities are utilized such that the unreliable submatrix of the binary parity-check matrix are initially reduced to a sparse form and then the BP algorithm is applied to the adapted parity-check matrix. ABP improves the performance of RS codes over HD decoding and traditional BP algorithm, but a major problem is to practically implement modifications of the graph at each iteration.
In this research, we introduce an iterative soft decision decoding for a class of binary cyclic codes, which involves transforming the parity-check matrix of the code, and is implementable on a real-time coding scheme.

II. PARITY-CHECK TRANSFORMATION ALGORITHM

The parity-check matrix transformation algorithm (PTA) [11] is developed based on the BP and is similar to the ABP, which improves the coding performance of a soft-decision decoder specifically for RS codes. The PTA functions as a simple symbol level iterative SD decoder and does not require removing 4-cycles from Tanner graphs, but rather transforms the parity-check equations at each iteration and utilizes the hard decision output of the decoder. The algorithm depends on the symbol reliabilities at each iteration, which are obtained by computing a soft reliability information matrix β.
A core part of the algorithm is the transformation stage. An efficient way to implement the parity-check transformation (HT ) is to form an invertible submatrix U containing unreliable symbols from the parity-check matrix H, and derive

HT = H · U −1.

This method of transformation is restricted to maximum distance separable (MDS) codes such as RS codes, which have every set of redundancy columns of its parity-check matrix to be linearly independent, but fails for other classes of linear block codes.
However, in this paper, we consider a more general transformation technique based on extending the parity-check transformation algorithm (PTA) [11]. The proposed method reduces the computational complexity of the ABP algorithm, particularly at the message updating stage, further enabling the parity-check matrix to be transformable at each iteration of the binary cyclic code. The algorithm can be implemented for any class of linear block code, but we focus on the class of cyclic codes due to the algebraic properties that can be utilized during encoding and decoding processes. The proposed EPTA is based on the soft reliability information matrix β and sorting the reliability bits of the parity-check matrix H according to the reliability matrix, which is not necessarily sparse, and computing the reduced row echelon form of H at each iteration, according to a certain principle.

IV. TIME COMPLEXITY ANALYSIS
The worst-case time complexity of the ABP, PTA and EPTA is presented using Big-O notation, allowing for a fair comparison of the message updating stage between the algorithms. At the initialization stage, the ABP requires one operation per bit (up to the nth bit from the channel observation) to compute the reliability of each bit, while the PTA and EPTA each requires two operations per bit to obtain the bit index with the highest value, which is repeated for each of the n columns in the β matrix.
The next stage of the algorithm involves obtaining the most and least reliable bits by sorting the reliability vectors L in ascending order. Each of the ABP, PTA and EPTA algorithms requires at worst n2 comparisons. The parity-check transformation stage of the algorithm takes a large component of the per-iteration time complexity. For ABP and PTA, the transformed parity-check matrix is obtained using Gaussian elimination with O(n3) arithmetic operations, while the EPTA requires O(n2) operations for matrix reordering.

V. SIMULATION RESULTS
A. Error rate performance analysis for different δ
We observe the simplicity of the proposed algorithm by considering the parameter δ, as it determines the number of iterations and bit error rate performance of the algorithm. We assume different δ values and investigate the performance of a high rate (15, 11) binary cyclic code for the same data using the BPSK constellation scheme with AWGN.
In Fig.1, we see that the smallest δ-value (δ = 0.001) requires more iterative steps, while δ = 0.9 quickly arrives at the negative element stopping condition before the algorithm properly decodes the received vector. However, it is clear from Fig.2 that δ = 0.05 is computationally efficient, since it yields a similar decoding performance with the smallest δ value and requires moderate number of iterations. Thus, we assumed δ = 0.05 as the incremental and decremental operator.

B. Performance comparison analysis of the EPTA with other iterative soft decoding algorithms
