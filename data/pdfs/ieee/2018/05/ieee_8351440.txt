Layer-by-layer Adaptively Optimized ECC of NAND flash-based SSD Storing Convolutional Neural Network Weight for Scene Recognition
Keita Mizushina, Toshiki Nakamura, Yoshiaki Deguchi and Ken Takeuchi
Department of Electrical, Electronic, and Communication Engineering, Chuo University, Tokyo, Japan E-mail: mizushina@takeuchi-lab.org

Abstract— Layer-by-layer Adaptively Optimized Error Correcting Code (ECC) is proposed to improve the reliability of triple-level cell (TLC) NAND flash-based SSD for the scene recognition using convolutional neural network (CNN) of IoT edge devices. Layer-by-layer Adaptively Optimized ECC is composed of Layer-by-layer Iteration-Optimized Low Density Parity-Check (LBL-LDPC) and Layer-by-layer Code-length Adjusted Asymmetric Coding (LBL-AC). The conventional techniques like LDPC ECC and Asymmetric Coding (AC) improve the reliability. However, they require large overheads of the ECC decoding time and the flag/parity cell. Proposed LBL-LDPC and LBL-AC decrease the ECC decoding time by 14% and the data overhead by 26%, respectively, without recognition accuracy degradation. In addition, the data-retention time extends by 230%.
Keywords— NAND flash memory; Convolutional neural network; SegNet; ECC; low-density parity-check; LDPC ECC; Solid-state drive; SSD; scene recognition;
I. INTRODUCTION Machine learning is applied in various fields, and its scope continues expanding. Especially, scene recognition technologies like semantic segmentation are useful. Semantic segmentation, which means classifying objects or persons pixel by pixel, is a key technology in IoT systems such as autonomous cars and industrial robots [1]. For example, scene understanding by semantic segmentation is critically important for autonomous cars to make decisions. Fig. 1 shows the concept and overview of this paper. Because deep neural network like convolutional neural network (CNN) needs huge amount of calculations in the training, the training is processed at data centers with GPU servers. The trained results of weights are stored of IoT edge devices such as autonomous cars and inference at edge devices. In this condition, SSD with TLC triple-level cell (TLC) NAND flash memory are used as a storage. TLC NAND flash memory cause many memory cell errors and error correcting code (ECC)

The deep neural network architecture used in this paper is SegNet [1], which is one kind of CNN for semantic recognition [5]. The structure of SegNet is composed of 10 convolution layers and the same number of deconvolution layers as shown in Fig. 2. Each layer has weights and biases at the filter. How much the precision of weights or data affects recognition accuracy are different layer-by-layer. Just like with weights of general artificial neural networks, weights are gradually optimized during the training at data centers. Finally, the calculated weights are stored in SSDs of IoT edge devices. In the convolution process, each filter gathers the features of input images. The accuracy of the output image classification is determined by weights. On the other hand, bit error rate (BER) of TLC NAND flash memory increases as the data-retention time increases as shown in Fig. 3 (a). The data-retention error is caused by electron leakage or de-trap from the floating gate/charge trap layer of memory cells, which decreases the threshold voltage (VTH) (Fig. 3 (b)) [6, 7, 8]. To decrease BER, the reliability enhancement techniques have been proposed but they require large data overheads [9, 10, 11]. To solve these problems, this paper proposes Layer-by-layer Adaptively Optimized ECC to improve the reliability of SSDs for CNN with low cost.

Fig. 7 (a) shows the flowchart of LDPC ECC decoding [4]. The strength of LDPC ECC depends on the number of iterations as shown in Fig. 7 (b). As more decoding iterations repeat, more errors are corrected. On the other hand, increasing the iterations also increases the LDPC decoding time as shown in Fig. 7 (c). Fig. 8 shows the measured results of LBLLDPC. By setting the minimum iterations for each layer, LBLLDPC can maintains the high ABER, which is almost equivalent to the conventional LDPC where the maximum decoding iterations, 30, are applied for all layers. By introducing LBLLDPC, high RAR is maintained up to 75% of BER. Because LBL-LDPC decoding time depends on the number of iterations, reducing the number of iterations is effective to improve data access latency and throughput. Fig. 9 shows the comparison of decoding time with proposed LBL-LDPC. Proposed LBLLDPC with the maximum iterations: (L1, L2, L3, L4) = (5, 15, 15, 30) reduces the LDPC decoding time by 14%.

III. LAYER-BY-LAYER CODE-LENGTH ADJUSTED ASYMMETRIC CODING
Asymmetric Coding (AC) [9, 10] is used to improve the reliability of TLC NAND flash memory as shown in Fig. 10. When random data is written to TLC NAND flash, measured BER differs for each VTH-state as shown in Fig. 10 (a). For TLC NAND flash memory, AC2 that increases the population of “A”state is the best way to improve the average reliability of all the input data as shown in Fig. 10 (b). To increase the population of “A”-state cells, user data are flipped or not flipped so as to increase “1” within the code length as shown in Fig. 10 (c). If half or more of the user data within the code length is “0”, AC2 encoder flips all data and appends a flag bit “1” to the user data.

When the user data includes “1” more than “0” within the code length, AC2 encoder just adds a flag bit “0” to the user data

On the other hand, during AC2 decode, AC2 decoder checks the flag bit and convert data stored in NAND flash to the original input data. If the flag bit is “1”, AC2 decoder removes a flag bit and flips all bits. Then, AC2 decoder outputs flipped data if the flag bit is “0”, AC2 decoder removes a flag bit and outputs user data as it is. Fig. 11 shows measured BER with AC2. BER is reduced as the code length (CL) becomes smaller, but the flag overhead becomes larger. Conversely, as CL becomes longer, the flag overhead becomes smaller. Thus, there is a clear trade-off of CL about BER reduction and the flag overhead.

Fig. 12 shows the concept of proposal 2, LBL-AC. LBL-AC adaptively optimize CL for each layer. By removing the large CL, the flag overhead is minimized. In this paper, two cases, Case 1 and 2, are evaluated. LBL-AC Case 1 protects central 4 layers and Case 2 protects central 6 layers. Because the layers near the center of CNN cannot tolerate errors as shown in Fig.5, small CL such as 4 is the optimal for central layers. IV. LAYER-BY-LAYER ADAPTIVELY OPTIMIZED ECC LBL-LDPC adaptively optimized the number of LDPC iterations layer-by-layer and reduces LDPC decoding time by 14%. LBL-AC modifies the code length of AC layer-by-layer and reduces the data overhead by 26%. Finally, Fig. 15 shows the measured results of Layer-by-layer Adaptively Optimized ECC that combines two proposals. Both the LDPC decoding time reduction by LBL-LDPC and the data overhead rate reduction by LBL-AC are achieved. The proposed ECC can increase the acceptable BER by 230%.
V. SUMMARY Table 1 shows the measured evaluation summary. This paper proposed and evaluated LBL-LDPC and LBL-AC for Layer-by-layer Adaptively Optimized ECC improving the reliability of SSD for CNN. LBL-LDPC decreases LDPC decoding time by 14% by changing the strength of LDPC iterations according to the importance of weight data. LBL-AC also reduces data overhead by 26% by changing the code length of AC according to the importance of weight data. By combining two techniques, Layer-by-layer Adaptively Optimized ECC extends the data-retention time by 230%. The proposed ECC is not limited to SegNet and can be applied to various CNN applications.
