Reconfigurable Decoder for LDPC and Polar Codes
Ningyuan Yang1,2,3, Shusen Jing1,2,3, Anlan Yu1,2,3, Xiao Liang1,2,3, Zaichen Zhang2,3, Xiaohu You2, and Chuan Zhang1,2,3,∗
1Lab of Efficient Architectures for Digital-communication and Signal-processing (LEADS) 2National Mobile Communications Research Laboratory, Southeast University 3Quantum Information Center of Southeast University, Nanjing, China
Email: {nyyang, shs.jing, anlan, xiao liang, zczhang, xhyu, chzhang}@seu.edu.cn

Abstract—With low-density parity-check (LDPC) code and polar code selected as the standard codes for 5G eMBB scenario, one challenge is how to improve the hardware efficiency when both decoders are required by one system. Since LDPC and polar codes can be decoded with belief propagation (BP) algorithms, this similarity allows us to design a reconfigurable decoder, which can decode both codes at the cost of only one decoder. Numerical and implementation results are also given in this paper to show that the proposed decoder achieves higher hardware efficiency than stand-alone LDPC or polar decoder, without harming the error performance.
Index Terms—LDPC codes, polar codes, belief propagation, reconfigurable decoder.
I. INTRODUCTION
Low-density parity-check (LDPC) code [1] was first proposed by Gallager in 1961, and became famous for its near Shannon-capacity performance and high decoding parallelism. The standard decoding algorithm for LDPC is belief propagation (BP), which takes advantage of the code sparsity and iterative decoding. For better balance of performance and complexity, variants of BP such as min-sum and layered minsum were proposed [2]–[4]. Nowadays, LDPC code has been widely adopted for applications and become the standard code for eMBB data channel by 3GPP.
Polar code [5] is another breakthrough in channel coding after LDPC code. Invented by Arıkan in 2008, it is the first code proved to achieve channel capacity of symmetric binary-input discrete memoryless channels (B-DMCs). Recently, polar code has been selected as the code for eMBB control channel by 3GPP. For polar decoding, successive cancellation (SC) was proposed together with polar code. Then, SC list (SCL) was proposed to further improve the error performance of SC. Although SCL significantly improves the performance, it suffers from high decoding complexity. Furthermore, since SC and SCL decoders are tree-searching based, they suffer from low parallelism and high delay. To address the problem, BP decoding, which can be carried out in parallel, was employed [6]–[9] for high throughput applications.
Since both LDPC and polar codes have been set as standard codes for eMBB, one of the utmost challenges is to implement both decoders efficiently at the same time. One straightforward solution is to have two separated decoders which take care of both codes, respectively. Admittedly, highlevel transforms such as folding and fine-grain optimization can help to improve the efficiency. The resulting decoders

remain not satisfactory. To this end, this paper devotes itself in proposing a reconfigurable BP decoder, which can decode both LDPC and polar codes with the same hardware. This target is achieved by: 1) revealing the similarity of LDPC and polar codes’ BP algorithms, and 2) reconfiguring a set of basic computational units. Complexity analysis and numerical results have indicated the efficiency advantages of proposed decoders over stand-alone ones. Such advantages have also been verified by FPGA implementations in this paper.
The remainder of this paper is organized as follows. Section II reviews the preliminaries. The proposed reconfigurable BP decoder for both codes is given in Section III with detailed module introduction. Complexity analysis and numerical results are given in Section IV. FPGA implementation is given in the same section. Section V concludes the entire paper.

For polar decoding, the MBCB’s function is the same to the BCB’s. For LDPC decoding, 2 L(rji)s are calculated by 2 CUs. As a result, 8 GFUs in a pair of MBCBs are reused to carry out a CN’s computation. Fig. 10 illustrates the corresponding architecture for a pair of MBCBs. GFUs and adders are connected according to Fig. 9. Besides, adders in MBCBs are reused to carry out VNs’ computation. Fig. 11 illustrates the architecture for the proposed reconfigurable BP decoder. It is made up of 48 GFUs and 48 adders, and its connection modes are shown in previous figures.
Blue squares representing 3 stages of MBCBs are connected to implement polar decoding. In a pair of MBCBs, 8 GFUs and 6 adders are reused to make up a CN and 2 VNs, respectively. For LDPC decoding, 6 CNs and 12 VNs are connected according to the factor graph. The input S determines which connection mode to be switched. If both decoding processes are implemented separately by a polar decoder and an LDPC decoder, 96 CMPs, 96 XORs and 84 adders are needed in total. As a result, CMPs, XORs and adders are reduced by 50%, 50%, and 43%, respectively.

D. Reconfigurable BP Decoder
To implement both decoding processes, we design the merged BCB (MBCB). Fig. 9 illustrates an architecture for the 9-input and 6-output MBCB, which is alternative to carry out a BCB’s computation or part of the 0th CN’s computation. There are two modes of inner connection in the MBCB. Black lines and red lines represents the mode for polar decoding and LDPC decoding, respectively. The input signal S is used for switching the two connection modes.

B. Numerical Results
Fig. 12 compares the bit error rate (BER) performance of the float scheme and the fixed scheme for both decoding processes. The fixed scheme employs 1 sign bit, 4 integer bits and 2 decimal bits. When BER = 10−2, compared with the float scheme, the fixed scheme has 0.2 dB and 0.5 dB loss for polar decoding and LDPC decoding, respectively. Hardware’s performance loss is acceptable under this condition. FPGA implementation results are given in Table II.
V. CONCLUSION
In this paper, we propose a reconfigurable BP decoder, which can flexibly decode polar code or LDPC code. Theoretical analysis shows that the decoder requires lower hardware

complexity, while numerical results indicate negligible performance loss. FPGA implementation results are also given Future work will be directed towards further optimization of performance and architecture, and its application in our 5G Cloud Platform.
