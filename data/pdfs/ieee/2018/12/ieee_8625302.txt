AbstractвҖ”This paper presents a technique to scale up an LDPC code to any desired block length. Under certain constraints, the scaled LDPC code results in a special case of globally-coupled-LDPC code. The proposed scheme can construct globally-coupled-LDPC codes from any short LDPC codes. A design example targeting for NAND flash memory applications is addressed and simulated. The results show the feasibility of constructing a structural long LDPC code by scaling from a short code.
Index TermsвҖ”low-density parity check code, length-scalable LDPC code
I. INTRODUCTION
Since the rediscovery of low-density parity check (LDPC) codes [1], [2], [3], vast results from code constructions, performance analysis, to hardware implementations were extensively brought out by numerous researchers over the past two decades. LDPC codes have been widely adopted as a strong error correction coding (ECC) scheme in many applications. Being a block code, the error correcting capability of an LDPC code increases with its block length. In applications, such as optical communication or memory controllers, longer block lengths are considered for low frame error rate scenarios, 10вҲ’15 for instance. Block length can be 4K-Byte or longer in many NAND flash memory applications [4].
Increasing the block length to enhance the performance is common when upgrading the system specification. Take the NAND flash memory for example, as its capacity grows, the stored data are even prone to errors. Manufacturers are considering LDPC codes of large block lengths as more powerful ECC candidates. Block lengths of 2K-Byte, 4K-Byte, or even longer have become the mainstream specifications. Most of the LDPC codes are in-house designs and have their product life-cycles. As the memory circuit architecture migrates to the next generation, a longer LDPC code may be required to compensate the increasing raw bit error rates. Redesigning a longer LDPC code takes a much longer time from code construction to performance evaluation.
In this paper, a technique to scale up an LDPC codeвҖҷs block length is presented. The proposed scheme, which will

The scaled LDPC code has a structural parity check matrix; thus the decoder designed to be compatible with the earlier system becomes possible. The girth after array dispersion is at least the same as the original LDPC code. Moreover, the column/row weight distribution proportions remain the same. Therefore the performance of the scaled LDPC code is predictable. It will be shown in this paper that some special cases of array dispersion will result in globally-coupled low-density parity check (GC-LDPC) codes [6]. Array dispersion can be employed to construct GC-LDPC codes based on any LDPC codes.
The rest of the paper is organized as follows. The array dispersion procedure will be addressed in Section II. It will be shown in Section III, under certain constraints, the array dispersion can be used to construct GC-LDPC codes. Subsequently, applying array dispersion to construct GC-LDPC codes for NAND flash memory applications is discussed in Section IV. The simulation results in Section V show the feasibility of the proposed scheme. Finally, a summary of this work will be given in Section VI.

IV. APPLICATIONS IN NAND FLASH MEMORIES
Data access in NAND flash memories must be page by page, which is the basic data access unit. In the current state of the art NAND flash architectures, a page can be 16K-Byte or longer. If all the information stored in one page is encoded as one LDPC codeword, the block length is at least 16K-Byte, and the LDPC decoders have to be designed with manageable complexity. If one page data is encoded by multiple LDPC codes, each LDPC code should have sufficient long block length to meet the low frame error rate requirement.
The structural parity check matrix of a GC-LDPC code facilitates efficient decoding. A GC-LDPC code constructed by array dispersion is capable of having more local codes to support multi-phase decoding when the globally-connecting part is designed with hierarchy. That is, Hglobal can be partitioned by layers such that in each layer, the connected local codes can be partitioned into disjoint subsets. The subset size grows with the layer index. In the last layer, all the local codes belong to the same subset.
A simple multi-phase decoding strategy can be realized as follows. For a GC-LDPC code with S local codes, the S local codes are decoded independently at decoding phase-1,

or termed the local phase. Once the phase-1 decoding fails,

the decoder switches to phase-2 decoding to incorporate more

local codes. If the phase-2 decoding fails the decoder switches

to phase-3 decoding, and so on. In the last phase, all the local

codes work collaboratively. The maximum number of layers

is the row number of Pglobal in (25). This multi-layer global-

connecting can be carried out by specifying the permutation

ОҰ in (24).

It may be more preferable that a GC-LDPC code has more

local codes such that most of the time the errors can be

corrected solely by the local decoders which have much lower

complexity. In the following, we present an example of an S-

local GC-QC-LDPC code for NAND flash applications. Each

local code has a block length around 1K-Byte.

Comparing their globally connecting part of the 2KB-GC-1 (conventional construction) and 2KBGC-2, the former has a more regular structure, which leads to better connectivity as compared to the random structure of 2KB-Code-2. Therefore better FER of 2KB-Code-1 can be perceived at high the SNR region. Random permutation in constructing the globally-connecting part can be optimized with a better connectivity.

VI. SUMMARY
In this work, an array dispersion procedure is introduced as a technique to scale up an LDPC code to any block length. The proposed scheme can be employed to construct GC-LDPC codes, and the maximum number of local codes is not restricted. The unconstrained number of local codes provides additional flexibilities in both decoding strategies and the decoder hardware architectures. The simulation results address the feasibility to construct GC-LDPC code by the proposed method. Furthermore, the frame error rate improved with the increased block length proves the concept of array dispersion, which is to construct a long LDPC code from any off-the-shelf well designed short LDPC code.
