Density Evolution Based on Two-Type Degree Distribution Framework of Raptor-Like LDPC
Genning Zhang , Yin Xu, Dazhi He, Jun Sun, and Wenjun Zhang, Fellow, IEEE

Abstract—The traditional degree distribution framework is a useful tool to analyze irregular low density parity check (LDPC) code. However, it cannot be used to analyze raptor-like LDPC code as it cannot describe the most important structural feature of raptor-like LDPC code, which is that any check node is connected to at most one degree-one variable node. In this paper, we propose a two-type degree distribution framework for raptor-like LDPC code. The proposed framework can describe the special structural feature of raptor-like LDPC code by dividing the check nodes into two different sets. One is the set of check nodes connected to those degree-one variable nodes, the other is the set of check nodes unconnected to those degree-one variable nodes. Furthermore, we propose a two-type discrete density evolution algorithm and a twotype Gaussian approximation density evolution algorithm based on the proposed two-type degree distribution framework to estimate the threshold of raptor-like LDPC code. During the decoding analysis, the proposed algorithms treat the check nodes belonging the two separate sets differently. Compared with the traditional multi-edge type (MET) algorithm, the proposed algorithms allow us to calculate the threshold more quickly without much sacrifice in accuracy. For some raptor-like LDPC codes adopted by the Advanced Television Systems Committee 3.0 (ATSC3.0) standard, the thresholds estimated by the proposed algorithms can be very close to the thresholds estimated by the traditional MET algorithm. The deviations are less than 0.07 dB. Meanwhile, the proposed algorithms can tremendously reduce the computation complexity because of the two-type degree distribution framework and the Gaussian approximation.
Index Terms—Degree distribution framework, raptor-like, LDPC, density evolution, Gaussian approximation.

Several techniques have been proposed in the literatures to analyze sum-product decoding by tracking the density of the messages exchanged in the decoder. This idea was first proposed by Richardson et al. [5], [6]. They introduced the conceptual degree distribution framework to represent LDPC codes. Thus, the density of messages transmitted between the variable nodes and check nodes of LDPC codes can be calculated. The calculation method is named density evolution. Generally, the numerical evaluation of density of messages is computationally intensive. So, Chung et al. [7] and Lehmann and Maggio [8] approximated the density of messages as Gaussian to simplify the analysis. Brink [9] proposed the extrinsic information transfer (EXIT) charts algorithm based on the Gaussian approximation. The contribution of Gaussian approximation is that the density of messages can be faithfully surrogated by a onedimensional quantity, without much sacrifice in accuracy. All these algorithms are based on the traditional degree distribution framework. Although, this traditional degree distribution framework is useful in most situations, it can not be used for some LDPC codes with special structures [10], such as the LDPC codes including degree-one variable nodes. Therefore, Richardson et al. [10], [11] introduce another generalization of LDPC codes, which is named multi-edge type (MET) framework [10], [11] of LDPC codes. And they extended density evolution algorithm to this MET framework. This framework can perform better than the above degree distribution framework, especially for the situations that density distribution framework can not work.
Recently, a new structure of LDPC called raptor-like LDPC code [12] has been recognized to have better performance at low code rate. And it also has advantages in the design of rate compatible codes. This kind of LDPC codes has been used in the latest communication standard, such as recently published ATSC3.0 standard [13], [14]. The most important structural feature of raptor-like LDPC code is that it has some degree-one variable nodes and any check node is connected to at most one degree-one variable node. Therefore, both density evolution and Gaussian approximation density evolution algorithms based on traditional degree distribution framework can not analyze raptor-like LDPC codes. Only the algorithm based on MET framework can analyze raptor-like LDPC codes. However, it is inconvenient to use MET algorithm to analyze the decoding process, because the computation complexity is too high [10]. There are two reasons resulting in the high computation complexity issue. One reason is that MET algorithm uses edge-type instead of degree to define the variable nodes and check nodes. For most LDPC codes, the amount of node types defined by the edge-type is more than the amount of node types defined by degree. The other reason is that we need to track the density of messages when we use the MET algorithm, and the density is an infinite-dimensional vector.
In order to analyze raptor-like LDPC code quickly and effectively, we propose a two-type degree distribution framework to represent the raptor-like LDPC code. This proposed framework divides the check nodes into two sets. One is the set of check nodes which are not connected to the degree-one variable nodes, the other is the set of check nodes which are connected to only one degree-one variable nodes. Therefore, this proposed framework can represent the structural feature that any check node is connected to at most one degree-one variable node. Based on this proposed framework, we also propose two algorithms which can analyze raptor-like LDPC codes more conveniently than MET algorithm without much sacrifice in accuracy. The first one is two-type discrete density evolution algorithm. This algorithm tracks the density of message during the variable node and check node operations. It considers the check nodes connected and unconnected to degree-one variable nodes, separately. The second one is two-type Gaussian approximation density evolution. This algorithm uses the mean of Gaussian, which is a one-dimensional quantity, to surrogate the density of message. This surrogate simplifies the analysis without much sacrifice in accuracy.
The rest of this paper is organized as follows. The two-type degree distribution framework and two-type discrete density evolution algorithm are introduced in Section II. The twotype Gaussian approximation density evolution algorithm is proposed in Section III. Finally, simulation results are presented in Section IV, followed by some concluding remarks in Section V.
II. TWO-TYPE DISCRETE DENSITY EVOLUTION BASED ON TWO-TYPE DEGREE DISTRIBUTION FRAMEWORK OF RAPTOR-LIKE LDPC CODES
In this section, we first propose the two-type degree distribution framework to represent raptor-like LDPC code. Raptor-like LDPC code has an important structural feature that it has some degree-one variable nodes and any check node is connected to at most one degree-one variable node. Under the proposed framework, we are able to take the important structural feature of raptor-like LDPC code into consideration when we track

the density of message. Based on this proposed framework, we further propose a two-type discrete density evolution and then deduce the density update operations at variable and check nodes.
A. Two-Type Degree Distribution Framework
The traditional degree distribution framework can not be used to describe the connection relationship between the degree-one variable nodes and check nodes. Thus, if we use the traditional degree distribution framework to describe a raptor-like LDPC code, it may lead to the situation that a check node is connected to more than one degree-one variable node. And this situation will inadvertently lead to error probabilities bounded away from zero [11].
To solve this problem, the key of the two-type degree distribution framework is that it should be able to describe the important feature of raptor-like LDPC code, which is that any check node is connected to at most one degree-one variable node. To this end, we consider dividing both the variable nodes and check nodes into two parts as shown in Fig. 1. Va is the set of variable nodes whose degree is not one. Vb is the set of variable nodes whose degree is one. C1 is the set of check nodes which are not connected to Vb . C2 is the set of check nodes which are connected to Vb . In order to satisfy the structural feature of raptor-like LDPC code, every check node in C2 is connected to a unique variable node in Vb , which means there is a one-to-one correspondence between the variable nodes in Vb and the check nodes in C2. Meanwhile, we use E1 to denote the set of edges which are connected to Va and C1, and use E2 to denote the set of edges which are connected to Va and C2.
The proposed two-type degree distribution framework is listed in Table I and Table II. We will introduce the type-onedegree and type-two-degree to describe the two-type degree distribution framework as follows. In an LDPC code, any variable node is connected to some edges. Among these edges, if there are dv1 edges in E1, and dv2 edges in E2, then the type-onedegree and type-two-degree of the corresponding variable node is dv1 and dv2, respectively. Similarly, among the edges which are connected to a check node, if there are dc1 edges in E1, and dc2 edges in E2, then the type-one-degree and type-two-degree of the corresponding check node is dc1 and dc2, respectively. Obviously, for a raptor-like LDPC code, only the variable nodes in Va have both type-one-degree dv1, and type-two-degree dv2. The check nodes in C1 only have type-one-degree dc1 (which means the corresponding dc2 = 0), and the check nodes in C2 only have type-two-degree dc2 (which means the corresponding dc1 = 0).
In this framework, we use the type-one-degree and type-twodegree to define the node type. If the type-one-degree and typetwo-degree of two nodes are both the same, then the two nodes have the same node type. We use NV T to denote the amount of variable node types and NC T to denote the amount of check node types. it is able to describe the structural feature that any check node is connected to at most one degree-one variable node. Thus, the density evolution based on this framework can analyze the message transmission accurately in a raptor-like LDPC code. III. TWO-TYPE GAUSSIAN APPROXIMATION DENSITY EVOLUTION BASED ON TWO-TYPE DEGREE DISTRIBUTION

IV. SIMULATION RESULTS
A. The Accuracy of Proposed Algorithms
In this subsection, the performances of the proposed two density evolution algorithms based on the two-type degree distribution framework are evaluated. We will use raptor-like LDPC codes [14] adopted in ATSC3.0 (which is a recently published standard) as examples in this section. In details, we use the proposed algorithms to estimate the thresholds of raptor-like LDPC codes adopted by ATSC3.0. And we compare the estimated thresholds to the reference thresholds. The reference thresholds are the actual thresholds and the thresholds calculated by the density evolution algorithm based on the famous MET framework [10], [11]. The MET algorithm has been proved accurate and useful to estimate the threshold of LDPC code with degree-one variable node, which makes it appropriate to be a benchmark.
The LDPC codes adopted by ATSC3.0 have two different structures, one is raptor-like structure especially for low rate codes and the other is quasi-cyclic irregular repeat accumulate (QC-IRA) structure especially for middle-to-high rate codes. For an LDPC code with the QC-IRA structure, the type-twodegree will be always zero for both variable and check nodes, because QC-IRA LDPC code does not include degree-one variable node. In that case, the two-type degree distribution framework will be the same as the traditional degree distribution framework, and the proposed density evolution algorithms will be the same as the algorithms introduced in [7], [18] which have been proved useful to estimate the thresholds of QC-IRA LDPC codes. Therefore, we will only consider the raptor-like LDPC codes in this section.
The raptor-like structure is applied to both the long and short LDPC codes in ATSC3.0 standard. And we use these codes to evaluate the proposed framework and algorithms. The corresponding degree distributions described by the two-type degree framework are listed in Table III–X. In each table, the variable

node types are listed in the left part and the check node types are listed in the right part. The theoretical and actual thresholds are listed in Table XI, and specially, to make it simple, “two-type GA” represents the proposed two-type Gaussian approximation density evolution algorithm, “two-type DE” represents the proposed two-type discrete density evolution algorithm, “MET DE” represents traditional discrete density evolution algorithm based on the MET framework, “MET GA” represents the MET mean density evolution algorithm based on the Gaussian approximation [21], and “Actual” represents the actual threshold. The actual threshold indicates the snr where BER performance is targeted around 10−7.
From the tables, we can draw three conclusions. First, all the estimated thresholds calculated by the two-type GA, two-type DE, MET GA, and MET DE algorithms are better than the actual threshold. This is because that our theoretical analysis (two-type GA, two-type DE, MET GA, MET DE) is based on the “local tree assumption”, which means all the messages sent to the same node are independent. For an actual raptor-like LDPC code whose codelength is not infinity, due to the existence of cycles, the messages sent to the same node is not independent. Therefore, the estimated threshold should always be better than the actual threshold and the gaps should decrease as the actual code length increases. As you can see in Table XI, the gaps between all the theoretical thresholds and the actual threshold are less than 0.38 dB for the long code, and the gaps are all less than 0.67 dB for the short code. These results prove that the gaps decreases as the code length increases.
The second conclusion is that the thresholds calculated by the proposed two-type DE algorithm are very close to the thresholds calculated by MET DE algorithm. In Table XI, all the differences between two-tyoe DE and MET DE are less than or equal to 0.02 dB. These results indicate that our proposed two-type discrete density evolution algorithm is accurate for raptor-like LDPC code, when MET DE algorithm is used as a benchmark.
The third conclusion is that the gaps between two-type GA algorithm and two-type DE algorithm are very small. To be specific, all the gaps between two-type GA and two-type DE are not more than 0.06 dB. These results prove that the mismatch in Fig. 4 is not important. Furthermore, the gaps between two-type GA algorithm and MET GA algorithm are all less than 0.03 dB, and the gaps between two-type GA algorithm and MET DE algorithm are all less than 0.07 dB. These small gaps can prove that the proposed two-type GA algorithm is very effective and accurate.

B. The Computation Complexity
In this subsection, we will compare the computation complexity between the proposed algorithms and the benchmark MET DE algorithm. In Section I, we have already introduced that there are two reasons which result in the high computation complexity of MET DE algorithm. One reason is that we need to track the density of messages when we use the MET algorithm, and the density is an infinite-dimensional vector. The other reason is that MET algorithm uses edge-type instead of degree to define the variable and check node types. For most LDPC codes, the amount of node types defined by the edge-type is more than the amount of node types defined by degree.
The proposed algorithms can reduce the complexity resulting from the two reasons mentioned above. For the first reason, the Gaussian approximation assumption in the proposed algorithm can effectively surrogate the infinite-dimensional vector by a one-dimensional quantity, which can significantly reduce the computation complexity. For the second reason, it is easily found out that the amount of node types defined by the proposed two-type degree distribution framework is less than the amount of node types based on MET framework for most raptor-like LDPC codes. V. CONCLUSION
In this paper, we propose a two-type degree distribution framework which can be used to describe and analyze raptorlike LDPC code. And based on this framework, we propose a two-type discrete density evolution (DE) algorithm and a twotype Gaussian approximation (GA) density evolution algorithm to estimate the threshold of raptor-like LDPC code. The framework divides the check nodes into two sets. One is the set of check nodes which are connected to only one degree-one variable node, the other is the set of check nodes which are not connected to degree-one variable nodes. This framework can well describe the structural feature of raptor-like LDPC code which is that any check node is connected to at most one degreeone variable node. The proposed two-type discrete density evolution algorithm tracks the message transmission based on the proposed framework. For the check node operations, the proposed algorithm consider the check nodes connected and unconnected to the degree-one variable node, differently. Moreover, the two-type Gaussian approximation density evolution uses the mean of Gaussian to approximate the message when analyzing the message transmission.
