FPGA IMPLEMENTATION OF PARITY CHECK MATRIX BASED LOW DENSITY
PARITY CHECK DECODER

H. Praveena, K.Kalyani

ME Communication Systems, Department of ECE, Thiagarajar College of Engineering, Madurai,

Abstract - Low Density Parity Check (LDPC) error correction decoders have become popular in diverse communication systems, owing to their strong error correction performance and their suitability to parallel hardware implementation. Low Density Parity Check Decoder is a class of Forward Error Correction Codes. The parameterization of a particular LDPC code is defined by its Parity Check Matrix. Parity Check matrix which describes the specific logical combination of the transmitted message bits into parity checks. PCMs are sparse matrices having far more zero entries than nonzero. It allows LDPC codes to be iteratively decoded using a low complexity message passing algorithm. Thus the objective of this project is to implement a PCM based LDPC decoder architecture on FPGA. Initially the components of LDPC decoder are identified then Verilog HDL coding for these components have been written, simulated using Altera Quartus II software. The implementation of these components are done on Altera Cyclone II FPGA kit to improve the error correction performance of communication system.
Keywords: Low Density Parity Check (LDPC) codes,
PCM, FPGA.
I INTRODUCTION
Low-Density-Parity-Check (LDPC) codes[l] have recently received a lot of attention because of their excellent error-correcting capability. LDPC codes have able to perform close to the Shannon limit [2]. They also can achieve very high throughput because of the parallel nature of their decoding algorithms. LDPC codes are a sub-set of linear block codes, defined by sparse parity check matrix H, to which a Tanner graph can be coupled as for any linear block code. This bipartite graph is formed by two types of nodes, Check Nodes (CN), one per each code constraint (H rows), and Variable Nodes (VN), one per each bit of the code word (H columns), with the

connections between them given by H. General LDPC codes is that a significant amount of memory is needed to store their parity-check matrices[3].Low Density Parity Check decoder needs a flexibility for design the space[4] and presented a high throughput implementation of communication standard such as DVB-S2.FPGA architecture has an effect on the quality of the device speed performance, area efficiency and power consumption and it develop the fundamental programming logic device[5]. The implementation of high performance flexible LDPC decoder design is difficult [6], so formulate the decoding algorithm to simplify the internal communication. The decoding architecture have a unique characteristics of a FPGA to improve the throughput [7]. It presents the multi level hierarchical quasi cyclic matrix for LDPC decoder and provide the flexibility in generate the different code lengths and code rate for communication standards [8].

A.VARIABLE NODE DECODER MEMORY
Variable Node Decoder first block is Block RAM. It consist of two extrinsic memory such as Variable Node Decoder Memory, Check Node Decoder Memory. Variable Node Decoder Memory read the sub matrix stored by the block RAM.
B. MULTIPLEXER
The Block RAM output is given as the input of Multiplexer. That selects one of several analog or digital input signals and forwards the selected input into a single line. Multiplexers are mainly used to increase the amount of data that can be sent over the network within a certain amount of time and bandwidth.
C. VARIABLE NODE PROCESSING UNIT
The VNPU has a priori LLR inputs and one intrinsic LLR input provided by the channel. It also has extrinsic LLR outputs and one a posteriori LLR output used as the basis of a decision for the corresponding LDPC- encoded bit. VNPU is required to calculate the sum of the intrinsic LLR input and all a priori LLR inputs besides the corresponding one. The a posteriori LLR output is obtained as the sum of the intrinsic LLR input and all a priori LLR inputs.
D. INTERLEAVER
The Inter leaver consist of Barrel Shifter (BS) is a digital circuit that can shift a data word by a specified number of bits without the use of any sequential logic, only pure combinatorial logic. One way to implement it is as a sequence of multiplexers where the output of one multiplexer is connected to the input of the next multiplexer in a way that depends on the shift distance. A barrel shifter is often used to shift and rotate n-bits within a single clock cycle.
E.CHECK NODE DECODER MEMORY
Check Node Decoder Memory will store the output produced by the de multiplexer. It consist of Block RAM and store the value calculated by the variable node decoder until they are needed by the check node decoder. Check node decoder process each row at single clock cycle. The process of check node memory is write the data.
