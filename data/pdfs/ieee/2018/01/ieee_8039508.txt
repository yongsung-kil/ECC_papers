A New Multi-Edge Metric-Constrained PEG Algorithm for Designing Binary LDPC Code
With Improved Cycle-Structure
Xuan He, Student Member, IEEE, Liang Zhou, and Junyi Du, Student Member, IEEE

Abstract— The progressive edge-growth (PEG) algorithm constructs an edge in each stage to maximize the variable node (VN) of interest’s local girth in real time. Thus, this VN’s local girths, after more than one edge is added to the current tanner graph (TG) setting, may not be maximized relative to that TG setting. To address this problem, we define the multiedge local girth and edge-trial, and based on these definitions, propose a new multi-edge metric-constrained PEG algorithm (MM-PEGA) to improve the design at each VN. The MM-PEGA constructs an edge in each stage that, relative to the current TG setting, can potentially maximize the VN of interest’s local girth after a certain number (up to the edge-trial) of edges are added to the TG setting. We first analyze the properties of the multi-edge local girth, and then propose an algorithm for calculating the multi-edge local girth. We also propose a method for accelerating the MM-PEGA. Moreover, we generalize the MM-PEGA for improving different PEG-like designs. According to the theoretical analysis, increasing the edge-trial of the MM-PEGA is expected to positively affect the cycle-structure and the error performance of resulting low-density parity-check (LDPC) code. This expectation is verified by simulations.
Index Terms— Approximate cycle extrinsic message degree (ACE), binary low-density parity-check (LDPC) code design, computational complexity, cycle-structure, edge-trial, metric, multi-edge local girth, progressive edge-growth (PEG).
I. INTRODUCTION
R ECENTLY, low-density parity-check (LDPC) code [3] has been adopted by the 3rd Generation Partnership Project (3GPP) as the single channel coding scheme for the 5G enhanced mobile broadband (eMBB) data channel. Consequently, the need to improve the design of LDPC code has become urgent and significant. It is well known that cycles with short lengths and poor connectivity to the rest of a Tanner graph (TG) [4] significantly degrade LDPC code’s error performance, and enlarging the girth [5], [6] and enhancing the cycle-connectivity [7]–[16] are effective in improving the error performance.

The progressive edge-growth (PEG) algorithm [5] is able to construct TGs with large girths. In addition, the authors of [10] found that cycle-connectivity, measured by cycles’ extrinsic message degrees (EMDs), or simply measured by cycles’ approximate cycle EMDs (ACE), has great impact on LDPC codes’ error performances. Since then, many related works based on [5] and [10] have been carried out. To the best of our knowledge, the improved PEG (IPEG) algorithm [11] is the earliest to apply the ACE of shortest paths in PEG-like designs. As a result, the IPEG algorithm achieves a similar girth property to but a better ACE property than the PEG algorithm. Inspired by this improvement, the authors of [12]–[15] focus on a better use for the EMD of shortest paths under their PEG-like designs; meanwhile, the number of shortest paths is also considered in [13]–[15]. On the other hand, the algorithms in [6]–[9] preset desired girth and/or ACE properties and then try to achieve the target LDPC code through large trials, but these approaches may lead to construction failure.
The authors of [17]–[19] design LDPC code by optimizing the direct indicator of error performance, i.e., the stopping set [20] in the binary erasure channel (BEC) or the trapping set [21] in the additive Gaussian white noise (AWGN) channel. However, due to the high computational complexity for approximating the stopping and trapping sets [22], only the simplest cases are analyzed in [17] and [19], and the method in [18] may not work well for long codes. Because of these disadvantages, the well-known indirect indicators of error performance, such as the girth [5], [6], the ACE [10], the ACE spectrum [9], [13], [14], [16], and the number of shortest paths [6], [13]–[15], have thus been used in this paper for the design and analysis of LDPC code.
This paper aims to improve the PEG-like designs for binary LDPC code. A simple example is given to show that the variable node (VN) of interest’s local girths, after more than one edge is added to the current TG setting by the PEG algorithm, may not be maximized relative to that TG setting. This problem arises because the PEG algorithm constructs an edge in each stage to maximize this VN’s local girth in real time. To address this problem, we define the multi-edge local girth and edge-trial, and based on these definitions, propose a new multi-edge metric-constrained PEG algorithm (MM-PEGA) to improve the design at each VN. II. PRELIMINARIES, NOTATIONS, AND BACKGROUND
A. Graph
A graph is denoted as G = (V, E), with V the set of nodes and E the set of edges. An edge connecting nodes u0 and u1 is denoted as (u0, u1). At the same time, (u0, u1) is called incident to u0 and u1, or is regarded as an (incident) edge of u0 and u1. A path of length L connecting nodes u0 and u L is denoted as a sequence of L + 1 nodes u0u1 · · · u L, where (ui , ui+1) ∈ E for 0 ≤ i < L. This path is called simple if ui = u j and u j = ut for 0 ≤ i < j < t ≤ L. (In this paper, only simple paths are considered.) Moreover, this simple path forms a length-L cycle (cycle of length (size) L) if u0 = u L. For each x, y ∈ V , if there exists at least one path connecting x and y, they are said to be connected. In this case, the distance between x and y is defined as the length of any shortest path connecting them. However, when x and y are not connected, their distance is defined as ∞.

C. ACE and ACE Spectrum
The ACE [10] value of a path is defined as j (dv j − 2), where dv j is the degree of the j -th VN of this path, and the summation is taken over all VNs of this path. The ACE spectrum [9] of depth dmax of a TG G is defined as a dmaxtuple η(G) = (η2, η4, . . ., η2dmax ), where η2i , 1 ≤ i ≤ dmax is the minimum ACE value of all 2i -cycles in G. If G does not contain any 2i -cycle, η2i is set as ∞. In addition, the comparison rule between two ACE spectra of depth dmax, say η(1) and η(2) (TGs are omitted), in [13] is defined as
η(1) > η(2) ⇐⇒ ∃ j ≤ dmax η2(1j) > η2(2j) and η2(1i ) = η2(2i ), 1 ≤ i < j .
With regard to this comparison rule, compared to TG with a smaller ACE spectrum, TG with a larger ACE spectrum has less chance to result in error-prone structures, such as small stopping and trapping sets. Thus, improving the ACE spectrum of an LDPC code has positive impact on the error performance, especially in a relatively high signal-to-noise ratio (SNR) region [9], [13], [14], [16].

E. PEG Algorithm
The PEG algorithm [5] constructs a TG by establishing edges between VNs and CNs in stages, where in each stage a CN is selected to connect to a certain VN. The edges are established in the following order: v0 takes the first dv0 stages to establish its edges, and then v1 takes the next dv1 stages to establish its edges, and the construction process continues until vn−1 takes the last dvn−1 stages to establish its edges. III. MULTI-EDGE METRIC-CONSTRAINED PEG ALGORITHM
A. MM-PEGA
Since the PEG algorithm selects CNs to maximize vc’s local girth in real time, the final local girth of vc (vc’s local girth at the end of the dvc -th stage of vc) may not be maximized relative to the TG setting at the beginning of the first stage of vc. This situation is demonstrated in the following example.

D. Generalization of MM-PEGA
The selection strategies of the PEG-like designs in [5] and [11]-[15] are made from the following selection criteria:
a. Select the CN with the largest distance [5], [11]–[15]. b. Select the survivor with the largest minimum path ACE
metric [11]–[14]. c. Select the survivor with the smallest number of mini-
mum ACE shortest paths [13], [14]. d. Select the survivor with the smallest number of shortest
paths [13]–[15]. e. Select the survivor with the largest total EMD of shortest
paths [12]. f. Select the survivor with the largest average EMD of
shortest paths [15]. g. Select the survivor with the minimal
degree [5], [11]–[15]. h. Select the survivor randomly [5], [11]–[15].
In the rest of this paper, for convenience, we refer to any of the above selection criteria by its index (in italic font), and any selection strategy made from the above selection criteria is denoted as a sequence of the indices of corresponding selection criteria of decreasing priority. For example, Strategy 1 is denoted as agh. Since each selection criterion from {a, b, c, d, e, f} corresponds to a metric measuring certain attributes of shortest paths (for example, a and d correspond to the length and number of shortest paths, respectively), we call these selection criteria and their corresponding metrics the shortest-path-related (SPR) selection criteria and SPR

metrics, respectively. If no ambiguity exists in a case, an SPR metric is referred to by its corresponding selection criterion’s index.
The authors of [11]–[15] have improved the PEG algorithm [5] by adding additional SPR selection criteria to Strategy 1. Each with the results in [5] and [11]-[15] belongs to the general form of aPgQh, where both P and Q refer to a sequence (may be empty) of the indices of SPR selection criteria of decreasing priority throughout this subsection, and the SPR selection criteria involved in aPQ differ. For the PEG-like design with selection strategy aPgQh, a simple improvement can be made based on the MM-PEGA by replacing the third and fourth selection criteria of Strategy 2 with PgQh. To achieve possibly better improvements, we would like to generalize the definitions of metric value, local girth, and multi-edge local girth under each SPR metric,3 and then generalize the MM-PEGA by applying certain generalized local girths and multi-edge local girths to Strategy 2.
Because the metric value of any shortest path/cycle under each SPR metric is as clear as that under a, defining the metric value, local girth, and multi-edge local girth under each SPR metric turns out to be a very intuitive and simple task with respect to the existing definitions under a (see Sections II-D and III-A). In particular, to keep to the original definitions of the local girth and multi-edge local girth, and meanwhile, to coincide with the empirical thinking that a larger local girth or multi-edge local girth implies a better quality of a node or an edge, the metric value corresponding to each SPR metric in {a, b, e, f} and {c, d} is made positive and negative, respectively. In this case, the notations in Section II-D and the multi-edge local girth in Definition 2 remain the same under each SPR metric. At this point, defining the metric value, local girth, and multi-edge local girth under a sequence of the SPR metrics of decreasing priority becomes very easy (see below).

V. CONCLUSIONS
The PEG algorithm constructs an edge in each stage to maximize vc’s local girth in real time. Thus, vc’s local girths, after more than one edge is added to the current TG setting, may not be maximized relative to that TG setting. To address this problem, we have defined the multi-edge local girth for referring to the maximum possible local girth that can be achieved after a certain number of edges are added to the current TG setting, and then proposed the MM-PEGA, which selects a CN whose multi-edge local girth is maximum in each stage. Certain properties of the multi-edge local girth have been analyzed, and a DFS-like algorithm has been proposed to calculate the multi-edge local girth. In addition, the SBM has been proposed to replace the BFS-like method so as to accelerate the MM-PEGA. Moreover, the MM-PEGA has been generalized for improving different PEG-like designs. The MM-PEGA is still locally optimal at each VN, but optimal in a less-local sense than the existing PEG-like designs. Increasing the edge-trial makes the MM-PEGA “see further” when selecting CNs, and is thus expected to positively affect

the cycle-structure and the error performance of resulting LDPC code. This expectation has been verified by simulations.
