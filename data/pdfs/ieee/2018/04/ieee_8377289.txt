Abstract—In this paper we optimize spatially coupled protographs for window decoding (WD) and arbitrary rate. Previous studies found that the belief propagation (BP) threshold of spatially coupled code ensembles achieves the maximum a posteriori (MAP) threshold of their underlying block code ensemble. This property requires a large coupling length L and thus the window decoder is considered to reduce latency and complexity of the decoding.
To approach the BP threshold fast in the size of the window W , it is well known that the code requires a special structure to avoid degree-1 variable nodes inside the window. We further require additional structure to construct a systematic code with low encoding complexity, which unfortunately forces degree-1 variable nodes inside the window.
Thus, we formulate an optimization problem to maximize the WD threshold and solve it by applying a differential evolution (DE) based algorithm. Compared to the regular protographs obtained by edge spreading, our optimized irregular protographs show a significant improvement in terms of WD threshold and finite length performance for small window sizes, hence leads to small decoding latency and complexity. Furthermore, for high rates our codes can compete with the highly optimized LDPC block codes from the WiMAX standard.
I. INTRODUCTION
Spatially coupled low density parity check (SC-LDPC) codes, introduced as LDPC convolutionial codes in [1], are constructed by connecting L independent LDPC codes into a coupled chain, which is equivalent to introducing memory into the code. It was shown in [2] that the belief propagation (BP) threshold of spatially coupled code ensembles approaches the maximum a posteriori (MAP) threshold of their underlying block code ensemble if L → ∞, which is called threshold saturation. Although these analytical proofs do not apply on SC-LDPC codes based on protographs, Mitchell et al. conjectured by numerical studies that threshold saturation is also true for protograph ensembles [3]. Note that the MAP threshold of block code ensembles approaches capacity by increasing the node degrees. Similarly spatially coupled code ensembles approach capacity under BP by increasing the node degrees and the code memory.
Since a large coupling length L is required to observe the threshold saturation of SC-LDPC codes, latency and decoding complexity becomes large, when BP is performed on the complete coupled chain at once. Hence, the sliding window decoder was proposed in [4] that makes the latency and complexity independent of L. Iyengar et al. have proven that the window decoding (WD) threshold approaches the BP threshold exponentially in size of the window W [5], i.e., the

WD threshold approaches capacity for W L. Moreover, it was shown in [4] that the careful construction of a protograph drastically reduces the required window size W to approach the BP threshold.
These facts inspired us to search for protographs of any rate that yield optimal performance under WD with practical constraints, such as low latency and low encoding and decoding complexity.
The paper is structured as follows. We start in Section II with an introduction to SC-LDPC codes based on protographs and discuss the window decoder and the WD threshold. In Section III we state an optimization problem to maximize the WD threshold of protograph based spatially coupled codes under practical constraints. The optimization results will be discussed in Section IV and subsequently we examine the finite length performance of our optimized protographs in Section V. Finally we conclude this work in Section VI.
For simplicity and the convenience of the reader, we will restrict the discussion in the asymptotic domain to the binary erasure channel (BEC). However, the results for the binary input additive white Gaussian noise channel (BI-AWGNC) are available and coincide with the presented results of the BEC. Due to more practical relevance, the finite length performance is evaluated on the BI-AWGNC.
II. PROTOGRAPHS FOR SPATIALLY COUPLED CODES AND WINDOW DECODING
In this section we want to introduce protograph based SCLDPC codes and discuss the window decoder and the iterative decoding threshold as performance metric for protograph ensembles.
A. Spatially Coupled Protographs
A popular way of designing LDPC codes is to use a protograph description of the code ensemble. The protograph of an LDPC block code is a small bipartite graph with nc check nodes and nv variable nodes, where multiple edges between a variable and a check node are permissible. The design rate of the code ensemble represented by the protograph is defined as R = 1 − nc/nv. C. Window Decoding Threshold

As a performance metric for protographs, we consider the
iterative decoding threshold that measures the average ensemble performance in the limit of N → ∞ and Imax → ∞. The BP threshold, denoted as BP(B), is defined as the supremum of all channel erasure rates for which the error probability of all variable nodes converges to zero, if Imax → ∞. In the case of WD, decoding is successful as long as the error
probability of the target values becomes smaller than a prior defined small value δ [4]. Thus, the WD threshold is denoted as WD(B, W, δ). We obtain the WD thresholds by applying the Protograph-EXIT analysis [8] in the same manner as described
in [4].

III. PROTOGRAPH OPTIMIZATION FOR WD

In this section, we state the problem of optimizing protographs for SC-LDPC codes to maximize the WD threshold and minimize the average variable node degree. Since this results in a multi-objective optimization problem, we need to discuss the definition of optimality in multi-objective optimization. Furthermore, we briefly describe a differential evolution (DE) based algorithm to find solutions to our optimization problem.

A. Optimization Problem

In the optimization problem (4), we wish to maximize the WD threshold and minimize dv with respect to B for given W , δ, ms, nv, nc and bmax. The minimization of dv is required, since only maximizing the WD threshold yields protographs with very large dv, what results into large decoding complexity (2).

C. Optimization Algorithm

Now arises the question of how to solve this problem. The objective function WD is some arbitrary non-differentiable function without a compact mathematical description. Moreover, the threshold is highly non-linear and has many local optima. Since we simultaneously want to minimize dv, the

problem is also multi-objective. A good choice for such complex optimization problems are evolutionary algorithms (EA). These seemingly simple metaheuristic algorithms can provide good solutions to all kind of often complex optimization problems. However, there is no guarantee that the optimal solution is found and in general it is not possible to assess the found solutions in terms of optimality. A method to obtain the optimal solution would be brute-force, but since the search space is increasing exponentially with the size of the base matrix, brute-force is not feasible.
Hence, we decided to apply an EA based on DE [9] to find approximate solutions to our problem. We will briefly describe the concepts and algorithms we used to adapt DE to our optimization problem. Details can be found in the cited publications.
DE was developed in 1995 by Storn and Price and soon rose to prominence, due to its simple nature and competitive results. It utilizes a population of NP solutions that is initialized by a uniform distribution over the whole search space. By iteratively applying mutation, crossover and selection DE produces new generations and explores the search space. The result of the algorithm is then the best found solution after Gmax generations.
The performance and convergence speed of DE heavily depends on the chosen control parameters that are optimized by massive trial and error experiments. A recent extension to DE is the success-history based adaptive differential evolution (SHADE) algorithm [10] that has shown to be highly competitive to other state-of-the-art evolutionary algorithms [11]. SHADE implements a control parameter adaption by maintaining a memory of size H that saves control parameters that have performed well in the recent past. New control parameters are sampled from the memory. The remaining control parameters are NP and H that must be obtained by experiments. We found NP = H = 50 to be a good choice.
The fact that DE, and thus SHADE as well, is designed for unconstrained single-objective optimization problems on the real vector space, imposes problems when we want to apply it on our optimization problem. At first, we do not operate on the real space, since a base matrix consists of integer entries. However, the only time where DE will leave the integer domain is during mutation. Hence, we simply round the mutation results to the nearest integer values.
To solve a constrained optimization problem, the algorithm must master two tasks. It must find the feasible region inside the search space and it must find the optimal solution inside the feasible region. A common way to handle this problem is to introduce a penalty function that gives an extra cost to the objective value of a solution, depending on how strong the solution violates the constraints [11]. Fortunately, our constraints are very easy to meet and we can produce a feasible initial population within a few random guesses. This makes a penalty function obsolete by only allowing feasible solutions to enter the population of the next generation.

new edges are spent on the increased memory. The existing

number of edges must be distributed over a larger memory,

what results in weaker coupling. Consequently, to maintain
the performance, dv must be increased if ms is increased. This observation is consistent with the results from [2]. If dv and W must be small for practical reasons, increasing the memory beyond ms = 3 is not beneficial.

VI. CONCLUSION
We formulated a multi-objective optimization problem, to optimize spatially coupled protographs of any rate and proposed a DE based algorithm to solve this problem in terms of Pareto optimality. The WD threshold is maximized while minimizing the average variable node degree, to keep the decoding

complexity low. Solving this problem yields protographs with
much better WD thresholds than the protographs proposed for
WD in [4]. Furthermore, our optimized protographs yield a better finite length performance. For R = 3/4, our codes can compete with the finite length performance of the WiMAX
block codes under similar decoding complexity but we observe a performance gap of 0.2dB for R = 1/2, although the WiMAX codes are QC.
This indicates that the excellent thresholds of our optimized
protographs lead to good finite length performance. Moreover,
we see from the comparison to the WiMAX codes, which are
highly optimized with respect to the finite length performance,
that further investigation is required to compete for all rates
with the best known block codes.
