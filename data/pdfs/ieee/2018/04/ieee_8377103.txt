Implementation Aspects of a Pipeline ADMM-based LP Decoding of LDPC Convolutional Codes
Hayfa BEN THAMEUR1, Bertrand LE GAL2, Nadia KHOUJA1, Fethi TLILI1, Christophe JEGO2 1 SUPCOM, GRESCOM Lab, University of Carthage, Tunisia.
2 Bordeaux INP, CNRS IMS, UMR 5218, Bordeaux University, France.

Abstract‚ÄĒThe enforcement of the linear programming (LP) decoding was recently extended to LDPC convolutional codes (LDPC-CC). It was demonstrated that their convolutional structure suites the message-passing based representation of the LP problem, thanks to the application of the alternating directions method of multipliers (ADMM). In this paper, a modified formulation of the ADMM-LP problem is introduced for pipeline decoding of LDPC-CCs based on the layered schedule. Moreover, an assessment of the fixed-point format required for hardware implementation is provided to evaluate the complexity of the underlying decoding algorithm. Simulations show that an 8-bit quantization scheme yields negligible error correction performances loss of about 0.05 dB regarding the floatingprecision ADMM based LDPC-CC decoder. Further, an algorithmic optimization of the Euclidean projection skipping technique, described in [1] [2], is proposed. This improved version enables to reduce the computational complexity of the decoding process without memory penalty contrary to the original formulation.
Index Terms‚ÄĒ802.16m, LDPC-CC, LP, ADMM, pipeline architecture, layered scheduling, fixed-point, hardware complexity.
I. INTRODUCTION
During the last decade, LDPC-CCs have attracted a great interest in the forward error correction area. They are considered as strong competitors to convolutional turbo codes as well as LDPC block codes (LDPC-BC) [3] for reliable decoding in many communication systems, such as wireless (WiMAX 802.16m), power-line (IEEE 1901) and optical communication systems [4]. Indeed, they provide good error correction performances and low latency features.
The hardware performances of LDPC-CCs have been consistently assessed for the belief-propagation (BP) algorithms such as the sum-product algorithm (SPA) and its Min-Sum (MS) approximation variants [4]. In the literature, LDPC-CC decoders based on a pipeline architecture were intensively studied. The interest of the pipeline decoding stems from its reduced latency and its scalability, making it suitable for realtime and high throughput demanding applications. In fact, implementation efforts are mainly devoted to the design of one processing unit that would be replicated as many times as the number of decoding iterations [4].
Recently, an LP technique was proposed by Feldman in [5] as an alternative to the conventional BP decoding. It yields in comparable error correction performance to the BP approaches [5] [6]. However, it suffers from high memory and computational complexities, making it impractical for hardware implementations. Later, an ADMM based version

of the original LP definition was introduced in [7], such that an iterative behavior of the LP decoding becames possible. Hence, this novel decoding technique is still viable for LDPCCCs (ADMM-CC) [8] [9].
Potentially, high hardware cost and time consuming projections are the major two practical issues of an ADMM decoder. In the literature, several strategies were investigated to address these issues. First, scheduling based optimizations were discussed in [9]. It has been shown that the horizontal layered (HL) schedule is the most memory efficient messages ordering approach for fast converging ADMM-CC decoder. Second, a hardware friendly algorithmic formulation of the Euclidean projection was proposed in [10]. Moreover, a skipping projection method was developed in [1] [2]. It consists in avoiding the projection operation whenever the evolution of the input vector is judged to be insignificant. Mainly, two manners were reported to verify if the projection can be ignored [1] [2]. Although the number of executed Euclidean projections is considerably reduced [1], extra computational and memory complexities are resulted. In this paper, a simplified formulation to estimate the projection input vector evolution without any memory overcost is provided.
So far, ADMM hardware decoders have been designed for flooding based LDPC-BCs. Yet, a 10-bit precision scheme was found to provide floating-point like error correction performances. In this work, we draw benefits from the layered schedule and the architectural specificity of pipeline decoding of LDPC-CCs to design a low hardware cost and scalable ADMM-CC decoder that is convenient for streaming data transmission scenarios. As a matter of fact, error correction performances close to the floating-point and 10-bit schemes, within 0.05 dB, are obtained by an 8-bit precision one.
The remainder of this paper is organized as follows: in Section II, conventional pipeline decoding of LDPC-CCs is described. In Section III, the modified ADMM-CC decoding approach is detailed. In Section IV, hardware complexity analysis is discussed and compared to BP. The proposed ADMM-CC decoder architecture is provided in Section V. Ultimately, experimental results are reported in Section VI.

C. Conventional Pipeline Decoding of LDPC-CCs
The diagonal form of the syndrome former HT of an LDPCCC induces a highly localized Tanner graph (Fig. 1). This is due to the following fact: consider two code blocks vt1 and vt2 , where t2 ‚Č• (t1+ ms+1), t1, t2 ‚ąą Z‚Č•1, since t2 ‚ąí t1 ‚Č• ms+1, vt1 and vt2 do not participate to any common parity check equation. The pipeline decoding of an LDPCCC exploits this structural property to split the related Tanner graph into independent windows of size (ms + 1) code blocks [3]. This allows the parallelization of P decoding iterations by assigning a separate processor to each decoding window as illustrated in Fig. 1 (ms = 2). Indeed, a processor is in charge of performing VNs and CNs updates, covered by its operating window, according to the chosen scheduling technique as well as the message-passing update rule.

III. PIPELINE ADMM -LP DECODING FOR LDPC-CCS

A. The ADMM-LP Decoding Problem

In [7], it was demonstrated that the LP problem for LDPC decoding stated in [5] can be reworded to match the ADMM template. Indeed, the code sequence v was relaxed to lie in the unit hypercube and auxiliary replica variables (z) were added. Although this decoding approach was applied to LDPC-BCs [7] and teminated LDPC-CCs [9], its conventional formulation does not straightforwardly fit the semi-infinite structure of LDPC-CCs.

B. Memory Complexity Analysis
A second useful hardware metric to compare studied LDPCCC decodings is the memory footprint (M) allocated to each processor. Let N and m be the number of Lv and CN-to-VN messages, respectively. For BP LDPC-CC decoders, M equals to (N + m). Notwithstanding, m additional memory elements are required by the ADMM-CC(-s) to store őĽ messages (M = N + 2m). While the ADMM-CC(+s) has the same memory footprint as ADMM-CC(-s), the one with the skipping projection proposed in [1] needs m further memory elements to store ancient w vectors (M = N + 3m).
In terms of memory access, it can be noticed from Table I that the total memory accesses within an ADMM-CC processor can be reduced by dc memory writes if the skipping projection condition is satisfied which inherently impacts the full pipeline ADMM-CC memory complexity. This fact makes it closer to the total memory accesses needed by the processor of a BP pipeline LDPC-CC decoder. Ultimately, memory access made by the ADMM-CC(+s) [1] remains the largest one amongst studied LDPC-CC decoders

VII. CONCLUSION
In this work, a first hardware-oriented ADMM formulation for layered based pipeline decoding of LDPC-CCs is investigated. Simulations, carried over the WiMAX 802.16m standard, show that an 8-bit quantization scheme is successfully achieved by the ADMM-CC-l2 approach with insignificant BER performances loss. Implementation costs are further reduced thanks to the proposed skipping projection algorithm. This achievement strengthen the assertion that the ADMM can be a competitive alternative to conventional BP for LDPC-CC decoders. It would also point to the avenue for future architectural optimizations of ADMM decoders to dynamically trade hardware complexity for throughput and latency constraints.
