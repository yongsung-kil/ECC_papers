A High-Throughput QC-LDPC Decoder for NearEarth Application

Shixian Li, Qichen Zhang, Yun Chen*, Xiaoyang Zeng
Fudan University State Key Laboratory of ASIC and system
Shanghai 200433, CHINA

Abstract—Turbo-like decoding message passing (TDMP) scheduling, which is suitable for quasi-cyclic low-density paritycheck (QC-LDPC) codes, can obtain better decoding performance and shorter convergence time than two-phase message passing (TPMP). Normalized min-sum algorithm (NMSA) is used to reduce complexity of soft message processing unit. In this paper, we propose a (8176, 7154) QC-LDPC decoder for Consultative Committee for Space Data Systems (CCSDS) standard. In order to avoid memory access conflicts, two cyclicshifted identity matrices forming a submatrix are processed individually. The decoder is synthesized by TSMC 65nm CMOS process and can achieve a throughput of 4.1Gb/s at clock frequency of 380MHz.
Keywords—component: Decoder architecture, quasi-cyclic lowdensity parity-check (QC-LDPC), turbo-decoding message-passing (TDMP), normalized min-sum algorithm (NMSA), cyclic-shifted identity matrix.
I. INTRODUCTION
Low-density parity-check (LDPC) codes achieve great interests for its near Shannon limit performance and ability of parallel decoding. LDPC codes were first proposed by Gallager in 1963 [1] but were ignored because VLSI implementation of LDPC codes were impractical in that period. Fortunately, LDPC codes were rediscovered by Mackay and Neal in 1996 [2] and now they are widely used in a number of storage systems and communication standards for various applications. First introduced by Mansour in 2003 [3], Quasi-cyclic LDPC (QC-LDPC) codes, which is composed of a group of submatrices based on a base matrix, attracted attention of many investigators. It can support multi-mode architecture because of its unique structure [4], and significantly reduce computational complexity of decoders.
Belief propagation (BP) algorithm is the primary decoding method for LDPC codes. Two-phase message passing (TPMP) and Turbo-like decoding message passing (TDMP) are two main message-passing schedules and both of them can be used for QC-LDPC decoding. In TPMP, variable nodes (VN) and check nodes (CN) are processed alternately and either of them can be processed fully parallel [5] or partially parallel [6]. TDMP is a kind of layered scheduling, in which parity check matrix is divided into several block rows and then are decoded sequentially.

Since 1990s, Consultative Committee for Space Data Systems (CCSDS) has proposed a set of standards, including (8176, 7154) LDPC code used for near-earth application. In [6], the decoder used sum-product algorithm (SPA) and partially parallel TPMP scheduling and was implemented in FPGA. Each submatrix in parity check matrix of (8176, 7154) code is superimposition of two cyclic-shifted identity matrices. Therefore, information from two check nodes will try to update one variable node simultaneously, leading to memory access conflicts in TDMP. A general architecture for QC-LDPC codes with high circulant weight was proposed in [7] to resolve this problem. A decoder with low complexity and high throughput for (8176, 7154) code is proposed in this paper. The interconnection between variable nodes and check nodes is realized by barrel shifter and structure of message processing unit is simplified. By using NMSA and TDMP scheduling, this decoder can achieve 4.1Gb/s throughput at 8 iterations.

To save hardware resources, proposed decoder only process one submatrix (block) every clock cycle instead of a whole layer. Submatrices of every block row are serially scanned for twice. In each clock cycle of the first scan, prior messages of 511 variable nodes belonging to one block are calculated. New extrinsic messages are obtained as (2) at the end of the first scan. In the second scan, posterior messages are updated block by block.

The ultimate algorithm only needs addition, shift and minimum search operations, which is beneficial for VLSI implementation.

III. VLSI ARCHITECTURE To balance hardware resources and performance, we use 6 bits to quantize the LLR information, including 1-bit sign, 3-bit integer and 2-bit fraction. Factor α is chosen as 0.75 [9] to reduce complexity as mentioned.

In proposed (8176, 7154) decoder, there are three types of memory: variable node memory (VNM), check node memory (CNM) and temporary registers (TR). Posterior messages are stored in VNM with 16 words and each word has 511×6 bits. Prior messages are stored in CNM with 2 words. For each row m, first minimum (5 bits), second minimum (5 bits), position of first minimum (5 bits, because the row weight is 32), and signs of all 32 Lmn which are associated with cm (32 bits) are stored for memory efficiency. Thus each CNM word has 511×47 bits to store information of one layer. TR is used to store information of new extrinsic messages for current layer. It’s the same with CNM in terms of organization but only has one word. For every row, three additional variables (implemented by registers) are needed to store temporary first minimum (Am), second minimum (Bm) and position of Am (POSAm).
Barrel shifter is used to realize connections between CN and VN. Shift values of the two cyclic-shifted identity matrices forming one submatrix are different so there are two barrel shifters. In the first scan, for each block, 511 posterior messages are loaded from VNM and permuted through the two barrel shifters. One word of CNM are loaded and converted to 1022 old extrinsic messages (2 extrinsic messages for each row, corresponding to two groups of shifted posterior messages) by recover unit (RU). For each row m, two extrinsic messages are subtracted from their corresponding posterior messages, generating two prior messages. Then a compare unit (CU) compares the generated two prior messages, Am and Bm and updates Am, Bm, and POSAm according to comparison results. Am and Bm are initialized to maximum value of 6-bit quantization before the first scan. After the first scan, all information of new extrinsic messages are stored in TR. V. CONCLUSION For QC-LDPC codes, TDMP is a primary decoding scheduling. But there are memory access conflicts for decoders of codes whose submatrices is superimposition of cyclicshifted identity matrices. In this paper, a low-complexity, highthroughput decoder for (8176, 7154) code is presented. This decoder uses NMSA and TDMP scheduling are changed to resolve memory access conflicts. The decoder achieves 4.1Gb/s throughput and occupies a core area of 2.3 mm2.
