Abstract—A new design framework based on the modified protograph extrinsic information transfer (PEXIT) algorithm is proposed for wireless body area networks (WBAN) communication systems. Using the proposed framework, a protograph LDPC code of rate 1/2 is designed for the WBAN channel which is statistically modelled as Rician distribution with K-factor obtained from the path loss model. The proposed protograph LDPC coded system achieves a significant coding gain of 17.5 dB over the uncoded system. This coding gain is much higher than the coding gain (6.1 dB) of irregular LDPC coded system in the additive white Gaussian noise (AWGN) channel reported previously for wireless sensor networks. This significant coding gain together with the low complexity encoder/decoder makes the protograph LDPC code an excellent candidate for WBAN communication systems where energy and complexity are among very demanding technical requirements.
I. INTRODUCTION
WBAN, a new generation of Wireless Sensor Networks (WSN), is a key solution to facilitate affordable and proactive health care services in the coming years [1], [2]. From the technical perspectives, WBAN has strict requirements on reliability, energy efficiency, and low complexity device [3]. To design a communication system that satisfies the aforementioned requirements is challenging, especially for the In VIVO WBAN communication channel since the capacity of power supply for the implant/in-body sensor is strictly limited, and the signal is highly attenuated when propagating via the human body [2]. Protograph LDPC codes, a subclass of LDPC codes, have yielded great attention from error control coding community recently [4]–[8] thanks to their excellent performance and low complexity encoder/decoder structures [4], [8]. In addition, experiment results show that LDPC codes are able to fulfill WSN performance requirements due to the high coding gain [9], [10], [11]. It was reported in [9] that an irregular LDPC code with the block length of 1024 bits can achieve the coding gain of 6.1 dB in the additive white Gaussian noise (AWGN) channel. Recently, A. Youssef et al. proposed a low complex LDPC hybrid decoding algorithm that requires the least number of iterations while maintaining outstanding bit error rate (BER) performance [10]. Motivated

by the aforementioned advantages of the LDPC code in the WBAN context, this paper proposes a framework to design protograph LDPC codes for WBAN communication channels.
A protograph [12] is a Tanner graph with a relatively small number of nodes. A protograph LDPC code is a larger derived graph constructed by applying a copy-and-permutation operation on a protograph. In the AWGN channels, one of the first protograph designs was proposed by Divsalar et al. [7]. In that work, several families of protograph codes whose minimum distance grows linearly with the information block length and iterative decoding thresholds approach the Shannon limit were designed. Then, a general design framework that results in better protograph codes was introduced by Nguyen et al. [4]. The framework in [4] can be used to design capacityapproaching rate-compatible protograph codes with a wide range of rates. Recently, Nguyen et al. [6] reported that the low-iteration optimized protograph LDPC code outperforms the off-the-shelf protograph LDPC code designed for a large number of iterations. For this reason, it is important and neccessary to have a framework to design protograph LDPC codes for WBAN communication systems where excellent BER performance, low device complexity encoder/decoder structures, and low delay are demanding technical requirements. Nevertheless, there has not existed research works on the protograph LDPC code design for WBAN in the literature. Hence, introducing a new design framework of the protograph LDPC code for WBAN communication systems is the core contribution of this paper.
The rest of the paper is organized as follow. Section II describes the WBAN communication system and the statistical model for the channel impulse response of the human body. The PEXIT algorithm for designing protograph LDPC codes is given Section III. Section IV proposes a framework to design an optimal protograph LDPC code. The BER performance is reported in Section V. Section VI concludes the paper.

III. MODIFIED PEXIT ALGORITHM FOR PROTOGRAPH LDPC CODES
The PEXIT algorithm was originally introduced by Liva and Chiani [16] for analyzing the protograph codes in the AWGN channel. Then, the algorithm was modified for fading channels with spatial diversity by Y. Fang et al. [17]. The modified PEXIT algorithm [17] is presented below.
A protograph G = (V, C, E) is defined by three sets: a set of variable nodes V, a set of check nodes C, and a set of edges E connecting check nodes and variable nodes. An edge em,n ∈ E if it connects a variable node vn to a check node cm. In addition, parallel edges are allowed. The protograph

code is an extended graph that is obtained by a ’copy-and-
permutation’ operation. By choosing the lifting factor, we can
achieve different code block lengths. Let B be the protomatrix of dimension M × N where M is
the number of check nodes and N is the number of variable nodes. The element bm,n of B indicates the number of edges connecting variable node vn to the check node cm. In order to present the modified PEXIT algorithm, we use following
notations:
• IAV (m, n) denotes the a priori mutual information (MI) between the input LLR value of vn on each of the bm,n edges and the corresponding coded bit vn.
• IAC (m, n) denotes the a priori MI between the input LLR value of cm on each of the bm,n edges and corresponding coded bit vn.
• IEV (m, n) denote the extrinsic MI between the LLR value sent by vn to cm and corresponding coded bit vn.
• IEC (m, n) denotes the extrinsic MI between the LLR value sent by cm to vn and corresponding coded bit vn.
• IAP P (n) denotes the a posteriori MI between the a posteriori LLR value of vn and the corresponding coded bit vn.
In addition, the MI function, J(σch), between a coded bit and its corresponding channel LLR value is calculated by the
following formula [18]

J (σch) = 1 − E [log2 [1 − exp (−u)]] .

(2)

More details about the approximation calculation of J(σch) and its inverse function, readers refer to [18].

V. EXPERIMENTS AND RESULTS

Based on the protomatrix in (15), the protograph LDPC code is constructed by copy-and-permutation operation, known as protograph lifting process. The lifting process is implemented in two steps. In the first lifting step, the protograph is lifted by factor of 4 using progressive edge growth (PEG) algorithm [20] in order to remove all multiple parallel edges. The second lifting step is determined according to the chosen information block length. In this experiment, with protograph having 3 check nodes and 6 variable nodes, the second lifting step is 86, making the information block length of 1032 bits. In the second lifting, the final LDPC code is constructed using PEG algorithm to determine a circulant permutation of each edge class to avoid short-length cycles occurred within a designed information block length. The decoder with a standard message passing algorithm has the maximum 20 iterations.

It is worth mentioning that AR3A was shown to be the best code over the regular and irregular LDPC codes [19], [21]. The simulation result indicates the performance improvement of the proposed code over the AR3A code. The improvement gap is about 0.1 dB which is less than the gap of the iterative decoding threshold 0.45 dB since the block length in simulation is 1032 bits.
Fig. 3 shows the BER performance of the proposed protograph LDPC coded system and the uncoded system. At the BER level of 10−4, the coding gain of the proposed protograph LDPC coded system is about 17.5 dB. This coding gain is much higher than the coding gain (6.1 dB) of the irregular LDPC coded system under the AWGN channel reported earlier [9] for wireless sensor networks. The significant coding gain of the proposed protograph LDPC code together with the low complexity encoder/decoder and advancement in effective implementation of LDPC decoder [10] strongly suggests that the protograph LDPC code is a good channel coding candidate to design an energy-efficient WBAN communication system.
VI. CONCLUSION
We introduced a new framework to design optimal protograph LDPC codes for WBAN communication systems using the modified PEXIT algorithm. A protograph code of rate 1/2 is proposed for the WBAN channel of the Anterior region. The proposed protograph coded system offers a high coding gain over the uncoded system for the WBAN channel. This significant coding gain is important for WBAN communication systems to meet the technical requirements, and thus the protograph LDPC code is a good channel coding choice for an energy-efficient WBAN communication system.
