Generalized Globally-Coupled Low-Density Parity-Check Codes

∗National Chiao Tung University, Hsinchu, Taiwan † University of California, Davis, CA95616 USA

Abstract—In this paper, two globally-coupled low-density parity check (GC-LDPC) code constructing methods are presented. The first is a Reed-Solomon-like GC-LDPC code that eases the design parameter searching in algebraic code constructions. The parameters such as the finite field, the associated prime factors, or the number of local codes can be more easily determined. The directly masking approach, the second proposed scheme, facilitates flexible structure designs of the parity check matrices. The two approaches can be regarded as generalizations of the original GC-LDPC codes. The design examples and the simulation results show the feasibility of the proposed techniques. In addition to code constructions, some applications and usage scenarios of GC-LDPC codes are addressed as well.
Index Terms—Globally-coupled low-density parity check code, structural LDPC code
I. INTRODUCTION
Globally-coupled low-density parity check (GC-LDPC) codes were first introduced in [1]. The parity check matrix of a GC-LDPC code comprises several disjoint copies, which are referred to as the local codes, and one globally-connecting part that connects all the local codes as one larger block lengthed codeword. Fig. 1 to Fig. 3 present a few parity check matrix structures of GC-LDPC codes. The check nodes in the globally-connecting part at the bottom connect all the variable nodes to provide high connectivity for each coded bit, which is an important characteristic of faster convergence in iterative decoding [2]. Code construction methods can be found in [1], [2], [3], [4]. These GC-LDPC codes are constructed on finite fields and have quasi-cyclic (QC) formats [5].The finite field chosen for code construction implicitly determines the structure of the parity check matrix, for example, the column/row degrees, the block length, and the submatrix size.
In order to have more flexible choices of parameters in code construction, a few extensions of GC-LDPC codes are proposed in this paper. A GC-LDPC code can be regarded as an LDPC code comprising several shorter block-lengthed LDPC subcodes, whether they are joint or disjoint. Based on the proposed code construction schemes, various GCLDPC structures can be carried out with ease. The rest of the paper is organized as follows. II. GLOBALLY-COUPLED LOW-DENSITY PARITY CHECK CODE REVIEWS
GC-LDPC code constructed by algebraic methods were first presented in [1][2][3][4]. These codes are QC-LDPC codes [5] and have the cascaded type parity check matrix of Fig.1. The construction can be summarized as
• Base matrix generation: Construct a matrix B1 over a finite field GF(q).
• Matrix dispersion: Replace each element in B1 with a circulant permutation matrix (CPM) or an all-zero matrix. A cyclically-shifted identity matrix is one common realization. The dispersed matrix becomes a low-density matrix.
• Submatrix masking (optional): Replace some of the dispersed circulant permutation matrices by an all-zero matrix. This is equivalent to remove some of the edges in the Tanner graph so as to potentially remove some short cycles.
The first GC-LDPC code proposed in [1] was constructed by matrix partitioning and superposition. Subsequently, a base matrix defined by two Reed-Soloman codes’ parity check matrices was introduced in [3]. These two base matrix generation methods are briefly reviewed in the following.

III. GENERALIZED GLOBALLY-COUPLED LOW-DENSITY PARITY-CHECK CODES
In a wide sense, all the local codes of a GC-LDPC code can be designed differently and optimized for different application scenarios. For instance, the block length and code rate of each local codes can be different. This can be easily carried out by defining different row numbers and shortening some columns in the corresponding GC-LDPC codes’ base matrix.
In this section, some code construction techniques are presented to generalize GC-LDPC code design so as to provide more flexibilities in practical applications.

B. Globally-Coupled Low-Density Parity-Check Codes Constructed by Directly Masking
From the matrix partitioning approach mentioned in Section II-A, a GC-LDPC code can be regarded as a QC-LDPC code with a special masking pattern. That means, many off-theshelf good LDPC code designs can be modified to GC-LDPC codes. The local and global structures are determined by a GC-masking pattern which is element-wise multiplied with the unmask LDPC codes.
The directly masking approach implies that one can modify an LDPC code to a GC-LDPC code, whether from its base matrix if it has a QC-LDPC structure, or from its binary parity check matrix. Direct masking does not reduce the girth of the code. Moreover, direct masking facilitates variations of the local-global structures. IV. APPLICATIONS
The highly structural parity check matrix of GC-LDPC codes preserves the low decoding complexity of smaller block-lengthed codes while the error correcting capability is comparable to codes of large block lengths. To leverage the locally disjoint and globally coupled parity check matrix structures, the decoding can be arranged in a hierarchically two-phase manner. In the local decoding phase, all the local codes are decoded independently. The information of the entire codeword is exchanged only via the global decoding phase. There is a great degree of freedom in designing the twophase decoding procedure for different applications. In the following, a few applications for GC-LDPC codes with twophase decoding are briefly introduced.
A. NAND Flash Memories with Unequal Page Bit Error Rate
For multi-level cell NAND flash memories, the data raw bit error rates among different physical locations can be very different. Encoding can be arranged such that data corresponding to physical locations of similar error rates are encoded to the same local codeword. The local codes corresponding to better channel conditions will converge first, and provides constructive probabilistic information to other local decoders. Moreover, it is possible to design a locally-joint GC-LDPC code to further increase the connectivity so as to improve the decoding convergence when the equivalent SNRs of each local codes are extremely different.

C. Transmission with Incremental Redundancy
In a communication system with time-varying channel environments, error correction codes with multiple code rates are necessary for efficient data transmission. Some systems are designed with an aggressive rate adaptation procedure such that a highest code rate for data transmission is set as the default scheme. Redundant information is transmitted by request. More parity bits are requested only when the receiver fails to recover the received data. This incremental redundancy in transmission can enhance the system efficiency if arranged properly. LDPC codes designed for this scenario requires a flexible structure in order to adapt to various code rates and block lengths. The structural parity check matrix of GC-LDPC codes can be leveraged to fulfill this requirement. As it is shown in Fig.7, the local and globally-connecting parts of the parity check matrix can be easily stacked to serve the redundancy requirement. The stacked local codes can have distinct code rates and lengths. For each appended local and global parts, the previously applied GC-LDPC parity check matrix will be regarded as a merged local part in the newly extended GC-LDPC code.
D. MIMO and MU-MIMO Transmission
In multi-input multi-output (MIMO) communication, the data transmitted via different (spacial) streams undergo different channel conditions. It is possible to decompose a MIMO channel into multiple parallel channels when the channel state information is available. This allows assigning different transmission rates for each link. That is, the modulation

scheme and the code rate are different. GC-LDPC code can be designed with varying local code rates and lengths to fulfill this purpose. By default, each stream, which can be mapped to distinct users, decodes the data independently. The globallyconnecting part in the parity check matrix provides an option for collaboratively joint decoding among the users for further performance enhancement.
V. SUMMARY
In this work, extensions of GC-LDPC code construction schemes are presented. These generalized GC-LDPC codes increase the degrees of freedom in defining the design parameters. The proposed Reed-Solomon-like GC-LDPC codes avoid searching over different finite fields for parameters. For instance, the maximum number of local codes and the CPM sizes can be easily determined to comply with specific block lengths and code rates. Moreover, the proposed directly masking approach allows one to design various GC-LDPC parity check matrix structures. Deriving GC-LDPC codes from many off-the-shelf well-designed LDPC codes are possible. The applications presented in Section IV explore different usage scenarios that best leverage the structures of the codewords. In addition to the design flexibility, our simulation results of the presented examples show that the proposed GC-LDPC codes have comparable BLERs as compared to conventional GCLDPC codes and QC-LDPC codes.
