Performance of Self-corrected APP-based Decoding Algorithm for Decoders with Quantized Input
Aleksei Kharin, Igor Volkov, Aleksei Ovinnikov, Evgeny Likhobabin and Vladimir Vityazev, Member, IEEE

Abstract—In this paper we explore the impact of self-correction (SC) modification on performance of a posteriori probability (APP) based decoding algorithm for LDPC codes. Influence of self-correction technique was investigated for decoders with 3, 2 and 1 bit input quantization.
It was shown that performance of decoders with quantized input can be significantly improved in comparison with regular APP-based decoding algorithm, and in some cases with Min-Sum algorithm. Convergence of algorithm is almost the same as for Min-Sum decoding.
Performance and convergence results are presented for two CCSDS (8176, 7156) and IEEE 802.3an (2048, 1723) LDPC codes for APP-based, SC APP-based and Min-Sum decoding algorithms.
Index Terms—LDPC, APP-based decoding, self-correction.
I. INTRODUCTION
The low-density parity check (LDPC) codes have been known for more then 50 years since they were proposed by R. Gallager [1] in 1963 and then rediscovered by MacKay and Neal [2] in 1990’s. Nowadays these codes are widely used in numerous communication standards like DVB-T2, DVB-S2, DVB-C2, WiFi, WiMax, IEEE 802.3an and also find place in other fields, including for example flash memory error correction techniques. Despite their long history decoding of LDPC codes still cause implementation difficulties due to sophisticated iterative belief-propagation (BP) based algorithm. That’s why a lot of simplified algorithms were proposed lately. They don’t provide near the Shannon limit performance as BP algorithm does, but allow to implement LDPC decoders under various power and computational restrictions. Due to these restrictions it’s very usual to have fixed point number representation inside decoder and quantized data at decoder’s input. In these conditions task of choosing the best decoding algorithm in terms of performance and computational costs is not simple taking into account big number of available modifications.

II. DEFINITIONS AND NOTATIONS
Any LDPC code can be represented as a check matrix which contains mostly zeros and only a small number of ones. A regular (N, K) LDPC code is represented as the matrix H which includes M ≥ N − K rows and N columns. We refer to the M rows of H as check nodes, and to all the elements of a LDPC codeword as variable nodes.
Also define N (i) the set of ones in i-th column, M (j) the set of ones in j-th row of matrix H. N (i) − {j} and M (j) − {i} are the same sets, but without j-th and i-th positions respectively.
Assume, that the binary LDPC code C is used for forward error correction over AWGN channel, with BPSK modulation. Let represent the codeword of C as x = [xi](xi ∈ {0, 1}), and s(x) = s = [si] as a corresponding transmitted sequence, then the received sequence y is equal to y = s + n, with yi = si + ni(1 ≥ i ≥ N ), where elements ni of n are statistically independent Gaussian random variables with zero mean and variance N0/2.
III. DECODING ALGORITHMS
During the research several LDPC decoding algorithms were used: Min-Sum, APP-based (APP) and self-corrected APP-based decoding algorithms. All of them are based on the logarithmic likelihood ratio (LLR) BP decoding algorithm [1]. Further each used algorithm is considered in details.
As we previously mentioned, Min-Sum decoding algorithm can be regarded as a simplification of LLR BP decoding algorithm [3]. This simplification is based on a simplification of check nodes updating step: in the Min-Sum algorithm there is no such complex operation as logarithm calculation. Its replaced by finding the minimum value of Li operation and multiplying sequence.

Generally, the check nodes updating step in APP-based algorithm is the same as that in Min-Sum algorithm, but there is one small difference: in APP-based computation performed only for one message for each variable node. Schematically, this detail is depicted in Figure 1.
Despite of the fact that only one message from variable node is calculated in APP-based algorithm, it’s possible to use the self-correction technique for this algorithm [7]. With this technique considered algorithm (SC APP-based) theoretically provide better performance than common APP-based [7]. The main idea of this modification is to set to zero all variable

nodes messages which change their sign between two consecutive iterations, as it is shown in Algorithm 1.
As it was mentioned in [7] several types of message erasing in SC APP-based algorithm are possible: message erasing (SCM), reliability erasing (SC-R) and message with reliability erasing (SC-MR). In this paper, we used the last type - SC-MR erasing which provides the best error correction performance and the minimum average number of decoding iterations.

IV. SIMULATION RESULTS
For our simulations we used the model from [8]. The main advantage of this model is high flexibility, it allows to run the developed model on various platforms and OpenCL [9] devices for different codes with minimal modifications.
Simulations were invoked for number of codes, but due to restricted size of publication results for only two codes are presented. The first is a quasi-cyclic EG N = 8176, M = 7156 CCSDS LDPC code [10] and the second is a algebraic constructed N = 2048, M = 1723 IEEE 802.3an standard LDPC code [11]. In both cases maximum number of iteration is 50.

It can be seen, that decoding performance for SC APP-based decoding algorithm is better than for APP-based decoding for both codes. Thus for unquantized implementation gain performance is about 0.1-0.2 dB, for 3 bit quantazation it grows up to 0.2-0.4 dB, for 2 bit - to 0.5-0.7 dB, and for hard 1 bit case - to 0.7-1 dB. Loss in performance to MinSum decoding for CCSDS code for unquantized and 3 bit implementations is about 0.3 dB, for 2 bit loss is only 0.1 dB, and finally for hard 1 bit quantazation SC APP-based algorithm outperform Min-Sum algorithm for 1.5 dB. For IEEE code SC APP-based decoding algorithm outperform Min-Sum algorithm for all these cases. In Figure 8 distribution of iterations on which decoding was successfully finished is shown. Simulation was made for CCSDS code on Eb/N0 = 5.5 dB. It can be seen that distribution for SC APP-based decoding algorithm have lower tail that for APP-based and even for Min-Sum decoding algorithms. That means that in equal conditions SC APP algorithm converges faster than APP and Min-Sum.
