Real-time FPGA-based Rate Adaptive LDPC Coding for Data Center Networks and PONs
Xiaole Sun, Mingwei Yang, and Ivan B. Djordjevic
Department of Electrical and Computer Engineering, University of Arizona, 1230 E. Speedway Blvd, Tucson, AZ 85721, USA, e-mail: sunxiaole@email.arizona.edu

Abstract We experimentally demonstrate an FPGA-based rate adaptive LDPC coding scheme suitable for data center networks and PONs. The proposed adaptive coding scheme can dynamically determine channel condition and adapt the code rate accordingly in frame-by-frame fashion.

Introduction Driven by the cloud computing, storage, and big data, modern enterprises benefit from data center (DC) that holds tens to hundreds of thousands of servers. To be profitable, the DC network (DCN) is required to be ultra-high bandwidth, low latency, flexible, scalable, and energy efficient. The employment of optical fiber in the DCN brings tremendous communication capacities and large scale. DCN is bandwidth hungry and heavy energy consumption. However, the average server utilization can be low and range from 5% to 20%1. Therefore, the DCN should be optimized by employing dynamically configuration approaches.
The low-density parity-check (LDPC) codes are widely employed in the transport layer to provide the bandwidth close to the channel capacity. Compared to using the fixed codes, which are designed for the worst-case scenario, assigning proper LDPC codes to different channel condition will bring more flexibility to improve the channel bandwidth, latency, and energy consumption2.
In this paper, we propose an FPGA-based rate adaptive FEC scheme suitable for data center and PON applications with quasi-cyclic (QC) LDPC codes of rates from 0.7 to 0.8, and evaluate this adaptive coding scheme in an arrayed waveguide grating (AWG)-based DCN3.

For given values of column weight Ȗ, row weight ȡ, block size b, and desired girth g, the shift parameters pi,j can be obtained such that the Theorem 2.1 in the reference4 is satisfied for the given girth. By combining with the concept of cyclic-invariant difference set (CIDS)5 in construction searching process, the parity-check matrix H can be represented by a set of shift values (0, p1, p2, …, pȡ-1). The most promising advantage of such codes is that it can be efficiently implemented in hardware, e.g. field-programmable gate array (FPGA). Highgirth QC-LDPC codes can provide excellent waterfall performance due to large minimum distance. Tanner’s bound shows the fact that minimum distance of an LDPC code is exponentially proportional to the value of girth. Furthermore, high-girth QC-LDPC also provides good error floor performance6. To achieve rate adaptation, shortening technique is adopted. The shortening operation is simply eliminating several block columns from the parent paritycheck matrix. Therefore, the shortened code is observed with early error floor since the girth of the shortened code does not increase. To address this problem, instead of designing the high rate parent code first, we first design a high-girth low rate code with small row weights, and then extend it to a higher rate code with large row weights. In this demonstration, we first design a girth-10 (3,10) QC-LDPC and append 5 block columns to get a girth-8 (3,15) QC-LDPC code. Column weight of three is widely adopted in optical communication systems due to the low complexity requirement. To make encoding suitable for hardware implementation, we use the systematic-circulant form of the generator matrix instead of the systematic form by Gaussian elimination. For details on how the systematic-circulant form of generator matrix can be obtained from the parity-check matrix, we refer interested readers to the reference7. In min-sum algorithm, the variable node processor (VNP) performs the sum operation, while the check node processor (CNP) performs the finding minimum part. The output of CNP will update a posteriori probability (APP) memory and the updated information will be used in the next iteration. To address the low latency requirement, we limit the maximum iteration number to 12. The FPGA emulation results of our adaptive code design are shown in Fig. 2. No error floor has been observed down to BER of 10-12.

Experimental verification

To further verify the proposed rate adaptive coding scheme in a DCN, a simplified AWGbased DCN is set up as shown in Fig.3. At the transmitter side, a tunable continuous wave (CW) laser with 10 kHz linewidth on a 50 GHz grid (C band) is sent to the Mach-Zehnder modulator at 1550 nm. PAM-2 (on-off keying) modulation is adopted in our system. The modulated signal is boosted by an erbiumdoped fiber amplifier (EDFA) with 6 dB noise figure. Then the signal is split into three branches. Each branch is connected to different input ports of the AWG- based DCN for data transmission, signal monitoring, and crosstalk simulation. More specifically, ports # (16, 17, 18) are used as inputs of the AWG. The signals at port 16 and 18 are used as the crosstalk channels. Optical delay line (DL) is inserted between the AWG and coupler to interleave the signals. At the receiver side, the target output port #1 at AWG is coupled with an amplified spontaneous emission (ASE) noise source, in which we can tune the noise level for testing the rate adaptive coding performance. Then the signal passes through an optical tunable filter (TF) to select the target wavelength at 1550nm. A photodetector with transimpedance amplifier converts the optical signal to the baseband. An

ADC/DAC FPGA mezzanine card (FMC) module from Analog Devices is used to transmit and receive the electrical RF signal. The ADFMCDAQ2-EBZ module is comprised of a 14bit, 1.0 GSa/s, JESD204B ADC, and a 16-bit, 2.8 GSa/s, JESD204B DAC. In the FPGA, the transmitted signal is up sampled by four times. We ensemble 4 parallel signals through the JESD204B interface and transmit at 1.0 GSa/s through the DAC. In this case, the data throughput is still 1.0 Gb/s. At the receiver side, ADC samples the signal at 1.0 GSa/s, and then the digital signal is down sampled by 4 times before being sent to the decoder. Both encoder and decoder operate on 250 MHz clock running frequency. (The throughput can be improved by the external clocking approach.) The code BER performance is shown in Fig. 4. There is no error floor observed at 10-9. With our rate adaptive coding scheme, we can correct up to 3.7102 raw BER by the shortening technique. When the raw BER exceeds 2102, the encoder will automatically switch to the 0.75 rate code. When the raw BER exceeds 2.9102, the encoder will change to the 0.7 rate code. Decoder will adapt accordingly with encoder. The raw BER calculation occurs right after the synchronization correlator finds a frame. Conclusions

An FPGA-based real-time rate adaptive FEC scheme has been proposed and verified in an AWG-based DCN. The LDPC encoder and decoder adapt the code rate dynamically based on the raw BER calculation. To maintain low post-FEC-BER, the adaptation has been applied on a frame-by-frame basis.
