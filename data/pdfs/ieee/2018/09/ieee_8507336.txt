QC-LDPC Codes with Fast Encoding for Error Control in NAND Flash Memories

Wojciech Sułek, Member, IEEE Silesian University of Technology
Gliwice, Poland Wojciech.Sulek@polsl.pl

Marcin Kucharczyk, Member, IEEE Silesian University of Technology Gliwice, Poland Marcin.Kucharczyk@polsl.pl

Abstract—In this paper we show the construction procedure for QC-LDPC codes with low encoding complexity and structure optimized for hard decision decoding and low precision soft decoding, which both are involved in a modern error correction control for NAND flash memories. The procedure is based on a computer search for codes with linear encodability, while error correction performance is optimized by reducing the number of weight-2 columns in the parity check matrix and optimizing structure of the code graph, particularly eliminating short cycles. We also show a simple method for code rate adjustment by extending the parity check matrix altogether with codeword shortening to enable more robust error correction when raw error rate of memory cells increases with the memory lifetime. The approach is validated via error rate Monte Carlo simulations with hard decisions and low precision soft decisions LDPC decoding, with a flash memory cell error model.
Index Terms—error correction codes, LDPC codes, QC-LDPC codes, LDPC encoding, NAND flash
I. INTRODUCTION
Strong error control coding (ECC) for Flash memories is becoming more important as the memory storage density increases. The commonly employed ECC in flash memories, BCH coding, is recently replaced by a modern Low-Density Parity-Check (LDPC) codes [1], characterized by an enhanced error correction performance and possibility to utilize soft decisions, which can be obtained by a finer-grained memory cell threshold voltage sensing.
A binary LDPC code (N, K) is defined by an M ×N parity check matrix H, where M = N − K, which is a sparse binary matrix. The number of nonzero elements in columns of H is known as a column weight distribution in H.
Methodical construction of LDPC codes is one of the major topics for many channel coding researchers. While the main objective of the research in this area is to provide possibly the best error correction performance for the specific use-case, the implementation issues typically constraint the design to some class of implementation oriented LDPC codes.
To be efficiently utilized in NAND flash controller, the LDPC code should satisfy the critical requirements, concerning the codeword length N , the code rate R = K/N , good error correction performance and possibility to implement high throughput hardware codec. Unlike to typical application

in telecommunications, where commonly soft decisions are available at the decoder input, the NAND flash memories sensing delivers hard decisions (bits) or soft decisions with strongly limited precision. This calls for a little different structure of H, because parity check matrices with relatively higher column weights tend to outperform their low-weight counterparts in a hard decision decoding scenario.
Currently, most of the LDPC codec designs are based on codes with block-circulant structure of H, which is a class of Quasi-Cyclic codes (QC-LDPC). Parity check matrix of a QCLDPC code is a grid of submatrices, which are either circulant permutation matrices or zero matrices. The block-circulant parity check matrix structure enables fast and efficient message routing in a decoder implementation. Additionally, the low complexity encoder implementation employing the classic Richardson-Urbanke method [2] requires dual-diagonal structure of H [3]. However, in a dual diagonal H, inevitably some columns are weight-2 columns, which is not necessarily a desired feature for hard decision decoding systems.
In this paper, we present a computer search algorithm for QC-LDPC codes encodable with a Richardson-Urbanke method as efficiently as a dual diagonal codes, but without the requirement of the dual diagonal structure, thus with reduced number of weight-2 columns. The algorithm optimizes structure of the code graph, particularly reducing existence of short cycles, which are known to strongly affect the error correction performance.
We also describe the rate adaptation method by the parity check matrix extension, which maintains the efficient encodability, altogether with code shortening. Such an adaptation scheme provides a more robust lower rate coding with unchanged block size, which can be used after some number of prgramming/erase (P/E) cycles in the NAND flash, when stronger error correction is required, at the cost of slightly decreased information bit rate.
The proposed code construction methods can be used for a variety of codeword lengths, code rates and blockcirculant sizes. In this paper, we present a case study design for a (2048, 1792) rate-0.875 code with submatrix size 64 and majority of columns being weight-4. For this code, we present block-circulant structure of the constructed parity check matrix, we present the extended lower-rate code and provide Monte-Carlo simulation results showing the error

II. ERROR CORRECTION IN NAND FLASH MEMORIES Gaussian distribution has been shown to be a decent approximation for flash cell threshold voltage distribution [4], [5]. Fig. 1 shows the normalized model of the threshold voltage distribution as a mixture of two Gaussian random variables comprising the SLC (Single Level Cell) model [5], in which the levels +1 and −1 represent two possible charge levels stored in the floating gate of a flash cell. Each time a cell is read, a single bit is obtained indicating whether the threshold voltage is above or below the applied word-line voltage. While single cell read provides a hard (1-bit) input to the LDPC decoder, reading the SLC cell multiple times with different word-line voltages produces soft input information. For example, three reads with word-line voltages −q, 0 and +q (Fig. 1) provide 4-level information in the form of probabilities delivered to the LDPC decoder [5]. While multiple reads increase the decoding performance, they also increase the sensing latency and decrease the throughput. Therefore, the most common strategy for the Flash controller is to apply the multiple reads only in a late state of a Flash lifetime, when after some number of P/E cycles, the hard decisions are not enough to guarantee error free decoding.

It has been well known that the LDPC decoding performance heavily depends on the column weight of the LDPC code parity check matrix. Irregular distributions of low-weight matrices, including weight-2 columns, have been shown to have stronger error correction capabilities than high-weight

LDPC codes in the case of soft decision decoding. However, a low column-weight LDPC code tends to have worse hard decision decoding performance than a high column-weight code [6]. This is also a case for soft decision input with low precision, as for 2-reads or 3-reads soft information from a NAND flash cell. Therefore, in an LDPC error correction system for NAND Flash, it is generally desirable to apply codes with high column-weight, particularly reduced number of weight-2 columns.

III. LDPC CODES
LDPC codes are a class of a linear block error correcting codes. Encoding process for a linear (N, K) code of rate R = K/N adds M = N − K redundant elements to the information vector u = {u1, u2, . . . , uK} forming the code vector (codeword) c = {c1, c2, . . . , cN }, where the vectors are over Galois field GF(q), particularly GF(2) for binary codes.
A binary (N, K) LDPC code is defined by a low density parity check matrix HM×N with GF(2) entries. In the decoder, a row vector ˆc of length N is recognized as a correct codeword if and only if it satisfies the parity check equation HˆcT = 0M×1, with GF(2) field arithmetic. When the check equation is not satisfied, then the error correction decoding is applied
by means of the iterative belief propagation algorithm [1].

When P is large, the base matrix Hb2 size is relatively small. For example, numerical results that will present in this paper are for a case code with I = 4 rows in the base matrix. For small base matrices, it is possible to search over all possible almost lower triangular matrices Hb2 and select solutions with Φb = 1. Among them, the one with the most favorable column degree distribution can be selected as a base for H2 construction. A. Rate Adaptation
In a complex error correction coding for NAND flash memory, a method for code rate adaptation is a necessity, because the NAND flash memory exhibits a diversified raw BER, depending heavily on P/E cycles number. It is desirable to apply a method, which offers an adapted rate basing on the mother code, decoded in the same hardware decoder and maintaining the low encoding complexity. At the same time, the codeword length should also be preserved to keep the number of codewords per flash pages intact. Then, the proposed rate adaptation is based on the following principles:
1) A QC-LDPC code (N, K), where K = (J − I)P , N = JP , is constructed with the method presented in the previous section. This should be a high rate (R = K/N = (J − I)/J) mother code utilized in the early flash lifetime.
2) The lower rate code is obtained by mother code matrix extension, with P additional columns and P additional rows, which corresponds to a single additional column and a single row in the base matrix Hb, giving the code rate R = (J − I)/(J + 1). The extension should be performed in a way maintaining the low complexity encoding structure.
3) To preserve the codeword length, a code block shortening of the first P bits of the codeword is applied. It means that at the encoder side, the first P bits in u are fixed to 0, and removed from the codeword after encoding. At the decoder side, the input decisions associated to these bits are also fixed to 0, with maximum reliability if soft decision decoding is used. The code shortening operation also provides enhanced performance, because of the relatively lower number of actual information bits in the codeword, which means the effective code rate is additionally lowered to R = (J − I − 1)/(J + 1).
The procedure for constructing the extension rows and columns is similar to the procedures provided above, but with a mother code matrix being a starting point for generation additional rows / columns. We can summarize it as follows:
Procedure: Extend the matrix H // Input: H = [H1 H2] – parity check matrix of a mother code // Output: He – matrix extended by P rows and columns Extend parity part of base matrix Hb2 into Heb2, with an additional row and column selected to keep Φb = 1 and maintain the almost lower triangular form. Extend information part of base matrix Hb1 into Heb1, with an additional row consisting of ones. Generate cyclic shift values associated with additional row and column in Heb2 in order to obtain Φ being a CPM. Generate cyclic shift values associated with an additional row in Heb1 in order to optimize code graph properties.

Create He from Heb1, Heb2 and the generated cyclic shifts. // End of procedure

VI. CONCLUSIONS
In this paper we have presented procedures for constructing LDPC codes that are: 1) efficiently decodable due to structured parity check matrix H, 2) efficiently encodable, without the need for a dual-diagonal form of H, 3) designed for hard decision and low precision soft decoding typical in a NAND flash applications, by employing relatively high column weight and optimizing the code graph structure. The construction is divided into 2 stages: parity part H2 construction with efficiently encodable structure and information part H1 with code graph optimization. We have also proposed an extension with shortening method for code rate adaptation. Numerical results for a case codes show a slight performance improvement of the proposed code over the dual diagonal code in the case of hard decision decoding as well as limited precision soft decision decoding.
