On the use of Probabilistic Parallel Bit-Flipping
decoder for the storage systems
Khoa Le, Fakhreddine Ghaffari, David Declercq. ETIS, UMR-8051, Universite´ Paris Seine, Universite´ de Cergy-Pontoise, ENSEA, CNRS, France,
{khoa.letrung, fakhreddine.ghaffari, declercq}@ensea.fr

Abstract—The soft-decision Low-Density Parity-Check (LDPC) decoders have been applied in several storage systems thanks to their powerful error correction capability. However, these systems may suffer a long read latency since the softdecision decoders require an intensive computations as well as a long sensing time for the soft-information before decoding. In this paper, we modify the recent-introduced Probabilistic Parallel Bit-Flipping (PPBF) LDPC decoder, to use on the storage systems in replacing the soft decision decoders, to improve the memory reading speed. The modified decoder is named Non-Syndrome Probabilistic Parallel Bit-Flipping (NS-PPBF). A special flipping mechanism is introduced such that the decoder can stop flipping without requiring the syndrome check results, which helps significantly improve the decoding frequency. We provide also the hardware architecture to implement NS-PPBF on the LDPC code used on the memory systems, which are usually very long block length with very high rate. The advantages of using NS-PPBF decoder in terms of error correction and decoding throughput are confirmed by the simulating decoding performance and the hardware synthesis.
I. INTRODUCTION
Thanks to the powerful error correction performance, LowDensity Parity-Check (LDPC) codes have been adopted not only to several communication standards such as IEEE 802.11n, 802.3an etc. [1][2], but also to the storage applications as in [3][4]. In the comparison to the traditional Bose-Chaudhuri-Hocquenghem (BCH) code which was widely used in storage systems, the LDPC code exhibits much better error correction capability and therefore, it is considered as a candidate to replace BCH code in these systems.
LDPC code is applied in the storage systems by the following principle. The user data (a sequence of bits) is encoded by adding some additional bits (the parity check bits) before being written into the memory. The coded data is called a codeword. The error may appear either when the codeword is being written/read into/out-of the memory or even during storing period in the memory, which leads to the fact that, the read codeword from memory is different to the one written. This read codeword (with errors) is called therefore the noisy codeword. An LDPC decoder is then applied to correct the errors possibly appeared in the noisy codeword. Note that, the reading latency of the memory includes the period of reading the stored codeword from the storing medium and the LDPC decoding delay.
The LDPC decoding is an iterative message-passing decoding process where it iteratively passes the messages between the two groups of processing units: the Variable Nodes (VNs) and the Check Nodes (CNs). One round (messages are sent from VNs to CNs, then CNs to VNs) is called one iteration

and the decoding process is stopped when a codeword is found or the maximum number of iterations allowed is reached. Depending on the message computations in the VNs and the CNs processing units, the LDPC decoding can be classified onto soft-decision (many bits are used to represent a message) or hard-decision algorithms (a binary message 0 or 1). As expected, the soft-information decoders provide an outstanding error correction performance thanks to more information provided from the soft messages. However, they require a long delay since the intensive computations are proceeded and an exhaustive hardware resource is required in the hardware realization. Furthermore, the soft-decision decoders require the soft information before starting the decoding process, which lengthens significantly the decoding delay and reduces the memory life time, as shown in [5]. On the contrary, the harddecision LDPC decoders are the lightweight and fast decoders thanks to the simple computations and short representation of messages. Moreover, when applied for error correction of memory system, the hard-decision decoders require only the hard information from the memory medium to start the decoding, which helps avoid the long latency in generating the soft-information.
We present in this paper a modification of the recently introduced hard-decision LDPC decoder, named as Probabilistic Parallel Bit-Flipping (PPBF) decoder [7]. The modified decoder is named Non-Syndrome Probabilistic Parallel BitFlipping (NS-PPBF). Compared to PPBF, in NS-PPBF we shorten the critical path (the longest data path which limits the maximum operation frequency of a design) by removing the syndrome check module from the decoder design. NSPPBF provides a very good error correction, being equivalent to that of the PPBF and approaching the soft-decision decoding performance, which will better protect the user data stored in the memory. Moreover, NS-PPBF is shown to be fast and lightweight hard-decision decoder, which will improve the memory reading speed.
The rest of the paper is organized as following. In section II, the LDPC decoding notations are recalled and the NS-PPBF decoding algorithm is presented. This section ends with an analysis of NS-PPBF decoder. The hardware architecture to realize NS-PPBF decoder is presented in section III. The NSPPBF is realized and implemented on the LDPC code used for storage applications on the Application-Specific Integrated Circuit (ASIC). The synthesis results as well as its decoding performance are shown in section IV. It is shown, as expected that, the NS-PPBF provides a very good error correction

II. THE NON-SYNDROME PROBABILISTIC PARALLEL BIT FLIPPING DECODING ALGORITHM
A. Notations
An LDPC code is defined by a sparse parity-check matrix H (M, N ), N > M . Each row represents a parity check function, computed by a CN, on the VNs represented by the columns. The VN vn (1 ≤ n ≤ N ) is checked by the CN cm (1 ≤ m ≤ M ) if the entry H(m, n) = 1 and they are called neighbors. LDPC code can also be represented by a bipartite graph called Tanner graph composing two groups of nodes, the CNs cm, (1 ≤ m ≤ M ) and the VNs vn, (1 ≤ n ≤ N ). The VN vn (1 ≤ n ≤ N ) connects to the CN cm (1 ≤ m ≤ M ) by an edge in the Tanner graph if the entry H(m, n) = 1. We denote the set of CNs connected to the VN vn as N (vn), (1 ≤ n ≤ N ), and |N (vn)| is called VN degree. Similarly, N (cm), (1 ≤ m ≤ M ) denotes all VNs connected CN m and |N (cm)| is the CN degree. We denote cm (resp. vn) as CN (resp. VN) itself while c(mk) (resp. vn(k)) shows value of the CN (resp. VN) at iteration k. A vector x = {xn} = {0, 1}N , (1 ≤ n ≤ N ) is called a codeword if and only if HxT = 0. We denote y = {yn}, (1 ≤ n ≤ N ) as the noisy version of x when being read from the memory.
B. The Non-Syndrome Probabilistic Parallel Bit Flipping Decoder
The proposed Non-Syndrome Probabilistic Parallel Bit Flipping decoder is a variant of PPBF decoder described in [7] and it is presented in Algorithm 1. The decoding starts by initializing the value for a vector p = (p0, p1, ..., pdv+1), p0 = 0, pdv+1 = 1 and 0 < pe < 1 for 1 ≤ e ≤ dv. It initializes also the VN values by copying the corresponding channel values.
The first half of a decoding iteration is to update the CNs values by calculating the CN equations. The CN computation is the parity check (the XOR operations, denoted by ⊕ ) on all values of its neighbor VNs. The computed values of CNs are sent to their neighbor VNs. In the second half of the iteration, each VN updates its value by the following steps. First, the VN evaluates a function, called energy function, denoted by En(k), 1 ≤ n ≤ N , in which the energy value is the number of unsatisfied neighbors CN values (the CNs with value 1) and the term vn(k) ⊕ yn which indicates the similarity between the VN current value and its channel output. The VN energy values are the integers and vary between 0 to dv + 1, (0 ≤ En(k) ≤ dv + 1). Second, each VN vn will update its values basing on the computed energy. When the energy En(k) = 1, a random variable (denoted by Rn(k), with values 0 or 1) is drawn from the Bernoulli distribution with probability pEn(k) . We denote in the algorithm that p(En(k)) is the En(k)-th element of p and B(p) is the Bernoulli distribution with probability p. The VN value is XOR-ed with the generated random values

Rn(k), which implies the fact that, the VN value is flipped with the probability pEn(k) ∈ p. When the energy En(k) = 1, the above process is proceeded, meaning the VN will flip its value with probability p1 ∈ p, if and only if vn(k) = yn. The updated values of VNs are sent to the next decoding iteration. The NS-PPBF decoding process is terminated when k reaches to the maximum value Itmax allowed.
It can be seen that, NS-PPBF differs from the PPBF in flipping when the energy En(k) = 1 and with this modification, NS-PPBF can halt the decoding process when converging to the correct codeword without requiring the SC value. Indeed, when converging to the correct codeword, all the CN values are 0 and the energy values of the VNs are either 0 or 1. The energy En(k) = 1 in this case is because vn(k) ⊕ yn = 1, in another word, vn(k) = yn. NS-PPBF will not flip any VN values in these cases and the VN values are retained until the end of decoding process.
Stopping the decoding without using SC provides the possibility to improve the NS-PPBF decoding frequency (fmax), compared to PPBF decoder. In fact, the SC in BF decoders is a global operation which calculate on M CN values and produces a binary indicating that all CN are 0’s or not. The flipping operation of PPBF in each iteration is proceeded after having the SC results while that of the NS-PPBF can be done locally and in parallel.
We suggest to use NS-PPBF for correcting errors in the memory system where the user allocate to the decoder a constant amount of time, τ , for decoding. For example, in the context of real time systems the designer needs a deterministic number of clock cycles for writing and reading operations of the memory. In this case, the decoder is allowed to go to Itmax = τ /(1/fmax) iterations (one iteration is finished in one clock cycle). With higher value of fmax, the NS-PPBF can decode with higher Itmax which typically helps correct more errors. To support this claim, we conducted a statistic on the Frame Error Rate (FER) of NS-PPBF as a function of number of decoding iterations on the Tanner code [8] and show the result in Fig. 1. It can be seen that, NS-PPBF preserves the error correction property of PPBF that continuously improves the FER when increasing the iteration number.
III. THE NS-PPBF HARDWARE IMPLEMENTATION
The architecture of NS-PPBF for the dv = 3 regular LDPC codes is presented in Fig. 2. The NS-PPBF implementation can be divided into 3 parts. The first part is composed by the D-FFs which store the channel values (y) as well as the current values of the VNs (v). The outputs of this part are connected to the second part, which is the CNUs, realized by dc-inputs XOR gates, by the connection network 1. The last part of the architecture includes the energy computation (EC) circuits by receiving the CNs values from the connection network 2 and the XOR outputs of the part 1 (vn(k) ⊕ yn). The architecture follows precisely the standard circuits for BF decoders and the detailed operation can be found in [7][9].

Furthermore, the syndrome check module is not necessary and is not implemented in this architecture thanks to the proposed stopping criteria of NS-PPBF. We apply the same principle in generating the probabilistic signal of [7], in which the Random Generator (RG) will produce for each VN 3 random signals with probability p1, p2 and p3.
In the FD block, the MUX5to1 controls the flipping operation of each VN since its output is connected to the XOR gate input in the part 1 of the architecture. The VN value will be flipped if the MUX5to1 produces 1 at its output (the current VN value vn(k) is XOR-ed by 1), otherwise the current VN value is retained (since it is XOR-ed by 0). The MUX5to1 is controlled by the VN energy value. Since we implement the case where p has p0 = 0 and p4 = 1, the input-0 of the MUX5to1 is assigned to 0 and the input-4 is assigned to 1, to always flip the VN value when the energy equals to

4 and to not flip when the energy equals to 0. The random signals with probability p2 and p3 are connected respectively to the input-2 and input-3 of the MUX5to1 and the added MUX2to1 multiplexer affects on the MUX5to1 output when the energy equals to 1. In this case, the random signal with probability p1 is passed to the MUX5to1 output only when the XOR gate computing vn(k) ⊕ yn produces 0, otherwise this MUX5to1 output is always 0. Therefore, the MUX5to1 produces 0 at its output and the VN value is not flipped in two cases: either the energy equals to 0 or the energy is equal to 1 with vn(k) ⊕ yn = 1. These are the cases when the correct codeword is found after decoding, as discussed in the previous section.
It can be seen that, the decoder stops flipping and retaining the VN values when converging to the correct codeword without requiring the global syndrome check results, and from that, the SC is not implemented. It is expected to improve the decoding frequency compared to PPBF decoder and it is confirmed in the synthesis results presented in the next section.
IV. NS-PPBF SYNTHESIS RESULTS AND DECODING
PERFORMANCE
A. Synthesis results
We implement the NS-PPBF decoder on the ASIC 90nm technology for the LDPC code with dv = 3, dc = 48, N = 8192, M = 512, code rate R = 0.9375. For the comparison purpose, we implement the PPBF decoder and the Gradient Descent Bit Flipping (GDBF) decoder in [9] on the same LDPC code. The value p = (0, 0.0001, 0.05, 0.9, 1.0) is used for both NS-PPBF and PPBF decoder. We also compare our proposed decoder with the Min-Sum decoder in [10].
It can be seen in the Tab. I that, NS-PPBF can operate at fmax = 500 MHz which is 100MHz higher than the PPBF and 2.5 times faster than GDBF decoder. At Itmax = 300, NS-PPBF provides the throughput of 13.65Gbps compared to 10.92Gbps of PPBF and 5.1Gbps of MS decoder. In term of hardware complexity, NS-PPBF is equivalent to GDBF decoder while being smaller to PPBF and MS (2.131mm2 compared to 2.367mm2 of PPBF and 2.825mm2 of MS). We also provide the average decoding throughput at the 10−4 of raw Bit Error Rate (Raw BER), which further emphasizes the advantages of NS-PPBF over other decoders.

B. Decoding performance
We compare the performance of NS-PPBF to other decoders in Fig. 3. NS-PPBF performs similarly to the PPBF and it exhibits a better performance to GDBF. At Raw BER 10−4, MS is 0.8 decade better than NS-PPBF at the cost of decoder complexity and decoding throughput as presented in the previous sub-section.

V. CONCLUSION
We propose in this paper a modified version of the Probabilistic Parallel Bit Flipping (PPBF), called Non-Syndrome Probabilistic Parallel Bit Flipping (NS-PPBF) to apply to the error correction in storage system. With a special stopping criteria, NS-PPBF can stop flipping the Variable Node values when converging to the correct codeword, without requiring the syndrome check module. NS-PPBF is shown to be very

high decoding frequency, low complexity while offering a very error correction performance.
