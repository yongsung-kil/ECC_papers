Comments on “Edge-Based Dynamic Scheduling for Belief-Propagation Decoding of LDPC Codes”
Qiyou Xie

Abstract— In this paper, we provide comments on the recent paper by Aslam et al. that proposed two low-complexity edge-based scheduling schemes for belief-propagation (BP) decoding of low-density paritycheck (LDPC) codes. In their schemes, the check-node reliability threshold (T ) is important to select the updating edges. However, it is found that the selected value of T in the above literature is not optimal. In this comment, we present a method that leads to a better choice of T .
Index Terms— LDPC codes, edge-based dynamic schedule, convergence performance.

III. COMMENTS AND SIMULATION RESULTS
To demonstrate the effect of γ on the performance of the proposed method for LDPC codes, we simulate the e-Flooding schedules with γ = 1, γ = 10−1, γ = 10−2, γ = 10−3, γ = 10−4 and γ = 10−5. The corresponding values of T and t in the e-Flooding schedules are listed in Table I. There are two types of LDPC codes selected in the simulations. One is the regular LDPC code (1008, 504), which is randomly constructed by the progressive edge-growth (PEG) algorithm with check-node degree dc = 6. The other is the irregular LDPC code (2112, 1056), which is constructed based on IEEE 802.16e.
Fig. 1 shows the performance of the regular LDPC code (1008, 504) employing e-Flooding schedules with different values of γ . The maximum iteration number of the BP decoding is set to 100. It can be observed that the performance of BP decoding becomes better as γ becomes smaller. When γ ≤ 10−1, the performance of BP decoding is stable. It also can be seen that there is about 2dB performance gap between the schedules with γ = 10−1 and γ = 1 (B E R = 10−5).

Then we compare the complexities of the e-Flooding schedules with different values of γ . Note that the numbers of C2V and
V2C update operations in the e-Flooding schedules are equal. Thus
the number of C2V update operations can be used to compare
the complexity. We take the LDPC code (1008, 504) for example. The considered Eb/N0 is the point where the BER reaches 10−4 in Fig.1. The maximum iteration number is set to 100. The number
of codewords in the simulation is 100. The results of the comparison
are shown in Table II, where Ia denotes the average iteration number of the codewords, Ea denotes the average C2V updated edges per iteration, and Esum denotes the total C2V updated edges per codeword. It is obvious that Esum = Ia × Ea. From Table II, we can see that Ea increases with decreasing γ . While Ia decreases with decreasing γ . Thus the values of Esum with γ = 10−3, γ = 10−4 and γ = 10−5 are close to that of γ = 1. That is to say the complexity of the schedules with small γ is almost the same as that of γ = 1.

IV. CONCLUSION
The value of T selected in literature [4] is not optimal in BP decoding. In this comment, we present a novel method to improve the choice of T . Simulation results show that the modified method can get better performance.

Fig. 2. BER performance of the e-Flooding schedules with different values of γ in BP decoding of LDPC code (2112, 1056).
In Fig. 2, we compare the e-Flooding schedules with different values of γ in decoding the LDPC code (2112, 1056). We find that the schedule with γ = 10−3 has the best performance and it has about 2.1dB performance gain compared to that of γ = 1 (B E R = 10−5). When γ ≤ 10−2, the performance of the BP decoding becomes stable.
