Abstract—Non-Surjective Finite Alphabet Iterative Decoding (NS-FAID) represents an LDPC decoding algorithm that uses reduced message storage, with similar or improved error correction performance with respect to Min-Sum. In this paper, we employ NS-FAID compression tables for a memory-centric flooded LDPC decoding architecture. Due to the approximate message storage, improved memory footprint and overall cost is obtained using the NS-FAID approach. We present FPGA synthesis results, in terms of LUT-FF pairs used, working frequency and throughput, as well as Throughput to Area Ratio (TAR). The estimates indicate that employing NS-FAID compression tables yield improvements between 25% and 110% in TAR with respect to the baseline Min-Sum decoder.
Index Terms—Forward error correction, LDPC, parity check, FPGA
I. INTRODUCTION
LDPC codes are part of error detection and correction components of modern communication systems being used in standards such as WIMAX [1], and recently even NANDbased Flash memories [2]. A class of structured LDPC codes that provide good error correction capabilities and are suitable for cost-effective hardware implementations is represented by the Quasy Cyclic Low Density Parity Check codes (QCLDPC). Key features in LDPC decoding architectures, with high impact in the overall cost and power consumption, are represented by memory blocks and the interconnection network used for message storage and routing [3] [4] [5]. One effective methodology for optimizing memory footprint in LPDC decoders is represented by the use of Non-Surjective Finite Alphabet Iterative Decoders (NS-FAID) [6]. NS-FAID provide a method for compressing messages, hence reducing the needed storage capacity. This approach relies on two working quantization domains: a lower quantization for the check-node computation, and a higher quantization domain for the variable node computation. This dual-domain quantization used for MS algorithms has been demonstrated in [7]. NSFAID provides a method that relies on Density Evolution (DE) for determining the most promising framing / deframing functions required for message compression. Furthermore, work in [6] indicated that the reduction of message quantization in NS-FAID with respect to the baseline Min-Sum decoding does

not necessarily lead to error correction capability performance loss.
For QC-LDPC codes, a wide range of partially-parallel decoding architectures can be employed, with different cost/throughput trade-offs. These architectures contain processing units - small adders and comparators -, memory blocks for message storage, and interconnection networks. Depending on the targeted throughput and/or cost, the parallelism degree at different layers can be applied. Several tradeoffs for throughput/area can be considered when designing LDPC decoding architectures. One such approach is represented by the memory-centric decoder for flooding scheduling proposed in [4] [5] [8] [9]. This architecture relies on memory blocks in order to pass messages between processing units - variable node units (VNU) and check node units (CNU) -, while eliminating the interconnection network. For this type of decoder architecture, memory represents the major contributor in the overall cost. We target the reduction of the memory footprint, and thus, of the overall decoder area, by employing the NS-FAID approach of approximate message storage. The memories will store messages from the lower quantization domain, while the frame/deframe functions will be implemented in the VNUs.
This paper is organized as follows. In section II, the theoretical background about flooding QC-LDPC decoding and NS-FAIDs is given. Section III presents a detailed technical overview of the memory-centric flooded LDPC decoder, while section IV highlights the architectural changes that have to be made for the NS-FAID concept integration. Section V discusses the synthesis results, while the last section is dedicated for concluding remarks.
II. NS-FAID DECODING OF LDPC CODES
A. Flooding QC-LDPC decoding
LDPC codes are a class of algebraic codes that can be defined either using parity check matrix H or in terms of sparse bipartite graphs called Tanner graph [10]. Decoding is an iterative process, which relies on passing messages on the Tanner graph between two types of nodes: variable nodes -

The quasi-cyclic LDPC codes (QC-LDPC) represent a subclass of LDPC codes, which are defined by the base matrix B [11]. The parity check matrix associated to the code is obtained by replacing each element of base matrix with a matrix of z×z elements. This expansion matrix can be either zero matrix for negative elements of B, either the identity matrix shifted by the non-negative value of the B. Table I denotes an example of B matrix for a QC-LDPC code. Due to their regular structure, QC-LDPC codes allow efficient hardware implementations.
Regarding the message-passing algorithms, Min-Sum (MS) and its variants - offset MS, normalized MS, self-correcting MS, and FAID - represent the preferred algorithms due to their efficient realization of hardware architecture. They rely only on additions and comparisons on a small number of bits, yielding the lowest hardware complexity for processing nodes [12]. The complexity of LDPC decoding architecture comes from the size of the H matrix, as codes with increased error correction capability have hundreds/thousands of rows and columns in the parity check matrix.
An important factor in decoding process is represented by the scheduling strategy. It establishes the steps concerning the order in which variable and check nodes are updated during the iterative process. Two types of scheduling strategies may be employed: flooding and layered. In flooding scheduling [13], each iteration relies on one update of the variable node messages, followed by one update of check node messages.
Flooded decoding using MS algorithm is presented in 1. We use the following notation: β for CNU output messages, α for VNU output messages, γ for the log-likelihood ratios (LLR) - the decoder inputs -, and γ˜ for a-posteriori log-likelihood ratios (AP-LLR). The hard decision - decoder outputs - is computed based on the sign of the AP-LLRs. The messages α, β, γ and γ˜ are quantized on a few number of bits, with typical values being 3-6 bits. Reducing the quantization leads to lower hardware cost, but also to lower error correction capability. The quantization of α and β messages in flooding decoding is denoted q.
Decoding is performed for several iterations. It stops either when the maximum number of iterations is reached (Itmax), either when the parity checks equations are satisfied (Syndromecheck) For the latter, an early termination criterion has to be implemented.
B. NS-FAIDs: Overview

The basic idea is to have two working domains in terms of quantization: a higher domain for the VNU process and the AP-LLR update, and a lower domain for CNU processing. III. MEMORY CENTRIC FLOODED LDPC DECODER
A straight-forward way to implement a flooding scheduling decoder is to directly map the Tanner graph in hardware. However, this leads to a large area cost due to massive parallelization and interconnect related bottlenecks. Taking into account the quasi-cyclic nature of QC-LDPC codes, an interconnect efficient unit has been proposed in [9], together with an algorithm to allow the overlapping of check-node and variable-node message memories. Note that the efficiency of the proposed approach depends on the QC-LDPC code structure. This paper uses the same architecture; however, it does not employ the aforementioned overlap optimization. Nonetheless, the NS-FAID concept is compatible with it. As depicted in Fig. 1, the number of VNU processing units is equal to the number of columns in B matrix, and the number

of CNUs is equal to the number of lines in the B matrix. The memory is organized in number of rows × number of columns memory blocks. Each block is used to store the variable-node and check-node messages corresponding to a B matrix entry and H matrix block. The depth is equal to 2 × z, and width is equal to the number of bits of the stored message. The main advantage of this architecture is represented by the fact that no interconnection routing is required.
The following components are used as decoder building blocks:
• Processing units - fully parallel processing units are employed. One row of H matrix is processed at a time by a CNU unit, while a VNU unit is responsible of processing one column of H. A total of B matrix number of columns × VNUs, and a number rows × CNUs are used. In order to execute one iteration circulant size z

Several optimizations targeting throughput/area have been performed for this type of architecture such as the overlapped message processing [9], vectorization, vector folding [4], or the multiple codeword processing optimization for FPGA [8]. These are complementary to the NS-FAID optimization used in this paper. With respect to other flooded LDPC decoding architectures, this architecture is characterized by low logic utilization, due to the relative small number of processing units and virtually no interconnect since no message routing is needed; therefore, in terms of cost, the memory footprint is dominant.

IV. ARCHITECTURE CHANGES FOR THE INTEGRATION OF THE NS-FAID CONCEPT
In order to accommodate the NS-FAID to the memorycentric LDPC decoding architecture, several changes have been performed. These are described below.
1) VNU changes: VNU operates in the high quantization domain. The messages from the lower quantization domain have to be ”transformed” into their higher quantization version. This is perfomed using de-frame operation for β messages, which are input to the VNU. A Look-up-Table (LUT) is used to implement the de-framing operation. Since the message representation of choice is sign-magnitude (SM), this operation is followed by SM to two’s complement (C2) conversion. We have implemented the MS decoder with q = 4 and the 20 variants of NS-FAID - 10 for w = 2 and 10 for w = 3 for the framing functions presented in Table II. The considered LDPC code is the regular (3, 12) code depicted in Table I. The results are for Xilinx Virtex-7 FPGA devices and are obtained using Xilinx ISE 14.7.

Furthermore, in order to have a fair comparison in terms of TAR metric, we have used the synthesis option distributed memory. Therefore, the cost in terms of BRAM blocks is 0. This is also a valid option in the case when the BRAM blocks are heavily underused, such as when the number of codewords is 1. Results for w = 3 show a TAR of approximatively 0.1 for all cases, while for w = 2, it varies in the range [0.12, 0.17]. With respect to the baseline Min-Sum with 4 bits of quantization, the improvement with respect to TAR metric is in the range 20%, up to 112 %. The reason for it is twofold: on the one hand the CNU, which is the main frequency bottleneck, is greatly influenced by the reduction of its operands size. The comparator tree is of substantial lower complexity if the operand size is reduced from 3 bits to 2 bits (for w = 3), and 1 bit (for w = 2). On the other hand, the main cost contributor in this architecture is memory. Therefore, a reduction of 25% for w = 2, and 50% for w = 3 in the width of the memory word will have a significant impact on overall cost.

In the architecture proposed in [14], TAR of the NS-FAID variants, denoted by ΔT AR with respect to the baseline MS implementation is improved mainly due to the frequency increase. The increase in working frequency for w = 2 with

respect to w = 3 is rather small, which leads to a rather modest increase in ΔT AR improvement - from 42% to 58%. For the architecture in this paper, as well as the one in [15], the main contributor to the ΔT AR increase is represented by cost savings. Using messages quantized on a smaller number of bits leads to important cost savings in both architectures. Furthermore, for w = 2, a substantial increase in working frequency is also obtained. Therefore, the TAR of the NSFAID with w = 2 can be more then double with respect to that of the MS decoder.
VI. CONCLUSIONS
In this paper, we have presented the implementation of memory centric LDPC decoder architecture using NS-FAID decoding. The work has been motivated by the approximate message storage employed by NS-FAID algorithm, which leads to reduced memory footprint. We have used 10 NSFAID framing/deframing functions for reduced quantization of both w = 2 and w = 3, and we have compared the proposed approach with the 4-bit quantization implementation of the baseline MS. Synthesis estimates indicate both cost reductions in the NS-FAID implementations of up to 45% and TAR improvements in ranges from 20% to 110%, with negligible error correction capability degradation – for w = 2, or slightly improved decoding performance – for w = 3.
