Abstract—This article mainly proposes a high-speed encoding and decoding method for LDPC code on FPGA. This method converts a quasi-cyclic LDPC code into a block quasi-cyclic LDPC code, and uses a similar transformation to generate a corresponding generator matrix, thereby improving the parallelism of encoder and decoder and making them have high throughput. Finally, we implemented high-speed encoding and decoding on the FPGA chip of the Kintex7 system by using the CCSDS-recommended (8176, 7154) LDPC code, and these encoder and decoder achieve a throughput of 2.97 Gbps under the condition of 5 iterations.
Keywords-LDPC code; quasi-cyclic LDPC code; high throughput
I. INTRODUCTION
In recent years, LDPC codes have become main research object in channel coding field. It is a block code whose performance is close to the Shannon capacity limit and has a sparse parity check matrix [1]. Compared with Turbo codes, the decoder of LDPC codes have low computational complexity. The hardware implementation of decoders can be roughly divided into parallel and partial parallel operations.
In the partially parallel decoding structure [2], the connection between check node and variable node is random if check matrix of the LDPC code is randomly constructed, which makes it difficult for time division multiplexing of processing unit. Therefore, it is difficult to implement highspeed partial parallel decoding for codewords that are completely randomly constructed. Thus, LDPC code must be designed as a special structure.
The parity check matrix of a quasi-cyclic LDPC code [3], [4] can be divided into an array of cyclic matrixes. For the parallel decoding of high-throughput, high-parallelism LDPC codes, a common challenge is to process a large number of memory for reading and writing in order to prevent memory access conflicts. Because check node and variable node must be read into or written to the memory at the same time when they process multiple messages.
Based on high-speed requirements of encoders and decoders, this article selects quasi-cyclic LDPC codes which are easy to design and implement. According to its pattern, it can be converted into corresponding block quasi-cyclic LDPC code and generator matrix, and high-speed encoder and decoder are designed in a high degree of parallelism, including decoding data input and output module, updating

of variable node and check node, storage of decoding information. The design methods for improving the throughput of encoders and decoders have been optimized, such as efficient use of hardware resources, high system clock stability, and effective reduction of maximum number of iterations. Based on the research and analysis of LDPC code correlation principle and its coding algorithm, we use MATLAB simulation to design and optimize LDPC encoder and decoder by using FPGA, and then write Verilog hardware program for testing and analysis. The test results show that this design can achieve the requirements of relevant indicators. And the high-speed encoding and decoding of encoder and decoder is realized at the rate of 2.97 Gbps under the condition of almost no loss of encoding and decoding performance.

B. Design Scheme of Partial Parallel Decoder
We use a 7-block parallel decoding structure by decomposing check matrix into block circulant matrix and its implementation block diagram is shown in Figure 5.
The decoder consists of the following parts: a memory address generation module, a confidence information memory group, a channel likelihood ratio memory group, a decision memory group, two check node update groups, 16 variable node update groups, and a decoding bit output module. Channel likelihood ratio memory group is used to store channel likelihood ratio (internal message), which is the initialization information of the decoder. In the process of decoding a frame of initialization information, initialization information continuously called by the iterative operation remains unchanged. The decision memory bank is used to store decoding decision output results. The belief information memory group is used to store intermediate calculation results (external information) in iterative
decoding process. Since cyclic matrix ( 511u511 ) is transformed into a block cycle matrix ( 73u 73 ), each subblock ( 7 u 7 ) is a cyclic matrix, so the depth of each
memory is 73 and memory width is 7 times width of quantization. Each node processing unit CFUG and VFUG contain 7 variable node message update function modules (CFU) and 7 variable node message update function modules (VFU) respectively, so each node processing unit CFUG and VFUG can complete 7 rows or 7 columns of message update at the same time.

The workflow for each iteration of entire decoder is as follows:
a) Horizontal scanning (check node update) The input of horizontal arithmetic module inputs 32 sets of data each time, each set of data contains 7 data. Since
the corresponding 7 u 7 small matrix is only a circular
matrix, row number and column number corresponding to one in sparse matrix do not correspond to each other in a part of cyclic matrix. That is, the number one in first
row of 7 u 7 small matrix is not necessarily in first
column (the starting position of the horizontal scan is not zero), and so it is necessary to sort each group of data before entering receiving end. According to check node update formula, the module can be divided into the position of absolute value, minimum value, secondary small value, the location of minimum value in each row of data, and the symbol function. The output of this module corresponds to input. It outputs 32 sets of data at a time and each set of data contains 7 data. Correspondingly, each set of data needs to be reordered and stored in corresponding RAM in original form. b) Vertical scanning (variable node update)

Similar to check node update module, the input of

variable node update module inputs 4 sets of data at a

time, and each set of data contains 7 data.

According to variable node update formula, the module

only needs to perform summation processing on initial

data and data on check node.

The output of module corresponds to input. It outputs 4

sets of data at a time and each set of data contains 7 data.

Finally, new data can be stored in corresponding RAM.

In addition, since iterative decoding process of LDPC

code is a process in which variable nodes and check nodes

alternately transfer external information, VNU and CNU are

input and output each other. Therefore, two frames of data

may be input at the same time, and alternate iterative

decoding may be performed in the form of variable updating

of another frame of data when one frame of data is checked

and updated. Figure 6 shows working sequence in iterative

decoding process.

V. CONCLUSION
The author proposes a constructing method that can realize high-speed parallel coding LDPC codes. The parity check matrix of quasi-LDPC codes is transformed into a block quasi-cyclic LDPC code by row or column permutation, and a corresponding method is used to transform generating matrix. Partial parallel decoding and partial parallel encoding of LDPC codes become possible by using this method. This method increases the parallelism of encoder and decoder without changing basic structure of single-channel partial parallel encoding and decoder. Address generator of memory group can still be implemented with a simple counter, and does not need extra resources to control multiple data, so this method is simple and effective. Finally, an encoder and decoder for a (8176, 7154) quasi-cyclic LDPC code is implemented on a Xilinx FPGA chip, and its information data throughput reaches 2.97 Gbps.
