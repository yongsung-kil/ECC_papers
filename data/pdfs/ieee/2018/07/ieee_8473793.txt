LDPC Codes Derived from Quasi-cyclic Code Design Suitable for Optical Communications

Xiaole Sun and Ivan B. Djordjevic
University of Arizona, Department of Electrical and Computer Engineering, 1230 E. Speedway Blvd., Tucson, AZ 85721, USA
e-mail: sunxiaole@email.arizona.edu
ABSTRACT We propose a rate-adaptive forward error correction (FEC) scheme based on spatially-coupled (SC) LDPC codes derived from quasi-cyclic (QC) LDPC codes with its field-programmable gate array (FPGA) architecture. By FPGA emulation, we show that, with comparable computational complexity, the proposed LDPC codes provide larger coding gain and smaller error floor compare to the QC-LDPC base code. As a result, due to its hardware friendly structure, they are presented to be a promising candidate for the next-generation intelligent optical communication systems such as long-haul optical transmission system. Keywords: forward error correction, LDPC, field-programmable gate array, spatially coupling LDPC.

1. INTRODUCTION
Spatially-coupled (SC) LDPC code was first introduced as LDPC convolutional code (LDPC-CC) in 1999 [1]. A simple approach to construct a LDPC-CC from a standard LDPC block code (LDPC-BC) was given. After the first reveal, many interesting research papers have been published on this promising type of code. More recently, more thorough analysis has been presented to show the underlying theory of such new code. In [2], an iterative decoding threshold analysis for terminated regular LDPC convolutional codes was given for both binary erasure channel and AWGN channel. It was shown that the decoding thresholds for the LDPCC codes are not only better than the belief propagation decoding thresholds of corresponding regular LDPC block codes, but also even very close to the maximum likelihood (ML) decoding thresholds. A more comprehensive and general proof was given in [3] to show that the spatial coupling of individual codes increases the belief-propagation (BP) threshold of the new ensemble to come closer to its maximum a posteriori (MAP) threshold of the underlying ensemble. The phenomenon was termed as вЂњthreshold saturation via spatial coupling.вЂќ The authors of [4] presented a windowed decoding scheme for LDPC convolutional codes to provide the flexibility to set and change the decoding latency on the fly. The multiple construction methods with hardware implementation architecture were proposed for this specific type of code in [5-7]. In this invited paper, we introduce a simple method to construct a set of SC-LDPC codes with different code rates.

2. THE CONSTRUCTION OF RATE ADAPTIVE QC-LDPC BASED SC-LDPC CODES
The method presented in this section demonstrate a straightforward construction of SC-LDPC codes from largegirth QC-LDPC block codes. We will first start with generating large-girth QC-LDPC codes. The performance of the large-girth QC-LDPC codes has been evaluated before and it has been shown to have good performance. Then we provide a simple construction method of designing SC-LDPC codes with the large-girth QC-LDPC codes as the base block.

2.1 Construction of Large-Girth QC-LDPC Codes
There exist various algebraic and combinatorial methods to construct structured LDPC codes. The large-girth QC-LDPC code design based on permutation matrices is of particular interest due to several practical advantages. The structure based on permutation matrices leads to an efficient implementation. It also provides large minimum distance, which gives well waterfall performance, since the minimum distance is exponentially proportional to the girth. A large girth QC-LDPC code with column-weight J and row-weight K can be represented by a parity-check matric H:

2.2 Construction of SC-LDPC Codes
The construction method of our SC-LDPC code is a simple copy-and-shift operation. To facilitate the implementation as much as possible, we use the same base QC-LDPC block code, which comes with the copy step. The main idea of spatially coupling codes is to introduce the irregularity into regular codes. Those higher column weight parts in a SC-LDPC code will benefits the performance of the whole codeword. In our method, the coupling process is sub-block based. For example, if we set the coupling length to 5, with three (3, 15, 1129) QC codes, the SC-LDPC code can be represented as

Given that the SC-LDPC codes have the same structure of its based QC-LDPC codes, we are able to implement the SC-LDPC codes with comparable complexity as the QC-LDPC codes with minor modifications. In this section, we will compare the architecture of QC-LDPC code and SC-LDPC code under the same emulation platform. It consists of an addictive white Gaussian noise generator, a QC/SC LDPC decoder, an error counter, and a virtual input/output (VIO) module. 3.1 FPGA Architecture of QC-LDPC Codes
The structure of the QC-LDPC decoder is shown in Fig. 2(a). We have a check node processor (CNP) and a variable node processor (VNP) to calculate the passing messages. There are two groups of memories, log-likelihood ratio (LLR) memory for storing the LLR values, and a posteriori probability (APP) memory for updating the passing messages. The controller generates the proper valid signals and organizes the read and write addresses to each memory modules. We adopt the layered scaled min-sum algorithm for the decoding algorithm with scale factor of 0.75. In this scheme, the VNP will perform the sum operation while the CNP will perform a minimum number search operation using a tree structure. In each layer, the CNP will update the APP memory with the most up-to-date information, so that, in the next layered iteration, the VNP will use the updated information from the previous layer. In the APP memory, there are JKвЂ™s block memory resources (BRAMs) with size of b Г— resolution of quantized value in the emulation. The parity-check matrix H information is stored in a BRAM associated with the controller. Due to the permutation structure of the QC-LDPC code, we only need to

3.2 FPGA Architecture of SC-LDPC Codes
For SC-LDPC decoder, there are several differences as shown in Fig. 2(b). First, the size of APP memory is larger than the corresponding base QC-LDPC codes by a factor of c. Because there are JKГ—c non-empty sub-matrices in the HSC matrix. The H matrix memory in the controller remains the same as before since we just copy the same base code c times. In this scheme, the controller needs to let VNP and CNP know which layer the current information is coming from. Then the CNP needs to update the corresponding position in APP memory. As illustrated in Fig. 3, by adding layer index to CNP and VNP, we do not need to reserve memory space for those empty sub-matrices. Therefore, the memory is independent of the coupling length m, which gives us a unified architecture to implement multiple SC-LDPC codes.

4. EMULATION RESULTS AND DISCUSSION We implement the proposed QC-LDPC code and SC-LDPC code in a Kintex UltraScale kcu105 FPGA device from Xilinx. In the emulation, we employ a uniform quantization scheme for all values in decoding processing. We choose 8 bits resolution to ensure that the degradation of code performance is negligible, and the error floor phenomenon is not caused by finite precision representation. The emulation is conducted over binary AWGN channel assuming the BPSK modulation. The maximum layered iteration number is set to be Imax = 36 for the QC code and Imax = 108 for the SC codes. As we can see in the figure above, spatially coupling of the base QC-LDPC code provides higher coding gain. The performance is getting better as the coupling length increases. It is worth to mention that there is a code rate loss associated with spatially coupling of LDPC codes. The effective code rates for the c = 3, m = 5, and m = 7 codes are 0.76, 0.733, and 0.706, respectively. The curves provided in Fig. 4 have already accounted for the code rate loss. In our emulation, we did not observe an early error floor down to 10-14. The results showed a SC gain ranging from 0.3 dB to 0.6 dB compared to the base QC-LDPC code. One thing might be interesting to further investigate is the limit of the coupling length. We know that, by doing the coupling, the girth of the new code can be reduced due to the extended structure. Higher coupling length means that additional cycles could be introduced.

5. CONCLUSIONS
We proposed a simple copy-and-shift method for construction of SC-LDPC codes derived from large-girth QC-LDPC codes. The complexity of the decoder of such code remains relatively low. The BER performance has been verified through FPGA based emulation. The results have been shown that the proposed SC-LDPC codes exhibit a superior performance over the base QC-LDPC code. From the implementation perspective, the structure of proposed SC-LDPC code is suitable for optical communication application thanks to its low complexity. It also provides a potential solution for rate adaptation FEC scheme.
