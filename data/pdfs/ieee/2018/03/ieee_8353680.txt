Error Elimination ECC by Horizontal Error Detection and Vertical-LDPC ECC to Increase Data-Retention Time
by 230% and Acceptable Bit-Error Rate by 90% for 3D-NAND Flash SSDs

Shun Suzuki, Yoshiaki Deguchi, Toshiki Nakamura, Kyoji Mizoguchi and Ken Takeuchi Department of Electrical, Electronic, and Communication Engineering, Chuo University, Tokyo, Japan
E-mail shun.suzuki@takeuchi-lab.org

Abstract— Cross Error Elimination (XEE) ECC with Horizontal Error Detection (HED) and Vertical-LDPC (VLDPC) is proposed to extend the data-retention lifetime of 3D-TLC NAND flash-based SSD. HED improves the error correction capability of LDPC ECC by evaluating the error bits in the horizontal direction which means the column direction. Moreover, V-LDPC improves the worst reliability in each WL in the vertical row direction through three (Upper/Middle/Lower) pages. This paper investigates the reliability improvement of 3D-TLC NAND flash memory by XEE ECC. As a result, the data-retention lifetime and the acceptable bit-error rate (BER) are extended by 230% and 90%, respectively.
Index Terms-- 3D-TLC NAND flash; error detection; LDPC; SSD
I. INTRODUCTION
3D triple-level cell (TLC) NAND flash memory achieves larger capacity than multi-level cell (MLC) flash by storing 3 bits per cell. However, the reliability of TLC NAND flash is degraded due to the narrow read VTH margins between each VTH state [1]. To improve the reliability of memory cells in 2D-TLC NAND flash, several coding methods have been proposed. Asymmetric Coding (AC) decreases the number of higher VTHstate cells [2-3]. Moreover, for 2D-TLC NAND flash, lowdensity parity-check (LDPC) error correcting code (ECC) achieves the best error correction capability, compared with BCH ECC [4]. In addition, the emerging 3D vertically stacked TLC NAND flash has different error characteristics than 2DTLC NAND flash [5]. This paper investigates the reliability of the 3D-TLC NAND flash with proposed ECC applying both AC and LDPC. By optimizing the measured characteristics of 3D-TLC NAND flash, Cross Error Elimination (XEE) ECC with Horizontal Error Detection (HED) and Vertical-LDPC (VLDPC) is proposed for higher reliability as shown in Fig. 1. Proposed HED estimates the number of error bits with AC in the horizontal column direction. Moreover, proposed V-LDPC improves the worst reliability in each word-line (WL) by vertical LDPC ECC through Upper/Middle/Lower pages in a WL. In XEE ECC, these two methods secure the horizontal and vertical reliability of 3D-TLC NAND flash. Proposed XEE can be applied to different 3D NAND flash array technologies (i.e., floating-gate (FG) and charge-trapping (CT)). Consequently, proposed XEE ECC increases the data-retention lifetime and acceptable bit-error rate (BER) by 230% and 90%, respectively.

II. PROPOSED ECC FOR 3D-TLC NAND FLASH
Fig. 2 shows the measured VTH distribution of 3D-TLC NAND flash with data-retention errors. The reliability of 3DTLC NAND flash is higher than that of 2D-TLC NAND flash because the memory cell size of 3D NAND flash is larger than that of 2D NAND flash, which means 3D NAND flash can store more electrons [6-7]. However, the higher VTH decreases drastically as data-retention time increases, which leads to increase the BER of higher VTH-states such as “G”-state. The two proposals, HED and V-LDPC, effectively reduce the dataretention errors.

For LDPC decoding, the error correction capability of LDPC depends on the predicted BER. For example, errors are more likely to be corrected when the predicted BERs of the error bits are 50% than when those are 25% (Fig. 3). In conventional LDPC decoding, BER is predicted by referring to the prerecorded tables which store the average BER information, e.g. 0.0% to 1.0% (BER in a block). On the other hand, proposed HED detects errors per code unit of AC, e.g. 8 bits, during read operations. The predicted BER of the error bit which HED detects is updated to higher BER, e.g. 10% to 70% (BER in a code unit of AC). This updated BER improves the reliability of 3D-TLC NAND flash after LDPC decoding.
Fig. 4(a) shows the algorithm of AC, which modulates the VTH-distribution to enhance the reliability. AC adds a flag bit, which determines whether the user data is flipped or not, for each code unit. AC is used to increase or decrease the population of reliable or unreliable VTH-state cells, respectively. In Fig. 4(b), the measured BERs of 2D and 3D-TLC NAND flash are shown. The measured results with AC for 3D-TLC NAND flash are first reported in this paper. In both of them, the higher VTH-states have higher BER. Fig. 5 shows BER prediction of conventional AEP-LDPC w/o upper lower cells to calculate LLR [8]. Based on the average BER information of each page and VTH-state, conventional LDPC predicts BER for each bit of read data. On the other hand, the proposed HED updates the predicted BER by detecting error bits to improve the error correction capability. The error detecting method of proposed HED is shown in Fig. 6. By using AC7 for 3D-TLC NAND flash, the population of “1” data is increased since “1”s are assigned to three pages of “F”-state increased by AC7 (Fig. 4(a)). In this paper, “1” is called majority data, and “0” is called minority data. To detect errors, the number of minority data is compared with that of majority data in a code unit. If the number of minority data is more than half of the code length in a read data, at least one error must be included in minority data because the number of minority data must be equal to or less than half of the code length in program operation.
Fig. 7 describes the example of HED with AC7 for 3D-TLC NAND flash. HED consists of two steps, the error detection (step1) and the update of predicted BER (step2). In step1, the number of minority data (Nmin) is counted, and then, if Nmin is more than half of the code length (NCL), HED detects errors in the minority data. For example, when NCL is 4 and Nmin is 4 as shown in Fig. 7, two bit errors are detected (Ndet = 2). In step2, predicted BER of the minority data where the errors are detected by HED is updated. If Ndet is one, 100% of BER is distributed to predicted BER of the minority data for updating. Similarly, if Ndet is two, 200% of BER is distributed. During this process, predicted BERs of the minority data are updated with considering those before update. In the example of Fig. 7 with Ndet = 2, 0.60% predicted BER, which is the highest BER in the code unit, is updated to the highest BER (65%). This BER can be calculated from the functions of Fig. 7. As a result, predicted BER for LDPC decoding is updated to improve the reliability of 3D-TLC NAND flash.

To avoid ECC fails in the upper page, V-LDPC is proposed (Fig. 10(a)). V-LDPC is the coding to eliminate the difference of detected error rate among pages. V-LDPC is similar to error dispersion coding (EDC) [9] in terms of LDPC coding order. EDC has been proposed to decrease burst errors while V-LDPC equalizes the number of errors detected by HED to improve the worst reliability in each WL. By combining HED with VLDPC, the error correction capability of the worst reliable upper page is secured as shown in Fig. 10(b). Compared with conventional AEP-LDPC w/o upper lower cells, proposed XEE ECC extends the acceptable write/erase cycles by 2.5 times as shown in Fig. 11. Consequently, the data-retention lifetime is

increased by 230% and the acceptable BER is increased by 90% as shown in Fig. 12.
IV. CONCLUSION
This paper proposes XEE-SSD with HED and V-LDPC to extend the data-retention lifetime of 3D-TLC NAND flash. By using HED with V-LDPC, the reliability of 3D-TLC NAND flash is improved. As a result, the data-retention lifetime is increased by 230% and the acceptable BER is increased by 90% in 3D-TLC NAND flash.
