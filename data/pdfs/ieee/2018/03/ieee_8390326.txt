Combating Bit Errors From Stuck Cells in Flash Memory Using Novel Information Theory Techniques

Ravi Motwani, Zion S. Kwok, and Poovaiah M. Palangappa Non-Volatile Systems Group Intel Corporation Santa Clara, California, USA
Email: ravi.h.motwani@intel.com, zion.s.kwok@intel.com, poovaiah.m.palangappa@intel.com

Abstract‚ÄîRecently, low-density parity-check (LDPC) codes have been successfully deployed in NAND Flash memory based Solid State Drives. As Flash memory scales and has now advanced from two-dimensional architectures to three-dimensional ones, defects in the form of stuck cells have increased. As far as algebraic codes like BCH are concerned, the errors from the stuck cells impact it just as any other errors. However, for LDPC codes, the stuck cells are three times as detrimental compared to other soft errors.
In this work, we first propose to flag the bits read from stuck cells as erasures. It turns out that it is better that the LDPC code be informed that bits are lost as erasures rather than being erroneously informed with high confidence about the stuck cells‚Äô values. This erasures and errors correction improves the performance of the LDPC code.
To realize further improvements in performance, we propose a method to reduce the raw bit error rate due to stuck cells in NAND Flash memory. We propose a divide and conquer strategy whereby we do not use all the available redundancy for LDPC parity. Instead we use some redundancy to first shape the data using sectionalized Flip and Write (FNW) so that it matches with the stuck cells read with high probability. This reduces the bit errors due to stuck cells. The residual small number of errors due to stuck cells needs to be corrected by the LDPC code.
Both of the proposals have been validated with simulation results based on a one kilobyte information block encoded for an LDPC code of rate 0.9. Errors and erasures decoding, and sectionalized FNW result in 1.92√ó and 1.94√ó raw bit error gains for soft-decision decoding, respectively.
I. INTRODUCTION
NAND Flash memory is used in Solid State Drives (SSDs) for its low latency, fast random access and high throughput. However, as Flash memory has scaled and now advanced from two-dimensional to three-dimensional architectures, various types of defects have manifested. An increase in the defects in the form of word line and bit line short and open circuits causes an increase in the raw bit error rate (RBER). The cells that are part of the word lines or bit lines which have either short or open circuits are called stuck cells in information theory literature. The bit error rate (BER) due to these defects hamper the performance of the error correction codes. While algebraic codes like the BCH and the Reed-Solomon codes are impacted by these defects in the same manner as any other bit errors, modern codes like LDPC codes are impacted very heavily by such defects.
In Figure 1, we show a single level cell (SLC) Flash memory threshold voltage distributions for the erase and the

program states. For a 5-strobe read, a read will give either a low, medium or a high confidence logical bit ‚Äò0‚Äô or ‚Äò1‚Äô. If the cell is open circuited, the cell will read as a high confidence ‚Äò0‚Äô. Since bits read from stuck cells result in high confidence errors with probability one-half, it derails the iterative decoding algorithm of the LDPC codes if the stuck cells occur with high probability.

Prior work on handling stuck cells consists of the work of Heegard [1] and work in [2], [3], [4] which mostly comprises of code construction for resilience to stuck cells. The rate of the codes is quite low for stuck cell probabilities of 2e3 we are considering. Recent work on using error correction pointers (ECP) [5] and a more refined implementation of ECP using pay-as-you-go (PAYG) [6] are very interesting works but they need extra memory to store the data in the stuck cells and separate data structures handling. Reads for the pointers and the stuck cell bits for NAND Flash memory impact the Quality of Service (QoS) since those pointers are stored in another NAND die and that die maybe busy during the codeword read.

In this paper, we propose two algorithms to counter the impact of such defects and improve the performance of LDPC codes. In the algorithms we propose, both the redundancy and the QoS are not impacted. Our algorithms need the locations of the stuck cells as input, either during the codeword read or write. Our first algorithm which consists of performing errors and erasures decoding does not need any extra redundancy and needs the location of the stuck cells as input during the codeword read operation. Our second algorithm needs a redundancy of only 10 bits which are obtained by puncturing the LDPC parity by 10 bits, so overall there is no extra redundancy needed. The location of the stuck cells is an input during write to this algorithm.

In the first algorithm, we propose to flag the stuck cells as erasures and perform an errors and erasures decoding of the LDPC codes. By flagging the stuck cells as erasures, we eliminate the evil effect of these cells whose intrinsic read is a high confidence error with a probability of one-half.

Instead if the redundancy is divided between the sectionalized FNW and LDPC code, the division of redundancy works better. The few flag bits needed by the sectionalized FNW is a very good use of redundancy towards combating errors from stuck cells. The number of flag bits needed for the FNW scheme is one per section. II. IMPACT OF HIGH CONFIDENCE ERRORS FROM STUCK CELL READS ON LDPC CODING GAINS
There are various sources of bit errors in NAND Flash memory. One of the major source of bit errors is the threshold voltage (Vt) distribution overlaps. As shown in Figure 1, the probability distribution of logical bit 1 or erase state and logical bit 0 or the program state of a single level cell (SLC) NAND Flash memory overlap. No matter what reference voltage we read at, the overlap will be one source of the raw bit error rate (RBER). The other source of bit errors is that

Fig. 1: Single Level Cell (SLC) Flash Memory threshold voltage distributions for the erase and program states.

there can be bit line or word line short circuits or open circuits. A bit line open circuit will cause that bit to read as a program state for all the word lines. For BCH or Reed-Solomon codes, this means that the bit will be read as logical bit 0 and be in error whenever the data was not a logical bit 0 in that location. However, for an LDPC code, the problem is more severe. Consider that a 5-strobe read is being performed to enable soft-decision decoding (SDD) for the LDPC code. An open circuit will not only read as logical bit 0 and will be in error when the data is a logical bit 1, but will actually read as a high confidence logical bit 0. The errors, which are due to Vt distribution overlaps, are mostly in the low confidence buckets and hence are less detrimental to the performance of the LDPC decoder, since it is easier to change their decision during the iterative decoding. In contrast, it is more difficult for the decoder to flip a high confidence error. Furthermore, if these errors exceed a certain threshold, they are highly likely to result in an LDPC decode failure. We observe that typically one high confidence error is equivalent to three low confidence errors.

IV. SECTIONALIZED FLIP AND WRITE TO SHAPE THE DATA IN ACCORDANCE WITH BITS READ FROM STUCK
CELLS
In the previous section, a solution to address the problem of stuck cells by considering the stuck cell data as erased was proposed to ease the soft read decoding of the LDPC code.

However, this proposal is still limited by the performance loss due to erasures manifesting as high confidence errors. In this section, we try to gain some performance by cleverly shaping the data. Consider the case when both the data and the stuck cell reads completely match. Then the stuck cell read is an advantage for the LDPC since it reads as high confidence correct data. Therefore, if we are able to pre-process the data and make it agree with the stuck cell reads, then there will be no performance loss due to the stuck cells. In order to exactly match the data to the stuck cell value, we need high redundancy. Our proposal is to allocate just sufficient redundancy to reduce the RBER due to the stuck cells. Given the total redundancy available, we distribute it between the data shaping flags and the LDPC parity. Since the LDPC parity by itself is not the best way to handle the errors from stuck cells, this divide and conquer strategy of dividing the redundancy and having separate redundancy for data shaping to handle the stuck cell errors and having the LDPC parity to mostly handle errors from the Vt overlap turns out to be optimal. The data shaping we propose is by using sectionalized Flip and Write. In the next sub-section we first describe the FNW method using an example and then elaborate on the sectionalized FNW scheme.

A. Flip-N-Write (FNW)

We now describe the FNW [9] data shaping procedure which is the building block of sectionalized FNW. At its core, FNW has two forms for the data to be written to the media‚Äîflipped or non-flipped forms. Depending on the data to be written to the stuck cells of the media, FNW procedure chooses to write the data in one of these two forms. We begin by describing FNW from first principles using theoretical formulation and an example. Second, we motivate and describe sectionalized FNW using an example, and theoretically estimate the error-rate gains from using sectionalized FNW.

Example: Without loss of generality and for the ease of understanding, consider an 8-bit data word to be written to the SLC NAND Flash memory, which has a stuck-at-0 pattern with 2 stuck-at-0 cells indicated by red 0‚Äôs in Figure 3(a). The Xs indicate the non-stuck (i.e., programmable) memory cells, which can be programmed to either 1 or 0.

An incoming data write, which has the data pattern shown in Figure 3(b) conflicts with both the stuck cells, resulting in 2 errors in the stored data. However, FNW algorithm weighs the two options‚Äîflipped and non-flipped‚Äîin terms of the resulting errors in the stored data. Since the flipped data results in zero errors in comparison to the 2 errors of the non-flipped data, the FNW data shaping procedure stores the data in its flipped format.

In order to indicate whether the data was flipped or not flipped, a flag bit is used. The flag bit set to 0 indicates that the data is in its non-flipped state and 1 indicates that the data is in its flipped state. The overhead for the FNW encoding is hence one bit.

Both of the proposals were validated with simulation results using an LDPC code. The code had information corresponding to two sectors‚Äì1 kilobyte of information, the code rate is 0.9, and we see 1.92√ó RBER gains with errors and erasures decoding and a 1.94√ó RBER gain with sectionalized FNW.
The punchline is that rather than use the LDPC to correct errors due to stuck cells and errors due to Vt overlaps, it is better to use two different techniques to correct the two types of errors. This divide and conquer strategy consists of assigning part of the redundancy to the sectionalized FNW to reduce the RBER due to stuck cells, while the remaining redundancy is given to the LDPC code to correct the residual errors.

V. CONCLUSIONS
In this paper, we proposed methods to improve the coding gain of the LDPC code which is impaired by the stuck cell errors. First we proposed an errors and erasures recovery by declaring the bits read from stuck cell as erasures. It turns out that rather than the stuck cells informing of high confidence errors to the LDPC decoder with probability one-half, it is better to erase the bit. Basically, no information is better than wrong high confidence information.
