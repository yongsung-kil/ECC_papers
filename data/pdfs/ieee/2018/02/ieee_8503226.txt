Abstract—This paper presents some new results on QCLDPC codes constructed based on Reed-Solomon (RS) codes. Results include designs and constructions of RS-based QC-LDPC codes with girth 8, cycle structure in their Tanner graphs, and correction of erasures.
I. INTRODUCTION
Some recent research [1]–[5] combines two powerful categories of codes, namely, RS [6] and LDPC codes [7], to form powerful classes of hybrid codes which, not only perform well, but are also practically implementable. These research results take two different approaches. The first approach is to encode and decode an RS code as a powerful LDPC code through a specific mapping [1], [2]. The second approach is to construct a structured QC-LDPC code based on the parity-check matrix of an RS code under certain constraints [3]–[5]. Such a code is referred to as an RS-based QC-LDPC code. In this paper, we further investigate several aspects of RS-based QCLDPC codes, including new designs and constructions, girth and cycle structure of their Tanner graphs, and corrections of erasures.
II. CONSTRUCTION OF RS-BASED QC-LDPC CODES
In this section, we present the basic construction of binary RS-based QC-LDPC codes. The construction of such a code begins with the parity-check matrix BRS of a chosen RS code CRS over a finite field GF(q) which satisfies the following constraint: any 2×2 submatrix of BRS is nonsingular (NS) [5], [8], [9]. We call such a constraint the 2×2 submatrix nonsingular (SNS) constraint, denoted by 2×2 SNS-constraint. Such an RS parity-check matrix BRS is referred to as a 2×2 SNS-constrained RS parity-check matrix. Once a 2×2 SNS-constrained RS parity-check matrix is constructed, we represent each of its entries by a circulant permutation matrix (CPM) of a fixed size, say n×n, where n is a factor of q − 1. This results in an array HRS,n of CPMs of size n×n. It is a matrix over GF(2) which has the following structure: any two rows (or two columns) do not have more than one position where they both have 1-entries. Such a structure is referred to as the row-column (RC) constraint [5], [8]–[10]. The null space over GF(2) of HRS,n gives an RS-based QC-LDPC code, denoted by CRS,n,ldpc. The RC-constraint structure of HRS,n ensures that the Tanner graph of CRS,n,ldpc has girth

at least 6 [8]–[10]. In this paper, we only consider designs and
constructions of binary RS-based QC-LDPC codes based on the 2×2 SNS-constrained parity-check matrices of RS codes over Galois fields of characteristic 2, i.e., q = 2s.

A. Construction of 2 × 2 SNS-Constrained RS Matrices
Let β be an element of order n in GF(2s) where n is a factor of 2s −1. The set Sn={1, β, ..., βn−1} forms a cyclic subgroup of GF(2s). Let d be a positive integer such that 1 ≤ d ≤ n. Form the following d×n matrix over GF(2s):

BRS,n(d, n) = (βi)j 1≤i≤d,0≤j<n .

(1)

The null space over GF(2s) of BRS,n(d, n) gives a 2s-ary (n, n-d, d+1) RS code CRS,n(d, n) of length n, dimension n-d, rate (n-d)/n and minimum distance d+1. The matrix
BRS,n(d, n) is the parity-check matrix of the RS code CRS,n(d, n). Let b be a positive integer such that d < b < n. Suppose we delete n-b columns from BRS,n(d, n). We obtain a d×b matrix BRS,n(d, b) over GF(2s) whose null space gives a (b, b-d, d+1) shortened RS code.
In general, the RS matrix BRS,n(d, n) does not satisfy the 2×2 SNS-constraint. In the following, we give three cases for

which RS matrices satisfy the 2×2 SNS-constraint.
Case 1: n = km, k ≤ m, 1 ≤ d ≤ k+1. Form the following d × m matrix over GF(2s):

BRS,n(d, m) = (βi)j 1≤i≤d,0≤j<m .

(2)

The null space over GF(2s) of BRS,n(d, m) gives an (m, m-d, d+1) shortened code CRS,n(d, m) of the RS code CRS,n(d, n). We label the column of BRS,n(d, m) from 0 to m-1 (the powers of β in the top row of BRS,n(d, m)) and label the rows of BRS,n(d, m) from 1 to d (the powers of β in the first column of BRS,n(d, m)).

B. CPM-Dispersions of Elements of a Finite Field
Consider the cyclic subgroup Sn={1, β, ..., βn−1} of GF(2s). For 0≤i≤n − 1, we can represent the element βi by a CPM over GF(2) of size n×n (with rows and columns labeled from 0 to n − 1, respectively) whose generator (the top row) has the unit-element “1” of GF(2s) as its single nonzero component at the position i. We denote this CPM by CP Mn(βi) where the subscript “n” in CP Mn(βi) stands for the size of the CPM (also the order of the cyclic subgroup Sn and β). The representation of the element βi by CP Mn(βi) is unique and the mapping between βi and CP Mn(βi) is one-toone. This matrix representation of βi is referred to as the n×n CPM-dispersion of βi with respect to the cyclic subgroup of Sn. For the 0-element of GF(2s), we represent it by a zero matrix (ZM) of size n×n.
Since n is a factor of 2s − 1, then 2s − 1 must be the product of a positive integer c and n, i.e., 2s-1=cn. Suppose c is a composite number and l is a factor of c. Then, ln is also a factor of q −1. Let δ be an element in GF(2s) of order ln, i.e., δln=1. Then, β can be expressed as the l-th power of δ, i.e., β=δl. Let Wln be the cyclic subgroup of GF(2s) with order ln generated by the powers of δ. It is clear that Wln contains Sn as a subgroup. If we disperse each element in Wln into a CPM of size ln×ln as described above, then the element βi=δil, 0≤i<n, as an element in Wln, is dispersed into a CPM of size ln×ln, denoted by CP Mln(βi), whose generator has its single 1-component at the location il. In this case, every element βi in Sn is uniquely dispersed into a CPM of size ln×ln which is referred to as the ln×ln CPM-dispersion of βi with respect to the super group Wln of Sn. Clearly, this CPMdispersion of each element βi, 0≤i<n, in Sn with respect to the super group Wln is unique and the mapping between βi and CP Mln(βi) is one-to-one with respect to Wln. The number ln is called the dispersion-factor with respect to Wln. Hence, each element in Sn can be one-to-one dispersed into a CPM of a size equal to the order of a cyclic subgroup of GF(2s) which contains Sn as a subgroup (including Sn itself).

C. Construction of RS-Based QC-LDPC Codes
Any of the three types of 2×2 SNS-constrained RS matrices
given in Section II.A can be used to construct RS-based QC-
LDPC codes. Consider the 2×2 SNS-constrained d×m RS matrix
BRS,n(d, m) over GF(2s) in the form of (2) constructed based on a cyclic subgroup Sn={β, ..., βn} of GF(2s). We form an lnd×lnm matrix HRS,ln(d, m) over GF(2) by dispersing each entry in BRS,n(d, m) into a binary CPM of size ln×ln with respect to a cyclic subgroup Wln of GF(2s) with order ln that contains Sn as a subgroup. The matrix HRS,ln(d, m) is a d×m array of CPMs of size ln×ln. This array HRS,ln(d, m) is called the ln×ln CPM-dispersion of the RS matrix BRS,n(d, m), denoted by CP Mln(BRS,n(d, m)). If n (or ln) is large enough, HRS,ln(d, m) is a sparse matrix with column and row weights d and m, respectively.
The array HRS,ln(d, m) consists of d (m) row (column)-blocks of CPMs. Each CPM row (column)-block of HRS,ln(d, m) consists of ln rows (columns). Each row (column) in a CPM row (column)-block contains m (d) 1-entries which reside in m (d) separate CPMs in the CPM row
(column)-block, one in each. No two rows (or two columns) in
the same CPM row (column)-block have any position where
they both have 1-entries, i.e., they are disjoint with respect to the locations of the 1-entries. Furthermore, the m CPMs in each CPM row-block of HRS,ln(d, m) are distinct, and the d CPMs in each CPM column-block of HRS,ln(d, m), except the 0-th column-block, are distinct.
The null space over GF(2) of HRS,ln(d, m) gives a (d, m)regular QC-LDPC code, denoted by CRS,ln,ldpc(d, m), of length lnm. The rate of CRS,ln,ldpc(d, m) is at least (m−d)/m which is the rate of the RS code CRS,n(d, m). Since CPMdispersion of BRS,n(d, m) may induce redundant (linearly dependent) rows in HRS,ln(d, m), the rank of HRS,ln(d, m) may be smaller than the number of rows dln in HRS,ln(d, m). The code CRS,ln,ldpc(d, m) is referred to as an RS-based QCLDPC code and BRS,n(d, m) is called the base matrix.
Label the columns of HRS,ln(d, m) from 0 to mln − 1. For 0≤j<m, the j-th CPM column-block of HRS,ln(d, m) consists of the jln-th to the ((j + 1)ln − 1)th columns of HRS,ln(d, m). Let v be a codeword in CRS,ln,ldpc(d, m). Divide the mln code symbols of a codeword v=(v0, v1, ..., vmln−1) in CRS,ln,ldpc(d, m) into m sections, vj =(vjln, vjln+1, ..., v(j+1)ln−1), 0≤j<m, each consisting of ln consecutive code symbols. Expressing v in terms of its m sections, we have v=(v0, v1, ..., vm−1). The ln code symbols in the j-th section vj of v corresponds to the ln columns of the j-th CPM column-block of the array HRS,ln(d, m). We call the sections of v the CPM-sections.

Following Theorem 2, it is clear that if the Tanner graph of an LDPC code has girth at least 6, its parity-check matrix must satisfy the RC-constraint.

III. RS-BASED QC-LDPC CODES WITH GIRTH 8 AND REDUCTION OF SHORT CYCLES
The error performance of an LDPC code with low-tomoderate rates very much depends on the girth g, and the number of short cycles of lengths g, g+2 and g+4, especially in the waterfall region. An LDPC code whose Tanner graph has girth 8 and a small number of short cycles of length 8, 10 and 12 generally performs very well in the waterfall region. If the code also has a relatively large minimum distance and its Tanner graph contains no small harmful trapping sets, it can achieve a low error rate without a visible error-floor.
A. Construction of RS-Based QC-LDPC Codes with Girth 8
The 2×2 SNS-constraint on an RS matrix over GF(2s) only ensures that the Tanner graph of the QC-LDPC code constructed based on the CPM-dispersion of the RS matrix has girth at least 6, a lower bound. It does not guarantee that it has girth 8, in general not. Theorem 3 only gives a necessary and sufficient condition on a base matrix whose CPM-dispersion gives a QC-LDPC code with girth at least 8. B. Reduction of Short Cycles with Masking
Masking is a technique for removing the short cycles in the Tanner graph of a QC-LDPC code constructed by CPMdispersion of a base matrix [5]. Masking the base matrix BRS,n(d, m) = [(βi)j ]1≤i≤d,0≤j<m can be modeled mathematically as follows. Let Z(d, m) = [zi,j]1≤i≤d,0≤j<m be a d × m matrix with the zero and unit elements of GF(2s) as entries. Define the following product of Z(d, m) and BRS,n(d, m): BRS,n,mask(d, m) = Z(d, m)⊗BRS,n(d, m) = [zi,j βij ]1≤i≤d,0≤j<m, where zi,j βij = βij if zi,j = 1, and zi,jβij = 0 if zi,j = 0. In this matrix product, the nonzero entries in BRS,n(d, m) at the locations corresponding to zero entries in Z(d, m) are replaced (or masked) by zeros. The binary ln × ln CPM-dispersion of BRS,n,mask(d, m) gives a d × m masked array, denoted by HRS,ln,mask(d, m), of CPMs and ZMs of size ln × ln. We call Z(d, m) and BRS,n,mask(d, m) the masking matrix and the masked base matrix, respectively. The null space over GF(2) of HRS,ln,mask(d, m) gives a binary QC-LDPC code, denoted by CRS,ln,mask,ldpc(d, m).

IV. A CLASS OF RS-BASED QC-LDPC CODES FOR CORRECTION OF CPM-PHASED BURSTS OF ERASURES
In this section, we present a class of RS-based QC-LDPC codes for correcting erasures over the binary erasure channel (BEC). An erasure means that a code symbol in a transmitted codeword v is erased during its transmission. If erasures in a received codeword are clustered in a span of consecutive locations (including end-around case), we call such an erasure pattern a burst of erasures. A burst of erasures is said to have length b if the burst is confined to b consecutive positions in which the code symbols at the first and the last positions are

Fig. 2: The BER and BLER performances of the QC-LDPC codes given in Example 2 and 3.
erased. Suppose in the transmission of a codeword v in an RSbased QC-LDPC code CRS,ln,ldpc(d, m), erasures occur and are confined in CPM-sections of v. A burst with erasures that are confined in a CPM-section of v is called a CPM-phased erasure-burst. The longest length of a CPM-phased erasureburst is ln (the dispersion factor). If all the ln code symbols in a CPM-section of v are erased, the burst of erasures is said to be solid, otherwise, it is not solid, i.e., at least one of the code symbol in the CPM-section is not erased.
A. A Process for Recovering Erased Code Symbols
An erased symbol is said to be recoverable if it is contained in at least one check-sum (the inner product of the received codeword and a row of the parity-check matrix of the code) as the only unknown. In this case, the erased code symbol is simply the modulo-2 sum of the unerased code symbols contained in the check-sum. When a word v with erased symbols is received, we check whether there are recoverable erased symbols in v. If there is no recoverable erased symbol in v, we stop the recovery process and declare a recovery failure. If v contains recoverable erased symbols, we recover them. This results in a new received word v(1) with fewer erased symbols or no erased symbols. If v(1) contains no erased symbols or no recoverable erased symbols, we stop the recovery process. In the latter case, we declare a recovery failure. It v(1) contains recoverable erased symbols, we recover them. This results in a new received word v(2). Then, we repeat the check and recovery (CR) process on v(2). Every successful CR-step reduces the number erased symbols. The CR process continues until either all the erased symbols in v are recovered or some remaining erased symbols are unrecoverable. If each CR-step results in at least one check-sum which contains one erased code symbol as a single unknown, then all the erased code symbols in the received word v will be recovered.
With the above CR-process, it can be shown that a (d, m)regular RC-constrained QC-LDPC code CRS,ln,ldpc(d, m) is capable of recovering up to ln erased code symbols confined in a single CPM-section plus d-1 erased code symbols scattered randomly in other CPM-sections of a received codeword v. We continue this recovery process in the same manner until all the erased code symbols in the CPM-sections vi and vi+1 of v are recovered. Since all the erased code symbols in vi and vi+1 correspond to VNs on the single cycle of length 4n in G(i, i + 1), there is always an unerased or recovered code symbol which is a neighbor of an erased code symbol. Hence, every erased code symbols in the CPM-sections vi and vi+1 of v can be recovered. The erasure recovery process will never fail as long as there is at least one unerased code symbol in the two CPM-phased bursts of erasures which are confined to two consecutive CPM-sections of v in CRS,n,ldpc(2, n).
Now, we consider the case for which n is a prime. For this case, since the 2n VNs in the subgraph of the Tanner graph GRS,n,ldpc(2, n) of CRS,n,ldpc(2, n) associated with any two CPM column-blocks of HRS,n(2, n) are on the unique cycle of length 4n in the subgraph, any two random CPM-phased bursts of erasures can be corrected as long as one is not solid.
The maximum number of erasures contained in two correctable CPM-phased bursts of erasures is 2n − 1 regardless whether n is a prime or not a prime. Since the number of parity-check symbols of the RS-based QC-LDPC code CRS,n,ldpc(2, n) is 2n−1, the maximum number of correctable erasures in two CPM-phased bursts of erasures is equal to the number of parity-check symbols of the code CRS,n,ldpc(2, n). In this sense, we regard CRS,n,ldpc(2, n) as optimal for correcting CPM-phased bursts of erasures.
We can perform masking on the 2 × n array HRS,n(2, n) given by (18) so that the code CRS,n,ldpc(2, n) can even correct two solid CPM-phased bursts of erasures. Label the CPM column-blocks of HRS,n(2, n) from 0 to n − 1. Note that each CPM column-block consists of two CPMs, one on top of the other. We first consider masking HRS,n(2, n) for the case with n not a prime. For each CPM column-block of HRS,n(2, n) labeled by an even number, we replace one of the two 1-enties in the last column of the CPM column-block by a 0-entry. The 1-to-0 replacement is performed alternatively between the top and the bottom CPMs. For even i, the 1-entry in the last column of the top CPM of the 2i-th CPM columnblock is replaced by 0, and for odd i, the 1-entry in the last column of the bottom CPM of the 2i-th CPM column-block is replaced by 0. With this masking, when two consecutive solid CPM-phased bursts of erasures occur, there is always a row in HRS,n(2, n) which checks only one erased code symbol. Using the check-sum formed by this row, we can recover the erased code symbol. Once an erased code symbol is recovered, the rest of erased code symbols in the two consecutive CPMphased bursts of erasures can be recovered. So all the 2n erased code symbols in the two solid CPM-phased bursts of erasures can be recovered.
For n prime, we replace one of the two 1-entries in the last column of each CPM column-block of HRS,n(2, n) by 0. The 1-to-0 replacement is done alternatively between top and bottom CPMs. With this masking, when two random solid CPM-phased bursts of erasures occur, two erased code symbols can be recovered in the first step of the erasure-

recovering process. Once this is done, all the rest of the erased code symbols in the two CPM-phased bursts of erasure can be recovered.
With the above masking, we obtain a masked 2 × n array of HRS,n,mask(2, n) which is a 2n × n2 matrix with rank 2n and two different column weights. The null space over GF(2) of masked array HRS,n,mask(2, n) gives a (n2, n2−2n) LDPC code CRS,n,mask,ldpc(2, n), but not a fully quasi-cyclic LDPC code. The maximum number of correctable erasures of CRS,n,mask,ldpc(2, n) is 2n which is equal to the number of parity-check symbols of CRS,n,mask,ldpc(2, n).
Example 4: Let GF(24) be the field for code construction. Factor 24-1=15 as the product of 3 and 5. Set n=5. Let β be an element of order 5 in GF(24). Next, we construct a 2 × 5 array HRS,5(2, 5) of CPMs of size 5 × 5 in the form of (18) which is a 10 × 25 matrix with column and row weights 2 and 5. The QC-LDPC code CRS,5,ldpc(2, 5) given by the null space over GF(2) of HRS,5(2, 5) is a (2, 5)-regular (25, 16) QC-LDPC code. Each codeword in CRS,5,ldpc(2, 5) consists of 5 CPM-sections, each consisting of 5 code symbols. Since n = 5 is a prime, the Tanner graph associated with any two CPM column-blocks of HRS,5(2, 5) is a single cycle of length 20 with all its 20 nodes lying on it. Hence, this code is capable of correcting any two CPM-phased bursts of erasures, one not solid. The maximum number erasures that can be corrected is 9 which is equal to the number of parity-check symbols of the (25, 16) QC-LDPC code CRS,5,ldpc(2, 5).
Suppose we set n=15 and construct a 2 × 15 array HRS,15(2, 15) of CPMs of size 15 × 15 in the form of (18). The array HRS,15(2, 15) is a 30 × 225 matrix with column and row weights 2 and 15, respectively. The null space over GF(2) of HRS,15(2, 15) gives a (225, 196) QC-LDPC code CRS,15,ldpc(2, 15). Each codeword in CRS,15,ldpc(2, 15) consists of 15 CPM-sections, each consisting 15 code symbols. The Tanner graph associated with two consecutive CPM column-blocks of HRS,15(2, 15) is a single cycle of length 60 with 30 VNs and 30 CNs lying on it. The code CRS,15,ldpc(2, 15) is capable of correcting two consecutive CPM-phased bursts of erasures. The maximum number erasures that can be corrected is 29 which is equal to the number of parity-check symbols of the (225, 196) QC-LDPC code CRS,15,ldpc(2, 15).
With masking, the first code becomes a (25, 15) LDPC code and second code becomes a (225, 195) LDPC code. The first code is capable of correcting any two CPM-phased burst of erasures, solid or not, with a maximum number 10 of correctable erasures. The second code can correct two consecutive CPM-phased burst of erasures, solid or not, with a maximum number 30 of correctable erasures.

V. CONCLUSION
In this paper, we investigated binary RS-based QC-LDPC codes. Methods were given to construct these codes with girth 8. We showed that masking can significantly reduce the number of short cycles in the Tanner graph of an RSbased QC-LDPC code. Also presented in the paper are two subclasses of RS-based QC-LDPC codes whose Tanner graphs have very interesting cycle structures. Based on these cycle structures, we showed a code in these two subclasses is capable of correcting two CPM-phased bursts of erasures with a maximum number of erasures equal to the number of paritycheck symbols of the code. Codes in these two subclasses may find applications in distributed storage systems.
