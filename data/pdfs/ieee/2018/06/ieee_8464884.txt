Abstract‚ÄĒMaximum-likelihood (ML) decoding is an important and powerful method in communications to obtain the optimal performance of a channel code. However, ML decoding is a challenging problem whose complexity grows exponentially with the blocklength of the code, making it unfeasible to decode optimally for large codes.
In this paper, we present a new approach to accelerate ML decoding by minimizing the number of ones in the code‚Äôs underlying matrix representation. We formulate this minimization problem as an integer program and use a heuristic algorithm to solve it. Using these minimized matrices, we significantly reduce the runtime of different ML decoders for various codes, resulting in speedups of up to 81% compared to the runtime on the original matrices.
I. INTRODUCTION
Channel decoding is an important task in any communication system. Thus, determining the optimal performance of a channel code is significant to qualify heuristic decoders by analyzing the gap towards the optimum. Although theoretical bounds like the Shannon limit [7] exist, these bounds do not rely on the underlying code‚Äôs structure, so they cannot give a hint about how much performance gain for heuristic decoders on specific codes is possible. A tight bound can be obtained by Maximum-Likelihood (ML) decoding, which solves the decoding problem optimally and individually for every code. However, solving the ML decoding problem is an NP-hard problem, and obtaining the Frame Error Rate (FER) of the ML decoder via Monte-Carlo simulations requires, e.g., 105 decoding runs for each Signal to Noise Ratio (SNR) and even more for low FERs. Thus, ML decoding is in general very challenging, especially for large blocklengths.
Techniques for optimal or near-optimal ML decoding are an active research field, that heavily relies on mathematical optimization (see e.g. [2], [3], [4], [8], [10], [12]).
In this paper, we present a method to reduce the ML decoding runtime of different linear codes by introducing a preprocessing step, that minimizes the number of ones in the code‚Äôs underlying matrix representation. Minimizing the density in matrix representations is oftentimes performed to reduce storage or simplify calculations, but has, to the best of our knowledge, never been formulated exactly in the context of ML decoding. This preprocessing step enables speedups in the ML decoding of up to 81%, resulting in many hours less computational time when simulating the FER and enabling the computation of a previously unknown ML performance of a large algebraic code. The preprocessing step only has to be performed once.

In detail, this paper makes the following contributions:
‚ÄĘ It is shown that minimizing the number of ones in a code‚Äôs matrix representation is NP-hard. The problem is formulated as an integer program (IP).
‚ÄĘ A heuristic for minimizing the one-entries in a given matrix is presented, and lower bounds on the minimum number of ones are derived. The bounds rely on the underlying code structure.
‚ÄĘ Several matrices of BCH and Reed-Solomon (RS) codes were one-minimized. With the help of the lower bounds we can show that the heuristic attains the minimum in many cases.
‚ÄĘ We show runtime improvements for the optimized matrices of different codes under different ML decoders, reducing the ML decoding runtime up to 81%. This improvement enabled us to simulate the ML FER performance of a large (155,115) RS code for the first time.

III. RELATED WORK
A. Matrix Manipulation
Using equivalent parity-check matrices to improve the decoding performance is a well-known task and has been considered for different codes. In [11], some general rules for paritycheck matrices to work well in belief propagation decoding are given. To obtain better matrices, redundant parity-checks are introduced that improve the decoding performance.
In [6], RS codes are decoded with a modified belief propagation algorithm. Upon receiving the reliability values from the channel, the matrix structure is changed to obtain best-possible decoding performance based on the obtained reliabilities.
The authors of [1] also try to reduce the density in a given RS parity-check matrix for improved belief propagation decoding. In contrast, our paper treats the problem of minimizing the number of ones in a given parity-check matrix from an optimization-theoretic point of view, giving an exact formulation as an IP and verifying the optimality of heuristic results by lower bounds.

B. ML Decoding
The authors of [4] solve the ML decoding problem using a branch-and-bound algorithm. In each step, the problem is divided into subproblems, for which upper and lower bounds are computed to prune parts of the search tree. The lower bounds are computed with an adaptive LP decoder [12], that iteratively computes redundant parity-checks if the solution of the LP was fractional. Well-known tools to solve linear and integer programs are the Gurobi Optimizer1 or IBM‚Äôs commercial solver CPLEX2.

V. HEURISTIC & RESULTS FOR DENSE CODES

Since the problem (MinOnes) is NP-hard and hard to solve in reasonable time, we will propose an integer programmingbased heuristic for this problem. Additionally, we will derive a lower bound for (MinOnes) and apply our heuristic to short BCH and RS codes.

A. Heuristic

applying a row-wise algorithm. This means that the heuristic starts with the original parity-check matrix H and reduces the number of one-entries in the first row of H, if possible. Then,

VII. CONCLUSION AND FUTURE WORK
In this paper, the problem of minimizing the number of ones in a given parity-check matrix was investigated, and an IP for the problem was formulated. With the help of lower bounds on the number of ones, a heuristic was shown

to yield optimal results in many cases. The benefit of one-
minimized parity-check matrices is a large reduction in ML
decoding runtime, where we have shown speedups of up to 81%. Furthermore, this improvement allowed us to simulate the ML FER performance of a large RS code for the first time.
As a next step, the advantages of one-minimized parity-
check matrices will be further investigated, including their
effect on the performance of different decoding schemes, and
the resulting reduction in complexity when moving towards
hardware implementation. In addition, the approach can be
extended to deal with non-binary matrices.
