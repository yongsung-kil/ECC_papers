Variable Correction for Min-Sum LDPC Decoding
Applied in ATSC3.0
Chao Chen1, Yin Xu1, Hao Ju1, Dazhi He1, Wenjun Zhang1, Yu Zhang2
1Cooperative Media Network Innovation Center of Shanghai Jiao Tong University, China 2Academy of Broadcasting Science, SARFT

Abstract—Min-sum (MS) algorithm is a low-complexity approximation of belief propagation (BP) algorithm, but has a severe decline in decoding performance. To reduce this degradation, Normalized min-sum (NMS) algorithm and offset min-sun (OMS) algorithm of a fixed correction factor were proposed. In the literature, extensive empirical results have been provided on NMS and OMS for ATSC 3.0 LDPC codes. It indicates that NMS is approaching BP’s performance, though NMS may have error floors for some codes. Meanwhile, OMS have better performance than NMS in most cases under perfect estimation assumptions. However, in real cases, due to OMS’s sensitivity to channel estimation which is imperfect, it suffers from performance loss. In this paper, the variable correction MS (VCMS) algorithm is introduced, based on which we run extensive simulations for all ATSC 3.0 LDPC codes. Results of frame error rate show that VCMS algorithm not only has a noticeable gain in waterfall region over NMS, but can also eliminate any possible error floors occurred. This algorithm is also better than OMS in many cases. Although in some cases OMS seems to be a little bit better than the VCMS, the VCMS is not sensitive to the imperfect channel estimation. The study of this paper aims to give more options and advice for the implementation of the ATSC 3.0 LDPC decoders.
Keywords—ATSC 3.0; LDPC decoding; min-sum; VCMS
I. INTRODUCTION
The Advanced Television Systems Committee (ATSC) has been committed to developing the next generation digital broadcast television standard, known as ATSC 3.0, which is not constrained by backward compatibility with ATSC 1.0 [1], [2]. In this new standard, high robustness and flexibility of reception have been considered to satisfy the requirements of different devices [3]. Bit-interleaved coded modulation (BICM), which is formed by forward error correction (FEC) codes, constellation mappers and bit interleavers, is part of input formatting module in the ATSC 3.0 physical layer, and it can provide wide operation range and high spectral efficiency [4]. The FEC consists of an inner and an outer code combined. The inner code is the low-density parity-check (LDPC) code[5]. LDPC codes in conjunction with bit interleaver and non-

LDPC codes were first presented by Gallager in 1962 [7]. Due to the remarkable performance that is close to the Shannon bound, LDPC codes have been widely used in many broadcasting systems [8]. As far as is known, belief propagation (BP) algorithm [9] is one of the best performing decoding algorithms for LDPC codes. However, because of the high computational complexity in check nodes updates, BP algorithm can hardly be implemented in practice. Then the min-sum (MS) algorithm was proposed by Fossorier [10], which has a low complexity, but degrades the decoding performance. Thereafter, considerable research efforts have been devoted to improving the performance of MS, such as normalized min-sum (NMS) algorithm [11] and offset minsum (OMS) algorithm which all introduce a correction factor to reduce the overestimation of MS [12]. To further improve the performance of MS algorithm, variable correction (VMS) scheme based on real log likelihood ratios (LLRs) is introduced, which indicates that the value of correction factor should change with the increase of the reliability of LLRs [13]. Nevertheless, this method is only applicable in check-node regular codes and varies solely as per iteration. What’s more, the correction factor is obtained via theoretical predictions, which implies it is not matched to the actual LLRs of the decoder when implemented in real cases. To solve these problems, a criterion based on generalized mutual information (GMI) and Monte Carlo analysis [14] is provided to produce the matched correction factors varying depending on iterations and check nodes degrees, which increases the complexity of computation to some extent.
In the paper [15], NMS algorithm and OMS algorithm are comprehensively analyzed and the corresponding optimal corrections factors are given for all ATSC 3.0 LDPC decoders respectively, which is quite meaningful and crucial for the implementation of the LDPC decoders of ATSC3.0.
However, we believe the decoding performance still has room to be improved. In this paper, we introduce a variable correction algorithm (VCMS) based on GMI with acceptable complexity, which is suitable for the decoding of ATSC 3.0 LDPC codes. Based on VCMS, we run extensive simulations and comprehensively compare the performance among different decoding algorithms when applied to ATSC3.0 codes, such as BP, MS, NMS, OMS and VCMS, aiming to provide more choices for a better implementation of the ATSC3.0 system.

II. LDPC CODES FOR ATSC 3.0
There are two different lengths of LDPC codes adopted in ATSC 3.0: the shorter length (16200) and the longer length (64800) codes [5]. Both lengths of LDPC codes have a wide range of code rates, from 2/15 to 13/15. Besides, for each length code, two structures are employed, which are irregular repeat accumulate (IRA) structure and multi-edge type (MET) structure [6].

As the correction values of α (i) increase with iteration, we search the optimal value of α (i) in the space of [α (i−1) ,1] to obtain a non-decreasing set of α (i) , using 0.01 as a step. The threshold is around BP’s threshold. Fig. 3 shows the non-decreasing curves of different rates of long ATSC 3.0 LDPC codes’ α (i) . It can be seen that these correction factors increase rapidly at the beginning and eventually converge, which implies LLR values is becoming more and more reliable as the number of iteration increases.

V. SUMMARY
In this paper, we introduce the variable correction with acceptable complexity for MS algorithm, which adopts a simplified criterion of GMI, aiming to give more options and advices for the implementation of the ATSC 3.0 LDPC decoders. To better match the real LLRs of LDPC decoders, we use Monte Carlo method to search the optimal set of variable correction factors. We also present the performance comparison of several LDPC decoding algorithms such as BP, NMS, OMS and VCMS. Simulation results of frame error rate show that VCMS algorithm has a noticeable gain over NMS algorithm, and also has similar or better decoding performances than OMS algorithm for many LDPC codes. Although OMS algorithm is a little better than VCMS algorithm in some cases, VCMS algorithm is not as sensitive to imperfect channel estimation as OMS algorithm. In addition, VCMS algorithm eliminates error floors occurred in NMS-based decoders. When implemented in hardware, those variable correction factors, obtained offline based on large data through Monte Carlo simulations, can be used directly. Therefore, the additional complexity of VCMS over that of NMS is trivial.
