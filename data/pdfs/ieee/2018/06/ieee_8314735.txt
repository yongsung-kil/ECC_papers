Rate-Adaptive Protograph LDPC Codes
for Multi-Level-Cell NAND Flash Memory
Pingping Chen
, Kui Cai, Senior Member, IEEE, and Shi Zheng
Abstract—The multi-level-cell (MLC) NAND flash memory
exhibits a diversity of the raw bit error rate (BER) over different
program/erase (P/E) cycles and different types of bits within a
memory cell. In this letter, we first apply a protograph-based
extrinsic information transfer chart analysis to the MLC flash
channel and design novel rate-adaptive protograph low-density
parity-check (RAP-LDPC) codes by using a code extension
approach. The proposed RAP-LDPC code has multiple code
rates, which can be adapted dynamically to different P/E cycles.
To mitigate the unbalanced raw BERs between different types of
bits within a memory cell, we further propose an optimum map-
ping between the variable nodes of the protograph and different
types of bits of the memory cell. Since the proposed RAP-LDPC
codes are based on the same parity-check matrix with specific
structure, a single protograph encoder/decoder is sufficient to
handle all the code rates. Simulation results demonstrate that the
proposed RAP-LDPC codes with optimum mapping outperform
the irregular LDPC codes for all the code rates with a faster
decoding convergence speed for the MLC flash channel.
Index Terms—Multi-level-cell (MLC) NAND flash memory,
rate-adaptive low-density parity-check (LDPC) code, protograph
code, extrinsic information transfer chart (EXIT) analysis.
I. INTRODUCTION
T
HE NAND flash memory has developed a significant
presence in the current non-volatile memories (NVMs)
market with wide applications. With the introduction of the
multi-level-cell (MLC) technology, the storage capacity of the
NAND flash memory has been improved significantly [1].
However, it comes with the side-effects of degraded reliability
and decreased lifetime. For example, information is stored
in the flash memory through an erase and program process.
The repeated program/erase (P/E) operations will induce wear
to the memory cells, leading to a significant increase of the
channel raw bit error rate (BER).
In MLC flash memory, each cell holds one of four symbols
that consist of two bits. The left-most bit is called the most
significant bit (MSB) while the right-most bit is the least
significant bit (LSB). The MSB and the LSB are associated
with the lower page and upper page of the flash memory,

and LSB pages exhibit unbalanced BERs [2] typically when
the signal levels are evenly placed.
Low-density parity-check (LDPC) codes can achieve better
performance than the conventional BCH codes adopted by the
flash memory, and hence are believed to be adequate for ensur-
ing data reliability for MLC flash memory. However, most of
the proposed LDPC codes are fixed rate codes [3]–[5]. Due
to the raw BER diversity of MSB/LSB pages, they have to be
designed for the worst case scenario, thus leading to a severe
capacity waste. To mitigate the performance degradation, [3]
proposed an intra-cell data placement interleaving scheme
and an improved min-sum decoding algorithm. Reference [4]
examined what ratio of the MSB/LSB should be connected
to the check nodes of the regular LDPC code to improve
the decoding threshold. References [6] and [7] presented
architectures of NVM-based storage systems that dynamically
adapt the LDPC code rates to the aging conditions of the
memory, without addressing the specific design of LDPC
codes for the rate adaptive scenarios.
The protograph LDPC codes have shown high potential
for the additive white Gaussian noise (AWGN) channel and
the partial response (PR) channel. The quasi-cyclic (QC)
protograph codes are particularly attractive for low-complexity
implementation, as they can be encoded with simple feed-
back shift-registers, and also have a readily parallelizable
protograph decoder structure [9]. Rate-compatible protograph
LDPC codes are also available in [9] and [10]. However, none
of them are designed specifically for the MLC flash channel.
In this letter, Section II starts from introducing the MLC
flash channel model, its capacity and mutual information
calculations. In Section III, we derive a protograph-based
extrinsic information transfer chart (P-EXIT) analysis for the
MLC flash channel. In Section IV, we design low-complexity
rate-adaptive protograph LDPC (RAP-LDPC) codes whose
code rate can be adapted dynamically to different P/E cycles to
extend memory lifetime. Moreover, to mitigate the unbalanced
raw BERs between the MSB/LSB pages, we propose an opti-
mum mapping between the variable nodes of the protograph
and the MSB/LSB pages. The simulation results are presented
in Section V. Finally, Section VI concludes the letter.

IV. RATE-ADAPTIVE PROTOGRAPH CODES WITH
OPTIMUM MAPPING
To design RAP-LDPC codes for the MLC flash memory
channel, we follow the work of [6] and [8] to keep the length
of the information word (i.e. the payload) to be the same,
and vary the LDPC codeword length nP. The variation of the
codeword length can be accommodated by the system by either
varying the number of codewords per flash page [6], or by
dedicating the unused bits at the beginning of life to the over-
provisioned spare area to boost performance [8].
We propose to start from the design of the high-rate code
and then use a matrix extension approach to derive the low-
rate code. Moreover, since the search space for the base matrix
of a high-rate protograph LDPC code is very large, we start
by producing a rate-0.5 protograph with a small base matrix
of size 3 × 6 (see (3)). We impose the following empirical
constraints to obtain the protogprah of the rate-0.5 code.
• The protogprah with low decoding threshold has one
high-degree variable node and several degree-2 variable
nodes.
• To preserve the linear minimum distance growth property
of the protograph having degree-2 variable nodes, limit
the maximum number of the degree-2 nodes to be the
total number of checks minus 1. This ensures that no
cycle is formed with only degree-2 variable nodes.
Since the MSB page has higher capacity than the LSB page,
we start with a π such that higher-degree and lower-degree
variable nodes of the protograph are mapped to MSB and LSB
pages, respectively. We name it variable-node-degree based
mapping (VNDM). After searching with the above constraints,
and with the guidance of the P-EXIT analysis, the following
protograph has a low decoding threshold, given by

V. SIMULATION RESULTS
Based on the proposed protographs, and by using the
progressive-edge-growth algorithm and circulant permutation,
we construct RAP-LDPC codes with code rates of 0.93, 0.89,
0.85, and 0.80, respectively. They have the same information
word length of 4095 bits, but with different codeword lengths
of 4410, 4620, 4830 and 5145, respectively. The maximum
number of decoding iterations is 10. The performance is
evaluated over the MLC flash channel with 6 reads.
Fig. 5 compares the frame error rate (FER) between the
RAP-LDPC codes and the irregular LDPC codes with the same
code rates, which are optimized through density-evolution for
the AWGN channel. For the R ≈0.89 case, the FERs of a
rate 0.90 irregular LDPC code optimized for the MLC flash
channel [12] is also included. Observed that the RAP-LDPC
codes achieve better FERs than the irregular codes for the
rate 0.93, 0.89, and 0.83 cases, while the FERs are similar for
the rate 0.80 case. In addition, the FERs with the optimum
mapping are also superior to those with random mapping,
which is illustrated for the high-rate cases of 0.93 and 0.89.
Moreover, Fig. 6 shows that the RAP-LDPC codes with
the optimum mapping requires less number of iterations to
converge than the cases with random mapping and with the
irregular codes. The reduction in iterations is up to 27% in the
early lifetime. Thus, the read latency of the flash memory can
also be reduced substantially.
VI. CONCLUSION
This work has proposed novel RAP-LDPC codes by using
the P-EXIT analysis and the code extension approach. The
proposed codes have multiple code rates which can be adapted
dynamically to the different P/E cycles. We have also proposed
an optimum mapping between the variable nodes of the
protograph and the MSB/LSB pages of the memory. The pro-
posed RAP-LDPC codes inherit the advantage of protograph
codes in terms of low encoding and decoding complexity,
while achieving superior error rate performance and faster
decoding convergence speed than the conventional irregular
LDPC codes. The proposed code design can also be easily
extended to the three-level cell (TLC) cases.
