Abstract—Reliability-Based-Layered Belief Propagation (RBLBP) decoding algorithm for LDPC codes is proposed in this paper. Being an informed dynamic schedule (IDS), it measures convergence and selects iterations by the absolute value of logarithmic likelihood ratio information obtained from channel (ABS-LLR-CH). Each variable node can select its own iterations by estimating its own convergence, and finish iterative decoding. Simulation results show that the proposed RBL-BP algorithm has better performance than those of flooding schedule, layered belief propagation (LBP) and variable-to-check residual belief propagation (VC-RBP). For the convergence property of IDS, the proposed algorithm has faster speed than those of flooding schedule and LBP with a little increase in complexity while it has almost no speed decrease with much lower computational complexity than VC-RBP.
Index Terms—LDPC codes, informed dynamic schedule, ABSLLR-CH, RBL-BP.
I. INTRODUCTION
LDPC codes were first proposed by Gallager in 1962 [1], and were rediscovered by MacKay and Neal in the 1990s, owing to the performance approaching to the Shannon limit[2]. The previous researches showed that the performance of LDPC codes has reached within 0.0045dB of the Shannon limit[3]. As expected, LDPC codes have attracted extensive attention of scholars as well as engineers. They have been applied to many usage scenarios, such as deep space communications, disk storage, and so on. Some communication standards, e.g., the wireless local area network standard IEEE 802.11n[4] and the mobile broadband wireless access systems standard IEEE 802.16e[5], use LDPC codes as their coding standard. In addition, It should also be noted that LDPC codes have been chosen as the candidate channel codes for 5G.
The iterative decoding algorithms are one of the study focuses for LDPC codes. The popular decoding algorithm among them is sum-product algorithm (SPA), also called belief propagation (BP). Being a soft decision algorithm, it has the decoding performance close to Shannon limit. In BP, the order of message passing is referred to as schedule. A good schedule strategy can improve convergence speed and bit error performance. Flooding schedule strategy[6] is a twophase schedule in which all the variable-to-check and checkto-variable message updates are simultaneously carried out in each iteration. Although flooding schedule receives much interest, it is not efficient in terms of convergence speed and error floor. As the alternative schedules, series schedule

and informed dynamic schedule (IDS) [7] are proposed to improve convergency speed and lower error floor. In [8] and [9], layered belief propagation (LBP) is presented as one of series schedule strategies, where each check node and its connected variable nodes are updated sequentially. Another one strategy is shuffled belief propagation (SBP) [10], in which, each variable node and its neighbor check nodes are updated in serial. Both the algorithms allow updated message for some node to be utilized more quickly for other nodes. The difference is that LBP is from check nodes to variable nodes and SBP is opposite. These series schedules are twice as fast as flooding schedule in convergence[11]. Faster convergence speed and better performance are obtained using the series schedules compared with flooding schedule.
Based on series schedule strategy, the further improvement is proposed taking advantage of the dynamic characters of message, called informed dynamic schedule (IDS). Some algorithms introduce residual to achieve dynamic nodes selection in message update process. Residual belief propagation (RBP) [12] and node-wise residual belief propagation (NW-RBP) [13] are proposed firstly. In these strategies, the check nodes with maximum check-to-variable edge residual are firstly updated. More algorithms with the dynamic schedule strategy are proposed to improve the convergence and performance. Variable-to-check residual belief propagation (VC-RBP) [14] is an improved dynamic schedule strategy which achieves message update process by maximum variable-to-check edge residual, and it obtains the better performance than RBP. Moreover, some further improved algorithms are also proposed, such as informed variable-to-check residual belief propagation (IVC-RBP) [15] and Lazy residual belief propagation (Lazy-RBP) [16]. IDS obtains fast convergence and good performance by updating the unstable information first. The drawback is its complex computation, residual operation in all edges of Tanner graph[17], which results in hardwarecomplexity increase. So we would like to find a way to decrease the number of edges which need calculate residual, or introduce a new criterion describing the reliability to decrease computation.
In this paper, we propose reliability-based-layered belief propagation (RBL-BP) which uses the absolute value of logarithmic likely ratio from the channel (ABS-LLR-CH) to measure reliability approximately and select the unstable variable

III. RELIABILITY-BASED-LAYERED BELIEF PROPAGATION Conventional BP algorithms treat all variable nodes and
check nodes equally. But for IDS, which variable nodes or check nodes are selected is decided by their current message state. Meanwhile, the decoding process is only operated for those selected nodes. IDS achieves the unequal treatment for nodes, which relies on the criterion used to estimate convergence. Residual is one of the criterions. It is used for some RBP algorithms, such as VC-RBP, NW-RBP. The nodes with low reliability(high residual for RBP) can not converge after few iteration times, but the nodes with high reliability can. The difference in reliability of variable nodes decides that the nodes need different number of iterations in decoding process, the nodes with low reliability need more iterations than the ones with high reliability.
According to the analysis above, we need a criterion to select the nodes with low reliability. Moreover, the nodes with low reliability hope to start decoding process early, and

those with high reliability start late. A layered strategy to achieve different number of iterations for nodes with different reliability is proposed. Define Ln(0) , the absolute value of
log likelihood ratio message from channel (ABS-LLR-CH), as the criterion to estimate reliability. Meanwhile, we also define layered threshold Lvalue threshold and layered stepping value ∂ to select nodes and distinguish the number of iterations. ABS-LLR-CH, layered threshold and layered stepping value are introduced to achieve dynamic schedule BP decoding as shown in TABLE III. The schedule is referred to as RBL-BP.

The proposed informed dynamic schedule strategy modifies the criterion describing the reliability of variable nodes. According to the algorithm, minimum ABS-LLR-CH is defined as the layered threshold Lvalue threshold to select variable nodes at the beginning of the iterations and the threshold can be increased in each iteration. Obviously, the initial threshold value is the minimum ABS-LLR-CH, so the variable node with minimum ABS-LLR-CH, which has the minimum reliability, will be updated in the first iteration. The update process is referred to as layered belief propagation(LBP). In the next iteration, more variable nodes are selected to perform update process with the increasing of the threshold.
The layered stepping value ∂ is introduced to control the range of increasing threshold between two consecutive iterations. More variable nodes, whose ABS-LLR-CH is less than threshold, are selected to perform update process when the layered threshold is added to a larger one by the layered stepping value. How many iteration times variable nodes achieve in decoding process is decided by their ABS-LLRCH, and the variable nodes with smaller ABS-LLR-CH can

Let ρ and λ denote the average degree of check nodes and variable nodes respectively, and the check matrix dimension is M × N . The computation complexity for different iterative decoding algorithms is shown in TABLE IV. For LBP, it is a part of RBL-BP decoding process, so RBL-BP adds a little more computation process than LBP. For RBL-BP and VC-RBP, finding the variable node satisfying the criterion is the main step. In RBL-BP, the N comparison operations are achieved before iterative update process, which makes it easy to select variable node satisfying criterion. Meanwhile, the variable parameter is only the threshold value Lvalue threshold among all the iterations, and updating Lvalue threshold relies on an addition operation. In VC-RBP, it needs calculating the residual for all the variable nodes, and finding the maximum residual needs ρM N comparison operations in each iteration. Thus it is difficult to select the variable node satisfying the criterion. Moreover, the number of variable node update and check node update is N for VC-RBP. The update number for RBL-BP is affected by parameter ε. When the decoding is performed by RBL-BP, the number of nodes which perform update process is less than N in each iteration, the average number of update process is less than N too. So the parameter ε is less than 1. We can note that RBL-BP has lower complexity than VC-RBP.
The value of ABS-LLR-CH is a general variable in decoding process. For different types of LDPC codes, the decoding process can be the same by calculating ABS-LLR-CH. For non-binary LDPC codes, the decoding steps of RBL-BP can also be used to perform iterative decoding. We are now trying to find the new criterion like ABS-LLR-CH to estimate the reliability for non-binary codes.

V. CONCLUSION Reliability-Based-Layered Belief Propagation algorithm (RBL-BP) is proposed in this paper. It introduces the value ABS-LLR-CH as criterion to select variable nodes with low reliability approximately and layered stepping value to achieve different iterations for each variable node. These strategies provide good performance and fast convergence for RBLBP. These advantages are obtained with a lower computation complexity. Moreover, the RBL-BP can be used for different kinds of LDPC-type codes using belief propagation algorithm in decoding, which will be attractive for practical purpose. For
