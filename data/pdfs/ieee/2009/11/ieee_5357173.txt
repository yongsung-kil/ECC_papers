A Real-Time Programmable LDPC Decoder Chip

for Arbitrary QC-LDPC Parity Check Matrices

Xin-Yu Shih, Cheng-Zhou Zhan, and An-Yeu (Andy) Wu

Graduate Institute of Electronics Engineering, National Taiwan University, Taipei, Taiwan
E-mail : genius@access.ee.ntu.edu.tw

Abstract – For the applications of next-generation channel-adaptive communication systems, a real-time programmable LDPC decoder architecture is proposed with three design techniques: divided-group comparison (DGC), adaptive wordlength assignment (AWA), and efficient early termination scheme (EETS). By utilizing programmable principle, the hardware architecture can support arbitrary Quasi-Cyclic LDPC parity check matrices, including various locations of 1’s, information bits, codeword lengths, and code rates. The prototyping LDPC decoder chip using 0.13um CMOS technology, which supports up to 23 code rates with a maximum block size of 1536 bits, only occupies 4.94 mm2 die area, operates at 125 MHz, and dissipates 58 mW power.
Keyword : Programmability, QC-LDPC, and Low-power.
I. Introduction
Low-Density Parity-Check (LDPC) codes were first introduced by Dr. Gallager [1]. At that time, it was very difficult to realize the hardware architecture due to the limited VLSI technology. After three decades, LDPC codes were rediscovered by Dr. MacKay [2]. The excellent errorcorrecting performance is proved much closer to the Shannon limit. Moreover, with the dramatic advancement of VLSI technology, it becomes feasible to implement LDPC codes as the kernel forward error correcting (FEC) module in the communication systems. In 2002, the first (1024, 512) LDPC decoder chip [3] was fabricated via 0.16um CMOS technology. From then on, there are many interesting research works on the VLSI hardware architecture for the communication systems, such as 10G BASE-T Ethernet (IEEE 802.3an) [4], and ultra wide-band system (IEEE 802.15.3) [5]. All of these chips belong to the single-mode configuration.
To improve transmission reliability in the various channel environments, the modern communication standards provide multiple fixed modes. The most popular implementation is the reconfigurable ASIC hardware design, including the chips for wireless metropolitan network (IEEE 802.16e) [6-7], and wireless local area network (IEEE 802.11n) [8]. Besides, the reconfigurable application-specific instruction-set processor (ASIP) is newly introduced, which can be considered as a weakly programmable processor [9]. By several pipelines and elaborated memory architecture, ASIP can overcome the inherent processor-based problems, low operating frequency and low parallelism. Unfortunately, the above LDPC decoder designs can only support multiple limited modes under very different channel environments, of which supported parity check matrices can not be changeable and

are limited to the hardware constraints. Especially, the hardware architecture is not able to satisfy the demands if the standards or the users have the new parity check matrix with better performance. Thus, re-designing the hardware designs would result in the extreme design efforts.
For the applications of next-generation channeladaptive communication systems [10], in which various LDPC codec circuit designs may be needed, it is desired to develop a low-area-cost and low-power LDPC decoder chip, supporting arbitrary Quasi-Cyclic-based LDPC codes with real-time programmability. In the short-distance transmission or less-noise channel condition, the chip can be programmed with shorter codeword lengths or higher code rates; otherwise, in the long-distance transmission or noisy channel condition, the chip can be programmed with larger codeword lengths or lower code rates. Even the LDPC codec can be programmed to suitably deal with the channel noise conditions, such as adjusting the location of 1’s and 0’s in the parity check matrix. In order to improve the matrix flexibility, the similar design idea of the programmable LDPC decoder chip [11] has been developed. However, the hardware architecture in [11] has large chip area and high power consumption with high design loads in the transistor level.
In this paper, we use Normalized Min-Sum Algorithm (NMSA) [12] for LDPC decoding. The hardware architecture of the real-time programmable LDPC decoder chip is proposed with downloading arbitrary Quasi-Cyclicbased LDPC parity check matrices. The real-time programmability feature provides a better solution to the adjustable FEC ASIC design in the next-generation channel-adaptive communication systems [10]. In order to enhance the code freedom and matrix diversity, the location of 1’s and 0’s can be easily user-defined. In addition, three design techniques are developed to take the different design considerations into account: 1) Divided-group comparison (DGC) for shortening critical-path timing. 2) Adaptive wordlength assignment (AWA) for enhancing decoding performance. 3) Efficient early termination scheme (EETS) for saving latency. By using TSMC 0.13um CMOS technology, the prototyping LDPC decoder chip, which supports up to 23 code rates and a maximum block size of 1536 bits, only occupies 4.94 mm2 die area and dissipates 58 mW power.
The remainder of this paper is organized as follows. Section II describes the LDPC codes. In Section III and IV, the proposed hardware architecture and three design techniques are demonstrated, respectively. The chip implementation in TSMC 0.13um CMOS technology and measurement results are both shown in Section V. Finally, Section VI draws the conclusions.

II. Low-Density Parity-Check (LDPC) codes

The (n, k) LDPC codes denote that k information bits can be encoded as the n-bit codeword. The code construction is highly based on a very sparse parity check matrix H. Take (n, k) LDPC codes for examples. The corresponding parity check matrix H(n-k)*n of (n, k) LDPC codes has (n-k) rows and n columns. As for LDPC decoding algorithm, an M-by-N parity check matrix can be transformed into a graphical representation, bipartite graph or Tanner’s graph [13]. M Rows and N columns can be transformed to M check node units (CNUs) and N bit node units (BNUs), respectively. The 1’s in the parity check matrix would be also mapped to the direct straight connection between corresponding CNUs and BNUs. In other words, the two ends of one connection belong to different computation units. By passing message on the straights in the bipartite graph, the mutual information can be exchanged iteratively.
Based on Quasi-Cyclic LDPC code construction principle, a 6-by-12 base parity check matrix has the elements, Ti,j, in the location (i, j), where i and j represent the ith column (1ЉiЉ12) and jth row (1ЉjЉ6), respectively. By zooming in the base parity check matrix, Ti,j can be extended to a p-by-p sub-matrix, where p is socalled extending factor. There are three choices in Ti,j, such as -1, 0, and positive number. The element of -1 and 0 denote an all-zero sub-matrix and an identity sub-matrix, respectively. The positive number denotes the identity matrix shifted right by the corresponding positive number. By the above code construction with setting p as 128 here, we can easily construct the (1536, 768) LDPC codes, as the design example shown in Fig. 1.

III. Proposed Hardware Architecture
Fig. 2 shows the proposed hardware architecture of the real-time programmable LDPC decoder design. There are four types of basic processing units, including check node unit (CNU), bit node unit (BNU), wordlength determination unit (WDU), and early termination unit (ETU). CNU and BNU are main computation units, which are responsible for performing horizontal updating and vertical updating, respectively. WDU and ETU are circuit modules dedicated to implement the design techniques of adaptive wordlength assignment (AWA) and efficient early termination scheme (EETS), respectively. Both two design techniques are specified in detail in Section IV. In addition, in order to achieve the programmability property, two kinds of adjustable-switch memory banks, switched channel-value memory (SCM) and switched mutual-information memory (SMM), are proposed.

Assume that the supported base parity check matrix has mb rows and nb columns maximally. The programmable hardware design needs mb*nb SMMs to store the message-passing between CNUs and BNUs because SMMi,j is directly mapping from the Ti,j in the mbby-nb base parity check matrix. On the other hand, it also needs nb SCMs to store initial channel values (input loglikelihood-ratio, input LLR). Instead of the traditional fixedly-connected storage elements, both two adjustableswitch memory banks are conditionally connected to main computation units. SCMi is connected or disconnected to BNU by the programmable switch, Bi. Then, SMMi,j is connected or disconnected to CNU and BNU by the programmable switches, Ci,j and Bi,j, respectively.
As shown in Fig. 3, the main purpose of the extra step (Step 0) in the proposed decoding method is to achieve real-time programmability as compared with conventional decoding method. In the real-time communication applications, the users can arbitrarily determine the locations of 1’s and 0’s in the QC-based parity check matrix by programming SCMi and SMMi,j in the Step 0. A. Divided-Group Comparison (DGC)
To shorten the critical-path timing, the computation time of CNU should be reduced. As the 12-input CNU shown in Fig. 4, we propose the design technique of divided-group comparison, DGC. The main minimum comparison can be divided into three stages. In the first stage, 12 inputs are separated into four groups and each group has three signals. In each group, not only the pairwise minimum values but also the local group minimums would be calculated. In the second stage, the minimum of any three groups in the previous stage would be generated in the parallel way. The third stage is used to compute the individual values. Each of three stages has the computed timing of one adder and one multiplexer. Finally, the shifter and the adder are used to implement the multiplication of the normalization factor (0.875). Based on the similar design methodology, the critical-path timing is extremely reduced due to the log3N computational complexity, where N is the input number of the CNU.

B. Adaptive Wordlength Assignment (AWA)
In the current literature, the (integer, fractional) wordlength assignment during decoding procedure are always fixed. Although it seems very regular to design, it may lead to the extremely limited decoding performance. Instead, the design technique of adaptive wordlength assignment (AWA) is proposed for improving the decoding performance as well as providing more flexible wordlength assignment. Initially, the wordlength in the chip design consists of 3-bit integer and 2-bit fractional parts, and the dynamic range is [-4, 3.75]. In every iteration, the design criterion would be checked in the WDU, as depicted in Fig. 2. Once the design criterion is met in the (Ik-1)th decoding iteration, the wordlength is adjusted to be composed of 4bit integer and 1-bit fractional parts from the Ikth to 8th decoding iterations, and the dynamic range is [-8, 7.5] with less computation precision.
The design criterion can be easily partitioned into 2 steps. First, the number of soft value of LLR, which is larger than one-fourth of initial dynamic range, would be accumulated. Second, the accumulation number would be checked whether it is larger than the specified threshold (Check_Num), which is empirically chosen as 95% of codeword length. C. Efficient Early Termination Scheme (EETS)
In order to make the LDPC decoder design more flexible, we propose the design technique of efficient early termination scheme (EETS). EETS is used to dynamically adjust decoding iteration number under various communication environments. EETS starts up only when the design criterion of AWA is met, i.e. the “Ctrl” signal (see Fig. 5 and Fig. 7) has true logic. As demonstrated in Fig. 7, EETS is highly related to the correction distance (CD), which is defined as the difference between soft value of LLR (SV) and channel value (CV). The decoder design is early terminated (“Early_flag” is true.) if all the correction distances in the decoded-bits are larger than onehalf of initial dynamic range. Again, we can take (1536, 768) LDPC codes with 8 decoding iterations for examples. Fig. 6 also demonstrates that EETS provides no decoding performance degradation.

V. Chip Implementation and Measurement Results
A prototyping LDPC decoder design is constructed based on the real-time programmable hardware architecture with merging three design techniques. The base parity check matrix has maximum dimensions of 6-by-12 and the expending factor has a maximum of 128. The codeword length can be chosen at most 1536 bits and there are 23 kinds of code rates, including 1/2, 2/3, 3/4,…, 10/11, 7/12, 11/12. As the chip summary in Fig. 8, the LDPC decoder chip is designed and fabricated in TSMC 0.13 um CMOS technology, only occupying 1.81mm x 1.36mm core area and 2.46mm x 2.01mm chip area.

VI. Conclusions

This paper presents the hardware architecture of the real-time programmable QC-LDPC decoder chip with three design techniques. It features real-time programmability, critical-path timing shortening, decoding performance improvement, latency-saving, small chip area, and low power consumption.
