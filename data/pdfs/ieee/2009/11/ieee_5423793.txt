A Spectral-Based Partitioning Algorithm for Parallel LDPC Decoding on a Multiprocessor Platform

Wen-Hsiang Hu, Chun-Yi Chen and Nader Bagherzadeh
Department of Electrical Engineering and Computer Science, University of California, Irvine, Irvine, CA 92697, USA {wenhsiah, cchen29, nader} @uci.edu

Abstract—Low Density Parity Check (LDPC) code is an error correction code that has near Shannon limit performance and is inherently suitable for parallel implementation. It has been widely used in several communication standards such as DVB-S2, WiMAX, and Wi-Fi. To address the need for supporting various LDPC codes in an era where diverse applications are integrated onto a single system, a multi-processor based implementation of the LDPC decoder was proposed. However, the heavy message exchange among processors limits the expected performance. In this paper, we present a partitioning algorithm based on graph spectral clustering to reduce the data communication during the decoding process. From the experiments, our approach successfully decreased the amount of inter-processor communication by 33% ~ 52%, as compared to the original sequential mapping approach. Together with the more balanced computation load from our algorithm, an improvement of up to 85% in the overall decoding time was observed.
Keywords-LDPC, multiprocessor, parallel processing, spectral clustering
I. INTRODUCTION
Low-Density Parity-Check (LDPC) code was proposed by Gallager in 1962 [1] and has been proven to have a performance close to the Shannon limit. The LDPC code is inherently suitable for parallel implementation, and it is adopted by several data communication systems such as DVBS2, WiMAX (IEEE 802.16e), Wi-Fi (IEEE 802.11n), 10Gigabit Ethernet (IEEE 802.3an) and ITU-T G.hn. It is also being considered for emerging standards for communication and storage applications.
With the continuing advance of semiconductor technology, multi-processor system-on-chip (MPSoC) and chip multiprocessor (CMP) are considered for building on-chip multicore systems to provide enough computation capabilities for applications requiring intensive computation and large amount of storage. As for the implementation of LDPC decoders, [2][3][6] attempted to use programmable techniques for designing flexible LDPC decoders. [2] implemented LDPC decoding algorithm on Cell/BE processor. In their approach, the Synergistic Processor Element (SPE) is used to decode codewords independently, so the codeword length that can be handled is limited by the local memory size. Applying Graphic Processing Unit (GPU) for LDPC decoding was investigated in [3], but the performance in long codes may still be restricted by the shared memory architecture. [6] used the Network-based Processor Array (NePA), a distributed multi-processor platform interconnected by network-on-chip (NoC), to perform

LDPC decoding. The authors suggested using a linked structure for storing intermediate data and partitioning the computation and data storage of the total decoding task among processors. Thus, the on-chip multiple processors can be used to decode long LDPC codes cooperatively and efficiently. Nevertheless, the speedup from the parallel implementation is constrained by the communication overhead.
To address this problem, there are two major solutions that can be considered. One way is to reduce the data communication time required for each message exchange by deploying an application specific hardware. For example, a Direct Memory Access (DMA) component can be added to accelerate the data receiving at the message destination node. The other way to tackle the problem is by reducing the number of required data exchanges. To this end, we propose a partitioning algorithm based on the graph spectral theory. The basic idea is that we transform the original partitioning problem into a graph partitioning problem. Then we make use of the spectrum of the graph as clues for performing partitioning. The experimental results from a 16-processor system demonstrate that our approach resulted in a reduction of 33% ~ 52% in the number of inter-processor data exchanges, and the overall LDPC decoding performance is improved by 14% ~ 85%.
II. BACKGROUND
A. LDPC decoding algorithm
LDPC codes are linear block codes characterized by a parity check matrix called H. LDPC codes can also be described by using a bipartite graph, called Tanner graph, in which nodes are divided into two groups: bit nodes (BNs) and check nodes (CNs), and BNs and CNs are linked by bidirectional edges. An edge links the check node m with the bit node n if the Hm,n of the parity check matrix H is 1. A LDPC code is referred to as a (N, K) code if there are N bit nodes and M check nodes in the graph where K = N – M. Fig. 1 shows an example for (8,4) LDPC code and its corresponding Tanner graph. A LDPC code is called regular if each row in H has the same number of 1’s and each column in H also has the same number of 1’s. Otherwise the LDPC code is called irregular.

B. Spectral clustering
Clustering refers to grouping similar objects together. Spectral clustering is a kind of clustering algorithm that utilizes the spectrum of a graph to partition the vertices of the graph. It was first introduced in the 1970s [8] and became popular in the 1990s [10]. It is simple to implement and usually outperforms traditional clustering algorithms. Therefore, it has become one of the most popular clustering algorithms in the machine learning and data mining community [11]. Here we briefly introduce how it works for graph bipartitioning problem, which aims at optimally partitioning the vertices of a graph into two subsets. Given a set of objects and the similarity between object pairs, we can use a weighted undirected graph G = (V, E) to represent the objects and their relations. III. PROPOSED PARTITION ALGORITHM

In [6], it has been demonstrated that it is better to parallelize

the LDPC decoding by assigning both BNs and CNs to PEs

because this approach can enhance the PE utilization and

attain more speedups. This approach also has the potential that

the inter-PE message can be reduced by carefully grouping the

BNs and CNs together onto the same PE. For instance, if CN

m and BN n are connected by an edge in the Tanner graph and

they

are

assigned

to

the

same

PE,

then

the

q (i) nm

and

r (i) mn

message exchange between these two nodes can be done by

intra-PE memory access, instead of inter-PE data

communication. This leads to time saving because inter-PE

communication is much more expensive than intra-PE data

exchange. Since the H matrix can be represented as a bipartite

graph, it is natural to a adopt graph clustering technique for

our purpose.

Based on the spectral clustering approach described in

Section 2.B, we propose the following partitioning algorithm:

(a) Given an H matrix, form the Laplacian matrix L. (b) Compute the first k eigenvector of L. (c) Execute the K-means algorithm on the eigenvectors from Step (b) to obtain the k-way partition.

In Step (a), the Laplacian matrix L is constructed according to (9). For an LDPC code with M CNs and N BNs, we number the CNs from 0 to (M-1) and the BNs from M to (M+N-1). We designate all the edge weights Eij to be 1. The dimension of L is (M+N) by (M+N). Then, in Step (b), the eigenvectors are computed by Octave [12]. In Step (c), we apply the K-means algorithm on the k eigenvectors corresponding to the k smallest eigenvalues of L to obtain k clusters. When executing the K-means algorithm, we require that each cluster has an equal number of BNs and CNs in order to balance the computation load on each PE.
IV. EXPERIMENTAL RESULTS
A. Evaluation environment
We used the NePA computing platform [7] with 16 processors for evaluation. NePA is a CMP environment which is comprised of an array of processing elements (PE). Each PE consists of an OpenRISC 1000 core, local instruction and data memory, network interface (NI), and a router. PEs are interconnected by a two-dimensional mesh on-chip network which provides inter-PE communication capabilities. The delivery of data among PEs is done by packetized data communication. For example, when PE A needs to send data to PE B, it initiates a data transfer by configuring the control registers in its NI. NI performs packetization to generate packets consisting of header flit and body flit, and injects them into the network. Packets are routed and transmitted to the destination PE and arrive at the NI of PE B. An interrupt is generated by NI to notify the processor core that a packet has arrived. The processor B has to execute an interrupt service routine to instruct the NI to do de-packetization and store the received data into the corresponding memory location. The NePA platform is described in cycle-accurate SystemC codes to perform simulation and measure performance.
The proposed partitioning algorithm is mainly implemented in C except that the eigenvector calculation is done by Octave. The C program performs the partition and automatically generates the LDPC decoder application codes for each PE. We used GNU tool chain to compile C programs and convert binary executable to hexadecimal codes for simulation. Several classes of LDPC codes were considered for evaluation, including 1) Code A-C: codes generated by the utilities from Radford Neal’s website [5], 2) Code D-G: WiMAX LDPC codes and 3) Code H-K: Wi-Fi LDPC codes. Table 1 lists a summary of the tested LDPC codes, which cover a wide range of code structures, code rates and code lengths. Code A-C are regular, and Code D-K are irregular.
B. Evaluation results and discussion
In Table 1, the number of edges and inter-PE data exchanges required for each decoding iteration is presented. The number in the parenthesis is the ratio of the number of inter-PE messages to the number of edges. The sequential mapping approach has about 6% reduction in inter-PE traffic while the spectral clustering approach saves 33% ~ 52% of the traffic. Fig. 1 and Fig. 2 visualize the resulted H matrix of two LDPC codes after spectral clustering by reordering the rows

and columns according to the partition results. In Fig. 2(b), BN 0-63 are mapped to PE 0, BN 64-127 to PE 1, and so on. CNs are mapped in the same way. The 1’s in H matrix are drawn as dots. The concentration of dots demonstrates that our approach can successfully congregate BNs and CNs so that the inter-PE data exchanges are transformed into intra-PE memory access. From Table 1, LDPC codes with lower code rates tend to have more traffic reduction from the spectral clustering approach.
The overall decoding performance comparison is presented in Table 2. The number in the parenthesis means the speedup compared to the single PE implementation. First of all, decoding time from spectral clustering are 14% ~ 85% less than those from sequential mapping because the time spent on communication are decreased by 22% ~ 51%. As for computation time, irregular codes (Code D-K) also benefit from the spectral clustering and require less computation time because the computation load is more balanced in spectral clustering than in sequential mapping. Computation time in regular codes (Code A-C) is not reduced in spectral clustering due to the fact that the computation load is balanced in regular codes as long as each PE has the same number of BNs and CNs. Therefore, irregular codes gain more performance improvement from the spectral clustering algorithm.
V. CONCLUSION
Parallel decoding of LDPC codes can be realized on multiprocessor platforms efficiently. To enhance the scalability of the implementation, we proposed to use a spectral-based partitioning algorithm to parallelize the LDPC decoding. The evaluation results from various LDPC codes including WiMAX and Wi-Fi LDPC codes have shown that our approach can effectively reduce the communication overhead, and the overall decoding performance is improved. As a conclusion, we look forward to additional application of spectral clustering in the parallel computing field.
