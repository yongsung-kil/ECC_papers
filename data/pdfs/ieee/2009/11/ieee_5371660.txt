A New Method of Detecting Cycles in Tanner Graph of LDPC Codes

Bo Li, Gang Wang, Hong-juan Yang
Communication Research Center, Harbin Institute of Technology 150001 Harbin, China

Abstract—To improve the performance of LDPC codes, we should detect and eliminate cycles in Tanner graph of the paritycheck matrix when we make LDPC codes. We improved a logic algebraic algorithm in computing entire routes between the nodes of a communication network, and found a new method of detecting cycles in Tanner graph of LDPC codes. This method applied the logic algebraic calculation rules. First transformed the parity-check matrix and then constructed the relative matrix of the Tanner graph. Through integrating and deleting rows of relative matrix and correcting some elements, this new algorithm can detect all the cycles in a Tanner graph of LDPC codes which code length is n and information bits number is k, just needing integrating and deleting n-1 times and correcting k times. The calculation steps of the algorithm is shown detailed by an (8, 4, 2) LDPC example and its correctness have been validated.
Keywords-LDPC codes; Tanner graph; cycle detecting; logic algebraic

II. CALCULATION PROCEDURE OF NEW METHOD
Through graph theory we have known that a Tanner graph has its relative matrix. So our new method constructs relative matrix according to the parity-check matrix of the Tanner graph firstly. After that, we assume one node as source node and then integrate the relative matrix, deleting other nodes progressively [6]. If the code length is n, the Tanner graph will have 2n-k nodes including variable nodes and check nodes. We can get all the cycles which pass this source node just needing n-1 times integration of the matrix and one time amendment of the expression. Then we can assume other nodes as source node and compute repeatedly. We will get all the cycles of the graph after each node participates in the calculation as a source node. We only use logic algebraic calculation in the computing progress of the new method. The calculation procedure will be introduced as follows.

I. INTRODUCTION
LDPC is an important achievement of coding field in recent years [1]. The binary LDPC codes with 1/2 code rate have a distance about 0.0045 dB to Shannon limit [2], and it is the nearest distance to Shannon limit among channel coding methods. The performance of LDPC codes depend on paritycheck matrix structure and it can be descript by Tanner graph [3]. For LDPC codes adopt iteration decoding algorithm based on belief propagation [4], the key point is that the information transmitted between nodes is statistical independence. When there are some cycles in Tanner graph, the information sending from a node will go back itself through a cycle. The result of that is breaking the statistical independence assumption and affecting the decoding accuracy seriously [5]. So when we construct a matrix, we should eliminate cycles or make their cycle length as big as possible. To the LDPC codes which have been constructed, cycles with length 4 can be found in the parity-check matrix, but we should use some algorithm to detect cycles when cycle length is more than 4.
Dai Fu-sheng and others proposed a logical algebraic algorithm to calculate entire routes between all nodes in a communication network [6]. We improved this algorithm and found a new method to detect cycles in Tanner graph of LDPC codes. This new method can detect all cycles in the graph accurately.

A. Construction of Relative Matrix
We can assume that code length is n, information bit length is k, and the check bit length will be n-k, the parity-check matrix can be H=[hij](n-k)×n, where hij is 0 or 1; i must be 1,2,…, n-k and j must be 1,2,…, n. The method needs to construct relative matrix M=[mij] n×n, where mij is the connection from node i to node j and i, j must be 1,2,…,n. We can get the value of mij as follows: if i=j, mij is a main diagonal element of the matrix and indicates the node itself, so mij=1; if there is no connection from node i to node j, mij=0; and if there is an edge Li connect from node i to node j, mij= Li.
We defined variable nodes and check nodes number as 1,2,…,n. If the total number of edges is s, we also defined edge number as L1,L2,…,Ls from left to right. After reforming the parity-check matrix, we can get a new parity-check matrix
H’=[h’ij](n-k) × n, if hij=0, h’ij=0; if hij=1, h’ij=Lx (x=1,2,…,s), where Lx is the edge connecting node i and node j. We can construct the relative matrix use formula (1).

IV. CONCLUSIONS
Our new method can detect cycles in the progress of constructing LDPC codes. Then, we can eliminate all or part of the cycles to keep the statistical independence of information and accuracy of decoding. The cycles detected by this method are comprehensive and exact, more importantly; they are in line with the rules of graph theory. The correctness and

feasibility of our method have been verified by computer programming. We know that this method is a development of graph theory algorithms and it can detect cycles not only in Tanner graph of LDPC codes but also in networks with all kinds of topologies. So the method has a wide range of applications.
