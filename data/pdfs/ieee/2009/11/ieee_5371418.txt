An Investigation in Iterative Decoding of Reed-Solomon Codes based on Adaptive Belief Propagation
Yushan Yang1, Ming Jiang1, and Xiaofu Wu2,1
1National Mobile Communications Research Laboratory, Southeast University, Nanjing, China Email: {young_ys, jiang_ming}@seu.edu.cn
2Nanjing Institute of Communications Engineering, Nanjing, China
Email: xfuwu@ieee.org

Abstract—In this paper, a practical approach to the iterative soft-decision decoding algorithms based on adaptive belief propagation (ABP) is investigated for Reed-Solomon (RS) codes. The novelty is cascading a simplified Chase type decoder to the ABP scheme in each iteration, which can be viewed as an efficient termination criterion deployed in the ABP algorithm. Simulation results show that the proposed algorithm improves the FER performance and converges with fewer iterations averagely, which declares a significant complexity reduction.
Keywords-Reed-Solomon (RS) codes; adaptive belief propagation (ABP); Chase type decoder; iterative decoding
I. INTRODUCTION
Reed-Solomon (RS) codes [1] are among the most celebrated forward error correcting codes. They are currently used in a wide variety of applications, ranging from state-of-the-art communication to data recording systems. In most of the existing systems, RS codes are decoded via an algebraic hard-decision decoding (HDD) [2, 3] algorithm. When soft information about the channel output is available, HDD incurs a significant performance loss compared to the optimal soft decision decoding. Consequently, soft decision decoding algorithms for RS codes are of research interests both for theoretical and practical reasons.
Belief propagation (BP) [4] based decoding has come to researchers’ interest for its appealing performance in decoding sparse graph based codes, such as low-density parity-check (LDPC) codes [5]. However, it is commonly believed that BP algorithm is not suitable for high density parity-check (HDPC) codes, e.g., RS codes, since there are a large number of short cycles in the factor graph which will incur error propagation.
The first successful iterative RS decoding technique, developed by Jiang and Narayanan, was the adaptive belief propagation (ABP) algorithm [6, 7]. This method is implemented using an iterative architecture, in which hard-decision vectors are produced using the output of successive iterations of BP algorithm. To obviate the inherent difficulties of BP decoding on the dense parity-check matrix of RS codes, Gaussian elimination (GE) is performed on the parity-check matrix before BP decoding, such that the variable nodes of lowest reliability connect to the check nodes only once. Although the performance of the ABP algorithm has been shown to be superior to any existing (practical) RS decoding techniques [9], the graph adaption renders a

hardware implementation of the procedure impossible, since GE is quite computationally expensive.
In this paper, we present an approach for the ABP algorithm, the novelty of which is to cascade a simplified Chase type decoder [8] to the ABP scheme in each iteration. The idea combing ABP and Chase recently appeared in Xia’s research on reliability-based RS decoding [11]. Xia applied the flipping concept to the hard-version of ABP algorithm for magnetic recording channels. Our investigation mainly focuses on the iteration of the ABP algorithm, especially the convergence behavior. The proposed algorithm can be viewed as an efficient termination criterion deployed in the ABP algorithm.
The remainder of this work is organized as follows: Some utilized preliminaries are given in Section II. In Section III, we briefly review the ABP algorithm and the Chase type decoder. The main contribution in this paper, the ABP algorithm cascaded with a simplified Chase type decoder (ABP-Chase), is presented in Section IV. Simulation results and a brief conclusion are provided in Sections V and VI.

The main novelty in the ABP scheme is to adapt the parity-check matrix at each iteration according to the bit reliabilities such that the unreliable bits corresponding to a sparse sub matrix and the belief propagation is then applied to the adapted parity-check matrix. This adaptation prevents the iterative decoder from getting stuck at pseudo-equilibrium points and, hence, the convergence behavior of the iterative decoder is significantly improved.
However, the matrix adaption requires a large computation, especially the implementation of GE. Though some research on reducing the adaption complexity has been explored, such as to avoid GE [13, 14] or to find fast GE algorithms [12], the algorithm remains still quite complex.

B. Chase type decoder
The Chase algorithm [8] is a class of decoding algorithms that utilizes channel measurement information, in addition to the conventional use of the algebraic properties of the code. The maximum number of errors that can, with high probability, be corrected is equal to one less than the minimum Hamming distance dmin of the code. This two-fold increase over the error-correcting capability of a conventional binary decoder is achieved by using channel measurement (soft-decision) information to provide a measure of the relative reliability of each of the received binary digits.
The objective of the channel measurement decoder is to use a binary decoder to help obtain a relatively small set of possible error patterns rather than just one error pattern and choose the error pattern of minimum analog weight. Among the three types of Chase decoder, Chase-II is a proper technique, since a considerably smaller set of possible error patterns is considered. The number of bits to generate error patterns is p = [ d min/2 ] . The set of test patterns has any combination of 1s, which are located in the p positions of lowest confidence values, i.e., the p positions with the lowest channel measurements. The output of Chase decoder is the ML decision of received symbols with all the error patterns.

IV. THE PROPOSED ALGORITHM
Although the ABP algorithm shows a superior performance for RS codes, it does not imply a practical low-complexity implementation. In this section, we try to reduce the average computational complexity of the ABP algorithm.
Mostafa in [9] mentioned that, the performance of the ABP algorithm largely depends on the decoder D (hard decision or BM decoder [2, 3]) and the termination criterion used. Besides, he creatively introduced an ASD (KV [10]) decoder as D. The performance of ABP-ASD with 20 iterations and 10 outer iterations is better than the ML upper bound and has more than 3dB coding gain over the BM algorithm at an FER of 10-4. However, the ABP-ASD algorithm requires a much larger computation.
In our work, we innovatively choose the Chase type decoder as the decoder D, cascading a Chase type decoder in each ABP iteration. The proposed algorithm can be viewed as an efficient termination criterion in which a list of codewords is iteratively generated. Note the algorithm as ABP-Chase in the following descriptions.
The termination criterion is as below:
If a decoding success is signaled by the Chase decoder, output the estimated bits; else if the number of iterations is equal to the maximum number of iterations, stop and declare a decoding failure; otherwise, continue with the next iteration.
Besides, one Chase type decoder is run before the ABP algorithm. If it succeeds to find a valid codeword, the ABP iterations can be avoided, which would save a lot of computation. The second Chase decoder is run after each ABP update, making full exploit of the algebraic hard-decision decoder, to obtain a valid codeword.
■
By statistics in our research, the value of p can be smaller than [ d min/2 ] with little performance loss, since the errors occurs on some least reliable symbols in highest probabilities. In this paper, p is set to (N-K) bits, which is much smaller than (N-K)m bits in the original Chase algorithm. And instead of ML decision, the iterations would stop if the Chase decoder finds a “valid” codeword satisfying the parity-check matrix. The direct output of the “valid” codeword is almost 99% the same as ML decision. Consequently, the simplified Chase type decoder can work faster.
The cascaded Chase decoder constructs a larger set of error patterns for algebraic hard-decision decoder, which could make some of the flipped LLR within the decoding bound and then lead to a decoding success. The new termination criterion improves the convergence behavior of ABP algorithm, performs better than the ABP-BM algorithm [7, 9], and reduces the computational complexity by implementing within fewer ABP iterations averagely. Besides, it can be seen that, the proposed algorithm is also a tradeoff between performance and complexity.

V. SIMULATION RESULTS
In this section, simulation results for ABP-Chase decoding of RS codes are presented. In the simulations, we assume pseudo-random sequence as source bits and BPSK modulation over AWGN channel. Frame error ratio (FER) performances of short length (31, 25), mediate length (63, 55) and long length (255,239) RS codes are provided. Besides, the average number of iterations is compared in this paper. Decoding algorithms compared are mainly algebraic hard-decision decoding (HDD), ABP-BM [7, 9], and ABP-Chase (proposed) algorithms. Since ABP-BM and ABP-Chase are enhanced schemes based on the soft-decision ABP algorithm, the performance of original ABP and Xia’s hard-version ABP [11] are not illustrated.
The initial damping coefficient α for RS ABP decoding is 0.1 for (31, 25) code, 0.12 for (63, 55) code and 0.1 for both (255, 239) code. The maximum number of ABP iterations is set to be 5 and 10.

VI. CONCLUSION
In this paper, we proposed an ABP-Chase algorithm for soft-decision decoding of RS codes. The algorithm is based on creating a new and efficient termination criterion for ABP algorithms. Cascaded with the simplified Chase type decoder, which constructs a larger set of error patterns for algebraic hard-decision decoder, the proposed algorithm helps improve the FER performance. Besides, ABP-Chase converges with much fewer average iterations, resulting in a great reduction in complexity.
However, the problem remains whether the complexity of ABP algorithm, especially Gaussian elimination process, can be decreased. The ABP-Chase algorithm just helps ABP to converge more quickly, but even 5 iterations of ABP algorithm require a large computation. And another question emerges how to find the optimal termination criterions for ABP algorithms, and even for BP algorithms, e.g., in decoding LDPC codes. Further investigations would be required.
