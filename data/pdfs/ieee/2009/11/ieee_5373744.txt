Memory Efficient Multi-Rate Regular LDPC Decoder for CMMB

So-Jin Lee, Joo-Yul Park and Ki-Seok Chung, Member, IEEE

Abstract — In this paper, we propose a memory efficient multi-rate Low Density Parity Check (LDPC) decoder for China Mobile Multimedia Broadcasting (CMMB). We find the best trade-off between the performance and the circuit area by designing a partially parallel decoder which is capable of passing multiple messages in parallel. By designing an efficient address generation unit (AGU) with an index matrix, we could reduce both the amount of memory requirement and the complexity of computation. The proposed regular LDPC decoder was designed in Verilog HDL and was synthesized by Synopsys’ Design Compiler using Chartered 0.18μm CMOS cell library. The synthesized design has the gate size of 455K (in NAND2). For the two code rates supported by CMMB, the rate-1/2 decoder has a throughput of 14.32 Mbps, and the rate-3/4 decoder has a throughput of 26.97 Mbps. Compared with a conventional LDPC for CMMB, our proposed design requires only 0.39% of the memory. 1
Index Terms — CMMB, Low Density Parity Check (LDPC) codes, Multi-rate support, Partially parallel decoder.
I. INTRODUCTION
Low Density Parity Check (LDPC) codes were first introduced by Gallager [1] in 1962, but they were rarely used since implementing them in HW was impractical in 1960s. The value of LDPC codes was rediscovered by Mackay and Neal in 1996 [2]. Since then, LDPC codes have gained a lot of attention as channel codes for the 4G mobile communication due to their excellent error correction capability [3].
LDPC codes are linear block codes based on a parity-check matrix called H-matrix of which rows and columns represent parity check codes and symbols, respectively. The H-matrix can be equivalently represented by a Tanner graph, which is a bipartite graph with the check nodes on one partite and the bit nodes on the other. Check node ci of a Tanner graph corresponds to row i in an H-matrix whereas bit node bj corresponds to column j. If the H-matrix has a 1 at row i and column j, the equivalent Tanner graph has an edge between check node ci and bit node bj [4][5]. An H-matrix and the equivalent Tanner graph are shown in Fig. 1.

China Mobile Multimedia Broadcasting (CMMB) is one of the new standards for 4G mobile communication, and its Hmatrix is known to have high error correction capabilities. CMMB is a mobile television and multimedia standard developed and specified by the State Administration of Radio, Film, and Television (SARFT) of China [8]. CMMB employs LPDC and Reed Solomon (RS) codes, and the control logical channel (CLCH) in CMMB adopts fixed channel coding modes: RS (240, 240) for RS codes and 1/2 rate for LDPC codes. Service logical channel (SLCH) coding modes can be flexibly configured according to the system requirement. The selected configuration mode is broadcast to terminals by using system control message [9]. LDPC codes adopt a 1/2 or a 3/4 code rate. To support a multi-rate SLCH, we need a decoder which is capable of handling multiple H-matrices because each H-matrix differs depending on the code rate. However, the decoding algorithm is the same for each data rate. So it’s wasteful to have separate decoders for each data rate. Thus, in this paper, we propose a dynamically reconfigurable decoder which accepts the code rate as a parameter, and reconfigures itself for the code rate. The Bit Node Unit (BNU) and the Check Node Unit (CNU) of the proposed design are reconfigurable to handle the two code rates to reduce the amount of area overhead.
To implement an LDPC decoder for long code words with acceptable performance and size, a partially parallel architecture is preferred over a fully serial or a fully parallel architecture [10]. A partially parallel architecture typically groups check nodes and bit nodes to carry out processing in parallel. Grouping is done in such a way to use multiple functional units and memories without conflicts. Also an

Many studies have suggested various methods for grouping nodes and designing an AGU to enhance the performance.
In [5], the authors propose a method where each element of the H-matrix is represented by the length of the codeword, the length of the information block, the row index and the column index. They map 360 nodes into 90 node groups based on this representation. Each node group is implemented as an operation module for the node processing. Each operation module has its local memory. Each operation module carries out computation by reading messages sequentially from its local memory, and using the address by an AGU, it delivers the result to the local memory of the other connected operation modules. Therefore, the decoder in [5] groups nodes to have simplified memory access patterns. As a result, the throughput of 520Mbps is achieved, but the size of the circuit is quite large. In [10], the authors map nodes into 360 node groups where each node group contains 90 nodes. A single operation unit for each node group processes both bit node and check node operations. Therefore, they use 360 operation units and 360 shared memory blocks. Since their approach is based on a shared memory structure, the size of the hardware is smaller than the structure with local memories, but due to the overhead of the shuffling network, its throughput was 255Mbps.
There have been other approaches of grouping for DVB-S2 and optimizing the AGU. In DVB-S2, addresses are computed by an address generation formula, and 360 nodes can be processed in parallel as shown in [10]. Therefore, by considering the trade-off between the size and the throughput, grouping can be easily done based on a pre-defined parallelization factor. However, CMMB has more constraints to consider than DVB-S2. Also, for a bigger parallelization factor, the complexity of routing increases very rapidly. Therefore, it is very challenging to get a satisfactory throughput. To the best of our knowledge, this paper is the first paper to design an LDPC decoder for CMMB.
In our design, grouping of bit nodes and check nodes is done in such a way that each group is processed by the same address from the AGU. Also, our design is based on shared memory modules, so the access control is a little complex, but the size is much smaller than the one with separate local memories. The AGU is simplified by generating addresses by pattern analysis. We store only an 18x6 sub-matrix and a 9x12 sub-matrix for the 1/2 rate and the 3/4 rate, respectively, and the rest of the H-matrix is generated by the AGU. Therefore, our proposed design is extremely memory-efficient. Furthermore, by utilizing an efficient Index matrix, we reduce the address computation cost as well.

II. LDPC DECODING ALGORITHM
Most practical LDPC decoding relies on soft-decision which is based on the concept of belief propagation (or message passing), because soft-decision decoding typically yields better decoding performance than hard-decision decoding. Soft decision decoding is done by passing messages which contain the amount of belief to be 0 or 1 between adjacent check nodes and bit nodes. Based on the delivered messages, each node tries to decode its own value. If the decoded value contains an error, the decoding process is repeated for a pre-defined number of times until the parity-check turns out to be error-free. Typically, there are two ways to deliver messages in LDPC decoding. One is to use probabilities, and the other is to use log likelihood ratios (LLRs) [6]. In the proposed decoder of this paper, among the methods of using LLRs, Modified UMP-BP algorithm [7] was employed since it shows the merit of reduced complexity and excellent performance. Let N be the length of codewords and M be the number of parity check equations. For a parity check equation with index m where 0 ≤ m < M, we define the set N(m) of codeword symbol positions that it checks,
N (m) = {n | n = 0,1,… , N − 1; H mn ≠ 0}.
Similarly, for a codeword symbol position n where 0 ≤ n < N, we define the set M(n) of indices of parity check equations that check the symbol position n,
M (n) = {m | m = 0,1,… , M − 1; H mn ≠ 0}.
That is, N(m) is the set of all bit nodes connected to check node m, M(n) is the set of all check nodes connected to bit node n.

IV. PROPOSED LDPC DECODER
The proposed LDPC decoder has a partially parallel architecture with 16 BNUs, 16 CNUs and exchange messages with 16 shared memory modules. Each CNU reads the memory from the address generated by the AGU and processes the operation. After completing each operation, the message to be delivered is stored in the same address. It is crucial for the AGU to generate precise addresses without causing any memory access conflict. Therefore, we propose a novel approach which enables us to parallelize processing with 16 shared memories without any conflict. Also, we propose a highly efficient AGU design which generates addresses based on an Index matrix. Furthermore, to support two data rates with a minimal area overhead, most of the blocks are reconfigurable so that they can be commonly used for both modes.
A. Partially parallel architecture
There are mainly three types of LDPC decoder architectures: fully serial, fully parallel, and partially parallel ones. Fully serial architectures are small, but they are too slow for high speed data transmission. On the contrary, fully parallel architectures are fast, but they are too costly in terms of area. Also fully parallel architectures are hard to implement when the block size is large [12]. A partially parallel architecture was first proposed by [13]. Partially parallel architectures are efficient in terms of both area and

Of course, groups of 4, 8, 32, 64, 128 or 256 nodes will generate the same result from the modulus operation. D. Reconfigurable BNU and CNU
CMMB supports two code rates. The H-matrix is different for each code rate as shown in Table I. To support two code rates by a single module, our proposed design is capable of reconfiguring the structure depending on the current code rate. Specifically, the two modes have different memory access patterns. In case of the 3/4 code rate, a CNU requires 12 values for a single processing, and in case of the 1/2 code rate, it needs 6 values. Therefore, we designed a memory with 6 read ports, and in case of rate-3/4, 12 values are read in two clock cycles. This will cause an increase in the number of clock cycles for decoding. But we can reduce the area overhead significant since designing a memory block with 12 read ports will incur too much delay and complexity. We may have an additional cycle for a read since the proposed rate-3/4 decoder with two cycle access still meets the performance requirement by the CMMB standard.

The throughput requirement for rate-1/2 decoder is 10.852Mbps, and that for rate-3/4 decoder is 16.243Mbps. As mentioned earlier, this paper is the first paper to design an LDPC decoder for CMMB. So we could not find any reference to compare our design with. Designs in [5] and [10] are LDPC decoders targeted for DVB-S2. So it is not reasonable to compare the designs with the proposed one because they are based on a totally different H-matrix.

VI. CONCLUSION
In this paper, we propose a novel LDPC decoder which supports multiple code rates for CMMB. The proposed design has a partially parallel architecture where multiple messages are transferred in parallel to enhance the performance. To support the two code rates of the CMMB standard, BNUs and CNUs are dynamically reconfigurable. Using an AGU, we could reduce the memory requirement considerably. Furthermore, to reduce the runtime overhead to compute the addresses with little memory overhead, an intelligent indexing method was proposed. The area of the synthesized decoder is 455K (in NAND2), and the operation frequency is 185MHz. The throughput of the rate-1/2 decoder is 14.32Mbps, and that of the rate-3/4 decoder is 26.97Mbps, both of which meet the throughput requirements for the CMMB standard.
