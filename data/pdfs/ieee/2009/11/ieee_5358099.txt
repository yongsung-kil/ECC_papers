A LDPC Decoder with All Single Port Memories

Ying-Hong Tian, Xiao-Jun Zhang, Zong-Sheng Lai
East China Normal University, Shanghai, China yhtian@ee.ecnu.edu.cn

Abstract—this paper proposes one LDPC decoder with all single port memories to lower requirements of logic and interconnection and save hardware cost. Considering the lower read-write speed of single port memories than that of dual port memories, the decoder needs to reduce computation complex and shorten the critical path delay so as to get high throughput. To get the purpose, the LDPC decoder uses TDMP algorithm and normalized MS algorithm to reduce computation complex, uses reusable and configurable CVPU to realize single cycle pipeline variable and check node updating calculation, uses optimized shuffle network to shorten the path delay, and uses memories dominated controller to avoid the read and write conflict of memories. The implementation results show for once iteration process the throughputs are about 890Mbps, 847Mbps, and 863Mbps for rate 0.4, 0.6, and 0.8 respectively.
Keywords-LDPC; TDMP;MS; single port memory
I. INTRODUCTION
The Low-Density Parity-Check (LDPC) codes have received much attention due to their excellent performance. These codes have been proved to achieve performance near to Shannon’s limits [1]. Their decoders can be fully parallelized and potentially have high throughout and low decoding latency. As a result, LDPC codes haven been adopted in DVB-S2, WLAN (IEEE 802.11n), WiMAX (IEEE 802.16e) and DMB-T.
LDPC codes are linear block codes defined by a sparse parity-check matrix. The sparse matrix can be classified into two classes: random or structured. Random matrices perform better than structured matrices in error correction ability, but they suffer from the complex interconnect issues. Structured matrices simplify the interconnect complexity, especially with the decoder architectures in mind. The quasi-cyclic LDPC (QC-LDPC), among structured codes, are used in all the standards mentioned above. QC-LDPC codes are defined by an array of sub-matrices, which are N x N circulant matrices with at most one “1” per column or per row. The QC-LDPC structure guarantees at least n messages can be computed in a parallel fashion with simple interconnection.
The original LDPC decoding algorithm is Gallager’s iterative two-phase message passing algorithm (TPMP), which involves check-node update and variable-node update as two phase schedule. There are many variations and approximations that have been studied since TPMP, including the widely used sum of products (SP), min-sum (MS) and Jacobian based BSJR (named by its discoverers Bahl, Cocke, Jelinik and Raviv). Mansour and Shanbhag proposed the turbo decoding message passing (TDMP), which is sometimes also called as layered

The hardware architectures of decoders mainly have three types: full parallel, partial parallel and full series. The full series architecture is too slow and used hardly. Full parallel architecture has the fastest speed but cost too much. It often used for small code length [3]. Partial parallel architecture is widely used for the long code [4]-[6]. Because of the long code length, the hardware implementations suffer from the high parallel process, the complex interconnection, and the huge memory cost. Usually, the parallel degree is equal to the rank of submatrix in order to simplify the interconnection [5]. The interconnection can be done by shuffle network [7]-[9] to support different code rate. The huge memories are mainly used to save messages of variable nodes and check nodes. In order to achieve the high parallel process ability, dual-port memories are used in almost all current LDPC decoders. Compared with single port memories, the dual-port memories increase the area, power, demand more route wire resource, and make the control logic complex.
In this paper, we proposed one LDPC decoder with all single-port memories for LDPC codes in the DMB-TH standard. To get the purpose, first, we reduce the decoding computation complex degree and speedup the iteration process by using TDMP algorithm and normalized MS algorithm, which are described in the section II. And then for the hardware implementation, we use some efficient techniques to shorten the delay path and increase the process ability so that we can use only all single port memories to meet decoding requirements, as described in the section III. In the section VI, the related implementation results are given and analyzed. In the section V, conclusions are drawn.
II. LDPC DECODING
The QC-LDPC codes are adopted in DMB-T standard. Three kinds of code rates are used: 0.4, 0.6, and 0.8. The three parity-check matrices are composed by the same style submatrices to simplify implementations. Two types of sub-matrix are used: all-zeros matrix and circulant identity matrix with different cyclic shifts. The sizes of all sub-matrices are the same, 127×127. Since the code length is 7493 bits, the submatrix column numbers of the parity-check matrices is the same, 59 columns and the sub-matrix row numbers are 35, 23, and 11 for the three rates, respectively. TDMP algorithm classifies the three parity check matrix into three mhn sub-matrices, where m is equal to 35, 23, or 11 and n is equal to 59 in the DMB-TH. All check nodes can be partitioned into m groups and all variable nodes can be partitioned into n groups. After one check-node group processing is finished, the new check nodes messages (Rmn) are immediately used to update the variable node messages (Qn). However, in previous TPMP algorithm the variable nodes are updated only when the whole check node updating have been finished. By the means, the TDMP increase the decoding speed without performance degradation. One check-node group processing and the associated variable-node group processing constitute once sub-iteration. The whole m check-node groups processing constitutes once iteration. The BCJR [2] and offset MS [6] have also been used for the TDMP. In this work the normalized MS is applied to TDMP.

III. HARDWARE IMPLEMENTATION
A. Reusable and Configurable CVPU
Equation (1) and (2) are used to calculate the check node updating and the related operations include substation, sign bit continued multiplication, minimum value operation, and constant multiplication. The minimum operation in equation (2) can be realized by computing the least minimum and the second least minimum among the entire set of the messages and recording the index value related to the least minimum. When given the index for some check message, its value will be the least minimum if the index is equal to the recorded index or be the second minimum value if the index is not equal to the recorded index. According to the method, check messages Rmn of one row variable node can be saved as one data Rm. By the means some bits are saved. The bit format of the data Rm is shown in Fig. 1.

Figure 2. the CVPU architecture
R1 - R4 are registers for saving the variable node message and check message. The mux0 and mux1 select the right input data for the adder. Data preparation unit finds the minimum check message and preprocesses it as one input data of the adder. Data preparation unit also processes variable node message to provide another input for the adder. The SMF unit does the sign bit continued multiplication, finds the least minimum and the second minimum, records the related index, and forms the bit format for check message Rm. The mux2 select right data to R4. When once check message calculation begin, the R4 is initialized to maximum values for the least minimum and the second minimum. The x0.75 unit finishes the normalized operation because the normalized factor is 0.75 in this work. CVPU finishes once check node message calculation from up to down without feedback and saves the check node message to register R4. When CVPU does variable node

B. Shuffle Network
The cyclic permutation submatrices determine the interconnect network between the variable-node memory and the CVNUs. The network can be directly implemented by s s×1 multiplexers. Generally, the network can be optimized with the well-known s×s Benes network [7], which is capable of performing any sort of permutation over s data by 2log2N-1 stages 2×1 multiplexers, where s is a power of 2. However, since only circular shifts are required for QC-LDPC decoder, an s×s Banyan network (1/2 Benes network) is preferable, because it only requires log2N stages 2x1 multiplexers. To accommodate multiple code rates, various block lengths, and different sizes of submatrices, reconfigurable networks have been proposed. These reconfigurable networks [8][9] improved Benes network structure and configured them with LUT or on the fly by auxiliary network.
In order to get optimized shuffle network for the DMB-TH, Two valuable properties of submatrices can be utilized: One is cyclic shifts and the other is fixed size (127 x 127). So we can use more efficient network: the SS-CS network, as shown in [10]. One 5×5 SS-CS shuffle network is shown in fig. 3. C. the proposed decoder architecture
The proposed decoder architecture supporting three code rate is shown in Fig. 4. The iteration processing unit ldpcdec_iter includes 127 CVPUs and can process one submatrix in parallel. Two shuffle networks are used to route data between variable node memories SRAM_Q0-SRAM_Q7 and ldpcdec_iter. The eight parallel single port memories SRAM_Q0-SRAM_Q7 can output 127 variable node messages simultaneously to make the ldpcdec_iter works 100%. Six parallel single port memories SRAM_R0 – SRAM_R4 are used to save check node message in which one unit saves two check messages. The three Parity-check matrices are saved in one 930x13 bit ROM Rom_H. The 13 bit data tells the nonzero submatrices position and first one position of submatrices.

In order to get the all single port memories implementation, the decoding control unit ldpcdec_ctrl is dominated by three kinds of memories. Namely, for check message updating calculation, the decoding system read check matrix memory Rom_H by giving read signal (read_ Rom_H) and address signal (addr_ Rom_H). The data_h from Rom_H consists of the information of the submatrix position and first one position. The information of the submatrix position is used to find the address of related variable node message in SRAM_Q0-7. The information of first one position is used to configure the two shuffle network. Then the decoding system can read SRAM_Q0-7 to get the variable node message. The SRAM_R0-4 can be red earlier since it only need to red once during one submatrix row calculation. After the check message and variable node message are ready, ldpcdec_iter can calculate the new check node message. V. CONCLUSIONS
In this paper, one LDPC decoder with all single port memories is proposed. Considering the lower read-write speed single port memories than dual port memories, the decoder needs to reduce computation complex and shorten the critical path delay. To get the purpose, the LDPC decoder uses TDMP algorithm and scaling MS algorithm to reduce computation complex. And then it uses reusable and configurable CVPU to realize single cycle pipeline variable and check node updating calculation, uses optimized shuffle network to shorten the path delay, and uses memories dominated controller to avoid the read and write conflict of memories. The implement results shown the throughput is enough high and can meet the application requirements.
