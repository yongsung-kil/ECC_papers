Abstract—This paper presents an area-efficient LDPC decoder architecture for the China Multimedia Mobile Broadcasting (CMMB) standard. Several techniques are adopted to reduce memory size, including the min-sum algorithm (MSA), optimal bit-width quantization of the iterative messages and reduced complexity for the interconnect network. A partially parallel layered decoding architecture is proposed. The decoder for the rate-1/2 9216-bit code is implemented using the 90nm 1.0V CMOS technology. It achieves the decoding throughput of 48Mbps at 5 iterations when operating at 60MHz and the power dissipation is only 34mW.
I. INTRODUCTION
With the development of mobile communication and multimedia technologies, demand for iterative mobile multimedia services in a high-speed mobile reception environment has grown rapidly. The China Multimedia Mobile Broadcasting (CMMB) standard [1], ratified in 2006, is mobile television and multimedia standard developed and specified in China by the State Administration of Radio, Film, and Television (SARFT). The high data rate and high reliability features of broadcasting network require high-throughput forward error correction (FEC) codes with excellent error performance, making low-density parity-check (LDPC) codes [3] an ideal candidate.
LDPC codes can be effectively decoded by the beliefpropagation (BP) algorithm. Research work in LDPC decoder architecture can be divided into three categories: fully parallel method, serial method and partly parallel method [5]. Among these classes of LDPC codes, Quasi-Cyclic (QC) LDPC codes are suitable for hardware implementation because of their regularly-constructed parity check matrix. The QC-LDPC codes can provide comparable error-correction performance compared with random LDPC codes [7].
In this paper, we present an area-efficient LDPC decoder architecture and adopt various techniques to reduce the area consumed by memories and interconnects. Also, some techniques to reduced power consumption are also involved in our proposed architecture. The rest of this paper is organized as follows. III. AREA-EFFICIENT ARCHITECTURE DESIGN
A. Area-Efficient Architecture Design
The memory usually dominates the overall chip area and power consumption in an LDPC decoder [5] because iterative messages at each check nodes and variable nodes must be stored for the processing in the subsequent iteration. The memory grows in size and power consumption if the code length of an LDPC code is large, such as the code in CMMB with a code length of 9216 bits. Hereby, we adopt three memory-efficient methods to compress the amount of memory required in the decoding process.
The application of MSA simplifies the storage requirement at a check node. The CTV messages corresponding to one row are now saved as a single message composed of the following four elements: the minimum magnitude, the second minimum magnitude, the relative location index of the minimum magnitude and the signs of all of the CTV messages in this row. Suppose a row weight of 6 and a quantized bitwidth of 3 for each CTV message, the required memory to store the CTV messages can be reduced from m × 6 × 3 = 18m bits to m × (2 + 2 + 3 + 6) = 13m bits. Therefore, 28% memory can be reduced using the MSA.
Another straightforward method to reduce memory amount is to compress the quantized bitwidth of the iterative messages. However, reducing the bit-width of iterative messages introduces quantization noise and degrades the BER performance of the LDPC code.
As demonstrated in Section II, the LDPC codes in CMMB are specially optimized to enlarge the girth in the code, resulting in superior error correction performances. As illustrated in Fig. 2, floating-point performance of the code can achieve a BER of 10−7 at the SNR of 2.5dB, exceeding the BER requirement set by the CMMB standard which requires a BER of 10−4 at the SNR of 2.5dB. Therefore, several sets of quantization schemes with different number of integer bits and fractional bits are tested in order to find the minimum quantization bitwidth. It can be observed from Fig. 2 that

B. Interconnect Optimization
In order to alleviate the interconnect complexity existing in randomly constructed LDPC codes, structural codes are constructed by dividing the H matrix into small submatrices. QC LDPC codes are among such structural codes and each of the l × l submatrices is weight-1 or weight-2 identity matrix or its permutation, or the zero matrix. The large m×n switching networks for message passing between CNUs and VNUs are now replaced by small simple mb × nb switching networks. Hence, the interconnect complexity is scaled down by a factor of l2 compared with the directly mapping mechanism in fully parallel architecture. In each submatrix, the message flow can be easily handled by a simple modulop counter, which causes little hardware overhead.
The total required memory for the extrinsic memory and the decoded bits memory using conventional architectures is 396 kbits in our design.
IV. IMPLEMENTATION OF THE PARTIALLY PARALLEL DECODER ARCHITECTURE
In this section, a partially parallel LDPC decoder architecture and its implementation for the rate-1/2 9,216-bit codes in CMMB standard are presented. MSA, LDA and other techniques aiming at reducing the overall chip area are considered. Based on the fixed-point analysis in Section III, we choose 3 bits fixed-point representation (2 bits for the integer part and 1 sign bit) as the bitwidth for all of our iterative messages. In addition, the algorithm level power reduction technique is considered in the implementation.
A. Overall Partially Parallel Architecture
Fig.3 shows the overall decoder architecture for the rate1/2 LDPC codes used in CMMB consisting of weight-1 circulant submatrices and one weight-2 compound submatrix. The CTV memory is used to store the CTV messages corresponding to one row in the H matrix of the QCLDPC code. Instead of storing 6 different CTV messages corresponding to one row (because the H matrix has a row weight of 6), only one message is stored. The Recovery block is introduced to recover the individual CTV message from its compressed form. The APP Memory Banks are used to store the APP messages and load the intrinsic messages from the channel when initialization. The Read Network fetch necessary APP messages from APP Memory Banks according to the nonzero locations within one row. The Write Network saves updated the APP messages back into APP Memory Banks after the check node processing. V. EXPERIMENTAL RESULTS
In order to demonstrate the effectiveness of our proposed design schemes, we implement a rate-1/2 9216-bit CMMB LDPC decoder in 90nm 1.0V CMOS technology with 8layer metals. Synthesis results show that the decoder can operate at a maximum frequency of 181MHz, corresponding to 145Mbps maximum decoding throughput under 5 iterations. In order to save power, the decoder is mandatory to operate at 60 MHz, resulting in an estimated power consumption of 34 mW with a decoding throughput of 48Mbps.
Table II shows the decoder implementation results compared with other LDPC decoders with long code length, such as LDPC codes in DVB-S2 standard. From the table, we can see that the proposed decoder can reduce the chip area and power consumption.
VI. CONCLUSIONS
In this paper, we proposed an area-efficient LDPC decoder architecture and its implementation for CMMB standard.

The decoder for a selected rate-1/2 9216-bit LDPC codes is implemented on CMOS 90nm technology. The chip area is 2.6mm2 and power consumption is only 34mW for the decoding throughput of 48 Mbps for the BER of 10−4 at 2.5dB SNR. Therefore, the proposed LDPC decoder is suitable for portable digital TV applications.
