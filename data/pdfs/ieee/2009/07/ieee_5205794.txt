Abstract-Extrinsic tree decoding of low-density parity-check codes operates on modified, finite computation trees created from the Tanner graph of the code. The goal of the extrinsic tree algorithm is to maintain or im prove the performance of existing iterative decoders, while providing a decoding algorithm for which upper bounds can be computed. The extrinsic tree algorithm is examined, along with the design of parity-check matrices on which the extrinsic tree decoder performs well.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are capable of near capacity performance when decoded with iterative messagepassing decoders such as the sum-product (SP) and minsum (MS) decoders. Unfortunately, the performance analysis of these decoders is difficult when the Tanner graph of the LDPC code contains cycles. There are several models for estimating the performance of MS and SP including stopping sets [1], trapping sets [2], pseudocodewords [3], [4] and density evolution [5], [6]. Unfortunately, none of these models is capable of bounding the performance of iterative decoding on the binary input additive white Gaussian noise (BIAWGN) channel for a specific code.
Wiberg [7] introduced the computation tree as an exact model for the behavior of SP and MS decoders at any given iteration. He showed that the MS decoder chooses the minimal cost valid configuration on the computation tree and the SP decoder chooses the highest probability binary value at the root node of the computation tree after considering each valid configuration. Here, a valid configuration is any binary assignment to the set of variable nodes such that each check node is connected to an even number of variable nodes assigned a binary 1, and for the remainder of this paper, cost refers to the received channel log-likelihood ratio (LLR).
The large number of valid configurations on the computation tree makes it intractable to consider each valid configuration even after a small number of iterations has been performed. For this reason, Wiberg introduced the concept of deviations. A deviation is a set of variable nodes on the computation tree assigned aI, such that the root node of the tree is assigned a 1 and no proper and non-empty subset of variable nodes in the deviation form a valid configuration on the tree. It is possible to compute performance bounds on iterative decoding using only the set of deviations on the computation tree. However, once the computation tree is large enough to allow non-backtracking walks of length equal to or greater than the girth of the code, deviations become difficult to enumerate. Thus, it is of interest to formulate a decoder whose performance is comparable to that of SP and MS, and whose performance analysis is tractable.

Recently, tree pruning (TP) decoding [8] has demonstrated the potential advantages of decoding on modified computation trees. However, the depth of un-truncated TP trees grows linearly with the diameter of the graph and the size of the TP trees is too large for decoding of codes with long block length. Like TP decoding, the extrinsic tree (ET) decoder [9] operates on modified computation trees. The ET algorithm begins by iteratively constructing trees with the goal of reducing the probability of error at the root node. Once the trees are constructed, the ET decoder computes the minimum-cost binary configuration on each tree, and outputs the corresponding values at the root nodes.

II. EXTRINSIC TREE DECODING
The primary goals of extrinsic tree decoding are to decode on smaller trees so that the performance can be analyzed and to design trees that yield a low probability of error at the root node. In particular, the ET decoder aims to construct trees with large minimum weight deviations in order to achieve low bit error rates at high SNR.
The first step of extrinsic tree decoding for a given code is to build an extrinsic tree for each variable node. The ET algorithm decodes on N different trees, where N is the length of the code. After the extrinsic trees are built, decoding operations are performed from the leaf nodes up to the root node.

Since the extrinsic tree decoder requires the storage of trees for each variable node , the memory complexity is significantly greater than algorithms like min-sum. If the average number of nodes in each tree is A, the decoder is required to store N x A nodes. Experiments indicat e that it is reasonable to expect A to be greater than or equal to N . Thus, a memory storage complexity greater than N Z should be expected. Since an operation is required for each node , the computational complexity would also be expected to be greater than N Z• However, parallel operations are allowed for all nodes on the same level across all extrinsic trees. Therefore, the time complexity of decoding will be proportional to the maximum depth of the extrinsic trees which is much less than N .

Decoding of low-density parity-check codes is typically performed on a full-rank parity-check matrix of dimension (N - K) x N . However, recent work by Kelley and Sridhara [10] demonstrates that decoding performance can be improved by adding redundant rows to the parity-check matrix. While their work focuses on the linear programming (LP) decoder, their results suggests that iterative decoders might also be improved by the addition of redundant rows to the parity-check matrix. The following example examines the performance of MS and ET decoding on a small code when redundant rows are added to the parity-check matrix.
Consider the length 6, dimension 3 bin ary linear code defined by the full rank parity-check matrix H 3 x 6 . The matrices H 5 X 6 and H 7 X 6 were created by adding redundant rows to H 3 x 6 .

V. CONCLUSION
A decoding algorithm has been developed for LOPC codes that is non-iterative and whose analysis is tractable. ET decoding is shown to be capable of exceeding the performance of MS at high SNR. Results in Section IV demonstrate that

the performance of ET decoding can be improved by adding
redundant rows to the parity check matrix.
Future work includes designing a method for building
parity-check matrices with desirable extrinsic tree structure. It
is also of interest to explore variations on the criteria for mak-
ing additions to the extrinsic trees. Although, maximizing the
minimum weight of the deviations results in good performance
at high SNR, other criteria for extrinsic tree construction might
yield better performance at low SNR. Finally, ways of avoiding
the brute force enumeration of deviations at each step of the
tree-building process could greatly reduce the computational
complexity.
