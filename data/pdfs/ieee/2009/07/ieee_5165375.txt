Low BER Performance Estimation of LDPC Codes via Application of Importance Sampling to Trapping Sets
Enver Cavus, Member, IEEE, Charles L. Haymes, Member, IEEE, and Babak Daneshrad, Member, IEEE

Abstract—We introduce an Importance Sampling (IS) method that successfully simulates the performance of Low density Parity Check (LDPC) Codes in an AWGN channel at very low bit error rates (BERs). By effectively finding and biasing bit node combinations that are the dominant sources of error events, called trapping sets, the developed technique provokes more frequent decoder failures. Consequently, fewer simulation runs and higher simulation gains are achieved.
Index Terms—LDPC, importance sampling, very low bit error rate, trapping set, error floor.
I. INTRODUCTION
L OW BER estimation of LDPC codes exposes the error floor regions that are beyond the reach of traditional simulation and helps the feasibility studies that investigate the use of LDPC codes in applications requiring very low BERs. So far, no analytical tool is available to evaluate the performance of LDPC codes, and traditional Monte Carlo (MC) simulations cannot be performed at very low BER regions due to prohibitive simulation times. In order to avoid long simulation times, a fast BER simulation technique, called Importance Sampling (IS), can be applied. IS methods are widely used in digital communications [6]-[8] and have also been effectively applied to FEC schemes [9]-[12]. If a proper biasing scheme can be developed for the required application, IS methods promise significant simulation gains with little to no loss in the accuracy of the results.
Recently, two independent IS approaches have been proposed to address the performance evaluation of the LDPC codes at very low BERs [1][2]. In [1], performance of a (96, 48) LDPC code was demonstrated up to a BER of 10−14 and, simulation gains on the order of 105 were reported compared to a standard MC technique. In the second approach [2], a two phase adaptive importance sampling method called Dual Adaptive Importance Sampling (DAIS), based on Multicanonical Monte Carlo (MMC) simulations is introduced. In [2], the BER and WER curves for a (96, 48) LDPC code are reported. The results show simulation gains of 107 for BERs as low as 10−19 Unfortunately, the complexity of both techniques [1][2] depends on the codeword length and therefore, sufficient simulation gains cannot be obtained to estimate the performance of longer LDPC codes. IS scheme for low BER performance estimation of LDPC codes. In [3], the combinations of bit nodes that prevent the LDPC decoder from converging to the transmitted codeword are defined as trapping sets. Based on these trapping sets, a computational technique is developed to predict the error floor of a given LDPC code. However, this technique requires MC simulations in the proximity of the low BER region of interest to identify the effective trapping sets. This trapping set search step requires prohibitively long simulation times especially for medium to long codeword lengths. In this letter an efficient two step algorithm, based on the concept of trapping sets, is introduced to estimate low BER performance of LDPC codes. In the first step, a novel trapping search procedure is used to identify the dominant trapping sets and then these trapping sets are categorized into “classes” based on the total number of bits involved in each trap. In the second step, IS is applied to only one trapping set from each class and the total BER of the code is estimated by adding the BER contribution of each trapping class. Compared to previous approaches [1][2], the proposed IS method achieves higher simulation gains and captures more information than the one point extrapolation in [3]. When applied to a (2000, 1000) LDPC code the proposed method yields a simulation gain of 1014 at a BER of 10−20.
The remainder of this letter is organized as follows. Section II discusses trapping sets and the proposed algorithm in detail. In Section III, the proposed IS approach is applied to two different LDPC codes, and the BER performances are reported. Finally, Section IV concludes the letter.
II. PROPOSED APPROACH: IS WITH TAPPING SETS
In this section, the proposed approach is described in detail. For a review of Importance Sampling and its application to the LDPC codes, we refer the reader to [6]-[12] and [1][2], respectively. In order to develop an efficient algorithm, it is necessary to accurately define the error regions for an LDPC code. In this work, unlike the previous approaches in [1] and [2], an error set partitioning approach based on trapping set bits is considered. At low BERs, the error space of LDPC codes consists of trapping regions or states, once captured, the decoding algorithm is never able to recover from these states. In our approach, the error space is partitioned based on these trapping regions and IS simulation is tuned for these specific regions by introducing biased noise samples to the corresponding trapping set bits. Consequently, an IS simulation technique, which is negligibly dependent on the codeword length with higher simulation gains is developed. B. Classifying Trapping Set Classes
Once all trapping sets are found, IS can be applied to all trapping sets with equally strong bias values of (−1). Then, the total BER could be estimated through addition of the BER values of the individual trapping sets. Although this method might work well for the codes that have relatively few trapping sets, the efficiency of the algorithm could deteriorate as the trapping set count grows. In order to eliminate this problem, it is necessary to group all the trapping sets having the same size into a class, and then simulate one set from each class. As the BERs for the trapping sets belonging to the same class are approximately equivalent [3], simulating only one trapping set is sufficient. This will effectively reduce the complexity of the algorithm to be proportional to the number of dominant trapping set classes in the very low BER regions. The cumulative BER for each class is obtained by multiplying the simulated BER of one set by the total number of trapping sets in that class. Finally, the total BER of the code is calculated as the sum of the BERs of each class. Unlike the previous approaches, the proposed technique is minimally dependent on the code’s block size and allows performance evaluation in the very low BER regions with simulation gains on the order of 1010.

IV. CONCLUSION
This work presented a highly efficient IS method for performance evaluation of LDPC codes at very low BERs. A graph based search is proposed to identify the dominant trapping sets. By effectively biasing these dominant trapping sets, the proposed technique estimates performance of a given LDPC code at very low BERs.

Fig. 1 shows the BER performance of a (96, 48) LDPC code with a standard MC simulation and the proposed IS method. This code is very similar to the code studied in [1] and [2] and it has two codewords at the minimum distance of 6. As is observed from Fig. 1, the IS simulation closely matches with the MC results. The small performance gap at 7 dB is most likely due to insufficient number of error events encountered in the Monte Carlo simulation. The IS simulations become very efficient as SNR increases and attains an unprecedented simulation gain of 1011 at BER of 10−14. The proposed IS method can also be applied to a longer (2000, 1000) LDPC code. Fig. 2 and Table II present the IS and MC simulation results. Again, the IS results are almost identical to those obtained via MC simulations and the simulation gain increases rapidly as lower BERs are evaluated. In this case, IS provides a remarkable simulation gain of 1014 at a BER of 10−20. To the best of our knowledge, this is the first result, able to simulate the performance of an LDPC code of length 2000 down to a BER of 10−20.
