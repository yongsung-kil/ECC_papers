Analysis of LDPC Decoding Schedules
Eran Sharon, Noam Presman and Simon Litsyn School of Electrical Engineering, Tel Aviv University, Ramat Aviv 69978 Israel;
e-mail: {eransh.presmann.litsyn}@eng.tau.ac.il

Abstract-Schedule is the order of passing messages between vertices of the bipartite graph defining an LDPC code during decoding. Schedules may significantly differ in the rate of decoding convergence. New efficient generalized serial schedules are described and analyzed. They provide significant convergence rate speedup factors compared to previously known schedules. For the proposed schedules, combinatorial and probabilistic analysis is presented, explaining the fast convergence observed in simulations. Using it, LDPC ensembles for which significantly better convergence rates can be obtained are identified. Keywords: LDPC codes, iterative decoding, belief propagation, density evolution.
Message-passing schedules determine the propagation speed of information over the code's graph. Hence, they affect the decoder's convergence rate and its computational complexity. The convergence speedup factor can be translated into reduced processing logic and power by the same factor. Alternatively, under severe latency conditions, if hardware resources are limited, it can allow increased coding gain. Furthermore, scheduling can affect other aspects related to the decoder implementation. E.g. certain schedules can allow lower storage requirements and/or facilitate the implementation of efficiently pipelined decoder hardware architectures [14], [4], [10]. Such advantages can make LDPC codes an attractive candidate for high throughput, low power and low complexity communications and storage applications. In this paper we extend results of [10]. We theoretically analyze the convergence rate of generalized serial schedules (a.k.a layers based schedules), based on combinatorial and probabilistic approaches. The analysis explains and predicts the fast convergence observed in empirical results. It indicates that as the sparsity of the LDPC codes in an ensemble increases, asymptotically (in the code length) unbounded convergence acceleration factors (compared to the flooding schedule) can be achieved with these schedules. Based on the analysis we identify LDPC ensembles for which significantly faster convergence rates can be obtained. We further propose specific code constructions based on lifted graphs, which can efficiently support generalized serial schedules. We show examples based on regular LDPC codes where the generalized serial schedules achieve convergence acceleration factors of up to 6. Higher speedup factors are predicted by the analysis for irregular codes.
I. BACKGROUND
An LDPC code is a linear error correcting code defined by a sparse parity-check matrix [3]. Alternatively, it can be defined by a sparse bipartite graph G == (V U C, E), consisting of a

set of variable nodes V, a set of check nodes C and a set of edges E. A variable node v E V represents a bit in the codeword and a check node C E C represents a parity check constraint on the bits. Edges connect variable nodes to their constraining check nodes.
In certain parts of the paper we focus on LDPC codes which are based on lifted graphs [18], [2]. In these codes, a small base graph, called a protograph, serves as a blueprint for constructing the code's graph via graph lifting. Graph lifting is performed by duplicating the protograph Z times, where Z is the desired lifting factor. Then every set of Z edges corresponding to an edge in the protograph is randomly permu,ted. We define the protograph ensemble Q(G 'Z) as the ensemble of all LDPC codes generated as the Z-lift of the
protograph G, by randomly selecting all the edge permutations
from the set of permutations on {I, 2, ... , Z} uniformly. LDPC codes can be efficiently decoded using iterative
message-passing decoding algorithms. These algorithms operate on the bipartite graph representation of the code by iteratively exchanging messages between the variable and check nodes along the edges of the graph. The results derived in this paper can be applied to any message passing algorithm. However, in this paper we will mainly concentrate on the ubiquitous Belief Propagation (BP) algorithm [9],[3]. The BP algorithm, for example, defines certain computation rules for the variableto-check and check-to-variable messages. However, it does not specify the order by which the messages need to be sent. The order of passing the messages between the nodes is referred to as an updating rule or a schedule. Formally, a schedule (J" is a permutation of the directed edges of the graph. A messagepassing algorithm that implements (J" operates as follows: in each iteration, the algorithm traverses all the edges according to the schedule (J" and computes the corresponding message for each edge. In such fixed, non-adaptive, schedules the decoding complexity is directly proportional to the expected number of iterations required for the decoder to converge.
The standard message-passing schedule for decoding LDPC code, already presented by Gallager [3], is the flooding schedule [6], in which in each iteration all the variable nodes, and subsequently all the check nodes, pass new messages to their neighbors. Although simple, the flooding schedule is not optimal, for example: on cycle-free graphs the flooding schedule is shown to be the worst schedule, converging in the maximum number of iterations. More efficient serial schedules have been proposed in various papers, e.g. the variable nodes based serial- V schedule [19], [5], [16], [7], which operates as follows: in each iteration the variable nodes are traversed

in some order and for each variable node v E V, all the check-to-variable messages are sent to the node v and then all the variable-to-check messages are sent from the node v. The serial schedules enable faster propagation of information on the graph and results in convergence rates which are twice faster compared to the flooding schedule [16].
In [10] we introduced a new family of schedules which are generalizations of the serial schedules, called layers based schedules or generalized serial schedules. A variable nodes based generalized serial schedule is constructed as follows. Start with the empty schedule 0". Enumerate the variable nodes in some order: for each v E V create its layers graph L(C,v,d) (which is a subgraph of G that includes all the nodes having their distance from the node v being smaller or equal to d). Then delete the edges of L(C,v,d) from G, and concatenate to 0" the optimal tree schedule on L(C,v,d). The resulting schedule can be considered as a generalization of the serial- V schedule,
since for d == 1 we get exactly the serial- V schedule.
As shown in [10], the generalized serial schedules provide an accelerated convergence rate in comparison with the serial schedules. Furthermore, they can be efficiently implemented with the same complexity as the serial schedules, both in terms of storage and processing logic. They can be applied to any given LDPC code. Indeed, we constructed generalized serial schedules for rate 1/2 LDPC codes taken from the IEEE 802.11n Wireless LAN and the DVB-S2 standards, which provided convergence speedup factors of rv 2.3 and rv 2.55, compared to the flooding schedule, respectively.
Even though the generalized serial schedules can be applied to any LDPC code, designing specific codes specially tailored for a given generalized serial schedule has several advantages: 1) structured designs, resulting in structured schedules, can facilitate simpler, lower complexity, decoder design 2) structured designs achieve the highest benefit in terms of convergence rate from the schedule 3) such designs allow theoretical convergence analysis for determining the achievable convergence acceleration factor of the schedule without resorting to Monte-Carlo simulations. This also enables us to identify fast converging code ensembles under a given schedule; In this paper we focus on such specific LDPC code designs, for which we can partition the underlying bipartite graph into a set of balanced depth-d subtrees (i.e. cycle free subgraphs), such that each edge in the graph belongs to exactly one subtree. Note that any LDPC code can be partitioned into
depth-d subtrees when d == 1. However, special structure is required when d > 1. For a depth-d subtree partitionable graph

II. COMBINATORIAL CONVERGENCE ANALYSIS BASED ON
COMPUTATION TREE EVOLUTION
In this section we examine the evolution of the messagepassing algorithm's computation tree under different schedules. We extend the results of [16] to generalized serial schedules. Consider a message passed from a variable node
v to a check node c along the edge e == (v, c) of a given
graph G during message-passing decoding. Depending on the iteration number, this message is a function of a tree spanned according to the code's graph G from the edge e towards the variable node v and of the received channel observations for variables that are contained in the tree. We refer to this tree as the computation tree of e, and denote it by T~l) . Note that the computation tree for a graph with cycles may contain replicated nodes. Through the rest of this section we will assume computation tree without replicated nodes. Such assumption is asymptotically justified, since if the code is long enough, at a given iteration the computation neighborhood of a given edge is cycle free with high probability, as proved in various concentration theorems [12], [16].
The depth of the computation tree at a certain iteration serves as an indicator for the decoding progress. It is easy to see that the computation tree of the flooding schedule is always a balanced tree, and its depth is increased by 2 at each iteration. On the other hand, the computation tree of a serial schedule is not balanced and its depth with respect to each of its leaves grows by at least 2 at each iteration. Thus, the computation tree of an edge under a serial schedule always contains the computation tree of the edge under the flooding schedule. This implies that the convergence speed of the serial schedule is not slower than the flooding schedule.

IV. SIMULATION RESULTS
The theoretical analysis described in Sections II and III predicts significant convergence rate speedup for generalized serial schedu les compared to the flooding schedule . We validated these results by simulations. Figure 5 shows the convergence rate of several schedules for BP decoding of a length 44032 rate 1/4 regular (3, 4)-LDPC code over the BEe. This code was constructed by a l 28-lifting of a protograph with the structure described at the end of Section l. The imposed code structure does not seem to degrade the performance. The code 's waterfall starts at erasure probability of rvO.63, near the asymptotic DE threshold for a (3, 4)-LDPC ensemble which is 0.6474. With respect to convergence rate, we obtain acceleration factors over the flooding schedu le of rv 2 and rv 3.1 for the Serial-VI and Serial-V6 schedules, respective ly. This coincides with the values predicted by the analysis in Section II.
