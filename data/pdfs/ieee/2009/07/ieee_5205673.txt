Abstract-The number of minimum weight codewords is an important parameter to measure the potential performance of a linear block code. This paper studies the number of minimum weight codewords of simple and full-length array (SFA) LDPC codes. The notion of a cyclic shift closure is introduced, and it is shown that the set of minimum weight codewords of the code is partitioned by cyclic shift closures of minim um weight codewords. Each cyclic shift closure is generated from a special codeword. With the help of com puter experiment, general algebraic forms of the special codewords are determined. As the result, the numbers of minimum weight codewords of several classes of SFA-LDPC codes are clearly expressed by formulas.
I. INTRODUCTION
To evaluate the potential performance of error correcting codes is one of significant subjects in the coding theory. One common approach for this sake is to estimate the performance by using the weight distribution of the code. It is widely recognized that the weight distribution has strong relationship to the performance of the code, though, it is quite difficult to determine the weight distribution unless the code is very small or the code has useful mathematical structures. If the weight distribution is not available, then the number of minimum weight codewords help approximating the performance of the code. To determine the number of minimum weight codewords is easier than to determine the entire weight distribution, though, a naive computer search does not help except for small codes because the computational complexity of the search will be in the exponential order to the code length.
We study the number of minimum weight codewords of simple and full length array LDPC (SFA-LDPC) codes. The SFA-LDPC codes, which are refereed as array type LDPC codes in literatures[l], [3], [7], were proposed by Fan[l] by extending the notion of array codes to LDPC codes. With moderate code length and high code rate, SFA-LDPC codes show good performance compared to randomly constructed LDPC codes with the same code rate[l]. An SFA-LDPC code is defined by a well-ordered parity check matrix, but the structure of the code is not studied sufficiently. The minimum distances of the codes have been studied in [3], [7], [5], [6], but as discussed by MacKay[2], the minimum distance is not very useful for discussing the performance of long powerful codes. Related to the number of minimum weight codewords, Yang studied the set of supports of minimum weight codewords of CA(q, 3), CA(5,4) and CA(7, 4)[7], where CA(q, j) denotes the

SFA-LDPC codes defined for code parameters q and j (see the following section for the formal definition of CA(q, j)). From Yang's results, we are able to determine the number of minimum weight codewords of those codes. For general CA (q, 4), Mittelholzer has derived a lower-bound limit on the number of minimum weight codewords[4], but the bound is loose as he himself writes in his manuscript [4].
In this study, we investigate the number of minimum weight codewords of SFA-LDPC codes CA(q,4) and CA(q, 5) with general q. We first formulate a cyclic shift closure of a minimum weight codeword, and show that the set of minimum weight codewords is partitioned by such closures. It is also shown that a cyclic shift closure is obtained from a special codeword which works as a "generator", and that the number of minimum weight codewords has strong relation to the number of the generator codewords. For preliminary investigation, we constructed the set of generator codewords for rather small code parameters. The results, interestingly, suggest that there is a universal structure which depends on the code parameter j but is independent of the code parameter q. With careful analysis, general algebraic forms of the generators are obtained, which determines the number of minimum weight codewords of SFA-LDPC codes.

V. CONCLUSION
The number of minimum weight codewords of SFA-LDPC codes are studied. For CA(q, 3), we explicitly derived a closed formula which gives the exact number of minimum weight codewords in the code. We have not yet showed that the
formulas derived for cases j = 4 and j = 5 give the exact
number of minimum weight codewords, but they can be used as lower-bound limits of the number. The results of computer

search show that the lower-bound coincides with the real value as far as we have examined, and we strongly conjecture that the formulas give exact numbers of minimum weight codewords (excluding some exceptional small q).
