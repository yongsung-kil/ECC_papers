Abstract-On the binary erasure channel (BEC), the performance of low-density parity-check (LDPC) codes decoded by iterative algorithms is estimated by the small stopping sets in the codes. The size of smallest stopping set is called the stopping distance. Since it depends on the structure of paritycheck matrix for the code, we can construct the parity-check matrix by adding dependent rows so that the stopping distance is equal to the minimum distance. The stopping redundancy is defined as the smallest number of rows in such a parity-check matrix. In this paper, we propose a probabilistic algorithm for computing the stopping redundancy of LDPC codes. Additionally, we show numerical results of computing the stopping redundancy of (155,64), (504,252) and (1008,504) LDPC codes.
I. INTRODUCTION
Low-density parity-check (LDPC) codes and their iterative decoding algorithms are represented by bipartite graphs called Tanner graphs [1][2]. On the binary erasure channel (BEC), the performance of LDPC codes decoded by iterative algorithms is dominated by a certain structure in the Tanner graph, which is called a stopping set. Di et al. introduced the concept of stopping set and analyzed the average bit and block error probability of LDPC codes under iterative decoding on the BEC. The asymptotic stopping set distribution of LDPC code ensembles has been analyzed [4]. In the literature, the size of the smallest stopping set in a code was termed the stopping distance. It plays an important role in understanding the performance of the code under iterative decoding on the BEC, akin to the role played by the minimum distance for maximumlikelihood (ML) decoding algorithm. As one would like to maximize the minimum distance if ML decoding is to be used, one should try to maximize the stopping distance in the case of iterative decoding. However, there is a difference between the minimum distance and the stopping distance. While the minimum distance is a property of a code, the stopping distance depends on the structure of a parity-check matrix for the code. For a given parity-check matrix of a code, the stopping distance can be improved by adding dependent rows to the parity-check matrix. Since the support of every codewords is a stopping set, the largest stopping distance of the parity-check matrix could be not more than the minimum distance of the code. In [5], the stopping redundancy is defined by the smallest number of rows in the parity-check matrix for which the stopping distance is equal to the minimum distance.

Schwartz et al. introduced the stopping redundancy and derived general upper and lower bounds on the stopping redundancy of binary linear codes, as well as more specific bounds for Reed-Muller codes and maximum distance separable (MDS) codes [5]. The stopping redundancy of Reed-Muller codes was further studied by Etzion [6], and upper bounds on the stopping redundancy of binary linear codes and MDS codes were improved by Han et al. [7], [8]. Also, upper bounds on the stopping redundancy of finite geometry LDPC codes were derived by Liu et al. [9]. Further, in the extension of the concept of stopping redundancy, structural characteristic of parity-check matrices for which the stopping set distribution is equal to the weight distribution were shown in [10]. However, there is no result of computing the exact stopping redundancy of practical LDPC codes in the literature. The time complexity for computing the stopping redundancy exponentially grows as the size of the parity-check matrix is larger.
In this paper, we propose an efficient algorithm for computing the stopping redundancy of LDPC codes. It is a probabilistic algorithm to compute the stopping redundancy of a given LDPC code by constructing the parity-check matrix with dependent rows. In the algorithm, we apply a probabilistic procedure to finding the dependent rows covering the stopping sets whose size is smaller than the minimum distance. Additionally, we applied the proposed algorithm to Tanner's (155,64) LDPC code [2], MacKay's (504,252) and (1008,504) LDPC codes [11], and obtained results of computing the stopping redundancy of these LDPC codes in short time.

III. STOPPING DISTANCE AND STOPPING REDUNDANCY
Iterative decoding algorithms for LDPC codes, roughly speaking, work in a message-passing manner on the Tanner graph. Di et al. characterized the key object needed to analyze the performance of iterative decoding algorithms on the BEC [3].
Definition 1: Let G == (V U P, E) be a Tanner graph. A
stopping set S is a subset of V, the set of variable nodes, such that every neighbor of S is connected to S at least twice.
The size of the stopping set is the cardinality of S. We dispense with this graphical representation of stopping sets in favor of an equivalent definition directly in terms of the underlying parity-check matrix.
Definition 2: Let H == [hj]l::;j::;n be a parity-check matrix, and J be a set to contain all column indices of H. Then S c J
is a stopping set if the submatrix [hj]jES of H has no row of weight 1.
The stopping distance is defined by the smallest size of stopping sets, and bears a striking resemblance to the definition of the minimum distance of a linear code. Recall that the minimum distance of C can be defined as the largest integer d(C) such that every d(C) - 1 or less columns of Hare linearly independent. For binary codes, this is equivalent to saying that d(C) is the largest integer such that every set of d( C) -lor less columns of H contains at least one row of odd weight. The stopping distance is similarly defined as follows [5]:
Definition 3: Let C be a linear code, and let H be a paritycheck matrix for C. Then the stopping distance of H is defined as the largest integer s(H) such that every set of s(H) - 1 or less columns of H contains at least one row of weight 1.
The definitions of s(H) and d(C) immediately derive s(H) ~ d(C). Indeed, since the support of every codeword is a stopping set, any linear code C is satisfied with s(H) ~ d(C) regardless of the choice of H. Thus, given a linear code C,
the largest stopping distance could be not more than d(C), no matter how cleverly the parity-check matrix for C is constructed. Furthermore, given a parity-check matrix H for
C satisfied with s(H) < d(C), a parity-check matrix H for
C can be constructed by adding dependent rows to H. This observation leads to the following definition [5]:
Definition 4: Let C be a linear code with minimum distance d( C). Then the stopping redundancy of C is defined as the smallest integer p(C) such that there exists a parity-check
matrix H for C with p(C) rows and s(H) == d(C).

The stopping redundancy is well defined as follows. Let H* denote the parity-check matrix for C consisting of all nonzero codewords of the dual code C..L. Then, any set of d(C) - 1 or less columns of H* contains all the vectors of
weight one among its rows. It provides a trivial upper bound
on the stopping redundancy p(C) ~ 2r (C ) - 1 where r(C) is the redundancy of C.

In this paper, we proposed a probabi listic algorithm for
computing the stopping redundancy of LDPC codes. In the

finding the dependen t rows of the parity-check matrix covering the stopping sets of size d(C) - 1 or less. Additionally,
