Candidate Bit Based Bit-Flipping Decoding Algorithm for LDPC Codes

Abstract-A novel hard-decision decoding algorithm for lowdensity parity-check (LDPC) codes is proposed in this paper. This algorithm employs the correlation information among the column vectors of the parity-check matrix and syndrome vector for decoding. It does not require soft information, and has low decoding complexity. Simulation results show that the proposed decoding algorithm could provide an effective tradeoff between error performance and decoding complexity.
Index Terms-Low-density parity-check (LDPC) codes, bitflipping decoding.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] can be decoded using various decoding schemes [1]-[7] such as harddecisions, soft-decisions and hybrid decoding. While iterative message-passing algorithms could provide a near-capacity performance [7], it does require soft information and its decoding complexity is high. In contrast, Gallager's bit-flipping (BF) [1] decoding has low complexity and requires no soft information. In BF decoding, code bits with the numbers of unsatisfied checks larger than a fixed threshold are flipped. An improvement for BF is proposed in [3], where only the code bits with the largest number of unsatisfied checks are flipped. In [2], a simple taboo-based soft-decision decoding (TBSD) is proposed, where soft information is utilized. Soft information is also employed in weighted bit-flipping (WBF) [3], which provides a good tradeoff between the achievable performance and the associated decoding complexity [4]. A type of modified weighted bit-flipping (I-WBF) [4] and reliability ratio based weighted bit-flipping (RRWBF) [5], [6] provide enhanced performance over WBF. More details on TBSD, WBF, I-WBF and RRWBF can be found in [2]-[6].
In this paper, we present a candidate bit based bit-flipping algorithm, in which soft information and real addition are not required. Simulation results show that it can provide a good tradeoff between performance and complexity for some LDPC codes. In Section II, a candidate bit base bit-flipping decoding is introduced. Simulation results are presented in Section III. Section IV discusses the computational complexity of the proposed algorithm, and conclusions are given in Section V.

IV. COMPUTATIONAL COMPLEXITY
Compared to BF, CBBF is a little more complex, mainly due to computations needed in steps 4, 5 and 6. In step 4, integer counter increment operations can be used to calculate W m , so the complexity of step 4 is low. Let K F denote the number of
cand idate bits with U i = U m ax in each iteration. K F is no more than the number of code bits with U i > 0, and is much smaller
than N . The number of integer additions needed in step 5 is no more than K F (A - 1). In step 6, K F - 1 integer additions are needed. Note that addition of real numbers, which has a higher complexity than integer operation, is not needed for CBBF decoding, while it is neces sary for TBSD, WBF, 1WBF and RRWBF. So the computational complexity needed for each decoding iteration with CBBF decoding is low.
In TBSD, WBF, I-WBF and RRWBF, only one code bit is flipped in each decoding iteration. But in the CBBF decoding,
there may be more than one code bit with iu = U m ax and ri = min{ril in one iteration, and all of them are
flipped. So CBBF converges faster than these algorithms . Table I compares the average numbers of iterations of different algor ithm s for converged cases when decoding (1057 , 813) Gallager code. We find that the average number of iterations with CBBF is lower than that with TBSD, WBF, I-WBF and RRWBF. Through simulation, we also observed that for uncon verged cases, CBBF decoding will automatically stop when no candidate bits are left before it reaches the predefined maximum number of iterations where as other algorithms will complete only until the maximum number of iterations is reached. Since CBBF has a low complexity for each decoding iteration, the overall complexity of CBBF decoding is low.
