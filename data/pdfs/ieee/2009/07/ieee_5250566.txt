Simplified Receiver for LDPC-Coded Interleave-Division Multiple-Access (IDMA) System

Tao Peng, Xiao Yue, Xu He and Shaoqian Li
National Key Lab. of Communications University of Electronic Science and Technology of China
Chengdu, 610054, China
Email: ptyszalx@uestc.edu.cn

Abstract-Low density parity check (LDPC) codes can be applied to IDMA system, called LDPC-Coded IDMA System, which is presented recently for combing the advantages of LDPC and IDMA. However, the high complexity for decoding and detection remains a problem. In this paper, a simplified receiver structure is proposed for LDPC-coded IDMA system. The main idea is based on a switch between LDPC and repetition code decoding processing, so as to make a better tradeoff between computational complexity and system performance. Simulation results show that the proposed receiver structure can achieve similar system performance but with reduced complexity by simplifying the processing of LDPC decoding.
I. INTRODUCTION
A s one of the most important technologies in wireless communications, multiple access technology has been widely researched. One of the most excellent multiple access technology is code division multiple access (CDMA). A conventional CDMA system involves separate forward error correction (FEC) codes and spreading operations. However, according to the current theoretical analysis [1] [2], combining FEC and spreading operations, which releases all the bandwidth to channel coding, has been demonstrated to achieve optimal multiple access channel (MAC) capacity and to maximize coding gain. Moreover, the performance of CDMA systems is mainly limited by multiple access interference (MAl) and intersymbol interference (lSI).
In CDMA scheme, the different signature sequences are employed for user separation. In this case, interleavers can be employed to distinguish signals from different users. The principle has been studied previously and its potential advantages have been demonstrated [3]-[9]. Ref. [3] shows the advantage of introducing chip-level interleavers for multiple access channel (MAC). IDMA, regarded as a special form of CDMA by treating interleaving index sequences as

that the chip-level interleavers should be different for different users. The interleavers are generated for differently and randomly. These interleavers disperse the coded sequences so that the adjacent chips are approximately uncorrelated, which facilitates the simple chip-by-chip detection scheme.
LDPC codes are excellent error correcting codes and its performance is almost as close to the Shannon limit as that of Turbo codes [10]. In paper [11] [12], a LDPC-Coded IDMA system is presented and performance comparison between LDPC-Coded and Convolutional-Coded IDMA systems is obtained. The advantage of the LDPC-Coded IDMA systems is obvious compared to the conventional IDMA systems. But the cost of LDPC decoding algorithm (including iterative processing, transcendental operators, and pseudo-random message exchange) is very high. For example, if N is the block size and R is rate of LDPC code, the LDPC decoding involves approximately 6Nt floating point multiplies per iteration, and the total number of operations per decoded bit (assuming 20 iterations) is about 120tlR [13].
To alleviate the above problem, in this paper, a simplified receiver structure is proposed for LDPC-coded IDMA system. The main idea is based on a switch between LDPC and repetition code decoding processing, so as to make a better tradeoff between computational complexity and system performance. Simulation results show that the proposed receiver structure can achieve similar system performance but with reduced complexity by simplifying the processing of LDPC decoding.
This paper is organized as follows: Section II presents the model of a simplified receiver structure for LDPC-Coded IDMA System adopted in this paper; in Section III, computer simulation results are given and the performance of the proposed system is also analyzed; the conclusions are given in Section IV.

B. Receiver Structure A simplified receiver structure for LDPC-coded IDMA System is illustrated in the lower part of Fig.l, consisting of an elementary signal estimator (ESE) and a bank of K single-user a posteriori probability (APP) decoders (DECs) . The DECs contain LDPC decoders and repetition code decoders. The main idea is based on a switch between LDPC and repetition code decoding processing, so as to make a better tradeoff between computational complexity and system performance. The multiple access and coding constraints are considered separately in the ESE and DECs . This greatly reduces the complexity involved in the receiver. During the turbo-type iterative process, {eESE(x5)} and {eoEc( c5)} are extrinsic information about {x5} and {c5} , generated by the ESE and DECs . After appropriate

deinterleaving/interleaving, {eESE(c5)} and {eoEc(x5)} obtained from the extrinsic information, is used as the a priori information in the DECIESE. The receiver completes the turbo-type iterative process to update the extrinsic information at the ESEIDEC interface. Suppose that we start from the ESE . All {eoEc(x5)} are initialized to zero, assuming
that there is no initial a priori information. The DECs produce hard decisions on information bits in the final iteration. In the following , we will focus on the ESE and the DECs function .

The optimal LDPC code rates in LDPC-Coded IDMA System are middle rates between 0.4 and 0.5 over AWGN channel [13] . Therefore, regular (3, 6) LDPC codes are applied in system simulation and the rate of LDPC code is R=1/2. The spreading sequence should contain a balanced number of + 1 and -1 so as to ensure randomness. We use {+I,-I , + I,- l... } for all users simply. We assume AWGN channel without power control and no path loss, thus all channel coefficients are set to h(k) = 1,k = 1,2, ..., K . Iteration number of receiver is 5, and iteration number of LDPC decoding is 20 . The number of users is K. The length of the information block is N= 1000 bits per user. The length of spreading sequence is S= 16. Thus total encoder rate Rc (including LDPC encoding and spreading) is 1/32. The system throughput is RcxK=K/32, which is a measurement of the overall bandwidth efficiency.

IV. CONCLUSION
A simplified receiver structure is proposed for LDPC-coded IDMA System. The main idea is based on a switch between LDPC and repetition code decoding processing, so as to make a better tradeoff between computational complexity and system performance. Simulation results show that the proposed receiver structure can achieve similar system performance but with reduced complexity by simplifying the processing ofLDPC decoding .
