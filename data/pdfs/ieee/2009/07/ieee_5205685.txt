Abstract- We search for good regular quasi-cyclic (QC)
LDPC codes with J == 2 ones in each column. In order
to simplify the search, QC LDPC codes are represented in the form of tail-biting (TB) convolutional codes. A modified BEAST algorithm is used for finding the free distance (minimum distance) and the girth of both parent convolutional and block LDPC codes. Representations of known bipartite graphs and LDPC based on finite geometries in the form of TB convolutional codes are found. This approach is further
generalized for J == 3 QC LDPC codes. Examples of good
short LDPC codes with large girth and minimum distance are
given. For example, we present a rate 2/5 J == 3 QC LDPC
(225, 92)- code with girth 8 and minimum distance 24.
I. INTRODUCTION
During the last decade low-density parity-check (LDPC) codes introduced by Gallager [1] started to attract huge attention as low complexity competitors for turbo codes. Known LDPC codes can be divided into two large classes: irregular (random or pseudo-random) LDPC codes (see, for example, [2], [3] and references herein) and regular LDPC codes [4]-[6].
A regular binary (J, L)-LDPC code is determined by a paritycheck matrix containing J ones in each column and L ones in
each row. This class of LDPC codes contains graph-based and algebraically constructed codes. The most intensively studied subclass of regular LDPC codes are quasi-cyclic (QC) regular LDPC codes having the property that the cyclic shift of any codeword by p positions, where p is multiple of L, is a codeword [5], [6].
Typically in order to achieve near-capacity performances rather long codes are required. Most of known good LDPC codes have a random structure and they have been found by trial-and-error method. Known long pseudo-random codes perform better than QC or other algebraically constructed LDPC codes. However, the QC LDPC codes of moderate and even short lengths may have better performances than random codes ([5]).
Short LDPC codes themselves are not good enough to be used in communication systems, but they can be used as an intermediate step in some concatenated constructions (protograph codes [7], woven codes [8], etc.). This motivates a search for good short QC LDPC codes.

As performance measures of a code we consider the girth of the corresponding Tanner graph [9] and the minimum distance of the code. One of advantages of the structured LDPC codes is that the underlying construction guarantees a lower bound on the code girth and(or) minimum distance. For example, for the regular LDPC codes with column weight J built from
permutation matrices the upper bound (J +1)! on the minimum
distance is known [10]. Moreover, explicit constructions can lead to easier decoding procedures. The serious shortcoming of algebraic constructions is that they usually exist with relatively few combination of parameters.
In this paper we represent QC LDPC codes based on known bipartite graphs and QC LDPC codes based on finite geometries in the form of tail-biting (TB) convolutional codes.
Some new examples of QC LDPC codes with J == 2 and J == 3 in the form of TB codes are found. The main advantage
of this representation is a possibility to apply to LDPC codes low complexity searching, encoding and decoding techniques well developed for convolutional and TB convolutional codes [11], [13]. Moreover, the free distance and the girth of the parent LDPC convolutional code upperbound the minimum distance and the girth of the corresponding TB LDPC code, respectively. It is shown that bipartite graph-based codes represented in the form of TB time-varying convolutional
codes [11] have minimum distances larger than (J +I)! bound.

IV. SEARCHING FOR GOOD CODES

We start with a parent convolutional LDPC code and find its dfree and gfree which upperbound minimum distances and girths of the corresponding block LDPC codes. We reduce the problem of finding gfree to the problem of finding the free distance of convolutional code determined by (4). In order to find the free distance we apply the BEAST algorithm (Bidirectional Efficient Algorithm for Searching Trees) [13] properly modified to exploit the sparseness of parity-check matrices.
Finding girth of block LDPC codes and minimum distance for J == 2-codes is a computationally simple problem. It is reduced to searching for the shortest path in a Tanner bipartite graph and therefore it can be solved by using the Dijkstra algorithm [14], see also [18] and references wherein.
For codes with J > 2 finding the minimum distance is
far more complicated. In order to find the minimum distance of QC LDPC code we used the modified BEAST algorithm [13] which can be applied to any block or convolutional code. Next example explains the minimum distance search procedure which exploits the properties of LDPC codes.

Two vertices are connected by an edge if the corresponding columns have nonzero positions in the same row. Any codeword of the graph code corresponds to a subgraph of such a graph and the minimum weight codeword corresponds to a cycle of the graph. However, the inverse statement is not true, some cycles are not codewords.
To distinguish between cycles and cycles-codewords we assign a syndrome to each path of the graph. This syndrome is equal to the sum of columns corresponding to the vertices of this path. Therefore, the problem of finding dm in is reduced to the problem of finding the shortest path with the zero syndrome.
