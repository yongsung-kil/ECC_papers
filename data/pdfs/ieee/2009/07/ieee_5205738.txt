Decreasing Error Floor in LDPC Codes by Parity-Check Matrix Extensions

Orner Fainzilber, Eran Sharon and Simon Litsyn School of Electrical Engineering, Tel Aviv University, Ramat Aviv 69978 Israel;
e-mail: {omerfain.eransh.litsyn}@eng.tau.ac.il

Abstract-High error floors in optimized irregular LDPC codes limit their usage in applications that require low error rates. We introduce new methods for lowering the error floor of LDPC codes, based on enhancing the code's parity-check matrix with additional linearly dependent and independent parity-checks. We prove NP hardness of certain optimization problems related to proposed methods and provide upper bound on the number of parity-checks that need to be added. We show that the proposed methods can lower the error floor of the code significantly, by several orders of magnitude, at negligible or no rate penalty.
I. INTRODUCTION
The near Shannon limit performance of optimized irregular LDPC codes comes at the expense of increased error floor. This may significantly reduce the achievable coding gain for applications operating at very low error rates. Hence, error floor reduction techniques, have been the subject of intense research in recent years. In this paper, we introduce new methods, improving over the existing ones.
An LDPC code is linear codes defined by a sparse paritycheck matrix H. Alternatively, it can be defined by a sparse bipartite graph Q(H) == (V U C, E), consisting of a set of variable nodes V, representing the transmitted bits, a set of check nodes C, representing parity-check constraints over the bits and a set of edges E, connecting the constraints to the participating bits. We will use the two representations interchangeably. The performance of an LDPC code under iterative decoding is governed by certain topological structures in its graph. For the Binary Erasure Channel (BEC) these structures are well defined and known as stopping sets [4]. A stopping set is a set of variable nodes whose induced graph contains no singly connected check nodes. Iterative decoding fails if the erasures encompass a subset of the bits whose corresponding variable nodes form a stopping set. Hence, the code's error floor is determined by small stopping sets in its graph. For other channels, the structures responsible for decoding failures are harder to define. They depend on the specific decoding algorithm and channel. They are known by different names, e.g. trapping sets [17], near-codewords [11] and pseudo-codewords [5].
Several approaches have been suggested for reducing error floor: (I) Modifying the decoding algorithm, either based on incorporating knowledge of the harmful graph configuration, or based on altering the decoder's convergence dynamics, such that the probability to converge into a trapping set is reduced [3], [9], [20], [2]. The main drawback of such solutions is high decoder complexity and limited effectiveness. (II) Using code generation algorithms, which try to reduce the probability for low weight harmful graph configurations [7], [19], [15]. (III) Performing changes to an existing code, such as removal of small cycles or construction of a longer code from an

original code, e.g. via graph covers [8]. (IV) Changing the graph representation of an existing code (without changing the code), e.g. by adding redundant parity-checks that eliminate small stopping sets [14], [10], [21].
In this paper, we concentrate on the last three approaches. First we describe a method for finding low weight redundant (linearly dependent) parity-checks. These can be used for eliminating small stopping and trapping sets by enhancing the code's graph with additional check nodes. Since the resulting code does not change, there is no rate penalty. However, this approach is effective only as long as the smallest stopping sets are not codewords. In order to further reduce the error floor when it is determined by low weight codewords, we introduce a greedy algorithm for finding a minimal set of linearly independent parity-checks, which eliminate the low weight codewords, at the expense of negligible code rate penalty. We prove that optimally solving this problem is NP hard. We further derive an upper bound on the number of parity-checks required for eliminating a given set of stopping sets. Finally, we propose a way to incorporate the described methods with a code generation algorithm.
II. FINDING Low WEIGHT REDUNDANT Rows
Given an LDPC code C of length n and dimension k, we would like to eliminate small stopping sets in its graph by adding redundant rows to its parity-check matrix H. We will say that a redundant parity-check eliminates a stopping set S iff the corresponding check node is connected to S exactly once. Addition of redundant rows does not change the code, hence no rate penalty is incurred. For the same reason, such rows can eliminate only stopping sets that are not equal to the support set of a codeword. In [14] and [6], upper bounds on the number of redundant rows needed to ensure that the weight of the smallest stopping set is the minimum distance of the code, were derived. In [10], an algorithm that finds redundant rows, named "Genie-aided random search" (GARS), was described. GARS algorithm guarantees finding a redundant row that eliminates a specific stopping set S if the weight of S is smaller than the minimum distance. A variant of GARS algorithm that eliminates stopping sets is summarized in Table I. The GARS algorithm has two main disadvantages. First, it provides (with high probability) high weight redundant rows. Adding high weight redundant rows increases the density of the resulting parity-check matrix. This has an impact on both the decoding complexity and efficiency. Iterative decoding operates well when the graph is sparse. Second, GARS algorithm performs a local search targeting a specific stopping set and as a result may miss good candidates that have lower weight and eliminates more stopping sets. IV. COMBINING REDUNDANT AND INDEPENDENT Rows
Parameters of an LDPC ensemble (i.e. its degrees spectrum) can be optimized using tools such as Density Evolution [16] or EXIT charts [1]. Codes generated from such optimized ensembles tend to exhibit high error floor, preventing from applications operating at low error rates to benefit from their high coding gain. In order to lower the error floor of these codes we propose to combine the methods described in Sections II and III. The approximated BSSE algorithm can be used for eliminating low weight codewords and increasing the code's minimum distance, by adding few linearly independent rows, incurring negligible rate loss (as shown in Section V).
