Abstract
In this paper, a method for constructing Low-Density Parity-Check (LDPC) codes of good performance in terms of Bit Error Rate (BER) using a computer search algorithm is presented. First, LDPC codes and encoding along with its Tanner graph representation is described. Then, a detailed description of the proposed construction algorithm is given. A Modified Shortest-Path (MSP) Algorithm of lower complexity is applied to obtain a code graph of the desired structure. The proposed algorithm works directly on the parity-check matrix of the LDPC code. The parity-check matrix of the code is constructed column by column subject to the bit-degree and girth constraints. Using the proposed algorithm, both regular and irregular LDPC codes have been constructed. Furthermore, not only have the girth been improved but also the rate of LDPC codes. Finally, by simulation results the constructed codes were shown to perform well over an additive white Gaussian noise (AWGN) channel. Keywords: LDPC Codes, Modified Shortest-Path, Girth, Short Cycles.
1. Introduction
Low-Density parity-check (LDPC) codes, discovered by Gallager in 1962 [1], were recently rediscovered and shown to form a class of Shannon-limit-approaching codes [2]-[8]. LDPC codes are advantageous to the recently developed powerful Turbo codes [9], for example, in two ways: 1) absence of lowweight code words, 2) iterative decoding of lower complexity [10]. Moreover, Turbo codes exhibit an error floor around a BER of 10-6, no such phenomenon occurs in LDPC codes. An LDPC code is specified by its paritycheck matrix H, which is sparse, that is, it consists mainly of Os and a small number of Is [11]. In particular, an LDPC code is denoted (n, t., tr), where n denotes the block length, te denotes column weight of matrix H, and t; denotes row weight of each row with t.> t.. The rate of such code is (1 - tcltr).
As originally suggested by Tanner [12], LDPC codes are well represented by bipartite graphs in which one set of nodes, the variable nodes, corresponds to elements of the codeword and the other set of nodes, the check nodes, corresponds to the set of parity-check constraints which define the code. Regular LDPC codes are those for which all nodes of the same type have the same degree. Irregular LDPC codes were studied in [4],[13],[14].
Construction of LDPC codes is generally based on one of four approaches: graph theory [15], finite geometry [16], combinatorial design [17],[18] or experimental design theory [19]-[22]. Authors in [19] and [20] provide a heuristic method named bit-filling to search for LDPC codes with large girth. The matrix generated by bit-filling has uniform column weight but non-uniform row weight. Hence the LDPC codes constructed by bitfilling are not regular in strict sense. In [21] a computer search algorithm known as progressive-edge-growth (PEG) is proposed. PEG attempts to construct Tanner graphs having a large girth by progressively establishing edges between symbol and check nodes in an edge-by-edge manner such that the placement of a new edge has as small an impact on the girth as possible.
In this paper, an efficient yet a very flexible method for constructing Tanner graphs having a large girth using a Modified Shortest-Path (MSP) algorithm is presented. Given the code parameters, i.e. the number of symbol nodes n, the number of check nodes m, maximum number of ones in each row b, and the symbol-nodedegree sequence Ds, an algorithm for adding a new column to the parity-check matrix, while ensuring that the addition of such a column has an impact on the girth g as small as possible, is introduced. Both regular and irregular LDPC codes with the desired code rate or girth were constructed. As a bench mark, the constructed codes have higher rate than those of [19, 20] and comparable or higher performance than those of [21, 22].

The proposed algorithm uses a parity-check matrix of size MN for representing the code graph, whereas the original Shortest-Path (SP) algorithm [23] uses an adjacency matrix of size (M + N)2. Furthermore, in original SP algorithm distances are calculated for all (M + N) vertices, whereas for the proposed algorithm distances are only calculated for M vertices (check-nodes). Hence the storage requirements and computational complexity of the proposed algorithm is favorable.
Motivated by the above facts, the proposed algorithm tackles the problem of constructing LDPC codes with the highest possible rate and a given girth or equivalently constructing LDPC codes with the highest possible girth and a given rate. In section 4, the details of the proposed construction algorithm are described. Simulation results and analysis are given in section 5. Finally, section 6 concludes this paper.

4. Algorithm Description
Herein, a detailed description of the construction algorithm is given. The input parameters comprise the dimensions of the parity-check matrix, m, n, the desired girth, g, symbol-node degree, d.; check-node degree, d., and the maximum number of ones in each row, b. The output is a Tanner graph (or equivalently a parity-check matrix) of girth at least g and dimensions m, n. The rate of the code can be adjusted via R = 1 - dfd.. The main idea is to start with a graph consisting of a single symbol-node, a single check-node, and a single edge connecting them. Then, the parity-check matrix H is grown column-by-column filling each column with the appropriate check-node sockets. Check-node sockets are being added in such a manner that the resulting Tanner graph (parity-check matrix) contains no cycles of length less than g the whole process terminates when it becomes impossible to add a new column. This could be caused either by all symbol-nodes having a degree equal to dSj or the impossibility of adding a new socket without violating the girth constraint.

The main role of MSP algorithm is to visit every node in the graph in a Breadth-First Search (BFS) [23] manner and return each node's shortest distance to the source node of interest. In our case, since the paritycheck matrix is built column-by-column, the source node is the current symbol-node (or column) and it is required to know the shortest distance from all the graph's check-nodes to our symbol-node. Therefore, it is only needed to calculate the shortest distances of the possible check-nodes.
The significance of the MSP algorithm lies in the fact that the local girth at symbol node} (column}) is given by: 9j = Dmin + 1, where Dmin is the smallest shortest distance of candidate check-nodes. Hence the aim is to pick those candidate check-nodes which have the largest possible shortest distance. This means that the girth of the corresponding graph is being maximized at each local symbol-node.
The proposed algorithm belongs to a family known as "greedy algorithms", i.e. it is required to construct some optimal structure by adding one piece at a time, and at each step a decision is made about which piece will be added next by choosing, among all available pieces, the single one that will bring us as close to the desired goal as possible. In other words, the global problem of finding the best structure is solved by the local procedure of being as greedy as possible at each step.
In implementing the MSP algorithm, the short distances of all check-nodes are initialized to infinity (line 1). Thereafter, the current check-nodes in column} are put on a queue of vertices to be visited (line 2). And then repeatedly delete the vertex x at the head of the queue Q, and scan the adjacency list Adj(x) with inserting to Q all neighbors ofx (of the type check-node only) which have never been inserted to Q (lines 6-10). The first set of check-nodes added to Q are at a distance (from the current symbol-node) equal to 1 (line 5), the second set are at a distance equal to 3, etc. Finally, all the check-nodes shortest distances from the current symbol-node are obtained.
As a final remark on how to choose a specific check-node from all possible candidates it should be noted in general that the check-node with the largest shortest distance is to be picked. If there exists more than one check-node having the same shortest distance the principle of homogeneity is applied, i.e. always choose the check-node that is least used so far. Again, there may be more than one check-node that is least used (has the same degree). Then, a selection is made between one of two approaches: 1) simply choose one check-node at random, or 2) choose a check-node c, with the least index. The second approach is more deterministic, but the first approach is more appealing in case of high-rate codes. Throughout the paper, approach 2 is adopted.

6. Conclusion
A method for constructing Low-Density Parity-Check (LDPC) codes of good performance in terms of Bit Error Rate (BER) using a computer search algorithm has been proposed. The proposed method employs a Modified Shortest-Path algorithm of lower complexity. High girth and high rate regular and irregular LDPC codes have been constructed. The codes obtained were shown to perform well over an AWGN channel, making them potential candidates for practical applications.
Compared to other constructions, like random methods, PEG, or Bit-Filling algorithms, the proposed algorithm seems to improve both rate and performance ofLDPC codes. Moreover, the flexibility of code design allows for the application of any other optimization method. Studying the application of the proposed algorithm for multimedia communication over some specific communication channels may be the goal for future work.
