Efficient Shuffle Network Architecture and Application for WiMAX LDPC Decoders
Jun Lin, Zhongfeng Wang, Senior Member, IEEE, Li Li, Jin Sha, and Minglun Gao

Abstract—In this brief, a new algorithm that can efficiently generate all the control signals for the shuffle network used in flexible low-density parity-check (LDPC) decoders is proposed. Employing the proposed algorithm, the hardware complexity of the controller of shuffle networks using the Benes network structure can be significantly reduced. In addition, a low-complexity reconfigurable shuffle network architecture for flexible LDPC decoders is developed. Both the Benes network and the controller can be tailored to fit specific applications. Consequently, an efficient shuffle network for WiMAX LDPC decoders is presented. Synthesis results demonstrate that with the SMIC 0.18-μm complementary metal–oxide–semiconductor process, the total gate count of the proposed shuffle network is only 16 000. The area saving is between 26.6% and 71.1% compared to related works in the literature.
Index Terms—Benes network, error correction codes, lowdensity parity-check (LDPC) codes, shuffle network, very large scale integration, WiMAX.
I. INTRODUCTION
L OW-DENSITY parity-check (LDPC) codes have been considered as one of the most attractive forward correction codes due to their remarkable error-correcting capacity and highly parallel implementation characteristics. Structured LDPC codes such as quasi-cyclic LDPC (QC-LDPC) codes [1] have received significant attention for their efficient hardware implementation and comparable performance to computergenerated random codes. The parity-check matrix of QC-LDPC codes is composed of cyclic permutation submatrices that determine the interconnection between the check node processor units (CNUs) and the variable node processor units (VNUs). The QC-LDPC decoder is proved to be well suited for very large scale integration implementation [2]. Recently, QC-LDPC codes supporting different code rates and submatrix sizes have

been adopted in several industrial standards such as the standards for wireless metropolitan area networks (WiMAX, IEEE 802.16e) [3] and wireless local area networks (Wi-Fi, IEEE 802.11n) [4] and the Digital Video Broadcasting-SatelliteSecond Generation (DVB-S2) [5]. The WiMAX LDPC decoders need to support 19 kinds of modes specified in the mobile WiMAX system. The code block sizes vary from 576 to 2304. Generally, LDPC decoders are designed and optimized for a specific code with a fixed code length and submatrix size. However, for LDPC decoders that support modern wireless communication standards, multiple block lengths and submatrix sizes have to be supported. As a result, the shuffle network between CNUs and VNUs should be flexible to support various kinds of parity-check matrices.
The well-known N × N Benes network [6] is capable of performing any sort of permutations over N input data, with N being a power of two. The network consists of 2 log2 N − 1 stages of 2 × 2 switches. Generally, the shuffle networks designed for QC-LDPC decoders are optimized for a given size of permutation matrix with the Benes network. To achieve the desired permutation result, the control signals for those 2 × 2 switches must be carefully selected. These control signals are stored in a dedicated lookup table (LUT) [7], [8]. However, it could be inefficient and impractical when the size of the submatrices is quite large. To reduce the memory requirement, a more efficient shuffle network compared to the design in [7] and [8] has been presented in [9], where the shuffle network consists of two Benes networks. One actually shuffles data, while the other configures the first one using on-the-fly computation. Although this approach can reduce the hardware complexity compared with the direct implementation using M M -to-1 multiplexers when the size of submatrix is M , it still suffers from significant hardware overhead. A novel controller design of barrel shifters for reconfigurable LDPC decoders using the Benes network structure was most recently presented in [10], in which control signals for switches are computed on the fly. However, those signals for switches within one stage are generated in serial, which increases the overall latency of the shuffle network.
In this brief, we propose a new reconfigurable shuffle network for flexible QC-LPDC decoders using the Benes network structure. It can support all the cyclic shifts for submatrices with a size smaller than the number of inputs of the Benes network. We also present an optimized shuffle network design using the proposed shuffle network structure, which can be applied to WiMAX LDPC decoders.

The proposed CSG module outperforms the controller pre-

sented in [10] in the following aspects.
1) All of the control signals for one stage of a Benes network are generated in parallel. The critical path of the SG module is shorter. The critical path of the controller in [10] is TDECODER + (K/2) · TOR + TAND + TXOR, where K is the size of a sub–Benes network. K can be very large in general. In contrast, the critical path of the proposed controller is TDEC + TNOT + TXOR + TOR + TMUX, where TDECODER and TDEC are almost the same.
2) The hardware complexity of the proposed SG module is lower in general, since we only consider about the maximum input number, which does not need to be a power of two. We can tailor the SG module to meet a specific requirement with minimum hardware cost.

V. CONCLUSION
In this brief, an efficient algorithm used to determine the control bits of a Benes network under cyclic shift mappings has been presented. A general architecture of the shuffle network using the proposed algorithm and an optimized shuffle network for WiMAX LDPC decoders have been proposed. From the hardware synthesis result, it has been shown that the proposed shuffle network for WiMAX LDPC decoders is very area efficient compared to previously published works. The method presented in this work is also applicable to the design of shuffle networks that are compliant to other standards such as IEEE 802.11n and DVB-S2, as well as general QC-LDPC decoders.

For each of the 4 × 4 Benes networks, one of those five 2 × 2 switches has only one useful input, and another has only one useful output. As a result, these two switches can be optimized, as shown in Fig. 8(b). By this kind of optimization, 32 switches can be saved in stage 5, and 16 switches, on the average, can be saved in stage 7, as shown in the gray part in Fig. 7. No modification has been made in stage 6. The optimized Benes network used in the shuffle network for the WiMAX application only has 11 stages. The number of input ports and output ports are both 96.
For SMIC 0.18-μm CMOS synthesis results, the reformed Benes network for WiMAX LDPC codes occupies 0.132 mm2 at 300 MHz. A complete 128 × 128 network occupies 0.238 mm2 at 300 MHz. The controller part of the proposed WiMAX shuffle network is composed of PG modules and CSG modules that can be implemented in the same way as discussed in Section III. For each Div module belonging to the last stage of PG, the width of inputs IN and SN should be expanded by 2 b without changing the original values. Both the PG and CSG modules can be optimized according to the proposed algorithm to achieve the minimum area consumption. With SMIC 0.18-μm CMOS technology, our synthesis result shows that the optimized controller occupies 0.028 mm2 at 300 MHz.

For the 0.18-μm CMOS synthesis result, the proposed shuffle network occupies 0.160 mm2. The area consumption under the 0.13-μm CMOS process is 0.080 mm2. The area saving is 26.6%–71.1% compared to previously published works. To support IEEE 802.11n (Wi-Fi) LDPC decoders, both the controller and the network should be modified. The extra area consumption caused by making the shuffle network compliant to Wi-Fi LDPC decoders is relatively low. The increment of area consumption is around 10%. We only need to modify the control logic for the first and last two stages and add nine more switches.
