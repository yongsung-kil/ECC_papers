Extended PEG Algorithm for high rate LDPC codes

Zhiheng Zhou1, Xiangxue Li2,+, Dong Zheng2, Kefei Chen1, Jianhua Li2 1 Department of Computer Science and Engineering, Shanghai Jiao Tong University
2 School of Information Security Engineering, Shanghai Jiao Tong University

Abstract—Progressive Edge-Growth(PEG) Algorithm is a good candidate to generate Tanner Graphs with a large girth by establishing edges or connections between symbol and check nodes in an edge-by-edge manner. In this paper, we propose an extended PEG algorithm for constructing Low-Density ParityCheck (LDPC) codes with very high rate when given a lower bound of girth. Simulation results show the bit error rates of constructed LDPC codes with very high rate or large girth.
I. INTRODUCTION
Low Density Parity Check codes were invented by Gallager in 1963 [1]. In a long period of time, these codes didn’t attract much attention. In 1996, MacKay and Neal rediscovered LDPC codes and found that these codes showed good performance in transmitting messages over a noisy transmission channel [2]. They illustrated LDPC codes could nearly reach Shannon limit when decoded by beliefpropagation (BP) or sum-product algorithm (SPA). Because of their good performance, low decoding complexity and easy implementation, LDPC codes is focused on more and more.
One interesting problem is how to construct LDPC codes with a finite block length and a low decoding error floor. In [1], Gallager proposed a LDPC code represented by a sparse parity check matrix, but his method is only fit for regular codes and not easy to implement when block length is very large. In 2004, Hu and Arnold proposed PEG algorithm for constructing LDPC code with moderate block length and large girth [3]. Simulation results showed that the decoding error floor of code constructed by this algorithm is quite low.
It is known that if the block length tends to be infinite, there is a signal-to-noise ratio (SNR) threshold, below which the code cannot be decoded successfully. Given degree distributions of check nodes and symbol nodes of a Tanner Graph, one can compute this threshold by using a powerful tool called density evolution [4].
The original PEG algorithm answered this question: given M, N and symbol node degree sequence Dsi (i = 1, 2, 3..., N), how to construct a M × N parity check matrix with largest possible girth g. And in this paper we answered another one: given M, lower girth bound g and symbol node degree sequence Dsi (i = 1, 2, 3..., N), how to construct a M × N parity check matrix with highest possible code rate. When an LDPC code is used to transmit messages in a channel where

the signal-to-noise ratio is much higher than its threshold, decoding error floor becomes not so important. In this case, we attach more importance to the code rate than decoding error floor. Thus, constructing high rate LDPC codes has its own practical significance.
The problem of constructing high rate LDPC codes comes to be another question: how to generate an M × N parity check matrix with given column weight si(i = 1, 2, 3, ..., N), given M and largest possible N. And this matrix has girth g, which is a even integer. In [5], MacKay gave some high rate LDPC codes that he found. In this paper, we propose an extended PEG algorithm for the sake of constructing higher rate LDPC codes.
This paper is organized as follows: Section II introduces some basic definitions of LDPC codes. Section III describes the extended PEG algorithm which can construct high rate LDPC codes. Section IV gives some simulation results and compares constructed codes with other high rate ones. At last, we conclude our paper in Section V.

II. BASIC DEFINITIONS
An LDPC code of block length N and dimension N − M can be represented by an M × N parity check matrix H. A bipartite graph with M check nodes on one side and N symbol nodes on the other can also represent an LDPC code, and this kind of graph is called Tanner Graph. Formally, Tanner Graph is defined as a 2-tuples {V, E} where V is the set of nodes, and E is the set of edges. The ith row (1 ≤ i ≤ M) of row weight ai in matrix H correspond to the ith check node ci in Tanner Graph, and the jth column (1 ≤ j ≤ N) of column weight b j in matrix H correspond to the jth symbol node s j in Tanner Graph. Similarly, ai is equal to check node degree dci , while b j is equal to symbol node degree dsj . Here is an example:

For example, when operating SNRs of the channels are always very high, we can use high rate LDPC codes to reduce the transmission cost. And when operating SNRs vary within a certain range, usually we construct rate-compatible LDPC codes which can change their code rate when messages transmitted in channels with different operating SNRs. If we use extending to generate RC-LDPC codes, first we need a base code which must have a high rate [6]. So in both cases, finding a method which can yield codes with given girth and very high code rate is of great importance.
For this purpose, we extend PEG algorithm, as showed in Algorithm 1. First, we try to maximize the cardinality of symbol nodes set N by adding symbol node to current graph. In this adding procedure, we always check current girth gi, if gi < g we stop adding, where g is the lower bound of girth we defined, which is an even integer bigger than 4.
Second, if gi < gi−1 we delete the edges incident to si and si−1, then redo the connection procedure for these two nodes. Because the check nodes linked to symbol node si are always randomly selected, the new connections are different from old ones. This procedure repeats until gi = gi−1 or stops after iter iterations, where iter is an integer predifined. In this way, we can slow down the decreasing of current girth gi.

When LDPC code length tends to be infinite, we can find
a noise threshold below which the decoding process can
succeed, and above which can not. Density Evolution can
be used to find this threshold. Relative work for sum-product
decoding on memoryless channels can be found in [4] and
[7]. The main idea of density evolution is as follows:
Messages passed in the Tanner Graph are random vari-
ables. The calculations of bit error probabilities under
message-passing decoding are functions of random variables. P(l) is defined as the probability density function (PDF) of message passed in lth iteration. Then P(l+1) can be calculated from P(l) under certain circumstances. At last, the bit error probability after l iterations P(el) can be represented by this formula:

computed by giving degree distribution of edges in Tanner

Graph of an LDPC code. By computing this for a range of variance σ, we can find that the threshold of a given degree

distribution for message-passing decoding is the supremum of the channel noise value for which P(el) goes to zero as the
iteration number increase indefinitely. Thus the threshold can

Because of the complexity of computing PDF for decoding, in our case, we just use LODE(LDPC Online Density Evolution) and degree distribution we calculated above to get thresholds of two codes [8].

The threshold represented by SNR of MacKay’s code is 4.25(dB), and threshold of ours is 4.62(dB). From here we can see that compared to MacKay’s code the error correcting ability of our proposed code is a little lower, we sacrificed error correcting ability for high code rate.

V. CONCLUSION
Up to now, PEG algorithm is one of the most successful approach to construct LDPC codes with given dimensions of matrix and degree distribution. At the same time, it can maximize girth of Tanner Graph corresponding to constructed codes, thus lower the decoding error rate. We extended this algorithm, and found it can construct LDPC codes with given girth and make code rate as high as possible. Although these codes do not have very good performance in AWGN channel when SNR is very low, they still can be used in transmission where transmission rate is of greatest importance.
