Low Complexity DVB-S2 LDPC Decoder

Botao Zhang, Hengzhu Liu, Xucan Chen, Dongpei Liu, and Xiaofei Yi Institute of Microelectrics & Microprocessor, School of Computer Science
National University of Defense Technology Changsha, HN, P.R.China
E-mail : {botaozhang, hzliu, xcchen, dongpeiliu, xfyi}@nudt.edu.cn

Abstract—The decoding complexity is the main issue in designing DVB-S2 Low Density Parity Check (LDPC) decoder. This paper proposes a low complexity decoder, which is based on a new row message passing Offset Min-Sum algorithm. The proposed algorithm can reduce the complexity of the decoder with no performance loss. The simplified node update units based on the proposed algorithm and the memory organization optimized across different code rates play the key role in reducing the complexity. The synthesized area of the decoder is 9.6mm2 in Chartered 90nm COMS technology. When the code rate is 9/10 and the work frequency is 320MHz, the net throughput of the decoder is 998Mbps.
Keywords- LDPC, DVB-S2, RMP-Min-Sum, low complexity decoder
I. INTRODUCTION
The newly developed digital video broadcast standard, DVB-S2 adopts the concatenation of BCH code and Low Density Parity Check (LDPC) [1] code as the Forward Error Correction subsystem [2].The decoder of DVB-S2 LDPC code is much more complex than other types of LDPC codes, mainly caused by its long code length. Another difficult issue in designing DVB-S2 LDPC decoder comes from the fact that the DVB-S2 standard supports two different frame lengths (i.e. 16200 bits, which is called as the short frame, and 64800bits, which is called as the normal frame) and a set of different code rates (i.e. 1/4, 1/3, 2/5, 1/2, 3/5, 2/3, 3/4, 4/5, 5/6, 8/9, and 9/10).
In order to reduce the complexity of decoders, many simplified decoding algorithms and the corresponding decoders are proposed recently. Frank Kienle et al. propose a decoder supporting all the frame lengths and code rates of DVB-S2 in [3]. They adopt standard message passing (SMP) BP algorithm, and simplify the update scheme of the check bits. However, they pay little attention to the large memory occupied. Marco Gomes et al. improve the work by re-constructing the paritycheck matrix and decreasing the parallelism factor of the LDPC code in [4]. Their solution really decreases the silicon area of decoder, especially the area of network logic, but it also decreases the throughput of the decoder. In [5], Arthur Segard et al. use Offset Min-Sum algorithm to reduce the operations of updates of nodes and the memory occupied. But they pay little attention to optimizing the memory organization across different code rates.
In this paper, we adopt Offset Min-Sum algorithm as the decoding algorithm, which can reduce the complexity of the operation-units. As the Bit Error Ratio (BER) performance of Min-Sum algorithm is not very well, we combine it with a new RMP scheduling scheme, and propose a new algorithm which

can be categorized into non-flooding algorithm. Here we called RMP (Row Message Passing)-Min-Sum algorithm. We then build a new decoder based on this algorithm. This decoder optimizes the memory organization across different code rates, and also reduces the complexity of check node update.
The rest of the paper is organized as follows: In Section II we present the RMP-Min-Sum algorithm. The architecture of the decoder is presented in Section III. Section IV presents the numerical results. We conclude the paper in Section V.
II. PROPOSED DECODING ALGORITHM
A. Existing Decoding Algorithms
The first LDPC decoding algorithm proposed in the literature is the SMP BP algorithm [7], which is also called the sum-product algorithm. It is an iteration algorithm with high performance. As an improvement to the BP algorithm, the log BP algorithm replaces the huge multiply operations in the BP algorithm with ADD operations, thus significantly decreases the complexity of the BP algorithm
The performance of the BP algorithm is excellent. However this algorithm adopts a non-linear function to update the check nodes. Designing the non-linear lookup table is a very challenging task. To overcome this challenge, in the Min-Sum algorithm the non-linear function is replaced by the combination of ADD and MIN operations, which successfully reduces the operation of check node update [8]. However, as a disadvantage, Min-Sum algorithm also introduces negative gain to the BER performance, especially under low code rates. Thus, Normalized Min-Sum and Offset Min-Sum algorithms are then proposed to improve the BER performance by properly increasing the complexity of check node update.
B. RMP-Min-Sum Algorithm
In [9], two kinds of message passing schemes are compared: SMP scheme and RMP scheme. Algorithms in Section II.A are all based on SMP scheme. In SMP based algorithms, all the variable nodes and check nodes are updated synchronously in each iteration. On the contrary, in the RMP based algorithms, a variable node is updated immediately after its corresponding check nodes, such that the variable node can utilize the already updated information in the same iteration. In this sense, one iteration of the RMP based algorithms is equal to multiple iterations of the SMP based algorithms. Thus, the RMP based algorithms converge faster than the SMP based.

III. RMP-MIN-SUM LDPC DECODER ARCHITECTURE
Our proposed decoder is based on RMP-Min-Sum algorithm. Fig. 2 gives the high-level architecture of the RMP-Min-Sum LDPC decoder. It is seen that the decoder is composed of two parts: the controller and the Process Element Array (PE Array).
The controller generates the control signals for PE array through Finite State Machine, Shifter RAM, VPIAddr (RAM V Information Part Address) RAM, VAGU2 (RAM V Address Generation Unit2), and CAGU (RAM C Address Generation Unit). Shifter RAM generates the shift control signals for the barrel shifter. VPIAddr RAM with its attached logic is one of the two Address Generation Units (AGU) of RAM V. Unlike the decoders proposed in [3] and [4], which use ROM to store these signals, this decoder uses RAM instead. Since DVB-S2 includes 21 sets of code rates and frame lengths, the solutions given by in [3] and [4] need 21 sets of Addr ROM and Shifter ROM. Our proposed decoder uses double buffers to support Adaptive Coding and Modulation (ACM), so it can save much memory area. The VAGU2 and CAGU will be described in Section III.A.
The depth and width of Shifter RAM and VPIAddr RAM should satisfy all the code rates and frame lengths. Accordingly,
the depth of the RAMs should be 648 (the maximum of q * Dc ,
where q is the code rate dependant parameter, and Dc is the degree of check nodes). The width of Shifter RAM is set to 9, as the maximum data within is 359. The width of VPIAddr RAM is 8, as the maximum data within is 135.

The parallelism factor of this decoder is 360, so it contains 360 sets of memory banks. Every memory bank includes one RAM V, which is for the LLR of variable nodes, and one RAM
C, which is for the data structure of Λ(mkn) referred above. The organization of RAM V and RAM C in the proposed decoder is shown in Fig. 3. The numbers in Fig.3 are the address indexes where the data is stored. M is the number of information bits. It is set to 32400 when the code rate is 1/2 and the code length is 64800 bits. The information bits and check bits are all stored in RAM V. The information bits are stored in the first part of RAM V, of which the address signals are generated by VPIAddr RAM(first Address Generation Unit). The check bits are stored in the second part of RAM V, of which the address signals are generated by VAGU2, and a MUX is designed to arbitrate which AGU is working. The boundary of the information bit part and check bit part in RAM V is determined by different code rates and code lengths. Further, the memory organization of these two types of bits is also different. The information bits are bank first, while the check bits are depth first. The depth of RAM V is 180.

In designing RAM C, the major concern is the utilization of the RAM space. If every check node occupies one row of RAM C, then the depth should be 135, and the width of RAM C should be designed according to the maximum of the check
nodes’ degree dc . Suppose the quantization of the minimum
and the second minimum value of | λ n − Λ(mkn−'1) | is in Q bits, and every check node occupies one row, the width of RAM C comes out to be 2Q + ⎡⎢log2 Q⎥⎤ + 30 . In this sense, the utilization of the RAM space will be un-acceptable, e.g., when the code
rate is 1/4, q is 135, dc is 4, so 86.7% of the bits for sign
vectors are wasted. Fortunately, we found that that when the q
is large, meanwhile the dc will be small, and vice versa. So we
can optimize the memory organization across different code

rates to reduce the memory occupied. The optimization methods vary from the locations of the check nodes. The worst one is that each check nodes occupies one row, so the width and deep of RAM C will be maximized, and then the best one is that the information structures of check nodes are all stored in order tightly, without any memory waste. Since the number of bits needed for storing the check nodes varies with the code rates, and at the same time the width of RAM C is fixed, in many cases, lots of check nodes are stored across the row boundary of RAM C, which increases the complexity of AGUs. Thus, the memory area saved might be even less than the area increased by the complex AGUs. In our proposed decoder, we balance the complexity of the AGUs and the utilization of the cells in RAM C. That is, assign two check nodes to one row under code rates 1/4, 1/3, 2/5, 1/2, 3/5 and 2/3, or assign every check node to one row under code rates 3/4, 4/5, 5/6, 8/9 and 9/10. Fig. 3 shows the memory organization of RAM C at 1/2 normal frame. It is seen that every row of RAM C contains two check nodes, so the number of rows occupied is 45. The address of RAM C is generated by CAGU. When a frame is decoded, CAGU is reconfigured according to the ACM information received from the demodulation subsystem.

Since the read and update operations of the RAM C in RMPMin-Sum algorithm do never occur simultaneously, RAM C is implemented by single port RAM. The message passing scheme adopted is check node based RMP. When a check node is updated, its connected variable nodes (including the information bits and check bits) are updated sequentially, with each update operation composed of one read and one write operations. The update operation is pipelined, so the read operation of the next update operation executes with the write operation of previous update operation. Thus, RAM V is implemented by double ports RAM. However, memory conflicts may still exist when the objectives of two successive update operations are the same variable nodes. To avoid the conflicts, we import read bypass logic into the decoder, such that when a conflict is detected, the write port data of the previous operation will be bypassed to read port in the next operation.
The decoder was originally designed using six-bit uniform quantization, which includes one sign bit, three integer bits and two decimal bits. However, simulation results show that the decoder does not work well in low SNR channels, due to the small represent range of the six-bit uniform quantization. IV. NUMERICAL RESULTS
The proposed decoder is implemented in Verilog HDL. The decoder supports all the code rates and code lengths specified in the DVB-S2 standard. The decoder is synthesized with the Synopsis Design Compiler based on Chartered 90 nm CMOS technology standard cell.

Table I compares the proposed decoder with four existing decoders. The memory occupied of the proposed decoder is small than the decoders proposed in [3] and [5]. The memory occupied by the decoder proposed in [10] is 3.18Mb, larger than the proposed decoder, although BCH decoder is included, but the memory cells occupied by BCH decoder are much less than LDPC decoder. The effective memory organization does some help in reducing the area of decoder. The area of the decoder proposed is fewer than the decoders proposed in [5] in the same CMOS technology. The area of the decoder proposed in [6] is fewer than the decoder proposed in this paper mainly because of the CMOS technology and the parallelism. As the parallelism of the decoder proposed by [6] is only 1/4 of this decoder, the net throughput of the decoder proposed by [6] is smaller than the decoder proposed in this paper, even though the working frequency of the first one is higher. The estimate area of the proposed decoder in 65nm COMS technology will be 4.8mm2, less than the decoder proposed in [10].

V. CONCLUSION
In order to reduce the complexity of DVB-S2 LDPC decoder, this paper studies the decoder algorithms of LDPC; proposes RMP-Min-Sum algorithm, which gets a gain of 0.6dB over BP algorithm when numbers of iterations are the same. After that, this paper proposes the decoder architecture based on RMP-Min-Sum algorithm, which only loses 0.1dB compared with float point RMP-Min-Sum algorithm, and the total area of the decoder is 9.6 mm2 in 90nm CMOS technology.
