Multi-rate QC-LDPC Encoder

Huxing Zhang
Electronic Engineering University of Electronic Science and Technology
Chengdu China Email: zh_hxing@yahoo.cn
AbstractвҖ”A multi-rate memory-efficient encoder for low-density parity-check (LDPC) codes is proposed in this paper based on shift-register-adder-accumulator (SRAA). The SRAA algorithm simplifies the encoder computation module and reduces the complexity of the operation. The LDPC code generator matrix is constructed by lots of quasi-cyclic square matrices in the Chinese digital TV terrestrial broadcasting standard (DMB-T), and the encoder is presented based on the quasi-cyclic character that reduces the memory cost. Simulations demonstrate that the proposed encoder can satisfy the DMB-T in three-rate according to different bit-rate option with lower complexity.
Keywords-Low-density parity-check codes; DTV; encoder; error correcting code.
I. INTRODUCTION
LDPC code (Low Density Parity Check Codes, LDPC) is a class of linear block error-correcting codes that can be defined by a very sparse parity-check matrix or bipartite graph, that first proposed by Gallager (1) and MacKay rediscovered the superiority of LDPC(2). In recent years, many studies show that LDPC code is a kind of remarkable performance code that is close to the Shannon limit over additive white Gaussian noise (AWGN) channels. For rate 1/2, the best code found has a threshold within 0.0045 dB of the Shannon limit of the binary-input additive white Gaussian noise channel and simulation results with a somewhat simpler code show that the performance can achieve within 0.04 dB of the Shannon limit at a bit error rate of 10вҲ’6 using a block length of 107 (3). LDPC codes have been used as the core of channel coding in a number of international standards, such as Europe's nextgeneration Satellite Digital TV Broadcasting Standard (DVBS2)(6), China Digital Television Terrestrial Broadcasting Standard (DMB-T)(8). For LDPC code, the parity-check matrix is sparse, but the generator matrix is not sparse, so in general coding, the check matrix is used to simplify the encoding methods, such as RU algorithm(3). RU algorithm, also called lower-triangular matrix algorithm, is from the simplified form of check matrix by Gaussian algorithm and the mathematical expression of the check bits can be attained from the lower-triangular matrix. In DMB-T, the LDPC code is very long, and the RU algorithm will increase the complexity and consume lots of hardware resource in FPGA. As a result, this paper adopts an approach based on the SRAA algorithm to simplify the encoding complexity and reduce the cost of memory using the quasi-cycle character of LDPC code generator matrix, compared with the traditional single-rate encoders, this encoder can suit for three-rate of the DMB-T.

II. QUASI-CYCLE LDPC CODES
Cyclic code is an important sub-category of linear block code, which has a variety of simple and practical decoding algorithms with lower decoding complexity and more efficiency due to its inherent structure of algebra. The disadvantage of general LDPC codes is that a significant amount of memory is needed to store their parity-check matrices (5). Quasi-cyclic LDPC (QC-LDPC) codes may be a good candidate to solve the memory problem, since their parity-check matrices consist of circulant permutation matrices or the zero matrix. In fact, the required memory for storing them can be reduced by a factor 1/L, when L*L circulant permutation matrices are employed. A good example for QC-LDPC codes is the array codes (4).Quasi-cyclic LDPC code (QC-LDPC) is a very important category of LDPC, and it is linear, systematic, quasi-cycle (7). Compared to the Turbo code, it has much lower decoding algorithm complexity. QCLDPC cyclic determinant is a square that has such characteristics: every row is the rotate right of the first row, and every column is the rotate down of the first column. As a result, the cyclic determinant is fully determined by its first line (or the first column). This section will briefly introduce the traditional RU encoding algorithm, and then deduce the encoding algorithm specifically.

III. LDPC ENCODER BASED ON FPGA
A. The Computing Module Design
The blocks matrices operations can be broken down into multiplications and additions, based on FPGA, multiplications can be achieved using AND, additions can be achieved using XOR. Researching the computing module based on the 0.8rate generator matrix as an example, the generator matrix of QC-LDPC (7493, 6096) code has 48 pieces of row-block matrices and 11 pieces of column-block matrices. The first row elements of the row-block matrices can be stored in a 127-bit wide, 48-depth memory. There are 1397 bits check bit, which is127*11, in order to increase the encoding speed, 11 pieces of the SRAA (shift-register-adder-accumulator) circuits (in figure 3-1) are called in parallel to compute all the checkbit (9). If using one ROM to store each first row elements of the block matrices, as the 11 SRAA parallel computing, each time reading data from ROM into the 11 127-bit shift registers will cost at least 11 clock cycles, and 48 pieces of row-block matrices operation will at least take 11 * 48 clock cycles to read the total generator matrix. However, we can configure separated ROM for each SRAA to store the first line of the block matrix, and all the register A can read data in parallel from the ROMs to increase the encoding speed.

Like the above 0.8-rate encoder computing module, the 0.6-rate LDPC (7493, 4572) can call 23 SRAA circuits parallel to compute the 2921 bits check bit, after 36*127 clock cycles, all the check bit can be computed in the SRAA circuits and stored in the cumulative register B; the 0.4-rate LDPC (7493,3048) can call 35 SRAA circuits parallel to compute the 4445 bits check bit, after 24*127 clock cycles, all the check bit can be computed in the SRAA circuits and stored in the cumulative register B. If just combine the three-rate encoder together, it will spend 11 +23 +35 = 69 SRAA circuits, since each time we can only choose one rate, the SRAA circuits can be reused to reduce the waste of resource, so only 35 SRAA circuits can achieve three-rate encoding. B. The Whole Structure of the Encoder
In DMB-T, there are three encoding rates, 0.4, 0.6, 0.8, and different generator matrix parameters are according to the three rates. The generator matrix parameters of three rates in DMB-T are shown in table 1.

Fig.3-2 Whole structure of multi-rate encoder
Control Module is responsible for every part of the encoder; G-SRAA Module has a total of 35 sub-modules to store the three-rate generator matrices and compute the check bit; Parallel to Serial Module reads check bit in parallel from register B and outputs them serially to FIFO; FIFO is used to buffer the data; RAM is applied to store the input data, and send them out after the check bit. Rate_control signal controls the select of three-rate, and Data_in is the input signal. 0.4-rate calls G0-SRAA to G34-SRAA, 0.6-rate calls G0-SRAA to G22-SRAA, 0.8-rate calls G0-SRAA to G10-SRAA. G0 to G10 are constructed by three ROMs to store the head 11 column block matrices of 0.4-rate, 0.6-rate, and 0.8-rate; G11 to G22 are constructed by two ROMs to respectively store the 12 to 23 column block matrices of 0.4-rate and 0.6-rate; G23 to G34 are constituted by one ROM to store 24 to 35 column block matrices of 0.4-rate. The multi-rate encoder using several ROMs to store the matrices separately in order to update SRAA shift register simultaneously and to avoid time delay of reading data from the ROMs.

D. Simulations and Verification
The multi-rate QC-LDPC encoder based on SRAA algorithm was implemented using Xilinx ISE9.1 platform, described by Verilog hardware language, conducted on Xilinx virtex2P xc2vp100 FPGA. The XST synthesis report showed that it cost 9707 slices, 9841 flip-flops, 17573 4-input-LUTs. To give a clear understanding, the simulation of the SRAA structure is shown in Fig. 3-5 based on Xilinx ISE simulator.
Fig.3-5 The simulation of SRAA encoding structure In Fig.3-5, data_in is the information bit; rate_contr signal is 2вҖҷb11, which means the rate is 0.8; after computed, the check bit is sent to sraa_data_out in parallel; this is the key of the multi-rate encoder. In order to verify the encoder, we have adopted a verification program in MatLab platform. A set of random binary sequence is produced and sent into both the encoder and MatLab simulation. Comparing the two encoding results can test and verify the correctness of the LDPC encoder proposed in this paper.
IV. CONCLUSIONS A multi-rate LDPC encoder, using SRAA circuits and taking up less memory, is proposed in this paper that based on the quasi-cyclic characters of the generator matrix of DMB-T. Using separated ROMs to store blocks matrices is effective in reducing delays in the system. Compared with the traditional LDPC encoder, this encoder can achieve in three-rate, occupy fewer resources, and have more practical value.
