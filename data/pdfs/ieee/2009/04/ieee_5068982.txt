Abstract - In this paper a fast version of the Parallel Weighted Bit Flipping LDPC decoding algorithm (PWBF) is presented. The idea, which allowed decreasing the computational complexity of the PWBF method, is based on the observation that in dependency on signal to noise ratio, many of the received bits in the codeword of LDPC code could be without error. The augmented algorithm makes tests based on syndromes and symbol involvement in parity checks, which are given by the factor graph corresponding to the chosen LDPC code in order to define so called guaranteed bits. These guaranteed bits do not participate in the decoding calculations further. Simulation results are presented, which show that there is not visible any significant performance deterioration by application of this method and that the approximate speed up of the decoding for the tested codes is about 25% on the selected hardware in comparison with the known PWBF.
I. INTRODUCTION
Gallager in his paper [1] introduced not only the famous LDPC codes but also a Bit-Flipping (BF) decoding algorithm for these codes. BF, which uses hard decisions about transmitted symbols from a detector as its input does not perform as well as soft decoding methods based on the ideas of Belief Propagation (BP) [2, 3], which are however much more computationally demanding. Newer the less its simplicity and resulting lower computational complexity makes it an viable solution for application, which need very high throughput and/or in situation of high signal to noise ratio, where only a small percentage of transmitted symbols in a received codeword could be in error. The flexibility of some Software Defined Radio Platforms [4] allows even to design adaptive decoder and use the BF as a decoding alternative to stronger algorithms BP based decoding algorithms such as Sum-product or Min-sum in opportunistic manner. Alternatively also improvement of some sorts of BP algorithms could be achieved using BF as an additional procedure [5].

Bit-Flipping algorithm after its introduction by Gallager was later improved in different ways, but we will in this short introduction only mention a branch of research leading to that variation, which is improved in this paper [6]. In order to find some trade of solution between the computationally demanding BP, which performs in many situations close to ML and the suboptimal but computationally simple BF, so called Weighted BF (WBF) was proposed and improved via different modifications [7-10]. Later in [6] the parallel weighted bit flipping (PWBF) was introduced, which lead to faster convergence of the decoding algorithm without deteriorating the performance. In this paper another modification of the parallel bit flipping algorithm is presented, which further reduces the computational complexity thanks to introduction of so called guaranteed bits or reliable symbols. The paper is organized as follows. In Section II. the PWBF is sketched closely following the explanation and notation in [6] for convenience of the reader. In Section III. the so called guaranteed bits are introduced and analyzed. In Section IV. the new Fast PWBF is presented and in conclusions some remarks and summarization of the achieved results is made.

Parallelization in BF decoding was introduced due to the fact of unnecessary large time and computational resources consumption of standard BF algorithms. The main disadvantage of the standard BF algorithms is, that they are able to flip only one bit per iteration. In other words it means that they are able to repair only one error per iteration and in cases, when many errors occur in transmitted codeword, the algorithm could not be able to repair all of them before the limit on the number of iterations is reached. Also in cases when the number of errors does not exceed the number of iterations, it wastes time and computational resources. On the other hand, if the algorithm is able to flip several bits per iteration, the probability of correcting more than one error per iteration is increased. III. ON “GUARANTEED” BITS
The new Fast PWBF algorithm is based mainly on the exploitation of a set of so called guaranteed bits. The set of guaranteed bits consists only of those bits of received codeword, for which we can say, that they are with high probability without error. This set is calculated in the beginning of each iteration of PWBF algorithm and depends only on the set of syndromes (sm) calculated in step 1 of PWBF algorithm. It is well known, that each syndrome sm ,
> @ m  1, M represents one check node of Tanner graph (or
one row of parity check matrix). If the syndrome of mth row of PCM equals to zero, it is visible, that all the bits participating in this row of PCM are either correct, or there should be even number of erroneous bits. To eliminate the fact, that erroneous bit could be marked as guaranteed, it is necessary to check all the parity check equations in which this bit participates. Only such bit is marked as guaranteed, for which all parity check equations in which it participates are equal to zero. In other words we can say that these bits are with high probability correct (with no error added by the transmission). The bits in this set will be called guaranteed bits. CONCLUSIONS
In this paper a Fast Parallel Bit-Flipping algorithm was presented, which exploits a so called guaranteed bits in order to reduce the computational complexity of the standard PWBF decoding algorithm for LDPC codes. The simulation results confirm that there is not visible any significant error control capability reduction caused by this modification. On the other hand the new algorithm works approximately 25% faster than the standard PWBF.
