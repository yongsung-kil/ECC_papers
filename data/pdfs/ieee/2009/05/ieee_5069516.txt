On the correlation between error weights and syndrome weights for belief propagation decoding
of LDPC codes

Abstract— This paper analyzes the correlation between the error weights and the syndrome weights during the sum product algorithm (SPA) decoding process of low density parity check codes (LDPC) in order to improve the error performance and the decoding speed. In this paper, we propose a modified SPA decoding algorithm to improve the decoding performance of the non-decoded codewords and investigate if this new algorithm takes indeed the best decoding decision for each codeword. The effect of the averaged decoding over the error weights and syndrome weights are analyzed as well as their correlation. The correlation coefficients between them are computed and a weight enumerator polynomial is used to jointly represent the error and syndrome weight distribution for each codeword. Keywords: LDPC, error weight, syndrome weight, SPA decoding.
I. INTRODUCTION
The LDPC codes were discovered by Gallager in 1962 [1], [2], but they were ignored until 1981 when Tanner gave them a new interpretation: a graphical one. In the last years they became more and more popular, due to their remarkable performance over AWGN channels. There are various methods to construct and decode LDPC codes and the researchers try to improve their performances by improving the construction methods or the decoding methods. We will concentrate our attention on the decoding process. Until now LDPC codes give the best performances with the SPA iterative decoding based on BP (belief propagation) and they can get very close to the Shannon limit on AWGN channels. But even for this algorithm, there are codewords that cannot be decoded in the established number of decoding iterations. Some of them can be decoded if we increase the number of decoding iteration, but some of them cannot be decoded not even several hundreds of iterations and they will never be. In other words, for this latter category of codewords, more we continue the decoding process, more the decoding error increases and the codeword moves away from the original sent codeword. This phenomenon is explained by the existance of trapping sets [3] (structures in the Tanner graphe that make a certain error propagate in the graph during the decoding process). In the paper [4] the authors tried to improve the decoding of the codewords that cannot be decoded, by proposing the

averaged SPA decoding algorithm, but this method made worse the decoding of the codewords that normally could be well decoded. The advantage of this method is that it eliminates the oscillations produced by the trapping sets in the BER performance curves, but overall it determines a higher error probability floor.
For the communications systems that demand a very high error performance for high signal to noise ratios (satellite communications, optical communications, the Ethernet or DVD), it is very important to solve these existing problems and improve their performance. We will try to improve the existing log-SPA algorithm [5], [6], by analyzing the problems of the codewords that cannot be decoded by the existing log-SPA algorithm. The proposed decoding algorithm is based on the idea of an existing correlation between the syndrom weight vectors and the error weight vectors during the decoding process.
II. MODIFIED SPA DECODING ALGORITHM
The sum-product algorithm (SPA) is an iterative, symbolby-symbol soft-in, soft-out decoding algorithm, that processes the received symbols iteratively in order to improve the reliability of each decoded code symbol, typically measured by the log-likelihood ratio (LLR). At the end of each decoding iteration, the calculated LLR-s become inputs for the next [7]. The algorithm stops in two situations: if it results in a null syndrome or if a predetermined maximum number of iterations is reached, even if it does not find the correct codeword. However, we observe that the error performance does not necessarily improve if the number of iterations increases. After a certain number of decoding iterations, the number of erroneous bits in the syndrome can increase, which may also increase the number of erroneous bits in the estimated codeword. It can also decrease. This means that at the end of all the iterations, one does not necessarily make the best decision i.e., to choose the nearest estimated codeword from the transmitted codeword. For improving the decoding decision, we modified the logSPA decoding algorithm for the case when the algorithm is stopped after reaching the maximum number of iterations. III. EFFECT OF AVERAGED SPA DECODING OVER THE
ERROR WEIGHTS AND SYNDROME WEIGHTS
The averaged SPA decoding algorithm keeps in memory, at every iteration, the extrinsic coefficients of the last q iterations and makes an average between them and the present coefficient [4]. This algorithm is conceived in order to eliminate the decoding oscillations caused by the trapping sets [3], by leading the decision to an average solution (a stable error floor that corresponds to the average of the oscillation amplitudes). The trapping sets are structures in the Tanner graph that can cause the propagation of an error level in the graph during the message exchanges in BP decoding.
We studied the effect of the averaged SPA decoding over the error weights and syndrome weights for each codeword. We observed that there is a noticeable reduction in the number of possible weight values for those codewords affected by significant oscillations during the decoding process. These codewords converge very rapidly towards an error floor, from which the weights start to have a constant value until the last iteration of the decoding process. Some of the results are shown in the Fig. 5.

Fig. 9 shows the error weight and the syndrome weight distribution. The coefficients of the least squares line are given by standard linear regression: the slope indicates the correlation between the syndrome weights and the error weights. If the line is horizontal, then there is no correlation between these variables. If the line increases, there is a positive correlation between the two variables while if the line goes down, their correlation is negative. The larger the slope, the stronger is the correlation between the error weight and the syndrome weight variables. One must be careful about the interpretation of the results because this slope indicates an apparent linear dependence between the variables, but one cannot detect a non-linear relationship between them (for instance if they are exponentially correlated). We must specify that even if the slope of the line and its correlation coefficients indicate a strong correlation between the variables, there is not necessarily a causal relationship between them.
We represented on the same figure several variables as a function of the number of decoding iterations: first the evolutions of the error weight vector and the syndrome weight vector and then their averaged distributions. The averaging was made over the last three samples of the two vectors. On the same figure we indicated the evolution of the slope of the averaged syndrome and an index which shows the oscillations of the slope between negative, positive and zero values. With all these parameters, we have tried to find a rule that could help us to take the best decoding decision for the codewords that cannot be decoded. We noticed once again that the proposed criterion, i.e., choosing the error that corresponds to the minimum weight syndrome, does improve the decoding performance, but does not always results in the best decoding decision (Fig. 10). Sometimes, the correlation between the error weight and the syndrome weight is larger, sometimes it is smaller: thus a general decision criterion cannot be drawn on the basis of these parameters alone. The correlation coefficient measures the existing correlation between two vectors of the same length. In our case, the two analysed vectors are the error weight vector and the syndrome weight vector during a fixed number of iterations: here this coefficient has values between 0 and 1. By calculating the probability p, we can assess if the calculated coefficient value corresponds to its real correlation value or one could obtain the same value even if the two vectors were not correlated. The rule is that if p ≥ 0.05, then the correlation coefficient is reliable, meaning that there is a very small probability that we could obtain the same correlation coefficient if the vectors were uncorrelated. These two coefficients were calculated for all the 14 non-decoded codewords of the PG-LDPC (273,191) code. We observed that for the codewords that converge after more iterations than the predetermined number of iterations, there is a relatively large correlation between the vectors and this correlation increases with the number of iterations (when they approach the zero converging point). For the codewords that do not converge at all, after a certain number of iterations, the correlation starts to decrease with the number of iterations (and the vectors never converge). This phenomenon can be explained by the existence of trapping sets in the code structure, which during the decoding process, after several iterations, create a fixed error pattern in the Tanner graph, which then propagates all over the graph, thus moving away from the correct decoding decision.
V. CONCLUDING REMARKS
In this paper, a modified SPA decoding algorithm is presented: it reduces the errors for those codewords that cannot be decoded. However, it does not take always the best decision, i.e., the minimum weight syndrome does not always correspond to the minimum weight error vector. Using average SPA

decoding, the syndrome and error weight values are reduced and they converge to a constant value after a certain number of iterations. The codewords that cannot be decoded are better decoded by this method, but we increase the decoding error probability for the codewords that are well decoded compared to the standard SPA algorithm. The average SPA algorithm eliminates the oscillations in the syndrome weights, but usually results into a higher error probability floor.
To view the relationship between the error and syndrome weight distributions we used a weight enumerator polynomial and studied the evolution of correlation variables. For the codewords that can be correctly decoded, there is a strong correlation between the error weight vector and the syndrome weight vector. For the codewords that are successfully decoded, the correlation values increase with the number of decoding iterations while for the codewords that do not converge, after a certain number of decoding iterations, the correlation values decrease with the number of iterations: the

decoded vectors get more distant (in the Hamming sense) from the correct decoding solution. Neverthless, a general decision rule for decoding that lead to an optimal decoding still needs further investigations.
