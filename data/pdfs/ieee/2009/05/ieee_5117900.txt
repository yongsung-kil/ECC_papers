A shuffled message-passing decoding method for memory-based LDPC decoders
Yeong-Luh Ueng, Chung-Jay Yang, and Chun-Jung Chen Dept. of Electrical Engineering, National Tsing Hua University, Hsinchu, Taiwan, R.O.C.

Abstract— The convergence speed of shuffled message passing decoding (MPD) is faster than that of standard two phase message passing (TPMP) decoding. Due to complex memory access and requirement of large storage space, the shuffled MPD is not suitable for hardware implementation especially for highrate LDPC codes. In this paper, we propose a modified shuffled MPD which can achieve a similar convergence speed but with reduced complexity in memory access and storage space as compared to the conventional shuffled MPD. We implement a rate-5/6 LDPC decoder based on the proposed algorithm.
Keywords— Shuffled, iterative decoding, message-passing decoding, low-density parity-check (LDPC) codes, LDPC decoder.
I. INTRODUCTION
Since the rediscovery of low-density parity-check (LDPC) codes [1] by Mackay [2], LDPC codes have attracted tremendous research interest because of their near-capacity performance and their potential of highly parallel implementation of decoder. LDPC codes are included in many standards such as IEEE 802.16e (WiMax) [3]. LDPC codes can be decoded by message passing decoding (MPD) algorithms based on the Tanner graph [4] of its parity check matrices (PCMs). The Tanner graph of an LDPC code is a bipartite graph which consists of variable nodes and check nodes. Sum-product algorithm (SPA) [2] and minimum sum algorithm (MSA) [5] can be used in the operations of check and variable nodes. It was demonstrated in [6] that the offset MSA (OMSA) can achieve the same bit error rate (BER) with a loss of 0.05 dB in signal-to-noise ratio and lower complexity as compared to SPA.
Based on the Tanner graph, different decoding schedulings can be used to achieve different convergence speeds. Two phase message passing (TPMP) algorithm which divides the decoding operations in one iteration into two phases, i.e., check-node-operation phase and variable-node-operation phase. Based on horizontal scheduling, layered MPD (LMPD) in [7] which was called turbo decoding message passing (TDMP) algorithm in [8] was used to achieve a convergence speed faster than that of TPMP.
Instead of horizontal scheduling, in [9], the authors proposed a shuffled version of MPD based on vertical scheduling which has about the same computational complexity and converges faster as compared to TPMP using SPA. Such a shuffled MPD using SPA is not suitable for hardware implementation. The main problem is that the complexity of memory access is significantly increased as compared to TPMP. This problem can be avoided by using the proposed shuffled MPD. To reduce the storage space and the complexity in memory access, we only store the first, second, and third minimum of variable-

to-check messages. As compared to the conventional shuffled MPD, the proposed shuffled MPD can achieve similar error performance but with reduced complexity. We also implement a rate-5/6 and length-2304 LDPC specified in the standard of WiMAX based on proposed shuffled MPD.
The remainder of this paper is organized as follows. Section II briefly reviews the TPMP and shuffled MPD based on SPA and the LDPC codes used in WiMAX. Section III presents a shuffled MPD with low complexity. We also compare the error performance of the proposed decoding method with those of TPMP, shuffled MPD in [9], and LMPD. In Section IV, we implement a rate-5/6 LDPC decoder and compare these results with those given in [10][11] which can support the rate-5/6 LDPC code. Finally, Section V addresses some concluding remarks.

II. PRELIMINARIES
A. LDPC codes in WiMAX
The M ×N PCM H ≡ [Hij] of the quasi-cyclic (QC) LDPC code specified in the standard of WiMax [3] is constructed based on an Mb ×Nb matrix Hb, where M = zMb, N = zNb, and z is a positive integer. In Hb, each 0 is replaced by a z ×z zero sub-matrix and each 1 at the position (i, j) is replaced by a sub-matrix that is obtained by right cyclically shifting a z ×z identity matrix by p(i, j) ≥ 0 columns, 0 ≤ i ≤ (Mb −1), 0 ≤ j ≤ (Nb −1). Fig. 1 shows the block-type PCM of the rate-5/6 LDPC code defined in the standard WiMAX.

III. A SHUFFLED MESSAGE-PASSING DECODING METHOD
WITH REDUCED COMPLEXITY
Although the shuffled MPD in [9] can achieve a convergence speed faster than that of TPMP, this shuffled MPD is not suitable for memory-based hardware implementation. The main problem is that the complexity of memory access is significantly increased. Now we describe this problem by using NG = 1, i.e, bit-based shuffled MPD in [9]. Suppose we use a memory bank to store values of Qji. In shuffled MPD, we perform the vertical step according to column j of PCM H and then perform horizontal step according to the associated rows i, i ∈ IC[j], of H. From (6), we see that the associated (|IR[i]| − 1) values of Qj i must be read from the memory bank for every i ∈ IC[j]. We repeat such a procedure for j = 0, 1, · · · , NGG − 1 to complete one iteration. Hence, each value of Qji is read from the memory bank (|IR[i]| − 1) times in one iteration. For the QC-LDPC codes used in WiMAX, the complexity of memory access of group-based shuffled MPD is the same as that of bit-based shuffled MPD if one group consists of one block column, i.e., z columns, of H. For the rate-5/6 LDPC code, |IR[i]| = 20 for all i and hence each value of Qji must be read from the memory bank (|IR[i]| − 1)=19 times in every iteration. In contrast, each value of Qji is read from the memory bank one time in every iteration for TPMP algorithm. The huge memory access of shuffled MPD will induce an obstacle of memory-based decoder.

of sorting, memory access, and the storage space, a small

a lot of competing candidates are discarded in Step 1 of the

B. Bit error rate (BER)
Fig. 3 shows the BER results in the AWGN channel for the rate-5/6 LDPC code using the proposed shuffled MPD with γ = 2, γ = 3, and γ = 4. Also included in this figure are the results of the same LDPC code using the conventional shuffled MPD in [9], TPMP, and LMPD. We see that the proposed shuffled MPD with γ = 3 can achieve a convergence speed similar to those of the conventional shuffled MPD and LMPD.

IV. A DECODER BASED ON PROPOSED SHUFFLED MPD
A. Decoder architecture
Fig. 4 shows the architecture of a rate-5/6 LDPC decoder based on the proposed shuffled MPD with NG = z. In the initialization step, the channel values are loaded into the

memory bank M Bλ. Single-port memory is used in M Bλ since during iterative decoding, we only need to read channel values from M Bλ. The operations in vertical step are performed in “vertical engine” and the operations in horizontal step are performed in “|Q| sorter” and “R calculator”. The rotator and inverse rotator, which can be implemented by flexible permutater in [12], are used to perform necessary permutation of values of Ri,j and Qj,i, respectively. Following the scheduling shown in Fig. 5, we can complete the vertical and horizontal steps for group g (equivalently block column g of H). Following the processing sequence shown in Fig. 2, we can complete one iteration. To achieve a high throughput, we use a pipeline architecture in our decoder. With such a pipeline architecture, we can easily perform the vertical step by accumulators. Although the pipeline technique results in the current processing group g at iteration k can not access some of the up-to-date V2C messages, i.e., Qj i[k], j ≤ gNG, and we must use Qj i[k − 1] in our decoder, this only slightly degrades the error performance. This is verified by the BER simulation. When considering both the effect of quantization and pipeline, the degradation in error performance is about 0.1 dB. We use B|Q| = 4 bits to represent one value of |Qji|. We only need M γB|Q| = 384×3×4 bits to store values of |Qji| in the proposed shuffled MPD while in the conventional shuffled MPD, we need M |IR[i]|B|Q| = 384×20×4 bits to store |Qji|. Although, we need additional space of M γ log2 |IR[i]| = 384 × 3 × 5 bits to store the ordered indeces jk, the storage space is reduced by the proposed method. Therefore, the complexity in memory access is also reduced.
B. Implementation results
The implementation results and the comparison of our decoder with those in [10] [11] are given in Table I and Table II, respectively. In [11], the authors proposed a WiMAX decoder based on their phase-overlapping message passing decoding (PO-MPD). Since it is difficult to implement LMPD for high-rate LDPC codes efficiently, the authors in [10] used TPMP to implement the decoder for rate-5/6 and rate-3/4 LDPC codes in WiMAX. They still used LMPD to implement rate-1/2 and rate-2/3 LDPC codes in WiMAX. Our rate-5/6 decoder can support other LDPC codes by simply increasing the size of the memory banks used to store Qji since the row number of Hb for low-rate LDPC codes in WiMAX is larger than the case of high-rate LDPC codes.
V. CONCLUDING REMARKS
We have presented a shuffled MPD algorithm for practical hardware implementation. As compared to the conventional shuffled MPD, the proposed shuffled MPD can achieve the same convergence speed with reduced storage space and complexity in memory access. The proposed decoder has the advantage of high throughput and low complexity.
