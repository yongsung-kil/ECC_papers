Improving the Hardware Utilization Efficiency of Partially Parallel LDPC Decoder with Scheduling and Sub-matrix Decomposition

Jie Jin and Chi-Ying Tsui
Dept. of Electrical and Computer Engineering, The Hong Kong University of Science and Technology, Clear Water Bay, Kowloon, Hong Kong {eetsui, eejinjie}@ee.ust.hk

Abstract—Partially parallel LDPC decoder is commonly used for practical applications due to its good tradeoff between the hardware cost and the throughput. In the partially parallel LDPC decoding architecture, two kinds of processor units are implemented: check node unit (CNU) and variable node unit (VNU). Because of the dependency between two kinds of processor units, the low hardware utilization efficiency (HUE) is one of the design issues for the partially parallel decoding architecture. In order to achieve the optimal hardware utilization efficiency, it is important to determine the order of the rows and columns in the LDPC parity check matrix processed by the processor units. In this paper, we model the scheduling problem as an optimization problem and use simulated annealing to find good solutions for the scheduling. In order to further increase the HUE of the partially parallel decoding architecture, sub-matrix decomposition scheme is proposed. By applying these two schemes, the HUE of some partially parallel decoding implementations can achieve 100%.
I. INTRODUCTION
Recently, low-density parity-check (LDPC) codes [1] have gained
significant attention due to their near Shannon limit performance [2].
They have been adopted in several wireless standards, such as
IEEE802.16e [3] and IEEE 802.11n [4], because of their excellent
error correcting performance.

An LDPC code is a linear block code defined by a sparse parity check matrix (Fig. 1(a)). It can be represented by a bipartite graph, called Tanner Graph [5] as shown in Fig. 1(b), which contains two sets of nodes: variable nodes that represent the bits of a codeword and check nodes that implement the parity-check constraints. The standard decoding procedure is the message passing algorithm, also known as “sum-product” or “belief propagation” (BP) algorithm [2], which iteratively exchanges the messages between the check nodes and the variable nodes along the edges of the graph. There are

different ways to implement the LDPC decoding algorithm, based on the number of processing units available. In general, the LDPC decoder architecture can be classified into three types: fully parallel architecture, serial architecture and partially parallel architecture. In fully parallel architectures [6], a check node processor is needed for every check node, which usually results in large hardware cost and complicated routing, and hence is less flexible. The serial architecture uses just one check node processor to share the computation of all the check nodes and is too slow for most applications. For partially parallel architectures, multiple processing units are used allowing proper tradeoff between the hardware cost and the throughput and are commonly adopted in the actual implementation [7].
In the partially parallel architectures, two kinds of processing units—check node unit (CNU) and variable node unit (VNU) are implemented to perform the check node and the variable node message processing in the decoding algorithm. A CNU can start a check node processing only when all the messages from the neighboring variable nodes have been computed; a VNU can start a variable node processing only when all the messages from the neighboring check nodes have been computed. Due to this dependency, the low hardware utilization efficiency (HUE) is one of the design issues when implementing the partially parallel decoding architectures [8]. The overlapped message passing (OMP) scheme [9] has been proposed to improve the hardware utilization efficiency. However, only partial solutions have been proposed and these algorithms cannot be applied to the general LDPC codes [8]. To increase the HUE of the general LDPC codes, heuristic algorithms have been proposed in [8] and [10] to schedule the whole LDPC matrix by making more empty spaces in the lower left and upper right corners of the scheduled matrix. However, neither of the two methods offers an optimal solution for the scheduling. In this paper, we first model the scheduling problem as an optimization problem. Simulated annealing is used to find the good solution for the scheduling problem, and better solutions for the scheduling are achieved, compared with the previous works [8, 10]. In order to further increase the HUE, sub-matrix decomposition scheme is proposed to break down the LDPC sub-matrix into several smaller sub-matrices. By doing so, the new LDPC base matrix will become larger, sparser and offers more searching space for the scheduling.
The rest of the paper is organized as follows. Section II gives the background of LDPC decoding algorithm and the partially parallel LDPC decoding architecture. The proposed scheduling scheme and the sub-matrix decomposition scheme are presented in section III. In section IV, experimental results are shown. Conclusions are drawn in section V.

II. BACKGROUND
A. LDPC codes and decoding algorithm
LDPC codes are linear block codes characterized by a sparse matrix, called parity-check matrix. The code can also be described by means of a bipartite graph, known as Tanner graph (Fig. 1(b)) [5]. A Tanner graph is made up of two entities, variable nodes (VN) and check nodes (CN), and they are connected to each other through a set of edges. The optimal LDPC decoding is achieved by using a message passing algorithm, also known as “belief propagation” (BP) [2], which can be described as an iterative exchange of messages along the edges of the Tanner graph [5]. The algorithm proceeds iteratively until a maximum number of iterations are elapsed or a stopping rule is met. The belief propagation algorithm consists of two types of message processing: variable node process and check node process. In the variable node process: the messages are got from the neighboring check nodes and are updated, then the updated messages will be propagated back to the check nodes. In the check node process: the check node combines together messages from the neighboring variable nodes to compute the updated messages, which are sent back to the corresponding variable nodes.
To facilitate the hardware design of the decoder, the quasi-cyclic LDPC codes (QC-LDPC) [9] were proposed. They are structured codes, whose parity-check matrix is built according to specific patterns. They are suitable for VLSI implementation, because they can significantly simplify the wire interconnections and the memory address generation of the decoder [9]. QC-LDPC codes have been adopted in several modern communication standards, such as IEEE 802.16e [3]. Fig. 2 shows an example of such parity-check matrix. It is of rate 1/2 with sub-matrix size (i.e. the size of the identity submatrix) of 6. The parity-check matrix is composed of null sub-matrix or identity sub-matrix with different cyclic shifts. The numbers stand for the cyclic shift value of the identity sub-matrix, and the “–” stands for null sub-matrix.
B. The partially parallel decoder architecture
The fully parallel LDPC decoder architecture results in large hardware cost and complicated routing due to the large number of processing elements required for every check node and variable node [6]. For the partially parallel architectures, multiple processing units are used allowing proper tradeoff between the hardware cost and the throughput and are commonly adopted in the actual implementation [7]. In this section, the partially parallel decoding architecture for the type of quasi-cyclic codes is briefly introduced. Take the LDPC code shown in Fig. 2 as an example. We may use two check node units (CNU) and four variable node units (VNU). Fig. 3 shows the block diagram of this architecture. The processor unit is shared for the computation of several check nodes or variable nodes. The check node process is computed row by row by the CNU. The variable node process is computed column by column by the VNU. The check-to-variable messages or variable-to-check messages of one non-null entry in the base matrix are grouped and stored in a bank of dual-port memory and the variable-to-check messages of a non-null entry in the base matrix share the same memory with the check-to-variable messages of the same entry in the base matrix.
In the partially parallel decoding architecture, the check node computation for a check node can only start after all the messages from the neighboring variable nodes have been computed, and vice versa. Due to this dependency, the partially parallel decoding architecture results in only 50% HUE of the logic. The OMP scheme have been proposed in [9] to improve the HUE of the decoder through properly scheduling the order of the check node and the variable node processed by the CNU and VNU such that the check

node processing phase can overlap with the variable node processing phase. However, only partial solutions have been proposed and these algorithms cannot be applied to the general LDPC codes [8]. To increase the HUE of the general LDPC codes, heuristic algorithms have been proposed in [8] and [10] to make more empty spaces in the lower left and upper right corners of the scheduled matrix. Unfortunately, the reordered parity check matrix would not keep the quasi-cyclic characteristics any more and it is impossible to directly map each non-null sub-matrix into the individual memory bank in hardware architecture [7]. The reordered parity check matrix is difficult to implement. Thus, in [7], in order to keep the regularity of the QC LDPC codes, the reordering of matrix using the heuristic algorithm of [10] is only performed on the LDPC base matrix. Only a sub-optimal solution is achieved even for the scheduling of the base matrix. In this work, we model the scheduling problem as an optimization problem, and simulated annealing [12] is used to find the good solution for the scheduling. The proposed scheme will be presented in the next section.

III. SCHEDULING ALGORITHM AND SUB-MATRIX DECOMPOSITION OF THE LDPC MATRIX
A. Scheduling of the LDPC matrix The scheduling of the LDPC matrix is determined before the
hardware design and the scheduling algorithm is computed offline. Thus the scheduling algorithm will not cause any extra hardware overhead and the algorithm complexity is not a major concern. To achieve the optimal HUE of the partially parallel decoding architecture, the optimal scheduling of the matrix should be found. For an m by n parity check matrix, there are m!*n! different permutation orders for the check node and variable node being processed by the CNU and VNU. It is impractical to find out the optimal scheduling with full-search method even for a medium size matrix.
We will take the LDPC matrix shown in Fig. 2 as an example to illustrate the scheduling of the parity check matrix. In order to maintain the regularity of the matrix, only the base matrix is

However, this nonlinear integer problem is difficult to solve. The optimization objective L can be easily evaluated, when the matrix permutation and the hardware configuration are given. So here we propose to use probabilistic search method to search for solution in the design space. Simulated annealing [12] is used as the searching algorithm. The cost function used in the simulated annealing is the optimization objective L. For each move, a pair of rows or columns is randomly selected and their orders in the decoding sequence are swapped. The objective function is updated after each move. We apply the proposed scheme on the rate 1/2 LDPC code defined in IEEE 802.16e [3]. The experimental results are summarized in Table I. with small number of processor units, the HUE can be increased by 13.3%~20%.

In the previous works [7, 8, and 10], only the LDPC matrix is considered in the scheduling and only one reordered matrix is generated and then it is used to calculate the HUE for different hardware configurations implementing this matrix. However, to achieve an optimal HUE, both the hardware configuration and the LDPC matrix should be considered, because the resource constraints are determined by the number of processing elements, which will affect the optimization objective L. We find that the scheduling that achieves high HUEs for different hardware configurations can be different. For example, the scheduling achieving the optimal HUE for the hardware with two CNUs and three CNUs are different. Although the HUE of the hardware configuration using small number of processor units with our scheduling scheme can achieve 100%, for the hardware configuration using large number of processor units, the HUE is low and it is only 50%. In order to further increase the HUE of the hardware implementation with many processor units, the sub-matrix decomposition scheme is proposed in the following section.

B. LDPC sub-matrix decomposition
To achieve the highest HUE for the decoder architecture with a large number of processor units, we can reorder the whole parity check matrix instead of the base matrix. By scheduling the whole parity check matrix, the idle time can be minimized. However, this will destroy the quasi-cyclic characteristics of the original parity check matrix and it is difficult to implement the scheduled matrix [7]. To maintain the regularity of the LDPC matrix and achieve a high HUE of the decoder at the same time, instead of scheduling the original base matrix or the whole parity check matrix, we propose to schedule a new base matrix generated by the sub-matrix decomposition. Fig. 6 shows an example of the sub-matrix decomposition scheme for a cyclic shift matrix with size 6 and decomposition factor 2. It can be seen that the cyclic shift matrix with size 6 can be reordered into two cyclic shift matrices with size 3. Fig. 7 shows the new base matrix generated from the parity base matrix shown in Fig. 2(a) by decomposition factor 2. Compared with the Fig. 2(a), the new base matrix is increased to 12 block rows and 24 block columns, and the sub-matrix size for every entry is reduced to 3. Table II shows the HUE of the partially parallel decoding architecture with one CNU and two CNUs for this example code. It shows that if the matrix is broken down by two, the HUE can be increased from 6/7 to 100%.
Different base matrices can be generated using different decomposition factor in the sub-matrix decomposition scheme. All the factors of the sub-matrix size q can be used as the decomposition factor. With sub-matrix decomposition scheme, the number of entries in the base matrix will increase which results in the increase of the memory banks in the decoder. The increase of the memory banks will increase the hardware cost because more decoders and more address generators are required for the memory banks. For different applications, the trade off between the hardware cost and the HUE of the decoder can be made.

IV. EXPERIMENTAL RESULTS

We apply the proposed scheduling scheme and the sub-matrix decomposition scheme to the LDPC codes defined in IEEE802.16e [3] to test the HUE for the different hardware configurations and the results are summarized in the Table I, Table III and IV. For some decomposed matrix with large entry, the simulated annealing may take too much time to finish the search. For these matrices, we use a heuristic algorithm based on that in [10] to obtain the schedule. This is indicated in Table III and IV. Table III and IV show that the HUE for the hardware configurations with large number of processor units can be increased by 50% using the sub-matrix decomposition scheme. For practical applications, the decomposition factor can be set based on the tradeoff between hardware overhead and the hardware utilization efficiency.

V. CONCLUSIONS

In this paper, we proposed a new scheduling scheme using the simulated annealing method to increase the hardware utilization for the partially parallel decoding architecture. To further increase the hardware utilization efficiency of the partially parallel decoding architecture with large number of processor units and maintain the regularity of quasi-cyclic LDPC codes, the sub-matrix decomposition scheme is introduced. Combining these two schemes, the hardware utilization for some of the partially parallel decoding implementation can achieve 100%.
