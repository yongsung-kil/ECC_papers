Abstract—For the realization of a safe train traffic environment, a wireless communication system which makes an information network between ground and vehicles has been needed. Low-Density Parity-Check (LDPC) codes have been widely considered as error-correcting codes for next generation communication systems. In this paper, Vehicle Condition Detect System (VCDS) channel coding methods were given according to [3].Then some reduced complexity decoding algorithms (BPbased, TSP, TDMP) are compared with the original BP algorithm. We find the optimum algorithms which are improvement in coding gain and suitable for VCDS credible information transmission. In the aspect of the hardware design, We analysis a highly efficient scalable decoder architecture for LDPC codes. This architecture adjusts the parallel processing factor based on the serial frame to increase decode speed.
Index Terms—vehicle condition detect system, credible transmission of vehicle information, low density parity-check codes, TDMP decoding algorithms, integration decoder architectures
I. INTRODUCTION
Real-time traffic information is an essential and important factor in such Intelligent Transportation System (ITS) as safe management, traffic control, route guidance, traffic accident detection and so on. Probe-vehicle technique is an effective method of collecting real-time traffic information on road networks. A technically feasible and cost-effective wireless communication network is in great demand so that large amounts of traffic data sampled by probe vehicles over a large geographical area can be sent to the vehicle working condition detect processing center [1].
The Vehicle Condition Detect System (VCDS) is showed in Fig. 1.The information is transmitted to the“existed vehicle detect system ” on the ground to realize the information processing of the black block in time; the statuses and faults of the vehicle detect systems are collected by the intellective complex sensors and the information of the black block is collected too. The collected information is saved by the vehicle information processor, the classify of the information and the faults , the intellective analysis of the faults and the alarm clew ,the showing to the driver are all done by the processor. The interface of communication and the conversion of the protocol in the wireless channel.
Wireless channel is one of the important parts of VCDS. The compressed vehicle data, in which bits are highly

correlated and variable length codes are liable to cause error propagation effect, are very sensitive to channel bit errors and erasures. Usually, the data communication quality can not be guaranteed under the condition that the channel Bit Error Rate (BER) is around 10-4 without any bit-error resilience measures taken. But, to decrease channel BER only by enhancing the power of the error correction codes will inevitably lead to high computation complexity and large coding delay, which is not suitable for real-time video communication.

Low density parity check (LDPC) codes were invented by Gallager in 1962. They are defined by sparse parity check matrices, with various construction methods. Simulation results show that such codes have near Shannon limit performance when decoded using an iterative probabilistic decoding algorithm. LDPC code has to be shown to perform as well as the turbo code but lower complexity and without the error floor problems [2].

III. LDPC DECODER IMPLEMENT ISSUES
LDPC codes are a class of linear error-correcting block codes. A LDPC code is defined as the null space of a paritycheck matrix H with the following structural properties:

1) Each row consists of ρ “ones” ; 2) Each column consists of ν “ones”; 3) The number of “ones” in common between any two columns, denoted λ, is no greater than 1; 4) Both ρ and ν are small compared to the length of the code and the number of rows in H.
Since ρ and ν are small, H has a small density of “ones” and hence is a sparse matrix. For this reason, the code specified by H is called a LDPC code. The LDPC code defined above is

known as a regular LDPC code. If not all the columns or all the rows of the parity-check matrix H have the same number of “ones” (weights), a LDPC code is said to be irregular.

The main challenge in the LDPC code decoder implementation is how to effectively manage the message passing during the iterative belief propagation (BP) decoding [4].

As mentioned in the above, the construction of the LDPC codes typically relies on random construction that results in unstructured bipartite graph topology. However, the partially parallel decoder implementation demands relatively structured LDPC code bipartite graph topology that could lead to the efficient realization of partially parallel message passing. Thus the partially parallel decoder implementation is not trivial. In this paper, we propose an approach to construct a class of LDPC codes that have relatively structured bipartite graph topologies that lead to very good performance and are directly fit to partially parallel decoder implementations.

One of the attractions of LDPC codes is their simple description in terms of a random sparse parity-check matrix, which makes it easy to construct LDPC codes of any rate. Many good codes can be built by specifying a fixed weight for each row and each column, and constructing at random subject to those constraints. However, the best LDPC codes use further design criteria. We review the basic constructions.

A. Gallager’s construction
In his original work Gallager imposed a fixed column weight j and a fixed weight k [2]. The parity-check matrix was divided horizontally into j equal size submatrices, each containing a single ‘1’ in each column. Without loss of generality the first submatrix was constructed in some predetermined manner (e.g. k concatenated identity matrices). The subsequent submatrices were random column permutations of the first.

B. MacKay’s construction
MacKay was intereated in keeping to a minimum number of short cycles in the bipartite graph representing the paritycheck matrix. The presence of short cycles can create difficulties for the belief propagation decoder [5][7].

C. Irregular matrices
Irregualr matrices are those for which there are several row and column weights. The matrix parameters are described in terms of row and column profiles. To build an irregular matrix, we use a construction described by Luby [9].

B. Design of decoder of VCDS based on TDMP algorithm
The design of VCDS receiver based on channel decoding is shown as Fig. 3.

Based on encoder of LDPC code, we apply TDMP algorithm to the VCDS receiver.

Iterative LDPC decoding algorithms such as sum-product or min-sum algorithms passes information back and forth between the check nodes and variable nodes in LDPC Tanner graph until correct codeword is found. In this section, we proposed a programmable hardware decoder architecture shown in Fig. 4. which built upon the optimizations performed in VCDS receiver. Optimized decoding operation sequence can

eliminate or reduce waiting time of data dependency [5]. The check node function unit (CNU) and variable node function unit (VNU) updates the check nodes and variable nodes respectively. The architecture allow CNU and VNU to access the dual port memory without conflict. The memory schedule places constraint on the parity check matrix generation. The constraint t＜1/2 is used as evaluation function. The offset value need to be less than half of the size of the circulant submatrix for our decoder architecture to function correctly. Lower offset value t also reduces the latency of decoders.

VI. CONCLUSION
In this research, we proposed a complete LDPC code and decoder design platform suitable for vehicle information credible transmission. Simulation results show that TDMP has a faster convergence speed compared with other LDPC decoding algorithm such as BP and TSP algorithm, and attains an order of magnitude improvement in coding gain at moderate to high SNR values.
Simulation results are reported that it not only has very good error-correcting performance but also is fit to efficient partially parallel decoder hardware implementations. The code construction scheme and the corresponding partially parallel decoder design have been described in details. The main advantage of this proposed approach that it could realize flexible trade-off between hardware complexity and decoding speed and support any arbitrary node distribution configuration,

hence shows great potential on achieving very good errorcorrecting performance.
This architecture adjusts the parallel processing factor of decoder of VCDS to increase decode speed.
As has been demonstrated by this work, the TDMP algorithm is an attractive solution for the VCDS communications standards in term of decoding throughput and requirements.
