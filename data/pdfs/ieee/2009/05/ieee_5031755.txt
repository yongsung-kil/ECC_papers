415 Gbit/in.2 recording in coaxial holographic storage using low-density parity-check codes
Kenji Tanaka, Masaaki Hara, Kazutatsu Tokuyama, Kazuyuki Hirooka, Yoshiki Okamoto,
Hidenori Mori, Atsushi Fukumoto, and Kanzo Okada
Tera Bytes Memory Development Department, Core Device Development Group, Corporate R&D, Sony Corporation,
Gotenyama Tec, 5-1-12 Kitashinagawa, Shinagawa-ku, Tokyo 141-0001, Japan Phone: +81-3-5448-6698, Fax: +81-3-5448-3257, E-mail: KenjiD.Tanaka@jp.sony.com
1. Introduction A coaxial holographic storage technology is one of the attractive candidates for a future optical storage
system. In recent years, we have reported our developed coaxial system [1-4], and have achieved the raw data density of 270 Gbit/in.2. In this setup, we have mainly applied optical components, such as a random-binary phase mask, a polarizing beam diffractor (PBD) for a noise reduction, a high NA objective lens (NA=0.85), and reducing optics for high data capacity per page. On the other hand, as generally known, signal processing is a powerful technique to improve recording performance, and low-density parity-check (LDPC) codes [5] recently attracts the great attention as a next-generation error correction code.
In this paper, we describe the recording performance using LDPC codes in our coaxial holographic system, and demonstrate 415 Gbit/in.2 recording combining LDPC codes with the optimization of optical conditions.
2. Experimental Setup The schematic diagram of the experimental setup is shown in Fig. 1. This setup consists of several
devices; an external cavity diode laser (ECDL) as a light source (λ = 407 nm), a spatial light modulator (SLM), PBD, an objective lens with a NA = 0.85, and a CMOS imager. The SLM is an integrated phase mask SLM (IPM/SLM) which was jointly developed with Displaytech Inc. It consists of ferroelectric liquid crystal (FLC) panel and a random-binary phase mask with a pixel size of 10.7 μm. The function of the phase mask is to suppress DC component for the matter of medium consumption and to spread optical distribution in the medium for an increase in the interference between a signal light and a reference light. The PBD, which also contains a quarter wave plate, can diffract the reference beam after retrieving a recorded hologram, thus decreasing the scattering noise. The aperture located at the Fourier plane of the relay lens has a typical size twice as the Nyquist size. The recording medium is a photopolymer with the thickness of 600 um. In the recording density evaluation, holograms are multiplexed two-dimensionally with certain pitches using piezoelectric stages.

diameter of the reference pattern are respectively 4.6 mm and 6.4 mm, and its data capacity is 135 Kbits. The white rate of the reference pattern is set to 0.75. As a signal data format, we combine a LDPC code with three kinds of modulation codes (Figs. 2(b)-(d)); no-modulated random code with a white rate of 50 %, E(4, 1, 2) sparse code, and E(16, 3, 8) sparse code. Here, E(n, m, k) means m blight pixels in n pixels with a modulation code rate of k/n [6]. The LDPC code is a regular code with a codeword length of 4947, code rate of 50 %, column weight of 3, and row weight of 6, and a min-sum algorithm is utilized in decoding.

4. 415 Gbit/in.2 recording In addition to applying the E(16, 3, 8) code, we optimize optical conditions for further improvement in
recording performance. The changes of optical conditions are as follows: 1. relay lens; new relay lenses are introduced into L1-L4 in order to improve lens aberration characteristics. 2. page data capacity; the page
data capacity is increased 5 % with the increase of data signal diameter. 3. aperture size; the aperture size is
changed from ×2.0 Nyquist size to ×1.6 Nyqusit size. We optimized this size by the SNR simulation of
retrieved page data and the noise evaluation in the experiment. The multiplexed recording performance shown in Fig. 6 (data D) clearly reveals that these optimization improved the SNR performance, realizing 415 Gbit/in.2 raw data density (SNR = 4.9 dB; error-free). This recording density corresponds to
approximately 180 GB capacity.

5. Conclusions
We have applied LDPC codes into our coaxial recording system, and revealed its fundamental characteristics. In addition, by applying the E(16, 3, 8) code and optimizing optical conditions, we have successfully improved the recording density, achieving the raw data density of 415 Gbit/in.2.
