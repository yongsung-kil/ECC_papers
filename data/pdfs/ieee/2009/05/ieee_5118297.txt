On the Structured Parallelism of Decoders for LDPC Convolutional Codes – An Algebraic Description

Marcos B.S. Tavares, Emil Matu´sˇ and Gerhard P. Fettweis Vodafone Chair, Technische Universita¨t Dresden, D-01062 Dresden, Germany
Emails: {tavares, matus, fettweis}@ifn.et.tu-dresden.de

Abstract— We propose an algebraic framework that captures the parallelism of decoders for LDPC convolutional codes. From this framework, an architectural template of a decoding core is derived and its main aspects are discussed. Furthermore, sophisticated decoding structures are built using the decoding core as basic element.

I. INTRODUCTION
It is well known that the main problem in the implementation of efficient decoders for low-density parity-check (LDPC) codes is due to the complex message exchanging operations required by the belief propagation algorithm [1]. In this case, memory architectures and routing networks should be designed such that low latency data transfers can be achieved and thus high decoding throughput is provided.
The complexities of memory architectures and routing networks for LDPC decoders are deeply related to the underlying graph structures of the codes to be implemented. For instance, codes that show totally unstructured (i.e., semi-random) graphs are likely to have more complex memories and routing networks than codes that show structured graphs (i.e., with certain algebraic symmetry). As it was already discussed in [2], LDPC convolutional codes (LDPCCCs) represent an elegant way to overcome the problem of complex interleaving operations. Furthermore, a system using LDPC-CCs might benefit from additional flexibility and lower implementation costs because the same encoder (decoder) can be used to encode (decode) code sequences with different lengths [3]–[5].
The main objective of this paper is to present an algebraic framework that models the structured parallelism that can achieved when implementing decoders for LDPC-CCs. From this abstract framework, an architectural template is derived that can be used to construct more complex decoding structures.

Graph Representation: LDPC-CCs have highly structured bipartite graphs of semi-infinite character. On the other hand, the bipartite graphs representing tail-biting LDPC-CCs have a cylindrical shape due to the syndrome former wrapping. Fig. 1 shows an arbitrary section of the abstract bipartite graph G = (C, V, E) of an (tailbiting) LDPC-CC. Nodes in the same horizontal line are said to be of the same type and they have the same connection profile [7]. Lifted Codes: An efficient way of constructing bigger codes from small ones is by lifting1. In this case, the ones and the zeros of a syndrome former with a relatively small ms are substituted by permutation and all-zero matrices of dimension X, respectively. In the graph domain, the lifting operation corresponds to substituting the nodes and edges by groups of X nodes and X edges, respectively. It is worth to mention that lifted syndrome formers are no longer restricted to be time-invariant. However, they still have very good parallelization properties as we will show later on.

IV. BASIC STRUCTURED PARALLELIZATION TECHNIQUES
The BP algorithm is intrinsically parallel: in each step of one iteration all CNs or VNs can perform their operations simultaneously since there is no data dependency. A decoder architecture for such a fully parallel processing was presented in [11]. However, in spite of the very high decoding throughput, this architecture shows a complex and inefficient routing network due to the semi-random character of the code considered. The objective of this section is to present structured parallelization techniques derived from the properties of the LDPC-CCs that can be used to construct simple, flexible and efficient decoder architectures.

Routing Network Complexity: To consider the complexity of a single shuffle and a single crossbar switch the following propositions need to be stated.
Proposition 1: Equivalent edges departing from consecutive CNs of the same type arrive at consecutive VNs of the same type, and vice versa. Proof: Because the codes are time-invariant, the connection patterns of all nodes are repeated for each time-instant tk.
Proposition 2: The temporal distance between connected nodes is less or equal to ms. Proof: Considering equation (1), we observe that the VNs involved in a parity-check equation at time tk are spread between tk − ms ≤ t ≤ tk. Conversely, the CNs checking a VN at time tk are spread between tk ≤ t ≤ tk + ms.
Once that the propositions above have been stated, one can conclude that the shuffles can be implemented as barrel shifters since edges from consecutive nodes are mapped to edges of consecutive nodes (Proposition 1). From Proposition 2, we have that the crossbar switch only needs to connect processing units containing in their vector memories edges from connected nodes, i.e., less or equal then ms time instants apart from each other.
Lifted Codes: For the best usage of resources, NΓ must be a multiple of X. In this case, the shuffle units will consist of NΓ/X independent shuffles, which can be simplified to barrel shifters if circulants are used instead of generic permutations. The crossbar switch architecture for lifted codes has the same properties commented above.

V. BLOCK PROCESSING
For sufficiently large block lengths, several decoding cores (Fig. 2) can be combined in one single architecture to increase the decoding throughput. The enabling fact for this combination is given by Proposition 2. In this case, each decoding core is assigned different blocks of data and neighboring decoding cores share memories with data corresponding to ms time instants. Fig. 3 shows the arrangement of the block processing architecture. The dashed data buses give support for tail-biting decoding. It is important to mention that the number of time instants within the memories of each processing core divided by NΓ must be a factor greater than ms divided by NΓ. Otherwise, there will be conflicts in the shared memories.
Observe also that this solution is particularly interesting when redesign costs are a critical issue. In this case, one only needs to interface the NB decoding cores with the shared memories in order to reach a throughput NB times higher than a single core.

VI. ITERATION PIPELINING
Another application of Proposition 2 is the pipelining of the decoding iterations. In this case, we can use the decoding core of Fig. 2 to parallelize the circular pipeline decoder proposed in [6].
The architecture of the parallelized circular pipeline decoder is schematically shown in Fig. 4 and it consists of 1 ≤ IP ≤ tN /(ms + NΓ) decoding cores Di (Fig. 2). The operation of the circular pipeline decoder is as follows. The code sequence of an (tail-biting) LDPC-CC corresponding to t ≤ tN time instants enters the decoder. Each decoding core Di is attributed a memory section (Fig. 4), which is large enough to avoid the memory access collisions between the processors and, thus, enable their parallel operation. Considering the Proposition 2 and the parallelism NΓ, we can easily note that the size of the memory sections should be at least ms + NΓ

VII. CONCLUDING REMARKS
The horizontal parallelization has been proposed for the first time
in [2] and its realization has been successfully integrated in [13]. The
realization of the horizontal-vertical parallelization was presented in
[7] and [14]. In all cases, the decoders can achieve hundreds of Mbit/s
still with relatively low power and area consumptions. Our further work includes the implementation of architectures for
lifted codes as well as for block processing and parallel pipeline
decoders.
