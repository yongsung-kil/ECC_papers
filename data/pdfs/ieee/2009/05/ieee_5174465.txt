Design of a Low-area, High-throughput LDPC Decoder Using Shared Memory Banks for DVB-S2

Chang-Soo Park, Seong-Woon Kim, and Sun-Young Hwang, Member, IEEE

Abstract — This paper presents a high throughput LDPC decoder architecture for DVB-S2, a second generation standard for European satellite digital video broadcasting system. DVB-S2 standard specifies higher order modulation and powerful FEC system based on LDPC codes concatenated with BCH code. DVB-S2 has been gradually replacing DVB-S for global satellite digital TVs, and many HDTV channels are served by DVB-S2 standard in Europe and Japan. The proposed decoder architecture clusters bitnodes and checknodes into groups by utilizing of periodic nature of parity check matrix. Each of these node groups are assigned to a functional modules which perform calculations required at bitnodes and checknodes. These functional modules exchange data through shared memory banks to maximize parallel accesses. Implementation of the proposed architecture exhibits the throughput of 277 Mbps, 9% improvements over previous architecture, while the area is reduced by as much as 52%1.

I. INTRODUCTION
In 1994, DVB-S was introduced as the European standard for broadband satellite broadcasting services. The DVB-S standard specifies QPSK modulation and FEC (Forward Error Correcting) structure which is concatenated convolution code and Reed-Solomon channel code. Since 1994, an evolution of digital satellite transmission technology has led to the second generation standard, DVB-S2. Higher order modulation and new powerful FEC system of DVB-S2 standard provides capacity gain of over 30%[1][2]. This new powerful FEC system is performed by LDPC (Low-Density Parity Check) codes concatenated with BCH (Bose-Chaudhuri-Hcquenghem) codes. DVB-S2 has been gradually replacing DVB-S for global satellite digital TVs, and many HDTV channels are served by DVB-S2 standard in Europe and Japan.
Gallager[3] proposed LDPC codes in 1962 and was rediscovered by MacKay and Neal[4] in 1996, which have introduced remarkable error correcting capabilities near the shannon’s limit. LDPC codes are defined by sparse parity check matrix H, which matrix H decides structure and performance of encoder/decoder. The (N–K) matrix H general-izes to create bipartite graph called Tanner graph[5], where N bitnodes and (N–K) checknodes are connected by

Fully parallel decoder is not suitable for the LDPC decoders with larger block length[8]. Instead, partially parallel decoders are practical and proposed by several researchers[9][10][11][12] [13].
This paper proposes a partially parallel decoder architecture suited for the DVB-S2 LDPC codes. Proposed decoder architecture composed 180 functional modules and two shared memory modules with 90 bank each, besides [11] composed by 360 functional modules and a shared memory modules with 360 bank each. This paper is organized as follows : Section II describes the DVB-S2 LDPC codes. The proposed decoder architecture is presented in Section III. Section IV shows the experimental results obtained by measuring throughput and area of an implementation of the proposed architecture. Section V concludes the paper.

III. PROPOSED DECODER ARCHITECTURE
Nodes and edges of Tanner graph are determined by matrix H. Rows of matrix H are mapped to checknodes and columns to bitnodes. Edge connections between checknodes and bitnodes are determined by the elements 1 in matrix H. As shown in Section II, matrix H consists of two parts: matrix A and matrix B. Columns in matrix A determine K bitnodes that represent the information and columns of matrix B determine the (N–K) bitnodes for parity. Edges connected to information bitnodes show a regular cyclic-shift pattern with the table specified in standard as the indices. Edges connected to parity bitnodes show a simple zigzag pattern based on matrix B, which is a staircase lower triangle matrix.
LDPC decoder can be designed to map Tanner graph directly. Nodes of the Tanner graph are allocated to a functional module and edges are implemented as memory and routing logic. Nodes can be clustered, based on the cyclicshift pattern of edges that connect information bitnodes and checknodes, to share functional modules and to share memory. The resulting design can prevent memory conflicts and the bottleneck problem.
In this section we propose a method of clustering nodes and implementing a shared memory bank by using the periodic nature of matrix A to maximize parallel access. The proposed decoder architecture is designed to be compatible with the characteristics of matrix A, where the parity portion based on matrix B is designed to suit the architecture determined by matrix A.

C. Structure of Shared Memory
The message passing algorithm repeatedly exchanges the computation results of bitnode groups and checknode groups that are connected by edges. Memory is required for implementation of edges representing connections between these node groups. Use of local memory simplifies memory accesses. Since each computation result is delivered to multiple FMs, duplicate data may exist in local memories of FMs. Therefore, shared memory is used for data storage. Each bitnode group allocated to a BFM and each checknode group allocated to a CFM require 720 cycles per iteration. Bitnode receives as input the computation result of the xi -th checknode and checknode receives the computation result of the yi-th bitnode at the first cycle. xi has 8 elements for i = 0, 1, … , 35 and 3 elements for i = 36, 37, … , 89 at coderate 1/2. Therefore, bitnode receives 450 data items as input. Since yi has 5 elements, checknode also receives 450 data items as input. Since xi and yi specified in the standard have random distributions, the 450 data items should be evenly distributed in each memory bank to ensure efficient data transfer to respective FMs.
The shared memory consists of two memory modules : one is check-to-bit memory, which stores checknode computation results and transfers them to bitnodes, and the other is bit-tocheck memory, which stores bitnode computation results and transfers them to checknodes. Each memory module consists of 90 memory banks. Fig. 8 shows the shared memory structure at coderate 1/2. Fig. 8(a) shows the computation result of the c-th

BFM requires either 8 or 3 input datas, but the check-to-bit memory provides 5 datas at each bank in each cycle. Each CFM requires 5 input datas, but the bit-to-check memory provides either 8 or 3 datas at each bank in each cycle. To overcome this mismatch, a shuffling network between the memory output ports and the BFM input ports is used. In the first cycle of decoding process, 0-th BFM receives the computation results of 54-th, 9318-th, 14392-th, 27561-th, 26909-th, 10219-th, 2534-th and 8597-th checknodes as determined by the index x0. These computation results are stored in 54-th, 48-th, 82-th, 21-th, 89-th, 49-th, 14-th and 47th banks, which are (x0 mod 90)-th banks of the check-to-bit memory module. In the second cycle, the computation results of 144-th, 9408-th, 14482-th, 27651-th, 26999-th, 10309-th, 2624-th, 8687-th checknodes are transferred to 0-th BFM. These computation results are stored in the same (x0 mod 90)th banks. In the j-th cycle, i-th BFM receives data from the (xi mod 90)-th banks. Therefore, the output ports of the check-tobit memory bank can be connected to the input ports of BFMs via 8 dedicated data lines. Since a BFM consists of 36 BFMs of 8-input and 54 BFMs of 3-input, the required shuffling network consists of total 450 dedicated lines.
In the first cycle, the 0-th CFM receives the computation results of 9421-th, 9822-th, 11094-th, 12961-th and 26820-th bitnodes as determined by the index y0. These computation results are stored in 26-th, 27-th, 31-th, 36-th and 75-th banks, which are (y0 div 360)-th banks of the bit-to-check memory. In the second cycle, the computation results of 9422-th, 9823th, 11095-th, 12962-th and 26821-th bitnodes are transferred to 0-th CFM. These computation results are stored in the same (y0 div 360)-th banks. In the j-th cycle, i-th CFM receives data from the (yi div 360)-th banks. The output ports of the bit-tocheck memory bank are connected to the input ports of CFMs via 5 dedicated data lines. Since a CFM consists of 90 CFMs with 5 input ports, the required shuffling network consists of total 450 dedicated lines.

IV. PERFORMANCE RESULT
The proposed decoder architecture was designed in verilogHDL and synthesized by a design compiler based on a 90nm technology. Figure 7 shows the synthesis results of the proposed decoder and those of previous decoders. The decoder presented in [11] achieves high throughput but requires huge area, and the decoder in [13] can be implemented in a small area but shows low throughput. Overall area of the proposed decoder is 4.76mm2 and throughput is 277Mbps. The throughput is improved by 9% when compared to that of the decoder in [11] , while the area is reduced by as much as 52%.

V. CONCLUSION
In this paper we propose a high-throughput LDPC decoder for DVB-S2. The proposed decoder architecture is designed by clustering bitnodes and checknodes into groups and by using shared memory banks. Node clustering makes it possible to achieve high throughput by exploiting maximal parallelism of FMs, and shared memory structure allows the implementations in small area without degradation in throughput by settling memory conflict and bottleneck problems, simultaneously.
