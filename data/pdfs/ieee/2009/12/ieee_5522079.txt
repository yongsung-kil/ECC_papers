Efficient Construction of Irregular LDPC Codes with Midterm Block Length and Nearly Optimal Performance
Qingwen JIN1, Pengjun WANG1, Shuzheng XU1, Huazhong YANG1, Habib F. Rashvand2
1. Department of Electronic Engineering, Tsinghua University, Beijing 100084, P. R. China. 2. Editor-in-Chief, IET & Director of Adv. Coms, School of Engineering, University of Warwick

Keywords: Irregular LDPC, Short Block, Downhill, PEG
Abstract
In this paper, we propose a general method for constructing irregular LDPC codes with midterm or short block lengths. It is a Monte-Carlo simulation based optimization procedure called Downhill Simplex which uses PEG (Progressive Edge Growth) method for Tanner graph construction in cost function evaluations. It outperforms DE (Density Evolution) method in both efficiency and performance. The efficiency of our proposal mostly depends on the Monte-Carlo simulation time which due to the midterm block lengths is relatively small compared to the time delay of conventional DE method. Simulation results show that for block length of 3200 and a relatively low code rate 3/8, LDPC codes constructed using our proposal noticeably outperform those constructed with DE method by more than 0.2dB in the entire SNR range under AWGN channel.
1 Introduction
Gallager introduced the concept of low-density parity-check codes (LDPC codes) in his thesis (1962), together with several iterative decoding algorithms. For an introduction to the basic knowledge of LDPC codes we refer the readers to [1]-[5]. The iterative decoding algorithms are best interpreted as message-passing algorithms executed on a bipartite graph which represents the code. Messages are passed along the edges of the graph between variable nodes and check nodes. Every variable node corresponds to one bit of the codeword and each check node corresponds to a linear constraint defining the code.

A simplex is a geometrical manifold consisting of 2(l-1) vertices, with each vertex representing a valid symbol-nodedegree distribution. We need a cost function to assess the
quality of a valid symbol-node-degree distribution / , and
then minimize this function by manipulating the simplex. In
general, for a symbol-node-degree distribution / , there are
two types of cost function evaluations for assessment, the density evolution based and the Monte-Carlo simulation based. For details about density evolution method we refer the readers to [11]. In [8], it was suggested to use Monte-Carlo simulation result as the cost function evaluation. More

specifically, for a valid symbol-node-degree distribution / , a PEG graph in accordance with / , is first constructed. Then evaluate the average block-error rate at a certain Eb / N0
using Monte-Carlo simulation with a prescribed number of
blocks. The value of Eb / N0 is initialized with a relatively
large number, then gradually decreases in small steps whenever a nonzero block-error rate is encountered. The final
Eb / N0 is called the Eb / N0 threshold, and used as the cost
function evaluation for a given symbol-node-degree
distribution / in [8].
However, we find this Eb / N0 threshold approach too
complex for cost function evaluation. It requires normally ten
or more Monte-Carlo simulations to get a Eb / N0 threshold
sufficiently precise assuming the initial Eb / N0 and its
decreasing step are both properly selected. This significantly lowers the efficiency of the evaluation procedure. In our proposal, we use a different and more straightforward
approach: for a given symbol-node-degree distribution / , the
block-error rate of the LDPC code constructed in accordance
with / at a fixed Eb / N0 in Monte-Carlo simulation is chosen directly to be the cost function evaluation for / . In
our implementation, the number of LDPC blocks for
simulation is not limited. Once a prescribed number $ of err-
blocks are detected the simulation stops and the block-error
rate is calculated. As long as $ is large (in our approach, it
was set to 50), the block-error rate calculated through simulation should be valid enough statistically. The advantage of our approach lies in that it doesn’t need multi-simulations to get a cost function which considerably improves the efficiency of the searching procedure. To further speed up this
process, one can set the Eb / N0 in Monte-Carlo simulation a
value relatively small so that the block-error rate of the LDPC code in simulation should be large enough for the simulation to end in a reasonable time. In our implementation the
Eb / N0 in Monte-Carlo simulation is selected such that in
the entire searching process the block-error rate of the LDPC code should be around 10-1~10-3.
This downhill simplex search algorithm is considerably more efficient than traditional density evolution approach in terms of time consumption. Though the exact time cost for downhill simplex search is hard to calculate since it includes a few random factors, experience shows that it costs normally half the time needed by DE method for LDPC degree distribution design.

4 Conclusion
A general method for constructing irregular midterm block length LDPC codes has been presented. Its main principle is to optimize the variable node degree distribution of LDPC Tanner graph. Numerical experiments show that our proposal is significantly more efficient than density evolution method for LDPC Tanner graph optimization in terms of time consumption. Moreover, simulation results show that for midterm block lengths, LDPC codes constructed using our proposal outperform those constructed with DE method in the entire SNR range. The performance gap between codes constructed using our proposal and DE method grows larger as the code rate decreases. For code rate 3/8, LDPC codes constructed using our proposal outperform those constructed with DE method by more than 0.2dB in the entire SNR range. As a result, our proposal, the simulation based downhill simplex search is a more efficient and powerful tool for designing LDPC codes with midterm or short block lengths than DE approach for all code rates.
