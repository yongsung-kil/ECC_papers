A Quasi-Parallel Encoder of Quasi-Cyclic LDPC Codes in IEEE 802.16e

Zhuo Ma
State Key Lab of ISN, Xidian University,
Xi’an, 710071, Shaanxi, China zma@mail.xidian.edu.cn

Abstract—We present a quasi-parallel LDPC encoder based on the quasi-cyclic characteristic of the parity check matrix of the LDPC code in IEEE 802.16e. A bidirectional recursion Arithmetic is used to improve its timing requires. The encoder is designed with simple shift registers and mod-2 adders, which is easy to be implemented in FPGA. By taking into account of some intermediate results, a low cost, high performance encoder is realized. The results show that, for the half rate LDPC code with code length 2304, the designed encoder can work at 50MHz on Xilinx’s XC3S1000 FPGA, With total equivalent gates consumes of 65964 and encoding throughput of up to kb*50 Mb/s.
Keywords- Quasi Cyclic Low Density Parity Check Code; QuasiParallel Encoder; FPGA implementation;
I. INTRODUCTION
Much attention has been given to LDPC codes recently due to their Linear decoding complexity and excellent performance which approach Shannon’s channel capacity [1]. However, there are some restrictions that limit the applications of LDPC codes in realistic systems, such as the encoding complexity and the delay caused by it. Motivated by these problems, Tanner proposed a type of quasi-cyclic LDPC (QC-LDPC) codes [2][3]. The parity-check matrix of QC-LDPC codes is composed of some circular shift matrices, which is highly adapted to VLSI implementation.
Based on the quasi-cyclic characteristic of the parity check matrix of the LDPC code in IEEE 802.16e [4], a new encoding algorithm is proposed in this paper.
II. DEFINITION OF THE LDPC CODES IN IEEE 802.16E

Each LDPC code in the set of LDPC codes is defined by a parity-check matrix H of size m-by-n, where n is the length of the code and m is the number of parity check bits in the code. For this parity-check matrix, three kind of encoding methods are proposed in IEEE802.16e/D8, which are serial encoding, fully parallel encoding and parallel encoding using multiplication of several small matrices. Serial encoding is simple but inefficient to be used in high data rate communication systems. Encoding throughput can be greatly improved by using fully parallel encoding, while the hardware complexity will be excessively grown. By decomposing original matrix into several small sub-matrices[5], one can reduce the implementation complication as well as achieve high throughput. But the encoding complexity will grow exponentially with the order of the sub-matrices, thus have restricted block size and code length.
For getting a great improvement in throughput and complexity, a bidirectional recursion scheme is introduced for encoding of this kind of LDPC codes.

IV. HARDWARE IMPLEMENTATION OF QUASI-PARALLEL LDPC ENCODER
Since the quasi-cyclic LDPC code check matrix is composed of some circular shift unit metrics, how to implicate the matrix multiplication of a input sequence by a circular shift unit matrix efficiently become the key problem of designing the encoder. Observed that the multiplication of a input sequence by a circular shift unit metrics is equivalent to circular shift the sequence by shift times of the circular shift unit metrics, Shift register is proposed to carry out the multiplication [6], but without any implementing structure. Based on the bidirectional recursion encoding arithmetic, a LDPC encoder composed of a kind of structure called two-state shift register will be introduced.
A. Implication of matrix multiplication with two-state shift register shift registers
In our design, circular shift unit matrix multiplication is realized by two-state shift register. First, the register perform arithmetic shift to store data to be multiplied, which is called storing state. Then the register perform circular shift, and the mth bit of the shift register output as the multiplying result, which can be called calculating state. Further more, many circular shifting with different shift times of the unit matrix can be implicated within the same register, which is showed in figure 1, where the shift times is 2, 4, and 8.

PCBG module consumes mb+1 two-state shift registers with order z and mb mod-2 adders. This module using the output of DSPE module to calculates v(0) and v(1) to v(mb-1), where formula (9) is used. v(0) is calculated from w(i) at the
same time as w(i) is shifted in, and is stored in a two-state
register synchronously.
Hardware structure of Quasi-parallel LDPC encoder is showed in figure 2, where two groups of DSPE module and PCBG module is used. They work alternately to encoding the input data continuously. The Mod-z clock divider divides the input clock by z, its output is used as the state select signal. The output of DSPC module and PCBG module is combined by a output register, forming the kb+mb bits coded data.

V. CONCLUSION
Based on the bidirectional recursion encoding arithmetic, the proposed quasi-parallel LDPC encoder is divided into two

modules: DSPE and PCBG, both implemented by two-state shift register. This scheme reduces the encoding complexity and improves the coding efficiency. This structure also has very good scalability: LDPC codes with different code length can share one two-state shift register, where the only distinctness is in the position of their MSB. Although this encoding scheme is proposed for LDPC code for IEEE.802.16e in this paper, the principle can be used for all Quasi-Parallel LDPC codes.
