EXPERIMENTS ON DECODING LDPC CODES USING TREES AND RANDOM FORESTS
Jake Gunther, Andrew Pound, Todd Moon
Department of Electrical and Computer Engineering Utah State University
Logan, Utah 84322-4120

ABSTRACT
This paper explores two alternatives to the standard message passing approach to decoding LDPC codes: regression trees and random forests. Regression trees are capable of approximating complicated functions. This paper reports on the performance of a tree-based low-density parity-check code (LDPC) decoder in which a regression tree is trained to approximate the message passing update function. This approach offers potential advantages over the message passing decoder. Single trees as well as groups of trees (i.e. forests) produce their outputs using only threshold comparisons. Therefore, the proposed decoders are arithmetic free.
Index Terms— LDPC decoding, regression tree, random forests
1. INTRODUCTION
Originally proposed by Gallager [1], low-density paritycheck (LDPC) codes have been shown to perform very close to the Shannon limit [2]. Due to the power of these codes and the availability of a reasonably low complexity decoding algorithm, LDPC codes are being adopted in industry standards and decoding hardware is being built. This paper explores two alternatives to the standard message passing decoder. One offers reduced computational complexity per iteration and the other decodes in a single step. The performance of the proposed techniques are compared to message passing decoding through simulations. This paper explores the use of regression trees and random forests to decode LDPC codes.
2. REGRESSION AND PREDICTION PROBLEMS

If the value to be predicted y is discrete or categorical, then the problem is referred to as classification. In general, the measured variables may also be continuous, discrete, or categorical. However, in this paper, the xi will always be continuous variables.

3. REGRESSION TREES

A simple regression tree [4] is a tree-structured function t that maps a vector of inputs x = (x1, · · · , xM ) to a scalar output y. At each node in the tree a binary decision is made by comparing one of the input variables xj to a threshold. If xj is less than the threshold, then the tree is descended downward along the left branch, otherwise the right branch is taken. The process is repeated at a node in the next level of the tree. The tree is traversed from the root node to a leaf node along a single path that is determined by the M variables in the vector x. Each leaf node stores an output value c. When a leaf node is reached, the c value is output. Through this process, regression trees effectively look up rather than compute the output value corresponding to a given input vector. A balanced tree with K leaf nodes1 can look up its output in log2 K comparisons. Regression trees provide an efficient (arithmetic-free) way to look up the output value.
The index of the comparison variable j and the threshold at each tree node are determined in an initial training phase. The output constants c in the leaf nodes are also found through training. For a discussion of the tree training procedure, see [5] in the proceedings of this conference.

4. RANDOM FORESTS
A random forest [6, 7] is a collection of trees. The trees in a random forest are usually simple relative to a fully grown regression tree. An element of randomness is integrated into the training of the trees in a random forest. When training a regular tree, the decision variable and corresponding threshold is found after a search over all input variables and over every possible threshold for each variable given the training data. When training a random tree, a set of three to five variables are randomly selected for the optimization process at each node. A second layer of randomness is introduced into training by randomly selecting from the full training set a subset
1By a balanced tree we mean one in which the leaf nodes are uniformly distributed among the branches of the tree.

of samples for training each random tree. The remaining samples are used in a cross-validation step to examine the quality of the tree thus trained. Random forests typically consist of 100 to 500 trees.
Random forests gain their prediction power by aggregating the outputs from a large number of trees. In a regression problem, i.e. predicting a continuous output variable given a vector of inputs, the output of the random forest may be obtained taken as the average of the outputs of all the trees in the forest. A variety of other possibilities are also possible. In a classification problem, the forest output decision may most easily be obtained through a majority vote of the trees in the forest. If a “soft” output is desired, the fraction of trees that voted in favor of a certain class could be used.
5. TREES AND FORESTS AS FUNCTION APPROXIMATORS
Based on the discussion above, it is evident that both trees and random forests can be trained to predict either continuous variables (for regression) or categorical variables (for classification). The typical usage of trees and forests is in machine learning. Here a data set is given and trees and forests are used to learn the function implicitly defined in the data.
In this paper we propose to use trees and forests to learn and approximate a specific function. Whereas in machine learning training data are given, here we are given a known function and use trees and forests to approximate that function. The specific function that is to be approximated here is involved in the decoding of LDPC codes. The data used for training is generated using the known function. In this way, the trees and forests are trained to approximate a specific function.
The motivation for this concept arises from the fact that trees “compute” their outputs without actually doing any computation. Only threshold comparisons are required. Therefore, if a given function can be approximated accurately enough for a given application, then trees offer an arithmetic free way to perform processing. As can be seen from (1), trees output a constant value ck for every input x in a given region Rk. By using a fine enough partition of the input space, a function can be approximated arbitrarily accurately. As the size of the tree grows, the approximation error decreases toward zero, but the memory required to store the tree parameters increases. Therefore, the application domain for tree-based approximations are those problems in which some quantization error is acceptable and where functions can be represented by reasonably small trees2. It is difficult to know in advance which domains possess these desirable

6.1. Iterative Decoder Approximation

Our goal was to replace the evaluation of f and g by a regression tree comparison-based look up. If such a replacement were possible, then LDPC decoding could be accomplished without performing any arithmetic.
Our approach to accomplishing the full goal has been to gradually replace some of the operations in the decoder equations by tree-based look ups. For example, early experiments have shown that simple linear and multiplicative transformations can be replaced by trees with very little loss in accuracy. This results in a savings in arithmetic. The core of the update consists of mappings of the form in (5), (6) and (7) composed with the affine transformation in (8). In the following, capability of modeling this highly nonlinear transformation using regression trees is explored.

7. CONCLUSIONS
This paper explored the idea of using a regression tree as a means to replace the arithmetic calculations of Gallager’s LDPC decoding algorithm with an efficient look up requiring only logical comparisons. Random forests were also proposed for the same task and are the subject of future investigations. Regression trees were trained to mimic a key calculation in the Gallager decoder. Tree based decoding was successful. Often the tree decoder required more iterations that the Gallager decoder, but the per iteration complexity was reduced due to the elimination of many double precision multiplies and adds. Further research will explore the trade off in tree size on the error rate of the decoded output.
