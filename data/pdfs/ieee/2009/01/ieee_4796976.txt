Design and Implementation for High Speed LDPC Decoder with Layered Decoding

Hong Ding, Shuai Yang, Wu Luo*, Mingke Dong Satellite & Wireless Communication Lab
National Laboratory on Local Fiber-Optic Communication Networks & Advanced Optical Communication Systems,
School of EE & CS, Peking University, Beijing, China, 100871 *Corresponding Author: luow@pku.edu.cn

Abstract
This paper presents a layered decoding algorithm for LDPC code and analyzes the advantages and challenge for high speed implementation. Simulation result indicates that layered decoding algorithm converge two times faster than traditional decoding algorithm. A new code construction scheme aimed to design high speed LDPC decoder with layered decoding algorithm is put forward. This code construction method helps to increase the parallel degree of decoder by suffering with little performance loss. Using the layered decoding algorithm and code construction scheme, a length 2304, rate 1/2 LDPC decoder which can achieve about 768 Mbps information throughput has been implemented on FPGA platform. Index Term--LDPC, layered decoder, high speed
1. Introduction
Low-density parity check (LDPC) code is a kind of forward error correction code (FEC) which has a very sparse parity check matrix. It shows excellent performance and can approach the Shannon limit. It was first discovered by Gallager in 1960s’ [1]. However, it was ignored for about thirty years because its coding and decoding algorithm was considered too complicated at those times. As the development of the computer science, MacKay and Neal [2] rediscovered LDPC code in 1996 and showed its advantages over Turbo Code.
Nowadays, LDPC code is considered as the coding scheme for next generation communication system. Because it is flexible to choose code parameters and has good balance between performance and complexity of implementation, LDPC code has already been

adopted by several protocols such as the digital video broadcasting (DVB-S2) via satellite [3]. Wireless local area network (WLAN), World Interoperability for Microwave Access (WIMAX) [4], etc.
Due to the requirement of high speed transmission in the future communication system, many researchers are attracted to implement the high speed LDPC decoder. Quasi-Cyclic (QC) LDPC code is a special type of LDPC codes designed for hardware implementation. The parity check matrix of a QCLDPC code is constituted by many squared blocks. Each squared block is generated by circularly cyclic shift of an identity matrix. The QC-LDPC code structure significantly reduces the interconnection complexity and is easy to be implemented. QC-LDPC code can be constructed by progressive edge-growth (PEG) algorithm, which is proved to be simple and effective for designing good LDPC codes [5].
Zhiyong Cui has implemented a QC-LDPC (8176,7156) decoder on the FPGA which adopted partially parallel structure and can reach over 170Mbps information decoding throughput at 15 iterations [6]. However, its code rate very high. Ahmad Darabiha has implemented a fully parallel decoder with the bit serial min-sum decoding algorithm which achieves about 650Mbps information throughput [7]. The lack of the design is that it costs too much resource and only fits for the short length code. Montreal has implemented a 706 Mbps LDPC decoder with Stochastic algorithm for the code length of 1024 [8]. Unfortunately, the reliability of this kind of decoding algorithm is not proved for long length code.
In this paper, we first present a layered decoding algorithm which is well suited for implementation of high speed LDPC decoder, and its benefits and limits are analyzed. Secondly, a constraint is added to PEG algorithm to enhance the parallel degree of this

Because it only has one phase, the clock cycles required are half of the Min-Sum algorithm in each iteration. In addition, Figure 1 shows that the convergent speed of layered decoding is two times faster than Min-Sum algorithm. Therefore, we can theoretically get four times benefits when implementing the high speed LDPC decoder with layered decoding structure.
However, there are some challenges for layered decoding algorithm while designing for high speed LDPC decoder. Because the row processor and the column processor work jointly, the rows which have intersection (there are non-zero elements at the same column) cannot be processed simultaneously. In addition, because the processing result will be calculated out in several clock cycles later, the

The strategy will be discussed in section 4.
3. Code Construction for Layered decoding
The performance of LDPC code is mainly decided by the shortest cycles in the parity check matrix. Progressive Edge Growth (PEG) algorithm is aimed to make each variable node has the maximal local girth, so it is an efficient way to construct the LDPC code with good performance. However, the computation complexity of the PEG algorithm is quite large.
While using the PEG algorithm to construct the implementation-oriented QC-LDPC code, the complexity is reasonable. The procedure of PEG algorithm for QC-LDPC code is as follows: 1. Ordering the variable nodes from small to large by
column degree. 2. Choosing the check nodes connect to the variable
nodes according to the rule. 2.1 expand the sub-graph from the variable node. 2.2 choose the check node which is farthest from
the variable node and has the lowest degree. 3. Select the offset value which leads to longest girth.
As discussed in section 2, the rows which have intersections cannot be processed at the same clock cycle. In order to increase the degree of parallelism, it is necessary to construct a LDPC code with more rows which are not intersected. A small constraint has been added to PEG algorithm to achieve this goal.
In step 2 of the PEG algorithm, all the check nodes is the candidate for the variable node to connect with. We can add the constraint that only the odd index of check nodes are the candidates for the odd index variable nodes and only the even index of check nodes are the candidates for the even variable nodes. The constraint of the code construction is shown in Figure 2. In the figure, it is a 32×64 QC-LDPC parity check matrix. The block size is 4×4. The black blocks represent the candidate check nodes for variable nodes. First, the left-part of the matrix is decided as the double diagonal matrix. While choosing the check nodes to connect with index 9 variable node, only

index 2,4,6,8 check nodes are candidates, and for index 10 variable node, index 1,3,5,7 check nodes are candidates, and so forth. By adding this constraint, we can easily find a block row permutation in which the successive rows have no intersection. In this example, [1 6 3 8 5 2 7 4] is a possible permutation. This permutation is corresponding to the processing order in the actual layered decoding.

4. FPGA Implementation of the High Speed Decoder

Based on the layered decoding algorithm, we have implemented a LDPC decoder of code length 2304, code rate 1/2 and size of block 96. To achieve the high speed design goal, the Altera Stratix2 EP2S180 FPGA is chosen as the platform. It can provide as many as 143,520 ALUTs, and 9,383,040 bits RAM. The CVNU processors read the LLR and exterior information from separate RAM, and do the joint row and column operation. The result is written back to the RAM and the sign of the updated LLR is written to the output RAM. When all the parity check equations are satisfied, the decoder starts to output.

4.1. Storage of the Channel Information
As discussed in section 2, when the degree of parallelism is high, conflicts may exist while accessing the RAM for channel information. To reduce the number of blocks of RAM, the channel information with no conflicts should be stored in the same RAM. In the parity check matrix, each column corresponds to one channel information. If the columns have no intersection, the corresponding channel information can be stored in the same RAM, because the channel information will not be read at the same clock. The maximum row degree is seven, so the minimum number of blocks needed to store the channel information is seven. Figure 5 is an example for this scheme. It is a parity check matrix expression for QCLDPC code. The non-zero element represents the offset of the identity matrix. The zero element represents all-zero block. The 1, 2, 5 column have no intersection, so the 1, 2, 5 block of channel information can be stored in the same RAM. For the same reason, the 3, 4 , 6 block can be stored in the same RAM, the 7, 9, 11 block and 8, 10, 12 block can also stored in the same RAM separately. The code construction method indicated in section 3 can obviously decrease the RAM blocks.

4.2. Joint Row and Column Processor

The joint row and column processor is the key computation unit for LDPC decoder. It accounts great importance to the total resources and the maximum frequency of the decoder. The number of clock cycles to calculate out the result should be as small as possible in order to increase the degree of the parallelism. On the other hand, the fewer the clock cycles required, the more resources are needed. Therefore we choose the structure indicated by Figure 6 to balance both of the factors. 5. Conclusion

In this paper, a layered decoding algorithm based on some modification of sum-product or min-sum

algorithm has been adopted to implement the high speed LDPC decoder. In the following, we have analyzed the advantages and limits of the algorithm. A scheme of code construction for implementation with layered decoding is proposed. Finally, a length 2304 and rate 1/2 LDPC decoder which can reach 768Mbps information throughput is implemented. The key points of the design are discussed and the result is given.
