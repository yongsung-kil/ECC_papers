A High-Parallelism Reconfigurable Permutation Network for IEEE 802.11n?802.16e LDPC Decoder
Zhixiang Chen*, Xiongxin Zhao, Xiao Peng, Dajiang Zhou and Satoshi Goto** Graduate School of Information, Production and Systems, Waseda University, Japan
*E-mail: zhixiangchen@ruri.waseda.jp **E-mail: goto@waseda.jp

Abstract—In this paper, we proposed a high-parallelism permutation network (PN) based on Benes network (BN) for LDPC decoder applied for IEEE 802.11n and 802.16e standards. By exploiting the symmetric property of BN, several groups of data can be cyclically shifted concurrently by our proposed network. Compared to the previous works our proposed PN achieves up to 4 times parallelism while maintaining small implementation area and high frequency. Additionally, based on the proposed PN, we present the architecture of a highparallelism decoder of which the throughput does not decrease when decoding a short code.
I. INTRODUCTION
First invented by R. Gallager [1] in the 1960s and rediscovered by D.J.C. MacKay [2] in the 1990s, Low Density Parity Check Codes (LDPC) as a forward error correcting code is now playing an important role in modern digital communication area not only because of its excellent bit error ratio (BER) performance, but also its high information throughput. Structured LDPC codes such as quasi-cyclical LDPC (QC-LDPC) codes have received significant attention since their friendliness to hardware implementation features and acceptable bit error ratio (BER) performance compared to the original random codes. Recently, QC-LDPC codes were included in IEEE 802.11n and 802.16e standards [3][4] which support multiple code rates and code lengths.
To support the multiple code rates, Sun [5] proposed new architecture of LDPC decoder which can be reconfigured for variable parity check matrix (PCM) on the fly.
To support the multiple code lengths, a reconfigurable permutation network (PN) is needed to cyclically shift data of variable group sizes according to the QC-LDPC PCMs and code lengths. Previous work has been done in [6] and recently some Benes network (BN) [7] based permutation networks have been reported in [8][9]. Unfortunately, all of the early works can only cyclically shift one group of data at one time even the size of the data group is much smaller than the input size of the network. This demerit leads to low throughput for LDPC decoder when the code length is small (small data group size).
In this paper, based on BN and the control algorithm proposed in [9], we propose a high-parallelism permutation network with new control algorithm. Several groups (up to 4)

of data can be cyclically shifted concurrently by the proposed network when the size of the data group is no bigger than half or quarter of input size of the PN. Based on the proposed PN, new architecture of a high-parallelism decoder is proposed to solve the low throughput problem for short code length.
The organization of this paper is as follows. Section II briefly introduces the related background knowledge of QCLDPC code and BN based permutation network. In section III our proposed BN based permutation network with its control algorithm is presented. The architecture of a high-parallelism decoder is briefly discussed in section IV. Section V concludes the paper.
II. BACKGROUND KNOWLEDGE
A. QC-LDPC Codes in IEEE 802.11n and 16e An LDPC code is defined by a parity check matrix (PCM)
the size of which is M*N where N and M equal to the code length and the number of redundant bits (parity bits) in a code block respectively. Quasi-cyclic (QC) LDPC code, the PCM of which is a combination of smaller sub-matrix consisting cyclically shifted identity matrix and all-zero matrix, is one of the structured LDPC codes invented for easy hardware implementation. QC-LDPC matrix can be represented by a base matrix consisting of positive integers and “-1” where the positive integer represents an identity matrix cyclically shifted by it and “-1” represents an all-zero matrix. An example of QC-LDPC is given in Fig. 1 where N=12, M=6 and the size of a sub-matrix is 4 by 4.

III. PROPOSED PERMUTATION NETWORK
All of the previously proposed PNs have a common demerit that is only one group of data can be shifted at one time even if the size of the data group is much smaller than the input size of the network. Because of this demerit, the throughput of the decoder will become very low when it is processing a short code. Note that throughput is proportional to the sub-matrix size which equals to the size of the data group fed into the permutation network.
As shown in Fig. 5a, if the expansion factor is z, only z data from memory are shifted and only z PEs (short for processing engine) are actively working at one time resulting low throughput and low hardware usage. For example, assume there are n PEs, if the code length is 576 (z=24) the throughput will be almost one-fourth of that the code length is 2304(z=96) and the PE usage will be only 25% (n=96). IV. A HIGH-PARALLELISM DECODER ARCHITECTURE
In this section the architecture of a high-parallelism decoder based on our proposed permutation network is briefly discussed.

The basic idea of this algorithm is to divide a big shift task to two small shift tasks and then combine their results together. For example if we want to cyclically shift one 2Ksize data group by 2s positions, we divide it to two K-size data groups and shift them by s positions respectively. The final result can be obtained by combining two shifted K-size data groups together. Based on the symmetric property of BN, two or four groups of data can be shifted concurrently by appropriately copying the control signals of cross bar switches that is listed in ALGORITHM I.

Fig. 7 Top level of proposed decoder architecture
As shown in Fig. 7 the main PN is used to cyclically shift logarithmic likelihood ratio (LLR) messages from LLR memory to appropriate PEs according to the PCMs which are stored in the ROM of the controller. The word width of the main PN equals to LLR message’s width which is 8 bits. The input size of main PN is set as 128*128 to support the biggest code length (2304 bit) in IEEE 802.16e of which the cyclically shifted data group size is 96. V. CONCLUSIONS
In this paper, we have proposed a Benes network based high-parallelism reconfigurable permutation network for LDPC decoder applied for IEEE 802.11n and 802.11e standards. The proposed PN achieves up to 4 times parallelism compared to the early works while maintaining small area and high frequency (0.676 mm2 and 177 Mhz @180nm). Meanwhile, the architecture of a high-parallelism decoder based on the proposed PN has been discussed and its throughput gain has been showed.

When the code length is 1296 (one 802.11n code) shown in Fig. 8(b), the decoder processes two independent blocks of codes (code 0 and code 1) whose LLR messages are stored in 108 blocks of memory divided into two groups. The LLR messages of each code are updated by 54 PEs in parallel so the PE usage is about 84% (54*2/128). A one-bit word width slave PN in Fig. 7 will be needed to output the decoded bits from LLR memory if that code achieves decoding success. To show the throughput gain of the proposed decoder architecture, we define the throughput of a conventional decoder working at a fixed frequency and decoding a (2304, 1152) code with a fixed maximum iteration time as one unit, then draw unified throughput to code length histogram for 0.5-code-rate 802.16e codes in Fig. 9.
Fig. 9 shows that the throughput of our proposed decoder is two to four times of the conventional one when the code length is small. The biggest throughput, 1.33 units, can be achieved if the code length equals to 768 or 1536 for which all the PEs in the decoder are active.
