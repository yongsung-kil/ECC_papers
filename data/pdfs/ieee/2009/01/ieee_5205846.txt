Abstract-We study an integer programming (IP) based separation approach to find the maxim um likelihood (ML) codeword for binary linear codes. An algorithm introduced in Tanatmis et at. is extended and improved with respect to decoding performance without increasing the worst case complexity. This is demonstrated on the LDPC and the BCn code classes. Moreover, we propose an integer programming formulation to calculate the minimum distance of a binary linear code. We exemplarily compute the minimum distance of the (204, 102) LDPC code and the (576,288) WIMAX code. Using the minimum distance of a code, a new class of valid inequalities is introduced.
I. INTRODUCTION
Linear programming (LP) decoding of binary linear codes was introduced by Feldman et al. [6]. LP decoding is motivated by the integer programming formulation of the maximum likelihood decoding problem which is NP-hard [1]. In the IP formulation, a linear function is minimized over the set of codewords C. Equivalently this linear function can be minimized over the convex hull of C i.e., conv( C) or the codeword polytope, which results in an LP formulation. In general, linear programming is polynomially solvable but computing the convex hull of C is intractable. Therefore in decoding algorithms based on linear programming, the linear cost function of the IP formulation is minimized on a relaxed
polytope P :2 conv( C). The relaxed polytope is easier to
describe and its vertices, the set of pseudocodewords, is a super set of the set of codewords. Consequently an integral LP optimum is the ML codeword [6].
Various LP formulations are used to describe P explicitly in [2], [6], [12], [13]. One common approach is to describe P via Forbidden set (FS) inequalities [4]. A set ofFS inequalities can be derived from each row of the parity check matrix H. The FS inequalities aim at forbidding the bit configurations which are not in the local code defined by a particular row. Note that the FS inequalities and the trivial inequalities 0 :::; x :::; 1, describe the convex hull of a local code completely and nonredundantly [6]. Several LP based decoding algorithms [4], [5], [6], [7], [9], [11] use the FS inequalities.
Redundant parity checks (RPC) are given by modulo 2 addition of two or more rows of the parity check matrix H. These checks are redundant in the sense that they do not alter the code. However, they induce forbidden set type inequalities in the LP formulation which tighten the relaxed polytope. Thus achieving better decoding performance at the cost of increased

complexity. In the algorithms of [6], [7], [9] and [11] the FS inequalities are also derived from the RPCs in order to tighten the relaxed polytope. In general it is not possible to describe the codeword polytope completely by using only inequalities of forbidden set type. The dual of the (7,4,3) Hamming code is a counter example [6].
We introduced the NEW SEPARATION ALGORITHM (NSA) in [11] as a solution approach to find the ML codeword. In the (NSA) we solve the relaxation of a novel IP formulation. In case the optimal solution is non-integral, we efficiently search for FS type of inequalities which cut off the non-integral optimal solution. We derive the FS inequalities from alternative representations of the code. This can also be interpreted as deriving FS inequalities from the RPCs. If the cut generation procedures of the NSA succeed, we add the new constraints to the LP formulation and resolve the LP problem. The NEW SEPARATION ALGORITHM stops if the optimal solution is integral or no FS inequality cutting off the non-integral optimal solution can be found. We demonstrated in [11] that the NSA performs better than the Belief Propagation (BP) algorithm on the Tanner's (155,64) LDPC code. We also showed that the NSA performs well on the (63,39) BCH code. Note that BP decoding does not work for this type of codes due to the dense structure of the associated parity check matrix.
The inequalities which are satisfied by all codewords but cut off the non-integral optimal solutions of the LP, are referred to as valid cuts. We continue our theoretical and numerical study on developing efficient methods to derive valid cuts in this paper. One of our major interests is the trade-off between the computational effort required to find new valid cuts and the decoding performance. Using the FS type of inequalities the NEW SEPARATION ALGORITHM is extended to include more cases for which valid cuts can be detected and constructed. We refer to the improved NSA as the REVISED SEPARATION ALGORITHM (RSA). It is notable that the computational load of our improvements do not change the worst case complexity of the NSA. We numerically demonstrate on three LDPC codes((64, 32), (160,80) irregular LDPC [10], (155,64) Tanner code) and (63,39) BCH code, that the REVISED SEPARATION ALGORITHM improves the NEW SEPARATION ALGORITHM in the range of 0.2 to 0.5dB. Moreover for the (64,32) LDPC code [10] and the (63,39) BCH code RSA approaches ML decoding.

We also make a first attempt towards a new class of inequalities, so called Minimum Distance (MD) inequalities, which can be derived from the columns of a parity check matrix. The MD inequalities are valid cuts and an algorithm to derive MD cuts from alternative code representations is developed. The minimum distance of a code has to be known to construct MD type of inequalities. Although the problem of finding the minimum distance of a code is in general intractable, for several codes this measure is known (e.g. some codes in the BCH code class). For some codes the minimum distance can be calculated by solving an integer programming problem. We propose an IP formulation and compute exemplarily the minimum distances of the (204, 102) LDPC code [8] and the (576,288) WIMAX code [14].

IV. NUMERICAL RESULTS
We tested the decoding performance of RSA on (64,32), (160 ,80) irregular LDPC codes [10], (155,64) Tanner code, and the (63,39) BCH code. We compared the results to NSA. RSA performs 0.2dB to 0.5dB better than NSA for the LDPC codes mentioned. For the (63,39) BCH code an improvement of more than 0.5dB is achieved as can be seen in Figure 1. Note that RSA approaches ML decoding for this code.
The minimum distance of a binary linear code can be computed by solving the IP problem given in Section lII-B by CPLEX [3] (a commercial IP solver). It is found out that the minimum distance of the (204, 102) LDPC code [8] is 8 and the minimum distance of the (576,288) WIMAX code [14] is 13.
V. FUTURE RESEARCH
Polyhedral analysis of the codeword polytope should be continued. It is interesting to identify the inequality classes defining the facets of the codeword polytope. The MD inequalities may offer new perspective in this direction. It may also be possible to find efficient cuts by combining the FS inequalities with the MD inequalities.
