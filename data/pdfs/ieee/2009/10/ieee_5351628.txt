Implementation ofLDPC decoder for 802.16e
Xiao Peng *, Satoshi Goto *

Abstract - The implementation complexity of the decoder for Low-density Parity-check Codes (LDPC) is dictated by memory and interconnection requirements. In this paper, we investigate the approaches to realize Turbo Decoding Message Passing (TDMP) algorithm. We compare the performance and implementation complexity of original approach, Jacobian approach, normalized min-sum approach and offset min-sum approach which are targeted for QuasiCyclic (QC) LDPC code defined in IEEE 802.16e standard. The normalized and offset approaches are more suitable for hardware implementation, which are realized on the FPGA. 1.
Index Terms - LDPC, min-sum, TDMP
I. INTRODUCTION
LDPC codes were originally proposed in 1962 by Robert Gallager[ 1]. Since then, since the contemporary investigations in concatenated coding overshadowed LDPC codes and the hardware of that time could not support effective decoder implementations, LDPC codes remained largely unstudied for over thirty years. Until the Mackay published his work in 1998, LDPC codes began to be strongly promoted.
There are potential advantages to LDPC codes which are discussed in literatures. Firstly, both abstractly and practically, LDPC codes have been proved to be capable of closely approaching the channel capacity. Secondly, LDPC codes have better performance than that of turbo codes which are also popular channel coding approaches in some cases, with iterative decoding algorithms which are easy to implement, and are also parallelizable in hardware. Thirdly, LDPC codes of almost any rate and block length can be created simply by specifying the shape of the parity check matrix, and the flexibility in rate is obtained only through considerable design effort. Fourthly, because the validity of a codeword is validated by its parity checks, even when errors do occur, they are almost always detected errors especially for long codes. Lastly, on the commercial side, LDPC codes are not patent protected.
For LDPC decoding, the main methods can be divided in to two kinds: one is the hard-decision and the other one is softdecision decoding. The former one was proposed by Gallager, and is always used to be an introduction example of LDPC codes. And the latter one would lead to much better results, and this paper will only talk about this method. Since soft-

decision decoding was discovered independently several times and as a matter of fact comes under different names. The most common ones are the belief propagation algorithm (BPA), the message passing algorithm (MPA) and the sum-product algorithm (SPA). MPA will be used in the following discussion.
Also, MPA could be classified into two categories: two phase message passing (TPMP) and turbo decoding message passing (TDMP) which is proposed by Mansour, et al[2]. LDPC codes are decoded iteratively using the TPMP algorithm was proposed by Gallager as well. It computes the probability values with each bit-node and the probability values with the each check-node iteratively. Each iteration is consists of two phases of computations: in the horizontal phase, messages in the form of probability vectors are passed to the check nodes, where the messages are combined, and in the vertical phase, messages in the form of probability vectors are passed to the bit nodes, where the messages are combined. Updates in each phase are independent and can be parallelized.
Up to now, there are many works on the TPMP algorithms and its variations, which come down are log likelihood ratio (LLR) MPA, a posterior probability (APP) MPA, min-sum MPA, normalized MPA and offset MPA. While the TDMP, which is also called as layered decoding algorithm, basically treat the parity check matrix as horizontal layers and update the extrinsic messages layer by layer. This layered decoding algorithm can improve the decoding convergence time by a factor of two and hence double the throughput.
In this paper, we investigate the approaches to realize TDMP algorithm and compare the performance and the hardware complexity of original approach, Jacobian approach, normalized min-sum approach and offset min-sum approach. During the investigation of these approaches, we point out the suitable approaches for implementation are normalized and offset min-sum approaches, which are implemented on the FPGA for IEEE 802.16e LDPC codes.

As shown in Fig.2, the TDMP decoder uses partially parallel iterative decoding architecture.
When the input code stream feed into the decoder, the buffer unit executes the pre-processing in order to give the rate and length information to the control unit and give the code to the initial unit. The initial unit is use to calculating (1) and (2) described in the min-sum algorithm. The control unit mainly has three functions: producing the control signal of network unit based on the number of seed matrix which comes from the ROM, the sequential controlling of the pipeline and the maintaining the memory reading and writing. The PE Core is used for computing (3), (4) and (5), which is the key operation part of the decoder. Since the iterative decoding is conducted layer by layer, and in each layer there are at most 96 lines, we need 96 PE cores for running simultaneously. Compared with fully decoding architecture, this is the so-called partially parallel decoding method. The L memory is used to store the LLR information. The network unit transfers the LLR

Fig. 3. Benes Network
To realize simultaneous transferring 96 numbers, the multiplexer is not a considerable choice regarding its huge hardware cost. In our design, we utilize the Benes network for the network unit. As shown in Fig.3, (A) is the 2 X 2 Benes network[6] which is the basic unit for constructing the larger network. For this unit, if the control signal is "0", the state of the unit is bar state which means the sequence of output is unchanged, while if the control signal is "1", the state is cross state that information would pass cross to the other direction. Using this unit, the 128 X 128 Benes network is constructed for the decoder recursively: make two copies of the unit, add two more stages, one before and one after the double-height network, and connect all the new units to the existing network. The principle for connecting the units is each unit in the front and end stage should connect both of two sub-network.

v. CONCLUSION
In this paper, we investigate the approaches to realize TDMP algorithm and compare the hardware complexity of original approach, Jacobian approach, normalized min-sum approach and offset min-sum approach. During the investigation of these approaches, we point out the suitable approaches for implementation are normalized and offset minsum approaches, which are implemented on the FPGA for IEEE 802.16e LDPC codes. The implementation result shows that the performance of the two approaches are almost the same, and they are all satisfy the 802.16e standard.
