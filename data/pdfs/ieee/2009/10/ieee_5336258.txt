ABSTRACT
Low-Density Parity-Check code is a kind of near-optimal error correction code. Developing a highly efficient decoding algorithm is the main challenge of the implementation of LDPC code. In this paper, based on the min-sum column-layered decoding algorithm, a new selective computation technique is proposed to reduce the number of comparisons. With the presented decoding algorithm, the total number of floating number comparisons can be reduced dramatically (range from 81% to 91%) at the cost of negligible hardware overhead. Simulation results show that the decoding performance and iteration times are almost the same as the original column-layered decoding algorithm.
Index Terms- Low-density parity-check (LDPC) codes, column-layered decoding, shuffled decoding, error correction codes, VLSI
1. INTRODUCTION
Low-density parity-check (LDPC) codes, which are one kind of linear block codes, were first introduced by Gallager in 1962 [1] and have been verified to have superior errorcorrecting capacities. LDPC codes have been proved to be near the Shannon limit under sum-product iterative decoding algorithm (SPA) and are capable of parallel implementation for high throughput implementations. LDPC code can be well expressed by a bipartite graph called Tanner graph [2]. Many decoding algorithms have been investigated for effectively decoding of LDPC codes. Gallager's two-phase message passing algorithm [1] decodes the received information by iteratively updating the messages travelling between check nodes and bits nodes which are described in Tanner graph. Turbo-decoding message-passing (TDMP) decoding algorithm, which updates check nodes messages and bit nodes messages concurrently, was proposed in [3]. TDMP decoding algorithm, which is also called horizontal layered decoding, reduces the average number of iterations significantly because it forces the updated information be utilized more quickly. TDMP has attracted lots of attention for its

simplicity and low memory requirement. Various kinds of decoder architecture based on TDMP or aiming at message memory compaction have been proposed in the public literature, such as [4]-[6].
Similar to TDMP, another kind of layered decoding, which is called shuffled decoding, was proposed in [7]. Shuffled decoding also can reduce the average number of iterations. However, due to the inherited high complexity of shuffled decoding schedule, its VLSI implementation has rarely been studied. A min-sum based column-layered decoding and the corresponding decoder architecture was proposed in [8]. Due to the approximation of original shuffled decoding algorithm, the complexity of the columnlayered algorithm in [8] was greatly reduced at the cost of potential degradation of decoding performance.
In this paper, an improved min-sum based columnlayered decoding algorithm is presented. The check node computing process is optimized to omit most of the unnecessary comparisons so that the computation complexity can be significantly reduced. The average number of iteration remains the same. More importantly, our simulation shows that the proposed algorithm causes nearly no performance loss.
The remainder of this paper is organized as follows. Section 2 introduces the original min-sum based columnlayered decoding algorithm and its simplified version. The improved column-layered decoding algorithm and two new kinds of check node computing techniques will be presented in Section3. Section 4 gives the simulation results and final conclusions are drawn in section 5.

By using sum-product algorithm [1], the shuffled algorithm significantly reduces the average number of decoding iterations. 2.2. Simplified Column-Layered decoding Algorithm

By investigating of the computation flow of the columnlayered decoding process, an incrementally message updating algorithm was proposed in [8]. To clarify the spirit of this algorithm, it is assumed that each check node has only one variable neighbor node in each block layer. This assumption can be easily verified by the regular structure of most of these QC-LDPC codes. At the beginning of a decoding iteration, for each check node, the minimum and second minimum absolute value, which is denoted as m, and

m2, are computed from messages coming from connected variable nodes. The check-to-variable messages corresponding to the first block column layer will be directly updated using minimum values that have just been computed. In order to update the rest of those block column layers, the new m, and m: value for the corresponding layer should be generated from old mi, m2 and Qij values that have just been updated. If the newly generated Qij has the same location as m, or m-, the corresponding m, or m2 will be replaced by the new Qij. As a result, the new minimum and second minimum will be determined by one of the old minimum values and Qij. Otherwise, these new m, and m2 will be found from old m., m2, and Qij. It is obvious that the computation of the second smallest magnitude may not be the same as the original algorithm according to [8]. This may cause certain degree of decoding performance degradation while reducing the overall computation significantly.

3. IMPROVED COLUMN-LAYERED DECODING ALGORITHM
With a close study of the check node computation process, it could be found that the complexity of the original minsum based column-layered decoding algorithm mainly comes from lots of redundant comparisons. In most conditions, it is unnecessary to use all the de magnitudes of variable-to-check messages associated with the check node i to get these two smallest values.
The key idea to reduce the computation complexity of min-sum based column-layer decoding algorithm is to use only a few magnitudes, such as the approach used in [8] which only use the original first minimum and second minimum as well as the updated variable to check message. Here we propose an alternative solution which can ensure good performance while introducing small computation overhead.

However, the power consumption can be decreased because redundant computations are eliminated from the decoding process. The proposed decoding algorithm may also be very suitable for implementation based on software platform where check nodes are processed in serial. Our simulation shows that the software decoding speed of the proposed algorithm is 5 to 10 times of that of original shuffle decoding algorithm for considered codes used in this paper. In addition, the presented algorithm will be more efficient for LDPC codes with relatively high check node degree since the extra costs will be insignificant compared to the reduced computations.
5. CONCLUSION
In this paper, an improved min-sum based column-layered decoding algorithm has been proposed. The contribution of presented algorithm is the use of selective computation and its changed version can reduce unnecessary comparisons during the decoding process. By extensive simulation, it is shown that the proposed min-sum based column-layered decoding algorithm can save 81% to 90% comparisons with little performance loss. Future work includes developing of more hardware-efficient column-layered decoding algorithm which can save more redundant comparisons while maintaining the decoding performance.
