An Multi-Rate LDPC Decoder Based on ASIP for DMB-TH
Xiaojun Zhang, Yinghong Tian, Jianming Cui, Yuyin Xu, Zongsheng Lai*

Abstract-Based on ASIP (Application Specific Instruction Set Processor), this paper propose a decoder architecture for LDPC (Low Density Parity Check Codes) in the DMB-TH standard The decoder use a five-stage pipeline, 32-bit RISC processor and it can supports three different code rates (0.4, 0.6 and 0.8) by only modifying the program. Based on XC4VLX150, at the maxfrequency of 126 MHz, the max throughput ofthe decoder can achieve 96Mps for 10 TDMP-decoding Iterations. Compared with other GPP and DSP implementations, this ASIP simplify the control logical and enhance the flexibility].
Keywords-decoder; multi-rate; ASIP;DMB- TH;TDMP
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1][2], which are one kind of linear block codes, were first introduced by Gallager in 1962 and were discovered by Mackay in 1996. It is received much attention because of their excellent performance and the large degree of parallelism that can be exploited in the decoder. Provided that the information block lengths are long enough, performance close to the Shannon limit can be achieved with LDPC codes. For instance, for code rate =1/2 and codeword length of 10E+7, bit error rate of 10E-6 can be achieved with threshold of 0.0045dB Shannon limit over AWGN channel [3], which shows the best performance among ECC(Error Correction Coding) codes up to now. LDPC codes are being widely considered as the most promising candidate FEC scheme for many applications such as telecommunications, storage devices and other fields. For example, LDPC code was used as channel coding in DVB-S2, IEEE 802.11n/16e, DMB-TH [4].

II. REVIEW OFLDPC CODE ANDDECODING ALGORITHMS
A. Low Density Parity Check Codes
The (n, k) LDPC codes are one kind of block codes, where k and n represent the number of information bit and codeword, respectively. LDPC codes are defined by a very sparse parity check matrix, which the most elements are O's and only several elements are l's. The corresponding code rate is defined as r = (n-k)/n. Fig. l(a) shows that the parity check matrix of (7, 3) LDPC codes with code rate of 4/7 has 3 rows and 7 columns. In the decoding aspect, a parity check matrix can be mapped into a bipartite graph [8] or a factor graph [9], as illustrated in Fig. 1(b). Rows and columns of the parity check matrix can be mapped to check node units (CNUs) and bit node units (BNUs), respectively. The direct connection can represent the mutual information transmission among CNUs andBNUs.

C. decoding algorithms
Although LDPC codes can be efficiently decoded with belief propagation algorithm using an iterative message passing algorithm, the normalized or offset min-sum algorithm would be preferred due to its simplified hardware implementation [10]. [11] proposes the TDMP (turbo decoding message passing algorithm), which can improve the decoding speed twice, so this decoder adapts TDMP algorithm based on normalized min-sum algorithm. For TDMP, We start with a base-matrix H of Dimensions MxN. Namely, the block LDPC with M block rows can be viewed as concatenation ofM layers or constituent sub-codes similar to observations made for AALDPC codes. In TDMP, after the check-node processing is finished for one block row, the messages are immediately used to update the variable nodes, whose results are then provided

After this instruction the check nodes start to-output newly calculated messages. These messages are shifted into the original order and stored at the same address where they were read from. After processing one row of the parity check matrix, the instruction ldpc_ck tests whether all parity checks were satisfied. In this case, i.e., decoding was successful, the ASIP powers down till a new block is available for decoding. If the parity check fails and less than the given number of iterations were performed, a new iteration is started.

The ASIP architecture adapts a RISC processor, which is shown in Fig. 5. The processor is a 5-stage pipeline architecture, i.e., fetch, decoding, executing, memory and writing back. With the pipeline architecture, it can shorten the executing period of processor. The architecture of decoder based on ASIP is depicted in Fig. 6, which is can be divided into four parts, namely, decoding, memory operation, updating nodes and hard decision. During the decoding, the processor generates the control signals. Based on the control signals, the memory ram_c and ram_v is read and written. The data from ram v and ram c are read into tern_v and tern_c, respectively. The data in temp_v through shffle network is read into updating units. v. CONCLUSIONS
A LDPC decoder based on RISC processor for DMB-TH is presented. Which make use of the character of check matrix (AA architecture), and decode the codes with one block row of base check matrix once. With some special instructions, the performance is greatly improved. Compared with DSP and ASIC, it obtain a good tradeoff between flexibility and performance.
