Abstract—In this paper, we comprehensively propose and 
investigate several  available decoding schemes for quasi-cyclic 
low-density parity-check (QC-LDPC) codes in physical network 
coding with bitwise XOR operation at the relay. 
Keywords- LDPC codes,  decoding,  XOR, Physical Network 
Coding
I.
INTRODUCTION 
The  basic idea of network coding [1] is that relay nodes in 
a network are allowed not only to route but to perform 
operations on incoming data to improve bandwidth efficiency. 
Generally, the operation at relay is XOR operation. Recently, it 
is introduced to the physical layer of wireless networks [2], 
and channel coding gennerally becomes a valuable problem for 
network coding. Specially, considering ther error control based 
on modulation level at relay [2],  Q-ary LDPC code is worth 
studying  further.  In this paper, we mainly investigate the error 
control performance of QC-LDPC codes based on bit level at 
sink,  and similar work has been proposed in recent study [3]-
[4]. For example, [4] first investigates the LDPC decoding 
schemes at sink of  the typical butterfly muticast network with 
XOR operation at relay shown in Fig.1. Furthermore, it 
proposed a new joint iterative message-passing decoding 
scheme for a good error control performance. However, there 
are still other available decoding methods  worth studying.  
This paper will further investigate the error control 
performance of several available LDPC decoding schemes at 
sinks of the typical butterfly multicast network with network 
coding.

Simulation Results 
In this section, we present the simulation results for different 
decoding schemes.  As shown in Fig.4 and Fig.5, two different 
rates approximately regular quasi-cyclic (QC) LDPC codes are 
applied, which are both constructed using modified 
progressive edge growth (PEG）algorithm [6].  The modulator 
uses the binary phase shift keying (BPSK) constellation. The 
message-passing decoding algorithm uses sub-optimal sum 
product algorithm (SPA). For all decoding schemes in Fig.2, 
maximum iteration number is set to 24 for decoder a, b and c.
And  equivalently , for the scheme_5  [3] in Fig.3, the fix inner 
iteration number is set to 3 for decoder A and B, while the 
maximum external iteration number is set to 4 between 
decoder A and B, which means that the maximum iteration 
number is also 24 for codewords a and b. Furthermore, in 
shceme_4, we preset the jointly decoding stop when a is 
decoded successfully or the iteration number reaches upbound, 
because experiential results show that the performance of a
will suffer more error corruption from b if decoding doesn’t 
stop until b is decoded successfully.

code rate decreasing. Therefore, it indicate that the reliability

III.
CONCLUSION
This paper prospose and investigates several available 
decoding schemes for quasi-cyclic low-density parity-check 
(QC-LDPC) codes in network coding with bitwise XOR
operation at the relay. However,  the scheme_1 and sheme_4 
assume that XOR codeword c is available to be decoded, on 
the condition that the codewords a and b belong to the same 
QC-LDPC ensemble. Otherwise, if a and b belong to two 
different LDPC ensembles,  the performance evolution of c
will be a challenging and interestiong work [3].  Additioanlly, 
the error control of Q-ary LDPC codes based on modulation 
level at  relay is also worth studying further.
