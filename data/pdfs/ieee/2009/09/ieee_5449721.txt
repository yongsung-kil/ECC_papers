Abstract—Layered decoding of Low Density Parity Check (LDPC) codes has been shown to give faster convergence and use less storage than the conventional Two Phase Message Passing (TPMP) algorithm in decoding such codes. To achieve higher throughput, multiple rows of the parity check matrix may be decoded in parallel, as in the Parallel-Turbo Decoding Message Passing (P-TDMP) algorithm. However, contention for globally shared memory permits only a subset of rows to be decoded in parallel thus limiting overall throughput. This paper uses a near optimal scheduling algorithm, based on graph colouring, to overcome memory contention, allowing processing of multiple rows without code modification, resulting in significant throughput gains over the standard P-TDMP algorithm. For the LDPC codes studied, throughput gains compared to the P-TDMP algorithm, of between ten and fifteen are shown to be possible.
I. INTRODUCTION
The need for high throughput channel decoding is a reality in many application areas ranging from hard disk and solid state data storage to personal, local and cellular wireless networks. A simple encoding/decoding scheme, naturally, presents a better candidate for achieving high throughput. Low density parity check (LDPC) codes, as described by Gallager [1], offer a comparatively simple means of decoding whilst achieving competitive error correcting performance. Hence, their inclusion in a number of recent wireless standards such as IEEE 802.16e (WiMax), IEEE 802.11n and DVB-S2.
Gallager [1] described an iterative two phase decoding algorithm, subsequently generalised by Pearl [2] and referred to as belief propagation. By combining the received sequence according to the parity check code, the likelihood (belief) of a checksum being satisfied is formed, this being the first phase. The second phase sees updating of symbol estimates based on the likelihood of checksum values (propagation), subject to code constraints. Multiple iterations of these two steps lead to an estimate of the received sequence that is optimal, in a maximum likelihood sense, for codes which do not contain cycles.
From an implementation perspective the standard two phase belief propagation algorithm is attractive as it is highly parallelisable whilst avoiding the high computational cost usually associated with trellis based algorithms. However, the process

of updating checksum values and symbol estimates, commonly referred to as check nodes and variable nodes, requires storage of intermediate values and propagation between the two sets of nodes. For long codes, a significant proportion of a typical hardware implementation is taken up by the routing network mirroring connectivity between the two sets of nodes [3]. Also, for long codes, storage of intermediate values results in large memory requirements. To address these problems alternative decoding methods, based on belief propagation, have been proposed [4], [5].
In [6], an alternative to the two phase decoding algorithm is given called Turbo Decoding Message Passing (TDMP). In contrast to the standard belief propagation approach of updating two types of message, the TDMP algorithm uses only a single message type, thereby reducing the storage requirement significantly. The TDMP algorithm also converges faster than the standard belief propagation algorithm due to using the most recently computed values in updating the next set of values.
In this paper we investigate the behaviour of the TDMP algorithm with a view to improving overall throughput. We describe a near optimal and computationally simple scheduling algorithm derived from graph theory. This scheduling algorithm, applied off-line, allows grater parallelism in the decoder architecture by overcoming memory contention resulting in a significant increase in throughput.
This paper is organised as follows: section II briefly explains the TDMP algorithm, highlighting memory contention as a limiting factor on decoder throughput. Section III introduces graph theory and describes a proposed edge colouring algorithm as means of scheduling contention free memory access. Section IV compares performance, in terms of number of colours used, of the proposed edge colouring algorithm with an established algorithm and gives the subsequent improvement in throughput. Finally, section V summarises our findings.

As seen from (2) the TDMP algorithm uses the most recently computed extrinsic value in updating bit estimates. Thus, in contrast to the standard belief propagation algorithm, updates are propagated as soon as they are made available resulting in faster convergence and fewer storage requirements.
To permit decoding of multiple rows simultaneously, as seen from (1), posterior memory locations accessed at any time instant must be unique to avoid memory contention. Specifically, multiple read operations from posterior memory are not subject to contention, write operations however need to be carefully scheduled to avoid contention. Extrinsic messages are stored local to each row processor and therefore do not suffer contention. One method for ensuring posterior memory locations being accessed at a given time instant are unique is by careful design of the parity check matrix. In [8] Architecture-Aware LDPC (AA-LDPC) codes are introduced as a means of avoiding memory contention by partitioning the parity check matrix into sub-matrices. Each sub-matrix has the property that every column has at most a single ’one’ entry and every row has at most a single ’one’ entry. A. Graph Colouring
In the graph colouring problem the aim is to find the smallest number (chromatic number) of colours needed to colour each vertex (node) in the graph such that no two vertices connected by an edge have the same colour, this is known as vertex colouring. Alternatively, a graph may be coloured with the minimum number of colours such that all edges incident on a vertex are of a different colour, this being termed edge colouring. For general graphs, vertex colouring is an NPcomplete problem whereas edge colouring can be achieved in polynomial time.
To ensure that each posterior memory location is accessed by only one row processor at any time instant, each column of the parity check matrix must have unique time slots in which row processors may access it. Since a row processor cannot access more than one posterior memory location at any time instant, due to the sequential nature of row processing, each row must have unique time slots in which the posterior memory locations are accessed. Fig. 3 shows contention of the posterior memory locations for the parity check matrix shown in Fig. 2 if nine parallel decoders were to be used in the decoding process. To apply graph theory to the memory contention problem and take the easier edge colouring approach a processor can be termed a vertex and a time slot an edge.
Thus, the posterior memory access problem may be seen as an edge colouring problem for a bipartite graph. For a bipartite graph G with a maximum vertex degree d, Konig’s elegant theorem [11] tells us the edge chromatic number χ′ (G) = d. Hence, for an LDPC code of maximum degree d it is possible to perform one complete iteration of the TDMP algorithm in d time slots. A simple and near optimal edge colouring algorithm is given below which can be applied, in reasonable time, to large bipartite graphs.

V. CONCLUSIONS
The turbo decoding message passing (TDMP) algorithm was discussed, highlighting the bottleneck prohibiting parallel decoding, namely contention for the globally shared posterior memory. Although the parallel TDMP algorithm allows for decoding of S rows in parallel, based on designing a parity check matrix from permuted identity matrices, it is limited to using S decoders. To overcome this limitation, a scheduling algorithm based on edge colouring of a bipartite graph was proposed and shown to outperform an existing edge colouring algorithm; using between 25% to 50% fewer colours whilst using significantly less computational time than an optimum edge colouring algorithm. Throughput analysis for six LDPC codes based on a scheduling derived from the proposed algorithm showed a gain of between approximately seven and fifteen times over the parallel TDMP algorithm.
