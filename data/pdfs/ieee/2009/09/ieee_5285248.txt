Seed Graph Expansion for Construction of Structured LDPC Codes

Wojciech Sulek #1
# Institute of Electronics, Faculty of Automatic Control, Electronics and Computer Science, Silesian University of Technology Akademicka 16, 44-100 Gliwice, Poland

Abstract—A lot of works concerning LDPC codes construction has been published so far. However, it is well known that efficient partially parallel hardware decoder architectures are allowed only for LDPC codes with blockwise partitioned structure of the parity check matrix, called structured LDPC codes. Two main steps in the structured LDPC code parity check matrix construction are the seed matrix (seed graph) construction and expansion of the seed matrix (seed graph). In this paper we present a flexible method for seed matrix expansion by computer search technique for the optimum shift values of the circulant submatrices. The proposed algorithm aims at reducing existence in the code graph of small cycles with low external connectivity that constitute structures known as Stopping Sets and Trapping Sets that are harmful to the code performance, especially in the high SNR region. The algorithm can be used for regular or irregular code construction of any block length and code rate.
I. INTRODUCTION
Low-density parity-check (LDPC), after their “re-discovery” in late 90’s [1], have attracted great research attention due to their excellent error-correcting performance and highly parallel iterative decoding scheme with complexity linearly dependent on block length of the code. An LDPC code is defined as the null space of an M × N sparse parity check matrix H. It can also be represented by a bipartite graph with M check nodes (representing parity-check equations) and N bit nodes (representing bits of the codeword).
Structured LDPC codes allow applying flexible and efficient partially parallel decoder architectures [2], [3], [4]. The main requirement for this efficient decoder implementation is the blockwise partitioned structure of the parity check matrix. The class of structured codes is also known as Architecture-Aware LDPC (AA-LDPC) or VLSI-Oriented codes [2], [3].
A lot of works concerning parity check matrix construction for LDPC codes with “good” graph properties, e.g. [5], [6], [7] has been published. On the other hand, the structured LDPC codes has been recently adopted for some industrial standards, e.g. [8]. However, the literature concerning construction methods for “good” structured LDPC codes is still quite poor. In this paper we present a flexible algorithm based on computer search technique that statistically reduces the existence of some harmful subgraphs in the code graph of structured LDPC code. The algorithm can be used for regular or irregular code construction of any block length and code rate.

It is caused by short cycles in the Tanner graph as well as some more complex subgraphs that are shortly discussed below.
A cycle is usually defined as a sequence of adjacent edges which starts and ends at the same vertex, and satisfies the condition that no vertex (except the initial and the final vertex) appears more than once. The length of the shortest cycle in a graph is called the girth of the graph. In this paper we also use a term of closed walk, which is a sequence of adjacent edges without restriction of their single appearance [10]. Cycles and closed walks are denoted as e1 ∼ e2 ∼ · · · ∼ ek ∼, where e1, e2, . . . , ek are consecutive edges in the cycle.
A Stopping Set SSa is a subset of a bit nodes, for which the subgraph of G induced by SSa and its neighbors contains no check nodes of degree one [11].
A Trapping Set T Sa,b is a subset of a bit nodes, for which the subgraph of G induced by T Sa,b and its neighbors contains b odd-degree check nodes [11], see Fig. 2.

As was proved [12], [11], the performance of an iteratively decoded LDPC code is limited by the existence of small stopping sets and trapping sets in the graph of the code. Since there are very few check nodes capable of correcting errors within the trapping set, this incorrect information remains “trapped” until the termination of the decoding process.
A special case is a Linearly Dependent Set LDa, which is a subset of a bit nodes whose associated parity check matrix columns are linearly dependent, but any subset of these columns is linearly independent [13]. Since columns are linearly dependent if and only if all check nodes in the induced subgraph are even-degree, every LDa is also a T Sa,0. A code with minimum distance dmin has at least one LDdmin , but no LDa where a < dmin. Thus the Tanner graph of the code with large minimum distance must not contain T Sa,0 with small a.
Finally, we define the parameter that indicates “harmfulness” of the cycles in the Tanner graph. An extrinsic check node of a bit node set is a check node that is singly connected to this set. The Extrinsic Message Degree (EMD) of a bit node set is the number of extrinsic check nodes of this bit node set [13].
A set of bit nodes with large EMD will require additional concatenation of nodes to become a stopping set. In a bipartite

graph free of degree-1 bit nodes, every stopping set contains cycles [13]. Similarly trapping sets are composed of interconnected cycles (Fig. 2). Hence, if short cycles are eliminated from the graph, also small stopping sets and trapping sets are eliminated. However, not only the length, but also the connectivity of cycles plays important roles, because cycles having small EMD are more prone to induce small trapping sets [13]. Minimization of short cycles with low EMD is the basis of many published code construction methods, e.g. [13], [6], [14]. In this paper the idea of low-EMD short cycles elimination is adopted for structured LDPC codes construction.
IV. STRUCTURED LDPC CODES CONSTRUCTION
The parity check matrix HDP ×LP that is suitable for efficient decoder architecture, as discussed earlier, is an D ×L matrix of P × P submatrices. In this paper we consider circulant submatrices, i.e. all non-zero submatrices are rotations of columns of the identity matrix.
The D × L matrix W with zeros indicating the all-zero submatrices and ones indicating the circulant submatrices we call the seed matrix and respective graph we call the seed graph. Thus, two main steps in the structured LDPC code parity check matrix construction are the seed matrix construction and expansion of the seed matrix, which means replacing its entries with appropriate P × P submatrices.
The seed matrix can be constructed in any known method for constructing LDPC parity check matrices without implementation constraints, e.g. Improved PEG (Progressive Edge Growth) [6]. Regardless of the method used for construction, the relatively small seed graph contains a lot of short cycles, which can be eliminated in the graph expansion. Thus, the crucial step for achieving “good” graph of the structured code is the expansion of the seed graph.
We define the seed graph expansion as follows. Let G = (Vc ∪ Vb, E) be a seed graph, where Vc = {c1, c2, . . . , cD}, Vb = {b1, b2, . . . , bL}, E = {e1, e2, . . . , e|E|}. Let P = {1, 2, . . . , p, . . . , P } be a finite set. The graph after expansion, called covering graph has vertex sets Vc × P and Vb × P. For each e ∈ E, we fix a translation: θe(p) ≡ p + se mod P . The edge set is E × P, where an edge (ei, p) = (bl, cd, p) ∈ E × P is incident with vertices (bl, p) and (cd, θei (p)). The expansion is thus characterized by the set of cyclical shift values S = {se1 , se2 , . . . se|E| }, 0 ≤ se < P . The covering graph we denote as G(P,Θ) = ((Vb × P) ∪ (Vc × P), E × P).
The crucial theorem characterizing the relationship between cycles in the seed graph and cycles in the covering graph is as follows.
Theorem 1: The covering graph G(P,Θ) contains a cycle of length k: (e1, p1) ∼ (e2, p2) ∼ · · · ∼ (ek, pk) ∼ if and only if there exist a closed walk e1 ∼ e2 ∼ · · · ∼ ek ∼ in the seed graph G and the following condition is satisfied for the cyclical shift values associated with edges e1, e2, . . . , ek:
se1 − se2 + se3 − se4 + . . . + sek−1 − sek ≡ 0 mod P (2)

A. Graph expansion algorithm
Based on considerations presented in the previous sections, an obvious direction to generate a graph G(P,Θ) for “good” structured LDPC code, is to select cyclical shift values S in a way to minimize existence of short cycles with low EMD. In the proposed method for seed matrix (graph) expansion, stated as Algorithm 1, at the beginning all closed walks in the seed graph of length lower than some predetermined Kmax are placed on the priority list LCW. Walks of length 4 begin the list, because of their well known destructive influence on the iterative decoding algorithm, then walks with EMD=0, because their bit nodes constitute LDk. Finally all remaining walks in order of value of the sum (k/2 + 1.1 EMD) are placed on the list.

The expression (k/2 + 1.1 EMD) is proposed in order to respect the length of the cycle (precisely: the number of bit nodes k/2 in the walk) as well as connectivity represented by EMD parameter. To the best knowledge of the author, general relationship between harmfulness of cycles and their length and connectivity does not exist. However, the proposed simple heuristic expression works well for cycles of length k > 4.
Consecutively for every closed walk on the created LCW list, the cyclical shift values S are adjusted by a computer search technique that has been developed. The proposed method is simply based on multiple trials of incrementing the values sei associated with edges ei belonging to the closed walk. Details of the search method are presented in Fig. 4, where the S values adjustment in order to fulfil condition (3) for a single cycle is presented. This condition in a vectors multiplication form is: Cyc · S ≡ 0 mod P , where Cyc is a row vector with 0, 1 and -1 components, according to the condition (3) formulated for considered closed walk.
Furthermore, for each ei, we set a constraint matrix CONei , where consecutive rows define conditions for all closed walks considered so far. Thus, the crucial condition at each stage is whether all elements of CONei · S are not congruent to 0 mod P . If “yes”, then the values of S fulfil all constraints formulated so far for ei. The S is a temporal variable (its elements sei are denoted by s (ei) for legibility) and Sbest is used to keep the best result, i.e. with the lowest sum of elements, S 1. We use the low sum criterion in order to achieve a slow increase in the values of sei , to leave more freedom for incrementing these values in the next steps. Index(Cyc) is a vector with indexes of non-zero elements of Cyc and [Index(Cyc)]n is n-th element of Index(Cyc).
In the case of missing the condition CONei · S ≡ 0 mod P , a trial is made to adjust s (ej) values for some ej that do not belong to the considered closed walk, but conditions for these walks are violated (it is denoted by “Stage II” in Fig. 4). We do not provide details of the “Stage II” here – it is quite similar to the algorithm presented in Fig. 4 and it allows to achieve a slightly better results (i.e. lower Sbest 1).
After some number of stages, as the values of S are increasing, conditions for some cycles cannot be satisfied without violating conditions for previously considered ones. In such a case, the result of the algorithm stage (Fig. 4) is “Cycle not removed” and the algorithm continues for the next closed walk from the list. In order to reduce the time consumption, in a practical realization of the presented algorithm, we set the number of consecutive “Cycle not removed” events (i.e. 1000), after which the algorithm is halted.

VI. CONCLUSIONS
The flexible algorithm for seed matrix expansion for structured LDPC codes construction has been presented in the paper. The algorithm aims to reduce the number of low-length cycles with low external connectivity (EMD) in the Tanner graph and – in consequence – statistically reduce the number of small Stopping Sets and Trapping Sets, which are harmful to code performance, especially in the low FER region.

A number of structured LDPC codes has been constructed
making use of the proposed algorithm. Performances of the
constructed codes in most cases are similar or even better
than performances of the respective “good” LDPC codes
from MacKays database [16], however the main drawback of
MacKay codes is that they are not VLSI oriented.
