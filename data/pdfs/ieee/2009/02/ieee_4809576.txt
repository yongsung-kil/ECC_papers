Low-Complexity Entirely-Overlapped Parallel Decoder Architecture for Quasi-Cyclic LDPC Codes

Weizhi Lu
School of Information Science and Engineering Shandong University Jinan, China weizhilu@yahoo.cn

Abstract-In this paper, we propose a low-complexity entirelyoverlapped partially-parallel decoder architecture for quasicyclic low-density parity-check (QC-LDPC) codes. For a (c, t)regular QC-LDPC decoder, this decoder can achieve approximate 100°t'c» hardware utilization efficiency (HUE) and entirely overlapped decoding by using only one check node process unit group (CNUG) to process c groups of check node messages sequentially. Compared with traditional partiallyparallel decoder, the quantities of check node process unit (CNU), barrel shifters and counters can be decreased prominently. Moreover, it is flexible in code rate and code length.
Keywords- QC-LDPC codes, HUE, entirely-overlapped, partially-parallel, decoder architecture
I. INTRODUCTION
Recently, quasi-cyclic low density parity check (QC-LDPC) codes have been considerably studied due to their hardwarefriendly structures and comparable performances to random LDPC codes [1]-[3]. Though the encoder ofa QC-LDPC code can be easily implemented using shift registers [4], the decoder design is still a challenge for its huge hardware demand.
Comparing with the fully-parallel decoder architecture [5], the partially-parallel architectures are more practical for its good tradeoff between hardware complexity and throughput. [6] presented a memory-basic partially-parallel decoding architecture with simple memory address generation, access and routing. But its hardware utilization efficiency (HUE) is only 50%.
In order to increase throughput and HUE, an overlapped message passing decoding algorithm for QC-LDPC codes is proposed in [7]. Because of memory access conflicts, the decoder architecture based on this algorithm can only realized partially overlapped decoding, which leads to low flexibility and high routing complexity [8] [9].
In this paper, we proposed a low-complexity entirelyoverlapped partially-parallel decoder architecture with approximate 100% HUE and high flexibility. Compared with traditional partially-parallel decoder architecture [6], we use only one check node process unit groups (CNUG), which can definitely realize entirely overlapped decoding without memory access conflicts. As a result, the number of barrel shifter and counter for routing can be reduced proportionally. Because there is only one relevant check node updated in one

clock cycle for a variable node, the computation complexity of variable node process unit (VNU) can be reduced proportionally.
This paper is organized as follows. In section II, first, we review QC-LDPC codes construction based on the progressive edge growth (PEG) algorithm and propose a new construction scheduling. In section III we review the general partiallyparallel decoder architecture with a simple switch network. In section IV, we present our LDPC decoder architecture in detail. Finally, section V concludes the paper.

B. Construction ofParity-Check Matrix of QC-LDPC code Using PEG Algorithm
The progressive edge growth (PEG) algorithm is flexible in finite-length QC-LDPC codes construction [3]. By using the PEG algorithm, we only need to optimize edges of the first variable node in a variable nodes group. Then the edges of other variable nodes in the same group can be determined by the circulant constraint automatically. All edges will be determined group by group. But latter determined variable nodes gets more short cycles than prior determined nodes, especially when the code is long and its code rate is high. In order to avoid performance imbalance among all the variable nodes, we propose to take all groups as an integer, first choose one edge for all nodes, and then tum to the next edge.

III. PARTIALLY-PARALLEL LDPC DECODER
A. Partially-Parallel Decoder Architecture
For the fully-parallel architecture, each node is assigned a special hardware process unit, which makes it possible to maximum the throughput. But the partially-parallel structure is more practical to balance the hardware costs and throughputs [6].
According to Hqc defined by (1), all variable nodes are divided into t groups and all check nodes are divided into C groups. Each group has b adjacent nodes. And then, there are t variable node process unit groups (VNUG) and C check node process unit groups (CNUG) in partially-parallel decoder, which is shown in Fig. 1. p process units in a process unit group are used to process b node messages in a time-division
multiplexed manner, where b = p x v and p is defined as
parallelism. A group of nodes are updated once in v clock cycles.
If hij is greater than zero, there is a dual-port memory block between the j-th VNUG (VNUGj) and the i-th CNUG (CNUGi), which stores the extrinsic messages of LDPC decoding. The structure of memory block connected VNUGj with CNUGi is displayed in Fig.2. In order to ensure messages storage and exchange accurately between VNU and CNU, each memory block are equipped with an address counter and a barrel shifter. The address counter is used to calculate the read/write memory addresses. And the barrel shifter is used to rotate the check node messages.

DECODER ARCHITECTURE
Based on the above partially-parallel decoder architecture, we can reduce the number of CNUGs to one and present a low-complexity entirely-overlapped partially-parallel QCLDPC decoder architecture, which both realizes entirely overlapped decoding and reduces the number of counters and

barrel shifters proportionally with accessible memory space supplemented.
As shown in Fig.3, the decoder comprises one CNUG and t VNUGs, so c groups of b check nodes have to be processed group-by-group. The extrinsic messages between the i-th check node group and the j-th VNUG are stored in MAij, 0::;i<c-1, 0~<t-1, where b messages are stored in the way mentioned in section II, that is, in the r-th address space of MAij, p messages {mn m r+v, ... ,mr+(p-J)v} are orderly stored, 0::;r<v-1.
In addition, to realize the overlapped decoding, all checkto-variable messages updated in the present iteration are available in the whole process of next iteration. However, MAij can only store the extrinsic messages in the current iteration, so specialized memory block MBij is appended to store the checkto-variable messages that the i-th check node group update for thej-th VNUG.
For VNUGj , c MAij and MBij can be taken as a group Aj and Bj , respectively. Moreover, Aj is equipped with a barrel shifter and counter for CNUG, and an extra counter is presented for VNUGj to simultaneously control its access to Aj andBj .
The overlapped decoding scheduling in the first two clock cycles can be summarized as follows.
In the first clock cylcle, VNUGj orderly read (c-1) groups of p check-to-variable messages from the same address value of (c-1) memory clocks in Bj , and then writes p variable-tocheck messages to the same address value of the memory block for the current check nodes group in Aj .
In the second clock cycle, VNUGj hold on updating messages for next group of p check nodes, and the CNUG orderly processes t groups of p variable-to-check messages which are separately updated by t VNUGs in last clock cycle.
Meanwhile, the addresses of MAij and MBij VNUGj accesses are both initialized by mod (hij, v), where hij denotes the cyclic-shift value of the submatrix that constraints the i-th check node group and the j-th variable node group.

V. CONCLUSION
In this paper, we have proposed a low-complexity entirelyoverlapped partially-parallel QC-LDPC decoder architecture for general QC-LDPC codes, which not only proportionally reduces the hardware complexity but also gets flexible in code rate and code length to a certain extent. Otherwise, while pursuing lower hardware complexity, the storage space is doubled, but it is accessible, especially when efficient decoding algorithms such as the modified min-sum decoding algorithm [10] can be applied.
