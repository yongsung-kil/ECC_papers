An Improved Decoding Algorithm for Finite-Geometry LDPC Codes
Telex Magloire Nkouatchah Ngatched, Member, IEEE, Fambirai Takawira, Member, IEEE, and Martin Bossert, Senior Member, IEEE

Abstract—In this letter, an improved bit-flipping decoding algorithm for high-rate finite-geometry low-density parity-check (FG-LDPC) codes is proposed. Both improvement in performance and reduction in decoding delay are observed by flipping multiple bits in each iteration. Our studies show that the proposed algorithm achieves an appealing tradeoff between performance and complexity for FG-LDPC codes.
Index Terms—Belief propagation algorithm, bit-flipping (BF) decoding algorithms, low-density parity-check (LDPC) codes.
I. INTRODUCTION

In this letter, a modification of IWBF is proposed. The modified algorithm, which we called multiple bit-flipping (MBF), updates multiple bits in each iteration. The modification is based on the observation that for LDPC matrices, the syndrome weight increases with the number of errors on average. Therefore, the idea is to use the syndrome weight in each iteration to estimate the number of bits to be flipped. This modification leads to improvement in performance in terms of both the bit-error rate (BER) and frame-error rate (FER), and drastically speeds up the decoding process.

L OW-DENSITY parity-check (LDPC) codes originally introduced by Gallager [1] and brought into prominence
by MacKay and Neal [2], have been attracting a great deal of research interest. LDPC codes can be decoded with various
decoding methods, ranging from low to high complexity
and from reasonably good to very good performance. These decoding methods include hard-decision ( [1, p. 39], [3, pp.
184-222], [4]), soft-decision ( [1, pp. 40-48], [5]–[9]), and hybrid decoding ( [10]–[16]) schemes.
A binary LDPC code is completely described by its sparse binary parity-check matrix H = [Hmn]. For an (N, K) LDPC code, H has N columns and M ≥ N − K rows since it may contain some redundant parity-check sums. For a regular LDPC code, H has a constant column weight wc and a constant row weight wr. The set of bits that participate in check m is denoted as N (m) {n : Hmn = 1}. Similarly, the set of checks in which bit n participates is denoted as M(n) {m : Hmn = 1}.
The algorithm in [15], designed for high-rate finite geometry (FG) LDPC codes [11], and which we call improved
weighted bit flipping (IWBF), was shown to offer one of the most appealing performance versus cost tradeoffs. FG-LDPC
codes are algebraically generated and allow effective, low-
cost, hardware encoder implementations.

III. MODIFIED IMPROVED WEIGTHED BIT-FLIPPING ALGORITHM
The above algorithm can be modified by flipping multiple bits in each iteration. The idea of the modification is based on the observation that for low-density parity-check matrices, the syndrome weight increases with the number of errors on average. Therefore, in each iteration, the syndrome weight gives an approximation of the number of bits to be flipped. In essence, we calculate p = wH (s(k))/wc , where wH (a) denotes the Hamming weight of a, wc is the column weight of the code, and x denotes the greatest integer less than or equal to x. We then find the set D(k) = j1(k), j2(k), . . . , jp(k) of the p smallest values of φn(k), 0 ≤ n < N , and flip the bits in this set.
Like the standard IWBF algorithm, the modified algorithm is a search algorithm. In some cases, the changing of bits can enter a loop which means that bits are changed and after some steps are changed back again ad infinitum if no stopping criterion is applied. In essence, if in the kth iteration the tentative binary hard-decision vector z(k) coincides with a previously considered vector z(k0), k0 < k, then D(k+1) = D(k0+1) and z(k+1) = z(k0+1). Since no valid codeword vector was found in z(k0), z(k0+1), . . . , z(k−1) , it would not be found if the decoding continues. When the loop detection mechanism is used, the effect on the different codes is not the same for both algorithms. Applying the loop detection mechanism to the modified algorithm provides an additional 0.1 dB gain in terms of both the BER and FER for the (1023, 781) code and a very marginal gain (about 0.025 dB) for the (4095, 3367) code. The gain achieved is marginal (about 0.05 dB) for the (255, 175) code, whereas there is absolutely no gain for the (16383, 14197) code. On the other hand, applying the loop detection mechanism to the standard algorithm provides a gain of 0.25 dB for the (255, 175) and the (1023, 781) codes, 0.175 dB for the (4095, 3367) code, and no gain for the (16383, 14197) code. Thus the loop detection mechanism is more effective when applied to the standard algorithm. This corroborates the fact that a good number of decoding failures of IWBF algorithm are attributed to infinite-loop traps. These results also suggest that, as the code length increases, the number of infinite-loop traps decreases. When comparing both algorithms with the loop detection mechanism, the improvement of MBF over IWBF is now 0.05 dB, 0.1 dB, 0.1 dB, and 0.25 dB for the (255, 175) code, the (1023, 781) code, the (4096, 3367) code, and

MBF (without loop detection) requires about 1/2 iterations and 2/5 iterations respectively, compared to IWBF (without loop detection) and IWBF (with loop detection). For the (1023, 781) code, MBF (without loop detection) requires about 1/5 iterations and 1/6 iterations respectively, compared to IWBF (without loop detection) and IWBF (with loop detection). Clearly, the reduction in decoding delay increases with the code length. Thus, though the cost per iteration of MBF is higher than that of IWBF, the average number of iterations is drastically reduced. As a matter of fact, without the loop detection mechanism, the average running time of the modified algorithm was found to be about 1.25, 2, 4 and 8 times less than that of the standard one for the (255, 175) code, the (1023, 781) code, the (4095, 3367) code, and the (16383, 14197) code respectively. It can also be observed that the improvement in performance obtained by applying the loop detection mechanism to the standard algorithm comes at the cost of not only an increased complexity, but an increased decoding delay as well. On the other hand, applying the loop detection mechanism to the modified algorithm does not significantly change the average number of iterations.
