Eliminating Small Stopping Sets in Irregular Low-Density Parity-Check Codes
Xiaopeng Jiao, Jianjun Mu, Jing Song, and Lihua Zhou

Abstract—We present a novel scheme to eliminate small stopping sets in irregular low-density parity-check (LDPC) codes. By adding several new check nodes and having their edges connected to small stopping sets in the original code, the improved code decreases the number of small stopping sets efficiently. Simulation results show that the proposed scheme decreases the frame error rate (FER) significantly in the error floor region, whereas having almost the same rate as the original code.
Index Terms—Error floor, low-density parity-check (LDPC) codes, stopping sets.

I. INTRODUCTION

Fig. 1. Eliminating small stopping sets by adding a new check node ca.

T HE error floor phenomenon of low-density parity-check (LDPC) codes is an important problem which should be considered carefully in designing practical codes with finite length. The performance of LDPC codes under iterative decoding in the error floor region is dominated by small stopping sets [1] on the binary erasure channel (BEC), and near-codewords [2] or trapping sets [3] on the additive white Gaussian noise (AWGN) channel. For the BEC, a straightforward way to improve the error floor is to construct LDPC codes without small stopping sets. It seems to be a difficult problem. However, there are a lot of fruitful works to improve the error floor in a sub-optimal way, such as progressive-edgegrowth (PEG) algorithm [4] (using girth metric), approximate cycle extrinsic (ACE) message degree algorithm [5] (using ACE metric), and partial elimination of stopping sets [6]. These algorithms always perform well but they may allow the existence of small stopping sets. In [7], Wang proposed a code annealing algorithm to prevent small stopping sets on any existing codes by swapping edges in the Tanner graph.
Recently, Schwartz and Vardy [8] introduced the stopping redundancy of a linear code, and showed that the performance of iterative decoders can be improved significantly by adding some redundant check nodes. Finding the exact value of the stopping redundancy is a difficult problem and some lower and upper bounds are derived in [8][9]. In this letter, we propose a method to eliminate small stopping sets in irregular LDPC codes by adding several new check nodes. The proposed method can improve both the stopping distance and the minimum distance at the expense of altering the original

III. STOPPING SET ELIMINATING ALGORITHM
A. The Basic Idea
To eliminate small stopping sets, we can add several new check nodes in the original code and have their edges connected to small stopping sets. It should be guaranteed that: (1) New small stopping sets should not be introduced; (2) Small stopping sets in the original code can be eliminated. Given a Tanner graph G, ca is a new check node added to G, and it has several edges connected to the variable-node set (see Fig. 1(a)). When choosing stopping sets for S, we should give the first consideration to small size stopping sets since they are more harmful than others. There may be many small stopping sets in G, so the number of small stopping sets in S must be determined carefully to ensure that the number of newly-added check nodes are small. Discussion: (1) Every codeword is a stopping set but not all of the stopping sets correspond to codewords. Therefore, the SSE algorithm can increase both the stopping distance and the minimum distance of the original code. However, we can not guarantee that the stopping distance of the improved code can be equal to its minimum distance. Indeed, with our observations, the stopping distance of the improved code is very close to or even equal to its minimum distance. (2) The proposed algorithm may introduce four-cycles. Though it is not deleterious for the BEC, it may degrade the performance of iterative decoders on other channels since small trapping sets may be introduced by four-cycles. For techniques to attack trapping-set errors, one is referred to [11] and the references therein1. (3) The algorithm [10] we used is only an estimate of the exact number of small stopping sets. Thus, we are not guaranteed to remove all stopping sets up to a certain size. This is, however, fully determined by the stopping set enumeration algorithm. Note that there is an algorithm can find all small stopping sets [12], but the algorithm [10] is simple and fast since it only uses belief propagation and an erasure decoding algorithm. Moreover, from [13] it can be seen that the algorithm [10] can find almost all small stopping sets if parameters in the algorithm are selected appropriately.

B. Construction of LDPC Codes with Same Rate
The proposed algorithm causes rate loss. In this subsection, a method for constructing LDPC codes with the same rate is presented. Firstly, we construct C4 with a block length of 1008 and rate of 0.503 by PEG algorithm (the degree distribution is λ(x)). Then three new check nodes are added to C4 by the SSE algorithm and C3 is obtained. C3 has the same block length and code rate as C1. The number of small stopping sets of C3 is given in Table I. Fig. 2 shows the FER performance of C3 under iterative decoding over the BEC. It can be seen that the proposed algorithm gives about two orders of magnitude gain in FER in the error floor region when compared with C1. The asymptotic curves in Fig. 2 are plotted according to equation (2). For iterative decoding curves, 50 block errors are collected, except the last two data points of C2 and C3, which contain at least 20 and 10 block errors respectively. We use a method [16] for approximating the minimum distance of C1, C2 and C3 and find they are 13, 20 and 23 respectively. Note that the stopping distance of C2 we found is greater than its minimum distance. This is mainly because the algorithm [10] we used may lose some small stopping sets.
