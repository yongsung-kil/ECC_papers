Abstractв”ҖThis paper concerns a new and powerful Low-Density Parity-Check (LDPC) stochastic decoding algorithm, called Controlled Start-up Stochastic (CSS) decoding, to implement fully parallel LDPC decoders. The proposed algorithm uses a clever starting point computing method to improve the convergence as well as to reduce the latency of the decoding process, without a significant hardware complexity cost. Using this stochastic method, in low signal-to-noise ratio (SNR) region, a short LDPC code can outperforms the fixed point state-of-theart (2048, 1723) LDPC code. To validate the performance of the proposed algorithm, a (48, 24) LDPC code are implemented on Xilinx Virtex-6 VLX240T field programmable gate array device, with and without the CSS method. The new algorithm provides a decoding speed seven times faster and a Bit Error Rate (BER) hundred times better at Eb/N0=4.5 dB. In addition, the results provide a BER better than the 6 bits offset min-sum algorithm (OMSA) and better than the 6 bits sum-product algorithm (SPA) for the (2048, 1723) LDPC code, below Eb/N0=3.9 dB and Eb/N0=4.3 dB respectively. The obtained results show that, the proposed algorithm is the first stochastic algorithm which can outperform the OMSA and SPA algorithms, with less hardware complexity.

I. INTRODUCTION
The Low-Density Parity-Check (LDPC) codes, firstly introduced by Gallagher in 1962 [1], are a valuable linear error-correcting codes which can offer a close Shannon Capacity decoding performance [2]. These codes have been adopted by several digital communication standards such as DVB-S2 [3], the IEEE 802.3an (10GBASE-T) [4], the IEEE 802.3ba [5], the IEEE 802.16 (WiMAX) [6], the IEEE 802.11 (WiFi) [7] and the ITUG.975.1 [8]. LDPC iterative decoding using stochastic approach was initially introduced by Gaudet and Rapley [9]. The recent stochastic decoding algorithm proved their compatibility for fully parallel decoding approach [10]-[15]. The edge memories (EMs) based decoding method [10] was introduced to avoid the Latch-up (locking) problem [16]. EMs can be implemented as shift register with a single selectable bit [11]. Fig. 3 shows the main structure of the controlled start-up stochastic variable node. In order to implement the one bit coded probability channel, 2-to-1 multiplexer is introduced between the variable node processor and the output D Fip Flop (DFF). The multiplexer selects one of the two input signals, the variable node processor output or the most significant bit (MSB) of Pch.
The Decoding Cycle (DC) corresponds to one iteration for the controlled start-up stochastic decoding. During N iterations in the decoding process (from 1 to N), the multiplexer forwards the Pch(k-1) signal to the output DFF at the first process cycle only. From 2 to the N cycle the multiplexer forwards the variable node processor output to the output DFF. With this approach, an important number of variable nodes start with a correct bit and escape the stochastic initial random state. In the case of the (2048, 1723) LDPC code, 98% of the variable nodes start with a correct bit at Eb/N0=5 dB.

IV. IMPLEMENTATION AND EXPERIMENTAL RESULTS
To validate the performance of the proposed algorithm, a (48, 24) LDPC code is implemented on Xilinx Virtex-6 VLX240T field programmable gate array (FPGA) device, with and without the CSS method. Delayed Stochastic (DS) decoding approach is used as an Advanced Stochastic Variable Node Processor. In comparison with the TFM and MTFM, DS decoding reduces the hardware complexity and the memory requirement of the stochastic decoders, for medium and long LDPC codes, with a reduced loss of performance.
Fig. 4 shows the implementation results of the BER performance of the proposed (48, 24) LDPC stochastic decoder. A maximum DC decoding of 4K is chosen. The CSS technique provides a BER hundred (100) times better than the common technique, at Eb/N0=4.5 dB. It is clear that the common DS approach cannot deliver a good performance with a short LDPC code, in contrast to the proposed one.

In addition to improve the convergence, the proposed CSS algorithm reduces the latency of the decoding process. It provides a decoding speed seven times faster than the common technique at Eb/N0=4.5 dB. The average number of decoding iterations decreases by increasing the SNR. It is around 1.06 DC at Eb/N0=5.0 dB.

In order to show the improvement of the proposed approach, the related (48, 24) LDPC decoder results are compared to the 6 bits offset min-sum algorithm (OMSA) and the 6 bits sum-product algorithm (SPA) for the (2048, 1723) LDPC code [17]-[19]. The FPGA implementation results of the developed LDPC stochastic decoder prove that the stochastic decoding can outperform both offset min-sum and sum-product algorithms.

V. CONCLUSION
In this paper, we proposed a new fully parallel stochastic LDPC decoding approach, which can outperform the fixed point SPA and MSA. This algorithm is called Controlled Start-up Stochastic (CSS) decoding. The improvement is obtained by introducing a clever and simple initialization process of the variable nodes. A Xilinx Virtex-6 VLX240T FPGA implementation of a very short LDPC code demonstrates the performance of the new stochastic decoder. The performance is widely enhanced with reduced latency of the decoding process. More performance is achieved by implementing medium and long LDPC codes.
