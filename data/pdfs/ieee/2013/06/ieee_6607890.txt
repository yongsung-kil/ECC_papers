An Improved Normalized Min-Sum Algorithm for LDPC Codes

Jinlei Chen, Yan Zhang and Ruiyi Sun Key Laboratory of Network Oriented Intelligent Computation
Shenzhen Graduate School, Harbin Institute of Technology HIT Campus of Shenzhen University Town Shenzhen, China, 518055
Email: jinleichen@gmail.com, ianzh@foxmail.com

Abstract—An improved normalized min-sum (IN-MS) algorithm is proposed for decoding low-density parity-check (LDPC) codes. In this algorithm, two normalized factors are used in check node computing, one for the minimum data and another for the second minimum data, and this algorithm achieves a better approximation to the BP algorithm. Simulation results show that the decoding performance of INMS algorithm is better than that of the normalized min-sum (NMS) algorithm.
Keywords- LDPC codes; LDPC decoding; normalized minsum algorithm
I. INTRODUCTION
Low-density parity check (LDPC) codes were proposed by Gallager in 1960s [1]. It was rediscovered by MacKay and Neal in 1996 [2]. A LDPC code is specified by a paritycheck matrix H which contains mostly zeros and only a small number of ones. A (N, K, dv, dc) regular LDPC code has a parity check matrix H of N columns and N-K rows. In H, each row contains exactly dc ones, and each column contains exactly dv ones. H can be represented by a Tanner graph [3], and the Tanner graph is a bipartite graph with nodes connected by edges. The nodes in the graph can be separated into two subsets (variable nodes and check nodes), and the edges only connect two nodes of different groups. The Tanner graph of a (N, K, dv, dc) LDPC code has N variable nodes, N-K check nodes, every variable node has dv neighbor check nodes and every check node has dc neighbor variable nodes.
Irregular LDPC codes were introduced in [4] and were further studied in [5], and good decoding performance could be achieved with carefully designed node degree distributions.
Quasi-Cyclic (QC) LDPC code [6] can ease the hardware implementation and achieve comparable performance to computer generated random irregular LDPC codes. The parity check matrix of QC-LDPC code is constructed by a base matrix and many cyclic shifted identity or zero matrixes, and its cyclic symmetry results in regular wiring and configurable structure in hardware implementation. The parity-check matrix of QC-LDPC code can be partitioned block wise and implemented with partially parallel decoder architecture, which achieves an efficient tradeoff between

very-large scale integration complexity and decoding throughput [7].
Iterative decoding of LDPC codes has received much attention [8-11]. Belief-propagation (BP) or called sumproduct algorithm can achieve excellent performance [12], but it is too complex to be implemented. The min-sum (MS) algorithm [9], which is an approximation to the BP algorithm in log domain, only needs add operations and comparisons, and it achieves very good tradeoff between complexity and error performance.
The MS algorithm has a performance loss compared to BP algorithm, and it comes from the overestimation of the message at check nodes. Therefore, the decoding performance can be improved by employing normalization or offset operation at check nodes, and these improved algorithms are called as normalized min-sum (NMS) and offset min-sum (OMS) algorithm [13]. In this paper, we proposed an improved normalized min-sum (IN-MS) algorithm, which can further improve the performance of the NMS algorithm.
The rest of the paper is organized as follows. In section II, the notations and NMS algorithm are introduced. In section III, the IN-MS algorithm is presented. The simulation results are described in section IV, and the conclusion is given in section V.

IV. SIMULATION RESULTS
Simulations are conducted to compare the performance of the IN-MS algorithm with the BP algorithm and NMS algorithm. In Fig. 1 and Fig. 2, the maximum iteration number Kmax is set to 100. In each BER curve, and for each point under BER of 10-4, over 105 code words are transmitted.
Fig. 1 is the fixed-point simulation results of a (8000, 4000, 3, 6)[14] regular LDPC code. In the fixed-point decoder, both the received LLRs are quantized to 6 bits and the extrinsic messages along edges are quantized to 8 bits, and two phase decoding is employed. The simulation results of floating-point BP algorithm are also shown in Fig. 1.

The normalized factor λ for the NMS algorithm is set to be 0.8125. and the normalized factors ( λ1 , λ2 ) for the INMS algorithm are set to be (0.84375, 0.78125).
With 8-bit quantization, a BER performance loss of about 0.1 dB is observed compared to the floating-point simulations. It is shown in Fig. 1 (a) that the IN-MS algorithm has about 0.1 dB improvement compared to the NMS algorithm, and this improvement is observed at the BER of 10-6. Fig. 1 (b) shows the average iteration number of various decoding algorithms for this (3,6) code, and INMS algorithm has less average iteration number than NMS algorithm.
Fig. 2 is the fixed-point simulation results of the 2304 bit irregular QC-LDCP code in IEEE 802.16e, and the code rate is 1/2. The normalized factor λ for the NMS algorithm and the normalized factors ( λ1 , λ2 ) for the IN-MS algorithm are the same as in the previous example. It is shown in Fig. 2 (a) that the IN-MS algorithm has about 0.1 dB improvement compared to the NMS algorithm. And the IN-MS algorithm has less than 0.1 dB BER performance loss compared to the floating point BP algorithm. Fig. 2 (b) shows the average iteration number of various decoding algorithms for this rate1/2 QC LDPC code, and IN-MS algorithm has less average iteration number than NMS algorithm.
V. CONCLUSIONS
In this paper, we have proposed an improved normalized min-sum algorithm for LDPC decoding. The proposed INMS algorithm has two normalization factors and it has the same decoding complexity as the NMS algorithm. We also showed how to obtain the normalization factors by simulation. Finally, simulation results showed that the INMS algorithm has better BER performance than the NMS algorithm.
