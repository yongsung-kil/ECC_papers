Multiple-rate Multiple-length QC-LDPC Codes Design with Near Shannon Limit Performance
Liwen Fan, Kewu Peng, Changyong Pan, and Jian Song

Abstract—Multiple-rate multiple-length LDPC codes with near Shannon limit performance at each rate are demanded in the next generation digital television terrestrial broadcasting (DTTB) systems. Row-Combining with Edge Variation (RCEV) codes, which generate higher rate codes by linearly combining the rows of parity-check matrix of a lower rate mother code, are rate-compatible with constant code length and simple hardware decoder architecture. This paper proposes an improved multi-rate scheme called Row-Combing-Splitting with Edge Variation (RCSEV) codes, where row splitting is introduced to improve the freedom of code construction at each rate. Multi-length code design can also be achieved by RCSEV codes with Quasi-Cycle (QC) structure. Finally, a set of multi-rate, multi-length candidate codes are proposed for the next generation DTTB systems. A comparison with DVB-S2 LDPC codes is performed and the better performance of our proposed code is validated via both extrinsic information transfer (EXIT) chart analysis and bit error rate (BER) simulations. Hardware implementation result demonstrates that the complexity of the multi-rate decoder is close to that of single rate counterpart.
Index Terms—Channel coding, low-density parity-check (LDPC) codes, multiple rate, quasi-cyclic.
I. INTRODUCTION
Channel coding with three important features are demanded in the next generation digital television terrestrial broadcasting (DTTB) systems: multi-rate, multi-length and near Shannon limit performance at each code rate. Near Shannon limit performance can be obtained by low-density parity-check (LDPC) codes [1], [2]. As a popular family of LDPC codes, QC-LDPC codes [12], [13], whose generator matrix has a block structure that consists of square sub-matrices each of size b×b, can provide excellent error-correcting performance with relatively simple multi-code-length schemes by adjusting b for different code length while sharing same template generator matrix.
Nowadays, multi-rate is a necessary requirement for most integrated FEC schemes. To keep the implementation as simple as possible, rate-compatible scheme is a quite efficient solution, which uses same basic hardware architecture to decode under

The basic idea of row combining is to generate higher-rate codes from a lower-rate code (called the mother code in this paper) by linearly combining the rows of mother code. Edge variation improves code performance by adding or deleting a few edges, which allows the code to have near-optimal variable node degree distributions and consequently near Shannon limit performance at each rate.
The main disadvantage of RCEV codes is that all the rates are tightly coupled. In the code design procedure, which rows to be combined and how they are combined are predetermined. Once the mother code is determined and optimized at its own rate, higher-rate codes are also determined except for several edges added or deleted. Thus, the performance of higher-rate and lower-rate codes is difficult to be optimized simultaneously.
Row splitting is proposed in an earlier paper [9]. It is more flexible than row combining method, because there are different ways to split a check node. In this paper, we introduce row splitting with RCEV codes to provide the flexibility and freedom of code construction at higher-rate codes, and it is named as Row-Combining-Splitting with Edge Variation (RCSEV) codes.
Multi-rate can be achieved by RCSEV codes via row combining or splitting with edge adding or deleting. Meanwhile, multi-length can be achieved by QC-LDPC code structure with different sub-matrix size b. Near Shannon limit performance is achieved by the optimization of variable node degree distributions with the help of extrinsic information transfer (EXIT) analysis [15]-[17].
In this paper, a set of multi-rate and multi-length QC-LDPC codes with near Shannon limit performance at each rate is constructed and provided as a possible candidate for the next generation DTTB systems. EXIT chart and bit error rate (BER) simulations demonstrate the superior performance of the proposed codes. Hardware implementation result shows that the complexity of the multi-rate decoder is close to that of single rate.

Row-combining is fairly strict: once given which rows to be

flexible: because there are different ways to split a check node

and the result depends on how it is split.

The main goal of row-combining and row-splitting is to

maintain the same fundamental hardware architecture for

multiple rates. In general, RCS won’t lead to a simpler decoder

for any particular rate but provides a simple overall architecture

with a smaller chip area. There are a couple of examples of how

decoder simplification might be accomplished [4], [8]. For a

typical LDPC decoder adopting Belief Propagation algorithm, the horizontal process done by Horizontal Process Unit (HPU) alternates with the vertical process done by Vertical Process Unit (VPU) throughout the decoding process. As for row-combining and row-splitting, HPUs of different rates can be multiplexed as one multi-rate HPU, and VPUs of different rates can be multiplexed as one multi-rate VPU. An efficient hardware implementation would decode the higher-rate code by performing the extra-message generation on top of the message computing done for the lower-rate codes, thus maintaining the basic hardware architecture.

B. Edge Variation
It is both the variable node degree distribution and the check node degree distribution that play the most important roles in the LDPC code performance. Regular check node degree distributions, in which every check node has the same degree, tend to approximate theoretical optimality [10]. Therefore, the check node degree distribution of all the rates should be near regular, if possible.
With strict row-combining or row-splitting, edges are neither created nor deleted. The number of neighbors of the variable nodes remains the same as the rate changes, thus all of the rates share the same variable node degree distribution but different check node degrees. This implies that the node degree distribution cannot be optimized for multiple rates simultaneously. As stated in [10], different rates require different variable node degree distributions for theoretical optimality, which is the exact reason that edge variation should be introduced to allow the addition and deletion of edges so that the degree distributions can be finely tuned for different rates simultaneously.

V. CONCLUSION
Multiple-rate, multiple-length LDPC codes with near Shannon limit performance at each rate are required in the next generation DTTB systems. Inspired by RCEV codes, this paper proposes an improved multi-rate scheme called RCSEV codes, where row-splitting is introduced to improve the freedom of code searching at each rate. Multi-rate, multi-length codes design can be achieved by proposed RCSEV codes with QC-LDPC structure and well-tuned node degree distributions. A set of possible candidate codes is proposed for the next generation DTTB systems with near-capacity performance at each code rate as well as low-complexity implementation of multi-rate decoder. As shown in EXIT chart analysis and BER simulations, our proposed QC-LDPC codes achieve better performance than that of DVB-S2 at the same rate.
