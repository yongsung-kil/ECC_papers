Implementation of Coded Modulation and FEC
Deyuan Chang1, Fan Yu1, Zhiyu Xiao1, Nebojsa Stojanovic2, Changsong Xie2, Liangchuan Li1, Xiaogeng Xu1, Qianjin Xiong1
1Huawei Technologies Co., Ltd., Network Research Department, Shenzhen, 518129, China 2Huawei Technologies Duesseldorf GmbH, European Research Center, Riesstrasse 25, D-80992 Munich, Germany
Abstract: We discuss implementation aspects of LDPC based soft-decision FEC schemes for high speed optical systems. Several BICM schemes with LDPC convolutional codes are proposed and compared in respects of both performance and implementation complexity.
OCIS codes: (060.2330) Fiber optics communications; (060.4510) Optical communications
1. Introduction
The digital coherent receiver enables people to employ higher spectrally efficient phase modulation formats, such as 16QAM, 64QAM [1] and even 256QAM [2], so as to meet the increasing transmission capacity demand. However, Euclidean distances between constellation points are decreased as the modulation order increases. Forward error correction (FEC) and coded modulation are required to address this issue by achieving higher performance gain.
Powerful FEC schemes using either hard decision (HD) decoding or soft decision (SD) decoding were proposed. A HD-FEC named CI-BCH code with 7% redundancy has a net coding gain (NCG) of 9.35dB [3]. A 20% overhead hard-decision Turbo product code showed an NCG of 10dB [4]. Compared to HD-FEC codes, SD-FEC codes achieve more than 1dB higher NCG at the expense of increased complexity. More attention has been given to the development of SD-FEC codes. Many SD-FEC schemes were proposed with more than 11dB NCG [5-9].
The use of different coded modulation schemes in optical systems has been actively considered. Concatenation of trellis-coded modulation (TCM) inner codes and block outer codes were proposed [10]. Multilevel coding (MLC) schemes were constructed [11] employing low density parity check (LDPC) codes as component codes. Bit interleaved coded modulation (BICM) schemes with shaped constellations were manifested in offline experiments [2]. However, consideration of implementation complexity is seldom given [12].
In this paper, we discuss the implementation aspects of soft-decision LDPC codes without the concatenation of outer codes. The wire routing congestion is the most obstructive problem when implementing an LDPC decoder. LDPC convolutional codes (LDPC-CCs) are suggested to be one effective way to reduce routing congestion. Furthermore, BICM schemes with LDPC-CCs are investigated by considering both performance and complexity.
2. Single Soft-Decision Codes for beyond 100Gb/s Optical Transmission Systems
Various SD-FEC codes schemes, especially those based on LDPC codes, were proposed by the optical community in recent years. It was well known that an LDPC code alone would suffer from the error floor at the bit error rate (BER) high above 10-15. One approach to suppress or completely eliminate the error floor is the concatenation of the LDPC code with an outer hard-decision code [13]. Reed-Solomon codes are conventionally chosen to eliminate the residual errors of inner codes [2, 8 and 13]. A triple-concatenated FEC was geniusly designed and implemented in [14]. An irregular LDPC code with short codeword is chosen as the inner code for its good error correction performance for high BER region from 10-2 to 10-3. The standard hard-decision concatenated code implemented in the OTU4 framer LSI, generally called “EFEC”, is used as the outer code. Some drawbacks are brought by the concatenation of codes. First, the concatenation limits the maximum achievable NCG performance. Next, since it requires an outer code and an additional interleaver (deinterleaver) to decorrelate the data sequences between the component codes, the complexity may become significant. Finally, more latency comes from the outer code.
There is a higher motivation to find a single LDPC code with low error floor than to focus on concatenated codes. We have proposed a single LDPC code with the codeword length of nearly 20000 [6]. The code was FPGA emulated with NCG of up to 11.3dB and no error floor down to BER of 10-15. A non-concatenated long LDPC code was also reported in [7], with the expected NCG result 11.3dB at BER of 10-15. Although many optimization techniques such as layered decoding algorithm and pipeline architecture can be used, it's still not straightforward to implement such long codes with 100Gb/s high throughput. One major challenge is the well known wire routing congestion, which becomes severe as the codeword length grows.
The number of wires in a conventional LDPC decoder is approximately proportional to N*q, where N is the codeword length and q is the number of quantization bits of soft information. Under the condition of fixed N of about 20000, one intuitive method is to reduce the value of q in order to alleviate the wire routing congestion. For QPSK signals, 3 or 4 quantization bits [14, 6] are required to obtain the performance less than 0.2dB away from the

floating point performance. If the number of quantization bits is reduced further, the performance will be degraded too much. Since the values of both N and q have a lower bound to satisfy the performance requirement, serial communication based decoding approaches, including bit-serial decoding [15] and stochastic decoding [16], seem to be inevitable to reduce the number of interconnect wires in LDPC decoders. In a bit-serial decoding scheme, a q–bit message is sent over a single sire in q clock cycles. While reducing wiring complexity, the bit-serial schemes have relatively poor throughput and high latency because of serial data processing. Stochastic decoders work in the probability domain and each message is represented by a Bernoulli sequence, that is, only one wire is used to transmit each message and the implementation of nodes processing is simplified to bitwise operations. Nevertheless, stochastic variable nodes may occasionally transit into the hold state failing to output the correct message, which can cause more than 0.2dB performance loss [16]. Hence, it is preferable to find a way out to reduce N without sacrificing performance.
As the counterpart of traditional LDPC block codes (all that discussed above), LDPC-CCs can obtain comparable coding gain with shorter N (called “constrained length” for LDPC-CC). Unfortunately, LDPC-CCs are traditionally decoded by message passing algorithms. Tens of iterations are needed to reach the decoding convergence, which incurs high latency and large implementation area and thus blocks the application of LDPCCCs in optical transmission system. We successfully applied layered decoding for LDPC-CCs [9]. Using half the number of decoding iterations, the layered decoding achieves the same performance as traditional message passing algorithms. As a result, the circuit area consumption and the latency of the LDPC-CC decoder are reduced by half.
A regular LDPC-CC with the constraint length of 10032 was constructed. Its performance was evaluated on the FPGA emulation system [9]. An NCG of 11.5dB at a post-FEC BER of 10-15 was achieved after 12 iterations of layered decoding, as is superior to all proposed LDPC block codes. The issue of wire routing congestion was alleviated a great deal due to the half reduced N.

3. Bit-Interleaved Coded Modulation with LDPC Convolutional Codes

In M-ary modulation systems where k bits are conveyed per symbol (2k = M), coded modulation schemes were proposed so as to obtain better system performance. Being viewed as an inner code without occupying redundant bandwidth, TCM hardly achieved better performance than uncoded modulations for 10-2 high BER region [10] where operating points of SD-FEC codes are generally located. As a special case of MLC [11], BICM schemes can provide dramatic performance with a single code [2]. Since LDPC-CCs have the advantages which have been discussed above, we specifically focus on LDPC-CC-coded BICM schemes with/without iterative demapping and decoding (ID) (named BICM(-ID) for short).

The block diagram of an LDPC-CC-coded BICM(-ID) scheme is shown in Fig. 1. The bit interleaver is omitted due to the built-in interleaving property of LDPC-CCs. BICM performs coding and mapping independently, where the application of LDPC-CCs is straightforward. Gray mapping is adopted because it gets near optimal performance for BICM [17]. As for BICM-ID systems, in order to achieve the best performance people usually optimize mapping and coding jointly to match each other. Four-dimensional (4-D) mapping is often employed to enlarge the optimization space [18, 19]. LDPC codes with irregular degree distribution are usually constructed by using extrinsic information transfer (EXIT) chart analysis [2]. Besides coding and mapping, constellation shaping provides additional gain by applying a Gaussian distributed constellation. Two classical shaped constellations are iterative polar quantization (IPQ) [20] and non-uniform QAM (NU-QAM) [21]. As an example, Fig. 2 depicts the 256-IPQ and 256-NU-QAM constellations.
The implementation complexity of a BICM(-ID) scheme is mainly contributed by the demapper and the decoder. A general soft demapper for BICM(-ID) is implemented according to the equation derived in [22]. As shown in Fig. 3, the number of floating point operations per bit (FLOPb) in one general demapper grows exponentially with the modulation parameter k. Meanwhile, the simplified demapper of Gray mapping in a BICM scheme has very low complexity [17]. However, irregular coded BICM-ID may suffer from high error floor and the concatenation of an outer code is essential, which increases the complexity and the latency.
Just like other signal processing modules, a realistic implementation of BICM(-ID) needs the tradeoff between performance and complexity to determine a good balance point. We use the case of M = 256 to illustrate our point of view. BICM(-ID) schemes with different constellations are investigated: IPQ, NU-QAM and U-QAM. Gray mapping is employed to NU-QAM and U-QAM, and a Gray-like labeling is designed for IPQ. A regular 37.14%overhead LDPC-CC with N = 9262 is constructed by shortening a rate-0.75 mother LDPC-CC code with N = 10032 so as to make a comparison with the results presented in [2]. The constructed code was employed in all BICM(-ID) schemes. The number of iterations is 13, exactly the same as that in [2]. Fig. 4 shows that all these schemes, even BICM without ID, achieve the comparable or superior performance to those illustrated in [2]. Among all the schemes, BICM with 256U-QAM has the worst performance with an estimated NCG of 14.7dB (compared to uncoded 256U-QAM). That demonstrates the suitability of LDPC-CCs for high order modulations. However, not all of the schemes have a realistic way to be implemented. The most pragmatic scheme is BICM with NU-QAM because of its good performance and the low complexity of Gray demapper in BICM. While the performance of BICM-ID with IPQ is about 0.1dB better than that of BICM-ID with NU-QAM, BICM-ID with NU-QAM is preferred to be implemented because it’s easier to apply the simplified sub-region demapper to NU-QAM. Nevertheless, unless the complexity of the sub-region demapper in BICM-ID can be further reduced by some newly proposed algorithm, the implementation of BICM-ID with NU-QAM is not pragmatic yet.

4. Summary

We discuss the measures to address the issue of wire routing congestion that is encountered when implementing a long-block soft-decision LDPC code. LDPC-CCs with relatively short constraint length are suggested to be one novel way to alleviate routing congestion. The suitability of LDPC-CCs for high order modulations is illustrated by proposed BICM schemes.
