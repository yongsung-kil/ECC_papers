Improved Check Node Decomposition for Linear Programming Decoding
Xiaopeng Jiao and Jianjun Mu

Abstract—For the linear programming decoding (LPD) proposed by Feldman et al., the number of constraints increases exponentially with check degrees. By decomposing a high-degree check node into a number of degree-3 check nodes, the number of constraints grows linearly with check degrees. In this letter, we show that the size of the LPD can be reduced by decomposing a high-degree check node into a number of degree-4 check nodes. The LPD using the degree-4 decomposition leads to almost the same number of constraints as using the degree-3 decomposition, while the number of auxiliary variable nodes is less than half of the one using the degree-3 decomposition. Moreover, when decomposing a high degree check node into a number of check nodes with degree d, d > 4, the number of constraints increases rapidly and the size of the LPD becomes larger than the degree-4 decomposition. It is demonstrated on an LDPC code and a BCH code that the decoding time of the degree-4 decomposition is the smallest among the different decomposition methods.
Index Terms—Check node decomposition, linear block code, linear programming decoding (LPD), high-degree check node.
I. Introduction
L INEAR programming decoding (LPD), first proposed by Feldman et al., is an alternative decoding method for linear block codes [1]. This method is based on an LP relaxation of the maximum-likelihood (ML) decoding problem. LPD offers some advantages over iterative message passing decoding (IMPD). First, LPD has the ML-certificate property, e.g., whenever it outputs a codeword, it is guaranteed to be the ML codeword. Second, LPD is easy to analyze because of its mathematical programming background. Finally, LPD is not limited to sparse matrices, like those of LDPC codes. The research on LPD is mainly twofold: lowering the decoding complexity and improving the error-correcting performance. For more details, please refer to the recent survey paper [2].
One problem of Feldman’s LPD formulation is that the number of constraints grows exponentially with the check degrees. For high-density parity-check (HDPC) codes, this number will be large and the original LPD becomes intractable. Though Feldman et al. proposed an alternative LPD formulation to overcome this problem [1], the number of variables and constraints still increases cubically with the check degrees. Yang et al. [3] and Chertkov et al. [4] independently proposed a new representation of the LPD formulation where each highdegree check node is decomposed into a number of degree3 check nodes and some auxiliary variable nodes, which is

Another appealing method with low complexity is the adaptive linear programming (ALP) decoding [5]. The ALP method can reduce the decoding complexity by solving a sequence of smaller LP problems. The number of constraints used by the ALP decoding is significantly smaller than other LPD formulations.
When the decomposition method is considered, a highdegree check node can be decomposed into check nodes with other degrees not necessarily with degree-3 (e.g., degree-4 or degree-5). The question one may ask is whether there exists a better decomposition method than the degree-3 decomposition. In this letter, we study the check node decomposition methods with different check degrees. The results show that the number of constraints needed in the LPD formulation using the degree4 decomposition is almost the same as using the degree-3 decomposition, while the number of auxiliary variable nodes is less than half of the one using the degree-3 decomposition. Simulations for an LDPC code and a BCH code show that the decoding time of the degree-4 decomposition is the smallest among the decomposition methods. Therefore, the proposed degree-4 decomposition method can be seen as a promising non-adaptive LPD formulation for HDPC codes or LDPC codes with high-degree check nodes1.
II. Preliminaries
A binary linear block code C with cardinality 2k and block length n is a k−dimensional subspace of the vector space {0, 1}n defined over the field GF(2). Its parity-check matrix H has m rows and n columns where m n−k. A codeword x ∈ C if and only if Hx = 0. The Tanner graph of H is denoted by G = (V, E). The vertex set V consists of two disjoint node sets, namely the check node set I = {1, 2, ..., m} and the variable node set J = {1, 2, ..., n}. An edge (i, j) ∈ E connects node i and j if and only if Hi, j = 1. The degree of the variable (check) node is the number of 1s in the corresponding column (row).
Let Ni = { j ∈ J : Hi, j = 1} denote the index set of variable nodes which are adjacent to check node i. Considering the subset S ⊂ Ni, the values corresponding to bits x j is assigned as follows. Set x j = 1 for all j ∈ S , and x j = 0 for all j ∈ Ni\S . For j Ni, x j can be chosen arbitrarily. If |S | is even, the value assignments to the code bits are feasible for the check node i. Otherwise, the value assignments are said infeasible if |S | is odd. Let x denote the transmitted codeword over a noise channel, and y denote the received vector. The original LPD

III. Improved Check Node Decomposition
By decomposing a high-degree check node into a number of degree-3 check nodes, the number of inequalities increases linearly (not exponentially) with the check degrees [3][4]. In this section, we will show that decomposing a check node into a number of degree-4 check nodes is better than the degree-3 decomposition in terms of the size of the LPD formulation.
For the degree-3 check node, its parity-check equation C3 is denoted by C3 = {(z1, z2, z3) : (z1+z2+z3) mod 2 = 0, z1, z2, z3 ∈ {0, 1}}. The minimum polytope P3 [3] describing the relaxed convex hull of C3 is defined as
P3 = (z1, z2, z3) : z1 ≤ z2 + z3, z2 ≤ z1 + z3, z3 ≤ z1 + z2, z1 + z2 + z3 ≤ 2, z1, z2, z3 ∈ [0, 1] .
To distinguish the check node decomposition into different check degrees, P3 is also called the degree-3 polytope. For the degree-4 check node, C4 is denoted by C4 = {(z1, z2, z3, z4) : (z1 + z2 + z3 + z4) mod 2 = 0, z1, z2, z3, z4 ∈ {0, 1}}. The corresponding degree-4 polytope P4 is described as follows.
P4 = (z1, z2, z3, z4) : z1 ≤ z2 + z3 + z4, z2 ≤ z1 + z3 + z4, z3 ≤ z1 + z2 + z4, z4 ≤ z1 + z2 + z3,
z1 + z2 + z3 ≤ z4 + 2, z1 + z2 + z4 ≤ z3 + 2, z1 + z3 + z4 ≤ z2 + 2, z2 + z3 + z4 ≤ z1 + 2,
z1, z2, z3, z4 ∈ [0, 1] .
Generally, for a check node with degree d, the constraints for the degree-d polytope Pd can be easily derived from the FS inequalities. The number of constraints for Pd is 2d−1. Note that the box constraints zi ∈ [0, 1] are not considered when counting the number of needed constraints throughout this letter.
A. Motivation
Taking a degree-12 check node as an example, Fig. 1 illustrates the decomposition of this check node into a number of check nodes with degree-3, degree-4 and degree-5, respectively. In Fig. 1, the circle denotes the original variable node,

the double circle denotes the auxiliary variable node, and the square denotes the check node. In the final process of the decomposition, there may be a polytope less than degree-d. See Fig. 1 (d) for example, there is a degree-3 polytope when we using the degree-5 decomposition.
When the decomposition is not used (Fig. 1(a)), the number of constraints we need is 212−1 = 2048 and the number of auxiliary variable nodes is 0. For the degree-3 decomposition, the number of constraints we need is 10 × 23−1 = 40 and the number of auxiliary variable nodes is 9. When the degree-4 decomposition is used, the number of constraints we need is 5 × 24−1 = 40 and the number of auxiliary variable nodes is 4. For the degree-5 decomposition, the number of constraints we need is 3 × 25−1 + 23−1 = 52 and the number of auxiliary variable nodes is 3. From this example we can see that the number of constraints needed in the degree-4 decomposition is the same as using the degree-3 decomposition. However, the number of auxiliary variable nodes needed in the degree-4 decomposition is less than half of the one using the degree-3 decomposition. For the degree-5 decomposition, the number of auxiliary variable nodes is the smallest, but the number of constraints is the largest among the three decomposition methods.

V. Concluding Remarks
To summarize, we have investigated the check node decomposition methods for linear block codes with LPD. The conclusion we obtained is that the degree-4 decomposition is better than the degree-3 decomposition and the degree-d decomposition with d > 4. For linear block codes with a HDPC matrix, the proposed method can reduce the size of the LPD formulation significantly.
