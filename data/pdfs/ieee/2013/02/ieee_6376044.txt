Turbo Adaptive Threshold Bit Flipping For LDPC Decoding
Mohamed Ismail, Justin Coon, Imran Ahmed, Simon Armour, and Joseph McGeehan

Abstract—In this paper, we present a novel bit flipping algorithm for high-throughput decoding of LDPC codes. The algorithm exploits the turbo principle to reduce the average number of iterations and employs a bit-local thresholding operation to remove the need for a block-wise minimum operation – which is common to many conventional bit flipping decoders – that hinders decoder throughput. Through simulations the proposed decoder is shown to achieve comparable bit-error rate performance to state-of-the-art techniques while requiring approximately half the number of iterations on average thus facilitating high-throughput decoding.
Index Terms—LDPC, bit flipping, decoder, high throughput.
I. INTRODUCTION
I N his thesis Gallager [1] not only described error correcting codes based on low density parity check matrices, but also detailed a number of practical decoding algorithms. A probabilistic soft decision message passing algorithm, often referred to as the sum-product (SP) algorithm, capable of achieving optimum performance was one class of decoding algorithm. Although SP techniques have been shown to perform exceptionally well, coming within 0.0045 dB of the Shannon capacity limit [2], they do suffer from a high implementation complexity, which ultimately hinders decoder throughput in practice. An alternative algorithm, falling into the class of bit flipping (BF) methods, was designed to be less complex than SP decoding [1]. At the expense of error correcting performance, BF methods allow the possibility of significantly higher throughput, smaller silicon area and reduced power consumption in practical systems.
In this paper, a novel BF algorithm for high-throughput decoding of LDPC codes is presented. The algorithm, termed turbo adaptive threshold bit flipping (TATBF), is capable of achieving a bit-error rate equal to the best known BF decoder to date (the multi-bit gradient descent bit flipping decoder [3]) while requiring fewer iterations on average to decode each codeword. The proposed method is well suited for decoding of layered codes where the parity check matrix is made of of circulant permutation sub-matrices and zero sub-matrices. The addition of a suitable perturbation is shown to improve error correcting performance and reduce the average number decoder iterations of the TATBF decoder.

BF decoders look at the set of parity checks each received bit is involved in; if the total number of check failures the bit is involved in exceeds some threshold, the sign of the received bit is inverted to form an updated received vector. The next iteration of the algorithm will re-calculate the syndrome vector using the updated hard-decision vector. Recently, Wadayama [3] formulated a new way of calculating the decision metric based on the gradient descent formulation. The Gradient Descent Bit Flipping (GDBF) algorithms outperform previously published algorithms in error correcting ability and, significantly, in the number of average iterations needed for successful decoding. The majority of bit flipping algorithms, including the GDBF ones, require locating a minimum value over the whole block length, which for LDPC codes can be large. This min operation incurs significant delay when implemented in an Application Specific Integrated Circuit (ASIC) causing a limitation to the maximum achievable throughput. The decoding algorithms proposed below remove the need for a global minimum operation, using bit local operations instead.

The ATBF algorithm uses the gradient descent formulation of (3) to calculate the contribution of each bit to the overall objective function. The second term of the right hand side of (3) represents the parity check a received bit is involved in. If all the participating parity checks are satisfied this term has a maximum value equal to the column weight in the parity check matrix. Conversely, if all the parity check are not satisfied, this term will have a minimum value equal to the negative of the column weight. The ATBF algorithm uses a negative threshold against which the Δk(GD) value is compared, and for each iteration a bit either changes sign if the corresponding Δ(kGD) is below the threshold value or the threshold value is scaled towards zero by a fixed amount. This way the ATBF algorithm causes the search process to move towards a global maximum by appropriate bit local decisions.

The loss in performance can be traced back to the nature of the gradient descent search process, which can become trapped at a local maximum, i.e., the objective function finds a local maximum, which the algorithm declares as a valid decoded codeword. The search process may be restarted by detecting the point at which the objective function reaches a maximum for which the decoded data is not a valid codeword. At such a point all the threshold values are saved and replaced with λk = θ2 + α where θ2 is a constant and α is a zero mean Gaussian distributed random number with variance σ, drawn for each λk value. The decoding process proceeds for one iteration with the new threshold values before continuing with the saved set of values.

IV. CONCLUSION
The gradient descent formulation of bit flipping provides an elegant representation of the procedure leading to good error correcting performance. However, the gradient descent based algorithms require locating a minimum value over the entire block length. A hardware implementation of such a function incurs a prohibitive delay in high-throughput applications due to the cascaded pair-wise comparison. In this paper, we presented a novel bit flipping algorithm called turbo adaptive threshold bit flipping that removes the need for locating a global minimum, instead using bit-local operations. The TATBF algorithm utilises the latest available information in the decoding process to reduce the overall number of decoding iterations. Additionally, a perturbation mechanism was introduced in the gradient descent search process that resulted in an improved TATBF algorithm giving the best error correcting performance and lowest number of decoding iterations out of all of the bit flipping algorithms tested. Future work will look at improving performance of the algorithm with irregular codes.
