12.7 A 45nm 6b/cell Charge-Trapping Flash Memory Using LDPC-Based ECC and Drift-Immune Soft-Sensing Engine
Kin-Chu Ho1,2, Po-Chao Fang1, Hsiang-Pang Li1, Cheng-Yuan Michael Wang1, Hsie-Chia Chang2
1Macronix, Hsinchu, Taiwan, 2National Chiao Tung University, Hsinchu, Taiwan
To satisfy the demand of higher storage density, storing multiple-bits-per-cell technique is widely adopted. As presented in [1], a 4b/cell Flash memory by using error-detection (ED) scheme stores 2b data on two sides of a memory cell individually. Since the noise margin becomes smaller, the distribution drifts due to program disturb, data retention and temperature variation will cause higher raw bit-error-rate (RBER) if the sensing level (voltage) are not adjusted accordingly. The ED scheme can detect the drift direction by counting and storing the number of cells (Ni) with threshold voltage (VTH) below the ith sensing level (VREF i). A simple example with page size 1KB is demonstrated in Fig. 12.7.1. During a read operation, the number of cells (Ni, measured) whose VTH below VREF i is counted and compared to Ni. The ED scheme can find out a sub-optimal sensing level when Ni, measured and Ni is close enough. In this paper, a production 16Gb 45nm 4b/cell ONO-based charge-trapping (CT) Flash memory is demonstrated to achieve 6b/cell capability. Since the adjacent distributions for 6b/cell are much closer to each other, even a BCH code with ED scheme fails to correct all the patterns. However, by using a new 1-3-3 mapping and LDPC codes with a developed drift-immune soft-sensing (DI-SS) engine, the 45nm 4b/cell CT Flash memory is boosted to 6b/cell. The data flow of programming data is also shown in Fig. 12.7.1.
Figure 12.7.2 shows a conventional 23-level 1-2-4 mapping, indicating it requires 1, 2 and 4 sensing times for high/middle/low pages to decode data respectively. The low page data in the 1-2-4 mapping usually suffers from higher RBER and longer read latency than high/middle pages due to its 4 sensing times. Hence, the low page data is always the worst case considered by error-correcting codes (ECC). We develop a new 23-level 1-3-3 mapping to balance the RBER between high/middle/low pages, reducing the RBER of low page data by 20% and the worst-case read latency of random access by 25%. In addition, the program sequence presented in [2] is also used to reduce the program disturbance.
In Flash storage applications, ECC plays an important role to achieve high reliability [3]. Although the low-density parity-check (LDPC) code can provide strong correcting capability, both the decoding performance and implementation cost highly depend on soft information quality that indicates the error probability of each code bit. To acquire accurate soft information, a drift-immune soft-sensing (DI-SS) engine is developed and the corresponding data flow is shown in Fig. 12.7.3. At the read stage, user data and soft information are obtained; then the log-likelihood ratio (LLR) with non-uniform quantization can be calculated for the LDPC decoder. If the LDPC decoding process fails, the controller will either get extra soft information to enhance the correcting capability or re-read both the hard and soft information, which is called Read-Retry operation. Due to the random telegraph noise (RTN) in Flash memory, error patterns are likely to be slightly different between each Read-Retry, indicating that it may help ECC to skip some uncorrectable error patterns. As shown in Fig, 12.7.3, the DI-SS engine performs the ED scheme with moving-read operation until the sub-optimal sensing level is found, and then obtains the accurate soft-2-bit information by two additional sensing near the sub-optimal sensing level. Note that each sensing result is updated to a buffer if it is not the sub-optimal one, implying that only one more soft sensing is required to get the soft-2-bit information. Moreover, the engine can also provide soft-3-bit information by another two sensing if stronger correcting ability is needed.

Figure 12.7.4 shows the measurement results based on the DI-SS engine. The code rate (R) is defined as k-bit user data divided by n-bit codeword. Since the noise margin is very small in a 6b/cell CT Flash memory, a conventional R = 0.85 BCH code with ED scheme fails to correct all the patterns at the very beginning. However, a soft-3-bit LDPC code with R = 0.875 can correct all the patterns up to 10-year retention time. The measurement results also show that Read-Retry helps both LDPC and BCH code to overcome more error patterns. At 10-year retention time, the soft-2-bit LDPC code can overcome over 80% error patterns, indicating that it is not necessary to acquire soft-3-bit information every time with the advantage of lower read latency. For the remaining uncorrected patterns, carrying additional soft sensing to provide soft-3-bit information can overcome more error patterns than Read-Retry operation by 9.9%. Once the soft-2bit LDPC code fails to decode, it is more efficient to get the extra soft-3-bit information for LDPC code to decode before using Read-Retry operation.
Figure 12.7.5 shows the number of decoding iteration of LDPC codes at 11-year retention time. When the DI-SS engine is applied to soft-3-bit LDPC codes, there is a 27% reduction in the average number of decoding iterations. In addition, the average number of moving-read for searching the sub-optimal sensing level in the DI-SS engine at VREF 2/VREF 5/VREF 7 (i.e., sensing low page) is about 0.3/3.6/3.9 respectively. It shows that the memory cells with higher VTH suffer more severely from data retention.
The R = 0.875 LDPC code described above is a quasi-cyclic (QC) LDPC code that can benefit from simple hardware architecture [4]. Figure 12.7.6 shows the decoder architecture uses shuffle decoding algorithm to achieve twice as fast decoding convergence speed [5]. The LDPC decoder possesses 1172 check node units (CNUs) and 293 variable node units (VNUs) to achieve high throughput for high-speed systems. Using 90nm CMOS technology, the gate count of LDPC decoder is 935K with throughput 5.7Gb/s with a 200MHz clock frequency with an average of 8 iterations. For low speed systems, the gate count can be reduced to 497K with throughput 520Mb/s.
Figure 12.7.7 shows the micrograph of our reliability measurement system of 6b/cell CT Flash memory. In summary, by using the new 1-3-3 mapping and LDPC codes with DI-SS engine, a 45nm 4b/cell CT Flash memory is boosted to 6b/cell. The new 1-3-3 mapping reduces the RBER of low page data by 20% and the worst-case read latency of random access by 25%. The measurement results also show that the LDPC codes with DI-SS engine correct all the errors up to 10year retention time and 27% reduction in decoding latency can be achieved. We believe that other non-volatile Flash memory such as NAND Flash memory can benefit from our techniques as well.
