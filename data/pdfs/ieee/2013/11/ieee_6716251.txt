LDPC code optimization based on Tanner graph mutations

Jan Broulím, Vjačeslav Georgiev, Jan Moldaschl, Lukáš Paločko

 Abstract — The paper presents LDPC error correcting
code optimization based on Tanner graph mutations. A statistical experiment focused on a heuristic LDPC code construction was performed with the genetic algorithm. The algorithm used is briefly described. Short wordlength LDPC code optimization was simulated. Results of these simulations are summarized in this paper.
Keywords — FEC, genetic optimization, LDPC.
I. INTRODUCTION
ERROR correction codes and especially LDPC codes [4] have become very popular these days because of their correction capabilities. Several algorithms for LDPC codes design have been presented in the literature and some are still being developed. This paper presents an optimization algorithm using Tanner graph mutations.
In general, Genetic algorithms [3] are a heuristic method to search for an optimal solution to some problem. These algorithms are based on creating a next generation set of solutions from the previous one by two methods crossover and mutation. All solutions in the set have to be evaluated whether they are useful or not. This is performed by a function called the fitness function. The function provides numerical information about the solution and its value is used to make a decision if the solution is good or if it should be dropped.
The paper discusses the possibility of using genetic algorithms for optimization of LDPC codes. All genetic algorithms require some pregenerated initial set of solutions (LDPC codes for our purpose). The initial set is generated by using the principle described in [1].
LDPC codes can be expressed by a parity-check matrix or by a Tanner graph [6]. The next generation of codes in presented optimization process is created by Tanner graph mutations of codes from the previous generation.

II. OPTIMIZATION ALGORITHM PRINCIPLES
As mentioned before, the initial set of different codes is pregenerated. All codes in the set are evaluated by the fitness function and sorted by their correction capabilities from the best to the worst. Codes are randomly selected after evaluation, mutated and consequently included in the next generation. But probability of the selection is not the same for every code. It decreases with increasing error rate (we expected that better codes would produce better mutants, but not necessarily).
In some cases mutations may not produce better codes. So, if we want to preserve the best code (or codes) in the next generations, it is necessary to include the code into the next generation without a mutation. See the explanation below.

B. Fitness function
The fitness function provides information about the correcting ability of LDPC code. On the basis of the return value of the fitness function we can decide whether a testing code is good or not.

C. Operations used in mutations
Operations used in the presented optimization algorithm can be divided into two groups - simple operations without usage of graph traversing and sophisticated operations

Function addLongDistEdges(maxBfsDepth, num : int, tg : TannerGraph) randomly selects one check node and computes the distance from all variable nodes (based on breadth-first search algorithm) to the selected check node. The distance means the minimum number of edges required to traverse the nodes. If the distance is larger than maxBfsDepth, it is considered as infinity. When distances are computed, a new edge is added between the randomly selected check node and the farthest variable node. The whole operation is repeated until the number of new edges reaches the desired number passed as a parameter.
Function breakCycles(maxLength : int, tg : TannerGraph) implements a parameterized cycle breaking. The correction capability of the LDPC code is negatively affected by the presence of short cycles in its Tanner graph. Thus, it is necessary to avoid short cycles. The function implements the modified breadth-first search algorithm originating from all check nodes. If the algorithm reaches the beginning node during graph traversing, a cycle is found and the distance from the beginning node is equal to the length of the found cycle. If the distance is larger than or equal to the maxLength parameter, the last edge in the cycle is removed.
Function minDegreeTwo(maxBfsDepth : int, tg : TannerGraph) searches for nodes with a degree less than 2. If that node is found, a new edge (or edges) is added to the node farthest from it. The algorithm of edge adding is similar to the addLongDistEdges function.

IV. SIMULATION RESULTS
A. Simulation parameters
LDPC code (n = 128, k = 64) optimization was simulated with these parametres: BFSDEPTH = 50, MAXPLUS = 20, MAXMINUS = 20, bit error rate THRESHOLD = 10-4. 30 codes were pregenerated by the algorithm described in [1]. Each set in consequently created generations contains 10 graphs and the simulation was stopped after 10 generations. The pregeneration and optimization were run in 76 separated threads. The AWGN (Additive White Gaussian noise) channel was chosen for bit error rate simulations. The SP algorithm for LDPC decoding was used.
B. Simulation results
30 codes were pregenerated in each initial set and the pregeneration was performed in 76 independent threads. A code with a minimum fitness value (it implies minimum Eb/N0 threshold) is considered as the best in each independent thread. Values of Eb/N0 [dB], at which the bit error rate drops below 10-4, are in tab.1 and tab.2.

V. CONCLUSION
A statistical experiment evaluating LDPC(128,64) code optimization was performed. The simulation results confirm an improvement in the LDPC codes bit error rate by using the genetic algorithm based on mutations. The maximum reached value of Eb/N0 threshold improvement was 1.15 dB, the average improvement was 0.56 dB. The code with the lowest bit error rate in the 10th generation falls below ber = 10-4 at Eb/N0 = 4.25 dB. Nevertheless, more simulations with different parameters and optimization of codes with longer codewords should be done.
