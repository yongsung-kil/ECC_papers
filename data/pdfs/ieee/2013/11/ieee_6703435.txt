Weighted Candidate Bit Based Bit-Flipping Decoding Algorithms for LDPC Codes

Qing Zhu, Le-nan Wu
School of Information Science and Engineering Southeast University
Nanjing, 210096, China Email: dreamathstat@163.com

Abstract—In order to improve the bit-flipping (BF) decoding algorithms, a weighted hard-decision based BF decoding algorithm for low-density parity-check (LDPC) codes is presented, in which the weights of variable-nodes are employed. A new metric is defined for the weight of candidate bits not only in unsatisfied parity check equations but also in satisfied ones. The connection between min-sum decoding algorithm and the proposed algorithm is presented. Simulation results show that the new decoding algorithm can provide better FER performance and faster convergence speed than the original candidate bit based bit-flipping (CBBF) algorithm.
Keywords—bit flipping (BF); low-density parity-check (LDPC); min-sum decoding; hard-decision
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] have received much attention since their rediscovery by Mackay in the 1990s [2]. LDPC codes are widely used in many scenarios such as 10 Gigabit Ethernet (10GBASE-T), visible light communication (VLC) [3-5] and deep-space communications due to their
Shannon-limited approaching characters. The decoding
algorithms can be divided into two types: soft-informationcomputation based decoding algorithms and hard-decision based decoding algorithms. In the former one, decoding algorithms such as belief propagation (BP) [6, 7] decoding algorithms and min-sum decoding algorithms usually have excellent decoding performance at the cost of high computation complexity. In the later one, decoding algorithms such as bitflipping (BF) algorithms are applied in some low time-delay scenarios, e.g., flash memory and VLC. BF decoding algorithms can be executed only using the modulo-two additions and simple comparison operations. The merits of high-speed and low complexity make BF and BF-like decoding algorithms attract much attention in practical applications. In [8], a candidate bit based bit-flipping (CBBF) algorithm is invented, in which a new metric defined by the number of candidate bits in a parity check equation is proposed to utilize the parity check information sufficiently. In [9], bias terms are introduced to improve the performance of CBBF algorithm. In [10], two extra bits are brought in the decoding process. One is to indicate the weight of each variable-node and the other is to represent the reliability of each check-node.

III. PROPOSED DECODING ALGORITHMS

A. Motivations
In [8], the concept of candidate bits is applied to measure the weight of unsatisfied parity check equations. The weighted function is linear with the number of candidate bits participated in the unsatisfied parity check equations. In this paper, optimized weight function is considered not only for the unsatisfied parity check equations but also for the satisfied parity check equations. Suppose that only candidate bits are possible to be flipped. For an unsatisfied parity check, if it contains only one candidate bit, it strongly suggests that this candidate bit should be flipped. If there are two candidate bits in the unsatisfied parity check, each candidate bit only has about 50% probability to be flipped, so the value of the weighted function is much lower than the aforementioned case. Similarly, for a satisfied parity check, in order to maintain the correctness of the parity check, it strongly suggests that this candidate bit should not be flipped when there is only one candidate bit in the parity check. If a satisfied parity check contains two candidate bits, both of the two candidate bits should be flipped or not be flipped. The probability of flipping both of the candidate bits is slightly bigger than that of not flipping due to the characters of candidate bits.

V. CONCLUSION
To improve the FER performance of BF-like decoding algorithms, an optimized weighted candidate bit based bitflipping algorithm is proposed. Both of the unsatisfied parity check equations and the satisfied parity check equations offer information for candidate bits. The proposed algorithm can be seen as a coarse hard-decision version of min-sum decoding algorithm. From the simulations, we can see that the proposed decoding algorithm achieves coding gains from 0.25 dB to 0.6 dB. In the subsequent research, the decoding algorithm will utilize soft information and generalize the idea to non-binary LDPC codes.
