A Low-hardware Consumption FPGA based Configurable LDPC Decoder

Lijun Zhang，Ying Jiang
School of Electr. and Inform. Eng. Beijing Jiaotong Univ. Beijing, 100044, China
Email: ljzhang@bjtu.edu.cn

Abstract—FPGAs are widely used for evaluating the performance of low-density parity check (LDPC) codes. But most of the existing LDPC decoders are designed for structured codes and extremely resource consuming. In this paper we propose a low-consumption configurable decoder architecture and a universal mapping algorithm for extrinsic messages to cope with structured or random regular LDPC codes. It can be implemented in low-priced products such as XILINX Spartan FPGAs family. In comparison with the decoders previously implemented, the proposed decoder significantly reduces the number of block RAMs used for extrinsic messages with no loss of throughput.
Keywords—configurable; LDPC decoder; low compuction; FPGA
I. INTRODUCTION
LDPC code has become one of the best choices for the applications that are required to achieve near Shannon capacity in communication [1]. It has been adopted in recent digital communication standards, e.g. DVB-S2 [2], and Mobile WiMax (802.16e) [3]. For some applications it is necessary to prove the appearance of error-floor down to 10−12 [4], which costs so much time on a conventional CPU that it is almost impractical to be implemented. As a result, FPGAs are widely used to emulate the performance of LDPC codes [4, 5, 6, 8].
In the implementation of a LDPC decoder, messages should be stored during the iteration, so the capacity of memories is a key limiting factor in an FPGA implementation. As to quasicyclic LDPC (QC-LDPC) codes, messages can be packed based on the circulant permutation matrix (CPM) location and mapped to embedded memory [4, 5]. But when the number of CPMs is much larger than the number of block RAMs, it becomes impossible to map each CPM to a block RAM. Moreover, the Gaussian elimination has destroyed the structure of parity-check matrix, so if we want to decode a systemic code and take the advantage of the QC structure of its parity-check matrix we have to rearrange the input of the initialization messages as well as the output of the hard decision vector. In [6] a decoder for QC-LDPC codes is proposed. But this approach has limited freedom in choosing the degree of parallelism, and is restrained by the structure of parity-check matrix. On the other hand, in many applications, we do not use QC-LDPC codes, such as in DVB-S2 standard, IRA LDPC codes are used, which should be implemented in a universal

decoder. In [8] a flexible decoder is proposed, but the price for this fully flexible architecture is very high.
All of these drawbacks motivate us to develop a lowconsumption decoder architecture which can cope with any structured or random LDPC code. Such a kind of decoder for regular LDPC codes is proposed in this paper, which can be implemented on the lower-priced FPGAs. There are three advantages of our decoder. (1) It is reconfigurable to deal with both structured and random code with no restrictions on paritycheck matrix. (2) The architecture is cost-saving, and the degree of parallelism can be regulated based on the resources on FPGAs. (3) We do not have to rearrange the input and output of the codeword, because the parity-check matrix used in the decoder is rearranged in the Gaussian elimination.
The rest of this paper is organized as follows. We give an introduction to LDPC codes and the min-sum decoding algorithm in section II. In section III, we describe the overall architecture including the mapping algorithm of our decoder, and present the performance analysis. At last, a conclusion of this paper is given.
II. LDPC CODES
The binary LDPC codes are defined by an m× n sparse parity-check matrix H. Equivalently the LDPC codes can be described by a Tanner graph with m check nodes (CNs) corresponding to the rows and n variable nodes (VNs) corresponding to the columns of H [1]. CNi is connected to VNj only if hij=1. We call a LDPC code is (j,k)-regular code if each VN connects to j CNs, and each CN connects to k VNs. In terms of encoding and decoding implementation, the most advantageous is the QC-LDPC codes whose special paritycheck matrix makes a high throughput be available [5]. The min-sum [9] algorithm and the decoding algorithm proposed in [12] are all reduced-complexity algorithms approximation of the SPA [10]. With carefully chosen related factor, the minsum algorithm can perform as well as SPA. III. DECODER ARCHITECTURERE
In the min-sum algorithm, the message L(qij ) and L(rji ) should be stored in the Extrinsic RAM during the iteration, so firstly, we introduce a mapping algorithm to determine the locations of the extrinsic messages.

A. Message Packing Algorithm
We use the well-known (96, 50) Gallager code from MacKay’s website to illustrate mapping algorithm [7]. As shown in Fig.1, the VN to CN messages are packed in RAM1, RAM2 and RAM3 and CN to VN messages are packed in RAM4, RAM5 and RAM6.rij represented the row index of the jth “1” position in the ith column. When RAM1, RAM2 and RAM3 are read, RAM4, RAM5 and RAM6 are written, which makes it collision-free when the memories are read (or written) by the calculation units.

B. Row Grouping
In an LDPC decoder, memory conflict arises if different calculation units try to access the same RAM at the same time. We have two methods to avoid memory conflicts. The simple method is to configuring the decoder as a serial architecture, performing two CNUs simultaneously. Apparently, this serial structure results in low throughput. For QC-LDPC codes, the messages of a row of CPMs and the other rows of CPMs are packed in different RAMs, because the first nodes in the columns are all in the first row of CPMs, similarly, the second nodes in the columns are all in the second row of CPMs, so are the other rows. Then we can perform more CNUs simultaneously as long as they read messages from different RAMs. Motivated by QC-LDPC codes, the other method is to divide the rows into groups like the row of CPMs to make it conflict-free and improve the throughput.

IV. CONCLUSION
A configurable LDPC decoder architecture with no limitation of parity-check matrix as well as a mapping algorithm is presented in this paper. We implemented it on a XILINX Spartan-6 FPGA and compared it with the existed partial decoder. The comparison result shows that the number of memories used by the proposed decoder is only one-forth of the partial decoder, keeping the throughput almost unchanged.
