Partially Decoder for Green Terrestrial Broadcasting
System

Abstract—In this paper, a partially decoding scheme, combined
with the rate compatible Raptor-like low-density parity-check
(LDPC) codes, is proposed to provide more robust, power saving
solutions for the future terrestrial broadcasting systems. Different
from the existing works, the proposed algorithm uses the least
information in the codeword and stops immediately after the
decoding is successful. We illustrate two different scenarios under
which this scheme will be quite useful. Simulation results show
that the proposed scheme is effective.
I. INTRODUCTION
The terrestrial broadcasting system is experiencing a very
challenging era [1]. It has been takeing up quite a lot of good
spectrum resources in UHF and VHF band, but providing
less and less attractive services, as the booming of other
technologies such as cellular networks, WiFi, Internet, etc.
However, terrestrial broadcasting still has its own need to
exist as a “one-to-many” free public services, such as news,
entertainment, and especially providing information to the
general public when natural disaster happens (flood, tornado,
forest fire, tsunami, earthquake, etc.). For these above services,
high data rate is actually not the first priority any more. It
is more desirable for the system to be more flexible, power
saving, has better performance and adapts to mobility.
Channel codes are very important for terrestrial broadcasting
system as they help improving the system performance and re-
liability in wireless channels. There have been several updates
of channel codes in broadcasting systems. The first generation
systems, such as DVB-T and ATSC adopt trellis code and
RS code. When time comes for the second generation, as
low-density parity-check (LDPC) codes become more and
more popular, many systems begin to use LDPC as the error
correction codes (e.g., The DTMB system in China and DVB-
T2[2]).
LDPC codes were proposed by Gallager [3] in the early
1960s but did not receive much attention until decades later
[4]-[6]. LDPC codes have a sparse parity check matrix and
could be decoded efficiently by the iterative belief propagation
(BP) algorithm. (See, for example, [7].) When used in the
broadcasting system, we need LDPC codes with different code
rates to adapt to different environments. Currently, different
parity check matrix are used for codes of different rates,
which improves the system complexity. In the literature, there
exist two other methods to generate rate compatible LDPC
codes from one single parity-check matrix: puncturing which
removes some of the parity bits from the encoded LDPC code-
word and extending which increases code length by expanding
the given parity-check matrix. However, puncturing [8][9] and
expanding [10] will generally change the degree distributions
of the codes and thus bring performance degradation.
Recently, fountain codes especially LT codes and raptor
codes have attracted much attentions for its capacity-achieving
character [11]-[13]. [14] proposed a type of protograph-based
raptor-like LDPC codes which consists of a high rate LDPC
code as “mother” code and an LT-like code (single-parity-
check code) as the incremental redundancy part. Moreover,
they realizes rate compatibility by puncturing on the LT-like
code part.
In this paper, we propose a partially decoding scheme for
the rate compatible Raptor-like LDPC code. We do not punc-
ture the transmitter code matrix to realize rate compatibility.
Rather, we achieve different coding gain by utilizing different
parity check bits in the receiver. The decoder will start by only
using check bits in the ”mother code”, which are presented in
the head of the codewords following the information bits. If the
decoding process fails, the decoder will wait for more check
bits to come and try again. Once the decoding is successful,
we stop receiving and processing of the following bits in the
same codeword. This scheme will greatly decrease the system
latency, especially when the channel condition is good.
We organize the rest of the paper as follows. Section
II briefly reviews the structure of raptor-like LDPC codes.
Section III details the proposed partially decoding techniques.
Section IV reports the simulation results and in Section VI,
we conclude the paper.

B. Raptor-like LDPC Codes
LT-codes and Raptor codes, which could be seen as ”foun-
tain” codes, are initially introduced for the purpose of scalable
and fault-tolerant distribution of data over computer networks.
However, the idea of infinite length is not suitable for many
other applications, such as broadcasting.
The Raptor-like LDPC codes are proposed by [14]. The
main differences between this new class of code and traditional
Raptor code is that it uses finite length LT-like code instead
of the LT code. And a high-rate LDPC is used as the precode.

III. THE PARTIALLY DECODING ALGORITHMS
The classic decoding algorithm for Raptor code could be
performed in two-stages: the LT code is first decoded and some
of the precoded symbols are recovered. Then the decoder will
try to recover the remaining symbols by decoding the precode
𝐶.
For the Raptor-like LDPC code, the decoding process also
includes two steps: BP decoding is first performed on the LT
code, and then on the LDPC part. For the initial algorithms
[15][16], the two-stage decodings use two different BP de-
coders, each exchanging their extrinsic information after one
iteration. In order to reduce system complexity, [14] considers
a joint decoding of the LT-like part and the LDPC part, which
guarantees that the BP algorithm always work for the high-rate
LDPC part as well as the whole codeword.
When see to the application of terrestrial broadcasting, we
find another decoding scheme which will be quite useful.
Assuming that the codeword of LDPC part is in the head of the
Raptor-like code as shown in Fig. 2, we will try to decode the
information bits as soon as we have received the LDPC part,
using traditional LDPC decoder. Once the decoding process
is successful, we stop the decoder. Otherwise, we will wait
for more check bits and try to decode the equivalent lower-
rate LDPC. This process will continue until the decoding is
successful or the entire Raptor-like LDPC codeword has been
received. A noticeable detail here is that we could use the soft
value of the last step every time we try anther round. The
whole decoding process could be represent by Algorithm 1.
Now we study two scenarios to explain why this partially
decoding scheme is useful for broadcasting applications.

1) Power Saving and Low Latency Mode for Terminals:
Assuming the case when someone is experiencing a certain
natural disaster. Let’s say a tornado. He is watching TV news
from the vehicular terminal. In order to cover as big as possible
areas, the TV station may use long error correction protection
in the transmitted signal. However, if the people is close to
the station which means he enjoys high signal to noise ratio
(SNR), his terminal will automatically shut down the decoding
modular after successful signal recovery using only the highest
rate “mother code”. This might enable his device to last longer.
Although this example seems to be extreme, we actually
could find some other similar situations. Because broadcasting
system is design for ”all” the people, it has to consider the
worst case, and eventually loss (waste of power, longer latency
etc.) of efficiency in better conditions. The use of Raptor-like
code and partially decoding scheme just provides the way of
distinguishing users by the adaptation in terminals.
2) Mobile Terminals under Single Frequency Network
(SFN) Conditions: In this case, assuming a car with a vehic-
ular TV terminal is moving across the border of the coverage
areas of two TV towers. The two TV towers are using the
same frequency and transmitting the same information with the
Raptor-like LDPC codes. Traditionally, this is a case when the
signal is easily to intermit, due to the strong multipath effect
(especially for systems using single carrier modulation such
as ATSC system in north America). However, when partially
decoding scheme is used, there are many chances of enabling
reliable reception in this case.
As shown in Fig. 3, if the SNR is sufficiently high, the
receiver could successfully get the message right after the
LDPC code part from TX1 arrives, before it crosses the border.
The worst case is that it can not complete decoding only using
the LDPC code part, and it misses some information when it
crosses the border, but it could continue to receive incremental
check bits from TX2 after it crosses the border and uses the
collected information to recover the transmitted signal.
This is basically the idea of network coding, but we find
the specific scheme to apply this idea in the broadcasting
applications.

V. CONCLUSION AND FUTURE WORKS
This paper proposes a partially decoding scheme for the
rate compatible Raptor-like LDPC codes. We explain the
reasons that this scheme is useful for broadcasting appli-
cations, by two different scenarios. The proposed decoding
scheme, combined with the Raptor-like LDPC codes has great
potential of providing more robust, power saving solutions
for the future terrestrial broadcasting systems. Moreover, the
simulation results of the decoding performance demonstrates
the proposed algorithm is effective.
Although the class of Raptor-like LDPC codes have been
proved to be able to have very good performance. It is still
possible to further improve the design criterion to make it
closer to the Shannon limit. Moreover, the combination with
other schemes are also attractive topics. For example, [19]
proposes a LDPC-RS product code for Mobile DTV system.
Using Raptor-like LDPC as substitute for traditional LDPC is
also an interesting direction for future research.
