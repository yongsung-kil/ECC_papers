Simulation and Implementation of LDPC Code in FPGA

Pavel Straus, Zdenek Kolka
Department of Radio Electronics Brno University of Technology
Brno, Czech Republic xstrau00@stud.feec.vutbr.cz, kolka@feec.vutbr.cz

Abstract—The paper deals with implementation of Low-Density Parity-Check (LDPC) codes [1] in FPGA-based bridge for FreeSpace Optical link. The coder was designed with a regular parity matrix for code rate 1/2. The matrix of dimension 8x16 for the experimental implementation was found using a random search in MATLAB. The main advantage of this matrix is the decoder can correct all single-bit errors. The simulation for all possible values shows that Bit Error Ratio (BER) is zero. This result was not obtained with other matrices. An experimental communication channel was realized with encoder and decoder implemented in FPGA Virtex 5 development board ML505. DIP switches are sources for information bits and these values are shown on LCD display. The bit-flipping method is used in decoder and result code word is shown in the second line on the LCD display.
Keywords— LDPC, Encoder, Decoder, Bit-Flipping
I. INTRODUCTION
Low density parity check codes are linear block codes using generator matrix G in an encoder and parity check matrix H in a decoder [1]. The advantage of LDPC codes is their error-correcting performance. This is a reason why LDPC codes have been selected for many applications [8]. Matrices H and G are orthogonal and the parity check matrix has M rows and N columns, where M represents check nodes and N represents variable nodes. This matrix is based on random construction techniques. The other way how to create H is based on Euclidean geometry and can be found, for example, in [6]. Information bits are based on check nodes and bits in the codeword are based on variable nodes. Matrix H contains only a few ones and a lot of zeros. If there is the same count of ones in columns and different count in rows, the matrix is called regular. Otherwise the parity check matrix is irregular. The structure of the parity check matrix is important for BER, where some matrices have better results than others. For example in [11], the random regular matrix H with column weight more than 3 may have asymptotically good results. This paper examines a low-size parity check matrix, which is regular and for which BER is zero for all single-bit errors in the codeword. The error-correcting performance has been demonstrated through MATLAB simulations. This paper studies design of an LDPC encoder and decoder in FPGA, which will be used in a network bridge for Free-Scape Optical or hybrid links [12]. This is the reason why encoder and decoder are implemented in FPGA device. This paper is organized as follows. Section II presents an introduction to

LDPC codes and shows an algorithm for creating and decoding a codeword. Section III present practical implementation of LDPC code, where simulation in MATLAB and ISE software has been done and realization on the development board ML505 with FPGA Virtex 5 is shown.

II. LDPC ALGORITHM A code word c is generated as

c= kG,

(1)

where k is the vector of information bits and G is the generator matrix. A valid codeword can be verified using

HcT= 0,

(2)

Where H is the parity check matrix. If the result in (2) is nonzero, the codeword c is invalid and an error correction procedure should be used in this case. The Bit flipping method uses a vector, called syndrome [4], which is computed as

s = HyT,

(3)

where y is the invalid codeword. The syndrome indicates which row in the H is not zeroed by vector y and some bits have to be repaired in the decoder [4].If the parity check matrix has low size, we can find an error floor of the LDPC code, where one erroneous bit is repaired and BER is close to zero or is zero. The bit flipping method is used in the decoder [4]. The advantage of LDPC codes lies in their performance, where the capacity is close to the Shannon capacity.
The first step in code design consists is creating a low density parity check matrix in MATLAB for code rate 1/2. First, random distribution is used for creating matrix H. The next step removes short cycles for better efficiency [10]. This matrix has to have a full rank, i.e. to have linearly independent rows. Then, matrix H is converted to the form H = [PTI] and generator matrix G may be created.

C. Simulation in MATLAB
The simulation for all possible input data vectors k was done in MATLAB. The vector k was realized as a binary counter with range from 0 to 255. Bit-flipping iteration method is used in decoder, where the syndrome is computed each iteration. The main idea of this decoding scheme is in(2). If the codeword is valid, the syndrome is zero. If the syndrome is not zero, it is used for computation of a new vector and this vector shows the position of error bit. This is repeated until the syndrome is zero or the maximum number of iterations has been reached. For every codeword, each bit is changed to simulate any possible single-bit error and this sequence is the input codeword to the decoder [4].
Packet error was simulated in MATLAB for Poisson distribution of errors, where the error probability was varied from 10-5 to 0.5. The result for 10000 random data vectors in each step is shown in Fig. 5.

D. Simulation in ISE FPGA Virtex-5 is installed on the development board
ML505, where LDPC transceiver is implemented. The IP core “Memory Block Generator” is used for matrix coefficients. This IP core is included in LDPC_encoder16bits module and LDPC_decoder16bits. The encoder is implemented like a state machine, where four states are implemented. The codeword is created from input information bits after 35 period of clock signal. If the clock period is 8 ns (125 MHz), the bit rate will be 57.14 Mbps and the frame rate 3.57Mfps [frames per second]. Bit flipping is used in the decoder, where six-state machine is implemented. The bit flipping is an iterative decoding algorithm, where syndrome is computed each iteration. The number of iteration actually performed is a random. The maximum number of iterations corresponds to the number of columns. This is the reason why time for decoding varies. The minimum number of clock periods is 22 for decoding a single error in codeword. If the clock period is 8 ns, the minimum time for decoding will be 176 ns. The decoder may decode 5681818 codewords per second with single error. If the received data will be without errors, the decoding requires 11 periods of clock signal and the decoding rate will increase to the 11363636 decoded words. The maximum number of iterations value is set to the number of columns of the matrix H. The unsuccessful decoding takes 177 periods of the clock signal in this case (1.416 us). After that, the state machine goes to the second state StSetAdr, where LUT address is incremented.
