Abstract—LDPC (Low Density Parity Check) is widely used in many telecommunication systems due to its excellent performance. However, real-time LDPC decoder in virtual radio system is difficult to realize. Taking CMMB (China Mobile Multimedia Broadcasting) for instance, this paper proposes a method to achieve high throughput decoder based on x86 processors which support SIMD (Single Instruction Multiple Data) instructions. By utilizing Normalized Min Sum decoding algorithm, normalized parameter as well as bit width of input variables and intermediate variables are determined. Then taking advantages of SIMD instructions, updating progress of variable nodes and check nodes in LDPC decoding algorithm is parallelized. Meanwhile, memory access operations are optimized as well. Tested on Intel Core i73960X, the throughput of the LDPC decoder using multithread processing can reach 92Mbps ∼ 722Mbps. Keywords—Virtual Radio; CMMB; LDPC; SIMD; fixed-point; Multithreading
I. INTRODUCTION
Virtual radio[1], which is a kind of software radio[2], based on general purpose computer was proposed by V.Bose in 1999. Compared with software radio which is based on special-purpose programmable devices, virtual radio own the advantages of lower cost, shorter-developing-period, easier update and more flexible configuration. It can support multiple wireless networks and, what’s more, can support more different networks in the future. In recent years, structuring practical virtual radio system has attracted more and more attentions due to the swift development of CPU technology. In 2009, Microsoft developed a virtual radio system based on PC, which is called “Sora”[3]. It utilized ordinary commercial PC, general RF front-end and kernel drivers to realize physical layer protocol and MAC layer protocol of IEEE 802.11a/b/g. In addition, Intel and IBM also developed a wireless communication system based on general purpose PC which realized LTE and WiMax. The project, implementing virtual radio system on general purpose portable computers, was supported

by the Science and Technology Commission of Shanghai Municipality [Project Number: 10DZ1500700]. It contains kernel protocol programs and multiple antennas RF front-end, aiming at being applied to wireless emergency communications and digital multimedia broadcasting networks. This research is based on this project.
The implementation and optimization of physical layer algorithms on general purpose computers is the key of virtual radio system due to the digital signal processing algorithms and coding/decoding algorithms of error correcting code in the physical layer own the features of large computations and real time processing. As the decoding algorithm is the main part of physical layer algorithms, the realization of real-time decoding is the vital matter in virtual radio.
Low Density Parity Check (LDPC)[4] code is a kind of linear block code. Due to its advantages like Shannon-limitapproaching performance[5], sparse check matrix and lowcomplexity decoding algorithm, it is widely used in many communication systems, like IEEE 802.11n/802.16e, DVBs2, CMMB, etc. From the very beginning of the rediscovery of LDPC, it enjoys a tremendous increasing and many efforts have been carried out in LDPC decoding algorithm and the design of decoders. Most of decoders are realized using FPGA/VLSI as semi-parallel or parallel structure[6][7], among which the highest rate can reach the level of Giga bps. However, these systems, using hardware description languages like VHDL and Verilog, are complex in structure and high in cost. Some systems are realized on GPU[8][9] and IBM CELL Broadband Engine (BE)[10] ,which maximum throughput is 160Mbps while the power consuming can be as high as 300W500W, which makes it too difficult to realize on portable terminal. There are also other approach like software-based LDPC decoder. But its throughput is too low to meet the requirements of communication systems.

II. REALIZATION OF LDPC DECODING A. Theory of LDPC Decoding Algorithm
First, classical LDPC decoding algorithm is an iterative decoding algorithm based on belief propagation (BP algorithm), also called sum-product algorithm[12]. The BP algorithm is a decoding method based on the Tanner graph, which contains Variable Node(VN) and Check Node(CN). As an example of LDPC with rate 1/2 in CMMB, Tanner graph of first three CNs is shown in Fig.2.
The BP algorithm is consisted of four parts: 1)Initialization of VN messages; 2)Update CN messages in iterative process; 3)Update VN and hard decision messages in iterative process; 4)Determining the next iteration according to hard decision messages. In each iteration, CN messages are updated by connected VN messages. Then, these updated CN messages are used to update VN messages as well as VN hard decision messages. After these operations, hard decision will be conducted to determine whether the entire information sequence is decoded. In the following discussion, LDPC sequence with rate 1/2 is modulated by BPSK and transmitted over AWGN channel. Parameter α in NMS algorithm is set to 0.75 for convenience in fixed-point simulation while it is 0.77 in floating-point simulation. As the result, a fixed-point data multiply by 0.75 is equal to itself subtracting its two-bit right shift.
In the decoding algorithm, input data LLR, intermediate variables Lq and Lr need to be converted to fixed-point. There are 3 steps in fixed-point operation. Firstly determine dynamic range and precision of LLR to select bit width. The precision is the noise range allowed over LLR (performance loss within 0.05-0.1dB). Secondly observe the influence on error performance resulting from saturated position to confirm bit width further. Saturated position is the amplitude limitation of LLR. For example, if saturated bit width is 4, the range of signed data is -7∼+7. Lastly, observe decoding process to confirm intermediate variables bit width, meanwhile, guarantee no overflow occurred.
According to a series of simulations, when the precision is 0.5 and BER = 10−5, performance loss between fixed-point data and floating-point data is about 0.05dB. When saturated bit width is 4, the performance loss is negligible. Thus, in this paper, precision is selected as 0.5 while the bit width is 4 (1 sign bit and 3 information bits). For convenience of programming, one LLR can be stored as a char type data.
When updating VN messages Lq, add operations are performed on intermediate variable Lr temp and the add operations may lead to overflow. According to simulation results, it is found that the iteration always end before overflow occurred. Hence, within the reasonable SNR (1∼10dB), no overflow is observed.
Fig.4 demonstrates the BER performance of both floatingpoint and fixed-point decoding algorithm with rate 1/2 and rate 3/4 in CMMB. (10 different SNRs, 10000 frames data in each SNR, 10 iterations for each frame.) It shows that the performance of fixedpoint decoding algorithm is 0.2dB less than that of floatingpoint decoding algorithm, that is to say, the conditon of fixed-point is fulfilled.
III. SIMD INSTRUCTION OF LDPC DECODING
Intel x86, x86-64 and IA-64 processors are equipped with Single Instruction Multiple Data (SIMD) technology[14][15], which can improve the efficiency by processing multiple data parallel in the same time. SIMD, including MMX, SSE(Streaming SIMD Extensions), SSE, SSE2, SSE3, etc, can process 64bit integer, 128bit integer as well as double precision floating-point number. These 64bit and 128bit data can also be the combination of multiple 8bit, 16bit and 32bit data.
In NMS algorithm, most of the time are occupied by two operations, namely operation No.6 and No.11 in Pseudocode Of NMS Algorithm. However, these operations can be optimized by utilizing SIMD technology. By storing data as m64 type or m128i type, SIMD instructions can be applied to boost the computation.

IV. DATA LEVEL PARALLELISM
All the previous discussions are based on the assumption that the CPU owns only one core. For multi-core CPU, multithread data level decoding can be applied. Assuming T threads are available, all LLR can be assigned to T independent decoders and the T results will be synthesized to get the final result. This procedure is shown in Fig.6, which make the throughput boost a lot comparing to only one CPU core.
In CMMB standard, the bandwidth of broadcast channel is 8MHz. By applying 16QAM modulation, during one time slot, 552960 LLR can be acquired from received data after deinterleave and demodulation, which requires p = 552960/9216 = 60 times decoding operations. If all these 60 groups LLR are transmitted to LLR FIFO and assigned to T independent threads, for example T=12, 12 groups LLR can be parallel processed in one call. All of these data will be decoded after 5 calls, instead of 60 calls.

VI. CONCLUSION
In this article, a LDPC decoding algorithm based on general purpose processor architecture is analyzed, simulated and optimized. By taking advantages of SIMD instructions and multi-thread technology, the decoding program can reach the throughput of 92∼722Mbps (varying from different SNR). Throughput of optimized program achieved the requirements of CMMB, which enables the system to perform real time decoding on virtual radio.
