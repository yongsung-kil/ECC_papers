Lowering Error Floors of Systematic LDPC Codes Using Data Shortening
Sung-Rae Kim, Student Member, IEEE, and Dong-Joon Shin, Senior Member, IEEE

Abstract—In this paper, data shortening methods for lowering error floors of systematic LDPC codes are proposed. Rather than attempting to analyze trapping (or stopping) sets of a given LDPC code rigorously, we search information bits associated with dominant trapping (or stopping) sets of systematic LDPC codes through simulation under various channels. Then, proper information bits forming dominant trapping (or stopping) sets are selected and known values are assigned to them before encoding to weaken the effect of dominant trapping (or stopping) sets. To decode codewords, fixed known values are assigned to the selected information bits, which gives rise to the disconnection of some edges in dominant trapping (or stopping) sets. Through simulation, it is shown that the proposed schemes result in remarkably better performance, especially at the error floor region, than the base LDPC codes under various channels with negligible loss of code rate.
Index Terms—Data shortening, error floor, stopping sets, systematic LDPC codes, trapping sets.
I. INTRODUCTION
L OW-density parity-check (LDPC) codes using iterative message-passing decoding have shown capacityapproaching performance for the binary-input additive white Gaussian noise (BIAWGN) channel [1], [2]. For finite-length LDPC codes, our understanding of the dynamics of iterative decoding algorithms is limited. In particular, iteratively decoded finite-length LDPC codes demonstrate an abrupt change in their error rate curves in the high signal-to-noise ratio (SNR) region, which is referred to as error floor. For binary erasure channel (BEC), error floor is well understood and known to be caused by particular graphical structures of LDPC codes, called stopping sets [3]. For the binary symmetric channel (BSC) and BIAWGN channel, the relationship of the error rate performance of LDPC codes and their graphical structures, called trapping sets, has been investigated and techniques to estimate the error floor were devised [4], [5]. In [6], the author provides a method to find all the dominant trapping sets in an efficient manner. Also many decoding-based strategies have been proposed to break trapping (or stopping) sets. In [7], they proposed a method to identify information bits of dominant trapping (or stopping) sets with lower complexity than the Monte Carlo simulation approach. Small trapping (or stopping) sets which are main contributors to error floor are called dominant trapping (or stopping) sets.
In recent years, data shortening schemes on array codes and irregular quasi-cyclic (QC) LDPC codes were proposed for obtaining codes of shorter length and lower rate from a given

In this paper, decoding-based design methods of systematic LDPC codes are proposed to lower error floor. Rather than attempting to analyze trapping (or stopping) sets of a given LDPC code as in [6], we search information bits associated with dominant trapping (or stopping) sets through Monte Carlo simulation under various channels as in [4]. To weaken the effect of dominant trapping (or stopping) sets of a given systematic LDPC code under various channels such as the BSC, BEC, and BIAWGN channel, Monte Carlo decoding simulation is first performed to estimate error floor and to count the number of errors at each information bit to estimate trapping (or stopping) sets of a given systematic LDPC code. Based on these simulation results, two bit-selection algorithms are proposed to select information bits associated with dominant trapping (or stopping) sets. Then, known values (e.g. 0) are assigned to those information bits when encoding is performed. Since selected information bits do not have to be transmitted, this is similar to the shortening of codeword and therefore this operation is called data shortening. Since a receiver knows the values of those bits, fixed values are assigned to them for message-passing decoding. The proposed data shortening schemes using two bit-selection algorithms weaken the effect of dominant trapping (or stopping) sets and improve the error-floor performance.
Especially, Gallager-B algorithm [2] is used for the BSC to demonstrate that our proposed schemes work well with hard decision decoding and the sum-product decoding [2] is used for the BEC and BIAWGN channel. For simplicity, we consider regular repeat-accumulated (RA) codes [10] as systematic LDPC codes to demonstrate the effectiveness of the proposed data shortening schemes.
The rest of the paper is organized as follows. Section II briefly introduces trapping (or stopping) sets and data shortening schemes based on two bit-selection algorithms are proposed. In Section III, simulation results under various channels are provided to confirm the validity of the proposed schemes and concluding remarks are given in Section IV.
II. DATA SHORTENING SCHEMES FOR SYSTEMATIC LDPC CODES TO LOWER ERROR FLOORS
A. Trapping Sets and Stopping Sets
A parity-check matrix H of an LDPC code can be represented by a bipartite Tanner graph G = (V ∪ C, E), where V is a set of variable nodes, C is a set of check nodes and E is a set of edges (i, j) connecting check node i and variable node j if and only if Hi,j = 1. IV. CONCLUSIONS
In this paper, simulation-based error-floor lowering schemes for systematic LDPC codes were proposed. Under various channels, information bits forming dominant trapping (or stopping) sets of systematic LDPC codes are selected by two proposed bit-selection algorithms and, by assigning known values to those bits, the decoding performance can be remarkably improved. Since the proposed methods can be easily applied to any systematic LDPC codes and achieve good performance, they can be practically used in any communication systems with negligible loss of code rate.
Finally, we want to note that it may be possible to establish a stronger and more direct connection between the proposed shortening techniques and dominant trapping set deactivation by using trapping set enumeration techniques [6].
