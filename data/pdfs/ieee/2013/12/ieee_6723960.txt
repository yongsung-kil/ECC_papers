Effects of Base Matrices on Iterative Decoding
Performance of Irregular QC-LDPC Codes
Sheng Tong, Qinghua Guo, Jiangtao Xi, and Yanguang Yu
School of Electrical, Computer, and Telecommunication Engineering
University of Wollongong, Australia
{sheng, qguo, jiangtao, yanguang yu}@uow.edu.au
Abstract—The edge connections in factor graphs of quasi-
cyclic (QC) LDPC codes, which affect their iterative decoding
performance, are determined by their base matrices. In this
paper, the effects of base matrices on the iterative decoding
performance of QC-LDPC codes are investigated by using the re-
cently proposed EXIT chart tool for protograph LDPC codes. Ex-
tensive experiments have been carried out and iterative decoding
thresholds of QC-LDPC codes with the same degree distribution
pair but base matrices having different structures and different
dimensions have been obtained and compared. Experimental
results show that 1) the base matrix structure affects not only the
waterfall performance but also the error floor performance; base
matrices having good waterfall performance may exhibit rather
poor error floor performance. 2) by increasing the dimensions
of base matrices for a given degree distribution, the threshold
means are reduced while the ranges of threshold distributions
are increased. 3) similar to the case of detailed represented
LDPC codes, base matrices with sufficiently large dimensions may
have thresholds surpassing the conventional ensemble threshold
with the same degree distribution pair. Moreover, as a concrete
example, an improved base matrix is designed for a rate-1/2
WiMax LDPC code.
I.
INTRODUCTION
Recently,
quasi-cyclic
low-density
parity-check
(QC
LDPC) codes (see [1], [2], [3] and the references therein)
have received increasing research interest and been adopted
in several industrial standards such as the WiMax standard
[4]. There are several reasons for this phenomenon. Com-
pared to general LDPC codes, QC-LDPC codes facilitate low-
complexity implementations for both encoders and decoders.
Moreover, well-designed QC-LDPC codes perform as well as
randomly constructed ones.
Due to the aforementioned advantages of QC-LDPC codes,
significant efforts have been made to the analysis and design
of QC-LDPC codes. Experimental results show that irregular
QC-LDPC codes with the same degree distribution pair but
different base matrices may have different performances in
the waterfall region. This observation suggests that except the
degree distribution pair, the base matrix structure, or more
precisely the connection characteristics of the factor graphs,
also affect the convergence behavior of iterative decoding of
irregular QC-LDPC codes. However, the conventional density
evolution (DE) tool [5] is not suitable for analyzing this
effect. To solve this problem, an extension to the EXIT chart
tool [6], termed as protograh-based EXIT (PEXIT) charts,
is developed in [7] for the convergence behavior analysis of
protogaph LDPC codes [8], which subsumes QC-LDPC codes
as a subclass.
With the PEXIT chart, this paper investigates the effects
of base matrices on the iterative decoding performance of
QC-LDPC codes and some useful observations are provided.
The rest of the paper is organized as follows. Section 2
briefly reviews the concepts of QC-LDPC codes, including the
definitions of base model matrix and base matrix. Moreover,
the failure of DE in analyzing the convergence behaviors
of QC-LDPC codes is explained and then the basic idea
of the PEXIT chart tool is briefly reviewed. In Section 3,
extensive experiments are carried out to generate threshold
distributions of QC-LDPC codes with base matrices having
different structures and different dimensions and some useful
observations are obtained. In addition, as a concrete example,
improved base matrix design for a rate-1/2 WiMax LDPC
code has been tried. Conclusion and future works are given
in Section 4.

B. Failure of DE to Protograph LDPC Codes and Protograph
EXIT Charts [7]
Iterative decoding of an LDPC code can be described in a
sparse bipartite graph that consists of a set of variable nodes
(VNs), an edge interleaver, and a set of check nodes (CNs).

The VNs (resp. CNs) represent the coded bits (resp. parity
check constraints). Iterative decoding is performed by passing
extrinsic messages between VNs and CNs along the edges.
DE [5] is a useful tool for analyzing the convergence behavior
of the iterative decoding by tracking the probability density
functions (pdfs) of extrinsic messages.
The original DE tool is not well suitable for analyzing
the convergence behavior of QC-LDPC codes mainly for two
reasons. First, the original DE does not take into consideration
the connectivity between VNs and CNs, or equivalently the
structure of H𝑏. This is because that the original DE takes
a degree distribution pair as its input, which is not sufficient
to describe the connectivity between VNs and CNs. Second,
the pdfs of extrinsic messages from CNs to VNs are firstly
averaged according to the degree distribution pair (see eqn.s
(7) and (8) in [5]). Then, all the incoming messages along
the edges of degree-𝑖VNs follow the same density and so
do the outgoing messages from degree-𝑖VNs. While, for
irregular QC-LDPC codes, due to its asymmetric connectivity,
the messages traversing along different edges of degree-𝑖VNs
(resp. CNs) may exhibit different pdfs.
The detailed represented LDPC codes in [9] to some extent
take the connectivity between VNs and CNs into consideration.
However, these codes only allow all degree-𝑖VNs to have
the same connectivity with CNs, while in irregular QC-LDPC
codes, two classes of degree-𝑖VNs corresponding to two
different columns of H𝑏may have different connectivities with
CNs. Moreover, in [9], the pdf-averaging operation is also
taken for incoming messages to VNs (or CNs) with a specific
degree.
Hence, it is necessary to develop an extended version of
DE for convergence analysis of irregular QC-LDPC codes. An
implementation of the extended DE can be briefly described
as follows. Draw a bipartite graph 𝐺𝑏according to H𝑏. Then,
𝐺𝑏fully describes the connectivity between VNs and CNs.
We shall associate each edge in 𝐺𝑏with two pdfs, one for the
extrinsic message from VNs to CNs and the other for that from
CNs to VNs. This is the major difference between the extended
DE and the original DE. Then, after initializing the densities
from the channel, we can track the evolution of the densities
along the edges and thus predict the convergence behavior of
irregular QC-LDPC codes.
To simplify the extended DE, PEXIT chart is developed
in [7] by tracking the evolution of the mutual information
related to the extrinsic messages rather than their true densities.

A. The Effects of Base Matrices on the Code Performance
To investigate the effects of base matrices on the iterative
decoding performance of QC-LDPC codes, we choose the fol-
lowing node perspective degree distribution pair and tried base
matrices with different structures and different dimensions.

increasing the dimensions to 12 × 24 and 18 × 36, the number

two dimension settings.
The threshold distributions of the three dimension settings,
obtained by using the PEXIT chart, are compared and shown
in Fig. 1. From Fig. 1, it is seen that 1) with the increase of
base matrix dimensions, the threshold distributions are shifted
leftwards. In fact, the threshold means of the three dimension
settings are 0.7908dB, 0.7782dB, and 0.7598dB, respectively.
This means that increasing base matrix dimensions leads to
better performance from a statistical point of view. Note that
these threshold means are much worse than the ensemble
threshold, 0.641dB. 2) increasing base matrix dimensions also
leads to large distribution ranges and better optimal threshold.
3) in the 18 × 36 dimension setting, the optimal threshold
is 𝐸𝑏/𝑁0 = 0.609dB, which is better than the ensemble
threshold, 0.641dB. A similar result is observed in the detailed
represented LDPC codes.
According to the above observations, we know that mixing
different types of edges as uniformly as possible (as what
have been done in conventional DE analysis) in constructing
factor graphs for LDPC codes is not a bad choice since
threshold means of structured codes are worse than ensemble
threshold; slight coding gains can be obtained by carefully
taking connectivity of factor graphs into account.
B. Improved Design of Base Matrices for a Rate-1/2 WiMax
LDPC Code
As a concrete example, we attempted to design an im-
proved base matrix for a rate-1/2 WiMax LDPC codes. All
rate-1/2 WiMax LDPC codes share the same base matrix H𝑏
of dimension 12 × 24 that also specifies the employed degree
distribution pair [4]. There are 19 block lengths and the longest
one is 2304 with an expansion factor of 𝑧0 = 96. H𝑏is of an
almost lower triangular form such that they allow an efficient
encoding algorithm. Using the same degree distribution as the
rate-1/2 WiMax LDPC codes, we randomly constructed 104
different base matrices and determine their thresholds by using
PEXIT chart. The distribution of the thresholds is shown in
Fig. 2. The gap of the best and worst thresholds is about
0.14dB, indicating that different base matrices may lead to
significantly different thresholds.
There is only one base matrix with the optimal threshold of
𝐸𝑏/𝑁0 = 0.681dB, while the base matrix of the WiMax LDPC
code has a threshold of 𝐸𝑏/𝑁0 = 0.816dB, which is very close
to the worst one, 0.821dB, in Fig. 2 We construct two (2304,
1152) codes, denoted as 𝐶1 and 𝐶2, using the base matrix
with the optimal threshold and one with threshold 𝐸𝑏/𝑁0 =
0.708dB, respectively. Fig. 3 compares the BER performance
of 𝐶1, 𝐶2 and the (2304, 1152) WiMax code. The maximum
iteration number is 80. We observe that 𝐶1 outperforms 𝐶2
and the WiMax code in the waterfall region due to its better
threshold, while 𝐶1 shows a much higher error floor than the
other two codes. This means that threshold is not the only
criterion in choosing good base matrix and we should also take
the error floor performance into consideration. In this aspect,
the base matrix for 𝐶2 is a better choice than the one for 𝐶1.

The design of base matrices that perform well in both
the waterfall and error floor regions is an interesting topic.
The available methods in literature for constructing bipartite
graphs that perform well in the error floor region, e.g., the ACE
method in [10], can be adapted to the design of base matrices
with good error floor performance. A practical approach is to
crack the problem into two steps: obtain a set of base matrices
with good waterfall performance and then from this set choose
base matrices with good error floor performance.
IV.
CONCLUSION AND FUTURE WORKS
With the help PEXIT chart, the threshold distributions of
QC-LDPC codes with base matrices having different structures
and different dimensions are generated. From these threshold
distributions, we observe that on average structured QC-LDPC
codes exhibit worse thresholds than randomly constructed
ones. However, by choosing base matrices with sufficient large
dimensions and carefully selecting a good base matrix, a slight
coding gain in waterfall performance may be obtained with a

risk of a poor error floor.
Interesting future works include developing an efficient
method for enumerating all base matrices satisfying a given
degree distribution pair. This work would help us to find the
base matrix with the best threshold. However, as pointed out
above, such a base matrix may exhibit a poor error floor
performance. Hence, a more meaningful work is to develop
an approach to constructing base matrices, which can provide
a good tradeoff between waterfall and error floor performance.
