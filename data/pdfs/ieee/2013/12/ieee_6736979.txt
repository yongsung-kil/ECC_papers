Low Overhead Correction Scheme for Unreliable LDPC Buffering

Amr M. A. Hussien, Wael M. Elsharkasy , Ahmed M. Eltawil, Fadi Kurdahi, and Amin Khajeh∗ Electrical Engineering and Computer Science Department, University of California, Irvine, Irvine, CA ∗Intel Labs, Hillsboro, Oregon, USA Email: {ahussien, wael.sharkasy, aeltawil, kurdahi} @uci.edu - {amin.khajeh} @intel.com

Abstract—Aggressive voltage overscaling (VOS) has been recently adopted to reduce the power consumption of embedded memories. While this affects reliability, several techniques have been developed to achieve reliable operation under aggressive VOS. In this paper, we present a comprehensive powerperformance study of LDPC decoders enabling aggressive VOS to internal memories. We propose a correction scheme based on defect map insertion that maintains reliable LDPC decoder under aggressive VOS. The proposed scheme maintains the LDPC decoder performance while achieving a power saving of up to 21%.
I. INTRODUCTION
Dynamic voltage and frequency scaling (DVFS) [1] has been widely adopted in low power mobile applications to reduce both the dynamic and leakage components of power consumption. To improve the effectiveness of DVFS, aggressive voltage over-scaling (VOS) [2] has been proposed where the supply voltage is reduced beyond the safe limit, while running at the frequency used at nominal supply voltage, allowing some degree of controlled hardware errors to propagate through the system (as long as an overall quality of service (QoS) metric is achieved). This approach allows hardware error tolerance based on the measured Signal to Noise Ratio (SNR); i.e for high SNR, additional hardware error is tolerated at the benefit of additional power saving. As a consequence, several approaches considered hardware error at the algorithmic level with the target of achieving low power error resilient architectures. In [3], [4], several hardware error resilient forward error correction (FEC) decoders were proposed, where the decoding takes into account the statistics of the channel noise, as well as aggressive VOS based hardware error. In [5], Turbo equalization was investigated under unreliable buffering, and a fault tolerant Turbo equalizer was derived to compensate for the memory errors. In [6], an error-resilient low power Viterbi decoder is presented. It enables algorithmic noise tolerance (ANT) in the Add-Compare and Select Unit (ACSU). The critical path delay of the ACSU is shortened to operate at lower supply voltages without timing errors.
Among the FEC decodes, low-density parity check (LDPC) proposed by Gallager [7] has been widely studied due to the promising near-Shannon limit performance. Additionally, there are several recent studies on the performance of LDPC decoders under unreliable hardware. In [8], the capacity and density evolution of noisy LDPC decoders were computed. The authors in [9], [10] studied the Bit Error Rate (BER) performance of the noisy Gallager B decoder in terms of transmission errors and processing errors. In addition, they

studied the optimal distribution of the LDPC variable and check nodes among the existing processing resources to minimize the overall BER.
This paper studies the power-performance tradeoff of minsum LDPC decoding with noisy buffers. Moreover, a solution is proposed to recover from internal memory errors through insertion of defect maps. The defect maps retain the desired performance, while maintaining considerable power savings. The paper is organized as follows. Section II presents the LDPC system model with internal noisy buffers. Section IV introduces the error resilient LDPC decoders using defect maps. Section V shows the performance results, and section VI concludes the paper.

The transmitter output stream is subject to an additive white Gaussian noise (AWGN) channel. The received data is down-converted, digitized, and stored in the buffering memory. To reduce power consumption, the buffering memory is subject to spatially uniform distributed errors due to aggressive VOS. The buffering noise affects the statistics of the received data, resulting accordingly in a different distribution at the buffer output. IV. HARDWARE ERROR RESILIENT LDPC
This section proposes a defect-map based scheme to combat VoS induced memory errors. This scheme proves to be a good tradeoff between performance and power consumption. The scheme assumes that a built-in self test (BIST) is applied prior to decoding. Afterwards, information about buffering memory errors are stored inside the defect map. Two different architectures are explored;
1) Location-aware: In this scheme, the addresses of the faulty locations

are stored in the defect map, and the corresponding LLR values are neglected in the decoding process. 2) Location and pattern aware: In this scheme, the addresses of the faulty locations as well as the error patterns are stored in the defect map. Erroneous bits are corrected upon reading from the buffering memory. Clearly, this scheme requires more area for the defect map to store both addresses and error patterns.
Since the target LDPC code is represented by structured LDPC code, the parity check matrix is composed of submatrices, allowing processing of parallel sub-codes. In the proposed architecture, the channel memory is organized as 1 bank of size 2K locations. The incoming data stream is written to the channel memory, and is read in sequential order. Since initially the internal LLRs are zeros, the channel memory output is written to the variable node memory, where the variable to check LLRs are stored. The variable node memory is organized as a set of 12 parallel banks, where each bank has a size of 2K locations. The data is written in permuted order according to the permutation of each submatrix. The output is read in a sequential order to the CNP, where it computes the minimum and second minimum of each check node. The Check node memory is organized as 12 banks, each with a size of 128 locations, where each location stores the minimum and second minimum. The CNP output is written to the check node memory in sequential order, while it is read in a permuted order. The read access of the check node memory is done in parallel to the channel memory, where the two outputs are processed by the VNP for subsequent decoding iterations. Each of the channel memory, variable node memory, and check node memory are subject to aggressive VOS, and each one is associated with a defect map.
While the defect map plays a key role in LLR memory error recovery, there is an associated overhead in area, power, and latency. Since the channel and variable node memory read operation is performed sequentially, an SRAM-based implementation is well suited for the associated defect maps. The defect map outputs the next address with faulty data. The read address of the Channel and Variable to check memories is incremented in sequential order until it matches the defect map output. At this step, the defect map SRAM address is incremented to point to the next address with faulty data. For the Check to variable RAM, the reading operation is not sequential. Therefore, each address has to be searched within the defect MAP. As a consequence, content addressable memory (CAM) is used instead. The corresponding memory organization and operation is represented as follows
1) Location-aware Defect map memory organization: The SRAM based defect maps for the channel memory and variable node memory banks store the addresses with faulty data. The defect map update is associated with each supply voltage level transition. For the CAM based defect map, a search operation is associated with each Check node RAM access. In case of a hit, the Check node RAM output is neglected by considering a zero LLR in the subsequent computation.

VI. CONCLUSION
This paper proposed a power saving scheme for LDPC decoders through enabling aggressive VOS to its internal LLR buffers. To overcome the performance degradation, we

proposed defect map insertion to the channel and internal LLR buffers to retrieve the performance similar to the hardware error free system, while maintaining the power saving. We utilized SRAM based defect maps for the channel buffers and variable to check RAMs where the LLR read address increments in ascending order, and CAM based defect maps for the check to variable RAMs where the LLR read address increments in permuted order. The proposed scheme achieves a power saving of 21% to the LDPC decoder utilized in IEEE802.11ac standard, with the same performance as the one operating at nominal supply voltage.
