AbstractέΑΦ This paper proposed a new decoder architecture which employs HSS algorithm and simple check node update algorithm. Memory architecture is a major point of in terms of high speed and area of implementation, therefore the efficient memory design to accelerate decoding throughput is studied. In conventional decoder architectures, p- parallelism, dc-serialism, and q-serialism are required. This paper attempted memory division to obtain dc-parallelism and memory sharing to reduce area, 7 times of decoder throughput may be improved compared to conventional one.
KeywordsέΑΦ LDPC, HSS, Normalized Min Sum(NMS), decoding throughput, High throughput, dcparallelism.
I. INTRODUCTION
LDPC codes [1], proposed by Gallager in 1962, are adopted with a high-order modulation scheme as a powerful Forward Error Correcting (FEC) structure of DVB-S2. They have a remarkable Bit Error Rate (BER) performance that is close to the Shannon limit [2]. An LDPC code is defined by a sparse parity check matrix H, where N bit nodes and (N-K) check nodes are connected by edges. In the matrix H, the bit nodes/check nodes are mapped to the columns/rows, where one element in a row is connected to another in a column by an edge. Some researchers have already reported their LDPC decoder for DVB-S2. [3][4]. LDPC codes can be decoded by implementing/using a message passing algorithm such as

belief propagation[5], whereby messages transmitted iteratively between bit nodes and check nodes have probabilities corresponding to node values of 0 and 1.
Recently, many authors have addressed about algorithms and memory utilization for high throughput implementation. In aspect the algorithms, we employed horizontal shuffle scheduling (HSS) which updates check nodes and bit nodes simultaneously [6], and simple normalized based on check node processing (CNP) is introduced. The other issue focused in this paper is efficient memory utilization. Memory architecture is a major point of in terms of high speed and area of implementation, therefore the efficient memory design to accelerate decoding speed is studied. Well defined saturation and an optimum partitioning of memory bank lead to a significant reduction compare to the state-ofthe-art. Even if state-of-the-art of the decoder architecture satisfied required throughput of DVB-S2 standard, the search of an efficient trade-off between high throughput and area makes implementation of the LDPC decoder still a challenge. The designer has to deal with many possible choice algorithms, parallelisms and code rates.

B. Conventional Decoder Architecture
In order to understand the scheduling, we should remind the structure of the LDPC matrix for the DVB-S2 standard. Let us focus first on the rate 1/2 LDPC code. The prototype matrix is of size (90, 180) for the rate 1/2 LDPC. Each element of this prototype matrix is a positive integer (0, 1 or 2 for this code rate). The summation of each row gives a total of dc=7 non zero value, the summation of the first 36 first column gives 8, the summation of the next 54 columns gives 3 and the summation of the last 90 columns gives 2. The summation of the total nonzero-value gives E = 630 (180 ├Ω7 for a summation row by row, or 36├Ω8 + 54├Ω3 + 90 ├Ω2 for a summation columns by columns). Then each element of the prototype matrix is expanded as a P├ΩP matrix, where P = 360 to obtain the final (32400├Ω64800)
LDPC matrix. The zeros of the prototype matrix are extended to the null P├ΩP matrix, the one are extended to a shifted (P ├Ω P) Identity Matrix (IM), the two are extended to the superposition of two shifted (P├ΩP) IM, a
situation called Double Diagonal (DD). The decoder for LDPC codes of 1/2 code rate has p=360 parallelism. Sj memory buffers p received symbol per one address. Therefore address depth of Sj memory is N/p=180.

Therefore, p-parallelism, dc-serialism, and q=serialism are required. All the design (control, check node) is derived from the variable node architecture shown in Fig. 3. IV. CONCLUSIONS
This paper proposed a new decoder architecture which employs HSS algorithm and simple check node update algorithm. Memory architecture is a major point of in terms of high speed and area of implementation, therefore the efficient memory design to accelerate decoding speed is studied. In conventional decoder architectures, p- parallelism, dc-serialism, and q-serialism are required. This paper attempted memory division to obtain dc-parallelism and memory sharing to reduce area, 7 times of decoder throughput may be improved compared to conventional one.
