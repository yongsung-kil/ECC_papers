Security Gap Performance of Some LDPC Code Constructions
Nicola Maturo, Marco Baldi, Marco Bianchi, and Franco Chiaraluce

Abstract—In this paper, we assess the behavior of some lowdensity parity-check (LDPC) code design techniques over the additive white Gaussian (AWGN) wire-tap channel, in terms of security gap. We consider the progressive edge growth (PEG) construction algorithm as a reference, and compare its performance with that of four approaches for designing structured LDPC codes. We consider both systematic and non-systematic transmissions, and show that some structured code design techniques are able to achieve comparable or even better performance than the PEG algorithm over the considered channel, while taking advantage of their simpler encoding and decoding procedures.
Keywords—Physical layer security, AWGN wire-tap channel, LDPC codes, security gap.
I. INTRODUCTION
P Hysical layer security is a breakthrough in communications security paradigms, since it allows to achieve secure transmissions without the need of any form of pre-shared secret within the group of authorized users.
It allows to achieve a first level of security, only based on the difference between the channels of legitimate receivers and those of potential eavesdroppers, which can suffice by itself or constitute a basis for higher layer cryptographic protocols.
This setting is well represented by the simple wire-tap channel model [1], in which we have a transmitter (Alice) sending some information to a legitimate receiver (Bob), in the presence of an eavesdropper (Eve). The transmission technique used by Alice is perfectly known by both Bob and Eve. However, the channel between Alice and Bob is different from the channel between Alice and Eve; hence, only based on this difference, there can exist some secret information sent from Alice to Bob with no chance for Eve to retrieve it.
The maximum amount of secret information which can travel from Alice to Bob can be estimated through information theoretic arguments, which provide the capacity limits for this channel. However, finding practical transmission techniques which are actually able to approach these bounds is still an open challenge.
In this paper, we focus on the AWGN wire-tap channel, for which transmission security can be achieved only when Bob’s channel has higher signal-to-noise ratio (SNR) than Eve’s channel [2]. As a metric for physical layer security, we use a parameter which allows for a straightforward assessment and comparison of practical transmission schemes, based on

the error probability achieved by Bob and Eve. This parameter is the so called security gap, first introduced in [3]. It measures the difference between Bob’s and Eve’s channel qualities needed to achieve a sufficient level of security, while ensuring that Alice achieves reliable transmission towards Bob. Hence, a successful transmission technique should keep the security gap as small as possible, since this means to have security even with a small degradation of Eve’s channel with respect to Bob’s channel.
We focus on state-of-the-art LDPC codes, which are capacity achieving codes under belief propagation (BP) iterative decoding [4] and represent a near-optimum coding scheme for many purposes [5]. LDPC codes have recently found application in cryptography too, thanks to the revitalization of their use in the McEliece cryptosystem [6], [7].
When using LDPC codes over the AWGN wire-tap channel, a first issue is that using the traditional systematic transmission yields large security gaps. It has been shown that an effective way of reducing the security gap consists in using punctured LDPC codes [3], [8], [9]. However, using punctured codes produces an increase in the power consumption. We have recently shown that similar, and even larger, reductions in the security gap can be achieved by using scrambled transmissions, and with considerable power saving with respect to punctured codes [10]–[12]. A similar approach can also be used in the context of IEEE 802.11 networks, where traditional convolutional codes are used in the place of LDPC codes [13].
In this paper, we address the problem of optimizing the LDPC code design, with the aim of further reducing the security gap over the AWGN wire-tap channel. For this purpose, we consider several LDPC code construction techniques, and assess their performance when used in this setting. We evaluate the security gap, through numerical simulations, for each of the considered code design techniques, also in conjunction with information scrambling. This way, we obtain a clear indication on the type of code design technique that is preferable to use for reducing the security gap between Bob and Eve.
The organization of the paper is as follows. In Section II, we introduce the notation and we remind the definition of the security gap and of scrambled transmissions. In Section III, we describe the LDPC code design techniques we have considered, whose performances are compared in Section IV. Finally, Section V concludes the paper.

An important target from the physical layer security standpoint is to keep the security gap as small as possible. Since systematic transmission yields large security gaps, in [3], [8], [9] the use of punctured codes is proposed, by associating the secret bits to punctured bits. As shown in [10]–[12], an alternative solution is to scramble the information bits prior to encode them. This approach allows achieving similar or even better reductions in the security gap than puncturing; moreover, smaller signal power is required to achieve Bob’s reliability condition.
Scrambling can be applied to single frames. Better performance, however, can be achieved when L information vectors (L > 1) are concatenated together, and then multiplied by a kL × kL non-singular scrambling matrix. Then, the resulting vector is divided again into L subvectors, which are individually encoded and then transmitted. At the receiver side, after receiving the corresponding L codewords, and exploiting the channel code to correct the errors affecting them, the received information vectors are concatenated again and descrambled together through the inverse scrambling matrix. This way, the message transmitted by Alice is recovered only if all the errors affecting the L vectors are corrected. Otherwise, during descrambling, a single residual bit error can be spread over all L vectors, thus causing maximum uncertainty. Hence, only Bob, who is able to correct all the errors induced by the channel, can recover the message transmitted by Alice, while Eve has bit error probability close to 0.5.
As we have shown in [10]–[12], when the scrambling matrix is a random dense matrix, it is able approach the effect of a perfect scrambler, which ideally spreads the residual errors over the received vectors. In this case, the bit error probability equals half the frame error probability, since a frame which is received in error corresponds to an error probability equal to 0.5 on its bits, after descrambling. This effect is further emphasized when L > 1 since, in this case, a single frame received in error within a block of L frames implies that all the L frames are in error after descrambling.
III. LDPC CODE CONSTRUCTION TECHNIQUES
In this section we describe five LDPC code design techniques which we consider for assessing their performance in terms of security gap over the AWGN wire-tap channel.
We use as a benchmark the PEG code design technique [14], which is a well consolidated solution for designing very good LDPC codes. The PEG technique aims at maximizing the length of the local cycles in the Tanner graph associated with an LDPC code. This allows to achieve very good performance under BP decoding, which is strongly influenced by the Tanner graph properties. On the other hand, LDPC codes designed through PEG have parity-check matrices without any inner structure, which gives significant complexity when encoding and decoding must be implemented in hardware.
For this reason, several techniques have been proposed in the literature to design codes having structured parity-check matrices, which facilitate the hardware implementation. We consider four different methods to design structured LDPC codes, and assess their security gap performance in compar-

A. Quasi-cyclic codes
Quasi-cyclic (QC) LDPC codes are a class of codes able to join the excellent performance of LDPC iterative decoding with the low complexity encoding which characterizes QC codes. In fact, QC codes have the desirable property of being encodable through simple barrel shift registers, which have complexity increasing in the code redundancy. QC-LDPC codes have structured generator and parity-check matrices, which are formed by circulant blocks. Several ways of designing QC-LDPC codes are known in the literature. Two main classes are those using cyclic permutation matrices, like array LDPC codes [15], and those using general sparse circulant matrices.
The latter, in particular, contain a class of very simple codes which are able to achieve good performance, especially at high code rate. They are characterized by a parity check matrix in the form of a single row of circulant blocks [16]:

B. Serially concatenated codes
Another powerful, though simple, class of structured LDPC codes is represented by multiple serially-concatenated multiple-parity-check (M-SC-MPC) codes [18]. These codes exploit the serial concatenation of very simple components, similarly to what proposed in [19], where single parity-check component codes are used. By using different component codes, which however remain very simple, M-SC-MPC codes are described through sparse parity-check matrices which ensure good performance under LDPC decoding. More recently, some variants have also been proposed, which are able to further improve their performance [20], [21].
An M-SC-MPC code is obtained as the serial concatenation of M component codes, each with length ni, dimension ki and redundancy ri, i = 1, 2, . . . , M . The serial concatenation is systematic, hence each component code appends its ri redundancy bits to the input codeword. The i-th component MPC code computes its j-th redundancy bit as the EX-OR of the codeword bits whose indexes are smaller than j by an integer multiple of ri. It follows from its definition that an M-SC-MPC code has a lower triangular parity-check matrix with a simple inner structure. An example of such structure is given in Fig. 2, for a code with M = 3. The black lines denote symbols 1, while the other symbols are 0. The structured nature of M-SC-MPC codes makes their encoding very easy [18], and also the implementation of LDPC decoders may benefit by the form of their parity-check matrix.

C. Interleaved product codes It is known that a special structure of LDPC code can be
obtained in the form of a bi-dimensional product code using two smaller LDPC codes as components [22], [23].
However, the codes, so obtained, rarely exhibit good performance under LDPC decoding, due to the very regular structure of their Tanner graph. This drawback can be overcome by introducing an interleaver between the two component codes [24]. If the interleaver obeys the so-called row-column (RC) constraint, the interleaved product code still has a minimum distance equal to the product of the minimum distances of its components, and interleaving also helps reducing the number of minimum weight codewords. On the other hand, the use of an interleaver produces a marginal increase in complexity, while preserving the very simple encoding procedure which is typical of product codes.
D. Progressive differences convolutional codes Recently, an increasing interest has been devoted to LDPC
convolutional codes, which are able to join the advantages of convolutional codes with the very good performance of LDPC codes. In particular, convolutional codes have the advantage to be encodable through very simple circuits, and to allow decoding based on a sliding window, which does not need the whole stream to be received before starting to decode it.
A particularly interesting class of LDPC convolutional codes is that of time-invariant codes, which are able to achieve very good performance with limited complexity [25]. In this paper, we focus on the design technique we have recently proposed in [26], which is able to design codes, named progressive difference convolutional low-density parity-check (PDC-LDPC) codes, that are characterized by a very small constraint length. This permits us to design even rather short convolutional codes, and to easily terminate them without incurring significant losses in the code rate.

V. CONCLUSION
We have assessed the performance achievable by some LDPC code design techniques in terms of the security gap over the AWGN wire-tap channel. We have considered both systematic and scrambled transmissions, and frame concatenation before scrambling. Making reference to the PEG construction algorithm, we have compared its performance with that of four techniques for designing structured LDPC codes. Our results show that M-SC-MPC codes can achieve the best results, which are comparable and even better than those obtained through PEG. Also QC-LDPC codes achieve good performance, with a moderate loss with respect to PEG. Hence, we can say that M-SC-MPC and QC-LDPC codes allow to take advantage of the inner structure of their characteristic matrices, while achieving very good performance from the physical layer security standpoint. Future work will concern deepening of implementation issues, for the various solutions we have compared, as regards the hardware requirements.
