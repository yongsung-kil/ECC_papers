FPGA Based Hardware Architectures for Iterative Algorithms Implementations
Bogdan Belean, Monica Borda and Adrian Bot

Abstract— The paper describes the FPGA technology together with its possibility to exploit spatial and temporal parallelism in order to implement hardware architectures for iterative algorithms. The development of hardware architecture using FPGA technology represents a reliable solution in case of various applications where fast processing in case of iterative algorithms it’s mandatory. Two applications are presented where the FPGA technology is used for processing. Thus, on one hand, automatic microarray grid alignment is performed using FPGA based hardware architecture, while on the other hand, an FPGA based LDPC decoder implementation is proposed in order to improve the decoder throughput compared to state of the art approaches.
Keywords—FPGA technology, Low Density Parity Check Codes, image processing, iterative algorithms.
I. INTRODUCTION
FPGA technology uses pre-built logic blocks and programmable routing resources for configuration and for implementing custom hardware functionality. Their main benefits are the low cost, the short time to market and the ease of reconfiguration. They were introduced by Xilinx as a new concept which combines the user control and time to market of PLDs with the densities and cost benefits of gate arrays. FPGA can be defined as an integrated circuit which consists on a two dimensions network of cells or logic blocks. Each logic block can be programmed for implementing any logic function of its inputs. This leads to the user possibility to design, program and make changes to the developed application specific hardware architecture. The FPGA also adds design flexibility and adaptability with optimal device utilization while conserving both board space and system power, which is often not the case with DSP chips. When a design demands the use of a DSP, or time-to-market is critical, or design adaptability is crucial, then the FPGA may offer the best solution. Indeed, FPGA technology offers the possibility to exploit spatial and temporal parallelism in order to speed up computation. In case of temporal parallelism, pipeline approaches are used for implementation. The algorithms to be implemented using FPGA technology are divided in independent subsequent task. Thus, multiple tasks can be performed at the same time. Considering spatial parallelism, multiple operations such as additions, multiplications or others can be performed at the same time, compared with sequential processing specific to general purpose processors. An introduction on microarray technology is presented next, were fast and automatic processing of microarray images using FPGA based hardware architectures is a plus, considering the iterative nature of the image processing algorithms.
Microarray technology is based on creating cDNA microarrays which represents gene specific probes arrayed on a matrix such as a glass slide or microchip. Nowadays, the whole human genome can be printed on a microarray. The microarray scanners deliver high resolution microarray images (6000x2160 pixels) of the microarray glass slide in order to analyze and compare genes from normal cells with abnormal or treated cells. In this way, information for understanding the genes involved in different diseases is provided [1]. The main disadvantage in microarray image processing is user intervention and the iterative image processing algorithms applied on large input data (approx. 50 MB), which slow down the process of microarray analysis if a large number of subjects is involved. In this context, a microarray image processing system, robust and independent of operator last time adjustments, is needed, which includes hardware implementations for iterative image processing algorithms.
Low-density parity-check (LDPC) codes are forward errorcorrection codes, first proposed by Gallager in 1962. Due to their iterative nature and increased codeword (50 kb), the computational cost for the LDPC implementations is extremely high. Thus, LDPC codes were neglected for over 35 years. Once with the increased computational power offered by graphic processing units, FPGA/ASIC technologies and digital signal processors, LDPC codes are considered a significant breakthrough in the world of digital communications. Thus, an open subject in current research is to improve the LDPC decoder throughput. The state of the art solutions used to overcome the last mentioned disadvantage involve decoding algorithms parallelization using application specific hardware architectures.

II. ITERATIVE ALGORITHMS IMPLEMENTATIONS
A. Automatic grid alignment for microarray image processing
The current section describes an FPGA based hardware architecture which reduces the computational time for iterative algorithms used for automatic microarray grid alignment. Further on, the image processing algorithms used for automatic microarray image processing are presented, together with their parallelization levels. In case of the shock filters for grid alignment, the levels of parallelization are low due to their iterative nature. Thus, an FPGA based implementation is proposed to reduce computational time.
The flow of microarray image processing techniques is presented next. After image enhancement using logarithm transformation, vertical and horizontal projections are computed in order to estimate spot location and dimension using autocorrelation. Once the spot location is established, grid alignment using shock filters is applied [1]. Spot centers together with the edge detection algorithms are used for pValue estimation and spot intensity extraction.
Image processing operations like median filters and gradient calculation for edge detection are based on the convolution, which is included in a class of algorithms called spatial filters. Convolution is used for implementing image operators which have as output pixel value a linear combination between pixels of the original image. Conceptually, each pixel in the output image is produced by sliding an NxM window over the input image and computing an operation according to the input pixels under the window. The hardware approach for convolution is presented in [1] and [2]. The levels of parallelization for the previously described image processing algorithms are discussed next. In case of image enhancement, we consider M, N the image dimensions and p the number of logarithm computation units. Due to the independent computation of logarithm for each pixel, the maximum level of parallelization for image enhancement is (MxN)/p. For image profile computation, the level of parallelization is M+N. Shock filters are applied on image profiles. Due to the recursive description of the algorithms they cannot be easily parallelized. Once the spot locations are estimated, k being the number of spots filters like Sobel or Canny for image segmentation can be parallelized, and the maximum parallelization level is k. In order to estimate the computational time, the highest level of parallelization was taken into account. We consider the number of logarithm units p = 100 for an M x N = 6100x2160 pixels Agilent image. The number of microarray spots k is 22575. B. LDPC decoding algorithm
In the last decade, making use of the increased computational power offered by graphic processing units, FPGA/ASIC technologies and digital signal processors is overcomes the disadvantage of the increased computational power needed by the LDPC decoding process. Further on, a hardware architecture is proposed, which improves the LDPC decoder throughput in case of the WiMAX standard.
An N dimension LDPC code represent a linear block code whose codeword of length N satisfy a set of M linear paritycheck constraints and its code rate is R=1-M/N. LDPC codes are defined by the sparse parity-check matrix H(M,N) composed by 1 and 0 values, each of its line specifying one of the parity-check constraints. Thus, each codeword c satisfies the relation c*Ht=0 (Ht = H transposed). The LDPC decoding process is characterized by iteratively passing probabilistic messages between two types of nodes, M check-nodes (CN) and N variable nodes (VN) according to Tanner graph [4],[5].

Figure 4 - Message passing
Each type of nodes collects incoming messages from its complementary nodes and produces an outgoing message. The message qij is sent from VN to CN and represents the probability for the yi bit from the codeword to be 0 or 1. The CN sends the rij message back to the VN, indicating the probability for the yi codeword bit to be 0 or 1, considering the parity check constrains. Thus, the LDPC decoding determines the correct codeword based on the resulted yi bits, after performing iterative updates for VN and CN nodes. The decoding algorithm is presented in Fig.5.

The main disadvantage of LDPC codes is the increased computational time needed for the iterative decoding process. The state of the art solutions used to overcome the last mentioned disadvantage involve decoding algorithms parallelization using application specific hardware architectures. Existing approaches used to increase the decoder throughput can be summarized as follows:
 GPU (Graphic Processing Units) [6] [7] exploits data parallelism using multithread capabilities and parallel memory access producing performances for LDPC decoder implementations comparable with ASIC dedicated decoders.
 ASIC/FPGA technology is used to develop application specific hardware architectures and processing elements in order to parallelize LDPC decoding process [8].
 The LDPC decoding process is parallelized using Application Specific Instruction Processors [9] [10] or RISC processors [11]. This approach involves using Single Instruction Multiple Data microprocessor architectures in order to realize parallel update for check nodes CN and variable nodes VN.

III. CONCLUSIONS
The present paper presents two FPGA based hardware implementation for iterative algorithms used in image processing and LDPC decoding process. Thus, hardware implementations for microarray image processing algorithms which take advantage of the FPGA technology features lead to an automated system for fast microarray image processing. Thus, a large number of microarray analyses can be performed per time unit, compared with the existing tools. In case of LDPC decoder implementation, the use of FPGA technology can lead to an improvement regarding the decoder throughput, considering the level of parallelism depending exclusively on the FPGA resources.
The experimental results based on algorithms parallelization proved that FPGA technology is an efficient solution for iterative algorithms implementations in case of high performance computing applications.
