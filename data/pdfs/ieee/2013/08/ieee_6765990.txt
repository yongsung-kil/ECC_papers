A Fast Convergence and Area-Efficient Decoder for Quasi-Cyclic Low-Density Parity-Check Codes

Zhibin Luan, Student Member, IEEE, Yukui Pei, Member, IEEE, Ning Ge, Member, IEEE Tsinghua National Laboratory for Information Science and Technology
Department of Electronic Engineering, Tsinghua University, Beijing 100084, China Email: gening@tsinghua.edu.cn

Abstract—The quasi-cyclic low-density parity-check (QCLDPC) codes have attracted much attention in space communication systems. However, the decoders are still difficult to be applied in practice for their large area and high memory requirements. Moreover, the clock cycles for the input and output interfaces can not be ignored due to the I/O resource is also limited on this occasion, which influences the throughput improvement significantly. This paper presents a parallel pipelined decoder architecture for QC-LDPC codes, which can largely reduce their area and memory size while maintaining a fast convergence speed. The decoding approach reformulates the original normalized min-sum algorithm and adopts a two pipelines architecture to eliminate the clock cycles for the I/O and reduce the number of clock cycles per iteration. Chip designed with TSMC 0.13-µm eight-metal-layer standard CMOS technology shows that it can achieve a throughput up to 767 Mbps with only 3.12 mm2 core area consumption. Especially, only 36 I/O ports are occupied, less than 10% of the conventional decoders.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are widely adopted by many modern communication standards, such as DVBS2, 802.11n, 802.11ad, 802.16e etc., for their excellent error correction capability and relatively simple decoding algorithm. As a special family of LDPC codes with circulant permutated submatrix, QC-LDPC codes are much more suitable for highspeed very large scale integration (VLSI) implementation at lower complexity and smaller memory size. Actually, LDPC codes applied in the above standards are QC-LDPC codes.
The decoding algorithms for an LDPC code mainly include the min-sum algorithm (MSA) [1] and the sum-product algorithm (SPA) [2]. Compared with the MSA, the SPA can obtain the better decoding performance. But since the MSA simplify the check node processing, it leads to significant silicon area, memory size and realization complexity reduction only at the cost of 0.5 dB performance loss. Especially, after the modified MSAs, namely the normalized MSA [3] and offset MSA [3], were proposed, the gap between the SPA and the modified MSA can be reduced to about 0.05 dB with proper parameters [3].
For ASIC implementation, two different timing schedules have been studied in the previous literatures. One is two-phase massage-passing (TPMP) [2] scheme, which updates all of the check nodes and variable nodes alternately. Another is layered schedule [4], which divides the check nodes (or variable nodes)

into many layers and updates a layer of check nodes (or variable nodes) and all variable nodes (or check nodes) that associated with the updated check nodes (or variable nodes) in two successive steps. In general, the layered schedule can achieve higher throughput with more complexity.
The first proposed layered schedule is row-layered decoding approach [4], which consumes less number of iterations but more clock cycles at each iteration. Totally, it does not accelerate the convergence speed. Indeed, it has longer critical path for heavier row weight code words. The decoder architecture [5], adopting column-layered schedule, can achieve higher throughput. However, it has higher memory requirement for storing the check-to-variable messages although the check node processing is approximated. Also, it is suffering from a little performance loss due to the approximation. The decoder architecture [6] is similar with [5], but it is not layered schedule because the current layer messages update without using the messages from the previous layer. It is memory efficient and improves the throughput by higher parallel degree with higher hardware cost and more decoding iteration numbers. In [7], a novel decoding approach is proposed, which is configurable for any QC-LDPC codes and can reduce the storage resource and power a lot. However, it still requires to store the channel intrinsic logarithm likelihood ratio (LLR). Additionally, it has slower convergence while the number of nonzero submatrices in H gets larger.
In this work, a parallel pipelined row-layered decoder for QC-LDPC codes is developed. We reformulate the conventional normalized MSA to achieve faster convergence speed and more area efficiency. The paper is organized as follows. Section II presents the reformulated normalized MSA and the performance curve. The implementation of the proposed parallel pipelined row-layered decoder architecture is presented in Section III. Section IV shows the ASIC design results, followed by the conclusions in Section V.

A. Transformed Normalized MSA
A binary QC-LDPC code (M, N, p, t) can be specified by an M ×N parity check matrix H. Each row of the parity check matrix is associated with a check node m and each column is associated with a variable node n. Each submatrix is a p × p either all zero or circulant permutated square matrix. The total number of nonzero submatrices is t and the row weight of the mth block row is tm.
We define the set of variable nodes that participate in check node m as N (m) = {n : Hm,n = 1}, and the set of check nodes in which variable node n participates as M (n) = {m : Hm,n = 1}. Let In be the channel intrinsic LLR for variable node n; Rmi n, the extrinsic LLR from check node m to variable node n in the ith iteration; Lm i n, the extrinsic LLR from variable node n to check node m in the ith iteration.
The definitions of the memories used in the decoding as follows. SumRAMn stores the posterior LLR of variable node n, 1stminRAMm and 2ndminRAMm store the first and the second minimum magnitudes of the variable-to-check messages from variable nodes N (m) to check node m respectively, posRAMm stores the position indexes of the first minimum magnitudes in row m, SignRAMmn stores the signs of the variable-to-check messages from variable node n to check node m, SignFlagRAMm stores module 2 sums of the signs of the variable-to-check messages from variable nodes N (m) to check node m.
The proposed decoding based on the normalized MSA is described in Algorithm 1. The scaling factor α is set as 0.5 for the convenience of VLSI implementation.
Although the proposed decoding is mathematically equivalent to the original normalized MSA, it has four key advantages for VLSI implementation.
i) Faster convergence speed. A parallel pipelined rowlayered decoding schedule is introduced which can significantly reduce both the iterative times and the requirement clock cycles at each iteration.
ii) More simple processing units. Unlike the conventional row-layer schedule, which requires serial concatenation of multiple comparators or adders, the check node unit (CNU) and variable node unit (VNU) in our decoder need only one counterpart for the nonzero submatrices are processed one by one. Moreover, the critical path is also shortened.
iii) Smaller memory size. Without storing the entire check-tovariable messages, Rmi n for each check node m can be recovered by the values stored in 1stminRAMm, 2ndminRAMm, posRAMm, SignRAMmn and SignFlagRAMm. In addition, no extra memory is required for the channel intrinsic messages which can be directly stored in SumRAMn.
iv) It is configurable for multi-rate QC-LDPC codes.

III. PROPOSED PARALLEL PIPELINED ROW-LAYERED DECODER ARCHITECTURE
This section presents the developed parallel pipelined rowlayered QC-LDPC decoder architecture based on the above normalized MSA transformation. Different from the traditional decoder architectures in [9]-[12], whose messages from the same rows or columns are updated simultaneously, in our decoder the messages corresponding to one nonzero submatrix are processed in one clock cycle and each iteration is accomplished serially by scanning check matrix H in the row-wise order. Two pipelines, one is two-stage pipelining of different code words and another is two-stage pipelining of check node processing and variable node processing, are introduced to eliminate the clock cycles for the I/O interfaces and reduce the number of clock cycles per iteration respectively. The proposed decoding approach generates p check-to-variable or variable-to-check messages corresponding to a nonzero submatrix within one clock cycle. Since the QC-LDPC code is partitioned into M row layers, each check node corresponds to a layer. After N (m) nonzero submatrices that associate to layer m are processed serially in the check node updating phase, the variable node updating phase starts and the next layer m′ = (m + 1) mod M is processed interleaved with it. IV. IMPLEMENTATION RESULTS
Based on (4, 24, 54, 85) QC-LDPC code adopted in 802.11n standard, a parallel pipelined row-layered decoder is designed using Synopsys tools and fabricated in the TSMC 0.13-µm eight-metal-layer standard CMOS technology. According to (3), when the decoder operates at a maximum frequency of 230 MHz, it can achieve a throughput of 767 Mb/s at 3 iterations while occupying an area of about 3.12 mm2 which contains two parallel decoders according to (4). The total memory bits of the designed decoder are about 38 508 bits at 8-bit quantization.
Table I compares the proposed parallel pipelined rowlayered decoder with the representative recent work on QCLDPC code decoder design. In order to compare at the same technology, the area of the decoder in [7] is roughly scaled down by 2. Considering the memory bits, silicon area, and the convergence speed described in the section (II), this comparison verifies the memory and silicon area efficiency of the proposed design approach, which also has a fast convergence speed. Moreover, the number of the I/O ports required by this work is reduced significantly as is shown in table I.
V. CONCLUSION
In this paper, by rearranging the conventional normalized MSA, we develop a fast convergence and memory-efficient decoder for QC-LDPC codes. As a result, the proposed decoder can significantly speed the convergence while largely maintaining decoding performance. Without storing the entire check-to-variable messages and the channel intrinsic messages can be directly stored in SumRAM, memory resources and silicon area can be drastically saved.
