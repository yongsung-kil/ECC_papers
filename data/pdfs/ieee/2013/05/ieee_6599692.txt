Design of high-Throughput QC-LDPC Decoder for WiMAX standard
Tahere Heidari*, Abumoslem Jannesari** * Tarbiat Modares University , tahere.heidari@modares.ac.ir
** Tarbiat Modares University, jannesari@modares.ac.ir

Abstract: In this paper, a high throughput low-density paritycheck (LDPC) decoder for 802.16e standard is presented. With simultaneous rows and columns processing, which reduced the number of clock cycles per iteration, the throughput of the decoder is improved. The proposed decoder architecture was designed for 802.16e standard with rate of 1/2 and code length of 2304 with 7-encodings style. It is synthesized on 130 nm CMOS technology by Synopsys Design Compiler. The obtained result in the operating frequency of 100 MHz shows total power consumption of 242mW and the chip area of 6.9 mm2.

Keywords: LDPC, decoder, QC_LDPC, High Throughput, Min Sum Algorithm.

4. Introduction

The Low Density Parity Check Codes (LDPC) are the

class of Forward error correction codes that are invented

at 1960 by Gallager[1]. In recent years, because of their

very high performance and efficient VLSI

implementation, are widely applied in the broadcast and

communication systems. Due to their near Shannon limit

performance in the high code rate range [2], they are

applied in different standards such as DVB-S2, WiFi

(IEEE 802.11), WiMAX (IEEE 802.16e) [3], [4], [5].

Belief Propagation (BP) algorithm, that also known

Sum-Product Algorithm (SPA), is used to decode LDPC

codes [6]. In addition, there are many algorithms that are

based on SPA. To reduce the complexity of SPA, Min-

Sum Algorithm (MSA) was introduced [7].

The architectures of LDPC decoder are classified in

three types: Fully-parallel, semi-parallel and Fully-serial

[8]. In fully-parallel architectures, for every check node

and variable node, it is associated an individual

processor. This architecture have good performance but

large hardware cost. In fully-serial, for all variable nodes

(check nodes), only one unit processor is used. In this

type, the hardware is low but the performance is too slow.

In semi-parallel, multiple unit processors are used. In this

architecture, there is a trade-off between hardware cost

and performance.

The decoding throughput of LDPC decoder can be

calculated as [9]:

Therefore, by reducing the number of cycles per
iteration, increasing the frequency or increasing the block length the throughput is improving.
In this paper, a new LDPC decoder architecture based on Min-Sum Algorithm is presented. This proposed architecture significantly reduces the clock cycles per iteration, which increase the throughput of decoder. The paper is organized as follow: section 2 expresses a brief review of LDPC codes. In Section 3, the proposed decoder architecture is presented. Section 4 gives the implementation results. The conclusion is presented in section 5.
2. Low Density Parity Check Code
The LDPC codes can be described by M*N parity check matrix (H) and its Tanner graph. The matrix should be sparse, which is means that most of the matrix elements are 0. The Tanner graph has two type nodes: check nodes and variable nodes. Check nodes define the number of parity check equation, which is equals to number of the rows of matrix H, variable nodes define the code length, which is equals to the number of matrix H columns.
The random codes and QC codes are two class of LDPC codes [10]. The random codes have better performance than QC-LDPC codes. But QC-LDPC codes due to the regularity of their parity check matrices have high speed decoding schemes. In QC-LDPC codes, the matrix H is decomposed to sub-matrices in which each sub-matrix is zero or circulated permutation of the identical matrix. Therefore, the QC-LDPC codes are more popular in IEEE 802.16e. The parity check matrix for rate 1/2 LDPC code in 802.16e standard is shown in Fig. 1.

2.2 Storage Spaces
To reduce the storage spaces, the message of check to variable nodes are store as first and second minimum values, sign each message and the position of first minimum value as ([Min_1st Min_2nd sign pos]). Since the algorithm is used for decoding, is based on soft decision, the input information are multi-bit resolution. Due to hardware limitations, in hardware implementation fixed points are used instead of floating points. With using fixed point, some information will be lost. Therefore, the decoding performance is decreased. We use 7 bits for channel LLR (including one bit for sign). The first and the second minimum values are quantized to 7 bits and to prevent occurring overflow, LQ are quantized to 8 bits. To increase the decoding performance, SMS*Min_1st and SMS*Min_2nd is represented by 7 bits. The correction factor is smaller than one. Therefore, the real Min_1st and Min_2nd are used, with quantized to 8.

3. Proposed Decoder Architecture
In SPA, updating check nodes and variable nodes are implemented in separate units. According to (6), the message of variable to check node is calculated when all message from check to variable node are calculated. In the semi- parallel architecture, the parity check matrix is divided into Lr row layer and Lc column layer. When the 1st layer was processed, the 2nd layer is processed and after each layer, the other layer is processed. When the row processing is finished, the column processing begins. In the proposed architecture, the parity check matrix is divided into L layers, and the message updating is performed layer by layer as illustrated in Fig. 2. When 1st layer is performed, 2nd layer will beginning processing, and at the same time, column processing is done. With this architecture, the number of clock cycles per iteration is reduced. Therefore, the throughput will be increased.

4. IPMPLEMENTATION RESULTS
The decoder was designed for 802.16e standard with rate 1/2 and code length 2304, which is compatible for any code length (576:96:2304). It is synthesized on 130 nm CMOS technology by Synopsys Design Compiler. The result obtained in the operating frequency of 100 MHz, showed total power consumption and chip area is equal 242mW, 6.9mm2 respectively. The implementation results and the comparison with previous decoders are shown in Table I. The comparison is based on the WiMAX system with 2304 code length. This design uses an extra RAM to save the posterior LLR of variable node among layers, which increase area and power consumption. To simplify the comparison, the area in [11], [12] are scaled to the 130 nm. To have a good comparison in throughput, the frequency and iteration in [11], [12] are scaled to 100 MHz and 10 respectively. As shown in Table I, the proposed architecture in comparison to the decoders presented in [11], [12] has better throughput. With use extra RAM, the area is increased. Therefore, in comparison with [12], we use bigger area (0.6 mm2) to achieve better throughput.

5. CONCLUSIONS
In this paper, a new architecture for QC-LDPC decoder is proposed. The architecture can improve the throughput of QC-LDPC codes by simultaneous rows and columns processing. The decoder can support different code length (576:96:2304). By saving SMS*Min_1st and SMS*Min_2nd instead of Min_1st and Min_2nd, not only reduced the number of multipliers but also improve decoding performance. The proposed decoder architecture was designed for 802.16e standard with rate 1/2 and code length of 2304.
