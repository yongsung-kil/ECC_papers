Abstract— In LDPC code design, the structure of code’s parity check matrix can be very crucial to code performance. In this paper, we have investigated a simple designed parity check matrix when the code is applied to a PMR channel of a storage system. Belief propagation algorithm and min-sum algorithm are two major decoding algorithms used to benchmarked the code’s merits. Additive white Gaussian noise channel (AWGN) and perpendicular magnetic recording (PMR) channel with SISO equalizer in PR2 target are the conditions of our investigation. The LDPC code itself is a modified array type. Its construction fairly simple compared to normal array and quasi-cyclic code. We used two decision variables to define the permutation matrix. A method proposed by Tanner’s et.al had been adapted to the modified array code. The decoders have been tested for 5 and 20. iterations. SNR versus BER and BLER were measured.
Keywords—Error correction coding, LDPC codes, min-sum
decoding, iterative decoding, magnetic recording
I. INTRODUCTION
In HDD manufacturing technology, the Longitudinal Magnetic Recording (LMR) is gradually replaced by Perpendicular Magnetic Recording (PMR) technique since the later holds good capability advantage. The higher the capacity of the media, the more difficult in signal conditioning to be imposed. With this regard, error correction code is obviously necessary. Viterbi and RS code have been used for years. However with the introductory of LDPC code, SOVA and LDPC have proved their superior in term of BER and SNR.
Low parity density check (LDPC) code is an error correcting code firstly introduced by Gallager [1]. It was recovered again by Mackey [2] some ten years later. The LDPC code construction is defined by sparse parity check matrix H. Normally, LDPC codes are decoded by Belief propagation algorithm (BP) or Sum-product algorithm (SP) [2]. Both of them are considerably complicated. The BP has been modified with the approximation idea to reduce the algorithm complexity by Fossorier [3]. It is so called MinSum approximation (MS).
In this paper, we demonstrate a fairly simple way to design a parity check matrix intended for a modified array code. Actually the design is targeted for a perpendicular magnetic recording channel.

The paper is organized as follows. In section II, we will give a brief on LDPC decoders. Belief propagation and Min Sum algorithms are most focused. In section III, the design of a parity check matrix with code rate 0.9 is given in quite detail. In section IV, a hard disk channel system is explained. We applied the designed code to this system. Based on that system, the performance of the codes is shown and discussed in section V. Finally, the conclusion of this work is made in section VI.

V. RESULTS
AWGN channel is considered in this experiment as only
wn is added as detailed in the previous section. The designed
matrix performance is investigated by employing 2 different LDPC decoders; BP and MS. In practical HDD system where the through is a constraint factor, only a small number of iteration is allowed for the decoder. In this work 5 iterations and 20 iterations are investigated. Bit error rate (BER) and block error rate (BLER or FER or SER) versus signal to noise ratio (SNR) are plotted according to the number of iteration specified. This is shown in Fig. 2 and 3. Quite obvious, BP shows better performance compared to MS when SNR is greater than 3dB. Increasing in number of iterations can improve both the BER and BLER performance.
It should be noted that the result shown in Fig. 2 and 3 are obtained when we consider only the encoder and additive white noise.

Fig. 3 Block error rate in AWGN channel
Regarding the system shown in Fig. 1, there is no need for the inner code and the outer code to perform the same number of iteration. These can be made vary when the smooth data flow is a case. Toward this investigation, we examined 3 iterations for the inner code and 5 iterations for outer code. The resulted BER performance is shown in Fig. 4 and BLER performance in Fig. 5 respectively. As shown in Fig. 4, it is obvious that the deployment of LDPC decoder is out performing compared to that of using only SOVA. The number of iteration seems to have less effect to both BP and MS.
