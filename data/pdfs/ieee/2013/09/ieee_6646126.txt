Abstract—LDPC codes have been intensively used in various wireless communication applications, due to their increased BER performance. The present paper summarizes the state of the art applications of short length LDPC codes and proposes FPGA based application specific hardware architectures for shortlength LDPC decoders. The decoding algorithms considered for implementation are both belief propagation and min-sum algorithm. Due to the increased BER performances, the proposed architecture make use of parallel computation capabilities offered by FPGA technology in order to implement the belief propagation algorithm. In spite of the iterative nature and increased computational complexity of the LDPC decoding algorithm, the proposed architecture achieves high-throughput, mandatory in real-time application and data transmission. The architecture for the LDPC belief propagation based decoder is based on arctangent hyperbolic function approximation used for check nodes update.
Keywords—LDPC codes; FPGA; decoding algorithms; hardware implementations; parallel computing
I. INTRODUCTION
Introduced by Galager in 1962, LDPC codes are used for error correction in communication channels. They are considered a significant breakthrough in digital communication, due to their remarkable BER performances which are 0.04 dB close to the Shannon theoretical limit [1]. The main disadvantage of using LDPC codes is the increased computational power needed by the iterative decoding process. Moreover, the code-word length in case of some LDPC codes (e.g. DVB-S2 standard) exceeds 50 kb, which demands high computational complexity. In spite of the aforementioned disadvantages, the increased computational power, available due to state of the art technologies like GPU or FPGAs, used in conjunction with reduced code-word LDPC codes offer the possibility to develop high-throughput applications which make use of LDPC codes. In this section, an introduction on LDPC codes, the LDPC decoding algorithms and their applications are presented. Further on, FPGA based architectures are proposed for high-throughput LDPC decoder implementations in case of both belief propagation decoding algorithms.
A. LDPC codes
An LDPC code represents a linear block code whose codeword of length N satisfy a set of M linear parity-check

constraints. The code rate for LDPC codes is denoted by R, with R=1-M/N. Each codeword c satisfies the relation c*Ht=0 (Ht = H transposed). The main characteristic of LDPC codes is the sparse parity-check matrix H(M,N) composed by 1 and 0 values, each of its line specifying one of the parity-check constraints. The generator matrix G for LDPC codeword is obtained as G=[I P], considering H=[Pt I]. The LDPC decoding is characterized by passing probabilistic messages between two types of nodes, M check-nodes (CN) and N variable nodes (VN). In Fig.1, the variable nodes are represented by ci while the check nodes by fj. The messages transmitted iteratively between the two types of nodes contain both the probabilities corresponding to the i node values 0 and 1 as denoted by LLRi=log(P(yi=0)/P(yi=1)), representing a Log-Likelihood Ratio. Each type of nodes collects incoming messages from its complementary nodes and produces an outgoing message (Fig. 1.a, 1.b). The message qi,j is sent from VN to CN and represents the probability for the yi bit from the codeword to be 0 or 1. The CN sends the ri,j message back to the VN, indicating the probability for the yi codeword bit to be 0 or 1, considering the parity check constraints. Thus, the LDPC decoding determines the correct codeword based on the resulted yi bits, after performing iterative updates for VN and CN nodes [2]. B. Application of short-length LDPC
In case of LDPC codes, LDPC efficiency regarding BER increases once with the code-word length. Thus, in case of digital satellite communication standards DVB-S2 the code length is up to 64 000 b. Nevertheless, medium or short length LDPC codes have also their applications.

Bit error rate – BER is used as a measure for security in this paper. The method is shown to be effective in sense that BER for the eavesdropper is close to 0.5, even if it has the ability to use a MAP decoder, while the BER for the desired receiver of the communication parties is close to 0. As presented in [4] FPGA based implementation for reduced code length LDPC decoders can also be used in order to correct errors occurred in data storage using flash memories. A biometric cryptosystem is constructed by Imamverdijyev et al. in [5] based on LDPC codes having the codeword of length n = 255 or 511. Moreover, LDPC codes found also their application in patterned media storage as shown in [6], where the major contribution is a stop update criterion for the LDPC decoding algorithm which leads to an increased LDPC decoder throughput. Reduced code length LDPC codes are also used in wireless communication standards like WiMAX (e.g. n = 576). The aforementioned LDPC code is used in Section III for comparison regarding the achieved throughput of our proposed implementation of the LDPC decoding algorithm.

In case of application where coding and decoding using LDPC codes are applied on real time data transmissions or in cases when benefits regarding physical layer security in wireless communication are achieved [3], efficient hardware implementation for the iterative LDPC decoding algorithms are mandatory. The hardware architectures aim to deliver high throughput while increased BER performances are desired.
II. HARDWARE ARCHITECTURES FOR LDPC DECODERS
FPGA technology offers the possibility to exploit spatial and temporal parallelism in order to implement custom hardware functionality. Its low cost, short time to market and ease of reconfiguration are used to develop application specific hardware architectures for reduced code-length LDPC decoders [10]. The proposed architectures can be used in application as the ones presented in section I.B. The algorithm used for implementation is the belief propagation decoding (Fig. 2), due to the increased BER performances. The choice of the decoding algorithm is motivated in the Results and Discussion section where its BER performances are compared with the min-sum algorithm in case of the chosen dimension for the LDPC codeword.

For an efficient implementation of the designed from Fig. 4, n value is chosen as a power of 2. A full pipelined architecture was developed for the proposed implementation which reduces the computational time for the TCU to 1 pixel/clock cycle with an initial 3 clock cycles delay. The same approach was used for implementing the tanh-1 function used also for the check nodes update, eq. (1). The logic block corresponding to the tanh-1 function is TCU-1. The possibility of parallel update of each variable node and check nodes within each of the iterations of the decoding algorithm described in Fig. 2, offers the possibility to design parallel application specific hardware architecture for the LDPC decoder. Thus, our proposed architecture described in Fig. 5 makes use of multiple instances of the TCU unit for the update of each check node. The LDPC code size is N=576 and M=288 with a code rate of R=1/2.
The check nodes are stored in R2 register of size M. The H matrix values are stored in look-up-tables and each line i specifies the variable nodes prev (R1) register values which are inputs for the appropriate TCU units and their accumulators. In this manner all check nodes are sequentially updated as follows (Fig. 5). For each R2i check-node, taking into account that each line of the H matrix has q=6 ones, 5 multipliers are necessary. The whole check nodes update process can be implemented as a 3 stage pipeline process, each stage denoted by the corresponding dotted rectangle shadowed area marked with S1, S2 and S3 respectively. Considering the available multipliers of FPGAs, the number of p check nodes for which the update is performed in parallel within one algorithm iteration is chosen.

III. EXPERIMENTAL RESULTS
The LDPC decoding algorithms are simulated within an AWGN communication channel with BPSK modulation. In order to ensure the compatibility with the application presented in Section I and with the WiMAX standard, the hardware implementations and algorithms simulations are performed in case of an LDPC code of size N=576 and M=288 with a code rate of R=1/2. The simulations are performed using C code implementations of the belief propagation and min-sum decoding algorithms. Results are compared in terms of BER performance. Taking into account the increased BER performances (Fig. 6), the belief propagation decoding algorithm was chosen for the implementation of the aforementioned LDPC code. The signal/noise ratio levels considered for simulation are EB/N0 = [4db…7db] on x axis. The number of code blocks is 106, while the maximum number of iterations performed for the decoding process is 30.
For the hardware implementation of the proposed LDPC decoder scheme, the Xilinx FPGA XC71500T was chosen, having a number of 1600 DSP48E. In this way, 1600 multipliers are available for the computation of the decoding algorithm. The number of p check nodes for which update is performed in parallel is 36, while M and N code size are 288 and 576 respectively. IV. CONCLUSIONS
LDPC codes bring up a remarkable potential regarding high-throughput channel coding for error correction.

Considering the most used LDPC decoding algorithms, the belief propagation one achieves better performances regarding BER compared with the min-sum algorithm, but the computational complexity is significantly higher. Thus, for a LDPC code having 576 bits codeword length, our proposed hardware implementation needs 85% hardware resource of XC71500T FPGA chip. Thus in case of increased codeword length (DVB-S2 standard – 60kb codeword) the hardware resource usage is too high. Nevertheless, our proposed approach for the LDPC decoder implementation proves that in case of short-length LDPC decoders, even if increased computational complexity is needed, hardware implementation can be considered for the belief propagation algorithm, in case BER performances are of major importance. The proposed FPGA based application specific hardware architecture for the LDPC decoder achieves 484 Mb/s, in spite of high computation complexity of the belief propagation decoding algorithm. Moreover, our LDPC decoder implementation is suitable for applications which use short-length LDPC codes, due to its parallel computation capabilities, low-cost and ease of reconfiguration. To conclude, FPGA based architectures for LDPC decoder represent a solution for easy integration of channel-decoding coprocessors in application using LDPC coder/decoder architectures.
