A Construction of LDPC Codes with Low Error Floors

Abstract—In this paper, we propose an effective method to construct practical LDPC codes with low error floors. We first consider a class of LDPC codes, called masked LazebnikUstimenko (LU) codes each of which is obtained by removing edges and nodes from the Tanner graph of an LU code and may have arbitrary large girth and high code rate. We then derive a lower bound for the stopping distance of a masked LU code and develop a simple method to further purge edges and nodes to obtain a small code ensemble with relatively lower error floors than the original masked LU code ensemble.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were invented by Gallager [1] in early 1960’s. In recent years, LDPC codes were rediscovered and attracted high attention due to their remarkable performance near to the Shannon limit [2] with lowcomplexity iterative message-passing decoding algorithms and sufficiently long codeword length. The performance of LDPC codes under iterative message-passing algorithms is dominated by certain structural properties of the Tanner graph [3] of the code. The error-prone patterns for the binary erasure channel (BEC) are stopping sets (SSs) and were introduced by Di et. al. in [4]. A stopping set is a subset S of variable nodes of the Tanner graph such that all neighbors of S are connected to S at least twice. The floor of the error performance of an LDPC code is determined by the minimum stopping sets of the LDPC code [5]. The size of a minimum stopping set is called the stopping distance of the code. For general memoryless binaryinput/symmetric-output channels, the error-prone patterns have different names motivated by different analytical techniques, such as trapping sets [6] and near-codewords [7]. An online resource maintained by Vontobel about these topics can be found in [8].
The construction of an LDPC code with large stopping distance implies the construction of an LDPC code with low error floor. However, the problem of computing the stopping distance of a general LDPC code is NP-hard [9], [10], and only cases for some structured LDPC codes have been solved. The result for a specific set of quasi-cyclic LDPC codes has been reported in [11], [12], and a combinatorial approach has been used to construct codes with better stopping set distribution [13].
Based on existing codes, algorithms are proposed to remove

small stopping sets by adding rows to the parity-check matrices [14], [15]. Instead of exhaustively searching for all small stopping sets and then removing them, it is possible to use indirect methods to construct LDPC codes with large stopping distance. The relation between stopping sets and the girth of an LDPC code has been explored in [16] where it is shown that large girth generally implies large stopping distance.
Lazebnik-Ustimenko (LU) codes were introduced in [17] which shows that the construction of LU codes is a promising method of constructing LDPC codes with an arbitrary large girth. However, the code rate of an LU code is usually too low for practical applications. By removing edges and nodes from the Tanner graph of an LU code, we are able to obtain a new LDPC code with code rate high enough for applications but still retain the same girth as the original LU code. This method is called the masking method and the newly created LDPC code is called a masked LU code [18], [19]. However, there is a question of how to choose the edges and nodes to be removed.
In this paper, we derive a lower bound of the stopping distance of a masked LU code, evaluate the masked LU code ensemble, and construct a small code ensemble with low error floors by further purging edges and nodes from the original masked LU code ensemble.
II. LU CODES
Since belief propagation decoding algorithms generally work better when the girth of an LDPC code is large, it seems that finding a Tanner graph with large girth might be a method of constructing good LDPC codes. Lazebnik and Ustimenko gave the construction of a family of Tanner graphs D(m, q) which has qm variable nodes and qm check nodes and each variable or check node has degree q, where m ∈ {2, 3, 4, · · · } and q is a power of a prime. The girth of the Tanner graph D(m, q) is at least 2 m/2 + 4 [17]. The family of LDPC codes corresponding to this family of Tanner graphs is called the family of LU(m, q) codes [20].
The original LU(m, q) is defined by an incidence relation of length m vectors in GF(q). This construction of LU(m, q) codes can be viewed as the construction by applying generalized Kronecker product on parity-check matrix recursively [19]. In other words, the construction of LU(m, q) codes can

III. MASKED LU CODES WITH HIGHER CODE RATE
Since having a qm × qm parity-check matrix, LU(m, q) is expected to be a low rate code. In many cases, the code rate of an LU(m, q) code is lower than 0.1. The code rate is too low to be used in practical applications. In this section, we will discuss a method of truncating some of the rows and columns of the parity-check matrix H(m, q) of an LU(m, q) code to render high rate LDPC codes [18], [21].

A. Masking
Let W be a J × L submatrix of W by removing (q − J) rows and (q−L) columns of W in (8). Let H be a J ×L block matrix by removing the corresponding blocks Hij. Then, the masked code, denoted as Hmk(m, q), is

Hmk(m, q) = W ⊗g H.

(9)

We call W the basic masking matrix of the Hmk(m, q) code. We also denote a masked LU code by LUmk(m, q). Note that an LUmk(m, q) code is not an LU code but still an LDPC code. Hereafter, we will call this masking method as the type
I masking method.

B. The First Order Suppressing Effect

There are many choices when we choose a basic masking
matrix in the type I masking method. In [5], the first order
suppressing effect of a cyclically lifted (CL) LDPC code
ensemble is given. This work gives us a hint of how to choose
a suitable basic masking matrix. Thus, we next introduce the
first order suppressing effect of a CL LDPC code ensemble.
The first order suppressing effect gives us a metric of the
“survival” probability of a stopping set in the base code of this CL LDPC code ensemble after cyclic lifting. Let CL be the CL LDPC code of length nK obtained from a base code CB of length n where K is the lifting factor. Let SL be a nonempty stopping set of CL. Note that SL is a subset of the set of all variable nodes {vi}i=0,1,··· ,nK−1 of the Tanner graph of CL. Rename all vi ∈ SL by v i/K and let SL/K be the set of these renamed nodes. Let SB be the set obtained from SL/K by removing the repeated nodes in SL/K . Then, we have that SB is a stopping set of CB [5].
The first order suppressing effect of the stopping set SB is of the order [5]

K −(0.5#E−#V +0.5#Codd),

(11)

where #V = |SB|, #E and #Codd are the number of edges
and the number of check nodes of odd degree in the subgraph
of the Tanner graph of CB induced by SB, respectively. The suppressing weight of SB is defined as Wsup 0.5#E − #V +0.5#Codd. In particular, for an LDPC code with variable nodes of the same degree dv, the suppressing weight Wsup of a stopping set S in the LDPC code is

Wsup = 0.5(dv − 2)#V + 0.5#Codd.

(12)

From the above results, the larger suppressing weight Wsup of a stopping set of the base code is, the smaller expected number of the first order survivals will be after cyclic lifting. Thus, if we choose a base code whose stopping sets are always with relatively high Wsup’s, we can have a CL LDPC code with relatively less small stopping sets.

IV. A CONSTRUCTION METHOD
In this section, we develop a method to construct an LUmk(m, q) code ensemble with low error floors when subject to the message passing decoding algorithms on the BEC.
A. Base Codes with Variable Nodes of Different Fixed Degree
In this subsection, we justify the idea that the suppressing weights Wsup of stopping sets of the base codes of LUmk(m, q) codes is a metric which can evaluate the effectiveness of basic masking matrices of LUmk(m, q) codes. However, it is not straightforward to construct a parity-check matrix of an LDPC code with stopping sets with preseted Wsup. But we find out that in general an LDPC code with variable nodes of higher degree has stopping sets with higher Wsup in average.
We compare the performance of LUmk(2, 17) code ensembles with base codes with variable nodes of different fixed degree. We let the basic masking matrices be 8 × 16 binary matrices. Thus these LUmk(2, 17) codes have code rate at least 0.5. Figure 1 gives the performance of five codes randomly chosen from each LUmk(2, 17) code ensemble with base codes with variable nodes of fixed degree from 2 to 6. The code ensemble with base codes with variable nodes of degree 3 has the best performance in average.
In [22] and [23], a minimum stopping set exhaustion algorithm for LDPC codes with short codeword length is given. The base codes of LUmk(2, 17) codes have codeword length 16 which is short enough for us to exhaustively search all stopping sets in these base codes. We compute Wsup of each stopping set and show in Table I the distribution of Wsup of the stopping sets of the base codes with variable nodes of different fixed degree.
From Figure 1, we find that the performance curves of the codes of each LUmk(m, q) code ensemble with base codes with variable nodes of a fixed degree form a cluster. And by Table I, we find that base codes with variable nodes of degree 2 have generally very low or even vanished Wsup value for their stopping sets and then benefit very little from lifting mentioned in [5], which explains why the codes of an LUmk(m, q) code ensemble with base codes with variable nodes of degree 2 have poor performance in Figure 1.
B. Base Codes with Different Stopping Distance
In this subsection, we compare the performance of LUmk(m, q) code ensembles with base codes with variable nodes of the same degree but different stopping distance.

Figure 2 gives the performance of five codes randomly chosen from each LUmk(2, 17) code ensemble with base codes with variable nodes of degree 3 and different stopping distance. This figure shows that by choosing the base codes with higher stopping distance, we can find a code ensemble with relatively good performance and lower error floors from the LUmk(m, q) code ensemble with base codes with variable nodes of a fixed degree.
By considering LUmk(m, q) code ensembles with base codes with variable nodes of a fixed degree and having a fixed stopping distance, we find out that the performance of these code ensembles is independent of the number of minimum stopping sets.
C. A Method to Construct a Code Ensemble with Lower Error Floors from a Given LUmk Code Ensemble
From the investigation in above, a procedure is implemented to obtain a small code ensemble with relatively good performance and low error floors from a given LUmk(m, q) code ensemble as follows:
1) Comparing the performance of LUmk(m, q) code ensembles with base codes with variable nodes of different fixed degree v to choose the best v, to which the corresponding LUmk(m, q) code ensemble has the best performance and lowest error floors compared with others.
2) Fix the best degree v for variable nodes in the previous step and then choose an LUmk(m, q) code ensemble with the highest stopping distance which we can reach.
Figure 3 shows the performance of five codes randomly chosen from each LUmk(2, 17) code ensembles with three different types of base codes: base codes with parity-check matrices whose entries are all 1’s, base codes which are randomly chosen regular (3, 6) codes, base codes which are constructed by our procedure in above.
Similar comparison is also applied to LUmk(2, 16) code ensembles as shown in Figure 4, where the best ensemble obtained from our procedure has variable degree 4.
Both Figures 3 and 4 show that the LUmk(2, 17) and the LUmk(2, 16) code ensembles constructed by our procedure have relatively good performance in average and lower error floors compared to the others.
V. CONCLUSION
In this paper, we introduce masked LU codes LUmk(m, q), derive a lower bound of the stopping distance of an LUmk(m, q) code, and propose a procedure to construct a smaller code ensemble with low error floors from a given LUmk(m, q) code ensemble. Simulation results also support the effectiveness of our construction.
