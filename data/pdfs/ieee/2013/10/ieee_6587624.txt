Tree-Structured Expectation Propagation for LDPC Decoding over BMS Channels
Luis Salamanca, Student Member, IEEE, Pablo M. Olmos, Member, IEEE, Fernando Pe´rez-Cruz, Senior Member, IEEE, and Juan Jose´ Murillo-Fuentes, Senior Member, IEEE

Abstract—In this paper, we put forward the tree-structured expectation propagation (TEP) algorithm for decoding block and convolutional low-density parity-check codes over any binary channel. We have already shown that TEP improves belief propagation (BP) over the binary erasure channel (BEC) by imposing marginal constraints over a set of pairs of variables that form a tree or a forest. The TEP decoder is a messagepassing algorithm that sequentially builds a tree/forest of erased variables to capture additional information disregarded by the standard BP decoder, which leads to a noticeable reduction of the error rate for finite-length codes. In this paper, we show how the TEP can be extended to any channel, specifically to binary memoryless symmetric (BMS) channels. We particularly focus on how the TEP algorithm can be adapted for any channel model and, more importantly, how to choose the tree/forest to keep the gains observed for block and convolutional LDPC codes over the BEC.
Index Terms—Channel coding, sparse linear codes, expectation propagation.
I. INTRODUCTION
B ELIEF propagation (BP) is the standard linear-time tool for iterative decoding of low-density parity-check (LDPC) codes [1] and LDPC convolutional (LDPCC) codes [2]. BP approximates the bitwise maximum a posteriori (MAP) estimate for each encoded bit and it needs to rely on irregular LDPC ensembles to achieve capacity as the block length tends to infinity [3]. Irregular LDPC codes suffer from irreducible error floors and a slow convergence to the asymptotic case with the block length [4], [5], which limits its performance for finite-length practical codes. LDPCC codes postulate as a practical capacity achieving technique that have an almost regular construction, although they present a rate-loss that it is only tackled by long codewords [6], [7]. Expectation propagation (EP) [9] can be understood as a generalization of BP for constructing tractable approximations of an intractable joint probability density function (pdf). EP minimizes the Kullback-Leibler (KL) divergence between the true intractable pdf and a tractable approximation, given by an exponential family distribution. In [9], it is shown that BP can be reformulated as EP by considering a discrete pdf that factorizes as the product of single-variable multinomial terms. EP generalizes BP in two ways: first, it is not restricted to discrete random variables. And second, EP naturally formulates to include more versatile, but still tractable, approximating factorizations [10], such as trees or forests of variables. In [11], we proposed EP with a tree-factorized family of distributions, named as the TEP decoder, to decode LDPC codes over the binary erasure channel (BEC). We have shown that the TEP outperforms BP for finite-length codes and we have generalized this algorithm to achieve maximum likelihood decoding [12].
There are several proposals [13]–[16] that improve BP performance over the BEC, with varying degree of success and complexity level, and all of them rely on the simple structure of the BEC to achieve those gains. Unfortunately, they cannot be extended to general (binary memoryless) symmetric channels and BP is the only algorithm in use to decode LDPC codes for general channels. Even though, several methods have been proposed in the machine learning community to improve the BP solution in loopy graphical models, see [17]– [21] and the references therein. Most of these algorithms can be understood as BP-like message passing algorithms that rely on better approximations to the graphical model entropy [21] and they are usually referred to as generalized BP (GBP) [22]. GBP is well suited for dense graphs, in which short loops limit the BP approximation [17]. For sparser graphs, characterized by medium or large loops, the GBP approximation to the distribution entropy quickly converges to the BP solution and no improvements are achieved. Actually, GBP has only been applied successfully for LDPC decoding over discrete channels with memory [23], [24], in which there is a uniform structure in the channel model that GBP is able to decode accurately. Hence, GBP helps with the memory, but not with the LDPC sparse graph structure.
The goal of this paper is twofold. First, we show how, given a fixed tree-factorized family of distributions, we can apply the EP algorithm for LDPC and LPDCC codes over any channel

III. TREE STRUCTURE CONSTRUCTION
Prior to running the TEP algorithm, it is necessary to fix a tree-structure, i.e. choose a family Ftree. As discussed in [9], while determining the cost of a given choice is straightforward, estimating the accuracy for each case is a non-trivial problem. In [10], it is suggested to link the variables with highest mutual information. However, for linear codes each pair of variables has a priori zero mutual information, and we thus need to find an alternative way for building the tree-structure.
For the BEC in [11], we define the tree-structure for each received word as follows: first we run the BP algorithm until it gets stuck and, second, we only link pairs of erased variables that are connected to a factor that has only two erased variables. This follows the highest mutual information criterion [10], because if there are only two erased variables in a factor node, those variables are either equal or opposite. The TEP decoder over the BEC keeps on building the graph, i.e. joining pairs of variables in factors with only two erased variables. When all the variables that form a loop in the graph of the code are joined through pairwise terms, we are then able to decode a set of variables that could not be decoded by the BP algorithm.
For other BMS channels, we follow a set of simple rules that emulate the TEP decoder over the BEC. First, we run the BP and, when it does not converge, we run the TEP algorithm. VI. CONCLUSIONS
We have recently showed for the BEC how the TEP decoder outperforms the BP in the finite-length regime [40] and we have analyzed its performance in [11]. In this paper, we show how to generalize the formulation of the TEP decoder over any BMS channel. We adapt an algorithm that benefits from the simple structure of the BEC to any general binary channel model. We also show how we can build the tree structure for each received word to retain, for any BMS, the performance proven for the TEP decoder over the BEC. When BP fails, the TEP algorithm operates by constructing a tree/forest of pairwise marginals that allows improving the BP solution to converge to the transmitted codeword. This procedure reduces both the error rate in the waterfall and error floor regions by building different kinds of trees/forests for each received word, providing measurable gains in the finite-length regime.

In this section we empirically show that the typical complexity of the TEP decoder is significantly lower than the maximal complexity of Pearl’s cutset algorithm, because the proposed algorithm tends to produce several short chains rather than a long chain connecting a set of variables of length O(n). The maximal computational complexity of updating each factor in the TEP approximation q(V) is O(rmax2rmax−1) if we had to apply Pearl’s cutset algorithm at every iteration and all the variables in the factor are involved. However, most iterations are BP-like and, in those that we need to, only one or two variables need to be considered as known. As a result, in practice, the TEP computational complexity only doubles
