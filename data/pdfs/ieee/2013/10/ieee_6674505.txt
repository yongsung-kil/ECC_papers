A CONFLICT-FREE MEMORY MAPPING APPROACH TO DESIGN PARALLEL HARDWARE INTERLEAVER ARCHITECTURES WITH
OPTIMIZED NETWORK AND CONTROLLER
Aroua Briki, Cyrille Chavet, Philippe Coussy
Lab-STICC laboratory / Université de Bretagne Sud Centre de recherche Christiaan Huygens, 56100 Lorient (France) {aroua.briki@univ-ubs.fr; cyrille.chavet@univ-ubs.fr; philippe.coussy@univ-ubs.fr }

ABSTRACT Recent communication standards and storage systems (e.g. wireless access, digital video broadcasting or magnetic storage in hard disk drives) uses error correcting codes such as LDPC (Low Density Parity Check) or Turbo-codes to reliably transfer data between source and destination. For high data rate applications, Turbo and LDPC codes are decoded on parallel architectures. However, parallel architectures suffer from memory access conflicts and efficient memory mapping algorithms are required to design parallel interleaver architectures which are one of the most critical parts of parallel decoders. In this paper, we present a method that finds a conflict-free memory mapping for any interleaving law and associated parallelism constraint. The proposed approach always complies with the interconnection network topology the designer wants to infer. Moreover, contrary to traditional methods, the resulting architecture is optimized by reducing the cost of network and controller (network and memory controller) architectures. Our approach is compared with state of the art techniques and its interest is shown through the design of parallel interleavers used in different industrial applications such as High Speed Downlink Packet Access (HSDPA), Multi BandOrthogonal Frequency-Division Multiplexing Ultra-WideBand (MB-OFDM UWB) and a WiMAX application.
Index Terms—Parallel architectures, memory systems, turbo-like codes, interleavers, memory mapping.
1. INTRODUCTION Recent communication and storage systems and standards like digital video broadcasting (DVB-Sx, DVB-Tx…), high-speed wired links (10GBASE-T, ADSL…), wireless accesses (WiMAX, Wifi, UWB…), telecommunications systems (UMTS, HSPA, LTE…) or magnetic storage in hard disk drives all rely on error correcting codes namely LDPC (Low Density Parity Check) [18][23]and Turbo-codes [17]. LDPC and Turbo Codes are wellknown, near Shannon limit, coding/decoding approaches that are able to achieve very low bit error rates for low Signal-to-Noise Ratio (SNR) applications. Decoding principle is based on message passing algorithms in which different processing elements iteratively exchange information in order to improve the error correction performance of these codes. In order to design high data rate applications, parallel architecture are needed [1]. To increase memory bandwidth, main memory is divided into different memory banks to provide concurrent access to all the processing elements. This allows to reduce the latency and thus to increase the throughput of decoders. Figure 1 presents an example of typical parallel decoder architecture with three processing elements (i.e. PE0, PE1, PE2) connected through a dedicated interconnection network to three memory banks (i.e. RAMA, RAMB, RAMC) and a dedicated control unit that drives the architecture. The network interleaves the data exchanged by the PEs according to a rule named interleaving law or permutation law defined by the standard or the application to design.

Unfortunately, depending on both the interleaving law and the memory mapping (i.e. data placement in memory banks), different PEs may try to simultaneously access the same memory bank which results in memory conflicts [2]. Since conflicting accesses to the same memory bank increases the cost and latency of the system, there is a significant interest in guaranteeing conflict-free accesses in parallel decoder architectures.
Figure 1. Architecture overview Three kinds of solution exist to avoid or minimize memory access conflicts:
(1) Define an interleaving law that naturally allows to map data simply in different memory banks so that all the processing elements can access them without any conflict at any time instance[4][8][9]. These approaches may also reduce the cost of the interconnection networkif the interleaving law is well defined. However, they are interesting only when the designer is free to choose the permutation law i.e. free not to comply with a given standard;
(2) Simply store data elements in different memory banks without considering conflicting accesses and then use different complex topologies (like crossbar and Benes network) and additional buffers to mange conflicts on runtime [11][12][14][15]. This kind of approaches increases the cost and latency of the system due to presence of complex interconnection network and buffer management mechanism to manage conflicts.
(3) Use memory mapping algorithms [5][6][13][19] to map data in different memory banks so that each processing elements can access them without any conflict at each time instance. This kind of approach either
(a) results in a non-optimized network [13][19] in order to find a conflict-free mapping for both turbo and LDPC decoders or
(b) considers the network as a constraint (but only for in-place turbo-decoders [5]) and fails to find a network-oriented memory mapping when the interleaving law and the throughput constraint (i.e. parallelism) are incompatible with the given interconnection network, or
(c) considers the network as a constraint for conflict free memory mapping for both Turbo Code and LDPC [6], but without reducing the cost of the control unit.

Objective of memory mapping methods is to find a conflict-free memory mapping given memory access schedule (i.e. interleaving law and parallelism) and, for some approaches, interconnection network.

2.2 Existing memory mapping approaches A first solution to solve this memory mapping problem could be to use the algorithm proposed in [5]. Unfortunately, this approach, that allows optimizing interconnection network, is not able to deal with LDPC memory mapping constraints (i.e. to handle data that are accessed several times). Other approaches, like simulated annealing based approaches [13] (that targets in place memory mapping) or [19] (that targets in-place memory mapping or multiread/multi-write memory mapping) are able to find a conflict free memory mapping. In order to apply the approach proposed in [13] on LDPC, the authors proposed to modify memory access schedule by using a Static Single Assignment (SSA) form which results in oversized memory architecture. In [19] authors removed this limitation by using several memory locations for each data and by sharing memory locations between data. However, like [13] interconnection network is not optimized.
Figure 3 exposes a conflict-free memory mapping obtained by applying [19] on the example presented in figure 2. At each time instance, each data has been assigned to one memory bank (data 1 is initially stored in memory bank C, data 2 in memory bank B…).

However, updated values can be stored in a different memory bank (e.g. updated value of data 1 is stored in memory bank B, updated value of data 2 is stored in memory bank C…). For that purpose, up to two memory banks can be assigned to each data: one for the read access to the data (first sub-column) and one for the write access to memory of its updated value (second subcolumn, Figure 3).

Unfortunately, none of these approaches takes into account the interconnection network, its controller nor the memory controller. Hence in this example, each processor has to access (read and/or write) to each memory bank. Thus, even if the designer would prefer to use an optimized network (e.g. Butterfly, Barrelshifter…), the resulting interconnection network must be a full crossbar or a Benes network, being both complex and requiring many control bits. In addition, because no dedicated approach has been used, data are randomly stored in memory banks. The resulting parallel interleaver architecture can thus be significantly optimized by reducing the cost of interconnection network, network controller and memory controller.
In order to find optimized conflict-free memory mapping for both Turbo-Codes and LDPC codes, authors of [6] proposed a memory mapping approach inspired from dual-access memory mapping described in [19]. This solution generates architectures with limited network controller cost and that respect a targeted interconnection network defined by the designer. This is made possible by adding registers in the architectures to deal with conflicting data and memory locations when there is no other solution. Nevertheless, by analyzing the results, the total area could be strongly optimized by reducing the memory controller area which is the most costly part of such architectures (compared to network cost, see [26]).
Figure 4 shows a conflict-free memory mapping obtained for the example presented in figure 2 by applying [6] with a Butterfly Network [28] as design constraint. This memory mapping allows to control the interconnection network complexity and consequently to reduce the cost of the network controller. Nevertheless, complexity of the memory controller has not been considered like in [13] or [19].

The ACG formalizes for each data the set of possible addresses. It is later used along with BMAP during Address Mapping (AMap) in order to select one memory address inside the memory bank data has been assigned to such that the memory controller is optimized. BMap and AMap steps are deeply interrelated and bank mapping widely impacts ACG. Indeed, as explained in the next subsection, when two data are stored in the same memory bank, two memory locations (i.e. addresses) are required if their lifetimes overlap (while only one is sufficient otherwise). Finally, our design flow generates the final VHDL architecture with a conflict free memory mapping and the associated optimized control units. Our approach allows:
- To generate a conflict free memory mapping - To respect the designer’s interconnection network constraint - To minimize the number and the size of memory controllers (ROMs) by maximizing similarity and regularity between the addressing sequences of each memory bank.
3.1 Modeling
3.1.1 Address Conflict Graph ACG formally models the conflicts that exist between addresses associated to data in a given memory bank. The set of vertex V={v0, … vn} represents the data accesses, and the set of edges E={(vi,vj)…(vk, vl)} represents the conflicts between the data

accesses in the memories. For example, an edge (vi, vj) exists if data vi and vj must be stored at different addresses in the considered memory bank i.e. due to their overlapping lifetimes.
Consequently, if two data accesses are linked in the ACG, they
must be performed in different addresses. The lifetime is defined by īvi = [Ĳmin (vi), Ĳmax (vi)], where Ĳmin(vi) is the date of a write access of data vi in a memory bank, and Ĳmax(vi) is the date of the consecutive read access of this data vi.

For the sake of simplicity, the different accesses to the same data

will be referred in the ACG with respect to their coordinates in the

matrix (see Figure 6): if a given data appears in the first line (i=0)

and in the third column (j=2) in the matrix, which is the case for

data 8, its write access will be referred as M(0,2) in the ACG.

Constraints and Objectives In order to guarantee a valid memory mapping, some memory and network constraints have to be respected for a given parallelism and interleaving law.
Memory constraints: 1- Data which are processed at the same time instance, i.e. data that belong to the same column in the mapping matrix (which are read or written concurrently), have to be mapped to different memory banks. 2- The ith read of data vj must be performed in the same memory bank bl that the (i-1)th write i.e. a data must be read at the same memory location it has been written. Moreover, because LDPC or turbo codes decoding are based on iterative algorithms, if the first read of data vj is done in the memory bank bk, then the last write of data vj must be done in bk . Network constraint: The memory mapping has to respect the user-defined network topology i.e. the set of permutations supported by the steering architecture. For example barrel shifter support only circular permutations while crossbar support any permutations. Controller objective: In order to be able to efficiently explore the address generation space, the generation of the ACG will directly impact the memory mapping step. The chosen memory bank will be the one which respects the memory mapping constraints but also the one which generates the less address conflicts in the ACGs (in order to reduce the size of each ROM) and which maximizes to addressing regularity between the memory banks at each cycle (in order to reduce the number and the size of ROMs). This is done through a dedicated at each memory mapping step, for each possible memory bank.

3.2 Bank mapping This first step of the proposed design flow starts by generating a partial conflict-free memory mapping (i.e. a partially filled memory mapping matrix). Indeed, each time a set of data cannot be mapped in a memory bank, without violating the previously defined mapping constraints, they are postponed i.e. momentarily unmapped. Then, the postponed data are assigned to memory banks as much as possible without modifying the previously mapping of data (i.e. it finds a memory mapping for a subset of conflicting data), see Figure 7 and Figure 8.

4. EXPERIMENTS Our approach has been applied to explore the design space of several test cases. The generated VHDL by our approach has been synthesized through Xilinx ISE Design Suite 2012, targeting a Virtex6 platform. Different test cases have been used: a High Speed Packet Access (HSPA) [25], an Ultra-WideBand interleaver (UWB) [22], and a Wimax application [27]. These experiments have been performed for several parallelisms and radix. Our results are compared to state of the art approaches [5] and [13] depending on their best target application (Turbo-Codes or LDPC; the approach proposed in [5] cannot be applied on LDPC codes). Since our approach is able to target any interconnection network, two families of such network have been explored: Barrel-Shifter network (for UWB test case) and Butterfly network (for HSPA and WiMAX).

It can be observed that the larger is the frame length compared to the parallelism (i.e. more data have to be stored in fewer memories), the better are the results. The reason is, in such cases, that the optimization space is bigger for our approach.

5. CONCLUSION In this paper we proposed the first parallel memory design space exploration methodology that is able to optimize interconnection network, its controller and the memory controller. It relies on a formal modeling describing data communication constraints and address conflicts. This methodology allows the generation of a conflict-free memory mapping. Resulting architectures respect the designer architectural constraints in any case and the controllers are strongly optimized (24,5% in average and up to 42%).
