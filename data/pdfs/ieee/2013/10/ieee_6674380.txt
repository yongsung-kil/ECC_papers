Error Detection and Correction using LDPC in parallel Hopfield Networks

Constantin Anton, Laurenţiu Ionescu, *Ion Tutănescu, Alin Mazăre, Gheorghe Şerban
Electronics, Computers and Electrical Engineering Faculty - University of Pitesti, Romania {constantin.anton, laurentiu.ionescu, ion.tutanescu, alin.mazare, gheorghe.serban}@upit.ro

* Corresponding author
Abstract - Low Density Parity Check (LDPC) coding is a classical method for the detection and correction of errors. Its problem is the relatively complex operations which must be performed at encoding, and especially at decoding, to detect and correct errors caused by communication channels. We present a solution for the errors correction using regular LDPC and Hopfield networkbased associative memories. Our solution solves this problem by using an associative memory based on the Hopfield network on the decoding stage, which stores the correct code words. This memory tends to transform the code words received with errors in errors free code words. We improved the ability of the associative memory by storing the code words in a specific format (pairs with their inverses, the constant number of “1” bits). These conditions are met by using LDPC coding. To be viable in communications, which require real-time correction, our proposed solution has used Hopfield networks operating in parallel, fully integrated into the Field Programmable Gates Array (FPGA) circuit.
Keywords: regular LDPC; associative memories; Hopfield neural networks; FPGA.
I. INTRODUCTION.
Low Density Parity Check (LDPC) coding is a classical method used in errors correction introduced by Gallagher in 1963 [1]. The method is actually used in communications, particularly in wireless communications. The process of encoding, by which are obtained the code words, takes place using a generator array. The generator contains a parity array. Decoding is in a similar way, using a second array which contains the transpose of parity array.
Encoding-decoding operations involve a fairly high time consume. For this reason our concern, as well as other researchers, is to reduce processing time by hardware acceleration. Hardware parallel processing increases performance of the error correction algorithm, compared to software solutions - as is presented in [2].
There are several solutions that have aimed optimization by implementing various hardware modules combined with their own processing methods. For example, it is presented in [3] a solution which combines MacKay scheme with an encoder implemented in VLSI. Also, it is presented in [4] a demodulator for Shaped Offset Quadrature Phase Shift Keying (SOQPSK) implemented in FPGA using soft output Viterbi algorithm (SOVA), with two PLL modules inside a FPGA.

Hardware solutions have been applied to LDPC codes for large message words with multiple values on letter (nonbinary codes) like the one presented in the paper [5], where circuits used are only for addition and comparison. LDPC codes are used in error correction, especially at wireless networks [9] or large data storing devices (HDD).
Together with hardware implementation, there have been attempted the optimization of processing time using different algorithms. Some research has reached the use of artificial neural networks, such as building a decoder using a feedforward multi-perceptron neural network [6].
Unlike proposals [7] [8] which use coding/decoding algorithms, we propose a new method. Our approach involves a special type of memory, associative memory, to be used in decoder implementation. These are based on Hopfield-type neural networks and require preliminary storing of correct code words [10].
Section II describes the LDPC codes, insisting on the model chosen by us. Section III presents our encoder and decoder (last one with associative memory). Section IV displays the experimental results. We present our conclusion in Section V.

- l bits of ‘1’: l represent the total number of connections which a message bit has with code bits (see Figure 1);
- k bits of ‘1’: k represent the total number of connection which a code bit has with messages bits.

The reception is more complicated because, in addition to code - message conversion must make error detection and correction. For that we use associative memory and detection correction of errors comes by default behavior of associative memory. In the associative memory are stored (by the learning process) all possible correct code words. If the code word is received with errors from the communications channel, then neural network is put into an unstable state. Experiments and analysis were performed for a Xilinx Spartan 3E FPGA 3S500, a circuit with one of the best capacity – price ratio. As it can be seen in Table II, for a neural network composed of 30 neurons, the resource is beyond the capacity of FPGA chip Spartan3 XC3S500. For a network of 90 neurons, it is estimated a resource allocation of 502%, so it would be necessary 6 Xilinx Spartan 3E FPGA 3S500 circuits. A problem that arises in this case is how to distribute network (which - do not forget - is composed of fully interconnected neurons) on six different integrated circuits?
A property of Hopfield network, which we exploit here, is that the energy function is low when the network is in equilibrium state, so the correction was made. This property is extremely useful in our case. Instead of using a large neural network (which occupies a large area), we use several small networks operating in parallel. Each network will provide outputs which reflect its state and its energy. The network with the lowest energy level has the correct code word.

Our solution involves use of H and H ' arrays, generated in the previous section. The H array is used as a generator for code words of size 15, with L = 6 and k = 3, which ensure a minimum code distance dH = 5. These words stored in an associative memory (built with parallel Hopfield networks) allow automatic correction, with very low response time, to a maximum of 2-bit errors and detection to 4-bit error. The matrix H ' allows to generate 10-bit code word size, with L = 6 and k = 3, but for a code distance dH = 2.

V. CONCLUSIONS
Even if from the point of view of Vt (code speed) LDPC coding solution using associative memories based on Hopfield network is not doing well compared to other established communication solutions, it is worth to mention the simplification of the structure of decoder. Instead of carrying out the syndrome computation in order to achieve and then

extracting the erroneous bits, here we use a default property the Hopfield networks have, that to achieve a stable state from non-stable state. The network is in equilibrium when there are achieved the words stored in it, namely the code words generated by the generator matrix H. In addition, a feature of our solution is to use multiple parallel Hopfield network for storing a big number of code words.
We stress that a big number of columns in generator array increases the correction capacity of code words, but their size. So, a future direction is to determine an optimal between code word size and Hamming distance to be superior to other forms of encoding.
Another future line of research is the use of other algorithms for generating code words that are stored in associative memories.
