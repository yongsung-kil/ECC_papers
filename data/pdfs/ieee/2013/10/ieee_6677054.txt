Length-Compatible PEG-CRT Algorithm
Xueqin Jiang, Hongyun Guan, Moon Ho Lee, Senior Member, IEEE and Soo Young Kim, Member, IEEE
Abstract—Among the existing methods for the construction of
random-like LDPC codes, one of the most successful approaches
is progressive-edge-growth (PEG) algorithm. This approach is
simple but the complexity of the PEG algorithm scale as 𝒪(𝑛𝑚),
where 𝑛is the number of symbol nodes and 𝑚is the number of
check nodes. The PEG-CRT algorithm deals with this problem
by construct a base matrix 𝐻𝑏of size 𝑚𝑏× 𝑛𝑏with the PEG
algorithm and expand this base matrix into a parity-check matrix
𝐻of size 𝑚× 𝑛via the chinese remainder theorem (CRT),
where 𝑚≫𝑚𝑏and 𝑛≫𝑛𝑏. The size of the base matrix is
expanded without decreasing the girth. Since a smaller matrix is
constructed with the PEG algorithm and the complexity of the
CRT computation is negligible compared to the PEG algorithm,
the complexity of the whole code construction process is reduced.
However, unlike the PEG LDPC codes, the code lengths of PEG-
CRT LDPC codes are not flexible. To solve this problem, in
this paper, we propose a length compatible PEG-CRT algorithm,
which preserves good properties such as large girths, flexible code
rates, flexible code length and low densities.
Index Terms—LDPC codes, Progressive Edge-Growth (PEG)
algorithm, Chinese Remainder Theorem (CRT), girth.
I. INTRODUCTION
Low-density parity-check (LDPC) codes, first proposed in
the early 1960’s [1] and re-discovered in 1996 [2], have recent-
ly attracted much attention due to their capacity-approaching
performance. Based on the methods of construction, LDPC
codes can be classified into two general categories: 1) struc-
tured codes; and 2) random (or random-like) codes. A method
to expand the structured LDPC codes via CRT was proposed
in [3]. Codes result from this method have large girths, flexible
code lengths and flexible code rates. Among the existing
methods for the construction of random-like LDPC codes, one
of the most successful approaches is progressive-edge-growth
(PEG) algorithm proposed in [4][5]. The PEG algorithm is
flexible, supporting flexible code length, but not efficient. In
the case of long length codes, the construction of LDPC
codes with the PEG algorithm requires long time consumption.
In [6], the PEG-CRT algorithm is proposed to reduce the
complexity of the PEG algorithm for long length random-like
LDPC codes. To reduce the complexity of the PEG algorithm,
the authors use the PEG algorithm in conjunction with CRT
that can expand the under-construction matrix 𝐻𝑏of size
𝑚𝑏× 𝑛𝑏without reducing the girth. The simulation results
show that the PEG-CRT LDPC codes of length 𝑛perform
better than the PEG LDPC codes of length 𝑛𝑏, 𝑛𝑏≪𝑛and
perform similar to PEG LDPC codes of the same length with
much less complexity.
However, unlike the PEG LDPC codes, the PEG-CRT
LDPC codes do not support flexible code length [6]. To solve
this problem, in this paper, we propose a length-compatible
(LC) PEG-CRT algorithm, which preserves good properties
such as large girths, flexible code rates, flexible code length
and low densities.
The remainder of this paper is organized as follows. Section
II reviews the CRT and the PEG-CRT algorithm. Section
III introduces the LC-PEG-CRT algorithm. The examples of
the proposed LC-PEG-CRT LDPC codes and their simulation
results are given in Section IV. Finally, Section V concludes
the paper.

V. CONCLUSION
The PEG-CRT algorithm can reduce the computational
complexity of the PEG algorithm for long length random-like
LDPC codes dramatically with negligible performance loss.
Since the code length of PEG-CRT LDPC codes equals to the
multiplication of co-prime numbers, the PEG-CRT algorithm
can not support flexible code length like the PEG algorithm. In
this paper, the proposed LC-PEG-CRT algorithm can achieve
the same code length as the PEG LDPC codes. Futhermore,
it preserves the good properties such as large girth, flexible
code rates and low densities. Simulation results show that the
LC-PEG-CRT LDPC codes perform similarly to PEG LDPC
codes of the same length with much lower complexities.
