Abstract— In this paper, we compare the performance of some 
different types of time-invariant LDPC convolutional codes. We 
first confirm the well-known fact that the performance can be 
increased by increasing the number of delay, i.e. memory size. 
Then, we show by simulation that increasing the encoder block 
size may possibly increase the performance when the memory 
size is fixed, but this must be done properly. We show also that 
the performance is comparable when the constraint lengths are 
comparable regardless of memory size and encoder block size. 
For all these, we introduce a matrix expansion operation for 
various sizes of sub-matrices of H matrix to be used in the 
simulation. 
 
Keywords— LDPC codes, convolutional codes. 
I. 
 INTRODUCTION 
Low-Density Parity-Check (LDPC) block codes were first 
introduced by Gallager in the early 1960’s [1]. As a counterpart 
of LDPC block code, time-varying LDPC convolutional 
(LDPCC) codes were proposed and its construction method [2]. 
The capability of LDPCC codes has long been recognized. 
Compared to LDPC block codes, they are suitable for practical 
applications due to continuous encoding and decoding [3] and 
capable of achieving low encoding and decoding complexity. 
To show its near-capacity performance, Lentmaier et al. 
showed a threshold analysis for the iterative decoding of 
regular LDPCC codes by using density evolution techniques 
over the binary erasure channel [4]. The result shows that 
LDPCC codes have both excellent thresholds and very low 
error flows compared to traditional LDPC codes. 
There have been some considerations about the construction 
of LDPCC codes. In [5], Tanner et al. proposed constructions 
of LDPCC codes by observing the close relation between 
quasi-cyclic block codes and time-invariant LDPCC codes. 
They also mentioned that LDPCC codes have much better 
performance than the QC-LDPC block codes on which LDPCC 
codes are based. In [6], graph-cover-based methods for 
constructing time-variant and time-invariant LDPCC codes are 
proposed, and it was shown how previous construction can be 
presented in terms of their methods. The LDPCC codes by 
their construction have been shown to be capable of achieving

passing decoding.  
In this paper, we compare the performance of some different 
types of time-invariant LDPC convolutional codes. We first 
confirm the well-known fact that the performance can be 
increased by increasing the memory size. Then, we show by 
simulation that increasing the encoder block size may possibly 
increase the performance when the memory size is fixed, but 
this must be done properly. We show also that the performance 
is comparable when the constraint lengths are comparable 
regardless of memory size and encoder block size. For all these, 
we introduce a matrix expansion operation for various sizes of 
sub-matrices of H matrix to be used in the simulation. 
This paper is organized as follows. In Section II, the parity 
check matrix and its corresponding encoder structure of 
LDPCC codes are briefly described. Here, we relate the 
parameters of the parity check matrix to the encoder block size. 
In Section III, we discuss the construction of LDPCC codes for 
the simulations and introduce a matrix expansion operation for 
expanding the size of sub-matrices of H matrix. In Section IV, 
the system setting, code parameters, and their performances are 
presented. In Section V, we conclude this paper.

a QC-LDPC code whose maximum cyclic shift value is now

V. 
CONCLUSIONS 
In this paper, we tested error performance of various 
LDPCC codes for various parameters. To do this, we discussed 
about the expanding method of sub-matrix. The larger number 
of delay blocks (i.e. memories) can give the better performance 
is confirmed by simulation. We also considered the size of sub-
matrix of the parity-check matrix. From the simulation results, 
we could confirm that a sub-matrix with larger size can 
improve the code performance. At last, we conjectured that the 
similar constraint length gives similar performance from the 
simulation result.
