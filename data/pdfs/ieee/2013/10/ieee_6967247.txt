Abstract—In this paper, we propose a design method for short block length low-density parity-check (LDPC) codes which use the structure of semi-random parity-check matrix and the optimal degree distribution pairs derived from density evolution algorithm. Simulation results show that the bit error performance of the designed code is better than Turbo codes with the same block length and code rate when Signal Noise Ratio (SNR) exceeds 2.5dB. At the same time, the complexity of decoder is lower than that of Turbo codes. This design method provides guiding significance for choosing an error-correcting code for reliable communication in narrow-band channel.

I. INTRODUCTION
In a practical communication system, short block message is often needed in many cases especially in narrow-band channel as the encoding delay required being less than 100ms or even less. To ensure reliable communication under these situations, short block length error correcting codes are employed such as Turbo codes which was first proposed by Berrou in 1993[1]. Turbo codes have very good error performance at short block length and low SNRs. However, error floor and high decoding complexity and delay restrict its use in some cases where SNR changes in a large range. To solve this problem, LDPC codes re-attracted researchers’ attention.
LDPC codes, originally proposed by Gallager in 1962 in [2], have Shannon limit performance [3] and low complexity and parallel decoding procedure. With the help of density evolution algorithm, Richardson [4] showed that LDPC codes randomly designed could approach capacity performance when decoded using the iterative belief propagation (BP) algorithm with appropriate degree distribution pairs. The optimized irregular LDPC codes’ asymptote is only 0.0045dB away from the Shannon limit for the binary-input additive white Gaussian noise (BIAWGN) channel [5]. The bit error performance of LDPC codes is better than that of Turbo codes when the code length is above 104[6].

LDPC codes can approach Shannon Limit only when the code block length is very large. However, large block length will bring a large coding delay under low bit rate channel where short block codes are often needed. It is known that the BP decoding is only optimal for cycle-free codes while good finite-length LDPC codes inevitably possess cycles especially in short block cases. Several methods have been proposed to construct short block length LDPC codes with good performance at high SNRs. A selection scheme was suggested to pick a code with the largest average girth from a code ensemble by Mao and Banihashemi [7]. Y.-J. Ko and J.-H. Kim proposed a new girth conditioning scheme which may ensure the local minimum girth maximization for construction of short block length LDPC codes [8].As is known that the minimum hamming distance determines the bit error performance of channel coding schemes including LDPC codes. However, it’s a Non-Deterministic Polynomial (NP) problem to calculate the exact value of the minimum hamming distance of LDPC codes. A method has been proposed to get the approximate value in [9] which can also be used to search a good LDPC code.
The short LDPC codes constructed in [7] and [8] are irregular and randomly generated which may have large encoding complexity and will need a large storage space to save the generator matrices. In this paper, we propose a new method to generate a group of short block length LDPC codes using the semi-random parity-check matrix and optimal degree distribution pairs derived from density evolution algorithm and then choose one code with the largest average girth or the maximum minimum hamming distance from the ensemble.
The paper is organized as follows. Section II presents the structure of semi-random parity-check matrix and gives the algorithm for iterative encoding. Section III gives the method for construction of short block LDPC codes. A performance simulation of LDPC versus Turbo codes is given in section IV. Section V discusses the complexity between LDPC and Turbo codes. Finally, Sections VI concludes this paper with some remarks and suggestions of further research work.

IV. PERFORMANCE SIMULATION
In this section, the error performance comparisons between the codes designed above and Berrou Turbo codes will be presented. In narrow-band channel, the code rate should not be very small, so that a high SNR can be achieved. Considering this reason, rate 1/2 codes are chosen combined with BPSK modulation over an additive white Gaussian noise (AWGN) channel. In the simulation, the maximum number of iterations is set to 50 of LDPC codes and 10 of Turbo codes as the increasing number of iterations cannot bring the performance improvement [13]. Considering two situations that bit speed rate of the narrow-band channel is 9.6kbps and 7.2kbps and the maximum encoding delay is 100ms, then the block length of LDPC codes is 960 bits and 720 bits. Fig.2 gives the information bit error rate Pb versus Eb/N0.
Fig.2 points out that the bit error performance of short block length LDPC codes is better than Turbo codes when SNR is higher than 2.5dB of 960 bits and 720bits. PCCC Turbo codes’ performance contains error floor and this property restricts its use when SNR changes in a large range. We can also obtain that LDPC codes chose by maximum average girth perform better than maximum minimum hamming distance. This is because BP decoding is only optimal for cycle-free codes especially at high SNRs.
V. COMPLEXITY
In this section, we present the complexity between LDPC short codes designed above and PCCC Turbo codes. For LDPC codes, iterative encoding can be used because the parity-check matrix contains a bidiagonal submatrix and the encoding only need the computation of parity check bits through some XOR operations [10]. The encoding of Turbo codes is also very simple; therefore, in this section we mainly discuss the decoding complexity between the two schemes.
Considering decoder complexity, it's is known that the LDPC BP decoding algorithm is parallel, and has lower complexity than turbo codes. To simplify the iterative updating

of the bit likelihood functions, all multiplicative operations are turned to additions by employing the log-domain implementation and this approximately needs 15  2v+9 additions per decoded bit per iteration as well as some operations for table lookup for Turbo codes assuming that v is the code memory length of the elementary encoder [13]. However, LDPC codes only need 4w additions where w is the average column weight of parity-check matrix. TABLE I compares the complexity of Turbo codes using Log-MAP decoders with LDPC codes using belief propagation algorithm in which it’s obvious that LDPC decoder is simpler than turbo codes per decoded bit per iteration [12].
However, to achieve proper error performance, LDPC decoder needs considerably larger number of iterations than Turbo codes [12]. Researchers have given the result that just eight to ten iterations may be needed to get a high performance for Turbo codes [13]. But the LDPC decoding process will get early termination when the received data is successfully decoded by BP algorithm; therefore, the actual number of iterations of LDPC decoding is small when the SNRs exceed 2dB. The results shown in Fig.3 give the number of iterations versus Eb/N0 for LDPC codes.
In Fig.3, the iteration number for LDPC decoding goes down along with the increase of SNR and is lower than Turbo codes when SNR exceeds 2dB. Table Ċ gives the exact number of additions for LDPC and Turbo codes when block length is 960 bits. A conclusion can be obtained from Table Ċthat LDPC codes have lower complexity than Turbo codes when the block length and code rate is the same.
VI. CONCLUSION
Considering the need of reliable communication of short delay and low complexity in narrow-band channel, we have designed a kind of short block length LDPC codes using semirandom parity-check matrix and optimal degree distribution pairs calculated by density revolution algorithm. Using maximum average girth as the selection criteria, we pick up a short code with better bit error performance against Turbo

Comparing with LDPC short codes designed by girth conditioning, the encoding complexity is very low and a lot of storage space will be saved. Simulation results show that the LDPC codes designed in this paper can be successfully used in narrow-band channel and short block length situation. However the block length in this paper is also very large. The method mentioned in this paper cannot be used to construct a code with good error performance when the block length is around 100 bits or smaller. A possible solution is to combine LDPC short codes with Polar codes.
