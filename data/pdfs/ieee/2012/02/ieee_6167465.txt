Abstract—Motivated by the recent emergence of storage technologies characterized by the intrinsic asymmetries in data recording, this work presents a preliminary look of two simple but practical high-rate code constructions for the Z-channel. The constructions are based on transforming LDPC codes designed for the binary symmetric channel to bias the number of ones that are sent through the channel. Using simulation results, the performance of this new class of codes is compared against some existing codes, and under certain conditions, new codes are shown to be superior existing asymmetric error correcting codes.

Excellent symmetric error correcting codes and their applications abound [11]. In certain situations, however, the cross-over probabilities are quite discrepant. The most dramatic case is captured via a (binary) Z-channel where, say, the 0-to-1 error occurs with some non-zero probability whereas the 1-to-0 error is impossible. Construction 1 and Construction 2 produce more desirable codes under different circumstances. In general the ability for Construction 1 to produce a code close to the desired distribution p depends on the underlying C[n,k,q] code. If C[n,k,q] code has small redundancy then the Construction 1 code will have rate ∼ H(p) and the probability a 1 is sent is p + δ where p represents the parameter used in Construction 1. With some assumptions on the codebook (e.g. size, minimum distance), it may be possible to determine a good approximation for δ. Construction 2 allows for a more precise control over the input probability distribution, but it has the disadvantage of possibly requiring a more complex iterative decoding algorithm.
V. EMPIRICAL EVALUATION

In this section the performance on the Z-channel of symmetric and asymmetric LDPC codes is compared to some existing asymmetric codes. The section begins with a motivating example that evaluates the performance of a known (symmetric) LDPC code and some known asymmetric codes with similar code rates, over various values of the channel parameter. It is demonstrated that even a relatively short LDPC code exhibits low error rates for a fixed code rate. Based on this observation, Construction 1 is applied to a longer LDPC code to produce a code that has even better performance. The error rate of this asymmetric code is then compared to a symmetric LDPC code of equal length and similar information rate.
It should be noted that the graphs in this section are somewhat unconventional in the sense that they focus on the low SNR region. The rationale for this analysis is that the difference in capacity is negligible for the high SNR region as demonstrated in Section III, and therefore any specialized codes will likely have minimum gain. Due to the restriction on the range of possible code parameters, some simulated asymmetric codes inevitably have short block lengths to ensure equivalency in comparison in terms of the code rate.
