Exploiting the Shift Property of Structured LDPC Codes for Reduced-Complexity Sliced Message
Passing based Decoder Design

Abstract—The sliced message passing (SMP) scheme breaks the sequential tie of CN update and VN update and presents an efficient way of elevating the hardware utilization ratio and increasing throughput. However, the forward and backward permutation networks complicate the message routing and make the SMP-based decoder not amenable for codes with large CN degrees. By utilizing the shift property in some structured codes, we reduced the permutation network by almost 59% when compared with the state-of-art designs. Overall, this design technique will make SMP-based decoder more cost-efficient and applicable for the future communications and storage systems.
I. INTRODUCTION
Low-density parity-check (LDPC) codes, discovered by Gallager in 1962, were rediscovered and shown to approach Shannon capacity in the late 1990s. LDPC codes are being considered for a wide range of applications such as highdensity flash memory, satellite broadcasting, WiFi and mobile WiMAX. LDPC codes are decoded iteratively using the sumproduct algorithm and its variations such as the normalized min-sum algorithm (NMSA) [1].
Over the past several years, researchers have been revolutionizing the LDPC decoder architecture for small silicon area, high throughput, and energy efficiency. LDPC decoder architecture can be categorized into two classes: fully parallel and partially parallel. For full-parallel decoders, each row and column of the parity check matrix is directly mapped to a processing unit; all these processing units operate in parallel. In general, while fully parallel decoders provide the much higher throughput than their partially parallel counterparts, they suffer from large logic area and routing congestion because of large number of processing units and long global wires between them.
Partially parallel decoders have not drawn much attention until the invention of structured codes constructed by algebraic and combinatorial methods. The semi-random structured codes such as LDPC codes based on Reed-Solomon codes with two information symbols (also known as RS-LDPC codes) [2] and quasi-cyclic (QC) LDPC codes [3] consist of square sub-matrices which are either zero matrices or permutation

matrices (PMs). Utilizing the regular structures of structured codes, partially parallel decoders provide a good tradeoff between routing complexity and parallelism. However, the data dependency between check node (CN) update and variable node (VN) update in the decoding often results in low hardware utilization ratio for decoder design. To improve the decoder throughput, scheduling schemes are proposed either to reduce the waiting time (overlapped message passing [4]) or to break the sequential tie (sliced message passing [5]) between the check node (CN) update and variable node (VN) update.
The sliced message passing (SMP) scheme is suitable for the high-throughput partially parallel decoder for two major reasons. First, it does not require the waiting time computation and achieves almost 100% hardware utilization ratio. Second, the SMP scheme employs serial-input CN processing units (CNU), which leads to reduced complexity in logic resources, especially for high-rate LDPC codes with large row weights. However, for each clock cycle in decoding, the interconnection networks between CNUs and VN-to-CN messages (or between VN processing units and CN-to-VN messages) are different and thus require expensive implementation using large fan-in multiplexors. Since the fan-in of each multiplexor is the same as the row weight of the corresponding row in the conventional SMP-based decoder, the implementation cost could be very high when the row weight is quite large.
To reduce the implementation cost of the interconnection network, we propose to exploit the shift property of the parity check matrices for some algebraically constructed LDPC codes, e.g., the (2048,1723) code [2] for the IEEE 10GBase-T standard and the (4095,3367) Euclidian geometry (EG) LDPC code [6]. Different from [7], which utilizes the shift property of RS-LDPC codes for only circularly shifting the registers storing CN-to-VN messages for the simplified decoder design, our scheme also dynamically reconfigures the permutation network in a circular shifting manner. Therefore, our method can be extended to other structured codes, e.g., the QCLDPC codes based on the multiplicative groups of finite fields (method 1) [8] and the QC form of the cyclic LDPC codes [9].

II. REVIEW OF LDPC CODES AND DECODING ALGORITHMS
An LDPC code C of length n is given by the null space of a J × n sparse parity-check matrix H = [hi,j] over GF(2). A binary n-tuple v = (v0, v1, . . . , vn−1) is a codeword in C if and only if vHT = 0. If each column has a constant weight γ (the number of 1-entries in a column) and each row has a constant weight ρ, then the LDPC code C is referred to as a (γ, ρ)-regular LDPC code. Otherwise, the null space of H gives an irregular LDPC code.
An LDPC code C can also be represented graphically by a Tanner graph which consists of two disjoint sets of nodes. Nodes in one set represent the code bits and are called variable nodes (VNs), and the nodes in the other set represent the check-sums that the code bits must satisfy and are called check nodes (CNs). The i-th CN is connected to the j-th VN by an edge if and only if hi,j = 1.
Soft decision decoding algorithms, such as the sum product algorithm (SPA) and its reduced-complexity approximation, the normalized min-sum algorithm (NMSA) [1], are used to decode LDPC codewords. With an optimized normalization factor, the NMSA performs as good as SPA. Therefore, the NMSA is usually chosen for practical implementation. The NMSA is summarized as follows.
For 0 ≤ i < J and 0 ≤ j < n, we define Ni = {j : 0 ≤ j < n, hi,j = 1}, and Jj = {i : 0 ≤ i < J, hi,j = 1}. Let Kmax be the maximum number of iterations to be performed. For 0 ≤ k ≤ Kmax, let z(k) = (z0(k), z1(k), . . . , zn(k−)1) be the hard decision vector generated in the k-th decoding iteration, Li(→ k)j be the extrinsic message passed from the i-th CN to the j-th VN, Li(← k)j be the extrinsic message passed from the j-th VN to the i-th CN, and Lj(k) be the reliability value of the j-th code bit. III. REDUCED-COMPLEXITY SMP-BASED DECODER DESIGN

A. Structured LDPC Codes and Shift Property

Structured LDPC codes simplify wiring between CNUs and

VNUs and allow partially parallel processing, and thus are

easier to implement compared with random codes. QC-LDPC

codes and RS-based LDPC codes are two most important

classes of structured codes. They are chosen by multiple

communication and storage standards. For example, DVB-

S2, mobile WiMAX, and 801.11n standard use QC-LDPC

code, while 10GBase-T standard uses RS-based (2048,1723)

code [2]. Commonly, the parity-check matrix of a structured

code is given as a γ × ρ array (or block) of submatrices

of the same size, say m × m. For QC-LDPC codes, these

submatrices are either circulant permutation matrices (CPMs)

and/or zero matrices. For RS-based codes, these submatrices

are permutation matrices (PMs).

Some subclasses of the structured codes, especially those

constructed based on the finite field theory, usually facilitate

decoder design. For example, QC-LDPC codes based on the

multiplicative groups of the finite fields (Method 1 of [8]),

the quasi-cyclic form of the cyclic LDPC codes [9], and the

RS-LDPC codes. These codes contain a decoder-friendy shift

property shown below. For example, the H matrix of a code

C1 with γ = 3, ρ = 6, m = 3 can be shown below and will be used to explain the reduced-complexity decoder.

The decoder has γm 1-in(out)put CNUs and m γ-in(out)put VNUs. There are two banks of registers to store the CN-to-VN messages of the previous iteration and the current iteration. Each register contains the minimum magnitude, the second minimum magnitude, and index of the minimum magnitude, and the sign bit. This compressed message storage scheme can achieve about 70% memory reduction for high-rate LDPC decoders [11].
As in different clock cycles, the interconnection networks of the CNs with VNs are different, CN-to-VN messages are reordered in group of m by the forward permutation network and passed to the VNUs. Similarly, after the VN update, the generated VN-to-CN messages will be reordered by the backward permutation network and passed to the CNUs, which compute the corresponding compressed version of check-tovariable messages iteratively. There is no need of storing VNto-CN messages since they would be computed and used right away. Thus, in each clock cycle, m columns are concurrently processed by m VNUs, and the computed VN-to-CN messages correspond to γm different rows. After receiving one Lvc message, each CNU can perform one step of the row process. Each row process is divided into ρ steps and is interleaved with the column processes. We can see that each iteration will be completed in ρ cycles without considering pipelining.

In this section, we present two case studies for two important LDPC codes: the (2048,1723) RS-LDPC code for the 10GBase-T Ethernet standard and the (4095, 3367) EG-LDPC code.

V. CONCLUSION
The sliced message passing scheme breaks the sequential tie of CN update and VN update and presents an efficient way of elevating the hardware utilization ratio and increasing throughput. However, the forward and backward permutation networks complicate the message routing and make the SMPbased decoder not amenable for codes with large CN degrees. By utilizing the shift property in some structured codes,

we reduced the permutation network by almost 59% when
compared with the state-of-art designs. Overall, this design
technique will make SMP-based decoder more cost-efficient
and applicable for the future communications and storage
systems.
