Additional Check Node to Improve the Performance
of LDPC Codes in the Error Floor Region
Kuntal Deka, A. Rajesh and P. K. Bora
Indian Institute of Technology Guwahati, India, email:{kuntal, rajesha and prabin}@ iitg.ac.in.
Abstract—This paper presents a new scheme to reduce the
detrimental effect of the trapping sets in the regular and irregular
low density parity check (LDPC) codes. By adding only one new
check node and properly selecting its edges, the performance in
the error floor region can be improved. Simulation results show
that the proposed scheme decreases the bit error rate (BER) and
frame error rate (FER) significantly in the high signal-to-noise
ratio (SNR) region at the expense of negligible rate loss.
Index Terms—Low density parity check (LDPC) codes, error
floor, trapping sets.
I. INTRODUCTION
The error correcting codes involving iterative decoding
schemes, like turbo and LDPC codes, suffer from the error
floor problem. The BER and FER curves for such codes show
a sudden decrease in the slope that occurs at moderate to high
signal-to-noise ratios (SNR). This flattening of the error curves
stands as a limiting factor for the successful implementation
of several LDPC codes in practical systems.
The error floor problem of LDPC codes is caused by
some combinatorial substructures of the parity check matrix.
Depending upon the channel and the way of analysis, different
authors have come up with different names for these combina-
torial substructures. For the binary erasure channel (BEC), the
cause of error floor is ascribed to the stopping sets [1]. For the
other channels like the binary symmetric channel (BSC) and
additive white gaussian noise (AWGN) channel, the concept
of stopping set is replaced by that of the trapping set (TS)
[2], the pseudo-codeword [3], the near codeword [4] and the
absorbing set [5]. However, from the point of view of graph
theoretic definition , all other terms may be classified as some
special types of TSs.
This paper presents a new method to reduce the detrimental
effects of TSs in LDPC codes. The rest of the paper is
organized as follows. The decoder failure caused by the
trapping sets is explained in Section II. Section III presents
the literature survey. In Section IV, the proposed method
is discussed. Section V presents the simulation results and
Section VI concludes the paper.
II. TRAPPING SETS IN LDPC CODES
Definition [6]: A (𝑎, 𝑏) trapping set is a set of 𝑎variable
nodes (VNs) which induces a subgraph with 𝑏odd-degree
check nodes (CNs) and an arbitrary number of even-degree
CNs.
In Figure 1, a (5, 1) TS from (96, 48) code available at
[7] is shown. As obvious from Figure 1, the CNs in the TS
induced subgraph can be divided into two groups: even-degree
and odd-degree CNs. If all the bits or VNs in the subgraph
are detected incorrectly then the even-degree CNs will be mis-
satisfied and the odd-degree CNs will be unsatisfied. During

III. LITERATURE SURVEY
A large number of techniques have been proposed by several
researchers to improve the performance of LDPC codes in the
error floor region. These techniques can be broadly classified
into two groups: construction based and decoder based.
The construction based strategies can be further divided into
two subgroups: direct and indirect. The direct construction
strategies take care of TSs directly during the construction
process. In [8], a construction method of a longer code from
an original code via the graph cover is proposed for removing
some known trapping sets. In [9], a code annealing algorithm
is proposed to prevent small stopping sets by swapping edges
in the Tanner graph. This method is further extended for small
TSs in [10]. The indirect strategies take care of TSs through
cycles. It is well known that dominant TSs involve cycles.
So, by controlling the cycles present in the Tanner graph, the
effect of TSs can also be controlled. Hu et al proposed the
progressive edge-growth (PEG) construction method which
maximizes the local girth at the variable nodes on an edge
by edge basis [11]. In [12], a measure of the connectivity
of the cycles, ACE (Approximate Cycle Extrinsic Message

version of the PEG algorithm (Generalized ACE constrained
PEG) is proposed. This method constructs the parity check
matrix according to the PEG algorithm but with additional
care to improve the ACE properties.
Some decoder based strategies have also been employed.
In [14], a post-processing technique is proposed to avoid pre-
known failures caused by some dominant TSs. Zhang et al
have proposed another post-processing technique which biases
the reliabilities of some messages selected by exploiting the
structure of the absorbing sets [15]. Landner and Milenkovic
have proposed an averaged decoding algorithm which de-
creases the frequency of TS induced error events by averaging
the messages [16]. Han and Ryan [6] have proposed several
decoder based techniques such as bi-mode syndrome-erasure
decoder, outer code concatenation combined with bit pinning
and a generalized decoder.
Deviating from the construction based strategies mentioned
above, several researchers have used the technique of aug-
menting the parity check matrix by adding some extra rows
for better performance. References [17], [18] and [19] provide
some lower and upper bounds for the minimum number of
rows in a parity-check matrix of a code in order to remove
all the stopping sets having sizes less than the minimum
distance of the code. In [20], some bounds for the smallest
number of rows of a parity-check matrix are derived so that
the code does not contain TSs having sizes less than some
fixed number. Recently a scheme for the extension of the
parity-check-matrix is proposed to eliminate stopping sets of
small sizes [21]. In this scheme, some extra rows are added so
that the difference between the contribution to decoding errors
from the eliminated stopping sets and reactivated stopping sets
is maximized. Reference [22] proposed a simple and effective
method to break small stopping sets in LDPC codes by adding
several new check nodes (CNs) and reported improved error
rate performance on the BEC. In the similar line, we propose
a practically feasible strategy for improving the error rate
performance in the error floor region for the case of the AWGN
channel by weakening some TSs via adding only one new CN.

The above idea can work properly only if the reliability
of the extrinsic message from the additional CN is high. For
that, the selection of the other neighboring VNs is very crucial.
Suppose RV is the set of VNs which are not present in any
of the TSs. Thus RV is the set of the most reliable VNs. We
want 𝑑𝑐−1 of these VNs to be connected to the additional CN,
where 𝑑𝑐is the degree of a CN in the case of regular codes.
In the case of irregular codes, 𝑑𝑐is taken as the majority CN
degree in order to ensure that the degree distribution profile
does not change drastically.
There may be three cases:
∙Case I (∣RV∣= 𝑑𝑐−1):
In this case, the number of elements in RV is just
sufficient. So all the VNs of RV are selected and the
job is done.
∙Case II (∣RV∣< 𝑑𝑐−1):
In this case, the number of elements in RV is less than
sufficient. So we have to also consider further level(s) of
reliable bits i.e. the VNs which are present in the list of
TSs once, twice etc..
∙Case III (∣RV∣> 𝑑𝑐−1):
In this case, we are left with too many choices. It is a well
known fact that a subgraph in the Tanner graph may be
detrimental if its connectivity with the rest of the graph
is very poor [12]. Taking this fact into consideration,
the selection space may be reduced. For that, the Tanner
graph is expanded into a tree by taking 𝑣𝑓as a root node
and then the most distant members of RV are selected.
In that way the connectivity of the TSs involving 𝑣𝑓with
the rest of the Tanner graph is also improved.
In selecting the VNs, one has to also ensure that the addition
of the extra row to the parity check matrix does not reduce
the girth of the original Tanner graph. A detailed procedure
of selection of all the VNs is presented in the Algorithm 1.
V. SIMULATION RESULTS
The proposed method of adding an extra CN and assigning
edges to the set of selected VNs is simulated for one regular
code and one irregular code over the AWGN channel. The
regular code considered is (96, 48) Mackay code (96.33.964)
available in [7]. For the irregular case, an 𝑁= 504, 𝑀= 252
code is constructed by the PEG algorithm with the degree
distribution provided in [24]. The list of TSs is prepared using
the method reported in [14].

just to include only the dominant TSs. It is observed that the
91st bit is the most frequent bit involved in the TSs. This code
is a regular code with 𝑑𝑣= 3 and 𝑑𝑐= 6. The remaining 5
VNs selected by the Algorithm 1 are 16, 35, 57, 63 and 72.
As a result of the addition of one row to the parity check
matrix, the rate of the code is decreased from 0.5 to 0.4896.

VI. CONCLUSIONS
This paper proposed a method for reducing the detrimental
effect of TSs by the addition of one new CN and connecting
it to appropriate VNs. Since only one CN is added, the
rate penalty is negligible. Simulation results showed that the
proposed method can improve the performance of both the
regular and irregular codes in the high SNR region.
