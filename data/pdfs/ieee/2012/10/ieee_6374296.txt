Abstract—Spatially-coupled codes have low-complexity structures and a low-delay decoding architecture, which nominate them as a very desirable candidate for communication standards. In this paper, a study on the asymptotic performance of spatiallycoupled ensembles over the binary erasure channel (BEC) with density evolution analysis based on the belief-propagation algorithm is conducted. We identify all possible members of base matrices with minimal coupling (corresponding to a code memory of one and two) for asymptotically (3, 6)-regular spatially-coupled codes. For ensembles with single memory, block decoding and window decoding thresholds are calculated and for ensembles with memory of two, window decoding thresholds are calculated. It is shown that not only block decoding thresholds of finitelength spatially-coupled ensembles over BEC is better than LDPC block codes, but also are their window decoding thresholds. Based on the results, spatially-coupled code ensembles with the best performance are identified and some practical code design criteria are presented.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are a family of capacity-approaching error correcting codes that were introduced by Gallager in early 1960s but they were forgotten for several decades until Mackay and Neal rediscovered them in 1996, [1], [2]. It has been shown that LDPC codes can operate within hundredths of a decibel to the Shannon limit with sufficiently large block lengths under belief propagation (BP) decoding on additive white Gaussian noise (AWGN) channels, [3]. A binary LDPC code, just like any other linear block code, can be fully described by a parity-check matrix H which is a sparse binary matrix. The code is called (J, K)-regular if H has a fixed row weight K and a fixed column weight J such that the relation m × K = n × J holds, where m and n represent number of rows and columns of H, respectively. Both K and J are very small compared to M and N . For an irregular LDPC code, row and column weights can vary. An LDPC code can be represented in terms of a bipartite graph (called the Tanner graph) in which variable nodes correspond to code symbols and the check nodes correspond to the set of parity-check equations.

There are several decoding algorithms used for the decoding of LDPC codes in the literature, including the hard decision (bit flipping) and soft decision (sum product algorithm, minsum algorithm) algorithms, each of which gives different tradeoffs between error rate performance and decoding complexity. Soft decision algorithms are preferred when high error performance is required and the hard decision alternatives are preferred when the computational resources are very limited at the decoder side. Regardless of the type of information they use, these algorithms are called iterative decoding algorithms (also known as message-passing algorithms) and are realized by exchanging messages between check nodes and variable nodes along the edges of the graph, in both directions and in an iterative manner [1].
In any iterative decoding algorithm, an important assumption is that a message sent from a node along an adjacent edge may not depend on the message previously received along the same edge. This guarantees that only extrinsic information is passed along the edges, which, in turn, improves the decoding performance. Messages exchanged on the bipartite graph travel in both directions: messages from variable node j to the check node i is the LLR value of the jth variable node, given the LLR values of all checks involving the variable node, except ith check node itself. Messages from check node i to variable node j is the LLR value of the jth variable node, given the LLR value of all variables checked by the ith check node except the jth variable node itself. In the first half of an iteration, all variable nodes send a message containing their LLR values to their connected check nodes. Note that for the first iteration, the only information available to a variable node is the received LLR from the channel. In the second half of an iteration, every check node calculates a response to every connected variable node. The response message contains the information that the check node believes to be the correct decision regarding the parity check equation for a specific variable node. If all check equations are fulfilled (or a maximum number of iterations is reached), then the decoding process terminates. At the final stage of an iteration, the variable nodes receive the messages from different check nodes and use this additional information to decide if their originally received value was the correct one. This operation is repeated until all parity-check equations are satisfied or until a maximum number of iterations is reached [1].
In this paper we consider a specific LDPC code family, named spatially coupled (SC) codes and evaluate their asymptotic performance over BEC by density evolution analysis. We propose code selection criteria for SC codes that are candidates for practical communication applications.
II. LDPC CONVOLUTIONAL CODES AND SC CODES
The convolutional counterparts of LDPC block codes are called LDPC convolutional codes. LDPC convolutional codes are similarly defined by sparse parity-check matrices that enable us to perform the decoding based on iterative messagepassing algorithms on them too [4]. Their main difference compared to LDPC block codes are the ribbon-shaped nonzero region they possess in their parity-check matrices. This allows them to be used to construct a family of codes with variable frame lengths and low-complexity encoding/decoding. Recent studies have shown that LDPC convolutional codes are indeed suitable for practical implementation in different communication scenarios, such as packet based communication systems with variable frame lengths [5]. A protograph is a small bipartite graph from which a larger graph can be obtained by a copy-and-permute procedure [6]. The protograph is first copied M times, and then the edges of the individual copies are permuted to obtain a single, large bipartite graph referred to as the derived graph. M is called the expansion factor. Fig. 2 demonstrates this copy-and-permute operation.

For a base matrix in this form, the copy-and-permute operation is equivalent to replacing each entry Bi,j in the base matrix with a sum of Bi,j permutation matrices of size M × M . As a result, in (8), each of the entries 1 and 2 is replaced by an M × M matrix such that each column and row contains only 1 and 2 ones, respectively. Note that, due to the special structure presented in (4) and (8), the first and last ms rows of the base matrix, or, equivalently, the first and last M ms check nodes of the constructed SC code have weight lower than the rest of the graph. Therefore, we are not able to call these codes (J, K)regular anymore, even if a big portion of the parity-check matrix satisfies the regularity condition. Instead, SC codes with parity-check matrices that differ slightly from a regular parity-check matrix are referred to as asymptotically (J, K)regular, meaning that if we neglect the first and last portions of irregularities in H, constructed from B[L], the remaining matrix can be called (J, K)-regular. This slight irregularity of SC codes resulting from the termination of the underlying LDPC convolutional code has been shown to be the reason for SC codes having better BP decoding thresholds compared to corresponding block code ensembles [8].

V. WINDOW DECODING OF SC CODES
The special structure of the parity-check matrix of an SC code allows the use of a window decoder that breaks down the BP decoding scheme into a series of decoding steps. A window decoder with a window size of W performs BP decoding over a portion of the codeword where the window consists of W × M rows and all connected columns. The symbols to be decoded within a window are referred to as the targeted symbols. After a successful decoding of the targeted symbols is done or when the maximum number of iterations has been achieved, the window slides over another portion to the right and again performs BP to decode the targeted symbols in the

The most desirable advantage of window decoding of SC codes is the simplicity of the structure and flexibility of code construction and decoding processes. In this method, decoding is done in a window which contains only a portion of the codeword, thus the delay will be lower since decoder does not have to wait for the whole codeword to reach the receiver. The window decoder starts decoding as soon as it receives the codeword portion corresponding to the first window position. Since low delay is a very desirable feature for many wireless communication applications, these codes can be very suitable candidates in communication scenarios where low latency is the primary design criterion. SC codes also allow decoding with variable frame lengths and variable maximum number of iterations for specific frames, which is also a very desirable feature for wireless applications. We can exploit this flexibility to design window decoders adjusted to use on wireless channels. If the fading is strong, the number of iterations is set to maximum and if the fading effect is weaker, the number of iterations is decreased to speed up the decoding process and reduce the delay. Furthermore, it has been studied in [9] that the asymptotic performance of window decoding of protograph-based SC codes is superior to the performance of block decoding of regular LDPC codes over the BEC and AWGN channels; a result that could possibly be extended to the fading channels.

Some rules for designing SC ensembles were presented in

the literature before [7]. However, these rules just emphasize

the need for having large weights on the left side of a window

to make sure that reliable messages are obtained early in the

decoding process and passed on to the following window

positions. In this work, we attempt to introduce additional

practical rules based on actual DE analysis results to obtain

low-delay and low-complexity SC communication systems.

Satisfying all of these rules may not be feasible but they can

be used as rules of thumb by a code designer.

• Ensembles with ms = 1, while having less complexity and decoding delay, show better performance in general. However using ensembles with ms = 2 increases the degree-of-freedom of the code construction process, allowing for additional code construction rules, such as

those for modifying the cycle/trapping set/absorbing set spectra, to be satisfied more freely. • If an ensemble description has one of summations (a1 + a2) or (ai−1 + ai) equal to zero, there is an unnecessary rate loss. In this case, it is better to use an ensemble with smaller code memory. • It is helpful to choose a1 or a2 ≥ 2. The best case is a1 = a2 = 2, if possible. This follows from the rules given in [7] and is verified here. • For ms = 2 ensembles, if max(a3, a4) ≥ 2, choosing max(a1, a2) > max(a5, a6) improves the performance. • For ms = 2 ensembles, if max(a3, a4) < 2, choosing ai such that (a1 + a2) − (a5 + a6) is minimized improves the performance.
Note that there may be some exceptions in the tables, but satisfying these rules in general, yields an SC ensemble with good characteristics.
VIII. CONCLUSION
In this paper, performance of asymptotically (3,6)-regular spatially-coupled LDPC ensembles with minimal coupling has been studied and their window decoding thresholds by using density evolution analysis were calculated. SC ensembles with minimal coupling enjoy the simplicity of structure and lowdelay scheme of window decoding and these features introduce them as an efficient candidate for communication standards.
Based on the results, ensembles with the best performance have been identified and some detailed good-ensemble design criteria for practical communication applications are proposed.
