Abstract—In this paper, we propose an efficient method for computing the weight spectrum of LDPC convolutional codes constructed from quasi-cyclic block codes. In the proposed method, we reduce the memory size of their parity-check matrices with the same distance profile as original codes, and apply a forward and backward tree search algorithm to the parity-check matrices of reduced memory. We show numerical results of computing the minimum free distance and the low-part weight spectrum of LDPC convolutional codes of memory about 130.

check matrices with reduced memory and the same distance profile as origianl codes. In the proposed method, a forward and backward tree search algorithm is applied to the code tree of these parity-check matrices, and compute the weight spectrum of given LDPC convolutional codes. Furthermore, we show numerical results of computing the weight spectrum of LDPC convolutional codes of memory 21, 57, 126 and 134.

I. INTRODUCTION
Low-density parity-check (LDPC) convolutional codes are a class of LDPC codes defined by convolutional parity-check matrices. Felstro¨m et al. proposed a construction of LDPC convolutional codes and showed simulation results of the performance of the codes decoded by the iterative algorithm [1]. In [2], Tanner et al. showed the performance comparison between LDPC block and convolutional codes based on circulant matrices of quasi-cyclic (QC) codes, and reported that LDPC convolutional codes exhibit better performance although the codes are defined by similar constraint graphs.
The minimum free distance and the weight spectrum of LDPC convolutional codes are important factors to evaluate their performance in error floor region. The error probability can be predicted by the low part of weight spectrum. In the analysis of LDPC convolutional codes, the lower bound on the free distance have been derived for some LDPC convolutional code ensembles [5][6].
To evaluate the distance profile of convolutional codes, the BEAST and the syndrome BEAST ware proposed in [8] and [9]. These algorithms efficiently enumerate the number of codewords of a given weight w in the code tree for a given convolutional code. These algorithms must store all state of codeword paths of weight w/2 for searching forward and backward trees, and must match each state in the forward search with one in the backward search. Thus, the BEAST and the syndrome BEAST are not useful for LDPC convolutional code of large memory.

As efficient algorithms for computing the weight spectrum of convolutional codes, the BEAST and the syndrome BEAST are presented in [8] and [9]. These algorithms are tree search algorithms to enumerate the number of codewords of a given weight w. These algorithms store all state of codeword paths of weight w/2 in the forward and backward search, and compare each state stored in the forward search with that in the backward in order to find codewords of weight w. When the BEAST and the syndrome BEAST are applied to convolutional code of large memory, these algorithms consume huge workspace on the RAM in order to store the nodes obtained in the forward and backward tree search. So, these algorithms are not useful for LDPC convolutional code of large memory. In fact, we were not able to compute the weight spectrum for H3(D), H4(D) and H5(D) using both the BEAST and the syndrome BEAST. Furthermore, in order to compute the weight spectrum using either the BEAST or the syndrome BEAST, we have to investigate each spectral component Adfree , Adfree+1, . . . , Adfree+N .
We compare the performance of our proposed method with the BEAST and the syndrome BEAST. Table II shows the number of extended nodes when these algorithms enumerate the number of codewords of weight 24 in the code tree of H2(D). In Table II, the number of nodes extended by our method is smaller than that by the BEAST and the syndrome BEAST. The BEAST show good performance when the minimal basic generator matrices are used. Although the BEAST is applied to the minimal basic generator matrices, the BEAST is not faster than our proposed method.
Similar to our method, the literature [10] reported an error/code-trellis reduction approach for convolutional codes

using shifted error/code-subsequences. However, they did not present a systematic procedure to reduce the constraint length of H(D). Further they discussed only how to construct error/code-trellis of convolutional codes, and did not give an application to computing the distance spectrum.
VI. CONCLUSION
In this paper, we have proposed an efficient method for computing the weight spectrum of LDPC convolutional codes constructed from QC block codes. In the numerical result, we applied the proposed method to LDPC convolutional codes of memory 21, 57, 126 and 134, and computed the minimum free distance and the weight spectrum of these codes.
