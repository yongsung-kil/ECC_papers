ABSTRACT
Increasing the pipeline depth of bit-parallel message-passing lowdensity parity-check (LDPC) decoders can be done by increasing the number of simultaneously decoded frames, or by decreasing the amount of parallelism in the decoder. In digit-serial decoders, the pipeline depth can also be increased by increasing message precision.
Digit-online decoders are a class of digit-serial decoders that use redundant notation to allow for most-significant-digit-first processing of LLR messages. This paper examines the effect of changing the precision of LLRs on the throughput, area and energy efficiency of digit-online decoders for the irregular WiMAX rate 3/4A length 1056 block code. Both single-frame and frame-interlaced decoding are considered. Digit-online decoders have a throughput that is largely independent of message precision. Frame-interlaced decoding has a higher throughput than single-frame, but has an increased energy/bit cost.
Index Terms— Forward error control (FEC), block codes, iterative decoding, belief propagation
1. INTRODUCTION
Low-density parity-check (LDPC) codes are a class of errorcorrecting codes first proposed by Gallager in [1]. They have good error performance and their decoders are simple and can be highly parallelized. They can be decoded using iterative message-passing algorithms.
Decoders that pass messages in a bit-parallel manner require a large number of interconnect wires. This, combined with the random nature of many codes, has meant that some ASIC implementations are area-inefficient. In [2] it is shown that for bit-parallel decoders, wiring overhead generally increases as parallelism increases. The area of the decoder reported in [3] was determined more by wiring congestion than logic area.
In an attempt to alleviate wiring concerns, bit-serial decoders pass messages serially between computation nodes [4–6]. However, since two’s complement addition cannot be done most-significantbit-first with low latency, the order bits are passed in must change at each node and continuous processing is not possible.

Digit-online LDPC decoders were introduced in [7] and use redundant signed-binary notation to allow continuous most-significant digit (MSD) first processing of messages. In this paper, we extend this work by examining in detail the effect of changing message precision on the area, throughput, and energy/bit of digit-online LDPC decoders. Section 2 explains the algorithm used to decode LDPC codes. Section 3 provides an introduction to digit-online arithmetic and section 4 applies this to LDPC decoding. We analyse the effects of varying message precision in digit-online LDPC decoders in section 5 and conclude in section 6.

2. LDPC DECODING

A two-stage message-passing algorithm can be used to decode LDPC codes. The messages are log-likelihood ratios (LLRs) representing a priori probabilities. Each stage of processing is performed by one type of processing node: variable nodes or check nodes. To begin decoding, each variable node is initialized with LLRs from the channel. Each node passes out a set of messages based on how likely it believes each bit is a one or a zero. Decoding is finished when all code constraints are satisfied or a set iteration limit is reached.

3. MSD-FIRST DIGIT-ONLINE ARITHMETIC
Our discussion of digit-online arithmetic is motivated by the operations needed to build an offset min-sum decoder. Addition is required for the variable node, and sign detection is required for the hard decision. The check node requires finding the sign and magnitude of operands, finding the minimum of two operands (compareselect), subtraction of an offset, and multiplication by ±1.
To do efficient digit-by-digit serial processing on a data stream, each output digit should be calculated as independently as possible from other output digits. Additionally, the output digits should be generated after receiving as few input digits as possible. The processing delay should be fixed and smaller than than the number of digits. Operations that satisfy these requirements are called digitonline [10–12]. These requirements ensure that the operation is done with minimum latency and minimum memory and allow for deep pipelining at the digit level.
If the serial operands are in conventional binary notation, it is impossible to satisfy these requirements [11]. If most-significant-bit (MSB) first processing is chosen, compare-select can be done with low latency, but addition requires a latency equal to the precision. If least-significant-bit (LSB) first processing is done, low-latency addition is possible but compare-select requires a large latency.
To solve this problem, a redundant number system is required where operands are stored using generalized digits rather than bits. This allows MSD-first addition to be performed with low latency and hence allows all calculations to be done in an MSD-to-LSD manner. One such redundant number system is signed-binary [13], which is used in our digit-online decoder. Signed-binary notation uses the digit set {1¯, 0, 1}, where ¯1 represents the value −1. This digit set allows for multiple representations of the same value due to the fact that borrows do not have to be fully propagated. As an example, 11¯ = 01, because the digit ¯1 borrows from the position before it.
Signed-binary numbers require two bits per digit to store, but do not require a separate sign digit. The sign of the number is determined by the sign of the most-significant non-zero digit since lesssignificant digits do not have a large enough magnitude to change the sign [11, 13]. Hence signed-binary numbers require one fewer digit than two’s complement to represent the same precision, albeit using more total bits.

4. DIGIT-ONLINE LDPC DECODING
This section contains only a brief overview of the digit-online LDPC decoder architecture. A full discussion of the architecture can be found in [7].
A system diagram of an digit-online LDPC decoder is shown in fig. 1. The variable nodes implement equation (1) with a simple tree of digit-online signed-binary adders.
The check node architecture is based on equation (2) and is shown in fig. 2. Inputs are split into their sign and magnitudes, and offset correction is performed. The corrected magnitudes are partially sorted using a selection network made up of compare-select modules to find the two smallest magnitudes. This allows the extrinsic principle to be satisfied. The output corresponding to the smallest input magnitude is given the second-minimum magnitude. All other outputs are given the smallest magnitude. The output signs are determined by an XOR tree and multiplied by the output magnitudes to give the output LLRs.
Messages flow in loops between variable nodes and check nodes. The full length of the decoding pipeline is equal to the length of the variable node pipeline (δv) plus the length of the check node pipeline (δc). Since one digit is stored in each pipeline stage, the maximum number of digits in a message l = δv+δc. However, some of these digits are used as guard digits to prevent overflow in the variable nodes. For dv,max = 4, 2 guard digits are required and for dc,max = 6, 3 guard digits are required. This overhead is required on each frame in the decoding pipeline.

WiMAX rate 3/4 base matrix A (which we refer to as rate 3/4A for short) codes have dv,max = 4, so the maximum useful precision is p1f = l−2 for single frame decoding, and p2f = l/2−2 for frameinterlaced decoding. A p-digit signed-binary number can store a pbit signed number without requiring an additional sign bit, so it is related to the channel quantization q by the equation q = p+1. Thus our pipeline can decode one frame at (q1f = l −1)-bit precision or two frames at a (q2f = l/2−1)-bit precision.
The chosen code requires variable nodes of degrees 2, 3 and 4 and check nodes of degree 14 and 15. The selection networks of the degree-14 and degree-15 check nodes are both 6 levels deep. The degree-14 check node requires 25 comparisons while the degree-15 check node requires 27 comparisons. These are the optimal numbers of comparisons for (2,14) and (2,15) selection networks [14].
To pipeline bit-parallel decoders more deeply, either more frames must be decoded at once or a partially parallel architecture must be used. Increasing the number of frames increases the complexity and size of the input buffers and control logic and is not always practical.
Digit-online decoders can also benefit from decoding multiple frames at once, but can also increase the amount of pipelining by increasing the precision of the messages. Higher precisions require more clock cycles per iteration but result in shorter delays in each pipeline stage.

5. RESULTS
To demonstrate the effects of message precision on decoder performance, several fully parallel decoders for the irregular WiMAX rate 3/4A length 1056 code were synthesized in a TSMC 65 nm general purpose process1. All results are reported for nominal process corners and the conservative wiring model. Automatic register retiming was enabled to balance the pipeline stages without designer intervention.
Decoder parameters are shown in table 1, and pipeline details are shown in table 2. Because of result forwarding between pipeline stages, there are many paths around the processing loop with fewer than the maximum number of registers. At the minimum pipeline depth of 4 for a degree-4 variable node there are combinational paths between input and output LLRs.
To ensure synchronization of values, the degree-2 and degree-3 variable nodes are pipelined to the same length as the degree-4 variable nodes and each will always have at least one register between input and output. The check node requires forwarding one stage ahead due to the offset subtraction.
Given these minimum forwarding values, the minimum number of registers in the processing loop can be found by subtracting 5 from the pipeline length. Thus a minimum pipeline length of 6 was chosen to ensure that each path contained at least one register. Additionally, 5 bits of precision are required for the optimal offset constant β to be greater than zero at efficient channel scaling.
Published WiMAX decoders generally use a channel quantization between 5 and 8 bits [15–20]. There is little improvement to error performance beyond 8 bits [20]. Decoders with 12 or more pipeline stages are included to obtain frame-interlaced results at higher precision. It is unlikely that 11 or more bits of precision would be used in practice.
Likewise, since error performance drops off sharply below 5 bits of precision, frame-interlaced decoders for less than 5 bits are included for comparison only and will not provide acceptable error performance.
Each decoder pipeline was synthesized over a large range of clock frequencies. The decoder with the highest throughput/area for each pipeline was chosen to present in this paper. Increasing the pipeline length past 20 increased the synthesis times greatly and made it infeasible to generate enough synthesis results to draw meaningful conclusions from.
Even at the pipeline lengths synthesized we are approaching the limits of clock frequency for an edge-triggered design. At higher number of pipeline stages the register clock→q and setup times become an increasingly large proportion of the clock cycle. About 100 ps of the critical path is consumed by flip-flop setup and clock→q times, which accounts for more than 20% of the delay at high frequencies.
The important decoder performance characteristics of LDPC decoders are throughput, the throughput/area ratio (TAR), and energy efficiency.

5.1. Throughput and Area
The throughput for each of the synthesized decoders is shown in fig. 3. As the message precision increases, the number of clock cycles per iteration also increases. However, the depth of the pipeline increases as well, allowing the clock cycles to shorten. Thus, for single-frame decoding with message precisions from 6 to 15 bits the throughput is relatively constant. The effect on throughput is magnified for frame-interlaced decoding, but throughput is relatively flat for 3 to 7 bits of precision. Using frame-interlaced decoding results in much higher decoding throughput for a given precision.
Fig. 4 shows the effect of changing message precision on decoder area. There is a weakly linear relationship between message precision and decoder area for single-frame decoding which is magnified for frame-interlaced decoding. The 6-stage and 7-stage

decoders actually requires more area than the 8-stage decoder due to the low number of registers in the worst-case loops. Frameinterlaced decoders are larger than single-frame decoders for a given precision which is part of the cost for their increased throughput.
The breakdown of the area for single-frame decoding is shown in fig. 5. The size of the control logic is dominated by the size of the input buffer. While the area of the control logic increases with precision, the greatest area increase is accounted for by the greater number of pipeline stages in the computation nodes. For larger numbers of pipeline stages the size of the input buffers could be decreased by allowing only frame-interlaced decoding. However, the area savings would not be significant.

6. CONCLUSION
In this paper we have examined the effects of varying message precision in digit-online LDPC decoders for the irregular WiMAX rate 3/4A length 1056 code. We have shown that in many cases there is no throughput cost to increasing the message precision, and decoder area grows slowly with precision (e.g. tripling precision, 5 to 15 bits, results in a 1.16x increase in area). However, the cost in decoding energy efficiency is significant and must be considered when designing a digit-online decoder.
We have demonstrated that frame-interlaced decoding is superior to single-frame decoding when it comes to throughput and throughput/area. However, frame-interlaced decoding comes at a higher energy/bit cost, so designers must consider whether the increased power consumption is acceptable.
