Improved Weighted Bit Flip Voting Decoding Algorithm for Generalized LDPC

Yibin Yang, Jintao Li, Hui Yu, Youyun Xu
Department of Electronic and Electrical Engineering Shanghai Jiao Tong University, P. R. China
Email: yangyibinde @126.com, Jintao_li @139.com, yuhui @sjtu.edu.cn, yyxu @vip.sina.com

Abstract—A generalized low-density parity-check code (GLDPC) is a low-density parity-check code in which the constraint nodes of the code’s graph are replaced with a small linear block code referred to as the constituent code other than single parity checks. For the optical system or ultra-high speed wireless data transfer system, it is hard to gain soft LLRs which need to be supplied by ultra-high speed A/D converters, so the hard decision decoder is important for these systems. Similar with Gallager’s bit-flipping algorithm for LDPC, the Weighted Bit Flip Voting (WBFV) algorithm has been proposed to decode GLDPC with only the hard decision which proved to be good performance. But the finding of max vote in the algorithm is intractability for the hardware implementation. If we find the max from thousands of data (code length is usually thousands) serially, thousands of cycles needed result to large latency; a lot of compare modules needed for parallel finding of the max, result to large resource. In this paper, we proposed an improved and simplified WBFV algorithm which without max voting finding is easy for hardware implementation. The proposed improved algorithm not only has no performance loss, but also speeds up the iteration convergence. Then we proposed a full parallel GLDPC decoder design for our algorithm with very low complexity.
Index Terms—Generalized LDPC, hard decision decoder, Weighted Bit Flip Voting.
I. INTRODUCTION
Low-Density-Parity-Check (LDPC) codes were originally proposed by Gallager [1] in the early 1960s, but it was hard to implement then, and was forgot for several decade. Until the 1990s, it was resurrected for its excellent performance [2] [3]. As an evolution of classic LDPC codes [1], Generalized LowDensity-Parity-Check (GLDPC) codes were introduced by Tanner [4] and then Hamming-code based GLDPC codes were further explored by Boutros et al.[5] (B-GLDPC) as well as by Lentmaier and Zigangirov [6] (LZ-GLDPC). GLDPC codes are constructed by replacing the constraint nodes of the LDPC code’s graph with a small linear block code referred to as the constituent code other than single parity checks. It has been shown that Hamming-code based GLDPC codes are asymptotically good in the sense of minimum distance and exhibit an excellent performance over both AWGN and Rayleigh channels [5][6][7]. When properly designed, GLDPC codes have very large minimum distance and exhibit performance approaching Shannon’s limit. Besides, with a benefit of high flexibility in terms of the selection of code

length and rate, also constituent codes, GLPDC codes are promising codes in many applications, such as digital audio and TV broadcasting, high speed packet data transmission and deep space applications.
Considering that B-GLDPC [5] has more regular structure and lower implementation complexity and good performance, in this paper, we will focus on the low complexity hard decision decoding implementation of B-GLDPC. In the constructing of B-GLDPC, apart from the parity check matrix of the constituent code, several other code parameters should be memorized, including the code length, the number of super codes and a permutation matrix.
Soft decision GLDPC decoding is based on a combination of simple and fast soft-input–soft-output (SISO) decoding of short, linear block codes. SISO for constituent codes of short length and high rate is trellis based maximum a posteriori probability (MAP) decoding, known as BCJR (Bahl-CockeJelinek-Raviv), or its low complexity version Max-Log-MAP. But it is too high complexity, so chase SISO rather than optimal SISO decoder was proposed to lower complexity with little performance loss.
However, chase is also high complexity; moreover, there is few ultra-high speed A/D converter to supply soft LLRs in the optical system or ultra-high speed wireless data transfer system, it is necessary to investigate hard decision. Hard decision GLDPC decoding usually employs linear code hard decoder and bit flip algorithm. A simple bit flipping scheme was originally proposed by Gallager for LDPC codes [1]. Based on it, the Weighted Bit-Flipping (WBF) algorithm was developed in [8] for the sake of achieving an improved performance by exploiting some bit-reliability information, which results in an attractive tradeoff between the achievable performance and the decoding complexity imposed. Then several improved WBF algorithms have been proposed [9][10][11][12]. Among these algorithms, algorithm in [9] is the best for hard decision decoding GLDPC. It [9] employs the concept of votes to improve bit flipping algorithm for GLDPC codes and hence it was termed as Weighted Bit Flip Voting (WBFV).
However, WBFV also has its disadvantages. Finding the max vote module is necessary from thousands of votes (code length is usually thousands) for WBFV hardware implementation. This is intractability, if we find the max from thousands of data (code length is usually thousands) serially, thousands of cycles needed result to large latency; a lot of compare modules needed for parallel finding of the max, result

II. REVIEW OF GLDPC AND WBFV
A. The review of GLDPC
The structure of B-GLDPC codes is introduced [5], GLDPC is usually described by the Parity-Check Matrix of LDPC codes [1].
The parity check matrix of B-GLDPC code is a sparse matrix H constructed in the following manner. The matrix is partitioned into Constituent matrices H1, H2, H3,ĂHJ , H1 is a block-diagonal matrix obtained from an identity matrix with ones on the main diagonal, by replacing these ones with the parity check matrix H0 of a constituent code C0(n, k),. Each of the Hj constituent matrices is derived from H1 through a random column permutation ±j-1, as Eq. 1 and Eq. 2 shown:

GLDPC codes contains J numbers of interleaved supercodes, [9] demonstrates the scheme of J=2, as shown in Fig. 3. After votes have been cast, each symbol will receive a vote pair, for the scheme of J=2, the vote pair is either VV, eV, ee, EV, Ee or EE; for the scheme of J=3, the vote pair is either VVV, VVe, VVE, Vee, VEe, VEE, Eee, EEe or EEE. The different vote pair indicators different bit reliability. An optimal vote pair ordering in terms of decreasing is implied: VV<eV<ee, EV<Ee<EE. The various votes V, e, E are given numerical values so that the votes sum of the J constituent votes indicator their reliability. It was suggested in [9] that using the weight of V =0, e =1and E =2, and EE=4, Ee=3, EV=ee=2, Ve=1, VV=0, is capable of producing the lowest possible BERs.

The bit-flipping strategy is passing current bit values from the symbol nodes to the constraint nodes and then employing the details of the constraints the HDDs at these nodes pass back n individual votes to the symbols they cover. The magnitude of a vote indicates how reliable a constraint node considers the current symbol’s value to be. The votes at each symbol node are summed to give an overall metric such that all N symbols may be ordered in terms of reliability and the set of least reliable bits, those with the maximum total vote, flipped before a new iteration begins. Decoding iterations will be early terminated if the maximum vote pair is vote pair VV.
The WBFV decoding algorithm is summarized as follows: 1) Use hard decision decoder to decode constituent codes,
which belong to the super-code C1 and C2, respectively, decide the vote type E, e or V for each bit in the supercode. 2) Compute the vote weight for each of the N symbols according to EE=4, Ee=3, EV=ee=2, Ve=1, VV=0 when J=2, for the sake of quantifying the reliability of all symbols. 3) Rank the symbols according to their reliability based on their vote weights. 4) Correct the lowest reliability symbols, i.e. those having the maximum vote weights. If the vote pair assumes the values of only ee or eV, the corresponding symbol will not be corrected, since no valid error magnitude was calculated. 5) Repeat steps 1) to 4). This process of symbol flipping continues, until we either arrive at the vote pair VV for all the N symbols or the maximum affordable number of GLDPC iterations is reached.
III. THE PROPOSED IMPROVED WBFV ALGORITHM
A. The motivation to improve the WBFV algorithm
As the above steps shown, the max vote finding module is the necessary part for the WBFV to hardware implementation. However, finding the max vote from thousands of (the code length is usually several thousands) votes is difficult. Traditionally, the max or min value is found serially, and a candidate data is compared every cycle. It takes thousands of cycles to get the max serially and a large latency for a decoding

iteration. However, when the max is found in the parallel manner, similar to FFT algorithm, the min number of compare module is n*log (n), and a lot of resources are needed. So it is need to improve the max vote method. After simulation we find the group voting method is a promising substitute of the max vote method.
B. The description of the proposed improved algorithm
In the new algorithm, the first process of voting is same with WBFV. The difference is the bit-flipping strategy, using the group voting method instead of the max vote method.
First, we also use constituent code HDD to decide vote type E, e or V. But we don’t need compute the weight and rank them according to the voting pairs and find the max vote. We group the close reliable voting pairs into same group and flip together. According to the simulation result, three groups will perform well. So there is no longer need to find max vote, but only to confirm the less reliable voting pair group and flip these bits first. For the scheme of J=2, vote pair EE and Ee are the first group; the scheme of for J=3, first group are EEE, EEe, EEV, and Eee. After grouping the vote pair, the decoding process only flip first group bits until there is no first group vote pair. If there is no any first group vote pair, the second group bits are flipped. If new first group vote pair arises, then only flip first group bits.
The proposed improved decoding algorithm is summarized as follows: 1) Use hard decision decoder to decode constituent codes,
which belong to the super-code C1 and C2, respectively, decide the vote type E, e or V for each bit in the constituent code. 2) Group the vote pair into three groups, according to the rank of WBFV, the high vote value into first group, the low vote value into third group, the middle vote value into the second group. As shown in table I, for the scheme of J=2, there is only one group dividing method, the first group is EE and Ee, the second group is EV, ee, the vote pair VV and eV are grouped into third group. For the scheme of J=3, we recommend the first group is EEE, EEe, EEV, and Eee, the second group is EeV, EVV, eee, eeV. The vote pair VVV, eVV are grouped into third group. 3) If there is only third group then iteration terminates, decoding finishes. Otherwise, only all the first group bits are flipped and all the second group bits keep the same until there is no any first group bit. If there is no first group bit but some second group bits, then all the second group bits are flipped. 4) Repeat steps 1) to 3). This process of symbol flipping continues, until we either arrive at the vote pair VV or VVV for all the N symbols or the maximum affordable number of GLDPC iterations is reached. Except the advantage of no module of finding max vote, the new proposed improved algorithm has no performance loss compared with WBFV. Besides, it speeds up the iteration

IV. THE SIMULATION RESULTS
We investigate the performance and iteration between the proposed algorithm and WBFV. Both J=2 and J=3 GLDPC are all simulated. 4 sets of GLDPC codes are chosen:
 J=2 GLDPC, length=3780, rate=0.619, with sub-code of BCH (63, 51), t=2;
 J=2 GLDPC, length=7650, rate=0.686, with sub-code of BCH (255, 215), t=6;
 J=3 GLDPC, length=3780, rate=0.429, with sub-code of BCH (63, 51), t=2;
 J=3 GLDPC, length=7650, rate=0.529, with sub-code of BCH (255, 215), t=6.
The group rule employs table I, which guarantees the best performance.
1) The BER performance between WBFV and the
proposed improved algorithm The first set of simulations investigated the BER performance between the two algorithms. Fig. 4. shows BER of 4 sets of GLDPC codes between WBFV and the proposed algorithm, Fig. 4. a for the scheme of
J=2 while Fig. 4. b for the scheme of J=3. The BER curves of two algorithms for the same code are almost overlapping.
According to the simulation result, no matter the scheme of J=2 or J=3, no matter different code length or different code rate, the BER performance of the proposed algorithm and WBFV is almost the same; there is no BER performance loss with the proposed algorithm.
So we can get a conclusion that the proposed algorithm has no BER performance loss with WBFV and is a promising substitute of WBFV.
2) The iterations performance between WBFV and the
proposed improved algorithm The first set of simulations investigated the iteration performance between the two algorithms; the simulation result is as following: Fig. 5. shows the BER VS iteration of the 4 sets of GLDPC codes between these two algorithms, Fig. 5. a for the scheme
of J=2 while Fig. 5. b for the scheme of J=3. Apparently, no matter which set of GLDPC, the iteration times at the same ber for the proposed algorithm is lower than WBFV. From fig. a, for J=2, length=3780 GLDPC code, ber of 10-6 for the proposed algorithm is at 7 while WBFV is at 10; for length=7850 GLDPC code, ber of 10-8is at 5 while WBFV is at 8. From Fig. 5. b, for J=3, length=3780 GLDPC code, ber of 10-6 for the proposed algorithm is at 7 while WBFV is at 30; for length=7850 GLDPC code, ber of 10-6 is at 6 while WBFV is at 25.

Because of the min iteration time for J=2 GLDPC is relatedly low, about 10, there is no too much to speed up, however, the min iteration time for J=3 GLDPC is usually big, above 25, so the speed up is significant for J=3 GLDPC. For J=3 GLDPC, The proposed improved algorithm will lowers the origin min iteration time of between 25 and 30 to between 5 and 8.
So we can conclude that the proposed improved algorithm will speed up the iteration convergence and reduce the min iteration times. Especially for J=3 GLDPC, the speed up is significant.
3) The conclusion from the simulation results As a summary, the new proposed improved algorithm simplifies the voting process with no performance loss while the number of iterations decreases apparently.

V. DECODER DESIGN FOR IMPROVED ALGORITHM

Now the new algorithm has not this max vote finding module, but module to recognize three group type votes, which is easy to implement in hardware.
There are different architectures for GLDPC decoder, such as serial, full parallel, partial parallel and so on. The serial decoder is extremely low complexity but a little low throughput; while the parallel decoder is high throughput but a little more resources.

VI. CONCLUSION
The construction and properties of GLDPCs have been reviewed. The classical WBFV algorithm also has been review and verified by simulation. Because of that it is hard to hardware implement the max vote finding process of WBFV, we proposed an improved and simplified WBFV algorithm which with no need of max voting finding and has no performance loss but is easy for hardware implementation. The algorithm use group the vote pairs method and several advantages have been gain. As the simulation result shown, the new improved algorithm not only has no performance loss, but also speeds up the iteration convergence, lowers the min iteration times. Especially for J=3, the min iteration times decrease from 25~30 to 5~8. Finally, we proposed a decoder design for our algorithm with very low complexity.
