Concatenated code constructions for error correction in non-volatile memories

Ju¨rgen Freudenberger, Uwe Kaiser and Jens Spinner HTWG Konstanz
University of Applied Sciences, Konstanz, Germany Institute for System Dynamics (ISD)
Email: {jfreuden,uwkaiser,jespinne}@htwg-konstanz.de

Abstract—This work investigates and compares different coding techniques for error correction in multilevel NAND flash memories. In particular, we consider strong error correction codes with an overall code rate of 0.8 and with a sector size of 1 kbyte. Two concatenated code constructions are compared to the state-of-the-art Bose-Chaudhuri-Hocquenghem (BCH) codes. First, we consider a serial concatenation of an outer BCH code and an inner LDPC or turbo code (TC). The second construction is a generalized concatenated code with outer Reed-Solomon (RS) and inner BCH codes. The presented simulation results and theoretical investigations demonstrate that the algebraic code constructions obtain a performance close or even superior to the iterative coding schemes.
I. INTRODUCTION
In this work we consider error correction for flash memory applications. The importance of flash memories as a nonvolatile mass storage is continuously increasing. A flash memory consists of floating gates in which the information is stored. These floating gates hold their state without a power supply. However, errors occur while the information is read. Hence, an error correction coding (ECC) unit is required, which has the task of correcting the errors. The error rate depends on the storage density, the used flash technology (multi-level cell (MLC) or single-level cell (SLC)) and on the amount of the read and write cycles [1]. In general the error statistic model of a flash memory can be assumed as a binary symmetric channel (BSC). Hence, typically BoseChaudhuri-Hocquenghem (BCH) codes (cf. [2]) are used for error correction [3], [1].
As NAND error rates continued to grow, the error correction capability of the ECC has to be increased. Furthermore, the correction block size changed from 512 byte sectors to 1 or 2 kbyte in order to further improve correction efficiency. The advent of 3-level cells and even smaller geometries will further increase the error correction requirements in NAND flash.
The design of an ECC unit for flash memories is a demanding task. For some applications, block error rates less than 10−16 are required, whereas the encoding and decoding complexity has to be moderate. Furthermore, the error correction module in the flash controller must be designed flexible in order to satisfy the different error correction capabilities for a large bandwidth of different flash technologies and applications. Moreover, the complexity of a decoding circuit

for BCH codes increases with the error correction capability of the code. Therefore, BCH codes will likely be replaced in the future by more powerful codes or codes that provide a better trade-off between performance and complexity.
The high decoding complexity and the high residual error rate are the main reasons, why code classes like Low-Density Parity-Check (LDPC) codes or turbo codes (TC) have not yet found their way into flash memory applications, despite their superior performance. Due to the error floor of the iterative decoding schemes, LDPC and turbo codes require an additional outer code. For instance, the concatenation of LDPC and BCH codes is used in the ETSI (European Telecommunications Standards Institute) EN 302 307 (DVB-S.2) standard. The outer BCH codes are used to correct sporadic errors made by the LDPC decoder. A similar code construction is also investigated in [4].
In this work, we investigate whether code concatenation is an alternative to stand-alone BCH coding. There exist two main methods of code concatenation [5]: With the first method the system of inner encoder, transmission channel, and inner decoder is seen as a super channel for an outer code. The outer code is designed to correct the residual errors of the inner code. The concatenation of LDPC and BCH codes is an example of such a concatenation scheme. Another method is the so-called generalized code concatenation which combines several outer and inner codes and optimizes the properties of the overall code. With generalized concatenated codes it is possible to combine the advantages of the used inner and outer codes, e.g. to reduce the decoding complexity.
In this work, we consider both concatenation methods. We investigate different concatenated coding schemes for error correction with flash memories. In section II, we investigate a serial concatenation of an outer BCH code and an inner LDPC or turbo code. In this case, the outer BCH code is designed in a way that the BCH encoder/decoder can be used as standalone error correction unit with SLC memories. With the MLC technology the reliability of the memory cells is much lower. Hence, higher error correction capabilities are required. This can be achieved with inner LDPC or turbo codes.
In particular, we consider the a low-density generator matrix (LDGM) code [6], because the complexity of the decoding and encoding procedures is very low compared to standard

The corresponding multi stage decoder has a lower decoding complexity than a comparable stand-alone BCH code, but achieves a similar error correction performance.
II. SERIAL CONCATENATION OF AN OUTER BCH CODE AND AN INNER LDPC OR TURBO CODE
In this section we investigate a serial concatenation of an outer BCH code and an inner LDPC or turbo code with iterative decoding. As mentioned above, a flash controller is typically designed to support different flash technologies. Therefore, the ECC unit has to provide different error correction capabilities for different flash technologies and applications. For instance, for todays SLC memories an error correction capability of 12 to 24 bit errors per codeword is sufficient while MLC memories may require an error correction capability of 48 or more bit errors. With the proposed concatenation scheme, the outer BCH code is designed such that the BCH encoder/decoder can be used as stand-alone error correction unit with SLC memories.
Flash memories are organized in sectors of e.g. 512 or 1024 bytes. In addition to this payload, typically some bytes of firmware information has to be stored. In the following, we consider codes for a sector size of 1024 bytes. Therefore, we choose a total of 1036 bytes of payload information which corresponds to ko = 8288 as dimension of the outer code. For the outer code, a binary BCH code with the length no = 8624 was chosen. This code is constructed based on a primitive element of the Galois field GF (214) and can correct at most t = 24 errors. Accordingly the dimension of the inner code has to be ki = no = 8624. For all code examples we consider a maximal code length of 10200 bits. Thus, the rate of the inner code LDPC or turbo code is about R = 7/8.

B. Simulation of the error distribution As said before, the estimation of the error floor assumes
statistically independent error events. If the residual errors are statistically independent we can assume that the super channel for the outer binary BCH code is a BSC with a channel error rate equal to the residual bit error rate of the LDGM code. In this case, the number of errors in an outer codeword would follow a binomial distribution. To see if this assumption is correct we examined the distribution of the residual errors for different channel error rates and compared it to the ideal binomial distribution. For example, with a channel error rate of p = 0.01 the LDGM code achieves a residual error rate of approximately 5 · 10−7. However, more than 24 errors occur with a probability of 10−5. Hence, the overall concatenated system has a block error rate of 10−5. However, for a BSC with channel error rate 5 · 10−7 the probability for more than 24 errors is 4.6 · 10−85.

Fig. 2 shows the simulated error floor of the (6,34) LDGM code compared to the theoretical error floor of an LDGM code with X = 6. It can be seen that with an error rate equal or less than p = 0.008 the simulated and the theoretical values agree. Tab. I compares the error distribution with the binomial distribution for p = 0.008. For this simulation, no codeword with more than 24 errors occurred with 5,000,000 transmitted codewords. However, compared to the binomial distribution with p = 0.008 there are codewords that contain too many residual errors. The binomial distribution was calculated with a bit error rate of 1.48 · 10−7. This is the simulation result of the LDGM code with p = 0.008. Hence, the assumption of statistically independence of the residual errors could not be confirmed. It is therefore not possible to estimate the performance of the overall system for very low block error rates. C. Comparison of LDGM and turbo codes

III. DESIGN OF GC CODES In this section we give a brief introduction to generalized concatenated (GC) codes. A detailed discussion can be found in [14], [5]. Then, we discuss the multistage decoding algorithm for GC codes and derive error bounds for the decoding error probability. These bounds are used for the code design. In the following we use the well known notation C(2m, n, k, d) to denote a code C of length n and dimension k over the Galois field GF (2m). The last parameter d denotes the minimum Hamming distance of the code C.
A. Encoding In a classical concatenated code, an information vector is
mapped to a codeword a = (a0, . . . , an−1), aj ∈ GF (2m) of the outer code A(2m, no, ko, do). Each code symbol aj of the outer code is mapped to a codeword bj of the binary inner code B(2, ni, ki, di). Hence, we require ki = m. This binary codeword bj of length no is then the j-th column of a ni × no matrix c, which is the matrix representation of a codeword of the overall code C(2, noni, koki, dodi). Note that this classical concatenation is different to the serial concatenation as described in setion II, where every single symbol of the outer code is fed to the inner encoder.
With a GC code this concatenation scheme is generalized to more outer and inner codes. A GC code of order L consists of L outer codes Al(2m, no, ko,l, do,l), l = 1, . . . , L of the same length no and over the same Galois field GF (2m). However, the dimensions and minimum distances can vary from level to level. For the inner encoding we use L binary codes Bl(2, ni, ki,l, di,l) of length ni, where the code Bl is a sub-code of the code Bl−1. Hence, we have the following relation BL ⊂ BL−1 ⊂ . . . ⊂ B1. The dimension of the l-th level is ki,l = lm.

symbol aj,l to form a vector of lm bits. The j-th column bj of the overall codeword matrix is obtained as

Decoding
Typically, a GC code is designed to optimize the overall minimum distance for a given code length and code dimension. However, we consider a design that minimizes the probability of a decoding error (incorrect decision or decoding failure) with a multistage decoding procedure. To illustrate the design and the decoding algorithm, we consider the GC code from example 1.
Next we discuss the multistage decoding procedure. Let r be the received binary ni × no matrix after transmission over a BSC with error probability p, where rj is the j-th column of the received matrix.

Hence, in the first step we decode the vectors rj with respect to the first inner code B1 which results in estimated codewords b˜j. For the example code, the first inner code has minimum distance one. Hence, we can omit this decoding stage.
For the outer decoding, we have to infer the code symbols a˜j,1 from the vectors b˜j. This requires a re-encoding of b˜j. Due to the systematic inner encoding the L-th outer symbol a˜j,L can by directly inferred from b˜j. Hence, we obtain an estimated inner codeword b˜j,L of the L-th level by reencoding a˜j,L. Successively re-encoding level by level, we obtain estimates for all inner codewords and all outer code symbols. For the example code, we directly infer the outer codeword a˜2. Re-encoding all symbols a˜j,2 to codewords b˜j,2, and obtain estimated codewords b˜j,1 = b˜j − b˜j,2 .
After the re-encoding we can proceed with the outer decoding. Thus, the next step is decoding the first outer code. In this step we obtain the estimated codeword aˆ1 which is the final estimate for the first level. After decoding the first outer code, we can again re-encode the symbols aˆj,1 to inner codewords bˆj,1. Next we decode the second level. Note that bj − bj,1 is a codeword of the code B2. Therefore, we decode rj − bˆj,1 with the inner code B2. From the estimated codewords we infer the symbols of the outer codeword and decode the outer code of the second level. In general, this decoding is processed for all L levels.

D. Computational complexity
The size of the decoding circuit and the achievable throughput depend on the decoder architecture. In order to provide at least a rough comparison of the computational complexity of the GC decoder with BCH decoding, we assume that the RS and the BCH codes are decoded by using the inversionless Berlekamp-Massey algorithm (BMA) to compute the error location polynomial and the Chien search to calculate its roots [2]. In this case, the most complex operation of a hardware implementation is the multiplication in the Galois field GF (2m). The size (number of gates) of the multiplier increases with order O(m2), whereas O(t) multipliers are needed for the decoding of a code with error correction capability t. A fully parallel implementation of the BMA requires 2t such multipliers and the Chien search with P parallel branches requires P t multipliers. The number of cycles is dominated by the Chien search which requires n/P cycles for a code of length n.

IV. COMPARISON OF THE CODE CONSTRUCTIONS Figure 5 presents simulation and numerical results for the word error rate (WER) for the two concatenated schemes as well as for the BCH code C(10192, 8288, 273). For the considered range of WER values, the LDGM code with sum-product decoding algorithm and outer BCH code outperforms the two codes with algebraic decoding. However, the gap between BCH and LDGM code is small. Moreover, the BCH as well as the GC code obtain better results as the turbo code construction or the LDGM code with min-sum decoding. Furthermore, very low residual error rates can only be guaranteed with the BCH or GC codes. For Pe = 10−16 the BCH code achieves a maximum allowable channel error rate of pmax = 6 · 10−3, whereas for the GC code we have pmax = 3.3·10−3. However, the GC code has a much lower computational complexity.
V. CONCLUSIONS In this work we have investigated and compared different concatenation techniques for error correction in future multilevel NAND flash memories. Therefore, we consider strong error correction codes with an overall code rate of about 0.8. For all considered constructions, the BCH code still provides the best performance for low residual block error rates.

For applications which tolerate residual block error rates of Pe > 10−6 the serial concatenation of an inner LDGM code with an outer BCH code outperforms the stand-alone BCH code. However, this performance requires the computational complexity of the sum-product algorithm and some means to reliably estimate the raw error rate of the memory cells. Therefore, the investigated LDGM code is not an attractive alternative for most flash controller applications with a sector size of 1KByte.
The presented generalized concatenated code construction is suited for applications which require a high reliability and a low decoding complexity. It can be easily adjusted to different code rates and reliability requirements. However, the BCH codes provide a superior performance. Remarkably, both algebraic coding schemes outperform the turbo code construction and the LDGM construction with mim-sum decoding even for high channel error rates.
