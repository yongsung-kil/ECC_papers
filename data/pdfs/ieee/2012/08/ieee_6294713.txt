2012 International Conference on Telecommunications and Multimedia (TEMU)

Novel Technique for Scaling Down LDPC Code Lengths in DVB-T2 Standard

Fatma A. Newagy, Member, IEEE and Salwa H. Elramly, Senior Member, IEEE
Department of Electronics and Communications Engineering, Faculty of Engineering, Ain Shams University, Cairo, Egypt.

Abstract— the paper proposes an approach to scale-down the LDPC code length, thus saving memory and processing power, without significantly sacrificing the code robustness. the paper describes how to use specific Low Density Parity Check (LDPC) codes for Digital Video Broadcasting Terrestrial Second Generation DVB-T2. While the original LDPC codes are rather long 16200 bits and 64800 bits, we propose to reduce their size (about 3k instead of 16k). To reduce the encoder/decoder complexity, the scaled down LDPC codes based on DVB-T2 of length 16200 are proposed in this paper. Scaling the codes may lead to the problems of girth-4 and low weight codes. This paper analyzed the possible reasons leading to the problems, and proposed the optimum scaled down LDPC codes of DVB-T2 that avoid girth-4 and low weight codes. The simulation results in the AWGN channel show that for code rate ½ at BER = 10-5, LDPC DVB-T2_3240 requires only additional 0.5dB above that required by DVB-T2_16200 and the proposed scaled down code by 5 still has no error floor as well as the former codes in [10]. The impact of the paper is not restricted to DVB-T2 systems, but also to DVB-S2 and all future technologies which are to adopt LDPC.
Keywords- LDPC codes; DVB-T2 standard; Sum Product Algorithm (SPA); Minimum Sum Algorithm (MSA); girth-4; minimum weight.
I. INTRODUCTION
Due to their near-Shannon limit performance, LDPC codes [1] have become promising candidate codes for current and future generation wireless communication systems. Currently, DVB-T2, which is an European high-quality digital video broadcasting system for terrestrial second generation standard, features the LDPC codes as its inner channel-coding scheme [2]. LDPC codes are linear block codes that were originally introduced by Gallager [3] in 1962. During that time, they attracted little attention, since hardware implementation of such decoding was impractical in the 1960s, and has been neglected since. However, the value of LDPC codes was rediscovered by Mackay and Neal in 1995, and many subsequent studies have shown that LDPC probabilistic decoding is very effective [4, 5]. Iterative LDPC decoding schemes based on the Sum-Product algorithm and MinimumSum algorithm [6] can be fully parallelized, leading to highspeed decoding. For excellent BER performance, the size of the block should be large. Decoding large blocks requires significant amounts of computation and memory. In this paper, reduced operations, complexity and power consumption of the parity-check matrix is proposed based on decreasing the size. Novel technique to re-compute the parameters defining the

parity-check matrix of DVB-T2 LDPC codes and to select the codes having good features among those obtained is proposed.
The rest of this paper is organized as follows. In the next section, the methods to detect girth-4 and minimum weight code are stated. The design of the code rate ½ DVB-T2 LDPC code of length 16200 is then carried out in Section III. Section IV presents the proposed design of scaled DVB-T2 LDPC code. Estimated results are then provided and discussed in Section V before the paper is finally concluded in Section VI.

II. DETERMINING THE GIRTH-4 AND MINIMUM WEIGHT
Suppose that the length of the LDPC code is N, within which the information sequence measures K bits, then the size of the parity check matrix H is (N-K) x N. H matrix can be described by a bipartite graph called tanner graph. A cycle in a tanner graph refers to a finite set of connected edges that starts and ends at the same node and satisfies the condition that no node (except the initial and final node) appears more than once [7]. The length of the cycle is simply the number of edges it contains. The length of the smallest cycle in the graph is called the girth. According to the DVBT2 standard, the code-length of the LDPC is 16200 or 64800. In this paper, we adopt the code-rate equal to 1/2 and the code length equal to 16200 so as to elaborate on the design method of the parity-check matrix of the DVB-T2 LDPC code. VI. CONCLUSIONS
The paper proposes an approach to scale-down the LDPC code length, thus saving memory and processing power, without significantly sacrificing the code robustness. In this paper, LDPC code of length 3240 based on DVB-T2 is proposed. As implementation of LDPC decoder is a tradeoff between error-correction performance and hardware complexity which depends on the parity check matrix size and construction. Proposed scaled down DVB-T2 LDPC code of length 3240 is simulated and BER performance illustrates that there is no error floor for different number of iterations, because there is no girth-4 and no minimum weight problems. LDPC DVB-T2_3240 requires only 0.5dB above that required by DVB-T2_16200 at BER = 10-5. The impact of the paper is not restricted to DVB-T2 systems, but also to DVB-S2 and all future technologies which are to adopt LDPC.
