Abstract—Absorbing sets have been identified as structures in the graph of a low-density parity-check code that cause error floors - in particular in combination with binary message passing decoding algorithms. In this paper it is shown that absorbing sets involving only variable nodes up to degree 3 are equivalent to cycles and a sufficient and necessary condition on the degree distribution to avoid these absorbing sets is derived. The results are extended to irregular graphs and simulation results demonstrate the improvement in the error floor region.
I. INTRODUCTION
Codes based on sparse graphs, e.g., low-density parity-check (LDPC) codes, are typically analysed in the asymptotic regime when the block length tends to infinity. In this case, tools like density evolution can accurately predict the decoding threshold of the ensemble average.
The analysis of the finite-length behaviour (including the error floor) of codes based on sparse graphs always relies on particular local structures in the graph. For the binary erasure channel (BEC) the key objects are stopping sets [1] – a set of variable nodes that, if erased, cannot be decoded by an iterative decoder. For other channels, trapping sets have been shown to be problematic [2].
A particular graphical structure, called an absorbing set, has been introduced in [3]. Absorbing sets are of particular importance for binary message passing decoders (also known as bit-flipping decoders) as they represent stable fixed points, i.e., if all variable nodes that form an absorbing set are in error then they cannot be corrected by a binary message passing decoder. For array-based LDPC codes there exists a theoretical analysis of absorbing sets [4].
In this paper we show how absorbing sets are related to cycles in the graph. We show that, for variable node degrees 2 and 3, absorbing sets and cycles are the same structures. Based on these results we develop a constraint on the variable node degree distribution that leads to codes without small absorbing sets.
We start by introducing absorbing sets in Section II. The relationship to cycles is described in Section III and numerical results are presented in Section IV.
II. ABSORBING SETS
Before stating our results on the relationship between absorbing sets and cycles, we summarise the definition of absorbing sets and a lower bound on their size from [4], [5].

Let V and F denote the set of vertices associated with the variable and check nodes respectively of the Tanner graph of an LDPC code. Furthermore, let g denote the girth of the Tanner graph. For any subset D ⊆ V let E(D) and O(D) denote the neighbours of D with even and odd degree, respectively. The subgraph formed by D, is the graph containing D, its neighbours and all the edges between them.
Definition 1 (Absorbing sets [4, Definition 1]). Given an integer pair (a, b), an (a, b) absorbing set is a subset D ⊆ V of size a, with O(D) of size b and with the property that each element of D has strictly fewer neighbours in O(D) than in F \ O(D). We say that an (a, b) absorbing set D is an (a, b) fully absorbing set, if in addition, all variable nodes in V \ D have strictly more neighbours in F \ O(D) than in O(D).

In the following lemma we show that for the case that γ ∈ {2, 3} the bound is in fact tight and a cycle of size g will always lead to an absorbing set of size g/2.
Lemma 2. Every cycle of size g with all of its variable nodes having degree γ ∈ {2, 3} corresponds to an absorbing set of size a = g/2.
Proof: In a single cycle, every node has two connections within the cycle. Assume variable nodes of degree γ = 2. In this case, all edges of a variable node are part of the cycle. Furthermore, every check node involved in the cycle has an even number of connections within the cycle (2 for a single cycle and multiple of 2 for overlapping cycles). By the definition of an absorbing set this corresponds to an (a, 0) absorbing set where a equals the number of variable nodes involved in the cycle.
Assume now variable nodes of degree γ = 3. In order to be part of a cycle, each variable node has two edges connected to the cycle and one edge connected to another check node. Therefore, this corresponds to the same setup as for the case of degree two variable nodes, and cycles with degree γ = 3 variable nodes lead to (a, b) absorbing sets where a equals the number of variable nodes involved in the cycle and 0 ≤ b ≤ a. The lower bound of b is achieved for multiple overlapping cycles and the upper bound is achieved when all a variable nodes have one connection outside the cycle.
The following lemma shows that the reverse is also true, i.e. by avoiding cycles in the subgraph formed by all the variable nodes with degree γ ∈ {2, 3} we can also avoid all absorbing sets solely in those nodes.
Lemma 3. Every absorbing set with all of its variable nodes having degree γ ∈ {2, 3} contains a cycle.
Proof: For γ = 2 we prove this lemma by contradiction. Assume the existence of an (a, b) absorbing set which does not form a cycle and let d denote the number of involved check nodes of even degree with respect to the absorbing set, i.e., d = |E(D)|. Consider the sub-graph consisting of the set of variable nodes D and the set of check nodes E(D). By the definition of an absorbing set, every variable node in this sub-graph has two edges and every check node has a nonzero even number of edges. Therefore, we have d ≤ a. If this sub-graph does not contain a cycle then it has to be a tree. For a tree, the number of edges e is at most one less than the number of nodes, i.e.,

e ≤ a + d − 1.

(1)

Due to the definition of an absorbing set, every variable node has γ = 2 edges in this sub-graph, i.e., e = 2a. This leads to a ≤ d − 1 which contradicts d ≤ a.
Now consider the case where one or more nodes have γ = 3.
From the definition of an absorbing set we know that every variable node from D has to be connected to E(D) at least

twice. Furthermore, the degree of every check node in E(D) is a multiple of 2. We start by assuming that every variable node in D has the minimum number (exactly 2) connections with E(D). This is the same constellation as for the γ = 2 case and
results in a cycle. Now consider a third edge connected to a variable node. If the third edge connects to a node F \ E(D) then it does not change the setting from the γ = 2 case. However, if this edge connects to E(D) then another third edge of a variable node in D also has to connect to this check node (otherwise this check node would be removed from E(D)). Supposing two of the variable nodes in D connect to E(D)
with all three edges. The sub-graph consisting of the set of variable nodes D and the set of check nodes E(D) now has e = 2a + 2 edges and E(D) could now have up to one extra node, i.e. d ≤ a + 1. Substituting into (1) we have

2a + 2 ≤ a + d − 1,

i.e a ≤ d − 3 which contradicts d ≤ a + 1. Similarly if any even number k of the nodes in D connect to E(D) with all three edges we have e = 2a + k and d ≤ a + k/2 and thus it is not possible for the sub-graph consisting of the set of variable nodes D and the set of check nodes E(D) to be cycle free when γ ∈ {2, 3}.
The benefit of this result is that by avoiding cycles in the subgraph formed by the degree 2 and 3 variable nodes we can increase the minimum absorbing set size of the code while retaining a sufficient number of degree 2 and 3 variable nodes to allow for a good threshold performance. Indeed, in [6] it was
shown that by avoiding cycles in variable nodes with degree γ ∈ {2, 3}, LDPC codes with very low error floors could be produced. Our results show why this was the case.
From [6] we know under what conditions it is possible to avoid cycles within graphs containing only degree 2 and 3 variable nodes. Here we extend this result to higher degree variable nodes. IV. NUMERICAL RESULTS
In this section we present optimised codes and simulation results for the binary message passing decoding algorithm with soft decisions from an AWGN channel (see [6] for details). The motivation for choosing the binary message passing decoder is that absorbing sets are particularly harmful for this algorithm (as they form stable fixed points). Furthermore, this algorithm allows an efficient and fast implementation making it useful in practical implementations.
