A LDPC Encoding and Decoding Scheme of Low Complexity Applied to Physical Layer 802.16e

Xu Li, Zhan Xu
State Key Laboratory of Rail Traffic Control and Safety, Beijing Jiaotong University, Beijing, China {xli, 10120173}@bjtu.edu.cn

Abstract—In the IEEE 802.16e OFDM (Orthogonal Frequency Division Multiplexing) physical layer, the optional LDPC (Low Density Parity Check) code channel coding scheme based on the IEEE 802.16e protocol is able to be applied in order to improve the whole system’s BER (bit error rate) performance. Due to the excellent characteristics and good error correcting performance of LDPC code itself, it has become the central issue of current channel coding field with broad application prospects. In this paper, we use the improved BF (Bit Flipping) decoding algorithm, which is based on the structure of check matrix and the protocol’s provisions of LDPC coding scheme, to replace RS-CC (Reed Solomon-Convolutional Codes) concatenated coding scheme on the consideration of decoding delay and further improvement of BER performance. Meanwhile, the storage method of constructed check matrix mentioned in this paper is also optimized. The improved BF decoding algorithm can be verified by simulation that improvement of BER performance is realized on basis of low complexity.
Keywords- LDPC code, Bit Flipping, BER
I. INTRODUCTION LDPC (Low Density Parity Check) code was first proposed by Gallager in 1962 [1]. However, due to lack of the knowledge of Graph Theory and the constraint of digital signal process capacity at that time, Mackay reintroduced LDPC code until 1995 and proved that LDPC is a highperformance channel code using the scheme of BP (Belief Propagation) iterative decoding algorithm, which is able to approach the Shannon limit [2, 3]. Comparing with the IEEE 802.16d protocol, the support for terminal mobility is enhanced in the IEEE 802.16e protocol and LDPC code optional scheme is added in the part of channel coding. The data sending and receiving procedure of OFDM physical layer in the IEEE 802.16e standard is shown in Figure 1.
Figure 1. Data sending and receiving procedure of OFDM physical layer in the IEEE 802.16e protocol.

Among the decoding algorithms of LDPC code, the BER performance of BP decoding algorithm is optimal [3, 4]. In the paper [5], BER performance analysis of LDPC code based on the IEEE 802.16e protocol has been done on the condition of different iteration times and different methods of modulation using LLR BP decoding algorithm. However, LDPC code has not been applied to IEEE 802.16e OFDM physical layer platform, thus the BER performance of LDPC code is not able to be compared with other coding schemes. In the paper [6], the construction method of check matrix and encoding scheme of LDPC code according to the IEEE 802.16e protocol are described in detail. However, which decoding algorithm to use is not mentioned in the paper. As a result, it is not able to be realized that LDPC code has the ability to approach the BER performance of Shannon limit. In this paper, low complexity BF iterative decoding algorithm [1] has been used with the encoding scheme according to the IEEE 802.16e protocol being realized. On basis of BF algorithm, improved decoding algorithm is come up with to reduce the occupied system resources during the decoding process and improve the overall system’s BER performance.
II. ENCODING SCHEME OF LDPC CODE BASED ON THE IEEE 802.16E PROTOCOL
Encoding scheme of LDPC code according to the IEEE 802.16e protocol is able to be divided into two steps. The first step can be the check matrix’s construction of LDPC code. And the second is proposed to be the encoding algorithm based on the check matrix.

III. IMPROVED BF DECODING ALGORITHM
Currently, there are two main decoding schemes for LDPC code, that is, Belief Propagation decoding algorithm based on soft decision and checking iterative Bit Flipping decoding algorithm based on hard decision [8]. In the IEEE 802.16e OFDM physical layer platform, the constructed check matrix according to the protocol is always in large scale in order to guarantee the BER performance of LDPC code. Therefore, this kind of matrix construction method will largely occupy more system storage resources, with the complexity and delay of decoding process increasing. In conclusion, Improved BF decoding scheme with lower complexity can be chosen to replace RS-CC concatenated channel coding scheme so as to further improve the overall system’s BER performance.
After hard decision operation, BF decoding algorithm is able to calculate the failure number of the check equation which is corresponded to each receiving bit. And then, it is appropriate to flip the bits which lead to the maximum number of check equation’s failures in every iteration. If the number of receiving sequence’s errors is less than or equal to the error correcting ability of code, the process of decoding can be completed within only a few iterations. Meanwhile, due to the good characteristics of LDPC code, BF decoding algorithm is able to correct many error patterns the number of which exceeds the error correcting ability of code itself.
Since the constructed check matrix based on the IEEE 802.16e protocol is in large scale, it will lead to a massive data computation during calculating checksum ‫ݏ‬௠ = ( ‫ݏ‬ଵ , ‫ݏ‬ଶ ,...‫ݏ‬ெ ) = H*‫ݔ‬௡் and error patterns ݁௡ = (݁ଵ , ݁ଶ ,...݁ே ) = ୬୘ *H. Therefore, a bit flipping threshold Ɂ might be considered to be set in order to flip the bits which make the failure number of check equation exceed the threshold Ɂ , instead of flipping the bits corresponding to the maximum of error pattern’s set ݁௡, namely, the bits making checksum ‫ݏ‬௠ correct failed for the most number of times within every iteration. So more receiving bits with high error probability can be corrected in each iteration in order to realize a faster convergence, minimize the times of computation of checksum and reduce the massive data computation effectively during the decoding process.

V. CONCLUSION
Under the condition of limited system storage resources, LDPC code using improved BF decoding algorithm can further improve the overall system BER performance and reduce the complexity of decoding process at the same time. Additionally, LDPC code itself has anti-burst error characteristic. Therefore, during the process of channel encoding and decoding, interleaver and deinterleaver modules are not necessary, which can avoid possible delay under the condition of guaranteed BER performance. Meanwhile, due to other excellent features of LDPC code, it will certainly have a good application prospect in the field of wireless WAN (Wide Area Network).
