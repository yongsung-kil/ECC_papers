Approximated Sum-Product Decoding for LDPC Codes

Sung Ik Park and Heung Mook Kim
Broadcasting System Department, ETRI Daejeon, Korea
e-mail: {psi76, hmkim}@etri.re.kr
Abstract—This paper proposes an approximated sum-product decoding algorithm for LDPC codes. The proposed decoding algorithm may be efficiently implemented through integer additions, comparisons, and bit-shift operations without multiplications and memories. Moreover, it doesn’t require any channel information such as signal to noise ratio and operate with just the received values from channel.
Keywords-LDPC code; decoding; sum-product

I. INTRODUCTION (HEADING 1)
Low density parity-check (LDPC) codes, first discovered by Gallager [1], were rediscovered to be Shannon limit approaching codes over additive white Gaussian noise (AWGN) channels [2]. Recently, for commercial use of LDPC codes, efficient decoding algorithms have been studied extensively. The sum-product algorithm (SPA) which is a conventional decoding algorithm for LDPC codes can be simplified as the min-sum algorithm (MSA) [3], which greatly reduces the implementation complexity. However, the MSA may lead to considerable performance degradation.
This paper proposes an approximated SPA based on a linear approximation of ln cosh(x) for LDPC codes. The
proposed algorithm may be efficiently implemented because it consists of integer additions, comparisons, and bit-shift operations without multiplications (divisions) and memories. Moreover, the proposed algorithm doesn’t need any channel information such as signal to noise ratio (SNR) and operate with just the received values from channel.

Simulations have been conducted to compare the performance of the proposed algorithm with those of the SPA and MSA. In the simulations, we used Digital Video Satellite Broadcast Standard (DVB-S2) LDPC code with length of

64,800 and rate of 1/3 [7]. Fig. 3 shows the bit error rate (BER) of LDPC code according over AWGN channel. In Fig. 3, values of σ op was found by computer simulations. The
number of iterations is set to be 50. The simulation results in Fig. 3 show that DVB-S2 LDPC code with the proposed algorithm has just 0.02 dB poorer performance as that of LDPC code with SPA at the BER = 10-5. While the performance gap between the proposed algorithm and MSA is about 0.6 dB at BER=10í5. These results indicate that there is a tradeoff between performance and decoding complexity.
IV. CONCLUSION
This paper proposed an approximated sum-product decoding algorithm for LDPC codes. Simulation results show that the proposed decoding algorithm not only shows much better performance than MSA, but also does not cause serious performance degradation, as compared with the SPA.
