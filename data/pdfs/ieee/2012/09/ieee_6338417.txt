Igor Zhilin and Victor Zyablov Institute for Information Transmission Problems Moscow, Russia

Abstract-The influence of the computational precision used in BP decoder on LDPe codes performance is studied. It is shown that an error floor can be lowered and pseudo-codewords are made possible to decode by increasing computational precision.
I. INT RODUCTION
LDPC codes are largely used now for a vast variety of error correction applications. These codes were first introduced by Gallager [I, 2] and are valuable for having efficient iterative message-passing decoder. This decoder can be realized using two approaches: either using symbol values or using a pos­ teriori probabilities at the channel output. The decoder using probabilities is also known as belief propagation (BP) decoder.
LDPC codes which have different number £ of non-zero
column elements in their parity-check matrix are known to exhibit different error-correcting behaviour. Codes which have large number of such elements have asymptotically good minimal Hamming distance. The problem arises when using BP decoding: in region of relatively bad signal-to-noise ratios
it works better with codes which have small £, typically three
or four. But, while they are the best when used in bad SNRs, they become worse in better SNRs and exhibit error floors. An example of this behaviour is shown on fig. I for half-rate 672-length Gallager code.

D. Computer Simulation
Sumulation was perforemed on a computer using single pre­
cision arithmetic but for ¢(x) function where double precision
calculations were used for intermediate results.
Code under study was binary Gallager code of length 672 and 1/2 rate with £ = 3 ones in each column of a parity-check
matrix. Channel with additive white Gaussian noise with soft deci­
sion output was used.
III. RESULTS
Figure 4 show code simulation results for various limits
of function ¢(x). It is clear that improving resolution (using larger limit on ¢(x» allows us to lower the error floor and to
improve its slope in the error floor region. Fig. 5 shows number of error in received codeword versus
belief propagation iteration number. It's clear that behaviour of both variants are equal until a key point where less precise realization halts but more precise is still able to continue decoding. Increase of iterations number up to 500 for lesser precise variant does not allow it to decode the codeword.

IV. CONCLUSION
Results of the simulation show us that while pseudo­ codeword degrade performance of LDPC codes, this degra­ dation can be lowered by improving computational precision. This allows to decode the pseudo-codewordes which trap belief propagation decoder with lower precision and, conse­ quently, to significantly lower error floors and increase their slope.
