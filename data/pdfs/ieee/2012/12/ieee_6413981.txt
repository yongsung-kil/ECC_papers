Abstract—In this paper LDPC encoding algorithms are presented, and the algorithms are analyzed based on CMMB standard by simulation. The performances of the encoding algorithms are compared in terms of decomposition effect and hardware implementation. The simulation results show that the LU decomposition based algorithm which finds the principal components in the row and column with the lightest weight can achieve high decomposition efficiency and is suitable for hardware implementation.
Keywords- CMMB;LDPCencoder;LUdecomposition
I. INTRODUCTION
CMMB is released in 2006 as the mobile multimedia broadcasting standard for China. A shorted RS code as outer code and a binary LDPC code as inner code is separated in CMMB. For LDPC code, two kinds of code rate(1/2and3/4) are proposed. LDPC codes were first presented by Gallager and were defined by the parity check matrix with a vast majority of zero elements and a little quantity of non-zero elements[1].The check matrix H with 4608 rows and 9216 columns is given by CMMB. There’s no generate matrix in CMMB, so we can’t encode the codes through the generate matrix directly. From the orthogonality between matrix H and matrix G, we can get G from H, but the matrix G is too dense to realize it in hardware. In this paper we simulate several LDPC encoding methods. We analyze and compare these methods to find a suitable method for hardware implementation.

B. Coding method based on LU decomposition
LU decomposition is proposed by Neal[5]. LU decomposition is to decompose a full rank matrix into an upper triangular matrix and a lower triangular matrix. Applying LU decomposition on the LDPC coding is to encode directly by the H matrix. We encode the LDPC codes through the use of two triangular matrices to avoid the inverse operation on matrix.
The purpose of the encoding of LDPC codes is seeking parity bits. Parity bits and information bits form a codeword. From the second part of the paper we have known that the outputting encoded code is not a systematic codeword, accordingly the H matrix giving by CMMB is not systematic. If we want to get the block of parity bits through the block of information bits we should get the systematic H matrix first. Putting the column COL_ORDER˄i˅into the column i, we can get the systematic H matrix, denoted by Hsys,
[ ] Hsys = Hp, Hs .The last 4608 columns of Hsys correspond
with information bits and the first 4608 columns correspond with parity bits.
From the orthogonality between Hsys and outputting
encoded code, Hsys × CsysT = 0 , we can get equation (5):

[Hp, Hs]× [p, s]T = 0

(5)

According equation (5) we can get equation (6):

Hp × pT + Hs × sT = 0

(6)

Because the encoding process is in the GF(2), equation (6) can be replaced by the following equation:

Hp × pT = Hs × sT

(7)

From equation (7) we can get the next equation:

pT = Hp −1 × Hs × sT

(8)

We can achieve the outputting encoded code when we get pT. From equation (8) we know that if we want to get pT we should know three variables: Hp-1ǃHsǃsT. Hs and sT are known, but we should get the inverse matrix of Hp. From above content, we know the H matrix, also Hp, is sparse but its inverse matrix is very dense. In order to avoid using the inverse matrix, LU decomposition is adopted. Decompose Hp into a product by L and U two matrices. Here, L is a lower triangular matrix and U is an upper triangular matrix. There are many frequently used LU decomposition algorithms. According to the different way to find the principle component, there’re these common methods: the principle component in the row, the row and column with the lightest weight, multiplication of the row with the lightest weight and the column with the lightest weight[7], and optimized partial pivoting strategy[8].Different algorithms are used in the LU decomposition to make the decomposition as sparse as possible, such as dynamic programming solution[9], Simulated Annealing[10].All these algorithms are used to make the triangular matrix have less nonzero elements. The algorithm of finding the principle component in the row and column with the lightest weight is detailed as follows:

1) Initialization:L=En*n,U=Hp,cl1=[1:n],cl2=[1:n],i=1.(cl1 and cl2 are used to confirm and record the transformation order of the row and column)
2) Finding the principle component: Find the row with the lightest weight from row i to row n. The number of the found row is recorded as a. Find those columns in which the element in row m is nonzero and find the column with the lightest weight in those columns. The number of the found column is recorded as b.
3) Row and column transformation of U:If a is not equal to i, transform row a and row i; If b is not equal to i, transform column a and column i. Update cl1 and cl2 with the same transformation. Exchange the first i-1 elements between row i and row a.
4) Modify the L matrix: Find the positions of nonzero elements in column i from row i+1 to row n of matrix U. Make the elements in the same positions in L to be 1.
5) Gauss elimination on U: Find those rows which element in column i is nonzero in U from row i+1 to row n. Do the binary addition on those rows with row i.

6) Judgment of loop operations: If i+1=n, end. Otherwise, jump into step2).

IV. CONCLUSION
In this paper we analyze several LDPC coding methods and simulate two methods in MATLAB. Compare the simulation results we find that the LU decomposition based algorithm which finds the principal components in the row and column with the lightest weight can achieve less nonzero elements and is suitable for hardware implementation.
