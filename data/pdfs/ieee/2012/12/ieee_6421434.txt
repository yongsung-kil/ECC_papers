Abstract-The Layered Belief Propagation L-BP algorithm is is a modified Belief Propagation BP algorithm, where the check nodes are divided in subgroups called layers and each iteration is broken into multiple sub-iterations. In this paper, we consider layered belief propagation decoding and propose an efficient variable node layering for decoding LDPC codes that performs well. We compare the performance of the first introduced LDPC decoding algorithm BP and Layered BP using check node layering with Layered BP using variable node layering in terms of bit error rate (BER). We show that the convergence for decoding LDPC codes is increasing by using a simple and efficient layering strategy.
I. INTRODUCTION
Low Density Parity Check (LDPC) codes, first introduced by R. Gallager [I] in the early 1960s, are actually widely investigated because of their excellent performance [2-4]. LDPC codes can be efficiently decoded us- ing the iterative belief propagation BP algorithm [13], which approximates the maximum likelihood decoding algorithm.
Recently, several papers have investigated different types of scheduling strategies in BP LDPC decoding.With sequential scheduling, the messages are generated sequentially using the latest available information. Sequential scheduling was introduced as a sequence of check-node updates in [5,6] and a sequence of variable-node updates in [7,8]. It is also presented in [18] under the name of Layered BP (LBP), in [9] and [10] as serial schedule, in [11] as row message passing and column message passing.
Check nodes Layered BP algorithm CL-BP is a modified BP algorithm that divides check nodes into small subgroups called layers and breaks each iteration into multiple sub­ iterations. In each sub-iteration one layer of check nodes and their neighboring variable nodes are processed [18]. In this paper, we propose an efficient layering strategy for the variable nodes VL-BP that significantly increases decoding convergence of LDPC codes as compared to CL-BP. We show that the decoding convergence of the proposed variable nodes layering outperforms CL-BP and BP decoding. In the second section, we introduce LDPC representation with bi­ partite graph. In the third section, we describe the principal of LDPC optimal decoding algorithm. In the fourth section, we describe BP and CL-BP algorithms and we give the details of the proposed VL-BP algorithm. Section V gives some simulation results, where the variable nodes layering technique

is compared with check nodes layering and BP algorithm. The last section briefly concludes the presented work.

II. LOW DENSITY PARITY CHECK CODES
A binary (N,j,k) LDPC code is a linear block code of length N having a small fixed number 'j' of ones in each column of the parity check matrix H, and a small fixed number 'k' of ones in each rows of H. A sparse ]v! x N parity-check matrix H can be viewed as a Tanner graph.A Tanner graph is a bipartite graph where the elements of a first class can be connected to the elements of a second class. In a Tanner graph of an LDPC code, elements of the first class are N variable nodes denoted by Vn corresponding to the encoded symbols and the elements of the second class are !v! parity­ check nodes denoted by Cm corresponding to the parity checks represented by the rows of the matrix H. A variable node Vn is connected to a check node Cm if and only if H(rn, n) has a non-zero entry. The Tanner graph representation of LDPC codes is very useful since their decoding algorithms can be explained by the exchange of information along the edges of these graphs. The notations related to the Tanner graph and an important hypothesis will be hereafter detailed. We take the same notation as it was done by Fossorier in [13], let M(n) denotes the set of check nodes connected to symbol node Vn (i.e. the positions of ones in the nth column of the parity-check matrix H) and let N(m) denotes the set of symbol nodes that participate in the mth parity-check equation (i.e. the positions of ones in the mth row of H). Furthermore, N(m)\n represents the set N(m) excluding the nth symbol node and similarly, M(n) represents the set M(n) excluding the mth check node. Let also <I>n,k the kth parity check constraint of M(n) with bit Vn excluded, k E {I, ... , IM(n) I}· In order to have independant equations, we consider the cycle free hypothesis. A graph is cycle free if it contains no path which begins and ends at the same check node without going backward.
