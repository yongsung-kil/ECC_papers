Fully-parallel LUT-based (2048,1723) LDPC Code Decoder for FPGA

V.Torres, A. Perez-Pascual, T. Sansaloni, J. Valls
Instituto de Telecomunicaciones y Aplicaciones Multimedia Universidad Politecnica de Valencia Valencia, Spain

Abstract— A good trade-off between performance and complexity is achieved if the min-sum algorithm with 2-bit nonuniform quantization is used to decode Low-Density ParityCheck codes. This paper proposes a method to design Variable Node Update (VNU) units based on Look-up tables suitable to design decoders for this algorithm. The method has been developed for the (2048,1723) LDPC code of the IEEE 802.3an standard and fully-parallel architectures have been implemented in a FPGA device. The results show that with the proposed method 35% area saving is achieved with respect to the use of the conventional VNU units.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] have received much research attention in recent years due to their remarkable coding performance, close to Shannon limit. They have been adopted in several industrial standards such as the European digital video broadcasting (DVB-S2) [2], Wireless Local Area networks (IEEE 802.11n) [3], Mobile Broadband Wireless Access Systems (IEEE 802.16e) [4] and 10-Gigabit Ethernet communication over unshielded twisted-pair (IEEE 802.3an) [5].
A binary LDPC code is specified by a sparse M×N paritycheck matrix H, where M represents the number of parity checks and N represents the number of bits in the code block. It can be represented graphically by a bipartite Tanner graph using H as incidence matrix, with M check nodes in one class and N variable nodes in the other. An edge exists between the variable node m and the check node n if and only if H(m,n)=1. An LDPC code is called regular when it has uniform column and row weights, defining weight as the number of 1s in a column or in a row. The LDPC code construction is very important, with computer generated random LDPC codes providing the best-known performance but also yielding high complexity hardware implementations. However, a class of LDPC codes named structured, that yield comparable performance to random generated codes, was proposed in order to reduce hardware complexity. These codes are generated by partitioning matrix H into blocks of z×z submatrices, where each sub-matrix has column and row weights

equal to one or zero for irregular codes, e.g. RS-based LDPC codes [6] or QC-LDPC [7].
The LDPC codes are decoded with message-passing algorithms: at i-th iteration, first, the variable-node update (VNU) operation is performed to compute the variable-tocheck messages λn,m, which denote an estimate of the n-th a posteriori log-likelihood ratio (LLR), and send them to the corresponding neighbor check nodes; second, in the checknode update (CNU) operation the check-to-variable messages, µm,n, are computed and sent back to the neighbor variablenodes. The message-passing decoding algorithm that achieves the best performance is the sum-product (SP) algorithm [8]. However, the SP algorithm has a high implementation complexity. A simplification called min-sum (MS) algorithm was proposed in [9]. Although, the MS algorithm causes a degradation in the decoding performance, two modifications [10], normalized-MS and offset-MS, make the MS algorithm performance closer to that of the SP. In [11] a low-complexity version of the MS algorithm that uses a 2-bit non-uniform quantification was proposed. It has a performance loss of 0.6dB with respect to normalized-MS but reduces significantly its complexity.
This paper presents a design method to implement with look-up tables (LUTs) the variable node update (VNU) units for the 2-bit non-uniform quantification decoding algorithm of [11]. The design method has been developed for the (2048,1723) RS-based LDPC code and is suitable to implement efficiently high speed decoders in an FPGA device.
The paper is organized as follows. Section II introduces the min-sum decoding algorithm with 2-bit non-uniform quantization. Section III depicts the check node and variable node update units to design fully-parallel and partial parallel architectures: Section III.A shows the conventional VNU cell structure based on arithmetic operations and Section III.B details the proposed method based on LUTs. Section IV gives the results of implementation in an FPGA device and, finally, Section V summarizes the conclusions.

III. DECODER DESIGN
This section presents the designs of the check and variable node update units suitable for a RAM-based partial-parallel [12] and fully-parallel [13] decoder architectures. In both cases the CNU and the VNU units update in parallel all the incoming messages. The design is customized for the structured (6,32)-regular LDPC code of the IEEE802.3an standard. Its parity-check matrix has a size of 384x2048, composed of 6x32 sub-matrices of 64x64 size with unitary row and column weights, so, the parity check matrix column and row weights are 6 and 32, respectively.
The half-broadcast technique [14] has been applied to reduce the number of messages (wires) that are sent from the CNU units. Therefore, instead of sending different messages from a CNU unit to each VNU unit, only one message (composed of sign, first minimum and second minimum) is sent and the VNU unit first compares this message with its current one and selects the proper one.
Due to the fact that the messages have only 2 bits, the CNU unit performs a very simple logic to compute eq. (2). Specifically, the CNU unit is only composed of three 1-bit logic functions to compute 1) the sign, 2) the first minimum and 3) the second minimum. Furthermore, the computation of the first and second minimum can be simplified as indicated in [11].
A. Conventional VNU unit structure
For the (6,32)-regular LDPC code the VNU unit calculates one output message from 6 incoming messages and the intrinsic one. A direct implementation of eq. (1) for that code is shown in Fig. 2. The first stage is composed of six 5-input LUTs, labeled CSF in Fig. 2, that perform the compare-andselect operations together with the application of the F(·) function. The CSF functions are applied to the 6 incoming messages and the F(·) function is applied to the intrinsic message. The outputs of the CSF functions are added and scaled by α. The result is added to the integer value of the intrinsic message, and the G(·) function is applied to generate the 2-bit output message. For the special case of the first iteration the output of the VNU unit is directly the intrinsic message, so a multiplexor is included after the G(·) function. In that case the INIT signal is activated.

B. LUT-based VNU units
Our proposal is to use tables to compute λn, as shown in Fig. 3. Table L1 stores the sum of 3 messages. Groups of 3 are preferred to take advantage of the FPGA LUTs sizes (for 3 messages a 6-bit input LUT is required). Function F(·) is not implemented as a separate block but as part of the L1 logic. L1 can have 16 different values (±3WH, ±3WL, ±2WH±WL, ±WH±2WL, ±WH, and ±WL). Therefore, its output can be codified with 4 bits. It should be noted that the output of L1 is not the arithmetic value of the sum of the 3 messages, but a 4bit code identifying one of the 16 possible output values. This approach reduces by 2 bits the word width of L1 and this is also important to reduce the size of table L2.
As shown in Fig. 3 table L2 stores the sum of 6 extrinsic messages. The output width of L2 would be 7 bits if it stored the arithmetic values of the sum, but as long as its output can adopt 49 different values it could be encoded using only 6 bits. Nevertheless, it is possible to take advantage of the fact that just one further addition has to be performed before applying G(·), the one with the intrinsic message. Therefore those 49 values can be classified in 9 groups so all the values in a group give the same λn after its combination with the intrinsic message. Those groups correspond with the zones shown in Fig. 4 for the values already scaled by α. Only the 4-bit group identifier is needed as input of table L3. For example, when the sum of the 6 extrinsic messages is -11.5, -11.0, -9.5, -9.0, 8.5, -7.5 or -7.0 table L2 outputs code 0001.
Finally L3 is a 6-bit input table that stores the 2-bit output message λn for all the L2 output groups and the 4 possible values of the intrinsic message. The comparison with the TL threshold is pre-computed for all the possible outputs so, as was the case with function F(·), G(·) requires no specific hardware.

V. CONCLUSIONS
A novel structure for the variable node update unit of a (6,32)-regular LDPC code decoder has been proposed. This structure is suitable for the 2-bit non-uniform quantized minsum decoding algorithm, as proposed by other authors. Our results show that working with tables and logical values, instead of arithmetic ones, a significant area saving can be achieved without spoiling the BER/FER performance.

Fig. 5 shows a further optimization of the proposed VNU cell. In this new scheme L3 stores the results λn for just 8 of the zones in Fig. 4. This comes from the fact that for zone 8 (the one in the middle in Fig. 4) table L3 output is equal to the intrinsic message and for that case the intrinsic message can be routed to the output by means of a multiplexer. In this case L3 is a 5-bit input table that stores the 2-bit output message λn for 8 of the zones. This leads to a small area saving per cell if compared with the previous version, but when the whole 2048 VNU cells are parallelized, as happens in the fully-parallel architecture, the saving may become relevant.
It should be noted that the LUT-based versions of the VNU output exactly the same messages as the conventional one.
IV. IMPLEMENTATION RESULTS
In this section, the implementation on FPGA of fullyparallel decoders for the (2048,1723) LDPC code are presented. Three decoders have been modelled with VHDL and implemented in Xilinx’s Virtex-VI XC6VCX550TL device: the first using the conventional VNU structure (Fig. 2) and two of our proposals (Fig. 3 and Fig. 5), both of them using a LUT-based structure. The three decoders have been tested and they give exactly the same BER/FER performance, the one shown in Fig. 1. This was expected as the VNUs of the three decoders have the same functionality.
The results of the FPGA implementation of the CNU and VNU units are shown in Table I for the three decoders. As can be seen the proposed LUT-based VNU requires less resources than the conventional one: the area saving per VNU is about 35%. Furthermore the proposed VNU exhibits 27% lower delay.
The implemented fully-parallel architecture for the target LDPC code calculates each iteration of the decoding algorithm in one clock cycle: first the 384 CNU units are computed in parallel and then the 2048 VNU units compute the variable-tocheck messages.
