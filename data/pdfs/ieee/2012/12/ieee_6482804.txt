Abstract— This paper presents a hardware design of a LDPC decoder where Min-Sum (MS) and Modified Min-Sum (MMS) algorithm has been investigated. The parity check matrix is complied with IEEE 802.11n recommendation (code rate of 0.5 , 648 bits block length). Verilog description language is employed for Xilinx XC2VP30-7 FPGA. With the matrix reordering technique, the final design shows that the data rate of 127 Mbps could be achieved when the pipeline stages are used.
Keywords- LDPC, modified min-sum, matrix reodering, Overlap.
I. INTRODUCTION
LDPC code or Low Density Parity-Check has lately drawn a great interest according to its excellent error-correcting capability that the code performance can be made approaching Shannon’s limit on AWGN and binary erasure channels. With this merit, when compared to Turbo codes, the decoder is slightly simpler. LDPC code now has been adopted and even further considered by a number of next generation communication standards including data storage. In the area of communication LDPC codes are now included in many standards such as DVB-S2 and DVB-T2 [2] for digital video broadcasting, Wireless Local Area Networks (WiFi) (IEEE 802.11n) [3], Wireless Metropolitan Area Networks (WiMAX) (802.16e) [4] and Wireless Regional Area Networks (WRAM) (IEEE 802.22). The IEEE802.11n, in particular, has defined matrix prototype of both n=648 (z=27) and n=1296 (z=54), and with the code rate of 1/2, 2/3, 3/4, and 5/6 respectively.
LDPC code is a linear block code defined by a sparse parity check matrix (H). The parity check matrix of the original Gallager’s LDPC code is randomly generated and the number of ‘1’ in each row and column of sparse matrix must be constant and the same. This is generally known as a regular LDPC code. The irregular ones, in contrast, this rule is not restricted. Both structures of LDPC codes are widely studies and reported in a number of published literatures. The major concerns of most works to LDPC codes can be divided into two main streams. The first one is focusing on how to construct an efficient parity check matrix of which the best performance of the code can be achieved. The second one is how to construct and implement a good decoder.

Tanner’s graph [1] can describe well the relation between a code word and its corresponding check matrix. M Rows and N columns of the check matrix can be directly transformed to M check node units and N bit node units, respectively. The 1’s in the parity check matrix can be transformed to the direct connection between corresponding check node units and bit node units. In other words, the two ends of one connection belong to different computational units. By passing message on the straights in the bipartite graph, the mutual information can be exchanged iteratively. In the believed propagation (BP) or sum product (SP) decoding algorithm, the passing message is computed for its probability and modified iteratively. According to the burden computational load of the sum product method, a modified version named as the logarithmic sum product has been later on offered. A said modified version has been further developed to reduce the computational complexity. The min-sum algorithm is finally achieved. The min-sum algorithm for LDPC codes, have been preferably implemented in hardware form, i.e., FPGA or ASIC. The main applications are wireless communications. This is because only hardware decoding that can satisfy the demanding of high speed processing. In [5] an overlap technique is used by reordering the parity check matrix of IEEE 802.16e system. [6] also used an overlapping method in their implement of IEEE 802.11n system. An alternative hardware architecture has been proposed but the parity check matrix is still left untouched. In this work, in contrast, we combine the re-ordering method proposed by [5] and architecture modification reported by [8]. On system level and quite standard architecture, [7] had suggested the hardware implantation of a LDPC Decoder.
In this paper, we consider reducing clock cycle of the decoder using a modified min-sum algorithm (MMS) together with reordered parity check matrix H. The MMS is a modified version of a min-sum algorithm (MS). It offers slightly better performance when compared to its original MS: -- however at the cost of higher hardware complexity. In the min-sum algorithm, row operations and column operations are alternately executed. Only row-row (or column-column) operations can be executed in parallel (in one iteration). Rowcolumn operation and column-row operation of any same iteration cannot be executed in parallel. In this paper, we report technique of re-ordering parity check matrix to enable a certain degree of parallelism. III. DESIGN TECHNIQUES FOR LDPC DECODER
Although the decoder hardware can be designed based on several approaches, we did target lowest power consumption and smallest chip area (or lowest gate counts) criteria. In this section, the design technique that can increase the processing speed while keeping not so excessive computing units is detailed. Such a technique is made achieve by mean of CNU and VNU overlapping computing arrangement.

A. Parity Check Matrix Reordering
In a LDPC decoding process, the bit node operation of first column needs to wait until the completion of the check node operations. In order to reduce the idle waiting time among those operations, it is possible to improve the situation by reordering the rows and columns of matrix H [5]. Such an effort does not make any influence on the error-correcting performance. As there are many possible ways to reorder the parity check matrix H. An important rule is:- row and column must be moved at the same time. More details about matrix reordering can be founded in [9].
B. Overlapped VNUs and CNUs Computations

Figure 1. Overlap patterns of CNUs and VNUs
Consider patterns given in Fig. 1, an iteration is defined as one round computation of row update (CNU) and column update (VNU). An operation cycle is just for one CNU or one VNU. This can be also made parallel too. Mathematical analysis; with no overlapping (pattern 0), 4 rows are processed in parallel (need 108 CNUs) and 8 columns are also processed in parallel (need 216 VNUs), therefore we need 120 cycles for all 20 iterations. Pattern 1: 4 rows, 8 columns per one cycle; requires 108 CNUs and 216 VNUs. The overlapped cycle is [(4*Ite)+1] and non-overlapped is [6*Ite], Ite denotes the number of iterations. A. Throughput and Pipelining

A first codeword is delivered after output data depermutation. The latency of 1636 clock cycles was measured. With the target device, the maximum clock frequency is 194 MHz. The clock rate is very much depended on the target device according to its fabrication technology. We have limited the iteration number to 5. Thus the implementation can delivered a maximum decoding throughput of (648u 194MHz) /(5 1636) 76.6Mbps .

Since the decoder has been designed with 3 main blocks; permutation, iteration process (VNU and CNU) and depermutation, it is possible to insert pipeline stages to increase the throughput. It can be noted that when the iteration computation is busy, both input permutation and output permutation are idle. Then there is a chance for us to make those blocks busy at most the times. We thus can insert pipeline registers at where we need.

VI. CONCLUSION
In this paper we have presented a reordering technique applied to an IEEE802.11n standard parity matrix. Matrix reordering enables us to arrange overlapping computation scheme that suits well MS or MMS algorithms needed in a LDPC decoder. To achieve the high degree of overlapping computation, unfortunately, only the code with 648 bits block length is applicable. This is because a matrix with several null sub-matrices can provide us a higher degree of matrix rearrangement.
