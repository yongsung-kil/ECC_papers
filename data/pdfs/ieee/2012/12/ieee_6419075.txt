A Layered QC-LDPC Decoder Architecture for
High Speed Communication System
Chiu-Wing Sham, Xu Chen, Wai M. Tam, Yue Zhao and Francis C.M. Lau
Department of Electronic and Information Engineering
The Hong Kong Polytechnic University
Abstract—The performance of a high-throughput long-distance
communication system such as an optical transmission system is
limited by the Net Coding Gain (NCG) of the Forward Error
Correction (FEC) system. Summarizing the previous research
works, Low-Density Parity-Check (LDPC) codes form one of the
most promising FEC schemes to be applied in high-throughput
communication systems. Designing a practical channel coding
scheme with high code rate, low complexity, high through-
put and extremely low error floor has always been a very
challenging problem. Quasi-cyclic low-density parity-check (QC-
LDPC) codes have been promising candidates to fulfill the above
requirements but the implementation issues remain. In this paper,
we propose a layered QC-LDPC decoder architecture with high
code rate, low complexity, high throughput and excellent error
performance. The architecture has been implemented using FPGA
and the error performance has been shown to be good.
I. INTRODUCTION
As the growth of the network traffic load is expected
to be a long-term trend, there is a great need to increase
the high network capacity. Compared with other possible
solutions, a strong but simple Forward Error Correction (FEC)
is the most desirable cost-effective candidate to recover the
noisy signal in long-distance communication systems. The
net error-correction power of a FEC system is known as
the Net Coding Gain (NCG). Among the different coding
schemes, Low-Density Parity-Check (LDPC) codes [1], [2]
form one of the most promising FEC solutions. Quasi-cyclic
LDPC (QC-LDPC) code is a class of LDPC codes with the
parity-check matrix consisting of circulant permutation sub-
matrices. Because of its regularized structure, the encoders
can be implemented with very low complexity [3], [4]. It
is evident that QC-LDPC codes represent a promising FEC
scheme [5]–[7] for the next generation optical transport net-
works. However, there are several challenges to be overcome.
First, the code rate should be greater than 80%. Second, a
higher NCG should be accomplished. It is necessary because it
implies that repeaters can be installed with a larger separation
distance. Hence, the cost of the optical system can be reduced.
Third, the decoder should be implemented with a reasonable
complexity. A decoder with a lower complexity represents a
lower fabrication cost and lower power consumption. Min-
sum (MS) algorithm is widely used [8]–[15] in the decoding
process of LDPC codes because it has a lower complexity.
Normalized/offset MS algorithms [16] are also proposed later.
However, the quantized MS decoding becomes less accurate
compared with the quantized Sum Product Algorithm (SPA) as
the check-node degree increases and the code rate is high [8].
In general, the previous architecture is not suitable for the FEC
system of optical communication because they cannot achieve
high code rate and NCG.
In this paper, we propose a layered QC-LDPC decoder
architecture with high code rate, low complexity, high through-
put and excellent error performance. First, our proposed
RAM-based architecture applies the dynamic storage of only
the quantized Check node-to-Variable node (C2V) or Vari-
able node-to-Check node (V2C) message for each connected
variable-check pair at any time. The memory storage re-
quirement is substantially reduced compared with existing
architectures [14], [15]. This architecture does not require
any shuffled network so the routing complexity is further
reduced. Second, we propose a novel Check-Node Processor
(CNP) which evaluates the C2V messages based on quantized
SPA. Its complexity is similar to that of MS decoding but
a much higher NCG can be acheived. Third, we propose a
deep pipeline scheduling such that all the CNPs and Variable
Node Processors (VNPs) will not be idle at any moment. The
throughput of this decoder can be increased with a lower com-
plexity. Fourth, we propose varying the number of iterations
of each codeword dynamically. This allows some codewords
that are more corrupted to go through more iterations.
The rest of the paper is organized as follows. Section II
reviews the basic concepts of QC-LDPC codes. Section III
elaborates the detailed RAM-based decoder architecture. We
explain the details of the main techniques including Dynamic
Storage of C2V and V2C Messages, Quantized SPA Computa-
tion in CNP and VNP and Deep Pipelined Scheduling of CN
and VN Processes. In Section IV, we show the performance
results in terms of hardware complexity, throughput and bit
error performance. Section V concludes the paper.

III. RAM-BASED ARCHITECTURE OF THE DECODER
The overall architecture of the QC-LDPC decoder is il-
lustrated in Fig. 1. In our decoder, the V2C messages are
stored in the sign-magnitude (SM) format whereas the channel
messages and the C2V messages are stored in the two’s
complement (2C) format. For each of the 𝐽layers, the V2C
messages, the channel messages and the C2V messages are
input to the layered decoder. First, the CNPs receive the
V2C messages and calculates the updated C2V messages.
The new C2V messages are converted to the 2C format.
These new C2V messages, together with the channel messages
and other C2V messages, are then used by the VNPs to
compute the updated V2C messages. Throughout the process,
the read/write operations of the RAMs are controlled by the
address controller. Once the address for each RAM is initiated
at the beginning of the processing of each layer, it keeps
incrementing by one so as to process the next layer. As a
result, the address controller is a simple counter thanks to the
quasi-cyclic structure of the QC-LDPC codes.
Some design techniques are applied in order to achieve high
code rate, low complexity, high throughput and excellent error
performance by this layered QC-LPDC decoder architecture.
First, our proposed RAM-based architecture requires the dy-
namic storage of only the quantized C2V or V2C message for
each connected variable-check pair as well as the quantized
channel messages at any time. In other words, for a particularly
memory location, it stores a C2V message at one time but
stores a V2C message at another time, and vice versa. Second,
a novel CNP is proposed which evaluates the C2V messages
based on quantized SPA in order to achieve a high NCG.
Its complexity is similar to that when MS decoding is used,
while the processor is implemented based on a look-up table.
Third, a deep pipeline scheduling is proposed. It allows more
codewords to be processored simultaneously. Consequently,
the throughput of the decoder can be increased with similar
complexity. We show that for a code with 3 check-node groups
per decoding layer and 4 decoding stages per hardware layer,
the throughput can be increased by 100% compared with the
conventional pipeline scheduling. Although more hardware
resources are required to store the messages of the additional
codewords, these can be easily implemented with the use of
more RAM blocks in our RAM-based decoder architecture.
Fourth, the number of iterations of each codeword can vary
dynamically. We can stop the decoding process for a particular
codeword if that codeword is found valid after checking.
The resources (number of iterations) saved can be applied to
decode other codewords which may need more iterations to
converge.
A. Dynamic Storage of C2V and V2C Messages
In our design, the newly updated V2C messages, which
will only be used in the next layer but not other layers,
are converted to the SM format. Subsequently, the updated
C2V messages (in 2C format) and the V2C messages (in SM
format) are output. The updated C2V messages are stored in
the RAMs corresponding to the input V2C messages, i.e.,
the RAMs storing the input V2C messages of layer 𝑗are
over-written by the updated C2V messages of the same layer;
whereas the V2C messages are stored in the associated RAMs
which are going to be accessed in the next layer. Since each
check node is connected to 𝐿VNs (i.e., row weight = 𝐿),
the corresponding CN requires 𝐿V2C messages from the
same layer (i.e, Layer 𝑗) in computing the 𝐿updated C2V
messages during the decoding process. In order to load these 𝐿
V2C messages simultaneously to the decoder without memory
collision, we allocate 𝐿separated RAM blocks (or simply
RAMs) to store these V2C messages.

We compare our quantized SPA with the quantized offset
MS and the quantized MS. The error performance results are
shown in Fig. 2. The code is a (18432, 4, 24) regular QC-
LDPC code. The quantized offset MS uses a conventional
non-layered scheduling with 24 iterations. For the quantized
MS and the quantized SPA, a layered scheduling with 10
iterations is applied. We can see that our decoding method
with quantized SPA performs much better than the other two
methods. The performance difference between quantized SPA
and quantized offset MS is 0.14 dB at a bit error rate of 10−6.
Note that this difference increases with the CN degree. For a
code with a higher code rate, the number of block columns and
hence the CN degree is larger. Consequently, the performance
difference between the quantized SPA and the quantized offset
MS becomes larger. Thus, we use our proposed quantized SPA
in our QC-LDPC decoder because of its better performance.
