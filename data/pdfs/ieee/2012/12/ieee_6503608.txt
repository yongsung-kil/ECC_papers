Abstract—In this work, we propose a scheme to extract soft information from hard-decision outputs in multi-level per cell (MLC) flash memory based on a cell-to-cell interference model. It will be shown that the soft information extracted in the form of log-likelihood ratio (LLR) by taking into account a dominant cell-to-cell interference term provides significant performance improvements when the error-control system is designed with an error-correcting code with iterative decoding algorithm, e.g. low-density parity-check (LDPC) code with the Belief-Propagation (BP) algorithm. To confirm the claims, we design error-control systems with a conventional Bose-ChaudhuriHocquenghem (BCH) code and an LDPC code with/without the soft information extraction. Performances of the systems are extensively evaluated and compared, which clearly shows that the soft information extraction based on the cell-to-cell interference model leads to a considerably better performance.
I. INTRODUCTION
Recently, there has been ever growing demand for storage devices using NAND flash memory due to its attractive features such as low power-consumption, compactness, and high data throughput. Although the NAND flash memory based storages have taken an increasing market share at a fast rate, their expensive bit cost as compared to its competitors, e.g. magnetic storages, hinders the path to their dominant position in the market. This motivates the semiconductor industry to aggressively pursue bit cost reduction by various approaches. Through continuous technology scaling, a higher density can be achieved to reduce the bit cost. NAND flash memory chips at 20nm technology are already widely used in industry. Furthermore, multi-level per cell (MLC) technology enabling a memory cell to store more than 1 bit has been developed to increase the storage density of NAND flash memory. In current design practice, 2 bits per cell NAND flash memories are most prevailing in the market but possibilities of 3 and 4 bits per cell NAND flash memories are extensively explored [7], [15].
Data integrity, i.e. reliable data retrieval, is one of the key requirements of data storages and generally ensured by relying on error-correcting codes (ECCs). Errors and erasures in NAND flash memories stem from various factors such as cellto-cell interference, charge leakage, read/write disturbance and over-programming. For flash memories with single-level cell (SLC) technology where each cell can store only a binary digit, simple ECCs with a modest number of redundant bits such as 3- or 4-bit-error-correcting Bose-Chaudhuri-Hocquenghem

(BCH) codes are sufficient to meet a required performance such as a target Page-Error Rate (PER) [1]. However, as the storage density gets higher, the reliability of the information bits stored in MLC NAND flash memory becomes gradually degraded and in turn deteriorates the data integrity. In particular, the MLC technology and the technology scaling make the noise margin smaller and the cell-to-cell interference from adjacent cells stronger, respectively. To address these issues, as the storage density grows, ECCs for flash memories have been consistently evolved to ones with stronger error-correcting capabilities, e.g. 24- and 55-bit-error-correcting BCH codes for NAND flash memories are commercially available [14]. In addition, there have been efforts to adopt more powerful error-control systems which consists of ECCs with iterative decoding algorithms such as low-density parity-check (LDPC) codes [18] with belief-propagation (BP) decoding algorithm and concatenated BCH codes [2] with iterative hard-decision decoding and/or reliability based decoding algorithms for improving error-correcting capability.
It is generally known that LDPC codes with the BP decoding algorithm provide much stronger error-correcting capabilities than the conventional BCH codes. However, the performance superiority is significantly sacrificed when only hard-decision results from NAND flash memories are available at the decoder. Unfortunately, in current practice NAND flash memory interface provides only hard-decision results to an off-chip error-control system. To overcome this technical challenge, a technique called read retry in which the charge levels on floating-gates are sensed multiple times with different reference voltages is considered as a practical solution. The read retry provides an equivalent discrete channel with a larger size of output alphabet than binary, and the mutual information between the input and output of the channel depends on the choices of the reference voltages. In [18], Wang et al. show that LDPC codes over the equivalent channel provide significant performance improvements when the reference voltages are chosen to maximize the mutual information. In spite of the performance improvements, the read retry technique usually results in a long latency and reduces throughput since the same user data must be read multiple times. Thus, it is desirable that the read retry is selectively performed only if the LDPC decoding fails with hard-decision results. In recent work, the cell-to-cell interference is extensively studied from theoretical perspectives [3], [4], [11], [12] and modeling perspectives [8], [9]. To see the effectiveness of the proposed idea, we design an LDPC code and evaluate the performance with/without the soft information. It will be presented that the soft information significantly reduces the decoding failure rate which leads to the decrease of the occurrence of read retry.

The amount of charges stored during writing/programming in the floating gate is quantized to q levels to express log2 q bits of information. During a reading operation, the amount of charges on the floating gate is measured by sensing the threshold voltage of the transistor through the control gate. In NAND flash memory illustrated in Fig. 2, the array of memory cells called the word, shares a common word-line in which all memory cells or a part of them are programmed or read simultaneously. For programming, we only consider 2 bits per cell NAND flash memory with multi-page programming [10], [17] and even/odd bit-line structure [13] in which the memory cells in a word are grouped into four pages, a basic user data unit to be read from and programmed to NAND flash memory: MSB (most significant bit)/even/odd bit-lines, and LSB (least significant bit)/even/odd bit-lines. For example, an MSB/even bit-line page consists of MSBs of memory cells on the even bit-line. It is assumed that the pages are programmed in the following order: LSB/even bit-line → LSB/odd bit-line → MSB/even bit-line → MSB/odd bit-line.
The conditional distributions of the threshold voltage are affected by noise, process variability, and cell-to-cell interference from programming of other memory cells among which the cell-to-cell interference rises as the dominant factor due to the aggressive technology scaling. A cell called as the victim cell has interference from physically adjacent cells called the interfering cell which are programmed after the victim cell.
