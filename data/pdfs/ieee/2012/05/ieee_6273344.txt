- Abstract Through researching the structural characteristics of
LDPC parity matrix and algorithm data flow features of LDPC code, an improved type of algorithm decoder of LDPC code is designed in this paper. The decoder includes expandable storage array, exquisite address-controlling unit and powerful sequential state-controlling machine. It possesses the advantages of expanding the decoding code length flexibly and lower complexity for hardware realizing and higher hardware resource utilization rate. After constructing communication system and testing the performance of hardware decoder, the results showed that the performance of the decoder and the theory simulation value can fit each other perfectly. The correctness of the design is proved. The decoder which is designed in this paper can provide valuable reference for the development of general chips for LDPC decoder.

III. THE DESIGN OF LDPC DECODER In 1.1, the sub matrix of Matrix H is classified into three types and they are unit matrix, cycle shift matrix of unit matrix and entire zero matrix. During the decoding process, the data in the sub matrix can be processed serially and the sub matrixes which don't have data transmission between each other can be processed parallel. So now we can get the data process of LDPC decoder which is designed based on partial parallel mechanism, and the hardware overall structure is showed in Fig1. The following are decoding steps. Step1: Serial and parallel conversion module makes partition with the received initialized decoding information according to the structure characteristics of the check-up matrix. Step2: After the process of data partition, it is stored in storage array for further calling of data processing unit.

V. CONCLUSION
A new type of LDPC decoder is designed in this paper, the advantages are listed below: CDa good scalability. If the code length of LDPC decoder is needed to increase, it is only needed to increase the number of RAM modules, there is no need to change the sequential controlling module. \2l The complexity of realizing hardware is low and complicated address controlling can be realized by using simple counting machine. It's easy to realize the master-slave structure sequential state machine, and the performance is very powerful. (3)The utilization rate of hardware resources is high and storage array can be used repeatedly. The address of nonzero elements in the check-up matrix can be recorded by using counting machines. By using these methods, large amount of storing space is saved and the utilization rate of hardware resources is improved.
