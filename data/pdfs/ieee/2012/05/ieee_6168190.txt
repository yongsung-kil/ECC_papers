On the Overflow Problem in Finite Precision Turbo Decoding Message Passing
Wojciech Sułek

Abstract—Much attention has been paid recently to the socalled layered decoding of LDPC codes, also known as turbodecoding message passing (TDMP). The TDMP algorithm for decoding LDPC codes is known to possess some desirable features, such as fast convergence speed, reduced memory requirements and reduced implementation complexity in comparison with standard two-phase message passing algorithm. In this paper we analyze an important issue connected with hardware implementation of TDMP algorithm, namely the finite precision representation of messages influence on the decoding performance. Constrained dynamic range of the finite precision representation of messages entails overflow errors. We present an analysis revealing that in the subsequent decoding iterations, the subtraction of non-overflowed intrinsic message from overflowed extrinsic message is a source of errors that have substantial impact on the decoding results. The analysis is confirmed by simulation results showing significant performance loss. However this performance loss can be almost completely eliminated with a basic modifications in the messages computation algorithm. Effectiveness of the presented modifications is confirmed by simulation results obtained with hardware TDMP decoder implementation that has been developed.
Index Terms—Low-density parity-check codes, iterative decoding, turbo decoding algorithm, overflow errors.
I. INTRODUCTION
T HERE is no doubt that Low-Density Parity Check (LDPC) codes are nowadays one of the most attractive error correcting coding methods considered by the scientific as well as industrial communities. First introduced by Gallager [1] and recently rediscovered [2], LDPC codes have outstanding error correction performance and a very large degree of parallelism of the decoding algorithm.
LDPC codes are decoded iteratively using the sum-product algorithm, also known as belief propagation [2], which closely approximates maximum-likelihood decoding. The algorithm was originally formulated in the form of two-phase message passing scheme, where a decoding iteration is divided into two rounds of computations, pertaining to variable and check nodes of the code graph respectively. However there exist variations of the standard message passing scheme that possess some desirable features. One such variation that has been studied by many in one form or another is labeled as layered decoding [3]–[7] or turbo-decoding message passing [8], [9].
An essential characteristic of layered decoding is the overlap of the standard variable and check node phases. Each

iteration is a concatenation of several layer elaborations or sub-iterations. Within a layer, the updated reliability of a bit in the codeword is immediately available to next layers. Therefore the updated information is utilized more quickly in the algorithm, thus speeding up the decoding [3]. Mansour et al. [8], [9] viewed an LDPC code as the intersection of interleaved super-codes with every decoding layer being associated with a single super-code. The decoding process employs constituent super-codes soft-input soft-output (SISO) decoders that communicate through interleavers. Therefore because of the similarity to the turbo-codes [10], the decoding algorithm is called turbo-decoding message passing (TDMP).
In [8], [9] the TDMP algorithm is thoroughly described and its advantages over the standard two-phase message passing are enumerated, which are mainly: 1) faster convergence, 2) improvement in coding gain at high signal-to-noise ratio (SNR), 3) reduced memory requirements, and 4) reduced decoder implementation complexity. The decoder chip implementing TDMP algorithm for architecture-aware subclass of LDPC codes (AA-LDPC) is presented in [11]. However there exist an important issue that has not been considered so far, connected with finite precision representation of messages in TDMP hardware decoder implementation.
Fixed-point numbers representing messages are characterized by a finite range as well as quantization, which is a source of overflow and quantization errors respectively. The influence of finite precision representation of messages in LDPC hardware decoders has been studied by several researchers [12]–[14]. The wordlength-performance tradeoff is well known, also some sophisticated quantization methods has been proposed, such as non-uniform quantization [15] and dual quantization domain [14]. However in the quantized TDMP implementation, another finite precision effect is crucial. In this paper we present an analysis revealing that in the subsequent decoding iterations, the subtraction of intrinsic messages from extrinsic messages burdened by overflow error (which is an operation specific to the TDMP decoding) is a source of errors that have very substantial impact on the decoding results. This problem is specific to a turbo-like dataflow and was previously analyzed in [16] with regard to Turbo codes fixed-point decoder implementation. We call this effect an overflow error burden, however it should be noted that the effect is not connected with the overflow in itself, but with the mentioned subtraction of “non-overflowed” data from “overflowed” data. In this paper we provide an analysis for TDMP finite precision LDPC decoder implementation as well as propose modifications of the decoding algorithm that significantly reduce the performance loss.

The paper is organized as follows. In the next section the TDMP algorithm description is recalled as well as the TDMP decoder implementation that has been developed is briefly presented. Section III that follows contains description of the overflow burden problem connected with limited range of quantized messages. We propose a model for the finite precision TDMP decoder operation, present formulas for the overflow errors calculation and explain significance of the problem. V. CONCLUSIONS
It has been presented that straightforward implementation of the TDMP decoding algorithm with quantized messages leads to overflow problem that significantly reduces decoding performance. The problem is connected with the subtraction of (non-overflowed) intrinsic messages from (overflowed) extrinsic messages, which is an operation specific to the turbolike decoding. We called this effect an overflow error burden and presented precise analysis of this phenomenon. Then we proposed modifications of the TDMP decoding algorithm that includes:
• Elimination of the main overflow error component. We called this modification “Stage I” error reduction.
• Several heuristic methods for the remaining overflow error components reduction (“Stage II”). Based on the decoding performance as well as decoder synthesis results, we recommend combined modifications that we called Normalized ρ and Limited λ.
Decoding results that are included show that performance of the TDMP FP decoder with the proposed modifications is near the performance of the full-precision decoder. With the proposed modifications all the mentioned virtues of TDMP decoding are maintained, whereas finite precision implementation has acceptable decoding performance.
