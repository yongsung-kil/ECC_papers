Abstract—1 Today’s mainstream electronic systems typically assume that transistors and interconnections operate correctly over their useful lifetime. For coming generations of silicon technologies, several causes of hardware failures, such as erratic bit errors, transient (soft) errors, and process variations, are becoming significant. In contrast to the traditional redundancybased reliability solutions, the aim of a probabilistic design is to achieve high quality results and efficiency using erroneous or imperfect components along with a judicious allocation of resources. In this paper we focus on a probabilistic analysis of an LDPC Gallager B decoder made out of unreliable hardware components. Our analysis reveals the dependencies between the final BER at the output of the decoder and the errors in the components of the decoder. We demonstrate that a system design guided by our analysis can produce higher quality results compared to an arbitrary resource allocation. This resource allocation is of particular relevance to emerging storage applications that need to maintain extremely high levels of reliability even as the underlying technology scales deep into the nano-regime.
I. INTRODUCTION
In classical communication theory, the hidden assumption behind the analysis of communication systems – including the transmitter, the channel, and the receiver – is that the channel is noisy but that the transmitter/receiver pair is noisefree. In the analysis of data storage devices, the assumption is that the encoding/decoding hardware is error-free. However, in modern CMOS technology, hardware reliability is facing many challenges such as erratic bit errors, transient (soft) errors, early-life failures (infant mortality), transistor aging, process variations and variations in operating conditions (e.g., voltage drops, temperature variations) [1], [2]. A traditional approach to achieving high reliability is to employ conservative design choices (e.g., guardbanding) or expensive redundancy schemes. However, the overhead of such traditional techniques is becoming unbearable [3]. One approach to reliable and efficient system design is to cooperatively employ multiple layers of the system for improved error resilience [9]. One example of such a principle is the Error Resilient System Architecture (ERSA) [9]. ERSA is based on a probabilistic design and achieves cost-effective system architecutre by using the application’s inherent error resilience to allocate hardware components with heterogeneous reliability levels according to the importance of the computation.
In this paper we study a probabilistic design of a popular low density parity check (LDPC) iterative decoder, as a representative example of iterative belief propagation schemes,

implemented on noisy hardware. We assume that the decoder runs the Gallager B algorithm [7]. The algorithm can produce sequences with arbitrary low bit error rate (BER) for small enough probability of transmission error. We investigate how the BER of the output sequence changes when computations are performed by erroneous processors. Our analysis follows the evolution of errors iteration-wise over the computation tree of the Gallager B algorithm. We provide a simple, approximate expression for the final error in the decoded sequence in terms of both the errors induced by the communication channel and by the errors in the processors. Finally, we use an implementation on ERSA to verify our results for a practical LDPC code. Our results can therefore serve as guidelines for the design of economically efficient LDPC decoders and memory units built out of less reliable computational resources.
In this paper we assume that, due to the faulty nature of the gates, the output of each node of the computation tree is erroneous with some fixed probability which is independent of errors at other nodes. The analysis of the decoding error for this model was done for the Gallager A algorithm in [16] (a simpler version of the Gallager B algorithm), and for the Gallager B algorithm with one iteration in [4], [13]. To the best of our knowledge, the current work is the first that considers a faulty Gallager B algorithm with multiple iterations.
In Section II we introduce the structure of an LDPC decoder made out of faulty gates and compute the overall BER. In Section III we derive some practical implications of our analysis regarding the design of error resilient systems. In Section IV we present simulation results of Gallager B decoder implemented on erroneous hardware, using ERSA hardware [9]. In Section V we conclude the paper.
II. FAULTY LDPC DECODER
In this section we analyze the error behavior at the output sequence of an LDPC decoder based on a Gallager B algorithm made out of erroneous gates.
A. Gallager B algorithm
We consider the two-level hard decision Gallager B algorithm decoder [7] which sends binary messages over the edges of the associated computation tree. The Gallager B algorithm on a binary symmetric channel (BSC) also serves as a useful proxy for more complex iterative decoding algorithms (e.g., sum product) in the region of very low BER, a region that is of primary interest in data storage applications. Consider a regular LDPC code with n variable nodes. Suppose that each variable node has degree dv and each check node has degree dc. Gallager B algorithm operates as follows: At iteration 0, each variable node sends its received value from the channel to the neighboring check nodes over the incident edges. In the decoding iteration i, the message sent from a check node to a neighboring variable node is the sum modulo 2 of all incoming messages except the one arriving from that variable node. After receiving the messages from all incident check nodes, each variable node sends a new message to each neighboring check node. The message sent from a variable node to a neighboring check node depends on all incoming messages except the one arriving from that check node.
Suppose that the majority threshold at iteration i is bi and that bi > (dv − 1)/2. The message from a variable node to a check node has value h ∈ {0, 1}, if at least bi of all incoming messages except the one arriving from that check node have value h, otherwise the variable node sends the message originally received from the channel to that check node. Similarly, if at least bi of the messages received from all neighboring check nodes at iteration i are h ∈ {0, 1}, the decoded message is h. Otherwise, the decoded message is the original message received from the channel at that variable node.

B. Analysis of faulty Gallager B decoder

To analyze the performance of a faulty Gallager B decoder we follow the assumption that the Tanner graph of the code is cycle-free, and therefore the messages sent by the nodes at each iteration of the decoding process are statistically independent [7]. In modern storage technology the block length is increasing [8]. In this setting the independence assumption becomes more realistic [14].
We assume that the output of each check node is flipped with some fixed probability pXOR and that the output of each variable node is flipped with some fixed probability pMAJ. We assume that the binary symmetric channel (BSC) flips each bit with probability p0. Therefore, at the 0th iteration, the estimate at each variable node has the probability of error p0.

IV. EMULATION RESULTS
We tested an LDPC Gallager B decoder on a probabilistic hardware platform to verify the analyzed LDPC decoder model and to highlight the effectiveness of using a theoretical analysis in designing probabilistic hardware. While the simulation can be easily done in software by modifying the conventional decoder, we used hardware for a more convincing demonstration. We performed experiments on an ERSA prototype implementation based on BEE3 FPGA platform [6]. We used a regular geometric projective code of type 1 [10] with dv = dc = 33 and n = m = 1053 on a BSC channel with p0 = 0.01.
The ERSA prototype has 1 Super Reliable Core (SRC) that reliably computes control-intensive operations and 8 Relaxed Reliability Cores (RRCs) that compute parallel computation tasks using low-cost unreliable hardware components. For our LDPC Gallager B decoder application, we mapped the main thread, RRC task management operations, and I/O operations to the SRC. Using our LDPC Gallager B decoder analysis, we predicted that using more reliable hardware for the variable node computation will deliver better result quality. Figure 3 confirms that the error injections into the variable node computations have bigger impact on the resulting decoding quality compared to the check node computations.
To demonstrate how a theoretical analysis benefits system design, we compared two different task mapping schemes on the RRCs in ERSA. The analysis-guided task mapping scheme utilizes the knowledge from the analyzed model that the variable node computations have bigger impact on the decoding quality compared to the check node computations. Instead of having uniform reliability levels across the entire RRCs, we divide 8 RRCs into two groups of 4 RRCs and apply different error injection rates for each group. The variable node computations are mapped to the RRC group with the lower error injection rate, while the check node computations are mapped to the group with the higher injection rate. The arbitrary task mapping scheme, in contrast, does not distinguish between task types and assigns any computation task to an arbitrary RRC. Figure 4 shows the comparison result of mapping schemes using various RRC error rates configurations. The results show that if we use the analysisguided task mapping scheme with non-uniform error rate configurations (configuration C1-C4), we can have a better output quality compared to the uniform error rate configuration or the arbitrary mapping scheme.
