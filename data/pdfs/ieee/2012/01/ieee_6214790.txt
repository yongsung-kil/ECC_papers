Abstract—This paper presents an asynchronous scheduling algorithm for high-throughput stochastic low-density paritycheck (LDPC) decoders. Stochastic computation provides ultralow-complexity hardware and can be implemented using binary or multiple-valued logic gates. Using asynchronous control, it also eliminates a global clock signal and therefore eases the worst-case timing restrictions. A timing model of asynchronouscomputation behaviours under a 90nm CMOS technology is used to demonstrate that the proposed algorithm with an optimized computation delay properly decodes a regular (1024, 512) LDPC code without the “lock-up” problem that potentially stops decoding before convergence and hence causes loss in coding gain. Based on our models, the proposed scheme achieves up to 7.37x improvement in decoding throughput with comparable BER performance in comparison with performance results of a conventional synchronous stochastic decoder.
Keywords-soft computing, stochastic computation, computer arithmetic, iterative decoding, forward error correction codes, communication systems, asynchronous circuits, circuit implementation
I. INTRODUCTION
Low-Density Parity-Check (LDPC) [1], [2] codes are forward-error-correcting codes that perform very close to the Shannon limit. LDPC codes have been used in digital communication standards, such as WiMAX [3], WiFi [4], and 10GBASE-T [5]. LDPC codes are typically decoded by using the sum-product algorithm (SPA) or one of its variants, which operates by iteratively passing a posteriori probability or log-likelihood ratio (LLR) soft-valued messages between two computation nodes on a factor graph [6].
Parallel decoder implementations tend to achieve high throughput, but the wiring complexity and large area consumption leads to an increase in the maximum wire length, which imposes an upper limit on achievable speed [7]. While some approaches have been proposed [8]–[13] to alleviate the wiring complexity, the area of computation nodes remains large. Stochastic decoding [14]–[21] is a technique where probability messages are represented by the statistics of a Bernoulli sequence. The computation nodes are implemented using simple binary or even multiple-valued logic [20], [21] gates and data transmission between the nodes is performed serially, which leads to low complexity decoders. However, high-speed clocking is required and the power dissipation of the clock tree will make up a large portion of the power

budget. Moreover, the wiring delay via an interleaver tends to have variations from wire to wire due to the complexity in the chip implementation causing very long interconnects, which limits the clock frequency and hence the throughput. The clock-related problems will be serious in large LDPC codes, which have better error correcting capabilities than that of shorter codes.
In this paper, we propose an asynchronous scheduling scheme for stochastic decoding of LDPC codes. An asynchronous scheduling algorithm has previously been proposed for min-sum decoders [22], but not for stochastic decoders. A wire-efficient data-transmission circuit based on multiplevalued logic was proposed for asynchronous min-sum LDPC decoders at ISMVL 2005 [23] and the asynchronous decoders designed using multiple-valued [12] or binary logic gates [13] achieve low energy consumption while maintaining comparable error-correcting performance to conventional decoders. In the scheduling scheme, all computation nodes and data transmission between nodes operate asynchronously using requestand-acknowledge handshaking. Consequently, throughput is not limited by the critical-path delay. Moreover, the use of the proposed algorithm eliminates the clock signal entirely in hardware implementation, which solves several clock-related problems. However, stochastic decoding has the potential of “lock-up” that stops decoding before convergence, causing a coding gain loss [16]. In synchronous decoders, this issue is resolved by using arithmetic blocks known as “edge memories” and through noise-dependent scaling [15]. To investigate the decoding dynamics of asynchronous decoders, we develop a timing model of asynchronous-computation behaviours and validate the proposed algorithm. This model can be applied to stochastic decoding of binary or nonbinary (multiple-valued) LDPC codes, but in this paper we investigate the characteristics of asynchronous stochastic decoding using binary codes.
The rest of the paper is organized as follows. A review of stochastic decoding and the proposed asynchronous stochastic decoding scheme are described in Section II. Section III describes out model and how it is used to simulate bit-error rate (BER) performance. Section IV describes the convergence dynamics of the proposed stochastic decoder and provides performance comparisons with conventional decoding under a 90nm CMOS technology. Finally, we conclude this paper in Section V.

II. ASYNCHRONOUS STOCHASTIC DECODING
A. Overview of LDPC codes
An LDPC code is defined by a sparse (n-k) x n parity-check matrix H. The transmitted data x containing k information bits and (n-k) parity bits over the channel is encoded using a generator matrix G, where GHT =0 with operations typically (but not always) over GF(2). An LDPC code can be represented by a factor graph [6] with n variable nodes and (n-k) paritycheck nodes as shown in Fig 1. A variable node is connected to a parity-check node when the corresponding element in the matrix H is nonzero (1 in the specific case of a binary LDPC code).
An LDPC code is often decoded using the sum-product algorithm or one of its variants, which operates by iteratively passing a posteriori probability messages along the edges of the factor graph between the variable and the check nodes. To reduce complexity of operations in LDPC decoding, loglikelihood ratio (LLR) messages are often used instead of probabilities [8]–[13].

B. Overview of stochastic decoding

Stochastic decoding [14]–[21] has been proposed for very low-complexity decoder implementations. Stochastic decoders were first introduced in [14] and have been demonstrated to lead to simple, yet very high throughput hardware [17], [18]. In stochastic decoding, a probability p is represented by a random sequence of bits that is a Bernoulli sequence. The frequency of ones in the sequence corresponds to p. For example, a probability 0.5 of a message can be represented by the stochastic stream 0100111010 which has 5 ones within 10 bits. The message representation is not unique. The message is represented by any sequence with statistically half of its bits being ‘1’.

Decoding is performed iteratively. First, channel values that are represented by a probability are received in variable nodes and converted into stochastic streams, which are set to variable-node outputs. Second, these streams are sent to paritycheck nodes via the interleaver connections. Third, paritycheck node outputs are calculated and sent back to the variable nodes, which update their outputs based on the updated inputs and channel values. This procedure, including the second and third operations, is repeated for a fixed number of iterations, or until a satisfactory solution is obtained.
C. Asynchronous scheduling algorithm
Stochastic decoders are usually implemented by using synchronous circuits, where all the variable-node outputs are updated simultaneously using a global clock signal. However, the wiring delay via an interleaver tends to have variations from wire to wire due to the complexity in the chip implementation causing very long interconnects, which limits the clock frequency. We have previously proposed an asynchronous scheduling algorithm [12], [13] to alleviate this problem for min-sum decoding.
In this work, we introduce an asynchronous scheduling algorithm for stochastic decoding to tackle the wiring complexity, especially for clock distribution. In that scheduling algorithm, since node computations are performed by using the most recent available messages rather than all updated messages at a global level that are required for stochastic decoding, nodes do not need to wait for the all updated messages including from tardy long wires, but can proceed when ready. The transmission delays between the computation nodes are determined by each wire delay, and thus the decoding convergence and hence

The asynchronous scheduling algorithm can be implemented by using previously reported multiple-valued currentmode [23] or binary asynchronous circuits [24], [25]. In the asynchronous circuits, local control signals are used instead of a global clock signal. Fig. 3 shows asynchronous data transmission between one variable node and one check node. Data (an stochastic stream) is transmitted by using local control signals, such as a request and an acknowledge signal, labeled as req and ack, respectively. First, the request signal, req, is asserted to transmit data to the check node. Second, the check node detects the request signal and then receives the data. Third, the check node transmits the acknowledge signal, ack, to the variable node to inform that the data is received. Finally, the variable node detects the ack signal and then goes back to the first step to transmit the next data. In this way, since the asynchronous circuits are controlled by a request-acknowledge-based handshaking protocol, the transmission delay via the interleaver is determined by local wiring delay characteristics.
Our goal is to implement a high-throughput stochastic LDPC decoder based on asynchronous scheduling. A BER simulation is required before decoder implementation to validate decoding and scheduling algorithms. In a synchronous stochastic decoder, all computation blocks operate at the same time so a BER simulation is straightforward to implement. However, in the asynchronous stochastic decoder, computation blocks and data-transmission blocks operate at different speeds based on each individual delay. To validate the asynchronous algorithm, we need a complex timing model that considers all delay information. In the next section, we describe out timing model.

IV. EVALUATION
We have developed a simulator based on the timing model of Section III for the asynchronous stochastic decoder. A regular (1024, 512) LDPC code is used. Assume that the variablenode computation delays (TX1, ..., TXn) are the same and are represented by TXi. Also, assume that the check-node computation delays (TY 1, ..., TY (N−k)) are the same and are represented by TY i. The data-transmission delays TXY and TY X have variations because of the range of wire lengths in LDPC decoders. Assume that TXY and TY X have the same variations and there is no handshaking overhead of asynchronous controls. To make the simulator credible, we use precise delay information from the synthesized result of a variable node with a 48-bit EM and a check node, which are synthesized by Synopsys Design Compiler using ASPLA 90nm CMOS standard cell library. TXi and TY i are 280 and 100 ps at the minimum, respectively as shown in Table I. TXY are set by using the data-transmission delays in the asynchronous bit-serial (1024, 512) LDPC decoder [12] that is implemented in ASPLA 90nm CMOS technology. The LDPC decoder uses the multiple-valued duplex asynchronous data-transfer circuit [23] for interleaving and eliminates the handshaking overhead. The data-transmission delays range from 300 to 2400 ps.
A. Decoder convergence
We observe the decoding dynamics of the proposed decoding by varying the VAR computation delay TXi from 100 to 1000ps shown in Fig. 5. TY i is 100 ps and TXY ranges from 300 to 2400 ps. Decoding terminates after a maximum of 5000 ns with a syndrome checker as an early stopping method. When TXi is smaller than 300 ps, the decoding does not converge and reaches the maximum latency. In this case, TXi is smaller than most values of TXY . The variable-node computation is often repeated more than once by using the same input value. An EM is also updated more than once using the same input value, which may incorrectly change the stored value. In contrast, when TXi is larger than 300 ps, the decoding converges. The minimum latency is 118 ns when TXi is 400 ps at Eb/N0 = 3dB.

D. Discussion
We have validated the decoding dynamics of the proposed decoding and evaluated the throughput and the BER performance in simulation. The proposed asynchronous stochastic decoding methodology has the potential for low-power and high-throughput stochastic LDPC decoding. The advantages of asynchronous decoding are very important at long LDPC block lengths because for those codes clocking power can be large. Also there will be an improvement in the wiring complexity for long codes, which leads to an increased advantage of the throughput in the proposed decoding technique.
In terms of the hardware implementation, asynchronous circuit techniques based on binary logic [24], [25] or multiplevalued logic [23] can be used. The asynchronous stochastic decoder is controlled by request-and-acknowledgment based handshaking instead of a clock. The handshaking signal is generated using asynchronous controllers, which can be large area overhead because the stochastic computation block is very small.
V. CONCLUSION
We have proposed an asynchronous stochastic decoding technique for parallel LDPC decoding. We model asynchronous stochastic decoding in a BER simulation and evaluate its dynamics, where the synthesis results under a 90nm CMOS technology are used for the timing model. The proposed decoding scheme achieves 2.52x and 7.37x increase in decoding throughput with comparable BER performance with respect to the synchronous and synchronous pipelined stochastic decoders, respectively. Based on these results, we can proceed towards a hardware proof of concept.
