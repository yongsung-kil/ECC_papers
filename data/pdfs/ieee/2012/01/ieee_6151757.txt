Analog Digital Belief Propagation
Guido Montorsi, Senior Member, IEEE

Abstract—We introduce a message passing belief propagation (BP) algorithm for factor graph over linear models that uses messages in the form of Gaussian-like distributions. With respect to the regular Gaussian BP, the proposed algorithm adds two operations to the model, namely the wrapping and the discretization of variables. This addition requires the derivation of proper modifications of message representations and updating rules at the BP nodes. We named the new algorithm AnalogDigital-Belief-Propagation (ADBP). The ADBP allows to construct iterative decoders for mod-M ring encoders that have a complexity independent from the size M of the alphabets, thus yielding efficient decoders for very high spectral efficiencies.
Index Terms—Belief propagation, APP estimation, iterative decoding, non binary LDPC, Analog Digital Belief Propagation.
I. INTRODUCTION AND MOTIVATION
I N modern telecommunication systems the need for high throughput with a limited bandwidth has pushed the interest toward coding schemes with very high spectral efficiencies. The common solution is to couple a powerful binary encoder with a standard high cardinality modulation, such as the QAM modulation. At the receiver, a suitable iterative decoder for the binary code is paired with the detector of the modulation, which provides Log-Likelihood ratios (LLR) of the coded bits. Iterations between detector and binary decoder may be avoided when Gray mapping is used for the modulation scheme, yielding a receiver scheme which is called “pragmatic”.
An attractive alternative for high spectral efficiencies is that of working directly with encoders processing high cardinality alphabets. Going from the binary alphabet to alphabets with cardinality M > 2 yields an improvement of performance but also increases the complexity and memory requirements as messages are represented in form of vectors with size M . Operations at BP check nodes for LDPC are consequently more complex ( [1]–[4]). Improvements have been obtained for the iterative decoding of non-binary LDPC codes reducing the complexity from O(M 2) of a straightforward implementation to O(M log M ) obtained with FFT processing at check nodes, to O(nm log nm), with nm < M of the Extended Min-Sum algorithm in [5].
Consider the problem of designing a bandwidth efficient transmission system making use of a linear encoder and a high-order M -PAM constellation. A straightforward extension of a linear (N, K) binary encoder is obtained by substituting the binary quantities at the input of a binary encoder with M ary symbols. Parity-check symbols are obtained by performing modM sums instead of mod 2 sums in the encoding procedure. These codes are a subclass of codes over rings, like those

At the receiver the use of the regular BP iterative decoding algorithm requires to compute the LLRs of the transmitted symbols in the form of M -ary vectors. The computation of LLR has complexity and memory requirements that grow linearly with M and thus exponentially with the spectral efficiency. Furthermore, also all known instances of the BP algorithm for non binary symbols have a complexity that grows at least linearly with M . This fact makes the use of this encoding system not common in practical applications. A receiver system using the proposed ADBP instead takes as input messages a pair of parameters identifying a member of a predefined class of likelihoods and performs the BP updating by constraining the messages in the graph to stay in the same class. The resulting receiver have then a complexity independent of M , thus allowing to achieve reliable communications also with very large M .

II. GAUSSIAN BP OVER LINEAR SYSTEMS

An important example of application of BP for APP estimation is when the underlying model is a linear system, i.e. where the variables are obtained as linear combination of other variables. For linear systems the underlying factor graph model only involves three types of nodes
1) Sum: The real sum of variables to the node is zero, 2) Repetition: Two or more variables are equal, 3) Scaling: The output is a scaled version of the input.
In this case, likelihoods Li are functions of the continuous alphabet of the variables and the message updating rules correspond respectively to convolution, multiplication, and axis scaling of likelihoods. Being the messages continuous functions these operations have an infinite complexity. However, when the input messages are Gaussian, the output messages from a node belong to the same class of input messages. VI. SIMULATION RESULTS OVER AWGN CHANNEL
ADBP gives performance very close to those that can be obtained with the exact BP, based on vector messages. In
Figure 4 we report the results of a set of simulations, obtained with a (5000, 3000) random regular mod-M ring LDPC code paired with an M -PAM modulation and transmitted over a wrapped AWGN channel, for M =10 and M =32.
The three curves show the performance of the iterative decoding algorithm (10 iterations) using the exact receiver, the ADBP-based receiver and an ABP only based receiver. ADBP has a performance loss of only about 0.3 dB w.r.t. the exact BP, while the ABP performance does not show any beneficial threshold effect due to coding. In fact, the threshold effect typical of powerful digital coding happens when the standard deviation of input messages becomes close to the sampling interval and approximation (10) is used in ADBP instead of (1). In Figure 4 we have also reported the performance of a system using the same code and decoder but with a modulation alphabet with cardinality 32: the curve shifts by an amount related to the capacity loss (the same effect would be observed with much higher cardinalities), showing that ADBP can be used to build a flexible transmission system with fixed complexity capable to achieve reliable transmission with unbounded spectral efficiencies.
