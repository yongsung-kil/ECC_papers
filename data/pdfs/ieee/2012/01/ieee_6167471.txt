Robust Decoder Architecture for Multi-level Flash Memory Storage Channels
Ravi Motwani and Chong Ong, Intel Corporation, NAND Storage Group, ravi.h.motwani@intel.com, chong.ong@intel.com

Abstract
Multi-level-cell (MLC) flash memory comprises of cells which can be programmed to multiple levels. Recent MLC flash memory systems support 3 bits per cell to 4 bits per cell which means that the individual cells are programmed to 8 or 16 distinct levels respectively. MLC flash has higher raw bit error rate (rber) than the single bit per cell flash. This calls for the use of sophisticated error control coding (ECC) schemes like LDPC codes.
The flash memory channel is usually modeled with level distributions having Gaussian probability density function. However, the Gaussian distribution is not a good fit for practical NAND channels. Even for the beginning of life of the flash memory device, this model has to be replaced by a more realistic model. With erases and re-writes, the channel towards the end of life of the device is far from Gaussian and floating-gate to floating-gate coupling and charge loss not only increases the raw bit error rate but causes the level distributions to become asymmetric.
Given such channel impairments, if the LDPC decoder assumes Gaussian level distributions, its performance can degrade considerably. Hence, modifications are required in the decoding algorithm to cope up with the channel distortions. In order to keep the hardware costs within limits, simple schemes with minimal hardware increase are desirable to keep up the performance. In this paper, the flash channel degradations are first enlisted and then simple solutions are proposed which keep the performance in check as the flash memory transits from a channel with moderate impairments to the end of life condition, where the level distributions for the different levels are highly asymmetric.

The bit error rate for an MLC-flash is worse as compared to the single bit per cell flash and the requirements on the output bit error rate are very stringent. Hence, for MLCflash, use of advanced error correction schemes like LDPC codes becomes inevitable.
LDPC decoders need soft information input in order to realize their full correction capability. The common LDPC decoding algorithm like the min-sum decoder or the 2-D normalization minsum decoder perform very well for the AWGN channel model [4]. As mentioned, the AWGN channel model is a misfit for the flash channel. Further, the flash channel degrades considerably over the life of the device, and if an error control coding scheme like LDPC code is used, the LDPC decoder is expected to adapt to the varying channel. Due to the time varying and non-Gaussian nature of the channel, appropriate changes to the decoder architecture are required. The architectural design challenge is to have a decoder which keeps up the performance throughout the life of the device with minimal increase in hardware.

1.1 Floating-Gate to Floating-Gate Coupling
Flash memory comprises of grid of cells arranged in a rectangular lattice. A cell is a floating-gate and the information is stored as charge in these floating-gates. A write consists of programming the cell by Fowler-Nordhiem tunneling till it is programmed to the ideal voltage value. However, due to programming time constraints, some tolerance is accepted and the actual programming voltage is allowed to be within some range of the ideal. Due to the capacitive coupling between neighboring cells, the threshold voltage of the cell changes depending on the amount of coupling with the neighbors and their charge levels. This phenomenon called as inter-cell-interference (ICI) can be from mild to extreme. Particularly, the cell at the lowest voltage level will have its threshold voltage substantially increased if there is coupling with neighbors which have high charge.
Depending on the programming method, the floating-gate coupling causes the level-0 sandwiched between two level-7 patterns ( the adjacent level-7 - level-0 - level-7 pattern) to have a increase in the threshold voltage if the neighbors are programmed after it. Thus program disturb which is a consequence of FGFG coupling can cause the level-7 - level-0 - level-7 pattern (abbreviated henceforth as 7-0-7) to migrate to a 7-1-7 or 7-2-7 pattern, if the two level-7’s are programmed after the level-0 is programmed.
In order to combat ICI, constrained coding is a possible solution [5, 6, 9]. Constrained coding entails forbidding certain level patterns so that effects of ICI are contained. For the example above, a constrained code can be employed which either elim-

inates or substantially reduces the 7-0-7 patterns from the level information to be programmed into the NAND.
Constrained coding requires spare bits, so it adds to the redundancy and decoding constrained codes causes error propagation which has to be combated by either using more sophisticated decoders which can add to the complexity and hardware or by adding more redundancy. Since constrained coding may not be a viable solution for applications which cannot afford extra overhead bits or encoding/decoding complexity, alternative solutions have to be explored. In the next section we propose pragmatic methods to address the mitigation of program disturb effects.
1.2 Single-bit charge loss
SBCL means charge loss of the high voltage level patterns like level-7 and level-6 which drift to the level-6 and level-5 patterns respectively. SBCL causes the distribution of the level-7 pattern to have a much higher below edge variance than expected. Hence, program disturb and SBCL both cause the tails of the distributions for level-0 and level-7 to flatten. Since program disturb and SBCL are mirror problems, the solution for both the problems is also the same.
1.3 Long Tails
Towards the end of life of the NAND, the levels have a distribution with a much higher variance than the beginning of life of the memory. Also the below edge variance of the distribution and the above edge variance values are different. Strictly speaking, the distribution is not Gaussian, a split-Gaussian is a better approximation. Also, each level has a different value of below edge variance and above edge variance. This asymmetry of the distributions of the different levels is called as long tail effect since it causes the tails of the cumulative distribution function (cdf) of the different levels to flatten.
1.4 Misplacements
While programming the MLC 3 bits per cell NAND, the sequence is to first program the lower page followed by the middle page and then the upper page. Misplacements occur when the read of the lower page to program the middle page has errors. This causes the NAND to program the cell to a level different from the actual intended level. Subsequently, while programming the upper page, these errors carry forward.
Further, errors during read of the middle page while programming the upper page will cause the NAND to program the cell to an erroneous level. The misplacement errors occur while programming and only the lower and middle page can suffer from them, hence misplacement errors remain localized to the read pages wherein they occur during programming.

2 LDPC Decoder Modification for Mitigation of Channel Impairments

To combat FGFG coupling, constrained codes have been proposed [5, 6, 9]. Constrained codes add to the redundancy and in applications which cannot afford extra spare area and encoding/decoding hardware, alternate solutions are to be explored.
Effectively, program disturb and SBCL lead to level-0 distribution to have a higher above edge variance and level-7 (and maybe level-6) to have a higher below edge variance. Hence, program disturb and SBCL can also be categorized as long tail effects and a method to combat long tails can be used to combat all these impairments. Hence, we are left with two problems to address- one is to combat long tails and the second to combat misplacements.

To combat program disturb, the LDPC decoder can be modified such that it takes into account the fact that a 7-1-7 pattern is suspect and with a certain probability is a 7-0-7 pattern transited to a 7-1-7 pattern due to program disturb.
Since the soft information is quantized to 7 bits, a read from the NAND which gives soft information can be made adaptive. Whenever a level-1 is read while reading the upper page, we check if this level-1 has neighbors which are both level-7’s. If they are, then the magnitude of the soft information is clipped down to account for its being a potential level-0 mapped to a level-1 due to program disturb.
Other options have also been tried, like flagging that bit 0 from level-1 with an erasure, if the neighbors are level-7’s. However, the best results are obtained by clipping the magnitude of the LLR, since erasures leads to substantial loss of information.
The mitigation of long tails by modification of the LLR values can be explained intuitively. Since the high confidence 0 bit can be a consequence of a 1 mapping to a 0 due to program disturb, the high reliability for the 0 bit has to be clipped to avoid the LDPC decoder from being mislead.
The same LLR modification method can also be used to combat SBCL. Since a level-6 read can be a level-7 mapped to a level6 due to SBCL, the bit 0 read due to level-6 trigger for the upper page must have its reliability clipped if its neighbors are low voltage cells. As mentioned earlier, the problem of program disturb and SBCL is actually the same problem, one occurring for erase level cells and the other for the highest voltage level cells. It is not surprising that the solution is also the same, since for both the cases a bit 0 is not reliable and hence its LLR magnitude has to be attenuated if its neighbors are either high voltage levels or low voltage levels respectively [7, 8].
Long tails can also be handled by the similar LLR modifications. Since long tails cause the above edge variance and the below edge variance of the distributions to have different values, proper LLR modification as derived here compensates for long tails. Towards end of life, a different LLR value for each level information is required to keep up the performance of the LDPC decoder.
2.2 Combating Misplacements
For MLC NAND flash, the lower page is first programmed. Subsequently, the middle page is programmed and to program the middle page the lower page information has to be read. While reading the lower page information, errors may occur and a 1 can be read as a 0 and vice versa. If this error occurs, then the final programmed voltage is in error. This impairment, termed as misplacement can be compensated for during the LDPC decoding. Misplacements can be single sided or double sided. Say, only two levels are programmed. In single sided misplacements, level1 is always correctly programmed, but level-0 can be erroneously programmed to level-1. This is called as single sided misplacement. 4 Conclusions

NAND flash memory suffers from various channel impairments. The FGFG coupling causes program disturb and SBCL causes charge loss of the high voltage cells. Misplacements cause the final programmed cells to be programmed to an incorrect level and the end of life condition level distributions look asymmetric, since the below edge variance of the distribution has a different value from the above edge variance.
