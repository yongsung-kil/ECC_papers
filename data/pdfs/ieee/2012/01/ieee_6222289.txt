Soft Information Single Error Correction for Interactive Concatenated Codes
John J. Metzner, Life Fellow, IEEE

Abstract—This paper describes a concatenated code method where the outer decoder hypothesizes possible block symbol values for the inner code to make a likelihood retest from its original soft information. These hypotheses come from low density check sets where the probability is significant that there is only one error in the check set, whose value would be the syndrome for that check. The method has an advantage over pure verifications in converting errors into erasures. Also, it is shown that, if the inner decoder can supply a list of two decisions for each symbol, the decoding method, for large first choice symbol error probability p, allows reliable communication at rates substantially exceeding the (1 − p) log2 Q QSC per symbol channel capacity with b-bit symbols.

I. INTRODUCTION
I T has become common to consider codes where the symbols are vectors with components over GF (2). Sending XOR combinations of such vectors has been a part of reliable multicasting [1]–[4] and linear network coding concepts [5]. In addition, the structure has been applied in vector symbol decoding [6] and majority logic-like decoding [7], [8], decoding with symbol lists [8], [9], packet symbol decoding with low density codes [10]–[12], convolutional code decoding [13], burst error correction [14] and decoding without sequence numbers despite deletion and out of order reception [11], [15].
Vector symbol decoding uses the idea of verification of the correctness of a subset of the received symbols [8], [10]. In [10], [11], [15], it is shown that vector/packet symbol decoding is especially attractive with low density codes.
An erasure channel is an idealized channel. Erasure decoding has been shown [16] to be rather simple with low density codes, and the capability to decode reliably and simply at rates that approach erasure channel capacity with code length has been demonstrated. The erasure model is not realistic for noisy channels, unless each individual symbol has enough error detection to make its validity almost certain. But this is an excessive redundancy if a symbol is, say, only about 50 bits.
In the past, it has been considered that twice as many erasures could be corrected as errors, based on minimum Hamming distance. But for nonbinary symbols with random,

The technique to be described, called soft information single error correction (SISEC), is for a concatenated code where the inner code makes a decision for a b-bit vector symbol, but retains soft information about that symbol. The outer code may hypothesize a block symbol value for the inner code to re-test based on its original soft information. These hypotheses come from low density check sets, where the probability of only one error in the check set is significant. It will be shown that the amount of error correction with verification and SISEC is greater than verification without SISEC. This can bring great improvements for short codes or sub-codes of a long code. The gains with SISEC in sub-codes extend to strong improvements in long codes in a way related to the advantages of recursive decoding methods [17] with low density codes.
Section II describes the structure of the concatenated code and vector symbol/verification decoding. Section III describes the SISEC steps. Section IV shows the relation between decoding on an erasure channel and decoding on an error channel using SISEC and verification. Section V describes the incorporation of short lists for each inner symbol decision. Section VI analyzes performance using both SISEC and lists. Section VII considers combinations of both errors and erasures. Section VIII presents some theorems showing performance of Majority-Logic-Like decoding can be improved through SISEC. Section IX discusses possible future extensions of the work.

The main principle in vector/packet symbol verification decoding [6]–[10] is to verify symbols in a check equation that adds to zero. It has been recommended [6] that each Ti be a different pseudo-random invertible transform. This reduces the probability that a set of error symbols in a check set adds to zero. The event that a check set adds to zero is critical because it is used as a verification that all symbols in the set are correct. For some channel types this step may not be necessary. Also, for SISEC the invertible pseudo random transform is important because it transforms a false outer code hypothesis into a random pattern.

After this stage, it will be found, for a certain HA, that most errors are corrected and verified, and among the unverified almost all are true errors. A. Steps in the SISEC Process for the HA Sub-codes

to have no errors initially, verifying the included symbols, or only one unverified symbol allowing immediate correction.

Step 1) For each sub-code, find any zero syndromes among a set of relevant syndromes. (In the main (12, 6) sub-code example, all 7 relevant check equations used have ≤ 4 symbols.) Mark as verified all symbols in any zero check equation. If all but one entry of a nonzero check set remains unverified, solve for the unverified symbol. If all symbols are verified, go to Step 4. Otherwise go to Step 2 .
Step 2) For an equation with two unverified symbols, test by SISEC one of the two by subtracting the equation syndrome and sending its transform to the inner decoder for a test. If the first test fails, test the other. Success corrects the error and creates two new verifications. If all symbols are verified go to Step 4. If success but not all symbols are verified, return to Step 1. If the equation with two unverified symbols fails on test, but there is an equation with two unverified not yet tested, repeat Step 2 with that equation. If all equations with two unverified symbols fail, go to Step 3.
Step 3) If SISEC fails to decode the sub-code completely in the first 2 steps, attempt to test equations with 3 unverified symbols, then successively greater up to the maximum in the relevant check equations. After any successful test return to Step 1. After a failed test, continue Step 3 with the next equation in the set. If all fail, no further corrections are possible. Go to Step 4.
Step 4) Decoding is complete where either all symbols are verified or no new verifications can be found.
The SISEC outer symbol hypothesis test gets randomized by an inverse transform back to the inner decoder. Any false hypothesis looks like a random set of bits. But a correct hypothesis almost surely will have a high score. Require a probability less than ε1 that a correct hypothesis of SISEC is not properly confirmed, and a probability less than ε2 that a false hypothesis gets accepted. Then the hypothesized symbol value should lie in a “sphere” around the received signal point covering 2αb vectors, where α is large enough so that a correct hypothesis is not in the sphere with probability < ε1, and small enough that the probability 2−(1−α)b a random false hypothesis falls in the sphere is < ε2. If this cannot be done, declare the symbol an erasure. In terms of sphere packing, for large b, the residual soft information about the symbol should be at least (1 − α)b bits.

Suppose instead that the x positions were errors instead of erasures. The last n − k − x rows contain no errors, and act as verifications, if found. The condition that all the n − x last positions can be verified is that no column of N be all zeroes. This is the condition that no column position c in the last n−x is dependent on the first x columns. It corresponds also to the condition that the errors do not come within one of covering a code word of the basic binary code. This is because position c and a set of columns among the x add to zero, meaning there is a code word whose positions are covered by errors except for position c (one from covering a code word).
Thus with verification all correct positions can be verified except for columns that are in the column space of Hx. If there are i such positions, then x + i positions remain unverified.

VIII. APPLICATION OF SISEC TO MAJORITY-LOGIC-LIKE CODES
Majority logic codes are binary cyclic codes that have the property that one symbol can be tested at a time with J check equations with the structure that the test symbol appears in all J equations and no other position appears in more than one equation. Usually the Hamming minimum distance is J + 1, which we will assume to be the case. For binary Majority Logic codes the correct test bit value is decided by majority rule. Each test symbol is changed or not, and all symbols are tested.
Majority logic-like (MLL) codes [7], [8] use the binary majority logic code structure, but instead the individual symbols are b-bit vectors, just as with vector symbol codes. With MLL codes, when a symbol is tested and at least two of the J syndromes show the same vector value and no syndrome is zero, that duplicate symbol value is judged to be the error and is so changed. If one or more of the J syndromes is zero, all symbols in the related equation(s) can be considered verified as correct.

IX. EXTENSIONS AND FUTURE WORK
Most work on interactive decoding centers around the ideas of turbo codes [24]. These methods combine likelihoods at the bit level. These bit likelihoods are modified through interaction between two encoder/decoders in parallel concatenation or between the inner and outer encoder/decoders in serial concatenation. Often a large number of iteration stages are required for good results.
SISEC could be used with more interactions in two ways. One way is through interaction between the HB code and the HA sub-codes. When the HB code corrects an error because all but one of the symbols in a check equation is unverified, that correction will result in many cases in correction of all 4 errors in a HA subset, since residual error sets of 4 are most common. These additional corrections in the sub-codes can lead to further corrections with other HB check equations, similar to the tornado effect common to low density codes.
The other way is for the inner code to have dependent symbols. Suppose the inner code is structured as a convolutional code terminated only after kb data bits, 2 ≤ k ≤ n, corresponding to k symbols. Due to this dependence, a SISEC correction of a b-bit block can lead to a correction of a following or preceding b-bit symbol error. Suppose that Viterbi decoding is used with a memory of m data bits. The transformed data from the hypothesis defines the state directly for the last b − m bits, and the first m bits define a trace back through the trellis to the initial state for the b-bit block. This unique b-bit path can have its likelihood computed from the soft information known to the inner code. This likelihood confirms or denies the hypothesis. If the hypothesis is confirmed, and the beginning state or the end state is different from the adjacent decision, it means that the respective adjacent decision almost surely was wrong. New end states may then be traced forward, or beginning states backward. This is likely to result in correction of adjacent symbol errors. Any changes in decisions on neighbor blocks can be reported to the outer decoder. This in turn can create a 2 to 4 multiplying correction effect at the outer decoder. The symbols in a sub-code could be interlaced in position to spread adjacent errors. Also, any verification corrections that occur in the outer code can be fed back to the inner code decoder.
These interactions should allow reliable rates that not only exceed substantially the QSC capacity (1), but perhaps allow approach to the larger value (10) and perhaps rival the performance of turbo codes. The capabilities of these interactions
