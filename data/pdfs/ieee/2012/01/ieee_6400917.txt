Abstract—Simple implementations of decoders for LDPC codes
are important practical problems when the decoders have the
severe energy consumption constraints or the computation time
constraints. A 3-bit decoding algorithm, proposed by Planjery et
al. [1], can be regarded as a simplified version of the sum-product
algorithm [3]. The algorithm has a very good performance of
frame error rate at the error flower region when the algorithm is
used for the BSC channels. Although the 3-bit decoding algorithm
is not so good for the AWGN channels, it can be shown that the
performance of the 3-bit decoding algorithm can be improved by
introducing bias terms. It will also be shown that the computation
time for the sum-product algorithm can be decreased by applying
the 3-bit decoding algorithm to outputs from the sum-product
algorithm.
I. INTRODUCTION
Low density parity check(LDPC) codes have sparse parity
check matrices and their performances are very good when
iterative decoding algorithms, e.g., the sum-product algorithm,
are employed.
There are simple decoding algorithms for LDPC codes,
called bit flipping algorithms. In the algorithms bits satisfying
some criteria are flipped at each iteration step. There are many
literature on bit flipping algorithms. In general the performance
of the bit flipping algorithm is improved by introducing
additional bits or operations, or complicate criteria for flipping.
Therefore there is a trade-off between the complexity (the
computation time or the size of circuits for the algorithm)
and the performance.
In this paper we are interested in very simple but good
bit-flipping algorithms where by ’simple’ we mean that the
decoding algorithm does not use any floating point arithmetic
operations and by ’good’ we mean that the performance is
better than the original bit flipping algorithm.
If we would implement a circuit for an error correcting
code in a very small device used under very bad conditions
like inside of an automobile or a human body, we need a
coding scheme which should be powerful enough but can also
be implemented by a small circuit. Therefore the trade-off
between the complexity and the performance is an important
issue.
This paper is organized as follows. First several algorithms
based on the idea of the BF algorithm are described. Then it is
shown that the performance of the 3-bit decoding algorithm is
improved by introducing bias terms. It also be shown that the
3-bit decoding algorithm can be used to improve the overall
performance of a coding system using the SP algorithm.

III. SIMPLE DECODING ALGORITHMS FOR LDPC CODES
The bit-flipping (BF) algorithm is the simplest decoding
algorithm for LDPC codes. One of important properties of
the algorithm is that the algorithm does not use any floating
point operation unlike the sum-product algorithm. Therefore
the BF algorithm can be implemented with a small circuit and
operated with small energy cost.
A BF decoding algorithm with bias terms (Quantized and
Modified BF, QMBF) is a variant of the BF algorithm. The
algorithm uses integer valued bias terms determined from
the received signal. A 3-bit decoding algorithm also uses no
floating point operation. We use these three algorithms in this
paper.
A weighted bit flipping (WBF) is yet another variant of the
BF algorithm but uses floating point values during iteration
steps[9]. A modified WBF is a variant of the WBF [11].

D. Simulation Results
Parity check matrices for the simulation is obtained from the
MacKay’s database [7]. In the following ‘LDPC(N)’ means
an LDPC code given by a parity check matrix of length 𝑁
and column sum 3 in the database.
The WBF algorithm flips only one bit at each iteration round
for almost all cases. This means that if the maximum iteration
number, 𝑙𝑚𝑎𝑥, is small then the WBF would not exhibit a
good performance. Therefore 𝑙𝑚𝑎𝑥of the WBF algorithm is
set to 100 but rather small values are used as 𝑙𝑚𝑎𝑥for other
algorithms.
The 𝜌in the MWBF is 0.5 and the 𝛼in the QMBF
is 0.5. The 𝛾in the 3BitBI, 3BitBR and 3BitBC is 0.3.
These constants are chosen so that good performances can
be obtained.
In Fig. 1 and Fig. 2, BER performances of algorithms
described in the previous section are given. In these figures the
performances of 3-bit decoding algorithm and its variants are
relatively poor at the low SNR region. But the performances
of them are good at the high SNR region. The performances of
the 3-bit and the 3BitBI decoding algorithms decrease rather
rapidly as the SNR gets large.
V. SECOND PROPOSED ALGORITHM
A. Performance of 3-bit decoding algorithm
First we show the performance of the 3-bit decoding al-
gorithm when the algorithm is applied to AWGN channels,
that is, an output from the AWGN channel is input to a hard
decision decoder and the output from the decoder is input to
the 3-bit decoding algorithm. A code used in the following
simulations is an LDPC(1008) and 𝑙𝑚𝑎𝑥is 45.
The BER and FER performance of the 3-bit decoding
algorithm is given in Fig. 3. The difference of the two
performances is not so large. We have checked the distribution
of positions where errors occurred in code words by displaying
on a monitor of a computer. It was found that (1) if the output
codeword is wrong, half of bits in the code word are erroneous,
(2) there may be no error pattern with small weight.
The bit error (BER performance) of the BF and the QMBF
algorithm is given in Fig. 4. There are significant differences
between the BER performance and the FER performance for
these algorithms.
When we watched the positions where errors occurred in
code words, the positions seemed distribute randomly.
These observations lead us to our second coding scheme in
the next section.

VII. CONCLUSION
We have proposed two coding schemes using the 3-bit
decoding algorithm. It has been shown that the performance of
the 3-bit decoding algorithm can be improved by introducing
bias terms. It has been also shown that the coding scheme con-
sisting of the SP algorithm and the 3-bit decoding algorithm
is better than the SP algorithm with respect to the number of
iterations.
