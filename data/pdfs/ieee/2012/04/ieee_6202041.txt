Efficient Encoding of Quasi-Cyclic LDPC Codes with Continuously Variable Length
Lu Liu1, Lei Liu2
(Affiliation 1): Beijing University of Posts and Telecommunications, Beijing, P. R. China, 100876 (Affiliation 2): University of Science and Technology of China, Hefei, P. R. China, 230027

Abstract—In this paper, we present a class of QC-LDPC codes with continuously variable length (CVL), which can be encoded in linear time. The method combines CVL codes [1] with approximate lower triangle structure. Thus the paritycheck matrix can achieve girth of 10, and the block length can vary continuously. Significantly, only cyclic-shifting and XOR operations are needed to realize the linear time encoding.
Index Terms —QC-LDPC codes, Efficient encoding, Continuously variable length, Girth
I. INTRODUCTION
The design of LDPC codes with high performance and low complexity is a hot topic in recent research of channel coding, among of which the quasi-cyclic (QC) LDPC code draws attention due to its linear time encodability and small size of storage. The parity-check matrix (H matrix) of QC-LDPC codes consists of quasi-cyclic matrices Paij (i=1,2,. . . ,m;j=1,2,. . . ,n), with each of them constructed by shifting the identity matrix to the right by aij times, where aij is the shift offset value defined in the ring RL. Hence, it is desirable that the block length and code rate can be fine adjusted for QC-LDPC codes [1].
In this paper, we propose a class of QC-LDPC codes with continuously variable length (CVL), and the complexity of encoding is linear with the block length. The H matrix can achieve girth of 10, and the memories needed for encoding is limited due to the quasi-cyclic structure.

V. CONCLUSION
In this paper, we design a class of QC-LDPC codes for efficient encoding base on the CVL method, which have continuously variable length and can be encoded in linear time. We construct a code with code rate 1/2 to verify this method, and the encoding structure of this code for hardware realization is proposed. The estimation for the resource consumption shows this structure requires limited memories and few calculations. At last, the simulation shows that the codes have good performances, and the higher the code rate is, the better it performs. Consequently, the encoding complexity is significantly low, while the performances are good, further more the block length can vary continuously so as to apply much more available codes.
