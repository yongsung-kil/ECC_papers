A Density Evolution Based Framework for Dirty Paper Code Design Using TCQ and Multilevel LDPC Codes

Yang Yang†, Zixiang Xiong†, Wu Yu-chun‡, and Philipp Zhang‡ †Dept of ECE, Texas A&M University; ‡Huawei Hisi Advance Research Department, China
Email: yangyang@tamu.edu, zx@ece.tamu.edu, wuyuchun@huawei.com, and pzhang@huawei.com

Abstract—We propose a density evolution based dirty-paper code design framework that combines trellis coded quantization with multi-level low-density parity-check (LDPC) codes. Unlike existing design techniques based on Gaussian approximation and EXIT charts, the proposed framework tracks the empirically collected log-likelihood ratio (LLR) distributions at each iteration, and employs density evolution and differential evolution algorithms to design each LDPC component code. In order for the approximated LLR distributions obtained using density evolution to better match the true LLR distributions, a novel decorrelator is added to the decoder to make channel LLRs and check node LLRs almost independent of each other. Simulation results show that at 1 bit per sample transmission rate, the dirty-paper codes designed using the proposed method operate within 0.53 dB of the theoretical limit, reducing the best known result (with a 0.58 dB gap to the limit) by 0.05 dB at the same complexity and block length, while this improvement can be as large as 0.21 dB (corresponding to a 0.37 dB gap to the limit) when we use higher complexity encoder/decoder with longer block length.
I. INTRODUCTION
Channel coding with side information (CCSI) has recently gained much research interests due to its applications to data-hiding [1]–[3], precoding for interference channels [4], coding for MIMO broadcast channels [5], [6], and transmitter cooperation [7], [8] in ad hoc networks. However, although the capacity results obtained by Gelfand and Pinsker [9] and by Costa on the special case of the so-called problem of “writing on dirty paper” [10] have been known for almost 30 years, practical approaches to CCSI did not appear until 1999.
The first practical dirty-paper code was developed by Chen and Wornell under the name of quantization index modulation [11]. Other designs were proposed by Eggers et al. [12], Yu et al. [13], Chou et al. [14], and Sun et al. [15]. However, the designs above do not perform well at low rate where the dirtypaper code design is more challenging due to magnification of the granular loss of the source code [16]. Recent works by Erez and ten Brink [16] and Sun et al. [17] used trellis coded quantization (TCQ) [18] in conjunction with irregular repeataccumulate (IRA) codes [19], while Bennatan et al. [20] and Uppal et al. [21] proposed dirty-paper coding (DPC) schemes based on TCQ and LDPC codes. The best performance at 0.25 b/s transmission rate is reported in [17] to be within 0.63 dB of theoretical limit, while at one b/s transmission rate, the most powerful DPC code suffers SNR loss of only 0.58 dB [21].
In this paper, we propose a systematic framework for TCQ-LDPC based DPC code design by introducing two new

elements into the DPC decoder to further improve the performance. First, observing the correlation between the input and output log-likelihood ratios (LLRs) of the BCJR decoder, we employ a novel decorrelator before feeding the LLR values back to the variable nodes. Although the decorrelator itself does not directly offer performance improvement, it makes the density evolution algorithm (which assumes independence between the channel LLR and check node LLRs) more accurate and hence eases the design process. Second, instead of using a Gaussian assumption on the equivalent modulo lattice channel [16], we find the true marginal conditional probability density function (pdf) of the channel by simulation and feed it to the decoder. This way, the BCJR algorithm can output a more accurate LLR and hence a better decoding performance is expected. This advantage becomes more pronounced when the quantization error distribution is far from Gaussian.
To design each LDPC component code, density evolution is employed to better emulate the decoding behavior. With the decorrelator added to the decoder, the per-iteration LLR distributions for the resulting DPC codes fits the density evolution approximations more accurately compared to the commonly-used EXIT charts/Gaussian approximation based design method.
The DPC codes designed using our proposed framework perform only 0.53 dB away from the capacity limit at a transmission rate of one b/s, showing a 0.05 dB improvement over the best existing designs [21] at the same TCQ memory size of eight1, block length of 2×105, and bit error rate (BER) of less than 10−5. We also simulate our code design at higher complexity with memory-11 TCQ and a longer block length of 5 × 105, and obtain convergence within 0.37 dB of the theoretical limit. As far as we know, these are the best DPC performance reported in the literature at the same rate.
The rest of this paper is organized as follows. Section II describes the proposed DPC scheme. Section III states our general density evolution based design framework. Section IV contains simulation results and Section VI concludes the paper.

III. PROPOSED CODE DESIGN FRAMEWORK
In this section, we describe our code design framework in detail. Let the target transmission rate be RDP C b/s, and our goal is to minimize SNR s such that good DPC codes can be designed to achieve a certain BER no more than .
The first step is to design the encoder, that is, the mapping Mb1,...,bT . We follow the same design philosophy as in [17] and [21], i.e., maximize the sum of the approximated capacities (computed as the area under the EXIT curve [22]) of all T component channels for a given SNR value. Assume that the optimal Mb1,j,...,bT,j corresponds to T approximated capacities of C1, C2, ..., CT . Then pick T numbers Ri, i = 1, 2, ..., T such that Ri = Ci − Δi and R1 + R2 + ... + RT = RDP C with Δi being some relatively small number (e.g., 0.02-0.05 b/s) that corresponds to the expected rate loss.
Once the mapping Mb1,...,bT and Ci, i = 1, 2, ..., T are fixed, we start to design the LDPC code profiles in a sequential way, i.e., when designing the i-th LDPC component code, i = 1, 2, ..., T , all the other profiles are fixed. The detailed design process is illustrated in Fig. 6.

V. CONCLUSIONS
We have described a dirty-paper code design framework based on density evolution. A novel decorrelator was used in the DPC decoder to enable the density evolution algorithm to better emulate the decoding process, which does not guarantee independent message passing if the decorrelator is not employed. Simulation results show a 0.05 dB - 0.21 dB performance improvement over the existing DPC designs at one b/s transmission rate.
