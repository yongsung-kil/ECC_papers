Protograph-based LDPC Codes for Partial Response
Channels
Thuy Van Nguyen†, Aria Nosratinia† and Dariush Divsalar‡
†Department of Electrical Engineering, The University of Texas at Dallas, Richardson, TX 75083-0688 USA ‡Jet Propulsion Laboratory, California Institute of Technology, Pasadena, CA 91109-8099 USA Email: nvanthuy@utdallas.edu, aria@utdallas.edu, Dariush.Divsalar@jpl.nasa.gov

Abstract— This paper addresses the design of a protographbased LDPC code which can approach the independent and uniformly distributed (i.u.d.) capacity of partial response channels. We propose a method to calculate the iterative decoding threshold of a joint graph between a protograph and the state structure of a partial response channel using the extrinsic information transfer (EXIT) chart. We then describe a simple method to search for a protograph code whose threshold is close to the i.u.d. capacity limit. This new class of codes is needed because experiments show a protograph that is capacity approaching in the AWGN channel may not perform well in partial response channels. In particular, protographs with punctured nodes are often used to produce good AWGN codes, but they perform poorly with the BCJR equalizer. Numerical results support our analysis.
I. INTRODUCTION
The topic of designing a capacity approaching coding scheme for channels with memory, especially inter-symbolinterference (ISI) channels, has received much attention recently. The problem of determining the capacity of ISI channels (also called partial response channels) with finite-alphabet inputs still remains open, however, if the source is restricted to be independent and uniformly distributed (i.u.d.), the capacity of binary-input partial response channels is known as the i.u.d. capacity [1]. Irregular LDPC codes have been designed to approach the i.u.d. capacity of partial response channels [2], [3], but they generally lack a structure to enable easy encoding and fast decoding. To our best knowledge, the problem of designing a capacity approaching structured code for partial response channels has been open until now.
In this paper, we design a protograph-based LDPC code over binary-input partial response channels. A protograph code [4] is a structured LDPC code that can be constructed from a small protograph. Protograph codes over AWGN channels have demonstrated very good thresholds with low encoder complexity as well as fast decoding [5], [6]. In the context of coding for channels with memory, this paper for the first time presents a method to compute an iterative decoding threshold of a protograph-based code. We then provide a simple optimization method for the threshold to find a protograph that is i.u.d. capacity approaching.
We briefly review a subset of past work in this area. The EXIT chart is a simple yet effective tool to design and analyze the performance of LDPC codes in many communication channels [7]. In particular, Franceschini et al. [8] propose a design method based on the EXIT chart to optimize an

irregular LDPC code for coded modulation in ISI channels. However, the general EXIT chart cannot be used to compute an iterative decoding threshold of protograph-based LDPC codes [9]. A general LDPC code design for partial response channels was proposed by Kavcic et al. [1] via a modified density evolution [10]. Following the method of [1], several irregular LDPC codes optimized for certain specific partial response channels are reported including [2], [3], [11] and others.
The main contribution of this paper is a structured coding scheme based on protographs that is i.u.d. capacity approaching. We find out that a protograph code that is good for AWGN channels may not perform well on partial response channels, in particular protograph codes containing punctured nodes perform poorly with the Bahl-Cocke-Jelinek-Raviv (BCJR) equalizer. Our rate-1/2 protograph has a threshold that is 0.4 dB away from the i.u.d. capacity limit, and an implementation of this code with 16k data words exhibits a word error rate of 10−6 at a gap of 0.9 dB from the i.u.d. capacity limit.
The organization of this paper is as follows. Section II briefly explains the system model. In Section III, we introduce the concept of a protograph code and its iterative decoding threshold in an ISI channel. Section IV describes the protograph design process. Section V provides code construction procedures and numerical results. Finally, Section VI concludes the paper.

III. PROTOGRAPH THRESHOLDS IN ISI CHANNELS
A protograph [4] is a Tanner graph with a relatively small number of nodes. A protograph code (an equivalent LDPC code) is a larger derived graph constructed by applying a “copy-and-permutation” operation on a protograph. The protograph is copied N times, then a large LDPC code graph is obtained by permuting N variable-to-check pairs (edges), corresponding to the same edge type of the original protograph. A simple example of a protograph is shown in Figure 2. This graph consists of 3 variable nodes and 2 check nodes that are interconnected by 7 different edge (variable-to-check) types. A derived graph is constructed by replicating the protograph 3 times and permuting variable-to-check pairings within the same edge type of the protograph. In this section, we propose a simple approach to search for a good protograph which processes a low iterative decoding threshold in a partial response channel. We concentrate on graphs that possess good protograph properties of having a low decoding threshold and the linear minimum distance growth property [6] that guarantees no error floor if random circulants are assigned when lifting the protograph. Such graphs should have degree-1 variable nodes [14], [15], as well as a fraction of high-degree and degree-2 nodes [16].

That observation motivates us to seek protographs that do not contain punctured nodes.

VI. CONCLUSION
This paper presents a simple method for designing capacity approaching protograph-based LDPC codes for partial response channels. A method of calculating iterative decoding thresholds of a joint graph of protograph codes with the state structure of partial response channels based on the EXIT chart is proposed. Our designed code allows easy encoding and fast decoding, and performs close to the i.u.d. capacity limits.
