Serial Scheduling Algorithm of LDPC Decoding for Multimedia Transmission

Minki Kim, Dongho Kim and Ye Hoon Lee

Abstract—In this paper, we consider a layered decoding algorithm for a performance improvement of LDPC. The proposed algorithm is based on a serial scheduling algorithm for the layered decoding of LDPC in which we consider check node degree, variable node degree, and the edge connection between the each node. Simulation results show that the proposed method improves about 0.2dB SNR gain compared with random schedule in a layered decoding. Especially when the maximum number of iteration and the code rate are low, the performance gain gets larger. This is desirable to the system which requires low per consumption and low latency such as WPAN system.
Index Terms—LDPC layered decoding, serial scheduling, layered BP, WPAN
I. INTRODUCTION
AS the demand for connection between the portable devices increases, the short-range wireless communication technology has been growing attention such as Bluetooth, UWB (Ultra Wide Band). Various AV equipments such as TV, camcorders, and HD cameras demand wireless connectivity with very high data rate. Especially when digital HDTV broadcasting systems are universally spread including the future ultra-HDTV, a high-speed wireless data transmission is indispensable because camcorder with HD resolution requires mass data storage and data transfer between two AV equipments. WiMedia Alliance whose members are the manufacturers of AV, home network equipment and wireless USB threw out a standard WiMedia 1.1 specification providing rates up to 480Mbps. Also in the Bluetooth or IEEE 802.11x family of WLAN, brisk research and standardization have been made in terms of power consumption, coverage, and transmission rates.
For the improved data rate, more sophisticated transmission technology such as high order modulation, MIMO etc should be considered. Also more reliable error correction technology is essential for the improved data transmission. In this paper, we consider LDPC code as an error correcting codes and propose a serial scheduling algorithm of layered decoding for a BER performance improvement with lower power consumption. The proposed algorithm is based on principle that updated message should be propagated to other nodes as much as possible. For the proposed layered decoding of LDPC we consider check

node degree, variable node degree, and the edge connection between the each node.
The remainder of this paper is as follows. In Section II, we describe LDPC decoding algorithms. In Section III, we propose the row-layered BP (Belief Propagation) algorithm based on the serial scheduling. The simulation results will be presented and conclusions are made in Section IV and Section V, respectively.

B. Layered Decoding of LDPC
Recently, active research has been done on layered decoding. There are two kinds of layered decoding algorithms, e.g. row-layered decoding [2-3] and column-layered decoding [4]. In the row-layered decoding, parity-check matrix is partitioned into multiple row layers (check nodes) and the message

Message update in a row-layered decoding of LDPC is shown in Fig. 2. In the layered decoding, intermediate updates of node messages and their propagation to the next layer within an iteration are the major factor to speed up the convergence and to improve the error performance. The improvement of error performance appears more pronounced when the maximum number of iterations is limited because of the decoding time requirement. In the first-half stage of update for 1st row-layer (Check Node layer), CN1 is updated using the messages from VN1, VN2, and VN3. The updated CN1 message is directly given to VN1, VN2 and VN3 for second-half stage updates. After the update for 1st row-layer, VN1 has more updated information compared with Flooding algorithm. The more updated VN1 is given to other CN update process, e.g. the 2nd row-layer CN2 is updated using the updated VN1 and initial VN4 and initial VN5. Likewise every check node is updated using more updated VNs. In a layered decoding, we can get improved performance using more updated VNs (or CNs) in less number of iterations.

III. PROPOSED SERIAL MESSAGE PASSING SCHEDULES
A. Row-layered Decoding
The implementation of an LDPC decoder with layered decoding has been studied widely [5-7]. It is reported that the optimum degree distribution in an irregular LDPC shows that the variable node degree is distributed widely, while the check node degree usually consists of two similar values, e.g.
λ (x) = 0.30780 x + 0.27287 x2 + 0.41933 x6
ρ (x) = 0.4x5 + 0.6x6 for a code rate 0.5 [8]. Therefore, the column-layered decoding in which message update process is column-layer by columnlayer requires so called “stall clock” because each variable node needs different number of clocks due to the various VN degrees. On the other hand, row-layered decoding doesn’t require the stall clock because each check node is connected to the similar number of variable nodes, i.e. the similar CN degrees and the required clock count for each check node update is similar. The stall clock degrades the decoding time and the decoding throughput. Considering the hardware design, we propose a serial schedule algorithm based on the row-layered decoding.
B. Proposed Serial Schedule for Row-Layered Decoding
In a layered decoding, sequential schedule of layers to be updated is a very important factor to the error performance. First, we choose a layer based on the parity-check matrix H of LDPC which is described in the bipartite graph. The bipartite graph of LDPC can be divided into smaller subgraphs and each subgraph consists of one check node and all their neighboring variable nodes. The layered decoding progresses sequentially through subgraphs by performing check node update and variable node updates in each subgraph.
We propose the serial schedules of the layer considering the characteristics of the parity-check matrix H of LDPC, e.g. degree distribution, variable node degree, check node degree, and edge connection between the nodes, etc. The proposed serial schedule method is based on the principle that updated message should be propagated to other nodes as much as possible. As described in the Section II, the layered decoding uses the newly updated node message into the next layer’s update process within an iteration. Therefore, as the newly updated node message has effects on other nodes more and earlier, the decoding performance gets improved more.
In the proposed scheme, we choose a row-layer schedule in accordance with the fast message propagation. First, we select a set of check nodes with large check node (CN) degree because they are connected to many variable nodes (VN) and can propagate updated messages to many VNs and the next layer’s CNs. In case that the check nodes have the same degrees, those CNs can be divided into different row-layer schedule group in accordance with their neighboring VN’s degree. When the neighboring VN has the large VN degree, it can also propagate the soft output to the next layer’s CNs and VNs. Consequently, we count the number of VNs with high VN degree and sort the check node with higher number. We also consider the edge

connection between the each node. When a check node constructs or involves a short-cycle in H matrix which degrades the error performance seriously, we set the CN with low schedule priority.
The schedule rule is as follows: 1. Select Check Node (CN) with large CN degree 2. If CN degree is same,
A. count the number of VNs with high VN degree B. Sort the Check Node with the higher number

3. Select Check Node with lower priority, if CN constructs or involves short-cycle in H matrix.
With the proposed schedule rule, every row (check node) in H matrix is classified into the different layer. Finally the layered decoding is processed sequentially in ascending order.
