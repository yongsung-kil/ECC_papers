Low Complexity Product Codes with LDPC Codes Achieving Ultra Low BER
Weigang Chen and Tongxin Dong School of Electronic Information Engineering,
Tianjin University, Tianjin, 300072, P. R. China Email: chenwg@tju.edu.cn; dongtongxin@gmail.com

AbstractŌĆöUltra low bit error rate (BER) is required in high speed communications such as the 100 Gbit/s long haul optic communications. In this paper, a type of simple product codes is proposed using low density parity check (LDPC) codes with very high rate algebraic block codes by exploiting the sparsity of LDPC decoding failure and sparse error distribution in an LDPC frame with errors. Specifically, different with traditional product codes, the information bits are first encoded with very high rate algebraic block codes of low encoding and decoding complexity. Even the block codes correcting a single error using algebraic decoding method can be used. Then, LDPC codes are used as the inner code to obtain a low decoding threshold using iterative decoding. This design scheme can achieve the ultra low bit error rate at relatively low signal-to-noise ratios. The cost of the proposed scheme is the relatively large latency, which is not so critical for the ultra high speed communications.
Index TermsŌĆö Product codes; LDPC codes; ultra low bit error rate; error floor
I. INTRODUCTION
With the increasing of communication traffics, ultra high speed communications have attracted wide interest and the throughput of communication systems has increased greatly. For example, 100 Gbit/s, 400 Gbit/s and even Tbit/s optical communications have been investigated [1], [2] and multiGbit/s wireless communications have also become the hot topics in the wireless communications [3]. The trend will be enforced when using 60 GHz and TeraHertz communications in the future [3]ŌĆō[4]. In these applications, for some latency insensitive services, long block length can be adopted to achieve large channel coding gain. Considering the complexity, it requires new design guideline for error correction codes with quite large block length [5].
Low density parity check (LDPC) codes can approach the Shannon limit, when the code length goes to quite large [6]ŌĆō [8]. They can be decoded with parallel iterative decoding algorithms and can support large throughput. However, due to the suboptimal property of the iterative decoding, error floor phenomena are often observed in the high signal-to-noise ratio (SNR) region of the error probability curves [9]. This limits the applications of LDPC codes in the high throughput systems. In the high throughput communications, usually very low bit error rate (BER) is needed in order to guarantee the scarcity of decoding failure events [5]. In addition, the complexity of decoding very long LDPC codes is also impractical for ___________________________________ 978-1-4673-2101-3/12/$31.00 ┬®2012 IEEE

the high throughput and storage requirement of extrinsic information.
In order to satisfy the BER requirements, there are mainly two approaches in the literature to low the error floors. One is to design the LDPC codes elaborately using some complex design guidelines and to improve the decoding algorithms [10]ŌĆō [12]. This concatenation scheme also has the problem of implementation complexity. The other approach is to use the concatenated codes with LDPC codes as inner codes [13]ŌĆō[16]. This scheme has been adopted in Digital video broadcasting in both Europe and China. However, this scheme can only be used for some special LDPC codes and is not a general method. In these cases, the outer code is only used to lower the error floors. For general LDPC codes used as inner codes, due to the residual error distribution, the rate loss with outer codes will greatly degrade the overall performance. Therefore, in this paper, an alternative strategy using product code [19]ŌĆō [24] is presented using general LDPC codes to reduce the decoding threshold and the error floor.
In this paper, a product construction scheme with LDPC codes and very high rate algebraic block codes is proposed. To design long channel codes, this scheme is different from the traditional product codes, which usually use multiple small block codes as component codes and iterative soft-in softout (SISO) decoding algorithms. In the proposed scheme, we first encode the information bits into K1 block codewords using high rate algebraic block code C1(n1, k1), and then the data are encoded using LDPC code C2(n2, k2) into K2 LDPC codewords. Here, n1 and n2 are the code lengths of the outer code and the inner LDPC code, while k1 and k2 are the information bit lengths of outer code and inner LDPC code, respectively. Therefore, the whole length of the encoded block is K2 ┬Ę n2 and usually is large. This scheme achieves very low error bit rate with relatively small decoding threshold degradation due to integrating the advantages of iterative decoding of LDPC codes and the algebraic decoding of outer codes. Furthermore, the encoding and decoding architecture can scale in proper parallel level according to the requirement of throughput.

II. ERROR PROPERTY ANALYSIS OF LDPC CODES
A. Residual Error Distribution in a Single LDPC Codeword
LDPC codes can achieve relatively low error floor compared with Turbo codes [8], [17], [18] and the residual errors can often be detected using the parity-check matrix, in that most of the errors are caused by some error-prone substructures in the Tanner graphs of the LDPC codes [8]. In this paper, in order to investigate the strategies to lower the error floor, we try to find the error property of the residual errors and to achieve the ultra low BER using some concatenated schemes.
In order to design the coding schemes, we need to match the error property of the outer and inner codes. For example, the concatenated code with algebraic outer codes and Turbo codes is a good example, which uses the outer codes to correct the residual low weight errors due to Turbo codes [17], [18]. Similar idea is adopted in [13]. In this paper, we design coding schemes to achieve ultra low BER using some universal LDPC codes.
Therefore, we first investigate the residual error property of typical LDPC codes. We choose the LDPC codes in the IEEE 802.16e standard [25] and present the error bit property with simulations. Fig. 1 presents the residual error distribution in a single codeword of the LDPC code with the code rate of 1/2 and the code length of 576 bits at the different SNRs. The simulation is under additive white Gaussian noise (AWGN) channel with Binary Phase Shift Keying (BPSK) constellation. All 1000 error events are recorded for analysis. It can be observed that though the number of residual error is usually small compared with the code length, it is still difficult to design concatenated codes using this LDPC codes. If we design concatenated codes to combat most of the residual errors, outer code with quite low code rate is needed. Therefore, in our previous work, we designed a kind of special LDPC codes with quite less residual errors in order to achieve very low bit error probability by using outer codes with very high code rate [13]. In this paper, we use another idea to explore the product codes to distribute the residual errors in multiple high rate algebraic block codes and thus very limited code rate degradation occurs.
B. Residual Error Property for Product Code Design
In this paper, the product code design is based on the following observations. First, as is known, if the bit signal-tonoise ratio, Eb/N0, is quite high, the frame error rate (FER) is quite low over the AWGN channel. The distance between two successive error frames is usually quite large. The cumulative distribution function (CDF) of the distance is illustrated in Fig. 2 for the LDPC code under simulation above. It is observed that the distances between two error frames are usually large and the distance becomes larger and larger with the increasing of the SNRs. This can be explained that the error frame rate decreases with the increasing of SNRs.

It can be observed that concatenated code proposed in [13] can be viewed as a special case, as shown in Fig. 4 (b). For the traditional concatenated codes, all the residual errors in one LDPC codeword are distributed to only a single outer code. We can adjust the number of outer codes, K1, and then the information bits for LDPC codes can be supervised by K1 outer codes. The extreme case is that k2 outer codes are used, i.e. K1 = k2, Then, each outer codeword only includes one bit of each LDPC code, as shown in Fig. 4 (c). In this case, the largest encoded block is formed and it causes the largest latency. The lowest latency can be achieved using the traditional concatenated codes with LDPC codes as inner codes. Indeed, the best error correction capability is achieved when each outer block code only contains one bit of the LDPC code, as shown in Fig. 4 (c).
The design criteria are different compared with traditional product codes. In our design, the outer code has the code rate near to one and can be encoded and decoded with very concise circuits. The LDPC codes should be optimized to have low decoding threshold, but can have a tail in low BER region. It is also the different guideline for designing LDPC codes in the literature. Indeed, we can even use outer code which can only correct a single error. Thus the lowest implementation complexity and the smallest rate loss due to outer code is needed.

A. Product Code Design with LDPC Codes
The proposed code is essentially product code using outer high rate block code C1(n1, k1) and LDPC code C2(n2, k2). It is different with traditional product codes, where usually short block codes are used as component codes. The design scheme in this paper is illustrated in Fig. 4. Universal LDPC codes and very high rate algebraic block codes are used as component codes, respectively. Specifically, the information bits are first decomposed into K1 parts and each part contains k1 bits. The K1 outer codewords are encoded and rearranged as in Fig. 4 (a). Then, all the encoded bits are decomposed into K2 parts and each part is encoded using LDPC code C2(n2, k2) as shown in Fig. 4(a). In this way, a product code with LDPC codes and outer block codes is designed. The whole code rate of the product code is R = (k1 ┬Ę k2)/(n1 ┬Ę n2). In our design the outer code rate R1 = k1/n1 is very high and approximates one, while the LDPC code has the code rate of R2 = k2/n2, approximating the whole code rate of the product code. It is different from the traditional product code design.
From the observation above, in the whole product code, there are very limited decoding failures for the K2 LDPC codewords and then the limited error bits in the failed codewords are distributed into K1 outer block codewords. In this way, each outer code only needs to correct very small number of errors. B. Implementation Issues
In this paper, we use the encoded block of length K2 ┬Ę n2 as a transmit unit, which is usually quite large. In the following, several issues related to implementation are addressed.
First, the main concern is the encoding/decoding latency. This design scheme is aimed for very high speed communications such as beyond Gbit/s wireless communications or over 100 Gbit/s optic communications. In these systems, for some latency-insensitive services, the proposed scheme can serve as a good candidate. Another concern may be the storage requirements. In our scheme, for the decoding of outer codes, we limit to use efficient hard-decision decoding algorithms. Therefore, the data needed to be stored in the encoder or decoder are only K1 ┬Ę n1 bits. This may be accepted for some applications. For the traditional LDPC codes with the same large length, the storage requirements for the extrinsic information are quite impractical and the routing also causes problems.
Furthermore, the proposed product code can be implemented in the limited complexity and has great flexibility. The traditional product codes are often decoded using iterative algorithms in modern coding literature [23], [24]. However, it is not the case for this paper. For the proposed product codes, LDPC codes are decoded using iterative decoding algorithms based on soft information and the low decoding threshold can be achieved. IV. SIMULATION RESULTS AND ANALYSIS
In this section, two examples are presented to verify the effectiveness of the proposed product codes. Both of the LDPC codes are chosen from the members in IEEE 802.16e standard. The first example uses the LDPC code with the code length of 576 bits and the code rate of 1/2. The second example also uses the LDPC code with code length 576 bits and its code rate is 5/6. For the outer code we choose the parameters of BCH(Bose-Chaudhuri-Hocquenghem) codes. For one error correction code, we choose BCH(1023, 1013, t = 1) and BCH(1023, 1003, t = 2) is used to correct two errors. The code rate degradation due to these outer codes is considered in the following simulation results.
In the first example, Fig. 5 presents the results when the outer codes are used. It can be observed that the BER can be significantly reduced sharply using the product code. We also evaluate the case when Eb/N0 =3.5 dB, after decoding theouter code BCH(1023, 1013, t = 1), there are no error events. Therefore, using this proposed scheme, ultra low BER can be achieved for typical LDPC codes by adding an outer code. It is very suitable for applications requiring ultra low error floors. The actual block length in this example is 1023 ├Ś 576 = 589 248 bits. It is quite large, but for high throughput applications, the latency is not so significant.
In the second example, Fig. 6 presents the results of the LDPC code with code length of 576 plus outer code BCH(1023, 1013, t = 1) or BCH(1023, 1003, t = 2). It can be observed that the BER can also be significantly reduced. The actual block length for this example is also also 1023 ├Ś 576 = 589 248 bits. V. CONCLUSION
In this paper, product codes based on new guidelines are designed using LDPC codes and very high rate algebraic block codes. This scheme is different from the traditional product codes. Very high rate algebraic block codes and LDPC codes are combined integrating the advantages of soft decoding of LDPC codes and algebraic decoding of outer codes. This design scheme can achieve the ultra low BER at the relatively low SNRs by exploiting the sparsity of LDPC decoding failure and sparse error distribution in an LDPC codeword with errors. The implementation achieves a good tradeoff compared with long traditional LDPC codes. The cost of the design scheme is the relatively large latency, which is not critical for high speed communications. In the future, LDPC codes with lower decoding threshold will be designed to optimize the proposed product codes.
