A Study Into High-Throughput Decoder Architectures For High-Rate LDPC Codes

Yeong-Luh Ueng, Chung-Chao Cheng Department of Electrical Engineering and the Institute of Communications Engineering
National Tsing Hua University, Hsinchu, Taiwan, R.O.C, Email: ylueng@ee.nthu.edu.tw

Abstract—This study investigates a variety of high-throughput decoder architectures designed for high-rate low-density paritycheck (LDPC) codes. To implement a high-throughput decoder, a fully-parallel architecture can be adopted, but with complex interconnections. In order to reduce the routing complexity, a Split-Row Threshold decoder can be adopted. However, the high check-node degree of a high-rate LDPC code leads to a long critical path when using the Split-Row Threshold decoder. The long critical path can be shortened by using partiallyparallel architectures combined with vertical scheduling. Twophase message passing and shufƀed message passing can be adopted in the vertical scheduling. The features of these state-ofthe-art high-throughput decoder architectures and the associated comparison are presented in this paper.
Index Terms— High-throughput, low-density parity-check (LDPC) codes, Split-Row Threshold decoder, two-phase message passing, shufƀed message passing, sliced message passing.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were ſrst described by Gallager [1] and were the rediscovered by Mackay [2]. LDPC codes have recently received increased attention due to their excellent error-correction capability. Nowadays, many communication standards, such as the 10 Gigabit Ethernet (10GBASE-T) protocol[3] and IEEE 802.11ad[4] have adopted LDPC codes. An LDPC code can be decoded iteratively based on its Tanner graph [5], which is a bipartite graph consisting of variable nodes (VNs) and check nodes (CNs). In [1] and [2], two-phase message passing (TPMP), which divides the decoding operations in a single iteration into the VN-operation and the CN-operation phases, was used.
A fully-parallel architecture can be used to implement a high-throughput decoder, but with complex interconnections. To overcome this difſculty, the Split-Row Threshold decoder presented in [6] can be used, where the parity-check matrix (variable nodes) is divided into several block columns (groups) and the interconnections between these groups are simpliſed. The authors in [6] demonstrated their architecture using a highrate 2048-bit (6,32)-regular LDPC code, where the variablenode and check-node degrees are 6 and 32, respectively. The Split-Row decoder can achieve an utilization of 97%. However, this decoder suffers from error performance loss [6].
In [7], the authors showed that the high check-node degree for high-rate LDPC codes leads to greater complexities in hardware, interconnects, and timing, which are difſcult to manage using a fully-parallel architecture without degradation

in error performance. Consequently, a partially-parallel architecture combined with vertical scheduling, where the variable nodes are partitioned into several groups and each group is processed sequentially in order to overcome the difſculties resultant from the high check-node degrees, can be used. The authors in [7] proposed sliced message passing (SMP), which is a TPMP-based and register-based partially-parallel architecture, to design high-throughput decoders. In this decoder architecture, the critical-path delay can be shortened and, hence, the throughput can be increased. However, the throughput loss caused by the idle cycles between consecutive iterations is signiſcant, especially when the parallelism is high. In order to minimize the idle cycles, the architecture presented in [8], which is also a TPMP-based architecture, can be adopted, where CN messages produced in both the current and previous iterations are saved. In order to further increase the throughput, the shufƀed MPD presented in [9] can be adopted, since fewer iterations can be used to achieve the desired bit-error rate (BER) performance compared to TPMP.

III. DECODER ARCHITECTURE AND SCHEDULING
A. Split-Row Threshold Decoder
A fully parallel decoder immediately maps each column and each row of the parity-check matrix to a different processing unit, and each of these processing units operate in parallel. However, the routing complexity for the decoder is high. To reduce the interconnect complexity due to message passing in the CN update step, the Split-Row Threshold decoder architecture presented in [6] uses column-wise partitioning of the parity-check matrix as shown in Fig. 2(a). Since the magnitude information of the CN processing only needs to propagate locally in this architecture, the routing complexity can be signiſcantly reduced, which is one of the major advantage of this architecture. For example, the Split-Row 16 decoder presented in [6] can achieve an utilization of 97%.

B. Sliced Message Passing Decoder
The Sliced Message Passing (SMP) decoder presented in [7] is a register-based partially parallel implementation. For a partially parallel architecture, VN and CN processing cannot be executed in parallel because one processing stage depends on the computed information passed from the other. In SMP, all the variable nodes, and, hence, all the V2C messages, are divided into G equal-sized segment, then check-node computation is performed sequentially segment by segment. The VN processing and CN processing stages are overlapped, as shown in Fig. 2(b). The CN update stage for each paritycheck equation is divided into G steps. In each step, the new incoming results from the previous VN update stage will be aggregated with to the former results. Instead of waiting for G clock cycles, CN can begin to update the C2V messages after only a single VN computation cycle. The VN in the subsequent iteration can begin immedicately after CN completes the computation of current iteration. Only G + 1 clock cycles are required to process a single decoding iteration. This scheme eliminates the sequential tie issue between the CN and VN update stages, and thus greatly shortens the critical-path delay and improves the throughput. However, when G is small, the idle cycle between consecutive iterations will signiſcantly reduce the decoder throughput, as shown in Fig. 3(b).

and is interleaved with the VN processing. It can be seen that each iteration where pipelining is not cosidered will be completed in G+1 cycles as shown in Fig. 2(c). D. Shufƀed Message Passing Decoder
Fig. 2(d) shows the architecture for the shufƀed MPD. In the shufƀed MPD [9], the VNs are partitioned into G groups, and each group is processed sequentially in order to overcome the difſculties resultant from the high CN degrees. Consequently, the critical-path delay can be shortened, and hence, the throughput can be increased. In addition, the shufƀed MPD can increase the convergence speed in BER, and, hence, requires fewer iterations for a given performance. Although the variable nodes from each group are processed sequentially, data dependency exists in this kind of group-bygroup real-time updating scheme if the updating message from the front group is not ready for processing by the executing group due to inappropriate scheduling. Hence, degradation in convergence speed will be observed. A distinct feature of the shufƀed MPD presented in [9] is that the CN memory, which is called ordered set memory in [9], contains information produced in both current and previous iterations. In addition, three minimum values for each CN are stored in the ordered set memory in order to achieve an improved error performance. Fig. 3(d) shows the processing scheduling for the decoder used in [9].
IV. CONCLUSION
Several high-throughput decoder architectures have been presented including both fully-parallel and partially-parallel architectures, for high-rate LDPC codes. The Split-Row fullyparallel decoder presented in [6] can reduce routing complexity but suffers from error-performance degradation. In addition, the utilization is high and the operation frequency is limited. The partially-parallel TPMP-based architectures presented in [7] and [8] can be used to increase the operating frequency. In the decoder architecture presented in [8], CN messages produced in both the current and the previous iterations are saved in order to reduce the number of idle cycles between consecutive iterations.
