Abstract—Low density parity check (LDPC) codes are one of the best error correcting codes known to approach the Shannon limit. This paper examines the impact of different rates and code lengths of Log Belief Propagation LDPC algorithm on the performance of digital video broadcasting -satellite-second generation (DVB-S2) on both Additive white Gaussian Noise (AWGN) channel and Rayleigh fading channel, it also examines low complexity LDPC algorithm which is minimum sum LDPC algorithm for DVB-S2 with different numbers of iterations and simulating it on both channels using matlab targeting to chose the appropriate algorithm for the architecture of DVB-S2 to be implemented hierarchically on semi-parallel LDPC design Field Programmable Gate Array .
Index Terms—Low Density Parity Check code (LDPC), log Belief Propagation(log-BP), minimum sum(MS),DVBS2, semiparallel LDPC design for Field Programmable Gate Array (FPGA).
I. INTRODUCTION
Low density parity check (LDPC) coding is a Forward error correcting (FEC) technique which is proposed by Gallager in 1963 and rediscovered by Mackay and Neal desiring to achieve bit error rate (BER) performance close to Shannon limit [1]. LDPC codes were adapted by recent communication standards such as 10 Gigabit Ethernet (lOGBASE-T), Worldwide Interoperability for Microwave Access (WiMAX) (802.16e), Wi-Fi (802.11n), 60 GHz WPAN (802.15.3c) [2] and Digital Video Broadcasting - satellite-second generation (DVB-S2)[3] .The latter is specified for satellite broadband applications, such as TV and sound broadcasting, internet access and professional services like TV contribution links and digital satellite news gathering [4,5]. This paper presents DVB-S2 decoder using soft decision decoding technique which includes Log-BP (Log Belief Propagation) LDPC algorithm, it is also known as sum product decoding algorithm. Log-BP is equivalent to maximum likelihood decoding algorithm. In order to achieve a balance between complexity and performance, many simplified algorithms have been proposed recently including the most representative Minimum Sum (MS) LDPC algorithm.

Although MS is easily implemented, it results in performance loss. After showing the performance of these decoders and comparing between them on different channels, it was desired to implement the MS LDPC algorithm using semi-parallel design implemented on FPGA.

II. DVB-S2 BASIC THEORY

The DVB-S2 was established by satellite broadcast service standard at 2003. This system has four different modulation methods, QPSK, 8 PSK, 16 APSK and 32 APSK. Different LDPC code rates handle each modulation method. High throughput modulation is used to achieve good SNR while if fading is introduced to the channel, the system must use lower throughput modulation. III. LDPC CODES DESCRIPTIONS LDPC codes are block codes specified by very sparse M∗ N parity check matrix H [2], in which most of the elements are zeros and few are ones. The LDPC parity check matrix H can be transformed into a graphical representation, Tanners graph ( bipartite graph) as shown in Figure.1 which composes of two types of processing elements. These elements are check nodes (CNi) and variable nodes (CNj). Variable nodes are equal to N columns of H matrix while check nodes are equal to M rows. VI. REVIEW ON FPGA IMPLEMENTATION DESIGNS
Analyzing the performance of both decoders sum product and MS, it is clear that it is easier to implement the MS LDPC algorithm shown in equation (2). After investigation on its implementation, it was found that there are three different ways for implementation as shown in Table.2. The fully parallel implementation requires a large circuit size to implement while series implementation will not meet the time constrain of most applications which require high throughput. As a result it was decided that semi-parallel design which achieve a good trade-off between hardware complexity and decoding throughput, will be used to implement DVB-S2 using MS LDPC algorithm.

VII. CONCLUSION
This paper presents two LDPC decoder algorithms named Log-BP and MS implemented using m-files on matlab. The performance of Log-BP on both channels was studied. At constant BER and code rate, the SNR of Rayleigh channel

is nearly twice that of AWGN channel. Therefore the performance on Rayleigh channel is worse, but nearer to real life. The difference between Log-BP and MS algorithms on both channels was analyzed. At constant SNR, code rate and number of iterations, MS algorithm gives higher BER than Log-BP algorithm. Therefore the performance of MS algorithm is worse than Log-BP algorithm, but better from the hardware implementation point. The number of iterations was also studied and it is clear that the higher the number of iterations the better the performance. The architecture of DVBS2 (8100, 4050) LDPC code using MS algorithm is ready for implementation on FPGA using semi-parallel design. The latter outperforms the other designs in complexity as it achieves a good trade-off between hardware complexity and decoding throughput.
