Abstract - Low density parity check (LDPC) codes with iterative decoding based on belief propagation (IDBP) achieve astonishing error performance close to the Shannon limit. Until now there is no known method for constructing these Shannon limit approaching codes systematically. Good LDPC codes are largely generated by computer search. As a result, encoding of long LDPC codes is in general very complex. This paper presents the first algebraic method for constructing LDPC codes systematically based on finite analytic geometries. Four classes of finite geometry LDPC codes with relatively good minimum distances are constructed. Theses codes are either cyclic or quasi-cyclic and therefore their encoding can be implemented with simple linear feedback shift registers. Long finite geometry LDPC codes have been constructed and they achieve an error performance only a few tenths of a dB away from the Shannon limit. Finite geometry LDPC codes are strong competitors to turbo codes for error control in commucation and digital data storage systems.
I. INTRODUCTION
LDPC codes were first discovered by Gallager [l]in early nineteen sixties and have recently been rediscovered and generalized [2-61. These codes with IDBP achieve an astonishing error performance close to the Shannon limit [3,5,6]. An LDPC code of length n is defined by a J x n parity check matrix H with the following structural properties: (1) each row consists of p “ones”; (2) each column consists of y “ones”; (3) the number of “ones” in common between any two columns, denoted A, is no greater than one; and (4) both p and y are small compared to the length n of the code and the number of rows, J , in H. The code is simply the null space of H. The density r of H is defined as the ratio of the total number of “ones” in H to the total number of entries in H. Then r = p / n = y / J . Since both p and y are small compared to the length of the code and the number of rows of H, the

density of H is small. For this reason, H is said to be a low density parity check matrix and its null space is hence called an LDPC code. Basically, H is a sparse matrix. Note that the third property of the parity check matrix implies that no two rows of H have more than one “1” in common. The LDPC code defined above is called a regular LDPC code. If not all the rows or all the columns of the parity check matrix H have the same number of “ones”, an LDPC code is then said t o be irregular. In this paper, we are mainly concerned with the construction of regular LDPC codes.
Although LDPC codes have been shown to achieve outstanding error performance, no analytic (algebraic or geometric) method has been found for constructing these codes. Good LDPC codes that have been constructed are largely computer generated, especially long codes. Encoding of these long computer generated codes is very complex. In this paper, we present an algebraic method for constructing regular LDPC codes systematically based on the points and lines of finite analytic geometries. Four classes of finite geometry LDPC codes are constructed. These codes have relatively good minimum distances and they can be decoded in various ways ranging from low to high decoding complexity and from reasonably good to very good error performance. Furthermore they are either cyclic or quasi-cyclic and hence their encoding can be implemented with simple linear feedback shift registers. Finite geometry LDPC codes can be extended and punctured to obtain a large class of LDPC codes. Extended finite geometry LDPC codes have been constructed to achieve an error performance only a few tenths of a dB away from the Shannon limit.

A type-I (or type-11) finite geometry LDPC code can be punctured by removing the columns of its parity check matrix H that correspond to the points (or lines) of a hyperplane of the geometry [7,8]from which the code is constructed. The punctuation results in a new low density matrix whose null space gives a shortened finite geometry LDPC code. The shortened code has at least the same minimum distance as the original base code.

V. CONCLUSION
This paper presents the first algebraic method for constructing LDPC codes based on finite geometries. Finite geometry LDPC codes can be decoded in a t least six different ways namely: one-step majority logic decoding, Gallager's bit flipping (BF) decoding, weighted majority-logic decoding, weighted BF decoding, a posteriori probability (APP) decoding and IDBP [9]. The one-step majority decoding requires the least decoding complexity but provides the least coding gain over the uncoded system. The IDBP gives the best error performance but requires the largest computational complexity. The BF and weighted

Finite geometry LDPC codes have several advantages over turbo codes such as: (1) they do not require a long interleaver to achieve good bit error performance; (2) they have much better block error performance; (3) their error floor occurs at a much lower bit error rate; (4)high rate codes can be easily constructed without puncturing; (5) with IDBP decoding, each decoding iteration produces all the decoded bits (information and parity bits) which can be used to compute syndrome for error detection to terminate the decoding iteration process; and (6) decoding is not trellis-based.
Since finite geometry LDPC codes are either cyclic or quasi-cyclic, their encoding is much simpler than the computer generated Gallager LDPC codes. For code length up to 1023, a finite geometry LDPC code performs better than the best computer generated Gallager LDPC code of the same length and the same rate with IDBP [9]. Simulation results show that IDBP of finite geometry LDPC codes also converges faster than their equivalent Gallager LDPC codes.
This paper gives an evidence that algebraic codes with a proper devised decoding perform just as good as random codes. So don’t give up algebraic codes.
