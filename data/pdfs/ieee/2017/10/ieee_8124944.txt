Abstract—This paper proposes cost efficient very high throughput layered decoding architecture for array quasi-cyclic LowDensity Parity-Check (QC-LDPC) codes, targeting tens of Gbps data rates. The targeted throughput is achieved by employing layer unrolling, with pipeline stages inserted in between layers. In order to obtain improved hardware efficiency for the decoder, multiple codewords are processed simultaneously. This leads to increased memory overhead. In order to reduce the associated cost, approximate message storage using Non Surjective-Finite Alphabet Iterative Decoding (NS-FAID) compression tables is employed. Cost efficiency is achieved by hardwired interconnects, compressed message storage using the NS-FAID, as well the A Posteriori Log-Likelihood Ratio (AP-LLR) message memory removal. Several compression tables are evaluated using the Throughput to Area Ratio (TAR) metric.
Index Terms—Forward Error Correction, LDPC Decoding, Layered Scheduling, NS-FAID
I. INTRODUCTION
Forward Error Correction (FEC) represents an important component in both wireless communication systems and NAND based Flash storage devices. With ever increasing data rate requirements, novel approaches on both algorithmic and architectural level are required to accommodate this trend. Low Density Parity Check (LDPC) codes are a class of linear algebraic codes, with decoding performance close to the channel capacity and increased potential for highly parallel decoding architectures [1]. Therefore, improved error correction capability at very high throughputs can be obtained for these codes. LDPC decoding is performed using message passing algorithms, such as Min-Sum (MS) and its variants [2]. These algorithms rely on passing soft messages (messages quantized on a number of bits) along a bipartite graph - Tanner graph [3] - between two types of nodes - variable nodes(VN) and check node(CN) - in several iterations. The hardware architectures implementing message passing LDPC decoding are composed of processing units, interconnection networks and memory blocks. The number and the size of these components are dependent on the desired area and parallelism, as well as on the given code.
Decoding can be performed using two types of scheduling strategies: flooded and layered. The latter has two important advantages: (i) increased convergence, which results in reduced number of iterations [4], and (ii) reduced memory requirements [5]. The increased convergence is due to the

fact that AP-LLR message updates are performed after each layer; thus, updating of these message is performed several times during one iteration. In order to increase the throughput of layered decoding architectures, layer unrolling has been proposed in [6]. Combined with multiple codeword processing, this technique has shown that throughputs of tens of Gbps can be obtained. Unrolled layered architectures are suitable for very high rate codes, as well as array LDPC codes.
In this paper, we aim to reduce the memory requirements for unrolled layered decoders by employing Non Surjective Finite Alphabet Iterative Decoding. NS-FAID represents a method that builds on the classical MS, while performing a tight fitting on message storage [7]. The basic idea is to have two working domains in terms of quantization during the iterative processing. The crossing between the two domains is made by means of the framing and de-framing functions. From the memory storage perspective, these are in fact compression/decompression tables.
This work evaluates the impact of using NS-FAID inside a high throughput layer unrolled decoder architecture implementation of a (3,12) regular array code. First, we need to derive the NS-FAID tables for this code, using the methodology proposed in [8]. Next, we assess the NS-FAID efficiency by implementing the corresponding FPGA unrolled layered architecture using the framing/deframing tables obtained earlier. Synthesis results for Xilinx Virtex-7 devices indicate improvements of up to 30% in logic resource usage with respect to MS based decoders, and an increase of TAR between 25% and 125%.
This paper is organized as follows: the theoretical background regarding layered decoding of LDPC codes using NS-FAID is described in Section II; the proposed unrolled layered architecture using NS-FAID based processing units is presented in Section III; Section IV is dedicated to FPGA synthesis estimates.
II. THEORETICAL BACKGROUND
A. Layered decoding of QC-LDPC codes
LDPC codes are a class of linear algebraic codes, defined by the parity check matrix H. Quasi-cyclic LDPC (QC-LDPC) are a sub-class of LDPC codes [9], for which the parity check matrix is obtained by expanding each element of the base matrix B with a square matrix of size z × z – with z

being the expansion factor. The special structure of the QCLDPC code allows efficient hardware implementation for the decoder; therefore, QC-LDPC codes are the LDPC codes used in communication and storage standards.The z ×z matrix used during the expansion is either the zero matrix, for negative entries of the B, either the identity matrix shifted by the nonnegative entry value of B.
LDPC decoding is performed using message passing algorithms. These rely on sending messages along a bipartite graph, called the Tanner graph [3]. The messages are passed between two types of nodes: variable nodes, corresponding to the codeword bits, and the columns of the H matrix, and check nodes, corresponding to the parity check equations, and the rows of the H matrix. The highest error correction capability is obtained by employing soft extrinsic message algorithms, such as the MS decoding [2]. MS and its variants (such as Offset MS, Normalized MS, Self-Correcting MS [10]) are suitable for hardware implementations, as these algorithms rely on simple arithmetic operations: additions at VN level, and comparisons at CN level. Given the fact the messages are quantized on a small number of bits (3-6), the processing units corresponding to VN and CN operations are fairly simple. However, due to the size of the H matrix (thousands of columns and hundreds/thousands of rows), the decoding architectures are complex, with a wide range of trade-offs between the cost (processing units, interconnects and memory blocks) and the throughput of the decoder.
The following messages are used in LDPC decoding: the channel LLR, γ, represent the input messages; check to variable nodes messages, β; variable to check node messages, α; the AP-LLR message, γ˜. Messages are quantized on a specific number of bits, with β and α quantized on q bits, while γ˜ quantized on p bits, with p ≥ q. The number of nonzero elements in a row of the H matrix represent the check node degree dc, while the number of non-zero elements in a column of the H matrix represent the variable node degree dv .
LDPC decoding can be performed by employing flooded or layered scheduling. The later relies on splitting the parity check matrix rows in several layers nl. The processing of layers is performed serially (one after another), while the processing within a layer is performed in parallel [5]. Each layer processing results in an update of the AP-LLR messages, which leads to an increase in the decoding convergence. For QC-LDPC codes, each layer consists of one row in the base matrix B (z rows in H). A description of layered decoding using MS algorithm is provided in Algorithm 1.

III. ARCHITECTURE
A. Unrolled layered architecture
Hardware unrolling has been applied for both flooded [11] and layered [6] LDPC decoders, in order to obtain throughput in range of tens of Gbps. For flooded, iteration unrolling for a fully parallel decoder implementation has been performed, while for layered scheduling layer unrolling has been used (for only one iteration). Combined with multiple codeword decoding, unrolled architectures have proved to have improved efficiency with respect to other decoding architectures.

Fig. 1. Unroll layered architecture overview

In this paper, we have implemented an unrolled layered decoding architecture for NS-FAID based decoding (Fig. 1). It consists of nl stages, corresponding to each of the processed layers of an iteration decoding. The processing of each stage/layer takes one clock cycle. Therefore, one decoding iteration requires nl clock cycles to complete. Pipelining is applied, with pipeline registers placed between each layer processing. In order to increase the hardware usage efficiency, nl codewords are decoded at the same time, with each codeword being processed in one pipeline stage during a clock cycle. Pipeline registers are used to store the γ˜ messages.
Each stage consist of z combined variable-check node (VCN) processing units. For each unit, a local memory for storing nl × β messages required for a codeword is employed. For FPGA devices, this memory is implemented using the dedicated shift registers (SRL). The depth of the employed SRL is nl. For efficient memory storage, a compressed form of the β message is employed: instead of storing all dc β messages corresponding to a row in the H matrix, we only store their signs, the first absolute minimum value, the second minimum value, as well as the index for the message with the first minimum value.
An important feature of the unrolled architecture with respect to other layered LDPC decoders (such as the ones in [12], [13], [14]) is represented by the lack of barrel-shifters based routing network. For the unrolled layered, the message routing logic for each stage is hardwired. This results in a significant reduction of logic resources when implemented in FPGA technology.
B. NS-FAID based processing units
NS-FAID decoding requires the implementation of framing/de-framing functions within the processing units. The corresponding combined VCN units perform the following operations (see Fig. 2):

The de-framing operation is performed using lookup tables designed according to the NS-FAID rules. In order to reduce the critical path, a look-up table which performs both the de-framing operation and the SM to C2 conversion is employed. 2) CN update - This operation requires the comparison between the absolute values of updated α messages in order to obtain the new β message. This comparison is performed in the low quantization domain. Therefore, the α messages require framing [7]. Due to the fact that absolute values are required, conversion from C2 to SM is also performed. Both the de-framing and the format conversion are computed by using a single combined LUT. The CN update requires a dc-input comparator. Therefore, for high rate array LDPC codes having large dc values, the reduced precision CN implementation will lead to significant cost savings. The newly update β messages are written in the local memory of each layer, being represented in SM framed format. 3) AP-LLR update - This operation requires the updating of the γ˜ message, by adding in full precision the newly obtained β message contribution. As the updated β message is framed and in SM representation, deframing operation and format conversion are required. Similar to the VN update case, these two are performed using a single look-up table.

IV. RESULTS AND DISCUSSIONS
In order to evaluate the NS-FAID approach, we have used the code described in Table II. It is an array code corresponding to a regular, rate 3/4, with variable node degree dv = 3, and code size 1296 bits. The number of layers unrolled in this case is 3 resulting in a total of 108 × 3 VCN units. The

baseline MS implementation considers w = q = 4 bits for CN message representation, and 6 bits for γ˜ messages; it is denoted as MS(4,6). For evaluation purposes we have implemented 4 NS-FAID decoders. Two for CNU message width w = 2, and 2 for w = 3. For all cases the size of the AP-LLR messages is the same as that of the baseline MS – 6 bits. As described in [6], the baseline MS(4,6) decoder requires 23328 bits for APLLR message storage, and 21384 bits for check-to-variable β storage. By using NS-FAID, the β storage requirements are reduced to 19440 bits for w = 3, and 17496 bits for w = 2. Although, by employing the compressed β message storage, the advantages in storage are diminished, there are significant gains in processing complexity reduction because of the large amount of parallel VCN units used in the decoder unit. As noted previously in [6], in unrolled architectures, the logic part dominates the storage requirements for FPGA technology. On the processing side NS-FAID requires the adding of compression/decompression as depicted in Fig. 2. However, for the VCN implementation, the comparator tree has the highest influence on the critical path, especially with the code rate increase. Hence, there is significant cost gain that counterbalances the additional tables. For the code in Table II, a 12-input comparator is needed.

V. CONCLUSIONS
This paper has investigated the applicability of NS-FAIDs for regular (3,12) codes that are implemented in very high throughput decoder architectures using layered unroll concept. A total of four architectures have been implemented in FPGA, for both w = 2 and w = 3 framing functions. The TAR improvement ratio suggest that NS-FAIDs implementation within this architecture yield improvement of 20% up to 125%, with a cost reduction of 10% for w = 3 and 30% for w = 2. The decoding performance estimated through DE predicts for the worst case a 0.5 dB degradation of the decoding performance in the case of w = 2, and for the w = 3 a 0.1 dB improvement.
