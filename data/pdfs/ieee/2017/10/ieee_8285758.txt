Hybrid Hamming GLDPC Codes over the Binary Erasure Channel

Rong Guan*, Lijun Zhang School of Electronics and Information Engineering, Beijing Jiaotong University, Beijing, China
15120069@bjtu.edu.cn

Abstract—Generalized low-density parity-check (GLDPC) codes are a class of iteratively decodable codes, which can change code rate flexibly and be close to the Shannon limit. Empirical evidence suggests that GLDPC codes have advantages over standard low-density parity-check (LDPC) codes in both of error floor and waterfall performance. In this paper, to further approach Shannon limit, the strategy of mixing single parity check (SPC) code with Hamming code to act as the component code of the check nodes (CNs) for GLDPC code is proposed. By using extrinsic information transfer (EXIT) chart, we can estimate the decoding threshold and degree distribution through adjusting the proportion of SPC code and Hamming code over the binary erasure channel (BEC). With the aid of EXIT charts, we construct GLDPC code based on the best degree distribution. Simulation results show that, in comparison with standard LDPC codes and the state-of-the-art GLDPC codes with hybrid variable nodes (VNs), the GLDPC code proposed in this paper has better asymptotic performance.
Keywords—GLDPC codes, EXIT chart, Shannon limit, BEC, Hamming code, hill climbing, degree distribution
I. INTRODUCTION
The idea of expanding and generalizing LDPC code[1] first proposed by Tanner in[2]. His research contributed greatly to understanding the belief propagation (BP) decoding algorithm[3] by proposed the bipartite graph, also known as Tanner graph. The process of iterative decoding between variable node decoders (VNDs) and check node decoders (CNDs) restrained by repetition (REP) code and SPC code. A lot of research indicates that, instead of SPC code, using Hamming codes[4], Reed-Solomon codes[5] and BCH codes[6] as component codes of CNs for GLDPC code can obtain better performance.
To evaluate the convergence behavior and tracking extrinsic log-likelihood ratios (L-values) of iterative decoder, the density evolution (DE)[7], Gaussian approximation (GA)[8] and EXIT chart[9] are effective methods for standard LDPC codes. These methods are also applicable to iterative decoding for GLDPC codes. Remarkably, the method of EXIT chart, showing the asymptotic performance by curve graph, has less computational complexity than DE and wider applicable scope than GA. The idea of EXIT chart is tracking the average mutual information transfer between the decoders. The process is simple over the BEC and empirical evidence suggests that a constructed GLDPC code is suited for many practical channels when it performs well over the BEC[10].
One aim of this paper is to analyze theoretical asymptotic performance by using EXIT chart. Iterative decodable codes, including GLDPC codes, have threshold effect when study the

To further approach Shannon limit, we can use the method of hill climbing to find the threshold and the corresponding best degree distribution when adjusting the component codes. Section IV presents simulation results of the proposed codes over the BEC to justify the advantage. Section V summarizes our results.

II. PRELIMINARIES
A. Construction of GLDPC codes
GLDPC code is an extension of LDPC code, so it can also be defined by check matrix which satisfies the sparse characteristic[12]. Through replacing SPC constraints with other component codes, we can construct the check matrix by using the check matrices H of LDPC code and h of component code. For example, consider mixing the CNs of (7,4) Hamming code and (7,6) SPC code. As shown in Fig. 1, in the first row of H, we replace the nonzero elements by one of the columns of h and zero elements by zero column-vector. This means that 1/3 of the CNs are Hamming codes and 2/3 are SPC codes.

A. Hill-climbing Algorithm
Consider a GLDPC code with fixed rate, we can use EXIT chart to find the best degree distribution over the BEC with erasure probability q. To initialize the algorithm, we set q=0 and then execute the following steps
1) Fix the degree distribution of VNs and CNs. Make VND curve approach CND curve gradually by increasing q. Record the critical value q when they are tangent..
2) Fix the q and adjust the degree distribution, we can see that VND curve and CND curve separate again.
3) Repeat Step 1 and Step 2, and find a decoding threshold qthres which can’t make the curves separate again no matter how to change the degree distribution.
Example 1: Consider a rate-1/2 GLDPC code VNs of degree dv = 2, 3, 4, and 6. We set 1/5 of CNs as (15,11) Hamming code and 4/5 as (6,5) SPC code. We can plot the EXIT chart via (1), (2), (5), (6), and (7) by using hill-climbing method. The resulting chart is shown in Fig. 3 with qthres = 0.48, and the corresponding degree distribution is λ2 = 0.52, λ3 = 0.20, λ4 = 0.00, and λ6 = 0.28. Consider a rate R GLDPC code over the BEC, the Shannon limit is 1 − R. So the rate-1/2 code performs 0.5 − 0.48 = 0.02 from the Shannon limit.

IV. SIMULATION RESULTS
A. Iterative Decoding of GLDPC Codes for the BEC
For correcting erased bits, the decoding algorithm used in this paper is based on the iterative decoding algorithm of

LDPC codes[15]. Consider mixing the CNs of SPC codes and Hamming codes. In the following, we describe a decoding method over the BEC concisely. Consider a LDPC code with parity-check matrix Hldpc. Choose part of rows and replace the 1-components in each row with the columns of Hamming parity-check matrix h. Hamming code is capable of detecting any erasure pattern with no more than two random erasures. Let H be the parity-check matrix and r be the received sequence according to a GLDPC code. Set k = 0 before the algorithm executes.
1) Determine the set of erased locations ξk, if it is empty, stoping decoding, otherwise go to Step 2.
2) Form Hξk from H with the columns corresponding to the erased locations.
3) (SPC constraint) Find a set θk of rows in Hξk . In θk, each row just has a single 1-component. We can use SPC constraint to correct the erased locations in ξk that correspond to the 1-component in each row of θk.
4) Remove the locations of the erasures which corrected at Step 3 form ξk.
5) (Hamming constraint) Let rξk denote the subsequence of r that consist of erased bits in r. Find a set ψk, each row in ψk denote the locations of 1-components which replaced with h. If the locations of r that correspond to ψk have no more than two elements erased, we can use the Hamming constraint to correct them.
6) Remove the locations of the erasures which corrected at Step 5 form ξk. Set k = k + 1, then go to Step 1.

V. CONCLUSION
For approaching Shannon limit, a technique (hill climbing algorithm) for searching best degree distribution with fixed code rate over the BEC has been proposed. With the aid of EXIT charts, this technique is used to present the asymptotic performance for a GLDPC code with hybrid CNs. Numerical

results show that the performance can be affected significantly by adjusting degree distribution.
In addition, a iterative decoding algorithm was proposed that applies to Hamming GLDPC code over the BEC. Simulation results show that the GLDPC code constructed in this paper performs well over the BEC. Moreover, it outperforms the Gallager code in terms of distance from Shannon limit.
