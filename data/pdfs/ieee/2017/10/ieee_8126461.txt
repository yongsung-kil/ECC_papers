Program and Read Methods with Offset in Quad-Level-Cell NAND Design

Liu Shijun, Zou Xuecheng, Wang Baocun

Huazhong University of Science and Technology, School of Optics and Electronic Information, 430074
Wuhan, HuBei, China 86-13916629018, liushijun_sh@hotmail.com

Abstract
3D QLC (Quad-Level-Cell) NAND technology with 16 voltage levels per cell will be one of the next generation memory technologies after 3D TLC (Triple Level Cell) NAND flash succeeded. Besides, program algorithm for 16 voltage levels is studied in this paper, the important read algorithms are investigated because the data errors of QLC device will be easily generated due to power loss, program distribute, etc. The read method with offset and soft message generation for Low Density Parity Check (LDPC) Belief Propagation (BP) soft-decision decoding are presented.
Keywords: QLC NAND; Program Method; Read Method;
Introduction
In the past 30 years, NAND Flash technology had been innovated from 2.0um [1] to 1Xnm [2] in 2D planar design, and in recent 10 years, 3D TLC (3bits/cell) NAND flash devices have been successfully developed [3]. But the nonvolatile memory (NVM) market continually requires new memory products with lower cost/Gb and acceptable performance. 3D QLC NAND Flash technology is being studied and 4 bits/cell causes the memory density increase.
The bit error rate (BER) of TLC is worse than MLC [4]. For QLC NAND which has 16 voltage levels with narrower threshold voltage (Vth) margins write the data by using the Incremental Step Pulse Programming (ISPP) Scheme[5] to increase the program pulse voltage (Vpp) step by step (ǻVpp) to charge QLC cells with more precise voltage control in tight slots. QLC will be worse than TLC. More read methods need to be used for accurate voltage verifying and data reading. The Low-Density-Parity-Check (LDPC) will be used to improve the reliability of QLC devices in system level application. The performance of LDPC decoding is based on log-likelihood ratio (LLR) [6].
The characters of QLC NAND flash are studied, and QLCbased program and read operating are presented. Moreover, in order to avoid the data errors and provide the internal soft message for QLC system application, more read method and the soft message generation by shift reading operation to provide different LLR rates for LDPC Belief Propagation (BP) decoding in system level application are studied.
Program Method in QLC NAND
QLC flash cell has a storage layer to store electrons for controlling the threshold voltage. Fig. 1 shows the voltage distribution of QLC cells: 4bits/cell (bit[3:0], 16 levels: L0~L15). Accordingly, 4 page definitions (page3~page0)

are mapped for bit [3:0]. The input data can be stored in four pages which are in one same wordline(WL). The relationship between the voltages and data values follows Gray coding with 1 bit change for the neighboring bit mapping.
Fig. 2 presents the properties of 3D QLC NAND flash design. The architecture is based on 3D Trap Charge (CT) NAND technology which has layers of Bitline (BL), BL Select (BLS), Wordline (WLs), SoureLine (SL), SL Select (SLS) and substrate. All the cells are around the pillars in vertical. One NAND flash cell is made of Channel layer, Tunnel Oxide (TOX) layer, SiN layer, Blocking Oxide (BOX) layer and Control Gate (CG) layer (alias WL).
Based on the study of QLC 16-voltage-level design, Fig. 3 illustrates one of QLC programming algorithms to write the data. Initially, the QLC cell is erased: the cell voltage is in L0 (data “1” - erased, and “0” - programmed). When programming, there are 2 steps: (1) data of page0 and page1 are written into cache first page by page, then they are computed as bit[1:0], and then the QLC cell is charged to reach to the voltage level according to bit[1:0]. (2) data of page2 and page3 are written to cache independently, at last compute the data of bit[3:2] and program the voltage to match the data of bit[3:2] at the same time. The voltage of the QLC cell is kept in one of the 16 levels (L0~L15).
As example is shown in Fig. 4, The data of 4 pages are stored into QLC cells a/b/c/d/etc according to step 1 and step2, e.g. bit[0] of page0 ~ 3 are stored to cell a, and so on.
But because of issue of program distribution in 3D NAND design, when programming the page in WLn, the cell in the adjacent WL, such as WLn-1 which has completed the programming operation should be interfered. As shown in Fig. 5 (a), the high voltage (20V) is set in WLn to do the programming operation, but according to 3D NAND’s pillar architecture, the high voltage has a high coupling effect to the cells in WLn-1, WLn+1 and other WLs. So the page mapping which means the programming order needs to be carefully considered to follow a staggered order from lower addresses to the higher addresses. Fig 5(b) shows one of the possible page mapping for QLC NAND design.
Read Method in QLC NAND
For NAND read operation, read voltages (Vrd) are used to judge the cell voltage (Vcell): if Vcell > Vrd, data=0, else data=1. Fig. 6 shows the read method of QLC NAND. 16 defined levels for reading use the “half-change” Gray coding which has 1-bit Hamming distance in order to reduce the data read errors. Due to the limitation of NAND flash cell, the max available voltage Vpass is about 5~8V. 15 read verify voltages (Vr1~Vr15) are designed in QLC NAND. L0 is mapped to data “1111” as “erase” status. Because of data errors in QLC NAND chip, such as the endurance issue because of program/erase (P/E) cycle [7], the retention issue because of charge loss [8], etc. The bit error rate (BER) in QLC will be increased. In order to better apply QLC chips in system level, LDPC encoding and BP softdecision decoding are needed for QLC device.
As shown in Fig. 7, the overlap area near each read voltage can apply several shifted voltages in each side like Va1 and Vb1 with +/- dv bias from the standard read voltage (Vrd). The higher the number of bias read levels for getting soft message, the more accurate the information for ECC is, but meanwhile, the more times for read operations which sacrifice the chip read performance.
Fig. 7 (a) is the standard read flow to read out the “hard bit” with the same quantization LLR {-1, +1} by the normal read (Vrd) for LDPC BP hard-decision decoding. Fig. 7 (b) shows that new data can be read out after using the bias voltages Va1/Vb1. After XOR 2 batch of these new data, the soft data is generated for BP soft-decision decoding. Fig. 7 (c) shows that the new data can be read out after using the bias threshold voltages Va1/Va2/Vb1/Vb2. Fig 7 (d) shows that because the data in areas a1/b1 have low confidence and the data in a2/b3 have high confidence, the soft messages in Fig. 7 (b) and (c) indicate the confidence levels associated to the “hard bit” of Fig. 7 (a) with different quantization LLR {-7, -1, +1, +7} and {-10, -5, -1, +1, +5, +10}. By using the bias read voltage design in QLC design, the LLR value indicates whether the “hard bit” is more likely to be a “0” or “1”.
QLC NAND chip can be designed to shift the bias read voltages by adjusting the internal registers. After introducing the above read mechanism in QLC chip design to generate soft messages for BP soft-decision decoding, the efficiency of LDPC error correction will be improved greatly.
Summary
QLC NAND technology with 16 levels is studied in this paper, and the data level definition and data mapping based on Gray coding with 1-bit Hamming distance are stated. Meanwhile, the mechanism which generates the soft-bit message by internal multiple read operations with the bias voltage judgment in QLC NAND design shows the good effects for BP-LDPC soft-decision decoding.
