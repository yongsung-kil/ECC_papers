An Efficient Procedure for Bilayer-Expurgated LDPC Codes Design in Cooperative Relay Channels

Sajad Mehrizi, Sara Khosravi, and Mahmoud Ahmadian

Abstract— This letter proposes a fast optimization procedure to design bilayer-expurgated low density parity check codes in the relay channel. The code optimization problem aims to maximize the code rate while guaranteeing the convergence of the density evolution relations in upper and lower layer codes. The original problem is linear programming. Nevertheless, it suffers from high computational complexity, as it is semiinfinite programming (SIP) in a 2-D continuous interval. To efficiently solve it, we first approximate the density evolutions by polynomials. Then, leveraging optimization problems over the solution of sum of square multivariate polynomials the problem is transformed to semi-definite programming (SDP), where the global solution can be found efficiently using available SDP solvers. Our simulation shows that our approach results in significantly lower computational complexity with respect to the well-known method of quantization for solving the SIP problem, while both achieve the same performance.
Index Terms— Bilayer-expurgated LDPC codes, relay channel, density evolution, code optimization, sum of square polynomials.
I. INTRODUCTION
C OOPERATIVE relaying strategy can significantly improve the channel capacity, power consumption and coverage in wireless communications. The relay channel was first studied in [1] and it was proved that decode and forward (DF) relaying schema is capacity approaching for a class of relay channel. However, designing a coding strategy to achieve the capacity is still an open problem.
Low-density parity-check (LDPC) codes have been of great interest due to their ability to perform close to the Shannon capacity [2], [3]. These forward error-correcting codes are described by sparse parity check matrices which make them efficiently decoded by iterative message-passing algorithms.
LDPC codes are promising coding strategy to apply in relay channels. Authors in [4] and [5] designed coding schemas in a relay channel consisting of one source, one destination and one relay to enhance the capacity of the system. Two LDPC coding strategies are proposed in [6] for DF relaying schema which are termed as bilayer-expurgated codes and bilayerlengthened codes. These two coding strategies are designed at two different SNRs; a higher SNR at the relay and a lower SNR at the destination.
The extrinsic-information transfer (EXIT) chart and density evolution analyses are powerful analytical tools which they have been developed for LDPC codes design [7, Ch. 3], [8]. The EXIT-chart is easier to implement, but it is not as accurate as the density evolution [7, Ch. 3]. Therefore, we will focus on the density evolution analysis.

Regarding bilayer LDPC codes, the authors in [6] designed

an optimization problem to find a code with maximum rate

by tracking error probabilities of message densities. However,

the problem is intractable to solve as it is a multidimensional

problem. In [9], authors proposed a simplified 1-D problem to

update message densities with sum-product decoding of LDPC

codes for AWGN channel. The main idea is to approximate

the message densities by a Gaussian distribution. By virtue of

the symmetry property of the messages, the variance of the

Gaussian messages is twice the mean. Therefore, the problem

of calculating the message densities leads to the problem of

tracking the mean of the messages only.

However, the simplified proposed algorithm in [9] is

still computationally intensive. More specifically, the code

design problem belongs to the class of semi-infinite program-

ming (SIP) which cannot be solved analytically or numerically

directly. Quantization is a prevailing method to solve the

problem, for example, [3], [9]. In this method, the infinite

feasible set of the optimization problem is quantized into a

finite feasible set, and can be solved by common optimization

methods such as gradient or evolutionary based algorithms.

The conic approach has been proposed to design LDPC

codes in binary erasure channel (BEC) [10], [11]. Taking

advantage of polynomial form of density evolution the infinite

feasible set is transformed into a semi-definite constraint.

However, this method cannot be directly applied to AWGN

relay channel as the density evolution is a complicated 2-D

function.

In this letter, we intend to design bilayer expurgated LDPC

codes under the Gaussian approximation proposed in [9] by

extending the idea suggested in [10] and [11] in order to

reduce the computational complexity of code optimization

problem. The objective is to maximize the code rate such

that a successful decoding is guaranteed. The code design

optimization problem consists of infinite number of constraints

in two dimensions and finite number of optimization variables

which forms a SIP problem. Our approach is to approximate

the density evolution relations in lower and upper codes with

polynomials, as polynomial optimization problems (POPs)

have had great development during the recent years. Then,

utilizing optimization problems over SOS polynomials we

translate the problem into SDP which can be effectively solved

by interior point method.

II. BILAYER EXPURGATED LDPC CODES PRELIMINARIES

We briefly introduce bilayer expurgated LDPC codes for

the relay channel. Consider a half-duplex relaying channel

where a pair of source and destination communicate with the

help of a relay. In the first stage, the source transmits its

message encoded by a high rate code (n, n − k1) which is

decodable at the relay to the relay and the destination. The

relay decodes the received message and adds extra nonzero

parity bits and encodes the message by code (n, n − k1 − k2)

and sends it in the second stage. The destination first decodes

the extra parity bits and then using the obtained information

decodes the source message. Therefore, the relay coding

strategy consists of two codes, such that one is a subset of

the other.

Like conventional LDPC codes, an ensemble of bilayer-

expurgated LDPC codes is described based on degree distri-

butions. Two sets of check nodes, i.e., lower and upper, and

one set of variable nodes form the graph of bilayer-expurgated

code. A lower (upper) edge is an edge connecting a variable

node to a lower (upper) check node. The lower (upper) degree

of a variable node is defined as the number of lower (upper)

edges connected to it. The degree distribution of a variable

node is λi, j if it has lower degree i and upper degree j .

V. CONCLUSION
We developed an efficient optimization approach to design bilayer expurgated LDPC codes. The original problem was SIP and computationally intractable to solve. The complicated density evolutions were approximated by polynomials. Using POPs techniques, we turned the resultant problem into SDP which can be efficiently solved. Simulation results confirmed that the proposed method has a significantly lower computational complexity compared with the quantization method.

The solution of problem (17) is a lower bound for problem (9) because the nonnegativity of constraint (9b) is relaxed to SOS. Fortunately, we found that gi, j (x1, x 2) and hi, j (x1, x 2) can be well-approximated by polynomials with degree 4 which satisfies the condition (10). Hence, the solutions of (17) and (9) are exactly the same.
IV. NUMERICAL RESULTS In this section, the performance of our code design method is evaluated by numerical results. The CVX package is used to solve problems (17) and (7a) . We set α = γ = 10−6 and β = γ = 0.5. j is assumed with maximum of 2. dc, dc and η are set 13, 4, 0.7803, respectively. M and k respectively are considered to be 8 and 4 for which the density evolution relations can be approximated with a small error. The transmission power for all transmission nodes is set as 0d B, and the noise variances for all receiving nodes are set as σn2. In addition, the source power division factor is set as 1 which is optimal for our scenario, [6]. All simulations are run with MATLAB version 2013a(8.1.0.604) on 64bit operating system with processor Intel(R) Core(TM) i5-2410M CPU @2.30GHz.
