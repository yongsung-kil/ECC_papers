Generic Architectures for Uniformly Reweighted APP Decoders
Velimir Ilic¬ī1, Elsa Dupraz2, Bane Vasic¬ī3

Abstract‚ÄĒWe propose generic architectures for memoryefficient Uniformly reweighted a posteriori probability (URAPP) Decoders of Low Density Parity Check (LDPC) codes, with the bit error rate close to Belief propagation decoding. The architectures enables a trade off between the throughput, computational and hardware resources. As a special case, we derive previously proposed flooding schedule URAPP decoder and propose new type of the URAPP decoder which uses shuffled scheduling.
We conduct a rigorous complexity analysis and provide new formulas for the URAPP decoder throughput in terms of hardware resources and memory requirements. The analysis confirms that the shuffled decoder represents an optimal solution if the hardware resources are limited, which makes it attractive in practical applications. The results are verified by simulations for the case of finite geometry LDPC codes.
Keywords - LDPC, APP decoding, complexity, memory, throughput, architecture
I. INTRODUCTION
Belief propagation (BP) is an iterative message-passing algorithm widely used for decoding Low Density Parity Check (LDPC) codes [1]. Despite its good error correction performance and capability of approaching the Shannon limit, BP suffers from large memory requirements for message processing and storage, proportional to the number of edges in the Tanner graph of the code [2]. Such large memory requirements, coupled with additional hardware resources needed for the message updating, make the BP less attractive in practical applications.
Uniformly reweighted a posteriori probability (URAPP) decoder is a suboptimal alternative to BP, with the performances close to the BP [3], [4]. In URAPP the variable node processing is simplified so that a message from a variable node corresponds to a posteriori value used to estimate that variable. In the URAPP decoder, the messages are thus sent in an intrinsic manner, which results in significant memory savings. In this paper we study the complexity of the URAPP decoder and provide the descriptions of generic architectures for the URAPP decoder. As a special case, we obtain new type

First we present a rigorous analysis of the decoders complexity. The following criteria are considered: 1) Memory requirements for the storage of intermediate results used in the decoding process, 2) Number of processors used for the decoder implementation, 3) Computational complexity defined as the total number of operations needed to perform the decoding 4) Decoder throughput defined as the number of information bits per time used for the decoding. Previously, the memory complexity was analyzed in [5], but only for a special type of URAPP decoder. Moreover, the exact expression of the memory complexity was not derived, but only an estimation was provided. Here, we derive the exact expressions of the memory requirements, number of processors, and computational complexity for the URAPP decoders which are general and applicable for the analysis of the generic architectures. We provide new formula which relates the throughput to the number of processors. The results presented in [6] are a special case of our expressions.
The order in which the messages are sent during the decoding process affects the decoding convergence speed and its performances, and is defined by the message passing schedule. In the case of BP algorithm, two types of the schedules has previously been considered, flooding [6] and shuffled [7] schedulings. On the other hand, the previous work on URAPP decoder includes only flooding schedules (flooding URAPP). In this paper we propose shuffled URAPP. The simulations on finite geometry codes verify the convergence speed advantage of the shuffled URAPP. In addition, the shuffled URAPP has lower memory requirements and, if the hardware resources are limited, higher throughput than the flooding URAPP.
The proposed generic architectures for URAPP decoders are compatible with both types of scheduling. They enable different configurations of URAPP decoders which depend on the number of used processors. We define two configurations: P configuration, which uses maximal possible number of processors, and S configuration, which uses minimal possible number of processors. In this way we specify four possible decoders: P-flooding, S-flooding, P-shuffled and S-shuffled decoders. Conducted analyzes and simulation results for finite geometry codes shows that the S-shuffled decoder represents an optimal solution if the hardware resources are limited, which makes it attractive in practical applications.

VII. CONCLUSION
Generic architectures for memory-efficient APP decoding of LDPC codes, with the bit error rate close to Belief propagation decoding are proposed. The architectures enables a trade off between the throughput, computational and hardware resources. As a special case, we derive previously proposed flooding decoder and propose new type of URAPP decoder which uses shuffled scheduling. Depending on the number of processors used for the decoder implementation we introduced P- and S-configurations for both of the decoders. In this way we analyzed four different cases, which include previously considered flooding decoder with P-configuration.
A rigorous complexity analysis is conducted and new formulas for URAPP decoder throughput are proposed in terms of hardware resources and memory requirements. It turned out that the S-shuffled decoder represents an optimal solution if the hardware resources are limited, which makes it attractive in practical applications. The results are verified by simulations for the case of P Gp2, sq LDPC codes.
