Abstract—It is known that the Gradient descent bit flipping (GDBF) algorithm is an effective hard-decision decoding algorithm for low-density parity-check (LDPC) codes. However, trapping in a local maximum limits its error-rate performance. This paper presents a modified GDBF scheme that can mitigate the trapping problem and hence can improve the error-rate performance. Compared to the conventional GDBF algorithm, the proposed method is able to improve the decoding performance of 0.3dB for an (18582, 16626) code. The (18582, 16626) LDPC decoder integrates 636k logic gates and achieves a throughput of 12.4 Gbps at a clock frequency of 200 MHz in a 90nm process.
Index Terms—LDPC codes, hard-decision decoding, bitflipping algorithm, gradient descent bit-flipping (GDBF) algorithm.
I. INTRODUCTION
L DPC codes have been adopted in many communication standards owing to their powerful error correcting capability. The soft-decision decoding algorithm such as the MinSum algorithm (MSA) can achieve excellent error-rate performance with practically-implementable hardware architectures [1][2]. However, for applications such as NAND flash or optical communications, soft information is hard to be obtained. As a result, the hard-decision decoding algorithms such as the bit-flipping (BF) algorithm [3] can be used, where both the single-bit flipping and multi-bit flipping were investigated. However, the bit error rate (BER) performance of the BF decoding algorithm is worse compared to the MSA decoding. Consequently, reducing the performance gap between the BF decoding and the MSA decoding is important.
In order to enhance the BER performance, the weight BF (WBF) algorithm [4] and the modified weight BF (MWBF) algorithm [5] were proposed. The authors in [6] proposed the gradient descent bit flipping (GDBF) algorithm to enhance the error-rate performance further, where a simple gradient descent formulation based on the maximum likelihood decoding was derived. In order to avoid to be trapped in a local maximum of the objective function, the authors in [6] also proposed a multi-

In this paper, a modified GDBF algorithm is proposed, where the positions of flipped bits are recorded in each iteration. When the number of occurrences of the oscillation behavior is higher than a threshold, the flipped-positions which were recorded previously are not allowed to be flipped again. Since only a few positions are recoded, the related increase in decoder complexity is low. Compared to the conventional GDBF algorithm, the proposed method is able to improve the decoding performance of 0.3dB for the (18582, 16626) code used in [2]. The (18582, 16626) LDPC decoder integrates 636k logic gates and achieves a throughput of 12.4 Gbps at a clock frequency of 200 MHz in a 90nm process.

III. A MODIFIED GDBF SCHEME
A. The Modified GDBF Algorithm
In the GDBF algorithm, the decoded vector is likely to be trapped in a local maximum and hence the error-rate performance is limited. We modify the GDBF algorithm such that the P bit positions that have been flipped are not allowed to be flipped again when the oscillation occurs. In the multibit GDBF mode, the decoder will flip the bits which have smaller inverse function values than the threshold. In addition, the P flipped positions will be recorded. When the oscillation happens, we lower the threshold by a factor λ, 0 < λ < 1. The decoder uses parameter χ to record the number of times when f (xt+1) ≤ f (xt) happens. If χ is over a pre-determined threshold ξ, the decoder will start the flipping-restriction mode, such that the decoder can escape from the local maximum successfully and reach the real maximum point eventually as shown in Fig. 2. The proposed modified GDBF algorithm is described in Algorithm 2.
Fig. 4 shows the BER results for the rate-0.89 (18582, 16626) LDPC code from [2] using the proposed algorithm, where different values of P are considered. According to the simulation results, an appropriate value of parameter P is 4. Based on the results from the Osc. detector, the modified GDBF decoder determines to execute either the multi-bit or the flipping-restriction mode and determines whether the P positions of the flipped bits should be ignored or not. The positions of flipping bits are determined by a threshold detector in the multi-bit flipping mode.
The position selector is used to find and record P positions, which are not allowed to be flipped again in the flippingrestriction mode. According to Fig. 4, when the value of P is equal to 4, it will lead to a better performance. As a result, P = 4 is used. As shown in Fig. 6, the position

selector contains a position memory and 4 position finders. We divide the n-bit vector from the threshold detector into 4 sub-vectors equally, and use the 4 position finders to locate the 4 positions, which are not allowed to be flipped again in the flipping-restriction mode. The results from the 4 finders will be combined into a new n-bit vector which is then stored into the position memory. The concept of carry-in bit operation is used in the implementation of the position finders. Each of the position finders keeps only a single ‘1’ from its input vector, and set the other bits to ‘0’. For example, here is a 5-bit binary input vector 10010, we first invert the vector, and then add 1 to it, which results in 01110. In the last step, we preform the AND operation with the input vector. Finally, the result of position finder is 00010. It can be seen that the resultant
