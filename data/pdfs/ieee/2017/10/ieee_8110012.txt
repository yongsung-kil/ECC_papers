Reduced Complexity ADMM-based Schedules For LP Decoding Of LDPC Convolutional Codes
Hayfa BEN THAMEUR1, Bertrand LE GAL2, Nadia KHOUJA1, Fethi TLILI1, Christophe JEGO2 1 SUPCOM, GRESCOM Lab, University of Carthage, Tunisia.
2 Bordeaux INP, CNRS IMS, UMR 5218, Bordeaux University, France.

Abstract—The ADMM based linear programming (LP) technique shows interesting error correction performance when decoding binary LDPC block codes. Nonetheless, it’s applicability to decode LDPC convolutional codes (LDPC-CC) has not been yet investigated. In this paper, a first flooding based formulation of the ADMM-LP for decoding LDPC-CCs is described. In addition, reduced complexity decoding schedules to lessen the storage requirements and improve the convergence speed of an ADMM-LP based LDPC-CC decoder without significant loss in error correction performances are proposed and assessed from an algorithmic and computational/memory complexity perspectives.
I. INTRODUCTION
Since their discovery in 1999 [1], LDPC-CCs have gained much attention in the error-correcting area. They have a great potential to become a key forward error-correcting code for improving reliability of modern digital communication systems and optical systems. The interest of LDPC-CCs stems from their channel universality property, their capacity to support versatile application scenarios and their flexibility to code frames of arbitrary size using the same code structure [2] which is not afforded by polar codes and LDPC block codes.
Recently, a novel approach based on linear programming (LP) was proposed for decoding binary LDPC block codes (LDPC-BC). The LP technique was first introduced by Feldman and al in [3] as a prominent alternative to solve the problem of decoding an error-correcting code. Later, an attractive relaxation of the original LP problem was stated by Barman and al in [4]. In fact, it relies on the Alternate Direction Method of Multipliers (ADMM) method [5]. Not only it reduces the LP complexity issues, the ADMM-LP technique can behave as a message passing (MP) algorithm that can be iteratively processed similar to the conventional Belief Propagation (BP) methods [4]. Multiple algorithmic optimizations of the ADMM-LP approach were recently proposed in literature. They aim to improve the error correction performance [6]–[8] and alleviate the decoding computational complexity and the execution time constraints [4], [9]–[13].
Generally, for MP decoders, the order of messages updates is important to improve the convergence speed. In this paper, a description of a flooding based ADMM-LP algorithm for decoding LDPC-CCs (ADMM-CC) is initially detailed. Then, two optimization ideas based on reordering message updates are proposed to make the ADMM-CC decoding more interesting for software and hardware implementations. Firstly, an On Demand Variable Node Activation (OVA) [14] formulation of the ADMM-CC algorithm is introduced. The underlined

approach is developed to accelerate the convergence speed of the flooding based ADMM-CC algorithm while keeping the same memory footprint. Secondly, an optimized formulation of the OVA based ADMM-CC schedule is detailed, termed as the Horizontal Layered (HL) ADMM-CC schedule. It yields the same convergence speed as the former schedule. Meanwhile, a reduction of memory requirements by almost 12.5% is reached for all possible code rates of the IEEE 1901 LDPC-CC.
The remainder of this paper is organized as follows: Conventional decoding of LDPC-CCs is reviewed in Section II. The ADMM formulation for decoding LDPC-CC codes is summarized in Section III. In Section IV, a detailed algorithmic description of the proposed OVA and HL based ADMMCC algorithms is provided. A complexity analysis of the three ADMM-CC scheduling schemes is also provided in Section IV. Ultimately, bit error ratio (BER) experimental results are reported in Section V.

B. Conventional BP-based Decoding of LDPC-CC Codes
Similar to LDPC-BCs, LDPC-CCs can be decoded with MP algorithms. The BP is the most efficient one in terms of error correction performance. However, it suffers from high computational complexity due to the costly tanh operations. A straightforward alternative to the BP is its Min-Sum (MS) approximation. Despite the resulting complexity reduction, a significant loss in terms of error-ratio is observed for the MS. It can be partially recovered by considering several MS variants namely the Normalized MS (NMS) and the Offset MS (OMS) [17]. In the next section, the application of a novel approach for decoding LDPC-CCs based on the LP algorithm is detailed.

B. Flooding based ADMM-LP Formulation for Decoding LDPC-CC
The conventional MP schedule for decoding LDPC-CCs is the flooding (FL) schedule [1] where a decoding iteration corresponds to a simultaneous update of all VNs followed by a concurrent processing of all CNs. Given that the ADMM algorithm can be reformulated as a MP algorithm for LDPCBCs [4], the application of such LP approach for decoding LDPC-CCs seems intuitively possible. Hitherto, in the following, a first detailed description of the ADMM based flooding decoding approach of LDPC-CCs is provided.

Graphically, messages are exchanged along the Tanner graph edges between connected VNs and CNs. Cfl = Imax × (M × O(CNfl) + N × O(V Nfl)), (3)

The memory complexity of the flooding scheduling can be computed as: Mfl = 2× (N + m). Indeed, during the decoding process, N channel LLRs (γi), N a-posterior messages (Lvi ), m z-replica and m λ exchanged messages are stored for computations.

IV. REDUCED COMPLEXITY ADMM-CC SCHEDULES
According to the ADMM flooding schedule, at each iteration, all VNs are activated just before CNs processing. Hence, the incoming information to a VN (z, λ) are included in its outgoing message Lv only after an iteration is accomplished. To overcome such inefficiency, a first modified schedule is proposed in this paper, named as the OVA schedule. Moreover, a memory optimized version of the OVA technique is also

introduced for the ADMM algorithm for decoding LDPC-CCs that bears resemblance to the BP schedule in [19].
A. The On-Demand Variable Node Scheduling for the ADMMCC Algorithm
Inspired by the OVA scheduling for the BP decoding of LDPC-CCs that was introduced in [14], the computation scheduling of the ADMM based LDPC-CC decoding is therein transformed accordingly. The main idea is to change the order of VN and CN updates. Algorithm 2 depicts a formalization of the OVA schedule for the ADMM-LP method.
Basically, compared to Algorithm 1, in the OVA algorithm, VNU and CNU steps are merged in a single one (Algorithm 2, Step 2). Each CN represents one layer. In fact, in the jth layer, j ∈ J , outgoing messages from the check node j (Algorithm 2, Step 2.2) are computed based on their immediately updated VN neighbors during the Pre-VNU Step (Algorithm 2, Step 2.1). As soon as a CN to VN message ( L(jk→) i, i ∈ Nv(j)) is counted, it is directly added to the intermediate message ψi (Algorithm 2, line 15). ψi is the sum of incoming messages to the VN i. Meanwhile, the information brought to the VN i by the CN j in the previous iteration (Lj(k→−i1)) is subtracted. As a prominent consequence, during the Post-VNU step, the activated VN neighbors get a part of their output message ( L(vki)) updated which helps spreading reliable information to the next layers. Attractively, this property would result in speeding-up the decoding convergence of an ADMM based LDPC-CC decoder. Note that layers are executed sequentially.
This modification of nodes’ updating order impacts the maximum computational complexity of the OVA based ADMMCC algorithm. Denoted as Cova, it can be approximated by:
Cova = Imax × M × (O(CNova) + dc × O(V Nova)), (4)

B. The Horizontal Layered Scheduling for the ADMM-CC Algorithm
Although the decoding convergence of an ADMM-based LDPC-CC decoder can be further speeded-up, thanks to its OVA version, the memory requirements is still high. In this sub-section, a reduced memory footprint ”CN centric” scheduling for the ADMM-CC algorithm is introduced.
Formulated as in Algorithm 3, memory savings come from the elimination of ψ messages while keeping the same VN/CN update’s order as for the OVA schedule. In fact, computation operations executed within the Pre-VNU and Post-VNU steps are modified in such a way that updated z-replica and λ messages are added to Lv LLRs directly (Algorithm 3, line 16). The projection of Lv messages onto [0,1] is said to be performed on the fly because, unlike previous schedulings (Algorithm 1, line 8 & Algorithm 2, line 9), they are not supposed to be kept in memory during the decoding process. Instead, each Lv LLR contains only the value of the a-posterior message (expressed as δi × ψi in Algorithm 2, line 9). Yet, the projection is needed to be done only if its related VN is requested by the active constraint node. Thereby, similar to the ADMM based OVA schedule, at the end of each layer processing, the VN neighbors of the activated CN are partially updated. Using this ADMM based approach, the following CNs are able to access the most recent information available from its connected VNs.
Theoretically, the maximum computational complexity of the ADMM based HL decoding algorithm is the same as for the OVA approach. Denoted as Chlayered, it roughly equals:
Chlayered = Imax × M × (O(CNhl) + dc × O(V Nhl)). (5)

In order to pinpoint the impact of the convergence speed on the error correction performance, the BER evolution of discussed schedules depending on the MDI is described in Fig. 3. Although the OVA exhibits similar performances to the HL, it seems that the HL is the best trade-off between considering complexity reduction, fast convergence and error correction performance. VI. CONCLUSION
In this paper, a first formulation of decoding T-LDPCCC based on the ADMM-LP approach was investigated. Besides, an algorithmic description of two iterative ADMMCC schedulings were proposed (OVA and HL schedules). The computational/memory complexity of discussed algorithms and their error correction performances were provided. In fact, a memory saving of about 12.5% was achieved by the HL based ADMM-CC applied to the T-IEEE 1901 LDPC-CC. Furthermore, it was found that the proposed schedules are 1.6 times faster than the FL one. Thus, they can be considered as a promising alternatives for futur software and hardware simplifications for decoding LDPC-CCs.
