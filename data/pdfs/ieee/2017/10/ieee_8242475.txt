Memory Compact High-Speed QC-LDPC Decoder

Tianjiao Xie School of Electronics and Information Northwestern Polytechnical University China Academy of Space Technology
Xi’an, China Email: xiexietianjiao@163.com

Bo Li, Mao Yang* and Zhongjiang Yan School of Electronics and Information Northwestern Polytechnical University Xi’an, China Email: libo.npu@nwpu.edu.cn yangmao@nwpu.edu.cn zhjyan@nwpu.edu.cn

Abstract—In this paper, compact memory strategies for partially parallel Quasi-cyclic LDPC (QC-LDPC) decoder architecture are proposed. By compacting several adjacent rows hard decisions and extrinsic messages into one memory entry, which not only reduces the number of memory banks for hard decisions, but also facilitates multiple data accesses per clock cycle, the throughput of the decoder is increased. We demonstrate significant benefits of using the proposed techniques with an FPGA implementation of a CCSDS LDPC decoder on Xilinx XC5VLX330 device. It shows that our decoder can operate at a maximum frequency of 250 MHz after place and route and achieve a throughput up to 2 Gbps at 14 iterations.
Index Terms—High-Speed, QC-LDPC, decoder, memory, FPGA, CCSDS.
I. INTRODUCTION
Recently, LDPC codes [1] have attracted a lot of attention due to their near Shannon limit performance and inherently parallelizable decoding scheme. Being a special class of LDPC codes, QC-LDPC codes, are not only achieve comparable performance to random codes [2], but can also be conveniently encoded with shift registers [3] and efficiently decoded with partly parallel decoder architectures [5]-[9]. A wide array of the latest communication and storage systems have chosen QC-LDPC codes for forward error correction in applications including Digital Video Broadcasting (DVB-S2), China Mobile Multimedia Broadcasting (CMMB), broadband wireless access (WiMax), wireless local area network (WiFi), space communications (CCSDS) and so on.
Therefore, many investigators studied the design of QCLDPC decoders. In particular, memory-bank-based partly parallel decoder architectures for QC-LDPC codes (see, e.g., [5][9]) have been intensively studied due to their property of a good tradeoff between hardware complexity and decoding throughput. However, in general, LDPC codes achieve outstanding performance only with large code word lengths (e.g., several thousand bits). Thus, the memory normally dominates the overall hardware, which significantly limits the performance of a decoder. To reduce memories without sacrificing error performance, various efficient architectures have been proposed [5]-[9].
However, these approaches need additional memory banks to store the hard decisions. In general, hard decisions memory banks occupy large ratio of the total number of the decoder

memory banks. Distinctly, we proposed a method of hard decisions sharing intrinsic and extrinsic memory banks for any QC-LDPC decoder with high parallelism. The proposed compact memory strategies not only employ significantly fewer memory banks, but also guarantee the high throughput of decoders.

B. High-speed QC-LDPC Decoder Architecture
Let us consider a regular QC-LDPC code, whose parity check matrix is formed by an array of m × n weight-1 shifted cyclically identity-matrix of size L × L. Typical partially parallel decoder architecture similar to those in [5-6] is shown in Fig. 1. Parallel degree is either the number of block rows m or the number of block columns n. In other words, m rows (n columns) from different block rows (columns) are updated simultaneously in the CNU (VNU) phase. All rows (columns) of each block are updated serially in L clock cycles. In Fig. 1, m CNUs with n-inputs, n VNUs with (m + 1)-inputs and m (PCUs) with n-inputs are adopted for processing. Here PCU (the decision and Parity Equation Check Unit) is introduced for parity check computation and tentative decision.
The architecture also requires m×n RAMM memory banks Ms,t(0 ≤ s ≤ m − 1, 0 ≤ t ≤ n − 1) for storing extrinsic messages, n RAMF memory banks Ft(0 ≤ t ≤ n − 1) for intrinsic messages from channel, and m × n RAMC memory banks Cs,t(0 ≤ s ≤ m − 1, 0 ≤ t ≤ n − 1) for hard decisions. Each unit of the memory banks for hard decisions has exactly one bit, while the wordlength Q bits for intrinsic and extrinsic messages depends on quantization schemes. In order to avoid data access conflicts, all memory banks are dual-port, one port is readable and the other is writable. Each memory bank contains L storage entries.
For QC-LDPC codes, the address generator for each RAMM memory bank can be simplified. In the VNU phase, the memory address generator successively generates 0, 1, . . . , L−1. In the CNU and PCU phase, the address generator sequentially outputs βs,t, βs,t + 1, . . . , 0, 1, . . . , βs,t − 1. Here, βs,t is the cyclic shift (or offset) value. Hence, the address generator can be implemented as a simple modulo-L counter. Therefore there is one modulo-L counter associated with each memory bank of RAMM to generate the corresponding memory address.
To improve the parallel degree and increase the throughput of decoder, we enable each node processing unit to process

the data corresponding to 1-components at multiple rows (or columns) of the parity check matrix at the same cycle. To facilitate multiple data accesses per clock cycle in a p-parallel (p > 1) architecture, the data in p adjacent rows or columns are stored into one memory entry. In general, barrel shifter and buffer or called network [5] are introduced to resolve the data access conflicts.
III. PROPOSED COMPACT MEMORY STRATEGIES
A. Hard Decisions and Extrinsic Memory Sharing Strategy
PCU is utilized for parity check equations. Clearly, PCUs run concurrently with CNUs because when VNUs generate the extrinsic messages for CNUs, they also generate the hard decisions needed by parity checks. In order to match parallel degree of CNUs and VNUs for partly parallel architecture, Similar to the CNUs, the m PCUs check the parity of the m block rows simultaneously and PCUs check the parity check equations in the same block row sequentially. Each PCUs needs n hard decisions, so m × n hard decisions should be read from memory banks in one clock period. In order to avoid memory access conflicts, as described above, a straightforward implementation [5] needs m × n memory banks Cs,t(0 ≤ s ≤ m − 1, 0 ≤ t ≤ n − 1) each entry with 1 bits, to store the hard decisions. In Xilinx FPGA, Each memory bank requires one 18 Kbits Block RAM. So the total memory banks for decoder is 2m × n + n.
In previous researches [6], reducing memory banks for hard decisions is considered. In PCU phase, it can be observed that hard decisions memory banks are only in read mode. Since dual-port are used to read, the memory banks for hard decisions can be reduced nearly twice, i.e., m/2 × n are required for hard decisions. This improved method can reduce memory banks to m × n + m/2 + n.
Another method employs registers of big length to store the hard decisions, which not only leads to more slices resources but also causes high critical path delay. It is unfavourable to high speed implementations.
However, we observed when CNU and PCU operates simultaneously, the corresponding m×n extrinsic messages (exactly is variable to check messages) and m × n hard decisions are required by reading from Ms,t and Cs,t, respectively. Meanwhile, it is crucial to note that extrinsic messages for CNU and hard decisions for PCU corresponding to the same row. So the data required by CNU and PCU come from the same memory access addresses, i.e., βs,t/p , βs,t/p + 1, . . . , 0, 1, . . . , βs,t/p − 1.
From the observation, distinction form extrinsic messages and hard decisions stored in separated memory banks, we package the two kinds of data in one memory entry and due to the same addresses so as to access without memory access conflict. From merged memory bank, we can get hard decisions and extrinsic messages simultaneously. B. Hard Decisions Memory Compact Strategy
However, the above hard decisions and extrinsic sharing strategy leads to a problem. When the check equations are satisfied or a preset maximum iteration number is reached, the hard decisions are as a finally decode results of last input frame, which will be outputted. Whereas, two ports of RAMMC are always simultaneously taken up by the input and output of CNU (PCU) or VNU. If the final output (hard decision) is extracted from RAMMC, memory access conflicts would be caused. In order to solve this problem, in VNU phase, at the same time as the outputs are written to RAMMC, n hard decisions are also written to RAMC memory banks. The memory bank RAMC only read by output are defined as C, which adopt a compact storage strategy. However, difference from RAMMC packaged extrinsic messages and hard decisions, RAMC package n hard decisions into one memory entry. As demonstrated in Fig. 3. In output phase, form memory bank C we can get n hard decisions simultaneously. All the codeword (hard decisions) can be outputted by sequentially selecting n hard decisions of each block column. In the case of p-parallel (p > 1), the width of memory banks for C is p × n bits and the depth is L/p . Hence RAMC only require one 18 Kbits BRAM in Xilinx FPGA. In addition to m×n RAMM memory banks, our decoder requires m × n + n + 1 ≈ m × n + n memory banks, could reduce the number of memory of m × n.
IV. CCSDS NEAR-EARTH QC-LDPC DECODER
In this section, we will present an FPGA design example on Xilinx Virtex 5 for the CCSDS Near-Earth (8176, 7154) QCLDPC code using the previously discussed memory compact strategies.

The CCSDS Near-Earth QC-LDPC(8176, 7154) code is a regular QC-LDPC code with a column weight of 4 and a row weight of 32 [10]. The parity-check matrix is a 2×16 array of L × L sub-matrix as shown in the following expression, here L = 511.

Initialization:
Flush the received intrinsic messages to the np RAMF Ft and the corresponding RAMMC M Cs,t is initialized to zeros. The data are stored column-wise in the memory Ft, C and M Cs,t. The VNU start first here since all memory banks M Cs,t = 0. Alternatively, the CNU start first if we initialize M Cs,t = Ft. Initialization iteration number i = 1.
Iterative Processing:
Variable node update (VNU): In VNU phase, two kinds of messages, Qp bits C2V messages and Qp bits intrinsic messages, are sent to VNUs, corresponding from RAMMC and RAMF in Fig. 5, respectively. Consequently, after the VNU computation, the updated Qp bits V2C messages and pbit hard decision form the sign of posterior message are packaged and are written back to the same address as read operations in the order of 0, . . . , L/p -1. VNU takes v + L/p clock cycles.
Check node update and parity checks (CNU-PCU): In our decoder, PCU concurrently proceeds with CNU. Similarly, in CNU-PCU phase, the (Q + 1)p bits compact messages are read out from RAMMC memory banks. Next, two kinds of messages, Q bits V2C messages and 1 bit hard decisions recovered by the combine/separate are sent corresponding to CNU and PCU, respectively. Then, after the VNU computation, the updated Qp bits V2C messages are written back to the same address as read operations in the order of βs,t/p , βs,t/p + 1, . . . , 0, 1, . . . , βs,t/p − 1. CNU-PCU takes c + L/p clock cycles. Meanwhile, the syndrome is calculated by PCU and is sent to stopping rules.
Decision: If preset maximum iteration number is reached

or the stopping rules are satisfied, go out of Iterative Processing and output np bits hard decisions, otherwise, continues Iterative Processing.
In the literature, there are many processing unit implementations [5-10]. Similar to those implementations, to reduce the critical path and increase the clock frequency fclk, some levels of pipeline stages are inserted in CNU and VNU. In this paper, assume that the computations required by CNU and VNU are split into c pipeline stages, and v pipeline stages, respectively. The heavier the row weight dc (column weight dv), the bigger the pipeline stages c of CNU (v of VNU) is implemented in decoders. From the decoding process outlined above, we know it takes (v +c+2 L/p ) clock cycles to complete one iteration of our decoder.
V. IMPLEMENT RESULTS
We now demonstrate the advantage of the proposed memory compact strategies decoder architecture through the use of the CCSDS near earth QC-LDPC decoder. The target implementation device is Xilinx Vertex5 XC5VLX330 FPGA. The synthesis and implementation are carried out using ISE 10.1.03.
For an LDPC decoder, the messages corresponding to row are updated by CNUs. Row weight is 32, so 2p 32-input CNUs are required for the CCSDS decoder. p enables increased parallelism of decoder, Here p is 8 in our decoder. Similarly, the messages corresponding to column are updated by VNUs. Similar column weight is 4, so 32p (4+1)-input VNUs are required for the decoder.
As shown in Fig. 5, our CCSDS decoder includes three kinds of memory banks used to store intrinsic messages RAMF, for extrinsic messages packaged hard decisions RAMMC and for hard decisions RAMC, respectively. In our decoder architectures, the depth of every memory bank is 2L/p , 2L/p depth of intrinsic memory bank enable the decoder to receive channel intrinsic messages while decoding. Similarly 2L/p depth of extrinsic messages enable CNUs and VNUs operate alternately on two different frames of data at the same time. In this way the decoder can do receive intrinsic messages, iteration and output operation at the same time. However, their widths are diversity. The memory banks width of RAMF, RAMMC and RAMC is Q × p bits, (Q + 1)× p bits and n × p bits, respectively. Besides all the memory banks are dual-port memory. Here Q=6 bits quantization scheme is adopted in our decoder. From the above description, 16 memory banks with 48 bits width and 128 depth are used to store intrinsic messages, totally, 16 × 48 × 128=98.304 kbits. 1 memory bank with 128 bits width and 128 depth is for output, i.e., 128 × 128=16.384 kbits. While, the number of memory blocks for extrinsic messages are proportional to the number of non-zero elements in its corresponding base-matrix. The depth is 128, width is 56 and the quantity equals to 64, totally, 64 × 56 × 128=458.752 kbits. Therefore, the entire memory is 98.304+16.384+458.752=573.440kbits. VI. CONCLUSION
For the partially parallel QC-LDPC decoder architecture, internal memory is required to store various messages, including intrinsic messages, extrinsic messages and hard decisions. The three messages are usually stored separately. However, this paper presents two compact memory strategies. The one is combining several adjacent rows extrinsic messages and hard decisions into a wider one memory entry to save memory blocks. The other one is performing hard decisions located in the same adjacent rows written to a single memory entry for avoiding memory access conflicts between processing and outputting. When the proposed compact memory strategies

are implemented for CCSDS near earth LDPC with Xilinx XC5VLX330 FPGA, a throughput of up to 2 Gbps at 14 iterations with a frequency of 250MHz is achieved.
