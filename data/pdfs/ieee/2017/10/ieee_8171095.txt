Construction of Multiple-Rate LDPC Codes Using Modified PEG
Tengfei Chu, Xue-Qin Jiang, Member, IEEE, Jia Hou, Wang Hui-Ming, Senior Member, IEEE, and Lingjun Kong

Abstract—Progressive edge-growth (PEG) algorithm was proven to be a simple and effective approach to design good LDPC codes. However, the low-density parity-check (LDPC) constructed by PEG algorithm is of a single code rate. In this paper, we propose a modified PEG algorithm to construct multiple-rate LDPC codes, which is referred to as the multiplerate PEG (MR-PEG) algorithm. The obtained LDPC codes with various code rates have the same blocklength and can share common hardware resources to reduce the implementation complexity. Simulation results show that the obtained codes outperform other multiple-rate LDPC codes constructed from random row-splitting operations.
Index Terms—Low-density parity-check (LDPC) codes, progressive edge-growth (PEG), multiple-rate PEG (MR-PEG).
I. INTRODUCTION
Low-density parity-check (LDPC) codes are a prominent class of error correcting codes. They were proposed by Gallager in the early 1960s [1] but did not receive much attention until decades later. LDPC codes are the most promising class of linear codes due to their capacity-approaching performance and low decoding complexity [2]. The girth, which is the length of the shortest cycle in the Tanner graph of LDPC codes, will affect the performance of LDPC code. Therefore, most LDPC codes are constructed in order to generate a larger girth [3].
Practical communication systems have to support various code rates for different user scenarios, with the same encoder/decoder pair shared for the whole set of codes, which is referred to as rate-compatibility [4]-[7]. The design of multiple-rate LDPC codes has been widely investigated using code modifying techniques such, as puncturing, extending, row combining and splitting. Puncturing is a way to obtain high-rate codes by dropping some parity bits [8]. It must be optimized carefully to guarantee good performance. Extending is also an efficient method for constructing RC codes, which starts with a high-rate mother code and then produces additional parity symbols to yield low-rate codes [9]. However, code lengthening changes the information block size. Row combining and splitting are effect ways to construct multiple-rate LDPC codes [10]. The multiple-rate LDPC codes constructed by row-splitting operation can share common hardware resources to reduce the hardware implementation complexity. But, they can not guarantee each row of the obtain matrix has the same degree, you must add or delete some ‘1’components to match the optimized degree distribution.

Progressive edge-growth (PEG) algorithm is a general method for constructing Tanner graphs of LDPC codes having a large girth by establishing edges or connections between symbol and check nodes in an edge-by-edge manner [11]. It has been shown to be a very efficient method for constructing regular and irregular LDPC codes. Its low complexity makes it suitable for constructing codes of very large lengths and, with a slight modification, for constructing linear-time-encoding LDPC codes. However the LDPC codes constructed by PEG algorithm is of a single code rate. In this paper, we combine the PEG algorithm and the row-splitting operation, and propose a multiple-rate PEG (MR-PEG) algorithm to construct multiplerate LDPC codes.
This paper is organized as follows. Section II describes the progressive edge-growth (PEG) algorithm, row splitting and combining operations. Section III introduces the MRPEG algorithm for multiple-rate LDPC code and. Section IV provides the simulation results of the LDPC codes constructed by the MR-PEG algorithm. Section V concludes the paper.
II. PRELIMINARIES
This section describes some background concepts that will be used throughout the paper.
A. Progressive Edge-Growth Tanner Graphs
Consider a binary LDPC code C represented by a Tanner graph G = {Vb ∪ Vc, E}, where Vb ={v1, v2, . . . , vn}, Vc ={c1, c2, . . . , cm} are the sets of variable nodes and check nodes, respectively, and E is the set of edges. In the Tanner graph G, two nodes vj and ci are adjacent if {vj, ci} ∈ E. Denote the variable degree sequence by Ds = {ds0 , ds1 , . . . , dsn−1}, where dsj is the degree of variable node sj, 0 ≤ j ≤ n − 1, in nondecreasing order, i.e., ds0 ≤ ds1 ≤ . . . ≤ dsn−1, denote the parity-check degree sequence by Dc = {dc0 , dc1 , . . . , dcm−1}, where dci is the degree of check node ci, 0 ≤ i ≤ m − 1, in nondecreasing order, i.e., dc0 ≤ dc1 ≤ . . . ≤ dcm−1. Fig. 1 shows an example of a Ds = {2, 2, 2, 3, 3} tanner graph, in which the check degree sequence is uniformly of degree 3, i.e., Dc = {3, 3, 3, 3}.
In the PEG tanner graph, for a given symbol node sj, define its neighborhood within depth l, Nslj as the set consisting of all check nodes reached by a subgraph (or a tree) spreading from symbol node sj within depth l. Its complementary set, N¯slj is defined as Vc/Nslj , or equivalently Nslj ∪ N¯slj = Vc.

B. Row Combining and Splitting Operation
The basic idea of row-splitting is to replace each single row of a parity-check matrix with several rows. The basic idea of row combining is to generate higher-rate codes from a lowerrate mother code by linearly combining the rows of mother code. We compare the bit error rate (BER) performance of MR-PEG codes over the additive white gaussian noise (AWGN) channel with BPSK modulation, respectively, the sum-product algorithm is adopted at the LDPC decoder and the maximum iteration number is set to be 50. A series of multiple-rate LDPC codes with target rates 0.5, 0.625, 0.75 and 0.875, code length 1024, 2048 are constructed. The corresponding codes with the same rates which are random row splitting, and code length 1024, 2048. They are employed as a comparison in Fig. 3, 4, 5 and 6 . As illustrated in Fig. 3 where L=2, Ds = 2, Ro = 0.75, R = 0.5 and N = 1204, at BER = 10−3, the obtained MR-PEG codes achieve approximately 0.25 dB performance gain against random rowsplitting codes for the rate 0.5. We have presented an improved algorithm called MRPEG algorithm for constructing multiple-rate LDPC codes. In the process of PEG construction, we have also constructed an index matrix P based on which we can split the highrate mother code to yield low-rate codes. Each row of the index matrix records the edges should be split into several rows. In comparison to the conventional random row-splitting based multiple-rate LDPC codes construction method, the MRPEG codes have large girth and wide range of code rates. Simulation results demonstrated that the BER performance of the proposed MR-PEG codes were better than that of LDPC codes constructed by random row-splitting methods.
