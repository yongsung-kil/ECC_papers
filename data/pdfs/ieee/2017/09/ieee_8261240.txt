A Simple Design for Large Girth LDPC Matrix

Somsak Choomchuay Department of Electronics Engineering, Faculty of Engineering
King Mongkut’s Institute of Technology Ladkrabang Bangkok 10520, Thailand
Email: somsak.ch@kmitl.ac.th

Abstract—This paper describes an alternative method for designing regular quasi-cyclic LDPC codes with large girths. The permutation matrix was designed based on the geometric sequence and prime number field. The design starts with a portion of the full-range matrix with column weight of 3. With a decided code rate, elements in the rows are extended by adding a certain column of the same full-range matrix into the designing matrix. Resulted codes can offer relatively good performance compared to the published literatures.
Keywords—Large girth, LDPC codes, Prime number
I. INTRODUCTION
Although Low Density Parity Check codes or LDPC codes has been invented and introduced since 1960, it was firstly not so popular according to its high complexity compared with BCH and RS codes that have been introduced in the same period. The introduction of Turbo code in 1993 had stimulated many researchers to look into LDPC codes more closer. Turbo codes are a class of error correction codes that come close to approaching Shannon’s limit. The rediscover of LDPC code in 1996 [1] has enhanced its application vastly. LDPC codes are superior over Turbo code according to their lower error floor and better performance in the higher code rate range. This class of codes become standards in several modern communication systems such as digital video broadcasting (DVB) [2], 10GBase-T Ethernet, and part of IEEE 802.11 Wi-Fi standards (optional part in HT, PHY) [3].
LDPC codes are linear block codes. The number of “1” in the large size parity check matrix ( H ) is very few and sparse.
Column weight ( wc ) is defined as a number of “1” in any column. Similarly the row weight ( wr ) is defined as a number
of “1” in any row. For a regular structure codes, one can have
(n − k )wr = nwc and the minimum distance is dmin ≥ wc +1, where n and k are block size and information size
respectively.
Codes’ performance is directly affected by the structure of their parity check matrix. The best design procedure for the best performance codes is still left to be exploited. Elements in the matrix can be determined by using 2 approaches; randomly and systematically (structurally). Several structural designs are proposed, for instance, array codes [4], modified array codes [5], quasi-cyclic codes [6], geometric Euclid codes [7], Projective geometry codes [8], Balance Incomplete Block Design (BIBD) codes [9], and codes based on progressive edge growth (PEG) algorithm [10].

For the design simplicity, this paper is focusing on the quasi-cyclic (QC) structure. Large girth design is taken as a problem. The rest of this paper is organized as follows: The matrix constructions based on geometric sequences are details in section II. The design start with girth 6 and the extended to higher girth as girth 8, 10 and 12. Parameters of designed coded are tabulated. In section III and IV, the evaluation of various designed codes are given in details. Obtained results are also discussed in depth. Finally, the paper is concluded in the last section.

IV. RESULTS
In the simulation, codes were evaluated on Gaussian distribution channel with BPSK modulation scheme. Logarithmic sum product decoders were employed. To save the computation time, only 20 iterations were run. Bit error rate (BER) and frame error rate (FER) were observed.
Shown in Fig. 2., codes are designed with the code rate of 0.5. BIBD (girth 6) class II can offer better performance than class I according to its larger column weight. However, the designed QC LDPC (girth 8) can give 1.25 dB bit error rate better than that of BIBD class II. Likewise frame error rate is also better. As such, the designed QC is better than BIBD even though the QC holds lower column weight.
For the code rate of 0.75, the designed codes are to be compared with those used Tanner’s method [13] and PEG method. None of the code can be constructed with the global girth of 8. Instead, the local girths of 6 and 8 can be obtained

(see also Table VI). As shown in Fig. 3, all code gives similar performance. Although a PEG designed code can yield slightly better performance compared to the proposed one, the proposed QC still better than that proposed by Tanner. Even though all code hold girth 6, but the number of girth 6 are different. By this observation, it can be implied that those numbers reflect the performance of resulted code.
The design method was employed to design a high rate code as the rate of 0.90. The performance of codes seems to deteriorate as the column weight increase. With the same SNR, code with lower column weight yield lower both BER and FER. When taking into account the close look of rate of change, the code with higher column weight can offer the better steepness.

The high minimum distance of any code is one of a desirable feature when the code is designed. The minimum distance depends on both girth and column weight. Codes with the same girth, one with larger column weight can have higher minimum distance. As such, to maintain the minimum distance as girth cannot be made larger, increasing the column weight can be an alternative choice. On the dim side, code with higher column weight holds the bigger size of the parity check matrix. This is not desirable for the low complexity hardware decoder.
