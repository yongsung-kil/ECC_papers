Adaptive Threshold Multi Bit Flipping for Low Density Parity Check Codes

Abstract—In this paper an adaptive threshold for flipping intensity based dynamic weighted multi bit flipping algorithm is proposed to overcome the existing complexities in finding the threshold. The change in the check node and bit node status are also considered here. Improvement in performance is observed using the method of adaptive threshold in place of fixed threshold.

Keywords-Low density parity check (LDPC), flipped bit selection rule (FBS), flipping function (FF), bit flipping (BF).

I. INTRODUCTION
Low density parity check (LDPC) codes comes under linear block codes and it was first presented by Gallager during 1960’s. Besides turbo codes, LDPC also approaches Shannon’s limit (1) [1]. The LDPC codes require a sparse parity ( H ) matrix which means it has less number of 1’s than 0’s [2]. The usage of sparse H matrix reduces the computational complexities. The messages we transmit is converted to codewords having ones and zeros. The LDPC belongs to class of forward error correction codes in which the message is augmented with parity bits to produce codewords.

The wc and wr be count of 1 s in columns and rows of an H matrix with dimension m × n. The matrix is mentioned low density if wc m and wr n . If the number of 1 s in all rows and columns is same then it is regular else irregular LDPC codes [3]. Tanner brought an intelligent graphical illustration for LDPC known as Tanner graph shown in Fig 1. It is a bipartite graph having 2 category of nodes, check (c-nodes) and variable nodes (v-nodes).
The LDPC codes can be decoded using different methods like majority logic (MLG) decoding, bit flipping (BF) decoding and sum product algorithm (SPA) [4]. The MLG decoding is the simplest decoding method whereas SPA has maximum complexity but better error performance. The complexity of BF decoding is low but has inferior performance. In order to improve the performance many variants of the bit flipping algorithm was proposed.

III. DYNAMIC WEIGHTED BIT FLIPPING ALGORITHM
In case of weighted bit flipping algorithms the received soft channel values determine the weights and they are constant all throughout the decoding process. Since these weights cast the decoder’s belief on syndrome calculation and also bit decisions an appropriate revisal of the weights are needed. The FF (En) (3) comprises of mainly two terms, the channel value and checksum reliability that express the decoder’s certainty on a bit flipping decision. All the FF’s defined for BF decoder’s have fixed weights. This does not reflect the dynamic nature in message passing process of the variable nodes. So when the reliabilities change checksums should be given suitable weights. Therefore dynamic weights are generated and the dynamic WBF (DWBF) algorithm [9] was introduced.

B. Estimation of threshold
For finding the threshold a regular H matrix of order 16 x 32 M × N with wc = 4 and wr = 8 was used. The initially given H matrix is modified to give a new Ht using procedure explained in section II to generate codeword of 32 bits. Then BPSK modulation is done to map the codeword into a bipolar sequence. The AWGN noise at various SNR values are added to the codeword.
At the decoder part, y = (y0, y1....yn−1) is the received soft channel values on which hard decision is done and denoted by z = (z0, z1....zn−1). The algorithm is done till the parity check constraints are fulfilled or it completes number of iterations.

1) In order to find the threshold ∆ the FF’s or En was calculated. The threshold was found using a trial and error method. Initially the values of FF’s was sorted in descending order and the difference between the maximum and minimum of FF’s was taken as ∆. Mostly the ∆ found using this approach are larger than the flipping intensity values determined by (19). In essence no flipping is done as Fn ∆.
2) Next the difference between the maximum and second maximum of F F s was taken as ∆. Mostly the ∆ found using this approach are smaller than the flipping intensity values determined by (19). Since random noise gets added to the codeword each time it is transmitted, the prospect of codeword being correctly decoded using above ∆ is only 40% -50%.

3) Later the difference between the maximum and second maximum of F F s multiplied by constant 2 was taken as ∆. This delivered ∆ values such that 30% - 40% it is correctly decoded. When constants greater than 2 was used ∆ value became more than the flipping intensity values. So finally 2 and 3 was combined. If Method 2 does not give a proper decoding Method 3 is used. Combining both the methods there is 50% - 70% chance of correctly decoding the codeword. Fig. 3. BER performance of various multi bit (M − BF ) decoding algorithms as a function of SNR

In this paper, another multi bit flipping decoding algorithm is proposed which uses flipping intensity principle along with adaptive threshold rather than fixed threshold. After each iteration the codeword and the unsatisfied check node status changes. These changes occurred in check nodes and variable nodes are considered and a new threshold ∆ is obtained. It means the dynamic changes in the codeword is adapted in the next iterations. Therefore the usage of adaptive threshold in the place of fixed threshold gave better BER performance but at the cost of an increase in computational complexity.
