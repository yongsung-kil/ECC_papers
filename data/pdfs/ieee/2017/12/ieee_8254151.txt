An Upper Bounding Technique on the Error Floor
Performance of LDPC Codes
Santhosh Kumar, Ravi H. Motwani
Intel Corporation, Santa Clara, CA
AbstractвҖ”Compared to algebraic codes, low-density parity-
check (LDPC) codes have higher coding gains in several configu-
rations. However, unlike algebraic codes whose performance can
be easily determined generally, the block error rate performance
of an LDPC code needs to be obtained by simulations. While
there are analytical methods for some structured LDPC codes
that predict this performance in the low error rate regime, these
provide little insight into the error floor of other (less structured)
LDPC codes. In this article, we present a simple method to
upper bound the error floor of LDPC codes based on a certain
monotonicity assumption on the decoder. This upper bound is
general enough so that under the monotonicity assumption it
is valid for any error-correcting code. This upper bounding
technique is useful when projecting very low block error rates
(such as below 10вҲ’14) for LDPC codes and there is no knowledge
of the trapping sets.
Index TermsвҖ”Decoding radius, error-correcting codes, error
floor, Low-density parity-check codes.
I. INTRODUCTION
Since the re-introduction of LDPC codes [1] in the mid-
1990s [2], [3], they have gained popularity in a wide range
of applications replacing legacy systems based on algebraic
or Turbo codes. Unfortunately, unlike the algebraic codes,
predicting the error rate performance of the LDPC codes
is a difficult task, particularly in the low error rate regime.
Though it is possible to design LDPC codes that operate
near Shannon limit in the moderate error rate regime [4],
these codes often exhibit the error floor behavior where the
slope of the error rate curve in the low error rate regime
degrades significantly compared to the high error rate regime
[5]. This behavior is particularly egregious when the block
length of the codes is small (up to a few thousand bits). This
is not an issue in wireless systems since they only require
moderate error rate from the error control code, but can
still obtain high reliability by implementing efficient feedback
mechanisms such as automatic repeat request (ARQ) schemes
or rateless codes [6], [7]. However, for applications such as
optical communications or storage systems which require high
reliability from the error control code, the error floor issue
precludes the use of LDPC codes.
The degradation of the performance in the error floor
regime is attributed to the so-called trapping sets [5]. These
manifest from the structure of the code, the channel and the
decoder. On the binary erasure channel, the trapping sets
are also known as the stopping sets and they have a simple
combinatorial definition [8]. However, on general channels
such as the binary symmetric channel or the AWGN channel,
their behavior is decoder dependent and hard to analyze [9].
A popular approach in the LDPC codes to remove error floors
in the moderate error rate region is to construct codes based
on graphs with large girth [10], [11]. It is well-known that
the minimum distance of the LDPC code is lower bounded
by a number that grows exponentially with girth [10]. In the
graph codes, while it is possible to ensure a girth of at least 6
for codes of practical interest, larger girths become infeasible
especially when bit-node degrees are large [11].
Techniques to mitigate the error floor of the LDPC codes
include modifying the decoder to eliminate some of the
identified trapping sets [12], [13], [14]. Using outer code to
eliminate low error floors is also explored [12], [15].
Beyond the mitigation of the error floor, it is also useful to
predict the point of onset of this behavior, which is typically
done by the numeration of the dominant trapping sets [5].
Enumeration of dominating trapping sets for certain codes
was done in [16], and for ensembles of codes in [17], [18].
For certain specific quasi-cyclic codes known as array-based
LDPC codes [19], there has been some effort to analytically
characterize the cardinality and size of the minimal trapping
sets [20], [21]. For these codes, a combinatorial description of
the trapping sets known as the absorbing sets was discovered
that accurately predict the performance in the error floor
region. However, array-based LDPC codes do not offer many
choices for the block length and rate, severely restricting the
code design. Unfortunately, these analytical methods are not
applicable for popular constructions of LDPC codes such as
those based on protographs [22], [23].
In this article, we present a simple method to upper
bound the error floor performance of the LDPC codes. When
deriving this upper bound, we assume a certain monotonicity
property on the decoder. This monotonicity can be shown
rigorously for the maximum-likelihood decoder, but it is also
observed in practice for several message passing algorithms
such as the sum-product or the min-sum decoder. With the
monotonicity assumption, the upper bound is general enough
to cover any error correcting code. This upper bounding
technique naturally incorporates operations such as shortening
or puncturing of the code. When there is no knowledge of the
trapping sets of the given LDPC code, our upper bounding
technique provides a quick and useful way to project low error
floors (such as below 10вҲ’14) that is beyond the simulation
range of even fast FPGAs. For systems that have a high code
rate and require very low block error rate, our technique turns
out to be quite useful.

Throughout, we consider binary linear codes and the chan-
nel of interest is BSC(рқ‘қ), which is defined as the binary
symmetric channel (BSC) with crossover probability рқ‘қ. More-
over, we assume that the correctability of a received sequence
depends only on the errors induced by the channel and not on
the transmitted codeword, which holds true for most decoders
of practical interest. Below, we refer to an error pattern as
the binary sequence indicating the errors induced from the
channel BSC(рқ‘қ). A decoder is said to correct a given error
pattern if the output sequence (from the decoder) agrees with
the input codeword in all positions.
Definition 1. A decoder is defined as monotonic if for every
error pattern рқ‘ўthat is uncorrectable, all error patterns рқ‘Ј
satisfying рқ‘ўвүӨрқ‘Јare also uncorrectable.
Intuitively, for a monotonic decoder, if it failed to correct
some error pattern, then adding more errors to that pattern
can never result in a successful decoding.

IV. CONCLUSION
In this article, we provide a simple method to upper bound
the error floor performance of an LDPC code without any
knowledge of the trapping sets. Our technique is based on
the assumption that the decoder behaves monotonically with
respect to the error pattern. It is general enough to handle
various variable-node and check-node degree profiles, and can
also handle shortening and puncturing. For systems that have
a high code rate, the upper bounds turn out to be exact in some
cases. We demonstrate the utility of our upper bounds with
a few numerical examples. It would be interesting to extend
these bounds to other channels such as the AWGN channel
to provide a simple method to extrapolate the error floor.
