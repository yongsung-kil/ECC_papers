An Energy Efficient Multi-rate QC-LDPC Decoder

Michaelraj Kingston ROBERTS
Assistant Professor, Department of TCE BMS College of Engineering Bangalore-19, INDIA king.pane@gmail.com
Abstract: This paper introduces, an energy efficient multirate decoder architecture for quasi-cyclic low-density paritycheck (QC-LDPC) codes. To achieve good error correcting performance with lower complexity, an improved normalized min-sum algorithm with row-merge scheme is adopted. By exploring the properties of the layered decoding scheme, an enhanced QC-LDPC code with overlapped matrix is employed to offer adequate flexibility for parallel degree optimization and convergence speed. This in turn reduces the linear encoding complexity and data correlation problems during the multi-rate operations of the proposed decoder. Furthermore, to avoid memory access conflicts without increasing the hardware resources, eDRAM is utilized. By utilizing eDRAM, the overall memory bandwidth requisite is reduced considerably along with the inter-connect complexity and hardware overhead. This overall multi-rate decoder is designed and implemented using TSMC 65nm GP CMOS Technology to support all the code rates of IEEE 802.16e applications. From the implementation results it is observed that the designed multi-rate decoder attains improved throughput-to-area ratio (TAR) with less power consumption when analysed with other multi-rate decoders
Keywords—Quasi Cyclic-LDPC codes ;Improved Normalized Min-Sum Decoding ; eDRAM; Row merge; Multi-rate decoder
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] are one of a variety of forward error correction codes which are proven to achieve remarkable decoding efficiency virtually near to the Shannon's theoretical bound. In general, LDPC codes are represented by different types of parity-check matrices. However, for the effectual hardware design of LDPC codes, the Quasi-Cyclic (QC-LDPC) codes [2] are often opted over random and shift-LDPC codes. Due to their linear time encodability, localized memory access and other graceful features, QC-LDPC codes are broadly adopted in many practical applications such as IEEE 802.11n and IEEE 802.16e [3].
In recent times, many research works on LDPC decoders were aimed at designing multi-gigabit-per-second multi-rate decoders with reasonable hardware complexity [4]. However, the main problem in designing low complex multi-rate decoder lies in its flexibility to support different code rates and multi-size LDPC codes with varying degree distribution [5]. A multi-rate QC-LDPC decoder compliant to Wi-MAX standard was proposed by Xin-Yn Shih et.al.,

Also, due to the requirement for many processing units and routing wires this decoder has a large silicon area of 8.29 mm2.
Chih-Hao Liu et. al., presented a multi-rate decoder design using shift-routing network [7]. This QC-LDPC decoder was designed compliant to support multiple rates of different standards. The implementation issues such as the critical path delay, routing congestion and hardware complexity were minimized to certain extent compared to [6]. However, to achieve a maximum throughput of 212 Mbps, this decoder consumes about 528 mW of power which is quite high for practical applications. In addition, the analysis of BER performance of the decoder is not done or is unavailable for comparison. In [8], a multi-rate QCLDPC decoder is proposed for mobile Wi-MAX applications. This decoder design is implemented with five different decoding techniques to boost the throughput gain with acceptable power dissipation. From the implementation results it was shown that this Wi-MAX decoder consumes about 282 mW power to achieve a maximum throughput of 287 Mbps. Furthermore, the interconnect complexity issues caused due to the memory bandwidth utilization of the proposed decoder are not analyzed in this research work.
In [9], Ji-HwanYoon, et. al., proposed an improved LDPC decoder using the memory-address remapping techniques. This proposed technique utilizes the 1-portembedded SRAM memories to reduce the memory size along with the overlapped message passing scheme. This decoder is designed compatible with Wi-MAX standard such that it can support all the code rates of that particular standard. Implementation results show that the designed chip can achieve a maximum throughput of 0.1137 Gbps at 100 MHz with a power dissipation of 104 mW and energy efficiency of 91 pJ/bit/iteration. II. PROPOSED MULTI-RATE DECODER ARCHITECTURE In this paper, an energy efficient multi-rate decoder
design which can support all the rates of IEEE 802.16e standard is proposed. In order to support the multi-rate operation an enhanced QC-LDPC code [11] is utilized instead of conventional QC-LDPC code. The overall functional schematic view of the proposed multi-rate QCLDPC decoder is shown in Fig.1.
Fig.1 Proposed multi-rate decoder block diagram
During the initialization phase, the channel output messages and the high precision intrinsic messages are stored in the (log-likelihood ratio) LLR memory is passed on to the input memory bank for further processing. The input memory bank in this work comprises of three

functional blocks to accommodate and process different code rate and code length messages of various Wi-MAX standard. In this work, in order to achieve good decoding performance with the least bit count, the bit precision value is chosen to be 6. The first functional block of the proposed decoder architecture is the channel value memory which is used to store the 6-bit precision value. In second functional block stores the overall check node sum in its memory. In this block, the data is stored by using the compression method [12] the first minimum and second minimum messages along with its magnitudes are stored along with the index of minimum magnitudes and sign bits.
By this compression method the total number of outgoing information bits in the input memory bank is reduced to 6. This in turn enhances the maximum available parallelism of the decoder. The third functional block of the proposed design is the variable node sum memory block. This functional block is mainly utilized to store and update the partial sum messages generated during the iterative decoding process. The input messages to be processed are then fed to the router block. The routing and reverse routing network is used to organize the interconnections of various storage and computation units. Therefore, the overall interconnect complexity of the decoder is determined by this routing and reverse routing networks which handle the interconnections.
A. Minimum Comparator and Selection Circuit (MCSC):
In this work, the router block is used to perform two main operations namely shuffling and distribution. The routing block reduces the routing congestion by shuffling the row layers into odd and even pattern. Then by utilizing improved multi-layer row merging scheme, multiple rows can be merged into a single row by applying the sparseness property of LDPC codes. Moreover, the shuffled data which are to be processed further are distributed to minimum comparator and selector circuit (MCSC). The overall functional block view of the pipelined MCSC unit is shown in Fig.2.
In this work, the MCSC unit is designed in a manner such that it can process the values of two (Odd/Even) layers at a single clock cycle. During the initial stage of the operation, the high precision soft information is found by subtracting the values of check-to-variable node from the a posteriori probabilities (LLR). Then the processed information is then stored in a compressed state in the intrinsic and extrinsic memory module. To mitigate overall reduction in the memory bandwidth of the proposed decoder, eDRAM [13] is utilized. Compared to other LDPC decoders which utilizes SRAM [14] and DRAM [15], eDRAM has four times higher bandwidth with low power and enhanced row, column-command interface. Furthermore, the eDRAM has its own counter, which is independent of other memory banks. Therefore, the drawback of LDPC decoders using DRAM and SRAM is mitigated in this work by utilizing eDRAM.

In order to attain high decoding throughput with low fall-throughput latency (pipelining latency) and less power consumption, a single stage hybrid pipelining is employed. In this work, the combinatorial logic of the hybrid pipelining method allows the data process (waves) to self propagate through the logic elements without the requirement of clock signals. Also, along with the initially transmitted data a reset process (wave) is initiated such that it refreshes the residual state of logic elements before the arrival of next set of data bits. Therefore, by this methodology, multiple data bits can be decoded simultaneously as there is no waiting as the next set of data bits are processed immediately after initiation of reset waveform. Furthermore, in this method the reliability of the processed data is preserved as there is no interference among the processed data bits which improves the decoding throughput.

IV. CONCLUSION
In this work, the design and implementation of an energy efficient QC-LDPC decoder has been successfully demonstrated. The implemented decoder is shown to successfully decode multiple LDPC codes of varying code rates belonging to IEEE 802.16e standard. The exhaustive experimental results using 65 nm CMOS technology show that the proposed decoder occupies a core area of 2.97 mm2. In addition, the proposed decoder is shown to achieve a throughput efficiency of around 1528.6 Mb/s and only consumes 389 mW power at 414 MHz, 10 iterations and 1.0V. Moreover, through performance analysis done with other recent LDPC decoders, the proposed decoder offers favorable TAR with greater flexibility, error correcting performance and power consumption.
