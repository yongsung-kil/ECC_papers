Iterative LDPC–LDPC Product Code for Bit Patterned Media
Seongkwon Jeong and Jaejin Lee
School of Electronic Engineering, Soongsil University, Seoul 06978, South Korea

Bit patterned media aims at high density recording of more than 10 Tb per square inch, but a burst error mostly occurred by media defects, and data write failure can be a serious problem. However, if the burst errors are compensated by erasure decoding, the performance of low-density parity check (LDPC) code, which is a strong candidate for the error correcting code for storage systems, can be improved. In this paper, we propose an iterative LDPC–LDPC product code and show that it performs better than a simple LDPC–LDPC product code when there are only random errors and both random and burst errors.
Index Terms— Bit patterned media (BPM), burst error, low-density parity check (LDPC) code, product code.

I. INTRODUCTION

B IT patterned media (BPM) is a high density storage system, and it has been studied as a next generation data storage method [1]. The present hard disk drives exploit perpendicular magnetic recording (PMR) substituting for the longitudinal magnetic recording (LMR) method. However, it is hard to increase the density of the channels due to the problem of super-paramagnetism [2]. Therefore, BPM is one of the possible solutions to overcome the recording limitations caused by the problem of super-paramagnetism.
BPM is a way to save space for the recording of bits by using a single magnetic domain instead of magnetic particles. In other words, it places one domain into a magnetic island by creating a 2-D patterned array. For this reason, it can resolve the noise problem of the particles, increase the recording density, and overcome the problem of super-paramagnetism, thereby offering advantages in a hard disk system [3]. Also, it can decrease the effect of non-linear transition shift, allow for timing recovery, and enable the tracks to be found more easily. BPM stores 1 b in a small island. When implementing a high density, BPM exhibits not only intersymbol interference, as in the case of current serial data storage systems, but also intertrack interference because of the narrowing of the gap between adjacent islands. To compensate this problem, many signal detection and error-correcting codes have been proposed [4], [5].
A burst error caused by physical impact on the storage system in proportion to increase the recording density becomes a very long burst error in contrast with a conventional storage system. Thus, error correction capability for the long burst error is needed [6]. Especially, low-density parity check (LDPC) code and product code are leading candidates in the storage systems. LDPC code was initially proposed by Gallager in 1962 [11] and rediscovered by Mackay and Neal in 1996 [12]. Elias [13] introduced product code in 1954. II. ITERATIVE LDPC–LDPC PRODUCT CODING
To improve the correcting capability for long burst error, product code has been used. Inner code has a minimum number of parities for detecting the burst error. Thus, it can correct most of random and short burst errors while the long burst errors in each row generate the decoding failure. For these uncorrected errors, the decoder declares erasures, which inform the locations of errors to the outer code decoder.

In general, the outer code has a maximum number of parities for correcting errors as many as possible. For instance, a 2-D data structure is 100×100 and a burst error of 5 × 100 occurs. Burst error of five rows can be detected by inner code decoder. Then, it declares erasure to these five lines, and log-likelihood ratio (LLR) values are changed to 0. Thus, outer code decoder can correct these errors using the erasure information.
Fig. 2 shows the block diagram of a simple LDPC–LDPC product code. After LDPC1 decoder corrects errors, it detects burst error if it cannot correct the errors. Then, the decoder transmits the erasure information (error location) and EI to outer code decoder (LDPC2 decoder). Outer code decoder can correct burst error using the erasure information. It is a merit that LDPC1 decoder not only detects burst errors but also corrects random or short burst errors. However, after finishing the decoding of LDPC1 code for rows, LDPC1 decoder transmits once the EI to LDPC2 decoder in a simple LDPC–LDPC product code. In this paper, we propose an improved LDPC–LDPC product code, which iteratively exchanges EI between LDPC1 and LDPC2 decoders to improve the system performance.
Fig. 3 shows the block diagram of the proposed scheme. After the received data pass through the soft output Viterbi algorithm (SOVA), LDPC1 decoder receives LLR value of data and parity of rows from SOVA. After the decoding of LDPC1 decoder is completed, LDPC2 decoder receives EI and erasure information from LDPC1 decoder and LLR values of data and parity of columns from SOVA. After LDPC2 decoder finishes decoding for columns, LDPC2 decoder informs EI to LDPC1 decoder. The erasure information is only sent at the first step, and EI is iteratively sent between LDPC1 and LDPC2 decoders.
III. SIMULATION AND RESULTS
A. BPM Channel Model
BPM with a pulse shape applied to PMR can be modeled in accordance with the soft underlayer (SUL) [14]. When reading data along a track and setting the horizontal direction, the specification of the modeled BPM is as follows. The length of sides of a square island saving one bit is 25 nm,

and PW50 of the pulse width relevant to 50% of the maximum value is 18.8 nm when there is an SUL and 21.2 nm when there is no SUL. The channel shape in the direction across the track is similar to the Lorentzian pulse shape [15]. A theoretical model of the step response commonly using for showing the signal of the LMR channel uses a Lorentzian pulse. The isolated transient response or transient response is referred to as a Lorentzian pulse and PW50 in this direction is 31.2 nm. To reduce the complexity of the simulation, we assume a 2-D channel as follows:

of h[pTx ,qTy]. Tx and Ty are the discrete points of direction along the track and across the track, respectively. In addition, n[ p, q] is additive white Gaussian noise (AWGN) with zero mean and variance σ 2.
