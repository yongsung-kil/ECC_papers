Efficient Multi-rate Encoder of QC-LDPC Codes Based on FPGA for WIMAX Standard∗

WANG Xiumin1, GE Tingting1, LI Jun1,2, SU Chen1 and HONG Fangfei1
(1. College of Information Engineering, China Jiliang University, Hangzhou 310018, China) (2. National Mobile Communications Research Laboratory, Southeast University, Nanjing 210018, China)

Abstract — An efficient multi-rate encoder for IEEE 802.16e LDPC codes which outperforms current single rate encoders with acceptable hardware consumption and efficient memory consumption is proposed. This design utilizes the common dual-diagonal structure in parity matrices to avoid the inverse matrix operation which requires extensive computations. Parallel Matrix-vector multiplication (MVM) units, bidirectional operation and storage compression are applied to this multi-rate encoder to increase the encoding speed and significantly reduce the quantity of memory bits required. The proposed encoding architecture also contributes to the design of multi-rate encoders whose parity matrices are dual-diagonally structured and have an Approximately lower triangular (ALT) form, such as in IEEE 802.11n and IEEE 802.22. Simulation results verified that the proposed encoder can efficiently work for all code rates specified in WIMAX standard. With a maximum clock frequency of 117 MHz, the encoder achieves 3 to 10 times higher throughput than prior works. The proposed encoder is capable to switch among six rates by adjusting the input parameter and it achieves the throughput up to 1Gbps.
Key words — Encoder, Multi-rate, FPGA, Dual Diag-
onal, WIMAX.
I. Introduction
Low density parity codes (LDPC) were proposed in 1960s by Gallager[1] and recalled in 1995 because of its competitive performance comparing to Turbo codes in high BER[2]. Nowadays, LDPC codes are included in diverse communication system standards such as IEEE 802.11n, Digital video broadcasting-satellite-second generation (DVB-S2), Digital terrestrial multimedia broadcasting (DTMB), World wide interoperability for microwave access (WIMAX)[3], etc..
Demand for high throughput and rate flexibility has been increasing in modern communication systems. In Ref.[4], a high-throughput encoder for IEEE 802.16e QC-

LDPC codes was presented. Although it achieved high throughput in excess of 10 Gbps, the encoder was only designed for the single-rate structure. A multi-rate encoder composed of rotate-left-accumulator circuits was presented under DTMB standard[5]. Using RLA circuits to implement a multi-rate encoder saves lots of resources and improves SRAA circuits’ disadvantages. However, it is more suitable for high-speed multiplication of φ−1, such as in the DTMB standard[6]. Ahmed Mahdi[7] presented high-throughput encoders of high flexibility for recursively-constructed QC-LDPC codes. The codes in Ref.[7] have similar density distribution with WIMAX codes (3/4A), and achieve throughputs of 71.5Mbps and 127.5Mbps in the serial architecture, for code rates of 1/2 and 3/4 respectively. Our proposed encoder has the corresponding throughput achievements of 1283.7Mbps and 1225.3Mbps for code rates of 1/2 and 3/4 respectively.
The multi-rate encoders[8,9] were designed for diverse standards, but they didn’t support the IEEE 802.16e standard. Kopparthi[10] proposed an encoder for implementing the structured LDPC codes defined in IEEE 802.16e standard but it needs to be re-synthesized for different code rates and lengths. A new architecture was presented for a configurable QC-LDPC encoder/decoder in Ref.[11]. The architectures were fully configurable at the pre-synthesis stage so that it was not only for a fixed block length or code rate. However, our design does not require reconfigurations for different code rates, which is more flexible in comparison with Ref.[11]. A truly flexible encoder in regards to code rates and lengths under the WIMAX standard was presented in Ref.[12], but required more memory resources and achieved a lower throughput than our proposal.

For QC-LDPC codes, we use k and n to denote information length and code length respectively. The code rate r is defined as r = k/n. The multi-rate encoder is composed of two main parts: a control module, and an encoding module (Fig.2). The control module plays a significant role in the encoding procedure, the flowchart of which is given in Fig.3.

According to the WIMAX standard, the multiplication operation of T −1 and other matrix can be converted into the accumulation form, in which the calculation is simplified. The logic complexity of encoder is 2(gt + kt) + t2 + (4g − 1)t − g in Table 4, so we can observe that the complexity is a linear function of code length, while m, g and t are constant under the same code rate. In conclusion, by taking advantage of the linear relationship between the complexity and the code length, the encoding complexity can be reduced greatly.

Consider input code rate r, then the control module (as shown in Fig.3) starts with the determination of the initial conditions consisting of A’s row block number j, information block number i, each ROM’s initial read ad-

dress, and the shift value in matrix B. Then the shifters’ common enable ports are opened for preparation to calculate the matrix-to-vector multiplications ArS and CS. The control module searches for the column number of non-negative values ‘col ’ stored in each ROM in parallel to find the shift value in the same address. Once the shift value is sent from the control module, the corresponding MVM unit starts to operate in the encoding module. This step continues until col equals i (the number of information vectors). The process takes about 9i clock cycles.

A detailed description of the control module is presented in Section III. The encoding procedure is mainly composed of 5 stages. The next step is to send a control signal to open the AS RAM’s write enable port and another control signal to calculate P0 at the same time. The two steps can be done in parallel and we need continuous j clock cycles to store ArS and one more clock cycle to calculate P0 (shown in Fig.4). In the process P0 update as P0 = ArS ⊕ P0 (r = 1, 2, · · · , j) and finally it ends with P0 = CS ⊕ P0. After that, it is time to shift P0 by a certain value which is defined by the code rate. Simultaneously, P0 is written to Dual Port RAM (DPRAM), without any conflict, which takes two clock cycles. These bidirectional operations described above contribute to the increase of the encoding speed. In order to obtain P1, a state signal ‘state bp0as’ is set to control the stages of P1 calculation. ArS is read from the AS RAM in z bits address by address, and the eventual result P1 is divided into z-bit vectors as P1 = (Q1, Q2, · · · , Qj). The three steps to get P1 are partially parallel and require j + 2 clock cycles in total, as shown in Fig. 5. The total latency to obtain the parity bits is about 9i+2j+5 clock cycles.

Stage 1 is to calculate the intermediate vectors ArS and CS. The information vectors S1, S2, · · · , Si are input sequentially block by block. After all the information blocks are completed, the MVM units output accumulated results ArS and CS. Stage 2 is a bidirectional operation. The outputs obtained are sent to XOR gates to calculate P0. Simultaneously, all groups of ArS are written to the AS RAM for preparation of next stage.

A resource-efficient and high-throughput QC-LDPC encoder has been proposed in this paper, and the encoder can be applied to all code rates in the WIMAX standard. Using the property of the parity matrices, the RU encoding algorithm is simplified by replacing the inverse matrix operation with back substitution. Furthermore, the utilization of MVM units, bidirectional operation and storage compression enormously reduce the required memory bits and increases the encoding speed. Simulation results confirm its correctness and feasibility with a slight increase of resource compared to single-rate encoders. The architecture of proposed multi-rate encoders is promising for using in other standards such as IEEE 802.11n and IEEE 802.22.
