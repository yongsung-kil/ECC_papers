Abstract—Advances in communication systems are drastically changing in day today’s life. Due to this the reliability of information will be always matter of concern, so there is need for improving the error correction mechanisms. The proposed work deals with improving the throughput of structured LDPC decoding defined for IEEE WiMAX 802.16e standards. BER simulation results shows improvement of 10−6 at 2.5 dB SNR, 0.1 dB early convergence compared to standard IEEE 802.16e WiMAX layered LDPC decoding, by employing the reordering and parallel row processing of the parity check matrix.
Index Terms—LDPC (Low density parity check codes), Layered decoding, Throughput performance, Wimax (Worldwide Interoperability for Microwave Access), min sum decoding, QC codes (Quasi-cyclic codes), Parity check matrix.
I. INTRODUCTION
Advances in Wireless technology are so fast increasing to cater the today’s and tomorrow’s demands such as high data transmission, highly reliable and secured data by using any specified high bandwidth media and also to provide the service in areas where, it will be very difficult for wired technology due to infrastructure limitations. Such wireless standard IEEE 802.16e WiMAX is used for small and medium distance communication, which uses LDPC as a powerful error correcting codes for reliable high data transmission. LDPC codes are called as highly computational codes and shown their error correction performance very close to Shannon’s limit. LDPC codes were developed by Gallager in 1962 [1], later these codes were rediscovered by Neal and Mackay [2] around 1993 till today they have been used in almost all communication standards. These LDPC codes are also used in Wi-Fi, IEEE 802.11ad, DVB-S2 and ITU standards [3]. Due to the low decoding complexity and well defined structure with low error floor LDPC codes are to proven to be best codes. Sum product probabilistic decoding performance to be better proven [4] but due to the complexity of implementing mathematical expressions of the check node computations and very difficult to implement hardware structure, so the min sum decoding is derived [2], where the check node computational complexity got reduced results obtained with this decoding ware approximately near to sum product decoding. To improve the min sum decoding, further normalized min sum decoding may be used [5], [6]. To improve the decoding performance still further modified decoding styles for both sum product and min sum algorithms were employed, among them layered decoding [7] and shuffled decoding are important. There are

two types of layered decoding styles namely column layered decoding and row layered decoding. The applications of DVB-S2 and WiMAX standards are always in need of high throughput multi rate decoding structure for next generation wireless communication systems. LDPC codes support the implementation of various data rate and multi code flexibility. The proposed work, here address the implementation of multicode rate LDPC decoder algorithm for IEEE 802. 16e WiMAX standard, where row layered min sum decoding is used. To improve the throughput, the reordering the rows of the parity check matrix and employing parallel processing of more rows of base matrix for code length N = 2304 with sub matrix size of p = 96 is employed. The throughput of the proposed work finds to be better compare to the reported work earlier through MATHLAB simulation results [10], [11]. The hardware structure of decoder architecture reported as fastest decoder for 1/2 code rate giving throughput of 680 mbps at 260 MHz [12].
Sections II describes overview of LDPC codes, construction of base matrix of IEEE 802.16e WiMAX standard, min sum decoding algorithm and brief about layered decoding, modification of parity check matrix and layered min sum decoding is proposed. Section III contains BER simulation results, which are discussed for different code rates. Section IV concludes the proposed work.

The structure of WiMAX matrix contains pij columns on the left part of the matrix, which is circulant permutation matrix or it is zero matrix. Circulant permutation matrix of size p is generated by cyclically shifting the rows of the identity matrix to the right by certain locations of p matrix. The size of p may vary from p = 24 with code length N = 576 to p = 96 with code length of N = 2304 bits. The right part of the structure contains special column in the position (i, k), the first column of the parity check bits and the weight of this element is 3 for all the WiMAX standard LDPC codes and double diagonal staircase structure which corresponds to the parity bits. WiMAX 802.16e code contains six different classes supporting four different code rates. All the six classes have same structure as that of Hbm. Depending on code rate and class the rows of the Hbm varies and number of columns are fixed as 24. For code rate 1/2 and code length N = 2304 the rows are 12 and columns are 24. Due to the quasi cyclic nature of the base matrix encoding complexity will be reduced and it is linearly proportional to code length N .

B. Min Sum Decoding Algorithm
Binary LDPC codes are decoded using both hard decision and soft decision algorithms. Soft decision decoding

algorithms such as sum product, min sum, max star are message passing algorithms between variable nodes and check nodes. Soft decision decoding are better performing ones compare to hard decision decoding. Soft decision decoding works on probability values of the received channel message bits. Consider N (i) = {j : Hij = 1} are set of variable nodes that are connected to the check node i. In the same way M (j) = {i : Hij = 1} be set of check nodes connected to variable node j. C. Layered Decoding

Structured LDPC codes such as IEEE WiMAX and Wi-Fi standards supports both parallel decoding and layered decoding. These schemes are used to enhance the throughput performance [7]. These layered decoding is carried either by horizontal row or vertical column partitioning. These partitioning decoding uses a new decoding data from the current iteration other than conventional decoding scheme, which uses data from previous iteration, this double the rate of convergence hence may increase the throughput. Layered decoding can be processed in both serial and parallel way.

To support all the block lengths 6 parity check matrices are generated as per the required code rate. The parity check matrix for code rate 1/2 of block length N = 2304 uses a standard matrix structure with sub matrix size of p = 96 ∗ 96 as (see Fig. 2).
Likewise the base matrix structure for other code rates are also generated. In the above matrix, the left side some columns filled with −1, which, indicates zero matrices of size p ∗ p and the number(value) in the columns of parity check matrix indicates the p ∗ p cyclically shifted matrix. The right part of the matrix consists of dual diagonal structure and −1 again this represents null matrix of size p ∗ p. Here the parity check matrices of different code rates of N = 2304 are generated for WiMAX 802.16e standard. Encoded the codeword with the BPSK modulated passed through AWGN channel with added channel noise are transmitted. At the receiver, min sum decoder which is described in Section II, used to decode the received message. The normal decoder works as explained by the flowchart in Fig. 3 exploiting the p-level parallelism, where the parity check matrix consists of p copies of the constituent decoders with identical structure which process the LLR message vectors of size p. To improve the throughput further modifications are suggested by looking into the structure of the parity check matrix two or more rows of the parity check matrix can be processed simultaneously by incorporating the layered min sum decoding instead of sequential row layered decoding.

IV. CONCLUSION
As per high data transmission demands of wireless communication systems, attempts are made here by processing the rows of parity check in parallel by reordering the standard parity check matrix of WiMAX standard for code rate of 1/2 of code length N = 2304. There is an improvement of throughput of 0.1 dB early convergence obtained compared to performance obtained from IEEE 802.16e WiMAX standard parity check matrix. The modified1/2 code rate parity check matrix with row layered decoding is able to achieve BER of 10−6 at 2.5 dB for WiMAX standards. The same technique may be applied for other code rates and for all the different code lengths to improve the throughput of structured LDPC codes.
