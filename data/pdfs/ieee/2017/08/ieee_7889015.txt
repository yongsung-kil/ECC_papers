Mitigating Stuck Cell Failures in MLC NAND Flash Memory via Inferred Erasure Decoding
Adnan Aslam Chaudhry, Student Member, IEEE, Cai Kui, Senior Member, IEEE, and Yong Liang Guan, Member, IEEE

Abstract— The multilevel-cell NAND flash memory experiences permanent hard errors due to cell defects (stuck cells). To overcome this problem, stuck cells are either regarded as erasures by the decoder based upon the knowledge of stuck cell location, or the entire memory block containing the stuck cells is marked as bad block and made unavailable for future usage. In this paper, a multiround inferred stuck-cell erasure belief-propagation (BP) decoding (ISED) is proposed in which the stuck cell locations are assumed to be unknown to the decoder. To perform the inferred erasure decoding, the input channel log-likelihood ratio (LLR) information is attenuated before BP decoding by modifying the threshold voltage distribution functions. In case of decoding failure, the probable stuck cell locations are inferred by using the flash’s read-back voltage signal and the decoded code-word bits. For all such inferred stuck cells, the input LLRs are set to zero for subsequent rounds of BP decoding. As the likely incorrect LLRs corresponding to the stuck cells are erased, the performance of BP decoder is substantially improved. Simulation results show that the error-rate performance is improved by more than two orders of magnitude with a moderate increase in decoding complexity under the proposed ISED scheme.
Index Terms— Belief-propagation (BP) decoding, error-rate performance, hard errors, input log-likelihood ratio (LLR), multilevel-cell (MLC) NAND flash memory, stuck cells.
I. INTRODUCTION
T HE NAND flash memory is the leading nonvolatile storage technology, which is being used in a wide range of applications, ranging from consumer electronics to largescale data centers. The state-of-the-art NAND flash memories employ the multilevel-cell (MLC) technology that can store up to m bits per cell (2–4 bits), which leads to tremendous growth in their storage capacity. However, the innovation of MLC technology along with the aggressive node scaling has made the flash memory cells vulnerable to various circuitlevel noise and interference effects. These impairments distort the flash’s threshold voltage signal and, consequently, degrade the storage reliability. To protect flash data against the noiseinduced soft errors, strong error-correcting codes, such as the

Besides the intermittent soft errors, the other type of imperfection that degrades the data reliability in NAND flash memory is the cell defect, also referred to as the stuck-at defect or simply the stuck cell, whose cell state is stuck (fixed) irrespective of the programmed value. A similar phenomenon is also observed in phase-change memory (PCM) [7], [8]. A cell defect may occur initially as a result of manufacturing fault or it may appear after going through a large number of program/erase (PE) cycles. In the latter case, the recurring PE cycles cause irreparable damage to the underlying floating-gate transistor, which prevents a memory cell to switch between its states. In this situation, a memory cell cannot be programmed to its desired value and may lead to a programming error. These stuck cell-induced errors, also known as the permanent hard errors, pose more serious threat toward maintaining the reliability of flash data, especially when the number of such hard errors exceeds the code’s error-correcting capability, and ultimately demand for advanced fault-tolerant solutions.
In the current design practice, stuck cell failures are handled by using three different system-level techniques, namely, the bad-block management, the stuck cell masking, and the stuck cell decoder design, as shown in Fig. 1. In bad-block management, a memory block is declared as bad block, once manifested with a large number of defective cells, and replaced with a good spare block [9]–[11]. However, this technique leads to an excessive wastage of storage space as the majority of nondefective cells within a declared bad block can still be used for data storage. Moreover, realizing the fact that the stuck cells are still readable, they can be utilized in a more efficient manner.
One way to make use of these stuck cells is through the stuck cell masking wherein the encoded code-word bits are mapped over the stuck cells, such that the values of each code

Since then, several other stuck cell masking schemes have been reported in the open literature [13]–[18]. Since the stuck cell locations are known at the time of cell programming, the stuck cell masking schemes try to find the code constructions, which mask a maximum number of stuck cells, while leaving behind the least possible mismatch between the code-word and the stuck values.
In contrast to stuck cell masking, the decoder design schemes for stuck cell mitigation are rarely discussed in the open literature. In this direction, an erasure-based decoding scheme is reported in [19] in which the message bits are encoded through LDPC code and stored along with a compressed stuck cell location vector. The location vector is used to erase the log-likelihood-ratio (LLR) information corresponding to the stuck cell locations. This scheme shows significant improvement in the error-rate performance as the likely incorrect LLRs do not take part in the beliefpropagation (BP) decoding. However, it requires additional storage and new write and read operations in order to store and retrieve the location vector. A very similar approach to handle the stuck cell failures in PCM is presented in [20], wherein the retrieved bits corresponding to the stuck cell positions are flipped if read at a different value than what was programmed by using the location vector. A hybrid decoding architecture is proposed in [21] in which the information bits are divided into two bit streams and decoded separately via RS and LDPC decoders. A hard-decision-based Gallanger-B decoder is presented in [22], which takes advantage of the LDPC code structure to detect and correct the permanent stuck-at cell errors.
In this paper, we present a novel inferred stuck cell erasure decoding (ISED) scheme in which the stuck cell locations are detected/inferred with the help of read-back voltage signal and the decoded code-word bits. The most prominent feature of the proposed ISED scheme is that it recovers the stuck cell failures without storing the stuck cell location information, thereby avoiding the additional read and write operations. Instead, we first attenuate the LLRs corresponding to the stuck cell state by modifying voltage distributions as a precautionary measure to diminish the magnitude of likely incorrect LLRs. In case of decoding failure, we innovatively infer the set of probable stuck cells by comparing the detected (predecoded) and decoded code-word bits, and perform either the complete or partial LLR erasure. We initiate a new round of BP decoding using the erased LLR bits. Providing erasures rather than wrong LLRs on these stuck cells makes it easier for BP decoder to correct the likely wrong code-word bits. Moreover, we explore the possibility of combining the stuck cell masking schemes with the proposed ISED scheme in order to achieve further improvements in the error-rate performance. Using numerical simulation, we demonstrate a noticeable improvement in the error-rate performance under the proposed ISED scheme.

II. OVERVIEW OF FLASH MEMORY
The flash memory cells, each made from floating-gate transistor, are organized into an array of rows and columns. To store information, cells are programmed by applying a specific amount of charge to the floating gate, which configures the cell’s threshold voltage over distinct write (verify) voltage levels. These voltage levels, in turn, represent the logical cell states. For instance, for 2 bits per cell flash memory, the cells are configured over four distinct voltage levels, denoted by V11, V10, V00, and V01, representing cell states s11, s10, s00, and s01, respectively. To retrieve the stored data, the programmed charge level is detected through the sense amplifiers by placing a specified word-line voltage across the control gate. Inadvertently, the intended programmed voltage level is affected by various circuit-level noise and interference effects. In Section II, we will present the probabilistic model of flash channel incorporating the programming noise, random telegraph noise (RTN), retention noise, and cell-to-cell interference (CCI) as major sources of voltage impairment. Flash channel models similar to ours are extensively reported in the open literature [23]–[29].

B. Stuck Cell Model

In addition to the above-mentioned voltage signal impairments, the NAND flash memory may also exhibit cell defects, especially after it has gone through a large number of PE cycles. In the defective state, the charged electrons are often trapped inside the floating-gate transistor, thereby preventing the configuration of cell’s threshold voltage. As a consequence, the read-back voltage signal stays fixed and becomes independent of any other channel statistics. III. INFERRED STUCK-CELL ERASURE DECODING (ISED)

We first briefly explain the BP decoding of LDPC code over the flash memory channel. Let us denote the set of N memory cells with C = {C1, C2, C3, . . . , C N } whose cell values constitute a single LDPC code word, where each cell can store 2 bits of information. Thus, N most-significantbit (MSB) values and N least-significant-bit (LSB) values constitute a single LDPC code word, which has a total code length of 2N bits. Let us further denote the set of stuck memory cells with C˜ = {C˜ j | j ∈ {1, 2, . . . , N}}, where C˜ ⊆ C. To initiate the BP algorithm, the LLR information of the entire LDPC code word is required. To compute the LLR information, the read-back voltage signal v j , for j = 1, 2, 3, . . . , N, is measured at discrete read threshold levels. The read threshold levels can be chosen so as to maximize the mutual information (MMI) between flash channel’s input and output voltage signals, as this can minimize the LDPC error-rate performance [33]. In order to reduce the memory read latency, a progressive sensing/decoding strategy that progressively increases the number of read threshold levels in a trial-and-error manner is often adopted [25], [34]. To start the progressive memory sensing for 2 bits per cell flash, at least three read threshold levels are required, denoted by R1, R2, and R3, to read both MSB and LSB positions. With three-level quantization, the voltage range is split into four nonoverlapping voltage regions, as shown in Fig. 3. B. Stuck Cell Inference
Despite the LLR attenuation step, if we encounter BP decoding failure, we perform the stuck cell inference through the proposed ISED scheme. In addition to LLR attenuation, the objective of the ISED scheme is to recover the stuck cell failures without performing new read and write operations. This is because the read and write operations of the NAND flash memory-based devices are the most time-consuming processes and incur significant latency and power consumption. Instead, we erase the incorrect LLR values based on the inferred stuck cell information and initiate a new round of BP decoding. We repeat the LLR erasure and BP decoding steps until a code word is successfully decoded or maximum erasure rounds, denoted by I , are exhausted. The block diagram of the proposed ISED scheme is shown in Fig. 4.
However, contrary to the work of [19] and [20] where the stuck cell locations are known to the decoder, the proposed ISED scheme needs to identify the stuck cell locations. To this end, we leverage on the decoded code-word bits to infer the stuck cell locations. In particular, we compare the detected (predecoded) and decoded code-word bits, denoted by {amj sb, aljsb} and {aˆmj sb, aˆljsb}, respectively, and record the cell indices j , for j ∈ {1, 2, . . . , N}, where the two cell values tend to differ, that is {amj sb, aljsb} = {aˆmj sb, aˆljsb}. These mismatched cell values will allow us to identify the stuck cell locations.

Fig. 4. Block diagram of memory sensing and BP decoding with the proposed inferred stuck cell erasure decoding (ISED) scheme.
It is pertinent to mention that the dependence on the code-word bits upon decoding failure may also lead to false inference of stuck memory cells as some of the decoded code-word bits are incorrect. Nonetheless, we exploit the fact that the first round of BP decoding already corrects a large number of detected bit errors. Thus, even the partially corrected codeword bits provide a reasonable detection performance, as shown in Section IV-B.
To infer the stuck cell locations, we reversely exploit the effect of stuck cells. In particular, we mark all the uncertain memory cells C j for v j > R3. These memory cells manifest the detected cell value {0, 1}. However, if the corresponding decoded code-word bits produce cell values {1, 1}, {1, 0} or {0, 0}, instead of the expected value {0, 1}, we would consider them as the likely affected stuck cells. These bit transitions are exactly what we expect to occur in case of stuck cell detection, whereby the intended cell values {0, 1}, {0, 0} or {0, 1} would have been failed to be configured at the desired write-voltage levels during the cell programming. It should be noted that some of the memory cells that were originally programmed to cell state s00 may inadvertently appear into stuck cell state s01 due to channel noise. The proposed ISED scheme will also identify such noise-induced soft errors by comparing the detected cell value {0, 1} with the decoded cell value {0, 0}.
Against all the inferred stuck cell locations, we erase the corresponding LLR values. In particular, for cell value transition from {0, 1} to {1, 1} or {0, 0}, we will perform the partial LLR erasure by erasing the MSB or LSB LLR, as only one bit flip is observed. For cell value transition from {0, 1} to {1, 0}, we will perform the complete LLR erasure by erasing both the MSB and LSB values, as shown in Fig. 5. E. ISED for 3/4 Bits per Cell NAND Flash Memory
In recent times, the main-stream NAND flash manufacturers have started to produce 3 bits triple-level cell (TLC) or 4 bits quad-level cell (QLC) per cell NAND flash configuration, which linearly increases the storage capacity by almost 3× or 4×. In this context, the application of the proposed ISED scheme over the TLC/QLC NAND flash technology can be realized by modifying the stuck cell inference strategy. To this end, the stuck cells can be identified by comparing the detected and decoded code-word bits. For instance, assuming the one-state stuck cell model, where cells may stuck at the highest programmed state s011, the stuck cells can be inferred when the decoded bit values appear to be different than the detected ones. For all such probable stuck cells locations, the corresponding LLR information can be erased for erasure BP decoding.
V. COMBINING ISED WITH STUCK CELL MASKING SCHEME
The proposed ISED scheme can be used in complementary manner with the existing stuck cell masking schemes, such as [13]–[18]. This feature provides further improvements in the error-rate performance. In the combined effort, the cell masking scheme partially precludes the occurrence of hard errors by matching the code-word bits over stuck cell values, while the ISED scheme handles the remaining hard errors via erasure decoding. VI. CONCLUSION
Stuck cells are the source of permanent errors in NAND flash memory that can severely degrade the storage reliability. To mitigate the stuck cell-induced errors, a novel concept of inferred erasure decoding is introduced. The main idea is to identify the stuck cell locations, which are unknown otherwise to the decoder, and erase the likely incorrect LLR values to facilitate the erasure BP decoding. In this paper, all the LLR values originating from the stuck cell state are attenuated. This step diminishes the effect of potentially wrong LLRs. However, to fully offset the effect of wrong LLRs, an iterative erasure decoding is performed. This is achieved by detecting the stuck cell locations, leveraging the flash’s readback voltage signal and the decoded code-word bits, and then erasing the corresponding LLR values. By simulating a modelbased 2 bits per cell flash memory with three and six levels of memory sensing precision, a significant improvement in the error-rate performance is realized. Besides, the proposed inferred erasure decoding scheme is used in conjunction with the existing DI scheme [16] to obtain further performance gain.
