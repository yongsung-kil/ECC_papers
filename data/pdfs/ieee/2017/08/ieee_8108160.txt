Complexity-Constrained Spatially Coupled LDPC Codes based on Protographs

Massimo Battaglioni, Marco Baldi Dipartimento di Ingegneria dell’Informazione
Università Politecnica delle Marche Ancona, Italy
Email:m.battaglioni@pm.univpm.it, m.baldi@univpm.it

Enrico Paolini Department of Electrical, Electronic, and Information Engineering “G. Marconi” University of Bologna, Cesena (FC), Italy

Abstract—Spatially coupled low-density parity-check (SCLDPC) codes are often thought as codes with very long block lengths. However, they can be decoded through sliding window (SW) decoders achieving near-optimal performance when the window size is a few times larger than the code syndrome former constraint length. This makes SC-LDPC codes with short constraint length suitable for low-latency transmissions, and they can even outperform their block code counterparts. Complexity of SW decoders increases linearly with the window size and with the number of decoding iterations. When complexity is constrained, an optimal trade-off between the window size and the maximum number of decoding iterations has to be found. In this paper, we propose a PEXIT-based method to find the best trade-off for codes with short syndrome former constraint length. We consider codes based on protographs, and validate the results of the PEXIT-based method through Monte Carlo simulations.
Index Terms—LDPC codes, protograph-based codes, spatial coupling, sliding window decoding.
I. INTRODUCTION
Low-density parity-check (LDPC) codes where firstly proposed by Gallager in 1962 [1]. After the introduction of turbo codes, LDPC codes were rediscovered, owing to the capacityapproaching performance achieved by iterative decoding algorithms working on their Tanner graph. Recently, spatially coupled LDPC (SC-LDPC) codes gained a prominent role due to their capability to achieve capacity under belief propagation (BP) decoding, owing to a threshold saturation phenomenon [2]. SC-LDPC codes, also known as terminated LDPC convolutional (LDPCC) codes [3], can be constructed starting from a very simple graphical structure, called a protograph [4], and applying a copy-and-permute operation. The design of protograph-based codes has been addressed in several works, e.g., [5]–[7].
The main techniques used to optimize LDPC code ensembles are density evolution [8] and extrinsic information transfer (EXIT) analysis [9]. A modified version of EXIT analysis called protograph extrinsic information transfer (PEXIT) analysis, able to capture the structure of the protograph, was proposed in [10]. LDPC code ensembles are usually designed for an asymptotically large codeword length and letting the number of decoding iterations tend to infinity. Nevertheless, practical applications may require a finite, and constrained, number of iterations. Code ensembles whose asymptotic decoding thresholds are optimized for an infinite number of

iterations may result in sub-optimal codes in an iterationconstrained setting. Recently, efforts have been devoted to the optimization of block LDPC code ensembles for a finite number of iterations, over the binary erasure channel (BEC) and the additive white Gaussian noise (AWGN) channel [11]– [14]. To our best knowledge, an analysis in this regime is missing for LDPCC codes. Although these codes are characterized by large block lengths, they can be decoded with a BP-based sliding window (SW) decoder [15], [16]. Compared with the classical full-size BP decoder, the SW decoder yields significant savings in terms of decoding complexity and latency. Moreover, when the window size is sufficiently larger than the code syndrome former constraint length, the degradation in terms of bit error rate (BER) performance of SW decoding with respect to full-size BP decoding is very small. Complexity of the SW decoder increases linearly both with the number of decoding iterations and the window size, i.e., ultimately, the code syndrome former constraint length.
In resource constrained devices, complexity must be kept limited due to scarcity of hardware and software resources. For these reasons, LDPCC codes may seem unsuitable for these contexts. However, SW decoding of LDPCC codes with short constraint length can be performed over short windows resulting in very good performance, often better than that of their block code counterparts. Therefore, it is worthwhile to investigate how LDPCC codes behave under severe complexity requirements. In this paper we propose a method that can be used to find the best trade-off between the SW size and the number of decoding iterations. For this purpose, we define a complexity metric capturing both aspects and we aim at finding optimized codes under complexity constraints. We focus on the matrix representation of protographs, called the base matrix, and exploit PEXIT analysis to find such a tradeoff. We then validate the results of our analysis through Monte Carlo simulations. We remark that the approach we propose is tailored to finding an optimal trade-off between the SW size and the number of decoding iterations for a given code, and not to designing new codes. Our approach can be applied to any family of protograph-based LDPCC codes designed through classical techniques.

A protograph can be unambiguously represented by its base

the ith check node to the jth variable node, i = 1, 2, . . . , c

and j = 1, 2, . . . , a; obviously, 0 ≤ Bij ≤ Dmax. In order to obtain an SC-LDPC code ensemble, a protograph is copied a

certain number of times, say 2L + 1, and then expanded by

a factor M , called the lifting factor. The edges are permuted

in a random-like way, under the constraint that the number

of edges leaving variable nodes and reaching check nodes is

invariant, except for few check nodes required to terminate the

code. The maximum distance between any two coupled blocks

(measured as number of blocks) is mh, which represents the memory of the code and is referred to as syndrome former

memory order.

The lifting procedure can also be seen as the substitution

of M × M random permutation matrices (or sums of random

permutation matrices, in case of parallel edges) to the nonzero

elements of B, and M × M zero matrices to the null symbols

of B. If the permutation matrices are chosen to be circulant,

the lifting is cyclic, and quasi-cyclic low-density parity-check

convolutional (QC-LDPCC) codes are obtained. Furthermore,

the design rate of the LDPCC code tends to the design rate

of the uncoupled code for L → ∞; thus, we will refer to the

former as asymptotic rate R∞. For the simple protograph in Figure 1, the above steps are illustrated in Figure 2, where

III. SLIDING WINDOW DECODING
A protograph-based convolutional code can be represented through a chain of coupled protographs or, equivalently, through a parity-check matrix containing a chain of syndrome former matrices. The latter has size a × Lh and is denoted by H0; its transpose is denoted by H0T . Due to the effect of lifting, both a and Lh must be multiples of the lifting factor M , that is, a = M a and Lh = M Lh. Since lifting is performed on a random basis, each copy of H0 in the chain will be generally different from the others: thus, the codes we consider are timevarying LDPCC codes. We denote as H0(i) the ith copy of H0, . When the code is not terminated, the protograph chain as well as the parity-check matrix are semi-infinite. Instead, when the code is terminated they both have a finite (though large) length. In both cases, performing decoding over the whole codeword may be prohibitively complex.
An approach to reduce complexity while retaining very good performance is the so-called SW decoding. Decoding is performed over a window of size W , measured in blocks, at a time; the corresponding size in bits is obtained by multiplying W by a . Then, when decoding over the current window terminates, a decision is made about the first a bits within the window. In the next step, decoding restarts over the next window, obtained by translating the previous window by a positions towards right (and by c = cM positions downward, in the parity-check matrix representation). Experimental assessments show that values of W roughly 5 to 10 times mh + 1 result in significant savings in latency and memory with minimal performance degradation.
At the beginning, the decoder executes BP over the first window with the aim of decoding the first a symbols in the window, called target symbols. Decoding is performed starting from the vector of log-likelihood ratio (LLR) values for the received symbols. A right message vector and a left message vector are iteratively updated according to the sum-product algorithm with log-likelihood ratios (SPA-LLR) check and variable node update rules. Therefore, we are able to find the optimal trade-off between W and Imax by minimizing those functions, for integer values of both W and Imax. This way, we are able to design the decoder parameters in such a way that the best asymptotic performance is achieved under the given complexity constraint.
These results are also confirmed in non-asymptotic conditions, through Monte Carlo simulations of binary phase shift keying (BPSK) modulated transmissions. This means that the complexity constraint used in the PEXIT analysis shall take into account such an expansion by a factor M when real codes are designed. However, we cannot neglect the stringent complexity constraints under which we are developing our analysis. If we relax such constraints and allow a fixed and large number of decoding iterations, as well as a large window size, from the PEXIT analysis we find that the codes with larger syndrome former constraint lengths asymptotically achieve better performance than those with shorter ones. V. CONCLUSION
We have studied the optimization of LDPCC codes with short constraint length under stringent complexity constraints, and provided a method to find optimal trade-offs between the decoding window and the number of decoding iterations. Our results have been developed in the asymptotic regime and validated through numerical simulations in the non-asymptotic regime.
