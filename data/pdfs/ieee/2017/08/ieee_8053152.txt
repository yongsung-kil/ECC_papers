A High-Performance FPGA-based LDPC Decoder for Solid-State Drives

Yanhuan Liu, Chun Zhang, Pengcheng Song, Hanjun Jiang
Research Institute of Tsinghua University in Shenzhen Institute of Microelectronics, Tsinghua University, Beijing, 100084, China
Email: zhangchun@tsinghua.edu.cn

Abstract‚ÄĒIn order to improve the throughput of error correction decoding for the high-performance solid-state drives (SSDs), a semi-parallel low-density parity-check (LDPC) decoding architecture is proposed in this paper. The circuit of the LDPC decoder which can be dynamically configured with bit rate and code length is implemented using the scheduling control flow mode of single instruction multiple data (SIMD) instruction. The Peripheral Component Interconnect Express (PCIe) interface is designed and the adaptive normalization factor is applied to achieve an average improvement of 35% in throughput with a signal-to-noise ratio (SNR) of 6.08 dB. The LDPC decoder is implemented on the Xilinx VC709 FPGA. With a rate-0.94 length35840 quasi-cyclic LDPC code, the decoder achieves a throughput of 1.97 Gb/s which compares favorably with previously proposed architectures.
Keywords‚ÄĒ LDPC decoder; dynamic configuration; adaptive normalization factor; SSDs; FPGA; PCIe
I. INTRODUCTION
In the era of large data, the solid state drives gradually replace the traditional mechanical hard disks and attracts much attention from both the industry and academy. The raw bit error rate becomes higher and higher with the increasing storage density in the modern storage devices. The error correction (ECC) codec is becoming more and more important. The LDPC code replaces the traditional BCH code with its superior performance close to the Shannon limit [1]. There are some scholars researching polar code [2] while they achieve the lower throughput and need much improvement to meet the needs of today's storage devices. The data transmission technology continues to develop in the modern storage devices. Peripheral Component Interconnect Express (PCIe) interface gradually becomes peripheral interface of SSD replacing the conventional Serial ATA Express (SATAe).
In this paper, we propose a high-performance LDPC decoder structure for NAND Flash storage media [3]. We adopt the normalized min-sum algorithm [4], a hierarchical messaging architecture [5] and the adaptive method to improve error correction capability and decoder convergence rate. The on-line programming of the matrix H is realized by introducing the instruction scheduling control flow. The designed decoder can support variable code rates and code lengths dynamically. It has a good versatility while greatly reducing the hardware overhead.

D. Instruction design
In this paper, the matrix H is compiled into the corresponding single instruction multiple data (SIMD) for controlling the whole decoder to achieve decoding and error correction [12]. A SIMD instruction is corresponding to a submatrix and relevant operations. In this paper, four types of basic instructions are designed. The N-type instruction is a regular instruction corresponding to the position of the sub-matrix in general and the general operation; The R-type instruction is a non-blocking instruction at the end of the line indicating that the relevant sub-matrix is the last sub-matrix of all rows. At this point you can use ‚Äę‹ģ‚Ä¨ŗĮ°ŗĮ†ŠąļŗĮĚŠąĽ to update ‚Äę‹•‚Ä¨ŗĮ†ŗĮ°ŠąļŗĮĚŠąĽand store the result in C-Ram. The M-type instruction is a matrix non-blocking instruction which indicates that the corresponding sub-matrix is the last sub-matrix in the iteration operation, so the read address in INS-Ram needs to be assigned to point the first instruction while updating C-Ram. The S-type instruction is a matrix-end blocking instruction which is used to make the CN module to keep not working for a number of cycles and wait for ‹ĽŗĮ°ŠąļŗĮĚŠąĽ being updated.
E. PCIe subsystem design
In this paper, we apply PCIe interfaces to transfer data between the host and AXI-4 bus replacing traditional SATA interfaces to improve the data transferring rate. We design PCIe interfaces referring the IP kernel in the development tool based on XILINX VC709 FPGA. The system automatically assigns addresses after completing circuit connection. The working principle is shown in Fig. 8. First, data is written into DDR3 memory from host; then, the source address, destination address and data length are written into register file, after that, specified register is written to generate interrupt signal for MICROBLAZE. V. CONCLUSION
In this paper, we design a LDPC decoder with highthroughput performance and error correction capability to meet the increasing requirement of NAND Flash memory products. The decoder is controlled by the instruction sequence so that the bit rate and the code length can be dynamically configured through compiling different matrix H. Besides, the decoder automatically adjusts the normalization factor to improve the performance while adding very few resources. The decoder is implemented on Xilinx VC709 FPGA. With (35840, 32768) LDPC code, the throughput of the decoder is 1.97 Gb/s while the original bit error rate of the input data is ÕĶ ŗĶą Õ≥Õ≤ŗ¨Ņŗ¨∑, which is required in SSDs applications.
