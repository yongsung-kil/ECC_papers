Irregular Repeat Accumulate Low-Density Parity-Check Codes Based on Residue Class Pair

Kai Tao, Li Peng, Kun Liang, Bo Zhuo
School of Electronic Information and Communications, Huazhong University of Science and Technology Wuhan National Laboratory for Optoelectronics, Wuhan 430074 Hubei China
e-mail: 591317194@qq.com, pengli@hust.edu.cn, liangkun@hust.edu.cn, zhoubo@hust.edu.cn

Abstract—This paper presents a new family of irregular repeat accumulate (IRA) low-density parity-check (LDPC) codes with the characteristic of random-like structure, a common feature of the practical LDPC codes adopted in most communication industrial standards. The systematic IRA-LDPC codes are defined by the parity-check matrix H = [Hd HP], in which Hd and HP correspond to the information bits and the parity bits respectively. We introduce the concept of residue class to create the residue class pair (RCP), from which we construct a compact representation of Hd matrix. It is demonstrated that, the family of designed RCP-based IRA-LDPC codes can provide better performance, higher structural level and lower memory consumption than the practical LDPC codes in some communication standards such as IEEE 802.16e and DVB-SII.
Keywords-irregular repeat accumulate (IRA) codes; lowdensity parity-check (LDPC) codes; parity-check matrix; residue class pair (RCP)
I. INTRODUCTION
The information and coding fields have long considered that random codes are good codes but are difficult for complete hardware realization, while the algebraic structural codes, on the contrary, are easier to implement but exhibit inevitable performance loss. So those codes with quasioptimal performance and feasible complexity, so-called random-like structure, have roused wide attention. Lowdensity parity-check (LDPC) codes, which were originally proposed by Gallager in 1962 [1] and were re-discovered in 1996 by MacKay and Neal [2], are just this case. The practical LDPC codes have been extensively studied and widely applied to several communication systems due to their Shannon-limit-approaching performance. At present, there are two classes of practical LDPC codes, i.e., the quisicyclic (QC) LDPC codes adopted in several IEEE 802 series standards [3] and deep-space communication standard, and the IRA-LDPC codes adopted in Digital Video Broadcasting Satellite-Second Generation (DVB-SII) [4] of European satellite communication standard. The practical random-like IRA-LDPC codes have the following advantages that the variable range of code rate is wider and more flexible, and the performance of long code of rate 1/2 is lower than 1dB, when the bit-error-rate (BER) below 10-5, all of which the practical random-like QC-LDPC cannot match. The existing literatures focus mainly on the QC-LDPC codes, but little on the IRA-LDPC codes, which becomes our dominant motive of investigating the construction of the IRA-LDPC codes.

II. DEFINITIONS AND PRELIMINARIES
Suppose that when two integers are divided by the same positive integer, the reminders are identical, that is, a1 = q1m + r, a2 = q2m + r, 0 ≤ r < m, a1 and a2 are said to be congruent modulo m, written a1 {a2 (mod m).
According to the concept of congruence, we can classify the integers congruent to r modulo m into one class from all the integers. Each class forms a set r̅ m = {a| a { r (mod m)}. All the integers can be divided into such m sets, namely, m residue classes. Obviously, any arbitrary integer must belong to one of m residue classes.
The following definition introduces a class of pairs of crucial importance in this paper.
Definition 1 (Residue Class Pair): We extract two integers r and q from the expression a = qm + r, to constitute a pair (r q)m, which is called a residue class pair (RCP) modulo m. By means of the expression a = qm + r, the pair (r q)m maps to the integer a, which means the qth element in the residue class r̅ m is a, when we classify all the integers by modulo m.
For example, let m be 5, (3 0)5 indicates the 0th element in residue classC35 is a = 0 u 5 +3 =5. To take another example, (2 3)5 indicates the 3th element in residue classC25 is a = 3 u 5 +2 =17.
Regarding a RCP as an element in arrays, we obtain the definition of RCP array.
Definition 2 (Residue Class Pair Array): Consider an m u k array D. The array D is composed of two elements: null set ∅ and residue class pair (r q)m modulo m. Because there is no ambiguity, in the rest of this paper, the residue class pair (r q)m modulo m is written (r q) for short. We stipulate there are arbitrary amount of RCPs in each column among the array D, and we denote the amount of pairs in each column by v0, v1, ⋯, vk-1. Therefore, the amount of RCPs in D is ∑kG=-10vG.

V. OPTIMAL SEARCH FOR RESIDUE CLASS PAIRS AND
SIMULATION RESULTS
We first summarize all the conditions listed below that the relation among these structural parameters in DH should meet, if the code length N = K + M and the rate R = K/N are given. 1) L is the common factor of K and M, that is L = K/k =
M/m. Under the condition that L exists, m+1 must be a prime number. 2) From all φ(m) generators in Galois field GF(m+1), select u generators from small to large.
3) u is the amount of RCPs in each row of [rθ,δ] = [r\,J]muu, which is also the row weight of He. This indicates the row weight of matrix Hd is the same, uL.
4) The column weight of He is either vmin = 3 or 3 < vmax < m.
5) K1 + K2 = K, and K1 vmin + K2 vmax = Mu. The possible values of L, m, k, vmax, u, K1, K2 and g0, g1,
⋯, gu-1 are lots. These parameters match well with each other, to satisfy the aforementioned 5 conditions, and form a parameter group of DH. Thus, the optimal search algorithm is: among this group, to search the best matching parameters which makes the SNR smallest, at the meantime, the bit error rate (BER) below 10-5.
Based on the above-mentioned search algorithm, we search out 10 well-performed arrays DH, which correspond to known parameters in two communication standards, i.e., code lengths N=576, 960, 1536, 2304 with 1/2 rate in IEEE

802.16e, and the rate R=1/4, 1/3, 2/5, 1/2, 3/5, 2/3, 3/4, 4/5, 5/6, 8/9, 9/10 with the same code lengths N=64800 in DVBSII. Detailed structural parameters are shown in Table I and II, respectively.

VI. CONCLUSIONS In this paper, we introduced a new method of constructing IRA-LDPC codes based on RCP. The resulting codes have a better performance compared with QC-LDPC codes adopted in IEEE 802.16e and IRA-LDPC codes adopted in DVB-SII, in consequence of the random-like architecture, which can also reduce the consumption of memory resource for the random parity-check matrix, providing the potential

probability for designing low complexity hardware circuit of the LDPC encoder/decoder.
