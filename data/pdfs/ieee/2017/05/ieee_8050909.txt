Abstract—LDPC codes are deployed in many modern wired and wireless communication systems. While fully-parallel LDPC decoders are very efficient, they typically suffer from routing complexity. The Split-Row method effectively reduces this complexity with a minor performance loss. This paper shows the importance of symmetry in Split-Row architectures and proves that the implementation of Split-Row decoders based on new proposed smart column-permuted versions of parity check matrices leads to a better error performance as well as a more efficient hardware. Moreover, in order to achieve optimized column-permuted parity check matrices, a heuristic approach is proposed. This method is then generalized to support QC-LDPC codes. Applied to IEEE 802.3an (10GBASE-T Ethernet) and IEEE 802.11n (Wi-Fi) LDPC decoders, the new technique improves the error performance, while leading to almost 3x speed-up in the synthesis compile time and about 10% reduction in the critical path.
I. INTRODUCTION
Although low-density parity-check (LDPC) error correction codes have been initially proposed by R. G. Gallager in 1962 [1], their rediscovery and real-world adoption was postponed until the late 90s. Today, these codes are deployed in many modern wired and wireless communication systems. This is not only because of their potential to achieve the channel capacity [2], but also their ability to be decoded iteratively using message-passing algorithms [3].
Message-passing algorithms such as sum-product (SP) and min-sum (MS) allow concurrent updates of messages passed between variable and check nodes. As a result, if all of the processing nodes are physically implemented, highthroughput, energy-efficient decoders can be designed [4]. However, such fully-parallel implementations come at the expense of increased routing complexity, which in turn leads to a low area utilization.
Various methods have been proposed in the literature to mitigate this routing complexity issue in fully-parallel decoders. While studies like [5] attempt to solve this problem through implementation-aware code designs, others introduce hardware implementation methods to overcome the routing complexity. For instance, bit-serial decoders were introduced to reduce the number of physical wires by serializing the message communication [6], or resource sharing was proposed to exploit sub-matrix structures in Reed-Solomon-based LDPC codes [7]. On the other hand, partitioning check node processors to smaller instances is offered by the Split-Row method, which leads to significant improvements in area utilization [8].

In order to realize more efficient fully-parallel schemes, this paper illustrates the importance of split symmetry in the parity check matrix of Split-Row LDPC decoders, something neglected by prior studies. Towards this goal, an index is introduced to parametrize the symmetry of partitions. Moreover, it is shown that using the proposed heuristic approach for column permutation can substantially increase the split symmetry of the parity check matrix (H). As a result of building Split-Row decoders based on the optimized column-permuted H matrices, the bit-error-rate performance and critical path of decoders are improved. The proposed optimization is applied to the standard H matrix of IEEE 802.3an (10GBASE-T Ethernet) as well as an irregular IEEE 802.11n (WLAN) H matrix, where the error performance is improved. Furthermore, the hardware advantages of optimized architectures are demonstrated by synthesizing two IEEE 802.3an decoders.

II. SPLIT-ROW BACKGROUND

Each LDPC decoder is formed based on a parity check matrix, H. This matrix is well expressed by its corresponding Tanner graph, a bipartite graph that contains variable nodes, representing columns, and check nodes, representing rows of H. Messages are passed along the edges of the Tanner graph in message-passing algorithms. Check nodes send messages to variable nodes (α messages), and vice versa (β messages).

Therefore, those Ns − 1 check nodes that are unaware of the global minimum can generate inaccurate α messages. This partial knowledge is the reason behind Split-Row’s error performance degradation and deteriorates as the number of splits increases.
Different methods have been proposed to improve the error performance of Split-Row LDPC decoders. Message normalization [5] is an effective approach that earlier SplitRow implementations utilized to compensate this inaccuracy partially [8]. The new normalization factor, SSR, is determined through simulations and is different for decoders with different split numbers. Researchers later introduced Split-Row Threshold [9], where the local minima are compared to a fixed threshold and adjacent partitions update one another when a minimum lower than the threshold exists. Requiring some implementation overhead, this idea overcomes the lack of global knowledge to some degree and improves the error performance. Finally, trust to the truthful check node (TTCN) [10] tries to find the partition that is most probable to create the global minimum in each decoding round. Therefore, other partitions can trust to it during some of the decoding iterations. This approach reduces the switching activity of Split-Row decoders and leads to a slight error performance improvement due to the propagated global knowledge.
This work explores how the symmetry of split partitions affects the error performance and hardware implementation of Split-Row decoders. After defining split symmetry metrics, a novel heuristic column permutation algorithm is introduced to optimize the H matrix before implementing the Split-Row decoder. Unlike the earlier studies, this methodology has no hardware overhead and is performed during the design process.

IV. SMART COLUMN PERMUTATION ALGORITHM
As shown later, increased split symmetry results in both a better error performance and more efficient hardware implementation of Split-Row decoders. We propose a heuristic column permutation algorithm to increase the split symmetry of such decoders. This algorithm is performed on a target H matrix during the design process with no extra hardware overhead. Under any column permutation, only the variable node indices in H’s corresponding Tanner graph are renamed and its structure remains intact. This guarantees that the cycle and girth characteristics of the Tanner graph remain untouched.

Also, in order to prevent long ineffective runs, sporadic random column swaps (mutations) are used. VI. ENHANCED HARDWARE UTILIZATION
In order to investigate the effectiveness of the increased split symmetry on decoder’s hardware implementation, two IEEE 802.3an (10GBASE-T Ethernet) LDPC decoders, called Dec1 and Dec2 here, are synthesized in lsi_10k technology library. Dec1 is based on the standard parity check matrix of IEEE 802.3an, while Dec2 is based on its split-symmetric version. As Fig. 4(b) depicts, all of the check node processors in the split-symmetric decoder have 16 interconnects. Therefore, instead of different types of check node processors, only one type, i.e. the 16-interconnect check node processor, is developed and instantiated. This reduction in instance types and complexity eases the optimization process, something that resulted in 2.92× speed up of the compile process using similar PC settings. Having only one type of node with a generalized routing pattern provides a great opportunity to have a hierarchical design that can be easily extended.
VII. CONCLUSION
In this paper, we have shown the importance of split symmetry in Split-Row LDPC decoders. We introduced split symmetry metrics together with a heuristic column permutation algorithm to increase the symmetry of parity check matrices. The resulting decoders have a better error performance and can run faster, due to a reduced critical path. Also, increased symmetry speeds up the synthesis process noticeably.
