Non Zero Syndrome Parallel Concatenated Gallager Codes

Abstract— Parallel Concatenated Gallager Code (PCGC) is a class of concatenated error correcting codes designed to reduce the encoding and decoding complexity of long block length Low Density Parity Check (LDPC) codes. In this paper, we present a novel class of PCGC in which one constituent code satisfies zero syndrome criteria and the other one satisfies non-zero syndrome criteria. Simulation results show that proposed non-zero syndrome PCGC outperforms existing non-zero syndrome PCGC in terms of its bit error rate (BER) performance in Binary Symmetric Channel (BSC). We also show that proposed concatenation method gives better performance than dedicated LDPC of same frame length and code rate over BSC.
Keywords— LDPC; Non-zero syndrome; Parallel concatenation; PCGC; BSC.
I. INTRODUCTION
Low-density parity-check (LDPC) code is a class of linear block codes, first proposed by R. Gallager in 1962 [1]. These codes are defined by sparse parity check matrices. However, these codes were largely forgotten as long block LDPC codes were too complex to be implemented at that time. After the discovery of Turbo codes in 1993 [2], these codes were revisited by D. MacKay and R. Neal in 1995 [13]. LDPC codes are well known for their error correcting performance near to Shannon limit which enables efficient and reliable communication over a noisy channel. Other advantages of LDPC codes includes design flexibility, decoding simplicity and lower error floor compared with Turbo codes.
The encoding complexity of LDPC codes increases quadratically with block length (O(N)2). Also fully parallel LDPC decoder for longer block length suffers from high implementation complexity due to widespread connectivity in bipartite Tanner graph. In order to reduce the encoding and decoding complexity of long block length LDPC codes, Parallel Concatenated Gallager Codes (PCGCs) [3] were introduced by H. Behairy and S. C. Chang in 2000. In Conventional PCGC [3], [4], [5], two component LDPC codes interact in parallel using interleaver-free Turbo code architecture. But Conventional PCGC has not been widely accepted because of its decoding delay and performance limitations. Later, different attempts for parallel concatenation of LDPC codes were introduced which are presented in [6], [7], [8] and [10] and have not been widely successful due to their limitations.

Non-zero syndrome PCGC is introduced in [11], where one component code is designed using conventional zero syndrome criteria and other component code is designed according to non-zero syndrome criteria explained in Section II. The name ‘Non-zero syndrome PCGC’ indicates the significance of using the concept of non-zero syndrome. This is in contrast to all other existing PCGCs explained in [3], [6], [8], [9], [10] and [16], where the concept of zero syndrome is only used. However non-zero syndrome PCGC does not received much attention due to its poor performance compared to a dedicated LDPC of same frame length and code rate. In this paper, a new class of non-zero syndrome PCGC is introduced, which overcome the demerits of existing non-zero syndrome PCGC.
The remaining sections of this paper are structured as follows: In Section II we introduce the concept of non-zero syndrome LDPC. In Section III we describe existing non-zero syndrome PCGC and mention its drawbacks. In Section IV we introduce proposed non-zero syndrome PCGC. In Section V simulation results for the BER performances of proposed nonzero syndrome PCGC in Binary Symmetric Channel (BSC) are presented. Finally Section VI concludes the paper.

III. EXISTING NON-ZERO SYNDROME PCGC
In non-zero syndrome PCGC [11], [15] two different parity sets are transmitted for the same message block. One set of parity bits satisfies the zero syndrome criteria while the other set satisfies the non-zero syndrome criteria. Encoder and concatenated codeword structure for non-zero syndrome PCGC is shown in Fig. 2.

In the next step of super iteration, decoder will perform one local iteration with the received sequence corresponding to non-zero syndrome codeword. Finally, the intrinsic information in the bit nodes are updated based on the extrinsic information available from the check nodes. Bit nodes corresponding to message bits and parity bits are updated separately. If the bit node corresponds to the parity bits, then intrinsic information associated with that bit node is flipped if majority of the extrinsic messages disagrees with the available intrinsic information. Hence for bit nodes which corresponds to parity bits, intrinsic information associated with both the registers are updated separately. But if the bit node corresponds to the systematic message bits, then the intrinsic information associated with that bit node is flipped if majority of combined extrinsic messages disagrees with available intrinsic information. Hence for bit nodes which corresponds to message bits, intrinsic information associated with both the registers will be same after every super iteration. This process of super iteration continues until a valid codeword is obtained or a maximum number of super iterations are over.

IV. PROPOSED NON-ZERO SYNDROME PCGC
One simple solution to make the extrinsic messages independent is to use reversed message block as input to the second component encoder. With this setting, we can make the parity check equations completely different and hence bit nodes will receive totally independent messages, thus increasing the performance.
In proposed non-zero syndrome PCGC, two different parity sets are transmitted. Zero syndrome parity corresponds to original message block while non-zero syndrome parity corresponds to reversed message block. In proposed non-zero syndrome PCGC, reversing the message bits makes the extrinsic messages received by the bit nodes totally independent, thus increasing the iterative decoding performance. Decoding delay for proposed PCGC can be reduced by using two component decoders in parallel, one for zero syndrome codeword and other for non- zero syndrome codeword. Both component decoders can perform local iteration simultaneously. Even though decoding complexity of proposed PCGC is slightly increased due to the incorporation of reverse operation, it can be nullified by the operations performed by additional check nodes and bit nodes in a dedicated LDPC. Also the performance of proposed nonzero syndrome PCGC is much better than existing non-zero syndrome PCGC and dedicated LDPC of same frame length and code rate.

VI. CONCLUSION
In this paper, we have proposed an efficient non-zero syndrome PCGC. We have shown that proposed non-zero syndrome PCGC has better BER performance than existing

non-zero syndrome PCGC over BSC. We have also shown that BER performance of proposed non-zero syndrome PCGC is much better than dedicated LDPC of same code rate and frame length. As future work, we suggest to look into the possibility of extending the proposed non-zero syndrome PCGC over AWGN and fading channels, using soft decision decoding.
