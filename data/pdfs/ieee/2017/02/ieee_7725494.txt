Optimization Techniques for the Efficient Implementation of High-Rate Layered QC-LDPC Decoders
Huang-Chang Lee, Mao-Ruei Li, Jyun-Kai Hu, Po-Chiao Chou, and Yeong-Luh Ueng, Senior Member, IEEE

Abstract— For high-rate low-density parity-check (LDPC) codes, layered decoding processing can be reordered such that the first-in-first-out (FIFO) buffer that stores variable-to-check (V2C) messages is not needed and, hence, the memory area can be minimized, but at the cost of increased data dependency. This paper presents three techniques that can be used to implement an efficient reordered layered decoder. First, with the assistance of a graph coloring method, the required minimum number of V2C sign memory banks can be theoretically determined, with the corresponding pipeline architecture also designed. After that, the integer linear programming technique is adopted so as to arrange the V2C sign memory banks in a manner that minimizes the number of pipeline stalls, thereby increasing throughput. In order to further simplify the decoder, the first minimum values are not stored if the proposed modified min-sum algorithm is used. The proposed techniques are demonstrated by implementing a rate-0.905 (18396,16644) QC-LDPC decoder using 90-nm CMOS technology. When using the proposed techniques, implementation results show that the throughput-to-area ratio (TAR) increases by 58.9% without sacrificing error-rate performance.
Index Terms— Belief propagation, error-control codes, layered decoder, low-density parity-check (LDPC) codes.
I. INTRODUCTION
H IGH code rate and high decoding throughput are desired for both storage and wireline communication applications [1], [2], [31]. These demands can be met using lowdensity parity-check (LDPC) [3], [4] codes, since their inherent structures are suitable for a parallel decoding architecture, and near-capacity performance can be achieved using a simple iterative Belief Propagation (BP) algorithm. In contrast, the implementation of a partially parallel architecture for Quasi-Cyclic (QC)-LDPC codes provides a balance between the throughput and the hardware complexity, and is more attractive for certain applications [5]–[11], [38]–[39].
The decoding messages include variable-to-check (V2C) messages, check-to-variable (C2V) messages, and APP (a posteriori probability) values. Compared to a flooding schedule, which generates new decoding messages based on information from the previous iteration, serial schedules can use the up-to-date information from the current iteration, thereby doubling the speed of the decoding convergence. The Layered BP (LBP) [12]–[14] has become one of the most popular types of serial scheduling. When LBP is applied to QC-LDPC codes, the decoding messages corresponding to the same block row of the parity-check matrix are usually grouped into a single layer, and can be easily combined with a block-row parallel architecture [15], [16].
For each layer, the LBP decoder generates the V2C messages based on the C2V messages from the previous iteration, together with the APP values. If the hardware-friendly minsum algorithm (MSA) [17], [18] is used, the first two minimum magnitudes of these V2C messages can be identified in order to generate the new C2V messages. The APP values are then updated as a combination of the new C2V messages and the V2C messages. Since the V2C messages from a single block row, i.e., a single layer, are used immediately after they are generated, the block-row parallel architecture stores only the APP values and the C2V messages. Using this architecture [15], [16], a high decoding throughput can be achieved, but at the cost of high decoder complexity, especially for long and high-rate codes.
By following the structure of QC-LDPC codes, the messages corresponding to each circulant block can also be processed sequentially, and decoders based on this block parallel architecture have a better area efficiency, but at the cost of a lower decoding throughput [9], [11], [19]–[21]. Multiple clock cycles are required in order to generate the V2C messages and then update the C2V messages for all blocks corresponding to a single layer. Subsequently, the V2C messages and the new C2V messages are combined so as to sequentially update the corresponding APP values for each circulant block. C2V messages and for updating the APP values, and are usually stored in a first-in-first-out (FIFO) buffer, as described in [11] and [22]. Consequently, an additional storage block is added to the block parallel architecture for the V2C messages for a single layer. In most cases, more than half of the silicon area is used for the storage of decoding messages [23].
Many approaches have been proposed that are able to further improve the area efficiency of the block parallel architecture via a reduction in the storage requirements, meaning that the number of quantization bits that are used to represent the decoding messages can also be reduced [9]–[11], [24]. However, this is achieved at the cost of a degradation in the error rate performance. In [25] and [34], the authors proposed removing the FIFO buffer by storing the V2C messages in the same memory blocks as those used for the APP values. This means that a single memory location is used to dynamically store either the V2C message or the APP value. Although the improvement in area efficiency was not significant, since a multi-bank, multi-port memory technique was adopted in order to avoid memory access conflicts, this architecture implies that it is possible to eliminate the FIFO buffer.
In our patent [26], the computations for the decoding messages within a single layer are reordered such that the storage requirements for the APP values can be removed, together with the FIFO buffer. The signs of the V2C messages are stored separately from the V2C magnitudes since they are required for the generation of both the C2V and the V2C messages, and are accessed more frequently. In addition, the V2C magnitudes are stored in the memory for a single layer, while the V2C signs are stored in the memory for all layers. This reordered block-parallel LBP decoder is successful because the results processed for each layer are updated to the V2C magnitudes rather than to the APP values. However, the reordered decoding operations result in a more complex data dependency. A direct solution to this problem is to introduce additional stalls in order to avoid memory access hazards. However, this will decrease the decoding throughput. As a result, although the decoder area is able to be significantly reduced by using the FIFO-free reordered LBP architecture, the throughput-to-area ratio (TAR) will be degraded if the reordered decoder described in [26] is directly implemented.
In this paper, several techniques for minimizing both the area and the number of stalls are proposed in order to increase the TAR. First, a graph coloring method [27] is introduced that is used to theoretically determine the minimum number of banks required for the storage of the V2C signs. Multiple single-port memory banks are required so as to avoid any memory access conflicts that are potentially introduced by frequently accessing the V2C signs. The corresponding pipeline architecture is also designed. We then propose using the integer linear programming (ILP) technique [28], [29] to locate the V2C signs in the memory banks so as to determine a decoding schedule that provides the minimum number of pipeline stalls. In order to further reduce the decoder area, a modified MSA is proposed such that the storage requirement for the first minimum magnitude is removed. Finally, a hardwire implementation of the decoder for a rate-0.905 irregular (18396,16644) QC-LDPC code is completed using a

combination of all three of these techniques. Implementation results based on APR (automatic place and route) show that the TAR can be increased by 58.9% when using the proposed techniques, without sacrificing error-rate performance. In addition, by using the proposed optimized reordered decoder, the TAR can be increased by 43% compared to the conventional layered decoder.
The remainder of this paper is organized as follows. The QC-LDPC codes and the LBP using the MSA are reviewed in Section II. Also included in Section II is the proposed modified MSA. The architecture for both the conventional and the FIFO-free reordered LBP decoder are described in Section III. In Section IV, optimization techniques for the FIFO-free reordered decoder architecture are introduced. Section V describes the implementation based on the proposed techniques together with a comparison of the results. Finally, a conclusion is given in Section VI.

C. Proposed Modified MSA
In order to further reduce the complexity of the C2V storage implementation, a variety of approaches proposed in the previous literature were considered, including the deep-quantization method proposed in [10] and the second minimum approximation methods presented in [36] and [37]. In this paper, a modified MSA is proposed, where the necessity for storing the first minimum value in the C2V memory is completely removed. When the first minimum value is required in the recovery of the C2V message, an approximate value that is deduced from the stored second minimum value is used. When the second minimum value is greater than a pre-determined threshold Tγ , the first minimum value is represented using a small nonzero positive value γ , otherwise, it is equal to zero. In this work, both the first and second minimum values are

quantized using 5 bits, with a step size of 0.5. In other words, four quantization bits are used to represent the integer part, while one quantization bit is used to represent the fractional part. If the threshold for the second minimum value is set as Tγ = 1, the possible selections of γ , i.e., the approximate first minimum value, can be either 0.5 or 1 because Tγ ≥ γ . Since the magnitude of the first minimum value is very small in most cases, any error between the actual first minimum value and the approximate value can be limited.
Fig. 1 shows the bit-error-rate (BER) performance for the rate-0.905 (18396,16644) QC-LDPC code, where each C2V message is represented using a 5-bit quantization. For the proposed MSA, we evaluate the performance for different combinations of Tγ and γ based on the same number of iterations, which is set to 20. It can be seen from Fig. 1 that the BER performance can be improved as both the threshold Tγ and the approximate first minimum γ reduce. For the 5-bit quantization scheme with a step size equal to 0.5, Tγ = 0.5 and γ = 0.5 can be selected as the minimum values, where the best BER performance is also achieved. Although it is possible to further improve the BER performance by using more precise quantization schemes, such as 6-bit quantization where the step size can be set to 0.25, any further reduction in Tγ and γ is not considered in this work due to the consequent degradation in area efficiency introduced as a result of the requirement for additional quantization bits. Therefore, the following discussion is based on the results when using Tγ = 0.5 and γ = 0.5. Also included in Fig. 1 is an illustration of the performance for both the original and the proposed MSAs for a different number of iterations. A degradation in BER performance can be observed if the number of iterations is insufficient, e.g., 8. However, when the number of iterations is sufficient, i.e., 50, the performance of the proposed MSA is very close to that of the original MSA. This means that the speed of the error rate convergence for the proposed MSA is slower compared to that of the original MSA.

III. BLOCK-PARALLEL LBP DECODER ARCHITECTURES
A. Conventional FIFO-Based LBP Decoder Architecture [11], [22]
Fig. 3 shows the architecture for the conventional block parallel LBP decoder presented in [11] and [22]. When a layer of the decoding process begins, the APP values are first read from the APP memory and then the V2C messages are generated based on (2). The magnitudes and signs for the V2C messages are separately stored in the FIFO buffer and the V2C sign memory, respectively. The FIFO buffer contains the V2C magnitudes for a single layer. For each check node, the neighboring V2C magnitudes are compared, and the first two minimum values are stored in the C2V registers, along with the product of all neighboring V2C signs together with the index value for the variable node with the minimum magnitude. According to (3) and (4), the C2V messages can be recovered after combining the values stored in the C2V registers with the corresponding V2C sign. The decoding process for a single layer is completed once the APP values have been updated using the C2V messages based on (5). The updated APP values for all variable nodes are written back to the APP memory.

B. FIFO-free Reordered LBP Decoder Architecture

Since the storage requirements for the decoding messages occupy a significant percentage of the silicon area of the conventional LBP decoder, the decoder area can be efficiently reduced if any unnecessary storage of decoding messages can be avoided. According to (3) and (5), the magnitudes of the V2C messages are respectively required for the generation of both the new C2V messages and the new APP values, and the update of the APP values cannot begin before the new C2V messages are available. Since the calculation of the C2V messages requires details of the magnitudes for all the neighboring V2C messages, a FIFO buffer is required in the conventional LBP decoder in order to store the V2C magnitudes for a single layer. In contrast, when the V2C messages are generated according to (2), only the APP value corresponding to the same variable node is required. This implies that it may not be necessary to retain the APP values for all variable nodes, meaning that it is possible to remove the APP memory.
As mentioned in Section II-A, the decoding process for each layer begins by reading all the APP values, and ends when the updated APP values are written back to the memory. In other words, the APP values are used as an ‘accumulator’ of the decoding results for each layer. If the APP memory is removed, the V2C messages may be used to assume the role of the ‘accumulator’. In order to remove the APP memory, the decoding operations implemented in the FIFO-free LBP decoder are reordered, and the V2C magnitudes are stored in the V2C memory rather than in the FIFO buffer. When the decoding process for the i -th layer of the k-th iteration begins, the computation of the V2C messages from the previous layer, i.e., the (i −1)-th layer are completed, then the C2V messages corresponding to the (i − 1)-th block row can be evaluated as:

Since the APP values can be eliminated after the V2C messages are updated, and since the decoding messages that correspond to each block column are generated individually, a pipeline register is sufficient for temporarily storing the APP values.
As can be seen from Fig. 4, the reordering operation is only applied to the blocks (circulant sub-matrices) for a single layer, thereby introducing a different layer boundary. Therefore, the reordered layered decoding can be regarded as an “operation-delayed” version of the conventional layered decoding method. The reordered layered decoding method consists of identical decoding operations and generates the same decoding messages compared to the conventional LBP decoding approach. As a result, the error-rate performance will not be affected by the reordered operations.
IV. OPTIMIZATION TECHNIQUES FOR RE-ORDERED LBP DECODERS
It is worth noting that the V2C sign memory of the reordered LBP decoder is not only accessed in order to write the new signs when the V2C messages are updated, but also for reading the corresponding V2C signs when recovering the C2V message. It can be expected that access to the V2C sign memory will occur more frequently than other memory modules. In order to avoid the possibility of the reordered LBP decoder being affected by access conflicts, the V2C signs are separately stored in multiple memory banks, where the minimum number of banks required can be theoretically evaluated using the graph coloring method. From Fig. 4, it can be perceived that the pipeline structure needs be redesigned after the FIFO buffer is removed. If the pipeline structure is directly implemented based on the original arrangement of the V2C sign memory, a large number of additional stalls must be inserted in order to prevent memory access hazards, and the decoding throughput will be slowed. The ILP approach is adopted so as to minimize the number of pipeline stalls, thus

However, the stalls cannot be arbitrarily removed. The V2C signs accessed by task t4,5 must be also moved to bank M1 based on the constraint that the V2C signs from the same block column must be stored in the same bank. Now, both tasks t4,4 and t4,5 access the V2C signs from bank M1 during the same clock cycle, which results in a memory conflict. As a result, additional stalls need to be inserted.
Since it has been established that the V2C signs from the same block column must be stored in the same bank, a number of stalls still exist that can be intuitively removed. For the final layer, task t4,8 can be executed during the second clock cycle, before t4,4, t4,5 and t4,6 are processed, meaning that three additional stalls can be avoided. V. PERFORMANCE EVALUATION
A. Implementation Results for the Proposed Decoder
In this work, a FIFO-free reordered decoder was implemented for a rate-0.905 irregular (18396,16644) QC-LDPC code using 90-nm technology. In order to boost the throughput, decoding messages from three circulant sub-matrices are processed in parallel. Since the stalls cannot be totally removed, 98 clock cycles are required in order to complete a single iteration, which is slightly more than E B/3 = 82. The number of quantization bits for the APP values, V2C magnitudes and C2V messages are denoted as qAPP, qV2C, and qC2V respectively, where qAPP = 6, qV2C = 5, and qC2V = 5 are used in the decoder implementation. Fig. 1 shows the associated fixed-point error-rate results for the implementation of the decoder, where Tγ = 0.5 and γ = 0.5 are adopted. Compared to the floating-point results which are not included in Fig. 1, the proposed quantization scheme introduces a degradation of 0.1 dB.
In order to achieve the desired throughput, the C2V storage is implemented using shift registers. Fig. 14(a) shows the architecture for the C2V storage used in a conventional decoder, where a 2-to-1 multiplexer is appended to each D flip-flop. This is because multiple clock cycles are required in order to complete the processing of a single layer, but all the values are only shifted to the right registers during the final clock cycle, otherwise the C2V messages must be retained in the same register during the decoding process for the same layer.
Considering the large number of multiplexers used in a conventional decoder, a gated-clock based architecture that

can be realized using a normal synthesis tool is used in the proposed decoder. The resultant architecture is shown in Fig. 14(b), where the appended multiplexers for all the D flip-flops are removed, and a single multiplexer is added in order to gate the clock that triggers the shift registers. The multiplexer is controlled using the same signal as was used in the original design, and data shifting can only occur during the final clock cycle for each layer. The flip-flop-based C2V memory consists of registers containing 21204 bits and occupies an area of 0.26 mm2.
In order to balance the memory bandwidth and area, both the V2C magnitude memory and V2C sign memory are implemented using register files (RFs) generated by the memory compiler. As the size of the RFs obtained from the memory generator is limited, most of the storage blocks are realized as a combination of multiple RFs. In the implementation for the proposed decoder, the V2C sign memory consists of fifteen 128×36 single-port RFs and three 36×36 single-port RFs and occupies an area of 0.44 mm2. In addition, the V2C magnitude memory consists of 35 122 × 22 two-port RFs and occupies an area of 0.95 mm2.

VI. CONCLUSION
This paper describes a FIFO-free reordered LBP decoder where the number of stalls required is minimized. Initially, the number of memory banks required for storing the V2C sign can be minimized to three with the assistance of a graph coloring method in combination with an additional pipeline stage. Then the ILP technique is introduced and is used to determine the schedule for the decoding tasks, as well as the bank arrangement for the V2C sign memory, such that the required number of stalls can be minimized and a higher decoding throughput can be achieved. In order to further reduce the decoder area, the first minimum values are not stored if the proposed modified min-sum algorithm is used. Compared to the LDPC decoders presented in previous literature, the proposed LBP decoder provides the best hardware efficiency in terms of TAR.
