LDPC Coded Modulation for TLC Flash Memory
Huang-Chang Lee∗, Jieng-Heng Shy†, Yen-Ming Chen3, and Yeong-Luh Ueng†§, ∗Dept. of Electrical Engineering, Chang Gung University, Taoyuan City, 33302, Taiwan.
†Dept. of Electrical Engineering, National Tsing Hua University, Hsinchu, Taiwan. ‡Institute of Communications Engineering, National Sun Yat-sen University, Kaohsiung, Taiwan.

Abstract—In this paper, a coded modulation scheme using extremely sparse low-density parity-check (LDPC) codes is proposed for the stored signal of triple-level-cell (TLC) NAND flash, where both the encoding and the decoding complexity can be significantly reduced with the advantage of the extremely sparse code graph. In order to enhance the performance of decoder, iterative detection decoding (IDD) is introduced to extract the extrinsic information from the symbol detector, and the cooperative non-Gray mapping is also designed. In addition, for error floor lowering, an interleaver is inserted to ensure the cascaded degree-2 variable nodes are separated to individual symbols. The simulation results show that the proposed coded modulation scheme can provide a practical error floor performance with a low decoding complexity.
Index Terms—Coded modulation, LDPC codes, TLC flash memory.
I. INTRODUCTION
NAND flash memory has been widely used as non-volatile memory. In order to increase the density of data storage, the technique adopted in the flash memory has progressed from single-level-cell (SLC) to triple-level-cell (TLS), where the number of bits stored in each cell increase from one bit to three bits. However, more errors can be introduced during the write/read operations as the density of the storage increase, the conventional correcting codes, such as BCH codes [1] and Reed-Solomon (RS) codes, are no longer capable of providing a reliable storage. Recently, with the advantage of near-capacity error control performance, low-density paritycheck (LDPC) codes [2][3][4][9] have been introduced as an error-correcting code for flash memory.
The voltage level stored in the cell of TLC flash memory can be regarded as a 8-PAM (pulse-amplitude modulation) signal. Ungerboeck mapping [5] is a popular symbol labeling scheme for coded modulation, but it is more suitable for the codes with clearly defined distance property. When the LDPC code is adopted in the conventional coded modulation, Gray mapping is the most intuitive symbol labeling, and the cooperated LDPC codes are designed independently. In this work, iterative detection decoding (IDD) is introduced in the coded modulation receiver. The symbol detection of the outer iteration loop is expected to provide extra information for the LDPC decoder of the inner iteration loop and improve the

Density evolution (DE) [6] is a powerful analytical tool to characterize the performance of LDPC codes in waterfall region, and can be used to determine the degree distribution of LDPC codes. The original density evolution is designed for BPSK in symmetric channel, and the performance estimation can be completed based on the case that only bits 0’s are transmitted. For the IDD scheme, the LDPC code must be designed to cooperated with the symbol mapping. Refer to the modified DE for asymmetric channel [8], the effect of the proposed 8-PAM mapping can be considered in the code design flow. The p.d.f. (probability density function) of both 1s and 0s transmitted bits are considered, and the p.d.f. of each received 8-PAM symbol is generated as the summation of the distribution of channel value and the a priori p.d.f.s from associated variable nodes, i.e., corresponding mapping bits. With the assistance of asymmetric density evolution, the LDPC codes that are suitable to cooperate with the Gray, Ungerboeck, and the proposed mappings can all be identified.
Comparing to the LDPC code designed to cooperate with the Gray mapping, the codes designed for the proposed mapping has has a much lower degree density. In other words, there are only few nonzero elements in the parity-check matrix, and the LDPC code adopted in the proposed coded modulation scheme has an extremely sparse code graph. Since the complexity of both the encoding and decoding of LDPC code are proportional to the number of nonzero elements in the parity-check matrix, the proposed coded modulation scheme can significantly simplify the implementation of both the encoder and the decoder.
The extremely sparse code graph of the proposed LDPC code implies that there are a large number of degree-2 variable nodes. According to our experiments, there is a high probability that these degree-2 variable nodes are connected in a sequence and from a 2-chain or a 2-loops. When these connected degree-2 variable nodes are mapping to the same symbol, the trapping sets can be easily triggered and the error floor rises. An interleaver is inserted between the symbol detection and and the decoder, where the degree-2 variable nodes are separated to individual symbols. With the assistance of the interleaver, the error floor can be significantly lowered,

IV. LOWER ERROR FLOOR VIA INTERLEAVER
When the proposed mapping of Table I is combined with the designed LDPC code in Table II, error floor can be observed before FER is about 10−6. The reason is that there are too many degree-2 variable nodes in the proposed LDPC codes, and many previous literatures point out the degree-2 variable nodes may degrade the error floor performance. Based on the observation, an interleaver is inserted between the LDPC decoder and the symbol detector, as shown in Fig. 2. The interleaver is designed to separate the degree-2 variable nodes connected in the same 2-chain or the same 2-cycle into distinct symbols. The interleavers corresponding to the Gray and the Ungerboeck mapping are also designed and inserted. However, for Gray mapping, there are no extra extrinsic information provided by the detector, and the effect of the interleaver is hardly observed. For the Ungerboeck mapping, the number of degree-2 variable nodes is not the majority of the parity-check matrix, and the interleaver cannot efficiently improve the error floor performance.

V. SIMULATION RESULTS
Fig. 5 show the improvement in the frame error rate (FER) performance of the proposed coded modulation scheme after the interleaver is inserted, where the error floor can be significantly lowered. The code length is n = 9216, and the degree distribution is listed in the third column of Table. II.
The FER performance of the coded modulation scheme using different mappings are compared in Fig. 6 and Fig. 7. The code length are selected as n = 9216 and n = 18432 in Fig. 6 and Fig. 7, respectively, and individual LDPC codes are designed for Gray, Ungerboeck and the proposed mappings using the degree distribution listed in Table. II. It can be observed that, for LDPC codes with both two different lengths, the coded modulation schemes using the Gray mapping have the lowest water fall region. It is consistance to the results

shown in Table. II, where the code designed for Gray mapping can achieve the lowest decoding threshold. It also implies that the asymmetric DE adopted in this work is effective.
Comparing to the one using Gray mapping, the coded modulation using the proposed mapping has slightly higher decoding threshold. However, for the application of the flash memory storage, the error floor performance is more important than the decoding threshold. With the assistance of interleaver, more than one order improvement in error-floor performance can be achieved using the coded modulation with the proposed mapping. Although the interleavers are also inserted to the coded modulation with Gray and Ungerboeck mapping, the error floor lowering effect is not significant due to that there are only few degree-2 variable nodes in the code graphs for these two mapping.

VI. CONCLUSION A design flow of the coded modulation scheme for TLC flash memory has been proposed in this work. The mapping of the 8-PAM signal is first determined, and then the corresponding LDPC code can be designed using asymmetric DE. Finally, the degree-2 variable nodes that connected in code graph are separated into individual symbols by an interleaver. The coded modulation scheme designed using this flow can provide practical error floor performance, and it also significantly reduce the implementation complexity of both encoder and decoder because of the extremely sparse LDPC code graph.
