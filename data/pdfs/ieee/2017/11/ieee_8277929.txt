On Practical LDPC Code Construction for NAND Flash Applications
Shiuan-Hao Kuo, Zhen-U Liu, Jeff Yang Silicon Motion, Inc.
Hsinchu County, Taiwan 30265 Email: {shiuanhao.kuo,leonhard.liu,jeff.yang}@siliconmotion.com

Abstract‚ÄĒAs increasing storage density accompanies increasing adoption of 3D NAND flash, the data integrity turns to more important. The error floor problem is known to have critical impact on the correction capability of LDPC code in NAND flash storage. Herein we propose an analytical method based on a simplified NAND flash error model to relate harmful trapping sets to small cycles, by which we can avoid certain cycle combination during code construction for NAND flash memory applications.
I. INTRODUCTION
With growing needs of massive data storage in various modern technologies, the low-density parity-check (LDPC) code is widely applied to the NAND flash memory devices to meet the rigorous demand on error correction capability. LDPC code provides capacity approaching performance with simple iterative belief-propagation decoder such as the sum-product algorithm, the min-sum (MS) algorithm and the bit-flipping (BF) algorithm. However, a sudden change of the slope of the error rate in high SNR region, much slower than which in the low-to-moderate SNR region, has been observed that for most LDPC codes decoded with belief propagation algorithm. Such phenomena, known as error-floor, is extremely unfavourable in storage systems.
The error floor occurs when belief propagation decoder gets trapped in a non-codeword steady state, which is related to a problematic structure of the Tanner graph of the LDPC codes. For additive white Gaussian noise (AWGN) channel, it is identified as trapping sets [1]. It has been proved that most error patterns in error floor region correspond to a subset of trapping sets, named elementary trapping sets [1]. A particular type of trapping set that is granteed stable under BF decoder is introduced as absorbing sets [2]. A lot of research effort has been focused on finding trapping sets [3] [4] [5]. Subsequent works provide methods such as importance sampling [1] [6] or linear model [7] [8] to estimate the error floor behavior based on trapping sets.
In [9] [10], the necessary condition and an exhaustive search algorithm to increase girth are studied. Usually, for NAND flash applications, high rate quasi-cyclic (QC) LDPC codes with rate around 0.9 and small column weight (i.e. variable node degree) are employed. In the most of the cases of 1KByte and 2K-Byte codes, the size of circulant permutation matrix (CPM) (also known as the lifting degree) cannot be made enough to be free of small cycles [9]. Therefore, con-

struction of good LDPC code with low error floor may rely on accurately avoiding harmful trapping sets or combinations of small cycles. The characterization of the structure of trapping sets have been studied in [11], [7] for column weight 3 and 6 respectively, and controlling cycles to remove trapping sets for column weight 3 and girth g = 8 LDPC code is studied in [12]. In this work, we further characterize the harmful trapping sets by the structure of connected cycle-6s. In this way, good codes can simply be selected by controlling certain connection between cycle-6s. We focus our attention on the trapping sets in the LDPC codes constructed based on RS code [13].
The method consists of the following steps. First, the RSbased LDPC codes are constructed at random and their cycle6 connections are profiled. Next, we analyze the harmful trapping set of each codes under high reliability error (HRE) model [14]. Finally, we provide an evaluation of good codes and bad codes that can be useful either in code construction or for error-floor analysis.

A (a, b) trapping set is a subgraph of a LDPC code‚Äôs Tanner graph that contains a variable nodes and all their associate check nodes, in which b check nodes have odd degree. These odd degree check nodes are called unsatisfied check nodes. On the other hand, the even degree check nodes are satisfied check nodes. An elementary trapping set (ETS) is a trapping set with all the check nodes having degree less than or equal to two. For example, a cycle-6 is a (3, 3(dv ‚ąí 2)) elementary trapping set. An absorbing set is a trapping set with more satisfied check nodes than unsatisfied check nodes.
The HRE model for NAND flash can be expressed as r = (‚ąí1)u‚äēe + n, where u is the transmitted binary signal, e is HRE and n is AWGN. The HRE is a flip bit error occurred while copying data form a SLC block to a TLC block without decode and re-encode, or the cell defect such that it cannot be erased entirely or cannot be programmed. III. CYCLE-6 AND TRAPPING SETS
A. The Connection of Cycle-6
Let C1 = {c1,0, c1,1, ¬∑ ¬∑ ¬∑ , c1,5} and C2 = {c2,0, c2,0, ¬∑ ¬∑ ¬∑ , c2,5} be two cycle-6. The possible relation of C1 and C2 is one of the following.
(a) one common node: c1,m = c2,n for some m and n. In this case, it is easy to prove using eqn. (4) that the set C1 \ {c1,m} ‚ą™ C2 \ {c2,n} will form a cycle-10.
(b) one common edge: c1,m = c2,n and c1,(m+1) mod 6 = c2,(n+1) mod 6 (we will omit ‚ÄĚmod 6‚ÄĚ in the following). Similar to the previous case, it is easy to prove a new cycle-8 is formed.
(c) two consecutive common edges: c1,m = c2,n, c1,(m+1) = c2,(n+1) and c1,(m+2) = c2,(n+2). In this case, there will be a new cycle-6 C3 = C1 \ {c1,m, c1,m+1, c1,m+2}‚ą™ C2 \ {c2,n, c2,n+1, c2,n+2}.
(d) two opposite common edges: c1,m = c2,n, c1,(m+1) = c2,(n+1) and c1,(m+3) = c2,(n+3), c1,(m+4) = c2,(n+4). In this case the connection of cycles is depend on the size of the CPM.
(e) three consecutive common edges: a new cycle-4 will be induced. Since the RS-based LDPC codes have girth g ‚Č• 6, this case does not exist.
For regular (dv, dc) LDPC codes with dv = 4 or 5, the most harmful connections between cycle-6 is the case that two cycles share a common horizontal edge (CHE), where the horizontal edge is defined as {c2m = (i2m, j2m), c2m+1 = (i2m, j2m+1)}, 0 ‚Č§ m ‚Č§ k ‚ąí 1. In particular, it indicates that two variable nodes are connected by the same check node. We can also define the vertical edge analogously. When two cycle6 share a CHE, each variable node on this edge is connected to at least three (which is more than dv/2) ‚ÄĚcorrupted‚ÄĚ check nodes that is in a cycle-6. This phenomena can be found in case (b), (c) and (d) of the cycle-6 relations.

B. ETS Category
The experiment is conducted using min-sum based decoder with HRE channel model. The ETS is collected when the decoder fails at its maximum iteration. Although the resultant ETS is not comprehensive, it covers the most of stable ETS and their frequency in direct proportion. We characterize the ETS with their cycle-6 prime structure. An ETS can be expansions of smaller ETS as shown in [4]. If an ETS is an expansion of a prime structure that does not involve new cycle6, then it belongs to the category of the prime structure.
To further interpret the relation between ETS and cycle-6, we extend the (a, b) ETS expression to (a, a , e1, e2, b), where a ‚Č§ a is the number of variable nodes that are associate to a cycle-6, e1 is the number of satisfied check nodes that are associate to a cycle-6, and e2 is the number of satisfied check nodes that are not in any cycle-6. If e1 + e2 ‚Č• b, then the ETS is an absorbing set. For dv = 4 the most harmful ETS is four cycle-6 connecting to each other and form a (4, 4, 6, 0, 4) ETS, which is discussed in [15]. Fig. 2(a) and Fig. 2(b) shows the cycle-6 connection and the Tanner graph of (4, 4, 6, 0, 4) ETS, the equivalent expression that hides check nodes (CN) is shown in Fig. 2(c). This structure is the smallest absorbing set [16] for dv = 4. For dv = 5, the smallest absorbing set has the same structure but one more unsatisfied check node for each variable node and thus become (4, 4, 6, 0, 8).
The ETS induced by HRE simulation on randomly generated RS-LDPC code are very different when the code contains (4, 4, 6, 0, 4) ETS or not. V. CONCLUSIONS
In this work, we establish an analytical method that uses HRE channel model for NAND flash to identify harmful ETS and their connection to cycle-6. The result is fed-back to the code construction where the cycle connection is easy to identify. We exam the result using RS-based LDPC codes with dv = 4 and dv = 5. By identifying the cycle-6 and their connection on the base matrix during code construction, it is able to avoid or control the existence of several harmful ETS. This technique is useful in lowering error floor or in selecting LDPC codes with improved error floor performance.
