Abstract—The existing LDPC encoder exists the problem of high computing complexity and much resources consumption, therefore, a strategy of LDPC fast encoding based on dynamic memory structure of linked list queue is proposed in this paper. Specifically, this strategy simplifies the design of encoder by making full use of the characteristics of the parity check matrix structure and sparsity. On the basis of this, the hardware structure of the quasi loop and the matrix vector multiplier of the barrel shifter are designed. Moreover, the FPGA implementation of the LDPC encoder is completed. Finally, we use Verilog language to implement the algorithm and XILINX ISE 14.7 as the functional simulation tool to simulate the timing of different code rate. And simulation results show that this method can achieve a linear relationship between the encoding complexity and the code length, effectively improve the encoding speed and reduce the amount of logic resources.
Keywords—LDPC code; Encoder; Matrix- vector multiplier
I. INTRODUCTION
In 1962, Gallager [1] proposed the LDPC (Low Density Parity Check Code) code, which is a linear block code with excellent performance. In addition, the performance of LDPC is very close to the Shannon [2] limit and it has strong error correction. In particular, quasi-cyclic LDPC codes [3,4] have a good application prospect and receive more and more academic attention.
In the coding algorithm of LDPC, there are three encoding algorithms. The first one is the traditional coding algorithm based on Gaussian elimination, in which the amount of computation and storage is huge. Besides, the hardware implementation is also very difficult. The second one is the RU (Richardson and Urbanke) coding algorithm. This coding algorithm only suitable for the situation of low throughput requirements, not too long code length and no adaptive coding demand [5]. The third one is the fast coding algorithm. The LDPC codes in the IEEE802.16e and in the DVB-S2 standard both adopt the fast coding algorithm [6-10]. The coding complexity of this algorithm is linear with the code length, and

it has the advantages of high coding speed, less resource occupation, and easy hardware implementation.
In the FPGA implementation of LDPC encoder, the improved RU coding algorithm and fast coding algorithm are the mainstream of thought now, the reference [11-13] are using improved RU encoding algorithm; the reference [14] employs fast encoding algorithm, and the traditional coding algorithm is used in [15,16]. In applications, LDPC code has begun to gradually apply to space communications, optical fiber communications, digital video, audio broadcasting and other fields. The main coding standards of the LDPC codes are DVB-S (Digital Video Broadcasting-Satellite) standard, DVBS2 standard, CCSDS (Consultative Committee for Space Data Systems) standard, GB20600 standard, IEEE802.16e standard and so on. However, the problems of high complexity calculation and more resource occupation in hardware implementation still exist. Therefore, a fast LDPC strategy based on linked list-based dynamic memory structure is proposed in this paper. We have verified the effectiveness of the method through the design and FPGA implementation of the LDPC encoder.

III. IMPLEMENTATION OF FPGA BASED ON FAST ENCODING ALGORITHM
The complexity of hardware implementation of LDPC encoder is an important index to evaluate whether it is practical. This paper focuses on the optimization of hardware implementation of fast iterative coding algorithm. The encoder design includes an input and output conversion module, a core coding module and a control module. The function of the input and output conversion module is to buffer the input and output data, and to operate the input and output data corresponding to the parallel serial conversion. The core coding module uses the stored base parity check matrix to dispose the input information bits, which includes the process of calculating the output parity bit under the operation of the loop, XOR, etc. The main function of the control module is to coordinate the connection between the memory matrix module ROM and each sub-module and to generate the corresponding control clock for each part of the system. Since the analysis above, core coding module is the focus of the design.

A. Design of Core Coding Module Structure
Core coding module structure is shown as Fig.1. It consists of the matrix multiplication module, code word generation module, vector addition module and cache module etc. The matrix multiplication module can finish the cyclic shift of the information vector; Code word generation module will realize the function that the check bits from the encoder and the input

Fig. 1. Core encoder block diagram
FPGA fast encoder works as follows: The first step initializes the value of the corresponding module to zero and changes the input information into parallel information s through the input conversion module; In the second step, s1 is multiplied by q(i,1) ≠ −1 in the first column of Hb , and the result of the multiplication is simultaneously transferred to the p1 module and the λi module, and then s1 is output. In the third step, s2 is multiplied by q(i, 2) ≠ −1 in the second column of Hb , and the result of the multiplication is simultaneously transferred to the p1 module and the λi module, and finally s2 is output. Until step kb +1 multiplies skb with the kb columns q(i, kb ) ≠ −1 in Hb and transfers the results to both the p1 module and λi module, and finally skb is output. From the second to kb +1 step, one clock cycle is consumed at a time, totally occupy kb clock cycle. The generation module p1 of kb + 2 step will generate the first parity bit p1 . All the values of λi are already produced and stored in the register set, at the same time the output terminal will output p1 . In the subsequent mb −1 clock cycle, by sending the value of p1 and the corresponding λi to the pi generating module, the following p= [p1 p2 ... pm] check bits can be generated and output according to the above equation (9). From the logical queue flow in Reference 14, it is easy to see that the queue storage information is fixed and limited to LDPC codes with a code rate of 1/2. In order to make the storage unit to achieve the insertion and deletion operations to adapt to the LDPC coding at different code rates, proposing a chain-based queue dynamic memory structure, as shown in Fig. 2.
The linked list memory is implemented by a dual-port memory. The linked list memory stores the link pointer of the linked list node and control information. The control information includes the frame length (column length of base matrix) and the start address of the transmission frame. The linked list memory needs only to store the frame length, The start address is obtained by the frame length and column number. Head and tail pointer memory stores queue head and tail pointer that is usually called the list node pointer; storage

address of head and tail pointer register corresponds to the queue number, a queue has a unique head and tail pointer register address. The sending queue length counter is used to record the length of the linked list. Once a frame data is written, the queue counter is incremented by 1. The counter decrements by 1 for each successful transmission of one frame of data. The dynamic memory structure of the queue based on linked list solves the problem of fixed column storage in matrix. Some nodes can be inserted, deleted and changed at will, so it only need to update the node information when switching LDPC codes of other coding rate. At the same time, the flexibility of linked list memory can be very good to avoid spending a lot of storage space on -1 elements in matrix. In addition, register is called quickly and orderly, which corresponds to the logical queue flow implemented by the fast coding algorithm, and uses only two sets of registers in the whole process, which greatly reduces the storage space overhead.

Compared with the reference [14], this design uses a list based on dynamic memory structure of the queue. after the compilation, the logic resources occupancy of the encoder is significantly less. Therefore, this method reduce the amount of logic resources. From the maximum clock frequency, the speed of the encoder is obviously faster in this paper.

Compared with the reference[14], this design uses the serial-parallel conversion module to process the data, which obviously improves the coding speed. LDPC encoder in[15,16] is designed on the basis of the traditional encoding algorithm, that is, the generated matrix is transformed by the check matrix firstly, and then the generated code is multiplied by the input information vector and the generated matrix. The complexity of the method is proportional to the square of the code length. While in this paper, the hardware encoder designed by fast coding algorithm use the parity check matrix to code directly, in which coding complexity and code length are linear relationship.

Conclusions
In this paper, we adopt the fast coding method in IEEE802.16e standard to design the LDPC encoder based on

linked queue dynamic memory structure, and finish the FPGA implementation of LDPC encoder. Finally, the coding speed is effectively improved and the occupancy rate of logic resources is reduced extremely. we carry out the LDPC coding design and analysis based on the code length and rate adaptive adjustment in the next step.
