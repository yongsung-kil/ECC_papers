Abstract—Low-density
parity-check
convolutional
codes
(LDPC-CCs)
are
generally
decoded
using
sliding-window
based message passing decoding. Based on the sliding-window
decoding, an informed dynamic scheduling (IDS) for LDPC-CC
is proposed in this work, where the decoding convergence can
be significantly accelerated. Since the number of processors
required for a satisfactory performance can be reduced, the
decoder can be simplified. A set of rate-compatible (RC)
puncturing patterns is also proposed and is used to construct
RC LDPC-CCs for the performance evaluation of the proposed
IDS. Although the proposed puncturing pattern cannot be
optimized for individual code rate owing to the rate-compatible
constraint, they can still provide a comparable error rate
performance compared to the codes defined in the IEEE
802.16m standard. It is worth noting that these standard codes
are not rate-compatible.
Index Terms— Low-density parity-check convolutional
codes (LDPC-CCs), informed dynamic scheduling (IDS), rate-

I. INTRODUCTION
Both low-density parity-check block codes (LDPC-BCs)
[1][2] and LDPC convolutional codes (LDPC-CCs) [3] have
received great research interests owing to their near-capacity
performance. For LDPC-BCs, multiple iterations for the mes-
sage passing decoding are required in order to achieve a
satisfactory error-rate performance, and the speed of the de-
coding convergence is related to the schedules of the messages
updating sequence. The informed dynamic scheduling (IDS)
has been used to accelerate the decoding convergence for
LDPC-BCs [4], and a variety of scheduling algorithms have
been proposed. However, the IDS is rarely applied to LDPC-
CCs.
For LDPC-BCs, one of the benefit of using IDS is the
significant reduction in the required number of iterations for
a satisfactory error-rate performance. When IDS is applied
to LDPC-BCs, a specific metric is used to measure the
importance of the decoding messages, and only the important
decoding messages will be updated. The comparison of the
metric can take place among the entire code graph, and the

updating of the decoding messages does not follow a fixed
order. The decoding messages that are regarded as more im-
portant can be updated more frequently and hence the updated
number of each message can be different. Consequently, one
iteration is counted when the number of updated messages is
equal to the number of edges in the code graph [4].
For LDPC-CCs, the decoding is generally implemented us-
ing a sliding window based message-passing decoding, several
(base) processors are cascaded to form a pipeline architecture,
and each processor is responsible for the decoding operation
of a single iteration [5][6]. Using this pipeline architecture, a
high-throughput decoder can be realized. Since the code length
of an LDPC-CC is not fixed, the boundary of code graphs
cannot be clearly specified, and only the decoding messages
included in the sliding window are processed. The selection
of the decoding messages is limited owing to the sliding-
window architecture. Since the comparison among the whole
code graph is impossible, the decoding messages cannot be
updated anymore after they are shifted out the sliding window.
Therefore, most of the IDS strategies proposed for LDPC-BCs
can hardly directly be applied to LDPC-CCs.
In this work, an IDS algorithm that can cooperate with the
sliding-window decoding is proposed. Based on the sliding-
window decoding, each processor contains only part of the
parity-check matrix, only few variable nodes have the chance
to propagate their information to all the other variable nodes in
the same processor, and the decoding updates are dynamically
selected from the messages associated to these variable nodes.
When the sliding window shifts, the partial parity-check matrix
corresponding to each processor is changed as well. As a
result, a different set of variable nodes can be selected to
propagate their channel messages. It can be expected that as
the sliding window shifts, all variable nodes will have an
equal chance to propagate their information and the silent
variable nodes [4] will not exist. In order to update sufficient
amount of decoding messages without increasing the number
of processors, multiple decoding messages are allowed to
be updated along with the most important one after each
comparison. Therefore, a more flexible trade off between the
error-rate performance and the decoding complexity can be
provided.

dedicated codes with different rates. Compared to LDPC-BCs,
LDPC-CCs have the advantage of variable-length data frames
and are more suitable to provide flexible code-rate through
puncturing [10]. An example is shown in the the proposal for
the IEEE 802.16m standard [11], where a set of puncturing
patterns are applied to a rate-1/2 mother code and the codes
with rates ranging from 5/6 to 1/2 are constructed. However,
the puncturing patterns are not rate-compatible (RC).
Besides the IDS for LDPC-CCs, a set of puncturing pat-
terns with RC capability is also proposed in this work. For
performance evaluation, the proposed IDS is applied to the
RC-LDPC-CC constructing using the proposed puncturing
patterns. The rate-1/2 LDPC-CC from the IEEE 802.16m
standard is adopted as the mother code. In order to achieve
RC capability, the design freedom of the proposed patterns
is limited compared to the original one [11] which does not
support the RC feature. Surprisingly, a comparable error-rate
performance can be still achieved for individual code rate with
the assistance of the proposed IDS. Take the advantage of
the RC capability provided by the proposed patterns, a higher
throughput is expected using the combination of the proposed
IDS and the proposed puncturing patterns in an HARQ system.

III. PROPOSED IDS FOR RC-LDPC-CC CODES
A. Proposed IDS
The proposed IDS inherits the conventional sliding-window
architecture, but we have to design a new processor for
dynamic scheduling. When the IDS decoder with three proces-
sors is applied to the example code, i.e., the (5,3,6) LDPC-CC,

In Fig. 3, the decoding graph corresponding to the new
processor is shown, where the variable nodes and check nodes
are depicted as circles and squares, respectively. It can be
observed that, there are 6 check nodes contained in a single
processor, and the neighboring variable nodes of all these
check nodes are also included. At each processing cycle, there
is a set of variable nodes connected to all check nodes. These
variable nodes of the selected set have the chance to propagate
their decoding messages to all the other variable nodes through
the check nodes in the processor. The selected variable nodes
will shift following the sliding window, then different variable
nodes are selected in distinct processing cycles. Following the
procedure, all variable nodes will have an equal chance to
contribute their intrinsic information to the decoding process,
and the silent variable nodes [4] can be avoided.
In Fig. 3, the variable nodes of the selected sets are
painted in gray, and the solid lines correspond to the candidate
C2V messages that contains the intrinsic information of these
selected variable nodes. When the proposed IDS is applied to
LDPC-CCs, the residual defined in (3) is used as the metric for
the importance of the C2V messages, and only the messages
with the maximum residual can be updated. After the update,
the sliding window will shift, and another set of variable nodes
are selected to propagate their intrinsic information.
When the proposed IDS is applied to the example (5,3,6)
LDPC-CC or the (491, 3, 6) LDPC-CC from the IEEE
802.16m standard, since only part of the decoding messages
are selected to be updates, the total number of updated
messages is only 1/3 compared to that of the conventional
decoders. Therefore, more processors are required for the
IDS decoder in order to provide the same number of updates
comparing to the conventional decoders.
Fig. 4 shows the BER performance when the proposed IDS
decoder and the conventional layered belief propagation (LBP)
decoder [10] are applied to the rate-1/2 (491, 3, 6) LDPC-CC
from the IEEE 802.16m standard. The rate-1/2 code is used
as the mother code, and the codes with rates ranging from
2/3 to 5/6 are derived using the puncturing patterns from
[11]. Also included in Fig. 4 are the simulation results of
these punctured codes. The default number of updated C2V
messages associated to each variable node of the selected set is
one. Consequently, when the LBP decoder has 5 processors,
there will be 15 processors in the proposed IDS decoder in
order to provide the same number of updates. According to
the simulation results shown in Fig. 4, the proposed IDS can
achieve a much better BER performance compared to the
LBP, where more than 0.5 dB gain can be observed from all
codes with different rates. The most significant improvement
is achieved for the rate-5/6 code, where about 2 dB gain is
provided when the BER is about 10−6. However, all these
improvements are achieved when the same number of updates

C. Proposed RC LDPC-CC using IDS
Compared to the original puncturing patterns that are
obtained by independently optimizing each code rate, the
proposed patterns suffer from more limitation, and the BER
performance degradation is expected if the conventional LBP
is used. In this subsection, we investigate the performance of
the proposed LDPC-CC using the IDS.
Fig. 6 shows the corresponding BER performance, where
the proposed 2-update IDS is applied to the proposed RC-
LPDC-CC. Also included in Fig. 6 are the BER results for
the original LDPC-CCs using either LBP or IDS. It can be
observed that, when using the LBP, the proposed puncturing
patterns introduce a significant BER performance degradation.
The reason is that the optimization freedom of the proposed
puncturing patterns is limited by the rate compatible constraint.
In contrast, the degradation in BER performance can be com-
pensated when the proposed IDS is used. The most significant
improvement in BER performance happens when the proposed
IDS is applied to the rate-5/6 code, where an 1.5 dB SNR gain
can be achieved when the BER is around 10−6. Since the
code with highest rate dominates the throughput in an HARQ
system, this SNR gain is expected to improve the throughput
performance for HARQ systems.
It is worth noting that, when using the 2-update IDS
decoder, the number of updated decoding messages is only 2/3
compared to that of the LBP decoder. Therefore, it is possible
for the IDS decoder to simplify the decoding complexity since
less updates for the decoding messages are required.
IV. CONCLUSION
In this work, an IDS for LDPC-CCs is proposed, along with
a set of rate-compatible puncturing patterns. The combination
of the proposed IDS and the proposed puncturing patterns
can provide a similar BER performance and can reduce the
number of updated decoding messages to 2/3 compared to
the combination of the convolutional LBP and the original
standard codes without RC capability.
