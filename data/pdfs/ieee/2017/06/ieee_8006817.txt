Abstract‚ÄĒA concatenated soft-decision channel coding scheme consisting of an inner LDGM code and an outer staircase code is proposed. The soft-decision LDGM code is used for error reduction while the majority of bit errors are corrected by the low complexity hard-decision staircase code. Decoding complexity of the concatenated code is quantified by a score based on the number of edges in the LDGM code Tanner graph, the number of decoding iterations, and the number of staircase code decoding operations. The inner LDGM ensemble is designed by solving an optimization problem, which minimizes the product of the average node degree and an estimate of the required number of decoding iterations. A search procedure is used to find the inner and outer code pair with lowest complexity. The design procedure results in a Pareto-frontier characterization of the trade-off between net coding-gain and complexity for the concatenated code. Simulations of code designs at rate 5/6 show that the proposed scheme achieves net coding-gains equivalent to existing soft-decision codes, with up to 57% reduction in complexity.
I. INTRODUCTION
The‚ąó design of channel coding schemes for long-haul optical transport-networks with soft-decision channel outputs is an active area of research. Proposed solutions include: turbo product codes [3], quasi-cyclic low-density parity-check (LDPC) codes [4]‚Äď[7], and spatially-coupled (SC) LDPC codes [8]‚Äď[10]. A rate 5/6 convolutional LDPC code achieved a net coding gain (see definition in Sec. VI) of 11.5 dB at an output bit error rate (BER) of 10‚ąí15 [8]. A rate 200/251 SCLDPC code achieved a net coding-gain of 12.0 dB at the same output BER [9].
However, compared to hard-decision coding schemes, softdecision coding schemes all require more complex decoders. The increased complexity leads to greater data flow (operations per decoded bit), power consumption, and heat generation. In addition, soft-decision schemes often have error floors above the 10‚ąí15 output BER required by optical transport standards, which must be reduced by post-processing [6] or concatenating with an outer code [4], [9].
In this paper, we propose a concatenated coding scheme consisting of an inner iterative soft-decision decoded lowdensity generator-matrix (LDGM) code and an outer iterative

In contrast to conventional concatenated coding schemes with a powerful inner code and a relatively simple outer code, [4], [7], we rely on the outer staircase code to correct the majority of errors. The inner LDGM code is used for error reduction, i.e., to sufficiently reduce the bit error rate without necessarily correcting any complete error pattern. This combination allows the powerful and low-complexity staircase code to be adapted to the softdecision channel, while incurring relatively small decoding complexity.
We describe a design method that minimizes the decoding complexity of the concatenated scheme for a given channel condition (or equivalently, net coding-gain). Assuming sufficient interleaving between inner and outer codes, the combinatorial analysis described in [11] can be used to determine the error floor of the concatenated scheme, requiring only simple software simulations of the inner code.
We focus on the quadrature phase-shift keying (QPSK) modulated system with a Gray-labelled constellation and normalized per-dimension average energy. We assume an additive white Gaussian noise (AWGN) channel with noise covariance matrix ŌÉ2I, where I is the 2 √ó 2 identity matrix. The signal to noise ratio (SNR) of the equivalent channel, in decibels, is denoted by Es/N0 ‚ąí10 log10 ŌÉ2.

III. STAIRCASE CODES
A staircase code [11] is constructed from a binary linear block code C(nc, kc), referred to as the component code. We denote the component code rate by Rc kc/nc, its systematic generator matrix by Gc, and its unique decoding radius by tc. We also denote the number of parity bits in the component code by rc nc ‚ąí kc.
Let M nc/2. The staircase code structure consists of a semi-infinite chain of blocks Bi, i ‚ąą N (see [Fig. 4] [1]). Each staircase block contains M 2 bits. Encoding begins by filling block B0 with all-zeros. For i ‚Č• 1, first fill the M √ó (M ‚ąí rc) matrix Mi with information bits. The matrix [BiT‚ąí1| Mi] is formed and each row of the matrix is encoded by the systematic generator matrix Gc. The parity bits from the encoding are stored in the M √ó rc matrix Pi. Finally, the staircase block Bi is formed by the concatenation of matrices Bi = [Mi| Pi]. The rate of a staircase code, Rsc, is given by Rsc = 2Rc ‚ąí 1 = 1 ‚ąí rc/M . Note that to have a positive rate, the component code must satisfy Rc > 1/2.
Decoding is performed by an iterative sliding window decoder containing W received blocks. At each iteration, for each i ‚ąą {0, . . . , W ‚ąí 2} the decoder forms the matrix [BiT | Bi+1] and decodes each row of the matrix by using the component code decoder. Decoding continues until the maximum number of iterations, Isc, has been reached or no more corrections are made by any component decoder. The window then slides by shifting out the decoded block B0 and shifting in the newly received block BW . The process continues indefinitely.

A. Pareto Optimal LDGM Design
Given Rcat and a staircase code, the LDGM ensemble must satisfy the rate constraint Rcat = RRsc and the maximum output BER constraint psc.
Let ő∑i EI/K = EI/(kRsc). The code design problem is defined as a constrained, multi-objective optimization with the objectives (Es/N0, ő∑i). We characterize the trade-off between these objectives by finding their Pareto frontier. For a given SNR, we find the LDGM ensemble (if it exists) of rate R, with the minimum ő∑i and maximum output BER psc. We note that the Pareto frontier of soft-decision LDPC codes was previously studied in [21]. However, our paper significantly improves on the scalability and efficiency of the optimization in [21].

B. Concatenated Code Optimization
We use a simple search procedure to optimize the concatenated code. The inputs to the procedure are Rcat and a table of staircase codes, such as Table I. The search procedure consists of 3 nested loops: LOOPi for i ‚ąą {1, 2, 3}. LOOP1 iterates over a discrete set of channel SNRs Es/N0. For each channel SNR, LOOP2 iterates over the entries in the staircase code table. Recall that each staircase code specifies Rsc and psc. Lastly, for each combination of channel SNR and staircase code, LOOP3 iterates over a discrete set of values for őĪ.
In the innermost LOOP3, we perform the LDGM ensemble optimization as described in V-A, which requires the values of Dc, őľňÜ, and U . Dc can be chosen freely. The value of őľňÜ is determined by psc [1].
The value of U is difficult to determine prior to the LDGM optimization. We use a binary search to find the smallest U such that the optimization has a feasible solution. Let ULi and UUi be the lower and upper limits on U i, the value of U at binary search iteration i. An LDGM optimization is performed using U i at each iteration. If a feasible solution is found, we set UUi+1 = U i. Otherwise, we set ULi+1 = U i.
At the end of an iteration of LOOP3, a feasible solution will produce the outputs ŌĀ(x) and J. At the end of LOOP3, we calculate the complexity scores ő∑i = EI/K ‚Čą őĪJ/Rsc based on the output J values, for each őĪ. The ŌĀ(x) that corresponds to the smallest ő∑i then specifies the minimum complexity LDGM ensemble for the staircase code specified in an iteration of LOOP2. At the end of LOOP2, we have obtained the LDGM ensemble and staircase code combination with the minimum ő∑ for a given channel SNR.

The following parameters were used: Dc = 100, Q = 80, channel SNRs Es/N0 for ŌÉ ‚ąą {0.480, 0.485, . . . , 0.520}, and őĪ ‚ąą {2.0, 2.1, . . . , 6.0}. BP decoding was used in LDGM ensemble design. We simulated LDGM codes sampled from the ensembles given in Table II. For ensembles designed for Es/N0 ‚Č• 2.92 dB (ŌÉ ‚Č§ 0.505), we sampled code graphs with k = 20000 and m = 2500. In Fig. 3, we compare the NCG and ő∑ of example concatenated code designs at rate 5/6, under BP and MS LDGM decoding, with several existing soft-decision codes [4]‚Äď[8]. The Pareto frontiers for BP and MS concatenated code designs are shown, along with simulated performances of sample codes. All of the referenced codes were decoded by an MS decoder with either a flooding or layered decoding schedule [10], [22].
We first compare with referenced codes decoded under a flooding schedule. The MS-decoded concatenated code design at ő∑ = 62 with an NCG of 11.3 dB achieves a 0.5 dB gain over the code in [4]. Similarly, it achieves a 0.3 dB gain over the code in [7]. It achieves the same NCG as the code in [6], while reducing ő∑ by 45%. Finally, MS-decoded concatenated codes achieves the same NCGs as codes in [8] (at 11.15 dB, for example) while reducing ő∑ by up to 57%.
Compared to the codes decoded under a layered schedule, the MS-based concatenated code designs show an improvement in ő∑ over the codes in [5] of up to 25% at the same NCG. They do not achieve the same NCG as the codes in [8]. This is due to the fact that as defined, ő∑ assumes a highly-parallel flooding decoding schedule (see Remark 1). Under a layered decoding schedule [10], [22], subsets of messages are updated sequentially. The decoder requires fewer overall iterations, resulting in a smaller ő∑. However, the loss of parallelism may increase decoding latency, which is not measured by ő∑. Hence, a more encompassing complexity score is required to make fair comparisons between these schemes.
