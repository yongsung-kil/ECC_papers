Design of LDPC Decoder Based on Adaptive Code Length

Bo Chen, Congcong Chen, Xiuli Du, Kangte Zhu, Shiming Li and Yuanyuan Guo
College of Information Engineering Dalian University
Dalian, Liaoning Province, China chenbo@dlu.edu.cn

Abstract²LDPC still receives extensive attention in the field of channel coding because of its superior error correction performance, high throughput and low decoding complexity. Considering the poor adaptability and limited expansibility of traditional LDPC decoder, a design method of LLR-BP decoder based on adaptive code length is proposed. A partial parallel LDPC decoder structure is designed under the ISE 14.7 environment, which adopts the pre-termination iteration strategy and effectively reduces the iteration number of it. And this decoder realizes the complex probability function operation by taking the limit in LLR-BP algorithm that can reduce the complexity of algorithm. Through simulation analysis, the decoder effectively improves the decoding speed, reduces the hardware logic resource share, and solves the decoding problem of multi-code length simultaneously in the premise of ensuring the bit error rate.
Keywords-LLR-BP decoder; multi-code length; adaptive; Partial parallel
I. INTRODUCTION
Low density parity check codes (LDPC) is a kind of linear block codes with sparse parity check matrix proposed by Robert G. Gallager in 1962 [1]. It not only has a good performance of Shannon limit, but also has low decoding complexity and flexible structure. It is still a hotspot in the field of channel coding in recent years, and has a good application prospect in mobile communication. However, with the increase of code length, the dimension of LDPC decoding matrix greatly increases, the complexity of the algorithm becomes extremely lager, and the hardware logic resource of decoding is occupied more. In recent years, domestic and foreign scholars research mostly in-depth from the hard and soft decision algorithm and LDPC decoder or other aspects.
A. Hard decision algorithm
The LDPC decoding algorithm based on hard decision has the advantages of simple operation and convenient hardware implementation, but its performance is poor. The main algorithm: Bit flip algorithm (BF) is proposed by Gallager when proposing LDPC code decoding algorithm [1]. Weighted Bit Flipping method (WBF) based on soft information is proposed by Y. gou, which utilizes some reliable information of the received symbol sequence to obtain better performance than simple hard decision algorithm. Z Liu proposes LP-WBF algorithm, which is close to the BP algorithm in the error correction performance by combining the reception symbol reliability, the introduction of loop detection and the bit reversal constraint mechanism, however it still has large complexity [3]. Ming Jiang proposes an improved modified weight BF decoding algorithm (IMWBF) to solve the problem that the reliable information of the calibration equation is the symbol value of itself, and provide a new weight calculation method based on KLF-WBF algorithm [4]. To achieve the optimal balance between performance and complexity, Ruan proposes JPWBF algorithm based on BF algorithm where the probability calculation value in the BP class algorithm is introduced into the BF class algorithm as soft information, which improves the reliability of reversal bit decision [6].
B. Soft decision algorithm
The LDPC decoding algorithm based on soft decision is better than hard decision in the bit error rate(BER), but it is not suitable for hardware implementation. Though Gallager creatively proposes the belief propagation (BP) algorithm, its complexity is very large [7]. The LLR-BP algorithm describes the probability information of the BP algorithm with the likelihood ratio, therefore a large number of multiplication operations become addition operations, reducing the operation time. Yuan Jianguo proposes a method to improve the LLR-BP algorithm by weakening the external information oscillation through introducing the weighting factor in the case that the external information oscillation of the variable node does not converge in the process of iterative decoding [8]. Weiberg N proposes the min-sum decoding algorithm (Min-Sum) for the problem that the decoding efficiency of LLR-BP decoding algorithm is not ideal, and the node updating in the iterative process is simpler on the premise of lower performance [9]. To simplify the hyperbolic tangent operation in the verification node part of LLR-BP decoding algorithm, Fossorier proposes a normalized minimum algorithm (Normalized Min-Sum) [10]. C. LDPC decoder
There are three kind of FPGA implementation structures of LDPC decoder: Full Parallel Structure, Serial Structure and Partial Parallel Structure [14]. As Partial Parallel Structure can quickly achieve decoding, but also save the advantages of hardware resources, it is widely adopted. Aiming at the characteristics of QC-LDPC code check matrix and the characteristics of BP algorithm, Zhang Lin improves the algorithm for hardware resource storage and data precision processing [15]. According to the BP decoding algorithm, Li Zhenhui designs a large-scale parallel QC-LDPC decoder structure [16]. In the design, a large number of complex function operations in BP decoding algorithm are realized by look-up table method. In accordance with the characteristics of no rate code decoding, Lin Li selects the appropriate design scheme to implement the BP decoder on FPGA and improve throughput [17]. Peter Hailes proposes a parameterized LDPC decoder structure in view of the low flexibility of LDPC code, meanwhile he proposes an off-line design flow to automatically generate optimized hardware description language and to support selecting codes [18]. Taking the limitations of hardware resources and power consumption into account, Mahdie Kiaee proposes a high throughput and product algorithm partial parallel LDPC decoder based on the time scheduling principle [19].
On the one hand, LDPC code performance is usually better when the code length becomes longer. But the bit rate is invariable and the check matrix H structure is fixed, so it is difficult to decode the multi code length at the same time. On the other hand, the improvement of decoding algorithm is based on the BP algorithm, but LLR-BP has the best BER performance. In view of this, in order to overcome the poor adaptability and limited expansion capability of the traditional LDPC decoder, a LLR-BP decoder design method based on adaptive code length is proposed in this paper.

III. THE DESIGN OF LDPC DECODER
This decoder adopts Top-Down design. Firstly, the whole structure of the decoder is designed, and then the sub modules are designed.

A. The Structure of Decoder
In order to realize the decoding of different code lengths, the 1/2 bit rate is used as an example, and the partially parallel structure is adopted. The decoder mainly includes the adaptive initial information cache module, check node process module (Check Node Processor, CNP), intermediate information storage module (Mess_Mem), variable node process module (Variable Node Processor, VNP), the control module (Control), the output cache module, decoding module etc., The block diagram of the partial parallel decoder is shown in fig. 1.

The check matrix H is divided into several sub blocks. In the block, all check nodes share a single CNP module, and all variable nodes share a single VNP module. A serial decoding is performed within the block, and a full parallel decoding is performed between the block and the block. The buffer module can automatically adjust the size of the storage matrix according to the length of the code.
The channel information is quantized and then stored into the initial data buffer module, and the control module sends out an instruction CNP_ON, and the decoding starts. The VNP module and the CNP module alternately perform the calculation of all variable nodes and check nodes under the control of the control module. VNP and CNP read the information from the intermediate information memory and rewrite the node update results back to the intermediate

information storage module. Because VNP and CNP work alternately, the design can arrange the access reasonably through the address control module, and there is no access conflict problem. All data processing data of this decoder is quantized by 12 bit, and the highest bit is the symbol bit.

B. Adaptive Initial Information Cache Module
The adaptive initial information cache module is used for receiving the data to be decoded from the AWGN channel where the module stores the initial data after the data is quantized. The buffer module is a 12*24 memory matrix. And the size of each storage matrix z * z is determined by the code length n, z n / 24 .In view of this feature, the module automatically adjusts the storage matrix neutron square Pij (i indicates the line, j indicates the column)according to the code length. Assuming the data to be decoded is [v1,v2«,vn-1,vn], n bits to be decoded are stored in the parity check matrix H in the order of column, where the value of Pij is 1 ( i presents row, j presents column), but Pij needn¶t store data, whose value is 0.
The read and write addresses of CNP can be in the order of 0~23. Channel information to be decoded are floating-point type, which need amplified after quantization stored in hexadecimal, the bit-width is 12 bit, and the highest bit stands for the sign bit. For example, Decimal number 0.52427 will be binary number 0.10000110001 with the accuracy of 2-11 , and then we will get 12'h431 through the shift register for calculation convenience.

C. Check Node Processor Module (CNP)
The check node process module bears the check node information updating function, and the degree of parallelism of the CNP is 12, wherein the parallel degree of the CNP is set according to the 1/2 bit rate of the parity check matrix in the IEEE 802.16e standard. The node updating operation is carried out after it read data from Pij, and the data is updated according to the row of each bit node of the parity check matrix H . The data is read from the Pij to perform the check node update operation, and the row of each bit node is updated in the order of row of bit node in check matrix H. The data in Pij is stored in column order to provide convenience for the read and write address sequence output of CNP. D. Variable Node Processor Module(VPN)
The VNP module starts working after the control module sends a begging signal VNP_ON (active high) of the VNP, and the VNP module reads the channel quantization information to be decoded from the initial information buffer ROM. The reads and writes addresses of VNP output in accordance with rem ((P(i,j)), +n, 24), n = 0,1,2... 23 of the sequential, P(i,j) is the submatrix of Pij. The variable node begins updating if rst =1¶b0 or VNP_ON=1¶b0. rji is read from the RAM in intermediate information storage module according to the (6). And the parallelism of the VPN is 24.
E. The Control Module
The Control module not only manages all the modules, but also provides read and write address signals for the storage module as the decoder core. The control module completes the function of controlling system, including the control of the iteration times. According to the finish signal donea1 (doneb0) of VNP (CNP), it decides whether the CNP module or VNP module will work in the next state. Success=1¶b0 stands for decoding success, which can end the iteration ahead and output the decoding result. Last_Iter (active high) presents the last iteration period, then the decoding ends. The Control module is implemented through Verilog HDL hardware language.
F. Decoding Decision Module
The decoding decision module receives Result_out from the output buffer module (Result_Mem), and decides whether the decoding is successful or not by the judging formula HcÖT 0 . Every bit in each row of the matrix H will AND with the decoding result cÖT after iterations, then the OR logic will be operated on what we got. Lastly the result of each row is implemented by the OR logic. The decoding fails and the next iteration will continue, if the result is 1.

In order to verify the performance of the decoder, this paper adopts Xilinx Virtex7 development board as decoder hardware verification platform, using Verilog HDL hardware description language programming. Functional simulation, timing verification and comprehensive analysis are carried out under the ISE14.7. Simulation conditions: IEEE802.16e standard and its support code length, AWGN channel, BPSK modulation, bit rate 1/2, binary LDPC code, signal to noise ratio (SNR) ranges from 0dB to 5dB.

V. CONCLUSIONS
In view of the shortcomings of traditional LDPC decoders with poor adaptability and limited scalability, a partial adaptive LDPC decoder structure with code length adaptive is proposed based on the LLR-BP algorithm, which can realize the decoding of the medium and long code. The iterative process adopts the early termination strategy to effectively reduce the number of iterations of the algorithm. At the same time, the complex probabilistic function of the LLR-BP decoding algorithm is realized by taking the limit method. The decoder perfectly completes the LLR-BP algorithm decoding
