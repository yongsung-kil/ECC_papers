Ultra High Throughput Unrolled Layered Architecture for QC-LDPC Decoders

Oana Boncalo University Politehnica Timisoara
Timisoara, Romania Email: oana.boncalo@cs.upt.ro

Alexandru Amaricai University Politehnica Timisoara
Timisoara, Romania Email: amaricai@cs.upt.ro

Abstract—This paper proposes a layered decoder architecture for array QC-LDPC codes which targets tens of Gbps data rates. It relies on layer unrolling with pipeline stages in between layers, allowing simultaneous decoding of multiple layers. The most important features of the proposed decoder are: (i) fully parallel processing units within each layer (ii) hardwired layer interconnect that allows the removal of high cost variable shift units, (iii) A posteriori log-likelihood ratio (AP-LLR) message memory type of storage is replaced by inbetween layers pipeline registers, as the messages are being forwarded from one layer to the next. Data dependencies is this masively parallel structure is resolved by decoding multicodewords at time. Hence, the proposed architecture allows optimum throughput/cost ratio. FPGA based implementation results indicate that for a 1296 bits LDPC code with 3 layers, throughput of up to 62 Gbps for an average of 4 iterations is by obtained using the proposed architecture.
Keywords-FPGA; LDPC decoder; Layered scheduling;
I. INTRODUCTION
Future wireless communication systems will require increased throughput, with data rates in the order of tens or hundreds of giga-bits per second (Gbps). Due to their high parallelism degree, Low Density Parity Check (LDPC) codes are suitable candidates for ensuring the forward error correction (FEC) of future communication standards. Another emerging application domain of LDPC codes is modern NAND based Flash storage drives. A more than 10x increased lifespan of these devices [1] has been reported due to these FEC scheme. Both multi-Gbps wireless communication and solid state drives FEC systems share two important requirements: high coding rates and high data throughput.
LDPC codes are a class of linear algebraic codes, with channel approaching capability, when large codewords are used (sizes of several thousands or ten of thousands of bits for a codeword) [2]. The architecture space area allows a variety of choices of the number of processing nodes and of the interconnection routes, leading to intricate architectures that offer different trade-offs between the area and the throughput outcomes.
Quasi Cyclic LDPC (QC-LDPC) codes are a sub-class of structured LDPC codes, suitable for hardware implementations [3]. For this sub-class, two main types of scheduling exist: layered and flooding. The layered scheduling has two important advantages with respect to the flooding: increased

convergence – less iterations for the decoding process –, and less memory requirements. [4].
This work proposes the development of ultra high throughput layered LDPC architectures, by applying two simple, but effective techniques: loop unrolling and pipelining. Unrolling is applied at layer level – layer unrolling– , while pipeline stages are inserted between layers. We optimize the interconnect for each layer, by hard wiring it to the required shift amount; thus, we remove the variable length shifting networks such as the typical barrel shifters. Regarding the memory organization, it is built out of shiftregisters for the check node messages, and from flip-flops, that make out the in between-layers pipeline registers for the AP-LLR messages. Due to pipelining, multiple codewords can be processed simultaneously. Hence, the throughput is increased by of factor equal to the number of layers. This architecture is appropriate for codes with small number of layers, which have zero or a very small number of nonzero elements on each row of the code base matrix. This is the case for the majority of the high rate standard codes. This requirement ensures efficient usage of in-between layer pipeline registers, as the majority of the buffered AP-LLR messages will be used by the processing units associated with the corresponding layer.
The remaining of this paper is organized as follows: Section II offers a short overview of QC-LDPC layered decoding principle, Section III discuses the main layered decoding architecture trade-offs with respect to prior art, Section IV presents the proposed decoder, and results, and Section V provides some closing remarks.
II. LAYERED DECODING: NOTATIONS AND OVERVIEW
QC-LDPC codes are a class of LDPC codes obtained by expanding the base matrix B by an expansion factor z [3]. The LDPC code has a length of N = z × ncol, with ncol denoting the number of columns of B. The parity check matrix H can be obtained in the following way: every element of B is replaced by a square matrix that is either the zero matrix for −1 entries of B, or the identity shifted by the non-negative entry value of B. An equivalent view of the H matrix is the graph representation referred to as the Tanner graph. It is a bipartite graphs, with two types of nodes, interconnected by edges. Lines from H, denoted

III. LAYER DECODER ARCHITECTURE TRADE-OFFS
The main metrics characterizing decoder performance are: (i) throughput – computed based on iteration latency and number of iterations; (ii) area – result of code length, parallelism choice, and memory organization; (iii) decoding performance – influenced by code parameters, choice of code rate, as well as message quantization. This work focuses on iteration latency reduction by efficiently increasing parallelization, in order to achieve multi-Gbps throughput.
The architecture parallelism has three dimensions: number of processing units, number of messages processed by each unit at a time, and number of layers processed. The number of processing units is selected either in relation to the circulant size z (i.e. expansion factor is a multiple of parallelization of the decoder), or with respect to the number of layers. For the first case, all rows within a layer are processed in parallel, while in the second case, a row within every layer is processed at a time. At the processing units

The last dimension, has been proposed by [9], and exploited for a special code structure by authors from [10]. The general case, from [9], is to process k layers in parallel, and add their contribution by some extra hardware and routing logic in case there are overlapping AP-LLRs. Essentially for k = 1 layer, we have the conventional layered scheduling, while for k = nl layers, we have the equivalent of flooding scheduling. The approach from [9] considers fully parallel unit processing, agues the general case of K layers, but restricts the implementation to k = 2 layers. Analysis has shown a degradation of decoding performance, consequence of the fact that for the k layers processed together the APLLR values are the same, hence no new information is propagated in between them. The work from [10] speculates on the B matrix structure, and considers the case when all k layers have no overlapping elements. In this case, all AP-LLR values are updated, same as the case of a singlelayer array code processing. Hence, no decoding degradation appears.
The last parallelism knob for reducing decoding latency, refers to the iteration level parallelism (i.e. the number of iterations executed in parallel). Each iteration is processed by separate hardware units, with buffers in-between. Iteration unrolling has been applied for flooded architectures in [11] [12]. Furthermore, in order to decrease the impact of the area overhead, while maintaining the error correction capability, the authors use variable message quantization: higher quantization is used in the middle stages, while the initial and final stages have lower quantization values. Synthesis results for ASIC technology for a 672-bit LDPC code indicates very high cost, but also throughputs of 100+ Gbps. In [12], the authors use dedicated look-up tables for VN computations in order to reduce the number of average iteration number and improve performance.
Since, LDPC design is memory centric, another critical

architecture decision to be carefully weighted is the memory building block. For designs with large memory bandwidth requirements, the memory aspect ratio becomes a limiting factor for SRAM usage, and register based storage is preferred. This is also the case of this work, where we target z × nl fully processing units. In terms of parallelism at processing unit level it is comparable to a fully flooded decoder implementation (i.e. a one-to-one correspondence of the Tanner graph and the hardware realization).
IV. UNROLLED LAYERED DECODER ARCHITECTURE
A. Proposed Architecture
The proposed architecture targets high throughput by means of layered based unrolling in the hardware LDPC decoder implementation. The idea is depicted in Fig. 1. Thus, one layer processing takes 1 clock cycle. Therefore, the iteration duration for one codeword is equal to nl clock cycles. Although higher working frequencies may be obtained if pipelining is applied inside layers, it has the drawback of data hazards between layers [8]. The data hazards require the introduction of stall cycles, which end up canceling the frequency increase benefits. In order to compensate this latency and efficiently use each hardware layer, we employ multiple codeword processing, same as for the iteration unrolled flooded decoder from [11]. The number of simultaneously processed codewords is equal to nl - see Fig. 3. The main building blocks for the decoder from Fig. 1 are:
Layer block: has z merged variable check node units (VCN), each with its own local memory for storing the β messages required for each codeword. It is implemented using dedicated shift register (SRL), and has a depth of nl entries. SRL data is shifted in First-in First-out order. Updated values are written at the end of the SRL, while currently processed values are read from the front. In order to save space, we use a compressed beta format. Control logic: is trivial. The state for each codeword is signaled using two flags: active (A), and done (D). These are propagated in the pipeline.
Routing: is made of simple hard-wired connections from one layer to the next, which arrange data at the inputs of the VCN units based on the B shift values. New codewords are loaded from the interface for layer 0 processing. For this purpose an array of 2-to-1 multiplexers selects between the LLR input values, and the output of the last layer buffer.

For MS algorithm, the VN computation is a simple subtraction. dc subtracters are needed for obtaining dc α messages in parallel. The compressed β message needs to be expanded to sign-magnitude (SM) messages, that are converted to 2’s complement by the SM-C2 block. The output of VN, is then transformed in SM format for CN processing. The min1, and min2 finders, are implemented by a comparator tree similar to the one from [13]. The last step is to add the α value to the newly obtained β value, and compute the updated AP-LLR value. The resolution for the α SM values is of q − 2 bits. The memory efficient SCMS version further requires the re-computation of β message signs from the α signs, the approximate evaluation of the erasure information based on the magnitude of the first minimum, as well as the cancellation of the α message.

B. Results
The base matrix used for implementing the architecture is described in Table II. It is an array code corresponding to a regular, rate 3/4, dv = 3 code, with code size N = 1296 bits. The code from Table II has girth (i.e. size of the shortest cycle in the Tanner graph) equal to 8, and the girth multiplicity 31428. It has 3 layers; hence, 3 codewords can be decoded at the same time. The iteration latency for one codeword is 3 cycles, while for the case of multiplecodewords is 1 clock cycle. Therefore, the iteration latency is comparable with the case of fully parallel flooded decoders, when 2 codewords are decoded at once.
Simulation performance curves for MS and SCMS with no explicit erasure information storage (denoted by SCMS:v2) are presented in Fig. 5 for several quantization values. The notation MS (4,6) refers to Min-Sum algorithm, with q = 6 bits for storing AP-LLRs, and 4 bits for storing CN messages. Three different fixed quantizations have been considered for MS and two for SCMS:v2 algorithm. Results from Fig. 6 suggest that the average number of iterations is smaller than 4 for a FER of less than 10−4.
Five LDPC decoders have been synthesized: three MS based decoders with quantizations (2, 4), (3, 5), (4, 6), and two SCMS:V2 decoders with quantizations (2, 4) and (3, 5). The synthesis has been performed for Xilinx Virtex-7 xc7vx485t device, speed grade -2, optimized for speed.
Since, we unroll 3 layers, and z is 108, a total of 3 × 108 VCN units are needed. The beta compressed message word size is denoted by qβ = dc + log2(dc) + 2 × (q − 3), with q − 2 bits needed for representing one individual β message. For AP-LLR quantization of q = 6 bits, this yields qβ = 22 bits. Note, that this number is significant smaller that dc × (q − 2) = 48 bits, corresponding to the case when the messages are represented in uncompressed format. The overall storage requirements for q = 6 are: AP-LLR storage – 108 × 24 × 3 × 6 = 23328 bits, check-to-variable β storage – 108 × qβ × 3 × 3 = 21384 bits. Although, at first glance, the layer unroll appears to consume 3 × the resources needed by conventional layer decoder, FPGA synthesis results have shown that at least some of them are masked by the FPGA underlying organization: 1 slice has 4 LUT-FF pairs, and the flexibility of Xilinx FPGA architectures that allow the look-up table (LUT) element configuration as logic, ROM/RAM, or SRL. The LUT SRL configuration allows a 4 bit shift register mapped inside a LUT. This implies that the 108 × qβ storage bits needed for the check-to-variable node messages storage corresponding to a layer, are mapped inside 3/4 × 22 × 108 =2376 LUTs. This result is consistent with the synthesis Slice LUTs uses as memory reported in the synthesis log file. V. CONCLUSION
In this paper, we propose an FPGA based very high throughput layered LDPC decoder that is based on the layer unrolling. It is suitable for high data rates LDPC codes, which have full or almost full layers. Pipelining is applied between the unrolled layers. For efficient resources utilization and throughput maximization, multiple codewords are decoded simultaneously. The results for a 1296 code with 3 layers revealed throughputs over 30 Gbps.
The proposed decoder presents a very good throughput/resource usage ratio. A similar has been obtained in iteration unrolled flooded architectures [11]. With respect to the flooded approach, the layer organization provides incentives for efficient memory organization, increased convergence, and natural layer unrolling.
