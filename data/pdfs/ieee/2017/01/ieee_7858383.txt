Improving LDPC Performance Via Asymmetric Sensing Level Placement on Flash Memory

Qiao Li§, Liang Shi§∗, Chun Jason Xue‡, Qingfeng Zhuge§, and Edwin H.-M. Sha§ §College of Computer Science, Chongqing University
‡Department of Computer Science, City University of Hong Kong

Abstract— Flash memory development through technology scaling and bit density has significant impact on the reliability of flash cells. Hence strong error correction code (ECC) schemes are highly recommended. With a strong error correction capability, low-density-parity code (LDPC) is now applied for the state-of-the-art flash memory. However, LDPC has long decoding latency when the raw bit error rates (RBER) are high. This is because it needs finegrained soft sensing between states to iteratively decode the raw data. In this work, we propose a smart sensing level placement scheme to reduce the LDPC decoding latency. The basic idea for the placement scheme is motivated by two asymmetric error characteristics of flash memory: the asymmetric errors at different states, and the asymmetric errors caused by voltage left-shifts and right-shifts. With understanding of these two types of error characteristics, the sensing levels are smartly placed to achieve reduced sensing levels while maintaining the error correction capability of LDPC. Experiment analysis shows that the proposed scheme achieves significant performance improvement.
I. Introduction
Continual technology scaling down, coupled with bit density increasing, has induced rising reliability issue for the state-of-the-art flash memory [1]. Strong error correction code (ECC) schemes are indispensable in ensuring the data integrity [2] [3] [4] [5] [6]. With a strong error correction capability, low-density parity code (LDPC) has been widely used for the state-of-the-art flash memory during the last decade [2] [3]. Different from harddecision code schemes, such as RS-codes and BCH, LDPC uses soft-decision log-likelihood ratio (LLR) information to achieve a high error correction capability. However, the challenge for LDPC is that its decoding latency is significantly increased for data with high raw bit error rates (RBER). The correction capability of LDPC is highly correlated with its input reliability information. The more accurate the input reliability information is, the more bit

errors can be corrected. However, for data with high RBER, acquiring high-accurate input reliability information is time consuming. This is because the adjacent states of memory cells have little noise margin or even overlapped for data with high RBER. In this case, LDPC needs to use fine-grained soft sensing scheme to iteratively acquire the reliability information. For example, for 2-bit per cell flash memory, the maximally used sensing levels are 21, 7 for each pair of adjacent states. In this work, we propose an approach to reduce the decoding latency of LDPC through reducing the number of sensing levels without sacrificing its error correction capability.
Several previous works have proposed to reduce LDPC sensing voltage levels on flash memory [2] [7] [8]. Dong et al. [7] proposed a nonuniform memory sensing strategy. Their design is motivated by the observation that the most overlaps between two adjacent cell states occur closely to the corresponding hard-decision reference voltage (i.e., the boundary of two adjacent states). With this observation, they proposed to sense the overlap regions with a much higher accuracy by placing more sensing levels. Zhao et al. [2] proposed to perform read-retry steps by starting from the lowest-level precision, and then adding extra sensing levels. This method outperforms the twostep decoding scheme [7] by not conducting the highest level but the suitable levels when it fails at hard-decision decoding. REAL [8] is designed to improve the input reliability information by taking the bit patterns into account. For example, for a 2-bit per cell, if the first bit is 1, the second bit can be 1 with a higher probability. With these information, the accuracy of the input reliability information can be increased. All of these works apply symmetric sensing levels between adjacent states, without taking the asymmetric state error characteristics of flash memory cells into consideration.
This work proposes a smart sensing level placement scheme to reduce LDPC decoding latency through exploiting error characteristics on flash memory. Specifically, errors of flash memory have two distinctive characteristics on cell states: inter-state asymmetric and intrastate asymmetric. Inter-state asymmetric errors are resulted from the asymmetric threshold voltage distributions for different flash cell states. For example, for 2-bit

Intrastate asymmetric errors are resulted from the asymmetric voltage shifts for each cell state. The voltage of cell states may shift in two directions: left-shift and rightshift, which are induced by different error sources. • Proposed an inter-state asymmetric sensing level placement scheme to reduce the sensing levels with awareness of the error characteristics of different states;

• Conducted experiments to verify the effectiveness of the proposed techniques. Experimental results show that our approach can achieve great reduction on the decoding latency of LDPC.

II. Background and Related Work
A. Basics of Flash Memory
Flash memory stores data by trapping charges in floating gate of memory cells. For the flash memory with n bits per cell, the data are presented by 2n different voltage states. Flash write operation is the process to inject charges to floating gates, to move the threshold voltage of each cell into one of 2n non-overlapping storage levels that are apart from each other with certain noise margin. However, flash memory is prone to errors for several factors, presented in the following.
P/E Cycling. Before being programmed, a flash cell has to be erased, by removing all the charges from the floating gate. During erase and program operations, a

high voltage is performed across the tunnel oxide, which will degrade the strength of charge trapping over time. As a result, charges can go through the tunnel oxide more easily, which causes both charge leakage and overprogramming. The flash memory program/erase (P/E) cycling causes damage to the tunnel oxide of floating gate transistors which directly results in threshold voltage shift and fluctuation [2] [5] [11] [12]. This will gradually introduce errors to flash cells. Cell-to-cell interference and retention time are two major error sources in flash memory.
Cell-to-cell interference. Cell-to-cell interference is the result of coupling effect that programming on one flash cell can raise the Vth of neighboring cells [5] [7]. Thus, the charge in the cells being affected will increase and the threshold voltage distribution tends to shift to the right. At the early age of flash memory, the insulation strength of the tunnel oxide is strong enough to avoid the Vth raising to the overlap region. Only when flash memory has endured certain P/E cycles, cell-to-cell errors become significant. As a result, we use P/E cycles to indicate the errors caused by cell-to-cell interference.
Retention time errors. The errors accumulated during retention time are another dominant error source in flash memory [5] [9] [10] [11]. It is caused by charge leakage of programmed flash cells over retention time. Retention errors leads to left-shift errors. Retention errors will increase with the increasing of retention time. Besides, the leakage rate is related to the amount of charge in floating gate. If there are more charge, the leakage is faster, which leads more errors. As a result, the state being programmed more charge will suffer more from retention errors.
B. Basics of LDPC on Flash Memory
The increasing bit density of multi-level cell (MLC) and advanced technology scaling on flash memory have raised the requirement for strong ECC, such as BCH and LDPC. The coding schemes for ECC can be grouped into two types: hard-decision codes and soft-decision codes. Hard-decision has single sensing level between adjacent cell states in the data decoding. In contrast, soft-decision has much more sensing levels for obtaining more accurate probability information for decoding input. Hard-decision is always much faster than soft-decision, but with a much worse error correction capability. In order to achieve a strong error correction capability, LDPC, a soft-decision code scheme, has been recommended for the state-of-theart flash memory [2] [3].
LDPC decoding is performed by aggressively increasing sensing levels between each pair of adjacent states. It starts with one sensing level between each pair of adjacent states. If the decoding fails, more sensing levels will be iteratively added to each pair. This process stops until the decoding succeeds or the sensing levels reach the maximal value. In each decoding iteration with a certain number of sensing levels (denoted as N ), there are two steps: sens-

C. Related work
To improve LDPC performance on flash memory, there are many previous works proposed on this issue. These works can be grouped into two types. First, exploiting various characteristics of flash memory to optimize LDPC without modifying LDPC decoding process. Second, exploiting flash errors to revise LDPC decoding process.
For the first group, AGCR [6] is proposed to take advantage of the characteristics that fast-programmed data presents high error rates, which requires more sensing levels to gain reliable input information for LDPC decoding and leads to longer read latency. Xie et al. [13] is motivated by the fact that lossless compression on flash memory can provide more redundancy for ECC with stronger LDPC decoding strength. They propose to exploit it for a better access performance. Besides, Guo et al. [14] proposes to reduce RBER by enlarging noise margins via Vth reduction of the data with high LDPC overhead.
For the second group, REAL [8] incorporates the numerical-correlation characteristic of retention errors into the process of LDPC decoding, to decode with extra numerical information from the other bit of the same flash cell. LDPC-in-SSD [2] is designed with a read-retry of progressive sensing method to progressively increase the sensing levels for stronger decoding capability. With this approach, it is able to perform just enough sensing levels for specific data error rates. Observing that most overlaps between two adjacent states occur at the boundary of two adjacent states, Dong et al. [7] proposes a nonuniform memory sensing strategy to reduce sensing levels on flash memory. However, all of the above works assume that the

errors for each state are symmetric, which is asymmetric in fact. Different from all of these works, we propose to take the specific asymmetric error characteristics of flash memory into consideration to reduce sensing levels.
III. LDPC Optimization via Asymmetric Sensing
Level Placement
In this section, we describe the asymmetric sensing level placement scheme to improve LDPC decoding performance. We first present the problem statement, by illustrating the asymmetric error characteristics of flash memory. Then, we propose two sensing level placement methods through exploiting the asymmetric error characteristics. Finally, we describe the implementation of the proposed approach on flash memory.
A. Problem Statement and Motivation
As described in Section II-B, the strong error correction capability of LDPC is realized at the sacrifice of decoding performance. This is because LDPC has to perform fine-grained sensing with a large number of sensing levels to provide accurate input probability information. The increased sensing levels and information bits result in a bad performance. In order to improve LDPC decoding, it is highly desirable to reduce sensing levels. However, it is a challenge that reduced sensing levels always induce reduced error correction capability. Existing approaches [2] [7] [8] proposed sensing level placement schemes to reduce them. They proposed to symmetrically place sensing levels to each pair of adjacent states. However, from our previous studies on the error characteristics of flash memory, we find that the errors of flash memory present the following two distinctive characteristics: inter-state asymmetric and intra-state asymmetric.

As described in Section II-A, all of the cell states can be impacted by several error sources, such as charge leakage, P/E cycling, and cell-to-cell interference [5] [7] [9] [10]. The reason for the inter-state asymmetric errors is that different states contain varied charges and are impacted by the error sources in different degrees. For example, states S2 and S3 are significantly impacted, while S0 and S1 are slightly impacted. This kind of asymmetric characteristics has also been identified and tested in previous work [5] [9] [11] [12].

Intra-state asymmetric errors. Intra-state asymmetric errors mean that the errors of a single state are asymmetric in different error directions. There are two error directions for a cell state, voltage left-shift and voltage right-shift. For the voltage left-shift induced errors, the dominant impact factor is retention time [10]. For the voltage right-shift induced errors, the dominant impact factors include the P/E cycling induced charge trapping [5], and cell-to-cell interference induced cell charging [5]. Fig.2 presents a description for the intra-state asymmetric errors. Take state S2 as an example. Its left-shift and right-shift errors are different and non-deterministic. When retention time is small, the left-shift could be small and when the P/E cycle is large, the right-shift could be increased. This kind of asymmetric characteristic has also been identified and tested in previous work [5] [9] [10] [12]. In this work, we will propose approaches to utilize these characteristics for LDPC performance improvement.

The above two asymmetric error characteristics motivate us to smartly place the sensing levels, so as to reduce LDPC sensing levels. To achieve this goal, error sources causing left-shift and right-shift are supposed to be known. This information is used to indict whether the right-shift errors are significant. We set a threshold TP E for P/E cycles. If P/E

This is because no matter the left-shift errors caused by retention time are significant, this strategy can reach the objective. IV. Experimental Analysis

In this section, we conduct experiments to verify the effectiveness of the proposed approach. First, we present the asymmetric errors aware sensing level placement, which maintains the error correction capability of LDPC with reduced sensing levels. Bit error rates are computed from the widely used flash memory error model [12]. To correct data with certain RBER, the corresponding sensing levels are placed. Table III shows the related information, which is produced by the error model [12]. In this table, Ntotal represents the total sensing levels of the 3 regions, bits represent the required information bits, and the 3-pair array in the table stands for the placement details (N0l, N0r)(N1l, N1r)(N2l, N2r). For example, when P/E cycle is 5K and retention time is 3 years, the traditional symmetric scheme places 7 sensing levels between each pair of adjacent states, with 3 levels on either side. The proposed approach reduces sensing level to 3 between S0 and S1, 5 between S1 and S2, and 6 between S2 and S3. Thus, for each flash cell, the total number of sensing levels Ntotal is reduced from 21 to 14, and the information bit, which represents the soft information of flash cell, is reduced from 5 bits to 4 bits.
To further quantitatively evaluate the influence on flash access performance, we conduct trace-driven simulation on the SSD module in DiskSim [15] with different readworld workloads from MSR traces [16]. Results show that asymmetric errors aware sensing level placement scheme can gain great read latency reduction by 54%, on average. Besides, when the retention time is longer, the improvement of read performance tends to be more significant. This is because more sensing levels are needed to decode data with longer retention time, which presents more potential for improvement. A special case is that when retention time is 6 months, the improvement decreases. V. Summary and Conclusions
With the generalization of LDPC in flash memory for error correction, there have been lots of works to optimize flash memory based on the LDPC decoding strategy. We studied two asymmetric error characteristics on flash memory. Motivated by these characteristics, we propose an asymmetric sensing level placement scheme to reduce LDPC soft-sensing levels. Our simulation results show that the proposed approach is effective, reducing read latency by as much as 54%, on average.
