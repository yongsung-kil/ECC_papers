A High Throughput Configurable Partially-Parallel Decoder Architecture for Quasi-Cyclic Low-Density Parity-Check Codes
Alaa Aldin AL HARIRI, Fabrice MONTEIRO, Loïc SIÉLER, Abbas DANDACHE Université de Lorraine, 7 rue Marconi, 57070 Metz, France
{alla-aldin.al-hariri, fabrice.monteiro, loic.sieler, abbas.dandache}@univ-lorraine.fr

Abstract—In this paper, we are proposing a new architecture for the fast decoding of Quasi-Cyclic Low-Density Parity Codes (QC-LDPC). QC-LDPC codes are becoming more and more popular in a wide range of applications, including data transmission (WiMAX, DVB-S2) in telecommunication systems, increasing the need for effective decoder architectures. In the present approach, support for a large subset of QC-LDPC codes is provided thanks to the configurability of the architecture at the synthesis level. High levels of parallelism can be reached and hence high throughput achieved, thanks to the modular decoder architecture that takes advantage of the highly regular structure of QC-LDPC parity check matrices. The architectural design has been validated through the implementation of different decoders related to DVB-T2 and DVB-S2 on FPGAs of the Altera Stratix II family. Very high data rates (up to 28.9 GB/s) have been achieved with still acceptable hardware consumption (about 32k logic elements) proving the effectiveness of the approach.
Index Terms—Error correcting codes, QC-LDPC, parallel and configurable decoder architectures, FPGA implementation.
I. INTRODUCTION
Modern communication systems must comply to high performance requirements such as huge volumes of data transmission, fast data transmission, diverse physical media, several transmission standards and many others. Most of the research work has been addressing these issues through classical design improvement techniques such as operational frequency increase, device dimensions reduction and use of additional modules, the counterpart being an increased circuit sensivity towards noise. Evidently, the main design objective is to be able to reach a high level of performance along with high reliability.
New technological and design constraints (higher sensitivity of circuits to disturbances, low power requirements and other) have shifted the need to much more massively parallel architectures. The most popular error correcting codes in present communication standards are Convolutional Codes (CC), Turbo codes and LDPC codes. The later are currently the most promising coding techniques to achieve the Shannon capacitie. LDPC codes can achieve up to 0.04dB of the Shannon’s limit [1]. LDPC codes are used in Digital Video Broadcasting – Satellite – Second Generation (DVB-S2), in particular to send sound and image from remote mobile units

located world-wide back to their home television stations. LDPC codes are also employed in many mobile wireless standards such as IEEE-802.11n (WiFi) and IEEE802.16e (WiMAX) [2].
Three main architectural approaches can be choosen to implement LDPC decoders: the first is based on serial or sequential decoding in which only a single word is processed at a time using a single row single column processor. Despite the interesting low hardware requirements, the throughput achieved is too low for practical use in demanding applications [3]. The second approach is based on full-parallel decoding where each node in the Tanner graph as its own dedicated processing unit. Very high throughputs can be achieved this way but the high costs, induced by the large area requirements and the high architectural and routing complexity, limits employability. This approach is generaly restricted to applications with very high throughputs requirements but limited code block lengths [4], [5]. The third approach is based on partial-parallel decoding [6]–[8]. The main objective of this intermediate approach (in the between of serial and full-parallel LDPC decoder architectures) is to allow a tradeoff between data throughput and hardware cost. While still employing multiple processing units, this architecture requires less than in the full-parallel approach. Each processing unit is shared by several rows and/or columns. Shared memories are used in addition to compensate the lower number of processing units. The lower requirements of the partiallyparallel architecture allow larger block sizes to be processed.
In this paper, we are proposing a new architecture for a reconfigurable partially-parallel decoder, compatible with a wide range of standardized QC-LDPC and different block length. The details of the proposed architecture are discussed in the next section.

In the Tanner graph, each row and each column of H correspond respectively to a check node (C_node) and a variable node (V_node). An edge exists in the graph between a check node Ci and a variable node Vj when and only when the value at coordinates (i, j) in matrix H is hi,j = 1 as shown in figure 1. For instance, row 2 in H corresponds to check node C2 in the Tanner graph. Check node C2 is connected to variable nodes V2, V6, V7 and V8 as denoted by the 1 located in row 2 of H. The total number of edges (connections) between variable and check nodes is n × wc or (n − k) × wr.

III. LDPC DECODING
A. Two-phase message-passing decoding
Several algorithms have been proposed for LDPC decoding, most of them deriving from the well-known belief propagation (BP) algorithm [1]. LDPC codes are usually decoded using an iterative message passing algorithm which consists in iteratively applying two sequential operations: row processing (for check node update) and column processing (for variable node update). In row processing, all check nodes receive messages from their neighboring variable nodes before performing parity check operations and then sending back the results to their neighboring variable nodes. Then in colomn pocessing, the variable nodes update the soft information associated with the decoded bits using the information received from the check nodes, sending back the updates to the check nodes. This process continues iteratively until the stop condition is finaly reachedh [13]. This algorithm is also referred to as two-phase message passing (TPMP).
The basic flow of the TPMP is shown in figure 2. After receiving the possibly corrupted information from an AW GN channel λ, the decoding can begin. First of all, the check node inputs are all initialized to 0. Then, the iterative algorithm starts. First, a check node update step (row processing) is done in which α messages are generated. Then, on reception of the α message by the variable nodes, a variable node update step (column processing) is done to generate β messages which will in turn be send back to the code nodes. This two-step iterative process repeats until all errors are corrected or a limit number of iterations is reached. Sum-Product [14] and MinSum [15] are widely used decoding algorithms. In this paper, we are using MinSum (MS) algorithm which is described in details in the following subsection.

VI. CONCLUSION
A new architecture for a configurable QC-LDPC encoder using a semi-random H-matrix has been presented. This method allowed us to effectively manage HW and memories requirements. To verify our design, a flexible encoder was implemented for the DVB-S2 standard and another for DVBT2. The two designs were synthesized and implemented on Altera StratixII. The FPGA encoder had a throughput up to 29 Gbps for DVB-S2 and up to 19.5 Gbps for the second encoder. A speedup of 3–20 times is provided compared prior designs.
