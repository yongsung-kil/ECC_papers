A Reduced-Complexity Iterative Scheme for Decoding Quasi-Cyclic Low-Density Parity-Check
Codes

Shu Lin, Keke Liu, Juane Li and Khaled Abdel-Ghaffar Department of Electrical and Computer Engineering, University of California, Davis, CA 95616, USA
Email: {shulin, kkeliu, jueli, kaghaffar}@ucdavis.edu

Abstract—This paper presents a reduced-complexity iterative decoding scheme for quasi-cyclic (QC) low-density parity-check (LDPC) codes which is devised based on the section-wise cyclic structure of their parity-check matrices. The decoding scheme significantly reduces the hardware implementation complexity of a QC-LDPC code decoder in terms of the number of message processing units and the number of wires required to connect the message processing units. A long high-rate high-performance QC-LDPC code is used to demonstrate the effectiveness of the proposed decoding scheme. Using this long code as the mother code, descendant QC-LDPC codes of various lengths and rates can be constructed by puncturing and masking the columns and rows of the parity-check matrix of the mother code. These descendant codes can be decoded by deactivating specific sets of message processing units of the mother decoder.
I. INTRODUCTION
LDPC codes [1] are currently the most promising coding technique to achieve close to the Shannon capacities for a wide range of channels. Many LDPC codes have been adopted as standard codes for many current and next generations of communication systems. Application of these codes to high-density data storage systems, such as flash memory, is currently being intensively investigated.
There are various types of LDPC codes [2]. Among them, the most preferred type of LDPC codes for practical applications in communication and storage systems are LDPC codes with quasi-cyclic structure, called quasi-cyclic (QC) LDPC codes. A QC-LDPC code is given by the null space of an array H of sparse circulant matrices of the same size over a finite field, binary or non-binary [2]. In most of the constructions of QC-LDPC codes, the sparse circulant matrices in the paritycheck array H of a QC-LDPC code are circulant permutation matrices (CPMs). Such a parity-check array H of a QC-LDPC code is said to possess CPM-structure and the code is called a CPM-QC-LDPC code.
CPM-QC-LDPC codes have advantages over other types of LDPC codes in hardware implementation of encoding and decoding. Encoding of a CPM-QC-LDPC code can be efficiently implemented using simple shift registers [2], [3] with complexity linearly proportional to its number of paritycheck symbols (or its length). In hardware implementation of an iterative CPM-QC-LDPC code decoder, the CPM-structure of the parity-check array of the code simplifies the wire routing for message passing between the variable-node (VN) message

processing units (MPUs) and check-node (CN) MPUs [4]. In this paper, we first give a review of a simple reduced-
complexity iterative decoding (RCID) scheme for decoding CPM-QC-LDPC codes, which was recently devised by the authors [5]. Then, we present an extension of this simple decoding scheme. The decoding scheme is devised based on the section-wise cyclic structure of the parity-check matrix of a CPM-QC-LDPC code and significantly reduces the hardware implementation complexity of its decoder in terms of the number of the CN-MPUs, the number of VN-MPUs and the number of wires required to connect the VN-MPUs and the CN-MPUs. It also reduces the size of memory required to store the messages to be passed between the VN- and CNMPUs for updating the reliabilities of the received symbols. This decoding scheme can be incorporated with any known reliability updating algorithm, such as the sum-product algorithm (SPA) or the min-sum algorithm (MSA) to form a reducedcomplexity iterative decoding algorithm for CPM-QC-LDPC codes. A proper combination of the proposed decoding scheme and a specific reliability updating algorithm can result in no (or very small) performance degradation.
The rest of this paper is organized as follows. Section II gives a brief review of CPM-QC-LDPC codes and their structural properties. Section III first presents a simple reducedcomplexity iterative decoding scheme for CPM-QC-LDPC codes that was recently devised by the authors [5] and then gives an extension. In Section IV, we incorporate the MSA into the decoding scheme presented in Section III to form reducedcomplexity iterative decoding algorithms for CPM-QC-LDPC codes. Section V presents a high-rate high-performance CPMQC-LDPC code to demonstrate the effectiveness of the proposed decoding scheme using the MSA for updating the reliabilities of the received symbols. We will show that the code can achieve a very low error-rate, say a bit-error-rate (BER) of 10−15, without a visible error-floor. Section VI gives an algebraic construction of rate compatible CPM-QC-LDPC codes of various lengths from a mother code which can be decoded by the mother code decoder. Section VII concludes the paper.

There are two major approaches to the constructions of arrays of CPMs and/or ZMs of the form given by (1), the graphtheoretic approach and the algebraic approach. Well known graph-theoretic approaches are the progressive edge growth (PEG) approach [2], [6], [7] and the protograph approach [2], [8]-[10]. Algebraic approach is based on mathematical tools such as finite fields, finite geometries and combinatorial designs. Well known algebraic constructions are well documented in [2], [11]-[21].
Each type of constructions has its advantages and disadvantages. Algebraic constructions are systematic and the Tanner graphs of the resultant CPM-QC-LDPC codes automatically have girth at least 6. However, the graph-theoretic constructions are, in general, pseudo-random constructions that require computer aid to construct arrays of CPMs and/or ZMs to produce CPM-QC-LDPC codes whose Tanner graphs do not have short cycles, such as cycles of length 4.
For 0 ≤ i < m, each row-block Hi of the array H given by (1) is an e × ne matrix. For 0 ≤ k ≤ e − 1, we label the e rows of Hi from 0 to e − 1. Let hi,k = (hi,k,0, hi,k,1, . . . , hi,k,n−1) be the k-th row of Hi which consists of n sections, hi,k,0, hi,k,1, . . . , hi,k,n−1, each containing e components. Each section is the k-th row of either a CPM or a ZM in Hi. If we cyclically shift all the n sections of hi,k simultaneously one place to the right within the sections, we obtain the (k + 1)-th row hi,k+1 of Hi which also consists of n sections and each section is the (k + 1)-th row of either a CPM or a ZM.
The above cyclic-shift within each section of hi,k is referred to as the section-wise cyclic-shift of the row hi,k. For k = e − 1, the section-wise cyclic-shift of hi,e−1 (the last row of Hi) results in the 0-th (or the top) row hi,0 of Hi. Consequently, all the rows of the i-th row-block Hi of H can be obtained by section-wise cyclically shifting the 0-th row hi,0 e − 1 times. This section-wise cyclic-shift of the rows of H maintains the CPM-structure of each row-block of H.
As shown in [3], the section-wise cyclic structure of the parity-check matrix of a CPM-QC-LDPC code simplifies its encoding which can be implemented with simple shift registers. In the next section, we will show that the section-wise cyclic structure can be used to reduce the hardware complexity of an iterative decoder of a CPM-QC-LDPC code significantly.

III. A REDUCED-COMPLEXITY ITERATIVE DECODING SCHEME FOR CPM-QC-LDPC CODES
A. Merry-Go-Round Decoding Scheme
Let l be a positive integer such that 1 ≤ l ≤ e. For 0 ≤ i < m and t = 0, 1, . . ., we form a sequence of submatrices, (Hi,t)t=0,1,..., of the i-th rowblock Hi of the array H given by (1). Each submatrix Hi,t in the sequence (Hi,t)t=0,1,... consists of l consecutive rows hi,(tl)mod r, hi,(tl+1)mod r, . . . , hi,(tl+l−1)mod r of the row-block Hi (including the end around case). Based on the section-wise cyclic-shift structure of a row-block in the array H, we can easily see that the submatrix Hi,t can be obtained by section-wise cyclically shifting the l rows of the first submatrix Hi,0 simultaneously tl times to the right. Let c = e/l . It is clear that any c consecutive submatrices in the sequence (Hi,t)t=0,1,... cover all the e rows of the i-th row-block Hi of H. If l is a factor of e, then each row of Hi is covered once. If l is not a factor of e, there are r consecutive rows of Hi for which each is covered twice where r is the remainder resulting from dividing e by l. From the m row-blocks of the array H, we can form m sequences of submatrices. Taking the first submatrix from each sequence, we form an ml × ne matrix M(l) = [HT0,0HT1,0 . . . HTm−1,0]T , which is a submatrix of H. Based on the section-wise cyclic structure of each submatrix Hi,0, it is clear that if we sectionwise cyclically shift the ml rows of the matrix M(l) c times (each time cyclically shifting the rows of M(l) l positions to the right), we cover all the rows (or row-blocks) of the array H. (If l is not a factor of e, some rows may be covered twice.) This structural property allows us to decode the CPMQC-LDPC code given by the null space of H iteratively based on the submatrix M(l) alone. Each decoding iteration based on M(l) is called a decoding sub-iteration. At the end of each decoding sub-iteration, the reliabilities of the received symbols are updated with a chosen reliability updating algorithm. Then, the reliability vector and the received sequence are sectionwise cyclically shifted to the left by l positions and used as the input information (maybe together with the channel information) to carry out the next sub-iteration based on M(l).
Every c consecutive decoding sub-iterations performed based on M(l) process all the rows of the entire paritycheck array H. At the end of each decoding sub-iteration, the syndrome s of the hard-decision of the received sequence (after section-wise cyclically shifted l positions to the left) is computed based on the entire parity-check array H. If s = 0, we stop the decoding process; otherwise, we continue the decoding process until a preset maximum number of decoding sub-iterations is reached. If l is a factor of e, c consecutive decoding sub-iterations is equivalent to one full decoding iteration based on the entire parity-check array H.
The above decoding process of a CPM-QC-LDPC code simply revolves around the submatrix M(l) iteratively. Since the CPM-structure of the parity-check array H is preserved, the wire routing advantage due to the CPM-structure of H is maintained. The matrix M(l) is referred to as the decoding

The parameter c is called the complexity reduction factor (CRF) for the RCIDscheme with M(l) as the decoding matrix. The RCID-scheme presented above for CPM-QC-LDPC codes is called a merrygo-round decoding (MGRD) scheme with CRF-c.
For l = 1, the CRF is e which is the size of a CPM in the parity-check array H. If the size of the CPMs in a constructed array H is large, the reduction of hardware complexity of the MGRD-scheme with CRF-e would be tremendous. The MGRD-scheme with CRF-e indeed reduces the decoder hardware complexity significantly, however it also increases the decoding latency. By choosing various values of CRF c, a wide spectrum of trade-offs between decoding complexity and latency can be obtained. For a specific application, c must be chosen to give an effective trade-off between decoding complexity and decoding latency.
IV. MGR-MSA
The MGRD-scheme for CPM-QC-LDPC codes can be incorporated with any known reliability updating algorithm, such as the SPA or the MSA, to form an MGRD-algorithm for decoding CPM-QC-LDPC codes. A proper combination of the MGRD-scheme and a specific reliability updating algorithm can result in no performance degradation. The incorporation of the MSA into the MGRD-scheme is referred to as the MGRMSA.
Let y = (y0, y1, . . . , yne−1) be the soft-decision received vector at the output of the receiver detector. We assume that the symbols of a codeword are transmitted over a binary AWGN channel with BPSK signaling. Let Imax be the maximum number of iterations to be performed based on the entire parity-check array H, each iteration consisting of c decoding sub-iterations based on the ml × ne submatrix M(l) = [ht,j ]0≤t<ml,0≤j<ne. VI. AN ALGEBRAIC CONSTRUCTION OF RATE-COMPATIBLE CPM-QC-LDPC CODES
Given a base matrix B over a finite field GF(q) for which any 2 × 2 sub-matrix of B is nonsingular or contains at least one zero element, a family of CPM-QC-LDPC codes with various lengths and rates can be constructed by taking submatrices of B with various sizes and dispersing them into arrays of CPMs and/or ZMs. The null space of the CPMdispersion of each of these sub-matrices of B gives a CPMQC-LDPC code whose Tanner graph has girth at least 6. The code given by the null space of the CPM-dispersion of B is called the mother code and the codes constructed from the sub-matrices of B are called descendant codes. The constraint that any 2 × 2 sub-matrix of B is nonsingular or contains at least one zero element is referred to as 2 × 2 sub-matrix (SM) constraint [21]. Clearly, a submatrix of B also satisfies the 2 × 2 SM-constraint. Various algebraic constructions of base matrices that satisfy the 2 × 2 and/or 3 × 3 SM-constraints can be found in [2], [5], [17]-[21].
Let B be an m × n base matrix over GF(q) which satisfies the 2 × 2 SM-constraint and H be the CPM-dispersion of B. Let C be the CPM-QC-LDPC code given by the null space of H, which is the mother code. For 1 ≤ m ≤ m and 1 ≤ n ≤ n, let B be a m × n submatrix of B and H be the CPM-dispersion of B . Then H is an m × n array of CPMs and/or ZMs of size (q − 1) × (q − 1) which is a subarray of H. The null space of H gives a CPM-QC-LDPC code C of length n (q − 1) and rate at least (n − m )/n . The code C is a descendant code of C. Since H can be obtained by puncturing n−n column-blocks and m−m row-blocks from H, we may consider C as a punctured code of C. Starting from the mother base matrix B, we can construct a family of descendant codes of C with various lengths and rates. All these descendant codes can be decoded with the decoder for the mother code C. This is done by deactivating the VNMPUs corresponding to the (n − n )(q − 1) columns of the punctured column-blocks and the CN-MPUs corresponding to the (m − m )(q − 1) rows of the punctured row-blocks. C and its descendant codes form a family of rate-compatible codes.
Suppose we replace a nonzero entry in B by the zero element of GF(q). In the CPM-dispersion H of B , this replacement results in replacing a (q − 1) × (q − 1) CPM in H by a (q − 1) × (q − 1) ZM which is referred to as masking [2], [5], [15]-[17]. Let λ be a nonnegative integer less than the number of nonzero entries in B . Replacing λ nonzero entries in B by λ zeros amounts to replacing λ CPMs by λ ZMs at the locations in H corresponding to the locations of the λ nonzero entries in B which are replaced by zeros. Masking λ CPMs in H amounts to removing λ(q − 1) edges from the Tanner graph G associated with H . Removing these edges from G may break many cycles. As a result, the resultant Tanner graph Gmask may have a smaller number of short cycles or a larger girth, or both. The subscript “mask” stands for masking. In choosing the nonzero entries in B to be masked, we have to avoid disconnecting the Tanner graph of H .

The operation of masking the base matrix B = αlk,s 0≤k<m ,0≤s<n can be modeled mathematically. Let Z = [zk,s]0≤k<m ,0≤s<n be an m × n matrix with the zero element and unit element of GF(q) as entries. Define the following product of Z and B : Bmask = Z ⊗ B = zk,sαlk,s 0≤k<m ,0≤s<n where zk,sαlk,s = αlk,s if zk,s = 1 and zk,sαlk,s = 0 if zk,s = 0. In this matrix product operation, some nonzero entries in B at the locations corresponding to the locations of 0-entries in Z are replaced (or masked) by 0’s. The CPM-dispersion of Bmask gives an m × n masked array Hmask of CPMs and ZMs of size (q − 1) × (q − 1). We call Z and Bmask the masking matrix and the masked base matrix, respectively. The null space of Hmask also gives a CPM-QC-LDPC code Cmask of length n (q − 1).
Properly masking a base matrix, we may obtain a code with better error performance. Puncturing and masking can be combined to produce good rate-compatible codes.

VII. CONCLUSION AND REMARKS
In this paper, a reduced-complexity iterative decoding schemes for QC-LDPC codes with CPM-structure was presented. We showed that the decoding scheme significantly reduces the hardware implementation complexity of a CPMQC-LDPC code decoder in terms of number of message processing units and the number of wires required to connect the message processing units. A long CPM-QC-LDPC code with rate 0.9519 was constructed to demonstrate the effectiveness of the proposed decoding scheme. We showed that this long high-rate CPM-QC-LDPC code decoded with the MSA can achieve a BER of 10−15 without visible errorfloor. This code is quite suitable for applications in highspeed communication and high-density data storage systems, such as high-speed optical communication systems and flash memory, where very low error rates and no more than 7% overhead (in terms parity-check symbols) are required. Also in the paper, we presented a algebraic method for constructing a family of descendant CPM-LDPC codes of various lengths and rates from a mother CPM-QC-LDPC code. These descendant codes can be decoded by deactivating specific sets of message processing units of the mother code decoder.
