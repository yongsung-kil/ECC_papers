Design and Implementation of LDPC(Low Density Parity Check) Coding Technique on FPGA( Field Programmable Gate Array)
for DVB-S2 (Digital Video Broadcasting-Satellite)
Rita Purnamasari, Heroe Wijanto, Iswahyudi Hidayat
Electrical Engineering Faculty, Telkom University Jl. Telekomunikasi, Dayeuh Kolot, Bandung 40257 Telp. (62-22)7504108 ext. 2229

Abstract- LDPC (Low Density Parity Check) is a channel coding technique is used to correct errors so that the validity of a data transmission on the noise transmission channel guaranteed for accuracy. LDPC is suitable for applications that require a large bandwidth, high reliability, and high noise channel like DVB-S2 (Digital Video Broadcast-Satellite) application. Despite the high level of complexity, LDPC coding technique is very possible to be implemented in line with the growing hardware capabilities. The purpose of this research later is to make DVB-S2 chips that can transmit digital video via satellite.
The method used in this research is to design a system, synthesize, and implement it on a FPGA (Field Programmable Gate Array) Virtex 4 series. LDPC coding Technique is designed with two stages, namely the stage of designing encoding and designing decoding. At encoding used lower triangular algorithms, while the decoding used to flipping bit algorithm. Parity matrix has 6x12 sized and code rate ½.
From the simulation results, the correction capability of flipping bits can only be correct for a 1-bit error correction. The resulting of bit rate is 21.4 Mbps, it is sufficient for video delivery that require wide bandwidth. In synthesis, when compared to some other designs, this design should be corrected because it takes a large memory capacity (209 936 Kbytes ) and is still quite long periods (4,9μs). The design of this coding technique can be implemented into the FPGA Virtex-4 because of its resource usage less than 100%. Keywords: LDPC(Low Density Parity Check), DVB-S (Digital, lower triangular, bit flipping , FPGA(Field Programmable Gate Array) Virtex-4
I. Introduction
LDPC (Low Density Parity Check) is a channel coding techniques are used to correct errors in a linear manner so that the validity of a data transmission on the transmission channel noises will be guaranteed for accuracy. Because of the error correction capability is known as near-Shannon (maximum theoretically corrects errors), then the LDPC code is suitable for applications that require a large bandwidth, high reliability, and channel noises too high. For applications that DVB-S2 (which require large bandwidth and high reliability due to the information that is sent in the form of videos and channels due to the high transmission noises transmitted via satellite) using LDPC codes in the channel coding. Despite the high level of complexity, LDPC

coding techniques will be very possible to be implemented in line with the growing hardware capabilities.
With a background as mentioned above, the problems that arise in this study are how to design blocks using the hardware description language VHDL (Very High Speed IC Description Language), integrating an architecture blocks into a single unit encoding and decoding system, synthesis architecture has been tested functions, and implementing on FPGA virtex 4.
In this paper have a limitation like Code rate only ½ with the size of the parity check matrix 6 x12, encode LDPC used Lower Triangular algorithm, the decode LDPC, used bit-flipping algorithm, and Using FPGA Virtex-4

CONCLUSSION AND SUGGESTION

5.1 Conclussion
1. Design of LDPC (Low Density Parity Chek) architecture with coderate ½ and the parity check matrix size of 6x12 can be designed, synthesized, and implemented on an FPGA board Virtex4. Test validation has been performed at every stage and get the same result between the VHDL simulation and implementation with the help of LA (Logic Analyzer).
2. With a clock frequency of FPGA Virtex-4 100 MHz obtained bitrate of 21.4 Mbps and the working frequency of the system is 3.57 MHz.
3. Based on test simulation of LDPC (Low Density Parity Chek) encoding system, the period required by the system to detect and correct bit errors are 28 clocks or 280 ns for each 6-bit processing in simulation. It is very short periode because the the system use parallel system. Based on implementation in FPGA, the system need 4,98 µs, and it is more longer than orther architecture design.
4. Based On The Synthesis Results Of Block LDPC (low density parity chek) encoding system obtained the amount of resource required in the architecture is less than 100%.
