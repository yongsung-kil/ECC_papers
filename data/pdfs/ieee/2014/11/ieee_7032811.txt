Area-Efficient Check Node Unit Architecture for Single Block-Row Quasi-Cyclic LDPC Codes

Chuan Zhang, Shenghui Weng, and Xiaohu You
National Mobile Communications Research Laboratory School of Information Science and Engineering Southeast University, Nanjing, China {chzhang, 213101699, xhyu}@seu.edu.cn

Zhongfeng Wang
Broadcom Corporation 5300 California Avenue, Irvine
CA 92617, USA zfwang@broadcom.com

Abstract—Single block-row quasi-cyclic low-density paritycheck (QC-LDPC) codes are recently proposed. This kind of codes are favorable in applications because of their construction flexibility and good performance compared to PEG codes and array codes. However, the corresponding high row weight will increase the hardware complexity of the check node unit (CNU). In this paper, an area-efficient CNU architecture for single block-row QC-LDPC codes is proposed by refining the searching method for the 2nd minimum. Implementation results of the rate0.9333 (2115, 1974) code have shown that compared with existing design approaches, the proposed method can achieve at least 15.2% hardware reduction while keeps the latency as the same.
Keywords—Check node unit; single block-row QC-LDPC codes; hardware efficiency; 2nd minimum search.
I. INTRODUCTION
Since the rediscovery of low-density parity-check codes (LDPC) [1], quasi-cyclic LDPC (QC-LDPC) codes [2, 3], which are well structured and therefore suitable for hardware implementation, have drawn a lot of attentions. In order to meet the growing requirement on data rate of communication systems, QC-LDPC codes which can achieve both high code rate and good decoding performance are in high demand. Recently, a new kind of QC-LDPC codes, with the name of single block-row QC-LDPC codes, are proposed in [4]. Constructed from cyclic difference families (CDFs) or perfect difference families (PDFs), single block-row QC-LDPC codes show the following advantages: 1) the code length can be flexibly chosen as low as the minimum achievable code length, 2) the construction flexibility does not affect the decoding performance.
On the other hand, the construction method of single blockrow QC-LDPC codes limits the column weight dv of the check matrix H to be 3, 4, or 5. Moreover, in order to achieve good performance, the length of circulant matrix array L can be as large as 1000. Therefore, the resulted row weight dc can be high, which becomes a bottleneck for efficient implementation of check node unit (CNU). Among few existing decoder designs for single block-row QC-LDPC codes, [5] proposed the CNU architecture based on the 1-element based decoding schedule and the sub-optimal low-latency searching method. However, with the increase of row weight dc, this method will suffer from the large number of required comparators. In order

to tackle this issue, a new 2nd minimum searching method named the hybrid low-latency searching approach is proposed. According to the FPGA implementation results, for the rate0.9333 (2115, 1974) single block-row QC-LDPC, the proposed CNU architecture can achieve 15.2% hardware reduction compared to the design proposed in [5]. And the corresponding latency keeps the same.
The remainder of this paper is organized as follows. Section II briefly reviews the single block-row QC-LDPC codes and the 1-element based decoding schedule for Min-Sum algorithm (MSA). In Section III, the proposed hybrid lowlatency searching method is proposed. Section IV presents the area-efficient CNU architecture. The implementation results and comparison are given in the same section. Section V concludes this paper.

Although, the 1-element based decoding algorithm helps to decrease the number of inputs per CNU, further optimization of CNU architecture is still required for single block-row QCLDPC codes, especially when L is large.
Since the 1st minimum can be easily found out with the binary tree search method, the main difficulty for CNU lays in figuring out the 2nd minimum. One straightforward approach is to compare every element in the array one by one and keep the pair of two temporary minimums until reach the end of this sequence. In order to offer the explanation more clearly, we employ a running example here. Suppose we are going to find out both the 1st and 2nd minimums of the sequence {5, 7, 2, 6, 4, 0, 1, 3}. The straightforward approach is illustrated in Figure 1.

However, this sequential approach suffers from a long processing latency. In general, for a sequence of length n, the processing latency equals (2n-3) comparisons. For single block-row QC-LDPC codes, whose block length L can be as large as 1000, this long latency is unacceptable. To this end, a parallel approach which employs the binary tree architecture is introduced. According to Figure 2, this binary tree architecture can help us to figure out the 1st minimum first. It is observed that there are two kinds of comparators employed by the proposed CNU architecture. The output “S”/“B” indicates the smaller/bigger one of two inputs, respectively. The “L” output is set as 0 when the left input of the comparator is the smaller one, otherwise it is set as 1. All variable-to-check messages are stored in registers before get processed.

Illustrated in Figure 6, the proposed CNU first generates the 1st minimum and the index of the 1st minimum. Its index can be simply found out with the aid of those “L” outputs. Thereafter, the index of the 1st minimum is employed to generate the index of the adjacent neighbor in the first level of Step 1 shown in Figure 5. This step can be done by checking the parity of the index of the 1st minimum. Finally, the 2nd minimum is identified by comparing the adjacent neighbor of the 1st minimum in the first level of Step 1 and those bigger values.

It means that the area benefit brought by the proposed design approach is considerable for the entire decoder design.
An additional comment which the authors would like make is that the possible use of the proposed area-efficient CNU architecture is not limited to either the single block-row QCLDPC codes or the 1-element based decoding algorithm. The proposed CNU architecture can be employed for any decoder algorithm as long as it is a variant of MSA. It also works well with any LDPC code that has a high row weight. Therefore, the proposed area-efficient CNU architecture shows advantages for general applications.
V. CONCLUSION
In this paper, an area-efficient CNU architecture for single block-row QC-LDPC codes is proposed. By employing the hybrid low-latency searching approach, the proposed CNU architecture successfully reduces the number of required comparators to (3n 2 − 2) , where n is the number of inputs. FPGA implementation results for the rate-0.9333 (2115, 1974) single block-row QC-LDPC code have shown that, compared to state-of-the-art design, the proposed design can achieve up to 15.2% hardware reduction while keeps the same processing latency.
