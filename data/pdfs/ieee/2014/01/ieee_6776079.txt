AbstractThis paper presents a block-serial fully-overlapped Quasi-Cyclic Low Density Parity Check (QC-LDPC) decoder for IEEE 802.11n. Based on the circuit retiming and message bypassing techniques, this decoder effectively improved the previous work proposed by Xiang et al. with an 11% clock-rate increase and a 3% decoding time reduction on average. Moreover, the proposed chip spends about 3.67 mm2 in 90 nm CMOS technology, its power approximately consumes 171 mW at 250 MHz, and throughput of the proposed design can reach 672 Mbps.

I. INTRODUCTION
Issues on wireless communications from a source to a destination via the air are not only how to communicate between each other, but also how the destination can receive correctly what the source sent. Because of the non-ideal environment, there are always a number of faulty factors, such as interference, attenuation, distortion, and noises, making the transmitted message to be noisy. To solve these problems, channel codes have been proposed, such as Error Correcting code (ECC). Channel coding is accomplished by introducing redundancy in the channel encoder and using such redundancy at the decoder to reconstruct the source message as accurately as possible.
Because of its lower decoding complexity than turbo codes, LDPC codes have been chosen in many modern communication standards, such as G.hn/G.9960, CMMB, DVB-S2/DVB-T2/DVB-C2, DMB-T/H, IEEE 802.3an, WiMAX, and IEEE 802.11n. In this study, the proposed LDPC design is set to meet the IEEE 802.11n standard.
Various LDPC hardware implementations have been proposed [1][4]. LDPC implementations can be divided into fully-parallel and partially-parallel architectures. The fullyparallel architecture implements all the check node and variable node units and all the interconnections between these two units on a chip. However, for the partially-parallel architecture, LDPC decoders are only implemented by one or a few number of variable node unit(s) and check node unit(s).

II. PROPOSED ARCHITECTURE
Based on the turbo-decoding message-passing (TDMP) algorithm [5], Fig. 1 shows the block diagram of our proposed block-serial fully-overlapped LDPC decoder for IEEE 802.11n. It consists mainly of a check node unit (CNU), a variable node unit (VNU), a permutation network (PN), a check node message memory, and a log-likelihood ratio (LLR) memory. Here, “block-serial” means that the input data processed by CNU and BNU in a partially-parallel way with a block size in IEEE 802.11n, and “fully-overlapped” means that the computations in CNU and VNU are concurrent processed over time, not interleaved or partially-overlapped each other.

According to the QC-LDPC decoding approach, because the parity check sub-matrix is either all zeros or cyclically shifted from the identity matrix by an offset value, a PN is needed to restore the real information connected between check nodes and variable nodes. Hence, such a PN is often connected to a CNU directly. However, it results in the delay time of each pipeline stage being imbalanced. To alleviate this problem, the proposed design uses a circuit retiming technique to move PN to the front of LLR Memory, as shown in Fig. 1. According to the simulation result, this change can approximately shorten the critical path by 11% compared to the original design.
After updating the variable node, the LLR Memory is used to store the posterior LLR messages. Meanwhile, CNU reads the same LLR messages from the LLR Memory. However, it is impossible that memory read and store operations are performed at the same time for some cases. Therefore, using a bypassing mechanism, posterior LLR messages from VNU can be directly bypassed to CNU for the initial first row and other appropriate processing phases. The benefit of this mechanism can not only save the decoding time for every iteration, but also save the power consumption for the LLR memory because reading messages in the LLR memory will not be needed, i.e., reducing the number of memory accesses, as shown in Fig. 2.

IV. CONCLUSION
This paper presents the design of a block-serial fullyoverlapped QC-LDPC decoder for IEEE 802.11n. Based on the circuit retiming technique, the proposed design can obtain an approximate 11% improvement in clock rate, which efficiently raises the throughput performance. Using the bypassing mechanism, the proposed design saves not only 3% decoding time than the previous work [4], but also reduces the power consumption. Furthermore, because the throughput of the proposed design reaches 672 Mbps at a code rate of 1/2, it meets the requirement of a maximum transmission rate of 600 Mbps in IEEE 802.11n-2009. Finally, performance evaluation with other previous works was made to validate that our proposed design had a smaller chip area and consumed less power dissipation.
