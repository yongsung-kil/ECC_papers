Status and Recent Advances on Forward Error Correction Technologies for Lightwave Systems
Andreas Leven, Senior Member, IEEE, and Laurent Schmalen, Member, IEEE
(Invited Tutorial)

Abstract—Since the introduction of coherent transponders, forward error correction based on soft decision is now established in optical communication. In this paper, we give a tutorial-style introduction of one class of commonly used codes, namely low-density parity-check (LDPC) codes. Also we discuss new developments such as convolutional LDPC codes and show how they can be employed as potential candidates for future optical communication systems.
Index Terms—Error correction codes, iterative decoding, lowdensity parity-check (LDPC) codes, spatial coupling.
I. INTRODUCTION
F ORWARD error correction (FEC) in optical communications has been first demonstrated in 1988 [1]. Since then, coding technology has evolved significantly. This pertains not only to the codes themselves but also to encoder and decoder architectures. Modern high-speed optical communication systems require high-performing FEC systems featuring [2]:
1) Support of throughputs of 100 GBit/s or multiples thereof, 2) Low power consumption, 3) Coding gain in the order of 11 dB, 4) Adaptation to the peculiarities of the optical channel. Transmission systems with 100 GBit/s and 400 GBit/s today typically use one of two coding schemes: Block-Turbo Codes (BTC) or Low-Density Parity-Check (LDPC) codes. Since in coherent systems, soft information is readily available, high performing systems typically utilize this information in a soft decoder architecture. This paper will focus on LDPC codes, since these are more commonly implemented. The paper introduces fundamental concepts of coding for lightwave systems in a tutorial style, but extends the discussion to new concepts, namely spatially coupled codes. It builds upon two tutorials presented at OFC 2012 [3] and ECOC 2013 [2]. FEC coding is based on adding redundant information systematically to a source information bit sequence. After transmission over a noisy channel, a decoding system tries to exploit the redundant information for fully recovering the source information. Several methods for generating redundant information

But if a bit participates in more than one parity equation, error correction can be performed. Low-density parity-check codes are built upon a clever combination of several parity check-equations. In this paper, we focus on binary block codes, as this class of codes is predominant in the area of optical communications. An FEC encoder is a device that encodes an information block comprising K information bits into a code block (also denoted code word) comprising N > K code bits.
The paper is structured as follows: In Section II, we give an overview and a short historical account on FEC an lightwave transmission systems. In Section III, we show by means of a tutorial example how most of the codes utilized nowadays in optical communications are constructed and especially focus on LDPC codes. We conclude the paper by illustrating the effect responsible for the performance of spatially coupled codes, present some design guidelines, and finally give pointers on the combination of FEC with modulation.
II. FORWARD ERROR CORRECTION IN LIGHTWAVE SYSTEMS
The first error-correcting codes, introduced by Hamming in 1950 [4], were binary linear block codes, which were designed to improve the reliability of electro-mechanical computing machines [5], [6]. Hamming codes are based on simple algebraic descriptions and have a straightforward hard-decision decoding algorithm to correct single errors. These codes were also one of the first applied in optical communications: In 1988, Grover [1] reported a performance improvement not only with respect to the required received power and flooring, but also with respect to chromatic dispersion tolerance and resilience to laser mode hopping.
Reed and Solomon [7] introduced non-binary cyclic error correcting codes that are able to correct and/or detect a predetermined number of erroneous symbols. The corresponding algebraic decoding algorithm was proposed by Berlekamp and Massey [8], [9]. A Reed-Solomon code, in short an RS(N, K) code over a Galois Field GF(2q ), is a non-binary code that consists of N q-bit symbols, where N ≤ 2q − 1. Each code word contains K information symbols and N − K parity symbols. The RS(255,239) code over GF(28 ) can correct up to eight symbol errors (or a single burst error of up to 57 bits) with 6.7%

Due to its low implementation complexity, this code is still in use for applications with low coding gain requirements. It is included in a number of standards and described in detail in ITU-T G.975 [12]. This code is commonly referred to as first generation coding for lightwave systems [13].
With code concatenation, long and powerful codes can be constructed that achieve significant coding gains with reasonable complexity. Concatenated codes were first introduced by Forney in 1966 in the form of serially concatenated codes, with a cascaded “inner” and an “outer” code [5]. Both component codes are decoded separately, resulting in a much lower complexity compared to maximum-likelihood decoding of the entire concatenated code. The discovery of parallel concatenated codes and the corresponding iterative “turbo” decoding algorithm described by Berrou et al. in 1993 [14] formed the breakthrough of this technique, ushering in a new era in error control coding. In each iteration, the component decoders exchange reliability information and update the a priori knowledge of the respective other decoder in a mechanism similar to a turbo engine.
The so called second generation FEC schemes are mostly based on concatenated product codes, meaning that the bit stream is first encoded by a simple algebraic code and that several of these code words are interleaved. The interleaver output is, after partitioning, again encoded by a simple algebraic code. Almost all 6.7% overhead codes specified in the ITU-T G.975.1 standard [15] are such concatenated product codes. These codes achieve very good performance with low error floors, and have a moderate implementation complexity. The component codes of such schemes are often Bose–Chaudhuri–Hocquenghem (BCH) codes. BCH codes are a class of cyclic error correcting codes defined over a finite field, which are able to correct multiple bit errors using syndrome decoding [16] based on efficient closedform solutions for computing the error locations. The decoding of concatenated block codes is usually performed iteratively by alternately decoding each of the component codes, improving the overall result after each iteration. For example, the code specified in G.975.1, Appendix I.9 is a BCH–BCH product code with a net coding gain of 9.24 dB at an output BER of 10−15 . All product codes specified in G.975.1 are very long codes with block lengths of around 500,000 bits.
The main advantage of concatenated product codes based on algebraic codes is that the internal decoder data flow is small once the syndromes have been computed [17]. Three very recent developments in second-generation hard-decision FEC codes are the continuously-interleaved codes [18], the staircase codes [17] and the braided BCH codes [19], that can be considered as a combination of concatenated product codes and convolutionalstructured codes, with a very efficient hard-decision decoding performance. The net coding gain of the staircase code amounts to 9.41 dB at a BER of 10−15 , which is only 0.56 dB away from the capacity of the hard-decision channel [17]. The advantage of the braided BCH code of [19] is mainly due to the use of a very simple extrinsic decoding scheme yielding larger

gains than conventional decoding algorithms based on intrinsic information.
With the advent of coherent transmission schemes and the utilization of high resolution Analog-to-Digital Converters, softdecision decoding became an attractive means to increase transmission reach. FEC schemes with soft-decision decoding are typically classified as third generation FEC schemes. Today, there are two competing classes of codes that allow for softdecision decoding and that are attractive for implementation in optical receivers at decoding throughputs of 100 GBit/s and above. The first class corresponds to the natural extension of the second generation decoding schemes: concatenated product codes. These codes are also often denoted as Block Turbo Codes (BTCs). Iterative soft-decision decoding of BTCs was pioneered by Pyndiah in [20], using a component decoder based on the sub-optimal Chase-II algorithm [21]. The application to optical communications has been demonstrated in [22]. The advantage of this decoding algorithm is that it is highly parallelizable. Furthermore, BTCs converge quite rapidly and only require a relatively small number of decoding iterations. As the product code usually has a large minimum distance, the error floor tends to be low, with a steep slope (with respect to asymptotic behavior versus SNR). However, these codes require large block lengths to realize codes with small overhead, leading to a larger decoding latency. A further disadvantage is the inflexibility with respect to varying frame sizes and overheads. With overheads of more than 15 to 20 percent, these codes no longer perform well, since the number of code words to consider in Pyndiah’s decoder becomes prohibitively large.
The second popular class of soft-decision decodable codes in optical communication systems are LDPC codes, developed in the 1960s by Gallager in his landmark Ph.D. thesis [23]. These codes were not further investigated for a long time due to the perceived complexity of long codes. With the discovery of turbo codes in 1993 [14] and the sudden interest in iteratively decodable codes, LDPC codes were rediscovered soon afterwards [24], [25]. In the years that followed, numerous publications from various researchers paved the way for a thorough understanding of this class of codes leading to numerous applications in various communication standards, such as, e.g., WLAN (IEEE 802.11) [26], DVB-S2 [27], and 10G Ethernet (IEEE 802.3) [28]. LDPC codes for soft decision decoding in optical communications were studied in [29]. In addition to conventional binary LDPC codes, non-binary LDPC codes were promoted for optical communication systems in [30]. Modern high-performance FEC systems are often constructed using a soft-decision LDPC inner code which reduces the BER to a level of 10−3 to 10−5 and a hard-decision outer code which pushes the system BER to levels below 10−12 [29]. An outer cleanup code is used as most LDPC codes exhibit a phenomenon called error floor: above a certain signal-to-noise ratio, the BER does not drop rapidly anymore but follows a curve with a small slope. This effect is mainly due to the presence of trapping sets or absorbing sets [31], [32]. The implementation of a coding system with an outer cleanup code requires a thorough understanding of the LDPC code and a properly designed interleaver between the LDPC and outer code for avoiding that the errors

With increasing computing resources, it is now also feasible to evaluate very low target BERs of LDPC codes and optimize the codes to have very low error floors below the system’s target BER [33]. Unlike classical coding schemes such as BCH or Reed-Solomon codes, there is no commonly accepted design rule for LDPC codes. III. LOW-DENSITY PARITY-CHECK CODES: A TUTORIAL INTRODUCTION

In this section, we will give a tutorial-style introduction of low-density parity-check coding based on a simple toy example. In general, FEC coding attempts to increase the reliability of transmission over a noisy channel by adding redundant information to the message to be transmitted. The art of coding deals with how the redundant information is added to the information so that the increase in reliability is maximized while the complexity of adding redundancy (encoding) as well as exploiting the redundancy for correcting errors (decoding) is manageable. In the following, we use italic font to denote scalar mathematical variables and boldface italic font to denote vectors. Vectors are usually written using lower-case letters whereas we use uppercase letters to designate matrices.
Let us now assume that we would like to transmit a message m comprising 25 information bits, i.e., m = (x1 , x2 , . . . , x25)T . We also decide to transmit additional 11 bits for increasing the reliability of the transmission. The 25 information bits and the 11 redundant bits form a 36-bit code word c = (x1 , x2 , . . . , x36 )T . Typically, the dimension (or length) of the code word is denoted by N and K denotes the size of the information words. In the example, we have K = 25 and N = 36. The amount of redundant information that is added is commonly expressed in terms of the code rate R which is defined as the ratio of the information block length K and the code dimension N , i.e.,

Code designers typically attempt to maximize the smallest Hamming distance out of all the Hamming distances of all possible code word combinations. This smallest Hamming distance between all possible code word combinations is denoted by the term minimum Hamming distance and the symbol dmin . A decoding strategy could now be to compare a received message with all possible code words and select the one with minimum Hamming distance as the most likely transmitted message. Unfortunately, even for this small toy example, the complexity of this decoder would be prohibitively large, since more than 33 million possible code words need to be compared with the received message. Therefore, it is beneficial to devise a set of rules that link information and redundant bits in a manner that can be exploited to design less complex encoders and decoders.

In the following, we evaluate the performance of this toy coding example. In order to assess the performance of the code using hard-decision decoding, we employ the additive white Gaussian noise (AWGN) channel model with antipodal ±1 signaling (binary phase shift keying, BPSK), meaning that a bit xi = 0 is mapped to a channel symbol si = +1 and a bit xi = 1 is mapped to the channel symbol si = −1. The AWGN channel model is a good model for coherent transmission systems, as due to the use of extensive filtering in the signal processor, the noise distribution at the input of the FEC stage is approximately Gaussian as a consequence of the central limit theorem.
In the field of coding and in the broad body of literature on FEC, the terms Es/N0 and Eb /N0 are frequently used. Therein, Es denotes the energy per modulation symbol. With our antipodal signaling, we have Es = E{s2i } = 1. The noise is characterized by the two-sided noise power spectral density N0 = 2σn2 where σn2 is the variance of the real-valued noise component ni. N0 refers to the complex Gaussian noise (in the complex I-Q plane of the modulation) but in the case of BPSK, we may only consider the real part. The received symbol zi in our model is obtained by zi = si + ni where ni is real-valued Gaussian noise of zero mean and variance σn2 = N0 /2.
Frequently, especially for comparing different coding schemes, Eb /N0 is used instead of Es/N0 . Herein, Eb denotes the energy per information bit whereas Es denotes the energy per coded bit. For example, if a code of rate R = 5/6, corresponding to an overhead of ρ = 20%, is used, the ratio of code bits N versus information bits K amounts to N/K = 6/5 = 1.2, i.e., 1.2 code bits are transmitted for each information bit. This means that if the code bits will be transmitted each with energy Es, the amount of energy conveyed by each information bit amounts to Eb := Es/R ≥ Es. As Eb is normalized to the information bits of the transmission system, it allows us to evaluate the net coding gain (NCG), i.e., the gain that is actually achievable by taking into account the coding. The net coding gain is frequently used to assess the performance of a coding scheme and is defined as the gain (in dB) of the coded information in terms of Eb /N0 relative to uncoded transmission for a given output BER. Note that the NCG takes into account the coding rate R, which is included in Eb (energy per information bit), in contrast to the (gross) coding gain, which uses the signal-to-noise ratio Es/N0 . We can see that in order to have a fair comparison between coding schemes, the BER at which the NCG is measured is especially important for the most advanced

IV. FINITE LENGTH PERFORMANCE
In the previous section, we have seen that with a simple toy LDPC code, we can already achieve coding gains in the order of several dB, however, we can also see that there are still significant gaps to the capacity limit. The immediate question that arises is whether these gaps are due to the code construction itself or due to other effects. It turns out that the capacity is not the best measure to quantify the maximum achievable coding gain in this case.
Although the capacity gives a lower bound on the required SNR for successful transmission over a given channel, Shannon’s capacity results assume infinite block lengths, which is not a valid assumption in practical lightwave systems. Much work has been dedicated to get precise bounds on the performance of transmission systems employing codes of finite block length N . A good overview of results can be found in [39] and [40], where the latter provides refined finite length performance bounds. Shannon already provided a lower bound on the code word error probability Pw (frame error rate, FER) [41], in the form Pw ≥ QN (Θs , A), with A = 2Es/N0 = 2REb /N0 . The quantity QN (Θ, A) denotes the probability that an N dimensional Gaussian random vector with mean (A, 0, . . . , 0) and covariance IN ×N (I: identity matrix) falls outside an N dimensional cone of half-angle Θ and Θs is the angle such that the N -dimensional cone of half-angle Θs encompasses a fraction 2−K of the total solid angle of the N -dimensional Euclidean space [39]. Using geometric arguments, Shannon derived an expression for QN (Θs , A) together with approximations that are valid for medium to large N . We refer the interested reader to [39] for more details.
In Fig. 4, we show the approximation of the sphere packing lower bound as described in [39], which gives an easyto-compute and good approximation to the gap to capacity, however, not all conditions to apply the bound are rigorously fulfilled. For instance, Shannon’s sphere packing bound assumes that the channel input is continuous. However, it is shown in [39] that the sphere packing bound is a good approximation

even when the channel input is constrained to be binary and

V. SPATIALLY COUPLED CODES
In the 1960s and the following decades, most coding research focused on block coding techniques, but many practical coding schemes were based upon convolutional codes [5]. With the advent of turbo codes, the rediscovery of LDPC codes, and advances in semiconductor technology, this suddenly changed so that today most new coding schemes are, again, block codes. Note that turbo codes as proposed in [14] are strictly speaking block codes due to the block interleaver, although the component codes are terminated convolutional codes. The trend is, however, to return to convolutional-like structures [44] that can be efficiently encoded and decoded using sliding-window techniques.
In the last few years, a class of spatially coupled code ensembles with several appealing properties has emerged [45], [46]. LDPC convolutional codes [47], which are one particular instance of spatially coupled codes, have been introduced more than a decade ago, but it was only recently noted that the belief propagation thresholds of certain (terminated) protograph-based LDPC convolutional codes approach the MAP thresholds of the underlying ensemble [48], [49]. Thus, contrary to a common belief, introducing structure into LDPC codes leads to a class of (degenerated) realizations of LDPC codes that demonstrate superior performance under belief propagation decoding. This effect of threshold saturation by introducing structure has been analyzed for the binary erasure channel (BEC) in [45] and it has been shown that spatially coupled LDPC codes can asymptotically achieve the MAP threshold of the underlying regular ensemble under belief propagation decoding. Recently, this result has been extended to more general channels and it has been shown in [46] that spatially coupled ensembles universally achieve capacity over binary-input memoryless outputsymmetric channels: most codes in this ensemble are good for each channel realization in this class of channels.
Spatially coupled codes are now emerging in various applications. Two examples mentioned previously are the staircase code [17] and the braided BCH codes presented in [19], which are both rate R = 239/255 codes targeted for 100 GBit/s applications with hard decision decoding. Both codes are basically spatially coupled BCH product codes that allow for a natural windowed decoder implementation. These codes can be interpreted as being generalized spatially coupled LDPC codes with variable node degree dv = 2 and every bit participating in two BCH component codes (instead of simple parity-check codes), where the component BCH codes are able to correct 3 errors each. A. Random Coupling
In the previous Section, we have presented the basic concepts of spatially coupled codes with a deterministic construction [48], [49], [51], [58], [59] which have appealing properties as we have demonstrated by an application example. These codes were originally termed LDPC convolutional codes [47] due to their resemblance with classical convolutional codes [60] with very large memories and sparse taps. As these constructions were difficult to analyze, Kudekar et al. introduced the concept of randomly coupled code ensembles [45], [46] and they coined the term spatially coupled codes.
The random, spatially coupled ensemble can be understood as a more general time-dependent construction where the submatrices forming the convolutional parity-check matrix Hc are time-dependent, i.e., they can be represented as H0(t), H1 (t), . . . Hμ (t). Time-dependent LDPC convolutional codes were already considered in [47]. The randomized construction according to [45] has an additional coupling parameter w, which is closely related to μ. The constraint equations in that code are constructed as follows: for each parity-check constraint equation, we first select with probability 1/w one of the code words (t − w + 1), (t − w + 2), . . . , (t) and within each code word

∞
where close inspection reveals the varying, time-dependent nature of the parity-check matrix. Kudekar et al. showed that for N → ∞, L → ∞, w → ∞ (in that precise order) and codes with large variable node degrees dv , the Shannon limit can be approached with relatively low complexity [45], [46]. There is a broad body of literature devoted to the analysis and understanding of this randomized spatially coupled code ensemble [45], [46], [54], [61] and it is tempting to apply this theory to the deterministic construction introduced previously. However, special care must be taken as the deterministic construction represents only a degenerate special case of the probabilistic construction meaning that the results do not necessarily apply directly.
The term spatially coupled code thus denotes a construction with time-varying, randomized coupling. However, the term spatially coupled code is also often employed slightly differently and is used to denote constructions with small μ (in the range of 1–5) and large N (in the range of several thousand bits) whereas the term LDPC convolutional code often denotes constructions with a large syndrome former memory μ (i.e., large overlap of the submatrices) and small sub-matrices (N small, in the range of several tens to several hundred bits). The toy example we have introduced is strictly speaking an LDPC convolutional code.

B. Selection of Code Parameters

With spatially coupled codes, the code and system designer has access to a broader range of parameters than with conventional codes. The selection of the memory parameter μ is important. It can be tempting to select a small value of μ and also a small value of N as we did in the toy example to obtain a very long code with simple repeating, time-invariant structural elements. However, as it is shown in App. C, there is an intricate relationship between the minimum distance of the code (and thus its correction capabilities) and the selection of μ and N for time-invariant codes, as

dmin ≤ (M + 1)(μM + 1)

(7)

where dim Hi = M × N . This bound, which is en extension of the Singleton bound, shows that either μ or M have to be selected large, in order to achieve a good minimum distance. Alternatively, time-varying constructions may be used. This is in line with the concept of spatially coupled codes (small μ, large N ) versus LDPC convolutional codes (large μ, small N ).

VI. COMBINATION OF CODING AND MODULATION
In this paper, we solely focused on the design of FEC schemes. In future optical communication systems, it is expected that higher order modulation formats will play a crucial role for increasing the spectral system efficiencies. If modulation formats other than BPSK and QPSK are used, a joint consideration and optimization of channel coding and modulation becomes beneficial. Such coded modulation schemes, which may include nonbinary codes, bit-interleaved coded modulation, trellis-coded modulation and multi-level coding, are however outside the scope of this paper. We refer the interested reader to several overview articles [11], [37], [68]–[70] and references therein.

Fig. 10. Convergence speed of slightly irregular code ensembles.
So far, we have only considered regular codes. Irregular LDPC block codes have been used to design codes that perform very well for high bit error rates, but these schemes suffer from relatively high error floors requiring the use of an outer code that leads to inherent capacity losses. In the case of spatially coupled codes, we can use the degree distribution to control the propagation speed of the decoding wave of the code [62]. Using the randomized spatially coupled ensemble introduced in Section V-A and the tool of density evolution [63], which numerically emulates the probabilistic behavior of the decoder, we have measured the number of iterations required to propagate the decoding wave by one step. This is the number of iterations that is necessary until a windowed decoder can advance by one step. For the analysis, we employ the randomized code construction with w = 3 (corresponding roughly to μ = 2) and we construct slightly irregular codes by allowing two different variable node degrees dv ∈ {3, 6}, i.e., each bit participates in either 3 or 6 parity-check constraints, respectively. We vary the fraction of degree-6 nodes av,6 between 0 and 1, where av,6 = 0 denotes a regular dv = 3 code and av,6 = 1 denotes a regular dv = 6 code. For a fixed design rate Rd = 5/6, this gives an average check node degree of dc,avg = (3av,3 + 6av,6 )/(1 − Rd ) = 18av,3 + 36av,6 . We then construct a code with irregular check node degrees dc,avg and dc,avg selected such that the desired average degree is obtained. We plot the contour lines of the required number of iterations at a certain Eb /N0 and for varying av,6 in Fig. 10. We can see that by proper selection of the irregular degrees, we can obtain codes that have an improved decoding convergence, however, we also see that depending on the selection, a worse convergence behavior than for the regular case can result. We also observe that at low convergence speeds (close to capacity operation, large number of iterations required), the optimum value of av,6 is larger than for more realistic values, where the optimum lies at av,6 = 0.2. Spatially coupled codes optimized with respect to the convergence speed have been successfully employed in [64]–[67]. In [62] we have derived simplified bounds on the convergence speed for the BEC (which is a good model of the AWGN channel at high signal-to-noise ratios) which can help to design fast-converging codes.

VII. CONCLUSION
LDPC codes and BTCs, decoded soft and iteratively, are the dominating coding schemes in long-haul transmission. Novel coding approaches like convolutional LDPC codes have the potential to bring even further performance enhancements. By a toy example, we have illustrated the concept and construction behind block-LDPC codes and convolutional LDPC codes and have shown the advantages that can be obtained by convolutional-type LDPC codes. Finally, we have shown some design guidelines for realizing fast-converging codes.
