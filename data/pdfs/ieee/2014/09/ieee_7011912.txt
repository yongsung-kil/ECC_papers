ASIC Design of 7.7 Gbps Multi-Mode 
LDPC Decoder for IEEE 802.11ac 
Thi Hong Tran, Yuhei Nagao, Hiroshi Ochi, Masayuki Kurosaki 
Grad. School of Compo Science and Systems Eng., Kyushu Institute of Technology

Abstract-In this paper, we propose a multi-mode low density 
parity check (LOPC) decoder architecture for high throughput 
wireless communication systems. Throughput of our decoder 
depends on code rate. The main blocks of our architecture 
are independent from value of the check matrix. Based on 
the proposed architecture, we design a twelve-mode Min-Sum 
LOPC decoder for IEEE S02.Uac system and synthesize it in 
ASIC 90nm technology. The ASIC synthesis results show that 
our decoder achieves up to 15.4 bits/cycle (bpc) or 7.7 Gbps at 
500 MHz operating frequency, and has the best efficiency which 
is 1.S9 (bpc/mm2) as compared to the previous works. In this 
paper, we also propose an efficient quantization method that can 
significantly improve the PER performance. The floating point 
and fixed point simulation results show that PER performance of 
the S02.Uac system is improved by at least 1 dB if using LOPC 
instead of BCe. Furthermore, the proposed quantization method 
can improve the LOPC PER performance up to 1.7 dB. 
I. 
INTRODUCTION 
Due to the high demand of wireless communication users, 
throughput of wireless cOlmnunication systems must be un­
remittingly increased. To increase the system throughput, there 
are common ways such as: 1) increasing the number of spatial 
streams, 2) expanding the channel bandwidth, e.g., from 40 
MHz to 80 MHz, 3) adopting high order modulation such 
as 256-QAM, and 4) improving the reliability of the system 
to eliminate the need for re-transmission. Because of the 
limited size of wireless device which results to the limited 
number of equipped antennas, and because of the limitation 
on frequency resource, methods 1 and 2 are not able to be 
utilized in some cases. If using method 3, the reliability of 
the system is degraded due to high density of constellation 
nodes. Finally, the overall throughput may be reduced because 
of the re-transmission request. For that reason, improving the 
reliability of the system as well as using high order modula­
tion are required. To obtain high reliability, employing high 
performance multiple input multiple output (MIMO) detection 
and forward error correction (FEC) is necessary [1]. In [2] we 
have presented our research on MIMO detection. To continue 
the work on improving system reliability, we focus on high 
performance FEC in this paper. 
Among various FEC codes such as binary convolution code 
(BCC), Turbo code, low density parity check (LDPC) code, 
etc., LDPC has been proven to achieve the best performance 
and is close to the Shannon limitation [3]. However, the 
hardware design of LDPC decoding is very complex due to 
multiple operation modes. To cope with the variety of channel 
condition and packet size, a wireless system supports many 
code rates and lengths, respectively. The low code rate results 
to low data rate but high decoding performance. It is preferred 
259 
to use in case the channel condition is not good. On the other 
hand, the high code rate results to high data rate and requires 
good channel condition. The large code length provides higher 
decoding performance but requires more complex circuit than 
the small code length does. Depend on situation, a suitable 
code length and rate would be used. For example, the IEEE 
802.16e WiMAX supports 76 operation modes combined from 
4 code rates (1/2, 2/3, 3/4, and 5/6) and 19 code lengths 
(ranging from 576 bits to 2304 bits). And the IEEE 802.11ac 
Wi-Fi supports 12 operation modes combined from four code 
rates (1/2, 2/3, 3/4, and 5/6) and 3 code lengths (648, 1296, 
and 1944) [4]. Each operation mode has a specific check 
matrix H, while the hardware design of the LDPC decoder 
are commonly dedicated for the pre-defined H. That results to 
a very high complicated LDPC decoder for multiple operation 
modes. Furthermore, long time is needed for developing and 
debugging the decoder if the design is affected too much on 
values of every entries of H. 
In this paper, we propose a multi-mode LDPC decoder 
architecture in which the main parts of the decoder are 
independent from H. A new operation mode can be added 
by simply inserting a new ROM that stores the H into the 
developed decoder. Because of the independence on H, the 
decoder developing and debugging are expected to be simple 
and require less time. Based on the proposed architecture, we 
design a min-sum LDPC decoder for 802.11ac system and 
synthesize it in ASIC. In this paper, we also show that how 
to quantize the log likelihood ratio (LLR) of the codeword 
bits is an important decision that significantly affects to the 
error correcting performance. We then propose an efficient 
quantization method. 
The rest of this paper is organized as follows. Section II 
presents the background of LDPC code and min-sum decoding 
algorithm. Section III describes our LDPC decoder architecture 
as well as quantization method. Section IV is the result 
evaluation in terms of packet error rate (PER) performance 
and ASIC synthesis. Finally, Section V shows our conclusion. 
II. 
BACKGROUND 
A. LDPC Code 
LDPC code is a special class of the linear block codes. It 
was introduced by R. Gallager [5] in 1963. It is presented by a 
sparse parity check matrix H in which H is an M x N matrix. 
If the value of each entry of H is whether zero or one, the 
code is called as binary LDPC code. N is the length of the 
codeword, and M is the number of parity bits.

C. LLR Quantization 
In terms of hardware implementation, we must quantize the 
LLR values of the estimated codeword bits and use a number 
of binary bits to present them. Let denote D as the number 
of bits that presents each LLR value, the range of LLR value 
will be divided into 2D discrete values. Fig. 6 shows the LLR 
values of a 64S bit-length codeword bits. Assume that we only 
quantize the values from -E to E, i.e., inside the red box in 
Fig. 6, the values that are smaller than -E or larger than E 
will be truncated to -E and E, respectively. The quantization 
resolution } will be: 
(11) 
From (11) we can see that the smaller the value of E is, 
the smaller the resolution } will be, and thus the higher the 
accuracy of the quantization will be. However, if E is too small, 
there are many values larger than E and must be truncated. 
The error due to truncation would reduce the accuracy of 
the decoder. In short, determining the threshold value E for 
quatization would be an issue that significantly affects to the 
performance of the LDPC decoder. 
In another aspect, the LLR An of the nth codeword bit is 
computed as (12).

In (12), Pr(ylxn = 0) and Pr(ylxn = 1) present the probabilities 
that the nth bit of the codeword is considered to be zero, and 
one, respectively. If Pr(ylxn = 0) = K x Pr(ylxn = 1) where 
K > 10 or K < 1/10, it is nearly correct to conclude that the 
nth bit is zero, or one, respectively. In that case, the error due 
to truncation may be inconsiderable. 
Based on our research, we propose to select the threshold 
E so that 2.3 < E < 4.6. It means that 100 > K > 10 or 
1/100 < K < 1/10. By doing so, the quantization range can be 
shorten significantly as compared to case of full range which 
is commonly from -15 to 15. The quantization accuracy is thus 
improved, while truncation error is inconsiderable.

RESULT EVALUATION 
A. P ER Simulation 
Fig. 7 shows the packet error rate (PER) performance of the 
S02.11ac PHY system when using BCC and LDPC codes. The 
simulation parameters are: 4 x 4 MIMO, SO MHz bandwidth, 
channel D, 64-QAM and 256-QAM modulation, 2000 packets, 
packet size of 1000 data bytes, 2D Sorter-based K-best MIMO 
detection mentioned in [2]. 
Fig. 7 shows that using LDPC codes can improve the 
PER performance of the system by about 1.1 dB, and l.5 dB 
(PER=lO-l) in cases of 64-QAM, and 256-QAM, respectively. 
To prove the correctness of our proposed quantization in 
Sect. III-C, we have simulated the S02.11ac PHY systems, 
256-QAM modulation, packet size of 500 data bytes. The other 
parameters are the same as mentioned above. We use D = 4 
bits to quantize the LLR values, and select E = 14, E = 7, E = 
3.5, and E = l.75. The simulation results are shown in Fig. S. 
The results show that the PER performance is improved when 
E value is reduced from 14, to 7, and to 3.5. That is because the 
shortening of quantization range has improved the quantization 
accuracy. However, if we continue to reduce the E value to 
l.75, the error due to truncation becomes considerable. The 
PER performance is thus not improved. The PER performance 
is improved by about l.7 dB (PER = 10-1) as compared to 
case E = 14. In addition, with fixed point simulation, PER 
performance of LDPC is much better than that of BCe.

CONCLUSION 
In this paper, we have proposed a high throughput multi­
mode LDPC decoder architecture for wireless communication 
systems. Most blocks of our architecture are independent 
from check matrix value. We have also proposed an efficient 
quantization method which is required for hardware imple­
mentation of the decoder. The floating point simulation results 
have shown that using LDPC code can improve the PER 
performance of 802.11ac PHY system by about 1.1 dB, and l.5 
dB in cases of 64-QAM, and 256-QAM, respectively. The fixed 
point simulation results have shown that applying the proposed 
quantization method can improve the PER performance by 
about 1.7 dB, and that LDPC performance is much better 
than BCe's. Based on the proposed architecture, we have 
designed a twelve-mode LDPC decoder for 802.llac system 
and synthesized it in ASIC 90nm technology. Our decoder 
achieves up to 15.4 bits per cycle or 7.7 Gbps at 500 MHz 
frequency. In terms of efficiency that already taken care the 
different in technology, code length, and number of iterations, 
our design achieves 1.89 (bpc/nun2), which is the best among 
the designs.
