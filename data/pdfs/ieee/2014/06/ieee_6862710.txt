Min-Sum Algorithm-Structure Based Decoding Algorithms for LDPC Codes

V.V. Vityazev, E.A. Likhobabin, E.A. Ustinova
The Department of Telecommunications and Fundamentals of Radio Engineering Ryazan State Radio Engineering University Ryazan, Russia
vityazev.v.v@rsreu.ru,lichobabinea@gmail.com

Abstract—Review of various reduced-complexity min-sum based decoding algorithms for LDPC codes are presented. New BPbased approach for LDPC decoding was proposed, and one new a-min*-min decoding algorithm was presented. Complexity, effectiveness, and average number of iteration for each algorithm are shown.
Keywords-Error correction, LDPC decoding, LDPC codes, minsum algorithm, a-min* algorithm.
I. INTRODUCTION
Low density parity check (LDPC) codes was proposed by R.Gallager [1] in 1963, and later rediscovered by MacKay and Neal [2]. Like turbo codes, LDPC codes can achieve near Shannon limit error performance [3], [4], and represent a very promising prospect for error control coding. Nowadays they are widely used in many applications and telecommunication standards, all modern standards such as DVB-T2, DVB-S2, DVB-C2, WiFi, WiMax, IEEE 802.15.3 used this codes [5][6].
An LDPC code is specified by a parity-check matrix which contains mostly zeros and only a small number of ones. A regular (N,K) LDPC code, with block length N and dimension K, has a parity check matrix H of M ı N - K rows and N columns with exactly dc ones in each column and dr ones in each row. We refer to the M rows of H as checks, and to all the elements of an LDPC codeword as bits. Then each check contains dr bits and each bit is checked by dc checks.
Gallager also proposed algorithms for decoding LDPC codes, the most popular of them is soft decision based, near optimum belief-propagation (BP) algorithm, also known as sum-product algorithm (SPA), and a hard decision based bitflip (BF) decoding algorithm. They are iterative decoding algorithms.
The main advantage of BP decoding algorithm is it high performance, that allows to achieve near Shannon limit, and the main drawback is it complexity, that remains relevance

nowadays for applications with limited computational and energy resources.
The BF algorithm has a reverse disposition. The main advantage is simplicity and the main drawback poor performance.
Due to high complexity of BP decoding algorithm, many authors try to compile advantages of BP and BF algorithms in one algorithm, and a lot of modifications and new ways of decoding LDPC codes were proposed from its invention. The most significant ways of LDPC decoding evolution are to simplify BP algorithm with loss in performance, and to complicate BF algorithm with gain in performance. Nowadays a lot of such modifications are known [7], [8].
In this paper a new approach to decoding LDPC codes was proposed, and one new decoding algorithm for illustration was presented. The new approach is based on simplification of BPalgorithm. The decoding strategy of proposed algorithm is similar to decoding strategy of a-min*[9] and min-sum decoding algorithms. All of them based on the idea that computation of outcoming check node message may be done by finding only two lowest values of reliability in the check. And only two outcoming messages until check nodes update generate, one for node with lowest reliability, and one for other nodes.
The paper is organized as follows. In Section II, we give a brief review of the min-sum algorithm, modifications of minsum algorithm and a-min* algorithm, then in Section III we proposed a generalized a-min* decoding approach and a-min*mindecoding algorithm. Some simulation results are provided in Section IV to illustrate the performance of the proposed min generalized a-min* algorithm. Finally, a conclusion is given in Section V.

II. ALGORITHM MIN-SUM AND ITS MODIFICATIONS

A. Min-sum algorithm
Min-sum algorithm (MSA) can be considered as a further simplification of operation for log likelihoods (LLR) summation in parity-check node decoder [5]. For two independent binary random variables a1 and a2 with probabilities p(ai=x)=px(l), x {0,1} and log-likelihood ratios LLRs Li=L(ai)=log(p0(l)/p1(l)). It can be shown, that LLR of the binary sum A2=a1+a2, defined as:
L(A2)=sign(L1)sign(L2)min(|L1|,|L2|)+g(L1,L2). (1)

For MSA only first part of this sum:

L(A2)=sign(L1)sign(L2)min(|L1|,|L2|).

(2)

One can observe [7], that the highest difference between (1) and (2) is concentrated in the region where the two message values are close. If the input reliabilities to the variable nodes are characterized by high LLR values, the average difference between the min-sum approximation and the BP is expected to be small.

There is loss of performance for MSA in comparison with BP algorithm, but for many practical applications it is enough.

The main advantage of such simplification is that outcoming message from every variable node can be computed by finding only two lowest values of reliability in the check.

The second valuable advantage is that decoder can operate with reliabilities received from the channel without any computations of LLRs. So this algorithm also known as uniformly most powerful belief propagation algorithm (UMPBP).

B. Modifications of min-sum algorithm
Due to their advantages MSA algorithm have widespread usage in different decoding applications, and moreover there are a lot of different modifications of this algorithm [4] [10] [11].

One of them is approximate min*[9] decoding algorithm. It’s one of BP-based algorithms, that have very attractive tradeoff between complexity and performance.

C. Algorithma-min*
Algorithm a-min* reduces complexity of check nodes update. It’s a combination of sum-product (SPA) and min-sum algorithm. Consider (1)

L(A2)=sign(L1)sign(L2)min(|L1|,|L2|)+g(L1,L2)=

= sign(L1)sign(L2)min*(L1,L2).

(3)

For MSA decoding operator min* is equal to

min*(L1,L2) = min(|L1|,|L2|).

(4)

And for a-min* algorithm

min*(L1,L2) = min(|L1|,|L2|)+g(L1,L2).

(5)

Where min(|L1|,|L2|)+g(L1,L2) sometimes called as box-plus operator, and denoted as +.
For a-min* algorithm as is for MSA, in every check node only two outcoming messages computed, the first for bit with lowest reliability, and the second for all other messages.
Since algorithm used LLR summation, any approximation of them may be used.

Generalized a-min* approach can be used to obtain a very simple modification with gain in complexity in comparisonto a-min* decoding algorithm. It’sbased on a very easy modification, that became evident using generalized a-min* approach.

IV. SIMULATION RESULTS
Performance and convergence research were investigated for two LDPC codes. First is a well-known for a number of publications PEG LDPC code N=1008, M=504 [12], and the second is a quasi-cyclic EG N=8176, M = 7156 LDPC code [5]. In both cases maximum number of iterations is 100.
V. CONCLUSION AND FUTURE WORK
Analyses of obtained results for PEG and EG codes show, that obtained a-min*-min decoding algorithm can achieve decoding performance close to BP and a-min* decoding algorithms, with a much lower complexity, that is very useful for applications with limited resources.
A-min*-min algorithm loses BP algorithm in convergence for large signal-to-noise ratios, but achievement bit error rate is quite enough for real applications. Also we can observe, that amin*-min algorithm have an intermediate performance between MSA and a-min* algorithms.
