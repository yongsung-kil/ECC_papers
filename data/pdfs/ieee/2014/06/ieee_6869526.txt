Optimized Encoder Architecture for Structured Low Density Parity Check Codes of Short Length

Silvia Anggraeni, Fawnizu Azmadi Hussin and Varun Jeoti
Electrical and Electronics Engineering Department Universiti Teknologi PETRONAS
Bandar Seri Iskandar, 31750 Tronoh, Perak, Malaysia silvia31804@yahoo.com, fawnizu@petronas.com.my and varun_jeoti@petronas.com.my

Abstract—This paper proposes an architecture for structured low density parity check encoder. The proposed architecture supports the limitation of input/output pins of field programmable gate array using division of information bits. The division of information bits generates latency of encoding. The proposed architecture does not store the required matrix for bitwise multiplication and does not use cyclic shift of barrel shifter. The proposed architecture is investigated using code length below 1000 bits and implementation of high code rate R = 5/6 and code length between 1000 and 2000 bits. Even though this architecture is optimized for short code length, it is shown that the proposed architecture achieves information throughput of 30.178 Gbps and area of 2737 logic element when code length N = 1944 and code rate R = 5/6.
Keywords: low density parity check (LDPC); encoder; architecture; structured codes.
I. INTRODUCTION
Structured low density parity check (LDPC) codes are constructed based on specific pattern. IEEE standards of 802.16e [1] and 802.11n [2] are communication standards which implement structured LDPC codes as their next generation of error correcting codes.
Encoding is done by a transceiver which is established in an encoder. High throughput encoders for standards [1, 2] are presented in [3-12]. All encoding of structured LDPC codes involve bit-wise matrix vector multiplication (BMVM) between information bits and their party check matrices.
The property of cyclic shifting in LDPC codes of standards [1, 2] implements cyclic shift of barrel shifter for solving the BMVM. The implementation of cyclic shifter in standards [1, 2] with high throughput encoder is described by the works in [3-7].
The BMVM of structured LDPC codes can be solved without cyclic shifter. The works of encoder in Gbps throughput are [8-12] which accomplish the BMVM in standards [1, 2] without cyclic shifter. Typical way of solving the BMVM operation stores the required matrix of BMVM in memory like the works [3-9, 12] then does multiplication and addition.
However, the work in [10, 11] does not store the required matrix and eliminate the process of multiplication. The works [10, 11] have shown a comparable area with the cyclic shifter

implementation. Moreover, both works have clock cycles of encoding below 10 clock cycles.
An LDPC encoder can be implemented using field programmable logic array (FPGA). However, the number of I/O pins in FPGA is limited to certain number which is defined by the series of device.
This paper utilizes BMVM method of the work [10]. The difference between the work in [10] and this paper is laid on the design of architecture which implements division for supporting limitation of FPGA I/O pins. The work in [10] employs division of information and parity bits while this paper proposes division of information bits only.
The division of information and parity bits for the work [10] leads to 9-clock cycles of encoding when implemented for short code length N = 1000 bits. In order to improve the encoder performance for small code length, this paper proposes an architecture of LDPC encoder with division of information bits. Investigation of the proposed architecture is done using code length below 1000 bits and high code rate implementation with code rate R = 5/6 and code length in the range of code length N = 1000 and N = 2000.
The remainder of the paper is organized as follows. In section II, encoding and method of BMVM are reviewed. Moreover, the proposed architecture is also presented in this section. Results and discussion of the proposed architecture are shown in section III. The last section is the conclusion.

Typical implementation of encoder will store sub-matrix P in the memory and encoding will be accomplished after the arrival of information bits s. The arithmetic operation of BMVM in (4) requires multiplication and addition between the sub-matrix P and the vector of information s.
In order to minimize area in the proposed architecture, we omit storing of the sub-matrix P and a multiplication of BMVM in (4). The multiplication of BMVM in (4) is replaced by the content of information bits s and is determined by the location of nonzeros in the sub-matrix P.
The proposed encoder employs only the addition of BMVM in (4). Since arithmetic operation of LDPC in this paper is based on binary bits using modulo-2 operations, the addition can be carried out by XOR gates. Therefore, the proposed architecture implements the XOR gates of (4).
A. BMVM
The method of BMVM is derived from [10]. The detail of this BMVM is given below. The bit-wise multiplication between the sub-matrix P and the vector of information s is accomplished in step 3. Step 4 does the addition which is completed by the XOR gates.
BMVM scheme: Step 1. Let the sub-matrix P and the vector s be binary bits. Step 2. Represent nonzeros of the sub-matrix P by column
location. The column location in this step will be used by the information bits s. The result is stored in P '. Step 3. Replace the nonzeros in step 2 by values in the s and save the result in P'' = s(P'(x, y)). Step 4. Do XOR operation in P'' to get the parity bits p.
B. Proposed Architecture
The proposed architecture is given in Fig. 1 which implements the BMVM in (4). The difference between our architecture and the architecture in [10] is on the division used. Our architecture employs division of information bits while architecture in [10] utilizes division of information and parity bits.
The proposed architecture splits the K bits of information into K/Col Div bits. The step 3 which is the multiplication of BMVM is completed by the input of each XOR in each Col Div denoted by s(P'(x, y)). The step 4 which is the addition of the BMVM above is accomplished by XOR module.
Variable of Col Div in the proposed architecture describes the number of division in the information bits. The value of Col Div is based on the proportion between the number of information bits, the parity bits and the number of I/O pins. Increment of the division leads to the increment of area for the proposed architecture. Therefore, the division will be made as small as possible which is Col Div = 2.
If the value of Col Div = 2, the first generation of parity bits is completed within four clock cycles and the second part is

completed by three clock cycles. If Col Div = 2, there are total of seven clock cycles for generation of all the parity bits.
If the I/O pins are not sufficient to generate the parity bits and the information bits of a code, the value of Col Div will be increased. The max value of Col Div is based on I/O pins for supporting the parity and the highest division of information bits.
The main module in our architecture is the XOR module. There are two XOR operation in this module. The first XOR operation does the step 4 of BMVM. In the first XOR, there are Col Div number of XORs units. Each XOR in each Col Div has multiple input. The second XOR operation updates the output in each clock cycle from the first one. Combinational circuit of the XOR module and the incoming information bits generates the parity bits p.

We do not compare the work of [12] in Table I since this work presents the throughput for code length above 2000. We also do not compare the works of [5-7] since there are no reported throughput in Gbps.
Since the I/O pins of Stratix EP1S80F1508C5 are 1212, we would like to investigate the encoder performance below code length N = 1000 and below code length N = 2000. In this case, we choose code length N = 576, N = 960 and N = 1920 derived from standard [1] and code length N = 1944 according to standard [2]. The comparison for code length N =1944 and code rate R = 5/6 is conducted with two reported throughputs of encoder in Gbps.
The cyclic shifter of Table I is intended for the cyclic shift of the barrel shifter. Derived from Table I, the non-cyclic shifter implementation for structured LDPC encoder in Gbps is made by the works in [8-11] while the cyclic shifter implementation is done by the works in [3, 4]. Both implementations of cyclic and non-cyclic shifter have comparable area and clock cycles of encoding.
Based on Table I, the work in [9] has the smallest area for code length N = 960 and N = 1920 since it solves arithmetic of inverse, matrix-matrix multiplication from R-U encoding [14] in the pre-process step and processes matrix vector multiplication into circularly right shifted registers and XORs.

transforms the arithmetic operations of inverse, matrixmatrix multiplication of modified R-U encoding in [15] into BMVM operations.
The work of [11] utilizes a device of Virtex 5 XC5VLX20T with 172 I/O pins while the implementation of N = 1944 and R = 5/6 requires 324 I/O pins for generating the parity bits. It means that the work in [11] does not count the limitation of I/O pins in FPGA.
Meanwhile, the work in [10] considers the limitation of I/O pins and maintains 9 clock cycles of encoding for code length N = 576, N = 960 and N = 1920.
Since the architecture of [10] applies the division of information and parity bits, the clock cycle of encoding could not be minimized for small code length like code length N = 1000. Therefore, we propose an architecture which employs only the division of information bits in order to minimize the latency of encoding for small code length N = 1000. The division of information bits is made to support the limitation of I/O pins in FPGA.
The proposed architecture maintains 7 clock cycles when code length below N = 1000 since the number division is Col Div = 2. The 7-clock cycles of encoding lead the proposed architecture into the highest throughput when code length N = 576 and N = 960.
When code length below N = 2000, which are N = 1920 and N = 1944, the proposed architecture has different clock cycles. When code length N = 1920 and code rate R = 1/2, the proposed architecture completes 13 clock cycles for generating all the parity bits. In order to support I/O pins, the number division is increased into Col Div = 4 for code length N = 1920 and code rate R = 1/2. Even though the Col Div is increased by 4, the proposed architecture has a comparable area and throughput with other works in this parameter of N and R. Based on Table I, it is concluded that the higher the division, the longer the latency of encoding and the larger the area of encoder.
Since the number of Col Div is made according to the proportion between the number of I/O pins, the information and the parity bits utilized in the architecture, the proposed architecture is also suitable for implementation with smaller bits of parity for example implementation of code rate R = 5/6 and above. Comparing a single rate encoder between the work of [11] and proposed one when N = 1944, R = 5/6, the proposed architecture has the smallest area and is able to generate all the parity bits in 7-clock cycles with information throughput of 30.178 Gbps.
IV. CONCLUSION
In this paper, we have presented an architecture for structured LDPC encoder of IEEE 802.16e and IEEE 802.11n standards with division of information bits. The division of the information bits determines clock cycles and area of encoder. The higher the division of the information

bits, the longer the clock cycles and the larger the area of
encoder.
Even though the proposed architecture is optimized for
short code length, it is shown that the proposed architecture
is able to achieve information throughput of 30.178 Gbps
with area of 2737 logic element when code length N = 1944
and code rate R = 5/6.
