On the Generalized LDPC Codes from Multiple Component Codes Suitable for High-Speed Optical Transport

Ivan B. Djordjevic* and Ting Wang
*University of Arizona, Department of Electrical and Computer Engineering, 1230 E. Speedway Blvd., Tucson, AZ 85721, USA NEC Labs, Princeton, NJ, USA

Abstract: A class of GLDPC-codes is proposed consisting of multiple local-codes suitable for code-rate-adaptation in high-speed optical-transport-networks, providing excellent coding-gains. GLDPC-decoder for this class of codes is more suitable for parallelization in FPGA/ASIChardware compared to LDPC-decoder.
OCIS codes: (060.0060) Fiber optics and optical communications; (999.9999) GLDPC codes.

1. Introduction
As the response to these never ending demands for higher data rates, 100 Gb/s Ethernet (100 GbE) has been already standardized, and effort has moved to 400GbE/1TbE. It has become evident that soft-decision FEC represents one of the key enabling technologies for the next generation of optical transport networks. Several attractive classes of LDPC codes including quasi-cyclic (QC) [1], convolutional [2], staircase [3], and spatially-coupled [4] LDPC codes have been proposed recently as next generation optical transport networks enabling LDPC codes. These various classes of LDPC codes, listed above, can all be derived from QC-LDPC code design as shown in [1].
In this paper, we show that the generalized LDPC (GLDPC) coding, introduced in [5]-[7], represents an excellent candidate for ultra-high-speed optical transport thanks to its low decoding complexity and good BER performance, in particular for column-weight-3 global codes. To improve the error correction strength of GLDPC coding we have proposed the usage of multiple component local codes; such as Hamming, BCH or Reed-Muller (RM) codes; instead of a single local code used in [5]-[7]. To solve for low code rate problem, typically associated with GLDPC coding, we propose the class of GLDPC codes composed of single-parity checks and several local block codes. Since the MAP decoding of a single-parity check is trivial, while the overall complexity of MAP decoder for RM code, based on fast Hadamard-Walsh transform (FHWT) [8] is of order nlogn (n-the codeword length of local RM code), this class of GLDPC codes represents a promising candidate for ultra-high-speed implementation, as the decoding complexity is lower compared to LDPC counterparts. We demonstrate that the record coding gains of proposed GLDPC codes, derived from multiple component codes, can be obtained. Finally, we show that the proposed class of GLDPC codes enables almost continuous tuning of the code rate. Therefore, the proposed class of GLDPC coding is extremely suitable for rate adaptive coding as well as adaptive modulation and coding.

2. Proposed GLDPC Codes Composed of Multiple Local Codes

Since the classes of GLDPC codes introduced by Boutros et al. [5] are inferior in BER performance to the class

of GLDPC codes due to Lentmaier and Zigangirov [6], called here LZ-codes, as shown in [7], in the rest of this

paper we restrict our attention to the class of LZ-codes derived from multiple component codes. LZ-codes are

obtained starting from a global LDPC codes by replacing every row of a global LDPC code by a parity-check matrix

of a single local code. (For additional details, please refer to [6]).

Notice that in [5]-[7] only GLDPC codes based on a single local code were studied. In this section, we apply a

different strategy. The i-th row of a global code in our design is replaced by the parity-check matrix of the s-th local

code Hs (s=1,…,S). Clearly, if the number of rows in a global code is larger than the number of available local codes, the repetition of certain local codes is unavoidable. The basic restriction in this design is that row-weight of

the s-th row in global code must be equal to the codeword length of employed local code ns. The global LDPC code could be either a regular or an irregular LDPC code. For ultra-high-speed applications, such as beyond 100 Gb/s

serial optical transport, the use of global LDPC codes of row-weight larger than 2 leads to unacceptably low overall

code rates of GLDPC codes. This is the reason why regular (irregular) LZ codes based on global codes of column-

weight of 2 were considered in [7]. On the other hand, by using this new strategy, in order to improve the code rate, some of local codes could be simple parity-check equations, whose MAP decoding complexity is trivial. For

instance, if only single-parity check codes and (n,k) local code are used in regular GLDPC code design, the

corresponding code rate R can be bounded as follows:

In conclusion, a class of GLDPC codes has been proposed composed of multiple local codes, providing excellent coding gains. The MAP decoding of local Reed-Muller codes employed in this GLDPC code design is feasible in terms of complexity. The decoder of proposed class of codes is much more suitable for parallelization than corresponding LDPC code. Finally, the proposed GLDPC codes enable almost continuous tuning of code rate.
