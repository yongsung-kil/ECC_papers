LDPC Code Construction as a Generalized Concatenated Code

Igor Zhilin, Victor Zyablov Institute for Information Transmission Problems
Moscow, Russia {zhilin, zyablov}@iitp.ru

Abstract—We propose a construction of general concatenated code that uses low-density parity check codes as component codes. This construction itself can also be considered as lowdensity parity-check code and decoded using Belief Propagation algorithm. Performances of several decoders are compared; it’s shown that this code can be efficiently decoded by proposed soft decoding algorithm or combining BP and proposed algorithm.
I. INTRODUCTION
In this work we propose a binary low density parity-check (LDPC) code construction that can be decoded by several soft decision decoders. Our construction is based on generalized concatenated (GC) code [3]. Unlike Gallager codes [1] that are mostly decoded by well-known belief propagation decoder [4], this construction can also be decoded by a new algorithm that we propose. It is a soft decoding algorithm, which is based on concatenated code decoding algorithm.
Encoding of this construction is essentially the same as for the construction presented in [5], thus we omit it for the sake of brevity.
Our main intention in this work is to compare performance of different decoders for a particular code. We didn’t optimize this construction for maximizing error correction performance and coding gain. It would be a different task that requires search of various code parameters, which won’t necessarily be the same for each decoder. Our task was to compare performance of different decoders in the same conditions.
The work is organized as follows. In section II we introduce the construction in subsection II-A and describe generation of its parity check matrix in subsection II-B. In section III we describe soft decoder based on decoder of concatenated codes in subsection III-A and Belief Propagation and hybrid decoding in subsection III-B. We show imitational modeling results in section IV and discuss them in section V.

III. DECODING

A. Soft decoding

The main decoding algorithm under consideration is soft decoding of this construction as concatenated code. It is an iterative algorithm which works in the following way.

From channel it gets a matrix of a posteriori probabilities P r{xi = 1|yi}, where xi and yi are inputs and outputs of the channel respectively. Since these values are actually probability distributions of symbols from GF (2) we will refer to them as PDs. We will denote matrix of these received values C. The
algorithm includes the following steps:

1) We calculate S using C as an input. We can denote it as a matrix multiplication S = H0 · C, but it has an important difference from the same formula in code definition: S and C are matrices which elements are probabilities of corresponding binary digits being 1.

It’s a code of length 3 and rate 1/3, its syndrome can actually be computed in two ways.

The first one is computing PD of each symbol individually using PDs of symbols it depends on, for example s2 = c1 ⊕ c2 = c1(1 − c2) + (1 − c1)c2. The second one includes computing PDs of the whole vectors (c1, c2, c3)T and (s2, s3)T . It is straightforward: a column Ci gives us the probability distributions of each symbol. We can use them to compute probability distribution of eight possible binary words of the inner code, these words correspond to four possible (values of) syndromes. IV. SIMULATION
Proposed decoders were implemented for computer simulation. We tested their behavior in a channel with binary phase-shift keying (BPSK) and additive white Gaussian noise (AWGN).
Total code length was n = 3000, rate R = 0.5, upper outer code length and rate nU = 1000, RU = 0.7, lower code nL = 2000, RL = 0.4.
Simulation results are given on figs. 1 and 2.
V. CONCLUSION
It’s easy to see that belief propagation algorithm shows rather high error floor just after the point it starts to work. Soft concatenated decoder doesn’t show any error floors on the probabilities we could reach. Unfortunately, proposed hybrid construction has higher effect of error floors than soft

concatenated decoder but it’s a significant improvement over Belief Propagation algorithm.
Our assumptions on the reason of error floor exhibited by Belief Propagation algorithm is a significant role of short cycles on Tanner graph [2] in the parity check matrix, which doesn’t affect soft concatenated decoder.
The error floor turns out to be an effect of used decoder but not the code itself.
