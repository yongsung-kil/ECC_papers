An Encoder/Decoder with Throughput over Gigabits/sec for Rate-compatible LDPC Codes with Wide Code Rates

Zhiyong He1,2, Paul Fortier2, Sébastien Roy3, and Hushan Xu1
1 Institute of Modern Physics, Chinese Academy of Sciences, Lanzhou, China, 730000 2 Department of Electrical and Computer Engineering, Laval University, Quebec City, Quebec, Canada, G1K 7P4 3 Department of Electrical and Computer Engineering, University of Sherbrooke, Quebec, Canada, J1K 2R1

Abstract— A challenge in the design of rate-compatible (RC) low-density parity-check (LDPC) codes is how to maximize the range of code rates. In this paper, we propose a class of RC LDPC codes with a very wide range of code rates. To ensure linear encoding, dual-diagonal form for the parity part of the mother parity-check matrix is used. Constructed from shifted identity matrices, the proposed codes are particularly well-suited for the high-speed implementation of parallel encoders and parallel decoders. To widen the range of code rates, we have proposed an optimal transmission scheme, which keeps the optimal degree distribution unchanged for the mother code and all daughter codes. Thus, the proposed technique pushes the upper bound of code rates to 0.96, which is the highest rate in RC LDPC codes in the world, based on our best knowledge. The implementation results into field programmable gate array (FPGA) devices indicate that a parallel encoder (decoder) for the proposed RC LDPC codes is capable of reaching a throughput of 7.2 (1.8) Gigabits per second using a clock frequency of 150 MHz.
I. INTRODUCTION
A critical issue in the design of rate-compatible (RC) lowdensity parity-check (LDPC) codes is how to widen the range of code rates, in order to maximize the total channel capacity and provide proper service according to channel conditions. Among existing approaches, puncturing provides an effective way to acquire a series of RC codes, where higher-rate daughter codes can be derived from a mother code by puncturing some parity bits (e.g. [1-5]). The important challenge in the code design with the puncturing approach is to keep the performances of the mother code and all daughter codes close to the capacity. Because puncturing may modify the degree distribution, which is a more important factor for determining the decoding performance, it is very difficult to obtain good daughter codes with very high code rate.
In this paper, we propose a class of RC LDPC codes with a wide range of code rates. By proposing an optimal transmission scheme and employing a zero-filling decoding algorithm, the approach to obtaining RC codes does not modify the degree distribution of the daughter codes. Thus, the proposed approach pushes the upper bound of code rates to a very high value, such as a rate of 0.96. Constructed from shifted identity matrices, the proposed codes are particularly well-suited for the high-speed implementation of parallel encoders and parallel decoders. The implementation results

C. Rate-compatible code versus rate-adaptive code
In [8-9], we have designed a class of rate-adaptive LDPC codes represented by a mother matrix, which is also comprised of an array of circularly shifted identity sub-matrices. The parity-check matrices of daughter codes with higher rates are deduced by removing the uppermost rows of the mother matrix, i.e. the rates are adapted by row-removing approach. On the other hand, the daughter codes of rate-compatible LDPC codes in this text are obtained by puncturing some parity bits from the mother code, i.e. the parity-check matrices of daughter codes with higher rates are deduced by removing some columns in the parity part of the mother matrix. IV. DECODING ALGORITHM AND PERFORMANCE ANALYSIS
The proposed RC LDPC codes can be effectively decoded using the sum-product algorithm, which performs the summation of all extrinsic messages in each column and the product of all extrinsic messages in each row. Instead of the separate row-column processing in the traditional sum-product algorithm, we use a joint sum-product decoding algorithm which incorporates the calculation of the extrinsic messages in the column processing into the row processing [10-11], i.e. a joint row-column processor is used to process each row from the lowermost row to the uppermost row in each iteration.

A great advantage of the proposed RC LDPC code is that

the proposed method pushes the upper bound of code rates up

to R=0.96 by transmitting only one parity bit vector, i.e. p24. A natural question is why bit vector p24 is sent. The frame error rate (FER) at the 50th iterations versus the signal-to-noise ratio

(SNR) per bit Eb /N0 are shown in Fig. 2 for the codes with a length of 1800 and a rate of 0.96. It is clear that the code with

bit vector p24 attains the best performance. To explain why the transmission scheme by sending vector pJ attains the best performance, we re-write (3) and (4) as follows.

A. Zero-filling decoding algorithm

Prior to the introduction of a zero-filling decoding algorithm for the proposed RC codes, we discuss the decoding of the mother code first. Let H be the mother matrix which consists of J×q rows and (K+J)×q columns, where (K+J) and J are the numbers of block-columns and block-rows. The mother code is decoded with a joint row-column processor from the lowermost row to the uppermost row within H in each iteration. During the decoding period, the joint rowcolumn processor reads both the log-likelihood ratio (LLR) Un and the extrinsic messages vmn, and then calculates a new loglikelihood ratio U’n and new extrinsic messages v’mn, with 1 ≤ m ≤ J × q and 1 ≤ n ≤ (K + J ) × q . At the first iteration,

the extrinsic messages vmn are set to be zeros, while the LLR of the n-th bit is extracted from the received signal, i.e.

U n = log( pn0 / p1n ) ,

(10)

where pn0 and p1n are the possibilities of the n-th bit to be 0

or 1, respectively. For the mother code with the lowest rate, all

bits including the systematic bits and parity bits are sent. The

LLR of all bits can be extracted from the received signal.

Zero-filling decoding algorithm: For the daughter codes,

some parity bits are not sent. The n-th bit is taken as a zero bit,

if the n-th bit is not sent. In other word, a large positive

constant is assigned to Un, if the n-th bit is not sent.
