Abstract—This paper presents an efficient min-sum-based decoder for high-rate low-density parity-check (LDPC) codes, where the first minimum and second minimum values are stored in registers. In order to meet a strict cost requirement imposed by NAND flash applications, we provide different upper limits for the first and second minimum values. Furthermore, we use non-uniform quantization for the second minimum value so as to reduce storage complexity. In order to enhance the errorrate performance, the normalization factor is determined based on the difference between the first two minimum values. Using the proposed techniques, a reduction in gate count of 13.36% can be achieved without suffering any degradation in errorrate performance. The implementation results for a rate-0.896 length-18624 layered decoder show that this decoder can achieve a throughput of 765.24 Mb/s at a clock frequency of 166 MHz with a gate count of 620K.
Index Terms—Low-density parity-check (LDPC) codes, NAND flash, non-uniform quantization, decoder.
I. INTRODUCTION
Recently, NAND flash memory has become increasingly popular since it can be used to store large quantities of data in a small device. Conventionally, NAND flash memory uses Bose-Chaudhuri-Hocquenghem (BCH) codes to mitigate the effect of memory cell errors. Due to the process scaling and the use of multilevel cells, BCH codes are no longer sufficient for multilevel NAND flash and, hence, powerful error correcting codes, such as low-density parity-check (LDPC) codes [1], are required.
LDPC codes can be iteratively decoded by passing messages between check nodes and variable nodes, where either the sum-product algorithm [2] or the min-sum algorithm [3] can be used to compute messages. For NAND Flash applications, high-rate and low-cost decoders are required. To meet the low-cost requirement, a less complex normalized min-sum algorithm (NMSA) [4] is usually used to implement an LDPC decoder in hardware. In order to increase the convergence speed in the error rate and, hence, increase throughput, layered [5][18] or shuffled [6][19][20] scheduling are usually adopted in decoder implementation [12][14]. Although the decoders reported in [12] and [14] are quite efficient, there is still room for enhancing decoder efficiency if the NMSA is refined.
In [7]-[11], several methods for refining the NMSA using different normalization factors for the first two minimum

values were proposed. The works presented in [7],[8] and [9], however, require additional logic cells to compute the variable-to-check (V2C) messages in order to determine the normalization factors. For the methods presented in [10] and [11], as well as additional logic cells, the previous V2C messages stored in memory are used to determine the normalization factors. Consequently, these methods are not suitable for efficient hardware implementation.
This paper presents a refined NMSA which is suitable for hardware implementation. We provide different upper limits for the first and second minimum values and use non-uniform quantization for the second minimum value such that deep quantization can be realized without suffering any degradation in error-rate performance. In order to enhance the error-rate performance, the normalization factor is determined based on the difference between the first two minimum values. Based on this refined algorithm, a check-node unit is thus proposed. In addition, a decoder for high-rate LDPC codes, which are suitable for NAND flash applications, is implemented. The implementation results for a rate-0.896 length-18624 layered decoder show that a reduction in gate count of 13.36% can be achieved using the proposed techniques. This decoder can achieve a throughput of 765.24 Mb/s at a clock frequency of 166 MHz with a gate count of 620K. Compared to the stateof-the-art decoders presented in [12] and [14], the proposed decoder can also achieve a higher throughput-to-area ratio.
The remainder of this paper is organized as follows. Section II reviews layered LDPC decoding using the NMSA. Section III describes the proposed techniques and the check-node unit. Section IV provides the implementation results and comparison. Finally, Section V concludes this paper.
II. BACKGROUND
A binary LDPC code can be defined using a sparse paritycheck matrix (PCM) and can be iteratively decoded through a bipartite (Tanner) graph corresponding to the PCM. In the Tanner graph, check node i corresponds to row i of the PCM, and variable node j corresponds to column j. There is an edge between check node i and variable node j if the (i, j) element of the PCM is 1. III. PROPOSED LOW-COMPLEXITY CHECK-NODE UNIT
A. Effect of deep quantization
In order to reduce the complexity of the decoder, we apply deep quantization to the first two minimum values. However, when this is applied, a severe error floor is observed. The reason for this outcome is as follows. It can be shown that (3) can be re-written as:
L(jk) = L(jk−1) + ∆R, ∆R = (Ri(,kj) − Ri(,kj−1)). (6)
It can be seen from (4) that the magnitudes of the C2V messages come from either nM in1 or nM in2 and, hence, will become saturated if these two values are saturated. The saturation of C2V messages is more likely to happen as the number of iterations increases. If there are no sign changes between Ri(,kj) and Ri(,kj−1) and the first two minimum values are saturated, the result will be ∆ R = 0 and, hence, the check node loses its error-correction capability even though there are still errors in the neighboring variable nodes.

B. Error-correction process
In order to apply deep quantization to the first two minimum values without producing any degradation in errorrate performance, we perform the following analysis. From simulation, it can be observed with a high probability that the signs of Q(i,kj) and Lj(k−1) are the same. As a result, the authors of both [15] and [17] used the signs of the V2C messages (or, equivalently, the total sign), to execute an early-termination function with negligible degradation in performance. Consequently, the following discussion is based

on the assumption that the signs of Q(i,kj) and L(jk−1) are the same. An erroneous code bit (variable node) j in the (k − 1)th iteration means that the sign of L(jk−1) and, hence, the sign of Q(i,kj), is not correct. If we want to correct it in the k-th iteration, the signs of Qi(,kj) and Ri(,kj), according to (3) should be different. This means that from (4) the associated total sign ST(k,i) should be -1 . This kind of check node is called an unsatisfied check node. In contrast, if ST(k,i) = 1, the signs of Qi(,kj) and Ri(,kj) are the same. This implies that the signs of Qi(,kj) and Lj(k) will be the same. Consequently, check node i cannot correct its associated erroneous bits if ST(k,i) = 1. This kind of check node is called a missatisfied check node. Fig. 1 shows examples for unsatisfied and missatisfied check nodes.
In decoding, a check node with either one or two neighboring erroneous variable (bit) nodes dominates the unsatisfied and missatisfied cases, respectively. Consequently, the following discussion is based on this scenario. From (4), it is easy to determine that for each check-node operation there is only a single Min2 receive node, which receives a C2V message with a magnitude of nM in2, and the others are Min1 receive nodes. As a result, in Fig. 1, we only show the nMin2 receive node for simplification. Since the reliability of the Min1 supply node is the lowest, the probability that this node contains an error is higher compared to other variable nodes. Hence, in Fig. 1, the error occurs in the Min1 supply node.
The error-correction process is performed to reverse the LLR sign of an erroneous node using the existing correct nodes, according to (3). Obviously, a check node should give an erroneous node a value with the opposite sign of the erroneous bit so that it is possible to correct it. However, for a missatisfied check node, the reliability of all connected nodes will be increased and, hence, any error correction is impossible. In contrast, for an unsatisfied check node, if we want to correct the erroneous bit, it must be the Min1 supply node and the nM in2 value should be larger than the magnitude of the V2C message for the Min1 supply node. Consequently, error correction is much easier to perform on erroneous node (the Min1 supply node) with a lower reliability and correct nodes (including the Min2 supply node) with a higher reliability.
Fig. 2(a) shows a numerical example for the case given in Fig. 1(b), where the Min1 supply node is an erroneous node with a V2C message of -5 and nM in1 = 4, where α = 0.8 is used. In addition, the Min2 supply node is a correct node with a V2C message of -9 and nM in2 = 7.2. In the next decoding phase, we can calculate an a posteriori LLR of 2.2 with a correct sign for the Min1 supply node. However, there are some negative effects on the correct nodes caused by the erroneous node since the reliability of these corrected nodes has been lowered, as shown in Fig. 2(b). From this example, the upper limit of nM in2 cannot be too low in order to correct any errors that occurred in the Min1 supply node, and the upper limit of nM in1 can be lower in order to prevent the erroneous node from affecting the other correct nodes.

C. Proposed check-node unit
From the discussion provided in Section III.B, fewer quantization bits can be used to represent nM in1, but the upper limit for nM in2 cannot be too low. In order to reduce storage complexity, we consider reducing the quantization bits of the stored nM in2 value. However, if we directly truncate the quantization bits of the stored nM in2 value, it will lead to a significant reduction in performance due to severe saturation in the C2V message. As a result, we perform non-uniform quantization In addition, a greater number of quantization bits are used to represent the second minimum value in the computational units in order to prevent saturation, while fewer bits are used to represent the second minimum value in storage so as to reduce complexity.
In the conventional NMSA, the normalization factor is fixed. In the proposed NMSA, the normalization factor depends on the difference between Min1 and Min2 and, hence, is dynamic. V. CONCLUSION
We have presented an enhanced NMSA and the corresponding check-node unit, where different upper limits are used for the first two minimum values. A rate-0.896 length18624 layered LDPC decoder for low-cost NAND flash applications has been implemented in 90 nm CMOS technology to demonstrate the proposed techniques. The proposed decoder can achieve a reduction in gate count of 13.36% without suffering any degradation in error-rate performance using either soft-decision or hard-decision decoding compared to the conventional decoder.
