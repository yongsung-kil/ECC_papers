Technologies for the next generation of digital terrestrial television broadcasting
-A study on Spatially Coupled LDPC codes-

Shingo Asakura, Takuya Shitomi, Susumu Saito, Tomohiro Saito, and Kazuhiko Shibuya

Abstract— NHK is conducting research on the next generation of digital terrestrial broadcasting to enable large-volume content services such as 8K Super Hi-Vision. In our previous study, we used a low-density parity-check (LDPC) block code scheme and proposed a decoding method that uses the channel response of a dual polarized MIMO transmission to further improve the bit error ratio (BER) performance [3]. LDPC block codes are robust forward error correction (FEC) codes and are Shannon-limit-approaching over an additive white Gaussian noise (AWGN) channel. Despite this feature, there is still room for improving the BER performance. In this paper, we propose spatially coupled (SC-) LDPC codes to further improve the BER performance. We also report on our verification using a computer simulation.
Index Terms—Low-Density Parity-Check (LDPC) codes, Spatially Coupled- LDPC codes, Dual-polarized MIMO, Ultra-multilevel OFDM
I. INTRODUCTION
The NHK Science and technology research laboratories (STRL) are conducting research on large-capacity transmission technology for next-generation digital terrestrial broadcasting systems that will provide large volume content services such as 8K Super Hi-Vision (SHV). As part of this development, we proposed large-capacity transmission technologies using ultra-multilevel (e.g., 1024QAM or 4096QAM) orthogonal frequency-division multiplexing (OFDM) and dual-polarized multiple-input multiple-output (MIMO) [1][2]. In our previous studies, we proposed an LDPC decoding method using the channel response of dual polarized MIMO transmission to further improve the BER performance.

This method is based on iterative computation of the log likelihood ratio (LLR) in the sum-product decoding process, and it has better bit error ratio (BER) performance than the conventional decoding method if the ultra-multilevel OFDM scheme is applied. In this paper, we propose spatially coupled (SC-) LDPC codes as FEC scheme. SC-LDPC codes are based on LDPC convolutional codes [4], and in theory, they are more robust than conventional LDPC block codes. Also, for their decoding method, the sum-product algorithm, which is a well-known decoding method of conventional LDPC block codes, can be used. We also report on our verification of SC-LDPC codes in computer simulations.

Its parity bits are only near the diagonal elements of the parity-check matrix, which is in contrast with LDPC block codes whose parity bits are randomly spread in the matrix. C. SC-LDPC encoding and decoding
The parity-check matrices of SC-LDPC codes are used in the same way as the conventional parity-check matrix. Consequently, the SC-LDPC encoding and decoding

algorithms are essentially the same as those of conventional LDPC block codes. That is, the sum-product algorithm can be applied to the SC-LDPC decoding. The structure of the SC-LDPC matrix is such that the memory needed in the decoding process can be reduced. In the sum-product algorithm, message passing is done between the variable node and check node and needs an amount of memory that depends on the code length. In the SC-LDPC decoding process, each variable node is connected to a nearby check node. This means the temporary memory size used in decoding process is smaller than in the conventional scheme. In other words, this characteristic of the matrix shows that the memory used in decoding process is not of the order of the length of SC-LDPC code but rather of the order of the size of the component base matrix. This means there is an advantage to using the conventional decoding process when SC-LDPC codes are implemented in hardware. With reference to Figure 3, the decoding method is as follows: 1. Perform the row operation at HU_0, HL_0 and HU_1. 2. Perform the column operation at HU_0 and HL_0. The decoding process at HU_0 and HL_0 is completed. 3. The memory used by HU_0 and HL_0 is released, and that to be
used by HU_2 and HL_1 is reserved. The row operation at HU_1 is already done. 4. Perform the row operation at HU_2 andHL_1. 5. Perform the column operation at HU_1 and HL_1. The decoding process at HU_1 and HL_1 is completed. 6. The memory used by HU_1 and HL_1 is released, and that to be used by HU_3 and HL_2 is reserved. The row operation at HU_2 is already done. The process is iterated over HL_i and HU_i (i<L). Furthermore, the longer the length of the LDPC code is, the better its BER characteristics become. We can easily generate LDPC codes whose length is long enough to perform well, and we can reduce the memory required in the decoding process because it depends on the code length of the component base matrix rather than on that of the SC-LDPC code. In other words, we can obtain robust LDPC codes without having to increase the amount of memory.
III. VERIFICATION
We compared the BERs of conventional LDPC block codes and SC-LDPC codes in an additive white Gaussian noise (AWGN) environment. We used DVB-T2 codes, which are well-known matrices of LDPC block codes. The code rates were 2/3, 3/4, and 5/6, and the SC-LDPC codes only had a small rate loss. The transmission signal format was based in the 32k-FFT system in dual polarized MIMO transmission. The number of iterations was 100, enough to achieve the best result. The code length was about 2.6x105. The transmission condition was ideal, and there were no cross-polarization components between the horizontal and vertical polarizations. A. Fundamental BER characteristics
The carrier modulation scheme was QPSK. Bit interleaving wasn’t applied because there is no improvement if the carrier modulation scheme is QPSK. B. Advantage of LLR iterative computation
In our previous work, we found that the LLR iterative computation in the decoding process of the LDPC block code, called the “pseudo LLR computation”, is superior to the conventional decoding method if an ultra-multilevel QAM scheme such as 1024QAM and 4096QAM is used [3]. Here, we verified this decoding method with SC-LDPC codes and confirmed whether there is advantage or not. Figures 7 and 8 show the CNR versus the BER characteristics in an AWGN environment. A bit interleaving scheme wasn’t applied because we wanted to compare the characteristics of forward error correction (FEC) between SC-LDPC and conventional LDPC codes. Likewise, the computer simulation showed that a pseudo LLR computation in decoding process of SC-LDPC code is also superior to that of conventional decoding if the ultra-multilevel QAM scheme is used. If 1024QAM is applied, the improvements for the code rates of 2/3, 3/4, and 5/6 were 0.4dB, 0.1dB, and none. If 4096QAM is applied, the improvements were 0.8dB, 0.3dB, and 0.05dB.

IV. CONCLUSION
We proposed SC-LDPC codes for the next generation of digital terrestrial broadcasting system. We generated the trial parity-check matrix of SC-LDPC codes with code rates of 2/3, 3/4, and 5/6, and compared the BER characteristics of these codes with conventional LDPC block codes in a computer simulation. The results of the simulation confirmed that the BER characteristics of SC-LDPC codes are superior to those of conventional LDPC block codes. Furthermore, we confirmed that the LLR iterative computation in the decoding process is superior to conventional decoding method if the ultra-multilevel OFDM scheme is applied. In the future, we will optimize the matrix and generate matrices with various code rates and code lengths.
