Low-Density Parity-Check Codes Based on Partial Unit Memory Codes
Fedor Ivanov, Victor Zyablov Institute for Information Transmission Problems,
Russian Academy of Sciences Moscow, Russia
{fii, zyablov}@iitp.ru

Abstract—In this paper we describe ensemble of Low-Density Parity-Check (LDPC) block codes based on Partial Unit Memory (P)UM codes. We study a decoding performance of received ensemble in the case of different parameters of constituent LDPC codes. The results of simulation of obtained code constructions for an iterative ”belief propagation” (Sum-Product) decoding algorithm, applied in the case of transmission of a code word via a binary channel with an additive Gaussian white noise and BPSK modulation, are presented.

II. QUASI-CYCLIC LOW-DENSITY PARITY-CHECK CODES
R. Gallager was the first to describe pseudo-random code construction with low-density parity-check (LDPC codes) and suggested the algorithm of generation of the check matrix H of these codes [7]. Apart from the random LDPC codes the algebraic LDPC codes are often used. Particularly, if the check-matrix of the LDPC code H consists only of cyclic shifts of a unit matrix I, it is called quasi-cyclic [8]. In this

paper we will employ such codes as a constituents in UM

codes. First, let us determine one particular way to obtain an

ensemble of quasi-cyclic codes based on permutation matrices.

Definition 1. The arbitrary code C ∈ EQC (l, n0, m, I) will be called quasi-cyclic LDPC code based on permutation matrices with base matrix I.

It should be noted that the way discussed above is not the only method to obtain quasi-cyclic LDPC code based on permutation matrices. We can choose an arbitrary permutation matrix P instead of I and then construct the parity-check matrix using cyclic shifts of columns (rows) of P. So our ensemble EQC (l, n0, m, I) is the special case of an ensemble EQC (l, n0, m, P ).
One of the most important characteristics of LDPC code is the absence of the cycle of a certain length. The cycle of length 4 can be understood as the formation in check matrix a rectangle, which vertices are ones. The absence of the cycle of length 4 can be defined with the help of scalar product of all rows (or columns) of the check matrix. If every pairwise scalar product of all rows (or columns) of the check matrix equal 0 or 1 (in real sense), that means the absence of the cycle of the length 4.
There is a rather simple criterion of 4-cycles absence in parity-check matrix of code from EQC (l, n0, m, I) [9].

IV. SIMULATION RESULTS
A MatLab functions was written for generating parity-check matrices of LDPC codes based on (P)UM codes. Simulation was made by methods of simulation modeling with the use of MatLab. For an information transmission channel, we chose a binary BPSK channel with additive white Gaussian noise. For a decoding algorithm, we chose an iterative algorithm SumProduct with ”soft input” working with a code representation in the form of a bipartite Tanner graph. The maximum number of iterations was limited by 50.

Fig. 1. Bit error probability versus signal-to-noise ratio on code symbol (Es/No) for different (P)UM LDPC codes, N = 30000, R = 0.8
For the rate R = 0.8 and N = 30000 five different codes were generated.

3) Matrices H0(t), t = 0, 1, 2, 3 are from EQC (3, 15, 500, I) and H1(t), t = 1, 2, 3, 4 – from EQC (1, 5, 1500, I). The resulted (P)UM LDPC is (4, 20) regular code with the length N = 30000. In the plot 1 it is denoted by (3, 15) + (1, 5).
