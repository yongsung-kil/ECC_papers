The Error-correcting Capabilities of Low-complexity Decoded H-LDPC Code as Irregular LDPC Code

Pavel Rybin Institute for Information Transmission Problems,
Russian Academy of Sciences Moscow, Russia prybin@iitp.ru

Abstract—This paper deals with the Low-Density ParityCheck codes with the constituent Hamming codes (H-LDPC codes) and two different iterative hard-decision low-complexity decoding algorithms. Both algorithms are based on the same main idea: the decreasing of the syndrome weight on each step of the decoding algorithm. The first decoding algorithm uses the properties of the constituent Hamming code. The best known lower-bound on the guaranteed corrected errors fraction for the H-LDPC codes under the first decoding algorithm was obtained in 2011. The second decoding algorithm considers H-LDPC as the irregular LDPC code and uses the well-known majority decoding algorithm. The lower-bound on the guaranteed corrected errors fraction for H-LDPC code under the second decoding algorithm is introduced for the first time in this paper. Numerical results for the lower-bound, obtained in this paper for H-LDPC code under the second decoding algorithm, significantly exceed the numerical results for the best known lower-bounds, obtained previously for H-LDPC code under the first decoding algorithm.
I. INTRODUCTION
The error correcting capabilities of Gallager’s low-density parity-check (LDPC) [1] for the binary-symmetric channel (BSC) were studied in [2], where it was shown that such Gallager’s LDPC code exists that capable of correcting the linear portion of errors, with decoding complexity O(n log n), where n – LDPC code length. Then in [3] new analytical easier to compute lower-bound on the error correcting capabilities of the Gallager’s LDPC code decoder was obtained, but the numerical results in most cases don’t exceed the numerical results, obtained by the old lower-bound from [2]. It should be noted, that the decoding algorithm, considered in [3], differs from the algorithm, described in [2]. In this paper we consider the algorithm similar to the algorithm from paper [2].
LDPC code with the constituent Hamming code (H-LDPC code) was considered in [4]. Then the code distance and the “soft” decoding of H-LDPC code were investigated in [5], [6]. It was shown in [7], that ensemble of H-LDPC codes contain such codes, that the minimal code distance almost reaches Varshamov – Gilbert bound. Then the result similar to the result from [2] for the first time was obtained for H-LDPC code in [8] by generalization of the methods developed in [2]. After that, using generalized methods from [8] similar results were obtained for q-ary LDPC code while transmitting over q-ary symmetric channel in [9] and for generalized LDPC codes while transmitting over the binary erasure channel in [10]. The authors of work [11] obtained a novel lower-bound on fraction of guaranteed corrected errors for the binary codes on graph while transmitting over BSC. The numerical results of

the lower-bound [11] for H-LDPC codes exceed similar results of the lower bound from [8]. Then in [12], [13] a new lower bound for generalized regular LDPC codes was obtained. It was shown that numerical results of the lower bound [12], [13] for H-LDPC codes exceed similar results of the previously known best lower-bounds on the error correcting capabilities of H-LDPC codes [11].
This work was inspired partly by [14] and partly by [13]. In this work we consider the H-LDPC code and two different iterative hard-decision decoding algorithms. Both algorithms are based on the same main idea: the decreasing of the syndrome weight on each step of the decoding algorithm. The first decoding algorithm uses the properties of the constituent Hamming code. The lower-bound on the guaranteed corrected errors fraction for the H-LDPC codes under this decoding algorithm was obtained in [12], [13]. The second decoding algorithm considers H-LDPC as the irregular LDPC code and uses well-known majority decoding algorithm. In this paper for the first time we obtained the lower-bound on the guaranteed corrected errors fraction for H-LDPC code under the second decoding algorithm. Numerical results, represented at the end of the paper, for the lower-bound, obtained in this paper for HLDPC code under the second decoding algorithm, significantly exceed the numerical results for the best known lower-bounds, obtained previously in [12], [13] for H-LDPC code under the first decoding algorithm.

B. Irregular LDPC code ensemble
Sometimes it is more convenient to specify LDPC codes using their Tanner graph representation [15]. The Tanner graph is a bipartite graph, where the nodes on the left side are associated with the code-word bits (variable nodes) and the nodes on the right are associated with the constituent code parity-check equations (check nodes). If check nodes are associated with the constituent Hamming code, then H-LDPC code, constructed above, is represented with regular Tanner graph with the left node degree and the right node degree n0. On the other hand if check nodes are associated with the single parity check (SPC) code, then H-LDPC code is represented with some irregular Tanner graph.

III. DECODING ALGORITHM
Let us consider two iterative hard-decision low-complexity decoding algorithms of the constructed H-LDPC code. The main idea of both decoding algorithms is the same: the number of constituent codes with unsatisfied checks decreases on each step of the decoding algorithm. The result of the decoding algorithm is the “corrected” sequence and the flag, indicating successful decoding or decoding failure.

A. Decoding algorithm A
Let us consider the first decoding algorithm A, proposed in [12], [13]. This algorithm uses the properties of the constituent Hamming code. One knows that if Hamming code paritychecks are not satisfied, then only one symbol, which inversion satisfies this Hamming code parity-checks, exists. So, the inversion of other n0−1 symbols doesn’t satisfy this Hamming code parity-checks. Then according to the construction of H-LDPC code each variable-node is connected to checknodes corresponding to the constituent Hamming code. Some of these check-nodes correspond to the constituent Hamming code which unsatisfied parity-checks become satisfied or remain unsatisfied after inversion of the considered symbol. Another check-nodes corresponds to the constituent Hamming code which satisfied parity-checks become unsatisfied after inversion of the considered symbol. So, the symbol is inverted if the number of constituent Hamming code which unsatisfied parity-checks become satisfied, exceed the number of constituent Hamming code which satisfied parity-checks

We will call the described above criterion of the symbol inversion as the “inversion” criterion.
The decoding iteration i, i = 1, 2, ..., imax, of iterative hard-decision decoding algorithm A consists of the following 2 steps:
1) invert the symbols of the tentative sequence r(i), where r(1) is the received sequence r, one by one, if the inversion of these symbols reduces the number of constituent codes with unsatisfied check (i.e. “inversion” criterion is fulfilled);
2) consider the updated sequence r(i) obtained on the previous step: • if the syndrome of LDPC code for the updated sequence becomes zero (i.e. there is no constituent code with unsatisfied checks), the algorithm returns the updated (corrected) sequence r(i), sets successful decoding flag and stops execution; • otherwise if the number of constituent codes with unsatisfied parity-checks for the updated sequence decreases, the algorithm proceeds to the next iteration i+1 with sequence r(i+1) that is equal to the updated sequence r(i); • otherwise the algorithm sets the decoding failure flag and stops execution.
For H-LDPC code under the decoding algorithm A the lower-bound on fraction of the guaranteed corrected errors was obtained in [12], [13]. Let α0ωγ denote the fraction of the guaranteed corrected errors obtained in [13].
B. Decoding algorithm B
Let us consider the second decoding algorithm B. On each iteration we invert one symbol after another that corresponds to a variable node with the left degree l that is connected with more than l/2 check nodes corresponding to the unsatisfied parity-checks (SPC). It is clear that inversion of such symbols reduces the syndrome weight, because more than l/2 constituent (SPC) code becomes satisfied.
The decoding iteration i, i = 1, 2, ..., imax, of the iterative hard-decision decoding algorithm B, consists of the following 2 steps:
1) consider the symbols of the tentative sequence r(i), where r(1) is the received sequence r, one after another: • determine a number of unsatisfied paritychecks corresponding to check nodes connected to a variable node corresponding to the considering symbol; • invert a current symbol if variable node with the degree l, corresponding to this symbol, is connected with more than l/2 check nodes corresponding to unsatisfied parity-checks; • proceed to the next symbol of the current sequence;

2) consider the updated sequence r(i) obtained on the previous step:
• if the syndrome of LDPC code for the updated sequence becomes zero (i.e. there is no constituent code with unsatisfied checks), the algorithm returns the updated (corrected) sequence r(i), sets a successful decoding flag and stops execution;
• otherwise if the syndrome weight for the updated sequence decreases, the algorithm proceeds to the next iteration i + 1 with the sequence r(i+1) that is equal to the updated sequence r(i);
• otherwise the algorithm sets a decoding failure flag and stops execution.
Remark 2: It is important to note that according to the description of algorithms A and B new errors can be added to the tentative sequence while decoding. So, algorithms A and B don’t guarantee that only erroneous symbols are corrected, new one may be added.
