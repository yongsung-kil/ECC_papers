Abstract_-Multi-rate  LDPC codes with near Shannon limit
performance are highly demanded in digital television terrestrial
broadcasting (DTTB) system. Several proposals have been carried
out to construct multi-rate LDPC codes,  among which, Row-
combining and Row-splitting with Edge Variation (RCSEV) helps
construct LDPC codes with constant code length and simple
hardware architecture. Besides, Column Extension (CE) is also an
effective method   for  rate-compatible LDPC code design: This
paper proposes an improved   scheme of Row-Combining and
splitting with Edge Variation and Column Deletion (RCSEV-CD);
which is the rational combination of RCSEV and CE. RCSEV is
carried out to generate multi-rate LDPC codes sharing the same
code length in basic code group. The codes in extended code group
are generated via CD, ensuring same code length in each extended
code groups Multi-rate codes within a code group improves the
flexibility of a service while the code groups meet the requirement
of multiple services. EXIT chart analysis helps guide the trade-off
between  column   degree   distributions  among  multiple rates A
comparison with multi-rate codes in DVB-S2 is performed via bit
error rate (BER) simulation and the result shows the superior
performance of our designed codes at typical code rates
Keywords_-low-density parity-check (LDPC); quasi-cyclic
(QC)-LDPC, Multi-rate, extrinsic information transfer (EXIT)
I. INTRODUCTION
The low-density parity-check (LDPC) codes [1] in digital
television terrestrial broadcasting (DTTB) systems should have
the following merits: multi-rate support, excellent performance
at each code  rate and low   implementation  complexity   for
multiple rates. Thus, LDPC code, which could achieve near
Shannon capacity performance, is gaining increased attentions
and extended applications these days Moreover, we hope the
LDPC coded modulation schemes employed in DTTB systems
have excellent performance at all supported rates and also enjoy
the benefit of simple structure for encoder/decoder. In
conventional practice, the parity-check matrix of each code rate
was usually designed independently to guarantee the
performance, causing the  significant differences of  degree
distributions of parity-check matrix   for multiple code rates,
which give rise to the difficulty and complexity of hardware
implementation of  multi-rate   code, as demonstrated in the
example of LDPC codes in DVB-X2 (DVB-S2 , DVB-T2 and
DVB-C2) [2]. Although the problem can be somewhat solved
Peng
Key Laboratory of Digital TV System)
by the universal decoding architecture like TDMP [3], which
could be applied in  general to decode any array-structured
LDPC code, the resource efficiency and the decoding
throughput will suffer accordingly. Hence it is better to meet the
requirements of modulation and coding scheme at the stage of
LDPC  code construction and the  solution to provide better
trade-off is still demanded.
As to the construction of LDPC code, a popular family of
LDPC codes QC-LDPC codes, have block  structures that
consists of square sub-matrices which have the special
circulant-matrix structure. The   parity-check matrix of QC-
LDPC code could be represented by the Template matrix, which
helps to simplify the design process of multi-rates and decrease
the complexity of implementation at the same time.
There were several proposals to construct new QC-LDPC
code at target rate from a given mother  LDPC code, while
maintaining the structure of the original parity-check matrix
As shown in [4], the scheme of Row-Combining with Edge
Variation (RCEV) is to combine two or more rows in Template
matrix of mother QC-LDPC code; which   corresponds to
changing the number of check nodes in Tanner graph: The
LDPC codes generated via RCEV are tightly coupled because
the result is unique once the rows of combination is determined.
Edge Variation (EV) helps tune the column degree distributions
in parity-check matrix of new QC-LDPC code by changing the
edges. Row-splitting (RS) in [5] is the counterpart of RCEV,
which generate low-rate LDPC code from high-rate   mother
codes It is apparent that the flexibility of RS can somewhat
compensate the disadvantages of RCEV, which constitutes the
scheme of another proposal Row-Combining-Splitting with
Edge Variation (RCSEV) [6].
However, when using RCSEV, the choice of code rate is
usually limited and  should be carefully selected in order to
guarantee the near-optimal column degree distribution for all
code rates and the low complexity in hardware implementation
of multi-rate decoder simultaneously. To satisfy the demand of
miscellaneous services, the choices should be increased, which
places a burden on the RCSEV method:
Another rate-compatible method called Column Extension
(CE) [7], whose main   operation is to add   columns to the
Template matrix of mother QC-LDPC codes to change the code
rate_ The process of CE follows the PEG algorithms [8] in each
column-extension step. The number of rows in Template

Column Deletion  (CD) method, which is   nothing but the

In this paper, we propose a QC-LDPC construction method
based on the combination of the above two techniques; named
RCSEV-CD, where, as an example, the RCSEV method is used
to build the basic code group of rate {1/2, 2/3, 5/6} and the
column deletion is used to build the extended code group ofrate
{2/5,3/5,4/5} and rate {1/4, 1/2,3/4} . Three code rates in each
group help to improve the flexibility for one service and three
code groups help meet the requirement of multiple services. As
the example in our DTTB system, LDPC codes in basic code
group is oriented to fixed-receptions with HDTV services [12].
The extended code group of {2/5,3/5,4/5} targets at portable-
receptions with SDTV services [13], while the extended code
group   of {1/4, 1/2, 3/4} targets at   robust   receptions   with
mobile/handheld services.
The construction of multi-rate high performance QC-LDPC
codes with low implementation complexity will be provided
and the related techniques will also be detailed in the following
sections. EXIT chart   analysis and bit error rate (BER)
simulation demonstrate the excellent   performance of the
constructed codes
The rest of this paper is organized as follows. Section II will
review the related techniques, including QC-LDPC code, Row-
combining and Row-splitting with Edge Variation, and Column
extension: In section III, the new proposal of RCSEV-CD will
be detailed followed by a design example. The EXIT analysis
and simulation results revealing the superior performance are
presented in Section IV. The conclusion of the whole paper is
drawn in section V.
II. TECHNIQUE REVIEW
A_ QC-LDPC codes
QC-LDPC code is a family of LDPC code which has an
encoding advantage over other   types of LDPC codes The
parity-check matrices of QC-LDPC codes have block structures
that each sub-matrix is a zero matrix or a circulant matrix of size
b, which could be described as the sub-matrix A i,j in (1). QC-
LDPC code with code length N and information bit length K,
the parity-check matrix H have cxt sub-matrices with entities
of A i,j, where c = (N-K)/b, t= N/b. We define Template matrix
T, which replaces the parity-check matrix H to simplify the
designing process, in which 1 represent a non-zero circulant
sub-matrix while 0 represent a zero sub-matrix.
First; it is shown in [9], for QC-LDPC codes, the complexity
of serial encoding is linearly proportional to the number of
parity bits and linearly  proportional  to the code   length in
parallel encoding: Besides that; the operating individuals of the
above techniques which help to generate multi-rate codes could
be the Template matrix instead of the high-dimension H matrix

B Design of RCSEV codes
The disadvantage of Row-combining lies in the limitation
of searching space as the resulting rows are determined when
the rows of mother code are given. In contrast, Row-splitting is
more flexible because there are more choices to split a row_
Thus   the   proposal of Row-Combining-Splitting with Edge
Variation could somewhat alleviate the constraints between the
target code and the mother code.
A LDPC code of length N and information bits K could be
specified as He-K N (2) and there are N variable nodes and (N-
K) check nodes in Tanner Graph. If there is an edge between a
check node and a variable node, the corresponding value in the
parity-check matrix is 1, and 0 for opposite, as shown in Fig.2.
It is apparent that if a variable node connect to different check
nodes at the same time, those   check nodes could not be
combined accordingly. Moreover, a check node could be split
into several nodes, as shown in Fig.3, if check node i could be
combined with check node k, then the sub-nodes i1, i2 could
also be combined with k, which could be defined as Row-
combinable, extending the original definition.

In addition to Row-combining and Row-splitting, Edge

could be   proceeded before or after the Row-combing and

degree distribution:
For QC-LDPC code, the object of RCSEV is the Template
matrix rather than the parity-check matrix, which indicates that
we could combine the rows in which the elements are sub-
matrices rather than a single number The low-dimension
Template matrix helps decrease the size of the searching space
and facilitate the designing process.
The deficiency of RCSEV is that the choices of code rate is
limited and should be carefully selected to guarantee both the
near-optimal column degree distribution for all code rates and
the low complexity in hardware implementation of multi-rate
decoder.
C Column Deletion
Column Deletion method, which is nothing but the
counterpart of Column Extension, following a nested matrix
structure. The number of rows in parity-check matrix of mother
code is constant, indicating the constant number of information
bits. In Column Deletion, some columns will be progressively
deleted in each deleting step, as shown in Fig.4.
Fig: 4_ Column Deletion_
The choice of deletion is guided by EXIT chart analysis and
the number of graph cycles. For a specific target code rate, the
number of columns to be deleted is determined by the target rate,
and the type of columns to be deleted is determined with the
help of EXIT chart analysis. Besides that; minimizing number
of graph cycles help to choose columns from the column group
of same degree.
The  multiple code-lengths in this proposal is helpful to
support   multiple services. Moreover; the    nested structures
between the parity-check matrices of mother code and the target
code facilitates the implementation of multi-rate decoder. The
object of CD for QC-LDPC code is the Template matrix, which

III. PROPOSED MULTI-RATE CODE DESIGN
A. The algorithm of RCSEV-CD
To meet the demand of channel coding in DTTB system, we
propose a new scheme based on the combination of those two
techniques above, named Row-Combining-Splitting with Edge
Variation and Column Deletion (RCSEV-CD): This proposal
takes advantage of the techniques above to obtain QC-LDPC
code ofhigh performance with low implementation complexity.

could be described as Hb = {Hi, Hz Hn} and the rates of
the codes Rb = {r1,T2 rn}: The code group extended from
the basic group could be defined as Hb = {Hi,Hz Hn} and
corresponding code rates are Rb = {ri,rz Tn}:

CONCLUSION
This paper propose a new method to construct multi-rate
QC-LDPC codes with low implementation complexity.
Inspired by   Row-Combining-Splitting and Edge Variation
(RCSEV) and Column-Extension (CE), we propose the
improved scheme named Row-Combining-Splitting and Edge
Variation & Column Deletion (RCSEV-CD); which exploits the
advantages of both techniques. RCSEV helps generate multi-
rate codes sharing a same code length which is beneficial to the
flexibility of a service. Apart from that, CD is employed to
provide more code-rate choices. Our designing   proposal
guarantees the low complexity implementation: Besides that;
the codes we design have the merits of multi-rate support and
excellent  performance at   each code rate, which  satisfy the
demand of channel coding in DTTB system: As shown in BER
simulations, our proposed multi-rate QC-LDPC codes
outperform DVB-S2 in most cases.
