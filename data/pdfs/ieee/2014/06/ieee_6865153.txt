Efficient Column-Layered Decoders for Single Block-Row Quasi-Cyclic LDPC Codes

Chuan Zhang, Xiaohu You
National Mobile Communications Research Laboratory School of Information Science and Engineering Southeast University, Nanjing, China {chzhang, xhyu}@seu.edu.cn
Abstract—The recently proposed single block-row quasicyclic low-density parity-check (QC-LDPC) codes are favorable for high-speed applications. However, conventional decoder design methods are not suitable for this kind of codes. To tackle this issue, this paper aims at designing efficient column-layered single block-row QC-LDPC decoder architecture without affecting the decoding performance. Moreover, the simplified version which only requires single minimum value is also proposed for further hardware reduction. Results show that, for the rate-0.9006 (1640, 1477) single block-row QC-LDPC code, the proposed two designs achieves significant advantages in both hardware and latency over their row-layered counterpart.
Keywords—Low-density parity-check (LDPC) codes, single block-row, column-layered, single-minimum
I. INTRODUCTION
Quasi-cyclic LDPC (QC-LDPC) codes have been receiving much attention due to their implementation-friendly structure and similar decoding performance compared to computergenerated random LDPC codes. Nowadays, the ever-increasing user demand forces the modern communication systems to become faster. Therefore, several high-rate QC-LDPC codes, which are suitable for high-speed applications, have been proposed in [1, 2]. However, their construction restrictions hinder those codes from being widely adopted. To this end, new single block-row regular QC-LDPC codes based on cyclic difference families (CDFs) or perfect difference families (PDFs) are proposed in [3]. They show great construction flexibility and error correcting capability, thus serve as a promising error correction code candidate for high-speed applications.
Though this new kind of QC-LDPC codes appear favorable in real-life applications, their efficient decoder design remains an open problem. This paper commits itself to the efficient decoder design for single block-row regular QC-LDPC codes. First, a new column-layered decoding algorithm, which proves well-suited for hardware implementation, is proposed. Then, its simplified version which only employs single minimum value is introduced for further hardware reduction. Comparison results for the rate-0.9006 (1640, 1477) code have shown that the proposed single-minimum decoder only requires only 10% VNUs and 6.25% decoding latency over the conventional rowlayered one, while achieving the same decoding performance.
The remainder of this paper is organized as follows. A brief review of the single block-row QC-LDPC codes is provided in Section II. In Section III, the new column-layered decoding algorithm is proposed. Its decoder architecture is given in Section IV. Section V introduces the single-minimum columnlayered decoder architecture. One possible solution is to employ column-layered minsum decoding algorithm instead. Similar to row-layered decoding, the column-layered one can also significantly reduce the maximum number of decoding iterations while keeping the same decoding performance. The major advantage is, in column-layered decoding, the main complexity corresponds to VNUs. Since VNU operation is mainly composed of simple accumulations, the column-layered approach is more favorable for efficient decoder design. Based on this consideration, a reduced-complexity column-layered decoding approach is recently proposed for efficient decoder design of QC-LDPC codes [6]. However, for the efficient decoder design of single block-row QC-LDPC codes, more specific optimizations are required. In what follows, a new column-layered min-sum decoding which is appropriate for hardware implementation of single block-row QC-LDPC codes is proposed.

B. Column Layer Partition Scheme
For the proposed algorithm, there are two column layer partition choices. For the first option, each column layer is composed of several consecutive columns, usually the columns associated with one circulant sub-matrix. However, the shifting set {sij :1 ≤ j ≤ dv} for each Hi varies when the value of i changes. Therefore, for this choice, different layers are not related with each other. It means that the interconnection

between CNUs and VNUs varies for different layers. This will increase the complexity and requires complicated barrel shifter and selection logic in VLSI implementations. In order to lower the connection complexity, we have the second choice. Suppose that the N-bit code-word (n1, n2, …, nN) is divided into g layers of the same size: N1, N2, …, Ng. The proposed column layer partition scheme is given as follows:

Ni = {nj : j = i : z : ((N − z) + i)},i = 1, 2,…, g(= z). (2)

For the second partition, layer Ni of H is exactly the j-step cyclic shifted version of layer N(i-j) mod z. This property indicates that the shuffle network between CNUs and VNUs now can be reused by different layers. As a result, the decoder complexity can be reduced. However, the row-weight for each layer is no longer unique now, which will introduce concatenation of multiple comparison for CNUs.

VI. HARDWARE COMPARISON
Benefitting from the column-layered decoding schedule, the decoder is of high efficiency. TABLE I lists the hardware complexity of different designs for the (1640, 1477) code. The proposed column-layered decoder and single-minimum one are denoted by this work 1 and 2, respectively. 5-bit quantization is assumed. Compared to the conventional row-layered design, the proposed two column-layered designs show much higher efficiency while achieving comparable decoding performance. For example, both designs only require 10% VNUs compared to the row-layered one. Although the number of required CNUs is higher, the complexity of each CNU is much lower. It should be noted that for both proposed designs, CNU here includes the CNU and vector gen blocks. Finally, the decoding latency of our designs is only 6.25% of that of the row-layered one.

VII. CONCLUSION
In this paper, we have proposed an efficient decoder architecture and its single-minimum variant for the single block-row QC-LDPC codes. By employing the column-layered scheme, which is inherently compatible with this specific kind of codes, we successfully achieves advantages in both complexity and latency without compromising the decoding performance. These promising results validate the practical use of both proposed decoder designs in real high speed communication environments.
