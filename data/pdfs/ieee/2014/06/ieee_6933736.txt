A General Folded Encoding Structure for Quasi-cyclic LDPC Codes

Chen Qi, Guo Xuan, Yang Zhanxin
Digital Engineering Center Communication University of China
Beijing, China daoting7@cuc.edu.cn

Abstract—A general folded encoding structure for multi-rate quasi-cyclic LDPC codes is proposed in this paper to implement LDPC encoder with the advantages of few resources consumption, adjustable parallelism and high coding rate. For the quasi-cyclic LDPC codes mentioned in this paper, folded encoding structure can provide high coding rate and save 66% of the resources compared with the traditional encoding structure.
Keywords-LDPC codes; SRAA; Foldable;
I. INTRODUCTION
LDPC (Low Density Parity Check) codes [1], which were first invented by Robert G.Gallager in 1962, have a good performance of approaching the Shannon limit and have characteristics of low decoding complexity and flexible structure. Since the quasi-cyclic check matrixes of QCLDPC(quasi-cyclic LDPC) codes can reduce the resources consumption and the complexity of encoding in practice, QCLDPC codes become a hot topic of the research in the field of channel coding [2,3]. The typical QC-LDPC encoder [4], which is based on traditional SRAA circuit structure, uses a lot of SRAA circuits and consumes large amounts of resources, especially for multi-rate QC-LDPC coding [5]. In order to solve the problem, we propose a general folded encoding structure, which could select appropriate number of SRAA circuits to fit the configuration of multi-rate QC-LDPC codes, to improve the performance of the LDPC encoder.
The rest of this paper is organized as follows: Section 2 introduces the principle of folded encoding structure. Section 3 introduces the encoding circuit of folded encoding structure. Section 4 gives a analysis between folded encoding structure and tradition encoding structure. Section 5 concludes the paper.

Control module controls the information sequence RAM, generator matrix ROM, SRAAi (i = 0,1,…,K-1)circuit and output module to work methodically to implement QC-LDPC encoding. The process can be described as follows:
1) Select the appropriate SRAA circuit number n and calculating time s based on the input encoding rate.
2) In the first time of calculating, the control module let the information sequence RAM and generator matrix ROM load the correct value to correspond SRAA circuits and enable the SRAA circuits to start to encode. When the calculating is done, output module will output the encoding results using shift register under the control of the control module, After shifting all encoding results, we will get n*q bits of the check bits. Repeat the operation s times, we will get all the bits of check code in a QC-LDPC code.
3) Output the information bits in the information Sequence RAM.
After all the steps above, we will get the QC-LDPC code.
Less SRAA circuits will consume less logic resources but need more encoding times. Accordingly, we should select appropriate SRAA circuit number and calculating time to improve the performance of the encoding. For QC-LDPC codes with 1 rate, we only need to select the SRAA circuit number and calculating time for the rate. While for multi-rate QC-LDPC codes, the appropriate SRAA circuit number n is different in different rates, calculating time is also different. So we need to combine the configurations of all rates to select the appropriate SRAA circuit number and calculating time in multi-rate QC-LDPC code encoding. The utilization rate of SRAA circuits for all rates also should be considered in the implementation of encoding to improve the performance of the encoder.

V. SUMMARY In this paper, a general folded encoding structure for multirate QC-LDPC codes is proposed to implement the encoder with the advantages of fewer resources consumption, adjustable parallelism and high coding rate. From the example of application of QC-LDPC encoding mentioned in this paper, an analysis and comparison between general folded encoding structure and the tradition encoding structure has been given. We will find the low resources consumption and the high coding rate in the process of implementation.
