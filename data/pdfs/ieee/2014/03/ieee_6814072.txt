Output Decisions for Stochastic LDPC Decoders
Kuo-Lun Huang', Vincent Gaudet" and Masoud Salehi'
tOepartment of Electrical and Computer Engineering, Northeastern University, Boston, Massachusetts tOepartment of Electrical and Computer Engineering, University of Waterloo, Waterloo, Ontario huang.ku@husky.neu.edu, vcgaudet@uwaterloo.ca, salehi@ece.neu.edu

Abstract-Stochastic decoding is a method to iteratively decode error-correcting codes such as low-density parity-check codes. Due to the computational simplicity of its decoding algorithm, stochastic decoding provides not only low silicon area but also competitive performance. In this paper, we research the different techniques used to determine the code bits generated by stochastic decoders during the final decision process. We investigate an efficient method where code bits are directly decided based on the final stochastic bits generated at output nodes. This observation contributes to saving chip area and providing high throughput for stochastic decoders.
Index Terms-Stochastic decoding, Low-Density Parity-Check (LDPC) codes, stochastic computation, quantization in statistics.

I.

INTRODUCTION

Harnessing computational complexity is a main priority in the design of signal processing algorithms. High computational complexity increases the delay, reduces battery life, and in most cases increases the physical dimensions of the devices. Hardware implementation of algorithms is constrained by application requirements, chip area, or power consumption limitations. Thus, efficient implementation of algorithms has become a crucial concern in the design of computing systems. Stochastic computation is an approach that has been considered for designing several types of low-precision digital circuits, and leads to good hardware and energy efficiency. Stochastic computation was introduced in the 1960s for logic circuit design [1], but its origin can be traced back to Von Neumann's work on probabilistic logics [2]. Its main advantages are that complex computations can be approximated by using simple circuits while providing comparable performance.

Over the years, stochastic computation has been recognized as a useful technique for smaller and reliable implementations. Its benefit in hardware efficiency has been shown in several applications, including neural networks [3][4], control circuits [5][6], approximate computing [7], and energy-efficient circuit designs [8][9].

Stochastic computation has also been considered for the implementation of iterative decoders for capacity-approaching error-correcting codes such as turbo code [15] and Low­ Density Parity-Check (LDPC) codes [10]-[14].

LDPC codes, a class of linear block codes, are typically decoded by a method based on iterative belief propagation,

such as the Sum-Product Algorithm (SPA), which iteratively
passes a posteriori probability messages on a graph that
represents the code structure. The complexity of the interconnection network in the graph can be significant and may lead to large chip area as the codeword length increases. However, the codeword lengths of practical LOPC codes are generally very large, usually thousands of bits, to provide better performance. In [10], Gaudet and Rapley demonstrated that stochastic computation can be used to efficiently implement iterative decoding of a simple parity-check code over factor graphs. Since all the messages are represented by streams of randomly generated elements and all the computational operations are processed in serial fashion, stochastic decoding greatly alleviates the routing congestion problem in conventional iterative decoders and can provide better logic utilization. Stochastic decoding has been utilized in a (16,8) LOPC code [10], a (1024,512) LOPC code [11], the (1056, 528) LDPC code in WiMAX standard [12], the (2048,1723) LOPC code in IOGBASE-T standard [13], and a growing list of other error-correcting codes [14][15].
Unlike the LDPC decoder introduced in [16], which has a fully parallel multi-bit architecture to transmit messages and requires a large number of wires, the inherently serial messages in stochastic decoding only require a single wire. Another similar technique to reduce the number of physical wires in iterative decoders is to perform decoding in a block-parallel bit-serial fashion. Only one bit is sent over a single wire at a time, and it accumulates the bits for later arithmetic operations. This method greatly reduces routing complexity, but it consumes more chip area for the memories to store the multi­ bit messages. For stochastic decoding, the statistics of the stochastic stream are more crucial than the actual value represented by the entire stream. Besides, bits in stochastic streams are independent of each other, and no memory is needed to store message bits. In many ways, stochastic decoding is simpler and more efficient.
Despite its efficiency both in hardware implementation and energy, stochastic decoding is prone to certain types of errors perhaps because of the message representation. At the end of the stochastic decoding process, the bit-wise output streams need a decision mechanism to determine the final code bits. N­ modular redundancy is a commonly employed fault correcting
technique where the computation is replicated N times and the final output is based on the majority vote of those N results. For

II. STOCHASTIC DECODING
The messages used in the SPA are soft values representing the probabilities or the likelihood ratios. In stochastic decoders, the messages are represented by streams of random symbols. For the binary case, each bit of the stream is equal to 1 with the value of the information to be encoded. The transmitted probability in iterative decoding is represented by the appearance of Is in the stream. The circuits that convert probabilities to stochastic streams are called stochastic number generators. The conventional stochastic number generators include a random number generator and a comparator. Fig. I(a) shows a conversion circuit from a probabilityp to a length-l stochastic stream. A random number R is generated and varies each clock cycle. The conversion process involves comparing R to the probabilityp, which is fixed for l clock cycles. The comparator outputs a 1 if the input probabilityp is greater than the random number R and a 0 otherwise. Assuming that the random number R is uniformly distributed over the interval
[0,1], the probability of a 1 appearing at the output of the
comparator at each clock cycle is equal to the probabilityp. For instance, a bit stream containing 25 percent of Is and 75
percent of Os indicates a probabilityp = 0.25, which means
that the probability of observing a 1 at an arbitrary position in the stream is p . Since each bit in the stream is generated independently, the position of Is is not fixed and different stochastic streams are possible to represent a given probability.
Conversion from a stochastic stream back to a probability value is much simpler, because the number of Is in the stream carries the value ofp. Fig. I(b) illustrates the conversion with an adder to count the number of Is in the length-l stream and then divided by l to extractp. Each clock cycle in stochastic decoding is called a Decoding Cycle (DC) [12]. At each DC, there is only one bit being computed for and transmitted between nodes, which are variable nodes and check nodes in the factor graph of the code.
The decoding process of stochastic decoders operates over DCs as follows.
Step 1. Initialization. Each variable node receives one bit from
the stochastic stream, which corresponds to the channel receiving probability.
Step 2. Variable to Check Node. At each decoding cycle,
variable node Vi propagates its outgoing I-bit message to the connected check node Cj based on the stochastic bit from the channel and the extrinsic bits received from all the other edges.
Step 3. Check to Variable Node. Check nodes check the
parities and send their I-bit messages back to their neighbors at each decoding cycle. The message sent from check node Cj to the variable node Vi on the edge e is the XOR of all messages received at node Cj on all edges except edge e.
Step 4. Codeword Test. Based on all the extrinsic bits from the
check nodes and the stochastic bit received from the channel, the output of each variable node at the end of a decoding cycle is passed to a decision mechanism to determine the code bit.
After completing this exchange of bits between variable nodes and check nodes, variable nodes load in the next bit from the stochastic streams and start the next decoding cycle. The process is terminated when the maximum number of decoding cycles is exceeded or all the check nodes are satisfied.

Ill. REDUNDANCY MECHANISMS
At the end of each DC, each variable node generates a bit to indicate the output probability. Since it is unnecessary to accumulate all the bits to calculate the probability Pop,t in stochastic decoders, a decision mechanism is needed to determine the final code bit based on the last few stochastic bits outputted from each variable node. A majority mechanism is
one of the designs proposed by Winstead et al. in [17] to make
the decision on code bits. This redundancy structure makes a majority vote upon the last few bits in the output stochastic streams. For example, when more than two of the last five bits are 1s, the code bit is decided as 1 and 0 otherwise. This method helps to improve the performance of decoder by increasing ability to against the transient errors. However, this error suppression technique consumes chip area for memories, and its time redundancy design delays the outputs.
In [12], Tehrani et al. employed an up/down counter in a
fully parallel stochastic LDPC decoder to determine the code bits. A 4-bit saturating up/down counter that counts from -7 to 7 is used. The counter is increased by one when the stochastic bit outputted from the variable node is I and is decreased by one when a 0 is received. Then, the code bit of each counter is determined by the sign bit of the counter. The decoded bit is 1 when counter is positive and 0 otherwise. We extend these results to show that the latest output bit from the i-th variable node can be directly determined as the i -th code bit in the codeword.
Since each stochastic bit is equal to 1 with its encoded probability and the actual value represented by the entire stochastic stream is not really critical in stochastic decoding, the redundancy mechanism can be neglected while the stochastic decoding process is convergent. Removing the decision mechanism and determining code bits based on the latest outputted stochastic bit at each variable node provide us a more efficient design of stochastic decoders.
