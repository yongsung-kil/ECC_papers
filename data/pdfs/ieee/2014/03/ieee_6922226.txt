Design and Implementation of Multi-Rate LDPC Decoder for IEEE 802.16e Wireless Standard
Vijaya Kumar K, Rahul Shrestha, Student Member, IEEE and Roy Paily, Member, IEEE Department of Electronics & Electrical Engineering, Indian Institute of Technology Guwahati Email: {k.vijaya, r.shrestha, roypaily}@iitg.ernet.in

Abstract—In this paper, a flexible architecture of multi-rate Low Density Parity Check (LDPC) decoder has been presented. It supports six different code-rates which are specified by IEEE 802.16e wireless standard. In the suggested decoder-architecture, column layered decoding technique has been employed to increase the convergence speed. Additionally, the decoder-design incorporates parallel architecture to achieve higher throughput which meets the requirement of IEEE 802.16e standard. An Application Specific Integrated Circuits (ASIC) implementation of this decoder-architecture has been performed at 130 nm Complementary Metal Oxide Semiconductor (CMOS) technology node. At the worst-case Process Voltage Temperature (PVT) corner with the supply voltage of 1.08 V, the implemented decoder has achieved a maximum information throughput of 159.6 Mbps at a clock frequency of 39.9 MHz.
Index Terms—LDPC codes, column layered decoding technique, multi-rate decoder and IEEE 802.16e standard.
I. INTRODUCTION
Superior performance of LDPC codes has made them acceptable by recent wireless communication standards such as IEEE 802.11n, IEEE 802.16e, ITUG.hn and DVB-S2. In the past few decades, various decoding algorithms have been developed in the field of coding theory for diverse applications. Some of the remarkable ones are probabilistic decoding [1], belief propagation algorithm [2], sum-product algorithm [3] and min-sum algorithm [4]. Basically, Sum Product Algorithm (SPA) and Min-Sum Algorithm (MSA) are the well accepted choices of decoding techniques because SPA delivers optimum decoding performance [5] whereas MSA reduces computation complexity of the decoding algorithm resulting in a simple decoder-architecture. Thereby, from the implementation perspective, sate-of-the-art designs of efficient decoders are based on MSA or the modified versions of MSA [6], [7]. Now a days, layered LDPC decoding schemes are widely considered as they can speed up the process of decoding and thereby reduces the required number of decoding-iterations [8], [9]. There are two variations of layered decoding: row-layered decoding [8] and column layered decoding [9], [10]. In [11], authors have proposed a min-sum based column layered decoding algorithm which uses a matrix permutation method to convert the generic QC-LDPC (Quasi Cyclic-LDPC) code into a shift-structured LDPC code to reduce the hardware complexity. Subsequently, a modified version of min-sum algorithm termed as three minsum algorithm has been used in their work. The contributions of our work are as follow.

∙ Two min-sum column layered decoding has been used to reduce the memory requirement and to improve the convergence rate of LDPC decoder.
∙ We have suggested flexible and simple parallel decoderarchitecture for LDPC codes. An ASIC implementation at 130 nm CMOS technology node is carried out and the results are compared with various reported works in literature.
∙ Bit Error Rate (BER) performance analysis has been carried out for different number of iterations using two min-sum column layered LDPC-decoding algorithm.
The remaining sections of this paper are organized as follows. In section II, the two-min sum column layered decoding used in decoder implementation is presented. Section III presents details of decoder implementation. Performance evaluation of the decoder is presented along with the results and comparison of the results in section IV. Finally the paper is concluded in section V.
II. DECODING ALGORITHM
Consider a binary QC-LDPC code which is specified by a parity check matrix H which has M rows and N columns. In the parity check matrix each column is associated with a Variable Node Unit (VNU) and each row is associated with a Check Node Unit (CNU). In a particular column, bit ‘1’ corresponds to an existence of connection between VNU corresponding to that column and CNU corresponding to the row that contains ‘1’. Let N (c) = {v : Hcv = 1} be the set of variable nodes that is associated with a check node c. Similarly, consider M (v) = {c : Hcv = 1} be the set of check nodes that is associated with a variable node v. Let messages from c-to-v and v-to-c be represented by Rc(kv) and Lc(kv), respectively, where k represents kth iteration, and the intrinsic message is denoted by Iv. G bits in the code word, where G ≪ N, are segregated into B groups of the same size G0, G1, ......GB−1, accordingly the parity check matrix is also portioned into B block columns. There are reported works in literature where authors have proposed a shuffled method to select G columns for each group [12]. However in our work, block columns defined in the parity check matrix are directly treated as a group. Specifically, we have used LDPC code which has a block length of 576 and each block column is a square matrix of size 24×24.

III. DECODER ARCHITECTURE
In this work, (96, 576) QC-LDPC code has been used and it contains 4 and 24 sub matrices in each column and row respectively. Each sub matrix is of a size 24×24 (expansion factor q = 24). In order to process each block column at a time, a total of 96 CNUs and 24 VNUs are required. In Fig. 2, a suggested architecture of LDPC decoder has been presented where each register file contains 24 registers such that each register is of 25 bit wide. Contents of this 25 bit registers are shown in Fig. 1, and thereby four such register files are used in the architecture of LDPC decoder, as shown in Fig. 2. Each register in such register files is associated with first and second stages of its corresponding CNU in CNU-A and CNU-B banks. Right shifters are used to shift the connections between CNUs and VNUs according to specified shift index for each sub matrix. Similarly, left shifters are used to shift the connections between VNUs and CNUs in order to compute the updated vectors mcn, inc and Scn and to store those in register file. In this decoder-architecture, we have used the parity check matrix without making any changes to it, few components of architecture like CNUs that can handle two Lcv values at a time are not needed, as it is guaranteed that each CNU is associated with only one VNU at a time.
The shift indices of associated parity check matrix are stored in a memory block, where the controller provides the required address. Generated Lcv values in VNU are used to update the mnc∼, inc∼ and Scn∼ vectors, so, there is no need for storing v-to-c messages. Intrinsic memory block is used to store the channel values corresponding to all VNUs and the controller provides the address to select the channel values corresponding to VNUs of the group that is currently processing. In each group the sign of the Lcv messages from all VNUs are stored in sign of Lcv memory, as shown in Fig. 2, so that they can be used to calculate Scn or Scn∼. A single flexible common controller has been used to provide address for all modules of the decoder, also the maximum number of iterations can be altered easily. Also for the shift indices with negative values all the CNUs and register file corresponding to that block will be made inactive and a multiplexer is used to choose zero at that instant. The implemented decoder can support the LDPC codes having their parity check matrices with expansion factor of 24, there are six such codes in IEEE 802.16e wireless communication standard and they are mentioned in Table I. It is also possible to decode the codes whose code lengths are greater than 576 with a small adjustment in controller and register files. In [13], authors have used 24 processing elements and reused the same processing elements to decode the codes that have expansion factor greater than 24, this kind of approach is energy efficient. However, it is not feasible to employ such idea from the perspective of high-throughput implementation.
