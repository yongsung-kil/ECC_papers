Efficient Progressive Edge-Growth Algorithm Based on Chinese Remainder Theorem
Xueqin Jiang, Xiang-Gen Xia, Fellow, IEEE, and Moon Ho Lee, Senior Member, IEEE

Abstract—Progressive edge-growth (PEG) algorithm construction builds a Tanner graph, or equivalently a parity-check matrix, for an LDPC code by establishing edges between the symbol nodes and the check nodes in an edge-by-edge manner and maximizing the girth in a greedy fashion. This approach is simple but the complexity of the PEG algorithm scale is O(nm), where n is the number of symbol nodes and m is the number of check nodes. We deal with this problem by construct a base matrix Hb of size mb × nb with the PEG algorithm and simultaneously expand this base matrix into a parity-check matrix H of size m × n via the the Chinese remainder theorem (CRT), where m mb and n nb. The size of the base matrix is expanded without decreasing the girth. For convenience, the PEG and CRT combined algorithm is referred to as the PEG-CRT algorithm in this paper. Since a smaller matrix is constructed with the PEG algorithm and the complexity of the CRT computation is negligible compared to the PEG algorithm, the complexity of the whole code construction process is reduced. Furthermore, the proposed algorithm has a potential advantage of saving storage space by storing a smaller matrix Hb and expanding it to H “on-the-fly” in hardware. The expanded matrix H preserves the important properties of base matrix such as large girth, flexible code rate and low density. The complexity analysis shows that the complexity of the PEG-CRT algorithm does not grow with the code length n. Simulation results show that compared with the PEG LDPC codes of length nb, the expanded PEGCRT LDPC codes have better bit error rate (BER) performance with the iterative decoding. It is also shown that compared with PEG LDPC codes of length n, which constructed with higher complexities, the PEG-CRT codes have similar BER performance.
Index Terms—LDPC codes, progressive edge-growth (PEG) algorithm, Chinese remainder theorem (CRT), girth.

Codes result from this method have large girths, flexible code lengths and flexible code rates. Among the existing methods for the construction of random-like LDPC codes, one of the most successful approaches is progressiveedge-growth (PEG) algorithm [5][6]. For an optimized degree distribution, usually generated by the density evolution algorithm [7], the PEG construction builds the Tanner graph, or equivalently a parity-check matrix, for an LDPC code by establishing edges between the symbol nodes and the check nodes in an edge-by-edge manner. It can maximize the local girth [5] in a greedy fashion.
LDPC codes belong to the class of linear block codes, whose performance improves as the code length n becomes very long [8][9]. Moreover, the optimized degree distributions obtained by the density evolution are designed for infinitelength LDPC codes. This implies that they may perform well when the code lengths of LDPC codes are long. However, the PEG algorithm is not efficient. In the case of long length codes, the construction of LDPC codes with the PEG algorithm requires long time consumption. In the worst case, the computational complexity scales as O(nm). Our goal in this paper is to reduce the complexity of the PEG algorithm for long length random-like LDPC codes. To reduce the complexity of the PEG algorithm, we use the PEG algorithm in conjunction with CRT that can expand the under-construction matrix Hb without reducing the girth. Notice that the idea to use CRT for the expansion in the random-like LDPC codes design was used in the short conference paper [10] but it was not fully or correctly developed. In this paper, this idea is fully and correctly developed. Hereafter, for convenience, the LDPC codes constructed by the PEG algorithm are referred to as PEG LDPC codes and the LDPC codes constructed by the PEG and CRT combined algorithm are referred to as PEG-CRT LDPC codes. The simulation results show that the PEG-CRT LDPC codes of length n perform better than the PEG LDPC codes of length nb, nb n, and perform similar to PEG LDPC codes of the same length with much less complexity.
The remainder of this paper is organized as follows. Section II reviews the PEG algorithm and the CRT. Section III proposes the PEG-CRT algorithm which combines the PEG algorithm and the CRT. Code design examples and simulation results are given in section IV. Some properties of the PEGCRT LDPC codes are given in Section V. Finally, Section VI concludes the paper.

V. PROPERTIES OF THE PEG-CRT LDPC CODES
A base matrix Hb constructed by the PEG algorithm guarantees good properties such as large girths, good minimum distance, flexible code rates, flexible code length and low densities. The parity-check matrix H constructed by the PEG-CRT algorithm preserves all these good properties. Furthermore, the PEG-CRT algorithm has lower computational complexity and practical complexity than those of the PEG algorithm.
A. Girth
It is well known that the iterative sum-product decoding algorithm converges to the optimal solution provided that the Tanner graph of the code is free of cycles. The shorter the cycles are in the graph, the sooner the algorithm will break down. Therefore, Tanner graphs or parity-check matrices with large girth are particularly desirable, and the girth is one of the important parameters determining the performance of the coding. Let us begin this section by analyzing the girth of the PEG-CRT LDPC codes.

VI. CONCLUSION
The PEG algorithm is simple but the computational complexity of the PEG algorithm scales as O(nm). We dealt with this problem by constructing a base parity-check matrix Hb with the PEG algorithm and expanding this base matrix to a larger parity-check matrix H via the CRT, simultaneously. Since the size of Tanner graph (or equivalently the paritycheck matrix) was reduced in the PEG construction, the computational complexity of the whole constructing process was reduced. The proposed algorithms achieved the similar code length as that of the PEG LDPC codes with negligible decrease in girth. Furthermore, practically the storage space of the decoder and the encoder was reduced by storing a smaller parity-check matrix Hb, instead of H, and the corresponding generator matrix. Simulation results showed that the PEG-CRT LDPC codes perform much better than the base PEG code and performed similarly to PEG LDPC codes of the same length. Note that compared to PEG LDPC codes, PEG-CRT LDPC codes of the similar length have much lower complexities.
