Abstract-A bit serial architecture is used which reduces the interconnect complexity in fully parallel low density parity check (LDPC) decoder. This can achieve better error correcting performance when the code length is moderate. By using a new approximation to the check update function in the Minsum decoding algorithm, the implementation is simplified. This new check update rule finds only the absolute minimum magnitude of the incoming message and if required correction is made to the outgoing messages. The proposed decoder is designed using Verilog HDL, simulated using MODELSIM 5.7g, synthesized by Xilinx 9.2i and implemented using Spartan 3E. The result shows that the proposed architecture requires fewer slices and LUT’s when compared with the existing methods.
Keywords- LDPC (Low density parity check) code, Minsum (MS) algorithm, Bit serial architecture, Check update function)
I.INTRODUCTION
Error correction coding is an essential component in digital communication and data storage systems to ensure robust transmission of information. Low Density Parity Check (LDPC) code is the most promising near optimal (i.e. near Shannon limit) error correction codes used in practice. These codes are discovered by Gallager in 1962.Since LDPC codes have superior coding performance and parallelizable decoder architecture, they have been recently used for several data communication applications. They allow a fine-level parallel message-passing decoding where all the check and variable nodes are updated concurrently. LDPC code has been widely used in many industrial standards such as WLAN (802.11n), WiMax (802.16e), DVB-S2, CMMB and 10GBaseT(802.3an) systems.
In this paper, a bit serial architecture is proposed for fully parallel LDPC decoders. This computation allows variable and check nodes to communicate multi-bit messages over single wires thereby reducing the interconnect complexity. In addition, a new approximation to the check update function in Minsum decoding is introduced. Here in each check node only one minimum magnitude is calculated over all the check node

inputs. A corrective constant is added to generate the proper check outputs, depending on the number of inputs that share the same minimum magnitude. The feasibility of bit serial decoding is illustrated by implementing a LDPC decoder on a Spartan3E FPGA based device.
The paper is organized as follows. Section I describes the basics of LDPC codes, Minsum algorithm and the LDPC decoder. Section II explains the bit serial architecture in detail. Section III elaborates the simplified check update function by its node architecture. Finally Section IV summarises the results which includes the implementation of bit serial architecture in FPGA.
A.LDPC Codes
LDPC codes form one class of shannon limit approaching codes .A LDPC code is a linear block code which is defined by a corresponding matrix called the parity check matrix H or the associated graph called Tanner graph. Tanner graph consists of two sets of nodes: check nodes {c1, c2, . . . , cM}that represent the rows of H and variable nodes {v1, v2, . . . . vN}that represent the columns. A connection is made between the nodes only if Hmn is nonzero. An LDPC code is said to be irregular if all the columns or all the rows of the parity check matrix H do not have the same weight.

B. Minsum decoding algorithm
The computational complexity of Sum Product algorithm (SPA) can be reduced by simplifying the computation of check node processing and variable node processing. Steps involved in Min-Sum Decoding Algorithm are as follows [4],
1) Initialize the iteration counter, I=1 and M=maximum number of iterations 2) Log Likelihood Ratio(LLR) Value received from channel is input to the decoders variable node. 3) Update the check nodes. Magnitude of the incoming message is calculated by XOR operation and sign is calculated by AND operation. 4) Update the variable nodes 5) Apply a hard decision If WHT = 0 or I  M ,stop decoding and go to step 6. Otherwise set I =I + 1 and go to step 3. 6) Output W(i) as the decoder output.
C. Decoder Implementation
Mostly, conventional decoders use multi-bit digital signals to represent the messages. In partially-parallel decoders [5],[6], the messages are transferred between the nodes through memory. This architecture reduces the decoder area by sharing the processing units but throughput is reduced. This can be increased by directly instantiating the variable and check nodes in hardware [7]. The major challenge in the implementation of fully parallel LDPC decoders is the complex and random interconnection between the variable and check nodes.

In this architecture since the CNU and VNU has multiple messages to process at a time, the hardware complexity is increased significantly. So Bit serial module is introduced to reduce the complexity. Check node is used to find the sign of the incoming messages and also find the minimum of magnitude of the input messages. In the proposed method, the sign of the incoming messages are computed by using XOR operation and the minimums are determined using AND operation. The check node output message is obtained by concatenating the sign bit and magnitude bit.
II. BIT SERIAL ARCHITECTURE
To reduce the complexity of the interconnect in fullyparallel LDPC decoders, in this paper a bit-serial approach is used for both communicating and computing extrinsic messages [8]. The difference between the conventional bitparallel scheme and a bit-serial scheme for transferring an nbit number, bn ……….b2b1 is in the number of clock cycles used. All the n bits are sent over n parallel lines in one clock cycle. Unlike bit-parallel, the bit-serial scheme will sent the message over a single line in n clock cycles. Stochastic computation is similar to bit-serial computation, in that it communicates extrinsic messages over single wires. It has a very simple check and variable node architecture but needs a significant amount of hardware overhead in order to translate the stochastic messages at the decoder inputs and outputs.
