Hardware-friendly Probabilistic Min-Sum Algorithm for Fully-parallel LDPC Decoders
Huang-Chang Lee, Chung-Chao Cheng, and Yeong-Luh Ueng Dept. of Electrical Engineering, National Tsing Hua University, Hsinchu, Taiwan, R.O.C.

Abstract—In order to simplify the check node operation of the low-density parity-check (LDPC) decoders, this paper presents a Normalized Probabilistic Min-Sum Algorithm (NPMSA), where the second minimum value is replaced by a probabilistic second minimum value. For NPMSA, the number of required comparisons can be reduced to about half compared to that of the conventional Normalized Min-Sum Algorithm (NMSA). It is shown that the simplification only introduces negligible impact on the bit-error rate performance, especially for codes with a high check node degree. When the proposed NPMSA is applied to the (2048, 1723) RS-LDPC code, the degradation in the error-rate performance is only about 0.05 dB. The hardware implementation shows that a throughput of 45.42 Gbps can be achieved using the proposed NPMSA.
Index Terms— High-throughput Decoder, Low-density Paritycheck (LDPC) Codes, Min-Sum Algorithm.
I. INTRODUCTION
With the capacity-approaching performance provided by using an iterative decoding algorithm, low-density paritycheck (LDPC) codes [1][2] have received significant attention. The related decoder can be realized using either the sum-product algorithm [2] or modified min-sum algorithms (MSAs), including both the normalized MSA (NMSA) and the offset MSA (OMSA) proposed in [3]. An LDPC code can be represented using a Tanner graph, which is a bipartite graph consisting of variable nodes (VNs) and check nodes (CNs). Two-phase message passing, which divides the decoding operations performed in a single iteration into the VN-operation and the CN-operation phases, was introduced in [1] and [2].
Nowadays, LDPC codes have been adopted in many communication standards, and a wide range of decoders have been implemented using the modified MSA [4][5][17][18]. For wired communications, high-throughput decoders for highrate LDPC codes are desired. In the IEEE 802.3an standard (10GBASE-T)[6], the throughput requirement is 6.67 Gbps, where the rate-0.84 regular (2048, 1723) Reed-Solomon (RS) LDPC code [7] is adopted. For next-generation high-speed communications, fully-parallel decoders are usually required in order to achieve the desired high throughput, where twophase message passing is an appropriate decoding scheduling approach.
For MSAs, the most crucial ingredient of the CN operation is the identification of the first and the second minimum values among all the inputs of each CN. The authors of [8] showed that the a high check node degree for high-rate LDPC codes leads to greater complexities in hardware and interconnects, which are difficult to manage using a fully-parallel architecture

without introducing degradation in error-rate performance. In order to simplify the CN operation, the Split-Row Threshold Algorithm (SRTA) was introduced in [11][12]. The paritycheck matrix (variable nodes) is divided into several block columns (groups) in SRTA, and the local minimum in each group is compared with a pre-defined threshold value.
Although the SRTA has simplified the comparison operation for CNs, the first and the second minimum values found in the block columns will be different to those found using conventional MSAs. Consequently, SRTA suffers from a loss in error performance of 0.2-0.3 dB compared to the NMSA [12]. In order to achieve a better trade-off between complexity and performance, a Normalized Probabilistic Min-Sum Algorithm (NPMSA) is proposed, where the first minimum value is kept the same as that generated using conventional NMSA, and only the second minimum value is replaced by a probabilistic second minimum value. Since the number of times the second minimum value is used is much less than that of the first minimum value, especially for a CN with a large degree, the proposed NPMSA can constrain the performance loss to within 0.05 dB, and reduce the number of comparisons required for CN operation by about half.
The proposed techniques are demonstrated by implementing a fully-parallel (2048, 1723) RS-LDPC decoder using a 90 nm CMOS process. Post-layout simulation results show that the proposed decoder achieves a throughput of 45.42 Gbps at a clock frequency of 199.6 MHz. This decoder achieves the highest throughput and throughput-to-area ratio among comparable works based on a similar or better error performance.
The remainder of this paper is organized as follows: The conventional NMAS is reviewed in Section II, and then the NPMSA is proposed. The impact of the probabilistic second minimum value is analyzed in Section III, and the performance of NPMSA is evaluated. In Section IV, the hardware implemetation is illustrated, and is compared with other decoder realiaztions. Finally, Section VI concludes this paper.

IV. HARDWARE IMPLEMENTATION
A. Realization Result
It was reported in [12] that a fully-parallel NMSA-based (2048, 1723) decoder can only achieve a utilization rate of 38%. This low utilization is due to the fact that the two minimum values and the corresponding index for each check node should be generated and propagated throughout the entire decoder, resulting in complex interconnects. Consequently, the SRTA was used in [11] and [12] in order to reduce the routing complexity of the LDPC decoder, where the paritycheck matrix is divided into several sub-matrices in a columnwise form, and only local minimum values for each sub-matrix are found and compared with a pre-determined threshold. As a result, the global-wire interconnects can be significantly reduced and a high utilization can be achieved for the decoder presented in [12]. However, the required first minimum value, min1, may not be the correct one since the decoding messages are not fully exchanged. This decoder suffers from an error performance loss of 0.2 dB compared to the conventional NMSA, as shown in Fig. 4. In contrast, the min1 value for each CN provided by the NPMSA is equal to the original NMSA. In addition, the proposed NPMSA can reduce the computational complexity of the CNUs with a minimal reduction in error-rate. The reduction in CN computations can not only reduce the hardware complexity of the computational units, but also reduce the numbers of connections required between these computational units, and, hence, achieve a high area efficiency. A fully-parallel (2048, 1723) LDPC decoder based on the proposed NPSMA where g = 2 was designed using 90 nm CMOS technology. The features of the chip are summarized in Table III.

A comparison of the metrics for the proposed LDPC decoder with the published results for other LDPC decoders is illustrated in Fig. 5, where normalized results are shown to compensate for the different CMOS processes that were used for these decoders. V. CONCLUSION Requiring only about half of the CN computations when compared to the conventional NMSA, the NPMSA proposed in this work can provide both the exact first minimum value and the probabilistic second minimum value. Since most of the C2V messages are generated using the first minimum value, the probabilistic second minimum value only introduces a negligible loss in error performance of 0.05 dB. The reduction in the computational complexity for the CN operation facilitates

the implementation of the fully-parallel LDPC decoder. The
decoder demonstrates a very high throughput with a compact
core area, which is highly desired for next generation high-
speed communication systems.
