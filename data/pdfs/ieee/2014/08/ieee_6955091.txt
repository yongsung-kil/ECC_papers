Rate-Compatible Spatially Coupled LDPC Codes via Repeat-Accumulation Extension
Wei Hou, Shan Lu, and Jun Cheng Dept. of Intelligent Information Engineering & Sciences
Doshisha University, Kyotanabe, Japan Email: hou.ita@outlook.com, shanluxd@gmail.com, jcheng@ieee.org,

Abstract—We propose the construction of rate-compatible spatially coupled low-density parity-check (SC-LDPC) codes. For a given high-rate SC-LDPC mother code, a family of low-rate member codes can be obtained by extending the protogragh of the mother code with repeat-accumulation (RA). Rate compatibility is obtained by adjusting the RA-extension parameters. Continuous code rate is achieved without extra cost. Density evolution analysis shows that the iterative decoding thresholds of all the member codes of the rate-compatible family are very close to the Shannon limits over the binary erasure channels.
I. INTRODUCTION
The rate-compatible code family [1]-[6], which consists of a set of member codes with different rates, can be processed by a single encoder at a transmitter and a single decoder at the receiver. A good design of a rate-compatible code family means that all member codes have excellent reliability. Because of good iterative decoding performance, the low-density paritycheck (LDPC) codes are widely used to design rate-compatible code families. There are two conventional design methods for rate-compatible LDPC codes: puncturing a mother code with a low rate [2]-[4] and extending a mother code’s graph with a high rate [4]-[6]. However, regardless which method is chosen, the degree distribution of each member code must be optimized to guarantee that the threshold of the beliefpropagation (BP) decoding is close to the Shannon limit.
On the other hand, by spatially coupling multiple protographs of a regular LDPC code, a convolutional-like LDPC code, which is also called a spatially coupled LDPC (SCLDPC) code, is obtained [7]-[12]. SC-LDPC code does not require complicated optimization of degree distribution because it has a BP threshold that approaches the capacity limit over discrete memoryless channels. The threshold saturation’s property was proved [13]. SC-LDPC code asymptotically achieves the maximum a posteriori probability (MAP) threshold of the corresponding regular LDPC block code with any rate. Because of these advantages, SC-LDPC code is a good option to structure a rate-compatible code family.
A feasible design of rate-compatible SC-LDPC codes is coupling the rate-compatible regular LDPC codes that were previously designed on a graph. An example of this design was presented [14]. This scheme achieved rate compatibility by successively extending the graphs of three-edge-type (TET) LDPC codes with higher rates, and all member codes can achieve the Shannon limit. However, a limiting factor of

the TET-extended scheme is that all member codes have to adopt the same check node degree. Because of this restriction, the TET-extended scheme only obtains discrete coding rates. When the coupled length is fixed, the smaller rate interval generates larger gaps between the BP threshold and the Shannon limit in low-rate regions. A hypothetical solution is removing this restriction of the check node degree in exchange for rate flexibility. But designing a family of regular LDPC codes with rate compatibility is not easy.
In this paper, we propose an alternative way to design rate-compatible SC-LDPC codes. Given a single SC-LDPC code, we structure a rate-compatible code family by extending the variable nodes at all of the coupled positions in the protograph of the given code with repeat-accumulation (RA). RA-extension consists of several complete accumulators and a partial accumulator. Rate compatibility can be achieved by configuring the number of complete accumulators and the proportion of the accumulated nodes in the partial accumulator. Unlike the convolutional TET-extended code family, we just use a single spatially coupled mother code to guarantee remarkable iterative decoding performance of all the derived member codes. Moreover, since the partial accumulator has good adjustability, our proposed scheme can achieve a continuous code rate without extra cost. Density evolution (DE) analysis shows that the BP thresholds of all the member codes of the proposed rate-compatible family are very close to the Shannon limits over the binary erasure channel (BEC).
II. RA-EXTENDED SC-LDPC CODES
We proposed rate-compatible codes that are obtained by extending a given SC-LDPC code with repeat and accumulation. Our proposed code, called RA-extended SC-LDPC code, is denoted by C(J, K, L, q, α). Its protograph consists of two parts: (J, K, L)-chain and (q + α)-extensions. Fig. 1 shows a protograph of code C(J = 3, K = 6, L, q, α). The circles and the squares are variable nodes and check nodes, respectively.
A (J, K, L)-chain is a protograph of a spatially coupled code, which is considered the mother code of the ratecompatible codes. In (J, K, L)-chain part, shown in the lower half of Fig. 1, J and K denote the degrees of each variable node and each check node of the mother code, respectively. L denotes the coupled length. 3 ≤ J < K and J, K, L are nonnegative integers.

The extension part, shown in the upper half of Fig. 1, consists of L (q + α)-extensions. Each is used for one group of variable nodes at the same coupled position of the mother code. Each (q + α)-extension consists of q 1-extensions and a single α-extension. q is a nonnegative integer, α is a real number, and 0 ≤ α < 1. We give more detailed explanation of the α-extension next.
As a small blueprint of code ensemble, the protograph of Fig. 1 derives the Tanner graph of a specific code by copying itself many times and permuting the edges that connect the same variable nodes and check nodes in each copy [15]. The above α-extension can be easily described in the Tanner graph illustrated on the right side of Fig. 1. Without loss of generality, we assume there are N variable nodes at each coupled position for the mother code’s Tanner graph. After permuting the N variable nodes, we accumulate the first αN variable nodes and ignore the rest of the (1 − α)N ones. Due to 0 ≤ α < 1, the α-extension is a protograph of the partial accumulator. On the other hand, the 1-extension is complete, since all of the N accumulated nodes are retained.
The following are the three features of proposed code C(J, K, L, q, α):
a) spatially coupling based construction, b) arbitrary rate, and c) rate compatibility. Beginning with feature a), we discuss the three features in order. Our proposed code is structured by extending a given (J, K, L)-chain that is actually spatially coupled mother code C(J, K, L, 0, 0). IV. NUMERICAL RESULTS
In this section, we give some numerical results of our proposed RA-extended SC-LDPC code. In the process of calculating the BP thresholds, we set 105 as the maximum iteration number of the variable nodes and set 10−7 as the breakout condition of the erasure probability at every coupled position.
Before describing the rate compatibility, we discuss the influence of different parameters on our proposed codes. Table 1 illustrates the BP thresholds of C(3, 6, 100, 5, q, α = 0) with various qs. Here the gap is the difference between Shannon limit ϵSh and BP threshold ϵBP ; there are only q 1-extensions. The mother code is C(3, 6, 100, 5, q = 0, α = 0). The smallest gap is 0.0132 with q = 2, and the largest gap is 0.0291. The BP thresholds of code C(3, 6, 100, 5, q, 0) for various qs are very near the Shannon limits.
Figure 3 shows the gaps between the Shannon limits and the BP thresholds of code C(3, K, 100, 5, q, α) with various q + αs, where the check node degrees of mother codes K = 6, 8, 10, and 12. From Fig. 3, when q = 0 and α = 0, the code with larger K has a smaller gap. On the contrary, when q + α > 0, the codes with smaller K have smaller gaps. By observing each of the four curves, we see a global maximum value of the gap at 0 < q + α < 1, showing that without the 1extensions, the α-extension may degrade the iterative decoding performance of our proposed code. For q ≥ 1, with an increase of q + α, the gap quickly decreases to a minimum value and slowly increases to a stable value that is actually very small. In the q + α > 25 regions, which is not illustrated in Fig. 3, the numerical value of the gap very slowly trends toward 0 as q + α goes to infinity. This shows that the (q + α)-extension is reliable for our proposed code when (q ≥ 1). Comparing the four curves, the codes with K = 6 and 8 are better since even their largest gaps are not larger than 0.065.

Due to the flexibility of q + α, the rate-compatible RA-extended code families achieve arbitrary points on the curves. In contrast, the curves of the TETextended code families are discrete. V. CONCLUSION
In this paper, we proposed a construction of a ratecompatible SC-LDPC codes and obtained the family of lowrate codes by RA-extending variable nodes in the protograph of a given high-rate SC-LDPC code. Rate compatibility was achieved by adjusting the RA-extension parameters. Density evolution analysis shows that the BP thresholds of our proposed scheme approach the Shannon limits over the BEC.
