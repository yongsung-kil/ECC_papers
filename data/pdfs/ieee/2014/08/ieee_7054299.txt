Work In Progress: A New Algorithm to Improve the Decoding Success Probability of Raptor Code

Huihui Li1, Xiangying Li1, Pei He1, Shanqiang Yi1, Dewei Yang1, Huafeng Liu2 1School of Information and Electronics, Beijing Institute of Technology, Beijing, China 2Research Center of Beijing Space Information Relay Transmission Technology, Beijing, China

Abstract—Raptor code is a kind of new fountain code which usually adopts belief propagation(BP) decoding algorithm. In order to improve the decoding success probability of Raptor code, this paper proposes two methods. On the one hand, we adopt a new construction method of check matrix for LDPC code, which can effectively avoid the girth of 4. On the other hand, we analyze and simulate the number of stopping set after BP decoding and propose a new decoding algorithm–the cascaded iterative algorithm based on the Ripple detection. Comparing our algorithm with BP, simulation results show that the proportion of encoded packages with degree 2, 3 in stopping set has decreased apparently and the success probability of Raptor decoding has improved.
Keywords—Raptor Code; stopping set; BP decoding; Ripple
I. INTRODUCTION
The concept of digital fountain was first proposed by John Byers and Michael Luby in 1998 [1]. The digital fountain code is no longer concerned about the channel conditions between transmitter and receiver, which is different from the traditional forward error correction coding techniques. There are two kinds of fountain code: LT code and Raptor code. LT code was the first real implementation of digital fountain [2]. In 2006, Shokrollahi made improvements based on LT code and then proposed Raptor code [3]. Raptor code adds the precoding technique into LT code, which significantly improves the performance. In order to design high-performance Raptor code, LDPC is usually used in the precoding [4]. Digital fountain code has low complexity for both encoding and decoding, no fixed code rate and doesn’t need feedback. All these advantages made it suitable for time-varing channel and was chosen by the third generation of cellular network in multimedia broadcast/multicast service(MBMS) and DVB-H standards.
We will organize the rest of the paper as follows: Section II introduces the principle of Raptor code, LDPC precoding and LT decoding are described. Section III describes the proposed algorithm-the cascaded iterative algorithm based on Ripple detection in detail. Section IV shows the influence of the improved algorithm on LT and Raptor code through simulation and compares it with BP algorithm . The conclusion is given in section V.

II. PRINCIPLE OF RAPTOR CODE
1) LDPC precoding: Using error correction capability of precoding, Raptor code can achieve lower decoding complexity and higher success probability under the same decoding overhead. LDPC (Low Density Parity Check), is such a kind of code with strong error correction capability [5]. Decoding performance of LDPC depends on the construction of check matrix H. The construction methods are divided into random method and structured method. The check matrix doesn’t have certain structural features using random construction method, which leads to high complexity. Therefore, structured construction method is more popular in practice. Structured extended irregular repeated-accumulate (SeIRA) code [6] is such a kind of LDPC code with linear complexity and structural features. In this paper, the construction method of patent [7] is adopted to generate the check matrix, because the matrix H constructed by this method doesn’t contain girth of 4.
2) LT decoding: BP algorithm and GE (Gaussian Elimination) algorithm are two main algorithms of LT decoding. GE algorithm can make full use of the encoded packages to decode the original packages [8]. However, it has high decoding complexity and the amount of calculation O(N K2) grows fast with the increase of K. Therefore, GE algorithm is not suitable for long code in practice. Since GE algorithm isn’t a practical choice, we focus on BP algorithm below.
a) Definition 1 decoding overhead: LT decoding needs to receive more than K encoded packets, the number of received packets divide K is called decoding overhead.
b) Definition 2 stopping set: when there is no package with degree 1, the decoding process stops, the collection of the remaining packages called stopping set.
Process of BP decoding is as follows: (1) Select an arbitrary encoding packet with degree 1. If such a packet exists, recover original packet connected with it, then delete this package and all the connections with it from the graph. If there is no such encoding package, the decoding stops. (2) Repeat process (1), until there is no encoding packet with degree 1. Fig. 1 shows a simple BP decoding process: there are three original packets s1, s2, s3 and four encoded packets t1, t2, t3, t4 , the size of each packet is one bit. Assuming t1, t2, t3, t4 = 1011 at the beginning of the decoding. From Fig. 1, we can see that BP decoding depends on the encoded packets with degree 1. If there is no such packet, the decoding stops. III. IMPROVED DECODING ALGORITHM

Ref. [9] presents an improved algorithm–BPML algorithm.

It is a cascade of BP and GE algorithm. When BP decoding

stops because of the stopping set, GE algorithm is used. It can

eliminate the influence resulting from stopping set and improve

the decoding success probability. However, this algorithm

performs well only when BP decoding has recovered the most

part of original packets.

In this paper, we present an improved decoding algorithm.

Theoretically, this algorithm can effectively eliminate the in-

fluence of stopping set, while the complexity doesn’t increase

a lot. We might call the proposed decoding algorithm as the

cascaded iterative algorithm based on Ripple detection, or

Ripple algorithm for short.

Assuming that the receiver has received N encoded data

packets, firstly we use BP algorithm for decoding, when there

is no encoded packets with degree 1 (Ripple = 0), the

decoding stops. We analyze the stopping set after BP decoding,

the results are shown in Fig. 2. Here we take the received

packets N = 1.1K, K changes from 100 to 1000. Through

the simulation, we find that the proportion of packets with

degree 2 and 3 accounts for 30% and 10% in the stopping set

respectively, and the total proportion accounts for about 40%.

Therefore it is necessary to find an algorithm to continue the

decoding process in this case.

Therefore, we should find out the encoded packets with degree 2, 3 in the stopping set after the BP algorithm fails. In order to make the equations solvable, we classify the unsolved source packets: put the packets with direct or indirect relationship into one class. For the encoded packets with degree 3, if any two of the source packets are in the same class, we can solve the other source packet out [10].
In most cases, a part of source packets can be derived. After obtaining these source packets, we update the generation matrix, similar to BP algorithm, then check the Ripple size. When the size of Ripple reaches a threshold value, BP decoding will be started again. The procedure of the cascaded iterative algorithm based on Ripple detection is as follows:
(1) Use the BP decoding algorithm. As BP algorithm has the lowest complexity, we hope to recover as many packets as possible using BP.
(2) Find out the encoded packets with degree 2 in the stopping set when the BP algorithm stops. Classify the source packets, named the classes D1, D2, D3 · · · .
(3) Search an encoded packet with degree 3 in the stopping set, as ti, the generate equation is si + sj + sk = ti, compare the three source packets with the classification, and check if any two of them are in the same class. If so, solve the equation and get the third source packet.
(4) Use the solved source packets to update the generation matrix, similar to the BP algorithm updating process. Add s with all encoded packets connected to itself, and delete the connection between them. This helps release more encoded packets with degree 1. Meanwhile, record the number of decoded packets. If the number equals to K, it means the decoding succeeds and the procedure stops, otherwise the procedure continues.

IV. RESULTS AND ANALYSIS
A. Improved LT decoding
In this section, the success probability of LT decoding are simulated respectively for BP and Ripple algorithm under different decoding overhead, when K = 100, results are shown in Fig. 3, when K = 1000, results are shown in Fig. 4. Considering erasure channel in this simulation, the erasure probability p = 0.3, choose robust soliton distribution as degree distribution, take δ = 0.05, c = 0.03,λ = 2, and code length K to be 100 and 1000 respectively, then repeat the decoding process 100 times.
It can be clearly seen from Fig. 3, success probability of Ripple decoding algorithm is higher than BP decoding on average when K = 100. Under the condition of decoding overhead changing from 1.2 to 1.5, the success probability increase about 20% on average, up to 30% at most. As the decoding overhead increasing, the performance gap between BP and Ripple decoding becomes unapparent, for the reason that BP algorithm can decode more efficiently with a larger overhead.

As shown in Fig. 4, the performance of Ripple algorithm is still better than BP algorithm when K = 1000 and the decoding overhead is between 1.1 and 1.25. The success probability can increase about 15% on average, up by 20% at most by using Ripple decoding algorithm compared with BP decoding algorithm.
From the analysis above, we know that the proposed algorithm performs well for short code. When the length of original code becomes longer, the performance of Ripple algorithm becomes poor. This is because BP algorithm already decode most of the packets successfully at small decoding overhead and the number of encoded packages with degree 2 or 3 is reduced in the stopping set.
B. Improved Raptor decoding
In order to prove the performance of SeIRA code constructed in the patent [7], we take quasi cyclic LDPC code (QCLDPC) in the IEEE 802.16e (WiMAX) standard as a contrast. The LDPC decoding is based on BP algorithm with code length 576 and code rate 1/2. The bit error rate and frame error rate performance curves are in Fig. 5.
In Fig. 5, SeIRA code constructed in patent [7] performed slightly better than the LDPC code in WiMAX. The reason is that the parity check matrix constructed by the SeIRA code avoids girth of 4, and that is why we choose SeIRA code as LDPC precoding method in this paper.
In order to understand the influence of precoding, Fig. 6 shows the decoding performance of LT code and Raptor code. In Fig. 6, we use SeIRA-LDPC code with code rate R = 1/2 and code length K = 576 for precoding. The erasure channel is adopted in simulation, the erasure probability p = 0.3, choose robust soliton distribution as degree distribution, take δ = 0.05, c = 0.03 and λ = 2. The decoding adopts Ripple algorithm proposed in this paper. The simulation is repeated 100 times.

As shown in Fig. 6, Raptor code starts decoding earlier than LT code thanks to the adoption of precoding. It can be seen from Fig. 6 that Raptor code starts decoding when receives 610 encoded packets, however, LT code starts decoding when receives 640 coded packets. V. CONCLUSION
Improving the decoding success probability with a small decoding overhead is a very important aspect in Raptor code research. Two methods are given in this paper. In the first method, we use a new construction method of LDPC check matrix, which can avoid the girth of four effectively. In the other method, we simulate and analyze the stopping set when BP decoding stops and present Ripple algorithm based on BP algorithm. The simulation results show that the Ripple algorithm performs much better than BP algorithm when LT code is short. Applying two improvements to Raptor code can improve the decoding success probability significantly.
