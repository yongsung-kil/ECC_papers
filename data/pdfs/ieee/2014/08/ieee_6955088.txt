Improving the finite-length performance of long SC-LDPC code chains by connecting consecutive chains

Pablo M. Olmos∗, David G. M. Mitchell†, Dmitry Truhachev‡, and Daniel J. Costello, Jr.† ∗Signal Theory and Communications Dept., University of Carlos III in Madrid, Spain, olmos@tsc.uc3m.es
†EE Dept., University of Notre Dame, Notre Dame, Indiana, USA, {david.mitchell, costello.2}@nd.edu ‡ECE Dept., Dalhousie University, Halifax, Canada, dmitry@dal.ca

Abstract—We propose a novel encoding/transmission scheme called continuous chain (CC) transmission that is able to improve the finite-length performance of a system using long spatially coupled low-density parity-check (SC-LDPC) code chains. First, we show that the decoding of SC-LDPC code chains is more reliable for shorter chain lengths, i.e., the scaling between block error rate and gap to threshold is more favorable for shorter chains. This motivates the use of CC transmission in which, instead of transmitting a sequence of independent codewords from a long SC-LDPC chain, we connect multiple chains in a layered format, where encoding, transmission, and decoding are now performed in a continuous fashion. Finally, we show that CC transmission can be implemented with only a small increase in decoding complexity or delay with respect to a system employing a single SC-LDPC code chain for transmission.
I. INTRODUCTION
Spatially coupled low-density parity-check (SC-LDPC) code ensembles are known to achieve capacity over binaryinput memoryless symmetric channels under belief propagation (BP) decoding [1]. SC-LDPC code ensembles are constructed by coupling together L (l, r)-regular LDPC code ensembles, each one of length M bits, into a chain with appropriate boundary terminations, thus creating strong subcodes at both ends of the coupled chain. When M tends to infinity and L is large, SC-LDPC code chains exhibit a BP threshold close to the maximum-a-posteriori (MAP) threshold of the (l, r)-regular code ensemble [1], [2]. In this paper, we are interested in the finite-length block error performance of SC-LDPC code chains. Scaling laws (SLs) to analyze the finite-length block error rate of SC-LDPC code chains over the binary erasure channel (BEC) have recently been proposed for large L values in [3], [4] and for small values of L in [5]. SLs are useful to understand the scaling between block error rate, gap to threshold, and code parameters, i.e., l, r, M , and L. In this paper, we provide a unified vision of SLs for short and long SC-LDPC code chains and show that they have important and practically-relevant differences. More specifically, when the chain is short, so that the boundary terminations are close, there is an improvement in decoding robustness beyond what SLs for long chains can predict. This result is not discussed in [3]–[5] and it is the first contribution of this paper.
Based on this new understanding of the decoding dynamics of short and long chains, we propose a novel encoding/transmission technique that improves the finite-length performance of a system using a long SC-LDPC code chain. When a single terminated SC-LDPC code chain is used for

transmission, the information is encoded into independent codewords corresponding to unconnected consecutive chains. We propose to link consecutive chains by creating a dependence between them, thus forming a special pattern that resembles the SC-LDPC code ensembles based on connected chains presented in [5], [6]. It should be noted that we are not proposing a novel construction for SC-LDPC code ensembles based on connected chains. Rather, our main contribution is to show that the stronger sub-codes at the termination points of unconnected chains can be exploited in a more effective way if we connect consecutive chains. The underlying principle is to generate reliable information at various points in the graph, effectively breaking long chains into better protected shorter chains, while maintaining the same rate as a single long SC-LDPC code chain. We refer to this encoding/transmission scheme as continuous chain (CC) transmission. We propose ensembles of SC-LDPC codes based on connected chains for which CC transmission is practically feasible. In particular, CC transmission only requires some additional memory in the encoding process and, to effectively implement a windowed decoder [2], [7], a different order in the sequence of transmitted bits compared to transmitting unconnected chains. To the best of our knowledge, this is a new concept in the field of SC-LDPC codes. Simulation results for the BEC show that CC transmission significantly improves the finite-length performance even for very long code lengths, e.g., M = 2000 bits per position and a chain length of L = 50.
II. THE SC-LDPC CHAIN ENSEMBLE
In this section we review the construction of a SC-LDPC code chain. We use the random construction presented in [3], [4] and the BEC for our analysis. A particularly interesting class of codes can be constructed by means of protographs, with important practical advantages [8]; however, these constructions significantly complicate the analysis with respect to the random SC-LDPC construction. Nevertheless, the results derived in the paper can be directly applied to the design of protograph-based SC-LDPC code ensembles.

III. FINITE-LENGTH SCALING BEHAVIOR OF THE SC-LDPC CODE CHAIN ENSEMBLE
In this section we provide insight into the estimation of the finite-length performance of SC-LDPC code ensembles using scaling laws. Analysis of the finite-length performance of LDPC codes is typically performed for the case of transmission over the BEC [10]. For the BEC, we consider an equivalent formulation to BP called peeling decoding (PD), which iteratively removes variable nodes with known (decoded) values from the Tanner graph. In [9], it was shown that if we apply PD to an LDPC code, the sequence of graphs follows a typical path or expected evolution. Based on the graph covariance evolution at those points where the expected evolution of degree-one check nodes in the graph presents a local minimum (the critical points), SLs for PD were proposed to determine the scaling behavior between the LDPC code ensemble block error rate, length, degree distribution parameters, and the gap to threshold [10]. To illustrate the main differences in the decoding of short and long SC-LDPC code chains, it suffices to observe the ensemble expected graph evolution over the PD process.

IV. CONNECTING CONSECUTIVE CHAINS
Now we consider the encoding and transmission of codewords using a code from the C(3, 6, L) ensemble. In the following, we restrict our discussion to long chains, e.g., L = 50, for which the asymptotic C(3, 6, L) BP threshold C(3,6,L) ≈ 0.48815 is close to channel capacity [2]. The information stream is divided into blocks of M LrC(3,6,L) bits, which are then independently encoded, transmitted, and decoded at the receiver. In the following, by transmitting independent chains we mean the transmission of independent codewords that belong to a C(3, 6, L) code chain. In this section, we show that the performance of this system can be significantly improved if we do not transmit independent chains but, instead, data is encoded in a continuous fashion using a convolutional-like structure based on connected SC-LDPC code chains. We refer to such an encoding scheme as continuous chain (CC) transmission of SC-LDPC codes. More specifically, N consecutive SC-LDPC code chains, that without CC transmission are independently transmitted and decoded, will be now connected such that the overall robustness against failures is improved and the decoding complexity and delay are not significantly affected.
For N = 2, this is precisely the effect achieved by the code ensemble shown in Fig. 3. This code consists of two layers with one SC-LDPC single chain of length L per layer, and it is denoted by S(3, 6, L, 2). The chains in the two layers have been connected to increase the degree of the variable nodes in the four middle positions of the layer 1 chain, from position L/2 to position L/2 + 3. In this way, we create a region with much better protection in the middle of the layer 1 chain, where the variable node degree profile is (4, 5, 5, 4). All variable and check nodes in chain 2 are of degrees 3 and 6 respectively. Therefore, to be effectively decoded, the layer 2 chain relies on the decoding of the four middle positions of the layer 1 chain. Note that, if positions L/2 , . . . , L/2 + 3 are correctly decoded, then the layer 2 chain has the same DD as the SC-LDPC single chain discussed in Section III.
By computing the expected evolution of degree-one check nodes for the S(3, 6, L, 2) ensemble [5], we observe that, despite the fact that the S(3, 6, L, 2) threshold is exactly the same as the single SC-LDPC chain ( S(3,6,L,2) ≈ 0.48815), all variable nodes in positions ( L/2 , . . . , L/2 + 3) in the layer 1 chain can be successfully decoded up to an erasure probability of = 0.505. Therefore, for ≤ S(3,6,L,2), these positions are decoded almost surely, which has two important consequences. First, the finite-length error rate in chain 2 will be similar to that of the single LDPC code chain and, second, low-degree check nodes will be created at positions L/2 − 1, L/2 , L/2 + 3 and L/2 + 4 in the layer 1 chain. Consequently, this chain is essentially decoded as

two interdependent chains of length L/2 . Further, if we use intermediate chain lengths, i.e., L ∈ [40, 70], the decoding of each of the two segments of length L/2 is well predicted by the single-critical point model in (7), and we can benefit from much better properties in terms of finite-length scaling behavior, as discussed in Section III. As a consequence, the block error performance of the layer 1 chain will be better than for a single SC-LDPC chain of length L.
In Fig. 4, we give simulated block error rate results for a code drawn from the S(3, 6, L = 50, N = 2) ensemble over the BEC.1 We show the block error rate computed for the chains in 1 layer (solid lines) and layer 2 (dashed lines) as well as the performance of a code drawn from the C(3, 6, 50) single chain ensemble. Note that the comparison between both ensembles is fair since we consider two independent consecutive codeword transmissions from the single chain ensemble and the transmission of the two layers in the connected chain ensemble. As predicted, we obtain a significant gain in performance for the layer 1 chain, of almost one order of magnitude, even for codes with 2000 bits per position. Also, note that there is no performance degradation in the layer 2 chain with respect to a single chain.
The design methodology followed to construct the S(3, 6, L, 2) ensemble can be extended to an arbitrary number N of layers, where the layer j chain is connected to layer j-1 chain using the same criteria as shown in Fig. 3. In general, we denote such ensembles as S(3, 6, L, N ), where N stands for the number of layers. For instance, in the S(3, 6, L, 3) ensemble, by using the low-degree check nodes of the layer 3 chain, we increase the protection of the variable nodes at positions L/2 , . . . , L/2 + 3 in the layer 2 chain. In addition to what we observed in Fig. 4, note that the performance measured for the chains in layers 1 and 2 is essentially the same, and hence there is no degradation in performance even though the layer 2 chain does not have low-degree check nodes at each end. This important result can be simply understood as follows: once the positions in the middle of the layer 1 chain have been decoded, besides splitting this chain into two sub-chains of length L/2 , low-degree check nodes are created at the boundary positions of the layer 2 chain. This chain now looks similar to the layer 1 chain at the beginning of the decoding process and, consequently, the performance in layers 1 and 2 is the same. This is critical to the success of the CC transmission approach for an arbitrary number of layers.
When the rate constraints are more strict and, as a consequence, we have to use even longer chains, e.g., L = 100, the S(3, 6, L, N ) ensemble might not be able to provide the same gain per layer as a shorter chain, e.g., L = 50. In this case, we can break each chain of length L = 100 into three shorter segments by creating two intermediate regions with stronger protection, using two chains at layer 2 to improve the performance in the layer 1 chain. If this CC scheme is extended for a number N of layers, where each strongly protected chain is broken into three shorter segments using two chains in the layer below, the ratio of the number of protected chains to the total number of chains tends with N to 1/2.
Compared to transmission of independent code chainwords, CC transmission using the S(3, 6, L, N ) ensemble only requires some additional memory and a different transmission order for the encoded bits. Regarding the decoding process, a window decoder [2], [7] is easily implemented with the condition that, when the middle positions L/2 , . . . , L/2 + 3 of the layer i chain are active, the channel information corresponding to the variable nodes in the boundary positions of the layer i + 1 chain must be available. Therefore, implementing efficient window decoding for CC transmission requires a change to the order in which the encoded bits are transmitted, so that the receiver has the necessary information at the appropriate time.
V. CONCLUSIONS
In this paper, we reviewed existing results to capture the main scaling behavior between error rate and code parameters for a SC-LDPC code chain. Then, we showed that there is an important design aspect regarding robustness against failures of short SC-LDPC code chains compared to long chains. Based on this result, a novel transmission scheme (CC transmission) designed to boost the performance of a system using long SC-LDPC code chains was introduced. We have shown that, by connecting consecutive SC-LDPC code chains rather than transmitting the codewords corresponding to each one independently, we obtain a significant performance improvement with only a minor change in the order in which coded bits are transmitted and some additional memory requirements at the encoder.
