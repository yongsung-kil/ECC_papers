Abstract- Low Density Parity Check codes (LDPC) have shown good error correcting performance which enables efficient and reliable communication. A subclass of LDPC codes known as quasi-cyclic LDPC codes are used whose parity check matrices consists of circulant permutation matrices. QC-LDPC codes require less memory as compared to LDPC codes. This paper presents a low complexity quasi-cyclic low density parity check (QC-LDPC) decoder. QC-LDPC codes require less memory as compared to LDPC codes. Partially parallel, low complexity decoder architecture has been designed for single-mode decoding. This decoder is modeled in Verilog, synthesized and performed place and route for the design using Xilinx ISE 12.1.

LDPC codes used in these high speed communication standards are irregular codes. The main advantage of using irregular codes is that they outperform regular codes and the major disadvantage is that it causes signal congestion problems during VLSI implementation and requires more area for routing wires. Due to quasi-cyclic structure, QC­ LDPC codes can be encoded efficiently with shift registers [5]. In addition, QC-LDPC codes also facilitate efficient high-speed decoding because of the regularity in its parity check matrix.

To attain reliable communication forward error correction coding schemes play an important role in high data rate wireless and multimedia applications. Low Density Parity Check codes [1] are a special type of error correcting codes that is known for their good decoding performance and high throughput. LDPC codes can be constructed by two methods: random codes and structured codes. Quasi-cyclic (QC) LDPC codes are the most promising class of structured LDPC codes due to their ease of implementation and excellent performance over noisy channels when decoded with message-passing algorithm. The decoding algorithm of LDPC is inherently parallel and can achieve performance close to the Shannon Limit. Newly high speed communication standards such as IEEE 802.16e [2], IEEE 802.lln [3] and DVB-S2[4] have considered employing

parallel. Serial decoder is simplest in terms of area and routing. It consists of single check node, single variable node and memory. The variable node is updated one at a time and check node is updated in serial manner. It has less throughput . Fully parallel decoders directly instantiate the bipartite graph of the LDPC code to the hardware [5]. These decoders can achieve very high decoding throughput but suffer from high implementation complexity. Partially Parallel LDPC decoders [6] map a certain number of variable nodes or check nodes in to a single decoding unit in time division multiplexing mode. In this paper we design a low complexity parity check decoder for single mode. Due to quasi-cyclic structure a small base matrix can be replaced by a large parity check matrix which results in good error correcting performance and directly fit in to partially parallel decoder implementations.

An LDPC code is a class of linear block code represented by a sparse M x N parity-check matrix H . The (n, k) LDPC codes shows that a k bit message can be encoded to be n bit codeword [2]. The parity check matrix H as shown in (2) consists of (n-k) rows and n columns and the corresponding code rate R is defined as kin. LDPC codes can be represented by a bi-partite graph called a Tanner graph. There are two classes of nodes in a Tanner graph, variable nodes and check nodes [2]. Check nodes represent the rows and variable nodes represent the columns of the parity check matrix (PCM). Tanner graph representation of PCM is shown in "fig.1". If the element in the parity check matrix is 1 it represents that check node is connected to variable node. The number of edges in tanner graph is equal to number of l's in parity check matrix.

QC-LDPC codes are defmed by very sparse base matrix[8]. QC-LDPC codes are codes in which rows or columns in a submatrix have similar and cyclic connections[lO]. Row column connections in QC-LDPC codes are constructed by shifting identity submatrices. Decoding principle of LDPC is based on message passing algorithm [7]. Message passing has broadly two categories: Hard decision and Soft decision. Bit flipping (BF) algorithm is the example of hard decision where as Belief Propagation (BP) is the example of soft decision.
BP decoding algorithm utilizes both the sign and the reliability information (magnitude) therefore it is also called Soft decoding. Soft decoding outperforms hard-decision decoding which relies only on the sign and therefore the preferred method. The main objective of BP algorithm is that the probabilities of bit nodes being 1 are exchanged between connected check nodes and variable nodes during each iteration cycle.

IV. PARTIAL PARALLEL ARCHITECTURE
In partial parallel method time division multiplexing is used in which a certain number of variable nodes or check nodes are mapped in to a single decoding unit.
The block diagram of single mode decoder depicting the hardware interfaces is shown in "fig 3". The decoder consists of 'Clock' and 'Reset' inputs[2]. The maximum permissible number of iterations is determined by the value supplied at the ITER input. This can be set at a value in the range 0-11. In order to get higher code length for code rate R=3/8 we require base matrix Hb of size 5x8. Then this matrix will be expanded to a bigger matrix by a factor known as expansion factor p=9. The size of expanded matrix H is 45x72. Thus, for single mode decoder we require 45 CNU's and 72 VNU's.

Using the LLR we can initialize the input buffers. N is the number of RAMs i.e. maximwn colwnn of base matrices. Z is the expansion factor which is also depth of each RAM. Thus the reconfigured maximal code length will be N x Z.
