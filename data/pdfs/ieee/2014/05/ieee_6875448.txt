Design and Test of Adaptive Computing Fabrics 
For Scalable and High-Efficiency Cognitive SoC Applications 
Pascal Nsame, Guy Bois and Yvon Savaria 
Microelectronic and Microsystems Research Group 
Electrical and Computer Engineering 
Polytechnique Montreal, Canada 
p.nsame@polymtl.ca 
 
 
Abstract—In this paper, a new adaptive computing fabric 
(ACF) that achieves both real-time multi-mode/multi-rate 
adaptation and lower error floor for cognitive SoC applications is 
presented. The VLSI architecture of the ACF is experimentally 
shown to meet the target specifications of several IEEE standards 
including DVB, 802.3an and 802.ad. Our ACF design delivers a 
Bit Error Rate (BER) of 10-14 at a bit energy-to-noise density of 
Eb/N0=5dB with an energy-efficiency of 0.61pJ/bit. Experiments 
using three ACF designs are conducted, comparing error 
correction performance for Low-Density Parity-Check (LDPC) 
codes adopted by the IEEE standards for commercial 
applications. We report on a 201.6Gbps 65nm CMOS ACF 
design and Xilinx ACF FPGA prototype, which demonstrates in 
hardware how real-time adaptive techniques can accelerate 
decoding convergence and lower the error floor. Finally, We 
show experimentally that our ACF design can achieve energy-
efficiency throughput speed-ups at scale in the range of 200x to 
5000x as compared to the same algorithm running in software 
(optimized C program) on a single CPU core. 
Keywords — Cognitive SoC; Adaptive Computing Fabric; 
Reliable Communication, Embedded Test Generation. 
I. 
INTRODUCTION  
The evolution of multi-core processors provides enhanced 
ways of accelerating performance mainly through Thread-
Level Parallelism (TLP). As a result, Chip Multiprocessor 
(CMP) has become the de factor architecture choice for 
processor design due to the ease of use of the share-memory 
model. However, shared-memory brings the overhead of 
memory consistency and coherency problems as shown by 
recent research that demonstrate the implications of cache 
coherence overhead on application performance including 
large high-performance clusters with multi-socket CMP based 
shared memory nodes [1]. This paper is motivated by the need 
for a new approach to VLSI architecture design, one that can 
scale with the application requirements in a manner that is 
energy-efficient. To that effect, we choose the LDPC decoding 
function to evaluate this new approach. Currently, no flexible 
LDPC decoder architecture has been published, that can 
achieve simultaneously, the throughput, power and error 
correcting performance required by all the IEEE standards that 
have adopted LDPC codes for commercial applications. 
Despite the fact that LDPC codes have made their way into 
virtually every modern high-performance wireless standard 
because of their powerful error correction capability, the 
length, irregularity and computation requirements of LDPC 
codes make their software [2] and hardware implementation 
[3] extremely challenging when support for multiple standards 
for storage applications including Flash, Solid-State Drive, 
MRAM and Tape as well as for communication applications 
including DVB, 802.3an, 8023ad, 802.15 and 802.16 is 
required. First, their software implementation on shared-
memory CMP platforms provides very limited energy-
efficiency despite a reported 26x speed-up on GPU devices, 
which comes with a high cost in chip area and power. Second, 
parallelism in the design [4] is no longer sufficient to make 
their VLSI hardware implementation energy-efficient with 
multi-mode/multi-rate flexibility. Alternative method of 
accelerating the performance of LDPC decoder have been 
investigated in related work including the use of adaptive 
quantization [5], binary differential message passing [6], 
layered architecture [7]. However, they target only regular, or 
short-length, or single rate, or irregular LDPC codes with 
limited energy-efficient throughput. In [8], Saejoon Kim 
proposed an adaptive informed dynamic scheduling scheme, 
however a VLSI implementation is not demonstrated. Modern 
applications require functions, including LDPC decoding, that 
are fully parallel and programmable with very high energy-
efficiency. In this paper, we focus on the design and test of an 
adaptive computing fabric architecturally optimized to evaluate 
multi-mode/multi-rate/multi-standard 
implementations 
of 
LDPC encoder/decoder (CODEC) for scalable and high-
efficiency cognitive SoC applications. In summary, this paper 
makes the following contributions: 
• To the best of our knowledge, this paper reports for the first 
time a VLSI hardware implementation of a multi-
mode/multi-rate LDPC CODEC that adaptively accelerate 
decoding convergence, while lowering the error floor for 
jointly designed architectures with the required flexibility 
to support multiple standards. 
• We experimentally demonstrate for the first time, that 
FPGA prototypes and ASIC implementations of adaptive 
computing fabric architectures can deliver throughput 
speed-ups that are by several orders of magnitude more 
energy-efficient when compared to publish alternatives 
including recent multi-core GPU architectures. 
The rest of the paper is organized as follow: Section II 
describes the ACF architecture used in our experiments. 
Section III provides design & test implementation details.

ADAPTIVE COMPUTING FABRIC ARCHITECTURE AND 
FUNCTIONAL DESIGN 
A. Scalability, Error Resiliency & Energy-Efficient 
Throughput Enhancements 
The integrated IBM PowerPC440 processor shown on Fig. 
1 is in charge of generating and applying manufacturing test 
and functional test programs including encoding frames, 
noising the code words from an AWGN channel and 
computing the BER performance. In order to exploit the 
resiliency limits of modern communication systems moving 
away from the 100% reliable computing paradigm, we 
designed the ACF to enable self-testing. This is done be 
extending the approach discussed in [9] and [10] and used to 
generate test patterns for small delay faults. During functional 
operation, from the decoder’s point of view, host CPU (e.g. 
dual-core ARM A9 application processor) send and receive 
data to/from global memory and/or I/O devices. All operations 
related to the LDPC decoding function including the new 
adaptive techniques evaluated in our experiments and 
summarized in Table I are transparent to the host CPU and 
performed by the adaptive computing fabric (ACF) in real-
time, concurrently and independently from the host CPU 
operations as illustrated in Fig. 2 and Fig. 3. These new 
adaptive techniques described in Section III include: an 
adaptive scheduling and quantization, an adaptive partitioning 
scheme, and adaptive synchronization scheme, and adaptive 
scaling scheme.

B. Adaptation Strategy 
In this section, we describe the process used to map the 
LDPC decoding function to the adaptive computing fabric 
(ACF) architecture. LDPC codes are defined by a sparse binary 
MxN parity check matrix H as shown in Fig. 4, which can be 
illustrated using a Tanner graph H as shown in Fig. 4, where 
each bit is represented by a Bit Node (BN) and each parity 
check is represented by a Check Node (CN). An edge connects 
BN i and a CN j, if and only if H(j,i)=1 as shown, in Fig. 4. In 
other to minimize the error floor and the number of decoding 
iterations while supporting a flexible ACF-parallel and 
programmable LDPC decoder, we choose to use a schedule 
and decoding algorithm that is based on a cognitive scheme 
because this provides the appropriate level of adaptation 
required for our ACF design. The cognitive decoding process 
is developed to operate as follows: more than one or more 
algorithms are concurrently scheduled using the ACF interface 
processor in Fig. 2 to performed the techniques summarized in 
Table I and described in section III using the ACF data stream 
processors in Fig. 3. The decoding sequence and results are 
then adaptively evaluated at every iteration, with a maximum 
number of iterations set to 5.

III. 
ANALYSIS, DESIGN AND TEST IMPLEMENTATION 
We experimentally found as shown in Fig. 6 that the 
adaptation scheme that performs well includes the following 
four algorithms evaluated concurrently with the highest 
available quantization range (e.g. 10 bits): Algorithm 1: Offset 
Min-Sum (OMS); In this method, messages smaller in 
magnitude than β are set to zero when equation (1) is 
computed to estimate the reliability of the parity check, thereby 
removing their contributions in the next node update step. 
Algorithm 2: Residual Min-Sum (RMS); In this method, we 
extend the notion of residual defined in [8] and utilize it to 
schedule the order of message updates in which the message 
with the largest residual is updated first. Algorithm 3:  RMS + 
OMS; In this method, RMS is scheduled first followed by 
OMS. Algorithm 4: OMS + RMS. In this method, OMS is 
schedule first, followed by RMS. We called this technique 
adaptive scheduling and quantization. In order to minimize 
wire congestion when accessing shared resources, each ACF 
unit in Fig. 2 integrates a dedicated ACF data stream 
processor, which is designed to compute the required bit node 
and check node operations (e.g. 32 bit nodes x 1 check node 
per ACF data stream processor for 802.an) in one cycle 
including generating the required addresses to load and store 
messages within and outside of the ACF unit. We called this 
technique adaptive partitioning. Similarly, in order to minimize 
access conflicts during load and stores to shared resources, 
each ACF unit is instrumented to use segments of a shared 
address, a shared read data bus and a shared write data bus with 
adaptive data word width, concurrent read and read operations 
using a bus protocol that supports multiple outstanding 
transactions to enable non-blocking communication across the 
on-chip bus bridges. We called this technique adaptive 
synchronization. A set of multi-port register files in Fig. 3 
which are unique to each ACF unit is used to implement the 
required message passing protocol between bit nodes and 
check nodes using the adaptive synchronization (per the 
appropriate Tanner graph as represented in Fig. 4) across all 
variable and check nodes operating in parallel. This approach 
eliminates the need for the bit nodes and check nodes to access 
global memory, (e.g. only the ACF interface processor in Fig. 
2 has access to global memory) and the need for 
interleaving/de-interleaving (e.g. matrix transposition) while 
also significantly reducing the workload (e.g. number of 
computations) per processor that would otherwise limit the 
effective throughput and increase the chip area of the LDPC 
decoder. We called this technique adaptive scaling.

IV. 
EXPERIMENTAL RESULTS AND DISCUSSIONS 
The experimental results were achieved by VLSI design, 
SoC 
integration 
and 
implementation 
of 
the 
LDPC 
encoding/decoding function using a 65nm ASIC standard cell 
technology with CAD tool suites from Synopsys, Cadence and 
Mentor Graphics.  We evaluated decoding of the LDPC codes 
using prototyping boards including the ZedBoard, miniBEE 
and BEE4 programmed with an ACF bitstream generated by 
the Xilinx Vivado high-level synthesis (HLS) tool suite.  In the 
case of the DVB standard, the ACF bitstream generated was 
configured to operate 360 independently operated ACF units 
shown in Fig. 2 and sharing the same LDPC H Matrix. 
Similarly, the ACF bitstream is configured to operate 42 ACF 
units in the case of IEEE 802.1ad standard, and 64 ACF units 
in the case of IEEE 802.1an standard. At 100MHz the LDPC 
decoder delivers an average throughput of 201.6Gbps at 5 dB 
as shown in Table II.  The energy-efficiency per coded bit is 
reduced to 0.61pJ/bit at the 100MHz operating frequency as 
Fig. 5 shows (e.g. a 5000x improvement compare to single 
core 
compiled 
software 
implementation). 
Furthermore, 
automated functional testing was used to collect error counts 
for a range of Signal-to-Noise Ratio (SNR) levels to generate 
BER plots. The cognitive LDPC decoding algorithm described 
in section II.B is applied to adaptively accelerate the decoding 
convergence (e.g. minimize decoding latency to 5 iterations) 
while lowering the BER and error floor (e.g. higher error 
correcting performance) to the levels required by the IEEE 
standards as shown in Fig. 6. These implementation results 
compare favorably to the state-of-the-art high-throughput 
LDPC decoder implementations [2], [5], [6], [7].

V. 
CONCLUSION 
We have developed, introduced and presented for the first 
time, adaptive computing fabrics (ACF) as an architecturally 
optimized design solution for synthesizable multi-mode/multi-
rate/multi-standard LDPC decoders using cognitive algorithms. 
We experimentally demonstrate an ASIC implementation and 
FPGA prototype that achieves up to 200x throughput 
improvement and up to 5000x energy-efficiency improvement 
as compare to the same algorithm running in software 
(optimized C program) on a single CPU core. Our 
implementation of the IEEE 802.3an LDPC code using ACF 
and cognitive decoding algorithms achieves an average bit 
error rates (BER) performance of 10-14 at Eb/N0=5dB with 
lower chip area, higher data throughput, higher error correcting 
performance and higher energy-efficiency compare to related 
recent LDPC decoders implementations. This approach is 
scalable to future generation of VLSI LDPC decoders that are 
expected to be fully parallel, re-usable and programmable. Our 
results strongly suggest extending this work to other 
applications including enabling the ACF for digital signal 
processing and high-performance computing as a topic of 
future work.
