Matrix Reordering Techniques for Memory Conflict Reduction for Pipelined QC-LDPC Decoder
Zhenzhi Wu1,2, Dake Liu1,2, and Yanjun Zhang1
1School of Information and Electronics, Beijing Institute of Technology, Beijing, 100081, China 2Department of Electrical Engineering, Linko¨ping University, Linko¨ping, SE-58183, Sweden

Abstract—Layered Decoding (LD) algorithm is widely applied in high throughput QC-LDPC decoders. Among all check node update algorithms in LD, Turbo-Decoding Message-Passing (TDMP) is adopted by many proposals. A-posteriori memory access conflict under pipelined TDMP decoder incurs serious throughput decline. In this paper, several matrix reordering techniques are introduced to reduce the conflict occurrences without incurring the performance loss, which includes Row Exchange method, element Sequence Reordering method, and a conflict detector with pipeline stall insertion. They are integrated in a joint recursive deep-first searching procedure. Test results show that the efficiency improvement reaches up to 60% compared to non-optimized scenarios for 802.11n and 802.16e standards.
Index Terms—QC-LDPC, Conflict reduction, Matrix reordering.
I. INTRODUCTION
Quasi-Cyclic Low-Density Parity-Check (QC-LDPC) codes are adopted in several high speed wireless standards, including 802.11n and 802.16e. TDMP [1] and Min-Sum (MS) [2] are two Check Node Update (CNU) algorithms applied in the Layered Decoding for QC-LDPC codes. Although MS has lower computational complexity, it only accept low accuracy approximation function (Min-Sum or Offset-MinSum, Scaled-Min-Sum) which slows down the convergence speed. TDMP also called Bahl-Cocke-Jelinek-Raviv (BCJR) [3] enables the adaptation of more accurate approximation functions [4], [5], [6]. It also advantages in Turbo and LDPC duo-mode architecture [6], [7]. The drawback of TDMP is the a-posteriori message access conflict caused by the forwardbackward recursion, which results in serious write-after-read conflicts. To resolve the conflicts, matrix reordering is a low cost and efficient method, where the elements of the base matrix are rescheduled to avoid the conflict happen.
Matrix reordering techniques are investigated in [8], [9] for conventional Two Phase Message Passing (TPMP) LDPC decoding. By arranging more blank area at the lower-left part and the upper-right part of the parity check matrix, the Check Node Update (CNU) and Variable Node Update (VNU) in overlapped schedule may suffer less conflict, and therefore less pipeline stalls are required. However, these methods can not be adopted in LD decoders. For pipelined LD decoders, the conflicts between consecutive layer and inter-layer are analyzed in [10], where the matrix reordering problem is summarized as ’Traveling Salesman Problem’ and the genetic algorithm is proposed. In [11], the consecutive layer conflict

problems are formulated. The reverse output method and layer reordering method are proposed.
Based on these previous work, we propose an off-line joint matrix reordering method for BCJR based LD, which has minor hardware change while brings to dramatic throughput improvement. Three conflict reduction methods including layer exchange, elements sequence reordering, and a conflict detector are integrated in a searching algorithm. The maximum efficiency (lowest amount of NOP insertions) is taken as the criteria for finding the best optimized result.

B. Proposed Hardware Architecture
In order to verify the methods proposed, a hardware prototype is established. The decoder architecture is shown in Fig. 3. Similar architectures using BCJR decoding are discussed in [5], [6], [7], [14], [15]. X and P are obtained by the reordered B. After serialization, the read address sequence Xr = [X1T X2T · · · XRT ]T and write address sequence Xw = [F (X1)T F (X2)T · · · F (XR)T ]T are obtained. The read/write sequence and corresponding shifting value P are stored in a Look-Up Table (LUT). The non-zero sub-matrices are recursively scanned. In the forward recursion (FR), (1) and (4) are processed. The APP-LLR messages are read by the address X and cyclic-shifted with P. The results including α and λ are stored in First-In-Last-Out (FILO) buffer. After FR is finished, the backward recursion starts. In which β, Λ are calculated and then the outgoing messages L are calculated, following (5), (6), (3). Λ is stored in local buffers, and shifted L is written back at the same time. The access conflicts happen in Memory Banks where APP-LLR is stored.

In Algorithm 2, we denote the amount of NOPs as N and the minimum N as Nmin. The row index is counted from one.
It tries to exchange each following row Brn with its next row Br+1, and test whether the amount of consecutive layer conflict is less or equal to c. If satisfied, the next row would become the current row, and the recursion proceeds. When the

last row is attained and successfully verified with the first row, an admissible optimized base matrix is formed. The procedure SequenceReordering and ConflictTest are called immediately. By comparing the number of NOPs with the minimum NOPs recorded before, the newly generated B is recorded only when it has less NOPs. The recursion continues until all possible matrices are tested, and then the most minimum number of NOPs is obtained with the corresponding B generated.
When the number of rows is large, the algorithm can be accelerated by quitting ’for’ loop when succeed is obtained or the searching time limit is attained. In such occasion, only one or several possible matrices are obtained. It reaches the same throughput (NOPs) in most cases, or obtains a slightly worse result.
B. Sequence Reordering Method
Firstly we propose two metrics called ’read distance’ and ’write distance’. In Fig. 4, the common elements between current layer and adjacent four layers are demonstrated. dj,θ represents the clock cycles between the write and read operation, called distance, where θ is the offset from current row. Conflict occurs when dj,θ < 0. Because the following rows are un-arranged, the distance can not be calculated directly. However, the larger distance for all layers, the less conflicts will happen in global perspective, which is formulated as maxdj,θ, θ = [−2, −1, 1, 2], for all j ∈ [1, J ]. The maxdj,θ can be optimized by carefully rescheduling the position of common elements in table X.
For the current row r, the common elements between row r and r + 1 are required to be written as early as possible for maxdj,1, while the common elements between row r and r −1 need to be read as late as possible for maxdj,−1. Similar requirements also exist for maxdj,±2. Writing early also refers to reading lately. In conclusion, all the requirements equal to one rule: common elements need to be read as late as possible. In addition, the consecutive layers r + 1 and r − 1 should be considered firstly because they have less distance than row r + 2 and r − 2.
According to the requirement, all the conflict related elements need to be placed at the right-hand side of the current row of read sequence (Xj). The common elements between r and r ± 1 are placed firstly, followed by common elements

C. Conflict Test and NOP Insertion
The remaining conflicts need to be detected and NOPs are inserted for decoding correctly. Two NOP inserting methods are proposed: Column NOP Inserting (CNI) and Single NOP Inserting (SNI).
In CNI approach, if one row encounters conflict, one NOP cycle will be inserted in front of all the rows, which means a whole column of NOPs are added to X.
Another approach is adding a single NOP at the exact row position in read sequence Xr and write sequence Xw when conflict is occurred in this row. NOPs are added in front of the row border in Xr, and added to the end of the row border in Xw. It performs best efficiency with least NOP cost.
Although SNI performs better or the same as CNI, we still take interest in CNI because it has no hardware modification, except that the base matrix LUT is filled with more NOP columns. It can also be implemented by a control flow which is able to stall several clock cycles for each row. Compared to SNI, which needs a table to distinguish different NOP value among different layers, it is easier to realize. For the study purpose, both of the methods are investigated.
A conflict test and NOP inserting program called ConflictTest is developed which returns the minimum inserted NOPs needed to make sure of the pipeline be conflict-free. The

IV. CASE STUDY FOR 802.11N AND 802.16E
The base matrices of all the modes in 802.11n and 802.16e are tested as a case study, assuming that the hardware can process z layers belonging to one layer in parallel. The maximum z is 81 in 802.11n and 96 in 802.16e. In simulation results, curve ’original’ refers to the results using the original base matrix without optimization, while ’RE’ refers to the Row Exchange method and ’SE’ the Sequence Reordering method. 802.11n contains 12 different base matrices for the 12 modes. In 802.16e, 19 modes share the same base matrix, and there are six base matrices providing different coding rates. ’Mode’ here represents for the different coding rates without distinguishing the 19 modes. All the following results provided are under the assumption of = 4. If hardware read/write pipeline latency is larger than 4, extra latency will contribute more NOPs added to N0, which means N = N0 + − 4. An exception is the conflict free occasions, which may still be conflict free.

V. CONCLUSION
In this paper, a joint low complexity searching technique for QC-LDPC base matrix reordering is proposed. Reordering techniques including RE and SE contribute to BCJR pipeline efficiency improvement dramatically. The proposed method requires minor hardware modification and negligible on-the-fly computational complexity.
