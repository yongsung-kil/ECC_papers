Abstract—In this paper, a simplified partially parallel decoder architectural based on field programmable gate array (FPGA) devices for Digital Video Broadcasting-Satellite 2 low-density parity-check(DVB-S2-LDPC) codes has been presented. We introduce the current research status about decoder design for LDPC code and described the motivation of this paper in the first part. Then, we present the simulating results on parameter selection and architectural design.The primary contribution of this paper is summarized as follows: firstly, we proposed a novel and more efficient barrel shifter design. secondly, we optimized sub-modules architecture including: Bit Node Update , Check Node Update and combined and flexible data storage strategy so that this decoder can achieve both high data throughput and low resource-on-chip consumption.
Index Terms—DVB-S2, LDPC decoder, FPGA architecture,
I. INTRODUCTION
Low Density Parity Check codes, firstly proposed in Gallager’s doctoral thesis[1] in the middle of last century, are linear block codes defined by sparse parity check matrix. Mackay and Neal gave these codes new life in the year 1999[2]. LDPC codes perform as good as Turbo codes, both of which show the potential closeness to Shannon bound. In many cases, if the code length is long enough and the check matrix owns proper form, the performance for LDPC codes can triumph Turbo’s. Tanner introduced bipartite graph description to indicate the structure of check matrix. Besides the outstanding error correcting performance, Belief Propagation (BP) decoding algorithm, as proved the optimal decoding strategy, proceeds in low-complexity which could not enhance with the code length. Researchers and engineers pay great attention on LDPC codes in recent decades due to the excellent bit error rate (BER) performance and low decoding complexity.
LDPC codes are exclusively defined by sparse check matrix H with few ’1’elements in it. The earliest structural approach is Gallager method adopted to generate regular random LDPC code. It could be relatively simple to create a check matrix according to the sub-matrix set by the row weight and the column weight. However, there are three disadvantages for this method. Firstly, short girth, severely affected BER performance, possibly appear in check matrix considering code length is long enough. Secondly, check matrix generated by Gallager method is irreversible, therefore, it is impossible to

transform check matrix into generate matrix. Finally, the fact ’1’ elements irregularly distributed in the check matrix leads to extremely high decoding complexity. The check matrix of Irregular Repeated Accumulate(IRA) codes adopted in DVBS2 system, on the other hand, formed by a group of quasicyclic matrices generated by one fundamental matrix cyclic shifting under different shift degrees. Algebraic structure of DVB-S2 LDPC codes are QC-LDPC codes being prone to decode in parallel which makes very large scale integration implementation and parallel decoding possible. Comparing serial decoding, full-parallel decoding architecture significantly improves the efficiency of the system, but this architecture demands high-level hardware platform. More balanced partially parallel decoding structural design is more urgent.
Research about LDPC decoding algorithm or decoder implementation start from the following aspect: Combine LDPC decoder with TDMP scheme, which could achieve both high decoding performance and low complexity, has been proposed in [5]. H.Jin,et al presented the improved BP decoding algorithm based on Generalized Distributive Law(GDL) [6]. The compensate BP decoding algorithm could better balanced the decoding complexity and performance of APSK modulated LDPC codes [7].The simplified DVB-S2 LDPC decoding algorithm could orderly decode or prior detect partial block so that the iteration and calculation amount can be reduced [8]. Sub-module parallel largely reduced the consumption of chip resource [9]. Parallel processing the fly-part among parity check bits could enhance the data throughput [10].

B. DVB-S2 LDPC Decoding
Since the check matrix could be described as Tanner graph, Probability Belief Propagation decoding procedure has been represented to process by rows and by columns. BP algorithm utilizes the probability to decode, in which the information transmitting between bit nodes and check nodes including not only hard decision results but also reliability of the results. If describe the probability as likelihood ratio, multiply operation could be replaced by addition operation, thereby, the calculation amount could be largely reduced. However, Log Likelihood Ratio Belief Propagation (LLR-BP)[3] decoding algorithm calculates the tangent and arctangent values in every row processing step, which is not practical for hardware implementation. Min-Sum (MS)[3] decoding algorithm could largely reduce the time consumption, which is at the expense of decoding performance, as the min value and the sign value need to be calculated only in the row processing step. In order to compensate the performance loss of MS algorithm, varieties of MS decoding algorithm have been proposed: NormalizedMS (NMS) algorithm and Offset-MS (OMS) algorithm[4], whose decoding complexities are almost identical with MS algorithm and decoding performances are close to BP algorithm.

IV. CONCLUSION
Experimental results we introduced in Decoding scheme simulation declare factors influencing the decoding performance in hardware implementation. The decoder design based on FPGA we propose in this paper could save the resource and enhance the data throughput from the following three aspects: firstly, combined LLR information and iterative results storage strategy; secondly,simplified Bit Node Update and Check Node Update module design; thirdly, novel method in barrel shifter design.
