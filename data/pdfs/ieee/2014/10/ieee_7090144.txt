Abstract – In this paper, the design of the LDPC decoder architecture in IEEE 802.11n/ac is proposed. The proposed decoder is used to the partial parallel architecture to provide 2 Gbps throughput at 200 MHz clock frequency. In IEEE 802.11n/ac, 12 parity check metrics is provide to support diverse code rates and block lengths. Therefore the network between nodes is configured to be satisfied parity check metrics. However, the structure of the network increases the complexity of LDPC decoder. Hence, the optimized network generator is also proposed to reduce the complexity of the network.
Keywords- LDPC decoder; optimized network; IEEE 802.11n/ac; quasi-cyclic low-density parity-check code
I. INTRODUCTION
Low density parity check (LDPC) codes, which is linear block code, have been proposed inceptively by Gallager in 1960s. In 1990s, Mackey and Neal have rediscovered the algorithm, and then many studies have been progressed [1]-[2]. Recently, LDPC codes are adapted to communication standard as Wi-Fi and WiMAX because of Shannon limit error performance [3]-[4].
The belief propagation (BP) algorithm is a message passing algorithm that is widely used in LDPC decoder algorithm. The sum-product algorithm (SPA) also known as BP algorithm has the best performance, however, it has the high computational complexity at the operation of the check node due to logarithmic and multiplicative computations. To reduce the complexity, the min-sum algorithm (MSA) is proposed. The algorithm is used to find the 1st minimum value and the 2nd minimum value instead of the calculation in SPA at the operation of the check nodes so that the complexity is lower than SPA. However the performance of the MSA is observed to degrade in comparison the SPA. The modified MSA such as offset-MS, normalization-MS algorithm (NMS) is proposed to improve the MSA’s performance, which is enhanced as the BP algorithm [4]-[6].
The LDPC decoder architecture also has a tradeoff between the complexity and the performance. According to the trade-off, the architecture is categorized as a full parallel architecture, a partial parallel architecture and a serial architecture. The full

The partial parallel architecture should be considered the trade-off between performance and complexity suitable for the target system [5]-[12].
In this paper, the design of the LDPC decoder architecture using the NMS algorithm is proposed. The network between variable nodes and check nodes constitute high complexity in the LDPC decoder, particularly. Because of the differential sub-block size or code rate of IEEE 802.11n/ac, the LDPC decoder should have several parity check metrics. Hence the network has a high complexity to support parity check metrics. To reduce the complexity of the network, therefore, the optimized network generator is also proposed in the partial parallel decoder architecture of IEEE 802.11n/ac.

III. DECODER ARCHITECTURE
Figure 1. Overall architecture of LDPC decoder.

The proposed LDPC decoder (Fig. 1) in IEEE 802.11n/ac is to use the partial parallel architecture. The architecture consists of an input/output controller and a core controller. The input controller rearranges the input data for LDPC codeword block length as shown in Fig. 2. The serial data, which entered from the forward block, is restored on a first-in-first-out (FIFO) buffer to a codeword.

The CNU is the operator of the check node unit with finding the minimum value and calculating an exclusive-OR because of the NMS algorithm. The operator of the variable node (VNU) computes the summation of the minimum values which is connected to variable node. The network unit (NETU) is the data path between check nodes and variable nodes defined by the parity check matrix of IEEE 802.11n/ac.

In IEEE 802.11n/ac LDPC codes, 12 parity check matrices are used to support various code rates and block lengths that are comprised of sub-matrices. Thus, the network between variable nodes and check nodes depends on the parity check matrix that should be connected to all paths of the maximum sub-matrix. This structure significantly increases the complexity of LDPC decoder.
In this paper, therefore, the generator of the optimized network between nodes is proposed. The method to generate the path is as follows: First of all, 12 parity check metrics divide into the number of pipeline. Then, the enable matrix (EM) generate in 3dimension consist of column of H × (row of H/pipeline) × (12 × pipeline) using divided parity check metrics. At this time, the zero-matrix does not save in EM. If, also, the value is equal at the point (H col, H row), it restore once. Finally, the network generates by the EM, and the network can reduced complexity compared to previous network.
IV. CONCLUSIONS
In this paper, the LDPC decoder architecture in IEEE 802.11n/ac has been presented. The decoder architecture utilizes a partial parallel architecture using a NMS algorithm. The generator of the optimized network between nodes has also been proposed. The optimized network is reduced by 86% as compared with the overall connected network.
