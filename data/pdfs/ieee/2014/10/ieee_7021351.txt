ABSTRACT Low density parity check (LOPC) code is an error
correct code that can achieve performance close to Shannon limit and it is widely adopted in communication systems such as WiMAX and Wi-Fi. This paper presents a design of a flexible LDPC decoder for WiMAX based on the NoC platform. The implementation adopts layered decoding algorithm and exploits high parallelism that lies in the algorithm. Considering large quantity of data exchange between cores, we use stepped data exchange strategy and master-slave structure to reduce communication cost. The Cycle accurate simulation results show high speed-up and scalability of code rate of 5/6 with different code length in WiMAX in 4/8/16 processor systems.
INTRODUCTION Low-density parity-check (LOPC) code featuring a
sparse parity-check matrix, was firstly invented by Gallager in 1962. It is proven to have a performance close to Shannon limit and it is widely used in communication standards,such as OVB-S2,Wi-Fi and WiMAX.
Many LOPC decoder implementations have been proposed ranging from serial to full parallel [1][2] and most of them are ASICs. The ASIC implementation may be efficient in performance and area,but it lacks flexibility to support various LOPC codes,which is required in some applications such as WiMAX.
In the recent years, multiprocessor system is proved to be both performance and energy efficient [3]. And we proposed a scalable,flexible approach for LDPC decoder in distributed processor based on the NoC interconnecting. We choose offset min-sum algorithm for the balance of performance and computation cost. The main decoding process are two phase message propagation decoding [4] (also called TPMP) and layered scheduling [5] (also called TOMP),which is more suitable for application of WiMAX according to [6]. So we choose TOMP decoding process in multiprocessor platform. We exploit high parallelism in the decoding process and adopt stepped data exchange strategy to eliminate congestion and master-slave structure to reduce communication. A LDPC code of WiMAX with different code length are implemented and evaluated in 4/8116 processor systems and the results show high speed up and scalability of our approach.
The rest of the paper is organized as follows: Section 2 describes the multiprocessor platform used for

evaluation. Section 3 introduces offset min-sum algorithm and software implementation in multiprocessor platform. The experiment results are shown in Section 4. Finally,we summarize our conclusion in Section 5.

depicted in Figure 1. The platform is comprised of 4x4
processing elements (PE). Each PE consists of a processor core, which is an OpenRISC core with a local memory, and a router. PEs are interconnected by a two-dimensional mesh network, which provides inter-PEs communication capabilities with high data exchange throughput by allowing simultaneous communication among PEs. The router uses X-Y routing algorithm for the balance of complexity and latency of communication. Thop, which is the cycles needed for data transfer from a PE to adjacent PE, can be configured to represent the network communication qualification.

TASK SCHEDULING The main section of the algorithm 1 is the 2nd step,
which is the process of row-by-row updating, 3rd step, which updates the result and 4th step, which checks the result and decides whether to begin next iteration. The decoding of the different LOPC code of WiMAX is similar and the main differences are row updating times in the 2nd step and column updating times in the yd step, which means by configuring some parameters, it is easy to implement different LOPC code with different length,thus achieving flexibility.
The 2nd is the process of updating row after row. However, the generation way of H by sub-matrix reveals that the column location that H mn=1 of rows in H that expanded from the same row of sub-matrix is different pair-by-pair, which means these rows can update in parallel. So given the processor number N, we update N rows simultaneously to speed up the 2nd step.
We parallel the 3rd step by dividing all column of y into N parts, the PEO update lowest part of y, the PEl update second lowest part of y and so on. In the 3rd step, the whole check process is to compute the vector product of each row in H and y and check whether the modular 2 of the result is zero. We check the vector product of N row in H and y to parallel the check process.

OPTIMIZED COMMUNICATION Due to the high parallelism of the decoding algorithm
1, we expect a high speed up in the multiprocessor platform, however, the quantity of data exchange among PEs is also large. In the 2nd step,each process of updating row inputs L and updates the new L,which will be used as inputs for some other rows. That means each PE must send its computed L to every other PEs and receive L from others PEs as well. Similarly, in the 3rd step, after computation the respective part of the y, each PE must send its part of y to every other PEs and receive other y for further row check in the 4th step. In the 4th step, each PE checks different row in H and if one of PEs finds that the result is wrong,there is no need to continue next row check, so each PE will get the information from both itself and other PEs to decide whether to do next row check.
We proposed stepped data exchange method in communication 3rd and 4th step of algorithm l. In this method, we regard the adjacent 4 PEs as a cluster. The stepped method is shown as follows:
Step I: we divide 16 PEs into 4 parts, each part corresponds to a cluster, each PE exchanges data among PEs that in the same cluster.
After step 1,PEs in the same cluster share all data. Step 2: 16 PEs is divided into 2 parts, which correspond to 2 clusters. In each part,PEs in the interface of the clusters exchange data and transfer data to inner PEs. After step 2,PEs in the same part share all data. Step 3: PEs in the interface of the 2 parts exchange data and further transfer data to inner PEs. So after step 3, all 16 PEs get data from other PEs, The whole communication step on 16 PEs is shown in

It can be easily seen that this communication method can avoid network congestion. Moreover, the sent data into network will be received immediately so FIFO depth in the network can be lower to contain all these data.
In the 4th step in algorithm 1, we adopt master-slave structure. In this structure, we choose a PE at the near­ middle location as master PE and other PEs as slave ones. After each check row process finishing, every slave PE sends its result to master PE and master PE process both received result and the result computed by itself, then master PE send data to slave PEs, which contains the message of whether to go on next check row process. By this way,the communication cost can reduce from O(N2) to O(2N).
