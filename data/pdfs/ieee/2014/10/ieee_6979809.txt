A Construction of LDPC Convolutional Codes with Close Distance Bounds

AbstractŌĆöA trellis code constructed by concatenating a delay processor and a signal mapper to a convolutional code with a short constraint length can be regarded as a convolutional code with a large constraint length. We can obtain a close upper bound and a close lower bound for such a code. In this paper, we use this code structure to construct low-density parity check convolutional codes which are 4-cycles free and hence can be efficiently decoded by the iterative message-passing algorithm.
Index TermsŌĆö LDPC Convolutional Codes, LDPC-CCs, Trellis Code, Delay Processor
I. INTRODUCTION
Low-density parity-check (LDPC) codes which are block codes were first proposed by Gallager [1] in 1960s and were rediscovered by Mackay and Neal [2] in 1996. The LDPC code has the feature that it can be represented by with Tanner graph [3]. LDPC codes with very long block lengths can achieve the near-Shannon-limit performance through iterative massagepassing decoding [4]. Quasi-cyclic LDPC (QC-LDPC) codes [6] [7] whose parity-check matrices are composed of permutation matrices, can be easily encoded with simple feedback shift-registers. The algebraic structure of QC-LDPC codes also allows an efficient high-speed very-large-scale integration (VLSI) implementation. Recently, a trellis counter part of the LDPC code, called the LDPC convolutional codes (LDPCCCs), is proposed in [15] [16]. Like the LDPC, the LDPC-CCs can also be represented by a Tanner graph with semi-infinite size which has a periodic structure. Hence, the LDPC-CCs can be decoded by a pipeline decoder which is a messagepassing algorithm with a continuous way of iteration. To our knowledge, there is still a significant room for studying the constructions of LDPC-CCs with close upper or lower bounds [15] [16] [17]. In this paper, a construction of LDPCCCs for which close lower bounds and upper bounds on free distance will be proposed. The proposed construction has the form of a trellis code with a delay processor and signal mapper (TCDPSM) [10] [11]. TCDPSM is constructed by concatenating a convolutional code with a small constraint length with a delay processor and signal mapper. The resultant TCDPSM is a convolutional code with a large constraint length. The large constraint length of the TCDPSM prohibits the feasibility of designing its optimum decoder. In [10] [11],

a sub-optimal decoder based on the multilevel structure of TCDPSM and the BCJR algorithm [13] is used. The BCJR calculates the soft-information for the convolutional code in a sliding window way, and then the soft information is passed to the demapper through a delay procedure so as to obtain updated soft-information for bits in lower levels of the multilevel structure. The updated soft-information was sent back to BCJR to be used by later bits. Only one iteration that means the the soft information of each code bit is updated on once is used in this sub-optimal BCJR decoder. Hence, there is room for improving the error performance by considering the decoders employing more iterations for each code bit such as the pipeline decoder [16] used in the conventional LDPCCCs. To this end, we have to derive a low density sparse polynomial parity-check matrix from polynomial generator matrix of TCDPSM. Special effort has been done to avoid the occurrence of undesired 4-cycles phenomenon. From computer simulations, better error performances obtained by the LDPC-CCs constructed from TCDPSM can outperform those obtained by some previously constructed codes and LDPC block Codes (LDPC-BCs) by considering the same window lengths or block lengths.
The remainder of this paper is organized as follows. In Section II, basics of LDPC-CCs and TCDPSM are described. Section III presents a new construction of LDPC-CCs, for which both of the upper and lower bounds on free distance are derived. Simulation results are shown in Section IV for performance verification. Section V concludes the paper.

V. CONCLUSIONS
We have proposed a new construction of LDPC-CCs based on the structure of the trellis code with a delay processor and signal mapper (TCDPSM). For the constructed LDPC-CCs, close lower and upper bounds on free distance can be obtained. The undesired 4-cycles in the Tanner graph for the proposed LDPC-CCs comes from the inherited property of TCDPSM. Special measures have to be employed to remove the undesired 4-cycles. Simulation results show that the newly constructed codes can obtain satisfactory error performances as compared to some previously known LDPC-BCs of comparable lengths.
