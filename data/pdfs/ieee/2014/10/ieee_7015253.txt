DUAL PORT RAM BASED LAYERED DECODING FOR MULTI RATE QUASI-CYCLIC LDPC CODES
A CASE STUDY: IEEE 802.16E WIMAX LDPC CODES
Soner Yes¸il, Murat Arslan
Communication & Information Technologies Division Aselsan Inc., Ankara, Turkey
syesil@aselsan.com.tr, muarslan@aselsan.com.tr

ABSTRACT
This paper presents a generic RAM based FPGA architecture for decoding of Multi Rate Quasi-Cycling LDPC codes. RAM based decoding enables us to reduce permutation networks into simple address controllers. Moreover, utilizing Block RAMs with various aspect ratios in an FPGA provides flexibility ranging from area driven compact designs to fully parallelized high throughput designs. Utilizing the read-first property of the RAMs, the proposed design efficiently exploits the dual port Block RAM resources by accessing all the four ports at the same time. Such facilities of recent FPGA devices have been combined with the well known layered decoding algorithm with non-linearly mapped Min-Sum approximation in order to obtain area efficient yet high throughput decoders. The proposed decoder architecture has been verified on Xilinx XC7Z020 FPGA device for IEEE 802.16e Wimax LDPC codes. 340Mbps of information throughput has been observed at an operating frequency of 150MHz.
Index Terms— LDPC, Quasi-Cyclic, Wimax, Layered Decoding, Min-Sum
1. INTRODUCTION
Low Density Parity Check Codes, first introduced by Gallager [1] in 1962 as a forward error correction (FEC) technique having very good bit error rate (BER) performance near Shannon-Limit, had to wait for a long time to take place in real-life applications due to its complex nature to implement in hardware. In parallel with the developments in the VLSI technology in the last quarter of 20th century, these codes were re-invented in the late 90s [2] and become popular for various high-throughput, low power, and reliable communication systems. Increasing demand on band-width and energy efficiency together with robustness in the presence of various channel impairments and minimum possible computation resources has also increased the research on construction of new implementation-friendly multi-rate LDPC codes with least possible degradation in BER performance. As a result, Quasi-Cyclic (QC) LDPC codes have been adopted as

the FEC technique in various well-known communications standards such as WLAN, WIMAX, and DVB-S2. QC-LDPC codes are specifically tailored to give way to efficient encoder and decoder implementations. Even so, realization of highthroughput QC-LDPC decoders with tight area requirements still remains a challenging problem. There are hundreds of various ASIC and FPGA implementations for various types of QC-LDPC Codes in the literature. Although the algorithms and approximation techniques are quite similar, the proposed architectures show much variety in each implementation. ASIC implementations [3, 4, 5, 6, 7, 8, 9] mainly focus on squeezing highly parallelized decoders into smaller silicon areas, whereas FPGA implementations [10, 11, 12, 13, 14, 15, 16, 17] focus on efficient utilization of the inherent resources. Therefore, the challenge of optimizing large permutation networks in ASIC designs has mostly turned into designing dynamic RAM-based networks in FPGA designs. There have been some fully-parallel and logic-based FPGA decoder implementations targeted for a specific code type [18, 19] that go beyond multi-Gbps throughputs. However, partially parallel designs aiming moderate throughputs and supporting multiple coding rates are likely to be more practical in concurrent system realizations, where FPGAs are expected to support extra duties ranging from physical layer processing to higher level control tasks.
In this work, we propose a generic and flexible RAMbased FPGA architecture for decoding of QC-LDPC Codes. The rest of the paper continues by giving a brief description of the QC-LDPC codes. Then the Turbo Decoding Message Passing (TDMP) Algorithm [20] (also known as Layered Decoding Algorithm), together with some approximations and implementation-oriented modifications is introduced in the following subsections. Section 3 presents the detailed explanation of the proposed Dual-Port RAM-based architecture and the techniques used in Check Node Processing (CNP) and Early Termination decision. An example implementation of IEEE 802.16e WIMAX LDPC Decoder based-on the proposed architecture is explained and compared to similar FPGA implementations within our knowledge. Finally, a

2. DECODING OF QUASI-CYCLIC LDPC CODES

LDPC codes with unstructured Parity Check Matrices (H) having randomly placed non-zero entries give the best performance among all. However these codes by nature are very difficult to implement in hardware. Therefore, research on code construction has been focused on finding implementable structured codes keeping the performance as good as possible. Reference [21] presents a good example showing how the structured LDPC codes can reveal significant reductions in hardware with a slight loss in the BER performance. QuasiCyclic (QC) LDPC codes are considered to be the most popular of such structured codes. In a QC-LDPC code, the HMatrix consists of z-by-z square sub-matrices, Pi,j, each of which is either zero or circularly shifted identity matrix as in (1). The circular-shift property makes it possible to simplify the troublesome permutation networks by reducing these into shift register or address counter based networks, which is the underlying reason for the popularity of these codes.

2.1. Layered Decoding Algorithm (LDA)
Layered Decoding Algorithm separates the Parity Check Matrix into its block rows, which are called layers. The HMatrix in (1) has mb layers, each having z rows and z · nb columns. Fig.1 depicts the Tanner Graph [22] representation of the variable and check nodes of a layer. The upwards and downwards arrows in this graph represent the variable and check node messages, respectively. Since each Pi,j is a circular shift of a z-by-z identity matrix, it is guaranteed that every variable node has at most one branch connected to the check node group of the corresponding layer. Different from the traditional Belief Propagation Algorithm (BPA), the variable node is now simply updated by subtracting the previous iterations check node message from the current Log Likelihood Ratio (LLR) information. This, however, means that every check node message occurring in the current iteration needs to be stored and used in the next iteration. A pseudo-code for the layered decoding is shown in Algorithm (1), where f (.) denotes a given check node update function, the use and choice of which does not depend on the application of the LDA.

2.2. Min-Sum Algorithm (MSA)

Min-Sum Algorithm, which is an approximation of the original Sum Product Algorithm (SPA), has been widely preferred in many implementations because of its hardware efficiency. The performance analysis of this approximation together with several modifications is presented in [23] showing that the MSA introduces very little penalty of loss in the BER performance. According to this algorithm, every check node outputs the absolute minimum of the input messages other than the one coming from the target variable node. Similarly, the sign of every output message is simply obtained by multiplying the signs of the input messages excluding the target variable node. 3. RAM-BASED LAYERED DECODING

3.1. General Architecture
The main challenge for the hardware implementation of a multi-mode QC-LDPC decoder is to design a routing network serving for all modes of operations [26, 27]. Due to the limited resources, recent FPGA designs tend to handle this problem by efficient utilization of the embedded RAM capabilities [10, 11, 12, 13, 14, 15, 16]. With this approach, a highly logicconsuming routing network is replaced by simple address management logic blocks. Moreover, the decoding architecture can be configured on-the-fly and adapted to different Hmatrices of various modes of operation. In addition, thanks to the flexible configuration capability and increasing RAM capacity in the recent FPGA devices, it is possible to meet extended degree of parallelization by exploiting different aspect ratios [11, 12]. Fig. 2 depicts the general view of the proposed RAM-based architecture with a parallelization degree of p. A main controller block runs a finite state machine to provide a higher-level control for the input-output processing and re-configuring the RAM controller blocks according to a specific mode of operation. The architecture includes nb identical RAM controller units, each of which is responsible for a single block column of the H-Matrix. The Check Node Processing units (CNPU) gather the variable node messages from the RAM controller units and return a message to each unit in parallel. With this architecture, full parallelization and hence maximum throughput can be achieved by exploiting zCNPUs in parallel. However, one must keep in mind that parallelization results in an increase in the resource utilization in

3.2. Dual-port Block RAM Address Management
The proposed RAM Controller Unit (Figure 3) consists of a true dual-port RAM, which enables accessing two separate ports at the same time. In addition, the “Read-First” property of the FPGA Block-RAMs makes it possible to both read and write on the same port at a single clock cycle, eliminating the need for clocking the RAMs at twice the clock frequency as in [12]. Therefore, every Block RAM Controller Unit becomes capable of processing all the steps of the Layered Decoding Algorithm at a single clock cycle after some fixed pipelining latency. This latency is caused by the register delays occurring in between the read and write ports of the RAM, including the CNPU. The number of registers in this path can be increased as required in order to obtain higher clock frequencies. This latency entails a circular shift in the re-organization of the LLRs to be stored and used in the next layer. However, this is a rather easy task for the address controller, which only requires starting the next layer with an offset. The LLRs and the CN messages are isolated in the address space in such a way that the Port-A is used to circularly store the updated LLRs layer by layer, whereas the Port-B is used to continuously store the CN updates during the whole iteration (Fig. 4). Further reduction in the number of clock cycles can be achieved by exploiting some degree of parallelization with multiple CNPUs. Depending on the desired level of parallelization, the physical RAM locations, which can be configured in various aspect ratios, can be shared by multiple variable node messages. Every variable node message would then be separately handled and directed to the corresponding CNPU. A data alignment process would be needed to bring the messages together and then deliver them towards the target CNPUs. This would be a simple multiplexor, the select pins of which are controlled according to the corresponding circular-shift value of the H-matrix. But as the degree of parallelization increases, extra resources introduced due to data alignment logic may become important. Therefore a trade-off occurs between throughput and FPGA resources. Another point that must be taken into account is that the degree of parallelization needs to be optimized for all modes of operation. Otherwise, extra logic resources would

3.4. Early Termination
Early termination decision, which stops the decoding before reaching the maximum number of iterations, is taken by checking the condition in step 4 of the LDA (Algorithm 1). Although written as a separate step in the algorithm, it is possible to do this check inherently during LLR updating process. One can easily obtain the required information to stop the decoding by accumulatively XOR’ing the sign bits of the LLRs of parallel columns throughout the whole iteration just before writing them to the RAM. The decoding is terminated if and only if the resulting XOR value is always a zero for all the rows of H-matrix. The decision is made by the main controller block which observes the sign bits coming from the identical BRAM controller blocks. Hence, with this method, the early termination condition is checked in parallel without a need for extra time.

3.3. Check Node Processing Unit
The Check Node Unit consists of a sub-block that computes the smallest two magnitudes (min1 and min2) among the inputs. Then a decision circuit prepares each reply message as in (2) by using these two minimum values and the sign of the inputs. This sub-block is based on a tree-structure composed of inner blocks computing the least two minima among four inputs [28]. Once the least two minima are found, the decision circuit sends min1 attached with the sign-bits to all the branches except for the one having min1 as the input. As for that branch min2 is sent, instead. The sign of each branch is simply determined by XOR’ing all the sign bits of the branches excluding the target branch. Regardless of the H-matrix, the proposed CNPU takes nb input messages and computes a reply for each. The input messages corresponding to the all-zero sub-matrices of each layer are set to their maximum possible values in order not to affect the check node processing. A desired level of pipelining can be exploited

The proposed architecture is implemented for the IEEE 802.16e Wimax LDPC codes. In this system, two parameters, rate (R) and expansion factor (z), together with a base model matrix for each rate, are used to define a code type among 114 different code types. There are six different rates (1/2, 2/3A, 2/3B, 3/4A, 3/4B, and 5/6) and 19 different z values obtained by z = 24 + 4f , f ∈ {0, 1, 2, ..., 18}. For any code type, the number of block columns, nb, is a constant equal to 24 and the number of block rows, mb, is obtained by mb = nb(1 − R) [29].
4.1. Implementation and Test Details
A multi-rate decoder for IEEE 802.16e LDPC codes has been implemented according to the proposed architecture. The implementation was held for two different parallelization levels of p=1 and p=4 with a latency of L=6. The check node processing unit based on 4-bit mapping of the messages was designed to operate in one clock cycle, whereas the other levels of pipelining registers took place in the data alignment, summation and mapping operations of the RAM controller blocks. The design has been verified on Xilinx XC7Z020 FPGA located on the Zedboard Kit at an operating clock frequency of 150MHz. An AWGN FPGA IP Core previously verified for un-coded BPSK BER performance, has been used as the noise generator. A TCL script running on a PC has been used to change the test parameters (information bits, noise gain, code type, base matrix, expansion factor) and to read the test results (BER, number of packets, iteration statistics, and throughput data). 5. CONCLUSION
A generic RAM-based decoder architecture based on the Layered Decoding Algorithm with a mapped Min-Sum Algorithm has been proposed for multi-rate QC-LDPC codes. The architecture eliminates the low throughput disadvantage of the RAM-based designs by exploiting the read-first property of the embedded dual-port BRAMs, by which read/write access at a single port can be done at the same time.
