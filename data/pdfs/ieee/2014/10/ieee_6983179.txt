Abstract—The design and optimization of low coding-rate channel codes is a hot topic for frequency-hopping spreadspectrum (FH-SS) transmission systems over harsh wireless channels. In this paper, we use extrinsic mutual information transfer (EXIT) chart to optimize a class of generalized lowdensity parity-check (GLDPC) codes over partial-band jamming (PBJ) transmission environment. With the proposed method, the channel capacity of the PBJ channel is illustrated, and based on which three code optimization methods are presented for achieving near-capacity performance. Simulation results show that, the GLDPC codes designed with the proposed method can achieve an information rate of 91.2% of the channel capacity, over the PBJ channel with 50% frequency band jammed and the channel signal-to-noise ratio (SNR) of the unjammed frequency band equals to -8 dB.
I. INTRODUCTION
Frequency-hopping spread-spectrum (FH-SS) technique is widely employed to combat the interference in wireless communication. In a FH-SS system, the carrier frequency of the modulated symbol is hopped in the chosen bandwidth. While over partial-band jamming (PBJ) channel, severe transmission errors occur and the channel codes are needed to recover the transmitted information. It has been shown that the design and application of channel codes with powerful error-correcting capability are crucial to the system performance [1]-[3].
Low-density parity-check (LDPC) codes attract most interests in recent decades due to their capacity-achieving performance [4]-[6]. However in PBJ channel, when most of the bandwidth has been interfered, the LDPC codes with high performance becomes difficult to design and hardware implementation due to their complicated parity-check matrix and the huge number of decoding iterations.
The Generalized LDPC (GLDPC) codes are constructed by replacing the check nodes of LDPC codes with more powerful block codes other than single parity-check codes [7]-[8]. The GLDPC codes have many potential advantages, including improved error-correcting performance in low signal-to-noise

We can know that P represents the probability that the hopped carrier frequency of the transmitted signal runs into the jammed band. The AWGN has the two-sided power spectral density (PSD) of N0/2 over WSS, which can be converted to SNR applied on the transmitted signal, and the APBGN has the two-sided PSD of NJ /2 over the interfered bandwidth, which is determined by the jammer.
The block diagram of a channel coding BPSK FH-SS system is shown in Fig.2 [10]-[11]. At the transmitter, every codeword of the channel code is modulated by the BPSK modulator, followed by a frequency hopper. The SNR of the unjammed frequency band is defined as SNR=Ebi/N0, where Ebi is the information bit energy. During a hop, several symbols are transmitted. The interval of time that a hop lasts is known as a dwell interval, and all symbols in a dwell interval have the same carrier frequency, which is determined by the hopping pattern. It is assumed that the interference will exist the whole interval when the carrier frequency band has been jammed.

IV. THE GLDPC CODE

The GLDPC code which is utilized in this paper is defined
based on the bipartite graph as shown in Fig.4 [12]. As we know the single parity-check node with degree-i in the LDPC code corresponds to an (i, i − 1) parity-check code constraint, while the general check node with degree-i in the GLDPC code corresponds to an (ni, i − 1) code constraint, and one bit of the generated block code combined with the i − 1 information bits defines a parity-check code constraint. Similarly to the variable node with degree-j in the LDPC code, the LDPC variable node with degree-j in the GLDPC code corresponds to a (j, 1) repetition code. Besides, there
are general variable nodes with degree-1 which are connected
to only one general check node. According to the description
mentioned above, it is known that the underlying parity-check
node constraint exists in the general check node, and it is
deduced that the LDPC variable nodes construct an LDPC
code.
In the following of this paper, we take one kind of GLDPC
code named LDPC-Hadamard code for example to illustrate
the optimization process. The LDPC-Hadamard code is de-
fined as replacing the single parity-check code constraint of
the LDPC code by Hadamard code constraint [12]. Each Hadamard check node with degree i corresponds to a (2i−2, i− 1) Hadamard code. We denote the i − 1 information bits as b(0), b(1), . . ., b(i − 2), and the 2i−2 Hadamard codeword as c(0), c(1), . . ., c(2i−2 −1). In systematic encoding of a length2i−2 Hadamard code, the positions of the information bits are indicated by indexes {0,1,2,4,. . .,2i−3}, then c(0) = b(0) and c(2t−1) = b(t), t = 1, 2, . . . , i − 2 .The other bits of the
codeword are parity bits.

VI. SIMULATION RESULTS
Table I gives the optimized results over PBJ channel based on the EXIT chart. The SNR of the AWGN is -8 dB and the frequency-band jammed probability is P = 50%. According to Section III, the channel capacity under such channel condition is 0.09907, which means the upmost information rate can’t exceed 0.09907 for arbitrarily small error probability. It is shown that we can achieve 91.2% of the channel capacity with the first method, 86.8% with the second method, and 89.0% with the third method.
Fig.9 gives an example of the proposed optimization method, which shows the EXIT chart of the original degree distribution and the optimized result with the first method according to Table.I. With our method, the information rate can increase from 0.0833 to 0.0904 over the PBJ channel with 50% frequency bands jammed and the channel SNR of the unjammed frequency band equals to -8 dB. From the figure, it is know that a far enough tunnel exists between the EXIT curve of original variable node degree and the curve of Hadamard check node at the left part, while with the proposed method, the variable node curve matches well with the Hadamard check node curve.
