Symbol-Level Synchronization Using Probability Lookup-Table for IDS Error Correction

Haruhiko Kaneko Dept. Computer Science, Graduate School of Information Science and Engineering
Tokyo Institute of Technology Meguro-ku, Tokyo 152вҖ“8552 Email: hkaneko@fuji.cs.titech.ac.jp

AbstractвҖ”Insertion/Deletion/Substitution (IDS) error correction will become important for future high-density storage devices, e.g., bit-patterned media. For IDS error correction, concatenated codings of an outer LDPC code and an inner marker code have been proposed. To improve the synchronization capability, marker decoder can employ symbol-level synchronization (SLS) with a large symbol size, while the computational time complexity of the SLS increases exponentially with the symbol size. This paper proposes an SLS using a probability lookup table to reduce the complexity. Probabilities in the table are precomputed for given channel and code parameters, and computationally intensive calculations of the SLS are replaced by table lookups. The proposed method is applicable to both single-path and multipath decodings. The bit error rate (BER) is evaluated for SLS of symbols sizes bS = 1, 2, 4, and 8 bits, and it is confirmed that larger symbol size gives lower BER.
I. INTRODUCTION
High-density storage devices and high-speed communication systems often require precise synchronization in data transmission. For example, in write operation of bit-patterned media (BPM), write signal and position of bit island should be synchronized precisely because magnetic grains of the BPM are discontiguous to improve the density. The precise synchronization, however, is difficult because of head vibrations, disk speed variations, and jitter in internal clock [5]. Missynchronization causes insertion/deletion errors in data, which cannot be efficiently corrected by conventional random bit/symbol error correcting codes. Thus these devices/systems require error control coding capable of correcting insertion and deletion errors as well as substitution (random bit) errors.
To efficiently correct insertion/deletion/substitution (IDS) errors, several coding methods have been proposed. LevenshteinвҖҷs code [1] can correct single IDS errors by using a constraint on the weighted sum of codeword bits. For multiple IDS error correction, concatenated codings are extensively studied, where the outer coding is by random error correcting code, e.g., LDPC code and RS code, and the inner coding is marker coding [3], [6] or watermarking [2]. The decoding of inner code employs forward-backward (FB) algorithm to calculate the likelihood of codeword bit/symbol.
The FB algorithms can be classified into two classes: bit level synchronization (BLS) and symbol level synchronization (SLS), where symbol is a group of bS bits. While the SLS provides much lower bit error rate (BER) compared to the BLS, the computational time complexity of the SLS grows exponentially with symbol size bS, and thus the SLS with a large bS, e,g, bS вүҘ 4, has been generally impractical for IDS channels. Table I lists examples of conventional concatenated coding for IDS and deletion channels. For IDS channel, Davey

and MacKay showed a BLS [2], and Wang et al. proposed an SLS [3] with relatively small bS, i.e., bS = 2 and 3. For a segmented deletion channel, Wang et al. presented an SLS [4] with larger symbol size, e.g., bS = 16. To the authorвҖҷs knowledge, however, SLS with larger symbol size, bS вүҘ 4, for IDS channel has not been evaluated.
To reduce the computational time complexity of the SLS with larger symbol size bS, this paper presents an FB algorithm which uses a probability lookup table. The table stores probabilities which are computationally intensive to be calculated. Since the probabilities are uniquely determined by the channel and inner coding parameters, the table can be reused until the parameters are changed. As shown in Fig. 1, the encoding/decoding flow of the proposed method is the same as that of the conventional concatenated codings except that the SLS uses the lookup table. The lookup table can be used in both single-path (SP) decoding (inner code decoding followed by outer code decoding) and multi-path (MP) decoding (iterative decoding between inner and outer codes).
This paper is organized as follows. Section II presents fundamentals of marker coding. Section III shows SLS using probability lookup table. Section IV evaluates the decoded BER, execution time of SLS, and size of lookup table, and Section V concludes the paper.

In the evaluation, the reduced lookup table is generated with parameters wi = wd = 1 for bS = 2, and wi = wd = 2 for bS = 4 and 8. V. CONCLUSION
For IDS error correction coding, this paper proposed a symbol-level synchronization (SLS) whose symbol size is larger than the conventional methods. To counter the problem of exponential increase of computational time-complexity with symbol size, a probability lookup table is employed in the SLS, where the table stores the probability, Pr(y|x), of transmitted symbol x being received as y in the IDS channel. This paper also showed a reduced probability lookup table to reduce the size of lookup table. The BER is evaluated for the SP and MP decodings with symbol sizes bS = 1, 2, 4, and 8 bits. The results showed that the BER can be lowered by using larger symbol sizes.
