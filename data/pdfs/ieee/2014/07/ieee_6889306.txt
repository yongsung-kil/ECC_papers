Modified Tree Structure Approach for Finding the First Two Minimum Values
Weiwei Cui, Yi Yang, Xueqin Jiang, and Sooyoung Kim

Abstract—In the min-sum decoding algorithm of low-density parity-check (LDPC) codes, for a given set of input values to one check node, efficient approachs for finding the first and the second minimum values are greatly needed for the low complexity decoder. In this paper, based on the tree structure (TS) approach, we propose the modified TS (MTS) approach, which requires less number of comparisons to find the first and the second minimum values. Therefore, the hardware cost of the proposed MTS approach is lower than that of the TS approach.
Index Terms—Low-density parity-check (LDPC) code, minsum algorithm, comparison, first minimum value, second minimum value.

I. INTRODUCTION

Low-density parity-check (LDPC) codes have been investigated extensively in recent years because of their powerful error-correcting performance with the iterative sum-product decoding algorithm [1-4]. However, the computational complexity of this algorithm is high and it is hard to implement in hardware. Min-sum algorithm is a good approximation of the sum-product algorithm which has the advantage of simpler computations compared to sum-product algorithm [5] [6].
An LDPC code is a linear block code defined by a sparse parity-check matrix H having dimension M × N . A Tanner graph with M check nodes in one class and N symbol nodes in the other can be created using H as the integer-valued incidence matrix for the two classes [1]. In the sum-product decoding algorithm and the min-sum decoding algorithm, the decoding message is passed iteratively between the symbol nodes and the check nodes. In the min-sum algorithm, the main task of the check node of degree n is to find the first minimum value min1st and the second minimum value min2nd of the n input values [7-9]. In [10] [11], the tree structure (TS) approach, which has low hardware cost, is developed to find the min1st and the min2nd. To further reduce the hardware cost of check nodes, in this paper we propose the modified TS (MTS) approach. We show that the MTS approach can achieve lower hardware cost in area than that of the TS approach.
The remainder of this paper is organized as follows. Section II describes the TS approach. Their hardware implementation is also described in this Section. Section III introduces our proposed MTS approach for the check node of n = 2k input values and the corresponding hardware implementation. Section IV compares the hardware cost of the MTS approach with that of the TS approach. Finally, Section V conclusions the paper.

II. PRELIMINARIES
A. The TS Approach
Following the literature [10], we denote MUX as a 1-bit 2-to-1 multiplexer. Suppose that an input value contains wbits, then the MUX also require w-bits. we denote MUX w as a w-bit 2-to-1 multiplexer. Considering the TS approach in [10], (2k+1 − 3) comparators are needed to find min1st and min2nd from n = 2k input values of a check node, where k is a positive integer. Totally, the TS approach requires (2k+1 −3) comparators and some low cost MUX ws and MUXs.
Given the n input values of a check node, the hardware unit that generates both min1st and min2nd is called the minimum-value generator (n-mVG). More specifically, we will consider the case of n = 8. III. THE MODIFIED TS APPROACH
In this section, we will first introduce the algorithm of the MTS approach and then introduce its hardware architecture. The proposed modified CU (MCU) of the MTS approach includes one less comparison than the CU of TS approach.

B. Hardware Implementation
Fig.8 shows the building block of an 8-mVG in the MTS approach. It is realized by 2 4-mVGs and 1 MCU. In total, the 8-mVG is realized by 10 comparators, 20 MUX ws and 4 MUXs.
Generally, when n = 2k, a 2k-mVG in the MTS approach can be constructed with the k stage architecture as shown in Fig.9. One 2k-mVG needs 2k−1 comparisons in the first stage (left most) for the 2k inputs, followed by (k − 1) stages of MCUs. Thus, the 2k-mVG in the MTS approach needs (2k−1 − 1) MCUs, and each MCU needs 2 comparators (1 comparator for 2-mVG and 1 comparator for mn0) and 4 MUX ws. Therefore, the total number of comparators of the 2k-mVG in the MTS approach is (2k +2k−1 −2). Note that the MTS approach and the TS approach need the same number of MUX ws and MUXs. Consequently, the hardware cost of a 2k-mVG in the MTS approach includes (2k + 2k−1 − 2) comparators, (2k+1 + 2k − 4) MUX ws and (2k − k − 1) MUXs. V. CONCLUSION
To reduce the hardware cost of the min-sum algorithm, in this paper we first propose the MTS approach, which includes less number of comparisons. Then, we also introduce the hardware architecture for the proposed MTS approach. Finally, since the MCU of the MTS approach needs less number of comparisons than CU of TS approach the hardware architecture of MTS has the advantage of lower complexity in area than that of the TS approach.
