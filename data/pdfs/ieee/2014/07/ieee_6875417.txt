2014 IEEE International Symposium on Information Theory

On the Upper Bound on Undetected Error Probability for LDPC code
Pavel Rybin and Victor Zyablov Inst. for Information Transmission Problems
Russian Academy of Science Moscow, Russia
Email: {prybin,zyablov}@iitp.ru

Abstract—This paper deals with the method of undetected error probability estimation for a low-density parity-check (LDPC) code under any given iterative decoding algorithm. We propose such modification of a given iterative decoding algorithm, that almost preserves a decoding failure exponent and decoding complexity of this algorithm. We obtain the upper bound on the undetected error probability for the modified algorithm. We show how to use the proposed method to estimate the undetected error probability of LDPC code under the belief propagation (BP) algorithm at the end of this paper.
I. INTRODUCTION
In the classical work [1] Forney considered exponential error bounds for different coding schemes. He obtained the trade-off relation for undetected error and decoding failure probability for nonlinear codes. Then Zyablov and Blokh obtained the analogical trade-off relation for linear codes in [2]. But the complexity of the decoding algorithms considered in [1] and [2] are exponential.
In this paper we consider the iterative decoding algorithms of low-density parity-check (LDPC) codes with the complexity about O (n log n). LDPC code was proposed by Gallager in [3]. A lot of works are devoted to estimation of the error exponent for LDPC codes under decoding algorithms with the exponential complexity, e.g. [4]–[6]. From [6] it is known that the lower bound on the error exponent under the ML decoding of LDPC codes meets the lower bound on the error exponent under the maximum likelihood decoding of good linear codes obtained in [7]. It is important to note that the complexity of the ML decoding is O 2min(R,1−R)n . But at this moment LDPC codes are very popular because of their very efficient decoding algorithms with low-complexity. That’s why the estimation of the probability of decoding failure for LDPC code under the iterative low-complexity decoding algorithm is very interesting and important question. Only few of papers are devoted to this topic. In [8] LDPC code with a special construction and a low-complexity decoding algorithm were proposed. The lower bound on the decoding failure for the code under proposed decoding algorithm was obtained. It was shown that such LDPC code with the special construction exists, that the decoding failure probability of the decoding algorithm with the complexity O (n log n) exponentially decreases for all code rates below the channel capacity. In [9], [10] the expander codes were considered with the rate near

the channel capacity under the iterative decoding algorithm. It was shown that the decoding failure probability has a positive exponent for all rates between zero and the channel capacity and the decoding complexity grows linearly with the code length.
In this work we follow [1], [2] and obtain the trade-off relation for the undetected error and the decoding failure probability. We develop the method of the undetected error probability estimation for LDPC code under some modification of any given iterative decoding algorithm with a known probability of decoding failure. We propose such modification of the given iterative decoding algorithm, that almost preserves the decoding failure exponent and the decoding complexity of this algorithm. And it lets us obtain the trade-off relation for the undetected error and decoding failure probabilities for LDPC code under a modified algorithm. We show how to use the proposed method to estimate the undetected error probability of LDPC code under a modified version of the belief propagation (BP) algorithm at the end of this paper.

III. DECODING ALGORITHM
A. Decoding algorithm description
In this paper we consider the modification At of some given iterative algorithm A. The main idea of the decoding algorithm At is the following. In the case of the successful decoding of the received sequence r with the algorithm A the algorithm At returns the code word c0 obtained by A iff the code distance between c0 and r is less than some value t0 (t0 ≤ d/2, where d is the code distance): w (c0, r) ≤ t0. Otherwise the decoding algorithm At fails. In other words, the decoding algorithm At works in some spheres with the radius t0 around the code words.
Let us consider the decoding algorithm At, that consists of the following two steps:
1) the received sequence r is decoded with the algorithm A:
• in the case of the successive decoding the code word c0 obtained by the algorithm A is saved and we go to the next step;
• otherwise a failure flag is returned and the execution of the decoding algorithm stops;
2) the distance w (c0, r) between the received sequence r and the obtained code word c0 is calculated:
• if w (c0, r) ≤ t0 then the code word c0 and a success flag are returned and the execution of the decoding algorithm stops;
• otherwise a failure flag is returned and the execution of the decoding algorithm stops.
Remark 1: Let us note that the complexity of the decoding algorithm At is increased by no more than n compared with the complexity of the algorithm A. We assume that the complexity of the algorithm A in a common case is more than O (n). So, in this case the asymptotic complexity of the algorithm At is preserved almost the same as the complexity of the algorithm A.
Remark 2: At the end of the paper we will consider the BP decoding (sum-product) algorithm over additive white Gaussian noise (AWGN) channel with “soft” decision on the receiver as an example for algorithm A. In this case it is important to note that for the decoding algorithm A the received sequence with the “soft” decision is used and for calculation of distance w (c0, r) the received sequence

r with the “hard” decision is used. So, in this case we can assume that the received sequence r with the “hard” decision is transmitted over binary symmetric channel (BSC) with crossover probability p.

V. NUMERICAL RESULTS
In this section we use the described above method to estimate the undetected error probability Pt(0e) and the decoding failure probability Pt(0τ) for LDPC code under the decoding algorithm At.
As the algorithm A we consider the BP decoding (sum-product) algorithm over additive white Gaussian nose (AWGN) channel with the “soft” decision on the receiver. In Fig. 2 you can see simulation results (frame error rate (FER)) obtained for LDPC code with n0 = 14, ℓ = 7, R = 0.5 and n = 2002 under BP decoding with 5 iterations.
Now let us consider the algorithm At based on BP decoding algorithm (algorithm A). To get the lower bound on the undetected error exponent we need first of all obtain the value of the radius τ0 using Lemma 1 (see Fig. 3). Then using Theorem 1 and obtained value of τ0 we get the lower bound on the undetected error exponent Et(0e). Using this lower bound on the undetected error exponent and Lemma 3 we obtain the upper bound on the undetected error probability Pt(0e) and the decoding failure probability Pt(0τ).
