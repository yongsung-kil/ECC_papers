Due to their error-correcting strength they are used in a variety of applications and standards, such as Wi-Fi, (e.g. IEEE 802.11n, IEEE 802.11ac) and Wimax. They are constructed by a generator matrix G. For a message m, the corresponding codeword c of the code is obtained by: c = mG. LDPC codes are commonly defined by their Parity Check Matrix (PCM), which is very sparse. The weight of a PCM row or column is the number of the nonzero elements it contains. LDPC codes can be distinguished into regular codes having identical row and/or column weights or irregular codes otherwise. Also, an important parameter is the code rate defined as the number of information bits to the number of bits the codeword has, R = k/n. The PCM is commonly graphically represented by a corresponding bipartite graph, called Tanner Graph (T G). A Tanner Graph contains two types of nodes, the variable nodes and the check nodes. Each variable node vi corresponds to the ith column of the PCM, while check node cj corresponds to the jth row of the PCM. An edge (vi, cj) exists in the graph if and only if the element at the jth row and ith column of the PCM equals to 1, i.e., hij = 1. Efficient decoding is implemented by iterative decoders based on message-passing algorithms. However, coding gain of LDPC codes under iterative decoding, is reduced by some structures defined on the Tanner Graph; prominent among them are the so-called trapping sets:

Several authors have proposed techniques to address (cf. [1], [6], [7] and [8]) the problem of identifying trapping sets. In this paper we propose a technique for the fast identification of trapping sets. The proposed technique contributes to improving the potential performance of LDPC codes and the corresponding decoders by identifying trapping sets and this could facilitate developing LDPC decoders that exploit the knowledge of particular trapping sets existing in a code to avoid them effectively. Furthermore the proposed technique could facilitate BER vs. noise performance prediction based on (1), avoiding extremely time-consuming simulations. Finally, it could be integrated to code-construction procedures aiming to LDPC codes with minor impact from trapping sets. We conducted experiments for the LDPC codes of IEEE 802.11n with block lengths 648, 1296, 1944 bits and code rates 1/2, 2/3, 3/4 and 5/6. Also, experiments were conducted on a code with block length 2016 bits and code rate 3/4 (irregular codes).

A. Identification of Unique Cycles At first, the proposed algorithm finds all unique cycles for
a given code by searching a tree that is constructed from the corresponding T G. We focused on cycles of length 4, 6, and 8, with distinct nodes, because trapping sets of small size (variable nodes ≤ 11) are mainly responsible for high error floor as they have high failure rates [9] [10]. We form the set Sd of unique cycles of length d. According to (1) and based on the knowledge of probabilities P {ξTa,b }, FER is completely determined by the distribution of trapping sets for all relevant values of a and b. The smaller the values a and b are, the stronger the influence the trapping set bears on FER. Trapping sets of larger size are rarely activated and therefore their influence to the error floor is very small [3]. The proposed method identifies unique cycles by constructing the TG of a code from its PCM and searching in it. If the (d + 1)st variable node is the same with the first of a path and the nodes in the path are distinct, a cycle is found and stored. The proposed method is general and flexible limited only by execution time. In particular, searching for cycles of length 10 and above requires long execution time and the results are too many, further encumbering the next steps of the algorithm. V. CONCLUSIONS We have introduced a technique for trapping sets identification, flexible and parametric for any LDPC code, block length and code rate. It requires low memory and exploits dynamic allocation, has great speed and reduces substantially the search complexity for the trapping sets identification. Thus it may offer a viable solution for the design of trapping-setaware decoders, accelerated experiments, and efficient LDPC code design.
