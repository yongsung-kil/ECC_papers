Code Construction Algorithm for Architecture Aware LDPC Codes with Low-Error-Floor
Dariusz Kania∗ and Wojciech Sułek∗
∗Faculty of Automatic Control, Electronics and Computer Science, Institute of Electronics Silesian University of Technology, Gliwice, Poland {dkania,wsulek}@polsl.pl

Abstract— The common approach for the design of an error correction system is first to construct a code and then to define the hardware structure of the encoder and decoder. However, in the case of LDPC codes (Low-Density Parity-Check) such a constructed code is generally not well suited for a hardware implementation. It has been recognized that the code construction and hardware design must be considered jointly to facilitate LDPC decoder and encoder implementation. In this paper, an efficient decoder structure for regular and irregular LDPC codes, based on TDMP (Turbo-Decoding Message Passing) scheme is designed first. The decoder has been implemented and verified in an FPGA device. Constraints for the parity check matrix of a code to be suitable for the decoder architecture are defined. Then an algorithm for LDPC parity check matrix construction subject to these constraints is presented. The algorithm aims at improving performance of the code in the low SNR region by employing irregular codes as well as in high SNR region by reducing the number of small Stopping Sets and Trapping Sets in the Tanner graph of the code making use of a computer search technique.
I. INTRODUCTION
LDPC code is a linear block code defined by an M × N parity-check matrix H, which is sparse. Matrix H has a graphical representation in the form of bipartite graph, called Tanner graph, that visualizes iterative decoding algorithms, known as message passing algorithms. Regular LDPC codes were first proposed by Gallager in the 60’s [1], but forgotten for years by the researchers. Their implementation complexity was exceeding the capabilities of the accessible technology, so they were not considered for practical applications. The codes were rediscovered in the late 90’s [2], [3], and after then they are under interest of the scientific world.
Considering an LDPC message passing decoder implementation, two challenges are encountered: how to realize the computation units and how to realize the message passing scheme. Fully parallel decoder achieves very high throughput, but for large code length it suffers from a huge number of computation units and routing lines. To achieve appropriate tradeoff between implementation complexity and decoding throughput, partially parallel decoder is of practical interest. Realizing message passing is not a trivial issue in this case, because the Tanner graph connectivity has to be realized part-by-part. Partially parallel decoder architecture consists of a reduced number of computation units, a configurable interconnection network and a message storage fabric. The parity check matrix in a block structured form enables efficient

memory and interconnection network organization. The matrix is partitioned such that each block submatrix is assumed to be either a zero matrix or a permutation of rows of an identity matrix. This family of structured codes is known as Architecture-Aware LDPC (AA-LDPC) or VLSI-Oriented codes [4], [5].
The turbo-decoding message passing algorithm (TDMP) presented in [5] has a faster convergence and reduced message storage requirements than standard two-phase message passing algorithm. However, the TDMP decoder chip, presented in [6] is capable only for regular LDPC codes. As is well known [7], the irregular codes with carefully chosen degree distribution outperform significantly the regular ones. In this paper an architecture capable for decoding regular as well as irregular AA-LDPC codes is presented. The decoder structure has been developed for FPGA devices with dual-port memories and formalized in the form of VHDL description.
Then we present an algorithm based on computer search technique that reduces the existence of harmful structures in the Tanner graph of AA-LDPC code. These structures are small Stopping Sets and Trapping Sets and they decrease code performance in the high SNR region. The algorithm is based on searching for graphs with minimized number of cycles with low extrinsic connectivity. The algorithm is introduced and simulation results are presented in section IV.
II. LDPC CODES OVERVIEW
The parity-check matrix HM×N of an LDPC code C (with rate r = 1 − M/N ) represents the relation between the N bits of the codeword and the M parity-check equations in GF (2) field. Vector x is a correct codeword (x ∈ C) if the parity check condition HxT = 0 is satisfied. Another way for defining the code is a bipartite graph (Tanner graph, [8]) with bit nodes representing bits and check nodes representing parity-check equations (fig. 1). Formally, a Tanner graph (V, E) contains the set of vertices V = Vc ∪ Vb, where Vc = {c1, c2, . . . , cM } is the set of check nodes, Vb = {b1, b2, . . . , bN } is the set of bit nodes and the set of edges E such that E ⊆ Vc × Vb. The edge (cm, bn) ∈ E if and only if hmn = 0, where hmn denotes the entry of H at mth row and nth column. A Tanner graph is (dc, db)-regular if all its check nodes have degree dc and all its bit nodes have degree db.

A. Turbo-Decoding Message Passing (TDMP)
Turbo-Decoding Message-Passing (TDMP) algorithm [5],
[6] constitutes a modification in message passing scheme of TPMP. Here C is considered as the concatenation of some number of codes, which means the set of rows of H is virtually
partitioned into a number of subsets. Each subset constitutes
a code, which will be called a subcode. A word is a correct
codeword if it belongs to all constituent subcodes. Let Ci denote the ith subcode and its parity check matrix
Hi, i = 1, . . . , D. The code is then C = C1 ∩ . . . ∩ CD and its parity check matrix is HM×N = H1; H2; . . . ; HD T. Furthermore, the check node set Vc of the Tanner graph of the code is partitioned as Vc = Vc1 ∪ . . . ∪ VcD. It is assumed that the rows in each submatrix Hi do not overlap (each bit node is incident to at most 1 check node in the subset Vci). The set of indexes of check nodes adjacent to bit node bn is denoted by M (n) and – similarly – the set of indexes of bit nodes adjacent to check node cm by N (m). Furthermore, N (m) \n represents the set N (m) excluding n and M (n) \m represents the set M (n) excluding m.
Inputs to the algorithm are measures of reliability of the
received data based on channel observations; they are called intrinsic channel reliability values and will be denoted as δn for nth bit. The TDMP algorithm determines a codeword iteratively in D sub-iterations, with one sub-iteration per constituent code Ci. belongs to subcode Ci and λni the reliability values calculated in the previous iteration. III. DECODER IMPLEMENTATION
A configurable decoder, based on TDMP scheme has been implemented in the form of synthesizable VHDL model. The decoder (fig. 3) is capable for decoding any regular or irregular AA-LDPC code. Γ-memory is initialized with soft channel values δ with wordlength W . Operations are performed in computation modules called Soft-Input-Soft-Output (SISO) units. Assuming P rows in a block row, it is convenient to map the algorithm into P/L SISO modules, where L is any integer number. If the maximum throughput has to be obtained then L = 1 and P SISO modules are implemented (as in fig. 3).
In a single time slot, messages are fed to the SISO unit in serial manner through Configurable Interconnection Network, updated, and then serially fed back to the memory. A memory word consists of P Γ-values, hence wordlength for Γ-memory equals P W and memory depth is B. H Structure Memory stores location of the permutation matrices in H and Πmemory stores respective permutations used for configuring Interconnection Network. Memory for extrinsic Λ messages is partitioned and included in SISO units as FIFO buffers. When Stop Criterion Check Module indicates correct codeword, decoding is halted and data is outputted in words of length P (the MSBs of the P Γ-messages in a single memory word are equivalent to hard decisions).
Reliability values Γ and Λ in a hardware implementation

are represented by fixed-point numbers, with two parameters: number of bits W and analog clipping threshold Am. Computations are performed under sign and magnitude representation, thus messages are quantized into 2W − 2 levels uniformly placed in the symmetrical range [−Am, Am]. The two parameters W and Am should be carefully selected to achieve the desired trade-off between decoder performance and implementation complexity.
The wordlength W and the clipping threshold Am can be determined experimentally. Fig. 4 presents simulation results for a code with blocklength N = 2048 decoded with Matlab software (floating point messages) as well as decoder FPGA implementation with worldlength W = 8 and W = 6. It can be observed that with 8-bit messages the performance is even better than in floating point case. If 6-bit messages are applied, the performance degradation is very little and exists only in higher SNR region (about 0.05dB loss).
Fig. 5 presents influence of the clipping threshold on the performance of a rate 0.5 regular code decoded with wordlength W = 6. It shows that this influence should not be neglected. The best results were obtained with Am = 14, the loss in performance is very little in this case.

IV. LOW-ERROR FLOOR CODE CONSTRUCTION
In this section an algorithm for AA-LDPC parity-check matrix construction is presented. The algorithm aims to reduce statistically the number of some structures in the Tanner graph that are harmful to code performance in the low SNR region (they cause an error-floor). First these harmful structures will be outlined.

Extrinsic Message Degree (EMD). An extrinsic check node to a set of bit nodes is a check node which is singly connected to the set. The Extrinsic Message Degree (EMD) of a set of bit nodes is the number of extrinsic check nodes connected to the set.
Stopping Set. A stopping set SSv is a subset of the set of bit nodes Vb with cardinality v, such that no extrinsic check nodes are connected to SS.
Trapping Set. A trapping set T S(v,s) is a subset of the set of bit nodes Vb with cardinality v, such that the subgraph induced by T S contains s odd-degree check nodes and strictly fewer even-degree check nodes.
As was proved ( [12], [13]), the performance of an iteratively decoded LDPC code can be limited by the existence of small stopping sets and trapping sets. A set of variable nodes with large EMD will require additional concatenation of nodes to become a stopping set. In a bipartite graph free of degree1 bit nodes, every stopping set contains cycles. Similarly trapping sets are composed of interconnected cycles. Hence, if short cycles with small EMD are eliminated from the graph, also small stopping sets and trapping sets are eliminated.
The parity check matrix HDP ×BP that is suitable for our decoder architecture, as discussed earlier, is an D × B matrix of P × P submatrices, where each submatrix is either an allzero or permutation of the identity matrix.
The D × B matrix with zeros indicating the all-zero submatrices and ones indicating the permutation submatrices we call the seed matrix for the construction and respective graph we call the seed graph. The seed matrix is expanded by replacing its entries with appropriate P × P submatrices to obtain the HDP ×BP parity check matrix called covering matrix.
We define the seed graph expansion as in [14]. Let (V, E), V = Vc ∪ Vb, E ⊆ Vc × Vb be a seed graph. Let G = {g1, . . . , gP } be a finite set with cardinality P . The graph after expansion, called covering graph has vertex sets Vc × G and Vb × G. For each e ∈ E, e = (cm, bn) we fix a permutation of G, θe. The covering edge set is then E × G = {((cm, gi), (bn, θe(gi))) : i = 1, . . . , P, e ∈ E}.
Important fact characterizing the relationship between the cycles in the expanded matrix and the cycles in the seed matrix is as follows [14]. Let e1 ∼ e2 ∼ · · · ∼ e2k ∼ be a cycle of length 2k in the seed graph. The covering graph possesses a respective cycle of length 2k if and only if

θe1 ◦ θe−21 ◦ θe3 ◦ θe−41 ◦ . . . ◦ θe2k−1 ◦ θe−21k

(9)

has a fixed point. Precisely, the number of cycles in the covering graph equals the number of fixed points in (9).
In this paper the permutation matrices in the form of rotations of the diagonal identity matrix are considered, that is θe(gi) = g(i+de mod P ) where de is an integer fixed for each e ∈ E. In this case (9) can be reformulated as (for prove see [14]): for the permutation matrices in the form of the rotations of the diagonal identity matrix, the covering graph possesses a respective cycle of length 2k if:

de1 − de2 + de3 − de4 + . . . + de2k−1 − de2k ≡ 0 mod P (10)

The algorithm for the defined AA-LDPC parity check matrix construction is as follows.
1) Determine the code and decoder parameters: the number of SISO units for decoder implementation and the size of submatrices P ; the size of the parity check matrix N , M determined by the desired code length and code rate of the specific application. Determine the node degree distributions dc, db. If irregular code is desired, the degree distributions can be determined by degree optimization [7].
2) Construct the seed matrix HD×B. The (relatively small) seed matrix can be constructed in any known method for constructing LDPC parity check matrices without implementation constraints, e.g. the improved PEG (Progressive Edge Growth) as proposed in [15].
3) For each edge ej in the seed graph, determine the cyclical shift value dej for the matrix expansion. This task is accomplished in a few steps:
a) Determine all closed walks in the seed graph with length lower than some predetermined value Lmax and EMD lower than another value EMDmax. Determine the EMD for all the closed walks.
b) For each closed walk, fix a value of harmfulness: the greatest for length-4 walks, then for all other walks with EMD equal 0, from the shortest to the longest, then for walks with EMD 1, and so on. For each closed walk with consecutive edges ei1 ∼ ei2 ∼ · · · ∼ ei2k ∼ define a non-congruence condition for eliminating the respective cycle in the covering graph as:

dei1 −dei2 + dei3 − dei4 + . . . . . . + dei2k−1 − dei2k ≡ 0

Performance of the obtained codes is generally not worse than performance of the codes from MacKay’s database [16] (however, those codes are not architecture aware). V. CONCLUSIONS
The first contribution of the presented work is a configurable LDPC decoder FPGA implementation for regular and irregular codes that are Architecture Aware. The decoder is based on TDMP message passing scheme, Min-Sum algorithm with correction and double recursion. In contrast with the ASIC decoder chip presented in [6], the designed decoder can be implemented in FPGA devices and is suitable for irregular codes. It has been verified in Xilinx Virtex-II device. Throughput of the decoder is lower than the mentioned ASIC chip, but it is similar to the other known FPGA implementation results. For

example the Virtex-II Pro decoder implementation presented
in [18] has throughput 2.46 bits per cycle per iteration, which
is less than for our decoder.
The second contribution is the AA-LDPC code construction
algorithm that aims at improving performance of the irregular
codes by reducing the number of small Stopping Sets and
Trapping Sets in the Tanner graph of the code. Simulations
have shown that reducing these harmful structures gives gen-
erally better results than simple girth conditioning algorithm.
