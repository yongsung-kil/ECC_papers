Abstract—Multi-Edge type LDPC codes are introduced by Richardson and Urbanke, and they show examples of their ensembles has better performance than other known ensembles. Orlitsky et al. derived the condition for irregular LDPC code ensembles with minimum distance linearly increasing in code length. We derive the condition corresponding to Orlitsky’s condition for two-edge type LDPC code ensembles which is simple example of Multi-Edge type LDPC code ensembles.
I. INTRODUCTION
Low-Density Parity-Check(LDPC) codes proposed by Gallager in 1963 [1] are linear codes defined by sparse bipartite graphs corresponding to parity check matrices, and have been shown to perform extremely well even with computationally efficient iterative decoding. In [3], Richardson and Urbanke proposed a generalized framework of structured LDPC codes such as irregular repeat accumulate (IRA) codes, accumulate repeat accumulate codes (ARA) codes [6], concatenated tree (CT) codes [7], and protograph LDPC codes [8]. Those codes are called multi-edge type LDPC codes and known to have better performance than standard irregular LDPC codes especially for short block lengths, while requiring lower complexity.
Minimum distance of linear codes is important characteristic of codes, which gives the upper bounds of the performance codes under maximum likelihood (ML) decoding. For the standard irregular LDPC code ensembles, Orlitsky et al. [2] showed that the number of small linear weight codewords decreases exponentially in code length iff λ (0)ρ (1) < 1, and a 1 − λ (0)ρ (1) fraction of codes have linear minimum distance if λ (0)ρ (1) < 1.
In this paper we consider the simplest multi-edge type codes with two edge types which parameterized two degree distributions (λL, ρL) and (λR, ρR). We call that codes twoedge type LDPC codes. We derive a condition of (λL, ρL) and (λR, ρR) that two-edge type LDPC code ensembles have exponentially decreasing small linear weight codewords. For simple symmetric two-edge type LDPC codes consisting of two irregular LDPC codes with the same degree distributions (λ, ρ), the condition is λ (0)(2ρ (1) + 1) < 1.
II. PRELIMINARIES
A. Two-Edge type LDPC Code Ensembles
Intuitively, two-edge type LDPC code ensembles can be seen as fusion of two irregular LDPC code ensembles. Part

of variables nodes in two graphs are shared and identified. In
Fig. 1, we give an example of a two-edge type LDPC code
ensemble. This is fusion of red (left) graph and a blue (right)
graph. Variable nodes of degree-type b2 and check nodes of degree-type d2 are shared by both graphs and unidentified.
In [3], multi-edge type LDPC codes are defined by two multinomials ν(r, x) and μ(x), where x is corresponding to edge-types and r is to channel inputs which we don’t care in this paper, therefore we set r = 1. We consider multiedge type LDPC codes with two edge-types with ν(1, x) = λ˜(xL, xR) := (i,j) λ˜i,j xLi xRj and μ(x) = ρ˜(xL, xR) :=
(i,j) ρ˜i,jxiLxRj . Let n be the code length, and for each variable node-type (i, j) the quantity λ˜i,jn is the number of variable nodes of degree-type (i, j) in the graph. Similarly,
for each check degree-type (i, j) the quantity ρ˜i,jn is the number of check nodes of degree-type (i, j) in the graph. Each
variable and check node of degree-type (i, j) has i left (red)
and j right (blue) edges. For each edge-type, we define sets of variable degree-types as BL := {(i, j)|i = 0, λ˜i,j = 0}, BR := {(i, j)|j = 0, λ˜i,j = 0}, and sets of check degreetypes as DL := {(i, j)|i = 0, ρ˜i,j = 0} DR := {(i, j)|j = 0, ρ˜i,j = 0}. Then, for a two-edge type ensemble, we define B := BL ∪ BR,D := DL ∪ DR. For (i, j) ∈ B and D, degree of each edge-type are denoted to be degL((i, j)) = i and degR((i, j)) = j, and then degree of a two-edge type ensemble is deg((i, j)) = i + j. We define design rate as R := λ˜(1, 1) − ρ˜(1, 1).

IV. CONCLUSION

In this paper, we give an approximation of the normalized average codeword weight distributions of two-edge type LDPC code ensembles for small weight. Moreover we derive the condition that the number of small linear weight

codewords decrease exponentially in code length. For a given two-edge type LDPC code ensemble with a parameter (λ˜(xL, xR), ρ˜(xL, xR)), using Theorem 2, we can easily check if a given ensemble has exponentially many or few codewords of small linear weight. That condition can be seen as a necessary condition that the codes have linear growing minimum distance. For our future work, we will extend the result for multi-edge type ensembles with more than two edge-types.
