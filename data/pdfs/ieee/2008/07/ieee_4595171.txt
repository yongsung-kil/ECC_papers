Abstract— Protograph-based doubly generalized LDPC (DGLDPC) codes are explored in this paper. We extend (and in the process simplify) the technique for computing the ensemble weight enumerators of protograph-based LDPC and GLDPC codes to DGLDPC codes. We find that with careful design, protograph-based DGLDPC codes can have a better asymptotic growth rate of minimum distance than that of the protograph-based LDPC and GLDPC codes. Simulation results confirm that protograph-based DGLDPC codes have a low error floor.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were invented by Gallager [1] in the 1960s. Tanner proposed generalized LDPC (GLDPC) codes by using short component codes other than single parity check (SPC) codes as check nodes [2]. Within this framework, many GLDPC codes with different component codes have been studied [3]-[8]. In [9], doubly GLDPC (DGLDPC) codes were introduced. These codes generalize not only check nodes but also variable nodes of LDPC codes. It has been shown that DGLDPC codes can achieve both better threshold and lower error floor compared with that of their LDPC and GLDPC counterparts [10][11].
In classical coding theory, the weight enumerator or weight distribution plays a crucial role in bounding the error performance of a code. It is usually difficult to obtain the weight enumerator of a given LDPC code. Therefore the average weight enumerator of an LDPC code ensemble is calculated instead. It is also useful to obtain the asymptotic weight enumerator of an LDPC code ensemble by letting the code length go to infinity. Asymptotic weight enumerators for LDPC ensembles have been studied in [1][14]-[19]. A similar analysis can also be applied to protograph-based codes.
Protographs or projected graphs were introduced in [12][13]. They are defined as a Tanner graph with a relatively small number of nodes. Each node and each edge in a protograph represent a node type and an edge type, respectively. Protograph-based LDPC and GLDPC codes and their ensemble weight enumerators have been discussed in [20] and [21], respectively. In this paper, we study protograph-based DGLDPC codes and derive their asymptotic ensemble weight enumerators. In the process, we simplify the optimization approach of [20][21] by considering the dual problem. We

find that protograph-based DGLDPC codes can have better asymptotic growth rate of minimum distance than protographbased LDPC and GLDPC codes.
II. PROTOGRAPH-BASED DGLDPC CODES
For protograph-based LDPC and GLDPC codes, a larger graph can be obtained from a protograph by a “copy-andpermute” process. In this process, a protograph is first copied p times and then the p edges of the same type are permuted under the constraint that they still connect to the same type of nodes. A protograph can contain parallel edges. However, these parallel edges should be eliminated in the copy-andpermute process in order to obtain an appropriate Tanner graph for the parity check matrix.
Protograph-based DGLDPC codes can be obtained in a similar way. As defined in [9], a DGLDPC code is composed of super variable nodes and super check nodes with each of them representing a component code. A DGLDPC protograph with nv super variable nodes and nc super check nodes is denoted as G = (V, C, E), where V = {v1, v2, · · · , vnv } is the set of super variable nodes, C = {c1, c2, · · · , cnc } is the set of super check nodes, and E is the set of edges. The codes corresponding to the super nodes in V and C are (lv1 , kv1 ), (lv2 , kv2 ), · · · , (lvnv , kvnv ) and (lc1 , kc1 ), (lc2 , kc2 ), · · · , (lcnc , kcnc ), respectively. A DGLDPC code is obtained by copying the protograph G p times and permuting edges among the p copies.
In order to apply the results in [23], we regard the group of p copies of super variable node vi as a constituent code with kvi length-p inputs and lvi length-p outputs. The weight vectors of kvi inputs and lvi outputs are denoted as di = [di,1, di,2, · · · , di,kvi ] and wi = [ωi,1, ωi,2, · · · , ωi,lvi ], respectively, where di,j and ωi,j are the weights of the j-th input and output of node vi, respectively. Similarly, we regard the group of p copies of super check node cj as a constituent code with lcj length-p inputs and no output. The weight vector of lcj inputs is denoted as zj = [zj,1, zj,2, · · · , zj,lcj ], where zj,i is the weight of the i-th input of node cj.

depicts the asymptotic weight enumerators of the protographbased (3,12) LDPC code and DGLDPC1. For the (3,12) LDPC code, the asymptotic growth rate of the minimum distance is 0.00206. The asymptotic growth rate of rate-3/4 random codes is 0.04169. Following (9), we calculate the asymptotic growth rate of DGLDPC1 as 0.01. Therefore, without changing the number of nodes and the graph connections, an LDPC code can be modified to achieve an improved asymptotic growth rate of the minimum distance by regarding a group of variable nodes as a different code. We constructed protograph-based (3,12) LDPC code and DGLDPC1 with length 2016 using the PEG method [24]. The weight enumerators of the LDPC code and DGLDPC1 are estimated as 4z12 + 13z14 + 171z16 + 859z18 + 2527z20 + · · · and 4z51 + z52 + z53 + z55 + 5z56 + 4z57 + 5z58 + · · ·, respectively, using the approach in [25][26]. Therefore the estimated minimum distance of these two codes is 12 and 51, respectively. They have a proportion similar to that of their asymptotic growth rate. The WER and BER performances of the protograph-based (3,12) LDPC code and DGLDPC1 are depicted in Fig. 3. We observe that DGLDPC1 has a lower error floor than the protograph-based (3,12) LDPC code with a slight loss in the waterfall region.
DGLDPC2 is constructed as in [28] by using (7,4) Hamming codes for all super variable nodes and (15,11) Hamming codes for all super check nodes. The protograph of DGLDPC2 consists of 15 super variable nodes and 7 super check nodes. The length of DGLDPC2 is 1800. Fig. 4 depicts the asymptotic weight enumerator of DGLDPC2. Its asymptotic growth rate of the minimum distance is 0.06, while that of a random code of rate 8/15 is 0.0993.
