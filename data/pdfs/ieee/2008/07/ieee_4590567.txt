LDPC-Based Advanced FEC for 100 Gbps Transmission
Takashi Mizuochi and Yoshikuni Miyata
Information Technology R&D Center, Mitsubishi Electric Corporation, 5-1-1 Ofuna, Kamakura, Kanagawa 247-8501, Japan, Mizuochi.Takashi@df.MitsubishiElectric.co.jp

Abstract —Practical FEC implementation of 100 Gbps longhaul transmission is discussed. An LDPC code using a novel algorithm is introduced to reduce circuit complexity. Simulation shows that the Q limit is 7.1 dB, and that the concatenation effectively suppresses unwanted error-floor.
I. INTRODUCTION
Recent digital signal processing technologies are quite rapidly driving the transition of high-speed optical transceiver architecture from an analogue-based binary mod/demodulation to a fully digitized multi-level one [1]. Fig. 1 shows a typical block diagram of a 100 Gbps class transponder. The incoming 100GbE client signal is wrapped by an OTU4 framer followed by a forward error correction (FEC) encoder and a symbol mapper, then the signal is multi-level modulated. Before I/Q modulation, the outgoing signal is pre-proccessed by the transmit DSP. The received signal is mixed with a local oscillator in an optical hybrid followed by balanced photodetectors. Each tributary is processed by the receiver DSP. In the FEC decoder, a soft-in/soft-out processor and iterative decoder efficaciously correct any errored bits. A log-likelihood ratio is fed-back to the equalizer embedded in the receiver DSP.

Recently, several types of powerful FEC for use at 10 to 40 Gbps have seen intensive development. For example, a Block Turbo code (BTC) LSI was reported with a Q-limit of 6.3 dB at 10 Gbps [2], and OTN compatible enhanced FEC LSI has been developed with a Q-limit of 9.1 dB at 43 Gbps [3]. However, no FEC LSI applicable to the 100 Gbps region and showing comparable performance has been seen yet. One of the reasons is that it is difficult to implement the algorithms in LSI in a fully parallel way with a practical circuit size. For example, while an infinite number of soft-decision bits would yield ideal correction capability, circuit implementation would be impossible. A near-infinite number of iterations would yield superior error correction performance, but unacceptable latency would result.
In this paper, we discuss the required error correction performance for 100 Gbps class FEC in terms of a net coding gain (NCG). Then we show a low-density parity-check (LDPC) code to be one of the best possible candidate for 100 Gbps FEC.

Simulation shows that the proposed concatenated LDPC + Reed-Solomon (RS) code provides a Q-limit of 7.1 dB without increasing circuit complexity and causing error-floor.

Circuit complexity is a severe problem when we try to implement 100 Gbps FEC in a single LSI chip. The latest LSIs for 40 Gbps made with the 90 nm CMOS process have equivalent gate counts of 30~40 million. For 100 Gbps, at least 3x higher throughput and a higher speed I/O are essential. Therefore use of a 45 nm process and a high-level system design would be necessary. If we can realize the same error correction performance with 1/3 the circuit complexity, the more mature and cheaper CMOS process are mandatory. Therefore, reduced circuit complexity is a very important issue for 100 Gbps class FEC implementation.

III. REDUCED CIRCUIT COMPLEXITY FOR LDPC-BASED FEC
Low-Density Parity-Check (LDPC) codes are being studied intensively as potential candidates for optical communications at 40 Gbps or more. Studies into applying LDPC to optical communications were instituted by Djordjevic et al [4]. The expected merits for LDPC codes are not merely confined to high error correction capability but also fit them for parallelization to reduce circuit complexity. In an LSI for high speed optical communications, ease of parallelized signal processing is essential for practicality.
An ideal LDPC decoder would need hundreds of decoding iterations and a huge circuit to process the numerous operations required each second. The result would be unacceptable latency and a circuit that was impossible to implement. The shuffled belief propagation (BP) algorithm [5] is well known to give remarkable performance, closing in on the Shannon limit. However, the hardware implementation for optical communications can be quite complex, requiring 6 or more soft-decision bits and an enormously complex circuit.
In order to reduce the circuit complexity, we propose a cyclic approximated δ-minimum algorithm, which only requires real additions, comparisons and fewer soft-decision bits [6]. Calculating the LLR of a bit sent from row-parity is an essential operation for an LDPC. However, such a calculation requires a huge amount of memory. To reduce the required memory significantly, we created an approximation of complex recursive function to calculate the minimum function. Calculating complex mathematical functions for BP decoding is a second essential operation for LDPC codes. However, a huge number of operations is necessary to calculate the decoding functions precisely. To achieve a significant reduction in the required number of operations, we created an approximated formula [6]. According to the proposed algorithm, the required number of operations is 10 times smaller than that for the shuffled BP algorithm. The required memory for cyclic approx. δ-min. is 5 times less than that for the shuffled BP algorithm.
IV. COMBATING ERROR FLOOR
In order to reduce the error floor of the LDPC codes, larger girth (~10) is necessary. There are two ways to increase the girth. One is expansion of the codeword length. According to a rough estimate, girths of 6, 8 and 10 are obtained from codeword lengths of 460, 4000 and 20000, respectively. By increasing the codeword length, a possible pattern of the parity check matrix which has only large loops can be discovered. However, longer codewords require increased circuit size. For 100 Gbps class LSI, a codeword length of 20000 is very difficult to implement. Another way to increase girth is increase the redundancy. Regular LDPC codes having redundancies of 3%, 10% and 35% enable their girths to be 6, 8 and 10, respectively. However, larger redundancy causes higher bit rate. In the case of 100GbE, 35% redundancy cannot be allowed because its bit rate would approach 140 Gbps. In slower speed systems, e.g. mobile and wireless, we can use

either of the above methodologies. However, especially for 100GbE, very long codewords and/or large redundancy can never be allowed.
We selected suitable LDPC and RS codes [7]. The total redundancy of the concatenated LDPC+RS is only 20.5%. LDPC(9252,7967) having a redundancy of only 16.1% was selected as the inner code, balancing error correction capability and circuit complexity due to codeword length. For the outer code, we chose RS(992,956), able to correct 18 symbols with only 3.8% redundancy. The additional hardware volume due to the RS code is very small compared to that of LDPC codes. We chose 2-bit soft-decision to reduce circuit complexity.
We compared the error correction performance of LDPC(9252,7967) alone and the concatenated LDPC(9252,7967) + RS(992,956) by Monte Carlo simulation. Fig. 3 shows the simulated pre-FEC Q vs. post-FEC BER for the LDPC code under test. The unwanted error-floor which appeared in the case of LDPC alone at a post-FEC BER of 10-9 is eliminated by concatenation with the RS code, at least down to a post-FEC BER of 10-11. Consequently, we expect that the proposed concatenated codes can achieve a Q-limit of 7.1 dB at a post-FEC BER of 10-13, which is 4 dB better than the standard RS(255,239) code.

V. CONCLUSION
The requirements for 100 Gbps class FEC were discussed. The novel algorithm for the cyclic approximated δ-minimum reduced the circuit complexity to 1/5 without sacrificing the error correction capability. The concatenation with RS code effectively eliminated unwanted error floor. According to the simulation, the proposed LDPC(9252,7967) + RS(992,956) having a redundancy of only 20.5% achieves a Q-limit of 7.1 dB. It is anticipated that the proposed FEC will bring a high possibility of practical LSI implementation for 100 Gbps class long distance transport systems with adequate system margins.
