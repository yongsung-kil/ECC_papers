Iterative Approximate Linear Programming Decoding of LDPC Codes with Linear Complexity

David Burshtein1 School of Electrical Engineering, Tel-Aviv University, Tel-Aviv 69978, Israel

Abstract‚ÄĒ The problem of low complexity linear programming (LP) decoding of low-density parity-check (LDPC) codes is considered. An iterative algorithm for efficient approximate solution of this problem was proposed by Vontobel and Koetter. In this paper the convergence rate and computational complexity of this algorithm are studied. In particular we are interested in obtaining a feasible vector in the LP decoding problem, with objective function value whose distance to the minimum, normalized by the block length, can be made arbitrarily small. It is shown that such a feasible vector can be obtained in linear, in the block length, computational complexity. Combined with previous results, that have shown that the LP decoder can correct some fixed fraction of errors, we conclude that this error correction can be achieved with linear computational complexity.
I. INTRODUCTION
The method of linear programming (LP) decoding of linear codes was proposed by Feldman et al. [1]. There are some close connections between LP decoding and iterative decoding of low-density parity-check (LDPC) codes. However, the LP decoder has some unique attractive properties, including the fact that its performance can be accurately described using pseudocodewords.
One major disadvantage of LP decoding concerns its complexity of decoding. Suppose that we operate below the density evolution threshold. Then under the tree (independence) assumption, which is valid asymptotically for LDPC codes, one can apply a constant (i.e. independent of the block length) number of iterations of belief propagation, using standard parallel scheduling, in a linear in the block length complexity, in order to retrieve the decoded word up to an arbitrarily small fraction of errors. On the other hand, when using a general purpose LP solver for LP decoding of LDPC codes, the computational complexity, although polynomial in the block length, may not be linear in the block length. Several authors have proposed algorithms for low-complexity LP decoding. In this work we focus on the iterative Gauss-Seidel-type algorithm of Vontobel and Koetter [2]. Using general properties of Gauss-Seidel-type algorithms for convex minimization, it was shown in [2] that the algorithm converges. In this paper we study the new approach of [2] under an appropriate scheduling scheme that we propose. We analyze the convergence rate and computational complexity of this algorithm. In particular we are interested in obtaining a feasible vector in the LP decoding problem, with objective function value whose distance to

IV. CORRECTION OF FIXED FRACTION OF ERRORS IN
LINEAR COMPLEXITY
In this section we show that the error correction results in [3] and [4] can be achieved in linear complexity using the approximated LP decoder (23)-(33).
The first step in the analysis is to show that the performance of the algorithm (23)-(33) is independent of the transmitted codeword for any binary-input output-symmetric (BIOS) channel. The detailed proof is provided in [5]. The proof is similar to the corresponding proof for message passing algorithms, and uses induction on the iteration index. Thus without loss of generality we can assume that the all-zero codeword was transmitted.
In the following we consider transmission over the binary symmetric channel (BSC). In this case, by the definition of ő≥i in (2), ő≥i = log((1‚ąíq)/q) if yi = 0 and ő≥i = ‚ąí log((1‚ąíq)/q) if yi = 1, where q the crossover probability. Hence we may re-scale the ő≥i‚Äôs such that ő≥i = 1 if yi = 0 and ő≥i = ‚ąí1

if yi = 1 without effecting the solution produced by the LP decoder.
In [3] it was shown, for left regular LDPC codes which are sufficiently good expanders, that the LP decoder can correct any error pattern with some fixed fraction of errors or less. This result was then combined with the fact that, asymptotically (in the block length), typical codes in the corresponding LDPC ensemble have the required expansion. The results in [3] were recently improved in [4]. Improved constants were obtained by relaxing the requirement that the code should be able to correct any set of some fixed fraction of errors to the requirement that it should only need to correct almost all such error patterns. Our result asserts the following,
Theorem 2: Theorem 1 in [3] and Theorem 1 in [4] remain valid, when the approximate iterative LP decoding algorithm (23)-(33) is applied, instead of the exact LP decoder, with complexity of order O(N ), except that we define success as decoded word with vanishing bit error rate.
The proof is provided in [5]. The proof relies on the proofs of [3, Theorem 1] and [4, Theorem 1], that make the all-zero transmitted codeword assumption, and then prove the success of the LP decoder by showing that a dual witness u exists.
There are various standard possibilities how to correct the remaining errors (i.e., how to augment the decoder so that it would have a vanishing word error rate). These standard methods are also applied in various iterative decoding schemes that guarantee a vanishing bit error rate, but not a vanishing word error rate. One possibility is to use concatenated coding, with asymptotically negligible rate loss. Another possibility [6], [7], [8] is to switch to the flipping algorithm after we have concluded the iterations of the approximate LP decoder (23)-(33) with additional complexity of order O(N ).
