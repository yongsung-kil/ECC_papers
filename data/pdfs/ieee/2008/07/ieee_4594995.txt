Abstract— On the binary erasure channel, the performance of LDPC codes decoded by iterative algorithms is estimated by small-size stopping sets. We have proposed a probabilistic algorithm for computing the minimum size of stopping sets of LDPC codes. In this paper, we analyze the probability and the complexity of finding the minimum-size stopping sets, and give an error probability of determining the minimum size of stopping sets after processing our algorithm. Additionally, we show the numerical results of computing the minimum size of stopping sets of several LDPC codes. In these result, we could compute the minimum size of stopping sets with high reliability.
I. INTRODUCTION
Low-density parity-check (LDPC) codes can achieve the performance close to the Shannon limit by iterative decoding algorithms based on the belief propagation (BP) [1]. LDPC codes and their decoding algorithms are represented by bipartite graphs called Tanner graphs. On the binary erasure channel (BEC), the performance of iterative decoded LDPC codes is dominated by a certain structure in the Tanner graph, which is called a stopping set. C. Di et al. introduced the concept of stopping set and analyzed the average bit and block erasure probability of LDPC codes decoded BP-based iterative algorithm over the BEC [2]. And, the asymptotic stopping set distribution of several LDPC code ensembles has been analyzed [3][4][5]. In these analyses, the average stopping sets of codes in LDPC ensembles are used to drive asymptotic properties of enumerating functions. Because finding small stopping sets in a given LDPC code is difficult combinatorial problem which requires exhaustive search. The problem of determining the smallest size of stopping sets of LDPC codes is conjectured to be NP-hard [6][7].
Recently, X.-Y. Hu et al. presented a method, called probabilistic subspace approach, for estimating the minimum size of stopping sets of LDPC codes [8]. In this method, submatrices are constructed by random choosing some columns of a given LDPC matrix, and the minimum size of stopping sets is exhaustively searched in the submatrices. In [9], a simple algorithm modifying the error impulse (EI) method [10] was presented. In this algorithm, the EI method which generates low-weight codewords by the decoding scheme is applied to finding the small stopping sets. Numerical results obtained by applying these methods to several LDPC codes were reported in [8] and [9]. However, these methods are not necessarily efficient to compute the minimum size of stopping sets, since these methods cannot generate small stopping sets positively.

On the other hand, in [11], E. Rosnes et al. proposed an efficient algorithm to find all stopping set whose size is less than a given threshold. It was introduced by modifying an algorithm presented in [12]. However, determining the minimum size of stopping sets of long random LDPC codes is difficult problem which requires exponentially growing complexity yet.
We have proposed a probabilistic algorithm for computing the minimum size of stopping sets of LDPC codes [14][15]. It is an effective algorithm to generate positively small-size stopping sets as an encoding scheme. In this algorithm, to reduce the complexity for finding small stopping sets, we focus on the relation between small-size stopping sets and lowweight codewords defined by sparse parity-check matrices.
In this paper, we analyze the probability and the complexity of finding the minimum-size stopping sets of LDPC codes by our probabilistic algorithm. And we give a probability of failing to determine the minimum size of stopping sets, called an error probability. The error probability can be use to evaluate the reliability of minimum size of stopping set computed by our algorithm. Additionally, we show numerical results obtained by applying our algorithm to widely-known LDPC codes, including MacKay’s (504, 252), (1008, 504) codes [16], (504, 252) progressive edge-growth (PEG) code [17], and Ramanujan-Margulis (17, 5) code with length 4896 and dimension 2474 [18]. For these codes, we could compute the minimum size of stopping sets with greater reliability than by the conventional method. These stopping sets dominate the performance of these LDPC codes decoded by BP-based iterative algorithms on the BEC.

A stopping set S is a subset of V , the set of variable nodes, such that every neighbor of S is connected to S at least twice.
The size of the stopping set is the cardinality of S. In the message-passing manner, the iterative decoder cannot determine the variable nodes contained in S. On the BEC, the decoding performance of BP-based iter-

sets. Especially the performance is dominated by the stopping
set with minimum size.

We give a probability of failing to determine the minimum

size of stopping set when we determine the minimum size

smin after rmax iterations of our algorithm. The probability is
called an error probability, denoted by εsmin . The events that the stopping set S of size s is found are

independent for each iteration, Thus the probability of failing

to find the stopping set S after rmax iterations is equal to

(1 − τ¯s,ρ,p,l)rmax . Therefore, the error probability that we fail

to estimate the minimum size smin of stopping sets is given

by

εsmin = (1 − τ¯smin,ρ,p,l)rmax .

When a stopping set of smallest size smin is found by our algorithm, we shall say that the minimum size of stopping sets of C is smin with probability 1 − εsmin .

VI. CONCLUSION
In this paper, we analyzed the probability and the complexity of finding small-size stopping sets, and gave an error probability for computing the minimum size of stopping sets by our probabilistic algorithm. Using the error probability, we can evaluate the reliability of the minimum size of stopping set

computed by our algorithm. Additionally, we showed numer-
ical results of computing the minimum size of stopping sets
and its error probabilities using our probabilistic algorithm.
