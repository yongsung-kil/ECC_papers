LP Decoding of LDPC Codes in HARQ Systems
Michael Lunglmayr ∗, Jens Berkmann ∗, Mario Huemer † ∗ Infineon Technologies AG, Am Campeon 1-12, D-81726 Munich/Neubiberg, Germany
† University of Klagenfurt, Embedded Systems and Signal Processing Group, Universitaetsstrasse 65-67, A-9020 Klagenfurt, Austria
{michael.lunglmayr, jens.berkmann}@infineon.com, mario.huemer@uni-klu.ac.at

Abstract— The combination of Low Density Parity Check (LDPC) Codes and Automatic Repeat reQuest (ARQ) has shown to be a promising option to increase the throughput of a communication system. A recently proposed method uses the decoding result to request unreliable bits for retransmission (reliability-based Hybrid ARQ). Commonly, Belief Propagation is used to decode LDPC codes. The result of this algorithm naturally provides an estimate for the probability of a bit to be erroneous. Another possibility to decode LDPC codes is via linear programming (LP) decoding. Despite some drawbacks (e.g. in terms of implementation complexity), linear programming decoding provides an interesting alternative to Belief Propagation. This work investigates the feasibility of combining LP decoding with ARQ. In lack of a probability measure for the decoded bits, when using LP decoding, request criteria using the integrality of the values in the LP solution are proposed. We present statistical investigations on the proposed criteria and the results of throughput simulations demonstrating the gains achievable by the proposed methods.
I. INTRODUCTION
Low Density Parity Check Codes [1] gained a lot of popularity during the last years. Recently, the combination of LDPC Codes (and other iteratively decoded error correcting codes) and Automatic Repeat Request, based on the bit reliability, has received great attention [2], [3], [4]. The result of a soft-in-soft-out (SISO) decoder is used to select unreliable bits for retransmission. This approach is often called reliability-based Hybrid ARQ (RB-HARQ).
Commonly, LDPC Codes are decoded with an iterative algorithm called Belief Propagation [5]. Considerable research was done for reliability-based HARQ with LDPC codes utilizing Belief Propagation decoding. To the best of our knowledge the use of linear programming (LP) decoding of LDPC codes in HARQ Systems has not been investigated so far. Starting with the work of Feldman [6], LP decoding of LDPC codes has become popular. LP decoding has a solid theoretical fundament in optimization theory, such as the maximum-likelihood property: if the result of the linear programming decoder is integer, it is provably the maximum likelihood codeword.
Although, there are still many open issues – such as the decoding complexity – there are promising results using LP decoding methods and its extensions [7]. In this work we analyze the use of LP decoding in conjunction with Hybrid ARQ. We propose methods using the result of the linear programming decoder to request bits from the transmitter. We present simulation results for the proposed methods, showing that notable gains in terms

of throughput can be achieved when using LP decoding with Hybrid ARQ.

II. LDPC CODES
Low Density Parity Check Codes, although already invented 1963 by Gallager [1], gained a lot of popularity during the recent years. LDPC codes are linear block codes with a sparse m×n parity check matrix H. We will only consider binary LDPC codes in this work. A binary linear block code C can be defined by vectors x ∈ {0, 1}n satisfying the equation Hx = 0:

C = {x ∈ {0, 1}n|Hx = 0}.

(1)

As stated before, often an iterative algorithm, called Belief Propagation, is used to decode LDPC codes. This algorithm operates on a graph, the Tanner graph, which can be built out of the parity check matrix. For a detailed description of this algorithm we refer to e.g. [5]. Another possibility for decoding LDPC codes is by solving proper linear programs. Suitable linear programs for linear programming decoding of LDPC Codes (as well as other binary linear block codes) are described in the next section.

III. LINEAR PROGRAMMING DECODING
Starting with the work of Feldman [6] decoding of LDPC Codes with linear programming has shown to be an interesting alternative to Belief Propagation. In general, linear programming decoding seems to be based on a much more solid theoretical fundament than Belief Propagation decoding [6]. But it is to note, that there are also drawbacks when using linear programming decoding. One major concern is the complexity of linear programming decoders or the worse decoding performance at low signal-to-noise ratios (SNR), especially for long codes. Considerable research for solving these issues is in progress [7], [8], [9].

The representations (4) and (5) are integer linear programs. The solution of these integer linear programs has the same time complexity as ML decoding [7]. In the worst case, every valid integer assignment of the variables of (4) and (5) –every codeword– must be examined. Linear programming decoding uses the corresponding linear program as a relaxation of the integer linear program, thus omitting the integer constraints. For both representations the integer constraints of (4d) and (5c) are altered to the linear programming relaxation by using values out of the range [0, 1] instead of out of the set {0, 1}. This allows using methods for solving linear programs for decoding. When the solution of a linear programming relaxation is integer, it provably represents the maximum-likelihood codeword [6]. If the solution of a linear programming relaxation has fractional values, usually a decoding error is reported. With a slight abuse of notation we will also refer to the equations (4) and (5) as linear programs, always keeping in mind that in this case for the last equations (4d) and (5c), respectively, the variables are out of the range [0, 1].
In general, the second representation (5) uses fewer variables than the first, by omitting the indicator variables, but uses more constraints. However, additional slack variables may be used for the second representation. In [6] it is described that the second representation (5) can be interpreted as the first representation (4), projected on the space {0, 1}n. Thus, both representations have the same optimal solution. For the second representation, a recently developed method dramatically reduces the number of required constraints by adaptively selecting only the required constraints depending on the received values [9].

IV. HYBRID ARQ
Many communication systems rely on two mechanisms for error control. One mechanism is the use of a forward error correction code (FEC), such as an LDPC code, the other mechanism is an automatic repeat request (ARQ) strategy. ARQ requests data blocks which could not be decoded correctly. The combination of the two mechanisms ARQ and FEC is often referred to as HybridARQ (HARQ). Recently, a reliability-based approach for HARQ was proposed [2], [3]. The (estimated) bit reliability of the decoding result is used to request unreliable bits. A. HARQ with LP Decoding
Similar to a reliability-based HARQ approach, we use the result of an LP decoder to request those bits from the transmitter which are assumed to be least reliable. Unlike when using Belief Propagation decoding, where the result of the BP algorithm can be used as an estimate of the error probability of the decoded bits [3], for LP decoding no such estimate is present. Due to lack of such a measure for LP decoding we propose to use the integrality of the LP result to evaluate the reliability of the decoded bits. In this context, the integrality of a variable is defined as the property of being integer.
B. System Description
Fig. 1 shows a schematic drawing of an HARQ system with LP decoding. Data bits (d) are encoded at the transmitter with a forward error correcting code (c), here an LDPC code, and then modulated and transmitted (t) to the receiver. The receiver demodulates the received packet (r). If a data block (s) was transmitted the first time, the receiver starts LP decoding. If the received data was requested as a retransmission of unreliable bits, the receiver first combines the received LLR values to the LLR values of previous transmissions (HARQ combining). After decoding, the fractional values of the decoding result are used to determine which bits should be requested. If a data block was decoded correctly, the estimates of the sent data bits (dˆ) are extracted out of the decoding result (x∗) and used for further processing. Two methods to identify which bits should be retransmitted are described in the next section. Here we assume a highly reliable feedback channel to request the bits which should be retransmitted. To reduce the number of feedback bits several strategies, such as source coding [4] or a rowbased approach [3] have been proposed. These strategies can also be applied when using LP decoding.
C. Identification of bits for retransmission
In this section we describe two criteria to identify which bits should be retransmitted. The first criterion uses only the integrality of the local codeword indicators qkl of the linear program (4) to determine which bits should be retransmitted. The second criterion uses the integrality of the bit variables xk of the LP decoding result.

For linear programs with a similar structure than (4), a work recently done in the field of statistical physics [10] describes a partial optimality property. Partial optimality in this context means that, if only parts of the solution of the linear programming relaxation are integer, they also appear in an optimum solution of the integer linear program. This property would especially be beneficial for our approach. Unfortunately, to the best of our knowledge there is yet no proof that partial optimality also holds for linear programming decoding.
To assume that integer values of the LP decoding result are correct is also widely used as a heuristics in branch-and-bound algorithms to improve the performance of linear programming relaxations of integer linear programs [7]. Furthermore, the good quality of this heuristics for reliability estimation is supported by the statistical investigations we present in the next subsection.
For the first method we use only the integrality of the local codeword indicators utilized in the equations (4). This means that a bit is only assumed to be decoded correctly if it is defined by at least one parity check with only integer values in the decoding result, thus a parity check where a local codeword indicator qjl is one. This approach is justified by the fact that, according to the linear program (4), if an indicator qji is set to one, then no other indicator of the same parity check can be set to one (4b), and all bit values of this parity check have to be integer (4c). However, because the integer constraints are omitted in the linear programming relaxation, values between 0 and 1 are also possible for the indicators. In this case the bits of parity checks with fractional indicators are assumed to be unreliable and requested for retransmission.
But, as used in the second method, additional to those bits defined by integer indicators, also bit variables from parity checks with fractional indicators can be integer. The empirical results presented in the following subsection show that using the integrality of bit values xk of the decoding result represents a practically much more useful criterion because there are usually much less integer local codeword indicators than integer bits. The results demonstrate that not only a remarkable part of the bit variables of the decoding result is integer, but also that the major part of those bits has the same values as the originally transmitted bits. Because of these empirical results, as a second method we propose to request those bits whose variables xk of the LP result are non-integer.

Thus, when using the communication system at low throughput areas a request of all bits instead of using an integrality criterion is preferable.
VI. CONCLUSION
In this work we investigated the feasibility of Hybrid ARQ with LP decoding. We showed that the integrality of the LP decoding result can be used as a reliability criterion to request bits for retransmission. We demonstrated that the throughput of an HARQ System with LP decoding can be considerably improved with the presented methods. Further investigations will focus on the comparison with Belief Propagation and other retransmission schemes

(e.g. with puncturing) for HARQ. An investigation on
the influence of the code structure on the performance
when using HARQ with LP Decoding may also be an
interesting research topic for the future.
