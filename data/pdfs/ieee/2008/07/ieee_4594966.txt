A Feedback Belief Propagation Algorithm for LDPC Convolutional Codes

Abstract—A feedback belief propagation (BP) decoding algorithm for low-density parity-check convolutional codes is proposed. The proposed algorithm can activate the variable nodes more efficiently by applying feedback decoding at each decoding iteration. Compared with the on-demand BP algorithm, the proposed algorithm has a doubled convergence speed and causes only about half of the decoding delay at similar error performances without any increase of storage requirement. Simulation results show that the proposed algorithm can offer a good tradeoff between the error-correcting performance and the decoding complexity.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were first proposed by Gallager in the early 1960s [1] and were re-discovered by Mackay and Neal in 1996 [2]. They have received much attention recently because of their capacity-approaching performance and their relative low decoding complexity.
Low-density parity-check convolutional codes (LDPC-CCs), proposed by Felstro¨m et al. [3], can be considered as the counterparts of LDPC-BCs. LDPC-CCs are coming into attention of many researchers mainly due to the comparable performance to LDPC-BCs [3] and the ability of continuous transmission. Additionally, LDPC-CCs can be easily encoded in a systematic way by using shift registers. Recent studies have shown that these codes are more suitable for practical applications in certain communication scenarios than LDPCBCs [4], [5].
Similar to LDPC-BCs, LDPC-CCs can be decoded with belief propagation (BP) algorithm [3]. At each iteration, first all check nodes and then all variable nodes are activated, respectively. Another schedule, named on-demand BP algorithm, was proposed by Pusane et al. [6]. In the on-demand BP algorithm, the check nodes are activated one-by-one and a variable node is activated whenever its message is demanded by a neighboring check node. It has been observed that such an on-demand BP algorithm can reduce the required number of decoding iterations. At each iteration, these two algorithms update variable message by using the variable message from the previous iteration. To update variable message more efficiently, we propose a feedback BP decoding algorithm for LDPC-CCs in this paper. In the feedback BP algorithm, the variable nodes are activated one-by-one and a

check node is activated whenever its message is requested by a neighboring variable node. At each iteration, some variable messages from the same iteration can be utilized to activate the current variable node. The feedback BP algorithm classifies the variable nodes into two classes: the past variable nodes and the future variable nodes relative to the current variable nodes. At each iteration, it uses the variable messages of some past variable nodes at the same iteration and the variable messages of some future variable nodes at the previous iteration to activate the current variable nodes. The efficient utilization of the variable messages from the past variable nodes increases the updating speed of messages. Compared with the ondemand BP algorithm, the feedback BP algorithm can provide a similar error performance with much less decoding delay and lower decoding complexity. Simulation results show that the feedback BP algorithm can offer a good trade-off between the error-correcting performance and the decoding complexity.
The rest of this paper is organized as follows. In Section II, a brief overview of LDPC-CCs is presented. The feedback BP algorithm and its complexity are presented in Section III. Simulation results of LDPC-CCs with the feedback BP algorithm, the BP algorithm and the on-demand BP algorithm are compared in Section IV. Conclusions are drawn in Section V.

Then go back to the Step B. In the iterative process of the decoding, the dependence will
be created among the inputs due to the presence of cycles in the Tanner graphs of the codes. The use of weighting factor can reduce the correlation existing between the inputs in successive iterations [7], [8]. As indicated in (9) and (10), the magnitudes of the LLRs are reduced by multiplying a weighting factor a which is smaller than 1. The best values of a are determined using simulations in order to obtain the minimum bit error rate (BER) at the last iteration.
For each processor, when the two variable nodes and their relative check nodes are activated, the formulas (9) and (10) are operated 1 time and the formula (11) is operated 2J times. There are 2(J − 1) add operations, J(K − 2) + 2 multiply operations, 2J minimum operations and 2J sign operations in all. The computation complexity of each processor for the feedback BP algorithm is similar to that for the on-demand algorithm. Due to the utilization of some variables message from the same iteration, the size of the registers of each processor should be 4(M + 1) − 2, which is about twice as large as that for the on-demand algorithm. Therefore, the storage requirement of each processor for the feedback BP algorithm is about twice as large as that for the on-demand BP algorithm. The total decoding delay for the feedback BP algorithm is (I(M + 1) − 1) information symbols.
IV. SIMULATION RESULTS
Simulations have been conducted for a rate-1/2 (513, 3(2), 5) LDPC-CC constructed according to the method proposed in [3]. Simulation results of this code with the feedback BP algorithm, the BP algorithm and the on-demand BP algorithm are presented in this section.
Fig. 3 and Fig. 4 compare the BER at Eb/N0 = 1.6dB and Eb/N0 = 1.8dB as a function of the number of the iterations for a rate-1/2 (513, 3(2), 5) LDPC-CC. The best value of a is 0.9 for the feedback BP algorithm. Fig. 5 presents the BER as a function of the values of Eb/N0. The number of iterations is set to 40 for the BP and on-demand BP algorithm. For the feedback BP algorithm, the numbers of iterations are set to 20 and 40. It can be seen that the feedback BP algorithm significantly outperforms the BP algorithm and the on-demand BP algorithm.
Simulation results show that the error-correcting performance provided by the feedback BP algorithm with 20 iterations is similar to that provided by the on-demand BP algorithm with 40 iterations. Therefore, the decoding hardware spending can be substantially decreased. Compared with the on-demand BP algorithm, the feedback BP algorithm has a doubled convergence speed. Based on the complexity analysis in Section III, we can see that the feedback BP algorithm produces only half of the total decoding delay caused by the on-demand BP algorithm without any increase of the total storage requirement.
