Introduction Different forward error correction (FEC) schemes suitable for use in optical communications systems can be classified into three different generations [1]: (i) hard-decision FEC schemes, (ii) concatenated codes, and (iii) soft iteratively decodable codes, such as turbo-product codes [1], and low-density parity-check (LDPC) codes [2]. LDPC codes are able to match and outperform turbo-product codes in terms of coding gain and decoding complexity [2]. Semi-random LDPC codes can be designed to closely approach the Shannon limit [3]. However the complexity of encoders is prohibitively high. The LDPC codes designed using algebraic, geometry or combinatorial concepts [4]-[8], sometimes called “structured” LDPC codes, were considered as possible alternatives to semi-random LDPC codes. The parity-check matrix of those codes can be put in cyclic or quasi-cyclic form leading to low–complexity encoder implementations based on shift-registers and modulo-2 adders. Moreover, they can be decoded using the low-complexity min-sumwith-correction-term algorithm [9], and as such are excellent candidates for future high-speed optical communication systems [2].
In this paper we are concerned with high-rates (R>0.9) structured-LDPC codes, and their application in turbo equalization to simultaneously suppress fiber nonlinearities [10], and compensate chromatic dispersion [11] and PMD [12]. We design four different LDPC codes, belonging to the following three classes: (i) LDPC codes designed from balanced incomplete block designs (BIBDs) [13], (ii) blockcirculant (array) LDPC codes, and (iii) semi-random MacKay codes. We experimentally demonstrate that those codes provide excellent coding gains, and are suitable for use in PMD compensation by LDPC-coded turbo equalization.
High-rate LDPC codes and PMD compensation by turbo equalization As mentioned in Introduction we design four different LDPC codes using three different approaches. The first approach is based on concept of BIBD [13]. A balanced incomplete block design, denoted as BIBD(v,b,r,k,λ), is a collection of subsets (blocks) of a v-set V with a size of each block being k, so that: (i) each pair of elements (points) occurs in exactly λ of the blocks, and (ii) every symbol occurs in exactly r blocks. If 20t+1 is a prime or prime power and θ is a primitive root of GF(20t+1), then the following t initial sets Si={θi,θ4t+i,θ8t+i,θ12t+i,θ16t+i} (i=0,2,…,2t-2) form a BIBD(20t+1, t(20t+1), 5t, 5, 1). The BIBD is formed by adding the elements from GF(6t+1) to the initial blocks Si. By observing the elements from BIBD blocks as position of ones in corresponding columns of a parity-check matrix (H-matrix), the code rate of an LDPC code such obtained is lower bounded by R≥1-1/t, and the codeword length is determined by b=t(20t+1). For example, for t=20 an LDPC(8020,7619) code of rate R=0.95, girth 6, and column weight 5 is obtained. (The girth is shortest cycle in corresponding bipartite representation of a parity-check matrix.)

Using this approach the following two codes of column weight 3 and rate R=0.906 are designed: LDPC(11936,10819), and LDPC(15328,13893).
The third approach is to design H-matrix semi-randomly [3]: H-matrix is generated with weight-wc columns, weight-wr rows, no two columns having overlap larger than one, and 4-cycles are avoided. Using this approach we design the LDPC(4376, 4095) (MacKay) code of rate R=0.936 and column weight 4.
To take full advantage of iterative decoding soft bit information/log-likelihood ratios (LLRs) are needed. To provide the log-likelihood ratios for every bit a Bahl-Cocke-Jelinek-Raviv (BCJR) equalizer is used, as we described in [10]-[12]. The equalizer operates on a dynamic discrete trellis, which models the channel as a nonlinear ISI channel with memory. If Y and X are the received the transmitted sequences, respectively the state is determined by s=(xi-m,..,xi,xi+1,…,xi+m) (xi∈X), and the channel is completely described by the conditional probability P(Y|X), obtained using training sequences. The set of triples: previous state, channel output, next state uniquely determines the trellis in time. The memory assumption for the channel indicates that the observed bit xi is affected by the preceding m (xi-m,xi-m+1,.., xi-1) bits and next m (xj+1,…,xj+m) bits in the transmitted sequence. The BCJR equalizer and the LDPC decoder iteratively exchange extrinsic information to determine the most likely transmitted sequence.
To evaluate the high rate codes introduced here, we perform a set of experiments in which LDPC codes are used as component codes for PMD compensation by turbo equalization, the concept we introduced in [12].

Experimental setup and experimental results The setup used in this experiment is shown in Fig. 1(a). The personal computer (PC) with GPIB card was used to load the test sequence into a pulse pattern generator (Anritsu MP1763C), and acquire sequences from oscilloscope (Agilent DCA 86105A). NRZ signal at 10Gb/s was used to drive a Mach-Zehnder optical modulator (JDSU). Controlled amount of DGD was introduced by PMD emulator (JDSU PE3), and PMD distorted signal was mixed with an ASE source through a 3dB coupler. Launch power was maintained at 0dBm, and different values of OSNR were achieved by controlling the power of the ASE source with variable optical attenuator before coupling. The optical signal was then pre-amplified (Optigain, Inc. 2000 series), filtered (JSDU 2nm band-pass filter), and converted to electrical domain using an optical detector (Agilent 11982A). Constant received power level of -6dBm was maintained by a variable optical attenuator.

In Fig. 2(b) four different LDPC codes proposed here are compared in terms of BER at DGD of 125ps. BIBD based LDPC code outperforms semi-random code at BERs below 10-4. Block-circulant LDPC code of length 11936, outperforms semi-random code rate by about 0.75 dB at BER of 10-6, while LDPC code of
length 15328 outperforms the semi-random code for almost 1.75 dB at the same BER. The LDPC code of
length 15328 is about 1.25 dB away from the channel capacity limit.

Conclusions The four different LDPC codes of rate above 0.9 are designed using one of three following concepts: BIBD based LDPC codes, semi-random LDPC codes, and block-circulant LDPC codes. The proposed LDPC codes are experimentally evaluated in the presence of PMD. For DGD=125 ps (and NRZ at 10-Gb/s) BIBD based LDPC code outperforms semi-random LDPC code at BER below 10-4, while block-circulant LDPC code of length 15328 outperforms the same code by even1.75 dB. Block-circulant LDPC(15328,13893) code is only 1.25 dB away from the channel capacity limit.
