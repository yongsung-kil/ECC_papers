Design of Systematic LDPC Codes Using Density Evolution Based on Tripartite Graph

Jun Ning and Jinhong Yuan School of EE&T, University of New South Wales, NSW 2052 Australia
Emails: {j.ning, j.yuan}@unsw.edu.au

Abstract—In this paper we consider the design of systematic low-density parity-check (LDPC) codes using density evolution. We show that systematic LDPC codes can be well represented by tripartite graphs, where the edges in the code graph are divided into two types, source edges and redundancy edges. Using the tripartite code graph representation, we consider the degree distributions for source edges and redundancy edges separately. In particular, we discuss the necessary conditions for the two types of edges to be viewed as a single type in terms of specifying the LDPC codes properly. Then we discuss the relationship between the conventional bipartite graph representation and the tripartite graph representation. For the design of systematic LDPC codes, we show that the tripartite representation can specify the ensembles that are not available for the bipartite representation. Furthermore, we show that codes optimized by deploying the tripartite representation can achieve better performance with respect to the bipartite representation.
I. Introduction
For binary-input memoryless channels, very long lowdensity parity-check (LDPC) codes exhibit a threshold phenomenon in their decoding performance: reliable transmission is possible only if a certain channel parameter, i.e., noise level, is smaller than a threshold, δ∗ [1]–[3]. Density evolution is a numerical technique, developed by Richardson and Urbanke [3], [4], to determine the threshold δ∗ to any desired accuracy, by tracking the evolution of message distribution under beliefpropagation decoding.
To employ density evolution for performance evaluation, one must specify a code in a proper code representation. Tanner [5] showed that the parity check matrix of an LDPC code can be effectively and uniquely represented by a bipartite (or Tanner) graph G. Conventionally, LDPC codes are specified by a degree distribution pair (λ, ρ), where λi or ρ j is the fraction of edges incident to a variable node of degree-i or a check node of degree- j, respectively. In the sequel, we refer to above conventional degree distribution pair (λ, ρ) as the bipartite representation of the LDPC codes. Kasai, Shibuya and Sakaniwa [6] showed that LDPC codes optimized by density evolution with a more detailed representation based on the bipartite representation can reach a higher threshold. To accommodate various code structures including concatenated tree (CT) codes, irregular repeat accumulate (IRA) codes and LDPC codes with puncturing mechanism, Richardson and Urbanke introduced a multi-edge type code representation in [7]. They showed that multi-edge type LDPC codes can have a high threshold with low variable node degrees. From the multi-edge type representation point of view, the bipartite

representation is simply its special case when the type of edges is one.
Unlike a code graph, the bipartite representation or the multi-edge type code representation specifies an ensemble of codes with identical degree distributions but equiprobable random edge connections. For a code ensemble, since all edges in the same edge type are statistically equivalent and interchangeable, density evolution tracks the probability density functions of messages passed between variable nodes and check nodes by considering an edge randomly chosen from the each set of the same type edges. In other words, at any iteration of the evolution, the density of the message passing along any edge is statistically averaged over all the possible connections specified by the degree distributions for the same type of edges. The threshold δ∗ evaluated by density evolution represents the predicted asymptotic decoding performance of an ensemble of codes specified by the code representation rather than that of a particular code.
The above code representations have not taken into account the natural systematic structure of the codewords. LDPC codes are normally in systematic form and each codeword of them consisting of two parts: information (or source) bit part and redundancy bit part. Without loss of generality, we can assume that the total m check constraints of a systematic LDPC code can be divided into two sets, Pc and Ps. The former set contains pc elements and each element involves only redundancy bits. The latter set has m − pc elements and each element has at least one source bit involved directly. If Pc is empty or pc = 0, we define the family of the corresponding LDPC codes as fully-source-involved LDPC (FSI-LDPC) codes. According to the codeword structure, we define a variable node as a source or redundancy node, if the variable node corresponds to a source or redundancy bit in the codeword, respectively. Then the edges emanating from source or redundancy nodes are called source or redundancy edges, respectively.
In [8], we proposed a tripartite representation based on tripartite graph. In a tripartite graph, we divide the edges of an ensemble of the systematic codes into two types, source edges and redundancy edges, following the nature of codeword structure. Thus we consider the degree distributions for source edges and redundancy edges separately. Unlike the tripartite graph proposed in [9] where the redundancy part is of a simple zig-zag form, the tripartite graph here admits a general form for the redundancy part whose degree distribution can

We showed in [8] that for certain code parameters, e.g., code rate and maximum variable node degree, source edges and redundancy edges have different degree distributions and thus cannot be regarded as having same statistical characteristics. In this paper, we consider the design of systematic lowdensity parity-check (LDPC) codes using density evolution based on the results in [8]. We first extend the results on the applicability of the bipartite graph in [8] to general systematic LDPC codes. Then we discuss the relationship between the conventional bipartite representation and the tripartite representation. For the design of systematic LDPC codes, we show that the tripartite representation can specify the ensembles that cannot be described properly by the bipartite representation. We prove and verify that it is possible to optimize the codes performance further by using the tripartite representation with respect to the bipartite representation. These are the major contributions of this paper. To show the advantage of differentiating source and redundancy edges for the design of systematic LDPC codes in this paper, we use the tripartite representation rather than the multi-edge type representation mainly due to three factors. Firstly, the tripartite representation can represent systematic LDPC codes in a natural way. Secondly, the applicable range of the multiedge type representation depends on how the edges are typed and will not be discussed here. Finally, we can prove the advantage of the tripartite representation over the widely-used bipartite representation in code optimization.
This paper is organized as follows. Section II introduces both the conventional bipartite representation and the tripartite representation. Also in this section, we discuss the applicable range of the bipartite representation. Section III investigates the relationships between the bipartite representation and the tripartite representation. Section IV shows the advantage of using the tripartite representation in code optimization. Section V gives some LDPC code ensembles optimized with the tripartite representation. Section VI presents the conclusions.

From Lemma 1 and 2 we see that if conditions (2) and (3) are satisfied the pair (λ, ρ) and its corresponding quadruplet (λ, ρ, α, β) are equivalent from the point of view of density evolution. Thus, we define the ensembles specified by the (λ, ρ) and the (λ, ρ, α, β) as density evolution equivalent ensembles. We see that a set of tripartite ensembles Cn(λ, ρ, α, β) can be generated from an bipartite ensemble Cn(λ, ρ) by splitting the latter according to different α and β. Given that inequalities (2) and (3) are satisfied, there is one ensemble Cn(λ, ρ, α, β) equivalent to the ”mother” ensemble Cn(λ, ρ). Conversely, any tripartite ensemble Cn(λ, ρ, α, β) can form a bipartite ensemble Cn(λ, ρ) by merging the two types of edges. However, if (2) and (3) are not satisfied, the (λ, ρ, α, β) cannot be converted into an equivalent (λ, ρ) with the same threshold. Clearly, in the merging the difference between the two sets of variable edges
is averaged. Therefore, the tripartite graph representation has
a wider applicable range.
The code design based on density evolution is to search degree distributions yielding the maximum threshold δ∗. As
the optimization is carried out over a large number of possible (λ, ρ) or (λ, ρ, α, β), we introduce some definitions as follows.

following pairs (λ, ρ) satisfy conditions (2) and (3). Let
A = {(λ, ρ)} denote the set of all pairs (λ, ρ), B = {(λ, ρ, α, β)}
denote the universal set of all possible elements (λ, ρ, α, β) under the applicable constraints, Bˇ = {(λ, ρ, α, β)} denote a proper subset of B, i.e., Bˇ ⊂ B, where for each element (λ, ρ, α, β) ∈ Bˇ there exists a density evolution equivalent element (λ, ρ) ∈ A and vice versa, and Bˆ denote the absolute complement set of Bˇ and Bˆ ∪ Bˇ = B.

VI. Conclusion
In this paper we consider the design of systematic lowdensity parity-check (LDPC) codes using density evolution based on tripartite graph. We study the degree distributions for source edges and redundancy edges separately based on tripartite graph. In particular, we discuss the necessary conditions for the bipartite representation to specify the LDPC codes properly. Then we discussed the relationship between the conventional bipartite representation and the tripartite representation. For the design of systematic LDPC codes, we showed that under certain conditions, the set of the systematic LDPC ensembles specified by bipartite representations is a subset of the ensembles specified by tripartite representations. Thus the ensembles optimized with tripartite representation outperform those with bipartite representation. This advantage of the tripartite representation is verified by numerical results.
