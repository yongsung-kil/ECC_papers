1. Introduction
Low-density parity check (LDPC) code was introduced by Gallagher [1] in 1962. It is a kind of binary linear block code whose parity check matrix is sparse which has much fewer l's than a common matrix. A sparse parity check matrix facilitates simple decoding algorithms and has low complexities in decoder designs.
There are two kinds of LDPC code, one is regular, and the other is irregular. If a mxn parity check matrix H has the same weight r and c in each row and column, it can be defined as a (n ,k ,r ,c)-regular LDPC code, and k is the number of parity bits. Otherwise, an LDPC code is said to be irregular. Check matrix of a LDPC code is often
represented by a bipartite graph, called Tanner graph [2],
which is composed of n variable nodes and m check nodes. Those variable nodes and check nodes are connected by edges defined by nonzero entries of matrix H. Figurel

Tanner graph shows a clear picture of all the infornation exchange links in a decoding process. The most timeconsuming operations in LDPC decoders are the comparison operations required in the check nodes. In the literature [3,4,5], there are some hardware realizations of LDPC decoders. A conventional CNU architecture [6] has a long delay time and is always the critical path of the decoder. Hence, [7] proposes a more efficient 6-input CNU architecture to increase the speed of the whole decoder. However, those designs are not specifically optimized for the efficient comparison operations. To address this design challenge, this work is aimed to achieve the fastest comparison speed with lowest comparator complexity as much as possible.

This paper presents a realization algorithm for CNU structure which can accept any number of inputs. The proposed technique is very close to the optimal solution and always has the shortest propagation time.
3. The Proposed architecture of CNU
In order to propose CNU architecture which suits for any inputs number, this paper suggests a new algorithm, called "Cyclic comparison algorithm". This algorithm postulates the shortest critical path and therefore reduces the number of comparator units as few as possible. To generate outputs in
Flog2 parallel, at least (n - 1)] comparators layers are
needed in a CNU, where n is the number of input operands to be compared. Subject to the minimum numbers of
comparators layers, this paper proposes a general
comparator architecture for CNU. Moreover, the proposed algorithm also can be used in VNU operation by replacing comparator units to two-input adder units. The detailed description ofthe algorithm is introduced below.

5. Conclusion

This paper presents a comparison algorithm for CNU/VNU, which generate the architecture of any outputs CNU/VNU by a novel systematic optimization algorithm, called cyclic comparison algorithm. The proposed design works well and can be process any input number. For very long LDPC code lengths, the proposed design techniques can provide close to optimal solutions. Finally, this algorithm can give the designers reference results to help them further improve their designs when they consider the hand-crafted structures.
