PERFORMANCE UNDER DELAY CONSTRAINTS ON ITERATIVE DECODING OF LDPC CODES
JaWone Kennedy and Daniel L. Noneaker (corresponding author)
Holcombe Department of Electrical and Computer Engineering 305 Fluor Daniel Engineering Innovation Building Clemson University, Clemson, SC 29634

ABSTRACT
The performance of a system with a quasi-cyclic lowdensity parity-check code is examined under various constraints on the decoding delay. Two alternatives are considered for the message-passing decoding algorithm: the sumproduct algorithm and the turbo-decoding message-passing algorithm. It is shown that their relative performance depends heavily on the stringency of the delay constraint; the TDMP algorithm results in substantially better performance
than the SPA if the constraint is stringent.
INTRODUCTION
Low-density parity-check (LDPC) codes enjoy widespread use in recently developed wireless communication systems. Good performance is achieved with reasonable decoding complexity using a well-designed LPDC code in conjunction with any of several iterative message-passing algorithms. Any implementation of a message-passing algorithm must limit the number of iterations used to decode a single code block or a sequence of code blocks. The limit is determined by the capabilities of the receiver's processor, the data rate of the system, the computational load imposed by each iteration of the decoder, and the nature and computational demands of the other tasks executed by the processor.
The performance of a system using an LDPC code and iterative message-passing decoding is usually characterized in terms of the probability of error that is achieved in a given channel when the number of iterations of the decoding algorithm is sufficiently large to approach its limiting performance. In some applications, however, the constraints on decoding delay are such that the algorithm can not employ such a large number of iterations. In these instances, the capability of the decoder should be characterized within the context of the system's decoding constraints.

In this paper, we compare the performance of two iterative message-passing algorithms under various decoding constraints. One algorithm is the well-known sum-product algorithm (SPA) [1], which uses a flooding schedule for message passing [2]. The other is the turbo-decoding message passing (TDMP) algorithm [3], [4], [5]. (The TDMP algorithm is also known as "shuffled belief propagation" [4].) The TDMP algorithm utilizes a different messagepassing schedule than the SPA [3]. The TDMP algorithm is naturally suited to partially parallel implementation [6] for "architecture aware" LDPC codes (including some quasicyclic (QC) LDPC codes) [3].
Nearly the same performance is approached by the TDMP algorithm and the SPA as the number of iterations is increased for each, but the TDMP algorithm arrives close to the limiting performance in fewer iterations than the SPA. Previous comparisons of the performance of the two algorithms have addressed that difference [3], [7]. In this paper, we focus instead on the performance of the two algorithms under a constraint on the decoding delay in which the decoder is not necessarily permitted enough iterations to approach its limiting performance. The algorithms are considered for a system employing Sridhara-Fuja-Tanner (8FT) codes [8], which are QC LDPC codes that use circulant matrices for fast encoding.

ITERATIVE MESSAGE-PASSING ALGORITHMS
The SPA and the TDMP algorithm are examples of iterative message-passing algorithms defined on a bipartite graph (a Tanner graph) determined by a low-density paritycheck matrix of the LDPC code [1]. A parity-check matrix H with M rows and N columns determines two kinds of graph nodes: N variable nodes that correspond to the bits in a codeword, and M check nodes that correspond to the parity checks (each represented by a row of H). The graph displays the relationship between the code symbols and the parity-check sums that determine a valid code word.
The SPA and the TDMP algorithm differ in the messagepassing schedule used to update the values held by variable nodes within each iteration. A flooding schedule is used in the SPA in which all check-node values are processed before the variable nodes are updated [2]. In contrast, the TDMP algorithm updates the variable nodes after each check node is updated [7]. Consequently, the two messagepassing algorithms can result in different decoding outcomes for a given channel output, and they yield different decoder performance in general.
The use of m x m permutation submatrices in the paritycheck matrix of an SFT code results in no sharing of variable-node values in the corresponding m check-node updates; thus the TDMP algorithm is naturally suited to partially parallel implementation with concurrent updates of m check nodes. We consider only serial implementation of the decoders in the comparison of performance results in this paper, however.

DELAY-CONSTRAINED PERFORMANCE OF THE SPA AND THE TDMP ALGORITHM
The performance of the SPA and the TDMP algorithm is compared in this section for different constraints on the decoding delay. Decoder performance is measured by the probability that the decoder fails to decode the data block correctly within the maximum allowable number of iterations (which is referred to simply as the probability of block error). The computational cost of using each algorithm to detect a block of data is measured by the number of iterations of the algorithm. The use of the SISO algorithm discussed in the previous section for the TDMP algorithm results in a computational cost per iteration that is the same for the SPA and the TDMP algorithm. Thus the number of iterations serves as a fair basis of comparison of the decoding delay incurred with the two iterative message-passing algorithms if they are implemented using serial processing. (Implementation of the TDMP algorithm using the BCJR-based SISO algorithm instead may result in a different computational cost per iteration for serial processing.)
Three SFT codes are employed in examining the effect of the decoding-delay constraint on the relative performance of the two decoding algorithms. They are a (305,124) code, a (1055,424) code, and a (2105,844) code. The rates of the three codes are 0.4065, 0.4019, and 0.4009, respectively; thus they have nearly identical rates but different block lengths. They are referred to in the subsequent discussion as "the short code," "the intermediate-length code," and "the long code," respectively. The parity-check matrix for the short code is constructed from circulant submatrices of size m == 61. The circulant submatrix size for the intermediatelength code is m == 211, and for the long code the size is m == 421. The parity-check matrix for each code has a row weight of three and a column weight of five.

The convergence of both the SPA and the TDMP algorithm with the short code is illustrated in Fig. 1. The figure shows the probability of block error as a function of the signal-to-noise ratio for each algorithm with limits of one, five, and 50 iterations in the decoding attempt for each block. The two algorithms clearly converge to nearly the same error probability as the number of maximum decoding iterations increases, and the performance of the two is very close if each decoder is allowed to perform up to 50 iterations for a block of data. As noted in [7], the TDMP algorithm converges to the limiting performance more quickly than does the SPA.
The performance of the two decoding algorithms with the short code is shown in Fig. 2 for circumstances in which the decoder operates under a hard real-time constraint that is specified by the maximum number of decoder iterations that can be used to detect any block of data. Specifically, the performance is shown as the signal-to-noise ratio required to achieve a target probability of block error as a function of the maximum number of decoder iterations. Results are shown for two values of the target block error probability: 10-2 and 10-3• (Results for a target probability of block error of 10-1 are not shown; they yield similar comparisons to the results that are shown.)
Suppose the target probability of block error is 10-2 for a system using the short code. If the maximum number of iterations per data block is large (e.g., 50), both the SPA and the TDMP algorithm approach their limiting performance. A signal-to-noise ratio of 2.5 dB is required to achieve the target performance with the TDMP algorithm; the performance is 0.1 dB poorer if the SPA is used instead. If a more severe real-time constraint is placed on the decoder, however, the TDMP algorithm results in much better performance than the SPA. The signal-to-noise ratio required to achieve the target performance is 0.3 dB lower for the TDMP algorithm than the SPA if the decoder is limited to ten iterations per data block. The difference in performance increases to 0.8 dB if the maximum number of iterations is five, and it is 1.8 dB with a limit of two iterations.
The relative performance of the two decoders as a function of the maximum number of iterations remains the same if the target block error probability is decreased to 10-3 for a system with the short code. The difference in performance ranges from nearly 2 dB if there is a very stringent realtime limit to only a small difference if a large number of iterations is allowed. Note that for an iteration limit of six or less, use of the TDMP algorithm allows the system to achieve a block error probability of 10-3 with a lower signal-to-noise ratio than is required to achieve a block error probability of 10-2 if the SPA is used.

CONCLUSION
In this paper we have examined the effect that a constraint on the decoding delay has on the perfonnance of a system using LDPC codes. Two iterative messagepassing algorithms are considered: the SPA and the TDMP algorithm. It is shown that the relative perfonnance of the two algorithms depends on the stringency of the constraint; the TDMP algorithm substantially outperfonns the SPA when the number of decoder iterations is limited to a small number. The difference in perfonnance can exceed 2 dB.
