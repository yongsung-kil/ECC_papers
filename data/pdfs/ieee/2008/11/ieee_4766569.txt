Improved Performance Irregular Quasi-Cyclic LDPC Code Design from BIBD’s using Threshold Minimization

Siddarama R. Patil1 and Sant S. Pathak2 , Senior Member, IEEE
Department of Electrical and Electronics Communication Engineering, Indian Institute of Technology, Kharagpur, India 1siddarama@iitkgp.ac.in, 2ssp@ece.iitkgp.ernet.in

Abstract-In this paper we propose a novel method for designing irregular quasi-cyclic low-density parity-check (LDPC) codes from Balanced Incomplete Block Designs (BIBD’s). The design method hinges around finding the optimum degree profile by minimizing the threshold using density evolution. The approach for designing short block length codes is robust for practical implementation and has been found to exhibit considerable performance gain that may be attributed to a good degree profile for the codes. The design takes into consideration the code rate and code length as variable parameters. The simulation results for the designed codes are compared with regular and irregular quasi-cyclic BIBD based LDPC codes and found a relatively higher performance in terms of BER.
I. INTRODUCTION
The most important feature of low-density parity-check (LDPC) codes is their ability to achieve a significant fraction of the channel capacity using belief propagation (BP) iterative decoding at relatively low implementation complexity and this have made the LDPC codes a probable candidate for next generation communication system [1]. There still exist many open questions regarding the exact influence that specific parameters of a code exhibit on its overall performance when used for finite-length data transmission.
An LDPC code is specified in terms of a sparse N × M parity check matrix H whose elements are 0 and 1. The rows of parity check matrix H represents a set of linear homogeneous modulo 2 equations called parity check equations, and the set of code words is the set of solutions of these equations. A (N, k, j) regular LDPC code is a code of
length N having H with j 1’s in each row and k 1’s in
each column. A Tanner graphs is used to shows the relationship between codeword bits and parity-checks of an LDPC code. Each of the N code bits and M parity-checks in H are represented by a node in the graph. The columns of H are represented by one set of nodes called as variable nodes and the rows of H are represented by another set of nodes called as check nodes in the corresponding Tanner graph. A graph edge joins a code variable node to the nodes of the parity checks that include it. The degree of a variable node is the number of check equations that it participates in. Similarly, degree of a check node is the number of variable nodes which take part in that particular check. For the case where all variable nodes have the same degree, k , and all the

check nodes have same degree, j , then it is a regular LDPC

code. These degrees are different for irregular LDPC codes.

For the irregular LDPC codes the irregularity is specified using two polynomials called variable node and check node degree profiles or degree distributions, and are given in (1a) and (1b) respectively.

Based on methods of construction, LDPC codes can be divided into randomly generated codes and structured codes. Randomly generated LDPC codes are constructed by computer search based on certain design rules or graph structures, such as girth and degree profiles of nodes [2] [3], while structured LDPC codes are constructed using algebraic and combinatoric methods. The problems of random LDPC codes such as encoder complexity due to matrix multiplication used for encoding, decoder complexity as one has to store all
the position of 1’s in the parity check matrix H and complex
routing requirement of edges which are randomly distributed in the hardware implementation are solved to a great extent by structured LDPC codes. Several classes of structured LDPC codes are known, including those based on finite geometries [4], combinatorial designs such as balanced incomplete block designs (BIBD) [5] and mutually orthogonal Latin squares [6]. Most of these construction techniques produce cyclic or quasicyclic (QC) codes, which can be encoded by a shift-register implementation. All these constructions are typically for regular LDPC codes that lack the capacity-achieving ability of irregular codes.

In [2] Richrdson et. al. introduced a method of designing capacity approaching irregular LDPC codes by finding the optimized degree distribution called Density Evolution. In this paper we find an answer for the problem of finding a good CWD. We discuss a novel method of designing and improving the performance of irregular quasi-cyclic LDPC codes from BIBD’s. In our design method we use Density Evolution for designing an optimum CWD by minimizing the threshold. Once a good CWD is available, then one can find the desired code by constructing a suitable parity check matrix H constrained to the optimized CWD.

II. LDPC CODE DESIGN USING BIBD
In the following, we first briefly discus the concept of BIBD’s, introduced by Bose using the method of symmetrically repeated differences (SRD) [9] [10].
A BIBD with parameters (v, k, λ) is an ordered pair (V , B) ,
where V is a v -element set and B is a collection of b k − subsets of V , called blocks, such that every element of
V is contained in exactly r blocks and every 2-subset of V is contained in exactly λ blocks. It is readily seen that the integers v, b, r, k, λ satisfy the equations

A BIBD can be completely represented by its v × b incidence matrix, Q , with entries in GF (2) . The rows of Q corresponds to the v objects in V ; the columns of Q corresponds to the b blocks of the design. The entry in the ith

row and the jth column of Q is ‘1’ if the ith object is
contained in the jth block. Otherwise the entry is ‘0’. It follows from the structural properties of BIBD that the incidence matrix, Q , has constant row weight, r , and constant column weight, k , and the maximum number of common ‘1’s between any tow columns is λ .
R. C. Bose [9] gave constructions for certain classes of (v, k,1)-BIBD in the additive group of GF (v) for k = 3, 4, 5, and these were subsequently generalized to other values of k by R. M. Wilson [11]. Here we are considering two of Wilson’s generalizations discussed by Chung et. al. in [12] for the construction of different BIBD’s. The objective therein has been to find larger number of low correlation blocks for optical orthogonal codes (OOC) design.

III. TMCWD ALGORITHM BASED IRREGULAR QC LDPC CODES
Density Evolution is an analytical technique which has been used to understand limits of performance of LDPC decoders. It also provides a tool which can be used in the design of families of LDPC codes, since their performance can be predicted using density evolution much more rapidly than the performance can be simulated. The Density Evolution introduces the idea of a channel threshold, above which the code performs well and below which the probability of error is non-negligible. This provides a single parameter characterizing code performance which may be used to gauge the performance compared to the ultimate limit of channel capacity.
For a check node with degree dc , each outgoing message L can be expressed by a function of dc −1 incoming messages with one incoming message along the same edge as the outgoing message being excluded. Then L is given by

L = φc (Z1, Z2 , … , Zdc −1)

(15)

where φc (i) depends on the decoding algorithm and Z1, Z2 ,… , Zdc −1 messages from variable nodes. Similarly, for
a symbol node of degree dv , each outgoing message Z can be expressed by

Z = φv (F , L1, L2 , … , Ldv −1)

(16)

where φv (i) depends on the decoding algorithm, {Li} are dv −1 incoming messages, and F is the a priori Log Likelihood Ratio (LLR) value of the bit. The density evolution typically assumes a loop-free graph and all zero code word is transmitted. Hence, all messages entering one node are independent and identically distributed. The key problem is to calculate the probability density function (pdf) of the messages of the current iteration based on the pdf of the

messages from the preceding iteration. For the simplicity of analysis, it is assumed that all messages are Gaussian and symmetric. The density evolution analysis tracks the mean of these Gaussian random variables through the decoding process for some values of Eb No , the mean converges to a small fixed point. The Gaussian pdf it represents would thus have both positive and negative outcomes, meaning that the messages represented by this distribution could have negative values, or that there is a non negligible probability that there are decoding errors. On the other hand, for some values of Eb No , the mean tends to infinity. The pdf has all of its probability on positive values. Thus, for a sufficiently large number of iterations the decoder would decode correctly.

Steps involved in the calculation of threshold in Eb No 1. Set Eb No to a small value 2. Run the Density Evolution for a number of iterations
(set to Imax) for a given Eb No 3. If mean is not increased to a large value with iteration
then increase Eb No by small step. 4. Continue this till mean of the messages increases to a
large value (set in the algorithm) with iteration
5. Take the corresponding Eb No as threshold.

V CONCLUSION
A systematic and practical approach for designing short block length irregular quasi cyclic LDPC codes from BIBD’s having a good degree distribution specifically for a given code rate and code length is developed. The algorithm converges to the best CWD which is having a minimum threshold. Simulation result shows that the designed codes are having considerable coding gain compared with regular quasi cyclic BIBD based LDPC codes. If the maximum variable node degree is increased, it is possible to get still better codes by this method.
