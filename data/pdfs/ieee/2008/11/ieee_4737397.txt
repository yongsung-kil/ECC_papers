Compressing Construction of QC-LDPC Codes

Shuqi Sun

Wuyang Zhou

shqsun@mail.ustc.edu.cn wyzhou@ustc.edu.cn

Wireless Information Network Lab

Abstract-- Two compressing construction methods of Quasi-Cyclic LDPC (QC-LDPC) codes are proposed, which can be called recursive-compressing and single-compressing respectively. Selecting a good QC-LDPC code as the Mother, the son codes constructed through the compressing methods are still good codes, i.e., have high girth and few short cycles, moreover, they have continuous code length and require smaller memory space. Analysis and simulation results show they can perform well over the AWGN channel.
Key words: LDPC codes; girth; circulant permutation matrices; compressing; quasi-cyclic.

I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were first introduced by Gallager[1] and rediscovered in the 1990s. Recently, Quasi-Cyclic LDPC (QC-LDPC) codes are becoming an attractive topic, as they have lower implementation complexity and excellent performance[2]. As we know, the performance of LDPC codes can be improved by increasing the girth and reducing the number of short cycles, several construction methods[3-6] were proposed towards this direction. However, these methods can’t generate codes of arbitrary code length with high girth, meanwhile, they need large memory space.
If a QC-LDPC code has high girth and few short cycles, it is a good code with excellent cyclic characteristic, we design two compressing methods (recursive-compressing and single-compressing) to construct this kind of codes. The main idea of the methods is selecting a good code as the mother code and compressing it. The generated codes have continuous code length and we only need small memory space to store them.

B. Storage of Codes
If we want to use a series of codes generated from other methods, all their parity-check matrices need to be stored in the hardware system, which ask for huge memory space. Adopting the compressing methods, we can reduce the space greatly.
For the recursive-compressing method, information stored includes: the Mother matrix and the Shift matrix of a mother code, the corresponding rate of compressing (q) of each recursion step, the compressing type (adopting S(H1) or S(H2 ) ) in each recursion step. Besides, if the element si′j or si′j′ needs to be updated in the recursion step, we should store the relevant location (i,j), however, the number of locations stored is always very small.
For the single-compressing method, information stored is similar with that of the recursive-compressing method, so we do not discuss it particularly.
We can make use of a mother code and the two methods to get a series of codes with arbitrary code length and need not store each code. When the number of the generated son codes is very large, we can reduce memory space greatly.
V. SIMULATION RESULTS
The method proposed in [4] can be used to get 3D codes with column weight 3 and code rate 0.4. The 3D code with girth 10 can be used as the mother code, using the proposed compressing methods, a series of son codes can be generated with column weight 3 and code rate 0.4. We present simulation results over the AWGN channel with BPSK modulation, and use the log-sum-product algorithm with 100 iterations.
Let a 3D code with code length 6000 be a mother code, apply the recursive-compressing method with two recursion steps totally, q is a fixed fraction 1/2, then lengths of the generated son codes are 3000 and 1500, their girth is 8. Compare them with 3D codes of girth 10, whose lengths are 3000 and 1375. Simulation results are shown in Fig.2.

VI. CONCLUSION
Through the analysis of QC-LDPC codes, we present a theorem about the cyclic characteristic of codes. Two compressing methods to construct codes effectively are proposed, i.e., a series of codes with arbitrary code length can be generated by compressing a mother code and they have good cyclic characteristic. Analysis and simulation results show that the compressing methods only need to store little information and the generated codes have good performance.
