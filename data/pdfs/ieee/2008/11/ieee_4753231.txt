Efficient Quantization Schemes for LDPC Decoders

Radivoje Zarubica, Ryan Hinton, Stephen G. Wilson, and Eric K. Hall* Charles E. Brown Department of Electrical and Computer Engineering
University of Virginia, Charlottesville, VA 22904 *L-3 Communications West, Salt Lake City, UT 84116

Abstract- LDPC codes have attracted much attention recently for their near-capacity performance and high throughput owing to parallel decoding architectures. While simulations are normally done with Boating point computation, any practical implementation (ASIC or FPGA) will be built with fixed-point computation. Obviously, decoder speed will increase, and resource requirements will drop, with low-precision implementations, say 3,4, or 5-bit architectures. In this paper we study the effects of quantization in this regime, using density evolution and decoder simulation. Detailed sum-product decoder implementations are given, and performance losses relative to Boating point decoding are given. In particular, 4-bit decoder architectures sustain only 0.1 dB penalty.1
I. BACKGROUND
Low density parity check (LDPC) codes have garnered a great deal of attention in the error control coding field recently, due to the ability to implement soft-decision decoders that perform very near the Shannon bound (for long blocklengths.) The decoding algorithm is iterative, using message passing among nodes in a Tanner graph representation of the code, and allows parallel, or semi-parallel, processing in the computation to speed decoding.
Given k message bits, an LDPC encoder constructs a
codeword of length n (n > k) code bits, yielding a code
with rate=k/n. The coding scheme relies on n - k possibly overlapping subsets of the code bits such that the parity of each subset in any codeword is even. These subsets are described by a (n-k) xn matrix H = [hij ], such that hij = 1 iff the i-th subset has bit j in it. The H matrix is called the parity check matrix of the code and it is very sparse, hence the name "low density parity check" codes. An LDPC code is called regular if the weights of rows and columns in its parity check matrix are equal and is called irregular otherwise. The parity check matrix can be visualized as a bipartite graph, referred to as the Tanner graph of the code. One such graph of a short (10, 5) code is shown in Figure 1. Check nodes correspond to n - k parity checks and variable nodes correspond to n bits of the codeword that is transmitted. The Hamming weights of rows and columns in this representation are the degrees of check and variable nodes, respectively. A variable node is connected to a check node if the bit corresponding to it participates in the parity check operation of the check node. For better visualization we say that a variable node j is connected to a check node i iff hij = 1.

III. QUANTIZATION IN CHECK NODES
As described in Section I, the SPA performs an iterative updating of real-valued messages flowing from variable nodes to check nodes and back, driven by the intrinsic real-valued likelihood data from the channel. A finite-precision implementation of the algorithm will send approximations of these messages, and thus we expect slightly different performance. The outcome on any particular channel realization may change due to quantization, and the statistical behavior (BER and FER versus SNR) will change. An efficient quantization scheme will attempt to minimize this impact on performance. Various quantization schemes are shown in the literature [2],[3] where some of implemented decoders use 8 bits to represent messages. Our results show that using 5 bits is sufficient to eliminate quantization effects on the performance, whereas quantization schemes designed for 3 and 4-bit precisions yield to 0.25 dB and 0.05 dB penalty, respectively.
The VNP is merely an adder, normally implemented with 2's complement arithmetic. We assume that check-to-variable messages are represented using bv bits of precision, with a step-size of ~v. Likewise the channel likelihood messages are represented with bv bits. Since return messages (variableto-check) are also represented with bv bits, the adder requires a saturation operation. The minimum message value is - (2 bv -1 ) ~v, while the maximum message is (2 bv -1 - 1)~v. In the work of Zhang et al [8] the values for ~v and ~e are restricted to be inverse powers of 2, which limits the search space for "good" quantization schemes.

It is evident there are five parameters, bv, be, ~v, ~e, and C that define the quantized decoder. For a given choice of bv, be, and C, there are good choices for step sizes that best approximate a floating point implementation. A better view is to choose step sizes to simply have best error performance, for it's possible that a quantized decoder can even beat a floating point decoder in some cases. Exhaustive simulation

is one unattractive possibility to optimize the choice, which would be slightly different at each SNR. Instead we have resorted to density evolution [4],[5] for study of quantized decoding, running density evolution for the various choices to determine the combination(s) with lowest SNR threshold. (Density evolution predicts the decoding threshold for message passing on arbitrarily large graphs without cycles.) To run density evolution, we need to specify a code ensemble, meaning its degree profiles. For purposes of this paper, we focus on regular rate-I/2 codes with variable degree 3 and check degree 6.

(Note that once a step-size pair is selected, only integer values reside in the LUT's, and the step-sizes are somewhat irrelevant; it is ~v however which is attached to channel quantization.) We should also note that a small range of step-size pairs are associated with a given set of tables. Finally, in each table, we list the degradation 1/J (in decibels) from the bi-AWGN Shannon bound on rate-I/2 codes. Some of this gap is implied by the regular code assumption (about 0.977 dB), and the remainder is implied by quantization.
We finally remark that the performance of a decoder is rather sensitive to the table entries, and changing just one value by one unit can result in significant penalty. For large precisions, say more than 6 bits, this sensitivity diminishes, but in cases of interest here the tables are more crucial.

IV. CONCLUSION
In this paper we report a detailed study of quantization in LDPC decoders, using density evolution to yield efficient designs. Performance losses relative to floating point decoding are minor, e.g. 0.1 dB in a 4-bit architecture. In future work we plan to further study quantization for irregular codes, in particular those with many degree-2 variable nodes, as we have observed that coarse quantization may introduce high error floors for these codes. Preliminary results indicate that a simple increase in quantizer scale can repair this deficiency.
