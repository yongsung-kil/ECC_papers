Abstract
The performance of LDPC codes decoded by iterative algorithms depends on the structural properties of their underlying Tanner graphs. For general memoryless channels, error patterns dominating the bit and frame error probabilities at the error floor region are termed trapping sets. In this paper, we propose an effective method for finding small-size trapping sets of LDPC codes. In the proposed method, a probabilistic algorithm to find low-weight codewords is applied to finding small trapping sets of LDPC codes. Furthermore, we show numerical results of examining small trapping sets of (504, 252) and (1008, 504) LDPC codes.
1. INTRODUCTION
Low-density parity-check (LDPC) codes exhibit the performance close to the Shannon limit by iterative decoders [1]. The performance depends on the structural properties of their underlying Tanner graphs. On the binary erasure channel (BEC), stopping sets determine the error floor performance, and also dominate the bit error rate (BER) and the frame error rate (FER) of LDPC codes [2]. For general memoryless channels, the dominating error patterns have different names motivated by different analytical techniques. These include near-codewords [3], pseudo-codewords [4] and trapping sets [5]. It is reported in [5] and [6] that the decoding failure at the error floor region is influenced by a certain configuration of the subgraph in the Tanner graph, which is termed trapping sets. Moreover, procedures that the low error rate performance is determined by small trapping sets have been presented in [7] and [8].
The problem of computing the minimum-size trapping set in a code is as difficult as that of determining the minimum distance and the minimum-size stopping set of a code [9][10][11]. In [12], an algorithm for exhausting all low-weight error-prone patterns have been proposed. However, determining the minimum size of trapping sets in long LDPC codes is difficult problem for which the search requires exponentially growing

complexity. It is conjectured in [11] that the problem is NP-hard.
In this paper, we propose an effective method for examining small trapping sets of LDPC codes. It is a fast algorithm to find possibly small trapping sets in an LDPC code. In the proposed method, Stern’s algorithm [13], which is a probabilistic algorithm to find low-weight codewords of linear codes, is applied to generating small trapping sets of LDPC codes. Furthermore, we examine small trapping set of MacKay’s (504, 252) and (1008, 504) LDPC codes [15] using the proposed method. In the numerical results, we found a number of small trapping sets which have not been reported in past literatures. Since the small trapping sets influence the FER of LDPC codes at the error floor region, our method is helpful to evaluate the error floor performance of LDPC codes.

The trapping set depends on the decoder input space and the decoding algorithm. For example, we could consider that the trapping sets are precisely the non-zero codewords in the case of a maximum likelihood decoder. If the channel is the BEC and the decoder is based on the belief propagation, the trapping sets are the stopping sets. In [5], it is shown that the FER of an LDPC code at the error floor region can be approximated by

FER = Pr{ξT },

where ξT denotes the set of decoder inputs that lead to a decoding failure on the trapping set T . This set varies

with the channel parameters. Although the probabilities Pr{ξT } cannot be characterized analytically, there exist numerical methods for estimating their values [5][8][14]. The general procedure is performed as follows:
1. Search trapping sets T contained in a code C.
2. For each trapping set T found in step 1, estimate the probabilities Pr{ξT } using the numerical simulation.
3. Calculate the FER from eq.(4) with the trapping sets T found in step 1 and the probability Pr{ξT } estimated in step 2.
Therefore, conditioned on the knowledge of these probabilities, the FER is completely determined by the distribution of the trapping sets T , for all relevant values of the parameters a and b. Usually, the smaller the trapping set, the stronger the influence of the set on the FER.

In the proposed method, we use Stern’s probabilistic algorithm to find small trapping sets which influence the FER of LDPC codes at the error floor region. Stern’s algorithm generates low-weight codewords from the parity-check matrix H˜ = [ImP] which is in systematic form. The matrix H˜ is obtained by applying the Gaussian elimination to H. Then we cannot simply apply Stern’s algorithm to finding small trapping set of H, because the connection of edges in E in the graph given by H˜ is different from that by H. On the other hand, we notice that the trapping set T = {v1, v2, v4} in Figure 1 is satisfied with the condition of codewords on the subgraph not to contain the parity-check nodes p1 and p2. Thus, when we eliminate rows corresponding to odd-weight row of H = [hj]j∈T from H, we can generate trapping sets T as codewords using Stern’s algorithm. Based on the property of TS matrices, we first make the submatrix H by eliminating some rows of H randomly, then generate low-weight codewords from H by using Stern’s algorithm. If we eliminate the rows of H corresponding to odd-degree parity-check nodes contained in the neighbors of T , we can generate small trapping sets of C as low-weight codewords of H .

4. NUMERICAL RESULTS
We evaluated small-size trapping sets of the following widely-known LDPC codes:
• C1: MacKay’s (504, 252) code
• C2: MacKay’s (1008, 504) code
where C1 and C2 are labeled as 252.252.3.252 and 504.504.3.504 in [15] respectively. Table 1 shows the results obtained by applying our algorithm to these codes. In these numerical experiments, the parameters used in our algorithm was set as follows:
• C1: α = 13, β = 45, p = 1, l = 15 and rmax = 1.0 × 106
• C2: α = 16, β = 45, p = 1, l = 17 and rmax = 1.0 × 107
We used computers of CPU Core 2 Duo E6700 2.67GHz and RAM 2GB, and it took 2.5 hours and 40 hours to obtain results for C1 and C2 respectively. In [8], the multiplicity of small trapping sets of the same codes has been evaluated. These are also shown in Table 1.
In Table 1, the probability to find (a, b) trapping sets by our algorithm is lager as the size a of trapping sets is smaller. We were able to find a lot of small trapping sets using our algorithm. In particular, we found (9, 1), (11, 1), (13, 1), (4, 2), (6, 2) and (8, 2) trapping sets in C1 which have been not found in [8]. And we found seven (14, 2) and fifteen (16, 2) trapping sets in C2 which have been not reported in [8]. We understand that the number of trapping sets contained in C1 and C2 is not less than the multiplicity shown in Table 1. When the frequency of found (a, b) trapping sets is much larger than the multiplicity of (a, b) trapping sets, we predict that the proposed algorithm could find almost all (a, b) trapping sets. 5. CONCLUSION
In this paper, we proposed an effective method for examining small trapping sets of LDPC codes. It is based on Stern’s probabilistic algorithm for finding lowweight codewords of linear block codes. In the proposed method, we focused on a relation between small trapping set and low-weight codewords defined by the sparse matrix, and applied Stern’s algorithm to generating small trapping sets in an LDPC code. Additionally, we showed the numerical results of examining small trapping sets of (504, 252) and (1008, 504) LDPC codes by our method. In the results, we found smaller trapping sets than that by conventional methods.
