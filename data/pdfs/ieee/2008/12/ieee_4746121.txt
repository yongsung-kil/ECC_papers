Dynamically Reconfigurable Architecture for Multi-Rate Compatible Regular LDPC Decoding

Akiyuki Nagashima Yuta Imai Nozomu Togawa Masao Yanagisawa Department of Computer Science and Engineering Waseda University 3-4-1 Okubo, Shinjuku, Tokyo, Japan
Email: nagashima, imai, togawa@togawa.cs.waseda.ac.jp

Tatsuo Ohtsuki

Abstract—Recently a demand for high-speed wireless network service on mobile devices is rapidly increasing. Error correcting codes are used to enhance network communication quality. Particularly, LDPC (Low Density Parity Check) codes show high throughput and achieve information rates very close to the Shannon limit. In this paper, we propose a dynamically reconfigurable architecture for multi-rate compatible regular LDPC decoding. Our proposed decoder deals with multi-rate codes by introducing a multi-rate compatible 1st-2nd minimum searching unit. The proposed decoder shows the better throughput over the wide range of S/N ratio compared to conventional rate-fixed LDPC decoders.
I. INTRODUCTION
LDPC codes are linear error correcting codes which were discovered by Gallager in 1960’s [2]. It had already been known that LDPC codes had a higher decoding performance compared to other error correcting codes at the time.
Recently a demand for high-speed wireless network service on mobile devices is rapidly increasing. However, noise environment may be unstable for most of the time in wireless network communication. There is a strong need for an LDPC decoder which can select the information rate depending on network noise environment.
Since then, there have been proposed several LDPC decoders. The LDPC decoder proposed in [3] is a decoder which can change code length but it only deals with the information rate R = 0.5. In [1], an LDPC decoder which deals with multi-rate decoding is reported. The multi-rate compatibility is realized by adding up the rows of a parity check matrix. All these LDPC decoders, however, deal with only one fixed information rate once decoding has started. The LDPC decoders in [3] does not deal with multi-rate decoding. The decoder in [1] does deal with multi-rate decoding but, in order to change the information rate, off-line reconfiguration is required.
In this paper we propose a dynamically reconfigurable architecture for multi-rate compatible regular LDPC decoding. Our proposed decoder deals with multi-rate information codes by introducing a multi-rate compatible 1st-2nd minimum searching unit. Experimental evaluations demonstrate that our proposed LDPC decoder shows the better throughput over the wide range of S/N ratio compared to conventional rate-fixed LDPC decoders.

III. DYNAMICALLY RECONFIGURABLE LDPC DECODER
ARCHITECTURE
In this section, we propose a dynamically reconfigurable multi-rate compatible LDPC decoder. First, we describe the specification of the proposed LDPC decoder and then briefly propose a column operational unit, which does not contribute to reconfigurability for information rate. Next, we introduce our block-partitioned parity check matrix, which leads to the reconfigurability of our decoder. After that, we propose a reconfigurable row operational unit. Finally, we discuss reconfigurable LDPC decoding using our proposed LDPC decoder.

A. Specification of the Proposed Decoder
As we mentioned earlier, we use the Min-Sum algorithm as a decoding algorithm. In [3], [6], code length is set to be 1536 bits at the information rate of R = 0.5. Then in our LDPC decoder, the code length for R = 0.625 and R = 0.75 is set to be 2048 and 3072, respectively, in a similar way. Each data has bit width of 6 and each sub-block has a size of 256×256. As in

[6], we employ partially parallel structure and set the parallel factor to be 4. We have 120 memory banks. These values are quite common in conventional LDPC decoders. The maximum number of iterations (which is set to be 8 in our decoder) is also determined based on [3], [6].
The proposed decoder structure is presented in Fig. 1. Each operational unit executes the corresponding operation. Each controller controls the read/write of the data required for each operation from/to memories, and the state-transition.
Column Operational Unit: Our column operational unit, which is also called a bit functional unit (BFU), executes the column operation. Since the parallel factor is set to be ‘4’, we have four BFUs for each column block. Thus we have 4 × 24 = 96 BFUs in total since there are 24 column blocks in our proposed parity check matrix.
Value of each row is calculated in one clock cycle and then we can obtain three β values. It also calculates the temporal decoded word, which is represented by cˆjq , to be used later in parity check operation in the decoding process.
B. Block-Partitioned Parity Check Matrix
A block-partitioned parity check matrix is quite common as it can also be seen in [3], [5], [6]. Since it is a regular matrix, it can be divided into wr × wc sub-blocks. Each sub-block is a square matrix whose size is k × k. A sub-block is either an identity matrix or the matrix which is obtained by shifting the elements of an identity matrix to the right by an integer value s.
Fig. 2 shows our proposed parity check matrix. First, we prepare a large matrix whose size is 768 × 6144. Then we divide it into 3 × 24 sub-blocks. This large matrix is evenly divided into 2, 3 and 4 small matrices, according to the information rate R as shown in Fig. 2. Since small matrices in each R have exactly the same size, wr and wc, we can decode 2, 3 or 4 code words simultaneously depending on R, by just using the original large matrix.
For example, assume that we have R = 0.5, and its code word has code length of 1536 bits. Since the number of columns in the large matrix in Fig. 2 is 6144, ‘four’ code words are decoded simultaneously using our proposed parity check matrix. In this case, we use a parity check matrix composed of 3 × 6 sub-blocks, i.e., whose size is 756 × 1536 (the matrix surrounded by the bold line in Fig. 2). Similarly, assume that we have R = 0.75,and its code word has code length of 3072 bits.

By applying the block-partitioned parity check matrix mentioned above to an LDPC decoder, we can have appropriate information rate R using a single hardware even though fluctuating wireless network environment is given. C. Architecture to Deal with Multi-Rate Decoding
If we employ the block-partitioned parity check matrix proposed in Section III.B, an LDPC decoder can consider the following two points:

Since 2) can be easily realized by extending the conventional parity check operational unit, we focus on 1), i.e., architecture design of a row operational unit which uses the blockpartitioned parity check matrix described in Section III.C.

Multi-Rate Compatible 1st-2nd Minimum Searching Unit: Eq. (3) requires searching 1st and 2nd minimum values among input values. Changing information rate R from 0.5 to 0.75 significantly affects this operation. In order to solve this problem, we apply the modified version of the tournament approach [4] to a reconfigurable row operational unit. As shown below, the tournament approach can effectively be applied to 1st and 2nd minimum value searching for the various information rates.
The tournament approach works as follows: First of all, each input value is paired with another input value. For each pair, comparison is done and the smaller value proceeds to the next comparison. Comparison continues until we finally have a single comparison result. The last value will give the minimum value among all the original input values. By using this tournament approach, we propose a multi-rate compatible 1st-2nd minimum searching unit which has only a little area overhead(Fig. 4).

D. Dynamical Reconfigurable LDPC Decoding Using the
Proposed Decoder
Now let us consider the tradeoff between error-correcting performance and decoding throughput performance.
When a fixed information rate LDPC decoder is focused on, this tradeoff can be solved by finding out the best point which gives the moderately good error-correcting performance and the throughput. This works very well when the communication environment is not changed. However, it is not good enough to consider communication via wireless networks.
Multi-rate compatibility is one of the answers to overcome this tradeoff. In order to make our decoder more efficient in terms of error-correcting performance and decoding throughput performance, the appropriate rate-switching points must be determined.
We carried out the LDPC code-decoding simulation using the C language to simulate AWGN channel environment with the BPSK modulation to find out the rate-switching points where reconfiguration is required and to see the throughput of the proposed decoder.
Fig. 5 shows the simulation result. In this figure, we define the decoding throughput as bit rates of correctly decoded codes.
As is shown in Fig. 5, switching the information rate R according to noise environment to obtain high decoding throughput has been proved to be effective. From these results, the decoder configuration of R = 0.5 gives the best performance until S/N ratio gets to 2.75 [dB]. From 2.75 [dB] to 3.65 [dB], the decoder configuration of R = 0.625 gives the best decoding throughput. The channel takes five S/N ratios of 2.5 [dB], 3.0 [dB], 3.5 [dB], 4.0 [dB], and 4.5[dB]. We prepared three time-ratio patterns where each S/N takes a certain period of time.
