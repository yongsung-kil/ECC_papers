An Effective Multibit-Flipping Algorithm for LDPC Decoding

Jui-Hui Hung1 and Sau-Gee Chen2
Institute of Electronics & Department of Electronics Engineering, National Chiao Tung University Hsinchu, 300, Taiwan, R.O.C.
1paholisi.nctu@gmail.com; 2sgchen@cc.nctu.edu.tw

Abstract—For LDPC decoding, bit-flipping (BF) algorithms are much simpler than the min-sum algorithms (MSA). However, BF algorithms have the disadvantages of poorer performances and higher iteration counts than MSA. To increase the BF performances, this paper presents a novel BF algorithm, called Low-Correlation Multibit-flipping (LCMBF) algorithm, which flips more than one bit in each iteration. High performances are achieved by flipping those bits with low correlation as much as possible. As such, the chances of introducing additional error bits are greatly reduced. Overall, the proposed algorithm achieves better BER performances and requires less iteration numbers than conventional BF algorithms.
I. INTRODUCTION
Low-density parity check (LDPC) code was introduced in 1962 [1], which can achieve performance close to Shannon bound. As such, LDPC has been adopted by many state-of-the-art communication systems, such as the recent DVB-S2, DMB-TH and 802.16e systems. It is a kind of binary linear block code whose parity check matrix is sparse which has much fewer 1’s than a common matrix. A sparse parity check matrix facilitates simple decoding algorithms and low-complexity decoder designs.
Check matrix of a LDPC code is often represented by a bipartite graph, called Tanner graph [2], which is composed of n variable nodes and m check nodes. Those variable nodes and check nodes are connected by edges defined by nonzero entries of matrix H. The number of 1s in each column of H determines the number of edges for each variable node connected to check nodes, and the number of 1s in each row of H determines the connections from each check node to variable nodes. Tanner graph shows a clear picture of all the information exchange links in a decoding process.
There are many LDPC decoding methods in literature. The sum-product algorithm (SPA) [3] has the highest decoding performance closed to Shannon limit. Due to its high complexity, it is impractical for applications. In comparison, the popular MSA presented in [4] has lower complexity, with comparable performance to SPA. Still, its complexity is too high to be efficiently realized in hardware implementation. Both MSA and SPA are based on the concept of message passing, which involves complicated message computations and updates. In contrast, the conventional BF (CBF) algorithm

[1] has a much simpler bit-update scheme than those two algorithms. However, its performance is much lower than that of SPA and MSA. To remedy the problem, the weighted bit-flipping (WBF) algorithm [5] and the modified WBF (MWBF) algorithm [6] attain better performances than the CBF algorithm, at the cost of additional computation. An improved MWBF (IMWBF) algorithm [7] and a reliability-ratio based WBF (RRWBF) algorithm [8] further improve the decoding performance over [5], [6].
Nevertheless, there still exist noticeable performance gaps between those algorithms and SPA. Beside the decoding performance, all the mentioned BF-based algorithms need larger iteration numbers than SPA, especially in low SNR conditions. To solve the problem, we will propose an efficient BF decoding algorithm which flips multibits in each iteration. The algorithm also combines an accurate decision algorithm for those bits to be flipped at a time. As a result, the iteration number can be reduced, while high decoding performances can be achieved, as will be detailed as follows.

III. CONCEPT OF THE PROPOSED ALGORITHM
Both mentioned algorithms only flip one bit in each iteration. Hence, they need high iteration counts to obtain the final decoding results. The most intuitive and simplest way to reduce iteration numbers is flipping more than one bit in each iteration. However, there is risk that one may induce additional error bits and propagate the errors to the later iterations, by flipping more than one bit at a time, as demonstrated below.
Consider a (10,4) LDPC code for an example, where its corresponding Tanner graph is shown in Fig. 1(a). Assumed WBF or RRWBF algorithm is used, and b2 is the first flipped bit as signified by the circle in Fig. 1, and b10 is correctly decided to be flipped next, according to the updated syndrome and value of error term after the b2 flipping, as shown in Fig. 1(b). On the other hand, if we adopt the strategy of flipping two bits at a time and assume that both b2 and b4 are to be flipped in the first iteration, then an additional error will be so introduced due to the wrongly flipped b4. It results in an inferior performance to the anticipation. One solution for this is to reduce the occurrence probability of additional error bits when flipping more than one bit at a time. Next, the basic concept of the proposed algorithm and the optimized scheme to select the candidate flipping bits (with low error probability) will be introduced first, followed by the summarized algorithm’s steps.
The basic concept of the proposed multibit flipping algorithm is very simple. For example, supposed that we want to flip two bits at a time. First, the reliability measure (based on any feasible measure such as (7)) is calculated and the bit with the maximum measure is chosen. Next, the second bit which has the maximum reliability measure among all the bits (not connected to the first bit by any links in the Tanner graph). Then these two bits are flipped and all syndromes associated with these two bits are updated. This ends an iteration and enters to the next iteration. Similar concept can be applied to flipping more than two bits at one time. In this case, note that all the later chosen bits should not be connected to those earlier

chosen bits. However, as the number of flipped bits is increased, it is hard to find any disconnected bits, especially for those parity check matrix Hs which have high bit connectivity. Under this situation, we can relax the previous unconnected link constraint by allowing choosing those bits with indirect connection, as will be clearly defined below. According to the Tanner graph, there exist three different kinds of relationships between any two bit nodes, as follows:
1) Unconnected: There is no any path connecting these two bit nodes.
2) Indirectly connected: These two bit nodes are connected by a path, and the path passes through more than one check node in between these two bit nodes.
3) Directly connected: These two bit nodes are connected by a path, and the path passes through only one check node in between these two bit nodes.
Take b2 in Fig. 1 for an example, which is unconnected to each of b7, b8, b9 and b10. It is indirectly connected to both b1 and b3. Besides, it is directly connected to b4, b5 and b6. Obviously, if a bit node is unconnected to another node, then the flipping of any of these two bit nodes will not affect the syndrome (and also error term E, such as (5) or (7)) of the other bit node. Similar to first relationship, if two bit nodes are indirectly connected, then the flipping of one of these two bit node will not change the corresponding syndrome of the other bit node. However, it may change the error term E corresponding to the other bit node, and possibly induce additional error bits. If the third relationship holds for a bit node pair, then flipping of one of these two bit nodes will mostly increase the number of 1s in the syndrome vector. Therefore, it will induce the chain reaction of incorrectly flipping bit nodes, and cause additional burst error bit nodes. This condition is the major concern in designing a multibit-flipping algorithm.
Fortunately, from intensive simulations, we find that the probability of additional errors induced by flipping an indirectly connected bit is very low. As a result, we can select those bits as the candidate bits which are indirectly connected or unconnected to each other. Based on the idea, we define the proposed algorithm as Low-Correlation Multibit-Flipping (LCMBF) algorithm.
Next, the number of flipping bits per iteration (denoted as Nfb) has to be decided. By intuition, a large Nfb is preferred. However, a large Nfb would increase the probability of wrongly flipping a correct bit. Hence, we should decide a suitable Nfb for the highest gain of the multibit-flipping scheme. This number highly depends on the code length, code rate and structure of the parity check matrix. However, it is very hard to derive the optimal value of Nfb due to the high complexity of theoretical analysis. Therefore, we use computer simulation to get the optimal value of this number. In what follows, the detailed steps of the proposed algorithm will be summarized.

V. COMPARISON AND SIMULATION RESULTS
Before conducting simulations of LCMBF algorithm, we need to select one of BF algorithms to work with. Here, this paper chooses RRWBF algorithm, because it has the best performance of all the BF algorithms. Thus, we combine the proposed algorithm with RRWBF algorithm which is denoted as LCMBF-RR algorithm. Some simulation conditions are described below:
1) Gallager’s method is applied to randomly generate the coding matrices, which has regular column and row weights.
2) AWGN channels and BPSK modulation are assumed.

First, we simulate the performance of RRWBF assuming code length of 816, code rate of 1/2 and column weight of 5. According to the simulation results in Fig. 2, we find that the decoding performance curves of RRWBF algorithm overlap for the cases of 100 and 200 iterations. That means 100 iterations are enough. It shows the maximum achievable performance of RRWBF algorithm in this case.
Fig. 3 shows the performances of LCMBF-RR algorithm at various Nfb. By this figure, we can find that the decoding performance will saturate when Nfb=10 but get worse when Nfb=20. As mentioned before, the worsening is due to the high error probability with a large Nfb of wrongly flipping the correct bits. In fact, flipping only those unconnected bits can avoid this kind of error. Unfortunately, this type of candidate bits is not always existent in high Nfb condition. Hence, to further improve the decoding performance of LCMBF algorithm, finding a suitable iteration number is necessary. Fig. 4 indicates the Compared decoding performance results of LCMBF algorithm at Nitr =20 and 10. It points out that LCMBF algorithm a higher iteration number has a better decoding performance than a lower iteration number in the same value of Nfp (if the performance is not saturated yet).
Note that the proposed algorithm’ performance is also bounded by the maximum achievable performance of the combined algorithm (i.e., in this case the RRWBF algorithm) as shown in Fig. 5. In this Figure, the performance of LCMBF algorithm cannot get any improvement in Nfp=2 compared with RRWBF algorithm at Nitr =100. It is because that the reliability measure of the combined algorithm is utilized in the proposed algorithm. Finally, Fig. 6 and 7 show that the algorithm still work well in various code rates and column weights at iteration number of 50.
VI. CONCLUSION
The proposed LCMBF algorithm for LDPC decoding can flip multibits at a time with very low error probability in wrongly flipping those candidate error bits. Compared with other BF algorithms, the proposed algorithm can achieve excellent decoding performances at small iteration numbers. Besides, any reliability measures commonly adopted in the existing BF algorithms can be combined with the proposed algorithm so that the current BF algorithms can be significantly improved in performances. Finally, low-complexity and accurate reliability measures are under further investigation so that higher-performance multibit-flipping algorithms can be derived.
