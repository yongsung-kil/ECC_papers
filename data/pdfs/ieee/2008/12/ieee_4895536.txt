Improvement of the Forced-Convergence Decoding for LDPC Codes

Abstract
An improvement of the forced-convergence decoding algorithm is discussed. Iterative decoding schemes such as the sum-product (SP) decoding bring out excellent performance of low-density parity check (LDPC) codes. The forcedconvergence (FC) decoding is also an iterative decoding algorithm obtained by modifying the SP decoding. The FC decoding succeeds to reduce the decoding complexity of the SP decoding, but the error correcting performance is severely degraded by the modification. The performance degradation of the FC decoding is expected to be mitigated if the convergence criterion used in the algorithm is appropriately modified. This paper discusses several types of modifications of the convergence criteria. Computer simulation shows that the proposed modifications improve the performance of the FC decoding algorithm.
1. Introduction
The sum-product decoding plays a crucial role in decoding low-density parity check (LDPC) codes. For long years, efficient realization of powerful decoding schemes has been one of the most significant issues in the coding theory. A number of efforts have been devoted to the subject, but the trade-off relation between the complexity and the performance could not be conquered. Long codes may have better potential than short codes in general, but many people believed that long codes are not practical because to decode such long codes requires huge complexity. The situation is, however, drastically changed in mid-90s when Turbo codes are proposed and LDPC codes are rediscovered. It was shown that there exist very efficient and very powerful decoding schemes for these classes of codes. Such decoding schemes make long codes practical, and the potential performance of long codes were brought to practical realizations. For example, if we decode appropriately constructed LDPC codes by using the sum-product decoding algorithm, then we will be able to obtain performance which is close to the Shannon limit[3]. In this sense, the sum-product decoding is the key to obtain a prominent per-

formance of LDPC codes.
The complexity of the sum-product (SP) decoding algorithm is almost in the linear-order in the code length, and the algorithm is the “most efficient” in an asymptotic meaning. The SP decoding can be understood as an iterative message passing algorithm over a Tanner graph. A Tanner graph is a bipartite graph whose set of nodes are divided to variable nodes and check nodes, and whose incident matrix is the parity check matrix of the code. In one iteration, a node of the graph sends and receives messages to and from its neighbor nodes, where the messages represent the estimated probabilities of the transmitted symbols. By repeating such exchanges of messages, the algorithm improves the estimation and eventually reaches correct estimation of the transmitted symbols. It is known that the number of iterations can be small. If we consider the number of iterations as a constant number, then the SP decoding has a linear-order complexity in the code length.
We would like to reduce the decoding complexity of the SP decoding. Asymptotically saying, there is little possibility to reduce the decoding complexity of the SP decoding, and thus our aim is to improve the “implicit constants” which are hidden behind the order-notation. This kind of improvement is essential from an engineering viewpoint, and we can find several studies in this direction. For example, in the min-sum decoding approach[1], we consider to perform probability computations in a log-likelihood region. This will transform the multiplications of probabilities into additions of log-likelihood values, and we will have simpler and more efficient decoders. Another well-known investigation is the study of bit-flipping algorithms[2]. In the bit-flipping approach, we use simple binary values as the messages, which can reduce the bus bandwidth in the decoder. This kind of efforts do not improve “asymptotic” complexity of the decoding algorithm, but surely reduce the complexity of implemented decoders.
The forced-convergence (FC) decoding proposed by Zimmermann[5] is a modification of the SP decoding algorithm. The primal purpose of the FC decoding is to reduce the complexity of the SP decoding, but the FC decoding

takes a different approach from the min-sum and bit-flipping approaches. In the FC decoding, we consider to stop computations which are related to “already converged” variable nodes. In the SP decoding, a variable node tries to estimate a transmitted symbol by iteratively updating probabilities associated with the node. The probabilities are expected to converge to correct values as the iteration proceeds, but we need to remark that the convergence behaviors of probabilities are different one by one. During the decoding computation, some variable nodes may successfully reach correct estimations of transmitted symbols in a first few iterations of the decoding. If we could distinguish a variable node which has reached correct estimation, then we can omit subsequent update of the probabilities which are related to that variable node. This will reduce the decoding complexity of the algorithm.
The focal point in the FC decoding is the criteria with which variable nodes are distinguished. If there is a sufficient condition for a variable node to have reached correct estimation, then the condition can be a good criteria which reduces the decoding complexity without affecting the decoding performance. However, such a condition is not known yet and we need to investigate other criteria which approximate the sufficient condition. The criteria considered in [5] is a simple inequality which is to test if probabilities are more than a given threshold or not. Experimental results show that FC decoding with this simple threshold criteria effectively reduces the decoding complexity, but remarkable performance degradation is observed unfortunately.
In this study, we consider to improve the criteria so that the performance degradation of the scheme is as small as possible. Several criteria are proposed, and their affect on the performance and the complexity are studied. Computer simulation shows that our modifications improve the performance of the FC decoding.
2. Preliminaries
Throughout this paper, let C be a low-density parity check code (LDPC code) which has an M × N parity check matrix H = (hi, j), where M and N are positive integers satisfying M < N. The matrix H can be represented by a bipartite graph G = (V, E) which is called the Tanner graph of the code. The graph G has M+N nodes, and the set of nodes are partitioned to two subsets Vc and Vv which respectively have M and N nodes. Nodes in Vc are called check nodes and denoted by c1, . . . , cM. Nodes in Vv are called variable nodes and denoted by v1, . . . , vN. The nodes cm with 1 ≤ m ≤ M and vn with 1 ≤ n ≤ N are connected by an edge in E if and only if hm,n = 1 in the check matrix H. In other words, the incident matrix of the Tanner graph G is the parity check matrix H where check nodes of G are associated with rows

of H and variable nodes of G are associated with columns of H. We say that two nodes are adjacent if the nodes are connected by an edge. For an integer m with 1 ≤ m ≤ M, let define A(m) = {n : hm,n = 1}, which is the set of indices of variable nodes which are adjacent to the check node cm. Similarly define B(n) = {m : hm,n = 1} for 1 ≤ n ≤ N, which is the set of indices of check nodes which are adjacent to the variable node vn.
Consider that a codeword x = (x1, . . . , xN) ∈ C is randomly and uniformly chosen and transmitted over a noisy communication channel. The channel may modify the transmitted symbols, and the vector y = (y1, . . . , yN) which is obtained at the receiver’s end can be different from x. We assume that the channel is memoryless and the statistical characteristic of the channel is known in advance. For a channel input symbol x and a channel output symbol y, let P(y|x) denote the probability that the symbol y is received under the condition that x has been transmitted.
3. Forced-Convergence Decoding
Under the setting described in the previous section, the forced convergence (FC) decoding[5] tries to estimate the codeword x from the received vector y. Similar to the original sum-product (SP) decoding, we associate each edge of the Tanner graph with several probabilities. For integers m and n with 1 ≤ m ≤ M, 1 ≤ n ≤ N and hi, j = 1, we introduce probabilities Qm,n(1) which are called a-priori probabilities and Rm,n(0) and Rm,n(1) which are called extrinsic probabilities, and Qm,n(0). The a-priori probabilities Qm,n(0) and Qm,n(1) are regarded as messages which are sent from the variable node vn to the check node cm, and represent the estimated probability distribution of xn. The extrinsic probabilities Rm,n(0) and Rm,n(1) are messages which are sent from the check node cm to the variable node vn, and carry the suggestions (on the value of xn) from other variable nodes. The probabilities are scaled so that Qm,n(0) + Qm,n(1) = 1 and Rm,n(0) + Rm,n(1) = 1.
In the SP decoding, all the extrinsic and a-priori probabilities are updated at each iteration. In the FC decoding, on the other hand, the update of some a-priori probabilities are stopped under a certain condition. In [5], this mechanism is described as deactivation of variable nodes. If a variable node vn with 1 ≤ n ≤ N is deactivated, then apriori probabilities Qm,n(0) and Qm,n(1) with m ∈ B(n) are fixed to constant values. As for the fixed constant value, we have two choices; In the push approach, the a-priori probabilities are quantized to either of 0 or 1. In the no-push approach, the a-priori probabilities are not modified, and fixed at the current values. Assume for example that the variable node vn is deactivated when the value of Qm,n(0) is 0.1. In this case, Qm,n(0) is forced to set to 0 in the push approach, while Qm,n(0) is fixed at 0.1 in the no-push ap-

The focal point of the FC decoding is the Step 2 in which we deactivate variable nodes. Intuitively, the inequality (1) holds if the variable node vn strongly believes that the n-th transmitted symbols xn is b. The essential idea of the FC decoding is to accept such strong belief positively, and to make use of the belief to omit some parts of computation. For inactive variable nodes, we can skip the computation for updating a-priori probabilities in the Step 4. If we can deactivate many variable nodes, then the decoding complexity will be reduced significantly. This aim may, however, backfire. If the belief of the variable node is not right, then a-priori probabilities are fixed to wrong values. We have no chance to correct the wrong probabilities, and the wrong probabilities continue sending wrong messages which can mislead computations at neighbor nodes.
Experimental results show that the FC decoding effectively reduces the decoding complexity, but remarkable performance degradation is observed unfortunately. The performance degradation is more serious in the push approach than the no-push approach. Maybe this is because that the wrong belief of variable nodes are “amplified” by quantizing the a-priori probabilities to 0 or 1 which are less accurate than the current a-priori probabilities.
4. Improvement of the FC Decoding
4.1. Techniques for the Improvement
The performance degradation of the FC decoding is mainly caused by inappropriate deactivation of variable nodes. In other words, the condition given by (1) is not a good criteria to select variable nodes which are having right belief. If the condition is replaced by a better criteria, then the performance of the decoding scheme can be improved. We consider in this section several techniques to improve the criteria.
The Timing to Deactivate Nodes The first improvement we consider is to hold deactivation of variable nodes in the first several iterations of the decoding. Roughly speaking, the probabilities used in the SP and FC decodings are expected to converge to accurate values as the iteration proceeds. Conversely, the probabilities are

not accurate when the decoding is just started. If we deactivate variable nodes according to less reliable probabilities, then we will have more risk that the deactivation is inappropriate and a-priori probabilities are fixed to wrong values. We must be conservative and cautious to handle unreliable probabilities which are obtained in the first several iterations, because wrongly fixed a-priori probabilities can mislead other computation. To be on the safe side, we consider not to deactivate variable nodes in the first several iterations.
Qualification by Syndrome We consider to make use of the syndrome information which are related to a variable node. In a linear block code, the consistency among symbols are represented in terms of the syndrome. The syndrome is the very essence of a linear code, and the syndrome information gives crucial information whether estimations of symbols are consistent or not. The FC decoding, however, does not use the syndrome information to determine if the belief made by a variable node is consistent or not. In this study, we consider that a variable node is deactivated if it satisfies the threshold criteria as given like (1), and additionally, the node passes a qualification which is posed by the syndrome. Let S 1, . . . , S m be syndrome components which are computed in the most recent execution of the Step 5 of the decoding. We consider two types of qualification criteria; A variable node vn with 1 ≤ n ≤ N is said to OR-qualify if S m = 0 for one or more m ∈ B(n), and vn AND-qualifies if S m = 0 for all m ∈ B(n). Note that S m = 0 if and only if the temporally estimated symbols xˆn with n ∈ A(m) are consistent with respect to the m-th parity check equation. Thus S m = 0 can be regarded as a positive evidence that the temporally estimated symbols which commit to the computation of S m is correct. The above qualification criteria is based on this observation. Obviously the AND-qualification is more tight than the OR-qualification in the sense that if a variable node AND-qualifies then it automatically OR-qualifies. The difference between these two qualification criteria will affect the performance of the decoding scheme.
Threshold Criteria The FC decoding distinguishes variable nodes by using apriori probabilities as in (1), which we call the Q-threshold in this paper, but the authors could not find an affirmative reason to use a-priori probabilities for the criteria. We may have another choice in which extrinsic probabilities are used instead of a-priori probabilities, that is, a variable node vn is deactivated if
Rm,n(b) > α|B(n)|.

We call the inequality R-threshold.

5. Simulation Results
The performance of the proposed scheme is evaluated by computer simulation. Figure 1 shows the block error probabilities of the SP decoding, FC decoding (with the push and no-push approaches) and the proposed decoding scheme (four combinations of {AND, OR}-qualification and {Q, R}-threshold). The target code is the one given as PEGirReg504x1008 in [4], which is an irregular half-rate LDPC code with code length 1008, constructed according to the progressive edge growth scheme. In the computer simulation, we assume the BPSK signaling over an AWGN channel, the maximum number of iterations is set to 20, and the proposed scheme holds deactivation in the first 10 iterations (l1 = 10 and l2 = 20 in the procedure given in the previous section). In the FC and the proposed decoding, the threshold constant α is chosen to be 0.99. Both of the push and nopush approaches are considered for the FC decoding, but we consider only the push approach in the proposed scheme for simplicity. The SP decoding shows the best performance in the investigated decoding schemes, while the performance of the FC decoding with the push approach is the worst. The performances of the proposed scheme are much better than that of the FC decoding, but not as good as the SP decoding. Among the proposed techniques, the ANDqualification and the R-threshold make more improvement than the OR-qualification and the Q-threshold, respectively.

6. Conclusion
Some techniques to improve the forced-convergence decoding have been studied. By replacing the criteria to deactivate variable nodes, the performance of the decoding scheme can be improved significantly. The proposed techniques are combinable, and we can change the combination according to the target code and communication channel, if needed.
