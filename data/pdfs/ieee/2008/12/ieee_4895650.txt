Abstract
We consider the challenges of finding good puncturing patterns for rate-compatible LDPC codes over additive white Gaussian noise (AWGN) channel. In previous works, the role of survived check nodes in puncturing patterns was considered with the limitations such as single survived check node assumption and simulationbased verification. In this paper, we analyze the performance according to the role of multiple survived check nodes and multiple dead check nodes. Based on theses analyses, we propose new algorithm to find good puncturing pattern for LDPC codes over additive white Gaussian noise (AWGN) channel.
1. INTRODUCTION
Design of good rate-compatible LDPC codes is a major concern for recent mobile communication standardization efforts. Pncturing is known as one of the most effective and convenient ways to change code rates. However the performance of punctured codes strongly depends on the nodes which are punctured (i.e., puncturing pattern). Therefore there have been many research studies to find good puncturing patterns for LDPC codes [2-8]. In [3], a puncturing scheme for LDPC codes with short and moderate block lengths was proposed in a systematic way and the performance of LDPC codes was analyzed by density evolution (DE) technique [9-10]. The punctured nodes were categorized by the required number of iterations to be recovered. The code structure, however, was too simplified such that each punctured node had only one survived

In this paper, we use the density evolution technique to analyze the positive effect of multiple survived check nodes and dead check nodes which are defined in section II. We show that the mean of log likelihood ratio passed between variable nodes and check nodes converges to correct value faster as the number of survived check nodes and dead check nodes increase. Based on this analytical result, we propose an algorithm to find good puncturing patterns and evaluate the performance of our algorithm.
2. BASIC DEFINITIONS AND NOTATIONS
In order to categorize punctured nodes following their characteristics, k-Step Recoverable (k-SR) nodes, survived check (SC) nodes and dead check (DC) nodes are defined as in [3]. The punctured nodes have zero channel LLR values and they spread to the other unpunctured nodes through neighboring check nodes during iterative decoding process. This is the main reason of the performance loss of punctured LDPC codes. Therefore, we should choose the punctured nodes which can have non-zero LLR values (It is defined as recover .) as fast as possible.
The k-SR node is defined as a punctured variable node which is recovered after k-th iterations where k corresponds to the recovery level. To become a k-SR node, it should neighbor at least one SC node and the rest of neighboring check nodes are DC nodes.

5. CONCLUSIONS
In this paper, we used the density evolution technique to analyze the positive effect of SC nodes and DC nodes. We showed that the mean of log likelihood ratio passed between variable nodes and check nodes converged faster as the number of SC and DC nodes increased. Based on these analytical results, new algorithm to choose good puncturing patterns for LDPC codes was proposed, which was maximize the number of SC nodes. Numerical results on the regular and irregular LDPC codes were well matched to the analytical

results. It is noted that the proposed algorithm shows better performance than the previous puncturing algorithm (grouping algorithm) especially at lower rates.
