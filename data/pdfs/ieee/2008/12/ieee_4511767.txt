Algorithms of Finding the First Two Minimum Values and Their Hardware Implementation
Chin-Long Wey, Senior Member, IEEE, Ming-Der Shieh, and Shin-Yo Lin

Abstract—Given a set of numbers , finding the minimum value of , min_1st, is a very easy task. However, efficiently finding its second minimum value, min_2nd, requires the derivations of min_1st and finding the minimum value from the set of the remaining numbers. Efficient algorithms and cost-effective hardware of finding the two smallest of are greatly needed for the low-density parity-check (LDPC) decoder design. The following two architectures are developed in this paper: 1) sorting-based (XS) approach and 2) tree structure (TS) approach. Experimental results show that the XS approach provides less number of comparisons, while the TS approach achieves higher speed performance at lower hardware cost. Since the hardware unit is repeatedly used in the LDPC decoder design, the proposed high-speed low-cost TS approach is strongly recommended.
Index Terms—Low-density parity-check (LDPC) codes, LDPC decoder, minimum values generator, tree structure (TS) approach.

I. INTRODUCTION

L OW-DENSITY parity-check (LDPC) codes [1] have been widely considered as a promising candidate error-cor-

recting coding (ECC) scheme for real-time communication

systems [2]–[18], such as DVB-S2 [8]. There exist design

tradeoffs among area, delay, power consumption, and perfor-

mance. Recently, a number of partially parallel LDPC decoders

using the min-sum (MS) scheme were proposed [2]–[4]. The

check-node function unit (CNFU) is used for row operation

to produce the outputs. Instead of producing the actual output

values, one can significantly reduce the memory space by

producing only the two smallest values, i.e., min_1st and

min_2nd, the index of min_1st, and signs of all output values

[18]. For long-code-length LDPC decoders with the iterative

MS decoding algorithm, the architecture requires 48 CNFUs

for block length of 1536 with a code rate of 1/2 in [2], and these

units are repeatedly used. Thus, it is absolutely necessary to

develop high-speed low-cost CNFUs.

This paper looks for the best possible architecture for se-

lecting the two smallest numbers, namely, min_1st and

min_2nd. Given a set of numbers,

For simplicity of discussion, the hardware unit that generates

architectures are developed and implemented: 1) sorting-based

(XS) approach and 2) tree structure (TS) approach. The XS ap-

that the XS approach indeed requires less number of compar-

isons for finding both min_1st and min_2nd. However, the TS

approach achieves better performance in both area and speed.

the algorithm and hardware implementation of the developed TS

approach. Finally, a concluding remark is given in Section IV.
