Rate Estimation Scheme for Weighted Bit-Flipping Decoding of Rate Compatible LDPC Codes
Kenji KITA∗† and Tetsuo TSUJIOKA∗‡
∗ Graduate School of Engineering, Osaka City University 3-3-138, Sugimoto, Sumiyoshi-ku, Osaka 558-8585, Japan †E-mail: kita@comm.info.eng.osaka-cu.ac.jp ‡E-mail: tsujioka-H20@info.eng.osaka-cu.ac.jp

Abstract
This paper considers a new rate estimation scheme for rate compatible LDPC (RC-LDPC) codes. We have investigated some estimation schemes for RC-LDPC codes even though there is no side information about code rates used at a transmitter. The previous schemes were based on log-likelihood ratio of sum-product decoders at a receiver. Because of too much computational complexity, simpler estimation schemes are expected to be developed. For this purpose, we focus on a simpler decoding scheme, thus weighted bit-flipping (WBF) decoding which have been much attention by many reseachers recently. In this paper, we propose a new rate estimation scheme and evaluate its performance. Simulation results show that the proposed scheme has advantages of much effectiveness and potential for estimating code rates.
1. INTRODUCTION
Low-density parity-check (LDPC) codes [1] are one of the most powerful error correcting codes and have decoding performance that is very near to Shannon limit [2][3]. Based on LDPC codes, rate compatible LDPC (RC-LDPC) codes have been proposed [4][5]. RC-LDPC codes are able to choose code rates flexibly. For an example, we can use RC-LDPC codes in flexible communication systems, such as adaptive encoding or unequal error protection (UEP), and so on. In the former system, code rates are changed according to channel state, while in the latter system, code rates are chosen for importance of each information. Owing to the flexibility, RC-LDPC codes are getting much attention lately in such system applications.
We have investigated some rate estimation schemes for RC-LDPC codes even though there is no side information about code rates used at a transmitter [6][7]. The previous schemes were based on posteriori loglikelihood ratio (LLR) of sum-product decoders at a receiver. We have showed good performance of these

schemes and suppression of decline in transmission efficiency (e.g. throughput) by computer simulations. In addition, we can flexibly change the code rate of block even in the same frame, therefore the system can adopt the rate of RC-LDPC codes in poor quality channels with rapidly changing channel state. Because of too much computational complexity, simpler estimation schemes are expected to be developed. For this purpose, we focused on a simpler decoding scheme, thus weighted bit-flipping (WBF) decoding [8] which has much attention by many reseachers. Especially, the WBF decoding is very effective for saving power consumption in such sensor networks. However, it is impossible to use the previous estimation schemes in the WBF decoder because the WBF decoding does not calculate posteriori LLR.
In this paper, we propose a new rate estimation scheme for the WBF decoding of RC-LDPC codes and a new transceiver applied the scheme. The receiver estimates code rate based on an evaluation value calculated in the WBF decoding. Simulation results show outstanding performance of the proposed scheme.
The correspondence is organized as follows. Knowledge about RC-LDPC codes and the WBF decoding is presented in Section 2. The rate estimation scheme for the WBF decoding and the transceiver corresponding to it are proposed in Section 3. Simulation results are discussed in Section 4. Finally, conclusions are given in Section 5.

2.2. WBF Decoding
The WBF decoding corrects errors by using syndromes computed by hard decision information and reliabilities of parity-check equations weighted by soft decision information. It is calculated which bit has the lowest reliability (the highest unreliability), and the WBF decoding progresses by flipping a corresponding bit.
Let H = (hm,n) be an M × N parity-check matrix. Define a set of bits which are related to a parity-check equation m as N (m) ≡ {n : hm,n = 1}. Similarly, define a set of parity-check equations which are related to a bit n as M (n) ≡ {m : hm,n = 1}. Let C denote an (N, K) LDPC code of block length N , information length K and parity length M , which is defined by a parity-check matrix H. Now, consider an

LDPC codeword c = (c1, c2, · · · , cN ) of C. Here, c is converted into x = (x1, x2, · · · , xN ) by a conversion equation of xn = 1 − 2cn, n = 1, 2, · · · , N , where we assume BPSK modulation. At a receiver, we map a

The receiver also includes three parts: decoders corresponding to candidate code rates, a rate estimator for the received codeword’s rate, and a demodulator. At the demodulator, we demodulate received signal into the received codeword buffer. In each decoder, an evaluation value zr for the code rate R(r) (r = 1, 2, · · · , rmax) is calculated, respectively, and forwarded to the rate estimator. Then, we suppose that a problem of frame synchronization is established. Thus, the first codeword can be synchronized by a preamble located at the head of each frame. The rate estimator compares all the evaluation values from decoders and estimates the code rate.

5. CONCLUSIONS

We proposed a new rate estimation scheme for RCLDPC codes for the WBF decoding even though there is no side information about code rates used at the transmitter. We evaluated performance of the proposed scheme by computer simulations. As a result of computer simulations, we obtained that optimum imax which is an important component of evaluation values is 1. This contributes to the reduction of the computational complexity greatly. Comparison PREEs with BLERs for RC-LDPC codes showed that the proposed scheme guarantees accurate transfer of rate information. We also observed that the proposed scheme is much better than code rate transfer using BCH encoding as side information. Moreover, from the point of view of data transmission efficiency, the proposed scheme is considerably effective due to lack of necessity of additive rate information. Therefore, the proposed scheme is quite useful and powerful scheme for the WBF decoding.
