Three-Level Error Control Coding for Dependable Solid-State Drives
Haruhiko Kaneko, Takuya Matsuzaka, and Eiji Fujiwara Graduate School of Information Science and Engineering, Tokyo Institute of Technology
2-12-1 Ookayama, Meguro-ku, Tokyo 152-8552, Japan. hkaneko@fuji.cs.titech.ac.jp matsuzaka@fuji.cs.titech.ac.jp fujiwara@cs.titech.ac.jp

Abstract
Solid-state drive (SSD) has advantages over hard-disk drive (HDD) in terms of power consumption, random access time, and resilience to shock and vibration. Large capacity SSD usually requires high-density multi-level cell flash memory fabricated with deep-submicron process. High-density memory chips, however, are vulnerable to soft errors caused by, for example, fluctuations of gate voltage and charge level in the floating gate. This paper proposes a hierarchical three-level error control coding suitable for the dependable SSD. The proposed coding is capable of correcting multiple random bit errors, as well as of recovering from single chip failures. Evaluation shows that the proposed coding scheme provides strong error correction capability. For example, bit error rate (BER) of the SSD is reduced from 1.08 × 10−4 to 2.44 × 10−19 by using the proposed coding, that is, using two BCH codes with different error correction capabilities for the first and the second levels, and the simple parity-check code for the third level. Extra one spare memory chip in the SSD improves mean time to data loss (MTTDL) from 13 yeas to 34 years.
Keywords: flash memory, solid-state drive (SSD), BCH code, bit error rate (BER), mean time to data loss (MTTDL).
1. Introduction
Storage capacity of the flash memory chip is rapidly increasing in recent years, and therefore this type of memory is widely used in digital camera, mobile phone, PDA, USB memory, and solid-state drive (SSD). The recent SSD has storage capacity comparable to small-sized hard disk drive (HDD), and is mainly used in laptop computers. Table 1 shows the typical specifications of the SSD [1] and the HDD [2]. This indicates that the SSD has some advantages over the HDD in terms of power consumption, random access time, and resilience to shock and vibration. Hence, the SSD is suitable for the resource-constrained computer

systems, such as small-sized computer and satellite data recorder, and for the high-speed database server requiring short random access time. Also, global warming issues strongly require computer systems to reduce power consumption [3], and from this point, low-power SSD would be suitable for the storage servers in the data center, originally consuming huge amount of electricity [4], that reduce much power consumption.
Flash memory cell stores charges in a floating gate, that is, the quantity of electrons inserted in the floating gate expresses a data value [5]. Single-level cell (SLC) flash memory chip has two charge levels in the floating gate, and thus it can store one-bit data in each memory cell, whereas multilevel cell (MLC) chip has 2b charge levels, and it can store b-bit data in each memory cell [6], where typical value of b is two or three. Hence, storage density of the MLC chip is higher than that of the SLC chip.
Similar to the conventional semiconductor RAM chips, the flash memory chips suffer from soft errors caused by, for example, noises and radioactive particles [7]. Soft error probability will increase when deep-submicron design, such as 56nm CMOS design, is employed in chip fabrication. Also, the MLC chip is more vulnerable to soft errors than the SLC chip because small fluctuation of gate voltage or charge level in the floating gate causes errors in the stored data. In addition, the large capacity SSD has multiple flash memory chips in the drive, and hence increased number of memory chips result in increased failure rate of

Error control coding is very effective to improve reliability of the storage systems [8]. From this, simple error control codes, such as Hamming code or BCH code, have been employed in each flash memory chip. With using these chips, large scale flash memory chip arrays employ twolevel coding, that is, Reed-Solomon code in each chip and simple parity-check code in the whole array [9]. This paper proposes a three-level error control coding suitable for the SSD, and demonstrates an evaluation of the coding in bit error rate (BER) and mean time to data loss (MTTDL) for the large capacity SSD.

2. Flash memory structure
Figure 2 shows a NAND flash memory chip which has an M × nb cell-array, that is, M × nb bits memory capacity. Each row in the cell array corresponds to a page, and hence the memory chip has M pages, each having n×b bits. Each page has an information part for data bits and a check part for check bits of the error control code. The information part is expressed by k × b bits, whereas the check part by r × b bits. For example, 4-Giga bit (Gb) flash memory chip has the parameters of k = 2048, r = 64, b = 8, and M = 256 × 210. The Hamming code and the BCH code are typical random bit error correcting codes applied to the flash memory chip.

3. Three-level error control coding for dependable SSD
Figure 4 presents check-bit allocation in the three-level coding. Each of the first N − 1 pages in the cluster has three segments: information, shared-check, and page-check segments. The last page has three check segments, namely, parity-check, shared-check, and page-check segments. The matrices Di, P, Qi, and Ri in Fig. 4 are defined as follows.
• Di = [ di,0 di,1 . . . di,b−1 ]: Binary k×b matrix indicating the information segment in the i-th page, where di,j is a binary column vector with length k.
• P = [ p0 p1 . . . pb−1 ]: Binary k × b matrix indicating the parity-check segment, where pj is a binary column vector with length k.

The information segment is encoded by three error control codes, that is, a parity-check code CPC for first level coding, an error correcting code CCL for second level coding, and an error correcting code CPG for third level coding. The check segments P, Qi, and Ri correspond to check bits of CPC, CCL, and CPG, respectively. The following presents three-level coding using parity-check code CPC, t -bit error correcting (n , k ) code CCL, and t-bit error correcting (k + r0 + r1, k + r0) code CPG, where k = kN and n = k + r1bN .

6. Conclusion
This paper has proposed a hierarchical three-level error control coding suitable for the dependable solid-state drives (SSDs). In this coding, a simple parity-check code is first applied to the whole chips in the SSD, next, a cluster including multiple pages is encoded by strong random bit error correcting BCH code CCL, and finally each column of a page is encoded by random bit error correcting BCH code CPG. This paper has also derived bit error rate (BER) and mean time to data loss (MTTDL) of the dependable SSD. Evaluation has demonstrated that the proposed coding provides strong random bit error correction capability

and long MTTDL suitable for the dependable SSD. For example, in the SSD having N = 9 memory chips with raw BER of 1.08 × 10−4, the proposed coding gives BER of 1.80 × 10−17. Also in the SSD with one spare chip where each chip has mean time to failure of 100 years, MTTDL of the SSD is improved from 13 years to 34 years.
Experimental reliability evaluation of the proposed method in a practical environment will be required in future. Also, evaluation of the encoder/decoder circuit of the proposed coding is remained in terms of complexity, delay, and throughput.
