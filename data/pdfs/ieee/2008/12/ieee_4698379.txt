Evaluation of the Concatenation of LDPC and RS Codes in Magnetic Recording Channel Using Field
Programmable Gate Arrays
Seungjune Jeon, Xinde Hu and B. V. K. Vijaya Kumar Data Storage Systems Center Carnegie Mellon University Pittsburgh, Pennsylvania 15213 Email: sjeon@andrew.cmu.edu

Abstract—Concatenations of inner low-density parity-check (LDPC) codes and outer Reed-Solomon (RS) codes were evaluated in magnetic recording channel models by simulations using field programmable gate arrays (FPGAs). Fixed inner LDPC codes were used and the optimal code rates for the outer RS code were obtained. It is observed that the outer RS codes do not always improve sector error rates for a fixed user bit density. The effect of various parameters on the sector error rates was evaluated and discussed.
Index Terms—concatenation, Reed-Solomon codes, low-density parity-check (LDPC) codes, magnetic recording, sector error rates.
I. INTRODUCTION
While Reed-Solomon (RS) codes have been used for a long time in magnetic data storage systems, low-density paritycheck (LDPC) codes are being investigated as future error correcting codes because of their potential coding gain over RS codes. In order to obtain improved coding gains by inner LDPC codes while handling burst errors and obtaining guaranteed error correction performance by outer RS codes, concatenation of LDPC codes and RS codes has been proposed and investigated [1]–[3] for magnetic recording channels. While one research study [1] suggests that combining the two can lead to better performance, another research study [2] indicated that combining LDPC and RS codes does not always improve performance when compared to using LDPC code only. Unfortunately, it is not possible to directly compare those results since many parameters are different. This work investigates under what conditions concatenation improves performance.
To our knowledge, the three papers mentioned [1]–[3] comprise the entire literature to date regarding the concatenation of LDPC and RS codes for magnetic recording channels. These papers have different results about an optimum T of outer RS codes, where T denotes the maximum number of erroneous symbols that an outer RS code can correct in a codeword. The variable T is optimized in order to achieve the minimum sector error rate for a fixed inner LDPC code, a fixed user bit density, and a fixed signal-to-noise ratio (SNR). In [1], it was shown that the optimum T exists and the range was 6 ≤ T ≤ 13 depending on the type of inner LDPC code and the number of global iterations. This suggests that the concatenation of LDPC and RS codes offers better performance than using LDPC code alone in some cases. On the other hand, it is interesting to note that another recent effort [2] showed that T = 0 can be the optimum value at which the minimum SNR is required to achieve a given sector error rate. In other words, in some cases using an LDPC code alone is better than concatenating LDPC and RS codes. Although there was another investigation [3] of the concatenation method, it was focused on designing inner LDPC code rather than finding the optimum outer RS code rates.
To deal with the differences shown in previous research, this paper evaluates the effect of major parameters on the optimum T of the outer RS codes and discusses the conditions in which the concatenation is an effective solution or not.

Equalizer coefficients were determined for different channel bit densities to obtain the partial response (PR) target 1221. Soft-Output Viterbi Algorithm (SOVA) was used as a detector but no data-dependent noise prediction (DDNP) for the transition jitter noise was used in this work. The number of local iterations and global iterations were set to predefined numbers. Their effect will be discussed later. At the output of the LDPC decoder, a symbol error counter which emulates the RS decoder was used. An RS symbol is defined to be 10 consecutive bits assuming no bit-level interleaving. If there is any error within a 10-bit RS symbol, it is treated as one symbol error. If the number of symbol errors is greater than T , one sector error is claimed. The RS code length is n = (LDPC message length)/10 = 438 symbols for this case. Since LDPC message length is 4376 bits, four extra bits without information were padded to make an RS codeword, which is 4380 bits long. Finally, the sector error rates can be obtained after counting symbol errors for a number of codewords during the simulation.

III. RESULTS

The two codes share the same code length 4923, the same code rate 8/9, the same column weight 3, the same quasi-cyclic structure, i.e., the same nine 547×547 circulants, and each of the two codes has a girth of at least 6. B. Sector Error Rates
Fig. 4 shows the sector error rates as a function of T for a fixed user bit density u = 1.78, the inner LDPC Code 1, 6 global and 6 local iterations. We can see that the optimum T = 0, i.e., the sector error rate is minimum when using LDPC code alone rather than using concatenation with outer RS code.
Fig. 5 shows the distribution of the number of bit errors and the number of symbol errors given that there is an LDPC codeword error at the output of LDPC decoder. In the bit errors vs. symbol errors figure, one dot represents one erroneous sector which contains the corresponding number of bit errors and symbol errors. Only AWGN was applied as noise and the SNR was 16.73 dB. It can be seen that whenever there is any error, there are 50 symbol errors in an erroneous codeword on average. No erroneous codeword has less than 27 symbol errors. Therefore, in order to correct these errors, T should be at least higher than 27 and effectively about 50.

C. Effect of Transition Jitter Noise
Fig. 6 shows the distribution of the number of bit errors and the number of symbol errors for the same condition except that transition jitter noise σj = 0.195Tbit were applied without AWGN. It can seen that the distribution has essentially the same behavior as in the AWGN only case. We conjecture that the mixture of transition jitter noise and AWGN case will also exhibit similar error distribution although that was not verified in this work. Although real data storage systems have both AWGN and transition jitter noise, the two types of noise were not mixed in this work in order to see the respective effects clearly and to avoid the difficulties in defining SNRs.

E. Effect of the Number of the Local Iterations
Figs. 8(a) and 8(b) show the sector error rates and the distribution of the symbol errors with the same conditions as in the previous subsection except that the number of local iterations is increased to 25. The symbol error distribution is similar and it can be seen that the outer RS code does not improve sector error rates in this case, either.
F. Effect of the Error Floor of the Inner LDPC code
Fig. 9 shows the effect of the error floor of the inner LDPC codes on the symbol error rates at the input of the RS decoder as a function of T for a fixed user bit density and fixed SNR value. For Code 1 with no error floor, the symbol error rates at the input of the RS decoder monotonically decrease as T decreases. On the other hand, for Code 2 with obvious error floor, the symbol error rates at the input of the RS decoder stop decreasing below a certain value of T depending on SNR as T decreases. Below that turning point, it is highly likely that the sector error rates at the output of the RS decoder may increase. Therefore, it is likely that an optimum T value may be found.
In order to test this assumption analytically, a symbol error probability mass function (pmf) at the output of the LDPC decoder is needed before applying the multinomial model [7]. Without the pmf information, however, it might be assumed that all symbol errors are independent and approximately evaluate the sector error rate from Fig. 9 using the binomial model. The results of this binomial approximation for Code 2 are shown in Fig. 10, which shows similar optimum T values as in [1]. It might be interesting to note that the sector error rates at the optimum T in this figure are so sensitive to SNR

value that a small increase in SNR and T values can greatly improve the sector error rate for this error floor case.
IV. CONCLUSIONS
Effect of various parameters on the concatenation of LDPC and RS codes was discussed. We conclude that using outer RS codes does not always improve the overall sector error rates for a fixed user bit density and a fixed SNR. Especially when the number of iterations is large enough and error floor of the inner code is not severe, using LDPC codes alone appears to lead to better performance than concatenation with outer RS codes. It was observed that although the 10-bit symbol error distribution of the LDPC decoder outputs is different for different number of global iterations, the optimum rate of RS codes did not change. It was not observed that transition jitter noise or the number of local iterations greater than six has significant effects on the optimum rates of the outer RS codes.
