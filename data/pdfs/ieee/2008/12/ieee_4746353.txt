Efficient Encoding for Dual-Diagonal Structured LDPC Codes Based on Parity Bit Prediction and Correction
Chia-Yu Lin, Chih-Chun Wei, and Mong-Kai Ku Department of Computer Science and Information Engineering
National Taiwan University, Taipei, Taiwan 106, ROC Email: {f93084, r95152, mku}@csie.ntu.edu.tw

Abstract—In this paper, an efficient encoding scheme for dual-diagonal structured LDPC codes is proposed. Our encoding algorithm employs parity bit prediction and correction to break up the data dependency within the encoding process. The encoder can achieve higher level of parallelism and better hardware utilization. The number of required clock cycles for encoding one codeword can be reduced to achieve higher throughput performance. The proposed scheme can be directly applied to IEEE 802.11n and 802.16e dual-diagonal codes without any matrix modification. A low-complexity encoder architecture is proposed and implemented to verify these characteristics. Results show that the proposed architecture outperforms conventional works in terms of throughput and throughput/area ratio.
Index Terms—LDPC codes, dual-diagonal, encoding.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1][2] have attracted much attention in the last decade due to their capacity-approaching error correcting performance. One drawback of LDPC codes is their high encoding complexity. The complexity of directly encoding by using the generator matrix is quadratic in the block length of the LDPC code. In order to reduce encoding complexity, LDPC codes with dual diagonal structure is adopted by the latest next-generation wireless LAN standard, IEEE 802.11n [3] and wireless metropolitan area network (MAN) standard, IEEE 802.16e (WiMAX). This class of codes can be encoded in near-linear time by the method proposed in [4] and in linear time by the sequential method in [5]. Another efficient encoding scheme [6][7] for dual-diagonal LDPC codes is based on arbitrary bit generation and correction. This approach achieves low encoding complexity and reduces the number of required cycles for encoding each codeword. The matrix structure is exploited in the encoding process to correct the parity bits. However, their approach can not be directly applied to encode IEEE 802.11n or 802.16e LDPC codes. Modification of the matrix structure is necessary when using this scheme. From their results in [6], this modification causes performance degradation and higher error floors compared to the original 802.11n dual-diagonal codes.
A generalized encoding scheme based on the technique of parity bit prediction and correction is proposed in this paper. The proposed scheme can be directly applied to encode IEEE 802.11n and 802.16e dual-diagonal LDPC codes without any

B. Proposed Encoding Scheme Based on the previous discussion, our proposed encoding
procedure is summarized as follows.

Proposed scheme:
Step_1._Set p0’ (i.e., b0, b1, …, bz-1) as any binary vector.
Step_2._Compute the vector λ = Hss by circularly shifting and accumulating the subblocks of s. (We denote λ λ = [c0 c1 … cm-1] and i = [ciz ciz+1 … c(i+1)z-1] for i = 0,
1, …, mb-1) Step_3._[Forward Derivation] Compute p1’, p2’, … , px’. Step_4._[Backward Derivation] Compute pmb-1’, pmb-2’ … ,
px+1’ and py. Step_5._Compute p0 by adding px’, and py. Step_6._Compute the correction vector f by circularly shifting
the sum of p0 and p0’ to the left by d positions. Step_7._[Correction] If f is a nonzero vector, then compute pi
by adding pi’ and f for i = 1 to mb-1. Otherwise, pi is simply pi’.

IV. PROPOSED ENCODER ARCHITECTURE An encoder hardware architecture is also proposed in [5] based on their sequential algorithm. In this architecture, kb barrel shifters are used for parallel computation of Hss. Nevertheless, in every computation cycle, certain barrel shifters will be idle due to the existence of zero submatrices in Hs. The proposed architecture is shown in Fig. 3. To minimize the number of idle barrel shifters, our proposed architecture
uses only 2 × α barrel shifters, where α is the maximum
number of nonzero submatrices in one block row in Hs. Multiplexers are used to select information subblocks (i.e., si)

Fig. 3. Proposed encoder architecture.
Following the proposed algorithm, the encoder architecture is partitioned into four phases. The initial prediction vector p0’ is set to zero to simplify the hardware. In the first phase, the multiplexer selects information subblocks and barrel shifters move the corresponding information subblocks from topmost and bottommost block rows of Hs simultaneously. In the second phase, λi values are computed and the prediction parity vectors pi’s are derived as soon as any λi is available. The number of required clock cycles for encoding can be reduced because the λi computation and pi’ derivation are done in parallel. In the third phase, the prediction parity vectors are stored into memory. In the meantime p0 and the correction vector are obtained by one XOR operation and one shifting operation respectively. In the fourth phase, the prediction parity vectors are corrected by the correction vector via XOR gates to get the correct parity bits. There is no data dependency among these correction operations so they can be computed in parallel.
By the definition in [8], CPC represents the required number of clock cycles required for encoding per codeword. The CPC of the proposed encoder is explained as follows. In the first phase, two block rows in Hs are simultaneously processed in each cycle, so mb/2 cycles are required. In the
second phase, it takes one cycle to accumulate the λi value to
compute a prediction parity vector for each block row. In the third phase, it takes one cycle to store the prediction vector computed from the previous cycle into memory, Since the operations of the first three phases are done in a pipelined way, it takes a total of mb/2+2 cycles. In the fourth phase, all prediction vectors are loaded and corrected in one cycle. Hence, it takes only mb/2+3 cycles per codeword.
In conventional sequential encoding schemes [5][8], all the
parity bits can not be derived until all the values of λi s and p0
are computed and the derivation of each parity vector is also

VI. CONCLUSIONS
We have proposed an efficient encoding scheme for dualdiagonal LDPC codes based on parity bit prediction and correction. The proposed algorithm and hardware architecture can be employed to encode dual-diagonal codes defined in IEEE 802.11n and 802.16e standards without any matrix modifications. Data dependency within the encoding process is relaxed to allow highly parallel operations and better hardware utilization. Our implementation results show that the proposed low-complexity hardware architecture achieves higher throughput than conventional sequential encoding based works for all but the shortest code length and better throughput/area ratio for all lengths.

As shown in Fig. 4, the proposed encoder architecture provides better throughput for a longer code length while the work in [8] does not have this kind of speed-up. This is because the proposed architecture has a constant CPC for all code lengths while the CPC in [8] increases as the code length gets longer. The proposed encoder provides up to 2.98 times throughput as compared with the work in [8]. In other words, the proposed encoder can achieve the same throughput as the work in [8] on lower clock frequency, which can lead to looser critical timing path constraint and lower power consumption.
