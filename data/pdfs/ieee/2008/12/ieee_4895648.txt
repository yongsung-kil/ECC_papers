Code Design and Performance Analysis Using a 2-Level Generalized Tanner Graph on the Binary Erasure Channel
Eirik Rosnes
Department of Informatics, University of Bergen N-5020 Bergen, Norway E-mail: eirik@ii.uib.no

Abstract
In this work, we consider code design and performance analysis using a 2-level generalized Tanner graph on the binary erasure channel. The 2-level generalized Tanner graph is composed of traditional variables nodes on the left side and generalized check nodes on the right side. The generalized check nodes are (dc, dc − 2), dc ≥ 3, binary linear codes. Iterative decoding is applied to the 2-level generalized Tanner graph using maximum a posteriori (MAP) erasure correction in the generalized check nodes. With MAP erasure correction, each check node decoder removes as much erasures as possible, even if it cannot resolve all erasures. Code design is done using density evolution, and we will show that the proposed scheme achieves both a better design rate and lower decoding complexity compared to the traditional scheme with only single parity-check nodes.

Note that the minimum distance dmin of the code defined by the parity-check matrix in (1) is 2 when either x0 > 1, x1 > 1, or x2 > 1, which is the same as for single parity-check codes. The code can correct all single erasure patterns, and x0 · x1 + x1 · x2 + x0 · x2 erasure patterns with two erasures. Under the constraint of n = x0 + x1 + x2 being a constant, the code can correct more erasure patterns with two erasures when x1 > 0 than with x1 = 0. For instance, with x1 = ⌊n/3⌋ and n ≡ 0 or 2 (mod 3), the code can correct ⌈n/3⌉(⌈n/3⌉ + 2⌊n/3⌋) erasure patterns with two erasures, while the corresponding maximum number with x1 = 0 is ⌊n/2⌋⌈n/2⌉. The motivation behind this choice of generalized check nodes is decoding complexity. In fact, the decoding complexity on the binary erasure channel (BEC) for each generalized check node assuming successful iterative decoding of a 2-level generalized Tanner graph, is independent of the value of x1 when n = x0 + x1 + x2 is a constant (see Lemma 1 below). Thus, any performance improvement we get by using x1 > 0 comes at no additional decoding cost. To improve performance further, one could try to incorporate check nodes with dmin ≥ 3, but unfortunately, the decoding complexity of maximum a posteriori (MAP) erasure correction in the generalized check nodes will increase. With MAP erasure correction, each check node decoder removes as much erasures as possible, even if it cannot resolve all erasures.

Iterative decoding on a 2-level generalized Tanner

graph on the BEC can be formulated as follows. As-

sociate three binary labels with each check node. The

initial value of the ith, i = 0, 1, 2, binary label is equal

to the modulo-2 sum of the constrained variable nodes

within the ith group connected to the check node. In

addition, if all the variable nodes within a group are

known, mark the group “finished”. Otherwise, mark

the group “unfinished”. For all unfinished check nodes

(i.e., at least one group connected to the check node is

unfinished) do the following.

a) If all the variable nodes within a group are known, mark the group “finished”.

b) If at least one group connected to the check node is finished and the values of all but one of the variable nodes within the second (or third) group connected to the check node are known, then set the missing variable bit within the second (or third) group to the XOR of the binary values of the second and first (or third and first) groups, add modulo-2 the value of the missing variable node to the binary value of all groups containing the missing variable node, and mark the second (or third) group “finished”.

c) Repeat for the considered check node until there is no further progress.

The procedure is done sequentially, i.e., one check node at a time, and continues until all check nodes are marked “finished”, or the algorithm cannot continue

further. We remark that the algorithm above is a generalization of the peeling decoder of Luby et al. in [7] to 2-level generalized Tanner graphs.
Note that the above algorithm implements MAP erasure correction in the generalized check nodes, since all codewords of the row space of (1) are used in an iterative fashion when decoding the 2-level generalized check nodes [8]. Furthermore, the above algorithm is slightly different from generalized iterative decoding on the BEC as described in [9], even in the case of order 2.
Lemma 1 The decoding complexity (in terms of the number of binary additions) on the BEC for each generalized check node assuming successful iterative decoding according to the above algorithm of a 2-level generalized Tanner graph, is at most equal to its degree minus 1, i.e., x0 + x1 + x2 − 1. Thus, the decoding complexity is independent of the values of x1 when the check node degrees stay the same.
Sketch of proof: The result follows from the above decoding algorithm, since the number of binary additions required in each 2-level generalized check node (assuming successful iterative decoding) is at most equal to its degree minus 1.
