Abstract‚ÄĒThe iterative algorithm, for low complexity linear programming (LP) decoding of low-density parity-check (LDPC) codes, proposed by Vontobel and Koetter, is considered. In this paper the convergence rate and computational complexity of this algorithm are studied using a scheduling scheme that we propose. In particular we are interested in obtaining a feasible vector in the LP decoding problem, with objective function value whose distance to the minimum, normalized by the block length, can be made arbitrarily small. It is shown that such a feasible vector can be obtained with linear, in the block length, computational complexity. Improved bounds on the convergence rate are also presented. The results extend to generalized LDPC (GLDPC) codes. It is also shown that previous results for LDPC and GLDPC codes, on the ability of the LP decoder to correct some fixed fraction of errors, hold with linear computational complexity when using the approximate iterative LP decoder.
I. INTRODUCTION
The method of linear programming (LP) decoding of linear codes was proposed by Feldman et al. [1]. There are some close connections between LP decoding and iterative decoding of low-density parity-check (LDPC) codes. However, a major disadvantage of LP decoding concerns its complexity of decoding. Suppose that we operate below the density evolution threshold and that the block length is large enough so that the tree assumption holds. Then one can retrieve the transmitted codeword, in linear in the block length computational complexity, up to an arbitrarily small fraction of errors. On the other hand, when using a general purpose LP solver for LP decoding of LDPC codes, the computational complexity, although polynomial in the block length, may not be linear. Several authors have proposed algorithms for low-complexity LP decoding. In this work we focus on the iterative GaussSeidel-type algorithm of Vontobel and Koetter [2]. Using general properties of Gauss-Seidel-type algorithms for convex minimization, it was shown in [2] that the algorithm converges. In this paper we study the new approach of [2] under an appropriate scheduling scheme that we propose. We analyze the convergence rate and computational complexity of this algorithm. In particular we are interested in obtaining a feasible vector in the LP decoding problem, with objective function value whose distance to the minimum, normalized by the block length, can be made arbitrarily small. It is shown that such a feasible vector can be obtained with linear, in

the block length, computational complexity. Combined with previous results, that have shown that the LP decoder can correct some fixed fraction of errors [3], [4], we conclude that this error correction can be achieved with linear computational complexity. An initial version of this work appears in [5]. In the current version an improved convergence rate analysis is presented in Section III after Theorem 1, and it is also shown that the results extend to generalized LDPC (GLDPC) codes.

V. CORRECTION OF FIXED FRACTION OF ERRORS IN
LINEAR COMPLEXITY
In this section we argue that the error correction results in [3] and [4] can be achieved in linear complexity using the approximated LP decoder (23)-(33). The first step in the analysis is to show that the performance of the algorithm (23)-(33) is independent of the transmitted codeword for any binary-input output-symmetric (BIOS) channel. The proof, provided in [6], is similar to the corresponding proof for message passing algorithms, and uses induction on the iteration index. Thus without loss of generality we can assume that the all-zero codeword was transmitted. We consider transmission over the binary symmetric channel (BSC). In this case we may re-scale the ő≥i‚Äôs such that ő≥i = 1 if yi = 0 and ő≥i = ‚ąí1 if yi = 1 without effecting the solution produced by the LP decoder.
In [3] it was shown, for left regular LDPC codes which are sufficiently good expanders, that the LP decoder can correct any error pattern with some fixed fraction of errors or less. This result was then combined with the fact that, asymptotically in the block length, typical codes in the corresponding LDPC ensemble have the required expansion. The results in [3] were recently improved in [4] by relaxing the requirement that the code should be able to correct any set of some fixed fraction of errors to the requirement that it only needs to correct almost all such error patterns. Our result asserts the following.
Theorem 3: Theorem 1 in [3] (Theorem 1 in [4], respectively) remain valid, when the approximate iterative LP decoding algorithm (23)-(33) is applied, instead of the exact LP decoder, with complexity of order O(N ) (o(N log N )), except that we define success as decoded word with vanishing bit error rate.
The complexity for Theorem 1 in [4] is o(N log N ) rather than O(N ) only since the ensemble considered is such that the right degrees of the code, which have a Poisson distribution, can only be bounded by some number o(log N ) (and not by a constant independent of N as in other standard LDPC ensembles). A similar computational complexity problem occurs for this ensemble when using iterative decoding.
The proof, provided in [6], relies on the proofs of [3, Theorem 1] and [4, Theorem 1], that prove the success of the LP decoder by showing that a dual witness u exists.
There are various standard possibilities how to correct the remaining errors (i.e., how to augment the decoder so that it would have a vanishing word error rate). These methods are also applied in iterative decoding schemes that guarantee

a vanishing bit error rate, but not a vanishing word error rate. One possibility is to use concatenated coding, with asymptotically negligible rate loss. Another possibility [9] is to switch to the flipping algorithm after we have concluded algorithm (23)-(33) with additional complexity of order O(N ).
The results in this section can also be extended to GLDPC codes. First, the proof that the iterative LP decoder is independent of the transmitted codeword easily extends to GLDPC codes. Now, [10] considers a GLDPC code, C, which comprises of a (l, r) regular bipartite graph, which is a sufficiently good expander, and with constituent linear codes with block length r and minimum distance «ęr. It is shown [10, Theorem 2] that the GLDPC code can correct a linear number of errors in any transmitted codeword. The approximate LP decoder can achieve the same error correction capabilities in linear complexity provided that we allow a vanishing bit error rate in the decoded word (or apply additional operations to correct the remaining errors as described above).
