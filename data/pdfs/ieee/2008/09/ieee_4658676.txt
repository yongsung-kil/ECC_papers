2008 5th International Symposium on Turbo Codes and Related Topics

High Throughput Hardware Architecture for (1440,1344) Low-Density Parity-Check Code
Utilizing Quasi-Cyclic Structure
Hiroyuki Yamagishi and Makoto Noda Core Device Development Group, Sony Corporation 5-1-12 Kitashinagawa Shinagawa-ku, Tokyo, 141-0001 Japan
Email: Hiroyuki.Yamagishi@jp.sony.com

Abstract—High throughput architecture of an encoder and a decoder for a quasi-cyclic low-density parity-check (LDPC) code is proposed. A new systematic encoding method is carried out by polynomial manipulation. The proposed decoder architecture, where the check-node process is split into two processes so that the memory access becomes column-wise, enables overlapped message-passing for any parity-check matrix. The hardware architecture for the check-node processes utilizing a quasi-cyclic structure does not require complex multiplexers. Hardware employing the proposed architecture for a (1440,1344) LDPC code designed for high throughput millimeter wave application is evaluated using 65 nm CMOS technology. The gate count of the encoder for 3 Gbps and 6 Gbps throughput is 2.5 k and 3.1 k, respectively, and the gate count of the decoder for 8 iterations is 304 k and 409 k, respectively. A bit-error rate of 10–6 is obtained at Eb/N0 of 5.9 dB, and the estimated power consumption of the decoder is 58 mW for 3 Gbps and 86 mW for 6 Gbps.
I. INTRODUCTION
Low-density parity-check (LDPC) codes first introduced by Gallager [1] have received considerable attention due to their excellent error-correction performance. Some recent technology standards, such as digital video broadcasting standard DVB-S2, and wireless local-area network standard IEEE 802.11n, have adopted LDPC codes. For their inherent parallelizable decoding algorithm, LDPC codes are also expected to be used for systems which require an even higher transmission rate. One such application is uncompressed video transmission using a millimeter waveband. To enable such applications, we designed a rate-14/15 quasi-cyclic regular LDPC code using a method previously reported [2], which is currently employed as a coding scheme in the draft document for the 60 GHz millimeter-wave-based wireless personal-area network standard IEEE 802.15.3c [3]. The rate 14/15, which may be the highest code rate for LDPC codes to date in communication standards, enables a throughput over 3 Gbps using quadrature phase shift keying (QPSK) modulation in a 2.16 GHz bandwidth.
Implementation of such a high throughput LDPC decoder with compact hardware using current LSI technology is a challenging task. The decoding algorithm iterates message-passing between two types of nodes, denoted as check nodes and variable nodes. The fully parallel decoder architecture can achieve very high throughput, however suffers

from inconveniently large hardware and interconnect complexity [4]. To reduce the amount of processing units, partially parallel decoder architectures have been proposed [5]-[7]. Because a check- or variable-node processing unit is usually shared by several check- or variable-node processes, the messages exchanged between the processing units have to be stored in memory. As a node processing unit requires messages generated by node processing units of the other type, message-passing between the two types of nodes cannot be processed simultaneously. Some methods have been reported to allow the processes to overlap by appropriate scheduling, which enables a throughput gain for some codes [6], [7].
When the throughput requirement is modest, the memory is efficiently implemented with a random-access memory (RAM). However, as the hardware needs to be parallelized for a higher throughput, a RAM with a large bit width and small depth would be area-inefficient. Meanwhile, implementing the memories with flip-flops (FFs) usually requires multiplexers and demultiplexers bridged with the processing units.
In this paper, we propose a simple encoding method for a quasi-cyclic code, and an LDPC decoding scheduling which improves the latency of overlapped message-passing. As the check-node operations are serialized, the decoder does not suffer from a large delay in the check-node calculation even for a large row weight. By utilizing the quasi-cyclic structure, composing memories used for the check-node processes of FFs with small additional circuits, which are connected cyclically, can eliminate the need for multiplexers and demultiplexers. We show the feasibility of an encoder and a decoder which can achieve a throughput over 6 Gbps with relatively small hardware for an LDPC code.
II. QUASI-CYCLIC LDPC CODES
In this section, the quasi-cyclic LDPC code is briefly reviewed. An (N, K) LDPC code of length N and dimension K is characterized by its M×N sparse parity-check matrix H. M rows in H correspond to constraints on the code. When every column of H has wc non-zero elements and every row of H has wr non-zero elements, the code is called a regular LDPC code, otherwise an irregular LDPC code. Usually, Step 1 and 2 are processed by check-node processing (CNP) units and variable-node processing (VNP) units, respectively. In a partially parallel decoder, each processing unit is shared by several node processes. The CNP units read Qm,n from memory with row-wise access, whereas the VNP units write Qm,n to memory with column-wise access, so the memory access control becomes complex.
The VNP unit which computes Qm,n for variable node n has to wait for all messages sent from check nodes m'  M(n). Similarly, the CNP unit which computes Rm,n for check node m has to wait for all messages sent from variable nodes n'  N(m). This implies that the computations by the VNP units and the CNP units generally cannot be overlapped. Fig. 3(a) shows the timing chart where the CNP and VNP units operate alternately [5]. If no separate initialization step exists, multiplexers in the check-node processes are required in the first iteration.
To improve the throughput, overlapped message-passing decoding has been proposed [6], [7]. Fig. 3(b) shows the timing chart of the overlapped message-passing decoding. Waiting time w that avoids memory-access conflict can be minimized by appropriate scheduling, however, the achievable minimum waiting time w depends on the parity-check matrix. In Fig. 3(b), the separate initialization step can avoid memory conflict of multiple read accesses to Fn.

In this scheduling, the check-node process is split into two processes, i.e. the first-half check-node process (FH-CNP) expressed by (17), (18) and the second-half check-node process (SH-CNP) expressed by (14), so that memory access is always column-wise. Equations (15), (16) are for the variable-node process. Immediately after the Rm,n values are obtained in (14), the Rm,n values are used to calculate the Qm,n values in (16), and the obtained Qm,n values are written to the Qm,n memory. The read access for (14) from the Qm,n memory becomes column-wise, just as the write access for (16) to the Qm,n memory. The obtained Qm,n values are also used to calculate Am(i+1) and Bm(i+1) by (17), (18) which are prepared for use in the next iteration. The FH-CNP for the first iteration is executed in advance during the initialization step as (12), (13). Fig. 3(c) shows the timing chart of this scheduling. As memory conflict does not occur because all memory accesses are column-wise, overlapped message-passing can be realized for any parity-check matrix. It should be noted that the bit-error performance obtained by this decoding scheduling is equivalent to that obtained by the message-passing algorithm described in Section IV-A.

C. Decoder Architecture Utilizing Quasi-Cyclic Structure
Fig. 4 shows a block diagram of the proposed LDPC decoder. The LDPC decoder consists of an Fn memory, a Qm,n memory, Np message-processing units, an Am update unit, a Bm update unit, and a syndrome-check unit, where Np is the degree of parallelization defined as the number of variable-node operations simultaneously executed. We assume that Np is a multiple of p, and d := Np/p is a divisor of L.
The Fn memory in Fig. 4 stores two entire codeword blocks of the received LLRs for continuous decoding. While a codeword block is iteratively decoded, the incoming channel LLRs of the succeeding codeword block are just stored. The memory is configured such that the bit width for each read access is Np×bF, where bF is the bit width for each quantized Fn value. The Fn memory can be implemented with 2 single-port RAMs or 2 shift registers.

The Qm,n memory in Fig. 4 stores the variable-to-check messages Qm,n. The size of the Qm,n memory is N×wc×bQ bits, where bQ is the bit width for each quantized Qm,n value. The Qm,n values obtained in the previous iteration are read by the message-processing units for calculating (14), and the updated Qm,n values obtained by (16) are written to the same address. The bit width for each access is Np×wc×bQ. The Qm,n memory can be implemented with a dual-port RAM, a first-in first-out memory, or a shift register.
The syndrome-check unit in Fig. 4 checks that the syndrome vector Hƙ(i) is zero. This unit can be a similar structure to the Am update unit described later.
Fig. 5 is a block diagram of the message-processing unit in Fig. 4, which is composed of wc SH-CNP units, a VNP unit, and wc FH-CNP units. The SH-CNP unit computes the check-to-variable message Rm,n using (14). The VNP unit receives a channel LLR Fn and wc check-to-variable messages Rm,n, and computes the hard decision ƙn of a posteriori LLR Qn and wc variable-to-check messages Qm,n, at one time. The FH-CNP unit computes the sign and ȥ function of (17) and (18). The number of output bits of the ȥ function is denoted as bȥ. The initialization step can be done by substituting zeros for the outputs from the SH-CNP units. The ȥ functions in the SH-CNP units may be omitted when the output values from the FH-CNP units instead of from the VNP units are stored in the Qm,n memory. But the memory size must be increased because ȥ(x) is a rapidly decreasing function for small positive x.
The Am update unit in Fig. 4 consists of 2 sets of M-bit memories and 1-bit multiplication circuits. The Bm update unit in Fig. 4 consists of 2 sets of (M×bB)-bit memories and adder circuits, where bB is the bit width for each quantized Bm value. The reason 2 sets of memories are used is to enable concurrent operations of the SH-CNP for the ith iteration and the FH-CNP for the (i+1)-th iteration. The Am and Bm update unit outputs Am(i) and Bm(i) values held in one set of memories to Np×wc SH-CNP units during N/Np cycles. The number of different indices of Am and Bm required by the SH-CNP units is denoted as NA. During the N/Np cycles, the Am and Bm update unit receives sign(Qm,n(i)) and ȥ(|Qm,n(i)|) from Np×wc FH-CNP units, and the values in the other set of memories are updated according to (17) and (18). After the N/Np cycles, Am(i+1) and Bm(i+1) are obtained. When a large Np value is employed, serial multiplications of (17) and serial additions of (18) are required because messages from multiple variable nodes are received.
When the parity-check matrix is not well structured, the Am and Bm update units require complex multiplexers and demultiplexers. We introduce hardware architecture here which can eliminate them when a quasi-cyclic code is used.

V. DESIGN OF A (1440, 1344) LDPC ENCODER AND DECODER
We have designed practical LDPC encoders and decoders adopting the architecture described in the previous sections. All designs were synthesized using a 65 nm CMOS technology library under the condition of a temperature of 125 °C and a voltage of 1.08 V. The gate count was obtained by adding the number of gates for all cells, where a gate is defined as four transistors. The actual chip was not available, so the power consumption was an estimation using the number of gates, operating frequency, and a roughly assumed switching activity.
The LDPC code employed in this paper is the rate-14/15 quasi-cyclic regular LDPC code designed for IEEE 802.15.3c with N = 1440, K = 1344, p = 15, q = 1, L = 96, wc = 3, wr = 45, and the minimum Hamming distance of 6. We chose a regular LDPC code, whose decoder could be simpler than that of an irregular LDPC code. Note that the threshold advantage of an irregular LDPC code over a regular LDPC code is at most 0.3 dB for a high code rate above 0.9 [10]. Furthermore, the advantage in coding gain vanishes when the codeword length is short enough to be practical for a high-rate code. For a communication system, a low-rate code is important for high reliability, and so is a high-rate code for high bandwidth efficiency. As the data rate of the uncompressed video of (1920 × 1080) pixels × 24 bits per pixel × 60 Hz is as high as 3 Gbps, the code rate should be high enough that the video can be transmitted even using simple modulation within a limited bandwidth. We set the target throughput to 3.2256 Gbps using the rate-14/15 LDPC code, i.e. the coded bit rate of 3.456 Gbps, at which data can be transmitted using QPSK with a symbol rate of 1.728 G symbols per second. A 15-parallel encoder was designed using the method described in Section III. Table II shows the coefficients of 15 generator polynomials in hexadecimal form. Power consumption was estimated to be 2.0 mW for 1.2 V supply, assuming the switching activity is 0.5. A 30-parallel encoder for 6.4512 Gbps was also designed. Its gate count was 3.1 k.

A 180-parallel decoder with 1-stage pipeline registers was synthesized with an operating frequency of 194.4 MHz. The total gate count of the decoder was 304 k. The decoder can operate at a coded bit rate Tin of 3.456 Gbps for Imax = 8. The power consumption was estimated to be 58 mW for 1.2 V supply, assuming the switching activity is 0.5 for (Iave+1)/(Imax+1) = 28 % of the entire duration and 0 for the rest. The power consumption will be reduced for a higher Eb/N0 value because the average number of iterations becomes smaller. A 360-parallel decoder with 1-stage pipeline registers operating at 216 MHz doubled the throughput of the 180-parallel decoder. The gate count of the 360-parallel decoder was 409 k. The 360-parallel decoder is a possible choice even for 3.2256 Gbps, because 26 % lower power consumption compared with that for the 180-parallel decoder is expected for its 39 % larger cell area when operated at 108 MHz. In [13], a high throughput decoder of 5.92 Gbps with 8 iterations for a similar codeword length N = 1200 using 0.13 µm CMOS technology was reported. Our decoder achieves 1.09 times this throughput with 0.36 times the gate count of [13], although we employed advanced CMOS technology.
