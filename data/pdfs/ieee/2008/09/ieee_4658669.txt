A Separation Algorithm for Improved LP-Decoding of Linear Block Codes

Abstract—We present improvements in linear programming decoding introduced by Feldman et al. with the help of generic integer programming methods. We propose a new IP formulation of the maximum likelihood decoding problem. This formulation makes use of indicator variables which help in detecting violated parity checks immediately. We derive Gomory cuts from our formulation and use them in a separation algorithm to find maximum likelihood codewords. We further propose a novel method of finding cuts induced by redundant parity checks. Under certain circumstances we can guarantee that these RPC cuts are valid and cut off the fractional optimal solutions of LP decoding. We demonstrate the performance of our separation algorithm on two LDPC codes and one BCH code.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes have attracted significant interest in the research community in the last decade. They are part of current and upcoming standards such as DVB-S2, WiMax, WiFi etc. Maximum Likelihood (ML) decoding is the optimal decoding algorithm for arbitrary linear block codes (e.g. LDPC codes). However, since the ML decoding is NP-hard [1], LDPC codes are generally decoded by the suboptimal algorithm known as Belief Propagation (BP) (or Sum-Product) algorithm. BP exploits the sparse structure of parity check matrix of LDPC codes very well and achieves good performance. However, due to the heuristic nature of BP decoding, the performance of BP decoders cannot be guaranteed at very low error rates. Moreover, the performance of BP is very poor for arbitrary linear block codes with a dense parity check matrix (which means that the corresponding Tanner graph contains short cycles).
The Shannon limit is generally used to benchmark the code performance. However, the Shannon limit is independent of the code structure and relies on an infinite blocklength for a given code rate. A better bound which also takes the code structure into account is the ML bound. The latter can be obtained by modeling and solving the ML decoding problem as an Integer Programming (IP) problem. As mentioned earlier, ML decoding is NP-hard which means, solving this IP problem is computationally feasible only for small instances. Nevertheless considering ML decoding as an IP problem yields a new approach to derive sub-optimal algorithms. These algorithms offer some advantages compared to BP decoding. First, they are not limited to sparse matrices. Second, these approaches rely on a well-studied mathematical theory which enables

quantitative statements (e.g. convergence, complexity, correctness, etc.) about the decoding process and its result [6], [8], [10].
Feldman et al. [8] relaxed the IP problem and proposed a new algorithm based on Linear Programming (LP) to decode binary linear codes. LP decoding algorithm utilizes a polytope which contains all valid codewords of a given code and a linear objective function. Minimizing this objective function over the polytope yields the ML codeword if the optimal solution is integral (known as ML certificate property). If the optimal solution is not integral, i.e., a pseudo-codeword is found, the LP decoder outputs an error.
Recently, LP decoding has been extended in various directions using nonlinear programming techniques [14], loopcalculus [2], duality theory and subgradient methods [11] or probabilistic approaches [5]. In this paper, we concentrate on improving linear programming decoding using integer programming tools. We introduce an alternative IP formulation for the decoding problem. Instead of solving the optimization problem, we attempt to find the ML solution by an iterative separation approach: We iteratively relax the IP formulation, solve the resulting linear program, derive cutting inequalities and add these inequalities to the IP formulation. This process continues until an optimal integer solution is found or further cuts cannot be generated. It should be noted that this general integer programming approach known as separation problem has first been applied to LP decoding by Taghavi and Siegel [10]. Our approach offers however the following advantages.
1) Our new formulation needs less constraints and all of them can be directly derived from the rows of the parity check matrix.
2) We employ indicator variables which can be efficiently used for identifying violated parity check constraints. We can prove that we detect violated inequalities faster than the adaptive algorithm of Taghavi and Siegel under some mild assumptions.
3) We formally show that the forbidden set inequalities [6] are a subset of the set of Gomory cuts (see [9]) which can be deduced from our formulation.
4) We provide empirical evidence that our new separation algorithm performs much better than LP decoding. This is mainly due to generating strong cuts efficiently using alternative representations of the codes at hand.

II. BACKGROUND AND NOTATION
A binary code C of dimension k and block length n is a subset (with cardinality 2k) of the set {0, 1}n. C is a binary linear code if it is a subspace of the vector space defined over the field GF (2). The linear code C is given by k basis vectors of length n which are represented by a k × n matrix G (generator matrix). Equivalently C can be described by a parity check matrix H ∈ {0, 1}m×n where m = n − k. We thus have x ∈ C, in other words x is a codeword, if and only if Hx = 0 in GF (2). We denote the ith row and jth column of H by Hi,., H.,j respectively. Hi,.x = 0 in GF (2) is defined as the ith parity check constraint. The index set I = {1, . . . , m} refer to rows and the index set J = {1, . . . , n} refer to columns of H. The matrix H is often represented by a Tanner graph G = (V, E). The vertex set V of G consists of the two disjoint node sets I and J. The nodes in I are referred to as check nodes and correspond to the rows of H whereas the nodes in J are referred to as variable nodes and correspond to columns of H. An edge [i, j] ∈ E connects node i and j if and only if Hij = 1.

inapplicable for high density codes. To overcome this difficulty an alternative formulation which requires O(n3) constraints is proposed in [8]. More recent formulations with size linear in length and check node degrees are proposed in [3] and [15]. Another approach applicable to high density codes is solving the corresponding separation problem of LPD [10]. The separation problem over an implicitly given polyhedron is defined as:
Definition 3.2: Given a bounded rational polyhedron P ⊂ Rn and a rational vector x∗ ∈ Rn, either conclude that x∗ ∈ P or, if not, find a rational vector (Π, Π0) ∈ Rn × R such that ΠT x ≤ Π0 and ΠT x < ΠT x∗ for all x ∈ P . In the latter case (Π, Π0) is called a valid cut.
In separation algorithms (see [9]) one iteratively computes families Γ of valid cuts until no further cuts can be found. In the separation algorithm of [10], which is called adaptive LP decoding by the authors, forbidden set inequalities are not added all at once in the beginning as in [8] but iteratively. In other words the separation problem for the fundamental polytope is solved by searching violated forbidden set inequalities. In the initialization step the LP min{cT x : 0 ≤ x ≤ 1} is solved. Let x∗ be the optimal solution. It is checked in O(mδmax + nlogn) time, if x∗ violates any forbidden set inequality where δmax is the maximum check node degree. If some of the forbidden set inequalities are violated then these inequalities are added to the formulation and the LP is solved again with the new inequalities.
Adaptive LP decoding stops when the current optimal solution x∗ satisfies all forbidden set inequalities. If x∗ is integral then it is the ML codeword otherwise an error is output. Note that putting the LP decoder in an adaptive setting does not yield an improvement in terms of frame error rate since the same solutions are found. On the other hand the adaptive LP decoder converges with less constraints than the LP decoder which has a positive effect on computation time.
The performance of LP decoding motivated researchers to find better approximations of the codeword polytope as part of ML decoding. One way is to tighten the fundamental polytope with new valid inequalities. Among some other generic techniques of cut generation, adding so called redundant parity check (RPC) cuts is proposed in [8]. Redundant parity checks are obtained by adding a subset of rows of H matrix in GF (2). These checks are redundant in the sense that they do not alter the code (they may even degrade the performance of BP [8]). However they induce new constraints in the LP formulation which may cut off a particular non-integral optimal solution thus tightening the fundamental polytope. An open problem is to find methods to generate redundant parity checks efficiently such that the induced constraints are guaranteed to cut off a non-integral LP solution.
To the best of our knowledge two approaches for generating potential cuts exist so far. First, adding redundant parity check cuts which result from adding any two rows of H [8]. Second, the approach in [10] which makes use of the cycles in the Tanner graph: 1) given a non-integral optimal solution x∗

remove all variable nodes j form the Tanner graph for which xj∗ is integral; 2) find a cycle by randomly walking through the pruned Tanner graph; 3) add the rows of the H matrix in
GF (2) which correspond to the check nodes in the cycle; 4)
check if the found RPC introduces a cut.

VI. CONCLUSIONS
In this paper we proposed a new IP formulation and its LP relaxation. Instead of solving the optimization problem, we solve the separation problem. The indicator variables z enable immediate recognition of parity violations and efficient generation of cuts. We used on one hand the forbidden set inequalities of [8] which are a subset of all possible Gomory cuts. On the other hand we showed how to generate efficiently new cuts based on redundant parity checks. Note that the rows in our Hˆ matrix can be considered as redundant parity checks.
