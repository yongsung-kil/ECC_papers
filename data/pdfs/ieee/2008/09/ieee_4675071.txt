Construction of the Cyclic Block-Type LDPC Codes for Low Complexity Hardware Implementation

Kuang-Hao Lin, Robert C. Chang, Member, IEEE, Chien-Lin Huang, and Sheng-Dong Wu
Department of Electrical Engineering National Chung Hsing University Taichung, Taiwan 89364306@ee.nchu.edu.tw

Abstract—In this paper, we construct the cyclic block-type lowdensity parity-check (CB-LDPC) codes for low complexity hardware implementation. The CB-LDPC code, which is a special class of quasi-cyclic LDPC (QC-LDPC), has an efficient encoding algorithm due to the simple structure of their paritycheck matrices. A distribution of irregular parity-check matrix for the CB-LDPC is developed so that we can obtain an areaefficient decoder design, good error correction performance, and low complexity architecture implementation. The CB-LDPC code decoding uses the iterative min-sum algorithm (MSA) and the block parallel connection design to improve the hardware architecture complexity and area.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes are linear block codes which were first discovered by Gallager [1] and they have better block error rate performance. In recent years, there has been a dramatic proliferation of research concerned with the systematic construction of LDPC code. Because the usage of high-throughput data rate increase, reliability requirements for error correction have become critical. LDPC codes technology have been widely used for the digital communication due to their high bit error rate (BER) and frame error rate (FER) performances.
Gallager’s theory [1] and Richardson et al. [2] offered a sounder theoretical basis for noisy channel coding theorem. Furthermore, these codes are either cyclic or quasi-cyclic (QC). The majority of research in hardware implementation has focused on QC. Myung et al. [3] provided an excellent review of these methods, results and instructional issues related to QC LDPC encoding and decoding. A special class of QC-LDPC codes, called Block-LDPC, was presented for practical LDPC coding system implementations [3, 4]. The main idea is to construct LDPC codes subject to certain hardware-oriented constructs that ensure the effective encoder and decoder hardware implementations. The reported works of decoding in the literature can be classified into two major categories of sum-product algorithm (SPA) or min-sum algorithm (MSA). The readers are referred to good tutorials in

[5] and [6] for an introduction to the above decoding algorithm approaches and their applications.
In this paper, the cyclic block-type LDPC (CB-LDPC) code is defined with a proposed irregular parity-check matrix to have almost lower triangular form. They are linearly encodable and have better memory efficiency as compared with the conventional LDPC codes [7]. The decoder architecture in the receiver is developed which uses the block parallel connection to improve hardware complexity. Simulation results show the performances of CB-LDPC decoder with different iteration numbers.
This paper is organized as follows. Section II discusses the CB-LDPC codes and the decoding construction. In section III, we present the efficient decoding architecture. In Section IV, the simulation and implementation results are presented. Conclusions are drawn in Section V.
II. THE CB-LDPC CODES
In the transmission, the transmitted data can be encoded through channel encoder then mapped to constellation. The transmitter constellation data suffer channel effect with additive white gaussian noise (AWGN). Consequently, the contaminated constellation data is received and can be corrected via soft demapping and channel decoder, respectively.
A. Encoding of the CB-LDPC codes
The method presented in this paper is developed from QCLDPC [3], [8], and we want to find a well-suited parity check matrix in encoding to reduce complexity in decoding. QCLDPC has been constructed and can encode efficiently in early years. But in hardware realization, QC-LDPC is not easily accomplished. Therefore, we propose a method to develop the parity check matrix so that the hardware can be achieved easier with lower cost.

III. HARDWARE ARCHITECTURE
Figure 2 shows the architecture of the CB-LDPC decoder design that comprises the bit node processing units (BNUs), check node processing units (CNUs), and memory (Mem) components. Here we use N BNU to sum LLR data of node and the result is stored to Mem. N is denoted as n/p, where n is the number of codeword length and p is the block matrix size. The output data of BNUs are stored into M two-port Mem through the counter to calculate address, where M is defined as the number of non-zero block of parity check matrix H. The K CNUs was adopted to determine the minimum value of
information from memory output. K is expressed as N×(1－R) and R is the coderate. In the first load, the LLR data from softdemapping is transported to BNU, and directly sent to Mem via BNUs component. In which, the BNU Connect determines the connection between BNUs and Mem block according to block matrix address of every column in parity check matrix. Similarly, the CNU Connect is determined according to block matrix address of every row in parity check matrix. Finally, hard decision (H.D.) results from BNUs are computed by BNUs and CNUs components with eight iterations respectively.
The most complex part in hardware implementation of LDPC is the connection between BNU/CNU and memory. The connection may occupy a very large area. According to different basic architectures, the routing in connection is different. For instance, using the partial parallel connection [3], BNU and CNU must employ more multiplexers to change data from source to destination because they are used by many memory banks. The number of the multiplexers is decided by the number of memory banks. The partial parallel connection exploits hardware sharing scheme that can save the hardware area. This architecture is widely used in convention LDPC, but it can decrease throughput and increase routing difficulty. Therefore, the block parallel connection is used in this paper.
The parity check matrix H takes on a block type in the CB-LDPC encode method. It is different from the conventional LDPC code encoding because the conventional LDPC uses 0 and 1 to encode, but the proposed CB-LDPC adopts O and I and uses rotation to enhance the girth number. Figure 3 illustrates BNUs and CNUs memory mapping in hardware implementation. Each BNU corresponding to a memory bank is designed so that this method can economize on many multiplexers in select connection. Therefore, the routing complexity using block parallel connection will be greatly reduced compared to that parallel connection [7], [9].

The BNU algorithm depicted in Fig. 4 allows a memory efficient connection implementation due to its inherent sequence and the fact that memory block (MEM) can express the original columns of H. The conn_BNUi indicates a BNU corresponding the number of memories, where i denotes the number of BNU. Figure 5 discuss the algorithm for CNU connection computation method. IV. SIMULATION AND IMPLEMENTATION RESULTS
Figure 6 shows the simulated BER versus signal-to-noise ratio (SNR) performance analysis with different number of iteration in receiver decoding, using codeword n=2304 and coderate 1/2 of CB-LDPC. The efficiency curve begins slow convergency from the eight iteration operation so that the hardware implementation employs eight iterations. The parameters in Table I are employed in the equation (1) to analyze BER and FER versus SNR. The parameters t and s indicate L3,t and L2,s location of matrix HI, respectively. The parameters a1 ~ a3 denote the times of L rotation. Figure 7 shows the performance simulation of different rotation matrixes. For the purpose of comparison, we construct 200 sets for 2304-bit, block matrix size 12×12, using eight iteration operations, and coderate with 1/2.
Assuming the block size L for all elements of HI is 12×12, the position of t and s can be distributed in the vicinity of row 5 and row 9 to have a longest distance with the 1st sequence L1,1. Thus, the girth number is raised which improves the efficiency of decoding. By simulation, we can find that the initial rotation number a1, a2, and a3 can be increased to reduce the probability of the same rotation number of other blocks in the same row. In Fig. 7, the CB-LDPC performance for the same FER and BER show a SNR gain about 1dB compared to the array code [3].
The implementation results are given in Table 2. Hardware implementation results show that the proposed decoder consumes about half of power dissipation and occupies half of cell area, compared to Refs. [7] and [9].

V. CONCLUSIONS
In this paper, an efficient CB-LDPC code decoding suited for MSA with irregular parity-check matrix H for highthroughput is proposed and analyzed. Our algorithm shows about 1dB improvements over array codes. The hardware routing complexity using block parallel connection will be greatly reduced compared to that parallel connection. The proposed architecture is able to reduce about 50% of power consumption and to use only half of cell area.
