ABSTRACT
In this paper, we propose a partially-parallel decoder architecture for irregular LDPC code targeting high throughput applications. The proposed decoder is based on a novel delta-value message-passing algorithm to facilitate the decoding throughput by removing redundant computations using the difference between the updated value and the original value. Techniques such as binary sorting, high performance pipelining are used to further speed up the messagepassing procedure. The synthesis result in TSMC 0.18 CMOS technology shows that for (648,324) irregular LDPC code, our decoder can increase 8 times in throughput, which reaches 418 Mbps at the frequency of 200MHz.
I. INTRODUCTION
A Low-Density Parity-Check (LDPC) code is an error correcting code first proposed by Gallager in the early 1960’s, and rediscovered by MacKay and Neal [1] in 1996. In [2][3], irregular LDPC codes are constructed enabling data transmission rates close to the Shannon Limit, the theoretical maximum. The LDPC decoding algorithm is inherently parallel and is much easier to be implemented than its comparator turbo codes, thus making it more attractive to researchers [3].
LDPC code is very suitable for hardware implementation by utilizing a parallel decoding algorithm called Message-Passing (MP) algorithm [4]. In the last few years, researches have done on designing specific decoder architectures for LDPC implementations, seeking for the best trade-off between area, power consumption and performance. [5]-[9]
The LDPC decoders that have been designed have the same primitive processing elements: a check functional unit (CFU) performing row operations for check nodes and bit functional unit (BFU) performing column operations for bit nodes. These processing elements are connected according to the Tanner graph, a graph representing the relation between bit nodes and check nodes. The MP algorithm exchanges messages between check nodes and bits nodes by performing row and column operations iteratively.
The authors in [8] proposed an accelerated message-passing schedule, which only performs those column operations whose corresponding check nodes have been updated by the row operations. A partially-parallel LDPC decoder based on the accelerated MP schedule is introduced in [9] to support for irregular LDPC code.
Although the partially-parallel irregular LDPC decoder proposed in [9] can improve the error correction performance, it suffers a main problem that the overall throughput is relatively low compared with the regular or fully-parallel irregular LDPC decoder (eg. [10], [11]). Considering the fact that throughput is an essential metric in realtime communication systems applying LDPC code, we propose an improved MP algorithm and decoder architecture for irregular LDPC

decoder, in this paper, to achieve a nearly 8X speedup. The novelties of this decoder in terms of high throughput are as follows:

1) A novel delta-value based message-passing algorithm is proposed to accelerate the decoding computation.
2) An improved binary sorting scheme is designed to reduce the computation time of row operation.
3) A fully-parallel structure of bit function unit is designed to speed up the column operation.
4) A high performance pipeline structure is used to further speed up the message-passing procedure.

The rest of the paper is organized as follows. Section II introduces the preliminary concepts and background materials. Section III discusses the improved message-passing algorithm proposed. Section IV discusses our LDPC decoder architecture. Section V shows the detailed implementation and timing of our decoder. In Section VI we demonstrate our implementation results and finally Section VII concludes.

II. BACKGROUND

In this section, we present the preliminary concepts, specifically, irregular LDPC code, and message-passing algorithm. The irregular LDPC codes can be defined by a parity check matrix H containing M×N sub-blocks, where M and N are the number of row-blocks and column-blocks respectively. Each sub-block matrix is a b×b square matrix, obtained through right shifting the identity matrix Ibǘb by a specific number. The parity check matrix used in this work is a 12×24 matrix (each is 27bits×27bits) defined in IEEE 802.11n standard with code rate of 1/2 and block length of 648 bits.

LDPC code can be decoded iteratively using the MP algorithm
described in [4]. Each iteration of the algorithm is composed of four
phases: row operation, column operation, error correction and parity check. The row operation updates message Į of all check nodes using message ȕ, and sends the message to bit nodes. The column operation updates message ȕ of all bit nodes based on message Į and initial message Ȝ. The error correction calculates the tentative decision and the parity check operation decides whether another
iteration is needed or not according to the tentative decision.

IV. DECODER ARCHITECTURE

In this section, we present the detailed implementation of row operation module and column operation module in our DVMP schedule based LDPC decoder. We mainly discuss a modified binary sorting scheme and fully-parallel column computation techniques used that enhance the overall throughput.
A. High-Throughput Row Operation Module
In the calculation of Į value according to Equation (1), the minimum and second minimum ȕ value among a total of |A(m)| values in the same row should be obtained through a proper comparison scheme. In the hardware implementation in [9], a bubble sorting scheme is utilized with two registers storing the minimum value and second minimum value respectively. Each time a new ȕ is read in to compare with the current minimum one and replace it if the new one is smaller. The serial comparison scheme requires nine clock cycles in total, which is the most time consuming step in row operation.

The detailed comparison procedure in row operation module is shown in Figure 2. The upper tree and the lower tree represent respectively the sorting for the minimum and the second minimum ȕ value among a total of eight values (|A(m)|=8 for the targeted matrix). In the figure, we assume the relationship between eight ȕ values in the same row (ȕ1, ȕ2,…, ȕ8) are ȕ1<ȕ2<ȕ3<ȕ4<ȕ5<ȕ6<ȕ7<ȕ8 and the resulting values are labeled on each data path. In the first step, eight ȕ values are compared in pairs. The smaller one (eg. ȕ1) is remained in the sorting tree for the minimum value while the larger one (eg. ȕ2) is eliminated to the sorting tree for the second minimum value. Then the values in both trees will continue the comparison process in pairs with the larger one in the upper tree eliminated to the lower tree. In this manner, we can get the minimum and the second minimum value at the third and the fifth step of the operation. Compared with a total 9 clock cycle comparison in [9], our proposed scheme can complete the comparison in only 2 clock cycles under the same frequency, a 4.5X speedup.

As discussed in Section III, the DVMP-based column operation can update ȕ values by the addition of its original ȕ and at most two updated ǻĮ values according to Equation (3). The number of updated ǻĮ values and the exact position of each updated ǻĮ in the column operation are determined by the parity check matrix. Then we compute the new ȕ value by adding the original one and the sum of ǻĮ in the second step.
By adapting the DVMP schedule, not only the computation time of each column operation is minimized, but also the area of hardware implementation is reduced. Based on this observation, we can further improve the throughput by applying a fully-parallel column operation here.

In the proposed decoder, we design 12 CFUs and 24 BFUs in order to execute the computation of the same row or column in each sub-block in parallel. Since the targeted LDPC decoding matrix is irregular, two different row operation modules (CFU_for_7in and CFU_for_8in), and three different column operation modules (BFU_for_12in, BFU_for_3in, and BFU_for_2in) are designed for rows and columns with different |A(m)| and |B(n)|. In previous implementation of the accelerated MP schedule [8][9], all N column-blocks are computed in parallel while different columns in one column-block are calculated in serial, which may require at most 11 clock cycles for column computation. In the proposed module, message ȕ of all columns, in which there exists an updated Į, are computed in parallel. A hardware implementation for computing k columns in parallel is shown in Figure 3.

The resulting parallel DVMP-based column operation can be computed in one clock cycle, which is 11 times faster than the implementation in [9].

In the proposed decoder, pipeline structure is utilized to achieve further speed up of the procedure, as shown in Figure 5. The row operation and column operation (including message read and write) are divided into four and three pipeline stages respectively to balance the computation time of each stage. After a further overlap of the row and column operation based on data dependency information, the message of a particular row operation and corresponding column operation can be updated after four clock cycle.
During the first clock cycle, Į and ȕ values are read in to prepare for a row operation. In the second and third clock cycles, a tree structure comparison is conducted for the minimum and second minimum ȕ value, and corresponding Į and ᇞĮ values are obtained. In the fourth clock cycle, all related column operations are computed based on ᇞĮ values, and Į values are written into memories at meantime.

VI. IMPLEMENTATION RESULTS
The proposed decoder is implemented under TSMC 0.18ȝm CMOS technology. The synthesis results are listed in Table 1. Because of the pipeline structure along with the improved row and column operation modules, our proposed decoder requires only 31 clock cycles for a single iteration and five iterations for codeword correction under SNR of 3.0dB. The area of the proposed decoder listed includes the wiring area, and we compromise the device area for throughput by applying a fully-parallel column operation. Compared with the implementation result in [9], the proposed decoder can achieve almost 8 times the throughput under the same clock frequency without losing bit error rate performance.
