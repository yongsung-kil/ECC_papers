AbstractÔÇöLow-density parity-check (LDPC) codes deliver very good performance when decoded with the BP algorithm. Unfortunately, this algorithm has the disadvantage to be complex for implementation. In this paper, we investigate the BP algorithm and its simplifications (BP-Based, normalized BP-Based, offset BP-Based). Previous works on normalized and offset BP-Based proposed theoretical values for the offset and normalization factors, we perform simulations which allow to validate these theoretical results.
Index TermsÔÇöLDPC codes, BP algorithm, BP-Based algorithm, normalized BP-Based algorithm, offset BP-Based algorithm.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were first discovered by Gallager [1] in 1962 and have recently been rediscovered by Mackay and Neal [2], [3]. They deliver very good performance when decoded with the belief-propagation (BP) algorithm [2]. As LDPC codes are being considered for use in a wide range of applications for example in the second Satellite Digital Video Broadcasting normalization (DVB-S2), the search for efficient implementations of decoding algorithms is being pursued intensively.
The implementation of the BP algorithm is difficult because it uses non linear functions. ThatÔÇÖs why different simplifications of this algorithm were discovered. The BP algorithm can be simplified in BP-based algorithm [4], which greatly reduces the implementation complexity but incurs a degradation in decoding performance. This has led to the development of different variants of the BP-Based algorithm which are the normalized BP-Based algorithm [5] and the offset BP-Based algorithm [9]. The optimal values of the normalization and the offset terms are found theoretically by density-evolution (DE) [9]. The purpose of our work is to validate by means of simulations these theoretical values.
This paper is organized as follows. Section II corresponds to a general description of the LDPC codes and the BP algorithm. Section III describes the BP-Based algorithm and its different variants. Finally, simulation results are proposed in section IV. The conclusion and perspectives of our work are given in section V.

II. LDPC CODES AND BP ALGORITHM

A. LDPC codes

Binary LDPC codes, are linear block codes defined by a

sparse parity check matrix H(M ├Ś N ), where N denotes the

codeword length and M the number of parity-check equations.

When the numbers of 1ÔÇÖs in each column and row are constant

the code is called a regular LDPC code. Otherwise, itÔÇÖs said

to be irregular.

LDPC code can be represented by a Tanner graph which is

a bipartite graph [6]. It is a graph where the elements of a first

class can be connected to the elements of the second class, but

not to the same class. In a Tanner graph for LDPC code, the

elements of the first class are the variable nodes denoted by vn. Those of the second class are the check nodes denoted by cm. Every bit of the codeword corresponds to a variable node, and

every parity check constraint is represented by a check node.

So, we have N variable nodes and M check nodes. When a bit

appears in a parity-check equation, the corresponding nodes

are connected by an edge.

IV. SIMULATION RESULTS
Simulations have been performed using regular(5,10) LDPC code of length N = 816 (given in [8]) and with 10 decoding iterations.

B. Optimization
It is clear that the BP-Based algorithm reduces the complexity of the check node update but this is made at the expense of a substantial loss in performance. This degradation is explained by the over estimation of the extrinsic information. Indeed, if E1 and E2 represent the values of En(i,)m computed by the BP algorithm and the BP-based algorithm with (4) and (6), respectively, it was shown in [5] that the following two statements hold:
1) E1 and E2 have the same sign, i.e, sign(E1) = sign(E2).
2) E2 has larger magnitude than E1, i.e, |E2| > |E1|.
Based on these two statements, two improved BP-based algorithms are shown by Fossorier and Chen in [9]. The overestimation can be partially corrected either by a normalization or by an offset term.

A. Optimal values of ╬▒ and ╬▓
To achieve the optimal performance, ╬▒ and ╬▓ should vary with different SNR and different iterations. However, for simplicity of implementation Fossorier and al. [9] propose to kept these terms constant for all iterations and all SNR values.
Fossorier and al. propose in [9] a theoretical approach to determine the normalization term and the offset term. They apply the density evolution (DE) to obtain the thresholds of the normalized BP-Based algorithm and the offset BP-Based algorithm with different values ╬▒ and ╬▓ and different code ensembles. The theoretical values of ╬▒ and ╬▓ for a regular (5,10) LDPC code are ╬▒ = 1.65 and ╬▓ = 0.2.
In a practical case (finite code length and finite number of iterations) the optimal values of ╬▒ and ╬▓ can simply be found through simulations. Indeed, we depict in figure (4) and figure (5) the evolution shape of the BER as a function of ╬▒ and ╬▓, for several SNR and for several maximal iteration number. After, we can find the values of ╬▒ and ╬▓ leading to the best BER. Then, according to simulation results shown in figures (4) and (5), the normalization factor ╬▒ = 1.85 and the offset factor ╬▓ = 0.7 correspond to the best terms for the (816, 5, 10) LDPC code.
Figure (4) and figure (5) show also that the experimental values (╬▒ = 1.85 and ╬▓ = 0.7) permit a best BER than the theoretical ones (╬▒ = 1.65 and ╬▓ = 0.2) for the case of regular (816, 5, 10) LDPC code .

V. CONCLUSION
In this paper, we have been concentrated on the decoding of LDPC codes. We have studied the BP algorithm and its simplification to the BP-Based algorithm. This algorithm reduces the complexity of the check node update but there is a degradation in performance compared to the BP algorithm.

The two improved BP-Based algorithms which are the normalized BP-Based algorithm and the offset BP-Based algorithm are investigated. Theoretical method have been proposed in [9] to determine the normalization and the offset factors in order to achieve the performance of BP algorithm. We showed that in practical case i.e finite code length and finite number of iterations, the optimal values of ╬▒ and ╬▓ can simply be found through simulations. We finally showed by simulations that with these experimental values the normalized BP-Based and the offset BP-Based algorithms, outperform the BP algorithm at high SNR values, for regular medium LDPC codes. These results are also found by Fossorier and al. in [10].
As a perspective on our work, we propose to perform a statistical analysis of the extrinsic information in order to establish an accurate model to be used instead of the normalization or the offset methods.
