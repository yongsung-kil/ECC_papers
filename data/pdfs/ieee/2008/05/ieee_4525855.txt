Long Length LDPC Code Construction and the Corresponding Decoder Implementation with
Adjustable Parallelism

Chia-Yu Lin, Mong-Kai Ku and Yi-Hsing Chien
Department of Computer Science and Information Engineering National Taiwan University, Taipei, Taiwan 106, ROC Email: {f93084, mku, d94012}@csie.ntu.edu.tw

Abstract—In this paper, we propose a class of implementation friendly structured LDPC codes with low error floors. The proposed codes exhibit no apparent error floors as compared with quasi-cyclic (QC) LDPC codes at long block lengths. A modified progressive edge-growth algorithm is used to construct the hierarchical quasi-cyclic (H-QC) LDPC codes. By adding implementation-friendly two-level hierarchy with limited types of second-level submatrices in the parity check matrix, coding performance is improved substantially over QC codes. We also show that QC-based decoder architecture can be easily applied to H-QC decoders to achieve better coding gain and higher throughput performance. Moreover, the degree of decoding parallelism and code length can be adjusted by changing the HQC code construction parameters.
Keywords-Low-density parity-check code; quasi-cyclic (QC) codes; progressive edge-growth (PEG); partially-parallel decoder
I. INTRODUCTION
Low-density parity-check (LDPC) [1] codes have attracted much attention lately due to their capacity-approaching performance and efficient hardware codec implementation. Long block length LDPC codes provide strong coding gain required for powerful communication systems. Next generation satellite communication standard DVB-S2 [2] employs LDPC codes with block length up to 64800 bits to achieve its system performance goals.
The goals of practical LDPC code design are good coding gain performance and low hardware complexity. Current design methodologies of LDPC codes can be divided into two categories: random and structured construction. The LDPC code parity check matrices constructed with random method have no specific structure restriction [3]--[5]. Well constructed random codes can perform close to the theoretical limit. However, the encoder/decoder hardware implementation for random codes always induces large area cost and high interconnection complexity due to the lack of structure in the parity check matrix. The other category of design methodologies, structured construction, creates the code parity check matrix under certain arithmetic structure [6][7]. By taking advantage of the matrix structure, linear-time encoding and partially-parallel decoding can be realized. Due to the

The performance of short to medium block length structured codes tracks closely to the performance of random codes. However, long block length structured codes suffer from coding gain degradation and error floors when compared with random codes. In this paper, the concept of matrix expansion is used to improve the coding performance of QC codes and facilitate modifying the decoder architecture to achieve higher throughput. We propose a class of implementation friendly structured codes named hierarchical quasi-cyclic (H-QC) LDPC codes. Simulation shows that the coding gain performance of H-QC codes matches random codes at long block lengths. Furthermore, there are no apparent error floors at high SNR. Efficient H-QC partially-parallel decoder can be achieved with minor modification to quasi-cyclic LDPC decoder architecture.

II. LDPC CODE CONSTRUCTION
A. Hierarchical Quasi-cyclic LDPC Codes A LDPC code with block length N and source length K is
defined by a M × N sparse parity check matrix. M, equal to N-K, denotes the number of parity bits. As shown in Fig. 1(a), the matrix of a quasi-cyclic (QC) LDPC code consists of square submatrices of size L. Each submatrix is either a zero matrix or a circulant permutation matrix which is formed by circularly shifting rows of the identity matrix to the right by several times. The m × n matrix obtained by replacing each circulant permutation matrix and each zero matrix with 1 and 0 is named the core matrix of the QC code.

Regular QC codes have very symmetric matrix structure that facilitates decoder hardware implementation [8]-[11]. For H-QC code construction, only circulant permutation matrices are used for matrix expansion to reduce storage and routing requirements. Multiple-level expansion is employed to increase the randomness of the matrix structure. Compared with other code construction approaches, the systematic recursive (SR) method [12] increases the block length by matrix expansion using random permutation matrices. SR method aims to generate LDPC codes as random as possible with no hardware complexity considerations. The decoder oriented code construction proposed in [13] creates the parity check matrix by random permutation matrices and zero matrices without multiple-level expansion. Since our proposed method expands the matrix using only circulant permutation matrices, the resulting H-QC codes have simple matrix structures and require less storage than the codes generated by SR method. Due to the regularity in the H-QC matrix, routing between functional units in the decoder architecture is also easier than the other two methods.
B. Code Construction The performance of LDPC codes is degraded due to the
cycles (especially small cycles) in the corresponding Tanner graphs. These cycles will harm the optimality of message passing decoding. Progressive edge-growth (PEG) algorithm is a graph conditioning technique to construct Tanner graphs of LDPC codes which possess large girths [14][15]. This

technique constructs the Tanner graph by progressively adding edges between variable nodes and check nodes. Edges are evaluated based on the girths of the resulting graphs, and only the one that results in the largest girth is selected.
To construct the s-level hierarchical parity check matrix, the block length N is factored to s+1 numbers n, q1, q2, …, and qs. The matrix construction begins by creating the first level m × n core matrix. The parity check matrix is hierarchically expanded by replacing 1s with qi × qi circulant permutation matrices and replacing 0s with qi × qi zero matrices for 1≤i≤ s.
Our PEG-based algorithm inserts nonzero elements into the matrix column by column according to the limitation set by circulant permutation H-QC structure. We use two-level H-QC matrix construction to illustrate our approach. As shown in Fig. 2(a), the two-level H-QC matrix with N=nq1q2 is composed of outer submatrices sized q1q2 and each outer submatrix is consisted of q12 inner submatrices of size q2. Since a nonzero outer submatrix is a special (1,1)-regular matrix such as Fig. 2(f), the outer submatrix will be constructed with the structure constraint set by our use of circulant permutation matrix. Only the shaded regions in Fig. 2(c)-(e) are the possible positions for nonzero elements. After a nonzero element is inserted into the leftmost column of this submatrix, all q1 nonzero inner submatrices in this outer submatrix are determined (Fig. 2(b)). As shown in Fig 2(c)-(e), the same principle applies to all the inner submatrices so they are guaranteed to be a circulant permutation matrix. To reduce hardware complexity, our PEG algorithm is also constrained to use limited types of distinct second-level matrices. This approach can be generalized to slevel H-QC code construction for s greater than two.

III. HARDWARE IMPLEMENTATION ISSUE EXPLORATION
Partially-parallel LDPC decoder can be optimized for the best combination of throughput and hardware complexity. The two-level H-QC LDPC codes (i.e., N=nq1q2) can be adopted for efficient partially-parallel hardware decoder implementations. In the following discussion we focus on the coding gain performance and hardware friendliness of the twolevel H-QC codes.
QC-based partially-parallel decoders [8]--[11] can be viewed as decoders for (j,k)-regular codes generated by the singlelevel expansion as shown in Fig. 1(a). Here the core matrix of size j× k is filled with 1s and the submatrix size q1=ab with the block length N=kab. This class of codes has very symmetric matrix structure that simplifies decoder implementation. However, for long block lengths, QC codes suffer high error floors as compared to randomly generated codes due to the bounded girth and minimum distance. As shown in Fig. 1(b), two-level H-QC codes with q1=a and q2=b can be used to avoid this limitation. The performance comparison of these two classes of codes is shown in section IV.
The first level expansion of H-QC codes produces a quasicyclic matrix such as Fig. 1(a). Thus, decoder architectures devised for this class of QC codes can be readily applied for H-QC codes with minor modifications. IV. SIMULATION RESULTS
In this section, we compare the proposed H-QC codes with QC codes used in [8]--[11] and randomly constructed codes for long block lengths. All results are simulated on the binaryinput additive white Gaussian noise (AWGN) channel. Both QC and H-QC codes are generated by the modified PEG method and random codes are generated by MacKay’s algorithm [4] (source code from [18]). Fig. 5 compares the performance of these three classes of LDPC codes with block length 10080, 20160, and 40320. We see that H-QC codes show no obvious error floors up to a BER of 10-7 while the performance of QC codes degrades at a BER of 10-5. For long block lengths H-QC codes outperform QC codes and match random codes.

V. CONCLUSION
In this paper, we have proposed a class of implementation friendly structured LDPC codes called hierarchical quasicyclic LDPC codes with coding gain performance that matches randomly constructed codes. Compared with QC codes, H-QC codes show no error floors at long block lengths. The parity check matrix structure of H-QC code is simplified to reduce hardware decoder complexity. Two-level H-QC codes are well suited for partially-parallel hardware LDPC decoder implementation. We show that only minor modifications are necessary to adapt quasi-cyclic LDPC decoder architecture to support two-level H-QC codes. To reduce routing and rotator complexity, the number of distinct second-level submatrices is cut to four without coding gain degradation. The degree of parallelism and code length can be

easily adjusted by changing the H-QC construction parameters. These properties make H-QC code a good candidate for realworld high performance communication systems.
