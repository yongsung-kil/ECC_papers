Abstract—This paper have developed farther the method of constructing sparse parity-check matrix (H-matrix) with determinate systemic structure as well as designing practice recursive encoding algorithm of the low-density parity-check (LDPC) codes. It discovers that the LDPC codes based on the circulant-shifted permutation Q-matrix created by N-queens searching algorithm perform better than them based on permutation identity matrix when Hd -matrix in H  ¢¡H p Hd °±¯ was constructed by using circulant-shifted permutation Q-matrix and identity matrix, respectively. It presents a simple and efficient algorithm of constructing Hd matrix without 4-cycle and 6-cycle. The presented LDPC encoders require O(M ) operation (M is check-bit size). They are rate-compatible and length- compatible encoders that can cover a very wide range of rate from 0.1 to 0.9 and length including any positive integer from 20 to infinity except prime number and multiple prime numbers. The presented LDPC codes with moderate 1ength and 1/2 rate can achieve reliable performance – 10-5~10-6 bit error rate (BER) – on an additive white Gaussian noise (AWGN) channel with signal-to-noise ratio (SNR) Eb/N0 about 1.42dB1.
Index Terms—Low-density parity-check (LDPC) codes, sparse parity-check matrix, permutation matrix, linear encoder, recursive encoder, N-queens problem.
҇ INTRODUCTION
Recently, the LDPC codes based on the array of circulant-shifted permutation sub-matrices [6, 7] in which row and column weights are w = 1 have extensively been paid attentions because they can provide both excellent error correcting performance and executable partial parallel decoder architectures [1]. If the H-matrix of this class of LDPC codes was completely constructed by circulant-shifted permutation sub-matrices, then two limitations exist. On one hand, the H-matrix has non-systemic structure which makes encoder execution become complicated; on the other hand, this kind of H-matrix includes redundant parity check rows which make practice code rate be larger than designing code rate. An effectively ameliorating method is to adopt the systemic formal H-matrix like H [H p Hd ] , in which

Ҋ. SIMULATION

Simulation tool is to adopt software version of LDPC-2001-11-18 [8] written in C and programmed by Radford M. Neal. For our research, we modified two blocks of this software version: parity-check matrix generation block and encoding block. Channel block and decoding block still use Neal’s original version. Simulation experiment is carried out according to this process: a random source sequence is encoded by the Q-LDPC codes based on H-matrix which consists of bidiagon H p matrix and Hd -matrix formed by the
subscript matrix S(Hd ) with similarity to (4) and (5) as
well as filled by the circulant-shifted permutation Q-matrix which can be obtained by the N-queens searching algorithm , modulated with BPSK, transmitted on an additive white Gaussian noise channel and decoded by belief propaganda iterative decoding algorithm. The maximum iterative time is 200 and the transmitted source block number is 1000.

ҋ. CONCLUSIONS
A family of Q-LDPC codes is presented in this paper. Its H-matrix H  [H p #Hd ] has such characteristics that the
bidiaganol matrix of H p and the subscript matrix corresponding to the array structure of Hd are determinate structural matrix, and permutation Q-matrix which is used to fill the array of Hd is random matrix of computer searching, To compare with other algebraic structural LDPC codes, the Q-LDPC codes have the following four predominance: 1) they have completely determinate structural H-matrix if we search the optimum Q-matrix off line and store it on line. The memory contents of the H-matrix only include the subscript matrix and an original Q-vector; 2) they have linear recursive encoder that only requires O(M) operation, low complexity decoder that boils down to structural design of the H-matrix in which “1” element is small in number, and natively partial parallel decoding architecture; 3) they can easily be designed based on flexible changed code rate and code length without adopting puncturing, extending and shortened techniques; 4) noteworthiness is that they have excellent performance up to 1.23dB away from their theoretical Shannon capacity limits for code length within 10000 under the circumstances of satisfying 1), 2) and 3).
