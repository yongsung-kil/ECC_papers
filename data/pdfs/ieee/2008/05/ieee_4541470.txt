Enhanced Delta-Based Layered Decoding of WiMAX QC-LDPC Codes

Tzu-Chieh Kuo and Alan N. Willson, Jr. Electrical Engineering Department
University of California, Los Angeles, CA 90095-1600, USA

Abstract—Rovini, et al., have proposed a delta-based layered decoding algorithm to mitigate the latency constraint imposed by the layered decoding algorithm on QC-LDPC codes whose adjacent block rows overlap. We apply their algorithm and the low-complexity offset-based Min-Sum check equation to the decoding of the codes specified in WiMAX. The performance is simulated for two representative hardware pipeline depths, showing a maximal 0.13-dB performance loss, with respect to the ideal latency-free layered decoding algorithm, for a block error rate of 10-2 in 15 iterations and a pipeline depth of eight. We further develop a systematic procedure to arrange the processing order of the block rows and block columns of a given QC parity check matrix, reducing the aforementioned performance loss to 0.03 dB at no extra hardware cost. The procedure can be applied to any QC-LDPC codes for any given pipeline depths.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are powerful advanced error control codes and have been chosen as mandatory or optional codes in new standards -- for instance, in IEEE 802.11n (Wi-Fi) and in IEEE 802.16e (WiMAX) [1]. Those standardized LDPC codes are based on structured quasi-cyclic (QC) codes, whose parity-check matrices are constructed from blocks of permutation sub-matrices [2]. A QC-LDPC code can be specified by an mb × mc base model matrix Hbm whose (i,j)-th entry is the cyclic-shift value for the (i,j)-th permutation sub-matrix. The full parity-check matrix H of size M-by-N is obtained by replacing each nonempty entry in Hbm with a corresponding z × z permutation sub-matrix and each empty entry with a z × z zero sub-matrix, where z is called the expansion factor in WiMAX. So, M = mb ∗ z and N = nb ∗ z .
For such codes, a block of z rows of parity-check equations, corresponding to a single row in Hbm, can be processed in parallel with no conflicts. So the code structure is very suitable for implementing the partly parallel decoder architecture, where a block of P ≤ z function units is instantiated and shared between block rows for reduced complexity and simplified interconnection. The function units are implemented in serial mode so that they can be flexibly reconfigured to accommodate irregular edge degrees [3].
However, the computation latency introduced by the serial function units make such architectures unsuitable for implementing the efficient layered decoding algorithm (LDA), proposed by Hocevar in [4], for those codes whose adjacent block rows overlap. For instance, Brack, et al., use

LDA only on WiMAX’s rate-1/2 and rate-2/3B codes which can be layer reordered to produce non-overlapping block rows, while still using a two-phase decoding algorithm for the other code groups [3]. Shih, et al., use a less-efficient overlapped two-phase decoding algorithm for WiMAX codes in [5].
Gunnam, et al., use an out-of-order message generation scheme to apply LDA on WiMAX codes in [6]. Their scheme is not applicable to other QC-LDPC code structures of different latency constraints. In contrast, Rovini, et al., have presented a delta-based layered decoding algorithm (DLDA) to completely remove the latency constraint on layered decoding of overlapped block rows [7]. Their algorithm is applicable to any QC-LDPC codes.
In this paper, we apply Rovini’s decoding algorithm, together with the low-complexity offset-based Min-Sum check algorithm (OMS), in [8], to the decoding of the code groups defined in WiMAX. The decoding performance is simulated for two hardware pipeline depths. We then develop a systematic procedure to permute the base model matrix, which specifies the processing orders of the serial functional units, in order to enhance the decoding performance of the delta-based decoding algorithm, at no extra hardware cost. The permutation rule is applicable to any given QC-LDPC codes of any pipeline-depth constraints.

The layered decoding algorithm and its architecture have been shown to be very efficient [4], [6]-[7]. Compared to the conventional two-phase message-passing (TPMP) decoding algorithm, LDA requires about half the number of iterations, half the number of clock cycles per iteration, and 45%-50% fewer memory bits for storing messages.
In a hardware implementation, (1)-(3) and (5)-(10) are computed by a serial check function unit that receives one input bit-sum qj per clock cycle from a bit function unit that consists of a memory block and an adder block implementing (11). The bit function unit is simplified to a memory block in LDA. The check function unit starts outputting one updated bit-sum per clock cycle after all the required bit sums have arrived. To retain the maximal throughput performance of the layered decoding, the check function unit is not stalled from

layer to layer. So a check function unit outputs an updated bit-sum for the current layer while receiving an input bit-sum of the next layer. The processing orders of check nodes (rows) and bit nodes (columns) in the check function units are completely specified by Hbm,; that is, bit nodes are processed in increasing column number, and check nodes are processed in increasing row number.

Let PD be the pipeline depth in the hardware. PD is defined as the clock cycles from the time of the last input bitsum being read for a check function unit to the time of the first updated bit-sum being written back in the bit function unit. If the weight of a row is dc, then the update latency for the j-th bit-sum of the row from the time it is read accessed to the time the corresponding new update is written back is

LAT = dc + PD −1 .

after it is read for the current layer, the new update generated in LDA by the current layer will be overwritten by the later requesting layer which still uses the same old bit-sum, as in (3), and an incorrect bit message will be generated during the next iteration in (1). In contrast, a new differential bitmessage in (10) will always be correctly added into its bitsum in the bit function unit in (11) for DLDA.

If a new update is not finished when its bit-sum is requested by a next layer, we call it a late update in DLDA because the new update is not generated promptly enough to be utilized by the next requesting layer. Those late updates result in performance loss in DLDA, with respect to the ideal latency-free LDA, shown in Section IV. Therefore, those late updates are quantitatively identified and used as a performance indicator.

III. MATRIX PERMUTATION PROCEDURE
In this section, we present a systematic matrix permutation procedure to reduce late updates in DLDA. This procedure is applicable to any QC-LDPC code.
Given a base model matrix Hbm and the hardware pipeline depth PD, we first create a new matrix Hp whose entries in row m are the column index numbers of the nonempty entries in row m of Hbm, as in Fig. 1. Hp represents the original row and column processing order for the serial check function units. We first perform row reordering to reduce the overlap between rows. Then, the columns of each row are reordered to best benefit the two immediate rows that follow, as the processing of these rows in the check function unit is scheduled closely after the given row, exhibiting the most strict latency constraints if they are to utilize the updates generated by that row.

1. Reorder and group the rows in Hp in ascending check weights.
2. Define the overlap count OC (ri,rj) as the number of common entries between row ri and row rj. Define COC as the cumulative overlap count along adjacent rows in a row sequence.

B. Column reordering to reduce late updates
In each operation, we observe the entries in three consecutive rows: a current row in Hp, denoted as R1, the next row, denoted as R2, and the row after the next row, denoted as R3.
The following positioning constraint is introduced to simplify the permutation rules.
• (C-1) Denote SA the subset of columns from R1 whose updates are promptly made to be utilized by R2. SA is placed first in R1, but it is placed last in R2 in order to minimize the number of late updates. Therefore, the columns (bit nodes) in SA are processed first by the check function units for R1, but they are the last columns requested for R2.
The column reordering procedure is stated as follows.
1. Set row index i = 1. R1, R2, and R3 are the first three rows of HP.

IV. DECODING PERFORMANCE SIMULATION
This section presents the decoding performance for the six code groups defined in WiMAX, in floating-point simulations under the following conditions.
• BPSK modulation and AWGN channels are simulated. The offset value β in (6) and (8) is set to 0.5, obtained from Monte-Carlo simulations.
• All rows in a layer are processed in parallel, with a parallelization factor P = z. Two pipeline depths, PD = 4 and PD = 8, are simulated. The pipeline depths are representative numbers for low clock-speed designs and for high clock-speed designs, respectively.
• For each code, the scheduling of row and column processing orders in the serial units are specified by two permuted base model matrices. The first matrix H1 is obtained by layer reordering Hbm in increasing check weights to allow for stall-free operations between layers, and the second matrix H2 is obtained from H1 being further permuted by the procedure in Section III to reduce late updates.
