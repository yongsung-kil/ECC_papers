Switching Activity Reducing Layered Decoding Algorithm for LDPC Codes

Shu-Cheng Chou, Mong-Kai Ku and Chia-Yu Lin
Department of Computer Science and Information Engineering National Taiwan University Taipei, Taiwan
Email: {p94003, mku, f93084}@csie.ntu.edu.tw

Abstract—A switching activity reducing decoding algorithm for Low-Density Parity Check (LDPC) codes is proposed. Our modified horizontal layered decoding algorithm reduces active node switching activities to lower LDPC decoder power consumption. Layered nodes are periodically refreshed to minimize coding gain degradation. A low hardware overhead partially parallel LDPC decoder architecture is also described. Simulation results show that our algorithm reduces the number of LDPC decoder operations up to 62.5% compared to the original layered decoding and improves the original vertical layered Lazy Scheduling much with little performance loss.
I. INTRODUCTION
The Low-Density Parity-Check (LDPC) codes [1] have received great attention for their capacity-approaching performance and parallelizable decoder VLSI architecture. As a result, LDPC codes have been adopted for several industrial standards, such as DVB-S2, WLAN (IEEE 802.11n) and WMAN (IEEE 802.16e). Low power LDPC decoder is crucial for these mobile applications. Message passing algorithm is commonly used to decode the LDPC codes. The iterative LDPC decoding process requires a number of iterations to converge to the correct codeword. For every iteration, huge number of messages is passed between check nodes and bit nodes with complex operation performed in every node. Due to the high complexity of message passing LDPC decoding algorithm, the key to lower power consumption is to reduce the total number of operations required to decode a LDPC codeword.
The operation complexity of the LDPC decoder consists of three factors: the number of operations per processing unit, the number of average iterations, and the number of active nodes in each iteration [2]. Many techniques are introduced to lower the complexity of each individual factor of LDPC decoding. To reduce the complexity of processing unit, scaling min-sum algorithm (SMSA) [3] can be used in place of sum-product algorithm with acceptable performance. Layered decoding [4] can cut the average number of iterations up to 50% by rescheduling the decoder operation. To reduce the number of active nodes within each iteration, bit-level stopping criteria

such as forced convergence [2] can be used. Lazy scheduling [5] is proposed as a hybrid of vertical layered scheduling and forced convergence to reduce both the number of average iterations and the number of operations in each iteration. But the complexity reduction is limited since this scheme requires complicated SPA node processor and more memory-hungry vertical layered scheduling.
For optimal result, we have to consider the complexity reduction of LDPC decoder from all three aspects mentioned above. In this work, we want to combine the strength of SMSA, horizontal layered scheduling and forced convergence to optimally reduce switching activity of LDPC decoder. The decoding algorithms must be modified to maintain good coding gain performance. Existing partially parallel LDPC decoder architecture can be easily modified to use our decoding algorithm. The remainder of this paper is organized as follows. Section II describes the proposed decoding algorithm in detail. Section III shows the proposed architecture design. Section IV presents the simulation results. Section V concludes this paper.
II. THE SWITCHING ACTIVITY REDUCING DECODING
ALGORITHM
Layered LDPC decoding algorithm [4] reschedules the traditional two-phase decoding algorithm to reduce hardware requirement while maintaining the same BER performance. Layered algorithm decodes in block-serial fashion. The number of iterations can be reduced by half due to its more reliable message propagation. In layered decoding, column operations are merged into row operations. Bit information is computed on-the-fly to eliminate the need of memory storage. Thus the decoder logic and memory requirement is reduced by about 50%. In this section, we explore how to further reduce the complexity of layer decoding algorithm.

Min-sum approximation decoding techniques can only be applied to horizontal layered decoding. In check node processors, the SPA uses “product” operations while SMSA uses “min” operations. When the decoding unit processes a layer, it starts by obtaining layer information from the previous layer. The layer information is refined using the message updates of current layer to produce more reliable information and pass to the next layer. To obtain successful decoding result, it is crucial to update the necessary layer information in a correct manner. As Illustrated in Fig. 1(a), while using SPA, both horizontal and vertical layered decoding schedules work well and layer information can be improved layer by layer. However, while using SMSA, only horizontal layered decoding can update layer information correctly. This is because, in vertical layered decoding, the characteristic of bit information should be an increasing value with iterative decoding algorithm. In Fig. 1(b), the “min” operation of C0 needs newly updated bit information from V0 and V1 but only has newly updated bit information from V0. Newly updated bit information from V1 will never send to C0 until the next sub-iteration is finished. The “min” operation compares the newly updated bit information, which should be larger in magnitude, with minimum magnitude of the row of the last iteration will always set layer information to the old value. As a result, layer information never updates and decoding fails.

B. The Switching Activity Reducing Decoding Algorithm Lazy Scheduling for Horizontal Layer with SMSA
In LDPC decoding, a large percentage of variable nodes reaches high reliability within only a few iterations in twophase decoding. When the magnitude of total information reaches high confidence, putting these variable nodes into sleep mode for the consequent iterations will have little impact on the correctness of the decoding result. By disabling the incoming and outgoing message update of these variable nodes, considerable amount of operations can be saved. In terms of memory usage, horizontal layered decoding requires smaller memory storage than vertical layered decoding. In horizontal layered decoding, layer information can replace the channel information and be stored into channel information memory. But in vertical layered decoding, intrinsic information is used to compute total information in each layer operation. Additional memory is required to store the intrinsic information. If the codeword length is N and q bits for quantization, (N ͪ q) bits of extra memory is needed to save layer information.

Appropriate threshold tv can be chosen based on the tradeoff between the amount of operation reduction and BER performance. However, if the threshold is chosen too low, error floor appears due to the wrong decisions in the processing nodes. We proposed an effective solution to eliminate error floor effect. The decoder wakes up the sleeping nodes periodically to update their messages. The update interval of wake up iterations is specified by iu . The exec(tv, Qv, iu) logic determines whether the variable node should participate in the process of update if the Qv is less then tv or current iteration reaches iu. All nodes is waken up to update messages for one iteration. After such iterations, all nodes decide their working status based on the newly updated information.

B. The Additional and Modified Processing Units Design
The key processing unit is the lazy scheduling controller that is responsible to manage for correct execution of the decoder. Variable node status vector, which is a vector of registers of length equals to the code length, keeps track of whether the variable node sleep or not. According to this vector, control unit can generate appropriate chip select (CS) or reset signal of the corresponding units. The other additional and modified processing units do not introduce much hardware overhead. The decision unit is simply a comparator shown in Fig 3(a). It decides whether Qv is exceeded the userdefined threshold tv and feedback the result to variable node status vector. The comparison result can be used by controller to determine whether the subsequent operations should be executed or not. Then Qv is passed to get bit information unit to generate bit information. As shown in Fig 3(b), the get bit information unit computes bit information by subtracting check information from channel information. The only modification is an additional multiplexer. The CS signal is generated by variable node status vector. While it is taken into sleep mode, the controller does not read Rcv. It simply passes Qv received from decision unit instead of the subtraction result.

III. THE PROPOSED ARCHITECTURE DESIGN
A. The LDPC Decoder Architecture
We propose LDPC decoder architecture of the H-LSSMSA. The goal of this architecture is to be applied to the next generation standards, both for regular or irregular LDPC codes, such as DVB-S2, 802.11n and 802.16e. In addition, flexibility for reconfigurable code rate is also considered. The LDPC codes specified in the IEEE 802.16e standard [6] with a block size of 96 are used to prototype this design. To take advantage of parallelism of the LDPC codes, 96 instances for several processing units are employed. By utilizing the blockserial decoding method, the architecture is highly scalable and reconfigurable. V. CONCLUSION
In this paper, we proposed a switching activity reducing decoding algorithm for LDPC codes. Our modified horizontal layered decoding algorithm reduces active node switching activities to lower LDPC decoder power consumption. The processing node complexity, number of active nodes, and average number of iterations are jointly optimized to lower total switching activities. Layered nodes are periodically refreshed to minimize coding gain degradation. Our algorithm can be easily applied to existing hardware decoder designs. Simulation results show that our algorithm reduces the number of LDPC decoder operations up to 62.5% compared to the original layered decoding and improves the original vertical layered Lazy Scheduling much with little performance loss. The application of our algorithm will lead to low power LDPC decoder realization in the future.
