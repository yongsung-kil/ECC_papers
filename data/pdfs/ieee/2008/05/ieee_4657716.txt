Efficient Encoding of Cycle Codes on Graphs with Large Girths
Weigang Chen, Liuguo Yin, and Jianhua Lu Department of Electronic Engineering, Tsinghua University, Beijing, 100084, China
{chenwg, yinlg, lujh} @wmc.ee.tsinghua.edu

Abstract-Low-density parity-check (LDPC) codes of column weight two, also called cycle codes, can be constructed based on connected simple graphs. In this paper, efficiently encodable cycle codes and modified Repeat-Accumulate codes based on Hamiltonian graphs with large girths are devised. Specifically, the Hamiltonian property of the graphs is exploited to obtain efficient encoder structures and a concise description for the Hamiltonian graph named LCF (Lederberg-Coxeter-Frucht) notation is used to reduce the complexity of the encoders, especially the storage requirement. Then, using the Hamiltonian property and the LCF notation, structured cycle codes with large girths and low encoding complexity are devised. Simulation results also reveal that the proposed structured codes exhibit better performance compared with other structured cycle codes.
1. INTRODUCTION
Low-density parity-check (LDPC) codes can asymptotically approach the Shannon limit when the code length goes to infinity [1], [2]. LDPC codes of column weight two, also called cycle codes [3], have many potential applications in partial response channels [4], [5], for their low decoding complexity and good performance.
The block or bit error probability of the LDPC code has a close relation with the minimum distance or the girth especially at high signal-to-noise ratios (SNRs) [5], [6]. The minimum distance, dmin , and the girth of the code, gc, have
the relation of drnin == gc/2 for the cycle code [6]. Therefore,
a large minimum distance can be obtained by constraining the girth. Simulation results also reveal that cycle codes with large girths exhibit good performance at high SNRs [5]-[7]. Cycle codes based on some graphs, especially on cages, may have very large girths. For example, the cycle code on (3, 17)-cage has a girth of 34 and shows prominent performance in the high SNR region [7].
However, the encoding problem of cycle codes still needs further investigation, though cycle codes with large girths have low decoding complexity and excellent performance [8], [9]. A pre-processing method to obtain efficient encoders by exploiting the sparseness of the parity-check matrix is proposed by Richardson and Urbanke in [8]. It is also verified that cycle codes can be encoded in linear time by transforming the parity-check matrix into an equivalent upper-triangular matrix [9]. However, the description of the parity-check matrix in the upper-triangular form may be still complex.
In this paper, more efficient encoding schemes of cycle codes are further investigated using their graph representations

and encoder structures with much lower complexity than the methods proposed in [8] and [9] are presented. Specifically, the Hamiltonian property of the graph is exploited to transform the parity-check matrix into the bi-diagonal form. The encoding using a bi-diagonal matrix is much simpler than using an upper-triangular matrix in [9]. This is the first contribution of this paper. Furthermore, the LCF (Lederberg-Coxeter-Frucht) notation for the Hamiltonian graph is exploited to concisely describe the parity-check matrix, thus reducing the storage requirement. Indeed, the rotational symmetry of the graph is exploited using the LCF notation. This is the second contribution. Thus, structured cycle codes are derived by constructing Hamiltonian graphs with large girths.
The paper is organized as follows. Section II gives a brief introduction to the graph representation of cycle codes and the cycles in graphs. In Section III, the Hamiltonian graphs with large girths are used to construct large-girth efficiently encodable cycle codes and the LCF notations of the graphs are exploited to describe the graphs aiming at further reducing the complexity of the encoders. In Section IV, two efficient encoding structures are presented. Conclusions are drawn in Section V.
II. GRAPH REPRESENTATIONS OF CYCLE CODES
A. Graph Representations of Cycle Codes
Cycle codes, a special type of LDPC codes, are linear block codes defined on very sparse parity-check matrices, and can be described using Tanner graphs. It is also shown that cycle codes without 4-cycles can also be represented by simple graphs following the edge-vertex construction [7], [10]. In this paper, we limit the graphs to connected simple graphs.
A connected graph with neither loops nor multiple edges is a simple connected graph denoted as G(V, E), where V is a nonempty set of vertices, and E is a set of unordered pairs of distinct elements of V called edges [11]. For example, Fig. I (a) illustrates a simple graph with five vertices and nine edges. In a simple graph, let each edge denote a column and each vertex denote a row in the parity-check matrix H, then the parity-check matrix H is obtained as illustrated in Fig. 1(b), which represents a cycle code without 4-cycle. The matrix H is indeed the incidence matrix of the graph G. If we insert a vertex on each edge as a variable node and let the original vertices in G denote the check nodes, the associated Tanner graph T of the cycle code is achieved as shown in Fig. 1(c).

B. Cycles in Graphs
A cycle in a graph is a closed walk in which the vertices and the edges are all distinct [11]. The length of a cycle is the number of edges or vertices it contains. The girth of a graph G is the length of its shortest cycle .
Obviously, in a Tanner graph T derived from G, a cycle consists of twice vertices and edges of the corresponding cycle in the original simple graph G. Therefore, if a simple graph G has the girth of g, the girth of the Tanner graph T derived
from G is 2g. In this way, a cycle code of girth ge == 2g can
be obtained by constructing a simple graph with girth g. There are many methods to generate graphs with large girths in the literature and cages are the best examples [11], [12].
A (k, g)-cage is a regular graph of degree k, girth 9 and
the minilTIum number of vertices v == v (k, g) [11]. Therefore,
a cycle code of check node degree k and girth 2g derived
on a (k, g)-cage has the shortest code length of kv /2. In the
sense of code length, cycle codes defined on cages are optimal among those with the same code rate and girth. However, it is tough to find the cages with large girths and interconnection structures of the cages are usually quite complex. Fortunately, many cages or graphs with large girths have been constructed in the literature [11], [12]. In this paper, a type of structured cycle codes are proposed based on Hamiltonian graphs with large girths and good rotational symmetry.

B. LCF Notations and Parity-Check Matrices
In this part, the LCF notation, a description of Hamiltonian graphs, is introduced to concisely describe the parity-check matrix, which was devised by Lederberg for the representation of 3-regular Hamiltonian graphs, then subsequently modified by Frucht and Coxeter et al., and hence was dubbed the LCF notation [14]. The LCF notation is also called the spoke vector in [15]. This notation achieves its symmetry and conciseness by placing a Hamiltonian cycle in a circle and then connecting specified pairs of vertices with chord edges.

C. Simulation Results
In this part, it is verified that the structured cycle codes derived on hamiltonian graphs with large girths exhibit excellent performance.
Fig. 5 shows the bit error rate (BER) performance of the cycle code on the 3-regular graph of vertices 2520, girth 17 and the LCF notation of (8). The girth of the cycle code is 34, the code rate is 1261/3780, and it is encoded with Structure A as depicted in Fig. 4 (b). The performance of the modified RA code encoded with Structure B derived from the cycle code above is also shown in Fig. 5. Its code rate is 1/3. It is observed that the BER performance with the two encoder structures are quite identical. The modified RA code encoded with Structure B has a more regular code rate compared with the cycle code encoded with Structure A.
In Fig. 5, the performance of another structured cycle code constructed in [6] is also shown. It has the code rate of 1/3, the code length of 4395 bits and the girth of 20. It has regular structure and can also be encoded efficiently. However, its performance is not as good as the proposed code. At a BER of 10-6 , the codes constructed based on Hamiltonian graph in this paper have 1.5 dB gain over the code in [6].
The error performance of the cycle code on (3, 17)-cage in [7] is also presented in Fig. 5. Its code rate is much similar to the code proposed in this paper. Its performance is also almost identical. However, the disadvantage of the code in [7] is that its encoding is more complex, for its Hamiltonian property and the LCF notation is hard to determine. Indeed, it is an NP-complete problem to find the Hamiltonian cycle of a specified graph. Indeed, the (8,6)-graph in [5] does have a Hamiltonian cycle and can be encoded in linear time. However, it is hard to determine a concise LCF notation. The method can also been extended to the codes with girth 12 in [6] and using the LCF notation.
In conclusion, the advantages of the proposed structured cycle codes fall into two aspects. One is that they can be efficiently encoded compared with the codes in [7]. The other is that the proposed codes show significant performance improvement compared with the structured cycle codes in [6].
V. CONCLUSION
Efficient encoding problem for large-girth cycle codes is investigated in this paper. Based on the graph representations of cycle codes, the Hamiltonian property of the graph is exploited to obtain efficient encoders. Furthermore, LCF notations of the Hamiltonian graphs are used to reduce the storage requirement of the encoders. With this two innovations, efficiently encodable cycle codes with large girths are obtained. Also, this type of structured cycle codes exhibits significant performance compared with other structured cycle codes.
