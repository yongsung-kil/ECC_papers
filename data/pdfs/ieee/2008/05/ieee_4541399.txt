Area Efficient Controller Design of Barrel Shifters for Reconfigurable LDPC Decoders
Daesun Oh and Keshab K. Parhi Department of Electrical and Computer Engineering University of Minnesota, Minneapolis, MN 55455 USA
Email: {dsoh, parhi}@umn.edu

Abstract— In this paper, we propose an efficient controller design of barrel shifter for reconfigurable low-density paritycheck (LDPC) decoders, whcih leads to significant reduction in hardware complexity. Since the structured LDPC codes for the most modern wireless communication systems include multiple code rates, various block lengths, and different sizes of submatrices, a reconfigurable LDPC decoder is desirable and the barrel shifter needs to be programmable. Even though the Benes network can be optimized for the barrel shifting networks of reconfigurable LDPC decoder, it is not trivial to generate all the control signals for numerous 2x2 switches on-the-fly. A novel simplified algorithm capable of generating all the control signals is proposed using the properties that both the full-size Benes network can be broken into two half-size Benes networks and the barrel shifters needed in the structured LDPC decoders require only cyclic shifts. The proposed algorithm can be easily implemented with a small numbers of gates. Compared with the direct implementation using a dedicated look-up table, the proposed algorithm achieves a significant hardware reduction in implementing a reconfigurable LDPC decoder.
I. INTRODUCTION
The highly parallel implementations and remarkable error correcting capabilities of low-density parity-check (LDPC) codes have led them into modern wireless communications standards, such as IEEE 802.11n, IEEE 802.16e, and DVBS2. Recently, many efforts have attempted to simplify the VLSI implementation of the related decoders called “BlockLDPC codes” [1] or “Architecture-Aware LDPC codes” [2]. Based on these design approaches, the structured LDPC codes have received significant attention due to their efficient hardware implementation and good performances [3]. The parity-check matrix H of these codes can be partitioned block-wise and implemented with partially parallel decoder architectures, which can achieve a good tradeoff between hardware complexity and decoding throughput.
In the structured LDPC codes, the parity check matrix is composed of cyclic permutation submatrices that determine the interconnection network between the check node units (CNUs) and the variable node units (VNUs). The barrel shifting network for interconnections between CNUs and VNUs can be pre-determined and optimized for a specific given code. However, since a LDPC decoder for the most

modern wireless communication systems must accommodate multiple code rates, various block lengths, and different sizes of submatrices, the barrel shifting networks need to be dynamically reconfigurable [4]-[6].
Generally, the barrel shifting networks are optimized for a given size of the permutation matrices with the wellknown S × S Benes network, which is capable of performing any sort of permutation over a S data where S is a power of 2 [7]. To properly control all the 2x2 switches in the Benes network, the configurable bits must be provided for 2S output combinations. However, when only cyclic shifts are required, the number of output combinations is reduced to ∑Si=1 i + (1 − S) . Typically, in the Benes network for a reconfigurable LDPC decoder, these control signals are stored in a dedicated look-up table (LUT) [5][6]. However, it could be inefficient or impractical for a large size Benes network.
Recently, a reconfigurable shuffle network architecture has been presented in [8], which is based on the use of two Benes networks: one actually shuffles data while the other configures the first one on-the-fly. Even though this can reduce the hardware complexity compared to direct implementations using S-to-1 multiplexers, it still requires a huge hardware complexity.
In this paper, we propose a novel efficient controller design of barrel shifter for reconfigurable LDPC decoders using the Benes network structure, which can support any kind of cyclic shifting network for smaller submatrices than the input size of Benes networks. The proposed algorithm makes use of two valuable properties: One is that the full-size Benes network can be broken into two half-size Benes networks, and the other is that the barrel shifting network in the LDPC decoder requires only cyclic shifts.

A simple block diagram for partially parallel LDPC decoder

B. Switch Networks
As shown in Fig. 2(a), the Benes network [7] is nonblocking when all input-to-output connections are known in advance. Each switch element can be set either in the BAR state or in the CROSS state as illustrated in Fig. 2(c). The Benes network known as an optimal non-blocking network achieves any desired permutation with S/2 × (2 log2 S − 1) 2x2 switches, where the input size of the Benes network, S, is a power of 2.
It has been shown that the Banyan network, which is only the first log2 S stages or the last log2 S of S-input Benes network, can perform all cyclic shifts for S inputs as shown in Fig. 2(b). In other words, with smaller number of switching elements than that of the Benes switch, it can be nonblocking

when a permutation represents a cyclic left or right shift. However, the Banyan networks have some difficulties to provide the flexibility when the value of S is not a power of 2 and is changeable [9]. In most modern applications, the reconfigurable LDPC decoder should support different values of S, which might be changed on-the-fly and does not belong to a power of 2. For example, the values of S in the LDPC codes for IEEE 802.11n proposed by TGn Sync and for IEEE 802.16e ranges from 24 up to 72 and from 24 up to 96, respectively. Therefore, the Banyan networks would be not suitable for reconfigurable LDPC decoders.

C. Reconfigurable Barrel Shifting Network

For structured LDPC decoder, it is necessary to implement

not all possible permutations but only cyclic shifted permu-

tations with each size of submatrix. Therefore, the number

of output combinations in the S-input Benes network would
be reduced to ∑Si=1 i + (1 − S) instead of 2S. In the S × S Benes network, the control signals for all the

2x2 switches can be determined based on the values of p

and c, where p and c denote the size of changeable permu-

tation matrix and the index of cyclic shifting, respectively.

Generally, with the same Benes network, there might exist

multiple sets of control signals which can perform the same

permutation. One typical implementation method is to pre-

compute and store one possible set of control signals in a

dedicated LUT for every supported combination of (p, c)

when the values of p belong to a limited range. However,

as the number of choices for p increases, the required size

of LUT increases to

As shown in Fig. 3, the Benes network with input size S can be constructed from two Benes networks with input size S/2. In other words, a full size Benes network can be broken into two half size ones. In this paper, denotes (p, c, S) cyclical shifting of p messages by c in the S-input Benes network. The proposed algorithm can be described as follows:

Algorithm A
Configurate(p, c, S) {
Step 0. Check the least significant bits of p and c, and compute (p − c)/2 and p/2 .

Step 1. Divide the p messages into two groups of p/2 and (p + 1)/2 messages by setting 2x2 switches in the first stage of S-input Benes network as follows: if p == even, Set all the switches to the BAR state
else Set top-most (p − c)/2 switches to the BAR state, and Set the others to the CROSS state.
Step 2. Cyclically shift two groups of p/2 & (p + 1)/2 messages by using two S/2 × S/2 Benes networks. In other words, call Configurate( p/2 , c/2 , S/2) and Configurate( (p + 1)/2 , (c + 1)/2 , S/2).
Step 3. Merge cyclically shifted two groups of p/2

and (p + 1)/2 into the p messages by setting 2x2 switches in the last stage of S-input Benes network as follows: if p == even and c == even,
Set all the switches to the BAR state else if p == odd and c == odd,
Set all the switches to the CROSS state else if p == even and c == odd,
Set top-most p/2 switches to the CROSS state, and Set the others to the BAR state else if p == odd and c == even, Set top-most p/2 switches to the BAR state, and Set the others to the CROSS state. }
In Fig. 4, four different examples are used to demonstrate the proposed algorithm instead of a rigorous proof due to limited space. By applying the same algorithm to half-size Benes network continuously, all the control signals for the S × S Benes network can be generated completely.
