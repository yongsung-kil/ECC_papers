Abstract— On the binary erasure channel, the performance of LDPC codes decoded by iterative algorithms is dominated by small-size stopping sets, especially minimal stopping sets. In this paper, we propose a probabilistic algorithm for computing the minimum size of stopping sets of LDPC codes. In this algorithm, we generate efficiently small-size stopping sets by using the relation between small-size stopping sets and lowweight codewords. In numerical experiments, we could compute the minimum size of stopping sets of LDPC codes of length about 500, 1000 and 5000 with high reliability, and could find smaller stopping sets than that found by conventional methods.
I. INTRODUCTION
Low-density parity-check (LDPC) codes show the decoding performance close to the Shannon limit by the belief propagation (BP)-based iterative decoders [1]. LDPC codes and their decoding algorithms are represented by bipartite graphs called Tanner graphs. On the binary erasure channel (BEC), the performance of iterative decoded LDPC codes is dominated by a certain structure in the Tanner graph, which is called a stopping set. C. Di et al. introduced the concept of stopping set and analyzed the average bit and block erasure probability of LDPC codes under BP-based iterative decoding algorithms over the BEC [2]. A. Orlitsky et al. showed asymptotic behavior of the distribution of stopping sets [3]. And, the asymptotic stopping set distribution of several LDPC code ensembles has been analyzed [4][5]. In these analyses, the average stopping sets and their distribution of codes in LDPC ensembles are used to drive asymptotic properties of enumerating functions. Because finding small stopping sets in a given LDPC code is difficult combinatorial problem which requires exhaustive search. The problem of determining the smallest size of stopping sets of LDPC codes is conjectured to be NP-hard [6][7].
Recently, some effective methods to find small-size stopping sets of LDPC codes have been proposed. X.-Y. Hu et al. presented a method, called probabilistic subspace approach, for computing the minimum size of stopping sets of LDPC codes [8]. In this method, submatrices are constructed by random choosing some columns of a given LDPC matrix, and the minimum size of stopping sets is exhaustively searched in subcodes defined by submatrices. In [9], a simple algorithm modifying the error impulse (EI) method [10] was presented. In this algorithm, the EI method which generates low-weight codewords by the decoding scheme is applied to finding the small stopping sets. Numerical results obtained by applying these methods to several LDPC codes were reported in [8] and [9]. However, these methods are not necessarily efficient

to compute the minimum size of stopping sets, since these methods cannot generate small stopping sets positively. On the other hand, in [11], C.-C. Wang et al. proposed an algorithm for exhausting all low-weight error-prone patterns including codewords and stopping sets of LDPC codes. And, in [12], E. Rosnes et al. proposed an efficient algorithm to find all stopping set whose size is less than a given threshold. It was introduced by modifying an algorithm presented in [13]. However, determining the minimum size of stopping sets of long random LDPC codes is yet difficult problem which requires exponentially growing complexity.
In this paper, we propose a probabilistic algorithm for computing the minimum size of stopping sets of LDPC codes. It is a novel algorithm to generate positively small-size stopping sets from a given LDPC matrix. To reduce the complexity to find small-size stopping sets, we focus on the relation between small-size stopping sets and low-weight codewords, and apply Stern’s algorithm [14] which is a probabilistic algorithm for finding low-weight codewords to generating small-size stopping sets. Using the proposed algorithm, we can compute the minimum size of stopping sets of long LDPC codes with high probability. Additionally, as the numerical experiments, we computed the minimum-size stopping set of widely-known LDPC codes, including MacKay’s (504, 252), (1008, 504) codes [17], (504, 252) progressive edge-growth code [18], and Ramanujan-Margulis (17, 5) code of length 4896 and dimension 2474 [19]. In these results, we could newly find a minimum-size stopping set of (1008, 504) code which has not been found by conventional methods, and could obtain the minimum size of stopping sets with high accuracy. These results indicate that our probabilistic algorithm is efficient to compute the minimum-size stopping set of LDPC codes.

Iterative decoding algorithms for LDPC codes, roughly speaking, work in a message-passing manner on the Tanner graph. C. Di et al. characterized the key object needed to analyze the performance of iterative decoding over the BEC as follows:
Definition 1: [2] Let G = (V ∪ P, E) be a Tanner graph. A stopping set S is a subset of V , the set of variable nodes, such that every neighbor of S is connected to S at least twice.
The size of the stopping set is also the cardinality of S. In the message-passing manner, the iterative decoder cannot determine the variable nodes contained in S. Since even if all other bits were known, every neighbor of S has at least two connections to S and so all messages to S will be erasure messages. VI. CONCLUSION
In this paper, we proposed a probabilistic algorithm for computing the minimum size of stopping sets of LDPC codes. It can efficiently generate small-size stopping sets from a given LDPC matrix by using an encoding scheme. In the method, we apply the probabilistic algorithm to find lowweight codewords to generating small-size stopping sets of LDPC codes. According to authors’ knowledge, no algorithm with the same scheme as our algorithm has been presented yet. Using our probabilistic algorithm, the minimum size of stopping set of long LDPC codes can be computed with greater

reliability than by conventional methods.
