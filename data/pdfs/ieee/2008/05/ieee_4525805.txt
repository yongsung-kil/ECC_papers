A New Approach for the Construction of Powerful LDPC Convolutional Codes
Christian Cardinal, Yu-Cheng He, and David Haccoun Department of Electrical Engineering E´ cole Polytechnique de Montre´al Montreal, Quebec, Canada
Email: {christian.cardinal, yucheng.he, david.haccoun}@polymtl.ca

Abstract—A novel approach for the algebraic construction of low-density parity-check (LDPC) convolutional codes is presented. It is based on the orthogonality structures of the codes. The proposed code construction leads to a girth of at least 10 in the Tanner graph. The error performance of these codes compares favorably with the usual LDPC convolutional codes, especially at low signal-to-noise ratio range.
Index Terms—Belief propagation; convolutional LDPC codes; iterative decoding, threshold decoding.
I. INTRODUCTION
Algebraic constructions for low-density parity-check (LDPC) codes yield low encoding complexity as well as efficient implementation of iterative belief propagation (BP) decoding. A large number of algebraic constructions, including quasi-cyclic and finite geometry constructions, have been presented [1]–[3]. As a class of algebraic LDPC codes, LDPC convolutional codes may further be divided into two types: nonrecursive and recursive, depending upon their encoder structures. It has been shown that LDPC nonrecursive convolutional codes operate at moderate signal-to-noise ratio (SNR) values expressed in Eb/N0 [3], [4]. The coding gains of iterative BP decoding for these codes are limited to their asymptotic coding gains under soft decoding. Therefore, LDPC nonrecursive convolutional codes suffer a modest degradation of error performance when compared with random LDPC block codes having similar lengths. However, LDPC recursive convolutional codes could achieve good error performances at low Eb/N0 values [5].
Iterative BP algorithm is a suboptimal decoding method for LDPC codes which have cycles in their Tanner graph representations. It is known that the iterative BP algorithm converges to the optimal maximum a posteriori (MAP) solution in a cycleless graph, whereas it has no such optimality for graphs with cycles [6]. However, avoiding short cycles in the graphs could improve the iterative BP algorithm. If the girth, or the length of the shortest cycle, of a graph is large, the messages exchanged between the nodes in the graph can keep independent for more iterations [7]. For this purpose, randomly constructed LDPC codes largely rely on the size and sparsity of the parity-check matrix in order for iterative BP decoding to achieve good error performance [8].
LDPC convolutional codes, whether recursive or nonrecursive, can be decoded by iterative BP decoding using a

pipelined architecture similar to the one proposed for the recursive time-varying periodic convolutional codes [5]. A fundamental requirement of BP for LDPC convolutional codes remains to be the orthogonality property imposed on the set of the parity-check equations for each code symbol [5], [9], [10]. Furthermore, it has been shown that for nonrecursive systematic convolutional codes, the double orthogonality yields a minimal number of cycles of lengths six and eight in the Tanner graphs. As a consequence, both the error performance and convergence speed of iterative BP decoding for these codes are improved [4], [11].
In this paper, we present a new code construction for a class of LDPC recursive systematic convolutional codes using the difference families associated with the code generator matrix. The code structure is easily analyzed and explained using a computation tree. It can be shown that the proposed code construction systematically avoids cycles of lengths smaller than ten, thus yielding a girth larger than eight in the Tanner graphs. The error performance of the proposed codes is examined by computer simulations over the additive white Gaussian noise (AWGN) channel.
The remainder of the paper is organized as follows. Section II presents the code structure. Section III gives message update rules for iterative BP decoding. Section IV illustrates the simulation result for a rate 1/2 code. Section V concludes the paper.

C. Definition of Recursive Convolutional Doubly-Orthogonal Properties
In this subsection, we define the recursive convolutional doubly-orthogonal (RCDO) properties required to improve the iterative BP decoding of the proposed LDPC recursive systematic convolutional codes above. The definition is deduced from the iterative application of the parity-check equations in (6)–(8). It may also be derived from the corresponding computation trees for the iterative BP decoding constructed by iteratively applying the parity-check equations.
Define ΛN ≡ {1, 2, . . . , N } and consider directly the elements gij of the code generator matrix G in (1), where i ∈ ΛN , and j ∈ ΛV . Following (1)–(3), we have gij = aij for 1 ≤ i ≤ K and gi,j = bi−K,j for (K + 1) ≤ i ≤ N . For any pair i, l ∈ ΛN , let ΛN \{i, l} denote the subset of ΛN with the two elements in {i, l} excluded. A set of the order-1 differences and a set of the order-2 differences associated with G is defined as follows

Dil ≡ {(gij − glj ) : j ∈ ΛV }

(11)

Eil ≡ {(gij − gkj) − (gln − gkn) : k ∈ ΛN \{i, l}; j, n ∈ ΛV , j = n } (12)

where in each set, the elements containing any zero entry gij = 0 with i > K, which corresponds to bi−K,j = 0, are excluded as indicated in (6)–(8).
Two families of the sets of differences with two different orders are then obtained, respectively, as

FD = {Dil : i, l ∈ ΛN , i = l}

(13)

FE = {Eil : i, l ∈ ΛN }.

(14)

Definition 1: The RSC code specified by a code generator matrix G is called a Recursive Convolutional DoublyOrthogonal LDPC (RCDO-LDPC) code if the following conditions are satisfied:
1) All the elements of every set Dil ∈ FD are distinct; 2) For any given pair i = l ∈ ΛN , the set Dil ∈ FD and
the set Eil ∈ FE contain no elements in common, i.e., Dil ∩ Eil = φ; 3) All the elements of every set Eil ∈ FE are distinct.
In fact, the three conditions above define an orthogonality of order 2 for the code generator matrix G [4], [11]. It can be shown that Condition 1) reflects the orthogonality property of the parity-check equations on each information symbol or each parity symbol. The orthogonality property identified by Conditions 2) and 3) is able to guarantee the independence of inputs over two successive BP decoding iterations. From the point of view of the Tanner graph, Condition 1) removes all the 4-cycles, Condition 2) removes all the 6-cycles, and Condition 3) removes all the 8-cycles, thus leading to a girth no less than 10 [4], [6]. It is worthwhile mentioning that the girth is independent of the choice for N and K and the code generator matrix G obtained following the code definition.
A large number of new RCDO-LDPC codes with various values of N and K have been determined using heuristic search algorithms developed by the authors. For each given pair (N, K), the code generator matrix with the smallest maximum entry, which corresponds to the smallest memory order m of the encoder, is selected following the conditions imposed on the associated differences sets. IV. SIMULATION RESULTS
The performances of RCDO-LDPC codes over the AWGN channel have been obtained by computer simulations. Fig. 2 presents the error performance of the code given by (15) and (16) using an iterative min-sum, or simplified BP, decoding algorithm similar to that of [5]. It is seen that the iterative BP algorithm for the code performs rather well at low Eb/N0 values and compares favorably with other LDPC convolutional codes [5]. Fig. 2 shows that a bit error rate of 10−6 is reached after 40 iterations at Eb/N0 = 1.25dB and that the error performance improves as the number of iterations increases.

V. CONCLUSION
A new algebraic construction for recursive convolutional LDPC codes has been briefly presented, satisfying the conditions of the second order orthogonality and thus leading to RCDO-LDPC codes. The new algebraic construction as well as an analysis of the orthogonality structures of these codes will be presented in detail. Using iterative BP decoding algorithm, these codes have been shown to perform rather well at low Eb/N0 values. The low complexity encoding together with the efficient decoding for the new codes indicates that these codes present an attractive alternative to the usual
