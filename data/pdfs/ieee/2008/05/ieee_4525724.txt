A 223Mbps FPGA implementation of (10240, 5120) irregular structured low density parity
check decoder

Wang Wenjun*, Wu Xiaoguang*, Zhu Xiaoxuan, Kang Guixia, Tao Xiaofeng Key Laboratory of Universal Wireless Communication， Ministry of Education,
Beijing University of Posts&Telecommunications, Beijing 100876, P.R.China
*These authors contributed equally Email: wangviking@gmail.com

Abstract—This paper presents a high speed decoder architecture for irregular structured low density parity check (LDPC) codes and its field programmable gate array (FPGA) implementation. Algorithm transformation and architectural level optimizations are employed to reduce the critical path. The enhanced semi-parallel architecture is easily scalable and reconfigurable for larger block sizes and can be well suited for achieving high decoding throughput. Based on the proposed architecture, a (10240, 5120) irregular structured LDPC decoder is implemented on Xilinx FPGA Virtex-4 VLX80, the FPGA implementation results show that the irregular LDPC decoder can achieve a maximum (information data) decoding throughput of 223 Mbps at 18 iterations.
Index Terms—LDPC, FPGA, Semi-parallel architecture
I. INTRODUCTION
LDPC [1] codes, as the next generation coding scheme, have replaced turbo codes to become the newest research focus of Forward Error Correction (FEC) codes worldwide. Research has shown that LDPC codes can achieve record-breaking performance for low SNR applications and are more amenable to rigorous analysis and design. LDPC codes require low implementation complexity [2] and can choose code parameters more flexibly. They have been adopted in the next generation digital video broadcasting (DVB-S2) via satellite [3], and considered for adoption in wireless local area network (WLAN) air interface (802.11), wireless personal area networks (WPANs) (802.15), mobile broadband wireless access

(MBWA) networks (802.20), and advanced magnetic and magneto-optic storage/recording systems.
A special type of LDPC codes, named structured LDPC [4] codes are defined by a sparse parity-check matrix H, which consists of blocks of circularly shifted identity matrices or zero matrices. The structured LDPC codes can reduce the complexity of the interconnections between bit nodes and check nodes and are well suited for hardware implementation. The encode of structured LDPC codes can be built with shift-registers. In addition, the great advantage of structured LDPC codes is that designers can easily change the throughput of the decoder by adjusting hardware cost, namely the number of check node function units (CNFUs) and variable node function units (VNFUs).
Zhang [5,6] researched the hardware implementation of the structured LDPC codes which can achieve up to 54 Mbps information decoding throughput. Chen [7] implemented a semi-parallel architecture for a rate 1/2, 8088 bit irregular LDPC code on both FPGA and ASIC. Their architecture can achieve up to 40Mbps at 25 iterations for FPGA. Karkooti [8] implemented a decoder for (3, 6) 1536-bit QC-LDPC codes which can obtain 63.5Mbps information decoding throughput. However, in order to facilitate the target level of parallelism, Parity check matrix should be generated under an additional constraint which unavoidably limits the performance of the LDPC codes.
In this paper, first, a well performed irregular structured parity check matrix is presented, which is suitable for semi-parallel hardware implementation. Second, a semiparallel architecture for decoding LDPC codes is designed, which is easily scalable and reconfigurable for larger block sizes. Memory management and alignment scheme are also discussed. Finally, the detailed FPGA implementation of (10240, 5120) irregular structured LDPC codes is presented which can achieve a maximum (source data) decoding

III. BELIEF PROPAGATION DECODING ALGORITHM AND MODIFICATION

LDPC codes can be effectively decoded by the iterative belief-propagation (BP) algorithm [1], which approximates the maximum likelihood decoding. In the hardware implementation, the Log-BP algorithm [5] would be employed in order to decrease the decode complexity. The conventional Log-BP algorithm has unbalanced computation complexity between the CNFUs and VNFUs, and the clock speed in the node processing will be upper-bounded by the critical path, thus the throughput of LDPC decoder will be limited.

denote N (m) \ n as the set N (m) with bit n excluded, and

M (n) \ m as the set M (n) with check m excluded.

IV. SEMI-PARALLEL ARCHITECTURE AND ENHANCEMENT
In this section, the block diagram of our semi-parallel architecture is presented in Fig.2 according to the algorithm described above. As we can see, Symbol Log-Likelihood-Ratio (LLR) that stored in the 32 LRAMs is fed to the decoder as an input value, and the decoded bit after parity check is stored in 32 CRAMs. The number of memory blocks MemA/ MemB depends on the number of non-negative-one element in shifting offset matrix Hb , and each memory block can save z (the expanded factor) node-to-node soft information, and each soft information can be represented by 5 bits. Each MemA/ MemB is a dual-port memory, one port is always configured for read and another one always configured for write. The READ address of each MemA/ MemB is generated by a binary counter which is initialized to 0 at the beginning of the variable node updating and initialized to shifting offset number at the beginning of the check node updating. The WRITE address is simply a delayed version of the read address depending on how many pipeline stages are inserted in the data paths between the MemA/ MemB data output and input ports.
In order to fully utilize the CNFUs/VNFUs and increase the decoding throughput, the input LRAMs store two blocks of codeword, one codeword is being decoded while the other codeword is stored for next decoding. Dual memories are adopted such that the MemA/ MemB store the node-to-node information for different two codeword respectively, and the MemA/MemB can be incorporated in one block RAM in FPGA implementation. During the computations of the current iteration, CNFUs process the variable-to-check information for one codeword, after the first half of the iteration is complete, CNFUs still should process the new variable-to-check information which just be updated by VNFUs in the first half of

Fig.2 Semi-parallel architecture for structured LDPC codes
Due to its simple data paths and fixed interconnection structure, the enhanced semi-parallel architecture is easily scalable and reconfigurable for a family of codes with different block lengths, parallelism ratios and information lengths. When the block length of the codeword changes, all the CNFUs, VNFUs and address generators still can be used, while only the expanded factor z needs to be changed. There will be a slight modification in the address generator units which can be done by changing the size and initialized number of the counters used in the address generators. Since the counters are parametric in the VHDL (Hardware Description Language) code, this can be done with a new compilation of the code using these new values.
A. Processing Units Architecture
For the parity check matrix Η here, we can update any one row information by formula (2) in the modified Log-BP algorithm. Since the row degree of the parity check matrix is always 8, the architecture of CNFUs with 8 inputs Zm,n and 8 outputs Lm,n is given in Fig.3. The sign bits of inputs Zm,n are put into the XOR tree and the absolute values of inputs Zm,n are put into the adder tree and LUT-C. Due to the symmetry property of the ψ function in CNFUs, the LUT-C’s size could be reduced by half if all the inputs are in the absolute value data format. B. Memory partition and data switch
The offset of cyclically shifted identity matrix Hb affects on the connection or initial address access between CNFUs, VNFUs and memory blocks. For the considered structured LDPC codes, because the cyclic shifting offsets are generally not multiples of the targeted parallelism level, to facilitate multiple data accessed for each cyclically shifted matrix in both row and column updating phases, memory partition and data switch schemes are developed as Fig.5.

With the consideration of double parallelism, the key issue for this enhancement is how to access two soft informations corresponding to each sub-matrix at each clock cycle. We define two information values as MEM_odd and MEM_even, and store them in a single memory address. When the sub-matrix’s shifting offset is even (multiple of 2), there is no data access confliction. When the sub-matrix’s shifting offset is odd, some data access adjustment need to be done. The input schedule is given as follows: at the first clock of column updating phases, MEM_odd and MEM_even should be stored in REG_E and REG_O after the data switch unit; then respectively the CNFU_odd and CNFU_even process the information read from REG_O and MEM_odd every clock; at the last clock, REG_O and REG_E are chosen to be processed by CNFU_odd and CNFU_even. The output schedule of CNFUs is similar to the input schedule: the first output information from CNFU_odd and CNFU_even should be stored in REG_O and REG_E; then REG_O and CNFU_even flow to the data switch unit every clock; at the last clock, REG_O and REG_E are chosen to flow to the data switch unit. In this way, the data access confliction is eliminated, and the proposed scheme can be easily extended to the higher parallelism case.

VI. CONCLUSION
In this paper, a low complexity, high speed semi-parallel architecture for irregular structured LDPC codes has been designed and implemented on Xilinx Virtex4 VLX80 FPGA. This architecture is easily scalable for different block sizes, information lengths and parallelism factors. Various techniques are incorporated to increase the clock speed. Furthermore, detailed memory partitioning and data switching schemes are also discussed. The detailed FPGA implementation of (10240, 5120) irregular structured LDPC codes is presented which can achieve a maximum (source data) decoding throughput of 223 Mbps at 18 iterations.
