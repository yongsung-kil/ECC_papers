1. Introduction

Forward error correction (FEC) techniques have been greatly developed for the demand of high reliability in real time digital communication. LDPC codes, first proposed by Gallager [1] in 1960, have been achieving great interest since the rediscovery by Mackay and Neel [2]. Among several FEC encoding schemes, such as convolution codes, Reed-Solomon codes, and turbo codes, LDPC codes provide nearest-capacity performance [3]. Consequently, LDPC codes have been adopted in broadcasting systems, such as next generation digital video broadcasting via satellite (DVB-S2) [4], and have been optional FEC techniques in networks, such as Wireless-LAN (802.1 In) [5] and Wireless-MAN (IEEE 802.16e) [6], etc.
There are mainly two families of algorithms utilizing iterative passing of soft messages in the decoding of LDPC codes based on soft-decision. The one invented earlier is two phase message passing (TPMP), which performs iterations between check node process and variable node process, including belief propagation algorithm (BPA) and enhanced/simplified variations [7], [8]. The other performs turbo-decoding message-passing (TDMP), main advantage of which is that convergence is accelerated by a factor of 2 for the message direct passing mechanism, as developed by M. M. Mansour [9]. To further reduce the complexity of TDMP algorithm, a novel free-of-Iookup-table updating scheme, which is channel-independent as well, is proposed in this paper.
On-chip memories occupy great part of the area and power dissipation ofLDPC decoder [10], [11], [12], and

need to be carefully arranged. In this work, dual-port and single-port memory are deployed, and the number of dual-port memory is reduced by careful arrangement. The paper [10] and [11] distribute read-only-memory (ROM) to store code configuration information. To insure programmability for further applications, cache memory, which can be reconfigured, is used in this paper for storing configuration information ofLDPC decoder.
Besides, another challenge of implementing a flexible LDPC decoder is to design a programmable message passing network without interconnection congestion. In this paper, a flexible message-passing network is devised to ensure free of interconnection congestion.
The structure of this paper is organized as follows. Section 2 briefs a class of quasi-cyclic LDPC (QC-LDPC) codes, and introduces the proposed M-TDMP algorithm. The proposed decoder architecture will be described in section 3. Section 4 demonstrates implementation results and compares them with recent works. Section 5 concludes the paper.
2. QC-LDPC codes and the proposed algorithm
2.1 QC-LDPC codes QC-LDPC codes or so-called Architecture-aware
LDPC (AA-LDPC) codes are well received for the inherent regularity apt to avoid congestion in decoder chip [9], [10] and [11]. LDPC codes can be defined by a parity check matrix H of size M-by-N, where N is the length of the code and M is the number of parity check bits. The number of information bits is K, and equals to N-M. Each QC-LDPC code can be sub-blocked for its regularity, and the matrix H is decomposed into multiple sub-matrices, which are either zero matrices or cyclic-shifted identity matrices of size Zr-by-Zf, along with a mb-by-nb base-matrix Hb for permutation pattern. And thereby, a class of sub-matrices and base-matrix integrally represent a set of QC-LDPC codes with code rate r = (nb-mb)lnb. QC-LDPC codes have been adopted in DVB-S2 and Wireless-MAN systems, etc.

Conventional stopping rule of parity check equation works inefficiently, and early stopping in iterative decoding ofLDPC codes was worked at by [13] and [14]. 3. The proposed decoder architecture
The proposed flexible decoder architecture is based on the four-step sub-iteration processing with a 6-bit quantization data-path. The pipelined partially-parallel architecture with scheduling and input/output buffering, as depicted in Fig.2, is devised to reduce the decoding latency, and the four-step processing only consumes Ci+3 clock cycles. The nominal number of processing cycles for a frame of 2304-bit is Ci+3 multiplying iteration number T, and the actual number is Ci+3 multiplying actual iteration numbers, which depends on channel conditions. To ensure flexibility, a pattern memory is deployed as cache, and can be reconfigured during decoding process. Different from the work [11], which overlaps the decoding process to improve convergence by doubling the processing elements, this architecture is area efficiently with the pipeline structure and turbo-decoding scheduling. All memories are implemented as register files, and posterior memories are of two-port, and extrinsic memories are of single-port.

Each network consists of 7 stages of logarithm shifters. The network is controlled by controlling code stored in the pattern memory for different code rates (1/2, 2/3A, 2/3B, 3/4A, 3/4B, 5/6).

The low complexity SISO engine performs four-step sub-iteration with 3-stage pipeline to reduce power and improve throughput. The newly generated extrinsic messages are written back to memory blocks and read for update using a decompression operation ahead. Fist-in first-out (FIFO) operations are manipulated to read/write intermediate p messages [10]. Comparison and selection (CS) are performed for message-magnitude updating and exclusive-or (XOR) operations are used to update sign of the messages. The SISO engine in [10] using forward-backward recursion consumes large power and the proposed SISO engine is of low power, which will be shown later.
The programmable network for message passing is devised to support flexible code rate of Wireless-MAN applications. There are two networks of the same structure are designed for reading and writing separately.

5. Conclusion Based on the M-TDMP algorithm and the early
termination scheme, a flexible LDPC decoder for Wireless-MAN applications was presented. The novel architecture was devised to achieve small area, high throughput and low power, which made the decoder advantageous over recent proposed ones.
