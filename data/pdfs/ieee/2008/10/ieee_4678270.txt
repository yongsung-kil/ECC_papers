Abstract— A joint optimization method of LDPC codes using EXIT chart is introduced in this paper. Based on the design method proposed by Brink, we take into account the variable nodes degree and check nodes degree together in the code design process, other than fixing the check nodes degree in advance. Compared with the non-joint design method, the joint optimization method can generate LDPC codes with nearer Shannon limit performance.
Keywords- LDPC, EXIT, Joint Optimization
I. INTRODUCTION
LDPC codes is a class of linear block codes specified by a parity-check matrix containing mostly 0’s and only a small number of 1’s. It was originally invented by Gallager in 1963[1], although it appeared a good performance at that time, people didn’t realize it’s advantage due to the limited computing ability. It was forgotten for decades until 1995, MacKay and Neal rediscovered it [2]. And since it can easily beat the best known turbo codes when code length is long enough, more and more researchers are now attracted by LDPC codes.
The encoding of LDPC codes is the same as that of common block codes. But it has a special decoding method--an iterative probability algorithm known as the sum-product or belief propagation. Brink first proposed to use mutual information transfer characteristics of soft in/soft out decoders as a tool to better understand the convergence behavior of iterative decoding schemes [3]. The exchange of extrinsic information is visualized as a decoding trajectory in the extrinsic information transfer chart (EXIT chart). And further more, Brink applied the EXIT chart to design LDPC codes, the iterative decoder can be viewed as wo separated decoders: the variable nodes decoder (VND) and check nodes decoder (CND). The code is optimized by performing a curve fitting on extrinsic information transfer charts of VND curve and CND curve. But in Brink’s design method, the check nodes degree is determined in advance, which decrease the freedom of degree, motivated by this observation, we propose a joint optimization taking into account the check nodes degree.
This paper is organized as follows: the design of LDPC codes using EXIT chart proposed by Brink is introduced in section II, the joint optimization of LDPC codes is illustrated in section III, and some simulation results are also given to demonstrate the advantage of the joint optimization. Section IV summarize the paper and gives the conclusion.

II. DESIGN OF LDPC CODES WITH EXIT CHART
Consider an LDPC code of length n and design rate R=k/n. Then in the graph of iterative decoding, there are n variable nodes and k-n check nodes, the jth variable node is involved in dvj parity checks, so its degree is denoted by dvj and the ith check node can check dcj variable nodes, so its degree is denoted by dcj. The distribution of dvj and dcj determines the structure and performance of LDPC code. And the iterative decoding process is performed between variable nodes and check nodes through edges connected them. From the view of EXIT chart, the sets of variable and check nodes are referred to as the variable-node decoder (VND) and check-node decoder (CND), respectively. Iterative decoding is performed by passing messages between the VND and CND. The decoder structure is shown in Fig. 1[4].

B. Decoding Threshold
Decoding threshold is an important parameter presenting the performance of LDPC with specific degree distribution. LDPC codes can’t be decoded correctly if the channel SNR (signal to noise ratio) is below the decoding threshold, no mater how many times of iteration we perform. And EXIT chart is a useful method to determine the decoding threshold.
From formula (1), we can see that the CND curve is only concerned with the degree of check nodes and doesn’t change with the SNR. But according to formula (2), the VND curve varies for different SNR, as shown in Figure 3, when the signal to noise ratio of 2 dB, VND curve is far away above the CND curve, then a successful decoding can be achieved after a certain number of iterations, which means the decoder is convergent, the iterative decoding process is shown in figure 3 (the dotted line), we can see that it needs approximately 10 iterations to achieve a correct decoding. Along with the decrease of SNR, VND curve gradually moves closer to the CND curve, when the SNR is 1 dB, the VND curve is still above the CND curve, then correct decoding will also be achieved but more iterations are required. Figure 5 illustrates the BER performance of rate 1/2 LDPC codes with length 504, 1008 and 2016 respectively. Table II lists average number of iterations needed in the simulation under different SNR, we set the maximum number of iterations as 100, we can see that when SNR is lower than the decoding threshold, the LDPC code cannot be decoded correctly even to the largest number of iterations, and with the increase in signal to noise ratio, it only needs a few number of iterations to achieve correct decoding. When SNR is 2 dB, only 10 average iterations are needed for the code (2016, 1008), which further verifies the accuracy of the EXIT.

IV. CONCLUSIONS
In this paper, we proposed a joint optimization method of LDPC codes using EXIT chart. The variable nodes degree and check nodes degree are considered together in the code design process. Compared with the non-joint design method, the joint optimization method can generate LDPC codes with nearer Shannon limit performance and can extended to arbitrary code rate design.
