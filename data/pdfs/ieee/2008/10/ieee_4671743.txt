ABSTRACT
Since Low-Density Parity-Check (LDPC) codes deliver excellent decoding performance, they are adopted in several recent communication standards like the IEEE 802.11n, IEEE 802.16e and DVB-S2. This raises the need for multi-standard, multi-mode decoder architectures. In this paper we propose to use the min-sum approximation together with a turbo-like decoding approach for decoding LDPC codes, and afterwards we perform a global trade-off analysis which enables the designer to choose the appropriate decoding algorithm.
Index Terms— LDPC, forward error correction, SDR, trade-off analysis, fixed-point refinement
1. INTRODUCTION
The trend of electronic devices to become an integral part in any thinkable application area raised the need for flexible solutions to cover a broad range of applications. For forward error correction (FEC) in general, the implementation approaches are typically very conservative. For the supported standards, individual rather fixed hardware accelerator blocks are deployed to perform the channel (de)coding. Due to the high throughput requirements of these blocks this was the only way to reach the design goals in the past. However, with the multiplication of coding schemes and options, there is a clear need for more flexible solutions, both to enable the support of future standards and for chip area reduction. This paper focuses on a sub-class of LDPC codes, namely architecture-aware (AA) ones, which are composed in such a way that they enable high-throughput, reduced-complexity decoder architectures [1–3]. Two recent communication standards, the IEEE 802.11n and 802.16e standard, are complying with this approach. A turbo-inspired decoding procedure has been introduced in [4] which has the advantage to significantly reduce the computation complexity. This approach is interesting for an application specific instruction set processor (ASIP) implementation [5] since the turbo-like decoding approach can efficiently be mapped to an instruction set architecture (ISA).

In this paper we show how the turbo-like decoding can also be used with a modified min-sum approximation, which further reduces the memory footprint significantly. Afterwards, a global trade-off analysis of the different decoding algorithms for LDPC codes, not only in terms of bit error rate (BER) performance, but also by taking the computational complexity, the fixed-point behavior and the achievable throughput into account, is performed. This is done having in mind a future ASIP implementation. We also propose a unified instruction set which allows efficient computation of the core kernels and which will build the basis for the complexity analysis.
In Section 2 the basics of LDPC decoding are presented. The trade-off analysis is then performed in Section 3 and the paper concludes with Section 4.
2. LDPC DECODING
2.1. Background
Decoding of LDPC codes is done in an iterative way and all computations are typically performed in the log-domain. The parity-check matrix H defining an LDPC code can be represented by a bipartite graph [1]. The graph contains two types of nodes: check nodes (CN) and bit nodes (BN), which correspond to the M rows and the N columns of H, respectively. Each CN represents a parity check equation, whereas each BN represents a coded bit. Denote the set of all CNs, directly passing probability messages to the BN with index i as C [i]. The degrees of CNs and BNs are defined by the number of edges which are connecting them with neighboring BNs or CNs and are denoted dcj and dvi , for the degree of CN j and BN i, respectively.
Instead of implementing the two-phase message passing algorithm, as summarized in [4], a check-node-centered processing scheme is considered. The reader is referred to [1–4] for further details about the basic algorithms.

2.2. Contribution

We propose to use the well known min-sum-based approximation (MSBA) [7] when implementing the turbo-like decoding algorithm. We will show in section 3, that this way of message computation has advantages compared to the MASTBA, depending on the design criteria.
In order to use the MSBA, the following adaptations to the TDMP algorithm are necessary: The function evaluated for the forward-/backward recursion and the output metric is computed by evaluating (9).

q(x, y) ≈ min(|x|, |y|) · sgn(x) · sgn(y).

(9)

Additionally, the boundary conditions need to be changed to Δα0 = Δβdcj = ∞.
Interestingly, the MSBA can further be optimized to reduce the memory footprint. The required CN message storage can be reduced by exploiting the fact that out of all CN LLRs of a single CN only two magnitudes are of interest, since only the smallest and the second smallest magnitude is used to produce LLRs for connected BNs. A more detailed discussion about the optimized MSBA message computation can be found at [6].

3.1. Fixed-point refinement
For hardware implementation, the fixed-point behavior of the introduced algorithms is of great interest. Simulations have shown, that for representation of LLR values of individual elements in the received codeword, a bitwidth of 6 bit, with 4 bit for the integer part and 2 bit for the fractional part, is sufficient. To identify the minimum bit width for the data path of the ISA, simulations have been performed for each decoding approach, as discussed next.
MSBA: In Fig. 1(a) the results of the fixed-point refinement for the MSBA are depicted. For the experiment, 6 bit have been used for representing received messages. The internal data path width has been reduced starting from 10 down to 7 bit. For the fractional part, two bit have shown to be sufficient. Intuitively, quantization means a degradation in BER performance, but the opposite effect can be observed here, where the quantized version of the MSBA delivers the same BER performance as the floating-point implementation with an signal-to-noise-ratio (SNR) reduced by half a dB. In fact, the MSBA benefits from the clipping performed to avoid overflows in the fixed-point implementation, where the magnitudes of overestimated probabilities are saturated. Similar effects have been observed in [9]. As seen in Fig. 1(a) the

algorithm delivers excellent results as long as the bit width is greater than 7.
MASTBA: In Fig. 1(b) the fixed-point experiments for the MASTBA are depicted. It was found that this approximation is more sensitive to quantization. The data path width has been reduced from 11 down to 9 bit. The fractual part was again two bit wide. The results show that the approximation delivers feasible decoding results with a minimum data path bitwidth of 10 bit. When further reducing the bitwidth, as seen in the plot, the algorithm does not converge to valid codewords anymore. Note that the quantized version of the MASTBA does not outperform its floating point implementation, as the MSBA does.
Optimized MSBA: The optimization of the message storage when implementing the MSBA has no impact on the decoding performance and convergence speed. Only the computation kernel gets a bit more complicated, since pack/unpack instructions need to be performed in order to store/extract individual messages, respectively. This reduces the achievable throughput.
The fixed-point implementation of the MSBA and the MASTBA are compared in Fig. 2(c). The MASTBA shows a BER performance degradation of around 0.1 dB compared to the MSBA. Also, in the low SNR region, the MSBA converges an average of around 9 iterations faster than the MASTBA, see Fig. 2(d).

3.2. Detailed operational complexity
To evaluate the detailed implementation complexity considering an ASIP implementation, we first developed an ISA appropriate for implementing the approximations.
As discussed in section 2, the MSBA and the MASTBA approximation share the same data path. Solely the core operations differ. Implementing both approximations enables the application to choose between them, depending on the environmental requirements, such as maximizing the throughput or increasing the decoding performance. An ASIP design methodology fits the high-throughput requirements and at the same time enables a flexible implementation of the different approximations and has therefore been chosen [5]. The ISA is depicted in Fig. 3. As introduced before, the processing

is organized in a check-node-centered way, where BNs are reduced to message storage and all computations take place in the CN processing unit. In order to increase the throughput, the decoding algorithms need to be parallelized. This can efficiently be done by implementing the processing unit in a Single-Instruction-Multiple-Data-based (SIMD) scheme, since all check equations within one subblock can be computed concurrently. Each SIMD slot then computes one check equation for each decoding round.
The corresponding instruction set is presented in Table 1 and a more detailed discussion about each instruction is given in the next paragraphs.
Memory access instructions: The instructions interfacing with the memory are vector instructions manipulating the SIMD word as one vector.

Core instructions: A special instruction has been implemented for each presented approximation, for the MASTBA the max instruction and for the MSBA the min+ instruction, respectively. This min+ instruction keeps track of the current sign and the current smallest magnitude, which needs to be found in the forward and backward recursion at each state. The cs operation was introduced as a special instruction for the optimized MSBA. It is used to find and keep track of the two smallest magnitudes when computing the forward recursion.
When implementing the SIMD based processing scheme, two implementation aspects are very sensitive. First, the minimum bitwidth of one SIMD slot in the data path, which results from the selection of the decoding approximation. Second, the number of SIMD slots utilized. Following, a few guidelines are given on how to choose a proper number.
Each LDPC code has its inherent number of concurrently computable check equations.

The interleaving operation needs to compensate for the cyclic permutation of the identity matrices between subsequent layers. When a BN is for example participating in the first check node equation in one layer and in the second check node equation in the subsequent layer, before writing back the results from the first computation, they need to be rotated by one position.
When not exploiting the inherent level of parallelism, the number of slots should always be a multiple of the subblock sizes of the codes, the system needs to support. For instance, when targeting the IEEE 802.11n standard, the codes are constructed of cyclic permutations of identity matrices with subblock sizes of 27 and multiples of it for different code lengths (e.g. 27,54,81). Then, interleaving also reduces to rotate operations. What still introduces additional overhead is the fact that the result from all computations of one subblock need to be available, before interleaving and writeback can take place. In case the number of SIMD slots do not match the subblock size or multiples of it, interleaving beyond SIMD word boundaries needs to be implemented, which incurs a significant overhead and should therefore be avoided.
3.2.1. Discussion
From the simulation results it was found that for the fixedpoint implementation of the MASTBA an overall bith width of 10 bit is necessary for internal message representation. The MSBA has shown to be less sensitive to quantization, only 8 bit are needed for message representation. This fact of course makes it more attractive for hardware implementations. The proposed optimized MSBA, where CN messages are stored in a packed way, reduces the CN memory footprint by around 30%, for the cost of a slightly more expensive CN computation kernel.

4. CONCLUSION
In this paper a new way of message computation based on the turbo-like decoding algorithm and the min-sum approximation, namely the min-sum-based approximation has been introduced and compared to existing approaches. The results from the fixed-point refinement, combined with the trade-off analysis enables a designer to choose the most appropriate approximation to be implemented on an ISA. By thoroughly choosing the number of SIMD slots, as discussed above, the

proposed approximations are suitable for high-throughput implementations.
