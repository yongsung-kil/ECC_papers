Construction of Short Block length IRA Codes
Based on Modified IPEG Algorithm
Junfeng Hu1, 2, Hailin Zhang1 1. State Key Lab of ISN, Xidian University, Xi’an, 710071, China
2. CETC No. 20 Research Institute, Xi’an 710068, China E-mail: hujunfeng_cetc@163.com, hlzhang@xidian.edu.cn

Abstract— Based on the particular structure of irregular repeataccumulate (IRA) codes, a weight function is given for codeword weight computing. Then, we propose a modification to the improved progressive-edge-growth (IPEG) algorithm. The modified IPEG algorithm gives degree-2 variable nodes more protection and tries to avoid low weight code-words corresponding to weight-one and weight-two encoder inputs. Simulation results show that the proposed algorithm improves the performance of short block length IRA codes at high signalto-noise ratio (SNR) region with no sacrifice in low-SNR performance.
Keywords- LDPC codes; irregular repeat-accumulate (IRA) codes; improved progressive-edge-growth (IPEG) algorithm, error floor
I. INTRODUCTION
The irregular repeat-accumulate (IRA) code [1], [2] is a popular class of irregular low-density parity-check (LDPC) code that not only can be encoded in linear time but also yields a very simple puncturing pattern for rate-compatible codes design [3], [4]. Short block length LDPC codes are of particular interest in delay-sensitive wireless communications. However, in iterative decoding, completely randomly generated short block length LDPC codes are prone to serious error floors.
The performance of the finite length LDPC code is directly related to the structure of its Tanner graph, as explained in [5], where the concepts of cycles, stopping sets, extrinsic message degree (EMD), approximate cycle EMD (ACE), and linearly dependent sets are introduced and clarified. Therefore, some designs aim at maximizing the girth of an LDPC code. This is the approach of progressive-edgegrowth (PEG) algorithm [6]. The improved progressive-edgegrowth (IPEG) algorithm [7] introduces the concept of ACE to PEG algorithm so that it can improve the performance of irregular LDPC codes at high signal-to-noise ratio (SNR) region due to larger minimum distance as well as improved trapping sets (near code words). Recently, several pseudorandom construction techniques for finite-length IRA codes design have been proposed in [2]-[4], [8].
In this paper, based on the particular structure of IRA codes, we propose a modification to the IPEG algorithm of [7]. The proposed algorithm improves the performance of short block length IRA codes at high SNR region with no sacrifice in low-SNR performance.

As pointed out in [2]-[4], a reasonable IRA code design algorithm should try to maximize the weight of the codeword corresponding to the weight-one and weight-two encoder input. That is, for wu = 2 and wu = 1, the distribution of the ones in H1 should try to maximize the minimum weight function value Fw (v) .

For IRA codes, as proposed in [8], the degree-2 variable nodes corresponding to the check bits only connect to two check nodes, which means degree-2 variable nodes have limited extrinsic information exchange with the rest of the bipartite graph compared to the variable nodes with higher degrees. We hence are motivated to give degree-2 variable nodes more protection.

III. MODIFICATION TO IPEG ALGORITHM

Based on the characteristics of IRA code that is given in section II, we propose a modification to the IPEG algorithm of [7] in this section.

As presented in [9], the generalized PEG algorithm consists of two independent procedures, “Check Nodes Shortest Paths Discovery” procedure and “Check Node Selection” procedure. The “Check Nodes Shortest Paths Discovery” procedure in our algorithm is the same as that in [7], and we focus on the “Check Node Selection” procedure
only. ĀCheck Node Selection” procedure classifies check nodes and selects one based on a suitably defined selection criterion. Selection procedures are sequences of selection criteria of decreasing priority. If multiple check nodes survive any selection criterion, the next criterion is applied on the surviving check nodes. Fig. 2 shows the performance comparison of the rate-1/2 codes in AWGN channel. It can be seen that, at high SNR values, the code constructed by modified IPEG algorithm outperform the corresponding IPEG code considerably. At lower SNR values, the codes perform more or less the same,

with modified IPEG code never performing worse. At high SNR region, both the modified IPEG code and the original IPEG code outperform the code proposed in IEEE 802.11n standard.
