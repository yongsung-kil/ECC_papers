Abstract
In this paper, we present an improvement on the Normalized Min-Sum (NMS) decoding algorithm for Low Density Parity Check codes. The modified NMS (M-NMS) algorithm introduces a more efficient adjustment for check node update computation. Taking the sub-minimum value among extrinsic information from the variable nodes into consideration, we make the correction factor varies with different iterations. Results show that, compared with standard Min-Sum (MS) and NMS decoding algorithms, the proposed method not only offers a better performance but also achieves a lower error floor in the high SNR region.
1. Introduction
Low Density Parity Check (LDPC) codes [1], rediscovered by [2] and [3], have recently attracted great interest due to the excellent performance and been adopted for several data communication applications.
To decode LDPC codes, the standard belief propagation (BP) algorithm achieves a good performance. Unfortunately, the hardware implementation of BP algorithm is limited by its complexity. So various work has been focusing on both improvement of decoding performance and reduction to complexity. The BP algorithm can be simplified using the so-called BP-based approximation [4] (also known as the Min-Sum [5]), but it incurs a degradation in decoding performance. This has led to the development of many variants of the MS algorithm that deliver near-optimum performance. One kind of the improvements is to modify the MS algorithm by employing a check node update that using a multiplied or additive correction factor which refer to the Normalized Min-Sum(NMS) or the Offset Min-Sum (OMS) decoding algorithm[6]. 4. Simulation results

Simulations have been conducted to compare the performance of the modified NMS (M-NMS) algorithm with various MS algorithms.
Figure 4 and 5 depict the bit error performances of the (250, 125) and (400,200) regular LDPC codes with ( dv , dc ) = (3, 6), respectively. The maximum number of
iterations is set to 50 and the optimum value α of NMS is fixed at 0.8, β of OMS is set to 0.19. For the (250,
125) code, the results show that at the average bit error rate below 10-4, the M-NMS algorithm outperforms the NMS algorithm by about 0.1dB. For the (400, 200) code, figure 5 shows that the proposed method not only offers a better performance but also achieves a lower error floor in the high SNR region .

5. Conclusion

In this paper, a modified Normalized Min-Sum (M-NMS) algorithm is proposed. For short regular LDPC codes, simulation results show that the improved algorithm outperforms the NMS algorithm and OMS algorithm. It can achieve performance gain with only a modest increase in computation complexity.
