LDPC block-coding schedule based on Queen-matrix

Luo Xiling, Zhang Jiayi, Zhang Jun
School of Electronics and Information Engineering, BUAA
Beijing, China luoxl@adcc.com.cn jiayi@ee.buaa.edu.cn buaazhangjun@vip.sina.com

Abstract—The general coding schedule of low density parity codes (LDPC) have the problems of slow coding speed and difficult implementation in hardware. Based on the irregular Queen-matrix LDPC construction algorithm derived from nqueens problem, this paper designed an optimized block-coding schedule according to construction codes’ dimension and memory’s bandwidth. The block-coding schedule can reduce the intermedial size of data storages and be propitious to hardware implementation by using data source and generating matrix̉s division. The experimental results show that comparing with the general coding schedule, the block-coding schedule can make coding speed be improved by 14.6㧑㨪36.1㧑 at the cost of a little increasing of bit error ratio (BER) because it arranges the submatrix multiplication in parallel prior to sum of them.
Keywords- Queen-matrix; block coding; LDPC
I. INTRODUCTION
Low density parity codes (LDPC) was first introduced by Gallager[1] in 1960. Because of the complexity of iterative decoding and the limitation of the technology of the time, LDPC has not been well noticed until 1995 when Mackay and Neal[2,3] rediscovered LDPC which has the as excellent performance as Turbo codes. So, it became the important element of air-craft communication, satellite formation and deep space detection which are the technical problems at domain of aeronautics and astronautics.
There are two steps to implement the encoding of LDPC. First, research the construction algorithms of parity-check matrix H ; second, research the encoding schedule according to the structure of H . This paper researched the construction algorithms based on queen-matrix coming from n-queens problem[4,5] and designed an optimized block coding schedule[6] dealing with the problems of slow coding speed and difficult implementation in hardware of the general coding schedule.
The n-queens problem[4] is to place n indistinguishable objects on an n × n grid so that no two objects are placed on the same row, the same column, or the same diagonal. Reference [6] designed a LDPC construction algorithm based on Queen-matrix derived from n-queens problem. This paper researched the characters of Queen-matrix and the construction algorithm of LDPC. The coding complexity of general coding schedule is direct ratio to the square of code length. Meanwhile, it is difficult to implement with hardware along with the increasing of code length. Reference [7, 8] presented a classic encoding schedule which can change parity matrix H to

approximate lower triangulations. It had proved that the coding complexity is possibly linear to the block length. This paper designed a block-coding schedule which has fast coding speed and easy implementation in hardware. The experimental results show that comparing with the general coding schedule, the block-coding schedule has a simpler structure of LDPC encoder and is easier to implement in hardware. The coding speed has improved by 14.6%~36.1% at the same time.

II. BLOCK-CODING SCHEDULE

A. Construct irregular Queen-matrix LDPC
Queen-matrix[6] is defined that any matrix element place
on an n × n square matrix at the GF(2) finite field so that no
two elements are placed on the same row, the same column, or the same diagonal. It has the excellent characters which are propitious to construct LDPC such as expressing with vectors, constructing by circulation and shift, orthogonality et al. Take one of five dimension queen-matrices as an example to explain these characters showed in Fig.1.

B. Block encoding algorithm
The coding complexity of the general coding schedule is direct ratio to the square of code length. Large code length of LDPC ( N >1000) is necessary in practical application so as to give the excellent performance of iterative decoding. But such large code length’s encoder is impossible. To be realized and therefore, it limited the application of LDPC largely. This paper designed a block encoding algorithm instead of the general coding schedule which divided the data matrices and the generate matrix by the dimension of sub-matrix according to the idea of constructing H by using sub-matrix in 2.1. It ensures that the realization of the LDPC encoder in hardware and improves the encoding speed.

It is showed that the block-coding schedule needs l ×u sub-matrices multiplications and two additions before getting the encoded codes. The key point of block-coding schedule is how to find out the sub-matrices’ dimension which means how
to find out the value of m,n,l,u,v . Three factors should be
considered. First, the dimension of queen-matrix when constructing LDPC; second, the bandwidth of multiplication in selected FPGA; third, the size of RAM and the data bus width of SDRAM. The block-coding schedule solved the problems of large memory size and large data throughput effectively by selecting the proper dimension of sub-matrices considering the three factors. It also ensures the encoder is realizable and improves the coding speed of the general coding schedule.

IV. CONCLUSIONS
This paper researched constructing algorithm of LDPC with queen-matrix form n-queen problem. Us-ing the idea of constructing parity matrix with divided sub-matrices, designed the block-coding schedule who divided the data matrix and the generate matrix by the dimension of construction matrix. The experimental results show that comparing with the general coding schedule with Mackay’s random codes, the blockcoding schedule can ensure the implementation of encoder in hardware device, make coding speed be improved by parallel idea instead of serial idea and reduce the memory size using queen-matrix constructing LDPC at the cost of a little increasing of bit error ratio.
