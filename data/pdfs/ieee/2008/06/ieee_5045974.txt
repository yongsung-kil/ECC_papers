Evaluation and Implementation of Quasi-Cyclic LDPC Codes for IEEE802.11n Based MIMO-OFDM System
Xiaonan SHI, Shingo Yoshizawa, and Yoshikazu Miyanaga Graduate School of Information Science and Technology, Hokkaido University, Sapporo 060-0814, Japan

Abstract— In this paper, the evaluation and the implementation of quasi-cyclic LDPC (QC-LDPC) codes in MIMO-OFDM systems are presented. QC-LDPC has comparative error correcting performance with random irregular LDPC and considerably reduces complexity. We evaluate communication performance of the QC-LDPC coded system based on the IEEE802.11n standard in a 4x4 MIMO scheme. In the LSI implementation, we have designed both the LDPC encoder and decoder for the QC-LDPC system, where an iterative parallel architecture is adopted to deal with such requirements of high-throughput data rates. The dsigned encoder and decoder have 61K and 850K in logic gates, respectively. Power consumption of encoder is 5.3mW, where decoder leading 199.6mW in power dissipation.
I. INTRODUCTION
Recently, LDPC codes proposed by Gallager have attracted much attention as powerful error correcting codes which can achieve the near Shannon limit performance with their capability [1]. IEEE802.11n standard proposed by the Enhanced Wireless Consortium (EWC) has effective bit rate of higher than 100Mbps [5]. MIMO-OFDM used over the standard allows a quantum jump from using the IEEE802.11a standard [3].
LDPC coded MIMO-OFDM system based on IEEE802.11n standard has been known to achieve good performance gain in data transmission at high speed compared to conventional convolution codes which is chiefly used for correction of errors. It is advantageous in effectively correcting the errors caused by noises occurring in the wireless channel to enhance the reliability of data transmission. The LDPC coded system also has an advantage in that it can perform decoding by using an iterative decoding algorithm based on the sum-product algorithm (SPA).
However, most methods for designing LDPC codes are based on random construction techniques; the lack of structure implied by this randomness presents serious disadvantages in terms of storing and accessing a large parity check matrix, encoding data, and analyzing code performance. In order to overcome these problems, QC-LDPC codes are designed by Tanner which are good codes with shorter block lengths and be adopted to reduce encoding complexity [6]. Moreover, the encoding facility of QC-LDPC codes can be combined with efficient LDPC decoding techniques, such as SPA [7]. Therefore, QC-LDPC is suitable for hardware implementation.

The rest of this paper is organized as follows. In section II, the construction of QC-LDPC code is illustrated. Section III gives the detail of QC-LDPC based MIMO-OFDM system. Simulation results also present in this section. Finally, Section IV describes a designing of LDPC encoder and decoder.

II. QC-LDPC CONSTRUCTION
Originally, LDPC code is a linear block code, if we set the m × n sparse parity check matrix H, with coding rate R = (n − m)/n. When encoding, a codeword c is carried out in the classic way by multiplying information bits m with the generator matrix G, follows by Eq. (1).

c = m d2(m × G)

The generator matrix is unknown from the code design. Generally, it is obtained by Gauss-elimination from equivalent parity check matrix Heq. Hence, the generator matrix G is not sparse. It happens on the high encoding complexity of a matrix quadratic for a large block size in the code length O(n2).
For the case of high encoding complexity of LDPC code, it is preferable that most of the parity bits are computable using sparse operation, leading to approximately linear encoding complexity.

A. Encoding Algorithm

QC-LDPC code is a kind of LDPC codes whose parity check matrices consist of the blocks of circularly shifted identity matrices [6]. A codeword is directly generated by a parity check matrix. Thus, QC-LDPC code is convenient to choose the parity check matrix H as an array of shifting fact (q × q) circulant permutation sub matrices, where some of the sub matrices are given an identity matrix or a zero matrix.
Figure 1 shows a QC-LDPC code defined by the parity check matrix H, which is decomposed into two parts for simple operation: one is the information bits part and the other is the parity check bits part. Position error of the decoding performance cannot avoid though it can be said that weight is optimum, since there is deflection in the position of the non-zero element compared with an ideal irregular parity check matrix. III. QC-LDPC BASED MIMO-OFDM SYSTEM
A. MIMO-OFDM system scheme
OFDM is one of the most promising physical layer technologies for high data rate wireless communications due to its robustness to frequency selective fading, high spectral efficiency, and low computational complexity. OFDM can be used in conjunction with a MIMO transceiver to increase the diversity gain the system capacity by exploiting spatial domain. MIMO-OFDM is considered as a key technology in emerging high-data rate systems such as IEEE 802.11n. IEEE802.11n features LDPC codes as optional forward error correction (FEC) scheme for high throughput applications. A 4x4 MIMO system based on IEEE802.11n used in this paper is shown in Figure 4. C. Simulation result
Transmission performance has been evaluated in BER performance. Figure 5 shows the BER comparison among the QC-LDPC coded system, the random irregular LDPC coded system, and the convolutional Viterbi decoding system on 16QAM and 64-QAM modulation. The coding rates are 1/2, 3/4, 5/6 respectively. When coding rate is 1/2, LDPC coded systems display best BER performance. Though systems analyze with different coding rates, the QC-LDPC coded system almost has the same performance with the random irregular LDPC coded system by evaluating BER. Furthermore, the LDPC coded systems show better performance than the convolutional Viterbi decoding system. Even 64-QAM modulation, a 5/6 coding rate is used in the system, LDPC codes demonstrate better performance than convolutional codes. Though QCLDPC codes performed not as well as random irregular LDPC codes, computational complexity is reduced drastically by QCLDPC codes (Shown in Table II ).
IV. IMPLEMENTATION OF LDPC ENCODER AND DECODER
A. Encoder architecture
We implemented a efficient QC-LDPC encoder by adopting the encoding algorithm given in Section II. An encoding circuit for QC-LDPC in Figure 6 can be devised based on the construction of parity check matrix H. In parallelism of each XOR processing, the encoding speed can be increased. Encoder for QC-LDPC code with various block lengths and coding rates are implemented using Verilog-HDL and synthesized result with a example code length of 960 in 100MHz clock frequency is indicated in Table III. This implementation requires row weight of 4 and column weight of 8. The data bits

vector is first muliplied by information part of parity check matrix H to obtain 1 and 2. Next, the multiplication is implemented by data vector and obtained vector in order to get 3 and 4.
B. Decoder architecture
SPA algorithm is used for our designed decoder. When an iterative parallel procedure is adopted in the decoding processing, a decoder making a calculation for all the rows and columns could be implemented by full parallel processing blocks [9]. However, a large block length in LDPC codes makes the full-parallel architecture unrealistic because of a huge circuit size. Therefore, we apply a partial parallel architecture that arrays processing blocks with the appropriate number of the parallel blocks.
Since the operation of tentative decision is similar with the column operation in decoding processing, a block having both column operation and tentative decicion is designed. The overall block diagram of decoder is given in Figure 7. All the input data needs to be stored in ”Memory Array”. If and only if all the data is gathered, the pre-processing can be pipelined. When the decoding processing begins, the data is input into BFU that calculates the bit node unit and the tentative decision unit as described in Figure 3. Figure 8 shows the architecture of internal processing of BFU. Each of the output is stored in ”Beta Memory Array”, and becomes an input of the CFU module afterwards. The other output data of Cn is sent to the parity check module. CFU is a block that processes the check node operation. The internal processing done in CFU is as shown in Figure 9. V. CONCLUSION
We have presented the QC-LDPC coded system in MIMOOFDM wireless communications. QC-LDPC coding has an efficient encoding quality even in reducing complexity. The communication performance of QC-LDPC coded system has been evaluated by the simulation on the IEEE802.11n standard. The simulation results have indicated that the QC-LDPC coding has comparative performance with the random irregular LDPC coding. Our hardware design benifited from QC-LDPC code in order to gain in flexibility and compatibility. In the LSI implementation, the construction enable us to design a low

complexity encoder with relatively small logic gate count. The
LDPC decoder executing partial parallel processing has been
designed for decoding of QC-LDPC codes. The decoder circuit
has 852,000 in logic gated and consumes about 200 mW in
power dissipation on a 90-nm CMOS process. The circuit area
and the power dissipation are expected to be improved by use
of more sophisticated techniques in further research.
