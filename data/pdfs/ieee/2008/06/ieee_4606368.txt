High-speed Design of Adaptive LDPC Codes for Wireless Networks
Zhiyong He and Sébastien Roy Department of Electrical and Computer Engineering, Laval University, Quebec City, Quebec, Canada, G1K 7P4

Abstract— In this paper, we propose a class of adaptive lowdensity parity-check (LDPC) codes for reliable data transmission in wireless networks, i.e. the code rate can be adapted according to channel conditions to maximize the total capacity. Constructed from shifted identity matrices, the first advantage of the proposed codes is that these codes are particularly well-suitable for the high-speed implementation of parallel encoders and parallel decoders. Since a unique mother parity check matrix is used to construct LDPC codes with several code rates, the second great advantage of the proposed codes is that a single universal encoder (decoder) is adequate to encode (decode) multi-rate codes, which makes it possible to efficiently implement multi-rate LDPC codes in a subscriber station. The implementation results into field programmable gate array (FPGA) devices indicate that a universal parallel encoder for LDPC codes with 9 code rates is capable of reaching a throughput above 3.6 Gigabit per second by using a clock frequency of 300 MHz and consuming only 1% of the total resources in a typical FPGA device.

I. INTRODUCTION
Low-density parity-check (LDPC) codes introduced by Gallager [1] have recently received a lot of interest for reliable high speed communication applications, because of their capacity-achieving performance and the existence of effective decoding schemes (e.g. [2][3]). Several standards regarding wireless communication have been recently proposed which suggested the incorporation of LDPC codes (e.g. [4] [5]). By employing multi-level modulation and multi-rate LDPC codes, adaptive modulation and coding can maximize the total channel capacity and provide proper service and quality of service for each user according to channel conditions and transmit power.

An LDPC code with a code length of n and a code rate R is commonly represented by an m×n parity-check matrix, where m is the number of rows and n is the number of columns. The code rate R of an LDPC code is determined by the numbers of rows and columns, i.e.

R = (1 − m / n) .

(1)

Thus, there are two alternative approaches to obtaining LDPC codes with multiple code rates. One is the puncturing or the extending approach which changes the number n of columns, while the other one is the row-removing approach which

changes the number m of rows.
The puncturing (extending) approach which punctures (add) some bits has been used in the design of rate-compatible LDPC codes for automatic repeat request (ARQ) protocol. The main advantage of the puncturing (extending) approach is its low encoding complexity, since a single encoder is adequate to encode the codes with different code rates. The main disadvantage is the degradation of its bit-error-rate (BER) performance, because the puncturing approach decreases the code length n. Since the punctured bits are not transmitted, these punctured bits have to be processed as bit ‘0’ in the decoder of a receiver, though some of these bits may be bit ‘1’. The puncturing approach also increases decoding complexity. On the contrary, the second approach which changes the number of rows, the so-called row-removing approach, does not change the code length and thus guarantees a good performance. However, the critical issue regarding the row-removing approach is its associated encoding complexity.
In this paper, we report our research results regarding the design of multi-rate LDPC codes using a row-removing approach. To reduce encoding complexity, we propose a class of linearly encodable LDPC codes by designing a mother parity-check matrix in a triangular plus dual-diagonal form. The code rate is adapted by removing the uppermost rows of the mother matrix. We will show that the row-removing approach neither increases the encoding complexity of the proposed LDPC codes nor degrades their performances. Since a single parity check matrix is used to construct a family of codes providing different rates, the main advantage of the proposed approach is that a “universal” encoder (decoder) is sufficient to handle the encoding (decoding) of these codes. Another important advantage is that all effective code rates have the same code length.

B. Encoding of daughter codes
For a daughter code with a code rate of 13/24, (x1, …, x13) represents the systematic part and (x14, …, x24) represents the parity parts. Since the computation in (5) is not required, the encoder starts from (6). Thus, the universal encoder works as follows.
1) To encode the mother code with a code rate of 12/24, i.e. 1/2, the XOR processors used for matrix-vector multiplication start from the first block-row of the mother matrix shown in Fig. 1.
2) To encode a daughter code with a code rate of (12+1)/24 = 13/24, the XOR processors start from the second block-row of the mother matrix.
3) To encode a doughter code with a code rate of (12+i)/24, the XOR processors start from the (i+1)-th block-row of the mother matrix.
The decoding process of LDPC codes with adaptive rates is conceptually similar to the encoding process. For the mother code with a rate of 12/24, a joint row-column processor based on the sum-product algorithm ([6, 7]) decodes the code from the last block-row to the first block-row. On the other hand, the decoder starts at the last block-row and stops at the ith
block-row for a daughter code with a rate of (12 + i) / 24 , where 1 ≤ i ≤ 8 .

V. CONCLUSION
This paper has studied high-speed design of LDPC codes with adaptive code rates for wireless networks. Since their parity check matrices are made of identity matrices or null matrices, highly-parallel architecture are feasible to increase the speeds of encoder and decoder. Characterized by a parity check matrix in a triangular plus dual-diagonal form, these structured LDPC codes with multiple code rates can be encoded in linear time using a universal encoder. The FPGA implementation results indicate that a 6-parallel encoder for the LDPC code with a code length of 2304 and code rates ranging from 1/2 to 5/6 attains a throughput of over 3.6 Gbps by consuming 1% resources in a typical FPGA device and using a clock frequency of 300 MHz.

B. Performances of adaptive LDPC codes
The designed code has been evaluated by performing simulations with binary phase-shift keying (BPSK) modulation in an additive white Gaussian noise (AWGN) channel. The iterative sum-product algorithm was used for decoding. The iterative decoder stops when either a valid codeword is found or 50 decoding iterations are reached.
