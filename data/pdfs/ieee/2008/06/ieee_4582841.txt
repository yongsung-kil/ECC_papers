TPCSPC Decoding over Coded
Partial-Response Channels
Xiaoxin Zou, Zhiliang Qin, and Kui Cai Data Storage Institute, Singapore, 117608 E-mail: {zou_xiaoxin; qin_zhiliang; cai_kui}@dsi.a-star.edu.sg

Abstract—In this paper, we consider turbo product codes with single-parity-check component codes as an low-complexity coding scheme for magnetic recording systems. Based on the observation that TPCSPC can be viewed as a special type of regular LDPC codes, three message-passing schedules of the sum-product algorithm (SPA) are studied. It is shown that the standard row-column decoding algorithm in the literature is essentially the same as the serial check-updating schedule. By viewing the code graph from a different perspective, we propose a serial bit-updating schedule that uses the MacLaurin Series expansion to approximate the computationally intensive check operation. The proposed algorithm converges faster than the parallel decoding schedule and can be implemented efficiently in hardware by using high-speed adders, comparators, and shift registers.
Index Terms—Single-parity-check (SPC) code, sum-product algorithm (SPA), turbo product codes (TPC)
I. INTRODUCTION
Recently, iterative decoding over coded partial-response (PR) channels (i.e., turbo equalization) has received considerable attention. In particular, the observation that a PR channel can be viewed as a rate-1 convolutional code leads to the natural format of a serially concatenated system where the channel is considered as an inner code and a low-density parity-check (LDPC) code or a punctured turbo code acts as an outer code. In [1]-[3], it is shown that an iterative receiver based on the BCJR channel detection algorithm [4] typically achieves a coding gain of 4-5 dB over uncoded systems.
Due to its high decoding complexity, turbo codes seem not to be a viable solution for high-density magnetic recording systems. On the other hand, LDPC codes have attracted much interest recently due to the low complexity of the sum-product algorithm (SPA) [5], [6]. However, with the exception of LDPC codes with quasi-cyclic nature, LDPC encoding generally has a quadratic complexity in terms of codeword length and also requires a large amount of memory to store a dense generator matrix. In addition, the LDPC decoder itself also requires large memory space to store all log-likelihood ratio (LLR) messages exchanged between variable nodes and check nodes. Reading data from these memory units repeatedly can be a time-consuming process especially for LDPC codes whose parity-check matrices have redundant checks or large row/column weights [7].

In [8]-[11], turbo product codes with single parity-check codes component (TPCSPC) codes were proposed as a promising candidate for magnetic recording systems. TPCSPC possess many attractive properties such as high code-rates, linear encoding/decoding complexity, and a highly parallelizable encoding/decoding structure. In spite of the fact that TPCSPC are intrinsically weak codes over additive white Gaussian noise (AWGN) channels, they were shown to provide BER performance comparable to that of LDPC codes when transmitted over precoded PR channels. Conventionally, TPCSPC was decoded by using an iterative algorithm [11] that first decodes SPC component codes in each dimension and then exchanges extrinsic information between two decoders.
In this paper, we view TPCSPC as a special type of regular LDPC codes with column weight 2 and investigate several SPA decoding schedules based on its code graph. It is shown that the row-column decoding algorithm in [8]-[11] is essentially equivalent to the serial-check-updating schedule where each check node is processed sequentially. By viewing the graph from a different perspective, we develop a serial bit-updating decoding algorithm, where all check-to-bit messages arriving at a bit node are processed simultaneously and the refined information is then used to process other bit nodes. In addition, we propose to use the MacLaurin series approximation [12] to reduce the computational complexity of the check operation (i.e., the “tanh-rule”). In contrast to the simplification based on the Jacobian logarithmic function that needs time-consuming table lookup [13], the proposed algorithm can be implemented efficiently by using high-speed adders and comparators.
This paper is organized as follows. In Section II, the system description is given. In Section III, various TPCSPC decoding algorithms are presented. In Section IV, simulation results over AWGN and MEEPR4 channels are presented, respectively. Finally, the conclusion is drawn in Section V.

III. TPCSPC DECODING
Of particular interest to magnetic recording systems are coding schemes with low encoding/decoding complexities. Since TPCSPC encoding involves adding a single parity-bit to each row and column, it is extremely simple and does not need to store explicitly a dense generator matrix as for most LDPC codes. Traditionally, TPCSPC is viewed as a serial concatenation of SPC codes separated by a block interleaver. In [11], a row-column decoding algorithm for TPCSPC was proposed, which uses the maximum a posterior (MAP) algorithm to decode the component code in each dimension and then iterates soft information in between.
Since each row and column of TPCSPC satisfies a check, we may also view it as a type of regular LDPC codes with column

weight 2. For a (t+1,t)2 TPCSPC, each bit node in the graph has a degree of 2 and each check node has a degree of t+1. For example, Fig. 2 shows the Tanner Graph of a (3,2)2 TPCSPC

First, saving results in a lookup table would inevitably introduce quantization errors caused by the truncation of the table input. Another problem is that multiple lookup tables are required for a wide range of signal-to-noise ratios (SNR) so as to minimize performance loss, thus increasing implementation costs. Moreover, accessing these tables constantly in the decoding process can be time-consuming and hence is not desirable. On the other hand, neglecting the correction factor completely (i.e., the Min-Sum algorithm [13]) normally results in a performance loss of 0.2-1 dB.

This method was also used in [12] to improve the performance of the max-log-map algorithm for turbo decoding and turbo equalization schemes, respectively. Clearly, (11) can be efficiently implemented by using high-speed comparators, adders and a shift register, which yields a significant saving in implementation costs.

IV. SIMULATION RESULTS
To be applicable to magnetic recording systems, we are interested in a rate-0.89 2-dimensional TPCSPC code, which are formed based on (17, 16) SPC component codes. We combine 16 (17, 16)2 TPCSPC codewords to form an effective data block size of 4K, which is compatible with current hard disk system configurations. In all simulations, two decoding iterations (i.e., I=2) are performed inside TPCSPC decoders.
We first consider the TPCSPC alone transmitted over AWGN channels. Fig. 3 shows the BER performance of three decoding schedules presented in the paper, i.e., the serial-check-updating schedule that is equivalent to the standard row-column decoding algorithm in [8]-[11], the fully parallel schedule, and the serial-bit-updating schedule. It is shown that serial schedules provides similar gains and converge faster than the parallel schedule. A performance gain of 1 dB is achieved at the BER of 10-4 by serializing the fully parallel decoding. In Fig. 3, we also simulate the serial-bit-updating schedule that uses the MacLaurin Series to approximate the check operation, which is shown to have negligible performance loss as compared with the exact implementation.
Next, we simulate the TPCSPC serially concatenated with a precoder with generator polynomial g(d)=1/(1+d) as an inner code over AWGN channels. The precoder is used in the system so as to provide an interleaving gain and improve the overall distance spectrum by mapping low-weight error-events to high-weight error-events. The precoder is also decoded by using the sum-product algorithm, which is shown in [10] to be equivalent to the BCJR algorithm. In the simulation, 15 outer iterations are used between the inner decoder and the outer TPCSPC decoder. For clarity purposes, Fig. 4 shows BER curves obtained at the 15th outer iteration. It is shown that both serial decoding schedules provide similar performance when concatenated with the precoder and achieve a gain of 0.5 dB over the fully parallel schedule at the BER of 10-5. To reduce the complexity of the check operation, we also simulate in Fig. 4 the serial-bit-updating schedule based on the conventional min-sum approximation [13] and the proposed MacLaurin Series approximation, respectively. The Min-Sum algorithm is shown to have a performance loss of 0.3 dB at the BER of 10-5 compared with the exact implementation. The approximation based on the MacLaurin Series, however, has a minimal performance loss of 0.1 dB through the simulated SNR range. Fig. 4 shows that, compared with the TPCSPC alone over AWGN channels, using a precoder can significantly improve BER performance for moderate-to-high SNR and achieve a gain of 4.2 dB at the BER of 10-5.
In Fig. 5, we present the performance of turbo equalization schemes over TPCSPC coded ideal PR channels. The precoder used in the system has a generator polynomial of g(d)=1/(1+d), while the ME2PR4 channel polynomial is given by h={5,4,-3,-4,-2}. The turbo equalizer consists of a soft-output detector based on the BCJR algorithm for the precoded ME2PR4 channel and a SPA-based TPCSPC decoder. In the simulation, we perform 15 outer iterations between the BCJR channel detector and the TPCSPC decoder, each of which involves 2 inner iterations inside the TPCSPC decoder. Fig. 5

shows that the serial check and serial bit schedules achieve similar performance and both outperform the fully parallel schedule by 0.5 dB at the BER of 10-5. The proposed implementation based on the MacLaurin Series approximation suffers from a performance loss of only 0.1 dB at the BER of 10-5 and performs better than the conventional min-sum operation [13] by 0.2 dB. For comparison purposes, we also include in Fig. 5 the performance of a length-4096 rate-8/9 random LDPC code with column weight 3 transmitted over the ME2PR4 channel. Four decoding iterations are used inside the LDPC decoder per outer iteration. To achieve better BER performance, no precoder is used over the LDPC coded channel (actually a performance loss of 1 dB at the BER of 10-5 is observed when the precoder is used.). For this channel, it is shown that the TPCSPC code based on the serial-bit updating schedule and the MacLaurin series approximation outperforms the LDPC code by 0.2 dB. Hence, considering its low encoding/decoding complexities, the TPCSPC code may be viewed as a viable channel coding scheme for magnetic recording channels.
V. CONCLUSION
In this paper, we consider the TPCSPC code as a special type of regular LDPC codes with column weight 2 and study several decoding algorithms that differ mainly in message-updating schedules. It is shown that the proposed serial-bit-updating algorithm based on the MacLaurin Series approximation converges faster than the fully parallel schedule and can be implemented efficiently by using high-speed adders and comparators.
