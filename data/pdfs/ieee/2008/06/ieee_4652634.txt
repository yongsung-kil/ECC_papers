Abstract
In this paper we examine the use of low-density paritycheck (LDPC) codes in binary asymmetric channels. The problem is interesting since in some types of binary input fading multiple access channels (MAC) without channel state information (CSI), which use the successive decoding scheme, some of the users may experience asymmetric channels. It is well known that the successive decoder is a set of single user decoders and since there is no CSI at the receiver, the corresponding single user channels may be asymmetric in general. In that case the rate of interest of the user with the asymmetric channel is achieved by an unbalanced input distribution of the binary input. We are interested in constructing LDPC codes for these channels that approach the desired rate tuple in the capacity region. A convenient way of making the output distribution unbalanced is by introducing a mapper at the output of the encoder. Here we explain the method of mapping and its effect on the iterative decoding and derive closed form expressions for the upper bound of the probability of error.

In our case however, although the mutual information for uniform input is almost the same as the capacity of the Z−channel, we have a multi-access channel and we need an unbalanced input not only because of the user that uses that code, but also because of the other user(s). A method to design an unbalanced code was suggested by Gallager in [11]. He suggests putting a mapper at the output of the encoder. The mapper takes the encoder output and maps it into another shorter sequence, making the input distribution non-uniform. However, the mapping function is not one-toone and therefore, it is not an invertible function. Thus, the decoding becomes more challenging. The parity check relations may be eventually used to decode and de-map at the same time. Methods used today, that allow to design good ensembles include density evolution [18] and EXIT (EXtrinsic Information Transfer) charts [22]. In [9], the authors

design capacity approaching LDPC codes for a multiple ac-
cess channel in which the input is binary and the additive
noise is Gaussian. They assume joint instead of successive
decoding. Our problem is different since we have reduced it
to a sequence of single user, possibly asymmetric channels.
Therefore, we need to design capacity approaching LDPC
codes for these single user channels. Consider the channel Y = X1 ⊕ A2X2, with α =
Pr{A2 = 1} = 0.615. Assume that we want to design codes that approach the rate pair R1 = R2, which is on the boundary of the capacity region (Fig. 1). We decompose this multi-access channel into two channels: X1 → Y that is a BSC (considering user 2 as noise) and X2 → Y |X1 that is a Z−channel. Since the channel X1 → Y is a BSC with crossover probability αp2, then p∗1 = 0.5, and the LDPC code for user 1 is constructed as for that BSC. By solving the equation R1 = 1 − h(αp2) = h(αp2) − p2h(α) = R2, we obtain that p∗2 = 0.251011 ≈ 0.25 < 0.5. In that case, R1 = R2 = 0.3793 bits (Fig. 1). One way to design an LDPC code for user 2, is to perform a mapping after the channel encoder, such that p2 = 0.25. In other words, in the codebook of user 2 there should be 3/4 of zeros and 1/4 of ones. One possible mapping is the following 00 → 0, 01 → 0, 10 → 0, and 11 → 1. Note that we are not interested in achieving the capacity of the Z−channel, but the mutual information h(p∗2α) − p∗2h(α). This follows from the fact that this is a multiple-access channel, i.e. rates of
both users depend on p2. In other words p2 has to be optimised with respect to the rates of both users. Note that the
de-mapping is done within the iterative decoder. In this example, we choose α such that we get p2∗ = 1/4, in order to explain the method on a simple mapping scheme. It could
be also generalized for more complex schemes.
The paper is organized as follows. First in Section 2 we
describe the structure of the iterative decoder when map-
ping is used and then in Section 3 we specify the sequence
in the message pasing algorithm. In Section 4 we analyze
the fractions of messages and bound the probability of error
during the iterations, and in Section 5 we show these results
for a certain family of codes. Section 6 concludes the paper.

4. Fractions of Sure Messages

In this section we are interested in the average probability of error as a function of the number of iterations. For simplicity we do an asymptotic analysis, as proposed in [23]. The assumption is that we have an infinite codeword, i.e. an infinite graph. Therefore, the messages that

are sent through the graph are i.i.d. random variables and it is enough to analyse one of them. In order to get the results in a closed form we compute the number of sure and unsure messages going through the graph. We call as sure message that message that determines the corresponding bit with probability one. Since we consider LLR messages, the LLR of a sure message takes value +∞ or −∞. Note that once the message becomes a sure message at some iteration, it remains sure in the following iterations. All the other messages will be called unsure. Since at the end the decision is taken upon the sign of the messages, calculating the fraction of sure messages is a pessimistic calculation of the probability of error.
At the channel output we have a fraction p1 of sure bits that are one and pQ = 1 − p1 unsure bits. There are no sure zeros at the beginning, because of the Z−channel. If the code is linear, it would be sufficient to consider all-zero codeword. In our case, because of the mapper the fractions depend on the transmitted codeword xi, i = 1, ..., 2nR. Hence, we are going to examine a typical codeword. Denote the fraction of ones in any codeword by β(i) = weight(xi)/n, where n is the length of the codeword. Obviously, β ∈ [0, 1], and if there exists at least one parity check of odd degree, then β ∈ [0, 1), since in that case the “all-one” word is not a codeword. On the other hand, the linearity of the code implies that the “all-zero” word is always a codeword. From the theory of LDPC codes with fixed rate, it is known that if n → ∞, then β converges to 1/2, [10, 17]. Since in our codewords the initial probability of ones is 1/2, this implies that for n very large, the number of codewords that have weight less than n/2 will be exponentially small. In the following text we derive all expressions for general β, although we found that β = 1/2 with high probability.
After the mapping vi = x2i−1x2i, the average weight of the sequence at the mapper output will be β2n/2 and at the channel output, αβ2n/2. Thus, for any codeword at the output of the channel the fraction of ones will be αβ2 and the fraction of zeros 1 − αβ2. That is, the mapper and the channel behave like the equivalent channel shown in Fig. 7. Note that the one at the output can be uniquely obtained from 11 at the input. Consider a code from the family λ = {λ2, λ3, λ4, λ5, λ9, λ11, λ17, λ33, λ35, λ99}={0.319949, 0.131809, 0.0223035, 0.129783, 0.0711008, 0.042702, 0.0792232, 0.0927079, 0.00366911, 0.106753}, ρ = {ρ4, ρ5} = {0.15, 0.85}, the rate is 0.19 and we get the fractions shown in Fig. 8. There we see how the fractions of sure bits evolve through the iterations. At the beginning, there are no sure zeros, there are α/4 sure ones and the rest 1 − α/4 are unsure messages. After sufficient number of iterations the fractions of ones and zeros approach 1/2 and the fraction of unsure messages vanishes. Since the threshold erasure parameter of this code if we use a BEC is ε∗ = 0.808565, we find an upper bound for the parameter α, that is 4(1 − ε∗) = 0.76574. This is true since the rate of erased bits is ε = 1 − α/4. Therefore, having a Z-channel with α > 0.76574 will guarantee a successful decoding. Unfortunately, the rate of this code is smaller than the desired mutual information on the Z−channel with a given α, that is h(p2α) − p2h(α). In Fig. 9 we present the lower bound for the rates achieved with these codes. This bound is not close to the desired rate. Our code performs in between these two lines. To determine the right performance of this code a numerical simulation is needed.

6. Conclusions
Successive decoder for multiple access channels without channel state information breaks up into a set of single user decoders that are matched to some single user channels. In general these channels are asymmetric and have a non-uniform optimising input distribution.
In this paper we proposed to concatenate a linear LDPC code with a non-linear mapper, to create a coding scheme that provides the desired non-uniform distribution at the channel input. We investigated how the iterative decoder can be implemented, derived bounds on error probability and discussed the density evolution of the messages in the graph.
Future research includes examining more sophisticated methods of mapping, while keeping the structure of the decoder as simple as possible.
