Low-complexity Real-time LDPC Encoder Design for CMMB

Wang Peng, Chen Yong-en Communication Software & ASIC Design Center, Tongji University
Shanghai, 200092, China E-mail: pengpwn@hotmail.com

Abstract
Based on modified LU decomposition theory with pivoting of sparse matrix, a low-complexity real-time LDPC encoder for CMMB was presented, which can support 2 different code rate (1/2 and 3/4). Multistaged pipeline and Pingpong buffer architectures were used to improve throughput. An efficient memory organization for storing sparse matrices was also presented. Whole design was synthesized and routed on Altera Stratix II EP2S90. Highest frequency achieved over 200 MHz, with pure encoding rate 32.44 Mbps (1/2 code rate) and 67.16 Mbps (3/4 code rate). Beside, fully-parameterized LDPC encoder can support arbitrary H matrix LDPC code with only necessary initialized data of memory blocks.
1. Introduction
LDPC code (Low Density Parity Check Code) was first introduced by R. G. Gallager[1] in 1962, however, it was overlooked for several decades because of high-complexity of encoding and decoding. In 1996, D. MacKay and R. M. Neal rediscovered its powerful performance as Error Correction Coding (ECC), [2]. For instance, for code rate = 1/2 and codeword length of 107, bit error rate of 10-6 can be achieved with threshold of 0.0045dB Shannon limit over AWGN channel [3], which shows the best performance as ECC up to now.
Presently LDPC code has been widely used on magnetic record, wireless communication and other zones. For instance, LDPC code was used as channel coding in DVB-S2, IEEE 802.11n/16e, and DMB-T, etc. In Chinese Mobile Multimedia Broadcasting standard (CMMB), LDPC code was also used as Forward Error Correction Code combined with RS code[4].
Despite of better performance compared to others codes especial Turbo codes. LDPC codes have large encoding complexity, which constitutes a main obstacle preventing its hardware implementation in modern communications.

Although parity-check matrix H of LDPC code is always very sparse, its corresponding generation matrix G is mostly dense, meaning quadratic complexity to code length with direct generation matrix encoding, which is nearly infeasible for hardware implementation. So currently widely-used LDPC codes are mainly QC-LDPC codes or its evolve version, whose parity-check matrix shows block-circular pattern, so that can be encoded with lower complexity[5].
LDPC code in CMMB is a kind of HighStructured (HS) code, but not QC-LDPC. Approach for QC-LDPC encoding can’t be used for CMMB. In this paper, based on LU decomposition of H matrix, a low-complexity LDPC encoder for CMMB was designed. To improve encoding efficiency, heuristic search strategy is used for LU decomposition with partial pivoting, leading to the lower-triangular matrix L and upper-triangular matrix U making up the LU decomposition as sparse as possible. So that LDPC encoder can be realized with very low complexity. Whole design was synthesized and routed on Stratix II EP2S90 of Altera Corporation. Highest operation frequency achieves 201.49 MHz.
2. LDPC encoding
2.1. General LDPC encoding algorithm
As a kind of linear block code, LDPC code can be fully described by its parity-check matrix H. only few elements of H matrix are nonzero, meaning H is a classic sparse matrix. However, its corresponding generation matrix tends to be dense even if H is very sparse. Implementing the encoder directly through dense matrix operations is very difficult, for it will result in complexity quadratic to code lengths, which is always unrealistic when code lengths are larger. So QC-LDPC was designed to solve this problem. Parity-check matrix and generation matrix of QCLDPC codes are both block-circular, so that its encoding can be easily completed with Linear Feedback Shift-Register (LFSR)[6]. LDPC code used in IEEE 802.11n/16e, and DMB-T, etc are all QCLDPC codes.

2.2. LDPC encoder for CMMB
There are 2 different LDPC code rates (1/2 and 3/4) used in CMMB standard. Both codeword lengths are 9216 bits. For 1/2 code rate, corresponding parity-check matrix H (4608×9216) data are cyclic-shifted 36 bits for each 18 rows. Similarly, parity-check matrix H for 3/4 code rate (2304×9216) takes on cyclic-shift pattern, too. H data of 3/4 are cyclic-shifted 36 bits for each 9 rows.

3. H matrix LU decomposition of CMMB
LU decomposition for H matrix is only used once in the whole encoding process, so it can be completed by software on PC, while the decomposed LU results are saved in encoder hardware.
Optimized partial pivoting strategy was used for LU decomposition, so that lower-triangular matrix L and upper-triangular matrix U are as sparse as possible, which could decrease memory requirement greatly.
Figure (a) showed LU decomposition results of 1/2 and 3/4 code rate. For 1/2 code rate, there are 28621 nonzero elements in all in L matrix, while 21676 nonzero elements in U matrix. 4. LDPC encoder design
Stratix II EP2S90 device of Altera is chosen to implement LDPC encoding for CMMB. There are 72,768 ALUTs, equivalent 90,960 LEs in all internal chip, and memory capacity also achieves over 4.5 Mbit, which is sufficient for real-time encoding of CMMB standard.
4.1. Decoder pipeline structure
Since LU decomposition is completed, whole encoding process could be accomplished according to approach listed in 2.2. To improve encoding efficiency, multi-staged pipeline architectures are introduced. An overview of whole hardware encoder architecture is shown in Figure 2:
Basically, 5 stages pipeline architectures were used in this design. Pingpong buffer was used in each stage to further improve throughput. Information bit was also buffered in each pipeline stage to ensure final outputting information codes and parity-check codes are counterparts.
In stage 1, inputting information bits are saved into upper and lower Pingpong buffer alternately. In pipeline stage 2, 3 and 4, steps 1, 2, 3 listed in 2.2 are completed one by one. Finally, solved parity-check bits and temporarily buffered information bits are combined together and reordered by codeword bits mapping vector (named IDX table) to form outputting encoded LDPC code.

6. Conclusion

Based on modified LU decomposition theory for sparse matrix, a low-complexity real-time LDPC encoder for CMMB was designed, 5 stages pipelines and Pingpong buffers were adopted to improve efficiency. Whole design was synthesized on Stratix II EP2S90 device. Maximum frequency achieves the rate of over 200 MHz. With necessary M-RAM initialized data, the LDPC encoder also can support other code lengths and code rates for arbitrary H matrix pattern.
