Efficient Code Construction of (3, k) Regular LDPC Codes

Silvia Anggraeni and Varun Jeoti Department of Electrical Engineering University Technology PETRONAS Bandar Seri Iskandar, 31750 Tronoh, Perak, MALAYSIA silvia31804@yahoo.com, varun_jeoti@petronas.com.my

Abstract
The efficient code construction of regular LDPC codes that can effectively reduce the encoding complexity is still attracting a lot of attention from the research community. This paper presents a code construction method for (3, k) regular LDPC code that utilizes combination of a deterministic part and a partially random part in parity check matrix design.
The parity check matrix (H), consisting of parity part (Hp) and information part (Hi), is so designed as to
have a girth of, at least, 6 and to generate various code
rates by maintaining the number of rows in H matrix
while only changing the number of columns in the
information part (Hi). Our code construction is able to
reduce encoding complexity by avoiding any preprocessing step during encoding.
1. Introduction
Low density parity-check (LDPC) codes were first invented by R. G. Gallager in 1960’s [5]. After the discovery of turbo codes, the LDPC codes were rediscovered and brought to the attention of the research community by Mackay and Neal [3]. Since that time, the interest in LDPC codes is back and many developments have been brought to this area.
Low density parity-check (LDPC) codes have a sparse density of one’s compared with zeros and belong to a special class of linear block codes. The code construction of LDPC code relates to designing
the parity check matrix (H) that is utilized in decoding
scheme. There are two main groups of LDPC codes:
• (j, k) Regular LDPC The matrix H has exactly j number of 1’s in each column and k number of 1’s in each row where j < k, and both are small compared to the number of columns in matrix H.

• Irregular LDPC
The matrix H has variable number of 1’s in each
column and/or variable number of 1’s in each row and both are small compared to the number of
columns in matrix H.
The irregular LDPC codes can achieve Bit Error Rate (BER) within 0.0045 dB of the Shannon limit in computer simulations with code length of 107 bit in Additive White Gaussian Noise (AWGN) channel [7].
Unfortunately, there are drawbacks of irregular codes: • The performances for medium code length ≤
10.000 bit for code rate ½ are generally not better than regular construction codes [6]. • The complexity in computation and hardware realization are higher than regular ones. Based on these points of views and the medium block length used in this paper, the choice for the proposed code construction is regular LDPC code. Since the code design of LDPC code has the sparse
density of ones and starts with parity check matrix (H)
which is applied in decoder, this has made decoding complexity is low.
To encode the code, we need to find a generator
matrix (G) by performing Gaussian elimination on H. However the drawback of this method, the matrix G
will mostly no longer be sparse and the encoding
complexity is in order of N² operation where N is the code length of LDPC code. As the value of N hovers
around hundreds to thousands of bits or more, the encoding scheme is generally complex.
There has been an approach to reduce the encoding
complexity by using parity check matrix (H) directly rather than finding a generator matrix (G). This method transforms parity check matrix (H) into an
equivalent systematic form and shows that the encoding scheme is upper bounded by O(N)+O(g2),
where g is the gap to measure the distance between a

Other approach to reduce the encoding complexity

This paper proposes the parity check matrix (H)
design which encodes and decodes efficiently, yielding
the code construction of (3, k) regular LDPC codes
with a girth at least 6 and generates various code rate by maintaining the number of rows in the parity check
matrix (H) and changing the number of columns only in the information part (Hi). The (3, k) regular LDPC
codes are usually preferred because of their good performance and lower complexity, compared with other regular LDPC codes.
The parity check matrix (H) consists of information part (Hi) and parity part (Hp). The construction of parity part (Hp) is deterministic and follows some
constraints which are described in the section II. The main purpose of this deterministic design is achieving lower triangular form matrix without additional process in pre-processing step of encoding.
Our code construction is able to avoid rank deficient and singular matrix condition of the parity check
matrix (H) without any row deletion or other additional
process such as row and column permutation in preprocessing step of encoding. The encoding complexity
is influenced by the factor of g² where the value of g is 2 × L .
This paper is organized as follows. Section 2 explains the construction of the proposed code for the
parity check matrix (H) including the parity and
information part. Section 3 describes the encoding scheme that is used in this paper. Performance results are discussed in section 4 and conclusion is in section 5.

2. Code Design

The code design of (3, k) regular LDPC code is represented by the parity check matrix H = [Hi| Hp].
The code construction is developed by applying two stage of expansion factor. The first stage of expansion factor which denoted by value of L, is done in the
parity part (Hp).
The second stage of expansion factor which
denoted by any value of L1, is applied after the construction H = [Hi| Hp] that consists of parity part (Hp) and the information part (Hi). The H = [Hi| Hp] matrix is scaled by (L1×L1) identity matrix. The second
stage of expansion factor expands longer code length than code length in the first stage of expansion factor which can reach up to million bits.
The detail construction of code design in parity part
(Hp) is described in part 2.1 and the information part (Hi) is explained in part 2.2 with explanation about the value of code rate (R).

2.2 Information Part (Hi)
The information part (Hi) is divided into some of
sub matrices Z. The sub matrix Z is (Z × Z ) identity
matrix with a random Right Cyclic Shift (RCS) of base equal to 15.

The value of Z is determined by equation below:

Z = (Base × L) ÷ j

since the value of j is 3 , base = 15 and L = 8, the value of Z is 40.

The total number of sub matrix Z in each row of information part (Hi) is determined by the value of ki that is the value of ones in each row of Hi matrix. The total number of sub matrix Z in each column is 3 since the value of j is 3 and the goal of information part (Hi) is to construct (3, ki) regular LDPC. The value of RCS in the information part (Hi) is created randomly to construct the parity check matrix (H) which is free of
girth 4-cycle.

The size of Hi matrix is Mi× Ni, where Mi is the number of rows in the Hi matrix which is equal to 3 sub matrix Z. The number of columns in Hi matrix is Ni which is equal to ki sub matrix Z. The value of Mi and Ni are given by Mi = Z × j = Z × 3 and Ni = Z× ki. After the construction of H = [Hi| Hp] that consists of parity part (Hp) and the information part (Hi), the H = [Hi| Hp] matrix is expanded by (L1 × L1) identity matrix which can be filled by any value of L1.
The second stage of expansion factor enables to construct LDPC code with code length of thousands of bits or more.

4. Performance Results & Analysis

4.1 BLER Performance
The decoding algorithm which is utilized in this simulation is sum product algorithm, also known as belief propagation. We set the number of iteration in the decoding scheme with iteration equal to 50. We simulate the code design in BPSK modulation and transmitted over AWGN channel.
We put the value of base = 15, j = 3, L = 8, L1 = 1
and change the value of ki = 1, 3, 5 and 9. This will
yield (3, 4) , (3, 6) , (3, 8) and (3,12) regular LDPC with
code length < 500 bits. The performance is measured by the probability
that a decoded codeword at the output of the decoder is in error, which is known as word error rate (WER), frame error rate (FER) or block error rate (BLER). We use the block error rate (BLER) term in this paper. The number of trial chosen is 10.000 in order to get the exact average of Block Error Rate (BLER).

4.2 Encoding Complexity Analysis
The original idea of encoding scheme in [10] divides into 2 steps: pre-processing step that is used Greedy algorithm to achieve lower triangular form and actual encoding step.
Our code construction uses only actual encoding
step since parity check matrix (H) is already in the
form of approximate lower triangular matrix indicated in Figure. 4. Here, there is no pre-processing step of
encoding which needs complexity in order of Ο(N ³)
operation as stated in step 1 in [10]. Our code construction is able to avoid rank-
deficient and singular matrix condition of the parity
check matrix (H) without any row deletion, row
permutation and column permutation. This yields reducing complexity of encoding scheme in the order
of Ο(N ³) operation in pre-processing step.
The actual encoding step in [10] needs operation in order of O(N)+O(g2) which is also the same operation
value as our code construction. The equation Φ = −ET −1B + D is very crucial in
the calculation of the parity p1 and p2. The encoding scheme isn’t success if the equation of − Φ −1 matrix can’t be solved.
This will occur in two conditions:
1. The − Φ −1 matrix is singular and the H matrix is
full rank. This condition can be solved by column
permutation of the whole element of Hi matrix and small part of element in Hp matrix. The Hp matrix starts in the first column until g column. Only
this condition occurred in our code design

2. The − Φ −1 matrix is singular and the H matrix is
rank deficient. This condition only can be solved by erasing the rows which are dependent to each other. This condition yields the reducing number
of rows in H matrix.
5. Conclusion
The construction of (3, k ) regular LDPC codes
which simplifies the encoding and decoding complexity is introduced in this paper. The constraints
of the deterministic matrix for the parity part (Hp) such as base matrix design in the parity part (Hp) and the
value of base are proven to reduce complexity of encoding scheme by avoiding any pre-processing step.
The encoding complexity is the order of
O(N)+O(g2) operation where the value of g is 2 × L . The variable value of ki in the information part (Hi)
generates various code rates by maintaining the value
of j equal to 3.
According to the performance results, our code design is shown better than regular repeat accumulate (RA) LDPC code, for example code rate ¾ at BLER of
10-5, our code construction has 0.272 dB improvement.
