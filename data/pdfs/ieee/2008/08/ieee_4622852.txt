Implementation of LDPC Decoder in DVB-S2 Using Min-Sum Algorithm

Haeseong Jeong, Jong Tae Kim School of Information and Communication Engineering
Sungkyunkwan University prometeuth@skku.edu, jtkim@skku.ac.kr

Abstract
A new market on digital broadcasting is opening because of the adoption of digital video broadcasting second generation as a digital broadcasting standard in Europe. The DVB-S2 uses a low density parity check, so DVB-S2 has much fast communication speed than conventional DVB which uses reed-solomon code and convolutional code for error correction. In this paper, we valuate performance and implement LDPC decoder of FEC which is important sub-system of DVB-S2 using VHDL. The simulated and implemented LDPC decoder is based on the Min-Sum algorithm.
1. Introduction
The DVB (digital video broadcasting) project is establish for digital television and global transmission of data service with over 260 broadcasters, manufacturers and network operators [1]. In the first specification is based on QPSK modulation and CC (convolution code) with RSC (reed-solomon coding) for FEC (forward error correction). There are increased demands for larger capacity and innovative services by satellite broadcast service since first system established. So the new specification DVB-S2 is enacting.
The DVB-S2 standard changes the FEC algorithm from CC and RSC in the DVB system to LDPC (low density parity check) and BCH (bose-chaudhurihocquenghem). Mackay confirmed performance of LDPC decoding algorithm [2]. The DVB-S2 system shows the prominent performance near the Shannon limit from 0.7dB to 1.2 dB, because LDPC decoding algorithm is applied. Also applied ACM (adaptive coding and modulation) typically 30% increased the performance in comparison with existing system.
In this paper we analyze the performance of LDPC decoder using Min-Sum algorithm. We synthesize it with RTL (revel transfer level) coding.

In the following sections of the paper, the LDPC decoder architecture and performance of DVB-S2 system will be discussed: section 2 describes the basic theory of LDPC decoder. Section 3 gives simulation results and synthesis results and section 4 concludes the paper.

2.2. LDPC

In 1962, Gallager designed sparse matrix H of linear block code called low density parity check matrix [4]. The matrix H was defined in a nonsystematic form. Each column of H had a small weight and the weight per row was also uniform.[5]. The parity check matrix H is able to decode very large block through iteration of decoding because it is sparse matrix. But, in those days there were not many applications for LDPC and it is difficult realized, so that LDPC had been generally forgotten.
After the mid-1990’s, LDPC rediscovered by Mackay and Neal [2]. LDPC performance is very close to the Shannon channel capacity limit like Turbo code [6].
The parity check matrix, H is made up K information bits and N code size, can be represented H(N-k)×N. It can be viewed as a Tanner graph consisting of N-K parity check nodes and N variable nodes. Figure 1 shows matrix and Tanner graph of H( n, j, k ) = H( 8, 2, 4 ).

3.2. Implementation LDPC Decoder

The LDPC decoder of DVB-S2 is designed for parallel processing structure which performs 360 operations at one time. It has equal parity check matrix form in blocks of 360. As mentioned, the decoder processes 360 nodes in parallel so 360 messages have to be supplied per cycle. Figure 4 shows 360 functional units mapping.

Figure 4. Functional units mapping (code rate 1/2) [7]
The rows of H matrix (check nodes) are connected to variable nodes by unit of q, which is the number of columns/360[3]. 4. Conclusion

In this paper we present the implementation and evaluation of LDPC decoder for the next generation broadcast service standard using MSA. The MSA reduces the complexity for computing message probability and it is effective for VLSI implementation. As a result of simulation we validated performance of the decoder for various channel environments, modulation methods and code rates. The result shows that the BER performance of the LDPC decoder is close to Shannon limit.
