Flexible Decoder Architectures for Irregular QC-LDPC Codes

Tzu-Chieh Kuo and Alan N. Willson, Jr.
Electrical Engineering Department University of California, Los Angeles, CA 90095-1594, USA

Abstract—A flexible and power-efficient decoder architecture employing the layered-decoding message-passing algorithm and the low-complexity offset-based Min-Sum check algorithm for irregular QC-LDPC codes is presented in this paper. The architecture is verified by implementing a programmable decoder chip compliant with the QC-LDPC codes in Mobile WiMAX standard. Compared to other published decoder implementations, the prototype decoder is 53% smaller and has better energy efficiency.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are powerful advanced error control codes that have been included as optional codes in new standards such as in the IEEE 802.11n (Wi-Fi) and 802.16e (Mobile WiMAX) standards [1]. These standardized LDPC codes are based on structured irregular quasi-cyclic (QC) codes which exhibit great levels of scalability, supporting multiple code structures of various code rates and code lengths. As a result, a decoder for these applications must be very flexible and reconfigurable. Additionally, the hardware implementation must have lowcomplexity and be energy-efficient in order to be practically integrated into VLSI chips for portable devices.
Published works do not sufficiently address these two important implementation issues. For instance, the LDPC decoders proposed in [2] and [3] are reconfigurable to realize the whole code structures defined in WiMAX, but the decoders employ 96 parallel processors for achieving a decoding throughput of 278~666 Mbps, far exceeding the maximum transmission rate of about 40 Mbps in a 10-MHz channel bandwith in the physical layer [4]. The resulting complexity and power consumption are much higher than that which is required. In contrast, the decoder in [5] consumes 52 mW in 0.13-μm technology which is more power efficient than other published works, but the decoder can be configured for just one out of the six code structures of WiMAX.
The decoder architectures proposed here consider both implementation issues. The programmable architecture is based on the fast-converging layered-decoding messagepassing algorithm by Hocevar [6] and the offset-based MinSum check algorithm by Chen [7]. It is implemented on a subset of check function units for reduced complexity. Memory architectures are further developed to simplify routing and reduce power dissipation. The performance of the architecture is demonstrated by a prototype decoder chip for WiMAX LDPC codes. The decoder is significantly

smaller than other decoders and exhibits better energy efficiency.
The decoding algorithms and the programmable architectures are described in Section II and III. Section IV presents the implementation results of the prototype decoder ASIC.
II. DECODING ALGORITHMS AND VLSI ARCHITECTURES
Fig. 1 illustrates the proposed decoding algorithms and the component function units for irregular QC-LDPC codes. A QC-LDPC code is represented by a base model matrix Hbm whose (i,j)-th entry is the cyclic-shift value for the (i,j)-th permutation sub-matrix in the full parity-check matrix H. Given Hbm, as in Fig. 1(a), H is obtained by replacing each non-empty entry in Hbm with a z × z permutation sub-matrix and each empty entry with a z × z zero sub-matrix, where z is called the expansion factor. Multiple code lengths can be generated from a single Hbm by changing z. The z rows in H expanded from a single row in Hbm is called a block row in this paper. Similarly, a block column represents the z columns expanded from a single column in Hbm. The columns in H are mapped to bit-node function units (BFU) and the parity-check rows are mapped to check-node function units (CFU). A bit node computes the a posteriori loglikelihood ratio (AP-LLR) of a transmitted bit-symbol and the bit-messages to its connecting check nodes while a check-node computes the check-messages to its neighboring bit-nodes. The AP-LLR of a bit-symbol is the sum of the check-messages produced by its connecting check nodes. The initial AP-LLRs are the channel LLRs from a preceding demodulator. In the layered decoding algorithm, BFUs degenerate into memory banks -- the Bit-Sum memories (BM) storing AP-LLRs, and CFUs process block rows as layers with messages exchanging from layer to layer for speed-up in convergence rate, as in Fig. 1(a), where the layerprocessing order is from block row 0 down to block row 1 derived from Hbm. The message computation tasks are done in CFUs.
Fig. 1(b) shows the architecture of a CFU. It is a serialmode implementation so that the unit can be reconfigured to accommodate irregular check degrees. A CFU receives one AP-LLR from BM per clock cycle, and the AP-LLR is subtracted by the old check-message to form a new bitmessage, which is further reduced by a programmable offset value for enhanced decoding performance. The minimum magnitudes, the edge index, and the sign bits are also stored in the Min-Sum memory for computing the old checkmessages in the next iteration.
A check-message generator consists of a multiplexer and a conditional inverter. The multiplexer selects between the two minimum message magnitudes based on the edge indexes of the participating bit nodes, and the selected magnitude is conditionally inverted based on the signs of the bit-messages. In addition, a CFU computes a tentative parity value for a check-node. The parity value is computed as the exclusive-OR over the sign bits of its bit-messages. If the parity value is zero, the check node is assumed to satisfy a parity check. If all parity checks in the current iteration are satisfied, the decoding operation can be early terminated for saving power if enabled.
A subset of P consecutive check nodes in a block row is processed in parallel when P CFUs are available. Fig. 1(c) shows the processing scheduling assuming z = 4 and P = 2, so each block row is divided into two sub-block rows. The natural processing order follows the increasing block column

indexes in Hbm. In addition to the pipeline latency introduced for meeting a target operating clock frequency, various wait cycles are inserted between layers because a CFU cannot request the AP-LLR of a bit-node before the on-going updating of the bit node is finished. Since the decoding performance is relatively invariant with the layer and column processing orders, matrix permutation on Hbm is conducted to find a better processing order for reduced wait cycles.
The complete decoder architecture is shown in Fig. 2. The code parameters are fetched from the configuration memory to drive miscellaneous controllers. The cyclic shifter, implemented as barrel shifters, routes AP-LLRs of a block column to destination CFUs. The updated AP-LLRs are stored back in BMs while their signs are also stored in the output memory as hard decisions.
Instead of using a set of z CFUs to achieve the maximal throughput constrained by the code structure, a subset of P processors, with P a factor of z, is employed for achieving the target application throughput with reduced complexity. Small memories such as FIFOs and Min-Sum memories in CFUs are combined together whenever a centralized memory consumes less power than the distributed memories.

III. EFFICIENT MEMORY ARCHITECTURES
Rather than storing a single AP-LLR per memory word, multiple AP-LLRs are cascaded in a memory word to reduce the number of memory instances. Using a multi-access scheme and memory output auxiliary circuits, the desired AP-LLRs can be selected from the consolidated memory banks of reduced area and power dissipation. The proposed memory architecture assuming P = 24 is shown in Fig. 3, where the memories are combined into six banks from the original 24 banks. The operating principle is illustrated in Fig. 3(b), where the current locations of the AP-LLRs, q0 up to q71, pertaining to a block column of 72 bit nodes are shown. If the cyclic-shift value of the block column is 34 from Hbm, it is computed that the read address of the first sub-block column, q34 up to q57, is from address 1, denoted ra0, for BM2 through BM5 and from address 2 for BM0 through BM2. The contents from BM2 are stored in two locations, so the content from BM2 at address 1 is fetched and stored in the register sets inside the auxiliary output circuits, followed by a secondary read from address 2. The read ports of BMs are thus operating at twice the system clock frequency. However, the other BMs are disabled during the secondary access to save power. Clock gating also applies to the auxiliary output circuits, and at most three out of the 24 register sets in the circuits are enabled during any read cycles. The multiplexers in the output circuits direct the appropriate contents to the cyclic-shifter, which then aligns the AP-LLRs in the right order for CFUs. The updated sub-block of APLLRs is stored back at the same address, computed as (ra0 + 1) in modulo 3. The read and write addresses of the second and third sub-block columns are computed similarly at their respective scheduling cycle. The cyclic shift value applied to the cyclic-shifter and the read-write addresses of BMs have to be computed on the fly with small overhead.
Using the consolidated memory structure and the centralized memory structure of Section II, the prototype decoder ASIC in Section IV only requires a total of 20 memory instances, compared to 38 memory instances in [3] and 100 memory instances in [5].
IV. IMPLEMENTATION RESULTS AND COMPARISONS
A prototype decoder ASIC has been implemented in TSMC 0.18-μm technology and is under fabrication. The decoder employs 24 CFUs to achieve a 60-Mbps decoding throughput in ten iterations. The decoder is reconfigurable to support all QC-LDPC codes of WiMAX. It can also be programmed to support other code structures of appropriate sizes, limited by on-chip memories.

Similar performance is achieved for the other code lengths, and there is no obvious error floor observed. The comparison of the decoder with other programmable decoders is summarized in Table I, where the throughput and power dissipation are characterized with the same rate-1/2, length2304 WiMAX code. Technology normalization is done with respect to 0.18-μm feature size and 1.8-V voltage. The normalized energy efficiency is defined as the normalized energy consumption per decoded bit per iteration. The prototype decoder is at least 53% smaller and employs significantly fewer logic gates than other WiMAX LDPC decoders shown. The estimated energy consumption of the prototype chip is 27% better than that of the fixed-code decoder in [5], which implements the traditional two-phase message-passing algorithm. The energy performance is actually 45% better, as the convergence rate of the layered decoding message-passing algorithm is about 1.7 times faster than the two-phase message-passing algorithm.
