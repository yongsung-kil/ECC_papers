Abstract-In this paper, a modified min-sum decoding algorithm based on classified correction is proposed for low density parity check (LDPC) codes. Different from the single correction in the normalized Belief Propagation (BP)-based and offset BP-based algorithms [4], the proposed algorithm utilizes two corrections for both minimum and sub-minimum magnitudes of input messages in check nodes. These two correction factors can be obtained by analyzing the offset of updated messages in check nodes between the BP and the min-sum algorithms associated with check node degree. Simulation results show that the proposed algorithm can achieve performance very close to that of the BP algorithm. Furthermore, the FPGA implementation of this algorithm can reach a throughput of 200Mbps at BER=10-6 with lower complexity and fewer resources than the BP algorithm.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] are a class of iteratively decoded error-control codes which have attracted much attention in recent years due to their excellent performance and parallelizable decoder architectures. These two peculiarities have significantly contributed to make LDPC codes among one of the most promising candidates for next generation data communication and storage systems, such as IEEE 802.16e, DVB-S.2 and 3GPP LTE, etc.
In general, LDPC codes are decoded by the belief propagation (BP) algorithm [2]. Theoretically speaking, the BP algorithm will converge to a posteriori probability for variable nodes if the Tanner graph (TG) of each code is cycle-free. Unfortunately, the messages passed through the edges of the graph in the BP algorithm are statistically dependent, due to the existence of short cycles in the TG. So there is no guarantee that the BP algorithm is optimal for short and moderate block lengths. Additionally, the hardware implementation of the BP algorithm is limited by high computational complexity.
In [3]-[4], several modified versions of the BP decoding algorithm have been introduced. M. Fossorier proposed the min-sum approximation [3]. Compared with the BP algorithm, the min-sum approximation greatly reduces the decoding complexity in implementation, but results in performance loss near O.5-ldB. Jinghu Chen analyzed the performance of two improved min-sum algorithms for LDPC codes, namely the normalized BP-based and the offset BP-based algorithms, by means of density evolution (DE) [4]. But these two algorithms

show performance degradation for some irregular degree distributions.
In this paper, we compare the extrinsic message values in check nodes delivered by the BP algorithm and the minsum algorithm respectively, and propose a modified minsum decoding algorithm based on classified correction. In the proposed algorithm, the two different correction factors are only related to the check node degree in the TG.
The paper is organized as follows. In Section II, we give a brief review of the BP algorithm and some approximation algorithms. Then we present the classified correction algorithm in Section III. Some simulation results are provided in Section IV to illustrate the performance of the proposed algorithm. Then, finite precision simulations and FPGA implementation are presented in Section V. Finally, a conclusion is drawn in Section VI.

IV. SIMULATION RESULTS

Simulation results are presented for comparison between the classified correction algorithm and other decoding algorithms discussed in Section II. The regular (3, 6) LDPC code with code rate 0.5 and code length 1008 [7] is considered, and the BPSK modulation of the all-zero codeword is transmitted over an AWGN channel. A maximum of 100 decoding iterations was used for decoding each transmitted codeword.
Fig.3 depicts the bit error performance and frame error performance of different decoding algorithms. It can be seen that the min-sum decoding algorithm is the worst one, due to the overestimation the reliability of the extrinsic messages without any correction. The gap between the BP and the min-sum algorithms is about 0.5dB. For the offset BP-based
decoding algorithm, with the parameter p = 0.15 derived by
means of DE [4], the performance loss related to that of the BP algorithm can be reduced to 0.2dB. Furthermore, for the classified correction algorithm, we choose different correction factors for AminI and Am in2 as (13) and (14), respectively.

V. FPGA IMPLEMENTATION
For applying the decoding algorithm presented in Section III, we select an (8176, 7154) Euclidean Geometry (EG) constructed irregular quasi-cyclic (QC) LDPC code [8]-[9] with code rate 0.875. The H is a 1022 x 8176 matrix with constant row weight 31 and two column weights, 3 and 4. We use partly parallel decoding structure to obtain a good tradeoff between the hardware complexity and the decoding speed [10]. The decoder mainly contains 16 variable node units (VNUs), 2 check node units (eNUs), and 78 message-RAMs.
Fig.S shows the VNU for a column schedule architecture. The dv successive L(rji) messages from the CNU are converted from Sign-Magnitude (SM) representation to 2's complement representation, then added together with L(P) to fonn (4). The most significant bit (MSB) of L(Q) is the sign bit which is used for hard decision. To avoid feedback in each iteration, the output L(qij) should subtract L(rji) from L(Q) as given by (3). After overflow protection, all the 6-bit results are converted from 2's complement representation to SM representation. Finally, the 7-bit results composed of 6-bit data and I-bit hard decision are stored in the message-RAMs for CNU read as input messages.
Fig.6 shows the correction part of CND for a row schedule architecture in the classified correction algorithm. This module receives de inputs. Each input is a 7-bit SM binary number. All the MSBs and the next most significant bits (NMSBs) of L (qij) are used for calculating parity check sum and the final

sign of output messages respectively. The rest 5bits represent magnitudes of L(qij) which need to be corrected as (13) (14). When the final AminI , A m in2 have been found, they are calculated as (15) and transferred to the same message-RAMs for the next iteration.
The word length of the LLR messages directly affects the memory size and the decoding performance. We compare the error performance of different quantization schemes in Fig.7. The (6, 8) uniform quantization (UQ) scheme in the classified correction algorithm can achieve decoding performance almost identical to that of the floating point case with 20 iterations as the maximum limit. It performs only about IdB from Shannon limit at BER=10-6 .
Both VNU and CND in this design are pipelined. It takes 2 x 519 = 1038 clock cycles to perform one iteration. The decoder is modeled in VHDL, simulated and synthesized targeting on the Xilinx Virtex2p x2vp50. After place and route, the report shows that the maximum clock frequency is 155.8MHz. The FPGA utilization statistics is shown in Table I.
Due to the decrease of the average number of iterations at high SNR, the iterative interrupt rule is set in CND, which means once the decoded code are valid, the next frame will be transferred to the decoder. The decoder throughput vs. SNR is shown in Table II. When BER is below 10-6 , the decoder throughput will reach over 200Mbps.
VI. CONCLUSION
In this paper, we have proposed a classified correction decoding algorithm for LDPC codes, which corrects both minimum and sub-minimum magnitudes of extrinsic LLRs in check nodes. Simulation results show that the proposed algorithm can achieve slight!y better performance than that of the BP algorithm at high SNR with the same computational complexity of the offset BP-based algorithm. By using this algorithm, we realize the decoding throughput of 1.2Gbps per iteration, and 200Mbps at BER=10-6 on the Xilinx Virtex2p x2vp50 FPGA chip. Furthermore, various techniques and decoder architectures can be incorporated to increase the clock speed [11], and more detailed decision thresholds can be set to recoup the extrinsic messages precisely [6], but the related complexity and resource consumption will be increased.
