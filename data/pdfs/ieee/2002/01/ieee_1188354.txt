1. INTRODUCTION
Tornado codes have been studied recently for erasure channels [I], [2]. The encoding and decoding complexities of tornado codes are significantly lower than other conventional erasure correcting codes (such as Reed-Solomon codes [3]).
This paper is concerned with a simple coding scheme based on the so-called semi-random low-density-parity-check (SRLDPC) codes introduced in [4]. We will show that the new scheme is equivalent to the first stage of a tornado encoder under the restriction of perfect reception of parity bits. This equivalence is helpful for deriving the characteristics of the new scheme using the existing work on tomado codes [I]. Furthermore, we will show that the restriction to perfect reception of parity bits can be removed due to the invariance of the new code structure in general random erasure channels. Thus the new scheme avoids the complicated tornadwtype cascade structure without compromising performance.
An SR-LDPC code [4] is defined by its parity check matrix H. This matrix is partitioned into a specially shuctured subblock and a randomly generated one (see (19) below). It has the advantage of a simplified encoding process compared with a standard LDPC code [5].n e IRA code [6] and the CT code [7] (including the zigzag code [E]) can be regarded as examples of SR-LDPC codes. (An IRA code is based on a single interleaver while a CT code based on several component interleavers.) For short codes, the CT approach has certain advantages, as explained in the Appendix.

IV. CONCLUSIONS
From (20). Remark 3, and Remark 5, an SR-LDPC code has a similar encoding and decoding complexity as the sub code formed by the first two columns of nodes in a tornado code. From Theorem 3, the former can achieve capacity in a general erasure channel without extra protection on parity check bits. Therefore, the proposed SR-LDPC code represents a simple, high performance coding scheme for erasure channels. We have carried out a simulation study. The results (not shown in this paper due to space limitations) confirm that SR-LDPC codes can achieve nearly the same performance as Tomado codes.
