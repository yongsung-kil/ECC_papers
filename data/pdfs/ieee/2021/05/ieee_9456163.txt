AbstractвҖ”This paper presents the simulation of the encoding scheme defined by CCSDS for satellite Telecommand operations and analysis of various soft decision and hard decision decoding algorithms for the same. The G(Generator), H(Parity-check) matrices are designed and verified for encoding and decoding of the message respectively. The decoding algorithms are implemented and the performances are analyzed and non-convergence of decoding algorithms, scheduling restrictions of the defined H, G matrix are explored. It is found that soft decision algorithms give 6dB coding gain and hard decision algorithms give 4 dB coding gain for the 4bit quantized inputs.
KeywordsвҖ”CCSDS, LDPC, channel coding gain, BER, Softdecision, Hard decision.
I. INTRODUCTION
Low Density Parity Check codes (LDPC) are linear ,Shannon limit approaching ,forward error correcting codes introduced by Gallager in 1960 [1] and were revived by MacKay and Neal [2] .LDPC is widely used in standardized communication systems like WiFi [3], WiMAX[4], DVBS2[5], Consultative Committee for Space Data Systems (CCSDS) [6] and ITU G.hn [7] .CCSDS has defined a LDPC code as one of its channel coding schemes for the use in Telecommand Space Data Link Protocol. This wide applicability of LDPC has been established due to the advancement of Very Large Scale Integration (VLSI) technology. In addition, decoding algorithms are also simpler and can be implemented with low hardware utilization.
LDPC can be constructed in different ways through varied placement of вҖҳ1sвҖҷ in Parity-check matrix. Since placement of вҖҳ1sвҖҷ in the Party-check matrix(H-matrix) has significant impact on error correction performance and decoding complexity as well. Generally, design of H-matrix should avoid stopping sets [15] and short cycles [9] to improve the error correction performance and for reduction in decoding complexity.
Code construction or H-matrix construction techniques are Random code, Quasi-cyclic code, Repeat-accumulate code, Progressive edge growth code method of construction [10]. CCSDS has employed Quasi-cyclic method of code generation [6], this method enables reduction of memory & ease of routing in hardware implementation. The designed

parity check matrix is used to construct Generator(G) matrix which is used in design of encoder.
Construction of decoder involves design of decoding algorithm which can be classified either as soft decision or hard decision algorithm. Soft decision algorithms receive Log Likelihood ratios вҖ“ (Multi-quantized bits along with sign bits) from the receiver and process it to decide on the received bit [10]. Soft-decision algorithms have good decoding performance but consume more resources for hardware implementation with reduction in throughput as well [11]. Hard-decision algorithms straightaway receive the binary code word (single bit quantized output) and further process it with syndrome checking as the preliminary step. Its decoding performance is not satisfactory in comparison to soft-decision decoding algorithms. Nevertheless, its throughput and hardware utilization are better than softdecision algorithms [11].
This work aims at the analysis of various existing decoding algorithms for the LDPC code defined by CCSDS since very minimal literature is available to analyze the decoding algorithms, in particular for LDPC code defined by CCSDS for satellite Telecommand operations. Hence the primary motive of this work is to bring out the coding gain achieved by existing decoding algorithms for CCSDS defined LDPC code. This will help in development of standardized LDPC decoder for satellite Telecommand operations, as in future, interoperability is required between ground stations of various private and government organizations.
The encoder is constructed according to the Generator matrix constructed in [6]. Decoding algorithms such as Belief-propagation, Approximate Belief propagation, MinSum, Normalized Min-Sum, Adaptive Bit-local Threshold Bit Flipping (ABTBF), Turbo Adaptive Bit-local Threshold Bit Flipping (TABTBF)have been implemented with the Hmatrix defined by CCSDS for Telecommand, and their BER performances are studied. Analysis have been done with floating-point inputs and quantized inputs in case of soft decision. The Adaptive Bit-local Threshold Bit Flipping and Turbo Adaptive Bit-local Threshold Bit Flipping algorithms are hard decision algorithms and latter belongs to soft decision category.

II. CCSDS DEFINED LDPC CODING
LDPC codes have certain characteristics on which it can be categorized. The LDPC code defined here is a systematic, irregular, half-rate, transparent code. A systematic code has its message part in MSBs followed by the parity check part. An irregular code does not have same column or row weights in its H-matrix or an irregular code is one where in information bits & parity bits are mixed. A half-rate code is advantageous when compared to quarter-rate code as itвҖҷs increased error correcting capability will not compensate the energy loss [1]. Transparent code is a code which is still valid after the frame phase inversion operation.

III. IMPLEMENTED LDPC DECODING ALGORITHMS вҖ“A
BRIEF
LDPC decoding algorithms are commonly referred as Message-passing iterative algorithms and numerous algorithms have been proposed in the literature [10]. In this section implemented soft and hard decision algorithms are discussed. Moreover, non-convergence of Min*belief propagation algorithm for quantized inputs and scheduling restrictions for the LDPC code defined by CCSDS telecommand space data link protocol are elaborated.

A. Analysis of various Soft decision decoding algorithms
LDPC decoders generally use the soft decision algorithms. Here inputs to the decoder will be LLR (Loglikelihood ratio) [12]. LLRвҖҷs sign dictates the binary value of the received codeword bit for which this LLR is calculated by the demodulator (positiveГҶ0, negativeГҶ1) [12]. The magnitude expresses the likelihood of that bit to be positive or negative. This LLR is otherwise known as вҖҳbeliefвҖҷ. This nomenclature (soft)is given because the demodulator does not decide the received bit to be вҖҳ1вҖҷ or вҖҳ0вҖҷ based on the received vector. But calculates how likely the received vector is to be either of the binary value.
The algorithm which gets this soft information (LLR) is called as soft decision decoding algorithm. Various soft decision algorithms have been proposed for LDPC decoding [10]. Among those, Belief propagation [15], Approximate Belief propagation [14], Min-sum, Normalized min-sum [16] are considered for soft decision algorithm implementation, and its performance are verified.

propagation algorithm

Although the Approximate Min*belief propagation

algorithm(ABP) gives significant hardware optimization, still

it involves complex arithmetic operations. ABP algorithm

with quantized input may result in a check-node operation

output as вҖңNaN (IEEE arithmetic representation for Not-a-

check-node is вҖҳ0вҖҷ because constraint message updates require

multiplication of вҖҳ0вҖҷ and вҖҳВ’. This happens due to

initialization of the ДҜ (storage variable) to вҖҳВ’вҖҷ in [14].Hence

it is recommended to initialize the storage variable to НҙаҜЎа¬ҝа¬ө аөҶ Ні ,where ЭҠ is quantization width.Neverthless even

was вҖҳ0вҖҷ i.e. the magnitude of likelihood of the received bit to

The Min-sum algorithm and its variants can be executed in certain sequence. Scheduling in LDPC decoding defines the sequence in which Variable node (node representing the columns of party check matrix) and check node (node representing the rows of party check matrix) processing is carried out.
Three major scheduling schemes applied for LDPC are Flooded Scheduling, Layered scheduling and Informed Dynamic Scheduling [10]. Flooded scheduling is processed in an iterative manner and in each iteration all variable nodes are processed simultaneously followed by simultaneous operations of all check nodes. Layered scheduling is also iterative scheduling but processing of check nodes is sequential, where in specific subset of the check nodes are activated at a time along with the activation of all variable nodes connected to the activated subset of check nodes and other remaining subsets of check nodes are not activated [10]. Informed Dynamic Scheduling checks the message passed between various nodes and activates the node which is expected to improve the belief [10].
3) Scheduling restrictions for CCSDS defined LDPC code
From the above mentioned scheduling schemes, Layered Scheduling cannot be implemented for H-matrix defined by CCSDS because one column may intersect with two subsets of check nodes, which is not the criteria for implementing

Layered Scheduling [15]. The Informed Dynamic Scheduling requires additional operations which intern increases hardware utilization. Hence Flooded scheduling is proposed for the decoding of LDPC code defined by CCSDS.
B. Hard decision algorithms
In hard-decision decoding, Bit-flipping (BF) algorithm is commonly used. It involves checksum calculation in each check node and if all the checksums are satisfied, algorithm is terminated, otherwise number of checksum failures are calculated at each variable node from the information received from respective connected check nodes. Then it is compared with the threshold for allowed checksum failures and the variable nodeвҖҷs outgoing bit is flipped if the checksum failed exceeds the fixed threshold.
Performance of the BF algorithm is not satisfactory in comparison to soft decision algorithms. Hence [11] proposes two algorithms-ABTBF, TABTBF.The ABTBF assigns individual thresholds to all variable node, and as the algorithm progresses in iterations the thresholds are reduced by 1 with the constraint given by column weight i.e. to stop varying the threshold if particular iteration reaches. The TABTBF algorithm is same as ABTBF except it updates all the syndrome involving the flipped bit and the requirement of syndrome calculation before the upcoming iteration is not required. But, here each iteration has latency greater than the previous algorithm nevertheless number of iterations required are fewer.

V. CONCLUSION
This work presents the performance analysis of the LDPC decoder for CCSDS defined LDPC code. Various hard decision and soft decision algorithms have been implemented and its performance have been analyzed. In

particular, soft decision algorithms have been analyzed with both floating point and quantized inputs. The results show that, for floating point inputs for soft decision algorithm, the best coding gain obtained is 7.7dB for NMSA and 6 dB for MSA with quantized inputs. The hard decision algorithm gives a coding gain of 4. 1dB.Hence soft decision algorithm gives better performance than hard decision. The work can be extended to hardware implementation of optimized soft decision algorithm for actual realization of Telecommand decoder for satellite operations.
