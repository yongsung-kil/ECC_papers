Logistic Regression for LDPC Decoding Failure Prediction

Taehyun Kim Network Business Samsung Electronics Co., Ltd. Suwon-si, Korea th8978.kim@samsung.com

JooSung Park Network Business Samsung Electronics Co., Ltd. Suwon-si, Korea jsjoo.park@samsung,com

Abstract—In this paper, we propose a regression analysis for the early stopping of low-density parity-check (LDPC) decoder to reduce computational complexity and decoding latency. The proposed scheme predicts the decoding failure of the LDPC decoder using logistic regression. The data sets for the decodability classification consist of the index of iteration and the number of unsatisfied parity checks. Simulation results show that with the proposed scheme, the average number of iterations of the LDPC decoder can be significantly reduced without performance loss. In addition, since the proposed scheme determines the decoding failure based on the instantaneous value, it has advantages in terms of memory and computational complexity compared to the early stopping schemes based on the changes in parameters.
Index Terms—low-density parity-check code, logistic regression, early stopping
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] have been used in various communication standards and are adopted for enhanced mobile broadband (eMBB) scenario in 5G new radio (NR) [2]. LDPC codes can achieve capacity-approaching performance through their iterative decoding algorithm such as the sum-product algorithm. However, depending on the channel state, the decoding may fail even though the maximum number of iterative processes has been performed. If the decoding failure is predicted during the LDPC decoding, the unnecessary computation can be reduced by terminating the iterative processing in advance.
There were several previously proposed stopping criteria to predict decoding failure. Depending on the kind of information used for the criteria, the schemes can be divided into two groups. The schemes involved in the first group observe the reliability variation of messages during the decoding process [3]–[6]. If the change in average reliability for a certain period is less than a threshold value prescribed in each scheme, decoding failure is predicted and the decoding process is immediately terminated. The drawback of this type of criteria is the high complexity caused by the calculation to obtain the average reliability.
The schemes in the second group predict decoding failure using the syndrome test [7]–[11]. The syndrome test is a wellknown stopping criterion and is used to verify the validity of the decoded output. If the change in the syndrome test

result during consecutive iterations is less than a threshold, it is predicted that decoding will fail. Compared to the first group, the additional complexity of the second group is relatively low.
In most of the previously proposed stopping criteria, the parameters for the decoding failure prediction were experimentally determined. Recently, there were various researches to optimize the parameters of the communication systems using machine learning. For the LDPC codes, machine learning has been used to optimize the normalization and offset factors for improving the performance of the decoding algorithm [12]– [15].
In this paper, we use machine learning to predict the decoding failure of LDPC decoders. In order to classify the decodability, the logistic regression (LR) with a multivariate polynomial model is used. The data set for training the regression coefficients consists of the index of iteration l, the number of unsatisfied parity checks at the l-th iteration, and the tag for indicating decodability. With a relatively simple metric, the proposed scheme performs accurate and fast decoding failure prediction. In addition, compared to previous early stopping schemes based on the changes in parameters during the decoding process, additional memory is not required due to the memoryless metric.
The remainder of this paper is organized as follows. In Section II, we briefly review the decoding algorithm of LDPC codes. Section III introduces the proposed early stopping scheme with LR. The error correction performance and complexity caused by early stopping schemes are discussed in Section IV. Finally, Section V concludes this paper.

III. EARLY STOPPING BASED ON LOGISTIC REGRESSION
In this section, a regression model for decoding failure prediction is introduced. First, we show the number of unsatisfied parity checks at each iteration to be used as data for the training of regression coefficients and as the input variable of the LR for decoding failure prediction. The proposed scheme uses only some of the parity checks for low complexity, but it is possible to use all parity checks. In this paper, we focus on the scheme using a quarter of the total parity checks. Then, the detail of the proposed LR is explained. At the end of this section, the threshold-based stopping criterion using the learning results of LR is also proposed.
A. Training Data Set for Proposed Regression Model
The decoding failure of the LDPC decoder means that at least one unsatisfied parity check exists during the iterative decoding process. Let NP(lC) denote the number of unsatisfied parity checks at the l-th iteration.
Figure 1 shows NP(lC) via iterations for a NR LDPC code with base graph 2 (BG2), information length K = 520 and code rate R = 1/3. The number of plotted data points is 10000. As represented in Fig. 1, the NP(lC) for the decodable case gradually converges to zero as the number of iterations increases, while that for the undecodable case changes irregularly. In the region that data points of the decodable case and undecodable case are coexistent, it is impossible to classify two cases.

Fig. 2. Confusion matrix for the proposed logistic regression classifier.
The problem that we try to solve through LR is the binary classification for the decodability of LDPC decoders. Figure 2 shows the confusion matrix of the classification problem considered in this paper. In the viewpoint of decoding performance, type I classification error is not a critical issue since decoding failure can be determined through cyclic redundancy check as conventional LDPC decoders. However, type II classification error causes an increase in the error rate of the decoder. In addition, when combined with a hybrid automatic repeat request system, the throughput decreases due to unnecessary retransmission requests.
When the data set with the coexistence region is used for training regression coefficients, several type II errors occur and decoding performance is degraded. To solve this problem, we exclude the data of the undecodable case in the coexistence region. Then, only the data of decodable case remains in the coexistence region of the training data set. When the regression model is trained using the data set refined as above, the decision region for the decodable case includes most of the coexistence region, so it becomes larger. However, the coexistence region still exists in the practical environment. Therefore, this solution increases the number of type I errors, but effectively reduces the number of type II errors that affect the error performance of the LDPC decoder.

If the output of sigmoid function y is greater than 0.5, the case is classified as undecodable. Otherwise, the case is classified as decodable.
The proposed regression analysis uses the following balanced cross-entropy function as a loss function that should be minimized through the training using the refined training data set.

L = −(αd log y + (1 − α)(1 − d) log (1 − y)), (6)

where d is the label represents that the case is decodable (i.e., d = 0) or not (i.e., d = 1), and the weighting factor α is in the range [0, 1]. In general, the balanced cross-entropy is used to solve the problem of class imbalance in the data set. However, in this paper, it is used to reduce the number of type II classification errors. The first term on the right-hand side of (6) is closely related to the type I classification error and the second term is related to the type II classification error. When α is less than 0.5, the loss of LR is more sensitive to type II classification errors and the regression coefficients are optimized to reduce the type II classification errors. In this paper, we set the parameter α as 0.1 to enhance the learning effect when the actual class is decodable.
To optimize regression coefficients, the adaptive gradient (AdaGrad) method [16] is used. The initial state of regression coefficients is set as 0. Figure 3 shows the accuracy of decodability classification using regression coefficients at each epoch. As we intended, it turned out that the type II classification error rate of LR with α = 0.1 is lower than that of LR with α = 0.5. If the factor α is set to 0.5, the lower type I error rate is obtained at the end of learning, but the higher type II error rate adversely affects the error correction performance of the LDPC decoder.

IV. SIMULATION RESULTS
Simulations are performed over an additive white Gaussian noise (AWGN) channel with binary phase-shift keying (BPSK) modulation. For the comparison with the proposed LR-based stopping criterion, we simulate previously proposed early stopping schemes. The scheme based on the convergence of mean magnitude (CMM) [4] is chosen among the schemes using the reliability of messages and the scheme based on the checksum variation pattern (CSVP) [8] is selected among the schemes using syndrome test results. The parameters of previous stopping criterions are set to avoid making wrong decoding failure decision. In the simulations, the syndrome test

is performed to check the valid output before decoding failure prediction is performed. In order to show the effect of the stopping criterion for the undecodable case, We also simulate the performance of the conventional LDPC decoding algorithm which only has the syndrome test (ST) for early stopping. In addition, we show the effect of α, which is the weighting factor of loss function, on the error correction performance of the proposed LR-based stopping scheme.
We simulate several practical LDPC codes with different lengths and rates. NR LDPC codes with various information bit length (260 and 520) and code rates (1/2, 1/3, and 1/4) are used to evaluate the performance. The decoding algorithm is the sum-product algorithm, and the maximum number of iterations is 30.
A. Error Correction Performance
Figure 4 shows the frame error rate (FER) of each stopping scheme. The wrong prediction of decoding failure results in the degradation of error correction performance. Since the LRbased stopping scheme with α = 0.5 has more type II classification errors, slight performance degradation is observed in the low error rate region. However, the scheme with α = 0.1 has the same error correction performance as the conventional LDPC decoder. Therefore, it is recommended to use α less than 0.5 to obtain the regression coefficients in a system that requires a low error rate.

Since most previously proposed stopping schemes, including CMM and CSVP, observe the variation in parameters during the decoding process, the early stopping for undecodable cases cannot be made in the first few iterations. In the LR-based stopping schemes, the lower number of iterations is obtained when the factor α is 0.5. The reason is that more type II classification errors cause more frequent decoding termination even if they are false predictions.

V. CONCLUSION
We propose a multivariate polynomial regression model to predict decoding failure based on the variation of unsatisfied partial parity checks. At the end of each decoding iteration, the LR is performed to classify the decodability of the codeword. If it is classified as undecodable, the LDPC decoder is immediately terminated. The LR model used in this paper has the effect of reducing the type II classification errors that cause the degradation of error correction performance. Also, a threshold-based stopping criterion using the decision boundary is presented. Simulation results show that the proposed LRbased stopping criterion predicts decoding failure earlier than the previously proposed stopping criteria without performance loss. Furthermore, the additional complexity caused by the LR is lower than the others.
