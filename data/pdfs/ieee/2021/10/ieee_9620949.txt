Machine Learning-Based Error Recovery System for NAND Flash Memory with Process Variation
Seonmin Lee, Jeongju Jee, and Hyuncheol Park School of Electrical Engineering
Korea Advanced Institute of Science and Technology (KAIST) Daejeon, South Korea
{sminlee, ji1718, and hcpark}@kaist.ac.kr

Abstract—The storage capacity of NAND flash memory has been significantly improved with advanced scaling and multileveling technologies. However, due to decreased intervals between voltage windows, NAND flash memory systems become vulnerable to retention and program/erase (P/E) cycling errors. Moreover, significant process variation resulting from the fabrication process results in different reliabilities among flash blocks. In this paper, we propose a machine learning-based error recovery system for low-latency NAND flash memory. In the NAND flash controller, the retention time and process variation are inaccessible in general. Therefore, it is difficult for the controller to select an appropriate decoding mode, resulting in increased latency. To replace the inaccessible information, we utilize the on-cell ratio as alternative information. The proposed error recovery system selects an optimal decoding mode which ensures successful error recovery with the minimum average read latency. Through simulation, we show that the proposed system predicts an optimal decoding mode with high accuracy. Moreover, average read latency performance is significantly improved compared to other baselines.
Index Terms—3D NAND flash memory system, error recovery system, process variation, machine learning
I. INTRODUCTION
NAND flash memory has been faced with higher density and capacity demands. To meet these demands, advanced downscaling and multi-leveling technologies have been developed, which have boosted the storage density. However, in expanding under 10-nm process technology, 2D NAND flash memory technologies have encountered fundamental limits such as lithography problem and cell-to-cell interference [1]. To solve these limits, 3D NAND flash memory in which cell arrays are vertically stacked is developed [2]. Unfortunately, as flash memory density increases, cells become more vulnerable to retention and program/erase (P/E) cycling errors [3]. In addition, process variation causes reliability difference among flash blocks [4]. In particular, in 3D NAND flash memory, there exists significant layer-to-layer process variation which causes variation of threshold voltage distribution among different wordlines [5].
To improve flash reliability, an error correction code (ECC) with strong error correction capability such as low-density parity-check (LDPC) code has been adopted to NAND flash memory systems [6]. LDPC decoding supports soft-decision

(SD) decoding that enhances error correction capability by obtaining soft information through additional sensing. Despite its stronger error correction capability, SD decoding results in higher latency during additional sensing.
There are several recent studies on improving the error recovery of NAND flash memory systems. In [7], a dynamic log-likelihood ratio (LLR) scheme based on the expectation–maximization (EM) algorithm for LDPC decoding is developed. This work always starts decoding from harddecision (HD) decoding, which has low latency and low error correction capability, regardless of the current memory state. Therefore, when the reliability of memory is degraded, a redundant decoding process can occur. The authors in [8] proposes a decoding mode selection scheme by estimating the raw bit error rate (RBER) value which represents the memory state. This work causes code rate penalty in the estimating RBER process, so the data storage efficiency is degraded. In [9] and [10], machine learning-based schemes for enhancing the decoding performance of NAND flash memory systems are provided. An error rate prediction model of NAND flash memory based on the random forest is presented in [9]. In [10], a machine learning-based algorithm which predicts the appropriate decoding status of NAND flash memory is provided. The works in [9] and [10] exploit retention time as an input feature, however, the controller is unaware of precise retention time in general. In addition, the process variation is not considered although it significantly affects the performance of multi-block NAND flash memory systems.
In this paper, we propose a machine learning-based error recovery system for low-latency NAND flash memory. Unlike previous works, we consider process variation and conduct experiment with accessible information at the controller without code rate penalty. The proposed system starts decoding in the optimal decoding mode which ensures successful error recovery with minimum average read latency. Simulation results show that the proposed system starts decoding from the optimal mode with higher than 94 % accuracy on average and closely achieves minimum average read latency in various blocks and wordlines with different reliabilities.

A. Basic operation
The basic operation of the NAND flash memory system is depicted in Fig. 1. The message bits are encoded into codewords, and the codewords are mapped to symbols. Next, electrons are injected into the memory cells up to the target threshold voltage, and this process is called as a program operation. The threshold voltage distribution of memory cells is distorted by several factors – e.g., P/E cycling error, retention error, process variation, etc.
To read the programmed data, the controller applies a predefined voltage between the voltage windows. The threshold voltage is identified by comparing it with the read reference voltage, which is denoted as sensing operation. We assume that the read references are ideally found as the intersection of adjacent distributions by existing algorithms [3], [11].
In the end, the noisy codewords are corrected through the decoder. The decoding modes used for the LDPC decoding are HD decoding, 2-bit SD decoding, and 3-bit SD decoding. In this paper, we often use ‘mode’ without ‘decoding’ in the word ‘decoding mode’ for convenience. Fig. 2 illustrates the sensing operation of LDPC decoding modes. HD decoding is performed with 1-bit LLR information obtained by sensing once. n-bit SD decoding is performed with n-bit LLR information through 2n − 2 additional sensing after HD sensing. Error correction capability can be enhanced by SD decoding, which exploits more precise LLR information, however, higher latency is required due to additional sensing.

IV. SIMULATION RESULTS
In this section, we evaluate the proposed machine learningbased error recovery system in terms of prediction accuracy and average read latency through simulation.

A. Simulation settings
In the NAND flash memory, we extract 9 blocks as follows. From (3), we generate 10,000 block variation parameter s and sort them in order of decreasing reliability. We obtain 9 samples for s corresponding to the top 10 % to 90 % and sequentially assign them into blocks. Thus, in our experiment, the larger the block number, the worse the reliability. Each block contains 32 wordlines, and the page size is 16 KB. We select 4 out of 32 wordlines from each block.
The ECC specifications are described as follows. Each 1KB data frame is encoded by the LDPC code with a code rate 0.9. The LDPC code is decoded by the min-sum algorithm. The maximum number of iterations for HD and SD decoding is set to 50 and 250, respectively. If the decoding succeeds, the iterative decoding is terminated at that moment.
In extracted blocks, we acquire data by sampling every 2 hours between 2 to 50 hours of retention time at 1, 100, 500, 1000, and 2000 P/E cycles. We train a classifier with data belong to blocks 1, 3, 5, 7, and 9 among the obtained data.

B. Prediction accuracy
Table I shows the prediction accuracy of the classifier for all blocks. Class 1, 2, and 3 denote the case when the optimal decoding mode is HD, 2-bit SD, and 3-bit SD, respectively.

The classifier predicts the optimal mode with higher than 94 % accuracy on average. Particularly, although the blocks 2, 4, 6, 8 have different RBER characteristics from the training data, the classifier achieves accuracy performance higher than 95 %. For data belong to class 2 and class 3, the classifier also predicts the optimal mode with about 100 % accuracy. Through this, most unnecessary error recovery processes can be omitted and latency can be greatly reduced. In addition, the classifier predicts LSB page with 100 % accuracy, and the accuracy decreases slightly in CSB and MSB pages. This is because CSB and MSB pages have relatively higher page error rates than LSB page [8].

C. Average read latency
We evaluate the average read latency performance at the 6th wordline of the 9th block, whose reliability is worst among the blocks. We assume that the latency for one sensing is 100/7 µs, decoding latency is 0.5 µs per iteration [17], and data transfer latency is not considered. In addition, we set the latency for prediction through the classifier to be nearly zero because the proposed classifier has a relatively simple inference algorithm. For comparison, the latencies of the following baselines are evaluated.
• Static denotes a scheme in which decoding always starts from HD decoding mode. If HD decoding fails, 2-bit SD and 3-bit SD decoding are sequentially performed until error recovery succeeds.
• OPT denotes a genie-aided baseline scheme that knows the optimal decoding mode in advance. Its latency becomes a lower-bound performance.
In addition, we add the three baselines below for the case when wordline and/or block-to-block RBER variation are unaware. These baselines are look-up table-based schemes.
• BU denotes a case where block variation is unaware. A look-up table is generated based on data from the 5th block, which is an intermediate block variation level.
• WU denotes a case where wordline variation is unaware. A look-up table is generated based on data from the 8th wordline, which has an intermediate FER performance among 32 wordlines.

V. CONCLUSIONS
In this paper, we propose a machine learning-based error recovery system for low latency NAND flash memory. To replace retention time and block variation, which are generally inaccessible information at the controller, we use on-cell ratio as alternative information. Moreover, we derive the average read latency taken when decoding starts in each mode. From this, we generate a dataset by finding a mode that ensures successful error recovery with minimum latency. By utilizing

generated dataset, we train the classifier based on the deci-
sion tree algorithm to predict the optimal decoding mode in
advance. By simulation, we show that the proposed scheme
predicts the optimal decoding mode with higher than 94%
accuracy even if there exists reliability variation among flash
blocks and wordlines. In addition, the proposed scheme shows
about 61% latency improvement ratio compared to the Static
scheme and achieves the minimum average read latency. From
this, low latency and high reliability can be guaranteed in
numerous flash blocks in spite of process variation.
