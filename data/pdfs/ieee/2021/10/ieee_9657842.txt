Abstract—We designed a FPGA based LDPC Decoder for 
decoding Type-I QC-LDPC codes with flexible code rate and 
code length, by which the channel adaptation could be realized 
with very fine granularity. In addition, the decoder shows good 
bit error rate (BER) performance whose performance loss is only 
0.6dB at the BER of 𝟏𝟎−𝟔, compared with the theoretical 
performance when decoding the same LDPC codes using 
normalized Min-Sum based layered decoding algorithm. 
Keywords—LDPC decoder, FPGA, ACM, broadband satellite 
I. 
INTRODUCTION 
Satellite communication has become an important and even 
the only means to solve the problem of global connectivity, due 
to its characteristics of “high” and “far” [1]. Satellite channel 
environment is complex and changeable, the natural weather, 
such as clouds and rainfall, will result in a large range of 
dynamic attenuation of satellite signals. It is necessary to 
reserve sufficient linkage allowance to ensure the link 
connectivity under the worst channel condition. The sufficient 
linkage allowance ensures the link connectivity, however, it 
greatly reduces the satellite system throughput. 
Adaptive coding and modulation (ACM) is one of the key 
technologies that can maximize the throughput of the satellite 
system under the premise of ensuring link connectivity. 
Channel estimation and prediction, adaptive switching strategy, 
flexible encoding and decoding, and multi-format-compatible 
modulation and demodulation are the core modules of the 
ACM scheme [2]. Adaptive range of signal-to-noise ratio 
(SNR), granularity of demodulation threshold and Estimation 
accuracy of SNR are the key indexes affect the performance of 
the ACM scheme [3]. 
In this paper, we mainly focus on the implementation of 
flexible decoding, which affects both the adaptive range of 
SNR and granularity of demodulation threshold. On the basis 
of the RAM-based architecture of irregular repeat-accumulate 
(IRA) LDPC decoder [4], some core modules have been 
improved so that Type-I QC-LDPC codes with flexible code 
rate and code length could be decoded correctly.In addition, 
layered decoding algorithm [5] is bound to the proposed 
decoder, the number of iterations can be reduced by up to half 
when achieving the same BER performance compared with the 
traditional non-layered decoding algorithm. Normalized Min-
Sum algorithm is used in the horizontal decoding process to 
reduce the complexity of the check node decoding module. The 
error performance of the proposed decoder has been verified in 
a real time point-to-point intermediate frequency loop system, 
and the performance loss is only 0.6dB at the BER of 10−6, 
compared with the theoretical performance when decoding the 
same LDPC codes.

After caching a frame of input data in the LLRs cache

data with given shift value.

successful or the maximum iterations are reached. The 
principle of the layered decoding algorithm can be found in [5]. 
III. 
DESIGN OF THE CORE MODULES 
This section focuses on the design of the core modules of 
the proposed decoder. Three core modules, refer to the LLRs 
cache RAM, barral shifter and check node unit, are carefully 
designed to support the decoding of Type-I QC-LDPC codes 
with different matrix parameters. 
A. LLRs Cache RAM  
The LLRs cache RAM is compose of two ram blocks. The 
two ram blocks work alternately, and they have the same width 
and depth.  
In order to support the caching of multiple Type-I QC-
LDPC codes with different matrix parameters, the width of 
each ram should be determined by the maximum CPM with 
size 𝑝𝑚𝑎𝑥× 𝑝𝑚𝑎𝑥, and the depth should not less than the total 
number of the non-zero CPMs of all the LDPC parity matrices 
required to be stored.

B. Barral Shifter 
We consider applying the multi-size circular shifting (MS-
CS) network [6] to complete the design of barral shifter. The 
basic unit of this network is a single size circular shifting (SS-
CS) network, which is composed of stages of 𝑛= 𝑙𝑜𝑔2(𝑝) 
stage of 𝑝 2-to-1 multiplexers each. Multiplexers on the 𝑖th 
stage of the shifter, 0 ≤𝑖< 𝑛 , are connected in such a way to 
circularly shifter (rotate) the input vector by 2𝑖 if the control bit 
is set; otherwise, the stage is transparent to the input. In this 
way, the arbitrary rotation 0 ≤ρ < 𝑝 is decomposed into 𝑛 
consecutive rotations by powers of 2. An example a 5 × 5 
barral shifter is shown in Fig. 3. 
Obviously, only one CPM size can be supported to realize 
the arbitrary rotation in SS-CS barral shifter. In order to 
support arbitrary rotation for multi-size CPMs, MS-CS 
network should be well designed. 
The core idea of MS-CS barral shifter is the parallel 
cascading of 𝑍= 𝑝𝑚𝑎𝑥/𝐵 SS-CS barral shifters and plus an 
additional stage of 𝑝𝑚𝑎𝑥 Z-to-1 multiplexers, where 𝑝𝑚𝑎𝑥 is the 
maximum block size of all the CPMs. The size of each SS-CS 
barral shifter is 𝐵× 𝐵. 𝐵 is the greatest common divisor 
between the block sizes of all the CPMs.

IV. 
PERFORMANCE VERIFICATION 
This section focuses on the performance verification of the 
proposed decoder architecture. On the one hand, verify the 
decoder’s capability of supporting multiple code rates, that is 
the decoder can correctly decode LDPC codes with different 
code rates; on the other hand, verify the BER performance of 
the decoder. 
Without loss of generality, three normal frame length 
DVB-S2 [7] LDPC codes with different code rates, 1/2, 3/4, 
8/9, are chosen to be implemented. According to the project 
requirements, the decoding parallelism is selected as 60. The 
decoder was implemented on the Xilinx's Virtex-6 platform, 
and the resource utilization of the decoder is shown in Table I.

the designed coder shows good BER performance. In addition, 
since the decoder can correctly decoding LDPC codes with 
different code rates sent continuously, the decoder has the 
capability of supporting multiple code rates. 
V. 
CONCLUSION 
As described in the above sections, on the basis of the 
architecture of the flexible LDPC decoder, by carefully 
designing each core module, multiple code rate supported 
decoder with good BER performance was implemented on the 
Xilinx's Virtex-6 platform. In the subsequent research work, 
the decoder should support decoding more types of QC-LDPC 
codes, verify and improve the decoder’s BER performance in 
high order modulation systems and in different channel 
conditions.
