Abstract—The applying of low-density parity-check codes is considered over two-state channels with memory described by Markov and Gilbert models. The error probability using different strategies of syndrome decoding is estimated. The method of choosing the coset leaders is proposed which provides the error probability close to maximum-likelihood decoding for the considered codes.
Index Terms—low-density parity-check codes, channels with memory, Markov channel, Gilbert channel, metrics
I. INTRODUCTION
In the modern transmission channels the low-density paritycheck (LDPC) codes are widely used. When constructing such codes, it is usually assumed that errors in the channel occur independently of each other, that is, channels without memory are considered, but in reality most systems have memory. Therefore, the issue of considering the decoding of LDPCcodes in channels with memory is relevant.
The standard array of linear code contains all correctable error vectors, which are the coset leaders corresponding to all possible syndromes. The problem of decoding a linear code is reduced to solving an optimization problem for some objective function. Most often, this function is the minimum distance or maximum likelihood. The maximum likelihood, for which the most probable vectors in the channel are selected as the coset leaders, is considered the optimal criterion, but often decoding is performed at the minimum distance in some metric. For channels without memory, the Hamming metric is traditional. When using a metric matched with the channel, the choice of minimum weight vectors as coset leaders ensures decoding according to the maximum likelihood.
To describe channels with memory, models of channels with a finite number of states are used [1], special cases of which are the Markov model and the Gilbert model. There is a metric that is matched with the Markov model — Markov metric. This paper discusses the possibility of using the Markov metric for decoding in the Gilbert channel, as well as other ways to select a set of corrected vectors and the probability of decoding error provided by them.
The paper is structured as follows. The section II discusses some models of channels with memory, their features and approaches to building a list of coset leaders for full decoding. Some approaches to constructing of LDPC codes are described in the section III. The section IV is devoted to the description

of experiments and the results obtained. The section V contains conclusions on the results.

The Markov chains are used to describe models of channels with a finite number of states. This model is called the Gilbert–Elliott channel [2]. It is characterized by the transition probabilities PGB, PBG, and the error probabilities in each of the states: 0 ≤ pG ≪ pB ≤ 1.

C. Gilbert Model
The Gilbert model is also a special case of the Gilbert– Elliott model [4]. In such a channel, pG = 0, and pB can take different values, but traditionally pB = 0.5, so we will use this value later in the article.
We will call the G-burst an error burst containing arbitrary elements between the extreme non-zero characters (see Fig. 3). If the code can correct all G-bursts of length bG, then, similarly to the Markov channel, no two G-bursts of length less than or equal to bG should give the same syndrome when multiplied by the parity-check matrix H. However, this now means that all columns corresponding to these bursts must be linearly independent. To determine the length of the correctable Gburst, there is a polynomial complexity procedure based on calculating the rank of some sub-matrices [5].

C. Evaluation of Methods for Selecting Coset Leaders
The correcting capability of the code is determined by the list of error vectors that the code can correct. Such a list can be obtained by constructing a standard array. In the standard array, each syndrome is associated with one vector from an coset — its leader. The method of selecting the leader depends on the decoding criterion. Optimal from the point of view of the probability of a decoding error per word is maximum likelihood decoding (ML-decoding):

To make a comparison, we can consider not only blockpermutation constructions, but also another way of constructing LDPC-code — the Progressive Edge Growth (PEG) construction [13]. It is based on the Tanner graph, for which the parity-check matrix is the incidence matrix. In this construction, the input of the procedure is the distribution of weights of the variable and check nodes of the graph. The purpose of constructing a PEG-code is to maximize the growth of the minimum cycle in the graph.

To use this criterion, you need to specify a metric — a function for calculating the distance. When using a channel-matched metric, decoding at the minimum distance is equivalent to decoding at the maximum likelihood.
To simplify the construction of the list of coset leaders, we can consider not constructing the entire standard array, but writing out vectors either in descending order of probability or in ascending order of their weight. For the Hamming metric, the procedure for such an assignment is defined quite simply. The procedure for writing out vectors in ascending Markov weight was considered in the II.
Let’s consider the methods of constructing a standard array when transmitting over the Gilbert channel. There are no metrics that are matched with the Gilbert channel, and the construction of vectors in descending order of probabilities would require recalculation when changing the channel parameters. Therefore, we propose the following approach to constructing a set of correctable vectors. Firstly, we will write out the vectors in ascending order of the burst lengths. Secondly, we will order all possible vectors with the same burst length by the number of nonzero elements inside the burst.

V. CONCLUSION
The paper considers the use of low-density parity-check codes in channels with memory. The error probability in the BSC, Markov channel and Gilbert channel is estimated for different strategies of syndrome decoding. It is shown that the use of a metric matched with the Markov channel leads to the worst result for the codes being considered. An approach to constructing a list of correctable vectors for syndrome decoding is proposed, which provides both a lower error probability in the Markov channel compared to the Markov metric, and an error probability close to the probability for maximum likelihood decoding in the Gilbert channel.
