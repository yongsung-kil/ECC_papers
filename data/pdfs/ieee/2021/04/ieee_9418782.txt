Abstract—This paper presents an Improved Offset Min-Sum (IOMS) algorithm for Low-Density Parity-Check (LDPC) decoders, used in 5th generation (5G) mobile communication. The proposed algorithm IOMS applies a multiplication factor to check-node update information messages. The simulation results show that the proposed IOMS algorithm offers a better error correction performance than the existing Min-Sum-based algorithms at the expense of a slight increase in complexity. At the BER of 10−5, the IOMS achieves a decoding gain of 0.1dB with respect to the OMS algorithm.
Index Terms—Offset Min-Sum, Min-Sum, 5G LDPC, Check node, Low-Density Parity-Check.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were first proposed by Robert. G. Gallager in 1962 [1]. They have been receiving much attention for the forward error correction in wireless communication systems because of their excellent performance and efficiency. In the past several decades, they were used in various applications such as WiMAX [2], Digital Video Broadcast (DVB) [3], IEEE 802.11 [4] and Advanced Television System Committee (ATSC) [5]. Especially, they have been selected by 3GPP in the enhanced Mobile BroadBand (eMBB) scenario as the error correction codes for data transmission in 5th generation mobile communication (5G) [6], [7].
An LDPC code, defined by a sparse parity-check matrix, can achieve iteratively between two processing groups named check-nodes (CNs) and variable-nodes (VNs). The most powerful message-passing algorithm is known as the Belief Propagation (BP) algorithm. The BP algorithm achieves the optimal decoding performance very close to the Shannon limit [8] but requires high computational complexity. For this reason, the Min-Sum (MS) algorithm was proposed to decrease the computational complexity by using only the comparison and addition operations [9]. Unlike the BP algorithm, in the CN process, the minimum function of the MS algorithm generates an approximated message. Thus, a significant decoding performance loss of error correction is observed. To compensate for this performance degradation, many variants of the modified original MS

algorithm were introduced such as Offset Min-Sum (OMS) and Normalized Min-Sum (NMS) [10], [11]. Compared to the MS algorithm, these MS-based algorithms improved the performance in exchange for a small increment in complexity. The performance gap between the existing MS algorithms and the BP algorithm gives a space to improve decoding performance. The preceding modified algorithms in the literature [12]–[15] provided a better trade-off between performance and complexity by using MS approximation.
This work proposes the IOMS algorithm for 5G LDPC decoders. Since 5G LDPC code uses variable-nodes of a special low degree, which connect to only one CN. The reliability of check-to-variable messages continuously involves these VNs. Thus, a CN-updated change is considered by multiplying to receive the required result. Based on this modification, the simulation results show that the error correction performance of 5G LDPC decoders is much improved.
The remainder of this paper is organized as follows. Section II presents the proposed Improved Offset Min-Sum algorithm. In section III, the simulation results are discussed. An architecture of a check node unit of LDPC decoder using the proposed algorithm is presented in Section IV and Section V concludes the paper.
II. THE IMPROVED OFFSET MIN-SUM ALGORITHM FOR 5G LDPC DECODER.
A. Notation
Let N denote the length of the codeword and K denote information bit size. The LDPC codes can be presented by a sparse parity-check matrix H with M rows and N columns where M = N − K. The code rate is denoted by R and R = K/N . M rows of H correspond to M check-nodes, while N columns correspond to N variable-nodes. Besides, the LDPC codes can be illustrated graphically using a bipartite Tanner graph [16], where each bit has its variable-node and each parity check has its factor or check-node.

The LDPC decoder can correct the information messages

during transmission. The method of LDPC decoding is to

exchange messages between CNs and VNs iteratively. This

process is repeated until the correct codeword is successfully

detected or the maximum iteration number is reached.

Quasi-Cycle LDPC (QC-LDPC) [17] is the majority

of LDPC codes employed in almost all practical LDPC

applications. QC-LDPC codes are generally specified a

base matrix B of size L × C with entries bi,j ≥

−1 (i ∈ {1, ..., L} ; j ∈ {1, ..., C}) [17]. The parity-check ma-

trix H of a QC-LDPC code can be defined by expanding

its base graph by an expansion factor Z. Thus, each entry

in the base graph is replaced by a sparse matrix of size

Z × Z, defined as follows: entries bi,j = −1 are replaced

by the all-zero matrix and entries bi,j > 0 are replaced by

a circulant permutation matrix, obtained by right-shifting the

identity matrix by bi,j > 0 positions.

Accordingly, 3GPP has agreed to consider two QC-LDPC

base matrices used in 5G, named BG1 and BG2, which

support rate compatible and scalable data transmission for

the channel coding [6], [18]. These matrices have a similar

structure, which is shown in Fig.1, a 1 in the matrix indicates

existence of a base edge. The base matrix BG1 consists

of four parts: the core matrix Hcore, the extension matrix

Hext, an identity matrix I and an all-zero matrix O. The

matrices Hcore and Hext represent the highest rate code and

the low-rate extension. The base graph columns are divided

into three parts: information columns, core parity columns,

and extension parity columns. The first four rows illustrate

core checks, and the others illustrate extension checks. One

special feature in the 5G LDPC is the two left-most columns,

which are punctured variable nodes. These nodes can achieve

improved performance at lower complexity and in practice

they can also have an effect on the number of base check

nodes. Furthermore, the 5G LDPC codes are extremely

irregular: row degree varies from 3 to 19 and column degree

varies from 1 to 30. In this paper, we focus on BG1, which

is a target for larger block lengths and higher rates. Base

graph BG1 has 46 rows, 68 columns, and 22 information bit

columns.

In (9) the optimization of η and γ factors is obtained by combining the Density Evolution (DE) [19] and simulation methods. Without loss of generality, for simplicity, first, γ is fixed to 1, η is optimized by the DE (i.e., same as the OMS algorithm). In this case, the optimal η value is 0.4. Next, with the value of η found in the previous step, to achieve a target BER closer to reality (i.e., the codeword length is finite, instead of assuming infinity as in the DE) the γ factor is optimized through simulation. As a result, η = 0.4 and γ = 0.95 are obtained as the optimal factors for the IOMS at the target BER of 10−5. Compared to the OMS and NMS algorithms, the additional complexity is slightly increased since only one modification factor (i.e., either γ or η) is introduced to the CN-update function.
