LDPC Decoders using Non-Uniform Quantizers and Associative Check Node Operations
Adrian Vidal Electrical and Electronics Engineering Institute, University of the Philippines, Diliman, Quezon City, Philippines
Email: adrian.vidal@eee.upd.edu.ph

Abstract—In this paper, we present a class of associative check node operations for LDPC decoders operating on non-uniformly quantized channel outputs over a binary-input additive white Gaussian noise channel (BIAWGNC). The proposed quantizer has thresholds that depend on the noise variance and can be easily calculated using binary search. A relatively small collection of check node operations is proposed where each operation can be described using a short list of integers. Simulation results using rate-1/2 and 2/3 DVB-S2 codes indicate that the proposed decoder under 4-bit quantization can come close to floating-point sum-product decoder error performance in the waterfall region to within 0.4 dB and 0.1 dB for the rate-1/2 and 2/3 codes, respectively. Moreover, tests with short LDPC codes indicate that the proposed 4-bit decoders may approach or even surpass floating-point error performance at high signal-to-noise ratios.
Index Terms—LDPC codes, iterative decoding, quantization, Gaussian channels
I. INTRODUCTION
Low density parity check codes (LDPC) are a class of capacity-approaching linear codes which were discovered by Gallager [1] in 1961, but were largely forgotten until they were rediscovered by MacKay in 1999 [2]. Due to their sparsity, LDPC codes lend themselves to an iterative decoding method known as the message-passing algorithm [1]. Traditionally, the algorithm takes likelihood ratios as inputs, which are then used as messages that are exchanged between variable nodes and check nodes. The outputs of the decoder are soft decisions which are used to recover the transmitted bits. This paper aims to speed up LDPC decoding by forcing the messages being passed between the nodes to be quantized.
With the introduction of LDPC codes in 5G cellular networks [3], fast decoders have gained importance to support ultra-reliable low-latency communications. To speed up LDPC decoding, various schedules of message-passing have been studied [4][5] to accelerate the convergence of the algorithm. In the sum-product algorithm, check node processing dominates the decoding time due to the use of floating-point hyperbolic tangent function. Various low-complexity approximations have been proposed [6], most notably the min-sum algorithm and its variants [7] with small error performance penalty, but still use floating-point arithmetic.
By limiting the messages to a discrete set we avoid expensive floating-point operations that are typically used in LDPC decoding. We use a non-uniform quantizer with thresholds that are dependent on the noise level. The thresholds are chosen

This approach was earlier explored for maximizing the resulting channel capacity [8], and for information-optimum mapping using the informationbottleneck method [9]. In this work, we use an alternative non-uniform quantization whose thresholds can be calculated easily without using expensive optimization routines.
More recently, several works have been dedicated to implementing low-precision message-passing algorithms using various criteria [10][11], and aim to exploit the use of lookup tables [12][14] to perform low-latency processing at the check nodes. By requiring check node mappings to be associative, the resulting decoder is guaranteed perform equally well regardless of the chosen grouping and ordering of input operands. The proposed collection of mappings can be succinctly described using a short list of integers, which may be useful for memory-constrained decoders that can support larger bit widths.
The contributions of this paper are as follows: we introduce a non-uniform quantization of the BIAWGN channel that quickly approaches the unquantized channel capacity for small numbers of bits per message. We then construct a family of decoders that preserves several properties of the boxplus operator used by check nodes in LDPC decoding.
The rest of the paper is organized as follows. In Section II, we discuss the system model consisting of the BIAWGN with a non-uniform quantizer. In Section III, we calculate the thresholds for the quantizer so that the resulting binary input discrete memoryless channel is weakly symmetric. Section IV develops a class of associative check node mappings over the set of quantized messages, followed by its extension in Section V. Section VI shows the simulation results using several LDPC codes. Finally, we conclude the paper in Section VII.

III. BIAWGN CHANNEL OUTPUT QUANTIZER

If the noise parameter σ is known, traditional LDPC decoders running the sum-product algorithm use the loglikelihood ratio as input to the decoder. In this section, we present a quantization scheme that encodes channel outputs as low-precision messages with little loss in capacity.
To provide tractable information-theoretic guarantees, we set the thresholds in H so that the resulting discrete memoryless channel from X to Yˆ is weakly symmetric. Treating X and Yˆ as rows and columns of the transition probability matrix P (Yˆ |X), weak symmetry implies that all rows are permutations of each other and that all column sums are equal [15]. A similar approach was taken in [8], in which the quantization levels are set to optimize the symmetric capacity of the resulting DMC. Our approach differs in that we solve for the thresholds to guarantee that equiprobable signaling is optimal for the resulting DMC.

VI. SIMULATION RESULTS
To test the performance of our quantized message-passing decoders, we used rate 1/2 and 2/3 DVB-S2 LDPC codes with block length 64800 and run the decoders for 50 iterations. All decoders presented in this section use the flooding schedule. Due to the symmetry of the channel and decoder [13], the bit error rate is independent of the transmitted codeword. Hence, we perform density evolution (DE) on the all-zero codeword, iteratively calculating the probability mass functions of the quantized messages. We further assume that the messages are independent, but we do not assume that they are identically distributed owing to the irregularity of the code. In the next round of simulations, we increased the description length to 4. For finer quantization levels, e.g. k ≥ 6, lookup tables may require too much memory, and calculation of the check node outputs can be slower than when we have the description length equal to 1. However, when the check node operation is implemented as a lookup table, the increase in description length should have no effect in the decoding speed. As Figs. 5 and 6 show, the longer description length closes the gap between the proposed decoders and the floatingpoint sum-product decoder to around 0.4 dB for the rate 1/2 code, and is very close at 5-bit quantization for the rate 2/3 code.

VII. CONCLUSION
In this paper, we presented a collection of associative check node operations that can be described using a short list of integers. We also presented a non-uniform quantizer whose thresholds can be easily calculated and with capacity at least as much as that of optimal uniform quantizers using at least four bits. The imposed structure on the check node operations allowed us to select good quantized decoders on the basis of bit error rate estimates from density evolution over the Tanner graph. Simulation results with long DVB-S2 LDPC codes show that 4-bit quantized decoders can be chosen from the collection that perform as close as 0.45 dB and 0.05

dB in the waterfall region for rate 1/2 and rate 2/3 codes,
respectively. Tests with short rate 1/2 LDPC codes indicate that
the performance gap between floating-point and our proposed
4-bit decoder narrows for shorter blocklengths, and may even
surpass the error performance of the sum-product decoder at
high SNR.
For future work, we consider looking at the combinatorial
structure of the check node operations to find good decoders
efficiently. Although the number of possible decoders in the
collection are small enough to allow individual testing, BER
estimation using density evolution is relatively expensive.
