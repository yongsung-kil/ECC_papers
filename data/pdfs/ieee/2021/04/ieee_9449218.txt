Abstract—5G data channel is latency-sensitive and demands 
high throughput. The Min-Sum (MS) algorithm, known as one of 
the LDPC Soft-Decision decoding algorithms, is widely used with 
guaranteed performance. However, the MS algorithm’s latency 
and resource consumption are non-negligible drawbacks, 
especially for 5G system. In this paper, different strategies for FEC 
(Forward Error Correction) of LDPC codes are discussed. An 
improved Weighted Hard-Decision algorithm is proposed 
targeting to reduce resource occupation and latency. The decoding 
process is given in detail and performance is validated via 
simulations. Comparing with the MS algorithm and IHD 
(Improved Hard-Decision) algorithm, our proposed algorithm 
shows its potentiality to be applied in 5G data channel. 
Furthermore, a test bench is launched in Cisco server using X86 
chipset to identify the advantage of this proposal.

I. INTRODUCTION 
To fit the requirements of 5G NR, the architecture of mobile 
network should provide higher throughput and lower latency. 
To also simplify deployment and maintenance, the solution in 
our context is C-RAN(Cloud Radio Access Network) [1]: The 
BBU (Base Band Unit) is centralized by moving the antenna 
sites to the CO(Central Office) which is also named as BBU 
hosteling or BBU pooling.  Therefore, only RRHs (Remote 
Radio Heads) are placed on the antenna sites. FECs (forward 
error codes) are suitably employed to ensure service quality in 
5G NR transmission.   
Our project aims to build 5G small cells that can apply in 
intelligent factories. For end to end test, the system is composed 
of BS (Base station) and UE(User Equipment) as shown in Fig. 
1. Three elements are in the BS side: BBU, RHub (Radio Hub), 
RRUs (Remote Radio Units). BBU and RHub are connected via 
10G fiber Ethernet using ECPRI (Enhanced CPRI). Rhub and 
RRUs are connected by 10G fiber using CPRI (Common Public 
Radio Interface). 5G CPEs (Customer Premise Equipment) are 
represented in the UE side, respectively. 
The RRU consists of a digital Chip: Xilinx Zynq 7000 and 
an analog Chip AD9009. To increase the coverage, maximum 8 
RRUs can be attached to one RHub which processes low 
physical layer modules (IFFT/FFT, subcarrier mapping/de-
mapping, etc.). RHub is connected to BBU which is 
implemented in a Cisco server and is exploited by X86 Chipset. 
An FPGA Board (aiding LDPC coding/decoding) is added in 
the BBU. This system, whose components are standardized by 
3GPP Release-15 aims at achieving a throughput of 2 Gb/s. 
 
Fig. 1. System model. 
The objective of this paper is to study different types of 
LDPC (Low Density Parity Check) codes[3][4] and improve the 
decoding algorithm which is proposed in our recent works[13] 
to fit better our proper 5G architecture (good trade-off among 
performance, latency and resource utilization). 
Generally, Soft-Decision algorithms and Hard-Decision 
algorithms are used to represent two famous categories in 
LDPC codes. Soft-Decision algorithms which show excellent 
performance consist of exchanging messages between 
individual processing nodes iteratively such as BP(Belief 
Propagation) algorithms, MS(Min-Sum) algorithms and their 
variants[6]-[8]. On the other hand, Hard-decision algorithms 
converge faster and perform not bad in regular PCM (Parity 
Check Matrix) such as BF (Bit-Flipping), WBF (Weighted BF), 
GDBF (Gradient descent BF) and their variants[9]-[13]. Hard-
decision algorithms commonly use the inversion function that 
computes node reliabilities to flip the most unreliable variable 
nodes to correct the sequences. Due to less information 
exchanging between variable nodes and check nodes, Hard-

A. Encoding Algorithm 
Commonly for LDPC codes, a matrix 𝐺 where (𝐻×
𝐺) 𝑚𝑜𝑑 2 = 0  must be presented for generating the coded 
sequence. The encoding algorithm is the most complicated 
process in the transmission module because of matrix operation 
[5]. To reduce the complexity of the circuit, the special parity 
check matrix shown as Fig. 2 is also used for input blocks 
encoding [15]. It is a mix of double diagonal and diagonal 
matrices. When the first parity column has been derived, the 
remaining parity can be calculated as each new layer is only 
adding one new column of parity. More comments, parity check 
bits can be computed from zone B and zone D. Parity check bits 
of zone B are obtained using systematic bits of zone A. 
Systematic bits presented in zone A and single parity check bits 
calculated from zone B will be utilized in zone C to compute 
parity check bits with the elements in zone D.

B. Algorithm Proposed 
In this part, we propose an IWHD (Improved Weighted 
Hard-Decision) algorithm. The algorithm is enhanced from our 
recent proposal [13], and is also based on the MS algorithm. 
To reduce latency and respect fundamentals of the Hard-
Decision algorithm, two ameliorations are mentioned before 
proposing the algorithm: part-matrix-decoding, early stop.  
 
 
Fig. 6. Part-matrix-decoding.  
Hard-Decision algorithm always deals with matrix-length 
bits due to its fundamentals: not decide the bits as Soft-Decision 
algorithms but flip the bits. In the 5G Uplink data channel, the 
whole matrix rows are always taken to decode the sequences. 
According to the code rate, even though the data length is 
always less than column length, we add some padding bits to 
match the matrix’s size for decoding. Those padding bits will 
not contribute to the decoding process and sometimes introduce 
additional errors.  As shown in Fig. 6, Part-matrix-decoding 
using real data length is suitable and necessary for our proposed 
algorithms.   
Early stop is to output decoded bits earlier while the 
maximum number of iterations is not achieved but no more 
errors in received sequences. The computation of syndromes is 
used in the first four rows which consist of information bits. As 
mentioned in the Hard-Decision algorithm [8]-[13], if all 
syndromes are equal to zeros, output decoded bits. Otherwise, 
we continue the decoding process until syndromes verified or 
maximum iteration achieved.  
 
 
Fig. 7. MS decoding procedure. 
In Fig. 7, two steps can be concluded for the MS decoding 
algorithm: horizontal and vertical. As described in (4), the 
horizontal step uses the global function to find minimum values 
of variable nodes and manipulate XOR function to obtain their 
sign to give the first metric of check nodes. For the vertical step, 
all metrics of check nodes are conditionally combined and 
finally gives the decision of each variable node.

V. CONCLUSION  
In this article, two common categories of LDPC codes are 
presented with descriptions of the encoding and decoding 
processes. The performance for different code rates of MS 
algorithm is given. The number of average iterations with 
different SNR is obtained as well. To improve performance and 
accelerate the decoding process, an enhanced Hard-Decision 
algorithm which is named IWHD (Improved Weighted Hard-
Decision) algorithm is proposed. We also illustrate the 
fundamentals and validate its performance.  Furthermore, A 
LDPC test bench is built in the Cisco server using X86 chipset 
to confirm the advantages of the proposed algorithm. The 
results show IWHD algorithm can achieve better performance 
with less resource utilization comparing to the IHD algorithm.  
In future research, our proposal will be implemented in an 
FPGA accelerate board to reduce more resource occupation and 
latency. Meanwhile, an architecture of FPGA is going to be 
proposed.
