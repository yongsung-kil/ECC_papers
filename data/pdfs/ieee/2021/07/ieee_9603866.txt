Abstract—In this paper,the logarithmic domain efficient decoding algorithm of LDPC code has been deeply studied,and the efficient algorithm with tree topology structure has been improved.The fixed-point representation and reweighting analysis of the receiving and processing information of LDPC code decoding has been carried out.At the same time,based on Quartus development software platform and Verilog HDL language,the efficient parallel decoding algorithm of regular LDPC code with length of 8designed by geometric construction method is designed, and the circuit design and simulation results of some main modules are given.The logic synthesis and implementation of the whole algorithm are carried out on StatixL EP2S60F484C3.The highest data processing rate is 10Mbps,and the error correction performance is ideal.After the reprocessing method is adopteda,ll the information in the decoding implementation process is the same standard,and there is no need for additional processing , which reduces the hardware complexity and calculation delay,and meets the needs of engineering implementation.

Keywords-F PGA; LDPC; code; decode

I.

INTRODUCTION

LDPC code is a packet error correction code with sparse check matrix proposed by MIT Robert Gallager in 1962. It is suitable for almost all channels. Its performance is close to Shannon limit, and its decoding can be operated in parallel. It is suitable for hardware implementation. At present, it has become the channel coding and decoding mode of 5G communication [1] [2] [3] [4]. The earliest LDPC codes defined by Gallager were codes of code length 1, and the number of
"1" in each row and column in its checksum matrix was fixed. The number of "1" in each column is A(A > 3) , the number of

"1" in each row is ( , and the number of "1" overlap between columns is less than or equal to 1[4].

B. Quantification of Efficient Algorithms
Because there is no core operation £(U@/’) in the improved efficient decoding algorithm, the information that needs to be quantized is only the core operation functions g(x) = logd+ el) in the core operation 1(U @V), in addition to decoding input information 2y/o*. The same standard quantification rules are applied to both types of information. That is, using 8-bit information to represent decimal digital information with symbol bits. And the first bit represents the symbol bit ("0" means "+" "I" means "-"), 2~4 bits represent integer bits, the last four digits represent the fractional part, so the range of values is 7.9375~+7.9375. Besides, because 8(=*lo)gd+ é"") is a number between 0~0.69, just use four bits to represent the decimal part, and the first 4 bits are all "0". Uniform quantitative criteria for initial and intermediate information, so all information operations no longer need to be handled separately in the hardware implementation of the information processing process, can be handled directly. To some extent, it also simplifies the steps of information processing in the process of hardware implementation.

two core operation modules of variable node update and check node update which are partially parallel processed, the interlacing processing module of information transfer between the two core operation modules, and the decoding decision output processing module. The update of variable node and check node adopts partial parallel processing, but the sequence tules of data input and output of the two modules are not consistent. 3) Intertwined information transfer processing module. The iterlace_module mainly completes the interleaving process of information transmission between the variable_module and the check_module. It ensures that both the variable_module and the check module can read the readout information in their correct order when the decoder partially processes the information in parallel. The judge module module completes the decision termination of the whole iterative decoding process
and stores the hard decision information of each iteration, and outputs the decision information of the iteration termination as
the decoding result after a fixed delay according to the decision termination signal. The ctrl_module generates the clock
needed by each functional module in the system, that is, the address, enable and zero signal needed by the storage module in
the system, ensures the normal operation of each module, and makes the whole system work in coordination.

CONCLUSIONS

LDPC code is a kind of superior performance error correction coding method, in all kinds of communication system has a broad application prospect and great application of power, has been used in satellite communication, 5G mobile

In this paper, the logarithmic domain efficient decoding algorithm of LDPC code is studied deeply, and the efficient algorithm with tree-like topology structure is improved. The fixed-point representation and quantitative analysis of the receiving and processing information of LDPC code decoding are carried out. At the same time, the software platform is developed based on Quartus and Verilog HDL language is used to design the efficient parallel decoding algorithm of (1008,3.6) regular LDPC code with girth of 8 designed by geometric construction method, and the circuit design and simulation results of some main modules are given. The whole algorithm is logic integrated and realized on Statix II device EP2S60F484C3. The resource occupation is as follows: logic elements
