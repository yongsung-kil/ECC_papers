Codes approaching the Shannon limit with polynomial complexity per information bit
Ilya Dumer and Navid Gharavi University of California, Riverside, USA Email: dumer@ee.ucr.edu, navid.gharavi@email.ucr.edu

2021 IEEE International Symposium on Information Theory (ISIT) | 978-1-5386-8209-8/21/$31.00 ©2021 IEEE | DOI: 10.1109/ISIT45174.2021.9517788

Abstract—We consider codes for channels with extreme noise that emerge in various low-power applications. Simple LDPC codes with parity checks of weight 3 are first studied for any code dimension m → ∞. These codes form modulation schemes: they improve the original channel outputs for any SN R > −6 dB (per information bit) and gain 3 dB over uncoded modulation as SN R grows. However, they also have a floor on the output bit error rate (BER) irrespective of their length. Tight lower and upper bounds, which are virtually identical to simulation results, are then obtained for BER at any SNR. We also study a combined scheme that splits m information bits into b blocks and protects each with some polar code. Decoding moves back and forth between polar and LDPC codes, every time using a polar code of a higher rate. For m → ∞ and a sufficiently large parameter b, this design yields a vanishing BER at any SNR above the Shannon limit of -1.59 dB and has complexity order of m log m per information bit.
I. INTRODUCTION
In this work, we address code design that protects informa-
tion transmitted on the AWGN channels with extreme noise.
One important future application is the Internet of things (IoT).
To efficiently employ it, current NB-IoT standards [4] are supposed to achieve a 20 dB reduction in the snr per channel bit over the current LTE networks (below snr = Es/N0 denotes the signal-to-noise ratio per channel bits and SN R = Eb/N0 is the ratio related to information bits).
From the theoretical standpoint, we consider binary linear codes C(n, k) of length n → ∞ and dimension k used on the BI-AWGN channels N (0, σn2 ) with noise power σn2 → ∞. To achieve any fixed signal-to-noise ratio SN R = 1/ 2σn2 Rn , these codes must still operate even when σn2 has the growing order of Rn−1. The fundamental Shannon limit shows that any binary code may achieve a vanishing BERs only if Rn → 0 and SN R > ln 2 (equivalently, -1.5917 dB). Given the gap ε = SN R − ln 2 to the Shannon limit, the two important figures of merit include complexity Φ(n, ε) and BER P (n, ε).
To date, it is an open problem to design efficient Shannon-
limit-approaching codes. In particular, a number of existing constructions have code rates Rn that decline exponentially in code dimension m. For growing m, this yields an exponential
growth in length, bandwidth, and decoding complexity. For example, biorthogonal codes C(2m−1, m) achieve the
Shannon limit; however, their code rate Rn = m/2m−1 declines exponentially in m. By contrast, the output word
error rates (WER) of these codes experience very slow decline, which is only polynomial in blocklength n. In particular, for low SN R ∈ (ln 2, 4 ln 2), codes C(2m−1, m) have WER [1]

Consider also low-rate RM codes RM (m, r) of order r = o(m) and length n = 2m. These codes approach channel capacity [5], [6] under maximum-likelihood (ML) decoding. However, ML decoding has huge complexity of exponential order 2(m r ) per information bit. Other efficient algorithms [7][9] known for RM codes still yield high error rates for low SN R < 1 dB or require unacceptably large decoding lists.
Finally, consider polar codes [10], [11] of rate Rn → 0 that operate at a constant SNR. One such construction [12] uses µ = O(log2 Rn−1) upgrading channels in the beginning, which form a repetition code of the growing length 2µ. This again calls for finding the minimal length at which polar codes achieve a gap ε to the Shannon limit. It is known [13] that such a length is a polynomial in ε−1. It is also important to further bound decoding complexity (n log n) /k per information bit.
For moderate lengths, one efficient design [14] concatenates a repetition code of length b = 4 with a (2048,40) polar code aided by the list of size 16. This gives WER of .002 for SN R = 2 dB and improves [4] by 1 dB. Another recent design [15] - called coded repetition - slightly modifies b polar codes used in b repetition blocks. This scheme can further improve performance on the noisy channels. Finally, paper [16] employs a two-stage decoding. The first stage uses a long low-rate LDPC code, which corrects information bits only. The second stage further protects information block using a short polar code. The scheme slightly improves on [14] by reducing three times its output WER. Below we advance codes of [16] using new multilevel code design and new techniques for estimating BER of the belief propagation (BP) decoding.

V. CONCLUDING REMARKS
Summary. We studied low-rate “modulation” codes Cm that employ parity checks of weight 3. BP decoding was applied to information bits only. Then codes Cm were modified to employ frozen information bits. The asymptotically exact bounds were obtained for the output BER of BP decoding. It turned out that the latter achieves ML-performance for codes Cm. Code Cm was further combined with b short polar codes Bs of length m/b. This combination was aided by the backand-forth data correction. The resulting codes of dimension k approach the Shannon limit ln 2 with complexity O(k ln k) per information bit. It can also be shown that an ε-gap to the Shannon limit is achieved with complexity poly(ε−1).
Work in progress. Detailed calculations show that our current design becomes practical only for the large lengths n ≥ 217. Our ongoing work replaces codes Cm with the shorter codes taken from the Gallager ensemble. This redesign can achieve the Shannon limit using ln2 k operations per information bit at the expense of a larger latency. This newer design also becomes more practical. In particular, new codes of length 213 and code rate of 0.05 achieve BER of 10−4 at SN R ∼ 1 dB using 32 candidates without CRC. This yields a 2 dB gain over the designs of [14], [16] and is on par with the best CRC-aided polar codes that use similar parameters [23]. An important problem is to further extend this design to the codes of lower rates that can withstand a higher noise per channel bit.
