FPGA Implementation of LDPC Decoder Architecture for Wireless Communication Standards

Ruslan Goriushkin, Pavel Nikishkin, Evgeny Likhobabin and Vladimir Vityazev, Member, IEEE
Department of Telecommunications and foundations of radio engineering Ryazan State Radio Engineering University, RSREU Ryazan, Russia
gorushkin.r.s@tor.rsreu.ru, nikishkin.p.b@tor.rsreu.ru, likhobabin.e.a@tor.rsreu.ru, vityazev.v.v@rsreu.ru

Abstract—This paper presents a decoder design for QuasiCyclic (QC) Low-Density Parity-Check (LDPC) codes. The design is parameterized and can be easily rebuilt to support various LDPC Parity-Check matrices taken from the WiMAX (IEEE 802.16e) and the WiFi (IEEE 802.11n) standards. New techniques such as parallelization of the decoding architecture cores are proposed. These cores calculate variable-to-check (VTC) and new check-to-variable (CTV) messages and also update posterior probabilities (APPs). The parallel multicore decoding architecture implies a prior shift of values based on the LDPC matrix and simultaneous calculation of values for the core. Our decoder is implemented on FPGAs of the Zynq-7000 Mini-ITX Evaluation Board (XC7Z100-2FFG900). The throughput of up to 1,2 GBit/s and the operation frequency of up to 240 MHz have been achieved.
Index Terms—Communication systems, Quasi-Cyclic codes, Low-Density Parity-Check codes, error correction codes, parallel decoder architecture, iterative decoding, min-sum (MS) algorithm, FPGA implementation.
I. INTRODUCTION
The Low-density parity-check (LDPC) codes concept was developed by Gallager in 1963 [1]. LDPC codes had been forgotten until the Gallager’s work was rediscovered by MacKay and Neal [2] in late 1990’s. Currently, there is a high interest in LDPC codes due to their excellent error correcting capability and their performance able to achieve up to 0.0045 dB of the Shannon’s limit at a BER of 10−6 [3]. These codes have become part of important communication standards such as IEEE 802.11n/ad/ay/ax (WiFi) [4], IEEE 802.16e (WiMAX) [5], and ETSI 5G [6]. They are mentioned in digital video broadcasting standards such as DVB-T2, DVB-S2, and DVBC2. Nowadays LDPC codes also find place in other fields, including error correction techniques for solid state drives (SSD).
A number of works devoted to parallel LDPC decoding architectures have been published in the last decade [7] - [10]. Only paper [8] presents a universal parametrizable decoder. The decoders proposed in [7], [9], [10] are based only on one communication standard or one codeword length. The authors of the papers widely use BRAMs to store the decoder internal values.
This work was supported by Russian Science Foundation under Grant 1779-20302.

In this paper we provide the parallel decoder implementation for QC-LDPC codes, which meets the requirements of modern specifications for throughput, and has the flexibility to work with various code rates and lengths. The proposed LDPC decoder can operate with matrices similar to IEEE 802.11 and IEEE 802.16e. Most of the previously proposed architectures do not pay much attention to the way the sparse parity check matrix is stored. So, the new way of storing the parity check matrix is proposed. It allows to reduce the amount of memory needed to store the code matrix.
Moreover, the details and results of implementing the above mentioned architecture on the FPGA part of the Zynq-7000 Mini-ITX Evaluation Board (XC7Z100-2FFG900) are provided.
The paper is organized as follows: the next section presents the overview of the decoding algorithm used herein. The proposed FPGA architecture of the LDPC decoder is described in section III. The hardware implementation results are presented in section IV. Finally, Section V concludes this paper.
II. MIN-SUM DECODING ALGORITHM
Low-Density Parity-Check (LDPC) codes are a class of linear block codes. As the name implies, they exhibit the linearity property and are applied to source data blocks.
Any LDPC code can be represented as a check matrix which contains a small number of ones and all other elements are filled with zeros. A regular (N, K) LDPC code is represented as a matrix H which includes M ≥ N − K rows and N columns. The M rows of H are defined as check nodes, and all the elements of a LDPC codeword as variable nodes.
The min-sum algorithm (MSA) is a simplified Belief propagation algorithm (BP) built by reducing complexity of the unit responsible for check node updates (CNU). The following definitions are given for the MSA algorithm: cn indicates the n-th bit of a codeword and yn is a value received from the channel. The check-to-variable (CTV) message rmn[k] is a message from the check node m to the variable node n during the k-th iteration. The variable-to-check message (VTC) qmn[k] is a message from the variable node n to the check node m. III. PROPOSED DECODER ARCHITECTURE
A. Decoder architecture definitions
The general structure of the proposed LDPC decoder is shown in Figure 1.
The soft decisions from the demodulator are used as input to the decoder. The input serial bitstream is converted into a parallel and is written to the block memory. For ease of parallel APP reading and writing, the block memory is divided into K = N/z blocks. The number of these blocks is determined by the length of the codeword (N ) and the size of the circulant (z). The size of each block depends on the number of symbols in the circulant and the width of the input data.
The Iterations counter block makes a decision on completion of the decoding process and output of the result. When a given number of iterations is reached, its output is set to a high level.
The Operating mode selection block is used to switch the operating mode to ”decoding” or ”result output” and to organize output of the decoding result. When the high level is received at the input from the Iterations counter block, the “result output” mode is switched on: a high level is set at the READY output, and the decoded word is obtained at the output. Otherwise, the next iteration is to be run.
The VTC calculation block is used to calculate the variableto-check messages based on APPs from the BRAM APP block. A simplified expression for calculating variable-tocheck messages is shown in expression (5).

B. Storage of the parity-check matrix, the CTV node and the APP in memory
A H matrix of the QC-LDPC codes taken from the WiFi and WiMAX standards is quasi-cyclic, so it consists of submatrices generated by circular shifting of an identity matrix. Only base parity-check matrixes Hb are stored for QC-LDPC decoder implementations. It allows to reduce memory consumption for matrix set storage significantly. The parity check matrices Hb used herein are specified in [5]. To further reduce the memory usage and to ease reading of matrix elements, they are stored in three vectors inside the proposed decoder.
The vectors for storing the matrix are shown below:
• vals is a vector of all matrix elements which is other than -1.
• pos is a vector containing the matrix column index of each element from vals;

IV. IMPLEMENTATION RESULTS
The implementation results show that the decoder can operate with any LDPC matrices of the WiMAX (IEEE 802.16e) and WiFi (IEEE 802.11n) protocols. The decoder can obtain the maximum decoding throughput of 1.2 Gpbs for one iteration (the codeword length is equal to 2304).
