Abstract—Since low-density parity-check (LDPC) codes have been studied and adopted in the DVB-S2 standard, many efforts have been made to improve their performance and optimize them up to the broadcasting standard. This paper proposes a family of protograph-based LDPC codes to provide a multi-rate coding for efficient DVB-S2 systems, considering low-complexity and excellent performance of the protograph codes. Moreover, we employ Bit Interleaved Code Modulation Iterative Decoding (BICM-ID) to enhance the performance of the ASK-modulated DVB systems. Both the protograph extrinsic information transfer (PEXIT) analysis and simulation results demonstrate the performance advantage of the proposed protograph-based LDPC codes over the conventional DVB-S2 codes for all the code rates. In particular, the proposed codes yield performance gains of more than 0.2 dB as compared to the conventional codes at high rates for 16-APSK modulations.
Index Terms—Low-density parity-check codes, protograph codes, DVB-S2, PEXIT, BICM-ID.
I. INTRODUCTION
Low-density parity-check (LDPC) codes are a kind of capacity approaching forward error correction (FEC) codes, which can perform very close to the Shannon limits over the additive white Gaussian noise (AWGN) channel [1]. LDPC codes have been widely adopted in broadcasting strandards, e.g., DVB-S2 strandard [2] and DVB-T2 strandard [3]. The application of LDPC codes in Bit Interleaved Code Modulation (BICM) system has been deeply studied [4], [5].
DVB-S2 is a satellite standard developed on the basis of DVB-S and DVB-DSNG. DVB-S2 LDPC short modes define 10 code rates, and each code rate defines a different LDPC code. The codeword length of the normal and short modes are uniformly specified as 64800 bits and 16200 bits, respectively. Although the encoding mechanism of DVB-S2 LDPC codes has similar properties and structures, the structure of the only

To solve this problem, a rate-compatible quasi-cyclic (QC) LDPC codes design method was proposed [6], which can provide an efficient multi-rate coding scheme. Also, a new threshold analysis method was introduced to predict the gap between Shannon limit and the decoding SNR threshold of practical QC-LDPC codes [7]. Ref. [8] presented three padding and puncturing schemes for DVB-S2 LDPC codes to obtain coding rates equivalent to DVB-SH specification.
Protograph LDPC codes was first proposed by the American JPL laboratory [9]. This kind of LDPC codes is constructed from a small base matrix, which is then expanded to LDPC parity-matrix by improved PEG (Progressive Edge-Growth) algorithm [10]. The protograph LDPC codes have shown excellent performance over the AWGN channel [11]. The protograph extrinsic information transfer (PEXIT) chart has been proposed to analyze the decoding threshold and performance [12], [13]. In particular, the QC-protograph codes are attractive for low-complexity implementations, because they can be coded by simple-feedback shift-registers [14]. In [15], three new types of protograph LDPC codes were proposed to achieve low error floor for 1-D intersymbol interference (ISI) channels.
In this paper, we propose a family of protograph LDPC codes for DVB-S2 systems considering its simple implementation and excellent performance. Guided by the PEXIT analysis and the empirical rules, we design protograph matrices of different code rates with low decoding threshold. Furthermore, we consider BICM-ID scheme in the LDPC coded DVBS2 with high-order modulations. Both the PEXIT analysis and simulation results show that the proposed LDPC codes perform better than the conventional codes for all the code rates, and the gain of up to 0.2 dB can be achieved at the high code rates.

II. SYSTEM MODEL AND PROTOGRAPH CODES A. DVB-S2 LDPC System Model
Fig. 1. The block diagram of the DVB-S2 BICM-ID system .
Fig. 1 shows the block diagram of the LDPC coded DVBS2 with BICM-ID scheme. The source binary bits are encoded by LDPC encoder and then divided into a q-bit stream. Each q-bit is mapped into a 2q-APSK constellation symbol by Gray labeling. At the receiver, the demodulator extracts the logarithm likelihood ratios (LLR) for each bits in the qbits from the channel observations, which are fed into the LDPC decoder. Then iterative message-passing between the demodulator and the decoder can be performed by using these soft messages. It is worth noting that an LDPC coded modulation system, such as the DTMB standard, may support both single-carrier and OFDM modulation modes and involve a symbol interleaver to combat multi-path and fading channels.
In the DVB-S2 standard, the LDPC codes adopt a unique coding structure that mixes regular codes and irregular codes. The LDPC encoder converts a block of information length K into a codeword of length N . Note that the DVB-S2 standard provides 11 sets of encoding parameters with code length of 64800 and 10 sets of encoding parameters with code length of 16200. DVB-S2 LDPC codes employ a multirate mode which has the same codeword length but different number of information bits and check bits. Since the paritycheck matrices are designed individually at each code rate, they have similar structures and properties but not exactly the same. Therefore, we follow the design guidelines of DVB-S2 LDPC code that the basic matrices of the protograph codes corresponding to each code rate are independent.
B. Protograph LDPC Codes The protograph can be seen as a Tanner graph with a rela-
tively small number of nodes [9]. A protograph is composed of variable nodes, check nodes and edges, each edge connects a variable node and a check node. However, unlike LDPC codes, the protograph codes allow parallel edges. We represent the adjacency matrix corresponding to the protograph as a base matrix. By copying and replacement operations on the base matrix, we can further derive a large protograph. The resulting

protograph LDPC codes can be decoded at a high speed, and provide excellent performance at high code rates in AWGN channels [11], [16]. After T copies of the protograph, the endpoints of the T copies of each edge should be replacement between the T copies of the variable nodes and the check nodes [17].

A. PEXIT Chart Analysis

A protograph G = (V, C, ε) consists of a series of D variable nodes V , a series of F check nodes C, and edges ε. Each edge ei,j ∈ ε connects a variable node vj ∈ V to a check node ci ∈ C. A protograph can be described by a base matrix B of size D × F . The element bi,j of B represents the number of edges connecting the variable node vj to the check node ci.
The PEXIT algorithm presented an initialization step [12], which provides channel information for a given channel parameter. The decoding threshold is the limit value of the channel parameter that allows the decoder to successfully converge to the error probability Pe = 0 [18].

IEc (i, j) : the MI between the message sent by ci to vj and the associated codeword bit, on one of the bi,j, edges connecting ci to vj. At the receiver, the channel decoder directly decodes the channel information, and the information is updated between the variable nodes and the check nodes. It is found that the degree-2 variable nodes in the protograph are committed to good error performance in the waterfall region, but too many degree-2 variable nodes may also introduce an error floor in the high SNR region [21], [22]. Hence, the desired protograph of low error floor for DVB-S2 standard may have a small proportion of degree-2 variable nodes but not too large [23], [24]. Based on these observations, we consider the rules of designing the protograph code for the DVB-S2 standard as follows
1) To preserve the linear minimum distance growth property of the protograph having degree-2 variable nodes, we limit the maximum number of the degree-2 nodes to be the total number of checks minus 1. This ensures that no cycle is formed with only degree-2 variable nodes.

V. CONCLUSIONS
This paper proposes a family of protograph-based LDPC codes for DVB-S2 systems. Guided by PEXIT analysis, we design protograph matrices of low decoding threshold for different code rates, respectively. Both the PEXIT analysis and simulation results show that the proposed LDPC codes can achieve performance gains over the conventional DVBS2 codes for all the code rates, and the gain is up to more than 0.2 dB as the rates increases. Moreover, for 16APSK modulations, the iterative decoding between the demodulation and the decoder can further enhance decoding performance.
