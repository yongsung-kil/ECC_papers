Counter Random Gradient Descent Bit-Flipping Decoder for LDPC Codes
Keyue Deng, Hangxuan Cui, Jun Lin, and Zhongfeng Wang School of Electronic Science and Engineering, Nanjing University, Nanjing, China Email: kydeng@smail.nju.edu.cn, haxcui@163.com, jlin@nju.edu.cn, zfwang@nju.edu.cn

Abstract—Tabu-list random-penalty gradient descent bitflipping (TRGDBF) is a hard-decision algorithm for decoding low-density parity-check (LDPC) codes, which offers a significant improvement in error correction. However, compared to the current gradient descent bit-flipping (GDBF) variants, it converges slower and has no obvious performance advantage unless enough iterations are allowed. To accelerate the convergence speed, this paper presents a counter random GDBF (CRGDBF) algorithm in which a bit is forbidden to flip in the next iteration after being flipped several times. Incorporating the random operation, the forbidden-mechanism shows a dynamic property, facilitating the decoder more efficiently to break trapping sets. Simulation results show that the CRGDBF can achieve up to 5 times better decoding performance than the TRGDBF algorithm within finite iterations. Additionally, an architecture is presented for implementing the CRGDBF algorithm, demonstrating it only brings limited hardware overhead compared to the TRGDBF.
Index Terms—Low-density parity-check codes, counter, random operation, tabu-list, bit-flipping, high-performance decoder.
I. INTRODUCTION
LOW-DENSITY Parity-Check (LDPC) codes have gained considerable research attention since first introduced in 1962 by Gallager [1], due to their near-capacity error correction capability. The soft-information message passing methods such as Sum-Product (SP) [2], Min-Sum (MS) [3], and Normalized Min-Sum (NMS) algorithms [4] provide good performance at the expense of the heavy calculation burden. Instead, the hard-decision decoding algorithms such as Bit-Flipping (BF) [5] algorithm require less computation resources but lead to non-negligible performance loss. BF algorithm is easier for hardware implementation because of the simple computation units. Many researchers have made every effort to boost up its error-correction performance while keeping the low complexity.
In general, BF-based algorithms decode by flipping one or more bits based on the inversion function, which indicates the reliability of each bit. The Gradient Descent Bit Flipping (GDBF) algorithm formulates the inversion function as a gradient descent problem [6] and it was adapted to Binary Symmetric Channel (BSC) by Rasheed et al. in [7]. As a variant of the GDBF, the Probabilistic GDBF (PGDBF) further improves the performance by flipping the candidates with a predefined probability of p0 (p0 < 1). However, there still exists a noticeable performance gap between PGDBF and softinformation algorithms. Inspired by their work, Cui et al. proposed the tabu-list aided PGDBF (TPGDBF) algorithm [8]

and the tabu-list random-penalty GDBF (TRGDBF) algorithm [9]. The tabu-list was designed to avoid repeated flips in several consecutive iterations, which significantly reduces the decoding failures. In addition, the random-penalty term λ (λ equals 0 or 1) introduced in the TRGDBF can help the decoder escape from trapping sets (TSs). Although the TRGDBF can achieve very low error-rate, it converges slower than the current GDBF variants.
In this paper, we propose an algorithm named Counter Random GDBF (CRGDBF) algorithm. A flip counter is designed by observing the misjudgments of the trapping sets. With the help of the counter, a bit will be added to the tabu-list in a more reasonable rule, which speeds up the decoding convergence. Our simulation results show that the CRGDBF algorithm offers a better performance than that of the TRGDBF algorithm, especially within finite iterations. Besides, we provide an architecture to implement the CRGDBF algorithm. Compared to the TRGDBF decoder, only about 30% additional hardware resource is needed. Moreover, the CRGDBF increases the throughput because of the faster convergence speed.
The rest of the paper is organized as follows. Section II presents the notations and reviews the TRGDBF algorithm. In Section III, the CRGDBF is described in detail. Section IV presents the simulation results including performance comparisons. The hardware architecture is presented in Section V. Finally, Section VI concludes the paper.
II. PRELIMINARIES
A. Notations
Generally speaking, a binary linear LDPC code can be defined by an M ×N parity check matrix H, where N > M ≥ 1. H is a sparse matrix containing mostly zeros and just a small number of ones. The code rate R = 1 − M/N . An LDPC code can also be represented effectively by a bipartite graph whose nodes can be separated into the variable nodes (VNs) and the check nodes (CNs). Element hij = 1 corresponds to a connection between the i-th check node and j-th variable node, i ∈ [1, M ], j ∈ [1, N ]. The set N (i) = {j : hij = 0} contains the VNs that participate in parity check equation i and |N (i)| is called the degree of the j-th CN. Similarly, the set M(j) = {i : hij = 0} denotes the neighborhoods of the i-th VN and |M(j)| is the degree of the i-th VN. u is transmitted through a channel and we denote y = {yj} as the channel output.

They will not be selected to flip in the next followed iteration and will be released from the tabu-list after the next iteration.

III. THE COUNTER RANDOM GDBF ALGORITHM
With the help of the tabu-list, TSs have been largely avoided. The number of decoding failures can be reduced indeed by the rule that prevents the bit from being flipped twice consecutively. But this rule is so stringent that the bit may be added to the tabu-list even if it doesn’t belong to the TSs. It can be seen in [9] that compared to the current GDBF variants, the TRGDBF converges slower. Hence, designing a more precise method to determine when the bit should be added to the tabulist is meaningful.

Based on the discussion above, the CRGDBF algorithm is proposed, which is presented in Alg. 1. The number of times a bit has been flipped before being added to the tabu-list is defined as χj, χj ∈ [1, θ]. For the sake of the simplicity of the algorithm, we introduce the parameter ηj, where P (ηj = 1) = p and P (ηj = 0) = 1 − p. p is a predetermined probability value and p ∈ [0, 1]. The value of χj can be set with a certain probability, where P (χj = 1) = p, P (χj = 2) = (1 − p)p, · · · , P (χj = θ) = (1 − p)θ−1. Through this operation, not only some bits that are flipped once or more and belong to the TSs are taken into account, but also the bits with same phenomenon but not belong to the TSs are considered. And the decoder does not need to know which bits belong to the TSs because of the random operation. Lj is the value of j-th counter and lj is used to mark whether the j-th bit is in the tabu-list or not. It is clear that the TRGDBF is an exceptional case of the CRGDBF when parameter θ or η is set to 1.

VI. CONCLUSION
In this paper, we propose an algorithm named Counter Random GDBF algorithm. In order to accelerate the convergence speed, a bit-flipping rule containing the counter and random operation is introduced. A bit will be added to the tabu-list after being flipped one or more times, which makes CRGDBF more efficient to break TSs. Simulation results demonstrate that the CRGDBF algorithm offers a faster convergence speed and better performance than that of the TRGDBF algorithm. We also implement the CRGDBF algorithm and present the synthesis results. Compared to the TRGDBF decoder, only small additional hardware resources are needed. Generally, the proposed CRGDBF is a promising decoder with highperformance and high-throughput characteristics.
