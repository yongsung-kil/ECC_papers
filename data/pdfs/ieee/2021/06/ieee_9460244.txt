Efficient BP-based Decoding Algorithms for QC-LDPC Codes

Konstantin Zavertkin, Anastasia Panarina, Aleksei Ovinnikov and Evgeny Likhobabin Department of Telecommunications and Radio Foundations Ryazan State Radio Engineering University Ryazan, Russia 390005
Email: zavertkin.k.n, panarina.a.a, ovinnikov.a.a, likhobabin.e.a@tor.rsreu.ru

Abstract—In this paper we propose to use a self-correction technique with two computationally efficient LDPC decoding algorithms: QRCBP and adjusted min-sum. It is shown that using such a technique allows to improve performance of this algorithms both in error floor and waterfall regions. Also comparison with other BP-based simplified decoding algorithms is presented for three quasi-cyclic LDPC codes.
Index Terms—LDPC, quasi-cyclic, self-correction.
I. INTRODUCTION Low density parity check (LDPC) codes were proposed by R.Gallager [1] in 1963, and later rediscovered by MacKay and Neal [2]. Like turbo codes, LDPC codes can achieve near the Shannon limit error performance [3]. Nowadays they are widely used in many applications and modern standards such as DVBT2, DVB-S2, WiFi, WiMax. The main drawback of LDPC codes is high complexity of the suboptimal belief propagation (BP) decoding algorithm proposed by R.Gallager. And nowadays there are a lot of algorithms for decoding of LDPC codes, which provide a wide range of implementation complexity and achievable error performance. Attractive tradeoff between error performance and decoding complexity is achieved by BP-based decoding algorithms. Among them the most popular are min-sum (MS) [4] and a-min* [5] decoding algorithms with various modifications. The other significant drawback of LDPC codes is the presence of error floors in BP decoding for some code constructions. One of them are quasi-cyclic LDPC (QC-LDPC) codes. It limits their usage in applications that need extremely low error rates. The error floor can be characterized as an abrupt decrease in the slope of a code’s performance curve. Error floors were first discovered by MacKay and Postol in [6] for (2640,1320) Margulis code. It was shown that the performance of LDPC codes with message-passing decoding in the low-error-rate region is determined by the multiplicity and structure of the trapping sets [7], rather then minimum distance of the code. It was shown [8] that using self-correction (SC) modification from [9] can reduce trapping set influence on the performance of decoder in error floor region. And as was shown in [10] improve efficiency of decoding in waterfall region. In this paper error performance for adjusted MS [11] and quantized reduced complexity box-plus (QRCBP) [12] decoding algorithms was investigated for QC-LDPC codes in

II. DEFINITIONS AND NOTATIONS Any LDPC code is specified by a parity-check matrix which contains mostly zeros and only a small number of ones. A regular (N,K) LDPC code, with block length N and dimension K, has a parity check matrix H of M ≥ N - K rows and N columns with exactly dc ones in each column and dr ones in each row. We refer to the M rows of H as check nodes, and to all the elements of a LDPC codeword as variable nodes. Then each check node is connected to dr variable nodes and each variable node is checked by dc check nodes. Also define N(i) – the set of ones in i’th column, M(j) – the set of ones in j’th row of matrix H. N (i) − {j} and M (j) − {i} are the same sets, but without j’th and i’th positions respectively.
III. BP-BASED DECODING ALGORITHMS Both considered algorithms are simplified implementations of the BP decoding algorithm. QRCBP decoding algorithm can be considered as a modification of the a-min* algorithm, and adjusted MS as a min-sum-like approximation of the BP algorithm.
A. Belief propagation decoding algorithm BP decoding algorithm is a suboptimal decoding algorithm,
that can achieve maximum-likelihood performance. BP decoding algorithm has high computational complexity as far as it utilize transcendental functions for check node processing. So reasonable decision is simplify Gallager’s BP decoding algorithm with control of decoding performance degradation.

B. Min-sum decoding algorithm Min-sum algorithm [4] is one of the most popular decoding
algorithms for LDPC codes. MS algorithm can be considered as a simplification of operation for log-likelihood ratios (LLR) summation in parity-check node decoder. For two independent binary random variables a1 and a2 with probabilities p(ai = x) = p(xl), x ∈ {0, 1} and LLRs Li = L(ai) = log(p0(l)/p1(l)). It can be shown, that LLR of the binary sum A2 = a1 + a2, defined as:
L(A2) = sign(L1)sign(L2)min(|L1|, |L2|) + g(L1, L2). For MSA only the first part of this sum computed as:
L(A2) = sign(L1)sign(L2)min(|L1|, |L2|). C. A-min* decoding algorithm
Although MS algorithm has a very attractive complexity, but in some applications error performance degradation in comparison with BP decoding may be unallowable. In such a case a-min* [5] algorithm can be very useful, as a combination of BP and MS decoding. A-min* algorithm reduces complexity of check nodes update in comparsion with BP decoder. Consider min-sum approximation:
L(A2) = sign(L1)sign(L2)min(|L1|, |L2|) + g(L1, L2) = = sign(L1)sign(L2)min∗(L1, L2). For MSA decoding operator min* is equal to
min∗(L1, L2) = min(|L1|, |L2|). And for a-min* algorithm:
min∗(L1, L2) = min(|L1|, |L2|) + g(L1, L2). Where expression min(|L1|, |L2|) + g(L1, L2) is called a box-plus operator. Box-plus operator also can be approximated by any appropriate function, in this paper in addition to a-min* we considered a-min* with one line approximation of box-plus operator from [13]. For a-min* algorithm as is for MS, in every check node only two outgoing messages are computed. The first one is for the bit with the lowest reliability, and the second one for all other bits. D. QRCBP decoder The QRCBP is an approximation of Flarion/Qualcomm decoder [12], which uses a reduced complexity box-plus operator. The main goal of QRCBP decoder is representation of a boxplus operator by a two-input look-up table. This table contains the integer representation of box-plus operator. Another innovation of QRCBP decoder is an optimization of check-node update method. The QRCBP decoder uses a variant of a-min* algorithm [5] to perform check-node message updates, but the QRCBP algorithm does not emulate a-min* exactly. The first difference is the approximation of box-plus operator. In addition to the use of the table, there is another performance tweak. Referring to the description of the a-min* algorithm, note that the first step in the algorithm is to find the variablenode message of minimum magnitude and label it vi#. This process requires dc − 1 comparisons. Then, in the a-min*

algorithm, the calculation of β is performed considering only the adjacent variable-node messages that do not emanate from vi#. This process requires full dc comparisons, where β message to all variable nodes except the least reliable variable node. In the QRCBP algorithm these two steps are merged into one so that only dc comparisons, rather than 2dc − 1 comparisons, are required to compute β [12].

E. Adjusted min-sum algorithm Adjusted min-sum algorithm provides high-performance de-
coding of LDPC codes by approximating the update function and determining magnitudes of outgoing log likelihood ratios (LLRs). Adjusted MS provides approximation of the two arguments function U :

U (A, B) ≈ (min (A, B) − f (|A − B|))+

(1)

where the + operation is given by x+ = x if x > 0 and x+ = 0 if x ≤ 0, f = ln(1 − e−x). Then the check node magnitude update may be defined in a nested fashion as:

Li = U (ld, ..., U (li+1, U (li−1, ..., U (l1, l2))))

(2)

A significantly better approximation of the U function, Uˆ (A, B), may be obtained by letting M = min (A, B) and takes the form:

Uˆ (A, B) ≈ M − f (|A − B| + h(M ))

(3)

where h(x) may be defined as ln(coth(x)) or −ln(1 − e−2x) or some value in between. For the approximation Uˆ (A, B) of the basic update function U (A, B) the minimum of A and B, denoted as M , should be computed at first. Then, the correction term which consists of difference of the two incoming LLR magnitudes |A − B| and a function h with a single argument which is applied to M are computed. At last the correction term quantity is subtracted from M .

V. SIMULATION RESULTS
Research of the performance was carried out for three codes with different code lengths, that were presented in [14], and all of them have error floors for BP decoding. Maximum number of iterations is 50. Random codeword transmission is assumed for BPSK modulation.
Simulation results for QRCBP and adjusted MS algorithms are depicted in figures 1, 2, 3 in comparison with BP-based decoding algorithms from [8]. Simulation results for SC modification of QRCBP and adjusted MS are depicted in figures 4, 5 and 6. In all figures solid lines are used for bit errorrate performance curves and dotted lines for word error-rate performance curves.
It can be seen, that the best performance is achieved with QRCBP, a-min* line and adjusted MS decoding algorithms, that outperform MS and a-min* algorithms in both waterfall and error floor regions. Applying of SC modification allow to improve error performance of each considered decoding algorithm. The best performance is achieved with a-min* line SC algorithm. QRCBP SC and adjusted MS SC algorithms are a little bit worse. It can be see, that applying SC modification for a-min* line SC algorithm has more effect than for QRCBP decoding algorithm.

VI. CONCLUSION AND FUTURE WORK Application of SC technique to QRCBP and adjusted MS algorithms for QC-LDPC codes decoding provides some positive changes. And such modification have performance gain both in waterfall and error floor regions. Error performance of such modifications is a little bit worse than for a-min* line SC algorithm. But none of the considered algorithms overcomes the error floor. In future work, it is planned to investigate the behavior of the considered algorithms when decoding the codewords with triggered trapping sets.
