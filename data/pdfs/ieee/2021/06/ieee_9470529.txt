Abstract—This article examines constructing blockpermutation low-density parity-check codes for error burst correction. Block-permutation constructions with superimposed zero masks are considered. We have analyzed the approach of construction such matrices consisting of two stages: finding the location of zero blocks and finding the base matrix. Experimental results containing analysis of correction capability of considered codes over the channel with memory are presented.
Keywords — low–density parity-check codes, block-permutation construction, burst error correction
I. INTRODUCTION
Nowadays, a huge amount of information is transmitted through various communication channels. Information has become an integral part of almost all areas of our life: mobile communications, the internet of things systems, satellite systems, etc. During the signal’s transmission from the source to the receiver, interference can occur, which leads to errors in the transmitted sequence [1]. In order to reduce the number of errors on the receiver error correcting codes are used. One of the most popular type of error-correcting codes are lowdensity parity-check codes that provide a low error probability and low complexity of coding and decoding procedures at the same time.
Most real communication channels have an effect that is often called «memory», which, as a rule, is not taken into account when constructing codes for modern telecommunications standards. Performance of low-density parity-check codes over channels with memory has been an actual topic of research for a long period of time, however, the task of constructing low-density parity-check codes for channels with memory remains open and contemporary. This article discusses the block-permutation design of low-density parity-check codes when some zero blocks inserted in the code structure instead of cyclic-shifts blocks. Also, the error correction capability of such constructions is examined.

II. BLOCK-PERMUTATION LDPC-CODES
Low-density parity-check codes (LDPC-codes) – are linear codes with a low density of parity checks. There were proposed by R. Gallager in 1962 [2], [3]. Low-density paritycheck codes can be specified by the parity-check matrix H. Parity-check matrix contains a small number of non zero entries. In most modern communication standard, the blockpermutation construction of codes is used. For example, BPCs are used in 802.11 standard (Wi-Fi) standard [4], 5G standard [5]. The probability of error provided by existing LDPC-code decoders in a channel with independent errors depends on the structure of the so-called Tanner graph, for which H is an incident matrix. This structure is known to provide less error probability if zero blocks are added to the block permutation matrix. However, the effect of inserting zero blocks in H was not analyzed for channels with memory before.

IV. DESCRIPTION OF EXPERIMENTS
As it was shown earlier, a block-permutation design without zero blocks cannot correct error packets greater than m – 1. The construction of block permutation matrices with zero blocks can be represented as the overlay of a mask on the base matrix. In this mask, each position corresponds to the block that might be replaced with zero block. In the future, the mask will be represented as a decimal number.
The following method of constructing block-permutation structures for channels with memory is used in the work:
1. A set of x random base matrices Hbase with a size of γ × ρ is generated.
2. N random binary masks are generated for the selected base matrix. For each mask, the maximum length of the bmax packet is calculated.
3. The masks are selected in order to achieve the largest increase in the length of the correcting error burst.
4. After the best mask is found, random base matrices are generated for which their correction capabilities are determined.

VI. CONCLUSION
In this article, the construction of block-permutation structures using zero blocks was considered. A method is considered, which selects mask for matrix with small size of block. Considered method allows to increase the bmax to

approximately 1.9 million for matrices with parameters γ = 4, ρ = 7, m = 48 and to 1.8 million for matrices with parameters γ = 4, ρ = 7, m = 73for matrix with large size of block.
