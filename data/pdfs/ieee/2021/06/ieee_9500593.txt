Abstract—In recent years coarsely quantized LDPC decoding using a flooding schedule has been extensively studied. However, there exist few works addressing coarse quantization for a layered schedule, which enables improved convergence speed of the message passing algorithm. The layered schedule can especially be beneficial for high throughput applications like fiber optical systems. This paper presents innovative layered decoding approaches, where the information bottleneck method is used for the design of different coarsely quantized decoder architectures. The varieties of investigated node implementations include lookup tables, computational domain techniques as well as reduced complexity approximations. All structures are designed offline using a layered discrete density evolution method. The performance of multiple node architectures is investigated in terms of evolution of mutual information in the design phase and in terms of error rates. We focus in this paper on regular quasi-cyclic codes. Our simulations running on GPUs also allow insights into the error floor behavior.
I. INTRODUCTION
In modern communication systems low-density parity-check (LDPC) codes play a very important role to mitigate transmission errors. Apart from being capacity approaching, their major advantage over competing techniques is that the decoding process is highly parallelizable enabling high decoding throughput designs.
An innovative discrete decoder design was proposed in [1] and a similar approach in [2], where mutual-informationpreserving mappings are used for the node operations. Instead of carrying probabilities in log-likelihood ratio format, the nodes exchange integers which encode the underlying probabilities. In [3] the design of this decoder was identified to be an application for the information bottleneck (IB) method. The IB method enables to compress observed messages while preserving relevant information, leading to a reduced space complexity in the decoding process.
Further complexity reduction can be reached by changing the message passing schedule. One very promising approach is known as layered decoding, where the number of required iterations is approximately reduced by half [4]. This is achieved by performing partial node updates within a single iteration.
In [5] an unrolled layered decoding architecture has been proposed, where compressed messages are exchanged between variable and check nodes. However, the nodes are not explicitly designed to maximize preserved relevant information, but

II. LAYERED SCHEDULING WITH QUASI-CYCLIC CODES
The basis for most LDPC decoders is the sum-product algorithm [7], also known as belief propagation, where messages representing probabilities of the code bits x ∈ X = {0, 1} are exchanged over a graph like the one shown in Fig. 1. The exchanged messages are updated in the variable nodes with degree dv and check nodes with degree dc. The variables x = [x1, x2, . . . , xN ]T share information about each other due to the encoding procedure at the transmitter. On a Tanner graph the code constraints are represented by a routing network, whose structure is defined by the parity check matrix H, such that Hx = 0. From an implementation point of view, well structured LDPC codes are preferred over randomly constructed codes, since they offer better routing characteristics leading to decoder designs with much higher decoding throughput. LDPC codes are decoded with lowest decoding latency when using a flooding schedule, where in a single iteration all check node updates and successively all variable node updates are performed in parallel. A single full variable node update is shown in Fig. 2a, where the messages N = {yc1→v, . . . , ycdv →v} from all connected check nodes and the channel message ych are used to generate dv output messages tv→c1 , . . . , tv→cdv .

III. LAYERED IB DECODER DESIGNS
For hardware implementations messages exchanged over the Tanner graph have low resolutions to reduce the space complexity. Restriction to messages of finite alphabet can be exploited to design node operations, which maximize the mutual information in every update step. We use discrete density evolution [1] and the IB method to achieve this [11]. Compared to the flooding decoder design, the layered decoder design additionally involves tracking the distributions of messages in memory. In combination with the IB method, this is done for the first time to the best of our knowledge.
Subsequently, we assume a binary input AWGN channel with a symmetrical IB quantizer for all IB decoders, as in [11]. The design Eb/N0 determines the initial joint distribution p(x, ych) used for local optimizations of the node operations.

B. Check Node Designs
The check node update for the layered schedule is designed similarly to that in a flooding schedule. In [11] the twoinput lookup table structure was used to break down the global optimization problem. In [12] a computational domain approach is provided, which we have slightly modified to enforce symmetrical distributions in discrete density evolution. Moreover, [2] proposed to replace the check node operation with the min-approximation for reduced hardware complexity.

V. COMPLEXITY CONSIDERATIONS
Beside the error rate performance, one very decisive characteristic is the required complexity of each individual solution. The layered schedule reduced the number of required iterations for similar performance approximately by half, compared to the flooding schedule. In contrast, the layered schedule requires additional memory elements to restore previous intermediate results. This amount of additional memory depends on the particular implementation details.
Accurate estimates concerning the hardware complexity of the flooding and layered IB decoders are part of future research. Especially characteristics like area and energy efficiency of an ASIC or FPGA decoder design are of interest.
However, we can make some initial considerations when being restricted to w-bit messages for the individual node designs. The first approach for the variable node involves dv − 1 two-input lookup tables as shown in Fig. 4a. Exploiting symmetry, each lookup table requires at maximum w · 2w+(w−1) bits, which are 96 bits if w = 3 and 512 bits if w = 4. In the computational domain approach we consider the structure of Fig. 4b. In each partial variable node update there must be dv messages translated which requires 2(w−1)(w −1) bits per translation table, assuming a symmetrical distribution of messages and w as the internal bit width. We have dv − 1 sum-operations where the largest adder must support up to log2(dv·(2w −1)) bits at the output. Finally, the symmetrical quantizer can be realized by using 2(w−1) thresholds with (w − 1) bits and a comparator like it is shown in [12].
At the check node we investigated three structures. All approaches share the same hardware required for computing the sign bit. In the two-input lookup table approach we can exploit the symmetries in each two-input lookup table. Then, an upper bound for the complexity in terms of required memory is (w − 1) · 22(w−1) bits, which are 32 bits if w = 3 or 192 bits if w = 4. However, further logic optimization of the lookup tables can reduce these numbers. The lookup table approach with its two-input operation might be arranged such that 3(dc − 2) two-input operations take place. The minapproximation requires only to compute the first and second smallest input magnitude by performing dc + log2(dc) − 2 compare-operations [15]. The OMSQ check node update comprises additionally two comparison and subtraction steps for the offset operation. The computational domain approach in the check node can be implemented using dc translations, dc − 1 additions, dc subtractions and dc quantization steps.
VI. CONCLUSION
In this paper, the information bottleneck method has been applied to design the node operations of a layered LDPC decoder. We analyzed the mutual information and error rate performance of multiple node configurations. Above the error floor region all 4-bit IB decoders are able to exceed the performance of conventional 4-bit OMSQ decoding by 0.06 − 0.14 dB. Our results showed that the mutual information tracked during density evolution is a valid performance

indicator for moderate error rate regions. However, when
getting closer to the error floor region of the code, the
preserved mutual information is not the dominant performance
factor since the density evolution model does not take into
account code properties like trapping sets. Interestingly, the
min-approximation in the check node was frequently able to
lower the error floor by 1 to 2 orders of magnitude. The
3-bit IB-decoder might be a good candidate for future high
decoding throughput systems with reduced space complexity
and performance close to conventional 4-bit decoding.
