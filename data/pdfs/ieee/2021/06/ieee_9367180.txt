MET-LDPC Code Ensembles of Low Code Rates With Exponentially Few Small Weight Codewords
Suhwang Jeong , Student Member, IEEE, and Jeongseok Ha , Senior Member, IEEE

Abstract‚ÄĒ This work studies the design of MET-LDPC code ensembles at low code rates with good threshold performances and exponentially few small weight codewords. There was a notable study on the condition, so called the ‚Äėt-value condition‚Äô, for exponentially few small weight codewords. Meanwhile, it was shown that by introducing degree-one variable nodes, the threshold performances of MET-LDPC code ensembles of low rates can be significantly improved. However, the degreeone variable nodes may result in small weight codewords and thus must be carefully introduced to MET-LDPC code ensembles. Despite the importance, the existing t-value condition was developed only for MET-LDPC code ensembles without degree-one variable nodes. This work extends the t-value condition to MET-LDPC code ensembles with degree-one variable nodes, which includes the existing work as a special case. The extended t-value condition provides useful insights into the contributions of degree-one variable nodes to the distribution of small weight codewords. Thus, the results of this work allow us to design MET-LDPC code ensembles of low rates with both good threshold performances and exponentially few small weight codewords. In addition, it will be demonstrated that MET-LDPC codes at finite lengths based on the designed code ensembles have good error-rate performances both in the waterfall and low-error-rate regions.
Index Terms‚ÄĒ Error correction, error correcting codes, channel coding.
I. INTRODUCTION
L OW-DENSITY parity-check (LDPC) codes were first proposed by Gallager in [1] and later rediscovered by MacKay and Neal in [2] as capacity-approaching codes over binary-input memoryless symmetric channels. Richardson et. al in [3] demonstrated how to design capacity-approaching LDPC codes using the density evolution technique. As a general framework of LDPC codes, Richardson and Urbanke in [4] proposed a new class of LDPC codes called multiedge type LDPC (MET-LDPC) codes in which edges between bipartite nodes, i.e. variable and check nodes, are of several

The multiple edge types in MET-LDPC codes make their structure richer and enable them to embrace variants of LDPC codes such as concatenated tree codes [5] and protograph-based codes [6] as special cases of MET-LDPC codes.
There have been extensive studies [7], [8] on designing MET-LDPC code ensembles which show notable performance improvements in terms of threshold over a wide range of code rates as compared to those of standard LDPC codes. Meanwhile, the average weight distributions of MET-LDPC code ensembles have been studied in [9]‚Äď[11]. In particular, Kasai et al. [10] derived a condition called the ‚Äėt-value condition‚Äô which MET-LDPC code ensembles have to satisfy for them to asymptotically have exponentially few codewords of small weights. The t-value condition turns out to be especially useful in the sense that the error-rate performances in the low-error-rate region can be notably improved by designing MET-LDPC codes of finite lengths based on ensembles satisfying the t-value condition.
Recently, error-correcting codes at very low code rates, e.g. ‚Č§ 0.1, have been gaining a great deal of attention due to some applications requiring error-correcting codes of low rates such as machine-to-machine communications and continuous variable quantum key distribution (CV-QKD) systems [12], [13]. While standard LDPC codes have been known to achieve capacity-approaching performances, they have poor performances at low rates [14] and thus are not appropriate for applications requiring error-correcting codes at low rates. To overcome the limit of standard LDPC codes at low code rates, repeat accumulate (RA) and Raptor codes were studied in [15] and [16], respectively. Meanwhile, Andriyanova and Tillich in [14] revealed the role of degree-one variable nodes in sparse-graph codes at low rates and showed that by introducing degree-one variable nodes, the threshold can be significantly improved. The authors in [14] also proposed a necessary condition for a sparse-graph code to have linear minimum distance. In [17], Otmani et al. studied necessary conditions which must be fulfilled to have LDPC codes with linear minimum distance. However, the conditions in both [14] and [17] are not directly applicable to MET-LDPC codes. In [13], it was shown that MET-LDPC codes with degree-one variable nodes are well suited for the key reconciliation of quantum cryptography which is conducted with error-correcting codes of low rates. While the RA [15] and MET-LDPC codes [13] with degree-one variable nodes have improved performances in the waterfall region, both suffer from performance losses in the low-error-rate region due to high error floors.

It seems natural to apply the t-value condition [10] to the design of MET-LDPC code ensembles with degree-one variable nodes in hopes of designing MET-LDPC codes with improved performances in both the waterfall and low-errorrate regions. However, unfortunately, the t-value condition in [10] assumes that MET-LDPC code ensembles have variable nodes of degrees larger than one and thus is not applicable to ensembles with degree-one variable nodes. Motivated by the limit of the existing t-value condition, in this work, we extend the t-value condition in [10] to MET-LDPC code ensembles with degree-one variable nodes, and thus the result in this work includes the t-value condition in [10] as a special case.
The remainder of this work is organized as follows. We first develop an extended t-value condition for MET-LDPC code ensembles with degree-one variable nodes. The extended t-value condition derived in this work provides useful insights into how the degree-one variable nodes affect the distributions of small weight codewords. The insights are further discussed by exploring examples of MET-LDPC code ensembles. Then, we design MET-LDPC code ensembles with degree-one variable nodes at low code rates, i.e. 0.1 and 0.02, based on the design rule in [8] with/without the constraint of extended t-value condition. To compare error-rate performances, MET-LDPC codes are implemented at finite lengths based on the designed code ensembles. Their error-rate performances are evaluated on a binary-input additive white-Gaussian noise (BI-AWGN) channel in terms of word-error rate (WER) and bit-error rate (BER), which clearly demonstrates that the MET-LDPC codes with the ensembles satisfying the extended t-value condition derived in this work have good error-rate performances both in the waterfall and low-error-rate regions. On the contrary, the ones based on the ensembles unsatisfying the extended t-value condition result in poor error-rate performances in the low-error-rate region, which mainly stems from decoder error events, i.e. decoding into wrong codewords, associated with small weight codewords.

V. PERFORMANCE EVALUATIONS

In this section, we compare error-rate performances of MET-LDPC codes with degree-one variable nodes based on ensembles with/without satisfying the t-value condition. The error-rate performances are measured in terms of both word-error rate (WER) and bit-error rate (BER). In addition, the practicality of the MET-LDPC codes is compared in terms of three metrics, i.e. the maximum variable node degree dmax, the maximum check node degree gmax, and a normalized edge density in [23] which is defined as the average number of edges per message bit, i.e. |E|/(R ¬∑ n) where |E|, R, and n are the total number of edges, code rate and code length, respectively. The maximum degrees are often used as a measure of decoding latency [24] while the normalized edge density is adopted to measure the decoding complexity in [23]. The MET-LDPC codes are implemented at two different lengths, i.e. 104 and 2 √ó 104, with the progressive edge growth (PEG) algorithm [25].
First off, we show that an MET-LDPC code without satisfying the t-value condition has a poor error-rate performance in the low-error-rate region. To this ends, we design an MET-LDPC code ensemble of rate 0.02 which is optimized to minimize the threshold for a BI-AWGN channel using the design rule in [8], and the degree distribution pair for the ensemble is denoted by Ref. 1 in Table II. Note that the designed MET-LDPC code ensemble, i.e. Ref. 1, does not satisfy the t-value condition (see inf T = 0.9873 < 1) and neither does the LDPC part in Ref. 1. As discussed in Section IV, we can turn the Ref. 1 ensemble into a new ensemble which satisfies the t-value condition by carefully tuning g3,2 and d2,2 values. Now, instead of tweaking existing ensembles to make them meet the t-value condition, we will design MET-LDPC codes targeting at good performances in both the threshold and lowerror-rate regions by performing the code optimization [8] for the threshold performance with the constraint of t-value condition. This work adopts the design rule in [8] since it efficiently rules out invalid ensembles in the design process, which significantly accelerates the code optimization. In addition, the design rule does not require the assumption of check node degree distributions in the concentrated form [26] and thus searches for the optimized code ensembles without any restriction on the search space. In particular, we exclude ensembles from the population in the differential evolution [27], [28] when they do not satisfy the t-value condition. An MET-LDPC code ensemble of rate 0.02 is optimized for a BI-AWGN channel and denoted by C1 in Table II, respectively. For performance comparisons, we also take a code ensemble of rate 0.02 from [12] which is denoted by Ref. 2 in Table II. Note that Ref. 2 has t-value of 0.9743 < 1 which does not meet the t-value condition while the designed code ensemble, C1 has its t-value of 1.1528, and thus the t-value condition is met. The thresholds of the code ensembles, Ref. 2 and C1, listed in Table II look very close each other. To evaluate error-rate

performances, MET-LDPC codes for the two ensembles are implemented at a length of 2 √ó 104, and their error-rates are compared in Fig. 4 where the codes for Ref. 2 and C1 have comparable performances in the waterfall region as predicted by their thresholds. However, it is clearly noticed that the MET-LDPC code for C1 drastically outperforms that of Ref. 2 in the low-error-rate region. The decoder-error events are also witnessed in the error-rate evaluations for the MET-LDPC codes based on Ref. 2 while the one for C1 does not have. The two ensembles are also compared in terms of complexity in Table II where it is noticed that C1 has smaller complexity than Ref. 2 in terms of dmax, gmax, and the normalized edge density while it provides the unparalleled error-rate performance advantage in the low-error-rate region.
