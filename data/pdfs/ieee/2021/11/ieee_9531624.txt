Across-Array Coding for Resistive Memories With Sneak-Path Interference and Lognormal Distributed Resistance Variations
Panpan Li , Guanghui Song , Member, IEEE, Kui Cai , Senior Member, IEEE, and Qiyue Yu , Senior Member, IEEE

Abstract— Resistive random-access memory (ReRAM) is a promising non-volatile memory technology for achieving highdensity and high-speed data storage. However, its crossbar array structure leads to a severe problem known as the sneak path interference (SPI), which is data dependent and correlated within a memory array. Meanwhile, variations of the memory fabrication process also cause deviation of memory cell resistances from their nominal values, which typically follows the lognormal distribution. In this letter, we first propose a cascaded channel model that incorporates these two key factors that affect the reliability of ReRAM. By analyzing the channel correlation of ReRAM caused by the SPI, we develop a novel scheme to estimate the probability of each memory cell to be affected by the SPI, based on which the log-likelihood ratio (LLR) of each channel bit can be generated. Moreover, we propose a novel two-step acrossarray bit allocation scheme, which distributes the low-density parity-check (LDPC) codewords to multiple memory arrays so as to minimize the correlation of the channel coded bits for decoding. Simulation results show that the proposed across-array bit allocation scheme with LDPC coding can achieve significantly better error rate performance than the prior art coding schemes, with similar implementation complexity and latency.
Index Terms— Resistive memory, sneak path, lognormal distribution, across-array coding scheme.
I. INTRODUCTION
T HE recently emerged solid-state non-volatile memory (NVM) technologies offer lower power consumption, faster read access time, and better mechanical reliability than the conventional magnetic and optical recording systems. Among various NVMs, the resistive random access memory (ReRAM) is widely considered the most promising technology due to its superior features of high-density, highspeed, long data retention, and good compatibility with the current CMOS technology [1]. It changes the resistance state of a memory cell to represent the binary-input user data: the high resistance state (HRS) corresponding to logical-0 and the low resistance state (LRS) corresponding to logical-1. The ReRAM cell is typically located in a crossbar array, which offers a huge density gain. However, a fundamental

drawback of the crossbar array structure is the sneak path problem [1]. Sneak paths are undesirable paths parallel to the target cell that is being read. The current goes through the sneak paths and decreases the measured resistance value. Such interference caused by the sneak path is data dependent and correlated within a memory array. Also, it is harmful only when the target cell is in HRS. Meanwhile, due to the non-uniformity of the memory fabrication process, there is a deviation of memory cell resistances from their nominal values. This random variation of the low and high resistances of ReRAM is another key factor that affects its reliability [2].
Several works have been proposed in the literature to tackle the sneak path problem from channel coding and detection perspectives. Y. Cassuto et al. [3] studies the maximum data storage efficiency of ReRAM when the constrained codes are employed to completely avoid the sneak paths. This scheme will incur a high code rate loss. More sophisticated sneak path models are considered in [4], based on which the data stored in each memory cell is detected separately by treating the sneak path interference (SPI) as an independent and identically distributed (i.i.d.) noise. In [5], data detection schemes by using the side information provided by the pilot cells are developed, which become unfeasible when the array size is large. In [6], an across-array coding scheme is proposed which can achieve a high data storage efficiency of less than 0.1 bit/cell from the capacity limit. The letter demonstrates that the coded bits from different arrays experience independent channels, and hence a diversity gain can be obtained during decoding. The root low-density parity-check (LDPC) codes [7] designed for the block fading channel are constructed based on similar ideas. However, they are low-rate codes that are not suitable for the data storage channels. In addition, in all the above prior art works for ReRAM, random variations of the cell resistances are assumed to be Gaussian distributed. However, experimental data shows that the resistance variation of ReRAM typically follows the lognormal distribution [2].
In this work, we propose a novel across-array coding scheme with LDPC code to mitigate both the SPI and the lognormal distributed resistance variations of the ReRAM channel. In Section II, we present a cascaded channel model for ReRAM. In Section III, we analyze the channel correlation of ReRAM caused by the SPI, based on which a novel scheme to estimate the probability of each memory cell to be affected by the SPI is developed. In Section IV, we propose a novel two-step across-array bit allocation scheme to minimize the correlation of the channel coded bits for decoding. The simulation results are illustrated in Section V. The letter concludes in Section VI.

II. RERAM CHANNEL MODEL

We consider an M ×N memory array of ReRAM where the

memristor that locates at the intersection of row i and column

j is denoted as memory cell (i, j), with i ∈ {1, . . . , M },

j ∈ {1, . . . , N }. Each array can store an M × N binary data

matrix X = [xi,j ]M×N , with xi,j ∈ {0, 1} being stored in memory cell (i, j). During writing, each bit is programmed

into the memory cell by changing the resistance value of the

memristor. For a logical-0 bit, the cell is changed to a high

resistance value R0; for a logical-1 bit, the cell is turned to a low resistance value R1. During reading, certain voltage is applied to the target cell and the resistance is detected by

measuring the current going through the memristor. However,

the memory reading becomes unreliable due to the existence of

the SPI and the random resistance variations. A sneak path is

an undesired path which originates from and returns to location

(i, j) while traversing typically three LRS cells through verti-

cal and horizontal steps. An example is shown in Fig. 1, where

cell (3,2) is the target cell for reading. The green line shows the

desired path for resistance measuring and the red line shows a

sneak path: (3,2)→(3,4)→(1,4)→(1,2)→(3,2). A direct effect

of the SPI is a reduction the measured resistance value, and

hence it is only detrimental when reading an HRS cell. The

most popular method to mitigate the SPI is to introduce a cell

selector to each memory cell, which is an electrical device

that allows current to flow only in one direction [1]. However,

cell selectors are also prone to failures due to imperfections

in the production, leading to the re-occurrence of the sneak

path. We assume that the selectors in a memory array fail

i.i.d. with probability pf = 0.001 [3], [6]. Hence, a sneak path event occurs when reading cell (i, j) if and only if the

following conditions are satisfied: (1) The cell (i, j) is in HRS,

i.e., xi,j = 0; (2) There exists at least one combination of k ∈ {1, . . . , M }, l ∈ {1, . . . , N } which forms a sneak path,

i.e., xi,l = xk,l = xk,j = 1; (3) The selector of cell (k, l) fails.

We use a random variable ei,j as a sneak path event indi-

cator, i.e., ei,j = 1 if the cell (i, j) is affected by sneak path,

and ei,j = 0 otherwise. The resistance value that incorporates

the SPI is then given by ri,j = R1 for xi,j = 1, and ri,j =

Unlike the work of [6] which simply assigns the LDPC

coded bits to the T memory arrays sequentially in a row-

by-row manner, in this work, we propose a novel two-step

bit allocation scheme. The first step is a Tanner graph based

across-array bit allocation to distribute the bits of one code-

word to different arrays, while the second step is a within-array

bit allocation to further shuffle the bits of the same codeword

assigned to an array into different rows and columns. The

overall objective is to reduce the effect of channel correlation

to the belief propagation (BP) decoding of LDPC codes.

Our proposed across-array bit allocation scheme is generally

applicable to different types of LDPC codes.

VI. CONCLUSION
We have considered the SPI and lognormal distributed resistance variations of ReRAM, and proposed a cascaded channel model to incorporate these two key factors. We have developed a novel scheme to estimate the probability of each memory cell to be affected by the SPI, based on which the LLR of each channel bit can be generated. We have further proposed a novel two-step across-array bit allocation scheme for the across-array coding with LDPC codes for ReRAM. Simulation results demonstrate that our proposed across-array LDPC coding system can effectively tackle the correlation of the ReRAM channel and achieve superior BER performance than the prior art coding schemes for ReRAM, with similar implementation complexity and latency.
