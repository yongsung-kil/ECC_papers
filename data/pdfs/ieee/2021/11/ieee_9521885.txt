Construction of Multi-Rate Quasi-Cyclic LDPC Codes for Satellite Communications
Chong Zhang , Xijin Mu , Jinhong Yuan , Fellow, IEEE, Huaan Li , and Baoming Bai , Senior Member, IEEE

Abstract— To provide reliable transmissions with flexible rates for satellite communications, this paper presents a novel method of constructing multi-rate quasi-cyclic low-density parity-check (LDPC) codes. The basic idea is to generate low-rate codes from a high-rate mother code by combining shortening and extending, which ensures that the generated code family owns the same code length, in order to maintain the same frame structure. The code construction involves the design of base matrices and exponent matrices for the designed codes. A progressive row elimination and addition algorithm is proposed for designing the code base matrices from a high rate to low rates. This algorithm leads to the nested and systematic structure of the parity-check matrices, which are desirable for practical implementations of their encoders and decoders, while ensuring the optimal decoding thresholds. In addition, we construct a circulation coefficient matrix based on finite fields and select the optimal rows in this matrix to construct exponent matrices while considering of cycle structures. We demonstrate that the designed codes achieve better performance for all the code rates than the LDPC codes in DVB-S2X standards. In addition, the proposed codes do not exhibit error floors for their block error rates down to 10−5.
Index Terms— Satellite communications, channel coding, LDPC.

LDPC codes were first introduced by Gallager in the early 1960s and shown to have capacity-approaching performance with iterative decoding [10]. Quasi-cyclic LDPC (QC-LDPC) codes are a class of structured LDPC codes that allow low-complexity encoding and decoding [11]. Due to their low implementation complexity, QC-LDPC codes are adopted in several wireless communication standards, such as DVB-S2X [9], IEEE 802.16e [12], 5G [13], and etc. In wireless communications, variable-rate LDPC codes and adaptive coding and modulation on a frame-by-frame basis are required to adapt the varying characteristics of wireless channels. The variable-rate codes should not only own acceptable error-correcting capability, but also be capable of sharing the same set of encoders and decoders for the sake of practical implementation complexity. According to the approaches to realizing rate compatibility, variable-rate LDPC codes can be classified into rate-compatible LDPC (RC-LDPC) codes and multi-rate LDPC (MR-LDPC) codes.
RC-LDPC codes were first proposed in [14], which achieved variable rates by varying the code length while maintaining the same information length. Ha et al. optimized the puncturing patterns of RC-LDPC codes based on the density evolution method [15]. In [16], a type of constructed RC-LDPC codes by extending a protograph were proposed, which can achieve a wide range of code rates. In [17], structured RC-LDPC codes are constructed based on combining algebraic-based and graph-based construction methods. The proposed codes have advantages in hardware implementations of encoder and decoder.
Note that most of the existing works to achieve variable rates are based on the construction of RC-LDPC codes, where a flexible code length ensures that the RC-LDPC codes can be used for incremental redundancy (IR) protocols for various wireless communication systems [12], [13]. On the other hand, MR-LDPC codes obtain variable rates by varying the information length while maintaining the same code length. Row combining with edge variation (RCEV) is a typical way to constructing MR-LDPC codes [18], which linearly combines the parity-check matrix’s rows to obtain high-rate codes. In addition to row combining, a few 1-elements can be added or deleted in the parity-check matrix to tune the code column degree distributions in order to achieve good performance. However, one of the big weaknesses of RCEV is that the strongly coupled feature of RCEV makes it difficult to guarantee universal good performance for all supported rates. Row-splitting is another method for constructing MR-LDPC codes [19], [20], which is achieved by splitting the parity-check matrix’s rows to obtain low-rate codes. This method is more flexible than the RCEV method and can support a larger code rate range. In [21], a more efficient method, named row-combining-splitting with edge variation (RCSEV), was proposed. The RCSEV method combines RCEV and row-splitting, so it can provide more freedom than the previous methods in the code construction. In [22], nonbinary MR-LDPC codes were constructed by applying a matrix-theoretic approach. The proposed codes can

achieve good performance in both the waterfall and error-floor regions.
In general, the methods mentioned above achieved the flexibility of code rates with row combining or row splitting. The code performance was optimized mainly by adjusting the degree distribution of the code parity-check matrices, but the structures of parity-check matrices were less considered from the implementation point of view. While RC-LDPC codes are mainly used for wireless cellular networks where HARQ can be applied, however, in other applications such as geostationary Earth orbit (GEO) satellite systems, ARQ is not feasible due to long delays. Fixed length MR-LDPC codes together with FEC protocols are more appropriate for satellite or deep space communications, which require a fixed frame length or frame structure [9].
Satellite communications will be an essential part of the future communication networks [23]. The increasing demands of highly reliable transmissions require powerful channel coding schemes for satellite communications. This paper focuses on the construction of MR-QC-LDPC codes for satellite communication systems and presents a new method for constructing MR-QC-LDPC codes. The basic idea is to generate low-rate codes from a high-rate mother code by combining shortening and extending techniques. For a given high-rate systematic code, the shortening decreases the number of information bits and the extending increases the number of check bits. With careful designs, the combination of extending and shortening ensures that the generated low-rate codes and the mother code own the same code length. The parity-check matrices of proposed codes are constructed following the protograph approach [24], which involves the design of base matrices and exponent matrices. The base matrices are first designed by optimizing the decoding thresholds with the protograph extrinsic information transfer (P-EXIT) chart. P-EXIT chart is a tool for calculating the decoding threshold and predicting the asymptotic performance of protograph LDPC (PTG-LDPC) codes. It has a relatively low computational complexity and good threshold estimation accuracy [25]. Then, the exponent matrices are designed to make the resulting parity-check matrices avoid 4-cycles (cycles of length 4) and own 6-cycles as few as possible. The proposed method enables the resulting parity-check matrices possess quasi-cyclic and nested structures, which are desirable for practical implementations of the encoders and decoders. In addition, the obtained codes are systematic codes, and they can also be directly encoded by using the parity-check matrices. Numerical results show that the proposed codes are able to achieve good performance within a wide range of code rates.
Compared with existing construction methods of MR-LDPC codes, such as the RCEV, row-splitting and RCSEV, the proposed method is able to achieve better performance with the typical code length around 1000 bits and 16200 bits. From the implementation point of view, both our proposed codes and the codes constructed with existing methods own quasi-cyclic structure. On this basis, our proposed method also takes more considerations on the compatibility of different code rates. A. Protograph LDPC Codes
A protograph is a relatively small bipartite graph, which consists of variable nodes (VNs), check nodes (CNs), and edges [26]. Since parallel edges are allowed in a protograph, there may be more than one edge between a VN and a CN. A Tanner graph [27] for an LDPC code can be obtained from a protograph with a copy-and-permute procedure, which is called lifting. Here, a protograph is copied several times, where the number of copies is referred to as lifting factor. The parity-check matrix corresponding to the obtained Tanner graph gives a PTG-LDPC code. The double-lifting offers a simple and effective method to generating a large base matrix from the original small base matrix. Consequently, the PTG-LDPC codes construction procedure consists of two steps, designing base matrices and designing exponent matrices. In the conventional base matrix design, computer search is the main method to obtain the optimal decoding threshold to achieve good error performance. The exponent matrix can be designed via computer search or

algebraic methods to avoid 4-cycles and reduce the number of 6-cycles. However, in the literature, most of the designs of the base matrix and the exponent matrix only consider a single code rate instead of all rates range. In this paper, we will study how to design good base matrices and exponent matrices for a family of codes, preferably with algebraic tools.
B. Shortening and Extending
Shortening and extending are two commonly used techniques for modifying the code rate and length for a given code [28]. Shortening is used to obtain low-rate codes by dropping some known information bits. This corresponds to eliminating some columns of a parity-check matrix, and thus the code length is decreased. Extending can yield low-rate codes by producing additional parity-check bits. Both the row and column dimensions of a parity-check matrix are increased by the same amount after extending operation, and thus the code length is increased.
For a QC-LDPC code, extending and shortening can be performed by changing the dimensions of a parity-check matrix with the size of sub-matrices. Fig. 1 illustrates the shortening and extending of a given half-rate QC-LDPC code. Each submatrix P is weight-1 circulant, which are also called circulant permutation matrix (CPM). Each submatrix 0 is an all-zero matrix, and each submatrix I is an identity matrix. Given a rate-1/2 mother code, we can see that the shortening operation eliminates a column block of the parity-check matrix of the mother code to obtain a rate-2/5 QC-LDPC code. From the same mother code, the extending operation adds a row block and a column block simultaneously to obtain a rate-3/7 QC-LDPC code. Shortening and extending operations can provide flexible code rates for a large range, whereas the code length of the new codes are not the same as the mother code. In the following, we will integrate these two methods approximately to design LDPC codes with flexible code rates and a fixed code length for satellite communications.

III. CONSTRUCTION OF MULTI-RATE LDPC CODES
This section introduces the construction method of the proposed MR-QC-LDPC codes with length N . Let the collection of proposed codes be {C1, C2, . . . , CJ }. Assume that rj is the code rate of the code Cj, and r1 > r2 > · · · > rJ . Let Hj be the parity-check matrix of code Cj. Here, Hj is an Mj × N array of CPMs and ZMs, Bj and Cj are base matrix and exponent matrix of code Cj, respectively. The size of both Bj and Cj is Mj × N . The lifting factor is L.
Let code C1 be the mother code with rate r1 = (N −M1)/N . All other lower rate codes of the code family will be generated from the mother code with the following proposed method. Here, we have rj = (N − Mj)/N . Let Mj = Mj−1 + Δj, j = 2, . . . , J, Δj ≥ 1. Considering Δj = 1 in our design, we obtain the smallest code rate change or the code rate granularity of 1/N for the code family.
In this paper, we assume that a high rate PTG-LDPC mother code has been designed and optimized with conventional methods, such as graph-based methods [24], [29], [30] and algebraic-based methods [31]–[33]. For the sake of the low complexity implementation, we particularly consider systematic PTG-LDPC codes, whose parity-check matrices are of block-wised single diagonal or double diagonal property. In the following, we will focus on how to construct MR-QC-LDPC codes from the mother code.
A. Combining Shortening and Extending
Unlike the current row combining and row splitting methods, we achieve the flexibility of code rates by combining shortening and extending, where the low-rate QC-LDPC codes are constructed from a given high-rate systematic QC-LDPC code.
The construction is illustrated by using an example in Fig. 2. Since the given code is systematic, the column blocks of the parity-check matrix can be divided into two parts, which correspond to the information bits and check bits, respectively. We first eliminate a column block corresponding to information bits by shortening, and then add a row block and a column block by extending. In general, both the eliminated column block’s position and the added row block in the code

In order to achieve this purpose, we perform the first lifting by replacing the elements in each base matrix Bj with L1 × L1 square matrices to generate the new base matrix Bj. The square matrices are used to keep the corresponding rates of Bj and Bj unchanged after lifting. The main idea of this lifting operation is building the corresponding relationship between the elements in Bj and the square matrices. We give two basic design criteria for the choices of the lifting matrices. The given criteria set the size and weight of square matrices for the first lifting. These settings guarantee that the maximum value in the resulted base matrix Bj is 1, which will ensure that the second lifting operation can be implemented directly by using CPMs, and that the degree distributions of the corresponding codes for Bj and Bj are the same. The proposed design criteria are shown in the following.
1) To ensure that the maximum value in Bj is 1, the lifting factor L1 should be greater than or equal to the maximum value in Bj and the maximum value in the lifting matrix is 1.
2) To ensure that the degree distributions for Bj and Bj are the same, the row weight and the column weight of each lifting matrix should be equal to the value of its replaced element bm,n in Bj.
With above criteria, we obtain a new base matrix Bj of size (MjL1) × (N L1) for the code Cj.

We first design a circulation coefficient matrix based on finite fields, and then design exponent matrices with the base matrices and the circulation coefficient matrix.
We choose a finite field GF(q), where q is the smallest prime number greater than the length N of the base matrix and the lifting factor L. Let α be a primitive element of GF(q). Then α−1 =Δ 0, α0 = 1, α1, . . . , αq−2 form all the q elements of GF(q). Let S1 = α0, α1, . . . , αq−2 , containing all q − 1 nonzero elements of GF(q). Let S2 = αk0 , αk1 , . . . , αkN−1 , where kn ∈ {0, 1, . . . , q − 2} for 0 ≤ n < N . S2 is a subset of N random nonzero elements in GF(q). E. Base Matrix Sparsification
Using the previous methods, we can achieve flexible and variable code rates from a single high-rate mother code with an incremental redundancy process. This incremental redundancy process can ensure the compatibility of different rate codes, and bring the nested structure of parity-check matrices. However, the existing matrix will also limit the choice of new base matrix. With the code rate decreasing, the right columns of the base matrix corresponding to the information bits are replaced with the weight-1 columns and the newly added rows are gradually sparse. These changes lead to that the elements in the base matrix mainly concentrated in the upper left corner. For very low rate codes, the resulting parity-check matrices are highly structured and their constituent permutation matrices are densely packed, which will affect the performance of the constructed codes. Here, we use the base matrix sparsification to reduce the impact of this limit. We propose to make the base matrices of the constructed codes sparse by reducing the density of its sub-arrays to further improve the performance of low rate codes.
This sparsification is performed by replacing the larger values in the base matrix by smaller ones. Fig. 4 illustrates an example of the base matrix sparsification operation. We give a base matrix with code rate 1/2 and obtain a new base matrix with code rate 1/3 by combining shortening and extending. Compared with the base matrix of code rate 1/2, the elements in upper left corner corresponding to the information bits for the base matrix of code rate 1/3 appears to be more crowded. We build the base matrices sparse by replacing the elements 3 in the upper left corner in the base matrix with 2 or 1 to improve the performance of resulting codes.

The sparsification is similar to masking, which can also reduce the density of a parity-check matrix by replacing a set of permutation matrices with zero matrices [38]. However, the masking performs on the base matrix with only 1-elements, and the result is changing 1-elements to 0-elements. The sparsification can perform on the base matrix with elements larger than one, and the result is changing nonzero-elements to smaller or 0-elements. The sparsification is more general, and can be viewed as a generalization of masking. The base matrix sparsification reduces the number of 1-elements in the parity-check matrix. Therefore, the design of wiring routing in the implementation process can be simplified. In addition, the current description of base matrix sparsification is mainly a schematic illustration, which shows its ability to enhance the code performance. However, the base matrix sparsification makes the parity-check matrices of low rate codes no longer meet the strict nesting characteristics, which means that it will affect the hardware implementation of the code family. Therefore, there is a trade-off between the hardware complexity and code error performance to be considered for designing the family of LDPC codes for practical systems.

V. CONCLUSION
This paper presented a new method for constructing a family of pragmatic MR-QC-LDPC codes for satellite communications. The method integrated the shortening and extending operations to achieve variable rates and a fixed length from a single QC-LDPC mother code. For the construction, the base matrices of the MR-QC-LDPC codes were optimized by using the P-EXIT chart. The obtained base matrices possess the nested structure of the parity-check matrices for the code family while ensuring the optimal decoding thresholds. The quasi-cyclic and nested structures are preferable for practical implementations of the encoders and decoders. Three code families of pragmatic MR-QC-LDPC codes were constructed based on the proposed method with code lengths 960, 5616, and 16200 bits respectively. We demonstrated that the constructed codes can achieve better performance than the existing codes for a wide range of code rates, and no error-floors as low as at 10−5 are observed.
