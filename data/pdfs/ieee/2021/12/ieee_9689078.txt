Performance Analysis of Sub-Optimal LDPC Decoder for 5G using Belief Propagation Algorithm

Abstract—Today’s modern wireless communication systems demand extensive quality and computational power with an increase in day-to-day data-rate standards. The latest communication standards focus on low latency, high throughput, and reliable error correction capabilities which inspired us to explore the 5G New Generation (NR) technology. Low-Density Parity-Check Codes (LDPC) are known to have error correction capabilities that can achieve Shannon’s maximum channel capacity. Encoding of LDPC Codes features a sparse property that enables effortless decoding at the receiver end. Advancements in deep learning have helped in estimating the channel noise more accurately. The Iterative Belief Propagation-Convolutional Neural Network (BP-CNN) architecture involves a CNN block which is trained with a quadratic loss function concatenated to a standard BP Decoder. The proposed Iterative BP-CNN architecture with a Batch Normalization layer results in a better performance compared to the Iterative BP-CNN architecture as it standardizes the input to the CNN block and also solves the problem of vanishing gradients during backpropagation.
Index Terms—5G NR, LDPC, BP, CNN, BP-CNN, LLR, BPSK, BER, SNR, BN
I. INTRODUCTION
Channel Coding is essential for digital communication systems as it adds controlled redundancy in the form of parity bits to provide immunity against the channel noise.Latest cellular communication technologies such as 5G uses longlength codes to provide high data rates. The primary issue for any communication system is the efficiency of the decoding process as it requires a lot of computational power.The sparseness of LDPC codes helps in reducing the decoding complexity with increasing code lengths.
LDPC codes belong to the family of linear block codes which utilize a sparse parity check matrix, containing less number of ones as compared to the number of zeros. [1] Sparse Parity Check matrices can also be represented in the form of a tanner graph containing two distinct nodes namely, bit nodes (represents the number of codeword bits) and check nodes (represents the number of parity check equations) [2].
The Belief Propagation decoding process involves the

iterative passing of the Log-Likelihood Ratios (LLRs) between the check nodes (c) and bit nodes(v) to produce the extrinsic information of the received codeword bits [3]. The final LLR computed is then decoded to produce the information bits. The Belief Propagation Algorithm is implemented in two forms namely the Sum-Product algorithm and Min-Sum algorithm.
Real communication channels exhibit noise correlation due to signal fading and multi-user interference. The performance of most soft-decision algorithms is based on a pre-defined model which can vary with channel conditions. Recently, deep learning models have become an integral part of communication systems because of their adaptive nature and robustness against channel noise. [4] In particular, CNN is the preferred choice for denoising as it models the noise distribution more efficiently.
The Iterative BP-CNN architecture in [5] is a highly adaptive and low complex decoder structure that exploits the channel noise correlation characteristics. The proposed Iterative BP-CNN architecture with added Batch Normalization layer standardizes the input scale and also solves vanishing gradient during weight estimation.
II. PROBLEM SETUP
The pyldpc package in python is used to obtain the generator matrix of row weight (wr) as 8 and column weight (wc) as 2. LDPC codes with a code rate of 0.75 are generated by multiplying the generator matrix with a random message matrix. 25,000 message samples, each of length 433, are encoded to create a code word vector of size (576, 25000). Channel Noise simulated is a correlated noise with a correlation parameter (η) equal to 0.8 and follows Gaussian distribution. The channel noise is taken as the dataset to train the CNN model.

C. Iterative BP-CNN Algorithm
Real communication channels exhibit noise correlation because of factors like oversampling, filtering, and multi-user interference. The Neural decoding architecture in [5] is suitable as it can be trained to adapt to the correlation in the channel noise. The decoding architecture concatenates a BP decoder with a CNN block, which improvises noise estimation in the BP Decoder. The CNN model is trained with the estimated noise symbols as features and channel noise symbols as the labels.

Iterating between the BP and CNN structure gradually improves the decoding process and results in the desirable Bit-Error Rate versus Signal-to-Noise Ratio (BER vs SNR) performance.

V. CONCLUSION
The Iterative BP-CNN decoding architecture is primarily used for LDPC channel decoding. The CNN suppresses the correlated noise, with a correlation parameter η of 0.8, and improves the efficiency of the BP decoder. The Quadratic Loss Function equipped with the normality test used in the CNN output layer improves the denoising capability of the CNN architecture. However, the above decoding architecture faces two shortcomings which are namely, vanishing gradient in weight estimation and irregular CNN Input Scale. The Iterative BP-CNN with Batch Normalization layer resolves these issues as well as enhances the error correction capability of Iterative BP-CNN decoder when measured against the Sum-Product algorithm decoder.
