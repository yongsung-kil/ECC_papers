Relation Between GCD Constraint and Full-Length Row-Multiplier QC-LDPC Codes With Girth Eight

Guohua Zhang , Yulin Hu , Senior Member, IEEE, Yi Fang , Member, IEEE, and Defeng Ren

Abstract— The greatest-common-divisor (GCD) method is a general framework employing a set of simple inequalities (called GCD constraint) to guarantee girth eight for a class of (J, L) quasi-cyclic (QC) low-density parity-check (LDPC) codes. However, an important problem, i.e., whether the GCD constraint is necessary for this class of codes to have girth eight, remains open. In this letter, the question is answered affirmatively, following which a novel algorithm aiming to find the shortest codes with girth eight in such a class is proposed. Besides, a close connection is established between the GCD method and the base expansion method, which are both applicable for any J and any L.
Index Terms— Circulant permutation matrix (CPM), girth, low-density parity-check (LDPC) codes, quasi-cyclic (QC).
I. INTRODUCTION
A QUASI-CYCLIC (QC) low-density parity-check (LDPC) code is defined as the null space of a sparse parity-check matrix (PCM) which is composed of circulant permutation matrices (CPMs) or zero matrices (ZMs). To improve the performance of QC-LDPC codes, great effort (such as [1]–[9]) has been made to increase the girth value, the length of the smallest cycles in the corresponding Tanner graph. Among these schemes, the greatest-common-divisor (GCD) method [3] features a simple and elegant construction of girth-eight QC-LDPC codes for any column weight and any row weight, which needs either no computer search or very low computational complexity to verify the GCD constraint which is represented by a set of simple inequalities [3], [7], [10]. Besides girth-eight QC-LDPC codes, the GCD method can also be extended to constructing affinepermutation-matrix (APM) LDPC codes with girth at least eight [11]. Moreover, as a basic module, the GCD method has been combined with traditional techniques of constructing LDPC codes, such as masking [3], [7], and row/column splitting [10], to build longer codes with good performance.
Although satisfying the GCD constraint enables a class of QC-LDPC codes to have girth eight, an important problem remains open in the literature to the best of our knowledge: whether the GCD constraint is necessary for such codes to have girth eight? In this letter, the problem is solved affirmatively, based on which two contributions are achieved.

First, an algorithm is developed to search for the shortest girth-eight codes belonging to the class. The lengths of the novel codes are noticeably shorter than the state-of-the-art results pertaining to this class of codes. Second, the recently proposed base expansion (BE) method [7] is proved to be a special case of the GCD method. Our results further enrich the GCD method and promote its application for constructing QC-LDPC codes with girth at least eight.

II. PRELIMINARIES
A (J, L) QC-LDPC code can be expressed by a J × L exponent matrix E = [e(i, r)] (0 ≤ i ≤ J − 1, 0 ≤ r ≤ L − 1) and a CPM/ZM size P , where there are J (resp. L) non-∞ entries per column (resp. row). From E,
the corresponding PCM can be readily generated by replacing every entry e(i, j) = ∞ by a P × P ZM and replacing every entry e(i, j) except ∞ with a P ×P CPM in which the nonzero element ‘1’ in the zeroth row is in the mod(e(i, j), P )-th column.
Now consider a class of QC-LDPC codes whose exponent matrices are defined by Ef = [ai·r] [3], where J = J, L = L and {a0, a1, . . . , aJ−1} are integers satisfying 0 ≤ a0 < a1 < · · · < aJ−1. For convenience, this class of codes is referred to as full-length row-multiplier (FLRM) codes. It has
been shown in [7] that girth of such codes cannot exceed
eight, since there exist cycles of length eight regardless of
the CPM size. The GCD method [3] is an efficient framework
to construct FLRM codes attaining girth eight.
Lemma 1 ( [3], [7]): Let i, j and k be three different row indexes (0 ≤ i < j < k ≤ J − 1). If the inequality (GCD

B. An Exhaustive Search Algorithm
According to Theorem 1, the GCD constraint is necessary and sufficient to guarantee a girth-eight FLRM code. Therefore, exhaustive methods can be devised via verifying the GCD constraint to find the shortest girth-eight FLRM codes. A direct exhaustive method is to search all combination of (a0, . . . , aJ−1). As this method fails to utilize the detailed knowledge of boundary of ai (0 ≤ i ≤ J −1), it becomes very slow for J > 6. B. On Be Method for L = 3 and Related Approaches
For L = 3, the BE method yields the sequence F(J,3) which is the same as the sequence [1] defined by the ES method. In this subsection, we explore the relationships among BE method for L = 3 and other closely related approaches. Here are some definitions and facts. (i) If a sequence does not include three integers x, y and z such that x + z = 2y, it is called a sequence with no three terms in arithmetic progression [1]. Such a sequence is called no-three-term for brevity. (ii) The ES method [1] uses a greedy manner to choose a new entry, i.e., the first possible entry x is selected such that the existing entries and x together satisfy the no-three-term constraint. (iii) The GCDg8 algorithm [3] employs a greedy way to choose a new entry, i.e., the first possible entry y is chosen such that the existing entries and y together satisfy the GCD constraint. (iv) A sequence is no-three-term if and only if it satisfies the GCD constraint for L = 3 [6].
According to (ii)-(iv), the sequence generated by the GCDg8 algorithm for L = 3 is identical to the one obtained by the ES method, which confirms the empirical observation in Remark 1 of [3]. Moreover, thanks to (iv) and Theorem 1, a no-three-term sequence corresponds to a (J, L = 3) FLRM code with girth eight, and vice versa. Obviously, this is just the special case of Theorem 1 for L = 3.
Remark 1: Since F(J,3) is identical to the sequence generated by the greedy GCDg8 algorithm with L = 3 or the greedy ES method, no integer in the range fL(0) < x < fL(J − 1) can be inserted into the set F(J,3) to jointly satisfy the GCD constraint. However, it is not the case in general for L ≥ 4. For example, if L = 4 and J = 9, the sequence F(J,4) from the BE method is {0, 1, 4, 5, 16, 17, 20, 21, 64}. However, there exist four integers (45, 47, 54, 56) such that the combined sequence still satisfies the GCD constraint.
Remark 2: Although no integer can be inserted into F(J,3) to form a larger set satisfying the GCD constraint, it doesn’t mean that there exists no qualified set of J entries with a smaller aJ−1. For the case J ∈ {5, 9, 10}, the value αJ−1 obtained by the BE method (L = 3) is not the smallest, because Algorithm 1 (L = 3) for J ∈ {5, 9, 10} yields a sequence with the smallest αJ−1, that is, {0, 1, 5, 6, 8}, {0, 1, 5, 6, 8, 13, 14, 17, 19} and {0, 1, 4, 6, 10, 15, 17, 18, 22, 23}, respectively.

VI. CONCLUSION
The GCD constraint in the GCD method can enable a special class of QC-LDPC codes (FLRM codes) to attain girth eight. In this letter, the GCD constraint is proved to be necessary for the FLRM class to possess girth eight. Based on the proposed necessity and the existing sufficiency, this letter conducts the following two main novel results which enrich the existing GCD method. First, an exhaustive algorithm is presented which is able to obtain the shortest girth-eight FLRM codes. Second, for the first time it is proved that the BE method is actually a special case of the GCD method. How to construct QC-LDPC codes based on girth-eight FLRM codes and free of small-size trapping sets [14] deserves further study.
