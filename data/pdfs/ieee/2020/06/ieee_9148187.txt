LDPC design for 5G NR URLLC & mMTC
Liguang Li1,2, Jun Xu1,2, Jin Xu1,2, Liujun Hu1,2 1ZTE Corporation, Shen Zhen, 518057, China 2State Key Laboratory of Mobile Network and Mobile Multimedia Technology, Shenzhen, 518057, China E-mail: li.liguang9@zte.com.cn, xu.jun2@zte.com.cn, xu.jin7@zte.com.cn, hu.liujun@zte.com.cn

Abstract—Low density parity check (LDPC) code in the fifth generation (5G or New Radio, NR) was mainly designed for the enhanced Mobile Broadband (eMBB) scenario, and specific optimization has not been considered for both scenarios of UltraReliable and Low Latency Communications (URLLC) and massive Machine Type Communications (mMTC), especially for small transport block sizes (TBSs). Therefore, we introduce a specific optimized design method of LDPC code for small TBSs of the latter two scenarios. According to the proposed method, a new LDPC code is designed and its performance is better than the LDPC code in 5G. Furthermore, an improved algorithm combining belief propagation (BP) and ordered statistic decoding (OSD) is considered in this paper. Compared with traditional BP+OSD algorithm, although there is slight performance loss, the number of OSD decoding implemented in the improved algorithm can be reduced to 1 or 2. Simulation results indicate that the performance of the improved BP+OSD decoding algorithm is obviously better than that of the popular BP decoding and also better than that of the polar code in 5G.
Keywords—5G, Small TBS, LDPC, BP+OSD decoder
I. INTRODUCTION
In order to meet growing requirements for mobile communication, 5G mobile communication system emerged. Three major application scenarios were considered for 5G: eMBB, URLLC and mMTC. For eMBB scenario, high-speed traffic service is supported, such as target peak rate of 20 Gbps for downlink and 10 Gbps for uplink. For URLLC, the userplane delay is limited to very small, such as 0.5 milliseconds. In addition, the peak spectral efficiency is 30 bps/Hz for downlink and 15 bps/Hz for uplink. The reliability for URLLC scenario is evaluated by the success probability of transmitting X bytes with a target reliability of 1-10-5 within 1 millisecond. For mMTC scenario, the connection density is 1 million terminals per square kilometer [1].
LDPC code was firstly proposed by Robert Gallager in 1963 [2]. In 2005, quasi-cyclic (QC) LDPC coding [3] based on the cyclic shift of identity matrix, which is also named structured LDPC code [4], was first applied in WiMAX (World Interoperability for Microwave Access) standard. The QC LDPC code can be defined by a lifting size and a base matrix. In 2006, LDPC code was also recommended for Long Term Evolution (LTE) channel coding instead of turbo code [5]. However, LDPC code was not adopted since it has no

obvious advantages over turbo code for low peak data rate (such as 100 Mbps). Later, the LDPC code with the same structure as in WiMAX was adopted by IEEE802.11n and IEEE802.11ad due to its remarkable performance and high decoding throughput. In October 2016, LDPC code was adopted by the 3GPP (3rd Generation Partnership Project) meeting as data channel coding scheme for 5G standard.
However, LDPC code in 5G is mainly oriented to the eMBB scenario, and it is not optimized for both scenarios of URLLC and mMTC. Small TBS may be mainly used in both latter scenarios. In this article, we present a method to construct LDPC codes for small TBSs, including two aspects: base graph (BG) design and base matrix design. Girth is defined as the length of the shortest cycle in an LDPC bipartite graph. The base graph for QC LDPC code is designed firstly, then the base matrix is constructed. In order to derive good base graphs, we consider the iterative decoding threshold (IDT) as in [6], as well as girth and trapping set. Both girth and trapping set are taken into account for base matrix design. We also provide a method how to identify the trapping set in an LDPC bipartite graph.
In addition, compared with the polar code in 5G, the LDPC code in 5G has slightly inferior performance for small TBSs. We consider a decoding algorithm of the combination of BP decoding and OSD decoding [7][8] to improve the LDPC decoding performance for small TBS. The BP+OSD decoding algorithm can achieve near Maximum Likelihood (ML) decoding performance for linear code when the OSD order is large enough. However, since one OSD decoding is required for each BP iterative decoding, the traditional BP+OSD decoding results in massive computational complexity and delay. Therefore, we provide an improved BP+OSD decoding which can obviously reduce the computational complexity.

Section III provides the detailed design algorithm to generate LDPC base graphs. Section IV presents the detailed algorithm to generate LDPC base matrices. In order to support flexible code rates and incremental redundancy for hybrid automatic retransmission request, the base matrix of the LDPC code with the form of extension structure [4][9] is considered. The QC LDPC base matrix includes two types of elements: circular shift value and "-1" elements. And, the base graph can be obtained by setting the circular shift values in base matrix as "1" and the "-1" elements as "0". The complete base graph can be generated by a matrix extension operation from a kernel base graph. For low code rates, some rows are added to the kernel base graph. Wherein, each row corresponds to a single parity check (SPC) code. An exemplary diagram of the LDPC base graph is shown in Fig. 1.

For LDPC decoding, errors may easily occur if poor

trapping sets [10] exist in the LDPC bipartite graph. A

trapping set is defined as a variable set with two parameters a

and b, where a is the number of variable nodes in the variable

set and b denotes the number of its neighbor check nodes

connecting to odd number of the a variable nodes. We call a trapping set with small values of a and b as a poor trapping set. During LDPC code design, poor trapping sets should be avoided. Here, a novel progressive variable node growth (PVG) method is presented to identify trapping sets. Firstly, a cycle in an LDPC bipartite graph is identified, then its variable nodes are regarded as an initial trapping set with parameters a and b. For simplicity, we have the following two definitions:
Definition 1 [extra neighbor check nodes of a trapping set T1]: check node neighbors connecting to the variable nodes in the trapping set T1 with odd number of times.
Definition 2 [extra neighbor variable nodes of a trapping set T2]: variable node neighbors of an extra neighbor check node of the trapping set T2 excluding the variable nodes in the trapping set T2.
A new trapping set can be found by adding a new variable node to the initial trapping set. Note that, the added variable node should belong to the extra neighbor variable nodes of the initial trapping set. During the PVG method, if the parameter b of the newly obtained trapping set equals 0 or the parameter a reaches the preset maximum value, it stops.
An example for PVG method is shown in Fig. 2. The variable nodes of {v0,v1,v2} and the check nodes of {p0,p1, p2} form a cycle with length of 6. Its variable nodes {v0,v1,v2} is an initial trapping set with parameters of a=3 and b=1. We can see that the extra neighbor check node of the initial trapping set is {p4}. The extra neighbor variable node of the initial trapping set is {v4}. Then, a new trapping set {v0,v1,v2,v4} is obtained by adding the new variable node {v4} into the initial trapping set. The trapping set {v0,v1,v2,v4} is regarded as an initial trapping set, and another new variable {v3} is added to form a new trapping set {v0,v1,v2,v3,v4}.

III. CONSTRUCTION OF LDPC BASE GRAPH

Base graph design is an important step for LDPC code design. If it is designed improperly, the poor trapping set may not be avoided during lifting process for base matrix. For small TBSs, the main factors affecting LDPC performance are girth and poor trapping sets. Therefore, here we consider the iterative decoding threshold, as well as girth and trapping sets.

IV. CONSTRUCTION OF LDPC BASE MATRIX
The base matrix of a QC LDPC code includes two types of elements: 1. the circular shift value, which is an integer greater than or equal to 0 and less than the lifting size Z, corresponds to the cyclic shift value of an identity matrix; 2. the "-1" element, which is used to describe zero sub-matrix, corresponds to ‘0’ in the base graph. Since both small girth and poor trapping sets lead to LDPC decoding errors and performance degradation. It is necessary to avoid the occurrence of poor trapping sets and short cycles. Similar to base graph design, the corresponding PCM priority principle for base matrix design is described as: a larger value of girth, a smaller number of shortest cycles, and larger values a and b of poorest trapping sets. In order to derive a high girth LDPC code, an algorithm in [11] is used to search a set of circular shift values with the largest value of girth. Then, each circular shift value is filled into the base matrix and the parameters a and b of trapping sets is calculated. We will select the circular shift value with maximization of a and b.

However, the combination of BP decoding and OSD decoding for iterative decoding can be used to reduce the order of OSD decoding and improve the decoding performance. We call it as a traditional BP+OSD decoding here. Wherein, one OSD decoding is needed for each BP decoding iteration. Although the BP+OSD decoding can improve the performance and reduce the OSD order, its computational complexity is very high since too many times of OSD decoding are needed. Therefore, an improved BP+OSD decoding algorithm is presented here to reduce decoding complexity.
According to the principle of OSD decoding, if the number of error bits is small, especially in the temporary
information sequence s , it is easier for OSD decoding to
traverse its original codeword. For BP decoding, check equation of H ⋅ c' ( c' is hard decision result of each iterative BP decoding) is used for early termination. In general, the smaller the number of non-zero bits in H ⋅c' is, the smaller the number of error bits is. Therefore, an improved BP+OSD decoding algorithm is as following: after performing all iterations of BP decoding, select one or two output LLR sequences with the least number of non-zero bits in H ⋅c' as inputs of the OSD decoding. The order of the OSD decoding equals 1.
The LDPC code in 5G with information length of 40 and 64 is used for evaluation. 10 BP iterations is performed. Performance results are shown in Fig. 8. The performance of the improved BP+OSD decoding is close to that of the traditional BP+OSD decoding. Because only one or two OSD decoding operations instead of 10 are performed, the computational complexity of the improved BP+OSD decoding can be reduced to 1/10 or 1/5. Moreover, compared with the the polar code in 5G with successive cancellation list (SCL) decoding of a list size 32, the performance gain is about 0.3 dB. Compared with popular BP decoding, the improved BP+OSD decoding has performance gain of about 1 dB.

VII. CONCLUSIONS
In this paper, we consider two ways to improve the performance of LDPC codes for scenarios of URLLC and

mMTC. On one hand, we propose a new design of base graph and base matrix for LDPC codes. The iterative decoding threshold, girth and trapping set are taken into consideration for base graph design. During base matrix construction, both girth and trapping set are taken into account. Simulation results indicate that these techniques are efficient and LDPC codes based on these methods have better performance than the one in 5G. On the other hand, the BP+OSD decoding algorithm can improve the LDPC performance greatly. We present an improved BP+OSD decoding algorithm which can reduce computational complexity significantly. Compared with traditional BP+OSD decoding, although the improved BP+OSD decoding has slight performance loss, there are ~1 dB and ~0.3 dB performance gains compared with the popular BP decoding for LDPC codes and the SCL decoding with a list size 32 for polar codes, respectively.
