An Approach to the Generation of Regular QC-LDPC Codes with Girth 8

Aleksei Kharin, Aleksei Dryakhlov, Evgeny Mirokhin, Konstantin Zavertkin, Aleksei Ovinnikov and Evgeny Likhobabin
Department of Telecommunications and Radio Foundations Ryazan State Radio Engineering University Ryazan, Russia 390005
Email: kharin.a.v, drahlov.a.a, mirohin.e.i, zavertkin.k.n, ovinnikov.a.a, likhobabin.e.a@tor.rsreu.ru

Abstract—In this paper, we propose a new approach to the generation of regular QC-LDPC codes with girth 8. The aim of the proposed smart exhaustive search (SES) algorithm is to construct codes with the desired lifting factor or to check whether codes with this parameter even exist. Also, this method allows us to reach smaller values of minimal circulant size compared with previously described methods: simulated annealing, hill-climbing, and guess-and-test.
Index Terms—regular LDPC, quasi-cyclic (QC), girth.
I. INTRODUCTION This work is devoted to the study of regular quasi-cyclic (QC) low-density parity-check (LDPC) codes. They require a smaller amount of memory for storing their parity-check matrices, and their structural properties are easier to analyze than those of random LDPC codes. It is well-known that such constructions are less widespread than irregular constructions [1] [2]. However, especially for high coding rates, regular QCLDPC codes are preferable to irregular ones in terms of error floor level. Two broad classes of methods have emerged for the construction of QC-LDPC codes: algebraic and random constructions. Algebraic codes have been constructed based on a wide variety of mathematical ideas, including finite geometries, finite fields, and combinatorial designs [3], [4]. However, their flexibility in terms of changes in the girth and lifting factor values leaves much to be desired. Randomized methods, such as Guess-andtest [5], Hill-Climbing [10], Improved PEG [6] and Simulated Annealing [7], are better suited for solving such problems. However, all of them are probabilistic, so they are not able to solve the lifting factor minimization problem. The remainder of the paper is organized as follows. Section II contains several common definitions and notations used in this paper. In Section III, the proposed smart exhaustive search (SES) algorithm is presented. Finally, we present numerical results that demonstrate the effectiveness of our algorithm and compare it with designs from the literature.

III. SES GENERATION ALGORITHM
As we aim to check the existence of a matrix with certain parameters, we need to test all possible combinations of shift values. To achieve this, we implemented a free forwardbackward search across all matrix elements. We use a PEG-like approach and utilize its predictive nature.
We move from the second column to the last one and from the second element to the last one inside each column. We do not touch pi,0 = p0,j, ∀i, j, as they are always equal to zeros. For every element, we form a list of cycles that are defined by

it and all previous elements. Based on equation (2) for a single cycle, we calculate the element’s shift value that will lead to a cycle in a lifted graph. This value is excluded from the list of possible values for the element. We perform this check for every cycle connected to the element and form a list of values that do not lead to unwanted cycles. We choose the smallest of them for the current element and switch to the next element. If for some element there are no available shift values, then we switch back to the previous element, choose the next available shift value for it, and switch forward again to check whether there are available shift values in this case.
It is highly important that lists of enabled shift values for every element are regularly updated. Every cycle connected to the current element is also defined by another element from the same column and several elements from previous columns (two or four depending on cycle length). Assuming shift values in previous columns are fixed, we can say that the solution for equation (2) is defined by the difference between the values of two elements in the current column. We call this difference a relative shift value. For a single element there are several lists of disabled relative shift values. One list is for every element that is placed before the current one. All lists of disabled relative shift values for every element in a column are calculated every time we switch to a new column from one with a smaller number, such as in the case of forward switch. Next, we can calculate a list of disabled absolute shift values for an element in the column. It is based on lists of disabled relative shifts and current shift values for previous elements in the column. The list of disabled absolute shift values is calculated every time we switch to an element with a bigger number. Excluding these values from all possible shift values we obtain a list of available shifts for the current element. With this mechanism, we have a relevant list of available shifts for every element in the matrix at any point of the search process.
In this manner, we move through all the elements in the matrix forward and backward. Actions applied at every switch depending on its direction are described in Figure 1. The search

stops in two cases. First is when we are able to find an available shift value for the last element in the last column, which means that a matrix with the desired parameters is found. Second is when we run out of available shift values for the second element of the second column, which means that a matrix with the desired parameters is not found, that is does not exist.
Doing so, we can check all combinations of shift values that may form a matrix with the desired parameters. Still, the number of combinations to check is very large. For example, in the case of a (3, 6) matrix with a lifting factor of 18 there are (z − 1)((dv−1)·(dc−1)) = 2015993900449 combinations. Of course, we do not check every single one of them yet the number remains high and becomes much bigger for matrices with more rows and columns and bigger lifting factors. Therefore, we use the properties of regular matrices to narrow the search space.
As already mentioned, we consider only matrices in which the first row and column have zero shift values. Then we can rearrange all other columns and rows so that the smallest value will be placed in the second element of the second column.

IV. RESULTS With the proposed algorithm we tried to generate regular matrices of girth 8. The search was run on a PC with Intel i7-8700 CPU clocked at 3.2 GHz and 32 Gb of DDR4 RAM. Our application was implemented in a single thread manner, so it utilized only one core of the CPU. We explored matrices with a number of rows varying from 3 to 5 and a number of columns from 4 to 12. We aimed to find the smallest possible circulant size. Yet it may take much time for big matrices, so we limited the search time for the single circulant size value, first up to 60 seconds and then up to 24 hours. We start with a rather big circulant size and try to generate a matrix within the time limit. If a matrix is found,

then we lower the circulant size by one and repeat the search. We keep lowering the circulant size until either we are unable to find a matrix within the time limit or the matrix with the circulant size does not exist.
For matrices with 3 rows, we also perform a search without a time limit to find the lowest possible circulant size. In this case, we start with a (3, 4) matrix and test circulant sizes from the theoretical minimum calculated based on Lemma 3 from [11] to the first one that will give us a matrix. Then, we move to a matrix with one more column and start with the circulant size that was found for the previous matrix. In this way, we go from 4 to 12 columns to show that matrices with a smaller circulant size do not exist.
We compare our results with the best results from previous studies. For matrices with 3 and 4 rows, we use the results from the GitHub repository [12] that were obtained with the algorithm described in [7]. For matrices with 4 and 5 rows, we use the results from Table II and Table III in [9] as no newer results were presented. All the results are presented in Tables I, II and III.
In the case of matrices with 3 rows, our algorithm achieves the lowest possible circulant size within 60 seconds with a number of columns from 4 to 10. For 11 and 12 columns, the search time limit should be increased to achieve results similar to [7]. And finally, without a time limit. we manage to find a (3, 12) matrix with a circulant size of 42:

We can state that the space of circulant sizes that allow us to form a matrix of an exact size with girth 8 is not continuous. This follows from the results of the (4, 6) matrix, in which matrices for circulant sizes of 24 and 21 were found while the full check of possible combinations of shift values in the case of sizes 22 and 23 showed that they do not exist.
Previously proposed algorithms can be characterized in terms of the success rate in finding a matrix with the desired parameters (Fig. 12 from [10] and Fig. 3 from [7]). This property cannot be applied to our algorithm as it guarantees that a matrix will be found if it exists. In this way, the search speed turns out to be the main characteristic of the algorithm. It is hard to apply this metric to previous algorithms as their speed depends on some random value chosen at the beginning of the search. Therefore, the search time was only mentioned in [7] for the search of a (3, 12) regular matrix with girth 8: 164, 509, and 8079 seconds for circulant sizes of 49, 48, and 47 respectively. With our algorithm, matrices with the same parameters were found in 1, 14, and 1557 seconds respectively. Further, for circulant sizes of 46, 45, and 42, the search was finished in 827, 44391 and 576922 seconds respectively.
In general, the smaller the circulant size, the bigger the search time as it is necessary to check more combination of shifts to find the one that will fulfill the restrictions. Still, there are cases in which the search for a matrix with a smaller circulant size may take less time. For example, a (3, 12) matrix with a circulant size of 46 was found almost twice faster than the same matrix with a circulant size of 47. This can be explained by fact that fewer combinations of shift values needed to be checked, so the search went faster.
