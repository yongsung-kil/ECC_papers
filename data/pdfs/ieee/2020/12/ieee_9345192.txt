Abstract— DOCSIS 3.1, the new international standard for high-speed cable television (CATV), uses Low Density Parity Check (LDPC) code for forward error correction (FEC). It is widely known that accurate soft information is critical to the performance of LDPC decoding, but existing bit log-likelihood ratio (LLR) expression only works well under high code rate. In this paper, an improved bit LLR expression is proposed to compensate the difference between the existing LLR expression under low code rate for Gray coded M-ary quadrature amplitude modulation (QAM). In addition to high accuracy, the improved LLR is simple and efficient to manipulate. Besides LDPC, the improved LLR is also applicable to other codes that require soft information. Simulation results show that the improved LLR expression achieves a significant improvement in performance for both LDPC and Turbo decoding.
Keywords-DOCSIS 3.1; log-likelihood ratio (LLR); quadrature amplitude modulation (QAM); soft decoding; low density parity check (LDPC)
I. INTRODUCTION
Data over cable service interface specifications (DOCSIS) is a standard for highspeed cable communication and is proposed by the North American operators [1]. The latest version of DOCSIS (3.1) is released in October 2013 and includes key technologies such as Low Density Parity Check (LDPC) code, high-order modulation (e.g., 1024-QAM and 4096-QAM), more than 1 GHz bandwidth, narrower (e.g., 25 kHz or 50 kHz) orthogonal frequency-division multiplexing (OFDM) subcarriers and provides support for low latency communication [2]. Comparing with the previous version, DOCSIS 3.1 improves transmission efficiency by 30% and boosts capacity significantly through the combination of high-order modulation and LDPC. DOCSIS 3.1 also maintains backward compatibility with existing DOCSIS equipment, and achieves higher cost efficiency by increasing network utilization and scale. For forward error correction (FEC), DOCSIS 3.1 uses LDPC code instead of the ReedSolomon (RS) code in earlier versions. Under Additive White Gaussian Noise (AWGN) channel and with a code rate of 1/2, the best LDPC code has a difference of only 0.0045 dB from the Shannon limit. LDPC along with frequency and time interleaving can be used to provide robustness against narrowband interferers and burst events. Therefore, LDPC has been widely applied in a number of

systems, such as high speed optical communication systems, LTE-A, DOCSIS 3.1, G.hn and DVB-S2. It is one of the main candidates for the channel coding scheme in the fifth generation mobile communication technology (5G) [3], [4], [5], [6]. There are two decoding methods for LDPC code, i.e., hard decision decoding and soft decision decoding. In the hard decision method, the demodulator maps the signals to 01 sequence by comparing them with the decision thresholds, and the decoder uses Hamming distance as the metric for decoding. In soft decision decoding, the demodulator outputs complex number soft bits indicating the likelihood of each bit rather than just 0-1 sequence. Then the decoder recovers the original bits following the maximum a posteriori probability (MAP) principle. One of the most commonly used soft-decision decoding algorithm is the sum-product decoding algorithm, also known as belief propagation (BP) algorithm [7], [8]. We can conduct the BP algorithm in the log domain, in which the probability information is expressed by log-likelihood ratio (LLR) and the multiplications are transformed into additions for efficiency. As the soft bits contains more information about the signal, soft decision decoding can outperform hard decision decoding by 2 to 3dB, so it has been a research hotspot since the 1980s, and has a wide range of applications.
As LLR is the key to soft decoding algorithms, there are many studies on how to compute LLR accurately and efficiently. Researchers deduce the exact LLR formulas and approximate them using some simplified formulas for reducing the computational complexity of the soft decoding algorithms. For example, Tosato et al. used some piecewise linear functions to approximate the LLR of 16-QAM and 64QAM in HIPERLAN/2, and became the most commonly used method for calculating soft information values in practical engineering [9]. K. Hyun et al. applied this method to 64-QAM or higher order Mary QAM, thus significantly improved decoding performance of the communication system using high order modulation [10], [11]. However, the formula of the piecewise function will become very complicated for the high-order constellation modulation, so the amount of calculation will increase greatly. Ardakani et al. adapted their method to Multiple-Input Multiple-Output (MIMO) system under the AWGN channel. To improve accuracy, they optimized the parameters in the piecewise linear functions [12], [13]. Yammine et al. proposed a

Although the algorithm reduces the computational complexity, its decoding performance is poor. Hochwald et al. showed a Listed Spherical Decoding (LSD) algorithm [15], [16]. When the dimension is low, the demodulation complexity of the algorithm is low, but with a higher dimension, the complexity of the algorithm increases significantly. The LLR is obtained by calculating the distance between the received symbol and a line assigned to each region [17]. However, these researches are focused on low-order constellations while it is not clear how to extend them to the high-order constellations in DOCSIS 3.1. To improve spectral efficiency, the FEC in DOCSIS 3.1 uses LDPC along with the interleaving of code words both time and in frequency domain prior to transmission. A simplified LLR expression is proposed which greatly reduces the complexity of LLR calculation for Gray coded higher order constellations. But there are still considerable difference between the actual LLR and the simplified LLR, which harms the performance of the decoder at low code rate. To address this problem, we proposed an improved algorithm, simplified LLR with error compensation, which compensates the difference by multiplying the simplified LLR with the ratio between its theoretical value and estimated value. In addition to LDPC code, the LLR calculation method can also be applied for Turbo decoder which requires LLR for decoding. Simulation results show that the proposed LLR calculation method achieves a significant improvement in performance for LDPC code and Turbo code.
The rest of the paper is organized as follows. Section II introduces simplified soft-output for DOCSIS 3.1. Section III is the table based compensation for simplified LLR. Section IV presents the simulation results and Section V concludes the paper.

III. TABLE BASED COMPENSATION FOR SIMPLIFIED LLR
The DOCSIS 3.1 system model is shown in Fig. 2. Downlink FEC in DOCSIS 3.1 only uses code rate r = 8/9. Fig. 3 is the simulation results on code rate r = 8/9 with three decoding schemes, i.e., hard decision, optimal and simplified LLR. We can see that the simplified LLR works well for modulation formats from including 16-QAM, 64-QAM, 1024-QAM and 4096-QAM. The performance of the simplified LLR is almost identical to optimal while optimal significantly outperforms hard decision.
However, the simplified LLR does not work well for low code rate. Fig. 4 shows the decoding performance of the three schemes under a code rate of 64-QAM, 1024-QAM and 4096-QAM with low code rate. The results show that simplified LLR outperforms hard decision, but there is still a large gap between the performance of simplified LLR and optimal.
Recall that the simplified LLR is an approximation of the exact LLR. Although the approximation enables us to calculate LLR more efficiently, it also introduces errors compared with exact LLR. However, if we are able to compensate the error between simplified LLR and exact LLR, decoding performance can be improved.
Let’s take 64-QAM for an example and Fig. 5 plots the difference between the exact and the simplified LLRs for the in-phase bits {b1 ˈb2 ˈb3 } of 64-QAM. For b1 , the exact and simplified LLRs are only the same when yI [i]  2d

simplified LLRs are identical for all values of yI [i] . For
higher order modulations, these differences can be big, and thus significantly degrade the performance of decoding.

To reduce the error in the simplified LLR for b1 and b2 ,
we propose a compensation method by dividing different zones. When the separation distance is 2d, an error compensation zones distribution is illustrated in Fig. 6. As the error between exact and simplified LLR is related to the
value of yI [i] , the x-axis is divided into serval regions.
The mean error in each region is calculated and stored in a table. When computing the LLRs, the simplified LLR is calculated first, then the compensation terms are added to the simplified LLR by looking up the error tables of each bit. As a demonstration, we plot the exact and compensated LLR when using a 64-QAM modulation in Fig. 7. The results show that the compensation significantly reduces the errors in the LLR. Fig.8 shows performances for different number of compensation zones with the proposed algorithm when using 64-QAM modulation. Obviously, it’s important to choose the right number of compensation zones. And the algorithm of simplified LLR with error compensation can also be applied to other modulation formats. IV. EXPERIMENT RESULTS In this section, we compare the performance of the LLR calculation methods under different modulation formats and code rates. Fig. 9 reports the performance on LDPC code. The results show that the compensation based LLR achieves almost the same performance as exact LLR, and at BER= 10-1 the compensation based LLR gain compared to
simplified LLR is approximately equal to 1 dB to 1.5 dB. Apart from LDPC code, we also compare the performance of the LLR calculation methods on Turbo code and Polar code in Fig. 10. As the decoder of Turbo code and Polar code also require soft information [18], any LLR calculation methods can also be used to provide input for the Turbo decoder and Polar decoder. The results show that the compensation based LLR also achieves good performance consistently for Turbo code and Polar code, and at BER= 10-1 the compensation based LLR gain compared to
simplified LLR is equal to 0.3 dB to 0.8 dB for Turbo code and 1 dB to 1.2 dB for Polar code.
We also report the time consumption of the LLR calculation methods in Table III. The results show that the compensation based LLR does not increase the complexity of simplified LLR significantly.

V. CONCLUSION In this paper, we derive the simplified LLR for the LDPC based FEC in DOCSIS 3.1 using a series of reductions. However, the simplified LLR can have a large error from the exact LLR and the performance of decoding will be significantly degraded under low code rate. To solve this problem, we propose a zone-based compensation method, which divides the domain of the received signal

into several zones and assigns a compensation constant for each zone. Experiment results show that the compensation based LLR significantly reduces the error of the simplified LLR. In the meantime, the low computational complexity of the simplified LLR is still maintained. The simplified LLR calculation methods can also be used to provide input for any decoder requiring soft information. We tested the decoding performance of the compensation based LLR on LDPC code, Turbo code and Polar code. The results show that it achieves almost the same performance as exact LLR but the computational complexity is significantly reduced.
