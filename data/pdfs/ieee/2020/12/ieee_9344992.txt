Semi-LDPC Convolutional Codes with Low-latency Decoding Algorithm

Zengzhe Chen1, Suihua Cai2, Li Chen1, Xiao Ma2*
1School of Electronics and Information Technology, Sun Yat-sen University, Guangzhou, Guangdong, 510006, China 2School of Computer Science and Engineering, Sun Yat-sen University, Guangzhou, Guangdong, 510006, China
e-mail: maxiao@mail.sysu.edu.cn

Abstract—In this paper, we propose a new coding scheme called semi-low-density parity-check (semi-LDPC) convolutional code, whose parity-check matrix consists of both sparse and dense sub-matrices. We propose a sliding-window decoding algorithm with a window size of two, in which a list decoding algorithm is utilized to reduce decoding latency. Simulation results show the proposed semi-LDPC convolutional code with list decoding can obtain a coding gain up to 1 dB over the existing LDPC convolutional codes with the same decoding latency.
Keywords-dense sub-matrices; low decoding latency; lowdensity parity-check codes; sliding-window decoding algorithm
I. INTRODUCTION
Low-density parity-check convolutional codes (LDPCCCs) [1], also known as spatially coupled LDPC (SC-LDPC) codes [2], are the convolutional counterparts of the LDPC block codes (LDPC-BC) [3]. A remarkable property of the LDPC-CCs is that their belief propagation (BP) decoding threshold can approach the maximum a posteriori (MAP) decoding threshold of the underlying LDPC-BC ensemble. This is known as the threshold saturation effect [4]. Another advantage of LDPC-CCs is their sliding window decoding (SWD) [5] which can trade off performance against message recovery latency. LDPC-CCs with a small memory yields the ground for realizing a lower latency SWD. Such LDPC-CCs may find applications in certain scenarios, e.g., the ultrareliable and low-latency communication (URLLC) defined in 5G.
In this paper, we focus on block-oriented LDPC-CCs with a memory of one. We propose a new coding scheme, called the semi-LDPC-CC. Different from the conventional LDPC-CCs, the semi-LDPC-CCs have parity-check matrices containing dense sub-matrices. A list decoding algorithm with only one-block-delay is proposed to achieve the lowlatency. Simulation results show that the semi-LDPC-CCs exhibit up to 1 dB coding gain over the conventional LDPCCCs under the constraint of the same decoding latency.
This paper is organized as follows. In Section Ⅱ, the parameter definitions of the semi-LDPC-CCs are briefly introduced and the encoding algorithm is proposed. In Section Ⅲ, a list decoding algorithm for semi-LDPC-CCs is proposed to reduce the decoding latency. In Section Ⅳ, complexity reduction methods are proposed. In Section Ⅴ, the performances of the semi-LDPC-CCs are simulated

compared with LDPC-CCs. Conclusions are drawn in Section VI.

For the conventional LDPC-CC, different window sizes d are set to obtain the SWD performance, and the list decoding algorithm is applied as well. For the semi-LDPCCC, the window size is fixed to two. The frame error rate (FER) is shown in Fig. 2. Note that this FER reflects the subframe error probability. We see that for the conventional LDPC-CC, the performance improves as the window size increases. At the FER of 103 , it can be observed that 1. As the list size enlarges, the coding gain can increase to 1 dB. 3. The conventional LDPC-CC using list decoding performs even worse than that with the original SWD algorithm, suggesting that introducing dense sub-matrix is helpful for list decoding.

VI. CONCLUSION
In this paper, a new coding scheme called the semiLDPC-CC has been proposed. It differs from the conventional LDPC-CC by accommodating the dense submatrices in its parity-check matrix. This yields ground for performing the list decoding algorithm within a decoding window. Our simulation results have shown that the semiLDPC-CCs with list decoding can substantially outperform the LDPC-CCs under the same decoding latency constraint.
