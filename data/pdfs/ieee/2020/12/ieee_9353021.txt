LDPC Decoder Based on Chisel

Bingrui Wu
College of Information and Electronic Engineering Zhejiang University Hangzhou, China
e-mail: 21831062@zju.edu.cn

Abstract—This paper proposes a sparse check matrix H generation method and a regular AA-LDPC (architectureaware low-density parity-check) code decoder architecture. The generated H matrix is suitable for regular LDPC codes, avoiding the appearance of four-line loops. The decoder uses TDMP (turbo-decoding message-passing) and Log-Map algorithms, which can achieve faster convergence speed and higher throughput compared with traditional decoding algorithms. Use the scoreboard algorithm to solve the problem of data conflict between TDMP algorithm initialization and the first iteration. Extrinsic messages and posterior messages are stored in 6bit and 8bit respectively, the memory usage is reduced by more than 60% compared with the traditional method, and the chip area is reduced. The decoder module is written in high-level Chisel language, which effectively improves the development efficiency compared with verilog. Uvm (Universal Verification Methodology) was used to randomly generate 10,000 stimuli to verify the verilog code generated by Chisel. With a clock frequency of 290MHz, this architecture can decode any regular AA-LDPC(3, 27) code with a code rate of 8/9 9216-bit, with ten iterations and a data throughput rate of 450Mbps.
Keywords-component; AA-LDPC; TDMP algorithm; BCJR algorithm; Chisel; uvm.
I. INTRODUCTION
LDPC code is a block code based on sparse parity check matrix proposed by Professor Gallager[1]. Research shows that LDPC codes can achieve record-breaking performance in low signal-to-noise ratios(SNRs) applications[2]. These studies have promoted LDPC codes to become strong competitors in many communication and storage systems that require high reliability. The randomness and sparsity of the non-zero elements of the sparse check matrix directly affect the complexity of decoding[3]. This paper proposes a method for generating an H matrix suitable for AA-LDPC codes. The distribution of 1 in the H matrix satisfies the randomness and There is no four-wire loop. The system adopts Chisel hardware language to realize TDMP decoding algorithm and Log-Map algorithm. Using saturated addition method to update extrinsic messages and posterior messages, effectively reducing the memory area. Analyze the characteristics of the Log-Map algorithm and simplify the circuit implementation of the algorithm. Experimental results show that this design effectively reduces the chip area and improves the decoding throughput rate.

The sub-matrix of the AA-LDPC code is composed of a cyclically shifted identity matrix, so this allows to make full use of the parallelism of the LDPC decoding algorithm, and the decoder generates higher throughput [5]. Each column of the sub-matrix has at most one non-zero element, which allows parallel decoding of the block row where the submatrix is located, and the complexity of the TDMP decoding algorithm increases linearly with the code length.

III. TDMP ALGORITHM AND LOG-MAP ALGORITHM
A. TDMP Algorithm
In the TDMP algorithm [6], each iteration uses the posterior message updated from the previous layer to calculate the external and posterior message of this layer. Since the updated message can be effectively used immediately, the TDMP algorithm can speed up the convergence speed and throughput rate. In addition, variable node message is calculated from posterior message and external message, which saves memory significantly. The TDMP algorithm steps are as follows：

C. Storage Unit
The decoder contains a 16×432bit rom, which is used to store the base matrix of the AA-LDPC code. The value of the posterior message uses 8bit storage, and a 144×64bit dualport ram is needed to store the sign bit of the posterior message and a 144×448bit dual-port ram to store the numerical value of the posterior message. The value of external message is stored in 6 bits, which requires 16 dualport rams of 27×384 bits.

D. SISO Unit
The internal structure of the SISO module is shown in Figure 5. Each SISO module contains 4 inputs and 4 outputs. SISO read from the dual-port ram, subtract the prior message from the external message, calculate the message from the check node to the variable node, and calculate α and β recursively.
