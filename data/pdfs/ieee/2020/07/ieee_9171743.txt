Abstract — Ensuring data reliability and adapting a very powerful error correction code of flash memory and DVB-S2 became a strategic demand nowadays. LDPC codes are designed for their high error correcting ability. Meanwhile, the error floor of LDPC codes cannot satisfy the significant low error rate requirements of flash memory applications. Therefore, we take into account a concatenated BCH and LDPC coding system for the potential usage of data preventive on flash memory. To solve this problem, the concatenation of BCH and LDPC code is performed. The result is a better balance between the potential to correct errors and the low error rate, which can be an optional coding structure. In this paper, our objective is to design a parallel BCH code concatenated with LDPC code to enhance BER and minimize the low error floor. Additionally, our purpose is to figure out the least number of BCH branches that achieve the best BER and consume the least hardware resources.
Keywords—DVB-S2, LDPC, BCH, BER.
I. INTRODUCTION
Interest in reliability of data for flash memory has increased recently due to new manufacture requirements, therefore developing new technology to assure data reliability of flash memory becomes an important research field. One of the BCH codes application is to implement a single-level cell flash memory to maintain data integrity. However, when progressive to multilevel cell flash memory, errors of data access transpire to be more likely to happen due to the reduction of a margin between state levels [4]. Therefore the BCH coding technique alone might not be suitable for the advanced multiple-level cell flash memory error protection requirement. In both industry and research, new errorcorrecting codes, like low-density parity-check LDPC codes, are developed to protect data and improve the ability to correct errors in next-generation flash memory. LDPC codes have the advantage of their high ability to correct errors in the waterfall region. Nonetheless, the error floor phenomenon in LDPC codes represents a problem that limits their usage for flash memory applications because they require intensely low error rates.
Lately, an alternative solution was proposed by using the serial concatenated coding system by integrating BCH and LDPC codes [13]. One of the advantages of combining BCH code and LDPC code is to improve the error floor [10] [11] and therefore it will be suitable for usage in flash memory. Additional research may however be required to reimburse for the resulting performance deterioration in the waterfall region resulted from the loss of rate attributable to code concatenation. The idea behind our method is to design a parallel BCH code concatenated with the LDPC code to overcome the loss of rate owing to code concatenation. Since a coding system that is suitable for low-cost hardware implementation is more desirable in flash memory, we work

on designing a new BCH code selection methodology such that the least number of parallel BCH branches is used to achieve the best BER and also consume the least hardware resources.
It is then shown by simulations that by selecting parallel equal length BCH codes, the new proposed design has improved the performance of error correction in the waterfall region in addition to maintaining low error floor. The sections of the paper are: Section II related work. Section III description of the BCH code, Section IV explanation of LDPC code. Then, Section V reviews the concatenation of parallel BCH-LDPC code, finally, simulation results are introduced in Section VI. After all, the conclusion is written in Section VII.
II. RELATED WORK
In [8], Researchers focused on designing a parallel BCH-LDPC code where the BCH codes having the same length. The input data are divided into β groups, where each group is encoded by its relevant BCH encoder, then the output of every β-BCH codeword is collected and get into the LDPC encoder. The same steps are applied in the concatenated BCHLDPC decoder, the results from this work led to the conclusion that the undesirable error floor will be erased.
As far to our knowledge, there is not sufficient data to determine the interface between BCH and LDPC encoder/decoder and the number of parallel BCH branches, therefore our main goal is to design a number of parallel BCH codes concatenated with LDPC code and calculate the BER against S/N and determine the number of BCH branches that achieve the best BER with the least hardware resources.

A. LDPC Decoder Process
The LDPC decoding relies on the algorithm of bit-flipping [11] or belief propagation (BP) algorithm [12]. In the Bit-flipping algorithm the detector makes a binary hard decision for each obtained bit and this is forwarded to the decoder. As shown in Fig. 4, the messages sent along the edges of the Tanner graph which are binary, a variable node sends a message indicating whether it is a zero or a one and every check node send back a message to variable node, specify on which value the bit is based. The check node states that it completes the parity-check equation if the modulo-2 sum of the received bit values is zero. The variable node turn over its current value, if most of the bits received by a variable node are vary from its received value. This process is replicated until all the calculations for parity tests are met, and then when a valid codeword is found the bit-flipping decoder will be terminated instantly. Otherwise, when the number of decoder iterations exceeds the maximum limit, the decoder gives up. Meanwhile, in belief propagation (BP) algorithm likelihood information is passed along tanner graph edges, variable nodes and check nodes continue to alternate the likelihood information until a null syndrome or a maximum number of iterations are passed.

VI. SIMULATION RESULTS
In this work, for a column weight equal 3 of (93, 186), (155, 310), (217, 434), (310, 620) LDPC code is constructed to concatenate with a (26, 31) BCH code. We consider the codeword is transmitted over an AWGN channel. The maximum number of LDPC iterations is set to be 5 iterations.

Fig. 6. Diagram of serial BCH-LDPC code.
Fig. 7, 8 shows our proposed scheme, the input data bits are divided into β-groups and each group is encoded by their relevant BCH encoder and the output of every β-BCH codeword is compiled and entered into the LDPC encoder. The received bits will then be decoded by LDPC decoder, and the output will be divided into β group and decoded by its related BCH decoders. Eventually, the BER will be calculated using different number of BCH branches and compare the value of BER=10-3 when using (3-5-7-10) parallel BCH branches in order to get the number of BCH branches that achieve the best BER and consume the least hardware resources.
Fig. 7. Diagram of parallel BCH encoder concatenated with LDPC encoder.

Fig. 9. LDPC performance codes with different code length.
Fig. 9 illustrates the comparison between different LDPC code lengths. The code length is chosen according to the matching between the number of BCH parallel codes and LDPC. These results confirm that as LDPC code length increases as the performance of error correction enhanced. Therefore, it is important to take into account the code gain difference between these LDPC code lengths. The later will allow the study of the suitable number of parallel BCH codes that give better gain compared to its complexity.
As seen in Fig.10, when comparing the performance of single LDPC with our proposed model at a BER = 10-3, the decoding design with three parallel BCH decoder outperforms the single (93, 186) LDPC by 0.3 dB and when using five parallel BCH decoder outperforms the single (155, 310) LDPC by 0.4 – 0.25 = 0.15 dB meanwhile-by using seven parallel BCH decoder outperforms the single (217, 434) LDPC by 0.22 – 0.15 = 0.07 dB finally by using ten parallel BCH decoder outperforms the single (310, 620) LDPC by 0.18 – 0.12= 0.06 dB.

VII. CONCLUSION
In this study, the concatenation of equal parallel BCH and LDPC codes has been shown to have potentially good error correction capabilities. To test the effectiveness of our concatenated coding system, simulations are recorded and performed in Figs. 10 and all LDPC codes to be compared within this simulation are designed based on the minimumweight codeword of LDPC code which equal three. When the proposed schemes are compared with each other, The BER will be increased with increasing the number of BCH branches meanwhile it will be at the cost of increasing hardware resources.
