A New Partially-Parallel VLSI-Architecture of Quasi-Cyclic LDPC Decoder for 5G New-Radio
Anuj Verma† and Rahul Shrestha‡, Member, IEEE School of Computing and Electrical Engineering, Indian Institute of Technology Mandi
d17029@students.iitmandi.ac.in†; rahul shrestha@iitmandi.ac.in‡

Abstract—This paper presents partially-parallel low-density paritycheck (LDPC) decoder architecture for binary-input additivewhite Gaussian-noise (BIAWGN) channel in 5G new-radio cellularcommunication technology based on layered min-sum decoding algorithm. This architecture exploits the iterative decomposition of checknode and variable-node combined-processing unit which leads to area optimization and lower hardware complexity. The proposed VLSI architecture of LDPC decoder processes quasi-cyclic LDPC encoded information which is received as log-likelihood ratios (LLRs) from soft demodulator at the receiver side. It achieves the adequate bit-error-rate (BER) of 10−6 between 2 to 4 dB of bit-energy to noise-power-spectraldensity (Eb/N0) with 10 decoding iterations for various code rates like 1/3, 2/5, 1/2, 2/3, 3/4, 5/6, and 8/9. The VLSI architecture of 5G newradio LDPC decoder has been field-programmable logic-array (FPGA) prototyped and its implementation results are compared with state-ofthe-art designs where the proposed LDPC decoder shows lower hardware utilization up to 87%. To the best of our knowledge, this is the first VLSIarchitecture of LDPC decoder reported for 5G new-radio compliant to the specifications of enhanced mobile broadband (eMBB).
Index Terms—Wireless communication, LDPC code, channel decoder, layered min-sum decoding, 5G new radio, digital very-large-scaleintegration (VLSI) architectures, and field-programmable gate-array (FPGA).
I. INTRODUCTION
Low-density parity-check (LDPC) code was pioneered by Robert G. Gallager, in the year 1960 [1], has been broadly adopted for forward error correction (FEC) in wireless, wireline, and optical communication systems over past two decades. Its key features are near-Shannon limit FEC performance, low-complexity, and hardwarefriendly decoding [2]. These qualities proliferated the use of LDPC code in several wireless-communication standards like IEEE 802.16e (WiMAX), IEEE 802.11n (Wi-Fi), and DVB-S2 [3] [4] [5]. Recently, LDPC code has been standardized by the 3rd-generation partnershipproject (3GPP) for 5G new-radio (NR) in release-16 [6] which is the phase-I standardization for 5G NR. Earlier, the primary coding scheme for 3G & 4G standards is turbo code [7] [8] that is now replaced by LDPC code for 5G-NR due to its compact Raptor-like protograph structure, higher throughput, area efficient, multi-edge and flexible shift-coefficient design [9]. Adherence of LDPC code in a modern standard like 5G-NR is empathically introducing of hardware-oriented transmutation of belief propagation layered decoding algorithm [10] and decoder architecture which is adequate for 5G-NR performance at low power. Hence, the scrutiny of hardware efficient LDPC decoder implementation attracts the focus which gradually promotes LDPC codes over the turbo codes. On the other hand, LDPC codes are defined by a bipartite graph which is known as Tanner graph [11] where internal message computation mechanism is performed at check nodes (CNs) and variable nodes (VNs) that lead into lower computational complexity from a hardware perspective. In 5G-NR standard, LDPC code pertains the quasi-cyclic (QC) LDPC codes and has a Raptor-like structure which is shown in base graph

matrix (Hb) below [8] [12].

Hb matrix structure is a standard base graph matrix for 5G NR, where A, E, O, B, C, I are sub-matrices. Here, A corresponds to information bits, E is a dual-diagonal matrix, O is zero matrix, B and C are extension check matrices, and I is the identity matrix. For 5G NR, QC-LDPC code adopted two base graph matrices of raptorlike structure: BG1 and BG2 [13]. BG1 has 46 rows and 68 columns where maximum information bits are 8448 (37.5% more bits than 4G-LTE) and supports code rates from 1/3 to 8/9 whereas BG2 has 42 rows and 52 columns with maximum 3840 information bits and supporting code rates from 1/5 to 2/3 [12] [14].
Our work focuses towards the design of QC-LDPC decoders compliant to enhanced mobile broadband (eMBB) specifications for 5G NR technology. It is the first phase of deployment for 5G NR that provides greater data-bandwidth which is complemented by moderate latency improvement. It supports a wider range of codewords and is an enhanced version of 4G Long Term Evolution (4G-LTE). As discussed above, a major challenge is to design various communication blocks (like LDPC channel decoder) in the physical layer to support such a large information bits (8448) with better hardware efficiency. There are various LDPC decoder implementations reported earlier for 3G/4G wireless standards [10] [15]. However, LDPC decoder implementation for 5G NR has been sparsely reported in the literature; like Liu et al. [16] recently reported low latency and reduced complexity LDPC decoder architecture for 5G NR ultra-reliable low-latency communications (URLLC). To the best of our knowledge, there are no LDPC-decoder hardware design and implementations reported for 5G NR compliant to eMBB. This is a high-time to conceive new VLSI-architectures for such decoders which play a key role in the physical layer deployment of 5G NR.
In this work, we propose a new partially-parallel LDPC-decoder architecture for iterative message passing mechanism based on layered min-sum decoding algorithm for 5G NR wireless communication standard. This architecture has been designed for a core code-rate of 1/3 for BG1. Various other code rates of 5G NR can be realized using puncturer/de-puncturer in a simple way. In addition, we have carried out comprehensive bit-error-rate (BER) performance analyses of our QC-LDPC decoding algorithm for all the standardized coderates of 5G NR. The proposed 5G-NR decoder architecture has been hardware prototyped in latest field-programmable gate-array (FPGA) platform. Eventually, we have compared our implementation results with the state-of-the-art works.

B. LDPC Decoding Algorithm
Specifically, QC-LDPC code has been standardized for 5G newradio enhanced mobile broadband (eMBB) communication system [6]. The H matrix for such QC-LDPC code is built using base matrix (Hb), as shown in Fig. 2 (represents the matrix form (1)). In this matrix, every non ‘-1’ element is replaced by an array of identity

matrix (Iz,z) with a circular shift of the corresponding permutation element and every ‘-1’ element is replaced by a zero matrix (Oz,z). On the other side, LDPC decoding is based on belief propagation technique which is iteratively performed by message passing between check nodes (CNs) and variable nodes (VNs) until the valid codeword is found or the maximum limit of iterations is achieved. III. PROPOSED ARCHITECTURES
This section starts with a top-level architectural description of the proposed QC-LDPC decoder for layered min-sum decoding algorithm on the encoded data. Subsequently, the VLSI architectures of various sub-modules of this decoder compliant to the 5G wireless standard are presented.
A. Overall Decoder Architecture
The suggested architecture of our LDPC decoder comprises of three basic modules: various memory blocks, CN & VN combined processing unit (CVCPU), and network of multiplexers, as shown in Fig. 3. At the input-side of this decoder, 8-bits of quantized LLRs are received and subsequently stored in initial memory bank (IMB) which is a stack of 68 initial memory units (IMUs). Here, each IMU has a depth of 384 memory locations with 8-bits of wordlength. Thus, IMB has 68×384 = 26112 memory locations which store 26112 received-LLRs of 8-bits each. It is to be noted that 26112 bits are the standardized frame size (for 1/3 code rate) of 5G new-radio wireless-standard and the corresponding LLRs from soft demodulator are fed to our decoder, as discussed in Fig. 1. In the real-world scenario, these received LLRs are fed sequentially where single LLR is stored in a memory location of IMB in every clock cycles and this consumes 26112 clock cycles to store all LLRs. These LLRs are read from all 68 IMUs and their locations are based on Hb matrix elements that are stored in the base-matrix ROM, as shown in Fig. 3, and thereby, IMB reads out 68 LLRs in every clock cycle. These 68 LLRs (that comprises of 68×8 = 544 bits) are passes through iterative multiplexer (IMUX) which passes one its inputs (i.e. Inp0) for first iteration and routes another LLR input (i.e. Inp1) via register for remaining iterations. Similarly, SUB-MUX passes Inp0 and Inp1 for first and rest iterations respectively. Outputs from

IMUX and SUBMUX are fed as inputs to CVCPU which performs the subtraction, min-sum approximation, and addition for selected LLRs (where non -1’s entries exist in a single layer of Hb matrix). Detail discussion on the design of VLSI architecture for CVCPU is discussion in next sub-section. Here, CVCPU generates updated LLRs (out ini mem) for the next layer of Hb matrix and intermediate LLRs (out inter mem) which are used for next iteration, stored in intermediate memory bank (IntMB), as shown in Fig. 3. All these 68 8-bits updated LLRs are fed back and overwritten in all 68 IMUs simultaneously in a single clock cycle. Hence, IMB requires 384 clock cycles to get updated with all generated LLRs for a single layer of Hb matrix. In the same clock cycle, 544-bits intermediate LLRs (68 LLRs where each LLR of 8-bits) are stored in IntMB. This IntMB comprises 46 intermediate memory units (INMUs) where each of these has 384 memory locations with 544-bits of word-length. Every INMU stores the intermediate LLRs corresponding to a single layer of Hb matrix. As soon as the IMB is updated with generated LLR of a single layer, IntMB simultaneously stores the intermediate LLRs (which are used for next iterations) in INMU in the same clock cycle. Both IMB and IntMB needs 384 clock cycles to update all 68 IMUs and single INMU, respectively, for processing a single layer of Hb matrix. Such updated values from IMB is used for the next layer of Hb matrix. Subsequently, 68 LLRs are again read from the IMB memory locations corresponding to the next-layer elements of Hb matrix. This layer also consumes 384 clock cycles for storing updated LLRs to IMB, and intermediate LLRs to next INMU of IntMB.
Aforementioned process of LLR updation in IMB and IntMB continues till 46th layer of Hb. At the end of 46th layer processing, all INMUs of IntMB are filled with their corresponding Hb matrix layers and this marks the completion of the first iteration of our LDPC decoder consuming 384×46 = 17664 clock cycles. As shown in Fig. 3, mostsignificant-bits (MSBs) of the first 22 IMUs are buffered in hard decision memory (HDM) which takes 384 clock cycles to store all 8448 decoded bits (i.e. 22×384 bits). Eventually, they are fetched sequentially in every clock cycle and hence the latency of our decoder is 177024 clock cycles (i.e. 176640 + 384 cycles).
B. Internal VLSI Architectures of Proposed Decoder
1) CVCPU Architecture: The CVCPU is fed with 68 received/updated (via IMUX) and 68 intermediate (via SUBMUX) LLRs read-out from IMB and IntMB, respectively, at every clock cycle for their corresponding Hb matrix layer and decoding iteration, as shown in Fig. 3. The suggested VLSI-architecture of CVCPU has been presented in Fig. 4(a) where the two sets of 68 LLRs with 8-bits are routed through two different multiplexer networks (MUX68by19) where each network comprises of 19 68:1 multiplexers. The 7-bits selection lines for these multiplexers are generated by Selection ROM (i.e. 7×19 = 133 bits), as discussed earlier in Fig. 3. Both MUX68by19 networks select only 19 LLRs each, corresponding to

the values of select lines from Selection ROM. Selection ROM has all non -1’s element locations for every layer of Hb matrix. These selected LLRs are fed to intermediate max multiplexer (IMX) and intermediate min multiplexer (IMM) networks, as shown in Fig. 4 (a), where IMX unit is the network of 19 parallel 2:1 multiplexers where one of the inputs is selected LLR and the other one is decimal number ‘31’. Similarly, selected 19 intermediate LLRs from second Mux68by19 network are fed to IMM network (comprising of 19 parallel 2:1 multiplexers) where intermediate LLR and ‘0’ decimal value are the multiplexer inputs. These IMX and IMM select inputs based on the number of non -1’s element in Hb matrix. For eg., 1st layer of Hb matrix has 19 non -1’s elements and thereby, all 19 parallel multiplexers route these 19 LLRs to the subtractor unit. However, 6th layer of Hb matrix, has 8 non -1’s elements so the first 8 multiplexers send LLRs and remaining 11 multiplexers pass the ‘31’ decimal value (via IMX) and ‘0’ decimal value (via IMM) to subtractor network. Subtractor network is a parallel network of 19 8bits 2-input subtractors and they subtract the intermediate LLRs with updated LLRs. Its output is limited to 6-bits quantization (via simple limiter unit) that means -32 and +31 are the lowest and highest values of LLRs respectively. Now, these limited LLRs are fed to min-sum approximation unit (MSAU) for min-sum approximation and minsum selection multiplexer (MSMUX) network for the selection of LLRs based on Hb layer. This MSMUX network is a parallel stack of 19 2:1 multiplexers whose one of the inputs is the output from MSAU and another input is the output from min-sum limiter, as shown in Fig. 4 (a). The 19-bits of selection input for MSMUX network (1-bit for each 2:1 multiplexer of MSMUX) are generated based on the non -1’s element in Hb matrix layers. Hence, MSMUX network passes selected 19 LLRs (19×8 = 152 bits) and are fed as one of the inputs to the adder network (ADN). Another 152-bits input to ADN is the selected output from the second IMM network.
Such ADN is collection of 19 parallel 8-bits adders which add updated min-sum and intermediate LLRs. Subsequently, the sum is limited to -128 to 127 via quantization unit, as shown in Fig. 4 (a). At the output side of CVCPU, there are two de-multiplexer networks (DeMUXNs) comprising 19 de-multiplexers for replacing the previous LLRs with the updated LLRs based on the selection inputs generation from selection ROM, as discussed earlier in Fig. 3. One of the DeMUXNs updates the LLRs in re reg mux input and passes updated LLRs (out ini mem) to IMB for next layer processing of Hb matrix. Another DeMUXN updates the intermediate LLRs in sub mux out input and sends new intermediate LLRs (out inter mem) to their corresponding INMU of IntMB, as shown in Fig. 3 and 4 (a).
2) MSAU Architecture: It is fed with 19 LLRs from the ‘-32 to +31 Limiter’ which are in 2’s complement format and thereby, these LLRs are converted to 8-bits signed-magnitude (SM) format using 2’s-complement to sign-magnitude (TCSM) units, as shown in Fig. 4 (b). Here, MSBs of 19 SM-LLRs are fed to sign unit (SU) and rest 7-bits of these 19 SM-LLRs are processed by magnitude computation unit (MCU). In SU, all input MSBs are XORed and this multiplies all signed bits and generates a product bit. This bit is further XORed with every 19 MSBs of SM-LLRs and produce 19 sign bits corresponding to the MSB of SM-LLRs, as shown in Fig. 4 (b). Finally, SU delivers 19 signs for the min-sum approximation. In MCU, two minimum magnitudes (represented as min1 and min2) are first selected among 19 magnitudes of SM-LLRs. Thereafter, a offset of decimal value 2 (i.e. 7’d2) is subtracted with min1 as well as min2. These subtracted values must be non-negative; however, if this value is less then zero then both min1 and min2 are assigned

IV. PERFORMANCE ANALYSIS, HARDWARE IMPLEMENTATION AND COMPARISON
In this work, we present a comprehensive BER-performance analysis of QC-LDPC decoding algorithm compliant to the 5G wireless standard. Comparative performance analyses of conventional and our hardware-implemented LDPC decoding algorithms are also included in this section. Fig. 5 shows the BER versus Eb/N0 plots of 5G new-radio compliant LDPC decoding algorithm which is extensively

simulated under AWGN channel environment where the information is BPSK modulated and demodulated for all code rates standardized for 5G standard. It clearly shows two categories of plots for different code-rates: conventional decoding performance without scaling and hardware-implemented performance with scaling, where LLRs are scaled by a factor of 1-2− I/S in every iteration [18]. Such scaling factor exponentially increases with decoding iterations (I) where its final value is 1 and therefore, scaling factor is approximated by a constant horizontal step (S).
For 5G new-radio standard, there are two base-graph (Hb) matrices: BG1 and BG2. Our simulations are based on BG1 matrix for code-word length (n) of 26112 bits and information length (k) of 8448 bits. Input LLRs fed to decoding algorithm are quantized in 6bits fixed-point format for simulation and 8 bits for implementation in order to mitigate the overflow problem. It delivers a BER of 10−4 at 1.1 dB of Eb/N0, for a code rate of 1/3, that has coding loss of 0.3 dB in comparison with the conventional decoding algorithm at same code rate, as shown in Fig. 5. It shows the BER simulation for different 5G-standardized code-rates of 1/3, 2/5, 1/2, 2/3, 3/4, 5/6, and 8/9. It can be clearly observed in Fig. 5 that the fixed-point BER-performance plots those are scaled in every iteration deliver better performance for lower code rates like 1/3 and 2/5. However, this performance degrades for higher code rates like 3/4, 5/6, and further.

Similarly, suggested design consumes lesser hardware by ≈87% compared to [10].
V. CONCLUSION
This paper presented the partially-parallel QC-LDPC decoder for 5G NR wireless standard with layered min-sum decoding algorithm. This design supported the decoding of 26112 encoded bits with the code-rate of 1/3 that is compliant to the 5G wireless standard. Our VLSI architecture is hardware efficient and first of its kind to support eMBB specification for 5G NR. This architecture can be enhanced to support all the code rates by controlling layer passing limitation on base matrix. This architecture can be made reconfigurable to support various parity-check-matrices compliant to the 5G standard. Eventually, the over-all memory bank size can be further reduced for fewer LLRs at the decoder input side. Finally, this proposed QCLDPC decoder architecture has iterative decomposition nature with layered decoding algorithm which makes our decoder area efficient and lowers hardware complexity.
