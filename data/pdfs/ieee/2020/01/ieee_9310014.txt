Neural Information Bottleneck Decoding
Maximilian Stark∗, Jan Lewandowsky∗† and Gerhard Bauch∗ ∗Hamburg University of Technology, Institute of Communications, Hamburg, Germany †Fraunhofer Institute for Communication Information Processing and Ergonomics (FKIE), Wachtberg, Germany,
Email: {maximilian.stark, bauch, jan.lewandowsky}@tuhh.de

Abstract—Receiver-sided channel decoding is a crucial, but computationally very demanding task. Recently, informationbottleneck-based decoding received considerable attention in the literature, as it achieves very good performance with coarse quantization and low complexity. Information bottleneck decoders replace the conventional node operations in the channel decoders with relevant-information-preserving mappings. In the literature, these mappings are typically represented as lookup tables. Unfortunately, the lookup tables are fairly large and the logic circuits needed to implement the lookup tables occupy significant chip area. This paper proposes a novel approach to overcome these impairments, which is based on a trainable function, i.e., a neural network. We show that a trained neural network can represent the mappings more compactly than the frequently used lookup tables. Moreover, our results show that even multiple-input-multiple-output mappings can be learned to replace lookup tables with prohibitive complexity. We investigate a coarsely quantized low-density parity-check decoder that realizes the relevant-information-preserving mappings for all decoding iterations with a single neural network to illustrate the practical benefits of the proposed concept.
I. INTRODUCTION
Efficient decoding of powerful channel codes like lowdensity parity-check (LDPC) codes and polar codes is a topic of high practical interest, as such codes are integral to achieve the required throughput and robustness of modern communication systems. The optimum decoding algorithms are computationally complex and need high precision. These factors often result in prohibitive implementation complexity. It is well known that reducing the precision, i.e., the number of bits required to exchange and process messages in the decoder can decrease its complexity. Unfortunately, this often comes at the price of drastic performance degradation, especially if the number of bits per message is very small [1].
Interestingly, the information bottleneck method can be applied as a very elegant tool to design channel decoders and other baseband processing units that achieve near-optimum performance despite very coarse quantization [2]. Originally, the method was proposed as a generic unsupervised clustering framework that aims to preserve so-called relevant information under compression [3]. Many applications of the information bottleneck method in signal processing were proposed already, including relaying, C-RAN, channel estimation, receiver design, and channel decoding [4]–[9].
Basically, the arithmetic node operations in a messagepassing LDPC decoder can be replaced with mutualinformation-maximizing mappings that map incoming q bit integers y = [y1, y2, . . . , yN ]T onto an outgoing q bit integer t

by application of the information bottleneck method [7]. The results from the literature indicate that typically q = 3 or q = 4 bit message representation is sufficient to approach double-precision performance. In the literature, the mutualinformation-maximizing mappings t = f (y) are often represented as lookup tables. Unfortunately, the size of the lookup depends exponentially on the number N of input messages to be processed by a node. As a result, the lookup table approach becomes intractable in practice when nodes shall be designed that process many incoming messages of a node at once. The solution to overcome this problem in the literature so far is splitting the node operation into many concatenated two-input operations, which is known to be sub-optimum and also causes an undesirable processing delay.
This paper proposes a novel solution to overcome the aforementioned problems in the design of information bottleneck decoders. The novel approach discussed in this paper is to represent the mapping t = f (y) as a trainable function fθ(y) which can be implemented using a neural network with parameters θ. The neural network is trained based on samples and learns both, its approximate design distribution p(x, y) and the mapping t = f (y) during training. In turn, it is shown that even long input vectors y, which hold a huge number N of incoming messages can be processed in one step. Interestingly, the neural network can also be successfully trained to output all extrinsic messages generated by a check/variable node in one step, rather than excluding each respective incoming message from the target edge separately. As another novelty, in this paper a single iteration-aware neural network is proposed which takes the iteration as additional input and can replace all lookup tables that vary for different iterations in the information bottleneck decoders presented so far. Please note that the neural network is not only used to learn to decode the channel code as, e.g. in [10]. Instead, the focus is to learn message-passing decoding with coarsely quantized messages. In brief, this paper contains the following main contributions:
• The design of neural networks that replace lookup tables in conventional information bottleneck decoders with much fewer parameters is proposed.
• A multiple-input-multiple-output setting (i.e., all outgoing messages of a node are generated at once) is considered.
• Training and design of a single iteration-aware neural network are proposed and investigated. Bit error rate simulations are provided which indicate that the performance of the iteration-aware neural network is similar to

II. PREREQUISITES
This section reviews the general ideas of information bottleneck decoding. Furthermore, deep learning is briefly revisited.
A. Information Bottleneck Based LDPC Decoding
Conventional belief propagation decoders for LDPC codes process log-likelihood ratios (LLRs) in the variable and the check nodes. These nodes exchange messages over the Tanner graph structure of a parity-check matrix iteratively. The check nodes evaluate the box-plus operation and the variable nodes sum incoming messages for the generation of extrinsic information. Due to the required precision of the exchanged LLR messages and also the computational complexity of the boxplus operation this decoding algorithm is very complex. To meet implementation requirements in practice, sub-optimum decoders quantize the exchanged messages to a lower precision and replace the box-plus operation with simpler approximations, for example, the min-sum operation [11]. Both results in performance degradations.
As discussed in the introduction, the information bottleneck method [3] is a generic clustering framework. It considers three random variables X → Y → T, which are termed the relevant, the observed and the compressed random variable, respectively. The method solves the problem of finding a conditional probability distribution p(t|y) that minimizes the mutual information I (Y; T) which reflects a compression goal. Simultaneously, p(t|y) shall maximize the amount of preserved relevant information I (X; T) between the compressed variable T and the relevant variable X. These two goals are contrary and hence an optimum solution represents the best trade-off between compression and preservation of relevant information.
In the context of information bottleneck LDPC decoding, the information bottleneck method aims to find a relevantinformation preserving mappings p(t|y), which replace the node operations of the check and variable nodes in the decoder. Typically, p(t|y) is restricted to be a deterministic mapping, i.e., p(t|y) = δ(t − f (y)), where δ(.) denotes the Kronecker delta function. Such a mapping can easily be implemented in a lookup table which holds each t for the respective y. Moreover, Y is considered to be a random vector with realizations y(i) = [y1(i), . . . , yN(i)]T that holds all messages processed by a

The information bottleneck obtains deterministic mappings t(i) = f (i)(y(i)) replacing the node operations in the decoder iteration i, by tracking the probability distributions of the messages exchanged in the decoder using discrete density evolution [12]. Interestingly, the mutual information I(X; T(i)) is independent of the elements in the event space of T(i). Hence, for simplicity, the literature proposes to use simple unsigned q bit integer messages. In turn, the high precision decoding task simplifies to an integer lookup operation at the nodes and an exchange of messages with low bit width q. In general, the performance of the considered approach is very good, even for bit widths as low as q = 3 or q = 4 bits per message [5], [7], [12]. As all incoming messages y1(i), . . . , yN(i) are q bit integers, the number of table entries to represent t(i) = f (i)(y(i)) is (2q)N and hence depends on the number of input messages N exponentially. This exponential dependency prevents a straightforward implementation of t(i) = f (i)(y(i)) using a single lookup table. Hence, several decompositions were proposed in the literature [5], [7], [12]. These decompositions require subsequent lookup operations, but in exchange eliminate the exponential dependency on N . Fig. 1 shows an exemplary decomposition to illustrate the idea. However, the resulting lookup tables, as well as their corresponding logic circuits are still large [13]. Moreover, the optimum operations vary for different decoding iterations i. Thus, alternatives to the classical lookup table approach are of crucial interest. In [14], it was proposed to translate the messages exchanged into a so-called computational domain at the nodes. Then, the nodes perform arithmetical operations using higher precision and quantize back their outgoing messages to the desired bit width q. The identification of proper domain transformations, however, seems to be very tricky.
This paper presents and investigates another alternative to the classical lookup table approach of information bottleneck decoding. The proposed idea is to represent the mapping t(i) = f (i)(y(i)) as a trainable function fθ(y(i)) using a neural network with trainable parameters θ. This approach maps the coarsely quantized inputs directly onto the outgoing messages and thus requires no domain transformation or anything comparable. B. Deep Learning Neural Networks

Deep neural networks experienced a tremendous research interest in the last two decades. Within the continuously increasing number of architectures and variants, multilayer perceptrons or feed-forward neural networks are their most basic variants. Formally, the feed forward neural network implements a mapping fθ(γ0) : RN0 → RNL of an input vector γ(0) ∈ RN0 to an output vector γ(L) ∈ RNL , where L defines the depth of the neural network. A feed-forward neural network consists of an input layer, an output layer and at least one hidden layer [15]. Such an architecture is shown in Fig. 2. The special case of a neural network with only one hidden layer is very useful to prove the universal approximation theorem of neural networks [15]. This theorem states that the neural network is a universal function approximator, i.e., a neural network with one hidden layer is able to approximate any continuous function arbitrarily well if the number of neurons is sufficiently large. This especially motivates us to apply neural networks to the problem at hand.
The function fθ(γ(0)) is implemented through L iterative processing steps

γ(l) = fθ(ll)(γl−1) , l = 1, . . . L .

(1)

Let γ(l) denote the output of layer l and Nl is the width of this layer, i.e., the number of units. This output is comprised of the previous layer’s output γ(l−1), multiplied by a weight matrix W(l) ∈ RNl×Nl−1 , added to a bias vector b(l) ∈ RNl , and then
fed into the activation function. The set of parameters for this layer is θl = W(l), b(l) . In turn, θ = {θ1, θ2, . . . , θL} is the set of all trainable parameters of the neural network. Thus,
the output of each layer can be written as

γ(l) = φa(l)(W(l)γ(l−1) + b(l)) .

(2)

The depth L, the width of each layer NL and the activation functions φ(al) are said to form the hyperparameters of the neural network. The literature considers many different activation funtions [15]. In this paper, we restrict ourselves to the rectified linear unit (ReLU) activation, i.e, φa(κ) = max(0, κ). The ReLU activation has several appealing properties for the application at hand, such as, sparsity and low-computational complexity [15]. Usually, neural networks are optimized using stochastic gradient descent. Stochastic gradient descent computes an estimated gradient using a small subset of the training data. This subset is referred to as minibatch B and |B| denotes the minibatch size [15]. An apparent advantage of the considered neural network architecture is that it is able to process all inputs y1, . . . , yN in one step. Moreover, if the number L of layers is small, the processing delay can be minimized. We will show in the following that a network with a single hidden layer (i.e., L = 1) is sufficient for decoding.

III. NEURAL NETWORK LEARNING OF INFORMATION BOTTLENECK DECODERS
In this section, neural networks will be trained which output the outgoing messages t(i) given the input vectors y(i) in an information bottleneck decoder. The general idea is to design the required lookup tables for the variable and the check nodes similar to [7], i.e., using discretized density evolution but without node decompositions as required in Fig. 1. Once constructed, we analyze the overall resulting N -input 1-output mappings t(i) = f (i)(y(i)) for each node type and each iteration i, which have prohibitive complexity for a one-shot implementation in a lookup table. Then we train neural networks fθ(i)(y(i)) that process all inputs y(i) of a node in iteration i in one shot for all decoding iterations i. These networks mimic and replace the iteration depending lookup tables. As generalizations, we also train one network fθ(y(i), i) for the variable nodes and another one for the check nodes, which take the iteration index i as additional inputs. Furthermore, these networks implement N -input N output mappings that generate all outgoing messages of a variable/check node at once.

Although this input is less sparse, the input dimensions are reduced and, in turn, the number of neurons and the complexity of the neuronal network is reduced as well. Furthermore, this input encoding is more common in signal processing and communications than the one-hot encoding and is thus of larger practical interest.
For training, samples are generated using known lookup tables similar to the design proposed in [7], i.e., the lookup tables designed using an information bottleneck algorithm and discrete density evolution. As a result, there is no fixed data set which is split into training set and a test set. Instead samples (t(i), y(i)) are constantly generated by sampling from p(y(i)|x) and feeding the resulting y(i) to the target network. In general, the design of information bottleneck decoders leverages a decomposition of the node operation into a concatenation of two-input lookup tables. This is mainly motivated by practical constrains as the size of the lookup without decomposition scales with |T |m, where m = dv at a variable node and m = dc − 1 at a check node. In turn, the size of the lookup table with decomposition is found as (m − 1)|T |2 [7]. Nonetheless, the concatenation of lossy compression results in an overall additional loss in relevant information compared to the non-decomposed lookup table design. This was discussed in [14]. In this paper, it will be shown that a trainable function can be learned to approximate the non-decomposed lookup table even with less parameters as required by a decomposed design but with a better performance than the decomposed design.

B. Extensions: Multiple-Output Nodes and Iteration-Aware Neural Network
The architecture shown in Fig. 2 and discussed in the previous sections, allows to compute one particular outgoing message t(i) from the incoming messages y(i) carrying the extrinsic information. However, in message passing decoding messages for each connected target edge of a node need to be computed. In information bottleneck decoding, this is usually achieved by copying the concatenated structure from Fig. 1 or by reusing the respective structure sequentially with a different removed incoming message received from the target edge. This either increases the required space or the decoding latency. The first extension of the approach proposed in the previous section, is a neural network which outputs all outgoing messages in one shot from all incoming messages. In turn, the neural network learns to consider only certain incoming messages to generate the extrinsic information but also to reuse possible intermediate results. Again we consider only one hidden layer with ReLU activation. Such an architecture is shown in Fig. 3 for an exemplary degree five check node indicated by the densely dashed area. In general, the optimum mappings f (i)(y(i)) for a variable node and for a check node, respectively depend on the iteration i. In an unrolled architecture, this iteration-dependency can easily be dealt with. In some applications, e.g. on a digital signal processor, however it is important that the same mapping can be used for all iterations. Therefore, an extended neural architecture is considered which takes the iteration i as an additional input.

IV. RESULTS
In this section, the performances of the trained neural networks are evaluated in a two-step process. First, we analyze the ability of the neural network to represent the node operations designed using the information bottleneck method as in [7]. The task of the network is to represent these mappings as accurately as possible in all decoder iterations to achieve the same performance as the conventional information bottleneck decoders from [7]. We analyze how many parameters θ are required for an adequate representation and aim for much fewer than there are lookup table entries in the original decoders. If adequate representation is possible with fewer parameters, the designed decoders should have the same performance as the lookup table based decoders, as they implement comparable mappings. Therefore, in a second step, we validate this assumption by analyzing the bit error rates of the respective decoders. Here, we also analyze the costs in terms of performance that follow from further reducing the number of parameters θ, such that only approximate representation of the actually designed node operations is possible. In detail we consider two architectures all with one hidden layer: First a neural network with N · 2q+1 + imax hidden neurons and one-hot input (one-hot in. NN) and second a neural network with N · q + imax hidden neurons and binary input (binary in. NN).
A. Analysis of the Learned Mappings
In order to evaluate if neural networks can learn to represent information bottleneck decoding mappings, we consider an

illustrative example of a mapping t(i) = f (i)(y1(i), y2(i)) with only two input messages. Fig. 4 shows a comparison of an exemplary two-input mapping from the check nodes of a q = 4 bit information bottleneck decoder (cf. Fig. 4a). The figure also shows learned mappings using neural networks which used the lookup table from the information bottleneck approach as training data (cf. Fig. 4b and Fig. 4c).
The used neural networks had the same architecture as in Fig. 2, i.e, only one hidden layer was used. However, the number of neurons in the hidden layer was 8 in Fig. 4b and 16 in Fig. 4c. Obviously, both networks can implement the lookup tables, quite well, but the accuracy of the representation depends on the number of parameters used. Please note that the number N of incoming messages yn(i) to be processed by a check node during decoding is determined by its degree dc. Table I shows the number of trainable parameters for check node degree dc = 6 and variable node degree dv = 3 and different architectures. Please remember that always only one hidden layer is used. Thus, even for higher node degrees all inputs are processed simultaneously. Therefore, fθ(i)(y(i)) can implement N -input 1-output mappings which have prohibitive complexity in a lookup table implementation. Furthermore, as shown in Table I, the number of parameters required by the neural network scales is much smaller compared to a one-shot lookup table, which shows an exponential increase in m. Interestingly, considering that imax = 50 decoding iterations are performed, the overall number of parameters for the neural network is even smaller than for the decomposed node operation. Thus, the neural network is shown to mimic the information bottleneck lookup tables accurately with fewer parameters. Similar observations can be made for the variable nodes.
