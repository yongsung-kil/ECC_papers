Bit-Labeling and Page Capacities of TLC Non-Volatile Flash Memories

Johann-Philipp Thiers, Daniel Nicolas Bailon, Ju¨rgen Freudenberger
Institute for System Dynamics HTWG Konstanz, University of Applied Sciences, Germany
Email: {jthiers,dnicolas,jfreuden}@htwg-konstanz.de Web: www.isd.htwg-konstanz.de

Abstract—The reliability of flash memories suffers from various error causes. Program/erase cycles, read disturb, and cell to cell interference impact the threshold voltages and cause bit errors during the read process. Hence, error correction is required to ensure reliable data storage. In this work, we investigate the bit-labeling of triple level cell (TLC) memories. This labeling determines the page capacities and the latency of the read process. The page capacity defines the redundancy that is required for error correction coding. Typically, Gray codes are used to encode the cell state such that the codes of adjacent states differ in a single digit. These Gray codes minimize the latency for random access reads but cannot balance the page capacities. Based on measured voltage distributions, we investigate the page capacities and propose a labeling that provides a better rate balancing than Gray labeling.
I. INTRODUCTION
Flash memories are important non-volatile data storage devices for consumer, industrial, and medical electronics. Their small size, mechanical shock-resistance, and fast readaccess times make them interesting for applications, where high data reliability or a small form factor are required. With this technology, the information is stored in floating gates which keep their electrical charge without a power supply. To read the stored information, reference voltages are applied to the control gate to distinguish different charge levels. However, this reading may result in errors. The error characteristics depend on many factors, e.g., the storage density, the used flash technology, the time since the cell was programmed, and on the number of program and erase cycles [1], [2]. Over time charge losses occur which cause a change of the threshold voltage distribution. Similarly, the memory reliability may suffer from temperature-related charge loss, read disturb, and inter-cell interference.
In this work, we investigate the bit-labeling of triple level cell (TLC) memories. A TLC stores three bits per cell. These bits are typically associated to three different pages, because different read thresholds have to be applied to read the bits stored per cell. The bit-labeling defines the number of reads that are required to determine a particular bit. Hence, the read latency depends on the bit-labeling. Moreover, this labeling impacts the bit error probability in the three pages. Today, mainly Gray codes are used to encode the cell states. A Gray code is a binary code where adjacent states are encoded such

that the code bits differ only in a single digit. These Gray codes enable page-wise read operations which minimize the latency for random access. However, Gray codes may not balance the error probabilities for the three pages.
Differences in the error probabilities lead to different page capacities. The page capacity determines the redundancy that is required for error correction. To enable page-wise read operations, the bits in different pages are encoded independently. However, this independent encoding imposes some restrictions for the error correction coding. Typically, the same error correction code is applied in all pages, because the amount of payload data and the number of parity bits are equal for all pages. This implies, that the page with the smallest page capacity determines the reliability of the flash.
The performance of error correction coding can be improved if reliability information about the state of the cell is available [3]. Hence, the channel is considered as binary input additive white Gaussian noise (AWGN) channel, where the channel output is quantized using a small number of bits [4]. In order to exploit the reliability information soft-input decoding algorithms are required. For instance, generalized concatenated codes (GCC) [5], [6], [7] and low-density paritycheck (LDPC) codes are well suited for flash memories with soft-input decoding [8], [9], [10], [11]. However, the threshold voltage sensing operation to obtain the soft information incurs a latency penalty and a larger energy consumption [12].
The capacity of a multi-level cell (MLC) was derived and investigated in [13]. This work demonstrates the evolution of the capacity over the lifetime of a flash memory and the dependency of the capacity on the word line or the page. In this work, we focus on the latter aspect. We investigate the bit-labeling and the resulting page capacities for a TLC. The aim is to avoid soft-reads by balancing the error probabilities in the pages. We propose a labeling that provides a better rate balancing than Gray labeling. Hence, the proposed labeling can improve the error correction coding with hardinput decoding. This labeling requires more read thresholds than ordinary Gray labeling. However, the number of read thresholds is significantly smaller than with soft-reads. In order to determine the channel capacity of the flash channel, we propose a parameter estimation approach for the voltage distributions based on measurement data.

II. PRELIMINARIES
The basic memory element in flash memories is a floating gate transistor. Data is stored in the form of a charge level, i.e., the floating gate can be charged and erased. Read reference voltages are applied to read out the data. The stored charge level changes the threshold voltage required to turn on the transistor. By sequentially applying different threshold voltages, the actual charge level can be inferred. TLC flashes store three bits and require at least seven thresholds to read all three bits.
An exemplary voltage distribution of a TLC flash memory is depicted in Fig. 1. In this figure, the x-axis represents voltages, and the y-axis represents the probability distribution of programmed voltages that correspond to different charge states. Each state (L0, . . . , L7) encodes a 3-bit value that is stored in the flash cell. Hence, seven reference voltages ri,j are required to differentiate the eight possible states during the read process. The reference ri,j is defined by the intersection of the distributions Li and Lj. Later on we will also use references for nonadjacent states.

The labeling 111, 110, 100, 000, 010, 011, 001, 101 in Fig. 1 is a Gray code, where the codewords of adjacent states differ only in a single digit. The first bit is the most significant bit (MSB), the second bit is called the center significant bit (CSB), and the last bit is the least significant bit (LSB). The flash device contains thousands of blocks, where each block contains a certain number of pages depending on the device manufacturing. Typically, the LSB, CSB, and MSB are mapped to different pages. With the labeling in Fig. 1, two reference voltages need to be applied successively to read the MSB page or the LSB page, whereas three read reference voltages are needed for the CSB page. This labeling supports a page-wise read operation, i.e., the LSB, CSB, and MSB can be read independently. However, this labeling does not necessarily balance the error probabilities in the different pages.
The code rate of a forward error correction code is defined as R = k/n, where k is the number of information symbols and n the number of symbols per codeword. The capacity C of a transmission or storage channel is an important performance measure. The channel capacity is the supremum of all achievable rates. Reliable transmission or data storage is only possible for R < C. III. MEASURED VOLTAGE DISTRIBUTIONS AND
PARAMETER ESTIMATION
In order to estimate the channel capacity, we assume that the channel is memoryless, i.e., we neglect inter-cell interference from neighboring cells. In this case, the capacity C can be obtained by means of numerical integration [15]. However, this calculation requires the conditional probability density functions f (x | Li), i.e., the voltage distributions. In this section, we propose a method to estimate the voltage distributions from measurement data.
By shifting the read voltage levels it is possible to measure the underlying threshold voltage distributions. An example of a measured histogram is depicted in Fig. 3. The x axis represents the possible voltage steps.

Such a tail can be seen in Fig. 3. Hence, the threshold voltage distributions are modeled by either Normal-Laplace (NL) distributions or exponentially modified Gaussian (EMG) distributions in the literature [17], [18], [19]. IV. LABELING AND READ THRESHOLDS
Gray codes minimize the latency for random access reads. However, in the next section we will see that Gray codes cannot balance the page capacities. On the other hand, a multilevel coding strategy would in general require a joint processing of all pages which impairs the random read performance. In this section we propose a new bit-labeling for the flash states. This labeling results from a multilevel coding strategy where only two pages are processed jointly. The proposed labeling provides a better rate balancing than Gray labeling, but reduces the latency compared with conventional multilevel coding [15], [14].
The capacity of a bit-page depends on the error probabilities of all required references. Fig. 5 shows a section of fitted distributions and different references. The reference ri,j is defined by the intersection of the distributions Li and Lj. For Gray labeling, only the references ri,j with j = i + 1 are required to read all three bits. Each bit-page requires one to four references depending on the labeling. For example with the Gray code shown in Fig. 1 the MSB-page requires the references r2,3 and r6,7, the CSB-page requires r1,2, r3,4, and r5,6, and the LSB-page requires r0,1 and r4,5.
Alternatively, we consider the 8-amplitude modulation scheme proposed in [15], which maximizes the Euclidean distance between possible states. While the first level requires all upper references to differentiate odd and even states, the

second level can use the information of the first level and therefore has to distinguish only even or only odd states using ri,j with j = i + 2. These decisions have a much lower error probability. This approach leads to a higher capacity than Gray codes but needs more read thresholds and therefore has a higher latency. Furthermore, it requires a joined processing of all three levels which reduces random read performance.
This concept is shown in Fig. 5. For an example, we assume a cell is programmed to state L1 with the label 011. For the first bit we need to distinguish even and odd states using all references ri,j with j = i + 1. To infer the second bit, we can use the knowledge that the first bit is zero. Hence, only the odd states have to be differentiated to read the second bit using r1,3 and r3,5. Similarly, with the knowledge that the second bit is one, only the states L1 and L5 remain. Hence, for the third bit only the threshold r1,5 has to be applied.
Note, that in a flash memory the read operation is performed on a page, which consists of many cells. Therefore, the same reference voltages are applied to all cells at the same time. Since the cells are programmed to different states we need to apply all possible reference voltages for each level. Hence, for the second level we have to apply all ri,j with j = i + 2 independent of what the value of level 1 was. When decoding, the relevant thresholds are chosen in order to determine the bits values in each level.
Soft-decoding approaches for flash memories improve the error correction performance [7]. A TLC with Gray labeling requires seven read references for hard-input decoding, whereas 21 read references are required to obtain a single softbit and 35 references for two soft-bits. These additional read references increase the channel capacity of the flash channel. However, the threshold voltage sensing operation to obtain the soft information incurs a larger energy consumption and a latency penalty [12].

V. RESULTS
In this section we investigate the proposed bit-labeling and the resulting page capacities for a TLC flash. Table II presents page capacities with different bit-labeling for a TLC at EOL. The values C0, C1, C2 are the page capacities for the MSB, CSB, and LSB page, respectively. C is the total capacity according (1). The value 3Cmin indicates the capacity for page-wise reads according to (2). The last two columns of Table II indicate the number of read references and the minimum required spare area. The last value indicates the minimum percentage of the total flash storage capacity which is required for error correction coding.
The three Gray codes use the same seven read references. Hence, these codes result in equal total capacities (with minor numerical differences). However, all Gray codes result in different page capacities. These differences lead to a capacity loss with page-wise read operations. The Gray codes require a spare area of at least 4.1%.
The size of the spare area can be reduced with other labeling codes. The row denoted by max. dist. in Table II presents results for a multilevel coding approach [15], [14]. This scheme would be the most efficient coding scheme from a coding point of view, i.e., it achieves the smallest spare area of 2.8%. A capacity achieving code would be required

only to protect the MSB page, whereas the capacities of the two other pages are very high. Hence, simple algebraic codes could be used for error correction in the CSB and LSB page. However, this maximum distance labeling requires additional read references. A total of 17 references is required which increases the read latency compared with Gray labeling. These additional references result in a small gain in the total capacity compared with the page-wise reads.
The two proposed codes use a joint processing for the CSB and LSB page, whereas the MSB page can be read independently from the other two pages. This approach reduces the capacity loss compared with Gray labeling. The minimum spare area is 3.25%. The increase of the number of read references is moderate.
For the Gray code 1 and the proposed code 2 Table II also shows results with one bit or two bits of soft information. Consider first Gray code 1. The results show an improved capacity. The required spare area is reduced from 4.2% to 2.46% or 2.21% with one soft-bit or two soft-bits, respectively. However, these capacity gains come at the price of a high read latency due to the 21 or 35 read thresholds that have to be applied to read the soft information for all read references. The highest capacity is achieved using only one soft-bit for the proposed code 2, where 2% spare area are required. At the same time, reading one soft-bit for the proposed code 2 requires only 23 threshold voltages and therefore has lower latency and energy consumption than Gray coding with two bits of soft information.
VI. CONCLUSION
In this paper, we have proposed a bit-labeling for TLC flash memories that provides a better balancing of the page capacities than ordinary Gray labeling. This labeling requires only two additional read thresholds compared with Gray labeling. Hence, the number of read thresholds is significantly smaller than with soft-reads. The new labeling achieves a significant capacity gain compared with Gray labeling and page-wise read operations. The price for this capacity gain is the joint processing of two pages which degrades the random access performance.
