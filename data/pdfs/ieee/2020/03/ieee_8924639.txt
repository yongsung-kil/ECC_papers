More on the Minimum and Stopping Distances of RS-LDPC Codes

Haiyang Liu and Xiaopeng Jiao

Abstract— Based on the extended Reed-Solomon (RS) code that has two information symbols over the field Fq , we can construct a binary regular matrix H (γ, ρ), where q := 2r for some positive integer r, γ and ρ are two positive integers such that γ ≤ q and ρ ≤ q. The matrix H (γ, ρ) specifies a low-density parity-check (LDPC) code C(γ, ρ), called an RS-LDPC code. In this letter, we provide more results on the minimum distance and stopping distance of this class of codes (denoted by d(C(γ, ρ)) and s(H (γ, ρ))) for the case γ = 6. For ρ = q, we derive an upper bound on s(H (6, q)) and d(C(6, q)), which is conjectured to be tight for q ≥ 32. For ρ < q, we investigate the choices of ρ such that d(C(6, ρ)) (resp., s(H (6, ρ))) can be improved compared with the original d(C(6, q)) (resp., s(H (6, q))).
Index Terms— Minimum distance, RS-LDPC codes, stopping distance, vector representation.
I. INTRODUCTION
C ONSIDER a binary linear code C, whose parity-check matrix (PCM) is H. The minimum distance of C, denoted by d(C), and the stopping distance of H, denoted by s(H), are important structural parameters of the code [1]. Despite their importance, determining the value of d(C) (resp., s(H)) for a given code C (resp., PCM H) is intractable in general [2], [3]. Even deriving meaningful upper and lower bounds on d(C) or s(H) is a difficult problem. Several algorithms, both exhaustive algorithms [4] and probabilistic algorithms [5], [6], for computing d(C) or s(H) have been proposed in the literature. If C or H possesses algebraic structures, however, then determining the values of d(C) and s(H) or providing their bounds by analytical methods may be possible.
In this letter, we consider a class of binary algebraic LDPC codes, RS-LDPC codes [7]. These codes have wonderful decoding performances, which make them suitable for practical purposes. In fact, an RS-LDPC code of length 2048 has been used in the IEEE 802.3an standard as the forward errorcorrection technique [8]. The minimum and stopping distances are helpful in understanding the performance of a given code, especially when the error or erasure probabilities are very low such that the simulation methods are infeasible. In addition, RS-LDPC codes have relatively high rates, which make them good candidates for the high-rate applications (e.g., storage systems). As a result, the investigation on the minimum and stopping distances may give more insights into the design of this class of codes.

An RS-LDPC code C(γ, ρ) based on the extended RS code over Fq can be described by a regular PCM H(γ, ρ) of size γq × ρq, where q := 2r for some positive integer r, γ ≤ q, and ρ ≤ q. Indeed, γ (resp., ρ) is the column (resp., row) weight of H(γ, ρ). If ρ = q, the code C(γ, q) is called fulllength. Otherwise, C(γ, ρ) is called non-full-length. With the help of the vector representation [9] of the PCMs of RS-LDPC codes, several previous works [10], [11] have considered the codes C(γ, q) and determined the values of s(H(γ, q)) and d(C(γ, q)) for 2 ≤ γ ≤ 5.
In this letter, we consider γ = 6. In contrast to the previous works [10], [11] that only focus on full-length RS-LDPC codes, both full-length and non-full-length RS-LDPC codes are considered in this letter.1 For full-length RS-LDPC codes C(6, q), we provide an upper bound on s(H(6, q)) and d(C(6, q)) using an extension of the analytical methods in [11]. Through computer search results, we conjecture that our upper bound is tight for q ≥ 32. Then we consider non-full-length RS-LDPC codes C(6, ρ) by investigating the choices of ρ such that d(C(6, ρ)) (resp., s(H(6, ρ))) can be improved compared with the original d(C(6, q)) (resp., s(H(6, q))). This is helpful in the construction of RS-LDPC codes with good parameters.
II. PRELIMINARIES
In the following discussions, the rows and columns of a matrix are both numbered from 0 for convenience.
A. Minimum and stopping distances
For a vector c, the set formed by the positions of the nonzero entries in c is called the support of c, whose size is called the weight of c. Suppose C is a binary linear code specified by the PCM H. The minimum distance of C, d(C), is the minimum weight of the non-zero codewords of C [1].
A stopping set [1] S of H is a subset of column indices such that the projection of H onto these columns, denoted by HS, contains no rows of weight 1. The stopping distance of H [1], s(H), is the size of the smallest non-empty stopping set of H.
Lemma 1 [1]: Let C be a binary linear code and H be a PCM of C. It holds that s(H) ≤ d(C).

Let us discuss the codes C(6, q) first. We provide an upper bound on d(C(6, q)) and s(H(6, q)) for any q ≥ 8. The basic idea in our discussion generalizes the method provided in [11]. First, we numerically collect some stopping sets of size 12 in H(6, q) for several moderate q using the probabilistic stopping set search algorithm in [5]. By careful verification of the collected stopping sets, we conclude that all the stopping sets correspond to the codewords of weight 12. Moreover, we can find some regularities: For each q in our searches, H(6, q) contains a stopping set S of size 12 such that Hˆ S(6, q) can be brought into a specific form after arranging its columns. Furthermore, the entries in certain positions of the form have some properties that are independent of q. Based on these facts, we guess that the form indeed corresponds to a stopping set of H(6, q) (also a codeword of weight 12 in C(6, q)) for any q ≥ 8.
