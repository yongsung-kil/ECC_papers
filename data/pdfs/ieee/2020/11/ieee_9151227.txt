Disjoint-Set Data Structure-Aided Structured Gaussian Elimination for Solving Sparse Linear Systems
Xuan He , Member, IEEE, and Kui Cai , Senior Member, IEEE

Abstract— Structured Gaussian elimination (SGE) is a class of methods for efficiently solving sparse linear systems. The key idea is to first triangulate the original linear systems. The maximum component (MC)-based strategies are widely used to implement the triangulation process. The most straightforward way to find the MC is through exhaustive search. Instead, in this letter, we propose to use a disjoint-set data structure (DSDS) to efficiently maintain the components. The extra storage and time complexity introduced by the DSDS are respectively linear to the number of unknowns and constraints involved in a linear system. Simulation results show that using the DSDS can be several times faster than doing the exhaustive search to find the components.
Index Terms— Component, disjoint-set data structure (DSDS), sparse linear system, structured Gaussian elimination (SGE), triangulation.
I. INTRODUCTION
S OLVING sparse linear systems efficiently is a key problem for many other applications, such as factoring integers and computing discrete logarithms [1], [2], maximum-likelihood (ML) decoding of erasure codes [3]–[9], fast encoding of low-density parity-check (LDPC) codes [10], decoding of non-binary LDPC codes [11], and so on. Let F be a field of interest, such as F = R or F = GF (2). We consider the linear system below:
Ax = y, A ∈ Fm×n, x ∈ Fn×l, and y ∈ Fm×l, (1)
where m, n, and l are given positive integers. The task is to compute x (the unknowns) given A and y, i.e. to solve the linear system. The number of nonzero elements in a row (column) of A is called the row’s (column’s) weight/degree. Let θ denote the number of nonzero elements of A. If θ is much smaller than mn, we say A is a sparse matrix and (1) is a sparse linear system.
Assuming the linear system of (1) is valid: there exists at least one x for Ax = y to be satisfied. A well-known result for solving (1) is that if and only if (iff) the columns of A are linearly independent, x can be uniquely recovered by using algorithms such as the Gaussian elimination (GE). However, the conventional GE has a time complexity of O(mn2 + mnl) (in terms of the number of operations over F, such as multiplications, additions, etc.), which can be

Clearly, smaller γ leads to lower complexity for solving (2). Thus, many strategies have been proposed in [1]–[10] to greedily minimize γ during the triangulation process. In particular, the most popular and efficient strategies are referred to as the maximum component (MC)-based strategies [5]–[9] (see Section III-B for the definition of component). To find the MC, the most straightforward way is to traverse the whole Tanner graph [12] corresponding to A by using, for example, the depth-first search (DFS) [13, Chapter 22.3]. (To the best of our knowledge, there is no more efficient method available in the literature including [5]–[9].) As a result, searching for the components may become the computational bottleneck for solving the entire sparse linear system.
In this letter, we propose to use a disjoint-set data-structure (DSDS) [13, Chapter 21] to efficiently maintain the components. The overall extra storage and time complexity introduced by the DSDS are linear to n and m, respectively, which are thus negligible compared to the cost of solving the entire linear system. Simulation results also verify that using the DSDS to maintain the components can be much faster than using the DFS to search for the components.

II. DISJOINT-SET DATA STRUCTURE
Let V be a set of n elements. Consider the scenario where at the beginning, each element forms a dynamic set that contains itself only. Then, the following two operations may frequently occur on V :
• FIND(v): for any v ∈ V , find the dynamic set which v belongs to.
• UNION(u, v): for any u, v ∈ V , if they belong to two different dynamic sets, say Su and Sv, respectively, unite Su and Sv into a new dynamic set.
A DSDS [13, Chapter 21] applied on V can efficiently support the above two operations. The idea is to consider the elements of V as the nodes of a rooted forest (a set of rooted trees) and associate each dynamic set (subset) of V to a rooted tree in the forest. For any v ∈ V , let par(v) ∈ V point to the parent of v in the rooted tree which v belongs to. In particular, v is a root iff par(v) points to v, i.e., par(v) = v. At the beginning, we have par(v) = v, ∀v ∈ V , since each node forms a rooted tree itself (an example is shown in Fig. 1(a)). Obviously, each rooted tree must have a unique root that can be used to identify itself.
For any v ∈ V , the operation FIND(v) is equivalent to finding the root of the tree that contains v. This is an easy task since par records the parent of each node. We summarize the FIND operation in Algorithm 1. Note that for each node u originally located on the path between v and the root r, r becomes the parent of u (i.e., par(u) = r) after calling FIND(v). For example, performing FIND(v5) on the rooted tree of Fig. 1(d) results in the rooted tree of Fig. 1(e). Such an operation is called path-compression [13, Chapter 21], which can shorten the distances between nodes and their roots so as to reduce the time complexity of future operations.

For any u, v ∈ V , if they belong to two different rooted trees, the operation UNION(u, v) is equivalent to uniting these two trees into a new rooted tree. This is also an easy task: Find the roots of u and v, and make one root as the parent of the other root if they are different. Examples of the UNION operation can be found in Fig. 1. We summarize the UNION operation in Algorithm 2, where for any root r ∈ V , rank(r) is the rank of r, indicating the upper bound on the height of the rooted tree identified by r. At the beginning when initializing par, we also initialize the rank by rank(r) = 0, ∀r ∈ V , since the height of a single-node rooted tree is zero. rank(r) is increased by one when r becomes root of the tree that is united from two rooted trees (whose roots have the same rank), corresponding to line 9 of Algorithm 2. Moreover, rank(r) may be larger than the actual height of the rooted tree identified by r, since path-compression may reduce the height of a rooted tree. For example, rank(v2) is equal to 2 in Fig. 1(d). After performing FIND(v5), the rooted tree of Fig. 1(d) becomes the rooted tree of Fig. 1(e), where the actual height decreases from 2 to 1, but rank(v2) = 2 does not change. When uniting two rooted trees, we always make the root with the larger rank as the root of the united tree, corresponding to lines 4 to 7 of Algorithm 2. Such an operation is called union-by-rank [13, Chapter 21], which can also shorten the distances between nodes and their roots.
Obviously, the storage complexity of a DSDS is O(n). On the other hand, assume that there are nf ≥ n FIND operations (each UNION operation can be considered as two FIND operations for simplicity). By implementing a DSDS with the heuristics of path-compression and union-by-rank, as shown in Algorithms 1 and 2, the time complexity for using the DSDS is O(α(n)nf ), where α(n) is the inverse Ackermann function and α(n) < 5 for any conceivable application of a DSDS [13, Chapter 21]. In the next section, we will apply the DSDS to efficiently implement the triangulation process that employs the MC-based strategies.

most straightforward way for finding the MC is to traverse the whole Tanner graph [12] corresponding to A. This can be done by using the depth-first search (DFS) [13, Chapter 22.3], each time with time complexity O(θ) (in terms of the number of integer operations). In this way, the overall time complexity of triangulation will be O(θγ).
Here, we propose to use a DSDS to maintain the components. Specifically, we regard the columns of A as the elements of the DSDS, each of which forms a rooted tree itself at Step 1. Then, each time at the beginning of Step 3, for any degree-two row of A, say ci, that has not been handled by the UNION operation, we handle ci by performing UNION(v, v), where v and v are the two neighbouring columns of ci. If v and v belong to two different rooted trees, performing UNION(v, v) can unite the two rooted trees into a new rooted tree. In this way, each rooted tree corresponds to a unique component, and each component must form a rooted tree. The rooted tree containing the largest number of columns is the MC that can be considered in Step 3.

IV. SIMULATION RESULTS

In this section, we investigate the actual running time for using the DSDS and the DFS. We realize Step 3 of the triangulation process using the following MC-based strategy:
• If A has degree-two rows, the root of the MC is moved into R. Otherwise, we follow the maximum accumulated weight-based strategy [7]–[9]: Find a row of the smallest degree. If there are multiple such rows, find one whose neighbouring columns have the largest number of nonzero elements. Select a neighbouring column of this row at random.

Fig. 2. Average (solid lines) and maximum (dash lines) running time in seconds per test case versus erasure probability pe for decoding the rate-0.5 irregular LDPC code, where (m, n, l) = (5000, 10000, 3200).
To the best of our knowledge, the above strategy is the most efficient MC-based one for triangulation that is available in the literature [1]–[10].
We mainly focus on the decoding of binary erasure codes over the binary erasure channel (BEC), where the decoding is equivalent to solving the linear system of (1) with F = GF (2). We remark that each row of x and y is a bit string of length l. Moreover, except the operations involved in the triangulation process, most of the other operations are bit-XORing. Thus, for the sake of efficiency, we compress every 32 bits of each row of x and y into an integer (as the bit-width of an integer is 32 in C++), which can significantly reduce the storage and computation cost. We will show the running time of both the triangulation process and the overall decoding process when using the DSDS (resp. DFS) to maintain (resp. search for) the components. Simulations are carried out using a desktop computer equipped with 2.4GHz Intel Xeon CPU E5-2620. For each simulation point, the reported running time is for 10,000 random test cases.
In the first example, we consider a rate-0.5 irregular LDPC code with variable and check node degree distributions 0.240602x + 0.210526x2 + 0.030075x3 + 0.125313x4 + 0.017544x6 + 0.375940x14 and 0.052632x6 + 0.902256x7 + 0.045113 x8, respectively. The parity check matrix of this code has m = 5000 rows and n = 10000 columns, and it is constructed by using the one-edge metric-constrained quasi-cyclic progressive edge-growth algorithm [14]. In the simulation, each sending codeword has n symbols. Each symbol is a bit string of length l = 3200. The symbols are transmitted one-by-one over the BEC with erasure probability pe: a symbol is totally lost with probability pe or correctly received with probability 1 − pe. The lost symbols are actually the unknowns we need to recover. The simulation results are presented in Fig. 2. We observe that for pe ≥ 0.45, using DSDS to maintain the components can be much faster than using DFS to search for the components.
