Abstract— In this paper, we propose a “bit duplication” design method for rate-compatible low-density parity-check codes (LDPC). The main idea of the method is to change the structure of codewords during its transmission over the communication system. This method can be effectively applied to create a set of codes, allowing to obtain lower-rate codes from the mother codes. To evaluate the proposed method, the dependence of the bit error probability on the signal-to-noise ratio (SNR) was simulated in Matlab. The result shows that the proposed method is superior to the conventional method of puncturing information bits.
Keywords— LDPC code, rate-compatible, codeword structure, multi-rate
I. INTRODUCTION
Low-density parity-check codes (LDPC codes) belong to the class of linear block codes. They were proposed by Robert Gallager in 1963. These codes have high noise immunity as the code length increases. However, for a long time the practical application of the codes is limited due to the complexity of the decoder implementation. Recently, with the development of computer technology and increasing requirements for the accuracy of information transmission, researchers began to pay great interest to the study of LDPC codes.
The practical communication system must support a large number of coding rates for different user scenarios using the same pair of encoder/decoder [1]–[6]. The property of such systems is called rate-compatibility. There are various methods of designing multi-rate LDPC codes, such as puncturing, extending, row combining and splitting. Puncturing is a way of obtaining high-rate codes by stripping some parity bits of the lower-rate codes (mother codes) [1], [4]. At the same time, stripping parity bits can lead to a loss of code performance. Extension is a way to obtain lower-rate codes from the higher-rate mother codes by adding extra check bits [2], [3]. However, the length of the code increases. Row combining and splitting [5], [6] are ways of getting code at different rates from the mother code by decreasing or increasing the number of rows in the check matrix. At the same time, the length of the code is preserved, but the characteristics of the parity matrix can change, resulting in reduction of code performance.
In this paper, we propose a “bit duplication” method for rate-compatible LDPC codes. The peculiarity of the method is that the lower-rate code is obtained from the parent code by changing the structure of the codewords. The advantages of this method are that the structure of the check matrix is completely preserved, which reduces the construction cost of

the encoder/decoder. And also the encoding rate can be changed very flexibly, with a fairly small step.
This paper is organized as follows. Section II describes the proposed method for obtaining codes with rates 1/3, 1/4 and 2/5. Section III provides the simulation results of the described method. Section IV concludes the paper.
II. METHOD DESCRIPTION
This section presents two methods to obtain LDPC codes with rates 1/3, 1/4 and 2/5 from mother code with rate of 1/2. The first one is the conventional method of puncturing information bits, and the other is the proposed method of bit duplication during transmission.
A. Puncturing of information bits method
Puncturing of information bits (PIB) is a method used to reduce the rate of the code by removing some information bits, instead of which all zeros or all ones are inserted that are known to the transmitter and receiver, accordingly, they do not need to be transmitted. They are replaced by infinity [7]. In this paper, we describe this method using the example of obtaining a code with rate of 1/3.

Comparing the performance of two methods at the same BER point 10-5, with different rates, it is clear that the proposed method outperforms the conventional one. At the rate of 1/4, the performance of our code is 0.5 dB better than the PIB code. At the rate of 1/3, the performance of our code is about 0.5 dB superior to the PIB code. And at the rate of 2/5, our code performs about 0.1 dB better than the PIB code.
IV. CONCLUSION In this paper, a new method for the rate-compatible LDPC codes design is proposed. The advantage of this method is that the parity check matrix structure and block length are unchanged, which reduces the cost of the encoder/decoder design. And also the obtained codes are compared with PIB codes using the Matlab modeling environment. Simulation results show that our codes are superior to PIB codes.
