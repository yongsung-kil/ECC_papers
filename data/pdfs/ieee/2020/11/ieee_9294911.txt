High speed LDPC decoding for optical space link

Vincent Pignoly, Bertrand Le Gal, Christophe Jego CNRS IMS Laboratory, University of Bordeaux, France

Benjamin Gadat, Lyonel Barthe Airbus Defense and Space, Toulouse, France

Abstract—In this paper, an architecture to implement horizontal layered Gallager E is proposed to decode QC-LDPC codes onto Field-Programmable Gate Array (FPGA). It arises as an attractive solution for wireless optical up-link space communication because of its low hardware complexity considering constraints in space. Very high throughput of several Gpbs is aimed. Moreover, at this bit rate, space systems imply hard inputs for the decoding process. Gallager E implementation represents an efficient trade-off between throughput, error correction capability and computational complexity. More than 1.5 Gbps are reached by the proposed decoders onto a Xilinx space-grade FPGA. One can note that higher throughput (3 Gbps) can be achieved on other recent FPGAs with increased working frequency.
Index Terms—LDPC codes, Gallager E, FPGA, optical space communication, iterative decoding
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes have become one of the most popular class of forward error correction (FEC) codes. They were proposed by Gallager [1] in 1962 but were too complex for workable implementations at the time. Mackay rediscovered LDPC codes [2] in 1996 and the increasing computing capacity made the use of LDPC codes more realistic. Their performance, close to the channel capacity [3], made them adopted by many standards such as WiFi, WiMax and 5G. Space standards including DVB-S2 and CCSDS also selected LDPC codes. Wireless optical space communication is not yet standardized and the efficiency of custom couples of LDPC codes and decoders has already been recently proven for this application field [4].
To achieve high throughput of several Gbps in optical space communication, FEC decoders are limited to hard input because of available technology. Hard input decoding presents lower error correction performance than soft input decoding. However, additive noise and fading are caused by optical channel involving high needs in terms of error correction capability. To minimize overhead of headers and signaling and because performance grows with block size, this application domain requires long codewords greater than 16k bits. Depending on channel conditions, different code rates, 1/2 and 4/5, are considered. Hardware complexity and power consumption are strongly constrained too. Lastly, a total throughput of 10 Gbps is expected.
Recent works have been done on LDPC decoder implementation with hard inputs on binary symmetric channel (BSC). Efficient FPGA implementation of Probabilistic Gallager B (PGaB) is proposed in [5]. Hinged on Gallager B algorithm

[1], this version consists in adding randomness in the processing of variable to check node messages. Then, an architecture for Probabilistic Gradient Descent Bit Flipping (PGDBF) is presented in [6]. The algorithm is an improvement of GDBF [7] where only a fraction of bits that satisfy the gradient condition is randomly chosen for flipping.
Another way to improve hard input algorithms is to increase the dynamic of data in the decoding process. An extension of Gallager B, that uses a third possible value for messages, has been proposed in [8]. Called Gallager E algorithm, it enables the value 0 in the exchanged message alphabet. It contributes to a very significant increase of error correction capability in exchange for slightly higher memory assignation.
Until now, Gallager E was only presented with a flooding scheduling. However, horizontal layered scheduling gives access to an important parallelism when used with Quasi-Cyclic (QC) LDPC codes. This scheduling highly considered in the literature [4] for high-throughput LDPC decoding.
In this paper, an architecture for Gallager E algorithm is proposed for up-link optical space communication. The resulting decoder reaches application needs in terms of error correction capability, latency, throughput and hardware resource requirements.
The rest of the paper is organised as follows. In section 2, Gallager E with horizontal-layered scheduling is introduced. Then, section 3 presents the decoding architecture implementation with a focus on the processing units. Finally, section 5 talks about decoder performance with regards to the literature.
II. HORIZONTAL-LAYERED GALLAGER E
A. Notations
A LDPC code is an error correction code that is defined by a (N, M ) parity check matrix called H. The code rate R is denoted by R = K/N with K = N − M representing the number of information bits. LDPC codes can be illustrated by a Tanner graph [9] with N variable nodes (VNs) and M check nodes (CNs). CNs connected to VNi and VNs linked to CNj are noted M (i) and N (j) respectively. The number of connections for a node is called its degree. It is denoted dv(i) for VNi and dc(j) for CNj. QC-LDPC codes have a structure composed of (Z, Z) sub-matrices. Each of it is a circulant permutation of the identity matrix.

Horizontal layered scheduling consists in doing the iterative process by considering one check node at a time with variable nodes update between each check node computation. It usually halves the number of necessary iterations for similar error correction performance [10]. Furthermore several check nodes can be simultaneously processed if they are independent. It involves that they are not connected to the same variable node.
The computation of c2v messages can be implemented as detailed in Algorithm 1 to decrease the complexity of decoder architecture. With the flooding scheduling in Equation (2), all v2cik messages except v2cij message are considered for c2vji computation. But for horizontal layered scheduling, it is more efficient to consider all v2cik messages first in Cj and then remove v2cij contribution depending on the aimed VN. With the product in Equation (2), it is not trivial to remove v2cij contribution because it is mathematically impossible. The proposed solution is to count the number of incoming messages equal to 0 and to use this information to compute c2v messages.

The first RAM stores the values of VN accumulator and their associated channel estimations. For LDPC codes with dv = 4, 4 bits are necessary for the accumulators Vi and one bit for channel information yi. So this RAM has N ×5 bits with a width of Z ×5 bits. The second RAM stores messages from check nodes. These messages are quantified on two bits because of the three possible values. The size of this RAM is twice the number of 1 in H with a width of Z × 2 bits. Only one interleaver can be used thanks to the QC-LDPC structure as done in [4]. In practice, a barrel shifter is used to implement the interleaver. At the end of the iterative decoding, a final vote is done considering VN accumulator values and channel estimation to choose the decoded K information bits.
Processing units are described in Figure 3. They are made up of two pipeline stages such as presented in Algorithm 1. The first one computes v2c messages and CN. The second one is in charge of c2v messages and VN update. Each stage takes dc clock cycles so the full process takes 2 dc clock cycles. The Sum0 registers that count the number of null votes is composed of 2 bits. On stage S1, this register is implemented with a simple shift register with an insertion of 1 when the vote is null. It is possible because it is only necessary to know if the value of Sum0 is 0, 1 or above 2 for the computation of c2v as explained in Algorithm 1.

IV. PERFORMANCE EVALUATION
A. Error correction performance
Gallager E algorithm does not present error correction loss once implemented because messages values are quantified in the theoretical algorithm. Figure 5 shows its error correction capability for the (1296, 648) LDPC codes and two home made QC-LDPC codes, especially designed for the optical space application and with friendly attributes for the decoder implementation. Their construction is based on Finite Geometry (FG) [14] with 32768x16384 for C2 and 20480x4096 for C3. These LDPC codes are irregular but with only two different possible dc values in order to make the CN execution efficient. Their average dv is 4 and Z = 256. It means that 256 processing units are assigned in the decoder implementation. It appears that these LDPC codes present a fast convergence and achieves high error correction performance even with 10 decoding iterations. It is a relevant property to reach high throughputs.
B. Hardware complexity and performance
Hardware implementation cost and performance of Gallager E decoder are given in Table I for the selected LDPC codes. The maximum working frequency is 309 MHz on a low speedgrade FPGA. This frequency is reached after synthesis, place and route of the decoder on a Kintex Ultrascale xcku060ffva1156-1 FPGA. This device is the pin compatible commercial part of the space-grade version xqrku060 with on-going QML-Y certification [15]. It is considered for current industrial aerospace systems. Higher frequency can be reached on other FPGAs. For example, the decoder has been implemented with 500 MHz clock constraint on the recent Zynq Ultrascale+ FPGA xczu9eg-ffvb1156-3-e.
Table I presents the hardware complexity of the decoder, its throughput and the latency. For the two long QC-LDPC codes, the same number of LUTs and FFs is necessary. Indeed, data have the same quantification values. Moreover, both decoders contain 256 processing units. But, the (32768, 16384) code needs more RAM Blocks (BRAM) because the codeword is longer and the number of messages higher. With 10 iterations, a throughput of 1.5 Gpbs is reached. At this low noise level, a few number of iterations is needed so the worst case throughput can be much lower. The proposed solution can reach more than 3 Gbps on similar FPGA. But as shown in Figure 4, Gallager E with the C2 code corrects more errors than PGDBF with the C1 code for a similar hardware complexity. V. CONCLUSION
In this paper, a flexible high throughput QC-LDPC decoder
is proposed in the context of wireless optical up-link space
communication. This architectural solution is based on a new
horizontal layered Gallager E algorithm. The association of FG
QC-LDPC codes and Gallager E algorithm is a very efficient
trade-off that fits for the space application.
