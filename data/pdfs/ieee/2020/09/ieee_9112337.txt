A Deep Learning Assisted Node-Classified Redundant Decoding Algorithm for BCH Codes
Bryan Liu , Graduate Student Member, IEEE, Yixuan Xie , Member, IEEE, and Jinhong Yuan , Fellow, IEEE,

Abstract‚ÄĒ This paper proposes a node-classified redundant decoding (NC-RD) algorithm based on the received sequence‚Äôs channel reliability for high-density parity-check (HDPC) codes. Two preprocessing steps are proposed prior decoding. The variable nodes of the parity-check matrix are firstly classified by the k-median algorithm based on the number of shortest cycles associated with each variable node before decoding. Then, by searching among the automorphism group of the HDPC codes, we generate a list of permutations for bit positions by computing and sorting the permutation reliability metrics. The redundant decoder conducts the message-passing decoding according to the sorted permutations, which limit the unreliable information propagation for each permutation. Besides proposing a list decoding algorithm on top of the NC-RD algorithm to augment the decoder‚Äôs performance, we show that the NC-RD algorithm can be transformed into a neural network system. More specifically, multiplicative tuneable weights are attached to the decoding messages to optimize the decoding performance. Simulation results of BCH codes over the AWGN channels show that the NC-RD algorithm provides a performance gain compared to the random redundant decoding algorithm. Additional decoding performance gain can be obtained by both the list decoding method and the neural network ‚Äúlearned‚ÄĚ NC-RD algorithm.
Index Terms‚ÄĒ High-density parity-check codes, iterative decoding, deep learning.
I. INTRODUCTION
L OW-DENSITY parity-check (LDPC) codes with iterative belief-propagation (BP) decoding algorithm have shown to approach the capacity limit for various channels [1], [2]. However, the BP performance is unsatisfying for high-density parity-check (HDPC) codes, such as the Bose‚ÄďChaudhuri‚Äď Hocquenghem (BCH) codes and the Reed-Solomon codes, due to a large number of short cycles in the codes‚Äô parity-check matrix (PCM) [3]. In recent years, many pivotal works have been proposed on how to improve the decoding performance of HDPC codes under soft-decoding [4]‚Äď[9]. More specifically, the group of bits with least accumulated a posterior probabilities (APPs) are allocated to the systematic part of the PCM after Gaussian elimination. This results in less unreliable information propagation during the message-passing decoding. To reduce the decoding complexity induced by sortation and Gaussian elimination of the ABP algorithm, a random redundant decoding (RRD) algorithm was proposed in [5]. During the decoding iterations, the positions of the bits are permutated following a oneto-one correspondence permutation function to perform the redundant decoding and reduce the accumulation of correlation between the bits in each iteration. The redundant syndromes are explored by the permutations of the bit positions and the permutation function is randomly chosen from a set of permutation functions that send the code to itself, where the permutation set is also known as the automorphism group. Further to the work in [5], a modified RRD (mRRD) algorithm was proposed in [6] and it has shown that by decoding the received sequence with distinct permutations in multiple times, a list of codewords can be generated to improve the decoder‚Äôs performance. The RRD and mRRD algorithms appear to approach the maximum likelihood (ML) decoding performance for very short-length and high-rate HDPC codes. However, for medium-length and low-rate HDPC codes, the detrimental influence of the large number of shortest cycles in the PCM becomes severe. The unreliable information could propagate if an unreliable bit is allocated to the variable node (VN) connected to a large number of check nodes. This results in unsatisfying soft-decoding performance.
In recent years, machine learning has shown to be a promising topic for the future development of the wireless communications [10]‚Äď[13]. Numerous deep learning assisted channel detection and decoding algorithms have been proposed to enhance the detection or decoding performances [14]‚Äď[20]. Particularly, regarding the decoding algorithms for HDPC codes, a ‚Äúlearnable‚ÄĚ BP algorithm was introduced by Nachmani et al. [21], [22], and Be‚Äôery et al. [23]. The traditional BP algorithm is transformed into a tuneable BP neural network framework (BP-NN). Instead of constructing the neural network in a traditional way, where all the neurons are fully-connected, the neurons in the BP-NN are connected following the decoding structures of the BP algorithm. Besides the investigations on HDPC codes [21]‚Äď[23], the weighted message-passing algorithms have been explored in various codes such as LDPC code [18], Polar code [24], [25] and Quantum-LDPC code [26]. These neural network decoding algorithms are constructed based on the connections of the Tanner graph and the multiplicative weights are employed to the decoding messages to improve the accuracy of the decoding messages. These works demonstrate the effectiveness of the neural network message-passing decoder and inspire researchers to investigate new soft-decision decoding algorithms, together with the neural network transformation, to further augment the decoder‚Äôs performance.
In this paper, we propose a node-classified redundant decoding (NC-RD) algorithm for a type of HDPC codes, the BCH codes. The preprocessing method introduced in [27] for a neural network decoder is further investigated in the message-passing decoding algorithm. With the adoption of the k-median algorithm, we propose to preprocess and classify the VNs according to the number of shortest cycles they involved in. Then, we propose a permutation reliability metric (PRM) based on the classified VNs and the reliability of the received bits. Instead of employing the random permutation as the RRD algorithm, we generate a sorted permutation list based on the PRM to conduct redundant decoding. Therefore, we name the proposed algorithm as node-classified redundant decoding algorithm. The proposed NC-RD algorithm performs as a soft-input soft-output decoding algorithm and it does not require the Gaussian elimination process as the ABP algorithm or the ordered-statistical decoding (OSD) algorithm [28]. On top of the NC-RD algorithm, a modified NC-RD (mNC-RD) algorithm is further complemented to enhance the decoder‚Äôs performance. Inspired by the mRRD algorithm, the mNC-RD algorithm generates a list of codewords by decoding the received sequence in multiple times, while the permutation of the bit positions in each time of decoding is selected from a preprocessed permutation list, instead of the automorphism group as in [6]. The preprocessed permutation list is a list of permutation to restrain the propagation of the unreliable information and during each time of decoding, the mNC-RD algorithm performs different permutation orders based on the preprocessed permutation list to increase the opportunities for the decoder to converge to the correct codeword.
Moreover, given the number of decoding iterations, we transform the NC-RD algorithm to a neural network system to optimize the scaling factors that are attached to the intrinsic information to be conveyed to the VNs in the proposed algorithm. The neural network is constructed following the decoding structure of the NC-RD algorithm. The neurons‚Äô connections in the neural network follow the updating rules of the decoding messages, where the decoding messages are updated based on the BP algorithm. This indicates that for a binary-input symmetric channel, the neural network decoder is ‚Äúlearnable‚ÄĚ from all-zero codewords corrupted by additive white Gaussian noise (AWGN). After an off-line training, it is

expected that the NC-RD neural network (NC-RDNN) decoder could compensate the detrimental effect of the large number of length-4 cycles in the PCM of the HDPC codes. Different from the conventional neural network message-passing decoding algorithms which have uniform neural network connections for all the received code sequences, origin from the preprocessing of the NC-RD algorithm, the NC-RDNN decoder‚Äôs neurons‚Äô connections are adaptive to the received code sequences. Simulation results of BCH codes show that the proposed NC-RD algorithm provides up to 0.5 dB performance gain compared to the conventional RRD algorithm in [5] under the same number of BP iterations. Furthermore, we simulated the mNC-RD algorithm to demonstrate the performance gain obtained by the list decoding method. A ‚Äúlearned‚ÄĚ NC-RD algorithm (NC-RDNN) is simulated and additional 0.18 dB and 0.2 dB gains are obtained for decoding the BCH(31,21) code and the BCH(63,36) code, respectively.
This paper is organized as follows. In the next section, we will briefly overview the RRD algorithm and in Section. III, we introduce the proposed NC-RD and mNC-RD algorithms. The detailed architecture of the neural network and the NC-RDNN algorithm are discussed in Section. IV. In Section V, we demonstrate the numerical results of the proposed schemes and the concluding remarks are presented in Section VI.

where u ‚ąą {0, . . . , m ‚ąí 1} and o ‚ąą {0, . . . , n ‚ąí 1}. The permutation is selected from the automorphism group, which contains all the permutations which send the code C into itself. For instance, any cyclic code has an automorphism group which includes a cyclic shift permutation [30]. By randomly permutating the bit positions, the redundant syndromes are checked correspondingly, which reduces the fast growth of the correlations between the decoding messages [4], [5]. Extension work of the RRD algorithm, such as the mRRD algorithm in [6], also implies random permutation to the bit positions, but the mRRD algorithm decodes the received sequence in multiple times and each time the decoder performs different permutations that are randomly selected from the automorphism group. In our work, instead of performing random permutation of the bit positions, we propose to permutate the bits following an order of the sorted permutation list which is generated based on the channel reliabilities of the received signals. This limits the accumulations of the correlations between the bits and reduces the effect of the unreliable information in the iterative decoder.

III. PROPOSED NODE-CLASSIFIED REDUNDANT DECODER
In this section, we will detail the proposed NC-RD algorithm. Since the deep learning is employed on top of the proposed NC-RD algorithm, in the following, we will firstly introduce the NC-RD algorithm. Specifically, compared to the conventional RRD algorithm, the NC-RD algorithm is proposed by introducing two preprocessing steps to limit the propagation of the unreliable information. The details for the VNs classification are described in Section III. A and Section III. B describes the computation of the PRMs. Section III. C delivers a toy example to interpret the proposed NC-RD algorithm. Lastly, in Section III. D, we propose a list decoding method, the mNC-RD algorithm, to augment the decoding performance of the NC-RD algorithm.
A. Variable Nodes Classification
In [27], Bennatan et al. proposed the preprocessing method that searches among the Per(C) to decide the bit positions to be permutated before decoding. The preprocessing method suggests to permutate the bit positions before decoding so that the sum of the reliabilities in the dense part of a systematic form of the PCM is maximized. The preprocessing method focuses on the systematic form of the PCMs and only one permutation in Per(C) is selected to be used for the decoder.
In our work, we generalize the preprocessing metric with the VNs which include a smaller number of shortest cycles in the PCM of the HDPC codes. In other words, we evaluate the number of shortest cycles that is contained by the VNs as a metric to preprocess the permutations to be invoked during the decoding once a code sequence is received. It is widely shown that for message-passing decoding algorithms, the decoder‚Äôs performance is highly affected by the short cycles in the PCM. Any VN that contains a smaller number of shortest cycles is considered to have less impact on other VNs and the bit associated with that VN is more likely to converge to the APPs [4], [31]‚Äď[35]. Following the concept to restrain the unreliable information to be propagated to other VNs, the idea of our work is to allocate the unreliable bits to a set of positions which contain a smaller number of shortest cycles. More importantly, for every decoding iteration, the unreliable bits based on the channel reliabilities are restrained by different syndromes and eventually, it is expected that the unreliable error bits are more likely to be corrected.
To identify the set of positions for the unreliable bits to be permutated in each iteration, we propose to classify the VNs of the PCM according to the number of the shortest cycles that each VN contains before decoding the received code sequence.
In particular, we propose a simple VNs classification method by using the k-median algorithm [36]. The k-median algorithm is known to optimize the L1-norm of a data-set [37]. Given a PCM H of a code, the non-zero elements in H are involved into different number of shortest cycles. This directly infers that the VNs contain irregular number of shortest cycles, as the positions of the non-zero elements in each column (VN) of H are different.
Let us define g = [g0, g1, . . . , gn‚ąí1], where gi is the number of the shortest cycles contained by VN vi, i ‚ąą {0, . . . , n‚ąí1}.

The proposed algorithm is simple to be employed for a given PCM. Since Algorithm 1 performs based on the number of shortest cycles that the VNs are involved in the PCM, it is general to different types of channel codes. Specifically, the proposed algorithm classifies the VNs into k clusters where the VNs of each cluster contain similar numbers of shortest cycles. We point out that the VNs classification is based on the characteristics of the PCM. Once the VNs are classified, the position set s0 is maintained and can be used for each received sequence. By permutating the unreliable bits to the set of VNs which contain less shortest cycles (length-4 cycles) for HDPC codes, the detrimental effect of these unreliable information propagation is reduced and these unreliable bits will be more likely to be corrected by the decoder.

In message-passing decoding algorithms for HDPCs codes, it is essential to avoid the accumulation of correlations between the bits during the iterations to achieve a good decoding performance [4], [5]. An effective way to reduce the correlation is to explore the redundant syndromes of the HDPCs codes by permutating the bit positions before the start of the next decoding iteration. Meanwhile, in order to limit the unreliable information to be propagated to other VNs, we propose to permutate the bit positions by following the sorted permutation list ŌÄsort which permutates the unreliable bits to the VNs with less number of shortest cycles. An inverse permutation (ŌÄhsort)‚ąí1 is appended to permutate the bit positions back to the origin for each h-loop iteration. Note that besides limiting the propagation of the unreliable information, permutating the bit positions by following the sorted permutation list prevents the decoder from selecting one permutation multiple times during the decoding, which could degrade the decoding performance.
We point out that using the channel reliabilities to compute the PRMs turns out to be a preprocessing step for the decoder. The PRMs can be computed in parallel and once the permutation list is finalized, the decoder does not need to search for the PRM in a second time during the decoding. This largely reduces the computation latency during the iterative decoding.

In summary, the propagation of the unreliable information during the decoding is restricted by allocating the unreliable bits to the VNs which contain no length-4 cycles in this example.
D. The mNC-RD Algorithm
The proposed NC-RD algorithm reduces the propagation of the unreliable information by allocating the unreliable bits to the group of VNs which contain the number of least shortest cycles. However, since the PCMs of the HDPC codes contains a large number of length-4 cycles, it is difficult for the decoder converges to the correct codeword in one round of decoding. To further enhance the decoding performance, inspired by the mRRD algorithm proposed in [6], we propose a list decoding method to generate a list of decoded codewords by decoding the received sequence several times with different permutation orders in the h-loop iterations. Here we name it as modified NC-RD (mNC-RD) algorithm.
The mRRD algorithm in [6] decodes the received sequence in multiple times and the permutation of the bit positions in the h-loop is randomly selected from the automorphism group. Inspired by this idea, our proposed mNC-RD list decoding algorithm randomly shuffles the permutation order from the preprocessed permutation list ŌÄprep. More specifically, the received sequence is decoded in multiple times as in [6], but each time the permutation order in the h-loop is shuffled based on the preprocessed permutation list. Since the preprocessed permutation list is generated to limit the unreliable information propagation during the message-passing decoding, selecting the permutation from the preprocessed permutation group will restrict the unreliable information propagation compared to randomly select the permutation from the automorphism group. The numerical results of decoding BCH codes by the mNC-RD algorithm will be shown in Section V.

B. Influence of the Number of Clusters for Classifying the VNs
In order to demonstrate the effect of the number of clusters to be preset prior decoding, in Fig. 9, we show the decoding performance of the BCH(255,207) and BCH(63,36) codes with different number of clusters preset for the NC-RD algorithm. The more number of clusters, the closer of the centroids between two adjacent clusters. In other words, the smaller the size of s0, which could influence the NC-RD algorithm‚Äôs decoding performance. The value of őĪ is set to be 0.05 and 0.1 for decoding the BCH(255,207) and BCH(63,36) codes, respectively.
As we previously discussed, the NC-RD algorithm intends to limit the propagation of the unreliable information during the message-passing decoding. For the BCH(63,36) code, when the VNs are classified into 7 clusters, the size of s0 is small (|s0| = 4), whereas some unreliable information could be allocated to the VNs with a relatively large number of shortest cycle that could degrade the decoding performance. That is why we see that the performance with 7 clusters of the VNs is 0.15 dB worse than the NC-RD algorithm with 2 to 4 clusters preset. For a longer BCH code, e.g. BCH(255,207), most of the VNs of the PCM contains a large number of shortest cycles, the decoding performances are not significantly different for various number of clusters when implementing the NC-RD algorithm.

C. Preprocessing Complexity Analysis
The generation of the preprocessed permutation list is pivotal for the proposed mNC-RD algorithm, which improves the decoding performance compared to the conventional RRD algorithm. The preprocessed permutation list is generated by the preprocessing steps which include the computation of the PRMs and the sortation of the PRMs. For a BCH code with codeword length n = 2m ‚ąí 1, regarding the complexity of computing the PRMs, the proposed NC-RD algorithm performs real additions, where the number of permutations grows as O nlog(n + 1) . The complexity of the addition process depends both on the codes‚Äô length and the cardinality of the cluster s0, which relies on the non-zero positions in the PCM. Therefore, in total, the preprocessing requires nlog(n + 1) √ó |s0| real additions for computing the PRMs. Furthermore, regarding the complexity of sortation of the PRMs, the NC-RD decoder‚Äôs sortation complexity depends on the number of permutations (h-loop iteration). Let hm ax be the number of permutations to be found based on the PRMs, i.e., hmax = min{hmax, m √ó n}. The NC-RD algorithm finds the hmax permutations of the minimum PRMs with a time complexity of O nlog(n + 1) + hmaxloghm ax . The smaller the number of h-loop iterations, the less processing time of sortation of the PRMs.
VI. CONCLUSION
In this paper, we proposed an NC-RD algorithm by introducing two preprocessing steps, the VNs classification and the PRMs sortation. The NC-RD algorithm explores the redundant syndromes and limits the propagation of the unreliable information during the message-passing decoding. Besides proposing a modified NC-RD algorithm which generates a list of codewords to augment the decoder‚Äôs performance, we show that the proposed NC-RD algorithm is ‚Äúlearnable‚ÄĚ by a neural network system. Numerical results show that the proposed NC-RD algorithm outperforms the conventional RRD algorithm and additional performance gains can be obtained by both the list decoding method and the ‚Äúlearned‚ÄĚ NC-RD algorithm.
