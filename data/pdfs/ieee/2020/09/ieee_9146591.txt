Concatenated LDPC/2-D-Marker Codes and Non-Iterative Detection/Decoding for Recovering Position Errors in Racetrack Memories
Ryo Shibata 1, Gou Hosoya2, and Hiroyuki Yashima1
1Department of Information and Computer Technology, Faculty of Engineering, Tokyo University of Science, Tokyo 125-8585, Japan
2Global Education Center, Waseda University, Tokyo 169-8050, Japan
Racetrack memories (RMs) are prone to alignment faults called position errors (PEs), which manifest as insertions and deletions of stored data bits. Conventional coding schemes for PEs demonstrate the promising results by employing low-density parity-check (LDPC) codes with an iterative detection and decoding algorithm. However, the computational complexity of detection is relatively high. In this article, for channels with PEs, we present a new coding scheme that can effectively decode corrupted data, even if the detection is executed only once (i.e., a non-iterative detection and decoding scenario). The proposed code consists of a concatenation of an inner 2-D marker code, which is specialized for PEs to mitigate the effect of insertion and deletion (ID) errors, and an outer irregular LDPC code. We also provide tractable design methodologies for these constituent codes. First, we identify the 2-D-marker code structures that offer higher achievable information rates in a non-iterative scenario and then optimize irregular LDPC codes to ensure good decoding properties. Through asymptotic-performance analysis and finite-length simulation, we confirm the effectiveness of the proposed coding scheme. Ultimately, the proposed coding scheme has the capability to reduce code rate loss and to provide excellent decoding performance under a non-iterative scenario, which also helps in understanding the reliability of RM when a low-complexity decoding algorithm is used to correct ID errors caused by PEs.
Index Terms— Low-density parity-check (LDPC) codes, position error (PE), racetrack memories (RMs), synchronization marker.

I. INTRODUCTION
R ECENTLY, racetrack memory (RM) or domain-wall memory [1]–[3] has been attracting significant attention as a promising candidate for a perceived non-volatile memory technology because of its potential to provide ultra-high storage density and low power consumption. Over the past decade, many researchers have developed various applications based on RM as universal memory technology, such as cache [4], digital signal processors [5], and storage [6]. RM stores data bits on a nanowire track, where each bit can be accessed with a fixed read port (RP). Accessing a data bit can be accomplished by shifting all of the data bits in either direction along the track such that the target bit is positioned over the RP. To improve latency, multiple RPs are usually employed in the RM, where multiple different data bits can be accessed simultaneously by shifting the track. Unfortunately, the bitserial access mechanism suffers from an unusual error, referred to as a position error (PR) [7], [8], which is known as one of the reliability issues of RM that must be overcome. A PE includes two kinds of imperfectly shifting errors, caused by the over-shifting and under-shifting of the track, which can be modeled as deletions and sticky insertions of the stored bits [7], [9], [10], respectively. A deletion error occurs because of an over-shifting of the track such that it is shifted by more than expected and the target bit is skipped. A sticky-insertion error occurs because of an under-shifting of the track, such

At first glance, insertion and deletion (ID) errors caused by PEs do not seem to be different from conventional types of ID errors. Hence, it would be possible to utilize conventional synchronization error-correcting schemes [11], [12] for tackling ID errors caused by PEs. Meanwhile, there are error-correcting schemes that are specialized for ID errors caused by PEs [9], [10], [13], [14]. Based on the algebraic number theory [9], [13] and modern (stochastic) coding theory [10], [14], these coding schemes leverage the fact that a PE occurs at all the RPs simultaneously.
Shibata et al. [10] introduced a stochastic memory-channel model of ID errors caused by PEs and provided the symmetric information rate (SIR) of a PE channel for different numbers of RPs, which revealed the relationship between the reliability of rms and the number of RPs. Moreover, an iterative algorithm consisting of maximum a posteriori (MAP) detection and low-density parity-check (LDPC) code decoding was proposed in [10]. With this iterative algorithm, spatially coupled codes and well-designed protograph-based LDPC codes have yielded promising results in terms of approaching the SIR of a PE channel [10], [14]. However, the iterative algorithm has a drawback with respect to computational complexity because the detector is implemented with the Bahl–Cocke–Jelinek– Raviv (BCJR) algorithm [15]. Unfortunately, the conventional coding schemes have not prepared for a non-iterative MAP detection and LDPC decoding scenario. In fact, under the non-iterative scenario, we found that the conventional coding schemes provide poor decoding performance in correcting ID errors caused by PEs.

In this article, for the PE channel, we propose a promising coding scheme that is effectively decodable with a non-iterative scenario. The basic idea of the proposed coding scheme involves a concatenation of an outer LDPC code and an inner marker code [16], where a transmitted sequence is formed by inserting markers (so-called pilot bits) into an LDPC codeword at regular intervals. The markers provide an advantage in terms of recovering synchronization impaired by ID errors. The proposed coding scheme utilizes a concatenation of an outer irregular LDPC code and an inner 2-D marker code. In contrast to conventional marker codes [16], 2-D-marker codes are specialized for a PE channel in terms of letting the MAP detector leverage the fact that a PE occurs in all the RPs simultaneously. To improve both achievable information rates and error-correcting performance in the non-iterative scenario, we also propose tractable design methodologies for each constituent code, i.e., 2-D-marker codes and irregular LDPC codes. We present some design examples of the concatenated LDPC/2-D-marker codes and demonstrate their effectiveness in terms of both the achievable information rate and asymptotic/finite-length decoding performances. These results can likewise provide new findings useful for understanding the relationship between the number of RPs and the reliability of RM when a low-complexity decoding algorithm is applied to correct ID errors caused by PEs.
The remainder of this article is organized as follows. In Section II, we review a stochastic model of PE channels, the MAP detection for the channels, and marker coding basics. In Section III, we outline the concatenated LDPC/2-Dmarker coding scheme. In Section IV, we describe the design methodology of inner 2-D-marker codes and present some design results. In Section V, we describe the optimization of outer LDPC codes, show some design results, and demonstrate asymptotic/finite-length performances of the designed concatenated coding scheme. In Section VI, we present our conclusions.

In this section, we review a PE channel model, the MAP detection for the PE channel, and a general marker coding scheme.
A. PE Channel Model
We considered the binary input–output PE channel model introduced by [10], where a PE is expressed as either an insertion or a deletion error. This model supposes that arp read ports (RPs) are employed to read stored data bits and ID errors caused by the PE occur with an equal probability of pid/2. Similar to [7], this study supposes that there are no bit-flip errors in RM.

C. Marker Coding Basics

For usual ID channels, there is a coding scheme that realizes

a considerable decoding performance under a non-iterative

scenario between a detector and a decoder, which is achieved

by a concatenation of outer LDPC codes and inner marker

codes [16]. The inner marker codes are predetermined bit

strings shared with the transmitter and receiver and are inserted

into an outer LDPC codeword at regular intervals. The markers

provide reliable a priori information for a detector and help in

removing the effects of ID errors from received bits. Putting

III. CONCATENATED LDPC/2-D-MARKER CODING SCHEME
A. System Model
This section outlines the concatenated LDPC/2-D-marker coding scheme, which is specifically devised for the non-iterative detection and decoding scenario for PE channels. The proposed scheme is shown in Fig. 1, where an outer LDPC code with a rate of rout and an inner 2-D-marker code with a rate of rin are used. Hence, the entire concatenated coding rate is r = rin · rout.
At the transmitter side, a binary information sequence b ∈ {0, 1}k is encoded into an LDPC codeword c ∈ {0, 1}nL by the outer LDPC encoder, and a 2-D-marker codeword m ∈ {0, 1}nM is embedded into the codeword c, thereby forming a 2-D input sequence X ∈ {0, 1}arp×n. Moreover, X is sent to a PE channel. Therefore, rout = k/nL, rin = nL/(nL + nM), and r = k/(nL + nM). At the receiver side, the MAP detector receives a 2-D output sequence Y ∈ {0, 1}arp×n and computes LLR messages for the LDPC decoder. Thereafter, the LDPC decoder delivers an estimate bˆ ∈ {0, 1}k, where only iterations within the decoder are performed.
The proposed coding scheme is characterized by the following four parameters: 1) the inner 2-D-marker coding rate rin; 2) the positions of the 2-D-marker codeword m in a 2-D channel input sequence X; 3) the outer LDPC coding rate rout; and 4) outer LDPC code parameters (e.g., degree distributions and protographs). In this study, we assumed that a 2-D-marker codeword m is a predetermined random binary sequence shared with the transmitter and receiver.

IV. DESIGN OF INNER 2-D-MARKER CODES
Our objective is to design a regular 2-D-marker code for PE channels such that the once bound Conce with the designed

code is as high as possible. Unlike the conventional marker code design for an ID channel [16], we need to select arp numbers of marker intervals (d(1), . . . , d(arp)) simultaneously. However, as arp increases, it becomes more difficult for us to compute Conce for all the possible combinations of the intervals. Therefore, in this study, we propose a tractable ad hoc design strategy as follows: 1) choosing a candidate inner coding rate r˘in that maximizes Conce under the random 2-D-marker coding scheme and 2) determining the regular 2-D-marker code structure (i.e., combination of intervals d( j)) that is able to maximize Conce and satisfy rin ≈ r˘in. In the remainder of this section, we will show four rules we derived for formalizing the design strategy.
A. Choosing the Candidate Inner Coding Rate
Our first task is to choose a candidate inner coding rate r˘in. For practical reasons, we considered the decimal places of r˘in up to κ digits. arp is the number of RPs of the design interest and pid indicates the target PE probability. We picked r˘in according to the following rule:
Rule 1: Choose r˘in (0 ≤ r˘in ≤ 1) that is able to maximize a BCJR-once bound Conce evaluated at the PE channel with arp and pid under the random 2-D-marker coding scheme.
The search for r˘in can be conducted systematically, starting with rin = 0 and increasing rin by 10−κ . Thereafter, we selected rin that is able to maximize Conce as r˘in.
As an example, we considered the PE channel with arp = 2 and pid = 0.05. The BCJR-once bound Conce under the random 2-D-marker coding scheme as a function of rin with κ = 2 is shown in Fig. 3. According to Rule 1, the candidate inner coding rate of r˘in = 0.81 is obtained because Conce is maximized at rin = 0.81.

VI. CONCLUSION
In this article, we have proposed a coding scheme based on a concatenation of an inner 2-D-marker code and an outer LDPC code for PE channels. The proposed concatenated codes were devised for a non-iterative MAP detection and LDPC decoding algorithm, i.e., a computation saving decoding scenario. The 2-D-marker code was specialized for PE channels, thereby enabling the MAP detector to exploit the fact that a PE simultaneously occurs at all of the RPs. To achieve a higher information rate with the MAP detection being applied only once, first, we provided an ad hoc methodology for designing 2-D-marker codes. Thereafter, we provided an optimization method for outer LDPC codes to ensure good decoding properties. Through asymptotic-performance analyses and finite-length simulation, we confirmed the effectiveness of the designed concatenated LDPC/2-D-marker codes. Ultimately, the proposed coding code is capable of reducing code rate loss and of providing excellent decoding performance under a non-iterative scenario, thereby contributing to the understanding of the reliability of RM when a computation saving decoding scenario is assumed to correct ID errors caused by PEs.
