The 5G New Radio Code: Elementary Absorbing Sets and Error Floor Performance

Abstract—The data channels in the 5G New Radio use quasicyclic low-density parity check (LDPC) codes for error control. Such codes, when used with a practical quantizer and an iterative decoder, typically have an error floor that is strongly impacted by absorbing sets present in the code’s structure. This paper leverages the particular structure found in the 5G code to describe a simple means of identifying its (elementary) absorbing sets and describes an algorithm for finding them. It then uses simulation results to show which of the absorbing sets are responsible for the error floor performance for a particular choice of code and quantizer parameters.
I. INTRODUCTION
The evolution from 4G to 5G cellular standards has resulted in a profound change in the error control codes used to ensure robustness - from the turbo codes used in 3G UMTS and 4G LTE to the low-density parity check (LDPC) [1], [2] and polar codes used in the data and control channels (respectively) in the 5G New Radio (NR) technology [3].
The main motivation behind the turbo-to-LDPC transition for data are NR’s stringent requirements: it must support high throughput, improved coverage, and high reliability as well as low latency applications that necessitates the support of a wide range of block lengths and rates – all in an incremental redundancy hybrid automatic repeat request (IR-HARQ) [5] protocol that requires rate compatible codes. LDPC codes provide higher coding gains than turbo codes, and they need remarkably fewer decoding operations to achieve the same frame error rate (FER) at a particular signal to noise ratio (SNR) [6].
It is common to implement LDPC decoders based on iterative message passing algorithms such as the sum-product algorithm (SPA) or the min-sum algorithm (MSA). Although these algorithms provide near-optimal performance at low SNR, their performance is often marred by the presence of an error floor, i.e. a (typically abrupt) change in the slope of the FER curve with respect to SNR [7]. The appearance of the error floor is attributed to absorbing sets (AS), substructures in the code that cause the iterative decoder to fail in the high SNR regime.
It may not be obvious why we should care about the error floor performance (and therefore the absorbing sets) of the NR LDPC code. As noted above, cellular systems use an IR-HARQ protocol that relies on re-transmission(s)

to achieve an acceptable error rate; the LDPC decoder by itself is not required to operate in the high-SNR (error floor) regime. However, the impending ubiquity of the 5G NR code - the billions of decoders that will be produced over the next ten years - suggests that, not only should its fundamental properties be well understood, but also that it may well be used outside the cellular context. For many applications (such as deep satellite communications), a low error floor is required, but structured LDPC codes like the 5G NR code, combined with practical finite-precision hardware, result in a high error floor [8]. Understanding the error floor performance of the 5G NR code may yield insights that will prove useful.
II. BACKGROUND AND PRELIMINARIES
A. QC-LDPC Codes: Definitions and Practical Issues
A quasi-cyclic LDPC (QC-LDPC) code can be constructed by assembling its parity check matrix from blocks of circulant matrices. The structure of such a code is represented by a small protograph or (equivalently) base graph matrix. Each entry in the base graph matrix is replaced with a square Z ×Z block to generate the code’s parity check matrix; this process is called lifting and Z is the lifting value. Each “0” in the base graph is replaced by an all-zero block, and each “1” is replaced by an identity matrix that is circularly shifted by a particular shifting value that depends on the location of the “1” in the base graph.
The SPA, MSA and their variations are iterative message passing algorithms that can achieve error rates very close to the channel’s Shannon limit. In a message passing algorithm, extrinsic information is passed along the edges of the code’s Tanner graph [9] and is updated during each iteration until the decoder converges or the maximum number of iterations is reached. These messages are in fact log likelihood ratios (LLR’s). Any practical implementation of a message passing decoder must inevitably use a finite (and small) number of bits to represent each LLR, and the resulting fixed word length effects can significantly degrade the decoder’s performance [8]. This paper employs uniform quantizers - more precisely, quantizers wherein d bits are used to represent the integer part of the LLR value, f bits are used to represent the fractional part, and a single bit indicates its sign, resulting in a q-bit quantizer, where q = d + f + 1. In such a scheme, which is referred to as a d.f quantizer, there are 2q quantization levels

B. Absorbing Sets and the Error Floor
In [10], Richardson developed the notion of a trapping set (and, more specifically, an elementary trapping set) to refer to the set of bit locations in a codeword where a message passing decoder tends to fail at high SNR. A code’s trapping sets depend on the kind of communication channel the code is used on and the kind of decoding algorithm that is employed. This idea was further developed in [11], and then in [12] the notion of an absorbing set was introduced - a special kind of trapping set that discloses the most error-prone substructures in the Tanner graph of an LDPC code on an additive white Gaussian noise (AWGN) channel. The collection of absorbing sets associated with a code is determined by the combinatorial properties of the code (i.e., its parity check matrix), independent of the channel or the decoder [7]. Each absorbing set associated with a code represents “weak spots” in the code’s structure - positions where significant channel noise can cause the decoder to get trapped, unable to converge to the most likely codeword estimate.
Denote the Tanner graph associated with a code’s parity check matrix by GH = (C, V, E), where C and V are respectively the sets of all check nodes and variable nodes and E is the set of all the edges connecting the neighboring nodes to one another. For a subset D of V , let O(D) and E(D) be the subsets of C that contain the neighboring check nodes of D with odd degree and even degree (with respect to the variable nodes in D), respectively.
• Definition 1: Given an integer pair (a, b), an (a, b) absorbing set is a subset D of V of size a, with O(D) of size b and with the property that each element of D has strictly fewer neighbors in O(D) than in C \ O(D) [7].
• Definition 2: An (a, b) absorbing set with O(D) of size b and E(D) of size c is elementary if all b check nodes are of degree one and all c check nodes are of degree two.
III. ABSORBING SETS IN THE 5G NEW RADIO CODE
The 5G NR channel code is a QC-LDPC code with two base graphs, BG1 and BG2; BG1 is used for longer blocklengths and higher rates, while BG2 is intended for shorter blocklengths and lower rates [3], [4]. The two base graph matrices have a similar structure; for reference, BG2 is illustrated in Fig. 1. They both include a dense (high rate) part called the core (red rectangle) and an extension part that adds additional redundant bits that are each the XOR of a set of bits within the core. The extension part enables the IR-HARQ protocol, in that additional parity bits can be generated for re-transmission as needed. For brevity, we focus on codes derived from BG2, a 42×52 binary matrix with a 4×14 core, but the observations and techniques are equally applicable to codes derived from BG1. (For more details, see [6].)

performance. First, a trivial observation: Any absorbing set in the 5G NR code must include at least one variable node in the codeword “core”, i.e. in the first 14Z positions of the codeword. This is because any sub-graph that contains only variable nodes from outside the core will be composed of multiple disjoint sub-graphs, each consisting of just a single variable node connected to a single check node. Keeping this in mind, we first focus on absorbing sets that include exactly one variable node in the core; in particular, we focus on elementary absorbing sets (EAS’s) with that property.
Here is a procedure for finding an elementary absorbing set with exactly one variable node in the core:
• Pick one column in the core with the property that there are more ones in that column outside the core than inside the core. Let i0 denote the index of that column1.
• Assume there are b ones in the selected column inside the core and c = a − 1 ones in the selected column outside the core. So by selection, b < a − 1.
• Let h1, h2, ..., hb index the rows inside the core that contain the b ones in the selected column.
• Let j1, j2, ..., j(a−1) index the rows outside the core that contain the c = a − 1 ones in the selected column.
• Let i1, i2, ..., i(a−1) index the columns outside the core that contain ones in rows j1, j2, ..., j(a−1).
• By construction, columns i0, i1, i2, ..., i(a−1) identify a variable nodes and rows h1, ..., hb, j1, ..., j(a−1) identify a + b − 1 check nodes. These check and variable nodes together form an (a, b) absorbing set with b check nodes of degree one and c = a − 1 check nodes of degree two, where b < c - i.e., an elementary absorbing set.

There are at least three differences that must be accommodated when looking for EAS’s with two variable nodes in the core compared with just one. First, it is possible to have two such variable nodes associated with columns from the same set of Z columns in BG2. Second, columns 4, 5, and 9 can now participate in the construction when combined with other core columns. Finally, there is a dependency on Z that did not exist in the single-variable node case.; for example, the combination of columns 9 and 13 yields a (12,3) EAS for Z = 32 as the smallest EAS, while for Z = 16 the same combination yields a (12,1) EAS as the smallest. We care about the smallest EAS because our simulation results showed that given a specific combination of core columns, only the smallest EAS that is constructed by that combination may become problematic in the error floor.
The first step to identifying all the two-core-column EAS’s smaller than amax for a specific lifting value Z is to determine all the possible combinations of core columns that yield EAS’s with a < amax. Then, the next step is to identify the neighboring check nodes and the extension variable nodes connected to the neighboring check nodes and to examine these combinations to find the smallest (a, b) EAS’s. Identifying and cataloging elementary absorbing sets with more than two variable nodes from the core is far more complicated than the cases considered above, and the reader is referred to [14] for more details. However, many of problematic EAS’s observed in the simulation results included more than two variable nodes from the core, such as the (25,2) EAS and the (34,1) EAS that were prevalent in the error floor region of the code lifted from BG2 with a lifting value of Z = 32.

B. Effect of Puncturing and Shortening on Error Floor Performance
The 5G NR standard specifies that 2Z information bits at the beginning of each codeword are to be removed (“punctured”) prior to transmission; these 2Z punctured bits are assigned LLR values of 0 (equally likely to be 0 or 1) at the decoder, but because they correspond to two columns in the dense core of the base graph, they participate in many parity checks and can typically be recovered at the decoder. Conversely, shortening F = 64 filler bits improves the waterfall performance significantly and almost eliminates the error floor. V. CONCLUSION
The structure of the 5G NR LDPC code provides a “road map” to identifying the associated elementary absorbing sets. By indexing the absorbing sets according to how many columns in the core of the base graph participate, we are provided with a means of reducing the size of the space that must be searched and an efficient means of finding the most problematic sets. The efficacy of the search was demonstrated by simulating a broad array of codes over a BPSK-modulated AWGN channel and observing the dominant error mechanisms.
We also demonstrated the factors that affect the composition of problematic EAS’s - for example, the dynamic range of the quantizer and the lifting value Z; however, the precision of the quantizer (for a given dynamic range) does not affect the composition of the EAS’s appreciably. The combination of puncturing information bits and shortening filler bits yields a smoother performance curve in which we do not see an error floor at least down to the FER of 10−6. The existence of the error floor after shortening and puncturing depends on the prevalent EAS’s related to the lifting value.

EAS participates in BG2 core columns 4, 6, 7, 9, 10. We also know that the 64 filler bits are padded into the information block to cover the gap between the 256th and 320th place, i.e. bits corresponding to columns 9 and 10 of BG2. Our conjecture is that because two variable nodes in each prevalent EAS are receiving maximally reliable LLR values from the filler bits, the composition of problematic EAS’s changes after shortening. After both puncturing 2Z systematic information bits and shortening F filler bits, the decoder yields a smoother improved waterfall performance with the error floor entirely vanishing, at least down to an FER of 10−8. The extent to which puncturing and zero padding can impact performance is relative to the number of bits being punctured and shortened, i.e. 2Z and F , and whether the prevalent EAS’s include any of those columns impacted by shortening.
