Encoding Algorithm of Binary and Non-binary Irregular LDPC Codes via Block Triangular
Matrices with Low Weight Diagonal Submatrices

Yuta Iketo and Takayuki Nozaki Dept. of Informatics, Yamaguchi University, JAPAN
Email: {b001vb,tnozaki}@yamaguchi-u.ac.jp

Abstract—In this paper, we propose a low complexity encoding algorithm for the binary and non-binary irregular LDPC codes. This algorithm transforms the parity part of the parity-check matrix into a block triangular matrix by row and column permutations. By lowering the weight of diagonal submatrices, we reduce the encoding complexity.

I. Introduction

Low-density parity-check (LDPC) codes [1] are defined

by sparse parity-check matrices H ∈ Fqm×n. The LDPC codes are decoded by the sum-product decoding algorithm

in linear time and come close to the Shannon capacity on

many channels [2].

for linear codes maps a message u ∈ Fqn−m into a codeword x ∈ Fqn by xT = GuT . Even if H is a sparse matrix, G
is not always a sparse matrix. Hence, the complexity of

encoding algorithm by a generator matrix is quadratic in

the code length. In other words, an encoding algorithm by

a generator matrix has a larger complexity than the sum-

product decoding algorithm for large code length. The

encoding can be a bottleneck in a communication system

with an LDPC code. Therefore, it is important to develop

a low complexity encoding algorithm for LDPC codes.

In this paper, we assume that the parity-check matrix

H has full rank, i.e., rank(H)=m. By performing suitable

column permutations into H, we get HP HI , where
HP is non-singular. Then the codeword x ∈ Fqn is split into two parts, namely the parity part p ∈ Fqm and the message part u ∈ Fnq −m as x = (p u). Since HxT = 0T , we get HP pT = −HI uT .

In general, the encoding for LDPC codes is accom-

plished in two stages, namely the preprocessing stage

and the encoding stage. In the preprocessing stage, we

split H into HI which corresponds to u and HP which

corresponds to p. In the encoding stage, we determine

p by solving HP pT = −HI uT . Since the computation of bT := −HI uT is the product of a sparse matrix and

a known vector, the computation complexity is linear in

the code length. Hence, in encoding for LDPC codes, it is

important to consider two following algorithms.

1) An algorithm to construct suitable HP .

2) An algorithm to solve efficiently HP pT = bT . We transform H only by row and column permutations so that we keep the performance of the sum-product decoding algorithm. Hence, by using two permutation matrices P ∈ Fqm×m and Q ∈ Fqn×n, we transform H into PHQ = HP HI . Richardson and Urbanke [3] proposed an efficient encoding algorithm by transforming HP into an approximate triangular matrix (ATM). The complexity of this encoding algorithm is O(n + δ2), where n is a code length and δ is a gap which satisfies δ n and is proportional to n. Hence, the complexity of this encoding algorithm is O(n2), but is lower than that of the encoding algorithm by a generator matrix. Shibuya and Kobayashi [4] proposed an encoding algorithm by block-triangulation. This encoding algorithm is more efficient than the Richardson and Urbanke’s (RU) encoding algorithm. However, Nozaki [5] showed some cases in which H cannot be transformed by block-triangulation. Nozaki [6] proposed a parallel encoding algorithm by transforming HP into a block-diagonal matrix. This encoding algorithm has lower encoding time than the RU encoding algorithm in the case of parallel computation. However, this encoding algorithm has slightly larger total computation complexity than the RU encoding algorithm. Huang and Zhu [7] proposed a linear time encoding algorithm for non-binary cycle codes, these are non-binary LDPC codes defined by parity-check matrices of which all the column weights are two. This algorithm transforms HP into an upper block-bidiagonal matrix whose diagonal submatrices are cycle or diagonal matrices. Irregular LDPC codes are generalization of regular LDPC codes and have lower block-error rate than regular LDPC codes [8], [9]. The goal of this work is to reduce the encoding complexity for binary and non-binary irregular LDPC codes. The main idea of this work is to transform HP into a block triangular matrix with low weight diagonal submatrices. More precisely, our proposed algorithm transforms each of the diagonal submatrices into a cycle or a diagonal matrix as much as possible. For non-binary cycle codes, the resulting matrix by the proposed algorithm coincides one by the Huang and Zhu’s (HZ) algorithm. In other words, the proposed algorithm

II. Preliminaries
This section defines LDPC codes and describes previous encoding algorithms.

A. LDPC codes
For sparse parity-check matrices H ∈ Fqm×n, non-binary LDPC codes are defined as Cq = {x ∈ Fnq | HxT = 0T }. In particular, when q = 2, C2 is called binary LDPC codes.
Each parity-check matrix H is represented as Tanner graph. The rows (resp. columns) of H correspond to the check (resp. variable) nodes. An ensemble of Tanner graphs is defined in terms of two degree distributions [10], namely the check degree distribution ρ(x) and the variable degree distribution λ(x). The check and the variable degree distributions can be expressed as ρ(x) = i ρixi−1 and λ(x) = i λixi−1, respectively, where ρi (resp. λi) is equal to the fraction of edges which connect to check (resp. variable) nodes of degree i. An LDPC code for which the degrees of nodes are chosen according to degree distributions is called an irregular LDPC code. Furthermore, an LDPC code which satisfies λ(x) = x is called a cycle code.

D. Property of proposed algorithm
When the associated graph is a connected graph, we say the cycle code is proper. The following theorem shows that the proposed algorithm is a generalization of the HZ algorithm. We omit the proof of this theorem for the space limitation.

Theorem 1. If the input matrix H is a parity-check matrix for a proper cycle code, the proposed preprocessing algorithm outputs H satisfying (i) L1 = C1, (ii) Li = Di (i = 2, 3, ..., ), (iii) Ki−1,i = O (i = 2, 3, ..., ), and (iv) Ki,j = O (j = i + 2, i + 3, ..., ).

V. Conclusion
In this paper, we have proposed an encoding algorithm for irregular LDPC codes. As a result, the proposed algorithm has lower complexity than the RU algorithm.
