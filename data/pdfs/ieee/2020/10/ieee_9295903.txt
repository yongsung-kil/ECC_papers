An Improved Bit Flipping Min Sum Algorithm with Difference to Sum Ratio Factor 
Based on Unreliable Received Messages

1State Key Laboratory of Networking and Switching Technology, Beijing University of Posts and Telecommunications,

Abstract—Low density parity-check (LDPC) codes is known 
for its outstanding performance near the Shannon limit. In this 
paper, an improved min sum algorithm with difference to sum 
ratio factor (DSR-MS) is proposed, which is called bit flipping 
min sum algorithm with DSR factor based on unreliable 
received messages (DSR-URMBFMS). It belongs to the hybrid 
decision decoding scheme. A novel assistant stage associated 
with the bit flipping (BF) algorithm is introduced after each 
failed iteration of DSR-MS, and the sequence with minimum 
sum of syndrome vector (SSV) is taken as the final decoding 
output. Simulation results show that the proposed algorithm 
achieves faster convergence rate and better performance than 
the DSR-MS algorithm. 
Keywords-LDPC; DSR-MS; BF; hybrid decision 
I. 
 INTRODUCTION 
Low density parity-check (LDPC) codes were first 
proposed by Gallager in the 1960s but ignored because of 
excessive computational complexity [1]. Until 1996, 
MacKay and Neal rediscovered LDPC codes and proved that 
they have excellent performance close to the Shannon limit 
[2]. The belief propagation (BP) algorithm and the log-
likelihood ratio belief propagation (LLR BP) algorithm are 
the representatives of soft decision decoding algorithms with 
great performance. Nevertheless, they are hard to implement 
for complex computations. Fossorier et al. proposed the min 
sum (MS) algorithm as a simplification of the LLR BP 
algorithm [3]. It makes the realization of soft decision 
feasible by using the minimum value to replace the tanh 
function at the cost of non-negligible performance loss. In 
order to reduce the gap between the BP algorithm and the 
MS algorithm, many improvements were proposed. The min 
sum algorithm with difference to sum ratio factor (DSR-MS) 
in [4] achieves a suitable tradeoff between complexity and 
error performance, which performs much better than the MS 
algorithm by adding little complexity. 
In addition to the soft decision scheme, there are two 
other decoding categories: hard decision and hybrid decision. 
The bit flipping (BF) algorithm has the lowest computational 
complexity, whose performance, however, is much worse 
than that of the BP algorithm. Even though the group of 
weighted BF (WBF) algorithms [5]-[8] and the gradient 
descent BF (GDBF) algorithm [9] make great progress in 
pursuing the soft decision decoding algorithms, the gap still 
exists. For the sake of combining the advantages of the 
above two schemes, the hybrid decision was proposed [10]-
[12], which generally consists of two separate iterative 
decoding stages serially. If the algorithm of the first stage 
fails to find the correct decoded vector in fixed iterations, the 
second stage will be executed until the iteration counter 
increases to its maximum value or the decoding is successful. 
Thus, in this category of decoding, the issue of stage-
switching (SS) timing or the problem of implementation 
complexity needs to be overcome. The two-stage BF (2SBF) 
algorithm and the three-stage BF (3SBF) algorithm [13] 
were proposed for the above problems, but they can’t get rid 
of the performance constraints of the BF algorithm. At the 
end of [4], the BF algorithm is regarded as an assistant stage 
after each failed iteration of DSR-MS to further improve the 
performance, which is called bit flipping min sum algorithm 
with DSR factor (DSR-BFMS). It provided us a good 
approach to avoid the problems of SS timing and high 
complexity, but its performance improvement is limited. 
For the problems mentioned above, we aim to find a 
faster way of decoding, which can be used for short burst 
communication, frequency-hopping communication, etc. In 
this paper, a new assistant stage embedded into the DSR-MS 
algorithm is proposed. It is a variant of the BF algorithm that 
flips suspicious bits one by one based on unreliable received 
messages. What’s more, a hard decision sequence with 
minimum sum of syndrome vector (SSV) in the second stage 
will be stored as the final result. This hybrid decision 
decoding scheme is named as bit flipping min sum algorithm 
with DSR factor based on unreliable received messages 
(DSR-URMBFMS). During the decoding process, the 
proposed algorithm improves the performance and reduces 
the number of major iterations in the first stage to accelerate 
the convergence rate. 
The rest of this paper is organized as follows. Section II 
reviews the description of LDPC codes and two classical 
decoding algorithms: the MS algorithm and the BF algorithm. 
In section III, the proposed algorithm, especially its assistant 
stage, is described in detail. The simulation results and the 
analysis of computational complexity are presented in 
section IV and section V, respectively. Section VI shows the 
conclusion.

B. Bit Flipping Algorithm 
In [1], Gallager also proposed the most basic BF 
algorithm for LDPC decoding. Let 𝑧 = [𝑧0, 𝑧1, ..., 𝑧𝑁−1] be the 
hard decision sequence initialized by 𝑦, as follows: 𝑧𝑖 = 0 if 
𝑦𝑖 > 0 and 𝑧𝑖 = 1 otherwise. We denote the number vector of 
unsatisfied parity-check equations associated with variable 
node 𝑖 by 𝑓 = [𝑓0, 𝑓1, ..., 𝑓𝑁−1], where 𝑓𝑖 = ∑ 𝑠𝑗, 𝑗 ∈ 𝑀(𝑖). If 
𝑓𝑖 is equal to the maximum value 𝑓max, the corresponding 𝑧𝑖 
will be added to a set Ω. Then, all the members of set Ω are 
flipped at the end of current iteration. The condition for 
ending decoding is that 𝑠 = 0 or the maximum number 𝑙max is 
reached.

Our algorithm is similar to DSR-BFMS by adding the 
assistant stage for better performance and faster convergence 
rate, which is proposed based on the DSR-MS algorithm and 
the BF algorithm. We name it bit flipping min sum algorithm 
with DSR factor based on unreliable received messages 
(DSR-URMBFMS). At the end of each DSR-MS decoding 
iteration, the novel bit flipping algorithm with certain 
iterations is introduced. The proposed algorithm involves 
two successive stages: The first stage is the same as the 
process of DSR-MS, and the assistant stage will be 
performed if the syndrome vector 𝑠 ≠ 0 in each major 
iteration. Of course, provided that 𝑠 = 0 occurs in the first 
stage, it means the correct codeword is found and the 
decoding terminates. 
With the help of information exchange of the first stage, 
lots of wrong bits can be corrected. But the smaller the 
absolute value of 𝑦𝑖 is, the more limited its effect is. We 
choose to flip the most unreliable 𝑧𝑖 of Ω based on the 
following rule [16]: for the additive white Gaussian noise 
(AWGN) channel, the hard decision bit 𝑧𝑖 is closer to the 
right result because of the greater |𝑦𝑖|. The syndrome vector 𝑠 
will be recalculated every time we flip the selected bit 𝑧𝑖. 
Throughout the decoding process, a hard decision sequence 𝑧 
with a minimum SSV 𝑠𝑠𝑢𝑚 is considered as a final result. The 
steps of the assistant stage are described as follows:  
1)   Calculate the number vector 𝑓 and find the maximum 
number 𝑓max of it. If 𝑓max is less than the threshold 𝛿, go to 
step 5). Otherwise, go to step 2). 
2)  Define the set Ω, including all the variable node 𝑖 
whose 𝑓𝑖 is equal to 𝑓max. Then, find the corresponding set of 
𝑦𝑖 and select the most unreliable 𝑦𝑖 with the smallest absolute 
value according to the rule mentioned above. Flip the 
corresponding decision 𝑧𝑖. 
3)   Compute the syndrome vector 𝑠 = 𝑧𝐻T and the SSV. 
If the 𝑠𝑠𝑢𝑚 isn't greater than the 𝑠𝑚𝑖𝑛, the latter will be 
replaced by the former. Meanwhile, update the candidate 
decoded vector 𝑢 with the latest 𝑧. 
4)   If 𝑠𝑠𝑢𝑚 = 0 or the counter 𝑡 increases to its maximum

In the whole process of decoding, the assistant stage is 
dedicated to correcting the remaining error bits in time. 
Once the target codeword is found by flipping bits, the 
decoding will be terminated directly and all the complex 
operations of later iterations will be omitted. Moreover, 
when the propagation of error messages leads to 
performance degradation in the first stage, the final output 
will not be affected because the optimal codeword is already 
stored in the assistant stage. Besides, even if some bits are 
flipped incorrectly in the second stage, it does not affect the 
next major iteration.

SIMULATION RESULTS 
In this section, we perform the simulations under the 
AWGN channel. Simulation results are presented under 
different signal-to-noise ratio (SNR) for comparison among 
DSR-MS, DSR-BFMS, and DSR-URMBFMS. Like [15], we 
also fix the parameter 𝜅 of those three algorithms to 0.0125 
for a lower complexity and an acceptable performance. With 
plenty of experiments, we select the maximum major 
iteration 𝑙max = 20 for related algorithms and secondary 
iteration 𝑡max = 10 for DSR-URMBFMS. In the assistant 
stage of DSR-URMBFMS. The optimal threshold 𝛿 = 2 and 
the original 𝑠𝑚𝑖𝑛 = 𝑀 are chosen.

Fig. 1 is performed by a regular (360, 180) (3, 6) LDPC 
code which corresponds to the code rate of 1/2. In contrast to 
DSR-MS, the proposed algorithm almost achieves a gain of 
0.13 dB at BER = 10-5. With the increase of SNR, the gap 
between curves is widened. The DSR-URMBFMS algorithm 
has more advantages in extremely short LDPC code since the 
assistant stage is single bit-flipping in each iteration. In 
addition, a smaller degree 𝑑𝑐 means less information is 
available to the check nodes in the first stage and more error 
bits, which helps the assistant stage in finding the true error 
bits. 
In Fig. 2, we consider the regular (1080, 720) (3, 9) 
LDPC code with the rate of 2/3 and display the BER 
performance. Although the gap is smaller in the long code, 
performance improvement proves that our method is feasible. 
In the case of long LDPC code, the first stage can correct 
more suspicious bits and has fewer bits to correct. The 
probability of flipping bits wrongly in assistant stage 
increases, which makes the algorithm unable to achieve the 
expected improvement of performance. 
V. 
COMPUTATIONAL COMPLEXITY 
In this section, the computational complexity of the three 
algorithms is analyzed. On the base of the DSR-MS 
algorithm, the DSR-BFMS algorithm adds the operations of 
subtraction and comparison in each major iteration. Besides 
these two operations, the DSR-URMBFMS algorithm also 
adds simple addition operations to find the minimum SSV in 
each assistant iteration. 
The computational complexity per frame of the (360, 180) 
(3, 6) regular LDPC code with 1/2 rate at SNR = 3.0 dB is 
summarized in Table I. 𝑙𝑎𝑣𝑒 represents the average number of 
major iterations per frame and 𝑡𝑎𝑣𝑒 represents the average 
number of the assistant iterations per 𝑙 iteration. The 
proposed algorithm reduces the number of major iterations 
by adding the operations of addition, which can be finished 
with single bit operations. Compared with DSR-MS and 
DSR-BFMS, it avoids lots of multiplication operations to 
shorten the decoding time and accelerate the convergence 
rate. Table II shows the computational complexity of the 
(1080, 720) (3, 9) LDPC code with 2/3 rate at SNR = 3.0 dB, 
where many multiplication operations are saved in our 
proposed DSR-URMBFMS algorithm, which is meaningful 
since the operation of multiplication is more complex than 
the other three operations.

VI. 
CONCLUSION 
In this paper, we present an improved decoding scheme 
combining soft decision and hard decision based on the 
DSR-MS algorithm and an enhanced BF algorithm. At the 
assistant stage, the novel BF algorithm flips unreliable

decision bits one by one and stores the decoded vector with 
minimum SSV, so that the target codeword can be found 
faster. Hence the DSR-URMBFMS algorithm reduces the 
overall decoding time and improves the working efficiency 
of the decoder at the cost of increasing simple operations and 
little storage space. As the simulation results show, our 
algorithm with the extremely short LDPC code gains more 
performance improvement than that with the long one, since 
the assistant stage is single bit-flipping in each iteration. As 
for future work, we intend to focus on finding more accurate 
judge rules of flipping bits for locating incorrect bits and 
improving the decoding performance.
