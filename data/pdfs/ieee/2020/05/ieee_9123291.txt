Abstract—In this paper, a LDPC decoder based on efficient variable node memory structure design is proposed to reduce the time cost of accessing data in the pipeline. Firstly, we prepare the table corresponding to the variable node memory structure. So we can access the data only by reading the table. On this basis, the check node update order is adjusted to avoid performance loss caused by the conflict between adjacent ones. This method is applicable to all code rates, and does not bring loss of decoding performance. It greatly increase the throughput of the decoder, with an affordable resource consumption. Based on the proposed architectures, a high speed LDPC decoder is implemented on Stratix V 5SGXEA7N2F45C2N FPGA, achieves a throughput of 2.05 Gbps for DVB-S2 code rate 2/3 and a throughput of 2.73 Gbps for rate 4/5.
Keywords-LDPC Decoder; DVB-S2; TDMP; Graph Coloring Problem; Traveling salesman problem
I. INTRODUCTION
As the increasing of the resolution of remote sensing satellites, including time resolution, spatial resolution, and spectral resolution, the rate of data transmitted by satellites to the ground is also dramatically increasing. In order to maximize the use of channel capacity and improve the spectrum utilization of the system, it is necessary to provide different modulation and encoding type according to different receiving environments.
In March 2005, the European Digital Video Broadcasting Organization released the second generation transmission standard DVB-S2 for broadband satellite applications.[1] It can provide Variable Coding and Modulation (VCM) and Adaptive Coding and Modulation (ACM). Therefore DVBS2 provides for many transmission modes (FEC coding and modulations), giving different trade-offs between power and spectrum efficiency.
FEC encoding shall be carried out by the concatenation of BCH outer codes and LDPC (Low Density Parity Check) inner codes (rates 1/4, 1/3, 2/5, 1/2, 3/5, 2/3, 3/4, 4/5, 5/6, 8/9, 9/10). Depending on the application area, the FEC coded block shall have length n= 64800 bits or 16200 bits. Mapping into QPSK, 8PSK, 16APSK and 32APSK constellations shall be applied, depending on the application area.[1]
Therefore, the high-speed LDPC decoder based on the DVB-S2 standard will be the key technology for the future upgrade of remote sensing satellite systems.

In the structure design of the LDPC decoder, the TPMP (Two-Phase Message Passing) scheduling algorithm is first proposed by [2][3]. The strategy is to update all variable nodes and all check nodes in turn. It can reach the max parallelism (up to 360). M. M. Mansour proposed the TDMP (Turbo Decoding Message Passing) algorithm in [4], which is called the layered decoding algorithm by D. Hocevar [5]. This algorithm update each variable node and each check node in turn, so that when we update each check node, we can use the latest variable node message. According to this strategy, TDMP has faster convergence speed than TPMP by a factor of two in terms of decoding iterations. But this algorithm also causes memory conflicts. Subsequent decoder designs are structurally improved under these two algorithms.
In [6] the authors adopt the TPMP algorithm, and use the decoder structure of shared memory and a posteriori probability accumulation storage, which reduces the memory consumption, but the throughput is low. In this paper, for example, with the parameter of the DVB-S2 short frame 3/5 code rate and 30 iterations, the throughput of the decoder is only 289Mbps.
The paper [7] also uses the TPMP algorithm, and adds a set of memory cache, so that check node updates and variable node updates can be performed simultaneously. And the throughput is doubled. Different rates for short frames reach 700-1000 Mbps.
The TDMP algorithm is used in [8]. It can support all rates of the DVB-S2. The algorithm reduces the number of iterations and uses 360 parallelism to improve the throughput of the decoder. Different rates for short frames reach 8001000 Mbps.
The memory conflict caused by the TDMP algorithm is first proposed in [9]. By means of grouping and reducing the number of parallelism, the memory conflict in the group is solved. And secondly, the memory conflict of the adjacent two updates is solved by adjusting the scheduling order of the pipeline.
Recently, the paper [13] proposes a new architecture. Through elementary transformation on the parity check matrices of DVB-S2 LDPC codes, a new matrix whose left is a QC sub-matrix and right is Transformation of Staircase lower triangular(TST) sub-matrix is obtained. This decoder can achieve the maximum decoding throughput of 2.5 Gbps.
However, the above papers have basically the same design of the memory cells, and are relatively simple. This design makes the whole decoding process have a large time

III. VARIABLE NODE MEMORY ALLOCATION PROBLEM
Based on the chronogram of a non-pipelined decoder mentioned in the last section, the current decoder all use pipeline to increase the throughput. However, in the process of reading and writing soft information, the above method is still used, which clocks are consumed to read data. The main reason is that the memory design is relatively simple, and there is no guarantee that the data read each time are distributed in different memory blocks. If the memory is properly designed, the data can be read at the cost of 1 clock. So an efficient memory design can increase the overall throughput by a factor of . Considering that the line weight ( ) of the 2/3 code rate in the DVB-S2 is 10, such an improvement is necessary.
A. The Graph Coloring Problem (GCP)

matrix corresponding to the information bits and the check bits are not identical, they can be designed separately. In this paper, we only talk about information bits part. The optimal memory design is such that when the given check node is updated, it is possible to consume one clock to read the variable node memory units from different memory blocks. For other rates, we only calculate the solution to these two problems.

VII. CONCLUSION
In order to solve the problem of large time consumption of the traditional LDPC decoder when reading data, this paper proposes an efficient memory design. This method preprocesses the table of the memory structure. It has no effect on decoding performance, but increases some hardware resources. On the basis of this, the check node update order is adjusted to avoid performance loss caused by data access conflict. After the targeted design of code rate 2/3 and 4/5, the throughput can be greatly increased, and the remote sensing satellite system capable of high-speed transmission in the future can be satisfied. Based on this method, we also find the solution of memory design and update order for other rates in DVB-S2, but the integration of all code rates in one decoder requires more complicated design of the decoder structure.
