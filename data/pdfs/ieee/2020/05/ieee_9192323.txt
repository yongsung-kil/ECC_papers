Real-time FPGA Verification for 25G-PON and 50G-PON LDPC Codes

Abstract: Employing 75-piece FPGA implementation, we investigate the performance down to 10-15 BER of hard-decision 25G-PON LDPC and soft-decision 50G-PON LDPC at a record 90Gbps throughput. FPGA emulations reveal soft-decision LDPC code offers more than 1.5dB coding gain with no error floor. ©2020TheAuthor(s)
1. Introduction
Forward Error Correction (FEC) is an essential technique to help high speed Passive Optical Network (PON) achieve required bit error ratio (BER) target and extend loss budget classes without increasing too much optical cost. In order to support 29dB and above loss budget class, an enhanced FEC, such as Low-Density Parity-Check (LDPC) code [1], has been adopted by IEEE 802.3ca 100G-EPON Task Force. Although FPGA verification of the adopted LDPC and its mother codes with hard-decision (HD) and soft-decision (SD) decoders has been presented respectively in [2] targeting 50G-PON, the actual throughput of around 0.1 Gbps is not sufficient to rule out the possible error floor at fully-loaded throughput before going for mass production. Therefore, in this paper, through a mega FPGA platform we analyze the real-time performance of LDPC codes with HD and SD decoders at a record throughput of 90 Gbps. In addition, the output BER was measured down to 10-15 within 30 hours and no error floor was observed. This would lay a solid foundation not only for current 25G/50G-PON, but also for beyond-50G PON development in the near future.
2. FPGA Implementation of the LDPC Codes
Figure 1 shows the schematic of the FPGA platform for emulating the LDPC schemes, which includes HD-LDPC and SD-LDPC codes with different code rate through zero padding and parity puncturing operation. The implementation employs 75 pieces of Xilinx ultrascale-XCVU125 FPGA clocked at 75 MHz.

Fig. 1. FPGA implementation of QC LDPC coding and emulation systems.
On the transmitter side, a 268−1 pseudo random bit sequence (PRBS) is generated as the payload and is fed into QC LDPC encoder. Parity puncturing and deleting padding zero module follows the QC LDPC encoders. Subsequently, the 32 parallel bit streams are mapped to binary phase shift keying (BPSK) symbols. To emulate an AWGN channel, a Box-Muller Gaussian variate generator (BM GVG) is implemented adding 32-lane uncorrelated GVs as the noise source to the modulated symbols. On the receiver side, the received symbols are converted into 6-bit log-likelihood-ratios (LLR) as soft decisions. After appending shortening and puncturing LLR operation, the subsequent QC LDPC decoder comprises 32 parallel sets of variable node units (VNU) and check node units (CNU) implementing the revised min-sumalgorithm for layered decoding. It only takes less than 40 minutes to achieve an output BER of 10-12 with 200 output errors counted, comparing with 10 days to achieve an output BER of 10-12 [2]. For an output BER of 10-15 with around 10 errors, it only takes 31 hours.
3. LDPC Performance Investigation with the FPGA Platform
The resource utilization of one FPGA implementation and the 90-Gbps throughput FPGA platform shown in Fig. 2. We choose the LDPC code adopted by the IEEE 802.3ca [4] and its mother code, where the code rates are 0.849 and 0.826 respectively. With these two codes, the HD-LDPC and SD-LDPC performances with the FPGA emulation platform are plotted in Fig. 3. For an output BER of 10-12, the rate 0.849 HD-LDPC code, the rate 0.826 HD-LDPC, the rate 0.849 SD-LDPC and the rate 0.826 SD-LDPC require the input BER to be 1.08e-2, 1.16e-2, 1.97e-2, and 2.86e-2, respectively. Compared to the reference HD-LDPC with 0.849 code rate, the SD-LDPC with the same rate provides 0.95dB more coding gain at an output BER of 10-12. For the mother code, the SD-LDPC provides 1.54dB more coding gain at an output BER of 10-12, which is significantly better than the 0.849 code rate. Moreover, we find there is no error floor down to 10-15 output BER for the SD-LDPC code.

4. Conclusions Employing 75-piece FPGA implementation, we investigate the performance down to 10-15 BER of hard-decision 25G-PON LDPC code and soft-decision 50G-PON LDPC code at a record 90-Gbps throughput. The FPGA emulations reveal that the soft-decision LDPC code offers more than 1.5dB coding gain and does not encounter error floor down to 10-15 output BER, which is a good candidate for higher-speed PON. 5.
