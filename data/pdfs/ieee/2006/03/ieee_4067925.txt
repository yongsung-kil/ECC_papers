High-Throughput Turbo-Sum-Product Decoding of
QC LDPC Codes
Yongmei Dai, Zhiyuan Yan and Ning Chen
Department of Electrical and Computer Engineering Lehigh University, PA 18015, USA
E-mail: {yod304, yan, nic6}@lehigh.edu

Abstract— In this paper, we first propose a turbo-sum-product (TSP) decoding algorithm for quasi-cyclic (QC) low-density parity-check (LDPC) codes and show that our proposed algorithm not only achieves faster convergence and better error performances than the sum-product (SP) algorithm, but also needs less memory in implementation. Compared with the turbo decoding algorithm, our TSP algorithm saves the same amount of memory and achieves a better tradeoff between sub-iterations and convergence rate, leading to a higher throughput. We also propose a shuffled-sum-product (SSP) decoding algorithm, which is the dual of the TSP algorithm.
Index Terms— low-density parity-check (LDPC) codes, quasicyclic (QC) codes, turbo decoding, sum-product decoding.
I. INTRODUCTION
Low-density parity-check (LDPC) codes [1, 2] and their efficient implementations have been receiving a lot of attention due to their near-capacity performances when decoded iteratively. Various (fully parallel, fully serial, and partly parallel) decoder architectures (see, e.g., [3–8]) have been proposed. In general, random code structure causes problems in the implementation of such codes, whereas regular code structure tends to ease the hardware implementation. Quasi-cyclic (QC) LDPC codes [9–11] not only have relatively good error performances [10] but also are suitable for hardware implementation: they can be encoded efficiently with shift registers (see, e.g., [12]) and decoded with partly parallel decoder architectures, which require simpler address generation mechanisms, less memory, and localized memory access (see, e.g., [6,7]). Thus, QC LDPC codes are of great interest and we focus on the decoding of QC LDPC codes in this paper.
One of the iterative decoding algorithms for LDPC codes is the sum-product (SP) decoding algorithm (see, e.g., [13]). An LDPC code can be represented by its Tanner graph [14], wherein the codeword bits and the parity check equations are represented as variable and check nodes respectively. Each iteration of the SP algorithm consists of two sequential steps: variable node updates (VNUs) followed by check node updates (CNUs). There is inherent parallelism among the CNUs and VNUs respectively in each iteration of the SP algorithm, but fully parallel decoder architectures require a large number of processing elements and are susceptible to congested routing network. Thus, fully parallel architectures have been considered only for LDPC codes with moderate block lengths [3, 4].
Other iterative decoding algorithms (see, e.g., [8, 15–23]) have also been proposed for LDPC codes, and some of

these algorithms have been implemented using partly parallel decoder architectures. Lu and Moura [19] proposed to split the bipartite graph into a few trees, with each tree defining a supercode. However, this approach seems difficult to implement efficiently. Prabhakar and Narayanan [18] proposed to split the bipartite graph into two disjoint sub-graphs of approximately the same size, with each sub-graph defining a supercode and to decode each supercode with the SP algorithm while exchanging reliability information between the two supercodes. However, their approach saves memory in implementation at the cost of slower convergence and inferior error performance than the SP algorithm. Shimizu et al. [8] proposed a high-efficiency message-passing algorithm that achieves faster convergence at the expenses of more VNUs and doubling the memory requirement. Mansour and Shanbhag [16] have proposed a turbo decoding algorithm, which essentially changes the order of the CNUs and VNUs in the SP algorithm (see Section II for details). Each iteration of the turbo decoding algorithm consists of J successive subiterations, each of which decodes a supercode of the given LDPC code. The turbo decoding algorithm requires fewer (up to 50%) iterations to converge and achieves better error performance [16] than the SP algorithm while leading to significant memory savings in implementation [16, 17]. A similar idea was proposed in the shuffled decoding algorithm [20–22] as well. One potential drawback of the turbo (or shuffled) decoding algorithm is that the serialization of the decoding of the J supercodes somewhat offsets its advantage in convergence rate. Suppose it takes T clock cycles to decode each supercode and the average number of iterations is Nt, then the average number of clock cycles to decode a frame is Nt × J × T . When applying the turbo decoding algorithm to a QC LDPC code, J is the number of block rows in the parity check matrix, which is at least 3 and greater than 3 in many cases (see [10]). Thus, despite a smaller Nt, the throughput of the turbo decoding algorithm is limited, especially when J is large.
In this paper, we propose a new hybrid decoding algorithm, referred to as turbo-sum-product (TSP) algorithm: Given any QC LDPC code, the parity check matrix is always split rowwise into two submatrices which define two supercodes; then the two supercodes are decoded sequentially using the SP algorithm while reliability information is exchanged between the two supercodes using the turbo decoding algorithm. Different from [18], our TSP algorithm leads to reduced memory requirement while achieving better error performances and faster

Since one iteration of the TSP algorithm only consists of sequential decoding of two supercodes, our TSP algorithm achieves better tradeoff between the convergence rate and sub-iterations, and hence higher throughput than the turbo decoding algorithm [16], especially when QC LDPC codes have more than 3 block rows. II. DECODING ALGORITHMS OF LDPC CODES
Given an LDPC code represented by a parity check matrix H = {Hl,i}L×N , where N and L denote the block length and the number of parity check equations respectively, we briefly review various decoding algorithms for the given LDPC code.

B. Turbo Decoding Algorithm
The turbo decoding algorithm [16] splits H into J submatrices row-wise, i.e., H = [H1T HT2 · · · HJT ]T such that for Hi (i = 1, 2, · · · , J) there is at most one 1 in each column. Each submatrix Hi defines a supercode Ci, and C = C1 ∩C2 · · ·∩CJ . Each iteration of the turbo decoding consists of J sequential sub-iterations to decode the J supercodes using the extrinsic information from the other supercodes as a priori information. To decode the supercodes, VNUs concatenated with either the BCJR algorithm or CNUs can be used (see [16] for details). We use the latter approach in this paper. The turbo decoding algorithm can be viewed as a variant of the SP algorithm that arranges the message passing in a different order. Let us denote the sets of variable and check nodes in the Tanner graph of the given code as V and C respectively. In the turbo decoding, the breakup of H partitions C into J disjoint subsets: C = C˜1 ∪ C˜2 ∪ · · · ∪ C˜J , where C˜i denotes the set of check nodes corresponding to Hi. Each iteration of the SP algorithm first computes the messages from V to C˜i (i = 1, 2, · · · , J) in parallel and then calculates the messages from C˜i (i = 1, 2, · · · , J) to V simultaneously as shown in Fig. 1(a). In the turbo decoding, the order of the messages is changed as shown in Fig. 1(b): first computes the messages from V to C˜1 and from C˜1 to V , then computes the messages from V to C˜2 and from C˜2 to V , and so on. Since within one iteration, one supercode can use the updated information from preceding supercodes, the turbo algorithm requires fewer (up to 50%) iterations and achieves better error performance [16] than the SP algorithm. Since the decoding of one supercode only needs information from the other supercodes and the new information can overwrite the old information from the succeeding supercode, this eliminates the need to save the LLR information of one supercode and leads to memory savings.

B. Turbo Decoding of QC LDPC Codes
The turbo decoding algorithm in [16] can be directly used to decode QC LDPC codes. Since for each H˜ s there is exactly one 1 in each column, each H˜ s defines a supercode C˜s+1 (0 ≤ s ≤ j − 1). Totally, there are J = j supercodes, and at any time, only the LLR information of j − 1 supercodes needs to be stored. Thus, the turbo decoding algorithm can save the memory corresponding to one block row in comparison to the SP decoding algorithm. However, due to serialization of the decoding of supercodes, the simplification method to compute (1) cannot be applied and the number of additions per variable node required for the VNUs increases from O(2j) to O(j2). Furthermore, as shown in Figs. 1(a) and 1(b), the serialization of the decoding of the j supercodes limits the throughput compared to the SP algorithm, where the decoding of the j supercodes is completely parallel. This motivates the following two-supercode based TSP algorithm.

Accordingly, the set of variable nodes V in the Tanner graph of the given LDPC code is partitioned into G subsets: V = V˜1 ∪ V˜2 ∪ · · · ∪ V˜G, where V˜i denotes the set of variable nodes corresponding to H¯ i. Each iteration of the shuffled decoding consists of G sub-iterations as shown in Fig. 1(d): first computes the messages from C to V˜1 and from V˜1 to C, then computes the messages from C to V˜2 and from V˜2 to C, and so on. The shuffled decoding algorithm also achieves better error performances and faster convergence than the SP
algorithm.

C. TSP Algorithm of QC LDPC Codes

H1 and H2 define two supercodes C1 and C2 respectively, and the original LDPC code C is then C1 ∩ C2. The breakup of H partitions the set of check nodes
in the Tanner graph of the given LDPC code into two disjoint
sets C1 and C2, which correspond to C1 and C2 respectively.
Since there is only one 1 in each column in H1, the decoding of C1 only requires the information from C2. Since there are j − 1 1’s in H2, the decoding of C2 requires extrinsic information not only from C1, but also from the other j − 2 1’s in C2. In order to save memory while exploiting the turbo

the TSP algorithm essentially changes the order of messages

IV. SSP ALGORITHM OF QC LDPC CODES
A. Shuffled Decoding of QC LDPC Codes
The group shuffled decoding algorithm [20] can be directly applied to decode a QC LDPC code. The parity check matrix of the QC LDPC code can be arbitrarily partitioned into G groups column-wise. However, similar to the turbo decoding algorithm [16], the group shuffled decoding algorithm can potentially lead to memory savings only when each group has

only one 1 in each row. Thus, the parity check matrix of a QC LDPC code is divided into G = k groups, corresponding to the k block columns. However, the implementation of the group shuffled decoding algorithm leads to a tradeoff between memory and computational complexity. Since the CNUs (2) in each sub-iteration are only for the variable nodes in one block column, we need to either re-evaluate the Ψ(·) functions in (2) in each sub-iteration or save the evaluations of them in memory. The former option saves the memory corresponding to one block column, but the numbers of additions and Ψ(·) function evaluation per check node required for the CNUs both increase from O(2k) to O(k2). The latter option maintains the same number of Ψ(·) evaluations as the standard SP algorithm at the expense of saving k−2 extra LLRs (the check to variable messages after Ψ(·) evaluations) in addition to k −1 LLRs per check node. That is, it actually requires more memory than the SP algorithm! Finally, similar to the turbo decoding algorithm, the throughput of the group shuffled decoding is limited by the serialization of the sub-iterations.
B. SSP Algorithm of QC LDPC Codes
Given a QC LDPC code, we can view its H as the concatenation of two submatrices: H = [H¯ 1 H¯ 2], where H¯ 1 is simply the first block column in H. Note that this concatenation ensures that for H¯ 1 there is exactly one 1 in each row. This concatenation of H partitions the set of variable nodes in the Tanner graph of the given LDPC code into two sets V1 and V2, which correspond to H¯ 1 and H¯ 2 respectively. As illustrated in Fig. 1(e), the SSP algorithm essentially changes the order of messages passed between C and V1 and V2: first the messages are passed from C to V1, from V1 to C, and from C to V2, then the messages are passed from V2 to C. However, if we want to have the same computational complexity, whether we can save memory or not depends on the implementation. Also, the time required for each iteration of decoding increases due to the serialization of the CNUs for the two sub-iterations, each of which needs the same amount of time as one CNUs in the standard SP algorithm. Given these problems, the shuffled and SSP decoding algorithms seem to be not as appealing as the turbo and TSP decoding algorithms.

The convergence rate advantage of the TSP algorithm over the SP algorithm is less significant compared with the turbo decoding algorithm. Also, as can be seen from Figs. 5, 6 and 7, the convergence rate improvement of the SSP algorithm is less significant compared to the TSP algorithm. fewer iterations respectively than the SP algorithm. For the (421, 4, 7) SFT code, at 2.4 dB, the TSP and SSP algorithms require 17.6% and 11.7% fewer iterations respectively than the SP algorithm. This can be explained by the weaker turbo effect between the two supercodes since as j and k increase the first supercode compares even weaker to the second supercode. Also, as j and k increase, the convergence rate advantage of the TSP algorithm over the SSP algorithm decreases. However, as mentioned before, in terms of memory saving and computational complexity, the TSP algorithm is more appealing.
