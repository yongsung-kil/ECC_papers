Approximately Lower Triangular Ensembles of LPDC Codes with Linear Encoding Complexity

Shay Freundlich, David Burshtein and Simon Litsyn1 School of Electrical Engineering,
Tel-Aviv University, Tel-Aviv 69978, Israel

Abstract— The complexity of brute force encoding of LDPC

codes is proportional to the square value of the block length.

Richardson and Urbanke have proposed efficient encoding algo-

rithms for LDPC codes. These algorithms permute the parity

check matrix of the code iteratively, such that it becomes

approximately lower triangular. We propose a new approach for

efficient encoding of LDPC codes in which we modify the code

ensemble to force an approximate lower triangular structure,

L

thus eliminating the need to apply the algorithms of Richardson

and Urbanke. We prove that the new ensemble has the same

asymptotic threshold as the corresponding standard ensemble.

The new ensemble can be used for linear time encoding of an

arbitrary code profile. Computer simulations confirm that the

performances of the standard and new ensembles are also very

similar when using finite length codes.

I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes were originally proposed by Gallager [4], and have lately attracted a great deal of interest. One disadvantage of LDPC codes, compared to other iteratively decoded codes such as Turbo-Codes is their encoding complexity. As any Linear code, they can be encoded using a generating-matrix. The complexity of such an encoding scheme is quadratic in the block length compared to linear in Turbo-Codes. This difference becomes a significant problem when dealing with long block lengths.
One way to approach this problem is the cascade code that was proposed in [7], [8] for transmission over the binary erasure channel (BEC). However the cascade structure results in performance loss compared to a standard LDPC code with the same block length. Another concatenation scheme for linear time encoding that uses expander codes was introduced by Spielman [13]. This scheme also results in performance loss compared to a standard LDPC code. In [5] it was suggested to perform the encoding in an iterative manner, using iterative matrix inversions. This algorithm converges in a finite number of iterations provided that the parity check matrix satisfies some condition. For some specific regular LDPC codes it was demonstrated using computer simulations that this condition can be imposed with no performance loss compared to the standard corresponding LDPC codes. However it was not shown for general irregular LDPC codes.
Other methods allowing for a linear time encoding of LDPC codes are based on an approximate lower triangular (ALT) form of an L × N (N is the block length of the code and L is

In such a matrix, the submatrix corresponding to the first L − G rows and the last L − G columns is lower triangular. G is referred to as the matrix gap. In [12], Richardson and Urbanke have described an algorithm, that exploits the ALT form of the matrix and its sparseness to efficiently encode a message block. It is based on fixing the N − L leftmost (systematic) bits, and computing the remaining L parity bits in two stages: first compute the G bits that correspond to the gap, and then compute the remaining L − G bits corresponding to the lower-triangular part recursively. The complexity of the process is dominated by the first stage (computation of the bits correspondi√ng to the gap) and is O(N + G2). Thus, if the gap size is O( N ), then the encoding complexity is linear. To achieve a sparse ALT representation of the parity-check matrix, Richardson and Urbanke [12] have suggested algorithms to transform the given parity-check matrix to an ALT form, by using row and column permutations, that keep the sparseness of the matrix. The purpose of the process is to reduce the gap as much as possible.
In [9] it was suggested to force the parity-check matrix to have an ALT form. For that purpose one needs to define a code profile in the form of a set of small matrices with fixed row and column degrees, which serve as building blocks for the entire parity-check matrix. This profile places rigid constraints upon the codes, resulting in some loss of performance. Furthermore, standard tools for optimizing the code profile cannot be used in this construction. It was demonstrated that one can choose linearly encodable codes according to some profile, and that these codes can perform almost as good as other codes from the same profile, but it was shown for specific profiles, and no general tools were developed to construct optimal linearly encodable codes. In [6] it was proposed to impose a lower triangular structure on the parity check matrix and incorporate it into the progressive edge growth algorithm.
In this work we introduce an ALT-LDPC ensemble, which places less rigid constraints upon the code, but still forces it to have an ALT form, so that linear time encoding can be realized. Given an arbitrary row and column weight distribution, we show how it can be modified to impose the ALT constraint. We then prove that the asymptotic threshold of the modified ensemble is identical to the asymptotic threshold of the original ensemble. Note that our method saves the preprocessing stage in the algorithm proposed in [12]. In addition it can be used for linear time encoding of an arbitrary given code profile, unlike the efficient encoding algorithms in [12]. Furthermore, the ALT ensemble can be defined with any desirable gap size. Thus it enables a tradeoff between the gap size (i.e. the encoding complexity) and the performance, for any given block length.
II. THE ALT-LDPC ENSEMBLE
A. Description
We use the standard bipartite graph based ensemble. We denote by λl, l = lmin, . . . , lmax (ρl, l = rmin, . . . , rmax) the fraction of edges with left (right) degree l. Similarly we denote by λ˜l (ρ˜l), the fraction of left (right) nodes with degree l.
The ALT ensemble is defined as follows. Consider the bipartite graph depicted in Figure 2, which is associated with the matrix in Figure 1. A graph, G, in the ALT ensemble satisfies the following constraints.
1) For i = 1, . . . , L − G, variable node i is connected by exactly one edge to check node G + i. This edge is a fixed (hardwired) edge.
2) For i = 1, . . . , L − G, variable node i is not connected to any check node j ∈ [G+i+1, L]. This constraint can also be formulated from the check nodes perspective by asserting that for j = G + 2, . . . , L, check node j is not connected to any variable node i ∈ [1, j − G − 1].
All the bipartite graphs with a constant left degree c and a constant right degree d, that obey the above constraints, with a relative gap size g (i.e. G = gN ), form the ALT regular ensemble, CALT (c, d, g). To randomly construct a graph from this ensemble with uniform probability, we follow the following procedure. First we generate N variable nodes, v1, . . . , vN and L check nodes, u1, . . . , uL. B. Performance
One of the key asymptotic properties used in the analysis of conventional LDPC codes is the tree (independence) assumption. It asserts that when the block length is sufficiently large, each fixed edge in the bipartite graph satisfies the following w.p. 1−o(1). The subgraph containing all nodes and edges that are at distance at most 2l from e, which we refer to as the 2l-neighborhood of e, is tree-like, i.e. it contains no cycles. The belief propagation decoding algorithm is based on this property, as it generates at each iteration a message, that is the conditional probability for the variable’s value, assuming that all the extrinsic information (the channel observations corresponding to variable nodes in the neighborhood) are statistically independent random variables. Richardson and Urbanke have shown in [11, Appendix A] that when the block length is sufficiently large, the tree assumption is justified with probability that approaches one. The following theorem asserts the same property for the expurgated ALT ensemble.

C. Systematic Codes, LT-LDPC Ensemble
Consider the CALT (λ, ρ, g) ensemble with g = 0 (in this case X1 = 1 to enable a valid matching condition (1) for i = 1). The parity check matrices of this ensemble are lower triangular. We term this ensemble, the lower triangular (LT), CLT (λ, ρ) LDPC ensemble. We note that the encoding procedure for this ensemble is much simpler than the encoding algorithm described in [12], since there is no need to implement the preprocessing stage, to calculate a matrix inverse, and to apply the special order of computations as described in [12]. Here the code is systematic. The systematic part of the codeword is an N − L vector, and the parity check part is L bits long. To encode an N − L bits message word we calculate the L parity check bits recursively starting from the L’th until the first. The encoding complexity is linear in the block length. However, a disadvantage of finite block length graphs in this ensemble is that they are expected to have a larger number of cycles (associated with variable nodes with small indices).

IV. CONCLUSION
We introduced the ALT ensemble, which offers practical linear time encoding of LDPC codes. Any standard irregular code can be converted to the ALT ensemble, thus providing linear time encoding. We proved that the asymptotic threshold of the ALT ensemble is identical to the threshold of the corresponding standard ensemble.
