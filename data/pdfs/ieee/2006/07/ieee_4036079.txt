Abstract‚Äî In this paper, we study and propose a puncturing algorithm of block-type low-density parity-check (B-LDPC) codes. This optimal puncturing algorithm is derived from the fact that puncturing of parity bits is equivalent to merging the check nodes. Furthermore, we propose a new decoding algorithm suitable for the punctured B-LDPC codes. This decoding algorithm needs not only smaller number of operations at each iteration, but also shows faster decoding convergence speed than the conventional erasure decoding algorithm. If the optimally punctured B-LDPC code is decoded by the new decoding algorithm, it results in the same performance as the unpunctured B-LDPC code of the same code rate.
I. INTRODUCTION
Low-density parity-check (LDPC) codes were introduced by Gallager [1] and rediscovered by MacKay and Neal [2]. Richardson, Shokrollahi, and Urbanke showed that irregular LDPC codes could have better performance on the AWGN channel than Turbo codes, which was nearly close to the Shannon limit. Also, the decoding complexity of LDPC codes is less than that of Turbo codes since LDPC decoding is based on sum-product algorithm. However, the encoding complexity of LDPC codes is quadratic in the block length, which results in slow encoding. Therefore, efficient encoding algorithm has been studied [3] and various fast-encodable LDPC codes with duo-diagonal form were proposed [4], [5], [6].
Recently, a new class of fast-encodable LDPC codes, called block-type LDPC (B-LDPC) codes, was proposed and adopted in IEEE 802.16e [7]. Another advantage of B-LDPC codes is the memory efficiency. For general LDPC codes, a significant amount of memory is needed to store their parity-check matrices. However, for B-LDPC codes, the required memory can be reduced by a factor of z if circulant permutation matrices (circulants) and all-zero matrices are employed. Also, B-LDPC codes can support various lengths by using simple methods such as scaling and modulo [7].
In this paper, we study and propose a puncturing algorithm of B-LDPC codes. Although the optimal puncturing for LDPC codes has been extensively studied [8], [9], the puncturing algorithm proposed in this paper is different from the previous approach. For B-LDPC codes, we verify that two check nodes can be merged if they are connected to the same punctured parity nodes. Therefore, the model matrix of mother B-LDPC code should be designed by considering not only the cycle properties of mother code but also those of the punctured codes. This is to avoid the performance degradation caused by the short cycles occurred in the punctured B-LDPC codes.
The decoding convergence speed of punctured LDPC codes is slower than the unpunctured LDPC codes of the same code rate since the punctured nodes are slowly updated during the iterative decoding. Therefore, punctured LDPC codes show worse performance, especially for the small number of iterations (say, < 50). Also, punctured LDPC codes have larger decoding complexity than the unpunctured LDPC codes of the same code rate since they are decoded on the more complicated Tanner graphs including punctured nodes. We propose a new decoding algorithm to solve these problems by using the simplified check-node merged Tanner graphs of punctured B-LDPC codes.
As an example, punctured B-LDPC codes of rate 2/3 are constructed from the mother B-LDPC codes of rates 1/5, 1/3, and 1/2 by using the proposed puncturing algorithm. Simulation results show that the proposed decoding algorithm gives faster decoding convergence than the conventional erasure decoding algorithm.

IV. FAST CONVERGENCE DECODING FOR PUNCTURED B-LDPC CODES
Finding good puncturing patterns of LDPC codes has been extensively studied and the puncturing patterns with good thresholds have been proposed for various LDPC codes [8], [9], [10]. However, the convergence speed of punctured LDPC codes is slow since the decoding is performed by assuming 0 as the initial LLR value of punctured nodes. Therefore, the performance of punctured LPDC code is worse than that of unpunctured LDPC code with the same threshold when the iteration number is not sufficient (say, < 50). Also, punctured LDPC code needs more decoding operations at each iteration

than unpunctured LDPC code of the same code rate since

it is decoded on the more complicated Tanner graph. These

shortcomings can be solved if the decoding is performed on

a simple Tanner graph obtained by removing the punctured

parity nodes using check-node merging defined in Section III.

As shown in Fig. 4, it is possible for the punctured B-

LDPC codes to be decoded using sum-product algorithm on

the Tanner graph in Fig. 4 (b) instead of that in Fig. 4 (a).

Since only the parity bits are punctured, we do not need to

recover the punctured bits after the decoding. In Fig. 4, it can

be easily shown that the decoding complexity is reduced by

half since the number of edges is reduced from 24 to 12 in a

subset.

From the simulation, we verify that new decoding algorithm combined with the proposed puncturing patterns shows good performance. We use the following mother model matrix of rate 1/3 whose size 16 √ó 24. Among 8 information nodes, 6 information nodes have degree 3 and 2 information nodes have degree 16. Also, 13 check nodes have degree 5 and 3 check nodes have degree 6. The rate-compatible puncturing patterns (2, 2, 2, 2, 2, 2, 2, 2), (4, 4, 4, 4) and (8, 8) are used to achieve code rates 1/2, 2/3 and 4/5, respectively. BPSK modulation, 1056 information bits (z = 132), sum-product algorithm, and AWGN channel are assumed. Fig. 5 shows that proposed decoding (PD) outperforms the conventional erasure decoding (CED) since the punctured parity nodes decrease the convergence speed. Also, the performance gains for PD become larger as the number of punctured parity bits is larger, i.e., the code rates are higher. At code rates 1/2, 2/3, and 4/5, and FER of 10‚àí3, the PD has 0.55 dB, 0.9 dB, and 1.3 dB improvements over those of CED, respectively. To show the good rate-compatibility and good performance of punctured BLDPC code, it is compared with Turbo code adopted in 3GPP [11]. For the simulation, the puncturing patterns of Turbo code are designed to locate the unpunctured parity nodes as regularly as possible. As generally known, when the CED is used, punctured B-LDPC code shows worse performance than Turbo code as shown in Fig. 6. However, when the PD is used, punctured B-LDPC code outperforms Turbo code at code rates 1/3, 1/2, 2/3, and 3/4.
