Irregular Low-Density Parity-Check Convolutional Codes Based on Protographs

G. Richter, M. Kaupper
University of Ulm Department of Telecommunications and Applied Information Theory
Albert-Einstein-Allee 43 D-89081 Ulm, Germany {gerd.richter,markus.kaupper}@uni-ulm.de

University of Notre Dame Department of Electrical Engineering

Abstract— Irregular low-density parity-check (LDPC) codes constructed from small protographs are one of the most powerful LDPC block codes. In this paper, we introduce the convolutional version of these codes. LDPC convolutional codes constructed from protographs have some advantages in comparison to LDPC block codes constructed from protographs, e.g, an effective pipeline decoding. We show that LDPC convolutional codes constructed from protographs have much lower error rates than analogous LDPC block codes with the same complexity and that they operate with low error rates near the Shannon limits even for a relatively small memory size.
I. INTRODUCTION
Low-density parity-check codes introduced by Gallager [1] more than 40 years ago, have been subject of many relatively recent research activities. The interest in these codes stems from their performance near the Shannon limit, their simple description and implementation, and their low decoding complexity.
Gallager considered only regular LDPC codes, i.e., codes that are represented by a sparse parity-check matrix with a constant number of ones in each row and in each column. Later, it was shown [2], [3], [4] that the performance of LDPC codes in the waterfall region can be improved by using irregular LDPC codes.
To analyze LDPC codes, density evolution, an analytic tool to calculate the thresholds of these codes, is introduced in [3], [5]. This tool is generalized [6], [7], [8] to determine the thresholds of irregular LDPC codes with a special structure. Particularly, in [8], an LDPC code is designed that can be described by a small protograph. This LDPC code has a threshold very close to the Shannon limit and very low error rates even for a relatively small block length.
The convolutional counterpart of LDPC codes is introduced in [9]. LDPC convolutional codes can be defined by a semiinfinite sparse parity-check matrix. Thus, they can be decoded by a pipeline version of the belief propagation decoding algorithm. It is shown in [9] that LDPC convolutional codes can be encoded in a systematic way by using a simple shiftregister.
Regular LDPC convolutional codes considered in [9] can be constructed by unwrapping of the parity-check matrix of LDPC block codes. It is shown in [10] that regular LDPC convolutional codes have lower error rates than regular LDPC

block codes and that the asymptotic thresholds can be improved by using terminated LDPC convolutional codes. The performance of LDPC convolutional codes can be further improved by using irregular LDPC convolutional codes as described in [11].
In this paper, we introduce a new construction technique to design irregular LDPC convolutional codes. We do this by connecting small protographs such that a semi-infinite paritycheck matrix is obtained, which can be decoded by the pipeline decoding algorithm. The fast encoding property is preserved by this construction algorithm, i.e., the encoding can be done by a simple shift-register. Furthermore, LDPC convolutional codes based on protographs have better performances than existing LDPC convolutional codes.
The paper is organized as follows: In Section II, some fundamentals of LDPC codes based on protographs are given, while we recall some fundamentals of LDPC convolutional codes in Section III. Section IV deals with the construction algorithm for LDPC convolutional codes based on protographs. After showing some simulation results and comparing the performance of our codes with some known LDPC convolutional and block codes in Section V, we conclude our paper in Section VI.
II. LDPC CODES BASED ON PROTOGRAPHS
An LDPC code of length n and dimension k can be represented by a Tanner graph [12]. This graph consists of two sets of nodes connected by edges. One set, the variable nodes, corresponds to the n columns of the parity-check matrix H and the other set, the check nodes, represent the rows of the parity-check matrix. The number of rows in H is denoted by m, where m ≥ n − k. A one in the parity-check matrix in row j and in column i corresponds to an edge between the i-th variable node vi and the j-th check node cj. The number of edges incident to the i-th variable node vi is called the variable node degree d(vi), which is equal to the number of ones in column i. Similarly, the number of edges connected with the j-th check node cj is called the check node degree d(cj) and is equal to the number of ones in row j. This is demonstrated in the following example.

In [8], a method is described how to construct irregular LDPC codes with very good performance even for short blocklengths n. This method is based on protographs and it is recalled here in a slightly different manner such that it is more suitable for the construction algorithm of LDPC convolutional codes described in Section IV.
A protograph is a Tanner graph with a relatively small number of nodes and edges. The number of edges between a variable node and a check node can be larger than one, i.e., parallel edges are not forbidden. A larger graph can be obtained by a combination of several protographs and permuting the edges such that the variable nodes of one protograph are connected with check nodes of other protographs. Connections are only allowed between variable nodes and check nodes of the same types as in the small protograph. This is demonstrated in the following example.

To obtain a protograph with rate R = 1/2, the variable node I is punctured. The asymptotic threshold σ∗ for an LDPC block code with infinite length constructed from this protograph is σ∗ = 0.9683, which is equal to Eb/N0 = 0.2798 dB. For the calculation of the threshold we use a generalized discretized density evolution with 11 bits to quantize the probability density functions instead of using a Gaussian approximation as in [8].
For the construction of the LDPC convolutional codes a modified PEG algorithm is applied. We first connect all unpermutable edges and then we run the PEG algorithm such that the variable nodes are connected only with check nodes of the correct types. As described in the original PEG algorithm, we first connect the variable nodes with low degrees to obtain no small cycles between these variable nodes.
In Fig. 9, we compare the bit error rates (BERs) of the LDPC convolutional code based on the protograph from Fig. 8 with a regular (3,6) LDPC convolutional code and with the irregular LDPC convolutional code designed in [11]. The average number of ones per column of the regular LDPC convolutional code is K = 6, while it is K = 8.355 for the code designed in [11]. The LDPC convolutional code based on the protograph has K = 5.8. The regular LDPC code and the code from [11] have memory ms = 1199. The memory of the LDPC convolutional code based on the protograph is ms = 239 and ms = 367, which results in approximately the same number of memory units and the same processor complexity as the LDPC convolutional code from [11]. All codes were decoded with a maximum number of 100 iterations and decoder stopping rules [15] were employed.

VI. CONCLUSIONS
In this paper, we proposed a new construction method for irregular LDPC convolutional codes, which is based on the concatenation of small protographs. We demonstrated that LDPC convolutional codes constructed with this new algorithm outperform irregular and regular LDPC convolutional codes constructed with known algorithms. Furthermore, these LDPC convolutional codes show a better performance than LDPC block codes with the same processor complexity, while they are only slightly worse compared to LDPC block codes with a much greater processor complexity.
