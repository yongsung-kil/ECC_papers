AbstractвАФ We consider the design of efficient rate-compatible (RC) irregular repeat accumulate (IRA) codes, a subclass of LDPC codes, over a wide code rate range. The goal is to provide a family of RC codes to achieve high throughput in hybrid automatic repeat request (ARQ) scheme for high-speed data packet wireless systems. We focus on a hybrid design method which employs both puncturing and extending. We propose a simple puncturing method based on minimizing the maximal recoverable step of the punctured nodes and a new extending scheme by introducing the degree-1 parity bits for the lower rate codes and obtaining the optimal proportions of extended nodes through density evolution analysis. Simulation results show that our designed RC codes offer good error correction performance over a wide rate range and provide high throughput, especially in the high and low SNR regions.
I. INTRODUCTION
Rate-compatible (RC) codes [1] are defined as a family of nested error correction codes where the codeword bits from the higher-rate codes are embedded in the lower-rate codes. Then the incremental redundancy can be achieved by sending the higher-rate coded bits first and sending additional parity bits in the lower-rate codes later if requested from the receiver. The receiver combines all received coded bits to recover the information. Another advantage of RC codes is that with such a nested structure, RC codes can be encoded and decoded using one single pair of encoder and decoder so that the complexity of transmitter and receiver can be significantly reduced.
As an important advanced coding technique, low-density parity-check (LDPC) codes [2] offer better performance than turbo codes, and also enjoy lower decoding complexity. Many works on RC-LDPC codes have been devised, recently. Basically, there are two alternative approaches to obtain RC-LDPC codes. One is puncturing, i.e., obtaining higher rate RC codes from a low rate mother code by puncturing some coded bits [3], [4]. The other approach is bit extending, i.e, obtaining the lower rate RC codes by appending more parity bits with a high rate mother code [5], [6]. However, both puncturing and extending have some limitations on the effective code rate region of RC code design. Either very low rate mother code (for puncturing) or very high rate mother code (for extending) is required to obtain RC codes over a large rate range if only one design approach is employed. Usually, it is difficult to design a good finite-length code (as the mother code) with such extreme rate. Therefore, a hybrid design scheme has been proposed in [5] which employs both puncturing and extending methods to obtain a wide dynamic rate range of RC-LDPC codes from a moderate rate mother code. In this paper, we employ the hybrid approach to design RC-LDPC codes.

As a special class of LDPC codes, irregular repeat accumulate (IRA) codes have drawn much recent attention because they enjoy extremely simple encoding and low complexity decoding [7]. Moreover, with the RA structure, we can design and construct good finite-length LDPC codes with unequal protection for different type of nodes to lower the error-floor [8], [9]. Therefore, we consider the design of RC codes for such IRA-type of LDPC codes. The IRA codes are naturally rate-compatible. However, random puncturing or extending will not provide good RC codes. Using the concept of recoverable steps introduced in [4], we propose a simple but efficient algorithm to obtain high rate RC-LDPC codes through puncturing. We also propose a new extending method by introducing degree-1 parity bits to obtain low-rate RC-LDPC codes. The performance of RC-LDPC codes through extending can be improved by optimizing the code profile based on the density evolution analysis. The resulting RC-LDPC codes from our proposed design methods offer better error control performance over a wide range of rates. Employed with the hybrid ARQ scheme, these codes provide high throughput efficiency over a wide signal-to-noise ratio (SNR) region in both AWGN and block fading channels.

III. RC-IRA CODE DESIGN вАУ PUNCTURING
In [3], a puncturing method for irregular LDPC codes is designed to optimize the puncturing degree profiles of LDPC code ensemble based on the density evolution analysis. However, these results presume infinite codeword length, which is not suitable for design of RC-LDPC codes with a short block length. Recently a new puncturing algorithm has been proposed specifically for finite-length LDPC based on grouping nodes with different recoverable steps [4]. In this paper, using the concept of recoverable steps, we propose a simple puncturing algorithm for IRA-type finite-length LDPC codes with medium to very high rate. We first review some definitions on recoverable nodes in [4]. Definition вАУ One-step recoverable node (1SR): A puncture node is called one-step recoverable (1SR) if it has at lease one check node neighbor whose all other connected variable nodes are unpunctured. This check node is called survived check node.
The 1SR nodes can be recovered during the first iteration of iterative message-passing decoder for binary erasure channels (BEC). For AWGN channels, the 1SR nodes may not be fully recovered but have one or more inbound messages with certain reliability from the single parity check decoding at the survived check nodes. For the sake of simplicity, we still call this recovered for the punctured nodes in AWGN channels.
Definition вАУ k-step recoverable node (kSR): If a punctured variable node has at least one check node neighbor which has at least one (k вИТ 1)SR node and others are mSR nodes, where 0 вЙ§ m вЙ§ k вИТ 1, this variable node is then called kSR. The kSR can be recovered after the kth decoding iteration.
Define Gk, k = 1, ¬Ј ¬Ј ¬Ј , KG, as the group that consists of all the kSR punctured nodes. If every punctured node belongs to one of the groups Gk, all the punctured nodes can then be recovered. Otherwise, there always exit one or more dead nodes that cause decoding failure. doing so is that empirically the puncturing pattern is always
more important to the higher rate punctured codes. Deliberate
puncturing on a lower rate code first may produce much larger KG when puncturing on a higher rate code later. For instance, suppose we have deliberately punctured Np parity nodes with one and the only one pattern of J consecutive punctured nodes, J > 1. So we have KG = J/2 . If we further puncture a parity node to form a higher rate code, it will produce a segment of 2J + 1 consecutive punctured nodes. Then KG = J + 1 J/2 . Such a large KG will significantly degrade the performance. This can be avoided by deliberately puncturing (Np + 1) nodes first so that we have the segments that have at most J + 1 consecutive punctured nodes. Then KG = (J + 1)/2 . From the studies in [11], degree-1 nodes play an important

Simplified Extending for Practical Consideration
The code profile obtained from the above optimization to achieve the threshold of Eb/N0 is for infinite-length random codes with cycle-free assumption [12]. For practical applications, it might be difficult to build a good finite-length code (i.e., a few thousand bits or less) from the optimized profiles since the cycles are impossible to avoid. Under the iterative decoding, the existence of cycles causes the error-floor. Here, we apply the design strategy in [8] to build extended code matrix of R < R0. Thus, in order to find the good tradeoff between low threshold decoding and low error-floors, we consider the following simplifications.

When SNR > 0dB, or ќЈ > 0.6, our RC-LDPC codes offer higher throughput than other RC codes. Since our RC-LDPC codes can achieve any high rate R through puncturing, the throughput is then much higher than that from other RC codes when SNR > 3 dB. By also considering Rmax = 1.0 for the uncoded system, the throughput can gradually reach one at high SNRs. This threshold rate is boosted up to 0.94 by our designed RC-LDPC codes. Furthermore, our RC-LDPC codes provide smooth throughput performance at low SNR region for the throughput below 0.3, where for other RC codes, there is no throughput result presented because of limitations of their design schemes which could not provide good low-rate codes.
