An 860-Mb/s (8158,7136) Low-Density Parity-Check Encoder
Lowell H. Miles, Jody W. Gambles, Senior Member, IEEE, Gary K. Maki, Senior Member, IEEE, William E. Ryan, Senior Member, IEEE, and Sterling R. Whitaker, Senior Member, IEEE

Abstract—Low-density parity-check codes achieve coding performance which approaches the Shannon limit. An (8158,7136) encoder was implemented in a five-metal, 0.25- m CMOS process. Use of generator polynomial reconstruction, partial product multiplication and functional sharing in the parity register results in a highly efficient design. Only 1492 flip-flops along with a programmable 21-bit look-ahead scheme are used to achieve an 860-Mb/s data throughput for this rate 7/8 LDPC code. A comparable two-stage encoder requires 8176 flip-flops.
Index Terms—Error correction, encoding, cyclic codes, LDPC, quasi-cyclic codes.

Virtually all LDPC code design procedures involve specification of the matrix after which an encoding procedure must be devised. A generator matrix may be obtained from using Gauss-Jordan elimination, but the resulting encoding operation via (1) is still too complex for long codes. To circumvent this, cyclic and quasi-cyclic (QC) LDPC codes have been proposed which lend themselves to lower-complexity shift-register based encoder implementations [3]–[6]. The low-complexity advantage derives from the fact that the matrix is composed of circulant sub-matrices. The code whose encoder implementation is described herein falls in the QC class.
This encoder actually implements two low-density parity-check codes. The first is a (4088,3360) code (4K) which is shortened from a (4095,3367) cyclic code [4]. The second is a (8158,7136) code (8K) which is a shortened version of a (8176,7154) subcode of a (8176,7156) QC LDPC code [5], [6]. Shortening is applied as a practical system consideration. By making the message a multiple of 16, no external packing is required for the data. Also included on the chip is a Consultative Committee for Space Data Systems (CCSDS) standard randomizer as well as a CCSDS standard attached synchronization marker (ASM) section.
The cyclic 4K coding block was designed based on well known techniques [3], [7]. The design of the 8K encoder portion of the chip will be the subject of this paper. In Section II, the generator matrix for the (8158,7136) code will be presented. A detailed description of the architecture for the 8K encoder will be presented in Section III as well as a comparison of the implemented architecture to other proposed 8K architectures. In Section IV, the design and layout techniques used to overcome the interconnect challenge inherent with LDPC codes will be covered.

An 860-Mb/s data rate can be sustained using a 54-MHz system clock. 446 16-bit data words are input to the chip and passed out of the chip with nine clock cycles of latency when no frame marker is inserted. When the two-word frame marker is prepended, data has an eleven clock cycle latency. 64 16-bit words follow, of which 63 are all parity bits. The final 16-bit output word contains the last 14 bits of parity with 0’s appended.

As an observation, implementation complexity of LDPC en-

coders can vary dramatically based on the matrix chosen.

Three encoding schemes for implementing a quasi-cyclic (8176,7154) LDPC code were compared in [6]. One of these schemes, a two-stage (TS) architecture, was a high throughput encoder requiring 511 clock cycles to encode the message bits for the 8K code. This compares directly with the 511 clock cycle throughput of the architecture described in this paper. In addition, two slower shift-register-adder-accumulator (SRAA) encoders are described in [6]. The first, SRAA-1, requires 7154 clock cycles to encode one block of data while the second, SRAA-2, requires 1022 clock cycles. An SRAA-1 implementation uses 2044 flip-flops while SRAA-2 needs 7154 flip-flops and TA requires 8,176 flip-flops.
The elimination of and functional sharing of flip-flops in the architecture presented in this paper results a very efficient design. There are a total of 2230 flip-flops used, 728 of which are in the parity register for the 4K code. An additional 10 flip-flops are in the randomizer and marker logic. This leaves 1492 flip-flops for the 8K encoder which includes all pipe registers into and out of the chip as well as all control and state machine flip-flops.
Assuming 5.5 logic gates per flip-flop, 2.5 logic gates per two-input XOR, and 1.5 logic gates per two-input AND, the SRAA-1 implementation requires a total of 15 330 logic gates while SRAA-2 requires a total of 67 961 logic gates and TS requires 50 078 logic gates. The 8K architecture presented in this paper requires 30 690 logic gates. Table IV summarizes these comparisons.
