Abstract 
A new "memory-aware" Low-Density Parity-Check codes decoding is presented. The proposed algorithm performs 
a single-phase update in which the variable node elaboration is embedded in the check node phase. This reduces the 
memory size to the number of bits in the codeword, for a saving of 70-80% depending on the code in use. Despite 
these rough approximations, simulation results on AWGN channel provide almost the same BERIFER performance 
of the exact belief propagation (only about 0.2 dB is lost in the transmission SNR) thanks to the introduction of 
stabilising factors driving the algorithm, which are adaptively updated through the decoding process. Remarkably, 
the decoding throughput is roughly doubled at high SNR and the computational complexity is also lower than 
current 2-phase serial implementations. 
1 
Introduction 
Recent advances in microelectronics technology have 
fostered the renaissance of Low-Density Parity-Check 
(LDPC) codes, and a large number of studies have been 
accomplished in the last decade as a follow up to Neal's 
and MacKay's bootstrap activity [1]. 
Introduced by Gallager in 1963 [2], LDPC codes 
were soon abandoned because of the high implementa­
tion complexity. However, the need of modern commu­
nication systems operating close to the Shannon limit of 
channel capacity [3] has renewed the interest in LDPC 
codes for practical applications. Next communication 
standards, such as DVB-S2, WLAN (802.11 n) and 
WMAN (802.16e), have adopted or are considering to 
use such codes. 
Nowadays, the main challenge to the success of 
LDPC codes remains the possibility of defining low­
complexity high-throughput decoders. It has been 
largely demonstrated that implementations based on 
parallel architectures require large chip areas even in the 
case of short block sizes [4], mainly caused by routing 
congestion. The adoption of "architecture-aware" [5] or 
"VLSI oriented" [6] LDPC codes, as well as reversing 
the code-decoder design 
[7], only partly solves the 
problem, and the decoder complexity remains an issue. 
In addition, memory requirements in terms of size and 
control (parallel access) also play a significant role 
in the overall complexity, especially for long-block 
codes as the DVB-S2 ones, where more than 200,000 
messages are updated and stored at each decoding step. 
In [8], a modified a-posteriori probability (APP) 
algorithm, called staggered decoding, is presented. This 
completely removes the variable node phase of the APP 
decoding, and limits the number of memory locations 
to the number of bits in the codeword, with evident sav­
ings. As for layered decoding [5], [9], the convergence 
speed of staggered decoding takes advantage of always 
up-to-date metrics; however a detriment in the BER 
performance is shown for a number of iterations greater 
than five. Moreover, a certain instability is exhibited, as 
the error rate is not always monotonic with iterations. 
Starting from these observations, this paper aims at 
improving the staggered decoding with the introduction 
of some stabilisation factors. Remarkably, the com­
parison w.r.t. the traditional message passing decoding 
reveals small losses in the BERIFER curves. The anal­
ysis under finite precision arithmetics is considered too. 
Finally, a VLSI architecture for the node processor im­
plementing the modified algorithm is briefly described, 
and main advantages w.r.t. conventional solutions are 
discussed.

Simulation Results 
The performance of the proposed Adaptive Single­
Phase Decoding (ASPD) algorithm has been compared 
with the Staggered Decoding (SD) and with the tradi­
tional BP over AWGN channel. Two codes have been 
used as a benchmark: the first one is structured and 
semi-random, with codeword size N = 1944 and rate 
1/2, proposed by the WWiSE consortium for WLAN 
applications [12]; the second one is drawn from the 
MacKay's on-line data base (label 4095.738.4.102), 
with codeword length N = 4095 and rate 0.82. In the 
first code dv takes the values 7,3,2 and 1, while in the 
second one it is constant to 4. 
Figures 1 and 2 compare the BER of the three 
decoding techniques for the WWiSE code. As current 
implementations are based on serial architectures of 
the node processors [13], the iteration time of the 
traditional BP decoding is roughly double the ASPD or 
SD one, which only perform the check update phase. 
Thus, in order to achieve the same decoding throughput, 
ASPD and SD are run for twice the number of iterations 
of BP. Accordingly, as to explore the asymptotic per­
formance, the former are reported up to 200 iterations, 
while 100 iterations are run for BP decoding. As shown 
in Fig. I, SD is very effective in the first iterations, 
but it sticks onto its asymptotic limit already after 5 
iterations, the gap to BP decoding being more than 
2 dB at BER=10-5, and even higher at lower error 
rates. On the other hand, ASPD largely outperforms SD 
and its gap to BP is only 0.2 dB down to BER=10-8. 
These observations are summarized in Fig. 2, which 
shows the BER of ASPD, SD and BP as a function 
of the number of iterations at Eb/ No=2.25 dB. SD is 
confirmed to be the faster algorithm in the first 2 - 3 
iterations, but ASPD corrects more errors than BP in the 
first 16 iterations; however, it does not reach the same 
asymptotic floor of BP. Note that in Fig. 2, the number 
of iterations has been normalised to the iteration time of 
BP, in order to compare the three algorithms for a given 
decoding time or, dually, a given throughput. Denoting

VLSI Implementation Issues 
The proposed LDPC decoding described in Section 2 is 
suitable to be implemented with a low-complexity serial 
architecture. Savings in complexity are twofold: on the 
one hand, memory requirements are highly reduced; on 
the other hand, only a single phase of the traditional 
log-BP decoding must be implemented.

5 
Conclusions 
As an enhancement of the staggered decoding presented 
by Yeo et al. in [8], a new algorithm for decoding of 
LDPC codes has been described in this paper. Basically, 
it adaptively updates a single metric for each variable 
node in the codeword, by means of a single-phase pro­
cessor fed with the channel a priori LLRs. The single­
phase processor outputs refined approximations of the 
APPs on each bit in the transmitted codeword, which 
are continuously cumulated to the node metric in a 
running sum with leakage. This makes the proposed so­
lution very attractive from the performance-complexity 
trade-otl point of view: for a reduction of the message 
memory of about 70-80% and of processing complexity 
of about 12%, only few tenths of dB are paid in the 
transmission SNR w.r.t. the traditional message passing 
decoding. Remarkably, this loss is even less when a 
coarser message quantisation is adopted (5 bits). This 
is particularly interesting for long codes where storage 
constraints are as crucial as processing complexity. 
Moreover, the use of metrics always up-to-date, 
along with a halved iteration time, can be exploited in 
a twofold way: either the average decoding throughput 
of ASPD is twice the BP one, or, if speed is not 
an issue, the chip clock frequency can be halved. 
Chip complexity and power consumption are therefore 
reduced. 
Although preliminary simulation runs confirmed the 
good performance of the presented algorithm with 
several codes, a dedicated, ad-hoc, refinement of the 
adaptive control law could be worth to further reduce 
the gap from BP decoding.
