A Low Complexity and Programmable Encoder Architecture of
the LDPC Codes for DVB-S2

Abstract
We propose a low complexity and programmable encoder architecture of the LOPC codes for OVB-S2 standard, where 21 variations of LOPC codes are defined. The proposed LOPC encoder takes advantage of the features of the codes, that is quasi cyclic codes and irregular repeat-accumulator codes. The key issue for realizing low complexity is a quasi-cyclic encoder using RAMs instead of shift registers. We implemented a programmable encoder circuit which is applicable to all LOPC codes defined in OVB-S2 standard, achieving easy operation.

1 Introduction
Low-density parity-check (LOPe) codes [1] are an active research topic due to its excellent error correct­ ing performance. The threshold performance of LOPC codes can be analyzed by density evolution [3], and Richardson showed that LOPC codes can achieve the Shannon limit when the code length goes to infinity [4]. In recent years, researchers have become increasingly interested in how to implement the encoder and decoder into hardware efficiently.
OVB-S2 [5] is the first digital video broadcast stan­ dard in which LOPC codes are adopted as forward error correcting (FEe) codes [6]. In this standard, 21 variations of LOPC codes are defined by structured parity check matrices, of code length 16200 or 64800, and the code rate varies from 114 to 9110. Some researchers [8] [9] have already reported their LOPC decoder for OVB-S2. One of them also implemented an LOPC encoder [8]. However, no mention is made about their hardware encoder architecture.
An LOPC code is a linear block code defined by a sparse parity check matrix. In contrast to turbo codes [2], the encoding scheme is not originally provided. When considering encoding of LOPC codes, they dis­ cuss the complexity for encoding. However, in the case of hardware implementation, easy operation is also required for an LOPC encoder circuit. That is, when integrated in a system, single clock circuit is easier to operate than multiple clock circuit. And, continu­ ous input/output interface is also better than complex constrained interface. In addition, programmability is a requisite for an LOPC encoder circuit for OVB-S2.
In this paper, we discuss encoding scheme of various LOPC codes. And, after we introduce the LOPC codes in OVB-S2, we show our encoder architecture which

achieve low complexity, programmability, and easy operation.

3.1 LDPC codes for DVB-S2 In OV8-S2 standard [5], 21 variations of LOpe codes are defined. Table I is a list of the parameters of all 21 codes, where N is code length, r n is the nominal code rate, and r is the real code rate. And, ko, no, and q = no - ko are the information vector length, code vector length, and the number of rows of the parity check matrices, respectively, which are divided by 360.
All codes are defined by structured parity check matrices which can be partitioned into two parts, such as the information part and the parity part. In the parity part, the staircase structure of an IRA code appears as same as Fig. 4. In the information part, a new column pattern appears in every 360 columns, and the following 359 columns are cyclic shifts of the first column by q. That is, the pattern of the xth column is obtained by the method below.
• copy the pattern of (360 x Lx/360J )th column. • shift cyclically by (x mod 360) x q, where, LaJ is the largest integer such that LaJ :S a.

4 Encoder Architecture
4.1 Conditions for the Encoder Circuit
In this section, we propose an architecture of an LOPC encoder circuit for OVB-S2 which satisfies the condi­ tions below.
• Programmability. • Low complexity. • Single clock. • Constant parallel I/O interface. The last condition is to achieve throughput of over 100 Mbps at lower clock frequency.

In general, one bit of register is quite a bit larger than one bit of RAM, and the total area size of RAM increases when divided into multiple macros. However, the area size of RAM of small word size and large bit size is not severely influenced. Furthermore, the bit size of 360 is too large to be realized within one RAM macro. In fact, the maximum bit size of a RAM macro of FPGA is 36 bits. So, it is concluded that the complexity of the proposed LOPC encoder for DVB-S2 codes is obviously less than an encoder which requires all information bits (57k bits) to be stored in shift registers.

A quasi-cyclic encoder using partitioned single port RAMs is decisive to achieve low complexity. The pro­ grammability is achieved by the controlling sequence for the cyclic shift block, the order controlling block, and the RAM block. And the fact that the proposed encoder works on a single clock, and provides a con­ tinuous byte-wise input/output interface makes it easy to operate when integrated in a system.
We implemented an encoder circuit for DVB-S2 codes on a FPGA, which is fully compatible to all LOPC codes defined in the standard. The synthesis result shows that it occupies 11840 LUTs and 11 RAM blocks (one for program sequence), and works on 50 MHz clock. The complexity is smaller than other existing encoder, and the maximum throughput is up to 400 Mbps.
