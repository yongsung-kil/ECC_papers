Designing a Self-orthogonal Quasi-cyclic Code with Extended Minimum Hamming Distance
Makoto Noda Core Tech. Develop. Group, Sony Corporation, 6-7-35 Kitashinagawa Shinagawa-ku, Tokyo, 141-0001 Japan

Abstract
A method to design a parity-check matrix with a generalized anti-Pasch configuration is developed for a self­ orthogonal quasi-cyclic code as a regular low-density parity-check code. This method can be applied to search a parity-check matrix with an arbitrary constant column weight and with an arbitrary number of rows. The minimum Hamming distance of a code designed by this method is at least the column weight plus two for an even column weight and the column weight plus three for an odd column weight. It is shown that the error floor and undetected error probabilities of a code with a high code rate above 0.95 designed by this method are significantly improved compared with those of a conventional self-orthogonal quasi-cyclic code based on finite geometry.

Introduction

One aspect of designing a low-density parity-check (LDPC) code [1] for practical use is how to construct a code simultaneously achieving a low error-rate performance and using low-complexity encoder and decoder circuitry. One approach to this problem is to design a structured LDPC code such as a self­ orthogonal (SO) quasi-cyclic (QC) code [2]. An SO­ QC code has advantages in the simplicity in both the encoder [3], [4] and decoder circuitries [5].
The SO code is a code such that the parity-check rules are chosen so that no two information symbols appear together in more than one parity-check equation. This property is equivalent to the property that the parity-check matrix of the code does not contain a cycle of length 4.
The QC code is a code such that every cyclic shift of a codeword by p symbols yields another codeword, where p is a positive integer. Townsend and Weldon developed the SO-QC code [3], and Kou, et al. showed that an SO-QC code constructed based on finite geometry is useful for an LDPC code [2].
It is well known that the minimum Hamming distance, dmin, of an SO code is at least the minimum column weight of the parity-check matrix plus one [3]. When a parity-check matrix of the SO code has a constant column weight, w, dmin of the code takes a value of w+ 1 in many cases. Several code­ construction methods extending dmin to at least w+ 2 for an SO code had been developed [6]-[8].
In the case of w = 2, Song, et al. showed a method of removing cycles of length 6 from a parity-check matrix for an SO-QC code [6]. dmin of the code is not described in [6] but at least 4. This method, however, cannot be applied to a QC code with w::>: 3.
In the case of w = 3, it is known that dmin increases to at least 6 from 4 by giving an anti-Pasch (or

"quadrilateral-free") configuration in a Steiner triple system on a parity-check matrix [7], [8]. The anti­ Pasch configuration can be generalized to the case of arbitrary column weight and Vasic designed this class of codes by using a balanced incomplete block design (BlBD) [8].
We propose a new method of designing a parity­ check matrix with the generalized anti-Pasch (GAP) configuration used for a computer search to obtain an SO-QC code with a large dmin•

3.2 Generalized Pasch configuration and the minimum Hamming distance
It has been shown that dmin of an SO code with a parity-check matrix with anti-Pasch configuration in the Steiner triple system, i.e. in the case w = 3, is six [8]. The Pasch configuration in Steiner triple system is known as a collection of four blocks whose union is a set of six points [7]. Similarly, the generalized Pasch configuration is a collection of w+ 1 blocks whose union is a set of u points, where u == w(w+ 1)/2 [8]. The relation between the generalized Pasch configuration and dmin of an SO code is described in this subsection.

Although it had been pointed out that a pseudo­ weight of an LDPC code was a good measure of a weight of an error pattern that would cause nonconvergence in the iterative detection [10], most of the iterative detections at the error floors were converged in our investigation because most of the errors at the error floors were undetected word errors, as can be seen in Fig, 3 and Fig, 4,
When an undetected word error occurred, the minimum number of bit errors in the codeword was 6 for the (2667, 2540) new code, but was 4 for the PG code and 5 for the EG code, These results support that dmin is 6 for the (2667, 2540) new code, 4 for the PG code and 5 for the EG code, In the (5355, 5101) new code, no undetected word error was observed during the simulation, dmin of the (5355, 5101) new code is at least 6,
