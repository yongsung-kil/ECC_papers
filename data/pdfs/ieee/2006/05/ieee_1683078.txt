Improved Belief Propagation (BP) Decoding for LDPC Codes with a large number of short cycles

Abstract-In this paper, we improve performance of Low Density Parity Check (LDPC) codes by adding a large number of short cycles. Short cycles, especially cycles of length 4, degrade performance of LDPC codes if the standard BP (Belief Propagation) decoding is used. Therefore current researches have focused on removing cycles of length 4 for designing good performance LDPC codes. We found that a large number of cycles of length 4 improve performance of LDPC codes if a modified BP decoding is used. We present the modified BP decoding algorithm for LDPC codes with a large number of short cycles. We show that the modified BP decoding performance of LDPC codes with a large number of short cycles is better than the standard BP decoding performance of LDPC codes designed by avoiding short cycles.

I. INTRODUCTION
Low Density Parity Check (LDPC) codes were first discovered by Gallager [1] and rediscovered by Mackay et al [2]. LDPC codes have comparable performance with Turbo codes [3]. The bipartite graph [4] of an LDPC code contains cycles. The presence of cycles in LDPC code graphs can introduce a noticeable loss in performance if the standard Belief Propagation (BP) decoding is performed. In order to improve BP decoding performance of LDPC codes, short cycles are usually removed when parity check matrix H is designed. However, we improve performance of LDPC codes by adding a large number of short cycles. Such performance improvement is obtained by using a modified BP decoding algorithm. The modified BP algorithm is based on a bipartite graph, which two check nodes with cycle of length 4 are combined together as a joint check node. We call this bipartite graph as the Joint Check (JC) bipartite graph.
In this paper, we improve performance of LDPC by adding a large number of short cycles and by decoding with a modified BP algorithm based on JC-bipartite graph. We found that a large number of cycles of length 4 improve performance of LDPC codes if a modified BP decoding is used. We present the modified BP decoding algorithm for LDPC codes with a large number of short cycles. We show that the modified BP decoding performance of LDPC codes with a large number of

short cycles is better than the standard BP decoding performance of LDPC codes designed by avoiding short cycles.

II. JOINT CHECK (JC) BP DECODING ALGORITHMS
It is well known that suboptimal iterative decoding for LDPC codes, such as the standard BP decoding, suffers large performance degradation due to short cycles, especially cycles of length 4. In bipartite graph [4], cycles of length 4 are represented as shortest close paths between check nodes and variable nodes. In other words, two rows in parity check matrix H have ones in two same column locations. Therefore, we propose a modified BP algorithm for LDPC codes with a large number of short cycles. The modified BP algorithm decodes jointly two check equations with a cycle of length 4. We call this decoding algorithm as "Joint Check (JC) BP" decoding because this algorithm is based on JC-bipartite graph. Fig. 1-(a)

IV. CONCLUSION
We improved performance of LDPC codes by adding a large number of short cycles. Short cycles, especially cycles of length 4, degrade performance of LDPC codes if the standard BP decoding is used. Therefore current researches have focused on removing cycles of length 4 for designing good performance LDPC codes. We found that a large number of cycles of length 4 improve performance of LDPC codes if a modified BP decoding is used. We presented the modified BP decoding algorithm for LDPC codes with a large number of short cycles. We showed that the modified BP decoding performance of LDPC codes with a large number of short cycles is better than the standard BP decoding performance of LDPC codes designed by avoiding short cycles for both regular LDPC codes and irregular LDPC codes.
