A Low Memory FPGA Based LDPC Decoder Architecture for Quasi-Cyclic LDPC codes
Paul Saunders and Antony D.Fagan
DSP Research Lab, School of Electrical, Electronic and Mechanical Engineering University College Dublin, Dublirn, Ireland
E-mail: paul. saunders@ee .ucd. ie tony. f agancucd. ie
Abstract We propose a novel low memory fully programmable FPGA decoder architecture to decode quasi-cyclic LDPC codes. By performing optimizations at the code construction, algorithmic and architecture levels we are able to achieve significant memory storage advantages over current FPGA decoder implementations. Our decoder employs the modified turbo decoding algorithm, to achieve a memory utilisation of 71Kb using a Xilinx Virtex-4 device.

I INTRODUCTION
Low-Density Parity-Check (LDPC) codes were introduced to the coding community by Gallager in
1962 [1] but it was not until 1995, when MacKay
and Neal [2] rediscovered them, for their full potential to be understood. LDPC codes have currently become a topic of great interest because of their excellent error correcting ability and their highly parallelizable decoding algorithm. Over the past few years many advances have been made in the design and analysis of LDPC codes. Some of these codes now offer near Shannon capacity performance and also out perform their Turbo Code counterparts [3]. However, these theoretical advantages have not been transferred into the practical arena. The realization of an efficient and practical LDPC decoder architecture still remains a big challenge and determines how well we can exploit the potential merits of LDPC codes in real world applications.
The typical design philosophy is to jointly consider the code design and decoder implementation. Our methodology not only considers the code design but the algorithmic and architecture design also. The code construction of most LDPC codes is typically random. which results in highly irregular hardware interconnectivity. This goes against efficient hardware implementation paradigms which require regular and structured design features. The class of quasi-cyclic LDPC codes targeted to the FPGA architecture in this paper, exhibit these design features. Secondly at the algorithmic level, the modified turbo decoding algorithm is used to address the memory overhead problem seen in current FPGA decoder implementations. Moreover, a parallel soft-input soft-output (SISO) message up-

date mechanism, in the form of a message decoding
unit (MDU), based on the Log-MAP algorithm [4]
is used. The MDU implements the recursions of the Log-MAP algorithm using metric differences based on simple max operations. Finally at the architecture level, a scalable architecture is implemented that can decode both regular and irregular LDPC codes of varying code rates and lengths each exhibiting the quasi-cyclic structure.
The two main contributions of this paper are as follows: firstly we present the first application of the modified turbo decoding algorithm to quasicyclic LDPC codes and secondly we are the first to provide an FPGA implementation of the modified turbo decoding algorithm. The remainder of this paper is organized as follows. We begin by discussing the theory behind LDPC codes and their graphical representation in Section II. In Section III we discuss the methods used to construct the quasi-cyclic codes that are targeted to the FPGA architecture in this paper. The modified turbo decoding algorithm which is employed in our FPGA architecture is discussed in Section IV. In Section V we present all the modules that constitute our FPGA architecture. The error correction ability of our codes as well as the final architecture statistics are given in Section VI. Section VII provides some concluding remarks.

a) Tanner Graph Representation
An LDPC code is a linear block code defined by a m by n sparse parity check matrix H. For block codes a vector x is a codeword if and only if it satisfies: H-x = 0. The parity-check matrix of a LDPC

III HARDWARE TARGETED CODE CONSTRUCTION
Our architecture aware LDPC codes are based on
the construction methods presented in [6] and [7].
The construction method proposed in [6] was limited to regular codes and therefore could not take advantage of the error correcting benefits offered by irregular LDPC codes. This shortcoming was observed in [7]and the construction method was modified to include irregular codes. The construction method of [7] involves two stages, base matrix construction and matrix expansion.
a) Base Matrix Construction
In this step we produce an irregular (C, R) LDPC code defined by a D by B base parity-check matrix Hbase whose girth is 6 and whose degree distributions are determined by a technique known as density evolution [8]. Specifically we chose C = {2,3,4} and R = {5,6}. We begin with an
all zero matrix Hbae, and randomly insert l's on a column by column basis into the Hbase subject
to the girth and degree distributions mentioned above.

V THE FPGA DECODER ARCHITECTURE
a) The Codes Targeted to the Architecture
In this paper we will present results for two codes derived from the same D by B base parity-check matrix Hbase. Our base parity-check matrix is constructed using the methods described in Section III and contains 54 rows and 100 columns. It is then expanded by a factor of S = 32 resulting in a length 3200 rate 0.46 LDPC code. The second code is constructed by expanding Hbase by a factor of S - 64 resulting in a length 6400 rate 0.46 LDPC code. The bit error rate (BER) performance of these codes using the modified turbo and turbo decoding algorithms is shown in Section VI.
b) A High Level Description Of The Architecture
We chose to design the partly parallel architecture to decode the length 3200 rate 0.46 code. We also chose to use 5 bits of resolution in our design as this offered the best compromise between performance and logic utilization (See Section VI). c) The Message Processing Units The structure of a MDU is shown in Fig.5.

Fig. 5: The structure of a MDU
Each MDU contains 4 inputs and 4 outputs. The 4 inputs are the left and right LLRs read from the ay and A memories. The 4 outputs are the left and right updated LLRs to be written back to the -y and A memories. The reason for providing both left and right inputs during a single clock cycle is that the a and 3 recursions performed by the MDUs are independent and can be calculated simultaneously. Therefore data corresponding to
the left permutation sub-matrix of a block row, appears on the left input ports. Similarly, the data corresponding to the right permutation submatrix of a block row, appears on the right input ports. The data appearing on the left input port moves from left to right along the trellis. Similarly, the data appearing on the right input port moves from right to left along the trellis. The point in
the trellis where the a and ,B recursions meet is known as the crossover point. The corresponding updated LLRs for each left and right permutation sub-matrix appears on the left and right output ports. Last In First Out (LIFO) buffers are provided to store the intermediate a and i3 values before crossover occurs. LIFO buffers are also provided to store the left and right p values which are summed with the updated A LLRs to produce the updated r LLRs.

d) The Memory Modules
The -y memory is a dual-port memory module containing 100 storage locations, each 160 bits wide. Both ports are configured for reading and writing. Each A memory is a dual port memory module containing 162 storage locations each 160 bits wide. One port is configured for reading, the other for writing. The H and i7r memories are combined into a single port read only memory (RUM). There are also three control bits that are appended to each storage location. These control bits give the trellis size of the block row being decoded. They are used
to control the read address and write address gen-
erators as well as the timing of the MDUs. This results in a memory module containing 162 stor-
age locations each 27 bits wide. The total memory utilization of our decoder is 71Kb or 72214 bits.
e) The Bi-Directiornal Transport Network
The transport network contains two 160 bit wide data paths in the forward direction and two 160 bit wide data paths in the backward direction. Each data path transports either the left or right reliability values from -y memory to the MDUs and back again. There are two cyclic interleavers in the forward direction. Each interleaver cyclicly shifts the data on its input to the left by the value stored in the ir memory. There are also two cyclic deinterleavers in the return direction. Each deinterleaver cyclicly shifts the data on its input to the right by the value stored in the ir memory. Both the interleavers and deinterleavers are implemented as 160 bit barrel shifters. Barrel shifters provide an efficient mxeans of cyclicly shifting a bit vector. They achieve this by performing the shift in stages of power of 2 i.e. in the first stage a shift of 1 is performed, in the next a shift of 2 followed by a shift of 4 etc.

VII CONCLUSION
A partly parallel FPGA architecture has been proposed based on the modified turbo decoding algorithm. The architecture achieves a memory utilisation of 71Kb for a length 3200 Quasi-Cyclic LDPC code and a BER of 10-6 at a SNR of 2.5dB over the AWGN channel. Our FPGA decoder is superior in terms of memory utilization when compared to other FPGA architectures presented in literature.
