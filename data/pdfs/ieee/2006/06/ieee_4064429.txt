Abstract-Channel coding is an important building block in communication systems. Low-Density Parity-Check codes is one kind of prominent error correcting codes being considered in next generation industry standards. This paper presents a memory efficient, very high speed decoder architecture suited for Quasi-Cyclic Low-Density Parity-Check codes using modified Min-Sum decoding algorithm. In general, about seventy percent of message memory can be saved over conventional decoder architectures, and the decoding speed can be largely accelerated because of the highly efficient VLSI architecture. Consequently, the proposed approach facilitates the applications of LDPC codes in area/latency sensitive communication systems.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) codes [1] have recently received a lot of attention because of their excellent error-correcting capability. A class of structured LDPC codes, namely quasi-cyclic LDPC (QC-LDPC) codes, has been proposed [2]. The QC-LDPC codes can achieve comparable performance with equivalent random LDPC codes [3] [4]. In addition, more importantly, they are well suited for hardware implementation because of the regularity in their parity check matrices. Especially in that the encoder of a QCLDPC code can be easily built with shift registers [5] while usually it is hardware complex to encode an LDPC code.
However, the implementation of QC-LDPC decoder is not trivial. Partially parallel decoding architectures for QCLDPC codes are well studied nowadays [6] [7], which uses RAM to store messages transferring between variable nodes (VN) and check nodes (CN). The huge memory requirement for message storage is the biggest problem. In addition, the low throughput or high decoding latency remains the major shortcoming of the traditional partially parallel architectures.
Among various LDPC codes decoding algorithms, the Sum Product algorithm (SPA) has the best decoding performance. The minimum-sum decoding algorithm [8] [9] is an approximation of the SPA. From the perspective of implementation, the Min-Sum algorithm requires less computation and estimation of noise power is unnecessary for an additive white Gaussian noise (AWGN) channel. Furthermore, the Min-Sum algorithm can help reducing the

message storage requirement because the messages transmitted from a check node to adjacent variable nodes per iteration have only two possible magnitudes. However, this advantage is not easy to take in hardware implementation. In [10], the authors implemented an LDPC decoder utilizing the Min-Sum algorithm. The memory requirement reduction is achieved, but the hardware is complex, and the decoding latency is extremely high. The fully efficient use of Min-Sum algorithm in decoding LDPC codes still remains unresolved.
This paper presents a (8192, 7168) (4, 32) regular QCLDPC decoder implementation which is very-high speed and memory efficient by efficiently using the Min-Sum decoding algorithm. The paper is organized as follows: after a brief review of the decoding algorithm in Section 2, the parity check matrix construction and transformation is explained in section 3. The VLSI architecture of the proposed decoder is presented in Section 4. Section 5 gives the implementation results, and Section 6 concludes the paper.

C. Conversion OfQC-LDPC To Shift-LDPC

The columns of a QC matrix can be rearranged to convert it to a corresponding shift matrix. We consider the code we implemented: (8192, 7168) (4, 32) regular QC-LDPC code with the width of sub-matrix P = 256.

The first 256 columns (in sub-matrix H11 H21 H31 H41) are relocated to [256k, 256k+7] (O<k<32) columns. The 0th column is unmoved; the Ith column is moved to the 256th column; the 2th column is moved to the 512th column, and so on. After the 3 1th column is moved to the 7936th column, the 32th column is then moved to the Ith column...

The second P columns (in sub-matrix H12 H22 H32 H42) are relocated to [256k+8, 256k+15] (O<k<32) columns... Rearrange all the columns until the matrix is done.

In brief, we can say that the qth column (q=256*a+b) is rearranged to column q' as following:

IV. DECODER VLSI ARCHITECTURE
256 VPUs and 1024 CPUs are instantiated in the decoder. We first present the decoding schedule.
Step 1: Receive the value from channel, and save them inside the VPUs.
Step 2: Do the decoding iterations. It needs 32 clock cycles per iteration. In each clock cycle, the decoder performs the column process and row process serially.
1) Column process by VPUs. In every clock cycle, one column block (256 columns) is processed. Each VPU get four check to variable messages input and compute back the variable to check messages, so that the 1024 CPUs can get one input each.
2) Row process by CPUs. Each CPU performs one step of the check node process. The whole check node process is divided into 32 steps. In every clock cycle, the CPUs do the computation and save the result in the next CPU so that every CPU can process the message from the same VPU all the time. This will be explained more clearly in the following.
Step 3: Terminate the decoding process when a satisfied codeword is found or when the maximum affordable iteration times has been exhausted.

We show the message memory management in Fig.3 and Fig.4. There are three groups of registers: old registers, new registers, and sign registers. The old register group contains the result of the last iteration. The new register group contains the values of the current iteration. They are both being shifted all the time to ensure that every CPU is receiving message from and delivering to the same VPU. After each iteration, the new reg i contains the min, 2nd-min, index and the xor of all the signs of row i-3 1, as explained above. The message storing method is similar to the method used in [4] in that this method can most greatly reduce the message memory requirement. These values are transferred to the old register group to provide messages from check nodes to variable nodes in the next iteration. The sign register group contains 32 message signs of the variable to check messages, and they are not shifting between each other. In every clock cycle, each sign register shifts in a sign bit of the variable to check message and shifts out a sign bit of a check to variable message (has to do xor with the sign bit in current old reg to gain it). Remember that they are always for the same VPU, so they do not need to be shifted between each other.

VI. CONCLUSION
In this paper, the implementation of a high rate (8192, 7168) (4, 32) regular QC-LDPC decoder is presented. The implementation results show that our decoder architecture will be very competent in future applications, especially in dealing with hardware complexity, throughput and decoding latency.
