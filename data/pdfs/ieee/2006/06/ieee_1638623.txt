Adaptive Offset Min-Sum Algorithm for Low-Density Parity Check Codes
Ming Jiang, Chunming Zhao, Member, IEEE, Li Zhang, and Enyang Xu

Abstract— An improvement is presented to the Offset MinSum decoding algorithm for Low-Density Parity Check codes. The proposed algorithm introduces a more efficient adjustment for check-node update computation in view of different minimum values. According to the optimal correction factor of Normalized Min-Sum algorithm, we can determine the adaptive offset item. The improved algorithm achieves a noticeable performance gain with only a modest increase in computation complexity.
Index Terms— LDPC codes, normalized/offset min-sum algorithm, iterative decoding.
I. INTRODUCTION
R ECENTLY, because of the near Shannon performance and the well-known Belief Propagation (BP) iterative decoding algorithm of Low-Density Parity Check (LDPC) codes [1], rediscovered by [2], various work has been completed focusing on both improvement of decoding performance and reduction to complexity. The update process of the variable node or the check node costs excessive storage and computation resources. For the standard BP algorithm in log-likelihood ratio (LLR) domain, a lot of logarithmic and multiplicative operations are involved in the check-node update. For simplicity, the check-node update of Min-Sum (MS) or BP-based algorithm [3] only selects the minimum and sub-minimum values among the input reliability from the variable nodes, therefore incurring decoding performance degradation. Much work has been done on the improved implementations for the MS algorithm during the past few years. These improvements can be distinguished by two kinds of schemes — one is the conditional correction scheme [4]–[8] aiming at improving approximation to the tanh-based check-node function, and the other is unconditional correction scheme [9]–[11] aiming at optimizing scaling factor evaluation via density evolution. In the Normalized Min-Sum (NMS) or the Offset Min-Sum (OMS) algorithm [9], a multiplied or additive correction factor is directly applied to the check-node output of the original MS algorithm in order to improve the decoding performance. For the algorithms with conditional correction, not only should the magnitudes of the input data be compared, but also their relative difference needs to be taken into account. For example, the optimized reduced-complexity algorithm [6] using linear approximation achieves the performance very close to the

III. IMPROVED DECODING ALGORITHM

In the OMS algorithm, a positive factor β is used to make

an additive correction to the check-node update. This constant

corrective item, however, does not take the output value into

account, so a reasonable adaptive scheme will be a good

improvement to the OMS algorithm. The multiplied factor α

is an efficient correction to the check-node update, but this

normalized process is almost completely determined by the

magnitude of the minimum value, and thus the adjustment

suffers performance degradation when the output is close to

zero.

propose an adaptive offset min-sum algorithm (AOMS), where

the additive offset factor is determined by the magnitude of

the minimum output of the check node. The added correction
