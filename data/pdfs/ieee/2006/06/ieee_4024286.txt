Trapping Sets in Irregular LDPC Code Ensembles
Olgica Milenkovic1, Emina Soljanin2, and Philip Whiting2 1Department of Electrical and Computer Engineering, University of Colorado, Boulder
2Bell Laboratories, Lucent Technologies

Abstract— Trapping sets represent subgraphs in the Tanner graph of a code that, for certain classes of channels, exhibit a strong influence on the height and point of onset of the errorfloor. We compute the asymptotic normalized distributions of trapping sets in random, irregular, binary low-density paritycheck (LDPC) code ensembles. Our derivations rely on techniques from large deviation theory and statistical methods for enumerating random-like matrices. Similar methods can be used for computing the spectra of other combinatorial entities in LDPC code, such as subcodes and/or minimal codewords.
I. INTRODUCTION
The performance and complexity of many well known classes of decoding algorithms are influenced to a large extent by combinatorial properties of the underlying linear block codes. For the Binary Erasure Channel (BEC), iterative decoders fail if the errors are confined to subsets of variable nodes forming a stopping set [2], while for the AWGN channel a similar phenomena can be observed with respect to trapping sets, also known as near codewords [3], [9].
The problem of enumerating and classifying combinatorial entities for a given code is usually intractable. It is therefore of interest to find probabilistic methods that allow for computing the average of a chosen set of parameters for a given code ensemble, which can then be used to describe the average performance of the ensemble as well as the performance of a randomly chosen code from the given category.
We will be concerned with random, irregular LDPC code ensembles and with combinatorial structures that bear an influence on the error-floor characteristics of these codes. LDPC codes have excellent performance at low-to-medium signal-to-noise ratio (SNR), but in certain cases these codes tend to exhibit saturation levels in the bit error rate (BER) curve for sufficiently high SNRs. This saturation in the BER curve is known as the error-floor of the code. For a large class of codes, including codes based on Cayley graphs [11] or certain types of combinatorial designs, error-floors appear at relatively low BERs. This prohibits the use of such codes in storage and deep-space communication systems, which require extremely reliable signalling techniques with BERs below 10−15. For many other codes, error-floors cannot be observed directly, because they are out of reach of standard simulation techniques. In order to assess the properties of such codes, one has to utilize error-floor approximation techniques based on stopping and trapping-set classification. Stopping sets were introduced in [2], while the more general notion of a trapping sets was first described in the context of error-floor analysis in [6]. The latter concept was further developed in the seminal

paper [9], in which trapping set configurations were shown to influence the point of onset as well as the slope of the error-floor curve of LDPC codes. The method described in [9] relies on computer-search strategies and does not provide a guarantee for identifying all relevant trapping sets.
This paper describes a new and conceptually simple method for analyzing the asymptotic behavior of the distribution of trapping sets, based on enumeration techniques described in [4] and large deviation theory [1]. The techniques described in this work also provide a simple framework for evaluating the distributions of other, more complicated, combinatorial entities. The results obtained for both regular [7] and irregular codes indicate a strong dependence of the asymptotic behavior of the number of such sets on the column and row weights of the ensemble, and the rates of the underlying code ensembles.
The paper is organized as follows. The terminology and definitions needed for subsequent derivations are given in Section II. Section III contains a summary of the main results: the asymptotic distribution of trapping sets of size linear in the length of the codes taken from an irregular LDPC code ensembles.
II. DEFINITIONS AND TERMINOLOGY
A. Trapping Sets
Let H be the parity check matrix of an [n, k, d] LDPC code, and let G(H) denote its corresponding bipartite Tanner graph. The columns of H are indexed by variable (left-hand side) nodes V of G(H), while the rows of H are indexed by check (right-hand side) nodes of G(H). We are concerned with subsets of V, obeying the following set of combinatorial constraints.
Definition 2.1: A general ((a, b)) trapping set Ta,b is a configuration of a distinct nodes from V, for which the induced subgraph in G(H) contains b > 0 odd-degree check nodes. An elementary ((a, b)) trapping set is a trapping set for which all check nodes in the induced subgraph have either degree one or two, and there are exactly b degree-one check nodes. The sets of all ((a, b)) general and elementary trapping sets in a Tanner graph of a code with parity check matrix H will be denoted by Ωa,b(H) and Φa,b(H), respectively.
For general classes of binary input, symmetric output channels, the influence of trapping sets on the onset of error-floors in LDPC codes is associated with the following phenomena arising both due to the properties of the code graph and decoding algorithm, as well as to the realization of certain special channel-noise configurations. After this initial increase in the reliability, external variables usually start adjusting the incorrect estimates towards the correct values. The findings described above motivate the study of the most general class of ((a, b)) trapping sets without any restrictions imposed on the relationship between the values of a and b. The distribution of trapping sets with more restrictive properties can be deduced from the general results.

The main components of the proof consist in establishing asymptotic expressions for two “nested” sums. For the first sum, a straightforward application of Sanov’s theorem is needed. For the second sum, Varadhan’s lemma provides an expression which has to be evaluated numerically.
