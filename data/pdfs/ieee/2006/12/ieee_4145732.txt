Abstract Low-Density Parity-Check (LDPC) code is one kind of prominent error correcting codes (ECC) being considered in next generation industry standards. This paper presents an FPGA implementation of array code based Low-Density ParityCheck code decoder. The advantages of the proposed architecture as compared to the fully parallel or partially parallel architecture are: less memory requirement, avoidance of complex global interconnects and its satisfying throughput. These advantages are derived from exploiting the well-defined structure of the parity check matrix of array code based LDPC codes.

I. INTRODUCTION
LDPC codes, due to their near-Shannon limit performance under iterative soft decoding have received significant attention [1] [2]. LDPC codes have been recently selected by the digital video broadcasting standard and are being seriously considered in various real-life applications such as magnetic storage, 10 Gb Ethernet, and high-throughput wireless local area network. In the past few years, significant improvement in LDPC code construction and performance analysis has been experienced. In addition, several efforts are given to the implementation of LDPC code decoder. [3-7]
In general, the complex routing [3], large message memory requirement [4] and limited throughput [5] are the main obstacles in designing an LDPC decoder. This paper presents an efficient FGPA implementation for array based LDPC (ALDPC) codes, which can achieve a good trade-off between
hardware complexity and decoding speed. Typically, over 70%
memory can be reduced and 117Mbps throughput can be achieved.
The structure of this paper is as follows. In Section 2, we briefly review ALDPC codes and the modified Min-Sum decoding algorithm. Section 3 gives the VLSI architecture of
our decoder and Section 4 gives the complexity analysis.
Synthesis and mapping results based on FPGA are given in Section 5. Section 6 concludes the paper.

The typical LDPC decoding algorithm is the sum-product belief propagation algorithm which has two phases [2]. In the first phase, the variable nodes compute updated information which is sent to adjacent check nodes. In the second phase, the check nodes compute updated information based on the new massages from the variable nodes. This update information is then sent back to adjacent variable nodes and the process is repeated.
The modified Min-Sum decoding algorithm [8] [9] is similar to the sum-product algorithm, with an approximation of check node process. It has some advantages in implementation against the sum-product algorithm, such as less computation and estimation of noise power is unnecessary for an AWGN channel.

From the decoding algorithm described above, it can be seen that the calculations are independent between variable nodes or check nodes at each iteration. Therefore, LDPC codes are especially suitable for parallel implementation. In addition, the computation in the decoder is fairly simple, which means low logic consumption. The main obstacle for full parallelism is the complicated interconnection, which makes condensed
chip layout impossible.[3] The problem can be alleviated
through reducing the parallelism (partially parallel or serial) and using storage elements to store the intermediate messages passed along the edges of the graph. Now the problem becomes how to reduce the complexity incurred by the message storage elements and their control logic. For random codes, the edges on the graph are picked randomly, so that additional memory is needed to save the parity check matrix. In addition, the memory addressing will be nontrivial for a partially parallel decoder.
Good code structures can simplify the data accessing.
ALDPC code is a kind of highly structured LDPC code which can achieve comparable performance to random codes. Efficient encoding, minimum distance properties, and performance of array-code based LDPC codes for binary as well as multilevel modulation are addressed in [10]. Fig.3 shows the four CPU Groups. Each group has 47 members (CPU). Group 1 contains from CPU-1 to CPU-47, in charge of the row operation of row 1-47. Group 2 contains from CPU-48 to CPU-94, in charge of the row operation of row 48-94. Group 3 contains from CPU-95 to CPU-141, in charge of the row operation of row 95 141. Group 4 contains from CPU-142 to CPU-188, in charge of the row operation of row 142 188. There is no communication between groups. Except the CPUs in Group 1 (CPU-1iCPU-47), every 47 CPUs in a group form a chain, and each CPU communicates with its two neighbors. For example, in CPU group 3, CPU-i only gets data from CPU-(i-2) and delivers date to CPU-(i+2). Our design relies on the structure of the H matrix. These communications between CPUs can alleviate the complex edge pattern between check nodes and variable nodes, which is the main obstacle in designing high parallel level LDPC code decoder. Here the decoding schedule is presented. It takes 47 clock cycles to finish per iteration. In the first clock cycle, the most
left 47 columns are processed by 47 VPUs. And then, the next 47 columns... In every clock cycle, each VPU gets 4 check-to-
variable messages and compute the 4 variable-to-check message according to equ.2, so that 188 CPUs get one input each. Then, each CPU can deal with one step of the check node process and store the results in memory. This is totally different from the traditional partially parallel decoder architecture [6] in that it combines the two decoding phases together. The whole check node process is divided into 47 steps.
Next we discuss how these simple regular connections implement the correct message passing. For CPU Group 1, it is the most simple. CPU-i receives variable-to-check message from and delivers check-to-variable message back to VPU-i, according to the 47 identity matrices of row 1-47. Hence, the operation of row i (i<47) is performed by CPU-i step by step in 47 clock cycles. For other rows, things are different. We choose the check operation of row 95 as an example. In the first clock cycle of an iteration, CPU-95 performs one step of row operation according to the first "1" element in row 95. Then the result is registered and transferred to CPU-97, owing to the chain of CPU group 3. CPU-97 is connected with VPU-3. In the second clock cycle, CPU-97 outputs a message to and receives back a message from VPU-3, according to the second "1" element of row 95... Similarly, the row operation result value is keeping shifting during the decoding process, according to the shifting matrices in the parity check matrix. Each row operation is performed by 47 CPUs in one group in 47 clock cycles, with each CPU finishing one step of it. The interconnections become localized thus make the routing complexity low.

IV. COMPLEXITY ANALYSIS
A. Storage Requirement Due to the effective appliance of the Min-Sum decoding
algorithm, a lot of message memory can be saved by our design. Here gives a quantitative comparison of the needed memory between the proposed architecture and the conventional designs. The memory bits needed by the traditional designs [6] [7] are 6x47x47x4 = 53016 bits (six bits for one message, 47x47x4 messages in total). In our design, the number is 47x47x4+17x47x4x2 = 15228 bits. It can be seen that more than 70% of memory is reduced. To the best of our knowledge, only the architecture in [5] and the proposed scheme can achieve this huge memory reduction. In addition, our architecture has significantly higher decoding speed than [5],

C. Decoding Speed The proposed architecture divides the check node operation
into 47 steps. Each step contains only one level comparator. Hence, the clock speed can be significantly increased. In traditional design, the check node processor will be an adder tree with 47 inputs. The decoding speed should be low.

V. RESULTS
The LDPC decoder is coded as a synthesizable Verilog HDL model. Results are obtained with the Altera Quartus 4.0 synthesis tool.
Table 1 gives the mapping result of our decoder on the Altera Cyclone EPIS30 device. The logic elements number used by CPUs and VPUs is 84*188+158*47=23218. The total usage includes some more control logic. The memory bits are composed of two parts: the intrinsic memory part (12288 bits) and the sign FIFO part (8836 bits). The maximum clock frequency is 50MHz. The throughput is 117.5Mbps (with 20 iteration times). Fig.6 shows the bit error rate (BER) performance of the decoder assuming binary transmission over the AWGN channel. Min-sum decoding algorithm performs better than BP because BP is not the optimized decoding algorithm. [8] We see that the implementation loss due to the
six bit quantization method is less than 0.2 dB at a BER of 10-7.
This decoder VLSI architecture can also be applied to some other kind of LDPC codes, such as Quasi-Cyclic codes. [11]

VI. CONCLUSIONS
In this paper, a memory efficient decoder architecture suited for the Min-Sum decoding algorithm for array code based LDPC codes is proposed. The approach discussed in this paper facilitates the applications of high rate LDPC codes in area/speed sensitive communication systems.
