Abstract— This paper proposes a memory-efficient accelerating schedule for LDPC decoder. Important properties of the proposed techniques are as follows: (i) Partitioning a pipelined operation not to read and write intermediate messages simultaneously enables the accelerated message-passing schedule to be implemented with single-port memories. (ii) FIFO-based buffering reduces the number of memory banks and words for the decoder based on the accelerated message-passing schedule. The proposed decoder reduces the memories for intermediate messages by half compared to the conventional one based on the accelerated message-passing schedule.
Keywords—Low-Density Parity-Check Codes, MessagePassing Algorithm, FPGA, FIFO buffer
I. INTRODUCTION
A Low-Density Parity-Check (LDPC) code was proposed by Gallager in the 1960’s, and it was re-discovered by MacKay and Neal in 1996. Inherent parallelism of a decoding algorithm called message-passing algorithm for LDPC codes makes it suitable for hardware implementations. The message-passing algorithm exchanges the intermediate messages between the check and bit nodes by performing the row and column operations iteratively. The schedule in Refs. [4], [5] performs the row and column operations concurrently using dual memory architecture (i.e. concurrent schedule)1. This architecture reduces a single iterative decoding delay, and makes the controller very simple. However, the messagepassing efficiency between the row and column operations is degraded, and the number of buffer words is double compared to the classical message-passing schedule2 as shown in Ref. [3]. The schedule in Refs. [6]–[7] performs the column operations first in association with the following row operations, and then the row operations are performed. This schedule improves the message-passing efficiency. However, the decoder in Refs. [6] approximates the column operations in order to reduce the column operations delay and the number of buffer words. This approximation degrades the decoding performance significantly. The decoder in Ref. [7] does not approximate the column operations. However, the decoder architecture partitions the memory into a large number of memory banks, and the number of buffer words

This paper proposes a memory-efficient LDPC decoder architecture based on the accelerated message-passing schedule which is shown in Ref. [8]. Important properties of the proposed techniques are as follows: (i) Partitioning a pipelined operation not to read and write messages simultaneously enables the accelerated message-passing schedule to be implemented with single-port memories. (ii) FIFO-based buffering reduces the number of memory words and accesses for the LDPC decoder based on the accelerated messagepassing schedule. The proposed LDPC decoder architecture keeps a single message for each non-zero bit in a parity check matrix as well as the classical schedule. Therefore, the proposed decoder reduces the memories for intermediate messages by half compared to the conventional one based on the accelerated message-passing schedule. Experimental results shows the proposed decoder achieves the best decoding performance, and is efficient for practical implementations.

III. MEMORY-EFFICIENT HARDWARE ARCHITECTURE
We propose a memory-efficient LDPC decoder based on the accelerated message-passing schedule.
A. Pipelined Operations based on Single-Port Memory
The decoder shown in Ref. [8] based on the accelerated message-passing schedule has two buffers for each of the messages α and β independently. The buffers are implemented with dual-port memories in order to read and write the messages simultaneously.
In this paper, we propose an efficient pipelined operation which enables the accelerated message-passing schedule to be implemented with single-port memory. The proposed pipelined operation avoids the simultaneous read and write accesses to the same memory bank. The proposed pipelined operation is shown in Fig. 2. The upper half of Fig. 2 shows the row operations and the lower half of that shows the column operations. The proposed pipelined row operation is composed of 7 stages. The row operation starts by reading message β and setting the messages in CFU registers. Six messages β(x, y1), · · · , β(x, y6) are fed to the CFU for decoding a (3,6)-regular LDPC codeword. In order to avoid reading and writing the message β at the same clock cycle, we partition the read and setting procedure into two stages. After that, the CFU computes absolute value |β| (stage 3), and determines the first and second minimum values (stage 4). The CFU compares the absolute values once in each cycle. After that, the CFU select one of the two (stage 5), and then add a sign (plus or minus) for each message α which is given to the column operations (stage 6). These two procedures (stage 5 and 6) are also performed in two stages in the proposed pipelined operation in order to avoid simultaneous read and write accesses.
The accelerated message-passing schedule performs the three column operations immediately after the row operations are performed. These row and column operations are performed in parallel. As shown in Fig. 1, three updated message α (i1(1,q), j1), α (i(22,q), j2), α (i(33,q), j3) are given to each column operation module, and these messages are stored in the registers of the column operation module. B. FIFO-based buffering for the accelerated message-
passing schedule
The schedule assures that the row and column operations are performed concurrently. However, the buffer size is double compared to the classical schedule since the decoder keeps messages α and β for every non-zero bit in a parity check matrix. A example of the sub-blocks in one column block is given in Fig. 3. The column block is composed of three sub-blocks. In this column block, the conventional decoder shown in Ref. [8] keeps eight messages α and β for each sub-block. Since the messages α and β have to be stored separately, six sets of a eight-word buffer are required for this one column block. At this point, we propose a FIFObuffering for the messages α and β considering read and write timings of them based on the accelerated messagepassing schedule in each sub-block. The read and write timings in one column block is shown in Fig. 3.
As shown in Fig. 2, read timings for the messages α by the column operations which are derived from x-th row operation occur during the next (x + 1-th) row operation. The read and write timings for the messages α are shown in Fig. 3 (a). Figure 3 (a) for the sub-block 1 shows that the message α which is written by the row operation is read out by the column operation j2 after a delay of six executive row operations, and it also read out by the column operation j3 after a delay of four executive row operations. Therefore, the messages α written by the row operation should be stored in a six-word FIFO buffer as shown in Fig. 3 (a)6. The messages α are shifted to the right word of the FIFO buffer at the timing when the message α is written by the executive row operation. The message α which is given to the column

In the proposed decoder, the FIFO buffers are implemented with shift registers for those whose number of words are less than 16. The other FIFO buffers are implemented with memory banks. Therefore, the proposed LDPC decoder has 18 memory banks for the messages α and β. The results show that the decoders based on the accelerated messagepassing schedule (Ref. [8] and the proposed one) achieve the best decoding performance. In addition, the number of memory banks and buffer words based on the proposed FIFO-based buffering is much less than that based on a concurrent schedule and the schedule shown in Refs. [7] and [8]. The number of memory banks and buffer words based on the classical schedule is almost same as the proposed FIFO-based buffering. However, in a classical schedule, all row operations are performed and then all column operations are also performed. Therefore, the single iterative decoding delay is double, and the decoding throughput decreases by half compared to that of the accelerated message-passing schedule. The number of memory banks and buffer words of the decoder shown in Ref. [6] is the least among the decoders since the decoder approximates the column operations. However, this approximation degrades the decoding performance significantly, and does not improve it effectively even though the maximum number of iterations increases.
We implemented the proposed decoder on an FPGA provided from Xilinx. Table II shows the comparison of the implementation results between the proposed decoder and the decoder shown in Ref. [8]. The throughput is estimated with the clock frequency of 100 [MHz] and the average number of iterations for decoding where SNR=4.5 [dB]. The memory banks are implemented with distributed block RAM in the FPGA device. Implementation results show that the proposed decoder reduces the FPGA resources for memories by 30 % where the memories for intermediate message α and β are reduced by about half compared to the conventional decoder based on the accelerated messagepassing schedule, as a result of which the proposed decoder architecture reduces the FPGA resources by 15 % compared to the conventional one. Experimental results shows the proposed decoder achieves the best decoding performance, and is efficient for practical implementations.
