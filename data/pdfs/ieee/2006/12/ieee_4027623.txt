Improvements in Belief-Propagation Decoding Based on Averaging Information from Decoder and
Correction of Clusters of Nodes
Nedeljko Varnica and Marc Fossorier

Abstract— In this letter we provide several methods to improve belief-propagation (BP) decoding of low-density parity-check codes. The methods use additional processing of the decoded information during the decoding process and are grouped into two categories. The first category is useful in the waterfall performance region and is based on processing averaged information obtained from the decoder. Several choices of the feedback information utilized in additional processing are introduced. The second category is useful in the error-floor performance region and is based on fixing clusters of symbols simultaneously, again utilizing the information obtained by monitoring the decoder. We demonstrate on a code example that our methods achieve sizeable gains compared to the original BP decoder.
Index Terms— Low-density parity-check codes, iterative decoding, belief propagation, augmented message-passing decoding.
I. INTRODUCTION
T HE success of belief propagation (BP) decoders for low-density parity-check (LDPC) is very well documented [1], [2], [3], [4]. Nevertheless, for many codes, it has been empirically observed that BP decoding still can perform far from the optimal maximum-likelihood (ML) decoding performance. For example, the gap between the BP and ML decoders of the well-known (155,64) Tanner code [5] on the additive white Gaussian noise (AWGN) channel is roughly 1dB, see [5], [6].
Recently, a class of algorithms that augment the BP decoder, and can significantly improve its performance if the number of decoding iterations is relatively large, has been proposed [6], [7]. These algorithms are especially effective on relatively short LDPC codes in the waterfall performance region of error rate plots. They also show improvements in the errorfloor region for both short and long LDPC codes. (Error floors are typically more pronounced for long LDPC codes.) It was demonstrated in [6] that this class of decoders can be successfully utilized on both memoryless and intersymbol interference Gaussian channels. In this letter, we are interested in enhancing the performance of this class of decoders and/or simplifying the processing overhead required by these decoders.

II. AUGMENTED BELIEF-PROPAGATION DECODER
Since we base our methods on the augmented BP decoder introduced in [6], [7] we briefly summarize it here. The following notation is assumed. In the LDPC (Tanner) graph G, the sets of variable nodes and check nodes are denoted by V and C, respectively. The length of the code is denoted by N = |V |. The messages passed in the decoding on this graph are log-likelihood ratios (LLRs). The decoder softoutput LLR on a variable node v ∈ V after the -th decoding iteration is denoted by m( )(v). Note that, according to this notation, the decoder input LLR on a variable node v ∈ V is m(0)(v). The LLRs corresponding to the highest likelihoods of bits “0” and “1” (and representing the saturation values in the decoder) are denoted by +S and −S, respectively. In the augmented decoder, a state of the decoder is monitored and used as an additional information which is processed and fed back to the decoder. We generally denote this state of the decoder after the -th decoding iteration by SD( ) = {SD( )(v0), SD( )(v1), · · · , SD( )(vN−1)}. For example, SD( ) = {|m( )(v0)|, |m( )(v1)|, · · · , |m( )(vN−1)|}.
