Abstract— The Reliability-Based Hybrid ARQ (RB-HARQ) scheme, which can be used with error correcting codes using soft-input soft-output (SISO) decoders such as convolutional codes and turbo codes, has been proposed. In the RB-HARQ scheme, the error rate performance is improved by selecting retransmission bits based on Log Likelihood Ratio (LLR) of each bit in the receiver. However, the receiver has to send the bit positions of retransmission bits to the transmitter. Therefore, the RB-HARQ scheme requires a large number of feedback bits. On the other hand, Low Density Parity Check (LDPC) codes are recently attracting a lot of interest, because LDPC codes can achieve near Shannon limit performance. In this paper, we evaluate the RB-HARQ scheme using LDPC codes. Moreover, we propose a RB-HARQ scheme that requires a fewer feedback bits by utilizing a code structure of LDPC code. We refer to the scheme as the RB-HARQ (row base) scheme. We show that the RB-HARQ and RB-HARQ (row base) schemes using LDPC codes have better error rate performance and higher throughput performance than no ARQ scheme. We also show that the RBHARQ (row base) scheme has a good trade-off between error rate performance and the number of feedback bits compared to the RB-HARQ scheme.
I. INTRODUCTION
Reliability-Based Hybrid ARQ (RB-HARQ) scheme that can be used with codes using soft-input soft-output (SISO) decoders such as convolutional codes and turbo codes has been proposed [1–3]. In the RB-HARQ scheme, a receiver decides the retransmission bits based on the log-likelihood ratios (LLRs) that are outputs of the SISO decoder. The RBHARQ scheme using the turbo code was shown to offer throughput close to capacity [1]. However, in the RB-HARQ scheme the receiver must feed back the indices of unreliable bits to the transmitter. Therefore, the number of feedback bits can potentially be quite large. For example, if the code length is approximately 1000 bits, then each bit position can be represented by a 10-bit index. If 100 bits are to be retransmitted, then the number of feedback bits will be 1000 bits if no source coding is applied. In [2] the RBHARQ scheme that exploits the time-correlation properties of convolutional codes to reduce the number of feedback bits was proposed. In [3] a source coding is applied for the feedback bits of RB-HARQ scheme.
In this paper, we evaluate the performance of RB-HARQ scheme using LDPC codes. Moreover, we propose a RBHARQ scheme that reduces the number of feedback bits of

RB-HARQ scheme by utilizing the code structure of LDPC codes. We refer to the scheme as the RB-HARQ (row base) scheme. The RB-HARQ scheme specifies the retransmission bits per each unreliable bit, while the RB-HARQ (row base) scheme significantly reduces the number of feedback bits by specifying the retransmission bits by each row that contains the unreliable bit. We show that the RB-HARQ and RB-HARQ (row base) schemes using LDPC codes achieve better error rate performance and higher throughput performances than no ARQ scheme with the same LDPC code on an AWGN channel. We show that the RB-HARQ (row base) scheme provides a good trade-off between the error rate performance and the number of feedback bits compared to the RB-HARQ scheme.
II. LOW DENSITY PARITY CHECK (LDPC) CODES
We assume an LDPC code with the parity check matrix H = [Hmn] that contains mostly zeros and only a small number of ones. The parity check matrix H has N columns and M rows. We describe an LDPC code by (N, j, k) LDPC code where N denotes the length of the codeword and j, k denote the column and row weights of the LDPC code, respectively. The code rate of the LDPC code is given by 1 − j/k = 1 − M/N .
III. RELIABILITY-BASED HYBRID ARQ (RB-HARQ) SCHEME
In the RB-HARQ scheme, a receiver decides the retransmission bits based on the LLRs that are outputs of the SISO decoder. For each error packet, the bits are ranked by the magnitude of decoder outputs (|zn|). The bit with the smallest |zn| is considered to be least reliable, denoted by “reliability (0)”, and the bit with the largest |zn| is considered to be most reliable, denoted by “reliability (N − 1)”. If the decoding fails, the receiver feeds back the indices of the bits with the reliability (0) ∼ (l − 1) to the transmitter where l denotes the number of retransmission bits. The transmitter then retransmits the codes bits corresponding to those requested information bits. Retransmitted codes bits corrupted by noise are received. When a new copy of the same coded bits (either information or parities) are received, old copies are not discarded, but are combined with the new ones to facilitate decoding. IV. RB-HARQ (ROW BASE) SCHEME
The RB-HARQ (row base) scheme can reduce the number of feedback bits of RB-HARQ scheme by utilizing the code structure of LDPC codes. The RB-HARQ scheme specifies the retransmission bits per each unreliable bit, while RB-HARQ (row base) scheme specifies the retransmission bits by each row of a parity check matrix that contains the unreliable bit. RB-HARQ (row base) scheme can specify the unreliable bit and (k −1) bits together by selecting a row of the parity check matrix, which leads to the reduction of the number of feedback bits. V. SIMULATION RESULTS
In our simulation, we use (1792, 3, 7) and (1057, 3, 13) LDPC codes with code rate 4/7 and 10/13, respectively. We assume BPSK modulation and an AWGN channel. We also assume that the feedback channel is ideal. For the LLRBP algorithm, the number of maximum iterations is set to 500. Each retransmission effectively reduces the code rate and hence increases the Eb/N0 at the receiver. We take this additional received energy into consideration by defining the effective Eb/N0 as the average Eb/N0 at the receiver.
Figs. 1 and 2 show the BERs and PERs of the RB-HARQ and RB-HARQ (row base) schemes using (1792, 3, 7) LDPC code, respectively, where l = 100. Fig. 1 also shows the BERs of no ARQ scheme with the (1792, 3, 7) LDPC code and the Type III HARQ scheme using (1792, 3, 7) LDPC code for comparison. The Type III HARQ scheme is an incremental redundancy coding scheme in the form of repetition code. The inside of the parenthesis (“max”) represents the maximum number of retransmission. We can see that the performance of the Type III HARQ scheme using LDPC codes is considerably degraded compared to that of no ARQ scheme with the same LDPC code. This is because the retransmission in the Type III HARQ scheme increases the effective Eb/N0 largely, although it improves the error rate performance. We can also see that the RB-HARQ and RB-HARQ (row base) schemes using LDPC codes achieve good BER compared to no ARQ scheme with the same LDPC code. In particular, when the number of retransmission is two, the performance of RB-HARQ scheme is about 0.8 dB better than that of no ARQ scheme with the same LDPC code and the performance of RB-HARQ (row base) scheme is about 0.5 dB better than that of no ARQ scheme with the same LDPC code at the BER = 10−4 on an AWGN channel, respectively. We can also see that the RBHARQ and RB-HARQ (row base) schemes using LDPC codes can achieve good PER compared to no ARQ scheme with the same LDPC code. When the number of retransmission is two, the performance of RB-HARQ scheme is about 0.6 dB better than that of no ARQ scheme with the same LDPC code and the performance of RB-HARQ (row base) scheme is about 0.4 dB better than that of no ARQ scheme with the same LDPC code at the PER = 10−2 on an AWGN channel, respectively.

VI. CONCLUSIONS
In this paper, we evaluated the performance of the RBHARQ scheme using LDPC codes. Moreover, we proposed the RB-HARQ (row base) scheme that utilizes the code structure of the LDPC codes to reduce the number of feedback bits. The RB-HARQ scheme specifies the retransmission bits by each unreliable bit, while the RB-HARQ (row base) scheme specifies the retransmission bits by each row that contains the unreliable bit. We showed that the performances of the RBHARQ and RB-HARQ (row base) schemes are better than that of no ARQ scheme with the same LDPC code. We also showed that the RB-HARQ (row base) scheme has a good trade-off between error rate performance and the number of feedback bits compared to the RB-HARQ scheme. Moreover, we showed that the RB-HARQ (row base) scheme with the high rate regular LDPC code can reduce the number of feedback bits compared to the RB-HARQ (row base) scheme with the low rate regular LDPC code. We showed that the RB-HARQ and RB-HARQ (row base) schemes using LDPC codes have a higher throughput than no ARQ scheme.
