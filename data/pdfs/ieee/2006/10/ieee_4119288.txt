Abstract — By employing a class of q-ary cyclic codes, i.e., Berlekamp-Justesen (B-J) codes, as base codes, we obtain two classes of structured LDPC codes by q-tuple and (q − 1)-tuple substitutions. These constructions can be viewed as generalizations of Reed-Solomon based LDPC codes proposed by Djurdjevic et. al. The B-J based LDPC codes, which could be regular or irregular according to the adjustment of three parameters, have much flexibility in choices of the code length, rate and minimum/stopping distance. Furthermore, the minimum distances and stopping distances of the B-J based LDPC codes are analyzed. We show that the stopping distance of a B-J based LDPC code is not smaller than the best known lower bound of its minimum distance. The B-J based LDPC codes perform well under iterative decoding and manifest low error-floors in simulations which could be explained in one respect by their large minimum distances and stopping distances.
I. Introduction
For the constructions of structured low-density paritycheck (LDPC) codes, it is desirable if we could choose arbitrarily the code length and rate. For a good structured LDPC code with fixed parameters, puncturing its parity-check matrix are usual skills to enhance the flexibility in choices of code length and rate. During the process of puncturing, it is important not to degrade the performance much. In order to understand the performance of an LDPC code under iterative decoding over a binary erasure channel (BEC), it is crucial to know certain combinatorial structures, called stopping set, of the parity-check matrix of the LDPC code [6][7]. The stopping distance, i.e., the smallest size of a nonempty stopping set, is an important performance indicator for iterative decoding over the BEC, alike the minimum distance for maximum-likelihood decoding [6][7]. Hence, it is interesting to investigate the stopping distance properties when puncturing a parity-check matrix.
Let C be a binary linear code with minimum distance d and parity-check matrix H. Let GH denote the Tanner graph corresponding to H. The girth of GH or H is the length of shortest circle in GH. If the rows of H have constant weight ρ and the columns of H have constant

weight γ, then H is said to be (γ, ρ)-regular and C is then called a regular LDPC code. Otherwise, C is said to be irregular. A stopping set is a set of column indices of H such that the submatrix formed by the corresponding columns of H does not contain a row of weight one. In other words, a stopping set S is a subset of the variable nodes in the GH such that all the neighbors of S are connected to S at least twice. The smallest size of a nonempty stopping set, denoted by s(H), is called the stopping distance of H. Unlike the minimum distance d of C, the stopping distance s(H) depends on the specific choice of parity-check matrix of C. Many experimental results show that the error-floor of an LDPC code with iterative decoding depends on not only the minimum distance d but also the stopping distance s(H). Hence, it is necessary to keep both d and s(H) reasonably large.
In [1], Reed-Solomon (R-S) based LDPC codes were constructed. A (2048,1723) R-S based LDPC code is actually the IEEE 802.3 standard code [1][3]. In section II and III of this paper, we employ another class of cyclic maximum-distance-separable (MDS) codes, i.e., Berlekamp-Justesen (B-J) codes, as base code and obtain two new classes of LDPC codes. These constructions can be viewed as generalizations of R-S based LDPC codes in [1]-[3]. The B-J based LDPC codes have much flexibility in choices of the code length, rate and minimum/stopping distance. Note that lack of flexibility in the choice of the length and rate is often believed a drawback of geometry based LDPC codes [10]. The B-J based LDPC codes could be regular or irregular according to the adjustment of three parameters γ, ρ and u. Furthermore, the minimum distances and stopping distances of the B-J based LDPC codes are analyzed in section IV. We show in theory that the stopping distances of a B-J based LDPC code is not smaller than the best known lower bound of its minimum distances, and these results on stopping distance are also suitable for the LDPC codes constructed by blocks of permutation matrix (see [4][5] and their references). Finally, several examples and some remarks are given respectively in section V and VI.

V. Simulation Results
In this section, we present several examples of B-J based LDPC codes and show their bit-error performance (BER) over AWGN channel, in which the sum-product decoding algorithm are employed.
Example 1 Let q = 25 and consider a [33, 2, 32] B-J code over GF(25). By q-tuple substitution, we obtain the first class of B-J LDPC codes. Set γ = ρ = u = 32. Then H(1)(32, 32, 32) is regular and has column weight 32 and row weight 33. C(1)(32, 32, 32) gives a regular (1056, 813) LDPC code with rate 0.7699 and minimum/stopping distance at least 33. At the BER of 10−5, the code performs 1.73 dB from the Shannon limit (see Figure 1). For comparison, we also calculate the BER of MayKay’s random LDPC code with the same length and rate. We can see that the two codes have almost the same performance at the BER of 10−5, but MacKay’s code has error floor at the BER of 10−6.
Set γ = 6, ρ = 32 and u = 6. Then H(1)(6, 32, 6) is irregular, and each row has the same weight 33, and the previous 6 columns have weight 32 and the last 1024 columns have weight 6. C(1)(6, 32, 6) gives an irregular (1030, 883) code with rate 0.8573 and minimum/stopping distance at least 8. From Fig.1, we see that at the BER of 10−5, C(1)(6, 32, 6) performs 1.47 dB from the Shannon limit, 0.3 dB better than Mackay’s code with same parameters.

VI. Conclusion In this paper, we employ a class of cyclic MDS codes, i.e., B-J codes with dimension 2, as base code and obtain two new classes of LDPC codes by using q-tuples and (q − 1)-tuples to substitute the elements of GF(q). For the first class of B-J based LDPC code C(1)(γ, ρ, u), the stopping distance is not smaller than γ + 1 for odd γ and γ + 2 for even γ. For the second class of B-J based LDPC code C(2)(γ, ρ, u), the stopping distance is not smaller than γ for odd γ and γ + 1 for even γ. C(1)(γ, ρ, u) and C(2)(γ, ρ, u) could be regular or irregular according to the adjustment of three parameters γ, ρ and u, and have much flexibility in choices of the code length and rate. Moreover, from Example 3, it can be found an interesting observation that the puncturing of B-J based parity-check matrix often does not remarkably degrade the performance of the corresponding LDPC codes while increasing the code rate.
