ABSTRACT

1 INTRODUCTION

Regular LDPC codes are a special class of lowdensity codes having an equal number of ones in each row and column of the parity check matrix describing the linear code. The uniform structure of regular LDPC codes allows a practical hardware implemen-
tation which can efficiently utilize the inherent paral-
lelism of the message passing algorithm (MPA) commonly used to decode low-density codes. The class of
{$3, 6} LDPC codes has been extensively studied and
they have been proven to provide very good error performance, especially at lower SNR. {4, 8} codes have not been analyzed nearly as much in the literature, mainly because their 'threshold,' the SNR where the waterfall region of the error performance curve begins, is typically a quarter of a dB or so worse than for comparable-length {$3, 6} codes. It has been proposed that {4, 8} codes have better high SNR behav-
ior, but until recently it was not possible to verify this conjecture. A new technique which can efficientlyfind error floors of LDPC codes now has the ability to il-
luminate just how good {4, 8} codes are in the high SNR region - a result which is of great interest for many practical applications. This paper will analyze the error floor characteristics of some {4, 8} codes and provide a simple algorithm for designing {4, 8} codes with low error floors. A newly-designed rate1/2 (1200,600) {4, 8} code with a vastly superior error floor compared to codes of similar parameters is introduced.
Keywords: LDPC, error floors, {4, 8} regular codes.
*This work is supported by L-3 Communications.

In the study of large block-length codes, it is now common knowledge that a tradeoff exists between achieving a low threshold where the code enters the 'waterfall' region and also achieving a low error floor in the higher SNR region. For turbo codes [1], a parallel concatenated structure leads to a lower threshold than serial concatenation, but also offers a higher (less desirable) error floor. In LDPC codes, irregular code designs exist which push the threshold very close to capacity, but sacrifice performance in the higher SNR region. For regular codes, meaning the same number of edges coming from each variable node, dv, and check node, dc in the Tanner graph, the decoding
threshold gets worse as d, increases above three. For
example, {3, 6} codes, meaning 3 edges coming from each variable node and 6 edges from each check node, are known to provide a lower threshold than {4, 8} codes. This paper will show that the {4, 8} codes have exchanged their degradation in the waterfall region with a much more desirable error floor. An implementation advantage of regular codes is the uniformity of processing required for each message in the message passing algorithm (MPA), which is the typical decoder used for low-density codes. The encoding procedure for arbitrary low-density codes is not quite as simple as for certain classes of LDPC codes such as repeataccumulate [2] codes, which are known to be lineartime encodable. However, there are straightforward encoding methods for arbitrary low-density codes [3] which can achieve almost linear-time complexity, a great improvement upon the naive method of computing uG, where G is a dense generator matrix and u is the k-length message to encode; the naive method requires a quadratic encoding complexity. This important result implies that a low-density H matrix can

This paper is organized as follows: Section 2 introduces some terms and concepts necessary to understand the decoding algorithm of LDPC codes and what causes their error floors. The final section summarizes the contribution offered in this paper.
2 LOCAL GRAPH STRUCTURE AND
TRAPPING SETS
LDPC codes, the revolutionary form of block coding that allows large block-length codes to be practically decoded with near-ML performance, were first presented by Gallager in the early sixties [5]. These codes have sparse parity check matrices, denoted by H, and can be conveniently represented by a Tanner graph [6], where each row of H is associated with a check node ci, each column of H is associated with
a variable node vj, and each position where Hij = 1
defines an edge connecting vi to ci in the graph. If a subset of variable nodes and check nodes is considered, members of this subset will be called the active nodes of a subgraph.
The iterative message passing algorithm (MPA), also referred to as Belief Propagation when using fullprecision soft data in the messages, is the method commonly used to decode LDPC codes. This algorithm passes messages between variable and check nodes, representing the probability that the variable nodes are '1' or 'O' and whether the check nodes are satisfied. The following equations, representing the calculations at the two types of nodes, will be considered in the log likelihood ratio (LLR) domain with notation taken from [7].

As with all linear codes, it is convenient to assume the all-zeros codeword as a reference vector for study. If a subset of variable nodes and check nodes is considered, members of this subset will be called the active nodes of a subgraph. By studying the local subgraph structure around each variable node, arguments can be made about the global graph structure. For example, bounds can be placed on the minimum distance of a code by only considering the local structure of a code [6]. Small subsets of non-zero bits which do not form a valid codeword but still cause the MPA decoder problems are well-documented in the literature and typically referred to as trapping sets (TS)

A TS, x, is a length-n bit vector denoted by a pair (a, b), where a is the Hamming weight of the bit vector and b is the number of unsatisfied checks, i.e. the Hamming weight of the syndrome xHT. Alternatively, from a Tanner graph perspective, a TS could be defined as the a nonzero variable nodes of x and all of the check nodes connected by one edge to those a variable nodes. A valid codeword is a TS with b = 0.
The essential intuition pointing to TS as problematic for the decoder is that if a bits have sufficient noise to cause them to individually appear as soft l's, while the others in the subgraph are soft 0's, the check nodes with which the a variable nodes connect will be satisfied, tending to reinforce the wrong state to the rest of the graph. Only the unsatisfied check(s) is a route through which messages come to reverse the apparent (incorrect) state. It is mainly the improved trapping set structure of the higher density {4, 8} codes which gives them an error floor advantage over {3, 6} codes.

3 {4, 8} CODE PROPERTIES

First, let's look at some properties of {4, 8} codes.

Theorem 1. {4, 8} codes have no (a, b) TS with odd b.

Proof This property can be seen by adding columns

of H modulo two. The total number of ones (edges

in a subgraph) in a group of x columns will always be

even since 4x is even for all x. These 4x edges must

be connected among check nodes in a subgraph of the

entire Tanner graph to form a TS. To satisfy a check,

an even number of edges must be connected to it. Af-

ter all satisfied checks have been accounted for, the re-

maining edges must belong to unsatisfied checks. Un-

satisfied checks use an odd number of edges. Since

there are an even number of edges left (an even num-

ber - another even number is always even), then the

number of unsatisfied checks must be even, since only

the sum of an even number of odd numbers will pro-

duce an even number. Thus no (a, b) TS can exist for

odd b.

Theorem 1 is useful because (a, 1) TS can create poor error floors for some {3, 6} codes. For example, most of the (96,48) {3, 6} codes on MacKay's website [9] have an error floor dominated by (5, 1) TS. The (504,252) PEG code [10] commonly studied in the literature has a dominating (9, 1) TS. There are many examples of {3, 6} codes which have (a, 1) TS with a small a and these typically create the error floor.
It has been noted that increasing the girth of a lowdensity code increases the lower bound on the code's minimum distance [6]. This increase in minimum distance is a function of the variable node degree, and for a given girth, a larger variable node degree will ensure a larger minimum distance. Similar arguments can be made concerning minimum-weight TS for a given girth [11] and variable node degree distribution. So, for example, if two (1200,600) codes each have a girth of eight, then over the ensemble average, the {4, 8} codes will have a lower error floor than the
{3, 6} codes.
An experiment which helps explain the lower error
floors of {4, 8} codes under MPA decoding is to take

This section presents a simple random girthconstrained LDPC H construction technique similar to those in [12, 10], which attempt to maximize girth when designing H. The procedure outlined below also adds edges to the graph with the goal of attaining a large girth, but in addition, the connection of edges in the graph is done probabilistically, governed by a rule to meet the regular check degree.

The maximum girth achievable is a function of the degree profile, n, and k. If a chosen girth constraint causes the above process to run out of available check nodes while many edges are still left to deploy, it is necessary to relax the girth constraint. For example, in the case of {3, 6} codes, this method allows the construction of a girth-8 code with n = 204, and a girth-10 code with n = 1400. If the code designer has access to a large pool of processing power, then it would be possible to search a vast number of H constructions, and these values of n could potentially be lowered.

5 RESULTS
Using the modified girth-constrained code construction procedure of Section 4, a (504,252) {4, 8} code was created with girth-8 until edges for the 355th variable node were added. From this node on, some of the added edges would produce 6-cycles, but the number of 6-cycles was cut down to under 1200, from an original value of about 5000 when only the girth of 6 constraint (unmodified construction method) was enforced. The code produced from this method has a single dominant (10,4) TS which produces the error floor. This code was better than the non-constrained girth-6 (504,252) code which has (5,4), (6,4), (7,4)

6 CONCLUSIONS
The work presented here demonstrates the utility of using regular {4, 8} LDPC codes for applications that must have a guaranteed very low error rate at a given SNR. A unique method to efficiently construct these codes is also presented and their performance is simulated, using a new 3-step IS procedure, down to block error rates previously out of reach by a traditional
7 of 7

Monte Carlo simulation approach.
