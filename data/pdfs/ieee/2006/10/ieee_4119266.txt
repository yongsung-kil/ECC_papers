Improvement of Shuffled Iterative Decoding

Zhao chuan-gang', Yuan jin-sheng', Lin xue-hong2, Lin jia-ru2

Abstract An improved version of shuffled belief propagation of iterative decoding of LDPC codes is proposed. The proposed scheme can speed up the convergence of iterative decoding. Simulation results verified our conclusions.
Index Terms Low-density parity-check (LDPC) code, Belief Propagation (BP) decoding, shuffled belief propagation (BP) decoding
I. INTRODUCTION
L ow-density parity-check codes was invented by Gallager [1] in 1962, but did not receive much more attention until
they were rediscovered independently by Mackey et al. [2]. Later, Luby [3] introduced the irregular LDPC codes. Because of its outstanding performance, LDPC codes have been put in the center of attention in the field of capacity-approaching codes.
Standard belief propagation (BP) decoder of LDPC codes often needs several tens or hundreds of iterations for the decoding process to convergence, which is not desirable because of high decoding delay. Furthermore, LDPC codes of interest can have large codeword length and it can be difficult to implement the decoding in hardware in a fully parallel way. In [4, 5, 6], shuffled BP decoding method was presented to reduce the required number of iterations for decoding LDPC. In this paper, an improved version of shuffled BP algorithm is introduced to accelerate the decoding process for LDPC codes.
II. ITERATIVE DECODING OF LDPC CODES
A binary (N,K) LDPC code [1, 2] is a linear block code
described by a sparse M x N parity-check matrix H, i.e., H has a low density of ls. The parity-check matrix H can be viewed as a bipartite graph with two kinds ofnodes: N symbol nodes corresponding to the encoded bits and M parity-check nodes corresponding to parity-check nodes represented by the rows of matrix H. The connectivity of the bipartite graph is such that the parity-check matrix H is its incidence matrix.

C. Improvement ofshuffled BPfor iterative decoding
Note that while one iteration of the standard BP algorithm is fully processed in parallel, that of shuffled BP algorithm becomes totally serial. The virtue of shuffled BP algorithm is that in one iteration, once one symbol node is processed, the more reliable messages based on the symbol node can be used for next symbol node, i.e., more reliable messages can be used as soon as possible, thus shuffled BP algorithm converges faster than the standard BP algorithm. In shuffled BP algorithm, the updated messages from check node m is
just used to update the bit-to-check messages of the current bit n. In order to speed up the convergence of shuffled BP algorithm, we propose an improved version of shuffled BP algorithm, in which the updated messages from check node

m is not only used to update the bit-to-check messages of node n, but also used to update messages of other symbol
node n adjacent to check node m. The following is the improved version of shuffled BP algorithm.

IV. CONCLUSIONS
In this paper, shuffled BP decoding of LDPC codes is studied. In one iteration of shuffled BP decoding, the horizontal process just compute the LLR messages sent to one symbol node, and the vertical process just compute the LLR messages sent from one symbol node, while our scheme compute all LLR messages sent to all symbol nodes adjacent the check node in horizontal process and all LLR messages sent from all the symbol nodes in vertical process. Simulations show our scheme converges faster than shuffled BP decoding algorithm.
