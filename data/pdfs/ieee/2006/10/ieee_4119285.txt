Abstract ‚ÄĒ In this paper, we propose an improved graph-based Chase-MP algorithm for iterative decoding of generalized low-density (GLD) codes, and also present analysis of the iterative decoding by using the Gaussian approximation. The simulation results on the additive white Gaussian noise (AWGN) channel show that the proposed algorithm can achieve good performance. During the iterative decoding analysis, we get a dynamic mathematical model of the Gaussian mean on iterations. As an application of the model derived, we compute decoding thresholds for several extended Bose-Chaudhuri-Hocquengnem (eBCH)-component GLD codes.
I. Introduction
Generalized low-density (GLD) codes refer to a class of pseudo-random error-correcting codes by generalizing Gallager‚Äôs construction of low-density parity-check (LDPC) codes. The concept of GLD codes was first introduced explicitly by Tanner [12], and was further developed by Boutros et al. in [2], where an iterative softinput/soft-output (SISO) maximum a posteriori (MAP) algorithm based on syndrome trellis (hereinafter, we call it trellis-MAP algorithm) was used for decoding of each component code. During the latest years, researches on Hamming-component GLD codes have been further made [6],[8]. Analysis similar to Gallager‚Äôs shows that GLD codes are asymptotically good and can achieve the Shannon capacity limit [6]. Due to their excellent decoding performance and high flexibility on the selection of code lengths and component codes, GLD codes turn out to be promising candidates for error correction in digital TV broadcasting and deep-space communications.
By using the turbo principle, GLD codes can be effectively decoded based on iterative SISO decoding of their individual component codes. Therefore, decoding performance and decoding complexity of a certain GLD code are heavily dependent on its employed SISO decoding algorithm. Among various decoding algorithms used for GLD codes [2],[6],[8],[9], the low-complexity one based on graphs has not been presented explicitly, and the convergence properties for decoding of GLD codes have not been researched, yet. Although, the decoding principle proposed by Lentmaier and Zigangirov [8] is similarly based on bipartite graphs, it is worth mentioning that the a posteriori probability (APP) algorithm used in [8] is of

high complexity. In this paper, we propose an improved graph-based Chase-MP algorithm for iterative decoding of GLD codes, and also present analysis of the iterative decoding using the Gaussian approximation (GA).
This paper is organized as follows. Section II shows how to construct GLD codes from component linear codes. In Section III, we investigate how to design the Chase-MP algorithm for decoding of GLD codes, and provide several simulations on the additive white Gaussian noise (AWGN) channel. Section IV is devoted to the analysis of iterative decoding with the proposed algorithm under the Gaussian approximation. In Section V, we show an application of the analysis for computing decoding thresholds. Section VI concludes the paper.

II. Construction of GLD Codes
A length-N binary GLD code is specified by its paritycheck matrix, which can be constructed from an (Nc ‚ąí Kc) √ó Nc matrix Hc that is the parity-check matrix of its component linear codes, and an (N ¬∑ J/Nc) √ó N lowdensity (LD) matrix A with row weight Nc and column weight J. The construction rule [6],[8] is that in every row of this matrix A we replace the (N ‚ąí Nc) zero elements by all-zero column vectors of length (Nc ‚ąí Kc), and the n nonzero elements by the Nc different columns of Hc.
If the resulting parity-check matrix of the constructed GLD code has a full rank, it is obvious that the rate of
the GLD code is defined as

R = 1 ‚ąí J ¬∑ (Nc ‚ąí Kc)/Nc.

(1)

In case of a smaller rank, it is evident that the rate is increased accordingly. However, as indicated in [8], with an increasing block length N a small number of linear dependent rows has only a minor effect on the rate, and equation (1) gives a good approximation of the actual rate of the GLD code.
We call the codes (N, J, Nc)Kc GLD codes, which can be constructed in the described way. As already stated in [8], a higher number J of component codes raises the dependence during the iterations and can lead to worse decoding performance in spite of the better distance properties. Therefore, in this paper we only consider the case of (Nc2, 2, Nc)Kc GLD codes with length-Nc Bose-Chaudhuri-Hocquengnem (BCH) or extended BCH (eBCH) codes used as the component codes.

A. How to Design Chase-MP Algorithm
With the LD matrix of any GLD code, we can associate a graph referred to as its bipartite graph [12] (an example of the graph can be found in [6]). In the LD matrix, each row corresponds to a constraint node, and each column corresponds to a symbol node. For the LD matrix A described in Section II, its bipartite graph is regular with degree ds = J of each symbol node and degree dc = Nc of each constraint node. Similar to designing the sparse matrix of an LDPC code, it is necessary to design A satisfying that its graph has large girth (the shortest length of cycles in the graph) in order to achieve better iterative performance. An instance of an LD matrix with girth 8 (J = 2) is given in (2), and one of its length-8 cycles is indicated by solid lines (see the top of the page).
Combining the Chase-II algorithm [3],[10] for linear block codes and the message-passing (MP) algorithm [11] for LDPC codes, we plan to design an improved algorithm named Chase-MP algorithm for decoding of GLD codes. This algorithm takes advantage of the iterative feedback decoding principles, and delivers extrinsic information mutually based on the bipartite graph of the LD matrix. In the graph, each constraint node calculates the extrinsic information for every symbol node connected to it by applying the Chase-II algorithm, and the MP algorithm based on continuous message alphabets is used during the entire iterations.
Generally, let A be an M √ó N matrix with its elements denoted by Am,n, for m = 1, 2, ¬∑ ¬∑ ¬∑ , M and n = 1, 2, ¬∑ ¬∑ ¬∑ , N . Moreover, let the row and column index sets of the LD matrix A be Mn = {m : Am,n = 1} and Nm = {n : Am,n = 1}, respectively. Let ¬Ķm (l),n denote the message from the constraint node m to the connected symbol node n at the l-th (l ‚Č• 0) iteration. Also, let őĽn(l) denote the a posteriori log-likelihood ratio (LLR) value associated with the symbol node n after l iterations; whereas, the message from the node n to its neighboring constraint node m is written as őĽn(l,)m (l ‚Č• 0). B. Simulation Performance with Comparisons to That of Trellis-MAP Iterative Decoding
In order to show the validity of the proposed Chase-MP algorithm, we use this algorithm to conduct experimental simulations for several BCH/eBCH-component GLD

codes on the BPSK modulated AWGN channel, and also make comparisons of bit error rate (BER) performance to that of the trellis-MAP iterative decoding. The GLD codes considered here are (642, 2, 64)57, (1282, 2, 128)120, and (632, 2, 63)57 codes. Each GLD code has an LD matrix in the form of (2). Their simulation results are shown in Fig.1 to Fig.4, respectively.
The (642, 2, 64)57 GLD code with (64, 57, 4) eBCH code as its component has length 4096, rate 0.7932, and minimum distance 16, and the (1282, 2, 128)120 GLD code with (128, 120, 4) eBCH code as its component has length 16384, rate 0.8789, and minimum distance also 16. For the Chase decoding in Chase-MP algorithm, as suggested in [1], we set its weight factor to be 1/2, and reliability factor to be 1 for every iteration. The number of least reliable positions (LRPs) for the Chase-II algorithm is respectively selected as p = 3, 4, 5. These simulation results are plotted in Fig.1 to Fig.3. In Fig.2 and Fig.3, the Shannon limits (SL) are also included for comparisons.

VI. Conclusion
We proposed an improved Chase-MP algorithm for decoding of GLD codes. The simulation results on the AWGN channel show that our algorithm can achieve good performance. In comparison with the iterative trellisMAP decoding algorithm for GLD codes, the Chase-MP algorithm is quite efficient. We presented analysis of the Chase-MP iterative decoding by using the Gaussian approximation and also computed decoding thresholds of several eBCH-component GLD codes.
