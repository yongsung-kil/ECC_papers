Generation of Rate-Compatible LDPC Codes Based
on Linear Transformation

Beijing University of Posts and Telecommunications P. 0. Box 93, Beijing, 100876, China
ningxia.chenggmail.com, yychanggbupt.edu.cn, yangdcgbupt.edu.cn

Abstract Practical communication systems require coding rates to be compatible. This report presents a new method to design rate-compatible low-density parity-check (LDPC) codes. The method is based on linear transformation and different from the existing methods. Decomposing rows of the parity-check matrices can produce the parity-check matrices for lower rates. It can also maintain low error floors for the codes at all rates, and the codes can be decoded conveniently and efficiently. It can achieve high throughput in HARQ with FEC system.
Keywords- LDPC codes, bipartite graph, rate-compatible, extending, degree distribution, Gaussian Approximation (GA), Density Evolution (DE), HARQ
I. INTRODUCTION
Rate-compatible codes are a family of codes where the codeword bits from higher rate codes are embedded in the lower rate codes. They can be encoded and decoded conveniently by using a common hardware implementation pair. Generating a strong rate-compatible code is important for raising channel utilization efficiency. And it is also important to achieve high throughput in hybrid automatic repeat request (HARQ) with forward error correction system. When the channel condition is good, the system could choose higher rate codes to transmit more data, otherwise the system could choose lower rate codes to ensure correct transmission. To achieve this target with LDPC codes we could generate ratecompatible codes by puncturing and extending techniques, as proposed in [1], [2] and [3]. However, it is so complicated to generate new rate codes in [1]-[3] that the encoder/decoder would have to be modified significantly.
In this report, we propose a new method to generate ratecompatible LDPC codes. This method can generate paritycheck matrices for lower rates just by linearly transforming the mother matrices. Linear transformation described above means linear row decomposition. Simulation results show that rate-compatible codes generated by this method achieve steeply falling error rate curves without detectable error floors. The analysis of code performance by using Gaussian Approximation reveals that linear row decomposition can increase the mean of message of the check nodes and improve code performance.

II. LINEAR ROW DECOMPOSITION
The basic idea of linear row decomposition to reduce the code rate is to add the number of rows in the mother paritycheck matrix. In bipartite graph, rows of parity-check matrix correspond to check nodes. Adding the number of rows by linear decomposition is equivalent to increasing the number of check nodes and check bits. The blocklength is constant, so the number of information bits is reduced and accordingly the rate becomes lower.
Linear decomposition increases the number of rows by means of replacing a single row with a group of rows which is decomposed from the chosen one. In bipartite graph a check node is replaced with a group of nodes split from the node.

Changing only the number of rows can achieve various rates. From the point of view of the bipartite graph, linear decomposition can change the rate without changing the variable degree distribution. It means that a single variable degree distribution can be used to describe the mother code and all extended codes.
However, the check degree distributions of the extended matrices are quite different from those of the mother matrix. Comparing the bipartite graphs of the mother code with the extended codes, we find that edges connecting variable nodes and check nodes are unchanged, and the increase in the number of check nodes is based on desired rates. The structure of the decoder for LDPC codes is based on the bipartite graph. After row decomposition, the variable degree distribution is

unchanged. So for the decoder, its basic architectures are not changed a lot in the decomposition.
Obviously as long as the mother matrix hasn't short cycles, the decomposition way described here keeps the extended matrices from short cycles. And the performance of the extended codes is independent of the positions of rows when the short cycles have been avoided.
On the other hand, we can also get flexible rate-compatible codes by using the linear column combination. The number of combined columns depends on the desired rate. However,
column combination makes the blocklength shorter and shorter, and it goes against the good performance of LDPC codes. We
prefer the partial row decomposition to column combination.

IV. SIMULATION RESULTS
In the simulation of the complete row decomposition, the mother code generated by Mackay [8] is a rate-0.87 LDPC code. The blocklength of the mother and extended codes is 495. The size of its parity check matrix is 62*495, and the matrix has one 4-cycle. The error performance of the proposed codes for AWGN channels are shown in Figs.3.
In order to comparing the column combination and the partial row decomposition, we use the same mother code to simulate these two proposed methods. This mother code is a rate-0.5 LDPC code generated by Mackay [8] and its blocklength is 1024. Fig.4 shows the simulation results of the two proposed methods.
In Fig.4, dashed lines are the performance curves of extended codes generated by the partial row decomposition and solid lines are by the column combination.
For all simulation results, the maximum number of iterations for the decoder is fixed to 100 and BP algorithm is used.
In Fig.3, we can see that all the extended codes have low error floors as long as the mother code has a low error floor. It is feasible to obtain extended codes with good performance by choosing a good performance mother code.
In Fig.4, from the solid lines we see that the performance improvement is 0.3dB from rate-0.5 to 0.4, but the performance of lower rate codes can't be improved much. However, the dashed lines exhibit the excellent performance of partial row decomposition. For the same mother matrix and the same rates, partial row decomposition can increase the SNR by 2.5 dB comparing with column combination.

V. CONCLUSIONS
We propose a new method to generate rate-compatible LDPC codes. It has low-complexity decoder for it only changes the variable degree distribution. In conclusion, partial row decomposition is a good choice to generate rate-compatible LDPC codes for it maintains near-capacity perfornance and meets the request of flexible rates. Furthermore, combining partial row decomposition with the method proposed in [7] can achieve almost arbitrary rates.
