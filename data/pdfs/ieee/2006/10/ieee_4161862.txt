Low Complexity Implementations of Sum-Product Algorithm for Decoding Low-Density Parity-Check
Codes
Daesun Oh and Keshab K. Parhi Department of Electrical and Computer Engineering University of Minnesota, Minneapolis, MN 55455 USA
Email: {dsoh, parhi}@ece.umn.edu

Abstract-In this paper, we propose low complexity implementations of sum-product algorithm (SPA) for decoding low-density parity-check (LDPC) codes. The finite precision implementations for SPA have an important tradeoff between the decoding performance and hardware complexity caused by two dominant area consuming factors. One is the memory for updated messages storage and the other is the look-up
table (LUT) for implementation of nonlinear function 1](x). The proposed two types of variable quantization schemes result in significant reductions of LUT sizes. Computer simulation results indicate that the performance losses of SPA with reduced LUT by proposed schemes are less than 0.05 dB compared to that of SPA with original LUT. Moreover, the proposed implementation offers a large reduction in the finite wordlength for storage of updated messages using a kind of compression technique without performance losses. As a result, using the proposed implementation based on variable quantization scheme and data compression technique, the novel proposed implementation for SPA decoder architecture offers significant reductions of 75% and 33% in hardware complexities of LUT and memories, respectively, without significant performance degradations over conventional uniform quantization scheme. Furthermore, since each LUT with variable (6:3) quantization scheme II has only 8 entries, its hardware implementation can be replaced with a small combination circuit instead of readonly-memory (ROM) for LUT.
I. INTRODUCTION
Low-density parity-check (LDPC) codes, first introduced by Gallager [1], have recently received a lot of attention because these can achieve near Shannon limit performance over the binary symmetric channel (BSC) as well as the additive white Gaussian noise (AWGN) channel [2]. The decoding of an LDPC code allows a high degree of parallelism, which makes it very suitable for high data rate applications such as wide-band wireless multimedia communications and magnetic storage systems [3]-[5].
Recently, many studies have been accomplished to simplify the VLSI implementation of the related decoders

called "Architecture-Aware LDPC codes" [6] or "BlockLDPC codes" [7]. Typically, the parity check matrix of these codes can be partitioned block-wise and implemented with partially parallel decoder architectures, which can achieve a good tradeoff between hardware complexity and decoding throughput. Many practical applications such as DVB-S2 [8], WLAN (IEEE 802.1 In) [9][10] and WMAN (IEEE 802.16e) [11] have adopted or are considering the use of these LDPC codes due to their easier implementations.
It is known that the sum product algorithm (SPA) has the best decoding performance among various decoding algorithms for LDPC codes. The finite precision implementations for the partially parallel decoder using SPA have a tradeoff between the decoding performance and hardware complexity caused by two dominant area consuming factors. It requires large memories for updated messages storage and a lot of look-up tables (LUT) for implementation of nonlinear
function V(x) increasing linearly with the number of edges
in the graph of LDPC codes. Therefore, for a practical decoder implementation, the finite precision effects and the efficient memory schemes are two important issues to be considered in LDPC codes.
For regular LDPC codes, the finite precision effects are
analyzed in detail in [12]-[14]. Recently, the finite precision
effects for irregular LDPC codes are discussed in [15], which proposes the modified sum product algorithms (SPA) to improve the performance of the finite precision implementations. The quantization effect with a total of 6 bits causes a small quantization loss which is no more than 0.2dB when compared with infinite precision case [12][15]. Also, memory efficient SPA decoder architecture by the algorithm modification are introduced in [16], but it causes severe performance degradations with limited iterations.
In this paper, we propose two kinds of variable quantization schemes which result in significant reductions of LUT sizes. Also, the proposed schemes offer a large reduction in the finite word-length for updated messages storage using a certain compression technique. B. Variable Quantization Schemes
Due to the high hardware complexity required by LDPC decoding, the finite precision analysis of SPA algorithm must be analyzed in order to address the desired tradeoff between hardware complexity and decoding performance. The quantization schemes need to be considered for the intrinsic information and the extrinsic information including
the nonlinear function W(x). The finite precision effects of
(3,6) regular LDPC codes were analyzed in [12]. It is mentioned that 6 bits quantization schemes are a good tradeoff between hardware complexity and decoding performance for extrinsic information which is quantized with 1 sign bit, 2 integer bits, and 3 fractional bits. The performance loss between this scheme and floating point seems to be no more than 0.2 dB. In this work, we analyze the quantization scheme for only extrinsic information with uniform 6 bits quantization for intrinsic information.
In log domain SPA, the values of OcQ is output of nonlinear function P(x) = log(tanh x/21) as shown in Fig. 3, which
is implemented as a LUT illustrated in Fig. 1. From Fig. 3, we can observe that the slope of P(x)
decreases exponentially with the increase of x. It is intuitive that there exist some redundancies when uniform quantization scheme with same quantization step is applied. Thus, under the same performance, a non-uniform quantization scheme will save the hardware complexity which is needed for LUT implementation of T(x). Therefore, instead of using
uniform quantization scheme for P(x), we propose two
novel variable quantization schemes where for the larger
magnitudes of input of P(x), larger quantization steps are
used.

III. SIMULATION RESULTS
In the following, we assume that LDPC codes are modulated by BPSK and transmitted over an AWGN channel. For our simulations, we consider two kinds of regular LDPC codes designed by procedures in [6]. For all simulation results in this paper, the maximum number of decoding iterations is chosen to be 20. Also, for each SNR, enough codewords are simulated to result in bit errors of more than 100.
To study the quantization effects, the BER and the FER performances versus Eb/NO under SPA with several quantization schemes are shown in Fig. 4 and Fig. 5 for (1728, 864) and (1728, 1296) regular LDPC codes, respectively.
The simulation results show that the SPA with uniform (5:2) quantization schemes have poor performances and high error floors while uniform (7:4) quantization schemes

achieve similar performances to floating point for both regular LDPC codes. The uniform (6:3) quantization schemes can be a good tradeoff between hardware complexity and decoding performance.
From this simulation results we observe that the performance losses between SPA with uniform (6:3) quantization scheme and that of variable (6:3) quantization scheme I can be almost not detectable. However, the proposed variable (6:3) quantization schemes can reduce hardware complexi-
ties for implementation of a nonlinear function V(x) by 50%.
Moreover, SPA by the proposed variable (6:3) quantization scheme II performs very close to conventional SPA with uniform (6:3) quantization schemes. Their performance gaps between two schemes are less than 0.05 dB. Since SPA implementations by this variable quantization scheme II have only 25% LUT sizes compared to those of uniform (6:3) quantization scheme, we can obtain a large hardware reduction at the cost of very small performance loss.
From these observations, we can expect that the SPA architecture by proposed variable quantization scheme II can be implemented with reduced hardware complexity and without significant performance degradations.

V. CONCLUSION
In this paper, we have proposed novel low complexity implementations of sum-product algorithm (SPA) for decoding low-density parity-check (LDPC) codes. The finite precision implementations for SPA have an important tradeoff between the decoding performance and hardware complexity caused by two dominant area consuming factors which are the memories for updated messages storage and the LUT's for
implementation of nonlinear function W(x).
Two kinds of variable quantization schemes we have proposed result in significant reductions in LUT implementations while the performance losses are less than 0.05 dB compared to that of SPA with original LUT. Moreover, the proposed implementation offers a large reduction in the finite wordlength for storage of updated messages using a kind of compression technique without performance losses. As a result, using the reduced LUT (RLUT) based on variable (6:3) quantization scheme II and data compression technique, the novel proposed implementation for SPA decoder architecture
offers significant reductions of 7500 and 3300 in hardware complexities of LUT and memory, respectively, without significant performance degradations over conventional uniform (6:3) quantization scheme. Furthermore, since each RLUT with variable (6:3) quantization scheme II has only 8 entries, its hardware implementation can be replaced with a small combination circuit instead of read-only-memory (ROM) for RLUT implementation.
