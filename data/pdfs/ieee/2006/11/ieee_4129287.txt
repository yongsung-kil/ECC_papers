ABSTRACT As the code length approaches infinite, LDPC codes have the potential of approaching Shannon limits. The original message passing (sum-product) algorithms for AWGN channel require the knowledge of noise variance. If the assumed noise variance deviates from the true value, the performance tumbled. In this paper, an iterative noise variance estimation algorithm using the decoded data is presented. The true noise variance is substituted by the estimated one. The computational load is analyzed. Simulation results demonstrate the effectiveness of this method.
1. INTRODUCTION
LDPC code was first proposed by Gallager [1] and was scarcely considered in the three decades that followed due to its computational complexity and the limited computational ability of receivers. The study of LDPC codes was rediscovered by Mackay, Urbanke and others [2,3,4,6,7]. LDPC code is a kind of linear block code where the check matrix is sparse. Every check equation may check a few bits, then the error position may be located easily, while every bit is checked by a few check equations.
LDPC codes include regular codes and irregular codes. Regular codes means the number of 1 in all rows of the check matrix are the same, while the number of 1 in all columns of the check matrix are also the same. The key points of the LDPC code are the construction of the checking matrix and the decoding algorithms.
One popular decoding criterion for any codes is the Maximum a Posterior Probability criterion (MAP). For LDPC codes, the MAP decoding criterion is implemented by iteration. Though there are different names for the LDPC decoding algorithm, such as belief propagation (BP), message passing (MP), sum-product algorithm (SPA), etc., they are essentially the same. Tanner graph is an easy way to describe the MP algorithm. Lattice graph is yet another useful description tool.

The traditional SPA decoding algorithm assumes that the noise variance is known. When the noise variance is unknown, it may be assumed to be a random value. The effect of noise variance error on decoding is discussed in the paper by Mackey[9]. If we choose a noise variance value randomly, the SPA algorithm may diverge; or even if convergence is obtained, it will be a slow one. Actually, the noise variance can be estimated iteratively by using the decoded data. With the estimated noise variance, the SPA decoding algorithm may then be carried out. These two processes can be done iteratively.

The above complex iterations can be represented effectively by a bi-partite graph called a Tanner Graph. A bi-partite graph is a graph whose nodes may be split into two classes, and where edges may only be connecting two nodes not residing in the same class. The two classes of nodes in a Tanner graph are Bit Nodes and Check Nodes. A Bit Node corresponding to a bit position, and a check node corresponding to a check equation. As described in Fig.1, Check Node Cj is connected to Bit Node Vi whenever element hi,j in H is a one. Step 2 (step 3) actually passes the message from the Bit Nodes (Check Nodes) to the Check Nodes (Bit Nodes). Nodes are information processing unit. Step 2 obtains the probability of check equation hold, but Step 3 gets the probability of one bit having 0 or 1 while not considering one particular check equation. Here, information is propagated bi-directional.

5. CONCLUSIONS
Sum-product algorithm of LDPC code requires the knowledge of noise variance of Gaussian channel. If the assumed noise variance different from that of the true value, the performance of decoding degrades greatly. The decoded data may be used to estimate the noise variance. In our simulation conditions, there is 0.2 dB performance loss between the estimating noise variance case and the theory result.
