Design of Variable-Rate Adaptive LDPC Codes for DVB-S2 Applications
Nishit Pandya, Bahram Honary
Dept of Communications Systems, Lancaster University, Lancaster, LA I 4WA, UK E-mail: n.pandya@lancs.ac.uk

Abstract
Both constant and variable Forward Error Correcting (FEC) coding is provided by the DVB-S2 system[ll]. Variable FEC can be used to provide different levels of error protection according to applications, & may be combined with the use of return channels, to achieve Adaptive Coding. The novel idea explained in this paper, introduces an algebric method for construction of well-structured Low Density Parity Check (LPDC) codes, enabling the system to switch between LDPC codes of different rates (9/10, 8/9, 5/6, 4/5, ... ) and perform FEC based on the required level of error protection. In this scheme instead of using different parity check matrices for the different code rates, LDPC codes of different rates are derived by structurally modifying a mother parity check matrix. It is different from the rate-compatible [8] code construction in the sense that the H-Matrix used for different code-rates is independent of the H-matrices used for all other code-rates in the set.
Keywords: Variable-rate, Adaptive, Low-Density ParityCheck, Tanner Graph.
1 Introduction
Discovered by Gallager in 1962[I], and rediscovered in 1990s, LDPC codes form a class of Shannon limit approaching codes and perform amazingly well. Ever since their rediscovery, many design, construction and applications for these codes, have been devised. Amongst the various construction methods proposed, the two broad general categories are, Construction of pseudo-random LDPC codes, using methods such as given in [2], [3] and Construction of well structured LDPC codes, using methods such as algebraic and combinatorial as given in [3], [4], [5]. However recently it is observed that variable FEC is essential in the design of many modem practical error control systems. As demands for using LDPC codes is increasing rapidly within the industry, some authors have proposed methods to design variable rate LDPC codes [6]. These types of LDPC codes are mainly referred to as rate-compatible LDPC codes [8]. In this paper we take a different approach and introduce a method to construct variable rate codes starting from higher rate codes going to lower rate codes based on matrix expansion with no puncturing.

This method is based on superimposing structured sparse constituent matrices onto a structured base matrix [4]. With this method, structured LDPC codes with girth at least 6 can be constructed. The structured Quasi-Cyclic Matrix design is proposed for the ease of practical implementation for DVBS2 applications . Encoding of QC-LDPC code can be implemented using simple shift-registers with complexity linearly proportional to the number of parity bits of the code. These codes have been simulated over A White Gaussian Noise and decoding structure used is sum-product algorithm [7]. The proposed construction method is presented in Section 2, with the structure of the bipartite graph [9] explained. Simulation results with parameters compliant to DVB-S2 standard are shown in Section 3.

2 Construction Method
This section describes the construction of the parity check matrix that begins with design of a base parity check matrix, Hb, derived from Vandermonde-like matrix (V-matrix), type of construction. Then to extend this matrix to a set of larger matrices supporting different code rates, each element of H, is replaced by another smaller matrix a and its cyclic shifts, which is small in comparison with the size of the base matrix. This method is referred here as replacement and expansion.

2.4 Structure of the Tanner Graph
Tanner [9] introduced bipartite or Tanner graphs to describe families of codes, which are generalizations of the LDPC codes. 1t is very convenient to construct message-passing algorithms based on the Tanner graphs. In this section with an example we represent a method in which bipartite graph of H(Hb, p, q) can be derived from the graph of Hb• The Hb with dimension M x N defines a code in which N elements of each codeword satisfy a set of M parity-check constraints. The Tanner graph contains N "bit nodes", one for each bit and M "check nodes", one for each parity check equation or constraint. The graph of H(Hb, p, q) contains q.N bit nodes

3 Simulation Results

The simulation is performed for code-rates compliant to the DVB-S2 standard [10]. One of the Code-length specified in the DVB-S2 standard is L= 16200 bits, which is selected for the simulations, with 8 different code rates as specified in the standard. The dimensions M x N of the base-matrix H; derived from a Vandermonde type construction, are selected to be 270
x 2700, to obtain a code-length L = q x N. The base-matrix is expanded by replacing each element of the base matrix by a a-matrix of initial dimensions p = q = 6 and then the value of p is increased to derive H-matrices for lower code-rates. The column weight (J)c =3 is constant for both the base-matrix H; and the expanded matrix H(Hb, p, q), whereas the row weight co, varies between 6 to 24 after expansion to achieve coderates between 9/10 and 1/2.
