Abstract— In this paper a modified bit-filling algorithm is used for constructing column-weight two quasi-cyclic low-density parity-check codes with large girths. Code rows are divided into groups. A column is connected to two rows from different groups, which are apart by at least a specified minimum length. Rows that meet the minimum length are searched sequentially. Group rows are connected to consecutive columns resulting in a cyclic structure in the code matrix. Larger codes can be obtained by using larger row groups. Compared to other methods, the algorithm provides an easy and fast way to construct codes of a wide range of rates,girths and lengths. Constructed codes show good bit error rate performance. They are also easily implementable in hardware compared to those obtained using the original bit filling algorithm.
I. INTRODUCTION
Low-density parity-check (LDPC) codes have been shown to have good error correcting performance[1][2]. However, LDPC codes with column weight of two have decoding performance increasing logarithmically with increasing code size, compared to a linear increase in codes with column weight larger than two[3]. Despite the slow increase of performance, column weight two codes have been shown to have potential in such applications as partial response channels [4][5]. LDPC concatenated with Reed-Solomon codes seem to be well suited for burst errors in magnetic recordings. These codes also have less computation because columns have only two connections.
Though LDPC code performance has been shown to be good, their hardware implementation still remains a challenge. This is mainly because of their large sizes and complex random (unstructured) row-column connections. Applications have power, area latency and cost constraints that LDPC encoders and decoders must meet. Structured codes have been developed to reduce hardware implementation complexity by constraining code constructions. However, structured codes tend to have less performance compared to random codes at large code sizes[6]. In most cases, increasing girth (smallest cycle length in parity-check matrix) of a code increases its decoding performance. The girth also determines the number of iterations before a message propagates back to its original node. Performance of structured codes could therefore be improved by increasing their girths. IV. PROPOSED ALGORITHM
We modify the bit-filling algorithm such that obtained codes are quasi-cyclic. This is achieved by dividing rows of a code into equal groups. Rows are treated as vertices and used to form a distance graph of the desired girth and vertex degree.Rows that are at a desired distance from each other are searched sequentially in each group. A connection is made between two rows of different groups. The method is described as follows.
1) Divide code rows into two or more groups of equal size. If the number of rows is unknown or not given, start with a theoretical minimum number of rows if known otherwise start with a group size of k. Rows are connected to columns k times, therefore the minimum group size is k.
2) Pair row groups such that each group appears k times. k is the targeted row-weight. If groups have different number of appearances obtained codes will be irregular.
3) For each row-group pair,sequential search for a row in the second group that is at least the desired distance from first row of the first group. Start searching from the last connected or used row in each group. If a row satisfying the distance condition is not found in one of the groups the algorithm fails or the group size is increased and the algorithm starts again.
4) The rest of rows in first group are connected relative or by the same shift as the first row to other row groups. That is, the second row in the first group is connected to rows one above those connected to the first row.The third is connected to rows two above those connected to the first row and so forth. Check if the row connections just made including that of the first row do not violate the girth condition. If the girth is violated go to step 3.
5) Construct parity check matrix from the columns formed. Starting with an all zero matrix with size equal to the number of rows by the number of row connections, change the ’0’ entries to ’1’s in all the rows involved in a connections or column.

III. BIT-FILLING ALGORITHM
The bit-filling algorithm introduced in [9] constructs a LDPC code by connecting rows and columns of a code one at a time provided the girth condition is not violated. The number of row connections is almost evenly distributed by randomly selecting rows with the least number of connections first. The algorithm obtains irregular codes with either a fixed row or column weight. Though the algorithm produces high-rate or high-girth codes given a particular code size, resulting codes are not easily implementable in hardware. This is because the structure of row-column connections is not consistent enough to be an advantage in hardware implementation. The objective of the algorithm is to optimize girth or rate. In the following section we modify the bit-filling algorithm such that we obtain regular and easily hardware implementable codes.

Figure 2 (a) shows row connections for a (25,2,5) LDPC code with girth eight constructed using the algorithm. There are five connections. Each connection is found by searching for a row in the second group that is a distance of at least four from row 1. Part (b) shows the distance graph representation of the code. Part (c) shows the resulting matrix. The top five rows are of the first row group. In these rows five unshifted identity sub-matrices are formed. Rows in the first group are connected in their natural order. In the bottom five rows, only the identity sub-matrices are shifted except the first one. The sub-matrices have size equal to the row groups. Each row group forms a sub-matrix in each connection.

A. Girth eight and twelve codes
When only two row groups are used, the minimum girth obtained is eight. The two row groups form a bipartite graph. The minimum cycle in a bipartite graph is four which translates to eight in matrix form representation. Obtained codes have a minimum size of 2k × k2, with a group size of k, for all row weights of k. This is the smallest code that could be obtained.A pair of rows are connected to the same column once otherwise they form a cycle of four. Larger codes can be obtained by using larger group sizes. Increasing the size of the row groups will not change the minimum cycle of four between the two groups.
Bipartite graphs have even cycles, so only girth eight and twelve are obtained with two row groups. Girth-twelve codes are formed by a bipartite graph with a smallest cycle of six. Table I shows code sizes of some obtained girth-twelve codes. There is no known general size representation. Larger codes can be obtained by using larger row groups. From our experiments larger groups than the minimum found by our algorithm maintained the girth. However, we could not prove that all group sizes will hold the girth.

A group could be mapped to a processing node as shown by the diagram in Figure 5. The interconnect between processing nodes has reduced complexity and cost as there are fewer groups compared to when individual columns and rows are connected. The complexity of addressing within groups depends on the permutation used to connect rows and columns between groups. Connecting the rest of a group rows relative to the first row (step 4 of algorithm) produces a cyclic structure in row-column connections. This simplifies addressing when messages are communicated between processing nodes. Examples of quasicyclic LDPC code decoder architectures are found in [10][11].
