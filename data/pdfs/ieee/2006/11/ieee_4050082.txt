Abstract — In this paper, the partial-min algorithms are proposed for updating the check node messages in decoding Low-density parity-check (LDPC) codes. The proposed methods can achieve a good performance-versus-complexity tradeoff by reducing both the check node processor complexity and the decoder memory requirements. Float-point and fixed-point simulation results show that a near optimum performance can be maintained. The architecture of a check node processor implementing the partial-min algorithms is also presented. The hardware expense especially the memory requirement is analyzed and compared with normal decoding architectures1.

I. INTRODUCTION
Low-density parity-check (LDPC) codes [1] can deliver capacity approaching performance when decoded with the log-likelihood-ratio belief-propagation (LLR-BP) algorithm [2]. Due to its superior performance and highly structural decoding architecture, LDPC codes are now being considered for use in a wide range of applications especially in the digital media broadcasting and the high speed Ethernet standards. Search for an efficient and low cost implementation of an LDPC decoder has now attracted great research interest.
The LDPC codes that are used in digital media broadcasting standards always have code lengths of several 103~104 bits, e.g. the LDPC codes in DVB-S2 standard have a maximum length of 64800bits and the LDPC codes in DMB-T (Digital Multimedia Broadcasting - Terrestrial) standard have a maximum length of 7488bits. For LDPC decoders with such code lengths, a good performance-versus-complexity tradeoff is important, because the implementation complexity directly affects the silicon area and chip cost and the decoding performance affects the quasi-error-free (QEF) threshold or

node operation. But this significant reduction in complexity is made at the expense of substantial performance loss. In order to compensate the performance loss, modifications of the “min-sum” algorithm are introduced in [5] and [6] by either subtracting an offset value or multiplying a correction factor to the check node output message. However neither the offset nor the normalized “min-sum” algorithm works well with the irregular LDPC codes.
In this paper, the idea of partial-min updating of check node messages is introduced and two versions of low-complexity LDPC decoding schemes based on partial-min algorithms are presented. The LDPC codes used in DMB-T standard are employed to evaluate the performance of the proposed algorithm.
The structure of the paper is as follows. Section II describes the decoding schemes based on partial-min algorithm. The simulation results and performance evaluations are presented in Section III. Section IV presents the check node architecture and the hardware complexity analysis. Conclusion is drawn in Section V.

Equation (1) shows that the magnitude computations in CNU are based on the Φ(x) function, which drops quickly with the increase of input magnitudes. The proposed partial-min algorithms use a quantity of smallest input values instead of one to compute the check node outputs.

The most noticeable thing in the fixed-point simulations is the significant error floors. Those error floors are mostly linked to the dynamic range, the bigger the dynamic range, the lower the error floors. The number of quantization bits does not seem to have so great impact on performance.
Since the error floor is a consequence of the fixed point representation and not a consequence of the LDPC code itself, a short BCH code is often used as a concatenated outer code to eliminate the error floors, e.g., the FEC (forward error correcting) codes in DVB-S2 standard are a kind of LDPCBCH concatenated codes, so do the FEC codes in DBM-T standard.

V. CONCLUSIONS
In this paper, the partial-min algorithms are proposed to reduce the memory requirement in LDPC decoding at the expense of little performance loss. The proposed decoding schemes can significantly reduce the number of outgoing messages in a check node update process. Simulations show that a near optimum decoding performance can be achieved with the partial-min algorithms. Memory reduction analysis show that in cases of LDPC codes with high check node degrees, the up to over 50% of memory can be saved by using the partial-min algorithms.
