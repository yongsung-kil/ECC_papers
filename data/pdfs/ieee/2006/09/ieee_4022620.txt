NEW CONSTRUCTION OF RATE-COMPATIBLE BLOCK-TYPE LOWDENSITY PARITY-CHECK CODES USING SPLITTING

Abstract In this paper, we construct rate-compatible block-type low-density parity-check (B-LDPC) codes using splitting that cover a wide range of code rates from 1/3 to 4/5. They outperform other rate-compatible B-LDPC codes for all rates and can be decoded conveniently and efficiently. A strong motivation for proposing splitting scheme comes from the observation that the quality of the initial transmission is the most important factor to achieve high throughput of type-II HARQ. Proposed scheme builds low-rate codes from high-rate codes by splitting rows of the given LDPC parity-check matrix. Thus, ratecompatible LDPC codes obtained by splitting have good FER (Frame Error Rate) performance in the first transmission. Contrary to other rate-control methods such as puncturing, shortening, and extending, the splitting algorithm not only needs smaller number of operations but also shows faster decoding convergence speed since more efficient Tanner graph is used for the decoding.
I. INTRODUCTION
Low density parity check (LDPC) codes [1, 2] provide better performance than turbo codes yet with lower decoding complexity. However, the encoding complexity of LDPC codes is quadratic in the block length, which results in slow encoding. As one of fast-encodable LDPC codes, Block-type LDPC (B-LDPC) code [3], a new class of fast encodable and memory efficient code, was proposed.
Rate compatible (RC) codes are a family of nested codes where the codeword bits from the high-rate code are contained in the codeword of low-rate code and, hence, can be encoded and decoded using a single encoder/decoder pair. To construct RC LDPC codes, many schemes such as data puncturing, puncturing, and extending have been proposed.
Data puncturing (shortening) is adopted in LDPC codes to construct RC codes [4]. Shortening is a novel approach to generate incremental redundancy for type-II hybrid ARQ (HARQ) protocol. By using shortening, low-rate codes are constructed from high-rate codes by puncturing data prior to encoding and the decoder architecture always operates on the highest-rate LDPC code. Thus, low hardware overhead is needed. However, the decoding complexity becomes larger as the code rate goes lower [4].
Puncturing is widely used to design high-rate codes

from good low-rate mother code. Rate-compatible puncturing patterns [5] are constructed by selecting the punctured parity bits from the unpunctured parity bits from low-rate code. Through proper puncturing, a series of higher-rate codes are obtained from the low-rate mother code. The encoder generates full set of parity bits, but some are not transmitted (punctured). The decoder assigns neutral value to the punctured positions and performs the decoding. However, puncturing is efficient only when the amount of puncturing is not large since punctured LDPC codes are decoded by giving 0 LLR values to the punctured positions, and this causes slow decoding convergence speed. Also, punctured LDPC codes have larger decoding complexity than the unpunctured LDPC codes of the same code rate due to the complicated Tanner graph.
Contrary to the puncturing, extending builds low-rate codes from good high-rate mother codes by adding more parity bits [6, 7]. In other words, additional parity bits are added to reduce the rate in such a way that the extended code provides sufficiently good performance at the proper lower rate. Since RC-LDPC codes obtained by extending have good FER performance in the first transmission, they are suitable for type HARQ. Another motivation for extending concerns the decoding complexity. Unlike puncturing where all parity bits are generated at the encoder whether they will be used or not, extending allows parity bits to be generated only as needed, to avoid unnecessary computations at the encoder and decoder. However, if the rate of mother code rate is too high, then the performance loss in the low-rate code obtained by extending becomes large.
In this paper, we propos new rate-control scheme called splitting in order to solve the above problems. The proposed scheme comes from the observation that the quality of the initial transmission is the most important factor to achieve high HARQ throughput. For RC-LDPC codes built from splitting, the initial transmission has a good FER in the first transmission since for the splitting mother code rate is the highest rate in desired rate coverage. In contrast to alternative methods that generate codes of different rates by puncturing, shortening, or extending, our method separates a high degree check node to two low degree nodes to produce lower-rate codes. Compared to the alternative methods, proposed scheme guarantees good performance since the number of distinct check node degrees can be as small as possible after splitting and they can be distributed as uniformly as

III. NEW RATE-CONTROL SCHEME OF B-LDPC CODES: SPLITTING
The aim of this paper is to construct nicely structured rate-compatible high-performance B-LDPC codes. The conventional rate-control schemes are shortening [4](or data puncturing), puncturing [5], and extending [6, 7]. As mentioned above, these rate-control schemes have the following disadvantages. The decoding complexity of shortening become larger as the code rate goes lower. And puncturing causes slow decoding convergence speed and has larger decoding complexity than the unpunctured LDPC codes of the same rate. In Extending scheme, if the mother code rate is too high, then the performance loss of the low-rate code obtained by extending becomes large.

The splitting scheme is proposed based on the observation that the quality of the initial transmission is the most important to achieve high HARQ throughput. For RC-LDPC codes built using splitting, the initial transmission has good FER in the first transmission since good mother code is designed for the highest code rate. The splitting algorithm splits a high-degree check node into two low-degree check nodes to produce lower-rate codes by using the Corollary 1. To construct good low-rate codes using splitting, the resulting check node degrees should be made constant if possible. Contrary to the extending method, splitting does not make short cycles and can increase the girth.

From now on, we consider decoding complexity and convergence speed. The convergence speed of punctured LDPC codes is slow since the decoding is performed by assuming 0 as the initial LLR value of punctured node. Therefore, the performance of punctured LPDC code is worse than that of unpunctured LDPC code with the same threshold when the iteration number is not sufficient. Also, punctured LDPC code needs more decoding operations than unpunctured LDPC code of the same code rate since it is decoded on the more complicated Tanner graph. These can be solved if they can be decoded on efficient Tanner graph which does not include the punctured node since check nodes are split to achieve the lower-rate.
As shown in Fig. 4, it is possible for the split B-LDPC codes to be decoded using sum-product algorithm on the Tanner graph in Fig. 4(b) instead of that in Fig. 4(a) when the systems require low-rate code from high-rate code. Since only the parity bits are punctured, we do not need to recover the punctured bits after the decoding. The proposed algorithm not only needs smaller number of operations but also shows faster decoding convergence speed than the conventional decoding algorithm.

B. Simulation results
For the simulation, BPSK modulation, AWGN channel, and 1056 information bits ( z = 132 ) are assumed. In Fig. 7, to show that RC B-LDPC code using extending together with puncturing is good to support various rates, it is compared with Turbo code adopted in 3GPP [10] and RC irregular LDPC code proposed in [6]. The RC BLDPC code using extending and puncturing has the rate coverage from 1/3 to 4/5 and 100 iterations and sumproduct algorithm are assumed. The puncturing patterns and tail-biting method for Turbo code are given in [10], and 8 iterations and Log-Map algorithm are assumed. The degree distribution of mother code and extending and puncturing method for RC irregular LDPC code are given in [6], the rate coverage is from 8/19 to 8/12, and 500 iterations and sum-product algorithm are assumed. For fair comparison, we fixed the maximum variable node degree to 16. Fig. 7 shows that RC B-LDPC code using extending together with puncturing outperforms Turbo code at code rates 1/3, 1/2, 2/3, and 4/5 and RC irregular LDPC code at code rates 1/2 and 2/3. Therefore, RC BLDPC codes using extending together with puncturing are reasonable to support various rates.
When the number of iterations is 20, Fig. 8 shows that, at rates between 1/2 and 4/5, RC B-LDPC codes using splitting have about 0.1dB-0.25dB and 0.5dB-1.2dB gains at FER=10-2 over RC B-LDPC codes using extending & puncturing and only puncturing.

V. CONCLUSIONS
In this paper, we propose new rate-control scheme called splitting. Contrary to other rate-control methods such as puncturing, shortening, and extending, the splitting algorithm not only needs smaller number of operations but also shows faster decoding convergence speed. By using splitting, we construct the RC B-LDPC code achieving four code rates 1/3, 1/2, 2/3, and 4/5. This is compared with other RC B-LDPC codes using puncturing only and puncturing together with extending, respectively. Our algorithm shows about 0.1dB-0.25dB and 0.5dB-1.2dB improvements over those of RC BLDPC codes by using extending and puncturing and by using only puncturing, respectively.
By computer simulation, we verify that our algorithm shows better performance than other RC B-LDPC codes when the maximum number of iterations is 20. Therefore, they can be a good candidate for the next generation communication system.
