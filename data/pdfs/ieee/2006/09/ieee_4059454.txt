Enhanced Channel Coding for OFDM-based UWB Systems

Torben Brack, Frank Kienle, Timo Lehnigk-Emden, Matthias Alles, Norbert Wehn Microelectronic System Design Research Group, University of Kaiserslautern, 67663 Kaiserslautern, Germany {brack, kienle, lehnigk, alles, wehn}@eit.uni-kl.de
Friedbert Berens STMicroelectronics N.V., 1228 Plan-les-Ouates, Geneva, Switzerland friedbert.berens @ st.com

Abstract- In this paper the authors present an enhanced channel coding scheme and architecture providing LDPC codes for the use in future generations of the WIMEDIA UWB industry standard. Main emphasis is put on the development of a low complex LDPC code which provides a throughput of at least 1.lGbit/s, with a reduced number of iterations even up to 1.6Gbit/s. By using such an enhanced channel coding scheme the range or the data rate of the WIMEDIA UWB system could be remarkably increased with a limited increase in implementation complexity with respect to silicon area.
Index Terms- LDPC, WIMEDIA, UWB, decoder, architecture
I. INTRODUCTION
The existing WIMEDIA UWB standard [1] for short range devices specifies a data rate of up to 480MBit/s within a range of around 2m. The deployed channel coding scheme is based on traditional convolutional code (CC) with a constraint length K = 7 and code rates between 1/3 and 3/4. Especially for the high code rate R = 3/4, the diversity gain is limited and thus the performance is poor leading to limited capabilities of the system for high data rates. To be able to support larger ranges for the high data rate modes up to 480MBit/s and above using higher order carrier modulation a more sophisticated channel coding scheme providing an increased coding gain is mandatory. To support streaming application with very stringent latency and delay jitter requirements a channel coding scheme with a very low packet error ratio (PER) below 10-3 is needed. Beside the use of concatenated codes using a CC code and an RS code LDPC codes are promising candidates for very high throughput and low PER channel coding systems. LDPC codes were invented by Gallager in 1963 [2]. They were almost forgotten for nearly 30 years and rediscovered by MacKay in the mid-90s and enhanced to irregular LDPC codes by Richardson et. al. in 2001 [3]. Now they are to be used for forward error correction in a vast number of upcoming standards like DVB-S2 [4], WiMax (IEEE802.16e)
[5], and wireless LAN (IEEE 802.1 in). Providing very

III. LDPC CODES
LDPC codes are linear block codes defined by a sparse binary matrix H, called the parity check matrix. The set of valid codewords C satisfies

HxT =, VxC C.

(1)

A column in H is associated to a codeword bit, and each row corresponds to a parity check. A nonzero element in a row means that the corresponding bit contributes to this parity check. The complete code can best be described by a Tanner graph [3], a graphical representation of the associations between code bits and parity checks. Code bits are shown as so called variable nodes (VN) drawn as circles, parity checks as check nodes (CN) represented by squares, with edges connecting them accordingly to the parity check matrix. Figure 2 shows a Tanner graph for a generic irregular LDPC code with N variable and M check nodes with a resulting code rate of R = (N -M) N.
The number of edges supplying each node is called the node degree. If the node degree is constant for all CNs and VNs, the corresponding LDPC code is called regular, otherwise it is called irregular. Note that the communications performance of an irregular LDPC code is known to be generally superior to which of regular LDPC codes. The
degree distribution of the VNs f[dm x...,3,2] gives the frac-
tion of VNs with a certain degree, with d"7aX the maximum variable node degree. The degree distribution of the CNs can be expressed as gdm x-1 with d maX the maximum CN degree, meaning that only CNs with two different degrees occur [6].
To obtain a good communications performance of an LDPC code the degree distribution should be optimized with respect to the codeword size N. The degree distribution can be optimized by density evolution as shown in [6]. Furthermore, the resulting Tanner graph should have cycles as long as possible to ensure that the iterative decoding algorithm works properly. A cycle in the Tanner graph is defined as the shortest path from a VN back to its origin without traveling an edge twice. The shortest cycle in the Tanner graph is called girth (G). Especially cycles of length 4 have to be avoided [3].

Updating the nodes can be done with a canonical, two-phased scheduling: In the first phase all variable nodes are updated, in the second phase all check nodes respectively. B. Layered Decoding
Layered decoding applies a different message schedule than the classical two-phase decoding. It was originally proposed by [8] and denoted as turbo decoding message passing (TDMP), then it was referred to as layered decoding by Hocevar [9]. The basic idea is to process a subset of CNs and to pass the newly calculated messages immediately to the corresponding VNs. The VNs update their outgoing messages in the same iteration. The next CN subset will thus receive newly updated messages which improves the convergence speed.
The major problem to utilize layered decoding within an LDPC decoder realization is the availability of the current message. The hardware realization of the WIMEDIA decoder is a partly parallel architecture presented in Section VI, at which a subset of nodes is instantiated as functional units (FU). To provide flexibility for upcoming codes and code rates, these VFU and CFU for variable and check node

V. UWB LDPC CODE DESIGN
The LDPC code design has been guided by the following main prerequisites
* Throughput higher than 48OMbit/s . Latency less than 2,us . Small silicon area in a 65nm CMOS process . Communications performance superior to that of con-
volutional codes * A fixed block length of N = 1200 bits and rate R =3/4
to ensure compatibility to the existing high data rate system
The main focus here is the extension of the possible data rate of the existing WIMEDIA UWB standard and the optimization of the existing 480MBit/s mode. Designing an LDPC code which can be efficiently processed by an LDPC decoder architecture under the given constraints requires knowledge of LDPC code and VLSI design. The best method is to design an LDPC codes with respect to hardware constraints which was first presented by [10]. Since then many further strategies were elaborated even in the UWB context [11][12], however the basic concept of an efficient LDPC code-decoder design is based on permuted I-by-I identity matrices. Thus always I entries of the parity check matrix can be described by the position of the identity matrix and its permutation. A permutation reflects a cyclic shift of the identity matrix. This results in a compact LDPC code description. The structure of our UWB LDPC code is shown in Figure 3. The size of the identity matrices is I = 30. Note that this size directly matches the number of parallelism of our architecture as shown in the next chapter. The two

However, the resulting decoder would result in a twice as large silicon area compared to our approach. Furthermore, the layered constraint explained above could not be preserved which would involve a high throughput penalty.
VI. LDPC DECODER AND ENCODER ARCHITECTURES
A partly parallel architecture template is mandatory to ensure flexibility for an LDPC decoder realization. A parallelization of P is established, which means that P edges are processed per clock cycle. This size P is determined by the identity matrix size I as described in Section V. To ensure code rate and codeword size flexibility the functional nodes are realized in a serial manner. Thus each functional unit can accept one message per clock cycle as already mentioned. The mapping of the VNs and CNs to the functional units is determined by the given code structure. Efficient mapping of the nodes to the functional units can be always guaranteed by designing LDPC codes using permuted identity matrices. Always I VNs/CNs with the connection described by an Iby-I identity matrix are allocated to I distinct VFU and I distinct CFU respectively. This was shown by many LDPC decoder realizations [10][15][16][17] as well as in the case of DVB-S2 LDPC decoding [18]. A permutation network has to realize the permutation of the identity matrix what can be done by a barrel shifter. To reach reasonable communications performance and minimize the area consumption, the check nodes are implemented with an optimized version of the Min-Sum algorithm (Section IV-A).
The overall decoder architecture can be seen in Figure 4. It applies layered decoding as described in Section IV-B at which the VN processing can be done within the check node block (CNB). This is achieved by storing the input information of the CFU in a FIFO and adding the new extrinsic information to the corresponding input message in the FIFO. Thus the correct a posteriori information is passed back to the channel memory which holds always the updated VN information. This basic concept was first

VIII. CONCLUSIONS
In this paper a new enhanced channel coding scheme for the use in a WIMEDIA UWB system based on OFDM has been presented. The communications performance in the typical transmission channel CM1 and CM2 has been
evaluated down to a PER of 10-3. The results show the superior performance of the LDPC codes compared to the standard CC codes used in WIMEDIA version 1.1.
Especially for the low PER region needed for the use in streaming video application the LDPC codes can gain up to 4.5dB. Two different LDPC decoder implementations and their corresponding encoders have been investigated using synthesis results for a 65nm state-of-the-art CMOS process:
* Low Complex LDPC decoder compliant with the frame format and the latency requirements of the existing WIMEDIA standard and
* High throughput and high performance LDPC decoder with a large block size.
The overall complexity of the first decoder implementation
is below 0.2mm2, the second utilizes an area of around 0.6mm2. Encoder complexity is an order of magnitude re-
spectively seven times below which of the corresponding decoder realization. The communications performance figures

and the complexity results clearly show the potential of the
proposed LDPC decoders for the use in future WIMEDIA
standard extensions.
