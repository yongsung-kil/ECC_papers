Abstract
Several semi-parallel decoding architectures have been explored by researchers for the quasi-cyclic low density parity check (LDPC) codes. In these architectures, the reconfigurability was seldom studied. In most of the published work, the shuffle network between the log-likelihood ratio (LLR) memory and the check- node units (CNU) is predetermined and optimized for a specific code.
However, the most of the modern wireless communication systems typically specify multiple code-rates, codeword lengths and sub-matrix sizes (for the QC-LDPC codes) to provide and guarantee quality-of-service (QoS) over challenging channel conditions. It is therefore desirable to define a reconfigurable decoder architecture that can support different parameters. In this paper, we propose an adaptive shuffling algorithm for QC-LDPC codes which together with the Benes network provides arbitrary cyclic shift for arbitrary submatrix size smaller than the input size of the Benes network. For the submatrix size larger than the Benes network input size, the proposed algorithm can be extended to obtain the cyclic shift in multiple stages. Compared with the direct implementation of m m-to-1 multiplexers, the proposed algorithm achieves significant savings on area and routing complexity.

It has been shown that the structure of QC-LDPC codes allows efficient semi-parallel decoder implementation by greatly reducing the complexity of the interconnect network [8]. Recently QC-LDPC codes supporting different code-rates and different size of permutation matrices have been proposed in the standards such as IEEE 802.16e and 802.11n. Typically the interconnect networks are designed and optimized for a given size of the permutation matrices [8, 9] making them unsuitable for multiple submatrix sizes.
In this paper, we propose a reconfigurable interconnect network for the LDPC decoder using the Benes network structure, which can support all the cyclic shifts for the submatrices of sizes smaller than the number of inputs to the Benes network. We also propose a simple memory addressing scheme, which enables the decoder to support the submatrix sizes larger than the size of the Benes network, with multiple clock cycles, thereby allowing area efficient semiparallel decoder architectures.

2 QC-LDPC Codes and Decoders
2.1 Code Construction QC-LDPC Codes
The LDPC codes are usually described in terms of a parity check matrix, H. With a valid code word, x, we have Hx = 0. The QC-LDPC codes are defined using a seed matrix Hseed of size ρ × χ and the full parity-check matrix is constructed by replacing each positive element hsi,ejed with an appropriately shifted identity matrix of size m × m. Negative or null elements in the seed matrix are replaced with an all-zero matrix of size m × m. The constructed parity check matrix H has size mρ × mχ and code rate R = 1 − (ρ/χ). Examples of seed matrix can be found in numerous references, including [1, 2, 5, 8].
2.2 Decoder Architectures of QC-LDPC Codes
The structured parity-check matrices make the QCLDPC codes attractive for hardware implementation. Specifically, they significantly simplify the memory access and the message routing in the decoder. A simple semi-parallel QC-LDPC decoder architecture is illustrated in Fig. 1 [4]. The architecture implements multiple parallel processing elements (PE) and each row (check-node) of a non-zero submatrix is processed by a separate PE. The PE is also referred as check node unit (CNU). The LLR data and check-node messages are fetched from and written back to corresponding memories in parallel. In this architecture, the variable-node (bit-node) messages are computed from accumulated LLR messages and check-node messages using a simple subtraction, thereby eliminating the need for variable-node message memory. The LLR data is shuffled according to the structure of the submatrix before it is sent to the CNUs (PEs). For the QC-LDPC decoder, because the submatrices are cyclically shifted identity matrices, the

shuffle network simply implements the cyclical shifts on the LLR data.
3 Proposed Algorithm
Direct implementation of the shuffle network requires m m-to-1 multiplexers. By smartly choosing the control pins of the multiplexers, it is possible to implement all cyclic shifts required in a parity check matrix H. However, the hardware complexity of such implementation increases exponentially as the value of m increases. Also, the routing complexity increases exponentially as m increases. Thus, it is desirable to design a low-complexity network to achieve the same function.
3.1 Benes Network and Its Application in QC-LDPC Decoders
Similar problems have already been addressed in the network router design to obtain non-blocking switching networks. Among these networks, the Benes network is an optimal non-blocking network that achieves any desired permutation with only S/2 × (2 log2 S − 1) 2-by-2 switches, where the input size of the Benes network, S, is a power of 2 [3]. The Benes network of input size S can be derived from two Benes networks with input sizes S/2 as illustrated in Fig. 2. In the Benes network, the permutation performed is determined by the control signals to the 2 × 2 switches. With the same Benes network, there exist multiple sets of control signals which can implement the same permutation.
For QC-LDPC decoder, it is not necessary to implement all permutations but only the cyclic shifts. It has been shown that, only the first or last log2 S stages of the S-input Benes network are required to achieve all cyclic shifts for S inputs, [9]. The implementation of the half or reduced Benes network is referred as Banyan switch. When the submatrix size in the QC-LDPC, m, is not a power of 2, it is possible to use an S-input Banyan switch with S > m and one additional multiplexing stage (at switch output or input) to provide all the cyclic shifts s = 0, 1, · · · , m − 1.
3.2 Reconfigurable Benes Network
In real applications, the decoder may need to support different value of m and the value typically changes on-the-fly. For instance, in IEEE 802.16e, the value of m in the proposed LDPC codes ranges from 24 up to 96 [1]. The Banyan switch, as explained in the previous subsection, cannot provide the flexibility over m and the complete Benes network is required instead. However, even for the Benes network, which has been proved to be non-blocking, the control pins of each of the 2x2 switches must be determined based on

Step 1. Generating inputs to the auxiliary Benes network: The input to the auxiliary Benes network is an array of integers from 0 to S − 1, where the first m integers are cyclically shifted to the left by s positions, i.e., the inputs are [s · · · m − 1 0 · · · s − 1 m · · · S − 1].

3.3 Bitonic Benes Sorting Network
In the second step of the proposed scheme, we need to find a bitonic sorting algorithm for the Benes network for the given inputs from the first step. In a general bitonic sorting network, we need log2(S) × S/2 switches and the same number of comparators to generate the control signals to the switches. In [6], the Benes network was applied as a bitonic sorting algorithm. However, in [6], in some stages, iterative comparison and switching are required, which makes it unsuitable for a pipelined decoder. In the scheme, the auxiliary Benes network must have the same structure as the main Benes network, i.e., it must be all feed-forward with no feedback loops. 4 Synthesis Results

Table 1 compares the hardware complexity of two implementations of the permuter for the LDPC decoder with LLR of word length 8 bits. While computing the gate count for the direct implementation, circuits generating the control signal to the multiplexers are not included. Moreover, the additional buffers in the direct implementations due to the large fan-in and fan-out requirement in the circuit were not included. In the proposed circuit, the control circuit includes both the ABN and the circuits to generate the sequence in Step 1 of the proposed scheme.
Even with the unfair comparison, in which the gate counts of the direct implementation were significantly underestimated, we still can observe dramatic savings on the hardware complexity. The savings are more significant as the value of S increases.
At the same time, due to the symmetric property of the Benes network, it is also possible to reuse ABN outputs between the reading circuit and writing circuit of the LLR memory, if the LLR update is included in the architecture.

5 Conclusion

In this paper, we have proposed a reconfigurable permutation network for the QC-LDPC code decoder. This network can implement all cyclic shifts for input sequence with size less than the Benes network size. Compared with the direct implementation of the shuffle network with S-to-1 multiplexers, the proposed algorithm achieves significant hardware complexity reduction. Furthermore, with little
