OVERLAPPED DECODING FOR A CLASS OF QUASI-CYCLIC LDPC CODES
Sang-Min Kim and Keshab K. Parhi
Dept. of Electrical & Computer Engr., University of Minnesota, Minneapolis, USA {smkim, parhi}@ece.umn.edu

ABSTRACT
In low-density parity-check (LDPC) code decoding with the iterative sum-product algorithm (SPA), due to the randomness of a parity-check matrix H, the overlapping of check node processing unit (CNU) and variable node processing unit (VNU) in the same clock cycle is difficult. This paper demonstrates that overlapped decoding can be exploited as long as the LDPC matrix is composed of identity matrices and their cyclic-shifted matrices, i.e., a parity-check matrix H belongs to a class of quasi-cyclic LDPC codes. It is shown that the number of clock cycles required for decoding can be reduced by 50% when overlapped decoding is applied to a (3,6)-regular LDPC code decoder.
1. INTRODUCTION
Low-density parity-check (LDPC) codes with iterative decoding based on belief propagation (usually known as sumproduct algorithm (SPA)) are considered to be more promising than turbo codes for many applications due to their great error correction capability and low error floor characteristic [1]-[2]. Furthermore, very high decoding throughput can be achieved since SPA decoding is inherently fully parallelizable. Recently, LDPC codes based solution has been adopted for the next generation DVB (digital video broadcasting) satellite communications standard [3].
Several LDPC code construction methods have been proposed, which are based on optimal degree distributions [2], pseudorandom generations [4], and finite geometries [5], etc. Since the overall structures of LDPC code decoder strongly depends on how LDPC codes are composed, efficient decoder VLSI implementation must be taken into account during the code construction step. In [6]-[10], the code designs and decoder implementations are jointly considered to induce the structural regularity required for a reduced complexity parallel decoder.
Most solutions for LDPC code decoding employ SPA decoding which consists mainly of horizontal (check node

Due to the randomness of a parity-check matrix H, the overlapping of the two computation units, the check node processing unit (CNU) and the variable node processing unit (VNU) in the same clock cycle, is difficult. Conventionally, the VNUs cannot start their computation until the whole CNU computations are completed and vice versa. Recently, overlapped message passing approach for a specific quasicyclic code has been proposed in [11].
In this paper, it is shown that the overlap between CNU and VNU processes can be exploited as long as a paritycheck matrix is composed of identity matrices and their cyclicshifted matrices. The null space of the constructed paritycheck matrix can be regarded as a case of quasi-cyclic LDPC codes. In case of such regular parity-check matrices, an overlap approach can be used to reduce the number of clock cycles per iteration. Up to 50% reduction of decoding latency can be achieved by applying overlapped decoding to a (3,6)-regular LDPC code decoder.
This paper is organized as follows. Overlapped decoding approach is proposed in Section 2. Modified (3,k)-regular LDPC code design for overlapped decoding is presented in Section 3. Finally, conclusions are given in Section 4.
2. OVERLAPPED DECODING
SPA decoding has three steps: 1. Initialization step, 2. Iterative decoding step, and 3. Decision step. The iterative decoding step consists of a horizontal (CNU processing) step and a vertical (VNU processing) step. Iterative decoding step is performed repeatedly until decoding is completed successfully or the number of iterations reaches the pre-set maximum number of iterations. In practice, during the initialization step, memories are initialized for storing the intrinsic information and preparing the next step.
Fully parallel LDPC decoders require very high complexity and suffer from large interconnection complexity. Partly parallel solutions overcome these bottlenecks and are of great interest.

H2 is a cyclic-shifted version of L × L identity matrix under a rule to obtain a high girth. The girth of [H1; H2] is 12. H3 is configured by a set of constrained random parameters to exploit decoder-first code design method. This (3, k)-regular LDPC code is implemented by partly parallel design approach, where both CNU and VNU processes are scaled-down by a factor L. Thus 3·k CNUs and k2 VNUs are used to compute one decoding iteration in 2L clock cycles.
H3 should be re-designed such that it has a quasi-cyclic property to obtain an overlapped decoding decoder. To do this, sub-matrices for H3 are generated such that they are cyclic-shifted versions of L × L identity matrix and the degree of shift for each sub-matrix is randomly defined as long as H is 4-cycle free. By this way, many such sets for H3 are generated and each H3 is plugged-in. Then, to extract a qualified code, the bit-error rate (BER) vs. SNR simulations over AWGN channel assuming the LDPC codes are modulated by BPSK are performed. Since its sub-matrices are either identity matrices or cyclic-shifted identity matrices, overlapped decoding approach can be exploited to reduce the number of clock cycles per iteration.

4. CONCLUSIONS
Usually, as long as a parity-check matrix consists of identity matrices or cyclic-shifted identity matrices, it gives a good encoder and decoder design solutions with linear feedback shift registers and partly parallel implementations respectively. However, it is inevitable that decoding latency increases due to partly parallel implementations.
In this paper, we show that overlapped decoding approach can be applied to any parity-check matrices with SPA decoding in conceptual level. Also, it can be employed with memory-based implementations to reduce increased decoding latency as long as a parity-check matrix belongs to such quasi-cyclic LDPC codes. By an example, it is shown that, by overlapped decoding, clock cycles required per iteration can be reduced by 25% with respect to one iteration and, furthermore, reduction by about 50% can be achieved using re-overlapping.
