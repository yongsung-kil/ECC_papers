Reduced Complexity Decoding Strategies for LDPC Convolutional Codes
Ali Emre Pusane, Michael Lentmaier, Kamil Sh. Zigangirov, and Daniel J. Costello Jr. Dept. Electrical Engineering, University of Notre Dame, Notre Dame, IN 46556, U.S.A. {Pusane.1, Lentmaier.1, Zigangirov.1, Costello.2}@nd.edu

Abstract — While low-density parity-check (LDPC) convolutional codes tend to significantly outperform LDPC block codes with the same processor complexity, large storage requirements and a long initial decoding delay are two issues related to their continuous pipeline decoding architecture [1]. In this paper, we propose reduced complexity decoding strategies to lessen the storage requirements and the initial decoding delay without significant loss in performance.

I. A Stopping Rule
The conventional stopping rule for LDPC block codes cannot be used in the continuous pipeline convolutional decoder because of the unlimited size of the parity-check matrix and the undetermined length of the input sequence. As an alternative, we propose a stopping rule that is tailored to continuous decoding of LDPC convolutional codes. Since decoding is done over a finite size window, instead of trying to determine whether the entire decoded sequence is a codeword or not, one can consider segments of the decoded sequence and determine whether they match a segment of a valid codeword. At each time instant, before processing the current constraint nodes within the operating region, each pipelined processor checks whether the corresponding parity-check equations are satisfied. If a sequence of P consecutive satisfied parity-check equations is observed, the processor enters a sleep mode and stops processing until the next violation is detected, where P is a parameter on the order of the syndrome former memory ms.
II. On-Demand Variable Node Activation
For decoding LDPC block codes, the standard messagepassing schedule calls for, in each iteration, all the variable nodes, and then all the check nodes, to be activated. This scheduling corresponds to the message-passing schedule of the pipeline decoder introduced in [1], in which, within a processing region, the check node activations are performed one-byone, and the variable node activations are done just before a set of variable nodes leaves the region. However, with this scheduling, the incoming information to a variable node is included in its outgoing messages only after an iteration is completed. As an alternative, we propose an on-demand variable node activation schedule for the pipeline decoder. According to this schedule, the activation order of the constraint nodes does not change, but instead of activating the variable nodes that are about to leave an operating region, we activate a variable node whenever its message is requested by an active constraint node. III. Compact Pipeline Decoder The requirement that the processing regions cannot overlap follows directly from the independence of the operations performed in the standard schedule. But, once on-demand scheduling is employed, we can let neighboring operating regions overlap to decrease the total storage requirements of the decoder. A large overlap will cause performance degradation. However, it has been observed that the performance degradation is negligible, as long as the overlap does not exceed half the processing region size.

IV. Conclusion Simulation results for the bit error rate (BER) performance of the proposed reduced complexity decoding techniques for (ms, J, K) LDPC convolutional codes with J = 3, K = 6, and different syndrome former memories ms over the additive white Gaussian noise (AWGN) channel with signal-to-noise ratio Eb/N0 per bit are presented in Figure 1. A maximum number of I = 100 iterations was allowed. It is observed that the stopping rule performs very well for various memory lengths without performance degradation. For the (128, 3, 6) code, the average number of iterations performed per bit at Eb/N0 = 1.00dB is 72, and this value drops to 7 at 1.80dB. Also, it is seen that the on-demand scheduling improves the BER performance for a fixed number of iterations. Or, conversely, for a fixed BER, the proposed scheme requires fewer iterations. This decrease in the number of required iterations reduces the initial decoding delay and the storage requirements of the decoder without a sacrifice in computational complexity. We can further reduce the storage and delay requirements by using the compact decoder implementation.
