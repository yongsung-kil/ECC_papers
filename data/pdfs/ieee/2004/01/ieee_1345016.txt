Abstract
In this paper we describe the construction method of a family of irregular rate-compatible low-density paritycheck (LDPC) codes by a combination ofpuncturing and extending techniques. In particular, we introduce a suitable structure for the extended parity-check matrices which preserves the structure of LDPC codes during extensions. Based on this construction, a family of eficient rate-compatible linear-time encodable codes are generated f.om an optimized irregular mother code of rate 8/13 and information block length k=1024. The rates of the codes varyfiorn 8/10 to 8/19 and employing them in a type-11 hybrid ARQ scheme results in a throughput which is only 0.7 dB away @om Shannon limit. This improves over the existing schemes based on turbo codes and LDPC codes by up to 0.5 dB.
Keywords: Rate-compatible low-density parity-check (RC-LDPC) codes, hybrid ARQ schemes, puncturing, extending, iterative coding schemes.
1. INTRODUCTION
Rate-compatible (RC) codes are a nested sequence of codes where the parity bits of higher rate codes are embedded in those of lower rate codes [I], [2], [5]-[7]. RC codes have many applications in packet data communications where adaptive coding and/or unequal error protection is required [2]. One of the main advantages of using RC schemes is that all the codes in the sequence can be encoded and decoded using a single encoder/decoder pair. In addition to having a low complexity, these schemes provide an efficient 6amework for the transmission of information using automatic repeat requesvfonvard error correction (ARQIFEC) protocols. In such a 6amework, referred to
as type-II hybrid ARQ,incremental parity bits of the next
lower rate code are transmitted in response to a negative

acknowledgement (NACK) from the receiver, which indicates the failure of the last code in correcting the errors in the transmitted packet [2], [5]-[7]. Since 1970’s, RC convolutional and block codes have been designed
using puncturing (for a list of references, see [Z], [5]). More recently RC punctured turbo (RCPT) codes [6], [7] (for more references, see [5]), and RC low-density paritycheck (RC-LDPC) codes [5] were also introduced. Both
schemes provide near capacity performance with throughputs about IdB from the capacity of additive white Gaussian noise (AWGN) channel. In another related work, Ha and McLaughlin [I] study the problem of finding puncturing distributions for RC-LDPC codes that optimize the threshold over a range of rates. Their results however focus mainly on asymptotic behavior of
codes at very large block lengths. In [5], the authors show that puncturing alone cannot
provide a sequence of well-performing LDPC codes with a wide range of rates. The problem with puncturing occurs at higher rates where the large percentage of punctured bits (eraswes) paralyzes the iterative softdecision decoder. To overcome this, both puncturing and extending is used in [ 5 ]to create a sequence of RC codes 60m a regular LDPC code. In this work, we devise RCLDPC codes that outperform codes constructed in [5] by up to 0.5dB in throughput performance. In our construction, we also use both puncturing and extending. Our work however can be differentiated from that of [5] based on the followings:
The structure used for extending the parity-check matrix is different. It provides a more uniform row weight distribution, and a stronger dependency between the columns of the parity-check matrix of the mother code and the newly added columns. These improve the performance of extended codes. The proposed structure for extension is deterministic. This makes the design and implementation of the codes modular and simple.

3. SIMULATION RESULTS

We consider using the designed RC-LDPC codes in a type-Il hybrid ARQ scheme based on selective-repeat ARQ protocol. We also assume that LDPC codes are transmitted over a binary input additive white Gaussian noise (BI-AWGN) channel with bipolar signaling, and are decoded using iterative belief propagation algorithm [8]. For decoding, we only need the decoder for the lowest rate code (8/19). To decode the higher rate codes, missing parity bits are replaced by erasures at the decoder. Packet combining (see, e.g., [ 5 ] ) would be used to combine different copies of the same coded bits until the receiver successfully decodes the transmitted packet. We also use LDPC codes for both error correction and error detection, and thus no extra redundancy for error detection is required. Moreover, we consider noiseless feedback channel and infinite buffer size at both transmitter and receiver. Also, in this work, we are not concerned about transmission latency, and the traffic of feedback channel, and only consider throughput q, defined below, as the measure of performance of the

4. CONCLUSIONS
Design guidelines for constructing RC-LDPC codes, which perform very well with iterative decoding and are also linear-time encodahle are presented. Important features of the proposed design are i) fmding the proper rate for the mother code, ii) properly extending and puncturing the mother code to preserve both performance and linear-time encodability, iii) constructing a good mother code which is also hear-time encodahle.
