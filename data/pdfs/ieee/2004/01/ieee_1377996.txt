On Unequal Error Protection LDPC Codes Based on Plotkin-type Constructions
Vidya Kumar and Olgica Milenkovic Electrical and Computer Engineering Department,
University of Colorado, Boulder e-mail: (kumarv, milenkov)@colorado.edu

Abstract— In this paper, a family of Unequal Error-Protection (UEP) Low-Density Parity-Check (LDPC) codes, based on Plotkintype of constructions, is introduced. The codes are decoded in multiple stages in such a manner that the order of decoding determines the level of error protection. The level of UEP among the code bits can be further increased by properly combining structured and random-like LDPC component codes with carefully chosen properties, and by using some new reliability features. The proposed scheme also offers a good trade-off between code performance on one side and encoding/decoding and storage complexity on the other side. To the best of our knowledge, the proposed approach represents the first iterative UEP method with analytically provable properties.

Low-density parity-check (LDPC) codes [6] are iteratively decodable codes on graphs that exhibit capacity approaching performance over channels such as the binary erasure or the additive white Gaussian noise (AWGN) channel [13]. In this paper, we propose to investigate the application of LDPC coding methods for channels that require Unequal Error Protection (UEP). UEP of data is sought for many practical applications, including joint coding for general parallel channels with nonuniform noise statistics, robust transmissions of still images in the JPEG standard and videos over wireless channels, or storage on holographic memories. The key idea behind the proposed UEP coding approach is to utilize the iterative Plotkin construction, which is a well known design technique borrowed from algebraic coding theory. For this technique, one starts with a set of random-like and/or structured LDPC codes to be used as component codes, and then creates a long length code in an iterative fashion. The construction allows for low complexity encoding and multi-stage iterative decoding, which provides a natural setting for achieving UEP. The last feature can be seen as arising from the irregular structure of the parity-check matrix, where different blocks of variable nodes have different degrees; or, it can also be viewed as a characteristic intrinsic to the decoding scheme employed for these codes. The underlying soft-decision decoding algorithm operates iteratively on two different levels: one is the level of the Plotkin construction itself, while the other is the level of iterative decoding of the component LDPC codes. A decoding method with the first feature only was also investigated by Dummer and Shabunov [2], [3], for soft-decision decoding of Reed-Muller (RM) [10], although without any reference to UEP.

II. PLOTKIN-TYPE CONSTRUCTIONS AND MULTI-STAGE DECODING
The Plotkin construction, also known as the |u|u + v| construction, is a code design method which increases the length of the code by combining component codes in an additive manner [10]. For the sake of completeness, we will briefly review the most relevant characteristics of this construction strategy.
Definition 1: Given a [n, k1, d1] code C1 and a [n, k2, d2] code C2, let C be a [2n, 2(k1 + k2), d = min(2d1, d2)] code, with codewords formed as the direct sum of vectors u, u+v, u ∈ C1, v ∈ C2, i.e.:

C = {|u|u+v|, u ∈ C1, v ∈ C2},

(1)

Such a code design strategy will be referred to as the Plotkin construction.
Applying the Plotkin construction recursively results in codes that usually have very good error-correcting properties. For example, the well-known class of Reed-Muller codes [10] can be derived in this manner.

In order to ensure a large error resilience for a given set of variable nodes, it is desirable to have these nodes involved in as many check equations as possible. Hence, one could expect the first half of the bits in C to have a higher level of error protection then the second half. But this cannot always be true, since some low degree and unreliable variables can share check nodes with large degree nodes. This would impose the requirement for high degree nodes to share many checks with other high degree nodes, and very few with low degree nodes. A code design approach satisfying such constraints is very hard to devise.
The algorithm proposed in this work combines new code designs and specialized multi-stage decoding operations and offers a simple and practical solution for the problem described above. The multi-stage decoder does not operate on the global code graph so that one does not encounter any four-cycles arising due to the Plotkin construction.
We will describe next the multi-stage (MS) decoding algorithm; in Section III we will discuss some novel modifications of the MS algorithm which use a threshold function to further improve the decoder performance. The description of the algorithm is valid for one level of the Plotkin design, but can be extended analogously for any other number of levels. Throughout the remainder of the paper we will assume that the codewords under investigation are BPSK modulated and transmitted over an AWGN channel.

The recursive Plotkin type construction can be represented

by a binary tree. The tree is constructed by starting from a

root node representing the codeword z itself, and then adding

two branches: the left branch, leading to the u component

of z, is labeled by 0, and the right branch, leading to the

v component, is labeled by 1. The same procedure is now

recursively repeated, starting with the components of u and v.

Therefore, each component code of the Plotkin construction has

an equivalent binary representation. This representation can be

used to describe the equivalent noise level seen by that specific

codeword component.

Improved Reliability-Based Decoding: For a Plotkin construc-

tion of large depth, the first few codes to be decoded will

experience a significantly degraded equivalent channel. The

reliability values of their code bits are used in all subsequent de-

of the decoding scheme. It is therefore desirable to modify

the decoding process in such a way as to include only very

sufficiently reliable v-based estimates. This can be accomplished
