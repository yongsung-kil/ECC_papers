Accelerating LDPC Decoding using Multiple-Cycle Eigenmessages
John S. Crockett, Todd K. Moon, Jacob H. Gunther, and Ojas S. Chauhan Utah State University

AbstractŌĆö The eigenmessage decoder [1] expresses a degree of nonlocality in a message passing decoder by representing an entire cycle in a single linear equation. The eigenvector for the linear message passing matrix represents a fixed point of the message passing algorithm around a cycle, and has been shown to significantly decrease the number of iterations required for lowdensity parity-check (LDPC) decoding using message passing. In this paper, we extend the eigenmessage idea to simultaneously solving for a fixed point of multiple cycles. The representation is thus nonlocal over a broader section of the graph. This multiple-cycle eigenmessage algorithm shows improvement over traditional belief propagation (BP) methods. We present and compare results for the new algorithm and consider possible variations.
I. BACKGROUND
Maximum likelihood decoding of a code on a graph is a global discrete optimization problem and suffers from computational complexity. Message passing decoding algorithms [2], [3], [4] reduce the global problem to a set of local problems which have lower complexity, finding a solution usually equal to the global solution. It is well known that for a graph without cycles (i.e., a tree), the message passing algorithm terminates in a number of iterations proportional to the diameter of the graph. However, some graphs, such as those associated with iteratively decoded codes such as turbo codes, low-density parity-check (LDPC) codes, or repeat-accumulate codes, do have cycles. In such cases, the performance of the message passing algorithm has been successful, but the decoding may take many iterations. For example, for a moderately large LDPC code, several hundred iterations may be necessary for decoding at low SNR. This high number of iterations diminishes the potential utility of such codes by increasing decoding complexity or latency. We have been seeking [1] decoding algorithms which are less local than conventional message passing in an attempt to reduce the number of iterations.
Behavior of message passing algorithms on graphs with cycles has been examined in some simplified cases. In [5], very small turbo codes with block lengths ┬Ī were examined. In [6], a graph with a single cycle was examined, where the simplified structure of the graph led to an explicit eigenvalue problem whose solution exactly characterized the fixed point of the iterated message passing; the analysis compared the ŌĆ£trueŌĆØ solution to the message-passing solution obtained using the eigenvalue problem.

VI. AVERAGING EXTENSION OF EIGENMESSAGE APPROACH

The multiple-cycle eigenmessage is one way of sharing

information among the cycles of the Tanner graph. However,

it is computationally complex. In this section we introduce

another method.

Information can be passed between cycles by averaging the

messages associated with edges shared among the cycles. For

instance, two cycles and might both include an edge and

However, the averaging approach did not have as dramatic

we had anticipated. It appears that the single-loop eigen-

message algorithm was already unwittingly taking advantage

of this information sharing. Since the eigenmessages for all

cycles are not updated simultaneously, the calculations for

cycles considered near the end of an iteration are based on

already-updated messages. By saving and incorporating the

contaminated our ŌĆ£freshŌĆØ calculation with older information.

VIII. COMPUTATIONAL COMPLEXITY
The multiple-loop eigenmessage approach adds significant computational complexity to the single-loop algorithm. While in the single-loop case, by calculating appropriately, the eigenmessages can be read directly off of a matrix, we know of no shortcut in the multiloop case that can circumvent the QR factorization.
The QR factorization is already an improvement to using a Lagrangian, but it still requires a lot of effort. For instance, using multiple-loop eigenmessages with a maximum group size of five, on the (15000,5000) code we tested, each iteration required just over 12 million FLOPS (floating point operations). Of these, 8 million FLOPS were in QR factorizations, 4 million FLOPS in the rest of the eigenmessage calculations, and only 200,000 FLOPS for a cycle of BP and 100,000 FLOPS to check if the output satisfied all checks. The QR factorization accounts for 66 percent of the floating point calculations in the decoding process. In a hardware implementation, these computations could be pipelined using CORDIC rotations, but the computational burden is still high.
