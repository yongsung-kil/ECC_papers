Abstract— We consider in this paper the encoding problem for low-density parity-check (LDPC) code. We investigate an efficient encoding approach for LDPC code. The straightforward the existing encoding scheme for LDPC code usually incurs too high complexity and should be changed to encoding scheme with low complexity. However, little consideration has been given to the LDPC encoder VLSI implementation.
We consider low complexity encoding of regular LDPC code, and we propose a pivoting and bit-reverse (PABR) algorithm to rapidly construct parity-check matrix for regular LDPC code. And We consider Improved Regular LDPC codes in BER performance. We propose that parity-check matrix is constructed with sub-matrixes which are made up by costas array. The code have sparse parity-check matrix. They are designed to perform well when iteratively decoded with the sum-product decoding algorithm and to allow low complexity encoding. Performance is not less than that of irregular LDPC code can be achieved.
We show approach to implementing LDPC encoder using PABR algorithm and Improving BER performance using parity-check matrix with costas array. This paper then describes FPGA implementation of regular LDPC encoder on hardware platform for 4th mobile communication system.
Keywords-channel codes, regular low density parity check codes, LDPC.
I. INTRODUCTION
Low-Density Parity-Check (LDPC) Codes were first discovered by Gallager [1] in 1962 and rediscovered by Mackay [2] in 1999. Recently, They have created much attention particularly in the field of channel coding for 4th mobile communication system. They have a threshold within 0.0045dB of the Shannon limit of Additive White Gaussian Noise (AWGN) channel [3]. They have validity in mobile communication system which required of low error rate and high speed data rate, because its decoder has low complexity, which can be calculated parallel and which can be use iterative decoding scheme as turbo code. The advantage of LDPC codes are lower decoding complexity, but its’ disadvantage is higher encoding complexity to get the optimal LDPC codes.
LDPC codes are classified into regular and irregular one in according to weight on parity-check matrix, H. Regular LDPC codes have same row weight and same column weight on parity-check matrix, H, and Irregular LDPC codes have

different row weight and different column weight on paritycheck matrix, H. LDPC codes are also classified into binary and non-binary one in according to elements on parity-check matrix, H. Binary LDPC codes are constructed from a given alphabet of elements consisting of two elements(0 and 1), and non-binary LDPC codes are constructed from the finite with q elements(denoted GF(q)), where q=2p for some integer p. We can select elements from(0,1,…,q-1), q>2 [4]. Non-binary LDPC codes have better BER performance than binary, But it has weak point which has complexity to calculate Galois field operation.
In general, BER performances of irregular LDPC codes are better than that of regular LDPC code[5], however irregular LDPC code increase complexity of encoder. Irregular LDPC code have merit which have not same Row weight, therefore we are free to construct of parity-check matrix, H with highgirth(minimum cycle) distributions. On the other wise, Regular LDPC code have weak point that we are not free to construct parity-check matrix, H with high-girth distribution.
It is suggested that using an approximate triangular paritycheck matrix to construct LDPC codes can reduce the encoding complexity significantly without performance degradation [6]. But, It takes a long time to reconstruct parity-check matrix, H to satisfy non-singular matrix, and its procedure is too complex and irregular to implement in hardware.
In this paper, we propose low complexity encoding of regular LDPC codes to implement in hardware. We introduce to a pivoting and bit-reverse (PABR) algorithm for its low complexity encoding. We describe the way to construct rapidly non-singular parity-check matrix, H using a PABR algorithm. And we propose improved regular LDPC codes in BER performance which has parity-check matrix, H constructed submatrix using costas array[7].
We show approach to implementing LDPC encoder to implement in hardware. And we verify functionality and performance of LDPC encoder with LDPC decoder using sumproduct algorithm on hardware platform for 4th mobile communication system.

An (n,j,k) LDPC code is specified by a parity check matrix, H having m rows, n columns and j 1’s per column. In this paper, j=3 and k=6, all the parity check matrix will have 3 ones per column, and 6 ones per row. The code formed form such a parity- check matrix is known as a regular Gallager code. But its performace is inferior to our proposed matrix. This parity-check matrix will be referred to as Costas array parity-check matrix, H.
Costas array is nxn pattern of n dots with one dot in every row and column having the property that any horizontal and vertical shifting would result in at most no dot position agreement. We can construct Costas array of n=p-1 for prime number p by Welch’s constructing method. We can first choice primitive element, a of GF(p), and then put a dot(‘1’) in the cell (i,j) of the nxn array only if i=ai(mod p) in 1 < i < n, 0 < j < n-1.
For example, ai(mod 11) can be solved to give 1, 2, 4, 8, 5, 10, 9, 7, 2, 6 in p=11, a=2. this Costas array is depicted in Fig.1.

Nevertheless A swapping is wholly proceeded from adjacent column and row up to the last column and row, a pivoting may fail. In that case, we can use bit-reverse method.
In this paper, we propose a Pivoting and Bit-Reverse (PABR) algorithm to make a non-singular matrix, A. It is feasible to make rapidly a non-singular using PABR algorithm. In first, PABR algorithm performs a pivoting procedure and then bit-reverse procedure.
A bit-reverse procedure is not independent but associate with the result from a pivoting procedure. The following depict a bit-reverse method for matrix A, which can be performed when a pivoting fails in Fig.5.

Bit-reverse method exchanges element as a bit reversed. That is, if its element is 0, exchange to 1, and if 1, exchange to 0.

By bit-reverse method, the result of a pivoting is also satisfied with keeping regular LDPC code done. In this paper, we consider a (3,6)-regular LDPC code with row weight, 6, and column weight, 3. So, we must exchange diagonal elements too. Thus, We can reverse each diagonal element not to vary row weights and column weights on H matrix.
A bit-reverse procedure will be searched to +y direction and +x direction.
Using PABR algorithm, we result in the effect of the eliminating of 4-cycle on H matrix and constructing rearranged H matrix from Costas array parity-check matrix with high-girth distribution.
