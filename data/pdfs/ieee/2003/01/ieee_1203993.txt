A Runlength Limited Low-Density Parity-Check Coding Scheme
Bane Vasic, Senior Member, IEEE and Karunakar Pedagani Department of Electrical and Computer Engineering 1230 East Speedway Boulevard Tucson, AZ 85721, USA E-mails: {vasic, karun}@ece.arizona.edu

Abstract.—In this paper we propose a novel approach to modulation and error control coding. The idea is to completely eliminate a constrained code and, instead, impose the constraint by the deliberate introduction of bit errors before transmission. The redundancy that would have been used for imposing the constraint is used in our scheme to strengthen the error control code (ECC), in such a way that the ECC becomes capable of correcting both deliberate errors as well as channel errors that occurs during the detection. The proposed ECC-modulation scheme is based on iterative decoding of low-density parity-check codes (LDPC) and a runlength constraint.
Index Terms—Inverse concatenation.
I. INTRODUCTION
Constrained (or modulation) coding is used in communication systems to translate an arbitrary sequence of input data to a channel sequence with special properties required by the physics of the transmission (or recording) medium [11]. One of the most common constraints, so called run-length (RLL) constraint [10], bounds the minimal and/or maximal lengths of consecutive like channel symbols, which improves timing recovery, reduces intersymbol interference in channels with excess bandwidth and reduces transition noise in nonlinear media. Most of the constraints used in practical channels have finite memory and can be represented by sofic systems [11]. A sofic system [1] is defined as a set of sequences generated by walks on a directed labeled graph. A graph representation of a constraint enables one to analyze statistical properties of a code [5], [10], but also provides a systematic procedure for code construction known as a state splitting algorithm [1], [11]. The results of a state splitting algorithm are the encoder in the form of a finite automaton and a sliding window decoder with limited error-propagation. The encoder translates the sequence (d (n) )n>0 of input words
of length p to a sequence (a(n) )n>0 of code words of length q. We refer to the ratio R=p/q as a code rate. The graph capacity (or Shannon noiseless channel capacity [12]), C, gives the maximal achievable rate of a code designed by the state splitting algorithm. (See [11] and [3] for complete theoretical characterization of the state splitting). Channel constraints are imposed in encoder by the introduction of a redundancy into a

The construction of high rate codes (that are the most interesting in practice) is especially difficult when the starting constrained graph is large. Since the automaton complexity grows rapidly with graph size resulting encoder and decoder obtained after several rounds of state splitting are complex as well. (The intractability of state splitting algorithm is discussed in [2].) Moreover, the decoding window size also grows as the number of rounds of state splitting increases [3], [4], which may cause unacceptable error propagation. Due to inherently nonlinear nature of the encoding and decoding algorithm, the effect of error multiplication in the decoder occurs. In order to limit error propagation in the decoder, block codes are typically used. The code redundancy is exploited more effectively in long block codes, but the code complexity rapidly increases with the block length, because, very often, the state splitting algorithm requires many rounds of splitting. Another problem in practical systems is that the errors multiplied in the decoder may overwhelm the error correcting system and cause the error correction failure. Besides the remarkable effort that has been devoted to the development of the constrained code theory, code construction remains to be difficult to be fully automated, and a code construction requires ingenuity and skills in the art.
In this paper we propose a novel coding approach to modulation and error control coding. We focus on maximum runlength constraint (k-constraint) although the concept can be extended to any type of constraint. The idea is to completely eliminate a k-constrained code and, instead, to impose a kconstraint by the deliberate introduction of bit errors before transmission. The redundancy that would have been used for imposing the constraint is used in our scheme to strengthen the error control code (ECC), in such a way that the ECC becomes capable of correcting both deliberate errors as well as channel errors that occurs during the detection. We consider an error control scheme based on iterative decoding of low-density parity-check codes (LDPC), a technique of great promise that is being extensively studied [13-16]. We demonstrate that the redundancy invested into LDPC error correction capabilities works, and that the novel approach outperforms the classical modulation-ECC concatenation scheme. II. BIT-FLIPPING BASED CONSTRAINED CODING SYSTEM
The channel coding portion of the block diagram of the state-of-the-art transmitter is depicted in Fig 1(a). Prior to the transmission through the channel, (for sake of simplicity we consider the additive white gaussian noise channel (AWGN) model), a data word d is encoded by k-constrained and error control encoders. A (0, k) runlength constrained encoder converts a sequence of unconstrained binary words to a sequence of constrained words, such that maximum number of 0s between two 1s is ‘k’. The k-constraint removes the possibility of occurrences of long strings of 0s, thus enhancing the timing recovery from the received modulated waveform.
The traditional implementation of error control coding subject to runlength constraint coding is illustrated in the Fig 1(a). A user data stream is first encoded in the error control encoder (in recording applications this is typically Reed Solomon (RS) code), and a runlength constraint is imposed on the ECC coded data. The runlength code establishes a nonlinear dependence between the data before and after the modulation coding. Because of this non-linear dependence, even a small amount of errors in this coded data can result in error multiplication and can greatly degrade the Reed Solomon decoder’s performance.

Consider now the alternative scheme shown in Fig. 1(c). The LDPC encoder encodes the data word d, and deliberate errors are introduced in the codeword to satisfy the runlength constraint. Whenever the constraint is not satisfied (as detected by the RLL monitor block), the bits in the data word are inverted in such a way that runlength constraint is satisfied with minimal number of inversions.
The part of the block diagram in Fig. 1(c) after LDPC encoder can be treated as an equivalent channel, in which, besides the Gaussian noise, hard bit errors are added to the transmitted bits. The bits are therefore corrupted by noise but sometimes are inverted as well. The ECC must be designed to be able to handle both noise and the deliberate bit flips.
The errors introduced due to these deliberate flips depend only on the input data and are independent of the operating SNR. A different code word is decoded when the total number of errors crosses the maximum number of errors a particular ECC can handle. Consequently, the distance properties of the code have greater impact on the decoder’s performance in the present scheme, especially at higher SNR values. V. CONCLUSION
The application of maximum run length constraint by deliberately inverting some of the input bits is considered. An increase in the floor value of bit error rate (BER) is seen with increase in the severity of the constraint. For a practical kconstraint, this scheme gives impressive results over traditional Reed Solomon codes with run length encoder and over its inverse concatenation. The performance is degraded with increasing SNR value because of the error floor. Increase in the minimum distance of the code suppresses the floor. By properly choosing the distance of a code, better results are obtained in a particular operating range of noise levels.
