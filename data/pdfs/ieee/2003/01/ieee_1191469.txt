I. INTRODUCTION
LDPC code proposd by Gallager in 1962 is a linear block code whose parity check matrix is composed of '0' elements domiiantly [I]. But, since realization was regarded to be impossible in those days, it had been forgotten for a long time until Mackay rediscovered it in 1996 (21. LDPC code shows good error correcting capability with iterative decoding by the sum-product algorithm [4]. For example, irregular LDPC code with
block length of IO6 and d e rate of U2 approaches to Shannon limit closer than turbo code 131. The perfor-
mance of LDPC code is affected by block length and distribution of 1's in panty check matrix [l],[Z]. So the methods to get the optimum parity check matrix have been studied,but the results are not satisfactory up to now. Thus,bit probabilitytransition characteristics are analyzed to determine the parameter which should be considered to obtain the optimum parity check matrix in this paper.
II. L D P C C O D E
Parity check matrix of LDPC code is composed of M rows and N wlumns, so the message length is K=N-M ' and, code rate is R=KM. In addition, the number of 1's in rows and columns of parity check matrix is very small
compared to block length N. Regular LDPC code has a
fixed number of 1's and irregular LDPC code bas variable

number of 1's in rows and columns of parity check matrix [21.
LDPC code can be decoded by probability propagation which is known as sum-product algorithm or belief propagation algorithm [Z]. And decoding process can be represented by Tanner graph with bit and check nodes [4]. Bit and check nodes correspond to columns and rows of parity check matrix respectively. If an element of parity check matrix is 1, the corresponding hit and check node are connectedwith an edge. For example, if aparity check matrix H is given as (I), Tanner graph of the parity check matrix is represented in Fig. 1.

The probability of hit node which is used to decide the correspondingbit is updated by the information delivered from the check node which is wnnected to the bit node in Tanner graph. And the updated bit probability is forwarded to check node which is connected to the bit node. Iterative decoding can be possible with this process in LDPC d e .
It is known that if there is a short cycle which is a Imp structure starting a bit or check node and ending the same bit or check node in Tanner graph, the iterative decoding is disturbed. So the cycle length of 4 should be removed in parity check code [ 5 ] .

Simulation was done in AWGN channel to investigate bit probability transition in iterative decoding with sumproduct algorithm. Though the longer block length gives the better performance in general [l],30x50 parity check matrix having three 1's in each column [5] is used to make analysis simple. And the number of iterations is set to 300.
Bit probability transitions can be classified into three types, namely Type I, 11, and 111 as results. Type I converges to 0 or 1, Type 11 converges to a value which is not 0 or 1, and Type I11 doesn't converges to any value but oscillates between 0 and 1. Each bit probability transition in the same codeword shows the same transition type in these types. It is due to that each bit in a codeword is affected by each other connected in Tanner graph in decoding process.
Fig. 3, 4, and 5 are the examples of three transition types. Type I usually shows no error but sometimes it results in error. And Type I1 and Type 1II show almost errors. The cause of these types is analyzed in order to reduce the errors in this paper.

IV. CONCLUSION
Bit probability transition characteristics are analyzed in this paper to investigate the characteristics ofiterative d d i n g of LDPC code. Bit probability transition shows three types after sufficient iterative decoding is done. Type 1 converges to 0 or 1, Type I1 converges to a value which is not 0 or 1, and Type 111oscillatesbetween 0 and 1. And it is found that each bit in the same block bas tbe same transition characteristic. In addition, Type I1 is
dominant in low SNR,but Type Ill becomes dominant as S N R increases.
The evident relationship between bit probability transition types and dismiution of initial probabilities are not observed. Rather, the probability transition types may
be dependent on the structure of parity check matrix.
Simulation results show that if parity check matrix is constructed to make no 6-cycle in check node which bas the highest degree, the e m r correcting capability of LDPC code can be improved.
Therefore, &cycle with the highest degree check node should be avoided in the parity check matrix to improve the performanceof LDPC code.
