### B. 알고리즘 요약

1. 대상은 QC-LDPC 블록코드(순환행렬 기반 parity-check matrix)와 이를 "unwrapping"하여 얻은 시간불변 LDPC 컨볼루션 코드(semi-infinite parity-check matrix, syndrome former memory `m_s`) 쌍.
2. 문제: message-passing iterative(MPI) 디코딩은 국소적(local) 연산이라 Tanner graph의 유한 커버(finite cover)를 구별하지 못해, ML 유니온 바운드보다 나쁜 성능(pseudocodeword로 인한 디코딩 실패)을 유발.
3. 핵심 가정/모델: 이진 선형 코드, binary-input output-symmetric 채널(AWGNC/BSC/BEC)을 가정하고, 모든 all-zero codeword 대칭성을 이용해 fundamental polytope/cone 중심으로 분석.
4. 핵심 기법 1단: QC 코드의 parity-check 다항식 행렬 `H(x)`을 `x→D`로 치환해 컨볼루션 코드의 `H(D)`로 "unwrap"하고, 반대로 `D→x^r` (r은 circulant 크기의 배수)로 "wrap"하는 대응 관계를 형식화.
5. 핵심 정리(Theorem 30/31): 조건(Assumption 23)이 성립하면, 컨볼루션 코드의 임의의 pseudocodeword `ω(D)`를 wrap한 벡터가 QC 코드의 pseudocodeword가 되며, AWGNC/BSC/BEC/max-fractional pseudoweight 모두 `w(ω(D)) ≥ w(ω_wrapped)`가 성립 — 즉 컨볼루션 코드의 최소 pseudoweight가 QC 코드보다 크거나 같음.
6. 확장: 유한 truncation을 위해 column distance/row distance 개념을 pseudoweight에 대응시킨 "truncated pseudoweight"·"bounded pseudocodeword pseudoweight" 두 수열을 정의해 컨볼루션 코드의 최소 pseudoweight를 상하한으로 조여감.
7. 부수 기법: LP(linear programming) 디코더의 판정 경계와 pseudoweight의 연결을 이용, BSC에서 bit flip 개수가 `w_BSC(ω)/2` 미만이면 LP 디코딩이 성공함을 증명(Theorem 32).
8. 검증: vertex enumeration 프로그램(LRS)으로 소규모 QC 코드의 정확한 최소 pseudoweight를 계산하고, 대형 코드는 하한/상한 bound로 근사.
9. 결과(수치): 예시로 rate 1/4, (3,4)-regular 코드에서 QC 블록코드 대비 unwrapped 컨볼루션 코드가 저~중 SNR에서 0.5~1.0 dB BER 개선을 시뮬레이션으로 확인, circulant 크기를 키울수록 블록코드 성능이 컨볼루션 코드에 근접.
10. 한계: 순수 이론/조합론적 분석(polytope, vertex enumeration)이며 실제 디코더 알고리즘(min-sum 등)이나 HW 구현은 다루지 않음, 결과는 소규모 예제 코드의 시뮬레이션에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC ↔ 컨볼루션(SC-LDPC 전신) unwrap/wrap 대응 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 변경(부호설계) 범주이나 실제 구현 대상이 아닌 순수 이론 |
| pseudoweight/pseudocodeword 분석(error-floor 원인 규명) | 대응 없음 | Prime ECC는 error-floor를 trapping-set/CRC 조기종료로 다루며 polytope 기반 pseudoweight 분석 도구 없음 |
| LP(linear programming) 디코더 | 대응 없음 | Prime ECC는 min-sum 기반, LP 디코더 미지원 |
| MPI(sum-product) sliding-window 디코더 | `decoder.cpp` `LDPC_Decoder()` | 개념적으로만 연결(SPA 기반), Prime ECC는 min-sum이라 직접 대응 약함 |

적용 가치: **낮음** — 정보이론 저널의 순수 수학적 pseudoweight 분석으로, 구체적 디코더 알고리즘·양자화·HW 설계와 무관하며 Prime ECC의 QC-LDPC 고정 구조에 그대로 이식할 실무적 요소가 없음.
