> 총평: row-column 연산을 파이프라인으로 겹쳐 iteration 수와 BER을 개선한 2006년 FPGA 스케줄링 연구로, 스케줄링 아이디어는 참고 가능하나 부호·양자화 스펙이 Prime ECC와 달라 이식성은 제한적.

### B. 알고리즘 요약

1. (3,6)-regular LDPC, codeword length 3336bit, AWGN 채널 대상 partially-parallel 디코더.
2. 기존 스케줄(Refs.[2]-[4])은 row/column 연산을 dual memory로 동시 수행하나, column 연산이 갱신된 메시지 `α'`를 즉시 반영하지 못해 message-passing 효율이 저하됨.
3. 채널모델은 AWGN, block-structured parity check matrix(`w_c x w_r` 서브블록, 각 서브블록은 `b x b` 순환 시프트 행렬).
4. 제안 스케줄: row 연산이 먼저 column 연산 위치를 결정하고, 그 직후 해당 위치에서 column 연산을 수행하는 2단계 구조.
5. row 연산 완료 시점마다 갱신 메시지 `α'`를 이용해 매 column 연산을 즉시 수행 - 기존 대비 3배 많은 column 연산을 단일 iteration 내 수행.
6. 파이프라인 BFU(Bit Functional Unit)가 3개 column 연산의 덧셈을 병렬 수행해 6클록 내 처리, row 연산(10클록)의 지연을 확장하지 않음.
7. CFU는 Gallager 함수 대신 근사 minimum 함수(min-sum)를 사용해 게이트 수 절감.
8. 별도 학습/최적화 절차 없음 (스케줄링·HW 구조 설계).
9. 결과: 기존 스케줄 대비 평균 iteration 수 감소(3.983→3.063 @Imax=4,SNR=5.0), throughput 최대 112Mbps, BER 0.000000(SNR=4.5, Imax=8)까지 개선.
10. 한계: (3,6)-regular 코드 및 2006년 FPGA(Xilinx Virtex-II)로 제한적 검증, 최신 QC-LDPC/고rate NAND ECC와 직접 비교 어려움, 본문 OCR 손상으로 정량적 세부수치(gatecount 등) 다수 확인 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| row/column 연산 파이프라인 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | iteration 루프 스케줄링 개선 아이디어로 참고 가능 |
| CN min-sum(근사) 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 Prime ECC가 min-sum 보유, 중복 기여 |
| 메모리 뱅크 구조/주소지정 | 디코더 HW 모델 (z=32, HCU, GT) | 메모리 뱅크 분할 방식이 상이해 직접 대응 어려움 |
| block-structured PCM (QC 유사 서브블록) | `ecc_top.cpp` `Load_PCM()` | (3,6)-regular 구조로 Prime ECC의 QC-LDPC와 상이, 부분 대응 |

> 적용 가치: 낮음 | (3,6)-regular 코드 기반 2006년 스케줄링 연구로 이미 Prime ECC가 min-sum·pipeline·HCU 등 유사 개념을 보유하고 있어 중복도가 높고, 코드 구조 차이로 직접 이식 난이도가 큼.
