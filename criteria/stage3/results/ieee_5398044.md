### B. 알고리즘 요약 (10줄 내외)

1. WiMax(802.16e) block-structured QC-LDPC, 최대 length 2304, cluster size `z=24~96`, rate 1/2 사례 구현.
2. 문제: LDPC 디코더 수작업 RTL 설계는 표준마다 다른 파라미터에 대응하기 어렵고 개발 기간이 김.
3. 해결: 고수준 합성(HLS) 툴 PICO를 사용해 untimed C 알고리즘 코드에서 직접 RTL을 자동 생성.
4. 핵심 알고리즘은 layered scaled min-sum: `Rmn = 0.75 × sign(Qmj) × min|Qmj|` (Sum-Product 근사, scaling factor 0.75 고정).
5. `Qmn = Pn - Rmn`으로 사전처리 후 최소/차소 최소값 기반 `Rmn` 갱신, `Pn = Qmn + Rmn`으로 갱신.
6. 두 아키텍처 제안: (1) per-layer 2-stage(core1/core2) 스케줄링, (2) 레이어 간 파이프라이닝하는 multi-layer pipelined 구조(코어 활용률 50%→개선).
7. 파이프라인 위험 방지를 위해 scoreboard 변수로 core1/core2 간 read-after-write 충돌을 검출·스톨.
8. `#pragma unroll` 지시어로 병렬도(z=48, 96 등)를 C 코드 레벨에서 조절해 스케일러블 데이터패스 생성.
9. 결과: TSMC 65nm에서 core area 1.2mm², 400MHz에서 415Mbps, peak power 180mW; clock-gating으로 sequential internal power 29% 절감(64.5mW→46.1mW).
10. 한계: 디코딩 알고리즘 자체는 개선 없이 기존 기법 적용, 정정성능(BER) 결과 없이 오직 면적/전력/처리량만 보고.
