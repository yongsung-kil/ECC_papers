> 총평: flooded Min-Sum binary QC-LDPC 디코더(circulant 54, 4bit LLR)를 대상으로 한 오버클럭 timing-error 신뢰성 분석 방법론으로, 디코더 성능/부호 개선 기여는 없어 이식성 하이나 QC-LDPC MS HW라는 점에서 NAND관련성 간접.

### B. 알고리즘 요약

1. 시스템: (3,6)-regular QC-LDPC(길이 1296, rate 1/2, circulant size 54, base 24x12), flooded Min-Sum 디코더 RTL, 입력 4bit 양자화 LLR, hard-decision 출력.
2. 문제: near/sub-threshold 전압 스케일링 시 process/PVT 변동으로 timing error가 확률적으로 발생 - 오버클럭 디코더의 신뢰성을 정확히 평가할 방법 필요.
3. 모델: 각 표준셀 전파지연을 Inverse Gaussian(IG) PDF로 모델링, 클록주기(timing constraint)에 대한 PO 오류확률로 fault map 구성.
4. 핵심 기법 1단: Monte-Carlo SPICE 기반 SSTA로 표준셀(INV/NAND2/DFF, PTM 45nm@0.35V) IG PDF 특성화.
5. 핵심 식: combinational block PO의 IG 파라미터를 critical path 상 셀들의 선형합성(mean·shape의 `2·INV+3·NAND2` 식(1))으로 근사, CDF로 주어진 지연의 오류확률 산출.
6. 핵심 기법 2단: RTL에 확률적 saboteur(switch 검출+RNG+XOR) 삽입, 각 PO의 CDF 오류확률로 데이터패스에 fault injection(제어부 제외).
7. 구현 디테일: 디코더 HW - 54 VNU/CNU, α/β 메시지 메모리, 6단 MUX barrel shifter 라우팅, 압축 β(15bit), BS 1 pipeline/VNU 4단/CNU 3단.
8. 학습/최적화: 없음(분석 방법론). Cadence Spectre/Encounter, Synopsys DC, Modelsim 상용툴 활용.
9. 결과: BiAWGN SNR 1~3dB. tclk 5.5→3.1ns는 성능손실 없음, 2.5ns(400MHz)에서 <0.1dB 저하, 2.2ns 유의미 손실, 1.9ns 복호불가. 오버클럭 2배까지 정정능력 무손실.
10. 한계: 디코더/부호 성능 개선 기여 없음, timing-error 신뢰성 평가에 국한, throughput 미보고, flooded MS·rate1/2 특정 부호 대상.
