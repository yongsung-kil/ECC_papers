### B. 알고리즘 요약

1. 대상: binary QC-LDPC (IEEE 802.16e WiMAX), 부호길이 `576`, block column = 24×24 (expansion factor `q=24`), 6개 rate(1/2, 2/3A/B, 3/4A/B, 5/6) 지원, BPSK/AWGN.
2. 문제: 다중 rate를 단일 유연 아키텍처로 지원하면서 layered로 iteration을 줄여 표준 요구 throughput을 맞추는 것.
3. 핵심 기법: min-sum 기반 **column-layered decoding** — parity check matrix를 B block column 그룹으로 나눠 그룹 단위(G0..G_{B-1}) 순차 CNU/VNU 갱신, block column을 그대로 그룹으로 사용.
4. VNU: `L(v→c) = Iv + α·Σ R(c→v)`, normalized min-sum scaling `α` 최적 0.8이나 HW 단순화 위해 `α=0.75` 사용.
5. CNU: `R = S × r`, `r = min(min_{va} L^{(k)}, min_{vb} L^{(k-1)})` — va는 이미 갱신된 그룹, vb는 이전 iteration 값 사용(column-layered 특성).
6. two-min-sum: 각 CNU가 최소 2개 `|Lcv|` (m1,m2)와 index (i1,i2, 각 5bit), sign 곱 Scn을 25bit 레지스터에 저장, CNU를 A(임시벡터 생성)/B(비교·갱신) 2단으로 분리.
7. 채널 soft값은 sign-magnitude 8bit 양자화(MSB=sign); v→c 메시지는 저장 불필요(VNU에서 재생성)해 메모리 절감.
8. 아키텍처: 96 CNU + 24 VNU 완전병렬, right/left barrel shifter로 sub-matrix shift index 라우팅, 단일 유연 controller가 iteration수·주소 관리, 음수 shift index 블록은 비활성.
9. 결과: 130nm CMOS ASIC(SYNOPSYS/CADENCE), area 0.96mm², clock 39.9MHz(임계경로 ~35%가 shifter), info throughput 95.76~159.6Mbps(5/10 iter), block column당 1 클록.
10. 한계: WiMAX 소형부호 고정, shifter 임계경로로 clock 저속·non-pipelined, 정정능력 자체 개선 아님(구현·convergence 위주).
