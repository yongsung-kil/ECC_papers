### B. 알고리즘 요약

1. 시스템: BIAWGN/BPSK, 짧은~중간 길이 binary LDPC(예 (120,56) r=0.467, (273,191) r=0.7, (504,252) PEG, (255,175) EG r=0.686)를 LP 디코딩.
2. 문제: Feldman LP 완화(fundamental polytope)의 barrier function 복잡도가 check degree `wr`에 대해 `O(N·2^wr)`로 지수 증가 → 고row-weight 부호에 비현실적.
3. 모델: ML 디코딩 정수계획을 LP 완화 `min γ^T x s.t. x∈P`, γ=음의 LLR. barrier method로 무제약화 `ψ=tf+φ`, centering(t↑)+Newton으로 해결.
4. 핵심기법: check node m당 `2^(wr-1)`개 parity 부등식 중 **최대 1개만 위반 가능**(Thm4.1, [6])이라는 사실 이용.
5. 핵심식: 근사 barrier `φ̃ = -Σ log(-σ_m) - box항`, `σ_m = max_V u(x)` (식15,16) — check당 유효 constraint를 1개(`I_m`)로 축약.
6. 확장: gradient(식22)·Hessian(식24)도 지배 항만 남겨 `O(N·2^wr)→O(N·wr)`로 축소, Algorithm 4로 `2^(wr-1)` subset 열거 없이 `I_m,σ_m` 계산.
7. 구현디테일: fractional value quantization(x≥0.5→1)로 조기 hard-decision, syndrome `s=z·H^T=0`이면 조기종료. 초기점 x=(1/2,...)가 wr≥3에서 strictly feasible.
8. 최적화: backtracking line search(α=0.3,β=0.5), μ=10~20, t0≈20, outer≤5·inner≤20.
9. 결과: 성능 저하 없이 표준 IPD 대비 실행시간 약 5~10배 단축, wr=16 EG부호는 표준 IPD로 불가하나 제안법으로 가능. 중~고 SNR(error-floor)에서 BP 상회.
10. 한계: HW 미설계·시뮬만, short/medium 길이, gate/latency 수치 없음, Newton(Hessian 역행렬)·부동소수 연산.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP interior-point 디코딩 (Newton/barrier) | decoder.cpp LDPC_Decoder() | 낮음 (Prime은 min-sum 반복복호, LP와 알고리즘 이질) |
| LLR(γ) 입력 / soft 정보 | channel.cpp Set_LLR_Th() | 낮음 (LP는 연속 LLR·부동소수 요구) |
| syndrome 기반 조기종료 (s=z·H^T=0) | full_CRC.cpp / 조기종료 로직 | 낮음 (Prime은 CRC 조기종료 이미 보유) |
```

적용 가치: **낮음** — LP interior-point 디코딩은 error-floor에서 BP를 능가하나 Newton step·Hessian 역행렬·부동소수 기반이라 min-sum RTL을 모사하는 Prime ECC 3.1의 HW 구조에 이식 불가에 가깝고, HW 검증도 없음.
