### B. 알고리즘 요약

1. Reed-Solomon(RS) 코드를 AWGN 채널 + BPSK/QAM 등 multilevel 변조로 전송, symbol 단위 soft-decision iterative decoding을 목표.
2. 기존 bit-based adaptive BP(ABP, Jiang & Narayanan)는 RS 코드를 binary 이미지로 확장해 처리하므로 긴 코드·큰 GF(q)에서 복잡도가 지나치게 큼.
3. RS 코드의 parity-check 행렬 `H_S`는 dense(비-sparse)하므로, BP를 적용하려면 신뢰도 낮은 symbol에 대응하는 부분 행렬을 대각화(diagonalize)해 sparse화하는 전처리가 필수.
4. 핵심 기법: 채널로부터 얻은 symbol 신뢰도 `P(c_j|y_j)`를 오름차순 정렬해 최소신뢰도 symbol들의 열이 대각(diagonal) submatrix가 되도록 Gauss-Jordan(GF(q)) 또는 dual RS 코드의 erasure decoding으로 매 iteration마다 `H_S^(l-1)` 재구성.
5. check node 갱신은 FFT 기반 BP(`r_ij = IFFT{∏ FFT[q_ij'(s)]}`, 식 12)로 계산, `q=2^m`일 때 DFT 복잡도를 `q^2`에서 `mq`로 축소.
6. 기존 방식과 차이: 모든 연산(정렬, 행렬 대각화, message 갱신)을 symbol 단위로만 수행해 bit-to-symbol/symbol-to-bit 변환을 완전히 제거(bit-based는 첫 반복 후 binary로 되돌아가 후속 반복을 bit 단위로 수행).
7. 별도 학습/최적화 절차 없음, 매 iteration 신뢰도 재정렬 및 대각화가 곧 adaptation 과정.
8. 결과: RS(31,25) AWGN/BPSK에서 HDD 대비 FER=10^-4 기준 1.3dB 이득, bit-based ABP 대비 성능은 0.9~1.0dB 열세이나 동일 FER 도달 시간은 bit-based 대비 약 3~3.6배 빠름.
9. iteration 5~20 범위에서 symbol-based ABP는 성능 변화가 0.2dB 이내로 안정적(bit-based는 iteration 수에 민감).
10. 한계: HW 구현 없음(시뮬만), RS 코드 전용(LDPC 아님), non-binary GF(q) 연산 기반이라 binary decoder로의 직접 이식 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| symbol-level(GF(q)) FFT 기반 BP 복호 전체 | 대응 없음 | Prime ECC는 binary LDPC 전용, non-binary/RS 복호기 미보유 |
| 신뢰도 기반 parity-check 행렬 adaptive 대각화 | 대응 없음 | H-matrix는 고정 QC 구조(`ecc_top.cpp` `Load_PCM()`)이며 매 iteration 재구성 개념이 없음 |
| soft-info 활용(채널 LLR/확률) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 채널에서 soft 정보를 받는다는 개념만 유사, 구체 알고리즘은 무관 |

> 적용 가치: 낮음 — Reed-Solomon(non-binary, dense H) 전용 알고리즘으로 Prime ECC의 binary QC-LDPC 구조 및 min-sum 디코더와 접점이 없음.
