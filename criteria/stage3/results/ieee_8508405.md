### B. 알고리즘 요약

1. 대상: 자기기록(MR)용 `SC-LDPC`(=LDPC convolutional) 부호, protograph base matrix `B`를 `K`회 복제·edge spreading으로 유도, code rate `0.8`, PEG로 `H` 생성.
2. 문제: 섹터 크기 증가(>4KB)로 SC-LDPC가 유망하나 sliding-window decoding(UWD)의 복잡도가 큼 - VN 업데이트 수를 줄여야 함.
3. 모델: turbo equalization - `BCJR` detector가 ISI(intersymbol interference)를 처리하고 LLR을 window 크기 `W=t·M`의 SC-LDPC 복호기에 전달, 각 decoding position이 code block `y_q` 담당.
4. 핵심기법 1 (DSWD): 각 code block의 추정 soft-BER가 유사하면 여러 block을 동시에 shift. shift factor `p_t`를 `1`~`K_dd` 범위에서 선택.
5. 핵심식: `c_t·log P(t) - log P(t) < ... ` (soft-BER 로그값 조건) 만족 시 block `t`를 shift, `c_t`는 Monte-Carlo로 결정.
6. 핵심기법 2 (DSNWD): NWD(non-uniform WD)를 수정해 DSWD를 강화 - window 내 일부 VN 업데이트를 조기 중단, shift된 block 수 `p_t`에 맞춰 update schedule 조정.
7. 구현: soft-BER는 복호 종료 시 `log P(t)` 항으로 추정, position별로 계산.
8. 최적화: shift factor `c_t`를 Monte-Carlo 시뮬로 튜닝.
9. 결과: BPMR 채널, `F_LDPC`/`F_Turbo` iteration 설정에서 DSWD가 UWD/NWD 대비 각각 복잡도 감소, DSNWD 적용 시 최대 감소 - 성능 손실 없이 평균 VN 업데이트 수 `R_avg` 감소.
10. 한계: HW 미설계, 시뮬만, SC-LDPC + 자기기록 turbo equalization 전제 (NAND와 무관), 정확한 복잡도 수치는 그림에만 존재해 본문 확정 불가.
