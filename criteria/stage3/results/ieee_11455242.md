### B. 알고리즘 요약

1. 시스템: 5G NR protograph LDPC(BG1/BG2), lifting factor Z, 다양한 code rate(예: BG1 최소 rate 약 1/3~0.917)와 2^q-QAM(q=2,4,6,8,10) BICM 변조.
2. 문제: few-iteration(URLLC/eMBB 지연 요구) layered decoding에서는 5G 표준 puncturing이 최적이 아니며(원래 대반복 가정 설계), 기존 puncturing 최적화는 변조·rate 호환성이 없고 greedy search(SPAT)로 지역 최적해에 수렴.
3. 핵심 가정: 채널은 AWGN 기반 BICM(2^q-QAM), layered(row-layered) 스케줄링이 정보 전파 경로를 바꾸므로 기존 flooding 기반 PEXIT 해석은 부정확.
4. 핵심 기법 1단: layered decoding neighborhood를 트리로 근사(Z→∞일 때 트리 수렴 확률 1, Theorem 1)한 revised(layered) PEXIT 해석으로 few-iteration 성능(decoding threshold `γ*`)을 정확히 예측.
5. 핵심 식: 상호정보량 갱신 `I(l_{j→i}) = Φ(Φ^{-1}(I(l_j)) - Φ^{-1}(I(l_{i→j})))` — layered 스케줄에 맞춰 레이어 단위로 순차 갱신(기존 flooding PEXIT과 갱신 순서가 다름).
6. 핵심 기법 2단: VN block 단위 puncturing 벡터 `β`를 progressive multi-path search(PMPS, Algorithm 1)로 최적화 — 기존 SPAT(pairwise swap greedy)의 지역 최적 함정을 과거 최적해 집합 `Bopt`+부최적해 집합 `Bsub`로 다중 경로 탐색해 회피.
7. 부수 트릭: 변조 호환성은 여러 modulation의 single-objective 해 사이 Euclidean distance 최소화(식 24)로, rate 호환성은 낮은 rate의 puncturing 집합이 높은 rate의 부분집합이 되도록 nested 최적화(Algorithm 3)로 구현. 실제 구현은 puncturing 순서 변경 없이 block interleaver(주소 재배치, sub-µs latency)로 삽입.
8. 최적화 절차: DE/GA/NN 없음, PEXIT 기반 결정론적 탐색(PMPS, 경로 폭 w=3 사용).
9. 결과: 5G 표준 puncturing 대비 최대 0.6dB 이득(few-iteration), IR-HARQ 결합 시 시스템 rate 최대 60.6% 향상(Eb/N0=3.5dB), 반복 횟수가 클 때는 성능 이득은 줄지만 평균 반복 횟수 감소.
10. 한계: HW 미설계(interleaver만 저복잡도로 언급), 5G NR 표준 규격(BG1/BG2, MCS 테이블, IR-HARQ) 및 BICM/QAM 변조 채널에 강하게 결속되어 NAND의 고정 rate·단일 채널 환경과 직접 대응 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN block puncturing 최적화(PMPS) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 punct_col_num=2로 고정된 puncturing 구조를 사용해 가변 rate matching 목적의 본 기법과는 대응 없음 |
| layered PEXIT 해석(few-iteration 성능 예측) | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 실측/시뮬 검증하며 PEXIT 같은 해석적 임계값 예측 도구를 사용하지 않음 |
| block interleaver(주소 재배치 HW) | 대응 없음 | 5G NR rate matching/IR-HARQ 전용 구조로 NAND ECC 파이프라인에 대응 모듈 없음 |
| layered(row-layered) 스케줄 자체 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 레이어(컬럼 그룹) 단위 갱신을 쓰지만 본 논문의 기여는 스케줄이 아닌 puncturing 설계이므로 간접적 관련 |

적용 가치: 낮음 — puncturing/rate-matching은 NAND 고전 binary QC-LDPC의 고정 rate 설계와 개념적으로 맞지 않고, BICM/QAM 채널 가정도 NAND 채널과 다르다.
