### B. 알고리즘 요약

1. 대상: 무선표준(WiFi 802.11n/ac, WiMAX 802.16e, WiGig 802.11ad, WPAN 802.15.3c)의 QC-LDPC PCM 집합, `z×z` circulant 서브행렬 base matrix `Hb`. LLR은 `W=4-bit` 2's complement.
2. 문제: 기존 FPGA 디코더는 완전유연(어떤 QC PCM이든 1클록 전환) 구현이 부재 - fully-parallel은 라우팅 폭발, serial은 throughput 부족.
3. 구조: partially-parallel, `P=Z/Q` VNPU (`Q`=병렬축소 인자로 자원/throughput trade-off), CNPU 수 `Pc=P/Gmin` (`G=nb/mb`).
4. 스케줄: modified flooding - block-row/block-column을 `z` 단위로 동시 처리, VND/CND 분리 데이터패스 병렬 동작.
5. 메모리: extrinsic LLR을 VMEM/CMEM 두 뱅크로 분리, 각 `NB` BRAM에 submatrix 단위 `P`개 LLR을 1워드로 묶어 저장 → BRAM 충돌 회피 (비선형 배치, Fig.5).
6. 라우팅: programmable barrel shifter(BS)로 shift값 `s`(0≤s<Z) 순환시프트, row/column-centric 표현 상호변환. BS가 다중 `z` 지원 시 자원 지배.
7. CN 연산: min-sum (`|min|` × 부호 누적곱), unused 입력은 VNPU=0 / CNPU=`2^(W-1)-1`로 turn-off. NPU는 log-depth adder/min tree로 critical path 최소화.
8. 고rate 최적화: 저degree CNPU 2개를 링크(`L=1`)해 고degree row 처리 - 고rate PCM의 적은 `mb`로 iteration당 클록 상쇄.
9. 자동화: offline design flow가 PCM집합→파라미터 추출→C++ 라이브러리로 SystemVerilog HDL 자동생성(Fig.11). Altera Stratix IV 합성, BPSK/AWGN, 최대 18 iteration, 목표 BER 10^-4로 특성화.
10. 결과: 단일PCM F1은 [45] 대비 throughput -63%지만 자원 -17%·에너지 0.6dB 우위·latency -33%. 전체12PCM F4는 [46] 대비 throughput +167%지만 자원 7.3배. 알고리즘 정정성능 개선은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CN 연산 (min×부호) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유(min1/min2+sign XOR) - 중복, 추가가치 낮음 |
| VNPU sum-tree / 4-bit 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, LLR 테이블 | 우리는 magnitude 양자화 테이블 기반, 표준 VN 합산과 유사 - 신규성 없음 |
| barrel shifter / VMEM-CMEM 라우팅·메모리 | 디코더 HW 모델 (z=32, GT) | HW arch 레이어 - 단, 우리는 고정 z=32 단일 PCM이라 멀티z BS 유연성 불필요 |
| 조기종료 (parity-check register) | `partial_CRC.cpp`/`full_CRC.cpp` | 우리는 CRC 기반 조기종료 보유 - 논문의 syndrome 방식보다 정교, 중복 |
| 다중 PCM ROM / run-time 전환 | 대응 없음 | Prime ECC는 단일고정 QC-LDPC - 멀티모드 전환 자체가 요구사항 아님 |

적용 가치: **낮음~중간** - 표준 min-sum QC-LDPC HW 설계로 정정성능 기여는 없고, 유일한 참고가치는 partially-parallel 메모리/BS 라우팅 아이디어인데 우리는 단일 z=32 고정이라 유연성 부분은 불필요.
