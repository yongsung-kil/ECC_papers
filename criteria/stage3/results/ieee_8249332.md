### B. 알고리즘 요약

1. 시스템: (γ,ρ)-regular Tanner 그래프 LDPC를 비신뢰 논리게이트로 만든 hard-decision 디코더(Gallager B, bit-flipping) survey. 예제 QC(155,64) rate 0.4, QC(768,192), PG(2,23), LS(155,64).
2. 문제: 전압 강하로 인한 게이트 고장(timing error)이 디코딩에 미치는 영향 — 기존 von Neumann 무상관 고장 DE는 arbitrary low error 불가라는 비관적 결론.
3. 핵심 모델: timing error = 게이트 출력이 이전과 바뀔 때만(`z(l)≠z(l-1)`) 확률 `ε`로 실패하는 data-dependent 상관 고장. `ε⊕`(XOR), `εMAJ`(MAJ) 구분.
4. 핵심 기법 1단: noisy Gallager B — timing error가 자주 스위칭하는 메시지(주로 trapping set 내부)만 교란 → trapping set 탈출 유도. 첫 iteration 무고장이면 codeword 무관·신뢰 디코더 수준 성능(Theorem 1, Corollary 1).
5. 식 의미: `Pr{e(l)=-1|z(l)≠z(l-1)}=ε`, 무변화 시 0 — 안정 메시지는 교란 안 되므로 correct bit 보존, trapping set만 흔들어 error-floor 개선.
6. 핵심 기법 2단: noisy bit-flipping — expander graph 인자로 최악 채널오류의 고정 비율 보증 정정, `|V1|<√(3(3+8)αn/32)−2CXOR` (Theorem 2), asymptotic Pn→0 (Theorem 3).
7. 구현 디테일: 첫 iteration 무고장(CXOR=0) 보장이 정정능력 선형 증가에 필수, 클록 주기 늘려 신호 안정화.
8. 학습/최적화: 없음(이론 정리 + Monte Carlo).
9. 결과: QC(155,64)에서 timing error가 구조적 결함 보상해 FER 한 자릿수 개선(LS 수준 근접), 수렴시간 큰 증가 없음(10~20 iter). γ≥8 코드 보증 정정 하한 표 제시.
10. 한계: hard-decision 전용, 실제 HW 미구현(이론+MC 시뮬 survey), expander 코드 실구성 난제, min-sum/soft 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager B / bit-flipping 반복 디코딩 | decoder.cpp LDPC_Decoder() | Prime ECC는 min-sum(soft) 기반 — hard-decision 패러다임 상이, 직접 이식 부적합 |
| trapping-set 탈출 / error-floor 개선 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 개념적으로만 관련(교란 기반), min-sum 구조엔 미적용 |
| noisy 게이트 timing error 모델 | 대응 없음 | Prime ECC는 신뢰 게이트 bit-exact 모사 — 비신뢰 HW 모델 없음 |
| expander 코드 / H-matrix 구조 | ecc_top.cpp Load_PCM() | 부호구조 재설계 필요, 고정 QC-LDPC와 부정합 |
```

적용 가치: **낮음** — hard-decision(Gallager B/bit-flipping) + 비신뢰 게이트 timing error 패러다임은 Prime ECC의 min-sum soft 디코더와 근본적으로 다르고, HW 미구현·부호 재설계 부담이 커 이식 이점 없음.
