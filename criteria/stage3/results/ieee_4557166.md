### B. 알고리즘 요약 (10줄 내외)

1. 시스템: BPSK + AWGN 채널, protograph 기반 irregular LDPC 코드(rate `R=4/9` unpunctured, `Rp=1/2` punctured, `R=2/3` 등 예제), belief-propagation 복호.
2. 문제: 기존 protograph 최적화는 모든 variable node에 동일 전송 에너지를 가정하여 threshold 개선 여지가 남아 있음.
3. 핵심 아이디어: protograph의 각 variable node `i`에 서로 다른 정규화 전송 에너지 `E_i` (`ΣE_i = n_p`)를 부여하는 "energy distribution" 개념 도입.
4. 핵심 기법: 각 variable node별로 다른 채널 확률밀도함수 `f_i(y)`(에너지 `E_i` 반영된 AWGN LLR 분포, 식 (2))를 사용하는 discretized density evolution 일반화로 threshold `v*` 계산.
5. 핵심 식 의미: `f_i(y)`는 variable node별 SNR이 달라지므로 반복복호 과정에서 각 edge/노드별로 별도 확률분포를 추적해야 함을 반영.
6. 최적화 절차: downhill(언덕오르기) 알고리즘으로 에너지 분포를 무작위 초기화 후 미세 변경하며 threshold가 개선되는 방향으로 반복 탐색(양자화 비트 8→10 단계적 증가).
7. 부수 기법: 짧은 cycle을 피하기 위해 PEG(Progressive Edge-Growth) 알고리즘을 변형하여 Tanner graph 구성.
8. 학습/최적화: density evolution 기반 threshold 계산 + downhill search로 에너지 분포 최적화(닫힌 해 증명 없음, 다양한 초기값에서 동일 수렴 확인).
9. 결과: 균일 에너지 대비 threshold 개선 0.02~0.44 dB (예: `R=4/9` 코드에서 0.44 dB, BER 10^-4에서 시뮬 결과와 일치).
10. 한계: HW 설계 없음, AWGN/BPSK 채널 전용 시뮬레이션(전송 파워 조절이 필요해 NAND read 채널과 물리적 매핑 없음), 기반 protograph의 minimum distance가 작으면 error floor 문제는 해결되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 전송 에너지(파워) 분포 최적화 | 대응 없음 | Prime ECC는 NAND read 채널(RBER/fixed-error) 기반이며 AWGN 송신 파워 할당 개념이 없음 |
| protograph 구조/density evolution | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드는 있으나 protograph 설계·DE 최적화 기능은 없음(대응 미흡) |
| belief propagation(soft, full-precision) 복호 | 대응 없음 | Prime ECC는 min-sum 근사 디코더이며 full BP는 미구현 |
```

적용 가치: 낮음 — AWGN 전송 에너지 최적화라는 물리 채널 특화 기법으로 NAND 채널 모델과 대응점이 없고, 복호기·HW 개선도 다루지 않음.
