> 총평: LDPC 자체가 아니라 TLC 페이지별 BER 불균형을 FTL 데이터 인터리빙으로 완화해 soft-read retry를 줄이는 시스템 최적화로, ECC 코어(부호/디코더/HW) 이식 대상이 아니라 이식성 하.

### B. 알고리즘 요약

1. 시스템: TLC NAND SSD, 셀당 8-state gray code, LSB/CSB/MSB 3비트가 서로 다른 BER(MSB가 최악, LSB 최선), LDPC로 신뢰성 확보 가정.
2. 문제: aged SSD에서 고BER 페이지(MSB)는 LDPC soft-decision을 위한 다중 read retry가 필요 → I/O request latency가 가장 느린 MSB 서브요청에 의해 지배됨.
3. 모델: I/O request는 여러 page 서브요청으로 분할, 완료시간=최느린 서브요청 서비스시간, 페이지 타입별 read latency 차등(LSB 50us/CSB 80us/MSB 100us, program 500/2000/5500us).
4. 핵심기법: DIR — 인접 LPN 3개 서브요청을 grouping, 각 그룹 데이터를 1/3씩 LSB/CSB/MSB device page(서로 다른 plane 블록)에 인터리빙 배치해 페이지 레벨 BER을 평준화 → read retry 감소.
5. 근거: page-level access locality(연속 쓰기된 페이지가 나중에 동시 접근되는 비율 ~80%)를 활용, group만 인터리빙해 write amplification 회피.
6. 확장: device page assignment는 예상 queuing time 기반으로 3 plane에서 LSB/CSB/MSB를 골라 동시 완료, relaxed program sequence[13] 채택.
7. 구현 디테일: host-transparent이므로 FTL이 grouped read 시 dummy 서브요청 2개 추가 생성, locality로 인해 중복 서브요청은 병합되어 read amplification 억제, reference count(초기3)로 update/GC 처리.
8. 최적화: 별도 학습 없음(휴리스틱 grouping + queue 기반 배치).
9. 결과: SSDsim 기반, MSR Cambridge 트레이스에서 aged SSD read latency 평균 36%↓(19~51%), 페이지 약 75%가 DIR 혜택, read amplification 3~23%(평균 10%).
10. 한계: LDPC 부호/디코더 자체 무변경(FTL 계층 데이터배치), HW 미설계·시뮬만, 매핑테이블 오버헤드(2GB->5GB), read/write amplification 유발.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TLC 페이지별 BER 불균형 인식 | 대응 없음 | Prime ECC는 부호/디코더 시뮬레이터, TLC 페이지 타입 개념 없음 |
| soft-read retry 감소(FTL 인터리빙) | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | soft-read/LLR threshold 계층은 있으나 논문 기여는 FTL 데이터배치라 직접 매핑 안 됨 |
| LDPC 인코딩/디코딩 엔진 | 대응 없음 | 논문이 ECC 코어를 개선하지 않음(블랙박스로 사용) |
| FTL 스케줄링/데이터 인터리빙 | 대응 없음 | 프로파일에 FTL/스케줄러 모듈 없음(시뮬레이터 범위 밖) |

적용 가치: **낮음** — LDPC를 블랙박스로 쓰는 FTL/SSD 시스템 최적화라 Prime ECC의 부호설계·디코더·HW 어느 레이어에도 이식할 알고리즘 요소가 없음.
