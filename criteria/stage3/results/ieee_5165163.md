> 총평: LP decoding 폴리토프의 기하학적 구조(fractional support, active set)를 이론적으로 규명하고 facet-guessing으로 성능을 개선하는 정보이론 논문으로, HW 구현이 없고 LP-solver 기반 디코딩 자체가 Prime ECC의 Min-Sum 아키텍처와 이질적이라 이식성은 낮음.

### B. 알고리즘 요약

1. 대상은 일반 이진 LDPC 코드이되, 이론적 분석은 `(c,d)-expander` 그래프 기반 코드(예: Tanner 코드, random (3,4) LDPC, n=200)에 한정.
2. 배경 문제: Feldman/Karger/Wainwright의 LP 디코더는 codeword polytope의 relaxation(fundamental polytope)을 풀며, 실패 시 대개 fractional pseudocodeword(정수 아닌 정점)를 출력해 오류가 됨.
3. 채널/가정: ML 디코딩은 `argmax` 우도 문제(식 1)를 polytope 위 선형계획으로 relax; pseudocodeword의 fractional support(비정수 좌표 집합)와 active set(폴리토프에서 정점에 인접한 facet 집합)을 정의.
4. 핵심 결과1(Proposition 1): `(c,d)`-expander 코드에서 모든 fractional pseudocodeword의 fractional support는 블록길이에 선형으로 비례(unique neighbor property로 증명).
5. 핵심 결과2(Theorem 1): 코드워드의 active set 크기는 `Θ(n)`이나, fractional pseudocodeword의 active set은 코드워드보다 최소 상수 비율만큼 작음(`|A(x)| ≤ |A(x_ML)| - Ω(n)`).
6. 핵심 기법(Facet Guessing Algorithm): 표준 LP 디코딩이 실패(fractional 출력)하면, 폴리토프의 facet 하나를 골라 그 facet으로 제한된 축소 폴리토프에서 LP를 재풀이 — exhaustive(EFG, 모든 facet) 또는 randomized(RFG, 상수 `L`회 무작위 샘플) 버전 존재.
7. 확장 알고리즘(Randomized Bit Guessing, RBG): `Θ(log n)`개 비트를 무작위 고정해 `2^k`개 조합에 대해 LP를 반복 — ML보다 우도가 높은 pseudocodeword가 다항 개(polynomial) 이하면 성공확률이 1로 수렴(Theorem 2).
8. 최적화/학습 절차 없음(이론적 알고리즘) — 실험은 sum-product/LP가 실패한 노이즈 realization에 대해서만 RFG(L=20) 실행.
9. 결과: 소규모 코드(Tanner group-structured code, random (3,4) LDPC n=200)에서 sum-product/LP 둘 다 실패한 케이스의 상당수를 RFG가 ML로 복구, 고SNR에서 최대 0.5dB 이득 관찰.
10. 한계: 순수 이론+소규모 시뮬레이션뿐(HW/ASIC 미설계), facet-guessing은 LP solver를 여러 번 호출해 실패 시에만 추가 복잡도가 붙지만 그 자체로 반복적 선형계획 문제이므로 NAND ECC급 저지연 HW 디코더에 이식하기엔 latency/복잡도 부담이 매우 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP 디코딩 자체 (fundamental polytope 최적화) | 대응 없음 | Prime ECC는 Min-Sum message-passing만 지원, LP-solver 기반 디코딩 아키텍처 없음 |
| Facet-guessing / bit-guessing로 실패 시 재시도 | `decoder.cpp` `LDPC_Decoder()` (조기종료·재시도 루프 개념만 유사) | 조기종료(Partial/Full CRC) 이후 재디코딩 개념과 표면적 유사성은 있으나 메커니즘은 완전히 다름(LP re-solve vs message-passing) |
| Expander code 구조/pseudocodeword 분석 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 분석이라는 점에서만 연관, Prime ECC의 QC-LDPC 특정 구조 분석은 아님 |

적용 가치: 낮음 — LP 디코딩은 Prime ECC의 Min-Sum 아키텍처와 근본적으로 다른 계열이며 HW 미구현, 소규모 이론 검증 수준이라 실이식 가치는 낮음.
