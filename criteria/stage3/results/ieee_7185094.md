### B. 알고리즘 요약

1. 시스템: block-fading 채널(코히런스 구간마다 서로 다른 Rayleigh fading 계수 `α_j` 적용) 위에서 protograph 기반 SC-LDPC(memory `m_cc`) 부호, rate R=1/2, `Nsc=2Nc`(F=2) 설정.
2. 문제: 기존 full-diversity root-LDPC 부호는 rate 1/2에서 diversity d=2로 한계가 있고, 임의의 목표 diversity `d_target`을 만족하는 체계적 설계법이 없었음.
3. 핵심 모델: 전송 패킷 시작점과 코드워드 시작점이 어긋나는 "synchronisation offset" `δ`(0≤δ≤1)를 도입 - 코드워드가 실제로는 F+1개의 서로 다른 fading gain에 걸쳐 수신됨.
4. 핵심 기법: diversity order `d`를 protograph `Pc`의 stopping set 이론으로 정의 - `d = sup_C lim -log Pe(γ,C)/log γ`이며, 고정 allocation `A`에 대해 `d = smin(A)`(최소 stopping distance).
5. 핵심 식 의미: 코드워드 비트를 전송 패킷에 어떻게 배치(allocation `A`)하느냐에 따라 동일 protograph라도 diversity가 달라짐 - deep-fade(α=0) 패킷 개수가 stopping set 크기 이상이면 반복복호가 실패.
6. 확장: `δ>0`일 때 allocation set `A_i`를 `±δ/2` 범위로 확장한 `A'_i`를 정의해 offset 하에서도 동일한 stopping-distance 프레임워크로 목표 diversity 설계.
7. 설계 절차: (1) `smin(Pc) ≥ d_target`인 protograph 설계(별도 논문[2] 참조), (2) `smin(A) = d_target`인 allocation을 탐색(현재는 체계적 방법 없이 exhaustive/random search).
8. 부수 트릭: SC-LDPC는 sliding-window 디코더(window 크기 W)로 복호, 동일 구조적 latency `N=WNsc` 기준으로 root-LDPC와 공정 비교.
9. 결과(시뮬): N=1000, W=5, F=2에서 SC-LDPC effective diversity 4로 root-LDPC(d=2)보다 우수, δ>0에서도 root-LDPC는 diversity가 즉시 1로 붕괴하나 SC-LDPC는 완만히 저하(W 증가로 손실 보상 가능).
10. 한계: 무선 모바일(Rayleigh block-fading, HSDPA/LTE 동기화 미스매치) 특화 시나리오, HW 미설계, NAND의 정적 채널·RBER 모델과 전혀 다른 문제.
