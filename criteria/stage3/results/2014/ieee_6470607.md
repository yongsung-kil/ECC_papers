### B. 알고리즘 요약

1. 대상은 binary (N,K) QC-LDPC block code(rate 5/6, J=4/L=24)와 이로부터 unwrapping으로 유도한 LDPCCC(convolutional, rate b/c=5/6, `ms=3`), 채널 AWGN.
2. 문제: 광통신급 BER 1e-15 검증에 CPU BP 시뮬은 비현실적, FPGA/ASIC은 비싸고 코드 변경 시 재설계 필요 → 유연한 저비용 가속기 필요.
3. 핵심 기법 1(GPU 데이터 구조): `θ`개(warp 배수, θ=32) codeword를 동시 복호, 동일 Tanner graph 공유하므로 같은 edge의 θ 메시지를 하나의 package로 선형 저장 → global memory coalesced access.
4. LUTc/LUTv 2D 룩업테이블에 CN/VN 갱신 인덱스 저장, constant/texture 메모리에 캐싱; 2D thread hierarchy(θ×노드)로 warp 내 divergence 최소화(hard decision 시만 분기).
5. 핵심 식: 표준 BP(horizontal step CN 갱신, vertical step VN 갱신), 초기 LLR `λ_n = 2y_n/σ^2`.
6. 핵심 기법 2(LDPCCC): QC-LDPC 유도 LDPCCC의 주기성(`T=ms+1`)으로 `I`개 파이프라인 프로세서가 동일 구조정보(Hbase 서브블록 라벨) 공유 → 메모리·주소제어 단순화.
7. 부수 트릭: 데이터·AWGN 잡음을 GPU에서 직접 생성해 host-device 전송 오버헤드 <2%; codeword/AWGN 온디바이스 처리.
8. CPU 비교군: C(직렬) 및 OpenMP(멀티스레드) 디코더 구현, 쿼드코어에서 4스레드까지 선형 개선.
9. 결과: GPU(GTX460, 336코어)가 직렬 CPU 대비 LDPC 148~162배, LDPCCC 170~200배 가속; 8스레드 CPU 대비 39~46배.
10. 한계: 순수 소프트웨어 시뮬레이터 가속 논문 - 정정능력/부호설계 기여 없음, HW/RTL 없음, 부동소수 연산이라 NAND 고정소수 HW와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 표준 BP CN/VN 갱신 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum 기반 - 알고리즘 자체 신규성 없음 |
| edge 인덱스 LUT(LUTc/LUTv) | 대응 없음 | GPU 메모리 coalescing용 자료구조 - z=32 row-병렬 HW 모델과 목적 상이 |
| QC-LDPC 유도 LDPCCC(convolutional) | 대응 없음 | Prime ECC는 block QC-LDPC 고정 - convolutional 구조 미지원 |
| θ codeword 병렬 package 저장 | 대응 없음 | 시뮬 처리량용 배치 병렬 - HW 라우팅/면적과 무관 |

적용 가치: **낮음** - 부호도 디코딩 알고리즘(BP)도 신규성이 없고 순수 GPU 소프트웨어 가속이라, NAND용 binary QC-LDPC의 부호설계->디코더->HW 어느 레이어에도 떼어 쓸 기여가 없다.
