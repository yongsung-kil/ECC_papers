### B. 알고리즘 요약

1. 대상: BPSK+AWGN 채널의 regular/irregular LDPC (예제 `(102,204)`~`(10000,20000)`), rate ~0.5.
2. 문제: GPU LDPC 디코더는 CN/VN 업데이트 시 랜덤 연결 때문에 read·write 중 하나가 non-coalesced가 되어 메모리 접근 지연이 큼.
3. 핵심 기법1(매핑): 32개 코드워드를 한 block에 매핑, warp 내 32 thread가 서로 다른 코드워드의 **같은 섹션**(같은 `rmn`)을 처리 → load·store 모두 coalesced.
4. 데이터 배치: 같은 인덱스 메시지를 32 코드워드 묶음(set)으로 연속 주소에 저장. 주소 `addr = (idblock*lencode + idedge)*w + idgroup`, `w=32`, `idgroup = idthread mod w`.
5. PCM 표현: sparse 행렬을 `VN2CN`/`CN2VN` 두 1-D 배열로 변환(공간 `O(n^2)`→sparse), warp 내 동일 섹션이라 배열 1개 원소만 broadcast.
6. 핵심 기법2(압축전송): 메시지를 8bit 고정소수로 표현, `char4` 벡터로 128B 캐시라인당 4개 코드워드 처리 → block당 코드워드 32→128, 산술명령 밀도↑로 메모리 지연 은닉.
7. 양자화: 원형 SPA는 양자화 민감 → `PLRA`(Parity Likelihood Ratio Algorithm) 사용, CN update를 100x100 LUT(shared memory) 참조로 대체, 데이터 `[-50,49]` 클립.
8. 최적화: Visual Profiler로 block당 thread=1024, grid size(A~F) 실측 튜닝.
9. 결과: GTX660Ti에서 코드 D `(4000,8000)` Eb/No>2.0dB, 9 iter에서 >550Mbps. 8bit PLRA가 32bit SPA보다 error-floor 낮음(작은값 0 절삭).
10. 한계: 순수 GPU 소프트웨어, 단일 코드워드 latency 길어 실시간 부적합. RTL/ASIC HW 미설계, NAND 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA/BP 이터레이션 루프(CN/VN update) | `decoder.cpp` `LDPC_Decoder()` | 알고리즘 골격은 대응되나 논문 기여는 GPU 매핑이라 무관 |
| 8bit PLRA + LUT 양자화 CN update | `decoder.cpp` `CNU_Update_New_Mag()` | 우리는 min-sum(비-tanh), PLRA-LUT는 이식 이점 없음 |
| GPU coalesced 메모리 매핑/`char4` 압축 | 대응 없음 | HW pipeline(z=32) 구조와 무관, GPU 전용 |
| PCM `VN2CN`/`CN2VN` 1-D 배열 변환 | `ecc_top.cpp` `Load_PCM()` | 소프트웨어 자료구조 표현일 뿐, 이식 불필요 |
