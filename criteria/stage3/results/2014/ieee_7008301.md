### B. 알고리즘 요약

1. 대상: 고throughput QC-LDPC 층별복호(LD), base matrix `B_{J×K}`를 `z×z` circulant로 확장, 한 layer가 `z` sub-layer 병렬.
2. 문제: TDMP(=BCJR) CNU는 forward/backward 재귀 때문에 이전 layer의 갱신 write 전에 현 layer가 APP-LLR을 read → write-after-read 메모리 충돌 → pipeline stall로 throughput 급락.
3. 핵심 알고리즘 3요소: (a) Row Exchange(layer 순서 교환, 성능 무손실), (b) Sequence Reordering(행 내 원소 순서 재배열), (c) 충돌 검출기+NOP 삽입.
4. BCJR 스텝: `λ_{j,k}=L_{X}-Λ`, forward `α_k`, backward `β_k`, `Λ_k=α_k ⊞ β_{k+1}`, `L_new=λ+Λ_new` (⊞는 log-sum 연산).
5. 핵심 원리: BCJR write-back(backward)은 input(forward) 역순이므로, read/write '거리' `d_{j,θ}`를 최대화(공통원소를 최대한 늦게 read)하면 충돌 감소 → `max d_{j,θ}, θ∈[-2,-1,1,2]`.
6. Sequence Reordering: 인접 layer(r±1 우선, 그다음 r±2)의 공통원소를 읽기순서 오른쪽에 배치, 홀/짝 layer에 원순서/역순서 교대 적용해 거리 증대. 선형 복잡도(재귀·반복 불필요).
7. 탐색: Row Exchange는 허용 연속충돌 `c`를 0부터 증가시키며 DFT로 최적 base matrix 탐색, `SequenceReordering`+`ConflictTest`를 호출해 NOP 최소해 선택.
8. NOP 삽입 2종: CNI(충돌 시 전 행 앞에 열단위 NOP, HW수정 불필요)와 SNI(해당 행에만 단일 NOP, 최소비용이나 layer별 NOP 테이블 필요).
9. 결과: 802.11n/802.16e 전 모드에서 효율 최대 60%, 정규화 throughput 최대 약 140% 향상. rate 1/2(11n mode1/5/9, 16e 1/2)는 RE+SE로 충돌 완전 제거. 고rate(5/6)는 inter-layer 공통원소 많아 잔여 NOP 존재.
10. 한계: TDMP/BCJR 층별복호 전제이고 min-sum 기반과는 충돌 양상 상이 가능, gatecount·절대 throughput은 본문 미제시(시뮬 효율 지표 위주).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 파이프라인 스케줄/충돌 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, pipeline(col_M1/Z/P1~P4) | 우리 6-stage pipeline의 read/write 충돌 스케줄 최적화 관점에서 직접 관련 |
| base matrix 재정렬(RE/SE) | `ecc_top.cpp` `Load_PCM()` (H-matrix 로드) | 재정렬은 로드 단계 오프라인 전처리로 반영 가능하나 우리 PCM은 고정 |
| CNU(BCJR/TDMP 연산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min-sum이라 BCJR 재귀와 다름 → 충돌 원인 구조 상이 |
| 메모리 뱅크 접근/NOP | 대응 없음 (전용 메모리 충돌 모듈 표에 없음) | z=32 구조에서 충돌 병목 존재 여부 별도 확인 필요 |

적용 가치: **중간** — 파이프라인 QC-LDPC 디코더의 메모리 충돌 제거라는 목표는 Prime ECC의 z=32 pipeline과 부합하나, 알고리즘이 TDMP/BCJR 전제라 min-sum 기반 우리 구조에 충돌 양상·이득이 그대로 옮겨지는지는 재검증 필요.
