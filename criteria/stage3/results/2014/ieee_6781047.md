### B. 알고리즘 요약

1. 대상: recursively-constructed(multi-level) QC-LDPC 부호 HW 인코더, 부호 rate 1/2·3/4, length 수천~16K bit(예: `(2016,1512)`, `(2016,1008)`, `(2304,1728)`), Wimax base matrix 재귀 확장.
2. 풀려는 문제: 시스테매틱 인코딩 `p = m·(H1⁻¹H2)ᵀ`에서 `H1⁻¹`가 dense → 저장·연산 비용 큼; 기존 RU/generator 방식은 code 크기에 복잡도가 종속.
3. 부호 구성: base matrix `H_B`를 extension factor `S`로 반복 확장(0→zero, 1→circularly shifted identity), unique base shift matrix로 기술되는 multi-level QC-LDPC.
4. 핵심 기법(LU): `H2` 대신 `H1`의 역행렬을 GF(2)에서 오프라인 계산 후 `P·H1⁻¹ = L·U`로 LU분해 → 3단계(P,L,U) row-vector×matrix 곱(VMM)으로 인코딩, 저자 관측상 `H1` LU분해가 `H2`보다 sparse.
5. 식 의미: 곱셈을 삼각행렬 forward/backward substitution으로 치환해 dense 역행렬 저장·연산 회피(선형 복잡도).
6. 핵심 기법(멀티레벨 압축): nonzero sub-matrix 위치를 각 확장 레벨의 base index+shifting factor로만 저장, 식(10)(15)로 상위 레벨 index 반복 계산 → 메모리 최대 77.4% 절감(단일레벨 대비 2.18배).
7. HW 트릭: circular shift는 Logarithmic Barrel Shifter(LBS)로 1클록 수행, L/U는 제한된 sub-matrix type만 존재 → type tag+LUT로 압축, MAC(modulo-2 VMM)+shift.
8. 아키텍처: serial/shared-serial/degree-P partial-parallel/fully-parallel 제공, 공통 base matrix면 부호 길이 무관하게 상수 clock(fully-parallel은 3 또는 1 clock) 인코딩.
9. 결과: Virtex-5 FPGA, TAR·area-time complexity가 prior art 대비 10~80배 우수, fully-parallel (2304,1728) 191.8Gbps, (2016,1512) serial 106.2Mbps/1035 slice, BER는 Wimax 동급 length에서 waterfall 동등·error floor 낮음.
10. 한계: 부호설계/디코더 성능이 아닌 인코더 HW 전용, gate count 공정 미표기(FPGA slice 기준), recursive 확장 구조가 아닌 PCM엔 이득 재검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LU분해 기반 systematic 인코딩(P,L,U 3단 VMM) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | 우리는 dual-diagonal 고정, LU/압축 인코더로 교체 시 재구현 필요 |
| 부호용 PCM 생성/재배치 | encoder.cpp Column_Reordering_Dual_Update() | 인코딩용 PCM 파이프라인 접점, 구조 상이 |
| recursive multi-level QC-LDPC PCM 로드 | ecc_top.cpp Load_PCM() | 우리 QC-LDPC는 base+circulant 고정, 재귀 확장 구조 아님 |
| circular shift(LBS)/멀티레벨 메모리 압축 | 대응 없음 | HW 인코더 특화 자료구조, 우리 디코더 z=32/HCU와 무관 |
```

적용 가치: **중간** — 인코더 레이어(dual-diagonal 대체 LU·메모리압축)만 건드리며 디코더·정정능력 기여는 없음. Prime ECC의 고정 dual-diagonal 인코더 대비 고throughput 대안으로 참고 가치는 있으나, PCM이 recursive multi-level 구조가 아니면 압축·LU 이득이 그대로 옮겨지지 않아 재검증 부담이 큼.
