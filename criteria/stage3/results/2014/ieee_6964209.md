### B. 알고리즘 요약

1. 시스템: 100Gb/s+ coherent 광통신, soft-decision FEC, LDPC/BTC와 차세대 후보 polar code·spatially-coupled(SC) code 비교, 25% overhead.
2. 문제: 기존 LDPC/BTC 대비 점근적 용량달성 부호 필요, irregular block LDPC는 낮은 SNR엔 좋으나 error-floor로 외부코드 rate loss 발생.
3. 모델: AWGN 채널, polar는 `x=uF`, `F=[[1,0],[1,1]]^{⊗n}`, SC는 무한확장 sparse binary `H`(`HxT=0`, `dim Hi(t)=m×n`).
4. 핵심기법1(polar): 극화행렬로 `N=2^n` 벡터 극화, K 정보비트+frozen, successive cancellation 복호 `O(N logN)`, 부호어오율 `O(2^{-√N})`.
5. 핵심기법2(SC): 좌측종단 LDPC convolutional을 windowed decoder로 복호, staircase/braided-BCH(rate 239/255) 같은 SC product code 사례 언급.
6. 확장(복호파 속도): degree분포로 decoding wave 전파속도(=윈도우 1스텝 전진에 필요한 iteration `Ireq`) 제어, degree-3와 degree-ℓ(ℓ∈{4,6}) 혼합, 비율 `av,ℓ` 조정.
7. 핵심식: density evolution으로 특정 `Ireq` 달성 요구 `Eb/N0` 산출, 실용값 `Ireq=1`에선 ℓ=6, `av,6=0.2`(degree-6 20%, degree-3 80%)가 최적.
8. 확장(lateral): PDL/OFDM 등 열화 병렬채널 대비 laterally-connected SC code 제안 - 서브채널별 독립부호를 sparse 연결해 상호부조.
9. 결과: 시뮬에서 SC(LDPC convolutional)가 regular LDPC(N=32640) 및 systematic polar(N=32768) 대비 우수, laterally-connected SC가 separate/interleaved 대비 이득.
10. 한계: HW 미설계, 순수 시뮬/개관 수준, error flare는 랜덤 구성 부작용, 광통신 특화.

```
| polar code + successive cancellation | 대응 없음 | Prime는 LDPC 전용, polar 미대상 |
| laterally-connected SC(병렬채널) | 대응 없음 | Prime는 단일 채널 블록부호, 서브채널 연결 개념 없음 |

```
