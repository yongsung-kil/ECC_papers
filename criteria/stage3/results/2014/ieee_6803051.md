### B. 알고리즘 요약

1. 시스템: sub-22nm 2bit/cell MLC NAND, 페이지당 9kB(8kB user+1kB redundancy), `rnorm=8/9`, QC-LDPC 사용.
2. 문제: MLC의 상위 페이지가 하위 페이지보다 BER이 높아 동일 rate 부호로는 상위 페이지가 내구성 병목이 되고 redundancy가 낭비됨.
3. 모델: 각 wordline이 `N+R` 셀 구동, `2N`-bit user + `2R`-bit redundancy, `rnorm=N/(N+R)`. 하위 rate `rl`, 상위 concatenated rate `rcon` 사용, `ru<rnorm<rl`.
4. 핵심 기법: 하위 페이지는 강한 rate-`rl` 부호로 보호(여유공간 `ur` 발생), 상위 페이지는 rate-`rnorm` 부호로 먼저 보호하고 각 세그먼트에서 `Ns/k` bit를 모아 rate-`rcon` 연접부호로 추가 보호하되 redundancy를 하위 페이지 여유공간에 저장.
5. 핵심 식: `ur = k·Ns·(rl-rnorm)/(rl·rnorm)` (여유공간), `rcon = rl·rnorm/(rnorm+k·(rl-rnorm))` (연접 rate). 하위 여유공간과 상위 추가 redundancy를 일치시키기 위함.
6. 디코딩 흐름: 상위 페이지 읽기 시 먼저 rate-`rnorm` 복호, 실패 시에만 하위 페이지에서 rate-`rcon` redundancy를 읽어 연접 복호(rcon→rnorm 반복). 실패 시 read-retry/soft-sensing.
7. 구현 디테일: 모든 component code(rl/rnorm/rcon)의 user data 길이를 동일하게 강제해 디코더 구현비용 최소화.
8. 최적화: 측정된 P/E cycling별 BER 기반으로 광범위 시뮬레이션하여 상/하위 페이지 내구성 gap 최소화하는 `rl`, `rcon` 선택 (rl=10/11이 최적).
9. 결과: 하위 min-sum(max 5 iter, hard-decision)로 P/E 내구성 20% 향상, 상위 페이지 read latency <7% 오버헤드, 65nm ASIC 합성 결과 1GB/s throughput.
10. 한계: 디코더 알고리즘(min-sum) 자체는 기존 [2]를 그대로 사용, hard-decision만 평가, 그림/표 기반 gatecount 및 BER 곡선 수치 손실.
