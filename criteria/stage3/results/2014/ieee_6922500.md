> 총평: LDPC-CC(컨볼루셔널) 전력선 표준 디코더로 부호 자체는 NAND 이식 대상 아니나, layered decoding + NMS의 CN기반 extrinsic 압축(min1/min2/idx/sign) 기법은 min-sum 디코더 메모리 최적화 관점에서 참고가치 있음.

### B. 알고리즘 요약

1. 대상 부호: IEEE 1901(전력선통신) 표준의 rate `1/2, 2/3, 3/4, 4/5` LDPC-CC(convolutional code), 주기적 시변·무한 PCM `H`(memory `M`), 주기 `Tp=3`, check polynomial로 정의.
2. 문제: 다중 rate LDPC-CC를 puncturing 없이 하나의 area/memory-efficient 디코더로 지원하고, MP(message-passing) 대비 수렴 가속.
3. 핵심기법1 layered decoding: extrinsic 저장을 VN 기반이 아닌 CN 기반으로 전환, 같은 CN의 extrinsic을 묶어 저장 → posterior `Lv`를 layer마다 즉시 갱신해 수렴 가속.
4. 핵심 갱신식: prior `Sv→c = Lv_old - Zc→v_old` (식2), extrinsic `Zc→v_new = α·∏sign·min|Sv→c|` (식3, NMS), posterior `Lv_new = Sv→c + Zc→v_new` (식4).
5. NMS 축약: CN당 extrinsic을 (1st·2nd 최소 절대값, 1st min의 위치 idx, 모든 sign, sign 곱)만으로 저장 → 저장량 대폭 축소.
6. 정규화 factor `α=0.75` (float/fixed 동일), `0.75x = x - 0.25x`로 구현.
7. 양자화: posterior/prior 8bit(sign+4int+3frac), extrinsic 6bit(sign+2int+3frac); overflow는 abspos>41 비교로 처리(42×0.75=31.5).
8. HW: 10 processor 파이프라인, extrinsic은 processor 외부 저장(9개 블록만 필요, 마지막 processor 출력 불필요), NMS min/sub_min은 3단 비교(CMP4 2회)로 계산, dual-port 메모리로 layer당 3 clock.
9. 결과(SMIC 130nm 실측): 최대 300Mb/s@180MHz, core 3.55mm²(1.7×2.09), 전력 138~200.4mW(rate별), 66.8pJ/bit/proc@rate4/5; MP 대비 절반 processor로 우수한 BER(Fig.2).
10. 한계: LDPC-CC 전용(블록 QC-LDPC 아님), layer 갱신을 3 clock으로 나눠 latency는 오히려 증가, gate count/공정별 복잡도 수치 없음.
