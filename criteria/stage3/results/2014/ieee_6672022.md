### B. 알고리즘 요약

1. 대상 시스템은 NAND flash + ReRAM 하이브리드 통합 SSD(USSS)로, 서버/디스크어레이/SSD 계층의 중복 ECC·redundancy 기능을 단일 컨트롤러로 통합한다.
2. 풀려는 문제는 종래 HDD 최적화 계층 스토리지의 중복 오버헤드와 NAND 특유 error-pattern 비대칭을 활용 못하는 점이다.
3. Reverse-Mirroring(RM): primary의 고BER 페이지 데이터를 mirrored NAND의 저BER 페이지에 배치(lower/upper·page번호 역순), ReRAM을 재정렬 버퍼로 사용.
4. Shift-Mirroring(SM): 버퍼 없이 shift number `s`(~128)만큼 페이지를 어긋나게 mirror 배치, 기존 상용 스토리지에 적용 가능.
5. Error-Reduction Synthesis(ERS): mirror 데이터를 bit-flip 후 비교, error 방향(0->1 vs 1->0) 비대칭을 이용해 불일치 시 다수결 정정.
6. Page-RAID: NAND string(bit-line 방향) 데이터를 XOR한 block-parity를 ReRAM에 저장, 블록당 1 ECC 실패 복구.
7. Error-Masking(EM): read 시 오류 위치를 RLE 압축 테이블로 ReRAM에 기록, 다음 read에서 마스킹.
8. 핵심 수식은 BCH ECC 실패율 (4)~(6)로 page-RAID의 BER after ECC 개선을 정량화(40bit/1KB BCH 가정).
9. 결과: mirroring 없이 ABER 4.4x, RM+ERS+page-RAID+EM 전체 적용 시 32x 개선, data-retention 34x·W/E 4.2x 향상(2X/1Xnm 실측).
10. 한계: LDPC 부호설계·디코더 알고리즘을 전혀 다루지 않고 ReRAM 하드웨어와 컨트롤러 레벨 데이터 배치에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RM/SM/ERS mirroring 데이터 배치 | 대응 없음 | 페이지 물리주소·ReRAM 버퍼 레벨, 디코더 무관 |
| Page-RAID block-parity(XOR) | 대응 없음 | RAID 계층 기법, LDPC PCM과 무관 |
| Error-Masking 오류위치 테이블 | 대응 없음 | 컨트롤러 read 후처리, 디코더 내부 아님 |
| BCH ECC 실패율 모델 | 대응 없음 | BCH 기반, binary QC-LDPC 아님 |

적용 가치: **낮음** — LDPC 부호/디코더/HW 어느 레이어도 건드리지 않는 순수 NAND 시스템 신뢰성 기법이라 Prime ECC 이식 접점이 없다.
