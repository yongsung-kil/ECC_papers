> 총평: 고rate QC-LDPC 복호기의 메모리·라우팅을 줄이는 HW 아키텍처 논문. min-sum 성능은 불변이고 부호구조(single block-row)를 전제하므로, 메모리 최적화 아이디어(1-element 스케줄, subblock-sharing, 저지연 min/submin 탐색)만 참고 가치.

### B. 알고리즘 요약

1. 시스템: CDF/PDF 기반 single block-row 정규 QC-LDPC(rate (L-1)/L), 예시 (1020,935) rate 0.9167(dv=3,L=12,z=85), (2115,1974) rate 0.9333(dv=3,L=15,z=141), 6-bit 양자화.
2. 문제: 이런 고rate single block-row 부호는 복호기 설계가 체계적으로 다뤄진 적 없고, circulant 배열 길이 L이 최대 1000까지라 메모리가 병목.
3. 핵심가정: MSA(min-sum)를 재배열해도 성능 동일. weight-3 circulant을 3개 weight-1 부행렬로 분해해 H를 3×L compound 행렬 H̃로 변환.
4. 핵심기법(1-element schedule): C2V 갱신을 dv=3 layer로 쪼갬 — `R_cv,k = α·∏sgn·min|L-R|`을 layer별 계산 후 3개 layer 결과를 merge unit(6입력 CNU)로 병합. 재배열 MSA라 TPMP와 동일 성능.
5. 스케줄 의미: L/R/T/R 메모리에 p인접 열/행을 한 엔트리로 묶고, R 메모리엔 soft가 아닌 4값(min1, min2, min1 index, 부호들)만 저장 → 메모리 대폭 절감.
6. VNU 최적화(subblock-sharing): C2V·V2C 단계가 겹치지 않으므로 permutation π와 depermutation π⁻¹을 barrel shifter 하나로 통합(p×(log2 p+2) 2-to-1 MUX), adder/subtractor도 단일 adder로.
7. CNU 최적화: min1/min2·min1 index 탐색을 binary-tree로 하되, 2단계 suboptimal low-latency 탐색(각 pair의 큰 값들 중 최소 = min2)으로 backtrack 없이 latency (log2 n + log2 log2 n) 비교 시간 달성.
8. 학습/최적화: 없음(순수 아키텍처 최적화).
9. 결과: (1020,935) 35,530 bit 메모리 — [3] 대비 42.3%, [4] 대비 20.8% 절감. (2115,1974)는 각 43.8%/23.5% 절감. 1 iteration latency 7×z/p 클록.
10. 한계: HW 소비 추정/합성 비교만(FPGA·실측 없음), throughput·gatecount 절대수치 미제시, dv=3 한정 설명, 특정 single block-row 부호구조 전제.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 1-element 분해 복호 스케줄(layer별 min-sum) | decoder.cpp LDPC_Decoder(), C2V_Cal() | 이터레이션 스케줄 재배열 아이디어(우리 Dual-Update와 유사 계열) |
| CNU min1/min2/index 저지연 탐색 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min1/min2 탐색 HW 대안, critical path 참고 |
| barrel shifter 기반 π/π⁻¹ VNU | decoder.cpp VN_Cal_HD() 등, GT.cpp Make_GT_HW() | 열↔행 permutation/라우팅 최적화 대응 |
| 4값만 저장하는 R 메모리 아키텍처 | decoder.cpp LDPC_Decoder() (메모리 모델) | 메모리 라우팅 절감 아이디어, 우리 z=32/HCU 구조와 정합 필요 |
| single block-row 부호구조(CDF/PDF) | ecc_top.cpp Load_PCM() | 부호구조 전제가 우리 고정 QC-LDPC와 달라 이식 부담 |
```

적용 가치: 중간 — 메모리·라우팅 절감과 저지연 min/submin 탐색은 HW 아키텍처 레이어(중 난이도) 참고 가치가 있으나, single block-row라는 특정 부호구조를 전제하고 정정능력 이득이 없어 Prime ECC 고정 QC-LDPC에 그대로 옮기긴 제한적.
