### B. 알고리즘 요약

1. 대상은 BSC 상의 `(dv,dc)`-regular binary LDPC(예: (155,64) Tanner code R=0.41, (732,551) QC code R=0.75), dv=3 girth=8 중심.
2. 문제: 일반 parallel/serial bit-flipping(BF)은 3-left-regular에서 보장 정정능력이 `⌈g/4⌉-1`로 낮아 weight-2 오류도 못 고쳐 실용성이 떨어진다.
3. 핵심 기법 1: 각 VN에 hard decision 외 strength 비트를 더해 `0s,1s,0w,7w` 4상태로 표현, `χ_u(v)=2` 같은 애매한 경우 flip 전에 강도를 먼저 낮춘다(TBFA1, 함수 `f1`).
4. 핵심 식: VN 갱신 `w^l_v = f(w^{l-1}_v, χ)` where `χ=(χ0p,χ0n,χ1p,χ1n)`, dv-tuple로 불만족 CN 조합을 세분한다.
5. 핵심 기법 2: CN에도 신뢰 비트 추가 — previously/newly satisfied·unsatisfied 4상태(`Φ` 갱신), 이전 iteration 대비 변화를 반영해 `χ_u≈⌊dv/2⌋` 경계의 신뢰도를 평가(TBFA2).
6. 실패 분석 프레임워크: 알고리즘별 trapping set을 (재)정의, 가장 compact한 harmful subgraph 집합 `E^r_I(F)`(trapping set profile)을 재귀절차 RA1/RA2로 열거.
7. 알고리즘 선택: 단일은 `nmin_{I,F}`(최소 trapping set 크기) 최대화, 다중은 서로 다른 trapping set profile을 갖는 상보적 알고리즘을 병렬 선택(collective error correction).
8. 최적화: 41,472,000개 후보 중 weight-3 전정정 358,851개 -> weight-4/5 상보집합 선별, 코드 Tanner graph 지식으로 무관 trapping set 제거해 정제.
9. 결과: 단일 D1이 Gallager A/B 능가, D4(35 알고리즘 병렬)는 error-floor 영역에서 SPA에 근접/추월, (155,64)에서 weight-5까지 보장 정정, FER slope 5~6.
10. 한계: HW 미설계(gatecount·throughput 없음), BSC hard-decision과 regular dv=3/girth=8 가정, 고성능은 다수 알고리즘 병렬(최대 1050 iter)에 의존.
