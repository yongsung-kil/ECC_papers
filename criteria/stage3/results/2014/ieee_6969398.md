### B. 알고리즘 요약

1. 레이어드(row-layered) LDPC 디코더의 CNU에서 반복 사용되는 min1/min2(또는 min3) 탐색을 대상, 입력 폭 3bit(sign+2bit magnitude), set size n=24(Wi-Fi/WiMAX base matrix 차원).
2. 문제: 기존 sorting/tree-structure(TS) min-value generator는 입력 set이 크거나 k=3(column-layered)일 때 latency가 급증, CNU가 디코더 병목.
3. 핵심기법 - Bit Searching(BS): 원소 간 비교 없이, MSB→LSB로 각 최소값의 비트를 하나씩 확정. 특정 비트가 0인 원소 개수를 세어 min들의 그 비트를 결정.
4. 예: U={0,4,5,1,7,6,2,3}에서 첫 비트 0인 원소 4개→모든 min의 첫 비트 0, 이후 비트도 카운트로 확정.
5. Cycle1: 각 비트에 대한 Boolean filter를 병렬 적용(`inp(1)'`, `inp(1)'inp(0)'` 등), Cycle2-3: filter 출력 1의 개수를 LUT-A/LUT-B로 카운트(0/1/≥2 구분), Cycle4: 카운터로 min1/min2 비트 확정.
6. adder 대신 LUT 방식(Xilinx 6-input LUT): n=24 기준 filter당 4 LUT-A + 1 LUT-B.
7. 자동 VHDL 생성 SW 툴 제공(입력 set size, bit-width, k를 받아 synthesizable RTL 출력), k=2(row-layered)/k=3(column-layered) 지원.
8. FEC 정정 에러 카운트를 별도 회로 없이 얻어 adaptive scheme에 활용 가능하다고 언급.
9. 결과: Virtex6 FPGA, TS 대비 latency 평균 51%·worst case 42% 감소(input 6bit), input 4bit에서 FF 3.7%·LUT 5.95% 감소, 항상 4 cycle 반환.
10. 한계: CNU min-search HW만 다루고 BER·정정능력·부호설계 무관, 특정 입력 폭/set size 예제 한정, throughput 수치 없음.

| LUT 기반 카운트 아키텍처 | 디코더 HW 모델 (z=32 row 병렬) | row 병렬 CNU 구현에 매핑 검토 가능, 구조 정합 필요 |
| min-sum offset/magnitude 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `C2V_Cal_New_Sgn()`, `ecc_data.h` `PARAM_LLR` | 3bit magnitude 양자화 가정, Prime ECC 양자화 테이블과 폭 상이 |
| k=3 (column-layered) 확장 | 대응 없음 | Prime ECC는 row(Dual-Update) 스케줄, column-layered 미사용 |

적용 가치: **중간** — min1/min2 탐색은 Prime ECC CNU의 핵심 연산과 직접 대응되어 HW 지연 절감 관점에서 검토 가치가 있으나, bit-exact 시뮬레이터에 RTL 지연 최적화는 알고리즘 결과가 동일하므로 이득은 HW 합성 단계에 국한된다.
