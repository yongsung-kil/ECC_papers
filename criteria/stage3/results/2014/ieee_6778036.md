### B. 알고리즘 요약

1. 시스템: `b×n` 2D 배열(열=저장장치, RAID 유사), 열 수 `n` 고정, 열 크기 `b→∞`, binary XOR 유닛.
2. 문제: 대수적 array code는 열 전체 erasure(`ε=1`)만 정정하고 `r` 초과 시 급격 실패 — 부분 열 실패라는 현실 모델 미반영.
3. 모델: mixed combinatorial-probabilistic — 최대 `r`개 열이 실패(combinatorial), 실패 열 내부는 bit가 확률 `ε`로 i.i.d. erasure(BEC).
4. 핵심기법 1: LDPC array code 앙상블 — parity constraint가 각 열에서 최대 1 bit만 포함(one-neighbor-per-column, Construction 1/2)하는 것이 핵심 제약.
5. 식 의미: 실패열만 남긴 `decoder induced graph` `G_R`에서 check degree가 hypergeometric `T_i^(n,r,d)`로 변환 — 설계 그래프가 아닌 유도 그래프로 BEC density evolution 적용.
6. 핵심기법 2: probabilistically-MDS 부호 — rate `1-r/n`, `λ̃(1-ρ̃(1-x))=x` 만족(Prop.4)시 임의 `ε_r<1` 정정, MDS 등가 fault-tolerance.
7. 구현 디테일: check degree `d=n`이면 유도그래프가 `(l,r)`-regular로 단순화, `d<n`은 `r≥4`·`d>n-r+1` 조건에서 존재.
8. 학습/최적화: density evolution threshold 조건 `x>λ̃(1-ρ̃(1-x))`로 앙상블 설계(반복 최적화 아님).
9. 결과: 2D 앙상블이 동일 파라미터 1D LDPC보다 rate가 엄밀히 큼(Thm.15, `R̃(x)<R̃^-(x)`), `r` 초과 실패에도 graceful degradation. 실험(n=16,b=1000,l=3,d=12,r=6): 2D가 1D 대비 항상 동등 이상, threshold `ε=0.56`.
10. 한계: erasure(BEC) 전용 이론, HW·soft-decision·AWGN bit error 미고려, systematic 부호는 open problem.
