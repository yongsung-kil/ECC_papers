### B. 알고리즘 요약

1. 대상은 binary LDPC 코드(`[155,64]` Tanner code, `[1057,813]` LDPC)로 AWGN 채널에서 ADMM penalized decoder(ADMM-PD)의 고SNR 거동을 분석.
2. 문제: ADMM-PD는 non-convex 페널티(`g1=-α1|x-0.5|`, `g2=-α2(x-0.5)²`)로 LP-BP SNR gap을 메우나, 시뮬 불가한 초고SNR(error-floor) 영역 성능이 미지수.
3. instanton = 디코딩 실패를 유발하는 최소 norm 노이즈 벡터. `ninst = argmax Pr(n)` s.t. 디코딩 실패 → AWGN에서 최소 `‖n‖²` 문제로 변환.
4. ISA-PD(1단): 현 출력 `ω`와 all-zero를 혼동시키는 최소 norm 노이즈를 Lagrange multiplier로 계산 `s.t. γ(n)ᵀω+Σg(ωi) ≤ N·g(0)`, 반복해서 최소 norm 노이즈 갱신.
5. 이 식의 의미: LP와 ADMM-PD가 동일 제약집합에서 목적을 최소화하므로 [7]의 반복 pseudocodeword 탐색을 non-convex 목적에 확장한 것.
6. ISA-R(2단): Nesterov random gradient-free 최적화 기반 refinement. ISA-PD 출력으로 초기화하고 non-zero 좌표에만 Gaussian search direction 부여.
7. 구현 트릭: 목적식에 디코딩 성공 시 non-zero gradient 부여(식(5)), step size를 `ℓ2 ≤ 1`로 제한, C=20000 사용.
8. 파라미터: `μ=3, Tmax=100, ρ=1.9, ε=1e-5`, 데이터점당 K=1000 랜덤 초기화.
9. 결과: 최소 instanton norm(BP 11.48 / ADMM-PD α2=2 시 13.8 / LP 16.35)이 고SNR WER slope를 잘 예측; 페널티↑ → norm↓ → 고SNR 성능↓.
10. 한계: 모든 최소 instanton이 trapping set(예: (5,3),(7,1)) 구조에 support를 가짐 → ADMM-PD도 trapping set에 취약; HW 미설계, 시뮬/분석뿐.
