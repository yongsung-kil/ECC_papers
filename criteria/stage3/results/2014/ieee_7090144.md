### B. 알고리즘 요약

1. 대상: IEEE 802.11n/ac QC-LDPC 디코더 HW. partial-parallel 아키텍처로 `200MHz`에서 `2Gbps` throughput 목표. sub-block size 27/54/81 지원.
2. 문제: 802.11n/ac는 12개 parity check matrix(다양한 rate·block length)를 지원해야 해서 VN↔CN network(interconnect) 복잡도가 급증.
3. 알고리즘: Normalization Min-Sum(NMS) 사용 — SPA의 log·곱 연산을 min1/min2로 대체한 MSA에 정규화 인자를 곱해 성능 보정 (offset-MS/NMS 계열).
4. 핵심식: CN은 `min` 기반(식2) + XOR sign, VN은 연결된 min 값들의 합(식3), 출력은 LLR 부호로 hard decision(식5).
5. 아키텍처: input controller(FIFO·shortening/puncturing/repeat 처리) + core controller(IBU, CNU, VNU, NETU, HCTRL) + output controller 구성.
6. network 구조: tri-state buffer로 CN↔VN 연결, H-matrix controller(HCTRL)가 enable 신호 생성. pipeline stage↑ 시 network 복잡도↓ 대신 core cycle↑ (Table I: pipeline 4/6/8에 따라 network 486×972~243×972).
7. 핵심기여(최적화 network generator): 12개 PCM을 pipeline 수로 분할 → column×(row/pipeline)×(12×pipeline) 3D enable matrix(EM) 생성, zero-matrix 미저장·중복 1회 저장으로 축소.
8. 트레이드오프: pipeline 수로 processing time vs network 복잡도 절충 (Table I 비교).
9. 결과: 최적화 network가 fully-connected network 대비 interconnect `86%` 감축, 목표 throughput 2Gbps@200MHz.
10. 한계: BER/gatecount/공정/합성 결과·연판정 bit수 미제시(본문 서술 없음), 다중 PCM 유연성이 목적이라 단일부호 NAND엔 부적합.
