### B. 알고리즘 요약

1. 시스템: CC(802.11a/g/n), Turbo(3GPP-LTE), QC-LDPC(802.11n/802.16e)를 하나의 tri-mode FEC 디코더로 통합, 메모리가 칩면적 50~70%를 차지하므로 메모리 공유가 목표.
2. LDPC는 QC-LDPC, 최대 병렬도 `Z=96`, TDMP(Parallel Layered Decoding) 적용. Turbo는 BCJR/MAP, CC는 soft-in-hard-out Viterbi. 세 알고리즘 모두 forward-backward recursion으로 보고 하나의 아키텍처로 병합.
3. 문제: 기존 연구는 대부분 data path 병합에만 집중, 메모리 시스템 공유는 미탐구.
4. Global Memory: `5P`개 8-bit 뱅크. 한 그룹(5뱅크)=40bit가 Turbo용 5x8bit 또는 LDPC용 8x5bit로 동시 대응 → Turbo의 큰 데이터폭과 LDPC의 큰 병렬도 불일치를 해소.
5. LDPC posteriori는 5-bit 양자화(<0.1dB 손실)로 충분 → global memory 압축으로 뱅크 1/3 절감.
6. 핵심 식: `Total = GM + P·(IPB+ALB+INB)`, `Saving = (Sum/Unified - 1)·100%`, `P=12` 선택.
7. Non-conflict Permutation Network: Viterbi 8-state 병렬 처리 시 write/read permutation(`S[3]` 기반 4-데이터 교환)으로 뱅크 충돌 제거, 덧셈은 or, 나눗셈은 shift로 대체.
8. PE 로컬 버퍼(Input/Alpha/Inner Buffer)도 세 알고리즘 간 병합 - Alpha Buffer 175% 공유율.
9. 결과: 65nm ST 공정 합성, 메모리 41%(전체 최대 48%) bit 절감, 1.06mm2, 250MHz datapath, LDPC 533Mbps/Turbo 186Mbps/CC 180Mbps.
10. 한계: 무선 다중표준 통합 특화, LDPC 디코딩 성능(정정능력/iteration) 개선 없음, 부호설계·min-sum 개선 무관.
