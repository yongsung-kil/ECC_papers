### B. 알고리즘 요약

1. 대상 부호는 CDF/PDF로 구성한 single block-row QC-LDPC (rate `(L-1)/L`, 예: rate-0.9333 (2115,1974), `dv=3`, `L=15`, `z=141`)로, 열중량 3~5·circulant 크기 L 최대 1000이라 행중량 `dc`가 매우 크다.
2. 큰 `dc` 탓에 표준 MSA CNU는 비교기가 폭증 → HW 병목. 이를 완화하려 [5]의 1-element based decoding schedule(MSA와 수학적 등가) 위에서 CNU만 최적화한다.
3. 1-element 알고리즘은 `R_cv` 갱신을 `dv`개 sub-step으로 분해해 CNU 입력 V2C 메시지 수를 줄인다(대신 latency 증가). CNU는 1st/2nd minimum과 1st min의 index를 찾는 것이 핵심.
4. 1st min은 binary tree search로 쉽게 찾지만 2nd min 탐색이 병목. 순차식은 `2n-3` 비교로 latency 과다.
5. optimal low-latency 방식: 2nd min은 Step1에서 1st min에 밀려난 adjacent neighbor들 중에 있으므로 그들만 비교. latency `log2 n + ceil(log2(log2 n))`, 비교 `(log2 n-1)+(n-1)`. 단 root까지 backtrack 필요라 HW 부적합.
6. sub-optimal([5] 채택) 방식: 각 pair의 bigger value들만 Step2에서 비교. latency 동일하나 비교기 `2n-3`로 면적 큼.
7. 제안 hybrid 방식: 첫 레벨은 adjacent neighbor를, 나머지 레벨은 bigger value를 취해 binary tree로 최소값을 구함 → optimal과 sub-optimal의 절충.
8. hybrid의 비교기 수는 `3n/2 - 2`, latency는 `log2 n + ceil(log2(log2 n))`로 optimal과 동일(Step1·Step2 병렬 수행).
9. rate-0.9333 (2115,1974) 코드, 6-bit 양자화, Altera EP2C5F256C6 FPGA 구현: 단일 CNU가 Design1 대비 20.2%, Design2([5]) 대비 15.2% logic element 감소(391/368→312), latency 15ns 동일. z=141개 CNU라 전체 면적이득 큼.
10. 한계: 정정성능은 개선 없음(면적/비교기만), throughput·전체 디코더 수치 없음, MSA 변형·고행중량 코드에만 유의미.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNU 1st/2nd min 탐색(min-sum) | decoder.cpp CNU_Update_New_Mag() | min1/min2 탐색 로직에 hybrid 2nd-min 탐색 대체 검토 |
| CN sign 처리 | decoder.cpp C2V_Cal_New_Sgn() | sign XOR은 그대로, magnitude 탐색부만 영향 |
| 디코딩 이터레이션 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | CNU 내부 교체로 알고리즘 동작 불변 |
| HW edge/MUX 축소 | GT.cpp Make_GT_HW() | Graph Thinning과 목적(면적축소) 유사하나 대상 계층 다름(비교기 vs edge) |
```

적용 가치: **중간** — 정정성능은 불변이나, 고rate QC-LDPC의 min-sum CNU 면적을 `3n/2-2` 비교기로 줄이는 순수 HW 최적화라 Prime ECC의 min1/min2 탐색부에 이식 여지가 있으나, z=32 병렬단위·현 CNU 구조와의 정합 재검증이 필요하다.
