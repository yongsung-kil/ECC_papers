### B. 알고리즘 요약

1. 대상: NAND Flash용 rate-`0.91` QC-LDPC (block size `8960`), AWGN 채널, layered LDPC 디코딩, 최대 iteration `I=8`.
2. 문제: 기존 조기종료(ET)인 parity check equation(PCE)·HDA·SDA는 block size가 커지면 복잡도·면적 overhead가 급증. 모바일 NAND의 유한 전력에서 부적합.
3. 기법 배경: Layer stopping(LS) 기준[6]은 layer 내 모든 bit node의 a posteriori 값을 preset threshold와 비교해 수렴 layer를 검출, 다음 iteration에서 해당 layer 연산 정지.
4. 핵심기법(LSNT): LS로 정지된 "수렴 layer 개수"를 counter로 누적하고, 이 개수가 LSNT threshold를 넘으면 전체 디코딩을 max iteration 전에 종료.
5. 핵심 로직: counter `k` 증가(수렴 layer마다) → `k > LSNT_threshold`이면 decoder 정지 (Fig.1 pseudo-code).
6. HW 구성: LSNT는 counter 1개 + comparator 1개만으로 구현 → 초저 하드웨어 오버헤드.
7. threshold 튜닝: LS threshold=19는 MSA 대비 동일 BER 얻도록 시뮬로 결정, LSNT threshold=300은 HDA 참조로 설정.
8. 결과(면적): TSMC 90nm 합성, LSNT gate count 900 (cell area 13)로 HDA(19,342)·SDA·PCE(16,129K) 대비 압도적으로 작음.
9. 결과(성능): @Eb/N0=5dB(raw BER 8.22e-3) LSNT total termination rate 56.67%, HDA 61.15%, PCE 63.23% → 소폭 손실이나 면적효율(ATE) 최고.
10. 한계: 정정능력 자체는 개선 안 함(에너지/iteration 절감용), 연판정 bit수·throughput·latency 수치 미제시, PCE 대비 termination rate 손실.
