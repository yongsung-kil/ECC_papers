### B. 알고리즘 요약

1. 시스템/코드: AWGN 채널, rate ≈ `8/9`, column weight `c=4`, block length `8192` bits의 이진 LDPC.
2. 문제: 랜덤 LDPC는 성능은 좋으나 인코딩/디코딩·생성/패리티 행렬 저장 복잡도가 커 비실용적 → 구조적(대수적) 부호 필요.
3. 핵심 대상: 위수 `p`의 순열행렬 중 `A²=I`를 만족하는 부분집합 `Sp`(제곱근-단위행렬, √I 계열, GF(2)에서 full rank)로 부호를 구성.
4. 타일 구성: 정수 `a`의 이진표현으로 `bin(i)+bin(j)=bin(a) mod 2`인 위치에 1을 채워 `p×p` 기본 타일 `φ(a)` 생성.
5. 대수 성질(Lemma 1): `A,B∈Sp`면 `AB=BA`, `(AB)²=I` (가환·involution). Lemma 2: `gfrank(A+B)=p/2`.
6. 부호 구성: 주어진 rate `R`, 짝수 거듭제곱 length `n`, column weight `c`에서 girth 제약을 만족하도록 √I 타일을 배치해 `H` 구성.
7. 인코딩 등가성(Theorem 1): 생성행렬이 √I 합 형태의 submatrix 타일로 표현 가능 → QC의 systematic-circulant(SC)와 유사한 systematic 형태.
8. On-the-fly 인코딩(Theorem 2): `H=[A|B]`에서 `gfrank(B)=n−k`가 on-the-fly 인코더 존재의 필요충분조건 → 즉석 패리티 생성.
9. 결과: AWGN에서 code failure 확률을 QC 계열과 비교(Fig.1), 두 계열 성능차 "very marginal"(정성적, 수치 미기재).
10. 한계: 디코더 알고리즘·HW 미설계, error-floor/girth 정량 데이터 없음, 이득이 QC 대비 미미.
