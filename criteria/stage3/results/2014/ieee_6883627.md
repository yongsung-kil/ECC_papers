### B. 알고리즘 요약

1. 시스템: 하나의 binary 인코더/디코더가 병렬 채널 집합으로 통신, bit mapper가 부호비트를 채널에 할당. `(dv,dc,L,w)` SC-LDPC(two-sided/circular), 예: `(4,8,L,w)`, `w∈{2,4}`, `L∈{10..40}`, design rate ~0.5.
2. 문제: BICM에서 부호비트를 병렬 bit channel에 어떻게 배분(bit mapper)해야 디코딩 threshold가 최적인가.
3. 모델: BICM 병렬 bit channel을 독립 BEC로 근사(threshold는 주로 채널 mutual information에 좌우), 4-PAM+BRGC로 2개 BEC(`ε1,ε2`), 평균 erasure `ε̄`.
4. 핵심 기법: 할당 행렬 `A=[a_i,j]∈R^(m×L)` — position `j`의 VN 중 `i`번째 BEC로 보낼 비율. 열 합=1, 행 합=`L/m` 제약. 입력 erasure는 채널 erasure의 가중평균.
5. 수식 의미: DE 식(2)의 각 spatial position 입력 erasure `εj`가 `A`로 결정되어, position별 VN이 서로 다른 채널 품질을 받게 됨.
6. 최적화: threshold 직접 최적화는 계산 부담 커서, 고정 `ε̄`에서 수렴 iteration 수 `ls(A,ε̄)`를 최소화(differential evolution)한 뒤 새 threshold 계산·반복하는 iterative routine.
7. 발견: 최적 mapper는 일부 VN class를 "overprotect"해 bad channel에 더 배정하고 나머지를 good channel로 보내며, blue 구간에서 `a1,j`가 0.5보다 약간 큼.
8. 구조적 효과: two-sided에서 최적 mapper가 one-sided decoding wave를 유도, circular(tail-biting, rate loss 0)에서는 mapper만으로 가상 termination boundary 생성해 wave 개시.
9. 결과: two-sided `(4,8)`에서 uniform 대비 threshold 개선, `Δ=0.04,w=4`에서 chain length `L=40→25` 단축; circular는 uniform threshold 0.3834(uncoupled (4,8) BP threshold)에서 최적화로 gap 대폭 감소(L 클수록 이득 증가).
10. 한계: BEC·DE 점근 threshold 분석만, 유한 length/AWGN/HW 없음, `L→∞`에서 이득 소멸(`a1,j→0.5`), 향후과제로 protograph 유한 length·다른 채널 제시.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit mapper(부호비트→병렬채널 할당) | 대응 없음 | Prime ECC는 단일 NAND read 채널, BICM식 병렬 bit channel/mapper 개념 없음 |
| SC-LDPC (dv,dc,L,w) ensemble/DE threshold | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 block QC-LDPC 고정, spatial coupling/convolutional 구조 아님 |
| 병렬 BEC/BICM 채널 모델 | `channel.cpp` `Set_LLR_Th()` | Prime ECC 채널은 AWGN/RBER 단일, 병렬 채널·PAM 라벨링 모델 없음 |

적용 가치: **낮음** — bit mapper 최적화는 BICM 병렬 채널 전제이며, 단일채널 block QC-LDPC 엔진인 Prime ECC에는 대응 개념이 없어 이식 불가.
