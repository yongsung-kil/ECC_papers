### B. 알고리즘 요약

1. 대상: `(J,K,L,w)` spatially-coupled LDPC ensemble, rate `R∈[0.1,0.9]`, BEC 채널, windowed BP 디코딩(복잡도 `w`에 비례).
2. 문제: regular SC-LDPC는 유리한 rate에서만 저복잡도·capacity 근접, 그 외 rate는 degree(`J,K`)가 커져 복잡도·성능 페널티 급증.
3. 모델: rate `R=1-J/K`, 복잡도 `C=R·J̄` (edge 수 = variable degree 종속), 정확도 `A=10^-i`에서 `K_Max=1/A`.
4. 핵심 기법: 같은 variable degree `J`의 두 regular 부호(check degree `Ki,Kj`)를 섞어 slightly irregular ensemble `NR(λ,ρ)` 구성 — `λ(x)=x^(J-1)`, `ρ(x)=ρ_Ki·x^(Ki-1)+ρ_Kj·x^(Kj-1)`.
5. 수식 의미: 계수 `ρ_Ki,ρ_Kj`를 목표 rate 식(8)에 맞춰 풀어, 두 favorable rate 사이를 "보간"해 임의 유리수 rate 달성.
6. 확장: mixing rule 2종 — boundary mix(최고/최저 rate 부호 혼합), direct neighbor mix(연속 rate 부호 쌍 보간).
7. 구현 디테일: `J≥3` 강제(degree-2 배제)로 최소거리가 block length에 선형 증가 보장, `w=3`이면 충분히 capacity 근접하며 windowed BP 복잡도 낮게 유지.
8. 최적화: spatial coupling의 threshold saturation으로 BP threshold가 uncoupled MAP threshold에 수렴, `J` 증가 시 MAP→Shannon.
9. 결과(BEC/DE): direct neighbor mix는 `J=3`에서도 전 rate 구간 Shannon 근접(gap 0.006~0.02), boundary mix는 중간 rate(~0.6)에서 gap 커져 `J=5` 필요.
10. 한계: BEC·점근 DE threshold 분석만, 유한 length BER·HW·양자화·AWGN 없음, 여러 check degree 혼합의 구현 부담 언급.

### C. Prime ECC 관련 모듈 핀포인트

대상이 code-design 이므로 C섹션(디코더 성능)은 전부 N/A. 부호설계 자체는 정정능력 개선이 아닌 rate 유연성이 목적이라 both 아님.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| nearly-regular SC-LDPC ensemble 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 high-rate QC-LDPC 고정, spatial coupling/convolutional 구조와 불일치 |
| degree distribution mixing rule | 대응 없음 | QC-LDPC base matrix 기반이 아닌 protograph/ensemble 관점 |
| windowed BP / DE threshold | 대응 없음 | Prime ECC는 block QC-LDPC min-sum, windowed convolutional 디코더 아님 |

적용 가치: **낮음** — SC-LDPC rate 유연성 구성법은 BEC 이론 수준이고, rate 고정 block QC-LDPC 엔진인 Prime ECC의 부호설계·디코더 구조와 근본적으로 다름.
