### B. 알고리즘 요약

1. 대상: SRA(Seamless Rate Adaptation), 본질은 RCM(rate compatible modulation) — 정보비트를 가중합으로 심볼 `ui=Σ gi,j·bj`(가중치 집합 `W={-4,-4,-2,-1,1,2,4,4}`, L=8) 생성, sparse projection matrix `G` (M×N), M=N=400.
2. 문제: SRA 복호는 산술영역 반복복호로 convolution 다수 → 곱셈기 폭증; 기존 FPGA 설계(4.8Mbps)와 직렬 Jacobian log가 throughput 병목.
3. 핵심 가정/모델: AWGN `N(0,σ²)`, LLR 반복(horizontal/vertical) 메시지패싱; 논문은 명시적으로 "SRA 인코딩은 LDPC와 다르다(XOR 대신 산술 덧셈)"고 구분.
4. Horizontal 처리: `ui→bj` LLR을 log영역 convolution `pk=Σ(pkn−pkd)`로 계산, `Tablel,k` 2D binary LUT + `ln(1+ex)` LUT 사용, 노이즈항 `z0/z1` 반영 후 `llrri,j=R0−R1`.
5. 핵심식 의미: `R0,R1`은 `ln Σ e^(pk+z)` 형태로 multi-variable Jacobian logarithm `ln Σ e^xi` 근사가 필수 — 이 연산이 하드웨어 자원 대부분 소모.
6. Vertical 처리: `bj→ui` LLR `llrqj,i=Σ_{m∈Cj/i} llrrm,j`, adder만으로 구성, VUP 2배 복제로 N/2L 슬롯 처리.
7. 핵심기법: multi-variable Jacobian log을 재귀(n슬롯) 대신 `F(x1..xn)=F(y1..y_{n/2})`, `yi=F(x2i-1,x2i)` 트리형 병렬화 → 23변수를 이론 5슬롯(실구현 4클록)으로 압축.
8. 부수 트릭: fixed-point 설계(sign 1bit + 정수/소수부 표 II, 범위초과 시 saturate), `ln(1+ex)` 80엔트리 LUT(x∈[-5,5), 간격 0.125), gate-clock으로 전력 절감.
9. 결과: 65nm SMIC ASIC, 300MHz에서 84Mbps(1424 cycle/블록), 면적 5×5mm², 전력 4.21W; 기존 FPGA 대비 throughput 약 17.5배, BER은 부동소수 시뮬과 거의 동일.
10. 한계: 무선통신용 RCM 복호기로 NAND/SSD·binary LDPC와 무관, gatecount 미제시, LogMAP 기반이라 min-sum HW와 직접 비교 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SRA 반복복호 전체(horizontal/vertical) | 대응 없음 | LDPC parity-check 복호가 아닌 RCM 산술영역 LogMAP — decoder.cpp의 min-sum 구조와 무관 |
| multi-variable Jacobian log 병렬화 | 대응 없음 | Prime ECC는 min-sum(min1/min2)로 tanh/log-sum-exp 미사용 — 이식 대상 아님 |
| `ln(1+ex)` LUT / fixed-point 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `Get_VNU_Table_Idx()` (개념적) | 양자화 테이블 접근이라는 일반 아이디어만 유사, 함수 내용은 무관 |
| AWGN 채널/노이즈 분산 | [channel.cpp](../Prime_ECC_3.1_Claude/) `Set_R_Offset()` (개념적) | AWGN 모델링만 공유, RCM 심볼생성과 무관 |
| partial-parallel L×L 유닛 아키텍처 | 대응 없음 | Prime ECC는 z=32 row-병렬 QC-LDPC 구조로 SRA의 M×N projection과 구조 상이 |

적용 가치: **낮음** — 본 논문은 binary QC-LDPC ECC가 아니라 무선통신용 SRA/RCM 복호기 ASIC이다. 부호구조(산술 가중합), 복호식(LogMAP/Jacobian log), 응용(무선 rate adaptation) 모두 NAND binary LDPC와 근본적으로 달라 Prime ECC 3.1 이식 대상이 아니다.
