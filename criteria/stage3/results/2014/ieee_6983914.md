### B. 알고리즘 요약

1. 시스템: BSC 상 `(dv,dc)`-regular binary LDPC, hard decision만 사용 (soft-info 불필요), 특히 column-weight-3 / girth-8 short~medium 코드 대상.
2. 문제: 단순 bit-flipping은 short/medium 코드에서 성능 약함 (cw-3 코드 보장정정능력 `≤ g/4-1`), Gallager A/B보다 열등하다고 여겨졌음.
3. 모델: VN·CN에 각각 1비트를 추가(2비트 상태)한 two-bit flipping(TBF). VN 상태 `Av={0s,0w,1w,1s}`(strong/weak), CN 상태 `Ac={0p,0n,1p,1n}`(previously/newly satisfied·unsatisfied).
4. 핵심기법1: VN 갱신함수 `f: Av×Ξ^dv→Av`가 인접 CN 상태 카운트 `(χ0p,χ0n,χ1p,χ1n)`로 상태 전이 (Algorithm 1, symmetry·irreducibility 제약). 후보 알고리즘 수 `|Fr|=41,472,000`.
5. 핵심개념: fixed set / trapping set 정의 (Thm 1: `⌈dv/2⌉` even-degree CN 이웃 조건)로 디코딩 수렴 실패의 충분조건 규정.
6. 핵심기법2: 각 TBF `F`의 trapping-set profile `E^r_I(F)`(inducing set I별 uncorrectable 유발 subgraph 집합)를 정의.
7. 알고리즘: 초기 corrupt VN으로 유도된 subgraph에 VN을 하나씩 추가하며 실패 여부를 판정, 이중재귀 RA1(그래프 확장)·RA2(iteration별 RA1 호출)로 profile 열거; 동형(isomorphism) 그래프는 1개만 유지해 복잡도 관리.
8. 선택: 열거된 profile로 uncorrectable weight-t 패턴을 파악해, 병렬/직렬 diversity로 함께 돌리면 더 많은 오류를 정정하는 "good" TBF 집합 선정.
9. 결과: cw-3/girth-8 코드에서 여러 rate·length에 걸쳐 Gallager A/B·min-sum·SPA를 error-floor 영역에서 능가; weight-4,5 오류를 높은 확률로 집단 정정 (inducing set 4/5/6 VN profile 구축). 정량 BER/gate 수치는 본문에 없음(저널판 참조).
10. 한계: hard-decision(BSC) 전용, HW 미설계, 이론/열거 위주 (시뮬 곡선·gate·throughput 미기재), 저·중 length·cw-3 코드에 국한.
