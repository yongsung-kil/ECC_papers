### B. 알고리즘 요약

1. 데이터 저장(HDD/광/테이프)의 modulation code + error-correction code 연접 구조를 다루며, 테이프 채널 실측으로 평가.
2. 문제: forward concatenation은 짧은 저rate 변조코드·detector→ECC로 soft-info 전달 불가; 기존 reverse concatenation은 구현이 매우 복잡.
3. 핵심: partial reverse concatenation - inner `C1` 인코딩은 modulation 인코딩 후에, outer `C2` 인코딩은 modulation 인코딩 전에 수행(순서만 재배치)해 forward 대비 소폭 변경으로 RC 이점 획득.
4. inner `C1`을 RS 또는 binary LDPC로 선택; LDPC-based에서 detector→ECC로 soft-decision 전달과 iterative detection/decoding 가능.
5. 사용 LDPC는 shortened array-based(Fan) binary LDPC, base `LDPC(4831,4550)` (`n'=p^2, p=71`, column weight `c=6`)를 shortened해 `LDPC(4768,4487)`, rate≈0.941.
6. 복호는 표준 belief propagation(`iterDec=4`)과 dual-max(NPML) detector 간 extrinsic 교환(`iterDet=8`)의 joint iterative detection/decoding.
7. 고rate 변조코드는 enumerative `(d=3,I=6,M=22)` + sliding-window substitution으로 rate-232/234 구성(변조코드 자체는 별도 설계).
8. 평가는 IBM TS1140 테이프 드라이브 실측 readback 샘플 + EPR4 등화 소프트웨어 채널로 BER/block-error rate 측정.
9. 결과: LDPC-based partial RC가 forward concatenation 대비 SNR 약 1.8dB, RS-based partial RC 대비 약 0.5dB 이득(BER 1e-20 목표, overall rate 0.876 동일).
10. 한계: HW 미설계, LDPC는 표준 array-code+BP 재사용일 뿐 부호설계·디코더 신규 기여 없음, 테이프 특유 product-code/deep-interleaving·modulation 제약 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modulation-code / ECC 연접 순서 재배치 | 대응 없음 | Prime ECC는 modulation code 미사용, 연접 아키텍처와 무관 |
| joint iterative detection/decoding (dual-max NPML) | 대응 없음 | NAND는 detector-decoder 반복검출 구조 아님 |
| array-based binary LDPC + BP 복호 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 표준 array-code 로드 수준, min-sum HW 디코더와 무관 |
| product/outer RS code | 대응 없음 | Prime은 QC-LDPC 단독, RS product code 없음 |

적용 가치: **낮음** — modulation code + ECC 연접 아키텍처는 테이프/디스크 채널 전용이고 NAND QC-LDPC ECC 파이프라인과 접점이 없다. LDPC는 표준 array-based 부호를 BP로 복호하는 참조용일 뿐 신규 부호설계·디코더 기여가 없어 이식 대상이 아니다.
