### B. 알고리즘 요약

1. 대상: array-based LDPC block code `H(γ,p)`(γ×p array of p×p circulant, high-rate QC-LDPC)를 `L`개 결합한 array-based SC-LDPC `C(γ,p,L,ξ)`. 초점은 `γ=3`, column-weight 3.
2. 문제: 고SNR error-floor 원인인 `absorbing set`(trapping set의 하위집합, 디코더 독립 조합적 객체)의 개수를 줄여 성능 개선.
3. 구성: "cutting vector" `ξ=[ξ0,...,ξγ-1]`로 edge-spreading(unwrapping)하여 성분행렬 `H0`,`H1`을 만들고 band 형태로 쌓아 SC-LDPC PCM 구성(memory `ms=1`).
4. 핵심 정리: 최소 absorbing set은 uncoupled와 동일 크기 `(3,3)`, 최소 fully absorbing set은 `(4,2)`(Lemma 7, Theorem 8·9), 개수는 `L(K1+K2)-K2` 형태로 L에 선형 증가.
5. 핵심 결과: coupling이 6-cycle(=（3,3) absorbing set)을 '깨뜨려' 심볼당 평균 개수 `A(a,b)`를 크게 감소(block은 `A(3,3)=p-1`, SC는 (K1+K2)/p² 수렴).
6. girth/distance: SC-LDPC girth ≥ block girth(≥6), `dmin≥6`(Corollary 4·6) — coupling이 성능을 해치지 않음 보장.
7. 최적화: `A(a,b)`를 최소화하는 "minimal cut" `ξ*`를 computer search로 탐색(p≤23은 (3,3), p≤11은 (4,2)); cut에 따라 개수 달라짐.
8. 결과 수치: p=5,`ξ=[2,3,5]`에서 (3,3)는 `35L-25`, (4,2)는 `5L-5`; block C(3,5)는 (3,3) 100개·(4,2) 150개 → coupling으로 A(3,3) 4→7/5, A(4,2) 6→1/5로 대폭 감소.
9. 관찰: p가 클수록 coupling의 absorbing set 감소 이득 증가; block과 달리 (4,2)<(3,3) 개수.
10. 한계: 순수 조합적 이론(예비 시뮬만 언급), HW 미설계, γ=3 한정, rate는 저rate 예제(L작을수록 rate 낮음), BER 곡선 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| absorbing set 스펙트럼 최소화 부호설계 | 대응 없음 | 부호구조 자체를 SC로 바꿔야 해 Prime ECC 고정 H-matrix와 정합 안 됨 |
| minimal cut 최적화(computer search) | 대응 없음 | SC 전용 최적화, block QC-LDPC용 대응 없음 |

적용 가치: **낮음** — 부호 클래스가 컨볼루션형 SC-LDPC로 block QC-LDPC 기반 Prime ECC와 근본적으로 달라 부호설계 이식 불가, error-floor 이론적 통찰(absorbing set은 우리 부호에도 존재)은 참고 가능하나 직접 재사용 어려움.
