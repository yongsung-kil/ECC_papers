> 총평: NAND/플래시 직접 타깃의 binary QC-LDPC CPM-RID-MSA 복호로 CN·배선을 q-1배 줄이는 실용적 HW 기법(FPGA 검증)이나, 성능 무손실 대신 서브이터레이션 증가로 latency가 커져 Prime ECC 구조 정합 확인이 필요.

### B. 알고리즘 요약

1. 시스템: CPM 구조(circulant permutation matrix + zero matrix) binary QC-LDPC, AWGN/BPSK, 고rate 예제 (16120,15345) rate 0.9519, 파생 (11700,10925)·(8320,7803)·rate-1/2 (1040,520).
2. 문제: 고rate·큰 column weight LDPC는 CN 처리유닛·배선이 방대(16120 코드: CN 780개, 배선 96720개) → HW 복잡도·전력 과다.
3. 모델: parity-check array `H`가 section-wise cyclic 구조를 가져, 각 row-block의 모든 행이 top row `hi,0`를 section 단위로 순환시프트해 생성됨.
4. 핵심기법1: CPM-RID — 각 row-block의 첫 행만 모은 축소행렬 `H0`(m×n(q-1))로만 복호, 매 sub-iteration 후 reliability·수신열을 section-wise 왼쪽 1칸 순환시프트; q-1 sub-iteration = 전체 H 1 iteration.
5. 핵심식: `H0`의 행수·1-entry가 `H`의 `1/(q-1)`배 → CN 유닛·배선이 `q-1`배 감소(16120 코드에서 130배: CN 780→6, 배선 96720→744).
6. 핵심기법2: CPM-RMSA — MSA를 CPM-RID에 결합, 식 (2) `L(t→k)j = λatten·sign(Rj)·min{|Rj'|}` scaled min-sum. RMSA[5,6] 대비 VN이 직전 CN 메시지를 빼고 전송(식 1,3)해 low row-redundancy 고rate 코드 성능 손실 제거.
7. 트릭: 조기종료 — 각 sub-iteration 후 hard-decision `z`를 section-wise 시프트해 전체 `H`로 syndrome `s=zH^T` 계산, s=0이면 정지.
8. 부호구성: GF(131) 두 부분집합 합으로 6×124 base matrix `B` 생성(2×2 submatrix 비특이 → girth≥6), CPM-dispersion으로 H 생성; masking(4×8 → (3,6)-regular)으로 girth-8 rate-1/2 코드.
9. 결과: (16120,15345) 코드 scaled MSA(factor 0.75) 50iter로 BER 1e-13 error-floor 없음, Shannon limit 1.33dB, uncoded 대비 8.71dB 이득; CPM-RMSA(factor 0.5)가 전체 MSA와 성능 곡선 거의 일치(무손실). FPGA로 성능 측정.
10. 한계: HW 축소는 시간다중화(sub-iteration 130배 증가: 50iter=6500 sub-iter) 대가 → latency 증가, throughput/gate 공정 수치 미상, CPM 대칭구조 전제.
