### B. 알고리즘 요약

1. 대상 시스템은 IEEE 802.15.4g/802.15.6 body area network(BCC)로, 64QAM에서 raw BER `10^-1~10^-2`를 `10^-4`로 낮추면서 `<10mW@100Mb/s` 저전력을 요구한다.
2. 기존 저전력 FEC인 convolutional+Viterbi를 대체하려는 것이 목표이며, LDPC-CC를 새 후보로 제시한다.
3. 부호는 `(23,3,6)` rate-1/2 LDPC-CC (memory length 23, column weight 3, row weight 6)로, QC-like parity check matrix를 unwrap해 highly regular 구조로 만들어 shift-register 기반 인코더를 단순화한다.
4. syndrome former polynomial `H[ti]` (식1)을 time-varying 표현(식2)으로 변환해 인코딩한다.
5. 복호는 well-known **normalized min-sum**을 min/sec-min sorter로 수행 (기법상 신규성 낮음).
6. 스케줄링은 On-demand Variable-node Activation(OVA) 기반, 5개 decoder processor가 각각 6 VN·1 CN 처리.
7. HW 트릭: 5-stage pipeline(전압 스케일링 timing margin 확보), inter-processor permutation network로 out-of-date LLR 폐기해 **storage 25% 절감**, 6 memory bank를 커널 간 shared.
8. multiple power island + voltage level shifter로 aggressive power scaling(0.6V까지 down).
9. 결과: 90nm 실측, 0.46mm^2, max 285MHz/285Mb/s, 100Mb/s에서 9.9mW·1Mb/s에서 0.5mW, 19.8pJ/bit/proc; 3~5 iteration으로 64/256-state convolutional code 대비 동등 BER@`10^-4`.
10. 한계: LDPC-CC(streaming convolutional)라 block QC-LDPC와 부호·복호 구조가 다르고, 저전력 무선(BCC) 특화라 NAND soft-read/error-floor 관점 기여 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized min-sum CN 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()` | 이미 보유(min-sum), 중복 기여라 가치 낮음 |
| permutation network로 out-of-date LLR 폐기 (storage 25% 절감) | 대응 없음 | LDPC-CC streaming 스케줄 전용, block QC-LDPC 메모리 모델과 불일치 |
| shared memory / 5-stage pipeline HW arch | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) (pipeline 6-stage 모델) | 구조 상이(OVA 커널 vs z=32 row 병렬), 직접 대응 안 됨 |
| (23,3,6) LDPC-CC 부호구성 | 대응 없음 | convolutional code라 부호 패밀리 자체가 다름 |

적용 가치: **낮음** — LDPC-CC(convolutional) 저전력 무선 칩으로, 복호 알고리즘은 이미 보유한 min-sum이고 부호구조가 NAND용 QC-LDPC block code와 근본적으로 달라 이식 대상이 아니다.
