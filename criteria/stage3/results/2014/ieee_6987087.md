### B. 알고리즘 요약

1. 시스템: flexible optical network용 software-defined optical transceiver(SDOT), reconfigurable QC-LDPC FEC를 채널 QoS에 맞춰 동적 교체. code rate 1/2·2/3·3/4, block(sub-block) size 27·54·81, codeword 648~1944 bit.
2. 문제: 광 채널 상태(BER, dispersion 등) 변동에 맞춰 FEC overhead/code length/LLR-width를 유연 조정해야 하나 고정 하드와이어 디코더는 불가.
3. 모델: 광 채널 특성(chromatic compensation, equalization 등)에 따라 DSP+FEC를 재구성해야 하는 flexible optical link.
4. 핵심 기법: FPGA reconfigurable area에 QC-LDPC FEC 모듈 배치, SDN(optical network) controller가 교체 지시 → control processor(ARM)가 라이브러리에서 해당 bitstream 로드.
5. 재구성 메커니즘: 코드별 configuration file(bitstream)을 DRAM/FLASH 라이브러리에 저장, PCAP(programming peripheral controller)로 프로그래머블 로직에 전송, 나머지 트랜시버는 무중단.
6. 확장: 동일 아키텍처를 DSP 모듈 재구성·dynamic coded-modulation에도 적용 가능.
7. 구현: Xilinx Zynq-7000(ARM 2개 + reconfigurable area), 수신 중 들어오는 패킷은 재구성 동안 버퍼에 임시저장.
8. trade-off 탐색: 코드rate·sub-block size에 따른 throughput/LUT/dynamic power, codeword length·LLR magnitude-width에 따른 column-layered QC-LDPC 디코더 자원(그림만, 본문 수치 없음).
9. 결과: 재구성 시간 항상 <12ms(648@1/2=4.0ms ~ 1944@1/2=12.1ms), area 675~2511 LUTs. 광 스위치/WSS 재구성 시간(수 ms)과 동등해 실용적.
10. 한계: 디코더 알고리즘·부호설계 신규 기여 없음, BER/정정성능 곡선 없음, LDPC 세부는 인용문헌 [8][9]에 위임.
