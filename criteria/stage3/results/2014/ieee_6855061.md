### B. 알고리즘 요약

1. 시스템: SDR용 GPU(NVIDIA GTX480, Fermi) 기반 802.11n `(1944,972)` 및 802.16e `(2304,1152)` QC-LDPC 디코더, submatrix 크기 `B=81`(11n)/32배수(16e).
2. 문제: SDR에서 FEC 디코딩이 계산집약적 → 표준 throughput(300Mb/s@20MHz)에 근접하는 GPU 가속 필요.
3. 알고리즘: Turbo-Decoding Message-Passing(TDMP, layered) — H를 행 단위 순차 처리, posterior `γ` 갱신, TPMP 대비 20~50% 적은 iteration.
4. TDMP 내부 루프: `ρ←γ(Ii)−λi` (read/subtract) → `Λ←SISO(ρ)` → `λi←Λ` → `γ(Ii)←ρ+Λ` (write back).
5. SISO는 Min-Sum, `Λj=[∏sign(ρn)]×min|ρn|`; 직접구현 대신 Two-Min Algorithm(TMA)로 2중 loop을 순차 2 loop으로 축약.
6. 병렬화: 한 layer의 `B` 연속 행은 읽기주소 충돌이 없어 `B` CUDA 스레드로 병렬 처리, thread block 1개가 codeword 1개 담당.
7. 메모리: check matrix 인덱스(`Hposition`,`Hvalue`)는 constant memory, `γ`는 bank-conflict-free shared memory, extrinsic `λ`는 coalesced global memory.
8. 핵심 트릭: zerocopy(mapped pinned memory)로 host→device global 중간단계 제거, kernel이 device-side pointer로 직접 shared memory에 채널값 로드.
9. 데이터형 최적화: float→16-bit short 변환으로 shared memory 절감, occupancy 37.5%→50%, BER 손실 없음; early termination(B checksum 전부 0 검사).
10. 결과: 802.11n 295Mb/s(512CW)~365Mb/s(4096CW), 802.16e 374~507Mb/s, 평균 iteration 2에서 최대 throughput.
11. 한계: HW(ASIC/RTL) 설계 없음, 무선표준 코드 전용, latency 최적화는 future work로 남김.
