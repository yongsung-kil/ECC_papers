### B. 알고리즘 요약

1. 시스템: IEEE 802.11ad(60GHz WLAN) QC-LDPC, base matrix `16×8`, 서브행렬 `Z=42`, full H `336×672`, 부호길이 `672`b, `R=1/2`.
2. 문제: 일반 LDPC는 디코더 아키텍처 선택지가 fully-parallel/fully-serial 양극단뿐이라 throughput-면적 중간 트레이드오프를 유연히 못 고름.
3. 부호구조: QC 구조 + parity matrix 행을 4개 non-overlapping layer로 분할(layer 내 열가중치 ≤1) → layered 복호 가능.
4. 복호: BP min-sum 근사 `r_m,n = min|q_j,m|·∏sign(q_j,m)`, VN은 `q_m,n = L_pr + Σr_j,n`.
5. Layered BP(LBP): 1 iteration을 layer 수(`p_max=4`)만큼 sub-iteration으로 분할, layer별 즉시 갱신 → 수렴 빨라 avg iteration 감소(BP 5.1 → LBP 4.1).
6. 아키텍처1(fully-parallel): 672 HW VN + 336 HW CN, 1 iter=1 clock, 최대 throughput·최소 latency·최고 HW효율.
7. 아키텍처2(layered): CN을 sub-iteration 간 재사용(HW CN 84개, 4배↓), 4 clock/iter, LUT 46%↓.
8. 아키텍처3(serial-parallel): QC 구조로 Z-to-Z permutator 활용, HW VN/CN 각 42개(16배↓), layer당 `N_ne(p)` clock, LUT 11%만.
9. 결과(Kintex7 FPGA, 100MHz, LLR 8bit/내부 5bit, FP대비 <0.1dB): throughput 6588/2064/159 Mbps, LUT 116779/63424/12814, HW효율 56.4/32.5/12.4 Mbps/1K LUT.
10. 한계: FPGA 합성만(ASIC gatecount/공정 없음), 짧은 부호(672b)·`R=1/2`, 정정능력 개선 아닌 순수 아키텍처 트레이드오프.
