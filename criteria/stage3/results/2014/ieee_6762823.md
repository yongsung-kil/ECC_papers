### B. 알고리즘 요약

1. 시스템: DVB-X 계열 binary structured LDPC(길이 4000~64800, R=1/2 등)를 GPU에서 실시간 복호, AWGN 채널, host CPU가 인코딩·noising 담당.
2. 문제: 기존 GPU LDPC 복호는 flooding 스케줄만 사용 — 병렬성은 좋으나 수렴 느림; layered는 빠르나 GPU 매핑 부적합으로 배제되어 왔음.
3. 모델: 8-bit 고정소수 LLR(soft), offset min-sum 변형.
4. 핵심기법 1: horizontal layered min-sum(Algorithm 2) — CN 계산 후 연결 VN을 즉시 갱신, iteration을 sub-layer로 분할.
5. 식 의미: layered 스케줄은 CN 처리마다 VN을 즉시 업데이트해 수렴을 flooding 대비 2배 가속.
6. 핵심기법 2: intraframe 병렬 대신 interframe(intercodeword) 병렬만 활용 — thread당 successive frame 처리로 core 사용률·memory coalescing 확보.
7. 구현 디테일: frame interleaving/deinterleaving(행렬 transpose 유사)로 정렬된 collapsed 메모리 접근 확보, 4개 VN/CN을 32-bit word에 packing(8-bit precision), SIMD로 4 frame 동시 처리.
8. 학습/최적화: 없음(구현 최적화 위주).
9. 결과: GTX 660 저가 GPU에서 Falcao decoder 대비 speed-up 11.8~26.5배, 타 연구 대비 2~18배, layered로 메모리 40% 절감(VN-to-CN 메시지 저장 제거), BER는 동일.
10. 한계: GPU/CUDA 전용 소프트웨어 최적화, 부호성능(BER) 개선 없음, HW(RTL) 미설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| CN offset min-sum 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum은 이미 보유(중복) |
| GPU interframe 병렬 / frame interleaving | 대응 없음 | GPU SIMT 소프트웨어 최적화라 RTL z=32 병렬 HW 구조와 무관 |
| 8-bit LLR packing(32bit word 4값) | [channel.cpp](../Prime_ECC_3.1_Claude/) LLR 테이블(`PARAM_LLR`) | 양자화 개념 유사하나 GPU 특화 packing은 이식 불필요 |

적용 가치: **낮음** — GPU 소프트웨어 throughput 최적화가 핵심으로 NAND RTL ECC 엔진에 직접 대응 없고, layered 스케줄·min-sum은 이미 보유.
