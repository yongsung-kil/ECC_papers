### B. 알고리즘 요약

1. 대상: systematic binary QC-LDPC(lifting matrix `A`, `Am',n'∈{-1,...,Z-1}`), BIAWGN, rate 0.1~0.9·length 504~32256를 단일 mother 부호로 커버.
2. 문제: 채널·서비스별로 rate·length가 다양한데 각각 전용 부호를 두는 것은 비현실적 → 하나의 lifting matrix로 유연성 확보.
3. 다중 rate: information shortening(`Keff`로 하위 rate `rl`, 식 2) + parity puncturing(`Mp` puncture로 상위 rate `rh`, 식 3) 결합, DE로 최적화된 DD·puncturing 다항식 사용([7]).
4. QC 적용: bit 단위가 아닌 submatrix(Z bit) 단위로 shorten/puncture → lifting matrix `A`상에서 연산, block 구조·동일 규칙을 여러 Z에 재사용.
5. 핵심 구성: QC PEG로 RC degree sequence `d`를 우→좌 처리, check node DD를 (거의) 집중시켜 shortened 부호들의 성능 보장.
6. ACE 개선: 무작위 degree 순서가 낮은 ACE cycle을 만드는 문제를 submatrix 내 오름차순 degree 배치 + [11]의 ACE 최대화 PEG rule로 억제(error-floor 영향).
7. Recoverability: punctured VN을 아직 미사용 guaranteed survived check node에 정확히 1개 연결해 1-step-recoverable 보장(Fig.2), 90% parity puncturing에도 few iteration 내 복구.
8. 다중 length: 다중 lifting factor `Z1<...<Ze`를 동시에 고려해, 작은 Z에서 좋은 후보를 유지하면서 큰 Z shift값을 modulo 정합으로 추가 선택(Alg.1).
9. 결과: rate 0.5 lifting matrix(M'=63,N'=126), Z∈{4,...,256}, 100 BP iter, BER 10^-4 기준 dedicated PEG 63개 대비 평균 0.19 dB loss로 단일 enc/dec 구현 가능.
10. 한계: HW 미설계, error-floor는 구성 중 직접 처리 안 함(BER 10^-4까지만 평가), BIAWGN 가정·통신 응용 중심.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC lifting matrix / H-matrix 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 부호구조 로드 지점이나 Prime ECC는 특정 QC-LDPC 고정, 재설계 시 재검증 부담 큼 |
| shortening/puncturing 기반 rate-compat | 대응 없음 | Prime ECC는 고정 rate·length 위주, RC puncturing 구조 미보유 |
| PEG/ACE cycle 최적화 부호 구성 | 대응 없음 | 오프라인 부호설계 도구, decoder/HW 모듈에 직접 대응 없음 |

적용 가치: **낮음** — QC-LDPC 부호설계 계층의 유용한 유연성 기법이나, 고정 QC-LDPC를 쓰는 Prime ECC에 떼어 붙이려면 H-matrix 전면 재설계·재검증이 필요해 실이식 가치는 제한적이다.
