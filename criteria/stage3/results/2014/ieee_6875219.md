### B. 알고리즘 요약

1. 시스템: rate-compatible Kite LDPC(내부부호, prefix rateless) + rate-1 accumulator 프리코더(외부부호)의 직렬연접, AWGN/BPSK, rate 0.1~0.9.
2. 문제: 기존 RC-Kite 부호가 중간 code-rate 영역에서 capacity 대비 gap이 크다.
3. Kite 부호: `C[∞,k;p]` - p-sequence로 정의, `vt = vt-1 + Σ ht,i ui`(Bernoulli(pt)), `Hv`가 dual-diagonal, 코드rate별 19구간 `ql` greedy 최적화.
4. 핵심기법: 정보비트 중 비율 `α`만 accumulator(`g(D)=1/(1+D)`)로 프리코딩 후 나머지와 MUX → RC-Kite 인코더 입력, 프리코딩이 최적화 자유도 추가.
5. 핵심식: EXIT `IE,VND = (1-α)·IE,VND^NPRE + α·IE,VND^PRE` (4) - 프리코딩/비프리코딩 VN EXIT 혼합.
6. 최적화: Acc-Rep vs SPC-ACC 두 EXIT 곡선을 fitting하도록 `α`를 1차원 brute-force 탐색(BEC duality로 accumulator EXIT 근사).
7. 결과 α: rate가 낮을수록 α 증가(rate0.1→0.37, rate0.9→0.01), rate-1/2에서 threshold `Eb/N0=0.91dB`(Shannon 0.19dB 근접).
8. 복호: serial concatenation - RC-Kite(SPA, max 10 iter) ↔ precoder(Box-Plus) 외부반복 max 100, extrinsic 교환.
9. 결과: RC-Kite 대비 waterfall에서 최대 0.3dB coding gain, k=10000에서 최악 capacity gap 1.2dB, Raptor 부호 대비 전 rate 우위(info bits 1890/10000).
10. 한계: HW 미설계, 시뮬만, error-floor 미분석, rateless/시변채널 지향 - NAND 고정 high-rate QC-LDPC와 무관.
