### B. 알고리즘 요약

1. 대상은 `(n,k)` binary LDPC, 예제는 랜덤 생성 `(180,90)` (rate 0.5), 채널은 이진대칭채널(BSC), 목적함수 계수는 LLR `µ_i=log(p(r_i|x_i=0)/p(r_i|x_i=1))`.
2. 문제: Feldman의 LP 복호는 ML 성질은 갖지만, 기본 폴리토프의 꼭짓점에 정수 아닌 pseudocodeword가 있어 ML 성능에는 못 미침.
3. 핵심: 기존 parity-check의 선형결합으로 만든 redundant parity-check(RPC)를 추가하면 현재 분수해(pseudocodeword)를 infeasible로 만들 수 있고, LP 성능이 향상됨(message-passing과 달리 RPC 추가가 LP엔 항상 안전).
4. 각 check node의 로컬 폴리토프 제약 `g_j(x)=(1-x_i 합)+(x_i 합)≥1`을 쓰고, `g_j(x)<1`이면 현재 해를 배제하는 절단(cut)을 찾은 것.
5. 명제1: `g(x)` 하한은 0.5 초과 bit는 S에, 미만 bit는 S^c에 넣어 구하며 |S|가 홀수면 그 하한이 최소값 → 이웃 bit값만으로 절단 존재 판정 가능.
6. 이를 이용해 최소 `g_j(x)`가 1 미만인 check node를 찾는 정수계획(IP) 모델을 구성: 변수 `ω_i`(새 check의 이웃 여부), `α_j`(원 check 참여 여부), 제약으로 |S| 홀수·최소 2개 check 합산 강제.
7. 각 bit 기여도 `x̄_i=0.5-|x_i-0.5|`, 위치벡터 `p`로 목적함수 `Σ ω_i x̄_i` 최소화 → active 분수해를 배제하는 RPC를 보장적으로 탐색(존재 시).
8. 학습/최적화: heuristic 없이 CPLEX 12.4로 IP·표준 LP를 풀며, 유효 정수 codeword가 나올 때까지 RPC 생성을 반복.
9. 결과: BSC에서 `(180,90)` 랜덤부호에 대해 ALP가 일정 반복 후 ML 복호 성능에 수렴; 한 문제 예시에서 약 `230`개 IP 모델을 풀어야 함.
10. 한계: IP는 NP-hard로 계산량 과다·HW 미설계, 시뮬만, avg-iteration 개념이 아니라 매 단계 IP 풀이라 실시간 latency 악화.
