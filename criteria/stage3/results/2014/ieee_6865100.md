### B. 알고리즘 요약

1. 대상: 2-bit MLC NAND, page size 8KB, QC-LDPC(512x512 circulant, 64 block col) + outer BCH over `GF(2^12)`, 28nm CMOS.
2. 문제: MLC 미세화로 인접 Vth 간격이 좁아져 raw reliability 급락 → LDPC 단독 보호는 utilization·codec복잡도·전력 비용이 큼.
3. 핵심 아이디어: multilevel coding(set partition)으로 MSB가 맞으면 LSB의 두 상태 간 거리를 `D`→`2D`로 확대 → lower page RBER를 최대 `6dB` set partition 이득으로 감소.
4. read 구조: 상위(MSB) page는 기존 3 reference(V0~V2), 하위(LSB) page는 MSB 결과에 따라 VL0/VL1 추가 reference 1회로 결정 → 계층적 hard-decision.
5. 부호구성: 32개 BCH codeword를 splitter로 분기, 일부 비트만 inner QC-LDPC로 재부호화. BOL은 BCH(3968,3776,t=16)+rate-15/16 LDPC(32768,30720), 전체 rate `0.922`.
6. EOL(고노이즈): BCH(3072,2880,t=16)+rate-1/2 LDPC(32768,16384), 전체 rate `0.703`. multi-rate로 수명 전구간 적응.
7. 역할 분담: outer BCH가 LDPC error-floor의 소수 잔류 오류 제거 → LDPC의 극저 error-floor 요구 완화; 동일 reconfigurable BCH/LDPC codec 재사용.
8. 디코더 HW: 3-stage(입력버퍼/layered min-sum LDPC/출력버퍼) 동시 활성, 코어가 512 메시지 병렬 접근, iteration당 256 clock; extrinsic memory RAW contention으로 VNU Plus stall, ~10% overhead.
9. 결과: 목표 BER `1e-16`에서 평균 iteration 최대 `4`, 700MHz에서 LDPC branch throughput `2.5GB/s`, 전체 FEC `40Gbps`; net coding gain 약 `10dB`(weak)/`12.5dB+`(strong).
10. 한계: 시뮬만(실측·gate count 없음), soft-read는 매우 큰 노이즈에서만 옵션 언급, 이종 BCH+LDPC+multilevel read 파이프라인이 시스템 레벨 재구성 요구.
