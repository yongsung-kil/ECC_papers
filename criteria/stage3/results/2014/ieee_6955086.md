### B. 알고리즘 요약

1. 대상: left-degree `dl` 고정(variable-regular) 이진 LDPC의 Tanner그래프, girth `g`, `(a,b)` ETS(a=변수노드수, b=홀수차수 미충족 체크수).
2. 문제의식: dominant trapping set 파악이 error-floor 추정/디코더 개선/저error-floor 부호설계에 핵심이나 일반적으로 hard problem.
3. ETS 표현: degree-1 체크 제거 + degree-2 체크를 edge로 치환한 normal graph로 induced subgraph를 축약, isomorphism 탐색을 nauty로 대폭 축소(예 (6,6): 5300만 -> 11 구조).
4. 핵심 개념: LSS(layered superset) - ETS `S`가 짧은 cycle `C`부터 노드 1개씩 커지는 nested ETS 수열 `C=S(0)⊂...⊂S`로 도달되면 S는 C의 LSS.
5. Lemma 1: 크기 a->a+1 확장 시 추가 변수노드 `v`는 S의 미충족 체크에만(최소 2개) 연결되고 충족 체크엔 연결 안 됨.
6. Routine 1(OneExpansion): 미충족 체크와 2개 이상 연결되고 충족 체크와 무연결인 후보집합 `O2(S)`만으로 확장 -> 복잡도 `O(b·dr)`, 메모리 `O(a·b·dr)`.
7. Algorithm 1: 입력 short cycle들에서 시작해 크기 k까지 재귀 확장, LSS 성질 가진 모든 ETS를 보장 탐색.
8. 특성화: dl=3,4,5,6 / g=6,8 / a,b<=10 각 클래스의 non-isomorphic 구조가 대다수 short cycle의 LSS임을 증명, LSSx의 x(요구 최소 cycle 길이)를 Table로 제공(dl=4,g=8: 크기<10, b<9 ETS 전부 LSS).
9. 결과: dominant ETS를 short cycle 탐색으로 guaranteed하게 열거 -> importance sampling 기반 error-floor 예측 정확도·속도 개선, 저error-floor 부호설계에 활용.
10. 한계: LSS 미충족 구조(예 dl=3, (8,2) ETS)는 어떤 cycle에서도 도달 불가 -> 탐색 누락, HW·디코더 미포함, 시뮬 검증은 [8]로 미룸.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dominant ETS 탐색(LSS/Algorithm 1) | 대응 없음 | 부호분석 도구로 Prime ECC 런타임 모듈 아님 |
| error-floor용 부호구조 진단 | `ecc_top.cpp` `Load_PCM()` | 로드된 H-matrix의 trapping set 진단·설계 기준으로 간접 활용 가능 |
| 수렴실패/코너 케이스 추적 | `corner.cpp` | ETS가 수렴실패 원인 -> 코너 분석과 개념적 연결 |
| trapping-set 인지 디코더 후처리 | 대응 없음 | 본 논문은 특성화만, post-processing 디코더 미제안 |

적용 가치: **중간** — 코드에 직접 이식할 알고리즘은 아니나, Prime ECC의 고정 QC-LDPC에 대해 dominant ETS를 진단해 error-floor 원인을 규명하고 저error-floor 부호설계·corner 케이스 분석을 지원하는 오프라인 도구로서 간접 가치가 있다.
