### B. 알고리즘 요약

1. 대상은 binary QC-LDPC 부호 여러 계열: Array(A-LDPC), circulant permutation matrix(CPM), Tanner, difference family(DF) 기반. parity-check `H`는 `r0×n0` 순환블록(`q×q`) 구성, 논문은 column weight ≤3에 집중.
2. 문제: QC-LDPC는 대개 `H`로만 정의되어 generator matrix `G`가 거의 연구되지 않음. sparse `G`를 얻으면 (i) 인코딩 회로 gate·연산 감소, (ii) minimum distance 상한 = `G`의 최소 행 무게로 도출 가능.
3. 핵심 결과: 위 여러 계열이 모두 Fig.1의 대각(diagonal) 구조 sparse generator를 가짐 → 즉 QC-LDGM(low-density generator matrix) 부호임을 증명.
4. 일반형: `G`는 `k×n`이며 `k0`개 블록 `G0..Gk0-1`이 대각으로 배치, 각 블록은 `1×(r0+1)` 순환행렬로 구성. 대부분 null이라 sparse.
5. Array LDPC: `G0=G1=...`로 동일, `r0=3`일 때 4개 순환블록 index set이 `{0}`, `{i1,i2,i3}`, `{i1+i2,...}`, `{i1+i2+i3}`로 닫힌형; `dmin ≤ min{8,2q}`.
6. CPM-LDPC: 인접 4열 창을 한 칸씩 밀며 각 `Gi` 계산, 블록들이 even weight라 full rank 아님 → `G`가 subcode 정의; `dmin ≤ min{24,2q}`.
7. DF-LDPC: `n0=2`에서 `G0=[P(I1)^T P(I0)^T]`처럼 index set 교환·transpose로 유도, `w` 짝수면 subcode.
8. rank 결손(H의 선형종속 행 `r0-1`개) 보정을 위해 extra row(2q 연속 1) 추가.
9. 결과: 계열별 닫힌형 index-set 공식 제시, duplicate index 발생 시 블록 무게 감소로 dmin 상한이 더 낮아지는 사례(예: CPM 예제에서 24→22) 확인.
10. 한계: 순수 이론(HW·디코더·BER 없음), 예제는 toy `q`라 실제 sparse 아니며 일부는 subcode만 정의.
