### B. 알고리즘 요약

1. 대상: 소~중 블록길이 이진 선형부호 (예 (155,64) Tanner, (204,102) MacKay LDPC, (127,85) BCH)의 정확 ML 복호와 최소거리 계산.
2. 문제: ML 복호는 NP-hard이며 최상위 상용 IP 솔버(CPLEX)조차 저SNR에서 느리거나 메모리 부족으로 실패.
3. 모델: 이진입력 무기억 대칭 채널(AWGN), LLR `λ_i=log Pr(r|c=0)/Pr(r|c=1)`, 목적함수 `ψ_λ(c)=λ·c^T` 최소화.
4. 핵심 기법: constraint set `F` 위의 branch-and-bound. 노드마다 하한/상한을 계산해 상한 초과 노드를 가지치기.
5. 하한: Zhang-Siegel(ZS) adaptive LP + cutting-plane 알고리즘으로 fundamental polytope에 redundant parity-check cut을 반복 추가.
6. 상한: SP(sum-product) 디코더 + order-2 re-encoding을 후처리로 사용, `F`의 고정비트는 LLR을 ±∞로 강제.
7. 분기 규칙: pseudocodeword에서 `1/2`에 가장 가까운 미고정 위치를 선택해 0/1로 분기.
8. 노드 선택: depth-first 기본 + `M`회마다 best-bound 혼합, 부모 하한 재귀 갱신으로 탐색공간 축소.
9. 튜닝: dual simplex warm-start, inactive constraint 제거(임계 `T`), redundant cut 횟수 `R`/cutoff `γ` 제한으로 속도-정밀도 절충.
10. 결과: (155,64) Tanner 1.0dB에서 CPLEX 대비 11배 이상 빠름; 최소거리 계산도 CPLEX 대비 수십 배 빠름. HW 설계 없음, 벤치마킹 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SP order-i 재부호화 상한 | 대응 없음 (full-tanh Sum-Product 계열) | Prime ECC는 min-sum HW 근사만 보유, 이 상한 로직은 무관 |
| branch-and-bound ML 탐색 | 대응 없음 | Prime ECC는 실시간 iterative 복호기, B&B 탐색 없음 |
| ZS LP cutting-plane 하한 | 대응 없음 | LP/IP 최적화는 코드베이스에 없음 |
| 최소거리 계산 | 대응 없음 | 오프라인 코드분석 툴, 시뮬레이터와 무관 |

적용 가치: **낮음** — LP/IP 기반 오프라인 ML/최소거리 벤치마킹 툴로, Prime ECC의 실시간 min-sum HW 디코더와 접점이 전혀 없다.
