### B. 알고리즘 요약

1. 대상은 `r×t`개의 `e×e` cyclic permutation 부매트릭스로 이뤄진 QC-LDPC 코드(예제 row weight 16, `e=128`)의 layered min-sum 복호기.
2. 문제: layered 복호에서 한 layer에 여러 block row를 넣어 throughput을 높이면(멀티블록로우) v2c 갱신이 여러 c2v에 의존해 갱신식이 복잡해지고 critical path가 길어짐.
3. 선행 [3]은 intrinsic message `L`을 공유(`L ⇐ L + Σ(v(j,·) − v(j−1,·))`)한 뒤 `u = L − v(j−1,·)`로 v2c 계산 — `L` 공유가 데이터 경로에 adder를 추가하고 routing network를 늘림.
4. 제안: `L` 공유 없이 각 block row의 v2c를 `u(j,pi+k) = u(j,p(i−1)+k) − v(j,p(i−1)+k) + Σ_{l≠k}(v(j,p(i−1)+l) − v(j−1,p(i−1)+l)) − v(j−1,pi+k)`로 직접 재정식화(식 4).
5. 이 식은 (2)(3)과 수치적으로 완전히 동일 — **복호 성능 손실 없음**, critical path에서 adder 1개 감소(p−1 add + 1 sub).
6. `p=2`(even/odd row) 케이스로 구체화(식 5): 4개 subtractor + 4개 routing network 구조(Fig.2).
7. HW 트릭: reverse routing과 forward routing을 하나의 MUX 제어 변경만으로 결합 → critical path에서 routing network 1개 추가 제거(면적 오버헤드 없음).
8. CNU는 표준 min-sum(min1/min2/idx + sign product, 2's-comp↔sign-mag 변환), `w=5`일 때 CNU critical path 약 13 게이트.
9. 결과: `e=128,w=5`에서 [3] 대비 clock frequency 26%↑, 면적 23%↑(주 penalty는 추가 buffer, `t` 감소 시 오버헤드 감소 — `t=8`이면 19%). gate count 214k(제안) vs 174k([3]).
10. 한계: BER 개선·iteration 감소 없음(순수 속도/면적), 합성·FPGA 없이 gate count 이론 추정, throughput 수치 미제시.
