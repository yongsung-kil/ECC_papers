### B. 알고리즘 요약

1. 시스템: DVB-S2 IRA QC-LDPC(하나의 기본행렬을 순환시프트로 생성), standard block 64800bit(11 rate)/short block 16200bit(10 rate), QPSK.
2. 문제: full-parallel은 HW 부담 크고 serial은 비효율 → 균형잡힌 부분병렬 구조 필요.
3. 복호 알고리즘: LLR-BP의 `tanh/atan` 룩업이 비실용적이라 Min-Sum 계열 채택, NMS(정규화계수 `α`)/OMS(오프셋 `o`)로 MS 성능손실 보상.
4. 파라미터 선정(시뮬): 알고리즘은 NMS 채택(BP에 근접), 최대 iteration=20(20↔30 사이 0.3dB 손실이나 자원 절감).
5. 양자화: 정수 2bit + 소수 3bit + 부호 1bit = 총 6bit (소수 3/4bit BER 거의 동일 → 3bit 선택).
6. 정규화계수: 시뮬 결과 `α=0.7~0.8` 부근이 최적(Fig.4).
7. 아키텍처: 체크행렬을 행 기준 파트(각 180행)로 분할, 파트 내 병렬(병렬도 180)·파트 간 직렬 처리하는 PDU.
8. Barrel shifter: 시프트도에 따라 배열을 두 부분으로 나눠 한쪽만 비교, 다른 쪽은 비교 없이 대입 → 비교/자원 절감.
9. 저장 전략: LLR RAM→RAM Group2 덤프로 파이프라인, 180개 Iterative Result RAM을 단일 광폭 RAM으로 통합해 자원 절감.
10. 결과(XC2VP100): [12] 대비 slice register 24521 vs 38804, LUT 56730 vs 70554로 자원 절감. 성능은 Matlab 시뮬과 동일. Throughput 수치는 그림에만 존재(본문 미서술).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS/OMS check node update | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min1/min2+sign은 Prime ECC 기보유 min-sum과 사실상 동일 |
| LLR 양자화(2정수+3소수) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 양자화 폭 참고 가능하나 우리 magnitude 테이블 방식과 다름 |
| Barrel shifter(순환시프트 HW) | 대응 없음 (z=32 circulant shift는 HCU/GT 계열이나 표에 barrel shifter 없음) | DVB-S2 shift 방식이라 직접 대응 불가 |
| 부분병렬 저장/RAM 통합 | `decoder.cpp` 파이프라인(col_M1/Z/P) | 개념 유사하나 구조 상이(병렬도 180 vs z=32) |

적용 가치: **낮음** — DVB-S2 IRA 부호 특화 + NMS·시프트 최적화가 Prime ECC 기보유 기법(min-sum, pipeline)과 중복이고 구식 FPGA 대상이라 이식 이득 적음.
