### B. 알고리즘 요약

1. 대상은 IEEE 802.11n irregular LDPC (block length 648/1296/1944/2016, rate 1/2~5/6)로, error floor를 유발하는 trapping set을 식별하는 것이 목표.
2. 풀려는 문제: trapping set 식별은 NP-complete로 알려져 대형 코드에서 전수 열거가 시간·메모리 부담이 큼.
3. 핵심 정의: `(a,b)` trapping set은 induced subgraph가 `b>0`개의 홀수차수 check node를 갖는 `a`개 variable node 집합이며, 작은 크기(variable node ≤ 11)일수록 error floor 기여 큼.
4. 3단계 알고리즘: (1) unique cycle 식별, (2) 공통 노드를 갖는 cycle을 block으로 묶음, (3) trapping set 식별.
5. Step1: PCM으로부터 각 variable node 기준 tree를 구성하고 길이 `d=4,6,8`의 distinct-node cycle을 탐색·저장 (Alg.1).
6. 핵심 메모리 트릭: TG 전체를 메모리에 올리지 않고 연결된 check/variable node의 한 경로만 생성·처리해 최대 0.8GB로 억제.
7. Step3 Case I: 같은 block 내 두 cycle의 모든 조합을 만들어 trapping set 여부 검사 (Alg.2).
8. Step3 Case II: 길이 6 cycle 자체가 trapping set을 이루는지 검사 (Alg.3).
9. FER 추정: 표본 block들(260/3444)의 누적평균으로 block당 약 15 TS로 추정 → 2016bit/rate 3/4 코드 전체 약 51660 TS로 예측.
10. 한계: HW 미설계, 디코더 알고리즘 개선 없음, 결과가 표본 기반 추정이며 non-binary/QC 구조 특화 처리 없음.
