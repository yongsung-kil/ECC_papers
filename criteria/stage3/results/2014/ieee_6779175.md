### B. 알고리즘 요약

1. 시스템: PEG regular LDPC(Code1 1008×504 col-deg3, Code2 504×252 col-deg3), rate 1/2, BPSK/AWGN, soft 입력 `y`로 bit-flip 복호(최대 150 iteration).
2. 문제: BP/BPA는 near-optimum이나 복잡도 매우 높음; bit-flip(BFA)은 저전력이나 iteration 수가 많음.
3. 기반: gradient-descent bit-flip(GDBF) 목적함수 `f_GD(x)=Σ_i Π_{j∈N(i)} x_j + Σ_n x_n y_n`, inversion function `∆_n^GD`가 threshold 미만이면 bit 반전.
4. 핵심 기법 1단: bit별 적응형 threshold `λn(k+1)=θ·λn(k)`, `En(k)≤λn(k)`이면 반전 — bit 독립 처리로 max-search 제거, `θ=2^-dX+2^-dY`로 곱셈을 shift-add 2회로 대체(HW 친화).
5. 의미: `θ` 작으면 대형 step(다중 flip), 크면 소형 step — 자연히 many→few flip으로 이동.
6. 핵심 기법 2단: dual-scaling switching(DSA) — fine/coarse 두 branch(예 θ1=0.5, θ2=0.97)를 병렬 실행, 매 iteration syndrome sum 낮은 branch의 hard bit·threshold 선택, 이후 한 branch 비활성화로 에너지 절감.
7. 부수 기법: noise perturbation — syndrome sum이 창 `W` iteration 동안 불변(stuck)이면 `λn+1=λn+K·zn`(Uniform/Gaussian)로 threshold 교란해 local minima 탈출. early-stopping은 `f(k)=f(k-10 또는 -11)`.
8. 최적화: 실험적으로 K≈1.6(Uniform)/1.0(Gaussian), `W`≈10~20, perturbation은 20 iteration 지연 후 적용이 최적(off-line 통계 방식).
9. 결과: DSA(θ=0.63/0.97)가 단일 θ=0.97과 동일 BER이면서 4dB에서 평균 iteration 66→39(41% 감소), early-stop은 17까지; perturbation 추가 시 BER 1e-3에서 0.5dB 이득, iteration 추가 절감(4dB에서 ~11 cycle).
10. 한계: HW 미설계, AWGN 시뮬만, PEG regular rate1/2 소~중길이 한정, IRRWBF가 최고 BER(복잡도도 최고), gate count·throughput 미제시.
