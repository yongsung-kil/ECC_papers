### B. 알고리즘 요약

1. 시스템: disk-array(RAID) 소거채널 대상, MDS 어레이코드(CGR)를 LDPC로 변환. 대표 예 `(5x5)` 어레이 → `N=15, K=25`, rate `r=2/5`, 최대 3 disk 손실 복구.
2. 풀려는 문제: LDPC는 장부호에서만 성능 좋고 disk array는 속도·용량상 단부호를 요구 → 단부호 MDS성 확보가 어려움.
3. 모델: 2차원 disk-array에 소거(erasure)가 무작위 분포, 그래프의 node=variable node, edge=check node로 사상.
4. 핵심기법: complete-graph-of-ring(CGR) 그래프를 `m x n` 어레이로 배치, 각 parity를 관련 2개 systematic bit의 XOR로 구성해 MDS 어레이코드 생성.
5. 변환식: `N = n*(2m-1)/3`, `K = mn`, `H = [A | I]` (I는 check node/parity disk, A는 정보행렬)로 LDPC PCM 구성.
6. 확장: `(2k+3, 2)` 어레이코드와 그 dual `(2k+3, 2k+1)`의 rich class 언급(지면상 2 사례만 제시).
7. 구현: 각 parity는 관련 data disk XOR, cyclic-left-shift로 parity 분산, systematic이 parity와 같은 컬럼(disk)에 놓이지 않는 규칙.
8. 복호: belief propagation algorithm(BPA)으로 AWGN·fading 채널에서 복호.
9. 결과: `10^-3` BER에서 EVENODD 대비 약 `1.0 dB` 이득, `10^-2`에서 repetition code 대비 약 `0.5 dB` 이득.
10. 한계: 소거채널·disk array 특화, 극단적 단부호, HW 설계·복잡도·throughput 전무, 순수 구성법.
