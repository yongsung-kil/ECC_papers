### B. 알고리즘 요약

1. BSC/AWGN 채널, variable-regular binary LDPC; 시뮬 코드는 (640,192) QC r=0.3, (2640,1320) Margulis r=0.5, (1280,1024) AR4JA protograph r=0.8, MacKay (4095,3358) r≈0.82.
2. 문제: 고정소수점 MP 디코더의 메시지 포화(saturation)가 작은 trapping set 탈출을 막아 error-floor를 유발.
3. 모델: trapping set의 `k-separation assumption` 하에서 EPS 밖 correct 메시지가 지수적으로 성장하면 min-sum/OMS/AMS/SPA가 결국 EPS 오류를 정정함을 증명(Thm 1,4).
4. 핵심 기법 1단: `(q+1)-bit quasi-uniform quantizer` — `|x|<dNΔ`는 q-bit 균일 양자화, 그 이상은 성장률 `d>1`로 지수 증가하는 구간에 매핑, indicator bit 1개 추가.
5. 핵심 식 의미: computation tree에서 correct 메시지 크기가 `(dv-1)^l|L0|`로 성장(식17)하므로, 큰 포화한계가 있어야 correct가 incorrect를 이긴다.
6. 확장: 일반화 quasi-uniform은 균일 구간 크기 `Nu`를 자유 배분; box-plus SPA(min-sum+bounded correction s(x,y))에도 적용.
7. 구현 디테일: CN/VN update rule과 부호(H-matrix) 무변경, 후처리 단계 없음; SPA는 φ(x) 대신 box-plus로 포화 회피, 2-piece 선형근사로 5배 가속.
8. 최적화: `Δ, d, Nu`, 입력 LLR 스케일은 heuristic 탐색(예: AR4JA d=1.3, Margulis d=1.2~1.3).
9. 결과: (640,192) AWGN에서 (3+1)-bit가 8-bit 균일·부동소수점보다 낮은 floor; quasi-uniform SPA가 부동소수점보다 적은 iteration으로 수렴(200 iter 제한 시 우위).
10. 한계: HW 미설계·게이트수/throughput 없음, 이득이 code-dependent, 파라미터 heuristic, 분석은 variable-regular 코드 위주(irregular 확장 비자명).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| (q+1)-bit quasi-uniform 양자화 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | magnitude 양자화 테이블을 지수 범위(indicator bit)로 확장 — 직접 대응 |
| VN 처리/양자화 적용 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | 양자화된 message로 VN update 수행 지점 |
| CN min-sum(box-plus offset) 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum/OMS와 호환(update rule 무변경)이라 양자화만 교체 |
| 입력 LLR 스케일/threshold | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 입력 LLR magnitude 스케일 조정과 대응 |

적용 가치: **중간** — Prime ECC가 이미 min-sum·magnitude 양자화 테이블·적응형 LLR을 보유하므로 quasi-uniform 확장은 테이블 교체 수준으로 이식 쉽고 error-floor 이득 기대되나, 지수 범위 표현이 우리 고정 비트폭 HW/pipeline과 정합하는지와 code-dependent 이득 재검증 필요.
