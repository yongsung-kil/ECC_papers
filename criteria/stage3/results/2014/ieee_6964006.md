### B. 알고리즘 요약

1. 시스템: 광통신(>100Gb/s) 소프트판정 FEC, 좌측종단 무한확장 SC-LDPC(LDPC convolutional) 코드, 오버헤드 20%(rate≈0.83), sub-block 크기 `N=7500`.
2. 문제: SC-LDPC는 우수한 waterfall를 보이나 error-floor가 존재, 외부코드로 정리하면 용량손실로 NCG가 감소 → 낮은 floor 부호가 필요.
3. 모델: AWGN 채널, all-zero codeword, LLR 5-bit(32레벨) 양자화.
4. 핵심기법: 무한확장 parity-check `H_conv`(syndrome former memory `μ=1,2`), 서브행렬 `H_i` 크기 `M×N`, 각 서브블록은 과거 `μ`개 서브블록만으로 인코딩 가능.
5. 윈도우 복호: `w·M` 행 × `(w+μ-1)N` 열 부분행렬에 대해 3단계(새 서브블록 수신→`I` layered iteration→결정) 슬라이딩, latency는 `w+μ-1` 서브블록.
6. 확장: 유효 iteration 수 = `(w+μ-1)·I`, `w=13, I=1`이면 conventional 복호 12+μ 반복에 대응.
7. 구현 디테일: layered decoding + scaled min-sum(및 offset-corrected min-sum) 검노드 연산, 순환치환행렬 크기 `32×32` QC 구조.
8. 참조 부호: regular block `d_v=3,d_c=15`(N'=31200)와 `d_v=4`(N'=30720); SC Code A(irregular, d_v={4,5,7,20}, d_c=18), SC Code B(regular d_v=4, d_c=20).
9. 결과: SC Code B는 늦은 waterfall 시작 후 급강하, 외삽으로 NCG≈10.5dB(scaled MS) / 11.1dB(offset-corrected MS) @ post-FEC BER=1e-15, error-floor 미관측.
10. 한계: HW 게이트수/throughput 미기재, NCG는 외삽 추정, error-floor는 post-processing 필요 가능성 언급뿐, 광통신 AWGN 가정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| scaled/offset-corrected min-sum CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 자체는 이미 보유, scaling/offset 파라미터만 참고 |
| layered decoding 이터레이션 | decoder.cpp LDPC_Decoder() | Prime는 dual-update 스케줄, layered와 상이 |
| LLR 5-bit 양자화 | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | soft-2~3bit만 지원, 5-bit는 범위 밖 |
| SC/좌측종단 윈도우 복호 구조 | 대응 없음 | Prime는 고정길이 블록 QC-LDPC, convolutional 구조 미지원 |
| 32×32 순환치환 QC 구조 | ecc_top.cpp Load_PCM() | QC 구조는 유사하나 부호 자체 이식 부담 큼 |
```

적용 가치: 낮음 - SC-LDPC 윈도우 복호와 광통신 AWGN 타깃이 NAND 고정길이 binary QC-LDPC와 근본적으로 달라 이식 실익이 적음.
