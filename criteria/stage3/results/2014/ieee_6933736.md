### B. 알고리즘 요약

1. 다중 rate QC-LDPC 코드용 인코더에서 자원소모를 줄이고 병렬도를 조절 가능한 general folded encoding structure를 제안한다.
2. 문제: 전통 QC-LDPC 인코더는 rate별로 다수의 SRAA(shift-register-adder-accumulator) 회로를 필요로 해 자원 소모가 큼, 특히 multi-rate에서 심함.
3. 부호는 `C=m·G`, generator matrix `Gqc`가 q×q(=256×256) quasi-cyclic 서브행렬로 구성된 systematic 형태로 마지막 q비트가 정보열과 일치.
4. 정보열을 길이 q인 K 그룹 `M0..M_{K-1}`로 나누고, `MiGj`를 순환시프트+AND+누산으로 계산해 같은 열을 누적하면 check code를 얻는다(식4~7).
5. SRAA 회로: 순환시프트 레지스터 B에 generator 다항식 저장, 레지스터 A에 중간결과/check bit 누산, 정보비트를 하나씩 입력하며 B를 우측 순환시프트.
6. Folded 구조: 입력 rate에 따라 적절한 SRAA 개수 `n`과 계산횟수 `s`를 선택, control module이 RAM/ROM/SRAA/output module을 순차 제어하여 n·q 비트씩 s회 반복해 전체 check code 산출.
7. SRAA 수↓→로직자원↓이나 계산횟수↑ 트레이드오프이므로 multi-rate 전체 구성을 종합해 최적 개수/시간을 선택하고 SRAA 이용률을 고려.
8. 예제: rate {3/4,1/2,1/3,1/4}, N=9216 고정, 서브행렬 256×256, (K,C)=(27,9),(18,18),(12,24),(9,27).
9. 결과: 전통 구조는 최소 27개 SRAA 필요, folded는 9개로 자원 66% 절감; EP4CE115F29C7N FPGA에서 최고 150MHz, Matlab 대조로 검증.
10. 한계: 인코더 자원공유 구조뿐이며 복호성능·BER·부호설계 개선 없음, throughput/gate count 미제시.
