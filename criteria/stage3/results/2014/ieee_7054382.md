### B. 알고리즘 요약

1. 대상은 irregular QC-LDPC 2종(length 4096 z=64 rate 1/2, length 8640 z=360 rate 1/2), NMS 디코더의 FPGA 구현.
2. 문제: 전통 TPMP(Two Phase Message Passing)는 CN과 VN이 서로 완료를 기다려 latency가 두 단계 합이 됨.
3. 채널: AWGN + BPSK, 수신 메시지를 FPGA에서 양자화 후 복호(비트폭 미상).
4. 핵심 기법: CNP/VNP 두 단계를 오버랩(OMP)하되, 기존 OMP의 보정항 `w_cv`/`w_vc`를 제거한 단순화 스케줄(Fig.1(c)).
5. 식 (1): CN 업데이트 시 V2C 메시지를 (k)차와 (k-1)차 두 종류로 나눠 즉시 갱신분을 반영 -> layered 유사, 수렴 가속.
6. 식 (2): VN 업데이트 시 C2V 메시지도 (k),(k-1) 두 종류로 나눠 처리; 대신 TPMP 성능 도달에 더 많은 iteration 필요.
7. 구현 트릭: dual-port RAM 파이프라인 접근 - m행 write 중 (m+1)행 read 병행(CNP/VNP 각각).
8. 학습/최적화: 없음(고정 스케줄).
9. 결과: serial 디코더 latency 4.30ms->2.62ms(39.1%↓), 36KB RAM 3개 추가·slice 거의 동일; partly-parallel (2,4) 24.3%↓; OMP16 성능 ≈ TPMP15.
10. 한계: 고병렬 구조엔 추가 조치 없이 적용 불가(본문 [5] 인용), 정정능력·error-floor 개선 없음, HD/NAND 채널 미검증, gatecount·throughput 미제공.
