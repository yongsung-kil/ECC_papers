### B. 알고리즘 요약

1. 대상: 802.11n 다중 rate QC-LDPC (base `Mb×Nb`, circulant `z=27`, code length 648, rate 1/2·2/3·3/4), FPGA 구현.
2. 문제: 표준 offset min-sum(OMS)은 iteration마다 horizontal/vertical 두 scan이 필요해 지연이 크고, `R_mn`·`Q_mn` temp 저장으로 메모리 낭비.
3. 채널/모델: AWGN, BPSK, `LLR_n = log(p(x=0|y)/p(x=1|y))`, 고정소수점 7bit(`b=7`) 균일 양자화.
4. 단순화 1(single-scan): `Q_nm = P_n - R_mn` 관계로 vertical/horizontal scan을 하나의 horizontal scan으로 병합, iteration마다 `R_mn`과 `P_n`(N개)만 저장 → `Q_mn`(N·dv개) 저장 회피.
5. 의미: `Q`를 accumulate로 `P`에서 복원 가능하므로 저장량을 dv배 줄임.
6. 단순화 2(compact CN 저장): 같은 CN의 `|R_mn|`은 최소값 `A`·차소값 `B` 두 개뿐 → `(A, B, min위치 n_m, sign s_mn)`만 저장하고 `R_mn = s_mn·f(A,B,n,n_m)`으로 복원(식(6)(7)).
7. 아키텍처: semi-parallel layer decoding(`Mb` sub-iteration), block 단위 병렬(z=27=M), 2-stage pipeline, CNU는 A_B finder + R calculate 2 state.
8. 결과: SLD가 같은 BER에서 OMS의 절반 iteration으로 수렴(수렴 2배), 메모리 약 절반 절감; fixed-point(7bit) vs float 열화 <0.1dB.
9. HW: Altera Stratix IV, 200MHz에서 rate별 throughput 104/134/162Mbps, ALUT ~8k·register ~4.8k·block mem ~18k bit.
10. 한계: 정정능력 개선이 아니라 속도·메모리 최적화, gatecount@공정·ASIC 없음, 802.11n·AWGN 가정.
