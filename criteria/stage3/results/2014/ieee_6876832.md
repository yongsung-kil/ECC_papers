### B. 알고리즘 요약

1. 대상은 semi-random(non-QC) LDPC, PCM은 Jordan normal matrix `J`와 permutation matrix `Πi,j`(offset·jump 벡터로 정의)로 구성, 예제 rate-2/3, length-15360, sub-matrix 512×512, base 10×30, col weight 2~5 / row weight 10~11.
2. 풀려는 문제: 전통 partly-parallel 디코더는 VNP·CNP가 교대 수행되어 하드웨어 이용효율(HUE)이 낮고, NQC 코드에서 OMP는 QC 코드에서만 throughput 2배가 됨.
3. 모델/채널: AWGN, BPSK, offset min-sum 기반 BP.
4. 핵심 기법 GOMP(general overlapped message passing): VNU와 CNU를 동시 가동, 메모리 접근·연산 지연을 `dc`(check-variable 갱신 지연), `dv`(variable-check 갱신 지연)로 파라미터화, 이 값이 L보다 훨씬 작아 HUE가 거의 100% 달성.
5. 핵심 식: `colth = (offset_{i,j} + jump_{i,j} × rowth) mod L` (식1)로 permutation 위치를 정의, jump=1이면 QC-LDPC가 되어 다른 구조부호에도 적용 가능.
6. 확장/코드구성: coset-based 구성으로 각 block row를 µ-block-divided row로 만들고 girth-4 제거, 이 제약이 메모리 압축 조건을 제공.
7. 구현 디테일: offset min-sum으로 CNU 출력을 min1(Q-1 bit)·min2(Q-1 bit)·min 위치(log2µ)·sign으로 압축, message storage를 µ개 Lb×QΛ + ρ개 L×1 block RAM으로 분할.
8. cycle bus 기법: 서로 다른 디코더 코어 간 동종 RAM을 데이터폭 확장으로 병합해 block RAM 수를 줄임, 대가로 iteration 1회분 대기(FIFO 필요).
9. 결과(FPGA, Stratix II, 140MHz, 4코어, 26 iter): throughput 305→598 Mbps(약 2배), logic register 8.35%↓, memory bit 22.3%↑, M4K RAM 효율 66.67→77.78%, 고 Eb/N0에서 offset min-sum 덕에 BP 대비 약 0.1dB 우위.
10. 한계: semi-random 부호 전용 구성이며 soft-read/양자화 비트수 명시 부족, cycle bus는 latency 증가, NAND/SSD 직접 타깃 아님.
