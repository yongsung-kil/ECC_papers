### B. 알고리즘 요약

1. 시스템: AWGN 채널, `(N,K)` LDPC 부호, message-passing 복호. 예시 `(504,252)`, `(1008,504)`, `(2032,1016)`, 모두 rate `1/2`, column weight 2~3.
2. 풀려는 문제: PEG 알고리즘은 큰 girth H행렬을 만들지만 sub-graph spreading 탐색이 무겁다(H가 크고 bit-node degree 높을수록 시간↑).
3. 모델: H를 Tanner bipartite graph로 보고 check node `V_c`, bit node `V_s`, edge set `E`로 표현.
4. 핵심기법: bit node를 무시하고 check node 간 연결만 담은 `T_{MxM}` topology matrix 도입 — `t_pq != 0`이면 p,q check node가 `t_pq`번째 bit node로 연결됨.
5. 핵심식: PEG의 sub-graph spreading이 `columnweight^(2(l+1))` 탐색이던 것을 topology matrix 사용 시 `columnweight^(l+1)`로 감소.
6. 절차: 첫 edge는 0이 가장 많은 행 선택, 이후 edge는 `T_{MxM}` 행을 따라 인접 check node로 spreading, 도달 못한/최원거리 그룹 중 최저 degree check node 선택 후 `T` 갱신.
7. 제약: girth-4 회피 위해 `N <= (M-1)/2` (column weight 2 기준), 최고 rate `R_max = 1 - 2/(M-1)`.
8. 학습/최적화: 없음(결정론적 그래프 구성).
9. 결과: 동일 `(N,K)`에서 PEG와 **동일한 H행렬·동일 BER** 생성, 탐색 시간(complexity)만 PEG 대비 감소(Fig.5/6 정성 비교).
10. 한계: 오프라인 H 구성 가속만, 부호 성능 이득 없음, HW·throughput·QC 구조 전무, message-passing 디코더는 일반론.
