### B. 알고리즘 요약

1. 시스템: systematic LT(rateless fountain) 부호, 무선 broadcast/incremental redundancy용, `k`=정보길이(100/1000), `R⁻¹=n/k=2.0~2.5`, BPSK/AWGN.
2. 문제: 기존 LT soft 반복복호(BP, `tanh` 기반, 식2~4)는 encoding/information 노드 그래프라 (a) 단순 조기종료 기준이 없고 (b) soft 정보 교환량이 적어 성능이 낮음.
3. 모델: 수신 soft demod 출력 `r`(길이 n)을 prior LLR로 사용, systematic이라 `c1..ck = u1..uk`.
4. 핵심기법: conventional Tanner 그래프의 encoding 노드를 bit 노드로 재해석해 `p1..p(n-k)` check 노드를 잇는 **modified Tanner graph** 구성 - LDPC와 동형이 되어 parity-check matrix 유도 가능.
5. 핵심식: 재표현 그래프에 LDPC min-sum 적용 - `Φ(a,b)=sign(a)sign(b)min(|a|,|b|)+LUTg(a,b)`, `LUTg=log(1+e^{-|a+b|})-log(1+e^{-|a-b|})` (식6~8).
6. 이점 분석: 1차 iteration soft출력 비교(식9 vs 10)에서 제안식이 더 큰 신뢰도 정보량을 전달 → BER 개선.
7. 조기종료: parity-check 방정식 만족 시 즉시 중단 - 기존은 단순 stopping 기준 없음, 평균 iteration 대폭 감소.
8. 부가: systematic raptor(LT+LDPC)에 적용 시 두 성분부호에 동일 복호 알고리즘 공유 → HW/메모리 단순화 기대.
9. 결과: k=100/1000, R⁻¹=2.5에서 conventional 대비 BER·평균 iteration 모두 개선(k 작을수록, Es/N0 클수록 격차 확대), 1차 iteration부터 우위(Fig.8).
10. 한계: HW 미구현, LT/rateless 특화이며 이득은 "LT를 LDPC처럼 다룬" 재표현 덕분 - min-sum/parity 조기종료 자체는 신규가 아님.
