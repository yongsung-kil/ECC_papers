### B. 알고리즘 요약

1. 대상은 LDPC convolutional code(LDPC-CC): semi-infinite transposed parity-check(syndrome former) `H^T`를 한 주기 `T`로 기술, rate `b/c`, memory `ms`, period `T`로 정의.
2. 문제: 기존 LDPC-CC 구성법은 대부분 좋은 block code를 먼저 만든 뒤 등가 syndrome former를 유도 - girth 보존 보장이 약하고 작은 frame size(작은 `ms`)를 효율적으로 다루지 못함.
3. 핵심 기법: block code를 거치지 않고 syndrome former에 **직접** `c×c` permuted(우측 회전) unit matrix를 무작위 위치에 삽입해 지정된 rate/memory/period/degree-distribution을 만족시키는 ensemble 구성법.
4. 사전 연결(pre-defined connections): `H0^T(t)`의 하위 `(c-b)×(c-b)`를 unit matrix로 만들어 fast encoding 보장, 회전량 `p`는 rate(`b/c`)에 따라 식 (1)로 결정.
5. 반복 절차: 각 phase 열 경계 `CS`(식 (2)) 내에서 uniform 난수 location·permutation을 뽑아 삽입, variable degree 충족까지 반복.
6. 4가지 체크로 후보 거절: 기존 연결 중복 금지, check-node degree 초과 금지, 길이-4 cycle 금지, ACE(`dACE,hACE`) 파라미터 위반 금지 - ACE로 stopping set 제거해 error-floor 완화.
7. 실패가 threshold 초과 시 forced-connection 모드로 모든 location·permutation을 직렬 시도, 그래도 실패면 `ms` 증가로 밀도 하향.
8. 복잡도: encoder `O(vs)`(`vs=c(ms+1)`), decoder LLR 저장 `c·Ip·(ms+1)` latching, latency는 termination bit `L`·processor 수 `Ip` 의존.
9. 결과: code A/B(girth 6, `ms=127`, `T=63`, rate 3/4, frame 1152) vs WiMAX LDPC-BC - ACE 적용 code B가 수렴영역에서 약 0.5 dB 우수, encoding 복잡도 약 10배 낮음, 단 decoding memory·latency는 더 큼.
10. 한계: 시뮬레이션(BER/FER)·복잡도 산정까지만, HW 미설계; termination bit로 인한 rate-loss(128 bit)와 큰 latency, 부호가 합성곱(LDPC-CC) 클래스.
