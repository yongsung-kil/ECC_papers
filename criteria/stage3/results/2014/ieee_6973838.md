### B. 알고리즘 요약

1. 시스템: Hadoop 등 클라우드 분산 스토리지의 복제(replication) 200% 오버헤드를 LDPC 부호 기반 프레임워크로 대체하려는 개념 제안.
2. 문제: erasure 부호는 공간효율은 좋으나 encoding/decoding·repair 계산·대역폭 오버헤드가 커 latency 악화.
3. 신규성1 (동적 파라미터화): QoS 정책·파일속성·리소스소비를 입력으로 LDPC의 `k`,`n`을 적응적으로 선택하는 QoS control policy.
4. 신규성2 (tailored LDPC): localized decoding 성질을 가진 systematic LDPC로 repair 시 모아야 할 chunk 수를 줄여 대역폭 절감; 가변 block size `n`으로 append 지원.
5. 신규성3 (예측모델): neuroevolution(진화 + ANN 학습)으로 파일 속성(read/write/access) 예측 → block placement·parameterization 유도.
6. 아키텍처: Cloud Storage Adaptation layer + LDPC Core(combiner/slicer→encoder/decoder) + block placement policy.
7. 구현 디테일: 구체적 H-matrix/부호구성·디코딩 알고리즘·양자화 서술 없음 (개념 수준).
8. 최적화: neuroevolution으로 예측 정확도 향상 목표 (구체 알고리즘·결과 미기재).
9. 결과: 없음 — RAID-6 대비 벤치마킹을 in-house/1000 VM/상용 클러스터 3단계로 "예정"이라고만 기술.
10. 한계: position/proposal 논문, rate/length·gate·throughput·BER 등 정량 결과 전무, HW 미설계, NAND 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 동적 k,n 파라미터화 (분산스토리지) | 대응 없음 | Prime ECC는 고정 QC-LDPC(H-matrix) 대상, 런타임 n/k 가변 개념 아님 |
| localized 디코딩 LDPC (erasure) | 대응 없음 | NAND용 error-correction(hard/soft decode)과 목적 상이, 분산 erasure 부호 |
| neuroevolution 예측모델 | 대응 없음 | 리소스/파일속성 예측 계층, ECC 코어와 무관 |
```

적용 가치: **낮음** — 클라우드 분산스토리지 시스템 제안 논문으로 NAND binary QC-LDPC의 부호·디코더·HW에 이식할 요소가 없다.
