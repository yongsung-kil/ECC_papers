### B. 알고리즘 요약

1. 시스템: 2b/cell MLC NAND, Gray 매핑(S0~S3), rate `8/9`·length `2KB` QC-LDPC(column weight `cw=4`), AWGN 채널, hard-decision 우선 후 실패 시 soft-sensing 2-step trial-and-error.
2. 문제: soft-decision sensing은 read latency가 sensing 정밀도에 비례해 커져 throughput을 떨어뜨림 — soft sensing 호출 빈도를 줄여야 함.
3. 관찰: MLC intra-cell 특성 두 가지 — (a) lower page BER `pl≈p`가 upper page `pu≈p/2`의 약 2배(불균형), (b) 같은 셀 두 비트 간 데이터 의존성(한 비트 값이 다른 비트 정오확률을 좌우).
4. 기법1(인터리빙): 같은 셀의 upper/lower page 비트를 한 codeword에 넣도록 sub-block(크기=병렬처리비트/2) 단위로 교대 배치 → intra-cell 의존성을 디코딩에서 사용 가능하게 함.
5. 부가효과: 인터리빙이 codeword 내 BER을 평균화해 lower page의 잦은 고비용 soft-sensing 호출을 줄임(cache read로 2nd/3rd sensing latency hide).
6. 기법2(의존성 인지 min-sum): 짝 비트 `Rc`와 그 hard-decision `dc`로부터 추가 belief `Ki`를 산출해 min-sum에 주입.
7. 핵심식: `Ki = αRi`(upper, `Rc<0 && dc<0`) / `0.5αRi`(upper, `Rc·dc<0`) / 양수(lower, `Rc·dc<0`) / else 0, α=0.75 — 짝 비트가 '1'일 확률이 높을 때 대상 비트 신뢰를 강화.
8. 디코딩 흐름: `Ej,i = α·(∏sign(Mj,i))·min|Mj,i|`(식3, scaled min-sum), `Mj,i = ri + Σ Ej',i + Ki`(식4), `Di = Ri + Σ Ej,i + Ki`(식5)로 매 iteration에 `Ki`를 더해 정정력·수렴 개선.
9. 결과: hard-decision 정정력이 baseline soft(2bit precision) 수준에 근접, 평균 iteration 감소, throughput 최대 85%↑(P/E 10K, sub-22nm 실측 BER 기반 시뮬, 디코더는 [10] 100MHz·76 cycle/iter 모델).
10. 한계: HW 미설계(기존 디코더 모델 차용), α·`Ki` 케이스가 heuristic, 2b/cell 전제라 TLC 확장은 정성적 언급뿐.
