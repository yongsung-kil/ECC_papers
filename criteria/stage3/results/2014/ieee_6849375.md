> 총평: TLC NAND 실측 기반 AEP-LDPC. LDPC 디코더 코어가 아니라 program disturb/retention/coupling을 EP table로 반영해 소프트정보(CER/LLR)를 정확화하는 채널 프런트엔드 기법. 아이디어는 우리 LLR 테이블에 이식 가능(중).

### B. 알고리즘 요약

1. 시스템: TLC NAND SSD, 셀당 8 VTH 상태·3 page, code rate `2/3`·`9/10` 측정.
2. 문제: 기존 EP-LDPC(MLC용)는 program disturb를 모델링 안 해 단기 retention(<2일) 정확도 부족, TLC는 VTH 마진이 좁아 더 심각.
3. 모델: 총 오류율을 셀별 CER로 분해, VTH는 1 state만 shift 가정; retention을 long(retention 지배)·short(program disturb+retention 공존) 2영역으로 분리.
4. 핵심 기법: read 시 3개 유닛 추가 — Unit1 retention time 추정, Unit2 short retention 처리, Unit3 EP table 접근해 CER 파라미터를 LDPC 디코더에 공급.
5. 핵심 식: `BER2nd_page = (|NcellA_init−NcellA_meas| + |NcellB_init−NcellB_meas|)/NP` — 2nd page 셀 재카운트로 retention time 추정.
6. 확장: `BER_total_est = (α+β+1)·BER2nd_page/3` — 1st/3rd page BER를 fitting 계수 α,β로 2nd page 기준 추정; short retention에선 Day0 최소값 사용.
7. 구현 디테일: soft-info를 49회 read 대신 7회 read + EP table lookup으로 대체(측정된 cycling/retention별 오류율); CER에 inter-cell coupling 승수 반영(예 1.086×1.187×…).
8. 학습/최적화: 실제 NAND 측정으로 EP table(neighbor VTH별 cell error ratio) 사전 구축.
9. 결과: BCH 대비 acceptable retention 12×, BER 2.8×; EP-LDPC 대비 max decode iteration 57% 감소; 디코딩 시간 8~32μs(10Gbps 가정).
10. 한계: LDPC 디코더 코어 알고리즘/부호구조 미기재, HW 미설계(측정+계산), EP table은 특정 TLC 공정 측정치 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CER/LLR 추정(EP table, coupling 반영) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` + `ecc_data.h` `PARAM_LLR` | 소프트정보 정확화 프런트엔드 — 우리 LLR/threshold 테이블에 대응 |
| 7-read soft 정보 생성 | `channel.cpp` soft-read 양자화 | 2SD/3SD read 비용 절감 아이디어와 정렬 |
| retention/disturb 모델 → 디코더 입력 | `decoder.cpp` `Get_VNU_Table_Idx()` (iteration별 LLR) | 채널 상태별 LLR 선택 로직에 개념 반영 가능 |
| LDPC 디코더 코어 | `decoder.cpp` `LDPC_Decoder()` | 논문은 코어 변경 없음(입력 LLR만 개선) |

**적용 가치**: 중간 — NAND 직접 타깃이고 EP table 기반 CER/LLR 추정은 우리 `channel.cpp`/`PARAM_LLR` LLR 테이블 설계에 이식할 여지가 있으나, LDPC 코어·부호구조는 손대지 않고 채널 프런트엔드에 국한되며 측정 데이터 의존도가 큼.
