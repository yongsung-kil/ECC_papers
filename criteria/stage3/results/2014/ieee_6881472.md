### B. 알고리즘 요약

1. 대상은 NAND flash SSD의 LDPC ECC. 19nm TLC는 2000 PE 후 raw BER ~10^-2, LDPC read latency가 오류수에 비례해 성능 오버헤드.
2. 문제: page retire/relocate로는 근본해결 불가(공정미세화로 신뢰도 급락, write disturb로 신규 데이터도 고BER).
3. 관찰1(error locality): flash 비휘발성으로 오류 셀이 erase 전까지 같은 위치 유지 → 같은 page 재read 시 오류위치 70~80% 동일.
4. 관찰2: LDPC 복호 latency는 입력 오류수에 직접 비례(오류 많을수록 flash sensing·iteration 증가).
5. 핵심기법 EC-Cache: 최근 read page의 검출오류(위치+valid value)를 SSD RAM에 캐시, 다음 read 시 raw data에 merge해 LDPC 입력 전 오류 사전정정.
6. 동작: read 시 EC-Cache 조회 hit면 line acquire·merge, LDPC 복호 후 diff로 EC-entry 재생성(LRU), write/GC 시 invalidate.
7. 구현트릭 offset coding: 절대위치 대신 상대 offset(8bit)+valid(1bit) 저장, 350bit 초과 시 dummy entry(255) 삽입. line 1kB(page 9kB의 1/9)로 data cache 대비 8배 라인 확보.
8. 실험: Virtex-5 FPGA 플랫폼으로 19nm TLC 실측 error locality, LDPC 시뮬(codeword 9kb, parity 922b)로 sensing/latency, SSDSim+DiskSim으로 MSRC 데이터센터 워크로드 시스템평가.
9. 결과: hit 시 오류 60%+ 사전정정 → BER 0.015에서 sensing 3→1, 복호 latency 25µs→<4µs, SSD read 응답시간 최대 2.6배 향상.
10. 한계: temporal locality 있는 SSD-cache 시나리오 한정(random access는 무효), LDPC 부호/디코더 알고리즘 자체 개선 아니고 시스템/펌웨어 레벨 전처리, memory partition 수동 프로파일 가정.
