### B. 알고리즘 요약

1. 대상 부호는 DVB-S2 IRA(Irregular Repeat-Accumulate) LDPC, Normal frame `N=64800`(Short `N=16200`), `M=360`, 11개 code rate.
2. 문제: G 행렬이 sparse하지 않아 인코딩 복잡도가 높고 대형 프레임에서 순차 처리 시 latency 과다.
3. 인코딩은 PCM의 하삼각 staircase(dual-diagonal) 구조 `B`를 이용해 `Pr = sum(m_z) XOR Pr-1` 식으로 G 없이 직접 수행.
4. Gomes 방식의 partially-parallel bit-node(vertical) 처리로 `M`개 정보노드를 동시 처리, barrel shifter로 circulant 회전 후 PARITY RAM 누적.
5. 디코딩은 full Sum-Product(Belief Propagation): LLR 초기화 후 CN update식 `psi(x)=-ln tanh(|x|/2)`와 sign XOR 사용.
6. BN update식 (6),(7)로 LLR 갱신·hard decision, parity check 만족 또는 iteration 한계(50)까지 반복.
7. LLR은 6bit 가변정밀 양자화(불신뢰 영역 finer, 신뢰 영역 coarse).
8. HW 트릭: psi(x)를 LUT로 구현, Zhang 26x6 LUT를 Oh&Parhi 가변양자화로 24x6(75% 축소), 메시지 메모리 폭 6->5bit(1sign+4mag) 축소.
9. 메모리 구조: TOP RAM(matrix A edge) + BOTTOM RAM(matrix B edge), 시프트 오프로딩으로 shuffle network 1개 제거해 throughput 개선.
10. 결과: Virtex-6 FPGA 이식, 인코더 LUT 3070/250MHz(기존 대비 축소), rate 3/5에서 BER 10^-8 시 6dB coding gain(실측). 고정소수점에서 rate 1/4,1/3,2/5 열화, 나머지 error floor는 10^-6 이하로 외부 BCH로 제거 가능. 정정능력 자체 개선은 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| full Sum-Product psi(x) CN update | 대응 없음 | Prime ECC는 min-sum 채택, full-tanh SPA 미사용 |
| psi(x) 24x6 LUT 가변양자화·메시지폭 축소 | decoder.cpp VN_Cal_HD() 등 / ecc_data.h PARAM_LLR | 양자화 아이디어는 참고 가능하나 SPA 전제라 직접 대응 낮음 |
| dual-diagonal(staircase B) 인코딩 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC도 dual-diagonal 사용, 개념 중복 |
| partially-parallel(M=360) bit-node 처리 | 디코더 HW 모델(z=32) | 병렬 개념 유사하나 lifting 단위·구조 상이 |
| TOP/BOTTOM RAM shift 오프로딩 라우팅 | GT.cpp Make_GT_HW() | edge 라우팅 최적화 개념만 유사, IRA 특화 |
```

적용 가치: 낮음 - DVB-S2 IRA·6bit soft·full Sum-Product 전제가 NAND용 고rate binary QC-LDPC + min-sum과 어긋나며 dual-diagonal 인코딩 등은 이미 보유.
