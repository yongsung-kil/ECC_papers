### B. 알고리즘 요약

1. 시스템: TDMR HDD, 듀얼리더 헤드로 인접 트랙 N/N+1 동시 판독, Voronoi 그레인 미디어(평균 9nm)·AWGN 전자잡음 채널모델, LDPC 외부코드 rate `R=0.9`/길이 0.5KB.
2. 문제: PMR 면적밀도 성장 둔화 → 듀얼리더 신호처리·코딩으로 추가 면적밀도 이득 확보 가능한지 시스템 관점 탐색.
3. 채널모델: 헤드응답 2D 가우시안, 미디어잡음 80%+전자잡음 20%, 트랙 동기 기록 가정.
4. 핵심기법1: 2-입력 2-출력 등화기(FIR 4개, `F11,F21,F12,F22`)를 4개 타깃으로 공동 최적화 후 2D 비터비 검출기로 두 트랙 동시 검출.
5. 식 의미: monic determinant 제약 `D=G11(0)G22(0)-G12(0)G21(0)=1`로 all-zero 해 방지, Lagrange `J=E[e1²+e2²]+λD` 최소화로 필터·타깃 도출.
6. 핵심기법2: cross-track ECC — write TMR로 트랙폭·SNR이 트랙마다 다를 때, 인접 두 트랙 비트를 한 LDPC 코드워드에 인터리빙해 코드워드간 SNR 평준화.
7. 복호: 표준 BP LDPC 디코더를 2D soft 채널검출기와 터보등화(채널반복 3회, 반복당 LDPC 8회)로 결합.
8. 최적화: 리더 배치 최적화(대칭배치), squeeze-to-failure로 1% BER 기준 면적밀도 산정.
9. 결과: 1D검출 듀얼리더 7.3%, 2D검출 10.17% 면적밀도 이득; cross-track ECC는 트랙 squeeze 2σ에서 ~3%, 3σ에서 ~5.5% 추가 이득.
10. 한계: SW 시뮬만·HW 미설계, HDD 자기기록 전용(write TMR·트랙폭 변동), 트랙폭 일치 시 cross-track 이득 거의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2D 등화기 / 2D 비터비 검출기 | 대응 없음 | HDD 자기채널 검출기, Prime ECC에 대응 모듈 없음 |
| cross-track 코드워드 인터리빙 | 대응 없음 | 트랙간 비트배치 스킴, NAND 페이지/QC 구조에 대응 개념 없음 |
| 표준 BP LDPC 디코더 | `decoder.cpp` `LDPC_Decoder()` (개념적) | Prime ECC는 min-sum 사용, 본 논문은 알고리즘 개선 없이 표준 BP 참조만 |

적용 가치: **낮음** — LDPC는 개선 없는 표준 BP 외부코드로만 등장하고, 기여는 HDD TDMR 채널·2D검출·트랙간 인터리빙이라 NAND binary QC-LDPC 디코더/부호설계에 이식할 요소가 없음.
