### B. 알고리즘 요약

1. 대상은 WiMAX QC-LDPC (rate 5/6, length 1536/1920/2304), 4x4 OpenRISC PE + 2D mesh NoC(X-Y 라우팅) 다중프로세서 플랫폼에 SW 구현.
2. 문제: ASIC 디코더는 면적/성능 효율적이나 다양한 LDPC 코드 지원 유연성 부족 → 다중프로세서로 유연성+속도 확보하려 함.
3. 디코딩은 layered offset min-sum(TDMP/layered scheduling): row별로 `Λmn=Ln-Λmn_prev`, `Λmn'=sign·max(min|Λmn|-β,0)`, `Ln=Λmn+Λmn'`, offset `β=0.125`.
4. 병렬화 핵심: 같은 sub-matrix row에서 확장된 H의 rows는 `Hmn=1` 열 위치가 pair-by-pair로 달라 병렬 갱신 가능 → N개 프로세서로 N개 row 동시 갱신(step2).
5. step3(y 갱신)도 열을 N등분해 각 PE가 분담, step4 체크도 N개 row 벡터곱을 병렬 검사.
6. 통신 문제: step2에서 각 PE가 계산한 L을 모든 PE에 송수신 → 데이터 교환량 폭증, 네트워크 혼잡 우려.
7. stepped 데이터교환: 16 PE를 4 PE 클러스터로 계층적 3단계 교환(cluster→part→전체)해 혼잡 회피, FIFO depth 절감.
8. master-slave: step4에서 중앙 근처 PE를 master로 두고 slave가 결과 전송→master 종합→재분배, 통신비용 O(N²)→O(2N).
9. 결과: cycle-accurate SystemC 시뮬(Thop=20cycle), rate 5/6에서 4/8/16 PE speed-up 약 3.45/5.91/9.34, 선행연구[8](TPMP 16PE, speed-up 6.06~7.68)보다 우수.
10. 한계: SW 다중코어 시뮬만(HW/ASIC 미설계), gate count·throughput(Gbps)·BER 곡선 미제시, 정정능력 개선 없음(병렬 구현 효율 논문).
