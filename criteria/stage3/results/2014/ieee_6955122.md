### B. 알고리즘 요약

1. 대상은 base(seed) matrix `B`의 nonzero를 circulant로 치환한 binary QC-LDPC block code (또한 `D`치환=convolutional, GF(2^m) companion=nonbinary); 예 rate `1/2~3/4`, length `576~64800`.
2. 문제: 밀도진화 degree distribution은 무한길이 점근최적이라 유한길이에선 short cycle로 error-floor↑ → 유한길이 조합최적화 필요.
3. 채널/모델: BPSK+AWGN에서 `FER(H,SNR)` Monte-Carlo를 탐색 기준으로, 채널모델을 최적화 루프에 삽입 가능.
4. 핵심 기법: girth profile `g=(g1..gc)`와 sliding-window girth `gw`, ACE-유사 기준으로 base matrix `B`와 degree matrix `W`를 열단위(greedy) 구성 (Algorithm 1).
5. 저weight 열은 큰 girth, 고weight 열은 낮은 girth를 허용 — short cycle 억제로 유한길이 FER 개선.
6. 확장: tailbiting `M→∞`로 LDPC convolutional code + sliding window decoding(SWD, window `ω`, shift `S`, delay `L`), nonbinary GF(2^8) QC로도 동일 골격 적용.
7. 구현 트릭: 인코딩 복잡도 위해 bidiagonal `Hbd` + 3-nonzero `h0` 형태로 선형복잡도 재귀 인코딩 확보.
8. 최적화: CDD→B→W 순차 preselect, 후보 수천개를 FER 시뮬로 선별하는 greedy 탐색(고속 girth 계산 활용).
9. 결과: R=1/2 신규 QC 코드가 WiMAX/DVB-S2 표준 대비 FER·BER 우위, nonbinary 코드는 4배 긴 DVB-S2보다 우수, FTN 최적화 시 약 0.6dB 이득.
10. 한계: HW 미설계, 시뮬만, nonbinary/convolutional/FTN까지 범위가 넓어 NAND binary QC 이식엔 일부만 해당.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| binary QC base/degree matrix 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 부호구조 로드부에 대응 — girth/ACE 최적화 결과를 PCM으로 반영 가능 |
| dual-diagonal 저복잡도 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 논문 bidiagonal 인코딩과 유사 목적(선형복잡도) |
| girth/ACE 유한길이 error-floor 개선 | `decoder.cpp` `LDPC_Decoder()` (간접) | 부호구조 개선이 복호 error-floor에 반영되나 디코더 알고리즘 자체는 무변경 |
| nonbinary/convolutional/FTN 부분 | 대응 없음 | binary QC-LDPC 전용 코드베이스라 이식 대상 아님 |

적용 가치: 중간 — binary QC-LDPC의 girth profile·SWG·ACE 기반 유한길이 부호탐색은 Prime ECC의 고rate H-matrix 설계·error-floor 개선에 참고 가치가 있으나, 고정 z=32·고rate 구조 재설계와 재검증 부담이 크다.
