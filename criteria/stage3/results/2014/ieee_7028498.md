### B. 알고리즘 요약

1. 대상은 SLC NAND(1bit/cell, ABL 아키텍처)로, 셀간간섭(ICI)을 encoder-side 부가정보로 정정하는 코딩 스킴이다.
2. 문제: scaling으로 ICI가 커지는데 encoder가 실제 ICI `S`를 알기 어렵고(read가 binary 결정), soft-decision LDPC는 read 속도 저하가 단점이다.
3. 채널 모델 `Y=X+S+Z=V+S+Z_read`(V=X+Z_write)로 두고, dirty-paper(Costa)/defective-cell(Gelfand-Pinsker) 관점으로 형식화.
4. 핵심 아이디어: write/erase 비대칭을 이용, **pre-read 1회**로 ICI 채널을 이진 `Y=X∘S⁺+Z` stuck-at defect 모델로 변환(`S⁺∈{0,1,λ}`).
5. `η_pre<η`로 pre-read level을 낮추면 vulnerable cell까지 stuck-at 1 defect로 간주해 subsequent ICI와 read noise 마진 확보.
6. 정정은 Heegard의 `[n,k,l]` **PLBC(partitioned linear block code)**: `c=G1·m+G0·d`, `d`는 defect masking parity.
7. 복호: syndrome `w=H^T·y`, `H^T·z=w` 만족하며 `‖z‖` 최소인 z 선택 후 `m=G1^T·(y+z)`.
8. 최적 masking parity `d`는 `‖c∘s⁺−c‖` 최소화가 지수복잡도라 [26,27]의 2-step encoding으로 근사.
9. 결과: `[n=1023,k=923,l]` PBCH로 `α`(ICI세기)·`σ_Zread`별 P(decoding failure) 저감, 최적 (l,r) 할당 존재(예 (10,90); η_pre=−1이면 (80,20)/(60,40)).
10. 한계: SLC 수치 시뮬만, HW 미설계, LDPC가 아닌 BCH계열 PLBC, encoder-side 정보로 write 속도 희생.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| pre-read로 ICI→stuck-at defect 채널 변환 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 채널/에러주입 전처리 관점의 유사성뿐, 부호는 다름 |
| PLBC additive encoding(defect masking) | 대응 없음 | Prime ECC 인코더는 dual-diagonal QC-LDPC, PLBC 아님 |
| syndrome 기반 error 선택 복호 | 대응 없음 | min-sum message-passing과 무관한 선형블록 복호 |

적용 가치: **낮음** - NAND ICI를 다루는 직접 논문이나 LDPC가 아닌 PLBC(PBCH) + encoder-side Gelfand-Pinsker 기법이라 Prime ECC의 QC-LDPC 디코더/인코더에 그대로 이식할 요소가 없고, pre-read defect 전처리 아이디어만 간접 참고 가능.
