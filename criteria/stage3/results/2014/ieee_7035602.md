> 총평: 표준 min-sum(NMS, S=0.875) TPMP를 (s,w) window로 부분병렬화한 합성-configurable QC-LDPC FPGA decoder로, Prime ECC가 이미 보유한 min-sum·부분병렬과 중복되고 알고리즘 신규성이 낮아 이식 가치 제한적.

### B. 알고리즘 요약

1. 시스템: WiMAX(1152/2304)·DVB-S2(64800, r=5/6)·DVB-T2(16200, r=7/9) QC-LDPC, Altera Stratix II FPGA, AWGN, λ 5bit 양자화.
2. 문제: full-parallel은 면적·라우팅 과다, serial은 throughput 부족 — 블록길이·rate가 다른 여러 표준을 하나의 유연한 구조로 처리 필요.
3. 코드구조: `H=[Hd Hp]`, Hp는 dual-diagonal, Hd는 `z×z` circulant 블록(`k=u·z`, `n-k=v·z`)로 구성한 semi-random QC-LDPC.
4. 핵심기법(min-sum TPMP): row processing `αij=∏sign(βij')·min|βij'|`, column processing `Zj=λj+Σαij'`, `βij=Zj−αij`, 매 iteration 후 `H·v^T=0` syndrome check로 조기종료.
5. 정규화: normalized min-sum scaling factor `S`(0<S<1) 곱으로 BER 개선, 실험적으로 `S=0.875` 채택.
6. 아키텍처: (s,w) calculation window로 sub-matrix 여러 개를 동시 처리, COMP(min1/min2/index)·SUM 유닛을 `t×w`개 배치, window를 이동시키며 동일 HW 재사용.
7. configurability: submatrix 크기 `T`, code rate `R`, block size를 합성단계 파라미터로 노출 → 여러 표준 대응.
8. 메모리: 채널 λ / check-node sum / min1·min2·index·sgn / CN→VN sign 4개 single-port RAM + shift register, FSM control-unit이 주소·window 이동 관리.
9. 결과: DVB-S2(64800,5/6) 최대 28.9Gb/s, DVB-T2 최대 19.5Gb/s, 약 32k logic element(≈8189~32734 표기 혼재), 선행 대비 3~20배 speedup, NMS가 SPA 대비 BER 열화 <0.2dB.
10. 한계: FPGA 합성만(ASIC/gatecount 없음), 정정능력 개선 아님(구조·throughput 중심), 본문에 encoder/decoder 서술 혼용 등 정합성 오류, 무선/방송 표준 특화.
