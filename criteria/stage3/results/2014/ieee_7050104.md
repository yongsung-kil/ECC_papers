### B. 알고리즘 요약

1. 대상은 systematic QC-LDPC block code(WiMAX 802.16e, WiFi 802.11n/ac), rate 1/2~5/6, block length 1152/1944/2304, `Hp`는 dual-diagonal·`Hd`는 circulant sub-matrix `A_ij`.
2. 문제: 기존 LDPC HW는 대개 단일 부호족/단일 block length·rate 고정 - 다양한 표준 QC-LDPC를 지원하는 configurable·고throughput 아키텍처 필요.
3. 인코더: `Hp`(dual-diagonal)·`Hd`(circulant) 규칙성 덕에 행렬을 메모리 저장 없이 `A_ij=A^{i·j}`로 생성, parity `p_i`를 XOR+flip-flop으로 계산(식 3,4); calculation window가 여러 sub-matrix를 병렬 처리.
4. 디코더: min-sum(MS) 기반 TPMP, check-node update `α_ij = ∏ sign(β_ij')·min|β_ij'|·S`(식 5, scaling factor `S`), variable-node update `β_ij = Z_j − α_ij`(식 7).
5. scaling factor `S=0.875`(normalized MS)로 min-sum BER 성능 개선, floating-point 대비 0.2 dB 손실.
6. 조기종료: column 처리 후 채널정보와 α 합으로 추정 codeword 계산, syndrome `H·v^T=0`(식 8) 충족 시 iteration 중단.
7. HW: 두 calculation window(check-node용 Comp, variable-node용 SUM), 각 `s×w` sub-matrix를 클록당 처리, 완전 병렬 shift-register 재사용.
8. 메모리 절감: 모든 메시지를 저장하지 않고 각 행의 COMP 결과(`min1,min2,index,sign`)와 각 열의 총합만 저장 - 선행 설계 대비 2~11배 적은 메모리.
9. 결과: encoder throughput 최대 32.8 Gbps, decoder 732 Mbps(WiFi rate 5/6), 양자화 (채널,extrinsic)=(5,7)bit, max 20 iteration; 선행 디코더 대비 2~9배 빠른 throughput.
10. 한계: FPGA 합성·BER 곡선까지만, gatecount@공정 없음; 알고리즘은 표준 normalized MS로 신규성 낮고 무선표준 부호 대상(NAND soft-read/error-floor 미언급).
