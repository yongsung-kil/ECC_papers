### B. 알고리즘 요약

1. 대상: WiMAX (1152,2304) rate-1/2 QC-irregular LDPC, expansion factor `m=96`, 메시지 양자화 `(γ,β)=(6,4)`bit(축소 시 (5,3)).
2. 문제: FPGA에서 layered QC-LDPC 디코더의 BRAM 블록·slice 자원 소모와 라우팅 복잡도가 커서 저비용·고주파 구현이 어렵다.
3. 알고리즘은 표준 `Min-Sum` layered scheduling: `α_z,n=γ_n−β_z,n`(VN), `β_z,n=(∏sgn)·min|α|`(CN), `γ_n=α+β`(AP-LLR 갱신)을 layer 순차 처리.
4. 핵심 기여1: VNU+CNU를 하나의 VCN 유닛으로 병합, AP-LLR(`γ`)을 한 번에 하나씩 **직렬 처리**해 AP-LLR 메모리 액세스 포트 수와 라우팅 네트워크를 축소.
5. 핵심 기여2: VCN 데이터패스의 산술·비교 연산 블록을 Distributed RAM 기반 **ROM(LUT)으로 치환**(sign-magnitude 표현), critical path 단축으로 주파수 최대 15% 상승(312MHz).
6. β 메시지 압축(`β_C`): sign들 + min1/min2 magnitude + min1 인덱스만 저장해 메모리 폭·크기 절감.
7. BRAM configurable port width(8~72bit)로 `γ`/`β` 메시지 패키징, 예시서 총 12 BRAM(baseline)/11 BRAM((5,3)).
8. 병렬화: layer당 `m`개 VCN 병렬 + aggressive pipelining, throughput 식은 직렬 `γ` 처리항이 지배적.
9. 결과: XC5VLX50T에서 slice ~6300, BRAM 11~12, throughput 800Mbps(coded), 타 FPGA 대비 slice 최대 3배·BRAM 1오더 절감; (5,3) 양자화는 BER 10^-6에서 (6,4) 대비 0.25dB 미만 열화.
10. 한계: FPGA(Xilinx) 자원 특화 최적화라 ASIC에 부적합, 오류정정 성능 개선은 없음(자원/주파수만), rate 유연성은 있으나 동일 base matrix·expansion 전제.

```

| layered/AP-LLR 갱신 VCN 처리 | decoder.cpp LDPC_Decoder(), VN_Cal_HD() | layered 스케줄·AP-LLR 갱신 대응, 신규성 없음 |
| β 메시지 압축(min1/min2+idx 저장) | decoder.cpp CNU_Update_New_Mag() | CN 메시지 min1/min2 압축 표현은 우리 구조와 유사 |
| ROM/LUT 데이터패스, BRAM 패키징 | 대응 없음 | Xilinx FPGA 특화 자원 기법, ASIC bit-exact 시뮬에 대응 모듈 없음 |
| γ barrel-shifter 라우팅 네트워크 | 대응 없음 | FPGA 라우팅 구조 특화, GT.cpp(edge MUX)와는 목적 상이 |
```

적용 가치: **낮음** — 알고리즘은 우리가 이미 구현한 표준 layered Min-Sum이고, 핵심 기여인 distributed-RAM ROM 치환/BRAM 패키징은 Xilinx FPGA 전용 자원 최적화라 ASIC 대상 Prime ECC 3.1에는 직접 이식할 것이 없다.
