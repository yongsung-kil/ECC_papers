### B. 알고리즘 요약

1. 문제: 집합 `X(M)={x0..x(M-1)}`에서 첫 `W`개(`W>2`) 최댓/최솟값을 병렬 VLSI로 찾기 (`W ≤ M/2`).
2. 배경: non-binary LDPC(EMS/CN)·K-best MIMO·turbo product code 디코더가 partial sorting을 요구하나, `W>2` 일반 병렬 구조 분석이 부재.
3. 핵심 착상: 비교기 기반 SN 대신 radix sort — 데이터를 MSB→LSB로 비트단위 분석.
4. 기법 1단(BWA): 각 `xi`에 인접비트 logic-AND를 재귀 적용, `hi,j = hi,j+1 ∧ xi,j`. MSB=1이고 단조(1→0 전이만) 시퀀스면 `H` 행렬의 첫 W 비영행 열이 max 위치.
5. 식 의미: `hi,0=1 ⟺ xi=2^N−1`; H를 LSB행→MSB행으로 훑으면 열=최댓값 인덱스.
6. 확장(완전판): 비단조/중복 원소 대응 위해 zero-row 처리 게이트 추가 → 수정행렬 `Ĥ`(N×M), `ĥi,0` 검사로 max 선택.
7. 구현 디테일: 2's complement는 MSB flip으로 비음수화. output generation circuit(g-slice + selection block + N combiner)으로 max를 0으로 치환한 새 집합 생성, W-stage 캐스케이드.
8. (학습절차 없음 — 순수 조합회로 설계)
9. 결과: 90nm CMOS P&R에서 area가 타 기법의 50% 이하(최악의 경우 약 50%), area–delay product 대부분에서 우수, 파이프라인 stage당 area overhead <35%.
10. 한계: 부호/디코더 알고리즘 자체 기여 아님(sorter HW 컴포넌트), binary min-sum의 W=2에는 과대, 절대 수치는 표에만 존재.
