### B. 알고리즘 요약

1. 시스템: binary BPSK / AWGN 채널, Sum-Product(BP) soft-input 복호, 최대 iteration 50, `N=30000`, `R=0.8`.
2. 문제: (P)UM convolutional code를 constituent LDPC로 구성해 sparse PCM을 갖는 block LDPC 앙상블을 얻으려 함.
3. 모델: QC-LDPC 앙상블 `E_QC(l,n0,m,I)` - identity `I`의 cyclic shift `I_pij`로 PCM 구성, `n=m·n0`.
4. 핵심 기법 1단: (P)UM code를 semi-infinite PCM `H`(식 2)로 정의, `H0(t), H1(t)`를 `E_QC`에서 등확률 선택.
5. 핵심 식: 제약 `l0·m0 = l1·m1 = M`, `n00·m0 = n01·m1 = n`으로 서브블록 크기 정합.
6. 확장: tail-biting으로 유한 block matrix `H4`(식 4) 구성 → `(L,N0)=(l0+l1, n00+n01)` regular LDPC, 길이 `N=4n`.
7. 구현 디테일: `H4`가 4-cycle-free일 조건을 8개 서브행렬의 4-cycle 부재로 판정(girth 관리).
8. 최적화: 학습 없음, MatLab로 PCM 생성, column weight를 3~6으로 변화시키며 실험.
9. 결과: column weight 낮을수록 성능 우수, `(2,10)+(1,5)`(column weight 3)가 최고, 랜덤 QC-LDPC(3,15)와 동등 수준.
10. 한계: HW 미설계, 시뮬만, 성능이 기존 랜덤 QC-LDPC 대비 우위 없음(not worse), gate count/throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC PCM 구성 (cyclic shift I_pij) | ecc_top.cpp Load_PCM() | 낮음 - PUM 기반 구성으로 Prime ECC 고정 base+shift와 다른 방식 |
| 4-cycle-free / girth 조건 | 대응 없음 | 없음 - Prime ECC PCM은 사전 확정, 구성 단계 모듈 미해당 |
| Sum-Product(BP) 복호 | decoder.cpp LDPC_Decoder() | 낮음 - Prime ECC는 min-sum 근사 사용, full BP 아님 |
```

적용 가치: **낮음** - 이진 QC-LDPC 재료를 쓰지만 PUM 기반 구성법이며 성능/HW 이점 없어 NAND 고정 QC-LDPC에 이식 실익이 적다.
