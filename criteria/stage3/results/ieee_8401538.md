> 총평: SC-LDPC 슬라이딩 윈도우 복호의 복잡도 절감 기법으로 자기기록(BPMR) 특화이며, NAND용 block binary QC-LDPC와 부호/복호 구조가 달라 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: SC-LDPC(=LDPC convolutional) 부호를 BPMR(bit-patterned media) 자기기록 turbo equalization에 적용, `(3,21)`-SC-LDPC, rate `6/7`, coupling length `L=50`, expansion `N=100`, memory `mcc=2`, 윈도우 `W=8`.
2. 문제: SC-LDPC 슬라이딩 윈도우 복호(U-WD)는 윈도우 내 모든 VN을 매 iteration 갱신 → 복잡도 과다, turbo iteration 반복 시 더 심함.
3. 채널 모델: 2-D BPMR 채널(areal density 2 Tb/in², 식 6-7), BCJR 검출기 + SC-LDPC 복호가 LLR 교환하는 turbo equalization.
4. 핵심 기법: 각 코드블록의 soft-BER 지표 `Pˆ`(식 5)를 추정하여 연속 블록의 BER가 같으면(조건 식 9: `(Pˆ_{t+s-1})^{αs} ≤ Pˆ_{t+s-2}`) 여러 블록을 한 번에 shift하는 DS-WD.
5. 식 의미: 첫 블록(`w=1`)이 가장 많이 갱신되어 신뢰도 최고 → 이미 수렴한 블록은 추가 갱신 불필요하다는 관찰(Fig.3)에서 다중 shift 유도.
6. 확장: NP-WD 비균일 스케줄을 DS-WD에 결합한 DS-NP-WD — shift 수 `s`에 따라 다음 위치의 업데이트 스케줄을 조정(Fig.4).
7. 구현 디테일: 최대 shift `s ≤ mcc+1`, shifting factor `α2=1.4`, `α3=1.7`를 Monte Carlo로 선정.
8. 최적화: shifting factor αs를 시뮬레이션으로 튜닝(BER 손실 없는 최대값 선택).
9. 결과: SNR 5.1 dB에서 평균 업데이트 `Uavg` U-WD 223.84 / NP-WD 133.34 대비 DS-WD 85.04(각 62%, 36.2%↓), DS-NP-WD 71.98(67.9%, 46%↓), BER 손실 없음.
10. 한계: 시뮬레이션만, HW 미설계, Sum-Product(`tanh`, 식 3) 사용, SC-LDPC/자기채널 특화로 NAND block QC-LDPC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| soft-BER 기반 동적 shift/조기종료 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 조기종료 개념은 유사하나 Prime ECC는 CRC 기반, soft-BER 지표와 메커니즘 다름 |
| 메시지패싱 LLR 갱신(CN/VN) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | 논문은 Sum-Product tanh, Prime ECC는 min-sum — 알고리즘 상이 |
| 비균일 업데이트 스케줄(NP-WD) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 루프 | 스케줄 개념 참고 가능하나 윈도우 전용이라 직접 이식 불가 |

적용 가치: **낮음** — SC-LDPC 윈도우 복호 + 자기기록 turbo equalization 특화 기법으로, NAND용 block binary QC-LDPC(min-sum, CRC 조기종료 보유)와 구조·알고리즘이 근본적으로 달라 재사용 가치가 낮다.
