### B. 알고리즘 요약

1. BI-AWGN 채널에서 파리티체크 행 각각을 SPC 대신 임의 컴포넌트 코드(Hamming, BCH 등)로 대체한 GLDPC(irregular 포함) 코드를 다룬다.
2. 문제: GLDPC는 컴포넌트 코드의 SISO 복호기가 복잡한 비선형 함수라 표준 Density Evolution(DE)으로 extrinsic 정보의 확률분포(pdf)를 계산할 수 없다.
3. 컴포넌트 코드 복호기로 Max-log-MAP(또는 Lipschitz 조건을 만족하는 임의 SISO)을 가정, `f_j(x)=½(max_{v∈X+} 〈v,x〉 - max_{v∈X-} 〈v,x〉)` 형태의 extrinsic 함수를 정의한다.
4. Theorem 1: extrinsic 누적합 `S_{j,n}`이 `l_{j,n}`을 Lipschitz 상수로 갖는 `H`-Lipschitz 함수임을 증명(`|S_{j,n}(x2)-S_{j,n}(x1)| ≤ l_{j,n}‖x2-x1‖_2`), `l_{j,n}`은 PCM `H`와 노이즈 분산으로만 결정.
5. 이 Lipschitz 성질은 concentration of measure 원리(Gaussian 측도 집중부등식)와 결합해, 총 LLR `T_j^{(n)}=R_j+S_{j,n}`의 꼬리분포를 Gaussian 분포로 상한시키는 데 필요.
6. Theorem 2: 반복 `n`회 후 LLR `T_j^{(n)}`의 CDF/생존함수가 평균 `a_j+m_{j,n}`, 표준편차 `(b_j²+l_{j,n}²)^{1/2}`인 Gaussian 함수로 상한됨을 도출(부등식 (8)(9)).
7. 부수 결과: regular code(`d_c`, `d_v` 고정)의 경우 `l_{j,1}` 값을 폐형식으로 정리 (`l_{j,1}=(2/σ1)√(d_v(d_c-1))`).
8. 학습/최적화 절차 없음(순수 수학적 증명, 시뮬레이션 없음).
9. 결과: 반복복호 오차분포의 꼬리가 Gaussian보다 가벼움(light-tailed) 을 증명 → Gaussian approximation/EXIT-chart 분석이 GLDPC 실제 iterative threshold의 상한(보수적 추정)을 제공함을 이론적으로 정당화.
10. 한계: 순수 이론/증명뿐이며 실제 수치 실험, HW 구현, 특정 코드 성능 비교 없음. 결과는 반복 임계값의 "상한"만 제공하고 실제 성능 예측치는 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC (SPC→Hamming/BCH 컴포넌트 코드 대체) 구조 | 대응 없음 | Prime ECC는 SPC 기반 표준 QC-LDPC(`PCM_b`, base+shift)로 GLDPC 컴포넌트 코드 구조 미보유 |
| Density Evolution / EXIT-chart 상한 이론 | `ecc_top.cpp` `Load_PCM()` (H-matrix 설계 단계 참고용) | H-matrix 설계·검증 시 이론적 배경 지식으로만 참고 가능, 직접 이식 코드 없음 |
| Max-log-MAP SISO 컴포넌트 복호 | 대응 없음 | Prime ECC 디코더는 min-sum 기반 SPC(single parity check) 구조로 컴포넌트 코드 SISO 복호기 없음 |

> 적용 가치: 낮음 — GLDPC라는 다른 부호 클래스(SPC가 아닌 컴포넌트 코드 사용)의 이론적 반복임계값 상한 증명으로, Prime ECC의 표준 binary QC-LDPC(SPC 기반) 구조·코드에 직접 이식할 요소가 없음.
