> 총평: 2/3-bit 초저정밀 LDPC 디코더의 LUT(quantization value+threshold)를 신경망으로 공동 학습하는 기법으로, Prime ECC의 hard-1bit/soft-2~3bit LLR 테이블 설계와 직접 맞닿아 이식 가치가 높음.

### B. 알고리즘 요약

1. 시스템: BPSK, binary-input AWGN 채널, 5G LDPC BG1 기반 QC-LDPC (lifting size `Z=384`), rate `R=0.5~0.92` 4종 코드에 대해 검증.
2. 문제: 저정밀(2~3bit) FAID(Finite Alphabet Iterative Decoder)/NS-FAID는 min-sum 대비 복잡도가 낮지만 기존 설계(non-surjective framing function `F(·)`)는 성능 손실이 크고, 기존 NN 기반 FAID(RQNN)는 quantization threshold를 사전 고정해야 해 유연성이 부족.
3. 가정/모델: 메시지 갱신은 CN-update `Φc`(min-sum과 동일한 sign·min 연산), VN-update `Φv`, likelihood 근사 `Υ`의 3종 함수로 구성되며, `Φv` 내부의 non-surjective framing function `F(x)`(양자화 값 `Q0,Q1,Q2` + 임계값 `t1,t2,t3`)가 LUT의 핵심.
4. 핵심 기법: 5-layer RQNN(CN-update layer 2개, VN-update layer 2개, likelihood layer 1개)을 2회 반복마다 unfold하여 `F(·)`의 양자화값·임계값을 학습 파라미터로 최적화.
5. 핵심 식: 임계값 이산탐색 문제를 softmax로 완화 — `p_ij(x) = exp(g_ij(x)/ζ) / Σ_k exp(g_kj(x)/ζ)` (온도 `ζ`를 학습중 점진적으로 낮춰 one-hot 분포로 수렴), 이를 통해 미분 불가능하던 threshold 선택을 학습 가능하게 만듦.
6. 확장: 양자화값 자체는 STE(Straight-Through Estimator)로 학습 — forward는 `round(x)`, backward는 미분가능한 `softround(x) = m + tanh(δ(x-m))/(2·tanh(δ/2))`로 근사.
7. 부수 트릭: loss는 BER의 soft 대리 지표인 `L = -1/n Σ [x_i log(1-λ(l_i)) + (1-x_i) log(λ(l_i))]` (λ=sigmoid)를 사용해 전 구간 미분 가능하게 설계.
8. 학습 절차: PyTorch/GPU, 15,000 샘플, 20 epoch, lr=0.01, batch=256, ADAM; rate 0.92는 10 iteration(파라미터 5세트), 나머지는 20 iteration(파라미터 10세트)으로 2-iteration 단위 파라미터 공유.
9. 결과: 2-bit/3-bit 모두 기존 NS-FAID([3]) 대비 FER에서 유의미한 이득, floating-point BP 대비 근접 성능, quantized MS 대비도 손실 미미(4종 rate 코드 전부에서 일관).
10. 한계: AWGN·BPSK 심플 채널 가정, FPGA/ASIC 등 HW 구현·복잡도(gate/throughput) 수치 없이 시뮬레이션(FER)까지만 검증.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| framing function F(·) (2/3-bit LUT, 양자화값+임계값) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC의 iteration별 LLR threshold 테이블 설계와 동일 목적(저비트 soft LUT 최적화) |
| CN-update Φc (min-sum 기반) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 min-sum 연산 구조, 논문은 그 출력을 저비트 LUT로 재양자화 |
| VN-update Φv / 양자화 | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD), `decoder.h` `Get_VNU_*` | 2~3bit 연판정(soft-2~3bit)이 Prime ECC의 2SD/3SD 지원 범위와 직접 대응 |
| RQNN 오프라인 학습 절차 | 대응 없음 | Prime ECC는 학습 파이프라인 없음, 별도 오프라인 최적화 도구로 이식 필요 |
```

적용 가치: `높음` — 2/3-bit LUT/threshold 공동 학습이 Prime ECC의 soft-2~3bit LLR 테이블(`PARAM_LLR`) 설계와 정확히 같은 문제를 다뤄, NAND read 비용을 낮추면서 waterfall 성능을 개선할 여지가 큼.
