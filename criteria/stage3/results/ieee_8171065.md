### B. 알고리즘 요약

1. 목표: 극저 SNR(deep-space, ECM) 채널용 rate ~0.03 LDPC 부호를 대수적으로 설계 (기존 PEG는 저rate 영역에서 성능 부족).
2. 채널/모델: BPSK + AWGN, 극저 SNR(-15dB급). 저rate에서 extrinsic 정보가 SNR에 따라 느리게 증가 → VN에 더 많은 edge 필요.
3. 기본 검사행렬: 원시다항식 `f(x)=x^k+x^p+1` (GF(2))로 생성한 m-sequence의 절단 구간을 codeword로 사용, 제약 `a_i+a_{i+p}+a_{i+k}=0`이 각 check row.
4. 4-cycle free 조건: `H(0)`은 `k=2p`일 때만 4-cycle 발생 → `k≠2p`로 회피.
5. 검사행렬 확장: sample sequence(`s=2^m`)로 파생 다항식 `f_m(x)=x^{2^m k}+x^{2^m p}+1`의 제약을 추가해 `H(m)`을 쌓아 VN degree 증가.
6. 확장 4-cycle free: `H`의 4-cycle 회피 조건 식(13)의 `k,p` 관계식으로 보장.
7. 가상변수노드 확장: m-sequence의 미사용 원소를 채널정보 없는 virtual node(수신값 0)로 추가, 실변수노드 간 extrinsic 정보 전달 다리 역할.
8. 인코딩/복호: codeword가 m-sequence 구간이라 인코딩은 LFSR로 단순 실현, 복호는 표준 BP.
9. 결과: rate 0.03, length 3000, 최대 50 iteration, BER 1E-5에서 PEG 최적화 부호 대비 약 2.2 dB 이득. 확장 길이 ≈ code length일 때 최적.
10. 한계: 극저rate 전용, HW 미설계, 시뮬만, Shannon limit과는 거리(짧은 길이 때문).
