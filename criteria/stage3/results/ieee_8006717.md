### B. 알고리즘 요약

1. 대상은 irregular binary LDPC(Tanner graph, girth `g>4`); 예제 C1/C2는 802.16e rate 0.5, 길이 576/1056, C3은 PEG girth-8 길이 504.
2. 문제: ETS(특히 leafless ETS=LETS)는 error-floor 주범인데, irregular 부호의 전수 ETS 탐색 알고리즘(branch-&-bound)은 길이·크기가 커지면 곧 불가능해짐.
3. 모델: parity-check matrix H를 Tanner graph로 표현, `(a,b)` = variable node `a`개·unsatisfied check `b`개 구조; ETS는 degree 1/2 check만 갖는 TS.
4. 핵심 기법: variable-regular용 dpl 특성화(depth-one tree `dot`, `path`, `lollipop` 세 확장)를 irregular로 일반화, simple cycle 또는 single VN에서 계층적 확장.
5. 핵심 표현: irregular ETS를 `normal graph`/`quasi-normal graph`로 사상 — 여러 non-isomorphic 구조가 소수의 normal graph로 투영됨(Prop 2).
6. 확장: leafless 아닌 ETSL을 LETS 하위구조 유무로 ETSL1/ETSL2로 분류하고 `dot1_k`(m=1) 확장으로 생성(Prop 5,6, Algorithm 1).
7. 구현 디테일: backward recursion으로 각 크기 a에 필요한 b 상한 `b_a^max`를 유도(Thm 2)해 탐색 범위를 보장·최소화, dv별 class 추적.
8. 최적화/이론: Prop 3,4로 각 확장이 `(a,b)` class를 어떻게 이동시키는지 정량화, dot 확장이 b를 가장 크게 감소시킴을 이용.
9. 결과: C1/C2/C3의 `a≤10,b≤2` ETS/LETS multiplicity가 [1]과 완전 일치, 런타임 38/18/20분 vs [1] 41/189/273분 — 블록길이 증가 시 오히려 복잡도 감소.
10. 한계: 탐색·특성화 도구일 뿐 부호/디코더 성능을 직접 개선하지 않음, HW 미설계, 4-cycle-free 가정.

| H-matrix(Tanner graph) 구조 분석 | `ecc_top.cpp` `Load_PCM()` | 로드된 QC-LDPC H-matrix에 trapping set 탐색을 적용할 접점 |
| trapping-set 기반 부호/디코더 설계 지침 | 대응 없음 | 본 논문은 탐색만, 실제 부호수정/디코더 post-processing은 미제공 |

적용 가치: 중간 — Prime ECC의 QC-LDPC H-matrix error-floor를 진단하는 오프라인 분석 도구로 쓸 수 있으나, irregular 대상·girth 가정 정합 필요하고 직접적 성능개선 기법은 아님.
