### B. 알고리즘 요약

1. 대상은 rate `R=1/q`인 Repeat-Accumulate-Accumulate(RAA) 코드 — 반복부호+2개의 rate-1 memory-1 누산기(accumulator)를 인터리버로 직렬연접한 구조(LDPC가 아닌 turbo-like 코드).
2. RA 코드는 인코딩 복잡도가 낮으나 최소거리가 block length에 선형으로 증가하지 않아(not asymptotically good) error-floor 성능이 나쁜 문제가 있고, RAA는 이를 개선하지만 반복 디코딩 하의 실제 저 BER 영역 성능(trapping set 영향)은 특성화가 부족함.
3. 채널 모델은 이진대칭채널(BSC), all-zero codeword 전송 가정.
4. 핵심 기법 1단: Gallager-Zyablov-Pinsker bit-flipping 변형 디코딩을 RAA의 factor graph에 적용해 `dtrap`(minimum trapping distance)의 앙상블 평균 weight enumerator `A_{d,d1,w}`를 유도.
5. 핵심 식 `(5) v1 - q*vq > 0`은 trapping set 부재 조건이며, 이를 통해 정규화된 trapping distance `ρtrap`의 점근 상한 `Ttrap(ρ)`(식 8)을 도출 — q≥3일 때 `dtrap`이 block length에 선형 증가함을 보임(계수는 매우 작음).
6. 핵심 기법 2단: BP 디코딩에 대해 `(a,b)` trapping set(변수노드 a개, 그 유도 부분그래프 내 홀수차수 체크노드 b개) 정의를 도입, 각 구성부호(반복부호+누산기)의 input-output trapping set enumerator(IOTSE)를 uniform interleaver 개념으로 결합해 RAA 전체의 TSE `A^{RAA}_{a,b}`를 closed form(식 11)으로 유도.
7. 부수 트릭: 누산기의 trapping set 계수를 trellis 표현(각 엣지를 `si/so|c` 3-tuple로 라벨링)으로 변환해 경로 열거 문제로 환원(Theorem 3, 식 12).
8. 별도 학습/최적화 절차 없음 — 해석적 앙상블 평균 계산 및 수치 evaluation(그림 2, 4, 5).
9. 결과: bit-flipping 기준 R≤1/3에서 `dtrap`이 선형 증가하나 성장률 계수가 극히 작음; BP 기준에서는 불만족 체크노드가 있을 때 TSE가 항상 더 크고, 최소 trapping set 크기가 선형 증가하는 영역이 존재하지 않음(negative result).
10. 한계: LDPC가 아닌 RAA 특유 구조에 한정된 조합론적 분석, 실제 코드 구성·시뮬레이션·HW 결과 없음, error-floor에 대한 (a,b) trapping set의 실질적 영향은 "별도 평가 필요"로 미완.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RAA 코드 구조(반복+누산기+인터리버) | 대응 없음 | Prime ECC는 QC-LDPC 전용이며 RA/RAA 계열 코드는 구조가 근본적으로 다름 |
| trapping set enumerator (error-floor 이론분석) | `ecc_top.cpp` `Load_PCM()` | error-floor 대책과 개념적으로 연결되나 QC-LDPC PCM에 대한 구체적 적용 방법 제시 없음 |
| bit-flipping 변형 디코딩 | `decoder.cpp` `VN_Cal_HD()` | hard-decision 계열이라는 점만 관련, RAA 특유 factor graph 기반이라 직접 이식 어려움 |

> 적용 가치: 낮음 — RAA(turbo-like) 코드에 특화된 trapping set 조합론 분석으로, QC-LDPC 전용 Prime ECC 3.1과 코드 구조가 달라 직접 이식 불가.
