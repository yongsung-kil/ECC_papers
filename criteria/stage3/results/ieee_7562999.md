### B. 알고리즘 요약

1. 시스템: column weight `j=3`인 QC-LDPC 부호의 exponent(shifting) matrix `E(H)`(크기 `3×K`)를 구성하는 문제. rate는 예제상 0.27~0.5, length는 150~671bit.
2. 문제: 기존 explicit construction(예: Zhang & Sun의 `(3,L)` girth-8 구성)은 매 block-column 크기마다 CPM(circulant permutation matrix) 크기가 상대적으로 크고 계산복잡도가 있음.
3. 채널모델: BPSK + AWGN, message-passing 복호(iteration=50)로 BER 시뮬레이션.
4. 핵심기법: "base matrix" 방법 - `(3,5)` 크기의 최초 base matrix를 알고리즘으로 생성한 뒤, 이를 그대로 재사용해 `(3,6)→(3,7)→…→(3,K)`로 한 열씩만 추가 계산(element-wise element method)해 확장.
5. 핵심식: 길이 `2l` short cycle 회피조건 `Σ(a_{ik,jk} - a_{ik+1,jk}) ≢ 0 mod L`; girth 8은 2행을 `E(H)_{2nd row} = (2^s · t) - 1` 규칙(s∈{1,2,3}, t=홀수)으로 생성 후 3행을 원소별 탐색.
6. 확장: girth 10/12는 별도 명시적 규칙 없이 2행부터 모든 원소를 element-wise element method(부분행렬 girth 검사 반복)로만 생성, CPM 크기 상한 `P ≤ 30K`에서 최적화.
7. 구현디테일: 3행 특정 인덱스에서 조건을 만족하는 값을 못 찾으면 해당 열 이전 인덱스를 삭제하고 처음부터 재탐색(백트래킹).
8. 최적화: 명시적 수식 기반 알고리즘(girth 8)과 탐색 기반 백트래킹(girth 10/12)의 결합, 별도 DE/GA/NN 없음.
9. 결과: girth 8 기준 CPM 크기가 Zhang의 방법 대비 크게(K=12일 때 108→121, 오히려 소폭 증가하는 경우도 존재) 비슷한 수준, BER 성능은 기존 QC-LDPC(Zhang) 및 shortened array code와 유사~근접.
10. 한계: HW 미설계, 시뮬만 수행(BER 1e-1~1e-6 구간), column weight 3의 low-density/low-rate 부호에 국한, error-floor 영역(초저BER)까지의 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| base matrix 기반 exponent matrix(girth 8/10/12) 생성 | `ecc_top.cpp` `Load_PCM()` | H-matrix/QC 구성 알고리즘이라는 점에서 대응되나 Prime ECC는 특정 QC-LDPC(dv=17, N_b=129) 고정이라 새 base matrix 적용은 재검증 부담 큼 |
| column weight 3 저밀도 구조 | 대응 없음 | Prime ECC는 VN degree 17의 고rate 구조로 column weight 3 저rate 부호와 목표 rate·degree 대역이 전혀 다름 |
| short cycle(girth) 회피 조건식 | 대응 없음 | 부호 구성 이론 수준 기법으로 디코더/HW 모듈과 직접 연결점 없음 |

적용 가치: **낮음** — column weight 3, low-rate 대상의 순수 부호 구성 알고리즘으로 NAND ECC급 고rate(~0.9) QC-LDPC와 목표 파라미터 대역이 맞지 않고, HW/디코더 이식 요소가 없음.
