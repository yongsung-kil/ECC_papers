### B. 알고리즘 요약

1. 시스템: BSC/AWGNC, sum-product(최대 100 iter) 및 Gallager A/B, 구조화 regular column-weight-3 LDPC, 예제 rate `0.5~0.85`, 길이 `155~4381`.
2. 문제: 고SNR error-floor는 Tanner graph 내 특정 trapping set 때문; girth만 키우면 길이↑/rate↓ 발생 → rate 유지하며 harmful trapping set만 표적 제거해야.
3. 모델: all-zero codeword 전송 가정, trapping set을 Gallager A/B의 `fixed set`으로 정의(Theorem 1 조건), harmfulness는 `critical number`로 측정.
4. 핵심기법: `TSO`(Trapping Set Ontology) - trapping set들을 line-point 표현으로 열거하고 successor/predecessor 위상관계(family tree)로 조직화.
5. 의미: predecessor가 없으면 successor도 없으므로, 작은 predecessor를 금지하면 상위 trapping set 일괄 제거 → 최소 제약으로 error-floor 개선.
6. 확장(부호구조): parity-check matrix를 Latin square 유래 disjoint-support permutation matrix 배열로 기술 → RC 제약 충족 시 4-cycle-free, array/QC-LDPC 포함.
7. harmfulness 판정: SPA/BSC에서 `(4,4)` trapping set의 successor 생성 여부로 그룹 분할, 성공복호율 차이 `S_G - S_bar`로 상대 harmfulness 정량화.
8. 구성절차: PEG류 check-and-select-or-disregard로 permutation block을 추가하며 금지 subgraph 제약 `C` 위반 시 교체(Algorithm 1); QC 구조로 search 복잡도 shift로 축소.
9. 결과: 예) `(3165,2554)` girth-8 코드가 array-masking QC-LDPC(3150,2520) 대비 낮은 error floor; `(530,373)` 코드가 PEG/integer-lattice 대비 우수; BSC 최적화 부호가 AWGNC에서도 우수.
10. 한계: column-weight-3 정규부호 한정, HW 미설계, 시뮬만; 고열/irregular 확장은 trapping set 폭증으로 복잡.
