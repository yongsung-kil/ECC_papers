### B. 알고리즘 요약

1. 시스템: AWGN/BPSK 채널, protograph 기반 P-LDPC(protomatrix `B`, copy-permute 후 circulant로 lift), 예제 rate-1/2, 정보블록 길이 4096, BP 복호.
2. 풀려는 문제: 최신 LDPC는 PCM 비영원소수(→복호복잡도)가 커지고 수렴에 많은 iteration이 필요 → 복잡도·latency·전력 증가.
3. 핵심 아이디어: PEXIT 분석의 최대 iteration `tPEXIT,max`를 실제 복호 iteration과 같은 작은 값(20/100)으로 설정해 "빠른수렴" 부호를 직접 최적화.
4. 설계 문제 정식화: protomatrix `B`에 대해 threshold `η(B, tPEXIT,max)`를 최소화, 제약 `f_m(B)≥0` (원소 min/max, 열/행 가중치, 선형 최소거리 성장, 비영원소 합 상한) (식1,2).
5. 최소거리 선형성장 제약(식7a)으로 error-floor 회피, 원소합 제약(식7b)으로 비영원소수=복호비용 상한.
6. 최적화: 복잡한 목적함수라 미분진화(DE) 사용 — DE/rand/1/bin + per-vector dither, 직접 제약처리로 수렴 가속.
7. DE 단계: 초기화→변이(식3, 정수 clamp `⟨·⟩`)→교차(식4, 확률 Pc)→선택(식5, 제약충족·threshold 비교)→G세대 종료.
8. 예제: 4×7 protomatrix, precoding+puncturing, 최대원소 3, AR4A와 동일 비영원소수 제약 하에서 `N1/2` 부호 도출(식8).
9. 결과: `tPEXIT,max=20`일 때 N1/2가 AR4A/ARJA/C150/regular(3,6) 대비 threshold 0.15dB↑; 시뮬 BER 10^-5에서 0.20dB↑; AR4A(26 iter) 대비 20 iter로 동등성능→throughput 30%↑.
10. 한계: rate-1/2 AWGN만, HW 미설계, 이득은 protomatrix 크기 의존(4×7→0.2dB, 13×25→0.26dB), 고rate NAND 대역 미검증.
