### B. 알고리즘 요약

1. 목표: column-weight 2 정규 LDPC(구현 단순)를 확장해 성능 좋은 비정규 LDPC로 변환하면서 code rate를 개선.
2. 기존 한계: column-weight 2 코드는 girth는 높으나 고정 column-weight 2 제약, 정규 코드는 비정규보다 threshold SNR이 약 0.6dB 열세.
3. 기반 구성(section II): row-weight `ρ`에 대해 decimal point set `X={0,1,3,7,...}`를 `Xi+1=2Xi+1` 또는 `Xi+1=2^i+Xi`로 생성, 순환 시프트로 H1 구성, 역순+시프트로 H2 구성.
4. sparse화: decimal 값을 Gray code로 표현(0이 많도록 선택)해 sparse H 생성, 사용 bit 수로 code length 유연화.
5. code 확장(길이): 각 정수 뒤에 `p`개의 0 삽입 -> 길이 `(p+1)`배, 그래프 정규성 유지.
6. 비정규화(section III): 기저 H0에 대해 `[H0 O0 O0; O0 H0 O0; ...; I0 I0 I0]` 형태로 쌓아 column-weight를 +1(Level-1), 반복 시 레벨만큼 증가.
7. 인코딩 개선: 종속 행 제거 부담을 피하려 `H=[Hs+1 | It]`로 항등행렬 연접 -> systematic form, generator 직접 획득 -> 인코딩 시간·메모리 절감(전체 G 대신 H0만 저장).
8. code rate: `rs+1 = nρ / ((m+n)ρ + (s+1)n)`.
9. 결과(Table 1): PEG/Gallager/Hoffman-Singleton/High-rate Gallager 대비, 예로 High-rate Gallager Level-1에서 제안법 0.8461 vs 표준 0.76 등 rate 개선. rate 범위 대략 0.54~0.85.
10. 한계: BER은 column-weight 3 초과 확장에서 개선 없다고 명시, 복호 알고리즘/HW 미다룸, 시뮬 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix / 부호 구조 구성(Gray code 기반 확장) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 고rate QC-LDPC 고정 - 비QC 확장 구성법이라 로드 대상 부호 구조와 불일치, 재검증 부담 큼 |
| 인코딩(항등행렬 연접 systematic form) | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코딩 고정, 본 논문의 G 기반 systematic 인코딩과 구조 상이 |
| 확장 기반 column-weight 증가 | 대응 없음 | QC circulant/lifting 구조가 아니라 대응 모듈 없음 |

적용 가치: 낮음 - QC-LDPC가 아닌 Gray-code 그래프 확장 기반 순수 구성법으로, Prime ECC의 고정 binary QC-LDPC 부호설계·복호·HW에 이식할 요소가 없다.
