### B. 알고리즘 요약

1. 대상: 임의의 (n,k) LDPC 부호 H에서 error floor를 지배하는 small `(a,b) trapping set` 열거(a=variable node 수, b=이웃 중 odd-degree check 수).
2. 문제: trapping set 최소크기 결정은 minimum distance/stopping set처럼 NP-hard 추정, 완전탐색은 `Σ C(n,a)` (n=1008,α=10일 때 ~2^78)로 불가능.
3. 핵심 관찰: T의 support vector `xT`는 짝수 weight row만 남긴 submatrix `X`에 대해 codeword(`X·xT^T=0`)가 됨 - 즉 trapping set을 저중량 codeword 탐색으로 환원.
4. 핵심 기법: H의 row를 랜덤하게 β개(`I`) 골라 `X`(제외 row), `Y`(선택 row)로 분리하고, `X`에 Stern의 확률적 저중량 codeword 알고리즘(파라미터 p,l)을 적용해 후보 생성.
5. 판정: 생성된 저중량 codeword c의 support T에 대해 `Y_T`가 b개 odd-weight row를 가지면 (a,b) trapping set으로 확정(Step6).
6. 근거: 희소 H에서 랜덤 submatrix의 row weight w 분포 `Pw = C(a,w)(wc/m)^w(1-wc/m)^{a-w}`가 작은 w에 편중 → small b trapping set이 codeword와 유사구조라 잘 잡힘.
7. 탐색 확률: (a,b) trapping set 발견확률 `τ = θ_{b,β}·π_{a,β,p,l}` (θ=Iodd⊆I 확률, π=Stern이 weight a codeword 찾을 확률)로 정량화.
8. 최적화 절차: rmax회 column permute 반복(Step2~7)으로 확률적 커버리지 확보.
9. 결과: MacKay (504,252)/(1008,504) 부호에 적용, 기존 [8] 미보고 trapping set 다수 발견((9,1),(11,1),(4,2) 등; C1 2.5h, C2 40h @Core2Duo).
10. 한계: 부호분석/평가 도구일 뿐 디코더·부호 개선 없음, HW 무관, 확률적이라 전수 보장 없음.
