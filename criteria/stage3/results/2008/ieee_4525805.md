> 총평: 차분족(difference family) 기반 대수구성으로 girth≥10을 보장하는 LDPC convolutional 부호 설계 — 개념은 우아하나 convolutional 부호·거대 memory order라 NAND 블록 QC-LDPC엔 이식가치 낮음.

### B. 알고리즘 요약

1. 대상은 (N,K,m) binary recursive systematic convolutional(RSC) 부호로, 예시는 rate 1/2 (10,5,1135) 부호(memory order m=1135), AWGN 채널.
2. 문제: 랜덤/기존 LDPC convolutional 부호는 Tanner graph의 짧은 cycle 때문에 iterative BP가 MAP 최적성을 잃어 성능이 제한된다.
3. 핵심 가정: girth가 크면 노드 간 메시지가 더 오래 독립 유지되어 BP 성능이 개선된다.
4. 핵심 기법: 생성행렬 `G=[A|B]`의 order-1 차분집합 `Dil`과 order-2 차분집합 `Eil`을 정의하고 RCDO(recursive convolutional doubly-orthogonal) 3조건을 부과.
5. 의미: 조건1이 4-cycle, 조건2가 6-cycle, 조건3이 8-cycle을 제거해 girth≥10을 보장(이는 N,K,G 선택과 무관).
6. 부호 탐색: 각 (N,K)에 대해 RCDO 조건을 만족하면서 최대 엔트리(=memory order m)가 최소인 `G`를 heuristic search로 선택.
7. 복호: parity-check 식 (6)-(8) 기반 BP 메시지 갱신 (19)-(23), `ψ(·)`는 sum-product 또는 min-sum 규칙, pipeline schedule로 초기지연 Mm 후 연속 출력.
8. 인코딩: V개 shift register + modulo-2 adder로 저복잡도 선형 인코딩.
9. 결과: (10,5,1135) rate 1/2 부호가 min-sum BP로 Eb/N0=1.25dB, 40 iter에서 BER 10^-6 도달, 기존 LDPC convolutional 부호 대비 저SNR서 우수.
10. 한계: HW 미설계·AWGN 시뮬만, memory order m=1135로 초기지연 Mm 방대, 블록 부호 아닌 convolutional 부호.
