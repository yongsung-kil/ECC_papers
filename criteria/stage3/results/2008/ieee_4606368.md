### B. 알고리즘 요약

1. 시스템: 무선망(802.16e/802.11n) adaptive modulation·coding용 multi-rate LDPC. AWGN·BPSK, SPA 복호(최대 50 iter).
2. 문제: rate 적응 시 puncturing은 code length 감소로 BER 열화, row-removing은 성능 보존하나 인코딩 복잡도가 관건.
3. 핵심 관점: mother PCM `H`를 J×L의 q×q shifted-identity/null 서브행렬 배열로 구성, `n=q×L`, `R=1−J/L`. 예: 12×24 배열, q=96 → n=2304, mother rate 1/2.
4. 핵심 기법: mother 행렬을 **triangular + dual-diagonal** 형태로 설계 → 상위 block-row를 1~8개 제거해 rate 13/24~20/24 daughter code 생성, **code length 일정**.
5. 핵심 식: `R=(1−J/L)` — block-row 수 J만 줄여 rate 상승, 길이 불변이라 waterfall slope 유지.
6. universal encoder: shifted-identity 행렬-벡터 곱을 XOR tree processor로, dual-diagonal 부분은 `x23(j)=x24(j-1)+v(j)`, `x24(j)=x23(j)+w(j)` back-substitution(DIDO 2-XOR)으로 선형시간 인코딩.
7. 구현 디테일: TYPE_RATE 파라미터(0~8)로 시작 block-row 선택 → 단일 encoder가 9개 rate 처리, 중간벡터 v/w는 dual-port RAM 저장, 6-parallel/12-parallel 구성.
8. 부호 최적화: shifted-identity shift 계수는 [8] 방식으로 설계해 mother/daughter 성능 보장.
9. 결과: XC4VLX160/XC2V6000 FPGA, XOR processor 34slice/18FF/61LUT(메모리가 자원 대부분), 전체 ~1% 자원, 300MHz에서 3.6~21.6Gbps, error floor 1e-7까지 없음.
10. 한계: 인코더 중심(디코더는 joint row-column SPA 개념만 언급), 무선 rate 1/2~5/6 범위, ASIC gatecount 아닌 FPGA 자원만, 고rate(>5/6) 미검증.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| universal 병렬 encoder(shifted-identity XOR tree) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | QC 행렬-벡터 곱 인코딩, 구조 유사하나 Prime ECC 기보유 |
| dual-diagonal back-substitution(DIDO 2-XOR) | encoder.cpp LDPC_encoder_4KB(), Column_Reordering_Dual_Update() | dual-diagonal 인코딩 이미 보유, 중복 |
| mother PCM row-removing rate 적응 | ecc_top.cpp Load_PCM() + H-matrix | NAND 고정 고rate엔 불요, 이식 가치 낮음 |
| joint row-column SPA 디코더 | decoder.cpp LDPC_Decoder() | 본문에 개념 언급만, 별도 논문 참조 |
```

적용 가치: 낮음 — dual-diagonal QC-LDPC 병렬 인코더 자체는 Prime ECC가 이미 보유하고, 신규 가치인 multi-rate row-removing은 NAND 고정 rate ECC에 불필요하다.
