### B. 알고리즘 요약

1. 시스템: 자성기록 turbo equalization — 내부 PR 채널(BCJR 검출) + 외부 TPCSPC. rate-0.89 `(17,16)^2` TPCSPC, 16개 묶어 4K 데이터블록.
2. 문제: 자성기록에서 turbo code는 복잡도 과다, 일반 LDPC는 조밀 생성행렬·대용량 LLR 메모리 부담. TPCSPC는 고rate·선형 복잡도·병렬 구조로 대안.
3. 핵심 관점: TPCSPC를 **column weight 2인 regular LDPC**로 보고 SPA 복호 적용(각 행/열이 하나의 check), `H`는 (2t+2)×(t+1)^2.
4. 스케줄 1: fully parallel — 표준 flooding SPA(단, 조기수렴 방지 위해 syndrome check 비활성).
5. 스케줄 2: serial check-updating — check 단위 순차 갱신, 문헌의 row-column 복호와 동치임을 규명.
6. 스케줄 3(제안): serial bit-updating — bit node의 모든 check-to-bit 메시지를 동시 생성 후 `z_n` 갱신, 다음 bit에 즉시 반영. 병렬 대비 빠른 수렴(특히 I=2).
7. check 근사(제안): tanh-rule을 `max*(0,a+b) − max*(a,b)`로 쓰고 보정항 `log(1+exp(-|a-b|))`을 MacLaurin 급수 `max{0, log2 − 0.5|a-b|}`로 근사 → LUT 불필요, comparator/adder/shift로 구현.
8. 대비: 종래 min-sum은 보정항 무시로 0.2~1dB 손실, MacLaurin은 정확대비 0.1dB 손실로 min-sum보다 0.2dB 우수.
9. 결과: serial이 parallel 대비 AWGN 1dB(@1e-4)·PR 0.5dB(@1e-5) 이득, precoder로 4.2dB 개선, ME2PR4에서 TPCSPC+MacLaurin이 length-4096 rate-8/9 random LDPC보다 0.2dB 우수.
10. 한계: HW 미합성(정성적 구현 주장만), 자성기록 PR채널·col-weight-2 SPC 특화, gate count·throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| serial bit-updating(layered) 스케줄 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 스케줄만 교체로 수렴 가속 이식 가능 |
| MacLaurin 급수 check 근사(max*) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum CN을 LUT-free max* 근사로 교체 후보(0.2dB 개선 주장) |
| 정확 tanh-rule / Sum-Product | 대응 없음 | Prime ECC는 min-sum, full-tanh SPA 미보유 |
| TPCSPC(col-weight-2) 부호구조 | 대응 없음 | 고rate QC-LDPC와 부호구조 상이 |
```

적용 가치: 중간 — layered 스케줄과 LUT-free MacLaurin check 근사는 min-sum 성능 개선 후보이나, 부호구조(col-weight-2 SPC)와 채널(자성 PR)이 NAND와 달라 근사식만 발췌 이식 필요.
