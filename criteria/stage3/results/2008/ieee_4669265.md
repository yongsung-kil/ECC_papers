### B. 알고리즘 요약

1. 대상은 802.16e WiMax LDPC 디코더로 BPSK/AWGN 채널, code length `n=576/2384`, rate `1/2·2/3·3/4·5/6`, 6-bit LLR 양자화, 20 iteration 조건이다.
2. 문제: fully-parallel은 라우팅 혼잡이 심해 짧은 블록에도 불가 → 전 code class를 지원하면서 목표 throughput을 맞추는 유연한 partly-parallel 구조가 필요하다.
3. 채널 모델 `rn = sn + vn`, LLR `un = (2/σ²)·rn`.
4. 복호는 BP/SP 기반 min-sum. CN update는 `g(x,y)=sign(x)sign(y)·Min(|x|,|y|)/α` (normalized min-sum, `α>1`).
5. 정규화 `/α`는 min 근사의 과대추정을 보정해 SP 대비 성능 손실을 줄이는 목적이다.
6. BN update `vn→m = un + Σ_{mj∈M(n)} wmj→n − wmi→n`; hard decision은 `vn→m + wmi→n` 부호로 결정.
7. 노드 클러스터링: CN/BN을 각 24개 단위 집합 `CSj/BSj`로 묶어 한 모듈 내에서 순차 처리, 서로 다른 집합은 모듈간 병렬 처리.
8. 아키텍처: 각 모듈은 BN유닛+CN유닛+메모리뱅크로 구성, 비지역 메시지는 인터커넥트 버스 `Bi`로 전송; 모듈수/버스수/뱅크/시나리오 수를 제약프로그래밍으로 최적화(본문 미상세).
9. 결과: 4-module 20-bus 구성이 전 WiMax code를 지원, Xilinx Virtex5 LX110T에서 192MHz 합성(10K FF/14%, 19K LUT/27%), throughput 10~30Mbit/s(@160MHz·20iter).
10. 한계: WiMax 특화·저throughput, ASIC gatecount 없음, 최적화 시스템 미공개, 정정능력/iteration 개선은 없음(구현 논문).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized min-sum CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 이미 보유(§3), normalization factor는 코드상 미확인 — 중복 기여 |
| BN(VN) 처리·hard decision | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | 표준 VN update, 중복 |
| partly-parallel 모듈/메모리뱅크/인터커넥트 | 디코더 HW 모델 (z=32, HCU, GT) | 부분 대응, 구조 상이(24-node 클러스터 vs z=32 row 병렬) |
| 6-bit LLR 양자화 | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | Prime ECC는 2~3bit soft; 6bit는 NAND read 비용상 부적합 |
```

적용 가치: **낮음** — WiMax 전용 저throughput 디코더 아키텍처로 핵심 알고리즘(min-sum)은 이미 보유하고 HW 구조도 z=32 기반 Prime ECC와 정합되지 않는다.
