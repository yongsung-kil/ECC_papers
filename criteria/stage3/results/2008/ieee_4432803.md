### B. 알고리즘 요약

1. 대상: 단일 인코더/디코더 쌍이 다양한 code length·rate를 지원하는 QC LDPC 시스템, mobile/DVB급 다중서비스 대상.
2. 부호 구조: `layer-block-cell` 계층 구조 QC 행렬 — 각 cell은 zero 또는 순환천이 항등행렬, 열/행 가중이 두 종류뿐(`approximately regular`).
3. 구성법(DEx): all-1 base matrix → PEx(permutation expansion)+full-ranking으로 mother matrix(LAR-I) → CSEx(cyclic-shift expansion)+short-cycle 제거로 PCM(LAR-II).
4. 부호 성능: 동일 degree분포·girth의 PEG 부호 성능에 근접(초과 아님), rate 1/2·7/8 예제에서 float/fixed 모두 확인.
5. 인코더: RU(Richardson-Urbanke) ALT 방식을 개선 — forward substitution 완전 제거, dense-matrix `E·T⁻¹·B` 곱을 QC성 이용해 feedback shift-register로 처리(면적 선형).
6. 인코더 유연성: rate 고정 시 CSEx factor만 바꿔 길이 변경, 길이 고정 시 파라미터 동시조정으로 rate 변경, 온라인 재구성.
7. 디코더 알고리즘 LMMS: modified min-sum(스케일링)과 layered decoding 결합, MS라 수신심볼 직접 사용, layer별 CN/BN 갱신.
8. 디코더 HW: 부분병렬(BNU/CNU + interconnection network), interconnect는 PEx 덕에 복잡도가 일반 LDPC 대비 낮고 작은 crossbar 배열로 구현, scaling factor를 compact binary로 골라 곱셈을 shift+add로 축소.
9. 결과: LMMS 10 iter ≈ BP 20 iter BER, fixed-point는 float 대비 ~0.1 dB 열화. Xilinx XC2V6000 FPGA 구현 시 인코더 최대 1200 Mbps, 디코더 60(단일)~180(3개) Mbps @100 MHz.
10. 한계: 부호는 PEG 성능 동등(이득 없음), 통신 채널 타깃, 고정소수점 데이터폭 등 세부 수치가 txt 추출에서 손상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LMMS layered modified min-sum 복호 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN + layered 스케줄, Prime ECC의 min-sum·Dual-Update와 개념 유사(중복 가능) |
| min-sum 스케일링(0.8/compact binary) | `decoder.cpp` `VN_Cal_HD()` 등, magnitude 양자화 | Prime ECC scaling factor 미확인이라 최적값 참고 가능 |
| 고정소수점 LLR 데이터폭 설계 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 양자화 폭 설계 참고(단, 실제 폭은 txt 손상) |
| layer-block-cell QC PCM 로드 | `ecc_top.cpp` `Load_PCM()` | QC 구조 로드 대상이나 Prime ECC는 고정 PCM |
| 개선 RU 인코더(FS 제거, FSR) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal, RU 방식과 상이 |
| BNU-CNU interconnection network(crossbar) | `GT.cpp` `Make_GT_HW()` | QC HW 라우팅, Prime ECC는 GT MUX(방식 상이) |

적용 가치: **중간** — layered min-sum과 QC crossbar interconnect는 NAND용 min-sum 디코더 설계에 참고할 만하나, Prime ECC가 이미 min-sum+Dual-Update+적응형 LLR을 보유해 상당 부분 중복이고 부호 자체는 성능 이득이 없다.
