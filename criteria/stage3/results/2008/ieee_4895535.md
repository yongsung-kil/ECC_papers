> 총평: RBP의 greediness를 V2C residual로 완화해 8 iteration에 0.3dB·빠른 수렴을 얻는 informed dynamic scheduling - 개념은 매력적이나 residual 정렬 기반 직렬성이 병렬 min-sum HW엔 부담.

### B. 알고리즘 요약

1. 시스템: AWGN 채널, IEEE 802.16e QC-LDPC mother matrix (length 576/1152/2304, rate 1/2·3/4), BP 계열 iterative decoding.
2. 문제: 기존 Residual BP(RBP)는 largest-residual edge를 먼저 갱신해 수렴은 빠르나 greediness로 새 오류를 만들고(정정성능↓) message 재계산·residual 재정렬 비용이 큼.
3. 핵심 아이디어: residual을 RBP의 C2V message(`m_{c→v}`) 차이 대신 **V2C message(`m_{v→c}`) 차이**로 계산 - `r(m_{v→c})`는 선택 edge의 check 하나를 뺀 모든 check equation 기반이라 덜 greedy·더 신뢰.
4. 핵심 기법(Algorithm A, VCRBP): 최대 `r(m_{v→c})`인 edge `m_{vi→cj}` 선택 → `r=0` 설정 후 `N(cj)\vi`의 모든 `m_{cj→va}` 갱신 → `N(va)\cj`의 `m_{va→cb}` 갱신하며 residual 재계산.
5. 복잡도 절감 근거: RBP Line5(residual 결정 때 이미 계산된 `m_{c→v}`를 갱신 시 불필요 재계산)와 Line10(edge마다 Q 재정렬)을 제거 - VCRBP는 한 스텝 단축되고 residual ordering 횟수도 감소.
6. 확장(Algorithm B, N-VCRBP): SBP처럼 node 단위로 갱신하되 다음 variable node를 스케줄이 아닌 largest-residual로 선택 - VCRBP 성능 유지하며 복잡도 추가 절감.
7. 구현 디테일: `r(m_{v→c})`가 모든 `c∈N(v)`에 동일하고 `r(m_{v→c})=r(m_{c*→v})` 성질을 이용해 residual 계산을 `r(m_{cj→va})`로 대체 가능.
8. 학습/최적화: 없음(휴리스틱 스케줄링).
9. 결과: length 576 rate 1/2에서 max 8 iteration 시 FER 10^-4에서 RBP 대비 약 0.3dB 이득, 초기 iteration 수렴 훨씬 빠르고 50 iteration까지 gap 유지.
10. 한계: HW 미설계·시뮬만, quantization/bit-width 미명시, residual 정렬 기반이라 병렬화 불리.
