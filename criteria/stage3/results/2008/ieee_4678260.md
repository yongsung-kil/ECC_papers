### B. 알고리즘 요약

1. 시스템: 전력선통신(PLC) 홈네트워킹, regular LDPC(column weight 3, Gallager 구성), 길이 `1024`, rate `1/2`.
2. 문제: 전력선 채널의 임펄스성 잡음을 AWGN용 BP 디코딩으로 처리하면 큰 성능열화(약 13dB) 발생.
3. 모델: Middleton Class A 잡음(AWAN) — 배경 Gaussian + Poisson 분포 임펄스, 파라미터 `A`(임펄스 지수), `Γ`(GIR).
4. 핵심기법: Log-BP(log domain BP)의 초기 LLR을 AWGN용 `λ=2y/σ²` 대신 Class A PDF `P_A(y)` 기반 `λ_A(y)=ln[P_A(y|+1)/P_A(y|-1)]`로 계산.
5. 식 의미: 수신값이 작으면 신호에 비례, 약 1 초과면 임펄스 개입 확률이 높아 LLR을 억제 → 임펄스 오류의 과신 방지.
6. 디코딩 절차: Step1 초기화 → Step2 check node 갱신(`sign`·`f(x)=ln((e^x+1)/(e^x-1))`) → Step3 message node 갱신 → Step4 hard decision + `ĉHᵀ=0` 검사, 최대 20 iteration.
7. 결과: AWAN에서 수정 Log-BP가 기존 BP 대비 BER=1e-5에서 약 `11dB` 이득.
8. 파라미터 영향: A 0.01→1 증가 시 약 10dB 열화(잡음이 Gaussian화), GIR 10에서 약 1.7dB 이득.
9. 한계: HW 미설계, MATLAB Monte-Carlo 시뮬만, 채널 특화(임펄스), full sum-product 기반.
