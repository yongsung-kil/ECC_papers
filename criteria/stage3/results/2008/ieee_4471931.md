### B. 알고리즘 요약

1. 대상: irregular LDPC 코드(예: `N=512, M=256, rate 0.5`; `rate 1/2 length 504`; `N=576 Z=24` 802.16e급) 구성, BEC/AWGN + BP(50 iter) 복호.
2. 문제: PEG/ACE 같은 기존 girth·ACE 조건화는 성능(block error)을 **간접** 최적화해 finite-length error-floor 예측·최소화가 부정확.
3. 모델: BEC에서 복호 실패는 그래프의 `stopping set`이 결정, 이를 `cycle set`(minimal cycle + intrinsic/extrinsic check)로 파라미터화.
4. 핵심기법: `(d,τ) cycle set`의 block-error 기여 상한 `PCS(d,τ)`를 union bound로 사전계산, edge마다 형성되는 e-cycle set 열거로 `PB(G,e,ε)` 추정.
5. 식 의미: `PB(G,e,ε)=Σ W^e_(d,τ) PCS(d,τ)` — edge가 유발하는 나쁜 cycle set 기여합이 최소인 edge를 선택(그리디)해 block error를 직접 최소화.
6. 확장: lifted 그래프(cyclic 순열, 즉 QC 구조)로 일반화 — 매 스텝 Z-edge 단위로 추가, `PBLB` 하한으로 정렬 후 검증.
7. 구현 디테일: 단일 tree-spanning으로 minimal path만 추적(EMD=ACE 활용), 열거 O(N), 저장 O(N·logN)로 PEG 수준 복잡도 유지.
8. 이론: 최적화 앙상블 block-error 상한 정리(Thm 4.1) 도출 — 시뮬 없이 cycle-set 스펙트럼만으로 error-floor 예측·코드 선택 가능.
9. 결과: 동일 앙상블 PEG 대비 stopping-set 스펙트럼·최소거리 개선(504코드 min dist 13→15), 802.16e 대비 AWGN 0.5dB↑ 또는 BEC ε 30%↑ 이득.
10. 한계: HW 미설계, BEC 기준 최적화(AWGN은 부수 확인), 예제가 rate 0.5·중간 길이라 고rate NAND ECC급 실증 없음.
