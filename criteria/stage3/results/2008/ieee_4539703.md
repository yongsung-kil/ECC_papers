### B. 알고리즘 요약

1. 대상: 2048-bit `(6,32)` regular structured LDPC(10 GBase-T, rate ~0.84, 행렬 384x2048), LLR-MP(sum-product) 복호, 최대 16 iteration.
2. 문제: 고rate 부호는 CN 입력수(row weight)가 크고(예 32) — fully parallel은 면적·라우팅 폭증(길이 제곱 이상), memory-sharing 부분병렬은 대역폭 한계로 수백 Mbps에 그침.
3. 핵심 관찰: CN 갱신 `L(r)`의 합산을 재귀(recursive)로 분해 가능 — 입력을 `S`개 청크로 나눠 `S` 클록에 걸쳐 부분합 누산(식 10~16), 재구성이라 성능(BER) 손실 없음.
4. SMP 핵심: CNU가 전체 입력 대기 없이 청크 부분합을 누산하고, VNU 갱신 1클록 뒤 CNU가 곧바로 시작 → V/C 단계 overlap, 단계간 latency 1클록.
5. CNU 입력이 row weight 전체(32)가 아니라 `S`(예 8)만 필요 → PLUT·adder tree 대폭 축소(23 adder·24 PLUT 절감).
6. HW: register-based 부분병렬 아키텍처(fragmentized memory 회피), 병렬도 `M=512`, MUX로 top-level 라우팅 제어.
7. 최적화: VNU/CNU 임계경로 rebalancing — RLUT·SM-2s·MUX 블록을 CNU→VNU로 이동해 전체 면적·HUE 개선, adder tree 단수 감소로 클록 주파수↑.
8. 양자화: `(5:3)`/`(4:2)` uniform quantizer, floating point 대비 각 0.2/0.5 dB 손실(Fig.17), 메시지 버스 4~6 bit.
9. 결과: IBM 90nm CMOS, core 14.5 mm², postlayout 임계경로 4.83 ns, 16 iter에서 optimized SMP 5.30 Gbit/s — 동일 면적에서 기존 대비 throughput 1.75배.
10. 한계: BER 불변(throughput/area 기법), min-sum엔 적용 가능하다 언급만(본 구현은 LLR-MP), SMP-adaptive 부호설계는 범위 밖, comms(10GBase-T) 타깃.
