### B. 알고리즘 요약

1. 시스템: 모바일 무선망용 regular LDPC 복호기, AWGN+BPSK, 데이터 폭 6-bit.
2. 문제: 기존 rate-fixed 복호기는 복호 시작 후 rate 고정, 채널 SNR 변동에 비효율(저SNR은 저rate, 고SNR은 고rate가 유리).
3. 코드 스펙: `768×6144` 대형 matrix를 `3×24` sub-block(각 `256×256` 순환이동 identity)으로 분할, R=0.5/0.625/0.75에 각각 code length 1536/2048/3072.
4. 핵심 기법: block-partitioned parity check matrix - 대형 matrix를 2/3/4 부분으로 나눠 rate별로 동일 구조(같은 `wr`,`wc`)의 부분행렬을 얻어, 하나의 HW로 여러 codeword 동시 복호.
5. 복호 알고리즘: Min-Sum, row op `αmn = (∏sign βmn)·min|βmn|`, col op `βmn = Σαm'n + λn`, max iteration 8.
6. 핵심 기법 2단: multi-rate 호환 1st-2nd 최소값 탐색 유닛 - tournament 방식으로 R에 따라 6입력 2출력(R=0.5) 또는 12입력 1출력(R=0.75)을 신호 `R`로 전환.
7. 구현 디테일: partially parallel(parallel factor 4), CFU 12개·BFU 96개, 120 memory bank.
8. 학습/최적화: C 시뮬레이션으로 rate-switching point 결정(R=0.5는 2.75dB까지, 0.625는 3.65dB까지, 이후 0.75).
9. 결과: CMOS 90nm 합성, critical path 3.07ns→326MHz, simple throughput 539Mbps; 다중rate row unit 면적 0.23mm²로 결합형 0.37mm² 대비 37% 절감.
10. 한계: 실측 없이 논리합성뿐, 무선 다중rate 시나리오 특화, 정정능력 자체 개선은 아님(rate 선택으로 throughput만 개선).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum row op / 1st-2nd min 탐색 | [decoder.cpp](../../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min1/min2 CN 연산은 이미 보유, tournament 구조만 상이 |
| 전체 복호 이터레이션 루프 | [decoder.cpp](../../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | Min-Sum 반복 구조 동일 |
| 다중 rate 재구성 minimum unit | 대응 없음 | Prime ECC는 단일 고rate QC-LDPC 고정, 다중rate 불필요 |
| block-partitioned PCM (rate 가변) | 대응 없음 | H-matrix 고정(`Load_PCM()`)이라 rate 분할 개념 없음 |

적용 가치: **낮음** - min-sum·min1/min2 CN은 이미 보유하고, 다중rate 재구성은 NAND의 고정 고rate ECC 시나리오에 불필요함.
