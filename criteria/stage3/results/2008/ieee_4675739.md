### B. 알고리즘 요약

1. 대상은 binary linear/LDPC code의 LP 복호(ML 근사); AWGN에서 random `(3,6)`/`(4,8)`-regular LDPC, 길이 `n=32~360`, rate `~0.5`.
2. 문제: Feldman et al. LP 복호는 check node degree `d_c`에 대해 제약 수가 지수적(고밀도 부호는 `n`에 지수적)이라 실용성이 낮음.
3. 핵심관찰: 완화된 parity inequality 중 각 check node가 만드는 cut은 최대 1개(`Theorem 1`)라 필요한 제약만 골라 추가 가능.
4. 핵심기법1 (Adaptive LP): box 제약 `0<=f<=1` 최소집합으로 시작(`Algorithm 2`), LP를 풀고 위반된 parity inequality(cut)만 순차 추가, 더 이상 cut이 없을 때 종료.
5. cut 탐색은 이웃 변수노드 값을 정렬 후 `Algorithm 1`로 `O(d_c log d_c)`에 처리; 수렴은 최대 `m`회 iteration 보장(`Theorem 2`), 최종 제약 수 `<= n + m`.
6. 결과관찰: 사용된 parity inequality 평균/최대 개수가 check degree `d_c`·길이 `n`과 거의 무관하게 상수에 근접 → 임의 밀도 부호에 LP 복호 적용 가능.
7. 핵심기법2 (RPC cut): LP가 nonintegral pseudocodeword를 낼 때, parity-check 행들의 modulo-2 합(RPC)으로 새 cut 생성해 해를 잘라내며 codeword feasibility 유지.
8. RPC cut 탐색은 fractional subgraph의 cycle과 연관(`Theorem 3/4`: noninteger pseudocodeword엔 항상 fractional cycle 존재); `Algorithm 3`은 pruned graph에서 random walk로 cycle 발견 후 RPC가 cut인지 검사.
9. 결과: warm-start로 LP 재풀이 가속(길이 `n`에서 약 2배 개선); RPC cut LP는 SNR 증가 시 LP·SPA를 능가하고 ML 하한에 근접(길이 32/100/240, AWGN, 약 200 failure 집계).
10. 한계: 범용 LP solver 사용으로 blocklength에 super-linear, message-passing의 선형 복잡도에 못 미침; HW 미설계, 순수 시뮬/이론.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP 복호 알고리즘 전체 | 대응 없음 (LP-solver 기반, min-sum과 다른 패러다임) | decoder.cpp LDPC_Decoder()는 min-sum, LP relaxation 대응 없음 |
| RPC cut / 수렴실패 후 post-processing | corner.cpp (수렴실패 추적) | 개념적 유사(복호 실패 케이스 처리)뿐, RPC cut의 동적 LP 재풀이 HW 대응 없음 |
| RPC = parity-check 행 조합 | ecc_top.cpp Load_PCM() | PCM은 고정 로드, 동적 redundant check 추가 기능 없음 |
| pseudocodeword/fractional cycle 분석 | 대응 없음 | error-floor 원인 분석 통찰 수준 |
```

적용 가치: **낮음** — LP 복호는 Prime ECC의 min-sum HW와 복호 원리가 근본적으로 달라 직접 이식 불가; RPC-cut·pseudocodeword 제거 아이디어의 개념적 통찰만 error-floor 대책 참고용.
