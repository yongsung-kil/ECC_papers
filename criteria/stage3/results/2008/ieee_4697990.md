### B. 알고리즘 요약

1. 시스템: BPSK/AWGN, `Imax=50` 반복. 예제 부호는 (1024,512) rate-0.5 irregular PEG 부호와 (3150,2520) rate-0.8 regular QC-LDPC(63x63 circulant, col weight 3, row weight 15).
2. 문제: iterative decoding의 error floor를 지배하는 trapping set(near-codeword)을 깨야 하나, 기존 decoder 기반 대책은 지배적 trapping set의 사전지식이 필요해 실용성 낮음.
3. 모델: `(a,b)` trapping set = 변수노드 a개 유도 subgraph에 odd-degree check가 b개뿐인 구조. 만족 check는 현재 결정을 강화, 불만족 check는 소수라 오정정을 못 바꿔 decoder가 갇힘.
4. 1단계: 수정된 iterative decoding. 반복 내내 불만족 check 최소개수 `nc`와 그 집합 `C`, 대응 hard decision `xˆ`를 추적. 실패 && `nc < t`이면 trapping set으로 판정.
5. 핵심 아이디어: `C`의 각 check는 오류 변수노드 집합 `V`에 인접. `C`의 matching set(각 check에 정확히 하나씩 인접하는 변수노드 집합) 중 하나가 `V`의 부분집합.
6. 2단계(SBF): `C`(또는 2-node subset)의 모든 matching set `L`을 생성. 각 `M`에 대해 해당 비트의 초기 LLR을 `-xˆ·γ`(최대 반대부호)로 뒤집고 재복호, 성공하면 종료.
7. 구현 트릭: `|L|`은 `dc^|C|`로 지수 증가 → 실제로는 `C`의 임의 2-node subset만 사용(`|L|~dc^2`)해도 성능 거의 동일(trapping set은 불안정 평형이라 1~2비트 섭동으로 붕괴).
8. 파라미터: `t=15` (실패 && `|C|<=15`면 2단계 진입).
9. 결과: 예제1 error floor는 undetected error(최소거리 15) 한계까지 저감; 예제2는 BER 10^-8까지 error floor 없음, 3.8dB에서 평균반복 4.4→4.5로 throughput 거의 불변.
10. 한계: HW 미설계·시뮬만. 2단계는 직렬 재복호라 실패블록 지연 큼. undetected error(최소거리)로 인한 floor는 못 낮춤.
