### B. 알고리즘 요약

1. 대상은 Block-LDPC(Architecture-Aware QC-LDPC): base matrix `Hb`(`Mb×Nb`), sub-matrix size `z`, cyclic-shift 순열 블록 — 802.11n/16e/15.3c 표준 부호.
2. 문제: 기존 flexible LDPC 디코더는 block-serial 스케줄로 throughput이 <640Mbps로 제한됨.
3. 핵심 가정/모델: AWGN LLR `yv=2rv/σ²`, 반복 message-passing (row op `Rcv` 식(1), col op `Lvc` 식(2)), Ψ(x)=log tanh(|x/2|).
4. 핵심 기법: horizontal layered 스케줄을 check node-based processor(CNBP)로 재정식화 — 한 block-row의 `dc`개 edge(m1,m4,m7)를 단일 클록에 동시 처리(block-serial은 `dc` 클록 필요).
5. 핵심 식: 식(6)(7)로 layered 업데이트 `R'cv`, `Q'v`를 직전 layer 값으로 on-the-fly 갱신 → 별도 `Lvc` 메모리 불필요.
6. 확장: `z`개 CNBP를 fully-parallel로 배치, VN_MEM/C2V_MEM를 `Nb`/`dc` 뱅크로 분할해 병렬 메모리 접근.
7. 구현 디테일: CNBP는 normalized min-sum(1st/2nd min 선택 + scaling)으로 근사, signed-magnitude↔2's complement 변환, dual-port 메모리 동시 R/W, Benes 스위칭망 `2×dc`개로 라우팅.
8. 유연성: control unit만으로 rate/block-size 재구성(CNBP/메모리 미변경), 미사용 입력은 MUX로 zero-feed.
9. 결과: throughput ≈ `R·N·fclk /(iter·Mb)`로 message edge수 `L`에 무관(block-serial은 `L` 종속) — 정성적 throughput 향상 주장(구체 수치 없음).
10. 한계: 합성/BER 시뮬/실측 결과 없음, gatecount·clock·throughput 수치 미제시; layered의 iteration ~50% 감소는 [4][5] 기법 차용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized min-sum CNBP(1st/2nd min+scaling) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime ECC min-sum과 동일 계열 — 이미 보유(중복) |
| horizontal layered 스케줄(on-the-fly R'cv/Q'v) | decoder.cpp LDPC_Decoder() 이터레이션 루프 (Dual-Update 교대 스케줄) | Prime ECC의 Dual-Update와 유사 개념 — 중복 |
| block-parallel edge 처리 / Benes SN 라우팅 | GT.cpp Make_GT_HW() (MUX 기반 edge 선택) | HW edge/라우팅 계층에 느슨 대응, 구조 정합 필요 |
| adaptive quantization(scaling) [11] | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | LLR/양자화 테이블에 대응 |
```

적용 가치: **중간** — 부호(QC-LDPC)·알고리즘(min-sum/layered)이 Prime ECC와 정합하나 대부분 기보유 기법과 중복이고 결과 미검증이라, block-parallel CNBP·메모리뱅킹의 throughput 아이디어를 참고하는 선에서 유용.
