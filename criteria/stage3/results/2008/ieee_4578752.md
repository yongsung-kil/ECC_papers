### B. 알고리즘 요약

1. 시스템: 완전병렬 bit-serial LDPC 디코더; PEG `(4,15)`-regular `(660,484)` rate 0.74 (0.13um 실측) + RS 기반 `(6,32)`-regular `(2048,1723)` 10GBase-T 코드 (90nm 합성).
2. 문제: 기존 fully-parallel 디코더는 대면적·복잡 배선으로 확장성 저하, 저전력 아키텍처 연구 부족.
3. 저전력 원리: 병렬도 `p`배↑ → 클럭 `p`배↓ 가능 → 공급전압↓ → 동적전력 `V^2` 절감 (같은 throughput 유지).
4. 배선 절감: bit-serial message passing으로 다비트 메시지를 여러 클럭에 걸쳐 1비트씩 전송, global wiring·CNU 면적(40%↓, 0.1dB penalty) 축소.
5. 아키텍처: 각 VN/CN마다 전용 VNU/CNU 배치, extrinsic 메모리를 배선으로 분산(주소생성 불필요), block-interlaced로 2프레임 동시 복호.
6. 조기종료: 이미 계산되는 V2C 메시지의 sign-bit parity를 OR-tree로 검사 → 유효 codeword 검출 시 전체 VNU/CNU 비활성화 (추가 배선·클럭 없음, gate 0.1%/0.5%만 증가).
7. min-sum: approximate min-sum(sign-magnitude MSB-first CNU, LSB-first 2's comp VNU), 4-bit 양자화 LLR.
8. 조기종료 특성: 표준 syndrome check 대비 평균 1 iteration 더 소요하나 배선 미증가; 동적전력 65% 절감.
9. 결과: `(660,484)` 10.4 pJ/bit/iter @1.2V 3.3Gb/s (0.6V시 2.7 pJ/bit/iter, 648Mb/s); `(2048,1723)` 2.23M NAND gate@90nm, 16Gb/s, Shannon 한계 3dB 이내 @BER 1e-5.
10. 한계: 완전병렬은 배선복잡도가 코드 길이 상한을 제한, 통신 표준(Ethernet/WiMAX) 타깃, 정정능력 개선 없음.
