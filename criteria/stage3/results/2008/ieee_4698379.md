### B. 알고리즘 요약

1. 시스템: inner LDPC(length 4923, rate 8/9, column weight 3, 9개 547×547 circulant QC-LDPC, girth≥6) + outer RS(10-bit symbol, T-symbol 정정, length 438 symbol), 자기기록 채널.
2. 문제: 선행연구 3편이 연접의 최적 outer RS 정정능력 `T`에 대해 상반된 결론(`6≤T≤13` vs `T=0`) - 어떤 조건에서 연접이 이득인지 불명.
3. 채널 모델: 수직자기기록 step response `A·erf(αt/PW50)`, transition jitter noise `Δtm`(i.i.d. Gaussian), AWGN 또는 jitter 단독 인가, FIR equalizer로 PR target `1221`, SOVA 검출(DDNP 미사용).
4. 평가 셋업: RS 인코더 없이 랜덤 유저비트를 LDPC 인코딩, LDPC 디코더 출력에서 10-bit symbol error counter가 RS 디코더를 대체, symbol error > T면 sector error로 집계.
5. 정규화: 유저비트밀도 `u=c·RLDPC·RRS`, `RRS=1-2T/n` 고정 u에서 T↑->channel density·noise↑ -> 공정 비교 위해 `σa(T)=σa(0)/(1-2T/n)`.
6. 핵심 관찰1: LDPC 오류 발생 시 codeword당 평균 ~50 symbol error(최소 27), symbol당 평균 1.9 bit error - RS로 잡으려면 T≥27~50 필요해 비효율.
7. 핵심 관찰2(그림4): 고정 u=1.78, Code 1, 6 global/6 local iteration에서 최적 T=0, 즉 LDPC 단독이 sector error rate 최소.
8. 파라미터 영향: global iteration(2 vs 6), local iteration(6 vs 25), transition jitter 유무는 최적 outer RS rate에 큰 영향 없음.
9. error floor 효과: error floor 있는 Code 2는 T 감소 시 symbol error rate가 어느 지점서 정체 -> 최적 T가 존재할 수 있음(binomial 근사로 [1]과 유사한 최적 T 재현), error floor 없는 Code 1은 T 감소가 항상 유리.
10. 결론/한계: 반복 충분+inner error floor 약하면 LDPC 단독이 연접보다 우수, 채널은 HDD 특화라 NAND 직접 대응 아님·새 알고리즘/HW 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| inner QC-LDPC(8/9) 복호 | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 유사 고rate binary QC-LDPC지만 논문은 복호 알고리즘 개선 없음 |
| inner LDPC H-matrix(547 circulant) | `ecc_top.cpp` `Load_PCM()` | QC 구조 로드 지점, 본 논문은 부호설계 대상 아님 |
| outer RS 연접 아키텍처 | 대응 없음 | Prime ECC는 inner LDPC 단독, 외부 RS 미포함 |
| error floor가 연접 유효성 좌우 | `corner.cpp`(수렴실패 추적) | error floor 특성이 시스템 판단에 영향(정성 참고) |

적용 가치: **낮음** - 자기기록 특화 연접 평가로 Prime ECC에 떼어 쓸 알고리즘/부호설계가 없고, "inner error floor 약하면 outer code 불필요"라는 시스템 인사이트만 정성적으로 참고 가능.
