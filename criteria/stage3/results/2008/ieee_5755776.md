### B. 알고리즘 요약

1. 유한 m차원 격자 `L = Z × Zp2 × ... × Zpm` (첫 좌표만 크기 `c`, 나머지 `pi` modulo)를 정의하고 격자 위의 "선(line)"을 검사노드, "점(point)"을 변수노드로 매핑해 `(ρ,c)`-regular Gallager LDPC를 구성한다.
2. 문제: 대수/조합 구성으로 girth·min-distance 보장 + 넓은 length/rate 선택을 동시에 얻기 어려움 → 격자로 유연성 확보.
3. slope 집합 `B={s1..sρ}`가 Condition 1(임의 η-부분집합 선형독립, η=3 or 4) + Condition 2(최소 m개 독립)을 만족하도록 설계.
4. Tanner 그래프에 proper 4-cycle·6-cycle이 없음을 Condition 1으로 증명 → girth 8 보장.
5. 핵심 결과식: proper 8-cycle 수 `N8 = nρ(ρ-1)(c-1)^2/8` (η=4일 때 정확 산출), 최소거리 `dmin ≥ 2m`.
6. 유연성: length `n = c·p2·...·pm`, rate 하한 `r ≥ (c-ρ)/c`; 대략 `n ≥ 3m/(1-r)^m` 및 `n ≥ 1/r^m` 범위에서 length/rate 자유 선택.
7. 복호: 블록행 `Hi`가 서로 겹치지 않아 partly-parallel 디코더(블록행 단위, `p2·...·pm`개 MPU)로 구현, a posteriori/extrinsic LLR 메모리 + interleaver `Πi` 사용.
8. MPU를 TDMP(turbo decoding message passing, [12]) 2-state trellis log-APP + "max-quartet"로 구현 → 저메모리·빠른 수렴(차용).
9. 결과: c=9,p2=10,p3=11(n=990,rate≈0.6), c=p2=p3=p4=8(n=4096,rate≈0.5) 두 예제 AWGN BPSK, 20 iter에서 Shannon 한계 근처의 가파른 waterfall.
10. 한계: HW 합성/gate count 없음, 기존 LDPC와의 비교 없음, error-floor 영역(10^-6 이하) 미측정, 비-QC 구조.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| FML 부호구성(격자 line = H행) | [ecc_top.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 로드부에 닿으나 QC(circulant)가 아닌 정규 격자구조라 이식난이도 높음 |
| girth-8 / dmin≥2m 이론 | 대응 없음 | 부호설계 구조특성, 특정 코드 모듈과 무관 |
| TDMP(log-APP trellis) partly-parallel 디코더 | `decoder.cpp` `LDPC_Decoder()` | 우리는 min-sum 기반 → TDMP와 알고리즘 상이, 직접 재사용 불가 |
| a posteriori/extrinsic LLR 메모리·interleaver | `ecc_data.h` `PARAM_LLR` | 메모리 개념 참고 가능하나 직접 대응 약함 |

적용 가치: 낮음 — 비-QC 정규 격자부호 + TDMP 디코더라 우리 QC-LDPC/min-sum 구조와 정합이 낮고, 정정성능 우위·고rate 검증도 없음.
