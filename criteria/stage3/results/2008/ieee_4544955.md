### B. 알고리즘 요약

1. 시스템: `(dv,dc)`-정칙 binary LDPC, AWGN/BPSK, **hard 0-1bit 경판정**, 최대 200반복.
2. 문제: Gallager 알고리즘 B(GB)는 무한길이 최적이나 유한길이·채널파라미터 과소추정 시 최적 아님 → 개선 필요.
3. 모델: 반복 `l`의 평균 traversing message 오류확률 `pl`을 EXIT 함수로 추적, 채널 crossover 오류율 `p0`.
4. 핵심기법: MB(majority-based) 알고리즘 차수 `b`를 반복마다 바꾸는 정규화 스위치기법(NSS) — `bn≤bn+1` 조건, GB·MB가 그 특수예.
5. 의미: 스위칭점 `si,j`(EXIT 곡선 `hp^i`와 `hp^j`의 교점)와 고정점 `q+/q-` 분석으로 열린 복호터널 유지 → 수렴 보장(Theorem 3).
6. 핵심기법2: exp-OSS(확장), TC-OSS(tail-cut), HA-OSS(head-appended), HATC-OSS 등 변형과 각각의 수렴정리(Theorem 4~7).
7. 트릭: 저차 MB로의 전환을 지연(exp-OSS)하거나 제거(TC-OSS)해 유한길이에서 오류메시지 증가를 안정적으로 억제.
8. 최적화: EXIT 곡선의 단봉성·Descartes 부호규칙으로 교점 유일성 증명, 임계값(Table I) 비교.
9. 결과: (5,10)len816 0.15/0.3dB, (4,9)len9000 0.65/0.9dB(BER 10^-6) GB 대비 개선, (5,7)QC len10297에서 90% 채널 과소추정 시 0.4dB↑, 추가 반복 불필요.
10. 한계: HW 미설계, 경판정 Gallager B/MB 계열 전용, 정칙부호·저~중 rate 대상, irregular 부호 스위치 설계는 향후과제.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager B/MB 경판정 반복복호 | decoder.cpp LDPC_Decoder(), VN_Cal_HD() | 우리 HD 경로는 min-sum 기반 — 복호 알고리즘 계열 상이 |
| 반복별 알고리즘 차수 스위치(NSS) | decoder.cpp Get_VNU_Table_Idx(), ecc_data.h PARAM_LLR | 우리는 반복별 LLR threshold 테이블 — 목적 유사하나 대상(MB 차수 vs LLR)이 다름 |
| EXIT 수렴/스위칭점 분석 | 대응 없음 | 오프라인 이론 분석, 런타임 코드에 없음 |
| majority-based bit-flip | 대응 없음 | 우리는 message-passing min-sum, bit-flip 미채택 |
```

적용 가치: **낮음** — 경판정 Gallager B 스위치 기법은 min-sum 소프트 복호를 쓰는 Prime ECC와 복호 계열이 다르고 대상 부호가 저rate 정칙이라 NAND 고rate QC-LDPC에 직접 이식 실익이 적다.
