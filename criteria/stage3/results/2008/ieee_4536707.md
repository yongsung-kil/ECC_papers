### B. 알고리즘 요약

1. 시스템: time-variant LDPC-CC(convolutional), syndrome former `H^T`는 무한·주기(period `Ts`), rate `R=(c-1)/c`, regular `(ms,3,K)` (예: (,3,6) rate-1/2, (,3,9) rate-2/3).
2. 문제: LDPC-CC 디코더는 processor로 iteration 차원 병렬이나 processor 내부 node 연산이 serial → throughput 한계, node 병렬화 시 memory access collision 발생.
3. 핵심: code와 decoder를 joint 최적화하여 architecture-aware LDPC-CC 구성 + node 차원 병렬 memory-based 디코더를 동시 설계.
4. 제약: 구성에 5개 제약 — C1(직접인코딩, `H0^T` full-rank), C2(CNU 1clk memory access), C3(node 병렬 충돌없음), C4(`ms` 최대화), C5(girth 최대화).
5. 구성: parity rows `P`(Ts×Ts, `p×p` circulant), info rows `S(r)`(4×4 block에 대각 block=0), base matrix 조립 후 cut/swap/repeat로 `H^T` 생성.
6. 디코더: circular buffer를 `8p` memory bank로 분할, x-y 평면 + z차원 `p` plane, `p-to-p` cyclic-shift switch matrix(Q1,Q2,Q3)로 `p`개 CNU 충돌없는 접근.
7. 스케줄: dual-port memory로 baud period를 2 clock으로 축소 → clock당 `p/2` info bit throughput, 회로 주기는 `g=Ts/p`.
8. 비용: CNU `320·q` gates, VNU `250·q` gates; 병렬화 `p`로 total memory bit 증가 없이 gate만 `p`배 증가.
9. 결과: `p=7`, 300MHz에서 rate-1/2 코드 ~1 Gb/s, latency `p`배 감소; BER은 architecture-independent LDPC-CC(ms=512)와 comparable (구속장 996/1004, (3,6) block length ~10000 수준).
10. 한계: convolutional 전용 아키텍처, HW는 gate 추정치(합성/실측 없음), 정정성능 개선은 없음.
