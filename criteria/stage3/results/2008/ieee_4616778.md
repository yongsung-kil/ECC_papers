### B. 알고리즘 요약

1. 대상은 WiMAX(802.16e)용 irregular QC-LDPC — base model matrix `Hbm`를 expansion factor `z`로 확장한 binary 패리티행렬 `H`, rate 1/2~5/6, length 576~2304, AWGN 채널.
2. 문제: 기존 프로그래머블 디코더는 96 병렬 프로세서로 278~666Mbps를 내지만 물리계층 최대 40Mbps에 비해 과잉 복잡·전력이거나(2,3), 저전력이지만 6개 코드 중 1개만 지원(5).
3. 알고리즘: Hocevar의 layered-decoding message-passing(수렴 가속) + Chen의 offset-based Min-Sum(저복잡), block row를 layer로 처리.
4. CFU는 serial-mode로 irregular check degree 수용, `new bit-msg = AP-LLR - old chk-msg`, 여기에 programmable offset `max(0,q-β)` 적용 후 min1/min2·최소 edge index 산출.
5. check-message generator = MUX(min1/min2 선택) + 조건부 inverter(sign XOR), sign XOR로 tentative parity 계산 → 전 parity 충족 시 조기종료(전력 절감).
6. subset-P: `z` CFU 전량 대신 `P`(z의 약수)개 프로세서만 써서 목표 throughput에 맞춰 복잡도 축소, `Hbm` permutation으로 layer/열 처리순서 최적화해 wait cycle 감소.
7. 핵심 HW 트릭: 한 메모리 워드에 다수 AP-LLR을 cascade + multi-access·auxiliary 출력회로로 24 뱅크를 6 뱅크로 통합, read 포트 2배 클록·나머지 뱅크 clock-gating으로 면적·전력·라우팅 congestion 감소.
8. barrel cyclic shifter가 block column의 AP-LLR을 목적 CFU로 정렬 라우팅, 주소는 on-the-fly 계산.
9. 결과: TSMC 0.18-μm, 24 CFU, 60Mbps@10iter, 55K gate·55576 mem bit; 타 WiMAX 디코더 대비 최소 53% 소형, 에너지 27%(layered 1.7배 빠른 수렴 포함 시 45%) 우수, error floor 미관측.
10. 한계: WiMAX 저rate 전용, partly-parallel 저throughput, 칩 fabrication 진행 중(post-layout PrimeTime 추정, 미실측).
