### B. 알고리즘 요약

1. 자기기록(perpendicular recording) read channel의 inner LDPC 부호 설계 — outer는 전통 RS, SOVA detector + LDPC를 iterative로 결합, 512B/4kB sector 대상.
2. 문제: LDPC를 MR에 쓰려면 (a) RS 대비 성능 우위 보장, (b) 인코딩/디코딩 복잡도 절감 두 가지를 풀어야 함.
3. 공통 아이디어: 여러 SPC 부호를 인터리버로 정의 → SPC의 인코더/디코더 단순성 상속, **H를 저장 안 하고 인터리버로 계산**, code length/rate/column weight를 몇 개 인자로 programmable.
4. Type-I: 사용자 비트를 2-D 배열에 두고 parity 1행+2열 padding, 3세트 SPC(열가중치 3,3,2), 예 (1640,1521) rate `0.927`, (4920,4563).
5. Type-II: parity 행 없이 3 parity 열, length/rate 더 유연하나 H 일부가 약함 → parity-on-parity 비트 추가나 세트간 상호 check로 regular화(단 SPC 단순성 상실).
6. Type-III: 직사각 정수격자[6] 기반 girth-8 QC-LDPC — slope-set을 triangle-free 조건으로 찾아 6-cycle 제거, 격자=코드워드 2-D 블록으로 재해석해 고rate화, 예 (4470,4023) rate `0.90`, (5379,4890) rate `0.909`(4kB용).
7. 인터리버: (pseudo-)random 열/행 permutation 또는 slope 기반, 두 세트 인터리버가 공통 entry 2개 안 갖도록 제약해 `d_min` 저하 방지.
8. Endec: 인코더는 SPC별 parity 누산(Type-I 병렬, Type-II 순차), 디코더는 factor graph의 3 subgraph에 대응하는 3 sub-decoder가 메모리 풀 공유, normalized min-sum 사용(min/submin/위치/부호만 버퍼, V2C 미버퍼, VN측 2N 메모리).
9. 결과: 512B/4kB sector에서 설계 LDPC가 SPC 대비 SFR 0.4/0.7dB 이득, Type-II는 Type-I보다 약간 열세지만 유연, girth 민감도 낮음(Type-III만 소폭↑), random Mackay 코드와 대등 이상.
10. 한계: 순수 시뮬(gatecount/throughput 없음), MR 채널·약한 SPC baseline, soft LLR 양자화 bit수 미명시.
