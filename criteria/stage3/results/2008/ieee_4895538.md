### B. 알고리즘 요약

1. 대상: IEEE802.16e QC-LDPC(`N=2304, R=1/2` 주 실험, AMC는 길이 576/1152/1728·rate 1/2~5/6), BPSK급/AWGN 및 3-path fading, 최대 iteration 50, BP 복호.
2. 문제: H-ARQ에서 복호 실패 프레임이 매번 최대 iteration까지 도달해 평균 iteration(=복잡도)이 급증. 낮은 SNR에서 실패를 조기감지할 필요.
3. 핵심 관찰: 오류 프레임의 미충족 체크노드 수 `w(s_i)`(= syndrome weight)는 몇 iteration 후 0이 아닌 값으로 수렴(정체)하며, SNR이 낮을수록 `w(s_i)` 평균이 높음(Fig.1).
4. 기법1 - Stagnancy check: 최근 `Istag` iteration 동안 `w(s_i)`의 최대-최소 차 `NmUaCx − NmUiCn ≤ δstag`(본 논문 `δstag=0`)이면 정체로 보고 복호 중단.
5. 기법2 - Early BP convergence speed check: 고정 시점 `i=Ichk`(예: 3)에서 `w(s_i) > δth`이면 회생불가 프레임으로 간주 조기중단. cycle-free 가정이 성립하는 극초기에 적용해 stagnancy가 아직 동작 못 하는 구간을 커버.
6. 임계 `δth`는 전체 체크노드 수의 백분율(10~30%)로 코드마다 동일 원리로 설정 → 가변 rate/length에 scalable(Table 2).
7. 결합: 기존 syndrome stopping(`w(s_i)=0`)에 두 기준을 추가하는 9-step 절차. 필요한 metric은 오직 `w(s_i)`라 추가 복잡도 거의 없음(Fig.2 flowchart).
8. 결과(AWGN): `Istag=7`로 WER 열화 거의 없이 1.0dB 이하에서 불필요 iteration 제거; 결합 시 극저 SNR에서 평균 iteration이 3~4(≃Ichk)까지 급감(Fig.5,6).
9. 결과(H-ARQ+AMC, Type-I 최대 4회 재전송, 288 OFDM subcarrier): 단일 임계 규칙으로 여러 코드에 걸쳐 복호 복잡도 70-80% 감소(Fig.9,10).
10. 한계: HW 미설계·PC 시뮬만, 무선 H-ARQ/fading 특화, 임계 튜닝 필요, 정정능력 자체는 개선하지 않음(복잡도/latency만 개선).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 실패 조기중단(stagnancy/speed) 결정 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 기존 CRC 성공감지 조기종료에 실패 조기감지 로직을 저비용 추가하는 지점 |
| w(si) 미충족 체크노드 카운트 | decoder.cpp LDPC_Decoder() | 매 iteration syndrome/checksum 계산 지점, 이미 계산되는 값 재사용 가능 |
| iteration 루프 조기탈출 제어 | decoder.cpp LDPC_Decoder() | 조기중단 조건을 이터레이션 루프에 삽입 |
| H-ARQ/AMC scheduling | 대응 없음 | Prime ECC는 재전송/변조적응 계층 없음 |
```

적용 가치: **중간** — syndrome weight만으로 uncorrectable 프레임을 조기중단하는 부분은 min-sum 복호와도 호환되고 이식이 매우 쉽지만(오직 기존 syndrome 재사용), NAND는 재전송 대신 read-retry/soft-read 승격이라 임계를 RBER에 맞춰 재튜닝해야 한다.
