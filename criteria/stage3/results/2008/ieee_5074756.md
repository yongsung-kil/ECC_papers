### B. 알고리즘 요약

1. 코드/시스템: flexible LDPC decoder 대상, IEEE 802.11n TGn Sync irregular LDPC 2종 `(1728,864)` rate-1/2, `(1728,1296)` rate-3/4, BPSK/AWGN, max iteration 20.
2. 문제: offset MS의 유한정밀 구현서 고SNR일수록 intrinsic LLR `2r/σ²`의 dynamic range가 넓어져 `(6:2)` 양자화가 고신뢰 정보를 최대값(±8)으로 절단 → error floor·성능저하. flexible decoder는 코드마다 다른 offset·양자화 필요.
3. 채널/모델: AWGN, intrinsic info `A_v = 2r_v/σ²` (`r_v`=수신 soft symbol, `σ`=noise std).
4. 기존 offset MS: CN `R_cv = S_cv·max(min_{n≠v}|L_nc| - β_σ, 0)`, VN `L_vc = ΣR_mv + A_v`, 판정 `L_v = ΣR_mv + 2r_v/σ²`.
5. 핵심관찰: 최적 offset `β_σ`는 SNR 따라 변하나 `σ²·β_σ`는 거의 상수 → offset을 채널독립 상수로 고정 가능.
6. 제안: 수신 데이터 직접 사용. `L_vc = ΣR_mv + θ·r_v`, `L_v = ΣR_mv + θ·r_v`, modified offset `β_n = (θσ²/2)·β_σ` = 코드별 상수(0.125), `θ = 1/(4σ²β_σ)` 코드별 사전결정.
7. 효과: intrinsic이 채널정보(σ²) 독립 → 하나의 양자화 scheme으로 여러 LDPC 코드 처리(flexible decoder), 채널추정 오차에도 강건.
8. 양자화: 균일 `(q:f)` 표기(q=총비트, f=소수부). 전체 데이터가 downscale되어 기존보다 소수부 비트 더 할당(`(6:3)`,`(5:3)`).
9. 결과: 제안 `(6:3)`은 floating point 근접, 기존 `(6:2)` 대비 BER=10^-6서 >0.2dB 이득, `(5:3)` 5비트도 기존 6비트 `(6:2)` 대비 ~0.1dB 이득, BER=10^-7까지 error floor 없음.
10. 한계: 시뮬(BER)뿐, HW·gatecount 없음, AWGN/BPSK·802.11n irregular 코드 전제, `θ,β_n`는 코드별 사전 튜닝 필요.
