### B. 알고리즘 요약

1. easy-encodable LDPC 부호족 제안: polynomial(Binomial Product Generator, BPG) 구조. BPSK/AWGN, 표준 SPA-LLR 최대 50 iter로 복호.
2. 문제: 좋은 BER(Shannon 근접)와 easy encoding·decoding을 동시에 만족하는 부호 설계가 미해결(product code는 rate 제약, turbo는 error floor).
3. 구성: M개 성분부호를 2D interleaver memory에 쓰고 성분별로 다른 reading rule로 parity 생성 → 부호어 `t(x)=m(x)g(x)`, `g(x)=Π_{j}(x^{rj}+1)`.
4. 두 변형: Layered Codes(LC, Fig.3, rank 부족→각 layer가 `rj−1`행)와 Serially Concatenated(SC, Fig.4, lower-triangular·full rank, redundancy = r).
5. SC는 M개 shortened cyclic code(`gj(x)=x^{rj}+1`)의 직렬연접으로 해석, `n = k + Σ ri`.
6. sparsity: 각 열 최대밀도 `M/Σrj` → `rj`를 크게 잡으면 sparse H 확보.
7. Theorem 1: distinct·coprime·increasing `rj`에서 `n ≤ r1·r2`면 length-4 cycle 없음(SC는 `n ≤ r1r2 + Σ_{j=2}^M rj`로 완화).
8. 설계 자유도: `rj` 선택으로 product code의 length/rate 제약을 극복, 기존 serially-concatenated SPC 대비 error curve 개선.
9. 결과(시뮬): rate~1/2 SC(3125,1603)·LC(3116,1598), rate~3/4 SC(8840,6630)·LC(8820,6615), rate~1/2 long SC(32900,16873); 동일 length/rate서 SC가 LC보다 우수하나 error floor는 더 높을 수 있음.
10. 한계: HW 미설계, 표준 SPA(50iter) 그대로, QC-LDPC 아님, rate 낮음, SC의 error floor 우려.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polynomial/BPG H-matrix 구성 | ecc_top.cpp Load_PCM() | Prime ECC는 QC-LDPC 고정; 다른 부호족이라 직접 대체 불가 |
| easy encoding (polynomial g(x)) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC는 dual-diagonal 인코딩; polynomial 구조와 상이 |
| short-cycle-free girth 설계 | 대응 없음 (H-matrix 생성 단계, Prime ECC는 고정 PCM) | girth는 부호 생성 시 결정, 이식 대상 아님 |
| SPA-LLR 복호 | 대응 없음 (Prime ECC는 min-sum) | standard full-tanh SPA 미사용 |
```

적용 가치: **낮음** — QC-LDPC가 아닌 polynomial/BPG 부호족이고 rate가 낮으며 HW 미설계라, QC 고정·고rate NAND용 Prime ECC에 이식 실익이 없다.
