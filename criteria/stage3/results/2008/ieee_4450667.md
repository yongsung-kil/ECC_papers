> 총평: 패킷기반 LDPC verification 복호의 false-positive를 다중검증으로 낮춰 FER 2자릿수 개선하나, bit-level soft min-sum QC-LDPC(Prime ECC)와 패러다임이 달라 이식가치 낮음.

### B. 알고리즘 요약

1. 시스템: 각 변수노드가 `b` bit 패킷(`xi=[xi1..xib]`)인 packet-based regular LDPC, BSC(crossover `p`) 전송, check는 이웃 패킷들의 bitwise XOR=0 제약.
2. 문제: 기존 VA(Luby-Mitzenmacher)는 단 하나의 check가 만족되면 그 이웃 변수노드를 모두 verified 처리 → false verification 확률이 높아 FER 악화.
3. 핵심 기법(EVA): 변수노드를 verified로 확정하려면 **동시에 `d`개 이상**의 incident check가 `sum(cvk)=0`(식1)을 만족해야 함(`numc≥d`).
4. threshold 스케줄: `d`를 `dv`에서 시작해 매 iteration마다 `d→d-1`로 완화, 최종 `d=1`까지 반복(VA = threshold를 상수 1로 둔 EVA의 특수 케이스).
5. 2단 반복: (verification) 식1 만족 check 수로 검증 → (correction) 한 check의 이웃 중 미검증이 1개면 나머지의 XOR로 그 값 확정(식2), 더 옮길 노드 없거나 `m`회 도달 시 종료.
6. 복잡도: 패킷연산(dc XOR 검증 + dc-1 XOR 정정)이 지배, 첫 iteration 후엔 값이 바뀐 check만 식1 재계산 → 재계산 횟수 제한적.
7. 간소화(EVA-x): 상한 threshold를 `dv` 대신 `x(<dv)`로 낮춰 계산량 절감, 성능-복잡도 trade-off.
8. 실험: (1008,504) regular LDPC(`dc=6, dv=3`), 패킷 32~256 bit, `m=5`(기본)/50, BSC.
9. 결과: `p≤10^-3`에서 EVA가 VA 대비 FER 100배 이상 개선(2 orders), 계산량 증가 <90%; 작은 패킷일수록 FER 낮음; EVA-2는 `p=10^-2~10^-3`서 EVA에 근접하며 저복잡도.
10. 한계: BSC 패킷채널 시뮬만, HW·gate count 없음; 검증 신뢰도가 패킷 크기(랜덤 XOR 충돌 희소성)에 의존 → 단일 bit에는 부적합.
