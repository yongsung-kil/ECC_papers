### B. 알고리즘 요약

1. 대상: array 구조 QC-LDPC (중국 DTTB inner code, `N=7493`, `K=3048/4572/6096`, circulant `b=127`, irregular) 소프트판정 디코더 HW.
2. 문제: parallel 디코더는 면적 과대, serial은 latency 과대, semi-parallel은 serial factor가 circulant 크기 `b` 이하로 못 내려가 저throughput 무선기기엔 여전히 HW 과잉.
3. 알고리즘: normalized UMP BP(정규화 min-sum), 스케일 factor `α=0.625`로 horizontal message 정규화 - LLR BP 대비 성능손실 없이 곱셈/`tanh` 제거.
4. 핵심 식 (2): `L_{mn}^i = α · sgn(∏ Z_{mn}^{i-1}) · min_{n'≠n} |Z_{mn'}^{i-1}|` - min1/min2 방식 min-sum CN 갱신.
5. 기존 semi-parallel: circulant 당 HPU/VPU 1개 재사용으로 serial factor `S=b`, 추가 PU G개로 `S=b/G`까지만 축소 가능(하한 존재).
6. 핵심 기법: folding factor `F`로 한 sharing group이 `F`개 circulant row/column을 커버 → PU가 `bF`회 재사용, serial factor `S=bF`, PU 수는 semi-parallel 대비 `1/(GF)`로 감소.
7. 구현 트릭: circulant 당 LRAM/ZRAM 블록, `F` RAM class를 tri-state 버스로 멀티플렉싱(MUX 대신 high-impedance 출력)해 PU에 교대 접속; irregular는 dummy 입력(HPU=양의 최대, VPU=0)으로 weight 정렬.
8. 파라미터: cycles/iter = `bF`, memory capacity = `2bΩ+N` (semi-parallel과 동일 - 메모리 불변).
9. 결과: Verilog+Stratix II FPGA 합성, `F=2`에서 HW ~40% 절감 대신 throughput 절반; ALUT 10.6k~26k, memory bits ~0.38~0.40M(구체 throughput 수치는 Fig.6 그래프뿐).
10. 한계: 성능 무손실(정정능력 개선 없음), throughput 감소가 대가, memory/routing 불변이라 folding이 커질수록 이득 체감.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 정규화 UMP BP(=정규화 min-sum) CN 갱신 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리 min-sum과 동일 계열 - 이미 보유(중복) |
| VP(합/차, hard-decision) | decoder.cpp VN_Cal_HD() | 표준 VN 처리, 보유 |
| folding PU time-sharing HW 아키텍처 | 대응 없음 (우리는 z=32 고정 병렬, folding 스킴 별도 미존재) | 무선 저throughput용 직렬화 - NAND 방향과 역행 |
| LRAM/ZRAM 메시지 저장/라우팅 | 디코더 HW 모델(z=32) | 메모리 구조 참고 수준, 우리 pipeline과 정합 부담 |
```

적용 가치: **낮음** - 핵심 알고리즘(정규화 min-sum)은 이미 보유하고, folding은 throughput을 깎아 HW를 줄이는 무선 저속용 기법이라 NAND 고throughput 요구와 상충한다.
