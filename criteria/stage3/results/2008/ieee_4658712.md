### B. 알고리즘 요약

1. 대상: 원래 PEG(Progressive Edge-Growth) 기반 bipartite Tanner graph 구성 개선. regular `(dv,dc)` 그래프(binary (3,6) 및 ultra-sparse `dv=2` NB-LDPC over GF(64)).
2. 풀려는 문제: 짧은~중간 길이에서 short cycle이 성능을 좌우 — 목표 girth `gt`를 최소 크기 `N=Ng(dv,dc)`에서 달성하고, girth를 못 올리면 길이-`gt` cycle 수(multiplicity)를 줄이기.
3. 기법1 truncated spanning tree: 최대 depth `lmax=(gt-2)/2`까지만 span → 후보군이 커져 randomization 여지 확보, 실패 시 전체 edge 폐기 후 재시작(Monte-Carlo식 다수 시행 후 best 저장).
4. diameter 논거: 최소 크기 그래프는 diameter `d=gt/2`이므로 tree를 `lmax=(gt-2)/2`로 span해야 `gt`보다 짧은 cycle 생성을 원천 차단.
5. 기법2 objective function: 후보 중 (1)depth `lmax` 정확히 위치, (2)생성 cycle 수 `nbCycles` 최소, (3)현재 degree 최소 후보만 남기고 무작위 sampling.
6. gap refinement: `Ng(dv,dc)<N<Ng+2(dv,dc)` 구간은 diameter 논거 미성립 → `lmax=(gt+gap-2)/2`로 시작해 진행 중 gap을 줄여 점차 작은 cycle 허용(regular 부호는 gap=2 전형).
7. dv=2 ultra-sparse 결과: dc≤50에서 g=6,8 minimal 그래프(하한 달성) 구성, dc=3,4에서는 g=16까지 하한 달성 (Table I, 표준 PEG 대비 N 대폭 축소).
8. NB-LDPC 응용: (2,4) N=160 GF(64) 부호 — 표준 PEG는 girth 12, RandPEG는 minimal girth 16 → BIAWGNC에서 error-floor 개선(Fig 1).
9. binary (3,6) 결과: N=504에서 길이-8 cycle 808→452개, N=1008에서 167→31개로 감소 → BSC+BP 복호에서 error-floor 영역 성능 개선(Fig 2,3, MacKay/PEG 대비).
10. 한계: HW/디코더 무설계(표준 BP 사용), 출력이 non-QC 그래프, 핵심 강조는 non-binary GF(q), rate 1/2 예제만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RandPEG 그래프 구성(girth multiplicity 최소화) | [ecc_top.cpp](Prime_ECC_3.1_Claude) `Load_PCM()` | 생성 그래프가 non-QC라 circulant/z_sb=32 구조와 부정합 → PCM 직접 주입 불가 |
| girth-g cycle 최소화 → trapping/stopping set 감소 → error-floor 개선 | 대응 없음 (부호설계 오프라인 원리) | 디코더(min-sum) 무변경, 원리만 QC-LDPC 설계에 참고 가능 |

적용 가치: **낮음** — error-floor 저감 원리(girth multiplicity 최소화)는 NAND에 관련성 높으나, 출력이 non-QC이고 non-binary 중심이라 Prime ECC의 고정 QC-LDPC에 직접 이식 불가, 개념 차용에 그친다.
