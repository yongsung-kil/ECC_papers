> 총평: 유연(코드 독립) 멀티프로세서 LDPC 디코더의 상호연결망(de Bruijn NoC) 논문 — 고정 QC-LDPC NAND ECC의 단순 cyclic-shift 구조와 정합 안 되어 이식성 하.

### B. 알고리즘 요약

1. 대상: 부분병렬(semi-parallel) 멀티프로세서 LDPC 디코더의 VNP-CNP 간 메시지 교환용 on-chip 상호연결망(NoC).
2. 문제: LDPC BP 복호는 연산은 단순하나 노드수·차수·iteration에 비례하는 통신량이 폭증(DVB-S2 예: 64800bit, 30 iter, iter당 30만 데이터) → 상호연결이 구현 병목.
3. 접근: 코드 독립(code-independent) flexible 디코더를 지원하려면 메모리 접근충돌을 라우팅으로 완화해야 함 → de Bruijn 그래프 위상 채택.
4. 위상: 이진 de Bruijn 디그래프 `dB(2,m)`, `N=2^m` 노드, 각 노드 2개 출력에지, 지름 `log2(N)`로 최적, 재귀구조로 확장성 우수.
5. 라우팅: shortest-path + deflection routing. 원래 순차적 계산(1~log2N cycle)을 for-loop unrolling으로 병렬화 → 라우터당 1 clock cycle에 출력포트 결정(Table 1의 병렬 비교).
6. 충돌처리: 버퍼링 대신 deflection, 중재는 iterative 수렴 기여도 기반 priority(4bit) → Time Stamp(6bit, age) → round-robin 순.
7. 패킷: header(목적지 id + TS 6bit + priority 4bit) + payload(목적지 메모리 write addr + soft-info LLR 8bit).
8. HW: 라우터(operative+control unit, MUX 스위칭/비교기/transcoder), network interface(header table, 송수신 FIFO), C 프로그램으로 generic RTL VHDL 자동생성.
9. 결과: ST 0.18µm 합성, 네트워크 면적 3.56mm², 266MHz, aggregate bandwidth 85.12Gbps, 병렬도 16, 1024bit 코드; 대역폭/면적 효율이 2D-mesh/MDN/Beneš 대비 최상, 평균 latency는 프로세서수에 log 증가(8→512P에서 4→22 cycle).
10. 한계: 정정성능/BER 기여 없음(순수 통신구조), 유연성 확보용이라 고정 QC-LDPC엔 과잉, 실측(fab) 아닌 합성 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN-CN 메시지 라우팅/에지 선택 | [GT.cpp](../../Prime_ECC_3.1_Claude/) `Make_GT_HW()` | 둘 다 MUX 기반 에지/메시지 경로처리이나 GT는 edge thinning, 본 논문은 임의코드 패킷라우팅 — 개념 상이 |
| 반복복호 루프(NoC가 서비스하는 디코더) | [decoder.cpp](../../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 복호 알고리즘 자체는 불변, NoC는 그 위 통신계층 |
| de Bruijn 위상/패킷스위칭/deflection 라우팅 | 대응 없음 | Prime ECC는 고정 QC-LDPC의 cyclic-shift(z=32) 라우팅 — NoC/패킷 위상 모듈 없음 |
| soft-info 8bit 메시지 폭 | 대응 없음 | 내부 message quantization 이슈로, NAND read LLR 테이블(PARAM_LLR)과 별개 |

적용 가치: **낮음** — 코드 독립 flexible 디코더용 NoC로, 고정 QC-LDPC(z=32 shift) NAND ECC 엔진에는 라우팅 병목 자체가 단순하여 de Bruijn 패킷스위칭의 오버헤드가 정당화되지 않음.
