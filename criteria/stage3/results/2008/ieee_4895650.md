### B. 알고리즘 요약

1. AWGN 채널에서 rate-compatible LDPC 부호를 위한 좋은 puncturing 패턴 탐색을 다룬다(mother code 1/2 → 0.6/2/3/0.7/0.75).
2. 기존 [3] 기법은 punctured node당 survived check(SC) node가 하나뿐이라는 단순 가정 + 시뮬 검증 한계가 있었다.
3. `k-SR`(k-Step Recoverable) node, `SC`(survived check), `DC`(dead check) node를 정의하고, punctured node는 채널 LLR이 0이라 이웃 통해 복구되어야 한다.
4. Gaussian approximation DE로 다중 SC node에서 오는 평균 LLR을 `m_u^{k+1,i}=φ^{-1}{1−[1−φ(m_v^0)]^{S(C_i)}}`로 유도한다.
5. VN 출력 LLR은 입력들의 합 `m_v^{k+1}=Σ_i m_u^{k+1,i}`이므로 SC node가 많을수록 LLR이 더 빨리 수렴(식 2).
6. DC node도 하위 punctured node가 모두 복구된 후 식(5) `m_v^{1,1}=φ^{-1}{...S(C1)}+φ^{-1}{...S(D)−S(C1)}`로 양의 기여를 줌을 보인다.
7. 오류확률 `Pe(V)=Q(√(m_v/2))`(식 3)로 연결하여 SC/DC node 수 증가가 오류확률을 낮춤을 결론.
8. 이 분석에 기반해 SC node 수를 최대화하는 puncturing 패턴 선택 알고리즘을 제안(block-puncturing 포함).
9. 결과: (3,6)-regular(PEG) 및 802.16e irregular block-type LDPC(info 1080bit, SP-BP 50 iter)에서 저rate(0.6, 2/3)에서 기존 grouping 알고리즘 대비 FER 우월, 고rate(0.75)에선 근접/열세.
10. 한계: HW 미설계, sum-product 복호 가정, error-floor·양자화 미분석, rate 대역이 NAND(~0.9)보다 낮음.
