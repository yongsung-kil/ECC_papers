> 총평: 가변 expansion factor(19+3종)를 단일 permutation shifter로 처리하는 다중표준 라우팅 네트워크 HW로, 고정-z NAND QC-LDPC에는 유연성이 과잉이라 이식가치가 낮다.

### B. 알고리즘 요약

1. 대상은 IEEE 802.16e/802.11n QC-LDPC로, `H`를 `zf×zf` circulant-shift/zero 서브행렬로 분해(802.16e `nb=24`, `mb≤12`, `zf=24~96` 19종 / 802.11n `zf=27~81` 3종), rate는 `mb/nb`.
2. 문제: 가변 서브행렬 크기 때문에 Omega/Benes 등 고정크기 crossbar를 못 쓰고, 크기별 전용 라우팅 HW는 신호 혼잡·복잡도가 커진다.
3. 모델: message passing 라우팅을 서브행렬 크기(zf)의 circular shift로 환원 - correction 알고리즘과 무관한 순수 순열(permutation) 문제.
4. 핵심기법 1단: enhanced self-routing(SR) 스위치 네트워크 - 3-stage(SRB 결합 → barrel shifter 순열 → 선택)로 `zmax` 라우팅 중 기대 `z`개를 self-routing bit로 선택, shift가 `zmax/2` 초과 시 단순화된 결정규칙으로 제어로직 축소.
5. 이유: 단일 barrel shifter만으로 모든 zf를 커버해 크기별 중복 MUX(신호 혼잡)를 제거하는 것이 목표.
6. 핵심기법 2단: two-way duplicated 스위치 네트워크 - `z<zmax/2`일 때 메시지를 복제해 서로 다른 서브행렬 두 그룹을 동시 라우팅(2zmax), 병렬성 2배.
7. 구현 디테일: 130nm 합성에서 enhanced SR/two-way duplicated 게이트 21.9k/37.4k, 면적 0.1095/0.1872mm², 동작 384MHz.
8. 학습/최적화: 없음(구조 설계).
9. 결과: 작은 zf에서 two-way duplicated로 디코딩 throughput 50%↑; 90nm 1P9M 디코더 칩 6.25mm², 107MHz에서 최대 63.36Mb/s, 203mW@1.0V, 20 iteration.
10. 한계: correction 성능 개선 없음(순수 라우팅), 다중표준 가변크기 지원 특화 - 단일 고정 z 시스템엔 불필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC circulant-shift 메시지 라우팅 스위치 | GT.cpp Make_GT_HW() | edge/MUX 라우팅에 대응하나 Prime은 고정 z=32라 가변-size 스위칭 불필요 |
| 디코더 message passing 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | 라우팅이 붙는 상위 복호 루프 |
| 가변 expansion factor(19+3종) 지원 | 대응 없음 | Prime ECC는 단일 고정 z=32 QC-LDPC |
```

적용 가치: 낮음 - 다중표준 가변-zf 라우팅이 핵심 기여인데 Prime ECC는 고정 z로 단일 barrel shifter면 충분하고 correction 이득도 없다.
