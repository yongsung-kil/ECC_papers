### B. 알고리즘 요약

1. 대상: binary (n,k) LDPC over BIAWGN. 실험은 regular `(504,252)` (rate 0.5) 코드.
2. 문제: 유한길이 Tanner 그래프의 짧은 cycle이 메시지 의존성을 만들어 BP/MS가 신뢰도를 과대추정, waterfall 성능 손실.
3. 핵심 개념: 반복 복호를 고정점 문제 `V = h(V,V0)`로 보고 수치해석의 successive substitution(SS, β=1) 대신 successive relaxation(SR) 적용.
4. 핵심 식: `Vk+1 = Vk + β(h(Vk,V0) − Vk)` — 새 메시지 방향으로 일부(β)만 이동해 신뢰도를 낮춰 과대추정 완화. β는 최적화 대상(보통 <1).
5. 도메인 확장: SR을 BP/MS 각각 LR·LLR 도메인에 적용(SR-BP-LR/LLR, SR-MS-LR/LLR) → 도메인마다 궤적(로그/지수 곡선)이 달라 결과가 달라짐.
6. 결론적 최적 조합: SR-MS-LLR이 최고 성능이자 최저 복잡도(VAR=합, CHK=부호·min). (504,252)에서 최적 β≈0.5.
7. 양자화 트릭: 수신값을 ±cth로 클리핑 후 균일 양자화(2^q−1 구간), 정수 연산, bit node 출력은 ±(2^{q-1}−1)로 클리핑. 최적 cth는 Eb/N0·q에 따라 증가.
8. 최적화: β는 Eb/N0·M에 거의 무관, 주로 코드·알고리즘 종속(SR-MS-LR만 β에 민감).
9. 결과: SR-MS-LLR이 표준 SS-BP 대비 최대 약 0.5dB 개선(M=200, 10000), MS-UC보다 우수(동일 avg iteration). 부동소수점 성능 근접에 6~7bit 양자화 필요.
10. 한계: 짧은 regular 코드 1종 시뮬만, HW 미설계, DE 해석 불가(메모리 존재), 직전 iteration LLR 저장 메모리 추가.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SR 완화 반복 갱신(β, 직전 메시지 메모리) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 루프 갱신식에 β 혼합 항 삽입으로 이식 가능 |
| min-sum CN 연산(sgn·min) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유(min-sum), 동일 연산 위에 SR만 추가 |
| 메시지 양자화(6~7bit, 클리핑 cth) | decoder.cpp VN_Cal_HD() 등, ecc_data.h PARAM_LLR | 양자화 테이블/폭 재검토 필요(6~7bit 요구) |
| 직전 iteration LLR 저장 메모리 | 대응 없음 | 신규 상태 메모리 추가 필요(HW 면적 영향) |
```

적용 가치: 중간 — 이미 보유한 min-sum 위에 β 완화항만 얹는 저비용 waterfall 개선이나, 직전 LLR 저장 메모리와 6~7bit 양자화 요구, 고rate QC-LDPC 재검증 필요.
