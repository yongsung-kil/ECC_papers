### B. 알고리즘 요약

1. 대상: [63,39,9]-BCH(dense 대수부호), rate-1/2 PEG-LDPC([50,25],[100,-],[200,100]), AWGN+BPSK, soft BP.
2. 문제: 짧은 대수부호는 high-density parity-check 행렬 탓에 표준 BP 성능이 나쁨(ML과 격차).
3. 배경/기법1(MBBP): 한 부호의 dual space 다중 basis로 비등가 parity-check 행렬 `l`개를 만들어 병렬 BP 디코더를 돌리고 결과를 결합, near-ML 달성(dense 부호는 automorphism group 순열로 생성).
4. 한계 지적: automorphism group은 크기 제한·PEG 부호엔 미지, minimum-weight cog 수 부족 시 표현 다양성 확보 곤란.
5. 기법2(Leaking): 초기 iteration엔 고신뢰 채널정보만 주입하고 저신뢰 bit는 erasure(Pr=0.5)로 두었다가 iteration 진행에 따라 threshold `δ_i`를 선형 감소시켜 점진 주입.
6. 핵심 식: `|y_ν|≥δ(y)=(σ²/2)ln(δ1/(1-δ1))`면 정보 통과, 아니면 erasure — girth-4 짧은 cycle 영향을 약화(AWGN→BEC로 채널 이동 효과).
7. Leaking은 동일 행렬 위에서 표준 BP와 다른 오류패턴을 정정하므로 MBBP의 추가 표현으로 활용(Leak MBBP).
8. 기법3(행렬 구성): Tanner graph의 cycle(`G_{c,τ}`) 지식으로 `c/2`개 check를 선형결합해 weight `wr=Σwi−c`의 redundant check 생성 → automorphism 없이 PEG 부호용 비등가 행렬 collection 구성.
9. 결과: BCH는 MBBP(l=6)가 ML union bound와 0.25dB 격차, Leak MBBP(l=30, I=100~300)로 ML에 근접; PEG(n≈50)은 Gallager bound 상회.
10. 한계: HW 미설계, full-SP/BP·다중 병렬 디코더로 복잡도 큼, rate1/2 단·중길이 부호에 국한.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 복호 이터레이션 | decoder.cpp LDPC_Decoder() | Prime은 min-sum, 본 논문은 full-SP/BP(tanh) → 방식 상이 |
| Leaking(iteration별 신뢰도 threshold gating) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | Prime 적응형 LLR 테이블과 개념 유사(중복) |
| 다중 parity-check 행렬(bases) 로드 | ecc_top.cpp Load_PCM() | Prime은 단일 H 고정, 다중 basis 미지원 |
| 다중 병렬 BP 디코더 결합 | 대응 없음 | NAND HW에 다중 full 디코더 비현실적 |
| automorphism 독립 H 구성/cycle 기반 redundant check | 대응 없음 | 부호구조 고정, 재검증 부담 |
```

적용 가치: **낮음** — full-SP/BP + 다중 basis 병렬 디코더 구조는 NAND 고rate 장부호 min-sum HW와 근본적으로 어긋나며, 유일하게 전이 가능한 Leaking 아이디어도 Prime의 적응형 LLR 테이블과 개념 중복이라 추가 이득이 제한적이다.
