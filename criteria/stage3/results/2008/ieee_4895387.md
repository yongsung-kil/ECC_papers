### B. 알고리즘 요약

1. 대상: binary-input AWGN 채널, regular LDPC(`n=1008, m=504`, col weight 3, PEGReg504x1008), rate 0.5.
2. 문제: BF 복호는 구현이 단순하나 BER이 sum-product/min-sum보다 크게 열세 — 이 성능격차 축소가 목표.
3. 핵심 아이디어: BF 복호를 숨은 목적함수 `f(x)`의 gradient ascent(에너지 `-f(x)` 최소화)로 재해석.
4. 목적함수 `f(x)=Σ xj·yj + Σ_i Π_{j∈N(i)} xj` — 1항은 수신어와 상관(ML 상관복호), 2항은 bipolar 신드롬 합(코드워드 강제 penalty).
5. inversion function `Δk(GD)(x)=xk·yk + Σ_{i∈M(k)} Π_{j∈N(i)} xj` = `∂f/∂xk`에 `xk`를 곱한 값; `xk·∂f/∂xk<0`이면 flip.
6. single GDBF: 매 iteration `Δk` 최소 위치 1비트 flip (좌표하강). WBF/MWBF는 목적함수 `f̃`의 특수형으로 통합 해석됨.
7. multi GDBF: 임계 `θ`(음수) 미만 `Δk` 비트를 한꺼번에 flip(큰 step)하다 목적함수가 감소하면 single 모드로 전환(step 축소).
8. escape process: 비-코드워드 local max에 갇히면 임계 `θ2`(랜덤, `1.7+N(0,0.01)`)로 강제 하강 후 재상승 — 국소최대 탈출.
9. 결과: BER=1e-6에서 multi GDBF가 MWBF 대비 약 1.6dB 이득; escape 추가 시 1e-5에서 naive multi 대비 약 1.5dB, 평균 반복 25.6(SNR 4dB).
10. 한계: normalized min-sum 대비 여전히 큰 성능격차; HW 미설계, AWGN rate-0.5 regular 코드 시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBF 반복 복호 루프 | decoder.cpp LDPC_Decoder() | 반복 골격만 재사용; BF는 message-passing 아님 |
| inversion function Δk (yk+신드롬합) | 대응 없음 | min-sum CN/VN(CNU_Update_New_Mag) 대신 BF 고유 연산 신규 구현 |
| soft 채널값 yk / |yk| 활용 | channel.cpp Set_R_Offset(), Set_LLR_Th() | soft-read 값 활용 유사하나 양자화 재설계 필요 |
| escape process 랜덤 섭동(θ2) | random.cpp (XOR25 난수) | 난수원만 대응 |
| syndrome=0 조기종료 | partial_CRC.cpp / full_CRC.cpp | BF는 parity check로 종료, CRC와 상이 |

적용 가치: **낮음** — min-sum 대비 BER 열세·반복수 과다로, 이미 보유한 복호기보다 우위 없음.
