> 총평: SPA 디코더 양자화(채널 LLR + 메시지 bit-width) 설계 가이드로, zero 포함 비대칭 양자화기·bit-width별 penalty(3bit 0.25dB, 4bit 0.1dB, 5bit≈float)는 Prime ECC의 채널·LLR 양자화 설계에 참고 가치. 단 CNP는 SPA φ-LUT라 min-sum과 상이.

### B. 알고리즘 요약

1. 시스템: BPSK/AWGN, rate-1/2 부호(regular (3,6) blocklength-1200, irregular PEG blocklength-1008)를 fixed-point SPA로 복호할 때 양자화 효과를 분석.
2. 문제: 시뮬은 부동소수지만 실제 ASIC/FPGA는 fixed-point → 3/4/5-bit 저정밀도로 속도↑·자원↓ 하되 성능 손실을 최소화해야 함.
3. 채널 양자화: 채널 LLR `L(Vi)=2Yi/σ²`의 pdf(두 Gaussian 합)를 임계값 집합으로 비교. rate-1/2 capacity 달성 SNR을 최소화하는 최적 임계값(3-bit 비균일: `τ=±5.61,±3.17,±1.4,0 ·Es/No`).
4. 핵심 트릭(비대칭 양자화기): 균일 양자화 후 중점값을 `Ai`로 쓰되, 덧셈 시 두 메시지 합이 0이 될 수 있으므로 **메시지 집합에 0을 포함**해야 함 → 비대칭 임계값(`-3.5Δv..2.5Δv`)과 2's complement 사용.
5. VNP: 단순 가산기(2's complement)에 saturation. 메시지는 `bv` bit, 최소 `-(2^(bv-1))Δv`~최대 `(2^(bv-1)-1)Δv`.
6. CNP: `φ(x)=-log(tanh(x/2))` 와 `φ^-1`을 table lookup(입력 `bv`bit, 출력 `be`bit, step `Δe`)으로 구현, `φ(0)=(2^dc -1)Δe`로 처리, C개 여분 bit로 합 표현.
7. 최적화: exhaustive 대신 density evolution으로 (bv,be,Δv,Δe,C) 조합의 threshold를 계산해 최저 SNR threshold 테이블 선정. 성능이 테이블 엔트리에 민감(1단위 변경도 큰 penalty).
8. 파라미터: bv=3(be=4,ψ=1.429dB), bv=4(be=3~5,ψ≈1.23dB), bv=5(be=5~6,ψ≈1.01~1.04dB). Δv/Δe는 시뮬로 결정.
9. 결과: 3-bit 0.25dB, 4-bit <0.1dB, 5-bit ≈ 부동소수 penalty. regular(3,6)·irregular PEG 모두 유사한 손실.
10. 한계: 합성/gate count 없음, SPA φ-LUT 기반(min-sum 아님), degree-2 많은 irregular는 coarse 양자화 시 error floor 상승(scale 증가로 완화, future work).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 채널 LLR 양자화(비대칭 임계값, zero 포함) | channel.cpp Set_R_Offset(), Set_LLR_Th() | soft-read threshold/양자화 설계에 직접 참고 |
| 메시지 bit-width / LLR 테이블 선정 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 3/4/5-bit별 penalty·step size 설계 참고 |
| VNP 가산기 + saturation | decoder.cpp VN_Cal_HD() 등 (HD/2SD/3SD) | 메시지 양자화·포화 처리 참고 |
| CNP φ(x)/φ^-1(x) LUT (SPA) | decoder.cpp CNU_Update_New_Mag() | Prime ECC는 min-sum(φ-LUT 없음) → 알고리즘 상이, 직접 대응 없음 |
```

적용 가치: **중간** — 채널/메시지 양자화 bit-width 설계와 0을 포함하는 비대칭 양자화기 원칙은 Prime ECC의 channel.cpp·LLR 테이블에 참고 가치가 크나, CNP의 φ-LUT는 SPA 전용이라 min-sum 기반 Prime ECC엔 직접 이식되지 않음.
