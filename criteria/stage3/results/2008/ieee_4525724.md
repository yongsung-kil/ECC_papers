### B. 알고리즘 요약

1. 대상은 structured(QC) LDPC로, base matrix `Hb`(16×32)에 expand factor `z`로 circulant permutation `P`를 확장한 (10240, 5120), rate 1/2 부호.
2. 문제: 기존 Log-BP는 CNFU/VNFU 연산부하가 불균형해 critical path가 throughput을 제한.
3. 부호 특성: column degree irregular(2~16), row degree는 항상 8로 고정해 CNFU 복잡도 단순화, error floor 없음 주장.
4. 핵심 기법: 수정 Log-BP — CN update `Lm,n`과 VN update `Zm,n`에 `ψ(x)=log((1+e^-x)/(1-e^-x))` LUT를 나눠 배치해 두 phase 연산부하·지연을 균형화.
5. 균형화 덕에 CNFU와 VNFU가 서로 기다리지 않고 동시 갱신 가능(MUX 네트워크로 입력 선택) → 디코딩 가속.
6. HW arch: semi-parallel + double parallelism(32 CNFU/64 VNFU), 32 LRAM/CRAM, MemA/MemB dual-port에 node-to-node soft info(5bit) 저장, 두 codeword 파이프라인.
7. 구현 트릭: CNFU는 sign을 XOR tree, magnitude를 adder tree+LUT-C로 처리(대칭성으로 LUT 절반), pipeline latch로 critical path 단축.
8. odd shifting offset 충돌 해결: MEM_odd/MEM_even을 REG로 스위칭하는 memory partition/data switch 스킴(Fig.5)으로 클록당 2 soft info 접근.
9. 결과: 5bit(1 sign/2 int/2 frac) 고정소수점이 double precision과 근사 성능(AWGN/BPSK), Virtex-4 VLX80서 129.2MHz, 18 iter에 223Mbps(식 (4)).
10. 한계: rate 1/2·AWGN 시뮬만, ASIC gate count 없음, min-sum 아닌 full Log-BP.
