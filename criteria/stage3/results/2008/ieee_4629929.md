### B. 알고리즘 요약

1. 목표: DVB-S2/WiMax/WiFi 등 특정 구조에 묶인 decoder-aware 설계를 벗어나, **구조·비구조 무관 임의 LDPC**를 동일 HW로 복호하는 완전 프로그래머블 디코더.
2. 문제: 기존 부분병렬 디코더는 constituent matrix 크기 공약수 등 제약이 커 부호설계 자유도가 낮음; 완전 유연 시 extrinsic 메시지의 메모리 접근 충돌 발생.
3. 아키텍처: 병렬도 `p`개의 extrinsic 메모리뱅크 `EM_i`, `p`개 VFU/CFU, 두 permutation network `pi`/`pi^-1`(Benes)와 `2p`개 `p:1` MUX로 메시지를 노드연산기에 라우팅, 제어신호는 컴파일된 부호를 `HM`에서 선형 read.
4. CN 연산은 **normalized BP-based(정규화 min-sum)** `R = alpha * min|Q| * prod(sign)`, `alpha in (0,1]`(합성후 0.71875~0.90625, 4bit 표현) - 단 아키텍처는 근사에 비의존, 다른 근사로 교체 가능.
5. VN 연산 `Q = In + sum(R)`, 반복 두 half-iteration 교대.
6. **매핑 알고리즘**이 각 edge에 제어벡터 `m=(addr,pos,mem)` 배정 - 같은 degree의 `p`개 노드를 그룹핑, check node 그룹에서 `mem` 중복이 `<=dC` 되도록 교환하여 충돌없는 접근 보장.
7. 실제 평균병렬도 `pa = 0.5*(E/NV + E/NC) <= p`, 활성 FU 비율 `eact = pa/p` - 고rate 부호(C7/C8)에서 96%, 저rate에서 77~79%.
8. 반복당 클록수 `Titer ≈ 2E/pa`, 정보 throughput `Dinfo ≈ R*N*f*pa/(I*2*E)`.
9. 결과: XILINX Virtex4 XC4LX160에 VHDL 구현, `f>=73MHz`, 최대 `Nmax=73,728`/`Emax=524,288`, throughput 4~11 Mbps(15 iter). FPGA 고정소수점(8bit=1s+5i+2f) BER은 부동소수 BP 대비 0.1~0.15dB 손실.
10. 한계: FPGA라 ASIC 대비 throughput 1/3 이하, 정정성능 이득 없음(구현 검증용); 유연성이 목적이라 고정 QC-LDPC 최적화와는 지향이 반대.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 정규화 min-sum(normalized BP-based) CN 연산 (alpha 스케일) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum과 동일 계열 - 이미 보유(중복 기여) |
| VN update / 8bit 양자화 | `decoder.cpp` `VN_Cal_HD()`, `decoder.h` `Get_VNU_*` | 표준 VN 처리, 신규성 없음 |
| 유연 메모리뱅크 매핑 + Benes 라우팅(비구조 부호용) | 대응 없음 | Prime ECC는 QC 구조(z=32)로 라우팅이 고정·정규 - 임의부호 매핑이 불필요 |
| valid codeword 조기종료 | `partial_CRC.cpp`, `full_CRC.cpp` | 개념 유사, Prime ECC가 더 정교(Partial CRC 4점) |

적용 가치: **낮음** - 임의 부호 복호를 위한 유연 HW+매핑은 고정 고rate QC-LDPC NAND ECC에 불필요하고, 핵심 연산(정규화 min-sum)은 이미 보유.
