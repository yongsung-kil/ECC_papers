### B. 알고리즘 요약

1. 대상은 DVB-S2 위성방송 FEC의 LDPC 복호기; 부호는 QPSK~32APSK 변조별 rate 1/4~9/10, normal FECFRAME `N=64800`(rate 1/2 시 열 32,400·`q=90`).
2. 문제: 차세대 위성방송에 Shannon 한계 근접 성능 필요 → SPA는 `tanh` 연산 비용 큼, VLSI 친화 근사 필요.
3. 채널/모델: AWGN 채널, LLR `L(V)=log(P(V=0)/P(V=1))` 입력.
4. 핵심: SPA의 `Q(x)=-log(tanh(x/2))` 성질로 최소 `x`가 결과를 지배 → CNU를 `L(rij)=α·min_{i'∈j\i} |β_{i'j}|`(부호곱×최소크기)의 **min-sum**으로 근사.
5. 식(1)의 의미: 곱셈/tanh 없이 min+sign만으로 CN 메시지 계산 → 복잡도 절감·VLSI 친화.
6. 복호 6단계: 초기화(`L(qij)=yi`)→CNU→VNU(`L(qij)=L(ci)+ΣL(rj'i)`)→soft decision(`L(Qi)`)→hard decision→parity check(`H·cᵀ=0` 시 종료, 아니면 최대 iteration까지 반복).
7. 구현 디테일: DVB-S2의 360-단위 QC 구조를 그대로 활용, 한 사이클에 360 연산 병렬 처리(360 functional unit 매핑).
8. 최적화 절차: 별도 학습/최적화 없음(표준 알고리즘 직접 구현).
9. 결과: C 시뮬(64000 프레임, AWGN)에서 SNR-BER가 Shannon 근접; RTL 합성 gate count 3,957,390, 메모리 982,320 bit.
10. 한계: scaling factor·양자화 비트·iteration·throughput·공정노드 미명시, 신규 알고리즘 기여 없음(표준 min-sum 재현), 대형 프레임으로 메모리 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CNU (`α·min`, sign 곱) | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 중복 (Prime ECC가 이미 보유한 min-sum) |
| VNU / soft·hard decision | `decoder.cpp` `VN_Cal_HD()` 등 | 중복 (표준 VN 처리) |
| parity check 조기종료(`H·cᵀ=0`) | `full_CRC.cpp` / `LDPC_Decoder()` 루프 | 낮음 (Prime는 partial/full CRC 조기종료 사용) |
| DVB-S2 QC(360 lifting) H-matrix 로드 | `ecc_top.cpp` `Load_PCM()` | 낮음 (부호 상이, NAND 고rate와 다른 rate군) |

적용 가치: **낮음** — Prime ECC가 이미 구현한 표준 min-sum의 교과서적 재현으로, 새로 떼어 이식할 신규 기법이 없다.
