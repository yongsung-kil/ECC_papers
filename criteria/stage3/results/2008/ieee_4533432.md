### B. 알고리즘 요약

1. 시스템: partial-response(PR) 채널(자기기록 모델 `Pn(D)=(1-D)(1+D)^n`)용 RA 기반 LDPC, PR4(`1-D^2`)에 rate `3641/4096=0.89` 부호 설계.
2. 문제: LDPC로 PR 채널 복호 시 채널 memory(ISI)가 복호 복잡도를 키움 → trellis-matched convolutional code의 LDPC 대응물이 필요.
3. 모델: precoder `1/Pn(D) (mod 2)`가 PR 채널을 "binary sense"로 반전, 무잡음 수신열과 D-codeword 간 1:1 관계 `xk≡|yk/2|(mod2)`(식 1).
4. 핵심기법: RA 인코더를 수정해 precoding을 정보열까지 확장(전송열=E-codeword), 원 RA와 동일한 선형 인코딩 복잡도 유지.
5. graph-matched: 전송 E-codeword가 아닌 D-codeword를 부호어로 보는 별도 Tanner graph로 복호 → PR memory 서브그래프 제거로 H가 더 sparse·systematic, 임의 `n`에 동일 graph.
6. 복호: SPA(sum-product)를 그 graph(Fig.4)에 적용, PR4 수신 `yk∈{0,±2}`에서 APP `Pk=Pr(xk=1|yk)`를 식 5로 계산 → memoryless 채널처럼 단순.
7. 등가 구현: 체계적 LDPC 인코더 → precoder `1/Pn(D)` → bipolar → PR 채널(Fig.5), 인코딩·복호 모두 O(N) 선형.
8. 부호설계: 정보비트 repetition 3, pseudo-random permutation 후 24비트 그룹으로 패리티 생성(열가중치 ≤3), SPA 최대 5회 반복.
9. 결과: PR4서 BER=1e-5에 `Eb/N0=5dB`, precoding 없는 LDPC 및 TPC/SPC와 성능 사실상 동일하나 복호 복잡도가 더 낮음(장점은 복잡도).
10. 한계: 정수계수 PR 한정(실수 GPR엔 modulo-2 precoding 불가), 자기기록 특화, 성능 이득 없음, AWGN 시뮬뿐 HW·gate count 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RA(accumulate) 기반 선형 인코딩 | [encoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_encoder_4KB()` | dual-diagonal이 accumulate 계열로 유사하나 PR precoding 확장은 NAND에 불필요 |
| graph-matched precoding(PR 간섭 제거) | 대응 없음 | NAND는 PR/ISI 채널이 아니라 제거할 memory 없음 |
| SPA(full sum-product) 복호 | 대응 없음 | Prime ECC는 min-sum 근사 사용, full-tanh SPA 미사용 |
| 고rate(0.89) 부호 rate대 | [ecc_top.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | rate대는 NAND ECC급이나 구조가 RA로 QC-LDPC와 다름 |

적용 가치: **낮음** — 핵심 기여인 graph-matched precoding은 PR/ISI(자기기록) 채널의 memory 제거용인데 Prime ECC는 memoryless(AWGN/RBER) NAND 채널이라 트릭이 불필요하고, 복호는 min-sum 대비 full SPA로 대응이 약함. rate 0.89 RA 인코딩 구조 정도만 참고 가치.
