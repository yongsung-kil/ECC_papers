### B. 알고리즘 요약

1. 대상: 이진입력 DMC 상 일반 선형/LDPC/GLDPC 부호에 대한 LP(선형계획) 디코딩의 근사 반복 알고리즘 분석.
2. 문제: Feldman LP 디코딩은 ML 근접·정정 보장 있으나 범용 LP solver는 블록길이에 비선형(다항) 복잡도.
3. 기반: Vontobel-Koetter의 Gauss-Seidel형 coordinate-ascent 알고리즘(Problem-DS)을 새로운 스케줄링 하에 분석.
4. 평활화: `min(x)≈−(1/K)log Σe^{−Kx}`(log-sum-exp)로 dual 목적함수를 매끄러운 `DS(u)`로 근사, entropy 항 포함 primal `PS`.
5. 업데이트: 좌표별 `u_{k,j}` 갱신(식23), `v_{k,j}`(20,25)·`λ_{i,j}`(27) 사용, 최종 `λ̃`(31) → 경판정 `ĉ`(33).
6. 스케줄링: `k(t)=argmax_k max_l|λ_{k,l}−λ_k|`(29)로 노드 선택, 두 리스트(<ε₀, ≥ε₀)로 정렬 회피 → iteration당 상수 연산.
7. 복잡도(Thm1): 모든 차수 상수 유계면 O(N) iteration(각 상수 연산)으로 `P(λ̃)−P*≤δN` 만족 feasible vector 확보.
8. 수렴률: 개선된 bound(46)-(50), Lemma4로 `A₂lN/δ²` iteration; (3,6) regular BSC(q=0.07) 예시 K=5에서 ε≤1/6까지 33N, ε≤0.1까지 95N iteration.
9. GLDPC 확장(Thm2): 단일 parity check를 일반 constituent code로 대체, dual distance≥3(`Ψ̃` full rank) 가정 하 동일 O(N).
10. 정정능력(Thm3): 기존 LP 정정([3][4] 고정비율 오류정정)을 근사 LP로도 O(N)(또는 o(N log N)) 달성(vanishing BER 정의). 한계: 순수 이론, 상수 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| approximate LP 디코딩 알고리즘 | 대응 없음 | Prime은 min-sum message-passing, LP 디코딩 미지원 |
| Gauss-Seidel coordinate-ascent 반복 | `decoder.cpp` `LDPC_Decoder()` (개념상만) | 반복 복호 루프이나 알고리즘 완전 상이 |
| log-sum-exp/entropy 평활 목적함수 | 대응 없음 | Prime CN은 min-sum(min1/min2), LP 목적함수 없음 |
| BSC γi=±1 입력 | `channel.cpp` | 채널 모델 참조만 |
| GLDPC(일반 constituent code) 확장 | 대응 없음 | Prime은 SPC-based binary QC-LDPC 고정 |

적용 가치: 낮음 — LP 디코딩은 min-sum 기반 NAND HW와 패러다임이 다르고, 선형 복잡도라도 상수가 커(33N~95N iter) 실용성이 낮으며 HW/시뮬 검증도 없다.
