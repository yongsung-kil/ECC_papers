### B. 알고리즘 요약

1. 대상: WiMAX QC-LDPC(`N=2304, R=1/2, 30 iter`) partly-parallel 디코더(parallelism 96, 333Mbit/s@400MHz, 1.31mm2@65nm), soft-decision 6bit 양자화.
2. 문제: CMOS 미세화로 soft/timing transient 에러 증가, TMR·radiation hardening은 면적/전력 과다 -> 알고리즘 고유 내결함성을 활용한 저비용 신뢰성 설계 필요.
3. 모델: 각 RAM 비트가 MTBF 확률로 flip, 순열망은 SER/TER(예 `10^-4`)로 bit-flip/timing 에러 주입되는 transient 에러 모델.
4. 핵심기법: 메시지패싱의 내결함성 활용 — (a) 상위비트일수록 중요, (b) 오염된 LLR은 정정 대신 `puncture`(0으로 리셋 = 해당 edge 일시 제거)로 영향 최소화.
5. 의미: sign 비트가 hard decision을 결정해 가장 중요, magnitude 오류는 min-sum류(3-Min)가 흡수 -> 보호 대상을 sign/MSB로 국한해 면적 절감.
6. 확장: 블록별 차등 — Sum RAM은 정보 누적이라 MSB triplication+voting으로 정정, Channel RAM/순열망은 MSB/sign duplication 후 불일치 시 puncturing.
7. 구현 디테일: sign-magnitude 표현 채택(전력·보호 용이), 순열망 timing 에러는 duplicated sign을 매 클록 반전시켜 검출(Razor FF 미사용), controller는 triplication+2/3 voting.
8. 학습/최적화: 각 서브블록 "error sensitivity" 탐색으로 보호기법 선택(controller 강보호, message RAM 무보호).
9. 결과: RAM MTBF 2ms + 순열망 SER/TER `10^-4` 하에서도 무에러 대비 성능 거의 동일(순열망만 error-floor 소폭↑), 면적 +21%(전체 보호)/+6%(controller만), throughput·latency 불변.
10. 한계: 정정능력/속도 개선이 아닌 칩 transient 에러 내성 목적, 특정 WiMAX partly-parallel 아키텍처·3-Min 종속, 면적 오버헤드 발생.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-Min(min-sum 변형) CN 연산 | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal()` | min-sum 계열이라 대응은 되나 논문 핵심은 알고리즘이 아닌 HW 보호 |
| sign/MSB 선택 보호·puncturing(0 리셋) | 대응 없음 | Prime ECC엔 transient 에러 대비 신호 보호/puncturing 레이어 없음 |
| 6bit LLR 양자화(sign-magnitude) | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 양자화 표현 관점만 부분 관련(Prime은 HD/2SD/3SD) |
| 순열망/barrel shifter 라우팅 보호 | 대응 없음 | Prime ECC HW 모델(z=32, GT/HCU)과 다른 순열망 구조 |

적용 가치: **낮음** — 신호 중요도 기반 선택보호+puncturing은 재사용 가능한 HW 강건성 원리지만, 대상이 칩 transient 에러 내성이라 Prime ECC의 정정능력·throughput·면적효율 목표와 직교하고 면적을 늘린다.
