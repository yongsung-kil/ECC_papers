### B. 알고리즘 요약

1. 대상: IEEE 802.16e QC-LDPC, block size `z=96`, 예제 rate-1/2·length-2304, regular/irregular 겸용 목표(DVB-S2/11n/16e).
2. 문제: message passing 복호는 iteration마다 방대한 노드 연산 → 저전력 위해 총 연산수(=처리단위 복잡도×평균 iteration×iteration당 active node) 감소가 핵심.
3. 3요소 결합: 처리단위는 scaling min-sum(SMSA, scaling `α`), iteration수는 horizontal layered(약 50% 감소), active node는 forced convergence.
4. Horizontal vs vertical: min-sum은 horizontal layered에서만 layer 정보 갱신이 올바름(vertical은 이전 sub-iteration 미갱신 bit 정보로 min 연산이 깨짐) → horizontal 채택(메모리도 유리).
5. 핵심기법(H-LS-SMSA): total information `Qv`가 임계 `tv` 초과 시 해당 variable node를 sleep — 식(6) total 갱신, 식(5) check 갱신, 식(3)을 식(7)로 단순화(check RAM read 1회 절약).
6. error-floor 대책: `tv`가 낮으면 오판정으로 error floor 발생 → 주기 `iu`마다 sleeping node를 전부 wake-up해 메시지 갱신(`exec(tv,Qv,iu)` 로직).
7. 아키텍처: partially parallel, 96 instance, 추가 유닛은 variable node status vector(코드길이 길이 레지스터)·decision unit(comparator)·get bit information unit(MUX 1개 추가)로 저오버헤드.
8. 전력근거: 메모리 access가 전력의 95% → node sleep 시 check/channel RAM read/write 생략으로 약 75% RAM access 절감.
9. 결과: AWGN/BPSK, 최대 30 iteration, tv=10·iu=5에서 SMSA 대비 연산 최대 62.5% 절감, vertical Lazy Scheduling(SPA) 대비도 우수, BER 열화 미미.
10. 한계: 연산량·BER 시뮬만(합성·gate·throughput 없음), NAND soft-read 무관, 정정능력 향상 아닌 전력 절감.
