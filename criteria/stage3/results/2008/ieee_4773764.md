### B. 알고리즘 요약

1. 대상: 짧은 binary LDPC((3,6)-regular MacKay (504,252)), BPSK/AWGN, short cycle로 BP가 ML 대비 성능 열화.
2. 문제: BP-OSD는 ML gap을 줄이나 Gaussian elimination·정렬로 복잡도 높음 → ABP[1]가 OSD 대체(초기 BP 실패 후 selected VN들로 디코딩 트리 형성).
3. 모델: BP 오류를 Type I(잘못된 안정 수렴)/Type II(오실레이션 미수렴)/Type III로 분류, VN·edge의 부호변화 횟수를 flipping times `Fv_i^k`, `Fq_ij^k`로 정의.
4. 핵심기법: ABP는 selected VN의 채널 메시지를 포화값 `±S`로 고정하고 두 갈래로 분기, `J`층 트리(총 `J`개 VN 선택)로 재복호.
5. 관찰: Type I은 오실레이션이 시작돼야 수렴으로 전환, Type II는 오실레이션을 억제해야 수렴, 오실레이션 심한 VN은 오실레이션 심한 CN과 연결됨(관찰 1~3).
6. 제안 node selection(Algorithm 1): `k` iteration 후 flipping times `Fv_i^k` 최대인 VN을 선택 → 사이클의 오실레이션을 멈추는 데 가장 유효한 노드 지정.
7. 부가: SUC 기반 기존법은 stable check가 필요해 Type II(오실레이션)에 취약, 제안법은 flipping 통계만으로 동작하고 BP에 쉽게 추가.
8. PABP: 서로 다른 두 알고리즘 디코더를 병렬 결합해 각자의 정정능력을 합침 → best BP+OSD와 거의 동일 성능(더 나은 노드선택법이 존재함을 시사).
9. 결과: J=11에서 NABP J=11 대비 FER 1e-5에서 약 0.25dB 개선, Algorithm 1 J=10이 NABP J=11보다 우수(절반 복잡도), PABP J=11은 best BP+OSD와 일치.
10. 한계: 시뮬레이션만, HW/gate count 없음, 트리 route 수 `2×2^J−1`로 복잡도 지수증가, short code 특화.
