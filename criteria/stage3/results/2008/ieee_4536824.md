### B. 알고리즘 요약

1. 시스템: 무선 멀티미디어 전송에서 채널 상태별 code rate 적응을 위한 rate-compatible(RC) QC-LDPC, 단일 encoder/decoder 쌍으로 전 rate 처리.
2. 문제: 여러 rate마다 인코더/디코더 쌍을 두는 것은 복잡도상 비현실적, puncturing만으론 고rate에서 반복 디코더가 마비됨.
3. 부호 구성: `H(q,j,k)` — prime `q`, `q≥k≥j`인 하삼각형(lower-triangular) modified array code를 모체로 사용, full rank 보장, girth≥6(4-cycle 없음).
4. RC 스킴 핵심: 모체 PCM에 새 행 블록만 추가하면 저rate 부호(`c'H'^T=0 ⇒ c'H^T=0`이라 모체 디코더로 복호 가능), 하단 행 삭제는 parity puncturing과 등가로 고rate 부호 생성.
5. rate: `R = 1 - j/k` (모체), 행 추가/삭제로 `m*` 조정.
6. 인코더 구조: 생성행렬 SC(systematic-circulant) 형태 `G_QC=[I | P]`, 각 circulant 생성자 `g_{i,j}` 순환시프트로 parity section 산출.
7. HW 1(직렬): SRAA(shift-register-adder-accumulator) 회로 `m*`개로 parity section 병렬 산출, 정보 직렬 입력, `(n*-m*)p` 클록.
8. HW 2(병렬): feedback shift-register 기반으로 `m*p` 클록 또는 full-parallel 시 `p` 클록에 완료, RC 상황별 일부 SRAA/shift-register 비활성화로 소비전력 절감.
9. 복잡도: 직렬은 최저rate 부호로, 병렬은 모체 부호로 결정 — 예: 직렬 `2·m_max·p` FF, `m_max·p` AND/XOR (심볼릭, 공정·절대치 없음).
10. 한계: BER/합성/FPGA 검증 없이 게이트수 공식 분석뿐, 무선 가변rate 전용이라 NAND 고정rate와 무관.
