### B. 알고리즘 요약

1. 대상/채널: DSL(DMT sub-channel≈AWGN) 전송용 고rate array-code LDPC, binary, AWGN 가정.
2. 문제: LLR-BP(sum-product)의 `tanh` 연산이 HW LUT 비용이 커, 복잡도-성능 절충 복호가 필요.
3. LLR-BP: check node `L(rmni)=2·tanh⁻¹(∏ tanh(L/2))`, bit node는 LLR 덧셈합(식 1~3).
4. BP-based(=min-sum): check node를 `∏sgn(L)·min|L|`(식 4/5)로 근사 → 덧셈·비교만, `tanh` 제거, check당 두 amplitude만 저장.
5. APP-based: bit node에서 `L(qn)`을 모든 `L(qnmi)` 대신 사용 → bit node당 한 값만 저장(메모리↓), 단 상관 유발로 성능↓.
6. 정규화: `L2>L1`이므로 `L2`를 `α=E|L1|/E|L2| (>1)`로 나눠 BP-based/APP-based 성능 복원, 전 iteration·SNR 단일 α 사용.
7. 부호구성: array code — 소수 `p`, 정수 `j,k`로 순환치환행렬 `α` 블록의 H(식 8), `N=kp, K=(k-j)p`, 상삼각 `Hp`로 선형시간 인코딩.
8. 부호 예: (529,460)/(2209,2021)/(4489,4154)/(7921,7387), rate `0.87~0.93`.
9. 결과: normalized BP-based ≈ LLR-BP(−0.1dB), normalized APP-based 대비 +0.3dB; 고rate에서 BP-based가 성능-복잡도 최적, `1e-7`까지 error-floor 없음.
10. 한계: HW 미설계(게이트 없음), 시뮬만, 복호 알고리즘·부호 모두 기존 문헌 재사용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP-based 복호(min-sum: sign×min) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum과 동일 — 이미 보유, 신규성 없음 |
| normalized BP-based(scaling α) | `decoder.cpp` `CNU_Update_New_Mag()` | 정규화 factor는 프로파일상 미확인이라 min-sum magnitude에 적용 여지 있음 |
| LLR-BP(full tanh sum-product) | 대응 없음 | Prime ECC는 tanh SP 대신 min 근사 사용 |
| array-code 고rate QC-LDPC 구성 | `ecc_top.cpp` `Load_PCM()` | 고rate QC 구조 로드에 대응하나 부호 고정이라 재검증 부담 큼 |
| LLR 초기화/저장(bit-node 메모리 축소) | `decoder.cpp` `VN_Cal_HD()`, `ecc_data.h` `PARAM_LLR` | soft LLR 처리·저장에 대응 |

적용 가치: **낮음** — normalized min-sum scaling은 우리 min-sum에 소폭 적용 여지가 있으나, 대부분 이미 보유한 기법이고 정정능력/error-floor 개선이 없음.
