### B. 알고리즘 요약

1. 시스템: BEC(binary erasure channel)용 (n,k) rate-1/2 LDPC, 길이 512/1024/2048, 패킷 소거정정 대상.
2. 문제: 유한길이 LDPC의 IT 복호는 waterfall 열화와 stopping-set 기인 error-floor 존재 → 이상적 MDS 대비 성능 손실.
3. 핵심 아이디어: IT 복호 실패 시 잔여 최대 stopping set을 ML로 해소하는 hybrid IT/ML(HIML) 복호 — ML 성능을 저복잡도로 근사.
4. ML 복호는 BEC에서 소거비트 부분행렬 `H_K`의 Gaussian elimination으로 등가, 복잡도는 길이의 3승.
5. 효율화: 희소성 이용해 `H_K`를 근사 삼각화, GE는 삼각화 불가한 `reference bits`에만 적용 → reference bit 수가 관건.
6. 부호설계 지침 R1~R6: systematic, 저복잡도 인코딩, ML usage 최소화, reference bit 최소, ML threshold `ε*_ML ≈ 1-R`, 낮은 error-floor.
7. GeIRA 부호: 외부 LDGM + 내부 rate-1 RCC 직렬연접, `g(D)` 계수 수만큼 Hp에 all-1 대각선 → SIRA류 초경량 인코딩 유지하며 degree-2 VN 수 감소로 `dmin` 개선.
8. 구성: parity VN은 다중대각 구조로, systematic VN은 PEG 알고리즘으로 연결; CN degree 9(예), 근정칙 버전은 [4,8].
9. 결과: (2048,1024) 부호가 ε=0.450에서 CER 10^-5 도달(이상 MDS는 0.453), 근정칙 (512,256) `dmin=40`으로 error-floor를 CER~10^-20으로 추정, SIRA 대비 error-floor 우위.
10. 한계: BEC 전용, HW 미설계, 복호가 소거 GE 기반이라 AWGN/NAND 연판정 min-sum과 무관.
