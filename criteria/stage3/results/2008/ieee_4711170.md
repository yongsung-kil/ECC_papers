### B. 알고리즘 요약

1. 시스템: eIRA(extended IRA) = 선형시간 인코딩(zig-zag/staircase 패리티) 가능한 irregular LDPC, DVB-S2 채택, 부분병렬 복호 지향, AWGN.
2. 문제: 동일 degree 분포라도 H 구조에 따라 성능 상이 — 저중량 부호어와 near-codeword(=trapping set)가 error floor를 지배.
3. 모델: BP 복호가 near-codeword에 갇혀 부호어로 수렴 실패; 성능은 최소거리+가장 확률 높은 near-codeword로 예측.
4. 핵심 기법: 열 단위 trial-and-error 준랜덤 설계 — degree분포 제약 + 사이클 구조 제약(systematic/mixed girth `Gs`,`Gm`).
5. 핵심 제약: 신규 `MNDR/MRD`(Minimum Number of Distant Rows / Min Row Distance) — weight-2 near-codeword 무게를 `2+(MNDR-1)·MRD` 이상으로 보장, `D1,2`로 weight-1,2 부호어 억제.
6. 근거: systematic VN 최소차수를 4로↑ → 짧은 사이클 EMD↑, WEF 분석상 저차 VN이 저중량 이벤트 주원인이라 error floor 급감(수렴 임계는 소폭 악화).
7. 병렬화: systematic부를 `M×M` 순환천이 항등블록 매크로열 구조(QC)로 설계 → 메모리뱅크 충돌 없는 부분병렬 복호(`Π`,`Θ` 벡터로 라우팅).
8. 다능성: puncturing(`2Rc/(1+Rc)`)·extension(`Rc/(2-Rc)`)로 rate 가변, parallelism degree/shortening으로 length 가변 — 소수 mother code로 0.33~0.80 지원.
9. 결과: (603,301) 부호 FER=10^-6서 [25] 최고부호 대비 ~1dB↑, 최소거리 15, extension/puncturing 부호도 성능손실 미미(min dist 17/14).
10. 한계: HW 미합성(아키텍처 스케줄링만), BP 가정, 80 iteration, gatecount/throughput 미제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| eIRA H-matrix 준랜덤 설계(error-floor) | ecc_top.cpp Load_PCM() | error-floor-aware H 설계 원리를 Prime의 QC-LDPC PCM 개선에 참고 가능(재설계 필요) |
| near-codeword/trapping-set 회피 분석 | corner.cpp | Prime의 수렴실패(near-codeword) 추적과 개념 연결, 설계단 반영은 별개 |
| 순환천이 항등블록(QC) + dual-diagonal 패리티 | encoder.cpp LDPC_encoder_4KB() (dual-diagonal), ecc_data.h PCM_b | Prime의 QC base+circulant / dual-diagonal 인코딩과 구조 동일 계열 |
| 부분병렬 메모리뱅크/라우팅(Π,Θ) | 대응 없음 | Prime HW는 z=32 행병렬 고정 구조라 이 매핑 직접 대응 아님 |
```

적용 가치: **중간** — near-codeword 회피 기반 error-floor 설계 방법론(MNDR/MRD·min VN degree↑)은 Prime의 QC-LDPC error floor 개선에 개념적으로 유용하나, 고정 H-matrix 재설계·재검증 부담이 커 즉시 이식은 아님.
