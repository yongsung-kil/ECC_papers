### B. 알고리즘 요약

1. 대상: AWGN/BPSK 채널의 log-domain Sum-Product(SP) LDPC 디코더 유한정밀도 분석, 테스트 코드 (120,64) irregular(`dv=3` 고정, `dc=5` 24개/`dc=6` 40개 check node).
2. 문제: 워드길이 결정은 방대한 시뮬레이션이 필요 → 워드길이와 성능을 잇는 해석적 오차 모델 필요, 그 첫 단계.
3. 모델: `Qvc = Σ Rmv + λv`, `Rcv = Πsign(Qnc)·Φ(ΣΦ(Qnc))`, `Φ(x) = -log(tanh(|x|/2))`, 판정량 `Qv = Σ Rmv + λv`.
4. 기법 핵심: `Φ`를 `i` 정수비트 `f` 소수비트로 균일양자화(LUT)할 때 성능을 저하시키는 두 오차 메커니즘을 식별.
5. Roundoff 메커니즘: 양자화 오차 `e = q̂ - q`가 `[-2^-f, 2^-f]`, 정규분포(zero-mean)로 작은 값.
6. Saturation 메커니즘: `Qnc`가 커서 모든 `Φ̂(Qnc)=0`이면 합=0, `Φ(0)=∞`라 `Rcv`가 최대값 `S = 2^i - 2^-f`로 포화 → 큰 오차. 확률 `p = Πpn`.
7. 해석식: 첫 iteration에서 `Qv` 오차가 `L·S` 근처일 확률을 regular 코드 대상 (10)으로 유도, `p`는 (12)의 erf로 노이즈·`f` 종속.
8. 히스토그램 특징: 오차 히스토그램은 `Nlobes = 2·dv + 1`개 lobe, `±L·S` 위치, 2·S/3·S lobe는 Lobe1 분포의 컨볼루션으로 확인.
9. 결과: `i=5,f=1`이면 7개 lobe(큰 오차 존재), `f`를 2→4로 늘리면 큰 오차 lobe 소멸, `i` 줄이면 lobe 중앙 병합.
10. 한계: HW 미설계, 1 iteration·단일 소형 코드 실험, 완성된 예측 모델은 미제시(첫 단계라고 명시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| `Φ(x)=-log(tanh)` LUT 양자화(SP) | 대응 없음 | Prime ECC는 full-tanh SP가 아니라 min-sum(magnitude 양자화 테이블) 사용 |
| 메시지/LLR 워드길이 양자화 | `decoder.cpp` `VN_Cal_HD()`, `ecc_data.h` `PARAM_LLR` | 양자화 지점은 대응하나 오차 메커니즘 자체는 min-sum에 다르게 적용됨 |
| Roundoff/Saturation 오차 모델 | 대응 없음 | 코드 모듈이 아닌 해석적 분석 프레임워크 |

적용 가치: **낮음** — Sum-Product Φ-LUT에 특화된 양자화 오차 분석으로 min-sum 기반 Prime ECC의 Saturation 거동과 직접 대응되지 않고, 새 알고리즘·HW 없이 미완성 모델의 첫 단계에 그친다.
