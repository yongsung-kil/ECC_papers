### B. 알고리즘 요약

1. 우주선 command 링크처럼 짧은 부호·높은 coding gain·극저 undetected error rate가 동시에 필요한 시나리오 대상, (32,16) protograph LDPC(circulant permutation로 확장) 사용.
2. 문제: 짧은 LDPC의 반복복호는 유효 codeword를 못 찾으면 erasure로 오류검출하지만, 검출된 codeword의 오정정(undetected error) 비율 `Pu`가 너무 높음.
3. 모델: 길이 n 부호, AWGN에서 `x = A·c_i + N`의 soft 수신벡터. angle은 진폭 A 지식 불필요(BD 대비 장점).
4. 핵심기법 BA-I: ML 대신 반복복호를 쓰되, 반복 중 검출된 codeword `c_i`와 `x`의 유클리드 각도 `θ_i = cos^{-1}(<x,c_i>/(|x||c_i|))`가 최대 각도 `θ_d`를 넘으면 erase.
5. 의미: 각도 임계로 "받아들일 codeword" 영역을 좁혀 오정정(undetected)을 걸러냄 — 부호 재설계 없이 디코더에 조건 하나만 추가.
6. 확장: `θ_d`를 Eb/N0에 따라 가변(예: 저SNR 31°→고SNR 50°, Table I)해 Pu는 낮게 유지하면서 Pw threshold 상승을 최소화.
7. 임의 CN 처리(min*, min, scaled min, offset min)와 무관하게 적용 가능, 임계 판단은 종료 후 각도 비교(그림4 flow chart).
8. 최적 θ_d≈30°: Tx/비Tx codeword 각도 분포(그림7)가 교차하는 지점 — 참 codeword 기각과 오 codeword 기각의 균형.
9. 결과: (32,16), min*, Imax=50에서 θ_d=28°/31°일 때 최대 Pu를 수 자릿수 저감, 대가로 Pw·threshold SNR 상승.
10. 한계: HW 미설계, (32,16) 짧은 부호 시뮬만, 부호 길어지면 이득 감소(CRC로 대체 가능), Pw 악화 trade-off.
