### B. 알고리즘 요약

1. 시스템: IEEE 802.11n WLAN용 (1944,972) binary QC-LDPC(rate 0.5) 디코더 칩, TSMC 0.13um.
2. 문제: min-sum(MSA)/NMSA를 고정소수점으로 HW화 시 wordlength·CNU 비교지연·메모리 throughput이 면적/속도 병목.
3. 알고리즘: NMSA 채택, normalization factor `0.875`(shifter+subtractor로 곱), iteration 8회 고정.
4. 핵심기법1 (GC): 7-input CNU의 min 탐색을 2-stage로 분할(1단 pair-wise 지역최소, 2단 그룹간 최소, Signal Network 경유), 입력 `N`개면 stage 수 `log3N`으로 critical path 단축.
5. 핵심기법2 (DWA): 총 wordlength 5-bit 고정 하에 초기 4 iteration은 `(3,2)`(정수3/소수2), 이후 4 iteration은 `(4,1)`로 동적 전환 — `TS_5/TS_6` threshold를 `Sel`로 선택.
6. 의미: DWA로 고정 wordlength 대비 면적 25%↓ + 고정소수점 열화를 회복하는 coding gain 확보.
7. 핵심기법3 (DPS): base PCM의 sub-matrix를 memory bank로 매핑하되 3개 data(`{D1,D2,D3}`)를 1 entry(15-bit)로 패킹, shift값 `floor(s(i,j)/3)`, 나머지 0/1/2별 DPU_0/1/2로 정렬 → bank 수 유지하며 throughput `k=3`배.
8. 레이아웃: 110개 memory bank(24 single-port+86 two-port)를 11x10 2-D surrounding-type 배치로 routing/전력 절감.
9. 결과: coding gain 0.48dB/0.63dB (vs (4,3) MSA / (3,2) NMSA @BER 1e-6), gate 443.9K, 111.1MHz, throughput 250Mbps(>165Mbps 요구), power 76mW.
10. 한계: 802.11n 특정 부호·rate 0.5 전용, iteration 감소 없음, error-floor 미논의, 구식 0.13um.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMSA 디코딩 루프(정규화 0.875) | decoder.cpp LDPC_Decoder(), C2V_Cal() | 이미 Min-Sum 보유, 정규화 factor 적용 방식 참고 정도 |
| GC (CNU min1/min2 비교트리) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | CNU 비교 파이프라인 critical path 단축 아이디어 대응 |
| DWA (iteration별 wordlength/양자화) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx(), VN_Cal_HD() | iteration 구간별 정수/소수 bit 재배치 — 적응형 LLR 테이블과 연결 가능 |
| DPS (memory bank 3-패킹) | 대응 없음 | Prime ECC는 z=32 row 병렬/HCU 구조로 메모리 패킹 방식 상이 |
```

적용 가치: **중간** — 실측된 binary QC-LDPC min-sum HW 디코더로 DWA(iteration별 wordlength 재배치)와 GC(CNU 비교트리 critical-path 단축)가 이식 후보이나, NMSA/적응형 LLR/파이프라인은 이미 보유해 순증분은 제한적.
