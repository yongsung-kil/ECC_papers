### B. 알고리즘 요약

1. 대상: IEEE 802.16e(WiMax) QC-LDPC, rate `1/2`, block length `960`, base matrix `Hb`를 `z=40` circulant permutation으로 확장, BPSK/OFDM(1024 subcarrier).
2. 문제: LDPC의 (a) 이차 인코딩 복잡도, (b) 패리티행렬 저장 메모리, (c) 복호 CNU의 `Ψ(x)=-log tanh(x/2)` LUT 비용을 HW에서 줄이기.
3. 인코더: RU(Richardson-Urbanke) approximate-upper-triangular 분해로 선형시간 인코딩, `u1=-φ^{-1}(ET^{-1}A+C)s`, `u2=-T^{-1}(As+Bu1)`, φ를 항등행렬로 만들어 O(g^2) 제거 (XOR/AND만).
4. 디코더: 표준 BP message passing - bit->check `q_ji = Σ r + LLR(p)`, check->bit는 `sign 곱 × Ψ(Σ Ψ(|q|))`.
5. 핵심 기법: CNU의 비선형함수 `Ψ(x)=-ln tanh(x/2)`를 CORDIC(hyperbolic rotation/vector 모드, 가산+시프트만)로 계산 - LUT 제거.
6. CORDIC 식: `X_{i+1}=X_i-mσ2^{-i}Y_i`, `Y_{i+1}=Y_i+σ2^{-i}X_i`; rotation 모드로 cosh/sinh→tanh, vector 모드로 `Z_n≈½ln(w)` (x=w+1,y=w-1).
7. 구현: 7-stage pipelined CORDIC(7 iteration), 완전병렬 디코더(노드 수 = PE 수), interconnection network.
8. 부호 성질: QC 구조로 저장 메모리 1/z 축소, bit-node column permutation으로 stopping-set 기반 burst-erasure 정정 능력 최적화.
9. 결과: Verilog-HDL을 Synopsys DC 150nm TSMC로 합성, CNU당 LUT 대비 HW 50% 절감, CORDIC 최대 250MHz / CNU 90MHz; AWGN·Vehicular-A에서 burst-erasure(Lbe) 성능(Lbe=200에서 BER floor).
10. 한계: gate count 미기재, throughput 수치 없음, sum-product(tanh) 전제, WiMax rate-1/2 단일 코드.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CORDIC 기반 CNU (full-tanh Ψ 계산) | 대응 없음 | Prime ECC는 min-sum(min1/min2+sign)으로 tanh 자체를 회피 - CORDIC tanh 트릭 무의미 |
| 완전병렬 message-passing 디코더 루프 | [decoder.cpp](../../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | Prime ECC는 pipeline 6-stage + Graph Thinning 구조라 완전병렬 아키텍처와 상이 |
| RU 선형 인코딩(approx 삼각화) | [encoder.cpp](../../Prime_ECC_3.1_Claude/) `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코딩 - 인코딩 방식 상이 |
| QC 패리티행렬 로드 | [ecc_top.cpp](../../Prime_ECC_3.1_Claude/) `Load_PCM()` | QC 계열은 동일하나 WiMax rate-1/2 base matrix라 코드 자체는 무관 |

적용 가치: **낮음** - 핵심 기여인 CORDIC로 sum-product CNU의 tanh를 계산하는 트릭은 tanh를 쓰지 않는 min-sum 기반 Prime ECC엔 적용 대상이 없다. 완전병렬 아키텍처·RU 인코딩도 Prime ECC의 pipeline/GT/dual-diagonal 구조와 달라 직접 이식 가치가 낮다.
