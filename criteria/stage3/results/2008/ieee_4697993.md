### B. 알고리즘 요약

1. 시스템: 블록길이 1024 irregular LDPC, normalized min-sum(스케일 0.8), 최대 100 iter, AWGN.
2. 문제: 유한길이 error floor는 최소거리 codeword가 아닌 소형 stopping set에서 기인; 최소 stopping set 크기 결정은 NP-hard이고 PEG의 큰 girth만으로는 유한길이 성능 보장 안 됨.
3. 이론(stopping set): stopping set `S`는 모든 이웃 check가 `S`에 2회 이상 연결된 변수집합. Theorem1: degree-1 없는 그래프에서 `S`의 모든 VN은 반드시 어떤 cycle 위에 있음.
4. 핵심 정의: length-2d cycle이 d개의 degree-2 VN으로만 구성되면 minimal stopping set이자 weight-d codeword. 일반적으로 minimal stopping set은 여러 ex-cycle의 결합.
5. ex-cycle/EMD: extrinsic message degree(EMD)=집합에 단일 연결된 constraint node 수. ex-cycle=EMD≠0인 cycle, `(d,e)`-cycle=length-2d·EMD e.
6. 핵심 아이디어: 동일 EMD인 두 ex-cycle이 결합해 EMD 0(=stopping set)을 이뤄 low-weight codeword를 만듦 → 이를 PEG 구성 중 회피.
7. 추정기: `ψ(σ,θ)=τ(σ,θ)+χ(ε(σ,θ)−1)·(min_σ τ(σ)+ε(σ,θ))`로 cycle-pair가 포함될 stopping set 크기를 근사(최적성 미주장).
8. 개선 PEG(Alg1): 새 cycle이 불가피할 때 p개 후보 중, 기록된 bad cycle 집합 `Θ`와의 worst-case 추정 stopping size를 최소화하는 edge 선택, 짧은 cycle은 `Θ`에 추가.
9. 결과: DE로 얻은 `λ(x)=448x^2+384x^3+192x^8` 분포 사용. 길이 1024에서 PEG-ACE 대비 error-floor 개선(더 큰 최소 stopping size), waterfall 성능 저하 없음(Fig.2 BER/FER).
10. 한계: HW 미설계·시뮬만, 2-cycle 군집만 고려, 구성 중 EMD가 미완성 edge로 부정확, 추정기 최적성 미보장, irregular non-QC 산출.
