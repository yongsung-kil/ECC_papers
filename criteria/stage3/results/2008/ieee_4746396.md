### B. 알고리즘 요약

1. 대상: 짧은 길이(N=1008, 2016) irregular LDPC, rate 1/2, AWGN + BP 디코더(max 50 iter), error floor 저감 부호설계.
2. 문제: 유한길이 LDPC의 error floor는 small stopping set·bad trapping set(low-weight/near codeword)이 주원인; 기존 ACE[5]는 개별 cycle만 보아 결합형 small stopping set을 놓침.
3. 정의: stopping set S = 모든 이웃 CN이 S와 2개 이상 연결; trapping set `[w,u]` = hamming weight w, syndrome weight u (w,u 작을수록 위험).
4. 기존 ACE: 길이 `2l` cycle의 ACE = `Σ(di−2)`. low-ACE short cycle이 small stopping set 기여하나, high-degree node 결합 케이스 누락(Fig.1: ACE=4 cycle 3개→size-7 stopping set=weight-7 codeword).
5. 신규 metric ACSE: 공통 root node로 연결된 cycle set이 singly-connected CN으로 뻗는 extrinsic-message edge 수 → cycle set의 타 노드 연결성 측정(Fig.3: L1=4, L2=2, L3=0).
6. ACSE 추정: root v에서 depth-d subgraph 전개, k=1부터 최소 ACE cycle을 추가하며 ACSE 최소가 되도록 cycle set `Lk` 확장, v의 모든 edge 포함까지 반복.
7. PEG-ACSE 구성: DE 최적화 degree 분포(λ1 max deg 15, λ2 max deg 10)로 VN degree 배정·정렬, degree-2 node는 zigzag로 큰 cycle 보장, 나머지는 PEG 연결 후 최소 ACSE<ζ면 재연결(Fail loop).
8. 임계 ζ: ζ≤3이면 small stopping/trapping set 제거 부족, ζ≥6이면 만족 행렬 구성 곤란; N=1008에 ζ=4, N=2016에 ζ=5, depth d=11.
9. 결과: PEG-ACSE는 waterfall(SNR<2dB)에서 PEG와 유사, high SNR(>2dB)에서 error floor 더 낮음; 최고 known code인 MacKay(1008,504) 대비 high SNR 우수(Fig.4,5).
10. 한계: irregular 비-QC 부호 생성(HW 구조화 안 됨), 일반 AWGN 통신 맥락(NAND 아님), HW 미설계·시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG-ACSE 부호구성(H-matrix 설계) | ecc_top.cpp Load_PCM() + H-matrix | Prime ECC는 고정 QC-LDPC 로드, 비-QC 구성법 이식 부담 큼 |
| error-floor(stopping/trapping set) 저감 개념 | 대응 없음 | Prime ECC엔 error-floor 전용 구성/검증 모듈 없음, 개념 참고 |
| BP 디코더 성능평가 | decoder.cpp LDPC_Decoder() | 평가용, Prime ECC는 min-sum(BP 근사) |
```

적용 가치: 중간 — error-floor(stopping/trapping set) 저감은 NAND 핵심 관심사이나, PEG-ACSE는 irregular 비-QC 부호를 생성하여 Prime ECC의 고정 QC-LDPC(circulant) H-matrix에 직접 적용 불가, ACSE metric·설계철학의 참고 가치.
