> 총평: 자기기록용 inner LDPC + outer BCH 연접에서 iteration 진동을 이용한 trial-and-error 복호로 error-floor를 회피 — NAND 연접 ECC에 개념 이식은 가능하나 외부 BCH 계층 추가가 전제.

### B. 알고리즘 요약

1. 시스템: perpendicular 자기기록 read channel, 4 kB 섹터, inner rate-19/20 `(4380,4163)` QC-LDPC(column weight 3) + outer `(33304, 32850, 28)` binary BCH, 전체 rate-15/16.
2. 문제: LDPC-only는 error floor와 매우 낮은 SER(`10^-11`) 검증이 FPGA로도 비현실적(4kB 섹터에 약 80일 추정).
3. 모델: perpendicular 채널 + media jitter(노이즈의 90%) + AWGN(10%), SOVA 검출, min-sum LDPC 복호(max iteration 8).
4. 핵심 기법: LDPC 복호 실패 시 iteration마다 bit-error 수가 진동한다는 관찰 → 각 iteration의 hard-decision을 외부 BCH 디코더에 순차 입력(trial-and-error)하여 BCH 성공 시 종료.
5. 핵심 식: overall SER를 식(1)로 `P_f`(inner LDPC 실패확률)와 최소 bit-error 확률의 곱 형태로 추정 — 저 SER를 직접 시뮬 대신 두 확률 추정으로 대체.
6. 확장: BCH 오정정(mis-correction) 대비 error detection 병행, `m`개 실패 시 실패별로 서로 다른 iteration 출력 조합 탐색.
7. 구현 디테일: 내부 LDPC 메시지 3-bit, SOVA soft output 3-bit, path metric 9-bit, 채널/AWGN/equalizer 출력 6-bit 양자화.
8. 결과: 연접 trial-and-error로 SER `10^-11` 도달, RS 대비 약 0.7 dB coding gain 손해지만 실리콘 비용은 LDPC-only/RS-only보다 낮음(65nm ASIC).
9. 한계: 자기기록 채널 특화, turbo equalization 미적용으로 RS 대비 gain gap 존재, gate count 등 수치는 표에만 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum LDPC 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유(min-sum) — 중복 기여 |
| iteration별 hard-decision 캡처(trial-and-error) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 각 iteration 출력 저장 훅 추가 필요 |
| 외부 BCH 연접 / 재시도 종료 | 대응 없음 (`full_CRC.cpp` 조기종료와 유사 위치) | Prime ECC엔 BCH 외부부호 없음 — 시스템 레벨 확장 |
| inner QC-LDPC H-matrix 로드 | `ecc_top.cpp` `Load_PCM()` | inner QC-LDPC 로드 대응 |
| 3-bit 메시지 soft 양자화 | `channel.cpp` `Set_LLR_Th()` | 양자화 폭 대응 |

적용 가치: 중간 — iteration 진동 이용 + 외부부호 재시도라는 개념은 이식 가능하나, Prime ECC에 BCH 외부 계층이 없어 decoder 단독 port가 아니라 시스템 레벨 확장이 필요.
