### B. 알고리즘 요약

1. 시스템: Wyner-Ziv/Slepian-Wolf 기반 분산비디오코딩(DVC); 소스=DCT 계수 bitplane, 디코더에 side information `Y` 존재, syndrome `s=Hx` 전송.
2. 문제: 상관잡음이 시퀀스마다 크게 변해 fine-granular 압축률(rate-compatible)이 필요하나, 기존 방식은 저rate sub-code를 split해 그래프 최적화가 어려움.
3. 핵심기법: rate=1 base code(H 6×6 예시)를 만든 뒤 두 check-node를 **병합**(H 두 행의 합)해 압축률을 점증적으로 높임.
4. rate-compatibility 확보: 병합쌍마다 2-degree syndrome node를 넣고 puncturing(`s0,s5` 등)해, 미수신 syndrome에 연결된 c-node를 디코더가 병합.
5. 제약: base/sub-code 모두 check-node degree 분포가 최대 2종·집중(concentrated)이어야 성능 유지.
6. accumulator 구조(repeat-accumulate 계열)로 syndrome node 연결, 전송순서를 puncturing period `Δ` 내 알고리즘으로 결정.
7. 그래프 컨디셔닝: ACE(approximate cycle EMD)로 stopping set 크기 증가, 2-degree v-node만의 4-cycle 금지, high-degree v-node를 주기 배치해 error burst 대응.
8. degree 분포 `λ(x),ρ(x)`는 [3]에서 채택, ACE 파라미터 `(dACE,ηACE)=(13,7)`.
9. 결과: node splitting(DISCOVER LDPC) 대비 WZ rate 최대 8%, turbo 대비 최대 19.2% 절감(Hall Monitor).
10. 한계: HW 미설계, QCIF 2개 시퀀스 시뮬 한정, 소스코딩 특화.
