### B. 알고리즘 요약

1. 대상: 이진 소거채널(BEC)에서 임의 LDPC 부호(예 MacKay `(504,252)`/`(1008,504)`, PEG, Ramanujan-Margulis `(4896,2474)`)의 최소 stopping set 크기 `s_min` 계산.
2. 문제: BEC 반복복호 성능은 최소 크기 stopping set이 지배하나, 최소 stopping set 탐색은 NP-hard 추정 조합문제로 exhaustive search 필요.
3. 핵심 관찰: stopping set(모든 이웃 check가 S에 2회 이상 연결)은 sparse PCM에서 저중량 codeword와 연관됨 (row weight 1이 없는 부분행렬 `H_S`).
4. 분해: 행 인덱스를 홀수-weight 집합 `I`와 나머지 `Ī`로 분리, `Ī` 부분행렬 `X`에 대해 `X x_S^T = 0` → SS 벡터가 `X`의 codeword가 됨.
5. 알고리즘: `H` 열을 랜덤 치환→`ρ`개 행 랜덤 선택으로 `X`(m-ρ행)/`Y`(ρ행) 분할→`X`에서 Stern 알고리즘(파라미터 `p,l`)으로 저중량 codeword 생성→`Y_S`에 weight-1 행 없으면 stopping set 확정, `s_min` 갱신, `r_max`회 반복.
6. 확률 분석: 홀수-weight 행 `I_odd`가 `I`에 포함될 확률 `θ_{t,ρ}`(식14)와 Stern이 weight `s` codeword를 찾을 확률 `π`(식15)로 발견확률 `τ̄_{s,ρ,p,l}` 유도.
7. Error probability: `r_max`회 후 `s_min` 오판 확률 `ε_{s_min}=(1-τ̄)^{r_max}`(식30)로 결과 신뢰도 정량화.
8. 복잡도/work factor: Stern 지배로 `N_{ρ,p,l}`(식31), stopping set 1개 생성 work factor `W=N/τ̄`(식32), 최적 `ρ,p,l`는 work factor 최소화로 선택.
9. 결과: C1 `s_min=16`, C3 `s_min=26`(기존 [8]의 28을 26으로 정정), 기존 subspace 대비 error probability 대폭↓(예 C3 0.972→3.0×10⁻¹⁴), SSE와 일치.
10. 한계: BEC stopping set 전용(soft/AWGN 아님), 소프트웨어 탐색으로 HW 무관, 소형 부호에도 수백 CPU시간(C3 3085h) 소요.
