### B. 알고리즘 요약

1. 대상: CMMB(중국 모바일 방송) LDPC 인코더, code rate 1/2·3/4, codeword length 9216 bit, H는 cyclic-shift 패턴이나 QC-LDPC는 아님(High-Structured).
2. 문제: LDPC의 G 행렬은 dense(행당 nonzero >1000)라 직접 인코딩은 길이 제곱 복잡도; CMMB는 비-QC라 QC용 LFSR 인코딩(효율적) 사용 불가.
3. 부호 특성: 출력이 non-systematic(정보/패리티 혼합), COL_ORDER 매핑(식(1))으로 재배열 → 역변환으로 systematic 형태 `H'*Csys=0` 복원.
4. 핵심 기법: `H'`를 `[Hp,Hs]`로 분할, GF(2)에서 `Hp*p_bit = Hs*s_bit`(식(3))을 sparse LU 분해 `L*U=P*Hp`로 해결.
5. 인코딩 절차: (1) `z=Hs_new*s_bit`, (2) forward substitution `L*y=z`, (3) backward substitution `U*p=y` → p_bit 산출 후 COL_ORDER로 재배열.
6. 최적화: partial pivoting heuristic으로 L·U를 최대한 sparse하게(1/2 rate: L 28621/U 21676 nonzero; 3/4 rate: L 8721/U 4747) → 메모리 대폭 절감. LU는 오프라인 SW 1회 수행.
7. HW 구조: 5단 pipeline + 각 단 Pingpong buffer로 throughput 개선, 정보비트도 각 단 버퍼링해 출력 정합.
8. 메모리: Hs_new·L·U·IDX를 4개 M-RAM(32K×16b)에 분산 저장, nonzero column index+new-row flag만 기록(all-zero 행은 illegal index 6으로 표기), code rate 전환 시 재로드.
9. 결과: Altera Stratix II EP2S90, Verilog, 합성·라우팅 완료, 최대 201.49MHz, 인코딩률 32.44Mbps(1/2)·67.16Mbps(3/4), ALUT 581(0.8%)·Memory 2.0Mb(45%).
10. 한계: 인코더만·복호 성능 무관, CMMB 비-QC 특화, FPGA 검증(ASIC 아님), QC-LDPC엔 미적용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LU 기반 parity 계산 인코딩 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC는 dual-diagonal QC 인코딩이라 LU 방식 불필요/대응 없음 |
| sparse H matrix 저장/로드 | ecc_top.cpp Load_PCM() | sparse 저장 아이디어 참고 가능하나 QC는 base+shift로 이미 압축 |
| 비-QC High-Structured 부호구조 | 대응 없음 | Prime ECC는 QC-LDPC 고정 |
```

적용 가치: **낮음** — 비-QC 부호 전용 LU 인코더로, dual-diagonal QC 인코딩을 이미 보유한 Prime ECC(QC-LDPC)에는 불필요하고 broadcast·FPGA에 특화됨.
