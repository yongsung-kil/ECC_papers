### B. 알고리즘 요약

1. 대상은 IEEE 802.11n QC-LDPC(AA-LDPC), `rate 1/2·2/3·3/4·5/6`, sub-block 27/54/81, `Nb=24`(size81 기준 N=1944); AWGN 채널.
2. 문제: layered(horizontal) 디코더는 수렴은 빠르나 Channel RAM·RAM1을 매 클록 read/write해 메모리 전력이 전체의 절반 이상을 차지.
3. 배경: layered 스케줄은 posterior를 layer마다 갱신해 평균 iteration을 최대 `50%↓`, min-sum/two-output 근사로 CN 출력 저장을 2개로 축소.
4. 기법1(memory bypassing): 연속 두 layer가 같은 column에 non-null이면(overlap) Add-array 결과를 Channel RAM에 쓰지 않고 다음 layer로 직접 전달.
5. 핵심: 802.11n H는 overlap column이 많아 Channel RAM 접근을 `57%~82%` 절감 가능(디코딩 순서 최적화가 관건).
6. 구현: overlap 최대화·idle cycle 최소화 위해 RAM1의 read/write 순서를 de-couple, update 순서용 ROM으로 SISO index 보정, mux·파이프 레지스터 추가.
7. 기법2(thresholding): 중간 soft magnitude가 임계 `T` 이상이면 magnitude를 RAM1에 안 쓰고 1-bit `S`만 저장(2개 묶어 1비트), 갱신 시 최대 표현값 사용.
8. 파라미터: bit-width 6bit(sign1+mag5), `T=21`을 시뮬로 결정, SNR 높을수록 saturate 값 많아 RAM1 접근 `5%~37%↓`.
9. 결과(TSMC 0.18um, 200MHz, 1.8V, 합성): Channel RAM 전력 `53%~72%↓`, RAM1 `11%~27%↓`, 총 디코더 전력 `11%~24%↓`, 성능 열화 경미.
10. 한계: throughput/gatecount 미제시(전력 중심), thresholding은 정정성능 소폭 희생, 기법이 layered 메모리 구조에 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered 스케줄 + column-overlap memory bypass | `decoder.cpp` `LDPC_Decoder()` | 스케줄·메모리 접근 루프 - Prime ECC dual-update와 정합 필요 |
| min-sum 2-output SISO CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min1/min2 2개 magnitude 저장은 이미 유사 보유 |
| magnitude thresholding(soft 값 clip) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 큰 magnitude clip으로 메시지 메모리 접근 축소 아이디어 반영 가능 |
| cyclic shifter / Channel RAM 라우팅 | 대응 없음 | z=32 HW 모델 대응 영역이나 §6에 대응 함수명 없음 |

적용 가치: 중간 — NAND 컨트롤러도 저전력이 중요해 magnitude clip·메모리 접근 축소 아이디어는 유용하나, 핵심 memory-bypass가 horizontal layered 스케줄과 특정 H overlap에 종속되어 Prime ECC(dual-update, z=32) 구조에 그대로 이식하기는 어렵다.
