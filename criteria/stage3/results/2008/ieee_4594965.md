> 총평: Min-Sum의 sign 변동 메시지를 erasure하는 단순·강력한 self-correction으로 Sum-Product급 성능 + 저복잡도 + 빠른 수렴 - Prime ECC min-sum 디코더에 직접 이식 가치 높음.

### B. 알고리즘 요약

1. 시스템: 일반 binary LDPC(rate 1/2, AWGN/QPSK) message-passing 복호. 최적화 irregular LDPC(2304, 8064bit) 및 IEEE-802.16e QC-LDPC(2304bit)로 검증.
2. 문제: Min-Sum은 CN 메시지 과대추정으로 Sum-Product 대비 성능 손실 - 근본 원인은 CN 메시지의 symmetric Gaussian 분포 상실(과대추정 자체가 아님).
3. 모델: AWGN(`σ2`), a priori LLR `γn`, VN->CN `αm,n`, CN->VN `βm,n`.
4. 핵심기법: Self-Corrected Min-Sum - CN 처리는 그대로 두고 VN 처리만 수정. `αtmp = γ̃n - βm,n` 계산 후 이전 iteration `αm,n`과 부호 비교.
5. 핵심식/의미: `sgn(αtmp)=sgn(αm,n)`이면 갱신, 다르면 `αm,n=0`(erasure). 즉 부호 변동하는 unreliable 메시지 삭제로 computation tree "cleaning".
6. 확장/분석: SCMS는 unerased 메시지의 sub-tree에 대한 Min-Sum과 동치(Proposition 2), 이로써 CN 메시지가 symmetric Gaussian 분포 회복.
7. 이론 분석: Gaussian approximation으로 (`P^l`,`E^l`) joint recurrence `(P^{l+1},E^{l+1})=φ(P^l,E^l)` 유도 - erasure 확률 `E^l` 별도 추적.
8. 학습: 없음(휴리스틱 erasure 규칙, noise variance 추정 불필요).
9. 결과: Sum-Product 대비 gap이 short(2304bit) 0.05dB / long(8064bit) 0.1dB, short 코드 고SNR(error-floor 영역)에선 Sum-Product 초과, Normalized Min-Sum이 보이는 error-floor 없음, Min-Sum보다 빠른 수렴.
10. 한계: HW 미구현(gate count 없음), erasure 판정 위해 이전 메시지 저장 필요, fixed-point는 내부 메시지 6-bit(step 0.25) 사용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Self-Corrected VN 처리(부호 변동 erasure) | `decoder.cpp` `VN_Cal_HD()`(및 2SD/3SD), `LDPC_Decoder()` 이터레이션 루프 | VN update에 이전 메시지 부호 비교 + erasure(0 set) 삽입으로 직접 이식 가능 - 가장 쉬운 레이어 |
| CN min-sum 연산(변경 없음) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | self-correction은 CN 근사 보정 아님 - CN 처리 그대로 유지 |
| 이전 iteration VN 메시지(부호) 저장 | 대응 없음 | erasure 판정용 per-edge 이전 부호 저장 로직/메모리 신규 필요 |
| LLR / 양자화 | `ecc_data.h` `PARAM_LLR`, `channel.cpp` `Set_LLR_Th()` | 저해상도(soft-2~3bit) 양자화 하 이득 재확인 필요 |

적용 가치: **높음** - Min-Sum 성능을 Sum-Product급으로 끌어올리면서 저복잡도·noise 추정 무관을 유지하는 기법으로 Prime ECC의 min-sum VN 루프에 직접 이식 가능(§3 보유 기법에 없는 신규 개선), 다만 이전 메시지 저장 메모리 비용과 저해상도 양자화 이득 검토 필요.
