### B. 알고리즘 요약

1. 대상은 turbo와 LDPC 장점을 결합한 turbo Gallager codes(TGC), 이진소거채널(BEC)에서 `n=1536, k=512`(rate 1/3).
2. 문제: 소거채널에서 TGC를 어떤 반복 디코더(turbo/LUT/BP)로 풀어야 성능·복잡도 균형이 좋은지 불명확.
3. 모델: BEC의 소거확률 `ε`, 반복마다 소거 수 `nei` 감소, 트렐리스 상태 수 `2^M`(M=인코더 메모리).
4. turbo 디코더: MAP 기반 branch/forward/backward metric + soft decision, 연산량 `NoT(k)=Σ(4k+74k·2^M)`으로 M에 지수 증가.
5. LUT 디코더: 소거채널에서 α·β 비영 값이 동일하다는 성질을 이용, 트렐리스 전이를 이진벡터로 표현해 lookup으로 대체 → 동일 BCJR 성능에 복잡도만 감소.
6. 간이 BP 디코더: 행에 소거가 하나뿐인 방정식만 계산, 복잡도 `NoBP`는 M과 무관하고 반복수에만 비례.
7. 하이브리드: 반복 디코더 뒤에 ML 성능의 "In-Place" 가우스 소거(열치환 없는 행렬역산)를 붙여 잔여 소거(stopping set) 해결.
8. 각 연산을 "등가 덧셈 수"로 환산해 디코더별 복잡도를 정량 비교(Table I~V).
9. 결과: BCJR계열이 소거채널에서 BP보다 수렴 우수, 저메모리 TGC(15/13)가 최고 균형, 제약길이 증가로 stopping set 깨져 error floor 하강.
10. 한계: 소거채널·turbo 계열 전용, HW 미설계, 시뮬레이션만.

| 소거채널 간이 BP | [decoder.cpp](../../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 소거채널 BP는 AWGN/RBER min-sum과 채널·연산 모두 달라 이식 불가 |
| In-Place ML 행렬역산 하이브리드 | 대응 없음 | Prime ECC 조기종료(partial/full CRC)와 성격 다름, ML 후처리 미보유이나 소거채널 전용 |
