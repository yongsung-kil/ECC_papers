### B. 알고리즘 요약

1. 대상: 임의 구조의 binary LDPC 디코더 HW, 예제는 regular 코드 `H`=48×96(가중치 8/4), 102×204(가중치 10/5), rate 0.5.
2. 문제: message-passing 디코더의 지배적 병목은 단순 처리유닛을 잇는 **인터커넥션 네트워크 복잡도**.
3. 기법1 GC(Grouped Calculations): 각 처리쌍(PP)에 Q-CPU(bit→check)와 R-CPU(check→bit)를 두고, `H`의 행 계산을 그룹으로 묶어 PP에 매핑, 그룹 간 통신 최소화.
4. 그룹핑: `H` 각 행을 노드로, 같은 열에 1이 겹치는 행쌍을 가중 에지로 연결한 그래프 `G`를 만들고, 최소 총가중치 컷으로 분할 → 그룹 간 누락 R 메시지 최소화.
5. 대가: 그룹 간 버스를 없애면 일부 R 메시지 폐기 → 성능 저하, iteration 증가로 완화. 소요시간 `TN = T/N + TR`.
6. 기법2 MB(Multi-Bus): 공용 버스 `b`개마다 Q/R CPU `b`개씩, 재스케줄링으로 프로세서 완전 활용, 성능 저하 없음. 실행시간 `1/n`, 총면적 `O(n^2)`.
7. 기법3 GCMB: GC의 각 PP를 MB 그룹으로 대체, 지연 `T = Tinit/(Ng+Nb)`. 면적은 그룹수 `Ng`에 선형(인터커넥션 추가 없음)이라 MB 단독보다 유리.
8. 메모리: 총 메시지 저장량은 버스 수와 무관하게 일정, 프로세서당은 `1/n`.
9. 결과: 2그룹 GC + 3~4 iteration이 full-interconnect 2~3 iteration과 유사 BER(소형 코드 SNR-BER 곡선), 정량 게이트/throughput 없음.
10. 한계: HW 합성/면적 수치 없음, 소형 regular 코드 시뮬만, QC 구조 미활용.
