### B. 알고리즘 요약

1. 시스템: partly-parallel QC-LDPC 디코더(CNFU `j`개 + VNFU `k`개 + PCFU), min-sum(MS) + 8-bit uniform LLR 양자화, AWGN/BPSK. 대표 구현: (3,6) regular, circulant `m=256` → 길이 1536, rate 0.5.
2. 문제: MP 복호의 intra-/inter-iteration 데이터 종속 때문에 CNFU와 VNFU가 교대로만 동작 → 하드웨어 이용률(HUE)·throughput 저하. 기존 OMP[10]는 모든 QC 코드에 적용 불가·최적 아님.
3. 모델: 각 CNFU는 block-row를, 각 VNFU는 block-column을 cyclic 순서로 메모리 순회. throughput gain `G = 2L/(L+Σ(αi+βi))`, `α`=intra-, `β`=inter-iteration 대기시간.
4. 핵심기법: 데이터 종속만으로 최소 intra-/inter 대기시간이 iteration-불변이고 서로 같음(`w`)을 증명(Lemma 1). 각 VNFU가 reference block row를 독립 선택 → `w = wG ≤ w[10]`.
5. 핵심식: `G_max = 2L/(L+ w·M)` 형태, 상한은 `G≈2`(HUE 100%)이며 `w ≤ ⌊(m)/2⌋` 조건에서만 상한 달성.
6. 확장: 최소 대기시간 `w`를 얻는 저복잡도 exhaustive 탐색(O(m^j))과 iteration마다 시작주소를 `w`만큼 shift하는 addressing scheme 제시.
7. 구현 트릭: 6-stage pipeline(step당 2 clock)로 memory read/write 병목 제거, PCFU를 인접 두 block-row에 대해 pipeline해 PCFU·hard-decision 메모리를 절반으로 축소.
8. 최적화: 없음(학습형 아님). 대신 coset 기반 QC 구성으로 `w`를 상한 조건 이내로 만드는 코드 설계.
9. 결과: FPGA(Xilinx Virtex-II xc2v6000)에서 non-OMP 대비 실측 throughput gain ≈1.69(오버헤드 포함), 이상적 ≈2, HUE 100%, BER 열화 없음, 메모리는 data-interleaving 대비 절반.
10. 한계: 정정능력·iteration 개선 없음(throughput 전용), 특정 CNFU/VNFU cyclic 아키텍처 종속, ASIC gate count·절대 throughput은 표에만 있어 본문 미기재.
