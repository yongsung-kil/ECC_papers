### B. 알고리즘 요약

1. 대상: 2차원 Projective Geometry(PG) 기반 binary regular LDPC, mother code `len 1057, rate 0.769`, reconfigurable로 `len 255(r0.686)`,`len 4095(r0.822)`.
2. 문제: LDPC 디코더 VLSI 구현의 핵심 병목인 memory/address clash와 interconnect routing을 회피하며 여러 길이·rate로 재구성 가능한 아키텍처 필요.
3. 모델: PG 코드는 `n=2^2s+2^s+1`, row/col weight `2^s+1`, 임의 두 열 교집합 1개(girth-4 없음), cyclic(incident vector shift)이라 shift-register 인코딩·majority-logic 복호(MLD) 가능.
4. 핵심기법: 3개 아키텍처 — 1개 fully-parallel(그래프 직접 인스턴스화) + 2개 semi-parallel(각 7 또는 151 PE, 1057의 인수).
5. 핵심식: location pointer `LPV=[C-mod(C,7)]/7`, `LPC=[I-mod(I,7)]/7` (식1,2; 151 버전은 mod 51) — bank selector=mod값으로 memory clash 없이 Tanner graph 연결 생성.
6. 확장: 동일 아키텍처로 shortened(무한 reliability)·punctured(50% reliability=LLR 0) 버전 복호, BP와 MLD/weighted-MLD 겸용(고SNR 시 BP→MLD 전환으로 전력 절감).
7. 구현: bank selector+location pointer register, permutation network은 logarithmic barrel shifter(단수=log2(bank수))로 구현, PE는 serial 처리로 pipeline 가능.
8. 학습/최적화: 없음(고정 구조 설계).
9. 결과: 세 아키텍처 설계 제시 — memory clash 없음, 재구성 가능. BER/throughput/gate-count 등 정량 결과·합성 없음(성능 이슈는 범위 밖 명시).
10. 한계: 설계 단계만(합성·측정 없음), PG(finite-geometry) 특정 부호족, memory-mapping식이 예제 코드 종속.
