> 총평: LDPC convolutional(무선용) 부호의 프로그래머블 multi-core VLSI 디코더 합성 논문으로, OMS는 기보유 min-sum과 중복이고 code 패밀리·아키텍처 패러다임이 Prime ECC와 상이.

### B. 알고리즘 요약

1. 대상: 시변 LDPC convolutional 부호(LDPCCC)와 tail-biting 버전(TB-LDPCCC, QC 대칭), 예시 `(ms=127,J=3,K=5)`, `R=0.4`, block length `L=3200/6400`, AWGN/BPSK.
2. 문제: 차세대 통신은 고throughput+capacity근접+유연성+저전력 동시 요구, BP의 데이터 전송이 병목-LDPCCC 그래프의 locality/regularity로 전송 복잡도 저감 가능.
3. 부호: TB-LDPCCC는 zero-tail rate-loss를 제거, mother LDPCCC의 free distance와 block length에 의존하는 최소거리, semi-random LDPC 대비 유사 성능·저복잡도.
4. 병렬화: horizontal(같은 type 노드 SIMD 병렬, `NΓ=64` FU) + vertical(다른 type 노드 multi-core 병렬, `NΛ=2`), 상보적·확장가능하게 결합.
5. 알고리즘: Offset Min-Sum(OMS) 사용-CN 갱신 식`(4)` `sign·max(min|m|-β,0)`, VN 갱신 식`(5)` 누산 후 자기기여 제거, offset `β` 프로그래머블.
6. 아키텍처: DMEM 2뱅크(vector misalignment 회피), ROM(base/offset/shift 파라미터), AGU, barrel-shifter+2x2 crossbar 라우팅(QC 대칭 활용, 3 pipeline stage), STA+VLIW.
7. 프로그래머블: CN/VN degree, block length, iteration 수, offset β, 처리 순서까지 런타임 조정-다양한 부호 지원(LLR buffer 16값 -> degree 16까지).
8. 양자화: 5-bit 고정소수 OMS(`β=0.11`) vs float BP -> 약 0.2dB 손실로 HW 복잡도 대폭 절감.
9. 합성: 0.13um CMOS, 200MHz, 코어 약 354KGates(top 1152KGates, 메모리 32KB inst/40KB data 포함), throughput 약 100Mbit/s@10iter, 전력 370mW.
10. 한계: convolutional/무선 타깃, non-symmetric 스케줄(J,K가 NΛ 배수 아님)로 코어 부하 불균형, 실측 없이 합성 추정, throughput NAND ECC엔 낮음.
