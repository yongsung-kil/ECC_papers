### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 채널 + LDPC FEC + Hybrid ARQ 재전송. rate 0.5 코드 3종(`96.3.963`, `PEGirReg252x504`, `816.3.174`).
2. 문제: RB-HARQ는 SISO(BP) 디코더의 비트 신뢰도로 재전송 비트를 고르는데, LP 디코딩은 확률적 신뢰도 척도가 없어 HARQ와 결합이 어렵다.
3. 모델: ML 디코딩을 정수 선형계획(ILP)으로 표현(`min yᵀx`), 정수 제약을 `[0,1]` 완화한 LP relaxation으로 복호.
4. 핵심 기법1: LP 해가 정수면 provably ML 코드워드(Feldman). 이 integrality를 신뢰도 대용으로 사용.
5. 식 의미: parity check `j`의 local codeword indicator `q_jl`이 1이면 해당 check의 모든 비트가 정수 → 신뢰.
6. 핵심 기법2(재전송 기준1): 정수 indicator `q_jl`에 참여하지 않는 비트만 재전송(PCHF).
7. 재전송 기준2: LP 해의 비트변수 `x_k`가 비정수(fractional)인 비트만 재전송(HF) — 실용상 더 유리.
8. 통계 근거: 저 SNR(-7dB)에서도 비트변수 63~75%가 정수이고 그중 ~70%가 정답, 고 SNR서 100%로 상승.
9. 결과: fractional 비트 재전송(HF)이 전블록 재전송(HA) 대비 throughput 약 1.5~2dB 이득. indicator 기준(PCHF)은 두 전략 중간.
10. 한계: HW 미설계, 시뮬만, LP 복잡도 문제 미해결, irregular 코드는 정수 indicator가 극소라 indicator 기준 무효.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP relaxation 디코딩 | 대응 없음 | Prime ECC는 min-sum message passing, LP와 완전 다른 패러다임 |
| HARQ 재전송 비트 선택 | 대응 없음 | NAND ECC에는 ARQ/재전송 개념 부재 |
| LLR soft input | [channel.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | soft-read LLR만 개념적 접점, 나머지 무관 |
