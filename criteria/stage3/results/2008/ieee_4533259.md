### B. 알고리즘 요약

1. 시스템: 두 RSC 성분부호(`Ca`,`Cb`)의 병렬연접 turbo code, m-ary(rate `m/(m+1)`) 또는 이진, rate 1/3~7/8, BEC(소실확률 `ε`).
2. 문제: iterative turbo decoding(TD)은 소실집합이 stopping set을 포함하면 실패 → ML 복호 필요.
3. 모델: BEC LLR은 소실=0, 수신=±∞만 → BCJR/dual MAP에서 상태·전이가 allowed/forbidden로만 진화(부호 오류 발생 불가).
4. 핵심기법1(CSI): TD 정지 후 각 성분 subcode를 span하는 기저 `ia`,`ib`를 SISO 복호기로 찾고, 잔여 소실비트를 기저의 선형결합 `ua = Wa·ia + za`로 표현.
5. 식 의미: 두 부호 제약을 합쳐 `W·i = z` (식 9) 선형계 구성 → full rank면 유일 turbo codeword가 교집합으로 결정(=ML).
6. 핵심기법2(dual trellis): m-ary 부호는 dual reciprocal 부호 trellis에서 처리, allowed 상태/전이 수를 dual 변수(`nA`,`nB`)로 계산(Property 1,2)해 기저 탐색.
7. 구현 트릭: 두 기저가 최대한 많은 비트를 공유하도록 선택→미지수 최소화; 오차사건 그룹 분할로 BCJR을 짧은 trellis 구간에만 실행해 평균 복잡도 대폭 감소.
8. 비교(BM): [6]의 LDPC용 sparse `Hy=0` gaussian elimination ML 복호를 turbo용으로 이식, reference 변수 비율 α=1~7%; CSI와 계산시간 비교(Table I).
9. 결과: gaussian elimination은 블록 <10^4에서 병목 아님; m-ary TC가 random coding bound를 WER 10^-6까지 하회, 평균 등가 turbo iteration <35 (Table II).
10. 한계: BEC 전용·turbo code 전용, HW 미설계, error floor는 `dmin`(MLA)로 하한 결정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CSI/BM ML 복호 (gaussian elimination) | 대응 없음 | Prime ECC는 min-sum iterative message passing, ML/gaussian elimination 없음 |
| dual trellis BCJR/MAP (turbo) | 대응 없음 | LDPC decoder(`decoder.cpp`)와 부호·복호 구조 무관 |
| BEC 소실채널 모델 | 대응 없음 | `channel.cpp`는 AWGN/RBER/fixed-error, 소실채널 아님 |

적용 가치: **낮음** - turbo code·BEC·ML(gaussian elimination) 조합이 NAND binary QC-LDPC min-sum과 완전 이질적이라 이식 대상 아님.
