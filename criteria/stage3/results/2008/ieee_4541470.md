### B. 알고리즘 요약

1. 대상: WiMAX(802.16e) irregular QC-LDPC 6개 code group, base matrix `Hbm`(mb×nb)를 z×z permutation으로 확장, rate 1/2~5/6, length 576·2304.
2. 문제: layered decoding(LDA)은 인접 block row가 겹치면(overlap) serial function unit 지연 때문에 파이프라인에서 late-update 발생 → 성능 손실.
3. 배경: Rovini의 delta-based LDA(DLDA)는 (3) 대신 차분 `ΔRmj=Rmj(i)-Rmj(i-1)`, `qj=qj+ΔRmj`(식10-11)로 latency 제약을 제거, 임의 QC-LDPC 적용 가능.
4. 핵심기법: DLDA + 저복잡 offset Min-Sum(OMS, offset β=0.5)을 WiMAX 부호에 적용하고, base model matrix의 row/column 처리순서를 재배열해 late-update를 최소화.
5. late-update 정의: pipeline depth `PD`, row weight `dc`에 대해 update latency `LAT=dc+PD-1`; 다음 layer가 `D<LAT`에 bit-sum을 요청하면 update가 늦어 손실.
6. Row reordering: check weight 오름차순 그룹화 후 인접 row 간 overlap count(OC)의 누적(COC)이 최소가 되는 순서 선택.
7. Column reordering: 연속 3개 row(R1,R2,R3) 관찰, R1에서 즉시 갱신 가능한 열집합 `SA`를 R1 앞·R2 뒤에 배치(C-1 규칙, 식13-17)해 late-update 저감.
8. 결과지표: rate-5/6, PD=8에서 원본 50/80 late-update → 재배열 후 16/80. 대부분 부호에서 late/total 비율 45%→~20%.
9. 성능: rate-1/2 length-2304, PD=8에서 이상적 LDA 대비 손실 0.13dB(H1)→0.03dB(H2), avg iteration도 이상 LDA에 근접. BP 대비 1.5× 적은 평균 iteration.
10. 한계: floating-point 시뮬만, 실제 HW 합성/gate/throughput 없음, NAND soft-read 무관, 성능 자체 향상이 아닌 파이프라인 손실 회복.
