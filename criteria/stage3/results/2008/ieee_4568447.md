> 총평: LDPC convolutional(SC-LDPC) 부호의 pipeline 디코더 구현·termination·인코더 논문으로, 고정 binary QC-LDPC block인 Prime ECC와 코드 패밀리가 달라 직접 이식 곤란.

### B. 알고리즘 요약

1. 대상: rate `R=b/c` (ms,J,K) regular LDPC convolutional 부호(=SC-LDPC), 주로 `R=1/2` (b=1,c=2), `(3,6)`, syndrome former memory `ms=513~4093`, AWGN/BPSK, sum-product 복호.
2. 문제: LDPC-CC의 pipeline 디코더는 초기 복호 지연 `I·(ms+1)`과 저장량 `(J+1)·I·νs`가 커서 실용성 저하.
3. 인코딩: syndrome former 실현과 partial syndrome former 실현 제안-후자는 `(c-b)·ms` 메모리로 termination을 간단히 처리.
4. termination: partial syndrome `pL`로부터 tail을 선형식 `(20)`으로 계산, 주기 부호면 동일 회로 재사용 -> 다양한 frame 길이에 단일 부호 사용.
5. on-demand VN activation: CN이 필요로 할 때만 VN 활성화해 최신 정보를 즉시 전파 -> 동일 복잡도로 고정 iteration 시 BER 개선(고 memory 부호에서 0.05~0.10dB@고SNR).
6. stopping rule: 각 processor가 syndrome 영벡터 지속(P회) 확인 시 sleep -> 평균 iteration 대폭 감소(예: `(512,3,6)` I=100에서 Iav 64@1.0dB -> 16@1.25dB), 성능손실 없음.
7. compact 디코더: processor 이격 `S`를 줄여 영역 겹침 허용, `S>ms/2`면 저장·초기지연을 거의 절반으로 감소, 성능손실 미미.
8. block vs conv 비교: 동일 processor 복잡도 시 conv이 block 대비 ~0.6dB 우수, 동일 지연/메모리 시 성능 대등.
9. 부호 관찰: `ms`가 크면 syndrome former 밀도가 큰 `(5,10)`이 `(3,6)`보다 우수(block과 반대 경향, threshold 포화 효과).
10. 한계: 전 과정이 convolutional/sliding-window 구조 종속, 실 HW 합성·gate count 없음(시뮬만), soft 양자화 미논의.

| on-demand VN activation 스케줄 | [decoder.cpp](Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` 이터레이션 루프 | informed/serial 스케줄 아이디어 참고 가능하나 window 구조 종속 |
| stopping rule (syndrome 기반 조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념 중복-Prime ECC는 CRC 기반 이미 보유 |
| partial syndrome former / termination 인코더 | `encoder.cpp` `LDPC_encoder_4KB()` | 인코딩 구조 상이(dual-diagonal vs syndrome former), 직접 이식 곤란 |
