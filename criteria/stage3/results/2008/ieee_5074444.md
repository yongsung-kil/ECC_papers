### B. 알고리즘 요약

1. 대상: (6,32) regular (2048,1723) 10GBASE-T LDPC 부호, AWGN+BPSK, 최대 15 iteration, MinSum normalized 기반.
2. 문제: 기존 SPA/MinSum은 row-col 프로세서 간 global 통신으로 interconnect 복잡도·메모리 대역폭이 커 고throughput HW가 어려움.
3. 배경: Split-Row는 row 처리를 2개 이상 near-independent 파티션으로 분할해 통신을 줄이나, 파티션이 상대 파티션의 최소값을 몰라 0.4–0.7dB BER 손실 발생.
4. 핵심 기법: 각 파티션이 자기 local min이 threshold `T`보다 작으면 인접 파티션에 1비트 `Threshold_en` 신호를 보내 min 과대추정을 보정.
5. 조건 분기: local `Min1<T`면 자기 min1/min2로 갱신, `Min1≥T`이나 이웃 threshold 신호 assert면 `T`값으로 갱신, 둘 다 아니면 local min 사용.
6. 파티션 간 배선은 sign 2개 + threshold 2개뿐이라 상호연결 이득 유지.
7. threshold `T`는 SNR/iteration 무관하게 코드별 고정값 사용 가능(본 코드 최적 T≈0.1–0.2, 정규화 인자 S=0.35).
8. Column 처리는 MinSum/Split-Row와 동일.
9. 결과: Split-Row 대비 0.15–0.2dB coding gain, 완전 MinSum normalized 대비 0.15dB 이내; row processor 게이트 296→317, 면적 766→805(65nm CMOS Synopsys DC).
10. 한계: 완전 MinSum 대비 여전히 손실, threshold 이론 최적은 SNR 의존(실사용은 고정), critical path에 mux 1단 추가.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Split-Row Threshold 복호 루프 | decoder.cpp LDPC_Decoder() | min-sum 이터레이션에 파티션 threshold 로직 삽입 가능 |
| 파티션 내 min1/min2 계산 + threshold 보정 | decoder.cpp CNU_Update_New_Mag() | CN min-sum magnitude 계산 지점에 threshold 분기 대응 |
| sign bit 파티션 간 전달 | decoder.cpp C2V_Cal_New_Sgn() | 파티션 분할 시 sign 전역 계산 대응 |
| 상호연결/row 분할 축소 | GT.cpp Make_GT_HW() | 목적 유사(edge/배선 축소)이나 방식 상이(GT는 MUX edge 선택) |
| 파티션간 threshold 신호선 | 대응 없음 | Prime은 row 분할 미채택 |
```

적용 가치: **중간** — binary LDPC min-sum의 상호연결 축소 실HW 기법으로 NAND 고rate QC-LDPC 디코더에 참고할 만하나, Prime이 이미 GT/HCU로 상호연결을 줄이고 있고 Split-Row는 완전 MinSum 대비 BER 손실이 남아 직접 이득은 제한적이다.
