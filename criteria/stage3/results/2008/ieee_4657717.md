> 총평: DTMB용 고속 병렬 QC-LDPC 인코더(PIPO) — binary QC-LDPC라 형식은 맞으나 인코더 전용·G-matrix 기반이라 dual-diagonal을 이미 보유한 NAND ECC엔 이식 가치 낮음.

### B. 알고리즘 요약

1. 대상은 binary QC-LDPC로 `Hqc`가 `b×b` circulant permutation/zero 행렬 블록(`c×t`)으로 구성, DTMB 코드는 `b=127`, `t=59`, `c=35/23/11`, block length 7493, rate 0.4/0.6/0.8(P4/P6/P8).
2. 문제: LDPC 인코딩은 복호보다 비싸고, 기존 SRAA 기반 SIPO/PISO 인코더는 클록 수가 정보/패리티 길이에 비례해 수천 클록 소요 → 고속 인코딩 난제.
3. 인코딩은 systematic-circulant 생성행렬 `Gqc`(dense circulant `Gi,j` + identity)로 `Cqc = a·Gqc`.
4. 핵심 기법 PIPO: 정보비트를 `b`폭 병렬 입력, 패리티도 병렬 출력. `Pj = a1·G1,j + a2·G2,j + ... `를 각 스텝마다 `ai`와 `Gi,j`의 generator를 병렬 투입해 누적.
5. 패리티 `Pj`(b비트)는 `t-c` 클록에 생성, 전체 패리티는 `c(t-c)` 클록에 완료 → SIPO의 `b(t-c)` 대비 대폭 단축(b가 t,t-c보다 큼).
6. 구현 트릭: 입력폭 `w`가 `b`와 같을 필요 없음(예 `w=⌈b/8⌉`), 출력폭도 가변 → 속도-복잡도 유연 trade-off, 서로 다른 클록 적용 가능.
7. 자원 비교(`c=t-c=b` 가정): PIPO는 레지스터 `b`개(SIPO `2b²`, PISO `b²`), AND/XOR는 세 구조 모두 `b²` 수준 → 레지스터가 b제곱이 아닌 b에 선형.
8. DTMB 아키텍처: S/P 변환→PIPO 인코더→패리티 버퍼→P/S. 생성행렬 ROM 단 1개(127×2196)로 다중 rate를 주소로 선택.
9. 결과: P4 기준 SIPO(12 SRAA) 9144 클록 대비 PIPO 840 클록(~10배), Stratix II에서 fmax>250MHz, 잠재 throughput >1Gbps, LC combinational은 SIPO의 1/2~1/3.
10. 한계: 순수 인코더(복호/정정 무관), FPGA 자원(LC/레지스터) 비교뿐 절대 ASIC gatecount 없음, 방송 QC-LDPC 대상.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 병렬 인코딩(PIPO) | encoder.cpp LDPC_encoder_4KB() | Prime ECC는 dual-diagonal(sparse H) 인코딩 — G-matrix 병렬 MAC 방식과 접근 상이 |
| 생성/부호 PCM 처리 | encoder.cpp Generate_PCM_encoding() | Prime ECC는 부호용 PCM_b_enc(sparse) 사용, dense generator ROM 불필요 |

```

적용 가치: 낮음 — encoder 전용이며 dense generator-matrix ROM 기반이라, sparse dual-diagonal 인코딩을 이미 보유한 고rate NAND QC-LDPC(Prime ECC)에 실질 이점이 적다.
