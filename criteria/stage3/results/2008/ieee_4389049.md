> 총평: HW친화 binary QC protograph 구성법이나 rate·정칙성이 NAND 고rate QC-LDPC와 맞지 않고 성능은 벤치마크와 동등 수준 — 이미 QC-LDPC를 보유한 Prime ECC에 새 이득 적음.

### B. 알고리즘 요약

1. 시스템: binary LDPC(GF(2)), 정칙 `γ=3`, rate `R=1-M/N` 0.4~0.8, `N`=200~3060, BPSK/AWGN·Rayleigh, SPA 최대 `I=100`반복.
2. 문제: 유사랜덤(MacKay/PEG) 부호는 성능 우수하나 HW 서술 복잡·인코딩 복잡도 준2차 → 성능-복잡도 절충 구조부호 필요.
3. 모델: base protograph를 `J`회 복제 후 엣지 순열로 derived graph 생성(Thorpe protograph), QC 구조 부여.
4. 핵심기법: 순열행렬 `Pq`(항등행렬 순환시프트)로 Vandermonde형 블록행렬을 만들어 base protograph PCM `Hb`(크기 `γq×ρq`) 구성, `Nb=ρ×q`, `R≥1-γ/ρ`.
5. 의미: `q`가 `ρ`보다 크고 소수 조건 → 같은 행/열에 순열행렬 중복 없음 → Tanner graph girth≥6 보장.
6. 핵심기법2: 수정 PEG 알고리즘 — (제약1) derived graph가 base protograph 구조 유지, (제약2) 직전 선택 체크노드에 최근접 선택으로 QC성 유지.
7. 구현 이점: QC 인코더는 길이 `K` 시프트레지스터로 선형 인코딩(`r(N-K)` 이진연산), 순환 PCM의 메모리 시프트로 노드간 메시지 주소 지정(MAG·배선 감소).
8. 최적화: 허용 체크노드 집합 크기를 이항계수 `C(N,γ)`에서 `C(Jγ,γ)`로 축소.
9. 결과: (504,252)·(1008,504)에서 MacKay 대비 BER 10^-6에서 약 0.2dB 이득, 비정형 PEG 대비 0.06dB 손실 — 동등하거나 근소 우위.
10. 한계: HW 미설계(복잡도 지표 비교뿐), 고rate·초단블록에서 구성 제약 미충족 시 성능 열화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VM 기반 QC protograph PCM 구성 | ecc_top.cpp Load_PCM() | 우리는 고정 고rate QC-LDPC PCM_b — 부호 전면 교체·재검증 부담 |
| QC 선형 시프트레지스터 인코딩 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | 우리는 dual-diagonal 인코딩, 구성법 상이 |
| 반병렬 protograph 복호 구조 | decoder.cpp LDPC_Decoder() | 우리는 z=32 행병렬 min-sum, 구조 개념만 유사 |
| 수정 PEG girth 최대화(오프라인 설계도구) | 대응 없음 | 런타임 코드베이스에 부호 설계툴 없음 |
```

적용 가치: **낮음** — binary QC-LDPC라는 큰 틀은 맞으나 rate(0.4~0.8)·정칙 γ=3가 NAND 고rate 요건과 어긋나고, 인코딩/반병렬 복호 이점은 Prime ECC가 이미 보유한다.
