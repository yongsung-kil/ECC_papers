### B. 알고리즘 요약

1. 대상: 소프트 LDPC 컨볼루셔널 부호(LDPC-CC) 디코더 IP, QC 패리티행렬 기반, R=1/2·2/3·3/4, 메모리길이 M=192~1536, iteration T=10/15/20.
2. 문제: LDPC-CC는 블록부호보다 인코더/디코더 구현 쉬우나, 일반 LDPC-CC 행렬은 저장·주소계산 비용 크고 처리량 병목.
3. 구성: `(M,J,K)` 행렬을 `J×J` 항등블록 정방행렬 `G`→각 블록 cyclic shift `Iab`→대각 절단·평행사변형 `L`→rate만큼 반복·행 인터레이스→대각 반복으로 무한 확장(Fig.1,2).
4. QC 이점1: 전체 H를 소수 정수로 기술 → 디코더는 파라미터만 저장, `K(J-1)`개 수만으로 전 주소 산출(vs `MK`개).
5. QC 이점2: 수직 주기가 수평 주기의 R배 → 모든 프로세서가 항상 동일 주소 공유(단일 address control).
6. 디코더: min-sum(가산·비교만), LLR을 `K` dual-port SRAM(P,Q)+`K/J` single-port SRAM(L)에 저장, in-place(write주소=직전 read주소).
7. CN HW: sign 전체 XOR 후 입력별 1회 XOR로 부호 산출, min-except-self는 comparator 공유(J=3,R=1/2 시 24→12개, 50% 절감), horizontal 출력에 0.75 scale(가산 1회).
8. 검증: RTL vs 고정소수 C모델, ASIC flow 전/후 layout gate-level sim, 코드커버리지 95%+, AMBA AHB VIP, FPGA/ARM 보드 emulation.
9. 결과(정성): iteration·메모리길이 증가 시 BER 개선, 수직스텝 3→1 cycle(vs [6] 67% 절감), [3] 대비 성능↑·복잡도↓(Fig.9). 수치는 그림에만 존재.
10. 한계: 컨볼루셔널 부호(스트리밍)로 NAND 페이지 block ECC 부적합, gate count/throughput 본문 수치 없음, 소프트 IP(합성까지, tape-out 아님).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum(comparator 공유, sign XOR 통합) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 min1/min2+sign XOR 보유, 대체로 중복 |
| 0.75 scale factor(normalized min-sum) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | scaling 개념 이미 사용, 참고 수준 |
| LLR SRAM 저장/공유 주소 스케줄 | ecc_data.h PARAM_LLR / decoder HW 메모리모델 | LDPC-CC 전용 스케줄, 직접 대응 낮음 |
| LDPC-CC(convolutional) 부호구조 | 대응 없음 | Prime ECC는 block QC-LDPC, 코드 패밀리 상이 |
```

적용 가치: 낮음 — LDPC 컨볼루셔널(스트리밍) 부호로 NAND block ECC와 코드 패밀리가 다르고, 재사용 가능한 min-sum HW 트릭(comparator 공유, sign XOR, 0.75 scale)은 이미 Prime ECC가 보유.
