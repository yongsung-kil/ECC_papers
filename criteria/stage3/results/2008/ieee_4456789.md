### B. 알고리즘 요약

1. 대상 부호는 Mobile WiMAX(802.16e) QC-LDPC: 12x24 base matrix를 확장인자 `z`(24~96, 4씩, 19종)로 확장, 각 원소 zero/identity/cyclic-shift 서브행렬, block size 576~2304.
2. 문제: 19-mode 지원 시 라우팅 복잡도·칩면적·전력이 크고, partial-parallel은 throughput↓·latency↑.
3. 알고리즘: Min-Sum(MSA), CNU는 min/2nd-min 탐색, BNU는 합산 후 sub·clipping, 8-bit 고정소수(4-int+4-frac) 메시지.
4. 핵심 기법1 `base matrix 재정렬`: 행/열 재정렬로 lower-left·upper-right를 모두 zero matrix화해 idle 감소, 단 QC 성질 유지 위해 전체 H가 아닌 base matrix만 재정렬(제어신호만 변경, HW 추가비용 없음).
5. 의미: 재정렬로 CNU/BNU 오버랩 가능해져 정정성능 저하 없이 latency 단축.
6. 핵심 기법2 `overlapped BNU/CNU`: 재정렬된 zero 영역 덕에 첫 block들을 조기 실행, 비오버랩 대비 latency 68.75%로 축소, HW 활용률 50%→75%.
7. 핵심 기법3 `HDA 조기종료`: 두 연속 iter의 hard-decision(LLR 부호)을 D-FF+XOR로 비교, 동일하면 종료; systematic code라 정보비트(1152)만 검사, SNR별 동적 iter 조절.
8. 핵심 기법4 `19-mode 재구성`: AGU+Counter+LUT로 저장/연산 유닛 변경 없이 19종 주소 생성, divider 대신 LUT로 shift값 계산.
9. 결과: TSMC 0.13um 1P8M, core 4.45mm^2/die 8.29mm^2, 420K gates, 83.3MHz·52mW 실측; throughput 60.6(@8iter)~222.2 Mb/s(@2iter), 0.85V시 16mW.
10. 한계: 정정능력·error-floor 개선 없음(표준 MSA), 기여는 WiMax 다중모드 대응 HW 스케줄·조기종료·저전력 레이아웃(checkerboard)에 국한.
