### B. 알고리즘 요약

1. 시스템: 고throughput generic QC-LDPC 복호기, `(4096,3584)(4,32)` QC-LDPC(PEG 생성), rate 7/8, AWGN(SNR ~4.0dB).
2. 문제: 대형 병렬 QC-LDPC 복호는 barrel shifter array 스위칭 네트워크가 면적·전력 과다, 반복 복호는 iteration 수가 throughput 제약.
3. 핵심 기법 1: 행렬 permutation - generic QC-LDPC `Hqc`(circulant sub-matrix)를 각 block column에서 열을 골라 one-step shift로 묶어 shift 구조 `H_s^b` (grouping level b=2/4/8)로 변환, barrel shifter 대신 one-step shift 라우팅.
4. 최적화: optimized matrix permutation - 변환 후 각 block column의 최대 row weight `dmax`를 최소화(시작열 `i0..iC-1` 선택)해 recursive CNU 입력·data path delay 축소.
5. 결과(permutation): 50개 (4096,3584) 행렬에서 `H_s^4`는 약 20% row만 2개 1-component (direct 대비 최대 row weight 크게 감소, Table I).
6. 핵심 기법 2: min-sum 기반 column-layered decoding(shuffled) - block column layer별 horizontal step `Rcv = ∏sgn(Lcn)·min|Lcn|`, vertical step `Lcv = Iv + α·ΣRmv`.
7. 효율 구현: sorted vector `m_c^g=[m1 m2]`와 sign product `Sc`를 layer 간 incremental update(scheme-A: 제거→Rcv 전송→재정렬; scheme-B: group 내 다중 neighbor는 최소값만 사용).
8. 아키텍처: CNU=CNUa+CNUb, M register array에 정렬벡터 저장, 한 clock에 block column 전체 병렬 처리(step-a→vertical→step-b).
9. 결과: column-layered가 TPMP보다 수렴 빠름; scheme-A/B는 표준 column-layered 대비 avg iteration이 SNR 4.0dB에서 0.6 증가하나 TPMP보다 여전히 훨씬 빠름, BER/FER 차이는 매우 작음.
10. 한계: 논리합성·throughput·gate count 없음(고수준 architecture+시뮬만), 양자화 bit 미명시.
