### B. 알고리즘 요약

1. 대상은 `(J,L)` regular QC-LDPC로 base matrix `B`(circulant shift 행렬, 첫 행·열=0)와 circulant 크기 `p`로 정의되며 `N=pL`.
2. 목표: 주어진 `(J,L)`과 girth `g`에 대해 되도록 작은 `p`(즉 `pmin`)를 갖는 base matrix를 찾는 것.
3. 기존 Fossorier의 guess-and-test는 non-zero 원소를 무작위로 뽑아 girth 조건식을 만족할 때까지 반복 — 큰 base matrix·`N≈Nmin`에서 매우 느림.
4. cycle 존재 조건: 길이 `2i` potential cycle을 따라 shift 차 `∆`의 교대합이 `0 mod p`이면 실제 cycle (Fossorier Thm 2.1, 식 (5)).
5. 핵심 아이디어: cost matrix `c_{j,l,z}` = 원소 `p_{j,l}`을 값 `z`로 바꿨을 때 남는 짧은 cycle의 가중합(짧은 cycle일수록 큰 weight `w_i`).
6. 각 potential cycle에서 "guilty value"(그 값으로 바꾸면 cycle을 만드는 값)를 교대합 `α`로부터 계산해 cost matrix에 누적 — 원소가 cycle 내 2회 등장하면 기여가 배가되어 처리 복잡.
7. hill-climbing: 랜덤 base matrix에서 시작, gain `g_{j,l}=c'_{j,l}-c̃_{j,l}`이 최대인 단일 원소를 greedy하게 변경, 더 줄일 수 없으면 종료(cost=0이면 성공, 아니면 실패 반환).
8. girth-10까지 구현(8-cycle 형성 경우가 많아 book-keeping이 가장 복잡).
9. 결과: g=8/J=3에서 guess-and-test 대비 `pmin` 상한 개선(예 L=12: 54→47), g=10에서는 격차 큼; success rate가 guess-and-test보다 압도적으로 높음(p=50, g=8에서 g&t는 평균 10^6회 시도 필요).
10. 한계: BER/디코딩 성능·error-floor은 본 논문에서 미측정(후속 논문 예고), HW·디코더 무관, Python 미최적화 구현.
