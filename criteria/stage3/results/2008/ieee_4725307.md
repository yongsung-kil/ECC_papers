### B. 알고리즘 요약

1. 대상 시스템은 대용량 MLC NAND SSD로, 칩 배열의 소프트에러(다중 랜덤비트) + 단일 칩 고장을 동시에 다뤄야 함.
2. 기존 2-레벨(칩내 RS + 배열 parity) 방식 대비 정정능력을 강화하기 위해 3-레벨 계층 부호를 제안.
3. 클러스터 = N 페이지 묶음. 각 페이지는 정보/shared-check/page-check 세그먼트로 구성, 마지막 페이지는 parity 세그먼트 보유.
4. 1레벨: 전체 페이지 XOR로 parity-check 세그먼트 `P` 생성 (단일 칩 소거 복구용, GF(2) 행렬합).
5. 2레벨: 클러스터 단위 `t'`-bit BCH 부호 `CCL`로 shared-check `Q` 생성 (칩 간 정정).
6. 3레벨: 각 페이지 열 `(d_i,j, q_i,j)`를 `t`-bit BCH 부호 `CPG`로 page-check `r_i,j` 생성 (열 단위 정정).
7. 복호는 CPG 우선 정정 → 미정정/오정정 시 parity 관계(b)와 CCL 관계(c)로 교차 정정 (syndrome 기반, error detection matrix `F` 활용).
8. 신뢰성 해석: correct-decoding `PC`, error-detection `PD`(Eq.1), miscorrection `PE`(Eq.2)를 균일 syndrome 분포 가정으로 근사, Fig.8 시뮬로 검증.
9. 결과: `b=8,k=2048,r0=4,r1=60,t=5,N=9,t'=19`에서 raw BER `1.08e-4` → 복호 BER `2.44e-19`; 1 spare chip로 MTTDL 13→34년.
10. 한계: 인코더/디코더 회로 복잡도·지연·throughput 미평가, 실환경 실측 미수행; 순수 하드판정 BCH 조합으로 반복 복호(LDPC) 요소 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-레벨 BCH/parity 계층 부호 | 대응 없음 | Prime ECC는 binary QC-LDPC + min-sum 반복복호, BCH 계층구조와 무접점 |
| 단일 칩 고장 복구 parity | 대응 없음 | 시스템(RAID)-레벨 소거정정, 칩내 LDPC ECC 계층과 별개 |
| syndrome 기반 교차정정 | 대응 없음 | 대수적(algebraic) 복호로 message-passing 디코더와 무관 |
```

적용 가치: **낮음** — BCH+parity 하드판정 시스템레벨 부호로 binary QC-LDPC 디코더/부호설계에 떼어 쓸 요소가 없음.
