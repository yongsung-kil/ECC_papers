### B. 알고리즘 요약

1. 대상: 생성행렬 `G`(systematic form)를 가진 이진 (N,j,k) LDPC/QC-LDPC 부호의 최소weight `wmin`·최소거리 `dmin` 추정 문제.
2. 문제: 최소거리 계산은 NP 문제(`K=2^M-1` 조합)로 코드가 길면 사실상 불가능, 기존 확률적 탐색은 유한 시간 내 최소값을 놓칠 수 있음.
3. Theorem 2/3: 부호어 `cn`이 `G`의 행벡터 `g`의 선형결합이므로 `wmin(c) ≤ wmin(g)`, `dmin(c) ≤ dmin(g)` — 생성행렬 벡터에서 상한 획득.
4. 의미: 최소거리 탐색을 훨씬 작은 파라미터 공간(생성행렬 행벡터 집합)으로 축소.
5. LWC(Low Weight Codewords) 탐색: 전0 부호어에 결정론적 길이 `Nr`의 연속 오류(all-1 sequence)를 삽입해 섭동, BP 디코딩으로 전0 부근 최저weight 부호어 근사.
6. 절차: `Nr=1`부터 all-1 부분수열을 all-0 수열의 모든 위치 `i`에 미끄러뜨리며 BP 디코딩, 디코딩 실패 시점의 `Nr`을 `wmin(c)`로 추정, `wmin(g)`까지 반복.
7. girth-4 판정: 테스트행렬 `O = H^T H`의 대각 외 원소가 모두 0/1이면 girth-4 없음(Theorem 4)으로 예제 검증에 사용.
8. Example 1(a=3,b=5,j=3,k=6,M=31,N=186): girth-4 free, `wmin(g)=28`, LWC로 `wmin(c)=11` → 좋은 BER(Fig.1).
9. Example 2(문헌[16]식 rate-1/2 QC, N=186): girth-4 free여도 `wmin(g)=6`, `wmin(c)=2` → 나쁜 BER, 길이를 M=127로 늘려도 `wmin(c)=2` 유지(개선 불가).
10. 한계: HW 미설계·소형 예제뿐, girth-4-free가 큰 최소거리를 보장하지 않음을 보이는 진단 도구 수준.
