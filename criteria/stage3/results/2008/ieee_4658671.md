> 총평: 고rate QC-LDPC의 error-floor를 지배 trapping set 비트 pinning으로 낮추는 저장장치용 논문. trapping-set 타깃 bit-pinning 아이디어는 Prime ECC error-floor 대책에 참고 가치 있으나, PR 채널/외부 BCH/trellis-pruning은 NAND에 그대로 이식 불가.

### B. 알고리즘 요약

1. 대상: 자기·광 저장용 QC-LDPC — 목표부호 rate-0.9 (4544,4096) IRA(7×71, 64×64 circulant, col-wt 5/2), 시뮬용 파생 rate-0.78 (2048,1600); PR1(`1+D`)·EPR4(`1+D-D²-D³`) 채널.
2. 문제: 반복 LDPC 디코더의 error-floor는 저가중 codeword가 아니라 trapping set(=USC/MSC 유발 부분그래프)에서 발생; 저장/광통신은 극저 오율 요구.
3. 핵심개념: `(w,v)` trapping set의 변수노드 하나를 외부 정보로 pinning하면 나머지 비트는 최대 2회 추가 반복으로 정정됨.
4. 핵심기법1(pre-pinning): 지배 trapping set을 덮는 특정 비트(예: (2048,1600)의 vp=0~3, 256비트)를 인코딩 전 '1'로 고정, 디코더는 해당 비트 LLR을 최대로 세팅(=code shortening 유사).
5. 의미: 640개 trapping set이 이 256비트 중 정확히 1개씩 포함 → 256비트 pinning으로 모든 trapping set 정정 가능.
6. 핵심기법2(post-pinning): 256비트를 (256,240,t=2) BCH로 보호; LDPC 출력에서 trapping set당 최대 1비트 오류 → BCH가 정정 후 그 값을 디코더/검출기에 피드백.
7. 구현 디테일: pinned bit를 검출기 trellis에 반영(trellis-pruning) — PR1은 1상태로 고정, EPR4는 trellis stage 분기 절반; pinned '1'을 8비트마다 배치해 runlength 제약(bit-timing) 부수 획득.
8. 채널 적응: AWGN에서 찾은 지배 trapping set이 PR1/EPR4에서도 지배적임을 관찰해 동일 타깃 사용.
9. 결과: pre-pinning은 floor ≥2자릿수 하강(trellis-pruning이 bit-pinning 대비 FER 10⁻⁶에서 ~0.4dB 이득, rate 0.76dB 손실은 거리스펙트럼 개선으로 상쇄); post-pinning은 rate 손실 0.04dB로 floor를 시뮬 한계 이하로 하강, waterfall ~0.6~0.7dB 이득(글로벌 반복 5회).
10. 한계: 지배 trapping set 사전 열거 필요, PR 채널/BCJR 전제, HW 미설계(baseline 부호만 FPGA), 채널·디코더 변경 시 trapping set 변동 가능.
