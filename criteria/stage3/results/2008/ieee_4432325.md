### B. 알고리즘 요약

1. 대상: 이진 irregular LDPC 부호 (예제 `n=504`, `n=1008`, rate `1/2`, DE 최적화 degree 분포), AWGN/BPSK.
2. 문제: iterative 복호의 error-floor는 작은 stopping/trapping set이 좌우하는데, stopping distance 최대화는 NP-hard라 명시적 제거가 어렵다.
3. 우회 아이디어: stopping set이 유발하는 cycle의 ACE(Approximate Cycle Extrinsic message degree)를 키우면 stopping distance가 커진다.
4. ACE metric: cycle 내 symbol node에 대해 `Σ(d_s − 2)`, 즉 cycle 바깥으로 뻗는 extrinsic edge 수를 근사.
5. 평가도구: 길이 `2i` cycle의 최소 ACE값들로 구성한 ACE spectrum `η=(η2,…,η_2dmax)`를 정의.
6. 핵심: PEG를 두 절차("Check Nodes Shortest Paths Discovery" + "Check Node Selection")로 일반화하여, 새 edge 추가 시 root와 candidate check node 간 모든 최단경로와 path-ACE metric을 BFS로 열거.
7. 제안 선택기준(우선순위): 최대 최소 path-ACE → 최소 ACE 경로 수 최소 → 총 경로 수 최소 → 최소 degree survivor. 기존 PEG/Improved PEG를 특수 경우로 포함.
8. 후처리: greedy 특성상 편차가 커 각 알고리즘당 100개 그래프 생성 후 ACE spectrum 비교로 상위 10개 선별(Viterbi류 선형시간).
9. 결과: `n=1008`에서 ACE spectrum 성분이 (13,4,3)→(17,8,3)류로 상승, error-floor 영역 BER이 original/improved PEG보다 개선(Fig.2, sum-product BP 100 iter).
10. 한계: HW 미설계, AWGN 시뮬만, irregular non-QC 단중길이 rate 1/2 한정, ACE 증가가 near-codeword/trapping set을 완전 제거하진 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG 기반 H-matrix/부호그래프 구성 | [ecc_top.cpp](../../Prime_ECC_3.1_Claude/) `Load_PCM()` | 부호구조 로드 지점이나 QC(circulant) 고정 - PEG 랜덤 irregular 구성과 정합 안됨 |
| ACE/stopping set 기반 error-floor 설계 | 대응 없음 | 코드베이스에 부호설계/최적화 모듈 부재 (H-matrix 고정) |

적용 가치: **낮음** — irregular non-QC rate 1/2 코드의 부호구성 알고리즘으로, 고rate 고정 QC-LDPC를 쓰는 Prime ECC에 직접 이식 대상 모듈이 없다. ACE로 short-cycle을 다뤄 error-floor를 낮춘다는 설계 원리만 참고 가치가 있다.
