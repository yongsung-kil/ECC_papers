### B. 알고리즘 요약

1. QC-LDPC 부호(Chinese DTTB), `N=7493`, rate 0.41/0.61/0.81, circulant size `b=127`, partially parallel 복호.
2. 문제: 전통 partially-parallel QC-LDPC 복호기는 HPU 조합논리 critical path와 HPU/VPU 교대 유휴로 throughput이 제한됨.
3. 복호 알고리즘: normalized UMP BP(=min-sum), horizontal process `L_mn = α·(∏sgn)·min_{n'≠n}|z_mn'|`, 정규화 계수 `α`는 shift/add로 구현.
4. 핵심기법1(HPU 파이프라인): 각 입력을 뺀 min(selective set `S`) 연산을 `T=⌈log2(λ-1)⌉+1` 단으로 분해해 critical path를 절단.
5. Cyclic HPU: 각 stage에서 cyclic하게 비교·레지스터링하여 selective min을 점증 누적, comparator `C(λ)=(T-1)·λ`, register `T·λ`.
6. Dimidiate HPU: 입력집합을 이진트리로 반분(⌈k/2⌉,⌊k/2⌋), 노드별 selective/total result 계산, comparator는 더 적고 register는 더 많음.
7. 핵심기법2(dual-word 스케줄링): 두 코드워드를 번갈아 처리해 HPU/VPU 유휴 제거(메모리 2배, dual-port rRAM/qRAM 필요).
8. VPU도 binary-adding-tree로 파이프라인, pipeline 단수만큼 qRAM/rRAM 쓰기를 지연시킴(stage 병합/삭제로 조정 가능).
9. 결과(Altera Stratix II FPGA, 30 iter): fmax 약 3배↑, throughput 25→110 / 34→172 / 31→237 Mbps(>4배), ALUT는 거의 동일하나 register 대폭 증가.
10. 한계: 정정성능 개선 없음(순수 throughput), FPGA ALUT/register만 보고(ASIC gatecount·공정 없음), 새 critical path가 controller→RAM 라우팅으로 이동.
