### B. 알고리즘 요약

1. 목표는 단·중 블록길이의 structured(HW 지향) LDPC를 PEG로 구성하는 것, 예제 `(2560,1280)`·`(2560,1920)`·`(2304,1152)`, rate 1/2·3/4, log-BP max iter 80, BI-AWGN.
2. 문제: 원 PEG는 H의 1 위치가 완전 random이라 VLSI 복잡도가 크고, 기존 PEG-QC(Z.Li)는 circulant 서브행렬이 girth-4 short cycle을 쉽게 유발해 성능이 저하된다.
3. 모델: variable/check node를 `p`-크기 subgroup으로 분할하고 subgroup 단위로 edge를 추가, 결과 H는 circulant-permutation(CP)/all-zero 서브행렬 블록(식(1)).
4. 핵심기법 1단(CP-PEG): node-by-node 대신 subgroup 단위로 edge를 추가하되 tree spreading으로 local girth를 최대화하는 최저 degree check node를 선택 → short-cycle 확률↓.
5. 이유: CP 서브행렬 구조를 유지하면서 girth를 키워 PEG-QC의 circulant-유발 short cycle을 회피한다.
6. 핵심기법 2단(ALT-CP-PEG): H를 approximate lower triangular로 제약하고 고차수 VN을 systematic part에 배치, gap `g`를 최소화(block-column permutation으로 특정 블록 identity화)해 인코딩 복잡도를 `O(g²)` 수준으로 절감.
7. 부수기법(EMD 기준): small stopping set 회피를 위해 tree-spreading 후보 check node를 EMD(및 ACE) 최대화로 선택 → error floor↓.
8. 학습/최적화: degree distribution은 density evolution으로 최적화(irregular은 max VN degree 지정).
9. 결과: 1000-code ensemble에서 girth-4 확률이 PEG-QC보다 낮고 BER/PER가 PEG-QC·IEEE 802.16e보다 우수(고SNR PER 1자릿수↑), EMD 적용 시 error floor 추가 저하, CP 구조로 layered decoding 약 2배 빠른 수렴.
10. 한계: 실 HW/gatecount 없이 시뮬만, AWGN·log-BP 가정, 부호 재설계 특성상 기존 고정 부호 교체 시 재검증 부담.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CP-PEG H-matrix/부호구조 구성 | ecc_top.cpp Load_PCM() | 새 구성법이나 Prime은 고정 PCM_b라 전면 교체·재검증 필요 |
| ALT-form 인코딩(gap 최소화) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime은 dual-diagonal 고정이라 ALT와 인코딩 구조 상이 |
| EMD/stopping-set 기반 error-floor 개선 부호설계 | ecc_top.cpp Load_PCM() (부호설계 레이어) | error-floor 설계 관점 유의미하나 코드 재검증 부담 |
| CP 구조 → layered decoding 2x 수렴 | decoder.cpp LDPC_Decoder() | Prime의 Dual-Update 스케줄과 대응(이터레이션 감소) |
```

적용 가치: 중간 - error-floor 인식 structured(QC) 구성법은 NAND 고rate 부호설계에 유의미한 참고가 되나 Prime의 고정 H-matrix 교체·재검증 부담으로 직접 이식은 어렵다.
