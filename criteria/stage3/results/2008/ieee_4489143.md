### B. 알고리즘 요약

1. 시스템: 8×8=64개 동종 PE(SIMD/MIMD) pool + PE 컨트롤러(32-bit RISC) + Data/Config Buffer로 구성된 재구성형 FEC 프로세서, 8~16bit 데이터패스.
2. 문제: 표준마다 다른 FEC(polynomial/블록크기/rate/termination)를 별도 ASIC coprocessor로 두는 관행 → 프로그래머블 다중표준 엔진으로 대체.
3. PE 구성: 각 PE에 ALU, ACS(Viterbi/Turbo용 Max/Max*), GF accelerator(RS용 8차 다항식), 16B LUT 탑재.
4. LDPC 매핑: 64 PE를 변수노드/검사노드로 교대 구성해 message passing 수행, 노드 수가 64 배수일 때 최대 병렬.
5. 검사노드 연산: `φ(x)=-ln(tanh(x/2))` 비선형 함수를 LUT로 구현(sum-product), variable node는 ALU로 계산.
6. 종료: 최대 iteration(20) 도달 or 모든 parity check 만족 시 hard decision.
7. 데이터 라우팅: Data Buffer 64 memory bank + 2단 MUX selection network로 임의 permutation/interleaving/data shuffling 지원(인터리버를 Config Table로 로드).
8. 부호: WiMAX/DVB-S2 H-matrix 사용(288×576, 528×1056, 704×1056; max row/col degree 4/7; 20 iter; 4-bit message resolution).
9. 결과: LDPC throughput 355/85/58 Mbps(행렬별), TSMC 90nm RTL 합성 시 PE당 전력 3.1mW, 고정 WiMAX(70Mbps) 요구 충족.
10. 한계: LDPC 알고리즘 신규성 없음(표준 sum-product), gate count 미제시, 64 PE 제약으로 부분 병렬, 무선표준 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PE를 VN/CN으로 교대 구성한 message passing | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 개념적 대응뿐, Prime ECC는 SIMD array 아닌 고정 6-stage 파이프라인 |
| 검사노드 `φ(x)=-ln(tanh(x/2))` sum-product LUT | 대응 없음 | Prime ECC는 min-sum(`CNU_Update_New_Mag()`)으로 대체 이미 보유, full-tanh SP는 역행 |
| 4-bit message resolution / soft 입력 | [channel.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | 양자화 개념만 겹침, NAND read 레벨과 무관한 무선 soft |
| DB selection network(permutation/interleaving 라우팅) | 대응 없음 | Prime ECC 고정 z=32 병렬·GT 구조와 다른 범용 SIMD 라우팅 |

적용 가치: **낮음** — 다중표준 재구성형 프로세서로 LDPC부는 표준 sum-product를 범용 PE array에 얹은 것뿐이고, 전용 고정 NAND QC-LDPC 엔진과 설계철학이 반대라 떼어 쓸 알고리즘 요소가 없다.
