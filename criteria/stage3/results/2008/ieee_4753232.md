> 총평: Layered/shuffled BP(TDMP)가 반복 제약 하에서 flooding SPA 대비 iteration·latency를 크게 줄인다는 정량 데이터. 다만 Prime ECC가 이미 Dual-Update(layered 교대 스케줄)를 보유해 기법 중복이며, 저rate·非양자화 wireless 시뮬 한정.

### B. 알고리즘 요약

1. 시스템: AWGN 채널, Sridhara-Fuja-Tanner(SFT) QC-regular LDPC (3,5)-regular. 세 부호 (305,124)/(1055,424)/(2105,844), rate≈0.40, circulant `m`=61/211/421.
2. 문제: 실제 수신기는 iteration 수를 제약(프로세서·데이터율·연산부하)해야 하므로, 수렴 한계 성능이 아닌 **지연 제약 하 성능**으로 디코더를 평가해야 함.
3. 부호 구조: `H`를 m×m permutation circulant `α`의 거듭제곱으로 배치(`α^{b^{st}}`, ord(a)=k, ord(b)=j) → circulant 기반 QC로 고속 인코딩(VLSI) 가능.
4. 두 스케줄 비교: SPA는 flooding(모든 CN 갱신 후 VN 갱신), TDMP(=shuffled/layered BP)는 각 CN(행) 갱신 직후 VN 갱신 → 같은 iteration에서 더 최신 메시지 사용.
5. TDMP 절차: 행 i마다 extrinsic `λ_i`를 posterior `γ(I_i)`에서 빼 prior `p` 생성 → `φ(x)=-log tanh(x/2)` SISO로 행 복호 → posterior 갱신, 모든 행 반복이 1 iteration.
6. 핵심 식: SISO는 SPA와 동일 φ 연산(또는 BCJR형 축소복잡도 SISO 대안). serial 구현 시 iteration당 연산비용이 SPA와 동일 → iteration 수로 공정 비교.
7. 병렬성: m개 permutation submatrix는 CN 갱신 간 VN 공유 없음 → TDMP는 m개 CN 동시 갱신의 partial-parallel에 자연 적합(단 본 논문은 serial만 평가).
8. Hard real-time(최대 iteration 제약) 결과: 목표 Pblk=10⁻²에서 10 iter시 TDMP가 SPA 대비 0.3/0.51/0.61dB(short/inter/long), 5 iter시 0.8/1.22/1.4dB, 2 iter시 1.9/2.17/2.2dB 우수. 블록 길수록 이득 증가.
9. Soft real-time(평균 iteration 제약) 결과: max 50 iter시 TDMP 평균 3.9 vs SPA 6.1 iteration. 평균 2.5 iter 제한 시 TDMP가 1.5dB 우수, 1 iter시 1.8dB.
10. 한계: HW 미설계(serial 시뮬만), 양자화 미고려, 저rate(≈0.4) wireless 대상, 수렴 후 한계 성능은 두 알고리즘 거의 동일(0.1dB).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP(layered/shuffled BP) 스케줄 | decoder.cpp LDPC_Decoder() 루프, encoder.cpp Column_Reordering_Dual_Update() | Prime ECC의 Dual-Update(layered 교대 스케줄)와 개념 중복, iteration 이득 근거 |
| 반복 제한(delay) 하 성능/조기종료 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | iteration 상한·조기종료와 직결되는 성능 트레이드오프 참고 |
| QC-LDPC circulant 부분병렬 | 디코더 HW 모델 (z=32 row 병렬) | permutation circulant → 부분병렬 구조 정합 |
| SISO φ(x) CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal() | SPA φ 기반, Prime ECC는 min-sum → 알고리즘 상이 |
```

적용 가치: **중간** — Layered(TDMP) 스케줄이 반복 제약 하에서 iteration/latency를 정량적으로 크게 줄인다는 QC-LDPC 데이터는 Prime ECC의 Dual-Update·조기종료 설계 근거로 참고 가치가 있으나, 기법 자체는 이미 보유한 Dual-Update와 중복됨.
