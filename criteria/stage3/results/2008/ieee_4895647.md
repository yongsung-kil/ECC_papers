### B. 알고리즘 요약

1. 대상: irregular GLDPC(CN이 SPC뿐 아니라 일반 선형블록부호 - 여기선 SPC-7 + (7,4) Hamming 혼합), BEC 채널, iterative 복호. 예제 부호 `(2000,667) R≈0.33`, `(6000,1200) R=0.2`.
2. 문제: GLDPC의 error floor(BEC)를 부호설계 단계에서 예측·억제하고 싶은데, CN 복호 방식(BD vs MAP)에 따라 stopping set 정의가 달라짐.
3. 모델: stopping set = BEC에서 정정불가 erasure 패턴에 대응하는 VN 부분집합. BD 복호는 Def.1(dmin=d CN에 d번 이상 연결), MAP 복호는 Def.3(CN sockets가 non-null CN codeword를 cover)로 정의.
4. 핵심기법: stopping set size 분포의 성장률 `G_SS(w)=lim (1/n)ln E[N_SS]`의 `w=0` 근처 1차 Taylor 전개(valid assignment 생성함수 `p(x)` 이용).
5. 핵심결과(Thm1, r=p=2): `G_SS,BD(w)=w·ln(λ'(0)ρ2(1))+O(w²)`, `G_SS,MAP(w)=w·ln(λ'(0)C)+O(w²)`. 계수가 degree-2 VN·dmin=2 CN에만 의존.
6. 의미: `λ'(0)ρ2(1)`(BD) 또는 `λ'(0)C`(MAP)가 <1이면 small linear-size stopping set 수가 지수적으로 작아짐(=low error floor 조건). 항상 `ρ2(1)>C`라 MAP는 낮아도 BD는 error floor 있을 수 있음.
7. 설계기법: 제약된 differential evolution(DE)로 BEC threshold를 최적화하되 `λ'(0)ρ2(1)≤β<1`(또는 MAP용 `λ'(0)C≤β`) 제약 부과, 그 분포를 PEG로 Tanner 그래프 구성. β가 1에 가까울수록 threshold↑·error floor↑ trade-off.
8. 학습/최적화: DE 앙상블 최적화 → VN degree 2~15, CN은 SPC-7 + (7,4) Hamming 혼합.
9. 결과: `(2000,667)`는 `λ'(0)ρ2(1)=1.2>1`이라 BD에서 FER 4e-5 error floor, MAP는 1e-6까지 floor 없음. `(6000,1200)`는 `λ'(0)ρ2(1)=λ'(0)C=0.9`로 MAP threshold q*=0.786(Shannon 0.8 근접)이고 BD/MAP 모두 1e-6까지 floor 없음.
10. 한계: BEC 전용·GLDPC(비-SPC CN) 특화, HW 미설계, AWGN/soft-read NAND 채널로의 확장은 미검증.
