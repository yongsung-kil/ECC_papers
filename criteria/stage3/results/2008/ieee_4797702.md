### B. 알고리즘 요약

1. 대상/채널: γ-left regular binary LDPC를 BSC에서 hard-decision Gallager A/B·parallel/serial bit flipping으로 복호할 때의 보장 정정능력 분석(리뷰 논문).
2. 문제: 최소거리가 length에 선형 증가해도 반복 복호에서 worst-case 오류의 선형 비율 정정을 보장하지 못함 → girth/column-weight로 정정능력을 특성화.
3. column-weight-3 상한: girth `g≥10`인 `(3,ρ)` 코드는 weight `g/2` 오류패턴을 모두 정정 불가(최단 cycle이 fixed point), 어떤 `α` 비율도 large `n`에서 보장 불가.
4. column-weight-3 하한: girth `g≥10` 코드는 Gallager A로 weight `g/2-1` 이하 오류를 `g/2` iteration 내 정정(Theorem 2) → 상한이 tight함.
5. 증명 기법: bad variable node configuration을 `Ne^{2k}` 트리 이웃에서 열거, valid configuration(`≤ g/2-1` bad, length `<g` cycle 없음)이 수렴함을 귀납적으로 증명(Theorem 3,4,5).
6. trapping set 정의: fixed point 조건(supp=T, 각 variable이 even-degree check에 최소 2개(γ≥4는 γ/2개) 연결, odd-degree check 공유 제한)으로 `(a,b)` trapping set 규정.
7. higher column-weight: Sipser-Spielman expander 정리 - `(γ,ρ,α,(3/4+ε)γ)` expander면 parallel bit flipping이 `α0` 비율 오류 정정.
8. expansion↔girth: Moore bound `n0(γ/2,g')` 이용, `k < n0(γ/2,g')`인 variable node 집합은 최소 `3γ/4`로 expand(Theorem 8) → 정정능력 하한.
9. cage graph 상한: 최소 potential trapping set 크기 = cage graph 노드수 `nc(⌈γ/2⌉,g')`, girth `2g'` 코드는 `nc(⌈γ/2⌉,g')` 오류 정정 실패(Theorem 10,11); `γ=3,4`에서 tight.
10. 결론: column-weight-3의 정정능력은 girth에 선형으로만 증가(한계), higher column-weight는 expansion 하한 + trapping set/cage 상한을 제공.
11. 한계: hard-decision Gallager A/bit flipping·BSC 특화 이론이라 min-sum/soft decision 및 구체 부호구성은 다루지 않고 시뮬 없음.
