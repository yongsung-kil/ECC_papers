### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 무기억 채널, rate-1/2 (2048,1024) irregular LDPC 두 앙상블(`4-8`, `5-9`)을 대상으로 error floor를 분석.
2. 문제: irregular 부호는 near-capacity threshold를 얻지만 유한길이·저-BER 영역에서 trapping set(TS)에 의한 error floor가 코드별로 크게 변동해 실용 응용(제한된 메모리/연산)에 위험.
3. 채널/모델: `LCi = 4*Yi*Es/No`의 LLR, sum-product(full BP) 메시지 전달, `Lrji = 2 tanh^-1(∏ tanh(Lqij/2))`, 최대 50 iteration.
4. 부호 구성: 단일 edge 추가 후 degree 분포·girth·extrinsic 제약 위반 시 롤백하는 반랜덤 배치. 초기상태는 parity부의 staircase(dual-diagonal) 패턴(`dv=2` n-k-1개 + `dv=1` 1개).
5. 두 degree 분포: `Λ1(x)` = 4/8차, `Λ2(x)` = 5/9차 시스템 노드; check 노드는 각각 `dc=7`, `dc=8` 균일 지향 → 빠른 인코딩 가능.
6. 핵심 통찰(정리 1,2): staircase 구성은 x=1..n-k에 대해 (x,1) TS와 n-k-x개의 (x,2) TS 존재를 **강제** → 어떤 staircase H도 이 저중량 TS를 피할 수 없음.
7. TS 탐색: 결정론적 잡음 임펄스를 Vnum-bit 조합에 주입, 발견 카운터로 dominant TS(작은 `d~`) 식별. 파라미터 `t1=3.5, Eb/No=4dB, Vnum=3` 등.
8. 3종 dominant error: Type1=저중량 codeword(x,0), Type2=parity(staircase)내 (x,1)/(x,2), Type3=staircase run + 소수의 `dv=4/5` bit 조합(error floor 개시 주원인).
9. 결과: 대표 4-8 부호 `dmin=15`, 5-9 부호 `dmin=22`. Eb/No≈2~2.5dB에서 error floor 시작, Eb/No=4dB 부근에서 두 부호 성능 수렴(Type2 지배). Monte Carlo + importance sampling 시뮬로 확인.
10. 한계: HW 미설계, full BP 가정(min-sum엔 SNR 불변성 다름), 특정 rate-1/2 irregular 앙상블에 한정, 정정 개선 기법이 아닌 분석/설계 도구 성격.
