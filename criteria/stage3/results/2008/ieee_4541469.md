### B. 알고리즘 요약

1. 대상: IEEE 802.16e(WiMAX) irregular QC-LDPC, base PCM `Hb`(Mb×Nb)를 z×z circulant으로 lifting, 예제는 rate-1/2·length-2304(`z=96`).
2. 문제: 기존 TDMP(layered MPD)는 TPMP보다 2배 빠르나 latency·memory access를 더 줄이고 싶음.
3. 배경기법: [9]는 C를 소형 선형 block code `Cl(0)..Cl(z-1)`(PCM `Hl`, Nb<Nl<N)로 분해해 순차복호, extrinsic을 코드간 공유.
4. 핵심기법: 본 논문은 `Cl(p)`,`Cl(p+1)`을 동시복호. `Hl`을 odd/even row로 나눈 `Ho`,`He`로 `He→Ho→He` 순차 처리하고 두 복호 간 extrinsic 교환.
5. 활용구조: `Hl`의 대부분 열은 weight 1, 마지막 12열만 weight 2 → 열을 RA(weight1)/RB(weight2 part B)로 분리해 B-part EVC로 extrinsic 재사용.
6. CVMU: m1-m2 selector로 min-sum(offset MSA) 계산, A-part register에 m1,m2,jm1,jm2 저장, Λ-R calculator로 APP·R 갱신(Step0~5).
7. HW: memory 기반(register 기반 [9] 대체), `MBΛ`/`MBR` 메모리뱅크 + exchange network, 단순 address controller(복잡 permutator 회피).
8. 타이밍: `(Cl(p),Cl(p+1))` 갱신 10 cycle, pipeline으로 latency `Ni×10×(z/2)+12` cycle.
9. 결과: TSMC 0.18μm, 105K gate, 200MHz, 4.6 avg iter로 BER 2e-6@2dB, 106Mbps(TDMP 71Mbps 대비). Ni=12가 TDMP Ni=15와 동일 BER → latency·memory access 80%.
10. 한계: WiMAX 부호구조 특화, AWGN/BPSK 시뮬+합성만(실측 없음), NAND soft-read 논의 없음.

적용 가치: **중간** — 수렴가속·latency 개선 목표는 Prime ECC 관심사와 맞으나, odd/even 스케줄이 기보유 Dual-Update와 중복이고 소형 block 분해가 WiMAX 부호구조에 종속.
