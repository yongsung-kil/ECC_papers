### B. 알고리즘 요약

1. 대상 부호는 IEEE 802.16e QC-LDPC: base matrix를 확장인자 `z`(24~96, 4씩 증가, 19종)로 확장, 각 원소는 zero 또는 cyclic-shifted identity 서브행렬, rate 1/2~5/6, length 최대 2304-bit.
2. 문제: 다중 rate·다중 서브행렬 크기(19종)에서 SPA의 노드 간 데이터 의존성이 스루풋을 제한하고, 가변 크기 crossbar 스위치는 라우팅 혼잡을 유발.
3. 채널: 6-bit 채널 확률정보(soft)를 LLR로 사용, BP/min-sum 기반.
4. 핵심 기법1 `phase-overlapping message passing`: 서로 다른 row(서브행렬) 간 phase 의존성을 decouple해, 이전 row의 check node 출력 메시지를 즉시 다음 bit node로 전달 → 인접 2개 서브행렬 동시 처리.
5. 의미: layered decoding처럼 데이터 의존성을 풀어 SPA 대비 약 50% 스루풋 향상.
6. 핵심 기법2 `self-routing switch network`: 소스 메시지에 self-routing 비트를 삽입, 단일 96-크기 permutation망(coarse 5bit + fine 2bit shift)으로 19종 크기 라우팅을 blocking 없이 수행.
7. 구현 디테일: check node cell = 1-bit 부호곱 + 5-bit 정렬기(min/2nd-min 탐색), bit node cell은 SM↔2's complement 변환 후 합산·클리핑, min-sum에 normalized factor 적용.
8. HW: CNP/BNP 각 96 처리셀 + 2개의 96x96 self-routing 스위치망, 메모리(bit sum/check sum/min group/channel value) + 버퍼관리·스케줄러.
9. 결과: 90nm 1P9M CMOS, 6.25mm^2, 380k gates, 150MHz에서 rate-5/6 2304-bit를 20 iter에 105 Mb/s, 최대 1.23 Gb/s, 802.16e 준수 109MHz시 186mW.
10. 한계: 정정능력·error-floor 개선 없음(표준 min-sum 구현), 기여는 WiMax 특정 가변크기 구조 대응 HW 라우팅·스케줄링에 국한.
