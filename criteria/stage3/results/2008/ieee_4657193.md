### B. 알고리즘 요약

1. 시스템: `Q`-user 상향링크 MAC, 사용자 `q`를 user-specific channel code `Cq`로 분리하는 Channel CDMA(CCDMA), IDMA류. half-rate `R=1/2`, `γ=3`, `N=1008` bit.
2. 문제: pseudo-random LDPC로 사용자 분리 시 각 사용자마다 별도 PCM 저장 필요(메모리 사용자수 비례) + 사용자별 girth 등 그래프특성 달라 불평등 보호.
3. 모델: 실수 AWGN MAC `y=Σ hq xq + n`, PIC 기반 반복 수신기(SISO detector + `Q`개 SISO LDPC 디코더 뱅크).
4. 핵심기법1: `J`-level MLS LDPC를 base matrix `Hb` + `J`개 constituent matrix `Qj` + adjacency matrix(=order-`J` Latin square)로 표현. constituent 비영원소는 base 비영원소 위치에만, 서로 배타적.
5. 핵심식: 부호율 `R=1-M/N`, Latin square 개수 `XJ=J!·(J-1)!·L(J,J)` → 6-level이면 `X6=6!·5!·9408`개 고유 PCM/사용자 표현 가능.
6. 핵심기법2: 모든 사용자가 같은 base+`J` constituent 공유, 사용자별로 다른 `(J×J)` Latin square(adjacency)만 할당 → 메모리 사용자수 독립.
7. 트릭: isotopic Latin square(row/col/symbol permutation triple)로 사용자 부호를 distinct하지만 isomorphic edge-colored Tanner graph로 만들어 girth 등 그래프특성 동일 → 동일 QoS.
8. 디코딩: SPA(sum-product) 최대 100 iteration, PIC-디코더 간 반복 `I=5`(2user)/`I=10`(3user).
9. 결과: AWGN/Rayleigh, BPSK, `N=1008`, `Q=3`에서 MacKay pseudo-random 대비 BER/BLER 열화 없음. LUT edge 저장 9072 → 612로 감소(84×168 constituent 6개 + 6×6 Latin square 3개).
10. 한계: HW 미설계, 시뮬만, 무선 다중접속 부호분리 전용 이득(사용자 다수 전제), RA 등 random-like 부호에도 적용 가능하다고만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MLS LDPC PCM 구조/로드 | `ecc_top.cpp` `Load_PCM()` | 구조 상이(Latin square multilevel), 단일부호엔 압축 동기 없음 |
| SPA(full sum-product) 디코딩 | 대응 없음 (Prime ECC는 min-sum 근사) | tanh 기반 full-SPA는 미보유이나 HW 지향 아님 |
| 사용자별 Latin square 부호분리 | 대응 없음 | NAND는 단일 고정 QC-LDPC, 다중사용자 개념 없음 |

적용 가치: **낮음** — 무선 CCDMA 다중사용자 부호분리·메모리 압축이 핵심 동기이며, 단일 고정 QC-LDPC를 쓰는 NAND ECC엔 그 이득이 성립하지 않고 디코더도 full-SPA라 이식성 낮다.
