### B. 알고리즘 요약

1. 대상: `[n,k,d]` 이진 linear block code(주 실험 `[63,39,9]` BCH, `[8,4,4]` 확장 Hamming), AWGN 이진 antipodal 채널, `-log` 영역 min-sum/min*-sum.
2. 문제: classical block code는 고밀도 PCM이라 Tanner graph에 length-4 cycle 다수 → 표준 message passing 성능 저하.
3. 핵심가정: 부호의 permutation group `Per(C)`에서 뽑은 치환 `β`로 `βH`에 복호하는 것은 `β⁻¹SI`를 `H`에 복호하는 것과 등가(고정 constraint + soft info 치환만으로 redundant 복호 구현).
4. 핵심기법(RRD): 매 inner iteration마다 `Per(C)`의 random 치환 `θ`를 soft info에 적용해 서로 다른 열치환 PCM 그래프에서 `I1`회 복호, 누적 soft `s` 갱신, 유효 codeword 나오면 치환 역적용 후 종료.
5. 핵심식: massive redundancy(예 50개 치환 concat한 1200×63 `HR`)로 aggregate 모델 iteration `I=1` 설정 → 중간 메시지벡터 저장 불필요(메모리 m배 증가 회피).
6. 확장: damping coefficient `α`(outer iteration마다 `α←α0+(1-α0)·i3/(I3-1)`)로 local minima 회피·수렴 지연.
7. 부호선택 트릭: Algorithm 2가 greedy row operation으로 short cycle 최소화(HC 32,625개 4-cycle→H2 3,162개), 초기 PCM 선택이 성능에 큰 영향.
8. 난수 치환: Celler et al. product-replacement로 permutation group의 near-uniform random element 생성(치환곱 1회로 효율적).
9. 결과: `{H2,GF,PA}` 설정이 algebraic HIHO 대비 BER 10⁻⁶에서 약 1.75dB 이득, 초기 PCM 선택으로 2dB, 전체 group이 cyclic subgroup 대비 0.25dB 우세.
10. 한계: HW 미설계·시뮬만, classical 고밀도 부호 특화(LDPC 적용은 future work), `I2×I3` 다중패스로 복잡도·latency 큼.
