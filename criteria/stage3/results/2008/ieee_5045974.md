### B. 알고리즘 요약

1. 시스템: IEEE802.11n 기반 4x4 MIMO-OFDM(40MHz, 16/64QAM, 128 FFT, 18-path Rayleigh, TGn)용 QC-LDPC FEC. 구현 부호 length 960·rate 1/2 (시뮬 rate 1/2·3/4·5/6, length 1728/2592).
2. 문제: 랜덤 irregular LDPC는 G가 non-sparse라 인코딩 복잡도 `O(n^2)` → 대형 H 저장·접근·인코딩 부담.
3. 핵심 기법(부호): H를 `q×q` circulant permutation 블록 배열로 구성한 QC-LDPC, dual-diagonal(approx lower-triangular)로 `g=0` → parity bit만 sparse 연산해 인코딩 `O(n)`.
4. 인코딩: `H·c^T=0`에서 parity `p1~p4`를 식(3)~(8) 순차 XOR로 계산, generator matrix 불필요.
5. 복호: SPA(Sum-Product) message passing, CFU는 `min` + sign Ex-OR 근사(=min-sum), BFU는 비트노드+tentative decision 통합.
6. HW 인코더: Verilog-HDL, XOR 병렬화, length 960·100MHz 합성.
7. HW 디코더: partial-parallel 20-processor, Metric/Beta/Alpha Memory Array, regular 구조로 CN-VN 메시지 메모리 대폭 절감(전체 area 절반↑가 메모리였음).
8. (별도 학습/최적화 없음 — 표준 802.11n QC-LDPC 채택)
9. 결과: QC-LDPC는 random irregular 대비 BER 약간 열세이나 근사, convolutional Viterbi보다 우수; 인코딩 복잡도 랜덤 대비 약 1/34; 90nm CMOS 합성: 인코더 67125게이트/5.3mW, 디코더 852743게이트/199.6mW.
10. 한계: 무선 802.11n rate-1/2 특화, decoder throughput 수치·양자화 bit 미기재, 기법 전부 표준(신규성 낮음).
