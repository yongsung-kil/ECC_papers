### B. 알고리즘 요약

1. 시스템: BPSK/AWGN, iterative 복호(max iter 100), CP-matrix 기반 binary QC-LDPC, rate 1/2·2/3·3/4.
2. 문제: [7]의 일반 QC-LDPC 효율 인코딩은 주어진 H에서 rank/full-rank square 부분행렬 탐색이 어렵고, array LDPC은 `m` 짝수 시 rank 미달.
3. 핵심 도구: row grouping cyclic shift(GCS, S bit 상향 순환이동)와 block elimination 기반 fast Gauss elimination — CP-matrix 지수 연산(mod L)만으로 소거.
4. 핵심 성질: `x=y (mod L)`이면 `P(x)+P(y)=P(∞)`; full-rank `H(L,m,n)`를 upper-trigonal화하는 데 `m`번 연산만 필요(일반 Gauss `L*m` 대비 대폭 축소).
5. full-rank 조건: 4-CP 블록에서 `a4=a0+a2-a1` 등이 rank를 full로 만들지만 length-4 cycle을 유발 -> 지수 careful 설정 필요.
6. 부호 구성: `[H1 H2]` 형태, `H1`은 length-4 cycle 없는 full-rank square(예 3x3, 5x5 grouping), `2c=a+b`류 조건으로 full rank 확보.
7. 확장: `H1(5L x 5L)` -> `H1'(7L x 7L)` 확장, `x=4, y=5, L>52`이면 length-4 cycle 0개·length-6 cycle L개 -> 다양한 length/rate 부호 생성.
8. 인코딩: fast Gauss elimination으로 `H1` 역행렬도 획득, [7] 방식으로 parity bit 수(직렬)/부호 length(병렬)에 선형인 인코딩 가능, column weight는 2 또는 3.
9. 결과: AWGN에서 동일 rate(1/2·2/3·3/4)·유사 length의 array-based LDPC 및 802.11n QC-LDPC보다 우수(short cycle 제거+column weight 분포) — 단 수치는 그림에만.
10. 한계: HW 미설계, column weight 2~3 저밀도·저rate 중심, 실제 BER 수치 본문 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| fast Gauss elimination 기반 systematic 인코딩 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC는 dual-diagonal 인코딩 보유 -> 대체 이점 낮음 |
| full-rank QC-LDPC H-matrix 구성/지수 배치 | ecc_top.cpp Load_PCM() | 새 부호 구성 대상이나 col weight 2~3로 Prime ECC 고degree와 부정합 |
| H1 역행렬(dual-diagonal화) | encoder.cpp Column_Reordering_Dual_Update() | Prime ECC도 dual-update column 재배치 보유 -> 중복 |
| short cycle 제거로 BER 개선 | decoder.cpp LDPC_Decoder() | 복호 성능 영향 있으나 부호구조 유래, 알고리즘 변경 아님 |
| QC 메모리 1/L 축소 | ecc_data.h PCM_b (base+shift) | QC circulant 저장 이점, Prime ECC도 base+shift로 동일 이점 보유 |

적용 가치: 낮음 — 인코딩 효율/QC 저장 이점은 Prime ECC가 이미 dual-diagonal로 보유하고, 구성 부호는 col weight 2~3·rate 0.5~0.75로 NAND 고rate 고degree QC-LDPC와 부정합.
