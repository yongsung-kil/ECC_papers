### B. 알고리즘 요약

1. 대상은 column weight 2 LDPC(=cycle code)로, connected simple graph의 incidence matrix가 `H`가 되고 rate `R = 1 - (|V|-1)/|E|` (예: 5/9, 1/3).
2. 문제: large-girth cycle code는 복호는 쉽고 성능 좋으나 인코딩이 복잡([8],[9]의 upper-triangular 변환도 여전히 복잡).
3. 핵심 아이디어: 그래프에 Hamiltonian cycle이 존재하면 그 cycle을 canonical bi-diagonal 행렬 `Hc`로 매핑하고, 나머지 chord edge는 `Hi`로 매핑.
4. 첫 기여: redundant row 삭제 후 `H=[Hi,Hc]` 형태가 RA(Repeat-Accumulate) 코드가 되어, `Hc`가 lower-triangular bi-diagonal이라 `p^T = Hc^{-1}(Hi·m^T)`로 선형시간 인코딩.
5. `Hc`가 bi-diagonal이므로 역행렬 연산이 단일비트 accumulator(register+XOR) 한 개로 구현 — upper-triangular([9])보다 단순.
6. 둘째 기여: LCF(Lederberg-Coxeter-Frucht) 표기 + direction vector `d`로 Hamiltonian 그래프/`Hi`를 online 생성 → 행렬 저장을 정수 몇 개+비트 벡터로 축약.
7. 예: 2520정점·girth 17인 3-regular 그래프를 정수 14개로 표현(직접 저장 `2(m-n)·⌈log2 n⌉` bits 대비 대폭 절감).
8. 인코더 구조 A: `n`개 accumulator, `2(m-n)+1` 클록에 정보비트를 LCF/procedure로 지정된 1~2개 accumulator에 누적(직렬).
9. 구조 B: `Hc`의 h1,n을 0으로 치환해 rank `n`의 modified RA code(rate `1-n/m`)로 만들어 rate를 규칙화, accumulator 1개만 추가.
10. 결과: girth 34, rate 1/3, length 3780 코드가 BER 10^-6에서 기존 구조 cycle code[6] 대비 1.5dB gain, (3,17)-cage[7]와 동등 성능이나 인코딩은 훨씬 단순.
11. 한계: gate count·throughput·HW 합성 없음(개념적 accumulator 구조만), 저rate·column-weight-2 한정.


적용 가치: 낮음 — column-weight-2 저rate cycle code의 인코더 전용 구성으로, NAND용 고rate binary QC-LDPC(dual-diagonal 인코딩 이미 보유)와 구조가 불일치한다.
