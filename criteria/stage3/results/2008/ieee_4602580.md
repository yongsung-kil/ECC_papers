### B. 알고리즘 요약

1. 대상: BIAWGN용 구조적 비정규 binary LDPC. rate 1/2·3/4, `N`=576/768/1056/2304. 목적: 짧은·중간 길이에서 좋은 min distance·stopping set 구조를 갖는 구조적 부호 설계.
2. 문제: density evolution은 degree distribution 등 고수준 정보만 줘 유한길이 부호의 min distance·stopping set을 못 다룸. 기존 대수적 구조부호는 정규라 capacity 접근 성능 부족.
3. 구조: `H=(Ĥ|H̃)`. `Ĥ`는 bidiagonal(zigzag) 정사각행렬 → degree-2 노드를 loop-free로 배열(IRA/accumulate 일반화). `H̃`는 열마다 1이 3개 이상.
4. 부호어 제약: `ĉT=Ĥ⁻¹H̃c̃T`, zigzag 덕에 `y=Ĥ⁻¹x` weight을 `S(x)=Σ(x_{l-2j}−x_{l-2j-1})`(식1)로 계산 가능.
5. `H̃=ΠA`: `Π`는 permutation(블록 인터리버 `Π(i)=(i mod p)s+⌊i/p⌋`), `A`는 `p×p` cyclic shift 행렬 `Δ^{pij}`와 zero 블록의 블록행렬. 쌍 `(φij,pij)`가 PCM을 완전 규정(template matrix `P`로 compact 표현).
6. `Π` 설계 목표: 저weight `c̃`가 `ΠAc̃T`에서 인접한 1들로 뭉쳐 저weight `ĉ`로 변환되지 않도록.
7. 최적화(랜덤 탐색): 각 열블록 `j`마다 `(φij,pij)`를 무작위 생성 → 최소 ACE≥`η` 보장(단주기 loop 억제) → `wt(c̃)≤w`인 입력의 출력 weight을 (1)로 계산해 `Smin` 최대화 → 반복 후 최적값 선택. `d̂=min_j d̂_j`가 min distance 상계.
8. 부수 제약: length-4 loop 제거조건 `(p_{i1j1}−p_{i2j1})+(p_{i3j2}−p_{i4j2})≡0 mod p` (식2), `wt(c̃)=1`일 때 `ĉ` weight `W_j=Σ(φ+sp)차` 하한 강제로 저weight 부호어 배제. cyclic shift 대칭으로 검사할 벡터 수 축소(`w≈5`면 충분).
9. 결과: rate1/2·3/4 부호가 IEEE 802.16 부호를 상회, 특히 고SNR서 최대 0.2dB gain, `FER=5e-5`까지 error floor 없음(Fig4,5). 최적화 40분(Athlon-2500). 한 template로 여러 길이 부호 family 규정 가능.
10. 한계: 순수 부호설계(HW·복호기 미설계), min distance는 저weight 입력만 본 근사, 인코더 latency가 순수 template부호보다 다소 높을 수 있음.
