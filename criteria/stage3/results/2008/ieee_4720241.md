### B. 알고리즘 요약

1. 시스템: BIAWGNC, binary irregular LDPC, code-decoder pair를 `(λ(x), ρ(x), β)`로 정의(`β`=MS를 수행하는 체크노드 비율, 나머지는 SP).
2. 문제: SP는 노이즈 분산 추정오차에 민감, MS는 노이즈 파워를 몰라도 됨 → 채널 추정오차 하에서 어느 한쪽만으로는 최적 아님.
3. 모델: 채널 추정오차 `α = σ_est^2 / σ_act^2`, `α ∈ [αmin, αmax]` 범위(예: ±3dB → `[0.5, 2]`).
4. 핵심 기법: robustness measure로 수정된 decoding threshold `(Eb/N0)_thr = max_{αmin≤α≤αmax} (Eb/N0)*` 정의 — 범위 내 최악 threshold.
5. 핵심 식(2): 최대 threshold는 `α`의 경계(αmin, αmax)에서만 발생 → density evolution을 두 번만 돌리면 됨(전 구간 스캔 불필요).
6. 최적화: `βopt = argmin_β (Eb/N0)_thr` (식3)로 고정 코드에 대한 최적 혼합비 도출.
7. 확장: `β`와 `(λ(x), ρ(x))`를 번갈아 최적화하는 iterative joint code-decoder 설계(보통 4회 미만 수렴).
8. 학습/최적화: DE 기반, 9-bit message 양자화, max LLR 30, max VN degree 15로 R=0.5 code-decoder 최적화(`βopt=0.18`).
9. 결과: (x2,x5) 예에서 pure MS 1.70 dB / pure SP 2.04 dB 대비 βopt=0.39로 1.55 dB; ±3dB mismatch에서 약 0.4 dB threshold 개선, length `10^4` BER 곡선으로 확인.
10. 한계: HW 미설계, AWGN 노이즈분산 mismatch 특화, SP 체크노드(tanh)를 요구.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MS 체크노드 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 부분만 대응 |
| SP(tanh) 체크노드 | 대응 없음 | Prime ECC는 min-sum 전용(full-tanh SP 미지원) |
| 이터레이션 루프 / β 혼합 스케줄 | `decoder.cpp` `LDPC_Decoder()` | 체크노드별 알고리즘 선택 훅 필요 |
| 채널 추정오차/LLR 강인성 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR과 목적 유사(이미 보유) |
| irregular 부호 degree 분포 설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix 고정이라 재설계 부담 |

적용 가치: 낮음 — 핵심 기법(SP/MS 혼합)이 full-tanh SP를 요구해 min-sum 전용 엔진에 비친화적이고, robustness 목적은 기존 적응형 LLR과 중복.
