> 총평: linear-time 인코딩 가능한 serial-turbo(staircase) LDPC를 비트묶음 nonbinary BP로 복호해 구조적 사이클을 회피·error-floor 계층을 복원하나, non-binary·RA 특화라 binary QC-LDPC NAND ECC엔 이식성 낮음.

### B. 알고리즘 요약

1. 대상: serial-turbo 구조 LDPC(`Repr`+interleaver+`Sums`+recursive conv `ψ(D)=(A+BD)^-1`), rate `R=1/2`(r=4,s=4), blocklength 300/600, AWGN(Eb/N0), 최대 50 iter.
2. 문제: 이들 코드는 linear-time 인코딩 가능하나 standard binary BP로 복호 시 성능 나쁘고 설계 파라미터 `d2ψ`(inner encoder 유효자유거리)와 무관하게 동작.
3. 모델: parity matrix `[H_N K_N]`, `K_N`은 staircase(A 대각/B 하대각), `H_N`은 행당 s개·열당 r개 1의 sparse 행렬(2-cycle 없는 interleaver 앙상블).
4. 핵심기법(이론): serial turbo ML 분석으로 word error `Pw(e)` 상·하한 도출, 지수 `μ=(r+1)/2`는 r에만, 계수 `d*`는 inner encoder `d2ψ`에 의존(Thm 1).
5. 의미: `d2ψ`(입력무게2 error event 최소무게)가 medium-high SNR error-floor를 지배 — accumulator(1+D)^-1는 d2ψ=1, (1+D+D^3)^-1는 d2ψ=4.
6. 관찰: standard BP는 staircase 구조부의 N-2개 연쇄 6-cycle 때문에 수렴 실패, ML 계층을 못 살림(Ex.2에서 d2ψ 큰 코드가 더 나쁨).
7. 핵심기법(디코더): k개 연속 비트를 하나의 nonbinary 노드(Z_2^k)로 묶은 factor graph(information/parity/check 노드) 구성 -> 구조부에 짧은 사이클 없음.
8. 알고리즘: 이 그래프 위 sum-product BP, parity/check 간 메시지는 Z_2^k 분포, information/check 간은 Z_2 분포, check 업데이트는 라벨 A,B 활용.
9. 결과: nonbinary BP가 standard BP를 크게 상회(Ex.2 극적), error-floor 영역에서 d2ψ 계층 이론과 일치(저SNR에선 계층 역전), iter당 복잡도 `2^(2k+1)·(r/ks)N` (binary 대비 k에 지수적).
10. 한계: HW 미설계, non-binary·특정 serial-turbo 구조 한정, 짧은 blocklength, irregular 확장은 future work.

| sum-product 메시지패싱 | 대응 없음 | Prime ECC는 tanh 대신 min-sum 근사 사용 |
| serial-turbo/staircase(RA) 부호구조·inner encoder 설계 | 대응 없음 | QC-LDPC 아님, [ecc_top.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()`의 QC 구조와 불일치 |
