### B. 알고리즘 요약

1. 시스템: `[P,Q]`-CR 부호 = `(P·(P+Q), P^2)` 선형블록부호, rate `R=P/(P+Q)`, AWGN+BPSK, 짧은프레임(<500bit) 고rate 대상.
2. 문제: 터보는 복잡도 과다, 짧은프레임 고rate LDPC는 구성 난이 → 저복잡·고rate 부호 필요.
3. 모델: `P×P` 정보비트 행렬을 동심원에 배치·위상회전, radial XOR로 검사비트 생성(식 3), H는 sparse `[A I; I A^T]` 형태.
4. 핵심 기법: 부호구성에서 블록크기 `P`를 소수로 강제 → uniform checking(각 비트가 각 차원서 정확히 1회 검사)으로 최소 해밍거리↑.
5. 핵심 식: `nk,i = ⊕_s m_{s,(i+k·s)P}` (modulo-P 인덱스) — 소수 P일 때 `(P-i)(P-k)/P` 비정수 보장이 uniform 검사 근거.
6. 확장: CR은 Q=1이면 SPC, Q=2면 2D 패리티체크로 환원되는 일반화 다차원 패리티부호.
7. 복호: Tanner 그래프상 iterative sum-product(SISO) — 초기 우도 `f±`, 제약노드측 `Δψ` 곱(식10), 변수노드측 정규화 갱신(식11~12), 식(5) 만족/최대 iter까지 반복.
8. 구현: Tanner 그래프 구성에 행렬 A만 필요 → 메모리 비용 절감, 인코딩 복잡도 O(K) 선형.
9. 결과: BER=10^-5서 SPC 대비 0.5dB↑, 터보(CTC)와 동급(10^-6 이하선 0.2dB↑, error floor 더 늦음), 소수 P가 비소수 대비 0.5~1.0dB↑; iteration은 30(CTC 6/SPC 8보다 많음).
10. 한계: HW 미설계·시뮬만, 짧은프레임 무선 특화, sum-product(full BP) 사용, 부호구성이 QC구조 아님.

