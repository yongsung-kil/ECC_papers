### B. 알고리즘 요약

1. 대상은 multi-Gbit/s fully/partially parallel binary LDPC 디코더의 배선 혼잡·throughput 문제 (10GBase-T, DVB-S2, WiMAX 표준용).
2. 문제: fully parallel 디코더는 랜덤 PCM 때문에 global interconnect가 혼잡, timing/전력 악화. 어떤 LDPC 부호에도 적용 가능하고 정정성능 저하 없는 기법을 목표.
3. sum-product를 `S = f(...)`, `P = g(...)` 형태로 재작성해 check/variable 노드 연산을 (3)~(6)으로 분해.
4. Half-broadcasting: check 노드가 이웃 VN들에게 개별 메시지 대신 **단일 메시지 `P`** 하나만 broadcast, 나머지 재구성 연산은 VN으로 이동 (message-passing 결과 불변).
5. 효과: fully parallel에선 배선 공유로 node간 wirelength 26% 감소(2048-bit, 1.88→1.40 mm, HW 오버헤드 0), partial parallel에선 CPU write가 `d`→`1`로 감소해 메모리 접근 24% 감소.
6. Full-broadcasting: check·variable 양방향 모두 broadcast, inverse 연산 중복으로 최대 2x 로직 오버헤드지만 배선 추가 축소(메모리 접근 45% 감소).
7. 워드길이 주의: broadcast 값(`P` 등)은 clip된 extrinsic보다 1~2bit 넓어야 할 수 있음(hard-decision 1-bit 메시지면 XOR라 증가 불필요).
8. Block interlacing: 두 연속 프레임을 교차 스케줄 — 한 프레임 VN update 시 다른 프레임 CN update. 유휴시간 제거로 HW utilization↑, throughput 최대 2배 목표.
9. 결과: (2048,1723) RS기반 (6,32)-regular hard-decision(32iter) 디코더 throughput +60% @ +5.5% gate, (660,480) (4,15)-regular PEG min-sum(4bit,16iter) 디코더 throughput +71% @ +9.5% gate (0.18um CMOS, post-layout).
10. 한계: 정정능력·부호·복호알고리즘 무변(순수 아키텍처), pipeline register가 critical path를 정확히 반분 못해 2배엔 못미침, 절대 gatecount/throughput은 표에만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| broadcasting (node간 배선/edge 축소) | [GT.cpp](../../Prime_ECC_3.1_Claude) `Make_GT_HW()` | MUX 기반 edge 축소와 개념적 유사, 다만 GT는 이미 보유 — 중복성 있음 |
| 디코더 이터레이션 루프 / pipeline | [decoder.cpp](../../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | block interlacing은 이 루프에 2-frame 교차 스케줄 추가 — throughput 참고 |
| CN min-sum 연산 재분할 | [decoder.cpp](../../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | broadcasting은 알고리즘 불변, 연산 배치만 이동 |
| partial-parallel 메모리 접근 | 디코더 HW 모델 (z=32) | 대응 위치 유사하나 논문은 fully-parallel 중심이라 직접 이득 제한 |

적용 가치: **중간** — HW throughput/interconnect 기법으로 참고할 만하나, broadcasting은 fully-parallel 배선 절감에 특화되고 Prime ECC의 GT/pipeline과 개념 중복이며 정정성능 개선은 없음. block interlacing의 2-frame 교차 아이디어만 throughput 관점에서 재검토 가치.
