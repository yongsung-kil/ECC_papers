### B. 알고리즘 요약

1. 대상: (n,k) binary LDPC (regular/irregular 모두), BPSK+AWGN, 예시로 DVB-S2 `n=64800`, rate `3/4`.
2. 문제: SPA belief-propagation은 H를 행/열로 접근하는 매우 불규칙한 메모리 접근이라 dedicated VLSI 외에 범용 stream 아키텍처로 매핑이 어렵다.
3. 핵심 기법 1단: SPA를 두 커널로 분리 - Kernel1(horizontal, CN→BN 갱신 식(1)(2)), Kernel2(vertical, BN→CN 갱신 식(3)(4)).
4. 핵심 기법 2단(자료구조): H의 non-null edge만 행기준(`HBN`)·열기준(`HCN`) 두 stream으로 코딩, 각 원소가 다음 원소 주소를 pointer로 보유하는 **circular linked list**.
5. 순환주소 의미: 시작점이 무의미해져 edge 하나당 프로세서 하나 할당 등 높은 병렬성 확보, 같은 (n,m) edge가 HBN/HCN에서 동일 위치를 점유해 pointer swap만으로 in-place 반복.
6. flooding schedule 채택으로 모든 BN 갱신 완료 전 다음 갱신 없음 → 진정한 병렬 실행 보장.
7. 구현 트릭: folded SDF(2커널을 MUX로 접어 program memory 축소), 여러 codeword 동시 복호(SSE 4-way packed / GPU pixel processor).
8. 플랫폼: x86 Core2Duo(SSE, C/VS2003), NVIDIA 8800 GTX GPU(OpenGL GLSL + Caravela).
9. 결과: brute-force 대비 speedup ≈ (H원소수/edge수) (예 999×111에서 SSE로 3.7배, 128ms→34ms/codeword@100iter); GPU는 16000 edge에서 100iter <109ms vs CPU-SSE ~300ms. Storage Compression Ratio DVB-S2에서 2733배(<0.04%).
10. 한계: 정정성능(BER) 개선은 정성 언급뿐 데이터 없음, HW(RTL/ASIC) 미설계, 부동소수 GPU/CPU 가속 전용.
