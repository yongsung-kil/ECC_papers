### B. 알고리즘 요약

1. 대상: QC-LDPC를 Mother matrix `M(H)`(0/1)와 Shift matrix `S(H)`(shift값)로 표현, circulant 크기 `t×t`.
2. 문제: 기존 고girth 구성법이 임의 code length를 못 내고 각 부호 H를 모두 저장해야 해 메모리 과다.
3. 핵심 정의: mother code `H0`(circulant `t×t`)를 압축률 `q∈(0,1)`(`qt` 정수)로 압축 → son code `H'`의 circulant는 `qt×qt`, `M(H')=M(H0)`.
4. Proposition[7]: `M(H)`의 2l-cycle 집합 `(s1..s2l)`에서 `r·Σ(-1)^i s_i ≡ 0 mod t`의 최소 `r`이면 그 2l-cycle이 H에서 2lr-cycle을 유발.
5. Theorem: `S(H')=qS(H)`, `M(H')=M(H)`이면 `H'`의 girth는 H와 동일하고 cycle 수는 q배 - 즉 짧은 부호를 girth 유지·cycle 감소로 생성.
6. recursive-compressing: `s'_{ij}=⌈q·s_{ij}⌉`(H1)와 `⌊q·s_{ij}⌋`(H2) 두 후보 생성 → cyclic 특성(girth·최단 cycle 수) 우수한 쪽 선택, 저장한 위치(i,j)에서 ±1 국소 조정으로 개선, 생성된 son을 다시 mother로 재귀.
7. single-compressing: 1)~7) 동일, 8)에서 같은 mother에 q만 바꿔 여러 son code 생성.
8. 성능 근거식 (12): girth `g`와 column weight `j`로 `d_min` 하한 - girth 클수록 `d_min` 크고 short cycle 적을수록 iterative decoding 유리.
9. 결과: 3D lattice mother(wt-3, rate 0.4), AWGN·BPSK·log-SPA 100 iter; 예) N=3000 son이 SNR 2dB에서 BER 10⁻⁷ error floor 없음, N=1500 son은 girth 8로 유지, 직접구성 3D 부호와 유사 성능.
10. 한계: rate 0.4·column weight 3 고정, HW 없음, girth는 유지(일부 son은 8로 하락), 실질 기여는 저장 절감·가변길이지 정정능력 향상 아님.
