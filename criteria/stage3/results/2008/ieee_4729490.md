### B. 알고리즘 요약

1. 시스템: 100Gb/s+ 광통신 FEC로 girth-10 QC-LDPC(24015,19212) `rate=0.8`, AWGN 채널에서 평가.
2. 문제: 100Gb/s 이상에서 고정밀 A/D 컨버터가 존재하지 않아 soft-reliability 기반 SPA/TPC LDPC 디코딩을 그대로 구현 불가.
3. 핵심 아이디어: 확률 `P`를 길이 `N` Bernoulli 시퀀스(1의 개수 `m`, `P=m/N`)로 표현하면 SPA의 곱/나눗셈을 hard-decision과 동일한 단순 Boolean 회로로 처리 가능.
4. 절차: 0)초기화 `qi->j=Pi=[1+L(vi)]^-1`, 1)check node `rj->i=ql(1-qm)+(1-ql)qm`, 2)variable node `qi->j=rs·rt/[rs·rt+(1-rs)(1-rt)]`, 3)VN 업데이트 후 `vH^T=0` 또는 최대 iteration까지 반복.
5. degree-3 한계: 기존 방식은 hidden state를 넣어 bipartite graph를 degree-3으로 변형 → 그래프 복잡도 증가.
6. 확장: step1)-2)를 step3)처럼 처리하여 hidden state 없이 arbitrary column/row-weight 노드에 적용, 그래프 변형 불필요.
7. latching(hold-state) 문제: 큰 girth 코드에서 노드가 고정 상태에 잠겨 성능 손실 발생.
8. 해결: second half-iteration만 `3-bit` precision으로, 나머지 단계는 stochastic logic으로 구현.
9. 부호설계: error floor 회피 위해 girth-10 QC-LDPC 설계(girth-10은 BER 10^-15까지 error floor 없음).
10. 메모리: `rj->i`, `qi->j` 두 확률만 저장(표준 SPA는 4개 확률+정규화 필요) → 저장량 절감.
11. 결과: BER 10^-7에서 RS(255,239) 대비 4.75dB, concat RS 대비 3.35dB 이득, net effective coding gain 10.95dB @ BER 10^-12 (30 iter, Bernoulli length 1000). 단 latency는 conventional SPA보다 증가, 시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stochastic 디코딩 알고리즘(Bernoulli Boolean 연산) | 대응 없음 | min-sum 기반 `decoder.cpp`와 연산 패러다임 자체가 다름 |
| 이터레이션 루프 구조 | `decoder.cpp` `LDPC_Decoder()` | 루프 골격만 유사, CN/VN 연산은 미이식 |
| girth-10 QC-LDPC 부호설계 | `ecc_top.cpp` `Load_PCM()` | 고rate QC-LDPC이나 rate 0.8/길이 24015로 NAND ECC와 규격 상이 |
| LLR 초기화 / 3-bit precision | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | soft 입력 양자화 관점만 부분 참조 |

적용 가치: **낮음** — 광통신 고속용 stochastic 디코딩은 min-sum 기반 Prime ECC와 HW 패러다임이 다르고 latency도 악화되어 떼어 쓸 부분이 없음.
