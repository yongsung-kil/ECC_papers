### B. 알고리즘 요약

1. 대상: binary LDPC min-sum 복호의 CNU/VNU HW, 임의 입력수 n(check node degree), 임의 code length.
2. 문제: 기존 CNU는 (a)conventional[6] min/2nd-min sorter가 critical path `⌈log2 p⌉+⌈log2(p-1)⌉`로 김, (b)efficient[7] exclusive CNU는 6-input 전용이라 일반 n으로 확장 불가.
3. 채택 알고리즘: SPA를 LLR로 근사한 **Min-Sum** 식(3) `Lc→v = (∏sign)·min_{k≠j}|L|` - CNU 각 출력은 자신 제외 입력들의 최소값.
4. 핵심 기법(Cyclic comparison): 각 CNU 출력 `Min{1..n}\{i}`들이 큰 중복 부분집합을 가짐을 이용, 입력을 순환형으로 배열해 층별로 부분 min을 재사용.
5. 층 구성: layer1은 `Min[1,2],Min[2,3],…,Min[n,1]` (n개), 이후 layer마다 `output[l][i] = output[l-1][i%n] ∪ output[l-1][(i+2^{l-1})%n]` 병합.
6. 확장: comparator를 2-input adder로 치환하면 동일 구조가 VNU 식(1) `L_{v→c}=channel+Σ L_{c→v}`에도 적용.
7. 결과(복잡도): CNU당 `n×⌈log2(n-1)⌉` comparator, `⌈log2(n-1)⌉` 층(각 층 n개) → conventional 대비 critical path 단축(공식상 최소 층수), 대신 space는 증가.
8. 검증: 문헌에 동일 주제 없어 conventional[6]과만 time/space complexity 공식 비교(Table 1). 합성/FPGA/BER 없음.
9. 특징: 손으로 최적화 불가한 긴 code length·큰 degree CNU에 체계적 near-optimal 구조 제공.
10. 한계: 정정성능·iteration은 min-sum 그대로(불변), area가 conventional보다 큼, gate/공정/throughput 수치 없음.
