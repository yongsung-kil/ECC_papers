### B. 알고리즘 요약

1. 목표: BEC에서 반복복호 성능을 지배하는 **최소 크기 stopping set**을 주어진 LDPC 부호에 대해 계산 (부호길이 약 500/1000/5000).
2. 문제: 최소 stopping distance 판정은 NP-hard로 추정되며 exhaustive search는 `Σ C(n,i)` 지수복잡도(예 n=4896,smin=24 → 약 2^215)로 불가능.
3. 핵심관계: stopping set `S`는 부분행렬 `HS`에 weight-1 행이 없는 열집합. 부호어 `c`의 support는 항상 stopping set(HS 모든 행이 짝수 weight) → **저중량 부호어 ⊂ stopping set**.
4. 핵심기법: H의 행을 무작위로 `ρ`개 골라 `Y`, 나머지 `m-ρ`행을 `X`로 분할. `HS`의 홀수-weight 행들이 모두 `I`(선택된 ρ행)에 포함되면, `X`의 저중량 부호어가 곧 stopping set의 SS vector `xS` (`X·xS^T=0`).
5. `X`의 저중량 부호어 생성에 **Stern 알고리즘**(확률적 저중량 부호어 탐색)을 사용해 작은 stopping set을 능동적으로 생성.
6. 검증: 생성된 `xS`가 실제 stopping set인지 `YS`에 weight-1 행이 없는지로 확인. `rmax` 반복하며 최소크기 `s` 갱신.
7. 확률분석: 목표 stopping set 발견확률 `τ = θ·π`(θ=홀수행 포함확률, π=Stern 발견확률), 실패확률 `ε_smin=(1-τ̄)^rmax`로 신뢰도 정량화.
8. 복잡도: Stern을 `(n,k+ρ)` 부호에 적용한 bit operation 수 `Nρ,p,l`; work factor `W=N/τ̄`는 C1~C4에서 약 2^37~2^52.
9. 결과: MacKay/PEG/Ramanujan-Margulis 부호에 적용, C3(1008,504)의 최소 stopping set을 기존 28이 아닌 **26**으로 새로 발견; C1/C4는 기존값과 일치하나 오류확률이 더 작음.
10. 한계: BEC 전용, 계산시간 방대(C3 3500h, C4 2288h), 디코더·부호설계·HW를 다루지 않는 순수 분석 알고리즘.
