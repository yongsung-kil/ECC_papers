### B. 알고리즘 요약

1. 대상은 표준 `(c,d)`-regular LDPC 앙상블(Tanner 그래프, 소켓 랜덤 매칭)과 Sipser-Spielman **parallel bit-flipping** 디코더(Algorithm 1).
2. 문제: 기존 결과는 left degree≥5에서만 "선형(블록길이 비례) 개수의 worst-case 오류 정정" 보장; 이를 개선·확장.
3. 채널이 비이진이면 **1-bit로 양자화** 후 flipping 적용(hard-decision). 각 변수노드는 unsatisfied check 이웃이 `c/2` 초과면 flip, 정확히 `c/2`면 확률적으로 결정.
4. 핵심 기법: 반복마다 변수노드를 bad/good × (unsatisfied이 `c/2`보다 적음/같음/많음)으로 분할하고, 반복 후에도 bad로 남는 집합과 good→bad 전이 집합을 **동시에** 분석(기존 [2],[3] 대비 개선 지점).
5. "possibly bad" configuration(오류가 줄지 않을 수 있는 tuple)을 union bound + 다항식 생성함수로 열거, saddle-point 부등식으로 확률 상한 산출.
6. Theorem 1(left degree>4): 함수 `g(·)`의 최소 양근 `δ*`로 정정 가능 worst-case 오류율 하한 도출 → expander graph 기반 [3] 대비 수 자릿수 향상.
7. 확장(Section III, left degree=4): 두 phase(오류 많음/적음)로 나눠 각 반복 오류 감소를 증명, Theorem 2로 degree≥4까지 선형 정정 보장.
8. left degree=3은 병렬간선(size-2 cycle) 존재로 단일 worst-case 오류도 정정 불가한 코드가 비무시적 비율 존재(trapping set [15] 인용) → degree 4가 하한.
9. random error 모델: 첫 반복에서 오류가 랜덤 위치라는 점을 이용해 first-iteration 경계를 개선, `-good` 코드 결합으로 더 강한 확률 보장.
10. 결과·한계: 모두 점근/유한길이 **해석적 경계**뿐, 시뮬레이션·HW·gate count 없음; 복잡도는 반복당 O(n), 총 O(n) / O(log n) 반복(정성).
