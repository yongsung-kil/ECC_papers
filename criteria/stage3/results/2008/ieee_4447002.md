### B. 알고리즘 요약

1. 시스템: BPSK/AWGN, rate 1/2 (2048,3,6) regular LDPC-CC(무한 주기 시변 PCM), Ethernet 프레임 640B~8064B, LLR 8bit.
2. 문제: LDPC-CC는 iteration마다 프로세서를 매핑하면 강한 코드(큰 code memory)에서 프로세서·메모리 비용 폭증.
3. 핵심 기법: 단일 프로세서 하나만 두고 종단(termination)된 프레임을 CRC 통과 또는 max iteration까지 반복 순환(serial in iteration dimension).
4. 구조: 무한 Tanner 그래프의 sliding-window(폭 `T·(m_s+1)`) 처리를 시간축으로 unwrap → 공간축 프로세서로 매핑, 단일 프로세서는 1 CNU + 2 VNU.
5. 메모리 설계: FIFO를 dual-port RAM 원형버퍼로 구현, 8개 뱅크(정보/패리티 LLR 분리), 원형버퍼를 base clock의 3배로 over-clock해 1 base cycle당 1 라운드 메시지 교환.
6. CN 처리: min-sum을 개선한 degree-matched check-node 근사[35] 사용(HW 단순, sum-product 대비 소폭 손실).
7. 조기종료: 매 loop마다 4-byte CRC로 정상복호 판정 → 통과 시 중단(고 SNR서 프레임당 2~3 iteration).
8. 확장성: 프로세서 수 p와 max loop n을 synthesis 변수화, p 증가 시 throughput은 선형 미만 증가(plateau).
9. 결과: Altera Stratix FPGA서 수천 LE + ~800Kb RAM만 사용, max 100 iteration서 최대 9Mb/s, (2048,3,6) LDPC-CC가 길이 10만 (3,6) LDPC-BC 능가, 급경사 waterfall·error-floor 미관측.
10. 한계: throughput 낮음(9Mbps), min-sum 근사로 0.1~0.2dB 손실, 짧은 프레임서 termination 오버헤드 큼, ASIC은 향후과제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 직렬 단일프로세서 sliding-window 아키텍처 | 대응 없음 | Prime ECC는 block QC-LDPC z=32 병렬 파이프라인, 컨볼루셔널 구조와 무관 |
| min-sum 개선 CN 근사 | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()` | Prime ECC 이미 min-sum 보유(중복) |
| CRC 기반 조기종료(loop stop) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 이미 보유(중복) |
| 8bit LLR 양자화 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | soft-read LLR 폭 개념 접점 |
| LDPC-CC(SC-LDPC) 부호구조 | 대응 없음 | Prime ECC는 block QC-LDPC 고정 |

적용 가치: **낮음** — 컨볼루셔널 코드 전용 직렬 아키텍처가 block QC-LDPC에 이식 불가하고, 재사용 가능 요소(min-sum 근사·CRC 조기종료)는 Prime ECC가 이미 보유.
