### B. 알고리즘 요약

1. 대상은 LDPC 컨볼루션 부호(LDPC-CC)의 인코더; rate-1/2 `(3,6)`-regular, code memory `m_s=128/512`(검증은 128~2048, rate 1/2~5/6), 90nm CMOS.
2. 문제: LDPC-CC는 유한 프레임에 적용 시 종단(termination)이 필요한데, 기존 single-phase 방식은 특정 phase 도달용 padding이 필요해 rate 손실·비용이 큼.
3. 모델: syndrome former `H^T`로 정의된 시변(주기 `T`) 부호를 FSM 인코더로 보고, 종단은 trellis를 all-zero state로 되돌리는 문제로 정식화.
4. 핵심기법 (all-phase termination): 종단 시퀀스를 GF(2) 선형계 `(6)/(7)` `Lx=state`로 풀되, 종단행렬의 **첫 행만** 저장해 매 state transition마다 on-demand로 종단비트를 1개씩 생성.
5. 핵심식: 종단길이 `L`은 `(7)`이 모든 phase에서 consistent하도록 `m_s`보다 약간 크게(대개 `<5%` 증가) 선택; free variable을 0으로 고정하면 detour 없이 `L` 스텝 내 partial-zero state 도달 보장(Appendix II 증명).
6. 확장: 종단행렬 크기 `2m_s x T`가 프레임길이와 무관 → 저장 공간복잡도 `O(m_s^2)`; single-phase 대비 padding 제거로 rate 손실 최소화.
7. HW 디테일: phase mask(one-hot shift-register FSM)로 시변 연결 제어, 파이프라인 XOR tree(3/9 stage)로 임계경로 단축, 종단회로는 인코딩회로와 phase mask 제어만 다름.
8. 검증: SW로 100만+ 프레임 무작위 종단 검증(실패 0), STM 90nm 표준셀 합성.
9. 결과: 최대 throughput 약 1Gb/s(1.2GHz), (512,3,6) 인코더 트랜지스터 약 350K/면적 약 0.55mm²; termination 추가로 면적 1.8배(128)~6.4배(512) 증가하나 디코더 대비 여전히 작음; [22] 대비 클럭 189MHz→1.2GHz 개선.
10. 한계: 순수 serial 구조(병렬화 미구현, future work), 시변 LDPC-CC는 종단 복잡도가 `m_s^2`로 커 대형 `m_s` 부적합; 합성까지만(실측 없음).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-CC 직접 인코딩(syndrome former) | encoder.cpp LDPC_encoder_4KB() | code family 상이(convolutional vs QC block), 인코딩 원리 다름 |
| all-phase termination / 종단행렬 | 대응 없음 | QC-LDPC block 부호는 종단 개념 자체가 불필요 |
| 종단 phase mask / XOR tree HW 구조 | 대응 없음 | dual-diagonal 인코더(Column_Reordering_Dual_Update)와 무관 |
| GF(2) 역행렬(G-J elimination)로 인코딩 | encoder.cpp Generate_PCM_encoding() | 역행렬 기반 인코딩이라는 일반 개념만 유사, 구조 이식 불가 |
```

적용 가치: **낮음** — LDPC 컨볼루션 부호(SC-LDPC 계열) 인코더의 종단(termination) 특화 HW로, NAND 컨트롤러의 QC-LDPC block dual-diagonal 인코더(Prime ECC)와 code family·인코딩 방식이 근본적으로 달라 이식 불가.
