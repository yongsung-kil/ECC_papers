### B. 알고리즘 요약

1. 시스템: BI-AWGN 채널 + SPA(sum-product) 복호를 baseline으로, 대상 부호는 rate-1/2 `(2640,1320)` Margulis 코드와 rate-0.3 `(640,192)` 짧은 QC 코드(circulant `Q=64`, col-weight 5).
2. 문제: LDPC error floor는 low-weight codeword가 아니라 `(w,v)` trapping set(부분그래프)에서 발생하며 storage/광통신처럼 극저 error rate 요구 시 핵심 장애.
3. 모델: 시뮬레이션으로 dominant trapping set을 열거(Margulis: `(12,4)`가 floor의 75%, `(14,4)`가 23%; QC: `(5,5)`가 90%+), Richardson의 semi-analytic + importance sampling으로 FER floor 예측.
4. 기법1 bi-mode(PP): SPA 수렴 후 syndrome weight가 target trapping set의 것과 일치하면 2모드 진입 → unsatisfied CN 이웃 VN들을 tree로 erasure flag 후 그래프 기반 erasure 복호(이진 XOR)로 trapping set 오류 정정.
5. 핵심 아이디어: erasure set이 stopping set만 포함하지 않으면 BEC 반복복호로 trapping set VN을 모두 복구 — 오류 bit LLR이 최소가 아닐 수 있어 unsatisfied CN의 `dc` 이웃 전부를 erasure 처리.
6. 기법2 bit-pinning+outer: 고rate outer BCH를 concat해 trapping set의 systematic bit 하나를 정정, 그 sign을 LDPC로 feedback해 해당 LLR을 max로 pin; overlap 활용해 t=1 BCH 4개로 rate loss를 0.5→0.49(0.086dB)까지 축소.
7. 기법3 G-LDPC(DM): trapping set의 unsatisfied SPC들을 묶어 generalized constraint processor(GCP)로 만들고 BCJR로 처리(SPA-GCP) — I/II/III로 GCP 그룹핑을 확장해 trapping set 제거.
8. trapping set 탐색: two-step importance sampling(bias -2 후 stable pattern 기록 → Gaussian RV로 실제 AWGN trapping set 확정)으로 G-LDPC trapping set을 효율 열거.
9. 결과: bi-mode는 Margulis FER floor를 `~10^-8`까지 소거·QC는 2자릿수↓, bit-pinning은 시뮬 한계 이하로 소거, G-LDPC III는 Margulis/QC 모두 floor 미관측(`~10^-8`).
10. 한계: HW 미설계·floating point SPA 시뮬, 기법이 baseline 복호기(SPA/양자화)에 따라 trapping set이 달라짐, G-LDPC는 BCJR로 복잡도 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA/반복복호 이터레이션 루프 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | baseline은 SPA, 우리는 min-sum — 루프 위에 post-processing 훅 삽입 지점 |
| CN 연산(우리 min-sum) | `decoder.cpp` `CNU_Update_New_Mag()` | 논문 baseline과 알고리즘 상이(SPA/BCJR), min-sum trapping set 재특성화 필요 |
| bit-pinning LLR max 고정 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | pin된 bit LLR을 max로 설정하는 것과 대응 |
| trapping set 수렴실패 추적 | `corner.cpp` | 수렴실패 프레임을 잡아 trapping set 후보화하는 데 활용 가능 |
| bi-mode erasure post-processing / G-LDPC BCJR | 대응 없음 | Prime ECC에 error-floor용 post-processing·GCP/BCJR 모듈 미보유 |

적용 가치: **높음** — error-floor는 고rate NAND QC-LDPC의 핵심 병목이고 bi-mode post-processing은 rate loss 없이 XOR 기반 저복잡도라 이식 후보. 단 min-sum/양자화 하에서 dominant trapping set 재특성화가 선행돼야 함.
