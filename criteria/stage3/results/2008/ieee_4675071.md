### B. 알고리즘 요약

1. 대상은 CB-LDPC(QC-LDPC 서브클래스), `n=2304`, rate `1/2`, `H=2304×1152`, block matrix `L`(96×96 또는 12×12 등), AWGN/soft-demapping.
2. QC-LDPC는 효율적 인코딩이 가능하나 HW 구현이 어려워, HW 친화적 parity-check 구조를 새로 설계하는 것이 목표.
3. `H=[HI|Hp]`로 나누고 `Hp`를 almost-lower-triangular(I/O 블록의 계단형)로 두어 선형 인코딩과 메모리 효율 확보.
4. `HI`는 cyclic permutation block `Lx,d=Lx,1^{(d-1)ax}` (rotation)으로 구성, `d1≠d2≠d3`, `L1,1<L2,1<L3,1`로 girth를 키움(`a1=2,a2=7,a3=13`, `t,s`는 L3,t·L2,s 위치).
5. girth 상승이 codeword 최소거리·복호 효율을 높여 array code 대비 약 1dB 이득.
6. 복호는 iterative Min-Sum(MSA): CN은 `sign` 곱 × 최소 magnitude(식2), BN은 `Lr(pb)+Σ Lr(rc'→b)`(식3), stop criterion(식4), 8 iteration 사용(8회 후 수렴 둔화).
7. HW는 BNU(N=n/p개), CNU(K=N(1-R)개), two-port Mem(M=비영블록수)로 구성; 각 BNU가 하나의 memory bank에 대응해 MUX를 절감.
8. LDPC HW 최대 병목인 BNU/CNU↔memory 연결(routing)을 partial parallel 대신 block parallel connection으로 구현해 MUX 수·routing 복잡도 대폭 축소.
9. 결과: 0.13µm에서 565Mb/s@100MHz, 342mW, cell area 4.3Mµm², Ref[7]/[9] 대비 전력·면적 약 절반.
10. 한계: rate-1/2·특정 CB-LDPC 구성, ASIC 미실측, soft LLR 양자화 비트폭 미제시.
