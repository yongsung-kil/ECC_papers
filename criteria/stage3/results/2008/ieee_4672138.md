### B. 알고리즘 요약

1. 대상은 WiMAX 802.16e irregular QC-LDPC (rate `1/2~5/6`, length `576~2304`, 채널 5-bit LLR).
2. 다중 rate/length를 하나의 칩으로 처리해야 해 재구성 가능·저복잡도·저전력 디코더가 필요.
3. `layered-decoding` message passing으로 H를 non-overlapping check row 블록(layer)으로 나눠 AP-LLR을 Bit-Sum memory 경유로 갱신 → 수렴 약 2배 가속.
4. CN 연산은 offset 기반 Min-Sum: bit message 중 최소 2개 크기(`qmin,qmin2`)와 그 index(`qidx`)만으로 check message 생성, offset으로 과대추정 보정.
5. `qjm/qj`는 8-bit, magnitude는 5-bit saturation으로 복잡도 축소(성능손실 미미), sign+MS message만 메모리 저장해 비트수 절감.
6. 아키텍처는 partial-parallel: 24개 CFU(최대 병렬 96의 subset)로 블록 병렬 처리, CFU 내부는 순차 연산이라 irregular check degree 수용.
7. Bit-Sum memory를 24뱅크가 아닌 6뱅크로 통합(consolidated), read port를 2배 클록으로 구동해 분산 저장 AP-LLR 회수, clock gating으로 40mW(20%) 절감.
8. message routing은 barrel shifter 기반 cyclic shifter로 forward routing만 구현(backward network 제거), 미사용 CFU엔 0 강제로 switching 절감.
9. 결과: rate-1/2 length-2304 10-iter에서 68Mbps@100MHz, 3.4mm2/165mW@0.18µm, 타 WiMAX 디코더 대비 면적 53%↓·게이트 86%↓·에너지효율 27~45%↑.
10. 한계: ASIC 제작중(post-layout 추정치), WiMAX 다중rate 특화, throughput 68Mbps로 NAND용엔 낮음.
