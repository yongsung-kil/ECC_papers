### B. 알고리즘 요약

1. 목표: binary LDPC의 log-domain SPA에서 **체크노드(CN) 프로세서**만을 저복잡도로 재설계, 벤치마크는 특허 디코더 FQ(Flarion/Qualcomm).
2. 문제: 정확한 CN 연산은 `box-plus`(Φ 보정항) 비용이 크고, FQ의 근사는 특허라 사용이 제약됨 → 특허 회피 + FQ 수준 성능/복잡도의 디코더 필요.
3. 모델: AWGN 채널, all-zeros 코드워드 송신, floating-point 수신값을 균일 스칼라 양자화기로 정수 message 로 변환.
4. 핵심 기법: `x ⊞ y` box-plus 연산자를 **2입력 look-up table**로 구현, 입력은 message 크기(magnitude)만 사용해 인덱스 접근.
5. 핵심 관찰: Φ 보정표는 (a) 인덱스 증가 시 비감소, (b) 경계에서 정확히 +1 증가, (c) 대칭 이라는 3가지 구조적 성질을 가짐.
6. 압축: 대칭성으로 절반만 저장 + 대표 열벡터 + 주대각선 기준 skip 거리로 표를 원래 크기의 일부로 **무손실 압축**(HW 구현 용이).
7. RC CN 알고리즘: 루프 1회에서 가장 신뢰도 낮은 인접 VN(포인터 β)을 찾으면서 box-plus 동시 계산 → `a-min`의 2d 비교를 d 비교로 축소(min1/min2 아이디어와 유사).
8. 양자화: 균일 step size, 소수 bit message, 양자화 오차는 대개 Φ의 미세한 over-approximation 으로 나타남.
9. 결과: 저rate IRA / 고rate IRA / Margulis 코드에 대해 RC 의 BER·FER 가 FQ 와 사실상 동일(수 dB 이내), 큰 degree CN 에서 RC 가 게이트 지연 더 빠르고 로직 게이트 수는 FQ 대비 일부만 사용(단, message 당 bit 는 RC 가 더 많음).
10. 한계: box-plus/SPA 특화(min-sum 아님), NAND/QC-LDPC 무관, HW는 게이트 수 분석 수준(공정·throughput 없음), 정확 수치는 텍스트 추출 손상으로 다수 미상.

| Φ 보정 2입력 LUT + 무손실 압축 | [ecc_data.h](../../Prime_ECC_3.1_Claude/) `PARAM_LLR` (magnitude 양자화 테이블) | 보정 LUT 개념은 닿지만 Prime ECC 테이블은 magnitude 양자화용이라 직접 대체 아님 |
