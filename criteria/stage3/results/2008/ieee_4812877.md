> 총평: IERRWBF(bit-flipping)의 syndrome oscillation 검출 조기종료로 iteration만 감소(BER 동일). 우리는 min-sum + CRC 조기종료 보유라 기능 중복 크고 BF 특유 성질 의존, 이식성 낮음.

### B. 알고리즘 요약

1. 대상: BPSK/AWGN+Rayleigh(무선) 채널의 binary regular LDPC, random(MacKay) 구성, 예제 `(250,125,3)`,`(1000,500,3)`,`(2114,1870,3)`.
2. 문제: IERRWBF는 저SNR에서 syndrome이 all-zero로 수렴 못하면 max iteration까지 무의미하게 flip을 반복해 시간 낭비.
3. 모델: RRWBF/IERRWBF의 4단계(초기화·변수노드·결정·체크노드), reliability ratio로 `En` 가중 계산.
4. 핵심기법: 체크노드 단계에 조건 추가 — 매 iteration 끝의 syndrome을 3-entry register에 저장하고 (i)와 (i-2) syndrome 비교.
5. 핵심식: `Tm=Σ|rn|`(초기화 상수), `En=Σ_{m∈M(n)} (1/|rn|)(2sm-1)Tm` (2); `Tm`·`rn`이 상수이므로 `sm(i)=sm(i-2)` → `En(i)=En(i-2)` → 같은 비트가 두 번 flip되어 디코드 벡터 반복.
6. 확장: syndrome 반복(oscillation) 감지 시 즉시 iteration loop 탈출 → 최종 디코드 벡터 확정.
7. 구현: register 크기 3(초기+1차+2차 syndrome 비교 최소값), codeword 길이 크면 전체 벡터 대신 (n-k) bit syndrome만 저장(고rate일수록 유리).
8. 학습/최적화: 없음(규칙 기반).
9. 결과: BER는 원본과 동일(무손실), 평균 iteration 대폭 감소 — 예 `(2114,1870)` max 1000 iter에서 694.95→32.22 (speedup 2056%), 코드 길이·max iter 클수록 감소폭 증가.
10. 한계: MATLAB 시뮬만, BF 계열이라 SPA/min-sum 대비 성능 열위, 2-iter oscillation은 BF 특유, HW·error-floor 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 반복(oscillation) 검출 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()` / `full_CRC.cpp` | 우리는 CRC 기반 조기종료 보유, oscillation 검출은 BF 특유 성질 의존 |
| bit-flipping(RRWBF) 복호 루프 | `decoder.cpp` `LDPC_Decoder()` | 우리는 min-sum, BF 계열 디코더는 대응 없음 |
| syndrome 계산(체크노드 단계) | `decoder.cpp` `C2V_Cal()` | syndrome/CN 연산 위치는 대응하나 min-sum 스케줄과 상이 |

적용 가치: 낮음 — BF 계열 특유의 2-iteration oscillation 검출에 의존하고, 우리 min-sum + partial/full CRC 조기종료와 기능이 중복되며 성능 개선 없이 iteration만 줄임.
