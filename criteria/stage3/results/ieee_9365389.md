### B. 알고리즘 요약

1. 시스템은 15-GBaud 16QAM optical coherent 통신이며 QC-LDPC의 parity-check matrix는 `H=[I(p_i,j)]`, circulant shift `p_i,j` 기반이다(rate 2/3~5/6, size 750×2250~1500×9000).
2. 기존 EXIT chart 기반 최적 degree distribution은 decoding threshold(무한 iteration 가정)를 최소화하지만, 실제 iteration이 제한(예: 8회)되면 성능이 저하되는 문제를 다룬다.
3. 채널/디코더 모델: VND-CND 간 mutual information(MI) 전달을 EXIT chart로 추적하는 표준 iterative decoding 모델.
4. 핵심 기법: decoding threshold 최소화 대신, 고정 iteration 수(예: 8) 이후 MI가 1에 도달하는 decoding trajectory를 기준으로 degree distribution `λ(x)=Σλ_d x^d`, `ρ(x)=Σρ_d x^d`를 선택한다.
5. 예시(rate 0.8): decoding threshold 기준 code I(2.38dB)가 code II(2.57dB)보다 우수해 보이나, 8-iteration 궤적 기준으로는 code II(필요 SNR 3.42dB)가 code I(4.22dB)보다 우수함을 EXIT chart로 보인다.
6. 확장 단계: greedy search 알고리즘([4] 인용)으로 constant check degree distribution 하에서 가능한 variable degree distribution 후보군을 먼저 생성.
7. 구현 디테일: 동일 sparsity·동일 girth(=8)로 고정해 conventional/proposed QC-LDPC를 공정 비교; hill-climbing 알고리즘([5] 인용)으로 최종 parity-check matrix 구성.
8. 학습/최적화 절차: (1) greedy search로 degree distribution 후보 생성 → (2) 제한된 iteration에서의 decoding trajectory 추적으로 최적 후보 선택 → (3) hill-climbing으로 H-matrix 구성.
9. 결과: rate 4/5, filter bandwidth 10GHz, 8-iteration 조건에서 BER 10^-6 기준 0.2~0.7dB 개선(코드율별), WER도 유의미하게 감소.
10. 한계: 광통신(OSNR, optical filter bandwidth) 채널 시뮬레이션에 한정되고 HW 구현·복잡도·throughput 분석 없음, iteration 수 자체를 줄이는 기법이 아니라 동일 iteration 수 내 성능 개선에 국한.

| Decoding trajectory 추적(EXIT chart 변형) | `decoder.cpp` `LDPC_Decoder()` | 이터레이션별 수렴 거동 분석 개념은 반영 가능하나 EXIT chart 자체는 오프라인 설계 도구로 코드 내 대응 없음 |

적용 가치: 낮음 — 부호 자체(H-matrix)를 다시 설계해야 하는 code-design 기법으로 Prime ECC의 고정된 QC-LDPC 구조에 이식하려면 전체 재검증이 필요하며, NAND 타깃도 아닌 광통신 특화 결과.
