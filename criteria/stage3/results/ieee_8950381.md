### B. 알고리즘 요약

1. 이진 LDPC 코드(길이 n, parity-check H)의 ML 복호 문제를 정수계획법으로 정식화 후, 3-변수 parity-check 식(`x1+x2+x3=0 mod 2`)을 보조변수로 분해해 선형 정수계획(9)으로 변환.
2. BP는 고SNR에서 error floor, 기존 LP/ADMM 디코더는 check polytope 투영 연산이 고비용이라는 문제의식.
3. 채널은 이진 대칭 채널(memoryless binary-input output-symmetric), LLR `γi=log(Pr(ri|xi=0)/Pr(ri|xi=1))` 기반 ML 목적함수 사용.
4. 핵심 기법: 정수제약 `v∈{0,1}`을 박스제약 `v∈[0,1]`로 완화하고 비볼록 이차 페널티항 `-α/2‖v-0.5‖²`를 목적함수에 추가한 QP 모델(식 10)을 제안, ADMM으로 `v,z,y`를 교대 갱신.
5. 행렬 A의 열-직교성(column orthogonality) 덕분에 `v`, `z` 갱신이 각각 `[0,1]`, `[0,+∞)` 구간으로의 단순 사영(clipping)만으로 분석적·병렬 계산 가능 — check polytope 투영(정렬 필요)을 완전히 제거.
6. all-zeros assumption 성질을 이론적으로 증명(대칭채널 가정 시 실패확률이 전송 코드워드에 무관), 정수해 획득 시 ML 여부를 1회 추가 반복으로 검증 가능(Proposition 1).
7. HW 친화 트릭: 행렬 A의 원소가 {0,1,-1}뿐이라 대부분 연산이 덧셈으로 처리되고, 사영은 min/max clip 수준으로 저복잡.
8. 별도 학습 절차 없음; μ, α 페널티 파라미터는 시뮬레이션 스윕으로 경험적 선정(μ≈1, α∈[0.3,1] 범위에서 양호).
9. 결과: 6종 표준 LDPC(Margulis, MacKay, IEEE 802.16e, 802.11n, 길이 576~13298, rate 0.5~0.89)에서 BP 대비 고SNR에서 error floor 없이 우수, 반복당 복잡도 O(n+Γa)로 코드길이에 선형(기존 ADMM 계열 O(n+m·d·log d) 대비 저복잡), 반복횟수 대부분 100회 미만.
10. 한계: HW(FPGA/ASIC) 구현·검증 없이 PC 시뮬(i5-3470, C++)만 수행, ADMM 비볼록 수렴성은 이론적으로 완전히 증명되지 않고 관측적 가정에 의존, 완전히 새로운 min-sum과 무관한 디코더 아키텍처.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QP-ADMM 반복 갱신 알고리즘 전체 (v/z/y 갱신) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | min-sum 이터레이션 루프를 완전히 대체해야 하는 이질적 알고리즘, 이식 시 CN/VN 갱신식 전면 교체 필요 |
| check polytope 투영 제거(사영을 단순 clip으로 대체) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum의 min1/min2 연산 자체를 QP 사영 연산으로 대체하는 개념이라 회로 재설계 필요, 직접 매핑 불가 |
| error floor 개선(all-zeros assumption 하 이론적 무오류 특성) | 대응 없음 (개선영역: error-floor) | Prime ECC는 min-sum 계열 error floor 대책(HCU, 조기종료)을 이미 보유, 근본적으로 다른 접근이라 관심 높음 영역이나 대응 모듈 없음 |
| LLR 기반 soft-decision 입력 | `channel.cpp` `Set_LLR_Th()` | ML 목적함수 입력으로 LLR 사용하나 양자화 방식은 논문에 미기술(soft-4bit+ 가정) |

> 적용 가치: 중간 — error floor 없는 신규 디코딩 패러다임(min-sum 계열과 무관)이라 이론적 흥미는 있으나 HW 미설계·미검증이며 Prime ECC의 min-sum 파이프라인(CNU/VNU, HCU, pipeline)과 구조가 근본적으로 달라 이식하려면 디코더 아키텍처 전면 재설계가 필요.
