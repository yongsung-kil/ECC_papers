### B. 알고리즘 요약

1. 대상: SLC/MLC NAND flash, 실제 ICI 채널모델(`γWL-to-WL,γBL-to-BL,γDiag`), PBCH `[n=1023,k=923]`·`[2047,1816]`, `R≈0.89~0.90`.
2. 문제: 셀 스케일링으로 ICI 증가, soft-decision LDPC는 다중 read로 read speed 저하 → read 대신 write 단계에서 대응하고자 함.
3. 핵심 통찰: flash의 write(page)/erase(block) 비대칭성으로 ICI 채널을 Costa dirty-paper → "결함셀 메모리(stuck-at defect)" 모델로 변환.
4. 변환 기법: 쓰기 전 pre-read(`ηpre`)로 threshold가 높은 취약셀을 stuck-at 0 defect로 식별, side info `s∈{0,λ}` 설정(식20).
5. 부호: Heegard `[n,k,l]` PLBC — `c = G1 m + G0 d`, 최적 `d*=argmin‖c∘s−c‖`로 defect masking(식10,12), `(d0,d1)`로 masking·정정능력 보증.
6. 인코딩: 2-step encoding(식13, Gaussian elimination `O(n^3)`), Step1 실패 시 `d0−1`개 defect만 선택 마스킹(Algorithm 1).
7. MLC 확장: multipage 아키텍처·page-address ordering 이용, MSB page에 2회 pre-read(`ηpre,1,ηpre,2`)로 stuck-at 0/1 defect 설정(Algorithm 3).
8. 이론: PBCH 복호실패확률 상한(식16, 결함+random error) 유도 → Monte-Carlo 없이 저확률 영역 추정.
9. 결과: SLC `P(m≠m)` 2.1e-5→1.1e-6, 밀도이득 약 19%(`α` 0.1 gain), PBCH가 LDPC 2-read 수준을 1-read read speed로 달성·error floor 없음.
10. 한계: HW 미설계·시뮬만·write speed 희생·기본 부호는 BCH/PLBC(LDPC는 AE-LDPC 결합옵션 언급뿐)·P/E cycle·retention 미고려.
