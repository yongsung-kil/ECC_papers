### B. 알고리즘 요약

1. 대상 시스템: BPSK + rate-1/2 protograph LDPC, 부호길이 2048, 수중음향(UWA) 다중경로 채널(Wuyuan Bay 실측 FIR 필터, 식(3) 8-tap).
2. 문제: 무한길이 EXIT/PEXIT은 ergodicity 가정으로 수백~수천 bit 단모/중길이 UWA 부호의 실제 성능을 못 맞춘다.
3. 채널모델: `y_n = Σ h_k x_{n-k} + w_n` (식1), coherence time 내 LTI FIR로 근사, 채널출력 LLR은 대칭 가우시안 분포로 검증됨.
4. 핵심기법: 유한길이 EXIT — inner(VN)/outer(CN) 복호기 각각 몬테카를로 BP 반복으로 EXIT band(central + ±3√var 상하한) 생성.
5. 식 의미: `I_ch = J(σ_ch)` (식6) — 채널출력 LLR 분산 σ_ch를 mutual information으로 변환, 유한 프레임 앙상블의 수렴 tunnel 판정.
6. 확장: outer 복호기는 `I_A,outer`로부터 `N_A=10000`회 반복해 EXIT band 형성, inner/outer band 사이 tunnel 개폐로 수렴 예측.
7. 구현 디테일: band는 central/upper/lower 3곡선, 각 앙상블은 동일길이 프레임의 개별 EXIT 곡선을 높은 확률로 포함.
8. 최적화: infinite EXIT로 4x8 protomatrix 후보를 빠르게 탐색(식9, degree-2 노드≤3, 병렬엣지·짧은 cycle 제한, 선형 최소거리 성장 보장).
9. 결과: 제안 HUWA 부호가 AR3A/AR4JA/(3,6) regular 대비 우수, BER=1e-5까지 floor 없음, (3,6) 대비 0.6dB 이득, EXIT 예측과 BER 시뮬 일치.
10. 한계: HW 미설계, 시뮬만, UWA 특정 채널 가정, 부호길이 2048 고정, 고rate 미검토.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| UWA FIR 다중경로 채널모델 | 대응 없음 | Prime ECC는 AWGN/RBER/fixed-error만, 다중경로 무관 |
| protograph 4x8 부호구성(HUWA) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | Prime은 고정 QC-LDPC(base+shift), protograph rate-1/2 저rate 부호와 구조·rate 상이 |
| 유한길이 EXIT 해석도구 | 대응 없음 | 부호평가용 분석툴, 디코더/인코더 코드 경로에 직접 대응 없음 |
| BP(belief propagation) inner/outer | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 원리상 message passing이나 Prime은 min-sum HW 근사 사용, full-BP 아님 |
