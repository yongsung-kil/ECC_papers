### B. 알고리즘 요약

1. 대상은 홀수 소수 `q`와 정수 `m≤q`로 대수적으로 정의되는 배열 LDPC 부호 `C(m,q)`, parity-check `H(m,q)`는 `mq×q²` 크기의 QC(순환행렬 `P` 기반) 구조.
2. 풀려는 문제: BEC 반복복호 성능을 좌우하는 최소 stopping set 개수 `Smin`과 최소 codeword 개수 `Amin`의 정확한 값(폐형식)을 구하는 것.
3. 배경 사실: stopping distance `s(H)≤d`이며, `s(H)=d`이고 `Smin=Amin`이면 이상적. 선행연구에서 `2≤m≤3`, `(m,q)=(4,5),(4,7)`에 대해 `s(H)=d`임이 알려짐.
4. 핵심 도구: 각 열을 길이 `m`의 mod-`q` 등차수열 벡터 `[i, i+j, ..., i+(m-1)j]`로 표현하고, 압축행렬 `H̃(m,q)`의 행별 성분 중복(partition) 구조로 stopping set 여부를 판정.
5. 핵심 판정식: 부분집합 `S`가 stopping set ⇔ `H̃_S`의 각 행에서 모든 성분이 최소 2회 등장. 4-cycle 부재(girth 6) 조건으로 가능한 행 partition을 제약.
6. Theorem 1 (`m=2`): `Amin(2,q)=Smin(2,q)=q²(q-1)²/4` (모든 최소 stopping set이 codeword support).
7. Theorem 2 (`m=3`): `q≥5`이면 `Amin=Smin=q²(q-1)(q-2)/6`, `q=3`이면 `Amin=3`이나 `Smin=12`(9개 예외).
8. Lemma 3/4 및 Theorem 3 (`m=4`): `(4,5)`는 `Amin=25, Smin=125`(codeword 아닌 stopping set 100개), `(4,7)`은 `Amin=Smin=147`.
9. 증명 기법: 각 partition 조합에 대해 mod-`q` 등차수열 제약으로 모순(`i2=i3` 또는 `j1=j2`) 유도하여 예외 stopping set 개수를 열거.
10. 한계: 특정 소수 케이스만 다루는 순수 조합 이론, 디코더·HW·시뮬레이션·BER 곡선 전무. 일반 `(m,q)`는 open problem.
