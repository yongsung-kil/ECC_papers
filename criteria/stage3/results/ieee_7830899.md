### B. 알고리즘 요약

1. 대상: Refs.[7,8]의 semi-random irregular NQC(Non-Quasi-Cyclic) LDPC, rate-2/3, length-15360(basic matrix 10×30, 512×512 sub-block, row weight 11, col weight 2~5), BPSK+AWGN.
2. 문제: NQC 부호는 partly-parallel BP 디코더에서 memory-access conflict·낮은 HW 이용효율(HUE)·과다 block RAM 발생. 기존 OMP는 무작위 `θ'` 파라미터 때문에 적용 불가.
3. 구성법: basic matrix에 수학적 제약(column parameter vector allowed set `Ω`, girth-4 회피)을 걸어 memory-access conflict를 회피하는 implementation-oriented 구성 제안.
4. 핵심 성질: 각 macro row가 `ρ`-block-divided가 되도록 구성 → row vector를 `ρ`개 simple two-port RAM에 균등 분산 저장 가능(충돌 없음).
5. MOMP(Alg.1): OMP를 수정해 VNP/CNP를 제약 없이 완전 오버랩. sub-matrix message passing을 3 region으로 나눠 n/n-1/n-2 iteration message를 혼용, `Q_v→c`/`R_c→v` 별도 RAM 저장.
6. 연산: Step 7/16의 갱신식은 offset min-sum[17] 또는 BP와 동일 formulation, 스케줄만 다름 → CNU는 min1/min2/position row-vector로 압축 저장(min-sum HW 관례).
7. 병렬 파라미터 `P=2^m`: 단일코어 병렬도 조절. `P=1`은 저메모리, `P>1`은 `N_b·P`/`M_b·P` VNU/CNU로 throughput 증가(단, throughput은 P보다 느리게 증가 → 큰 P에서 logic 효율↓).
8. Multi-core: non-cooperative(단순, 자원 M배) vs cooperative(cycle bus로 같은 sub-matrix RAM 결합·address generator 공유 → block RAM 최소화).
9. 결과(Altera 5SGXMA7N2F40C2 FPGA, 100MHz, 19 iter): 전통 구조 대비 logic/Mbps 23.8~52.6% 감소, message-memory bits/Mbps 29.0~90.0% 감소; 최대 throughput ~1924.82 Mbps; 8.43dB coding gain, 6-bit 양자화.
10. 한계: MOMP는 저효율 message-passing 확장 구간 때문에 BP 대비 ~0.2dB, P 증가로 추가 0.07dB 손실; 대상이 Non-QC라 QC-LDPC 코드베이스로 직접 이식 곤란.
