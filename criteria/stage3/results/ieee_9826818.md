### B. 알고리즘 요약

1. 시스템: block fading(BF) 채널, L=4 블록, BPSK, (N,K) binary linear code, block length `T=N/L`.
2. 문제: r=2/L~(L-2)/L 중간 rate 구간의 BF 채널용 diversity-approaching LDPC 구성/분석이 부족.
3. 모델: 각 블록 fading gain `α_i`(i.i.d. Rayleigh), 순시 SNR `γ_i = r(Eb/N0)α_i²`, 채널 outage로 FER 하한 정의.
4. 핵심기법1: Resolvable Block Design(RBD; BTD(n) for k=2, KTS(v) for k=3)의 incidence matrix를 protograph base matrix로 사용해 정규 protograph LDPC 구성.
5. 핵심 성질: RBD의 resolvable 특성으로 각 CN이 서로 다른 fading 블록의 VN과 정확히 한 번씩 연결 → 큰 girth(base girth 6 → QC lifting 시 g≥18) 보장.
6. 확장: k=2는 rate `(L-2)/L`, k=3는 `(L-3)/L`의 두 정규 protograph 부호(Construction 1) 도출.
7. 구현 디테일: KTS(9)/BTD(3) protograph를 lifting size `Z=200`으로 lifting → girth `g=14` (2400,1200)/(2400,600) 부호 생성.
8. 학습/최적화: 없음(조합적 설계). 대신 min-sum 복호 하에서 AP 메시지의 순시 SNR을 추적하는 `gamma evolution` 분석 도입.
9. 결과: gamma evolution BER 상한 식 (7)(8)이 실제 시뮬 BER slope와 일치, iteration 진행 시 slope가 가팔라지고 FER slope가 채널 outage에 근접.
10. 한계: BF(무선) 채널 특화, HW 미설계, 시뮬레이션 수준, AWGN·NAND 시나리오 미검증.
