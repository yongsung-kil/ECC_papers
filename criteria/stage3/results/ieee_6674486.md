### B. 알고리즘 요약

1. 대상: irregular rate 1/2 (2016, 1512) LDPC를 Log Sum-Product(Log-SP)로 복호하는 기준 대비, Min-Sum/NMS/OMS 근사가 유발하는 성능저하를 예측.
2. 문제: 근사 디코더의 BER는 시뮬 없이는 예측 불가, word-length·근사 설계 시간이 큼.
3. 핵심 보조정리 `Lemma 1`: `BERB = (1-2·Pdc)·BERA + Pdc`. 근사 알고리즘 B의 BER를 원본 A의 BER와 결정변경확률 `Pdc`로 표현.
4. 결정변경 메커니즘: 결정메시지 `m`과 근사오차 `e = mdec - m̂dec`의 부호가 뒤집힐 조건 (`-e < m < 0` 또는 `0 < m < -e`)을 카르테시안 평면 영역으로 정의.
5. 확률 계산: 두 시나리오 영역 적분합 `pC = pA + pB` (식 9)로 결정변경확률을 구함.
6. 핵심 기여: 결합 pdf `fm,e(m,e)`를 bivariate 2D 정규분포로 모델링(식 10-12, 대칭화 `(gm,e + g-m,-e)/2`, 상관계수 `ρ`).
7. 파라미터 `(μm,σm),(μe,σe),ρ`는 코드 degree 분포·근사방식에 종속, 시뮬 로그에서 fitting으로 실험적 결정.
8. 방법 검증: Log-SP를 5 iteration 실행한 BER를 기준으로 NMS(α=0.3), OMS(β=2)의 BER를 (1)로 예측, 시뮬 BER와 근접.
9. 결과: MS/NMS는 모델 정확도 양호, OMS는 offset factor로 오차분포가 불연속이라 정확도 저하.
10. 한계: HW 미설계, 순수 성능예측 도구, 파라미터를 매 코드/근사/iteration마다 실험으로 얻어야 함.
