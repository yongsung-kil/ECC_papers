### B. 알고리즘 요약

1. 대상: 프로그래머블 임베디드 멀티코어(SIMD 확장)에서 실시간 LDPC 복호, DVB-S2(N=16200/64800)·WiMAX 802.16e irregular 부호(rate 1/2 등), BPSK/AWGN.
2. 문제: 표준마다 다른 LDPC 부호 → 유연성 필요, 전용 ASIC은 비유연·NRE 높음 → 프로그래머블 멀티코어로 실시간 복호 달성이 목표.
3. 핵심 가정: 코드워드 간 의존성이 없으므로 다중 코드워드를 독립 병렬 복호(data parallelism) 가능, 단일 공유버스로 충분.
4. 핵심 기법 1단: MSA(min-sum) CN처리 `Lrmn = sign·min|Lqnm|`, BN처리 `Lqnm = LPn + ΣLrmn` 을 수평/수직 커널로 분리.
5. 핵심 자료구조: Tanner 그래프를 row-major/col-major로 스캔해 non-null edge만 주소로 담는 컴팩트 구조 `HBN`(수평)·`HCN`(수직), CRS/CCS 대체.
6. 핵심 기법 2단: intra-codeword(그래프 분할 task) + multi-codeword(SIMD 8/16-way) 병렬화, 분산 scratchpad 메모리 + DMA.
7. 구현 디테일: 8-bit 고정소수점으로 128-bit 접근당 16 데이터 packing, 정렬 불가 시 runtime data realignment로 read/write 중 하나만 정렬.
8. 학습/최적화 절차 없음 (연산량 `Nop>12×#Edges`, 코어수 `np` 추정식 제시).
9. 결과: Cell/B.E.(PS3, 6 SPE)에서 96 코드워드 병렬 10 iter시 78~79.6Mbps(5 iter시 ~145Mbps), 참조 ASIC 180코어 90nm 13.6mm²/6.2Mgates/185mW, DVB-S2 3.9mm²·WiMAX 1.3mm²@65nm.
10. 한계: 256KB SPE 로컬메모리 초과 큰 부호는 throughput 급락(≤9Mbps), 알고리즘 신규성 없음(표준 MSA), 무선 표준 타깃.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MSA(min-sum) CN/BN 처리 | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | Prime ECC도 min-sum 사용 — 기법 중복(신규성 없음) |
| HBN/HCN 컴팩트 Tanner 자료구조 + 라우팅 | `GT.cpp` `Make_GT_HW()` (부분), 대체로 대응 없음 | 프로그래머블 멀티코어 SW 자료구조 ≠ 고정 QC-LDPC HW 배선 |
| 다중 코드워드 SIMD 병렬화 | 대응 없음 | Prime ECC는 z=32 row-병렬 단일 코드워드 pipeline |
| ASIC/FPGA 멀티코어 아키텍처 | 대응 없음 | Prime ECC HW 모델(HCU/GT/6-stage)과 구조 상이 |

적용 가치: **낮음** — 알고리즘은 이미 보유한 표준 MSA 그대로이고, 기여는 무선 표준용 프로그래머블 멀티코어 SW 병렬화/자료구조라 NAND 고정 QC-LDPC HW 디코더에 이식할 요소가 거의 없다.
