### B. 알고리즘 요약

1. 대상: 32-layer 3D MLC NAND(Micron MT29F256G08, 256Gbit), 페이지 16kB user + 2208B OOB(ECC), block당 1024 page.
2. 풀려는 문제: TID(감마 조사, ~50 krad)로 셀 `Vt` 하강→default read FBC가 ECC 정정한계(~0.5~1%, 최대 3%) 초과.
3. 채널/물리 모델: 방사선이 floating-gate 전자 손실을 유발해 program state Vt가 하향 이동, MSB 페이지가 LSB보다 fail 많음.
4. 핵심 기법: Read-Retry(RR) - ONFI Set Feature(0xEF, addr 0x89)로 16개 내부 read 기준전압 중 최적 옵션 선택(datasheet에 전압값 비공개).
5. 핵심 관찰: 조사 후 default(#0)가 최소 FBC가 아니며 예로 20krad에서 옵션 #11, 10krad에서 #10이 최적(dose별 상이).
6. 확장 변수: 최적 RR 옵션이 chip-to-chip 편차, dose, LSB/MSB, 3D layer 위치에 의존 → 각각 precharacterization 필요.
7. 제안 알고리즘: 블록당 dosimeter 페이지(all-zero)를 먼저 읽어 raw FBC로 dose를 추정, 사전 characterized LUT에서 최적 RR 옵션 선택.
8. 절차: raw FBC가 ECC 한계 내면 default read+ECC, 초과 시 LUT로 최적 RR 옵션 적용 후 read.
9. 결과: 20krad에서 평균 page FBC 171→5 미만(~98% 정정), 50krad에서도 ~95% byte 정정(잔여 5%는 강한 ECC로 커버 기대).
10. 한계: ECC/디코더 자체 기여 없음, HW 변경 불필요하나 전압 offset 독점정보, 데이터 유지시간·온도 미반영, TID 특화.
