### B. 알고리즘 요약

1. 대상은 semi-infinite time-invariant SC-LDPC 코드(QC 서브행렬 구성, syndrome former memory `µ`), AWGN 채널, windowed decoding 기반.
2. 기존 windowed decoding은 고정 크기 window(`w`)로 BP를 수행하지만 순차 처리로 latency가 크고 pipeline decoding 대비 메모리 소모가 큼.
3. layered 스케줄(check node 순차 업데이트)을 기반으로, window 내 인접 sub-block(`µ+1`개)만 참조하는 지역성을 이용해 병렬화 가능함을 관찰.
4. 두 가지 개선: (1) `P`개 프로세서가 서로 다른 stage를 동시에 처리하는 parallel computation, (2) 이미 충분히 갱신된 LLR sub-block을 즉시 메모리에서 제거하는 memory-efficient management.
5. 핵심 수식: `Cbit = P(1-R)(I(µ+1)+1)Ccheck`(연산량), `Storage = P(µ+1)mSdc + (P(µ+1)+µ)nS`(저장량), `∆delay = (P(µ+1)+µ)mS(I(µ+1)+1)Tcheck`(latency).
6. 파라미터 `P`(병렬 프로세서 수), `I`(내부 반복 횟수)로 성능-복잡도 trade-off를 조절.
7. Algorithm 1로 반복문 구조 명시: 매 stage마다 LLR/R sub-block 삽입 → `P`개 프로세서 병렬 갱신 → shift.
8. 별도 학습/최적화 절차 없음(폐형 알고리즘, 파라미터 스윕 시뮬레이션만).
9. Code1(µ=2)에서 windowed decoding 대비 storage 75%, latency 11%; Code2(µ=1)에서 storage 67%, latency 7%로 감소하며 BER 성능 저하 없음(windowed decoding과 동일 BER 곡선).
10. AWGN 채널 시뮬레이션에 국한되고 HW 아키텍처/게이트 설계는 제시되지 않음; SC-LDPC 특화(구조적 코드에만 적용 가능)라 QC-LDPC block code에는 직접 적용 불가.
