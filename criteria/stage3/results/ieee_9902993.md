### B. 알고리즘 요약

1. 시스템: NAND Flash용 (N=36864, K=33028, M=3840) QC-LDPC, CPM `Z=256`, `dv=5`, rate≈0.896, hard-decision, fixed-EBC 채널로 FER 평가.
2. 문제: global-searching GDBF 계열(CRGDBF, tabu-list)이 4KB 코드에서 error-floor 발생 + throughput 병목.
3. error-floor 원인: tabu-list의 지배적 penalty `Mtabu=-∞`가 과보정(over-correction) 시 에러비트를 배제해 버스트 에러 유발 (Fig.2 trace 분석).
4. 핵심기법: penalty method로 디코딩을 제약없는 최적화 `arg max{fi+Mi·hi^2}`로 해석, 에너지 `Ei = (xi⊕yi) + Σsj + ρ(di) + ηi`.
5. 통합: correlation penalty·flipping penalty(momentum `ρ(di)`)·random penalty(`ηi`)를 하나의 energy로 통합 → UP-GDBF (Algorithm 3), randomness를 soft constraint(`Mη=1`)로 두어 flip-miss 감소.
6. active iteration: `ηi = Bernoulli(P=0.9) if t>tactive else 1`, `tactive`는 fixed-EBC 오프라인 Monte-Carlo로 결정 → 초기 오류 randomness 억제.
7. HW: partial-parallel, 144 col-block×256bit 병렬, 4-stage pipeline, C2V/V2C barrel shifter, ECU(one-hot adder)+value collapse unit.
8. energy-based backtracking: max-finding phase에서 동일 에너지 col index를 backtracking memory에 연결, flipping phase에서 유효 col만 접근 → idle cycle 제거.
9. 결과: 10~20% iteration 감소, backtracking로 42%+ cycle 감소(1.72x throughput), 16nm 0.120mm2 1GHz, 19.3Gbps(ONFI 5.0), backtracking overhead 0.8%, 4KB에서 non-global 대비 10^4배 신뢰도.
10. 한계: hard-decision(1bit)만, fixed-EBC 채널 가정, `tactive` 채널별 오프라인 튜닝 필요.
