### B. 알고리즘 요약

1. 시스템: 통신용 QC-LDPC(802.16e/802.11n/5G 3GPP), `N`=648~8192, rate 1/2~8/9, `Z`=27~256 lifting.
2. 문제: 통신표준의 빠른 변화에 hand-coded RTL은 IP 개발이 느림 → 유연·자동화된 고효율 FPGA 디코더 생성 필요.
3. 접근: x86 소프트웨어 LDPC 디코더 소스[4]를 SystemC 행동모델로 재작성 후 Vivado HLS로 RTL 자동생성.
4. 핵심기법: horizontal layered min-sum(Algorithm 1) — Kernel3.1 `Z`개 incoming msg 병렬계산, Kernel3.2 `Z`개 outgoing msg+VN 갱신, H 구조에 맞춘 interleave/de-interleave.
5. 병렬화: intra-frame 병렬화, QC의 `Z`가 병렬 CN 계산 수를 규정(SIMD 벡터폭=32 고정, AVX2 대체).
6. 아키텍처: `Z` memory bank + `Z` PU, interleaver를 barrel shifter로 구현, layered 스케줄로 flooding 대비 2× 빠른 수렴.
7. 구현 디테일: scalar 라이브러리(saturated add/min/XOR/sign) + 가변 bitwidth(`sc_int<T>`), 기본 양자화 VN 8bit·msg 6bit.
8. 최적화: HLS pipeline/unroll/inline pragma로 처리량 튜닝, 100/400MHz 두 클럭 트레이드오프.
9. 결과: Virtex-7에서 단일 디코더 31~147Mbps(10 iter), 기존 HLS 대비 throughput 1.1~16.7×·면적 1.6~23.8× 개선, float 대비 0.05dB 열화.
10. 한계: HW 생성 방법론일 뿐 정정능력·error-floor 개선 없음, NAND 무관, gate count·throughput은 Gbps급 미달.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered min-sum 디코딩 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | 알고리즘 자체는 표준 min-sum, 이미 보유(중복) |
| CN min-sum(min/sign) 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 연산 이미 구현, 신규성 없음 |
| SIMD/HLS 자동생성 흐름 | 대응 없음 | Prime ECC는 bit-exact C++ 수동 모델, HLS 생성 흐름과 무관 |
| barrel shifter interleaver(라우팅) | 대응 없음 | QC shift 처리 방식 다름, 직접 대응 함수 없음 |

적용 가치: 낮음 — layered min-sum은 이미 보유 기법이고 논문 핵심(HLS 자동생성)은 Prime ECC의 bit-exact 수동 모델과 무관하며 정정성능 개선도 없음.
