### B. 알고리즘 요약

1. 대상은 시불변(time-invariant) LDPC 컨볼루션 부호(LDPC-CC, spatially-coupled)로, polynomial syndrome former matrix `HT(D)`로 정의되며 rate `R=b/c`.
2. 문제: 고rate rate-compatible 부호를 펑처링으로 얻을 때 무작위 펑처링은 error-floor와 waterfall 손실이 큼. 기존 distance-spectrum 기준은 대형 memory 부호에서 계산 불가.
3. 그래프 기반 3기준으로 펑처링 패턴 `a`(주기 `P`, `P×c` 0/1 행렬)를 선택.
4. 기준1 - recoverability: 펑처된 VN의 `m`-step-recoverable(`m`-SR) 열거자 `Ec_ai` 계산, `∞`-SR(복구불가) 노드를 최우선 제거하고 작은 `m` 선호.
5. 기준2 - CPCTS: cycle trapping set의 모든 VN이 펑처된 completely-punctured cycle trapping set 수 `Eτ_ai`를 최소화(error-floor 억제).
6. 기준3 - 단주기: 길이 `w` 단주기 cycle에 포함된 펑처 VN 수 `Eb_ai`를 최소화(waterfall 개선).
7. Algorithm 1이 이 3기준을 순차 필터(Step1.1→1.2→2→3)로 적용해 nested(rate-compatible) 패턴을 도출.
8. cycle 열거는 참고문헌 [25] 방법으로 girth `g`부터 `Nτ,Nb`까지 계산, mother code당 1회만 수행.
9. 결과: (21,3,5)/(57,3,5) Tanner LDPC-CC 등에서 rate 4/10~8/9 계열 생성, `P`를 키우면 `∞`-SR 회피 범위 확대·중간 rate 증가, rate 8/10에서 약 2.1dB gain(P=2 대비 P=4).
10. 한계: AWGN/BPSK 시뮬만, HW·복잡도·throughput 없음. 대상이 블록부호가 아닌 컨볼루션 부호이고 판정은 ML이 아닌 iterative BP 기준.
