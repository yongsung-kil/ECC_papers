### B. 알고리즘 요약

1. Multi-Dimensional Spatially-Coupled (MD-SC) LDPC 코드: `M`개의 SC 코드 사본을 대각선에 배치하고 일부 circulant를 인접 사본으로 "relocation"하여 결합한 QC(quasi-cyclic) 구조, 파라미터 `(γ, κ, z, L, m, M)`.
2. 기존 MD-SC 설계는 자유도(SC memory `m`, 사본 수 `M`)가 커질수록 조합 탐색 복잡도가 폭증해 좋은 relocation을 찾기 어려움 — 이를 확률적 프레임워크로 완화하고자 함.
3. 핵심 가정: Tanner graph의 짧은 cycle(`cycle-6`, `cycle-8`)이 absorbing set을 구성해 성능(error-floor)을 저해한다는 표준 LDPC 결함 모델을 채택.
4. 핵심 기법 1단: `(m+1)×M` 확률분포행렬 `P`(원소 `p_i,j`)로 각 circulant가 어느 auxiliary matrix로 relocation될지의 결합확률을 표현, 이를 gradient-descent로 최적화(`MD-GRADE` 알고리즘).
5. 핵심 식: 기대 cycle-6/8 개수 `N6(p_con)`, `N8(p_con)`을 two-variable coupling polynomial `f(X,Y;P)`의 계수 형태로 유도(식 (5),(6)) — 이 값을 최소화하는 local-optimal `P`를 closed-form 조건(식 (8),(10))에서 구함.
6. 핵심 기법 2단(확장): local-optimal `P`를 초기값으로 하는 `Finite-Length Algorithmic Optimizer (FL-AO)`가 실제 circulant relocation을 수행해 유한장 GD-MD 코드를 생성(활성 cycle이 가장 많은 circulant부터 순차 relocation).
7. 부수 트릭: `force(·)` 함수로 각 행의 확률 합을 `p*`(기존 SC 코드의 local-optimal edge distribution, [8]에서 획득)로 유지하며 음수 성분 보정.
8. 최적화 절차: MD-GRADE(gradient descent, step size `α`) → FL-AO(반복적 relocation, 활성 cycle 감소 없거나 iteration bound 도달 시 종료), 학습 없이 결정론적 수치 최적화.
9. 결과: 1D SC/TC 코드 대비 cycle-6 92~100%, cycle-8 83~98% 감소(예: SC Code 1.1 79,917개 → GD-MD Code 1.1 6,375개), FL-AO 반복 횟수는 항상 20회 미만.
10. 한계: LDPC 디코딩 BER/FER 시뮬레이션은 수행하지 않음(cycle count만으로 성능 논증), HW 미설계, NAND/Flash를 직접 타깃하지 않는 일반 SC-LDPC 구성법 논문(장문판 arXiv:2401.15166에 증명·시뮬 별도 수록 가능성).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MD-SC/circulant relocation 기반 H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base+circulant(QC-LDPC) PCM을 고정 로드하는 구조라, 새 H-matrix 설계 자체는 재검증 부담이 큰 최고난도 영역 |
| cycle-6/8 최소화를 통한 absorbing set 억제 | 대응 없음 | 디코더 알고리즘이 아닌 부호구조(H-matrix) 설계 문제로 decoder.cpp 이터레이션 로직과 무관 |
| gradient-descent 확률분포 기반 최적화 절차 | 대응 없음 | Prime ECC 코드베이스에 부호 자동설계/최적화 모듈 없음(H-matrix는 외부에서 고정 로드) |

> 적용 가치: 낮음~중간 — QC-LDPC(circulant) 구조라는 점에서 개념적 유사성은 있으나, 이 논문은 특정 H-matrix 구성법(설계 시점 산출물)이며 NAND 타깃 실증·디코딩 성능 검증이 없어 Prime ECC decoder/encoder 코드에 직접 이식할 요소가 없음. 향후 고rate QC-LDPC 신규 설계 시 참고 가치는 있음.
