### B. 알고리즘 요약

1. 타깃: NAND SSD 컨트롤러의 Garbage Collection(FTL 모듈), LDPC는 read 정정용 블랙박스로만 등장.
2. 문제: LDPC read가 고지연 → GC의 페이지 마이그레이션(external GC) 비용 상승 → 시스템 성능 저하.
3. 모델: 두 GC read 방식 비교 — external GC(컨트롤러에서 LDPC decode/re-encode, 신뢰성↑ 지연↑) vs internal GC(플래시 레지스터 내 복사, 지연↓ 무결성 희생).
4. 핵심기법(LaGC): 블록 retention time을 문턱과 비교해 위험 블록(고RBER)만 external GC, 나머지는 internal GC 선택.
5. 판정 근거: 고정 RBER 문턱(BER0)에 근접하는 data kept time을 [11] 에러모델에서 도출, P/E cycle에 따라 문턱 조정.
6. 자료구조: Block Retention Information(BRI, flash) + Block Retention Table(BRT, 컨트롤러 100 엔트리), GC Assistant(valid page detector + LDPC selector).
7. 오버헤드: 블록당 2B, 전체 BRI 128M(SSD의 4%), BRT 100엔트리는 무시가능.
8. 최적화: 알고리즘 학습 없음 — 단순 문턱 기반 정책.
9. 결과: Disksim(SSD ext) 6개 워크로드, external GC read 지연 78% 절감, GC 비용 5% 개선, 시스템 응답 약 2% 개선. (32GB, P/E 8000, BER0=0.006, retention 1년)
10. 한계: HW 미설계(시뮬만), 부호종류/rate/length/디코더 알고리즘 세부 전무, ECC 코어 기여 없음.
