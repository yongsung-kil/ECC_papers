### B. 알고리즘 요약

1. IEEE 802.16e 표준 QC-LDPC(rate 2/3, length 1920), AWGN + BPSK 채널로 시뮬레이션.
2. 전통 MP(row/column 분리) 디코더는 메모리 충돌·라우팅 혼잡, iteration당 1회 LLR 갱신으로 수렴이 느림.
3. TDMP(layered decoding)를 일반화해 row 처리 안에 column 처리를 결합한 joint row-column processor 제안.
4. 핵심 갱신식: `uij = Uj - vij`, `v'ij = sign(uik)·φ[Σφ(|uik|)]` (row별로 순차 처리하며 즉시 `U'j = uij + v'ij` 갱신).
5. 이 갱신 방식으로 각 row 처리 후 즉시 최신 LLR을 다음 row에 반영, iteration당 여러 번 LLR이 갱신되어 수렴 가속.
6. min-algorithm 변형: `v'ij = sign(uik)·min|uik|`, 보정 `v''ij = α×v'ij - β` (α=0.75, β=0)로 연산 단순화.
7. row processor의 φ 함수는 7bit LUT(4bit fractional) 구현, extrinsic message는 8bit(부호 1bit + magnitude) 표현, tree 구조로 덧셈 전파지연 단축.
8. 별도 학습/최적화 절차 없음(보정계수는 실험적으로 고정).
9. 결과: 40-parallel 디코더가 XC4VLX160의 20% 로직으로 2Gbps 달성, 동일 BER(10⁻⁵)에 필요한 iteration이 14→7로 감소, serial decoder HW 비용 3% 감소·병렬 decoder는 30% 감소, 속도 4배(병렬) ~ 8배(serial) 향상, error floor는 10⁻⁷에서 10⁻⁸ 이하로 개선.
10. 한계: 단일 코드(rate 2/3, length 1920)만 검증, ASIC 합성 결과 없음(FPGA만), min-algorithm 버전은 FPGA 구현 결과 미포함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| joint row-column 디코딩(layered 일반화, iteration당 다회 LLR 갱신) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 스케줄링 변형으로 Dual-Update와 비교 이식 검토 가능 |
| min-algorithm 기반 CN 연산(min-sum 근사 + 보정계수 α,β) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum 보유, 보정계수 방식만 참고 가치 |
| extrinsic message 저장/메모리 충돌 회피(컬럼 프로세서 제거) | 디코더 HW 모델 (z=32, HCU, GT) | 메모리·라우팅 최적화 아이디어로 참고 가능 |
| φ 함수 LUT 양자화(7bit LUT, 4bit fractional) | `channel.cpp` `Set_LLR_Th()` | 양자화 정밀도 설계 참고 |
| H-matrix (802.16e QC-LDPC, sub-matrix 80) | `ecc_top.cpp` `Load_PCM()` | 특정 부호 고정이라 이식 난이도 높음 |

> 적용 가치: 높음 — layered decoding 변형으로 iteration 감소·error floor 개선·HW 비용 절감을 동시 달성했고 스케줄링 레이어(decoder.cpp)에 낮은 난이도로 이식 가능.
