### B. 알고리즘 요약

1. 시스템: QC-LDPC 코드, parity check matrix (640,128) — 즉 rate `128/640=0.2`가 아니라 문맥상 (n=640, 정보/체크 관계 불명) 이므로 부호rate는 본문의 (640,128) 표기를 그대로 rate 후보로 간주(체크비트 128 가정 시 rate≈0.8). AWGN 채널, BPSK 변조.
2. 문제의식: min-sum(MSA)이 sum-product(SPA) 대비 CN 근사 오차로 성능 저하가 있고, 반복복호 중 trapping set/absorbing set에 갇히면 error floor가 발생. 기존 균등양자화는 CN 업데이트 값의 지수적 증가(iteration에 따른)를 표현하기 어려움.
3. 핵심 가정: trapping set에 걸리지 않은 VN은 최소 반복 내 saturation 값에 도달하지만, 균등양자화된 디코더는 낮은 saturation 값 때문에 trapping set을 벗어나지 못함.
4. 핵심 기법: box-plus 연산 `a⊞b = sgn(a)sgn(b)·min(|a|,|b|)`을 정규화/오프셋(`α`,`β`)으로 근사하는 기존 normalized/offset min-sum에, **non-uniform quantization** `Qq(x)`을 추가 적용. 선형구간은 균등양자화(`N=2^q` level), 급증 구간은 지수증가 factor `d`와 그 거듭제곱으로 스텝 크기를 스케일링(식 19).
5. 식의 의미: 이진 표현에서 마지막 2비트(00,11)로 어느 양자화 구간(선형/지수1/지수2)인지 구분하고 1비트로 부호를 표현 — CN 업데이트 규칙 자체는 변경 없이 magnitude 표현만 바꿔 saturation 값을 키움으로써 trapping set 탈출을 유도.
6. 부수 기법(post-processing): 동일 에러 패턴이 6회 이상 반복되면 LLR 값이 큰 노드를 다음 반복에서 드롭(dynamic node dropping)해 decoding deadlock을 탈출.
7. 시뮬레이션 세팅: 실제 비트폭 증가는 2비트(구간 판별용)+1비트(부호)로 제한적, 실험은 `Δ=1, d=3(전반부), 2d=6(후반부)` 파라미터로 수행.
8. 학습/최적화 절차: 없음(수동 파라미터 선정, DE/GA/NN 미사용).
9. 결과: SNR 5dB 이상에서 non-uniform quantization이 낮은 error floor(약 10^-5)를 탈출시키는 extrinsic LLR을 제공, 균등양자화 대비 고SNR 영역에서 약 0.5dB 코딩이득.
10. 한계: 소규모 시뮬레이션(MSA 10 iter vs SPA 100 iter 비교)뿐이며 HW 설계·gatecount·throughput 데이터 전무. 양자화 비트 매핑 세부사항(정확한 룩업테이블 구조)이 본문에서 다소 불명확.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN 업데이트 시 non-uniform(지수적) LLR magnitude 양자화 | `decoder.cpp` `CNU_Update_New_Mag()`, `ecc_data.h` `PARAM_LLR` | Prime ECC도 magnitude 양자화 테이블 기반 min-sum이므로, 비균등 saturation 값 설계 아이디어를 LLR 테이블 갱신에 참고 가능 |
| trapping set 탈출을 위한 반복 중 saturation 값 조정 | `decoder.cpp` `Get_VNU_Table_Idx()` (iteration별 LLR threshold 테이블) | 이미 iteration 구간별 적응형 LLR threshold 테이블을 보유 중이라 유사 개념이 일부 구현되어 있음 — 신규성 낮으나 saturation 값 확장 아이디어는 참고 가능 |
| 반복 에러 패턴 감지 후 노드 드롭(post-processing) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 별도 post-processing 로직 추가 여지 있으나, Prime ECC는 조기종료(Partial/Full CRC) 중심이라 이 방식과는 결이 다름 |
| normalized/offset min-sum (box-plus 근사) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 계열 기법으로 중복 기여, 신규성 낮음 |

**적용 가치**: 중간 — 이미 보유한 min-sum/양자화 프레임워크에 비균등 saturation 값 설계라는 구체적 아이디어를 더할 수 있으나, HW 검증이 전혀 없어 실제 이식 시 재검증 부담이 있음.
