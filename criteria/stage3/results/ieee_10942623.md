### B. 알고리즘 요약

1. Reed-Muller RM(m, r) 코드(길이 `n=2^m`)를 BPSK+AWGN 채널에서 CPA(collapsed projection-aggregation) 디코더로 복호하는 문제를 다룸.
2. 기존 CPA는 flooding 스케줄로 `n_B = C(m, r-1)`개 서브스페이스를 모두 처리 후에만 갱신해 연산량이 큼.
3. LDPC의 layered decoding(최신 정보를 즉시 반영하는 순차 스케줄)을 CPA에 적용, 부분 집계된 LLR `L_partially_aggregated`를 가중치 `ω`로 현재 LLR에 더해 `L_biased = L + ω·L_partially_aggregated` (식 8)로 편향시켜 조기수렴 유도.
4. Layered 방식은 서브스페이스마다 다른 LLR을 broadcast해야 해 `(n_B-δ)·n`개의 가중 바이어스 연산이 필요, 오버헤드가 큼.
5. LDPC의 broadcast-based BP(check node가 동일 메시지를 모든 variable node에 broadcast)에서 착안, 모든 서브스페이스에 동일한 평균 LLR `L̂` (식 10)을 전파하는 broadcast 수정을 제안해 오버헤드를 `n`회 곱셈·덧셈으로 축소.
6. adaptive syndrome check(주기 `δ`, decay factor `d_SCH`)와 결합해 조기종료(Algorithm 1) 수행.
7. 복잡도는 [10]의 연산량 가중치 테이블 기준으로 측정(layered 오버헤드 `4n` 연산 가정).
8. 파라미터 `α`(바이어스 가중치), `δ`(syndrome check 주기)는 시뮬레이션 스윕으로 코드별 선택(Table I).
9. RM(8,3) 코드, Eb/N0=2.5dB에서 FER 열화 없이 평균 연산량 47% 감소, RM(7,3)은 기존 대비 반복 횟수 1회 감소.
10. 한계: 순수 알고리즘/시뮬레이션 수준으로 HW 미설계, RM 코드 특유의 projection/FHT/aggregation 구조에 결합돼 있어 일반 binary LDPC나 QC-LDPC 파리티체크 디코더에 직접 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CPA layered decoding (부분집계 바이어스) | 대응 없음 | RM 코드 전용 projection-aggregation 구조로 min-sum CN/VN 업데이트와 무관 |
| broadcast-based 메시지 전파 | `decoder.cpp` `C2V_Cal()` | LDPC layered BP에서 유래한 broadcast 개념이나 Prime ECC는 이미 dual-update/HCU 등으로 라우팅 최적화 기 적용, 신규 이식가치 낮음 |
| adaptive syndrome check(조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 아이디어 개념적으로만 유사, Prime ECC는 CRC기반이라 직접 대응 아님 |
| FHT 디코딩 | 대응 없음 | RM 코드 전용 Hadamard 변환, binary QC-LDPC와 무관 |
```

적용 가치: **낮음** — RM 코드 전용 디코더 구조이며, 차용한 LDPC layered/broadcast 개념 자체가 Prime ECC에는 이미 유사 형태로 구현되어 있어 신규 이식 가치가 없음.
