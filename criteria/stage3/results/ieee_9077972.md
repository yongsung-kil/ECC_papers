### B. 알고리즘 요약

1. 시스템: 5G NR uplink, PUSCH slot(14 symbol, DMRS 2 + PUSCH 12), N 사용자가 무선자원 균등 분할, BG1(mother rate 1/3)/BG2(1/5).
2. 문제: peak throughput용으로 설계된 디코더가 낮은 code rate·낮은 lifting size의 transport block을 동일 자원·시간 내 복호하지 못할 수 있음.
3. 모델: transport block→CRC 부착→Ci 코드블록 분할, lifting size `Zi`(2~384)로 base matrix lifting, frequency-first 매핑으로 on-the-fly 처리 가정.
4. 기법: sub-base matrix를 base matrix 앞쪽 `φi` 행·`ϕi` 열로 잘라 구성, core part(4행)는 pruning 불가 → `φi = max(⌈min(Ωp,Φp)/Zi⌉, φi)`.
5. 핵심식: 코드블록당 복호지연 `ti = I·Ui/fc` (I=iteration, Ui=sub-base matrix circulant 블록 수, fc=동작주파수) — block-parallel에서 circulant 수에 비례.
6. 확장: 전체 사용자 지연 `t`를 Algorithm 1로 계산, jth 코드블록이 `(2+⌈12j/Ci⌉)`번째 symbol에서 끝나는 순차 누적 모델.
7. 트릭: LBRM(limited buffer rate matching)으로 circular buffer `Ni = min(Λ, ...)` 제한, redundancy version 순서 0,2,3,1.
8. 최적화: 재전송 시 effective rate 하락 → sub-base matrix 커짐 → 재전송 지연이 초기전송보다 큼.
9. 결과: 수치해석(fc=1.2GHz, I=12, BW 100MHz, N=3~273)에서 peak throughput 디코더가 저rate/저lifting 블록을 제 시간에 복호 못함을 확인, 지연은 N 증가 시 증가.
10. 한계: HW 미설계, 순수 시스템 지연 해석뿐, BER/정정능력·gate 수치 없음, 5G NR rate-matching 규격 특화.
