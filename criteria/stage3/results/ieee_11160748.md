### B. 알고리즘 요약

1. 4G LTE(turbo)와 5G NR(LDPC, PDSCH), NR PDCCH(polar, PAC) Tx-Rx 체인에서 반복횟수 `I`(turbo/LDPC) 또는 list size `L`(polar/PAC)을 SNR에 맞춰 적응적으로 정하는 문제를 다룸.
2. 기존엔 코드별로 개별 조기종료/적응 기법만 있었고, 여러 디코더 타입에 공통 적용 가능한 통일된 방법이 없었다는 것이 동기.
3. 채널 모델은 3GPP AWGN 및 EPA-5Hz(LTE)/TDL-A(NR) 페이딩, `r(l,n) = h(l,n)x(l,n) + v(l,n)`.
4. 핵심 지표는 received bit information rate (RBIR): 각 참조신호(RS) 톤의 추정 SNR `Γj`을 mutual information LUT에 매핑해 평균한 값 `RBIR = Σ MI(Γj/γcod) / (m·N_RS)`.
5. `γcod`는 code rate별 보정계수(trial-and-error로 획득), MI는 SNR-MI LUT 참조.
6. 오프라인 단계: AWGN 채널에서 RBIR-SNR LUT, 그리고 각 후보 `Ki`(iteration/list size)별 SNR-BLER LUT를 구축해 목표 BLER를 만족하는 RBIR 임계값 `γ_RBIR,Ki`를 산출.
7. 온라인 단계: 실시간 추정 SNR로 RBIR을 계산하고 임계값들과 비교해 가장 작은 충분한 `K_Adap`을 선택, 해당 반복횟수/list size로 디코더 실행 후 종료.
8. 별도 학습(DE/GA/NN) 없이 Monte-Carlo 시뮬레이션 기반 LUT 구축만 사용.
9. 결과: LDPC 반복횟수 최대 54.7%, turbo 최대 67.4% 감소; polar/PAC list size 최대 ~92% 감소, BLER 성능은 고정 파라미터 대비 손실 없음.
10. 한계: LUT 기반 사전 계산 필요, 셀룰러 페이딩 채널·기존 통신 부호(turbo/LDPC/polar/PAC) 대상 시뮬레이션뿐이며 HW 구현·NAND read-channel 검증 없음.
