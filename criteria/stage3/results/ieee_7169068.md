### B. 알고리즘 요약

1. 대상 코드는 IEEE 802.3an (10GBASE-T) regular (2048,1723) QC 계열 H-matrix, `Wr=32`, fully-parallel Min-Sum Normalized 디코더.
2. Split-Row 기법은 각 check node를 `Ns`개 인스턴스로 분할해 라우팅 복잡도를 낮추지만, 각 인스턴스가 local min만 계산해 global unawareness로 오차가 생기는 문제가 있음.
3. 채널은 AWGN, SNR 스윕(2.6~3.8dB)으로 시뮬레이션.
4. 핵심 관찰: 한 check node 행에서 `min(|β|)`(M in1)를 만드는 variable node는 반복(iteration)마다 대개 동일 - 이를 만드는 인스턴스를 "truthful check node"(TCN)라 정의.
5. TCN이 계산한 `M in1`을 나머지 `Ns-1`개 인스턴스가 그대로 재사용("trust")하는 iteration을 trust iteration으로 지정, 해당 구간에서 나머지 인스턴스의 min 계산 트리를 게이팅해 동적 전력을 제거.
6. TCN 판별은 초기 iteration에서 한 번만 수행 후 이후 iteration까지 유지된다고 가정(하드코딩), 회로적으로 1클록 앞서 계산해 critical path 영향 없음.
7. 구현: TCN들 간 M in1 공유용 버스 추가, one-hot 방식으로 1개 송신·`Ns-1`개 수신 설정; 추가 HW는 gray 표시된 소규모 로직.
8. 최적 trust iteration 세트는 시뮬레이션으로 탐색("not too early, not too late" 경험칙), 본 논문은 (4,5,6,7) 채택.
9. 결과: 4-split 기준 활성 min 계산 트리 평균 32% 감소, 이는 check node 동적 전력 약 30~40% 감소로 이어지고 error performance도 소폭 개선(early termination 유발).
10. 한계: 알고리즘/게이트 레벨 시뮬 위주(실측/합성 결과 없음), Ns가 커질수록 TCN 판별 로직과 inter-CN 버스가 더 global해져 라우팅 복잡도 증가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Split-Row 기반 check node 분할 및 fully-parallel 구조 | 대응 없음 | Prime ECC는 HCU/GT 기반 partial-parallel(z_sb=32) 구조로 Split-Row 방식의 fully-parallel 인스턴스 분할과 다름 |
| CN min-sum 연산 (M in1/M in2) | `decoder.cpp` `CNU_Update_New_Mag()` | Min-Sum min1/min2 계산 개념은 공통이나 TTCN의 게이팅 최적화는 Split-Row 전용 |
| 동적 전력 게이팅(min 계산 트리 클록 게이팅) | 대응 없음 | Prime ECC 시뮬레이터는 bit-exact 기능 모델로 전력/게이팅은 프로파일 범위 밖 |

적용 가치: `낮음` — Split-Row 전용 저전력 게이팅 기법으로 Prime ECC의 부분병렬 아키텍처와 구조적으로 맞지 않고, 정정능력 개선도 부수적 수준.
