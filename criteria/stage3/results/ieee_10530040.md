### B. 알고리즘 요약 (10줄 내외)

1. BPSK/AWGN 채널에서 LDPC(576,432, rate 0.75)·BCH(127,106)/(63,45)·Hamming(7,3) 코드에 대해 NBP 계열(NNMS, NOMS)을 Tanner graph unfolding으로 신경망화한 기존 구조를 사용.
2. 문제의식: NNMS/NOMS는 학습된 가중치(`αv,c`, `βv,c`)와 레이어 수가 훈련 후 고정되어, 채널/노이즈 변화나 레이어 수 조정 시 전체 재학습이 필요함.
3. 가정: 채널은 가우시안 잡음의 BPSK-AWGN, 디코더는 `T`회 반복의 CN층(`(3)`,`(10)`)·VN층(`(4)`,`(11)`)·출력층(`(6)`,`(13)`)으로 구성된 unfolded 신경망.
4. 핵심 기법: 별도의 `l`층 dense hypernetwork `f(:,W)`가 입력 LLR을 받아 메인 NNMS/NOMS 디코더의 학습가중치(`α`, `β`)를 직접 생성 — 메인 네트워크는 재학습 없이 hypernetwork만 학습.
5. 핵심 식 `u^t_{c,v}=sign(...)·min(f(:,W_α^{(t-1)})×u^{t-1}_{v',c})` (식 14): CN 업데이트의 스케일 가중치 `α`를 hypernetwork 출력으로 대체.
6. 확장(dynamic hypernetwork): 레이어별로 독립된 hypernetwork 유닛 `G`를 두어 임의 레이어 수로 확장 가능하게 함(레이어 수 변경 시 재설계/재배포 불필요).
7. 부수 트릭: hypernetwork는 tanh 활성함수의 4-layer(32 뉴런) dense망, SGD와 cross-entropy loss로 학습, 초기 가중치는 zero-mean 단위분산 Gaussian.
8. 학습 절차: hypernetwork만 학습하고 메인 디코딩 네트워크 구조는 고정(unsupervised는 아니며 cross-entropy 기반 지도학습), TensorFlow 2.0으로 구현.
9. 결과: HyperNNMS는 원본 NNMS 대비 BCH/LDPC에서 고SNR 영역 BER/FER 약 0.1~0.2dB 개선(12/32-layer 실험), 반면 HyperNOMS는 BCH에서 오히려 0.5dB 이상 성능 저하.
10. 한계: 이론적 하드웨어 구현/합성 없음, 순수 시뮬레이션(Tensorflow)뿐이며 hypernetwork 추가로 저장·연산 복잡도가 증가(모델 압축은 별도 참고문헌 언급뿐 본 논문 미구현), 코드 길이도 576비트 단(短)-중간 길이로 NAND용 장길이 QC-LDPC와 스케일 차이가 큼.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NNMS/NOMS 학습가중치 생성(hypernetwork) | 대응 없음 | Prime ECC는 학습형 NN 가중치 없이 고정 min-sum + LLR 테이블(`ecc_data.h` PARAM_LLR) 구조라 hypernetwork 개념 자체가 대응 없음 |
| CN min-sum 업데이트(offset/normalize factor) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 골격은 유사하나 스케일 팩터가 학습형(hypernetwork)이 아닌 고정 테이블 기반이라 개념적 참고만 가능 |
| iteration별 적응형 파라미터(레이어별 가변 가중치) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 적응이라는 아이디어는 유사하나 실시간 신경망 추론이 아닌 사전계산 테이블 참조 방식이라 구조적으로 다름 |
```

마지막에 한 문장으로 **적용 가치**: `낮음` — 학습형 hypernetwork로 min-sum 가중치를 생성하는 소프트웨어/시뮬레이션 기법이며 Prime ECC의 고정 min-sum·HW 구조에 이식하려면 NN 추론 엔진 전체 도입이 필요해 실효성이 낮음.
