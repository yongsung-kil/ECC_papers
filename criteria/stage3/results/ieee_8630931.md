> 총평: IoT 이미지 JSCC용 DP-LDPC(R4JA/AR4JA) 유한길이 error-floor 최적화 논문으로, NAND binary QC-LDPC ECC와 도메인이 달라 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: IoT 무선 이미지 전송용 double protograph LDPC(DP-LDPC) JSCC, BPSK/AWGN, 짧은 블록(`N=400/720/3200/9600`), source rate `Rsc`+channel rate `Rcc`.
2. 문제: 유한길이·고 source 확률 `p`에서 source code 원인의 error floor가 심각(BER `5e-4` 수준).
3. 모델: joint base matrix `BJ=[Bsc BL1; BL2 Bcc]`, source LLR `Zvsc=log((1-p)/p)`, source 엔트로피 `H(s)`가 `Rsc`에 근접하면 floor 발생.
4. 핵심기법1: PEXIT-S를 유한길이용 `FPEXIT-S`로 개선 — 실제 확률 `p'=p+sqrt(p(1-p)/N)*Q^-1(1-β)`로 finite-length 효과 반영.
5. 의미: 무한길이 가정의 threshold 대신 길이 `N`·신뢰수준 `β` 반영 threshold로 짧은 블록 floor를 정확히 예측.
6. 핵심기법2: cascaded 2단 GA 최적화 — step1은 `(Bsc,Bcc)` 결합해 channel threshold 최소화, step2는 waterfall 손실 허용 하에 source threshold `pstep2` 최대화(floor 저감).
7. 구현디테일: `JBSC(·)`의 MI를 Monte Carlo 대신 curve fitting으로 근사해 최적화 시간 단축; PEG로 lifting.
8. 최적화: GA(교차 `pc=0.8`, 변이 `pm=0.2`, `Num=50`, `Maxgen=100`)와 PEXIT 분석 결합.
9. 결과: 짧은 블록 floor를 기존 대비 개선(`N=3200`이 타 방식 `N=6400/9600`과 유사 floor), 0dB에서 FLC rate allocation으로 PSNR 약 15dB, 추가 code 최적화로 약 10dB 이득.
10. 한계: HW 미설계·시뮬만, JSCC(source+channel 결합)·AWGN 특화, 일부 고Rsc/저Rcc code는 GA 최적해 미도달.
