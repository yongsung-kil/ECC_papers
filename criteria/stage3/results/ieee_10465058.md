### B. 알고리즘 요약

1. 시스템: 5G NR QC-LDPC, base graph BG1(46×68, Hcore 4×26) / BG2(42×52), lifting `Z`=2~384(51종), 각 `Z`행=1 layer로 병렬.
2. 문제: 기존 flexible 5G 디코더는 short code(작은 `Z`)에서 처리유닛이 놀아 처리율·자원활용률(HUE)이 낮음.
3. 채널/복호모델: 계층복호(layered) + normalized min-sum(NMS), `L_c2v = α·M(i)·∏sgn`, `α∈[0.5,1]`, min1/min2만 계산.
4. 핵심기법 1단(reconfigurable 구조): [8]의 segmented shifter를 개조해 제어신호를 path별 분리·재그룹, `Z<Zmax`일 때 서로 다른 길이 codeword를 concat해 동시복호 → HUE↑.
5. 의미: 384 path를 6 segment로 분할, 최대 6개 입력 동시복호(단, lifting 합 ≤384)로 놀던 자원을 재사용.
6. 핵심기법 2단(dynamic planning): 계층 내 노드 재배열 - 이전 layer 갱신 미완료 노드는 rightmost로 이동(conflict node), shared node는 NPU 2단에서 우선 갱신하여 메모리 read/write 충돌 회피.
7. 부수트릭: offset permutation으로 LLR 원위치 복귀 생략, 4-stage pipeline shifter, ping-pong으로 sign bit 선캐싱, segment별 독립 XOR early termination, buffer로 완료 codeword freeze.
8. 학습절차: 없음.
9. 결과: idle cycle 10→3 감소; Z=192에서 평균 throughput 23.713Gbps(비교군 [3] 16.250Gbps), Zmax에서 [7] 대비 Mbps/kLUT +25.14%·Mbps/BRAM +165.5%, Fclk 346MHz, LUT 122839, quantization (9,9,6).
10. 한계: 정정능력(BER) 곡선 없이 HUE/throughput만 보고, 시뮬 아닌 FPGA 구현 결과이나 부호성능 개선은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered NMS 복호(min1/min2, α scaling) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | min-sum 코어는 이미 보유, α scaling은 코드상 명시 미확인이라 참고 가능 |
| dynamic planning 노드 재배열(메모리 충돌 회피) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 우리 z=32/HCU/pipeline의 메모리 라우팅 개선에 개념적 적용 여지, 이식 난이도 중 |
| reconfigurable permutation shifter(다길이) | 대응 없음 | Prime ECC는 단일 고정 QC-LDPC라 다코드 shifter 불필요 |
| segment별 XOR early termination | `partial_CRC.cpp`, `full_CRC.cpp` | 조기종료 개념 대응, 우리는 CRC 기반이라 방식 상이 |
| ping-pong sign-bit 선캐싱 | 대응 없음 | 6-stage pipeline과 다른 별도 메모리 스킴 |
| HW edge/MUX 스케줄 | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 edge 선택과 유사 목적(클록당 처리 최적화) |

적용 가치: **중간** — dynamic node reordering을 통한 메모리 충돌 회피 아이디어는 HCU/GT 기반 파이프라인의 라우팅 병목에 참고할 여지가 있으나, 핵심인 다길이 동시복호 유연성은 단일 고정 부호를 쓰는 NAND용에 불필요하고 부호성능 개선도 없음.
