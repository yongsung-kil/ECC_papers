### B. 알고리즘 요약

1. AWGN/BPSK 채널에서 중간길이 LDPC 부호 (96,48) 및 QC-LDPC (155,62)의 error floor를 낮추는 것이 목표.
2. 문제: trapping set이 iterative decoder의 error floor 주원인이며, 기존 대책(코드길이 증가·새 check node 추가)은 원부호 파라미터를 바꾸고 설계 복잡도를 키움.
3. 핵심 아이디어: `(a,b)` trapping set은 겹치는 short cycle 구조를 포함하므로, cycle을 깨면 trapping set이 제거된다는 관계를 이용.
4. 식별 단계: iterative decoding 기반이 아닌 graph search로 Tanner graph의 구조적 trapping set을 전수 탐색.
5. 제거 단계: 변수노드 `v`의 인접 even-degree check node `c`를 다른 check node `c'`로 재배치 → 해당 check node degree가 even→odd로 바뀌어 trapping set 소멸.
6. 제약: 재배치 대상 `c'`는 같은 trapping set subgraph의 check node가 아니어야 하고(`c' ∉ CTi`), 새 short cycle을 만들면 안 됨.
7. PEG 사상 차용: 변수노드 `v`를 깊이 `l` subgraph로 확장(`N_l(v)≠∅, N_{l+1}(v)=∅`)해 최단 cycle 길이를 최대화하는 `c'` 선택.
8. 알고리즘 step4에서 재배치 후 최단 cycle이 girth `g`보다 짧으면 edge를 원복해 더 작은 trapping set 유입 방지.
9. 결과: (96,48)은 cycle-6 16개 감소·cycle-8 4개 증가, BER=10^-5에서 약 0.5 dB 이득; (155,62)는 약 0.2 dB. 개선 decoding 알고리즘 결합 시 고SNR BER 약 1.5 orders 추가 감소.
10. 한계: HW 미설계, 소규모 부호 시뮬만, trapping set 다수 시 완전제거 불가로 개선 제한, QC 구조가 unstructured로 붕괴.
