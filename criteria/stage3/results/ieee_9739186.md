### B. 알고리즘 요약

1. 대상 시스템: OpenAirInterface(OAI) 기반 5G NR open-RAN, PUSCH(ULSCH) 수신 체인의 LDPC FEC 디코딩.
2. 문제: 물리계층 중 LDPC FEC가 가장 무거워 단일 CPU에서 채널복호가 RX 처리시간의 91%(예: MCS28, 106 PRB에서 965 μs) 차지 → 가상화 RAN 병목.
3. 채널/설정: AWGN 및 TDL-C(30ns delay spread), 30kHz SCS, 273 PRB(100MHz BW), 12 OFDM symbol, DMRS type1.
4. 핵심 기법: LDPC 디코딩을 Xilinx T1 telco accelerator card(ZU19EG MPSoC + ZU21DR RFSoC)로 오프로딩.
5. 인터페이스: DPDK bbdev API를 통해 OAI ULSCH descriptor를 LDPC 디코딩 파라미터로 변환하는 shared library 구현(PCIe 연결).
6. 참조 디코더: OAI soft LDPC 디코더(iteration 10), thread-pool로 세그먼트 병렬 복호.
7. T1 카드 스펙: 50Gbps Fronthaul, 17.7Gbps 인코더, 7.8Gbps 디코더 throughput.
8. 성능 결과: 10% BLER 도달 SNR 기준 soft LDPC와 T1 하드 디코더가 대체로 동등(저/고 MCS에서 soft가 약간 우수, outlier 가능).
9. 시간 결과: 오프로딩 디코더가 최대 2배 빠름(고 MCS는 이득 작음) — code block을 하나씩 순차 전송하고 결과를 기다리는 구현 제약 때문.
10. 한계: 부호설계/디코더 알고리즘 기여 없음, HARQ 미통합, transport block 일괄 전송 모드 미지원(드라이버/비트스트림), 순수 5G 무선 시스템 통합.

| Xilinx T1 FPGA 하드 디코더 | 대응 없음 | 특정 상용 telco accel 카드, Prime의 z=32 QC-LDPC HW 모델과 무관 |

| DPDK bbdev/shared library 통합 | 대응 없음 | 소프트웨어 오프로딩 인프라, ECC 알고리즘 밖 |
