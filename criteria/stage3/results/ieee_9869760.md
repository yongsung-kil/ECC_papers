### B. 알고리즘 요약

1. 시스템: PLDPC-Hadamard code(PLDPC-HC), BPSK, AWGN. rate R=0.0494, r=4(d=6), 이중 lifting z1=32·z2=512, code length l=1327104. 심우주/IDMA 등 극한 채널용.
2. 문제: LDPC-Hadamard는 Shannon 한계(-1.59dB)에 근접하나 HW 복호기가 없어, PLDPC-HC 계층복호를 실제 HW로 구현.
3. 모델: base matrix를 z1(병렬엣지 제거)·z2(CPM으로 QC화) 이중 lifting, 각 H-CN이 d개 P-VN + (2^r-d)개 degree-1 Hadamard VN(D1H-VN) 연결.
4. 핵심 기법: symbol-MAP Hadamard sub-decoder를 커널로, 채널·a priori LLR로 `L_app^H` 계산 후 계층별 `L_ex` 갱신. min-sum이 아니라 FHT+DFHT 변환 기반.
5. 식 의미: DFHT 내부 `max*(a,b)=ln(e^a+e^b)`(Jacobian logarithm) = max + LUT(ln(1+e^-|a-b|))로 exact-MAP을 근사, Hadamard 행렬의 butterfly 구조 활용.
6. 확장: 두 파이프라인 케이스 — Case I(tloading≤t1st, FIFO 불필요) / Case II(FIFO로 갱신 LLR 임시저장), latency·throughput 식 유도(식14~19).
7. 구현 디테일: LLR을 z2 크기 세트로 나눠 Nh개 RAM에 뱅킹, CPM 오프셋 p를 cyclic shifter로 인터리빙(qu/re 분할), dual-port RAM으로 접근충돌 회피.
8. 최적화: 고정소수점 비트폭 S1→S2(정수부 +1bit)→S3(소수부 +1bit)로 error-floor를 BER 10^-2→2×10^-8까지 제거.
9. 결과: Xilinx VCU118, fc=130MHz, Nh=128에서 throughput 1.48Gbps(I=20), BER 10^-5 @ Eb/N0=-0.40dB. sub-decoder 완전활용 시 최대 4.5Gbps. 고정소수점 저하 0.08~0.10dB.
10. 한계: ASIC gate count 없음(FPGA LUT/BRAM %만), 초저rate 특수부호, 복호커널이 min-sum 아닌 Hadamard FHT/DFHT.
