### B. 알고리즘 요약

1. 대상은 임의 linear block code(논문 평가는 5G CRC-aided polar `(128,105+11)`, rate 0.75~1)로, 부호구조에 무관한 **code-agnostic** 복호.
2. 풀려는 문제: capacity-approaching 부호마다 전용 디코더가 필요한데, GRAND 계열은 부호에 무관하게 어떤 linear code든 복호 가능하고 short-frame(M2M)에 매력적.
3. 핵심 아이디어: 수신어를 직접 복호하지 않고 **잡음(error pattern)을 추측**하여 `y⊕e`가 codeword인지 `H·(y⊕e)==0`로 검증(GRAND); ORBGRAND는 soft-LLR로 error pattern을 **logistic weight(LW=뒤집힌 인덱스 합)** 오름차순 생성.
4. 핵심 변수: LLR 절댓값 오름차순 정렬 permutation `ind`, 최대 logistic weight `LWmax`, integer partition `λ=(λ1..λP)` — 각 `λi`가 뒤집을 인덱스.
5. 복잡도 파라미터: query 수는 `LWmax`와 partition 크기 `P`에 의존, `LWmax` 128→64 시 복잡도 355× 감소·성능 0.2dB 열화, `P≤6`로 추가 축소(FER=10^-7 무시할만한 열화).
6. HW 확장: distinct integer partition을 병렬 생성하는 **3개 shift register**(`P=3` 대응) 구조로 특정 LW의 크기 2·3 partition을 1 time-step에 모두 검사; `P>3`는 controller가 `sc` 결합.
7. 구현 디테일: 입력 LLR 5bit 양자화(sign 1 + frac 3), `n×(n-k)` H-메모리로 임의 부호/rate 로드, bitonic sorter(`log2(n)` stage)로 LLR 정렬, XOR gate array로 syndrome 병렬 계산.
8. (학습 절차 없음 - 순수 조합적 탐색 복호)
9. 결과(TSMC 65nm ASIC 합성): `n=128`, LW≤64/P≤6에서 avg throughput 42.5Gbps, area 1.82mm2, avg latency 2.47ns(SNR 10dB); DSCF polar 대비 49× throughput, GRANDAB(AB=3) 대비 FER 2dB 우수.
10. 한계: short code(n=128) 고rate·soft-input 특화, 부호가 길어지면 query 폭증, LDPC 특유 sparse 구조를 전혀 활용하지 않는 별개 복호 패러다임.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ORBGRAND noise-guessing 복호 전체 | 대응 없음 | Prime ECC는 min-sum message-passing(`LDPC_Decoder()`), error-pattern 추측 복호 미보유 |
| syndrome `H·(y⊕e)` 검증 | 대응 없음 | min-sum CN 연산(`CNU_Update_New_Mag()`)과 무관한 전수 syndrome 체크 |
| soft-LLR 정렬/양자화 | `channel.cpp` `Set_LLR_Th()` (개념적 접점만) | LLR 양자화 개념은 닿으나 정렬기반 사용법 상이 |
| shift-register/XOR HW 병렬 아키텍처 | 대응 없음 | GT.cpp의 MUX edge 선택과 목적 다름, QC-LDPC pipeline과 정합 안됨 |

적용 가치: **낮음** — ORBGRAND는 부호구조를 쓰지 않는 code-agnostic ML근사 복호로, min-sum QC-LDPC 기반 Prime ECC와 알고리즘 계열이 완전히 달라 이식 대상이 없고 short high-rate code 특화라 NAND 긴 부호에도 부적합.
