### B. 알고리즘 요약

1. (2209, 2024) array LDPC code, (4,47) regular, p=47 소수 기반 순환치환(circulant) 행렬로 구성된 PCM 사용, rate≈0.916.
2. 완전병렬 구조는 배선 복잡도 문제, 기존 partially-parallel 구조는 메시지 메모리량과 처리량 한계라는 문제를 해결하려 함.
3. AWGN 채널 가정, modified Min-Sum 디코딩 알고리즘 사용(스케일링 팩터 α로 sum-product의 tanh 연산을 근사).
4. 핵심 기법: CPU(check node processor) 188개를 47개씩 4개 그룹으로 나누고, 각 그룹 내 CPU들을 체인으로 연결해 이웃 CPU와만 통신(`CPU-i ↔ CPU-(i±2)` 등)하도록 설계 — array code의 규칙적 shift 구조 이용.
5. 핵심 관찰: PCM의 순환행렬 구조상 row 연산 결과가 매 클록 shift되며 전달되므로, 47클록에 걸쳐 check node 연산을 47단계로 분할(`Rcv` 식 (1))해 처리 가능.
6. VPU는 조합논리로만 설계, 매 클록 4개의 check-to-variable 메시지를 받아 variable-to-check 메시지(식 (2))를 계산.
7. 구현 디테일: CPU는 min1/min2 비교(Part 1) + 부호/인덱스 기반 메시지 출력(Part 2)의 2파트로 극도로 단순화, sign은 FIFO에 push.
8. 별도 학습/최적화 절차 없음(고정 array code 구조 활용).
9. 결과: 메모리 15,228bit로 기존 대비 70%+ 절감(기존 53,016bit), throughput 117.5Mbps @ 50MHz·20 iteration, 6bit 양자화 손실 0.2dB@BER=1e-7.
10. 한계: 특정 (2209,2024) array code 한정 실험, 정정성능/iteration 개선 언급 없음(구조는 QC-LDPC로 확장 가능하다고만 언급).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified Min-Sum (min1/min2 스케일링) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 보유한 min-sum과 동일 기법, 신규성 낮음 |
| CPU 체인 기반 로컬 통신(라우팅 단순화) | 디코더 HW 모델 (z=32, HCU, GT) | 배선 단순화 아이디어로 GT/HCU와 목적 유사하나 구현 방식 상이 |
| array code(순환치환 PCM) 구조 | `ecc_top.cpp` `Load_PCM()` | 특정 array code 고정 구조로 QC-LDPC(base+shift)와 개념은 유사하나 이식 난이도 높음 |
| 6bit soft 양자화 | `channel.cpp` `Set_LLR_Th()` | Prime ECC의 soft-2~3bit 범위 내로 참고 가능 |

> 적용 가치: 낮음 — 이미 보유한 min-sum 알고리즘의 재구현이며, array code 특유의 체인 통신 구조는 Prime ECC의 QC-LDPC/HCU/GT 구조와 이식 정합성이 낮음.
