> 총평: BP를 EP tree-structure로 개선해 finite-length에서 waterfall·error-floor 동시 감소를 보이나, 지수복잡도 cutset과 채널별 tree 구성 heuristic이 우리 min-sum HW 파이프라인과 근본적으로 안 맞아 이식성 낮음.

### B. 알고리즘 요약

1. 대상: rate `r=1-k/n` binary LDPC, BI-AWGN 채널, (3,6)-regular 및 irregular(λ=1/6x+5/6x³, ρ=x⁵), `n=128~1024`.
2. 문제: BP는 loopy Tanner graph에서 suboptimal이고 finite-length short 코드에서 성능 저하 — 대안 추론기법 부족.
3. 모델: posterior `p(V|y) ∝ p(y|V)·ΠCj(V)` (식 (1)), 이를 tree factorization `q(V)=Πqi(Vi|Vpi)` (식 (2))로 근사.
4. 핵심: EP로 inclusive KL divergence `DKL(p||q)` 최소화 (식 (3)) → moment matching, parity factor `Cj`를 `Wj=Πwz,j` (식 (4))로 치환.
5. 절차: iteration마다 factor `m` 골라 auxiliary `f(V,ℓ,m)=Cm·q^{ℓ-1}/Wm` (식 (5)) 계산 후 KL 최소화로 `q^ℓ` 갱신(식 (6)).
6. 확장: `f`가 cycle 형성 시(Fig.1b) Pearl's cutset conditioning으로 loop 변수 가정해 message passing — 복잡도는 가정 변수수에 지수.
7. 트릭: tree-structure 구성 heuristic — BP 미수렴 시 llr≈0/오류 check 연결 변수를 seed로 이웃 연결해 loop 직전까지 chain 형성(§III).
8. 대비: GBP(cluster variation)는 좋은 LDPC에서 size-4 loop 교집합이 없어 개선 불가하나, TEP는 4보다 큰 loop에도 pairwise 정보 전파해 개선.
9. 결과: (3,6) BER=10⁻⁴ waterfall gain 0.2~0.3dB, error-floor는 약 한 자릿수 개선; loop 크기 대개 d=2라 평균 iteration 비용은 BP의 거의 동등(cTEP=2^{d-1}L+(1-L)).
10. 한계: HW 미설계, cutset 지수복잡도(최대 `2^{rmax-1}`), tree 구성이 채널별 heuristic, moderate-length short 코드에 특화.
