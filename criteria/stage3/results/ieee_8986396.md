### B. 알고리즘 요약

1. 시스템: `(N,K)` polar code(예 `(1024,512)`, `R=1/2`)를 AWGN 채널에서 평가, inner code로 변환행렬을 삽입.
2. 문제: polar code의 SC/CRC-list-SC 복호는 직렬이라 latency가 크고 throughput 제한, 원래 BP는 parity-check 특성이 없어 성능 낮음.
3. 목표: polar 인코딩 결과를 등가 LDPC code로 변환해 병렬 BP 복호를 직접 적용.
4. 핵심 기법: rate-1 inner 변환행렬 `Tinner` 설계 — `Gpolar·Tinner·Hldpc^T = 0` 조건을 만족하도록 구성.
5. 유도: `Gpolar`를 `[GL GR]`, `Hldpc`를 `[HL HR]`로 분할하고 `Tinner`의 대각 블록 `U`, `D`만 사용, `D=I`로 두면 `U=(GL)^-1·GR·HR^T·(HL^T)^-1`.
6. 의미: 이 조건이 성립하면 수신열에 LDPC의 `Hldpc` 기반 BP 복호를 그대로 쓸 수 있음.
7. 확장: BP로 복호 실패 시 `(Tinner)^-1` bipartite graph를 따라 LLR을 message passing으로 전달, 이후 polar SC 복호로 잔여 오류를 추가 정정.
8. 결과: `(1024,512)`에서 LDPC 20-iter만으로 [6] 및 CRC-list-SC(L=2) 수준 FER; 50-iter면 BLER 크게 개선.
9. 한계(BER): SC 추가 복호는 error propagation 탓에 BER가 LDPC 단독보다 오히려 나빠짐.
10. 한계(범위): 설계 흐름이 rate-1/2 polar code에만 적용, HW 미설계·시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar→LDPC 변환행렬 `Tinner` | 대응 없음 | polar code 전용 인코딩 변환, binary QC-LDPC 부호구조와 무관 |
| 등가 LDPC의 BP 복호 | 대응 없음 (full-BP, min-sum 아님) | Prime ECC는 min-sum 기반, polar 그래프 결합 구조와 정합 안 됨 |
| SC 복호 후처리 | 대응 없음 | polar successive cancellation, LDPC 디코더에 없음 |

적용 가치: **낮음** — polar code 전용 변환 기법이며 NAND binary QC-LDPC 부호설계/디코더/HW 어디에도 떼어 쓸 대응 요소가 없음.
