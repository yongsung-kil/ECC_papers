### B. 알고리즘 요약

1. 대상은 3D TLC NAND flash(cell당 3bit, LSB/CSB/MSB page, 8-state Gray code), ECC(BCH/LDPC)는 codeword당 최대 120 error bits/1KiB 정정 한계를 가진 배경으로만 언급.
2. 풀려는 문제: retention/read-disturb/program-interference로 인한 threshold voltage shift(특히 가장 취약한 G state)를 ECC나 read-retry는 사후 교정만 할 뿐 셀 전하분포 자체를 복구하지 못함.
3. 핵심 관찰(모델): G state는 offset이 가장 크게 shift(-16, 타 state는 최대 -10)하며, G state를 재프로그래밍하면 인접 셀에도 program interference로 전하가 보충되어 MSB뿐 아니라 CSB/LSB도 영향받음.
4. 핵심 기법 CellRejuvo: 대상 워드라인을 읽어 LSB/CSB/MSB 비트패턴이 `1-0-1`(G state)인 위치만 식별(`PD[i] = ~LSB[i] & CSB[i] & ~MSB[i]`), 그 위치만 G state로 재프로그래밍하고 나머지는 Erase(전하 미충전) 상태로 프로그래밍.
5. 재프로그래밍 시 CSB 페이지에 PD를, LSB/MSB 페이지에는 전부 `1`(Erase)을 실어 프로그램 명령을 내리는 것으로 G state 셀만 실질적으로 충전됨 — 표준 wordline 프로그램 명령 재사용, 새 HW 불필요.
6. 확장 기법: 재프로그래밍 대상 워드라인 밀도로 DCRD(모든 wordline, 조밀)와 SCRD type-1/type-2(4개 layer 중 1개씩 선택적, 성긴)를 제안 — DCRD는 LSB 오류를 악화시키는 부작용이 있어 SCRD가 이를 완화.
7. 구현 디테일: 96-layer 블록(384 wordline) 기준 SCRD type-1은 매 layer의 두 번째 wordline, type-2는 짝수 layer만 선택(`i+9`/`i+7` 인덱싱)해 재프로그래밍 빈도와 program interference 트레이드오프 조절.
8. 학습/최적화 절차 없음(경험적 실측 기반 설계, G state 선택은 사전 실측 관찰(Fig.4, Fig.7)로 결정).
9. 결과: 실제 SSD에서 평균 RBER 38.28% 감소(SCRD Type-1 최대), late-stage 평균 read latency 21.86% 개선(최대 30.53%, rsrch_1 워크로드).
10. 한계: ECC/디코더 자체를 다루지 않고 셀 전하 레벨의 재프로그래밍(FTL/컨트롤러 레이어) 기법이며, MSR-Cambridge 워크로드 기반 실측이나 GC/wear-leveling 등 FTL 내부 동작은 미고려.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| G state 선택적 셀 재프로그래밍(DCRD/SCRD) | 대응 없음 | 셀 전하 분포를 물리적으로 재조정하는 FTL/컨트롤러 레벨 기법으로 LDPC 부호설계·디코더·인코더 어느 것과도 대응 없음 |
| BCH/LDPC 정정한계(120bit/1KiB), read-retry 비교언급 | ecc_top.cpp, decoder.cpp `LDPC_Decoder()` | 배경 설명·비교대상일 뿐 논문이 개선하는 대상이 아니므로 참고용 |
| RBER 감소로 인한 read latency 개선 | 대응 없음 | ECC 반복횟수/read-retry 감소 효과는 있으나 Prime ECC decoder 내부 알고리즘 변경이 아닌 입력 RBER 자체를 낮추는 상위 레이어 기법 |
```

적용 가치: **낮음** — LDPC ECC의 부호설계·디코더·HW를 개선하는 논문이 아니라 3D NAND 셀 재프로그래밍으로 입력 RBER 자체를 낮추는 flash reliability(FTL 레벨) 기법이라 Prime ECC 코드 이식 지점이 없음.
