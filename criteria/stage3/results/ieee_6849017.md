### B. 알고리즘 요약

1. 대상은 spatially-coupled(SC) LDPC 부호(LDPC convolutional 부호의 terminated 형태)로, ribbon형 `H` 구조 덕분에 window decoding 가능.
2. 문제: 기존 연구는 (3,6)-regular SC에 집중 — irregular SC 앙상블이 더 높은 임계값을 가질 수 있는지 규명.
3. 채널/분석 모델: BEC 상에서 density evolution(DE)으로 임계값 계산, AWGN에서 BER 시뮬로 검증.
4. 핵심 기법: 프로토그래프 base matrix `B[L]`을 copy-and-permute로 확장, 서브행렬 원소 `0≤bi≤9` 제약 하에 모든 memory 1·2 앙상블을 DE로 탐색.
5. 임계값 정의: window decoding threshold `ε*wd` = window 내 목표 심볼 erasure 확률을 `δ=1e-6` 이하로 낮추는 최대 채널 erasure율.
6. 확장: check node degree `dc`를 6~10으로 고정하고 각 degree별 최고 임계값 앙상블 탐색(Table V).
7. 구현 디테일: `H` 생성에 random permutation(저복잡도, but stopping set 유발) vs PEG(girth 최대화로 short cycle 제거) 두 방법 비교.
8. 최적화: DE로 100만개(ms=2) 앙상블 전수 탐색, 임계값 0.4993 이상 16개 발견 — 단 큰 `dc`가 short cycle/stopping set 유발.
9. 결과: irregular SC(예 [12 26], `dc`=11)가 regular SC 대비 Shannon 한계에 근접(gap 0.0019); PEG 적용 시 AWGN error floor 소멸(3dB 이상).
10. 한계: HW 미설계, BEC 이론 + AWGN 시뮬뿐, rate~0.5 SC(block 아님) 구조라 NAND 고rate QC-LDPC와 직접 연결 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC 부호 구성 (base matrix copy-permute) | 대응 없음 (Prime ECC는 고정 binary QC-LDPC block, `ecc_top.cpp` `Load_PCM()`) | SC convolutional 구조는 block QC-LDPC와 상이 |
| PEG로 short cycle/error-floor 완화 | `ecc_top.cpp` `Load_PCM()` (H-matrix) | H 설계 개선 아이디어는 개념적 참조 가능하나 SC 특화 |
| window decoding (BP) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 block decoder라 window 스케줄 부적합 |

**적용 가치**: 낮음 — SC-LDPC convolutional + rate 0.5 + window decoding 구조로, NAND 고rate binary QC-LDPC block 코드베이스와 부호·복호 구조가 근본적으로 달라 이식 실익이 낮음.
