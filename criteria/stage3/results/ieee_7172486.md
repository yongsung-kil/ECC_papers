### B. 알고리즘 요약

1. 문제: haplotype assembly의 MEC(minimum error correction) 최적화가 잡음 채널을 통해 전송된 부호어를 복호하는 문제와 수학적으로 동일함을 관찰.
2. 배경: SNP fragment matrix `R`(read x SNP)에서 read select 벡터 `s`와 haplotype `h`를 결합한 메시지 `m=[h s]`를 GF(2) 위에서 생성행렬 `G`로 부호화, `c=[h s]G`로 관측치 `y=c+e`를 얻는 구조.
3. 채널 모델: BSC(binary symmetric channel) 유사 - SNP calling 오류를 codeword의 비트반전 오류(e)로 모델링.
4. 핵심 기법1(Bit-Flipping): 원래 LDPC 복호용으로 제안된 BF 알고리즘을 haplotype assembly에 적용 - parity check 위반이 가장 큰 마진을 가진 비트를 순차적으로 flip.
5. 핵심 기법2(Belief Propagation): `G`(패리티체크 행렬 불필요)를 그대로 사용하는 BP 알고리즘 설계, 메시지 `y_{i,j}=(1/2-p_e)(2R(i,j)-1)+1/2`로 read-error 확률을 인코딩.
6. 확장: BP를 polyploid(k>2 배수체) haplotype assembly로 확장 - select 변수가 다치(k-ary)로 바뀌며 multiplexer형 팩터노드 사용.
7. 복잡도: BF/BP 모두 haplotype block 길이 n에 대해 시간복잡도 `O(n)`, 공간복잡도 BF는 `O(n²)`, BP는 `O(n)`.
8. 성능한계 분석: cut(SNP coverage) 기반으로 오류확률/switch error rate(SWER)의 하한을 해석적으로 유도(식 6 등).
9. 결과: 1000 Genomes/Fosmid 실데이터에서 HapCUT·HapCompass 대비 MEC 점수 우수, 최대 40배 빠른 실행시간(chr.6에서 BP vs HapCUT).
10. 한계: 통신 채널/부호 설계가 아닌 유전체 데이터 구조에서 유도된 응용 도메인 특화 알고리즘, HW 미설계, LDPC 자체의 부호/복호 성능 개선이 목적이 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Bit-Flipping 알고리즘 (parity check 위반 기반 비트 반전) | 대응 없음 | Prime ECC는 min-sum 기반 soft-decision 디코더이며 hard-decision BF 계열 알고리즘을 별도 보유하지 않음 |
| Belief Propagation (tanh-rule, message passing) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 표준 BP/SPA 개념 자체는 이미 min-sum 근사로 구현되어 있어 중복 |
| Haplotype 데이터 특화 그래프 모델(SNP/read select) | 대응 없음 | NAND ECC의 부호/채널 구조와 무관한 응용 도메인 특화 모델 |

> 적용 가치: **낮음** — 통신이론의 디코딩 알고리즘을 유전체학 문제에 유추 적용한 응용 논문으로, 부호설계·디코더 성능·HW 어느 축에서도 NAND ECC에 이식할 신규 기법이 없음.
