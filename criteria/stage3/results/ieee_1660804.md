### B. 알고리즘 요약

1. Rate 4/5 irregular QC-LDPC, GF(31) 상 circulant matrix 기반, block size 930 (mb=6, nb=30, Z=31).
2. Layered(row-block) belief propagation 디코딩 시 인접 block row 간 동일 block column에 non-zero 서브행렬이 있으면 데이터 의존성이 발생해 병렬 처리량이 제한됨.
3. 기존 block row 재정렬만으로는 고rate 코드에서 의존성을 완전히 제거할 수 없다고 가정.
4. 핵심 기법: 서브행렬에 identity matrix 2개를 aggregate한 'II' 블록을 도입(ACM), cycle-4 'I'-circle 구조를 찾아 대각 위치 2개는 'II'로, 나머지 2개는 'O'로 치환.
5. `Tn,m = Ln,m - En,m + 2*E_new_n,m` (II인 경우) 식으로 bit update를 재정의해, 병렬 처리되는 두 체크노드의 결과를 `LnNew=(Tn,m1+Tn,m2)/2`로 합산해도 원래 알고리즘과 결과가 동일함을 증명.
6. 이 update 재정의는 BP뿐 아니라 min-sum, λ-min 등 임의의 소프트 디코딩 알고리즘에 적용 가능.
7. HW: extrinsic memory/variable node memory 이원화, shuffling/inverse shuffling network, bypass MUX로 클록 사이클 1개 절감.
8. Design example: Θ=3 임계치로 degree≥3 비트노드부터 aggregation 수행(GF(31), a=3, b=6).
9. 결과: BER=10^-5에서 non-ACM 대비 약 0.1dB coding gain, block row당 cycle 수 4→2로 감소, TSMC 90nm 300MHz 3-stage pipeline에서 930Mbps(20 iteration), datapath 복잡도는 20% 증가.
10. 한계: GF(31) 특정 알헤브라익 구성 예시 하나로만 검증, 대형 블록/타 rate에 대한 일반화 및 HW의 error-floor 성능 데이터는 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 서브행렬 aggregation(ACM) 부호구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC H-matrix 고정이라 구조 변경은 재검증 부담 큼 |
| 병렬 bit update 재정의(Tn,m 식) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | layered/row-parallel 스케줄과 결합 가능하나 Prime ECC의 Dual-Update 스케줄과 개념적으로 유사, 직접 대체는 낮음 |
| CN min-sum 자체 | `decoder.cpp` `CNU_Update_New_Mag()` | 논문은 BP 기준, min-sum 적용 가능성만 언급하고 실증 없음 |
| shuffling/inverse shuffling network | `decoder.cpp` HW pipeline (col_M1/Z/P1~P4) | 메모리 라우팅 구조가 유사한 목적이나 세부 대응 불명확 |

> 적용 가치: 중간 | 데이터 의존성 제거로 처리량 2배라는 아이디어는 유효하나, 부호구조(H-matrix aggregation) 자체를 바꿔야 해 Prime ECC의 고정 QC-LDPC 구조에 이식하려면 상당한 재설계가 필요함.
