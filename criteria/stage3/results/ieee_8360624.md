### B. 알고리즘 요약

1. 대상 시스템은 SLC/MLC NAND flash로 블록당 `L=128` 페이지, 페이지당 `k=8192` bit, 2회 write(1회 rewrite) 지원.
2. 풀려는 문제: 기존 WOM 코드[14]는 단일 LDGM 고정 rate라 invalid 페이지의 unprogrammed 비율이 rate보다 낮으면 재기록 실패 → 페이지를 버리게 됨.
3. 모델: NAND를 WOM 장치로 보고, 재기록을 binary erasure quantization(BEQ) 문제로 정식화. cell state를 erasure location vector `v`로 취급.
4. 핵심 기법: BEQ 인코딩을 LDGM 생성행렬 `G_Q`(dual LDPC의 `H`) 상의 message-passing으로 수행. `z=mH_Q^{-1}`, `r=BEC(z,v)`, `u=Q(r)`, `x=uG_Q+z`.
5. 핵심 식: 재기록 성공 조건은 erasure 패턴 `r`이 이분그래프의 stopping set을 포함하지 않는 것 → BP quantizer가 무손실 `D(r,uG_Q)=0` 해를 찾음.
6. 확장(다중 rate): `β+1`개 multi-rate BEQ 코드 `C(Q_0)..C(Q_β)`를 순차 적용, 실패 시 invalid 페이지를 추가해 길이를 늘리고 rate `R_{2,i}=1/(2+α+i)`를 낮춤.
7. 구현 트릭: fractional page `α`(0.125/0.25)로 유한길이 성능저하 보정, page selection으로 rewritable 페이지(local stopping set 없는)를 사전 선별.
8. 코드 구성: R4JA rate-1/2 base matrix를 protograph extension+puncturing으로 raptor-like RC-LDGM 구성, PEXIT(over BEC)로 최적 threshold protograph 선택, CPEG(lifting 1024, girth≥8)로 lifting.
9. 결과: writing efficiency `η`가 Rivest-Shamir(고정 1.333) 대비 최대 1.469(α=0.125)로 상승, θ 증가 시 1.311→1.443, MLC LLH에서 1.234.
10. 한계: HW 미설계, 수치 시뮬만, 오류정정이 아닌 재기록(WOM) 목적 - Prime ECC의 LDPC 오류정정과 층위가 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BEQ/WOM 재기록 인코딩 (LDGM message passing) | 대응 없음 | Prime ECC는 오류정정 LDPC 디코더/인코더로 WOM 재기록 계층 없음 |
| raptor-like RC-LDGM protograph 구성 | 대응 없음 | 부호설계가 erasure quantization용이라 QC-LDPC ECC H-matrix와 목적 상이 |
| page selection (invalid page 재사용) | 대응 없음 | 컨트롤러/FTL 층 기능, ECC 엔진 범위 밖 |

적용 가치: **낮음** — WOM 재기록 코드로 NAND 수명연장을 다루나 오류정정 LDPC 디코더/인코더와 목적·계층이 달라 Prime ECC 이식 대상 없음.
