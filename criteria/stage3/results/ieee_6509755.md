### B. 알고리즘 요약

1. 시스템: (3,6)-regular binary LDPC, 검증 부호 크기 `(50,25)~(400,200)`, rate `1/2`, AWGN+양자화(2~4bit LLR), 완전병렬 아키텍처, Xilinx Spartan-3E/Virtex-5 FPGA.
2. 문제: 대형 LDPC 디코더를 HDL로 수작업 배선/검증하는 것이 복잡·시간소모적 — 노드 간 상호연결이 큰 행렬에서 사실상 불가능.
3. 모델: min-sum 복호(sum-product 특수형), soft-decision 메시지패싱 + 최종 LLR 부호로 hard-decision.
4. 핵심기법: Matlab 스크립트가 LDPC 행렬을 읽어 V-node/C-node를 자동 instantiate·routing하여 Simulink 디코더 모델 생성 → HDL Coder로 VHDL/Verilog 자동생성 → FPGA 합성.
5. 핵심식: VN `Lcv = Iv + Σ Rmv` (식1), CN `Rcv = ∏sign(Lcv)·min|Lcv|` (식2) — 표준 min-sum.
6. 구현 트릭: `find3min` 모듈로 6입력 C-node의 최소값 계산을 재사용, 원본 대비 slice 105→56, LUT 186→98로 축소(XC3S1200E).
7. 라우팅: Simulink routing label 이름 매칭으로 V/C-node 포트 자동 연결(수작업 배선 제거).
8. 검증전략: 테스트데이터 생성→인코딩→AWGN 삽입→양자화를 자동화, Simulink/VHDL/FPGA 3계층 결과를 text 파일로 비교.
9. 결과: (200,100) 2-bit, 83.2MHz, 평균 3.7 iteration(3.5dB)에서 throughput `2.25Gbps`, 대형·고급 FPGA 시 10Gbps 초과 가능 주장; 합성표에 slice/LUT/FF/Fmax 제시.
10. 한계: 알고리즘 신규성 없음(방법론 논문), 완전병렬이라 소형 부호만 실장(≤400), 절대 gatecount@공정 없음, NAND/QC-LDPC와 무관.
