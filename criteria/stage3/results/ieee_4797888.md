> 총평: LDPC-BC가 아닌 LDPC Convolutional Code(무한 PCM, sliding-window 디코딩) 전용 90nm ASIC로 부호구조 자체가 Prime ECC의 QC-LDPC와 근본적으로 다르며, 정정성능 개선도 없는 순수 HW 구현/전력 최적화 논문이라 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: rate-1/2 (128,3,6) LDPC Convolutional Code(LDPC-CC, 열가중치3/행가중치6 규칙 부호, code memory `m_s`로 정의되는 constraint length), time-varying period 부호, 90nm CMOS ASIC.
2. 문제: LDPC-CC는 PCM이 무한/band-diagonal이라 LDPC-BC(block code)와 달리 sliding-window 기반 디코딩과 파이프라인 프로세서 체인이 필요, 기존 구현([20])보다 전력·성능 개선이 목표.
3. 모델: PCM `H`는 하삼각(lower-triangular) band-diagonal 무한 행렬, phase `t mod T`로 시변 parity 제약을 인덱싱; 정정 알고리즘 자체는 표준 min-sum.
4. 핵심 기법 1단(인코더): one-hot 위상 인코딩으로 각 사이클에 활성화되는 로직 비율을 최소화(3단 파이프라인 XOR tree)해 저전력·고주파수(1.1GHz) 달성, all-phase termination 회로로 프레임 종단 처리.
5. 핵심 기법 2단(디코더): 슬라이딩 윈도우 디코딩을 동일 구조의 프로세서 여러 개를 파이프라인 concatenation으로 구현, 프로세서 간 controller 공유(기존엔 프로세서마다 개별 controller)로 라우팅 혼잡 감소.
6. 부수 트릭(메모리): 단일포트 메모리만 사용(듀얼포트 대비 면적 1.5배 절감), graph-coloring 알고리즘으로 필요한 메모리뱅크 수를 최소화(24 read/write에 28뱅크 필요, LLR 분산도 균형화).
7. 부수 트릭(테스트): on-chip AWGN 노이즈 생성기(combined Tausworthe PRPG), BIST(CRC 시그니처, 가변길이 FIFO), 온칩 PLL로 외부 IO 한계(100MHz) 이상 속도 테스트 가능.
8. 정량화: 8-bit LLR 사용(6-bit로도 동등 BER 가능하다고 언급했으나 시간부족으로 미구현).
9. 결과: 인코더 1.1GHz@22mW(40% 에너지/bit 절감 vs 이전작), 3-processor 디코더 600MHz@369mW(0.61nJ/decoded-bit @SNR 2dB), throughput/area가 비교 LDPC-BC 디코더보다 10% 우수; BER은 약 1e-6에서 error-floor 관측(기존 LDPC-BC보다 뚜렷).
10. 한계: 정정능력/디코딩 알고리즘 자체의 개선 없음(표준 min-sum), 부호가 LDPC-CC(무한 PCM)로 QC-LDPC block code와 구조가 상이, 30-processor 확장은 시뮬레이션 추정치일 뿐 실측 아님, NAND/SSD 적용 언급 없음(streaming video/packet-switching/sensor network 타깃).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-CC 슬라이딩 윈도우 디코딩 / 무한 band-diagonal PCM | 대응 없음 (Prime ECC는 QC-LDPC block code, 고정 base matrix) | 부호 패러다임 자체가 상이해 재사용 불가 |
| min-sum 디코딩 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()` | 이미 보유한 표준 기법, 중복 기여 |
| 파이프라인 프로세서 간 controller 공유 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` (pipeline 6-stage와 유사 개념) | 컨트롤 로직 공유로 면적 절감 아이디어는 개념적으로 참고 가능 |
| graph-coloring 기반 메모리뱅크 최소화 | 대응 없음 (Prime ECC는 z=32 lifting 기반 병렬 메모리 구조) | 메모리 충돌 회피 최적화 개념만 참고 가능 |
| one-hot 인코딩 저전력 회로 | [encoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_encoder_4KB()` | 우리는 dual-diagonal 고정 방식이라 LDPC-CC 특화 인코더 로직은 직접 대응 없음 |

적용 가치: **낮음** — LDPC Convolutional Code는 무한/가변장 PCM과 sliding-window 디코딩이 핵심인 부호 패러다임으로 QC-LDPC block code 기반 Prime ECC와 근본적으로 다르며, 정정성능 개선도 없는 순수 ASIC 구현·전력 최적화 논문이라 부호설계·디코더 알고리즘 이식 가치가 낮다.
