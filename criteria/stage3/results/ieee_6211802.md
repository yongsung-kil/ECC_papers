### B. 알고리즘 요약

1. 대상은 LLR-domain BP/Min-Sum LDPC 디코더의 check node processing unit(CNPU)로, VNPU보다 훨씬 복잡해 디코더 전체 복잡도를 지배.
2. Min-Sum은 식 (2)의 `2 tanh^-1(∏ tanh(|l|/2))`를 `min(|l_j→a|)` 근사(식 3)로 치환, sign은 XOR(식 1)로 분리 계산.
3. 체크노드 출력 magnitude는 두 값만 가지므로(min 1st가 아닌 위치는 min 1st, 그 위치는 min 2nd), 핵심은 n개 입력에서 최소 2개값과 1st min index를 찾는 Minimum Value Generator(MVG).
4. 기본 블록 `2-MVG`: 비교기 1개 + i-bit MUX 2개로 두 입력의 min1st/min2nd/index 산출(MVU 정의).
5. `4-MVG`: 두 2-MVG(A,B) + Connection Unit(CU)로 구성, min1st=min{m1A,m1B}, min2nd는 pos에 따라 min{m1B,m2A} 또는 min{m1A,m2B} 선택(m2A와 m2B는 비교 불필요).
6. 일반화 `2^k-MVG`: tree-search 방식으로 2개 2^(k-1)-MVG + CU 재귀 구성. 2^k-MVG는 2-MVG용 2^(k-1) 비교 + CU용 3·2^(k-1) 비교.
7. 비-2의거듭제곱 `n=2^k+2^r` 구조도 제시(index1을 k-r zero로 left-pad 후 IndexG).
8. 구현은 SystemC 실행가능 스펙 -> Verilog -> Quartus II 9.0 합성(Stratix II GX EP2SGX90 FPGA), n=4~64, 비트폭 4~12bit로 파라미터 스윕.
9. 결과: 면적(ALUT/ALM)은 n에 선형 증가, critical path delay는 log(n)에 선형(ASIC 경향과 일치). CNPU 자원의 60% 이상이 두 최소값 탐색에 소모.
10. 한계: 부호설계·복호성능 개선 없음, throughput/BER 미제시, 특정 부호 스펙 없음. 순수 CNPU building block 코스트 분석에 국한.
