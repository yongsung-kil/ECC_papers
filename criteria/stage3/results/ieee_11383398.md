### B. 알고리즘 요약

1. IEEE 802.11n 규격 LDPC(N=2048, K=1723, R=0.84, row weight 6, column weight 5, regular)를 AWGN+BPSK 채널에서 사용.
2. 문제: 기존 직렬/균일분할 병렬 디코더는 고속 통신 환경에서 병렬도 부족, 자원 낭비, 다양한 코드 파라미터 적응성 부족.
3. 채널/노이즈 모델: AWGN, SNR 0~5dB 구간 시뮬레이션(BPSK 변조).
4. 핵심 기법 1단: 파리티체크 행렬 행별 positional entropy `H(j) = -Σ p_ij log2(p_ij)`를 계산해 체크노드를 엔트로피 내림차순 정렬 후 greedy 알고리즘으로 겹치지 않는 그룹으로 분할(비균일 그룹핑).
5. 엔트로피 식은 행 내 0이 아닌 원소의 분산/집중도를 정량화해 병렬 갱신 가능한 노드 부분집합(최대 독립집합에 가까운)을 찾는 데 사용.
6. 핵심 기법 2단: P×Q 재구성 연산 유닛(ROU) 어레이 + 우선순위 기반 동적 자원 스케줄링(`S = ω1·s + ω2·t`)으로 그룹 크기·연산시간에 따라 유닛 재할당.
7. 구현 디테일: 8-bit(N≤1024)/16-bit(N>1024) 가변 고정소수점, tanh/tanh-1은 LUT 기반 분할선형 근사(오차 3% 이내), pre-configuration으로 재구성 오버헤드 1클록 이하.
8. 조기종료: 연속 2회 반복 간 사후확률 변화가 임계값(10^-3) 미만이면 종료 → iteration 30% 감소, BER 열화 0.1dB 이내.
9. 결과: P=16에서 throughput 3.2Gbps(직렬 대비 4배), 자원활용률 85.2%(정적 대비 +32.9%p), 에너지효율 51% 개선, BER 열화 0.15dB 이내. Xilinx Kintex UltraScale KU040 FPGA(200MHz)로 검증.
10. 한계: 부호 자체는 802.11n 표준 그대로(부호설계 기여 없음), gatecount 등 면적 수치 미제공, sum-product(tanh) 기반이라 min-sum HW와 구조가 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 비균일 노드 그룹핑(엔트로피 기반 그룹 분할) | 대응 없음 | Prime ECC는 z=32 고정 lifting 기반 row-parallel 구조로 엔트로피 기반 가변 그룹핑 개념이 없음 |
| 재구성 가능 ROU 어레이/RIN(크로스바) | 대응 없음 | Prime ECC HW 모델은 고정 pipeline(col_M1/Z/P1~P4) 구조이며 재구성형 아키텍처가 아님 |
| tanh/tanh-1 LUT 기반 sum-product 연산 | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 min-sum 근사를 쓰므로 tanh LUT 자체는 대응 없음, CN 갱신 로직 위치만 참고 가능 |
| 동적 자원 스케줄링(우선순위 기반 유닛 할당) | 대응 없음 | Prime ECC는 z=32 고정 병렬 구조로 동적 재할당 스케줄러가 없음 |
| 조기종료(사후확률 변화 임계값 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념은 유사하나 Prime ECC는 CRC 기반, 논문은 확률 수렴 기반으로 메커니즘 상이 |

적용 가치: `낮음` — 부호설계 기여 없이 표준 부호 그대로 사용하며, 재구성형 FPGA 아키텍처 자체가 Prime ECC의 고정 QC-LDPC/min-sum HW 구조와 근본적으로 다른 패러다임이라 이식 실익이 적음.
