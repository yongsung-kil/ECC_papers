> 총평: 저전압(sub-threshold) 노이지 HW에서 flooded Min-Sum 디코더가 얼마나 견디는지를 gate-level fault injection으로 평가하는 신뢰성 분석 논문으로, 이식할 새 디코딩 알고리즘이 없어 이식성 하.

### B. 알고리즘 요약

1. 시스템: flooded Min-Sum LDPC 디코더, regular `(3,6)` 코드, base matrix `12×24`, circulant size `54`(→ 24 VNU, 12 CNU), rate `0.5`, BiAWGN 채널.
2. 문제: 전력절감 위한 near/sub-threshold CMOS는 게이트가 동작주파수 의존 확률적(비결정) 거동 → 디코더 HW 자체가 error-prone, 이 결함이 정정성능에 미치는 영향 평가 필요.
3. 모델: gate-level 확률 결함모델 GISP(Gate Input Switching Probabilistic, 입력 전이패턴별 상이 고장확률), Vdd=0.35V, 50℃, delay-dependent(critical/non-critical path별).
4. 핵심기법: CNU/VNU 처리유닛을 combinational/sequential 서브블록으로 계층분해, combinational 서브블록만 Synopsys DC로 gate-level 합성 후 각 게이트에 mutant(Vdd·delay·온도 함수) 삽입.
5. 근거: timing report로 critical path(최다 게이트 경로) 추출, path별 게이트 delay 산출해 현실적 타이밍 제약 기반 고장확률 부여.
6. 확장: 서브블록별 출력비트 고장확률(PTF)을 Monte-Carlo(대량 프레임)로 추출 → 원본 RTL의 각 CNU/VNU 출력에 saboteur 삽입해 계층적으로 디코더 전체 BER/FER 재시뮬.
7. 구현 디테일: fault 전파는 무시(각 combinational 서브블록은 정상 입력 수신), C++ 전송체인 모델과 RTL을 DPI-C로 cross-simulation.
8. 최적화: 별도 학습 없음(SFI 신뢰성 평가 방법론).
9. 결과: 300,000 프레임/캠페인. CNU 결함 200ns 제약 시 BER 10^-2에서 fault-free 대비 약 0.1dB 열화, 133ns 0.3dB, 100ns 0.5dB↑. VNU 결함 시 BER 10^-3~10^-4에서 error-floor 발생(CNU+VNU 동시에도 유지). 5MHz(200ns)에서 CNU sub-threshold 디코더는 fault-free로 수렴 경향.
10. 한계: 새 디코딩/부호 기법 제안 아님(신뢰성 평가), rate=0.5 regular(3,6) 저rate 코드로 NAND 고rate와 상이, gate count/throughput 본문 수치 없음.
