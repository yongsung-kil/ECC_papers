### B. 알고리즘 요약

1. 시스템: 1B-construction Generalized LDPC `(N,J,n)`, SPC check 대신 ext-Hamming 구성부호(예 (32,26,4),(16,11,4))를 GCN으로, BSC hard-decision, overall rate `R≥1-J(1-rc)`; N=2048/4096, Imax=20.
2. 풀려는 문제: SDD(BP)는 성능 좋으나 복잡도 과다, HDD인 BF는 저복잡도지만 성능 낮고 trapping set이 uncorrected error 유발; 부호 구성으로 trapping set 제거는 처리속도·복잡도 한계로 아직 불가.
3. 핵심 아이디어: trapping set을 부호에서 없애는 대신, 발생 시에만 추가 자원을 써 디코더에서 그 영향을 줄임(양측 state-aided).
4. 핵심 기법(Failure Analysis): ext-Hamming(dmin=4) 국소복호 실패를 P(정상단일오류)/U-set(검출불가)/A-set(오정정)/D-set(zero-syndrome 미검출)로 분류.
5. 식 의미: VN↔GCN 사이에 원래 비트 외 신뢰도 extra bit 1개 추가 → 4단계 신뢰도(1+,1-,0-,0+ = strong flip/weak flip/weak keep/strong keep)로 결정 강도 표현.
6. Horizontal(GCN) 처리: GCN 상태 `dc(l)`∈{0,1,2}(zero-syndrome/단일오류정정/실패)에 따라 메시지 송출, 국소 카운터 `αc(l)`(상태 반복수)·`βc(l)`(들어온 suspect 수)로 D/A/U-set 가정하 신뢰도 승강 및 재-syndrome 검사(Table I).
7. Vertical(VN) 처리: J=2, 4신뢰도의 10개 입력조합을 s1/s2/s3로 나눠 VN 상태 `rw(l)`∈{0,1,2} 결정, VN 카운터 `γw(l)`로 rw=1이면 flip 대신 suspect로 표시해 GCN에 신뢰도 전달(Table II,III).
8. 최적화 절차: 상태 반복 임계(2,3회) 등은 규칙 기반, 별도 학습 없음.
9. 결과: (32,26) subcode(R=0.625)에서 기존 HDD BF([12],[13]) 대비 0.5dB 이상 이득, (16,11)(R=0.375)에서 더 큰 이득, 15~20 iteration에서 수렴; 추가연산은 trapping set 발생 시에만 동작해 latency 영향 무시 가능 주장.
10. 한계: HW 미설계(BSC 시뮬만), ext-Hamming 구성부호 대수복호 의존, gate count/throughput 수치 없음, non-SPC GLDPC 구조 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC bit-flipping 디코더 전체 | 대응 없음 | 우리 디코더는 SPC check + min-sum message passing이며 Hamming subcode 기반 GCN/BF 구조와 불일치 |
| GCN 상태·양측 신뢰도 extra bit | `decoder.cpp` `VN_Cal_HD()` (HD 경로) | 개념적으로만 hard-decision 신뢰도 보강과 닿으나 GCN(구성부호) 전제라 직접 대응 불가 |
| trapping-set 영향 완화 상태카운터 | 대응 없음 | 우리는 min-sum post-processing/조기종료 사용, BF 상태카운터 접목 근거 없음 |

적용 가치: **낮음** — GLDPC(ext-Hamming 구성부호) 전용 hard-decision bit-flipping 기법으로, SPC check의 binary QC-LDPC min-sum 디코더인 Prime ECC와 부호·복호 구조가 근본적으로 달라 떼어 쓰기 어렵다.
