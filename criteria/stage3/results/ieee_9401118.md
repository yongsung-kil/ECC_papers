### B. 알고리즘 요약

1. 대상은 Wi-Fi/WiMAX/5G QC-LDPC 디코더의 재구성 가능 cyclic shifter(CN↔VN 메시지 permutation), lifting parameter로 codeword 길이 가변.
2. 문제: 기존 시프터는 한 번에 입력 하나만 shift → 입력 크기가 최대(ZMAX) 미만이면 HW 일부가 유휴, codeword도 한 번에 하나만 복호.
3. 모델/구조: left/right 로그시프터(각 `K=ceil(log2 ZMAX)` stage) + combiner로 임의 크기 cyclic shift 구현.
4. 핵심 기법: 로그시프터의 각 "path"별 제어신호를 분리하고, 연속 path를 재그룹해 segment 구성 → 각 segment가 서로 다른 shift값·입력크기를 독립 수행.
5. 핵심 식: `SR = CSR % Z`, `SL = Z - SR`, combiner `S[i]=1 if i < (SL + segment_offset_n)` — segment offset 반영이 다중 병렬화의 핵심.
6. 확장(sectioned): 384 path를 24 section(각 16 path)으로 묶어 제어신호 수를 `9×384×2=6912`에서 `9×24×2=432`로 축소, 유연성-복잡도 절충.
7. 구현 디테일: section 단위 제어로 최대 24개 입력 동시 shift, 큰 입력은 인접 section 결합; 조기종료(Early Termination)용 시프터를 별도 인스턴스.
8. 학습/최적화: 없음(순수 회로설계).
9. 결과: 0.18um CMOS 테스트칩(3.1×3.6mm, 96 path 구현), lifting 12/24/24/12 codeword 4개(기대 iter 1/3/3/1)를 40MHz 입력·100MHz 복호로 병렬 처리, 6개 codeword 동시 복호 실증; MUX 6786개, 10 logic level.
10. 한계: 디코딩 알고리즘·부호설계·정정성능·soft-read와 무관한 permutation HW 빌딩블록, 무선 가변길이 다중 codeword 특화, throughput/BER 수치 없음.
