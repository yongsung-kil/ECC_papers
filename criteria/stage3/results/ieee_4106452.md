### B. 알고리즘 요약

1. 시스템: SPIHT/JPEG2000 이미지 비트스트림을 LDPC(수평, irregular, rate ~0.469, N=8272)와 RS(수직, GF(2^8), UEP)로 구성한 product code로 BSC(BER=0.1) 또는 Gilbert-Elliot 채널을 통해 전송.
2. 풀려는 문제: 기존 IRA/product-code 기반 이미지 전송은 반복 복호 시 RS로 확정된 비트 정보를 LDPC 재복호에 충분히 활용하지 못함.
3. 핵심 가정: RS 복호 후 일부 패킷은 부분 복구되며, 그 결과 특정 비트값이 결정적(determinate)으로 알려짐.
4. 핵심 기법 1단: determinate 비트에 대해 SPA의 LLR `L(c_i)`을 매우 큰 양/음 값으로 강제(bias)하고 관련 `L(r_ji)`, `L(q_ij)`도 해당 부호로 편향시켜 LDPC 재복호 성능을 향상.
5. 핵심 식: `L(Q_i) = L(c_i) + Σ_{j∈C_i} L(r_ji)` — 결정적 비트는 `L(Q_i)`와 `L(c_i)` 부호를 일치시켜 강제, 이는 표준 marginal LLR 계산에 사전지식을 주입하는 방식.
6. 핵심 기법 2단: RS rate(각 블록의 protection level `Q(k)`) 할당을 반복 product 복호를 고려한 동적계획법(AEC, adaptively-estimated channel)으로 최적화 — 기존 APC(사전 채널조건 고정)보다 개선된 손실확률 `P_L(k)` 추정.
7. 부수 트릭: S-random interleaver(S=15)로 오류를 수평 방향에 고르게 분산시켜 determinate bit exploitation 효율 향상.
8. 최적화 절차: 트렐리스 기반 동적계획법으로 블록별 RS 심볼 수 `Q(k)`를 왜곡 최소화 관점에서 선택, 반복마다 손실확률 재추정.
9. 결과: BSC BER=0.1, 0.505bpp에서 기존 product code 대비 PSNR 약 1.5dB 개선, product 복호 반복은 통상 3회 이내로 추가 연산비용 작음.
10. 한계: LDPC 디코더 알고리즘 자체(min-sum/scaling 등)는 변경 없이 표준 log-SPA를 그대로 사용, HW 구현·양자화·NAND 채널 모델은 전혀 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Determinate bit LLR biasing (외부 정보 재주입) | 대응 없음 | Prime ECC는 단독 QC-LDPC ECC 엔진으로 RS와의 product code 구조나 외부 소스 결정 비트 개념이 없음 |
| RS rate allocation (UEP, 동적계획법) | 대응 없음 | 이미지 소스 왜곡 최적화 문제로 NAND ECC rate 설계와 무관 |
| Irregular LDPC decoder (log-SPA) | `decoder.cpp` `LDPC_Decoder()` | 표준 log-SPA 사용, Prime ECC의 min-sum 대비 이미 상위 성능이나 HW 비친화적(교과서 수준) |
| S-random interleaving | 대응 없음 | product code array 내 오류 분산 목적, Prime ECC 단일 코드워드 구조와 무관 |

적용 가치: **낮음** — 이미지 전송을 위한 LDPC+RS product code 프레임워크와 UEP 최적화가 핵심으로, LDPC 디코더 알고리즘 자체는 표준 log-SPA를 그대로 사용해 NAND 고전 binary LDPC ECC에 이식할 만한 새로운 기법이 없음.
