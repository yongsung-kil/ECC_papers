### B. 알고리즘 요약

1. 시스템: QC-LDPC 코드, superposition(SP) construction 기반 exponent matrix `P=[p_i,j]`, CPM(circulant permutation matrix) 크기 `L`(lifting size)로 base matrix `B`를 확장.
2. 문제: 기존 QC-LDPC는 하나의 exponent matrix가 단일 lifting size만 지원(5G만 예외적으로 5~8개 지원), 5G 방식은 large-lifting-size-first라 확장이 어려움.
3. 핵심 가정: L2=aL1(a는 양의 정수)일 때 작은 lifting size L1의 exponent matrix P1으로부터 큰 lifting size L2의 P2를 유도하는 small-lifting-size-first 전략.
4. 핵심 기법: `P2 = P1 + L1·W` (식 7), W는 weight coefficient matrix(`w_i,j∈Z_a`)로 P1과 동일한 base matrix를 공유해 동일한 반복복호 threshold를 가짐.
5. 핵심 식 의미: 정리1(Proposition 1)은 이 구성법이 girth를 비감소(`g2≥g1`)시킴을 증명; 정리2는 short cycle 개수의 비증가는 보장 못함을 보여 별도로 cycle distribution 최적화가 필요.
6. 부수 트릭: Algorithm 1은 W를 무작위로 갱신하며 cycle distribution `C(X)`가 개선될 때만 채택하는 탐색(`N_search`회 반복), 완전탐색 대신 무작위 탐색으로 축소.
7. 트레이드오프: 작은 base matrix+큰 lifting size는 부호가 지나치게 구조화되어 성능 저하 우려 → 짧은/중간 길이 부호의 우수한 threshold를 기반으로 긴 부호를 확장하는 실용적 접근.
8. 검증: CCSDS(R=1/2, K∈{1024,4096,16384}, L∈{128,512,2048})와 5G(R=2/3, K∈{1920,3840}, L∈{192,384}) 표준 부호를 대상으로 예제 설계.
9. 결과: 구성된 부호가 표준 CCSDS 대비 BLER≈10⁻⁶, BER≈10⁻⁸까지 유사 성능, 5G 대비는 더 낮은 error-floor 성능을 보임; 디코딩은 offset min-sum(OMSA, correction factor 0.5, 최대 50 iteration).
10. 한계: 이론/시뮬레이션 수준(HW 미설계), short cycle 비증가는 보장되지 않아 별도 metric 필요, 표준 부호(CCSDS/5G) 대체 목적의 구성법이라 특정 응용 H-matrix에 직접 대입 시 재설계·재검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| exponent matrix 다중 lifting size 구성법 (`P2=P1+L1W`) | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 설계 영역과 직결되나 Prime ECC는 특정 QC-LDPC(z_sb=32) 고정이라 이식 시 새 H-matrix 재검증 필요(이식 난이도 높음) |
| girth/cycle distribution 최적화 | `ecc_top.cpp` `Load_PCM()` | 부호 구조 품질 지표로 동일 모듈 범주지만 코드 자체는 Prime ECC 자체 최적화 로직 없음 |
| OMSA(offset min-sum) 디코딩 | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC의 min-sum(min1/min2+sign XOR) 구조와 원리 유사(min-sum 계열), correction factor 0.5는 이미 보유한 min-sum 변형과 중복 기여 가능성 |
| 다중 lifting size로 저장복잡도 절감 | `ecc_data.h` (PCM 구조체) | 여러 code length용 PCM을 하나의 exponent matrix로 저장하는 아이디어로 메모리 라우팅과 연관 |

적용 가치: 중간 — QC-LDPC exponent matrix 다중 lifting size 구성법은 부호설계 레이어(이식 난이도 높음)에 속하나, 저장복잡도 절감 아이디어와 OMSA 디코딩이 Prime ECC의 QC-LDPC/min-sum 구조와 원리적으로 정합되어 있어 참고 가치는 있음.
