### B. 알고리즘 요약

1. 시스템: NAND flash memory ECC(BCH, LDPC) 성능평가를 위한 두 가지 RBER-vs-P/E cycle 채널 모델(모델1: 실측 3x nm 공정 기반, 모델2: 완만한 증가 가정) 사용.
2. 문제: 기존 ECC 설계는 용량(capacity) 우선 고려로 rate 0.9 이상 고rate 부호 위주였으나, BER 증가에 따라 신뢰성(P/E cycles)과 용량의 트레이드오프를 함께 평가할 지표 부재.
3. 핵심 가정: RBER은 P/E cycle 증가에 따라 지수적으로 증가하며, UBER이 `10^-15` 초과 시 flash 수명 종료로 간주.
4. 핵심 기법: Error-free Information Capacity(EIC) 정의 - `EIC = sum_{i=1}^{P/E_cycles} C_i` (i번째 write에서 UBER이 기준 이하일 때의 유효 저장용량 C_i의 누적합).
5. 핵심 식: BCH 코드는 `UBER = (1/k) * sum_{n=t+1}^{N} C(N,n) * RBER^n * (1-RBER)^(N-n)` 로 t-error-correction 한계 기반 acceptable RBER 계산, LDPC는 시뮬레이션으로 계산.
6. 확장: Per-unit EIC(PEIC) = EIC를 P/E cycles로 정규화한 평균 지표도 함께 정의.
7. 구현 디테일: BCH(length 65535/16383)와 LDPC(length 64800) code rate 1/2~9/10 구간에서 EIC 비교, 데이터량은 write당 1GB 가정.
8. 학습/최적화 절차: 없음(수치 계산/시뮬레이션 기반 비교 분석).
9. 결과: 동일 rate에서 LDPC의 EIC/P·E cycle이 BCH 대비 수배~수십배 우수, 채널모델1에서는 rate-2/3, 채널모델2에서는 rate-1/2에서 EIC 최대(고rate=0.9+ 설계 관행과 상반되는 결론).
10. 한계: 특정 디코딩 알고리즘/부호 구성/HW 설계가 전혀 없는 순수 평가 방법론 논문이며, LDPC의 acceptable RBER은 "computer simulation"으로만 언급되고 세부 디코더 파라미터 미공개.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EIC 평가 지표 | 대응 없음 | 디코더/인코더 알고리즘이 아닌 상위 레벨 성능평가 방법론으로 코드 모듈과 직접 대응 없음 |
| RBER-P/E cycle 채널 모델 | `channel.cpp` `Set_R_Offset()` | RBER 기반 채널 열화 모델 개념은 유사하나 본 논문은 P/E cycle 장기추이 통계모델로 실제 채널 주입 로직과는 상이 |
| rate 선택 가이드(저rate가 EIC 유리) | `ecc_top.cpp` `Load_PCM()` | H-matrix rate 선택 시 참고용 정성적 시사점 정도, 직접 이식 요소 아님 |

> 적용 가치: 낮음 | 특정 알고리즘/HW 구현이 없는 순수 평가지표 제안 논문이며, 코드/모듈 이식 대상이 존재하지 않음. rate 트레이드오프에 대한 정성적 참고자료로만 가치.
