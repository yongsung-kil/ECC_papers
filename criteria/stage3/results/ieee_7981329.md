### B. 알고리즘 요약

1. 시스템: SDR용 FPGA LDPC FEC 유닛을 3종 HLS 툴로 개발; 부호는 Wi-Fi(QC-LDPC), DVB-S2(LDPC-IRA), 비바이너리 GF(2^m) 코드셋.
2. 문제: RTL 수작업 설계는 시간·전문성 비용이 큼 → HLS로 생산성↑ 하면서 RTL급 throughput/latency 달성 가능한지 검증.
3. 모델: 표준 message-passing. binary는 `MSA`(CN=min·sign 식(2), VN=식(3),(4)), non-binary는 `FFT-SPA`(Fourier 도메인 CN 식(5)~(7), WHT 사용).
4. 핵심기법1 - Wide-pipeline(Altera OpenCL): work-item을 파이프라인에 수백~수천 개 채워 처리, thread-per-node(TpN) multi-kernel로 CN/VN/edge 커널 분리, 성능지표는 initiation interval `II`.
5. 핵심기법2 - Dataflow(MaxCompiler): `M`개 FU(QC 확장인자 `zf`의 약수)로 부분병렬 스트리밍 디코더, FU별 전용 BRAM bank, dual-mode(CN/VN 동시).
6. 핵심기법3 - Loop-annotated(Vivado HLS): C/C++에 `#pragma`로 loop unroll/pipeline + array partition/reshape 지시, 바깥 루프 pipeline·안쪽 루프 unroll이 최적.
7. 구현 디테일: OpenCL 메모리모델은 반복마다 데이터가 off-chip DRAM을 왕복해야 해 비효율, dataflow/loop는 BRAM에 근접 배치·reshape로 대역폭↑.
8. 최적화: solution I→VI로 loop+memory 지시 조합 시 latency 2 orders 감소(5ms까지), CU/decoder 복제(kdec)로 throughput↑하나 라우팅으로 clock 12~16%↓.
9. 결과: HLS 디코더 throughput Mbps~Gbps, dataflow는 RTL survey 추세를 상회, wide-pipeline은 근접(logic 더 큼), non-binary는 throughput 동등하나 logic 훨씬 큼. 수치는 표(Table 4/5)에만 있어 본문 확정 불가.
10. 한계: 신규 알고리즘/부호설계 없음; 설계 방법론·툴 비교 논문. Prime ECC(C++ bit-exact 시뮬)와 설계 패러다임 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MSA CN/VN 처리 (표준 식(2)~(4)) | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 낮음 - 표준 min-sum 재사용일 뿐 신규 기여 없음 |
| FFT-SPA non-binary 디코딩 | 대응 없음 | non-binary GF(2^m)는 binary QC-LDPC 대상 밖 |
| HLS FPGA 파이프라인/dataflow 아키텍처 | 대응 없음 | Prime ECC는 C++ bit-exact 시뮬, HLS/FPGA 플로우와 무관 |
| BRAM banking / array reshape (메모리) | `decoder.cpp` `LDPC_Decoder()` (HW 모델) | 낮음 - 메모리 병렬화 개념만 유사, FPGA-특화 |

적용 가치: **낮음** — HLS 툴 생산성·설계공간탐색(DSE) 비교 방법론이며 표준 MSA/FFT-SPA 재사용에 그쳐, min-sum 알고리즘·부호설계·HW arch 어느 레이어에도 이식할 신규 기법이 없다.
