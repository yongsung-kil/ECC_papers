### B. 알고리즘 요약

1. 시스템: NAND Flash SSD, MLC(3~4 bit/cell, 8~16 level), rate 0.9 LDPC `n=16384`(2KB), `k=14746`. 리소그래피 공정변이로 die마다 raw BER가 크게 다름.
2. 문제: 기존 ECC는 worst-die BER 기준 설계라 parity 과잉(overkill). die간 BER diversity를 활용해 평균 BER 기준 설계로 낮춤.
3. 채널모델: MLC read를 8-PAM+AWGN으로 모델, die별 BER는 log-normal 분포(variance 6). 여러 die에 걸친 코드워드는 block-fading 채널과 등가.
4. 핵심기법1(splitting): 하나의 LDPC 코드워드를 여러 die에 분산 저장. 저BER die 비트는 고신뢰, 고BER die 비트는 저신뢰로 처리 → 코드워드 평균 BER의 평균·분산 감소.
5. 핵심식 의미: 유한 read 영역 `b_i`별 LLR (Eqn.1)이 die 노이즈 분산(=BER)에 따라 자연스럽게 스케일 → die BER만으로 LLR 스케일링 결정 가능.
6. 핵심기법2(customized code): split을 block-fading 채널로 보고 density evolution으로 최적 degree distribution 설계, 4개 sub-matrix weight 분포 수정([7] 방식)에 log-normal BER 반영.
7. 부수기법(3.1): split 불가한 저지연 시스템은 저BER die에 짧은 read time을 주는 variable read time으로 평균 read latency 감소.
8. Outage: 무선의 outage 개념을 고BER die 선택 확률로 번역, code rate를 outage 확률이 시스템 한계 이하가 되게 선택.
9. 결과: 6 die split시 median log-normal BER 기준 coding gain 12 orders(BER ratio), 2 die split도 5 orders. custom DE 코드는 AWGN 코드 대비 2 orders 추가 이득(FPGA 시뮬, UBER target 1e-15).
10. 한계: HW 미구현(FPGA 시뮬), gate/throughput 없음, die split은 random access QoS 악화, die 선택 상세는 특허 참조.
