### B. 알고리즘 요약 (10줄 내외)

1. Micron 관점의 엔터프라이즈 데이터센터 SSD 설계 개요 논문으로, 특정 알고리즘 제안이 아닌 산업 트렌드·요구사항 정리다.
2. 데이터센터 SSD 요구(용량 128GB~4TB+, UBER 1e-17, MTBF 2M시간, 5년 수명, 3 drive fills/day)와 클라이언트 SSD를 대비한다.
3. IOPS·latency·endurance를 FOB→transition→steady state 성능 곡선(Fig.3)과 QoS(9's, Fig.4) 지표로 설명한다.
4. NAND 고유 특성(tPROG/tREAD/tERASE, cycling 기반 RBER, retention)이 셀·스트링·페이지·블록 설계 선택에서 비롯됨을 서술한다.
5. Program Disturb, program noise, cycling trap-up, 고온/저온 data retention(de-trapping, SBCL) 등 신뢰성 열화 메커니즘을 정성적으로 나열한다.
6. SLC→TLC로 갈수록 page/plane 증가 → 컨트롤러·펌웨어의 garbage collection·valid data 이동 부담 증가를 지적한다.
7. 성능 기법으로 multi-plane overlapping read/write로 tPROG를 "숨기는" 병렬 동작을 언급한다.
8. SSD 스택(FTL, flash file system, error control, ONFI 드라이버)을 Fig.8로 제시한다.
9. ECC는 16~20nm MLC/TLC에서 "더 강한 BCH와 LDPC"가 필요하다고 IV절 마지막에 단 한 문장으로만 언급한다.
10. 한계: LDPC 부호설계·디코더·HW·수치 결과가 전혀 없는 시스템 개요 논문이라 ECC 이식 정보가 없다.
