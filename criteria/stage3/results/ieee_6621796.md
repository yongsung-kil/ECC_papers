### B. 알고리즘 요약

1. 시스템: 변환영역 Wyner-Ziv(TDWZ) 분산비디오코딩, WZ 화면을 4x4 정수 DCT→16 sub-band 양자화→bit-plane, rate-adaptive LDPCA(accumulate)로 syndrome 생성, 피드백채널로 partial syndrome 요청.
2. 문제: DVC 채널부호(LDPCA)의 error-floor가 RD 성능을 열화시킴.
3. 모델: source `X`와 side-info `Y`가 i.i.d. binary noise `Z`로 상관, 조건부엔트로피 `H(X|Y)`가 target rate이며 crossover prob `p`로 산출.
4. 기법1 large-cycle LDPCA: syndrome merging + PEG로 bipartite graph의 girth 최대화(짧은 cycle이 error-floor 유발하므로).
5. 기법2 source revealing: encoder가 선택한 k개 source bit을 함께 전송→해당 VN이 degree-1로 그래프에서 제거되어 해당 syndrome을 완전복원, 저상관 영역에서 이득.
6. 핵심식: `H(X|Y) = -p·log2(p) - (1-p)·log2(p)` — 채널신뢰도(=required Rmin) 측정 근거.
7. 적응형 스킴: Rmin>RThd(=0.85)면 partial syndrome+source bit 요청, feedback index가 Lcutoff(=46) 넘으면 source member 전송; source 위치는 사전공유 random permutation으로 위치정보 전송 불필요.
8. 구현: source bit 수신 시 LLR을 ±∞(시뮬은 절대값 200 clamp)로 갱신((2)), BP max iter 100 + hard-decision 조기종료.
9. 결과: length 1584에서 large-cycle LDPCA가 degree-3 regular 대비 개선(H(X|Y)=0.8 부근 제외); Soccer 기준 DISCOVER 대비 ΔPSNR 0.24dB, Δrate -4.40%, source revealing 자체 절감 0.23~0.26%.
10. 한계: 시뮬만(HW 미설계), DVC/피드백채널 전제, LDPCA·source revealing이 NAND ECC 채널과 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| large-cycle(PEG girth 최대화) 부호구성 | ecc_top.cpp Load_PCM() | error-floor 대책 개념 참고 가능하나 LDPCA 구조라 QC-LDPC에 직접 이식 불가 |
| LLR 계산/갱신(source revealing 시 ±∞) | channel.cpp Set_LLR_Th() | LDPCA/DVC 전용 LLR 갱신, NAND soft-read와 대응 약함 |
| BP 복호 + hard-decision 조기종료 | decoder.cpp LDPC_Decoder(), partial_CRC.cpp | 조기종료는 Prime ECC가 이미 보유(중복 기여) |
| source revealing / syndrome accumulate | 대응 없음 | 분산소스코딩 특유 구조, NAND ECC에 대응 모듈 없음 |
```

적용 가치: **낮음** — Wyner-Ziv 분산소스코딩용 LDPCA(누적 syndrome)+source revealing 응용이라 NAND binary QC-LDPC ECC와 부호구조·채널 모델이 근본적으로 다르다. large-cycle(PEG) error-floor 대책만 개념적 참고 대상.
