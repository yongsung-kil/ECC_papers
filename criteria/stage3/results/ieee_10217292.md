### B. 알고리즘 요약

1. 시스템: Joint Source-Channel Coding(JSCC) 구조에서 source SC-LDPC와 channel SC-LDPC를 concatenate, BPSK/AWGN 채널에서 sliding window decoding으로 검증.
2. 풀려는 문제: 기존 SC-LDPC base matrix 확장은 성능을 위해 random 구성을 쓰는데, HW 구현 시 매트릭스 저장·검색이 비효율적이고 클록 동기화가 복잡.
3. 핵심 가정/모델: source 압축비=channel code rate 관계, coupling length 제약 `Lsc + 2ms = Lcc`, source check node ↔ channel information node 1:1 대응.
4. 핵심 기법: 저장 친화적 base matrix를 선형 복잡도로 구성 후 각 `1`을 `M×M` shift-identity(오프셋 K)로, `0`을 zero matrix로 QC 확장.
5. 의미: QC(shift-identity) 구조는 random 대비 매트릭스 정보 저장량을 크게 줄여 HW clock 동기화·데이터 저장/검색을 용이하게 함.
6. 확장: base matrix를 `ms+1` submatrix로 주기적 coupling, 우측 identity 예비코딩 구조는 고정, 나머지는 최소 ring length·column weight 합 제약 하 random 배정.
7. 구현 디테일: 확장인자 `M=100`, offset K는 홀수 또는 짝수로 sparse 성능 보장, short cycle 제거로 decoding 성능 확보.
8. 복호: sliding window decoding(SWD, BP+계층 복호 결합), source check node↔channel variable node 간 LLR 교환, 최대 iteration/error exit 조건.
9. 결과: 제안 QC 구조가 기존(원) 구조 대비 복호 성능 우수, random 구조 대비는 열세이나 차이가 수용 범위이며 window size↑ 시 BER이 빠르게 0에 수렴.
10. 한계: HW 미설계 시뮬만, JSCC(source+channel 결합) 특화 SC-LDPC라 NAND 단일 binary QC-LDPC와 성격 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC(shift-identity) base matrix 확장 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | Prime ECC도 base+circulant shift QC 구조지만, 논문은 SC-LDPC 시간확장/coupling 구조라 직접 이식 불가 |
| sliding window BP 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | Prime ECC는 block QC-LDPC min-sum, SWD(convolutional)와 스케줄 상이 |
| tanh 기반 BP CN/VN 갱신 | 대응 없음 | Prime ECC는 min-sum 근사(full-tanh sum-product 미사용) |
| source-channel concatenation / JSCC | 대응 없음 | Prime ECC는 순수 channel ECC, source coupling 개념 없음 |

적용 가치: **낮음** — JSCC용 concatenated SC-LDPC의 저장 친화 QC 구성법으로, NAND 단일 block binary QC-LDPC(고정 H-matrix, min-sum) 파이프라인과 부호구조·복호 스케줄이 달라 실질 이식 접점이 약함.
