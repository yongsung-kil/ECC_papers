### B. 알고리즘 요약

1. 대상은 5G NR QC-LDPC로 최대 code length·최고 rate 선정(info `8448 bits`, rate `0.92`, expansion factor `384`), LLR 8-bit 고정소수 구현.
2. 풀려는 문제: MS 알고리즘이 SP의 `tanh` 근사 시 다른 항을 1로 근사해 생기는 오차를 저복잡도로 보정하는 기존 방안 부재.
3. 채널: 사전 LLR `Fn = log2(P(xn=0|yk)/P(xn=1|yk))`로 초기화(로그영역 처리).
4. 핵심 기법(LCC-MS): CN 근사식 `Lmn ≈ 2·tanh^-1(∏tanh(Zmn'/2))`에 보정계수 α를 곱해 `Lmn ≈ α·(-1)^(σm⊕σmn)·min|Zmn'|`로 오차 보정.
5. 핵심 트릭: LLR에 양의 상수를 곱해도 hard decision 불변 → `Lmn/α^p`, `Zmn/α^p` 형태로 iteration마다 α 누적, 이로써 보정계수 곱셈이 매 iter O(N0·M1)→O(N0)로 감소(α^p는 사전계산 저장).
6. 확장 최적화 1: 보수집합(complement set) 최소값 대신 완전집합의 min/2nd-min을 구해 사용 → 비교연산 O(M0·N1^2)→O(M0·N1)(수식 2M0N1).
7. 확장 최적화 2: VN에서 `Zmn = Zn − Lmn`로 계산 → 누적연산 O(N0·M1^2)→O(N0·M1)(2N0M1).
8. HW 구현: 자체 DSP UCP에 전용 명령어 CompMR(min/2nd-min), CompAbsSel(부호항 부착), CompSel(hard decision) 추가, CN을 pre-computation+parallel processing 2단으로 SIMD 병렬화.
9. 결과(TSMC 16nm 테이프아웃, 1.2GHz): BER 1e-5에서 MS 대비 0.16dB 이득, fixed-point는 float 대비 0.04dB 손실; 단일 code block 3.34μs, 0.5ms 슬롯당 149 블록, throughput 2.53Gbps.
10. 한계: 5G 특화·자체 DSP 명령어 의존, 이득 소폭(0.16dB), error-floor/iteration 수치 미제시, gate count 미상.
