### B. 알고리즘 요약

1. 대상: type-I protograph QC-LDPC(`b_ij∈{0,1}`), base matrix `B` (Mb×Nb), circulant 크기 L, layered scheduling 복호. 예: (3,6) L=128 N=768 R=1/2, WiMAX N=2304 Mb=12.
2. 문제: 반복복호 지연을 줄이려면 조기종료 필요하나 full-syndrome은 HW비용·지연 증가, 저비용 OTF는 unsafe(undetected error).
3. 모델: AWGN + BPSK, offset Min-Sum, layered, 최대 50 iteration.
4. 핵심기법: 인접 2 layer 결합 submatrix `H_{i,i+1}=[H_i;H_{i+1}]`에 대한 **in-between layers partial syndrome** `s_{i,i+1}=H_{i,i+1}·x̂_i`를 layer 사이 hard-decision으로 계산.
5. 의미: 같은 x̂에 대해 두 layer PC를 동시에 검증 → OTF(단일 layer)가 놓치는 layer간 x̂ 진동을 포착해 안전성↑.
6. criterion 계열: `IBL-PS(θ)` = 연속 θ개 IBL-PS가 모두 0이면 종료. θ=1은 최저지연, θ↑ 이면 safer.
7. 구현: on-the-fly (AP-LLR memory write-back 시 H_i, read 시 H_{i+1} PC 평가), XOR gate만 필요 → FPGA overhead OTF와 유사, 전체 아키의 2% 미만.
8. 약점 분석: IBL-PS(1) undetected error는 H_{i,i+1}의 cycle/trapping set에서 발생. 2 full layer girth ≤ 12 → 부호를 girth-12·저multiplicity로 설계하면 안전.
9. 결과: IBL-PS(1)은 FER≈10⁻⁴까지 full-syndrome 무손실. IBL-PS(2)는 SS(3)만큼 안전. 동일 지연 조건에서 IBL-PS(5)만 full-syndrome 안전성 근접. IBL-PS(2)가 SS(3) 대비 avg iteration 최대 30%, OTF(9) 대비 108% 감소.
10. 한계: layered 아키 전제, all-zero block 있는 irregular 부호(WiMAX)는 덜 안전, gate count·throughput 절대수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 조기종료 criterion (partial syndrome) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 이미 partial(T4)/full CRC 조기종료 보유 → 대안 종료 신호 개념 |
| syndrome/PC 평가 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 종료 판정 삽입 지점, Prime ECC는 dual-update 스케줄이라 layer 개념 매핑 필요 |
| avg iteration/latency 감소 | `decoder.cpp` `LDPC_Decoder()` | 반복수 감소로 throughput/전력 이득 목표 동일 |
| layered CNU 병렬 처리 | 대응 없음 | Prime ECC는 layered가 아닌 z=32 row 병렬 + HCU/GT 구조 |

적용 가치: **중간** — 조기종료 안전성-지연 trade-off는 Prime ECC 관심 영역이나, 이미 partial/full CRC 종료를 보유하고 layered 아키가 아니어서 IBL-PS의 layer간 개념을 dual-update 스케줄로 매핑해야 실이득이 나온다.
