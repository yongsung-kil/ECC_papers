### B. 알고리즘 요약

1. 대상은 3p check node, pL variable node를 갖는 **(3,L)-regular QC-LDPC**(3=column weight, L=row weight), CPM(circulant permutation matrix) 기반, exponent(shift) 행렬 `P`로 정의.
2. 문제: 유한체/protograph/array/컴퓨터탐색으로 만든 수많은 (3,L) QC-LDPC 중 주어진 길이·rate에서 최적 부호를 어떻게 고르나.
3. 접근: Tanner graph **isomorphism** 관점 — 동형 부호는 반복복호 성능이 동일하므로 비동형 부호만 탐색(Theorem 1: 행/열 permutation, modulo-p 가감/승산으로 동형 판정, `P`의 첫 행·열을 0으로 정규화).
4. 사이클 존재조건: 2i-cycle은 `sum(P_{lj,kj} - P_{lj+1,kj}) = 0 (mod p)`일 때 존재(식 5).
5. 핵심 알고리즘 1: exponent 행렬 `P`에서 식(5)를 만족하는 cycle type을 열거하고 Definition 1의 등가관계로 비등가 type 수 `Num2i`를 구해 사이클분포 산출.
6. 비등가 type과 실제 사이클 수 대응: (4i-2)-cycle은 type당 p개, 4i-cycle은 p, p/2, p/4... 대응(Table I).
7. 핵심 알고리즘 2: 4-cycle 없으려면 `P` 둘째(또는 셋째) 행 원소가 서로 달라야 함 → 후보 행 집합 `S`에서 P1,P2 조합, girth < g0인 부호를 pruning하는 **단순화 exhaustive 탐색**(공간 p^{2(L-1)} → 대폭 축소).
8. 최적화: 추정 최적 girth `g0` 이하 부호를 제외, 짧은 사이클(≤12) 총수가 적은 부호를 선택(girth값 하나가 아니라 총 사이클 분포 기준).
9. 결과(Example 2, (258,131) L=6 p=43): 비동형 5개 부호 중 짧은 사이클 최소인 C2가 WER 10^-7까지 error floor 없음, 동일길이 (256,128) CCSDS 대비 BER 10^-6에서 0.3dB 우수(SPA 50 iter, BPSK/AWGN).
10. 한계: column weight 3 고정·짧은 저rate 부호 예제, 디코더/HW 미구현, gate count·throughput 없음, soft-read 무관(부호구성 이론).
