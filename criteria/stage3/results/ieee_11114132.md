### B. 알고리즘 요약

1. MDPC(127,106) irregular cyclic code(최소 해밍거리 6, 2-error correcting), BPSK 변조 후 AWGN 채널 전송을 가정.
2. 문제의식: BP는 성능은 좋으나 복잡도가 크고, 기존 GDBF(gradient descent bit-flipping)는 HW 구현이 단순하지만 trapping set에 의한 진동으로 성능이 낮음.
3. 채널/디코딩 모델: 에너지 함수 `E_j^(ℓ) = a_j^(ℓ) y_j x̂_j^(ℓ-1) + b_j^(ℓ) Σ_{i∈P(v_j)} s_i^(ℓ) + m_{l_j}`로 각 비트의 flip 여부를 결정(threshold `E_th^(ℓ) = min E_j + δ`).
4. 핵심 기법 1단(AD-GDBFwM): 여러 GDBFwM 디코더를 cascade로 연결, `t`번째 디코더 입력 `z_t = (1-r_t)x_{t-1} + r_t y`(restart flag `r_t`)로 이전 디코더 결과와 원 수신값을 혼합해 trapping set 탈출 유도.
5. 식의 의미: 기존 GDBF(`a_j=b_j=1, m=0`) 대비 momentum항 `m_{l_j}`(최근 flip된 비트 재flip 억제)과 irregular 코드용 가변 가중치 `a_j, b_j`(variable node별 독립)를 도입해 저-weight 비트의 정정 난이도를 보정.
6. 핵심 기법 2단(파라미터 학습): 유전 알고리즘(GA)으로 `a_j, b_j, m` 벡터를 오프라인 학습 — 초기 GDBF로 error log(uncorrectable pattern) 수집 → cascade 단계별로 순차 최적화, fitness=정정된 코드워드 비율.
7. 구현 디테일: 파라미터는 비트열로 양자화해 GA 염색체(chromosome)로 표현, single-point crossover + mutation(p_m=0.05) + roulette wheel selection + elitism, 모멘텀 벡터는 조합 행렬 `M`(크기 `N_m = C(m_max+L, L)`)에서 선택.
8. 학습/최적화 절차: `Dec_0`(기본 GDBF)로 고 Eb/N0에서 Monte Carlo 시뮬레이션으로 error log 생성 → `Dec_1`부터 순차적으로 GA 학습, cascade별 반복(각 `L_t=15`, `M_D=4`단).
9. 결과: AD-GDBFwM이 BP 대비 FER 기준 1.5dB, GDBFwM 대비 0.5dB 이득, NBP20 대비 iteration>3에서 우수; 단 miscorrection 36건/5×10^8 코드워드(주로 Dec_1) 관찰.
10. 한계: 단일 short irregular MDPC(127,106) 코드로만 검증, HW 구현·게이트/throughput 분석 없음(순수 알고리즘/시뮬), 코드rate 확장은 future work로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBFwM/AD-GDBFwM bit-flipping 반복 디코딩 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC 주력은 min-sum message-passing이라 bit-flipping 계열 알고리즘 자체는 구조가 달라 별도 디코더 모드 추가에 준하는 이식(중간 난이도) |
| Cascade 구조(restart flag로 이전 결과·원 신호 혼합) | 대응 없음 | Prime ECC엔 cascade/restart 개념이 없어 신규 도입 필요 |
| Momentum 항(최근 flip 비트 재flip 억제) | `decoder.cpp` `VN_Cal_HD()` 계열 | trapping-set 회피 아이디어로서 VNU 갱신 로직에 상태(최근 flip 이력) 추가 형태로 참고 가능 |
| GA 기반 파라미터(a_j,b_j,m) 오프라인 학습 | `local_opt.cpp` | Prime ECC의 LLR 자동최적화 모듈과 목적(파라미터 튜닝)은 유사하나 GA 알고리즘 자체는 대응 없음 |
| soft LLR quantization/threshold | `channel.cpp` `Set_LLR_Th()` | 에너지 함수 y_j(수신 soft값) 활용은 LLR 임계값 설계와 유사한 맥락이나 직접 대응 아님 |

적용 가치: 중간 — bit-flipping 계열 디코더는 Prime ECC의 min-sum 구조와 이질적이라 전면 이식은 어렵지만, cascade+momentum을 통한 trapping-set 회피 아이디어와 GA 기반 파라미터 자동 튜닝 컨셉은 Prime ECC의 LLR 최적화(local_opt.cpp)에 참고할 여지가 있음. 단, 단일 short 코드 시뮬 검증뿐이라 실효성은 추가 확인 필요.
