### B. 알고리즘 요약

1. 시스템: AWGN + BPSK, 부호는 BCH(15,11)/BCH(15,7) 초단 linear block code, BP iteration `NIter=5` 고정.
2. 문제: 짧은 부호는 PC matrix가 sparse하지 않고 short cycle이 많아 표준 BP 성능 저하, 게다가 수신기가 부호를 몰라도 복호(blind)하려 함.
3. 핵심 기법: NBP(Nachmani)를 gated RNN cell로 재구성 - `Σ`(V2C), `Π`(C2V), `OUT`(a posteriori LLR) 블록을 미분가능 그래프로 구현.
4. Gating weights `wG`(sigmoid `σ(wG)`)로 factor graph 위상(topology)을 이진 선택 학습, NBP weights `wΣ`,`wOUT`(1 근처 실수값)로 메시지 가중.
5. Eq.(1) V2C, Eq.(2) C2V(`2·arctanh(∏ tanh(µ/2))`), Eq.(3) posterior LLR을 각각 weighted 버전으로 학습 - short cycle의 악영향 완화가 목적.
6. 효율화: dense layer(O(Nvar²·Ncheck²)) 대신 개선 computational graph로 곱셈수를 O(Nvar·Ncheck)로 축소(Table I~III).
7. 구현 디테일: arctanh gradient explosion 회피 위해 Taylor 근사, refined gate function으로 saturation 영역 gradient 개선, LLR을 [-1,1] 정규화.
8. 학습: TensorFlow auto-encoder 프레임, binary cross-entropy loss, RMSProp + cyclic LR(1e-2~1e-1), 250 epoch, SNR 4dB 고정, many-to-many 출력 가중합.
9. 결과: BCH(15,11)/(15,7) 모두 표준 BP 이상 성능, BCH(15,11)에서는 NBP 효과로 BP 상회(FER, Fig.10). 총 trainable param 610/310개.
10. 한계: 초단 BCH·시뮬(FER)뿐, HW 미설계, 큰 부호로의 확장·학습 반복성(가끔 실패)은 미해결.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| gated RNN NBP 디코더 전체 | 대응 없음 (NN/딥러닝 디코더) | Prime ECC는 bit-exact min-sum HW, NN 학습형 디코더와 무관 |
| C2V `tanh`/`arctanh` SP 업데이트 | 대응 없음 (full-tanh Sum-Product) | Prime ECC는 min-sum 근사 사용, full-SP는 미채택 |
| iteration별 message 가중 | [decoder.cpp](../../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | 개념적 유사(iteration별 파라미터)뿐, 학습형이라 직접 이식 불가 |
| LLR 정규화/전처리 | [channel.cpp](../../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | soft LLR 처리 접점은 있으나 NN 학습 정규화라 직접성 낮음 |

적용 가치: **낮음** - NN이 BP를 전면 대체하는 short BCH 전용 학습형 디코더로, binary QC-LDPC min-sum HW인 Prime ECC에 이식할 수 있는 요소가 없다.
