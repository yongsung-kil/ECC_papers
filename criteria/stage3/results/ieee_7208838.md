### B. 알고리즘 요약

1. Regular binary LDPC `(N,K)(dv,dc)` 코드, BPSK+AWGN 채널. 저rate/저차수 Code 1 (816,272)(4,6) rate-0.333과 고rate/고차수 Code 2 (1023,781)(32,32) rate-0.763 두 코드로 검증.
2. 기존 BF(bit-flipping) 계열 디코더는 checksum weight `wmn`을 초기 채널값으로 고정해 이후 이터레이션의 VN/CN 신뢰도 변화를 반영하지 못해 성능이 정체되는 문제.
3. flipping function(FF) `En`은 VN의 tentative decision 신뢰도 지표이며, checksum(패리티 잔차) `sm`의 함수로 정의됨(`En = -α1·φ(uˆn,yn) - Σ wmn(1-2sm)`).
4. 핵심 기법: Dynamic Weighted BF(DWBF) — checksum weight `rmn(l)`을 매 이터레이션 이전 FF 값들의 클리핑 함수로 재계산 (`rmn(l) = min_{n'∈N(m)\n} Γ(-En'(l))`, `Γ(x)=x-η (x≥η), 0 (x<η)`).
5. 새 FF는 `En(l) = -yn(1-2uˆn) - α2·Σ rmn(l-1)(1-2sm)`로, forgetting factor `α2`(0<α2<1)로 과거 신뢰도 반영 비중 조절. η=0이 실험적으로 최적.
6. 확장: 다중 비트 선택(FBS) 규칙 M1(임계값+최대값 fallback), M2(flipping intensity `F̃n` 기반, UCN/PCN 신호 결합)로 수렴 속도 개선.
7. 계산량 절감을 위해 selective weight-updating schedule SWUS-A/B 제안 — 전체 CN이 아닌 flip된 비트 인접 CN만 갱신(FWUS 대비 visited CN 수 대폭 감소).
8. 파라미터(α1, α2, α3, FI 임계값 θ0~θ2)는 수치 시뮬레이션으로 오프라인 최적화(학습 알고리즘 없음).
9. 결과: BER=10⁻⁵에서 S-DWBF-B/F가 RRWBF 대비 2.5~2.6dB 이득; M2-DWBF는 NMS(min-sum) 대비 0.1dB 이내로 근접하며 고SNR에서 NMS보다 낮은 연산복잡도.
10. 한계: 순수 시뮬레이션(이론/HW 검증 없음), 파라미터(α2, η, FI 임계값)가 코드별 오프라인 튜닝 필요, hard-decision 순수 버전은 별도 언급 없이 soft yn 값에 의존하는 경우가 대부분.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BF 디코딩 알고리즘 전체(FF/FBS/weight-update) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 BF 계열 자체는 대응 없음. 알고리즘 대체 검토 시 참조 위치만 동일 |
| checksum weight 동적 갱신(DWBF) | 대응 없음 | Prime ECC는 iteration별 LLR 임계값 테이블(`PARAM_LLR`) 방식으로 이미 유사한 동적 조정 개념 보유, 직접 대응 모듈 없음 |
| iteration별 파라미터 최적화 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 개념적으로 유사(이터레이션별 파라미터 테이블)하나 BF의 checksum weight와는 다른 메커니즘 |
| 선택적 갱신 스케줄(SWUS-A/B, 복잡도 절감) | `GT.cpp` `Make_GT_HW()` | Graph Thinning과 유사하게 "일부만 갱신"하는 발상이나 대상 알고리즘이 달라 직접 이식 어려움 |

> 적용 가치: 낮음 — Prime ECC는 min-sum 기반 soft-decision 디코더 계열이며 본 논문은 hard-decision 기반 BF 계열 디코더 전체를 대체 제안하는 것이라 알고리즘 구조 자체가 상이함.
