> 총평: fully-parallel LDPC 디코더의 라우팅 병목을 배선분할+pipeline register로 완화하고 짝수 register로 2 codeword 동시복호까지 하는 순수 HW arch 기법으로, LDPC HW 라우팅 병목 관점에서 참고가치는 있으나 완전병렬 전제·정정능력 무개선이 한계.

### B. 알고리즘 요약

1. 코드/시스템: length-1024 (3,6)-regular LDPC, rate-1/2, min-sum 복호, 4-bit LLR(sign 1 + magnitude 3), BPSK/AWGN, 90nm CMOS, fully-parallel(모든 노드 물리 인스턴스화).
2. 문제: fully-parallel 디코더는 노드 간 배선이 복잡해 가장 긴 wire의 지연이 clock을 지배 → throughput 저하; bit-serial은 라우팅은 줄지만 message당 cycle 증가.
3. 모델: 긴 wire를 register로 나눠 message를 여러 clock에 걸쳐 전송, 노드는 도착 대기 대신 "가장 최근 available(다소 outdated) message"로 연산(flooding-type update schedule).
4. 핵심기법1(wire partitioning): 최장 wire를 2개로 분할, 노드출력 `x_A^t`를 이전 iteration의 register 저장값(outdated) 포함해 계산 → critical path 지연 감소, BER 큰 손실 없음.
5. 핵심의미: outdated message 사용이 min-sum 수렴에 치명적이지 않다는 관찰([22])을 HW 지연단축에 활용.
6. 핵심기법2(2-codeword 동시복호): 긴 wire에 짝수(2)개 register 삽입 시 두 codeword message가 섞이지 않아 동시복호 가능, 면적 소폭 증가로 throughput 배증.
7. 구현 디테일: register 삽입을 pre-layout 추정(predicted flow) 대신 register 없이 initial place-and-route한 실제 배선정보로 결정(initial layout flow) → register 수 89~95% 감소, 4-bit라 wire당 4개 register.
8. 학습/최적화: 없음(CAD place-and-route 기반 wire length 히스토그램으로 threshold=최대길이 절반 이상 wire만 분할).
9. 결과: Proposed2가 uncoded 13.21Gb/s(Conventional2 대비 +28%), 0.098nJ/bit(-30% energy), core area +1.6%; 동일 코딩성능에 필요한 clock cycle 33→31로 감소(16 iteration 등가).
10. 한계: 완전병렬 전용 arch, 정정능력/error-floor 무개선, gatecount 본문 미기재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| wire partitioning / pipeline register 삽입 | 디코더 HW 모델 (pipeline 6-stage col_M1/Z/P1~P4) | Prime ECC는 이미 6-stage pipeline 보유, 배선분할 개념은 부분 중복이나 라우팅 최적화 아이디어 참고 |
| flooding-type outdated-message schedule | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 Dual-Update(even/odd 교대) 스케줄; outdated-message 허용 스케줄은 별개 접근 |
| min-sum CN(min+sign XOR), 4-bit LLR | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()`, `PARAM_LLR` | min-sum·양자화는 이미 보유(중복), 4-bit는 Prime ECC 3SD 상회 |
| 2-codeword 동시복호(짝수 register) | 대응 없음 | Prime ECC에 다중 codeword 동시복호 스케줄 없음 |

적용 가치: **중간** — LDPC 디코더 라우팅/배선 병목이라는 HW 실병목을 겨냥한 arch 기법이라 Prime ECC의 메모리/라우팅 개선 참고가 되나, fully-parallel 전제·min-sum 중복·정정능력 무개선으로 z=32 partial-parallel 구조에 직접 이식은 제한적.
