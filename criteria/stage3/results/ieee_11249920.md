### B. 알고리즘 요약

1. 대상은 quantum surface code, BP(belief propagation)를 디코딩에 사용하나 quantum LDPC 특유의 topological product 구조로 인해 short cycle이 많아 non-convergence 문제 발생.
2. 풀려는 문제: 기존 BP+post-processing(OSD 등) 방식은 복잡도 증가와 heuristic 의존이라는 단점이 있음. 후처리 없이 BP 자체의 수렴성을 개선하고자 함.
3. 핵심 가정: quantum code degeneracy(오류 정정 조합이 stabilizer를 이루면 동등 취급) 덕분에 모든 큐비트를 디코딩 그래프에 유지할 필요가 없다는 성질을 활용.
4. 핵심 기법: `s-sparsification` — 연속된 `s`개 edge 라인을 제거하고 `(s+1)`번째 라인만 유지(수평/수직/대각 방향), 디코딩 그래프를 구조적으로 희석.
5. 핵심 식: correctability loss `δ = (t(G)-t(Gs))/t(G)`이며 Theorem 1에서 `δ_D,δ_V ≤ (s+1)/(s+3)`, `δ_H ≤ (s-1)/s` 로 상한이 코드 크기 `N`과 무관함을 증명 — 즉 희석해도 최악의 정정능력 손실이 코드 크기에 상관없이 bounded.
6. 핵심 기법 2단(확장): Graph Dilution — sparsification ratio `{s0,...,sT}`를 점진적으로 증가시키며 단계별로 BP 수행 후 diluted 큐비트를 decimation(soft 메시지 기반 0/1 고정).
7. 부수 트릭: 각 단계에서 짧은 사이클을 줄여 그래프가 "동적으로 locally tree-like"해지도록 만들어 BP 가정(tree-like graph에서 유효)에 근접시킴.
8. 결과: X-noise 하에서 후처리 없는 표준 BP가 threshold 9.0% 달성, BP-OSD의 9.1%와 근접. worst-case 복잡도 `O(N^{3/2})`, 평균 복잡도는 거의 linear.
9. 한계: 순수 시뮬레이션(코드 capacity noise model)뿐 HW 미설계, quantum degeneracy라는 quantum 특유 속성에 전적으로 의존하는 기법이라 classical binary LDPC(non-degenerate)에는 이론적 정당화가 성립하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 디코딩 그래프 sparsification (short cycle 제거) | `decoder.cpp` `LDPC_Decoder()` | classical BP도 short cycle이 성능 저해 요인이라는 점은 공통이나, 본 기법은 quantum degeneracy(오류 조합의 stabilizer 동치성)를 전제로 큐비트를 그래프에서 제거하는 것이라 classical binary LDPC에는 근거가 없음(정보 손실 없이 변수노드 제거 불가) |
| Graph Dilution(단계적 sparsification+decimation) | 대응 없음 | quantum 특유의 degeneracy 기반 decimation으로 `decoder.cpp`의 min-sum/HCU 등과 대응되는 classical 기법 없음 |
| Girth 증가를 통한 수렴성 개선 | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 단계에서 short cycle 억제는 QC-LDPC 부호설계의 일반적 목표와 개념적으로 일치하나, 본 논문은 디코딩 시점의 동적 그래프 변형이지 부호설계 기법이 아님 |

적용 가치: **낮음** — quantum surface code degeneracy에 근본적으로 의존하는 BP 그래프 희석 기법으로, classical binary QC-LDPC ECC의 부호설계·디코더·HW 어느 레이어에도 직접 이식할 근거가 없음.
