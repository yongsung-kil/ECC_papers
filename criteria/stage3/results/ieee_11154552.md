### B. 알고리즘 요약

1. QC SC-LDPC(quasi-cyclic spatially-coupled LDPC) 코드의 girth를 크게, lifting size(constraint length)는 작게 유지하는 구성 알고리즘.
2. 기존 SC-LDPC high-girth 구성법은 coupling width가 크거나(예: girth-6 protograph 방식), 시간가변(time-variant)이 되어 구성 복잡도가 spatial position 수에 지수적으로 증가하는 문제.
3. time-invariant SC-LDPC의 주기적 구조에서 유도한 `cycle relevant matrix(CRM)` `H^[w,ℓ]_S`(coupling width w, 길이 ℓ 사이클을 모두 포함하는 부분행렬)만으로 cycle 탐지를 축소.
4. block LDPC용 HQC(hierarchical QC) 구성법[Wang et al.]을 확장: 다항식행렬 `H(x)`를 2단 lifting `H(x,y)`(x-lifting `Sx`, y-lifting `Sy`)로 표현, 사이클 조건은 `Σ(-1)^i s_i ≡ 0 mod S`.
5. Alg.2(Block Construction and Spread): 블록 프로토그래프로 despread 후 HQC 구성, edge spreading하면 girth가 유지된다는 성질(spreading은 cycle을 늘리지 않음)을 이용.
6. Alg.3(CRM 기반 modified HQC): CRM `H^[w,g-2]_S(x,y)`에서 지수 중복도(multiplicity κ)를 고려해 `κ(s0-s') ≡ Σx mod Sx`로 문제값을 계산, 첫 replication 블록열만 탐색해 복잡도 축소.
7. cost function table(그리디 최적화): 각 미지수 지수 후보값마다 발생하는 짧은 사이클 수(N4,N6,...)에 가중치(`wi=5^k`, w4=125,w6=25,w8=5,w10=1)를 곱한 비용을 계산, 비용 감소가 가장 큰 지수부터 갱신(Alg.1).
8. 최적화 절차는 학습이 아닌 결정론적 그리디 탐색(반복적으로 모든 cost table entry가 non-negative가 될 때까지 갱신).
9. 결과: girth-10, rate 0.8(3,15)/(4,20)-regular 구성에서 Alg.3이 Alg.2 대비 lifting factor 약 절반; Smin=461 사례(batch size 36880, w=4)를 실용적이라 제시. girth 10 코드가 girth 6 대비 BER 평균 성능 대폭 우수(Fig.4).
10. 한계: HW 미설계, 순수 시뮬레이션(BER은 예시 하나만), 부호 구성이 확률적("try and check")이라 임의 protograph·girth·lifting에 대해 성공 보장 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix/protograph 구성(HQC, girth 최적화) | `ecc_top.cpp` `Load_PCM()` | 부호설계 레이어에 닿지만 SC-LDPC 특화 구성법으로 Prime ECC 고정 QC-LDPC 구조 재검증 부담 큼 |
| 2단 lifting (x-lifting/y-lifting, hierarchical QC 구조) | 대응 없음 | Prime ECC의 base+circulant shift(`PCM_b`)는 단일단 lifting, HQC 2단 구조와 직접 대응 없음 |
| SC-LDPC edge spreading/convolutional 구조 | 대응 없음 | Prime ECC는 block QC-LDPC 전용, SC 구조 자체를 지원하지 않음 |
| girth 증가에 따른 error-floor 개선 결과 | 대응 없음 | 부호설계 재검증 필요, 디코더 알고리즘 변경 아님 |

> 적용 가치: 낮음 — SC-LDPC 전용 부호 구성 이론이며 Prime ECC의 고정 QC-LDPC(block) 구조에 부분 이식하려면 부호 자체를 재설계해야 해 이식 난이도가 매우 큼.
