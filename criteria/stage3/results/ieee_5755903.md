### B. 알고리즘 요약

1. 대상은 DVB-S2 표준에 정의된 21종 QC-LDPC(IRA, Irregular Repeat-Accumulate) 코드로, 코드 길이 `N=16200 또는 64800`, 코드율 `1/4~9/10`이다.
2. 풀려는 문제는 LDPC는 인코딩 스킴이 원래 정의되어 있지 않고, generator matrix `G`는 sparse하지 않아 인코딩 복잡도가 크다는 점 - 21종 코드를 하나의 HW로 지원하는 programmability도 필요.
3. IRA 코드 parity part는 staircase 구조(accumulator에 대응), information part는 매 360열마다 새 컬럼 패턴이 나타나고 나머지는 순환 시프트(quasi-cyclic) 구조를 가진다.
4. 핵심 기법: shift register 대신 **single-port RAM**을 사용하는 quasi-cyclic 인코더 구조 - 정보부 `H_I`의 파티 생성 행렬 `G_P^I = H_I^T`를 barrel shifter + m개 EXOR + RAM으로 계산.
5. 인코딩은 `m=360` 비트 단위로 `p=m/s` 병렬 입력을 받아 시프트 후 RAM에 mod-2 누적(read-modify-write, 2클록/워드) - 정보벡터 전체(`m×k_0` 비트) 처리 후 parity 완성.
6. 확장 기법: 순서 문제(permuted matrix 순서 vs. 원래 parity check 순서 불일치) 해결을 위해 RAM을 `b`개로 분할하고 데이터를 interleave 저장하는 partitioned RAM 기법(diagonal read).
7. 부수 트릭: `p`비트 parallel accumulator(8비트 예시, EXOR 8개+레지스터), 최대 column weight `C_max ≤ s/2` 조건 하에서 동작.
8. 별도 학습/최적화 절차는 없음(구조적 설계).
9. 결과: FPGA(Altera Stratix EP1S80) 합성 결과 11840 LUTs, 11 RAM 블록, 50MHz 클록으로 최대 400Mbps 처리, 기존 shift-register 방식(57k bits 레지스터, 약 0.5M게이트) 대비 복잡도 대폭 절감.
10. 한계: 인코더 HW 구조에 국한되며 디코더/정정성능(BER, error-floor, iteration)에 대한 언급이나 실험이 전혀 없고, ASIC이 아닌 FPGA 합성 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Quasi-cyclic RAM 기반 인코더 구조 | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 고정 인코더를 사용하나, RAM 기반 저복잡도 QC 인코딩 아이디어는 인코더 HW 재설계 시 참고 가능 |
| 부호용 PCM(H_I) 기반 parity 생성 | `encoder.cpp` `Generate_PCM_encoding()` | 정보부 PCM 구조에서 parity를 직접 생성하는 방식이 유사한 레이어 |
| Partitioned RAM order control(column reordering) | `encoder.cpp` `Column_Reordering_Dual_Update()` | 컬럼 순서 재배치라는 컨셉은 유사하나 목적(순서 문제 해결 vs dual-update)이 다름 |
| Programmable 다중 코드 지원 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정 구조로 21종 코드 지원 개념 자체가 대응 없음 |

> 적용 가치: **중간** — 인코더 HW 복잡도 절감을 위한 RAM 기반 quasi-cyclic 인코딩 아이디어는 참고 가치가 있으나, 정정성능/디코더 개선과 무관하고 Prime ECC는 이미 dual-diagonal 인코더가 고정되어 있어 재설계 부담이 있음(이식 난이도 중~높음).
