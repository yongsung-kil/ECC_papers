### B. 알고리즘 요약

1. 시스템: 딥스페이스 BPSK/AWGN 채널, IF 70MHz, 정보율 16.67kbps, 심볼율 100ksps, `(5120,1024)` QCARA-GLDPC (rate 1/5).
2. 문제: 극저 SNR(-4.6dB)에서 PLL 기반 동기화는 매우 좁은 loop 대역이 필요해 설계·디버깅이 어려움 → PLL 없는 수신기 필요.
3. 모델: 잔류 timing/carrier 오차 `τk=θt+kΔft`, `φk=θc+kΔfc`를 가진 BPSK 신호 (식 1).
4. 핵심 기법: LDPC 복호 posteriori LLR 합 `ψ(κ)=Σ L(Qi|κ)`을 동기화 오차 `κ`의 목적함수로 정의 (식 4), 완전 동기화 시 최대.
5. 의미: Gaussian approximation상 posteriori LLR 합이 입력 SNR에 양의 상관 → LLR 합 최대화가 곧 동기화 오차 최소화.
6. 확장: 4파라미터(θt,Δft,θc,Δfc)를 carrier/timing 2개 평면으로 분리, 각각 Nelder-Mead simplex search로 peak 탐색.
7. 구현 디테일: coarse 보정(FFT 65536-pt periodogram으로 Δfc, DA phase 추정, 4-path 상관기 frame 동기)으로 simplex 수렴 영역 안에 초기오차 진입.
8. 절차: simplex 20 iteration, 참조 시 LDPC 3 iteration, 최종복호 50 iteration.
9. 결과: FPGA 실측 BER 10^-5 @ SNR -4.6dB, 이상대비 열화 <1.3dB (양자화 손실 고려 시 <0.3dB), 8-bit 양자화 시 BER 10^-5에서 약 1dB 손실.
10. 한계: 저rate GLDPC + 딥스페이스 동기화 특화, 디코더는 표준 sum-product partially-parallel일 뿐 신규성 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-product LDPC 복호(partially parallel, 8-bit) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 우리는 min-sum 채택, full sum-product는 대응 없음에 가까움 |
| 동기화용 LLR 합 목적함수 | 대응 없음 | NAND에는 carrier/timing 동기화 개념 자체가 없음 |
| Simplex search 기반 파라미터 추정 | 대응 없음 | ECC 코어와 무관한 수신기 전처리 |
| QCARA-GLDPC 부호구조 | 대응 없음 | GLDPC/저rate, Prime ECC 고rate QC-LDPC와 이질적 |

**적용 가치**: `낮음` — 기여의 핵심이 딥스페이스 수신기 동기화이고 LDPC 부호/디코더는 표준 재사용이라 NAND binary QC-LDPC ECC에 떼어낼 요소가 없음.
