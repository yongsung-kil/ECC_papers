> 총평: min-sum과는 이질적인 majority-logic 기반 저복잡도 반복 디코딩 기법으로, 저bit 양자화(3~4bit)와 빠른 수렴이 매력적이나 heavy weight 구조 코드 한정 및 CNU/VNU 코어 재설계 필요성 때문에 이식성은 중간 수준.

### B. 알고리즘 요약

1. 대상은 heavy row/column weight를 갖는 structured binary LDPC(예: (961,721) QC-LDPC γ=ρ=30, (255,175) EG-LDPC γ=ρ=16), BPSK+AWGN 채널, 수신 신호를 정수로 양자화 후 복호.
2. 기존 OSMLGD/MOSMLGD는 1회 flip만 수행해 성능 저하가 크고, SPA는 성능은 좋으나 복잡도가 매우 높은 trade-off 문제를 해결하고자 함.
3. 채널모델은 표준 AWGN+BPSK이며, 신뢰도 척도로 수신 심볼 크기 `|y_j|`를 비균일 양자화한 정수값 `q_j` 사용.
4. 핵심 기법: check-sum 기반 가중 신뢰도 `p_j^(k) = Σ_{i∈M_j}(1-2s_i^(k))w̃_{ij}^(k)` (식 11)를 매 iteration 갱신, `w̃_{ij}^(k) = min_{j'∈N_i\j}|R_{j'}^(k)|` (min-sum과 유사한 min 연산이나 majority-logic 프레임워크).
5. flip 대신 turbo-like 갱신식 `R_j^(k+1) = R_j^(0) + sign(R_j^(k))[α·p_j^(k)]` (식 13)로 신뢰도를 점진적으로 이동, `α`는 사전정의 scaling factor.
6. **iterative set** `J_presented = {j : p_j^(k) < 0}` (식 12) 개념 도입 - 이 집합에 속한 VN만 갱신 수행, 시뮬레이션상 활성 VN 비율 약 5%로 계산량 대폭 절감.
7. 비균일 양자화 함수(식 9): 파라미터 `r`(0<r<1), `b`(bit 수)로 원점 근처일수록 촘촘한 양자화 스텝 적용, 3~4bit 해상도로도 우수한 성능 확보.
8. 별도 학습 절차 없음(파라미터 r, α는 사전 설계값 사용).
9. 결과: (961,721) QC-LDPC 코드, 4bit 양자화 기준 BER=10⁻⁴에서 OSMLGD 대비 1.6dB, WBF 대비 0.6dB, RBI-MLGD 대비 0.35dB 개선, SPA 대비 0.5dB 이내로 근접; 평균 반복횟수 Eb/N0=3.8dB에서 약 2.3회(SPA와 유사 수준)로 RBI-MLGD(약 5.3회) 대비 약 3배 빠른 수렴.
10. 한계: HW 설계 없음(순수 알고리즘/시뮬레이션), heavy column weight 구조(finite geometry/QC-LDPC)에만 적용 가능하며 저 column weight의 random-like LDPC에는 심각한 성능 저하 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| iterative reliability 갱신(majority-logic 기반, min-sum 대체) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN/VN 갱신 루프 자리에 대응되나 알고리즘 구조(check-sum 기반 majority vote)가 min-sum과 달라 코어 로직 교체 수준의 개선 필요 |
| iterative set(활성 VN만 갱신, 계산량 절감) | `decoder.cpp` iteration 루프 내 VN 처리 | 저활성률(5%) 갱신 아이디어는 HW pipeline/GT(Graph Thinning)와 유사한 목적이나 알고리즘 트리거 조건이 다름 |
| 비균일 양자화(3~4bit) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | LLR 양자화 테이블 설계에 참고 가능, NAND soft-read 비용 절감과 직결 |
| 조기종료(syndrome=0 체크) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 매 iteration syndrome 검사로 조기종료하는 방식은 기존 CRC 조기종료와 유사 목적 |

> 적용 가치: 중간 — 저복잡도·저bit 양자화·빠른 수렴은 매력적이나 min-sum 프레임워크와 이질적인 majority-logic 코어이며 heavy column weight 코드 한정이라 Prime ECC(VN degree 17, 상대적으로 저weight) H-matrix에 그대로 이식 가능한지는 별도 검증 필요.
