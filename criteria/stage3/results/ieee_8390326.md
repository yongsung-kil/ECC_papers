> 총평: high-confidence stuck-cell 오류를 erasure로 처리(LLR=0)하는 아이디어는 채널/LLR 계층에 저비용 이식 가능하나, sectionalized FNW는 write-side 데이터 shaping·stuck 위치 사전지식 의존이라 이식성 중.

### B. 알고리즘 요약

1. 대상은 NAND(2D→3D SLC 예시) SSD의 rate 0.9, 1KB(2섹터) LDPC 부호, 3-strobe soft-decision 복호(SDD).
2. 문제: bit/word line short·open으로 생기는 stuck cell은 확률 1/2로 "high-confidence 오류"를 유발해 LDPC 반복복호를 탈선시킴(BCH/RS와 달리 LDPC엔 3배 치명적, "1 high-conf 오류 ≈ 3 low-conf 오류").
3. 모델: 총 RBER = `RBER_overlap + q/2`(q=cell stuck 확률), stuck 기여 `q/2`가 고신뢰 오류라 SDD 이득이 2.6×→1.36×로 붕괴(q=2e-3에서 1.92× 열화).
4. 알고리즘1(errors-and-erasures): stuck 위치를 read 시 입력받아 해당 LLR=0(erasure)로 설정 → 잘못된 고신뢰 정보보다 "정보 없음"이 유리(RS는 무의미하나 LDPC는 신뢰도 중요).
5. 효과1: SDD에서 stuck-read 대비 1.92× RBER 이득(erasure 채널 `q/2` 기여가 동일 총RBER의 error 채널보다 우월), HDD는 1.175× 이득.
6. 알고리즘2(sectionalized FNW): redundancy를 LDPC parity와 데이터 shaping flag로 나눠, 각 section 데이터를 flip/non-flip 중 stuck read와 Hamming 거리 작은 쪽으로 저장 → stuck cell을 고신뢰 정답으로 만듦.
7. FNW 이론: stuck s개일 때 저장오류를 최대 `floor(s/2)`로 절단, 평균오류율도 s 작을수록 감소(식 (1)) → 512B 섹터를 k개 section으로 분할(작은 s 유지).
8. 파라미터: k=10 section, section 길이 n=820, section당 flag 1비트(총 10비트, LDPC parity 10비트 puncturing으로 조달→추가 redundancy 0), post-FNW RBER 4.33e-4(약 23배 감소, parity 미보호 반영 시 4.89e-4).
9. 결과: sectionalized FNW로 stuck-read 대비 1.94× RBER 이득(시뮬), "divide-and-conquer"(FNW=stuck 오류, LDPC=Vt overlap 오류 분담)가 단일 LDPC redundancy보다 우수.
10. 한계: HW 미설계·시뮬만, stuck cell 위치를 read/write 시 안다고 가정(위치추적은 별도 특허, 범위 밖), FNW는 write-side 전처리 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stuck cell을 erasure로 플래그, LLR=0 주입 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` + [ecc_data.h](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `PARAM_LLR` | soft-read LLR 설정 지점에서 특정 비트 LLR=0(erasure) 지정으로 저비용 접목 |
| errors-and-erasures 반복복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `VN_Cal_HD()` 등 | erasure 입력(LLR=0)을 min-sum 루프가 수용하는지 검증 필요(알고리즘 자체 변경은 없음) |
| sectionalized FNW 데이터 shaping(write-side) | 대응 없음 | 인코딩 전 데이터 전처리 계층으로 Prime ECC(dual-diagonal 인코더)에 해당 모듈 부재 |
| FNW flag 비트를 parity puncturing으로 조달 | [encoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_encoder_4KB()` | puncturing/오버헤드 편성은 인코더·PCM 구조와 정합 필요 |

적용 가치: **중간** — stuck cell을 erasure(LLR=0)로 다루는 첫 기법은 channel/LLR 계층에 거의 무비용 이식 가능(고rate binary LDPC·NAND 직접 타깃). 다만 sectionalized FNW는 write-side 데이터 shaping과 stuck 위치 사전지식에 의존해 별도 계층 신설이 필요.
