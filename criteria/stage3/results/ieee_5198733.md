### B. 알고리즘 요약

1. 대상은 (Wc,Wr) regular LDPC, 예시로 (6,32)(2048,1723)와 (4,16)(1536,1155) 코드, AWGN+BPSK 채널.
2. Split-Row는 check node(CN) 처리를 2개 이상 파티션으로 나눠 상호 통신을 줄여 interconnect 복잡도를 낮추지만, 파티션이 전체 row의 global minimum을 모르는 문제로 0.4~0.7dB 성능 손실 발생.
3. 기존 Split-Row Threshold는 로컬 `Min1`만 threshold `T`와 비교해 다른 파티션에 `Threshold_en` 신호로 전달하나, `Min2`는 비교하지 않아 일부 오차를 못 잡음.
4. 제안 기법(Improved Threshold)은 각 파티션에서 `Min1`, `Min2` 둘 다 `T`와 비교해 4가지 Condition으로 분기: Condition1(둘 다 ≤T)은 그대로 전파, Condition2(Min1≤T<Min2)는 이웃 파티션의 `Threshold_en`에 따라 `Min1,T` 또는 `Min1,Min2` 전파, Condition3(Min1>T, 이웃 en=1)은 `T`만 전파, Condition4(Min1>T, 이웃 en=0)은 기존 Split-Row와 동일.
5. `α_ij = S_MS × sign(β_ij) × min(|β_ij|)`(Eq.6, MinSum Split-Row 기본식)에서 Min2 비교 추가만으로 파티션 간 정보 손실을 줄이는 것이 핵심.
6. HW는 기존 CN 프로세서에 "Thresholding Logic" 블록(mux 2개 + select logic)만 추가하는 최소 변경.
7. 검증은 AWGN/BPSK 채널 시뮬(80 error block, 최대 15 iteration)과 65nm CMOS synthesis.
8. threshold T는 SNR·iteration 수에 무관하게 고정값(예: T=0.2, T=0.3) 사용 가능함을 실험으로 확인.
9. (2048,1723) 코드에서 Split-Row 대비 0.27dB, Split-Row Threshold 대비 0.10~0.20dB 개선, SPA와는 0.08~0.13dB 차이. CN 게이트카운트 3%, 면적 16% 증가에 그침. 별도 칩(65nm, 8.9mm²)은 5.5Gbps@15iter.
10. 한계: Split-Row 계열(다중 파티션 CN) 아키텍처 전용이며, 본 논문은 알고리즘/합성 결과만 제시하고 전체 디코더 chip은 별도 논문[17] 인용뿐. FPGA/실측 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum 연산(Min1/Min2 산출) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2 기반 CN 연산을 이미 수행하나 단일 파티션 구조 |
| Split-Row 파티션 분할 구조 | 대응 없음 | Prime ECC는 z_sb=32 row 단위 병렬이며 CN을 다중 파티션으로 쪼개는 구조가 아님 |
| Thresholding Logic (mux 기반 조건 분기) | 대응 없음 | 유사 개념 없음, 이식 시 신규 로직 추가 필요 |
| iteration/SNR 무관 고정 threshold | `ecc_data.h` `PARAM_LLR` (개념적 유사) | Prime ECC는 iteration별 LLR threshold 테이블을 이미 사용 중이라 아이디어는 유사 |

> 적용 가치: 낮음 — 대상 아키텍처(Split-Row 다중 파티션)가 Prime ECC 3.1의 단일 CN 구조와 근본적으로 달라 이식 시 구조 재설계가 필요하고, 성능 개선폭(0.1~0.3dB)도 이미 Prime ECC가 보유한 min-sum+HCU 대비 크지 않음.
