### B. 알고리즘 요약

1. 시스템: HDD magnetic recording iterative read channel — rate-8/9 regular QC-LDPC(column weight 4) + SOVA 검출 + dominant error event post-processor, 512B sector, 목표 1.5Gb/s(component 2Gb/s), 최대 채널 iteration 4, 65nm.
2. 문제: 고급 iterative 검출/복호가 실리콘 면적·에너지 오버헤드를 크게 유발. eDRAM(SRAM 대비 3배 밀도, 0.8배 에너지)을 on-chip 메모리로 써 이를 완화.
3. baseline: recursive 아키텍처(component throughput를 목표보다 높이고 equalizer-SOVA 사이 버퍼 삽입). on-chip SRAM이 총면적 68% → eDRAM drop-in 시 총면적 45% 절감.
4. 핵심기법1 (조건부 post-processing): 매 sector마다 무조건 post-proc 하지 않고, 먼저 LDPC 복호 후 실패한 sector만 post-proc 호출. 대부분 sector가 1st pass에서 성공하므로 post-proc 연산 99.99%+ 제거.
5. 이를 위해 두 프레임(6bit 채널출력 + 1bit hard decision) 저장 버퍼 필요; SRAM 0.31mm2 vs eDRAM 0.1mm2 → eDRAM이 이 방식을 경제적으로 정당화.
6. 핵심기법2 (전압 오버스케일링): LDPC 복호 iteration 수가 sector마다 크게 변동(평균 << 최대). 대부분 decoder가 idle → 공급전압을 낮춰 에너지 절감.
7. 트릭: fine-grain 동적 스케일링은 오버헤드 크므로, eDRAM 버퍼를 detector-decoder 사이 삽입해 fixed voltage scaling 적용; 전압 낮추면 decoding overflow↑ → 버퍼 용량으로 상쇄.
8. 분석: 버퍼 overflow 확률 `P` upper bound를 iteration 통계 `p_i`로 추정(식4), circuit delay ∝ `V/(V-V_th)^α`로 전압스케일 팩터 `K` 산출(식5), velocity saturation index α=1.2/1.5/2.
9. 결과: post-proc 조건부실행으로 35mW 절감(추가 0.1mm2), LDPC 디코더 전압스케일링으로 최대 67% 복호 에너지 절감(버퍼 에너지 오버헤드 고려한 total도 유의미).
10. 한계: HDD read channel 특화, LDPC 부호/디코더 알고리즘 개선 없음(SP+QC-LDPC baseline 재사용), 전압스케일링 이득이 큰 버퍼에서 eDRAM 에너지에 상쇄되어 포화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| iteration 수 통계 기반 조기수렴/에너지 최적화 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` (조기종료) | 낮음: Prime ECC는 CRC 조기종료로 iter 변동 이미 활용, 전압스케일링은 시스템/전력 레이어라 코드 무관 |
| baseline sum-product QC-LDPC 디코더 | `decoder.cpp` `LDPC_Decoder()` (Prime ECC는 min-sum) | 낮음: 부호/알고리즘 자체는 baseline 재사용, 신규 기여 아님 |
| 조건부 post-processing (error event detection) | 대응 없음 (SOVA + magnetic error event post-proc) | 낮음: NAND에는 SOVA/dominant error event 개념 없음 |
| eDRAM 버퍼/전압 오버스케일링 (시스템 전력) | 대응 없음 (HW 전력·메모리 레이어) | 낮음: bit-exact 시뮬레이터 범위 밖 |

적용 가치: **낮음** — 부호설계/복호 알고리즘 기여가 없고 HDD read channel의 eDRAM 버퍼·전압스케일링이라는 시스템 전력 최적화라 NAND용 Prime ECC(부호→디코더→HW)에 떼어 쓸 알고리즘 요소가 사실상 없다.
