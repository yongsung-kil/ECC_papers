> 총평: DVB-S2 mother code에서 puncturing 대신 bit duplication으로 저rate 코드를 만드는 통신 방송용 rate-compatibility 기법으로, NAND의 고rate 단일 부호 요구와 방향이 달라 이식 가치가 낮음.

### B. 알고리즘 요약

1. 시스템: DVB-S2 irregular LDPC mother code (`n=64800`, `rate=1/2`)를 기반으로 AWGN 채널, BPSK 변조, sum-product 디코딩(최대 100 iteration)에서 다중 rate(1/3, 1/4, 2/5) 코드를 얻는 문제.
2. 문제: 기존 puncturing(정보비트 스트리핑)은 저rate 획득 시 성능 손실이 있고, extension/row-combining은 코드 길이나 parity 구조가 바뀌어 재설계 비용 발생.
3. 핵심 가정: check matrix `H(n,k)` 구조와 block length를 완전히 고정한 채로만 rate를 바꾼다는 제약.
4. 핵심 기법: "bit duplication"(BD) — 인코딩 전 MUX.1에서 원 메시지 뒤에 0-시퀀스(길이는 목표 rate에 따라 `k/3`~`k/2`)를 붙여 인코딩한 뒤, MUX.2에서 codeword의 0-시퀀스 자리를 정보비트의 중복(duplicate) 구간으로 치환해 전송 codeword `w`를 구성.
5. 핵심 식: 수신단 DEMUX.1에서 중복된 두 위치의 LLR을 합산 `L(ui)=L(ui)+L(ui+offset)` (i.e. rate 1/3: `i=1...k/3`, offset=`2k/3`)로 반복코딩 이득을 얻고, 남은 0-시퀀스 자리는 `L(ui)=inf`로 채워 mother code 디코더에 그대로 입력.
6. 확장: rate 1/3, 1/4, 2/5 세 가지 목표 rate에 대해 각각 duplication 길이·오프셋을 다르게 설계 (예: rate 1/4는 정보부 전체를 완전 중복).
7. 부수 트릭: 디코더/인코더는 mother code(rate 1/2)의 H-matrix를 그대로 재사용하고, MUX/DEMUX 전단 처리만 추가해 인코더·디코더 재설계 비용을 없앰.
8. 학습/최적화: 없음(구성법 논문, DE/GA/NN 없음). Matlab 시뮬로 BER만 비교.
9. 결과: BER=10^-5에서 PIB(puncturing) 대비 rate 1/4에서 약 0.5dB, rate 1/3에서 약 0.5dB, rate 2/5에서 약 0.1dB 성능 우위.
10. 한계: 대형 방송용(64800bit) irregular LDPC 대상이며 목표가 rate를 낮추는 것(1/3~2/5)이라 NAND ECC의 고rate(~0.9) 요구와 반대 방향. HW 복잡도/latency 분석 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit duplication 기반 rate-compatible 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 고rate QC-LDPC 단일 구성이며 rate-compatible 다중 rate 지원 없음, 구조 재사용 난해 |
| LLR 합산(duplicate 위치 결합) | `channel.cpp` `Set_LLR_Th()` | 개념적 유사성만 있고, 대상이 저rate 반복코딩용이라 고rate NAND ECC와 무관 |
| sum-product 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum(Sum-Product 근사)이라 알고리즘 자체 이식은 아니고 참조 배경일 뿐 |
| H-matrix 고정, MUX/DEMUX 전처리 | `encoder.cpp` | rate 조절 목적이 반대(저rate 생성)라 인코더 구조 참고 가치 낮음 |

적용 가치: `낮음` — DVB-S2 방송용 저rate rate-compatible 구성법으로 NAND 고rate 단일 QC-LDPC 요구와 목적이 어긋나 이식할 요소가 거의 없음.
