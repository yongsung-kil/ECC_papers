### B. 알고리즘 요약

1. 대상은 rate 0.9의 protograph 기반 quasi-cyclic S-IRA(4544,4096) 부호(M=7, K=64, Q=64, VN degree wc=5), AWGN 채널, FPGA 상에서 SPA(5-bit)와 min-sum(6-bit) 두 디코더로 비교한다.
2. 문제는 LDPC가 waterfall 영역에서는 우수하나 저장장치급(10⁻¹²) BER에 필요한 낮은 error floor를 달성하지 못한다는 것으로, 이 floor는 저무게 near-codeword(trapping set)에 기인한다.
3. `(ω,ν)` trapping set을 무게 ω인 오류패턴이 ν개의 unsatisfied check를 유발하며 반복복호가 벗어나지 못하는 부분그래프로 정의하고, FPGA 실측(디코딩 실패 통계) + H-matrix 서브행렬 탐색 + importance sampling(IS) 3단계로 dominant trapping set을 식별한다.
4. 발견된 3대 클래스: (5,5) 오각형 구조(6-cycle 5개 union), (5,7)(6-cycle 7개 union), (6,2)(8-cycle 6개 union, oscillating error 유발); 각 오류블록의 50% 이상이 5-systematic-bit 오류로 발생.
5. 대응책 1(bit-pinning/code-shortening): trapping set에 속한 VN group(예: `vp=62`, 64bit)을 인코더에서 0으로 고정 후 삭제, 디코더에서 해당 위치 LLR을 최대값(+15/+31)으로 고정 — 부호를 (4480,4032)로 축소하되 floor를 크게 낮춤.
6. 대응책 2(on-off attenuated min-sum, aMS): min-sum CN 출력이 SPA보다 과대평가되는 경향을 짝수 반복에서만 CN 메시지를 1/2로 스케일링(1-bit shift로 HW 구현)하여 완화 — aMS는 waterfall 0.1dB 열화로 floor를 100배 낮추고, on-off aMS는 추가로 100배(총 waterfall 손실 0.05dB 미만) 개선.
7. 대응책 3(outer 부호 연접): trapping set의 bit-error 위치를 미리 알고 있다는 점을 이용해 RS(410,406, GF(2^10), t=2)나 BCH(4096,4018, t=6) 심볼 매핑을 trapping set 클래스에 맞춰 설계 — RS-LDPC는 floor 10배, BCH-LDPC는 1.9×10¹⁰ 코드블록 시뮬 중 오류 0건.
8. 별도 학습 절차는 없으며, trapping set 탐색은 (a) FPGA 실패통계 카탈로그 → (b) H-matrix 서브행렬(예: 4×14 generalized submatrix) 계산적 탐색(multiplicity Q=64로 탐색량 축소) → (c) IS로 변종(6,5) trapping set까지 확장하는 3단계 알고리즘.
9. 결과: bit-pinning으로 FER floor 대폭 개선(레이트 손실 미미, 0.9→0.899), on-off aMS로 waterfall 손실 0.05dB 이하에 FER floor 10⁴배 개선, BCH-LDPC 연접으로 5.2dB에서 사실상 error floor 제거(오류 0건/1.9×10¹⁰ 블록).
10. 한계는 결과가 특정 S-IRA(4544,4096) 부호와 그 dominant trapping set 구조에 특화되어 있고(랜덤 코드나 trapping set이 많은 코드에는 비효율적), 저자 스스로 임의 코드로의 일반화는 명시적으로 제한함.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Trapping set 탐색(FPGA 실측+H-matrix 서브행렬+IS) | `ecc_top.cpp` `Load_PCM()` | Prime ECC의 H-matrix(PCM_b)에서 동일 절차로 dominant trapping set 사전 카탈로그 가능 - 부호 검증/개선에 직결 |
| On-off attenuated min-sum (짝수 iteration만 CN 메시지 1/2 스케일링) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산에 1-bit shift 어테뉴에이션만 추가하면 되는 매우 낮은 HW 비용의 error-floor 개선 기법 - 이식 난이도 낮음 |
| Bit-pinning / code shortening | `encoder.cpp` `Generate_PCM_encoding()`, `ecc_top.cpp` `Load_PCM()` | 특정 trapping set VN group을 고정 LLR로 처리하는 방식, 부호구조(H-matrix) 수정이 필요해 이식 난이도 중~높음 |
| 조기종료/오류블록 재처리(outer BCH/RS 연접) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC의 CRC 조기종료와 별개 레이어(외부 대수부호)로, LDPC 디코더 실패블록에 대한 2차 방어선 아이디어로 참고 가치 |
| 5-bit SPA vs 6-bit min-sum 양자화 비교 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 양자화 비트수와 error floor 상관관계 검증에 참고 가능 |
```

적용 가치: **높음** — on-off attenuated min-sum은 min-sum CN 연산에 최소 HW 비용(1-bit shift)만 추가해 error floor를 크게 낮추는 기법으로 이식 난이도가 매우 낮고, trapping set 사전 탐색 방법론은 Prime ECC의 실제 H-matrix에 그대로 적용해 error floor 취약점을 사전 진단할 수 있음.
