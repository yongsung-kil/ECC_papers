### B. 알고리즘 요약

1. 대상은 regular time-invariant LDPC convolutional(RTI-LDPCC) 부호로, 복잡도 척도는 syndrome former constraint length `vs = q·n0`.
2. 기존 [6](Tanner)·[9](PDC-LDPC) 방식은 high-rate에서 constraint length가 크거나 error-floor가 높아, 짧은 `vs`와 좋은 성능을 동시에 달성하기 어려움.
3. 채널은 AWGN + BPSK, 복호는 sliding window 위의 belief propagation(sum-product, LLR) 가정.
4. 핵심: array LDPC 블록부호(순환 permutation 행렬 `P`의 지수를 `Ha` 원소로 지정, 소수 `q` 사용)를 시작점으로 삼음.
5. array 블록부호 `H`를 행·열 재배열해 "circulant of blocks" 형태 (식 3)로 만든 뒤 [7]의 unwrapping을 시간불변 cutting pattern으로 적용해 반무한 `Hconv`(식 4) 유도.
6. 결과 부호의 파라미터: 점근 rate `R=k0/n0`, 열무게 `r0`, memory order `ms=q`, constraint length `vs=q·n0`.
7. Δ 집합을 proper(`{0,1,...,r0-1}`)/improper로 바꾸고 shortening으로 min-distance를 조정(improper Δ는 휴리스틱 최적화).
8. [6]의 시작점 지수행렬 (식 5)이 특정 array 부호의 열재배열·shortening 특수경우임을 보여, 더 일반적인 array 부호를 시작점으로 삼는 것을 정당화.
9. 결과: rate 0.9에서 C3가 Ca[6]·Cb[9] 대비 0.6 dB 이상 이득이면서 `vs`는 절반 이하; rate 0.75에서 C4/C5가 더 높은 min-distance(10~12) 달성.
10. 한계: irregular time-varying LDPCC 대비 0.8 dB 손실, HW 미설계, error-floor 정량곡선은 그림에만 존재.
