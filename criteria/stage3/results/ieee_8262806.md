### B. 알고리즘 요약

1. 시스템: LDPC를 블록 단위로 적용(분산스토리지 HDFS/위성-지상 통신), erasure BP(peeling) 복호, 예시 (8192,7561) rate 0.923 정규 `(dv=3,dc=39)`.
2. 문제: BP가 못 푸는 stopping set이 남았을 때, **최소 개수의 erasure를 제거**(원격 회수/재전송)하여 나머지를 `k` iteration 내 복호되게 만드는 SSEk 문제 정의.
3. 모델: stopping graph `G=(V∪C,E)`, 모든 check node degree ≥2, k iteration 내 복호되면 Decodable Set.
4. 핵심 결과 1: `SSE∞` NP-hardness — Set Cover에서 파생한 Pseudo Set Cover(NP-complete)를 gadget `D_{i,j}`(방향성: `si` 알면 `tj` 복호, 역은 불가)로 환원.
5. 핵심 결과 2: 유한 `k`(심지어 `k=1`)도 NP-hard — Not-all-equal SAT을 gadget `Vi`/`Wj`+H-bar 연결의 `Gsse`로 환원, one-iteration elimination set 크기 `n+3k`가 satisfiability와 동치.
6. 근사 알고리즘(k=1): white/gray/black 3색 그리디, 이웃 `Λ(v)` 제거로 elimination set 구성.
7. 근사비: 최대 degree `dv,dc`에 대해 approximation ratio `dv(dc-1)`, 시간복잡도 `O(dv²dc²|V|)`.
8. 트리 최적 알고리즘: stopping graph가 tree/forest일 때 BFS 후 역순 처리로 gSSEk(파라미터 `δ(v),ω(v)`)를 하위트리로 축약, 선형시간 `O(|V|+|C|)` 최적해.
9. 결과: 순수 이론(정리·보조정리), stopping-set 평균크기가 RBER 0→1에서 0→8192로 선형 증가하는 통계 그림 외 성능 수치 없음.
10. 한계: BEC/erasure·peeling BP 전용, HW·복호성능 개선 아님, 응용이 분산스토리지 원격회수·위성 재전송으로 NAND ECC와 무관.
