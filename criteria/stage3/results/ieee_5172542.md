### B. 알고리즘 요약

1. 대상은 임의의 구조적(QC) 또는 비구조적(unstructured) 이진 LDPC 코드 `(N,K)`, sparse parity-check matrix `H` 기반 belief propagation(BP) 디코딩, 병렬도 `p`는 자유롭게 선택 가능.
2. 기존 QC 전용 디코더는 확장계수 `z`의 배수인 병렬도에서만 무충돌 메모리 접근이 가능하고, 다른 `z`를 갖는 여러 코드를 동일 칩에서 디코딩하기 어려움(활용률 100% 미만) — 이를 임의 코드/병렬도로 일반화하는 것이 목표.
3. 채널/노이즈 모델은 표준 BP 방정식(식 1~4): intrinsic `In = ln(P(yn=+1|rn)/P(yn=-1|rn))`, check-node update `Rm→n`(tanh 기반), variable-node update `Sn = In + Σ Ri→n`.
4. 핵심 기법은 sum-value `Sn`(기존 [8]의 extrinsic value 대신)에 매핑 함수 `T`를 적용하는 동적(dynamic) 메모리 매핑: `E`개의 sum-value 복사본을 두 개의 파티션 `P`, `P'`로 나누고 그래프 컬러링 문제로 풀어 `p`개 메모리 뱅크에 충돌 없이 배치.
5. 핵심 식은 매핑 함수 조건 `vi,vj ∈ Vk ⇒ T(vi)≠T(vj)` — 같은 서브셋(subset) 안의 값은 항상 다른 메모리뱅크에 저장되어야 충돌이 없다는 그래프 컬러링 제약.
6. 확장: CN 처리와 동시에 VN 누적합을 갱신(`Sn,k+1 = Sn,k + Ri→n`)하여 CN/VN을 병렬 처리, 기존 [8]의 교대(alternate) CN/VN 처리 대비 클록당 데이터 갱신량이 `p → 2p`로 2배 증가.
7. HW 디테일: `p`개의 intrinsic/extrinsic/sum(A,B) 메모리 뱅크(dual-port), 4개의 permutation network(Beneš 네트워크로 구현 가능), 반복마다 SMA/SMB 뱅크를 교환하고 역방향 데이터패스 사용.
8. 학습/최적화 절차 없음 — 매핑은 그래프 컬러링 알고리즘(존재성 증명은 [8] 인용)으로 결정론적 계산.
9. 결과(정성적 비교, Table I): 기존 heuristic 매핑([9]~40%, [10]~80%) 대비 제안 구조는 임의 코드에서 100% 활용률, 클록당 갱신 수 `2p`(기존 `p` 대비 2배, [9] 대비 최대 5배); 메모리 크기는 `3N+E`(평균 코드 E=3.5N 기준 6.5N)로 [10]/[8]보다 크지만 [9]보다는 절반 이하.
10. 한계: BER/FER 정정성능 실험 없음(순수 아키텍처/매핑 이론), gatecount·throughput(Gbps) 수치·ASIC/FPGA 합성 결과 없음, 비구조 코드 지원을 위해 random parity-check matrix를 control memory에 저장해야 하므로 QC 대비 제어 메모리 오버헤드 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN/VN 병렬 처리(동시 갱신으로 throughput 2배) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 pipeline 6-stage(col_M1/Z/P1~P4) 설계와 개념적으로 유사한 병렬화 아이디어이나 Prime ECC는 이미 고정 QC z=32 구조라 직접 이식보다는 참고용 |
| sum-value 기반 동적 메모리 매핑(그래프 컬러링) | 디코더 HW 모델(z=32, HCU, GT) 및 `channel.cpp` LLR 저장 | Prime ECC는 고정 QC-LDPC라 barrel shifter 기반 정적 매핑을 사용 중, 범용 비구조 지원용 동적 매핑은 오버엔지니어링(H-matrix 고정 전제와 배치) |
| 임의 병렬도 p에 대한 100% 활용률 보장 | GT.cpp Make_GT_HW() (MUX 기반 edge 선택) | Prime ECC의 GT(Graph Thinning)와 목적(효율적 edge/메모리 처리) 유사하나 접근 방식(그래프 컬러링 vs MUX 테이블)이 다름, 참고 수준 |

적용 가치: `중간` — Prime ECC는 이미 특정 QC-LDPC(z=32) 고정 구조라 범용 프로그래머블 아키텍처 자체는 불필요하지만, CN/VN 병렬 동시처리로 throughput을 2배로 늘리는 아이디어와 무충돌 메모리 매핑 설계 원칙은 HW 파이프라인 최적화 시 참고 가치가 있음.
