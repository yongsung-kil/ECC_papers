### B. 알고리즘 요약

1. IEEE 802.16e(WiMax) 블록 LDPC(rate 1/2, 2/3, 3/4, 5/6, base matrix 24열, expansion factor z=24~96)를 대상으로 하는 layered(TDMP) 디코더.
2. 기존 TPMP(two-phase message passing)는 check/variable node 갱신을 완전히 분리해 처리하므로 iteration당 throughput이 낮고 메모리 소모가 큼 — 이를 개선하려는 문제의식.
3. 채널은 BPSK+AWGN 가정, offset min-sum(OMS)을 check-node 업데이트 알고리즘으로 채택.
4. 핵심 성질: 각 check node m에 대해 `R_mn`이 실질적으로 `M1, -M1, ±M2` 세 값만 가짐(offset 보정 포함) — 이를 이용해 CNU가 k개 값 대신 `M1, M2, index, cumulative sign` 4개 정보만 저장하는 value-reuse 구조 설계.
5. `R_mn = M1` (n이 M1의 인덱스가 아닐 때), `= M2` (n이 M1의 인덱스일 때) — 이 압축으로 R 메모리 57% 절감.
6. TDMP는 j개 block row를 layer로 보고, 한 layer의 check-node 처리 직후 바로 variable-node 메시지를 갱신해 다음 layer에 사용 — iteration을 최대 50% 절감.
7. 서로 다른 expansion factor(z=24,48,96)를 지원하기 위해 Master-Slave Benes router로 cyclic shifter의 제어신호를 동적 생성(고정 메모리 대신 실시간 정렬 기반 생성) — 기존 Omega/Benes 방식 대비 제어신호 메모리 대폭 절감.
8. 최적화 절차 없음(구조 설계 논문, DE/GA 미사용).
9. 결과: 기존 TPMP 대비 총 메모리 63% 절감, BCJR 기반 TDMP[2] 대비 55% 절감, 라우터 50% 절감, throughput 2배(iteration 절반), SNR 페널티 0.15dB 미만.
10. 한계: FPGA 구현(110MHz)에 한정, ASIC 합성 결과 없음; 특정 WiMax 표준 부호 구조(24열 base matrix)에 최적화되어 범용성은 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CNU value-reuse (M1/M2/index 압축) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2 기반 min-sum을 사용하므로 메모리 압축 기법 비교 가능 |
| layered(TDMP) 스케줄 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC의 Dual-Update(even/odd 교대) 스케줄과 유사한 layered 개념 |
| cyclic shifter / Master-Slave Benes router | HW arch (병렬화, 메모리, 라우팅) | Prime ECC z_sb=32 고정 병렬 구조와 달리 다중 expansion factor 지원용 라우팅 — 구조 상이 |
| FS/PS 메모리 압축 구조 | `ecc_data.h` PARAM_DEC | 유사 개념(간결화된 메시지 저장) 참고 가능 |

> 적용 가치: 중간 — value-reuse 기반 메모리 절감 아이디어는 Prime ECC의 min-sum 구조와 개념적으로 겹치나(이미 보유), 다중 rate/z 지원용 Benes 라우팅은 Prime ECC의 고정 QC-LDPC 구조에는 이식 필요성이 낮음.
