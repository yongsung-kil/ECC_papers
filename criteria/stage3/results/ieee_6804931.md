> 총평: NAND 블록/페이지에 고정rate Raptor 외부부호와 BCH·block-wise product 내부부호를 연접해 hard-decision만으로 error-floor를 낮추는 부호설계로, binary QC-LDPC 디코더인 Prime ECC와는 부호 패밀리가 달라 직접 이식 대상이 아니다(2-bit soft LDPC와 성능 대등, 4bit+ LDPC엔 열세).

### B. 알고리즘 요약

1. 시스템: NAND MLC, page 8KB(내부 BCH 1KB×8), block 256 page(~2MB). 채널은 hard-decision 기반 BSC로 가정, 전체 rate 0.93.
2. 문제: 긴 BCH는 복잡도 급증, LDPC/turbo는 full 성능에 soft read(다중 sensing)로 latency↑ → hard-decision만으로 강한 정정을 원함.
3. 모델: 내부 디코더 실패를 erasure로 선언해 BSC를 symbol-erasure 채널로 변환, 외부 fixed-rate Raptor(R10) code로 erasure 정정.
4. 핵심 기법 1단(off-line block recovery): NAND 블록 전체를 하나의 Raptor codeword로, 내부 BCH 실패 codeword를 erasure로. 실패 확률 `P(E1|NE=i)=min{1,2^-(Ns(N-K)-Ns·i)}`.
5. 핵심 식: `m=A_pre^-1 t`(intermediate symbol), `r=G_LT m`(LT parity), 디코딩은 `A'm=c'`를 Gaussian elimination/inactivation decoding으로 해 full rank면 성공. mega-byte 크기라 직접 처리 불가.
6. 핵심 기법 2단(page correction): 내부부호를 block-wise product code(BW-PC, 교차부 다중공유 bit)로 두고, 실패한 row·column 교차 block을 erasure symbol로 태깅해 외부 Raptor로 정정. `nB`(교차 block bit수)가 핵심 설계변수.
7. 부수 트릭: look-up table(position vector) 기반 encode/decode로 controller 대용량 buffer 회피(SRAM<2.6MB). reduced decoding sphere(t=3->2, 6->5)로 inner miss-correction 저감.
8. 최적화: 목표 PER<10^-12, rate 0.93 만족하도록 `nB,kr,kc,tr,tc,N` 선택. Raptor symbol ≥1000개 위해 `nB=k/N≤64`.
9. 결과: BW-PC C2((2026,1960,6)/(1970,1904,6))가 raw BER 3.3e-3까지 PER 10^-12 달성, 8KB BCH 초과 성능, 2-bit LDPC와 대등. Raptor decode O(1e3) XOR로 RS(O(1e5)) 대비 우위.
10. 한계: HW 미설계, 시뮬/해석만, miss-correction 저감 가정은 강한 component code 전제, 4bit+ soft LDPC엔 열세.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Raptor(fountain) 외부부호 encode/decode | 대응 없음 | Prime ECC는 binary QC-LDPC 전용, fountain/erasure code 및 inactivation decoding 미보유 |
| BW-PC/BCH 내부부호 반복 hard-decision | 대응 없음 | product/BCH 연접 구조는 코드베이스 밖 |
| hard-decision sensing → erasure 변환 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()` | hard-decision 모델만 겹칠 뿐 erasure 선언 파이프라인은 없음 |
| 조기종료/실패판정 후 상위 정정 호출 | [full_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) | 내부 실패 판정 개념은 유사하나 외부 Raptor 정정과는 무관 |
| 비교 baseline QC-LDPC(min-sum) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 논문에선 우리 부호와 유사한 QC-LDPC를 비교대상으로만 사용, 기여는 Raptor 쪽 |

적용 가치: 낮음 — NAND 직접 타깃이나 부호 패밀리(Raptor+BCH/product)가 Prime ECC의 binary QC-LDPC와 근본적으로 달라 디코더/부호설계 어느 레이어에도 떼어 붙이기 어렵고, 성능도 2-bit soft LDPC와 대등한 수준에 그친다.
