### B. 알고리즘 요약

1. IEEE 802.3an 표준 (2048,1723) LDPC, rate 0.84, min-sum 기반 디코더의 면적(A)×지연(T) 복잡도 최적화가 목표.
2. 기존 두 아키텍처(bit-parallel: 저지연·대면적, bit-serial: 소면적·고지연) 중 어느 하나가 아니라 AT-complexity 관점의 최적점을 찾는 체계적 탐색이 부재했던 문제를 해결.
3. 채널/부호 모델은 표준 min-sum 근사 BP만 가정(변형 없음), 순수 아키텍처(데이터패스) 최적화 논문.
4. 핵심 기법: 디코딩 루프를 노드 연산(bit/check node arithmetic)과 노드 간 인터커넥트로 나누고, 각각을 bit-parallel/bit-serial로 조합하는 설계공간(tree diagram, Fig.7)을 전수 탐색.
5. 핵심 관찰: check node를 bit-serial, bit node를 bit-parallel로 구현한 조합(No.14)이 반복당 클록 사이클을 9 cycle로 최소화(`γ`, 식 2-1의 반복주기 결정 변수).
6. 확장: digit-serial 인터커넥트(2bit, 3bit word) 도입으로 bit-serial의 지연을 완화, logic/routing 면적이 균형점에 도달할 때까지 word length를 늘려 AT를 추가 절감(9→6→5 cycle).
7. bit node 아키텍처는 부분합(partial sum)을 클록마다 누적계산(Fig.9)하여 bit-parallel 구현 대비 노드 복잡도·critical path 축소.
8. 학습/최적화 절차 없음. 대신 45nm CMOS full-custom leaf cell 기반 파라미터화된 cost model(면적·타이밍)로 아키텍처를 비교/도출.
9. 결과: 기존 최저 AT-complexity 구현([11]) 대비 60% AT 절감, 면적효율(throughput/area) 2.5배 향상. 반복주기: bit-serial 인터커넥트 19ns, 2bit digit-serial 14.2ns, 3bit digit-serial 13.2ns. 면적 3.1/3.6/4.1 mm².
10. 한계: 정정성능(BER/error-floor) 개선 없음, 순수 HW 면적-지연 트레이드오프 연구. cost model이 특정 word length(6bit)·특정 코드(2048,1723)·45nm 공정에 종속되어 다른 조건(NAND 저rate, 작은 block)으로 일반화 검증은 없음. block interleaving 미적용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-serial/digit-serial 인터커넥트 아키텍처 | 디코더 HW 모델 (z=32, HCU, GT) | 우리는 병렬(z=32) 구조 고정, bit-serial/digit-serial 트레이드오프는 우리 아키텍처와 근본적으로 다른 방향 |
| min-sum CN 연산(부분합 기반 bit node) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum 계열 보유, 부분합 방식 자체는 이미 유사 구현 존재 |
| AT-complexity cost model / 설계공간 탐색 방법론 | 대응 없음 | 특정 알고리즘 모듈이 아닌 아키텍처 비교 방법론으로 코드 모듈 대응 없음 |

> 적용 가치: 낮음 — 우리 디코더는 이미 완전병렬(z=32) HW 구조가 고정되어 있어 bit-serial/digit-serial 트레이드오프 연구는 아키텍처 방향성이 근본적으로 다르고, 정정성능 기여도 없음.
