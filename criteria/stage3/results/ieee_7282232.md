### B. 알고리즘 요약

1. 시스템: 반도체 메모리(SRAM/DRAM) 방사선 soft error 보호용 ECC, EG-LDPC 예제 `(15,7)` 부호(7 정보 + 8 parity).
2. 문제: ML decodable 부호는 다중오류 검출은 좋으나 codeword 길이만큼 복호 cycle 소요(예: 21bit→21cycle) → 메모리 접근 지연.
3. 모델: 알파입자/우주선/boron fission 유발 soft error를 대상, 본문은 flash/FRAM/MRAM엔 soft error가 없다고 명시.
4. 핵심기법1: EG-LDPC(Euclidean Geometry over GF, `EG(m,2^s)`, m=2) one-step majority-logic decodable 순환부호, PCM 행/열 동수.
5. 핵심식: parity-check sum(수신벡터·PCM 행 내적)을 다수결게이트에 입력, 출력 ‘1’이면 해당 비트 반전 정정.
6. 핵심기법2: 처음 3 iteration만 xor matrix 평가 → 전부 ‘0’이면 error-free로 조기종료(대부분 워드가 무오류라 평균 복호시간 대폭 감소).
7. 구현디테일: serial one-step corrector(cyclic shift register + comparator), 고error rate 시 병렬 corrector(n copies) 사용.
8. 학습/최적화: 없음.
9. 결과: Xilinx/ModelSim VHDL 시뮬, 기존 MLD가 n-bit에 n-cycle인 반면 제안은 3 cycle에 검출, 5비트 초과 flip도 확률≈1로 검출.
10. 한계: 정정 시뮬 결과는 미제시(검출만 제시, 정정은 under process), 소형 부호, min-sum·QC-LDPC 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EG-LDPC 다수결논리(one-step MLD) 복호 | 대응 없음 | Prime ECC는 min-sum message-passing, 다수결논리 복호 미보유 |
| first-3-iteration 조기 error 검출 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | 조기종료 개념만 유사, 판정근거(다수결 vs CRC)가 완전 상이 |
| EG(m,2^s) 부호구조 로드 | 대응 없음 | 특정 유클리드기하 순환부호로 QC-LDPC base+shift 구조와 무관 |

적용 가치: **낮음** — SRAM/DRAM 방사선 soft error용 소형 EG-LDPC 다수결논리 복호로, NAND flash·min-sum·고rate binary QC-LDPC 어느 쪽과도 정합되지 않는다.
