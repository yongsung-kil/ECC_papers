### B. 알고리즘 요약

1. 시스템: ABL(all-bit-line) 4-level MLC NAND, i.i.d. 입력 pmf 0.25, SIQ 1.8 bits/cell ≈ 부호rate 0.9에 해당하는 동작점 비교.
2. 문제: 스케일링으로 ICI 심화·레벨간 마진 축소 → 기존 hard(post-comp/coupling canceller)·양자화 soft 검출기는 채널 양자화 없는 정확한 soft-output을 못 준다.
3. 채널모델: 1D causal 및 2D anticausal ICI 모델 `Y_k = X_k + ΣΓ(k)(Y_{k-ℓ}-E_{k-ℓ}) + W_k + U_k`, victim cell이 이후 프로그램된 3개 이웃에 영향받음.
4. 핵심기법1(MAP): 입력 xˆ를 trellis 상 Viterbi-like MAP로 검출, 상태 `S_k=X_{k-M+1}^k`, branch metric `Λ_MAP`.
5. 핵심식: branch metric의 조건부 pdf `f(y_k|x_k,y_{k-L}^{k-1})`는 해석적으로 어려워, 조건부 특성함수 `G_Yk`의 FFT로 pdf를 수치 계산(식 8).
6. 핵심기법2(GA-MAP): Γ(k)를 상수 근사한 Gaussian approximation으로 `Λ_MAP` 추정, FFT 없이 2개 FIR 필터(θ_k, ν_k)만으로 구현.
7. 구현 디테일: 정확 MAP는 2L개 FIR 필터 필요, GA-MAP은 곱셈기·덧셈기 등 DSP로 구현 가능해 HW 친화.
8. 결과: BER은 post-comp와 대등하나, soft output 품질(SIQ=I(X_k;O_k))에서 MAP가 기존 대비 0.23dB, GA-MAP 0.14dB 이득(SNR 가변, s=0.75).
9. 의미: SIQ는 random LDPC 부호의 달성 가능 최고 정보율 → 0.23dB 이득은 동일 LDPC로 0.23dB 약한 부호를 써도 됨을 시사.
10. 한계: LDPC 디코더 자체 미설계, HW 미구현, 시뮬만. ICI 채널모델·MAP trellis에 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft-output 검출기(확률벡터/SIQ) → LDPC LLR 입력 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 검출기 출력을 우리 soft-read LLR로 변환하는 앞단, 알고리즘 자체는 이식 대상 아님 |
| ICI/2D 채널모델·error injection | `channel.cpp` (AWGN/RBER/fixed) | 우리 채널모델과 다른 ICI trellis 모델, 대체·이식 부담 큼 |
| MAP/GA-MAP trellis 검출 | 대응 없음 | LDPC 복호(min-sum message passing)와 무관한 pre-decoder 검출기 |

적용 가치: **낮음** — LDPC 부호설계·디코더가 아닌 채널 검출기(pre-decoder)라 Prime ECC 디코더/인코더에 직접 이식할 요소가 없고, soft-read LLR 품질 개선 아이디어만 간접 참고 가능.
