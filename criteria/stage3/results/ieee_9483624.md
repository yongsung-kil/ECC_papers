### B. 알고리즘 요약

1. 시스템: SCC(staircase code) + BCH 컴포넌트 코드(예: BCH(256,239,2), rate `R=0.87`), M-PAM/AWGN, sliding window(`L=9`) 복호.
2. 문제: 표준 SCC의 iterative BDD는 miscorrection·stall로 이득 한계, SD 완전 대체는 복잡도 과다.
3. 기법 기반(SABM): 채널 LLR 절댓값 `|λ|`로 bit를 HRB(고신뢰)/HUB(고불신)로 마킹해 HD BDD를 보조(miscorrection 방지 + BF).
4. 핵심 개선(iSABM) ①: 정렬 제거 - 두 임계값 `δ1`,`δ2`로 HRB/UB/HUB 3분류(`δ1=10`, `δ2=2.5`), HW 정렬망 불필요.
5. 개선 ②: BDD 실패/miscorrection 시 HUB를 **랜덤 선택**해 flip → iteration마다 다른 시도로 탈출 확률↑.
6. 개선 ③: 마지막 2블록만이 아닌 window 내 마지막 `L-K` 블록으로 soft-aided 확대(`K=2`가 최적).
7. HW 고려: max-log LLR 근사, 마킹 정보 2-bit(HRB/HUB/UB) 정적 RAM 저장, 1-bit 균일 양자화도 가능.
8. miscorrection 검출: HRB 미flip + zero-syndrome codeword와 충돌 없음 두 조건.
9. 결과: 표준 SCC 대비 최대 0.91 dB, SABM 대비 최대 0.53 dB 이득(post-FEC BER 10⁻⁶); 1-bit 양자화 시 0.25 dB 손실로 마킹 메모리 50% 절감.
10. 한계: HW 미설계(시뮬만), gate/throughput 수치 없음, BCH-staircase 전용 알고리즘(LDPC 무관).
