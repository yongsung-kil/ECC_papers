### B. 알고리즘 요약

1. LDPC 부호(rate `R=k/n`, 예시로 (21,3,5) Tanner code) BPSK/AWGN 채널에서 BP(belief propagation) 복호를 다룸.
2. Flooding은 지연은 낮지만 성능이 열등하고, NW-RBP는 성능은 좋으나 residual 계산·정렬로 지연이 큼 — 이 트레이드오프를 동시에 해결하려는 문제.
3. 채널 모델은 BPSK 변조 + AWGN, residual은 갱신 전후 C2V 메시지 LLR 차이의 절댓값 `R(Ci→Vj)`로 정의.
4. 핵심 기법: 먼저 flooding으로 최대 반복까지 복호를 시도하고, 실패 시(`v·H≠0`)에만 NW-RBP를 이어서 수행하는 2단 순차(concatenated) 스케줄링.
5. V2C/C2V 갱신식은 표준 BP 식 `L(Ci→Vj)=2tanh^-1(∏ tanh(L(Vj→Ci)/2))`이며, NW-RBP는 residual queue `Q`에서 최대 residual을 가진 check node부터 갱신.
6. NW-RBP 실행은 flooding 실패 시에만 트리거되므로 residual 계산·정렬 빈도(접근 횟수)가 크게 줄어듦.
7. 별도 양자화/HW 변형은 없음 — flooding과 NW-RBP가 동일 factor graph 회로를 시분할로 재사용한다고 서술(HW 복잡도 증가 없음 주장, 설계는 없음).
8. 별도 학습/최적화 절차 없음(고정 스케줄 규칙).
9. SNR 2dB에서 평균 반복수: flooding 22.39, NW-RBP 12.38, concatenated 33.49; FER 10^-4에서 NW-RBP 대비 concatenated가 약 0.05dB SNR 이득; concatenated의 NW-RBP 접근 빈도는 SNR 2~3.5dB에서 15.4%→0.418%로 감소.
10. 한계: (21,3,5) Tanner code 단일 예제로만 검증, HW 구현/합성 없음, 대형 QC-LDPC나 NAND 채널 모델에서의 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 디코딩 스케줄링(flooding→NW-RBP 순차 결합) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 flooding이 아닌 min-sum 기반 Dual-Update(odd/even 교대) 스케줄을 사용해 스케줄 자체가 상이, 순차결합 개념만 참고 가능 |
| residual 계산/정렬(NW-RBP) | 대응 없음 | Prime ECC에 residual queue 기반 동적 스케줄링 구조 없음 |
| 조기종료(v·H=0 판정) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념적으로 유사(정지 조건) 하나 논문은 syndrome check, Prime ECC는 CRC 기반으로 메커니즘 다름 |

> 적용 가치: 낮음 — Sum-Product 기반 flooding/NW-RBP 스케줄링이며 Prime ECC의 min-sum·Dual-Update·Partial-CRC 조기종료 구조와 메커니즘이 달라 직접 이식 어렵고, 검증도 소형 코드 시뮬레이션에 그침.
