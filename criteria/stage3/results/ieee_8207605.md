### B. 알고리즘 요약

1. 대상: graph-기반 binary LDPC(랜덤 MacKay/PEG 및 QC-LDPC), rate 0.5~0.9, `N`=96~4032, ZM/CPM(`L`=27~330) 구조.
2. 문제: PEG 등 그리디 구성은 국소 girth만 최적화 → 전역 Tanner graph는 준최적, 고rate에서 짧은 사이클 다수 발생 → error floor.
3. 핵심 지표 CM: 각 edge가 낀 길이 `2l` 사이클 수 `N^{2l}`에 가중 `β^{l-2}`(β=0.01) 합산, `M_T = Σ M_{v_j}`; CM↓이면 사이클 수↓/길이↑ → floor 개선.
4. PMP: MPA 변형으로 각 edge에 고유 벡터 `X_k(J,I)=[J I]` 초기화, C2V/V2C에서 서로 다른 벡터 개수만 기록해 사이클 수를 병렬 계산(복잡도 `O(l·Ne^2)`).
5. 식 의미: edge `E_{j,i1}`의 초기벡터가 l회 후 `E_{j,i2}`의 C2V 메시지에 나타난 횟수 = 두 edge 공통 사이클 수 → 노드별 특정 없이 사이클 수 산출.
6. PMPE(랜덤 코드): CM 최대 edge들을 골라, UEP/IEP가 아닌 두 edge의 연결 CN을 교환(edge exchange)해 `M_T`가 줄면 수락, 줄지 않으면 undo — degree 분포 불변.
7. QC-PMP(QC 코드): base matrix `H_nb` 원소를 1~L 범위로 순환 조정(0으로는 안 바꿈 → degree 불변)하며 CPM 매핑 후 `M_T(H_bqc)` 감소 방향으로 최적화; 802.11ad는 BDD 유지 위해 일부 열 고정.
8. 검증(간접): bit-reversing 기반 근사 최근접 codeword(ANC) 탐색으로 저Hamming-weight codeword 수 감소를 확인 → 최소거리 개선.
9. 결과: PEG rate~0.9 (440,396)에서 0.5dB@BER1e-7; QC 예로 (3960,2640) floor 낮춤, (4032,3264) 0.15dB@8e-8, (2304,1920) 802.16e 대비 0.2dB@2e-7, 802.11ad (648,486) 0.2dB@3e-8 등 대부분 floor 영역 이득.
10. 한계: SPA/AWGN 시뮬만, HW(인/디코더) 설계 없음, NAND 채널 미검증, throughput/gate 정보 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC base matrix 원소 조정(QC-PMP) | ecc_top.cpp Load_PCM() | 우리 H-matrix를 offline로 재최적화해 로드 — 재검증 부담 큼 |
| PCM/부호구조 정의 | ecc_data.h PCM_b (base+shift) | base+circulant shift 표현과 직접 대응, shift 값 조정 대상 |
| 인코딩용 부호 PCM(구조 보존) | encoder.cpp Generate_PCM_encoding() | dual-diagonal/BDD 유지 제약과 정합 필요 |
| CM/사이클 카운트(PMP) | 대응 없음 | offline 설계 도구, 런타임 디코더에 없음 |
| SPA 디코딩(성능 평가) | decoder.cpp LDPC_Decoder() | 우리는 min-sum — 평가 알고리즘 상이, floor 이득 재검증 필요 |
| error-floor/trapping-set 개선 | 대응 없음 | 부호설계 단계 개선, 디코더 post-processing 아님 |
```

적용 가치: 중간 — code-design 단계 offline 최적화라 min-sum/HW와 무관하게 우리 QC base matrix에 적용해 error-floor를 낮출 잠재력은 있으나, 특정 QC-LDPC 고정·dual-diagonal/z=32 제약·min-sum 하 재검증 부담으로 즉시 이식은 어려움.
