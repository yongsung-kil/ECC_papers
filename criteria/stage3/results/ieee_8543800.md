### B. 알고리즘 요약

1. 무선 SDR 베이스밴드용 Polar/LDPC/Turbo/CC 4모드 통합 FEC ASIP(QFEC ASIP)로, 기존 3모드 TASIP에 Polar 복호를 추가한 확장 설계다.
2. 문제: 표준마다 다른 FEC를 단일모드 ASIC 4개로 모으면 면적·개발비가 과대 - 하드웨어 공유형 프로그래머블 프로세서로 해결.
3. 통합 알고리즘 프레임워크(Algorithm 1): 4모드 모두 양방향 재귀(bi-directional recursion) 구조로 환원 - Polar는 LP/RP, LDPC/Turbo는 BCJR, CC는 Viterbi ACS.
4. LDPC 모드는 TASIP의 `TDMP`(turbo-decoding message passing)+BCJR trellis forward-backward recursion 커널을 그대로 사용(WiMAX 대상). min-sum QC-LDPC와 다른 계열.
5. Polar는 scaled min-sum 근사 BP: `f(x,y)=0.9375*sign(x)sign(y)min(|x|,|y|)`, 0.9375 곱은 `x-(x>>4)`로 구현.
6. 데이터 정밀도: 8bit 고정소수(정수 5bit + 소수 3bit)로 부동소수 대비 성능 근접, N=4096에서 0.34dB 손실@BER 1e-5.
7. SIMD 병렬: 16 PSISO x 각 32 BCU 병렬(Polar). LDPC/Turbo/CC용 TSISO 12개는 별도(8 서브레이어/8 state 병렬).
8. 핵심 기여인 quad-mode 충돌없는 글로벌 메모리(GMS): Polar용 8x(128bit x 32) 뱅크를 4모드가 공유, LDPC/Turbo/CC 데이터 재배치로 재활용.
9. 결과: 65nm 합성 4.26mm2, 344MHz, throughput Polar 1345 / LDPC(WiMAX) 917 / Turbo 320 / CC 387 Mbps. 메모리 뱅크 65.2%, 비트 48.9%, 면적 29.7% 절감.
10. 한계: LDPC는 부수 모드로 정정능력·error-floor 개선 기여 없음, NAND read 비용/soft-read 논의 없음, 무선표준(WiMAX/5G) 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC TDMP/BCJR trellis 복호 커널 | 대응 없음 | 우리는 min-sum 반복 message passing, trellis/BCJR 커널 없음 |
| Polar scaled min-sum BP `f(x,y)` | `decoder.cpp` `CNU_Update_New_Mag()` | min 근사라는 점만 유사, 대상 부호(Polar)가 달라 직접 이식 불가 |
| quad-mode 공유 글로벌 메모리(GMS) | 대응 없음 | 멀티모드 메모리 공유는 단일 QC-LDPC 엔진에 무의미 |
| 8bit 고정소수 양자화 | `channel.cpp` + LLR 테이블(`PARAM_LLR`) | 양자화 개념만 참고, Polar 특화 값이라 재사용 가치 낮음 |

적용 가치: **낮음** - LDPC가 WiMAX용 TDMP trellis 부수 모드에 불과하고, 핵심 기여(멀티모드 통합/메모리 공유)가 단일 NAND binary QC-LDPC 엔진과 무관하다.
