### B. 알고리즘 요약

1. 대상 코드는 BCH(63,45), BCH(127,106) (rate 0.714, 0.835), AWGN 채널·BPSK 변조, LLR `si = 2ri/η²`.
2. 기존 BP 기반 GNN 디코더는 반복마다 그래프 전체를 처리해 계산복잡도가 노드 수에 선형으로 증가하는 문제.
3. PCM을 인접행렬로 변환한 이분 그래프(variable/check node)에서 GNN 메시지 패싱으로 BP를 모사.
4. 핵심 기법: Pick-Prune-Link(PPL) Top-k 풀링. 노드 차수 기반 점수로 top-`k`개 노드 선택(`k = ⌈ρ·|V|⌉`), 이후 pruning과 재연결로 서브그래프 `G' = (V, C, E')` 구성.
5. 풀링된 특징행렬 `F_pool`, 인접행렬 `A_pool`을 재희소화하여 후속 GNN 레이어 계산량 축소.
6. Dynamic MLP 레이어(`F_i(X) = σ_i(W_iX + b_i)`)가 풀링에 따른 입력 크기 변화에 자동 적응, 서로 다른 코드율에도 재구성 불필요.
7. 노드 특징에 균등분포 노이즈 `ε~U(0,α)`를 추가해 다양한 채널 조건에 대한 강건성 확보(정규화 트릭).
8. 학습: Binary Cross-Entropy 손실(반복별 LLR 추정치 기준), Adam 옵티마이저, SNR 2~9dB, GNN 8회 반복, 임베딩 20차원, hidden unit 40.
9. 결과: 고 SNR(6~9dB) 구간에서 풀링 미적용 GNN 대비 BLER/BER 유사 성능 유지하면서 디코딩 시간 30~35% 단축.
10. 한계: HW 구현 없이 GPU 시뮬레이션(TensorFlow)만 수행, 저 SNR에서는 풀링 미적용 대비 성능 열화, BCH 전용이며 LDPC 확장은 향후 과제로 언급됨.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GNN 기반 BP 디코딩 전체 | 대응 없음 | NN 전면 대체 디코더로 Prime ECC의 min-sum 기반 구조와 무관 |
| BCH 코드 대상 | 대응 없음 | Prime ECC는 binary QC-LDPC 전용, BCH 코드 미지원 |
| PPL Top-k 풀링(그래프 축소) | 대응 없음 | 그래프 뉴럴넷 특화 기법으로 HW min-sum 디코더 구조에 대응 불가 |
```

적용 가치: **낮음** — BCH 코드 대상 GNN 전면 대체 디코더로 NAND binary QC-LDPC ECC와 부호 종류·디코딩 패러다임 모두 상이하며, 저자 스스로도 LDPC 확장을 향후 연구로만 제시함.
