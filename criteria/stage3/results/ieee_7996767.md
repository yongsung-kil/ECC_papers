### B. 알고리즘 요약

1. 시스템: 단일 FPGA에서 30Gb/s~1Tb/s gross rate를 지원하는 configurable HD-FEC(hard-decision) 유닛, 광통신 대상.
2. 문제: 1Tb/s급 광링크에서 soft-decision LDPC는 FPGA transceiver 수·복잡도 제약으로 목표 rate 달성 곤란 → hard-decision 선택.
3. 부호: expurgated primitive `BCH` 성분부호(최대 `n=1023`, `t=3` 정정, 최소거리 6, `n-k=32`)를 row/column 양방향에 적용한 quadratic product code(frame), overhead 6.5~13%.
4. 복호: syndrome `s_i=r(α^i)` 계산 후 error-locator 다항식을 lookup table로 풀어 근을 찾는 Gravano/Okano-Imai BCH 디코더, double parity check로 오복호 확률 저감.
5. 반복복호: rows→columns→rows... 를 한 iteration으로 정의, 성분부호 t=3이면 이론상 pre-FEC BER ~4.5e-3(2.44 errors/row) 미만에서 성공, NCG ~9.4dB(BSC 한계보다 0.4dB 낮음).
6. HW: 여러 Gravano decoder를 병렬로 두고 correction을 FIFO로 처리, syndrome은 한 번만 계산 후 정정 시 갱신.
7. RAM: 프레임을 tile 단위로 저장하고 tile 열 수만큼 slice로 분할해 병렬화, 실병목은 slice간 correction 분배.
8. 확장성: input width는 2의 거듭제곱, clock 250MHz 목표, Gravano decoder 수를 늘려 고rate에서도 iteration 확보(최소 4 iteration 필요).
9. 결과: Arria10에서 32 병렬 frame decoder로 gross 1Tb/s(net 883Gb/s) 달성, latency는 µs 범위(고rate에서 <0.5~2µs), 10Gb/s 랩실험으로 post-FEC vs pre-FEC BER 검증.
10. 한계: LDPC 아닌 BCH product code, hard-decision 전용, error floor는 3x3 core로 결정(~1e-15 목표), 광통신 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BCH product code 반복복호 | 대응 없음 (Prime ECC는 LDPC min-sum) | 부호·복호 알고리즘 상이 |
| Gravano BCH syndrome 디코더 | 대응 없음 | LDPC CN/VN 연산과 무관 |
| tile/slice RAM 분할·라우팅 | `decoder.cpp` HW 모델(z=32) 개념적 유사만 | 구조 다르고 직접 이식 불가 |
| iterative row/column 스케줄 | 대응 없음 | product code 전용, LDPC 이터레이션과 무관 |

적용 가치: 낮음 — 광통신용 BCH product code hard-decision FEC로, binary QC-LDPC min-sum 기반 Prime ECC와 부호종류·복호 알고리즘이 근본적으로 달라 이식할 요소가 없다.
