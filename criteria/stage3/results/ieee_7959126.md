### B. 알고리즘 요약

1. 시스템: 2-bit/cell MLC NAND, rate-0.90 (8000,7200) column-weight-5 regular LDPC (PEG 구성), shuffled BP, round당 최대 25 iteration.
2. 문제: retention time 경과로 셀 전압 하강 → 초기 전압분포·read threshold가 outdated → LLR 부정확 → BP 실패. 재추정·재복호에 추가 memory sensing이 latency·전력 병목.
3. 모델: `V = Vs + (nu+np+nw+nr+ni)` 5-noise NAND 채널(ISPP, programming, RTN, retention, CCI), Gaussian mixture 근사, retention `μrs=(Vs-x0)·(...)·log(1+T)`.
4. 핵심기법 RABP: 1차 BP로 hard-decision `{ĉmsb,ĉlsb}` 획득 → 실패 시 read-back 전압 v와 복호비트로 charge-loss victim 셀 검출 → LLR re-mapping 후 2차 BP 재복호.
5. LLR 재매핑: erasure 영역(E1~E3) 셀은 `L̂(n)←ζn·L(n+1)` (retention-erasure correction), data 영역이나 상위 state로 복호된 셀은 `L̂(n)←ζn·L(n+2)` (retention-error correction).
6. 양자화: MMI 기반 6-level(Q=6) non-uniform 양자화(4 data D1~D4 + 3 erasure E1~E3)가 full-precision soft BP 근접, LLR은 read threshold Ri와 전압분포로 산출.
7. 채널갱신 RABP-CU: 복호 전후 error/erasure count 비교 → 측정·예측 count의 MSE 비용함수 `C(μ,σ)`를 gradient descent로 최소화 → best-fit μ,σ 재추정 (추가 read 불필요).
8. 실패 시(RABP-CUF): 측정 count 없으면 검출 셀 count `ZDi,ZEj`로 error/erasure count 근사해 CU 수행.
9. 결과: RABP가 CBP 대비 retention time 한계 최대 70%(FER 1e-4 기준 60%) 연장, FER 최대 3자리수 개선. RABP-CU는 retention 한계 최대 41×(6K PE) 연장, GD 복잡도 DPE 대비 낮음(34 ADD/165 MUL/33 LUT per iter), 추정오차 <0.02V.
10. 한계: 모델기반 2-bit 시뮬만(실측 없음), 2차 복호로 latency↑·2N 추가 저장, ζ,η 가중치 미최적화, HW 미설계.
