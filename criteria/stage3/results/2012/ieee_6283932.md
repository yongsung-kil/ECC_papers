### B. 알고리즘 요약

1. 대상: BEC(소거채널) 상 LDPC 앙상블 `(λ(x),ρ(x))`의 유한길이 TEP(tree-expectation propagation) 디코더 성능 분석.
2. 문제: TEP가 short LDPC에서 BP를 개선함은 [2]에서 경험적으로만 보였음 → 어느 코드길이 범위에서 개선되는지 이론적 예측 필요.
3. TEP 디코더: Tanner graph peeling으로 degree-1 check(변수 확정) + degree-2 check(그래프 축소, `V1`이 `V2`의 연결 상속) 처리 = 최대 2개 1을 가진 row의 가우스소거, 복잡도 `O(n)` (BP와 동차수).
4. 기대 그래프 진화: 좌/우 차수 edge 비율 `l_i(τ)`,`r_j(τ)`를 미분방정식으로 추적, 정규화 시간 `τ=ℓ/E`, 잔여그래프 크기 `e(τ)`.
5. TEP가 BP보다 나으려면 degree-1 check를 많이 생성해야 함(그림2(a) 시나리오), 확률 `P_S(τ) ≥ (l_avg-1)²(r_avg-1)/(e(τ)E)`, `n→∞`이면 `→0`이라 극한성능은 BP와 동일.
6. Critical point: `lim_{ε→ε_TEP} r1(τ)=0`인 `τ`, Lemma 1로 TEP critical point 수 = BP critical point 수.
7. 유한길이 효과: `r1(τ,n,ε_TEP) ≈ γ_TEP·n⁻¹` (예: (3,6) `γ_TEP⁻¹=0.3194`, 식(8) irregular `0.2925`), 길이 `n_th` 이상이면 0으로 수렴.
8. Scaling law: waterfall 영역 WER을 `Q((√n(ε_TEP-ε)+γ_TEP/(n·δ))/α_TEP)` 형태로 예측(식14), `α_TEP≈α_BP`, 분산은 BP critical point 값으로 근사(상한).
9. 결과: (3,6) 및 irregular 부호 `n=2^10~2^12`에서 SL 예측(식14)이 시뮬 WER과 잘 일치, TEP가 BP 대비 waterfall 개선 확인. Raptor ML 성능 추정에도 적용 가능.
10. 한계: BEC 전용·이론만(HW 없음), 다중 critical point·일반 binary memoryless 채널 확장은 open problem.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TEP peeling/가우스소거 디코더 | 대응 없음 | BEC 전용, min-sum iterative(decoder.cpp LDPC_Decoder())와 계열 다름 |
| 유한길이 scaling law (waterfall 예측) | 대응 없음 | 성능예측 이론 도구일 뿐 코드 구현 요소 아님 |
| degree-1/2 check node 처리 | 대응 없음 | soft-read LLR/CNU(CNU_Update_New_Mag())와 무관한 소거 전용 연산 |
```

적용 가치: **낮음** — BEC 소거채널 전용 TEP 디코더의 유한길이 이론 분석으로, NAND soft-read min-sum 파이프라인에 이식할 요소가 없다.
