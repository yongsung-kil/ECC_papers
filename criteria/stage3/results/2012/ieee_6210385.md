### B. 알고리즘 요약

1. 대상은 BSC(bit-flipping) 채널의 binary linear/LDPC 부호이며, LP 복호(Feldman et al.)의 marginal polytope 완화를 기반으로 한다.
2. 문제: 표준 LP 복호가 실패하면 fractional pseudocodeword를 내는데, 이 정보를 버려 성능 손실이 발생한다.
3. 핵심 개념 `FCP`(fundamental cone property): 복호 실패 시에도 pseudocodeword가 실제 codeword에서 얼마나 벗어났는지 bound(robustness) 하는 성질.
4. 채널 likelihood와 pseudocodeword 좌표 차이 `|w - y|`로 각 bit의 불신뢰도(disagreement)를 측정한다.
5. 이 불신뢰도로 목적함수를 재가중: 신뢰 bit는 변경 비용↑, 불신뢰 bit는 비용↓(음수 가중 `-a1,-a2`)하여 2차 LP를 재실행.
6. `Algorithm 1`: 1차 LP → 정수해면 종료, 아니면 상위 `k`개 편차 bit 집합 선택 → 가중 LP 재실행(총 2배 복잡도).
7. HER(high error rate) subset 정리(Thm 6.2): pseudocodeword 편차 상위 집합에 실제 오류가 상수 비율 이상 몰려 있음을 증명.
8. Thm 7.1: expander code / high-girth code에 대해 재가중 방식의 strong/weak recovery threshold가 기존 LP의 provable robust threshold보다 엄격히 높음을 증명.
9. 결과: 랜덤 LDPC(3,4) `n=1000`, `R=1/4` 시뮬에서 p 작을수록 BER 개선, p=일부 구간에서 최소 1 order magnitude 개선(mixed-integer/facet-guessing 대비 우수, 추가 iteration 1회).
10. 한계: HW 미설계, BSC 전제, cvx MATLAB 시뮬만, true threshold 개선 여부는 미증명(bound 개선일 뿐).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP 복호 + 재가중 postprocessing | 대응 없음 | Prime ECC는 min-sum message passing, LP/cvx 계열 미보유 |
| pseudocodeword 불신뢰도 기반 LLR 편향 | decoder.cpp Get_VNU_Table_Idx() (개념적 유사) | 적응형 LLR 테이블과 발상만 유사, 알고리즘 이식은 불가 |
| BSC bit-flip 채널 | channel.cpp Set_LLR_Th() | hard-1bit read와 대응하나 논문은 AWGN/soft 미지원 |
```

적용 가치: **낮음** — LP 기반 복호는 Prime ECC의 min-sum HW 파이프라인과 계열이 완전히 달라 떼어 쓸 수 없고, 기여도 provable threshold bound 개선에 국한된다.
