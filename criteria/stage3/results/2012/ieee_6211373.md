### B. 알고리즘 요약

1. 대상은 short/medium block length의 regular/irregular LDPC 부호, BPSK/AWGN, BP(LLR) 복호 max 50 iter.
2. 문제: 기존 PEG 변형(ACE[10], generalized-ACE[8], ACSE[11])은 error floor를 `ACE`/`ACSE` 같은 간접 지표로만 다뤄 dominant trapping set을 직접 제어하지 못함.
3. 핵심 지표: `(a,b)` elementary trapping set(ETS, 모든 check가 degree 1 또는 2) 중 a·b가 작은 것을 dominant로 정의, depth-d `ETS spectrum` `(ξ_{g/2},...,ξ_d)`를 최대화 목표.
4. 핵심 기법: 원래 PEG로 변수노드 `v_i`의 edge 후보를 만들되, `Algorithm A`로 `v_i`를 포함하는 dominant ETS를 재귀 탐색해 target spectrum을 만족하는 edge만 채택.
5. `Algorithm A`는 [3](regular)/[4](irregular)의 dominant-ETS 탐색을 `v_i` 포함 ETS만 찾도록 단순화(정확도 유지).
6. 복잡도 저감 수정: (i) 초기 집합을 `v_i`를 포함하는 short cycle로 한정, `l_max`로 긴 cycle 보완, (ii) low-degree VN 추가(Step4) 제거 후 Step7,8로 2-chain 처리 보완.
7. 파라미터 `l_max, β, T_ACE, d`로 정확도-복잡도 균형 조절.
8. absorbing set 등 다른 그래프 구조도 동일 프레임으로 target 가능(탐색 알고리즘 tailoring).
9. 결과: rate-1/2 (1008,504)·(504,252)·(2016) 예제에서 dominant (a,b) ETS 다중도가 원본/generalized-ACE-PEG 대비 크게 감소, error floor 개선(예: n=2016, 2.4dB에서 BER 6e-9 vs 1.46e-8).
10. 한계: HW 미설계, 부호구성(오프라인) 알고리즘 시뮬만, QC-LDPC/circulant 제약 미고려.

```
| dominant ETS/error-floor 개선 목표 | 대응 없음 | 우리 코드는 부호를 재설계하지 않음(디코더 side), error-floor 보증 구조 미보유 |
| trapping set 회피 부호구성 | encoder.cpp Generate_PCM_encoding() (개념적) | 부호 PCM 생성부와 닿으나 PEG는 QC lifting 방식과 상이 |
```

적용 가치: **중간** — error-floor를 직접 낮추는 부호설계 발상은 NAND 고rate ECC에 가치가 있으나, Prime ECC의 QC-LDPC circulant·고정 H-matrix 구조에 PEG 구성을 이식하려면 lifting 결합과 전면 재검증이 필요하다.
