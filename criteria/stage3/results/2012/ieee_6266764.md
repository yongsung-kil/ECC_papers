### B. 알고리즘 요약

1. 시스템은 AWGN 채널 위의 protograph 기반 LDPC 부호로, HARQ/적응형 코딩 응용을 위해 rate 0.32~0.9 범위를 커버하는 nested/rate-compatible 부호 family 설계가 목표.
2. 기존 puncturing 기반 rate-compatible 방법은 고rate로 갈수록 mother code보다 capacity gap이 커지는 문제가 있어, 이를 대체할 code-extension 기반 설계를 제안.
3. protograph는 작은 protomatrix(변수/체크노드, 병렬엣지 수)로 표현되는 Tanner graph이며, "copy-and-permute" 연산으로 실제 LDPC 부호를 생성.
4. 핵심 기법: PEXIT 알고리즘으로 protomatrix의 iterative decoding threshold를 계산하고, 좋은 threshold를 위한 경험적 제약(degree-1 accumulator, 단일 degree-2 노드 허용, 나머지 degree≥3)과 linear minimum-distance growth 조건(degree-2 노드 수 < 체크노드 수, degree-2 노드 간 cycle 금지)을 만족하는 protomatrix를 systematic search로 탐색.
5. 고rate 부호는 저rate protograph `H1/2`를 확장행렬 `He`로 lengthening(`H_high = [H1 He]`)해 만들며, 완전한 rate-compatibility를 위해서는 열/행을 동수로 추가하는 code extension(`Hn+1/n+2` 구조)으로 정보블록 크기를 고정.
6. error-floor 방지를 위해 protograph 앙상블의 (a,b) trapping-set 크기가 블록길이에 선형으로 증가함을 weight enumerator 상한(Theorem 1, Lemma 1)으로 증명.
7. 구현 디테일: protomatrix를 PEG(progressive edge growth) 알고리즘으로 두 단계 lifting(1차 factor 4로 병렬엣지 제거, 2차로 circulant permutation 결정)해 최종 QC 유사 구조 생성.
8. 최적화 절차는 제약 하 discrete search(최대 병렬엣지 수 제한 등)로 축소된 설계공간에서 최저 threshold protomatrix 선택.
9. 결과: rate>2/3에서 capacity gap 0.09dB 이내, rate-2/3는 0.122dB; rate-1/2 신규 코드가 AR4JA 대비 약 0.2dB 개선, FER 10^-6까지 error floor 미관측(정보블록 16k, max iteration 200).
10. 한계: HW/게이트카운트/throughput 설계 전무, 통신(HARQ, deep-space, AWGN) 표준용 다양한 rate 부호설계이며 NAND용 고정-rate 고rate QC-LDPC의 실제 이식엔 H-matrix 재설계 부담이 큼.
