### B. 알고리즘 요약 (10줄 내외)

1. 시스템은 AWGN 채널 위의 regular `(dv=5, dc=13)` LDPC 코드, `N=1008`, `K=504`, rate `0.5`.
2. 표준 Min-Sum(MS)은 BP 대비 저복잡도지만 BER 성능이 열화되는 문제를 다루며, MS의 check-to-bit(C2V) 갱신 과정에 반복 간 부호(sign) 급변을 억제하는 보정 팩터를 추가해 성능을 개선하고자 함.
3. 핵심 가정: 반복이 진행될수록 추정 코드워드가 수렴하며 부호 변화(sign flip) 확률이 감소해야 하는데, 표준 MS는 이를 명시적으로 반영하지 않는다는 점을 문제 삼음.
4. 핵심 기법 1단은 DSR(Difference-to-Sum-Ratio) 팩터 `φ ∈ [0,1]`을 도입: 현재·직전 반복의 추정 코드워드 값의 차이와 합의 비율로 계산하고, 이를 C2V 갱신식에 곱하는 형태로 표준 MS 업데이트를 수정(`φ=1`이면 표준 MS와 동일).
5. DSR 팩터는 스칼라가 아니라 parity-check 행렬 차원과 같은 벡터로 매 반복 갱신되어, 반복이 진행될수록(부호 변화 확률이 낮아질수록) 연산량이 자연히 줄어드는 방식으로 설계됨.
6. 핵심 기법 2단(BFMS)은 DSR-MS 반복이 끝난 후 syndrome `S = xH^T`을 계산해 0이 아니면 Bit-Flipping 하드 디시전을 추가 적용: 만족된 check node에 연결된 variable node(valid variable node)를 찾고, 거기 속하지 않는 비트를 flip.
7. 부수 트릭: DSR 팩터로 인한 추가 연산(곱셈)은 반복 간 부호 변화가 발생하는 경우에만 필요해 후반 반복일수록 오버헤드가 줄어든다고 주장(정성적 서술, 수치 없음).
8. 학습/최적화 절차 없음 - 고정된 DSR 계산식을 매 반복 재계산하는 방식.
9. 결과: `(1008,504)`, iteration 10/20에서 DSR-MS와 DSR-BFMS 모두 표준 MS 대비 BER이 낮아짐(SNR -1~5dB 구간), 반복 20회일 때 개선폭이 반복 10회보다 넓은 SNR 범위에서 나타남. 복잡도는 DSR-MS는 부호변화 시에만 곱셈 1회 추가, DSR-BFMS는 이보다 더 증가.
10. 한계: HW 미설계·시뮬레이션(BER 곡선)만 존재, 검증 코드가 규칙적(regular) 짧은 코드 1종뿐이며 irregular/non-binary/고rate에 대한 검증은 "향후 과제"로 명시. 정확한 게이트카운트·throughput·정량적 복잡도 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DSR factor를 곱한 C2V(check-to-bit) 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 표준 min-sum CN 갱신식에 스케일 팩터를 곱하는 형태로, 기존 min-sum 커널에 추가 곱셈기 삽입 위치에 대응 |
| 반복 후 Bit-Flipping 하드 디시전 후처리 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | soft(min-sum) 디코딩 후 hard bit-flip 후처리를 추가하는 구조로, 이터레이션 루프 말미에 post-processing 스테이지 삽입 개념과 대응 |
| Syndrome 검사 기반 valid variable node 판별 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 직접적인 CRC 조기종료는 아니지만 syndrome(parity check) 계산으로 반복 종료/보정 여부를 판단하는 점은 유사 |

**적용 가치**: `중간` - min-sum CN 갱신에 스칼라(벡터) 곱셈 팩터를 추가하는 정도로 디코더 루프 이식은 비교적 쉬우나(§4 "디코더 알고리즘" 낮은 이식난이도에 해당), 검증이 짧은 regular 코드 1종·시뮬레이션 수준에 그쳐 Prime ECC의 고rate QC-LDPC(z=32, HCU 구조)에서 실제 이득이 재현될지는 추가 검증 필요.
