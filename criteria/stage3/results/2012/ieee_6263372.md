### B. 알고리즘 요약

1. 시스템은 warm shallow water acoustic(WSWA) 채널의 LDPC-coded OFDM으로, 부호는 `1000×2000` random PCM(`R=0.5`), BPSK, 최대 iteration `L=30`.
2. 문제: snapping shrimp 임펄스 노이즈는 넓은 대역에 걸쳐 여러 OFDM 서브캐리어를 동시에 오염 → 노이즈 성분이 **종속(dependent)**이고 heavy-tail SαS 분포라 기존 LDPC soft decoder 성능이 급락.
3. 모델: 노이즈 벡터를 등방성 SαS로 두고, dependency를 쓰려면 multivariate PDF marginal이 필요한데 binary code에서 복잡도 `O(2^d N)`(d=종속성분 수)로 지수 폭발.
4. 핵심 기법: sub-Gaussian 표현 `X = A^(1/2) G`로 SαS 벡터를 스칼라 `A`(우편향 α/2-stable)와 독립 Gaussian 벡터 `G`로 분해 → 성분이 조건부 독립이 되어 marginal 불필요.
5. 핵심 식: `w_n^G = (1/A^(1/α))(r_n − t_n)` = fading 채널의 `w = r − s·t`와 동형 → fading 추정 알고리즘으로 `A`를 추정.
6. 확장: Tanner graph에 a posteriori probability 노드 `P_j`를 붙여 같은 노이즈 벡터에 영향받는 symbol 노드끼리 `λ_n(A_i)`/`π_n(A_i)` 메시지 교환으로 symbol posterior `p_n(x)`를 매 iteration 갱신.
7. 구현 디테일: 연속분포 `A`를 이산화(`A_d`)하되 원점 근처는 조밀(δ1), 먼 곳은 성긴(δ2) 세그먼트로 이중 양자화, `A_max` 상한.
8. 학습절차 없음 - a priori `π(A_i)`를 CDF 근사로 사전 생성.
9. 결과: dependency 무시 대비 큰 BER 개선(성분 수 10~50에서 gain 증가), δ2로 `N_A`를 320→12까지 줄여도 성능 유지, marginal 방식은 d 큰 경우 시뮬 불가한데 본 방식은 `O(N_iter·N_A·N)`로 d와 무관.
10. 한계: marginal 대비 sub-optimal, HW 미설계·시뮬만, SαS 임펄스 채널 전용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| symbol a posteriori / soft-input LLR 계산 | channel.cpp Set_R_Offset(), Set_LLR_Th() | LLR 생성 위치이나 SαS 노이즈 모델은 우리 채널(Gaussian/RBER)과 불일치 |
| sum-product 이터레이션 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | 논문은 full sum-product 확장 - Prime ECC min-sum과 다름 |
| A 이산화 / 노이즈 dependency 처리 | 대응 없음 | NAND read 채널에 SαS·성분 dependency 개념 부재 |
```

적용 가치: **낮음** - 수중음향 임펄스 SαS 채널과 성분간 dependency라는 매우 특수한 전제 위에 세워진 채널모델 확장 디코더로, NAND의 Gaussian/RBER read 채널에는 전제가 성립하지 않아 이식 여지가 거의 없음.
