### B. 알고리즘 요약

1. 대상은 `(dl, dr, L)` protograph SC-LDPC(band base matrix, lifting `M`), BEC에서 BP 복호; rate `R = (k-1)/k - (dl-1)/(kL)`.
2. 문제: SC-LDPC는 종단 때문에 rate-loss가 발생하고, 종단 단계가 dense `Hterm` 역행렬 계산으로 `O(M^2)` 비용이 들어 sequential encoding/decoding보다 비싸다.
3. 인코딩 구조: sequential encoding(`O(LM)`)으로 section별 parity `xki`를 syndrome `si`와 permutation `Pi,j`로 결정, 마지막 `Nterm` parity는 종단으로 결정.
4. 핵심 기법: base matrix `H`에서 아래쪽 `dl-2`개 row(및 대응 check node)를 제거한 수정 부호 `H~(dl,dr,L)` 정의.
5. 효과: rate-loss가 `(dl-1)/(kL)` → `1/(kL)`로 감소(=(dl-1)배 완화), coding rate `R~ = (k-1)/k - 1/(kL)`.
6. 종단 축소: 수정 부호는 종단 parity 수 `N~term = 2`로 `dl,dr` 무관하게 항상 2비트만 결정하면 됨.
7. 종단용 `H~term`을 `[[I,I],[I,I]]` 형태로 약간 수정(full-rank 확보, M 큰 경우 성능 영향 무시가능)해 2M accumulation으로 풀이.
8. 결과 비용: 종단 `xkL-1,i = xkL,i-1 + sL,i`, `xkL,i = xkL-1,i + sL+1,i` 순차 계산으로 `O(M)` (RA 부호처럼 accumulator).
9. 성능: Table I에서 `(3,6,L)`, `(4,8,L)`, `(3,9,L)`, `(4,12,L)` 여러 L에 대해 수정 부호가 더 높은 rate이면서 BP threshold는 5자리 정밀도까지 동일; 다만 수렴은 다소 느려짐(Fig.5).
10. 한계: BEC BP threshold 이론 분석뿐, HW 미설계, 유한길이/BMS·AWGN 성능은 future work.

| 수정 base matrix / 부호구조 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 block QC-LDPC H-matrix 고정, coupling/band 구조 미보유 |
