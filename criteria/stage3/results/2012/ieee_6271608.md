### B. 알고리즘 요약

1. 대상: 10GBASE-T Ethernet용 `(2048,1723)` (6,32)-regular RS-LDPC, rate≈0.841, `GF(2^6)` 기반 2-정보심볼 shortened RS 부호.
2. 문제: 기존 RS-LDPC는 coset 형성 시 무관 codeword 조합으로 parity-check matrix가 **랜덤 구조** → 스위치망(Benes) HW 복잡도·메모리 큼.
3. 핵심 기법: coset 생성 단계에서 곱셈 대신 **cyclic shift**로 symbol location vector를 `ps-2`회 순환 이동 → 각 submatrix가 circulant permutation form이 되어 전체 H가 quasi-cyclic(`HQC`)이 됨.
4. cyclic shift를 쓰는 이유: 곱셈은 `α∞`(무한원소)를 생성 못하지만 순환이동은 `α^{ps-2}→α∞→α^0` 포함 가능 → QC 구조와 α∞ 처리를 동시 만족.
5. RS-LDPC의 두 핵심 성질(같은/다른 member matrix 행 간 공통 1이 최대 1개 → girth≥6, 4-cycle 없음)을 그대로 유지하므로 BER 성능 저하 없음.
6. 예: `M=384 rows, N=2048`, γ=6 row group × ρ=32 col group의 64×64 submatrix, 행 weight 6·열 weight 32, `dmin≥8`.
7. 복호는 BPA / offset Min-Sum(BPSK/AWGN)으로 시뮬, QC화가 BER에 영향 없음 확인.
8. QC 구조 덕에 랜덤 Benes 대신 QC 전용 **QSN/Banyan** 스위치망 사용 가능.
9. 결과(90nm 합성, submatrix 1개): Benes 12,673 gate·67,584 bit → Banyan 9,087 gate·1,152 bit(메모리 2%), critical path 5.5→3.2ns, clock 180→300MHz.
10. 한계: 부호구성·스위치망 HW만 다루고 디코더 알고리즘/정정능력 개선 없음, 특정 RS-LDPC 부호 한정.
