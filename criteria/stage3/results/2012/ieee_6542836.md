### B. 알고리즘 요약

1. 시스템: LTE 터보부호(2개 RSC + QPP 인터리버, PCCC), rate 0.33(원)~0.9(펑처링), block length 40~6144, AWGN+BPSK.
2. 문제: 짧은 부호·고rate 펑처링 시 전통 터보 복호(Max-Log-MAP) 성능 급락, HARQ throughput 손실.
3. 모델: 터보 복호 후 soft output LLR 크기를 신뢰도로 사용, order statistics 기반 대수적 재복호.
4. 핵심기법: 복호 실패 시 OSD 수행 — LLR 크기순 정렬 → hard-decision·생성행렬 열 재배열 → Gaussian elimination으로 systematic `G̃sys` → 고신뢰 K비트로 재인코딩 → CRC로 검증. order-1 채택.
5. 핵심식 의미: 누적 LLR `LkS = Σ bt·Lkt`(식10), `bt=1 if t≥p`(식11). 초기 몇 iteration LLR은 진동(oscillation)하여 부정확하므로 p번째부터 누적해 신뢰도 안정화.
6. 확장: OSD 수행 빈도 `f`(f=0 없음, f=1 마지막, f=T 매 iteration) 3가지 스킴(tailed/repeated/acc-tailed).
7. 구현 디테일: bit-flipping 재인코딩 — i번째 비트 flip 시 `G̃sys` i행만 관여, O(K²)→O(K)로 복잡도 절감. 누적은 저장 3K + 곱셈-덧셈 3K 추가로 파이프라인 가능.
8. 최적화 절차: p를 1~8 스윕, p=3에서 최적.
9. 결과: (132,40)에서 BLER 1e-2 기준 repeated-OSD가 0.75 dB 이득; 펑처 rate 0.9에서 최대 1.2 dB.
10. 한계: HW 미설계, 시뮬만, 대상이 터보부호(LDPC 아님), OSD의 Gaussian elimination이 매 iteration 시 비용 큼.
