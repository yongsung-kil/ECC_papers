### B. 알고리즘 요약

1. 시스템: LDPC 인코더 → BPSK → partial response(ISI) 채널 `yᵢ=Σhₖxᵢ₋ₖ+Nᵢ`, 수신단은 BCJR equalizer + 표준 message-passing LDPC 디코더 연접.
2. 문제: AWGN에서 capacity-approaching인 protograph가 PR채널에서 성능 저하(특히 punctured node가 BCJR equalizer feedback에 불리), 구조적 부호로 i.u.d. capacity 근접이 미해결.
3. 모델: protograph를 copy-and-permute로 lifting한 구조적 LDPC, PR채널 상태(trellis) 구조를 protograph에 결합한 joint graph.
4. 핵심기법: joint graph의 반복복호 threshold를 EXIT chart로 계산 — BCJR equalizer의 EXIT 분석과 protograph EXIT 분석 결합, `J`함수 기반 mutual-information 전달식 반복.
5. 핵심식: VN→CN, CN→VN 업데이트를 `IEᵛ=J(Σeₛⱼ[J⁻¹(IAᵛ)]²+...)` 형태로 계산, APP-LLR `IAᵖᵖ=1` 수렴하는 최소 `Eb/N0`가 threshold.
6. 확장: BCJR equalizer↔LDPC 간 extrinsic 정보 `IAd` 교환을 반복하여 joint threshold 산출.
7. 트릭: linear minimum distance growth 보장(마지막 2행 edge합 ≥3), degree-1/degree-2 노드 포함으로 탐색공간 축소, `xᵢ∈{0,1,2}` 제한.
8. 최적화: punctured node 없는 rate-1/2 protograph(4 check, 8 var)를 dicode 채널 threshold 최소화로 단순 탐색 → ISI code `HISI` 도출.
9. 결과: rate-1/2 protograph가 i.u.d. capacity로부터 0.4dB, 16k blocklength에서 WER=10⁻⁶, dicode에서 AWGN code 대비 0.7dB 이득, capacity gap 0.9dB, error floor 없음.
10. 한계: HW 미설계, 시뮬만, dicode/EPR4 등 ISI 채널 특화, memoryless(AWGN/NAND) 채널 부호와 목표 다름.
