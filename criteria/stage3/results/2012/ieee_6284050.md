### B. 알고리즘 요약

1. 대상: BSC 채널, column-weight 3(dv=3) 바이너리 LDPC. (155,64) Tanner 코드, rate-0.753 (732,551) latin-square 구조 코드(dmin=12).
2. 문제: error floor(보장 정정능력이 ML 한계에 못 미침). trapping set이 저잡음 오류패턴에서 디코더 실패 유발, BSC에서 floor 기울기는 보장 정정능력이 좌우.
3. 모델: FAID `F=(M,Y,Φv,Φc)`, 메시지 alphabet `M={0,±Li}` (7-level = 3-bit), CN은 min-sum형 `Φc`, VN은 LUT/LT/NLT 규칙 `Φv`. isolation 가정 하 critical number로 성능 분석.
4. 핵심 기법: **decimation** - 일부 VN 비트값을 `b*`로 확정(항상 최강 메시지 `±Ls` 송신), 매 decimation round 후 나머지 메시지를 0으로 리셋해 재시작.
5. 의미: 오류패턴 부분그래프를 나머지 그래프에서 격리(주변 정상노드 확정), 리셋으로 인접 정상노드 추가 확정 유도. 오류노드는 절대 확정 안 되게 규칙 설계.
6. 핵심 확장: **적응형** decimation - 규칙집합 `B={β(1),β(2)}`, `β(2)`는 부분집합 `Ξ(2)[j]⊂Ξ(2)[j+1]`로 보수적→공격적 순차. 실패 시 더 공격적 규칙으로 전체 재디코딩.
7. 구현: `β(1)`은 3번째 iteration 후 1회, `β(2)[j]`는 2 iteration마다. VN 업데이트에 별도 맵 `Φvd`(decimation용)/`Φvr`(잔여 노드용) 사용, `|M|=7` 특화.
8. 설계: `Φvd`·`Ξ` 규칙은 코드의 minimal stopping set 내 오류패턴을 isolation 하 분석해 수작업 설계(critical number t+1이면 t-오류 보장). Corollary: 잔여그래프가 stopping set이면 실패 경향.
9. 결과: Tanner(155,64)에서 7-level ADFAID가 6-오류 모두 정정(FAID 5, BP는 5-오류 실패); (732,551)은 관측 영역서 5-오류 실패 패턴 없음. 여전히 3-bit·BP보다 저복잡도.
10. 한계: BSC·dv=3 전용, LUT 규칙 코드별 수작업 설계, 다중 재디코딩(latency↑), HW 미설계, 시뮬만.
