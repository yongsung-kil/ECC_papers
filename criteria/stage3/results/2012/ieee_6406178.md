### B. 알고리즘 요약

1. 시스템: 2D ISI(자기저장 TDMR/TwoDOS/patterned-media) 채널, `Mh×Nh` 응답행렬, AWGN, LDPC로 부호화 후 1D→2D 매핑(interleaver).
2. 문제: 2D ISI의 최적 MLSD는 NP-complete라 실용 불가, 반복 검출로 근사 필요.
3. 채널모델: `r(i,j)=h∗x(i,j)+v(i,j)`, 2D convolution 간섭에 AWGN 추가.
4. 검출기 구조: 직렬연접된 down-track + cross-track 두 SISO 검출기, IRCSDFA와 달리 soft-info를 가중없이 교환.
5. down-track: symbol-based BCJR, trellis state `S=(xi,k-Nh+1,...)`, `max*` 로그도메인 forward/backward metric(식 5~8).
6. cross-track: `j`열 대상 trellis, state `2^(Mh-1)`개, LLR `L(x(i,j))` 계산(식 10~15), bit↔symbol 변환기(식 20)로 교환.
7. LDPC: log-domain SPA(식 16~18, `tanh` 기반), regular LDPC `dv=4`, 내부 50 iter, 검출기-복호기 external 5 iter, 검출기간 3 iter.
8. density evolution: 메시지 밀도 추적(식 22~25), 채널검출기 밀도진화 `ξc(.)`는 closed-form 없어 Monte Carlo, threshold `σth`(식 26).
9. 결과: regular LDPC `N=1998, R=0.89`에서 제안 검출기가 MCMC 대비 0.4dB, IRCSDFA 대비 0.6dB 이득, threshold와 0.9dB 차(BER 10^-5).
10. 한계: HW 미설계, 시뮬만, 자기저장 2D ISI 특화, LDPC는 표준 SPA로 자체 기여 없음.

### C. Prime ECC 관련 모듈 핀포인트

대상이 other(2D ISI 검출기 중심)이며 LDPC 기여가 표준 SPA라 C섹션 성능 항목은 N/A.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2D BCJR turbo 검출기 | 대응 없음 | NAND에는 2D ISI 검출 구조 부재, Prime ECC는 채널등화 없음 |
| log-domain SPA LDPC 복호 | 대응 없음 | Prime ECC는 min-sum(`CNU_Update_New_Mag()`), full-tanh SPA 미사용 |
| density evolution threshold | 대응 없음 | 코드설계 분석기법으로 시뮬레이터 런타임 모듈에 대응 없음 |

적용 가치: **낮음** — 핵심이 자기저장 2D ISI turbo 검출기이고 LDPC는 표준 SPA 컴포넌트라 min-sum QC-LDPC NAND ECC에 이식할 요소가 없다.
