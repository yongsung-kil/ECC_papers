### B. 알고리즘 요약

1. 대상: BPSK/AWGN 채널의 binary LDPC 디코더 고정소수점 구현 비교, 예제 코드 2048-bit / 384 check node (802.3 계열).
2. 문제: error-floor(BER≤1e-11)에서 고정소수점 근사별 성능차 비교는 C-model(≈1M sym/s)로 불가 → HW 가속 HDL 에뮬레이터 필요.
3. Sum-Product 표준식: `L(r_i,j)=Φ⁻¹(ΣΦ(|L(q)|))`, Φ는 `-log(tanh(x/2))` 초월함수 (식3~4), HW에선 LUT로 구현.
4. Sum-Product 근사: Φ를 선형 `Φ'(x)=a-b·x`로 근사, MSE 최적 `a=2.125,b=1.125`; 실제 emulation 최적은 `aΦ=1,bΦ=0.25 / aΦ⁻¹=3,bΦ⁻¹=0.5`(2ⁱ꼴이라 shift+adder 1개).
5. Min-Sum(식7): Φ 대신 min1/min2 검색, 양자화 스킴 1개만 필요해 구현 자유도↓, post-processing만 가변.
6. Min-Sum 근사1(식12): 2nd-min을 `min+1LSB`로 대체 → 노드 복잡도↓, 정답 비율 `(dC-1)/dC`.
7. Min-Sum 근사2(식13): CS-tree에서 상/하 반 min들의 max를 2nd-min으로 → 정답비율 `(dC-0.5)/dC`(dC=32시 96.9%→98.4%), 단 error-floor는 상승.
8. WGN 소스: Box-Muller+Tausworthe(3 LFSR)+Cordic, Ziggurat는 error-floor 미관측이라 배제; 20 fractional bit(5σ), 상관계수 ≈1e-4로 무상관 검증.
9. 결과: 정규화 Min-Sum(계수 0.5)이 SNR 3.55dB@BER1e-9로 최고, Sum-Product 선형근사 4.16dB, 미정규화 Min-Sum/split-row는 4dB↑; Pareto-최적 3종 도출(면적 1.8~4.5mm²@90nm, 6.4Gbit/s 기준 IMAX 15~47).
10. 에뮬: 64×Virtex-4(ZeBu-XXL) 560kHz, early termination시 67.4Mbit/s로 Matlab-C 대비 최대 70배, 코드양자화 4정수+2소수 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum min1/min2 + 2nd-min 근사(식12,13) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 우리 CN 연산과 직결 - 2nd-min 근사로 CNU 면적 절감 여지, 단 error-floor 영향 재검증 필요 |
| 정규화 계수(0.5)·post-processing | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 우리는 magnitude 양자화 테이블 기반 - normalized factor 도입 검토 대상 |
| 고정소수점 양자화(4int+2frac, Φ LUT) | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()`/`Get_VNU_*`, [ecc_data.h](../Prime_ECC_3.1_Claude) `PARAM_LLR` | soft-2~3bit 양자화 설계와 정합 - LLR 테이블 폭 최적화 참고 |
| early termination | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 우리는 Partial/Full CRC 조기종료 보유 - 논문은 error-free 시 중단, 중복 |
| 면적·타이밍 코스트모델(라우팅 2차 의존) | 대응 없음 | HW 코스트 추정 방법론 - 코드베이스 이식 대상 아님 |
| Sum-Product Φ/Φ⁻¹ (full-tanh) | 대응 없음 | Prime ECC는 min-sum 전용, tanh Sum-Product 미채택 |

적용 가치: **중간** - min-sum 2nd-min 근사·정규화·양자화 트레이드오프는 우리 CNU/LLR 설계에 직접 참고되나, 핵심 min-sum/CRC 조기종료는 이미 보유하고 부호·NAND 타깃이 아님.
