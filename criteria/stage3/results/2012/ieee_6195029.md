### B. 알고리즘 요약

1. 시스템은 rate-1/2 주기적 time-varying regular-`(479,3,6)` LDPC convolutional code(LDPC-CC)로, sliding-window 기반 message-passing으로 복호한다.
2. 문제: 기존 iteration-parallel LDPC-CC 디코더는 two-phase MPD라 20개 내외 processor(=iteration)를 register로 구현 → 칩 면적이 큼.
3. 모델: AWGN 채널, check node 알고리즘은 offset min-sum(OMSA), 6-bit 양자화 채택.
4. 핵심기법: shuffled MPD를 LDPC-CC에 적용 - processor 내에서 최신 메시지를 즉시 반영해 수렴을 가속, two-phase 대비 iteration 약 절반으로 동일 BER 달성.
5. 핵심식: C2V/V2C 갱신 시 `t'<t` 최신 메시지를 즉시 사용(shuffled 스케줄), CN당 least-reliable V2C 크기를 압축 저장한 ordered set으로 CN memory 크기 축소.
6. 확장: node 병렬화 인자 `p=8`을 iteration 병렬화와 결합, block column 단위로 8개 노드 동시 처리.
7. 구현 디테일: architecture-aware(AA) 부호를 제약조건 C1~C6(인코딩 full rank, 병렬 인코딩, 메모리 접근/공유, 수렴, girth>4)로 구성 → 메모리 충돌 없이 단일 word 공유(CV/CN memory), permutation network(PU)로 processor간 메시지 교환.
8. 학습/최적화: global/local index를 mod 조건으로 결정해 length-4 cycle 회피 및 메모리 공유 동시 만족(별도 NN 학습 없음).
9. 결과: 90nm CMOS, 11 processor(10 iteration), 면적 5.36mm^2, 256.4MHz서 information throughput 1.025Gbps, power 682mW. [25](30 processor, 27.72mm^2) 대비 면적 대폭 감소.
10. 한계: LDPC-CC(convolutional) 전용, code length 본문 서술상 명확치 않음, 정정능력 자체는 two-phase와 동일(수렴속도만 개선).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| shuffled MPD 스케줄(iteration 절반) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 이터레이션 루프 스케줄 대응 - 단 Prime ECC는 Dual-Update(even/odd) 스케줄 이미 보유 |
| offset min-sum CN 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` | min-sum 자체는 이미 보유(중복 기여) |
| 메모리 공유/permutation network HW | 디코더 HW 모델(z=32, GT/HCU) | LDPC-CC 전용 메모리 구성이라 block QC-LDPC 구조와 정합 어려움 |
| AA-LDPC-CC 부호 구성(circulant+CC) | 대응 없음 | Prime ECC는 block QC-LDPC이며 convolutional 구성법 미대응 |

적용 가치: **낮음** - 부호가 LDPC convolutional code라 block QC-LDPC 기반 Prime ECC에 부호·HW를 그대로 이식할 수 없고, 유일하게 재사용 여지가 있는 shuffled MPD 아이디어도 이미 보유한 Dual-Update 스케줄과 중복된다.
