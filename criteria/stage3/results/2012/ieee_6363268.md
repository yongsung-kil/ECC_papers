### B. 알고리즘 요약

1. 시스템: irregular LDPC (code A 1512/2016, code B 1008/2016), Min-Sum layered decoding, max 10 iter, AWGN 채널 BPSK, Virtex5 FPGA 프로토타이핑으로 BER<1e-9 관측.
2. 문제: LLR 유한 워드길이 표현이 정정성능·메모리·라우팅·면적에 영향, 최적 워드길이 탐색에 긴 시뮬 필요 → 분석 모델로 대체.
3. 모델: 양자화 오류를 saturation(범위 이탈)과 roundoff(범위 내 반올림) 두 종류로 분리, 2's-complement `(i,f)` 표현 `L_d=-2^{i-1}, L_u=2^{i-1}-2^{-f}`.
4. 핵심기법: check node의 min 연산 특성상 출력이 saturated면 입력 전부 saturated → `P(R_kj sat)=Psat^{d_k-1}`, degree 분포 `ρ(x)`로 가중합.
5. 핵심식: `IFRE=PrndErr+PsatDec·(PsatErr-PrndErr)` - 반복복호 후 결정메시지의 saturation 확률로 전체 성능 열화 지표 산출.
6. 확장: variable node update는 단순 합산이므로 saturated 메시지 수에 비례, LLR 항 비율 `LF=V/N`으로 채널 a priori 기여 반영.
7. 부수 트릭: iteration별로 `Psat(i+1)`을 재귀 갱신(Algorithm 1), 결정메시지는 `LF=V/(N+V)`로 보정.
8. 최적화: 학습 절차 없음 - 이전 연구[7]의 오류전력(PsatErr/PrndErr)과 결합해 여러 `(i,f)` 스킴 중 최적 선정.
9. 결과: code A에서 (5,1) 스킴 우세 예측이 시뮬과 일치, (6,0)은 Eb/N0>5dB에서 개선. 내부 메시지는 18bit 표현.
10. 한계: cycle에 의한 메시지 상관을 무시(독립 가정), AWGN Gaussian mixture 전제, HW는 검증용 FPGA뿐 신규 아키텍처 아님.
