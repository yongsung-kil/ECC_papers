### B. 알고리즘 요약

1. 대상: tailbiting convolutional로 표현되는 QC-LDPC block code, rate 1/2·2/3·3/4·5/6, block length `n≈576`, regular((3,K))·irregular 모두.
2. 문제: WiMAX 802.16의 짧은 블록(n≈576) 코드보다 나은 BER를 내는 QC-LDPC를 찾기 — 통상 girth만 목표로 하는데 base matrix 구조가 BER에 더 큰 영향이라는 통설.
3. 핵심 개념: `girth profile` 도입 — 열가중치 작은 앞쪽 서브행렬들의 free girth를 정렬한 튜플 `Sg=gfree,i^(n(i))`로 girth를 세분화.
4. 핵심 기법 1(Alg BD): 목표 degree distribution `λs`와 행/열 최대 중첩 `Pc,Pr`을 만족하는 base matrix `B`를 랜덤 greedy 생성 (`Pc=2` → gB=4이나 d2=7이면 girth 14 라벨 가능).
5. 의미: `Pc,Pr`이 base graph girth `gB`·2차 일반화 Hamming distance `d2`를 결정 → girth 하한 `g ≥ 2max{gB+⌈gB/2⌉,d2} ≥ 3gB` (식 4)을 좌우.
6. 핵심 기법 2(Alg GP): 주어진 base matrix에 대해 girth 부등식계를 풀어 목표 free girth profile을 만족하는 monomial 라벨 `wij`를 greedy 탐색, tailbiting `M` 반영 시 `g=gfree` 보장.
7. 구현 디테일: regular은 all-one(AO)·Steiner triple system(STS) base matrix 사용, 라벨 탐색이 최복잡 단계(열당 10^3~10^6 라벨링, 코드당 평균 1~3시간).
8. 학습/최적화: greedy + rejection(중첩·girth 부등식) 기반, 후보 10개 중 BER 10^-5 SNR로 최종 선택.
9. 결과: good girth profile로 소~중 SNR에서 통상 0.5~1 dB BER 개선; R=1/2·5/6은 WiMAX 대비 유의미 개선, R=2/3·3/4은 고SNR에서 소폭; good girth profile 코드에서 error floor 미관측.
10. 한계: HW 미설계, AWGN/BP 시뮬만, 짧은 블록·저~중rate 중심, 고rate에선 base matrix 탐색 실패, latency/iteration 기여 없음.
