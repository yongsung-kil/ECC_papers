### B. 알고리즘 요약

1. AWGN/BSC 채널용 일반 LDPC 부호를 Spartan-3E XC3S500E FPGA에 Verilog로 구현한 데모 논문.
2. 부호 rate `1/2`, `1/3`, `1/4`, H-matrix 크기 `6x12`~`4096x8192`(codeword 6~8192 bit)로 다양하게 구성.
3. Repeat-Accumulate(RA) LDPC 부호도 함께 구성.
4. 복호는 표준 log-domain Belief Propagation(log-BP)과 hard-decision bit-flipping 두 가지.
5. log-BP: data update `M_{j,i}=Σ E_{j',i}+r_i`, check update에 `f(x)=log(tanh(x/2))`, `g(x)=2tanh^{-1}(exp(x))` 사용(교과서 sum-product).
6. bit-flipping: mod-2 parity check 불만족 시 다수결로 비트 반전, 모든 parity 만족 or 최대 iteration까지 반복.
7. syndrome check(`H·z=0`)로 조기종료.
8. C, Verilog-HDL, MATLAB(Simulink) 3가지 코딩 스타일로 교차 검증.
9. 결과: codeword 4096 bit에서 BP가 오류 75개, BFD가 50개 복구(rate=1/2 비교표); 면적 7693 gate(Leonardo Spectrum), critical path slack 0.69ns.
10. 한계: 순수 통신용 일반 LDPC, gatecount·throughput·BER 곡선 등 대부분 그림/표에만 있어 손상, NAND·QC-LDPC 연관성 없음.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| log-BP (sum-product) 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 우리는 min-sum 사용, full-tanh sum-product는 미채택 → 대응 낮음 |
| bit-flipping 복호 | 대응 없음 | hard-decision BF는 Prime ECC 미사용 |
| syndrome check 조기종료 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) / full_CRC.cpp | 개념만 유사, 우리는 CRC 기반 |
| RA/일반 LDPC 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 우리는 QC-LDPC 고정, RA 구성 무관 |

적용 가치: **낮음** — 교과서 log-BP/bit-flipping FPGA 데모로 이미 보유 기법과 중복이거나 하위 수준이며 QC-LDPC/NAND 특화 요소가 없다.
