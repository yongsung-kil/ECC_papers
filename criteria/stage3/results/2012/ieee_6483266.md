### B. 알고리즘 요약

1. 대상: regular binary LDPC(`(dv,dc)`, 예 `(3,6)/(4,8)/(6,12)`, rate 1/2)를 **hard-decision Gallager-B** message-passing으로 복호, BSC(cross-over `po`) 채널.
2. 문제: 복호기 자체를 faulty 게이트로 구성 시 성능 저하를 정량화하고, 낮은 잔여오류 `pres`를 적은 복잡도로 달성.
3. 고장 모델: VN/CN 함수 평가가 확률 `φ`로 출력 반전(deviation). VN·CN deviation을 등가 CN deviation `φ = (1-(1-2φv)^(dc-1)(1-2φc))/2`로 통합, edge당 정규화 `φe`.
4. 핵심 기법: Gallager-B VN 규칙(임계 `b`)을 **메시지 반복** `n`으로 확장 — 각 edge 메시지를 `n`번 계산·전송, 확장 다수결 `MAJR`과 임계 `bn`으로 반복코드 길이를 `dv`와 독립 제어.
5. 반복코드 효과: VN 입력이 길이 `dv-1` 반복코드를 형성해 `pres < φe`; `n` 증가로 `dv` 고정한 채 내결함성↑.
6. EXIT 함수 `f_bn(p[t-1],po)`를 `MAJR`로 재유도(식 15), 채널 threshold `po*`와 deviation threshold `φe*` 알고리즘적으로 계산(Alg.1 + binary search).
7. 결과: `(4,8)`에서 reliable `po*=0.0516`; `φe`가 특정값 넘으면 `po*` 급락, 낮은 `pres`일수록 `φe*`는 `po`에 거의 무관.
8. 최적화: iteration별 `(n,bn)` 규칙 선택을 trellis 최소비용경로(gear-shift, 복잡도 식 20)로 스케줄링.
9. 결과 수치: `(4,8)`, `po=0.04`, `pres=1e-11`에서 reliable 스케줄 cost 14, overhead 50%(cost 21) 허용 시 `φe`를 2.94e-4까지 상향(반복은 마지막 3 iteration만). 작은 overhead로 tolerable deviation을 수 자릿수 개선.
10. 한계: cycle-free 점근 EXIT 분석. gatecount/throughput/BER 실측 없음, soft-decision·min-sum·NAND와 무관.

