### B. 알고리즘 요약

1. 시스템: MLC(2bit/cell) NAND, LDPC 등 soft-decision ECC 사용 시 세밀 sensing이 on-chip sensing + memory-to-controller 전송 지연을 크게 늘리는 문제.
2. 문제: soft sensing 결과를 fixed-length(`⌈log2 n⌉`-bit)로 전송하면 비균일 양자화에서 엔트로피 낭비 → 전송 데이터량·지연 과다.
3. 배경 기법: (i) progressive sensing(HD 먼저, 실패 시 SD로 fail-and-retry), (ii) non-uniform quantization(오버랩 영역에 양자화 집중)이 이미 존재.
4. 핵심기법 1: entropy coding — 비균일 sensing 영역의 확률차를 이용해 sensing 결과를 무손실 압축, 3-bit fixed → 평균 2.45-bit(약 18% 감소) 예시.
5. 핵심 아이디어: 전송량 = 엔트로피에 근접시키는 것이 목표. 각 threshold 영역 확률이 다르므로 fixed-length는 준최적.
6. 핵심기법 2: zoned entropy coding — progressive sensing에서 이전(HD) 결과가 threshold 가능영역을 이미 한정 → 같은 zone 내 영역만 유일 부호화하고 zone 간에는 코드워드 재사용.
7. 구현 디테일: 2-step 예시(1st: 4-level HD, 2nd: 7-level SD), zone = {I,II},{II,III,IV},{IV,V,VI},{VI,VII}, 높은 확률로 셀당 1-bit만 추가 전송.
8. 소자모델: erase/program Gaussian, RTN(지수), retention(`ln(1+t)` shift), cell-to-cell interference(절단 Gaussian) 합성곱으로 최종 threshold 분포 `pf` 생성.
9. 결과: fixed-length 대비 complete entropy coding 20.4% 감소, zoned entropy coding 64.8% 감소(2nd-step SD 전송지연 기준).
10. 한계: LDPC 부호·디코더 자체 개선 없음, 가설적 device 모델 기반 시뮬만, HW 구현·2-step 예시에 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft-read 양자화 / threshold 설정 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 비균일 sensing 영역·확률은 LLR threshold 설계와 접점, 다만 논문은 압축이 초점 |
| progressive(HD→SD) sensing 흐름 | `channel.cpp` (soft-read 양자화) + 조기종료 재시도 | Prime ECC의 HD/2SD/3SD 지원과 개념 정합, fail-and-retry는 재사용 가능 |
| entropy / zoned entropy 압축 | 대응 없음 | Prime ECC는 sensing 결과 압축(전송지연) 모듈 없음 — 컨트롤러 read path 밖 |
| LDPC 디코딩 자체 | `decoder.cpp` `LDPC_Decoder()` | 논문이 디코더를 개선하지 않음, 무관 |

적용 가치: **중간** — NAND soft-read의 전송지연을 줄이는 직접 관련 기법이나, Prime ECC의 부호설계·디코더·HW가 아닌 read/전송 파이프라인 계층 최적화라 참고 수준(채널·LLR 경로 아이디어)에 그친다.
