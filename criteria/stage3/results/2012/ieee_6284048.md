### B. 알고리즘 요약

1. 대상: AWGN 채널, LLR-BP 복호, regular PEG(504,252,3,6)·irregular PEG(1024,518,15,8) 부호.
2. 문제: 고SNR에서 trapping set이 유발하는 error floor. 진동(oscillating) trapping set과 elementary trapping set을 구분해 대응.
3. 모델: trapping set (x,y) = x개 VN 유도 부분그래프에 홀수차수 미충족 CN이 y개. 진동은 사이클 중첩으로 LLR 진동, elementary는 저SNR floor 주원인.
4. 1단(진동 대응): 각 노드 belief `B(vi)=|L(vi)|`의 iteration간 증감을 관찰, 급감(`>β`)·급증(`>ν`)이면 선택(`Ω(vi)>0`).
5. 선택 노드의 VN 출력 메시지를 직전 iteration과 평균 `m=(m^(l)+m^(l-1))/2`. CN 메시지는 미변경.
6. 의미: 오류 정보 전파를 늦춰 신뢰 노드가 먼저 수렴하도록 유도, 진동 trapping set 완화.
7. 2단(elementary 대응): 1단 실패 & 미충족 CN 수 `<CNthreshold`이면, 미충족 CN 연결 VN 중 다른 공통 CN 없는 집합 `Vnc`의 채널 LLR을 `-η`배(bit flip)로 재디코딩.
8. flip된 정상 노드는 공통 CN 없어 재수렴, trapping set 노드는 평형 파괴. `Ω=2`로 항상 평균 유지. backtracking 회피.
9. 결과: irregular(1024) SNR 3.5dB에서 BLER 2.13e-8(제안) vs 6.33e-8(averaging) vs 1.8e-7(standard); elementary trapping set 해결율 regular 1/4, irregular 5/8.
10. 한계: 시뮬만, HW 미설계, 짧은 코드 2종, full LLR-BP 기반. 파라미터 β=3.2·ν=1·η=0.8·CNthreshold=10 실험적 튜닝.
