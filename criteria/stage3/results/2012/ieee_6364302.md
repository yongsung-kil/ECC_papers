### B. 알고리즘 요약

1. 시스템: BICM 수신기, 이진입력 무기억 채널로 `N`비트 codeword 전송, ML 디코더가 L-value 사용.
2. 문제: 수신기가 계산한 L-value가 mismatched(모델 부정확/파라미터 오차/근사)이면 ML 디코딩이 suboptimal, 이를 선형보정 `l̃ᶜ = α·l̃`로 교정하고자 함.
3. 모델: L-value는 consistency `p(l|1)/p(l|0)=eˡ` 및 symmetry 조건을 따르는 확률변수 `Lₙ`으로 취급.
4. 핵심기법: 보정 후 ML 디코더의 PEP(pairwise error probability)를 직접 최소화하는 `{αₙ}`를 찾음. `argmin PEP({αₙ}; ĉ)`.
5. 핵심식: PEP를 CGF `κ_Σ(s)`의 Bhattacharyya 상한 `PEP ≤ e^{κ_Σ(ŝ)}`로 근사, saddlepoint `ŝ`에서 최소화.
6. 핵심결과: 최적 보정계수 `α̂ₙ = 2ŝₙ` (또는 `2|ŝₙ|`), 즉 L-value CGF saddlepoint의 2배 (Theorem 1, Prop. 1).
7. 트릭: pdf 대신 CGF만 알면 되므로 수치적분 회피 가능; Gaussian 가정 시 `α̂ = 2μ̃/σ̃²`로 기존 [6] 결과와 일치.
8. 최적화: 간섭 있는 BPSK 채널 예제에서 saddlepoint 방정식 `hₙ - σ_z²ŝ = gₙ·tanh(gₙŝ)`를 Newton-Raphson `Iₘₐₓ=2`회로 풀거나 SNR/SIR 테이블 보간.
9. 결과: `Nb=1000`비트, rate-1/2 convolutional code(Viterbi) 및 turbo code(5 iter)로 BER 시뮬, 제안 보정이 true L-value 성능에 근접(고SNR에서 두드러짐), GMI 기반 보정과 유사하나 수치적분 불필요.
10. 한계: HW 미설계, 시뮬만, LDPC는 응용 예시조차 아님(convolutional/turbo만), BICM 간섭채널 특화.
