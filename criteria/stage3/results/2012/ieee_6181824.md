### B. 알고리즘 요약

1. 대상: BSC(Binary Symmetric Channel) 위의 regular column-weight `dv=3` LDPC 코드, 사례로 `(N=155, K=62, dmin=20)` Tanner code (rate 0.4) 사용.
2. 문제: 반복복호(BP/min-sum)는 유한장 코드에서 trapping set에 걸려 error floor가 발생하며, 최소 해밍거리 대비 guaranteed error correction `t`가 MLD보다 훨씬 작음.
3. 채널/디코더 모델: FAID는 메시지를 유한 알파벳 `M={-Ls,...,0,...,Ls}` (`Ns=2s+1` levels, 3-bit precision이면 `Ns=7`)로 제한한 message-passing 디코더. check-node 갱신 `Φc`는 min-sum과 동일(min-sum의 sign·min 연산), variable-node 갱신 `Φv`는 비균일 양자화 함수 `Q(.)`와 비선형 파라미터 `mc`로 정의.
4. 핵심 기법(decoder diversity): 같은 코드에 대해 서로 다른 동역학을 갖는 FAID 규칙 집합 `D={D1,...,D_ND}`를 구성, 어떤 error pattern을 한 디코더가 못 고쳐도 집합 내 다른 디코더가 고칠 수 있도록 함(`E = ∪ E_Di`).
5. 선택 알고리즘: 브루트포스 대신 trapping set 유형 `(a,b)` 주변의 오차패턴 집합 `E[t](Λ(A,B))` (A=2t)만 고려하는 그리디 알고리즘(Algorithm 1)으로 최소에 가까운 diversity set을 탐색(NP-hard Set Covering 문제의 휴리스틱 근사).
6. 확장 기법: 여러 반복 후에도 안 풀리면 decoder를 랜덤 재초기화(random re-initialization, Algorithm 2) - 하드결정과 채널값 비교해 확률 `δ`로 비트 플립, 진동하는 attractor에서 탈출시켜 MLD에 근접.
7. 구현 디테일: 코드의 quasi-cyclic 대칭(σ,π,ρ automorphism)을 이용해 trapping set 개수를 최대 10^5배까지 축소, 탐색 복잡도를 실용 수준으로 낮춤.
8. 최적화 절차: Table-driven greedy selection(Algorithm 1) + Monte-Carlo 기반 재초기화 파라미터(`NI*=20, ε=5, δ=0.05`) 튜닝.
9. 결과: Tanner code에서 t=5(기존 FAID)→t=7(decoder diversity, 243개 디코더)로 guaranteed error correction 향상. Diversity set(9239개 FAID + 랜덤 재초기화)으로 BP 대비 최대 5 decade FER 개선, MLD lower bound에 근접(α=1e-2에서).
10. 한계: HW 설계 없음, 순수 시뮬레이션(Monte-Carlo)뿐. 수천 개 FAID 규칙을 순차 실행하며 디코더당 최대 4000 iteration이 필요해 latency가 크게 증가하고, 어떤 디코딩 규칙(LUT)을 썼는지 세부는 "future publication"으로 미공개.

> 적용 가치: **중간** — min-sum 코어 위에 FAID 규칙 다양성(diversity)을 추가하는 아이디어는 이식 난이도가 낮은 편이나, 다수 디코더 규칙 저장/전환과 latency 증가(최대 4000 iteration)가 NAND 실시간 제약과 상충되어 HW 적용은 신중한 검토 필요.
