### B. 알고리즘 요약

1. 시스템: IEEE 802.16e OFDM 물리계층, QC-LDPC (rate 1/2, `N=1536`, info 768-bit), SUI-3 무선채널 + QPSK/BPSK.
2. 문제: 기존 RS-CC concatenated 방식의 복호 지연·복잡도가 크고, BP는 대규모 H-matrix에서 저장·연산 부담이 큼.
3. 인코딩: 프로토콜 base matrix `Hb=[Hb1|hb|Hb2]` (hb 홀수 weight 열, Hb2 double-diagonal), z×z permutation으로 확장, approximate lower triangular 방식으로 patiry `p1,p2` 산출 (식 3,4).
4. 핵심 기법: hard-decision Bit Flipping. checksum `s_m = H·x^T`, 오류패턴 `e_n = s^T·H` 로 각 비트의 check 실패 횟수 계산.
5. 개선 포인트: 실패 횟수 최대인 비트만 flip하는 대신, flip 임계값 `δ`를 두어 δ 초과 비트를 iteration당 다중 flip → 수렴 가속.
6. 의미: 한 iteration에 고오류확률 비트를 여러 개 정정해 checksum 계산 횟수·연산량을 줄임.
7. 구현 디테일: check matrix를 sparse matrix(비영원소 위치·값만 저장)로 저장해 메모리·지연 절감.
8. 채널/판정: `r_n=2y_n-1` 이중극성 후 hard decision, 최대 iteration 도달 또는 전 checksum 만족 시 종료.
9. 결과: BER 0.2 기준 LDPC가 RS-CC 대비 약 4dB coding gain, iteration 30이면 수렴 만족; interleaver 제거해도 BER 유지(오히려 소폭 개선).
10. 한계: HW 미설계, 시뮬(Matlab Simulink)뿐, hard-decision BF라 정밀 정정능력은 BP/min-sum 대비 낮음, 복잡도는 정성적 O(N) 언급뿐.
