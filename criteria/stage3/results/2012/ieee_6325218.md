### B. 알고리즘 요약

1. 대상은 무선 IR(incremental redundancy)/HARQ용 rate-compatible protograph LDPC convolutional code family (`{C_k}`, `K=11` members, rate 약 0.2~0.8).
2. 문제: puncturing 기반 rate-compatible은 고rate 초기전송 성능이 나쁘고, irregular extension은 rate마다 degree distribution 재최적화가 필요.
3. 모델: raptor-like precode `B(1)`에 IR step마다 variable/check node를 하나씩 append하는 nested base matrix `B(k)` (식 (1)).
4. 핵심기법: **spatial coupling** - `B(k)`의 edge를 `m_s+1` 시간구간에 **edge spreading**하여 convolutional base matrix `B_[0,L-1]` 생성 (식 (2), `m_s=1`).
5. 제약식: `B = Σ B_i` (식 (3)) - node degree와 computation tree 구조 보존; check degree-1 발생 방지를 위해 저degree row는 spread 금지.
6. rate식: `R_k = 1 - ((L+m_s)/L)(M_k/N_k)` (식 (13)) - termination rate loss는 `L→∞`에서 소멸.
7. 분석: 각 IR step protograph에 **density evolution** 적용하여 BEC/AWGN threshold 산출 (여러 `L`, 정칙 dc=10/36 대조군과 비교).
8. 복잡도지표: `C_LDPC = d̄_v/R` (평균 variable degree / rate) - message 계산량/정보bit.
9. 결과: 결합 시 threshold가 모든 rate에서 Shannon 한계에 균일 gap으로 근접; 정칙 IR family는 저rate에서 복잡도가 거의 지수적 증가하나 제안 irregular는 선형 증가.
10. 한계: DE threshold(무한길이 점근) 분석뿐 - 유한길이 BER/FER, gate count, HW, latency 없음; NAND 아닌 무선 IR 채널 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC/protograph rate-compatible 부호구성 | 대응 없음 | Prime ECC는 고정rate binary QC-LDPC, convolutional/rate-compatible 구조 미보유 |
| spatial coupling edge spreading | 대응 없음 | 우리 부호에 coupling 구조 없음 |
| density evolution threshold 분석 | 대응 없음 | Prime ECC는 bit-exact HW 시뮬레이터로 DE 미수행 |

적용 가치: **낮음** - 무선 IR용 rate-compatible SC-LDPC로 NAND 고정rate binary QC-LDPC ECC와 목적·구조가 달라 떼어 쓸 부분이 사실상 없음.
