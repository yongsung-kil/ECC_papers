> 총평: SC-LDPC의 sliding window 디코더가 버스트 erasure에 취약한 점을 저지연 convolutional interleaver로 보완하는 무선 통신 시스템 논문. 채널(erasure)·부호(SC)·타깃(wireless low-latency)이 NAND용 binary QC-LDPC와 이질적이고 인터리버는 시스템 레이어라 이식성 하.

### B. 알고리즘 요약

1. 시스템: (3,6)-regular SC-LDPC, `ms=1`, `B0=[2 2] B1=[1 1]`, L=40, base matrix 41x80, rate 0.5, 저지연 무선 통신 타깃.
2. 문제: SC의 ribbon-shaped H + sliding window 디코더는 버스트 erasure(채널 메모리)에 성능 급락, block interleaver는 지연 과다로 SC 이점 상쇄.
3. 모델: 상관 erasure 채널 — 길이4 버스트(erasure율 `εB=0.6`)가 codeword 임의 위치에서 발생, 나머지는 표준 erasure율 `ε`.
4. 핵심기법: convolutional interleaver(Λ delay line, λ번째 line은 `1+(λ-1)Ψ` delay 소자)로 버스트를 W보다 넓게 분산 → window 디코더가 연속 erasure를 랜덤처럼 관측.
5. 핵심 성질: `π=π^-1` (동일 블록으로 인터리브/디인터리브 가능) → 메모리·HW 재사용 이점.
6. 지연 분석: 예제(길이10)에서 총지연 5, 초기화 zero 소자 수(6)로 상한; 길이80(L=40)에서 지연 6으로 block length 대비 무시가능.
7. DE(BEC): `q_{i→j}=1-∏(1-p)`, `p_{j→i}=ε∏q`, 성공=목표심볼 erasure<`δ=10^-6`, max iter 1000.
8. 특이 관찰: 버스트가 홀수 위치 시작이면 check node 1개 완전 erase → 임계값 최소, 짝수 시작이면 완전 erase 없어 임계값 소폭 상승(교번 패턴).
9. 결과: 버스트 없을 때 `ε*wd=0.4843`, 버스트 시 `0.2182≤ε≤0.2790`로 급락 → convolutional interleaver 적용 시 임계값 대폭 회복(지연은 무시가능).
10. 한계: BEC/erasure 점근 DE 이론뿐, HW 미설계, AWGN·NAND·유한길이 BER·gate count 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| convolutional interleaver / 버스트 분산 | 대응 없음 (Prime ECC는 인터리버 레이어 없음) | 시스템 레벨 기법, ECC 엔진 내부 아님 |
| SC-LDPC / sliding window 디코딩 | 대응 없음 (Prime ECC는 QC-LDPC block 디코딩) | 부호종류·스케줄 상이 |
| BEC/erasure DE 임계값 | 대응 없음 | Prime ECC는 AWGN/RBER(channel.cpp), erasure 미대상 |
| BP message-passing | decoder.cpp LDPC_Decoder() (개념적) | erasure BP라 min-sum과 직접 대응 아님 |
```

적용 가치: 낮음 - 무선 저지연 SC-LDPC + interleaver 시스템으로 채널·부호·타깃이 NAND binary QC-LDPC와 상이하고, 이식 대상이 ECC 엔진 외부(인터리버)라 떼어 쓸 요소 없음.
