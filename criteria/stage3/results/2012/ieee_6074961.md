### B. 알고리즘 요약

1. 시스템: MLC NAND SSD 컨트롤러(FTL)에서 asymmetric coding + stripe pattern elimination algorithm(SPEA)를 BCH ECC 앞단에 추가.
2. 문제: 측정 결과 data retention error가 비대칭적(특정 상태로의 천이가 지배적)이고, bit-line 스케일링으로 column-stripe 프로그램 패턴 시 program peak current가 과도하게 커짐.
3. 핵심 가정: cell error 분포는 데이터 패턴(“1”/“0” 비율, 컬럼 stripe 여부)에 강하게 의존한다는 실측 기반 모델.
4. 핵심 기법 1: asymmetric coding — 데이터 unit 내 “0” 비트가 다수면 전체 반전(flag 1비트 추가)하여 lower/upper page에서 “1”/“0” 비율을 각각 60%로 편향, 최고 상태(state) cell 개수를 줄임.
5. 핵심 식: 코드 길이 `k`일 때 flag overhead `2^{-k}` 및 확률 `P(1)` 계산(식 (1)(2)); overhead-확률 트레이드오프가 code length로 결정됨.
6. 핵심 기법 2: SPEA — 짝/홀수 컬럼 “1” 개수 차이가 임계값(`Th`) 이상이면 데이터를 재배열·반전(flag1/flag2)해 worst-case column-stripe 패턴(최대 inter bit-line capacitance 충전)을 제거.
7. 부수 디테일: 인코딩 순서는 SPEA→asymmetric coding→BCH ECC, 복호는 ECC→asymmetric→SPEA 순(플래그 비트 오류를 ECC로 먼저 정정해야 하므로).
8. 별도 학습/최적화 절차 없음 — 회로 레벨 카운터/비교기/MUX로 구현(순수 조합 로직).
9. 결과: data retention error 90% 감소(asymmetric coding), program peak current 43% 감소(SPEA, 3Xnm), 동일 오버헤드에서 24bit→61bit BCH correction 수준 신뢰도 등가.
10. 한계: 제안 기법은 ECC(BCH) 앞단의 데이터 전처리일 뿐 ECC 알고리즘 자체(부호설계/디코더)는 변경하지 않음, "LDPC systematic encoding과 결합 가능"이라는 언급만 있고 실측/시뮬 데이터는 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| asymmetric coding (데이터 반전 전처리) | 대응 없음 | ECC 앞단 FTL 레벨 전처리로 Prime ECC의 부호/디코더 로직과 별개 |
| SPEA (bit-line stripe 패턴 제거) | 대응 없음 | NAND program 전력 최적화이며 ECC 알고리즘·HW 구조와 무관 |
| BCH ECC (블랙박스로 사용) | 대응 없음 | 논문은 BCH를 그대로 사용, LDPC 관련 기법 제시 없음 |

> 적용 가치: 낮음. ECC 알고리즘·디코더·HW 구조를 전혀 다루지 않는 NAND 컨트롤러 레벨의 데이터 전처리(전력/신뢰성) 기법으로 Prime ECC 코드에 이식할 요소가 없음.
