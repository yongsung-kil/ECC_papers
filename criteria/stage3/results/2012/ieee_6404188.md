### B. 알고리즘 요약

1. 대상 시스템: LDPC 인코더 + AWGN 채널 + LDPC 디코더 + 오류계수기 + 2개 LFSR(RNG)로 구성된 FEC 코덱을 SUT로 함.
2. 부호: WiMAX(802.16e) degree distribution polynomial로 구성한 여러 rate `R`/길이 `N`의 irregular LDPC, 대표 인스턴스는 504 check node + 2016 variable node.
3. 디코더: Normalized Min-Sum(정규화 계수 `a=0.75`)의 layered decoding 변형을 VHDL로 구현.
4. 풀려는 문제: error-floor(BER < 1e-9) 영역의 rare event(1/1e12)를 소프트웨어 시뮬로는 관측 불가 - 고속 관측 수단 필요.
5. 핵심 기법: 동일 parametric HDL 모델로 병렬도(degree of parallelism) 조절 가능한 디코더 생성 → FPGA 에뮬레이션이 double-precision Matlab 대비 4자릿수 이상 빠름(BER 1e-12까지 도달).
6. 검증 계층: double-precision Matlab ↔ bit-true Matlab ↔ HDL(behavioral/post-route) ↔ FPGA 실측을 단계적으로 등가성 확인.
7. 임베디드 시스템(ESPVF): MicroBlaze/PowerPC + DDR2 SDRAM + Ethernet + 커스텀 C OS/웹서버로 SUT 데이터를 원격 캡처(Chipscope 대비 저장·유연성 우위).
8. 저장 방식 비교: Serial(7,854,000 CPU cycle) vs CompactFlash(3,372,400) vs DDR2(843,056) → DDR2+Ethernet 채택.
9. 결과: FPGA 시뮬이 Matlab 대비 4자릿수↑ 가속, 검증 32,754 파일 44시간 100% 안전저장, ESPVF는 SliceM 8%/SliceL 7%만 점유.
10. 한계: 트래핑셋/거리문제 관측 인프라일 뿐 새 디코더·부호 알고리즘은 없음, 60GHz backhaul(무선) 응용 타깃.
