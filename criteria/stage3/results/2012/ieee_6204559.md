### B. 알고리즘 요약 (10줄 내외)

1. Spatially coupled LDPC 코드는 convolutional LDPC 코드를 유한 길이로 종단(terminate)한 부호로, parity-check matrix `H[L]`가 대각선을 따라 리본(band) 형태를 가짐 (식 (2)).
2. 기존 블록 LDPC 코드는 코드 길이에 비례한 복호 복잡도를 갖지만, spatially coupled 코드는 리본 폭(코드 메모리 `ms`)에만 비례하는 저복잡도 windowed decoding이 가능함이 문제의식.
3. protograph 기반 구성: base matrix `B[L]`를 서브매트릭스 `B0...Bms` (크기 `J×K`)로 정의하고, 각 원소를 `M×M` 순열행렬의 합으로 치환(lifting factor `M`)해 실제 H 행렬 생성 (식 (3), (4)).
4. `ms=1`(최소 결합 메모리) 고정, `B0`, `B1` 두 서브매트릭스의 성분 조합으로 만들 수 있는 16가지 (3,6)-asymptotically-regular 코드 앙상블을 전수 탐색.
5. Windowed decoder: 전체 `L+ms` 블록의 parity-check 행 중 앞의 `W`개 구간(`W·M·J` 체크노드와 연결된 비트노드)만 반복복호하고, 사전 정한 반복횟수 후 앞 `M·K` 비트노드를 확정, 창을 한 칸씩 이동시키며 처리 (Fig. 3).
6. 실험 파라미터: `L=100`, `M=20`, `W=6` → 코드 길이 `n=4000`; 표준 블록 복호 지연 `n=4000` 심볼구간 대비 windowed 복호 지연 `W·M·K=240` 심볼구간으로 대폭 감소.
7. AWGN 채널에서 16개 후보 코드의 BER 성능을 비교, 코드 메모리 `ms=1`이면서 실질적 결합이 일어나는 `B0=[1 2]`, `B1=[2 1]`(및 미러) 조합이 최고 성능.
8. 반대로 `B0=[0 0]`, `B1=[3 3]`처럼 한 서브매트릭스가 0이 되는 조합은 사실상 결합이 없는 독립 소형 블록 LDPC의 연쇄로, 최악 성능.
9. 결과: 제안 spatially coupled 코드는 유사 복잡도의 짧은 블록 LDPC(n=78, (3,6)-regular) 대비 AWGN BER 성능에서 유의미하게 우수.
10. 한계: 순수 컴퓨터 시뮬레이션(AWGN)에 국한, HW 구현/게이트카운트/throughput 분석 없음, NAND 채널이나 SSD 적용 언급 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Spatially coupled 부호 구조(H[L] 리본 행렬) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 base matrix 구조로, convolutional/SC 구조 자체가 다름 (대응 없음에 가까움) |
| Windowed decoding(창 이동식 반복복호) | 대응 없음 | Prime ECC는 블록 단위 전체 PCM 반복복호이며 windowed 스케줄 미보유 |
| protograph lifting(순열행렬 치환) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC circulant shift와 유사 개념이나 SC 결합 구조는 미보유 |

> 적용 가치: 낮음 — SC-LDPC 코드 구조와 windowed decoding은 Prime ECC의 고정 QC-LDPC 블록 코드 아키텍처와 근본적으로 달라 이식 난이도가 높고, 논문 자체도 AWGN 시뮬레이션만 있는 순수 이론/코드탐색 연구로 NAND HW 관련성이 거의 없음.
