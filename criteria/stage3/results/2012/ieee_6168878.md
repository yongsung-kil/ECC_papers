### B. 알고리즘 요약

1. 시스템: AWGN 채널, sum-product(최대 100 iter) 디코딩, 예시 rate 0.5, 정보길이 `k=2000~10000`, 목표길이 집합 `{0.2k,...,k}`.
2. 문제: 고정 rate에서 다양한 길이를 단일 encoder/decoder로 지원하는 length-compatible(LC) LDPC 필요; 기존 shortening/puncturing은 degree distribution이 크게 변해 성능 저하.
3. 모델: dual-diagonal(accumulator) parity 구조로 제한, degree distribution `(λ(x),ρ(x))`를 density evolution으로 설계.
4. 핵심기법: mother code를 `splitting`(puncturing 역연산: super-check node를 두 check로 분할)과 `appending`(shortening 역연산: 정보노드 추가) 으로 구성.
5. 의미: shortening/puncturing 후에도 `λ̂(S,S,x)=λ(x), ρ̂(S,S,x)=ρ(x)`(S=P)를 만족하도록 설계 → 파생 코드가 mother의 좋은 threshold 유지.
6. 확장: 목표 정보길이 집합 `{N_I(1..L)}`에 대해 Step1(PEG로 최소길이 코드)→split→append→PEG 엣지 설정 반복.
7. 구현 디테일: appended submatrix 엣지는 PEG algorithm으로 생성, dual-diagonal 구조는 puncturing 하에 super-check 병합으로 보존.
8. 최적화: shortening은 정보비트 뒤쪽부터, puncturing은 복구 스텝 최소화 순으로 순서 결정.
9. 결과: rate 0.5, k=10000에서 제안(LC-LDPC-I) noise threshold 0.70dB로 mother와 동일, 기존 PEG 방식(LC-LDPC-II)은 0.87~1.24dB로 열화; N_I=6000에서 BER=1e-4 기준 약 0.54dB 이득.
10. 한계: 순수 부호구성법, HW 미설계, 시뮬만, error-floor/디코더 성능 분석 없음.

### C. Prime ECC 관련 모듈 핀포인트

부호구성법(code-design)이나 dual-diagonal 인코딩 구조가 우리 인코더와 접점이 있어 참고용으로만 기재.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-diagonal(accumulator) parity 구조 | [encoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_encoder_4KB()` | 우리도 dual-diagonal 인코딩 사용, 개념 공유(간접) |
| mother code splitting/appending 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 생성 단계 대응이나 QC-LDPC circulant과 방식 상이 |
| puncturing 시 super-check 병합 | 대응 없음 | 우리 부호는 고정 길이, length-compatibility 요구 없음 |

적용 가치: **낮음** - dual-diagonal 개념은 공유하나 목적이 length-compatible 부호구성이라 고정 길이 NAND QC-LDPC에 직접 이식할 요소 없음.
