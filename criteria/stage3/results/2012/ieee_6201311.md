### B. 알고리즘 요약

1. 시스템: 브로드밴드 PLC(AWGN 채널, QPSK/64QAM), FEC로 다중rate QC-LDPC. rate 1/3~5/6, N=2304~9216, 정보길이 K는 rate별 상이.
2. 문제: ITU-T G.9960의 다중rate QC-LDPC는 각 rate를 독립 설계(비-nested)라 rate-compatible 하지 않고, T matrix 행수가 적어 variable node degree 선택폭·distribution 최적화가 제약됨.
3. 핵심 기법: "column extension" — c(행수) 고정, t(열수)만 rate에 따라 증가시키는 nested T-matrix. 최저rate(1/3, 24×36) matrix에서 왼쪽으로 열을 점진 추가해 상위 rate matrix 생성.
4. 열 추가는 PEG 알고리즘으로, 새 열의 shifting value는 short girth 회피하도록 설계(b=p^m-1 or p^m, p 소수 방식 활용).
5. degree distribution 최적화: variable node degree를 모든 rate에서 (2,3,12)로 통일, EXIT chart로 VND/CND 곡선 매칭 튜닝해 near-optimal 분포 탐색.
6. EXIT chart: VND EXIT 곡선 Ci와 CND 역곡선 Co^-1 사이 tunnel이 (1,1)까지 안 닫히는 최저 SNR을 threshold로 삼음(area property).
7. 부수 유연성: 확장인자 b(64 or 180) 조정으로 같은 nested 구조에서 codeword length/정보길이 요구를 맞춤.
8. 학습/최적화: 별도 학습 없이 EXIT 곡선 매칭 기반 수동 degree 탐색.
9. 결과: 설계 코드가 R=1/2,2/3,5/6에서 G.9960 대비 EXIT threshold 0.30/0.41/0.27 dB 개선, BER 10^-5에서 QPSK 0.16~0.33 dB, 64QAM(R=2/3) 0.35 dB 우수. 복호는 SPA, 최대 50 iteration.
10. 한계: HW 미설계(구현 이점만 언급), PLC AWGN 가정, 저~중rate 위주로 NAND 고rate ECC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |

| EXIT 기반 degree distribution 설계 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정, 오프라인 부호설계 도구 없음 |
| SPA 복호 (참조) | decoder.cpp LDPC_Decoder() | Prime ECC는 min-sum 사용, SPA는 미보유이나 이식 대상 아님 |
```

적용 가치: **낮음** — PLC용 rate-compatible 구성법으로 Prime ECC의 고정 고rate QC-LDPC 부호와 rate대·응용이 다르고 HW/디코더 개선 기여가 없음.
