### B. 알고리즘 요약

1. 대상: column weight 3인 regular LDPC 코드(LDPC(1008), LDPC(1057)), AWGN 채널(bipolar {±1}, 분산 σ²), 하드결정 `z_n`으로 변환 후 처리.
2. 문제: floating point 연산 없는 저복잡도 BF(bit-flipping) 계열 알고리즘은 회로 구현이 쉽지만 성능이 낮고, Planjery의 3-bit 디코딩은 BSC에서는 error-floor 성능이 좋으나 AWGN에서는 성능이 떨어짐.
3. 채널 모델: AWGN + 이진 bipolar 변환, 수신벡터 `y`의 하드결정 `z_i = 0 if y_i≥0 else 1`.
4. 핵심 기법 1단(3-bit 디코딩): variable-to-check 메시지 `β_mn=(β1,β2,β3)`, check-to-variable 메시지 `α_mn=(α1,α2,α3)` 3비트로 표현, column/row 연산과 lookup table(Table I)로 갱신.
5. 제안 확장(3BitBI/3BitBR/3BitBC): 초기값(`|y_n|<γ`이면 초기 신뢰도 낮춤), row 처리 단계(`α_mn,2`를 bias `ε_n=q`이면 감소), column 처리 단계(`β_mn,2`를 동일 방식으로 조정) 각각에 QMBF의 bias term 아이디어를 이식.
6. 핵심 기법 2단(SP+3-bit 결합): SP 알고리즘을 먼저 적용 후 코드워드가 아니면 그 출력을 3-bit(또는 3BitBI) 알고리즘의 초기값으로 넘겨 추가 복호.
7. 구현 디테일: 모든 연산이 정수/이진 연산(곱, mod-2 합, lookup table)만 사용해 floating point 불필요 — 소형 회로 구현에 적합하다고 서술(HW 설계는 없음).
8. 파라미터 결정: bias 상수 γ=0.3(3BitBI/BR/BC), α=0.5(QMBF), ρ=0.5(MWBF) 등은 실험적으로 좋은 성능을 보이도록 선택.
9. 결과: LDPC(1008), l_max=5/45 조건에서 3-bit 계열이 고SNR에서 BF/WBF/QMBF보다 우수한 FER 확인. SP+3-bit 결합 스킴은 LDPC(1057)에서 동일 BER 성능 도달에 필요한 (SP iteration + 3-bit iteration) 합이 SP 단독 대비 감소하는 것을 그래프(Fig.5)로 제시.
10. 한계: column weight 3 regular LDPC로 알고리즘이 제한되며, HW 합성/게이트카운트 없이 순수 알고리즘 시뮬레이션 수준에 그침. Toshiba/SRC(Storage Research Consortium) 감사 표기가 있으나 본문에 NAND/SSD 직접 언급은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-bit finite-alphabet 메시지패싱 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Min-Sum 대체 저비트 메시지패싱 후보이나 column weight 3 regular 코드 전용이라 Prime ECC의 VN degree 17 QC-LDPC에는 직접 이식 어려움 |
| bias term (초기값/row/column) | `decoder.cpp` `VN_Cal_HD()`, `ecc_data.h` `PARAM_LLR` | 적응형 LLR/threshold 테이블과 유사한 컨셉이나 Prime ECC는 이미 iteration별 LLR threshold 테이블을 보유해 중복 기여 |
| SP+3-bit 결합(iteration 절감) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | SP(min-sum 근사) 실패 시 저비트 후처리로 iteration 절감 아이디어는 Partial CRC 조기종료와 목적이 유사하나 별도 알고리즘 추가 필요 |
| soft-2~3bit 양자화 메시지 | `channel.cpp` `Set_LLR_Th()` | Prime ECC의 2SD/3SD soft-read 비트폭과 유사한 범위, 참고 가능 |

적용 가치: **중간** — soft-2~3bit 저비트 메시지패싱과 bias term을 통한 성능 개선, SP 결합을 통한 iteration 절감 아이디어는 Prime ECC 방향성과 맞으나, column weight 3 regular 코드 한정 실험이고 HW 미검증이라 직접 이식 시 재검증 부담이 있음.
