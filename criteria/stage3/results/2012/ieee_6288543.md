### B. 알고리즘 요약

1. 대상: binary QC-LDPC(regular/irregular), PCM은 `Q×Q` circulant permutation/null 블록 구조, 예제 rate 1/2, 길이 256/512, `Q=8/16`.
2. 문제: 짧은 블록 QC-LDPC는 short cycle이 SPA 메시지 독립성을 해쳐 성능(특히 error-floor)이 나쁨.
3. 모델: BPSK/AWGN 채널, log-domain SPA 디코더, DE 최적 degree 분포(식9, max VN degree 8)에서 QC 제약 반영.
4. 핵심기법: QC-PEG tree expansion으로 최소무게 후보 CN 집합 `N^mw,l`을 얻고, 후보가 복수일 때 SPA 디코더로 최적 edge를 선택(QC-DO-PEG).
5. 핵심식: 수렴메트릭 `CVM` — 후보부호로 AWGN 하 인코딩·SPA 복호 후 `w·|L(Ti)|`(식7~8, 부호 일치 여부 가중) 합을 SNR·메시지 평균.
6. 확장: `CVM(g)` 최대인 후보 CN을 선택(`argmax`)해 해당 circulant을 배치, 나머지 열은 하향 순환이동으로 채움.
7. 디테일: weight-2 VN만으로 된 cycle 금지 제약, 구성은 오프라인이라 전송복잡도는 일반 QC-LDPC와 동일.
8. 최적화 절차: 각 edge 배치마다 후보별 행렬역변환(인코더 유도)+행렬곱+SPA 복호 반복 → 구성비용 급증.
9. 결과: 길이 256/512 rate 1/2에서 QC-PEG 대비 error-floor 영역 BER 개선(cycle 연결성 향상), 구성시간 QC-PEG 3s vs QC-DO-PEG 348s.
10. 한계: HW 없음, 시뮬(MATLAB)뿐, 저rate·단블록만 검증, 구성비용이 커 short~medium 길이에 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC PCM / edge 배치 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 부호구조 로드에 대응, 단 Prime ECC는 특정 고rate QC-LDPC 고정이라 구성법 교체는 이식난이도 높음 |
| SPA 기반 후보 평가(디코더) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | Prime ECC는 min-sum 기반, 논문은 full SPA를 구성단계에서 사용(런타임 디코더와는 별개) |
| error-floor 개선(cycle 연결성) | 대응 없음 | 부호설계 단계 최적화, 코드베이스 런타임 모듈 직접 대응 없음 |

적용 가치: **중간** — binary QC-LDPC라는 부호족은 Prime ECC와 일치하고 error-floor 개선 아이디어(디코더 피드백 기반 edge 최적화)는 참고가치가 있으나, 저rate·단블록 예제와 오프라인 SPA-in-loop 구성비용, 고정 H-matrix 재검증 부담으로 직접 이식은 제한적.
