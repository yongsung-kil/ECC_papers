### B. 알고리즘 요약

1. 목표: 여러 통신표준(WiFi 802.11n, WiMAX 802.16e, DVB-S2/T2/C2)의 이질적 LDPC 코드를 하나의 유연한 디코더로 지원하는 ASIP 설계.
2. 문제: 전용 HW는 면적/전력 효율은 좋으나 멀티표준 재구성 불가, GPP/DSP는 유연하나 저전력 임베디드에 부적합.
3. 접근: 처음부터 프로세서를 만들지 않고 **공개 MIPS softcore(Plasma)** 재사용 + 전용 SIMD PU 매트릭스 결합 (중간형 접근).
4. 코드: IRA 계열 LDPC(sparse + staircase 하삼각), 주기성 도입으로 저장량 축소, horizontal layered decoding 채택.
5. 알고리즘: **horizontal layered offset Min-Sum** — CN 처리를 최소값 선택+offset `η`로 단순화, layered로 수렴 2배 가속.
6. 핵심식: `Em(t+1) = Max(Min(|Tnm|) − η, 0)`, `Tn(t+1) = Tn(t) + ΣEmn` (offset로 MS 단순화 손실 보정).
7. 아키텍처: Plasma 컨트롤러 + P개 PU/블록메모리로 구성된 homogeneous SIMD 매트릭스, PU는 2단 pipeline(V2C 처리 → CN sign/mag 계산).
8. 인터커넥트: PU-메모리 간 LLR 전송 병목을 **Benes 네트워크**(P-to-P permutation)로 해결, 충돌회피 위해 PU 실행 스케줄링(List-Based).
9. 설계흐름: H-matrix 분석→degree/병렬도 추정→메모리매핑(3방식)→scheduling-binding→VHDL RTL 자동생성 + Plasma 펌웨어 C코드(11개 신규 명령어 추가).
10. 결과: Virtex-6에서 WiMAX(576,288) P=24 33Mbps, DVB-T2(64800) P=64 65Mbps, (5,7,5) 양자화, 20 iter. rate 2/5에서 error floor 관측.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| horizontal layered offset Min-Sum | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | offset MS는 우리 min-sum과 유사, 이미 보유 |
| LLR (5,7,5) 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` / `ecc_data.h` `PARAM_LLR` | 통신표준용 5bit 양자화, NAND soft-read와 상이 |
| PU-메모리 Benes 인터커넥트 | 대응 없음 | 우리 구조(z=32 row병렬, HCU/GT)와 정합 안 됨 |
| MIPS softcore + SIMD ASIP 아키텍처 | 대응 없음 | Prime ECC는 전용 HW 모델, 프로세서형 아님 |

적용 가치: **낮음** — 알고리즘(layered offset MS)은 이미 보유, ASIP/Benes 아키텍처는 멀티표준 유연성 목적이라 NAND 전용 고rate QC-LDPC 엔진에 이식 부적합.
