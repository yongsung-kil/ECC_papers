### B. 알고리즘 요약 (10줄 내외)

1. 대상 부호: 3GPP rate-1/3 (7692,2560) PCCC turbo 코드, DVB-S2 rate-1/3 (16200,5400) LDPC 코드, BPSK+AWGN 채널.
2. 문제의식: turbo/LDPC의 실용 디코더(MAP/SPA)는 ML 디코더 성능에 못 미치는데, 실제 ML WEP(word error probability)를 측정할 실용적 방법이 없음.
3. 핵심 아이디어: 수신단이 스스로 생성한 side information("gift", 특정 비트 위치에 대한 사전 확률 지정)을 반복 주입해 디코더가 여러 후보 codeword를 출력하게 하고, 그중 최고 likelihood(`normalized path metric`)를 선택 → 이 값들로 ML WEP의 상한(디코더 자체 WEP)·하한(더 높은 likelihood를 가진 codeword 출현 확률)을 각각 구성.
4. 저(low) WEP 영역: 랜덤 perturbation noise `ν`를 입력신호에 더하는 perturbed decoding(PA)으로 상·하한이 수렴, ML WEP를 정확히 추정 가능.
5. 고(high) WEP 영역: PA의 계산량이 과도(perturbation 횟수 5만~20만회로도 한계)해지므로, 정답 codeword의 특정 비트값을 직접 지정하는 gift 방식으로 전환. gift 크기가 `trigger threshold`(turbo 10, LDPC 24)를 넘으면 `normalized path metric`이 선형적으로 상승하는 성질을 이용.
6. Gift-assisted decoding algorithm (GDA): 사후확률이 `[0.25, 0.75]` 구간(불확실 비트)에 속하는 위치를 우선 선택하는 greedy/(+b,-c) grow-up 절차(ant colony optimization과 유사한 반복 성장)로 매칭 gift를 점진적으로 구축.
7. 복잡도 절감: 무작위 gift 매칭 시 필요한 반복 횟수를 기존 대비 약 6자리수(10^6배) 줄여, gift 크기 20~24 수준에서 실용적으로 매칭 gift 탐색 가능.
8. 파라미터: turbo/LDPC 각각 초기 gift pool `Nini`=10^7/2.5×10^6, 반복 디코딩 18/100회, appending/removing bit `b=3,c=1` 등 GA 유사 파라미터로 GDA 튜닝.
9. 결과: WEP 10^-4~10^-5 영역에서 turbo/LDPC 모두 상·하한이 수렴해 ML WEP 확정 — best message-passing 디코더(MAP/SPA)가 ML 대비 최소 0.15~0.2 dB 열등함을 정량 확인.
10. 한계: 순수 시뮬레이션/사후분석 기법으로 실시간 디코더 구조가 아니며, 고 WEP 영역에서도 여전히 10^6회 이상의 반복 gift 탐색이 필요해 HW 구현이나 NAND 적용과 무관.

| Gift-assisted decoding (ML WEP 벤치마킹) | 대응 없음 | 실제 디코딩 알고리즘이 아닌 오프라인 성능 상한 측정 기법 |
| Perturbed decoding / multiple-output decoding | 대응 없음 | Prime ECC는 단일 출력 min-sum 반복복호 구조로 다중출력 개념 미보유 |
| DVB-S2 LDPC 부호 자체 | `ecc_top.cpp` `Load_PCM()` | 참고 부호 스펙일 뿐 논문 기여가 아니며, non-QC/방송용 부호로 이식 무의미 |

> 적용 가치: 낮음 — 논문은 ML 성능 상한을 시뮬레이션으로 추정하는 분석 방법론이지 실전 디코더/HW 기법이 아니므로 Prime ECC(실시간 HW 디코더)에 이식할 요소가 없음.
