### B. 알고리즘 요약

1. 시스템: LDPC (2016,1512) inner code + 외부 binary BCH (756,x) 2병렬 unit을 연접, AWGN 채널 BPSK, FPGA hardware emulation으로 BER<1e-10 측정.
2. 문제: LDPC 디코더는 trapping/absorbing set 때문에 저노이즈 error-floor가 남아 conventional LDPC/BCH 단독으로 정정 불가.
3. 모델: iterative min-sum 계열(NMS a=0.75, OMS b=0.5) 디코더가 수렴 실패 시 oscillatory 거동을 보인다고 가정.
4. 핵심기법 1: 외부 BCH가 LDPC가 남긴 소수 오류를 정정 - `c(x)=m(x)x^{n-k} - R_g[m(x)x^{n-k}]` systematic encoding, iBM + Chien search 복호.
5. 핵심식: `S_j=r(α^j)` syndrome 계산으로 오류 검출, iBM `d=ΣΛ_i S_{k-i}`로 error locator polynomial 산출.
6. 핵심기법 2: Syndrome-based(SB) LDPC 디코더 - 각 iteration에서 unsatisfied check 수(syndrome weight)가 최소인 추정 codeword를 출력으로 선택, oscillation 보상.
7. 부수 트릭: BCH를 q개 소형 병렬 unit(n/q)으로 분할, `d(q,c,n)` 식으로 병렬 수 q 결정 → 동일 면적에서 평균 정정능력↑, throughput↑. 모든 컴포넌트 shortening 지원.
8. 최적화: SB post-processing으로 LDPC 잔여 오류를 t 이하로 줄여, 더 작은 t의 BCH(BCH-1)로도 동등 성능 달성 → BCH HW 축소.
9. 결과: BCH-SB-LDPC 조합이 저노이즈에서 0.8~1dB coding gain (단독 LDPC/BCH 불가), Virtex5 FPGA에서 BCH-LDPC ~94Mb/s, Multiple decoder 대비 면적 효율적.
10. 한계: 연접으로 code rate 하락 → Eb/No 도메인 waterfall 손해, LDPC 부호구조 상세 미기술, latency는 pipeline으로 거의 2배 증가.
