### B. 알고리즘 요약

1. 대상: binary protograph 기반 QC-LDPC, 예제 (3,4)-regular Tanner code 계열, rate ≈0.255, 길이 n=124~496.
2. 문제: 1단 circulant lifting QC-LDPC는 min Hamming distance가 `(nc+1)!`로 상한(binary base면), 이 한계가 error-floor를 유발.
3. 핵심 기법: 2단 리프팅 - (1) protograph를 m-fold graph cover로 pre-lift해 `B'`(m×m permutation), (2) `B'`를 r-fold circulant lifting해 block-circulant `P_i,j = diag(I_{p_1}..I_{p_m})·B̃'_i,j` (식2).
4. 핵심 결과: `H'∈ξ_B(mr)`이지만 `ξ_B^QC(mr)`엔 없어 min distance가 `(nc+1)!` 초과 가능 (near-codeword 형태로 초과 입증).
5. 정리4: block-circulant들이 모두 commute하면(각 P_i,j의 shift 동일) min distance는 여전히 `(nc+1)!` 상한 → 이를 피하려 non-commuting 유도.
6. Design Rule 1: pre-lifting을 circulant permutation으로 하되 shift를 다르게(σ에 fixed point 없게) 하여 non-commuting 확보.
7. Design Rule 2: pre-lifting을 non-commuting permutation으로 하고 각 block-circulant엔 동일 shift 사용(state-of-the-art QC code shift 재활용).
8. girth 축소: Lemma1/Thm2로 `girth(B')≥g`면 `girth(H')≥g` 보장 → 42개 girth-8 조건을 m=5시 4개로, m=9시 0개(무조건 girth 8)로 축소.
9. 결과 수치: Example2 [392,100,24]→Rule1 적용시 dmin 32~56로 증가, girth 10; Example3 [496,126] dmin 28~68; BPSK/AWGN 100 iter syndrome-check 복호에서 pre-lifted 코드가 BER 1e-5에서 SNR 1dB↑ 이득.
10. 한계: HW 미설계, AWGN 시뮬만, 저rate short-block 예제, block-circulant 구조로 디코더 라우팅 복잡도 증가 가능.
