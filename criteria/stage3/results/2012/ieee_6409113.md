### B. 알고리즘 요약

1. 대상 부호는 IEEE 802.11n WLAN QC-LDPC (`n=1296`, parity `432`, rate `2/3`, sub-block `Z=54`), regular (`Wr=8`, `Wc=12`).
2. 문제: 기존 sum-product 복호는 매 iteration마다 모든 노드에서 `rij`/`qij`를 재계산해 연산량·면적·전력이 큼.
3. 관찰: 높은 LLR을 가진 노드는 매 iteration에서 거의 동일한 메시지를 전달 → "static node"로 간주 가능.
4. 핵심 기법: static node 감지 시 sparse H와 동일 크기의 buffer를 갱신, 해당 노드는 이후 iteration에서 계산 제외(캐시값 사용). iteration이 진행될수록 static 노드가 늘어 연산량 감소.
5. static 판정: 노드 메시지가 사전 정의 threshold를 넘으면 threshold로 clip하고 이후 계산에서 배제(row/col weight는 유지).
6. `rij`는 식(2) `phi()` piecewise-linear 근사 + LUT/선형보간으로 구현, `qij`는 식(3).
7. HW: semi-parallel 아키텍처 - 4 row processing unit(각 108행), 8 column processing unit(각 162열), 11-bit fixed point, block RAM 분리 메모리.
8. syndrome check 개선: `HV^T` 이진 곱셈 대신 sparse H의 '1' 위치에서 lookup·even/odd 카운트로 modulo-2 합 계산 → 연산 대폭 축소.
9. 결과: throughput 522 Mbps(표준 SPA 90 Mbps 대비), 연산량 저SNR 20.8%·고SNR 87.6% 감소, slice count 감소, BER는 소폭 증가.
10. 한계: FPGA 구현·시뮬만, static clipping의 BER 열화 정량치 본문 부재, WLAN 단일 부호에만 검증(다양한 길이/표준은 future work).
