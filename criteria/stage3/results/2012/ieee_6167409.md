### B. 알고리즘 요약

1. 시스템은 이중반복(double-iterative) MIMO-BICM: MIMO 검출기와 LDPC 디코더가 반복적으로 extrinsic 정보를 교환.
2. 블록길이 `N=3720~14880`, 코드율 `RC=1/2~4/5`인 유연한 QC-LDPC 부호를 지원해야 함(UMIC 규격, OFDM 프레이밍).
3. 채널모델은 MIMO 페이딩(`yt = Ht·st + nt`, 복소 가우시안 `Ht`)이며 목표 시스템 처리율은 200Mbit/s이지만 4회 외부 피드백×5회 내부 반복 시 LDPC 디코더 자체는 Gbit/s급 처리율 필요.
4. 디코딩은 정규화 Min-Sum 알고리즘(`λm→n = α·sgn(...)·min(|λn→m|)`, `α=0.75~0.875`)과 TDMP(layered decoding, Gauss-Seidel식 갱신)를 사용.
5. 처리율 근사식 `#bits/cycle ≈ P·R/(iter·dVN)`으로 필요 병렬도 `P`를 역산 — `dVN≈3.2` 가정 시 목표 처리율 만족을 위해 `P=256` 도출.
6. 부호-아키텍처 공동설계: 서브행렬 크기 `I=93`으로 고정하고 3개 서브행렬을 동시 처리(`P=3·93=279`)해 짧은 블록에서도 multi-diagonal 문제 회피.
7. Layered 구조 특유의 variable node 블로킹 구간 `B = 3 + dCN/3` 을 고려해 동시에 처리되는 행 그룹이 blocked column과 겹치지 않도록 PCM 배치를 설계(PEG 기법으로 순열 결정).
8. 짧은 블록은 `VNmax=5`, 7440비트 이상 블록은 `VNmax=15`로 제한하는 등 layered 제약이 부호 설계에 직접 반영됨.
9. 65nm P&R 결과: 면적 4.588mm2, 275MHz에서 1271mW, LDPC 디코더 처리율 3.5Gbit/s(5회 반복 기준), 전체 시스템은 275Mbit/s.
10. 한계: BER/통신성능 시뮬레이션 결과는 지면 제약으로 미제시, MIMO/무선 채널 전용 설계로 NAND read 채널 모델과 무관.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 정규화 Min-Sum + layered(TDMP) 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 min-sum/layered 계열 기법과 개념적으로 유사, 신규 기여 아님 |
| 서브행렬 병렬처리 및 blocking 제약 기반 PCM 설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체가 특정 무선 표준(UMIC) 규격에 종속되어 이식 난이도 높음 |
| soft-info(다중 비트 LLR) | `channel.cpp` `Set_LLR_Th()` | AWGN/MIMO 기반 연판정으로 NAND soft-read 모델과 상이, 참고용 |
