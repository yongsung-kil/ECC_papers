### B. 알고리즘 요약

1. 대상: `(J,L)` regular QC-LDPC, `p×p` circulant permutation matrix, base matrix `B`가 cyclic shift 값 `pj,l`을 저장 (예제 `J=3, L=10, p=765`, `N=7650`, rate `7/10`).
2. 문제: girth를 키우면 small trapping/stopping set이 줄어 복호성능 향상되나, 기존 hill-climbing 구성법은 실패 가능성이 높고 computer search가 느림.
3. 기준식: Fossorier의 girth≥`2i+1` 필요충분조건 `Σ Δjk,jk+1(lk) ≡ 0 (mod p)`을 위배시키는 shift 값을 찾는 문제로 환원.
4. 1단 - cycle 통계 알고리즘: 각 element `pj,l`가 만드는 4/6/8-cycle 개수를 행조합별로 세어 `N^i` 행렬 산출. GPU에서 행조합마다 block, 열마다 thread를 배정(4/6/8-cycle에 `L`, `L×L`, `L×L×L` thread).
5. 판정: `A−C+D−B`(4-cycle), `A−C+D−E+F−B`(6-cycle) 등이 `p`의 배수면 cycle 존재로 기록.
6. 2단 - shortened cycle elimination: path cost matrix `A`(식 8, `aj,l,z`=값 `z`로 바꿀 때 잠재 path 수)를 계산, cycle 수가 최대인 element를 한 번에 하나씩 최적화.
7. 수렴 트릭: 어떤 값으로도 short cycle을 못 없애면 해당 column을 통째로 삭제(shortened)하여 알고리즘 수렴 보장.
8. 학습/최적화: local optimal greedy(매 스텝 최대 cycle element 선택 후 재계산), array code 아이디어 결합.
9. 결과: girth 6/8/10 base matrix 예시 제시, `pmin` 상한이 random/hill-climbing 대비 소폭 개선(예 `L=12`에서 54→52); girth-12로 확장 가능하다고 주장.
10. 한계: 구성법만 제시 - 디코더·HW 미설계, BER/error-floor 실측 없음, 속도(GPU 가속) 이득도 정량 수치 미제시.
