### B. 알고리즘 요약

1. 시스템: BEC 상의 causal streaming 전송, 정보 시퀀스 `u[1,t]`가 시간에 따라 증가, 매 시각 code block `vt = E(u1..ut)` 방출. rate `R=1/2`.
2. 문제: delay-sensitive real-time(control-over-network)에서 신뢰성과 지연 제약이 상충 → anytime 신뢰성(block `j`의 erasure 확률이 decoding delay `d=t-j`에 대해 지수 감쇠) 필요.
3. 모델: LDPC Convolutional Code(LDPC-CC), lower-left band-triangular H로 causal 인코딩 가능, non-terminated·메모리 증가형.
4. 핵심기법 1단: protograph base matrix `B[1,t]`(식 8)로 반무한 LDPC-CC 앙상블 정의, `B0=[1 1]`, `i>0`에서 `Bi=[x 0]`, systematic 구조(짝수 인덱스 VN은 degree-1).
5. 핵심기법 2단: expanding-window MP 디코더 — 새 블록 도착 시 1~t 전체 수신 시퀀스에 MP 수행, 창을 계속 확장. sliding-window[8] 대비 lift factor `M` 작게 유지.
6. 분석: P-EXIT 알고리즘(식 10~13, BEC에서 multi-edge DE와 등가)을 이 protograph/디코더에 맞게 수정, 상호정보 `IEv,t`,`IAv,t` 갱신.
7. 핵심 결과: `IE∞v,t → 1` (t·iteration 증가 시), `PAPP,t(j) ≈ ε·PAPP,t(j+2)` (식 32) → 정보노드별 erasure 곡선이 shift된 형태로 동일.
8. anytime exponent: `PAPP,t+1(j) = ε·PAPP,t(j)`로 지수 감쇠 확인, `α = -log(ε)` (일반적으로 `α = -x·log(ε)`).
9. 결과 수치: `ε=0.3`, `x=1`에서 P-EXIT 슬로프가 이론 exponent `-log(ε)`와 정확히 일치, `x=3`에서 `-3log(ε)` 확인(Fig.3,4).
10. 한계: BEC·무한 블록길이·무한 iteration 점근 가정, HW/유한길이 구현 미포함(후속 논문 예고).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-CC protograph 구성 (`B[1,t]`) | 대응 없음 | Prime ECC는 QC-LDPC 블록 부호(`Load_PCM()`), convolutional 아님 |
| expanding-window MP 디코더 | 대응 없음 | Prime ECC 디코더는 고정 블록에 대한 min-sum 이터레이션이며 창 확장 개념 없음 |
| P-EXIT / density evolution 분석 | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터, DE 분석 도구 아님 |
| BEC anytime 신뢰성 이론 | 대응 없음 | NAND 채널(AWGN/RBER, `channel.cpp`)과 무관 |

적용 가치: **낮음** — BEC 전용 LDPC 컨볼루셔널 부호의 anytime 이론 증명으로, NAND용 binary QC-LDPC 블록 부호의 부호설계·디코더·HW 어디에도 이식 요소가 없다.
