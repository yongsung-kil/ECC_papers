### B. 알고리즘 요약

1. 시스템: AA-LDPC(구조화 LDPC) BPSK/AWGN, TDMP(층별) 복호, FPGA(VirtexII) 디코더로 검증. 부호 (1056,816) regular, (2048,1024) irregular.
2. 문제: TDMP는 빠른 수렴·저메모리 장점이 있으나, 유한정밀(fixed-point)에서 특유의 overflow 오류로 성능이 붕괴.
3. 핵심 관찰: 다음 iteration에서 `ρ = γ̃ - λ` 계산 시 "포화된(overflowed) posterior γ̃"에서 "비포화 intrinsic λ"를 빼는 연산이 오류원 - turbo형 dataflow 고유.
4. 핵심기법 1단(Stage I): posterior 업데이트당 발생하는 overflow 오류 `ε`를 additive 성분으로 모델링, 주성분 `ε^{m1}`을 수정 extrinsic `[λ]ALT = λ + ε`로 흡수해 제거(SISO당 뺄셈기 1개 추가).
5. 핵심식: `[λ]ALT = Sat_Z(λ + ρ) - ρ = λ + ε` → 다음 iter의 `ρ̃`에서 동일 subiteration 유래 overflow 성분 소거.
6. 핵심기법 2단(Stage II): 잔여 overflow 오류를 통계적으로 근사 보정 - "Offset ρ"(γ=±Z일 때 상수 C 가감) 또는 "Normalized ρ"(λ에 A<1 곱).
7. 부수 트릭: `A=0.5` 선택 시 곱셈이 bit-shift로 HW 불필요. "Limited λ"(λ 저장범위를 [-Mλ,Mλ], Mλ≈Z/2~3Z/4로 제한)로 overflow 빈도 자체 감소.
8. 최적화: 실험적으로 `C=0.19Z`, `A=0.5`, `Mλ=3Z/4`를 최적/근최적으로 도출(성능 민감도 낮음). CNU는 BCJR/Min-Sum+비선형보정.
9. 결과: W≥6에서 full-precision에 근접, W=8은 일부 full-precision 상회(클리핑 효과). FPGA 합성 오버헤드 ~5% slice(RAM블록 불변).
10. 한계: layered/TDMP 전용(two-phase에는 문제 없음), throughput/gatecount 본문 수치 없음, 특정 wordlength/부호에 튜닝된 상수.
