### B. 알고리즘 요약

1. 시스템: 2b/cell(및 3b/4b) NAND 기반 SSD, LDPC ECC를 NAND 컨트롤러에 내장, 코드율 `CR=k/n` 2/3~9/10 범위로 실측.
2. 문제: 기존 LDPC는 정확한 `VTH` 정보 위해 최대 21 Vref 필요 → 50μs×21=1050μs로 read 지연이 과도(3b/4b cell은 2×/5× 악화).
3. 핵심 아이디어: 3 Vref만 쓰는 error-prediction LDPC(`EP-LDPC`) — 부족한 VTH 정보를 write/erase cycle `NW/E`와 retention time `TRetention` 기반 BER 예측으로 보완.
4. Step1~2: lower page의 "1" 개수 `N"1"measured`를 세고 write 시 저장한 `N"1"initial`과의 차로 `BERLower = |N"1"measured − N"1"initial|/NP` 추정 (retention은 N"1"↑, program-disturb은 N"1"↓).
5. Step3~5: `BERLower`·`NW/E`로 `TRetention` 추정 → pre-recorded EP table로 inter-cell coupling 추정 → 각 데이터 BER 추정 후 그 BER를 LDPC 디코더에 전달해 복호.
6. 테이블은 출하 전 test 과정에서 flash에 저장, 예측 로직은 단순 logic gate로 컨트롤러 면적 증가 무시 가능.
7. Error-recovery(ER): 복호 실패 시 error mode 판정 후 program-disturb엔 PDRP(CG-IPD 계면 전자 de-trap, VTH↓), retention엔 DRRP(FG 전자 주입, VTH↑) 펄스 인가 후 재복호.
8. 결과(실측): CR=2/3에서 lifetime 11×, 9/10에서 2.8× 연장, 허용 BER 3.7×/1.8×↑, PDRP로 program-disturb 76% 회복, DRRP(500회)로 retention 56% 회복.
9. HW: LDPC gate count는 기존 LDPC 수준, BCH의 2배(수치 미기재), read는 기존 LDPC 대비 7× 빠름.
10. 한계: LDPC 디코더 내부 알고리즘·부호 구조·양자화 미기술, 기여가 채널 BER 예측 및 셀 레벨 recovery pulse에 집중(디코더 이식 요소 부재), hard-read 기반.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BER/retention 예측 후 디코더에 BER 전달 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()`, `Set_LLR_Th()` | 예측 BER로 soft-read threshold/LLR 초기화하는 아이디어 참조 가능 |
| N"1" 카운트 등 pre-processing 로직 | 대응 없음 | Prime ECC는 채널 이후 복호만 담당, 셀 read/예측 전처리 계층 없음 |
| PDRP/DRRP 셀 레벨 recovery pulse | 대응 없음 | NAND 셀 물리 프로그래밍 영역으로 ECC 코드베이스 범위 밖 |
| LDPC 디코더 알고리즘 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 논문이 디코더 내부를 기술하지 않아 직접 대응 불가 |

적용 가치: **낮음** — NAND 직접 타깃이나 기여가 채널 BER 예측과 셀 물리 recovery에 있어 디코더/부호 이식 요소가 없고, LLR 초기화 아이디어 정도만 간접 참조 가능.
