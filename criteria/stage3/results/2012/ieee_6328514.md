### B. 알고리즘 요약

1. 시스템: binary LDPC(IEEE 802.11n), BPSK, AWGN 채널, 수신값을 quantized LLR(`QLLR` `ξ`)로 변환.
2. 문제: min-sum BP가 trapping set에 빠져 실패하면 iteration을 늘려도 못 벗어나므로, 총 iteration을 줄이며 성능 유지가 목표.
3. 2단 구조: 1단 min-sum BP를 `I_BP` iter 수행, `ĉH^T=0`이면 성공 종료.
4. 실패 시 2단: BP 출력 `ĉ` 중 `|QLLR|`가 가장 낮은 `X`개 좌표를 인공 BEC로 erasure(`Δ`) 처리해 `z` 생성.
5. erasure iterative 복호: parity-check가 단일 erasure를 포함하면 해당 비트를 정정, 반복.
6. 정정된 비트가 원래 `ĉ`와 다르면 해당 `QLLR` 부호를 반전해 `ξ`를 갱신, 다음 HD cycle의 BP 입력으로 사용(`I_HD` cycle 반복).
7. `X` 선택이 핵심: `X≤s(H)`(stopping distance)면 모든 erasure 정정 가능하나 너무 작으면 오류를 못 덮음. 코드별 최적 `X`는 시뮬로 결정(예 (648,432)→70).
8. density evolution으로 ε=X/N에 대한 erasure 복호 최소 iteration 산출.
9. 결과: (648,432)서 2-cycle HD(총 50 BP iter)가 200-iter BP 대비 error-floor서 0.2dB 이내, (1296,864)는 2-cycle·25 iter로 동등 성능 → 훨씬 적은 iteration으로 등가 WER.
10. 한계: HW 미설계, 시뮬만, 코드별 `X` 수동 튜닝 필요, stopping set 의존.
