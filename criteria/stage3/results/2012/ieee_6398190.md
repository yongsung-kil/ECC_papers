### B. 알고리즘 요약

1. 대상: irregular LDPC 부호 `H (M×N)`, rate `1/2`, 매우 짧은 길이 `(8,4)~(160,80)`; AWGN + BPSK, BP(belief propagation) 복호.
2. 풀려는 문제: 유한길이 LDPC의 error floor와 인코딩/설계 계산복잡도를 낮추는 좋은 H 구성 (특히 length-4 cycle 제거).
3. 배경: Tanner graph의 4-cycle이 BP 수렴/독립성을 해쳐 BER/FER를 악화시킨다는 잘 알려진 사실을 근거로 삼음.
4. 핵심 기법: 주어진 길이에 대해 H를 랜덤 생성 → 연관 Tanner graph에서 관찰된 length-4 short cycle 제거 → 각 행렬 성능 평가 → 최선 성능 행렬을 최종 H로 채택.
5. 절차 의미: 랜덤 후보 중 4-cycle 없는 것들의 성능을 비교해 고르는 방식이라, 별도 최적화 이론 없이 "생성-필터-선택" 구조.
6. 비교 기준: 동일 길이로 랜덤 생성했으나 short cycle이 남아있는 행렬과 BER/FER/CPU time을 비교.
7. 구현: MATLAB 플랫폼, all-zero codeword 전송, 고정 iteration·block-in-error 수 기준으로 시뮬.
8. 학습/최적화: 명시적 girth 제약이나 학습 절차 없음 (랜덤 + cycle 제거 heuristic).
9. 결과: 예로 `(40,20)`에서 BER `0.1046→0.0669`, FER `0.822→0.506`, CPU time `1.43→1.11s`; `(60,30)`에서 CPU `3.30→1.92s` 등 제안법이 전반적으로 우수(Table I).
10. 한계: 부호가 극단적으로 짧고(≤160bit) MATLAB 시뮬만, 알고리즘 서술이 얕아 재현 세부(제거 방법·선택 기준) 불명확, HW·QC 구조·gate/throughput 전무.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| irregular H 랜덤 생성 + 4-cycle 제거 (부호구성) | ecc_top.cpp Load_PCM() (개념적 참조) | Prime ECC는 확정된 binary QC-LDPC를 로드만 함 - irregular 비-QC 부호 구성법이라 직접 대응 함수 없음 |
| 4-cycle 제거로 error-floor/BER 개선 | 대응 없음 | Prime ECC 부호는 이미 girth 설계 완료된 고정 QC-LDPC - 이 heuristic이 닿을 실행모듈 없음 |
| BP 복호 성능평가 | decoder.cpp LDPC_Decoder() (개념적 참조) | Prime ECC는 min-sum 기반이며 논문은 알고리즘 변경이 아닌 부호선택이라 코드 이식 대상 아님 |
```

적용 가치: **낮음** - 부호가 극단적으로 짧은 irregular(비-QC)이고 "랜덤 생성+4-cycle 제거+최선 선택"이라는 교과서 수준 heuristic이라, NAND용 고rate binary QC-LDPC인 Prime ECC에 떼어 쓸 신규 알고리즘/HW가 없음.
