### B. 알고리즘 요약

1. 대상 부호: IEEE 802.15.3c irregular LDPC `(672,336)` - CN 672, BN 336, edge 2184 (본문 표기 기준, rate≈0.5).
2. 문제: irregular LDPC는 노드별 edge 수가 달라 GPGPU 병렬화 시 메모리 접근 패턴이 불규칙해 coalescing이 어렵고 성능 저하.
3. 채널/모델: BPSK + AWGN, `r_i = x_i + n_i` (식1,2), noise 분산 `sigma^2`.
4. 핵심 알고리즘: Two-Phase Message-Passing min-sum - horizontal(`Lc_j->x_i` = sign 곱 * min) / vertical(`Lx_i->c_j`) / posteriori / hard decision / syndrome 종료(식은 Fig.4 pseudo code).
5. 데이터구조: `Lxc`, `LxcBegin`, `LxcSize`, `LcxPosition` 배열로 irregular edge를 표현 - coalescing과 data reuse를 트레이드오프.
6. irregular 접근은 여러 cache line을 L1으로 읽어 evict 전에 최대한 재사용, Fermi의 L1(최대48KB)/L2(768KB) 계층 활용해 spatial locality 극대화.
7. 병렬화 1: 노드당 1 thread 생성(one thread per node), horizontal/vertical 각 phase 처리.
8. 병렬화 2: 다중 코드워드 동시 복호(GPU_MC=16 codeword, 10752 thread)로 Tesla C2050 448코어 활용률 극대화.
9. host-device 통신 최소화: `Lint` 배열 1회만 복사, iteration마다 syndrome 정수 1워드만 회신.
10. 결과: iter20 기준 CPU 355.82s → GPU_MC 6.39s, 최대 55.68X / 평균 46.85X 가속. HW 미설계·throughput(Gbps) 미기재.
