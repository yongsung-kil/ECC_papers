> 총평: TLC NAND 실측 오류특성을 규명하고 셀레벨 Z4+binary 연접 ECC를 제안한 NAND 직접 논문이나, 신규부호가 LDPC 디코더/QC-LDPC 부호설계가 아니어서 Prime ECC 이식성은 하이며 채널·오류모델 인사이트로만 참고.

### B. 알고리즘 요약

1. 시스템: TLC(3bit/cell) NAND, cell=8레벨, 셀당 MSB/CSB/LSB 3페이지가 서로 다른 페이지로 분리 저장, 블록당 384페이지, 페이지 2~8KB.
2. 문제: MLC/TLC 미세화로 Hamming/CRC로는 부족, 강한 ECC(BCH/LDPC)가 필요하며 TLC 실측 오류특성 기반 설계가 요구됨.
3. 실측 모델: 수천 회 P/E 사이클로 오류 DB 구축, 채널은 BSC(crossover `p`=실측 평균오류확률)로 근사. 관측: (1)셀이 오류나면 高확률로 3비트 중 1비트만 오류, (2)비트오류확률이 target cell state와 무관.
4. 핵심 기법: 셀레벨 연접 ECC - `C1`은 `Z4` 상 `t1`-오류정정 부호, `C2`는 binary `t2`-오류정정 부호(`t1>t2`), 매핑 `φ:{0,1}^3->Z4`로 3비트 triplet을 Z4 심볼로 변환.
5. 핵심 식/의미: type-1(1비트)/type-2(2비트)/type-3(3비트) 셀오류를 분류. `wH(u-u')=e1+e2`(Lemma1), 매핑 `ψ` 적용 후 `wH(eMSB)=e2+e3`(Lemma2) - 1비트 오류를 Z4 단에서 흡수해 binary 단 부담 감소.
6. 정정능력: `e1+e2<=t1` 且 `e2+e3<=t2`이면 3페이지 정확 복호(Theorem 3) - 우세한 type-1 오류를 저비용 Z4 코드가 담당.
7. 인코딩: pMSB에서 C2 여분 `s2` 저장 -> `u=φ(pMSB,pCSB,pLSB)` -> C1 여분 `s1`(Z4) 저장. 디코딩은 C1로 Z4 오류 e 추정 후 ψ 적용, C2로 MSB 잔여오류 정정.
8. 비교 baseline LDPC: Gallager (3,k)-정규(len 216bit, rate 0.8/0.9/0.925), protograph PCC(rate 4/5), AR4JA(CCSDS, rate 4/5, len 1280/5120/20480), MacKay DJCM-3/4(col weight 3/4, no 4-cycle) - SPA float 소프트 시뮬.
9. 결과(수치): 8500 P/E 이후 대부분 LDPC가 BCH 우월, rate~0.9/0.925에서 신규 ECC가 BCH·Gallager·DJCM-3/4 모두 상회(rate 계산 C2=16·t2, C1=18·t1 여분 가정).
10. 한계: 신규부호 HW 미설계, LDPC는 비교용 float 시뮬뿐, C1(Z4) 효율적 구성은 미제시(sphere-packing 보수적 가정), 소수 블록 실험으로 블록 간 변동성 미반영.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TLC 셀레벨 오류특성 / BSC crossover 모델 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | 실측 오류통계를 LLR/RBER 채널모델에 반영할 인사이트 |
| Z4 C1 + binary C2 셀레벨 연접 ECC | 대응 없음 | Prime ECC는 binary QC-LDPC 단일, 비-LDPC 연접 부호 미지원 |
| 비교 LDPC(Gallager/protograph/AR4JA/DJCM) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` (간접) | 부호족이 QC-LDPC와 상이, 직접 로드 대상 아님 |
| SPA float 복호 | 대응 없음 | Prime ECC는 min-sum HW 근사, full-tanh SPA 미사용 |

적용 가치: **낮음** - NAND 직접 논문이나 신규 ECC가 셀레벨 Z4/binary 연접이라 Prime ECC의 QC-LDPC 디코더/부호설계에 이식할 요소가 없고, TLC 오류특성·채널 모델 인사이트 정도만 참고 가치가 있다.
