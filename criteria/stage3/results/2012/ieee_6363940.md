### B. 알고리즘 요약

1. 시스템: binary-input ternary-output channel(BITOC, 출력 `{+1,0,-1}`, 파라미터 `p1`(erasure 확률), `p2`(flip 확률))을 가정하고, 3-level quantized BP 디코더(메시지값 `{-1,0,1}`)를 정의.
2. 문제: 기존 error-floor 원인 구조(stopping set은 BEC, fully absorbing set은 BSC/bit-flipping decoder 기반)가 실제 AWGNC 위의 floating-point BP 디코더 동작을 충분히 반영하지 못함.
3. 핵심 가정: BITOC은 AWGNC를 3레벨로 양자화한 근사 채널이며, all-one 코드워드 전송을 가정해 에러패턴을 분석.
4. 핵심 기법 1단: 두 종류의 새로운 EPS 정의 - 1-shot EPS(`m_vi^(1) = m_vi^(0)` 이고 최소 하나는 오류, 즉 1회 반복 후에도 결정이 안 바뀜)와 static EPS(모든 반복 `l`에서 메시지가 고정, 식 (8),(9)).
5. 핵심 식 의미: 목적함수 `Obj(s) = log((1-p1-p2)/p2)·Σx_i + log((1-p1-p2)/p1)·Σy_i` (식 10)를 최소화하는 것이 수신 채널출력의 최대우도(가장 그럴듯한 에러패턴)를 찾는 것과 동치.
6. 핵심 기법 2단: 3-level 메시지 `{-1,0,1}`을 이진 변수쌍 `(x_i,y_i)`, `(z_l,u_l)`로 인코딩해 `sgn`/parity-sum 같은 비선형 연산을 선형 정수계획(IP) 제약으로 변환, branch-&-bound로 목적값 `smax` 이하 EPS를 전수 탐색.
7. 부수 트릭: 각 branch 단계에서 relaxed mixed-IP(완화된 IP)로 하한값 `b`만 계산해 효율을 높임; compatibility 조건(`s0`와 호환)으로 탐색 공간을 분기.
8. 최적화 절차 없음(학습이 아닌 exhaustive search 알고리즘, DE/GA/NN 미사용).
9. 결과: (3,6) regular LDPC(M204, M504) 및 PEG형 코드(PEGR504, length 204/504)에 적용해 1-shot/static EPS와 기존 FAS의 대응관계를 확인(예: type-(-1,0,0,0,0) 1-shot EPS 28개가 PEGR504의 (5,3) FAS 14개와 대응). FER 시뮬레이션(5-bit fixed point decoder, 10^-9~10^-13 영역)으로 동일 위상의 EPS라도 error-floor 기여도가 크게 다름을 확인.
10. 한계: 코드를 개선하는 신규 구성법이 아니라 진단/열거 도구이며, 저자 구현은 기존 SS/FAS exhaustive 탐색보다 느려 코드 설계 단계에서만 사용 가능. HW 구현이나 실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-level(soft-2~3bit) quantized BP 디코딩 메시지 정의 | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD), `decoder.h` `Get_VNU_*` | 3레벨 양자화 메시지 표현이 2SD 계열 VN 처리와 유사한 개념이나 직접 대응 함수는 없음 |
| 1-shot/static EPS(trapping-set류) exhaustive 탐색 | 대응 없음 | Prime ECC는 시뮬레이터로 이런 오프라인 코드 진단(IP solver 기반) 모듈이 없음 |
| H-matrix/부호 구조에 대한 error-floor 원인 분석 | `ecc_top.cpp` `Load_PCM()` | H-matrix를 로드하는 지점은 대응되나, EPS 탐색 자체는 별도 오프라인 도구로 이식해야 함 |

> 적용 가치: 중간. 정정능력을 직접 개선하지는 않지만 NAND용 QC-LDPC H-matrix의 error-floor 원인을 진단하는 오프라인 코드설계 도구로서, 3레벨 soft decoding 환경(2SD와 유사)에 특화된 trapping-set 분석 방법론은 Prime ECC의 코드 검증/설계 단계에 참고할 가치가 있음.
