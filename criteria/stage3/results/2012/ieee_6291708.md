### B. 알고리즘 요약

1. 대상: 초저rate(`R=1/10`, `1/6`) binary sparse-graph 코드(TLDPC 앙상블), AWGN/BEC, base code+bipartite graph 일반 구성, 반복복호.
2. 문제: 저rate에서 표준 LDPC는 성능 나쁨 — 좋은 iterative threshold와 linear `dmin`(error-floor 방지)을 동시 만족하는 저rate 부호 일반 설계법이 없음.
3. 모델: sparse-graph 코드를 base code `B`(rate `Rb`, 길이 `m`) + degree 분포 `Λ`의 bipartite graph `G`로 일반 표현(`R=1-(1-Rb)λ̄`).
4. 핵심 기법1: "partial weight 2 codeword 그래프" `G`를 정의, 그 평균차수 `Δ≤2`가 linear `dmin`의 필요조건(초과 시 `dmin=O(log n)`)(Theorem 1).
5. 핵심 식 의미: LDPC 예로 `λ̂2·ρ̄≤2` — degree-2 VN 비율과 check 평균차수 곱이 2 이하여야 linear `dmin` 보장.
6. 핵심 기법2: degree-1 bit(`λ1>0`) 도입 — EXIT area theorem `ΔA(p)=(C(p)-R)/(λ̄(1-λ1))`로 `λ1↑`이 threshold를 capacity에 근접시키고 waterfall 기울기·수렴속도 개선.
7. 구현/설계 디테일: linear dmin(작은 λ1)과 빠른 수렴(큰 λ1)이 상충 → tail-biting convolutional base code로 cluster 크기 최소화, `G`에 cycle 없게 permutation 설계.
8. 최적화: Gaussian 채널에서 `Λ̃(x)` 1차원 최적화(EXIT/DE), rate 1/10에서 `λ1=1/3`, `λ̃2=0.4`.
9. 결과: rate 1/10 TLDPC threshold ≈ -0.9 dB(capacity -1.286 dB 대비 0.4 dB), WER 10^-3~10^-5부터 ME-LDPC(λ1=22.5%)보다 우수(λ1=33%), error-floor 없음(양쪽 다 good dmin).
10. 한계: HW 미설계·대부분 이론/시뮬, TLDPC의 linear dmin은 증명 아닌 추측, 초저rate 전용.
