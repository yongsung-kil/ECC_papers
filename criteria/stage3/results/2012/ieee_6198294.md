> 총평: NMS + row-based layered로 iteration 반감, 65nm 703k 게이트 5.79Gb/s 실측 칩. 알고리즘(NMS/layered/early-term)은 Prime ECC 기존 보유와 중복이나, multi-rate 공유 sorter·routing 감축 HW 기법은 참고 가치가 있는 중간 이식성.

### B. 알고리즘 요약

1. 대상: IEEE 802.15.3c 4개 irregular QC-LDPC (`672,336`/`672,420`/`672,504`/`672,588`), rate 0.5~0.875, AWGN + 16-QAM/QPSK.
2. 문제: 다중 rate·블록 지원 시 Gb/s throughput 유지하며 HW 복잡도(특히 routing congestion) 낮추기 어려움.
3. 복호: NMS(normalized min-sum, scale factor `0.75`) + row-based layered scheduling. layer 내 C-to-V 직후 V-to-C가 최신 메시지 즉시 사용 → 표준 BP 대비 iteration 절반.
4. 구조 활용: 4개 PCM이 4-layer로 분할되며 layer별 등가 행 구조 → permutation block·연산유닛을 4개 코드에 재사용, layer당 column degree ≤ 1.
5. CNU: min-sum이라 min/min_index/2nd_min 필요 → 재구성형 8/16/32-input sorter (32=8×4 합성)로 row degree 8~32 대응.
6. V-to-C routing: sorter 입력 재배치(reallocation)로 12-to-1 MUX를 4-to-1로 축소, 선택신호는 layer index만 의존.
7. C-to-V routing: pre-coded routing switch(coarse 선택, 21×4개 3-to-1 MUX) + 672개 4-to-1 MUX → 전역 배선 대폭 감소. 전체 MUX 입력 177408 → 최적화로 64% 감축.
8. 인코더: QC-LDPC systematic 인코딩을 SRAA 대신 AASR(adder-accumulator-shift register)로 구성, 계수 0 위치 제거해 register/AND/XOR 감축 (r=1/2에서 54.2% 감소).
9. 조기종료: syndrome=0 시 clock gating. low-power 모드 / high-throughput 모드 2종.
10. 결과: 65nm 1P10M, 703k gate, core 1.56mm², 5.79Gb/s @5 iter/1V, 62.4pJ/b, 3.7Gb/s/mm². 한계: 802.15.3c 단일 표준 특화, 정정능력 자체 개선 아님(HW 효율/속도 개선).
