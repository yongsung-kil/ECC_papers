> 총평: min-sum CNU의 first-two-min 회로 latency를 줄이는 순수 HW 최적화 브리프. 알고리즘/정정능력 무관하나 CNU 임계경로 개선 시 참고 가치 중간.

### B. 알고리즘 요약

1. 대상 연산: 집합 `X={x0..x_{M-1}}`에서 첫째·둘째 max/min(`y0,y1`) 동시 탐색 — min-sum LDPC의 CN 출력(min1/min2)과 turbo의 Jacobian max*에 필수.
2. 문제: 기존 radix-2 트리 구조는 `O(log2 M)` 스테이지로 delay가 커서, 피드백 루프 있는 iterative decoder에서 pipeline으로도 throughput이 안 오름.
3. 접근: 일반 radix-K 고정 트리(FRA)로 확장 — 각 CS(comparing stage)가 K입력의 first-two를 `O(1)`에 뽑고, 트리 delay는 `O(log_K M)`으로 단축.
4. 비용: radix-K CS는 `K(K-1)/2` 비교기 필요, 총 비교기 수 `C = M/2·(K + 2M/2·... )` 형태로 area가 radix에 따라 증가(식(1)~(3)).
5. 확장1 IR-MRA(inter-level): 트리 레벨마다 다른 radix 사용, `∏K_l = M` 제약 하에 비교기 수 `C` 최소화(식(4)~(10)).
6. 확장2 IA-MRA(intra-level): 같은 레벨 내 CS마다 다른 radix — 더 유연하나 최소화가 Diophantine이라 exhaustive 탐색(식(11)(12)).
7. 회로 구성: 각 CS = 비교기 배열 + one-hot index generator(OHIG) + mux-like structure(MLS). 부호비트 `s_{p,q}`와 and/or gate로 max1(식13,14)·max2(식15,16) 인덱스를 OH로 산출.
8. 트레이드오프 지표 `μ3 = (A_radix/A_radix2)·(L_radix/L_radix2)`; `μ3<1`이면 radix-2 대비 area·latency 곱 우수.
9. 결과(180nm P&R): M=32,N=3에서 IA-MRA로 `C=115`까지 감소; M=9에서 FRA-3가 radix-2 대비 area 21025→14426 µm², latency 2→1.6 ns; 4/2/3 MRA는 latency 25%↓·area 1.1%↑. LDPC용 기존 [14][15] 대비도 우위.
10. 한계: 순수 회로 latency/area 최적화로 BER·iteration·정정능력과 무관; 부호/디코딩 알고리즘 변경 없음, 시뮬 BER 결과 없음.
