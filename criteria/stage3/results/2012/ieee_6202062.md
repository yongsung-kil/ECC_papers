### B. 알고리즘 요약

1. 대상은 `(ms, J, K)=(3,4,24)` LDPC 컨볼루셔널 부호(LDPC-CC), rate `R=b/c` (예제 rate-1/3), AWGN 채널.
2. 문제: block LDPC는 가변길이 패킷(Ethernet 등)에서 오버헤드가 크므로 스트림 처리에 LDPC-CC가 유리하나, 복호 계산량이 커 시뮬이 느림.
3. 복호기는 다수의 동일 processor를 파이프라인 연결한 구조 - 각 processor가 1 iteration, sliding decoding window로 스트림 처리.
4. 복호 알고리즘은 표준 `SPA`(Sum-Product, tanh 사용)와 `MSA`(Min-Sum) 두 가지 - 신규 알고리즘 아님.
5. 각 processor에서 (i) 새로 진입한 VN에 연결된 CN 갱신, (ii) 떠나는 VN 갱신 후 다음 processor로 전달.
6. 세 플랫폼에 구현: 단일스레드 CPU, OpenMP(멀티코어 CPU), CUDA(GPU, Nvidia GTX460 336 core).
7. GPU 구현: PCM을 device memory 복사, AWGN 생성·LLR·VN/CN 메시지·hard decision을 SIMD 병렬 처리.
8. 병목: "Register Shift" 과정은 여전히 CPU에서 수행 → device-main memory 왕복 전송이 속도 제한.
9. 결과: OpenMP는 CPU 대비 3~4배, CUDA는 OpenMP 대비 최대 5배·CPU 대비 최대 19배 빠름. BER은 세 플랫폼이 거의 동일.
10. 한계: 부호설계·복호 성능 개선 없음(속도만), HW 미설계, LDPC-CC 특화로 block QC-LDPC와 무관.
