### B. 알고리즘 요약

1. 대상: WiMAX QC-LDPC(`(576,288)` rate 1/2, avg row-weight 6.25, submatrix 24, 12 layers 등), 멀티코어(4클러스터×8프로세서, MIPS32-like RISC + 2D-mesh NoC) SW 디코더.
2. 문제: ASIC 대비 유연성 위해 멀티코어 SW 디코더를 쓰지만, 기존 구현은 디코딩 시간의 ~50%가 프로세서 간 통신에 소모됨.
3. 복호 알고리즘: `TDMP`(turbo-decoding message passing, layered) + offset Min-Sum. `Lvc=Sv-Rcv`, `Rcv=η·∏sgn·min`, `Sv=Lvc+Rcv` 3스텝.
4. 기법1 (통신 트래픽 감소): 알고리즘 분할을 수평/수직으로 정량 분석 → `CPB=ρB` vs `LPB=2ρB-2MkB`, `dc>2`이면 **수평분할**이 통신량 최소(대부분 LDPC가 해당).
5. 핵심 식 의미: 수평분할은 한 행의 모든 nonzero submatrix 연산을 한 task로 묶어 수평 message passing을 task 내부화, task 간엔 수직(layer 간) passing만 남김.
6. 기법2 (통신 거리 감소): task 간 수직 통신 횟수가 불균형함(WiMAX 통계상 max 11 vs min 0)을 이용, branch-and-bound로 자주 통신하는 task를 근접 배치 → 통신거리 ~20% 감소.
7. 기법3 (수신 식별비용 감소): 패킷 헤더에 행 식별용 **R-flag** 추가 → 수신측이 dc개 카운터 대신 현재 행 카운터 1개만 확인, 패킷 1개 수신 사이클 ~53% 감소.
8. 최적화·매핑 절차: 파라미터화 LDPC로 분할전략 일반화 후 branch-and-bound 자동 매핑.
9. 결과: 통신시간이 전체 디코딩의 **12.2%**(기존 ~50%), throughput 7.91Mbps@700MHz(27코어, 10 iter), 정규화 지표 TNDC 0.4185로 비교군 중 최고.
10. 한계: 부호설계·정정능력 개선 없음, ECC 자체가 아닌 멀티코어 SW 매핑/통신 최적화, throughput이 Mbps급으로 매우 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP(layered) + offset Min-Sum 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC도 min-sum 보유(중복), layered 스케줄은 이미 Dual-Update로 유사 |
| 수평/수직 알고리즘 분할·NoC 매핑 | 대응 없음 | Prime ECC는 SW 멀티코어가 아닌 고정 z=32 병렬 HW pipeline, 프로세서 간 통신 개념 부재 |
| R-flag 수신 식별 기법 | 대응 없음 | 전용 HW엔 비동기 패킷 수신/카운터 문제 자체가 없음 |

적용 가치: **낮음** — 멀티코어 SW 디코더의 NoC 통신 최적화라 전용 HW pipeline(z=32, HCU, GT) 기반 Prime ECC와 아키텍처 전제가 다르고, ECC 알고리즘·정정능력 기여가 없음.
