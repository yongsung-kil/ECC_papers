### B. 알고리즘 요약

1. 대상은 binary-input memoryless symmetric channel(BSC/BIAWGNC)의 binary linear block code(LDPC/turbo/BCH/RM 포함), 복호를 수학적 최적화로 정식화한 서베이다.
2. 문제: 반복 message passing(SPAD/MSAD)은 수렴 보장·ML certificate가 없고 dense parity-check code에 약하다. LP decoding(LPD)은 이를 분석적으로 대체한다.
3. 핵심 모델: MLD를 LLR `γ` 목적함수 `min γ^T x`의 정수계획(IP)으로 두고, codeword polytope `conv(C)`를 fundamental polytope `P`로 완화한다.
4. 기법 1단(BLPD): local code convex hull의 교집합 `P` 위에서 LP를 풀어 정수해면 ML codeword(ML certificate 성질), 비정수해면 실패 처리한다. 정식화 BLPD1/2/3, forbidden-set(FS) 부등식.
5. 핵심 식 의미: `d_frac ≤ d_min`(fractional distance가 최소거리 하한)으로 LPD 성공조건을 폴리토프 기하로 정량화한다.
6. 기법 2단(효율화): ALPD/MALPD(FS 부등식 adaptive 추가), CLPD(check node 분해로 선형 크기), 내부점법, BCQPD(2차계획) 등 복잡도 축소 계열.
7. 기법 3단(성능개선): RPC(redundant parity check) cutting plane, facet guessing, branch&bound로 fundamental polytope를 조여 MLD에 근접(error-floor 저감).
8. (학습 절차 없음 — 조합최적화/폴리토프 이론 기반).
9. 결과: BCH/RM/Tanner code 등에서 개선 LPD가 SPAD·BLPD를 상회(정성). 단 SPAD 대비 복호시간은 여전히 느리고 부호 길이 30~1920 소형 예제 중심.
10. 한계: HW 미설계, LP solver 기반이라 min-sum 대비 복잡도 폭증, non-RPC facet 특성화·복잡도 축소가 미해결(결론에서 명시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP/IP 기반 복호 알고리즘 | 대응 없음 | Prime ECC는 min-sum message passing, LP solver 기반 복호는 대응 모듈 없음 |
| ML certificate·fundamental polytope | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 개념적 참고뿐, 코드 구조상 직접 대응 아님 |
| coordinate-ascent(min-sum 유사) SDLPD | `decoder.cpp` `CNU_Update_New_Mag()` | LP dual이 min-sum과 유사하다는 이론 연결일 뿐 구현 이식 아님 |

적용 가치: **낮음** — LP/IP 복호는 min-sum HW 파이프라인과 계보가 다르고 복잡도가 커서 NAND ECC에 떼어 쓸 실질 요소가 없다.
