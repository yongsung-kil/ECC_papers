### B. 알고리즘 요약

1. 시스템: IEEE 802.3an (6,32)-regular LDPC, 길이 `n`=2048, rate 0.84, sum-product(SPA) 복호를 CUDA-GPU(GTX480)에서 병렬 구현.
2. 문제: LDPC bipartite 그래프의 비대칭성 때문에 bit-to-check/check-to-bit 양방향 모두에서 read·write coalesced 접근이 동시에 불가.
3. 모델: BPSK+AWGN 채널, `Λn` LLR 소프트정보, `smn`/`qmn` 메시지, 32-bit 정밀도 시뮬.
4. 핵심기법: check-to-bit 엣지 번호를 재배열해 한 방향(bit-to-check)은 translating array 없이 read·write 완전 coalesced 달성 = "unbalanced memory coalescing(UMC)".
5. 핵심식: SPA `qmn = Λn + Σ skn`(V2C), `smn = 2·tanh⁻¹{Π tanh(Λi/2)}`(C2V) — 표준 sum-product.
6. 확장: 나머지 방향(check-to-bit)은 read·write 동일한 단일 translating array 사용 → 배열 1개만 필요(기존 [13]은 2개), off-chip 접근 절반 감소.
7. 구현: translating array를 on-chip shared memory에 적재, 메시지·AWGN·PCM을 texture cache에 적재해 off-chip 접근 최소화.
8. 최적화: shared memory 파티션 수 최적화, parallel reduction으로 에러카운팅.
9. 결과: CPU 대비 234x~250x 속도향상, `Tdec` UMC로 35% 감소, texture caching으로 `Ttotal` 24% 감소, throughput 24.5~146.6 Mbps.
10. 한계: 순수 GPU 시뮬 가속(정정능력·iteration 개선 없음), HW(RTL/ASIC) 미설계, GPU 메모리 계층 특화로 NAND ECC HW와 무관.
