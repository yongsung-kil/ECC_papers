### B. 알고리즘 요약

1. 대상은 구조적(structured) LDPC: 10GBase-T의 RS-LDPC `(2048,1723)` (γ=6, ρ=32, 서브행렬 64×64 PM)와 EG-LDPC `(4095,3367)` (QC form, 63×63 CPM).
2. 문제: SMP(sliced message passing)는 CN/VN 업데이트의 순차 의존을 끊어 HW 이용률 ~100%를 얻지만, forward/backward permutation network가 큰 fan-in MUX를 요구해 row weight가 큰 고rate 부호에서 비용이 폭증.
3. 모델: NMSA(normalized min-sum, 식(1)의 `α` scaling) + syndrome 조기종료. 서브행렬이 CPM/PM인 부호.
4. 핵심 기법: 일부 구조부호의 `shift property`(같은 block row의 연속 PM들이 시프트 관계) 이용 - 레지스터 뱅크를 순환(R0->R1->R2->R0)으로 배선하고 CNU 출력도 순환 시프트 라우팅.
5. 핵심 식/의미: `(γ,ρ)`-정규부호에서 기존 `γm`개 `ρ-to-1` MUX 대신 `x`개 `⌈(γ+ρ-1)/γ⌉-to-1` + `y`개 `⌊(γ+ρ-1)/γ⌋-to-1` MUX로 분할(`x+y=m`) - fan-in을 row weight에서 약 `ρ/γ`로 축소.
6. 확장: RS-LDPC뿐 아니라 유한체 곱셈군 기반 QC-LDPC, cyclic LDPC의 QC form 등으로 일반화 가능.
7. 구현 디테일: CN-to-VN 메시지는 min1/min2/index/sign 압축 저장(고rate 부호 메모리 ~70% 절감), VN-to-CN은 즉시 계산·소비하여 미저장.
8. (2048,1723): 기존 512개 32-to-1 MUX -> 512개 8-to-1 MUX, gate 218,880 -> 68,224 (본 논문 32%, [10] 대비 68% 절감, [7] 대비 59% 절감).
9. (4095,3367) EG tri-mode 디코더: slices 10% 감소, block RAM 95% 감소(153->7), throughput 15% 향상 2.2Gbps.
10. 한계: shift property가 없는 일반 QC-LDPC엔 미적용, 정정능력/BER 개선은 없고 라우팅·면적 절감에 국한, FPGA 검증(ASIC 아님).
