### B. 알고리즘 요약

1. 코드: (1008,504), (504,252) regular LDPC, AWGN 채널, 이진 BF 계열 디코더.
2. 기존 multi-bit GDBF는 각 파리티검사행렬·SNR마다 실험으로 threshold `θ1, θ2`를 찾아야 하는 문제가 있음.
3. 채널 모델: AWGN, bipolar 변조 후 평균0 분산σ² 노이즈 추가, 하드결정 오류는 |noise|>1.0이고 부호가 반대일 때 발생.
4. 핵심 기법: 목적함수 `Ek = xk*yk + Σ_{i∈M(k)} Σ_{j∈N(i)} xj` (Eq.2)을 최대화하는 방향으로 비트 flip. 초기 flip 개수 `Bmax = |{j : |yj|>2.0}|`로 AWGN pdf 특성에서 직접 추정(threshold 불필요).
5. `Bmax` 산정 근거: |noise|>1.0이며 부호 일치하는 확률 ≈ 실패판정 확률이라는 AWGN pdf 대칭성 이용.
6. 반복마다 `Bl = max(⌈Bl-1/2⌉, 1)`로 flip 비트 수를 절반씩 줄여가다 단일비트 모드(`Bl=1`)로 수렴, 동일 비트 반복 flip 시 single-bit 모드 전환.
7. 로컬 최대값 탈출: 동일 local maximum 재진입 시 `Bmax = Bmax+1`로 flip 개수를 증가시켜 같은 패턴 재발 방지(부수 개선안).
8. 별도 학습/최적화 절차 없음(threshold 실험 탐색을 제거하는 것이 본 논문의 목적).
9. 결과: 제안법 BER은 기존 threshold 기반 multi-bit GDBF와 거의 동일, 반복당 연산시간도 거의 동일(43.2µs vs 42.3µs, (1008,504) 기준); local-maximum 재진입 방지 기법 적용 시 SNR 3dB 이상에서 BER 추가 개선.
10. 한계: HW 미설계, waterfall 영역 시뮬 결과만 제시, SNR≥1.5dB에서 100회 반복 normalized min-sum보다 BER 열위.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBF 목적함수 기반 bit-flipping 디코딩 전체 | 대응 없음 | Prime ECC는 min-sum 기반(`decoder.cpp` `LDPC_Decoder()`)이며 BF 계열 알고리즘 자체가 없음 |
| threshold-free flip 개수 추정(Bmax, AWGN pdf 이용) | 대응 없음 | BF 알고리즘 종속 개념으로 min-sum 파이프라인에 직접 대응되는 모듈 없음 |
| soft 수신값 |yj| 기반 채널 통계 활용 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | LLR/threshold 설계 아이디어 참고 가능하나 BF 전용이라 간접적 |
| local maximum 탈출(Bmax 증가) | 대응 없음 | min-sum에는 local maximum 개념이 없어 대응 모듈 없음 |

> 적용 가치: 낮음 — BF/GDBF 계열은 Prime ECC의 min-sum 아키텍처와 근본적으로 다른 디코딩 패러다임이며, HW 미설계·시뮬뿐이라 이식 시 재설계 부담이 큼.
