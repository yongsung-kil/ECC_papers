### B. 알고리즘 요약

1. NAND SSD 컨트롤러용 ECC로 LDPC 대신 2D-TPC(각 차원에 BCH 성분코드)를 제안, `(315×246, 288×228, 7×5)` TPC 구성, `t1=3, t2=2`.
2. LDPC-soft 디코딩은 soft sensing으로 인한 긴 read latency가 문제이며, TPC는 hard-decision만으로 이를 회피하려는 동기.
3. 채널 모델은 RBER 기반 NAND raw bit error, UBER 목표 `10^-15`.
4. 전통적 HIHO(hard-input hard-output) 반복 행/열 스캔 디코딩에서 정정 불가능한 uncorrectable submatrix(USM)가 발생하는 것이 한계.
5. USM 크기가 `2(t1+1)×2(t2+1)`보다 작으면 USM 비트를 flip한 뒤 HIHO를 재실행하는 BFE-TPC(bit-flipping enhanced TPC) 제안, UBER 상한식 `UBER ≤ (n1/2(t1+1))(n2/2(t2+1)) f(U1,U2) RBER^(2(t1+1)(t2+1))`.
6. 디코더 엔진은 inverse-free Berlekamp-Massey 알고리즘을 사용하는 BCH 디코더 4개를 행/열 공용 풀로 재사용.
7. Transpose memory(플립플롭 기반)로 매 클록 행/열 데이터를 전환 공급, 별도 embedded processor가 bit-flipping 및 USM 탐색을 관리(메인 CPU 부담 경감).
8. 별도 학습/최적화 절차 없음(구성법 기반 설계).
9. FPGA(Xilinx VU19P, 250MHz)에서 encoder 29.21Gbps, decoder 17.96Gbps 달성, 2kB LDPC 대비 면적 42% 절감(28nm 기준).
10. HW는 FPGA 프로토타입 수준(28nm ASIC은 timing report만 언급, 실측 아님), 부호가 LDPC가 아닌 TPC/BCH 기반이라 LDPC 이식 관점에서는 구조적 이질성이 큼.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BFE-TPC 디코딩 전체(BCH 기반) | 대응 없음 | LDPC가 아닌 TPC/BCH 계열 디코더로 min-sum 구조와 무관 |
| bit-flipping post-processing(USM 재정정) | decoder.cpp `LDPC_Decoder()` 이터레이션 루프 | 개념적으로만 유사(수렴 실패 시 후처리 아이디어), 알고리즘 자체는 이식 불가 |
| embedded processor 기반 조기종료/상태관리 | partial_CRC.cpp `partial_crc_HW_T4()` | 조기종료 관리라는 역할만 유사, 구현은 무관 |
| transpose memory / 디코더 엔진 풀 재사용 | 대응 없음 | BCH 디코더 풀 공유 구조로 QC-LDPC HW(z=32, HCU)와 대응 안 됨 |
```

> 적용 가치: `낮음` — 부호종류가 LDPC가 아닌 TPC(BCH 기반)이며, Prime ECC는 binary QC-LDPC 전용이라 부호설계/디코더 모두 이식 불가.
