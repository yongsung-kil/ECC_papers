> 총평: layered decoding의 두 row grouping을 동시 처리해 clock 수 기준 디코딩 속도를 높이는 스케줄링 변형으로, BER/FER 성능 개선은 없고 정정능력 기여가 아닌 순수 처리 속도(latency) 트릭이라 Prime ECC의 기존 Dual-Update와 개념이 겹칠 가능성이 있음.

### B. 알고리즘 요약

1. IEEE 802.11n/802.16e 표준의 QC-LDPC 부호(H는 L×L circulant permutation matrix `P(a)` 블록들로 구성, base matrix `B(H)`, exponent matrix `E(H)`)를 대상으로 함.
2. 표준 decoder들이 채택하는 layered decoding(row grouping 단위로 순차 처리, 매 layer 후 즉시 variable node 갱신)은 BP 대비 iteration 수를 절반으로 줄이지만, layer별 순차 처리로 인해 총 디코딩 속도(clock 기준)는 충분히 빠르지 않다는 문제를 제기.
3. 채널 모델은 AWGN + BPSK, 표준 sum-product/tanh 기반 check-node 연산(`φ(z) = -log(tanh(z/2))`)을 그대로 사용.
4. 제안하는 double layered decoding은 row grouping 수가 짝수(i)일 때, `t`번째와 `t+i/2`번째 두 row grouping을 동일 sub-iteration에서 동기적으로 처리(식 11~16): 각각 독립적인 variable node 메시지 `P1,n`, `P2,n`을 유지하다가 마지막 sub-iteration(`t=j/2`)에서 두 결과를 합산(식 17).
5. 핵심 식(17) `P_n^(k) = P1,n^(k)(j/2) + P2,n^(k)(j/2) - P_n^(k-1)`는 두 병렬 layer 그룹의 갱신량을 합쳐 최종 사후값을 만드는 부분으로, 초기값 중복 차감을 통해 두 그룹 처리를 하나의 total iteration으로 등가화한다.
6. 확장/2단 기법 없음 — 단일 스케줄링 변형(row grouping을 2개씩 짝지어 동시 처리)이 핵심.
7. 구현 디테일: check-node grouping 처리에 동일한 S clock pulse가 소요된다고 가정, H(100,6,18) 예시로 Layered는 10×6×S, Double Layered는 14×3×S clock이 소요됨을 수치 비교(clock 수 절감으로 속도 우위 주장).
8. 별도 학습/최적화 절차 없음.
9. 결과: block length 1800bit, rate 2/3 QC-LDPC(H(100,6,18)), 최대 50 iteration, Eb/N0=2.2dB에서 double layered decoding의 clock 수가 일반 layered decoding 대비 적어 속도가 빠름을 주장; BER/FER 성능은 BP와 거의 동등, 그러나 layered decoding보다는 열세이며 평균 iteration 수는 layered decoding 대비 오히려 증가.
10. 한계: FPGA/ASIC 등 HW 구현·합성 결과 없이 clock 수 가정에 기반한 이론적 속도 비교뿐이며, row grouping 수가 짝수여야 하는 제약, 정정성능(BER/FER) 개선은 없음(오히려 layered 대비 열세).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered decoding / double layered 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 iteration 루프 스케줄링과 직접 관련, 다만 이미 유사한 Dual-Update(odd/even 교대) 보유 |
| 두 row grouping 동시 처리(속도 향상) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 (Dual-Update 옵션) | 개념적으로 기존 Dual-Update와 유사한 스케줄링 최적화, 중복 기여 가능성 |
| check-node min-sum/sum-product 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문은 sum-product(φ함수, tanh)를 그대로 사용, Prime ECC는 min-sum이라 알고리즘 자체는 대응 없음 |

> 적용 가치: 중간 — 스케줄링 아이디어는 decoder.cpp 이터레이션 루프에 닿지만, Prime ECC가 이미 보유한 Dual-Update와 개념이 겹쳐 중복 기여 가능성이 크고 정정성능 개선이 없어 우선순위는 낮음.
