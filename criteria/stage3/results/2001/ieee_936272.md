### B. 알고리즘 요약

1. 문제: 정수 `a`(열 weight), `b`(행 weight 상한), 짝수 girth `g`, check node 수 `m`이 주어질 때 girth≥`g`를 만족하는 최대 열 수(=최고 rate) `m×n` H-matrix 구성.
2. 동기: 기존 random/algebraic 구성은 지수시간이라 작은 column weight·`g=6`에만 실용적.
3. 핵심 아이디어: 이미 제약을 만족하는 H에 (n+1)번째 열(=bit node)을 추가하며 최대 `a`개 check node를 연결, 각 edge가 길이 `(g-2)` 이하 cycle을 만들지 않도록 test.
4. cycle 회피: check node `c`의 2-hop 이웃집합 `N_c`로부터 `U_j`(`c`에서 길이 `2j-2` 경로) 정의, 금지집합 `U = ∪ U_j` (`1≤j≤g/2-1`)를 피해 feasible 집합 `F0 = A \ U` 선택(`A`=degree<`b`인 check).
5. 의미: `U_j`에 속한 check를 `U1`에 넣으면 길이 `2j` 이하 cycle 발생 → 이 집합을 배제해야 girth 보장.
6. 주 휴리스틱: `F0` 중 지금까지 가장 적게 쓰인(최소 degree) check 선택 → Tanner graph를 최대한 균질(homogeneous)하게. `1-h`(first-order homogeneity)와 `c-h`(complete-homogeneity, 더 넓은 이웃까지 비교) 두 변형.
7. 확장: fixed girth·high rate, fixed rate·high girth, fixed girth·fixed length·high rate(binary search로 최소 `m`), irregular(bit-degree 분포 지정) 등에 쉽게 적응.
8. 복잡도: 전체 `O(bm^3)`, `1-h` 단순버전은 `O(bm^2)`; GIRTH&BI-DEGREE 결정문제는 NP이나 (P≠NP 가정) NP-hard 아님.
9. 결과: MacKay 최고 rate 코드 9개 중 7개에서 더 높은 rate 달성(예 a=4,n=444: MacKay 3584 vs c-h 12360); 동일 파라미터(m=222,n=1998,a=4,b=36,g=6) BER은 세 코드가 거의 동일(AWGN).
10. 한계: HW 미설계, 주로 girth-6 regular 구성이며 QC/circulant 구조 아님, 성능 개선이 아닌 rate·유연성 향상이 초점.
