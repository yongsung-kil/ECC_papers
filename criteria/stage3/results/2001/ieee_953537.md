### B. 알고리즘 요약

1. 시스템: (1268,456) LDPC(construction 2A, ATM cell용), BPSK/AWGN, sum-product(belief propagation) 복호, 최대 500 iter.
2. 문제: flooding schedule은 짧은 블록(≤10000bit)의 작은 cycle 때문에 APP 최적에서 멀어져 성능 저하.
3. 모델: cycle-free TG면 sum-product가 최적 APP지만, 실제 TG는 cycle 다수 → girth `g_u`(노드 u 통과 최단 cycle)가 최적성 붕괴 시점을 결정.
4. 핵심 기법: 각 symbol node `u`에 확률 `p_u = g_u/g_max` 부여, 매 iteration마다 확률 `p_u`로만 outgoing 갱신·`1-p_u`로 idle 후 check node 갱신.
5. 의미: girth 작은 노드(비최적 메시지 조기 유발)의 갱신 빈도를 낮춰 `g_max/2` iteration마다 한 번씩만 최단 cycle을 돌게 함 = "equal double counting" 근사.
6. 확장: 첫 iteration만 flooding으로 초기 메시지 전부 주입, 이후 확률 스케줄 적용.
7. 구현: 평균 갱신 메시지 `E + Σ g_u d_u/g_max` (< flooding의 `2E`)로 iteration당 연산은 동등/소폭 감소.
8. 학습: 별도 학습 없음, girth distribution만 사전 계산.
9. 결과: SNR 2.5dB에서 BER·MER를 flooding 대비 한 자릿수 이상 개선, undetected error 대폭 감소; 단 avg iteration은 최대 3회 증가.
10. 한계: HW 미설계, 시뮬만, 저rate 단블록 full sum-product 대상, latency(iteration 수) 증가.
