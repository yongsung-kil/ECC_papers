> 총평: LLR 도메인 SPA를 serial(trellis, forward-backward)·parallel(tree) 토폴로지로 통일 정리하고 core연산의 보정항을 저해상도 LUT·piecewise-linear·단일 상수로 근사해 SPA를 거의 손실없이 재현하는 HW 지향 고전 논문. sign-min은 min-sum, 상수보정은 offset-min-sum에 해당해 우리 디코더에 직접 이식·비교 가치 높음.

### B. 알고리즘 요약

1. 대상은 binary regular/irregular LDPC, AWGN 채널, LLR 메시지 기반 message passing(LLR-SPA); MacKay 온라인 코드 등으로 검증.
2. 풀려는 문제: brute-force LLR-SPA의 CN update는 `tanh`와 행degree `dc`에 비례하는 곱셈이 필요해 고rate·고속 HW에 부적합.
3. 핵심 모델: CN 연산을 두 독립 이진변수의 XOR LLR core연산 `f(a,b)=sgn·min + 보정항`으로 정의(log-likelihood algebra [8]).
4. 핵심 기법 1단(serial): CN update를 single-state trellis 위 forward-backward로 계산 → `3(dc-2)`회 core연산, latency `O(dc)`.
5. 식 의미: core연산은 `sgn(a)sgn(b)min(|a|,|b|) + [ln(1+e^-|a+b|) - ln(1+e^-|a-b|)]`로, 보정항이 min-sum과 정확 SPA의 gap.
6. 핵심 기법 2단(parallel): 보조변수 `δ=⊕`의 LLR을 tree 토폴로지로 계산 → CN당 `2dc-3` core연산, latency `O(log2 dc)`로 축소(고throughput용).
7. 구현 트릭: 보정항 `ln(1+e^-|x|)`을 max-star(2-bit coarse LUT, Table I) 또는 8-segment piecewise-linear(계수=2의 거듭제곱, shift; Table II)로 근사; tree용 `log|e^x-1|`은 Table III 8구간 근사.
8. 근사 변형: sign-min(보정항 제거, add-free 비교만), 단일 상수 보정(c=0.8, offset), max-star LUT 등 다양한 design point 제공.
9. 결과: LLR-SPA1/2의 LUT·piecewise-linear는 3000 parity-check 코드에서도 SPA 대비 손실 <0.05dB; 단일 상수(c=0.8)도 SPA 근접; sign-min만 0.3~0.5dB 손실.
10. 한계: gatecount/throughput 등 정량 HW 수치 없음(Monte Carlo 시뮬, max 80 iter), 부호 rate/length 본문 명시 없음, error-floor 미다룸.
