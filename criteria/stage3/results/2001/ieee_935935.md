### B. 알고리즘 요약

1. 유한상태 binary Markov 채널(변조기-실채널-복조기 이산채널의 error pattern) 위에서 LDPC를 복호하는 수정 알고리즘 제안.
2. 문제: Markov 채널은 같은 정상상태 BER의 memoryless 채널보다 용량이 크나, 그 높은 용량을 실제 복호에서 활용하기 어렵다.
3. 채널을 표현하는 hidden Markov model(HMM)을 LDPC의 bipartite 그래프에 통합, 전체를 Bayesian network로 간주.
4. 핵심: LDPC의 error pattern 노드를 HMM과 "link"하고 belief propagation을 그에 맞게 수정.
5. HMM 도입은 그래프 사이클 수를 늘리지만, 실험상 복호 성능에 큰 악영향은 없음. LDPC 구조 덕에 채널 인터리버 불필요.
6. 메시지 스케줄: check 노드 → error pattern 노드 → HMM 노드에서 forward/backward BP → error pattern 노드로 복귀·결합 → check 노드로 전파(1 full iteration).
7. 채널 미지 파라미터에 대해서는 joint estimation/decoding으로 파라미터를 복호와 동시 추정.
8. 채널 모델: 2-state Gilbert-Elliott, `b=.0044`, `g=.0156`, bad state `PB=.5`, good state `PG` 변화로 정상 BER `p` 조절.
9. 결과: rate-1/2 (3,6)-regular LDPC, 35000 coded bits. `p=.124`에서 낮은 BER 달성(메모리 무시 시 `p>.11`에서 신뢰전송 불가). BSC 대비 큰 개선.
10. 한계: 시뮬만, HW 미설계, 미지 파라미터 joint estimation 시 더 많은 iteration 필요, 특정 Markov 채널 가정.
