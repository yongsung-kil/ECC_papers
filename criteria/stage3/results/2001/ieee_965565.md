### B. 알고리즘 요약

1. 문제: 주어진 `m`(check수)·차수열 `{a_j}`·최대 row weight `b`에서 최대 rate(=최대 n) 또는 최대 girth의 sparse H를 구성 (Problem I.1/I.2).
2. 핵심 아이디어: 비트(열)를 하나씩 Tanner 그래프에 추가하되 항상 girth 제약을 유지하는 탐욕(greedy) 방식.
3. 확장의 핵심: girth 제약 `g'`를 실행 중 가변. 큰 `g'`로 시작→추가 불가 시점에 `g'`를 2씩 감소하며 계속 (기존 고정 girth 대비 개선).
4. 그래프 연산: check node `c`의 2-edge 이웃 `N_c`로 `U_j` 정의(식1), `U=∪_{1≤j≤g'/2-1}U_j`를 회피 집합으로 하여 길이 `<g'` cycle 생성 방지.
5. 가용 집합 `F0 = A\U` (`A`=차수 여유 있는 check). `F0`가 비면 girth 감소, `g'<g`이면 중단.
6. 효율화: `U`를 증분 갱신(식3,4)해 매 열 추가 시 재계산 회피. 전체 복잡도 `O(bm^3)`(1차 휴리스틱 시 `O(bm^2)`).
7. 차수분포: 낮은 차수 비트부터 삽입하면 저차수 subgraph의 girth가 커져 small-weight cycle을 회피 → Richardson 지적대로 성능에 유리.
8. 고rate 결과: MacKay 최고rate 9개 부호 중 8개에서 더 높은 rate 달성(예: `4/222`에서 n 1998→2967), 1개는 동등.
9. girth/성능: 동일 파라미터에서 girth를 6→8→10→12로 키우면 regular(m=2000,N=3502)에서 BER 대폭 개선(Fig.4), irregular도 완만한 개선(Fig.5). 고SNR에서 랜덤 대비 열화 없음.
10. 한계: HW 미설계·AWGN 시뮬만, 비정형 sparse H(QC 구조 아님), gatecount/throughput/latency 정보 없음.
