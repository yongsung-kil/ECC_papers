### B. 알고리즘 요약

1. 대상: magnetic recording용 LDPC, sector 4096 user-bit, 목표 throughput >1Gbps. 부호는 random(512×4608, rate 8/9)과 2D GF(2^5) finite-geometry(1023×4092, rate 3/4) 두 종.
2. 문제: 직렬 LDPC 디코더는 그래프 edge 수에 비례하는 대용량 메시지 메모리(파이프라인 시 edge×4배)가 병목.
3. 모델: bit/check 노드 간 message passing(`Qnm`,`Rmn`), check 갱신은 `Φ(x)=-log tanh(x/2)` 기반 sum-product.
4. 핵심 기법(staggered schedule): (2)식의 `-Rmn` 항을 생략한 근사 `Q'n(k)=Q'n(k-1)+Σ Rm'n`로, 각 bit당 단일 running 메시지 하나만 저장.
5. 스케줄: 매 step 활성 check 부분집합 `S(k)={m: k mod M}`만 처리(예제 `|S(t)|=1`) - 한 iteration = 전체 parity check 1회 순회.
6. 효과: bit당 메시지 리스트 대신 하나만 저장 → edge가 bit의 4~8배인 전형 LDPC에서 메모리 75%+ 절감, arithmetic 복잡도는 동등.
7. HW 구현: GF 코드의 row cyclic 성질 + 1:4 column splitting 이용, RAM을 4병렬 shift-register 체인으로 대체(1/4 주파수로 동일 throughput). check-to-bit는 signed-magnitude(MSB XOR tree + 하위3bit adder tree).
8. random 코드는 4개 128×4608(col wt 1, avg row wt 36) 부착으로 512×4608 구성.
9. 결과: BER=1e-5, 3 iteration에서 staggered가 concurrent 대비 0.3~0.4dB 개선(빠른 수렴), 그러나 5 iteration에선 cycle 누적으로 이득 소멸·<0.2dB 열화.
10. 한계: BP 근사라 iteration 많으면 성능 하락, 순수 시뮬(HW 합성/실측 없음), 채널이 magnetic recording·부호가 비 QC-lifting.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| staggered(근사 BP) 스케줄·running sum | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 스케줄링 변경 레이어 - Prime ECC dual-update 교대 스케줄과 대비되는 대안, 이식 지점 명확 |
| check-to-bit sum-product(`Φ=-log tanh`) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min-sum - full-tanh LUT는 직접 대응 아님 |
| 직렬 디코더 메시지 메모리 축소 | 디코더 HW 모델(z=32, HCU, GT) | 메모리 병목 개념은 공유하나 QC row-병렬 구조라 edge-메모리 이슈 성격이 다름 |

적용 가치: 중간 - staggered 스케줄(메모리 축소 근사 BP) 아이디어는 참고할 만하나, Prime ECC는 이미 QC 구조·dual-update로 메모리를 다르게 다루고 min-sum이라 그대로 이식할 부분은 제한적.
