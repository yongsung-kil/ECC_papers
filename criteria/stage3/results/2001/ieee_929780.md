### B. 알고리즘 요약

1. 채널/코드: 1024-bit, rate-1/2 irregular LDPC, soft-decision, AWGN(수신값=LLR), 3G 무선 turbo 블록 크기 대응.
2. 문제: turbo/block-turbo의 iterative 복호가 block-serial이라 throughput↓·전력↑; LDPC는 block-parallel 가능.
3. H 스펙: 평균 열가중치 3.25 (열가중치 t=3,6,7,8), 평균 행가중치 6.5, 행가중치 k=6이 256개·k=7이 256개.
4. 핵심 기법: LDPC bipartite 그래프를 그대로 HW에 인스턴스화 → 모든 variable node/check node를 교대로 완전병렬 갱신.
5. 메시지: sign 1bit + magnitude 3bit(총 4bit)로 VN↔CN 전달; 수신값도 4-bit.
6. Check node: 행 parity(XOR)를 각 입력과 XOR해 각 edge용 값 계산; reliability는 log-도메인에서 덧셈으로 갱신 후 역변환(log-MAP류 근사, 소수 게이트).
7. Variable node: 입력 메시지+수신값을 2's complement로 합산, 합의 부호가 현재 추정; 출력=그룹합−개별 입력(합/출력이 0이면 수신 비트 부호 사용).
8. I/O: scan chain(width w=16, depth d=64)으로 3-block pipeline; 반복 수 = d = 64회, 64MHz 클럭으로 1Gb/s.
9. 결과: 6-iter MAP turbo와 PER<1%에서 동등 coding gain; 1Gb/s에서 평균 220mW(최악 500mW), 수렴 후 스위칭 activity 1%.
10. 한계: rate/length 고정(1024,R=1/2), 완전병렬 라우팅(26000+ wire, 3mm 평균)이 실병목; error-floor/정정능력 개선 아님.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 완전병렬 message passing 이터레이션 | decoder.cpp LDPC_Decoder(), C2V_Cal() | 알고리즘 골격은 동일하나 병렬 단위(전체 vs z=32) 상이 |
| Check node reliability(log-domain 덧셈 근사) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum과 다른 log-log 근사; 직접 대응 아님 |
| Variable node 합산/양자화(3bit mag) | decoder.cpp VN_Cal_HD() 등 | 양자화 폭·표현 상이 |
| 완전병렬 HW 라우팅(그래프 직접 배선) | 대응 없음 | Prime ECC는 z=32 부분병렬, GT/HCU 구조 |
```

적용 가치: **낮음** — 완전병렬 그래프-직결 아키텍처와 4-bit log-log 근사는 QC-LDPC 부분병렬 min-sum 기반 Prime ECC와 구조가 달라 이식 가치가 낮다.
