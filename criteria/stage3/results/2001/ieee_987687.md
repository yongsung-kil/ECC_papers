### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 채널, 짧은 블록(<10000bit)의 (3,k)-regular binary LDPC를 대상으로 부분병렬 VLSI 디코더에 맞춘 부호·디코더 동시설계.
2. 문제: 완전병렬 디코더는 복잡도 과다, 랜덤 LDPC는 임의 코드에 대해 효율적 부분병렬 변환이 사실상 불가능.
3. 접근: decoder-first 설계(`[3]`)의 랜덤수생성기 복잡성을 제거하려 code·decoder를 joint 설계.
4. 1단: 순환시프트로 `Ho`(L×L 단위행렬 블록)·`H1`(cyclic shift 블록 `P=T^u(I)`, `u=((z-1)·y) mod L`)을 구성해 girth-12인 (2,k)-regular 코드 `C2`를 결정론적으로 만듦.
5. 정리 3.1: `L`이 `a·b (a,b∈{0..k-1})`로 인수분해 안 되면 girth=12 보장 → 짧은 사이클 억제로 BP 성능 확보.
6. 2단: (3,k) 디코더가 `t_{x,y}`·hash `f`·g-layer shuffle로 세 번째 서브행렬 `Hp`를 삽입, `C2`의 sub-code인 (3,k) 코드 앙상블을 정의(각 코드 4-cycle free).
7. 구현 디테일: `k^2` 메모리뱅크(RAM I/E1/E2/E3/C), l-layer·g-layer shuffle network, k개 CNU, `k^2`개 VNU, 뱅크별 modulo-L 카운터 AG; 1 iteration = 3L 클록.
8. 부호선택: girth average 비교 + 컴퓨터 시뮬로 앙상블에서 좋은 코드 선별(500개 생성→상위 5개→최적 1개).
9. 결과: 두 예제 `Cj1`(2304,1154), `Cj2`(4608,2306), rate≈0.5. 정정성능이 fully-random (3,6) 코드와 거의 동일; 수신 4bit·내부 6bit 양자화 유한정밀 시뮬.
10. 한계: 실제 HW 구현·gatecount·throughput 미제시, rate 0.5 저rate·soft 4bit, 시뮬레이션 수준.
