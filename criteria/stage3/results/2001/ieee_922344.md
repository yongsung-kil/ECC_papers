### B. 알고리즘 요약

1. 대상: irregular binary LDPC, 블록 1024bit, rate 1/2(3G 무선 turbo 대안), soft-decision, BPSK/AWGN 가정.
2. 문제: turbo/block-turbo 디코더는 block-serial 의존성으로 병렬화 불가 → 반복마다 대용량 상태 fetch/store로 throughput↓·전력↑. LDPC는 block-parallel 복호 가능.
3. 복호: message passing(iterative BP) — 그래프에서 VN↔CN 메시지 교환, hard-decision(1bit 패리티 XOR/다수결) 또는 soft-decision(신뢰도 전파, tanh 기반 reliability update).
4. 핵심 관찰: 그래프상 노드는 반대 타입 노드에만 의존 → 모든 VN 또는 CN을 block-parallel 갱신 가능. 그래프 사이클 최대화(row/col overlap 최소화)로 성능↑.
5. 아키텍처 비교: hardware-sharing(소수 유닛+메모리 fabric, 면적↓·throughput↓·제어복잡)  vs 완전병렬(그래프를 HW에 직접 인스턴스화, 배선이 곧 그래프 연결).
6. 핵심 트릭: 반복 진행 시 변화하는 메시지 비율이 급격히 수렴(switching activity 감쇠) → 완전병렬 아키텍처의 activity factor가 매우 낮아 초저전력.
7. 구현: 1024bit R=1/2 코드, avg col weight 3.25(실제 3/6/7/8), avg row weight 6.5(weight-6 256개+weight-7 256개 CN), cycle 최대화 systematic 구성, 메시지=1 parity+3 reliability bit(4bit).
8. 물리설계: 0.16um 1.5V CMOS 5-metal 표준셀, VN 1024개를 16 macro(vgrp0~15)로 그룹화, 26000+ 메시지 배선을 custom placement/buffer 삽입으로 라우팅 혼잡·타이밍 해결. chip 7.5x7.0, utilization 50%(라우팅 제약).
9. 결과: iter당 1클록, 64iter/packet. 64MHz에서 1Gbit/s throughput, avg power 220mW(PER 1%, switching 1%)·worst 500mW. turbo(6 iter)와 성능 거의 동일. 저전력이 turbo 대비 압도적.
10. 한계: 코드가 배선에 고정되어 코드 변경 시 재설계 필요, 라우팅 혼잡으로 utilization 낮음, 정정능력 개선 아님, gate count 미제시(합성).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| message passing 복호 루프 (VN↔CN) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 동일한 반복 메시지패싱 골격이나 완전병렬 vs 부분병렬 상이 |
| soft-decision reliability(tanh) update | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 min-sum 근사 — 알고리즘 이질 |
| 완전병렬 그래프 직접 인스턴스화/라우팅 | 대응 없음 | Prime ECC는 z=32 부분병렬(HCU/GT) 구조로 그래프 통짜 배선 아님 |
| switching activity 기반 저전력 아키텍처 | 대응 없음 | 부분병렬·pipeline 구조라 완전병렬 activity 논거 직접 적용 불가 |

적용 가치: **낮음** — LDPC를 실리콘에 직접 인스턴스화한 완전병렬 디코더 실증으로 역사적 의의는 크지만, 코드가 배선에 고정되고 라우팅 혼잡이 병목인 완전병렬 방식은 z=32 부분병렬 QC-LDPC(HCU/GT/pipeline)인 Prime ECC HW와 구조가 이질적이라 직접 이식 가치는 낮다.
