### B. 알고리즘 요약

1. 대상: rate `1/2` irregular LDPC (`N=1020`, `M=510`, function degree `df=7`, variable degree `dv∈{2,3,7}`), sum-product 반복복호.
2. 문제: 임의 구조 LDPC를 `n`개 병렬 파이프라인 프로세서에 매핑할 때 메모리 접근 충돌로 iteration latency가 커진다 - 이를 최소화하는 NP-hard 조합최적화.
3. 아키텍처: `n` variable processor + `n` var2fun 뱅크 + `n` function processor + `n` fun2var 뱅크, 각 프로세서는 한 뱅크에만 write / 모든 뱅크 read, 사이클릭 read로 충돌 회피.
4. 핵심 기법1(mapping): 노드를 `n`색 모듈로 원(circle)에 coloring하는 문제로 모델링, K-L mincut 휴리스틱 변형으로 색 간 노드 이동.
5. 비용식: `Cost(vi)=Σ ej²` (`ej`=n-color circle상 `Nbr(fj)`를 잇는 최단 arc의 빈 색 개수), 제곱으로 큰 stall을 강하게 억제.
6. gain 갱신을 상태(`sj=|Nbr(fj)∩C1|`) 기반으로 재사용해 gain-update 복잡도를 `O(N²)`→`O(N)`으로 축소.
7. 핵심 기법2(scheduling): 매핑 후 각 프로세서 내 노드 처리순서를 Linear Ordering(NP-hard) node-compaction으로 국소 최적화, degree monotonic 강제.
8. 구현 디테일: edge word 크기 약 4bit, 로컬 버퍼로 memory miss stall 완화 (버퍼 32 word 이상은 이득 미미).
9. 결과: `n=16`에서 optimized mapping+ordering이 randomized 대비 75.1% latency 개선(zero buffer), 32-word 버퍼 시 speedup ≈ `1.12n-0.024n²`.
10. 한계: 순수 시뮬(PII 500MHz PC), 정정성능/BER 개선 아님, 임의 코드 전제 - 컴파일 타임 1회 실행용 최적화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 병렬 프로세서 노드 mapping/scheduling | 대응 없음 | Prime ECC는 z=32 고정 row병렬 QC 구조로 컴파일러형 임의매핑 불필요 |
| var2fun/fun2var 메모리 뱅크·라우팅 | 디코더 HW 모델(z=32, HCU, GT) | 메모리/라우팅 병목 개념은 겹치나 아키텍처가 다름 |
| sum-product(full-tanh) CN/VN 연산 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum 사용 - full-tanh는 대응 대상 아님 |

적용 가치: 낮음 - 임의 LDPC용 코드독립 컴파일 최적화라 고정 QC-LDPC HW인 Prime ECC 3.1에 떼어 쓸 부분이 사실상 없음.
