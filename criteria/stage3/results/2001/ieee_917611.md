### B. 알고리즘 요약

1. 시스템: 자기기록 partial-response 채널을 inner encoder로 쓰는 serial-concatenated turbo 구조. outer는 16-state conv 또는 rate 8/9 LDPC, sector 4096 user bits, throughput 목표 >1Gbps.
2. 문제: SISO 반복복호(MAP/SOVA/LDPC)의 VLSI 구현 시 연산HW·메모리·제어로직 비용이 큼. 이를 정량화하고 저복잡 구조 제안.
3. 모델: LLR을 `4~6bit` 고정소수로 표현(BER vs HW 트레이드오프). throughput 유지 위해 fully-unrolled+pipelined(반복수에 선형 복잡도 증가).
4. MAP 핵심: log-domain BCJR을 Add-Compare-Select-Add(ACSA)로 구현, `max*` 연산자는 comparator+LUT+adder로 근사.
5. Windowed-BCJR: 2개 overlapping window로 backward 연산해 메모리·지연 감소, branch-metric 메모리를 양방향 shift register로 구현(제어로직 축소).
6. SOVA: register-exchange survival memory + Compare-and-Mux(CAM, XOR 기반 등가검사)로 SRAM traceback 대체. ACSA 3→1로 66% 연산유닛/30% 메모리 절감(BER은 소폭 손실).
7. LDPC 핵심: 예시 부호 col-weight 4, row-weight 36, 512 check/4608 bit node. check-to-bit는 sgn+LUT(`f(x)=g(x)` 동일 LUT), bit-to-check는 4-input tree adder.
8. LDPC 메모리 설계: 비정규 H로 인해 각 행 메시지를 FIFO(row-weight 36)/4-entry stack에 저장 → 컬럼 인덱스 추적 불필요, 주소디코딩을 카운터/MUX로 대체.
9. 결과: LDPC는 연산유닛 극소(16 adder+8 LUT)지만 메모리 요구가 MAP/SOVA보다 2자릿수 큼(단일 iteration >73000 words). 정규/구조화 부호 필요성 지적.
10. 한계: BER 성능 개선이 목표 아님(구현 복잡도만), 비정규 부호 전제, HW 실합성/실측 없음(이론·아키텍처 분석).
