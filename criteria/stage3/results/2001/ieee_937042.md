### B. 알고리즘 요약

1. 시스템: outer code(TPC/SPC, TPC/MPC, LDPC 중 하나) + inner (precoded) PR 채널(PR4:1-D², EPR4:1+D-D²-D³, PR1:1+D, PR2:1+2D+D²), 자기/MO 기록. 유효 블록 L=4K bit.
2. 문제: 고밀도 기록에서 turbo 코드는 복잡도 과다, 대안으로 graph 기반 block 코드(LDPC/TPC) 성능·error 통계를 RS-ECC 관점에서 평가.
3. 코드 스펙: regular LDPC column weight `s=3`(rate 0.94), TPC/SPC는 (16/17)²=0.89, (32/33)²=0.94 2-D 곱부호; TPC/SPC 최소거리 항상 4(bit degree 2, cycle 길이 8).
4. 통합 복호: 세 코드 모두 bipartite graph에서 LLR message-passing; LDPC는 batch 모드, TPC는 두 성분부호 반씩 갱신하는 semi-sequential 모드(약간 빠른 수렴).
5. Turbo equalization: inner=BCJR log-MAP equalizer, outer=코드 디코더; extrinsic 정보를 상호 a priori로 교환.
6. Precoder 역할: PR 채널을 recursive하게 만들어 spectral thinning(저weight error event 다중도 감소)→interleaving gain. LDPC는 큰 최소거리라 이득 없어 precoder 없이 더 우수(precoding이 rate-0.89 LDPC/PR4에 0.5~1dB 손실).
7. Threshold 분석: density evolution + Gaussian approximation으로 mean `m^(q)` 추적, MAP inner는 Monte-Carlo로 `m_o=(m_i)` 관계 도출; TPC/SPC upper bound가 LDPC 대비 ~0.5dB 이내.
8. Error statistics: RS-ECC(symbol 단위 t=10~20 정정) 성공 여부는 BER이 아니라 error 분포가 결정; 160,000 블록 관찰.
9. 결과: PR4/EPR4에서 세 코드 BER 유사(무부호 대비 4.5~5dB gain), PR2에서 TPC/SPC가 LDPC보다 0.4dB 우위; LDPC는 bursty error, TPC는 소수 bit 오류 블록 다수→RS-ECC와 더 궁합.
10. 한계: HW 미설계·시뮬만, 자기/MO PR(ISI) 채널·precoder·BCJR 전제, 결론은 오히려 LDPC보다 TPC/SPC 추천.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC message-passing 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 개념적 대응이나 논문은 tanh 기반 SP·turbo equalization으로 min-sum·memoryless read와 다름 |
| turbo equalization / BCJR inner MAP | 대응 없음 | Prime ECC는 ISI 채널 equalizer 없음(NAND read는 memoryless) |
| precoder / PR 채널 모델 | 대응 없음 | Prime channel은 AWGN/RBER/fixed error, PR/ISI 미지원 |
| DE + Gaussian approximation threshold | 대응 없음 | 코드베이스에 threshold 분석 로직 없음 |

적용 가치: **낮음** — 자기/MO 기록 PR(ISI) 채널의 turbo equalization·precoding 비교 연구로, memoryless NAND read + binary QC-LDPC min-sum을 쓰는 Prime ECC와 채널·복호 구조가 근본적으로 다르며 LDPC를 오히려 열위로 결론 낸다.
