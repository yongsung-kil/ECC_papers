### B. 알고리즘 요약

1. 시스템: LDPC 반복복호(message-passing)를 재구성 가능 HW에 매핑하는 domain-programmable(code-independent) 병렬 아키텍처. 대표 부호 (510,1020, col-deg 2/3/7, row-deg 7) irregular LDPC, rate ~0.5.
2. 문제: LDPC 병렬성은 크지만 실HW 매핑 시 **통신(라우팅) 병목**이 지배적. 연산자원을 과다 투입해도 통신에 막혀 활용률 낮음.
3. 모델: factor graph의 logical variable/function node를 physical node에 매핑. 메시지 4-bit 해상도, 내부처리 7-bit 고정소수.
4. 핵심기법: 네트워크를 3개 동일 2D sub-network(function 1 + variable 2)로 분할해 각 타입 메시지를 **병렬 라우팅** → effective diameter 축소.
5. 핵심 아이디어: 라우팅/상태관리(physical node)와 연산(FU)을 **분리**하고 FU를 여러 physical node가 **공유** → 통신·연산 병렬성 decoupling, 각자 개별 최적화.
6. 확장: FU sharing-factor를 설계변수로. 최적값 2~4(sub-network 크기 무관) → 연산자원 50%+ 절감하면서 성능 저하 미미.
7. 구현 디테일: physical node = router + FU interface server + send server. logical node는 4-state FSM(wait/process/send/finished), self-synchronized 반복.
8. 라우팅: unidirectional 2D torus + store-and-forward + minimal dimension-ordered. deadlock 회피 위해 대각선 "hot node" + normal/urgent 우선순위 신규 알고리즘(virtual channel 불필요).
9. 결과: 동일 core FU 기준 full-parallel 대비 FU 활용률 +251%, sequential +116%, FPGA +209%; SGI 공유메모리 머신 대비 확장성 +75.6%, 성능 ~3자릿수 개선. 25 iter 프레임당 cycle 예: full-parallel 975.
10. 한계: 부호정정 성능 개선 아님(구현 효율만), code-independent 재구성 지향(software-radio), 실측/합성 없이 cycle-accurate C++ 시뮬, 일반 factor-graph 대상.
