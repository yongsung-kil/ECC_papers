### B. 알고리즘 요약

1. 시스템: 짧은 블록(<10000bit) BPSK/AWGN용 (3,k)-regular LDPC, 예제 `N=L·k²` (L=64→N=2304, L=128→N=4608, k=6, rate≈0.5), soft BP(sum-product) 복호.
2. 문제: fully-parallel 디코더는 복잡도 과다, 랜덤 LDPC는 partly-parallel 변환이 사실상 불가능 — decoder-first 설계[3]는 random number generator 다수로 복잡.
3. 핵심기법: joint code&decoder 설계 — 결정론적 high-girth (2,k)-regular 부호 `C2`에 디코더가 랜덤 check node를 삽입해 (3,k)-regular 부호 앙상블 정의.
4. 핵심구조: `H=[H0^T,H1^T]^T`, `H0`는 L×L identity 블록, `H1`은 identity의 cyclic shift 블록 `P_{x,y}=T^u(I)`, `u=((x-1)·y) mod L` — circulant-shift 기반.
5. 핵심정리(Thm 3.1): `L`이 `a·b (a,b∈{0..k-1})`로 인수분해 안 되면 girth=12 보장, 짧은 cycle 억제로 좋은 성능.
6. 디코더 아키텍처: `k²` memory bank, k개 CNU, k² VNU, 1-layer+g-layer shuffle network, Address Generator(modulo counter), 한 iteration을 `3L` clock에 완료.
7. 부호선택: 500개 랜덤 생성 후 girth average 높은 것 5개 → 시뮬로 최적 선택; 4-cycle-free 보장, redundant check 2개로 실제 rate 약간 상승.
8. 효율 인코딩: column permutation `π_c`로 approximate upper-triangular `H^en` 생성, sparse 서브행렬 D/E/T/A/B와 `T^{-1}=T` 성질 활용해 선형 복잡도 인코딩(Richardson-Urbanke 개선).
9. 결과: 4bit 수신/6bit intrinsic·extrinsic 유한정밀 시뮬에서 implementation-oriented 부호가 fully-random (3,6)-regular 부호와 거의 동일 BER(성능 손실 없음), 단 random은 partly-parallel 디코더 구현 불가.
10. 한계: 부호 성능 자체는 개선 없음(구현성이 목적), gatecount/throughput 수치 미제시, general (j,k)/irregular 확장은 future work.
