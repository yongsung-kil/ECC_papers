### B. 알고리즘 요약

1. 대상: 단~중길이(수백~수천 bit) LDPC 및 DSC(different set cyclic) 부호, 예시는 (3,6) Gallager 부호 `(504,252)`, AWGN.
2. 문제: BP 복호는 MLD에 못 미치고(suboptimum), MLD는 해당 부호에 구현 불가할 만큼 복잡 → 그 성능 격차를 메우고 싶음.
3. 핵심: 매 BP iteration의 soft-output 값을 신뢰도(reliability)로 보고, 저복잡도 soft-decision 복호를 추가 수행.
4. soft-decision 복호로 OSD(ordered statistic decoding)의 order-1 reprocessing 선택 → 복잡도를 감당 가능 수준으로 유지.
5. 의미: 각 iteration이 BP + order-1 OSD로 완성되어 BP와 MLD 사이 성능 격차를 bridge.
6. 조기종료: syndrome check 및 test `T(2)` 기반 stopping criterion으로 평균 iteration 수(따라서 평균 OSD 횟수) 감소.
7. 부수 효과: iteration당 연산은 늘지만 목표 성능 도달에 필요한 iteration 수는 감소.
8. 부가 가치: iterative 복호의 suboptimality와 부호 구성 자체의 성능 손실을 분리 측정하는 도구로 활용 가능.
9. 결과: `(504,252)` (3,6) 부호에서 WER `10^-3.5`, order-1 reprocessing + 최대 50 BP iteration이 최대 200 iteration BP 대비 0.55 dB coding gain.
10. 한계: 1쪽 요약, 시뮬만이며 HW·복잡도 정량·긴 고rate 부호 결과 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP + OSD 결합 복호 루프 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프에 후처리를 얹는 위치는 우리 decoder지만 min-sum 기반이라 OSD 정렬·소거 연산 삽입 부담 큼 |
| iteration별 soft-output → 신뢰도 후처리 | `decoder.cpp` `C2V_Cal()`, `VN_Cal_HD()` 등 | soft-output 활용은 대응되나 order-1 OSD 자체는 대응 모듈 없음 |
| 조기종료(syndrome check/stopping) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 우리도 Partial/Full CRC 조기종료 보유 — 중복 기여 |

적용 가치: **낮음** — 디코더 후처리(near-MLD) 방향은 관심 영역이나 order-1 OSD의 정렬/Gaussian 소거가 min-sum HW 파이프라인에 무겁고, 조기종료는 이미 보유(CRC)라 중복이다.
