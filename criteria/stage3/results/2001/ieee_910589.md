### B. 알고리즘 요약

1. AWGN 채널, 중~고rate(`R>1/2`) 대상. CT 부호는 큰 span의 SPC tree code 여러 개를 turbo식으로 병렬 concatenate한 특수 LDPC로, interleaver 길이 `L=1024~65536` 시뮬.
2. 문제: 순수 LDPC(SPC 구성)는 수렴 느리고, turbo(BCJR)는 iteration당 연산·메모리 큼 — 둘의 장점(저비용+빠른 수렴+고성능)을 동시에 얻고자 함.
3. 각 constituent code는 loop 없는 tree code(Tanner graph)이며 Bayesian network로 edge 방향 따라 쉽게 인코딩(일반 LDPC의 복잡한 인코딩 회피).
4. `M`개 local tree encoder를 interleaver로 묶어 전체 CT 부호 구성. 정보노드 degree는 `M`배 증가하고 전체 그래프엔 loop 발생.
5. 핵심 설계기준: 모든 정보비트가 recursive part에 최소 2회 등장해야 좋은 성능(turbo의 interleaving gain 원리) — low-weight codeword 다중도 감소.
6. Interleaver 설계: constituent code당 2개의 독립 random interleaver(recursive `J_r`/nonrecursive `J_n` 분리)로 모든 비트가 정확히 `J_r`,`J_n`회 등장 보장.
7. 복호: serial(two-way schedule) turbo식 BP 복호 — local APP 복호기들이 순차로 정보비트 LLR(extrinsic) 교환, delayed extrinsic 감산으로 순환 방지. parallel(flooding)보다 수렴 빠름.
8. 복잡도(Appendix): CN식(5)·VN식(6)이 edge당 ~3 mult 수준. 전체 42 mult+35 add/info-bit/iter로 16-state turbo(256 mult+192 add) 대비 ~6배 저복잡, likelihood-ratio 방식[28]으로 양자화 비트 절감 가능.
9. 결과: `R=1/2`,`L=1024`에서 (J_r,J_n)=(3,1)이 (4,0)보다 BER 1e-4에서 ~0.3 dB 우수, `M=4` 최적. 고rate(1/2,3/4,9/10) 모두 이론한계 ~0.5 dB 이내, 짧은 L에서 LDPC/RA 능가.
10. 한계: HW 미설계·시뮬뿐, error-floor는 추측(2-interleaver로 uniform interleaver 가정 불가 → weight-distribution bound 미완), turbo계열이라 interleaver 필수.
