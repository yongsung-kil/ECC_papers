### B. 알고리즘 요약

1. 대상은 실시간 셀룰러 통신용 "LDPC turbo code" 디코더의 VLSI 구현, 채널은 AWGN, 송신단은 J개 dimension의 정보/패리티 행렬을 인터리빙하여 전송.
2. 문제: 기존 iterative 디코더는 조합산술·지수/로그 연산·긴 인터리빙·대용량 중간저장 메모리로 VLSI 구현이 복잡·대면적.
3. 복호 알고리즘은 PLR(Parity Likelihood Ratio) [ref 4] — Sum-Product의 확률 뺄셈을 나눗셈으로 치환한 고속 근사, 핵심 함수 `f(x,y) = (1+xy)/(x+y)`.
4. 복호는 Initialization → Updating(`q` 갱신) → Horizontal Forward(`F,G`) → Vertical Backward/Forward → Horizontal Backward → Extrinsic(`U`) → Output 단계를 dimension별·iteration별 재귀 수행, 한 dimension의 extrinsic을 인터리빙해 다음 dimension으로 전달.
5. 양자화: ADC 출력을 64레벨(base 2)로, 로그영역 6-bit sign-magnitude(레벨인덱스 ±0…±31)로 표현.
6. 핵심 HW 트릭: PLR의 horizontal step을 `64x64` 레벨인덱스 룩업테이블 검색으로 대체해 산술 제거.
7. 메모리: 3개 RAM(q, q~, G)에 대해 나눗셈·f함수 3연산을 병렬 수행, in-place 알고리즘으로 중간값 별도 저장 없이 동일 위치 갱신.
8. 주소생성: dimension controller의 14-bit 카운터(dimension 2b + stage 2b + row 8b + col 2b), stage 3은 역순 처리를 주소 반전으로 down-counter 없이 구현. 인터리버는 랜덤 패턴을 ROM에 저장해 인터리빙 latency 제거.
9. 파라미터: J=4 dimension, m=256 row, n=4 col, 16 iteration. 합성은 Synopsys, 0.35um CMOS, 3.0V, 최악 동작주파수 85MHz. 블록 처리 16,777,216 클록 → 입력 1.328MHz에서 블록당 0.197s (< 0.2s 실시간 가능).
10. 한계: FSM과 메모리 요구로 칩면적이 여전히 큼(주 단점), 정정능력 개선 없음(구현 논문), 향후 병렬 FPGA 재구성 아키텍처 제안. BER/게이트카운트 수치는 본문 서술에 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PLR `f(x,y)=(1+xy)/(x+y)` 룩업 복호 | 대응 없음 | Prime ECC는 min-sum(min1/min2+sign), PLR/Sum-Product 계열 룩업 방식과 무관 |
| 로그영역 6-bit sign-mag 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` (양자화) | 양자화 개념은 겹치나 PLR 레벨인덱스 방식이 달라 직접 이식 불가 |
| in-place 메모리 / ROM 인터리버 | 대응 없음 | turbo dimension 인터리버 구조로 QC-LDPC circulant 라우팅과 무관 |
| 조기종료(final iteration 결정로직) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | Prime ECC는 CRC 기반 조기종료로 접근이 다름 |

적용 가치: **낮음** — turbo/PLR 계열 디코더의 오래된(2001, 0.35um) VLSI 구조로, 부호종류·복호 알고리즘·인터리버 구조 모두 Prime ECC binary QC-LDPC min-sum과 계열이 달라 떼어 쓸 요소가 거의 없음.
