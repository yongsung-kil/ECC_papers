### B. 알고리즘 요약

1. (155,64,20) Tanner 코드가 AWGN 채널에서 Linear Programming(LP) 디코딩될 때 발생하는 pseudo-codeword(유효거리가 Hamming거리보다 짧음) 기인 error-floor를 다룸.
2. 기존 BP/LP는 Bethe free energy 최소화의 근사(트리 가정)라 그래프에 루프(loop)가 있으면 정확도가 떨어지고, 이 루프가 error-floor의 원인이 됨.
3. 핵심 가정: 각 BP 실패 사례(instanton)마다 partition function에 bare BP와 비슷한 크기로 기여하는 "critical loop" `Γ`가 하나 존재한다고 가정(conjecture).
4. loop calculus(선행연구[1][2])로 partition function `Z = Z0(1 + Σ_C r(C))`를 표현, `r(Γ)`가 1에 가까운 single-connected loop을 triad 곱 `µ̃α(bp)`로 탐색(threshold 낮춰가며 검색).
5. `r(Γ)=1`이면 해당 loop 기여가 bare BP와 동등해 디코딩 실패의 직접 원인임을 의미 — critical loop 식별이 핵심.
6. 확장 기법: critical loop을 반영해 Bethe free energy `F`를 수정(`F = -ln Z0 + Σ_Γ Z_Γ`)하는 "modified BP equation"을 유도(Loop-corrected BP, 논문 내 수치실험은 없음).
7. 실전 구현은 휴리스틱 단순화인 "LP-erasure": LP 실패 시 critical loop 위 bit들의 log-likelihood를 완전/부분 소거(ε 배율, 논문 실험은 ε=0 완전소거)한 뒤 LP 재실행.
8. 학습/최적화 절차는 없고, critical loop 탐색 자체가 threshold 조정 기반 수작업 휴리스틱.
9. 결과: (155,64,20) 코드에서 기존 발견된 ~200개 pseudo-codeword instanton 전부를 LP-erasure로 교정, 최소 LP-instanton 유효거리 ≈16.4에서 Hamming거리 20까지의 error-floor 갭을 완전히 해소.
10. 한계: 단일 소형 코드(n=155)의 proof-of-concept이며 BER/FER Monte-Carlo 비교, 다른(긴) 코드, 표준 min-sum BP 적용, 복잡도/자동화는 모두 향후 과제로 남김. HW 구현 없음.
