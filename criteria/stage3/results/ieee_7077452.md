### B. 알고리즘 요약

1. 대상은 임의의 구조화/비구조화 (N,K) LDPC 코드이며, 표준 min-sum이 아닌 tanh 기반 sum-product BP 디코딩(`R(l)_m→n`, `S(l)_n`)을 사용한다.
2. 기존 프로그래머블(재구성형) 디코더는 휴리스틱 메모리 매핑으로 인해 collision을 완전히 피하지 못해 idle cycle이 발생하는 문제가 있었다.
3. 핵심 가정은 파티셔닝 함수 `T: {v0,...,vE-1} → {0,...,p-1}`로, 같은 서브셋 `Vk`에 속한 두 값이 항상 서로 다른 메모리 뱅크에 매핑되도록 보장하는 것이다.
4. 핵심 기법: sum value `Sn`을 변수노드 degree만큼 복제한 뒤(step1: E개 copy에 access index 부여), 각 access index별 서브셋을 partition P/P'로 재배열해 동적 매핑 함수 T를 적용(step2), 최종적으로 복제본을 제거해 N개 원본 sum value만 저장하되 반복마다 서로 다른 뱅크를 오가도록 함(step3).
5. 이 동적 매핑(정적 mapping을 쓴 기존 [7] 대비)이 핵심으로, extrinsic value 대신 sum value에 대해 충돌 회피를 적용해 check/variable node를 동시에(one-phase) 병렬 처리 가능케 한다.
6. 확장: flooding schedule 구조(2p unit 병렬 갱신) 외에 layered decoder 구조도 제시, 메모리(SMBi, IMi, 2개 permutation network) 절감.
7. 부수 구현: permutation network는 Beneš network로 구현, layered 구조는 짝/홀 iteration마다 체크노드 처리 순서를 역순으로 바꿔 수렴속도 개선.
8. layered 구조의 체크노드 처리 순서 재배열에는 simulated annealing을 사용해 idle state를 5% 미만으로 최소화.
9. 결과: 기존 정적매핑 프로그래머블 디코더([7]) 대비 clock당 update 수 p→2p(2배), 기존 휴리스틱 매핑([5],[6]) 대비 유틸리제이션 40~80%→100%; 단, 동일 parallelism에서 QC 전용 디코더([9])와 throughput은 동일(우위 없음).
10. 한계: ASIC/FPGA 합성 결과 없이 이론적 구조·비교표(Table 1)만 제시, gatecount/throughput 수치 미제공, unstructured 코드 지원을 위해 control memory에 임의 H-matrix 정보를 저장해야 하므로 QC 대비 메모리 오버헤드가 본질적으로 큼(저자도 인정).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| collision-free 동적 메모리 매핑(임의 코드용) | 대응 없음 | Prime ECC는 고정 QC-LDPC PCM(`PCM_b`)에 특화된 z=32 barrel-shift 병렬구조라 임의 코드 지원용 동적 매핑이 불필요 |
| one-phase message passing / layered scheduling | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 옵션) | 개념적으로 스케줄링 방식과 관련되나 Prime ECC는 이미 QC 고정 구조에서 parity even/odd 교대(Dual-Update)를 보유, 중복 |
| permutation network(Beneš) 기반 라우팅 | 대응 없음 | Prime ECC는 QC 구조상 barrel shifter로 충분해 범용 permutation network가 불필요 |

> 적용 가치: 낮음 — 임의 LDPC 코드 지원이 핵심 목표인 범용 재구성형 아키텍처로, 고정 QC-LDPC 하나에 최적화된 Prime ECC의 설계 철학과 반대 방향이며, 저자 스스로 QC 전용 대비 throughput 이점이 없다고 명시.
