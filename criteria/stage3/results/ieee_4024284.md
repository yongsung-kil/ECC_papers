### B. 알고리즘 요약

1. 채널: BSC, 복호: Gallager B(=column weight 3에서는 Gallager A와 동일) 알고리즘, 메시지는 binary(hard-decision)만 전달.
2. 문제: 낮은 SNR(높은 α) 영역이 아닌 error floor 영역은 Monte Carlo 시뮬로 확인 불가능하여 반해석적 추정법이 필요.
3. 가정: 디코더 고정점(fixed point)은 코드워드가 아닌 `(v,c) trapping set`으로 정의(`v`개 variable node, 유도 부분그래프에 `c`개의 odd-degree check).
4. 핵심 기법: trapping set 클래스 `X`마다 critical number `m`(그 trapping set으로 끝나는 데 필요한 최소 에러 수)과 strength `s`(실제 decoding failure를 유발하는 조합 수)를 정의.
5. 핵심 식: `Pr{X|r-errors} ≈ s|X|·C(r,m)/C(n,m)`, `Pr{r-errors}=C(n,r)α^r(1-α)^(n-r)` — trapping set 클래스의 FER 기여도를 조합론으로 계산.
6. 확장: sensitivity `β`(trapping set 밖 노드가 추가로 에러일 때도 여전히 그 trapping set으로 수렴할 확률)를 정의했으나 계산 결과 β≈1로 근사해도 무방함을 확인.
7. 구현 디테일: column-weight-3 코드에서는 trapping set이 girth `g`, `g+2` cycle 및 그 union으로 환원되어 그래프 탐색(cycle 카운팅)만으로 클래스 크기 `|X|` 산출 가능.
8. 최적화 절차 없음(탐색 기반 열거, 학습 알고리즘 아님). `M`(trapping set에 흡수되는 최대 에러 수)은 휴리스틱+시뮬+최소거리로 결정.
9. 결과: Margulis(2640,1320), Tanner(155,64), MacKay 랜덤코드, QC코드 3종에 대해 이론 FER 곡선이 시뮬레이션과 낮은 α 영역에서 잘 일치, error floor를 정확히 예측.
10. 한계: BSC + Gallager B(hard-decision) 전용, column-weight-3 코드로 한정, AWGN/soft-decision 및 HW 설계는 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping set 기반 error-floor 분석 | 대응 없음 | Prime ECC는 min-sum(soft) 디코더이며 hard-decision Gallager B 이론과 직접 대응하는 모듈 없음 |
| Gallager B 메시지 패싱(1-bit hard) | `decoder.cpp` `VN_Cal_HD()` | HD(hard-decision) 모드가 존재해 개념적으로만 참조 가능 |
| H-matrix 구조/girth 분석 | `ecc_top.cpp` `Load_PCM()` | 코드 구조 분석 시 참고 가능하나 이 논문은 부호설계 도구 자체를 제공하지 않음 |

> 적용 가치: 낮음 — soft-decision QC-LDPC HW 디코더가 목표인 Prime ECC와는 채널/디코더 모델이 근본적으로 다르며(hard 1-bit vs soft multi-bit), 코드 자체보다는 error-floor 원인론(trapping set 개념)으로서만 참고 가치가 있음.
