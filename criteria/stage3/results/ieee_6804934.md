### B. 알고리즘 요약

1. 시스템: 4-level MLC NAND, 고rate LDPC(1KB, rate 0.88, fixed-point min-sum, quantized LLR)를 outer, tensor-product classification code(CC)를 inner로 concatenate(Fig.2).
2. 문제: soft read는 다중 read로 read time·채널 점유(throughput)가 커 비싸다; hard read만으로 coarse soft를 만들고 싶다.
3. 모델: 등간격 4-level Vt에 동일 분산 AWGN, RBER 1E-3~1E-2 영역.
4. 핵심기법: CC = C1(single-parity, "chunk") ⊗ C2(t-정정 알고리즘 부호)의 tensor product → "location code". syndrome이 오류 chunk 위치를 지목.
5. 핵심식: `NSoftBits = log2(Nreads+1) - 1`; CC는 6개 구분 LLR로 `NSoftBits ≤ log2(6)-1 = 1.585` bit 상당의 soft를 hard read만으로 생성.
6. LLR 배정: ≤t chunk 오류→해당 chunk "weak", 나머지 "strong"; t+1→전부 "don't know"; t+2 이상→오배정(성능 소폭 저하, RBER 불변).
7. 구현 디테일: 예) C1=(25,24)⊕C2=(8,4) Hamming → CC (200,196) rate 0.98(2% overhead). chunk 짧을수록 fine soft이나 오버헤드↑. LLR은 RBER 무관 사전값.
8. 분석: CC 유무의 LDPC 입력 MI를 비교(Fig.5), 5% CC가 고RBER에서 2%보다 MI 개선 큼.
9. 결과: 3 iter, RBER 3.7E-3에서 CWER 1E-5 달성(CC 없으면 2.8E-3); RBER<5E-3에서 avg iteration ~16% 감소(2.47→2.06), CWER-RBER 곡선 기울기 급해짐.
10. 한계: 반복수 충분 시 동일 14% 예산을 순수 14% LDPC에 쓰는 편이 우세(CC는 정정 안 함). HW 미설계·시뮬만, turbo-equalization은 미탐색.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CC가 생성한 weak/strong/don't-know 3레벨 soft LLR | [ecc_data.h](../Prime_ECC_3.1_Claude) `PARAM_LLR`, [decoder.cpp](../Prime_ECC_3.1_Claude) `Get_VNU_Table_Idx()` | coarse soft를 LLR 테이블로 매핑하는 부분과 연결 (2SD/3SD와 유사 정밀도) |
| hard read → soft 생성(추가 read 회피) | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | soft-read 비용 절감 관점에서 채널 LLR 생성 로직과 접점 |
| CC decoder(syndrome 위치검출) 자체 | 대응 없음 | tensor-product inner code는 Prime ECC에 별도 신설 필요 |
| min-sum scalar/offset을 RBER 추정으로 조정 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` | CC의 RBER 추정으로 min-sum 파라미터 조정하는 개념적 접점 |
| avg iteration 감소(조기수렴) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 빠른 수렴은 조기종료와 시너지, 다만 기여 원천은 다름 |

적용 가치: **중간** — soft-read 없이 coarse LLR을 만들어 avg iteration을 줄이는 발상은 Prime ECC의 read-cost 절감 목표와 부합하나, tensor-product CC 인코더/디코더를 새로 붙여야 하고 순수 고rate LDPC 대비 이득이 저-iteration 구간에 한정된다.
