### B. 알고리즘 요약

1. 대상: partially-parallel LDPC 디코더에서 P개 PE가 B(=P)개 메모리 뱅크를 매 time instance마다 충돌없이 read/write하는 문제.
2. 문제: K개 데이터를 B뱅크에 배치해 각 시점 P개 PE가 병렬 접근 시 같은 뱅크에 중복 접근(collision)이 없어야 함. 기존 SA 기법은 종료시점 예측불가, node/edge coloring은 데이터당 1뱅크 제약으로 뱅크 과다.
3. 모델: data access matrix (P행 × N열), 반복 처리로 각 데이터가 여러 시점 접근됨. Tanner graph 기반 bipartite graph `G=(T∪L,E)` (T=시간노드, L=데이터노드)로 모델링.
4. 핵심 기법: multiple read/write access 개념 도입 — in-place가 아니라 한 뱅크에서 읽고 다른 뱅크에 쓰기 허용해 최소 뱅크 수 달성.
5. 제약: (1) 매 시점 모든 뱅크 1회씩만 사용, (2) 데이터의 마지막 write 뱅크 = 첫 read 뱅크. LDPC 반복 접근 특성상 그래프는 항상 semi-regular(모든 time node 동일 degree `d_t`).
6. Step I: data access matrix로 bipartite graph 구성, direct edge(read) / induced edge(write) 구분(placement property).
7. Step II: 그래프를 proper partition으로 분할. partitioning constraint(파티션 내 시점당 read/write 2개 이하) 하에 traversal+elimination 프로세스로 path 구성. Lemma 1/2로 `d_t/2`개 파티션.
8. Step III: 각 파티션을 최대 2색으로 coloring — read 충돌 제거 후 induced edge 추적하며 write 충돌 없게 색 배정(recursive).
9. 결과: K=6,P=B=3 예제로 결정론적 충돌없는 매핑 도출. Konig 정리로 색수 = 최대 degree 보장.
10. 한계: 방법론/의사예제뿐, 실제 HW 합성·gatecount·throughput 수치 없음. barrel shifter/butterfly/de Bruijn 등 특정 interconnect 미지원(future work).
