### B. 알고리즘 요약

1. 시스템: 종단(terminated) LDPC convolutional code = SC code, (3,6)-regular, rate `R_L=1-(1+ms/L)(1-R)`, base matrix `B=[3 3]`, L=40.
2. 문제: SC 앙상블 중 최소결합(`ms=1,2`)에서 어떤 base matrix 분해가 최선의 반복복호 임계값을 주는지 미정.
3. 모델: BEC(binary erasure channel), belief-propagation, protograph 기반 copy-and-permute로 H 생성.
4. 핵심기법: base matrix `B`를 `B_i=[a_{2i-1} a_{2i}]` 서브행렬로 분해, `ms+1` bin에 `b1,b2` ball 분배하는 조합수 `C(b1+ms, ms)·C(b2+ms, ms)`로 전 앙상블 전수열거(ms=1→16개, ms=2→100개).
5. 핵심 식: `RL = 1-(1+ms/L)(1-R)` — 종단이 rate loss와 임계값 개선을 동시 유발.
6. 확장: window decoder(윈도우 크기 W)로 BP를 부분수행, 지연 축소; W=4,6에 대해 임계값 비교.
7. 구현 디테일: 목표심볼 erasure 확률이 `δ=10^-6` 이하로 내려가면 성공으로 정의.
8. 최적화: DE 반복식 `q_{i→j}=1-∏(1-p)`, `p_{j→i}=ε·∏q`로 임계값 `ε*` 산출, mirror 앙상블은 동일 임계값.
9. 결과: ms=1 최선 `[22 11]` block `ε*=0.4830`, window `ε*=0.4843`; SC의 window/block 임계값 모두 (3,6) block LDPC보다 우수. `a1=a2=2` 권장 등 설계규칙 5개 제시.
10. 한계: BEC 점근 임계값·이론뿐, HW 미설계, AWGN/NAND 채널·유한길이 성능·복잡도 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC base matrix 구성/분해 | 대응 없음 (Prime ECC는 QC-LDPC 고정, SC 미지원) | SC↔QC 구조 상이, 부호구성 이식 불가 |
| BEC density evolution 임계값 | 대응 없음 | Prime ECC는 AWGN/RBER 채널(channel.cpp), BEC 미대상 |
| window decoding 스케줄 | 대응 없음 | Prime ECC 디코더는 block(전그래프) 스케줄, window decoder 없음 |
| BP iterative message-passing | decoder.cpp LDPC_Decoder() (개념적 유사) | Prime ECC는 min-sum, 논문은 BEC BP(erasure)라 직접 대응 아님 |
```

적용 가치: 낮음 - BEC 상 SC-LDPC 부호구성·점근이론으로 NAND용 binary QC-LDPC(부호종류·채널·rate 모두 상이)에 떼어 쓸 요소가 없음.
