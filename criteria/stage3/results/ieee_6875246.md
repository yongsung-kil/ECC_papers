### B. 알고리즘 요약

1. 채널은 non-ergodic block-fading(코드워드를 F개 서브블록으로 나눠 각기 독립 Rayleigh fading `aj`), BPSK, 수신 LLR `Li = 2aj yi / σ²`(식2).
2. 문제: block-fading에서 full diversity를 주는 root-LDPC는 diversity>2를 위해 rate<1/2가 필요하고 설계가 복잡함.
3. 성능지표는 outage probability `Pout = P(I(x;y|α) < R)`(식3)와 diversity order `d = -lim log(Pwe)/log(γ)`(식5).
4. 핵심 기법: protograph 기반 (3,6)-regular LDPC를 edge spreading으로 spatially couple, 결합 memory `mcc`가 최대거리를 정의하고 base matrix가 종단(terminated) convolutional 형태(식7).
5. 핵심 통찰: 각 check node가 최대 `F(mcc+1)`개의 독립 fading에 연결되므로, R·F를 유지한 채 `mcc`만 키우면 diversity order가 증가.
6. 성능 계산은 정확 density evolution 대신 reciprocal channel approximation(RCA)으로 fixed SNR의 PDEO(식6)를 Monte Carlo로 산출.
7. Table I: mcc=1→d=3, mcc=2(EnsA2)→d=6, mcc=3,4→d=10 (R=1/2 유지); memory 3→4는 큰 이득 없음(node degree 동시 증가 필요).
8. SC-LDPC는 특정 F에 맞춘 설계 불필요, F 변동에 robust(root-LDPC 대비 강점).
9. 결과: 유한길이(N=200, F=2, 최대 50 iter) 시뮬 WER이 DEO 예측과 거의 일치, latency 제약 sliding window decoder(Wmin=5~10)로 block decoder 대비 성능손실 미미.
10. 한계: 무선 block-fading 특화, HW/양자화/soft-read 미설계, diversity 상한의 해석적 bound 미확보.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC(spatially coupled) 부호구성 | 대응 없음 | Prime ECC는 고정 QC-LDPC(base+circulant) 구조, SC-LDPC convolutional 구조와 비호환 |
| edge spreading / protograph 결합 설계 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 로드는 존재하나 논문의 SC 구조 자체를 담을 수 없음 |
| sliding window BP 디코딩 | 대응 없음 | Prime ECC는 block QC-LDPC min-sum 이터레이션, windowed convolutional decode 부재 |
| block-fading outage/diversity 해석 | 대응 없음 | NAND 채널(AWGN/RBER) 모델과 무관 |

적용 가치: 낮음 — 무선 block-fading용 SC-LDPC diversity 이론으로, NAND binary QC-LDPC 부호/디코더/HW 어느 레이어에도 직접 이식할 요소가 없다.
