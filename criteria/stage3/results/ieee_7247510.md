### B. 알고리즘 요약

1. 짧은 블록길이(N=100~576) regular/irregular LDPC 코드의 AWGN 채널 SPA 디코딩에서 수렴 지연 문제를 다룸.
2. 기존 RBP(residual belief propagation)는 모든 엣지의 잔차 `r(µ_ci→vj) = |µ_ci→vj^(k) - µ_ci→vj^(k-1)|`를 매 스텝 재계산해야 해 스케줄링 자체의 계산비용이 과도함.
3. 관찰: 초기 상태에서 최대 잔차는 항상 입력 LLR 크기(reliability)가 가장 작은 두 엣지에서 발생한다는 경험적 사실을 이용.
4. 제안하는 Rel.-RBP는 각 check node마다 신뢰도(입력 메시지 크기)가 가장 작은 두 메시지 `µ_vn1→cm`, `µ_vn2→cm`만 골라 그 두 개에 대해서만 잔차 `Rc_m^(k+1)`를 계산, 전체 엣지 대비 재계산량을 대폭 축소.
5. check-node-residual `Rc_a^(k+1) = max_m Rc_m^(k+1)`이 가장 큰 check node를 골라 해당 메시지만 업데이트하는 순차(one-message-at-a-time) 스케줄.
6. 불필요한 재계산 방지를 위해 이진 indicator 벡터 `i_m`을 두어, 새 정보가 없는 엣지는 재계산 생략.
7. 반복 횟수 비교의 공정성을 위해 "modified iteration"(check node 업데이트 총 횟수를 flooding/layered와 동일하게 맞춘 지표) 개념을 별도 제안.
8. 학습/최적화 절차 없음 (규칙 기반 휴리스틱 스케줄링).
9. 결과: modified iteration 기준으로 RBP/NS-BP 대비 훨씬 빠른 수렴, check node update 연산량은 Table I 기준 RBP/NS-BP의 `M·dc(dv-1)(dc-1)` 대비 Rel.-RBP는 `2M·dc(dv-1)`로 감소.
10. 한계: 모두 이론/시뮬 결과이며 HW 구현·파이프라이닝·병렬화 논의 없음, NAND/SSD 관련 채널·응용 언급 없음.
