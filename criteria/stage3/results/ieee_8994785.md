### B. 알고리즘 요약

1. 시스템: `n=1296`, `dv=4`, `dc=8`, rate `0.5` QC-LDPC code를 대상으로 코드워드 생성·노이즈 주입·복호·통계까지 전 시뮬 흐름을 FPGA(Xilinx Zynq XC7Z020)에 구현.
2. 문제: LDPC 오류특성/error-floor 분석은 `10^-12` 수준까지 `10^14` 코드워드 처리가 필요해 CPU/GPU로 수년~수십년 소요.
3. 대상 알고리즘: hard-decision bit-flipping류 — Gallager B(`GaB`)와 Probabilistic Gallager B(`PGaB`), CNU는 xor, VNU는 majority vote.
4. 핵심 기법: controller/codeword gen/noise gen/RNG/decoder/statistic의 7모듈 파라미터화 테스트베드, 디코더 모듈만 교체 가능(HDL instantiation).
5. `PGaB`: 비수렴 cyclic 상태에서 1-bit RNG값 `P=1`일 때 채널입력 `rn`을 반전시켜 trapping set 탈출 시도.
6. 파라미터: `Alpha`(crossover prob), `max_iter`, `max_cwf`(=100 기본)로 사용자가 해상도 설정.
7. GPU 베이스라인: Tesla K40 CUDA, stream 24·batch 16에서 백만 코드워드당 6.5분(비교 기준용).
8. 결과(throughput): FPGA 테스트베드 9172 Mb/s, GPU 3608 Kb/s, CPU 219 Kb/s (standalone PGaB는 15.69 Gb/s).
9. 결과(시간): `Alpha=0.005`에서 FER `0.917×10^-11`를 24시간에 도달(GPU 12년, CPU 199년 추정); GaB 4-error 패턴 전수탐색(117B 코드워드)을 4.5시간에 완료.
10. 한계: 디코더 알고리즘 개선이 아니라 시뮬 가속 프레임워크, hard-decision류 전용, soft-decision/min-sum은 미대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FPGA 시뮬 테스트베드 전체 | 대응 없음 | Prime ECC는 C++ bit-exact 시뮬레이터, FPGA 검증 인프라와 층위 다름 |
| trapping set / 4-error 패턴 탐색 | `corner.cpp` (수렴실패 추적) | 개념적 유사 — 수렴실패 코드워드 추적 기능과 목적 겹침 |
| GaB/PGaB hard-decision 디코더 | 대응 없음 (min-sum 기반이 아님) | Prime ECC는 min-sum, bit-flipping류 아님 |
| LFSR 기반 노이즈/RNG | `channel.cpp`, `random.cpp` (XOR25 난수) | 에러 주입 개념은 유사하나 알고리즘 무관 |

적용 가치: **낮음** — 디코더 알고리즘이 아니라 오류특성 분석용 FPGA 인프라이고 대상이 hard-decision bit-flipping이라, Prime ECC의 min-sum QC-LDPC 파이프라인에 떼어 쓸 요소가 없음(방법론적 참고만 가능).
