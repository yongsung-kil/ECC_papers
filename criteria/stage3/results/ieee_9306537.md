### B. 알고리즘 요약

1. 시스템: QC-LDPC 디코더, layered scheduling 기반, WiMAX rate 5/6 코드(length 2304, `z=96`, base matrix 4행)로 검증.
2. 문제: 기존 unrolled layered 아키텍처는 array LDPC(음수 엔트리 없는 base matrix)에만 효율적이고, 일반(non-array) QC-LDPC는 negative entry가 있는 layer에서 AP-LLR을 그냥 통과시켜 저장 자원을 낭비.
3. 핵심 가정: 여러 개의 연속 layer를 하나의 generalized layer(GL)로 묶어 처리하면 파이프라인 단수와 메모리를 줄일 수 있다는 관찰.
4. 핵심 기법: residue 기반 layered scheduling(`δn`)을 도입해 같은 GL 내 동일 컬럼의 중복 업데이트로 인한 Read-after-Write 해저드를 residue 변수로 우회.
5. 핵심 식: `γn = αm,n + βm,n_new + δn` (no-hazard), `δn = βm,n_new − βm,n_old` (hazard 시) — GL 병합 시에도 AP-LLR 값을 정확히 유지하기 위한 보정.
6. 하드웨어 구조: 각 GL이 layer cluster block(파이프라인 레지스터 + Residue Merger Unit)으로 매핑되는 unrolled 아키텍처, VNU(β 압축 해제/감산)-Routing-CNU(min 비교)-AP-LLR update 순서로 처리.
7. 구현 디테일: AP-LLR 6-bit, check node 메시지 4-bit 양자화. β 메시지는 압축된 sign-magnitude(min1/min2/index/sign) 형태로 저장 후 twos-complement 변환.
8. 최적화 절차 없음(설계 파라미터는 layer grouping 3종: (1,1,1,1)/(2,1,1)/(2,2)을 수동 비교).
9. 결과: 4 iteration 기준 35~41 Gbps throughput, layer grouping이 커질수록 slice register/LUT 감소(69211→53844, 360163→328952)하나 주파수·throughput은 소폭 하락(41→35 Gbps).
10. 한계: FPGA 자원 75% 이상 사용해 routing congestion 발생, 주파수 30~35MHz로 낮음. 정정성능(BER/error-floor) 개선 없이 순수 HW 비용/throughput 트레이드오프만 다룸.
