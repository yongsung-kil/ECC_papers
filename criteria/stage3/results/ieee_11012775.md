### B. 알고리즘 요약

1. QC-MDPC McEliece PKC는 순환행렬 기반 parity-check matrix `H`로 암호화하며, 복호는 QC-MDPC 디코더(bit-flipping 계열)로 수행되고 실패확률(BLER)이 공격 취약점이 된다.
2. 기존 MLABF-1(bit-flipping 변형)은 NUPC(`α_j` = 미충족 parity check 수)와 회귀선 기울기 기반 보조지표 `β`로 매 스텝 1비트씩 플립하는 반복 디코더.
3. 본 논문은 Chase decoding 개념을 차용해 디코딩 전에 test pattern `T`(Hamming weight `q`)를 암호문에 미리 더해 여러 후보를 만드는 BCLD(Bi-evaluated Chase-like Decoding)를 제안.
4. `T`의 0이 아닌 위치는 NUPC 상위 `d`개 값 그룹에서 `β`가 큰 후보 집합 `I`에서 무작위 `q`개 선택 - `Algorithm 1 Calcβ`로 가상 플립 후 syndrome 재계산.
5. `c' = c + T`를 임의의 QC-MDPC 디코더 `dec(·)`(여기선 MLABF-1)로 복호, 실패 시 새 test pattern으로 재시도(최대 `l`회) - 핵심 흐름은 Algorithm 3.
6. 변형 BCLD2는 원본 암호문 복호 실패 시에만 BCLD를 수행(가중치 `w(c')` 증가 방지 트릭).
7. 부수 트릭: `β` 계산 시 `τ`(candidate 수)를 iteration마다 감소시켜 계산량 제한.
8. 학습/최적화 절차 없음 - 파라미터(`τ0, δ, σ, l, d, q`)는 사전 실험값 고정.
9. 결과: `(n,r,w)=(9602,4801,45)` QC-MDPC에서 BCLD+MLABF-1, BCLD2+MLABF-1이 MLABF-1 대비 BLER 대폭 감소(오차 수 `t` 작을수록 개선폭 확대), BCLD가 대체로 최저 BLER.
10. 한계: 순수 시뮬레이션(BLER 곡선)뿐, HW 미설계, 최대 `l`회 반복 재디코딩으로 latency 증가, ADMM 등 최신 QC-MDPC 디코더와 비교 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping 기반 QC-MDPC 디코더(MLABF-1) | 대응 없음 | Prime ECC는 min-sum soft decoder이며 bit-flipping hard decoder 계열 미보유 |
| Chase-like test pattern 재시도(BCLD) | `decoder.cpp` `LDPC_Decoder()` (조기종료 후 재시도 개념 유사) | 반복 재디코딩 아이디어는 partial-CRC 실패 시 재시도 구조와 유사하나 test pattern 생성 로직 자체는 부호(MDPC)/디코더(BF) 전용 |
| syndrome/NUPC 계산 | `decoder.cpp` `C2V_Cal()` | parity check 결과 집계라는 점에서만 개념적 유사 |
| QC-MDPC H-matrix 구조 | `ecc_top.cpp` `Load_PCM()` | 대응 없음 (MDPC는 moderate-density, Prime ECC는 QC-LDPC 고정) |

> 적용 가치: 낮음 - 암호용 QC-MDPC bit-flipping 디코더의 BLER 개선책으로, NAND binary QC-LDPC min-sum 디코더와 부호밀도·디코딩방식이 근본적으로 달라 직접 이식 대상이 아님.
