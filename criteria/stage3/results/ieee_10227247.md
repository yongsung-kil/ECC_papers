### B. 알고리즘 요약

1. 대상: (32,64) H-matrix, `n=64` VN, `m=32` CN, code rate `Rc=0.5`, BPSK+AWGN, 4-byte 정보길이 IoT 단블록.
2. 문제: 기존 decoder-in-the-loop GA 부호설계[1]는 girth-4 cycle과 BP 디코더 때문에 단블록 성능이 제한적.
3. 모델: GA로 후보 H-matrix 집단(`Npop,max=50`)을 진화(mutation/crossover), fitness는 decoder-in-the-loop BLER.
4. 핵심기법 1단(개선①): "Detection and Removal of Girth 4" 블록을 penalty로 추가 — girth-4(사각 4변 1) 검출 시 사각 성분 재배치로 제거해 열악 fitness 후보를 조기 제거.
5. 핵심 식: girth `g`에서 최소거리 하한 `dmin >= 1+wc+wc(wc-1)+...` (wc=열가중치) — girth 증가로 dmin·복호효율 향상 근거.
6. 핵심기법 2단(개선②): 디코더를 BP 대신 min-sum으로 교체 (`Eji=∏sign(Mji')·min|βji'|`, VN은 addition), SPA 대비 복잡도 절감·성능 유사.
7. 구현 디테일: min-sum 5단계(초기화 L=-rx, CN update, total LLR `Li^total=Li+ΣEji`, code recovery `c*H^T=0` 체크, VN update), `Nit,max=200`.
8. 학습/최적화: GA는 fitness function 평가 횟수로 성능 측정, 시간·비용 균형 위해 population size 50 선택.
9. 결과: Eb/No=3dB에서 Scheme3(BLER=0.0336)이 Scheme1(0.0378) 대비 11.1%, Scheme2 대비 4% 개선; 5dB에서 Scheme1 대비 최대 33.3% BLER gain.
10. 한계: HW 미설계, Matlab 시뮬만, 극소형 단블록, 향후 Rayleigh/Rician·burst error·IoT 적용은 미완.
