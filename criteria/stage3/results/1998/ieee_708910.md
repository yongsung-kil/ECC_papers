### B. 알고리즘 요약

1. Gallager LDPC에 착안한 rate `R=1/2` binary `(M,J,K)`-convolutional code: memory `M`, 주기적·희소 parity-check 행렬 `H`, column weight `W_H=J`, row weight `W_H=K`.
2. 풀려는 문제: turbo code에 근접하는 성능을 convolutional LDPC 구조로 달성.
3. 정규 구성 절차: 정보심볼 column은 `J=3`, parity-check 심볼 column은 `J=2`, 그리고 `K=5`로 구성.
4. 반복복호: log-likelihood ratio `l(n)`, `A(·)=tanh(l(n)/2)` 기반 공식으로 각 부호심볼을 갱신하며 parity-check 집합 `N(j)`, `J(n)`을 사용.
5. 식의 의미: `H`의 원소 `h_jn`으로 정의되는 parity 제약 `S(n)`을 만족하는 tuple `v`에 대한 통계로 반복 갱신.
6. 병렬 구현: `H`의 주기성 덕분에 `I`개 병렬 프로세서로 쉽게 구현 — 프로세서 `i`가 iteration `i` 통계를 계산, `I`번째 프로세서가 최종 판정.
7. 복잡도: 디코더 복잡도는 memory `M`에 무관, 병렬 프로세서 수 `I`에 선형.
8. 이론 결과: `M→∞`일 때 오류확률이 `J,K` 및 채널 SNR에 의존하는 하한으로 lower-bound; `M`을 키워도 이 하한을 넘을 수 없음.
9. 결과: AWGN 시뮬에서 `(M,3(2),5)` 부호가 rate 1/2 block LDPC 및 매우 긴 LDPC(N=65389, 19839)와 비교됨; 정확 공식 (1) 사용 시 근사 대비 0.2~0.3 dB 개선.
10. 한계: HW 미설계, 1페이지 초록, 상세는 라이선시엇 논문·투고본[2][3]에 위임.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-convolutional(SC-LDPC) 부호 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC block 고정, convolutional H 구조는 미대응(재검증 부담 큼) |
| tanh 기반 반복복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 근사 채택, full-tanh sum-product는 대응 없음 |
| I-병렬 프로세서 파이프라인 | 디코더 HW 모델(z=32 파이프라인) | 주기성 기반 병렬화 개념뿐, 우리 QC row-병렬과 구조 상이 |

적용 가치: **낮음** — SC-LDPC/convolutional 부호 원류 초록으로, QC-LDPC block·min-sum 기반 Prime ECC에 떼어 쓸 구체 부호·디코더 기법이 없다.
