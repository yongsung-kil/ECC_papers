### B. 알고리즘 요약

1. 대상 채널: 3bit/cell(TLC) MLC NAND, 8-level Gaussian로 모델링되나 실제는 program disturb·SBCL로 비대칭(split-Gaussian)화됨.
2. 문제: 표준 AWGN 가정 LDPC 디코더는 채널이 EOL(end-of-life)로 갈수록 비대칭·long tail이 심해지면 성능 저하.
3. 채널 모델: 레벨별 above/below edge variance가 다른 split-Gaussian, `σ1² > σ0²`인 variance mismatch(VM)로 정의.
4. 핵심 기법: LLR을 `LLR(r) = log(σ1/σ0) + (r²+a²)(σ0²-σ1²)/(2σ0²σ1²) + ra(σ0²+σ1²)/(σ0²σ1²)` 형태의 비선형 함수로 재계산(기존 선형 LLR 대체).
5. 의미: variance mismatch가 있으면 LLR이 read voltage `r`의 비선형 함수가 되어야 오분류를 방지(고신뢰 비트 misleading 방지).
6. 확장 기법(program disturb 전용): 7-0-7 패턴이 program disturb로 7-1-7이 될 가능성을 고려해, level-1 read 시 이웃이 둘 다 level-7이면 해당 bit의 LLR magnitude를 clip.
7. 구현 디테일: 7-bit soft quantization(128 LLR values), erasure flag 방식도 시도했으나 clipping이 정보 손실이 적어 더 우수.
8. Misplacement 대응: 편측(single-sided) misplacement 확률 `α`를 이용해 `|LLR(r)| ≈ min(2ar/σ², |log(α)-log(1-α)|)` 형태로 clip.
9. 결과: VM=2.5인 long tail 상황에서 clipping 미적용시 target UBER(1e-12) 달성 rber가 저하되나, clipping 적용 시 1.875e-2 rber에서 target UBER 회복(무손상 기준 1.9e-2에 근접). Misplacement 1% 상황도 clipping으로 1.8e-2 rber에서 target 달성.
10. 한계: HW 구현/게이트카운트 없음, 순수 알고리즘·시뮬레이션 수준이며, program disturb 대응법은 이웃 셀 추가 read가 필요해 read latency 오버헤드 발생.
