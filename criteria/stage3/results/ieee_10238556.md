### B. 알고리즘 요약

1. 시스템: BSC 채널, 하드결정(hard-decision) bit-flipping 디코더. 대상 부호는 irregular LDPC (4986.93xb.329, PEGirReg504x1008; N=1008 등).
2. 문제: 기존 BF/PGDBF는 regular 코드용이라 irregular 코드의 서로 다른 column degree `dv`가 inversion value 범위를 불균형하게 만들어 저degree 열이 거의 flip되지 못함.
3. 모델: inversion value `Ev(i) = (vi⊕yi) + Sv(i)`, `Sv(i)`는 연결된 unsatisfied parity-check 수의 합.
4. 핵심 기법1(열가중치): degree별 가중치 `w`를 두 번째 항에 곱해 `w1·dv1 = w2·dv2`가 되도록 범위 정규화 (예: dv1=3, dv2=9 → w1=3, w2=1).
5. 식 의미: `w1×dv1 = w2×dv2`로 서로 다른 degree 열이 flip될 공정한 기회를 갖게 함.
6. 확장(다중 degree): degree가 3개 이상이면 LCM/degree 가중치가 너무 커져, degree를 두 그룹으로 나눠 그룹당 동일 가중치(예: w=3,1) 적용.
7. 구현 트릭(수정 확률 flip): 열가중치 전에 각 그룹의 `Smax`를 먼저 찾아 bit-width 증가 회피. `Emax`와 같으면 확률 `p1`, `Emax+1`이면 `p2`로 flip.
8. 최적화: `(p1,p2)`를 0.1~1.0 그리드로 수치탐색. p2=1.00이 최적이라 RG(난수생성) 모듈 제거 가능.
9. 결과: TRGDBF/ISBF가 error floor를 보이는 구간에서 CW-PGDBF는 error floor 없이 더 나은 FER. 짧은 길이·높은 column weight 코드에서 하드웨어 오버헤드까지 감소(PEGirReg504x1008).
10. 한계: BSC 하드결정 전용, Tmax=300으로 반복이 많음, throughput/gatecount 절대값은 표에만 있어 본문 서술로 확정 불가.
