### B. 알고리즘 요약

1. 길이 `n=1000~2000`, rate `R=1/2, 3/4`의 irregular LDPC 코드(예: `λ(x)=0.283x+0.281x²+0.436x⁸`)를 대상으로 PEG(progressive edge-growth) 알고리즘으로 Tanner graph를 구성한다.
2. 문제: irregular LDPC는 waterfall 성능은 좋으나 작은 stopping set(BEC)·낮은 minimum distance/trapping set(AWGN)에 의한 error floor가 발생, 기존 PEG/ACE는 이를 충분히 억제하지 못함.
3. stopping set은 `S⊆V`에서 모든 이웃이 S에 2번 이상 연결된 집합으로 정의(Definition 3), erasure 복호 실패의 직접 원인.
4. 핵심 기법: PEG의 variable-check node 연결 시 복수 후보가 있을 때, degree-3 variable node 조합이 만드는 stopping set 크기를 check node distance `d_c(i,j)=|i-j|` 합으로 미리 계산해 목표 최소 stopping set 크기 `ŝ`보다 작아지는 연결을 원천 배제(modified PEG).
5. 핵심 식: 1~4개 degree-3 노드 조합별로 거리합 임계값(`ŝ-1`, `ŝ-2`, `ŝ-4`, `ŝ-5`)을 두어 단계적으로 금지 조건을 적용, 코드길이 n과 무관하게 `ŝ` 크기에만 의존하는 복잡도로 tree-spanning 탐지.
6. 4개 이상 degree-3 노드가 얽힌 stopping set은 cycle distance `c`를 최대화하는 check node를 선택해 확률적으로 회피(완전 배제는 미보장).
7. 부수 트릭: degree-2 variable node는 zigzag(staircase) 방식으로 연결해 degree-2 노드 간 cycle을 원천 차단.
8. 별도 학습 없음(결정론적 그래프 성장 알고리즘, 무작위 seed 50개로 앙상블 구성).
9. 결과: n=1000, R=1/2에서 평균 minimum distance가 12.0→16.7로 증가, size<18 stopping set 수가 1042→96개로 감소; BEC/AWGN FER 시뮬에서 modified PEG 최악 코드가 원본 PEG 최선 코드보다도 우수.
10. 한계: 오프라인 부호설계(H-matrix 생성) 알고리즘일 뿐 디코더·HW 변경 없음, AWGN 채널은 최대 500 iteration shuffled BP 가정으로 NAND HW 디코더(min-sum, 제한 iteration)와 직접 비교 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG 기반 H-matrix 구성(stopping-set 회피) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 새로 생성하는 부호설계이나 Prime ECC는 QC circulant(base+shift) 고정구조라 PEG의 자유도(임의 edge 연결)와 직접 호환 안 됨 |
| minimum distance 증가로 error-floor 개선 | 대응 없음 | 개선영역은 decoder.cpp의 min-sum 알고리즘이 아닌 부호 자체 구조에 기인, 코드 교체 없이는 이식 불가 |
| stopping set/trapping set 크기 분석·탐지 알고리즘 | 대응 없음 | Prime ECC coverage에는 별도 trapping-set 분석 도구 부재 |

> 적용 가치: 중간 — error-floor 개선 아이디어(stopping set 회피 기반 부호설계)는 개념적으로 유효하나, Prime ECC의 QC-LDPC 고정 구조(§4 "부호설계 이식 난이도 높음")로 인해 직접 H-matrix 대체 없이는 이식 불가.
