### B. 알고리즘 요약

1. 시스템: TLC NAND flash(3bit/cell), 코드워드 length 1KB, code rate 0.9의 LDPC ECC, 페이지 크기 16KB, HD/2bit-SD 센싱과 read-retry를 포함하는 error recovery flow(ERF).
2. 문제: 기존 고정순서 ERF(Default HD → Default SD → Optimal HD → Optimal SD)는 메모리 열화 상태와 무관하게 순차 재시도해 고RBER에서 불필요한 지연이 큼.
3. 가정: 채널모델은 프로그램 분포(Gaussian) `*` RTN(Laplace) `*` retention noise(Gaussian)의 convolution(식 1~6), PE cycle과 retention time에 따라 Vth 분포가 변화.
4. 핵심 기법: 센싱 출력(7개 read voltage에서의 ON-cell 비율), PE cycle 수, (선택적) HD 디코딩 출력(FER, 평균 iteration)을 입력특징으로 하는 4-class 지도학습 분류기(ANN/SVM/DT)로 최소 latency를 달성하는 시작 모드를 예측.
5. 핵심 식: 각 클래스(1~4)의 latency는 `t_HDsen+t_HDdec`, `t_HDsen+t_SDsen+t_SDdec`, `t_RR+t_HDsen+t_HDdec`, `t_RR+t_HDsen+t_SDsen+t_SDdec`로 정의(Table I) - 레이블은 성공하는 모드 중 최소 latency 모드.
6. 확장: 특징 선택에 따른 trade-off 분석 - 디코딩 출력을 특징에 추가하면 정확도가 약 10%p 상승하지만 매 시도마다 불필요한 HD 디코딩이 추가되어 latency overhead 발생.
7. 구현 디테일: 센싱 출력만 특징으로 쓰면 원래 read 과정에서 어차피 수행하는 HD 센싱을 재사용하므로 추가 latency 없음(`t_infer≈0` 가정).
8. 학습 절차: ANN(은닉층 없음, softmax+cross-entropy), 선형 SVM(binary complete coding 다중클래스), CART 기반 Decision Tree(Gini index, 최대 40 split) 3종 분류기 비교.
9. 결과: 분류 정확도 79~91%(특징 조합별), Default ERF 대비 평균 latency 최대 85% 개선, 정확도 낮은 구간에서도 41% 이상 개선.
10. 한계: LDPC 디코더/부호 자체 개선 없음(기존 HD/SD 디코더를 블랙박스로 사용), MATLAB 시뮬레이션뿐 HW 미설계, ML 모델의 온칩 구현 비용/latency(`t_infer`)는 사실상 0으로 가정만 하고 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ERF(HD/SD 센싱, read-retry 순서 결정) | 대응 없음 | Prime ECC는 디코더 엔진 시뮬레이터로 컨트롤러 레벨 read-flow 정책은 범위 밖 |
| HD/SD 디코딩 latency, iteration 수 | `decoder.cpp` `LDPC_Decoder()` (HD/2SD/3SD 지원) | 논문의 특징(FER, 평균 iteration)은 Prime ECC 디코더 통계와 유사한 개념이나 논문 자체는 디코더 알고리즘을 개선하지 않음 |
| soft-read 양자화(2-bit SD sensing) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 논문의 2bit SD 센싱 개념은 채널/LLR threshold 모듈과 개념적으로만 연관, 논문이 이를 개선 대상으로 삼지 않음 |
| ML 분류기(ANN/SVM/DT) | 대응 없음 | Prime ECC에 ML 기반 정책 결정 모듈 없음 |
```

적용 가치: 낮음 - LDPC 부호/디코더/HW를 다루지 않는 시스템(컨트롤러) 레벨 read-flow 스케줄링 논문으로 Prime ECC 모듈 지도상 대응되는 이식 대상이 없음.
