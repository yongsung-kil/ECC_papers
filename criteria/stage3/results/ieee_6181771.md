### B. 알고리즘 요약

1. 대상: protograph로 만든 (3,6)-regular SC-LDPC 체인(길이 `L`), BEC/AWGN 채널, iterative(BP) 복호.
2. 문제: 단일 SC-LDPC 체인의 iterative threshold를 더 높이고 특정 BER 달성에 필요한 복호 복잡도를 줄이려 함.
3. 모델: 체인을 protograph의 edge로 보고 여러 체인을 상호연결. base matrix `B`는 (3,6) 체인의 incidence matrix.
4. 핵심 기법: 체인 끝단 CN의 여유 edge(2/4개)를 다른 체인의 VN에 연결 → loop `L(3,6,L)`, triangle `T(3,6,L)`, mixed-loop `L1/L2(3,6,4,8,L)` 앙상블 구성.
5. 식 의미: 연결점 위치를 `L/3`(내부 loop변 `2L/3`)로 두면 초기 iteration에서 수렴부스트가 필요한 끝단에 도움이 도달, threshold 최대화.
6. 확장: mixed-loop는 (4,8)체인이 더 강한 수렴부스트를 요구하므로 12개 연결 edge를 분산 배치(`L2`)해야 (4,8)측도 개선.
7. 구현 디테일: 없음(HW 무관). 연결 edge 배치·연결점 간격이 성능 민감 파라미터.
8. 학습/최적화: density evolution(식 (1)~(3))으로 노드별 error prob 추적, 개선제약 `θ` 있는 update schedule로 복잡도 측정.
9. 결과: `L(3,6,12)` BEC threshold 0.5237 vs 단일 `C(3,6,12)` 0.495; mixed `L2` 0.5105; BER 1e-5 달성 update 수 감소. min distance 선형증가(δmin 0.0071~0.0109).
10. 한계: HW 미설계, 이론(DE/threshold)만, 저rate 비QC SC-LDPC, 유한길이/NAND 성능 없음.
