### B. 알고리즘 요약

1. 대상은 rate-`b/c` LDPC 콘볼루션 부호(LDPC-CC), IEEE 802.16m의 rate-1/2 `(491,3,6)` 부호를 mother code로 사용.
2. LDPC-CC는 슬라이딩윈도우 기반 message passing으로 복호되며, 여러 base processor를 캐스케이드해 pipeline(각 processor=1 iteration)을 구성.
3. 문제: LDPC-BC용 IDS는 전체 그래프 비교가 필요한데, 슬라이딩윈도우는 윈도우 밖 메시지를 갱신 불가라 기존 IDS를 그대로 적용 못 함.
4. 핵심기법1: 각 processor 내에서 모든 CN에 연결된 variable node 집합을 선택, 이들의 C2V residual `r = |m_pre - m|` (식 3)을 metric으로 최대 residual 메시지만 갱신.
5. residual 의미: 갱신 시 변화가 큰(=수렴에 중요한) 메시지를 우선 갱신해 수렴 가속, silent variable node 회피. residual pre-computation은 복잡도 위해 min-sum(MSA)로 근사.
6. 핵심기법2: 슬라이딩윈도우가 이동하면 선택되는 variable node 집합이 바뀌어 모든 노드가 균등하게 intrinsic 정보를 전파.
7. 트릭: processor 증설 없이 갱신량 확보 위해 한 번 비교 후 한 variable node에 대해 C2V 메시지를 1개가 아닌 2개까지 갱신(2-update).
8. RC 펑처링: rate-5/6부터 시작해 상위rate 패턴의 "0"을 "1"로 되돌려(parity 추가) 하위rate 패턴 생성 → rate-compatible 보장(Table II).
9. 결과: 동일 갱신수 조건에서 IDS가 LBP 대비 전 rate에서 0.5dB↑ gain, rate-5/6는 BER 1e-6에서 약 2dB gain. RC 패턴+2-update IDS는 갱신 메시지 2/3로 복잡도 절감.
10. 한계: HW 미설계(시뮬만), 동일 갱신수 확보 시 processor 수 증가로 latency·복잡도 증가, convolutional 부호·무선/HARQ 특화.
