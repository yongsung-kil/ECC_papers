### B. 알고리즘 요약

1. Hard-decision LDPC 복호(soft info 없이 binary 입력)를 대상, `H=(h1,...,hn)` m×n parity check matrix, 각 열을 "atom"으로 정의, syndrome `s=Hcˆ=He`.
2. 문제 설정: 에러패턴 `e`를 `argmin‖e‖0 s.t. s=He`로 정의(가장 sparse한 해 탐색), GF(2) 특성상 `‖e‖0=‖e‖1`로 compressed sensing의 matching pursuit(MP)과 연결.
3. 핵심 가정: LDPC parity check matrix가 RIP(restricted isometry property)를 만족해 MP 알고리즘으로 sparse error pattern 복원 가능.
4. BMP(Binary MP) 알고리즘: 매 iteration `argmax⟨r_{i-1}, h_ω⟩`로 residual과 최대 내적을 갖는 atom 선택, 동일 atom이 연속 선택되면 "mismatching"으로 간주해 재선택 금지 리스트 `Ω`에 등록.
5. `ri = r_{i-1} ⊕ h_{λi}` (GF(2) XOR)로 residual 갱신, `‖ri‖1=0` 또는 최대반복 `κ`(≈`√(nPe)`, Pe는 무부호 채널 에러확률) 도달 시 종료.
6. IBMP(Improved BMP) 확장: atom별 "matching reliability" `L`(과거 선택 빈도 카운트)을 도입해 최대내적+최대신뢰도 atom을 동시에 만족하는 atom을 매칭으로 선택, mismatching 시 신뢰도를 `d=dv`(열 degree)만큼 감산.
7. 부수 트릭: 첫 iteration엔 reliability가 0이라 정보가 없어 오매칭이 잦은 문제를 다중 복호모듈 병렬 협업(collaborative decoding)으로 완화, 최종 결과를 결합.
8. 학습/최적화 절차 없음(고정 파라미터 `d=dv`는 시뮬레이션으로 결정된 경험값).
9. 결과: code length 128, rate 1/2, PEG-LDPC 기준 BMP/IBMP가 BF·SMP를 능가하고 hybrid soft 알고리즘인 WBF보다도 우수(단 log-BP엔 못 미침); iteration당 parity check 불필요해 연산량 절감, WBF 대비 복잡도 낮음(정수/논리 연산 위주).
10. 한계: 소규모 code length(≤512) 시뮬만 수행, 조기종료·HW 설계·NAND 특유의 고rate(~0.9)·대용량(수천~수만 bit) 코드에서의 검증 없음.

| Matching reliability(atom 선택 빈도 카운트) | `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 상태 테이블 참조 구조와 개념적으로 유사하나 실존 대응 함수는 아님 |
| 다중 복호모듈 협업(collaborative decoding) | 대응 없음 | Prime ECC엔 병렬 다중 인스턴스 협업 복호 구조가 없음(Dual-Update와는 다른 메커니즘) |
| GF(2) atom(=parity check 열) 선택 기반 복호 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산과는 수학적으로 다른 접근(내적 최대화 vs magnitude 근사)이라 대체 검증 필요 |

> 적용 가치: 중간 — hard-1bit 경로에서 BF 대비 BER 개선 여지가 있는 신규 알고리즘이나, 소규모 코드 시뮬 수준 검증과 HW 미설계로 즉시 이식보다는 개념 검토용.
