### B. 알고리즘 요약

1. 대상은 rate `k/n`의 time-invariant LDPC convolutional code 앙상블 `(n,k,W)`; 예로 `n=6,8,10`, `k=4`.
2. 문제: 좋은 SC-LDPC를 만드는 데 얼마나 적은 randomness가 필요한가 — 파라미터 수를 극단적으로 줄이려 함.
3. 채널 모델: BEC(parameter ε)만 사용, all-zero codeword 가정, peeling decoder로 복호.
4. 핵심 기법: `n-k`개 shift-invariant parity-check 각각이 `n`개 stream에 tap 1개씩, tap 위치는 `{0,...,W-1}`에서 균등 랜덤 monomial.
5. 각 코드는 `n(n-k)log2(W)` bit로 기술 — 극소 파라미터. `W`는 constraint length(=effective blocklength)를 결정.
6. 확장/성질: staircase 순서를 만족하면 인코딩이 순차적으로 간단히 수행됨.
7. 구현 디테일: polynomial 대신 monomial만 허용해 stopping set 분석을 용이하게 함(성능은 다중 tap이 더 나을 수 있음).
8. 분석: 최소 stopping set 크기가 `(3c-1)/(2√c)` (c=n-k) 하한, 즉 shift-invariant check 수에 지수적으로 증가.
9. 결과: 시뮬에서 `W`가 커질수록 waterfall이 가팔라지고 Shannon 근접(threshold saturation); error floor slope로 stopping set 크기(W=40→d≈12, W=1000→d≈15) 추정, 하한과 정합.
10. 한계: BEC·peeling decoder뿐, asymptotic threshold는 해석적으로 못 구해 시뮬 의존, HW 미설계.
