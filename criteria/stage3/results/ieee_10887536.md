> 총평: LDPC가 아닌 짧은 이진 블록코드(RM/BCH/eBCH/Polar)에 대해 4-cycle-free·고cycle연결성 sparse PCM을 체계적으로 추출하여 NMS-BP 복호 성능을 ML 경계 근처까지 끌어올리는 부호설계 기법으로, NAND QC-LDPC와는 부호족이 다르지만 trapping-set/girth 최적화 아이디어는 개념적으로 참고 가능.

### B. 알고리즘 요약

1. 시스템: AWGN 채널 위 이진 짧은 블록코드(RM, BCH, eBCH, 5G NR Polar) `C(N,K)`를 대상으로, NMS(normalized min-sum) 기반 BP(sum-product 근사) 복호를 수행. LLR `λ=2y/σ^2` 입력.
2. 풀려는 문제: BP는 긴 LDPC엔 우수하지만 짧은 코드의 표준 PCM은 밀도 높고 4-cycle이 많아 성능이 급락 — 기존 4-cycle 제거법은 AWGN에서 보조 VN(AVN)의 무정보(zero-LLR) 메시지 때문에 오히려 성능 저하 유발.
3. 핵심 가정: 4-cycle과 결합된 작은 trapping set(특히 elementary trapping set, ETS)이 error floor의 근원이며, ACE(approximate cycle EMD, `η(C)=Σ(dv-2)`)로 cycle connectivity를 정량화할 수 있다는 모델.
4. 핵심 기법 1단 — 최적화된 4-cycle 제거: 기존 [56] 방식(그리디하게 `s`행·`t`열의 4-cycle군을 한 AVN으로 제거, `δ=C(|Sv|,2)*C(|Sc|,2)` 최대화)에 degree-1/degree-2 CN 생성을 방지하는 후처리(Alg.1의 Line 17-23)를 추가해 AVN 수를 최소화.
5. 핵심 식: `Q4(H)=ΣΣ C((HH^T)_i1,i2, 2)` (4-cycle 개수), 정리(Theorem 1)로 4-cycle-free 행렬 `H̃`의 NMS-BP FLOP/BOP 복잡도가 원행렬 `H`보다 항상 작음을 증명(`s(7t-7)-7t≥0` 등).
6. 핵심 기법 2단 — Progressive Row Growth(PRG): LDPC의 ACE-constrained PEG를 본떠, 이중부호 `C^⊥`의 저weight parity row로 구성된 탐색공간 `Hs`에서 rank·AVN 최소화·column weight 분산 최대화 기준으로 중복 행을 점진 선택해 girth-6·`η_min` 최대 sparse PCM `Hb`를 구성.
7. 부수 트릭: 빠른 4-cycle 판별(Alg.4)로 후보 행마다 전체 4-cycle 제거를 반복하지 않고 degree-2 CN 병합으로 조기 스킵, damping factor `γ=0.7` NMS 사용.
8. 최적화 절차: 탐색공간 생성(Alg.2, Gaussian elimination 기반 저weight parity row 생성) → PRG로 순차 행 선택(Alg.3) → 최종 `η_min` 최대인 `Hb` 채택. 별도 학습(NN 등) 없음.
9. 결과: RM(32,16)에서 240-edge PCM-II로 기존 최고 성능(1560-edge NBP)과 동등 성능 달성, eBCH(64,24)에서 SOA 대비 0.4dB gain, BCH(63,45)에서 ML과 0.35dB 이내로 근접; 3회 MBBP 시도로 ML급 도달.
10. 한계: 순수 시뮬레이션(HW 미설계), 코드 길이 증가 시(RM(128,64)) 탐색공간이 94488×128까지 커져 grid heuristic의 최적성 저하; 대상이 QC-LDPC가 아닌 RM/BCH/Polar 등 non-LDPC 짧은 코드이며 PCM이 원래보다 행/열이 늘어나는(redundant AVN) 구조.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS(normalized min-sum) BP, damping factor | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 min-sum 계열 알고리즘이나 Prime ECC는 이미 min-sum 보유(중복 기여, 신규성 낮음) |
| 4-cycle 제거/AVN 삽입으로 girth 확대 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 이미 고정 QC-LDPC(base+circulant) 구조라 짧은 코드용 PCM 재추출 기법 자체는 대응 없음, 다만 trapping-set 회피 설계원리는 참고 가능 |
| Trapping set / ACE(cycle connectivity) 최적화 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 시 error-floor 완화 목적과 개념적으로 연결되나 논문의 degree-1/2 CN 방지 알고리즘 자체는 QC 구조에 그대로 이식 불가 |
| PRG(progressive row growth) 탐색 | 대응 없음 | LDPC PEG류 구성법과 유사하나 짧은 non-LDPC 코드의 중복 parity row 탐색이라 QC-LDPC 설계 파이프라인과 별개 |
```

적용 가치: **중간** — min-sum BP 알고리즘 자체는 Prime ECC가 이미 보유해 직접 이식 기여는 낮으나, 4-cycle/작은 trapping set을 girth·cycle-connectivity(ACE) 관점에서 정량 분석하고 제거하는 설계 원리는 향후 H-matrix 개선(error-floor 대책) 검토 시 참고 가치가 있음.
