### B. 알고리즘 요약

1. 대상: binary LDPC, fixed-point scaled(normalized) Min-Sum 디코더, 채널 LLR 입력, VN-degree `dv=3,4,5,6` 예제(rate 0.5 Margulis `Nv=2640`, AB-LDPC `p=31`, RS-based, 802.3an `(2048,1723)`).
2. 문제: 실무 quantized MS 디코더의 error-floor는 message 포화레벨(`Umax`)·양자화 선택에 강하게 의존하는데, 어떤 absorbing set(AS)이 비활성화되는지 판정할 정량 기준 부재.
3. 모델: AS 내부 extrinsic message 전개를 quasi-linear state-space `x^(k+1)=sat(α(A(x−1)+d−1+ℓ))`로 모델링, 외부 message는 +1로 포화 가정.
4. 핵심기법: scaled-MS의 AS threshold `τ = max_ℓ min(ℓ)` (misleading equilibrium 존재/부재를 가르는 max-min 최적화)로 정의, α 인자를 (2)식 CN 또는 (3)식 VN에 적용 가능(결과 동일).
5. 의미: 모든 VN의 channel message가 `τ`를 초과(`ℓ_i>τ`)하면 유일 equilibrium이 `x=+1`(정답)로 수렴 → 성공 복호 충분조건. `τ<0`이면 `Umax` 조절로 항상 만족시켜 AS를 "비활성화".
6. 확장: minimum equilibrium이 존재하고 min channel message로 순서화됨(Property 3,4)을 증명 → threshold를 minimum equilibrium 추적으로 빠르게 계산.
7. HW 친화 디테일: message는 `Λmax`/`Umax`로 clip·양자화, 정규화(`u=U/Umax`)로 saturation을 `sat(x)=sign·min(|x|,1)`로 단순화.
8. 최적화 절차: Algorithm 1 — `ℓ=−1/α`에서 시작해 Δ씩 증가시키며 equilibrium이 `x=+1`될 때까지 반복(bisection 가속), threshold lower bound `τ_min(α,d)=1/α+1−d`.
9. 결과: Margulis 지배 (12,4)·(14,4) AS의 threshold가 α=1에서 0 → α=3/4에서 음수화(τ=−3/50)되어 qe=5bit scaled-MS에서 error-floor 대폭 감소(Monte-Carlo·Importance Sampling 확인); AB/RS/802.3an의 모든 지배 AS도 낮은 α로 비활성화.
10. 한계: HW 미설계(시뮬만), elementary AS·regular VN-degree 가정, 통신용 규칙적 부호 대상이라 특정 부호마다 threshold 재계산 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| scaled/normalized Min-Sum CN 연산 (α 인자) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC Min-Sum(min1/min2+sign)에 scaling factor 도입 지점, error-floor 개선용 |
| extrinsic message 양자화·포화(Umax) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | AS threshold가 `Umax` 포화레벨과 직결, 양자화 테이블 튜닝으로 AS 비활성화 |
| AS threshold 기반 error-floor 분석 | `corner.cpp` (수렴실패 추적) | 수렴실패 케이스를 AS로 특성화해 threshold로 정량 진단 가능 |
| iteration별 LLR threshold 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR과 scaling/saturation 상호작용 검토 대상 |
| AS가 속한 H-matrix 구조 | `ecc_top.cpp` `Load_PCM()` | 우리 QC H-matrix의 지배 AS 스펙트럼 추출이 threshold 계산 전제 |

적용 가치: **중간** — Prime ECC가 이미 쓰는 min-sum + magnitude 양자화 위에 scaling factor를 얹고 우리 부호의 지배 AS threshold를 이 빠른 알고리즘으로 계산하면 HW 변경 없이 포화레벨/α 튜닝만으로 error-floor를 낮출 수 있으나, NAND 고rate QC-LDPC로의 threshold 재계산과 우리 부호 AS 추출이 선행되어야 한다.
