> 총평: Stochastic computing 기반 완전병렬 LDPC 디코더로 55nm ASIC 실측(57.6Gbps, 4.86pJ/bit) 성능은 우수하나, min-sum/LLR 기반 Prime ECC와는 완전히 다른 연산 패러다임(bitstream probability propagation)이라 알고리즘·모듈 이식성은 낮음. 다만 shared-resource(IM/RNG 공유)로 VN당 하드웨어 비용을 줄이는 설계 철학은 참고 가치가 있음.

### B. 알고리즘 요약

1. 5G NR 표준의 (2160,1760) QC-LDPC(길이 2160, rate≈0.815)를 대상으로 stochastic computing(SC) 기반 완전병렬 디코더를 ASIC으로 구현.
2. 기존 방식은 처리량을 높이려면 파이프라인 단계 추가(스톨 발생) 또는 병렬도 증가(면적·전력 급증)가 필요해, 특히 stochastic LDPC의 VN 업데이트 함수 수렴(latch 문제)이 느리고 자원 소모가 컸다는 문제의식.
3. 채널 정보는 LLR을 확률로 변환한 Bernoulli 비트스트림으로 표현하며(`I_n = e^{L_n}/(1+e^{L_n})`), stochastic sum-of-product algorithm(SPA)으로 C2V/V2C를 반복.
4. 핵심 기법: VN 업데이트 함수를 "공유 확률(shared probability)"과 "갱신 확률(updating probability)" 두 부분으로 인수분해하고, 이를 2단계(2-bit IM(Internal Memory) + 3-bit SCNT(stochastic counter))로 나눠 `m-1`개 노드 간 IM을 공유하는 TSM(Two-Stage shared Memory) 구조를 제안.
5. 핵심 식 `Q^i_n = I_n · [Σ_{m'≠i} P_{m'→n}] / [...]` (식 6~8)에서 `P_{m'→i}`(shared probability)를 IM으로 근사해 하드웨어를 공유 — VN당 자원 비용을 크게 낮추는 근거.
6. 확장으로 전체 2160채널에 대해 RNG(LFSR) 하나만 공유하는 shared bitstream generator를 사용, 기존 2160개 RNG 대비 2159개 절감.
7. 구현 디테일: fixed-point min-sum 대비 word length 6bit 비교 실험, bit-flipping 후처리 모듈과 codeword selection(체크섬 검증) 병행, 최대 iteration 100회.
8. 별도 학습/최적화 절차 없음 — 수식 인수분해 기반 구조 설계.
9. 결과: 55nm ASIC 실측 throughput 57.6Gbps, 면적효율 33.68Gb/s/mm², 전력효율 4.86pJ/bit, 코어면적 1.71mm², 400MHz 동작. BER 10⁻⁶ 도달 SNR 4.63dB(floating SPA 4.6dB, fixed-point min-sum 4.76dB보다 우수).
10. 한계: stochastic computing 특유의 latch/수렴 문제에 대한 국지적 개선이며 결정론적 LLR 기반 min-sum 디코더와는 연산 모델이 근본적으로 달라 타 아키텍처로의 직접 이식은 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic VN 업데이트(TSM: IM+SCNT) | 대응 없음 | Prime ECC는 결정론적 min-sum LLR 방식(`VN_Cal_HD()` 등)이며 확률 비트스트림 연산 자체가 없음 |
| Stochastic C2V/V2C 반복 (sum-of-product) | decoder.cpp `LDPC_Decoder()`, `C2V_Cal()` | 반복 구조(iteration loop) 개념은 대응되나 내부 연산(min-sum vs stochastic SPA)이 전혀 달라 코드 재사용 불가 |
| 전 채널 RNG 공유(2160→1) | 대응 없음 | Prime ECC는 난수 발생기를 검증/코너 케이스 추적용으로만 사용(`random.cpp` XOR25) — 디코딩 연산 자체에 RNG 불필요 |
| bit-flipping 후처리 + codeword checksum 검증 | partial_CRC.cpp `partial_crc_HW_T4()`, full_CRC.cpp | 조기종료용 체크섬 검증 개념은 Prime ECC의 CRC 조기종료와 유사하나 구현 메커니즘(SC vs 결정론)은 다름 |
| (2160,1760) QC-LDPC 5G NR H-matrix | ecc_top.cpp `Load_PCM()` | 5G NR 부호이며 NAND 전용 부호 구조와 무관 |

적용 가치: **낮음** — stochastic computing 패러다임 자체가 Prime ECC의 결정론적 min-sum 아키텍처와 근본적으로 이질적이라 직접 이식할 요소가 거의 없음. VN당 자원 공유를 통한 면적 절감이라는 설계 철학만 개념적으로 참고 가능.
