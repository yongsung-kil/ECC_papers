### B. 알고리즘 요약

1. 시스템: 5G NR C-RAN(BBU/FPGA aiding), BI-AWGN+BPSK, BG1 QC-LDPC, `CB size=26112`, `Z=384`, code rate `0.3~0.8`, 목표 throughput 2Gb/s.
2. 문제: MS는 성능은 좋으나 floating-point·최소값 탐색으로 latency·자원 소모 큼 — 5G latency-sensitive 채널에 부적합.
3. 모델: hard-decision은 inversion function `Δk(x)=α·φ(xn,yn)+Σωi·Πxj`(식 7)로 신뢰도 낮은 비트를 flip; regular PCM에서만 threshold 유효.
4. 핵심기법: IWHD — MS의 horizontal(check node sign XOR)/vertical(variable node update) 절차를 따르되, check node sign `C^horizon=XOR{sign(V^vertical)}`(식 8), VN update `V=1·sign(y)+ρ−β`(식 9)로 계산.
5. 식 의미: `β=sign(ρ)·Σ_{i∈1..4}Π xj`는 앞 4행(정보비트) syndrome을 reweight 항으로 사용 — IHD의 check node belief γ(평균)를 syndrome으로 대체.
6. 개선점: 저자들이 error-bit tracking으로 γ가 수렴을 늦추는 오류를 유발함을 발견, syndrome이 reweight와 early-stop 양쪽에 더 유효하다고 판단.
7. 구현디테일1: part-matrix-decoding — code rate에 따라 실제 데이터 길이만 디코딩(padding bit는 추가 에러 유발하므로 배제).
8. 구현디테일2: early stop — 앞 4행 syndrome=0이면 조기 출력.
9. 결과: rate 0.3에서 IWHD가 IHD보다 1.6dB 우수(단 MS보다 4.5dB 열화), rate 0.8에선 IWHD-IHD 0.5dB·MS-IWHD 2dB. X86 test bench cycle: MS 530k, IHD 220k, IWHD 190k(수렴 빠름). core usage MS 82% vs IHD/IWHD 53%(iter=4).
10. 한계: FPGA 미구현(future work)·hard-decision이라 정정능력 자체는 MS 대비 열위·irregular PCM에서 threshold 불안정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN sign XOR / VN update (BF 절차) | [decoder.cpp](../Prime_ECC_3.1_Claude) `C2V_Cal_New_Sgn()`, `VN_Cal_HD()` | HD 경로 존재하나 알고리즘이 min-sum이라 가중 BF는 구조 상이 |
| syndrome 기반 early stop | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 루프 (Prime은 partial/full CRC 조기종료 보유) | 개념 유사하나 Prime은 이미 CRC 조기종료 보유 |
| inversion function 가중치 α,ωi | 대응 없음 | Prime ECC은 min-sum LLR 기반, bit-flipping inversion function 미보유 |
| part-matrix-decoding(가변 길이) | 대응 없음 | Prime은 고정 QC-LDPC(z=32) 구조 |

적용 가치: **낮음** — 가중 bit-flipping은 Prime ECC의 min-sum 기반 soft/HD 파이프라인과 알고리즘 계열이 다르고, MS 대비 성능 열화가 커서 고rate NAND QC-LDPC의 정정능력 요구를 만족하기 어렵다.
