### B. 알고리즘 요약 (10줄 내외)

1. 5G NR eMBB 표준 QC-LDPC(BG1, `46×68` sub-matrix, `z=384`, 부호길이 `n=26112`, 정보길이 `k=8448`, code rate `1/3~8/9`)를 BI-AWGN+BPSK 채널에서 복호하는 FPGA 아키텍처를 제안.
2. 문제의식: 5G NR eMBB 규격에 맞는 LDPC 디코더 HW 구현 사례가 희소하여, 대형 프레임(8448비트) 처리에 적합한 면적효율적 아키텍처가 필요.
3. 채널모델은 표준 BI-AWGN, LLR 입력은 6bit(시뮬)/8bit(구현) 고정소수점 양자화.
4. 핵심 기법: layered offset min-sum 디코딩(`Algorithm 1`) - `α_{m,n}=min|x_{m,n}| - offset`(offset=2), `β_{m,n}=max(α_{m,n},0)`, 부호 XOR로 `sign(x_{m,n})`.
5. 하드웨어 구조상 CN/VN 결합처리유닛(CVCPU)이 층(layer) 단위로 68개 LLR을 읽어 subtraction→min-sum→addition을 수행, 초기메모리뱅크(IMB, 68×384)와 중간메모리뱅크(IntMB, 46×384)로 반복 갱신.
6. 부분병렬화: 매 클록마다 68 LLR을 처리(19개 non-`-1` 원소가 있는 층 기준 `MUX68by19` 19-way 병렬 선택), 전체 레이어(46개)를 순차 처리해 면적 절감.
7. 구현 디테일: MSAU(min-sum approximation unit)에서 2's-complement↔sign-magnitude 변환 후 min1/min2 선택, iteration별 LLR 스케일링 팩터(`1-2^(-I/S)`, S=상수)로 하드웨어 성능(coding loss 0.3dB) 보정.
8. 별도 학습/최적화 절차 없음(고정 min-sum 근사 + 스케일링).
9. 결과: 10 iteration으로 rate 1/3에서 BER 10⁻⁴ @ Eb/N0 1.1dB(비스케일 대비 0.3dB 열화), FPGA(Xilinx Kintex-7) 합성 시 LUT 28450/Slice Reg 3749/MUX 858, 기존 [16] 대비 약 7%, [10] 대비 약 87% 자원 절감, critical path 26.7ns(34MHz).
10. 한계: rate 1/3 core만 실제 아키텍처 설계, 나머지 rate는 puncturer/de-puncturer로 처리 가정; throughput(Gbps) 수치 미제시; 정정능력 자체 개선 없이 표준 offset-min-sum·layered 스케줄만 사용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered offset min-sum (CN 연산, offset=2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 offset 상수 적용/layered 스케줄 비교 참고 가능 |
| CVCPU (CN/VN 결합처리 + MUX68by19 부분병렬) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 부분병렬 아키텍처(19-way MUX 선택)는 Prime ECC z=32 병렬 구조와 스케일 다르나 면적-처리량 트레이드오프 설계 참고 가능 |
| IMB/IntMB 메모리뱅크 분할 및 갱신 | HW arch (병렬화, 메모리, 라우팅, pipeline) 항목 - 코드 대응 파일 특정 어려움 | 대형 프레임(26112bit) 메모리 라우팅 설계는 참고할 수 있으나 Prime ECC HCU/GT와 직접 매핑 어려움 |
| iteration별 LLR 스케일링 팩터 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블과 유사한 목적(iteration별 보정)이나 구현 방식(exponential scaling)은 상이 |
| Puncturer/de-puncturer (rate-matching) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 rate 부호라 5G식 다중 rate 펑처링 구조 이식 난이도 높음 |
```

**적용 가치**: `중간` — 정정능력 개선은 없으나 대형 프레임 QC-LDPC의 부분병렬 HW 아키텍처(멀티플렉서 네트워크 기반 CVCPU, 메모리뱅크 이원화)와 FPGA 합성 결과가 구체적으로 제시되어 Prime ECC의 HW 면적/메모리 최적화 설계 시 비교 참고 자료로 유용.
