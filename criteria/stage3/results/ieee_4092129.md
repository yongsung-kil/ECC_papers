> 총평: TDMP(layered decoding)+offset min-sum의 value-reuse 압축을 결합해 CNU 비교기/메모리를 크게 줄인 2007년 array-LDPC ASIC 논문으로, HW 구조 이식 관점에서는 유용하나 이미 유사 기법(min-sum, layered decoding)이 Prime ECC에 존재해 추천도는 중간.

### B. 알고리즘 요약

1. Array LDPC 코드(길이 2082, j=3, k=6, p=347, rate 0.5)에 대해 offset min-sum + TDMP(layered/turbo decoding message passing) 기반 완전병렬 ASIC 디코더를 설계.
2. 기존 fully-parallel LDPC 디코더는 interconnect(배선) 복잡도가 병목이며, TPMP 대비 TDMP는 iteration을 최대 50% 줄이지만 CNU 자체의 하드웨어 재사용은 최적화되지 않았다는 문제를 지적.
3. 채널은 BPSK+AWGN 가정, 체크노드 업데이트는 offset min-sum `R_mn = δ_mn * max(κ_mn - β, 0)` (β=0.15, (3,6) rate 0.5 코드 기준 density evolution으로 결정).
4. 핵심 관찰(value-reuse property): 한 체크노드의 모든 출력 `R_mn`은 실제로 입력 중 최소 2개 값(M1, M2)과 그 부호 조합으로만 결정되어 전체 집합이 `{M1, -M1, +M2, -M2}` 중 3가지 값만 가짐.
5. 이 성질로 binary-tree 기반 최소/차소 탐색기(comparator tree + trace-back mux)를 설계해 비교기 수를 `k(1+0.5(log2 k -1))`(기존 [8])에서 `k+log2 k -2`로, 덧셈기를 `4k`에서 `k+4`(5-bit)로 축소.
6. TDMP는 array code의 H행렬 구조(연속 블록열 간 순환시프트 차이가 상수)를 활용해 두 종류 순환시프트(하향 n-1, 상향 (j-1)n)만 배선으로 구현하고, p개 병렬 CNU로 한 레이어(block row) 전체를 동시 처리.
7. CNU 출력(M1,-M1,±M2, 최소값 위치 index)을 압축 저장하는 FS 레지스터뱅크와 부호 별도 저장으로 R 메시지 메모리를 20~72%(구현 코드 k=6에서는 20%) 절감.
8. Q/R 메시지 5-bit, P 메시지 6-bit 정밀도로 floating-point SP 대비 0.2dB 이내 성능 저하만 발생함을 시뮬로 확인(BER 곡선, Fig.5).
9. 0.13um 표준셀로 ASIC 구현: 칩 면적 2.3mm×2.3mm(5.29mm2), 100MHz, 최대 15 iteration 기준 decoded throughput 4.6Gbps(user data 2.3Gbps) — 기존 관련 연구([2],[5],[9]) 대비 면적·메모리 효율 우수.
10. 특정 array LDPC 구조(순환치환 기반, rate 0.5, (3,6))에 특화된 레이아웃(concentric layout)이며, 정정성능(BER) 개선이 아닌 HW 효율화가 목적이라 error-floor 등 디코더 성능 개선 기여는 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum 체크노드 처리 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2 기반 min-sum 사용, value-reuse 최적화 방식 비교 가능 |
| TDMP(layered decoding) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 iterative C2V/V2C 및 Dual-Update 스케줄 보유, 레이어드 스케줄링 세부는 대응 |
| R 메시지 압축 저장(FS 레지스터) | `decoder.cpp` `Get_VNU_*` (VNU/양자화 관련) | 메시지 메모리 절감 아이디어로 참고 가능하나 코드 명시 함수 없음 |
| array code 순환시프트 배선(concentric layout) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조(z_sb=32)라 array code 전용 배선은 직접 대응 없음 |
| 5/6-bit soft 양자화 | `channel.cpp` `Set_LLR_Th()` | Prime ECC의 2SD/3SD 양자화와 비교 가능한 수준(soft-4bit+는 Prime ECC 미지원) |

> 적용 가치: 중간 — CNU value-reuse 압축과 TDMP 결합은 HW 효율화 아이디어로 유용하지만, Prime ECC는 이미 min-sum/HCU/dual-update 등 유사 최적화를 보유하고 있어 신규성 기여는 제한적.
