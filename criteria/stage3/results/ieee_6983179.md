> 총평: FH-SS 재밍채널용 초저rate GLDPC(LDPC-Hadamard)를 EXIT 차트로 설계하는 무선 특화 논문 — Hadamard check node 구조·rate 0.09가 NAND 고rate binary QC-LDPC와 완전히 상반, 이식 가치 없음.

### B. 알고리즘 요약

1. 시스템: FH-SS(주파수도약 확산) 무선전송, BPSK, partial-band jamming(PBJ) 채널, GLDPC(LDPC-Hadamard) 부호로 초저rate(~0.09) 채널코딩.
2. 문제: PBJ 채널에서 대역 상당수가 재밍되면 저rate 코드가 필요하나 일반 LDPC는 PCM 복잡·반복 과다로 설계·HW 구현 난이도 높음.
3. 채널모델: `CPBJ(σ)=(1-P)·CBIAWGN(σ)` — 재밍확률 `P`(대역 중 재밍 비율), 재밍대역은 APBGN으로 수신불능(Y=0) 처리, 비재밍대역 AWGN.
4. 핵심기법: GLDPC = LDPC의 SPC check node를 더 강력한 블록코드로 대체; 여기선 degree-`i` Hadamard check node가 `(2^(i-2), i-1)` Hadamard code 담당, LDPC variable node는 repetition code.
5. 식 의미: Hadamard 부호어 재귀생성 `ci+1=[ci, b(i-1)⊕b(0)⊕ci]`, 짝수 degree는 systematic/nonsystematic 모두, 홀수 degree는 nonsystematic만 parity 제약 성립.
6. 확장: 전송되는 degree-1 Hadamard variable node 수 `k`를 조절해 code rate 세밀 조정.
7. 최적화: EXIT 차트 곡선정합 — variable node `IE,V=J((j-1)J^{-1}(IA,V)+J^{-1}(Ich))`, Hadamard check node는 Monte Carlo로 EXIT 곡선 산출, 두 곡선 사이 tunnel 유지하며 `R` 최대화.
8. 최적화 방법3종: (1) VN degree 분포 `{λj}` 조정, (2) `k` 조정, (3) Hadamard CN degree `{ρi}`+`k` 동시 조정.
9. 결과: SNR=-8dB, `P`=50%에서 채널용량 0.09907 대비 방법1이 91.2%(R=0.0904), 방법2 86.8%, 방법3 89.0% 달성.
10. 한계: SW 시뮬만·HW 미설계, 초저rate FH-SS 재밍채널 전용, GLDPC(Hadamard) 구조라 표준 binary SPC LDPC와 노드 구성 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Hadamard check node(GLDPC 구조) | 대응 없음 | Prime ECC는 SPC check node의 binary QC-LDPC 전용, Hadamard component code 미지원 |
| EXIT 차트 degree 분포 최적화 | `ecc_top.cpp` `Load_PCM()` (개념적, 부합 안 함) | Prime ECC PCM은 고정 QC 구조, degree 분포 재설계는 부호구조 전면 교체로 이식성 높음(부담 큼) |
| PBJ 채널 용량/모델 | `channel.cpp` (개념적, 부합 안 함) | FH-SS 재밍채널로 NAND AWGN/RBER와 성격 다름 |

적용 가치: **낮음** — GLDPC(Hadamard check node) 부호구조와 rate 0.09가 NAND 고rate binary QC-LDPC와 정반대이고, EXIT 최적화는 무선 재밍채널 전용이라 Prime ECC 부호설계/디코더에 이식할 요소가 없음.
