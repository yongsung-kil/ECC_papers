### B. 알고리즘 요약

1. Min-sum LDPC 디코더의 CN(check node) 처리는 입력 VTC 메시지들 중 첫 두 최소값(`min1`, `min2`)과 `min1`의 위치(index)를 찾아 CTV 메시지를 계산해야 함.
2. 기존 rejection-based tree 방식([25])은 `min1`의 인덱스를 이진수(binary index)로 출력하는데, min-sum 디코더 HW 구현에는 one-hot 시퀀스(예: `n=4`일 때 `1000`)가 더 유리함(멀티플렉서 select 비트로 바로 사용 가능, 추가 비교기 불필요).
3. 핵심 기법: rejection-based의 기본 유닛(`3×2-MIN`, `4×2-MIN`)의 1단계 비교 결과 플래그를 행렬 `G=[g_i,j]` (`g_i,j=1` if `x_j<x_i`)로 재구성.
4. 핵심 식: `m_i = NOR[g_i,0,...,g_i,n-1]` (식 5) — `min1`에 해당하는 행은 전부 0이므로 NOR 결과가 1이 되어 one-hot 위치를 바로 얻음.
5. 부수 트릭: 이 NOR 연산은 기존에 이진 인덱스(`I1I2`)를 계산하던 2단계 연산을 대체할 뿐이라 추가 클록 사이클(latency) 없이 동작.
6. 학습/최적화 절차 없음 (조합논리 회로 설계).
7. Latency 식: rejection-based `L_RS = (2(⌈log2 n⌉-1)+1)/f_clk`, bit-serial `L_BS = (k+2)/f_clk` (`n`=입력 개수, `k`=비트폭).
8. 결과: Xilinx Virtex-7 FPGA에서 두 아키텍처 모두 최대 클록 ~500MHz 동작 확인. Bit-serial이 일반적으로 LUT/전력 소모가 더 적음(단 `n<5`에서는 예외), rejection-based는 `n` 증가에 따른 latency 증가가 더 큼.
9. 비교 결과는 FPGA LUT/전력 수치로 제시되나 "정확한 값은 특정 ASIC 공정에서 재합성해야 신뢰 가능"이라고 저자가 명시 (본문 서술).
10. 한계: 순수 min1/min2 finder 모듈 단위 비교로, 전체 LDPC 디코더의 BER/throughput 등 시스템 성능은 다루지 않음. ASIC 합성 결과 없음(FPGA만).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum 연산(min1/min2, 인덱스 탐색) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문의 핵심 대상(rejection-based min1/min2 finder)이 바로 이 CN 연산 유닛과 대응 |
| one-hot 인덱스 기반 CTV 멀티플렉싱 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | CTV 메시지를 min1 위치에서 min2로, 나머지는 min1로 배정하는 로직에 select 비트 형태로 직결 적용 가능 |
| tree-based vs bit-serial 아키텍처 비교(latency/면적/전력) | 디코더 HW 모델(z=32, HCU, GT) | 병렬화 폭(z_sb=32)에서 CN 처리 유닛 다수 인스턴스화 시 면적/전력 트레이드오프 참고 가능 |

적용 가치: **높음** - CN 최소값 탐색은 min-sum 디코더의 핵심 반복 연산이며, Prime ECC가 이미 min1/min2 기반 min-sum을 채택하고 있으므로 이 HW 최적화(추가 지연 없는 one-hot 출력)는 `decoder.cpp`의 `CNU_Update_New_Mag()` 주변에 직접 이식을 검토할 가치가 있음.
