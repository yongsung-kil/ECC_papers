### B. 알고리즘 요약

1. 대상: 32GB TLC NAND SSD, page 4kB/block 1MB, 4채널 per-channel BCH + 공유 LDPC 1개 하이브리드 ECC. BCH `(8262,8192,5)`, LDPC `(8360,8192)`, LDPC redundancy overhead 9.39%.
2. 문제: 전통적 wear leveling(WL)이 BCH lifespan을 초기에 소진시켜 조기 성능 저하 유발, 강한 LDPC 디코딩 지연이 read 성능 악화.
3. 핵심 아이디어: 의도적으로 불균등 마모(anti-WL)를 만들어 hot 데이터를 소수 BCH 블록에 집중, LDPC 블록 접근을 최소화해 디코딩 오버헤드 은닉.
4. 블록 3분류: `BCHold`(P/E 큰 소수 블록, hot write 집중), `BCHyoung`(read-hot 저장), `LDPC`(P/E 임계 초과 or read-cold). 임계 초과 BCH 블록은 LDPC 모드로 전환(비가역).
5. anti-WL write: write를 BCHold에 집중 → 대부분 erase가 BCHold에 몰려 hot/cold 자연 분리, WA(write amplification) 감소.
6. anti-WL GC(Alg.1~4): LDPC→BCHyoung→BCHold 순 재귀 GC, R/W flag 기반 second-chance victim 선택. LDPC 블록은 확률 재활용 `blkRP = ρ·w_erase + (1-ρ)·w_invalid`.
7. 부수: BCH parity가 spare에 들어가 BCH 블록이 더 많은 user page 저장(용량 이득), LDPC redundancy는 별도 page 예약.
8. 오버헤드: 저장 오버헤드 ~1%(32GB에 32.3MB), Alg 시간복잡도 O(Nblock)/O(Npage)로 무시 가능.
9. 결과: MSR trace(web/media/source)에서 write ART가 LDPC 대비 50.5~66.5%, read/write 수명 ~50.6% 향상, 전체 수명 1.5x, 전력 LDPC 1.4x 대비 anti-WL 1.18x.
10. 한계: HW 미설계(trace 기반 시뮬), LDPC/BCH를 블랙박스 latency 파라미터로만 모델링, process variation 미고려(향후).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| anti-WL write/GC 정책 | 대응 없음 | FTL/컨트롤러 블록 관리, ECC 부호/복호와 무관 |
| 하이브리드 BCH/LDPC 모드 전환 | 대응 없음 | LDPC를 latency 블랙박스로 취급, 디코더 알고리즘 미변경 |
| LDPC 접근 빈도 저감 | 대응 없음 | 시스템 계층 접근 회피이지 min-sum/soft-read 비용 절감 아님 |

적용 가치: **낮음** — 부호설계·디코더·HW 어느 레이어에도 닿지 않는 FTL 계층 wear-leveling/GC 관리 기법으로, Prime ECC 코드에 이식할 요소가 없음.
