> 총평: irregular LDPC convolutional(SC-LDPC) 부호를 shift-register로 고속 인코딩 가능하게 구성하고 동일 하드웨어 복잡도에서 block code 대비 waterfall 성능(~0.8dB) 개선을 보이나, 파이프라인 슬라이딩 윈도우 디코더 구조와 초기 지연이 NAND 블록 단위 ECC와는 정합성 이슈가 있어 이식성은 중간 수준.

### B. 알고리즘 요약

1. 시스템: rate `R=b/c` LDPC convolutional code, 준무한(semi-infinite) syndrome former 행렬 `H^T`로 정의, syndrome former memory `m_s` (구속장 `ν_s=(m_s+1)c`).
2. 문제: 기존 LDPC convolutional code 구성법([12])은 weight-1 순열행렬 타일링으로 regular 부호만 가능, irregular degree distribution 미지원.
3. 핵심 가정: AWGN 채널, syndrome former는 주기 `T=m_s`로 periodic.
4. 핵심 기법: 원하는 edge-perspective degree distribution `(λ_e^C, ρ_e^C)`으로부터 node-perspective를 구하고, 이를 `X`, `X^2`로 나눠 축소한 `(λ_n^B, ρ_n^B)`을 갖는 길이 `m_s·c`의 LDPC block code를 먼저 구성(PEG 알고리즘으로 girth 확보).
5. 핵심 식 의미: block code의 syndrome former 행렬을 대각선 방향으로 `c×(c-b)` 단위로 잘라 우측에 이어붙이고, 각 행 쌍 좌측에 `[1 1]^T` 서브블록을 추가해 `H_0^T(t)`가 항상 풀랭크가 되도록 함 -> fast encoding(shift-register) 성질 보장.
6. 확장: rate `R=b/c=1/2`, 주기 `T=m_s`인 경우로 구체 예시(m_s=4)를 통해 구성 절차 시연.
7. 부수 트릭: 인코딩 복잡도는 코드워드 길이/`m_s`에 무관하게 `(K-1)`에 비례(block code는 길이 N에 비례) -> 저복잡도 실시간 인코더.
8. 복호: sliding window pipeline decoder(I개 독립 프로세서, on-demand 스케줄), 최대 100 iteration.
9. 결과: `m_s=1199` convolutional code가 동일 프로세서 복잡도의 block code(N=2400) 대비 BER=10^-5에서 0.80dB 이득; N=240000 block code보다는 0.30dB 열세.
10. 한계: 시뮬레이션만(HW 미설계), 초기 디코딩 지연 `I·(m_s+1)` 타임유닛 존재, 매우 큰 `m_s`로 인해 사실상 긴 유효 프레임 필요, block 단위 프레이밍이 아닌 continuous transmission 지향.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Irregular SC-LDPC 구성(H-matrix, degree distribution) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 block QC-LDPC 구조로, convolutional(semi-infinite) 부호 구조 자체가 대응 없음 |
| Shift-register 기반 고속 인코더 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 인코더와 유사한 목적(저복잡도 실시간 인코딩)이나 convolutional 구조 특화 기법이라 직접 이식 어려움 |
| Sliding window pipeline 복호(I개 병렬 프로세서) | `decoder.cpp` `LDPC_Decoder()` | 반복 message-passing 복호라는 점은 공통이나, Prime ECC는 고정 block 파이프라인(z=32 등)이라 convolutional sliding window 구조와는 상이 |

> 적용 가치: 낮음 — SC-LDPC(convolutional) 고유의 continuous 프레이밍/sliding window 복호 구조가 Prime ECC의 고정 block QC-LDPC 구조와 근본적으로 달라 직접 이식은 어려우나, waterfall 성능 개선 기법(irregular degree 최적화) 자체는 참고 가치가 있음.
