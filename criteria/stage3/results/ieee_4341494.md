### B. 알고리즘 요약

1. IEEE 802.16e 표준의 AA-LDPC(architecture-aware) 코드 전체(rate 1/2, 2/3A/B, 3/4A/B, 5/6, 총 114개 codeword 크기, `zf=24~96`, prototype matrix 24열)를 지원하는 유연한 디코더 설계.
2. 문제: serial 처리 유닛은 저전력·저면적·유연성이 좋으나 처리량이 낮아, layered decoding + pipelining으로 수렴속도와 처리량을 동시에 확보해야 함.
3. 채널 모델은 별도 명시 없이 표준 LDPC soft-decision 디코딩(LLR 기반) 가정.
4. 핵심 기법: layered(shuffled) decoding — VN/CN phase를 겹쳐서 레이어별 micro-iteration으로 처리, 갱신된 reliability를 다음 레이어에서 즉시 사용(`y_n^(q)[k+1] = y_n^(q)[k] - ε_{m,n}^(q) + ε_{m,n}^(q+1)`)해 수렴속도를 2배로 향상.
5. 핵심 식(3)-(5)는 CN 단위 sign/magnitude 갱신(mod-min-sum, `Φ^{-1}` 근사 기반)이며, 802.16e 코드 구조상 레이어를 `W=zf`개 병렬 parity-check로 구성해 column weight ≤1을 보장, 병렬 처리 가능.
6. 확장: 레이어 간 의존성(공유 SO 메시지)로 인한 idle cycle을 레이어 순서 최적화 및 메시지 정렬로 최소화(클래스별 idle cycle이 iteration 시간의 1~39%).
7. 부수 트릭/구현: Multi-Size Circular-Shifting(MS-CS) 네트워크(24개 4×4 Banyan barrel-shifter + adaptation network)로 19가지 codeword 크기 지원; FIFO-based(기존, V2C 메시지 버퍼링) vs memory-based(FIFO 제거, V2C를 CN-core 출력에서 on-the-fly 재계산, 3-port 메모리를 dual-port 2뱅크로 구현) 두 아키텍처 비교.
8. 학습/최적화 절차: memory-based 뱅크 할당은 오프라인 컴퓨터 탐색으로 redundancy 최소화(SO memory 29%, C2V memory 4.5% redundancy).
9. 결과: 65nm CMOS 합성, 400MHz, 15 iteration에서 throughput 93~497Mbps(class별), FIFO-based 161Kgates/0.59mm², memory-based 117Kgates/0.55mm²(RTL 27%, 면적 7% 절감). 15 iteration으로 100 iteration 대비 SNR loss 약 0.2dB.
10. 한계: 802.16e 표준 특화(고정된 prototype matrix 구조), NAND 채널이 아닌 무선 채널(AWGN) 대상, 정정능력 자체 개선이 아닌 HW 복잡도/처리량 최적화에 집중.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered(shuffled) decoding 스케줄 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 옵션) | 이미 Dual-Update로 유사한 odd/even 교대 스케줄 보유, layered decoding의 idle-cycle 최적화 기법은 참고 가능 |
| FIFO 제거 및 V2C on-the-fly 재계산(memory-based 아키텍처) | `decoder.cpp` `LDPC_Decoder()`, HCU 관련 메모리 접근 로직 | 메모리/면적 절감 목적이 Prime ECC의 HCU와 유사한 방향, 재구현 시 참고 가치 |
| Multi-Size Circular-Shifting(MS-CS) 네트워크 (다중 codeword 크기 지원) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 단일 z_sb=32 고정 구조라 다중 크기 지원 목적과 직접 대응 안 됨 |
| mod-min-sum(2-output 근사) CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min-sum 계열(min1/min2) 보유, 중복 기여로 추천도 낮음 |

> 적용 가치: 중간 — layered decoding·idle-cycle 최적화·memory-based(FIFO 제거) 아키텍처는 이미 보유한 Dual-Update/HCU와 방향이 겹치나 구체적 메모리 뱅크 할당·redundancy 최소화 기법은 HW 면적 절감에 참고할 만함.
