### B. 알고리즘 요약

1. 대상은 irregular LDPC 코드(block length 2000, 3000)의 AWGN 채널, BP 디코딩(최대 200 iteration).
2. 풀려는 문제: PEG/IPEG로 구성한 코드도 짧은 stopping set이 남아 BEC/error-floor 성능을 제한.
3. 배경 정의: variable node 부분집합 `S`가 stopping set이면 모든 인접 check node가 `S`에 2번 이상 연결됨. `EMD(S)`(extrinsic message degree)는 `S`에 단 1번만 연결된 extrinsic check node 수, `ACE`는 길이-`2d` 사이클의 `Σ(d_i-2)`.
4. 핵심 기법: 기존 IPEG는 새 edge 추가 시 후보 check node 중 최소 ACE를 최대화하는 노드를 선택하고, 동률이면 무작위 선택 - 이를 개선해 동률(`Φ_sj^k`) 시 tree-expanded subgraph의 column 합에서 1의 개수(EMD)가 더 큰 check node를 우선 선택.
5. EMD가 클수록 stopping set 형성 가능성이 낮아짐(`EMD(stopping set)=0`)이라는 관계가 핵심 근거.
6. 추가 동률 시엔 subgraph의 variable node 수가 더 많은 쪽을 선택, 그래도 동률이면 무작위 선택(3단계 tie-breaking).
7. 부수 디테일: 별도 양자화/HW 변형 없음, 순수 그래프 구성(오프라인) 알고리즘.
8. 최적화 절차: 기존 최적 degree distribution(`λ1(x)`, `λ2(x)`, Richardson-Shokrollahi 기반)을 그대로 사용하고 edge 배치 알고리즘만 개선.
9. 결과: length 2000 코드에서 기존 IPEG는 FER 1e-5에서 error floor 발생하나 제안 코드는 해당 구간에서 floor 미발생, length 3000도 유사(IPEG FER 1e-4서 floor, 제안 코드는 없음); waterfall 영역 성능 손실 없음.
10. 한계: 순수 코드 구성법으로 디코더/HW 설계 없음, 검증된 길이(2000~3000)가 NAND ECC 대비 상당히 짧고 rate 정보 미기재.
