### B. 알고리즘 요약

1. 대상 부호는 BCH code(HDPC, high-density parity-check) — LDPC와 달리 PCM이 조밀해 짧은 사이클(특히 4-cycle)이 많아 BP 디코딩 성능이 저하되는 문제.
2. 기존 RRD(random redundant decoding)는 자기동형군(automorphism group) `Per(C)`에서 무작위 permutation을 선택해 반복 복호하나, 중간~저rate HDPC에서 short cycle 밀집으로 비신뢰 정보(unreliable info) 전파가 심함.
3. 채널 모델: AWGN, BPSK 변조. LLR `L_i = log(Pr(b_i=1|r)/Pr(b_i=0|r))`.
4. 핵심 기법 1단: variable node(VN)를 PCM 내 포함된 최단 사이클(4-cycle) 개수 `g_i` 기준으로 k-median 알고리즘(Algorithm 1)으로 k개 클러스터로 분류, 최소 클러스터 `s0`(사이클 적은 VN 집합) 도출.
5. 핵심 식: permutation reliability metric `η_πu,o = Σ_{πu,o(i)∈s0} |L_vi|`(식6) — 비신뢰 비트를 `s0`(사이클 적은 위치)로 보내는 permutation을 우선 정렬해 비신뢰 정보 전파 억제.
6. 핵심 기법 2단(확장): 정렬된 permutation 리스트 기반 NC-RD 알고리즘(Algorithm 2, e/h/q 3중 루프)과, 다중 복호로 codeword list를 생성하는 mNC-RD(list decoding) 확장.
7. 부수 트릭: NC-RD를 feed-forward 신경망(NC-RDNN)으로 변환 — e-loop/h-loop의 C2V 메시지에 곱셈형 tunable weight `w̃`, `ŵ` 부착, permutation 행렬은 각 realization별 개별 적용(기존 Perm-RNN과 차별점).
8. 학습 절차: TensorFlow 기반, cross-entropy loss(식9)로 all-zero codeword(AWGN 오염) 기반 오프라인 SGD 학습.
9. 결과: BCH(31,21)에서 RRD 대비 NC-RD 0.2dB, NC-RDNN 추가 0.18dB, mNC-RD 추가 0.25dB 이득(OSD-2/ML 성능에 근접). BCH(63,36)에서 NC-RD가 RRD 대비 0.5dB 개선.
10. 한계: HW 미설계(순수 알고리즘 시뮬), 대상이 BCH(HDPC)로 short/moderate length(n=31~255)에 한정, LDPC와 달리 PCM이 원래 조밀(dense)해 "4-cycle이 많다"는 전제 자체가 저밀도 QC-LDPC와 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN short-cycle 기반 클러스터링(k-median) | 대응 없음 | Prime ECC PCM(QC-LDPC)은 이미 4-cycle-free 설계(RS-LDPC 마스킹)라 BCH의 조밀 PCM 전제와 무관 |
| Redundant permutation decoding(NC-RD/mNC-RD, e/h/q 루프) | `decoder.cpp` `LDPC_Decoder()` | multi-permutation 반복 복호 구조는 개념적으로 이터레이션 루프와 유사하나, automorphism group 기반 permutation은 BCH cyclic code 특유의 성질(`π_{u,o}(i)=2ui+o mod n`)로 QC-LDPC 구조와 미대응 |
| BP C2V/V2C 업데이트(tanh 기반) | `decoder.cpp` `C2V_Cal()` | Prime ECC는 Min-Sum이며 논문은 full-tanh Sum-Product 기반, §5 판정필터상 "대응 없음" 예시(full-tanh SPA)에 해당 |
| 신경망 가중치 보정(NC-RDNN) | 대응 없음 | NN/딥러닝 디코더는 프로파일 §6에서 "대응 없음" 명시 사례 |

적용 가치: **낮음** — BCH(HDPC, dense PCM) 전용 redundant permutation decoding + NN weight tuning 기법으로, Prime ECC는 binary QC-LDPC(sparse PCM, 이미 4-cycle-free) 전용이라 문제 전제 자체가 달라 이식 근거 희박.
