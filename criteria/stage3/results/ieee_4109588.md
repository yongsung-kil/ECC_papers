### B. 알고리즘 요약

1. 대상은 IEEE802.16e에 규정된 짧은 LDPC(192,96) 및 (384,192), rate=1/2, CRC-8(`X8+X4+X3+X2+1`) 연접.
2. 짧은 codeword 길이에서는 LDPC가 예측 성능(sphere-packing bound)에서 크게 벗어나는 문제(Tanner Graph의 short cycle로 인한 BP 최적해 도달 곤란)를 해결하고자 함.
3. AWGN 채널, BPSK 변조(`x_i=1-2c_i`), 표준 LLR 도메인 BP(min-sum 아님, tanh 기반 Sum-Product) 사용.
4. 1차 BP 복호 결과가 실패(패리티 불만족)이거나 CRC 불일치이면, 초기 수신신호 `Y'`에서 신뢰도(|LLR|)가 가장 낮은 L개 비트를 선택해 perturbation을 가하고 재복호.
5. Perturbation은 `y_i = y_i + n_i'` (선택된 L개 비트에만 분산 `σ_p^2`의 가우시안 잡음 추가)로, 초기화 시 belief 분포를 바꿔 다른 국소해로 수렴을 유도.
6. 재복호 루프는 최대 N=100회까지 반복하며, 매 루프마다 CRC로 결과를 검증해 통과 시 종료.
7. 부수 디테일: perturbation 대상 비트 수 L과 전력 `σ_p`는 시뮬레이션으로 최적화된 고정값 사용(L=48/σ_p=0.35 for (192,96), L=72/σ_p=0.30 for (384,192)).
8. 학습/최적화 절차: L=24+12i (i=0..6), σ_p=0.25+0.05j (j=0..5) 격자 탐색으로 SNR 구간 1~4.5dB에서 최적 조합을 시뮬레이션 탐색.
9. 결과: FER=10^-4에서 IEEE802.16e 대비 약 0.5dB 이득, sphere-packing bound와 1.12dB(N=192)/1.19dB(N=384) 차이로 근접.
10. 한계: HW 미설계, 재복호 루프(최대 100회 x BP iteration 최대 50회)로 연산량이 크게 증가하며 실제 구현 시 대역폭 효율은 이득이나 계산 파워 부담이 크다고 저자도 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CRC 기반 오류 검출 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료/오류검출 목적은 동일하나, 재복호 트리거 용도로 사용하는 점이 다름 |
| 저신뢰도 비트 선택 + perturbation 재복호 | 대응 없음 | 우리 코드는 min-sum + LLR 테이블 기반 적응형 조정은 있으나 랜덤 잡음 주입 재복호 기법은 없음 |
| BP(Sum-Product, tanh) 복호 알고리즘 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 우리는 min-sum 근사를 사용하므로 tanh 기반 정확 SPA는 이질적(§3: full-tanh Sum-Product는 대응 없음) |
| 데이터 메모리에 원본 수신신호(Y') 보관 | `channel.cpp` | 재시도용 원본 보관 개념 자체는 채널 모듈과 관련되나 세부 구현 대응 없음 |

> 적용 가치: 낮음 — 정정능력(error-floor)을 개선하지만 tanh 기반 SPA 전제, HW 미설계, 최대 100회 재복호 루프로 latency 악화가 뚜렷해 NAND read latency 제약과 상충되며, 핵심 아이디어(perturbation 재복호)는 우리 코드 구조에 대응 모듈이 없어 이식 난이도가 큼.
