### B. 알고리즘 요약

1. Braided convolutional code(BCC, (2,3)-regular compact graph)를 임의의 `(dv, dc)`-regular 그래프로 확장한 CC-GLDPC 코드 앙상블을 정의, rate R=1/3 ((2,3)/(4,6)/(6,9))과 R=1/2 ((2,4)/(3,6)/(4,8)) 예시를 다룸.
2. 문제: 기존 SC-LDPC는 variable node degree를 늘려야 threshold/최소거리가 개선되나, 유한 길이에서는 degree 증가가 short cycle을 늘려 BP 성능을 해침 — degree 대신 constraint node(component code)의 강도(trellis state 수)를 높여 같은 효과를 낼 수 있는지 분석.
3. constraint node를 rate-1/2 mother recursive systematic convolutional code(2/4/8-state, generator (1,1/3),(1,5/7),(1,13/15))의 trellis를 puncturing해 rate `(dc-1)/dc` component code로 구성, 채널은 BEC(erasure probability `ε`).
4. 핵심 기법: `q(i)(ej,k)`(VN→CN erasure확률), `p(i)(ej,k)`(CN→VN)의 density evolution 재귀식을 punctured trellis의 extrinsic transfer function `fs, fp`(식 8-9)로 정의, uncoupled(식 4-7) 및 spatially-coupled(coupling memory `m`, 식 13-16) 양쪽에 대해 유도.
5. 핵심 식: MAP threshold 상한은 area theorem 기반 `∫p̄e(ε)dε = R`(식 17)로 계산 — BP와 MAP threshold의 격차(및 spatial coupling에 의한 saturation)를 정량화하는 핵심 도구.
6. 확장: single-edge type 앙상블(비구조 그래프, Fig.3(b))로 단순화해 Gallager 스타일 weight enumerator `Ā(w)_dv,dc`(식 2)를 유도하고 최소거리 하한 `d̂`(식 3, α=0.5)를 계산.
7. 부수 디테일: puncturing은 각 세그먼트 내 균등 랜덤(패턴 최적화는 미해결 문제로 남김), single-edge type 근사로 수치적 처리가능성 확보.
8. 최적화/학습 없음 — 순수 해석적 DE 반복 계산 + weight enumerator 조합론.
9. 결과: trellis state를 늘리면 최소거리·MAP threshold 개선되나 BP threshold 저하; 예로 (4,6)-4state 앙상블은 (2,3)-8state나 (6,9)-2state보다 낮은 dv로 유사/우수한 MAP threshold와 최고 수준 minimum distance 확보, spatial coupling 적용 시 BP threshold가 MAP threshold로 saturate.
10. 한계: BEC 채널·이론(DE/weight enumerator) 분석에 한정, binary AWGN이나 실제 FER 시뮬레이션·HW 구현이 전혀 없으며 복잡도-성능 트레이드오프 분석도 범위 밖으로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CC-GLDPC 앙상블 구성(convolutional constraint node) | 대응 없음 | check node가 SPC가 아닌 convolutional trellis로 대체된 GLDPC 구조로, Prime ECC의 QC-LDPC(SPC check node) 구조와 근본적으로 다름 |
| Density evolution 임계값 분석 (BEC) | 대응 없음 | BEC 채널 기반 이론 분석 도구이며 Prime ECC는 AWGN/RBER 채널의 bit-exact 시뮬레이터로 DE 자체를 사용하지 않음 |
| 최소거리 하한(weight enumerator) | 대응 없음 | 앙상블 이론 지표로 구현 코드에 대응되는 모듈 없음 |
| Spatially coupled 확장(threshold saturation) | 대응 없음 | Prime ECC는 단일 block QC-LDPC 고정 구조, spatial coupling 개념 미적용 |

적용 가치: `낮음` — convolutional constraint node 기반 GLDPC 앙상블의 BEC 채널 이론 분석으로, 부호 구조·채널·검증 방식 모두 Prime ECC의 binary QC-LDPC HW 시뮬레이터와 대응점이 없음.
