### B. 알고리즘 요약

1. 부호: rate 1/2, 길이 `N=m*k=337*24=8088` bit irregular LDPC (정보 4044 bit), 병렬인자 `p=24`, 최대 25 iteration.
2. 부호구성: Irregular Partitioned Permutation(IPP) - 작은 `12x24` irregular 블록행렬 `H'`의 각 원소를 소수 `m`의 순환치환행렬(shift `= a^s b^t mod m`)로 치환해 rate `R>=1-j/k`, 길이 `m*k` 부호 생성.
3. 복호: log-domain belief-propagation. V2C 메시지 `Lqmj = Lqj - Rmj`, C2V 메시지 `Rmj = -{prod sign(Lqmn)} Psi(sum Psi(Lqmn))`, 여기서 `Psi(x)=log(tanh(|x/2|))`.
4. 정지: 최대 iteration 도달 또는 `H·x=0` (조기종료 회로 내장).
5. 아키텍처: 부분병렬 구조 - Parallel Adder Block(PAB, 식(2)+패리티체크), Parity Check Update Block(PCUB, 식(3)), Column Sum Block(CSB, 식(1) 누적) + router/reverse router로 그래프 연결 실현.
6. PCUB: `Psi` 대칭성 이용해 sign-magnitude 입력으로 LUT 절반 축소(32개 5-bit 엔트리), 클록당 7~8 데이터(=행 가중치 min/max) 처리.
7. Router: 블록행렬을 9개 섹션으로 분할해 섹션당 연결 1개만 존재하도록 -> 라우팅이 블록 내 고정, 블록 전환 시에만 MUX 스위칭 (log2(p)=5층 circular shifter).
8. CSB(가장 복잡): dual-memory A/B ping-pong으로 이전 iteration 누적값 읽기와 현재 누적을 동시 수행 -> 처리량 향상; alignment/reverse alignment로 행순서<->열순서 재배열(gap closing으로 dummy 처리).
9. 결과: FPGA(Xilinx XC2V8000, 44MHz) 정보 처리량 40Mbps; ASIC(TI GS-40 0.11um, 212MHz) 데이터율 188Mbps, 335K logic + 407K memory gate(2.61mm^2 메모리).
10. 한계: rate 1/2 저rate·full-BP(Psi LUT) 특정 IPP 부호에 최적화, 정정능력·iteration 개선 기여는 없음(순수 구현 논문).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PCUB check-node 갱신(Psi LUT full-BP) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리는 min-sum, 논문은 log-tanh LUT — 알고리즘 상이하나 CN 갱신 위치 대응 |
| CSB variable-node 누적 | decoder.cpp VN_Cal_HD() 등 | VN 누적/양자화 대응 위치 |
| router/alignment 부분병렬 라우팅 | decoder.cpp 디코더 HW 모델(z=32, GT) | 부분병렬 edge 라우팅 개념 대응, 우리 Graph Thinning과 목적 유사 |
| 조기종료(H·x=0 체크) | full_CRC.cpp / partial_CRC.cpp | 우리는 CRC 조기종료, 논문은 syndrome 체크 |
| 순환치환 IPP 부호구조 | ecc_top.cpp Load_PCM() | 부호 로드 위치이나 IPP≠우리 base+circulant QC 구조 |
```

마지막에 한 문장으로 적용 가치: `중간` — 부분병렬 메모리 정렬·ping-pong dual-memory·라우터 스케줄은 HW 아키텍처 참고가치가 있으나, 저rate·full-BP·IPP 특정부호라 Prime ECC 고rate QC-LDPC min-sum 파이프라인에 그대로 떼어 쓰긴 어렵다.
