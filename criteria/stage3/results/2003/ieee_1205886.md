### B. 알고리즘 요약

1. 시스템: 프레임 2304-bit LDPC, 5-bit datapath, 0.18um 1.8V CMOS, 200MHz에서 1.92Gbit/s 목표 디코더.
2. 문제: 랜덤 sparse H는 병렬 디코더의 인터커넥트 폭발과 직렬 디코더의 메모리 부담을 야기 → 저지연·저전력 응용에 부적합.
3. 핵심기법1(부호설계): H를 `B×D` 블록행렬로 구성, 각 블록은 `S×S` all-zero 또는 permutation 행렬, `n=S·B`, `R≥1-D/B` — architecture-aware LDPC(AA-LDPC).
4. 라우팅 이득: 블록구조 덕에 `rS`(n:1) 대신 `rS`(J:1), `J<B<<n` mux만 필요, S개 mux를 공통 sequencer로 제어 → 인터커넥트 60.5% 감소.
5. 핵심기법2(복호): TDMP(turbo-decoding message-passing) — 블록행을 순차(sub-iteration) 처리하는 turbo-style 스케줄, check-to-bit 다중 메시지 저장 제거(75% 저장 절감), 수렴 20~50% 빠름.
6. MPU: BCJR 키 방정식을 differential 형태의 `Q(x,y)=max(x,y)-max(x+y,0)` "max-quartet"로 근사 → LUT 불필요, 병렬 MPU 구현비 절감, 이상 대비 성능손실 미미.
7. 아키텍처: B 메모리 모듈 + S개 병렬 MPU + read/write 전송 네트워크, 블록행별 read(누산·permute)→MPU→write 파이프.
8. 부호품질: cyclotomic coset / Ramanujan graph 기반 AA-LDPC가 랜덤 부호와 대등한 BER임을 인용([7,8,10,11]).
9. 결과: TPMP 대비 전력 89.13%·면적 62.80%·인터커넥트 wire 60.5% 절감, 메모리 면적 16.06%, 네트워크가 면적 74.6% 차지, 1.92Gbit/s.
10. 한계: BER 곡선/정정능력 개선 데이터 없음(HW 효율 논문), 복호 알고리즘이 turbo-decoding(BCJR) 기반으로 min-sum과 상이, 부호rate 명시 없음.
