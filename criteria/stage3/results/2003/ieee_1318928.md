> 총평: turbo-like tree+interleaver로 large girth 부호를 유연 rate/length로 만드는 부호설계 논문. error-floor 완화·메모리 절감 아이디어는 매력적이나 시뮬 검증 없고 QC 구조가 달라 이식성 중.

### B. 알고리즘 요약

1. 시스템: 구조화 regular binary LDPC, column weight `j`, row weight `k`, rate `p=1-j/k`(예 j=3,k=27→8/9), 예시 (2457,3,9) rate 2/3.
2. 문제: cyclic/QC-LDPC는 Tanner 정리상 girth 상한(j≥3→6, j=2→12)으로 제한, random 부호는 girth 좋으나 rate/length 유연성·HW 단순성 부족.
3. 모델: factor graph girth = 최단 cycle 길이; large girth → message passing 의존성↓·수렴 빠름, min distance↑로 error floor 완화.
4. 핵심기법1: 두 height-balanced sub-tree(upper `Tu`, lower `TL`)를 leaf끼리 interleaver로 연결 → 각 tree 내부는 cycle 없고 cycle은 interleaver가 도입.
5. 핵심식: leaf에 auxiliary node 도입, index를 `p-q-alternate decimal`(p=k-1,q=j-1)로 표기; type-I cycle 길이 `du(a1,a2)+dL(a3,a4)+2`, 상한 `2h+4`.
6. 핵심기법2: Theorem 1~2 — symbol-wise reversal `πs(·)`+shift `S`로 `x_{q-p}`↔`πs(x_{q-p})` 매핑해 type-I cycle girth 최대화.
7. 부수트릭: Theorem 3 — group 개념+shift `S_{i,j}` 조건식(1~8)을 피해 length-4/6 type-II cycle 제거; j=2는 length 8·12 제거로 girth≥16.
8. 결과(구성): (2457,3,9) girth 8 부호를 819×2457 H로 구성; j≥3 girth≥8, j=2 girth≥16 보장, rate/length 유연.
9. 메모리 이득: random (n,j,k)는 nj LUT 필요(복잡한 메모리접근), turbo-like는 (j-1)(k-1)×(k-1) shift 행렬만 — (2457,3,9)에서 7371→128 entry로 대폭 절감.
10. 한계: BER/FER 시뮬·HW 구현 없음(이론·구성만), 상세 증명은 별도 submitted 문헌 참조, QC circulant 아닌 tree+interleaver 구조.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| turbo-like tree+interleaver H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base+shift QC-LDPC 고정, tree+interleaver 구조와 상이해 직접 로드 불가 |
| symbol-wise shift 행렬 저장(메모리 절감) | `ecc_data.h` `PCM_b`(base+shift) | QC의 circulant shift 저장과 개념 유사, large-girth shift 설계 원칙만 참고 |
| large girth로 error-floor 완화 | `corner.cpp`(수렴실패 추적) | error-floor 개선 목표는 부합하나 검증(시뮬) 부재 |
| 인코딩 메모리/구조 | `encoder.cpp` `Generate_PCM_encoding()` | turbo-like는 소메모리 인코딩 주장, dual-diagonal과 방식 상이 |

적용 가치: **중간** — large girth로 error-floor를 완화하고 shift 행렬로 부호 표현 메모리를 줄인다는 방향은 Prime ECC의 고rate QC-LDPC 부호설계 목표와 부합하나, tree+interleaver 구조가 circulant QC-LDPC와 다르고 BER 검증이 없어 shift 설계 원칙을 QC lifting에 재해석하는 추가 작업이 필요.
