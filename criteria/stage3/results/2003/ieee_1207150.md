### B. 알고리즘 요약

1. 대상 부호는 `(504, 252)` regular LDPC, rate `1/2`, 무선 flat/2-path Rayleigh 페이딩 채널에서 평가.
2. 문제: 표준 BP(sum-product)는 `tanh`류 곱셈·나눗셈이 많아 HW 복잡, 또 채널상태정보(CSI)를 요구해 채널이 자주 변하는 무선에 부적합.
3. UMP(uniformly most powerful) BP-based 알고리즘 도입: 실수 덧셈만 사용하고 채널 사전정보(a priori) 불필요.
4. CN 처리: 부호는 입력 부호 XOR(`u_mn = ⊕ z_n'`), 크기는 이웃 최소값 `min|z_n'|`으로 근사 — 전형적 min-sum 형태(식 4,5).
5. VN 처리: `z_mn = |c_n| + Σ l_mn`로 LLR 누적 갱신(식 6,7), 부호 판정 후 `Hx=0`이면 조기 종료.
6. 복잡도표(Table I): rate 1/2 부호 1 iteration당 BP는 `3Nj+N` 덧셈 + `11Nj-9N` 곱셈 + `Nj+N` 나눗셈, UMP는 `Nj/2(8j-7+⌈log2 2j⌉)` 덧셈만.
7. 양자화: 수신신호를 클리핑(`±A_max`) 후 midrise 균일 양자화(3-bit 예시: ±D~±4D 8레벨)하여 디코더 입력.
8. 결과1: max iteration 50에서 UMP BER은 BP와 거의 동일; 수렴 임계(BP 200 / UMP 50 iter)에서는 UMP가 flat 0.35 dB, 2-path 0.40 dB 열세.
9. 결과2: 5-bit·7-bit 양자화 BER이 무양자화와 거의 동일(BP·UMP 양쪽, 두 채널 모두).
10. 한계: HW 미설계, 시뮬레이션만, 무선 Rayleigh 페이딩 가정, gate count·throughput 미제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| UMP BP-based CN min 근사 (식 5) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리 min-sum과 동일 계열, 신규성 없음 |
| VN LLR 누적 (식 6,7) | decoder.cpp VN_Cal_HD() 등 | 표준 message-passing, 중복 |
| 수신신호 클리핑·균일 양자화 | channel.cpp Set_LLR_Th(), Set_R_Offset() | soft 입력 양자화, 다만 Rayleigh 가정이라 NAND와 상이 |
| 5/7-bit 양자화 성능 등가 | channel.cpp + LLR 테이블(PARAM_LLR) | 참고성, NAND는 2~3bit 목표라 직접 이득 없음 |
```

적용 가치: **낮음** — UMP BP-based는 우리가 이미 가진 min-sum과 동일 계열이고 기여가 무선 페이딩 채널 평가·양자화 비트수 확인에 국한되어 NAND binary QC-LDPC 이식 신규 가치가 없다.
