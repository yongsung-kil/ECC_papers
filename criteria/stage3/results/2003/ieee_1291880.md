### B. 알고리즘 요약

1. 시스템: LDPC coded M-ary modulation (QPSK/16QAM/64QAM) over Rayleigh fading, 코드 rate 1/2, block 9036 bit(regular (3,6)).
2. 문제: 고차변조·고rate에서 초기 입력 LLR의 dynamic range가 100 이상으로 커져 유한정밀도 디코더의 양자화 손실이 심각.
3. 채널모델: frequency non-selective Rayleigh fading `y=hx+n`, coherent detection, complex Gaussian fading `h`.
4. 핵심기법1: 균일양자화 `(q:f)` — 총 q비트 중 f비트를 소수부에 배정; f↓하면 dynamic range↑ 정밀도↓ trade-off.
5. 핵심식: check node 갱신을 `tanh` rule로 표현, MSC는 `sgn(AB)·min{|A|,|B|} + correction(|A+B|,|A-B|)` — 근사항이 저SNR 정확도 보정.
6. 핵심기법2: 세 디코더 비교 — 직접 SPA(LUT), min-sum(MS), min-sum+correction(MSC); MSC는 비선형함수 불필요해 정밀도 제약 없음.
7. 구현디테일: SPA는 LUT saturate 필요(Xmax/Ymax), 큰 입력에서 테이블에 0 다수 발생→decode 실패; MSC는 correction항 계산에 추가 HW 로직 필요.
8. 최적화절차: discretized Density Evolution(FFT 기반 pmf convolution)로 code ensemble 별 threshold 계산, 양자화 스킴별 비교.
9. 결과: MSC(5:1) 부동소수 대비 0.1dB 손실(전 변조/rate), SPA(6:3)는 저rate만 양호(고rate 0.4dB↑ 손실), MS는 0.5dB 손실; FPGA(Virtex II) 60Mbps@24 iter.
10. 한계: Rayleigh fading + BICM 전제, error-floor 미분석, avg iteration 감소 없음, 부호설계 기여 없음(양자화·디코더 알고리즘 비교에 국한).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum + correction term(MSC) CN 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min-sum(min1/min2)만 보유, correction항은 미보유 → 저SNR 성능 개선 여지 |
| 입력 LLR 유한정밀도 양자화 `(q:f)` | `channel.cpp` `Set_LLR_Th()`, `Set_R_Offset()` + `ecc_data.h` `PARAM_LLR` | soft-read LLR 양자화 비트 설계에 참고, 단 채널 상이 |
| discretized DE로 양자화 threshold 산출 | 대응 없음 | Prime ECC는 bit-exact 시뮬, DE 해석툴 없음 |
| VN 처리/양자화 dynamic range | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | 양자화 비트폭·saturation 설계 대조용 |

적용 가치: **중간** — MSC의 correction항은 Prime ECC 미보유 기법이라 min-sum 저SNR 성능 개선 아이디어로 참고할 만하나, Rayleigh+M-ary 전제와 큰 LLR dynamic range가 NAND(AWGN/RBER, hard/2~3SD)와 정합되지 않아 직접 이식은 제한적.
