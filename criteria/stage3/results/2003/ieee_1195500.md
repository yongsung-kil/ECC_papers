### B. 알고리즘 요약

1. 시스템: 연결그래프 `G(V,E)`로부터 binary regular LDPC 부호를 구성, 예제 rate `0.845~0.945`, length `640~10429`, BPSK/AWGN + SPA 복호.
2. 문제: 기존 그래프기반 LDPC는 large girth를 요구했으나, 본 방법은 large girth 없이도 좋은 부호 구성.
3. 핵심 정의: 길이 `l` 경로집합 중 임의 두 경로가 disjoint이거나 한 정점에서만 교차하는 disjoint-crossing(DC) 제약 만족 집합을 사용.
4. 핵심기법: 경로들의 incidence matrix `H`(행=정점, 열=경로)를 만들면 각 열 weight=`l+1`, DC 제약상 두 열/행이 1을 최대 1개만 공유 → null space가 girth>=6 LDPC.
5. 식 의미: 각 행 weight 동일하면 regular LDPC, Massey 직교성 정리로 최소거리 >= `(행weight)+1` 보장(정정능력 하한).
6. 경로탐색: path trellis(`l` section)에서 extend-select-eliminate(ESE) 알고리즘으로 DC 제약 만족 survivor 경로집합 탐색, 노드별 우선순위로 균형 유지.
7. 확장(divide-and-conquer): 큰 complete graph를 subgraph로 분할, 각각 ESE 후 connecting trellis로 결합; `q+1`개 정점이면 shortened RS code(min-dist로 DC 보장)로 connecting path 직접 생성.
8. 예시: complete graph 100/200/400 정점 → (640,541)/(1287,1087)/(2738,2339) 부호; 582정점 divide-and-conquer + GF(97) RS → (10429,9852) rate 0.945 부호.
9. 결과: (2738,2339) rate 0.854 부호가 BER 10^-6에서 Shannon limit 1.5dB 이내; (10429,9852)는 1dB 이내, BER 10^-6 위로 error floor 없음.
10. 한계: HW 미설계, ESE 복잡도 정점수에 지수적(large graph는 divide-and-conquer 필요), QC-circulant 구조 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DC 경로집합 → H-matrix 구성(부호설계) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base+circulant QC-LDPC 고정, 일반 그래프 incidence matrix와 구조 상이(§4 높음) |
| girth>=6·최소거리 하한 보장 | 대응 없음 | Prime ECC H-matrix 구조 종속, 본 구성법의 직교성 보장을 그대로 이식 불가 |
| SPA(sum-product) 복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 근사 사용(§3 보유), full tanh SPA는 미대응 |

적용 가치: **낮음** — binary LDPC 구성법이나 QC-circulant 구조가 아니고 HW/NAND 타깃도 없어 고정 QC-LDPC Prime ECC 파이프라인에 이식 실익이 낮다.
