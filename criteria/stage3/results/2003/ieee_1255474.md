### B. 알고리즘 요약

1. 대상: 랜덤 LDPC의 복호기 HW 병목(복잡 인터커넥트·메모리 오버헤드)을 code/algorithm/architecture 3계층 동시 최적화. 예: rate-2/3 irregular LDPC `n=2304` (Ramanujan graph).
2. 문제: 랜덤 `H`는 병렬(인터커넥트 폭증)/직렬(메모리 4×Hamming weight) 양쪽 다 비효율, TPMP(2-phase)는 다중 check→bit 메시지 저장 필요.
3. 핵심1 (부호): AA-LDPC — `H`를 `S×S` pseudo-permutation(각 행·열 ≤1) 부분행렬로 분해 → 인터커넥트를 `S`배 축소, 라우팅을 작은 인터리버로 factoring, mux `S`개로 감소.
4. 핵심2 (알고리즘): TDMP — LDPC를 supercode 간 터보복호 문제로 변환, 인접 supercode 간에만 SISO 메시지 tandem 전달 → 다중 check-msg 저장 제거(메모리 ≥75%↓), 수렴 20~50% 빨라짐.
5. 핵심3 (연산): SPC 트렐리스에 BCJR forward/backward 재귀 적용, `f(x,y)` 비선형 함수를 metric-difference로 변환해 정규화 불필요.
6. max-quartet `Q(x,y)`: `f`/`q` 보정함수를 LUT 없이 signed-magnitude 논리게이트로 근사 → LUT-free, 이상적 대비 coding gain 손실 미미.
7. 양자화: 5-bit MPU가 6-bit Gallager 방정식보다 우수, 4-bit에서 1dB+ coding gain 개선(iteration/switching↓).
8. 아키텍처: 프로그래머블 TDMP 디코더 — column-wise 분산 메모리 모듈 + read/write transport network(block selector·permuter·column accumulator), 포인터 기반 갱신으로 데이터 이동↓.
9. 결과: 0.18μm 합성 200MHz, throughput 1.92 Gb/s(10 iter 시 192 Mb/s), 1176mW, 9.41mm²; TPMP 대비 power 89.13%↓·area 69.83%↓·interconnect 60.5%↓.
10. 한계: BCJR/트렐리스 기반 SISO 복호(터보식)로 min-sum과 패러다임 상이, GF(2) binary이나 부호가 Ramanujan/cyclotomic 특정 구성.
