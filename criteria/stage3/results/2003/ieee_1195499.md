### B. 알고리즘 요약

1. 시스템: binary-input/continuous-output AWGN + ISI 메모리 부분응답 채널(Dicode, EPR4), block length `10^6` bit, rate `0.5~0.9`.
2. 문제: memoryless 채널에서만 성립하던 capacity-approaching irregular LDPC 설계를 메모리 있는 부분응답 채널로 확장(미해결 문제).
3. 모델: 채널 법칙 `y = h*x + n` (h=채널 메모리 계수, polynomial form), i.u.d. capacity를 랜덤 선형부호 달성 상한으로 사용.
4. 핵심기법: Richardson-Shokrollahi-Urbanke의 memoryless degree sequence 최적화를 ISI용 밀도진화(density evolution)에 맞게 수정, variable/check degree 벡터 `λ, ρ` 최적화.
5. 식 의미: check→variable, variable→check 메시지 pdf를 convolution으로 진화(식 3,4), 오류확률을 `p0`→`ε`로 낮추는 근사 iteration 수를 최소화.
6. 확장: BCJR을 매 update마다 돌리지 않고 1회 BCJR 후 variable-to-check density update를 여러 번 반복하는 스케줄로 변형.
7. 트릭: linear programming으로 λ 최적화, ρ는 nonzero entry 소수로 제한해 실행시간 단축, 이산화 pdf의 수치정밀도 floor 대비 확률 limit `δ` 도입.
8. 최적화: Algorithm II - 초기 degree sequence에서 threshold σ* 증가가 멈출 때까지 반복, local 최적이라 다른 초기값으로 재시작.
9. 결과: noise threshold가 i.u.d. capacity의 0.15dB 이내, length `10^6`·rate 0.7 coset code가 BER `10^-6`를 capacity 0.22dB 이내 SNR에서 달성(10000 blocks 시뮬).
10. 한계: HW 미설계, 시뮬만, 부분응답/ISI 채널·BCJR 검출 전제, 점근적 최적성은 증명 없음(경험적).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| irregular degree sequence 최적화(부호설계) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 binary QC-LDPC H-matrix, degree 분포 재설계·재검증 부담 큼(§4 높음) |
| ISI 채널용 밀도진화 threshold 평가 | 대응 없음 | NAND는 memoryless read 채널, ISI/BCJR trellis 진화 없음 |
| BCJR + LDPC 반복 스케줄 | 대응 없음 | Prime ECC 복호는 min-sum message passing 단독, trellis 검출 없음 |

적용 가치: **낮음** — 부분응답(ISI 메모리) 채널 전용 irregular LDPC degree 최적화로 채널모델과 검출구조가 NAND memoryless binary QC-LDPC와 달라 이식 실익이 없다.
