### B. 알고리즘 요약

1. core-based(IP 재사용) 설계용 파라미터화 매크로셀(PMC) 방법론 - 소수 scaling parameter로 트랜지스터 크기를 아키텍처 수준에서 조정.
2. 문제: 기존 IP core는 application-specific이라 커스터마이즈가 어렵고, 커스터마이즈하면 재특성화가 불가피해 성능 예측이 어려움.
3. 1-D 레이아웃 전략(단일 P/N 트랜지스터 행 + 공유 power/ground rail)으로 back-to-back 스택 규칙성 확보, metal-1만 셀 내부 사용.
4. 핵심: PMC 하나에 1~3개 scaling parameter만 부여(예: 2's complementor는 `Q=(aR,aY)`), delay optimizer가 최적 파라미터 결정.
5. 1차 macro delay 모델 `delay = to(Q) + m(Q)*C(QL)` - 개별 셀·fanout만으로 저수준 트랜지스터 타이밍 없이 delay 예측(HSPICE 대비 오차 6% 이내).
6. PMC 라이브러리 구축(D-latch, full adder, MUX, shuffle-exchange switch 등) - Cadence + 0.18um 1.8V TSMC CMOS.
7. 응용 사례: 통신용 LDPC 디코더 서브시스템 합성 - BCJR 차분형의 compare-select+correction 커널 `Q(x,y)=max(x,y)-max(x+y,a)+...`.
8. SISO-MPU(soft-in soft-out 메시지처리, BCJR recursion 3식)와 message 전송용 Omega(shuffle-exchange) 네트워크를 PMC로 구현(parallel/serial).
9. 결과: Q-block 4-bit 임계경로 지연 `1.6ns`, 면적 `4864um2`; 서브시스템들 delay `1.6~10ns`, 면적 `40x100~380x200um2` (0.18um).
10. 한계: LDPC는 방법론 시연용 사례일 뿐 부호/디코딩 알고리즘 기여 없음; BCJR 기반 SISO 복호로 min-sum·NAND와 직접 무관.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|
| BCJR SISO-MPU 메시지처리 유닛 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | Prime은 min-sum, BCJR/SISO 대응 없음 |
| Q(x,y) compare-select 커널 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()` | CN 연산과 개념적 접점이나 알고리즘 다름 |
| Omega/shuffle-exchange 메시지 라우팅 | 대응 없음 | Prime은 GT(MUX edge)·z=32 라우팅으로 구조 상이 |
| PMC 레이아웃/delay 모델 CAD 방법론 | 대응 없음 | Prime은 bit-exact C++ 시뮬로 물리 레이아웃 계층 없음 |
