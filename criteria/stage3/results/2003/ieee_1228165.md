> 총평: cyclic-shift 기반 QC-LDPC 구성에 소규모 random permutation을 Kronecker로 더해 long code의 error-floor를 낮추는 설계법 — QC HW 친화적이나 random 성분이 Prime ECC의 순수 QC 구조와 충돌.

### B. 알고리즘 요약

1. 대상은 `Jp×Kp` 패리티검사행렬 `H=(m_{j,k})`로, 각 블록 `m_{j,k}`가 `p×p` cyclic shift 행렬 `c(l)`(항등행렬을 `l mod p`만큼 순환 시프트)인 QC-LDPC.
2. 풀려는 문제: cyclic shift만으로 짧은 cycle(특히 length-4)이 적은 부호를 쉽게 구성하고, 긴 부호에서 나타나는 error-floor를 낮추기.
3. 구성법: block을 순서대로 채우며, 후보 shift `l`이 만드는 부분행렬 `H_{j,k,l}`의 variable node 최단 cycle 길이 `g_{j,k,l}`이 최대가 되는 `l`을 greedy 선택(동률이면 random).
4. 이 selection으로 `p<100`인 임의 소수에서 몇 번 시도 내에 length-4 cycle 없는 `(3,k)`-regular 부호를 `k=p`까지 구성 성공.
5. 핵심 확장(Kronecker): `H=(c(l_{j,k}) ⊗ d_{j,k})`로, `d_{j,k}`는 `q×q` 소규모 random permutation. Kronecker product는 다시 permutation이 되어 QC 구조 유지.
6. 의미: cyclic shift는 commutative라 `(3,k)`-regular 부호가 임의 길이에서 weight-24 codeword를 가져 error-floor 유발; 작은 random `d_{j,k}`가 이 대칭을 깨 floor 제거.
7. 결과: `(3,6)`-regular 부호에서 (2)식 `p=5000` 부호는 floor가 존재하나, (3)식 `p=500, q=10` Kronecker 부호는 관측 FER 범위에서 floor 없음.
8. HW 친화성: 구조가 partly-parallel message-passing decoder[4]에 적합 — `c(l_{j,k})`는 VN↔CN 간 shuffle network, `d_{j,k}`는 message memory의 address generator에 대응. cyclic shift라 HW 복잡도 낮음.
9. array code 대비 약간 우수한 BER/FER 성능(Fig.1).
10. 한계: 시뮬 결과뿐(HW 미구현), 구체 rate/length·gatecount 없음, error-floor 개선을 위해 순수 QC에 random 성분을 도입해야 함.
