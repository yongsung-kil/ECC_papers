### B. 알고리즘 요약

1. 대상: LDPC convolutional code(LDPC block code의 convolutional 대응). 연속(continuous) 복호 가능이 잠재 이점.
2. 문제: 기존 LDPC convolutional 구성[1][2]은 약한 irregular 그래프뿐. block 부호처럼 irregular 차수로 threshold 크게 개선 가능([3]).
3. 부호 표현: time-invariant, parity-check 행렬 `H(D)_{j,k}` (`D`는 지연연산자)로 기술.
4. 핵심 기법: 먼저 이진 `j×k` 행렬 `P` 설계 → `P`의 '1'을 단항식 `D^a`(a≥0)로 치환, '0'은 유지하여 `H(D)` 획득. 변수/제약 노드 차수 = `P`의 열/행 무게.
5. 링 해석: `H(D)` 원소를 `GF2(D)/(D^m-1)`의 원소로 보면 길이 `km`의 QC block code가 됨.
6. 최적화: 특정 `P`가 그래프 edge 연결을 결정. `P` 고정 후 단항식을 랜덤 선택한 부호 앙상블 고려, `D`의 최대차수→∞ 극한 성능이 threshold. threshold 좋은 `P` 선택.
7. girth 규칙: `P`에 0 없는 2×3 또는 3×2 부분행렬이 있으면 단항식 선택 무관하게 항상 length-12 cycle 발생. 따라서 girth 최대 12를 주는 단항식 선택이 유리.
8. 예제: `H(D)` 크기 (a) 4×8, (b) 5×10, 최대 `D` 차수 200. 차수분포 (a) λ4=3/8 λ2=3/8 λ3=2/8, ρ5=2/4 ρ2=2/4 등.
9. 결과: irregular 부호가 저 SNR에서 regular (3,6) LDPC convolutional 대비 상당히 우수(threshold 개선). 5×10이 4×8보다 threshold 좋음 → 더 큰 행렬로 capacity 근접 가능성.
10. 한계: (b) 5×10에서 error floor 높음 — threshold 좋은 분포가 거리(distance) 특성은 나쁠 수 있음. 디코더/HW 없음, rate 1/2.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| irregular LDPC convolutional 부호 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 block QC-LDPC 고정, convolutional 부호형식과 정합 안 됨 |
| `H(D)` girth-12 최적화(threshold) | 대응 없음 | Prime ECC 부호는 확정, convolutional threshold 설계 미사용 |
| BP 복호(연속 복호) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 block 단위 min-sum 반복복호, 연속 sliding-window 복호 아님 |

적용 가치: **낮음** — LDPC convolutional(SC-LDPC 계열) 구성법으로 threshold(waterfall)는 개선하나, Prime ECC의 block 고rate binary QC-LDPC 구조 및 block 반복복호와 부호형식이 근본적으로 달라 이식이 어렵다.
