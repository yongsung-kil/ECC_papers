### B. 알고리즘 요약

1. 대상 부호: array code 기반 binary quasi-cyclic LDPC(ALDPC), 파라미터 `(p, i, j)` (`p`=홀수 소수), PCM은 single-cyclic-shift 행렬 블록으로 구성(식 1).
2. 구조: column weight `i`, row weight `j`, 4-cycle free(girth≥6), 부호 length `jp`, dimension `jp-ip+p-1` → quasi-cyclic이라 shift-register로 선형시간 인코딩 가능.
3. 동기: random LDPC는 인코더 메모리·행렬곱 복잡도가 큼 → cyclic/quasi-cyclic 대수적 LDPC로 인코딩 부담 완화, array code가 좋은 성능 후보.
4. 문제: array code는 대수적 hard-decision 복호로 erasure 정정력이 뛰어남 → BP(soft) 복호에서도 error·erasure 정정이 유사하게 좋은지 조사.
5. 시스템 모델: RLL 후 LDPC, EPR4 등화 MRC, log-MAP(채널)+BP(LDPC) 두 APP 디코더, ALDPC엔 random interleaver 사용.
6. Erasure 모델: TA/media defect로 인한 full/partial erasure를 rectangular window(fading depth `α`, erasure length `L`)로 모델, deep fade 시 CSI 있으면 해당 LLR을 0으로 clear.
7. 비교 부호: ALDPC (4572,4067) `(127,4,36)`, random binary (4608,4096), q-ary (1152,1024) — 유사 length/rate. binary는 log-BP, q-ary는 FFT-BP, 최대 50 iteration.
8. 채널: Lorentzian(전자 AWGN) 또는 Lorentzian-Gaussian(10% AWGN + 90% jitter media noise), 정규화 밀도 고정.
9. 결과: q-ary가 긴 erasure(`L≥256`)에서 FER 기준 0.2~0.25 dB 우수하나, 그 외에선 차이 없음. ALDPC(+random interleaver)는 random binary와 동등하면서 q-ary 대비 복잡도 ~1/12, 메모리도 훨씬 적음 → 성능-복잡도 tradeoff 우수.
10. 한계: HW 미설계, 복호는 표준 log-BP, erasure 정정(자기기록 TA/MD) 특화 채널.
