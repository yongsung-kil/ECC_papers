> 총평: 무선 대역효율 목적의 LDPC convolutional(SC-LDPC)+multilevel modulation 부호화 논문 — NAND 고전 binary QC-LDPC 이식성 낮음.

### B. 알고리즘 요약

1. 대상: memory `M` LDPC convolutional 부호 두 개 `C1(M,J1,K1)`(rate `R1=b1/c`), `C2(M,J2,K2)`(rate `R2=b2/c`)를 합쳐 전체 rate `R=R1+R2=1`.
2. 문제: 기존 단일 rate-1/2 LDPC convolutional + Gray 매핑 대비 대역효율(bandwidth-efficient) 개선 필요.
3. 모델: QPSK, AWGN 채널. 정보열 `u`를 분할해 각각 `C1`,`C2`로 부호화한 뒤 신호점에 multilevel 매핑.
4. 핵심기법: multilevel coding/modulation (Imai-Hirakawa) + 결합 복조·복호(combined demodulation and decoding)로 신호 성좌 내 의존성 활용.
5. 복호: [1]의 iterative pipeline decoder(시변 주기 convolutional LDPC) 기반.
6. 대조군: 단일 rate-1/2 부호 `C'(M',J',K')` + Gray 매핑.
7. 분석: iterative limit `(Eb/N0)_0` 상한을 두 방식에 대해 계산 → multilevel이 낮음.
8. 결과 수치: Gray `(Eb/N0)_0≈1.1dB` vs multilevel `≈1.025dB`; SNR>0.9dB에서 multilevel이 우세(M'을 4배 키워도).
9. 예시부호: multilevel `C1(1019,3,4)`+`C2(1019,3,12)` vs 기존 `C'(1025,3,6)`, 반복 진행 시 BER에서 제안이 우세(Fig.1).
10. 한계: 시뮬/이론 threshold뿐, HW 미설계, QPSK 변조·무선 채널 특화로 NAND 저장채널과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC convolutional(SC-LDPC) 부호구성 | 대응 없음 | Prime ECC는 block QC-LDPC 고정, convolutional 구조 미지원 |
| multilevel coding/modulation | 대응 없음 | NAND ECC는 변조 없음, 대응 모듈 없음 |
| iterative pipeline 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | pipeline 반복복호 개념만 공통, convolutional 스케줄은 이식 불가 |

적용 가치: **낮음** — SC-LDPC convolutional + 다중레벨 변조는 무선 대역효율 목적으로 Prime ECC의 block binary QC-LDPC·저장채널과 정합되지 않고 이식 요소가 없다.
