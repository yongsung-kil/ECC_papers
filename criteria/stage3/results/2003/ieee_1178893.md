### B. 알고리즘 요약

1. 대상: binary irregular quasi-cyclic LDPC. PCM은 circulant 행렬들의 배열 (식 1), 하나가 invertible이면 systematic generator (식 2) 구성 -> shift-register로 선형 인코딩.
2. 문제: 최적화된 random irregular LDPC는 성능 좋지만 인코딩 복잡도가 길이의 제곱(matrix mult). 구조적(QC) irregular 부호로 인코딩을 선형화하면서 irregular 이득을 살리려는 것.
3. 핵심 도구: difference family - 그룹 `Z_v`를 크기 `k` 부분집합들로 배열해 모든 비영 원소가 차이로 정확히 `lambda`번 나타나게 함. 여기선 `lambda=1` family 사용.
4. 핵심 기법(Construction 1): `(v,k,1)` difference family의 각 set를 circulant 열로 매핑. regular는 set 전체 원소, irregular는 set 원소 일부만 임의 선택(단일 set로 서로 다른 원소로 두 circulant 구성 가능).
5. 열가중 분포 벡터 `W=[w_1,...]`로 비정칙성 부여, 행가중치는 상수(식 4). 최소 한 circulant가 invertible하도록 원소 선택.
6. 4-cycle-free 보증(Lemma 2): difference family에서 두 원소는 translate 전체에서 정확히 `lambda=1`번만 함께 나타나므로 두 열이 같은 두 행에서 겹칠 수 없음 -> 4-cycle 없음. irregular는 원소 제거일 뿐이라 4-cycle 추가 안됨.
7. 인코딩 복잡도: QC 부호는 code dimension 크기 shift-register 회로, `(rho-1)` 규모 binary 연산 (`rho`=행가중치) vs random의 matrix mult (제곱).
8. 복호: sum-product algorithm, max iteration 50. (양자화/HW 관련 서술 없음.)
9. 결과: (101,5,1) difference family로 rate-3/4[404,303], 4/5[505,404], 5/6[606,505], 6/7[707,606] 4개 부호 구성. random 및 regular QC 대비 비교(Fig.2-3): regular QC 대비 modest 이득, 짧은 길이·고rate에서 random 대비 개선. 단, 최적화된 random irregular는 능가 못함(대신 인코딩 복잡도 낮음).
10. 한계: 짧은 길이 예제뿐, HW 미설계, difference family 존재성에 부호 파라미터가 제약됨.
