### B. 알고리즘 요약

1. 시스템: RAID 서버용 2Gbps 광섬유 채널의 FEC로 LDPC를 적용, BER < 1e-12 요구. AWGN 채널, rate 1/2 `(3,k) regular` LDPC, N~1000~2040.
2. 문제: serial 디코더는 latency가 블록길이 `N`에 비례해 고throughput 불가, 완전 parallel은 gate count 과다. 둘 사이 절충 필요.
3. 모델: soft-input iterative message passing, LLR `L=log(Pr(c=1)/Pr(c=0))`, log-MAP 알고리즘.
4. 핵심기법1(serial): bit-node당 1클록 처리, 3 edge 병렬. `Psi(x)=log(tanh(|x/2|))` 회로로 check-node parity 합산, FIFO로 Psi값 저장 후 subtract로 bit update. LLR 4bit, 20 iteration.
5. 핵심식: parity check는 연결된 bit-node들의 `Psi` 합, 자기 정보 제외(subtraction)로 extrinsic 산출 — 이것이 tanh-domain SPA 구현.
6. 핵심기법2(parallel): 모든 bit-node 동시 처리, 3*N개 Psi/Psi^-1 회로+레지스터, 1클록/iteration으로 최고 throughput(10Gbps@100MHz) 그러나 gate 2.69M.
7. 확장기법(scalable): 조합논리(adder/subtractor/Psi)를 병렬화계수 `M`배 스케일, 메모리를 3*M 블록으로 분할, switch logic으로 라우팅. latency = `N/M`.
8. 트릭: M-way 메모리 분할을 BER 저하 없이 하기 위해 code construction에 parallelizing constraint 추가.
9. 결과: TSMC 0.35um 합성, M=170 확장형이 2Gbps 사양을 최소 gate(1.29M)로 충족, 2.51Gbps@295MHz. M=340은 4.62Gbps/1.77M gate.
10. 한계: BER 곡선/error-floor 분석 없음(HW 트레이드오프 중심), log-MAP full-tanh 기반, 광통신 특화, NAND soft-read 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 반복 message passing 디코더 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 알고리즘 골격은 대응하나 Prime은 min-sum, 본 논문은 full-tanh log-MAP이라 CN 연산 상이 |
| Psi(=tanh-domain) CN 연산 | 대응 없음 | Prime은 `CNU_Update_New_Mag()` min-sum 근사 사용, tanh full-SPA 미사용 |
| M-way 병렬화/메모리 분할/switch logic | 디코더 HW 모델(z=32, GT) | Prime은 row-단위 z=32 병렬+Graph Thinning 구조로 접근이 다름, 라우팅 개념만 유사 |
| LLR 4bit 초기화 테이블 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | 개념적 대응, 그러나 AWGN 광채널 기준이라 NAND soft-read와 무관 |
