### B. 알고리즘 요약

1. 대상은 AWGN 채널의 여러 known regular LDPC 부호에 대한 양자화 sum-product(Q-SP) 반복복호, 4-bit LLR 표현.
2. 문제: 제한된 비트수로 LLR 메시지를 어떻게 최적 표현하여 BER·평균 반복수·error floor를 개선할지.
3. 핵심 파라미터는 양자화의 step size와 largest level 두 가지; step size가 임계값 `2log(d-1)`(d=최대 체크 차수)를 넘으면 Q-SP가 부호 무관하게 quantized min-sum(Q-MS)과 등가가 됨.
4. 부호 의존성 제거를 위해 변수·체크 노드 pairwise updating rule 사용 → 임계값이 상수 `2log2`로 고정, regular/irregular 동일 알고리즘·저복잡도 복호 가능.
5. largest level이 작으면 양자화 overflow가 discrete density evolution을 훼손해 error floor 유발 → 경험적으로 largest level ≥ 15 필요, 균일 양자화 시 Q-BP가 Q-MS로 열화되지 않으려면 최소 5비트 필요.
6. 비트수<5에서는 비균일(nonuniform) 양자화만이 step size와 largest level 요구를 절충; 채널 출력 정보를 담은 초기 LLR은 더 충실히, 작은 LLR 영역은 더 세밀히 양자화.
7. 방법1(segmental): 대칭 양자화 `Q(x)`, 레벨 집합 `{X_k}`, pairwise rule `R_c`; `(T_{L-1}, K, X_K)` 선택 후 세 구간에 나머지 레벨 균등 배치, `T_0=0` 및 `Q(R_c(l,l))<|X_k|` 보장.
8. 4-bit Q-SP는 `L=7, (15,3,2.6)`이 거의 모든 known regular 부호에 최적.
9. 방법2(A-law): 점증 step size가 density evolution에 더 잘 부합; 정규화 입력 20·`A=6.5`가 threshold 기준 최적, BER·평균 반복수·error floor에서 segmental 대비 실질 최우수.
10. 결과: max iteration 200에서 4-bit threshold를 unquantized SP(`σ_SP`)·균일(`σ_u`)·두 비균일과 Table I로 비교(구체 수치는 표에만 있어 본문 미기재), 한계는 HW 미설계·AWGN/regular 한정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LLR 비균일 양자화 테이블(segmental/A-law) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 작은 LLR 영역 세밀화 아이디어를 Prime ECC LLR 테이블 설계에 참고 가능 |
| VN/CN 양자화 메시지 처리 | decoder.cpp VN_Cal_HD() 등, CNU_Update_New_Mag() | 4-bit 양자화 메시지 처리와 대응, 단 Q-SP↔min-sum 차이 주의 |
| soft-read LLR threshold | channel.cpp Set_LLR_Th() | 양자화 threshold/largest level 설계가 soft-read LLR 설정에 참고 |
| pairwise updating rule로 부호 의존성 제거 | 대응 없음 | Prime ECC는 고정 QC-LDPC·min-sum, pairwise rule 자체는 미사용 |
```

적용 가치: **중간** — 작은 LLR 영역을 더 세밀히 하는 비균일(A-law) 양자화 설계 아이디어는 Prime ECC의 LLR 테이블(`PARAM_LLR`)·soft-read threshold 튜닝에 참고 가치가 있으나, 대상이 Q-SP(full sum-product)·AWGN·regular 부호이고 Prime ECC는 min-sum·2~3bit soft라 직접 이식보다 설계 지침 수준.
