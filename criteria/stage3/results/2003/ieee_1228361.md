### B. 알고리즘 요약

1. 대상: LDPC 일반화 - variable/check 노드가 repetition/SPC보다 일반적인 작은 부호(atomic code `C_i(n_i,k_i)`)를 나타내는 "molecular code"(deep-space, JPL).
2. 구조: 각 atom은 network-side `b_i` bonded edge, channel-side `c_i` charged edge를 가짐; bond가 atom쌍을 잇고 원래 `n_i-k_i` 제약 위에 1개 제약 추가.
3. 길이/rate: 최소차원 `K=Σk_i - B`(`B`=총 bond 수), 길이 `N=Σc_i`, 최소 rate `R=K/N=(F̄-ρ/2)/γ` (`F̄`=가중평균 atom rate, `ρ`=bonded 심볼 비율, `γ`=charged 심볼 비율).
4. rate 설계: `γ<2F̄`이면 채널 연결 심볼 비율 `γ` 조정으로 임의 `0<R<1` 설계 가능.
5. 설계 예: shortened Hamming `H(6,3)` q개 + SPC(3,2) 2q개를 랜덤 이분그래프로 연결해 rate 1/6 부호 2종 구성(Hamming/SPC 중 한쪽 fully charge, 두 부호는 등가).
6. 관계: Tanner의 그래프 부호 구성을 양측 정점 모두 임의 소부호로 일반화(Barg-Zemor expander code 계열).
7. 복호: bonded atom 간 extrinsic APP 교환(LDPC와 동일 방식), 각 atom은 자기 trellis에서 BCJR로 parity 제약 결합 계산.
8. 성능: Fig.2 부호의 iterative 복호 threshold를 Gaussian density evolution으로 산출 → AWGN에서 `Eb/No ≈ 0.2 dB`.
9. 결과: (5,6) regular LDPC threshold보다 크게 우수하나 rate-1/6 capacity 한계보다 여전히 1dB 이상 높음; irregular molecular 구성으로 gap 축소 여지.
10. 한계: 순수 이론(DE threshold)·HW 미설계, 저rate deep-space 대상, BCJR trellis 복호 요구.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| atom별 BCJR trellis APP 복호 | 대응 없음 | Prime ECC는 SPC-노드 min-sum(`decoder.cpp`), 일반부호 trellis/BCJR 미지원 |
| molecular code 부호설계(atom 네트워크) | 대응 없음 | 특정 binary QC-LDPC H-matrix 고정, generalized-LDPC 구성 파이프라인 없음 |
| rate 설계(charged 심볼 비율 조정) | `ecc_top.cpp` `Load_PCM()` (개념상 근접) | rate 설계 접근이 다름(QC circulant 대 atom charge) |

적용 가치: **낮음** — 노드를 임의 소부호로 일반화한 generalized-LDPC 구성·BCJR 복호로, min-sum 기반 binary QC-LDPC NAND ECC에 이식 접점 거의 없음.
