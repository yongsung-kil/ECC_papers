### B. 알고리즘 요약

1. 대상: column weight `j=2`, 큰 girth(`g=16`, `g=20`)를 가지는 규칙적 LDPC 부호 두 부류를 그래프 모델 기반으로 구성.
2. 문제: 짧은 cycle이 sum-product 반복복호에서 extrinsic 정보 독립성을 해쳐 성능 저하 → 큰 girth 부호가 필요하나 규칙적·구조적 구성이 어려움.
3. 표현: `j=2`인 H의 각 column을 두 점 사이 edge로 보고 "structure graph" 정의 → K-cycle = K/2 점과 K/2 edge의 loop로 cycle 식별 단순화.
4. cylinder 구조: `v=8p`(g=16) 또는 `v=10p`(g=20) 점을 8/10 subset으로 나눠 인접 subset끼리만 연결, 각 subset 간 edge 묶음을 section `S_i`로 정의.
5. slope/ASP: edge의 slope `s=j-i`, admissible slope pair `(s_i,s_j)`로 각 point degree를 균등 증가시킴. slope 0은 특수 ASP `(0,0)`.
6. cycle 회피 조건: ASP `(s,s')`와 `(0,0)`이 만드는 최소 cycle 길이 `L=4[lcm(|s|,|s'|)/|s| + lcm(|s|,|s'|)/|s'|]`가 girth 미만이면 배제.
7. 탐색 알고리즘: 각 section마다 reference point에서 first m step 경로를 전개, 두 경로가 만나면(=cycle) 해당 ASP 폐기, 인접 section ASP 제약 하에 순차 선택(Construction 1/2).
8. 최소 p: g=16은 `p≥23`(→`v≥184`, `n≥368`), g=20은 `p≥13`(→`v≥130`)에서 성공 구성 존재.
9. rate: g=16은 row weight 4, `T=(k-j)/k=1/2`; g=20은 row weight 3, `T=1/3`. 최소거리 `d_min=g/2`(=8, 10).
10. 결과: AWGN·sum-product에서 high-SNR 시 random 대비 g=16이 0.8 dB, g=20이 1.1 dB(BER=1e-5) 우위. 단 low-SNR에서는 random보다 열세, 원인은 girth 길이 cycle 분포로 추정. HW 미설계.
