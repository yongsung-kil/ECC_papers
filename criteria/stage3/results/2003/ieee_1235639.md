> 총평: LDPC 디코더의 메모리·interconnect 병목을 code-design/알고리즘/아키텍처/레이아웃 4계층에서 공동 최적화한 landmark HW 방법론으로, TDMP의 iteration 감소·메시지 메모리 절감은 NAND QC-LDPC 디코더에 참고가치 큼.

### B. 알고리즘 요약

1. 문제: 기존 LDPC 디코더는 random H의 interconnect 랜덤성 때문에 parallel 구현은 wire·area가 code length에 불리하게 스케일(`N(n)=24n`), serial 구현은 edge 4배 메모리·throughput 병목.
2. 목표: code-design/디코딩 알고리즘/디코더 아키텍처/물리 레이아웃 4계층 공동 최적화로 memory overhead·interconnect 복잡도 해소, code-programmable 코어 생성.
3. AA-LDPC 부호: 길이 `n=BS`, block PCM을 `S×S` all-zero 또는 permutation 행렬 블록으로 구성 → `[D,B,S,c,r]-AA-LDPC`. 블록행 내 1들이 non-overlap이라 독립 처리 가능, rate `R≥1-D/B`.
4. 구조적 이점: 인접 블록행 간에만 메시지 전달 → interconnect가 `rSlog(S)` (2:1)-MUX로 축소(랜덤 대비 `O(n)`→`O(log n)` 제어 오버헤드).
5. TDMP 알고리즘: 각 블록행을 constituent code로 보고 `S`개 MPU가 병렬로 SISO 복호하는 turbo-decoding message-passing, 블록행별 sub-iteration 순차 진행 후 posterior `γ`로 hard decision.
6. TDMP 이점: check-to-bit 다중 메시지 저장을 최신 1개로 대체(`75%` 메모리 절감), 수렴이 빨라 동일 SNR에서 TPMP 대비 iteration `20~50%` 감소.
7. 저복잡도 메시지 연산: Gallager 식 대신 syndrome-trellis BCJR를 근사한 "max-quartet" 이변수 함수 `Q(x,y)=max(x,y)-max(x+y,0)`로 LUT 제거, 양자화 잡음 내성.
8. 디코더 아키텍처: `S` MPU + 분산 메모리(λ/γ/H/π-Memory) + shuffle-exchange(Omega) 라우팅 `R,R^-1` (`S/2·log(S)` 2×2 switch), sub-iteration당 `(r+4)` cycle, iteration당 `(r+4)D` cycle.
9. 레이아웃: parameterized macro-cell(PMC/PLC) 라이브러리로 transistor sizing·power-rail scaling을 파라미터로 core-optimizer가 power-delay-area 최적화.
10. 구현 결과: rate-1/2, length 2048, `[16,32,64,6]-AA-LDPC`, 4-bit datapath, 0.18µm 1.8V CMOS → 14mm² (기존 [5] 대비 3.75배 소형), 1.6 Gb/s @125MHz, 760mW; 전력 memory 53.9%/MPU 37.7%/R-net 8.4%.
