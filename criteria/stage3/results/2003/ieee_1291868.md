### B. 알고리즘 요약

1. 대상: **cycle code** = column weight `j=2` LDPC (예제 (8,3), (200,2,8)). PCM `H`는 sparse지만 생성행렬 `G`는 dense.
2. 문제: 기존 `Z=GS` 인코딩은 `O(n^2)` 복잡도로 cycle code 응용을 가로막음. `O(n)` 인코더 달성이 목표.
3. 기법1 "label-and-decide": Tanner graph에서 bit node를 정보비트/패리티비트로 라벨링하며, 패리티비트로 라벨되는 순간 그 값을 parity-check 식으로 즉시 계산.
4. 개념 pseudo-tree: `2P+1` tier로 구성(홀수 tier=bit node, 짝수 tier=check node), 각 check node가 상위 tier에 유일한 parent bit(조건 iii)를 갖는 계층 Tanner graph.
5. Lemma 1: 임의 pseudo-tree는 선형시간 인코딩 가능. parent를 패리티비트로 잡고 하위 tier부터 tier별로 패리티 값 계산.
6. Lemma 2: 임의 cycle code는 pseudo-tree들의 **합집합**으로 표현 가능. 연결 부분그래프 `Gi`마다 종속 행 1개 제거 후 tier 구조로 재배치, 조건 iii 위배 노드는 하위 tier로 드래그.
7. 비용: `l`번째 parity-check 식(비트수 `kl`)당 `(kl-2)` XOR. regular `(n,2,k)` code 전체 인코딩은 약 `2n` XOR → 선형.
8. Theorem 1: 임의 cycle code는 선형시간 인코딩 가능(Lemma1+2 결합).
9. Corollary 1: cycle code의 `H`는 행·열 순열만으로 upper-triangular `H_upper`로 변환 가능 → causal system으로 해석.
10. 한계: **순수 인코딩 이론**(HW·시뮬·성능수치 없음), j=2 cycle code는 거리특성이 j≥3보다 나빠 RS 연접 등 특수 용도 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| cycle code 선형 인코딩(label-and-decide) | `encoder.cpp` `LDPC_encoder_4KB()` | 우리는 dual-diagonal 인코딩 고정, 부호 클래스(j=2)도 달라 대체 유인 없음 |
| PCM 순열→upper-triangular | `encoder.cpp` `Generate_PCM_encoding()` | 개념만 겹칠 뿐 QC-LDPC 구조와 무관 |
| 부호 구조/H 로드 | `ecc_top.cpp` `Load_PCM()` | j=2 cycle code는 우리 고rate QC-LDPC와 클래스 상이 |

적용 가치: **낮음** — j=2 cycle code 전용 인코딩 그래프 이론으로, 이미 dual-diagonal 선형 인코딩을 갖춘 고rate QC-LDPC(VN degree 17) Prime ECC에는 부호 클래스·인코딩 방식 모두 맞지 않음.
