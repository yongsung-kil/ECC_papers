### B. 알고리즘 요약

1. 시스템: regular LDPC(`row weight wRow=8`, `col weight wCol=4`), rate 1/2, code size 8K, AWGN 가정, soft-decision SPA(belief propagation).
2. 문제: 대형 LDPC의 near-Shannon 성능을 위해 확장 가능(scalable)하고 저지연인 FPGA SPA 디코더 아키텍처가 필요.
3. 알고리즘: SPA 4단계 - init(채널 posterior `f_n(1)`), horizontal step(`δq_mn`→`δr_mn` 행 곱), vertical step(`r_mn`→`q_mn` 열 곱), decoding check(`ĉ_n H^T=0`).
4. 핵심 식: horizontal은 `δr_mn = ∏_{n'≠n} δq_mn'`(식5), vertical은 `q_mn = k·f_n·∏_{m'≠m} r_m'n`(식8/9), codeword는 `q_n(1)>0.5` 임계.
5. 메모리 모델: nonzero 원소만 (row-idx, col-idx, value)로 저장, 총 저장량 = 4×(H nonzero 수) + 2×code length.
6. HW 매핑: figure1(horizontal)의 `wc` 메모리 블록 수=row weight, figure2(vertical)의 `wr` 메모리 블록 수=col weight로 두어 병렬 곱셈기 배열 구성.
7. 병렬화 구조: 곱셈기 배열은 n입력에서 n-1개씩 취해 n출력(binomial, k=n-1)을 내는 구조, `ĉ_n H^T` modulo-2는 32 섹션 병렬 up-counter로.
8. 타이밍 식: `t_Col=N_Col/f_Mem`, `t_Row=N_Row/f_Mem`, `t_Est=N_Row·wRow/(K·f_Mem)` (packing factor K).
9. 예제 결과(Virtex-7 XC7V485T, 400MHz BRAM, K=32): `t_Col=20.48µs`, `t_Row=10.48µs`, `t_Est=2.56µs`, wc=16 mult/wr=6 mult, 평균 100 iter·수렴 0.01s 가정 시 throughput ≈400kbps, 자원 Logic 51.9%/BRAM 63.0%/DSP48 56.0%.
10. 한계: 64비트 double precision 부동소수 연산(HW 비효율), min-sum 미적용, 매우 낮은 throughput(1차 설계값), BER 성능 시뮬 미제시, 비QC regular 코드.
