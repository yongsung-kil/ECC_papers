### B. 알고리즘 요약

1. 대상: iterative sparse-graph 부호(inner LDPC 또는 TPC)의 error-floor를 outer block code 직렬연접(SCC)으로 낮추는 시스템레벨 FEC, 100Gb/s OTN 광통신 타깃.
2. 풀려는 문제: 기존 SCC는 error-floor 제거에 길고 복잡한 outer code+긴 interleaver가 필요해 고속 시스템 구현이 어려움(복잡도·latency).
3. 핵심 모델: inner 디코더 출력 error-pattern 집합 `Ω`, 그 중 error-floor 유발 subset `Ω0`, `wmax = max w(ω)`를 outer code 정정능력 설계 기준으로 사용.
4. 핵심 기법: 긴 outer code+interleaver를 짧은 outer block code 2개(`C2`, `C3`)+새 encoding/decoding으로 대체 — `C3`는 각 dataword에 parity 부여, `C2`는 parity 블록들을 묶어 보호.
5. 디코딩: inner `C1` 복호 후 uncorrectable codeword 검출→오류없는 dataword에서 `P3i` 재생성→`C2` erasure decoding으로 손상 codeword parity 복원→`C3` 재복호(erasure 기반이라 저복잡).
6. 복잡도 이득: `C2`/`C3` 디코더 throughput이 `(τ/m)T`로 감소 → SCC-II 대비 약 1자리수(5~6.7배) 게이트 절감.
7. 확장 1(τ=1 최적화): outer `C2`를 SPC로 대체, `C2` 디코더를 `n3-k3`개 XOR+FF 재귀구현(`Φ(C2^d)=23·(n3-k3) gates`), latency·buffer(`3mk1→2mk1`) 감소.
8. 확장 2(일반화 SCC): `C3`의 `g`개 parity로 inner 잔여오류 codeword를 검출, minimum-distance codeword(undetectable)까지 커버해 TPC에 적용.
9. 결과: Margulis [2640,1320] LDPC error-floor를 outer로 저감; TPC-EH(8192b, dmin16)+SCC로 NCG ~11.2dB @ BER1e-15, error-floor ~7e-17, 22% OH; 복잡도 107Kgates@28nm(SCC-II 575K 대비 5.38배↓).
10. 한계: HW 미설계(게이트 추정만, 시뮬), 타깃이 고속 optical(고rate long-block, MDS RS/BCH outer), wmax/trapping-set 사전 분석 필요, inner LDPC 디코더 자체는 개선 안 함.

| outer RS/BCH/SPC 직렬연접 | 대응 없음 | Prime ECC는 QC-LDPC 단일 부호 엔진, outer block code 연접 계층 없음 |
