### B. 알고리즘 요약

1. 시스템: BPSK+AWGN 채널, 구조화 binary LDPC(`Code1 [255,175] GF(16) 2D cyclic`, `Code2 [529,374] array QC`)를 BF 계열로 복호.
2. 문제: GDBF는 랜덤 LDPC엔 좋으나 구조화 LDPC는 column weight γ가 커(예: 16) 에너지함수 두 항의 값 범위 불균형으로 성능 급락.
3. 모델: 수신 `rk=yk+nk`, 판정 `zk∈{+1,-1}`, 신드롬 `sj=∏zi`, 목적함수 `f(z)=Σrizi+Σsj` 최대화(ML 근사).
4. 핵심기법1: 비트별 에너지함수 `Ek=rkzk+Σ_{j∈M(k)} sj`에서 최솟값 비트 `i*=argmin Ek`를 플립.
5. 핵심식 의미: 두번째 항 값범위 `[-γ,γ]`가 첫항(-3,3)보다 훨씬 커 첫항 기여가 묻힘 → 불균형 해소가 관건.
6. 핵심기법2: 기존 WGDBF의 `ωj=min|ri|`(ρ-1회 비교 필요)를 버리고, 단순히 `1/γ` 정규화 `Ek=rkzk+(1/γ)Σsj`로 값범위를 `[-1,1]`로 맞춤 → 비교연산 제거.
7. 부수트릭: 반복 중 각 비트 플립횟수 `Fk`를 세어 `Ek=rkzk+(1/γ)Σsj+Fk`에 더해 연속 플립·저Hamming weight 흡수집(absorbing set) 정체를 억제.
8. 최적화: 별도 학습 없음. 최대 반복 `Imax=100`, 신드롬 all-one 시 조기종료.
9. 결과: Code1 BER `1e-4`에서 GDBF 대비 약 0.6dB, WGDBF 대비 약 0.4dB 이득, SPA와는 약 0.7dB 격차. Code2 BER `1e-3`에서 GDBF 대비 약 0.5dB. ANI(평균반복수)도 최저 → 수렴 최속.
10. 한계: 시뮬만(HW 미설계, gatecount/throughput 없음), 대상 부호가 우리 고rate QC-LDPC와 rate·column weight 상이, soft-read 비트수 명시 없음(GDBF는 채널 실수값 사용).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBF/IGDBF 반복 복호 루프 | decoder.cpp LDPC_Decoder() | BF 계열은 min-sum과 다른 복호 패러다임이나 이터레이션 루프 위치는 동일 |
| 에너지함수 1/γ 정규화 + Fk 플립카운트 | decoder.cpp C2V_Cal() (min-sum 자리) | 우리는 min-sum 사용, BF 에너지항으로 교체 시 CN/VN 연산 전면 대체 필요 |
| 조기종료(신드롬 all-one) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 우리는 CRC 기반, 논문은 신드롬 체크 → 개념 유사, 구현 상이 |
| soft 수신값 rk 사용 | channel.cpp Set_LLR_Th() | GDBF는 채널 실수값 직접 사용, 우리 LLR 양자화 테이블과 정합 필요 |
```

적용 가치: **낮음** — Prime ECC는 min-sum BP 계열 디코더이고 본 논문은 BF 계열 복호 전면 대체를 요구하므로, 부분 이식보다 별도 디코더 파이프라인 재설계가 필요하다.
