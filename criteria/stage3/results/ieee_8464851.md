### B. 알고리즘 요약

1. 시스템: binary Slepian-Wolf source coding — source X를 syndrome `s=H·x`로 전송, 디코더는 side info Y와 syndrome으로 BP 복원. BSC 상관채널, 부호길이 짧음(192/256).
2. 문제: 기존 LDPCA rate-adaptive 구성은 accumulator 구조 고정이라 저rate에서 short cycle 다량 생성·일부 source bit(VN) CN 제약에서 소멸 → 유한길이 성능 열화.
3. 가정: mother code `H1`(rate 1/2)에서 CN들을 결합해 낮은 rate 딸행렬 `H2=H1→2·H1` 생성, `H1→2` full-rank면 rate-adaptive 조건 만족.
4. 핵심기법: 중간행렬 `H1→2`의 연결을 설계 — 각 `s_i∈S`는 U의 CN 하나에만, `u_j∈U`는 낮은 degree `d`로 연결하여 결합 CN 선택.
5. 식 의미: `H2=H1→2·H1`은 여러 CN을 XOR 결합해 rate를 `m2/n<1/2`로 낮추는 연산이며, 결합 선택이 H2의 cycle/연결성을 좌우.
6. 확장: 여러 rate `R_I<…<R2<R1`를 `Hi=Hi-1→i·Hi-1` 재귀로 생성.
7. 구현 디테일: `u_j` 구성 시 (1)공통 VN 없는 CN만 선택해 VN 소멸 회피, (2)T개 후보 랜덤 시도 후 length-4/6/8 cycle 수를 최소화하는 조합 선택(Algorithm 1).
8. 학습/최적화: density evolution 미사용(유한길이 집중), degree `d_k`는 가능한 작게, cycle 카운팅은 [18] 기법 사용.
9. 결과: C1(128×256), C2(96×192, WiMax protograph) rate 3/8·1/4·1/8에서 LDPCA 대비 BER 최대 1자릿수 개선(rate 3/8 C1만 예외), N4 cycle 항상 더 적음.
10. 한계: HW 미설계, source coding 전용, 채널부호 정정능력 논의 없음, N6 cycle은 오히려 증가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| rate-adaptive H-matrix 구성(`H2=H1→2·H1`) | `ecc_top.cpp` `Load_PCM()` | Prime은 고정 고rate QC-LDPC H-matrix 로드 — rate 낮추기 결합 구조는 목적 불일치 |
| short cycle(4/6/8) 최소화 CN 선택 | 대응 없음 | QC 리프팅/base matrix 설계 도구 아님(source-coding CN 결합) |
| syndrome 기반 BP 복호 | 대응 없음 | Prime은 codeword BP(min-sum), syndrome source-decoding 아님 |

적용 가치: **낮음** — Slepian-Wolf source coding용 rate-낮추기 구성법으로 채널부호가 아니며, NAND 고rate binary QC-LDPC의 부호설계(고정 QC 구조)·min-sum 디코더와 목적·채널·rate 방향이 근본적으로 달라 이식 가치 낮음.
