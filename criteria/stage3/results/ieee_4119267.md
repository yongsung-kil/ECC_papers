### B. 알고리즘 요약

1. 대상 코드는 accumulator 기반 직렬연접부호(반복/확장 Hamming + rate-1 accumulator, rate 0.8125 등)와 LDPC 코드(rate-1/2 Margulis (2640,1320), rate-0.8 (2000,1600) irregular LDPC)로 AWGN·BPSK 채널.
2. 반복복호에서 error floor는 대부분 trapping set이나 Tanner graph 내 짧은 cycle로 인한 검출 가능한 오류(detectable error)에서 발생하며, 이를 낮추는 것이 목표.
3. 핵심 가정은 outer 코드워드의 하드결정 추정치를 accumulator로 재부호화하면 큰 weight을 갖는 detectable error가 trapping set을 형성한다는 것.
4. 1단(interleaver 설계): 2차원 `L×M` 배열 interleaver `π = φπcπrφ⁻¹`를 사용해 accumulator subgraph 내 짧은 cycle을 회피, 조건식 `Σ μ_{i,j} ≡ 0 (mod L)`(식 1)을 만족하는 λ_j 선택을 피함.
5. 식(1)의 의미: 특정 λ 조합이 이 합동식을 만족하면 accumulator subgraph에 짧은 cycle이 생겨 trapping set 유발 → interleaver 설계 시 이 조건 회피가 error floor 억제의 핵심.
6. 2단(message scaling): 표준 반복복호(α1=1)가 정지조건 미충족 시, extrinsic L-value에 스케일 계수 `α_i<1`(예: α2=0.7)를 곱해 재복호하는 2-stage decoding.
7. LDPC 확장 시 variable node에서 다른 edge 메시지 합에 스케일 계수를 곱하는 방식(Chen&Fossorier 방식)을 채택, 특정 degree(예: degree-4) variable node에만 선택적 적용.
8. 별도 학습/최적화 절차 없음(스케일 계수 α2=0.7 고정값을 실험적으로 선택).
9. Margulis code(rate-1/2, 2640,1320)에서 WER 10^-6·Eb/N0=2.4dB의 trapping set(size 12/14, 50개 이상) 전부 2단계에서 정정되어 error floor flattening 미관측; RA/LDPC 계열에서 공통적으로 약 0.2dB 추가 이득.
10. HW 설계 및 gate count/throughput 분석 없음, interleaver 설계는 특정 코드 구조(QCRA, 2차원 배열) 종속적이며 2-stage decoding은 최악 케이스에서 iteration 수(및 latency)가 최대 2배로 증가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2단계 message scaling (α<1로 extrinsic 재조정) | `decoder.cpp` `LDPC_Decoder()`, `ecc_data.h` `PARAM_LLR` | iteration별 LLR threshold 테이블(적응형 LLR)과 유사한 스케일링 개념, 재실행 방식은 조기종료(Partial CRC) 실패 후 후처리로 대응 가능 |
| 조기종료(정지조건) 후 실패 시 재복호 트리거 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 정지조건 체크 로직과 대응, 실패 시 2단계 재시도 로직은 Prime ECC에 명시적으로 없음 |
| interleaver(2차원 L×M) 설계로 trapping set 회피 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 설계 영역과 대응되나 QC-LDPC 구조(circulant shift)와 무관한 별도 interleaver 개념이라 이식 난이도 높음 |
| Margulis/irregular LDPC 특정 부호 구조 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정 구조이며 본 논문 코드와 직접 대응 없음 |

> 적용 가치: 중간 — error-floor 개선을 위한 message scaling 아이디어는 적응형 LLR과 결이 비슷해 참고 가치가 있으나, latency 악화와 특정 코드 구조 종속성 때문에 직접 이식은 제한적.
