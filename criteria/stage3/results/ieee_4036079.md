### B. 알고리즘 요약

1. 대상은 IEEE 802.16e 채택 block-type LDPC(B-LDPC, QC-LDPC 계열) 코드: base matrix `Hb`(mb×nb)를 z×z circulant/all-zero 블록으로 확장한 구조, model matrix `Hbm`에 shift 값 저장.
2. 문제: parity bit puncturing은 code rate를 유연하게 높이지만, punctured node의 초기 LLR을 0으로 가정하는 기존 erasure decoding(CED)은 수렴이 느리고(특히 iteration<50) Tanner graph가 복잡해져 연산량도 늘어남.
3. 핵심 가정: puncturing은 parity bit에만 적용되며, 동일한 punctured parity node에 연결된 두 check node는 병합될 수 있다는 관찰(Theorem 1: 두 even-parity 집합 `Ai`, `Aj`의 대칭차 `Ai⊕Aj`도 even-parity 성립).
4. 핵심 기법: puncturing pattern `(l1,...,lp)`을 적용하면 mb개 check node가 p개로 병합되며, 병합된 check node의 degree는 `2+Σ(d(i)-2)` (Theorem 2)로 계산됨 — punctured node가 사라진 축소 Tanner graph를 얻음.
5. 핵심 식의 의미: 병합 후 check node degree 분포가 punctured 코드의 실제 threshold를 결정하므로, puncturing 설계가 "check node degree 분포 최적화" 문제로 환원됨 (Criterion 1: 병합 후 degree 종류는 최소·연속이어야 함).
6. 확장: mother code의 check node degree 배치(Criterion 2, 식(2))에 따라 동일 degree 분포라도 punctured 코드 성능이 달라짐을 보이고, 이를 만족하는 4단계 탐색 알고리즘(Step1~4)으로 최적 puncturing 위치 결정.
7. 부수 트릭(디코딩): 병합된 축소 Tanner graph에서 바로 sum-product 디코딩(PD, Fig.4(b))하며 punctured bit 복원 불필요 — 예제에서 edge 수가 24→12로 절반 감소, 연산량 절감 비율 `Ratio=(Rp-Rm)/(Rp(1-Rm))`의 `2×Ratio`만큼 variable/check node 연산 감소.
8. 학습/최적화 절차: 별도 학습 없음, 조합적 탐색(Search algorithm)으로 puncturing pattern 결정.
9. 결과: rate 1/3 mother code(16×24, z=132, 1056 information bits)에서 rate 1/2·2/3·4/5로 puncturing 시 PD가 CED 대비 FER=10^-3에서 각각 0.55dB, 0.9dB, 1.3dB 이득; punctured B-LDPC(PD)가 3GPP Turbo code보다 rate 1/3~3/4에서 우수한 성능을 보임(20~50 iteration).
10. 한계: AWGN/BPSK 시뮬레이션에 국한, HW 구현·gatecount·throughput 데이터 없음; error-floor 영역 곡선은 제시되지 않고 waterfall(FER~10^-3~10^-4) 영역 위주.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| punctured-node 제거형 축소 Tanner graph 디코딩 (PD) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 반복 루프 내 특정 노드/엣지 축소 처리 아이디어로 이식 난이도 낮음, 단 Prime ECC는 puncturing 미사용이라 적용 맥락 재검토 필요 |
| check node degree 병합/merging 이론 (Theorem 1,2) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 다루므로 재설계 부담 |
| rate-compatible puncturing 설계 절차 | 대응 없음 | Prime ECC는 고정 rate 구조, 가변 rate 지원 요구가 없으면 적용 불가 |
| shift value 결정 시 cycle 특성 고려 | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 참고용 |

적용 가치: **중간** — punctured node를 아예 제거하고 축소 그래프로 디코딩해 iteration/복잡도를 줄이는 아이디어는 흥미롭지만, Prime ECC는 현재 puncturing을 사용하지 않는 고정 rate QC-LDPC이므로 직접 이식보다는 "불필요 노드 제거형 디코딩 단순화"라는 개념적 참고 가치가 중심.
