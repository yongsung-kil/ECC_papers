> 총평: LDPC error-floor 검증을 위한 FPGA 에뮬레이션 가속 기법(BRAM 폭 활용 벡터 메시지 패킹)으로 디코딩 알고리즘 자체 개선은 없고 순수 처리량 향상 목적이라, Prime ECC의 실제 정정성능·복잡도 개선과는 직결되지 않으나 메모리 대역폭 활용 아이디어는 참고 가치가 있음.

### B. 알고리즘 요약

1. 시스템: 정성적 (γ,ρ) binary QC-LDPC 부호(circulant permutation matrix 기반 base matrix), FPGA(Xilinx Virtex-2/4/5) 기반 partially-parallel 디코더로 error-floor(최대 1e-12) 검증용 에뮬레이션이 목적.
2. 문제: 기존 scalar 디코더는 Block RAM 워드(18~72bit) 하나에 메시지 1개(5~6bit)만 저장해 메모리 대역폭을 크게 낭비, error-floor 측정처럼 방대한 시뮬레이션에는 처리량이 병목.
3. 모델: 디코딩 알고리즘은 normalized min-sum algorithm(NMSA, `σ_cv = α·min|q_cv|·sign(q_cv)`, `α=0.5`)을 그대로 사용, 새로운 채널/부호 모델 없음.
4. 핵심 기법: **Vector Message Passing(VMP)** — Block RAM 워드 하나에 K개의 메시지를 패킹해 K배 CNU/VNU 처리유닛을 동시 구동, 이중버퍼(CNU memory/VNU memory)로 read/write 충돌 회피.
5. 핵심 식: 패킹 위치 `L[k] → Lc[⌊(k-ci)/K⌋][(k-ci) mod K]`(CNU 메모리), `Lv[...]`(VNU 메모리) — CPM offset 때문에 CNU/VNU 접근 순서가 달라 Write Alignment 유닛(레지스터+MUX)으로 재정렬 필요.
6. 핵심 기법 2단: 기존 Overlapped Message Passing(OMP, CNU·VNU 스케줄 겹침)과 VMP를 결합한 Vector-OMP로 iteration 내 대기시간 `w̃`를 최소화(`w̃ = max{⌈m/K⌉-x_c, ⌈m/K⌉-x_v}+1`).
7. 부수 트릭: dual-port BRAM을 2배 클록으로 시분할해 quad-port처럼 사용(CNU/VNU 동시 read+write), hard-decision bit를 extrinsic 메시지에 embed해 별도 메모리 절감.
8. 설계 자동화: QCSyn(Python 툴)이 임의 QC-LDPC 부호와 벡터 길이 K를 입력받아 synthesizable Verilog 생성, FPGA 자원 제약 하 최적 K 탐색.
9. 결과: (3,6) regular m=256 코드에서 K=4일 때 기존 최고 기법 대비 5.5배 throughput(99.1→545.6Mbps), BRAM 사용은 오히려 36→24개로 감소; (3369,3213) irregular 코드 K=4에서 1.234Gbps 달성(3.71배).
10. 한계: 정정성능(BER/error-floor 수치) 자체는 논문 범위 밖(NMSA 그대로 사용, 개선 없음), K 증가 시 Slice/FF/LUT 자원이 비례 이상 증가해 대형 코드는 큰 FPGA 필요, ASIC 공정/gatecount 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Vector Message Passing(BRAM 워드 벡터 패킹) | 대응 없음 (Prime ECC는 FPGA BRAM이 아닌 bit-exact RTL 모사, z=32 lifting 기반 병렬화가 이미 존재) | FPGA 자원 특화 기법이라 ASIC/HW arch 관점에서 직접 대응 없음 |
| Overlapped Message Passing(OMP) 스케줄링 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` (Dual-Update 스케줄과 유사 개념) | 이터레이션 내 CNU/VNU 겹침 아이디어는 이미 유사하게 보유 |
| normalized min-sum(NMSA, `α=0.5`) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()` | 이미 min-sum 기반, 중복 기여 |
| Write Alignment(레지스터+MUX 재정렬) | [GT.cpp](../Prime_ECC_3.1_Claude/) `Make_GT_HW()` (MUX 기반 edge 선택과 유사 개념) | 데이터 정렬용 MUX 개념 정도만 참고 |
| QCSyn 자동 설계 툴 | 대응 없음 | 설계 자동화 툴 자체는 Prime ECC 코드 모듈이 아님 |

적용 가치: **낮음** — 이 논문은 FPGA error-floor 에뮬레이션 처리량을 높이는 순수 HW 자원 활용 기법으로 디코딩 알고리즘/정정성능 개선이 없고, Prime ECC는 이미 z=32 lifting 및 GT/HCU로 유사한 병렬화·라우팅 최적화를 보유해 중복도가 높다.
