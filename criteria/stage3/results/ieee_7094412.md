### B. 알고리즘 요약

1. 대상: (N,K) binary LDPC, BSC(cross-over `α`) 전송, 예제는 (7807,7177) QC-LDPC (dmin=10, column-weight 3).
2. 문제: 짧은/구조화 LDPC는 error-floor에서 trapping set 때문에 저중량 error pattern을 못 고침. BP·quantized Min-Sum도 실패.
3. FAID: `|M|=2s+1` 레벨 유한 알파벳 iterative decoder, 4-tuple `(M,Y,Φ,Ψ)`. CN 갱신 `Ψ(n)=sgn(n)·min(|n|)` (min-sum류), VN 갱신 `Φ`는 Boolean LUT(예: 7-level dv=3 표).
4. 핵심 아이디어 - decoder diversity: VN map `Φ`만 다른 FAID 집합 `{Di}`을 순차 적용하면 각 FAID가 서로 다른 error pattern을 고쳐 집합적으로 더 많은 pattern 정정 (`E=∪EDi`).
5. 최소 diversity set 찾기는 Set Covering Problem(NP-hard) → greedy 알고리즘으로 근사.
6. FAID 생성: 코드 내 harmful (a,b) trapping set `Λa,b`만 대상으로, 그 support의 저중량 error pattern `E[t]`을 커버하도록 `Dbase`에서 FAID를 탐욕적 선택. Tanner graph automorphism으로 탐색 topology 수 축소.
7. 반복적으로 `D[k]→D[k+1]→…→D[t]` 확장, 미해결 pattern `Er`가 공집합이 될 때까지 FAID 추가.
8. 결과: (7807,7177) 코드에 `ND=7` FAID로 guaranteed t=4 정정, error-floor를 단일 FAID 대비 한 decade 개선, 6-bit offset Min-Sum도 수 자릿수 능가 (Fig.1, 최대 100 iter).
9. HW: fully-parallel flooding schedule. 7개 FAID는 VN LUT만 다르고 대부분 엔트리 공유 → 7-table VNU가 단일 대비 +89% 면적. 전체 디코더는 Min-Sum 대비 면적 52%, 33.3Gbps @180nm CMOS 합성.
10. 한계: BSC(hard) 전용, column-weight 3 특화, FAID LUT는 코드 종속(코드마다 재설계), soft-decision/고rate 확장은 본문 미검증.
