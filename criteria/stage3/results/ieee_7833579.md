### B. 알고리즘 요약

1. 대상: binary QC-LDPC, base matrix + `(p^s-1)×(p^s-1)` circulant permutation matrix(CPM) 전개, AWGN+BPSK, SPA 복호(max 50 iter). 예제 `(2048,1024)` rate-0.5.
2. 문제: 짧은 cycle(girth 4/6)은 error-floor를 높임 - girth 8+ 대수적 QC-LDPC를 간단히 구성하는 프레임워크가 필요.
3. 핵심 가정: `GF(p^s)`의 primitive element `α`로 nonzero 원소 `w_{i,j}`를 CPM `A(w_{i,j})`로 치환해 PCM 생성(0은 ZM=masking).
4. 핵심 기법 1: Generalized RS(GRS) 부호로 base matrix `W_GRS(m,n)` 구성 - 원소는 `v_j(α^{i-2}β_j+γ_0)` 형태, MDS 성질 활용.
5. 핵심 식: girth 4 없음 조건은 임의 2×2 부분행렬 `det≠0`(Theorem 1) - GRS 구조가 `v·γ_0·(α차이)·(β차이)≠0`로 자동 만족(Lemma 1).
6. 핵심 기법 2: girth 6 없음(=girth≥8) 조건은 임의 3×3 부분행렬이 식(13)의 6가지 shape 합≠0을 모두 만족(Theorem 2).
7. 구현 트릭: masking matrix `B(m,n)`로 일부 CPM을 ZM으로 바꿔 girth 8+ 및 원하는 degree distribution 달성.
8. 최적화: Algorithm 1 - 3-row/3-col 조합의 식(13) 위반 개수(H_test)를 세어 최다 위반 위치를 mask(-1)로 제거, base row<10이면 빠름.
9. 결과: `GF(2^8)`, `β=(α^-1..)`, `v=(α^1..)`, `γ_0=1`, WGRS(4,8) 서브행렬로 (2048,1024) 생성. masking girth 8+ regular/irregular 코드가 no-masking 대비 우수, irregular는 CCSDS와 거의 동등(BER 10^-6에서 masking vs no-masking gap ~0.4dB).
10. 한계: 순수 시뮬(SPA 50iter), HW/throughput/latency 미평가, error-floor 곡선은 그림에만 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GRS 기반 base matrix + CPM 전개(부호구조) | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 base+circulant shift binary QC-LDPC - H-matrix 교체로 이식 가능 |
| masking으로 girth 8+ / error-floor 개선 | `ecc_top.cpp` `Load_PCM()` (H-matrix 설계 단) | 부호설계 레이어 대응, 단 §4상 이식난이도 높음(고정 QC 재검증) |
| SPA 복호(max 50 iter) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 근사 사용 - full-tanh SPA는 대응 없음(HW면적상 미채택) |
| 인코딩(구성된 PCM) | `encoder.cpp` `Generate_PCM_encoding()` | dual-diagonal 고정이라 GRS-masked PCM용 인코더 재구성 필요 |

적용 가치: 중간 - 부호류(binary QC-LDPC, base+CPM)와 error-floor 겨냥 girth-8 masking은 Prime ECC 부호설계에 개념적으로 이식 가능하나, Prime ECC의 고정 QC 구조(z=32·N_b=129·고rate)·dual-diagonal 인코더·양자화와의 정합 재검증 부담이 크고 예제 rate(0.5)가 NAND ECC급 고rate와 달라 실이식 리스크가 있음.
