### B. 알고리즘 요약

1. 8192비트, rate 7/8 (`(8192,7168)`), (4,32)-regular LDPC 코드를 0.18μm CMOS ASIC으로 구현한 사례 연구.
2. 문제의식: LDPC 완전 병렬 디코더는 대형 코드워드(>1000비트)에서 라우팅 복잡도가 칩 면적의 절반 이상을 차지하는 병목이 됨.
3. 채널/노이즈 모델은 별도 정의 없이 표준 AWGN + Min-Sum 근사(체크노드 tanh 대신 min 근사) 채택.
4. 핵심 기법 1단: `shift-LDPC` 부호 구성 — parity check matrix를 서브행렬 단위로 분할, 좌측 서브행렬은 항등행렬의 랜덤 열치환, 우측 서브행렬들은 이를 1칸씩 cyclic shift하여 생성.
5. 핵심 식: check-to-variable 메시지 계산 시 `α`(스케일 팩터, ~0.75)를 곱하는 정규화 min-sum 근사식 사용(수식 (1)); variable-to-check 메시지는 식 (2)로 계산.
6. 핵심 기법 2단: 이 구조적 규칙성을 이용해 CPU(check node processor unit) 간 "CPU communication network"를 도입, VPU-CPU 간 shuffle network 대신 CPU끼리 고정 이웃과 통신하도록 하여 라우팅을 국소화(intra/inter-iteration 통신 분리).
7. 부수 트릭: 메시지는 최소/차소 magnitude, 최소 위치, sign bit만 압축 저장(old/new/sign 레지스터 3종); 양자화는 b=6bit; girth-4/6 사이클을 제거하는 girth 최적화로 성능을 랜덤 코드 수준까지 끌어올림.
8. 최적화 절차: girth-optimization 소프트웨어로 구조적 shift-LDPC 앙상블 내에서 큰 girth를 갖는 코드를 탐색.
9. 결과: 전통적 partially-parallel 구조 대비 메시지 메모리 67% 절감(65,536bit vs 196,224bit), 15 iteration에서 5.1Gb/s throughput, 기존 UWB용 ultra-sparse 코드 대비 hardware efficiency 70%+ 개선.
10. 한계: 부호 성능은 랜덤/QC-LDPC 코드와 "comparable" 수준(개선 아님)이며 irregular WiMAX 코드보다는 열등; 구조가 특정 shift 기반 PCM에 종속되어 일반 QC-LDPC로의 재사용에는 열 치환(column permutation) 변환이 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| shift-LDPC 부호 구성(구조적 PCM) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base+circulant shift) 고정 구조라 부호구성 자체 이식은 난이도 높음; 다만 shift-LDPC는 QC-LDPC 변환 가능하다고 저자가 언급 |
| Min-Sum 체크노드 연산(정규화 스케일 팩터 ~0.75) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 보유(scaling factor 미확인 상태), 유사 근사 방식이라 참고 가능하나 신규 기여는 아님 |
| CPU communication network (라우팅 국소화) | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC의 GT.cpp(Graph Thinning)와 유사한 목적(edge/라우팅 축소)이나 구현 방식은 상이 - HW arch 이식 난이도 중 |
| 메시지 압축 저장(min/2nd-min/sign만 저장) | `decoder.cpp` `VN_Cal_HD()` 계열, `ecc_data.h` `PARAM_LLR` | Min-sum 메모리 절감 기법으로 Prime ECC 메모리 구조와 비교 시 참고 가치 있음 |
| girth 최적화 알고리즘 | `ecc_top.cpp` `Load_PCM()` | 부호구조 최적화 절차, H-matrix 재검증 부담 커서 이식 난이도 높음 |
```

적용 가치: `중간` — 오류정정 성능 개선은 없으나 메시지 메모리 67% 절감과 라우팅 국소화 아이디어(CPU communication network)는 HW 효율화 관점에서 Prime ECC의 메모리/라우팅 최적화에 참고할 여지가 있음. 다만 부호구조가 shift-LDPC 특유의 구성에 종속되어 있어 Prime ECC의 고정 QC-LDPC 구조로의 직접 이식은 재검증 부담이 큼.
