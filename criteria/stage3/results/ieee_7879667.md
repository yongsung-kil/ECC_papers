### B. 알고리즘 요약

1. 대상은 프로토그래프 기반 종단 LDPC convolutional code(SC-LDPC)로, R4JA 프로토그래프를 원형으로 삼는다.
2. 문제: 프로토그래프 convolutional code의 에지 확장 방식이 성능을 좌우하는데, 좋은 종단 구성법이 필요하다.
3. 반무한(semi-infinite) 패리티체크 행렬 `H_T`로 memory `ms`, cycle `T`를 정의하고 constraint length `vs = c·(ms+1)`로 코드 구조를 규정.
4. 기법: base matrix `B`를 `ms+1`개 부분행렬 `B0,...,Bms`로 분해(변수확장)하되 `B = Σ Bi`로 차수분포를 보존, 이후 공간결합(spatial coupling) 형태로 배치.
5. 예시로 `R=2/3` R4JA 행렬 `B=[2 1 2 1 1 3; 1 3 1 1 3 1]`을 대각분해하여 `B0,B1`로 나눔(분해 후 원소 ≤2, 차수분포 균등화).
6. 에지 확장은 대각분해법·공약수분해법·저차분해법 세 가지 중 성능이 가장 좋은 대각분해법을 채택.
7. 실제 lifting/확장은 PEG 알고리즘으로 수행하며 에지 위치가 성능(neighboring node 연결성)에 영향.
8. `R=1/2, 2/3, 3/4` 세 rate로 프로토그래프 블록코드와 종단 코드를 각각 구성해 BPSK+AWGN에서 BP 복호로 BER 비교.
9. 결과: 동일 코드길이·rate에서 종단 코드 BER은 프로토그래프 블록코드와 유사하나, 복호 HW 복잡도는 코드길이가 아닌 constraint length에 비례해 더 낮음.
10. 한계: HW 미설계, 시뮬만, 짧은 L에서는 이득이 없고 긴 L(긴 데이터)에서만 유리하며 표의 핵심 수치가 .txt에서 손상.
