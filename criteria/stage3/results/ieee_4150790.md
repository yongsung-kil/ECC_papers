### B. 알고리즘 요약

1. 대상 코드는 permutation submatrix로 구성된 구조적 LDPC 계열(RS-based, array-based, lattice-based, Ramanujan-Margulis 등) 중 (6,32)-regular (2048,1723) RS-LDPC(rate≈0.841), 표준 min-sum 유사 message-passing 디코딩(`Φ(x)=-log tanh(x/2)` 기반 sum-product 근사) 사용.
2. 문제의식: 낮은 BER(10^-10 이하) 영역까지 소프트웨어 시뮬로는 검증이 사실상 불가능(월 단위 소요)하므로 FPGA 하드웨어 에뮬레이션으로 초당 240Mb/s 처리하여 짧은 시간(1시간)에 BER 10^-10~10^-13대까지 관측.
3. δ×δ(=64×64) permutation submatrix 구조를 활용해 열(column) 기준 32개 병렬 처리유닛(ρ=32)과 행(row) 기준 6개 그룹(γ=6)으로 나누는 parallel-serial 아키텍처, M0/M1 이중 뱅크 메모리(bit-to-check/check-to-bit 교대 저장)로 글로벌 인터커넥트 최소화.
4. 5/6/9-bit 균등양자화(3.2/3.3/4.5 fixed-point) 세 가지 워드길이로 구현·비교; 워드길이가 짧을수록 저BER 영역 error floor가 심함.
5. 핵심 관찰(오류 특성 규명): 최종 16회 iteration 동안 비트값이 진동(oscillation)하는 경우와, 비트값이 고정되어 non-codeword로 수렴하는 absorbing set 오류로 구분. 저정밀도(5-bit)에서는 oscillation이 지배적(74건 중 71건), 고정밀도(6/9-bit)에서는 absorbing set이 지배적(31건 중 27건).
6. Absorbing set 정의(신규 제안): (a,b) subset 중 a개 bit node가 유도된 subgraph 내에서 홀수차수 check b개(b>0)를 가지며 각 bit node가 홀수차수 check보다 짝수차수 check에 더 많이 연결된 경우; 최소(a'<a)인 것만 absorbing set으로 인정 — trapping set/stopping set/near-codeword 개념을 구체화.
7. 부수 트릭: oscillation 오류는 워드길이를 1~2bit만 늘린 재실행(post-processing)으로 대부분(4~9 iteration 내) 정정 가능하지만, absorbing set 오류는 float precision 재실행으로도 정정 불가(코드 구조 자체 문제).
8. 최적화 절차: 관측된 (8,8) absorbing set(8 bit node가 28개 check 중 20개와 degree-2로, 8개와 degree-1로 연결)을 전수 열거(enumerate)하고, 해당 8-bit sublist 중 한 비트씩 flip한 뒤 소수 iteration(10회) 재실행 → 수집된 모든 absorbing-set 오류가 성공적으로 정정됨.
9. 결과: 6-bit 구현에서 BER 10^-10 부근 slope 변화(error floor) 관측; absorbing set 기반 bit-flip 후처리로 해당 오류 전량 정정 성공. peak throughput 240Mb/s(100MHz 클록), 소프트웨어(C, Xeon 2.4GHz) 대비 260kb/s로 약 900배 가속.
10. 한계: 특정 (2048,1723) RS-LDPC의 (8,8) absorbing set 구조에 한정된 사후분석/후처리이며, 실시간 디코더에 absorbing set 검출·flip 로직을 통합하는 구체적 HW 설계는 제시하지 않음(future work로 남김); 코드 설계 단계에서 absorbing set을 원천 회피하는 방법도 향후 과제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 계열 message-passing 디코딩 루프 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 동일한 iterative C2V/V2C 구조로 직접 대응 |
| 5/6/9-bit 고정소수점 양자화, Φ LUT 기반 soft 메시지 | `decoder.cpp` `VN_Cal_HD()`/`Get_VNU_*`, `ecc_data.h` `PARAM_LLR` | 워드길이-error floor 트레이드오프 통찰이 LLR 양자화 설계에 직접 적용 가능 |
| absorbing set(흡수집합) 검출 및 bit-flip 후처리 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프(post-processing 단계 추가 위치) | Prime ECC에 없는 error-floor 대책으로 신규 이식 가치 높음 |
| δ×δ permutation submatrix 기반 병렬-직렬 아키텍처 | HW arch (z=32 lifting, HCU) | 구조적 유사성(circulant/permutation block) 있으나 QC-LDPC lifting 구조는 이미 보유 |
| FPGA 기반 저BER(10^-10+) 에뮬레이션 방법론 | `TV_Dec_Inter/TV_Dec` (HW 검증용 신호 출력) | 검증 방법론 자체가 유사한 목적(HW 정합성/저BER 특성 확인)에 참고 가능 |

적용 가치: 높음 — error floor의 두 가지 근본 원인(양자화로 인한 oscillation, 코드구조로 인한 absorbing set)을 명확히 분리하고, 후자에 대해 실효성 있는 bit-flip 후처리 기법을 제시해 min-sum 디코더의 error-floor 개선에 바로 참고 가능.
