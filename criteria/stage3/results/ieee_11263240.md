### B. 알고리즘 요약

1. IEEE 802.3ca 표준 LDPC(codeword 17664 bit, information 14592 bit, `R≈0.826`)를 16-QAM으로 전송하는 광통신 시스템에서 AWGN + 버스트 잔류 위상잡음(residual phase noise) 채널을 다룸.
2. 문제: 위상잡음이 시간적으로 버스트성(급격한 위상 교란)을 가지면 기존 정적 LLR 계산이 부정확해지고, 인터리빙은 버스트를 분산시키지만 패킷 단위 재전송 구조상 오히려 PER을 악화시킬 수 있음.
3. 채널 모델: `Y = Xe^{jθ} + N`, θ는 Gilbert-Elliott(GE) 2-state(Good/Bad) 마르코프 모델을 따르는 위상잡음 분산(`σ_G^2`, `σ_B^2`)으로 상태별 크기가 다름.
4. 핵심 기법 1단: 2-state Viterbi 기반 채널상태추정. 트렐리스 branch metric `BM_i = p(Y_i|S_i)·P(S_i|S_{i-1})`으로 각 시점의 G/B 상태를 추정.
5. 핵심 기법 2단: burst-aware LLR 계산. 추정된 상태 S를 이용해 `LLR_m(Y) = log[Σ_{X∈X0m} p(Y|X,S)P(X) / Σ_{X∈X1m} p(Y|X,S)P(X)]`로 상태별 우도함수(LT 또는 BLT 근사, 식 4·5)를 적용.
6. 베이스라인은 상태정보 없이 G/B 분산을 정상상태확률로 가중평균한 `σ_eff^2 = σ_G^2·P_G + σ_B^2·P_B`로 대체.
7. 부수 트릭: 우도함수 근사에 linear transform(LT)과 bilinear transform(BLT) 두 방식을 비교, BLT가 일관되게 더 우수.
8. 별도 학습/최적화 절차 없음(파라미터는 GE 모델 전이확률 `P_GB=2×10^-4`, `P_BG=2×10^-3`, 평균 버스트 길이 500 심볼로 고정).
9. 결과: 인터리빙 없을 때 JBA-BLT가 noJBA-BLT 대비 BER 4×10^-3에서 1.5dB 이득, 인터리빙 있을 때 0.5dB 이득, PER 1%에서 0.7dB 개선. 단, 심한 버스트(σ_B^2 큰 영역)에서는 인터리빙 없는 JBA가 인터리빙 있는 JBA보다 PER 우수.
10. 한계: HW 미설계, 시뮬레이션뿐(실측 없음), 위상잡음이라는 특정 광통신 손상 모델에 한정(코드 자체는 표준 재사용이라 순수 디코더 측 LLR/상태추정 전처리 기법).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Viterbi 기반 채널상태(GE) 추정 | 대응 없음 | Prime ECC는 AWGN/RBER/fixed-error 채널 모델만 지원하며 마르코프 버스트 상태추정 블록이 없음 |
| burst-aware LLR 계산 (식 4, 5, 8) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | LLR 양자화/threshold 설계 레이어와 개념적으로 닿아있으나 위상잡음 특화 우도식이라 그대로 이식 불가 |
| IEEE 802.3ca LDPC 부호 자체 | `ecc_top.cpp` `Load_PCM()` | 부호설계 기여 없음(표준 재사용), 대응 관련성 낮음 |
| LDPC 반복복호(min-sum 아님, BP 암시) | `decoder.cpp` `LDPC_Decoder()` | 복호 알고리즘 자체 개선은 없음, 직접 대응 없음 |

> 적용 가치: 낮음 — 부호설계 기여가 없고, 채널상태추정+LLR 보정은 광통신 위상잡음 특화 기법으로 NAND read-channel(전하손실/셀간간섭 등) 모델과 직접 매칭되지 않음.
