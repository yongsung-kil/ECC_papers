### B. 알고리즘 요약

1. 대상은 `(dv, dc, L)` 앙상블의 rate `R = 1 - dv/dc` binary spatially coupled LDPC 코드이며, BEC 채널에서 recursive(shift-register) 인코딩을 사용.
2. 기존 recursive 인코더([5])는 인코딩 후 레지스터가 zero state가 아니어서, 종단(termination)을 위해 tail bit를 선형방정식으로 풀어야 하고 이 계산량이 `O(M^2)`(M=lifting number)로 크다.
3. [9]는 accumulator 기반 `O(M)` 종단을 제안했으나 tail bit가 이후 위치의 비트에 의존해 시간 지연(time delay)이 발생.
4. 본 논문은 protograph에 RA(repeat-accumulate) tail 변수노드를 추가한 modified SC-LDPC 구조를 제안 - 추가 변수노드는 위치 `L, L+1, ..., L+ms-2`의 체크노드에 연결.
5. tail bit `zi`는 식 (4)로 계산되며 이전 위치의 인코딩된 비트에만 의존하므로 지연 없이 `O(M)` 복잡도로 계산 가능.
6. 부수 트릭: 추가된 변수노드로 인해 설계 rate가 `R = 1 - (L+2dv-2)/(L·dc/dv+(dv-1))`로 약간 손실되지만 L이 커지면 `1-dv/dc`에 수렴.
7. 성능분석은 BEC 위에서 density evolution으로 BP threshold를 계산해 Shannon limit과의 gap을 비교.
8. 결과: (3,6)~(4,16) 여러 `(dv,dc)` 조합에서 RA tail 구조의 BP threshold gap이 기존 구조([5])보다 약간 크지만(예: (3,6)에서 0.0444 vs 0.0332) 무시할 수준.
9. 시뮬레이션(BEC, 유한 길이 L=50)에서 bit erasure rate가 기존 구조와 거의 동일한 곡선을 보임.
10. 한계: BEC 채널에만 한정된 이론/시뮬 결과이며, HW 설계나 디코더 성능 개선은 다루지 않음 - 순수 인코더 종단 기법.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 인코더 종단(RA tail) | 대응 없음 | Prime ECC는 dual-diagonal 고정 QC-LDPC 블록코드 인코더(`encoder.cpp`)이며 SC-LDPC 컨볼루션 구조·종단 문제 자체가 없음 |
| 인코더 복잡도 감소 | `encoder.cpp` `LDPC_encoder_4KB()` | 인코더 복잡도라는 주제만 공유, 구조적 대응 없음 |
| BEC 채널 density evolution | 대응 없음 | Prime ECC는 AWGN/RBER/fixed-error 채널 시뮬레이터로 BEC 이론분석 대상 아님 |
```

> 적용 가치: 낮음 - SC-LDPC 컨볼루션 코드의 인코더 종단 문제로, binary QC-LDPC 블록코드 고정 구조인 Prime ECC와 부호구조 자체가 다르고 디코더/HW 개선도 없음.
