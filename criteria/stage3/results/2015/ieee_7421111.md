### B. 알고리즘 요약

1. AsAP2(164개 16-bit RISC 프로세서 2D mesh, 65nm) many-core 플랫폼에 소프트웨어 LDPC 디코더를 구현, 길이 4095(rate 0.82)와 16129(rate 0.95) 두 코드를 대상으로 함.
2. 목표는 전용 HW 디코더의 높은 설계비용/유연성 부족 문제를 소프트웨어 many-core로 대체하는 것.
3. 채널/에러 모델 특정 서술 없음(일반 LDPC 디코딩 문제로 다룸).
4. Chen et al.의 Reduced-complexity Min-Sum 알고리즘을 사용, `Compact Set`(두 최소 magnitude와 XOR sign `T` 추출) → `Unpack Set` → `Update Variable`(`D_jk` 부분합 갱신, `Q_j` 산출) → `Correct Variable`(`V_ij = Q_j - C_ij`) → `Valid Codeword Detection` 5단계 소프트웨어 파이프라인.
5. 핵심 데이터 압축: 체크노드 메시지 `C_ij`는 두 개의 최소 magnitude만으로 표현 가능(min-sum 특성)해 메모리 오버헤드를 줄임.
6. 코드 길이별로 Compact Set lane, Update Variable lane을 다수 코어에 복제·스택하여 병렬 처리량 확장(스케일러블 매핑).
7. 메모리 매핑을 코드 길이에 따라 인터리브(4095) 또는 패킹(16129, 2개 변수/주소) 방식으로 구현, 주소 생성기를 다수 코어에 분산.
8. 반복적 프로파일링으로 각 코어의 최소 필요 주파수/전압을 찾아 에너지 최적화(전력 9.3%/12.1% 절감, throughput 손실 0.56%/0.44%).
9. 결과: 21.3Mbps(4095) / 13.4Mbps(16129) throughput, 131/188 nJ per bit, GPU 대비 throughput/area 2.15배, CPU 대비 75배 우수(22nm 스케일 비교).
10. HW ASIC 설계 없이 범용 프로그래머블 코어 기반 소프트웨어 구현이며, BER 등 정정성능 검증 없음(throughput/에너지 효율만 평가).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| many-core 소프트웨어 매핑/데이터 라우팅 | 대응 없음 | Prime ECC는 bit-exact HW 클록 모델 시뮬레이터로 범용 프로세서 소프트웨어 매핑과 무관 |
| 주소 생성(H-matrix 접근 패턴) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 개념적으로만 유사, 실질 이식 요소 없음 |
