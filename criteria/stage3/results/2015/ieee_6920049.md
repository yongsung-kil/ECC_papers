### B. 알고리즘 요약

1. 대상: `Mb×Nb` block 구조 QC-LDPC(802.11n 길이 648을 확장), 구현부호 length `96000`, expansion factor `b=4000`, rate `1/2, 2/3, 3/4`, AWGN/BPSK.
2. 문제: TDMP(layered BP)는 TPMP 대비 수렴 ~2배 빠르나, 고속화 시 인접 block row가 같은 변수노드를 공유해 data correlation·메모리 접근 충돌 발생.
3. 복호: hybrid normalized MS(HNMS) - `Rmn = -min|Qnm|×∏sign`에 normalization factor `γ`(0.5<γ<1) 적용, `Qn_new=Qnm+γ·Rmn`. 8bit 양자화(sign1/int4/frac3).
4. 핵심 기법1(병렬도 최적화): 큰 expansion factor block row를 subblock row로 분할, 반복지연 `dL`을 고려해 병렬도는 `b/(dL+2)` 이하로 제한(너무 작으면 clock cycle 증가).
5. 핵심 기법2(offset iterative sequence rule): 비영 submatrix를 common/individual 영역으로 나누고, 반복 시작점 `Sp_next`를 이전 block row 시작점 + 최대 offset `Fmax`로 갱신(식 6)해 아직 미완료된 공유 변수노드 접근을 회피.
6. 의미: base matrix의 복잡한 reordering 없이 shared 변수노드가 이전 block row에서 이미 갱신되도록 순서를 어긋나게 배치해 충돌 제거·clock cycle 감소.
7. γ는 `1/2` 및 `1/4`의 2의 거듭제곱 합(예 0.625=1+1/8 아닌 (1)+(1/8) 식 표기)으로 근사해 HW에서 shift-add로 구현.
8. 아키텍처: central controller, interleaver/deinterleaver, CNU processors, EMC RAM(MAG_R=min1st/min2nd, POS, SIGN), VNS/VND RAM, dual-port RAM으로 pipeline; chip utilization ~85%.
9. 결과: 90nm 1P9M CMOS, chip 13.32mm²(core 9.73mm²), 378929 gates, 100MHz, 10 iter에서 throughput `1.7~3.0Gb/s`, 평균전력 502mW, 17pJ/bit/iter, 반복당 ~240~480 clock cycle.
10. 한계: HW 미측정(postlayout 시뮬 전력), 무선 표준 확장부호·rate 0.5~0.75·8bit soft LLR로 NAND 고rate·저비트 read와 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| HNMS(normalized min-sum) CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum과 중복, normalization factor만 참고 |
| TDMP layered 스케줄·offset iterative sequence | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 스케줄/충돌회피 아이디어, dual-update 교대스케줄과 유사·부분 중복 |
| 병렬도 분할·pipeline 메모리 충돌 회피 | 디코더 HW 모델(z=32, HCU, GT) | z/pipeline 구조 정합 필요, 이식난도 중 |
| min1/min2·sign 저장(EMC RAM) | `decoder.cpp` `CNU_Update_New_Mag()` | min1/min2 구조는 Prime ECC와 동일 개념 |

적용 가치: **중간** — offset iterative sequence rule 기반 layered 스케줄 충돌회피는 iteration/latency 절감에 참고할 만하나, HNMS·min-sum·min1/min2 기여는 Prime ECC 보유기법과 겹치고 부호/양자화 조건이 NAND와 달라 직접 이식은 제한적이다.
