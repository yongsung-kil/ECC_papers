### B. 알고리즘 요약

1. 대상은 cyclic code(EG-LDPC) 기반 majority-logic(ML) 디코딩을 사용하는 SRAM/일반 메모리 시스템으로, hard-decision 1비트 판정만 다룬다.
2. 기존 ML 디코더는 오류 유무와 무관하게 항상 `N`(코드워드 길이) 사이클을 소모해 무오류 시에도 불필요한 지연·전력이 발생하는 문제가 있다.
3. 채널/오류 모델은 SRAM Single Event Upset(SEU) 기반 soft error이며 별도 채널 통계 모델은 제시하지 않는다.
4. 핵심 기법은 XOR 매트릭스 출력을 첫 3사이클 동안만 관찰하는 fault-detector(control unit)로, 3사이클 내내 `0`이면 오류 없음으로 간주해 즉시 종료한다.
5. Control unit은 3-stage shift register로 최근 3사이클의 XOR 매트릭스 출력을 OR 결합(`OR1`, `OR2`)해 finish 신호를 생성하며, 오류 존재 시에만 정상 N-사이클 ML 디코딩으로 전환한다.
6. 확장 없이 단일 기법(early-exit control unit)만 사용한다.
7. 구현 디테일은 cyclic shift register + XOR matrix + majority gate로 구성된 기존 ML 디코더 구조를 그대로 두고 control unit만 추가하는 방식이다.
8. 별도 학습/최적화 절차는 없으며 (7,3) cyclic code 예제로 Xilinx 기반 시뮬레이션만 수행했다.
9. 결과는 무오류 시 total delay 14.006ns→12.578ns, 전력 49mW→47mW(약 4% 절감)로 보고된다.
10. 한계는 (7,3) 소형 코드 예제에 한정되고 gatecount/throughput 등 대용량 NAND급 수치가 없으며, HW 검증도 FPGA 시뮬레이션 수준에 그친다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 무오류 조기종료(control unit) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 이미 CRC 기반 조기종료가 구현되어 있어 개념적으로 중복, majority-logic 방식은 직접 대응 없음 |
| Majority-logic 디코딩 전체 | 대응 없음 | Prime ECC는 min-sum 기반이라 majority-logic 디코딩 구조 자체가 없음 |
| EG-LDPC 부호구조 | 대응 없음 | binary QC-LDPC와 무관한 EG 기반 cyclic code |

> 적용 가치: 낮음 — 조기종료 아이디어는 이미 Partial CRC로 구현되어 있고, majority-logic 하드판정 디코더 구조 자체가 Prime ECC의 min-sum soft 디코더와 이질적이라 이식할 요소가 거의 없음.
