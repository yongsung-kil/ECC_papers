### B. 알고리즘 요약

1. 대상 코드는 WiMAX 표준의 (1056,528) `Rc=1/2` 불규칙 QC-LDPC 코드, 44×44 순열행렬 기반 구조.
2. 문제의식: stochastic decoding은 HW/에너지 효율적이나 SPA 대비 성능 손실(~0.5dB)과 error floor 악화가 있으며, 그 원인이 되는 trapping set 구조가 아직 특정되지 않음.
3. 채널 모델은 AWGN + BPSK(및 BSC), stochastic decoder는 확률을 1-bit Bernoulli 스트림으로 표현해 노드 연산을 비트단위 논리로 대체.
4. Variable/check node 연산은 `Pc = PaPb + PaPb(1-Pa)(1-Pb)` (VN, 식1), `Pc = Pa(1-Pb)+(1-Pa)Pb` (CN, 식2)로 정의되는 degree-3 확률 결합 연산.
5. 이 두 식은 SPA의 확률 도메인 메시지 전달을 비트-시리얼 stochastic 연산으로 근사한 것으로, decoding cycle(DC) 단위로 반복됨(SPA 32 iter, stochastic 700 DC 비교).
6. SNR 3.5dB(error floor 영역)에서 SPA는 성공하나 stochastic decoding만 실패하는 노이즈 패턴을 시뮬레이션으로 탐색해 "stochastic-decoding-specific trapping set"을 정의.
7. QC 구조 특성상 44개 순열행렬 shift에 대해 동일 크기의 trapping set이 반복 발견됨(shift δ 표로 44개 multiplicity 나열).
8. 별도 학습/최적화 절차 없음 - 순수 시뮬레이션 기반 탐색.
9. 결과: 크기 10의 elementary absorbing set T(10,1) 확인, degree-2 VN 3개+degree-3 VN 7개 구성, 내부 cycle 길이 12/14/18. degree-2 노드가 trapping set에 더 취약함을 관찰.
10. 한계: HW 미설계, 순수 시뮬레이션 관찰 수준이며 코드 구조 개선안(actual redesign)은 제시하지 않고 "가이드라인 제공" 수준에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic decoding 알고리즘(비트 스트림 기반 VN/CN 연산) | 대응 없음 | Prime ECC는 min-sum 기반 디코더로 stochastic 비트-시리얼 연산 방식 자체가 없음 |
| Trapping set / error floor 원인 분석 | `decoder.cpp` `LDPC_Decoder()` | 개선 대상 자체(error floor)는 공통 관심사이나 stochastic 고유 현상이라 직접 대응 안 됨 |
| QC-LDPC 순열행렬 shift 구조 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 관점에서만 부분적 유사성(QC 구조) |

적용 가치: `낮음` — stochastic decoding 고유 현상 관찰 논문으로 Prime ECC(min-sum 기반)에 이식할 알고리즘·구조 개선안이 없음.
