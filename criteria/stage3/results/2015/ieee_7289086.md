### B. 알고리즘 요약

1. (61440, 20480) irregular LDPC 코드(rate 2/3), BPSK/AWGN 채널에서 min-sum 계열 디코더의 정규화 팩터 `α`를 다룸.
2. 문제: 기존 정규화 min-sum(NMSA)의 상수 `α`나 기존 적응형(ANMSA)의 단일 `α`는 irregular 코드에서 체크노드 degree `d_c`별 편차를 반영 못해 SPA 대비 성능 열화가 큼.
3. 가정: `α`는 이론적으로 체크노드 degree `d_c`와 사전 상호정보량 `I`(EXIT 분석 기반)에 의해 결정되나, 실제 구현에선 `I` 측정이 어려움.
4. 핵심 기법: 실패 parity-check 비율 `p = Σsᵢ/m` (hard-decision `d=Hᵀ·d`로 계산)을 `I`의 대리 지표로 사용, 체크노드를 degree `d_c`별 서브그룹 `T`개로 나눠 각 그룹마다 `α = g_i(p)` lookup table을 둠.
5. 핵심 식: 수평처리(check-to-variable) `L_mn = α × sign(Z)곱 × min|Z_mn'|` (식 12) — 여기서 `α=g_i(p)`가 매 이터레이션·매 서브그룹마다 갱신되어 min-sum의 soft 정보 정확도를 SPA에 근접시킴.
6. 두 종류 lookup table: 코드 degree 분포만으로 학습하는 "General-adaptive function"(범용)과 특정 코드로 재학습하는 "Aiming-adaptive function"(코드 특화, 성능 더 우수).
7. 부수 기법: Genie-Aided Training으로 SPA 출력과 min-sum 출력의 MMSE 기준 최적 `α`를 Monte Carlo로 곡선적합(curve fitting)·샘플링·양자화해 lookup table 생성, 이론상 상한(Genie-Aided Bound)도 함께 도출.
8. 학습 절차: SNR을 바꿔가며 반복 시뮬레이션으로 `(p, α_i)` 데이터 수집 → 곡선적합으로 `α=g_i(p)` 함수화 → lookup table로 양자화.
9. 결과: BER 10⁻⁵ 이하에서 기존 NMSA 대비 약 0.24dB 코딩이득, SPA와의 갭은 0.06dB로 축소.
10. 한계: HW 구현·gatecount·throughput 등 실측 없음(순수 알고리즘 시뮬), 매우 긴 코드(61440)·특정 irregular degree 분포 전용, iteration 수 감소 효과는 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 체크노드 degree별 적응형 정규화 팩터 min-sum | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 스케일링 팩터를 degree별·iteration별 lookup으로 바꾸는 아이디어를 CN 연산부에 이식 가능 |
| 실패 parity-check 비율 p 계산 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | syndrome 계산은 이미 존재(조기종료용); p 통계를 정규화 팩터 갱신에 재사용하는 로직 추가 필요 |
| 정규화 lookup table (p→α) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 iteration별 LLR threshold 테이블 구조와 유사한 형태로 degree별 α 테이블 추가 이식 가능 |
| Genie-Aided Training(오프라인 학습) | 대응 없음 | 코드 내 오프라인 학습 파이프라인 없음, 논문 자체 Monte Carlo 절차로 별도 구축 필요 |

> 적용 가치: 중간 - min-sum CN 연산에 degree별 적응 정규화를 추가하는 기법은 `decoder.cpp`의 CNU 연산에 비교적 낮은 난이도로 이식 가능하나, Prime ECC는 이미 iteration별 적응형 LLR 테이블(§3)을 보유하고 있어 중복 기여 소지가 있고 irregular 전용(degree 다양성) 가정이 Prime ECC의 QC-LDPC 구조(균일 VN degree 17)와 정합성 확인 필요.
