### B. 알고리즘 요약

1. 대상은 rate-1/2 단블록 LDPC 3종(`96.33.964` n=96, `PEGReg252x504`, `PEGirReg252x504` n=504), AWGN 및 frequency non-selective fading 채널.
2. 문제: 고SNR에서 trapping set(near-codeword)과 cycle이 유발하는 error floor를 그래프 edge 재설계 없이 낮추고 싶다.
3. 핵심 개념: `(a,b)` trapping set = a개 bit error + b개 unsatisfied check, 짧은 cycle 조합이 dominant. 큰 girth 최적화 부호에서 trapping set이 뭉쳐 있어 소수 pilot으로 커버 가능.
4. 핵심 기법: message frame 내 trapping set/cycle 위치에 값이 이미 알려진 pilot bit을 인코딩 전 삽입, systematic 부호라 전송 전 puncture하고 디코더에서 high-LLR soft bit으로 재삽입한다.
5. 효과: pilot이 induced sub-graph에 고신뢰 LLR을 공급해 해당 trapping set의 trail을 비활성화(`disable`), (4,4) trapping set은 pilot 1개로 무력화 가능.
6. 유효 rate는 `Rp=(k-p)/n`로 저하되므로 pilot 수 p는 최소화해야 하며 위치 최적화가 관건이다.
7. 제안 알고리즘: (1) trapping set 탐색 (2) all-zero codeword와 Hamming 거리 계산 (3) `PE≈Q(sqrt(2·d²))`로 SNR별 BER 기여 계산 (4) bit별 기여 히스토그램 f(i) (5) 최대 기여 위치에 pilot 삽입 (6) 그 pilot 포함 trapping set 제거 (7) 목표 pilot 수까지 반복.
8. 최적화는 dominant trapping set의 경험적 열거 기반(별도 학습 없음).
9. 결과: `96.33.964`에 pilot 3개(위치 91,55,79, 유효rate 0.484)로 dominant (6,0) 오류 66%->대폭 완화, 고SNR error floor 가시적 하락. noise threshold(waterfall)는 불변.
10. 한계: HW 미설계, 짧은 블록 시뮬뿐, rate loss가 pilot 수를 제약하고 trapping set이 많은 부호엔 효과 감소.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| pilot bit을 high-LLR로 재삽입 | `channel.cpp` `Set_LLR_Th()` / `ecc_data.h` `PARAM_LLR` | known 위치에 고LLR 강제 주입 형태로 LLR 테이블/채널단에서 구현 가능 |
| pilot puncture (systematic 인코딩) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 메시지 일부를 known bit으로 예약·puncture하려면 인코더 수정 필요 |
| trapping set 열거·타깃팅 | `corner.cpp` (수렴실패 추적) | dominant trapping set 식별에 근접, 다만 pilot 위치 최적화 로직은 신규 필요 |
| error-floor 완화 효과 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 디코더 알고리즘 불변, 입력 LLR 조작만으로 error-floor 개선 |

적용 가치: **중간** — decoder/encoder 코드 수정 없이 LLR 주입만으로 error-floor를 낮출 수 있어 이식 자체는 쉽지만, 고rate NAND ECC에서 pilot 만큼의 rate loss와 QC-LDPC trapping set 사전 열거 부담이 실효성을 제한한다.
