### B. 알고리즘 요약

1. 대상: SSD의 두 극단 응용 — archive(초장기 retention·소량 W/E)와 enterprise(대량 W/E·단기 retention); ECC(BCH/LDPC)는 컨트롤러에 상시 병존하는 배경 요소로만 언급.
2. 문제: TLC는 bit-cost 낮지만 retention BER가 MLC의 ~10배로 장기 보관 부적합; 최신 BCH(60bit/1KB, 허용 raw BER 0.25%)·LDPC(1~5%)로도 TLC retention은 10년 이하로 제한.
3. 기법1 - n-out-of-8 level cell(nLC): 측정된 상태별 BER 편중(Fig.3)을 이용해 BER 높은 memory state를 제거하고 read-reference 전압을 가상 조정해 read margin 확대; 7LC는 P6 제거로 P7 margin 최대화.
4. nLC 데이터 매핑: base-n 변환(host binary→base-n digit→memory state) — (k,j) 최적화로 `k`비트를 `j`셀에 저장, 제약 `2^k ≤ n^j`, 7LC는 (14,5)로 effective cell level 6.96.
5. 셀레벨 선택: 요구 retention에 맞춰 bit-cost 최소화하는 cell level `n`을 selector가 선택 (retention 길수록 낮은 n).
6. 기법2 - universal asymmetric coding(UAC): W/E cycle·retention에 따라 최고 BER memory state가 바뀜을 실측(E→P1, E→P2 등)하고, "1"/"0" 편중을 조절하는 asymmetric code(Original, Code-1~4)를 상황별로 선택.
7. UAC 구조: multi-page(upper/lower) 조합으로 4가지 state 분포 실현, code length 5bit(4-bit data+1-bit flag, "1" 최대 69%) 또는 17bit(flag overhead 6.3%, 60%).
8. HW: 7LC codec은 5 clock/14bit(200MHz), UAC codec은 1 clock; nLC·UAC 모두 단순 로직, 버퍼 공유 가능해 면적·전력 penalty 작음 (Fig.25 encoder/decoder).
9. 결과(실측): 7LC는 1k년 retention에서 BER 79% 저감, UAC는 random 대비 최대 52% 저감; 고정 AC는 조건에 따라 오히려 37% 악화 → 적응 선택 필요성.
10. 한계: ECC가 아니라 그 앞단의 셀레벨 변조·데이터패턴 코딩이며, 단일 블록 실측이라 die/generation variation 미고려(future work).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| nLC/UAC 셀레벨·비대칭 코딩(ECC 전단) | 대응 없음 | ECC 바깥 계층 — 우리 LDPC 부호/디코더/인코더 어디에도 대응 없음 |
| raw BER 저감이 채널 통계에 주는 영향 | `channel.cpp` (RBER/fixed-error 주입) | 간접: UAC/nLC로 낮아진 raw BER를 우리 채널 RBER 입력에 반영해 시뮬 조건으로만 활용 가능 |
| LDPC ECC (배경 언급) | `decoder.cpp` `LDPC_Decoder()` | 논문은 LDPC를 배경으로만 언급, 디코더 자체 개선 없음 |

적용 가치: **낮음** — NAND 직접 타깃이고 실측 데이터가 풍부하지만, 제안 기법이 ECC 전단의 셀레벨 변조·데이터패턴 비대칭 코딩이라 binary LDPC 부호설계·디코더·HW 이식과 무관하다. 기껏해야 raw BER를 낮추는 전처리로서 우리 채널 통계 입력에 참고할 수 있는 수준이다.
