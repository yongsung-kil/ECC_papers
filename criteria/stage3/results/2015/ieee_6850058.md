### B. 알고리즘 요약

1. MLC NAND flash memory(2b/cell)의 P/E 사이클 endurance 개선을 위한 기법. 각 셀 컨텐츠(11/10/00/01)별로 물리적 damage가 다름을 이용.
2. 기존 [7]의 independent enumerative coding은 상위/하위 페이지를 독립적으로 매핑해 damage 저감이 제한적.
3. 채널/노이즈 모델은 없으며, 대신 damage factor `C(i) = ρ_min / ρ_max^(i)` (특정 데이터 콘텐츠 i를 기록했을 때 목표 BER 도달까지의 P/E 사이클 수 기반)로 물리 손상을 정량화.
4. 핵심 기법: 압축된 데이터를 lower page는 기존 enumerative coding으로, upper page는 이미 코딩된 lower page 데이터를 고려해 damage를 최소화하는 subset `B_k`를 선택(`t = argmin_k Σ C(2ν_i + ω_i^(k))`, 식 2).
5. `{0,1}^n`을 해밍 웨이트 기준 `2^(n-m)`개 subset으로 분할, 각 m-bit 입력을 3단계(Hamming weight 결정 → index 계산 → bit-by-bit 복원)로 enumerative coding하여 1-to-1 매핑(식 3~4).
6. 확장: 후보 subset 수가 크면 제약된 부분집합 `Ω ⊂ [1, 2^(n-m)]`만 탐색해 구현 복잡도 절감(|Ω|=1이면 기존 independent coding과 동일).
7. HW 구현 디테일: 조합수 `C(n,i)` 값을 on-the-fly 계산 대신 on-chip 메모리에 사전 저장, LZ 압축/해제와 함께 65nm ASIC 합성.
8. 별도 학습/최적화 절차 없음(damage factor는 실측 BER 곡선 기반 정의).
9. 결과: 압축비 1.5:1에서 기존 independent coding 대비 endurance 25% 개선, 무압축 대비 50% 개선; 1GB/s throughput 목표로 실리콘 비용 미미(8kB 페이지당 약 8μs 지연).
10. 한계: LDPC(length-2kB, rate-0.94)는 endurance 평가용 배경 ECC로만 사용되고 본 논문의 기여는 압축+매핑 기법이며 LDPC 디코더/부호 자체를 개선하지 않음. 실측은 sub-22nm 상용 칩 기반이나 ECC 알고리즘 상세는 불명.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| selection-based enumerative coding (데이터 매핑) | 대응 없음 | ECC 부호/디코더가 아닌 프로그래밍 전 데이터 매핑 계층 기법 |
| damage factor 기반 endurance 모델 | 대응 없음 | Prime ECC는 복호 성능/HW 모델이며 셀 물리손상 모델 없음 |
| length-2kB rate-0.94 LDPC (평가용) | `ecc_top.cpp` `Load_PCM()` (참고 수준) | 단순 배경 ECC로 사용, 논문이 이 LDPC 자체를 개선하지 않음 |

적용 가치: 낮음 — LDPC ECC 설계/디코더와 무관한 NAND 프로그래밍 전처리(압축+매핑) 기법으로 Prime ECC 코드베이스와 접점이 없음.
