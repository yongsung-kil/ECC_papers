### B. 알고리즘 요약

1. 시스템: 이진 구조적 (γ,ρ)-regular LDPC 두 클래스(finite geometry 기반 EG-LDPC, finite field 기반 QC-LDPC), AWGN 채널 + BPSK, 예시 코드 (961,721) rate≈0.75 γ=ρ=30, (4095,3367) rate≈0.82 γ=ρ=64.
2. 문제의식: 기존 SPA는 성능은 좋으나 실수 연산(tanh 등)으로 복잡도가 크고, majority-logic 계열(WOSMLGD/RBI-MLGD/RBI-MSD)은 복잡도는 낮지만 저비트 양자화(3~4bit)에서 성능 손실이 크다.
3. 핵심 가정: 반복 과정에서 신뢰도(reliability) 정보를 extrinsic message가 아닌 syndrome message로 대체하고, 문턱값을 반복마다 자기조정(adaptive)하면 저비트 양자화에서도 SPA에 근접 가능.
4. 핵심 기법: check node에서 `U_{i→j} = (2s_i-1)w_{i→j}`(syndrome 기반 메시지), variable node에서 proportionality λ로 정의된 적응형 문턱값 `η(j) = round((2λ-1)Σw_{i→j})`와 비교해 갱신 여부 결정.
5. 핵심 식 (18): `R_j^(k+1) = R_j^(0) - sign(R_j^(k))·round(α·Γ_j^(k))` (Γ>η인 경우만), turbo-principle 방식 갱신으로 flipping 대신 reliability를 목표 방향으로 이동.
6. proportionality-logic 확장: λ→1/2이면 (W)OSMLGD와 동일, λ→0이면 모든 VN 처리(RBI-MLGD/RBI-MSD와 동일)하는 일반화된 형태이며 λ<λ*(예: 0.3~0.35)로 최적화.
7. 부수 트릭: 실수 입력을 정수로 clip·양자화하는 함수 `Q(y)=[y/Δ]` (b bit, Δ 파라미터), η를 고정값으로 둬(η=-2^4) 추가 복잡도 절감 가능(성능 열화 0.1dB 이내).
8. 최적화 절차: λ와 양자화 파라미터(Δ, b)는 시뮬레이션 기반 1차원 탐색(one-dimensional search)으로 코드별 선택, scaling factor α는 density evolution으로 최적화 가능.
9. 결과: (961,721) 코드에서 8bit 양자화 시 (W)OSMLGD 대비 2.1~2.5dB 이득, 3bit 양자화로도 SPA 대비 0.26dB 이내 성능이면서 평균 iteration 2.8회(Eb/N0≈3.6dB, SPA는 약 2.0회) 달성.
10. 한계: 하드웨어 미설계(FPGA/ASIC 검증 없음, gatecount/throughput 없음), 큰 column weight를 갖는 구조적(finite geometry/finite field) LDPC에 특화되어 비구조적 random-like irregular LDPC에서는 성능 저하 가능성을 저자가 직접 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 CN 메시지, majority-logic 갱신 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | min-sum이 아닌 majority-logic 계열이라 CN 연산 로직 자체를 교체해야 함 |
| 적응형 문턱값 η(λ) 기반 VN 부분 갱신 | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD) | 일부 VN만 처리하는 proportionality 갱신 방식은 이터레이션 루프 로직 변경 필요 |
| 저비트 양자화 함수 Q(y) | `channel.cpp` `Set_LLR_Th()` | 3~4bit soft quantization 임계값 설계와 관련, LLR 테이블 구조와 연계 가능 |
| iteration별 λ/η 최적화(1차원 탐색, DE) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 반복별 파라미터 테이블 최적화 개념은 기존 적응형 LLR 테이블과 유사 |

> 적용 가치: 중간 — 저비트 양자화에서 SPA 근접 성능과 빠른 수렴은 Prime ECC의 soft-2~3bit 환경에 유효할 수 있으나, min-sum이 아닌 majority-logic 알고리즘 자체를 교체해야 하므로 검증 부담이 크고 HW 미설계로 실비용은 불확실.
