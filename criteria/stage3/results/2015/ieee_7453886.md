### B. 알고리즘 요약

1. AWGN(BPSK) 채널, column weight 3의 irregular LDPC 부호, 코드길이 1470~1764bit, rate 0.9197~0.9246(고rate, NAND ECC 유사 영역).
2. 풀려는 문제: 저 column weight LDPC는 4-cycle이 흔해 반복복호 성능(waterfall)이 저하되는데, 이를 대수적(algebraic) 구성으로 원천 차단하려 함.
3. 핵심 가정: Tanner graph에서 RC-constraint(임의 두 행/두 열이 공통 1을 2개 이상 갖지 않음)를 만족하면 4-cycle이 없다는 그래프 성질을 이용.
4. 핵심 기법: `(7,4)` Hamming code의 dual code 생성다항식(`1+X^2+X^3+X^4`)으로 만든 circulant를 column splitting하여 column weight 2, 4-cycle-free `(14,8)` extended Hamming code(EHC)를 구성.
5. Hybrid 구성 알고리즘: base matrix `H1`(column weight 7, PEG 또는 Gallager 방식)의 각 열의 1-성분을 `(14,8)` EHC의 대응 행으로 치환하고, 끝에 `14×14` 단위행렬을 추가해 최종 column weight 3의 `H=(m+14)×14n` 행렬 획득.
6. 확장/비교: base matrix를 PEG 알고리즘(girth 최대화, PH) vs Gallager 방식(GH) 두 가지로 만들어 성능 비교.
7. 부수 트릭: 단위행렬 추가는 낮은 column weight로 인한 AWGN 반복복호 성능 저하를 보완하기 위한 구현 디테일.
8. 별도 학습 절차는 없음(PEG는 girth 최대화를 위한 그리디 edge-growth 절차 사용, 논문 자체 최적화는 아님).
9. 결과: `(14,8)` EHC는 4-cycle 0개(vs GH 계열 최대 10811개); PH가 Shannon limit 대비 약 1.5dB 이내로 근접, SNR 4dB 이상에서 PH가 GH보다 BER/FER 우수; 평균 반복수는 SNR 낮을 때 GH가 낮지만 SNR 상승 시 PH가 더 적은 반복수로 수렴(정성적 언급, Fig.3 그림 수치는 본문 서술 없어 확정 불가).
10. 한계: HW 설계 없음, SPA(표준 sum-product) 복호만 사용해 디코더 자체 개선은 없는 순수 구성법, QC-LDPC 구조(circulant shift 기반)가 아닌 일반 sparse 행렬 구성.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix 구성(hybrid, (14,8) EHC + PEG/Gallager base) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base+circulant shift, z=32) 고정 구조라 이 논문의 EHC column-splitting 구성법을 그대로 적용하려면 QC 형태로 재설계 필요, 이식 난이도 높음 |
| 4-cycle 제거를 통한 waterfall 개선 | `ecc_top.cpp` `Load_PCM()` | girth 확대(4-cycle 회피)라는 설계 원칙 자체는 QC-LDPC 부호설계에도 참고 가능하나 구체 구성법은 대응 없음 |
| 복호 알고리즘 | 대응 없음 | 표준 SPA만 사용, Prime ECC의 min-sum/HCU/조기종료 등과 무관 |

> 적용 가치: 낮음 — column weight 3, rate>0.9의 고rate LDPC 구성법으로 rate 영역은 NAND ECC와 유사하나 QC-LDPC가 아닌 일반 대수적 구성이며 디코더/HW 개선이 전무해 Prime ECC 이식은 girth 설계 원칙 참고 수준에 그침.
