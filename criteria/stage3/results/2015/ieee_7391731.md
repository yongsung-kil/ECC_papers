### B. 알고리즘 요약

1. 시스템: rate `Rc`, `(dv,dc)`-regular LDPC, AWGN 채널을 `|Y|`(예 2048) 레벨로 세분한 뒤 `|T|`(4bit=16) 클러스터로 양자화하여 디코더 입력으로 사용.
2. 문제: LLR 기반 BP/Min-Sum은 quasi-continuous 연산(비선형 함수, 부호있는 수)이 필요해 HW 구현 부담이 큼 — 목표는 unsigned integer만으로 노드 연산을 대체.
3. 채널/가정: 대칭 이진입력 discrete memoryless channel(AWGN 근사), 메시지는 LLR 순서로 정렬 가능하다는 separating hyperplane 조건 활용.
4. 핵심 기법: Information Bottleneck(IB) 알고리즘으로 각 노드 연산 `gl(yl)`을 두 입력씩 순차 압축하는 discrete density evolution 수행. 목적함수 `min I(Y,T) s.t. I(T,X)≥D`.
5. 핵심 식: `tl = gl(yl)`이며 check node는 `p(y_l^c, x_l^c)`를 `⊕` 기반 재귀(식 16~17)로, variable node는 `+` 기반 재귀(식 18~19)로 갱신 — 정보손실 최소화하며 메시지 폭 고정.
6. 확장: 대칭성 제약 `Pr(Tl=tl)=Pr(Tl=|T|-1-tl)`, `L(xl|tl)=-L(xl|T|-1-tl)`을 추가한 전용 IB 알고리즘(Algorithm 3)으로 대칭 trellis 보장, 부호 비트는 상태 인덱스와 `|T|/2` 비교만으로 결정.
7. 구현 디테일: 노드 연산을 두 정수 벡터(iteration마다 길이 `|T|²(dc-2)` 또는 `|T|²·dv`)로 사전 계산해 저장, 디코딩 시 `tl = a[t_{l-1}|T| + y_{l+1} + offset]` 인덱싱만 수행 — LLR/부호 저장 불필요.
8. 최적화 절차: sequential Information Bottleneck(Slonim) 변형 알고리즘으로 채널 출력 양자화 및 노드 매핑을 오프라인 설계(density evolution 임계 σn² 부근에서 1회 생성 후 고정 사용).
9. 결과: (3,6)-regular, Nv=10^4/10^5 LDPC, |T|=16(4bit)에서 BP 대비 0.4dB 이내 손실, Min-Sum(양자화 채널) 대비 waterfall 영역에서 약 0.3dB 개선.
10. 한계: HW 설계·gatecount·throughput 전무(순수 알고리즘/시뮬레이션), 무작위 (3,6)-regular 코드에만 검증, error-floor 영역 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| discrete node operation(check/variable) trellis 대체 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Min-Sum 연산을 정수 룩업으로 대체하는 대안 알고리즘으로 참고 가능 |
| iteration별 정수 메시지 매핑(오프라인 생성 벡터) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 LLR 테이블 구조와 유사한 형태이나 완전히 다른 설계(IB 기반) 필요 |
| 채널 출력 양자화(soft-read threshold) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | IB 양자화 알고리즘 자체는 대응 없음, 결과물(threshold)만 유사 위치 |
| 부호구조(무작위 (3,6)-regular) | `ecc_top.cpp` `Load_PCM()` | 대응 없음(QC-LDPC 미검증) |

> 적용 가치: 중간 — Min-Sum을 대체할 정수 전용 노드 연산이라는 아이디어는 HW 면적 절감 관점에서 흥미로우나, 부호구조·HW 검증 부재로 Prime ECC의 QC-LDPC/HCU/pipeline 구조에 바로 이식하기 위해서는 상당한 재설계와 재검증이 필요함.
