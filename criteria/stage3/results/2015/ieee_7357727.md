### B. 알고리즘 요약

1. 시스템: `(γ,ρ)-regular` LDPC 부호워드를 불안정한 메모리 셀에 저장, 셀 값은 주기적으로 반복(iterative) 정정회로로 업데이트(Taylor-Kuznetsov 메모리).
2. 문제: 메모리 셀뿐 아니라 정정 회로를 구성하는 로직 게이트 자체도 고장(failure) 날 수 있는 환경에서 정보를 무한히 보존 가능한가(`memory stability`).
3. 채널/오류 모델: 셀 오류는 crossover probability `p_m`의 BSC(soft error)로, 게이트 오류는 von Neumann 모델(독립 확률 `c`) 또는 데이터 종속 GOS 모델(`Pr{ε(k)=1|z(k)≠z(k-1)}=c`)로 별도 정의.
4. 핵심 기법: 세 가지 디코더 기반 메모리 아키텍처 비교 - Bit-Flipping(`Φ^(v)_BF`: majority-vote), Gallager-B(`(γ-1)-input MAJ gate`), Gallager-A(`(γ-1)-input comparator`).
5. 핵심 식: BF 기반 아키텍처의 redundancy `R_BF ≤ (1+D_γ+γ(ρ-2))/(1-γ/ρ)` — `D_γ`는 γ-input MAJ gate 복잡도(Lemma 1로 상한 유도).
6. 확장: expander code 이론(`(γ,ρ,α,δ)-expander`)을 이용해 BF 기반 메모리가 견딜 수 있는 컴포넌트(셀+게이트) 고장 비율의 유한장(finite-length) bound를 유도(정리 2, 3).
7. 부수 디테일: 게이트 지연 오류를 inverse Gaussian 분포로 모델링한 실측 파라미터(Table I) 인용, γ-input MAJ gate를 2-input AND/OR 트리로 분해하는 구현법(Appendix A).
8. 별도 학습/최적화 절차 없음(수학적 상한 유도 + 수치 예시).
9. 결과: (3,6)-regular LDPC 기반 Gallager-A 메모리가 최소 redundancy(약 34배) 달성; girth `g`가 8→14로 늘면 허용 가능한 컴포넌트 오류 수가 다항적으로 증가(예: γ=15일 때 g=12→12개, g=14→20개 오류 허용).
10. 한계: 실제 시뮬레이션·HW 구현 없이 이론적 상한/서베이 위주, NAND 채널(비대칭 read 오류)이 아닌 로직 게이트 고장 모델을 다뤄 표준 ECC 설계와 목표가 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Bit-Flipping / Gallager-A / Gallager-B 디코더 비교 | `decoder.cpp` `LDPC_Decoder()` | 개념적으로만 유사(hard-decision 디코더 계열), 하지만 논문 목적은 게이트 고장 허용이지 정정성능 개선이 아님 |
| 게이트 고장(von Neumann/GOS) 모델 | 대응 없음 | Prime ECC는 게이트 자체의 확률적 고장을 모델링하지 않음(bit-exact 결정론적 시뮬레이터) |
| expander code 기반 유한장 오류 허용 bound | 대응 없음 | H-matrix 설계와 무관한 이론적 fault-tolerance 증명 |

> 적용 가치: **낮음** — NAND 채널 오류 정정이 아닌 "로직 게이트 자체 고장 허용" 문제를 다루는 이론 서베이라 Prime ECC의 목표(채널 오류 정정 성능/HW 효율)와 직접 연결되지 않음.
