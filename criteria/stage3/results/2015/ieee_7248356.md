### B. 알고리즘 요약

1. Regular LDPC 코드(rate 1/2, (3,6)/(4,8)/(5,10) 계열), AWGN 채널, row-layered offset min-sum(OMS) 디코더를 대상으로 함.
2. 문제: CMOS 회로의 propagation delay 변동으로 인한 동기식(synchronous) 설계의 에너지 비효율을 해결하고자 하나, 기존 ANT(algorithmic noise tolerance)류 하드웨어 보상 기법은 별도 저정밀 블록/비교회로가 필요해 면적이 늘고 "최상위 비트에서 먼저 타이밍 위반 발생" 가정이 항상 성립하지 않음.
3. 핵심 모델: 타이밍 위반으로 왜곡된 회로 출력 `Z[t] = Y[t] + D[t]`를 메모리리스(memoryless) 채널로 모델링, deviation `D[t]`는 게이트레벨 회로에서 Monte-Carlo 시뮬로 측정.
4. 제안하는 quasi-synchronous 설계는 보상 회로 없이 타이밍 위반을 그대로 허용하되, 전체 디코더 대신 1-iteration 연산을 수행하는 test circuit(Fig. 2)만 특성화해 대표 삼음.
5. Density evolution(semi-Gaussian 방식)이 타이밍 결함이 있는 디코더에도 여전히 유효함을 보이고, `pe(t+1) = f_γ(pe(t), pe(0))` 형태의 ExIT 함수와 에너지함수 `c_γ(pe(t), pe(0))`를 test circuit에서 동시에 측정.
6. 에너지 최소화 문제를 설계변수 시퀀스 `γ = [Vdd, Tclk, α]`의 동적계획법(trellis, Gear-Shift Decoding 확장)으로 풀어, iteration별로 다른 (전압, 클록, 스케일링) 조합을 선택.
7. HW 친화 디테일: 메시지 6비트 양자화, offset min-sum의 offset 파라미터도 DE 분석 기반 선정, VNP/CNP 파이프라인 2단.
8. 최적화 절차: error probability를 decade당 1000단계로 양자화한 trellis 위에서 dominance pruning을 적용한 최단경로(최소에너지) 탐색, 수 분 내 수렴.
9. 결과: (3,6) rate 1/2 코드에서 동일 성능/레이턴시(pe0=0.12, pres=1e-8, 144ns) 조건에서 표준회로 530pJ 대비 quasi-sync 회로 381pJ로 28% 에너지 절감(간소화 시퀀스로도 25% 절감).
10. 한계: TSMC 65nm 표준 합성툴 기준 게이트레벨 시뮬·정적타이밍분석 결과이며 실리콘 실측 없음, 정정성능 자체는 개선하지 않고 동일 성능을 더 적은 에너지로 달성하는 것이 목표, non-QC regular 코드로만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset Min-Sum row-layered 디코더 아키텍처 | `decoder.cpp/.h` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 min-sum/layered 구조와 동일 계열이나, 본 논문은 알고리즘이 아닌 회로 전압/클록 스케줄링이 기여점이라 로직 이식 대상은 아님 |
| VNP/CNP 파이프라인, 메시지 6비트 양자화 | `decoder.cpp` pipeline 6-stage 모델 | 이미 bit-exact pipeline HW 모델을 보유해 개념적으로 유사하나 quasi-sync 전압/타이밍 기법은 RTL/실리콘 레벨이라 C++ 시뮬레이터 범위 밖 |
| iteration별 (Vdd,Tclk,α) 파라미터 시퀀스 (에너지 최적화) | `ecc_data.h` `PARAM_LLR` (iteration별 LLR 테이블과 유사한 개념) | 개념적으로 "iteration별 파라미터 스케줄"이라는 점만 유사, 실제로는 전력/타이밍 도메인이라 대응 없음 |
| 대상 없음(정정능력·부호설계 관련 기법 아님) | 대응 없음 | H-matrix/부호설계, 디코딩 알고리즘 자체 변경 없음 |

> 적용 가치: 낮음 — 정정성능이 아닌 실리콘 합성 단계의 전압/클록 오버스케일링 에너지 최적화 기법으로, bit-exact C++ 시뮬레이터인 Prime ECC 3.1의 판정 범위(알고리즘/HW 아키텍처) 밖이며 특정 공정·합성툴에 종속적.
