### B. 알고리즘 요약

1. Layered QC-LDPC 디코더(WiMAX rate-1/2 irregular `dv=3`, regular `dv=3/4` rate 1/2·3/4)를 대상으로, base matrix `B`와 circulant 치환행렬로 구성된 QC 구조 위에서 layered scheduling(층별 순차 처리, 층 내부는 병렬) 기반 Min-Sum(및 Self-Corrected MS) 디코더를 다룸.
2. 문제의식: LDPC 디코더 아키텍처 탐색 시 파라미터(병렬화도, 양자화, 파이프라인 단수 등)를 바꿀 때마다 RTL/테스트벤치/합성 스크립트를 수작업으로 재작성해야 하는 비효율.
3. 핵심 가정: 코드 속성(base matrix, circulant 크기, 양자화, 알고리즘)과 아키텍처 파라미터(병렬화도 `p`, 코드워드 수, 파이프라인 단수, iteration 수)를 분리해 템플릿화 가능하다고 가정.
4. 핵심 기법: Verilog HDL에 smart tag를 결합한 템플릿 기반 자동 생성 툴로, VCN(Variable-Check Node) 처리블록·AP-LLR 메모리/라우팅망·check-node 메시지 메모리 3대 모듈을 계층적 템플릿(디코더 전체/컴포넌트/기본블록)으로 정의.
5. 핵심 식: `α_{z,n} = γ_n - β_{z,n}` (VN 메시지), `β_{z,n} = (∏sgn(α))×min(|α|)` (CN 메시지, Min-Sum), `γ_n = α_{z,n}+β_{z,n}` (AP-LLR 갱신) — 표준 layered MS 반복 갱신식.
6. 부수 트릭: β 메시지를 압축형(`β_comp`: 최소값 2개+인덱스+부호)으로 저장해 메모리 폭 절감, barrel shifter 파이프라이닝으로 라우팅 포트 제약 완화.
7. 확장 기법: multi-codeword decoding — 파이프라인 지연으로 인한 "delayed update" 문제를 여러 코드워드를 시차 처리해 해결(BRAM 활용률 향상, throughput/주파수 저하 없음).
8. 학습/최적화 절차 없음(설계 자동화 툴, DE/GA/NN 미사용). 대신 Greedy 알고리즘 기반 메모리 뱅크 충돌 회피 스케줄링(ROM 주소 생성)을 사용.
9. 결과: Virtex-7 FPGA 합성 결과, WiMAX(N=2304, m=96) r=1/2 P=3에서 91050 LUT-FF pairs/37 BRAM/178MHz/1318Mbps; 기존 수작업 설계([12][13]) 대비 동등~우수한 throughput/BRAM 효율 달성.
10. 한계: 디코딩 알고리즘 자체(MS/SC-MS)는 기존 기법 재사용, 정정능력/error-floor 관련 실험 없음(순수 HW 설계자동화/합성결과 위주), NAND 특화 실험 없이 WiMAX/일반 표준 코드만 대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered QC-LDPC decoder HW 아키텍처(VCN, 병렬화도 p) | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC도 row 단위 병렬(z_sb=32) layered 유사 구조를 가지나, 이 논문은 파라미터화 생성 툴이라 이미 고정된 Prime ECC 아키텍처에 직접 대응되지 않음 |
| AP-LLR 메모리·barrel shifter 라우팅망 | `decoder.cpp/.h` (pipeline, HCU) | barrel shifter 기반 라우팅/직렬화 기법은 Prime ECC의 memory routing 설계 시 참고 가능한 개별 아이디어 |
| Check-node message 압축 저장(`β_comp`: min1/min2+idx+sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 이미 min1/min2+sign 기반 압축 저장을 사용 중이라 중복 기여(신규성 낮음) |
| Multi-codeword 파이프라인 은폐 기법 | `decoder.cpp` pipeline 6-stage 모델 | 파이프라인 지연 은폐 아이디어로 Prime ECC의 6-stage pipeline과 개념적으로 유사, 신규 이식 요소는 제한적 |
| 템플릿 기반 RTL 자동생성 툴체인 자체 | 대응 없음 | Prime ECC는 특정 QC-LDPC로 고정된 시뮬레이터라 파라미터 생성 프레임워크 자체는 이식 대상이 아님 |

> 적용 가치: 낮음 — 디코딩 알고리즘 개선이 아닌 설계 자동화 툴/프레임워크 기여이며, 개별 HW 요소(압축 저장, 파이프라이닝)는 Prime ECC가 이미 보유하고 있어 중복. barrel shifter 라우팅 직렬화 기법만 제한적으로 참고 가치.
