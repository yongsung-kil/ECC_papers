> 총평: HDD/자기기록 채널용 4kB QC-LDPC + single-parity outer code product 구조로, min-sum 기반 row-code soft LLR을 실패 섹터 재복호 시 사전정보로 주입해 iteration 감소와 SNR 이득을 얻는 기법. inner code가 QC-LDPC라는 점에서 부분적으로 이식 가능하나 채널·시스템 전제가 다름.

### B. 알고리즘 요약

1. HDD 자기기록 채널(T50/T=1, 80% media noise + 20% electronic noise)에서 4kB QC-LDPC(rate 0.8782)를 column(inner) code `C1`, single-parity check code를 row(outer) code `C2`로 하는 product code 구성.
2. 문제: 4kB 이상으로 LDPC 블록을 키우면 SNR 이득은 있지만 실리콘 면적/전력 비용이 커, 블록 확장 없이 SNR 이득을 얻는 것이 목표.
3. 채널 모델은 표준 자기기록 디스크 채널(soft-output-Viterbi/MAP 검출기 + turbo iteration), RLL 변조 제약 하 outer code 인코딩은 비선형 제약을 보존하지 못함.
4. 핵심 기법: 컬럼 LDPC `C1` 디코딩에 실패한 섹터만 골라 row 부호 `C2`의 syndrome 제약식(식 1, 2)으로 부분 syndrome을 계산하고, min-sum 원리로 row-code soft 정보 `L_{k,j}(C2)`를 산출(식 3, 4: min(|L_{i,j}(C1)|) 및 부호 XOR).
5. 식 (3),(4)의 의미: 실패 섹터를 제외한 나머지(수렴한 섹터의 syndrome + 다른 실패 섹터들)의 LLR로 각 실패 섹터의 사전 확률정보를 만들어 재복호에 활용 — single-parity라 1회 반복만으로 충분.
6. 확장: row-code soft 정보를 실패 섹터의 detector 입력에 추가 prior로 주입하여 detector BER을 낮추고 컬럼 LDPC 수렴 가능성을 높임(product code turbo iteration 최대 3회 허용).
7. 구현 디테일: row-code soft 정보 계산은 첫 turbo iteration의 detector LLR 출력을 사용(마지막 iteration 대비 신뢰도는 낮지만 latency 감소), 메모리 병목은 실패 섹터 수(최대 6개) 제한한 equalized sample 버퍼링과 partial syndrome/최소 LLR 저장.
8. 별도 학습/최적화 절차 없음 (구성법 기반, DE/GA/NN 미사용).
9. 결과: 무제한 메모리 가정 시 ∼0.7 dB SNR 이득(sector failure rate 기준), 실용 조건(최대 6개 실패 섹터 버퍼, 1회 product 반복)에서 ∼0.3 dB 이득 @ 1e-5; 실패 섹터의 컬럼 LDPC 평균 turbo iteration이 크게 감소(Fig. 5).
10. 한계: 컬럼 LDPC 재기록 시 read-modify-write 오버헤드 발생(shingled recording 제외), 매우 낮은 SNR(<11.95dB)에서는 실패 섹터 과다로 메모리 부족 및 코드율 차이로 인해 기존 4kB LDPC 단독 대비 성능 역전.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 기반 row-code soft 정보 계산(식 3,4) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum min1/min2+sign XOR 연산과 원리적으로 동일한 min-sum 근사를 outer 부호에 적용 |
| 실패 섹터 재복호 시 사전 LLR 주입 | `decoder.cpp` `VN_Cal_HD()` 등 VN 처리 / `ecc_data.h` `PARAM_LLR` | 외부 정보를 VN 초기 LLR에 prior로 결합하는 구조로 LLR 테이블 확장 여지 |
| 4kB QC-LDPC inner code (rate 0.8782, quasi-cyclic) | `ecc_top.cpp` `Load_PCM()` | inner code 자체는 고rate QC-LDPC로 부호 구조 유사하나 특정 H-matrix 재검증 필요 |
| single-parity check outer/product code 구조 | 대응 없음 | Prime ECC는 단일 QC-LDPC 디코더로 outer product-code 계층 개념 없음 |
| 실패 섹터 버퍼링/메모리 관리 | 대응 없음 | HW 메모리 아키텍처 상세 미기재, 코드 대응 모듈 불명확 |
```

적용 가치: **중간** — inner code가 QC-LDPC이고 min-sum 기반 외부 정보 주입으로 iteration 감소·SNR 이득을 얻는 기법 자체는 디코더 레벨에서 참고 가치가 있으나, outer product-code 계층(다중 섹터/트랙 단위) 구조는 NAND 페이지/블록 단위 시스템과 전제가 달라 시스템 레벨 재설계가 필요.
