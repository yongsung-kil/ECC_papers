### B. 알고리즘 요약

1. 시스템: 근본적으로 근사연산(approximate computing) 프레임워크로, 두 응용 중 하나가 min-sum LDPC 디코더(irregular (3,6)-regular 코드, length 2640, B=5비트 sign-magnitude 양자화)의 variable-node 메시지 저장 메모리.
2. 문제: 나노 CMOS 공정 스케일링으로 임베디드 메모리(SRAM/레지스터)가 태생적으로 신뢰성이 낮아지는데, 기존 ECC 보호는 추가 메모리 셀(오버헤드)이 필요함.
3. 채널모델: 메모리 셀의 각 비트는 독립적인 binary symmetric channel(BSC, cross-over probability ρ)을 통과하는 것으로 모델링(von Neumann bit-flipping model); 통신 채널은 별도로 AWGN(σ) 가정.
4. 핵심 기법: ACOCO(Adaptive Coding for approximate Computing) — 손실 있는 source encoder로 비트를 줄이고, 그만큼의 비트를 channel code 중복비트로 재사용해 "추가 메모리 없이" sign bit/MSB를 보호. min-sum 디코더용 cost function `f(xs,xo)=1(|xs|<τ)1(xs≠xo)+1(|xs|≥τ)[(xo-xs)²+λ|sign(xo)-sign(xs)|]`.
5. 식의 의미: min-sum 디코딩은 sign bit 오류(부호반전)가 결정오류로 직결되고 작은 크기(|xs|<τ) 메시지는 모든 비트가 동등히 중요하므로, 메시지가 τ 이상(큰 확신도)일 때만 sign bit를 (3,1) 반복코드로 보호하고 LSB 2비트를 희생.
6. 확장: density evolution 분석으로 linear transformation `p^(ℓ)=H·A(ρ)·G·p̂^(ℓ)` 유도(G=인코딩 매핑, H=디코딩 매핑, A(ρ)=BSC 전이행렬)해 잔여오류율(residual error rate)을 이론적으로 예측.
7. 구현 디테일: sign-magnitude 표현(B=5비트), 메시지 크기 τ 임계값으로 "큰 값"과 "작은 값" 분기 처리, (3,1) 반복코드로 sign bit 다수결 복원.
8. 별도 학습 절차 없음(cost function 기반 결정론적 코드 설계, density evolution으로 이론 검증).
9. 결과: 저잡음(σ) 영역에서 잔여오류율이 무보호(NC) 대비 제곱 오더로 감소(`p_r^AD ≈ (p_r^NC)²`), decoding threshold가 무잡음 디코더(0.811)에 근접(AD: 0.803 vs NC: 0.798 vs QC: 0.781); finite-length(N=2640) 시뮬레이션으로도 확인.
10. 한계: HW 합성/게이트카운트 없이 이론(density evolution)+시뮬레이션만 검증. 대상은 min-sum 알고리즘 자체가 아니라 그 디코더가 올라가는 메모리의 신뢰성이며, 실제 NAND 저장 채널 오류가 아닌 디코더 내부 트랜지언트 하드웨어 오류를 다룸.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 디코더 variable-node 메시지 저장(내부 RAM) | `decoder.cpp` `LDPC_Decoder()` (V2C/C2V 메시지 버퍼) | 디코더 내부 메시지 저장 구조가 개선 대상과 대응, 단 Prime ECC는 bit-exact 시뮬레이터라 실제 트랜지언트 오류 모델 없음 |
| sign bit 우선 보호(반복코드) | `decoder.h` `Get_VNU_*`, LLR 부호/크기 표현 | sign-magnitude 양자화 구조가 유사하여 개념 이식은 가능하나 추가 인코딩/디코딩 로직 필요 |
| 메시지 크기 기반 적응적 보호(τ 임계값) | `ecc_data.h` `PARAM_LLR` (양자화 테이블) | LLR 임계값 테이블 구조와 유사한 형태로 참고 가능 |
| density evolution 분석 | 대응 없음(오프라인 분석 도구) | Prime ECC는 시뮬레이터이며 별도 이론 분석 프레임워크 없음 |

> 적용 가치: 중간 — NAND 저장 채널 자체 개선은 아니지만, 디코더 HW 내부 메모리(SRAM/레지스터)가 저전압·공정변이로 신뢰성 저하될 경우 대비한 저오버헤드 보호기법으로 참고 가치가 있음. 단 min-sum 알고리즘 자체 개선은 아니라 우선순위는 낮음.
