### B. 알고리즘 요약

1. 시스템: IEEE 802.11n(2012) 표준 QC-LDPC 코드(circulant-1 identity submatrix 구성), base matrix `mb×nb = 12×24`, `z = 27/54/81`, rate 1/2, code length `n = 24z = 648/1296/1944`.
2. 문제: 5G/mm-wave급(수 Gbps, latency<1ms) 요구에 맞는 FPGA 기반 고처리량·재구성 가능 QC-LDPC 디코더가 필요하나 기존 FPGA 구현은 fully-parallel(코드 특화·routing congestion) 위주.
3. 핵심 가정/모델: AWGN + BPSK, 6-bit(1 sign + 부호 포함 총 6bit, 4 fractional bit) 고정소수점 LLR 양자화.
4. 핵심 기법: scaled min-sum algorithm(scaled-MSA), CTV 메시지 `r_ij = a·sign(Π q)·min|q|` (스케일 팩터 `a=0.75`)에 row-layered scheduling 적용. 노드처리를 Global NPU(1st/2nd min 계산)와 Local NPU(overall min 계산)로 분리해 `O(d_c)` 선형 복잡도 달성.
5. 핵심 식: PCM `H`를 base matrix `H_b`(shift값 `s∈{-1,0,...,z-1}`)로 표현, submatrix `I_s` z개 CN이 겹치는 VN 없이 병렬 처리 가능(Fact 1) → z-fold NPU array 병렬화.
6. 확장: PCM을 layer×block으로 분할 후, invalid(-1) block을 제외한 valid block만 담는 compact 표현 `β_I`(index)·`β_S`(shift) 도입. compaction ratio `λ = J/n_b`(사례 λ=1/3)만큼 처리할 block 수 감소 → throughput 1/λ배(사례 3배) 잠재 이득.
7. 부수 트릭: `β_I` 재배열로 2개 layer를 GNPU/LNPU 파이프라인으로 동시 처리(superlayer 내 `|L|`은 pipelining efficiency `η_p=|L|/(|L|+1)`를 최대화하는 짝수 인수 선택, 사례 `|L|*=6`), 추가 HW 자원 없이 2x 파이프라이닝 달성.
8. 학습/최적화 절차: 없음(고정 코드/구조, NI LabVIEW CSDS FPGA IP 컴파일러로 알고리즘 레벨 설명을 자동 VHDL 변환).
9. 결과: Xilinx Kintex-7(k410t)에서 2x(파이프라인) 버전 throughput 608Mb/s@260MHz, latency 5.7μs, 1x 대비 1.8배 처리량 향상에도 자원 사용량은 유사(FF 9.1%→5.3%, BRAM 4.7%→6.4%, LUT 8.7%→8.2%); 고정소수점 vs 부동소수점 BER 차이 약 0.5dB.
10. 한계: 코드가 IEEE 802.11n(rate 1/2) 표준에 국한된 사례연구, HW 정정능력 개선이나 error-floor 대책 없음(순수 아키텍처/throughput 최적화), 파이프라이닝 상한 유도가 base matrix 구조(mb=12)에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Scaled min-sum(CTV 계산, 스케일 팩터 a=0.75) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이나 scaling factor 명시 미확인 — 스케일링 값 튜닝 참고 가능 |
| Layered decoding(row-layer 순차 처리) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 iterative 구조와 유사, layered 자체는 이미 보유한 개념과 중복 |
| GNPU/LNPU 분리 + PCM compact 표현(βI/βS)을 통한 2-layer 파이프라이닝 | 디코더 HW 모델 (`decoder.cpp` pipeline 6-stage, z=32 HCU/GT 구조) | 파이프라이닝 깊이 확장 아이디어로 참고 가능하나 이미 6-stage pipeline+GT/HCU로 유사 최적화 보유, 신규성은 제한적 |
| z-fold NPU array 병렬화(circulant 기반) | `decoder.cpp` 병렬 row 처리(z_sb=32) | Prime ECC의 lifting 병렬 단위(z_sb=32)와 동일 개념, 이미 구현됨 |
| PCM 로드/base matrix(circulant-1 identity) | `ecc_top.cpp` `Load_PCM()` | 802.11n 표준 코드 구조이며 Prime ECC의 고정 QC-LDPC 구조와 별개 부호이나 로딩 방식은 유사 |

적용 가치: **중간** — layered scaled-MSA와 z-fold 병렬화는 Prime ECC가 이미 보유한 구조(z_sb=32, pipeline)와 개념이 겹치나, PCM의 invalid block을 skip하는 compact index 표현(βI/βS)과 GNPU/LNPU 파이프라이닝 재배열 기법은 메모리·타이밍 최적화 아이디어로 참고할 만함.
