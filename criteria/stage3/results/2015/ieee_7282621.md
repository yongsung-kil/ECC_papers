### B. 알고리즘 요약

1. 대상은 column-weight `dv=3`, girth `g=8`인 regular QC-LDPC 부호(binary), PEG(Progressive Edge-Growth) 계열 구성 알고리즘 기반.
2. 문제: LDPC error-floor는 Tanner graph 내 작은 trapping set(TS)에서 기인하는데, 기존 PEG/RandPEG는 girth만 최대화할 뿐 TS를 직접 회피하지 않음.
3. 채널/모델: elementary trapping set `(a,b)` 정의(Richardson), `dv=3,g=8`에서 가장 작은 TS인 `(5,3,8^3)`, `(6,4,8^2,12)`, `(6,4,8,10^2)`을 조합론적으로 특성화.
4. 핵심 기법: PEG의 depth-k computation tree에서 check node의 복사본(copy)이 공통 조상 변수노드 개수 `n = |A(c1)∩A(c2)|`를 만족하면 특정 TS가 형성됨을 정리(Theorem 5~7)로 증명.
5. 핵심 식: 두 경로의 사이클 길이 `g = l1+l2+6-4n`(예: l1=l2=7일 때 n=3 → g=8 → (5,3) TS 발생)로 TS 발생 조건을 판별.
6. 확장 기법: RandPEG(Venkiah et al.) 비용함수에 TS 카운트(`nbtrapping[0]`=(5,3) 개수, `nbtrapping[1]`=(6,4) 개수)를 추가 — (5,3) 만드는 후보 check node는 완전 제거, (6,4) 최소화 후보만 선택.
7. QC-LDPC 특화: 변수노드 `v=nL`(L=순환행렬 크기) 단위로 computation tree를 확장하고 한 PEG step에서 순환행렬 전체(L개 엣지)를 동시 할당.
8. 최적화 절차: greedy 방식(RandPEG 확장), 학습/DE/GA 없음 — 매 스텝 후보 check node 집합에서 TS 유발 후보를 제거하는 결정론적 필터링.
9. 결과: L=31, dc=5 예제에서 기존 RandPEG 대비 (5,3) TS 31→0, (6,4,8²,12) TS 930→434로 감소(Table I); BPSK-AWGN 시뮬에서 waterfall 기울기 개선 확인(Fig.3,4).
10. 한계: 짧은 코드길이(L=18/31/52)로 error-floor 영역 자체는 시뮬레이션에서 관측 못함(waterfall 기울기 개선만 확인), HW 미설계, dv=3/g=8 특정 케이스에 한정된 증명.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC H-matrix/PCM 구성(PEG 기반 trapping set 회피) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC H-matrix 고정 로드 방식이라 부호 재구성 자체는 이식 난이도 높음(§4 표 "부호설계"=높음), 다만 error-floor 개선을 위한 신규 H-matrix 후보 생성 시 참고 가능 |
| trapping set (5,3)/(6,4) 조합론적 검출 | 대응 없음 | 디코더 실행시 아닌 부호 설계 단계(오프라인) 기법으로 decoder.cpp의 런타임 로직과는 무관 |

> 적용 가치: 중간 — 새 H-matrix를 설계할 기회가 있을 때 error-floor 완화용 참고 기법이 될 수 있으나, Prime ECC의 고정 QC-LDPC 구조 재검증 부담이 커서 즉시 이식보다는 향후 부호 재설계 시 참고 가치.
