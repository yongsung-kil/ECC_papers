### B. 알고리즘 요약

1. WLAN(648,324)/(1296,864), Wi-MAX(2304,1152) regular LDPC, AWGN/BPSK 채널, rate 1/2~2/3.
2. 기존 SPA는 tanh/arctanh 연산으로 계산복잡도가 매우 높아 HW 구현에 부적합, MSPA/SSPA는 양자화로 복잡도는 낮췄지만 BER 성능이 저하됨.
3. 채널은 AWGN, 수신값 `y_n = x_n + w_n` (w_n: 가우시안 잡음, 분산 `σ² = (2R_c·E_b/N_o)⁻¹`).
4. Check node에서 tanh/arctanh 대신 FFT with time-shift(FFTh)를 적용: `r_mn = sign(·)·FFTh⁻¹[FFTh(2|y_i|/σ²)]` (식 10).
5. FFTh는 입력을 순환 시프트한 뒤 FFT를 적용하는 변형으로, 일반 FFT/tanh 대비 연산시간·메모리 요구량이 적음.
6. Variable node update에 최적 정수 상수 `X_i`를 추가(`X_i = M/2−1` if `m_i=0`, `(M−1)/2` if `m_i=1`)해 FFTh로 인한 성능저하(>0.1dB)를 보상(식 11~14).
7. `X_i`는 휴리스틱 시뮬레이션으로 찾은 값이며, 초기 LLR 값은 매 반복마다 갱신되어 재사용됨.
8. 별도 학습/최적화 절차는 없음(정수 상수는 시뮬레이션 기반 수작업 튜닝).
9. BER 10⁻⁴~10⁻⁵ 기준 SPA 대비 0.02~0.04dB 열화, FFT-SPA/SSPA/MSPA 대비 0.04~0.46dB 이득; alphabet size 2/4/8에서 SPA 대비 산술연산 42%~67% 감소.
10. HW 구현/합성 없음, MATLAB 시뮬레이션에 한정, 20회 고정 iteration만 평가(조기종료·복잡한 채널모델 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FFTh 기반 check node 갱신 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Min-Sum 기반 CNU와 알고리즘 계보 다름(full tanh/FFT 대체), 직접 대응 아님 |
| variable node 정수 상수 보정 | `decoder.cpp` `VN_Cal_HD()` 등 | LLR 보정 아이디어 자체는 유사하나 Prime ECC는 min-sum 기반 magnitude 양자화 테이블 사용 |
| 산술연산 수 비교(alphabet size별) | 대응 없음 | HW gatecount/throughput과 직접 연결되는 근거 없음 |

적용 가치: 낮음 — SPA/FFT 계열은 Prime ECC의 Min-Sum HW 구조와 근본적으로 다르고, HW 검증도 없어 이식 실익이 적음.
