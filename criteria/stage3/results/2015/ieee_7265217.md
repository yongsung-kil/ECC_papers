### B. 알고리즘 요약

1. LDPC(수평코드) n개를 쌓아 만든 codeword matrix에 SPC 또는 Hamming 코드를 수직코드로 추가한 simple product code 구조를 사용(`P=C⊗C1`, `H`: 수직코드의 parity-check).
2. 문제: finite-length LDPC는 고SNR 영역에서 error-floor가 발생, 기존 post-processing 기법들은 알고리즘만 바꾸고 코드 구조는 그대로 둠.
3. 채널모델: 표준 LDPC AWGN 채널, LLR `r_ij`를 사용하는 soft-decision 수신.
4. 핵심 기법: 먼저 일반 LDPC 복호를 각 수평코드워드에 순서대로 적용, 그룹 내 word error 없으면 지연 없이 다음으로 진행. 실패(e개)가 있으면 수직코드의 확장 parity-check `H_E`(모든 nonzero 선형결합) 중 punctured `H_P`(실패 코드워드에 대응하는 열만 선택)에서 최소 weight `e_min` 행을 찾아 재복호 순서를 정함.
5. 핵심 식: LLR 결합 연산 `r_ij ⊕ r_i'j = 2·tanh⁻¹(tanh(r_ij/2)·tanh(r_i'j/2))` — 동일 코드워드에 대해 독립적으로 두 번 받은 것처럼 취급해 soft 정보를 합성(H-ARQ type-I 유사, 약 3dB 이득).
6. `e_min=1`이면 단순 XOR 정정, `e_min=2`이면 두 개의 독립 LLR을 합해 재복호(식(6)), `e_min≥3`이면 실패 코드워드를 두 그룹으로 나눠 같은 방식 반복 적용.
7. 부수 기법: SPC/Hamming을 수직코드로 쓸 때 "combined-decodability"(ε-combinable, η combined-decodable) 개념을 정의·증명해 재복호 성공 가능 조건을 수학적으로 특성화(SPC=2, (7,4)Hamming=5).
8. 별도 학습절차 없음(구성적/조합론적 분석).
9. 결과: IEEE 802.16e LDPC(R=1/2, n=2304 및 R=5/6, n=1152)에 (24,23) SPC 수직코드(puncturing rate 1/24)를 적용해 고SNR 영역에서 기존 LDPC 대비 WER 대폭 개선(waterfall+error-floor 동시 개선), 저SNR에서는 추가 연산 부담이 있으나 고SNR에서는 무시 가능.
10. 한계: HW 설계 없음(시뮬뿐), 재복호 시 지연(decoding delay) 및 추가 메모리/버퍼 필요, undetected error가 있으면 성능 저해, product decoding(turbo iteration) 대비 성능은 낮지만 메모리·연산·지연은 훨씬 적음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 실패 코드워드 재복호(soft LLR 결합, HARQ 유사) | `decoder.cpp` `LDPC_Decoder()` | 조기종료 실패 시 후처리 단계로 이터레이션 루프 뒤에 추가 가능하나, 다중 코드워드 버퍼링이 필요해 구조 변경 폭이 큼 |
| 수직코드(SPC/Hamming) product 구조 | 대응 없음 | Prime ECC는 단일 QC-LDPC PCM 구조이며 product code 개념 없음 |
| 조기종료 판정(word error 카운트) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 그룹 단위 실패 판정 아이디어는 CRC 기반 조기종료와 개념적으로 유사하나 코드는 별개 |
| LLR soft 결합 연산(`⊕`) | `channel.cpp` `Set_LLR_Th()` | LLR 테이블/양자화 구조에 결합 연산을 추가하려면 별도 로직 필요, 대응 약함 |

> 적용 가치: 중간 - error-floor 개선을 위한 post-processing 아이디어(실패 코드워드만 골라 soft 정보 재결합)는 개념적으로 유용하나, product code 구조·다중 코드워드 버퍼링이 Prime ECC의 단일 QC-LDPC 파이프라인과 맞지 않아 이식 시 구조 변경 부담이 큼.
