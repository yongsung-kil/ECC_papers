> 총평: LDPC Tanner graph의 짧은 사이클을 세는 오프라인 알고리즘 개선(속도 2배, QC 코드는 p배 추가)으로, 디코더/HW와 무관한 부호설계 검증 유틸리티 성격의 논문.

### B. 알고리즘 요약

1. 기존 Karimi-Banihashemi의 message-passing algorithm(MPA)은 girth `g`인 이분그래프(Tanner graph)에서 길이 `2k` (`g ≤ 2k ≤ 2g-2`)의 짧은 사이클을 정수 덧셈/뺄셈 메시지로 세되, `k`번 반복(2k 타임슬롯)이 필요해 복잡도 `O(gE^2)`.
2. 개선된 IMPA는 사이클을 이루는 두 방향 메시지가 그래프상 중간 지점(시간 `t=k-1`)에서 서로 다른 edge로 만난다는 관찰을 이용해, `k` 타임슬롯만에 길이 `2k` 사이클 존재를 검출.
3. 메시지는 정수 벡터(모노미얼) 형태로 표현되며, 노드 `u`의 두 edge를 공유하는 경로 쌍을 조합해 식 (1)-(4)로 사이클 수 `N(u_2(t+1))`, `N(2(t+1))`를 계산.
4. IMPA는 MPA 대비 반복 횟수를 절반으로 줄여 계산시간·복잡도를 약 2배 절감, 메모리는 `min(d_U, d_W)·E` (MPA의 `2·d_U·E` 대비 절감).
5. QC-LDPC 코드(circulant permutation matrix/zero matrix로 구성된 H)에 특화한 QC-IMPA는 각 열 블록(U^(r)) 내 노드들이 circulant 구조상 사이클 수가 동일함을 이용해 대표 노드 1개만 계산, 복잡도를 circulant 크기 `p`배 추가 절감(`O(gE^2/p)`, row-block cyclic 구조면 `O(gE^2/n)`).
6. 순차 구현은 계산 완료된 노드를 그래프에서 제거해 메모리·계산량을 점진 축소하나, 병렬 구현은 전체 그래프를 유지해야 하므로 속도는 빠르지만 계산복잡도는 더 큼.
7. 실험: PEG 기반 (504,1008) irregular/regular 코드, (504,252)/(2640,1320) QC-LDPC, IEEE 802.11n의 (648,486) rate-3/4, (648,540) rate-5/6 QC-LDPC 코드에 적용.
8. 결과: IMPA는 MPA 대비 약 2배 빠름(예: Code IV 919.51s vs 2028.60s), QC-IMPA는 QC 코드에서 IMPA 대비 수백 배 빠름(예: Code IV 0.40s vs 919.51s), 기존 protograph 고유값 기반 알고리즘[5]보다도 대체로 빠름.
9. 한계: 디코딩 알고리즘이 아니라 부호 설계/분석 단계에서 오프라인으로 그래프의 girth·사이클 분포를 검증하는 도구이며, HW 구현이나 실시간 복호 성능과는 무관.
