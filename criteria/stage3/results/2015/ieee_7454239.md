### B. 알고리즘 요약

1. (j,k)-regular QC-LDPC, `L=256`(circulant 크기), (3,6) 코드로 검증, Normalized Min-Sum Algorithm(NMSA), 8-bit quantization, partially-parallel 아키텍처.
2. 문제: 기존 partially-parallel QC-LDPC FPGA 디코더들은 hard-decision(1비트) 저장에 별도 메모리 뱅크를 추가로 써서 BRAM 낭비가 큼.
3. 핵심 가정: VNU(variable node update) 출력의 부호(sign)가 곧 hard-decision이므로 이를 extrinsic 정보(Q비트)와 합쳐 `(Q+1)`비트로 같은 메모리 주소에 저장 가능하다는 관찰.
4. 핵심 기법: `Ms,t`(extrinsic, j×k개), `Ft`(intrinsic, k개) 두 종류 메모리에 각각 hard-decision 비트를 최상위 비트로 결합(`combine`)해 저장, CNU와 PCU가 같은 메모리·같은 주소를 공유해 읽음(`separate`로 분리 추출).
5. 핵심 식 `r = 1/((1+j)/(j/2)+1)`, `1/5 ≤ r ≤ 1/3` — 기존 대비 절감률 r이 체크노드 degree `j`에 따라 20~33% 범위임을 도출.
6. 확장: PCU(parity check unit)가 CNU와 동일 주소 생성 메커니즘을 공유해 별도 주소생성기 불필요 → 추가 로직 절감.
7. 구현 디테일: 8-stage 파이프라인(기존 6-stage 대비 critical path 단축), 대안으로 register/distributed RAM 기반 저장도 검토했으나 slice 자원 소모가 커서 BRAM 결합 방식이 최적이라 결론.
8. 학습/최적화 절차: 없음(구조적 설계, 수식 유도).
9. 결과: reference [6](Non-OMP, j=3) 대비 메모리 뱅크 24개 vs 36개(33% 절감), critical path 4.739ns vs 6.802ns, throughput 84.4 vs 58.8(단위 미상, 상대비교), 평균 iteration 3.0으로 동일(정정성능 변화 없음).
10. 한계: (3,6) regular 저rate(0.5) 코드 한 조합만 검증, error-floor/waterfall 등 정정성능 실험 없음(순수 HW 자원 절감 논문), j가 커지는 고차수 코드에서는 절감률이 감소(최소 1/5).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| hard-decision을 extrinsic/intrinsic 메모리에 결합 저장 | `decoder.cpp` `VN_Cal_HD()`, `decoder.h` `Get_VNU_*` | VN 출력 hard-decision을 별도 저장하지 않고 기존 메시지 버퍼에 결합하는 메모리 절약 기법으로 직접 이식 검토 가능 |
| CNU/PCU 주소생성기 공유(조기종료 syndrome 계산과 결합) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `decoder.cpp` `C2V_Cal()` | 파리티체크(syndrome)와 CNU가 동일 메모리·주소 공유하는 방식은 Prime ECC의 조기종료·CNU 로직에도 참고 가능 |
| Q-bit 양자화 메시지 저장 구조 | `ecc_data.h` `PARAM_DEC`, `decoder.cpp` 메시지 버퍼 | Q+1비트 결합 저장 방식은 Prime ECC의 min-sum magnitude 저장 구조에 참고 가능 |

적용 가치: **중간** — 정정성능과 무관한 순수 메모리(BRAM) 절감 HW 트릭으로 Prime ECC의 decoder.cpp 메시지 저장 구조에 낮은 난이도로 이식 검토 가능하나, NAND ECC의 고rate·비대칭 degree(HCU) 구조에서 절감폭 재검증 필요.
