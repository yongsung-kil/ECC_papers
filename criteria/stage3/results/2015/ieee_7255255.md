### B. 알고리즘 요약

1. 시스템은 광통신 링크(장거리 optically-amplified 또는 단거리 unamplified 데이터센터 인터커넥트)로, LDPC(32768,26803, rate 0.818) 및 검증용 LDPC(672,588, rate 0.875) 두 코드로 성능 비교.
2. 문제의식은, 저비용 저해상도(2~3bit) ADC 기반 soft-decision FEC 수신기에서 multi-branch(hybrid/optical fan-out) 구조가 표준 single-branch보다 어떤 이득/손실을 갖는지 사전에 분석할 방법론이 없었다는 점.
3. 채널/잡음 모델은 광송신기 RIN, 채널 ASE beating noise, 수신기 shot/thermal noise를 branch별로 공통(common) 잡음과 비상관(uncorrelated) 잡음으로 분리해 Gaussian PDF로 모델링.
4. 핵심 개념은 "inconsistent thermometer code": hybrid/optical fan-out처럼 신호가 여러 photoreceiver로 분리되면 비상관 잡음 때문에 D1,D0,D-1 3개 판정 회로 출력이 표준 thermometer 패턴([000]~[111])을 벗어난 조합(예: [101])이 발생.
5. 이 inconsistent code들도 버리지 않고 `LLR(y) = ln(P(x=1|y)/P(x=0|y))` (식 1,2) 계산에 포함시켜 6가지 LLR 레벨(기존 4개보다 많음)을 SPA 디코더에 공급 — 더 정밀한 초기 LLR 추정으로 복호성능 향상.
6. 확장 기법으로 optical coupler 분할비(예: hybrid 50/50, 90/10 등)에 따른 branch별 Q-factor 저하를 수식화(`ΔQ ≈ 20log10(X_C)`)하고 최적 커플링비를 탐색.
7. 구현 디테일: Monte Carlo 시뮬레이션(10^6~10^7 비트)으로 thermometer 코드 발생 빈도표(logic table) 생성 후 조건부확률 → LLR 매핑, 2bit(4레벨)/실제로는 inconsistent 포함 최대 6레벨 양자화.
8. 별도 학습/최적화 알고리즘 없음 — 문턱값(threshold) 스윕과 post-FEC BER 계산으로 최적 hard/soft threshold 결정(부록 C).
9. 결과: 증폭 장거리 링크에서 hybrid fan-out 50/50이 inconsistent code 활용 시 XNOR 방식 대비 0.7dB 개선(2bit 전기적 fan-out 대비는 여전히 0.32dB 열세); 무증폭 단거리 링크 실측(LDPC 672,588)에서 2bit 전기적 fan-out이 무부호 대비 NCG 4.13dB, 경판정 대비 0.915dB 개선을 실험으로 검증.
10. 한계: 순수 아날로그 프론트엔드/잡음 분석과 실험 검증에 국한되며, LDPC 부호 자체나 디코더 알고리즘(min-sum, 스케줄링 등) 개선은 전혀 다루지 않음(표준 SPA 그대로 사용); HW(ASIC/FPGA) 설계도 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Soft-decision LLR 양자화(2~3bit) | `channel.cpp` `Set_LLR_Th()` | 광통신 채널 특화 잡음모델(RIN/ASE/shot noise) 기반이라 NAND RBER 채널과 물리적 근거가 달라 직접 이식 불가, 개념적 유사성만 존재 |
| 디코딩 알고리즘(SPA) | `decoder.cpp` `LDPC_Decoder()` | 표준 sum-product 그대로 사용, Prime ECC의 min-sum 대비 개선 요소 없음 |
| Inconsistent thermometer code 활용 다중 LLR 레벨 확보 | 대응 없음 | NAND는 단일 read 경로이며 광학적 multi-branch 분리 개념이 대응되지 않음 |

> 적용 가치: 낮음 — 광학 하드웨어(포토다이오드/커플러) 잡음 특성에 기반한 프론트엔드 설계 논문으로 LDPC 알고리즘/부호 기여가 없고 NAND 채널 특성과 무관.
