### B. 알고리즘 요약

1. QC 코드는 `ℓ×ℓ` circulant 서브행렬로 구성된 `m×n` 블록 행렬로 표현되며, 각 circulant를 링 `F_ℓ=F[x]/(x^ℓ-1)`의 다항식으로 대응시켜 행렬-벡터 곱을 다항식 곱셈으로 취급.
2. 기존 shift-register 기반 QC 인코딩은 비트 복잡도가 생성행렬 비영원소 수에 비례해 고속 통신/저장 응용에 부적합, FFT 기반 다항식 곱셈은 점근적으로 빠르나 실용적 ℓ에서는 오히려 비효율적.
3. 핵심 가정: 인코딩은 G-based(`p=G_p^T u`) 또는 H-based(`H_p p = s`, `s=-H_u u`) 두 방식 중 하나이며, 제안 CRT 알고리즘은 이 QC 행렬-벡터 곱 단계에 적용됨.
4. 핵심 기법: `p(x)=p_1(x)···p_s(x)` (서로소 다항식 분해)로 CRT 도메인을 구성, 입력벡터를 변환행렬 `T`로 CRT 도메인으로 옮긴 뒤 포인트와이즈(작은 링별) 곱셈 `v'=A'u'` 수행, 역변환 `T^{-1}`로 원 도메인 복귀.
5. `T`, `T^{-1}` 변환은 `O(ℓ log²ℓ log log ℓ)` field 연산, 도메인 내 곱셈 단계는 `O(nmlogℓ)` 연산 — 전체 CRT 알고리즘 복잡도가 직접곱보다 점근적으로 우수함을 보임.
6. 확장: H-based 인코더의 `H_p`가 비가역(non-invertible)인 고rate QC-LDPC의 경우 QC generalized inverse `H_p^+`(`H_pH_p^+H_p=H_p`)를 정의해 `p=H_p^+ s`로 해를 구함.
7. field F2에서는 `T`, `T^{-1}`를 `ℓ×ℓ` binary 행렬곱으로 구현 가능하며 이 경우 최대 `ℓ²` 2-input XOR 게이트로 HW 구현 가능하다고 언급(수치 예시는 없음).
8. 별도 학습/최적화 절차 없음(순수 대수적 유도).
9. 결과: 복잡도가 기존 direct 곱셈 `O(ℓ²)` 대비 CRT 방식은 `O(nm logℓ)` (+변환 오버헤드 `O((n+m)log²ℓ loglogℓ)`)로 점근적 개선; 수치 시뮬레이션·게이트카운트 등 실측치는 본문에 제시되지 않음.
10. 한계: 이론적 복잡도 분석에 그치며 실제 코드/HW 구현·시뮬레이션 결과가 없고, generalized inverse가 QC 구조를 유지하며 존재하는 조건이 제한적(circulant size와 field characteristic이 서로소일 때 항상 존재).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| systematic 인코딩 (H_p p = s 풀이) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 구조로 CRT 기반 일반 QC 곱셈 가속과는 접근이 다름 — 재검증 부담 큼 |
| QC circulant 행렬-벡터 곱 가속(CRT 도메인 변환) | 대응 없음 | Prime ECC 인코더는 shift/XOR 기반 dual-diagonal에 특화되어 다항식 CRT 변환 개념이 없음 |
| H-matrix / 부호구조(QC 서브행렬) 로드 | `ecc_top.cpp` `Load_PCM()` | QC 구조 자체는 공유하나 CRT 기반 가속은 부호 자체가 아닌 곱셈 알고리즘 층이라 이식 난이도 높음 |

적용 가치: 낮음 — 이론적 QC 행렬곱 가속 기법으로 실제 HW 구현·시뮬 검증이 없고, Prime ECC의 고정 dual-diagonal 인코더는 이미 저복잡도이므로 CRT 계층을 별도 도입할 실익이 낮음.
