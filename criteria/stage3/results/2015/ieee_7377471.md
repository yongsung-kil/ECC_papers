### B. 알고리즘 요약

1. 시스템: Tanner 그래프 기반 regular/irregular LDPC 코드를 대상으로 하는 완전병렬(fully-parallel) FPGA 디코더 아키텍처, AWGN 채널 시뮬레이션으로 검증.
2. 문제: irregular LDPC 코드(비균일 노드 차수)를 다양한 코드 인스턴스에 유연하게 대응하면서 병렬 처리 가능한 확장형 HW 구조 필요.
3. 핵심 가정: 표준 log-domain min-sum belief propagation(체크노드-변수노드 메시지 패싱)을 사용, `tanh` 기반 sum-product 대신 min 근사로 연산 단순화.
4. 핵심 기법: 각 check node/variable node마다 독립적인 전용 연산 유닛(check node unit, variable node unit)을 인스턴스화하는 완전병렬 구조 — 유닛 수는 코드의 실제 check/variable node 수와 동일.
5. 핵심 식: check node 업데이트 `r_ij = sign(prod q_ij) * min_{j'∈N(i)\j} |q_ij'|`, variable node 업데이트 `q_ij = L(p_j) + sum_{i'∈M(j)\i} r_i'j` (표준 min-sum 메시지 갱신식).
6. 확장: 코드 생성 프레임워크(자체 개발)가 파리티체크행렬/Tanner 그래프로부터 노드 간 연결 인덱스(entity_index, input_index, edge_index)를 자동 계산하여 VHDL 파일로 출력, 디코더 소스는 고정.
7. 구현 디테일: 초기 확률(LLR)은 Look-Up-Table(LUT)로 log-domain 변환, 각 노드 연산은 1 clock cycle에 완료되도록 설계.
8. 학습/최적화 절차: 없음(자동 코드 생성 알고리즘은 조합적 인덱스 매핑이며 학습 기반 아님).
9. 결과: Stratix V/Cyclone IV/Cyclone V FPGA에 다수의 LDPC 코드를 합성하여 clock frequency 및 회로 활용률(ALM/LE 사용률) 확보(표 수치는 텍스트 인코딩 손상으로 정확한 판독 불가).
10. 한계: 완전병렬 구조는 FPGA 자원 제약상 짧은 블록길이 코드에 국한되며, 저자가 결론에서 "긴 블록길이 LDPC 코드에는 semi-parallel 구조에 대한 추가 연구가 필요"라고 명시함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum check/variable node 메시지 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()`, `VN_Cal_HD()` | 표준 min-sum 알고리즘으로 Prime ECC가 이미 보유한 기법과 동일(중복 기여) |
| 완전병렬(노드별 전용 유닛) 아키텍처 | 대응 없음 | Prime ECC는 z_sb=32 lifting 기반 부분병렬(row 단위) 구조로, 노드별 완전 인스턴스화 방식과 근본적으로 상이 |
| 코드 생성 프레임워크(파리티체크행렬→연결 인덱스 자동 생성) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 개념은 유사하나 본 논문은 완전병렬 HW 배선을 위한 인덱스 생성이 목적으로 용도 상이 |
| 초기 LLR LUT 변환 | `channel.cpp` `Set_LLR_Th()` | LLR 초기화/양자화 개념 유사 |

> 적용 가치: 낮음 | 표준 min-sum 알고리즘은 Prime ECC가 이미 보유하고 있으며, 유일한 기여인 완전병렬 자동생성 아키텍처는 장길이·고rate NAND QC-LDPC(z=32 lifting 기반)와 구조적으로 맞지 않아 이식 난이도가 높고 실익이 낮음.
