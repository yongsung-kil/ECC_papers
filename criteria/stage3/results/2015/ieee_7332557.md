### B. 알고리즘 요약

1. 시스템은 rate `R = 1 - dv/dc`의 semi-infinite SC-LDPC 코드 앙상블을 사용하며, BEC(binary erasure channel) 상에서 스트리밍 데이터를 전송한다.
2. 기존 block LDPC는 블록 길이에 비례하는 지연이 커서 streaming/real-time 전송에 부적합하다는 문제를 다룬다.
3. 채널 모델은 erasure probability `ε`의 BEC이며, 정보 시퀀스 `U`를 블록 단위로 나눠 순차 인코딩/디코딩한다.
4. Recursive encoder는 SC-LDPC의 convolution 구조(`Hi(t)`가 `M x M` permutation submatrix)를 이용해 프레임 단위로 순차적으로 parity bit를 계산한다.
5. `AWD = Tw + Tdec(W)` 식으로 sliding-window decoder의 지연을 정의, 윈도우 크기 `W`가 latency와 복잡도를 결정한다.
6. 논문이 제안한 modified sliding-window decoder는 타겟 심볼뿐 아니라 인접 심볼까지 조기 복구되면 여러 프레임 단위(`s`블록)로 빠르게 슬라이드한다.
7. HARQ(chase combining)로 디코딩 실패 시 재전송을 요청, 최대 재전송 횟수 초과 시 해당 윈도우를 폐기하고 재인코딩한다.
8. Density evolution으로 (3,6) 앙상블의 belief propagation threshold를 계산해 앙상블 설계에 활용한다(GA/NN 등의 별도 최적화 절차는 없음).
9. 결과: (3,6) 앙상블에서 타겟 심볼은 약 20 iteration 내 erasure probability가 0으로 수렴하며, 이론적 throughput이 Shannon limit에 근접함을 보임.
10. 한계: BEC 채널 전용 이론/시뮬레이션 결과만 제시하며 HW 설계나 AWGN/NAND 채널 특성은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호구조 (convolutional H matrix) | 대응 없음 | Prime ECC는 QC-LDPC 고정 구조이며 SC-LDPC convolutional 구조 미지원 |
| Sliding-window decoder | 대응 없음 | 스트리밍 프레임 단위 윈도우 디코딩으로 Prime ECC의 블록 기반 `LDPC_Decoder()`와 구조적으로 상이 |
| HARQ / chase combining | 대응 없음 | Prime ECC는 재전송 프로토콜 미보유 (NAND read-retry와 개념 유사하나 본문 근거로 직접 매핑 불가) |
| Density evolution (BEC 기반 threshold) | 대응 없음 | AWGN/RBER 채널 기반 Prime ECC 채널 모델과 상이(`channel.cpp`) |
