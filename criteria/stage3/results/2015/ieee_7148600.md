### B. 알고리즘 요약

1. 대상 시스템은 LTE(rate 1/2~5/6, N=576~2304)와 WiFi(802.11n/ac, rate 1/2~5/6, N=648~1944)의 랜덤/PEG-like QC-LDPC 부호, SPA(sum-product) 디코더, AWGN 채널.
2. 문제의식: 종래 SPA 디코더는 초기 몇 iteration에서 방향이 결정되면 이후 iteration을 늘려도 오답 상태에서 벗어나기 어려움(entropy가 너무 낮아짐) - 반복 자원 낭비.
3. 채널/오류 모델은 표준 AWGN, cycle·trapping set이 짧은 blocklength에서 성능을 저해한다는 전제.
4. 핵심 기법: 컨벤셔널 SPA로 `Ic` iteration 시도 후 실패 시, `ns`개의 서로 다른 서브그래프(각각 비율 `δ`만큼만 노드 참여, 나머지는 정규화 LLR을 0.5로 설정해 erasure)로 각 `Ir` iteration씩 재시도. `IC = Ir·ns + Ic`로 전체 iteration 예산 동일 유지.
5. 핵심 의미: erasure된 노드의 LLR=0.5(불확실 상태)는 entropy를 인위적으로 높여 디코더가 나쁜 수렴 경로에서 이탈해 다른 경로를 탐색하게 함 - 어느 한 서브셋이라도 유효 코드워드에 도달하면 성공.
6. 확장 없음(단일 기법). 서브셋 수 ns 최대 5로 저장 오버헤드(ns x n) 제한.
7. 구현 디테일: erasure는 LLR 재할당만으로 구현되어 추가 연산복잡도 거의 없음("virtually free"); 서브그래프별 병렬 처리 가능성 언급(HW 자원 있으면).
8. 최적화 절차 없음 - δ, ns는 경험적으로 선택(논문에서 "specific selection remains an open problem"이라 명시).
9. 결과: 동일 iteration 수(IC=100)에서 코딩 이득 0.11~0.26dB (LTE (576,288) 0.25dB, WiFi (1944,1458) 0.13dB, (648,324) 0.26dB 등), BER 10^-6~10^-8 영역에서 관찰.
10. 한계: HW 미설계, 저장 오버헤드 존재, 파라미터(δ, ns) 선택 기준 부재, 저~중간 rate(최대 5/6) 및 blocklength(최대 2304) 범위만 검증, NAND 고rate(~0.9) 환경 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 디코딩 실패 시 서브셋 erasure 재시도(recovery) | `decoder.cpp` `LDPC_Decoder()` | 조기종료(Partial/Full CRC) 실패 후 fallback 단계로 이터레이션 루프에 추가 가능 |
| LLR erasure(0.5 재할당)로 entropy 상승 유도 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | LLR 재설정 방식이 기존 LLR 테이블 구조와 유사해 삽입 지점 명확 |
| 서브그래프 병렬 시도 | (대응 없음) | Prime ECC의 z=32 row 병렬화와는 다른 차원(다중 디코딩 시도)의 병렬성으로 별도 HW 리소스 필요 |
| 조기종료 CRC 판정 | `partial_CRC.cpp` `partial_crc_HW_T4()` | recovery 시도의 valid codeword 판정에 기존 CRC 검사 재사용 가능 |

> 적용 가치: 중간 — 실패 후 low-cost erasure 기반 재시도라는 아이디어는 디코더 루프에 조기종료 실패 시 fallback으로 결합 가능하나, 저장 오버헤드(ns x n)와 다중 서브그래프 재시도로 인한 latency 증가 우려가 있어 HW 재설계 필요.
