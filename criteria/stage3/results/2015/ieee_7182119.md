> 총평: min-sum CNU 연산 자체를 근사화해 비교기 트리를 AND 게이트로 대체하는 HW 최적화로, 정정성능 저하는 거의 없이 CNU 면적을 56% 절감한 이식성 높은 결과.

### B. 알고리즘 요약

1. (3,6)-regular QC-LDPC, base-matrix 12×24, 확장계수 `Z=54`, `H`는 648×1296, layered scheduling(`L=3` layer)으로 복호.
2. 기존 OMS는 offset factor `λ`를 CNU에서 감산해야 하므로 비교기 트리·부가 로직이 필요해 FPGA에서 면적·처리율 손해가 큼.
3. 채널은 AWGN, 메시지는 `q=4`bit(αm,n), AP-LLR은 `q̃=6`bit로 양자화.
4. 핵심 기법(POMS): CNU 입력 `|αm,n^SAT|`의 LSB를 0으로 지우고(`x[LSB←0]`) 최소값을 구해 offset 보정을 "부분적으로만" 수행.
5. `|β|_{m,n}^{POMS} = min|αm,n^SAT|[LSB←0]`; λ=1일 때 OMS와 동일하려면 LSB=0인 경우만 일치 — 즉 MS와 OMS 사이의 절충.
6. 2단 근사(I-POMS): CNU의 `Detect_0` 신호(입력 중 0 존재 여부 판별)까지 생략, `a*_{m,n}`로 치환해 AndMsb*/AndLsb* 두 신호만으로 β 계산.
7. HW 구현: CNU는 2bit(am,n)의 AndMsb/AndLsb/Detect_0 조합 논리로 대체, β 메시지는 2bit(|β|)+1bit(sign)=3bit로 저장 → β 메모리 25% 절감.
8. 별도 학습/최적화 절차 없음(고정 논리식 유도).
9. 결과: CNU LUT 56% 절감(OMS 대비), 처리율 1.95~2.41Gbps(OMS 대비 48~83% 증가), BER 10⁻⁵에서 POMS는 OMS 대비 0.07dB, I-POMS는 MS 대비 0.08dB 열화.
10. 한계: (3,6)-regular rate-0.5 코드에서만 검증, ASIC 미검증(FPGA 전용), irregular/고rate 코드 확장성 미확인.

| CNU 비교기 트리 대체(AND게이트 기반) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 min1/min2 기반 CNU 사용 중 — HW 면적 절감형 대체 연산으로 검토 가치 |
| β 메시지 양자화(3bit) 및 메모리 절감 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | soft-4bit+ 환경에서 β 저장 비트폭 축소 아이디어로 참고 가능 |
| layered scheduling / permutation·barrel shifter HW | HW 모델(z=32 파이프라인) | 우리 z=32 병렬구조와 z=54 구조 차이 있어 직접 이식보다는 참고 |

> 적용 가치: 중간 — CNU 근사 아이디어는 min-sum 변형으로 이식 난이도가 낮은 편이나, rate-0.5 (3,6)-regular 코드 기준 검증이라 NAND 고rate QC-LDPC에서 impreciseness로 인한 성능 열화 재검증이 필요함.
