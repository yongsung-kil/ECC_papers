### B. 알고리즘 요약

1. 대상은 CMMB(China Multimedia Mobile Broadcasting) 표준 LDPC(rate 1/2) 소프트웨어 디코더를 GPU(OpenCL)로 병렬화.
2. 기존 OpenCL 구현은 host-device 간 explicit 데이터 전송이 PCI-e 버스를 거쳐 큰 latency를 유발하는 문제가 있음.
3. 채널/노이즈 모델 서술 없음(디코딩 알고리즘 자체 개선이 아니라 실행 플랫폼 최적화가 목적).
4. LDPC 디코딩을 INIT, CNU(check node update), BNU(bit node update), PC(parity check) 4단계로 나눠 각 단계를 OpenCL data-parallel 모델(work-item/work-group)로 GPU PE에 매핑.
5. GMAC(Global Memory for ACcelerators)의 `eclAlloc`로 CPU-GPU 공유 가상주소 공간을 만들어 `eclCallNDRange`, `eclSetKernelArg`로 커널 실행 시 명시적 memcpy 없이 데이터 접근.
6. 확장 없음(단일 GPU, 단일 알고리즘 프레임 반복 처리).
7. pre-pinned 메모리 객체(OS page-out 방지)로 host-GPU 전송 속도 향상, 이는 HW 친화적 트릭이라기보다 소프트웨어 런타임 최적화.
8. 학습/최적화 절차 없음.
9. 결과: CPU 단일스레드 대비 8.26배, 일반 OpenCL 대비 1.28배 속도 향상(SNR별 프레임당 평균 디코딩 시간 24.95ms→0.43ms 범위, AMD Radeon HD 7800 GPU 기준).
10. 한계: 디코더 정정성능(BER/FER) 개선 없음(순수 속도 최적화), 소프트웨어(GPU) 구현으로 ASIC/FPGA HW 설계 없음, CMMB 특정 H-matrix에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC 디코딩 알고리즘(INIT/CNU/BNU/PC 단계) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 단계 구성은 유사하나 논문은 GPU 소프트웨어 병렬화만 다루고 알고리즘 변경 없음, 이식 가치 낮음 |
| GPU host-device 메모리 전송 최적화(GMAC) | 대응 없음 | Prime ECC는 bit-exact HW(RTL) 시뮬레이터로 GPU 소프트웨어 런타임 최적화와 무관 |
| CMMB H-matrix 기반 실험 | `ecc_top.cpp` `Load_PCM()` | 부호 자체는 표준 고정 H-matrix이며 이식 대상 아님 |

> 적용 가치: 낮음 — 디코딩 알고리즘 개선이 아닌 GPU 소프트웨어 플랫폼(OpenCL/GMAC) 성능 최적화 논문으로 Prime ECC의 HW RTL 시뮬레이터 구조와 접점이 없음.
