### B. 알고리즘 요약

1. 시스템: IEEE 802.11ad QC-LDPC, sub-matrix `Z=42`, 코드길이 672, 코드율 `1/2, 5/8, 3/4, 13/16` 전체 지원, ASIP(Synopsys IP Designer) 기반.
2. 문제: layered 디코딩은 flooding 대비 수렴이 빠르나, 이전 layer 결과에 의존하는 특성 때문에 layer 간 파이프라이닝이 불가능해 고속화가 어려움.
3. 핵심 기법: half-layer paralleled 아키텍처 - 코드워드 절반의 bit node만 VFU(381개, 8-slice)에 매핑하고, 42개 CFU가 8개 slice에서 온 최대 8개 사전정보로 min-sum 처리.
4. 한 layer를 두 개의 half-layer로 나눠 처리하며, 두 half-layer 간 메모리 접근 충돌이 없어 half-layer 레벨 파이프라인 적용 가능(연속 layer 간 1cycle 오버랩).
5. 사이클 수 = `(4×Nlayer)×Niter+1` - 4개 명령어 슬롯(scalar/load-rotate-subtract/min search/output-add-store)의 VLIW/SIMD 구조.
6. 확장: 저코드율(예: rate 1/2)에서는 두 layer를 하나의 virtual layer로 결합해 5cycle 내 처리, throughput 저하를 완화.
7. 구현 디테일: column re-ordering으로 APP 메모리↔VFU/CFU 간 라우팅을 규칙적으로 만들어 배선 복잡도 감소(Fig.3~4).
8. 4-core 멀티코어 구성 + 전압/주파수 스케일링(VFS)으로 1.8~18.4Gbps 범위 처리율 지원, 저처리율에서는 4코어 저주파 동작이 1코어 고주파보다 에너지효율적.
9. 결과: 28nm CMOS, 630MHz, rate 1/2에서 최대 4iteration 18.4Gbps, 평균 2iteration 시 18.0pJ/bit, 23.58Gbps/mm² - 기존 최고 ASIC([6], flooding+frame-pipeline, 12Gbps) 대비 40% 에너지효율·53% 면적효율 개선.
10. 한계: 정정능력/error-floor 개선 없음(순수 HW 아키텍처 논문), 802.11ad 특정 QC-LDPC(Z=42) 구조에 최적화되어 코드 변경 시 재설계 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Half-layer 파이프라이닝(layer 간 1cycle overlap) | `decoder.cpp` `LDPC_Decoder()` | layered 스케줄링 최적화 기법으로 파이프라인 6-stage 구조와 비교 검토 가치 있음 |
| CN min-sum + sign 연산 (min1/min2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 방식과 동일 계열(중복), half-layer 분할 방식만 차별점 |
| Column re-ordering(APP memory-VFU 라우팅 규칙화) | `decoder.cpp` `VN_Cal_HD()` 등, HW 라우팅 관련 | 메모리 라우팅 최적화 기법으로 HCU/GT의 라우팅 최적화와 비교 가치 |
| VLIW/SIMD ASIP 명령어 슬롯 구조 | 대응 없음 | Prime ECC는 고정기능 HW 시뮬레이터로 프로그래머블 ASIP 구조 자체는 대응 없음 |
| 멀티코어 + VFS 처리율 스케일링 | 대응 없음 | Prime ECC는 단일 코드/디코더 시뮬레이터로 멀티코어 관리 계층 대응 없음 |

> 적용 가치: 중간 — 정정성능 개선은 없지만 half-layer 파이프라이닝과 column re-ordering을 통한 HW 활용률/처리율 개선 기법은 Prime ECC의 layered 스케줄·파이프라인 구조에 참고할 수 있음. 단, VLIW ASIP·멀티코어 구조는 Prime ECC의 고정기능 시뮬레이터 구조와 이질적.
