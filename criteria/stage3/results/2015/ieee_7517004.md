### B. 알고리즘 요약

1. WiMAX(576~2304, rate 1/2·2/3·3/4)와 Wi-Fi(rate 1/2·2/3·3/4) 두 표준의 QC-LDPC 코드를 하나의 디코더로 지원.
2. 기존 partially-parallel TDMP 디코더는 NZS-block을 순차 처리해 고rate에서 iteration당 clock cycle이 커지고 throughput이 떨어짐.
3. AWGN 채널, BPSK 변조 가정 하에 RTL 시뮬레이션 및 ASIC 합성으로 검증.
4. Full Layer Parallel(FLP) 구조: 한 layer의 모든 NZS-block을 한 clock에 동시 처리, TDMP-NMS(정규화 min-sum) 알고리즘으로 `β_mn = L_n - α_mn`(1), CN 갱신 `α_mn = ε·min|β|·sign product`(2, ε=0.75) 수행.
5. 식 (2)의 정규화 factor 0.75는 min-sum 근사 손실을 보정해 sum-product 대비 성능차를 줄이는 역할.
6. WiMAX rate 1/2에서 인접 두 layer를 base PCM reordering으로 겹치지 않게 재배열해, CNU를 재사용한 2-layer 동시 처리로 iteration당 clock 수를 절반으로 축소.
7. CNU는 min-value finding tree(MVFT) 구조로 1st/2nd min과 위치를 4-stage만에 동시 산출, 128×128 MUX-array 기반 shifter(PN)로 128 PE 배열에 재구성 가능한 라우팅 제공(PRN I/II로 sub-block↔row 재배열).
9. 40nm 공정, 7-bit 양자화(1-4-2), 290MHz에서 WiMAX 789~2227Mbps, Wi-Fi 470~1879Mbps 달성, BER은 TDMP-NMS floating point 대비 0.2dB 손실.
10. HW는 특정 표준(WiMAX/Wi-Fi)의 PCM 구조에 최적화되어 있고 NAND ECC 고유의 고rate·비대칭 구조에는 직접 적용이 어려우며, 정정능력 개선 관점의 기여는 없음(순수 throughput/면적 최적화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP-NMS min-sum 알고리즘, 정규화 factor 0.75 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 min-sum 계열과 유사, 정규화 factor값만 참고 가능 |
| CNU MVFT(1st/2nd min 동시 탐색, 4-stage) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN 연산 HW 구조 대체안으로 참고 가능 |
| Full Layer Parallel(전 layer 동시처리), PN/PRN 셔플 네트워크 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z_sb=32 row 단위 병렬로 구조가 다름, 재설계 필요 |
| Matrix reordering(2-layer 동시처리) | `ecc_top.cpp` `Load_PCM()` | H-matrix 고정 구조 특성상 이식 난이도 높음 |
| 7-bit 양자화(1-4-2), 채널 모델(AWGN/BPSK) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 무선 AWGN 전용, NAND soft-read 모델과 상이 |

> 적용 가치: 낮음 | WiMAX/Wi-Fi 표준 전용 HW 최적화(throughput/면적)로 정정능력 기여가 없고, 코드 구조가 특정 무선표준 PCM에 종속되어 NAND ECC 이식 시 재설계 부담이 큼.
