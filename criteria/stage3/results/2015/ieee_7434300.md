> 총평: DVB-S2 위성통신 고차 변조용 soft-demapper의 LLR 계산을 저복잡도화한 논문으로, LDPC 디코더 자체 개선이 아니며 변조방식이 NAND 채널과 무관해 이식 가치가 낮음.

### B. 알고리즘 요약

1. DVB-S2 표준의 8-PSK(코드rate 3/5)와 16-APSK(코드rate 2/3) 변조, 짧은 프레임(16200 bit) LDPC 부호에 대해 AWGN 채널에서의 soft demapper를 다룸.
2. 기존 정확한 LLR 계산(식 2, 3)은 제곱근·지수·로그 연산이 많아 고차 변조에서 HW 면적/메모리 부담이 크고, Max-Log 근사(`ln(e^x1+e^x2)≈max(x1,x2)`)도 여전히 전체 성상 심볼과 max 비교가 필요함.
3. 채널은 AWGN, 수신 심볼 `r`과 성상점 `si` 간 유클리드 거리 기반 우도(`Vi`)를 가정.
4. 핵심 기법: 성상을 π/8 회전(8-PSK) 후 사분면별 소구역으로 분할, 각 비트마다 인접 2심볼(`bj=0`, `bj=1`)만으로 `LLR(bj) ≈ Di - Dj` 형태로 근사(식 6~8).
5. 최종 일반식(식 9)은 `LLR(bj) ≈ (1/σ²)[I(Isx-Isy)+Q(Qsx-Qsy)]` — 수신 심볼의 실수부·허수부를 미리 계산된 상수(μi, μq)와 곱셈만으로 LLR 산출, 제곱항이 상쇄되어 소거됨.
6. 확장: 16-APSK는 진폭이 다른 링(ring) 구조이므로 Equivalent Received Symbol(ERS, `ri=Gi·r`, 스케일 팩터 `Gi=Ri/|r|`)을 도입해 각 링을 정규화 후 동일 방식 적용.
7. 구현 디테일: 곱셈 상수(`μi`, `μq`, `DRef` 등)를 메모리에 사전 저장해 곱셈-덧셈만으로 연산, 비교 연산 수도 Max-Log 대비 대폭 축소(표 II·III).
8. 별도 학습/최적화 절차 없음(수학적 유도와 경계조건 설계).
9. 결과: 8-PSK는 Max-Log 대비 성능 저하 거의 없음, 16-APSK는 Max-Log 대비 약 0.5dB 성능 열화; 연산 개수는 Max-Log 대비 덧셈/곱셈/비교 모두 대폭 감소(표 II·III, 구체 게이트/처리량 수치는 본문에 없음).
10. 한계: LDPC 디코딩 자체는 기존 Min-Sum을 그대로 사용(디코더 알고리즘 개선 아님), HW 합성/실측 없이 시뮬레이션(BER 곡선)에만 근거하며 스케일링 팩터 부재로 인한 16-APSK 성능 열화 원인은 추가 분석 필요로 남김.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft demapper LLR 근사 (PSK/APSK 성상) | 대응 없음 | Prime ECC는 NAND read 기반 LLR 테이블(`channel.cpp` `Set_LLR_Th()`)을 사용하며 PSK/APSK 성상 복조 자체가 대응 없음 |
| LDPC 디코딩(Min-Sum) | `decoder.cpp` `LDPC_Decoder()` | 논문에서 디코더는 기존 Min-Sum을 그대로 사용, 개선 없음 |
```

적용 가치: 낮음 — 위성통신 고차 변조(PSK/APSK) 성상 특화 soft-demapper 단순화로, NAND read 채널의 LLR/threshold 설계(`channel.cpp`)와는 물리 계층이 근본적으로 달라 이식할 요소가 없음.
