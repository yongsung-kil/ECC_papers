### B. 알고리즘 요약

1. 대상은 regular binary LDPC(예: (3,5), (155,93) Tanner code rate≈0.4, (621,1035))이며 채널은 BSC(파라미터 `α`), 복호기는 Finite Alphabet Iterative Decoder(FAID).
2. 문제: 미세공정 HW는 transient 연산오류를 유발하는데, 기존 noiseless threshold만으로는 faulty-HW에서 어떤 복호기가 강건한지 예측 불가.
3. 모델: CNU/VNU/APP를 결정함수 대신 조건부 PMF(`P_Φc`, `P_Φv`, `P_Φa`)로 정의하고, 강건성 분석을 위한 대칭조건(Def.1)과 "noiseless 함수 + 출력단 noise"로의 분해(식11)를 도입.
4. 핵심 기법 1단: 두 가지 단순 오류모델 — Sign-Preserving(SP, 부호 보존·진폭만 오염)과 Full-Depth(FD, 부호+진폭 모두 오염) 전이행렬(`Π^SP`, `Π^FD`)로 faulty 함수 표현.
5. 핵심 식: noisy-DE 재귀 4식(18)-(21) `r=Πc·r(μ)`, `q̃=Πv·q`로 iteration별 메시지 PMF를 갱신하고 오류확률 `Pe(α)`를 산출.
6. 확장: functional threshold `ᾱ`(식26)를 Lipschitz 상수 기반으로 정의해 저오류/고오류 영역의 sharp/smooth 전이를 판별, HW noise 과대 시 예측 실패를 보이고 Def.6으로 재정의.
7. 구현 디테일: VNU를 7-level LUT(`μ1,μ2` 입력)로 표현, offset Min-Sum도 FAID의 한 LUT 인스턴스로 취급(Table II).
8. 설계 절차: trapping-set 분석에서 얻은 ND=5291 FAID 집합에 noisy-DE를 돌려 functional threshold 최대화로 SP/FD 각각 robust/non-robust FAID를 선별(S1∩S2∩S3∩S4).
9. 결과: (155,93) Tanner code, 100 iter Monte Carlo에서 robust FAID가 error-floor 최적화 `Φopt`보다 faulty(SP pv=pc=pa=0.05, FD 0.02) 조건에서 낮은 BER, non-robust는 큰 성능손실; SP-robust가 FD엔 non-robust인 모델 의존성 확인.
10. 한계: 채널은 BSC 한정, transient error만 고려(permanent 제외), HW 미설계로 gate count/throughput 없이 noisy-DE+finite-length 시뮬만.
