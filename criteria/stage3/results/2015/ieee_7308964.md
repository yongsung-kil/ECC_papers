### B. 알고리즘 요약

1. 목표: LDPC 구성의 두 축인 대수적(superposition) 방법과 protograph 방법이 사실은 하나임을 그래프·행렬 양면에서 증명 (binary 한정, Part I).
2. Protograph 구성: base 그래프 `Gptg`(인접행렬 `Bptg`)를 k copy 후 edge permute → `Gptg(k,k)`; 연결행렬을 circulant로 잡으면 결과가 QC-LDPC.
3. 핵심 대수화: `Bptg`를 k개 부분행렬 `D0..Dk-1`로 분해하고 block-cyclic 배열 `Hcyc` 구성 → 행/열 permutation `πrow,πcol`으로 `m×n` CPM 배열 `Hptg` 획득 (그래프 edge permute를 행렬 분해로 대체).
4. Superposition 구성: base `Bb`의 1-entry를 replacement set `R`의 member 행렬로, 0을 ZM으로 치환 → `Hsp`. member가 k×k regular이면 protograph 구성과 동일 (t=k 특수경우).
5. 핵심 주장: protograph 구성은 superposition 구성의 특수 케이스이며, 2002년 이후 대수적 구성 다수가 superposition에서 파생됨.
6. CPM-dispersion(CPM-D): GF(q) base 행렬 `Bq`의 각 nonzero `β^l`을 k×k binary CPM `A(β^l)`로 치환 → binary CPM-QC-LDPC. base 구성만으로 코드 결정.
7. Girth 조건: 2×2 SM-constraint → girth≥6 (Thm1), 3×3 SM-constraint → girth≥8 (Thm2). RC-constraint/pair-wise RC로 short cycle 방지.
8. Masking: 2×2 만족 base에 masking matrix `Z`(Hadamard product `Z⊙Bq`)로 일부 CPM을 ZM화 → girth 확대 + short cycle 대폭 감소.
9. 결과: masked (2640,1320) rate-0.5 (3,6)-regular CPM-QC-LDPC (girth 8), MSA 50 iter로 BER≈10^-10 까지 visible error-floor 없음. protograph (2664,1334) 코드와 waterfall 유사.
10. 한계: 순수 구성/증명 논문, HW·디코더 설계 없음, 최적 분해(decomposition) 탐색은 미해결.
