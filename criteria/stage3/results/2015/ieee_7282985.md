### B. 알고리즘 요약

1. 대상은 protograph 기반 LDPC 앙상블(copy-and-permute로 base matrix `B`를 N배 확장), 목표는 평균 codeword weight enumerator `A(d)`(→ 점근 최소거리 `dmin`) 계산.
2. 풀려는 문제: enumerator 병목은 CN enumerator `A_C(w)` 평가로, 고rate(고연결 CN)·generalized CN·큰 코드길이에서 계산이 급격히 비현실적이 된다.
3. 모델: CN codeword 행렬 `M_C`(qc×K, K=2^{qc-1}), 입력가중치 제약 `w=M_C·n`, `logA_C(w)=max* over {n} logC(N;n)`, Stirling 근사로 `logC(N;n)≈N·H(n/N)` (다변수 엔트로피).
4. 핵심 기법 1(Abu Surra 추측 증명): Eq.(8) 최대화에서 동일 SWV(subset-weight vector)를 갖는 codeword는 동일 비율로 나타남 → 동일 type 엣지는 "as-uniform-as-possible(AUAP)" 가중치일 때 최적.
5. 핵심 식: 3-edge 동일 type의 경우 순환행렬 `Δ`(det≠0, Lemma 2)로 `aδ1+bδ2+cδ3`이 AUAP가 되는 `(a,b,c)∈[0,1]^3, a+b+c=1` 유일해 존재, Jensen 부등식으로 `H(p*)≥H(p1)` 증명(Theorem 3).
6. 확장: 4개 이상 동일 type 엣지는 순환행렬 고유값이 0이 되어 직접 증명 불가 → 엣지 3개씩 골라 순차적으로 AUAP로 맞추는 반복 절차로 일반화.
7. 핵심 기법 2(연속완화): 이산 최적화 `max H(x) s.t. M'_C·x=b, x∈ℒ^K`를 연속 완화 `max H(y), y∈[0,1]^K`로 바꿔 IP/SQP로 효율적 해결.
8. Theorem 4: 이산해집합 `ℒ^K`가 `[0,1]^K`에서 dense이므로 연속해로부터 임의의 작은 오차 ε 내 이산 근사해 구성 가능(엔트로피 연속성 이용).
9. 구현: Moore-Penrose 유사역행렬 `y0=M'_C^+ b`로 초기점, `H`의 해석적 연속(analytic continuation)으로 정의역 밖 값 처리.
10. 결과: N=10^5로 여러 regular protograph의 점근 weight enumerator·dmin 산출(예 (3,6) dmin≈0.023, (5,6) dmin≈0.254). trapping/stopping set·pseudocodeword 열거에도 응용 가능. HW·복호·시뮬 없음.

### C. Prime ECC 관련 모듈 핀포인트

대상이 `code-design`이므로 디코더 성능(C섹션 D-표) 항목은 모두 N/A. 모듈 대응은 아래와 같다.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph 앙상블 weight enumerator / dmin 계산 | 대응 없음 | Prime ECC는 특정 QC-LDPC H-matrix 고정 시뮬레이터로 앙상블 enumerator 계산 기능 없음 |
| Abu Surra 추측(동일 type 엣지 균등 가중치) | 대응 없음 | 부호 앙상블 해석 도구로 부호구성/디코더 코드에 직접 대응 모듈 없음 |
| 연속완화 CN enumerator 최적화 | 대응 없음 | 최적화 도구, 이식 대상 모듈 아님 (부호설계는 `ecc_top.cpp` Load_PCM에 닿으나 본 기법은 H-matrix 생성이 아닌 분석) |

적용 가치: **낮음** — protograph 앙상블의 dmin/enumerator를 빠르게 계산하는 해석 도구로, NAND binary QC-LDPC 부호구성·디코더·HW 어느 모듈에도 이식할 실행 알고리즘이 없다. 다만 error-floor 예측용 부호분석 보조로만 참고 가능.
