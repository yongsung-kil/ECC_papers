### B. 알고리즘 요약

1. 대상은 systematic binary `(32643, 30592)` LDPC 코드(rate 0.937, VN degree 7, CN degree 111/112, edge 228501개)로 ITU G.975.1 해저 광통신 표준 유래 하드디코더용 코드를 soft OMS로 복호.
2. 풀려는 문제: 이런 초장길이 코드를 완전병렬로 구현하면 228만 개 배선이 필요해 라우팅 혼잡으로 사실상 구현 불가능 - 기존 partially-parallel 설계는 DVB-S2 등 저throughput 응용에 한정.
3. 핵심 가정: 코드가 생성자/패리티 행렬 대신 112행×293열의 connectivity matrix `M`(7개 slope로 정의된 대각선이 parity-check 라인)로 표현되며, 행 간 연결관계가 행 번호에 무관(Theorem 1).
4. 핵심 기법 1단: 이 row-invariant 구조를 이용해 293개 processing node로 구성된 단 1개 행만 구현하고, 111행의 memory node(순환 시프트 레지스터)로 전체 매트릭스를 표현하는 "line-passing" 아키텍처.
5. 핵심 식: OMS check-node 업데이트 `m_{c→v}^(l) = max(min|m_{v'→c}^(l-1)|-β,0)·sgn(...)`, variable-node `m_{v→c}^(l) = L_v + Σm_{c→v}^(l)` - β=0.8~1.0 보정 min-sum.
6. 핵심 기법 2단: 각 processing node는 7개 parity-check 라인에 대해 first/second minimum과 누적 sign만 계산·전달(5bit min값 2개+1bit sign×7 = 노드당 edge 6153개, 전체 45122 wire, 완전병렬 대비 1/50).
7. 부수 트릭: 메모리 노드를 병합(merged memory design)하면 노드당 메모리 비트 75% 절감(성능손실 0.1dB, β=0.8)되며, 수집과 분배를 동시에 수행해 추가 지연 없이 critical path 단축.
8. 별도 학습/최적화 절차 없음(구조 기반 결정적 설계).
9. 결과: SNR 5.25dB에서 평균 4.2 iteration 수렴, 완전병렬 대비 면적 92% 절감(122.5mm²→7.99mm²@65nm), TSMC 65nm 합성으로 370MHz에서 24Gbps 달성 (ITU 요구 40Gbps에는 미달, 노드 복제로 확장 가능하다고 언급).
10. 한계: HW구현은 있으나 정정능력/error-floor 개선은 없음(표준 OMS 그대로), 특정 connectivity-matrix 구조를 가진 코드 전용이라 임의의 QC-LDPC로 일반화 시 추가 분석 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset min-sum (min1/min2+sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 알고리즘 자체는 Prime ECC가 이미 보유(§3) - 중복 기여, 신규성은 HW 아키텍처 쪽 |
| line-passing 로컬 인터커넥트/순환 시프트 메모리 구조 | 디코더 HW 모델 (z=32, HCU, GT) | 배선 혼잡 완화라는 목표는 Graph Thinning(`GT.cpp` `Make_GT_HW()`)과 유사한 취지, 순환 시프트 기반 구조는 대응 없음(circulant shift 기반 QC-LDPC와는 다른 방식) |
| connectivity matrix / row-invariant 구조 (부호구조) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base matrix+circulant shift(`PCM_b`) 고정 QC-LDPC로, connectivity-matrix 방식 자체는 대응 없음(부호설계 재검증 부담 큼) |
| iteration별 수렴(평균 4.2 iteration) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration 수 자체는 참고 가능하나 정정능력 개선 주장 없음 |
```

> 적용 가치: 중간 - 알고리즘(OMS)은 이미 보유해 직접 기여는 낮으나, 대형/고degree 코드에서 배선 혼잡을 없애는 로컬-인터커넥트·순환메모리 HW 아키텍처 아이디어는 Prime ECC의 병렬화/메모리 라우팅 최적화에 참고할 여지가 있음.
