### B. 알고리즘 요약

1. 대상 코드는 QC-LDPC, code rate `R=3/8`, base matrix `Hb` 크기 5x8을 확장 인자 `p=9`로 확장한 `H` 크기 45x72(즉 codeword length 72) - AWGN 채널 가정, NAND 특화 채널 모델 없음.
2. 풀려는 문제는 QC-LDPC의 준순환 구조를 활용해 서로 다른 code rate/길이를 지원하는 재구성 가능한(reconfigurable) 디코더를 partial-parallel 구조로 설계하는 것.
3. 채널/노이즈 모델은 일반 AWGN이며 NAND/플래시 특화 요소는 본문에 전혀 없음(무선/일반 저장 통신 대상 서술).
4. 핵심 기법은 offset min-sum 기반 modified belief propagation - check node update에서 `L(rji) = Π sgn(L(qij')) · max[min|L(qij')| - offset, 0]`(offset 값은 density evolution에서 결정).
5. variable node update는 `L(Qi) = L(ci) + Σ L(rji)`로 LLR을 누적하고 부호로 hard decision을 수행하는 표준 sum-product/min-sum 변형 식.
6. 확장 기법으로 single-mode 디코더(고정 rate, 45 CNU + 72 VNU)와 multimode 디코더(node RAM 배열 + reconfigurable router + control unit으로 여러 base matrix/rate 지원)를 각각 설계.
7. 구현 디테일은 node RAM(깊이=확장인자 Z, 개수=최대 non-zero 서브행렬 수 X), lookup table 기반 control unit으로 모드별 라우팅 재구성.
8. 별도 학습/최적화 절차 없음(offset 값은 기존 density evolution 이론값 참조).
9. 결과는 Virtex-7 XC7V2000T FPGA 합성 기준 - single-mode 대비 multimode 자원사용이 소폭 증가(Slice Registers 81→572, LUTs 221→36798), 최대 클록 353→370MHz, iteration 12 고정 시 throughput 100Mbps 이상.
10. 한계: 정정성능(BER/FER, error-floor 등) 시뮬레이션 결과가 전혀 제시되지 않고 순수 HW 자원/클록 비교에 그침, 코드 길이 72로 매우 작아 실용 NAND ECC와 스케일 차이가 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset min-sum BP 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC가 이미 Min-Sum(min1/min2+sign XOR) 방식을 보유하고 있어 중복 기여, 신규성 없음 |
| Partial-parallel 구조(node RAM, 병렬 CNU/VNU 매핑) | `decoder.cpp` `LDPC_Decoder()` (z_sb=32 병렬단위) | 개념적으로 유사하나 논문의 구조가 더 단순(소규모 예제)하며 Prime ECC의 HCU/pipeline 대비 구체성 낮음 |
| Multimode(reconfigurable router, 여러 rate 지원) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC H-matrix 고정 구조라 multimode 재구성 개념 이식은 난이도 높음(부호설계 레이어 변경 필요) |

> 적용 가치: 낮음 - NAND 무관의 일반 QC-LDPC FPGA 디코더 설계이며 알고리즘은 Prime ECC가 이미 보유한 offset min-sum과 중복되고, 정정성능 데이터 부재로 개선 근거도 없음.
