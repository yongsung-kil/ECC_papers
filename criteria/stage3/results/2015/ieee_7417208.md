### B. 알고리즘 요약

1. 시스템: IEEE 802.11n 표준 LDPC 코드, `n=1944` 고정, rate `R=1/2`와 `R=5/6` 두 종류로 AWGN 채널에서 시뮬레이션.
2. 문제: layered(serial) BP는 flooding BP 대비 iteration을 절반으로 줄이지만, 기존 informed dynamic scheduling(RBP/NWRBP)은 residual 계산 비용이 크고(edge 단위) syndrome check 빈도도 과도함.
3. 핵심 가정: variable node j의 APP(a posteriori probability) LLR 갱신 전후 차이 `ξj = ρ'j - ρj`(식 4)를 스케줄링 지표로 사용, edge 단위가 아닌 node 단위로 residual을 정의해 계산량 축소.
4. 핵심 기법 1단: 매 sub-iteration마다 `ηj`(residual 사본, 이미 갱신된 큐 Q 내 노드는 0으로 리셋)의 최댓값을 갖는 `j*`를 골라 업데이트하는 APPRBP 알고리즘(Fig. 3).
5. 핵심 식: threshold 비교 `if ηj* < δ then break`(Line 7~8) — 남은 residual이 충분히 작아지면 해당 반복(k-loop)을 조기 종료해 불필요한 노드 업데이트를 생략.
6. 확장: δ를 0~0.2(R=1/2) 또는 0~0.05(R=5/6) 범위에서 조절해 BER-AIN(평균 iteration 수) trade-off를 유연하게 조정하는 파라미터화된 스케줄러.
7. 구현 디테일: 큐 `Q`로 이미 업데이트된 노드를 추적해 특정 노드 과다 업데이트를 방지, 업데이트 카운터 `C`를 `n`(codeword length)으로 정규화해 `Nused = C/n`으로 iteration 수 산출.
8. 학습/최적화 절차: 없음(δ는 시뮬레이션으로 사전 튜닝하는 정적 파라미터).
9. 결과: `n=1944, R=1/2`에서 `δ=0.1` 시 `Eb/N0=1dB`에서 NWRBP 대비 평균 iteration 수 거의 절반으로 감소, BER은 LBP 수준 유지(δ=0일 때 NWRBP와 거의 동일 BER).
10. 한계: HW 구현/gatecount/throughput 전혀 없음(순수 알고리즘 시뮬레이션), δ가 코드 rate·length에 따라 달라져 범용 고정값 부재, δ 증가 시 BER이 FBP보다 열화될 수 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| variable-node 단위 residual 기반 동적 스케줄링(조기 업데이트 종료) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 기존 고정 스케줄(Dual-Update)을 APP residual 기반 조기종료로 개선 가능한 후보 위치 |
| APP residual threshold `δ` 비교 | `decoder.cpp` `VN_Cal_HD()` 등 VN 처리부 | VN 업데이트 여부 판단 로직을 추가 삽입할 지점으로 대응 |
| 조기종료(반복 축소) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 기존 CRC 기반 조기종료(전체 반복 단위)와 달리 이 논문은 노드 단위 조기종료라 상호보완적, 직접 대응은 아님 |
| Dual-Update 스케줄(odd/even 교대) | `encoder.cpp` `Column_Reordering_Dual_Update()` | 유사 개념(업데이트 순서 최적화)이나 이 논문의 동적/적응형 스케줄과는 다른 고정 스케줄 |

> 적용 가치: 높음 — variable-node residual 기반 동적 스케줄링은 min-sum 이터레이션 루프에 추가 로직만 삽입하면 되어 이식 난이도가 낮고(§4 "디코더 알고리즘" 범주), avg iteration·latency 감소라는 Prime ECC 관심 영역에 직접 부합.
