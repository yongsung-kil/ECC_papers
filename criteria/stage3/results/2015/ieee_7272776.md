### B. 알고리즘 요약

1. IEEE 802.11n 기반 4x4 MIMO 시스템, length-1944 rate-0.5 QC-LDPC 코드, BPSK/QPSK/16-QAM(비교용 64/256-QAM) 사용.
2. 기존 IDD(iterative detection and decoding) 수신기는 검출기-디코더 간 데이터 의존성으로 처리량이 낮고(예: 44Mb/s, 1.58M gates) 하드웨어 복잡도가 큼.
3. 채널은 fast-fading 4x4 MIMO 채널(원소는 i.i.d. 복소 가우시안), 검출기와 LDPC 디코더가 turbo 원리로 LLR을 반복 교환.
4. Non-resetting IDD: outer iteration마다 C2V 메시지를 리셋하지 않고 a priori LLR만 갱신 - 적은 inner iteration으로도 resetting 방식과 동등 수렴 성능.
5. Layered decoding 채택: 체크노드를 layer로 나눠 순차 갱신, `R_mn`(C2V), `Q_mn`(V2C) 메시지를 offset min-sum으로 계산.
6. Fractional inner iteration 스케줄링(Fig.4(c)): 검출기와 디코더의 처리 사이클 수 불일치로 생기는 idle cycle을 layer 단위로 제거해 100% HW 활용.
7. 검출기-디코더 인터페이스: LDPC intrinsic memory 없이 APP 메모리만 사용, 8bit LLR 기준 31,104비트 메모리 절감(direct-mapped 대비).
8. MMSE-PIC 검출기는 QR분해+2x2 서브행렬 역산으로 division-free 행렬역산 구현, CORDIC 기반 QRD로 division 4회로 감소(기존 20회).
9. 결과: 40nm CMOS, 998k gates, 794Mb/s throughput, 135mW@0.9V, 170pJ/bit; 선행연구[12] 대비 10x throughput, 1.42x 면적감소, FER 2dB 개선.
10. 한계: MIMO 검출-복호 결합 시스템 전체 최적화가 핵심이며, LDPC 디코더 자체는 offset min-sum + layered 스케줄의 표준 구현(신규 디코딩 알고리즘 기여 없음); ASIC 실칩 검증이나 무선 MIMO 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered decoding, offset min-sum (CN/VN 연산) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 offset-min-sum과 동일 접근, 신규성 없음 |
| MMSE-PIC MIMO 검출기, QRD 행렬역산 | 대응 없음 | MIMO 무선 채널 전용, NAND 단일 채널 ECC와 무관 |
| Non-resetting IDD 스케줄링, fractional inner iteration | 대응 없음 | 검출기-디코더 결합 반복 구조 자체가 NAND 파이프라인에 대응 없음 |
| APP/CN 메모리 절감 인터페이스 | `decoder.cpp`/`ecc_data.h` (APP·R 메모리 구조) | 메모리 절감 아이디어는 일반적이나 MIMO 인터페이스 특화라 이식 난이도 높음 |

> 적용 가치: **낮음** — MIMO 검출-복호 통합 수신기 칩으로 핵심 기여가 무선 검출기(MMSE-PIC)와 IDD 스케줄링에 있으며, LDPC 디코더 자체는 표준 layered offset-min-sum이라 Prime ECC 3.1(NAND binary QC-LDPC)에 이식할 신규 요소가 없음.
