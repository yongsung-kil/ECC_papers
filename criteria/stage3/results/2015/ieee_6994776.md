### B. 알고리즘 요약

1. 대상은 double/triple-parity RS 부호 `[n, n-2, 3]`, `[n, n-3, 4]` (GF(2^q), 예: [31,29,3], [63,60,4], [255,252,4]).
2. 문제: RS 패리티체크행렬은 dense해서 이진 이미지에 직접 BP 적용 시 결정론적 오류가 커 성능 부족.
3. 모델: AWGN 채널, RS 코드워드를 GF(2^q) 원소의 이진벡터 `b`로 전개한 이진 이미지 `Hb`.
4. 핵심 기법: 각 root `α^i` (i=1..d-2)에 대한 패리티체크식을 `Σ b^(ij) ω_j^i = 0` 형태로 재배열, 이를 compound binary Hamming code 제약으로 해석 (Theorem 1).
5. 핵심 식: `ω_j^i = Σ_k c_<(ij-k)/i>,k`로 정의하면 각 식이 길이 `n=2^q-1` Hamming code의 codeword 제약이 됨.
6. 확장: root `α^0`은 여러 single parity-check 식에 대응하므로 전통 message passing 병행.
7. 구현 디테일: variable/check/Hamming 3종 노드 구성, Hamming 노드는 BCJR로 soft 메시지 계산 (다른 fast soft Hamming 디코더 대체 가능).
8. 학습/최적화: 없음 (고정 그래프 반복 BP).
9. 결과: [31,29,3]에서 KV 대비 0.9dB@BER 1e-5, [63,60,4]에서 PSPA/KV 대비 0.5dB@1e-6 이득.
10. 한계: HW 미설계·시뮬만, RS(non-binary) 전용이며 root 3개 초과로 일반화 시 성능 저하.
