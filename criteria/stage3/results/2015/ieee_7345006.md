### B. 알고리즘 요약

1. NAND SSD向 (36864, 32768) irregular QC-LDPC 코드(rate 0.889, b=512, PEG 생성), layered NMSA 기반 디코더.
2. 기존 NMSA는 normalization factor `α`가 고정값(예: 0.75, 0.25)이라 신뢰도 변화에 대응 못해 iteration 낭비.
3. 채널/모델: NAND read LLR `I_j`, iteration별 check-to-variable `R_ij^(k)`, variable-to-check `L_ji^(k)` 메시지.
4. 핵심 기법: 매 iteration 후 unsatisfied parity-check 개수 `E(k) = Σ e_i^(k)`를 카운트해 α를 적응적으로 선택 (`α(k) = Φ(E(k))`).
5. E(k)가 크면(에러 많음) 작은 α, E(k)가 작으면 1에 가까운 큰 α 선택 — LLR 신뢰도 반영해 수렴 가속.
6. HW 구현 디테일: 원래 연속함수 `Φ(·)`를 2개의 이산 룩업테이블로 단순화, 임계값 E(k)=256 기준으로 두 테이블 중 선택.
7. R-Ram 최적화: 행마다 (min, sub-min, index, sign-all)만 저장해 88.9% 메모리 절감(576Mb→64Mb, 4-bit 양자화 기준).
8. Cyclic-Shifter 1개로 축소 + pipeline 삽입으로 주파수 1GHz 달성.
9. 결과: SNR 6.14dB에서 평균 iteration 22.5~30.3% 감소(throughput 29.2~43.4% 증가), FER 0.2dB 개선, 면적 오버헤드 0.87%, 최종 throughput 14.9Gb/s @ TSMC 65nm, 5.365mm2.
10. 한계: 임계값(256)과 LUT 출력값은 실험적으로 결정되었고 이론적 최적화는 미실시; 4-bit quantization 기준 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| adaptive normalization factor (E(k) 기반 α 선택) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산에 unsatisfied-check 카운트 기반 α 선택 로직 추가 가능 |
| iteration별 LLR/정규화 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 적응형 LLR 테이블 구조에 유사 방식으로 통합 가능 |
| layered message passing (C2V/V2C) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 layered 구조와 동일 계열 |
| R-Ram (min/sub-min/index/sign-all) 압축 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | 이미 유사 압축 저장 구조 보유 가능성 높음 (중복 기여) |
| 다중rate QC-LDPC 지원 (INS-Ram 기반) | `ecc_top.cpp` `Load_PCM()` | H-matrix 교체로 유사 기능, 이식 난이도 높음 |

> 적용 가치: **높음** — adaptive normalization factor는 min-sum 디코더 코어(`decoder.cpp`)에 국한된 개선으로 이식 난이도가 낮고, 이미 보유한 layered/적응형 LLR 구조와 결합 가능하며 iteration 감소·waterfall 개선 효과가 HW 오버헤드 대비 크다.
