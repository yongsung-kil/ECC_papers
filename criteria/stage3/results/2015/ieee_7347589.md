### B. 알고리즘 요약

1. 시스템/코드 스펙: IEEE 802.22 WRAN 표준의 QC-LDPC, base matrix `Hb`(mb×nb) + z×z permutation/circulant shift 확장, code rate 4종(1/2, 2/3, 3/4, 5/6), word length 21종(384~2304bit, z=16~96), 총 84개 조합 지원.
2. 문제: 생성행렬 `G`를 이용한 인코딩(`c=u·G`)은 G가 dense하여 복잡도가 `O(N²)`로 비현실적 - H-matrix만으로 인코딩하는 [11] 알고리즘을 채택하되, 84개 조합 모두를 낮은 전력·면적으로 지원하는 유연 HW 구조가 필요.
3. H-matrix는 `Hb1`(sparse 일반부) + `Hb2`(double-diagonal 구조, dual-diagonal과 유사)로 분할되어 이 특수 구조가 순차적 parity 계산을 가능케 함.
4. 핵심 기법: 각 행 `i`에 대해 `Λ_i = Σ_j P_i,j·u_j`(순환左shift 누적)를 계산해 저장(식 8), 이후 double-diagonal 구조를 이용해 `p_0`을 구하고(식 9), 나머지 parity `p_i`를 역순(`p_{mb-1}`→`p_1`)으로 재귀 계산(식 13, 14).
5. 핵심 식 `P_i,j·u_j = u_j <<< P_i,j` (circular left shift)로, permutation 행렬 곱셈을 barrel-shifter 없이 MUX/DEMUX 기반 shift로 대체해 게이트 수 절감.
6. 확장: 두 아키텍처 제안 - (a) Serial: 비트 단위 순차입력, 내부 SRAM(576×4) 사용, MUX/DEMUX로 shift 구현; (b) Coprocessor: 공유 시스템 메모리 사용(내부 work RAM 제거), sparse Hb의 null 값을 저장하지 않고 non-null 주소만 저장해 사이클 절감.
7. 구현 디테일: 작업 word 크기 4bit(모든 z값의 최대공약수), Hb 저장은 512×7 ROM(serial) 또는 non-null 값+주소만 저장(coprocessor); z<96일 때 shift 값 재조정 공식(식 1) 적용.
8. 별도 학습/최적화 절차 없음 (구조적 HW 설계).
9. 결과: 55nm ASIC 합성, 300MHz 동작, coprocessor 아키텍처 면적 14,811μm²/전력 1.96mW, serial 26,284μm²/3.60mW; throughput은 coprocessor가 R=5/6에서 236.065Mbps, R=1/2에서 324.324Mbps로 802.22 표준 요구치(22.69/13.61Mbps) 충족, 기존 문헌([11],[16],[20]) 대비 면적/전력 우위, coprocessor가 serial보다 최대 13배 빠름.
10. 한계: 인코더 전용(디코더 미포함), 802.22 WRAN 표준 특유 base matrix에 특화되어 타 부호로 일반화 시 재검증 필요, 정정성능/BER 분석 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix 기반 dual/double-diagonal 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 구조의 부호 PCM(`PCM_b_enc`) 기반 인코딩을 사용 - 구조적으로 유사한 접근 |
| 순환 shift 기반 accumulate (barrel-shifter 대체 MUX/DEMUX) | `encoder.cpp` | 게이트 절감형 shift 구현 트릭으로 인코더 HW 면적 최적화에 참고 가능 |
| 다중 rate/length 유연 지원 (84 조합) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(H-matrix) 고정 구조로, 논문처럼 다수 rate/length를 하나의 재구성 가능 HW로 지원하는 접근은 부호설계 유연화 관점에서 참고 가치 있으나 이식 난이도 높음(§4 표 기준 "높음") |
| Coprocessor식 공유 메모리 접근 (내부 RAM 제거) | `encoder.cpp` | 메모리/라우팅 최적화 아이디어로 인코더 HW arch에 참고 가능 |
| ASIC 합성 결과(면적/전력/throughput) | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 실제 게이트 합성 결과 비교 대상 아님 |

> 적용 가치: 중간 - 인코더 전용 논문으로 디코더 성능과 무관하나, H-matrix 기반 double-diagonal 인코딩과 shift 연산 게이트 절감 기법은 Prime ECC의 encoder.cpp(dual-diagonal 고정) 대비 HW 면적 최적화 아이디어로 일부 참고 가능. 다만 802.22 WRAN 특화 다중 rate 지원 구조 자체를 이식하기엔 코드 재검증 부담이 큼.
