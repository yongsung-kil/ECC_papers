### B. 알고리즘 요약

1. 대상은 IEEE 802.11n(WiFi) 표준의 QC-LDPC(rate 1/2, 2/3, 3/4, 5/6, block length 648/1296/1944)이며 offset min-sum algorithm(O-MSA) 기반 row-layered(block-parallel) 디코딩.
2. 풀려는 문제는 layered 스케줄의 강한 데이터 의존성으로 인한 파이프라인 병목과, 다중입력(최대 24개) 중 최소 2개(`min1`, `min2`) 값을 단일 clock cycle에 찾는 연산 비용.
3. 채널은 AWGN, BPSK 변조를 가정, 별도 간섭/noise 모델 없음.
4. 핵심 기법 1단은 rejection-based scheme(RS) 비교기: 다수 입력을 계층적(radix-3/4) 트리로 비교하며 불필요한 입력을 조기에 제거(reject)하여 area는 sorting 기반과 유사하면서 delay는 tree-search에 근접.
5. 2's complement 비교는 carry-look-ahead adder(CLA) 기반 `O(logN)` delay 회로, sign-magnitude 비교는 뺄셈 후 부호 확인 방식으로 단순화하여 `O(1)` 비교 delay를 달성.
6. 핵심 기법 2단은 dynamic multi-frame processing: 프레임 길이(648/1296/1944)에 따라 동시 처리 프레임 수(3/3/2)를 동적으로 정해 파이프라인 버블(4-stage bubble)을 다른 프레임의 유효 연산으로 채움.
7. 부수 트릭으로 barrel shifter + memory addressing 기반 순환 시프트(9-length vector를 3-input shifter로 처리), PCM ROM 단일화 및 base-address 룩업테이블 기반 재구성(reconfiguration).
8. 별도 학습/최적화 절차 없음(HW 아키텍처 설계 및 시뮬레이션 기반 성능 검증).
9. 결과: 90nm CMOS 합성/post-layout에서 336MHz 클록, throughput 5.13Gbps, 에너지효율 0.01nJ/bit/iteration, 기존 최고 기록([13] 1.96Gbps) 대비 62% 향상, gate count 509k(hierarchical)/4.36k(leaf cell 관련 수치는 본문 표 종속).
10. 한계: 단일 표준(WiFi 802.11n) 전용 구조로 다중 표준 지원 없음(대비 대상 [13]은 다중 표준 지원), 정정능력(BER waterfall/error-floor) 개선은 목표가 아니며 순수 HW throughput/latency 최적화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min1/min2 다중입력 비교기(rejection-based) | `decoder.cpp` `CNU_Update_New_Mag()` | CN의 min-sum 크기 연산 회로 최적화 기법으로 직접 대응, gate/delay 개선 시 이식 가능성 있음 |
| Dynamic multi-frame processing (pipeline bubble 제거) | `decoder.cpp` `LDPC_Decoder()` pipeline(col_M1/Z/P1~P4) | 파이프라인 구조가 다르나(6-stage 고정 vs 논문의 동적 프레임 수) 버블 최소화 아이디어는 참고 가능 |
| Layered(row-layered) O-MSA 스케줄 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 Min-Sum + 반복 message passing 보유(§3), layered 스케줄 자체는 중복 기여 |
| QC-LDPC barrel shifter 기반 순환 시프트 | 대응 없음 | Prime ECC 프로파일에 shifter/circulant 관련 회로 세부 항목 없음 |

> 적용 가치: 중간 — 정정성능 개선은 없지만 min1/min2 비교기 HW 최적화와 파이프라인 버블 제거 기법은 decoder.cpp 이터레이션 루프 HW 최적화에 참고할 만함.
