### B. 알고리즘 요약

1. QC-LDPC (3,6) regular 코드, codeword length 1296bit, circulant size 54, rate=0.5(=(1296-648)/1296 추정, 논문 명시는 아님). Flooded(병렬) 아키텍처로 MS, SCMS(Self-Corrected MS), FAID(Finite Alphabet Iterative Decoder) 3종 비교.
2. 딥 나노미터 CMOS 공정에서 저전압 구동 시 FEC 디코더 자체 회로(연산/메모리)에도 확률적 타이밍 에러가 발생할 수 있는데, 기존 연구는 단순 확률 오류모델·C++/Matlab 모델 수준이라 아키텍처(메모리 구조, 라우팅망 등) 세부를 반영하지 못하는 문제.
3. 핵심 가정: 0.3V 저전압, 45nm PTM 공정에서 클록 주기 단축 시 조합논리/메모리 출력에 확률적 지연 fault가 발생(BIAWGN, BSC 두 채널모델 모두 평가).
4. 핵심 기법: 3단계 fault 평가 방법론 — (1) SPICE Monte-Carlo 기반 SSTA로 표준셀 지연 PDF 추출, (2) gate-level PDF propagation으로 각 combinational block 출력의 clock-constraint별 실패확률(CDF) 도출, (3) RTL(Verilog)에 saboteur 기반 fault injection(SFI) 삽입.
5. 핵심 결과 해석: 데이터패스에만 fault 주입(control은 fault-free 가정), 정정 성능은 FER(Frame Error Rate)로 측정. 디코더 내부 데이터패스 오류도 LDPC 디코더가 스스로 정정 가능함을 확인(전송채널 오류와 마찬가지로 취급 가능).
6. 확장 비교: MS(4bit LLR), SCMS(4bit+erasure/부호 메모리 추가), FAID(3bit, binary input, 룩업테이블 기반 VNU) — FAID가 fault 발생 개수가 유의미하게 더 적음(Fig.2).
7. 부수 디테일: 양자화는 AP-LLR 6bit, 입력 LLR/CN 메시지 4bit(MS/SCMS), CN/VN 메시지 3bit(FAID); CNU 3-stage, VNU 4~6-stage(MS/SCMS/FAID), Barrel Shifter 1-stage pipeline.
8. 별도 학습/최적화 절차 없음(fault-injection 시뮬레이션 기반 실증 분석).
9. 결과: 클록주기 3.1ns(MS)/2.5ns(SCMS)까지는 fault-free와 동일 성능 유지, 클록 상한 초과 시 error-floor 발생 또는 복호 불능; 전체적으로 77~150% 오버클럭(처리량 증가) 여유를 정정성능 저하 없이 확보 가능함을 정량 제시.
10. 한계: 정정 알고리즘 자체의 개선은 없음(기존 MS/SCMS/FAID 재사용), 특정 공정(45nm, 0.3V)·특정 코드(1296,648) 조건에서만 검증, gate-level netlist synthesis 결과(gatecount/throughput 수치)는 본문에 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Flooded MS 디코더 데이터패스 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC는 min-sum 기반이나 flooded가 아닌 pipeline(6-stage) 구조라 타이밍-fault 내성 방법론은 직접 대응 어려움 |
| RTL saboteur 기반 fault-injection 방법론 | 대응 없음 | Prime ECC는 bit-exact 클록 레벨 시뮬레이터이나 확률적 타이밍-fault 모델은 미보유(신규 검증 인프라 필요) |
| SCMS의 erasure/부호 메모리 | 대응 없음 | Self-Corrected MS 자체가 Prime ECC 미보유 기법(§3 목록에 없음) |
| 저전압 오버클럭 여유 분석(voltage scaling) | 디코더 HW 모델(z=32, HCU, pipeline) | Prime ECC HW 모델에 동일 방법론 적용 시 전력/처리량 트레이드오프 정량화 가능 |

> 적용 가치: 중간 — 알고리즘 이식 대상은 아니지만, Prime ECC HW 설계에서 저전압 구동 시 타이밍 마진과 정정성능 트레이드오프를 정량화하려는 목적이라면 이 fault-injection 방법론 자체(SSTA+RTL SFI)는 참고할 만함.
