### B. 알고리즘 요약

1. Left-terminated LDPC convolutional code, semi-infinite parity-check matrix `H_conv`(submatrix `H_i(t)`, syndrome-former memory `m_s`), rate `R=(c-1)/c`(정수 `c`), constraint length `ν_s=(m_s+1)c`.
2. 기존 protograph 구성은 그래프 탐색(PEG, ACE 등) 휴리스틱 의존이라 girth-6를 보장하는 최단 constraint length 탐색이 비효율적 — 이를 해석적(대수적) 방법으로 대체하려는 문제.
3. 채널모델은 AWGN 가정(BER 성능평가), protomatrix `Z`의 `(c-b)×c=1×c` 행벡터(`c-b=1`)로 구성, 병렬엣지 미허용(이진 0/1).
4. 핵심 기법은 protomatrix 내 두 bit-"1" 원소의 행 인덱스 차이인 거리 파라미터 `x^(j)_{i,k}`를 정의, 4-cycle 발생조건(정리4: 두 거리값이 동일하면 4-cycle 발생)을 통해 girth-6 조건을 "모든 거리값이 서로 다른 양의 정수"로 환원.
5. 최적 protomatrix 탐색 문제는 `max_{i,k,j}[x^(j)_{i,k}]`를 최소화하는 서로 다른 양의 정수 배치 문제로 단순화(예: `c=5, d_v=3`에서 `m_s=15`가 최소).
6. 확장으로 6-cycle 조건(정리6: 두 거리의 합이 다른 거리와 같으면 6-cycle) 제시, 구조적 제약상 6-cycle은 완전히는 제거 불가함을 증명(단, 4-cycle-free는 보장).
7. 구현 디테일: 각 protomatrix 비영원소를 8×8 type-1 circulant(순환치환)로 치환하는 periodic QC lifting을 적용해 실제 큰 girth(6, 8, 10) LDPC convolutional code 도출, shift value는 modulo 방정식을 simulated annealing/hill-climbing으로 최적화.
8. 최적화 절차: girth-6 protomatrix는 대수적으로 직접 도출(탐색 불필요), 이후 QC lifting shift 값만 heuristic(annealing)으로 최적화하는 2단계 설계 — 기존 전체 그래프탐색 대비 탐색공간 대폭 축소(예제에서 가능 배치 5.5×10^13 중 최적해 80640개, 비율 1.5×10⁻⁹).
9. 결과: rate R=0.8, d_v=3 코드에서 windowed min-sum 복호 시 girth-6(C3, 무최적화 shift)도 AR4JA(girth-4) 대비 waterfall 우수, girth-10(C4)이 girth-6(C3) 대비 error floor 영역에서 명확히 우수(작은 absorbing set 회피); 동일 계산복잡도 비교에서 채널메모리 L=12480(C1)이 AR4JA(S=512, block length 4096) 대비 BER 10⁻⁸에서 0.1dB 이상 우수하면서 메모리는 더 작음.
10. 한계: HW 미설계(회로/게이트 미상), windowed decoder(고정소수점) 시뮬레이션 결과이며 6-cycle은 구조상 완전 제거 불가, 주로 convolutional(spatially-coupled) 코드 설계이고 block code는 tail-biting을 통한 부산물.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-6 protograph 대수적 구성 + QC lifting → H-matrix | `ecc_top.cpp` `Load_PCM()` | tail-biting으로 QC-LDPC block code(H-matrix) 도출 가능해 대응되나, Prime ECC는 특정 고정 QC-LDPC 구조라 재검증 부담 큼(프로파일 §4: 부호설계 이식 난이도 "높음") |
| girth 최적화(4-cycle/6-cycle 회피)로 absorbing set 축소 → error-floor 개선 | `decoder.cpp` `LDPC_Decoder()` (간접, H-matrix 품질에 의존) | Prime ECC 디코더 자체 변경은 아니고 입력 H-matrix 품질 개선을 통한 간접 기여 |
| windowed min-sum 복호 성능평가 | `decoder.cpp` `CNU_Update_New_Mag()` | 표준 min-sum 사용(Prime ECC와 동일 계열)이나 convolutional 구조 특유 windowed decoding은 대응 없음 |

> 적용 가치: **중간** — 코드 구성법(부호설계) 논문으로 이식 난이도는 높지만, girth-6/8/10 최적화를 통한 absorbing-set 감소·error-floor 개선 원리는 Prime ECC H-matrix 설계 시 참고할 가치가 있음.
