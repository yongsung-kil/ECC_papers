### B. 알고리즘 요약

1. SPC(single-parity-check) 코드의 신뢰도 추론(각 변수에 대해 나머지 N-1개 변수의 결합 LLR)을 계산하는 문제; LDPC는 다수의 SPC(=check node)로 구성되므로 이는 LDPC 디코더의 CN 연산에 직결.
2. `O_i = 2·tanh^-1(Π tanh(I_j/2))` (식 1) 형태의 비선형 연산을 N개 변수 각각에 대해 동시에 계산해야 하는데, 기존 cascaded(forward-backward, F/B partial-sum) 구조는 커널 수는 최소(`3(N-2)`)지만 latency가 `(N-2)τ`로 N에 선형 비례해 문제.
3. 커널(⊙)은 논문 추상화상 임의의 결합연산(2-input, 곱셈/tanh 로그도메인/min-sum/LUT 등)이며 실제 구현은 4-bit quantization LUT (`S(O)=S(I_i)⊕S(I_j)`, magnitude는 min 기반 식 16, min-sum과 유사한 형태).
4. 핵심 기법: symmetric-tree 구조로 converging network `U`(입력들을 이진트리형으로 결합해 partial-sum 후보 전부 생성)와 diverging network `V`(U의 출력을 재조합해 최종 O 생성)를 재귀적으로 구성(식 3~13).
5. `U`, `V` 각각 N-2개 커널을 써서 총 커널 수는 기존과 동일하게 `3(N-2)`이나, 트리 깊이가 `log2(N)`이므로 latency는 `2(log2(N)-1)τ`로 로그 스케일로 감소 — 이 latency 감소가 핵심 결과.
6. N이 2의 거듭제곱이 아닌 경우, 가상 입력을 가정해 U_n/V_n을 구성한 뒤 존재하지 않는 노드/연결을 제거하는 pruning 절차(Algorithm 1~3, `Rec_Cons_U`/`Rec_Cons_V`)로 일반 N에 대응.
7. 구현 디테일: 커널은 4-bit sign-magnitude LUT(`M(O)`는 min-sum과 유사한 magnitude 계산 규칙), 정확도·복잡도 불변을 이론적으로 보장(연산 순서만 재배치, 대수적으로 동일).
8. 학습/최적화 절차 없음(구조적/조합적 설계, HDL 자동 생성 가능한 pseudo code만 제시).
9. 결과: FPGA(Altera Stratix IV) 구현 시 N=32에서 fmax 186% 향상(60~70MHz → 190~200MHz), ALUT 사용량은 기존과 거의 동일(322→316); (28672,24576) QC-LDPC 10-iteration layered decoding에서 BER 곡선이 기존 구조와 거의 겹침(양자화 오차 경로 감소로 미세 개선).
10. 한계: 정정능력 자체를 개선하는 기법이 아니라 순수 HW 라우팅/latency 최적화이며, N(CN degree)이 작을 때(N≤4)는 기존 구조와 동일해 이득 없음; throughput(Gbps) 수치는 본문에 미기재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPC(CN) 신뢰도 추론 라우팅 구조 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN degree가 높은(9/7) 컬럼의 min-sum 계산 라우팅을 트리 구조로 재설계해 critical path 단축 가능 |
| pipeline 단계별 critical path 단축 | `decoder.cpp` pipeline 6-stage (col_M1/Z/P1~P4) | 파이프라인 스테이지 내 CN 연산 latency를 줄여 클록 상승 여지 |
| LUT 기반 magnitude 계산 (식 16) | `decoder.cpp` `Get_VNU_*` (magnitude 양자화 테이블) | LUT 기반 CN 연산이라는 점에서 Prime ECC의 min-sum LUT 테이블과 구조적으로 유사, 라우팅 재배치 적용 가능 |
| HCU(비대칭 degree 컬럼 2-cycle 처리) | `decoder.cpp` HCU 로직 | degree가 큰 컬럼에서 이 논문의 트리 구조 적용 시 이득이 더 크므로 HCU 대상 컬럼과 시너지 검토 가치 |

적용 가치: 높음 — CN(SPC) 신뢰도 계산의 라우팅 구조를 바꿔 정정능력 저하 없이 critical path/latency를 개선하는 순수 HW 아키텍처 기법으로, Prime ECC의 min-sum LUT 기반 CN 연산 파이프라인에 비교적 낮은 난이도로 이식 검토 가능.
