### B. 알고리즘 요약

1. 시스템: NAND flash ECC(Hamming/BCH/QC-LDPC) 평가 및 설계, RBER은 P/E cycle 증가에 따라 증가하는 채널 모델 2종(channel model 1: 3x-nm 실측 기반 급증형, channel model 2: 완만한 증가형) 가정.
2. 문제: 기존 ECC 설계는 "capacity 우선" 관점(rate>0.9)에 치우쳐 있으나, BER 증가와 P/E cycle 감소가 심화되는 상황에서는 용량과 신뢰성(수명)을 동시에 고려할 평가지표가 없음.
3. 핵심 지표: Error-free Information Capacity `EIC = Σ_{i=1}^{P/Ecycles} C_i`, 여기서 `C_i`는 i번째 write에서 `P_e(i) ≤ Pe_max`(UBER<10^-15)를 만족할 때의 저장 용량.
4. 보조 지표: Per-unit EIC `PEIC = (1/P/Ecycles) Σ C_i`, EIC가 최대(또는 무한대)가 되는 코딩이 이상적.
5. 계산 절차: (N,k,t) BCH는 UBER 공식 `UBER=(1/k)Σ_{n=t+1}^N C(N,n) RBER^n (1-RBER)^{N-n}`으로 acceptable RBER 산출, LDPC는 시뮬레이션으로 산출 → 두 채널모델에서 P/E cycle과 EIC 매핑.
6. 관찰: 고정 code length에서 rate가 높을수록 acceptable RBER과 P/E cycle이 급격히 줄어듦; 동일 rate에서 LDPC가 BCH보다 EIC/P-E cycle이 수배~수십배 큼(Shannon한계 근접 성능).
7. 핵심 기법: 적응형 LDPC ECC 구조 - 고정 code length(64800)에서 P/E cycle 구간별로 rate를 9/10(0~9300)→5/6(9300~42500)→3/4(42500~87400)→2/3(87400~170000)→1/2(170000~358000)로 단계적으로 낮춤.
8. 최적화 없음: rate 전환 임계값은 채널모델 시뮬레이션 결과(acceptable RBER)에서 직접 도출, 별도 학습/최적화 알고리즘 없음.
9. 결과: 제안 적응형 ECC의 EIC는 rate-9/10 고정 ECC 대비 channel model 1에서 5배, channel model 2에서 한 자릿수(order of magnitude) 향상.
10. 한계: 특정 LDPC 부호 구조나 디코더 알고리즘 자체는 제시하지 않음(기존 rate별 LDPC 재사용), HW/회로 구현 없음, rate 전환에 따른 실제 컨트롤러 오버헤드(멀티 PCM 저장, 재인코딩 등) 미분석.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 적응형 rate LDPC 구조(P/E cycle별 rate 전환) | `ecc_top.cpp` `Load_PCM()` | 다중 rate PCM 로드/전환 정책과 연관되나 특정 알고리즘 이식은 아니고 시스템 설계 참고용 |
| EIC 평가지표 | 대응 없음 | 코드베이스는 디코더 시뮬레이터로, 수명 전주기 용량 평가 지표는 별도 상위 레벨 분석 도구 영역 |
| BCH/LDPC UBER-RBER 산출식 | 대응 없음 | Prime ECC는 binary QC-LDPC 전용이며 BCH 비교 로직 없음 |

적용 가치: 중간 — 특정 디코더/부호 이식 항목은 없으나, "P/E cycle 증가에 따른 rate 적응" 아이디어는 Prime ECC의 rate 정책/PCM 선택 전략(`matrix_sel`) 설계에 참고할 만한 시스템 레벨 통찰 제공.
