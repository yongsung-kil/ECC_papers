### B. 알고리즘 요약

1. LDPC 디코딩 계열 리뷰: BP(sum-product, SPA)는 고성능·고복잡도, bit-flip(BF)은 저복잡도·저성능이며, min-sum 계열이 그 중간 절충안.
2. Min-sum(MSA)은 check node update에서 `L(A2)=sign(L1)sign(L2)min(|L1|,|L2|)`로 정확식 `L(A2)=sign(L1)sign(L2)min(|L1|,|L2|)+g(L1,L2)`의 보정항 `g(L1,L2)`를 생략한 근사.
3. a-min* 알고리즘: check node에서 `min*(L1,L2)=min(|L1|,|L2|)+g(L1,L2)`(boxplus 연산)를 사용하되, 가장 신뢰도 낮은 비트(최소 LLR)용과 나머지 비트용, 총 2개의 outgoing message만 계산해 복잡도 절감.
4. a-min*-min 알고리즘: a-min*을 더 단순화 — 최소신뢰도 비트를 제외한 나머지 비트의 message는 boxplus 대신 단순 min 연산으로 대체.
5. Self-correction(SC) 기법: variable node message의 부호(sign)가 연속 두 iteration 사이에 뒤집히면(fluctuation) 해당 메시지를 신뢰불가로 보고 0으로 소거(erase) — variable node update 단계에 적용.
6. Self-correction은 a-min*, a-min*-min의 variable node update가 min-sum과 동일하므로 동일하게 적용 가능(check node update는 각 알고리즘 고유 방식 유지).
7. 부수 트릭: 별도 HW 근사·양자화 언급 없음, self-correction은 부호 비교(XOR)만으로 구현 가능한 저비용 로직.
8. 학습/최적화 절차 없음 — 알고리즘 자체는 고정, 시뮬레이션으로 성능 비교만 수행.
9. 결과: PEG(1008,504) 코드에서 self-corrected a-min*가 약 0.1dB waterfall 성능 개선, self-corrected a-min*-min이 a-min*보다 우수, 평균 iteration 수는 두 알고리즘 모두 미세하게 증가; EG(8176,7156, rate 0.8752) 코드에서는 개선폭이 더 작음(높은 code rate 때문으로 추정), iteration 수는 a-min*와 거의 동일 또는 a-min*-min에서 소폭 감소.
10. 한계: max iteration 100회 고정 시뮬레이션뿐, HW 미설계, 두 코드(PEG/EG)에만 한정 검증, NAND 채널·부호 미언급, 다른 min-sum 변형에 대한 self-correction 효과는 "추가 연구 필요"로 명시(미검증).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Self-correction (VN 메시지 부호 fluctuation 소거) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 순수 min-sum의 VN update 단계에 추가 로직으로 결합 가능, 낮은 이식 난이도 |
| a-min* / a-min*-min (boxplus 근사 CN update) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 순수 min-sum(min1/min2) 사용 중이라 boxplus 근사항 추가는 복잡도 상승 요인, 이식 시 성능/비용 트레이드오프 재검토 필요 |
| Iteration별 성능/평균 iteration 수 비교 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블과 함께 self-correction 결합 시 상호작용 검증 필요 |

> 적용 가치: 중간 — self-correction 자체는 min-sum에 저비용으로 결합 가능한 waterfall 개선 기법으로 관심도 있으나(§5 "min-sum 성능 개선"에 해당), 함께 검증된 a-min*/a-min*-min은 Prime ECC가 이미 보유한 순수 min-sum보다 복잡도가 높아 그대로 채택하기보다 self-correction 단독 적용 검토가 더 적절.
