### B. 알고리즘 요약

1. 시스템: DVB-NGH 2x2 cross-pol MIMO(eSM), 16k LDPC(length 16200, rate 3/15~11/15), 16-QAM+16-QAM 등, 입력 10bit 양자화, Helsinki 이동 UHF 채널.
2. 문제: MIMO LLR 계산이 후보 심볼마다 유클리드거리 `‖Y-HS‖²`를 요구 → nbpcu=10에서 후보 1024개, 실수곱 4096회로 HW(모바일) 면적 병목.
3. 모델: `Y=HS+W`(AWGN σ²), Max-log LLR `LLR(bi) ≈ (1/2σ²)[min_{S∈Ci0}‖Y-HS‖² - min_{S∈Ci1}‖Y-HS‖²]`.
4. 핵심기법: 거리계산을 2차원 근사 `α·max+β·min`의 일반화로 대체 — `D = f1(f0(a0,b0), f0(a1,b1))`, `fi(w,z)=λi·max(w,z)+γi·min(w,z)`, `ai,bi`는 실/허수부 절대차.
5. 식 의미: 제곱·곱셈 대신 abs+max/min+가중합으로 유클리드거리 근사 → 곱셈기(면적 지배 요소) 제거.
6. 계수 최적화: LLR 출력 RMSE 최소화로 brute-force 탐색 → `λ0=0.7,γ0=0.3,λ1=0.9,γ1=0.4`, 이를 2의 거듭제곱으로 반올림(`λ0=0.5,γ0=0.25,λ1=1,γ1=0.5`)해 곱셈을 bit-shift로 구현.
7. 구현 디테일: 근사는 두 최소값 탐색에만 사용, 확정된 2개 후보는 정확 거리로 재계산해 LLR 값 왜곡 방지.
8. 검증: Verilog-HDL로 거리계산 셀 모델링 후 180nm 합성, NAND 게이트 면적 기준 등가 gatecount 비교.
9. 결과: 거리계산 면적 약 80%↓, 검출 전체 곱셈 25%↓(덧셈 10%↑), BER 성능손실 모든 code rate에서 0.1 dB 미만.
10. 한계: LDPC 디코더 자체가 아닌 MIMO demapper(LLR 생성 프런트엔드)만 다룸, 채널행렬 H·MIMO 후보집합 전제라 SISO/NAND엔 직접 대응 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MIMO demapper 유클리드거리→LLR 생성 | 대응 없음 | NAND엔 MIMO 채널행렬·심볼 후보가 없음, LLR은 셀 read로 생성 |
| max+min(2의 거듭제곱) 거리근사 | 대응 없음 | demapper 프런트엔드 근사, 우리 min-sum CN 연산과 무관 |
| 10bit LLR 양자화 | [channel.cpp](../Prime_ECC_3.1_Claude/) `Set_LLR_Th()` | LLR 양자화 개념은 접점이나 우리는 soft-2~3bit, 4bit+ 미지원 |
| LDPC 디코더(참조만) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 논문은 디코더 내부 미개선, LLR을 공급만 |

적용 가치: **낮음** — 이 논문은 LDPC 디코더가 아닌 MIMO demapper의 LLR 거리계산 경량화라, MIMO/채널행렬이 없는 NAND binary QC-LDPC ECC(Prime ECC)에 떼어 쓸 접점이 사실상 없다. shift 기반 max+min 근사 아이디어만 일반 참고 수준.
