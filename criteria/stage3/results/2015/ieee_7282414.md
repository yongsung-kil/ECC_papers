> 총평: PEG 계열 H-matrix(Tanner graph) 구성 알고리즘을 다단계 lookahead와 ACE 제약으로 확장해 girth/cycle-connectivity를 높여 error-floor를 낮추는 코드 설계 기법으로, Prime ECC의 H-matrix 생성(`Load_PCM`) 단계에 원리적으로 이식 검토 가능하나 QC-LDPC circulant 구조로의 변환 및 재검증 부담이 있음.

### B. 알고리즘 요약

1. 대상: irregular LDPC 코드의 parity-check H-matrix(Tanner graph, TG)를 오프라인으로 구성하는 PEG(progressive edge-growth) 계열 알고리즘 개선. rate 1/2, length 1008, VN degree distribution 다항식 기반 시뮬레이션.
2. 문제: 기존 PEG는 매 stage에서 현재 edge만 국소적으로 girth를 최대화(greedy)하여, VN의 최종 local girth가 최적이 아닐 수 있음. ACE-constrained PEG는 girth 외에 cycle-connectivity(ACE)까지 고려하지만 여전히 1-step greedy.
3. 가정: standard sum-product algorithm(SPA) iterative decoding, BPSK/AWGN 채널, DE(density evolution)로 최적화된 VN degree 분포를 그대로 사용.
4. 핵심 기법 1단(MSPEG): `r`-step lookahead로 각 stage에서 CN 선택 시, 향후 `r`개 edge까지 established 되었다고 가정한 "r-step local girth"가 최대인 CN을 DFS(`Algorithm 2`)로 탐색해 선택. `r=1`이면 원래 PEG와 동일.
5. 핵심 식: r-step 계산 복잡도 `O(m^r·E + m^(r-1)·E²)`(BFS 기반, BBV) — r이 커질수록 재탐색 비용이 급증하는 것이 문제.
6. 핵심 기법 2단(MSACPEG): MSPEG에 ACE constrained PEG의 minimum path ACE metric 선택 기준을 결합, 선택 우선순위는 `multi-step local girth 최대 → 해당 girth에서 ACE metric 최대 → 기존 ACE-PEG 규칙`.
7. 부수 트릭(DBV, distance-based version): distVC(CN-vc 거리 벡터)와 distCC(CN쌍 거리 행렬)를 증분 갱신하는 방식으로 BFS 재탐색을 없애 계산량을 `O(m^r·E + m²n)`로 낮춤 — `r>1`일 때 BFS 기반(BBV)보다 훨씬 효율적.
8. 최적화 절차: DFS 탐색 시 distance sequence(DS)가 nonincreasing이 되도록 TCNS(tried-check-node sequence) 순서를 재배열하는 Theorem 1을 이용해 탐색 공간을 가지치기.
9. 결과: ACE spectrum이 `r`(algorithm-step) 증가에 따라 커짐(예: B4가 가장 큰 ACE spectrum), BER 시뮬(Eb/N0 2.2~2.9dB)에서 B4가 가장 낮은 BER, 즉 ACE-property가 클수록 error-floor가 낮아짐을 확인.
10. 한계: 매우 짧은 코드(n=1008)·중간 rate(0.5)에서만 검증, HW 구현·복잡도(gate/throughput) 논의 전혀 없음, `r` 증가 시 설계 시간(오프라인)이 늘어나는 트레이드오프가 있고 고rate QC-LDPC(circulant 구조)로의 확장은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix/Tanner graph 구성(PEG/ACE) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 생성하는 부호설계 단계 — 다만 Prime ECC는 QC-LDPC(base+circulant shift) 구조라 직접적인 edge-by-edge PEG 구성과는 매핑 난이도 있음 |
| error-floor 개선(cycle-connectivity/girth 최적화) | 대응 없음 | 디코더 알고리즘이 아닌 코드 구성 단계 개선이라 `decoder.cpp`에 직접 대응 없음, H-matrix 설계 시 참고 가능 |
| ACE spectrum 기반 trapping-set 회피 | 대응 없음 | Prime ECC 프로파일 상 trapping-set/ACE 관련 명시적 모듈 없음 |

적용 가치: **중간** — 순수 부호구성(H-matrix 설계) 개선으로 Prime ECC의 고정 QC-LDPC 구조 재검증 부담(§4 "부호설계" 이식난이도=높음)이 있으나, error-floor 저감이라는 목표와 구성법 원리(girth/ACE 최적화)는 QC-LDPC 설계 시에도 참고 가치가 있음.
