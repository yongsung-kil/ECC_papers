### B. 알고리즘 요약

1. 시스템: 소스 S1, S2와 릴레이 R로 구성된 cascade 기반 2-소스 코드협력(coded cooperation) 시스템. 각 소스는 LDPC-S1/S2로 부호화하고, 릴레이는 두 코드워드를 cascade한 뒤 LDPC-R로 추가 패리티 `p`만 목적지에 전송.
2. 문제: 기존 방식(개별 소스/릴레이 LDPC를 독립 설계)은 결합 패리티검사행렬 `H`에서 소스 간(Type-II) girth-4 사이클을 방지하지 못해 BP 복호 성능이 저하됨.
3. 채널/모델: Rayleigh fading(협력 vs 비협력 비교) 및 AWGN(joint/separate/random LDPC 비교) 두 시나리오로 시뮬레이션.
4. 핵심 기법: base matrix `M(H)`와 exponent matrix `E(H)`로 QC-LDPC를 정의(`H`의 각 블록은 `p_{j,l}`-순환시프트 항등행렬). 결합 `H`는 `H1`, `H2`, `HR=[A B C]`를 블록 대각+하단행으로 배치.
5. 핵심 식: girth-4 사이클을 이루는 네 원소의 순환시프트 값이 `(p_jk,lk − p_jk+1,lk) + (p_jk+1,lk+1 − p_jk,lk+1) ≠ 0 (mod B)`를 만족해야 사이클이 사라짐(Theorem 1).
6. 확장: Type-I(개별 코드 내부) 사이클은 base matrix 구성(bit-filling)으로, Type-II(코드 간) 사이클은 exponent matrix의 순환시프트 값 할당(Theorem 1)으로 각각 제거하는 2단계 절차.
7. 구현 디테일: exponent matrix의 "1" 항목을 열 단위로 순차 배정하되 무작위가 아니라 Theorem 1 조건을 만족하는 값만 선택. bit-filling은 지수시간 무작위 구성법보다 계산복잡도가 낮음.
8. 최적화 절차: bit-filling 기반 base matrix 구성(Step 1) + 조건부 exponent 값 배정(Step 2), 별도 학습/DE 없음.
9. 결과: Rayleigh 채널에서 협력 방식이 비협력 대비 BER 10⁻⁴에서 약 5dB 이득. AWGN에서 joint design이 separate design 및 random LDPC보다 우수(10회 반복 후 차이 명확).
10. 한계: 릴레이/다중소스 협력통신 특화 구조이며 HW 설계·복잡도·throughput 분석 없음. 순수 시뮬레이션 수준의 부호구성 논문.
