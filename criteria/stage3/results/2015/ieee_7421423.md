### B. 알고리즘 요약

1. 대상 코드는 (3,6,72) protograph 기반 QC 구조 SC-LDPC 코드(`dv=3`, `dc=6`), 슬라이딩 윈도우 디코딩(윈도우 크기 `W=15`, partial syndrome-check window `Ws=8`, step `s=2×Z`) 적용.
2. 문제의식: 나노공정(sub-45nm) 임베디드 메모리는 공정편차로 stuck-at 결함이 발생하는데, SC-LDPC 디코더의 결함 저항성은 아직 연구되지 않은 open question.
3. 채널모델은 BPSK+AWGN이며, 메모리 결함은 stuck-at fault 모델(각 bit-cell이 확률 `Ps`로 독립적 stuck-at 0/1, 균등분포)로 모델링, 채널 메모리·LLR 윈도우 메모리 등 디코더 내 여러 메모리 블록에 적용.
4. 핵심 기법 1: MSB protection - LLR의 부호(sign)를 담는 MSB만 신뢰성 높은 셀로 보호(`Ps=0` 가정), 나머지 비트는 결함 허용.
5. 핵심 기법 2: Self-Corrected Min-Sum Algorithm(SCMSA) - 반복이 진행될수록 VN 출력 메시지의 부호가 자주 바뀌지 않아야 한다는 가정 하에, 반복 간 부호 flip 발생 시 결함으로 간주해 해당 메시지에 erasure를 삽입(2's-complement 표현에서 MSB 임계값 0x80 기준).
6. LLR은 8-bit fixed-point Q6.2(부호+정수 6비트, 소수 2비트) 표현 사용.
7. 부수 트릭: stuck-at fault는 message symmetry를 깨서 DE의 all-zero codeword 가정이 무효화되지만, XOR 기반 무작위화로 stuck-at을 random bit-flip으로 변환해 symmetry 복원 가능(단 self-correction과는 DE 결합 불가로 계산 불가능).
8. 학습/최적화 절차 없음(오프라인 DE 분석은 결함 있는 SC-LDPC에 [13],[20] 결합으로 유도 가능하다고만 언급, 상세 생략).
9. 결과: 채널메모리만 결함 시 `Ps=10^-4`에서 SNR 손실 0.15dB(BER 10^-4 기준), MSB protection 적용 시 손실 0.1dB로 감소. 전체 메모리 결함(`Ps=10^-3`) 시에도 MSB protection+SCMSA 병용으로 열화를 거의 0으로 완화.
10. 한계: 특정 (3,6,72) SC-LDPC 앙상블 한정 검증, HW 구현/합성 없이 Monte Carlo+DE 시뮬 수준, self-correction과 조기종료(early termination) 결합 시 DE 분석 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Self-Corrected Min-Sum(SCMSA, 부호 flip 기반 erasure 삽입) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Min-Sum 반복 루프 내 VN 메시지 부호 이력을 추적해 erasure를 주입하는 후처리로, 이터레이션 루프에 낮은 난이도로 삽입 가능 |
| MSB protection (LLR sign bit 보호) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` LLR 파라미터 | LLR 저장 비트 중 sign 비트 우선 보호 개념은 메모리 구조·양자화 설계에 참고 가능 |
| SC-LDPC 슬라이딩 윈도우 디코딩 / LLR 윈도우 메모리 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 블록코드이며 SC-LDPC 슬라이딩 윈도우 구조 자체는 대응 없음 |
| Stuck-at fault 모델 / DE 결함 분석 | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 결함 주입·DE 분석 프레임워크 없음 |

적용 가치: `중간` — SCMSA(부호 flip 기반 self-correction)는 min-sum 디코더 루프에 낮은 비용으로 이식 가능한 오류완화 기법이나, 논문의 주 대상인 SC-LDPC 구조와 stuck-at 메모리 결함 모델 자체는 Prime ECC(QC-LDPC 블록코드, bit-exact 시뮬레이터)와 직접 대응되지 않음.
