### B. 알고리즘 요약

1. 시스템: Gaussian wiretap channel — Alice가 비밀메시지 M(ks bit)과 랜덤메시지 R(kr bit)을 조합해 code rate `Rc=(ks+kr)/n`, 비밀메시지율 `Rs=ks/n`로 부호화, Bob(정당 수신자)과 Eve(도청자)에게 각각 다른 AWGN 채널로 전송.
2. 풀려는 문제: 기존 연구는 무한길이(asymptotic) 코드에서만 보안성능(secrecy capacity)을 다뤘고, 유한길이 실용 부호의 보안성능 설계/평가 기법이 부족.
3. 핵심 가정: Eve와 동일한 채널을 갖지만 비밀메시지 M을 알고 있는 가상의 수신자 "Frank"를 도입해, Frank의 복호오류율 `η`로 Eve의 equivocation rate 하한 `Re* = Rc - CE - (Rc-Rs)η - 1/n`을 유도.
4. 핵심 기법: Bob의 부호(전체 `H`)와 Frank의 부호(서브행렬 `H'=[B|C]`)를 동시에 만족하는 조건 하에 density evolution 기반 공동 최적화로 `λ(x)`, `ρ(x)` degree distribution 설계.
5. 핵심 식: 공동최적화 제약 C5 `λ̃_{B,i} ≥ λ̃_{F,i}` — Bob의 Tanner graph가 Frank의 그래프를 부분집합으로 포함해야 함을 강제, 이를 통해 두 부호가 동시에 최적점 근처에 있도록 유도.
6. 확장: check node degree distribution은 두 개 정도(concentrated)로 제한해 `ρ̃(x) = ax^⌊cm⌋ + bx^⌈cm⌉` 형태로 단순화.
7. 구현 디테일: PEG(Progressive Edge Growth) 알고리즘으로 실제 parity-check 행렬 생성, systematic 인코딩 가정(실무에서는 스크램블링 필요 언급).
8. 최적화 절차: 단일 부호용 density evolution(Gaussian approximation, [13] 방식)에 공동최적화 제약(C5) 추가한 변형 DE.
9. 결과: n=10000 길이에서도 fractional equivocation rate가 0.8 근처까지 도달, Rc=0.75일 때 유한길이-무한길이 성능 gap이 0.2~0.4dB; 기존 punctured LDPC 방식(n=10^6) 대비 짧은 길이에서도 우월한 경우 존재(Rs=0.43).
10. 한계: HW 미설계, 시뮬레이션(density evolution + PEG)에 국한, AWGN 무선 wiretap channel 전용 모델로 NAND flash 채널(RBER 등)과 무관, 정정능력/error-floor 개선이 목표가 아니라 정보이론적 보안 메트릭이 목표.
