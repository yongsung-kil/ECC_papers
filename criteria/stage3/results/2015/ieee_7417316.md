### B. 알고리즘 요약

1. 대상은 (1024, 512) Polar code, rate 0.5~7/8, BP 기반 디코딩, factor graph의 stage 수 `m = log2(n)`.
2. 기존 min-sum(MS) BP는 매 iteration마다 전체 factor graph를 순회해 연산량이 크고 SC 디코딩 대비 throughput 병목이 있다.
3. Polar code의 재귀적 인코딩 구조상 factor graph 내부에 반복적으로 나타나는 짧은 constituent code(부분 코드)들이 존재한다는 것이 핵심 가정.
4. 4종 constituent code를 정의: all-frozen `N0`, all-information `N1`, repetition `NREP`, single-parity-check `NSPC`. 각각에 대해 메시지 갱신 규칙을 단순화.
5. `NREP` 갱신식 `Ri,j = Σ Lk,j`(6), `NSPC` 갱신식 `Ri,j = sgn(·)·min(·)`(7)로 원래 O(l log l) 연산을 O(l)로 축소.
6. round-trip 스케줄링: 기존 conventional scheduling(좌→우 단방향 반복) 대신 각 iteration을 L 방향/R 방향 두 단계로 분리해 갱신 효율을 높임(Fig. 3).
7. 조기종료 기법: 추정 코드워드 x̂에 대해 `x̂H = 0`(parity check) 검사로 종료 판단, uˆ 없이도 적용 가능.
8. 별도 학습/최적화 절차 없음(고정 알고리즘, DE/GA/NN 미사용).
9. 결과: round-trip scheduling만으로 평균 iteration 83.7% 감소(24.5→3.98@Eb/N0=3.5), XJ 단순화로 iteration당 연산 40.6% 추가 절감, 전체 90.4% 복잡도 절감, SMS와 동등 성능.
10. 한계: HW 구현 미설계(시뮬레이션 단), (1024,512) 단일 코드 예제 중심, Polar code 전용이라 LDPC Tanner graph에는 직접 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 디코딩 알고리즘 전체 (constituent code 단순화) | 대응 없음 | Polar code factor graph 전용 구조로 QC-LDPC Tanner graph와 불일치 |
| round-trip scheduling | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 스케줄과 개념적 유사) | 이미 보유한 Dual-Update(odd/even 교대) 스케줄과 유사한 아이디어이나 Polar 전용 구현이라 직접 이식 불가 |
| 조기종료(parity check 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념 자체는 이미 Prime ECC에 CRC 기반으로 구현됨, 중복 기여 |

> 적용 가치: 낮음 — Polar code 전용 factor-graph 단순화 기법이며 부호구조가 달라 QC-LDPC로 직접 이식 불가. round-trip scheduling 개념만 참고 가치 있으나 이미 유사 기법(Dual-Update) 보유.
