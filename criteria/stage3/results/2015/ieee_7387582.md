### B. 알고리즘 요약

1. Mother QC-LDPC 부호(rate 1/2, z=64, 정보블록 1152bit)를 기반으로 확장(rate 1/3까지)·펑처링(rate 3/4까지)으로 rate-compatible 패밀리를 구성.
2. 기존 RC LDPC는 mother code가 근사적 linear encoding만 가능해 인코딩 복잡도가 부호길이에 완전히 비례하지 않는 문제를 해결하고자 함.
3. 채널은 BPSK + AWGN, BP(belief propagation) 디코딩 가정, 최대 50 iteration.
4. 파리티 검사행렬의 parity part를 lower triangular(LT) 형태(`I(S_j,i)` shifted identity 블록 배치)로 설계, 재귀적 인코딩 전략(식 4~10)으로 파리티 비트 `q_1,...,q_m`을 순차 계산.
5. 인코딩 복잡도 `M = Σ(ρ_j+2) ≈ n(1-R)(ρ+2)` (ρ=평균 row weight) — 부호길이 n에 완전 비례하는 linear complexity를 보장하는 핵심 식.
6. Girth 최적화를 위한 LSCO(layered short-cycle optimization) 알고리즘: shifting value `S_a,b`에 대해 theory 1(식 12, mod z 조건)로 4-girth/6-girth를 배제하도록 particle-swarm류 탐색.
7. 시스템 부분(systematic part)은 PEG(progressive edge-growth) 알고리즘으로 구성, degree-2 변수노드를 파리티 비트에 배정해 펑처링에 적합하게 설계.
8. Degree distribution은 layered density evolution([4] 방식 재사용)으로 각 rate별 최적화; 펑처링은 [1]의 progressive node puncturing 채택.
9. 결과: mother code girth 8, rate 2/5·1/3 girth 6; FER=10^-5, BER=10^-7까지 error floor 미관측; BER=10^-6에서 Shannon limit 대비 1.477dB 갭; rate 0.4에서 기존[4] 대비 0.28dB, rate 0.7 대비[1] 1dB(FER=10^-4) 이득.
10. 한계: 시뮬레이션(AWGN/BPSK)만 수행, HW 구현·게이트카운트·throughput 언급 없음. 정보블록 1k 고정 크기에서만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LT form 재귀 인코딩 (식 4~10) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코더 사용, 구조가 달라 직접 이식은 어려움 |
| H-matrix / mother code 구성 (Fig.1) | `ecc_top.cpp` `Load_PCM()` | 특정 QC-LDPC 고정 설계라 재검증 부담 큼 |
| LSCO girth 최적화 | 대응 없음 | 부호설계 단계 도구로 Prime ECC 코드 구조에 대응 모듈 없음 |
| Rate-compatible puncturing/extension | 대응 없음 | Prime ECC는 고정 rate 구조, RC 개념 자체 대응 없음 |

적용 가치: 낮음 — 순수 QC-LDPC 부호구성(RC, LT-encoding) 논문으로 Prime ECC의 고정 고rate 구조와 직접 대응되지 않으며 디코더/HW 개선 요소가 없음.
