### B. 알고리즘 요약

1. 대상은 결맞음 광통신(coherent optical) 100Gbit/s급 FEC, 목표 BER 10^-15, soft-decision 디코딩(고해상도 ADC 활용).
2. 기존 LDPC/BTC(block turbo code)는 낮은 오버헤드를 위해 100,000비트 이상의 긴 블록 길이가 필요하거나 hard-decision에서 성능이 떨어지는 문제.
3. 채널모델은 hard-decision AWGN(간접 비교) 및 통상적 AWGN 광채널 가정, 별도의 NAND/저장 채널 모델 없음.
4. 핵심 기법: Spatially-Coupled LDPC(SC-LDPC)는 여러 sparse PCM `H_i(t)`를 컨볼루션적으로 이어붙인 무한 확장 구조(`H_sc`), threshold saturation으로 BP 복호 성능이 ML에 근접.
5. 핵심 식은 `H_sc x^T = 0` (구조화된 무한 PCM), windowed decoder로 유한 레이턴시 복호 가능.
6. 비교 대상으로 Polar code(길이 32,768, successive cancellation decoder, 복잡도 O(N log N))도 함께 다룸.
7. 세 코드(systematic Polar, block LDPC, weakly-coupled SC-LDPC) 모두 rate 0.8, 약 32,000비트, 유사 복잡도(~25 iteration/bit) 조건으로 통제해 비교.
8. 최적화 절차 없음(설계 비교 위주), windowed decoder의 window 수 w·반복 I 설정이 성능에 결정적임을 언급(잘못 설정 시 성능 손실 도시).
9. 결과: 시뮬레이션에서 SC-LDPC가 block LDPC 및 systematic Polar 대비 우수한 BER 성능(특히 error-floor 영역)을 보임(Fig.2, 정량 수치는 그래프에만 존재).
10. 한계: 2쪽 초청 발표 요약으로 상세 부호구성/디코더 HW 설계·검증 없음, 결과 수치는 그래프에만 표시되어 본문 서술 수치 부재, 광통신 특화 스케일(32k+ 길이)이라 NAND ECC와 rate/length 격차 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호구성(H_sc 컨볼루션 구조) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조라 SC-LDPC 채택 시 H-matrix 전면 재설계 필요(이식 난이도 높음) |
| Windowed decoder | 대응 없음 | Prime ECC는 codeword 단위 block 디코더로 SC 코드용 windowed 복호 구조 없음 |
| Polar code / successive cancellation | 대응 없음 | binary QC-LDPC 전용 코드베이스와 무관한 코드 계열 |
| Error-floor 개선(SC-LDPC threshold saturation) | 대응 없음 | 개념은 흥미로우나 본문에 구체적 메커니즘·수치 없어 판정 불가 |

> 적용 가치: **낮음** — 광통신용 SC-LDPC/Polar 개념 비교를 다루는 초청 발표 요약(2쪽)으로 구체적 알고리즘·HW 설계가 없고, NAND binary QC-LDPC 고정 구조인 Prime ECC 3.1에 이식하려면 부호설계 전면 재작업이 필요해 실익이 낮음.
