> 총평: FPGA 자원 증가 없이 stochastic decoder의 내부 메모리를 확장해 수렴성을 개선한 하드웨어 최적화이나, stochastic computing 자체가 Prime ECC의 binary min-sum LDPC 구조와 근본적으로 이질적이라 이식 가치가 낮음.

### B. 알고리즘 요약

1. 시스템: `(n,k)` regular LDPC, 예시로 medium `(1024,512)`·short `(200,100)` 코드를 fully-parallel stochastic decoder로 구현.
2. 문제: fully-parallel LDPC 디코더는 throughput은 높지만 HW 복잡도가 크고, 기존 stochastic decoder(CSS, Delayed Stochastic)는 variable node(VN) 내부 메모리(IM)가 2bit로 제한되어 수렴 속도가 느림.
3. 핵심 가정/모델: 확률값 `Pch`를 Bernoulli random bit sequence로 변환해 비트-시리얼로 처리하는 stochastic computing 기반 디코딩(연판정 LLR 대신 확률 비트열 사용).
4. 핵심 기법: 기존 CSS decoder의 VN 구조를 개선해 disagreement(hold) state에서 IM 내 무작위 주소로 저장된 과거 비트 중 하나를 랜덤 선택해 출력 — IM 길이를 2bit에서 최대 64bit로 확장.
5. 핵심 식: `Pvout` 갱신 시 agreement/disagreement 판정 후 `Pvout=1 if (M_in AND random_bit=1) & (IM length ≤ 64)`(식 7) — IM 길이 확장이 수렴성 개선의 핵심.
6. 확장: degree-3 VN을 3개의 sub-node로 분해, majority state 신호와 random address를 모든 sub-node에 공유해 최종 출력 비트를 결정.
7. 구현 디테일: FPGA distributed RAM(SLICE)으로 IM 구현, decoding cycle(DC)은 1 iteration에 대응, 첫 cycle은 `Pch(k-1)`을 그대로 VN 출력 DFF에 전달(초기화).
8. 학습/최적화 절차: 별도 없음(구조적 HW 개선).
9. 결과: Xilinx Virtex-6 VLX240T에서 (200,100) 코드는 IM 2bit→64bit 확장 시 Slice LUT 사용량(logic 5507/memory 600/VN용 4200)이 변화 없음, (1024,512) 코드도 동일(logic 26561/memory 3072/VN용 21504) 유지.
10. 한계: BER/BLER 곡선, iteration 수 감소량, throughput 수치가 본문에 제시되지 않음(정성적 "수렴 개선"만 주장); low-cost FPGA(Spartan-3)에서는 IM 길이가 32bit로 제한됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stochastic VN/CN 연산(Bernoulli 확률 비트열) | 대응 없음 | Prime ECC는 결정론적 min-sum LLR 기반, stochastic computing 패러다임 자체가 없음 |
| VN 내부 메모리(IM) 기반 랜덤 선택 구조 | 대응 없음 | `decoder.cpp`의 VN 처리는 LLR magnitude 갱신 방식이라 구조적으로 무관 |
| fully-parallel FPGA 구현 | `decoder.cpp` `LDPC_Decoder()` (병렬화 구조 참고용) | 완전병렬 개념만 참고 가능, 구체 회로는 무관 |

> 적용 가치: 낮음 — stochastic LDPC decoding은 Prime ECC의 min-sum 기반 binary QC-LDPC 구조와 알고리즘 패러다임이 근본적으로 달라 이식 불가에 가까움.
