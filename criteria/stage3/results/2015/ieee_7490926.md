### B. 알고리즘 요약

1. 대상 코드는 CCSDS 표준 rate-1/2 (480, 240) QC-LDPC이며, 아날로그 VLSI 상에서 연속시간(continuous-time) iterative decoding으로 구현.
2. 기존 디지털 표준정지기준(SSC)은 오류불가능(uncorrectable) 블록을 구분 못해 저-중 SNR에서 전력·시간 낭비가 크고, 아날로그 디코더의 연속적 동작 특성상 SSC 자체를 적용하기 어려움.
3. 채널은 memoryless binary-input output-symmetric channel로 가정, factor graph 기반 sum-product 알고리즘으로 복호.
4. 각 check node `c_j`에 확률 전파용 보조 변수 `s_j`를 추가해 probabilities of satisfied checks(PCS)를 계산하는 별도 tree 구조(OR-gate tree)를 부가.
5. `s_j`는 `c_j`로부터 메시지를 받되 다시 `x_j`로 메시지를 보내지 않아 기존 복호 경로에 영향을 주지 않음 — 순수 모니터링 branch.
6. 전체 PCS가 특정 임계 확률 `P`를 넘으면 조기종료; `P`는 실험적으로 0.5~0.9 범위에서 스윕.
7. 회로는 Gilbert multiplier 기반 soft-XOR/soft-EQU 모듈과 정규화 회로(PMOS 기반)로 구성, 처리 지연은 1차 RC delay 근사(`τ`)로 모델링해 이산시간 재귀식으로 시뮬레이션.
8. 별도 학습/최적화 절차 없음 — `P` 값을 0.5~0.9로 스윕한 시뮬레이션 비교만 수행.
9. 결과: `P=0.5`일 때 성능 저하 없이 평균 iteration을 크게 감소, `P`가 커질수록 iteration은 급증하나 BER/FER 성능엔 큰 영향 없음 (구체 수치 그래프로만 제시, 본문 텍스트에 수치 없음).
10. 한계: 순수 시뮬레이션(behavioral model)뿐이며 실제 HW(ASIC) 설계·합성 없음, 아날로그 특화 구조라 디지털 구현과 직접 비교 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 조기종료(check-satisfied probability 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념적으로 조기종료라는 목적은 같으나 CRC 기반이 아닌 아날로그 확률 누적 방식이라 직접 대응 불가 |
| 아날로그 sum-product 회로(Gilbert multiplier 등) | 대응 없음 | Prime ECC는 bit-exact 디지털 시뮬레이터로 아날로그 회로 모델 없음 |
| CN/VN 업데이트(factor graph) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 개념(sum-product 반복)만 유사, 아날로그 회로 구현이라 실질 이식 불가 |

> 적용 가치: 낮음 — 아날로그 VLSI 전용 구현으로 Prime ECC의 디지털 bit-exact 아키텍처와 계층이 달라 조기종료 "개념"만 참고 가능하고 실질 이식 요소 없음.
