> 총평: 대형 통신용 SPA LDPC 디코더를 FPGA에 loop-unroll/hybrid로 구현할 때의 자원·throughput·latency trade-space 설계 방법론 논문으로, 신규 알고리즘·부호설계 없이 엔지니어링 지침만 제공해 NAND QC-LDPC ASIC 파이프라인 이식 가치가 낮음.

### B. 알고리즘 요약

1. 대상: 대형 regular LDPC(PCM 8K~64K element), rate 1/2, 데이터율 1Mbps 이상, full-tanh Sum-Product Algorithm(SPA), FPGA 구현.
2. 문제: 대형 부호 + 고데이터율 + 다수 iteration으로 near-Shannon 성능을 얻으려면 FPGA 자원·latency가 급증, 이를 정량화하는 설계지침 부재.
3. SPA 4단계: 초기화 → horizontal step(행 비영원소 곱, CN) → vertical step(열 비영원소 곱, VN) → decoding check(조기종료 판정) 반복.
4. 메모리 모델: regular 부호라 저장 비영원소 수 `N_Mem = wRow·N_Row = wCol·N_Col`, 행/열 인덱스+값 sparse 저장.
5. 설계식: 디코더율 `R_DECODER = R_DATA/R_CODE`(식2), 블록율 `R_BLK = R_DECODER/LDPC_BLK_SIZE`(식3), 단일반복시간 `t_ITER = 1/(R_BLK·N_MAX_ITER)`(식4).
6. latency: `t_LATENCY = t_INIT + N_MAX_ITER·(t_H+t_V) + t_CHK`(식5), codeword 추정은 다음 iteration과 병렬 실행 가정.
7. 아키텍처 선택: `t_ITER`와 FPGA 클록으로 결정 — closed-loop(최소자원)/완전 unrolled(최대자원, N_MAX_ITER배)/hybrid, N개 병렬 디코더로 throughput N배.
8. 자원 산정: FPGA 개수 `M_F = ceil((R_IN + N_MAX_ITER·R_HV)/(K_CHIP·K_UTIL) + R_DC)`(식6), DSP48/BlockRAM/Logic 자원 타입별 계산.
9. 예제: 100Mbps, rate 1/2, 64K 부호, N_MAX_ITER=30 → t_ITER≈10.92us, Virtex-7 485T @400MHz, hybrid 구조로 총 7개 FPGA 필요, Block RAM이 지배 자원.
10. 한계: 신규 알고리즘·부호설계 없음(설계방법론·노모그래프뿐), BER 곡선 없음, full-tanh SPA·대형 통신부호 전제로 NAND ECC와 무관.

|---|---|---|

| latency/iteration 자원 산정식(식4~6) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` 주변 통계 | 자원·latency 견적 방법론일 뿐 알고리즘 대응 없음 |
| decoding check(조기종료 판정) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 개념적으로 조기종료에 해당하나 논문은 codeword estimate 단순 언급뿐 |
| loop-unroll/hybrid FPGA 아키텍처 | 대응 없음 | 우리는 ASIC급 z=32 고정 파이프라인, FPGA loop-unroll 방법론 비적용 |
