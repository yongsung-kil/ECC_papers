### B. 알고리즘 요약

1. 시스템은 (504,252)/(1056,528) rate-1/2 regular LDPC를 8PSK/16QAM SP-labelling과 결합한 BICM(bit-interleaved coded modulation) 시스템으로, demapper와 LDPC 디코더를 하나의 belief propagation 그래프로 결합한다.
2. 기존 BICM-ID는 demapper-decoder 간 반복이 별도라 복잡도가 높고, Brink의 joint iterative 알고리즘도 check node 갱신에 `tanh` 연산이 남아 복잡한 것이 문제다.
3. 채널은 AWGN 가정, 초기 LLR은 demapper의 Max-Log-MAP 출력을 사용한다.
4. 핵심 기법은 check node 갱신식(`tanh` sum-product, 식(3))을 UMP-BP(min-sum 유사) `r_{m,(k,i)} = ⊕sign · min|q|`(식(11))로 대체해 지수 연산을 최소값 비교로 단순화한다.
5. UMP-BP는 정확도가 떨어져 large column weight LDPC에서 1dB 이상 성능 손실이 발생하므로, consistency condition(`ln[p(L|b=1)/p(L|b=0)] = L`, 식(12))을 만족하도록 check node 출력 LLR에 상수 스케일 팩터(예: 0.8)를 곱해 보정한다.
6. 추가로 초기 demapper LLR도 GMI(generalized mutual information) 최대화 기준으로 별도 스케일 팩터 `s_i`를 찾아 보정한다(식(9)(10)).
7. 구현 디테일로 GMI 계산 시 기대값 연산 대신 합 연산으로 근사하고, 시뮬레이션상 SNR 관심구간에서 check node 스케일 팩터가 반복 전 구간 상수(0.8)로 수렴함을 확인해 매 iteration 재계산을 생략한다.
8. 스케일 팩터 탐색은 BER 곡선을 스캔하는 방식(그림3)으로 결정하며, 별도 GA/NN 학습은 없다.
9. 결과는 BER 10^-5 기준 제안기법이 BICM-ID 대비 0.6~0.8dB, 표준 joint iterative 대비 0.2dB 이득을 보였다.
10. 한계는 HW 설계·gatecount·throughput 수치가 전혀 없고 무선 변조(8PSK/16QAM) BICM 채널에 특화되어 NAND read 채널 모델과 무관하다는 점이다.

| check node LLR 스케일 팩터 보정 | `decoder.cpp` `LDPC_Decoder()`, `ecc_data.h` `PARAM_LLR` | normalized min-sum factor와 유사 개념이나 Prime ECC 구현 세부(양자화 테이블) 방식과 불일치, 참고용 |
| 초기 demapper LLR GMI 보정 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | BICM 변조 특화 보정이라 NAND soft-read threshold와 직접 대응 어려움 |
| joint demapping-decoding 그래프 결합 | 대응 없음 | NAND 채널엔 변조 demapper가 없어 구조 자체가 무관 |
