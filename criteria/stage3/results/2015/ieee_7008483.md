### B. 알고리즘 요약

1. 시스템: flash memory용 BCH(inner group) + LDPC(outer) concatenation; 정보비트를 β그룹으로 나눠 각 BCH 인코딩 후 결합해 LDPC 입력 `KLDPC=ΣNj`, LDPC codeword `NLDPC=18480`, overall rate ~0.9 (rate-0.95 BCH × rate-0.95 LDPC).
2. 문제: LDPC 단독은 waterfall은 좋지만 flash가 요구하는 극저 error rate(10^-12)에서 error floor 발생; concat이 해법이나 rate-loss로 waterfall 성능 저하.
3. 채널 모델: AWGN을 BSC crossover ε로 근사, ε=10^-2~10^-3 영역에서 BCH UER/BLER 분석.
4. 디코딩: outer iteration(BCH↔LDPC) + LDPC 내부 min-sum(MSA); 성공한 BCH 블록 비트 신뢰도를 ∞로 saturate해 LDPC가 해당 hard decision을 못 바꾸게 함(식(1)).
5. MSA 핵심: `Ei,j = α·si,j·min|Li',j|` (α normalization scale, si,j는 sign 곱), 성공 BCH 비트는 `Ri=2(1-2d̂i)/σ²`로 초기화.
6. 기여1 - outer iteration 수정: [6]의 max outer iteration 상한 대신, 직전 대비 성공 BCH 블록이 늘 수 있는 한 계속 진행(`Bi>Bi-1`).
7. 기여2 - unequal-length BCH 선택: UER를 binomial weight 분포로 근사(식(2)), 목표 BLER Pt로 nmax 결정, residual BER(90%-tile) 곡선을 따라 [N1..N4]=[2000,2400,5800,7272]로 점증 선택.
8. 구현 디테일: unequal 길이지만 mother BCH code + shortening으로 단일 BCH HW 재사용 가능 → HW 복잡도는 equal-length와 유사.
9. 결과: column-weight-3 LDPC concat이 error floor 제거하며, equal-length는 등가 BCH t=187, 제안 unequal-length는 t=208 수준으로 waterfall 개선; raw BER<10^-3이면 BCH만으로 정정(Table I).
10. 한계: 시뮬레이션만(HW 미설계), gate count·throughput 미보고, LDPC 부호/디코더 자체가 아니라 outer BCH 계층 설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC min-sum(MSA) CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문 MSA는 우리 min-sum과 동일 계열 — 이미 보유, 신규성 없음 |
| 성공 BCH 비트 ∞ 포화로 LLR 초기화 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 외부(BCH) 확정 비트를 강신뢰 LLR로 주입하는 인터페이스 — 부분 대응, outer 계층 신규 필요 |
| 조기종료(BCH 전부 성공 시 종료) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 우리 CRC 조기종료와 개념 유사하나 논문은 BCH validity 기반 outer 종료 |
| outer BCH↔LDPC concatenation 계층 | 대응 없음 | 우리 코드는 단일 LDPC 엔진 — BCH inner/outer 계층 미보유 |

적용 가치: **중간** — NAND 직접 타깃이고 waterfall+error-floor 동시개선이 우리 관심사와 맞지만, 핵심 신규성이 LDPC 디코더가 아닌 outer BCH concatenation 계층 설계라 우리 단일 QC-LDPC 엔진에 떼어 붙이려면 BCH 계층 추가라는 아키텍처 변경이 필요하다.
