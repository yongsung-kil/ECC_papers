### B. 알고리즘 요약

1. 대상은 MLC NAND flash 기반 SSD로, in-page ECC(BCH 등)로 정정 불가능한 raw bit error를 across-chips 데이터 리던던시로 복구하는 H2-SSD 아키텍처를 제안.
2. 기존 across-chips 리던던시(CR5M, EECC)는 parity 갱신에 따른 추가 write가 MLC 칩 성능/수명을 저하시키거나 HDD 병목·충격저항 저하 문제가 있음.
3. 핵심 관찰: MLC는 write-intensive 환경에서 chip failure보다 uncorrectable bit error로 먼저 소진되므로, chip failure가 아닌 bit error 복구에만 집중하는 비대칭 parity 생성 규칙을 채택.
4. SLC 칩 1개를 전용 parity 칩으로 배치, MLC 칩들의 across-chips parity를 저장(RAID-4 parity disk와 유사). SLC는 MLC 대비 P/E latency·수명이 우수(예: program 0.23ms vs 1.6ms, 최대 P/E 100,000 vs 3,000).
5. Segment error rate `Ps`(식 3)와 uncorrectable BER `Pu`(식 4)를 BCH (n,k,t) 파라미터로 계산, 계층적 리던던시 적용 시 `Pstripe`, `PsH2`(식 5,6)로 재계산해 수명 개선을 정량화.
6. Strip(=segment, 자체 ECC checksum 보유)과 stripe(across-chips 복구 단위) 구조로 설계, MLC/SLC 페이지가 stripe 경계를 넘지 않도록 strip 크기 결정.
7. HW 구현 없음, 순수 시스템 레벨 시뮬레이션(Micro-Add-on SSD 시뮬레이터, 약 1,400줄 C코드 추가).
8. 최적화/학습 절차 없음 (해석적 확률 모델 + trace 기반 시뮬레이션).
9. 결과: MLC 칩 8개+SLC 1개 구성에서 P/E 수명 23~178% 개선, I/O 처리량 저하는 대부분 10% 미만(F1: 최대 13.0%, D1: 최대 19.5%).
10. 한계: BCH 코드/in-page ECC는 기존 기법 그대로 사용(수정 없음), 시스템 아키텍처·트레이스 시뮬레이션 수준이며 LDPC 자체 알고리즘·HW 설계는 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| across-chips parity/RAID형 데이터 리던던시 | 대응 없음 | Prime ECC는 단일 채널 in-page LDPC 디코더/인코더 시뮬레이터로 칩 간 RAID 구조를 다루지 않음 |
| in-page ECC(BCH) 정정능력 모델링 | 대응 없음 | 논문은 BCH를 예시로만 사용, 알고리즘 세부사항 없음 |
| SLC/MLC 이종 메모리 SSD 아키텍처 | 대응 없음 | HW 디코더/인코더 시뮬레이터 범위 밖 |

> 적용 가치: 낮음 — 시스템(SSD 아키텍처) 레벨 리던던시 논문으로 LDPC 부호설계·디코더·HW와 직접 관련 없음.
