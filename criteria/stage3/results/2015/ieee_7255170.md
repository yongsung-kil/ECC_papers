### B. 알고리즘 요약

1. 시스템은 BI-AWGN 채널, BPSK 변조, rate-compatible LDPC 코드(mother code rate `a/b=8/13`, `W=256`, 확장 `NE=12`)를 사용하는 무선 통신 링크.
2. 기존 RB-HARQ(`[4][6]`)는 개별 code bit 단위로 재전송 요청을 하여 피드백 오버헤드(`log2N`bit/index)가 크다는 문제를 해결하려 함.
3. 채널 모델은 표준 AWGN(`rk = uk + nk`)이며, 코드워드를 `S`-bit 클러스터로 나눠 클러스터 단위로 재전송을 요청하는 구조를 가정.
4. 핵심 기법은 디코딩 실패 시 각 클러스터의 "신뢰도 지표"를 계산해 가장 신뢰도 낮은 클러스터를 재전송 요청하는 것 — 우도비 `L(uk,ūk,rk) = Pr(uk|rk)/Pr(ūk|rk)`를 기반으로 3가지 지표(Metric A/B/C) 유도.
5. Metric C는 relative entropy(`D(p||q) = Σp(x)log(p(x)/q(x))`)로 수신분포와 Kronecker delta 분포 간 거리를 측정, `k̃ = argmax_k Σ_i(1+exp(-|Lk,i|))`로 최소 신뢰 클러스터 선택 — 세 지표 중 성능 최우수.
6. 클러스터 크기 `S`가 작을수록 재전송 정밀도는 높아지나 클러스터 인덱스 지정 비트수(`log2(N/S)`)가 늘어 피드백 오버헤드 증가, 이 트레이드오프를 정량 분석.
7. 부수 기법: 재전송된 클러스터는 maximum ratio combining으로 기존 수신값과 결합, mother code는 Ha's puncturing 알고리즘(`[2]`)으로 펑처링.
8. 별도의 학습/최적화 절차는 없음(분석적 유도 + 시뮬레이션 검증).
9. 결과: 피드백 오버헤드 보정 후에도 `S=48`(오버헤드 14%)에서 non-feedback 대비 최대 약 2dB 이득, Metric C가 Metric A(기존 `[7]` 휴리스틱)보다 throughput 우수.
10. 한계: 순수 시뮬레이션(AWGN/BPSK)뿐이며 HW 설계 없음, 근본적으로 피드백 채널이 존재하는 무선 ARQ 프로토콜이라 피드백이 없는 NAND flash 저장 채널에는 구조적으로 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Rate-compatible 부호 확장/펑처링 | 대응 없음 | Prime ECC는 고정 rate QC-LDPC(`Load_PCM()`)이며 rate-compatible 확장 구조 없음 |
| 클러스터 기반 재전송(HARQ) | 대응 없음 | NAND flash에는 피드백 채널/재전송 프로토콜 개념 자체가 부재 |
| LLR 기반 신뢰도 지표 계산 | `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 값을 참조한다는 점만 유사, 클러스터 재전송 목적의 지표로 디코더 내부 로직과는 무관 |

적용 가치: **낮음** — LDPC 부호를 사용하나 본질적으로 무선 링크의 재전송(ARQ) 프로토콜 설계 논문으로, NAND flash ECC의 디코더/부호설계/HW 어느 레이어에도 대응되지 않음.
