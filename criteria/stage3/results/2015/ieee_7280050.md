### B. 알고리즘 요약

1. 시스템: NAND Flash 블록별 적응형 ECC 시스템, ECC selection module + ECC codec module(Hamming/BCH(4122,4096,5)/BCH(4148,4096,9)), 섹터당 data `4096`bit.
2. 문제: 블록 신뢰도가 P/E cycle·retention time에 따라 변하는데 고정 ECC는 저BER 구간에서 정정능력·coding time 낭비.
3. 모델: 오류를 hard/soft error로 구분, soft(SEU/MBU)만 ECC 대상, retention>1day면 retention error 지배.
4. 핵심기법1: reliability map(P/E `N` × retention `T` 2D 테이블에 BER 저장), 환경센서로 `N,T` 측정 후 BER 조회.
5. 핵심식: 이중선형보간(bilinear interpolation) `BER = Σ BER_{i,j}·(면적가중)` 으로 격자 사이 BER 추정 (식 1).
6. 핵심기법2: UBER-BER 관계로 임계값 결정 — BER 구간별 no-ECC / 1bit Hamming / 2bit BCH / 4bit BCH 선택.
7. 구현디테일: TMR 기반 ECC flag bit(‘00’/‘010101’/‘101010’)로 어느 ECC인지 표시, check bit는 OOB에 저장.
8. 학습/최적화: 읽기 후 실측 BER로 reliability map 갱신(피드백).
9. 결과: ADSP-TS201(600MHz)에서 coding time 3.1x speedup, redundancy 1.25%→0.72~0.80%, UBER~1e-15 유지, throughput 871.5Mbps.
10. 한계: LDPC 없음(BCH/Hamming), HW 미설계(DSP 소프트 구현), 부호/복호 알고리즘 신규성 없음, 실제 NAND 적용은 future work.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Hamming/BCH suite 및 적응 선택 | 대응 없음 | Prime ECC는 QC-LDPC 전용, BCH/Hamming 부호 미보유 |
| reliability map(P/E·retention→BER) | 대응 없음 | 채널 신뢰도 추정 프레임워크로 부호/복호 코드에 대응 모듈 없음 |
| TMR ECC flag / OOB check bit 저장 | 대응 없음 | 시스템 레벨 저장 정책, Prime ECC 시뮬 범위 밖 |

적용 가치: **낮음** — NAND 직접 타깃이나 LDPC 부호·복호 알고리즘이 없고 Hamming/BCH 선택 프레임워크뿐이라 binary QC-LDPC 코드베이스에 이식할 요소가 없다.
