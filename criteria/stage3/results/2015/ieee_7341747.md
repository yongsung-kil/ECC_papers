### B. 알고리즘 요약

1. 시스템: 100Gbps급 장거리 광통신(coherent, DP-16-QAM/DPQPSK) 링크용 소프트결정 FEC. AWGN 채널, BPSK 모듈레이션으로 시뮬레이션.
2. 문제: 기존 NMSA(normalized min-sum) 기반 LDPC 디코더는 성능은 좋으나 회로 복잡도·에너지 소모가 커서(100Gbps 링크 전체 에너지의 15.8%) 저에너지 대안이 필요.
3. 핵심 가정: `(n,k)` LDPC, 희소 `H`(m×n), Tanner graph로 표현. 디코더는 채널에서 받은 LLR을 `q`비트로 양자화해 입력.
4. 핵심 기법 1단: IDB(Improved Differential Binary) 알고리즘 — NMSA와 달리 VN당 `q`비트 메모리 1개만 사용하고 노드 간 메시지는 **1-bit**만 전달(NMSA는 `q`비트 메시지 × `d_v`개 메모리). 대신 `d_v ≥ 6`인 코드가 필요.
5. 핵심 식/구조: `H`를 근사 하삼각형태(`T` 서브행렬이 lower unitriangular)로 만드는 Richardson-Urbanke(RU) 인코딩 사용, "gap size `g`"가 설계 파라미터. 이를 통해 인코딩 복잡도가 `O(n²)`(G행렬 곱)에서 `O(n+g²)`로 감소(실측 100~280배 XOR 연산 감소).
6. 핵심 기법 2단: `g`가 작으면 `H` 우하단이 밀집해 짧은 사이클·작은 trapping set이 생겨 error-floor가 높아짐 → `g=600`까지 늘리면 graph search+importance sampling으로도 trapping set이 발견되지 않아 BER 10⁻¹⁵ 이하로 추정.
7. 부수 트릭: irregular 구조(대부분 `d_v=6, d_c=36`, 일부 35개 노드만 `d_v=5, d_c=35`)로 full-rank 확보. IDB는 완전병렬 배선 복잡도가 낮아 fully-parallel ASIC 구현에 유리.
8. 최적화 절차: 별도 학습 없음. Monte Carlo 시뮬 + trapping set graph search + importance sampling(IS) 조합으로 error-floor 특성화, linear extrapolation으로 BER 10⁻¹⁵에서 NGC 추정.
9. 결과: (30000,25000)/(60000,50000) 코드에서 예측 NGC 10.55dB/10.75dB (NMSA 24576,20482 QC-LDPC의 10.7dB와 유사). ASIC 합성(65nm): 125Gbps/100Gbps throughput, 11.3pJ/bit·18.8pJ/bit (동급 NMSA 디코더 86pJ/bit 대비 대폭 절감), latency 200ns/500ns.
10. 한계: 검증된 코드가 초장블록(3만~6만 bit)·랜덤 irregular 구조로 NAND용 QC-LDPC(수천 bit급, 순환구조)와 코드 스펙 차이가 큼. Iteration 수(50/100회 고정 max)나 iteration 감소 효과에 대한 별도 분석은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IDB 저복잡도 디코딩(1-bit 메시지) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | min-sum 대체/보완 알고리즘으로 이터레이션 루프에 접목 가능성 있으나 1-bit 메시지 축소로 인한 성능손실 재검증 필요 |
| VN당 단일 q-bit 메모리 구조 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | VN 메모리 절감 아이디어는 HW 메모리 최적화 관점에서 참고 가능 |
| RU 인코딩(근사 하삼각 H, gap size) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코딩이라 RU 방식 도입은 인코더 구조 재설계(높음 난이도) |
| gap-size 기반 error-floor/trapping-set 억제 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 시 trapping set 회피 설계 원칙은 참고 가능하나 특정 QC-LDPC 고정이라 재검증 부담 큼 |
| 완전병렬 ASIC, 에너지 최적화(65nm 합성) | 디코더 HW 모델(z=32, HCU, GT) | 저에너지 완전병렬 아키텍처 방향은 Prime ECC HW 최적화와 목표가 유사하나 구현 스케일(3만~6만 bit vs 4KB급) 차이 큼 |

> 적용 가치: 중간. IDB의 1-bit 메시지 절감 아이디어와 저에너지 완전병렬 ASIC 설계 철학은 참고 가치가 있으나, 검증 코드가 NAND ECC와 스펙(길이·rate·랜덤구조)이 크게 달라 직접 이식보다는 개념적 참고에 그침.
