### B. 알고리즘 요약

1. 대상은 irregular binary LDPC(rate 1/2, N=2000, dv,max=5~50), AWGN·Rayleigh fading 채널, shuffled(variable-node 기반 serial) BP 디코딩.
2. 문제: 기존 shuffled BP는 variable node 처리 순서가 무작위이거나 고정이라 정보 갱신 활용이 비효율적이고, IDS류 개선은 매 반복 residual 계산이 필요해 오버헤드 발생.
3. 가정: 한 반복 내 처리되는 edge 수는 순서와 무관하게 동일하지만, 처리 순서에 따라 "aggregated messages"(다음 노드 처리 시점까지 갱신된 정보량)가 달라짐.
4. 핵심 기법: variable node를 column-weight(차수) 내림차순으로 정렬한 벡터 `S`를 오프라인 1회 생성, 이 순서대로 순차 갱신(고차수 노드 먼저 처리).
5. 핵심 식: `M_H-L = Σ_i [Σ_k(Σ_{j>i} j·Nλ~_j + ik)]` (고→저 순서의 누적 메시지 수)가 `M_L-H`(저→고 순서)나 `M_R`(무작위, `≈ d̄_v·N/2·(N+1)`)보다 항상 큼 — 이 값이 클수록 후속 노드가 더 최신 정보를 활용해 수렴이 빨라짐.
6. 확장 없음(단일 스케줄링 규칙), 최대 column-weight `dv,max`에 비례해 개선폭 증가.
7. 구현 디테일: run-time 계산 없이 오프라인 정렬만 필요해 IDS 대비 추가 복잡도 없음; PEG 알고리즘으로 H-matrix 구성(다른 구성법과도 호환 가능 언급).
8. 별도 학습 없음(Richardson-Shokrollahi-Urbanke 최적화 degree distribution 사용, Monte Carlo 시뮬로 mutual-information·iteration 수 측정).
9. 결과: dv,max=50 코드에서 AWGN 채널 기준 low-to-high 대비 약 37%, random 대비 약 29% 빠른 수렴(Rayleigh fading은 각 35%/22%); BER/FER은 스케줄 순서와 무관하게 동일(성능 열화 없음).
10. 한계: 순수 시뮬레이션(이론/알고리즘 수준)이며 HW 설계·게이트카운트·throughput 없음; irregular 코드의 불균등 column-weight 특성에 의존해 QC-LDPC의 고정/대칭 구조에 그대로 적용되는지는 논문에서 다루지 않음.

> 적용 가치: 중간 — 추가 하드웨어/연산 비용 없이 순서만 바꿔 평균 iteration을 줄이는 아이디어는 매력적이나, Prime ECC의 고정 QC-LDPC base matrix(column-weight 상대적으로 균등, HCU 예외 2열)와 기존 Dual-Update 스케줄에 이 정렬 규칙이 실질적 이득을 주는지 재검증 필요.
