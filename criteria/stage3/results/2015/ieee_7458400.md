### B. 알고리즘 요약

1. 대상은 Binary Erasure Channel(BEC) 상의 protograph LDPC 코드이며, base matrix `H(j,i)`의 각 원소는 variable node `Vi`와 check node `Cj` 사이 edge 개수를 나타냄.
2. 문제의식: protograph의 정확한 density evolution threshold `E*`는 계산이 어려워, 다변수 evolution 함수를 단일변수 함수로 상한 지어 손쉽게 평가/설계에 쓰려는 것.
3. 채널모델은 BEC(ε)이며, 메시지는 지워짐(erasure) 확률로만 표현되는 표준 message-passing 복호를 가정.
4. 핵심 기법은 "all-edge bound"와 "single-edge bound" 두 가지 상한. all-edge bound는 `E* ≤ min_{x≠0} Π f_ij(x)`로 모든 edge의 update function을 하나로 묶어 단순화.
5. single-edge bound(Lemma 2)는 "가장 빨리 수렴하는 edge" `(a,b)`(최대 check-degree, dominant column 조건)만으로 상한을 구해 all-edge bound보다 더 타이트함: `E* ≤ E*_ab = min_{x≠0} f_ab(x)`.
6. 이 상한을 다시 부호화도(degree) 항으로 닫힌 형태로 근사(`W1`, `W_N` 최소화식)하여, 협의 검색 없이 protograph degree 조건만으로 threshold 상한을 해석적으로 평가 가능.
7. 설계 휴리스틱: dominant column 조건, dominant column의 degree 합 최소화(sparsity), degree-2 variable node cycle 회피, 연결성, 비반복 column 등 조건을 만족하도록 랜덤 구성.
8. 최적화 절차는 H' 부분을 무작위로 생성하고 dominant column을 threshold 최대화 방향으로 검색하는 반복적 랜덤 구성(유전 알고리즘 대비 훨씬 단순).
9. 결과: 2x4~10x20 크기 protograph에서 BEC threshold 0.43~0.4852로, capacity 상한(0.5)에 근접. 기존 유전 알고리즘 기반 방법보다 단순한 절차로 유사 성능 달성.
10. 한계: BEC 전용 분석(다른 채널로 일반화 안 됨), 유한장 실제 코드의 error-floor·waterfall 시뮬레이션 없음, HW/복호기 설계 전무.
