### B. 알고리즘 요약

1. 대상은 N=1008, K=504, dv=3, dc=6 정규 LDPC 코드(MacKay 데이터셋), 메시지 5bit 양자화(1bit fractional), min-sum 디코더.
2. LDPC 디코더 전력의 50% 이상이 메모리 접근(VNU/CNU/channel memory)에서 발생 - voltage scaling으로 전력 절감하되 오류 유발이 문제.
3. 메모리 오류 모델: 공급전압을 1V→0.65V로 낮추면 공간적으로 균일한 랜덤 비트플립 발생(`Perr`=0.0027), channel/VNU/CNU 메모리에 additive random variable로 모델링.
4. 오류 허용성 통계: `Perr > ΔBER`이면 오류 허용(무손실), `Perr < ΔBER`이면 오류가 반복복호를 통해 전파·증폭.
5. 핵심 관찰: MSB(부호 결정 비트)에 오류가 나면 성능 저하가 크고, LSB 오류는 대체로 허용됨 - 따라서 하위 비트만 scaled voltage 적용, MSB는 nominal 전압 유지.
6. CNU 메모리(MIN 연산 피연산자) 오류가 VNU 메모리(SUM 연산) 오류보다 성능에 더 치명적 - 위치별 차등 전압 정책 필요(CNU는 last 2bit만, VNU/channel은 last 4bit).
7. 반복횟수 증가로 메모리 오류 영향 상쇄 가능(10→11→12 iteration) - 단 throughput 저하와 에너지 오버헤드 트레이드오프 존재, `E_total=(P_mem+P_logic)×N_iter` 식으로 총 에너지 모델링.
8. 구현: 기존 reconfigurable SRAM 아키텍처([8])를 활용, 비트별로 sub-array 분리해 LUT 기반 select 신호로 Vlow/Vnorm 전환.
9. 결과: Case 1(모든 메모리 last 4bit scaled)에서 메모리전력 58.4%, 전체 에너지 79.2%로 절감(0.11dB SNR 손실); Case 4는 SNR 손실 -0.03dB(개선)이나 반복 증가로 에너지 85.9%.
10. 한계: HW 합성/실칩 검증 없이 시뮬레이션(전력모델 기반 추정)뿐이며, 무선통신용 SRAM 저전력 기법으로 비휘발성 NAND ECC와는 적용 맥락이 다름; 이터레이션 증가로 오히려 latency 악화 트레이드오프.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNU/VNU 메모리 비트별 오류 민감도 분석 | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` (R/APP 메모리 접근부) | R메시지(C2V)·APP메시지(V2C) 저장 구조와 대응, 저전력 정책 아이디어만 참고 가능 |
| Voltage scaling 기반 메모리 전력 절감 | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 SRAM 전압 레벨 모델링 대상 아님 |
| LLR 양자화 비트 민감도(MSB vs LSB) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 테이블 비트폭 설계 시 MSB 보호 우선순위 참고 가능 |
| 반복횟수 증가로 오류 상쇄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 조기종료/반복수 조정 로직과 개념적으로만 연결 |

> 적용 가치: **낮음** — 무선통신 SRAM 저전력 voltage-scaling 기법으로 Prime ECC(bit-exact HW 시뮬레이터)의 판정 대상인 디코딩 알고리즘/부호설계와는 층위가 다르며(회로 전력설계 영역), 비휘발성 NAND 메모리 컨트롤러 맥락과도 직접 연결이 약함.
