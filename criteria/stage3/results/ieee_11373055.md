### B. 알고리즘 요약

1. 시스템: 3-D TLC NAND flash, QC-LDPC rate 8/9, length 18432, min-sum decoder, 최대 40 iteration.
2. 문제: RBER 상승 시 LDPC decoding iteration이 늘어 read latency가 심화되며, 기존 기법들은 외부 보조정보나 read voltage 최적화에 의존해 추가 저장/연산 오버헤드가 발생.
3. 가정/관찰: 프레임 내 information bit와 check bit는 오류 확률은 동일하지만 decoding latency에 미치는 영향은 다르고(check bit 오류가 더 큰 영향), LSB/CSB/MSB 페이지 간에도 고유한 신뢰도 격차(CSB가 가장 신뢰도 높음)가 존재.
4. 핵심 기법 1단 - DCDR(Disparity Characteristic-based Data Reconstruction): 신뢰도 높은 CSB 페이지에 check bit(CB)를 집중 배치하도록 워드라인 내 LSB/CSB/MSB 프레임의 IB/CB를 재배열(추가 저장 오버헤드 없음).
5. 핵심 식/의미: check bit LLR = `2.5 × information bit LLR`. 이 비율이 실측·시뮬 모두에서 iteration 수를 최소화하는 지점이며, 너무 낮으면 신뢰도 높은 check bit가 오판되고 너무 높으면 오류 check bit를 정상으로 오인.
6. 핵심 기법 2단 - HFMLLR: DCDR로 만든 재배치 데이터 위에 (a) check bit LLR 스케일링, (b) 동일 페이지 내 다중 프레임 오류의 공간적 상관성(대부분 "전부 성공" 또는 "다중 프레임 실패"로 82.90%+ 몰림)을 이용해 한 프레임 실패 시 해당 페이지 나머지 프레임을 skip하고 즉시 read retry를 트리거.
7. 부수 디테일: IB/CB가 모두 CSB 페이지에 있는 경우 LLR 조정 없음. 재구성은 순서 무관, 연산 없이 병렬 수행 가능해 추가 latency penalty 없음.
8. 최적화 절차: LLR 배율(2.5배)은 실측 800 P/E/60일 유지 조건과 시뮬레이션 5208 프레임 대상 파라미터 스윕으로 결정(자동 최적화 알고리즘 아님, 그리드서치 성격).
9. 결과: 1200 P/E cycle·180일 유지 조건에서 CSB 최대 81.98%, MSB 최대 83.40% iteration latency 감소; 모든 조건에서 decoding 후 BER=0 달성(traditional scheme 대비 개선); ABEL/ILD/EMAL 대비 우수.
10. 한계: 워드라인 단위 재구성이라 단일 페이지 read에는 부적합, 실측은 특정 칩(micron B16A, 64-layer)에 한정, HW(RTL/게이트) 설계 없이 알고리즘·컨트롤러 아키텍처 레벨 제안에 그침.

| min-sum decoder 자체 | `decoder.cpp` `LDPC_Decoder()` | 논문은 min-sum을 그대로 사용, 알고리즘 변경 없음 |

적용 가치: `중간` — 디코더 알고리즘 변경은 없지만 LLR 초기화 테이블에 IB/CB 차등 스케일 팩터를 추가하는 것은 낮은 비용으로 시도해볼 수 있는 아이디어이며, 데이터 재배치/조기retry 정책은 Prime ECC 시뮬레이터 범위 밖(컨트롤러 레벨)이라 이식 난이도가 있음.
