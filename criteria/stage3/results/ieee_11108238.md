### B. 알고리즘 요약

1. 시스템은 Slepian-Wolf 비대칭 소스압축(LSCSI): 소스 `x^n`(Bern(q))을 syndrome `s^{n-k}=Hx^n`으로 압축, 수신기는 side-information `y^n = x^n ⊕ z^n`(noise `z^n`~Bern(p))을 관측.
2. 기존 GRAND는 균일 소스(q=0.5)만 가정한 ML 근사 디코더이며, 비균일 소스(q≠0.5)에서는 MAP 디코딩이 필요하나 BCH/짧은블록LDPC/랜덤코드용 범용 MAP 근사 디코더가 없다는 문제를 다룸.
3. 소스 x^n과 side-info noise z^n은 각각 독립 Bernoulli(q), Bernoulli(p) 이진 대칭채널로 모델링.
4. 핵심 기법: MAP 기준을 `argmin_{x^n:Hx^n=s^{n-k}} α·wh(x^n) + (1-α)·wh(x^n⊕y^n)` 형태로 변환(Theorem 1), 여기서 `α = [log(q/(1-q))/(log(q/(1-q))+log(p/(1-p)))]`.
5. noise 추정 변수 `ẑ^n=x̂^n⊕y^n`으로 재정의하여 손실함수 `l(ẑ^n)=α·wh(y^n⊕ẑ^n)+(1-α)·wh(ẑ^n)`를 정의, 이 손실이 작은 순서로 coset 내 후보를 탐색하는 것이 핵심.
6. 비제약 최소점은 α≤0.5면 `ẑ⋆=0^n`(GRAND와 동일), α>0.5면 `ẑ⋆=y^n`(SO 디코더와 동일)로 이분화(Theorem 2).
7. 구현 디테일: 정렬 지표 `s(ẑ^n)=wh(ẑ^n)-2α·wh(y^n⊙ẑ^n)`을 이용해 `(m,m1)` 쌍별 등가집합 `Z^m_{m1}`을 사전 계산한 S-m-m1 테이블 순서로 순차 탐색, 최대 쿼리수 `g_max`로 조기 포기(abandonment).
8. 별도 학습/최적화 절차 없음(closed-form α 계산 + 전수탐색 순서 결정).
9. 결과: (256,240) QC-LDPC(base-graph 명시, Z=16, girth 16) 등에서 비균일 소스(q=0.01) 시 AP-GRAND가 GRAND/SO/BP 대비 BLER 우수, 평균 쿼리수도 GRAND보다 적음. 균일 소스(q=0.49)에서는 AP-GRAND/GRAND/BP 성능이 근사.
10. 한계: 쿼리 수가 많아 저잡음·고rate 영역에서만 경쟁력 있고, HW 미설계·시뮬레이션뿐이며 이론적 복잡도 분석은 future work로 남김. LDPC는 BP와의 비교 대상 코드일 뿐 LDPC 디코더 자체의 개선이 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| AP-GRAND (syndrome coset 전수탐색형 MAP 디코더) | 대응 없음 | min-sum 반복복호와 무관한 별도 계열(guessing decoding) 알고리즘, NAND 채널 모델 아님 |
| LDPC(256,240) BP 비교 대상 | `decoder.cpp` `LDPC_Decoder()` (참고용 대응) | BP 성능 비교 벤치마크로만 LDPC 등장, AP-GRAND 자체가 이식 대상 아님 |
| Slepian-Wolf/PUF 정보 조정 시나리오 | 대응 없음 | NAND read channel과 무관한 별도 응용(비대칭 소스압축, SRAM PUF 키생성) |

> 적용 가치: 낮음 — Slepian-Wolf 소스압축을 위한 guessing 기반 hard-decision 디코더로, NAND binary QC-LDPC의 soft-decision min-sum 복호 개선과는 알고리즘 계열·응용 목적이 모두 다름.
