### B. 알고리즘 요약

1. 시스템: 3D TLC NAND(FG), user data 2KB를 rate 8/9 LDPC로 인코딩(ECC 0.25KB), layered min-sum decoding, HSD(hard sensing) 중심, Gaussian 채널 근사.
2. 문제: retention/P/E 증가로 RBER이 층·페이지(EP/UP, 상하위 층)마다 크게 편차, 고RBER 페이지는 iteration 최대(30)에 도달해 정정 실패·SFSD 필요 → read 비용 급증.
3. 핵심 관찰: FPGA 실측으로 inter-layer RBER 편차 확인, RBER > 6.5e-3에서 Norm LDPC HSD가 실패(threshold).
4. 핵심 기법(Sege 인코딩): RBER>threshold 페이지는 원정보 N을 N/2씩 2분할, 각 절반의 빈 N/2를 0으로 채워 길이 N codeword 2개로 별도 인코딩 → 오류가 두 codeword로 분산되어 codeword당 정정부담 감소.
5. 핵심 기법(디코딩): 채운 N/2 기지 0비트를 known information으로 활용, 같은 check equation에 참여시켜 unknown 비트 디코딩을 가속.
6. LLR 최적화: unknown은 bit0→+1, bit1→−1(부호만), 기지 0비트는 LLR 진폭을 5 또는 10으로 상향(`|P0|>|L1|`), 판정규칙(Tj<0이면 1) 정합.
7. 적응 스위칭: RBER이 threshold 미만이면 Norm LDPC, 초과면 Sege LDPC 자동 선택(page type/layer/PE/retention 룩업테이블 기반).
8. 오버헤드: buffer overhead 무시가능(~1.37MB/chip), storage overhead 0.03~0.34%/block, 4KB random read만 부분 페널티(+45us).
9. 결과: HSD 허용 RBER을 6.5e-3→~1.4e-2로 확장, RBER 1.3e-2에서 iteration 30→합 10(약 66.7% 감소), UBER/BER·decoding time·layer 대폭 감소, 5K P/E까지 수명 연장(RT 3-month).
10. 한계: 컴퓨터 시뮬(디코더는 min-sum simulation), HW 미설계, 정정이득은 부호율 유지 위해 user space 희생 대가, 부호구조는 표준 rate 8/9 LDPC 재사용.
