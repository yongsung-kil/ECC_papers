### B. 알고리즘 요약

1. 시스템: Wyner-Ziv 기반 분산비디오코딩(DVC) 디코더, WZ프레임을 LDPCA로 복호, QCIF@30fps 목표, block length `396`(HW), `1584`(SW).
2. 문제: DVC 디코더의 SI 생성과 LDPCA 복호 연산량이 커 SW로는 0.156fps에 불과 → 실시간 불가.
3. 모델: LDPCA는 LDPC syndrome을 누적(accumulate)하여 low-rate에서도 모든 VN이 CN에 연결되도록 함, rate-adaptive 소스부호.
4. 핵심기법1: syndrome bit `C_k = ⊕ H_ki v_i`, 누적 syndrome `A_k`, comparing-tree로 basic CN(`66`개)에서 stacking CN을 code rate별로 삽입.
5. 핵심식: rate 66/66→2/66로 갈수록 병합될 두 CN 포인터를 새 stacking CN으로 재지정 → rate 변화 시 tree 확장.
6. 핵심기법2: de-accumulator(조합회로)가 누적 syndrome에서 병합 syndrome bit 생성.
7. 구현디테일: soft input computation을 lookup table로 단순화, block length 396 채택(HW 비용 comparing-tree가 block length에 비례).
8. 학습/최적화: 없음 (BFS로 tree 순회하여 wiring 결정).
9. 결과: TSMC 90nm, gate count `690K`, chip area `4.67mm²`, 100MHz, 302mW, RD 성능이 DISCOVER 동등 이상.
10. 한계: NAND 채널부호가 아닌 분산소스부호, degree-3 regular LDPCA over BSC, min-sum 성능/error-floor 논의 없음.
