### B. 알고리즘 요약 (10줄 내외)

1. 50G-PON 표준(IEEE 802.3ca/ITU-T G.9804) 기반 QC-LDPC(17280, 14592), code rate 84.44%, protograph 구조에서 384 bit parity puncturing 적용.
2. IMDD-NRZ 50 Gbit/s 신호가 25km SSMF를 통과하며 chromatic dispersion·laser chirp로 인한 ISI가 발생, 기존 연구는 디코더 파라미터 최적화나 iteration 수 대비 수신광파워(ROP) 분석을 다루지 않음.
3. 채널 모델은 실측 광채널(TOSA-SSMF-APD/TIA)이며 등화(equalization) 없이 슬라이싱 후 soft-input(SI, double-precision) 또는 hard-input(HI, sign-only)으로 디코더에 입력.
4. 디코더는 layered min-sum, weight factor `α`(0.55~0.85)를 적용하는 iterative 디코더, 최대 14 iteration, intra-layer 병렬/inter-layer 직렬 처리.
5. `α`는 min-sum 근사의 보정 계수로, 낮은 ROP 구간에서 평균 iteration 수를 줄이는 데 기여(특히 25km SI에서 효과적).
6. SI/HI 두 방식을 비교하여 HI가 SI 대비 약 1dB 성능 손실만 보이면서도 오히려 더 가파른 BER waterfall을 보임(디코딩에 유리한 Gaussian 유사 프로세스 추정).
7. 특별한 양자화/HW 근사 기법 언급 없음(SI는 double-precision floating point 그대로 사용).
8. 학습/최적화 절차 없음(파라미터 스윕에 의한 실험적 특성화만 수행).
9. 25km SSMF에서 SI/HI 모두 평균 6회 iteration이면 에러프리 동작 가능(0km는 4회); HI가 SI 대비 낮은 ROP에서 더 많은 평균 iteration 필요.
10. 한계: 실시간 하드웨어 구현이 아닌 오프라인 처리(RTO 캡처 후 후처리), 새로운 부호/디코더 알고리즘 제안 없음, 270개 codeword만 측정해 표준 요구 BER(10⁻¹²)까지 직접 검증 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum layered 디코더 + weight factor α | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 min-sum/스케일링 계열 기법과 개념적으로 겹치나, 본 논문은 알고리즘 개선이 아닌 실험적 파라미터 특성화라 신규성 없음 |
| soft-input(double precision) vs hard-input 비교 | `channel.cpp` `Set_LLR_Th()` | HD/soft 양자화 레벨 설계와 관련 있으나 본 논문은 4bit+ soft 대비 1bit HD 성능차 실측일 뿐, NAND read 비용 관점의 양자화 설계 기여 없음 |
| iteration 수 대 ROP(채널 SNR) 특성 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 조기종료/평균 iteration 감소 목적과 유사한 관측이나 신규 조기종료 기법 제안 없음 |
| QC-LDPC(17280,14592) 부호구조 | `ecc_top.cpp` `Load_PCM()` | 광통신 표준 고정 부호이며 NAND용 고rate 구성과 무관, 대응 없음에 가까움 |

> 적용 가치: **낮음** — 새로운 알고리즘/HW 기여 없이 기존 표준 부호와 min-sum 디코더의 광채널 실측 특성화에 그치며, 도메인도 NAND가 아닌 PON 광통신이라 이식 가치가 낮음.
