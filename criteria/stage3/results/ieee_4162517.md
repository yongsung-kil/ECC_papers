### B. 알고리즘 요약

1. 대상: 고rate QC-LDPC 코드(예제: (4608,4096), (4,36), rate 8/9 regular, read channel용), modified min-sum algorithm(MSA) 기반 partially parallel 디코더.
2. 문제: LDPC 코드 성능은 큰 코드워드 길이(N≫1000bit)에서 발휘되므로 메모리가 전체 하드웨어의 대부분을 차지 - 메모리 절감이 핵심 과제.
3. 가정/관찰: check-to-variable 메시지 `R_cv`는 modified MSA 식(1)에 의해 한 행(row) 내에서 크기(magnitude)가 최소/차소 두 값만 가짐(data redundancy) → 압축 저장 가능.
4. 핵심 기법: 디코딩 절차를 재배열(rearranged MSA)해 `R_cv^(k)` 갱신에 이전 `S_v^(k-1) - R_c^(k-1)` (column sum 기반)을 직접 활용, 개별 `L_cv` 메시지 대신 column sum `S_v` 만 저장·갱신 (식 3, 4).
5. 의미: `R_cv` 메시지를 한 행당 4개 요소(최소값 m1, 차소값 m2, 차소 위치 index, 부호 signs)로 압축 저장함으로써 R-memory 용량을 크게 줄임; 필요시 data distributor로 개별 메시지 복원.
6. 확장: weight-1 circulant 전용 구조(Fig.1)에서 weight-2 circulant(중첩 순환치환행렬) 포함 H-matrix로 확장 - 행 분해(decompose) 후 CNU_a/CNU_b 두 유닛과 merge unit으로 처리(Fig.4).
7. 구현 디테일: CNU 핵심 연산(최소/차소 크기 탐색)을 위한 최적화된 2x2 PROF(pseudo-rank order filter) 설계, index bit를 stage별로 aggregation; p-parallel 처리 시 인접 p개 row/column 데이터를 barrel shifter 기반 data scheduling/reverse scheduling unit으로 배치 충돌 해소; single-port memory + 이중 버퍼로 dual-port 메모리 대체(Fig.3)해 면적 절감.
8. 최적화 절차: 없음(알고리즘 학습형 아님) - 아키텍처/데이터패스 최적화 위주.
9. 결과: 6bit 양자화 가정 시 제안 구조 총 메모리 163kbit vs 전통적 접근 258kbit로 약 37% 절감(고rate 8/9 예제); 일반적으로 rate>2/3 고rate 코드에서 30% 이상 메모리 절감 가능. 디코딩 사이클 수 `⌈t/p⌉+2n·J(⌈t/p⌉+1)` 명시.
10. 한계: BER/FER 곡선, 실제 게이트카운트, throughput(Gbps) 수치는 본문에 없음(이론적 사이클/메모리 분석뿐); 두 디코딩 phase가 겹치지 않아 throughput은 latency 대비 동등 수준(overlap 시 2배 가능하다고 언급하나 상세 미제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified min-sum CNU 연산 (최소/차소값 탐색) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2 기반 min-sum을 사용하므로 PROF 기반 최소/차소 탐색 최적화는 CNU 로직과 직접 대응 |
| R_cv 메시지 압축 저장(m1,m2,index,signs) 및 data distributor | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 메시지 저장 방식/메모리 구조 최적화로 디코더 내부 메모리 관리 로직에 이식 가능성 있음 |
| p-parallel data scheduling / reverse scheduling unit (barrel shifter 기반) | 디코더 HW 모델 (z=32, HCU, GT) | z_sb=32 병렬 처리 시 row/column 데이터 배치 충돌 해소 기법으로 병렬 아키텍처와 관련 |
| single-port memory + 이중 버퍼로 dual-port 대체 | 대응 없음 | 메모리 뱅크 구현 세부(면적 절감) 기법이나 코드 프로파일에 대응 모듈 명시 없음 |
| weight-2 circulant 분해 처리(CNU_a/CNU_b) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조(compound circulant) 관련이나 Prime ECC의 고정 QC 구조와 직접 대응 확인 어려움 |

> 적용 가치: 높음 — QC-LDPC 고rate 디코더의 메모리 병목(§5 "HW 면적/throughput 개선" 관심영역)을 직접 겨냥한 실용적 아키텍처 기법으로, min-sum 압축 저장/PROF 최적화는 decoder.cpp 로직에 낮은 난이도로 이식 가능.
