### B. 알고리즘 요약

1. 대상: short/medium 블록의 binary linear code (LDPC/BCH). 예: (204,102) 정규 LDPC, (63,39) BCH, BIAWGN 채널.
2. 문제: ML 디코딩은 NP-hard지만 짧은 부호에서는 정수계획(IP) + 범용 solver로 정확히 풀 수 있어, 여러 IP 정식화의 계산 효율을 비교할 필요.
3. 모델: ML 디코딩 = `min λ^T x, x∈C`, λ는 LLR 비용벡터. C를 conv(C)로 치환 시 LP, integrality 강제 시 IP.
4. 정식화들: IPD1/IPD2(Breitbach, `Hx=2z` 짝수 패리티), IPD3(Feldman local code `w_{i,S}`), IPD4(Forbidden Set 부등식), IPD5(parity polytope, `O(n^3)`), IPD6(Yang, degree≤3 check node 분해).
5. 핵심 관측: IPD3는 변수 수가, IPD4는 제약 수가 check node degree에 지수적 → 고밀도 부호에서 비효율. IPD5는 다항이나 `O(n^3)`.
6. CPLEX의 branch-and-cut(Gomory/MIR cut, simplex LP relaxation)으로 각 정식화를 풀며 변수·제약·simplex iteration·cut·node·CPU time 측정.
7. 결과: IPD1/IPD2는 변수·제약 적어 LP relaxation이 느슨 → cut 다수. IPD3~IPD6는 tight relaxation이나 큰 tableau. 고밀도 부호에서 IPD1/IPD2가 iteration 100배 이상 적음.
8. 결론: ML 디코딩 IP는 "적은 변수·제약으로 시작 후 cut을 점진 생성"이 유리하다고 추정.
9. 수치: (204,102) LDPC SNR 2.1에서 IPD1 59.2s vs IPD5 306s(100 instance). 고밀도 BCH는 IPD3/IPD4 실행 불가(*).
10. 한계: 범용 solver 기반 오프라인 분석 도구 — HW 미설계, 실시간 복호 아님, 짧은 블록 한정.
