### B. 알고리즘 요약

1. IEEE 802.16e 표준의 Block-type LDPC(B-LDPC, base matrix + z×z permutation 확장 구조)를 대상으로, 하나의 모체 parity-check matrix로 여러 rate(1/2, 2/3, 3/4, 5/6)를 지원하는 rate-matching 문제를 다룸.
2. 기존에는 rate별로 별도 H-matrix를 저장해야 해 메모리·복호기 복잡도가 커지는 문제가 있음.
3. 세 가지 펑처링 방식을 비교: (a) RS - 비트 단위 recovery-step(GS 알고리즘) 기반, (b) BRS - RS를 base matrix(block) 단위로 단순화 적용, (c) BPS - systematic 코드의 parity 블록만 정해진 순서로 펑처링.
4. 핵심 변수: k-step-recoverable(kSR) 노드/블록 개념 - 펑처링된 변수노드(블록)가 survived check node를 통해 k번째 이터레이션에 non-zero 정보를 받아 "복구"되는 순서.
5. BRS의 핵심 관계식 `γ_μ = μ + q_i,j (mod z)`로 base matrix의 1개 원소(shift value q_i,j)가 확장 후 z개 변수노드-체크노드 쌍에 동일하게 대응됨을 이용, block 전체를 한 단위로 그룹핑/정렬.
6. BPS는 그룹핑/정렬 계산 없이 parity 블록만 고정 순서 `{k_b+2, k_b+4, ..., k_b+3, k_b+5, ...}`로 펑처링해 계산복잡도를 낮추고 최고 rate 한계를 더 높임.
7. 부수 트릭: BRS는 RS와 동일한 GS 원리를 base matrix 레벨(nb×mb, 확장 전)에 적용해 grouping/sorting 연산량을 z배 줄임(메모리·연산 절감 언급).
8. 별도 학습/최적화 절차 없음 - 세 방식 모두 결정론적 규칙 기반 펑처링.
9. 결과: rate 0.5 모체부호(576,288)/(960,480)에서 rate 0.6~0.8까지 펑처링, top rate 한계는 RS 0.816~0.819, BRS 0.857, BPS 0.923~0.947로 BPS가 가장 높음; BLER 0.01·rate 0.8에서 Block Puncturing이 RS/BRS 대비 0.3~0.4dB 손실.
10. 한계: AWGN·BPSK·BP 디코딩 시뮬레이션에 국한, HW/복잡도 수치 없음, 정정능력 자체의 개선이 아니라 다중rate 지원을 위한 펑처링 방법론 비교에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| base matrix + z×z circulant 확장 QC-LDPC 구조 | `ecc_data.h` `PCM_b`, `ecc_top.cpp` `Load_PCM()` | 부호 표현 구조(base+shift)가 Prime ECC의 `PCM_b`와 유사한 QC-LDPC 개념이나 논문은 가변rate 펑처링이 목적 |
| 펑처링 기반 rate-matching (RS/BRS/BPS) | 대응 없음 | Prime ECC는 고정 H-matrix 단일 rate 설계로 펑처링·다중rate 지원 개념 없음 |
| systematic parity 블록 선택적 펑처링 | 대응 없음 | Prime ECC 인코더(`encoder.cpp` dual-diagonal)는 고정 구조로 펑처링 대상 아님 |

> 적용 가치: 낮음 - QC-LDPC base matrix 개념은 공유하나 목적이 무선표준의 가변rate 지원(펑처링)이라 고정 고rate NAND ECC에는 직접 이식할 요소가 없음.
