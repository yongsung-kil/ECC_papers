### B. 알고리즘 요약

1. 시스템: 이진 AWGN 채널, irregular LDPC 부호, M×N parity check matrix H(Tanner graph 표현). column weight(symbol node degree) 고정, row weight(check node degree)는 결과로 결정됨.
2. 문제: 원래 PEG는 "M,N,symbol 차수열이 주어졌을 때 girth를 최대화"하는 문제를 풀지만, 본 논문은 "M, girth 하한 `g`, symbol 차수열이 주어졌을 때 code rate(N)를 최대화"하는 역방향 문제를 다룸.
3. 핵심 가정: girth가 클수록 SPA 반복복호의 독립 반복 횟수가 늘어 오류율이 낮아지나, SNR이 threshold보다 충분히 높은 채널에서는 error-floor보다 code rate가 더 중요하다는 실용적 관점.
4. 핵심 기법: Algorithm 1(Extended PEG) — symbol node를 하나씩 추가하며 현재 girth `gi`가 목표 하한 `g` 이상인 동안 계속 N을 증가시킴; `gi < g`가 되면 노드 추가 중단.
5. 핵심 식/절차: symbol node 연결 시 depth-l tree 확장으로 `N^l_si`(neighborhood)를 구하고, 이전과 달리 최저차수 check node 대신 `N^l_si`에서 **무작위** 선택(원 PEG는 최저차수 선택) — 더 많은 girth 여유 확보.
6. 확장 트릭: girth가 감소하면(`gi<gi-1`) 직전 두 노드의 edge를 삭제하고 재시도(최대 `iter`회 반복)해 girth 감소를 늦춤.
7. 부수 결과: rate-compatible LDPC(RC-LDPC) 설계 시 필요한 "고rate base code"를 생성하는 용도로도 언급.
8. 학습/최적화: Density Evolution(LODE 툴)로 무한 length 극한에서의 SNR threshold를 계산해 두 부호의 정정능력을 비교.
9. 결과: 동일 M, girth=6 조건에서 MacKay 부호(N=4376) 대비 제안 부호(N=5867, rate 상승)는 BER 성능은 유사하나 DE threshold가 4.25dB→4.62dB로 소폭 악화(정정능력 손실); girth를 6→8→10으로 늘리면 rate는 낮아지지만 BER은 개선.
10. 한계: 순수 그래프 구성 알고리즘(랜덤 성장), HW 설계·복잡도 분석 없음, 정정능력 향상이 아닌 "rate 증가 대가로 정정능력 일부 희생"하는 트레이드오프 논문.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Extended PEG 기반 H-matrix(고rate) 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base+circulant) 고정 구조 사용, 본 논문은 비-QC 랜덤 irregular 그래프 성장 방식이라 구조적 이식 난이도 높음 |
| rate-compatible base code 설계 언급 | `ecc_top.cpp` `Load_PCM()` | RC-LDPC 관련 개념적 언급뿐이며 실제 구현/검증 없음, 대응 약함 |
| Density Evolution 기반 threshold 비교 | 대응 없음 | 이론적 무한장 근사 분석 도구로 Prime ECC의 유한장 bit-exact 시뮬 구조와 무관 |

> 적용 가치: 낮음 — 디코더/HW 요소가 전혀 없는 순수 랜덤 irregular PEG 그래프 구성법이며, Prime ECC의 고정 QC-LDPC 구조와 정합성이 낮고, 제안 기법 자체가 정정능력을 일부 희생하며 rate만 높이는 트레이드오프라 NAND ECC 이식 매력도가 낮음.
