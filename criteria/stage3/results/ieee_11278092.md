### B. 알고리즘 요약

1. 대상: `[[n,k,d]]` CSS 기반 QLDPC 코드(예: `[[882,24,18<d<24]]`, `[[1922,50,16]]`, bivariate bicycle code), depolarizing 노이즈(code-capacity model) 및 circuit-level noise 모두 다룸.
2. 문제: 기존 syndrome-based min-sum(SB-MS)은 Tanner graph의 짧은 cycle로 인해 트래핑셋에 빠져 error floor가 발생, OSD/SI 등 후처리는 복잡도(O(n³) 가우스소거 등)가 커 실시간 처리 불가.
3. 채널모델: code-capacity model(데이터 큐빗만 depolarizing 오류, `p_X=p_Y=p_Z=p/3`)과 circuit-level noise(게이트/측정/보조큐빗 오류까지 반영하는 detector error model).
4. 핵심 기법 1단(SB-LP): 고전 LP 디코딩(Vontobel-Koetter 알고리즘)을 QLDPC용으로 변형 — syndrome bit `s_i=1`일 때 검사식을 odd-parity 집합 `C_i`(보수 집합)로 확장, `u_{i,j} = (α_LP/2)(T^{(0)}_{i,j} - T^{(1)}_{i,j} - S_{i,j})`로 완전병렬 갱신.
5. 핵심 식 의미: extrinsic 대신 a posteriori LLR을 직접 갱신함으로써 짧은 cycle을 통한 정보 되먹임에 의한 진동(oscillation)을 회피 — SB-MS의 트래핑셋 문제를 구조적으로 완화.
6. 확장(early-stopping): 연속 반복 간 추정 syndrome의 Hamming distance `d_H(ŝ,ŝ_pre) ≤ d_v`(VN 차수)이면 SB-MS를 조기 종료하고 그 사후 LLR을 SB-LP 초기값으로 전달, SB-MS+SB-LP 결합.
7. 부수 트릭: circuit-level noise에서 CN 차수가 30 이상으로 커지면 `O(2^{|P_i|})` 최대화 연산이 폭증하므로, 양수합 기반 그리디 근사(Algorithm 5)로 `O(|P_i|)`로 축소.
8. 학습/최적화: 별도 학습 없음. `α_BP`, `α_LP`(수렴 가속 스케일링 팩터) 및 `δ0,δ1,δ2,δ3`(circuit-level noise 초기화 가중치) 그리드 탐색으로 튜닝.
9. 결과: B1 code(`p=0.04`)에서 SB-MS 대비 SB-LP 결합 시 error floor 2~3자리 개선; 3단 구조(SB-MS+SB-LP+OSD-0)는 bivariate bicycle code `[[144,12,12]]`에서 logical error rate `1.67×10^-7`(BP+OSD-0 대비 개선), OSD 호출 횟수 약 절반 감소, 실행시간 1.9~2.9배 단축.
10. 한계: 순수 소프트웨어 시뮬레이션(Stim 기반)뿐 HW 설계 없음; 저자도 "코드 최적화·병렬화 안 됨"이라 명시, 대형 distance 코드에서는 최신 디코더 대비 아직 미경쟁.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SB-MS 디코더(min-sum 기반 반복 메시지패싱) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | quantum syndrome(`s_i` 0/1 모두 가능) 처리라 CN 갱신식이 classical parity(`s_i=0` 고정)와 다름 — 구조는 유사하나 그대로 이식 불가 |
| Early-stopping(syndrome Hamming distance 기반 진동 검출) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 목적(조기종료)은 유사하나 판정 지표가 CRC가 아닌 연속 syndrome 간 거리라 별도 로직 필요 — 개념적 참고만 가능 |
| SB-LP(LP 완화 기반 후처리 디코더) | 대응 없음 | Prime ECC는 min-sum만 보유, LP-relaxation 디코더 자체가 없음 |

> 적용 가치: 낮음 — quantum stabilizer code 전용 디코더로 binary classical QC-LDPC와 부호모델이 근본적으로 다르며, 유일하게 참고 가능한 것은 트래핑셋 진동 감지에 의한 조기종료 아이디어 정도.
