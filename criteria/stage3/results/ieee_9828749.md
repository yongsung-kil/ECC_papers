### B. 알고리즘 요약

1. 대상은 QC-LDPC(base graph, `z×z` circulant block), block-parallel + layered scheduling 디코더, 5G-NR BG1(rate 1/3)/BG2 예시.
2. 문제: pipeline 도입 시 같은 column index `j`의 a-posteriori message `A_j`에 대한 read(Thread1 첫 stage)/write(Thread2 마지막 stage)가 서로 다른 row에서 충돌(read-write conflict) 발생.
3. 모델: two-thread waveform graph — Thread2가 각 row에서 Thread1보다 지연, row `i`의 block 스케줄 벡터 `TiX/TiY`, row weight `di=|Ni|`, 충돌 인덱스 `Ci,i+δ = Ni∩Ni+δ`.
4. 핵심기법: block 재스케줄(충돌 감소)과 NOP 삽입(충돌 완전 제거, throughput↓)을 결합해 "충돌 0 + NOP 최소"인 명령어 집합 생성.
5. 제약식: 명령어 non-overlap `Xi+di-1<Xi+1`(식1,2), thread간 순서 `Xi+PX+di-2<Yi`(식3), 레지스터 버퍼 보존 `Xi+1+PX>Yi`(식4) → Algorithm 1로 conflict 무시한 `Xi,Yi` 결정.
6. 확장: 2-row 충돌은 Theorem 1(Lemma 1·2)로 `TiY, TiX+δ`를 최소 NOP `e`로 배치, `e=max(0,p1-q1+1)`.
7. 다중 row: 충돌 최대 row 거리 상한 `Δ0 ≤ ⌈(PX+PY-2)/dmin⌉+1`(Theorem 2)로 고려 범위 한정, Algorithm 2·3로 `TiX/TiY` 설계(이미 해소된 인덱스는 line5에서 제외).
8. 통합: Algorithm 4가 위를 결합, `SY/VY` 버퍼 1개 + `SkX/VkX` 버퍼 Δ개를 포인터 `k`로 순환 사용해 최적 스케줄 생성.
9. 결과: BG1 rate1/3(PX=4,PY=2)에서 총 명령어 354, NOP 비율 0.1073; 단순 natural-order NOP삽입 대비 NOP 68.6%·명령어 19.0% 감소(PX=5,PY=3시 71.2%/25.6%), 정규화 throughput `(PX+PY)/L` 분석.
10. 한계: FPGA/ASIC 실측 없이 스케줄 최적성만 증명, 복호 성능(BER) 개선 아님, row schedule은 고정 가정.
