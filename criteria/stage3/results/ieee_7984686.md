> 총평: QC-IRA 부호구성에서 저중량 부호어 제거로 error-floor·iteration을 낮추는 아이디어는 유효하나, 항법용 단코드·IRA/zigzag 구조라 NAND용 dual-diagonal QC-LDPC인 Prime ECC엔 직접 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: 위성항법(GPS L1C/BeiDou) FEC용 IRA-like 부호, `(1200,600)` rate `1/2`, BPSK/AWGN, normalized sum-product `max_iter=100`.
2. 문제: 항법 메시지는 짧은 프레임·저비용 수신기 요구 → 짧은 코드에서 높은 coding gain과 낮은 codec 복잡도 동시 필요.
3. 구조: `H=[H1|H2]`, `H1`은 QC 구조(ICPM shift matrix `S`), `H2`는 double-diagonal(zigzag) accumulator, PN degree=2(마지막 제외).
4. 핵심 관찰: circulant permutation을 직접 이어붙이면(direct splicing) zigzag 연결에서 다수의 length-8 cycle과 weight-5 저중량 부호어 발생 → error floor·미검출 오류 증가.
5. 정량화: degree-3 VN마다 최소 5개 weight-5 부호어, `k/p` 그룹이면 최악 `C(p,2)×5` 규모 저중량 부호어.
6. 핵심 기법: CN을 `(n-k)/p` 그룹으로 나누고 각 PN을 인접 CN group의 특정 CN에 교차 연결하도록 행순서(연결) 재배치 → 저중량 부호어 제거.
7. 구성 절차: (S1) PEG로 `H1` base matrix `B` 생성(같은 열 중복 '1' 회피), (S2) '1'/'0'을 `p×p` circulant/zero로 lifting, (S3) 행순서 조정으로 저중량 부호어 제거, (S4) `H1`+`H2` 결합.
8. HW 배려: 행순서를 double-diagonal 형태로 정렬해 하드웨어 구현 용이.
9. 결과: direct splicing 대비 2.5~3.5dB에서 BER 2~4자릿수 개선, error platform 낮춤, 필요 iteration 대폭 감소(2.0dB에서 최대 약 60배 빠름).
10. 한계: HW 미설계·AWGN 시뮬만, 항법 단코드 타깃, IRA/zigzag 구조 특화(gate/throughput 수치 없음).
