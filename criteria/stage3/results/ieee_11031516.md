### B. 알고리즘 요약

1. 5G-NR/WiFi/위성/IoT 통신 채널을 대상으로 QC-LDPC 부호에 Min-Sum 계열 디코딩을 적용한 FPGA 구현.
2. 기존 CPU/GPU 기반 LDPC 디코더는 전력·지연 측면에서 실시간 처리에 부적합하다는 문제의식에서 출발.
3. 채널 모델은 AWGN 기반이며 SNR 0.5~5dB 구간에서 BER/FER 성능을 측정.
4. 핵심 기법은 QC-LDPC 구조 + Min-Sum Algorithm(MSA)이며, Offset Min-Sum(OMSA)으로 보정계수를 추가해 성능을 개선한다고 서술.
5. 핵심 식은 `L(q_i→j) = L(r_i) + Σ_{k∈N(i)\j} L(q_k→i)` (SPA의 LLR 업데이트, MSA는 이를 min 연산으로 근사).
6. 레이어드 디코딩 스케줄링과 조기종료(valid codeword 검출 시 중단)로 반복 횟수를 줄인다고 언급하나 정량 수치는 없음.
7. 구현상 fixed-point 연산, DSP slice/LUT 활용, BRAM에 parity-check matrix 저장 등 HW 친화적 설계 요소를 사용.
8. 별도의 학습/최적화(DE, GA 등) 절차는 없음 — 설계 파라미터는 수작업 선택으로 보임.
9. 결과: SNR=2.5dB에서 BER 3.2×10⁻³, FER 6.5×10⁻², throughput 3.85Gbps, latency 7.8µs, 전력 4.8W(GPU 대비 처리량 20%↑, 전력 대폭 절감).
10. 한계: 시뮬레이션 기반 FPGA 결과만 제시되고 구체적 H-matrix/부호 rate/length가 명시되지 않아 재현·비교가 어려우며, 알고리즘적 신규성이 낮음(표준 Min-Sum/OMSA 재사용).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum 기반 디코딩 알고리즘 전체 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC가 이미 Min-Sum 보유, 본 논문은 표준 MSA/OMSA 재사용이라 신규 기여 없음 |
| Offset Min-Sum 보정계수 | decoder.cpp `CNU_Update_New_Mag()` | scaling/offset factor 개념은 이미 대응 위치 존재, 구체 수치 미제시로 이식 불가 |
| 파이프라이닝/병렬 CN·VN 처리 | decoder.cpp pipeline 6-stage 구조 | 이미 pipeline 구조 보유, 본 논문은 구체 스테이지 설계 없이 일반론만 서술 |
| QC-LDPC 부호 구조 선택 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | rate/length 등 구체 스펙 미상으로 비교 불가 |
| BRAM 기반 parity-check matrix 저장 | 대응 없음 | Prime ECC는 시뮬레이터로 온칩 메모리 구조를 직접 모델링하지 않음 |

적용 가치: **낮음** — 구체적 알고리즘/부호 혁신이 없고 표준 기법(Min-Sum, OMSA, 파이프라이닝)을 개괄적으로 서술한 수준이라 Prime ECC에 이식할 새로운 요소가 없음.
