### B. 알고리즘 요약

1. 대상: PEG로 생성한 rate-`1/2`, length `500` 최적 degree 분포 LDPC, AWGN 채널, BP 복호.
2. 문제: sequential scheduling(IDS)은 trapping set 오류 정정에 강하나 short code에서 일부 non-trapping set 오류를 오히려 놓친다 (greedy 구조 때문).
3. 배경: residual = 메시지 업데이트 전후 차이. residual 최대인 메시지를 먼저 갱신 → 그래프의 가장 불안정한 부분만 갱신, 수렴 가속.
4. 기존 mixed scheduling [2]: 결정적 스케줄(LBP)로 non-trapping 오류 처리 후 동적 스케줄(NWRBP)로 trapping set 처리.
5. 핵심 기여: RBP → LBP → NWRBP 3단(3S-RLN) 적응형 전환. RBP를 초기 iteration에 먼저 배치.
6. 전환1 (RBP→LBP): `S_{l-1}/S_{l-2} ≥ S_l/S_{l-1}`가 `l≥4`에서 2회 성립 시(또는 최대 10 iter). `S_l`=불만족 check node 이웃 수.
7. 전환2 (LBP→NWRBP): `Σ_i |LLR(i)|/deg(i)`가 두 번째 국소최대 도달 시(또는 최대 30 iter). deg(i)=변수노드 차수.
8. 근거: RBP는 초기 iteration 최고 성능, NWRBP는 중후반 asymptotic 성능이 좋음 → 각 구간 강점을 적응적으로 활용.
9. 결과: 70 iteration에서 모든 SNR에 대해 flooding/RBP/NWRBP/2S-LBP+NWRBP보다 WER 우수, SNR 증가할수록 개선폭 확대(trapping set 정정 효과).
10. 한계: 시뮬(AWGN)만, HW 미설계, 추가 계산비용은 code length에 선형이라 하나 sequential 스케줄 자체가 parallel HW와 상충. rate/length 단일 예제.
