### B. 알고리즘 요약

1. 5G 표준 QC-LDPC 코드(rate 0.7, 0.9)의 layered decoding을 대상으로, M-ary(QAM-16/256) 변조로 생성된 초기 LLR `λ_i`의 특성을 이용해 복잡도를 줄임.
2. 기존 forced-convergence 계열 기법([4]-[9])은 반복마다 VN 신뢰도를 동적으로 판단해 추가 연산이 발생하는 문제가 있음.
3. M-ary 입력 채널은 `m`개의 비트레벨 채널로 분해되며(`p(y|b(l)=b)`, 식 (1)), 고차 변조일수록 비트레벨 채널 간 신뢰도(평균 LLR 크기) 차이가 커짐(가장 신뢰도 높은 레벨 집합 `L*`).
4. 비활성 VN 집합 `S = {i, (i mod m)∈L* : |λ_i|>ω}` — 최신뢰 레벨에 속하면서 초기 LLR 절댓값이 임계값 `ω`를 넘는 VN을 디코딩 시작 전에 한 번만 결정.
5. Algorithm 1(layered decoding)에서 `S`에 속한 VN은 매 반복 C2V LLR 갱신(Step 7)과 AP-LLR 갱신(Step 8)을 생략, MSA의 min1/min2 탐색 대상에서도 제외.
6. 임계값 `ω`는 원하는 오류확률 `P_d = 1-(1-P_e)^D` (식 (2))로부터 설계 — `D`는 평균 dVN 개수, `P_e`는 dVN에서 오판정 확률, 둘 다 초기 LLR PDF `p(λ(l)|b(l)=b)`(식 (3))로 계산.
7. 부수적으로 shortening VN(초기 LLR=∞)과 SPC VN(이웃 CN 1개뿐)도 함께 비활성화해 별도 이득 확보(BLER에 영향 없음).
8. threshold는 오프라인 lookup table로 구성 가능(코딩 파라미터·SNR별); 본 논문은 on-the-fly 설계로 시뮬.
9. 결과: `P_d=10^-3` 설정 시 R=0.7, 0.9 모두 약 22.5% VN 비활성화하며 BLER 손실 무시 가능; 평균 반복횟수(ANI)는 최대 1.7 감소(`P_d=10^-2`일 때, 단 이 경우 최대 1.6dB BLER 손실 발생).
10. 한계: HW 구현/게이트카운트 미제시, 시뮬레이션(5G QAM 채널)에만 국한되며 SPA(sum-product) 기준 검증, min-sum 기준 정량 결과는 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN LLR 갱신 생략(dVN 스킵) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 반복 루프 내 특정 VN의 C2V/AP-LLR 계산을 조건부로 skip하는 로직은 이터레이션 루프에 훅 가능 |
| 초기 LLR 기반 사전 판단(임계값 ω) | `channel.cpp` `Set_LLR_Th()` | soft-read LLR 양자화·threshold 설정 위치와 개념적으로 유사하나 M-ary 비트레벨 채널 전제가 NAND에는 없음 |
| min-sum 탐색에서 dVN 제외 | `decoder.cpp` `CNU_Update_New_Mag()` | min1/min2 탐색 시 특정 V2C LLR을 제외하는 최적화는 CN 연산부에 적용 가능 |
| M-ary bit-level channel 신뢰도 분석 | 대응 없음 | Prime ECC는 binary QC-LDPC + NAND read 채널로, 변조 기반 비트레벨 채널 개념 자체가 없음 |

> 적용 가치: 중간 | VN 갱신 스킵으로 반복당 연산량과 iteration 수를 줄이는 아이디어는 디코더 최적화 관점에서 유효하나, 임계값/신뢰도 설계가 M-ary 변조 채널에 특화되어 있어 NAND soft-read 채널 모델로 재유도하는 추가 연구가 필요함.
