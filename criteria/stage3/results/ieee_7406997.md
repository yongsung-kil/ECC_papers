### B. 알고리즘 요약

1. 시스템: Polar code `(N,K,A,u_Ac)`, code length N=64/256, rate=1/2 (예: (64,32), (256,128)), AWGN 채널, stochastic bit stream(길이 `l=1024`)으로 확률값 표현.
2. 문제: list SC 복호는 list size `L` 증가에 따라 HW 복잡도가 선형 증가(`O(LN log N)`)하여 큰 L(≥8) 적용 시 구현 부담이 큼; stochastic computing으로 복잡도를 줄이려 하나 직접 적용 시 성능 저하 발생.
3. 핵심 문제 원인: L>1일 때 decision metric이 조건부확률 `p_k(u_i|û_1^{i-1})`이 아니라 누적곱 확률 `P(û_1^i)=Π p(u_n|û_1^{n-1})`이어야 하는데, 이 값이 단계가 진행될수록 지수적으로 작아져 고정 길이 stochastic bit stream이 무작위성을 잃고 성능이 저하됨.
4. 핵심 기법 1: f/g node를 stochastic 버전으로 재정의(`Pr_f = Pr_a(1-Pr_b)+Pr_b(1-Pr_a)`, `Pr_g`는 `usum` 조건부 결합), channel message scaling factor `α=0.5`로 채널 LLR을 스케일링해 초기 stochastic stream 품질을 개선.
5. 핵심 식: ordered probability 재귀식 `P(û_1^i)=P(û_1^{i-1})·p(u_i=û_i|û_1^{i-1})`(식 10) — L개 후보의 누적확률을 매 단계 곱셈(AND gate 1개)으로 갱신해 최유력 L개를 유지.
6. 핵심 기법 2(Doubling Probability): 두 후보 확률이 모두 0.5 미만이면 2배로 rescale하여 확률값이 (0.5,1] 구간에 머물게 함으로써 stochastic stream의 무작위성 손실을 방지.
7. HW 친화 근사: 정확한 doubling은 HW 구현이 어려워, 확률 구간별 근사식 `2P ≈ P + 1/(n+1)`(표 기반, 이웃 0의 개수마다 하나를 1로 flip)로 NOT 게이트 1개만으로 구현.
8. 최적화 절차: α 값(0.4~1) 스윕 시뮬레이션으로 최적값 0.5 결정, 별도 학습 알고리즘 없음.
9. 결과: L=1 stochastic 디코더는 deterministic 대비 게이트 수 O(7N) vs O(10qN)(quantization q=10 기준 약 90% 감소); L=2 stochastic+doubling은 deterministic SC보다 우수한 BER; latency는 stochastic L=2가 `8l+27` clock으로 stochastic L=1(`8l+6`)보다 김.
10. 한계: Polar code 전용(LDPC 아님), l=1024인 긴 bit stream으로 인해 latency가 매우 큼(수천 clock), FPGA/ASIC 합성 없이 게이트/레지스터 수 추정치만 제시, deterministic list SC(L≥8) 대비 성능비교 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic decoding 전체 패러다임 | 대응 없음 | Prime ECC는 magnitude-quantized min-sum 기반 결정론적 디코더로, bit-stream 확률표현 방식과 근본적으로 다른 아키텍처 |
| f/g node, mixed node (SC 복호 트리) | 대응 없음 | Polar code 고유의 butterfly 구조로 QC-LDPC의 min-sum CN/VN 구조와 대응 없음 |
| channel message scaling(α) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 채널 LLR 스케일링이라는 개념적 유사성은 있으나 stochastic stream 생성용으로 직접 이식 불가 |

적용 가치: 낮음 — Polar code decoder이며 stochastic computing이라는 이질적 패러다임이라 binary QC-LDPC min-sum 디코더에 이식할 요소가 거의 없음.
