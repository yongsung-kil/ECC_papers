### B. 알고리즘 요약

1. 시스템: NAND flash용 4KB급 QC-LDPC 계열, GF(197) 기반 구성, (37632, 33712~33908) 코드, rate 0.895~0.901.
2. 문제: 기존 GC-LDPC/SC-GC-LDPC는 계층적 구조 최적화를 충분히 활용하지 못해 성능·복잡도 개선 여지가 있음.
3. 핵심 가정: NAND flash 채널(문턱전압 분포, program interference, retention noise 통계적 모델) + AWGN 비교 채널.
4. 핵심 기법 1(부호설계): 기존 GC-LDPC의 global part를 global-local(`G_{L,k}`, k=1..u)과 global-global(`GG`) 컴포넌트로 재분할하는 2계층 결합(HC-GC-LDPC) 구조, `g × u = n × t` 제약.
5. 식의 의미: u(global-local 컴포넌트 수)와 v(GG 행 수)를 독립 조정해 coupling 세밀도와 강도를 분리 제어, u가 클수록 성능 향상, v가 클수록 성능 저하.
6. 핵심 기법 2(디코딩): AFA-TPA는 로컬 위상에서 실패한 서브코드 벡터 `f`(4×1)를 추적, `|f|_1` 개수에 따라 필요한 global 체크노드 부분집합만 활성화(1개 실패 시 최소 subset, 3개 이상 시 전체 사용).
7. 구현 디테일: local phase 최대 IL, global phase 최대 IG, 전체 Imax=(3,2,7); 디코딩 알고리즘 자체는 normalized min-sum(NMS, factor 0.7, 최대 반복 20)과 comparable CNU로 정렬.
8. 최적화 절차: masking, fold expansion으로 rate/컬럼weight 튜닝(Example1: weight6·rate0.895, Example2: weight5·rate0.901, Example3: u=3,v=1로 균형).
9. 결과: AWGN에서 BER 10⁻⁵ 기준 SC-GC-LDPC 대비 약 0.1dB 이득, AFA-TPA는 TPA 대비 CNU 약 3.5~4.1%, ITPA 대비 약 1.8~2.72% 감소.
10. 한계: 시뮬레이션만 수행(HW 미설계), gatecount/throughput 수치 없음, GF(197) 기반 특정 대수구조에 종속된 부호설계로 일반화 난이도 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| HC-GC-LDPC H-matrix (local/global-local/global-global 2계층 결합) | `ecc_top.cpp` `Load_PCM()` | GF(197) 기반 GC-LDPC 특유 구조로 base matrix 자체가 다름, 이식 난이도 높음 |
| AFA-TPA 적응형 부분 global 활성화(실패 서브코드 수 기반 subset 선택) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 실패 위치 기반 조건부 부분 디코딩은 iteration 스케줄링 최적화 아이디어로 이식 검토 가치 |
| Normalized min-sum 디코딩(factor 0.7) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 보유(min-sum 변형)로 중복 기여, 신규성 낮음 |
| 조기 종료(로컬 phase 성공 시 즉시 종료) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 조기종료와 유사하나 CRC 대신 로컬 parity 체크 기반, 대응은 부분적 |

> 적용 가치: 중간 — 부호설계(GC-LDPC 계열, GF 기반)는 우리 QC-LDPC 구조와 달라 이식 난이도가 높지만, AFA-TPA의 "실패 서브코드 수 기반 부분 활성화" 스케줄링 아이디어는 디코더 이터레이션 최적화(iteration/CNU 절감) 관점에서 검토할 가치가 있음.
