### B. 알고리즘 요약

1. 시스템: NAND Flash ECC용 QC-LDPC 코덱. rate-0.90 (8320,9280) 1KB 페이지용, rate-0.95 (32832,34560) 4KB 페이지용 두 코드로 검증, LLR word length 8-bit(soft-decision).
2. 문제: NAND용 LDPC codec은 (1) 벤더/기술마다 다른 페이지 크기로 재설계 부담 큼, (2) 긴 codeword(1~4KB)로 온칩 메모리 비용 큼, (3) throughput이 메모리 접근 속도를 넘어야 함.
3. 핵심 가정: circulant sub-matrix 크기가 8의 배수라는 제약을 두면 "byte 단위" 재구성만으로 다양한 QC-LDPC를 지원 가능(완전 재구성 대비 overdesign 방지).
4. 인코더: 기존 private-memory 구조(서브 패리티 생성기마다 독립 메모리) 대신 단일 deep-depth 메모리를 순차 접근하는 shared-memory 구조 제안, 오버헤드는 encoding latency에 `z`(circulant 크기) cycle 추가뿐.
5. 디코더: layered min-sum(LMS) 알고리즘, block-serial 스케줄에서 종전엔 prior message `L`를 FIFO와 posterior memory에 이중 저장했으나, rescheduling architecture(순서를 permutation→CNU→BNU에서 BNU→permutation→CNU로 재배치)로 FIFO 제거하고 prior message memory만 유지.
6. HW 재구성: `p`(sub-matrix 크기), `L`(block row), `M`(block column)을 configuring mode에서 설정, 8의 배수 제약(byte-reconfigurable). BNU/CNU/permutation network 모두 enable 신호로 부분 활성화.
7. Permutation network: 기존 2-cycle 구조 대비 1-cycle에 완료하는 modified reconfigurable permutation network(두 개의 barrel shifter + mux) 제안.
8. SIB-ET(sub-iteration based early termination): 매 sub-iteration(layer)마다 parity check equation `Hĉ=0`을 XOR-누산으로 점진 검사, 종전 LC-ICA 대비 iteration 내에서 조기종료 가능(1 sub-iteration의 추가 지연만 필요).
9. 결과: shared-memory로 인코더 면적 23%↓, rescheduling으로 디코더 면적 15%↓(256K gate 상당 메모리 절감), SIB-ET는 raw BER 특정 조건에서 LC-ICA 대비 sub-iteration 수 29.6%↓. 최종 칩: TSMC 90nm, core 6.72mm², 222MHz, 인코더 222MB/s, 디코더 최소 163MB/s, 디코더 1443K gate(조기종료 95.4K, 오버헤드 6.6%), rate-0.95/8-iter 기준 전력 700mW@0.9V.
10. 한계: BER 성능 개선(waterfall/error-floor) 목적이 아니라 순수 HW 면적·throughput·조기종료 최적화 논문. 특정 두 QC-LDPC 코드(rate 0.90/0.95)로만 검증, normalized factor 0.5 고정값 사용(최적화 근거 상세 미제시).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered min-sum 디코딩 | decoder.cpp LDPC_Decoder(), C2V_Cal() | 동일 계열 알고리즘(min-sum), 스케줄링 재배치 아이디어 참고 가능 |
| CNU (min1/min2, sign 누산) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 논문 CNU 구조(1st/2nd min, position index, sign 합)와 직접 대응 |
| BNU 처리 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | posterior message 갱신 로직 대응 |
| SIB-ET(sub-iteration 조기종료) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | Prime ECC의 Partial CRC 4 termination point와 유사한 목적(조기종료), sub-iteration 단위 PCE 누산 방식은 추가 아이디어로 참고 가능 |
| Shared-memory 인코더 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | 인코더 메모리 절감 아키텍처, dual-diagonal과는 다른 QC 생성행렬 방식이라 구조 자체는 참고용 |
| Rescheduling(FIFO 제거) 디코더 파이프라인 | decoder.cpp pipeline (col_M1/Z/P1~P4) | 파이프라인 스테이지 재배치로 메모리 절감하는 아이디어, HW arch 레이어 대응 |
| Byte-reconfigurable 파라미터화(p, L, M) | ecc_top.cpp Load_PCM() | 다중 코드 지원을 위한 재구성 개념, Prime ECC는 특정 QC-LDPC 고정이라 이식 난이도 중~높음 |
```

적용 가치: 높음 — NAND Flash ECC를 정면 타깃으로 한 QC-LDPC 인코더/디코더 실칩 설계이며, min-sum CNU/BNU 구조·조기종료·메모리 절감 아키텍처가 Prime ECC의 decoder.cpp/encoder.cpp/partial_CRC.cpp 레이어와 직접 대응되어 HW 면적·throughput 개선 아이디어로 즉시 참고 가능.
