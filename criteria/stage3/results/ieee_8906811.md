### B. 알고리즘 요약

1. 대상은 DVB-S2/S2X LDPC FEC 인코더로, 55개 code rate(rate `0.2~0.9`), block length `16200/32400/64800` bit(normal/medium/short frame) 전부 지원.
2. 문제: 기존 인코더 구현은 code rate 일부만 지원하거나 rate당 자원이 커서 full-rate-compatible 저자원 인코더가 부재.
3. 부호는 DVB-S2 표준 IRA 계열로, parity `p`를 정보비트 `i`의 accumulate(XOR)로 계산하는 dual-diagonal 구조.
4. 핵심: 표준 표(B.1~C.10)의 첫 비트 주소 `x`에서 나머지 359 비트 주소를 `(x + (m mod 360)*q) mod (nldpc-kldpc)` 식(3)으로 반복 산출.
5. 이 반복 주소 생성으로 전체 주소 ROM을 저장하지 않고 소수 register+counter만으로 parity accumulator 주소를 즉석 계산.
6. 최종 parity는 `p_i = p_i XOR p_(i-1)` (식 4)의 running XOR로 완성(dual-diagonal 마무리 단계).
7. 구현 트릭: modulo 덧셈을 비교기 2개+감산 1개로 대체, Informational Sequence RAM과 Output(parity) RAM을 전 설계 공용화해 중복 제거.
8. 아키텍처: preamble로 code rate 선택 -> Bank of ROMs/Address Generators를 MUX로 스위칭, Output Circuit이 read-modify-write 1-cycle로 accumulate.
9. 결과: Kintex 7에서 55 rate 전체가 LUT 4796 / FF 2988 / BRAM 6, target clock 350MHz, 기존 구현 대비 rate당 자원 대폭 절감(유사 throughput).
10. 한계: 인코더 전용, 병렬화·throughput 수치 미제시, DVB-S2 표준 주소 테이블에 강결합.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-diagonal(IRA) parity accumulate 인코딩 | encoder.cpp LDPC_encoder_4KB() | 개념(dual-diagonal)은 공유하나 DVB-S2 IRA vs QC-LDPC로 부호구조 상이 |
| 표준 주소테이블 기반 PCM 참조 | encoder.cpp Generate_PCM_encoding() | Prime ECC는 QC circulant shift로 PCM 생성, 주소 ROM 방식과 불일치 |
| RAM 재사용/자원 절감 HW 트릭 | 대응 없음 | FPGA 자원 최적화로 bit-exact C++ 시뮬 범위 밖 |
```

마지막에 한 문장으로 **적용 가치**: `낮음` - Prime ECC 인코더도 dual-diagonal이나 DVB-S2 표준 IRA 인코더의 주소테이블·counter 재사용 기법은 QC-LDPC 구조와 정합되지 않아 이식 실익이 낮다.
