### B. 알고리즘 요약

1. 대상은 중국 지상파 디지털 방송 표준 DMB-TH의 QC-LDPC(RS 기반 2-symbol) 코드, code rate 0.4/0.6/0.8 3종, codeword length 7493bit 고정.
2. 기존 GPP/DSP/FPGA(ASIC) 구현은 유연성과 성능/전력 최적화를 동시에 만족하기 어려운 문제를 다룸.
3. Base matrix `H`는 127×127 순환행렬(circulant) 블록으로 구성된 quasi-cyclic 구조.
4. 핵심 기법은 TDMP(turbo decoding message passing)를 normalized min-sum 위에 적용한 것으로, 한 block row의 check node 갱신 후 즉시 variable node 갱신에 반영해 수렴 속도를 2배 향상.
5. `ldpc_up` 전용 명령어가 CNU/BNU 갱신을 담당하며, `ldpc_ck`가 매 row 처리 후 parity check 성공 여부를 검사해 조기 종료.
6. 하드웨어는 5-stage RISC 파이프라인(ASIP) 위에 decoding/memory operation/update nodes/hard decision 4개 모듈로 구성.
7. Shuffle network는 benes/banyan 대신 barrel shifter(6-stage)를 채택해 자원 사용을 줄이고 benes(13-stage) 대비 성능 개선.
8. 별도의 학습/최적화 절차 없음 (고정 알고리즘 + ASIP 명령어셋 설계).
9. XC4VLX150 FPGA에서 최대 117MHz, throughput 96Mbps(10 TDMP iteration 기준) 달성, DSP/ASIC 대비 유연성-성능 트레이드오프 우위 주장.
10. 알고리즘 개선(오류정정능력, error-floor 등)은 다루지 않고 순수 HW 아키텍처(ASIP) 논문이며 hard-decision만 지원.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP(normalized min-sum) 갱신 루프 | [decoder.cpp](decoder.cpp) `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC도 min-sum 기반이나 이미 보유한 기법, ASIP 재구성은 무관 |
| ASIP 전용 명령어(ldpc_up/ldpc_ck) 구조 | 대응 없음 | Prime ECC는 고정 HW 파이프라인이라 명령어셋 기반 유연 아키텍처와 대응되지 않음 |
| shuffle network(barrel shifter) | [GT.cpp](GT.cpp) `Make_GT_HW()` | 라우팅/edge 선택 개념상 유사하나 실질 재사용성 낮음 |
| 조기종료(parity check) | [partial_CRC.cpp](partial_CRC.cpp) `partial_crc_HW_T4()` | 유사 개념(조기종료)이나 구현 방식(hard parity check vs CRC) 상이 |

> 적용 가치: 낮음 — ASIP 유연 아키텍처 자체가 논문의 핵심이며 NAND ECC와 무관한 방송표준 대상, hard-decision만 지원해 이식 실익 없음.
