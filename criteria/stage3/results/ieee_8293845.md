### B. 알고리즘 요약

1. 시스템: 하드웨어 LDPC 디코더의 Node Processing Unit(NPU) 내부 구조. NPU는 2입력-1출력 subnode 블록들의 배치로 구성. 대상 부호는 IEEE 802.11n QC-LDPC 등 다중 PCM(rate/length는 표준 참조, 본문 미명시).
2. 문제: partially-parallel 디코더는 하나의 NPU가 여러 CN/VN을 시분할 처리 → irregular/다중PCM 지원 시 입출력 수 `I`가 run-time 가변, 이를 지원하면 HW 자원·critical path가 급증.
3. 모델: CNPU는 `I=dc` 입출력, VNPU는 `I=dv+1`(채널 intrinsic + a posteriori 출력). 각 출력은 자신 대응 입력 제외 나머지 입력의 조합. subnode 연산은 알고리즘별로 다름(본 논문은 연산 종류에 무관한 배치 최적화).
4. 핵심기법: 일반화된 flexible `dual-tree` NPU 구조. summing 섹션(binary tree로 전 입력 결합)과 combining 섹션(각 stage 신호 `Tst`를 이전 stage와 `2S-s` stage 신호로 결합)으로 각 출력이 `I-1` 입력 조합을 갖게 함.
5. 핵심 식: subnode 지연 `D = max(2, 2⌈log2(I-1)⌉ + ...) ≈ 2⌈log2(I-1)⌉` → forwards-backwards의 `I-2` 대비 크게 짧은 critical path.
6. 확장/flexibility: bypass MUX를 key subnode 출력에 배치, 활성 입력 `i≤b`(bypass threshold)면 subnode 우회 → null 입력 없이 run-time 가변 degree 지원(stochastic처럼 null 없는 연산에도 적용). clock/power-gating으로 미사용 subnode 절전.
7. 구현 디테일: full flexibility엔 `2(I-2)` MUX만 추가. relevance 파라미터 `r`로 실제 필요한 degree 집합 `isup`만 지원하도록 불필요 MUX 제거(예: 802.11n VNPU I=13, isup={3,4,5,7,8,9,12,13}, 9 subnode 제거).
8. subnode 수: 제안 구조 `3I-6`(forwards-backwards와 동일)이나 critical path는 더 짧음. multi-tree(`~I(I-2)`)보다 subnode 훨씬 적음.
9. 결과: Altera Stratix IV FPGA 합성. 4-bit unsigned min subnode(MSA CNPU) 기준, 802.11n(I=DC=22) CNPU에서 HW효율(fmax/ELB)이 multi-tree 대비 122%, forwards-backwards 대비 248% 향상. dual-tree만 inherent flexibility 보유.
10. 한계: 순수 HW 배치 최적화(정정능력/BER 무관), FPGA 합성만(ASIC/gatecount 미제시), 이득의 상당부분이 다중PCM run-time flexibility에서 옴, 4-bit MSA 예시.
