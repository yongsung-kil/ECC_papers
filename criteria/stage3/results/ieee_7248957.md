### B. 알고리즘 요약

1. 시스템은 BEC(binary erasure channel) 상에서 소스 블록을 causal하게 인코딩·디코딩하는 anytime coding 프레임워크, 코드율 `R=K_B/M_B`.
2. 기존 S-LDPCC(결정론적 base matrix)는 error floor가 높고, SC-LDPC(완전 랜덤)는 유한지연 성능이 나쁨 — 두 방식의 중간을 노림.
3. 채널모델은 순수 BEC(erasure probability `ε`)이며 NAND read-channel 모델(AWGN/RBER)과 다름.
4. 핵심 기법은 base matrix `B[1,t]`의 entry `Z_{j,i}`(연결수)를 랜덤 변수로 하여, 각 변수노드가 체크노드를 클래스(크기 `w`)로 분할하고 클래스당 정확히 `x`개 연결을 갖도록 구성(P-LDPCC).
5. 제약식 `Σ Z_{i+(k-1)w+l,i} = x` (식 8)로 각 변수노드의 총 연결수를 고정, `x/w` 비율이 클수록 erasure 확률은 낮아지나 error floor는 높아지는 trade-off.
6. `w=x=1`이면 S-LDPCC로 환원되어 P-LDPCC가 S-LDPCC의 일반화임을 보임.
7. lifting 규칙(L1, L2)으로 base matrix를 실제 parity-check matrix `H[1,t]`로 변환(순열행렬 및 회전형 0/1 패턴 사용).
8. 학습 없음. multi-edge density evolution(식 11-14)으로 점근 성능 및 delay exponent를 해석적으로 유도.
9. 결과: delay exponent `α ≈ (x/w)log(ε)` (식 15) conjecture를 DE와 유한길이 시뮬레이션 slope로 검증(예: x=1,w=5,ε=0.4 → 0.1828 vs DE 0.1831); 동일 error floor 대비 SC-LDPC보다 P-LDPCC가 더 나은 유한지연 성능.
10. 한계: HW 미설계, BEC 전용 이론/시뮬(AWGN·NAND read 채널 미검증), delay exponent는 증명 없는 conjecture.

### C. Prime ECC 관련 모듈 핀포인트

| P-LDPCC 부호 구성(랜덤 base matrix, lifting) | `ecc_top.cpp` `Load_PCM()` | convolutional/spatially-coupled 부호 구조로 Prime ECC의 고정 QC block-code 구조와 근본적으로 다름 - 대응 미약 |
| Expanding-window message passing decoder | `decoder.cpp` `LDPC_Decoder()` | BEC 전용 소거정정 디코딩으로 min-sum soft decoding과 무관 - 대응 없음 |
| Density evolution / delay exponent 분석 | 대응 없음 | 이론 해석 도구로 코드베이스에 대응 모듈 없음 |

> 적용 가치: **낮음** — BEC anytime 통신용 convolutional 부호 이론 논문으로 NAND binary QC-LDPC ECC(부호설계·디코더·HW)와 채널모델·부호구조·목적이 모두 상이함.
