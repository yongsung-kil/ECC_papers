### B. 알고리즘 요약

1. 대상은 특정 부호가 아니라, IEEE 802.11n/16e/15.3c/3an 등 최신 표준의 다양한 QC-LDPC 디코더를 한 칩에 담는 **재구성형 HW 가속기 설계 방법론**.
2. 문제: 고전 high-level synthesis(HLS)는 RTL micro-아키텍처만 다뤄, memory·communication 병목과 macro/micro 상호작용이 큰 demanding 응용의 전체 아키텍처 설계를 지원하지 못함.
3. 관찰: LDPC 디코딩은 VN/CN 노드 연산이 fully-serial~fully-parallel 스펙트럼을 가지며, 병렬화 정도를 바꾸면 메모리·인터커넥트 구조도 함께 바뀌어 분리 설계가 불가능.
4. 핵심 주장: computing unit + memory + communication + macro/micro-아키텍처를 **하나의 결합 최적화 문제**로 다뤄야 함(기존은 분리 처리).
5. 방법론: quality-driven model-based 설계 패러다임 — 요구 품질을 behavior·구조/파라미터 제약·목표로 모델링하고, pre-designed generic architecture template를 인스턴스화해 design-space exploration 수행(Fig.2, Fig.3).
6. 재구성 전략: FPGA의 총체적 재구성은 오버헤드가 커 고throughput/저전력 요구를 못 맞추므로 ASIC 기반, 단 basic processing unit 내부가 아닌 **유닛 간 interconnect만 재구성**(부호 구조 유사성 활용).
7. 탐색 절차: 가장 demanding한 code class(예: 802.15.3c rate7/8, CN degree 32는 serial이 유리; rate1/2는 parallel이 유리)의 아키텍처를 먼저 확정 후, 덜 demanding한 case와 구조 유사성을 찾아 자원 재사용.
8. 학습/최적화 절차: template 인스턴스화 → quality metric(throughput/area/energy) 대비 제약 비교 → 미충족 시 새 모듈 설계 또는 요구 완화 후 재탐색(iterative).
9. 결과: 구체 수치·합성 결과 없음; "현재 이 방법론을 실제 LDPC 가속기 설계에 적용 중"이라고만 언급.
10. 한계: 순수 방법론 프레임워크로 구체 부호·디코더 알고리즘·gate count·throughput 전무, LDPC는 설명용 예시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| macro/micro 아키텍처·메모리·인터커넥트 결합 탐색 방법론 | 대응 없음 | 방법론 자체는 코드 모듈에 대응 안 됨 |
| VN/CN 노드 병렬화 스펙트럼(serial~parallel) | `decoder.cpp` `LDPC_Decoder()` (z=32 row 병렬, HCU) | 낮음: Prime ECC는 병렬단위·구조가 이미 고정 |
| PCM을 CNP/VNP·메모리에 배정하는 재구성 | 대응 없음 | Prime ECC는 단일 고정 QC-LDPC라 다표준 재구성 불필요 |

적용 가치: **낮음** — 다표준 재구성형 가속기 설계 방법론으로, 단일 고정 binary QC-LDPC를 bit-exact로 모사하는 Prime ECC에는 떼어 쓸 알고리즘·구조 기여가 없음.
