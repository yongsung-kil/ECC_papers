### B. 알고리즘 요약

1. 시스템: Micron B17A 3D TLC NAND(62 layer, page 16KB+2208B) 실측 채널, LDPC는 PEG로 만든 regular (4000,3600) 부호(dc=30, dv=3, rate 0.9), 최대 iteration 10.
2. 문제: LDPC 반복 복호가 iteration마다 posterior LLR을 갱신해 read latency가 큼 - iteration 수를 줄여야 함.
3. 모델: page를 binary channel로 보고(0↔1 오독), 채널 LLR `LLRch`를 상태 전이확률로 산출. mutual info `J(δL)`는 LLR 평균·표준편차 증가 시 단조 증가(Ten Brink).
4. 관찰: iteration이 늘수록 bit 1의 single-symbol LLR 평균이 전역적으로 상승하고 error bit 수는 감소(Fig.2/3). 또한 같은 page 내 intrapage frame들의 RBER이 서로 유사(Fig.4).
5. 핵심 기법(FFIA-DA): 한 frame(auxiliary)의 복호 전/후 양의 LLR 평균비 `λ = ELLR/ELLRI` 를 feedback으로 계산.
6. 핵심 식 의미: `λ`는 LLR 평균 상승 폭 = 수렴 속도 지표. 유사 RBER frame(adaptive)에서 이 상승 경향을 미리 반영해 수렴을 앞당김.
7. Adaptive decoding: 반복마다 동적 scale factor `v = λ·ELLR/ELLRI`를 계산해 check-node 갱신식 `CI = v·2·atanh(∏ tanh(0.5·VI))` 에 곱함(식10). VN/posterior는 표준식(4)(5).
8. 절차: Algorithm 1(auxiliary, λ 산출) → Algorithm 2(adaptive, v로 CN 스케일). frame 간 저장 오버헤드는 λ 하나뿐.
9. 결과: PE=5K/12mo(RBER 9.5e-3)에서 block BER 1e-5 도달 avg iteration 10→5, BER 1.1e-5→2.9e-6, FER 4.6e-3→1.7e-3. CSB page에서 EMAL 대비도 iteration 9→6로 우수.
10. 한계: HW 미설계(복잡도는 연산횟수 수식만, gatecount/throughput 없음), λ 최적화 미해결(future work), SP(tanh) 기반이라 Min-Sum 대비 계산량 큼.
