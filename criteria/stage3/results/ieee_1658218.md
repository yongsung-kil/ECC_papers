### B. 알고리즘 요약 (10줄 내외)

1. (3,6) 또는 (4,8) regular LDPC 코드(예제: rate-0.5, N=200000), AWGN+BPSK 채널에서 반복 메시지패싱 디코더를 대상.
2. 고복잡도 알고리즘(sum-product)은 정정능력은 좋으나 iteration당 연산시간이 길고, 저복잡도 알고리즘(min-sum, Algorithm E, Gallager Algorithm B)은 연산은 빠르나 threshold가 나빠 오히려 총 반복횟수 증가로 전체 디코딩 시간이 더 길어질 수 있는 문제를 다룸.
3. EXIT chart 함수 `p_out = f_i(p_in)` (intrinsic/extrinsic message-error rate 추적)로 각 알고리즘의 iteration별 성능을 근사, DE 대비 계산 용이.
4. 핵심 기법: "gear-shift decoding" — iteration마다 다른 디코딩 알고리즘을 선택하는 gear-shift sequence `(m_1,...,m_l)`를 정의, 알고리즘별 iteration당 계산시간 `t_i`가 다를 때 목표 오류율 `p*`에 도달하는 최소 총계산시간 조합을 탐색.
5. 양자화된 EXIT chart 값들을 정점으로 하는 "gear-shifting trellis"를 구성하고, 각 알고리즘 전환을 edge/weight `t_i`로 표현해 Viterbi류 동적계획법으로 최소비용 경로(최적 gear-shift sequence)를 탐색.
6. 확장: 알고리즘 간 메시지 알파벳(soft/hard) 호환성 문제를 클래스(Class 0/1) 제약으로 trellis에 반영, hard 알고리즘(E, B)에서 soft 알고리즘으로의 전환은 별도 매핑 테이블(Table I/II) 필요.
7. 파이프라인 HW 응용: decoding latency 대신 회로면적을 비용으로 사용, 각 파이프라인 스테이지가 알고리즘별 다중 iteration을 수행하도록 동일 동적계획법을 적용해 목표 throughput에서 최소 HW 비용 구조 도출.
8. 별도 학습 절차 없음(EXIT chart 계산 + 동적계획법만 사용, DE로 정확도 검증).
9. 결과: (3,6) 코드 SNR=1.6dB 예제에서 sum-product 단독 61.5us/b, min-sum 단독 52.7us/b 대비 gear-shift(2×min-sum+5×sum-product+10×min-sum) 40.9us/b로 최대 33~50%대 지연 감소. HW 예제(rate-1/2 (4,8)코드, throughput 0.4Gb/s)에서 sum-product 3블록 대비 gear-shift(sum-product+AlgE+AlgB) 조합으로 면적 44% 미만으로 축소. 정리: 정리 1(Theorem 1)로 gear-shift 디코더의 수렴 threshold가 구성 알고리즘 중 최선과 같거나 우수함을 증명.
10. 한계: 실제 HW 구현 없이 면적을 이론적 스케일링 가정으로만 추정, 서로 다른 알고리즘 간 메시지 호환성 매핑이 성능에 민감, irregular code는 이미 특정 알고리즘에 최적화되어 gear-shift 이득이 줄어들 수 있음(논문 자체 언급, 개방문제로 남김).

| EXIT chart 기반 동적계획법으로 최적 스케줄 탐색 | `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 테이블 인덱스 결정 로직에 참고 가능하나 Prime ECC는 EXIT chart/DP 최적화 도구를 사용하지 않음 — 대응 없음에 가까움 |

| soft-hard 알고리즘 간 메시지 호환 매핑 | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD) | HD/2SD/3SD 모드 전환 시 메시지 재매핑이라는 유사한 문제가 이미 존재, 참고 가능 |

> 적용 가치: 중간 — 직접적인 모듈 대응은 약하지만, iteration 진행에 따라 디코딩 정밀도/알고리즘을 동적으로 전환해 latency·HW비용을 최소화한다는 설계 철학은 Prime ECC의 적응형 LLR·다중 정밀도(HD/2SD/3SD) 스케줄 최적화에 아이디어 차원에서 참고할 가치가 있음.
