### B. 알고리즘 요약

1. 대상은 circulant-based SC-LDPC(CB-SC), 파라미터 `z,κ,γ,m,L`; 예 `κ=z=17,L=30,길이 8670bit`(γ=3 rate 0.818, γ=4 rate 0.757), 저rate 예 `2940bit rate 0.564`, AWGN 채널.
2. 문제: 유한길이 SC 코드의 error-floor는 absorbing set(AS)/trapping set 때문인데, 기존 cutting-vector 분할은 overlap이 커 finite-length 성능이 나쁘고 m=1로 제한적.
3. 모델: γ=3/4/5별 지배적 AS(예 (3,3),(4,2),(5,3)/(4,4),(6,4))의 공통분모가 `(3,3(γ-2))` AS/TS = cycle-6, 이 cycle-6 개수를 최소화하면 상위 AS도 함께 감소.
4. 핵심 기법1(OO 분할): protograph의 cycle-6 수를 overlap 파라미터 `t_{i...}`의 함수(Theorem 2)로 표현, 독립 overlap 파라미터 `Nind=(m+1)γ-1`개에 대한 이산최적화 `F*=min F`로 최적 분할 결정.
5. 식 의미: brute-force `(m+1)^{κγ}` 분할 대신 소수의 overlap 변수 최적화로 축소(예 γ=3,m=1은 7개 변수), balanced 제약 `t0+t1+t2=3κ/2`로 편중 방지.
6. 핵심 기법2(CPO): 최적 분할 후 circulant power `f_{i,j}`를 휴리스틱 조정해 lifting 시 조건 (8)을 깨뜨려 `(3,3(γ-2))` AS/TS를 추가로 제거(cycle-4는 0 유지).
7. 구현 디테일: enumeration은 replica 반복구조(Theorem 1) `F=Σ(L-k+1)F1k`로 `Π_1^χ`만 탐색해 복잡도 절감, active cycle 가중치 `wk`로 조정 대상 1 선정.
8. 최적화 절차: PC(Core i7 3.1GHz)로 하루 이내 실용 파라미터(γ∈{3,4},m∈{1,2}) 최적 분할 도출, γ=4 m=1 예에서 cycle-6 35,280→2,870.
9. 결과: cycle-6 최대 89% 감소(m=2,γ=3에서는 0 → girth 8), BER error-floor 최대 ~2자릿수(γ=4는 5자릿수) 개선·0.6~1.3dB SNR gain(min-sum/windowed min-sum 디코딩).
10. 한계: HW 미설계·AWGN 시뮬만, SC 코드 특유의 band-diagonal 구조와 windowed decoding 전제, non-binary 확장은 별도 참조([2] flash용 NB-SC).
