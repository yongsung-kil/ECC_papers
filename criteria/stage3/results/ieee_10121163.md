### B. 알고리즘 요약

1. 대상: CC-GLDPC(CN이 rate `r_j=k_j/n_j`, memory `m_j` convolutional code인 GLDPC), VN degree `dv=2` 고정·CN 종류 2개, design rate `R=1/3,1/2,2/3`.
2. 문제: GLDPC에서 CN을 강한 부호로 바꾸면 design rate가 변해 공정 비교가 어렵고, 부호강도를 rate/length 불변으로 조절하기 어려움.
3. 모델: BEC에서 정확한 density evolution(DE) 유도 가능 → 이를 최적화 기준으로 사용, AWGN은 사후 검증.
4. 핵심기법: CN 측에 세 종류 비정칙성 도입 — (a) trellis memory 비정칙 `m1≠m2`, (b) edge fraction 비정칙 `ρ1=ρ, ρ2=1-ρ`, (c) component code rate 비정칙 `r1≠r2`.
5. rate 제약식: `R(ρ,r)=2·rc(ρ,r)-1`, `rc=ρr1+(1-ρ)r2`로 design rate 고정하에 `r2(ρ)`가 `r1,ρ`로 결정 → 탐색공간을 `r1,ρ,m`으로 축소.
6. 부호구성: rate 1/2 mother trellis를 `k_j`개 결합·puncturing해 `r_j≥1/2` component code 생성.
7. threshold 계산: DE 재귀 `q(i)=ε·fT(q(i-1))`와 BP-EXIT `hBP(ε)`로 BP threshold, 그 적분으로 MAP threshold 산출.
8. 최적화: `ne≈512`로 이산화한 exhaustive grid search로 BP-optimized/MAP-optimized 앙상블 도출 (예 R=1/2: `ρ*=0.5212, m=(3,2), r1=0.6554`).
9. 결과: 비정칙성으로 BP threshold가 fully-regular 대비 유의미 개선, MAP-optimized는 regular와 근소차; `n=102400` AWGN BER 검증. 단 저rate에서 waterfall 개선 큼.
10. 한계: BP-optimized 부호는 error floor 뚜렷(R=2/3에서 `r2=0.9540` 심한 puncturing 탓 특히 심각), HW 미설계·시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN=convolutional code(trellis/forward-backward) 복호 | 대응 없음 | Prime ECC는 min-sum single-parity CN이라 트렐리스 CN 미지원 |
| GLDPC 앙상블/부호구조 설계 | 대응 없음 | binary QC-LDPC(base+shift) 구조와 상이, `Load_PCM()` 적용 불가 |
| BEC/AWGN DE·threshold 최적화 | 대응 없음 | 부호설계 이론이며 우리 코드 로드 대상 H-matrix가 아님 |
| Density Evolution/BP-EXIT 기반 rate 설계 | 대응 없음 | 교과서 이론 수준 재사용에 그침 |

적용 가치: 낮음 — CN을 convolutional code로 대체한 GLDPC 앙상블 threshold 최적화 이론으로, min-sum 기반 binary QC-LDPC인 Prime ECC와 부호구조·복호기가 근본적으로 달라 이식 경로가 없다.
