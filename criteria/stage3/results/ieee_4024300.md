### B. 알고리즘 요약

1. 채널은 burst erasure channel으로, hidden Markov chain(HMC-EC) 기반 상태전이로 패킷 단위 소거 확률이 결정되는 모델(예: Gilbert-Elliott, CLBuEC)을 사용.
2. 기존 memory-less erasure channel 가정(패킷이 독립적으로 소거)은 실제 burst 손실(위성/무선 패킷 오류)을 과소평가하므로 이를 제거하고 burst 특성을 직접 다룸.
3. 핵심 가정은 코드워드가 대부분 하나의 burst에만 영향받는 영역(burst 길이가 코드워드 길이의 상당 부분)을 다룸.
4. 핵심 기법은 stopping set 개념 기반 `Lmax`(어떤 위치에서 시작해도 성공적으로 복구 가능한 최대 burst 길이) 최대화를 위한 variable node 순열(permutation) 알고리즘.
5. `Lmax ≈ p*n`(p*: memory-less erasure channel threshold)이며, 알고리즘은 concentrated stopping set을 그래프상 먼 위치로 분산시켜 `Lmax`를 키움.
6. eIRA(구조화된 parity 부분을 가진 IRA 코드) 코드는 연속된 weight-2 column 배열 때문에 `Lmax`가 낮으므로, 이를 균일 분산시키는 systematic-only 최적화도 제시.
7. 부수 트릭으로 systematic symbol만 최적화하는 준최적(suboptimal) 버전을 도입해 계산량 절감.
8. 최적화 절차는 burst 길이를 1부터 점진 증가시키며 non-resolvable 지점을 찾아 variable node를 그래프 밖 노드와 교환하는 그리디 탐색.
9. 결과: rate 1/2 (2000,1000) random 코드에서 `Lmax` 836→904로 개선, decoding failure rate 기준 b(천이확률)에서 1~2 자릿수 개선.
10. HW 미설계, packet-level erasure 채널 시뮬레이션에 한정되며 bit-level NAND 채널 모델과는 직접 대응 관계 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Variable node 순열 재배치(Lmax 최적화) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 변경이라는 점에서 위치상 유사하나, burst erasure 대응 목적이라 binary symmetric/AWGN 채널 기반 NAND ECC 정정능력과 직접 연결되지 않음 |
| Stopping set 분산(erasure decoding) | 대응 없음 | Prime ECC는 hard/soft decision min-sum 기반 decoder로 erasure decoding 알고리즘 자체가 미대응 |
