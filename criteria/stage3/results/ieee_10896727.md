### B. 알고리즘 요약 (10줄 내외)

1. 시스템: AWGN 채널, BPSK 변조, hard-decision bit-flipping(BF) 계열 LDPC 디코더. 대상 부호는 dv4R050N1296(rate 0.5, column weight 4)와 (2048,1723) IEEE 802.3an 부호.
2. 문제: PGDBF-w/M(확률적 GDBF+momentum)은 GDBF-w/M의 error-floor를 완화하지만 하드웨어에 난수생성기(RNG)가 필요해 구현 복잡도가 높음.
3. 관찰: GDBF-w/M이 국소최적(local optima)에 걸릴 때 송신 부호어 근방에서 진동(oscillation)하는 경향이 있음(Fig.1,2로 실증); 확률적 flipping은 초기 iteration에서 오히려 성능을 악화시킴.
4. 핵심 기법: 기존 GDBF-w/M의 flipping function `Δ_n^(ℓ) = αx_n^(ℓ)y_n + Σs_m^(ℓ) + d_{l_n}`에서, 특정 iteration `ℓ_th` 이후 스칼라 `α`를 수신값 `y_n`의 크기 구간(κ개 구간, threshold `θ_i`)별로 `α_n`으로 분할.
5. 핵심 식(6): `α_n = A1 (|y_n|≥θ), A2 (otherwise)` (κ=3 경우) — 채널 신뢰도(수신값 크기)에 따라 다른 스케일을 적용해 perturbation 효과를 유도, 이것이 오실레이션 탈출의 핵심.
6. HW 친화 트릭: `αθ`, `A1/α`, `A2/α`를 사전계산 파라미터로 두어 `(α_n/α)·(αy_n) = α_n y_n`로 계산 — 추가 연산은 최대 2N 비교 + N 곱셈, ℓ_th 이상 진행하는 프레임에서만 1회 수행.
7. 결정론적 특성: PGDBF-w/M과 달리 RNG 불필요, deterministic 동작이라 HW 구현이 상대적으로 단순.
8. 파라미터(α, δ, momentum vector d, ℓ_th)는 실험적으로 GDBF-w/M 디코딩 실패 프레임을 수집해 grid search로 선정(Table I).
9. 결과: dv4R050N1296에서 FER=10⁻⁵ 기준 GDBF-w/M 대비 약 0.8dB, 802.3an 부호에서 FER=10⁻⁶ 기준 약 0.25dB 개선; 평균 반복횟수(ANI)는 GDBF-w/M과 거의 동일(추가 연산은 무시할 수준).
10. 한계: HW 설계·합성 미수행, 시뮬레이션(FER)만 검증; hard-decision 기반 BF 계열이라 soft-info(LLR) 기반 min-sum 디코더와는 알고리즘 패러다임이 다름.
