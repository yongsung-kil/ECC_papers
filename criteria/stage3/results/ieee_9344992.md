### B. 알고리즘 요약

1. Memory-1 LDPC 컨볼루션 부호(LDPC-CC/SC-LDPC)를 대상으로 하며, parity-check matrix는 sparse `H0`(현재 서브프레임)와 원래는 sparse였던 `H1`(이전 서브프레임과의 연결)로 구성.
2. 기존 LDPC-CC의 sliding-window decoding(SWD)은 지연-성능 trade-off가 있으나, 작은 window(=낮은 지연)에서는 성능 손실이 큼.
3. 채널은 BPSK + AWGN, 부호는 `K=32, N=64, L=32` 예제(rate ≈ KL/(N(L+1)-K) ≈ 0.49)로 시뮬.
4. 핵심 기법은 `H1`을 sparse 대신 Bernoulli(1/2) 랜덤 **dense 행렬**로 대체한 semi-LDPC-CC와, window size 2의 BP+list decoding 결합: 현재 서브프레임 `c(t)`의 BP 사후 LLR로 K개의 most reliable bits(MRB)를 뽑아 flipping+재부호화로 후보 리스트 `ĉ(t,l)` 생성.
5. 핵심 식은 두 서브프레임 LLR의 soft metric `Γ(ĉ(t,l)) = Σ(-1)^{ĉ_j} Λ_j^(t) + Σ(-1)^{ĉ_j} Λ_j^(t+1)`로, 다음 프레임까지 반영해 최적 후보를 선택(2-프레임 lookahead).
6. 확장 없음(단일 기법), 다만 Code III로 `H0=[I,I]`(sparser 구조) 변형도 비교.
7. 복잡도 절감 트릭: syndrome 기반 LLR 근사식(`Λ_i^(t)`를 `tanh`합으로 계산)으로 두 번째 프레임 전체 재디코딩 없이 후보 검증하는 simplified list decoding(SLDA, Algorithm 3) 제안.
8. 별도 학습/최적화는 없으며, serial 구현 시 조기종료용 통계적 threshold `T`를 off-line으로 학습해 사용한다고만 언급(구체 절차는 [10] 참조).
9. 결과: FER 10⁻³에서 semi-LDPC-CC가 동일 지연의 LDPC-CC 대비 list decoding으로 0.2dB~1dB(list size 증가시) 코딩이득, simplified 버전은 0.5dB 이득 유지.
10. 한계: K=32/N=64 소규모 예제·시뮬 전용, HW 설계·gatecount·throughput 없음, dense sub-matrix 도입으로 인코딩/저장 부담 증가 가능성 미검토.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sliding-window BP + list decoding(MRB flipping) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 QC-LDPC block code로 컨볼루션 windowed decoding 구조 자체는 대응 없음; list/OSD 후처리 아이디어만 참고 가능 |
| dense sub-matrix(H1) 기반 부호 구성 | `ecc_top.cpp` `Load_PCM()` | 전부 sparse QC 구조인 Prime ECC PCM과 배치되는 설계라 이식 난이도 매우 높음 |
| simplified list decoding(조기 후보 검증, syndrome 근사) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기 종료라는 목적은 유사하나 구현 메커니즘(list 후보 검증 vs CRC)은 상이 |

> 적용 가치: 낮음 — dense sub-matrix를 포함하는 SC-LDPC 컨볼루션 구조 자체가 Prime ECC의 고정 QC-LDPC(전부 sparse) 아키텍처와 근본적으로 달라 부호설계 이식은 불가하며, list decoding 개념도 소규모 예제에서만 검증됨.
