### B. 알고리즘 요약

1. 시스템: NAND flash용 (21150,19050) GC-LDPC, rate `R=0.9007`, `2K-Byte+` 블록, 2-bit soft 입력(내부 4/6-bit 양자화).
2. 문제: 고rate·장블록 LDPC는 storage/critical-path/throughput이 병목, BCH는 t 증가 시 비용 폭증.
3. 부호구조: `H_GC` = 대각의 3개 disjoint local code `H_local` + 하단 `H_global`(전역 결합), GF(151) Latin-square 초행렬 `B0`에서 대수적 구성(식 10~13), CPM size 150.
4. 부호 최적화: random/structural masking으로 cycle-6/8 제거(cycle-4-free 보장), waterfall 개선.
5. 복호 핵심: normalized min-sum(NMS, 식 9, first/second-min 탐색) + row-layered schedule.
6. local-global 2-phase: local끼리 독립 복호(`T_L=3`) 후 global 결합 복호(`T_G=1~2`) 교대, itr=5~6 내 수렴; 단순화한 feed-forward two-level도 제시.
7. HW: layer size=CPM=150, CNU 150개 3-stage pipeline(대row-weight 47/141의 sort critical-path 완화), VNU 4 col-block(600 VN)씩 갱신, 3 local이 동일구조라 barrel shifter/주소 제어 공유.
8. 메모리: L_post+sign 6-bit(126900b), CNU min1/min2/sign/idx 27300b, SRAM 뱅킹으로 대역폭 확보.
9. 결과: UMC 65nm, 3.376mm2, 322.6MHz, 평균 1.8 iter로 4.32Gb/s(조기종료 포함); 선행 [18] 대비 2.65배 throughput, TAR 12.8배.
10. 한계: gatecount 미제시, GC-LDPC 특정 구조 전제, NAND 채널은 RBER로만 근사(AWGN 시뮬).
