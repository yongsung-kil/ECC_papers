### B. 알고리즘 요약

1. `(8192,7168) (4,32)` regular QC-LDPC, sub-matrix 크기 `P=256`, 전체 256 VPU / 1024 CPU로 구현.
2. 기존 partially-parallel 아키텍처는 VN-CN 간 메시지 저장용 RAM이 커서 면적의 약 70%를 차지하는 문제가 있음.
3. AWGN 채널 가정, 6-bit 균등 양자화 사용, 양자화 손실 0.1dB 미만.
4. 체크노드는 modified min-sum: `R_cv = a * sign(prod) * min(|L_n|)`, `a=1/2`로 고정해 HW 단순화.
5. QC-LDPC의 열을 재배열해 "shift-LDPC" 구조(`H_ij[k,l] = H_i(j-1)[(k-1) mod P, l]`)로 변환 — 서브블록당 1's가 대각선으로 1칸씩 이동하는 형태로 shuffle 배선을 단순화.
6. 체크노드 프로세서(CPU)를 min/2nd-min/index/sign을 저장하는 old-reg/new-reg/sign-reg 3그룹 shift-register-circle로 구성해 CPU 간 순환 방식으로 메시지 전달.
7. 이 shift-register 구조 덕분에 메시지 저장량이 전통 방식 196224 bit → 65536 bit로 67% 절감.
8. 별도 학습/최적화 절차 없음 (구조적 재배열 및 HW 스케줄링만).
9. 결과: 0.18um 공정 150MHz, throughput 1.8Gbps, max iteration=20, die 4.3mm x 4.3mm, 메시지 메모리가 전체 면적의 35.5%(기존 대비 약 절반).
10. 한계: BER/정정성능 개선은 논문 목표가 아니며(코드 자체는 PEG 기반, Shannon limit 1dB 이내라고만 언급), 순수 HW 메모리·처리량 최적화 논문. Error-floor 곡선 등 구체적 시뮬 데이터는 본문에 제시되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified min-sum CN 연산 (`a=1/2` scaling) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum과 유사, scaling factor 고정값 참고 가능 |
| VN 처리 / 6-bit 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | 소프트 양자화 비트폭 설계 참고 |
| shift-LDPC 열 재배열 (메시지 메모리 절감) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | old/new/sign register 순환 구조가 메모리 절감 아이디어로 참고 가능 |
| QC-LDPC → shift-LDPC 변환(부호구조) | `ecc_top.cpp` `Load_PCM()` | 부호구조 자체는 특정 PEG 기반 H-matrix로 고정, 이식 난이도 높음 |
| 인코더 | 대응 없음 | 논문에 인코더 구현 상세 없음 |

> 적용 가치: 중간 | 메시지 메모리 67% 절감 아키텍처(old/new/sign register 순환)는 decoder.cpp의 HW 메모리 최적화에 참고할 만하나, 정정성능 개선이 없고 코드가 특정 QC-LDPC(P=256, (4,32) regular)에 고정되어 있어 재검증 부담이 있음.
