### B. 알고리즘 요약 (10줄 내외)

1. 시스템/부호: LDPC + ISI 채널의 serial concatenation, length 2048·rate 1/2 irregular LDPC(`λ(x)=0.283x+0.399x^2+0.318x^7`, `ρ(x)=0.6x^5+0.4x^6`), PEG zigzag로 systematic triangular H 생성.
2. 문제: 부호화-채널 컨볼루션(ISI)의 등화와 복호를 별도 엔티티로 Turbo 반복하면 복잡, 이를 하나로 통합하려 함.
3. 채널/모델: minimum-phase 임펄스응답 `F(z)=1+f1 z^{-1}`(시뮬은 `1+0.5z^{-1}`, 2-tap), AWGN, DFE(feedback filter로 tail 상쇄) 구조 전제.
4. 핵심 기법: DFE의 hard slicer 대신 VN의 LLR 추정치를 soft-bit `λ(ξ)=tanh(ξ/2)`로 변환해 아날로그 도메인에서 ISI tail을 `ri - f1·tanh(ξ_{i-1}/2)`로 상쇄한 뒤 다시 LLR화(식 2,4).
5. 핵심 식: 수정 VN 식 `v_m(i)=(2/σn^2)[r(i)-Σ f_m tanh(ξ_{i-m})] + Σ u_k`(식 5) — 대괄호항이 아날로그 DFE-like 상쇄, CN 식(6)은 표준 그대로.
6. 확장: `M`(채널 메모리=feedback filter 길이)차 ISI로 일반화, `ξ_{i-m}` = 해당 VN에 들어오는 모든 LLR 합(통상 복호에선 최종 단계에만 계산하나 여기선 매 iteration 필요).
7. 스케줄링 주의: 코드 보호 전(intrinsic만으로) 사전 등화 금지 — 잡음 큰 값 처리로 성능 저하, Tanner graph 통과 후 등화해야 함.
8. 이론 확장(미구현): density evolution의 수렴식(식 9)에 intrinsic `2/σ^2` 항 옆에 등화 성분을 add-on해야 함을 지적(근사 필요, 실제 미수행).
9. 결과: 2-tap ISI에서 20 iteration 제한, 통합 방식이 sequential(선등화 후 복호) 대비 coding gain, iteration 늘수록 이득 증가하나 no-ISI AWGN엔 못 미침, BER ~10^-6에서 error floor(등화 링크가 만든 짧은 cycle 탓).
10. 한계: HW 미설계, 시뮬(첫 결과)뿐, error floor 존재, H-matrix(PEG) 재설계는 후속 과제, ISI 채널 전제라 NAND 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 수정 VN 식(등화 상쇄 항 포함) | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` 등, `LDPC_Decoder()` | VN 처리 접점이나 ISI 등화 항은 NAND에 불필요 |
| soft-bit `tanh(ξ/2)` 아날로그 변환 | 대응 없음 | 우리는 min-sum(tanh 미사용), full-tanh soft-value 도입은 역행 |
| ISI tail 상쇄(DFE 통합) | 대응 없음 | NAND는 ISI 채널이 아니라 등화 개념 자체 부재 |
| PEG zigzag H-matrix 설계 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 우리는 고정 QC-LDPC, PEG irregular 부호와 상이 |

적용 가치: **낮음** — ISI 통신채널의 등화-복호 통합 기법으로, ISI가 없는 NAND flash에는 등화 개념이 적용되지 않는다. tanh soft-bit·irregular PEG 부호도 min-sum 기반 고정 QC-LDPC인 Prime ECC와 방향이 어긋난다.
