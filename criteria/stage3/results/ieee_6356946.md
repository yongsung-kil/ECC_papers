> 총평: CCSDS 딥스페이스 표준 AR4JA(Turbo-like RA 계열) 부호의 protograph 확장(구성) 기법 논문으로, PEG+ACE 조합 자체는 QC-LDPC 구성에도 원용 가능한 표준 기법이나 NAND 직접 타깃이 아니고 HW/복잡도 데이터가 전무해 이식성은 낮음.

### B. 알고리즘 요약

1. CCSDS 딥스페이스 통신 표준 채널코드로 채택된 AR4JA(Accumulate-Repeat-4-Jagged-Accumulate, RA 계열) protograph 부호를 대상, code rate 1/2, k=1024/n=2048 예제로 검증.
2. 기존 구성법(random expansion, PEG 단독, density evolution)은 성능-구조성(하드웨어 구현 용이성) 간 트레이드오프가 존재해, 우수한 BER과 QC(quasi-cyclic) 구조를 동시에 만족하는 구성법이 부재했음.
3. 채널은 AWGN(딥스페이스 채널 근사), 변조는 BPSK로 가정.
4. 1단계(First expansion)는 개선된 PEG 알고리즘으로 protograph를 작은 확장계수(f=4)로 복제하며 check node degree 제약까지 고려해 girth를 최대화.
5. 2단계(Second expansion)는 1단계 결과의 '1' 원소를 순환치환행렬(circulant permutation matrix)로 치환하는 개선 ACE(Approximate Cycle EMD) 알고리즘으로, EMD가 목표치(`d_ACE=10, K_ACE=8`) 이상이 되도록 무작위 순환 시프트를 재탐색해 loop 간 연결성을 확보 - error-floor 저감.
6. 확장으로 base matrix 크기 128의 quasi-cyclic 구조(`H_qc`, 순환행렬 블록)를 형성해 shift-register 기반 하드웨어 구현을 용이하게 함.
7. 부수 트릭: 1단계에서 4-loop를 우선 제거하고, 2단계에서 EMD 제약(`L≤2d_emd → l_emd≥K_emd`)을 만족할 때까지 재탐색하는 방식으로 girth와 loop 연결성을 분리 최적화.
8. 최적화 절차는 PEG(그리디 girth 최대화) + ACE(제약 만족 재탐색) 2단계 알고리즘이며 별도 학습(NN/GA) 없음.
9. 결과: two-step expansion이 random/PEG 단독 대비 낮은 BER(그림7), CCSDS 표준 AR4JA와 유사한 BER 곡선(그림6)을 시뮬레이션으로 확인. 구체적 gain 수치(dB, gate count)는 본문에 없음.
10. 한계: 시뮬레이션(AWGN/BPSK)만 수행, HW 구현·gatecount·throughput 전무. AR4JA는 accumulate 구조를 포함한 RA 계열로 고전 binary QC-LDPC와 부호 구조가 상이하며, 딥스페이스 채널(장거리, 저SNR) 특화로 NAND 채널모델과 다름.
