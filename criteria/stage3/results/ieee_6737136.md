### B. 알고리즘 요약

1. 시스템: architecture-aware(AA) QC-LDPC, base matrix `Hb`(Mb×Nb), sub-matrix 크기 `z`, 802.11ad(rate 13/16 등)·802.11ac(rate 5/6) 다표준, LLR 양자화 7bit->4bit 가변.
2. 문제: 멀티-Gbps LDPC 디코더에서 완전병렬 ASIC은 라우팅 혼잡·면적·전력 문제, 저전력 모바일용 유연 고throughput 설계 필요.
3. 모델: layer scheduling SPA/Min-sum, posterior `Qn`(column order 저장)과 extrinsic `rmn`(row order 저장), permutation network shift `P(m,n)`.
4. 핵심 기법: layer decoding — 각 iteration은 `Mb` sub-iteration, a priori `qmn=|Q|n - rmn`, `rmn=f(qmk)` (min-sum), `Qn=qmn+rmn` 후 permutation network로 되돌림.
5. 식 의미: extrinsic을 posterior에서 빼 a priori를 얻고 CN 갱신 후 다시 더하는 layered 갱신으로 수렴 iteration 감소(교과서 layered 기법).
6. 핵심 확장: half-row-based layer decoding — BFU 병렬도를 code length의 절반으로 낮춰 라우팅 완화, CN 계산을 2 phase로 분할.
7. 구현: 8 slice + 1 cross slice의 19-way SIMD ASIP(213bit 명령어), barrel rotation/add-sub/min-sign, half-layer 또는 quarter-layer 병렬 구성 가능, layer compression으로 저rate 모드 cycle 축소.
8. 최적화: Target tool suite로 ASIP 생성, C+inline assembly 펌웨어, normalized/offset min-sum 선택, 최대 iteration·양자화 프로그래머블.
9. 결과: 40nm(40G) TSMC 500MHz 0.9V, 802.11ad rate 13/16에서 5.3Gbps@5iter, latency <150ns, 4.3 pJ/bit/iteration, 면적 0.216mm2. 802.11ad 3iter 시 7.6Gbps@90ns.
10. 한계: 무선표준 특화, 부호설계/정정능력 개선 없음, min-sum은 기성 알고리즘, NAND read soft-info(1~3bit) 대비 4bit+ 양자화.
