> 총평: MIMO MCMC 샘플링을 단거리 LDPC에 이식한 완전히 다른 복호 패러다임 — Prime ECC의 min-sum QC-LDPC와 알고리즘·구조가 무관하여 이식가치 없음.

### B. 알고리즘 요약

1. 시스템: `(N,K)` binary LDPC, AWGN 채널, generator matrix `G` 기반 표현 `y=Gx+n` (MIMO 검출식과 동형화).
2. 문제: BP는 factor graph에 짧은 루프가 있으면 단거리 부호에서 suboptimal — 짧은 부호에서 BP 성능을 낮은 복잡도로 개선하는 것이 목표.
3. 모델: LDPC 복호를 MIMO 검출과 동일 형태로 보고 GF(2) 이진 곱셈으로 대응, MCMC(마르코프 체인 몬테카를로) 샘플링 적용.
4. 핵심기법(MCMC): 비트별 LLR `L=(1/2σ²)(||y-Gx_{=0}||²-||y-Gx_{=1}||²)` `(2)`로 `P(x=0)` 계산, 균일난수 `r`과 비교해 비트 샘플링 `(4)`.
5. 식 의미: 후보 codeword와 수신벡터의 유클리드 거리 차로 각 비트 posterior를 추정 — 근사 MAP 표본 생성.
6. 개선1(MCMC-S): 초기 iteration에서 잔차가 noise+에러라 `σ*²=||y-Gx||²/2` `(5)`로 분산 보정, 비트 flip 변화량 `Δ` `(7)`(8) 정렬해 에러확률 큰 비트부터 갱신.
7. 개선2(MCMC-L): 매 iteration 후 `L`개 최적 경로 보존(polar SCL 유사) — 정확값 대신 리스트 유지.
8. 구현 디테일: `Gxj`를 binary addition으로, flip 갱신 `-Gxj+L=-Gxj-G` `(10)`로 증분 계산해 유클리드 거리 재계산 절감.
9. 결과: `(20,10)`에서 MCMC-S/L이 BP 대비 각 0.5dB/1dB(@BER 1e-3) 이득, `(32,16)`에서도 MCMC-L 우세; VLSI 아키텍처(초기화+iteration+sort) 제시.
10. 한계: 시뮬만(HW 합성 결과 없음), 단거리 부호 전용, 복잡도 `O(N·K·L)`로 무거움, sparse H·병렬 message passing과 무관한 패러다임.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MCMC 샘플링 복호 (Alg.1~3) | 대응 없음 | Prime ECC는 min-sum message passing — 몬테카를로 샘플링 복호와 알고리즘 자체가 다름 |
| generator matrix G 기반 유클리드 거리 계산 | 대응 없음 | Prime ECC는 sparse PCM(`PCM_b`) 기반, 조밀 G·거리계산 없음 |
| 비트 flip 정렬 / 리스트 경로(MCMC-L) | 대응 없음 | polar SCL 유형, LDPC HW 복호 구조와 무관 |
| 하드결정 초기화 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` | 개념만 유사, 이식 무의미 |

적용 가치: **낮음** — MIMO MCMC를 단거리 LDPC에 이식한 별개 복호 패러다임으로, min-sum QC-LDPC HW를 쓰는 Prime ECC와 알고리즘·자료구조·타깃 부호길이가 모두 불일치한다.
