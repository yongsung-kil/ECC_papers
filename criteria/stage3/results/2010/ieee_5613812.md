### B. 알고리즘 요약

1. 대상은 Steiner triple system(STS)의 point-line incidence 행렬을 `H`로 쓰는 STS LDPC 부호, 예제는 anti-Pasch STS(81)로 만든 [1080,999] (rate ~0.925).
2. STS LDPC는 열 무게 3, 행 무게 `(v-1)/2`의 `(3,(v-1)/2)`-regular 부호다.
3. 문제: error-floor를 지배하는 최소 trapping set을 찾는 것은 일반적으로 NP-hard지만, design 기반 부호는 부분적 특성화·효율적 열거가 가능하다.
4. `(a,b)` trapping set을 STS 구성(configuration)으로 해석하고, elementary trapping set이 `((3a+b)/2 | a)`-configuration에 대응함을 보인다.
5. 지배적 오류 구성을 뽑는 property `P1`(연결성, line당 홀수차수 점 최대 1개 등)을 정의; column weight 3이라 이들이 absorbing set과 일치한다.
6. 4~5 line configuration(C14, Pasch C16, Mitre D1, D7, D18, D19)을 P1로 필터링해 (4,2)/(5,1)/(5,3) 등 elementary trapping set을 도출한다.
7. 6~8 line까지 비동형 trapping set 유형을 표(Table II/III)로 분류·계수하고, anti-Pasch/anti-Mitre 부호가 특정 small TS를 반드시 포함함을 명제로 증명한다.
8. AWGN에서 250-iter BP 시뮬(6.75dB)로 오류 configuration이 최대 4-iter 동안만 transient하게 변하며 (5,1),(4,2),(5,3),(6,2),(6,4),(7,1),(6,0),(8,0)이 지배적임을 확인한다.
9. BSC serial bit-flipping은 terminal error configuration이 `(2a,0)`,`(2a+1,1)` TS로 국한됨(Lemma 5.1)을 보이고, configuration 전이 그래프(Fig.4)로 BP 시뮬 없이 FER를 계산한다.
10. 결과: [1080,999] 부호의 FER이 (7,1),(5,1),(6,0) trapping set에 의해 90% 이상 지배됨을 조합적 계산과 Monte-Carlo로 교차검증.
11. 한계: 순수 이론+시뮬, HW 미설계, STS 특정 구조 종속으로 QC-LDPC에 직접 적용 불가, 개선 알고리즘은 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping/absorbing set 열거·수렴실패 분석 | `corner.cpp` | 수렴실패 프레임 추적과 연계해 error-floor 원인 분석에 방법론 참고 가능 |
| STS point-line incidence H-matrix | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 지점이나 STS 구조는 QC-LDPC와 상이해 부호 이식 부담 큼 |
| BP/bit-flipping 반복 복호 | `decoder.cpp` `LDPC_Decoder()` | 논문은 표준 BP/bit-flip 사용, 디코더 개선 기여는 없음 |
| error configuration 전이 기반 FER 예측 | 대응 없음 | Prime ECC에 조합적 FER 예측기 부재 (시뮬 기반) |

적용 가치: **중간** — trapping/absorbing set을 조합적으로 열거해 error-floor를 시뮬 없이 예측하는 방법론은 Prime ECC의 error-floor 진단에 개념적 참고가치가 있으나, STS 특정 부호 종속이고 QC-LDPC 부호구조·개선 알고리즘·HW는 제공하지 않는다.
