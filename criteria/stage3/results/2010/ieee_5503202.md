### B. 알고리즘 요약

1. 시스템: regular `(dv,dc)` binary LDPC, BIAWGN, block length `N`=300k~500k, rate 1/7~2/3.
2. 문제: SR-BP/MS 및 DD-BMP처럼 메모리를 가진 반복복호는 메시지 독립가정이 깨져 기존 density evolution(DE)으로 threshold 계산 불가.
3. 모델: 변수노드→체크노드 링크에 memory unit `B(ℓ)` 추가, `B(ℓ)=ΨB(B(ℓ-1),X(ℓ))`, 출력 `MV→C=ΨM(B(ℓ))`.
4. 핵심기법: 반복번호 `ℓ`와 트리 깊이 `d` 2차원에서 메시지·메모리 의존성을 Bayesian network(DAG)로 표현, d-separation으로 조건부 독립 판별.
5. 핵심식: `Pe(ℓ)=P(B(ℓ)<0)+½P(B(ℓ)=0)`(식3) — 메모리 내용 분포로 비트오류확률 계산.
6. 확장: SR은 relaxation factor `β`로 `B(ℓ)=(1-β)B(ℓ-1)+βX(ℓ)`, DD-BMP는 `B(ℓ)=B(ℓ-1)+X(ℓ)`에 `sgnr` 이진 메시지.
7. 트릭: 순진한 DE는 `ℓ`에 이중지수 증가 → memory truncation으로 `B` 시퀀스를 유한차수 `m` Markov 근사(식12), 복잡도 `ℓ` 선형화. 양자화 비트 `q`=8, step `Δ` 최적화.
8. 최적화: `(3,6)`에서 m=4,5 결과 0.01dB 이내 일치, 최적 Δ≈0.08 → threshold 3.26dB.
9. 결과: DD-BMP threshold를 BP/MS와 비교 — 고rate·고degree에서 DD-BMP가 MS를 능가(rate-1/2 dv=7에서 0.30dB 우위), BP와 격차는 고rate에서 1dB 미만.
10. 한계: threshold 해석 이론뿐, HW 미설계, 실제 디코더 구현/복잡도·latency 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| density evolution / threshold 해석 | 대응 없음 | 부호설계 해석 도구, 런타임 디코더 아님 |
| DD-BMP / SR 메모리형 복호 | `decoder.cpp` `LDPC_Decoder()` (개념상) | 메모리형 이진 message passing — Prime ECC는 min-sum 기반, 별도 알고리즘 |
| 메시지 양자화 (q bit, Δ) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 양자화 개념만 공유, DE 해석은 무관 |

적용 가치: **낮음** — 메모리형 복호의 threshold 해석 이론이며 실제 디코더/HW 설계·정정능력 개선 결과가 없어 Prime ECC 이식 대상 아님.
