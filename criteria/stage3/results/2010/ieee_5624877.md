### B. 알고리즘 요약

1. 대상: 길이 1023/4095/16383, rate 0.75~0.87의 고rate EG-LDPC(Euclidean geometry) 및 등가 BCH 코드, BPSK/AWGN 채널.
2. 문제: LDPC의 soft-decoding은 고복잡도라 고rate(3/4~7/8)에서 BCH 대비 경쟁력이 정량 비교된 바 없음.
3. 복호: EG-LDPC는 `layered bit-flipping(BF)` 사용 — 행/열 weight(32/64/128)가 커서 soft SPA 부적합, hard-decision만 적용.
4. BF 절차: check-sum 계산 → 각 변수노드의 non-zero check-sum 수가 threshold 초과 시 flip, syndrome=0까지 반복.
5. Layered 모드: 첫 iteration 후부터 변수/체크노드를 열(vertical layer) 단위 순차·동시 갱신해 iteration당 클록 축소.
6. Threshold adaptation([7]) 적용, 초기 flipping threshold 23/41/77, iteration 한계 14/18/22로 설정.
7. BCH: syndrome → SiBMA(inversion-less BM) key-equation → Chien search, encoder-assisted syndrome + resource sharing으로 면적/throughput 개선.
8. HW: EG-LDPC는 cyclic 구조라 shift-register 기반 serial→parallel(factor p) 설계, CPU는 Wallace tree+CLA adder.
9. 결과(0.18um 합성): 긴 부호에서 LDPC 면적이 BCH보다 작고, throughput 고정 시 고SNR에서 LDPC 전력 ~34% 절감, 조기종료로 peak 10.5Gbps 달성.
10. 한계: EG-LDPC는 부호길이/rate 구성 자유도가 극히 제한적, 오류성능/정정능력 자체 개선은 아니며 비교연구 성격.
