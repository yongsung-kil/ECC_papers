### B. 알고리즘 요약

1. 대상: 임의의 특정 LDPC 부호(Tanner graph / parity-check matrix `H`가 주어짐), 표준 부호들(Margulis 2640, 802.11n 648/1296/1944, 802.3an 2048 등)로 검증.
2. 풀려는 문제: iterative 복호기의 error-floor를 만드는 `(a,b)` trapping set을 exhaustive search 없이 실용적으로 열거.
3. 핵심 정의: `(a,b)` trapping set = 크기 `a`의 VN 집합이 induce하는 subgraph에서 홀수차 CN이 정확히 `b`개.
4. 핵심 기법 1단(그래프 변환): 원 그래프 `G`의 각 CN에 auxiliary "flag" VN을 붙여 `G̃` 생성 — flag VN=1 ⟺ 해당 CN이 홀수 가중치 관측. 변환 PCM `H̃ = [H | I]`.
5. 식 의미: `H̃`의 codeword에서 원 VN의 무게가 `a`, flag VN의 무게가 `b` → codeword 열거 = trapping set 열거로 환원.
6. 핵심 기법 2단(customized impulse): Declercq의 improved impulse method [8]로 `H̃`의 저가중치 codeword를 근사 열거. min-sum 초기화 + OSD-1 list decoding + box-and-match.
7. 커스터마이즈: 원 VN LLR=4, flag VN LLR=150(큰 양수)으로 초기화해 nominal failure rate trapping set 유도; `a+b` 상한과 `b/a` 비율(≤0.5~0.75) 제약으로 무의미한 large-`b` set 배제.
8. 최적화 절차: impulse 방법의 반복 message-passing + list decoding(파라미터 `ncb=7`).
9. 결과: Margulis (2640,1320)의 (12,4)/(14,4) trapping set 수 1320개가 Richardson [1] 값과 일치; QC 부호는 trapping set 개수가 circulant 크기(27/54/81)의 배수 경향 확인. 단, 실행에 부호당 1분~1주.
10. 한계: 부호 특성화(enumeration)에 그침 — HW 미설계, 복호 성능/정정능력 자체 개선 없음, 결과는 실제 enumerator의 하한(일부 누락 가능).
