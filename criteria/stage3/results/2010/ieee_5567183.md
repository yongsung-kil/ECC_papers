### B. 알고리즘 요약

1. 시스템: AWGN, BP 디코더, 단부호(1008/2016 bit) rate-0.5 irregular LDPC, DE로 최적화된 degree 분포(DE15 max deg15, DE10 max deg10).
2. 문제: 단부호 LDPC는 error floor가 심각(스토리지·심우주 통신은 BER 1e-15 요구). PEG는 girth는 키우나 error floor는 여전.
3. 핵심관찰: error floor의 주원인은 elementary trapping set(ETS, 모든 CN degree 1 or 2). 같은 [w;u] 라벨이라도 ICS 구성이 다르면 기여가 다름 → cycle 수로 추가 구분 필요.
4. 핵심지표 e: `e = 0.5*(sum(d_i-2) - u) + 1` (식3), ETS-ICS 내 "구별가능(distinguishable) cycle" 수. e가 클수록(다중 cycle) T-type 변수노드↑ → 오류 메시지 증폭, 유해.
5. 유해 정의: 소형 w, 소형 u, e>1인 [w;0;e]·[w;1;e]를 "detrimental ETS"로 정의, 회피 대상.
6. 신규 metric CSE: cycle set(공통 root 변수노드로 묶인 cycle 집합)의 extrinsic message degree, cycle indicator e'로 정량화. ACE(단일 cycle)의 cycle-set 확장.
7. 구성 알고리즘 PEG-ACSE: 변수노드를 degree 오름차순 연결, root마다 depth D subgraph에서 근사 최소 CSE(ACSE=Ω) 추정, threshold ζ 미달 시 재연결 → 소형 detrimental ETS 회피.
8. 이론: PEG류가 만드는 최소 가능 ETS 크기를 정리(Theorem 3, Corollary 2~4)로 도출, `|S(R_k,t)| = s_k/2 + (ceil(s_k/4)-1)(t-2)`.
9. 결과: 시뮬(BP 50iter, 블록에러 100개까지). PEG-ACSE-8-4가 waterfall은 동등, high-SNR error floor 최저. 고SNR 오류의 ~90%가 다중 cycle(e>1) ETS 기인 확인. BLER 예: DE10 3.0dB에서 PEG-ACSE 1.98e-8 vs PEG-only 4.89e-8.
10. 한계: HW 미설계, 시뮬만, rate-0.5 단부호에 한정, 방법이 부호 자체를 새로 생성(구성법). Monte Carlo로 3dB 이상 평가 불가(향후 importance sampling).
