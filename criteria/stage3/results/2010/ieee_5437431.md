### B. 알고리즘 요약

1. 대상은 GLDPC/D-GLDPC 부호: CN에 일반 선형블록부호(BCH 등), VN에 일반 선형블록부호를 쓰는 LDPC 일반화, 채널은 BEC.
2. 문제: threshold가 capacity에 가까운 LDPC는 최소거리가 sublinear라 error-floor가 높음. GLDPC/D-GLDPC로 최소거리 개선을 노리나, component code의 (split) information function을 몰라 EXIT 분석 불가.
3. 핵심 가정: component 부호를 특정 부호 대신 최소거리 `d>=2`인 랜덤 선형블록부호 앙상블(expurgated ensemble `G`)에서 뽑는다(random component code hypothesis).
4. 핵심 기법 1: `G` 위에서 기대 information function을 rank 계산 기반 재귀식(`M`, `N` 함수)으로 정확히 산출 → CN EXIT 함수(식 3)를 앙상블 평균으로 대체.
5. 식 의미: EXIT 함수 = information function의 조합, expurgation(zero/independent column 제거)이 EXIT chart 유효조건(9)(10)(11)을 보장.
6. 핵심 기법 2: VN용 기대 split information function도 별도 재귀식으로 계산(식 22~29), generalized VN의 EXIT 함수를 랜덤 표현으로 근사.
7. 구현 디테일: `d`-bounded-distance decoding EXIT(식 4)로 MAP 대신 저복잡도 CN 복호 모델링, MAP는 `d=n`.
8. 최적화: 위 기대 EXIT를 Differential Evolution(DE, `NP` population)과 결합해 capacity 근접 GLDPC/D-GLDPC edge 분포 설계.
9. 결과: rate-1/2 length-128000 랜덤부호 시뮬에서 D-GLDPC가 LDPC/GLDPC 대비 waterfall 소폭·error-floor 1~2 orders 개선; generalized CN edge 하한 제약 시 floor 추가 개선.
10. 한계: BEC 전용 순수 이론/시뮬, HW·양자화·soft-read 없음, generalized VN/CN 구조 자체가 표준 binary LDPC와 다름.
