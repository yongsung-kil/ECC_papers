### B. 알고리즘 요약

1. 시스템: half-duplex decode-and-forward 릴레이(S→R→D), AWGN 3링크(SR/RD/SD), BPSK, `α=2` path-loss, codeword `n=150000`, 100 iteration.
2. 문제: 소스코드가 릴레이에서 복호되어 parity bit `bR = bS⊗H2^T`를 생성해야 하고, 전체 코드가 목적지에서 복호돼야 해 두 채널 파라미터에서 동시에 동작하는 코드가 필요.
3. 모델: overall relay 용량 `C = sup min{t·CSR, t·CSD+(1-t)·CRD}`, effective SNR `SNR*`로 릴레이 채널을 등가 직접링크로 취급해 설계.
4. 기법1(Single-user scenario [4]): `H1`과 확장 `H`를 각각 단일유저 LDPC로 취급, DE Gaussian approx로 degree distribution λ/ρ를 linear programming 최적화.
5. 핵심식: 코드율 `Rc = 1 - ∫ρE/∫λE`를 목적함수로, 성공복호 조건 `Σλi(fi(Lch,p)-p)<0`과 stability 조건을 선형제약으로.
6. 기법2(Bi-layer [5]): check node를 left/right로 구분, 2차원 degree distribution `λE(w,z)`로 릴레이 parity를 구분 처리 (더 정밀하나 `400^2=160000` 제약으로 복잡도 폭증).
7. 기법3(제안 Punctured): overall relay용 mother code 하나만 설계 후 parity bit을 랜덤 puncturing해 SR링크용 소스코드 생성 (`Rc` → `Rc,S`로 rate 상승), 동일 디코더 재사용.
8. 최적화: DE+Gaussian approximation, MATLAB `linprog()` 선형계획.
9. 결과: d=0.5에서 Rc,S=0.8/Rc=0.55, d=0.8에서 Rc,S=0.6/Rc=0.5; error propagation 고려 시 single-user가 최고, punctured가 최악(rate 0.55→0.8 상승 영향).
10. 한계: HW 미설계, 순수 부호설계/시뮬만, 릴레이 3링크 특화라 NAND 단일링크 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DE 기반 degree distribution 최적화 | 대응 없음 | Prime ECC는 고정 QC-LDPC H-matrix, irregular DE 설계 흐름 없음 |
| Puncturing 기반 rate 조정 | 대응 없음 | NAND ECC는 고정 rate, rate-compatible puncturing 미사용 |
| 릴레이 분산 코드(H1/H 확장) | 대응 없음 | 단일링크 NAND와 무관 |

적용 가치: **낮음** — 무선 릴레이 채널 특화 분산 LDPC 부호설계로, binary QC-LDPC NAND ECC의 디코더/HW/부호구조 어디에도 이식 접점이 없다.
