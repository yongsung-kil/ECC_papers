### B. 알고리즘 요약

1. 대상은 BIAWGN 채널의 binary LDPC로, SPA(sum-product, `tanh` 기반) BP 디코딩의 error floor를 낮추는 것이 목적(deep space·storage 응용 언급).
2. error floor를 유발하는 trapping set을 사전 열거하지 않고, 디코딩 중 전달되는 belief(부호)만으로 식별하는 수학적 분석을 제시한다.
3. Theorem 1: 정상 복호된 VN은 이웃에서 오는 belief 부호가 정답 값과 항상 일치하며, 불일치(inconsistent) belief는 최소 1개의 odd-degree CN에서 나온다는 것을 SPA 식(5)의 `tanh` 곱 부호로 증명한다.
4. 이 불일치 belief를 보내는 CN을 unsatisfied로 marking(marker=1); trapping set은 작은 pseudo-cycle들로 구성되며 두 unsatisfied CN 사이 최단경로가 trapped bit들로 이뤄진다고 본다.
5. 핵심 기법: 1단계에서 conventional BP를 돌리고(syndrome≠0이면), 2단계에서 임의의 두 unsatisfied CN 쌍 사이 모든 최단경로를 열거해 그 경로상 VN들의 값을 flip한다.
6. Searching Path Algorithm 1: 두 odd-degree CN `c_a`,`c_b`의 이웃을 root로 depth-1 트리 group a/b를 만들고 교대로 i+2 level씩 확장해 `v_ai ∩ v_bi ≠ ∅`인 교차 경로를 찾는다.
7. flip 후 수정된 codeword를 같은 conventional 디코더에 재입력해 재복호하며, trapping set의 형태·cardinality를 알 필요가 없어 임의 LDPC에 적용 가능하다.
8. quantization level이 낮으면 VN이 0으로 몰려 steady trapping set이 생기나, 값이 0인 VN은 최종결정을 random으로 하여 영향 배제한다.
9. 결과: rate-1/2 (1024,512) irregular 부호에서 Eb/N0>2.2일 때 error event의 32% 완전교정·40%는 error bit 80%↓, Eb/N0=2.6~2.8에서 약 80% 교정; error floor 영역(Eb/N0>2.4)에서 BER·FER 약 1자릿수 개선.
10. 한계: 복잡도가 conventional BP의 3배 미만(2회 BP pass + 트리 탐색), HW 미설계·시뮬만, traversable trapping set(c≥2)에만 유효하고 oscillating trapping set 23%는 미교정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2-stage BP + trapping set flip 후처리 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | error-floor 후처리 단계로 이터레이션 루프 뒤에 삽입 검토 가능 |
| 불일치 belief 기반 unsatisfied CN marking | `decoder.cpp` `C2V_Cal()`, `C2V_Cal_New_Sgn()` | CN 부호 계산부에서 marker 추가 필요, min-sum 부호 XOR과 정합 검토 |
| syndrome 기반 unsatisfied check 위치 | `partial_CRC.cpp`/`full_CRC.cpp` | 조기종료 syndrome 재활용해 unsatisfied CN 위치 파악 가능 |
| SPA `tanh` full sum-product | 대응 없음 | Prime ECC는 min-sum(min1/min2) 근사, full-tanh SPA 미보유 |

적용 가치: **중간** — trapping set 목록 없이 error-floor를 낮추는 decoder-level 후처리라 개념은 이식 여지가 있으나, SPA(tanh) 전제·latency 최대 3배·HW 미설계로 min-sum 기반 Prime ECC에 바로 붙이려면 재검증 부담이 있다.
