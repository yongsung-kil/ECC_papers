### B. 알고리즘 요약

1. SLC/MLC NAND 블록에서 수백만 회 erase-write-read 반복으로 block/page/bit 레벨 오류를 실측 특성화 (SLC 3.6M회, MLC 1.05M회 iteration).
2. 문제: MLC ECC가 MSB/LSB 페이지를 독립 정정 → dominant single-level cell-state 전이(01→11, 11→10)를 비효율적으로 처리.
3. 관측 모델: 오류는 column 방향 클러스터링, 11-bit symbol당 대개 단일 bit 오류 → RS보다 BCH 유리; MLC LSB가 MSB보다 BER 높음.
4. 핵심 기법: 호환형 BCH 쌍 `C1`(t1 정정), `C2`(t2>t1 정정) 사용. `C2`의 roots가 `C1` roots를 포함하도록 설계 → `C1` redundancy `r1`이 `C2` redundancy `r2`의 앞부분과 동일.
5. 의미: 선형성으로 `pLSB`의 `r1` redundancy = `s1` + `s2`의 앞 `r1` bit → MSB/LSB가 redundancy를 공유하며 LSB도 t1 정정 가능.
6. 인코딩: `s1`=C1(pMSB) 계산, `s2`=C2(pMSB+pLSB) 계산 (2단계).
7. 디코딩: C2로 pMSB+pLSB의 t2 오류 정정 → 셀 상태 1레벨 보정 → C1로 pMSB t1 정정 → 합으로 pLSB t1 정정.
8. 부수: WOM-code(Rivest-Shamir 3-cell 2-bit twice, rate 2/3)로 block 재기록 → erase 횟수 감소, wear-leveling 연계.
9. 결과: C2=35오류/C1=5오류 정정 vs BCH 20오류/page (동일 총 redundancy). 32k~40k P/E cycle 구간에서 신규 스킴이 MSB/LSB 모두 정정 성공, BCH는 실패.
10. 한계: HW 미설계, C2 디코더 miscorrection 취약성 미분석, 특정 MLC 칩 실험 데이터 기반이라 소자 변동성 미반영.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 호환형 BCH 인코딩(C1/C2 공유 redundancy) | 대응 없음 | Prime ECC는 QC-LDPC dual-diagonal 인코더, BCH 무관 |
| MSB/LSB 공유 정정 디코딩 | 대응 없음 | LDPC min-sum 디코더와 알고리즘 상이 |
| WOM-code SLC 재기록 | 대응 없음 | ECC 부호와 별개 write 전략 |
| NAND dominant 전이/BER 실측 통계 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()`, `Set_LLR_Th()` | 채널/soft-read 모델링 참고 여지(약) |

적용 가치: **낮음** — BCH/WOM 전용 논문으로 binary QC-LDPC 디코더·부호설계에 이식할 알고리즘 요소가 없고, NAND 오류 통계만 채널 모델 참고 수준.
