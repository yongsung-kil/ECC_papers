### B. 알고리즘 요약

1. 대상 부호: 10GBASE-T(IEEE 802.3an) (6,32)-regular RS-LDPC `(2048,1723)`, rate≈0.84, `H`는 384x2048을 6x32개 64x64 순환 permutation 블록으로 구성.
2. 문제: 정규 message-passing 디코더는 (8,8) absorbing set(fully absorbing set) 때문에 BER≈10^-10 부근에서 error-floor 발생, 10GBASE-T의 <10^-12 요구 미충족.
3. 채널/모델: AWGN, LLR `(1)`로 초기화. sum-product는 유한 wordlength에서 tanh 근사 포화로 majority decoding화 → absorbing set 악화.
4. 핵심 기법 1(디코더): offset min-sum `(5)` 채택 — magnitude를 min1으로 근사 후 offset 보정, 6bit로 sum-product 대비 0.5dB SNR 이득, CN 동적전력 절감.
5. reordered schedule `(6)(7)`: V2C를 posterior LLR - C2V로, posterior를 이전값+C2V로 갱신해 메모리·연산 재사용.
6. 핵심 기법 2(post-processing): 2-step. 정규 복호 실패 시 unsatisfied check의 edge를 tag → biasing 단계에서 tagged VN이 satisfied check로 보내는 message magnitude를 약화(absorbing set 내 상호 강화 차단) → follow-up 정규 iteration으로 정리. VN 내장(pre-biasing/biasing/follow-up 3-phase)이라 외부 배선 추가 없음.
7. HW 디테일: wordlength 6→4bit 축소로 wire length 41.2%·core area 37.9% 감소; post-processor 추가는 area +13.7%, 전체 배선 +1.7%뿐. 4bit+post-proc가 6bit 디코더 성능 상회.
8. 아키텍처: 그룹화(32VNG-1CNG)로 irregular local wire를 국소화하고 global wire를 정규화, AEF(area expansion factor) 지표로 병렬도 선택; 7-stage pipeline, iteration간 6-cycle stall.
9. 결과(실측 65nm ASIC, 5.35mm^2): 47.7Gb/s@5.5dB, early termination 시 평균 1.5~2.6 iter로 수렴; V/F 스케일링으로 6.67Gb/s@144mW. 측정 BER<10^-14에서 error-floor 미관측, 최소거리 14 undetected error가 남는 near-ML 성능.
10. 한계: 특정 (6,32)-regular RS-LDPC·(8,8) absorbing set 대상, AWGN 채널(4bit는 통신 SNR 기준), all-zero/all-one codeword 테스트.

| unsatisfied check tag / parity | `partial_CRC.cpp`/`full_CRC.cpp`(조기종료 판정) | 중: check 만족 판정 인프라 재사용 가능 |

| 그룹화/pipeline/wiring HW arch | 디코더 HW 모델(z=32, HCU, GT) | 중: 배선 국소화 전략 참고, 구조 정합 필요 |

적용 가치: **중간** — offset min-sum 자체는 이미 보유(중복)이나, absorbing-set용 message-biasing post-processing은 조기종료 후 실패 프레임에 붙일 수 있는 error-floor 저감 후처리로 신규 이식 가치가 높음.
