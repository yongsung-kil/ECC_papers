### B. 알고리즘 요약

1. 시스템: AWGN 채널, code rate 4/5, length 1675 bits LDPC 디코더 + H.264 계층(data partitioned) 영상 전송.
2. 문제: 모바일 영상 디코더의 baseband 에너지 소비를 줄이되 화질(PSNR) 손실 최소화.
3. 핵심 아이디어: 모든 데이터가 동일한 오류보호(EEP)를 요구하지 않는다는 UEP 관점 — H.264 DP가 NAL을 우선순위 A/B/C로 분할.
4. 디코더 알고리즘: offset-min-sum(`Rmn = δmn·max(κmn−β,0)`, `κmn=min|Qnm|`)을 기존 VLSI 설계[11] 그대로 사용.
5. 핵심 기법: 우선순위별로 서로 다른 LDPC iteration 수(`iterA/iterB/iterC`)를 할당 — 중요 데이터는 많은 iteration.
6. 탐색 절차 1단(Coarse Search, Alg.1): binary search로 목표 PSNR을 만족하는 iteration set의 대략값을 A→B→C 순으로 탐색(itermax=20).
7. 탐색 절차 2단(Fine Search, Alg.2): iterA를 조금 늘리고 iterB/iterC를 크게 줄여 `(ΔiterB+ΔiterC)−ΔiterA` 최대화.
8. 에너지 최소화: 탐색된 iteration 수를 룩업테이블에 저장 → DVFS 컨트롤러가 우선순위별로 주파수/전압(`fdec`,`VDDL`) 조절, `E∝V^2`.
9. 결과: TSMC 0.13um 합성, 디코더 175MHz/1.2mm2/~200mW; EEP 대비 최대 15%/35%/35% 에너지 절감.
10. 한계: LDPC 정정성능 자체 개선 없음, 영상 PSNR-품질 트레이드오프에 특화, HW는 디코더+DVFS 컨트롤러 합성 수준.
