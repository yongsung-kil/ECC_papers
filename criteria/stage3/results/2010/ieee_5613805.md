### B. 알고리즘 요약

1. 대상은 두 사용자가 서로 relay가 되는 cooperative MAC(coded cooperation), half-duplex, 코드워드를 2 프레임으로 분할한다.
2. 채널을 Rice 모델로 통합해 k-factor로 block fading(k=-inf)~Gaussian(k=+inf)을 하나의 프레임에서 다룬다.
3. 문제: block fading에 좋은 LDPC와 Gaussian에 좋은 LDPC는 서로 다르며, 양쪽 모두 잘 되는 universal 앙상블이 부재했다.
4. 기존 Rate-Compatible RootCheck (RCRC)-LDPC를 기반으로 full-diversity를 보장하는 `H2`(RootCheck) 구조를 유지한다.
5. 부호 C는 `C1`(rate Rc1=2/3)과 `C2`(rate 0.5)를 결합해 전체 rate Rc=1/3로 구성, 4개 bit class(V0~V3)로 나뉜다.
6. multi-edge type(MET) LDPC 프레임워크로 type1(H1)·type2(H2) edge 관계를 EDV `d=[d1,d2]`와 node 다항식 `v(x),mu(x)`로 표현한다.
7. 최적화는 2단계: 먼저 point-to-point로 `C1`의 (10)(11)을 최적화, 이후 H1 고정한 채 H2의 type2 edge를 differential evolution + MET density evolution으로 최적화한다.
8. profile 제약식 (14)~(16)으로 C1과 C2의 check node 비율 및 marginalization 일관성을 유지한다.
9. fading space 분석으로 outage gap이 threshold `f_C1^-1(Rc1)`, `f_C^-1(Rc)`에 좌우됨을 보이고 이를 최소화하도록 설계한다.
10. 결과: N=2000, rate 1/3에서 기존 RCRC 대비 고 k-factor에서 1.5dB 이득, C의 threshold를 Shannon limit +0.42dB까지(기존 2.3dB) 개선, BF 채널 coding gain도 동반 향상.
11. 한계: 시뮬(WER)뿐, HW 미설계, cooperative relay/fading 특화 구조.
