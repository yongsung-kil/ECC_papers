### B. 알고리즘 요약

1. 대상: DVB-S2 IRA-LDPC(BCH 연접), 부호길이 n=64800, 예시 coderate 1/2(q=90), BP 메시지패싱, HDTV FEC.
2. 문제: DVB-S2 고화질 서비스의 latency/throughput 요구를 serial은 못 맞추고 full-parallel은 면적 과대 → partially-parallel에서 throughput을 더 높이려는 HW 최적화.
3. 부호구조: H=[A|B], A는 (n-k)×k sparse(360열마다 cyclic 패턴), B는 (n-k)×(n-k) staircase 하삼각; 패리티 `pj⊕=im`, `j={xt+m mod360×q} mod(n-k)` (식1~2), q는 coderate 종속(Table I).
4. 핵심기법1: 노드 클러스터링 — A의 cyclic 성질로 360개 bitnode/checknode를 그룹화(bitnode 180그룹, checknode 90그룹), 그룹당 전용 신호선으로 메시지 전달.
5. 의미: 클러스터링으로 라우팅을 정규화해 partial-parallel 하드웨어 매핑 가능.
6. 핵심기법2: 통합 B/CFM — BFM과 CFM의 cascaded adder가 유사함을 이용해 하나로 병합, 둘 중 큰 adder를 취해 bitnode/checknode 연산 시분할 공유(면적 절감), 이전 BFM+CFM 대비 처리모듈 증가로 throughput↑.
7. 구현트릭: data aligner를 각 메모리뱅크 입력포트 앞에 배치, offset이 홀수일 때 crossbar/DEMUX/register로 올바른 위치에 정렬 저장; 2개 concatenated LLR을 한 entry에 담아 2-port 메모리로 2개 B/CFM 동시접근(4-port 회피).
8. 데이터플로우: iteration당 360 clock(bitnode 180 + checknode 180), 180 B/CFM plane, 메모리뱅크 630개(그룹 간 edge 수), 각 뱅크 360 LLR.
9. 결과: 90nm Verilog 합성, throughput 1,020Mbps(참조[13] 대비 +95.8%, [10] 대비 +1,033%), 면적 15.5mm2(+25.2%/+278%); BER/gatecount 수치 없음.
10. 한계: 정정능력 개선 아님(순수 HW throughput), IRA staircase 부호 특화, min-sum 아닌 full LLR adder tree 기반, 실측 아닌 합성.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BFM/CFM 통합 B/CFM(연산 공유) | `decoder.cpp` `LDPC_Decoder()` (CN/VN 처리) | Prime은 min-sum CN/VN 분리 처리 — B/CFM식 adder 공유는 알고리즘·구조 상이 |
| 노드 클러스터링/라우팅 | `ecc_top.cpp` `Load_PCM()` | QC 구조 로드와 유사하나 IRA staircase는 Prime의 QC-circulant와 다름 |
| data aligner / 2-port 메모리 접근 | 대응 없음 | Prime의 z=32 partial-parallel 메모리 모델과 별개 구조, IRA offset 정렬 전용 |
| 메모리뱅크/selection network | `GT.cpp` `Make_GT_HW()` | MUX 기반 경로선택이라는 점만 개념 유사, 목적 상이 |

적용 가치: **낮음** — DVB-S2 IRA staircase 부호와 BP full-adder-tree 기반 HW라 Prime ECC 3.1의 QC-LDPC min-sum 파이프라인과 알고리즘·부호구조 모두 이질적이며, B/CFM 공유·data aligner는 특정 아키텍처 종속으로 이식 실익이 낮음.
