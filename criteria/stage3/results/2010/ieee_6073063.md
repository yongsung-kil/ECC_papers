### B. 알고리즘 요약

1. 코드: circulant identity-shift 기반 `j×k` array QC-LDPC, index matrix `Ps,t = a^t·b^s mod m` (`a,b∈{2..m-1}, a≠b`), (3,6)-regular rate-1/2 예제.
2. 문제: 기존 QC-LDPC 대수구성이 small stopping set과 small girth를 동시에 고려 안 해 random LDPC보다 BER 열화·복호실패 발생.
3. 모델: BPSK/AWGN, iterative BP, 최대 20 iteration; stopping set = 이웃이 모두 최소 2회 연결되는 variable node 부분집합(BEC 복호실패 원인).
4. 기법1(Theorem 1): shift 인덱스 6개조 `(x,x',x'',y,y',y'')`가 특정 곱차 방정식 `(bx'-bx)(ay-ay')+(bx''-bx)(ay-ay'')=0 mod m` 만족 시 6-cycle 존재 → 위반시키면 6-cycle 배제.
5. 핵심식: 4-cycle 배제조건 Corollary 2 `(bx'-bx)(ay-ay')≠0 mod m` — circulant shift 차가 겹치지 않으면 short cycle 없음.
6. 기법2(Theorem 2,3): 6-cycle + row-weight 조건 `H(i,y)+H(i,y')+H(i,y'')=1`로 size-3 stopping set, `(bx'-bx)(ay-ay')=0` + `H(i,y)+H(i,y')=1`로 size-2 stopping set 존재조건 규명.
7. 구현: 최소 identity order `mmin` 도출(Corollary 3) 후 `mmin`의 배수를 곱해 small girth/stopping set 없이 긴 길이 코드 확장.
8. 최적화: 학습 없음, index matrix `P`의 대수조건 만족 여부만 검사.
9. 결과: (498,3,6)/(1290,3,6)/(3018,3,6) 예제 설계, 동일 길이 random regular LDPC 대비 BER 우수(특히 length 3018 장블록에서 algebraic이 random과 대등 이상). 수치 BER은 그림에만 있어 미상.
10. 한계: HW 미설계, (3,6) rate-1/2 짧은 길이 예제뿐, avg iteration/latency 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| circulant shift 대수조건 QC-LDPC 구성 | `ecc_top.cpp` `Load_PCM()` (H-matrix 로드) | Prime ECC는 고정 base+shift QC-LDPC라 shift 인덱스 배치가 정확히 이 논문 대상; girth/stopping 검증에 응용 가능 |
| 4/6-cycle·stopping set 배제 (error-floor) | 대응 없음 (부호구조 자체 검증 단계) | Prime ECC 코드베이스에 girth/stopping 검사 모듈 없음; 부호설계 재검증 부담 |
| iterative BP 복호 | `decoder.cpp` `LDPC_Decoder()` | 복호는 Prime ECC min-sum과 별개, 논문은 부호구조만 다룸 |

적용 가치: **중간** — 부호구조(circulant shift) 관점의 error-floor 개선 정리라 Prime ECC의 고정 QC-LDPC shift 배치를 girth/stopping set 기준으로 검증·개선하는 데 이론적 참고가 되지만, HW·고rate 예제 부재로 직접 이식이 아닌 설계검증 보조에 그침.
