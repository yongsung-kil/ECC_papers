### B. 알고리즘 요약

1. 대상은 BSC 상의 binary linear code에 대한 LP 복호(Feldman et al.의 ML 완화); 실험 부호는 rate `R=3/4`, `n=1000`, `dv=3/dc=4` random LDPC.
2. 문제: LP 복호가 실패하면 fractional pseudocodeword를 출력하는데, 이 부분적 정보를 활용해 성능을 더 끌어올리려는 것.
3. 채널 모델: BSC(bit-flip 확률 `p`), 목적함수는 likelihood `γ^T x` 또는 BSC에서 `||x - x(r)||_1` 최소화(polytope `P` 위).
4. 핵심 기법: 1차 LP 출력 pseudocodeword `x(p)`로 편차벡터 `x(d)=x(r)-x(p)`를 만들어 각 비트의 불신뢰도를 측정.
5. `x(d)` 절댓값 정렬로 신뢰(작은 `pn`개=집합 `L`)/불신뢰 비트를 구분 — 재가중의 근거.
6. 확장(2차 LP): `min λ1||(x-y)_L||_1 + λ2||(x-y)_{L^c}||_1` (`λ1<0, λ2>0`)로 신뢰 비트 변경비용↑, 불신뢰 비트 변경비용↓.
7. 이론 도구: fundamental cone property `FCP(S,C)`와 robustness factor `C`를 정의, expander/girth 부호에 대해 provable strong/weak threshold 하한 유도.
8. 최적화 절차: `λ1,λ2`는 각 `p`마다 실험적으로 평균 성능이 최적인 값 선택(`-3≤λ1≤-0.5`, `1≤λ2≤3`).
9. 결과: Algorithm 1은 LP 대비 2배 복잡도(1회 추가 LP)로 mixed-integer(Draper), random facet guessing 대비 BER 개선, provable strong threshold를 엄격히 개선.
10. 한계: LP는 min-sum보다 성능이 비슷~약간 우수하나 sum-product보다 열위, MATLAB/cvx 소프트웨어 구현뿐, HW 미설계, BSC(hard) 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP 복호 알고리즘 전체 | 대응 없음 | Prime ECC는 min-sum message-passing 계보로 LP polytope 복호와 무관 |
| pseudocodeword 편차 기반 신뢰도 재가중 | `decoder.cpp` `LDPC_Decoder()` (개념적 유사) | 불신뢰 비트 가중 개념은 적응형 LLR과 발상은 겹치나 알고리즘 구조가 달라 직접 이식 불가 |
| BSC hard-decision 채널 | `channel.cpp` `Set_R_Offset()` | hard-1bit 채널 모델만 대응, soft-read 이득 없음 |

적용 가치: 낮음 — LP 복호는 Prime ECC의 min-sum HW 계보와 다른 알고리즘 축이며, 이론/MATLAB 시뮬뿐이라 NAND ECC 이식 경로가 없음.
