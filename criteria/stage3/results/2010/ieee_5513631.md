### B. 알고리즘 요약

1. 대상: LDPC 컨볼루션 부호(LDPC-CC), 특히 rational parity-check matrix(RPCM) `H(D)`를 가진 것. 실험은 블록길이 `10^3~10^6`, BPSK/AWGN, pipeline 디코더 iteration 50~100.
2. 문제: RPCM을 Laurent 급수로 전개하면 memory order `ms=∞`(pipeline 복호 불가) + Tanner 그래프에 length-4 cycle(girth≤6) 발생 → BER 열화.
3. 핵심 아이디어: RPCM이 규정하는 제약은 유지하되, 등가 Tanner 그래프로 변환해 유한 `ms`와 큰 girth를 확보.
4. 기법: rational 항 `p(D)/q(D)`에 보조변수 `d_t = Σ_{i≥1} v_{t-2}^{(3i)}` 를 도입해 무한합 패리티식을 2개의 유한식으로 분할(식(10)), 공통항 제거로 length-4 cycle 소거.
5. 결과 등가부호: 보조변수 `d(D)`를 추가한 `Heq(D)`는 더 낮은 rate·더 긴 mother code `C2`, 원부호 `C1`은 `d(D)` puncturing으로 얻는 child code.
6. 일반 절차(Procedure 1): rational 항을 `(1,c)` 위치로 이동 후 `κ(s)(D)`를 최저차항씩 분해하며 행/열을 추가해 `Heq(D)` 생성. 다중 rational 항은 하나씩 반복 적용.
7. 복호: `Heq(D)` 위에서 표준 BP 또는 punctured-LDPC 스케줄링 복호 사용(수렴·BER 개선).
8. 예제1: girth 4→10, `ms` ∞→409, BER `10^-5`@1.9dB(원방식은 uncoded보다 나쁨).
9. 예제2: girth 4→8, `ms` ∞→203, 원방식 대비 유의미한 SNR gain. QC-LDPC(trinomial+ 항)로도 확장 성공 언급.
10. 한계: HW 미설계(시뮬만), 컨볼루션 부호·pipeline 복호 전용, 등가변환이 rate 저하(mother code) 수반.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RPCM→등가 Tanner 그래프 변환/부호구성 | `ecc_top.cpp` `Load_PCM()` | 부호구조 로드에 대응하나 블록 QC-LDPC와 그래프모델 상이 |
| pipeline/sliding-window BP 복호 | 대응 없음 | Prime ECC는 블록 min-sum(z=32) 구조, 컨볼루션 pipeline 아님 |
| puncturing 기반 등가부호 | `encoder.cpp` (부호 PCM) | punct_col 개념은 있으나 목적(dual-diag)이 달라 직접 대응 아님 |
| length-4 cycle 제거/ girth 개선 | 대응 없음 | 부호설계 원리(참고), 코드 모듈 직접 대응 없음 |

**적용 가치**: 낮음 — LDPC 컨볼루션 부호(RPCM) 전용 그래프변환으로, 블록형 binary QC-LDPC + min-sum HW 디코더인 Prime ECC와 부호패밀리·복호구조가 근본적으로 달라 이식 대상이 아님.
