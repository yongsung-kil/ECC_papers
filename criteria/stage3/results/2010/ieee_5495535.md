### B. 알고리즘 요약

1. 대상은 고처리율 turbo-like(LDPC/Turbo) 병렬 디코더의 인터리버 아키텍처로, 병렬 PE가 동시에 같은 메모리 뱅크에 접근하는 collision 문제를 다룬다.
2. 문제: 표준 준수 인터리빙 법칙을 바꿀 수 없을 때, 충돌 없이 데이터를 뱅크에 배치하면서 인터커넥트 네트워크(barrel-shifter/butterfly 등)까지 저비용화해야 한다.
3. 형식화: L개 원소를 자연순서 파티션 `Nat`과 인터리브 파티션 `Int`로 나누고, 매핑 함수 `M:{1..L}->{1..X}`가 같은 subset의 두 원소를 서로 다른 뱅크에 보내야 한다는 두 제약(1)(2)을 정의(`X=L/N`=병렬도).
4. 핵심 기법(SAGE): 자연/인터리브 두 순서를 각각 행렬 `MNat`,`MInt`로 표현하고, 대응하는 매핑 행렬 `MAPNat`,`MAPInt`를 채운다.
5. 구조 제약: 두 행렬에 공통인 데이터는 같은 뱅크로, 각 열 안에서 한 뱅크는 한 번만 사용. 아키텍처 목표: 각 열이 다른 열의 순환치환(barrel-shifter 조건)이 되도록 배치.
6. 알고리즘: 가장 제약이 강한 열을 선택→빈 셀마다 유효 뱅크 리스트를 목표 아키텍처 우선순위로 정렬→첫 원소 배정→다른 행렬에 전파→재귀. 리스트가 비면 백트래킹.
7. 구현 디테일: barrel-shifter 목표 시 한 열은 A-B-C의 순환 시프트(C-A-B 등)만 허용해 스티어링 네트워크가 성립.
8. 기존기법 [13](simulated-annealing 매핑)과 대비: [13]은 매핑은 항상 찾지만 스티어링 컴포넌트를 고려하지 않아 barrel-shifter 아키텍처를 생성 못 함.
9. 결과: 12원소 3-PE 교육용 예제에서 barrel-shifter 기반 충돌 없는 매핑(Bank A={0,1,6,3} 등)과 네트워크 제어스텝을 도출.
10. 한계: HW 합성/면적/throughput 수치 없이 교육용 예제 검증만; 부호 자체나 복호 성능과 무관한 순수 매핑 방법론.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| 병렬 인터리버 메모리 충돌 회피 매핑 | 대응 없음 | Prime ECC는 QC-LDPC circulant 기반 z=32 병렬로 인터리버 매핑 문제를 별도로 풀지 않음 |
| 스티어링(barrel-shifter) 인터커넥트 최적화 | 대응 없음 (근접: HW edge/MUX는 [GT.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Make_GT_HW()`) | Graph Thinning의 MUX edge 선택과 개념적 유사성뿐, 인터리버 매핑과 직접 대응 없음 |
| 메모리 뱅크 라우팅 | 대응 없음 | 논문은 turbo 인터리버 뱅크 배치, Prime ECC HW 모델과 정합 안 됨 |

적용 가치: **낮음** — turbo/LDPC 인터리버의 메모리 매핑 방법론으로, NAND binary QC-LDPC의 부호설계·복호 알고리즘·정정성능 어디에도 직접 닿지 않는다.
