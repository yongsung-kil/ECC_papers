> 총평: TDMP pipeline idle cycle을 seed matrix 재정렬로 없애 throughput을 올리는 HW 최적화로, 부호성능 무해하고 우리 z=32 QC 구조에 적용 검토 가치 있음(중).

### B. 알고리즘 요약

1. 대상: AA-LDPC(Architecture-Aware, `H`가 `P×P` permutation/all-zero 서브행렬 `D×L` 격자 = QC 구조)를 파라미터라이즈 VHDL로 구현, 타깃 Xilinx VirtexII FPGA.
2. 문제: 고클럭 위해 pipeline register를 많이 넣으면 처리경로 지연이 커져 subiteration 사이에 idle cycle이 강제되어 throughput이 상쇄됨.
3. 복호: TDMP(Turbo-Decoding Message-Passing) — `C`를 supercode 교집합으로 보고 `D`개 subiteration으로 layered 갱신, `Q_n^d=γ_n−λ_n^d`, CNU는 `ψ(x)=−ln tanh(x/2)` 근사(실제 SISO는 normalized Min-Sum).
4. 핵심관찰: subiteration `d`의 첫 read가 이전 subiteration의 해당 memory cell write 뒤 1cycle 이상이어야 하므로, seed matrix 연속 행들의 "같은 열 겹침"이 idle을 유발.
5. 식: 보조변수 `X(d2,d1)`(공통 열 없으면 ∞, 있으면 좌·우 1의 개수 차−1)로 idle 수를 pipeline delay `TP`와 연동해 정확 산출(`TP=2·TNET+TSISO=11`).
6. 확장: `d−1,d−2,d−3` subiteration 완료 대기까지 고려한 `T1d/T2d/T3d`와 그 max로 `T_idle^d`를 확정(식 10~13).
7. 트릭: 행 셔플은 부호 불변, 열 셔플은 성능 불변(비트순서만 바뀜)이라는 성질을 이용 → 재정렬이 error performance에 무해.
8. 최적화: heuristic(Alg.1) — 열을 weight 오름차순 정렬 후 1000회 반복으로 각 행을 idle 최소가 되게 배치, `T_idle=ΣT_idle^d` 최소 seed matrix 선택(PEG로 생성한 부호들).
9. 결과: idle cycle이 다수 케이스에서 0까지 감소(예 W55×110: 217→0), throughput 약 2배(P=32, R=0.5, fclk=145MHz, imax=10에서 >60Mb/s).
10. 한계: gatecount 없음, throughput 절대치 낮음(소형 부호·FPGA), 정정능력 개선 아님(순수 HW 스케줄 최적화).
