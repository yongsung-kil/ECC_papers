### B. 알고리즘 요약

1. 시스템: HDD perpendicular PRML read channel SoC(Hitachi Travelstar 5K500.B), 65nm, 1700Mbps, 1316mW, IDRC 면적 5.8mm², 512B 섹터당 LDPC codeword 2개.
2. 문제: 종전 Viterbi+parity post-processor(PPP) 대비 반복검출로 ISI 하 SNR/용량을 높이되 실리콘 비용·전력·안정성을 만족해야 함.
3. 코딩 구조: reverse concatenation(변조 rate 0.995 → RS(10-bit symbol) → LDPC), LDPC rate=4096/4272≈0.959(redundancy ~4.1%).
4. 핵심 부호: `column-weight-1` LDPC — 각 data bit가 정확히 1개 parity check에만 참여, 전통적 short cycle이 원천 배제되어 저가중치 잔여오류가 RS로 정정 가능.
5. 부호 생성: 선형 합동생성기로 97-square permutation matrix `P` 형성→주대각 1개 row/col puncture해 96-square로, `P^i` powers를 이어붙여 전체 H 구성(QC형, 효율적 인코딩/MP).
6. 반복검출: feed-forward 고정지연 3.5-iteration 파이프라인, 16-state Viterbi→여러 SISO(ISI 근접비트 정보교환, error-event filter로 LLR 생성)↔MP block 교대.
7. 핵심 트릭: `{1 2 1}` 등화 타깃+matched-filter metric으로 Viterbi ACS 단순화, media noise용 context-dependent biasing, extrinsic에 0.75 attenuator, 미충족 parity check 카운트 기반 조기종료+clock gating.
8. MP block: LDPC 96 parity check(각 24~25 bit even parity)에 대해 well-known `Min-Sum`(least/2nd-least reliable bit LLR + add/compare/select)으로 extrinsic 생성.
9. 결과: 종전 PPP(100/102 parity) 대비 SFR 1e-5에서 0.81dB SNR 이득, 드라이브 용량 최대 8% 증가, 실제 양산 mobile HDD 통합(실측).
10. 한계: HDD PRML 채널·ISI 결합 특화, column-weight-1 저거리 LDPC는 RS 후단 의존, NAND용 고열 QC-LDPC 독립 정정과는 목적·구조 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MP block min-sum extrinsic 생성 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 min-sum 계열이나 Prime ECC 이미 보유(중복), 이득 없음 |
| 불만족 parity check 카운트 조기종료 | `partial_CRC.cpp`/`full_CRC.cpp` (조기종료) | 조기종료 개념 유사하나 Prime ECC는 CRC 기반으로 이미 구현 |
| permutation matrix powers로 H 구성 | `ecc_top.cpp` `Load_PCM()` | QC 구성 개념만 유사, column-weight-1은 NAND 고열 QC-LDPC와 상이 |
| SISO(ISI)/Viterbi/matched-filter/timing recovery | 대응 없음 | Prime ECC는 채널 등화·검출기 계층 없음(순수 복호기) |

적용 가치: **낮음** — HDD 반복검출 read channel의 ASIC 구현으로, LDPC가 ISI SISO와 결합된 column-weight-1 보조부호라 NAND 독립 binary QC-LDPC 복호/부호설계에 이식할 신규 요소가 없음.
