### B. 알고리즘 요약

1. 대상은 binary LDPC로, 패리티검사행렬을 순열행렬(permutation matrix) 배열로 구성한 structured LDPC(QC-LDPC 일반화)이다.
2. 기존 girth 최대화 방식은 유한길이·고rate에서 harmful trapping set을 충분히 제거하지 못해 error-floor가 남는다는 문제를 푼다.
3. Latin square(quasigroup Cayley table)에서 1이 겹치지 않는 q개 순열행렬 집합 `M`을 얻고, 이 집합이 행렬연산 `⊞`,`⊡` 하에서 `GF(q)`와 동형인 체를 이룬다는 것을 보인다.
4. `W`(quasigroup 원소 배열)를 `f(·)`로 순열행렬화하여 `H=f(W)` (크기 `µq×ηq`)를 만든다; `Mt=P^t M0 Q^t` 구조.
5. Theorem 1(cross-addition 제약): `w_{i1,j1}+w_{i2,j2}=w_{i1,j2}+w_{i2,j1}` 이면 girth≥6(4-cycle 없음) — 4-cycle 검사가 대수적으로 단순화된다.
6. 구성 알고리즘: 순열행렬 블록(GF(q) 원소)을 순차 추가하며 조건 `τ`(girth/최소거리/특정 trapping set 부재)를 검사, 위반 시 블록 교체 (PEG + integer lattice 방식 결합).
7. `τ`는 TSO(Trapping Set Ontology) 데이터베이스에서 회피할 (a,b) trapping set을 지정 (예: (5,3) girth-8, (6,4)).
8. column-weight 3~4·중소 길이에서 실용적 (예: (1111,808) girth-8·d_min≥10 코드 2.4GHz PC에서 2분 미만).
9. 결과: (530,371) R=0.7 `C8(new)`가 동일 파라미터 array code보다 AWGNC SPA(최대 50 iter) 성능 우수; d_min≥10 부여 시 girth 6이라도 고rate·양호 성능 유지 → girth만으론 부족함을 입증.
10. 한계: HW 미설계, column-weight-3 전용 예제, error-floor 곡선은 그림에만 있고 본문 수치 서술 부족, non-binary 확장·저널판 알고리즘 상세는 미제공.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 순열행렬 배열 H-matrix / 부호구조 로드 | ecc_top.cpp Load_PCM() | 논문의 GF 기반 QC 부호구조를 로드하려면 여기 대응, 단 degree 3 vs 17 불일치 |
| TSO trapping-set 회피 부호설계 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정, 부호설계 재구성 레이어 부재 |
| SPA/AWGNC 복호 성능 평가 | decoder.cpp LDPC_Decoder() | Prime ECC는 min-sum 기반, 논문은 SPA 성능만 보고 |
```

적용 가치: **낮음** — error-floor 개선 아이디어(TSO 기반 trapping-set 제거)는 흥미로우나 column-weight-3·저rate 부호설계에 국한되어 Prime ECC의 고정 고rate QC-LDPC 구조에 이식하려면 부호 전면 재설계·재검증이 필요하다.
