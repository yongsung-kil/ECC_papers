### B. 알고리즘 요약

1. 대상: length `n`의 binary column-weight-three(변수노드 degree 3) 정규 LDPC, BSC 상 hard-decision Gallager A 반복복호(`Part II`, [18] 후속).
2. 문제: girth `g`와 보장 정정능력(모든 t-error 정정)의 정확한 관계, 그리고 error-floor 유발 trapping set의 회피 조건 규명.
3. 핵심결과1: girth `g>=10`이면 최대 `(g/2 - 1)`개 오류를 `g/2` iteration 내에 항상 정정 가능(Main Theorem).
4. 증명기법: bad variable node의 depth별 configuration을 tree 가정하에 재귀 열거(Lemma 1~2), incorrect message 전파 하한을 귀납으로 도출.
5. 핵심식: bad node 수 하한 `|F(m_i)| >= ...` 형태로 valid configuration(cycle<4l+4 free, <=2l+1 bad nodes)을 girth마다 최대 3개로 한정(Theorem 2).
6. 핵심결과2: girth `g=6/8`은 girth만으론 부족 — weight `<=g-2` codeword 회피가 추가로 필요(Theorem 7 등).
7. Trapping set: girth `g=8`에서 (5,3)·(8,0) trapping set이 critical number 3을 가져 3-error 실패 유발 → 이 subgraph들을 회피해야 3-error 보장(Theorem 8).
8. Decision rule: girth<10엔 Rule A(전원 일치 시만 갱신)가 Rule B(majority)보다 수렴 빠름 — 오류 적을 때 유리.
9. 결론: column-weight-3 코드의 Gallager A 보장 정정능력은 girth에 선형(=코드길이에 로그) 증가(최소거리는 선형 증가와 대비), bit-flipping보다 약 2배 정정.
10. 한계: 실HW·시뮬 없음(정리·증명뿐), hard-decision BSC·degree-3에 국한; soft/양자화 BP로의 확장은 open problem으로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping-set/codeword 회피 부호설계 | `ecc_top.cpp` `Load_PCM()` | error-floor 관점 부호설계 원리이나 degree-3 정규 전제라 고degree QC-LDPC에 직접 대응 안 됨 |
| Gallager A hard-decision 복호 | `decoder.cpp` `LDPC_Decoder()` | Gallager A ≠ min-sum(Prime ECC), 알고리즘 상이 |
| girth/critical number 분석 | 대응 없음 | 이론 분석 프레임, 코드 모듈에 직접 대응 요소 없음 |

적용 가치: **낮음** — trapping-set 회피=error-floor 개선이라는 원리적 시사점은 있으나, degree-3 정규 + Gallager A hard-decision이라 Prime ECC의 고rate QC-LDPC/min-sum 파이프라인에 떼어 쓸 실체는 거의 없음.
