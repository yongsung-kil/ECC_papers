### B. 알고리즘 요약

1. 대상: WLAN(IEEE 802.11n)/MIMO-OFDM용 regular LDPC, SPA(Sum-Product/Belief Propagation) 복호 디코더 HW.
2. 문제: 기존 pipelining은 latch로 인한 area·latency 오버헤드가 커서 이를 wave-pipelining으로 완화.
3. 모델: Tanner graph 이분 그래프상 check node(`r_ij`)·bit node(`q_ij`) 메시지 교환, 채널값·메시지는 LLR.
4. 핵심 기법: check node는 `r0_ij/r1_ij = [1±Π(q0-q1)]/2` (곱셈·감산·shift `>>1`), bit node는 `q_ij = α·p·Π r` — SPA 원식을 CFU/BFU 하드웨어로 직접 매핑.
5. wave-pipelining: 비임계경로 idle time을 줄여 클록 주기를 데이터 최대-최소 지연차까지 단축, pipeline latch 제거로 면적·전력·클록분배 단순화.
6. 구현 디테일: delay balancing 위해 조합블록에 버퍼 삽입, 입출력에 레지스터 배치; multiplier는 IEEE 754 32-bit floating point로 재설계.
7. 검증: Modelsim 7.0 기능검증 + Xilinx 9.1 합성; single/two-stage pipelining과 wave-pipelining 비교.
8. 결과(디코더): slice FF 134/140→37(wave), LUT 173/184→2로 감소; latency clock cycle 4/5→3.
9. 결과(FP multiplier): total gate count 24115/25366/26080→22505(wave), FF 601/751/846→456, 지연 12.5/13.5/15.5→11.5 cycle.
10. 한계: 부호 rate/length/BER·throughput 미제시, FPGA 합성 수치만 있고 실측·error-correction 성능 분석 없음.
