### B. 알고리즘 요약

1. 대상은 통신용 시분산(time-varying) LDPC 컨볼루셔널 부호(LDPC-CC), rate `b/c`(1/2, 2/3, 3/4 예시), 대표코드 (472,3,6) regular.
2. 문제: LDPC-CC 디코더는 보통 iteration 차원만 병렬(각 프로세서=1 iteration)이라 throughput이 제한되고, node 차원 병렬화 시 메모리 접근 충돌이 발생.
3. 핵심 접근: 부호·인코더·디코더를 공동설계(architecture-aware)하여 PCM에 대수구조를 삽입, 병렬 인코딩/디코딩 제약(C1~C6)을 만족시킨다.
4. 핵심 기법: syndrome former를 `M×M` circulant(weight-1 순환, permutation 행렬 특수형) 블록으로 구성 — parity-row/information-row 행렬을 circulant로 채워 C3(CNU 1클록 접근), C4(노드 병렬 충돌 없음)를 자동 충족.
5. circulant 구조 이유: 같은 circulant에 속한 노드들이 서로 다른 메모리 뱅크로 정렬되어 switch matrix(SW1, `ρ-to-ρ` 순환 시프트)만으로 충돌 없는 병렬 접근 가능.
6. 인코더: partial syndrome 인코더를 `ρ`-병렬화(SW0 switch matrix), all-phase termination을 circulant 링 상의 행렬 역변환으로 병렬화 — termination 회로 면적을 [9] 대비 15%로 축소.
7. 부수 디테일: `ρ=8`(byte 정렬, modulo 단순화), CNU는 min-sum(단순성), 메시지 6bit 균일 양자화(BER 손실 없음, min-sum은 SP 대비 ~0.4dB 손실).
8. 학습/최적화: 없음. 부호는 modified edge-growth(girth 유지, C6)로 구성, ACE/protograph 대체 가능.
9. 결과: 90nm CMOS ASIC 합성에서 encoder 2.0Gbps @250MHz(0.1mm²), 단위 디코딩 프로세서 2.0Gbps @250MHz(<1mm²), FPGA 디코더 400Mbps/프로세서(직렬 대비 10배), 에너지 ~70% 절감.
10. 한계: circulant 병렬도 `ρ` 상승 시(대략 upper-20s) error floor 발생, block code가 아닌 컨볼루셔널 부호 전용 sliding-window/circular-buffer 구조.

|---|---|---|

| min-sum CNU (6bit 양자화) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 min-sum 보유 → 중복 기여 |

| 병렬 partial syndrome 인코더 + termination | [encoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal block 인코더 → CC용 syndrome 인코더와 대응 안 됨 |

적용 가치: **낮음** — HW/ASIC 완성도는 높지만 LDPC 컨볼루셔널 부호 전용 공동설계로, NAND용 block QC-LDPC(Prime ECC)와 부호 구조·디코더 스케줄이 근본적으로 달라 떼어 쓰기 어렵다.
