### B. 알고리즘 요약

1. 대상은 AWGN 채널의 rate-compatible protograph LDPC 계열(rate `0.375~5/6`)로, HARQ/adaptive coding용 동일 정보블록 여러 rate 지원.
2. 문제: puncturing 기반 rate-compatible는 mother code 대비 capacity gap이 크게 벌어짐 → 대안이 필요.
3. 핵심 가정: protograph를 copy-and-permute로 확장한 LDPC 앙상블, threshold는 PEXIT chart(Liva-Chiani)로 계산.
4. 핵심 기법: 고rate daughter code에서 시작해 lower-rate/higher-rate를 base matrix `H=[H1 He]` 확장(lengthening)으로 얻어 nested embedded 구조 구성.
5. `H_{n+1}` 을 `H_n`에 소수 원소의 extension graph(6원소)를 붙여 만들고, 각 확장 graph를 threshold 최소화로 최적화(식 (5)~(10)).
6. 확장: HARQ용으로는 고rate protograph를 저rate로 extend(Fig.4 nested), rate-2/3에서 0.428, 0.375까지 확장(식 (11)).
7. 구현 디테일: degree-1 variable, degree-2/고차 혼합, linear minimum distance 조건을 만족하도록 base matrix 원소(평행 edge 수) 제약 탐색.
8. 최적화: 확장 graph 원소를 0~3 범위로 제한한 그리디/전수 탐색으로 PEXIT threshold 최소화(DE/EXIT 대체).
9. 결과: rate>2/3에서 capacity gap 0.08dB 이내, rate-2/3 0.147dB, AR4JA 대비 약 0.2dB 개선, 모든 부호가 linear min-distance 성질 보유.
10. 한계: 순수 부호설계 이론(PEXIT threshold만), 유한길이 BER/error-floor·HW 구현 없음, 단일 하드웨어로 계열 전체 복호는 장점으로만 언급.
