### B. 알고리즘 요약

1. 대상: short-to-medium 블록 GLDPC 부호. check node 제약이 SPC가 아니라 선형부호 `C(n,k)`(Hamming (15,11), RM(3,5)); AWGN 채널.
2. 문제: 기존 GLDPC는 효율적 인코딩과 rate-compatible(펑처링)이 어려움 — LDPC용 IRA/EERC 아이디어를 GLDPC로 확장.
3. 핵심 구조: `H=[H1 H2]`, `H2`를 하삼각(bi-diagonal 계열)으로 두어 iterative decoder로 인코딩 가능. 부호클래스 `Π`(변수노드 degree 2 + 마지막 degree 1) 정의.
4. IRA-like GLDPC: `n=[1,...,1]`, `D=M`. `H2`의 2차 대각 1을 (n-k) 확장 시 length-4 cycle 방지 위해 일부 degree-1 노드 허용, PEG로 2번째 에지 배치 → 인코딩 `O(N)`.
5. EERC-like GLDPC: `n_i`를 공비 0.5 기하급수로 재귀 결정(식 6) → `H2` 대각에 fractal 0 삼각형, 인코딩 `O(log2 N)`.
6. degree-1 노드 수 = `(1/2)(n-k+1)(n-k)` (IRA-like) 또는 그 하한(EERC-like); degree-1 수와 인코딩 스텝 간 trade-off.
7. 펑처링: iterative decoder가 최소 스텝에 복원하도록, 그리고 check node에 균일 분산(uniform)하도록 펑처 → 집중 펑처보다 성능 우수.
8. 설계 알고리즘: component code·degree 선택 → 결정적 `{I_ni}` 배치 → degree-1 처리 → PEG로 `H2` 후 `H1` 완성 → 펑처.
9. 결과: (603,301) Γ3(EERC-like, Hamming 컴포넌트)이 동일 길이/rate irregular LDPC 대비 BER=1e-6에서 0.4~0.5 dB 우세, 양호한 error floor. 복호는 trellis MAP.
10. 한계: HW 미설계·시뮬만, component code trellis/MAP 복호 요구, 무선 rate-compatible 지향(NAND 아님).
