### B. 알고리즘 요약

1. 시스템: AWGN 채널, LDPC convolutional code(SC-LDPC)를 블록 LDPC와 동일 복잡도에서 BER 비교. 예제 R=1/3, ms=2, 시변 Ts=1.
2. 문제: convolutional LDPC가 블록 LDPC 대비 왜 더 좋은 BER("convolutional gain")을 보이는지, 그 그래프 구조적 원인 규명.
3. 모델: 부호를 semi-infinite parity-check `Hconv` (식1)로 정의, `(ms,J,K)`-regular, 제약길이 `νs=(ms+1)c`, 주기 `Ts`로 시변.
4. 핵심기법(구성): `(J,K)`-regular 블록 LDPC의 H를 대각선 따라 `(c-b)×c` step으로 절단해 `H0`(하삼각)/`H1`(상삼각)으로 분할, 이를 쌓아 `Hconv` 생성 ([6] Feltström-Zigangirov 방식).
5. 핵심 관찰: 절단 step이 작을수록 연속 bit-node가 그래프상 항상 연결(ms>0), step이 전체(1024×2048)면 H1=0·ms=0이 되어 블록 부호와 동일 → gain 소멸.
6. 핵심기법(복호): sliding-window 디코더 — 프레임 전체 수신 전 복호 시작 가능, `νs`보다 먼 두 bit-node는 공통 check에 안 묶여 독립 → I개 병렬 프로세서로 iteration 처리.
7. 구현디테일: 시변 주기 부호로 무한 H 문제를 한 주기만 설계해 해결, window 기반으로 유한 처리.
8. 최적화: 없음(구성+시뮬 분석 위주). gain 원인으로 개선된 minimum distance, pseudo-codeword, girth, trapping-set 언급([9]-[12]).
9. 결과: JPL 비정칙 블록 부호(n≈2500, R=1/2~4/5) 대비 각 SC-LDPC가 0.6~0.9dB 이득. 절단 step을 1×2~1024×2048로 바꾼 11개 부호로 gain의 절단-step 의존성 확인.
10. 한계: 소개/분석 수준, HW 미설계·throughput 미제시, error-floor 정량분석 없음, window 복호는 큰 latency·메모리 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC(Hconv) 부호 구성 | ecc_top.cpp Load_PCM() | 낮음 (Prime은 고정 block QC-LDPC, convolutional 미지원) |
| sliding-window 반복복호 | decoder.cpp LDPC_Decoder() | 낮음 (Prime은 block 단위 min-sum, window 스케줄 아님) |
| window 내 병렬 프로세서 | 디코더 HW 모델 (z=32) | 낮음 (병렬 축·메모리 구조 상이) |
```

적용 가치: **낮음** — SC-LDPC(convolutional) 부호와 sliding-window 복호기는 무한 H·window latency·연속 프레임 구조를 전제로 하여, 고정 4KB block QC-LDPC를 클록레벨로 모사하는 Prime ECC 3.1과 부호·복호 양측 모두 이질적이고 HW 근거도 없음.
