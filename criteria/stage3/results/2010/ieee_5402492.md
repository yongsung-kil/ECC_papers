> 총평: trapping-set 타깃 bit-pinning/G-LDPC로 QC-LDPC error floor를 수 자릿수 낮추는 저장장치용 기법으로, NAND error-floor 대책으로 개념 가치가 높으나 PR채널·BCJR·외부BCH 전제라 memoryless NAND로 이식 시 pinning 커버리지·rate loss 재검증이 필요.

### B. 알고리즘 요약

1. 대상은 storage용 rate-`0.78 (2048,1600)` QC-LDPC(rate-`0.9 (4544,4096)` IRA에서 파생, `Q=64` circulant), PR1/EPR4 ISI 채널 + AWGN + BCJR 검출기.
2. 문제: LDPC iterative 복호의 error floor는 저중량 codeword가 아니라 trapping set(`(w,v)`: w VN, v USC)이 원인 — 저장장치 적용 지연 요인.
3. 핵심 가정: dominant trapping set은 HW/graph-search로 열거 가능하고(이 부호는 (5,5) 1개·(5,7) 9개 = 640개 trapping set), PR채널에서도 AWGN의 dominant set이 그대로 지배적.
4. 핵심 기법1(pinning): 각 trapping set의 한 비트를 외부적으로 '1'로 pin하면 subgraph 구조가 깨져 나머지 오류는 2회 추가 iteration으로 정정 — 256비트(vp=0~3 그룹)가 640 trapping set을 모두 커버.
5. pre-pinning: 256비트를 사전 '1' 고정(code shortening 유사), decoder는 해당 VN LLR을 최대치로 설정; PR채널은 memory 때문에 pin 비트 전송 필요, trellis-pruning으로 검출기 trellis 가지치기.
6. 핵심 기법2(post-pinning): `(256,240,t=2)` BCH로 256비트를 보호, LDPC 실패 후 BCH가 trapping set당 1비트를 정정해 검출기/복호기에 피드백(mis-pinning은 syndrome weight θ·flip 수 검사로 억제).
7. 핵심 기법3(G-LDPC): 문제 trapping set의 USC SPC 노드들을 pairwise로 묶어 GCP(generalized-constraint processor, BCJR 기반 SISO)를 구성 — 128 SPC→64 GCP로 trapping set 간 신뢰정보 브리지.
8. 최적화/시뮬 가속: Richardson importance sampling을 PR채널로 확장(trapping-set 방향 `Δ_T`로 noise biasing, Gram-Schmidt 정규직교기저)해 floor 추정을 Monte-Carlo 대비 수 자릿수 가속.
9. 결과: pre-pinning은 floor를 최소 2자릿수↓, trellis-pruning은 bit-pinning 대비 FER 10^-6에서 0.4dB↑; post-pinning은 rate loss 0.04dB로 시뮬 범위 밖까지 floor 제거, G-LDPC는 waterfall 성능 손실 없이 floor 대폭 개선.
10. 한계: 소프트웨어 시뮬 중심(baseline SPA만 FPGA), PR ISI채널·BCJR 전제, pinning은 dominant trapping set 사전 열거 필요·외부 BCH/rate loss 수반, iteration 감소 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping set pinning(VN LLR을 max로 고정) | `decoder.cpp` `LDPC_Decoder()`, `VN_Cal_HD()` 등 | 특정 VN LLR을 최대 신뢰로 강제하는 post-processing으로 이터레이션 루프에 삽입 가능 |
| dominant trapping set 열거 / 수렴실패 추적 | `corner.cpp` (수렴실패 추적), `ecc_top.cpp` `Load_PCM()` | 우리 H-matrix의 trapping set 목록을 먼저 확보해야 pinning 대상 비트 선정 가능 |
| G-LDPC GCP(SPC 묶음 SISO) | `decoder.cpp` `CNU_Update_New_Mag()` (CN 처리) | CN 처리를 일반화 제약(GCP)으로 확장하는 구조 변경 — 이식 난이도 높음 |
| 외부 BCH post-pinning | 대응 없음 | Prime ECC는 단일 LDPC 엔진, 외부 BCH concatenation 계층 부재 |
| PR trellis-pruning / BCJR 검출기 | 대응 없음 | NAND는 memoryless라 ISI trellis pruning 무관 |

적용 가치: 중간 — trapping-set 타깃 bit-pinning은 error-floor 개선이라는 우리 관심 영역에 직결되고 decoder.cpp 루프에 삽입 가능하나, dominant trapping set 사전 열거·외부 BCH·PR채널 요소는 NAND 환경에서 제거/재설계가 필요.
