### B. 알고리즘 요약

1. 대상: binary/non-binary QC-LDPC 두 큰 class를 대수적으로 구성; binary 예로 (4032,3708) rate0.92, (65280,61201) rate0.9375 등 고rate 코드.
2. 문제: PEG/ACE 등 컴퓨터 탐색 대신 systematic·QC 구조를 가지면서도 pseudo-random 코드에 필적/우세한 부호를 원함.
3. 핵심구성: 유한체 `GF(q^m)`의 두 additive subgroup `G1,G2`로 `α-multiplied RD-constrained` base matrix `W_add,c,n`을 만들고(식4,5), 각 원소를 `(q^m-1)×(q^m-1)` binary CPM으로 array dispersion → RC-constraint(girth≥6) 자동 만족.
4. 핵심정리: Theorem2 — `W_add,c,n`이 RD-constraint를 만족 → dispersion 결과 `H_b`가 RC-constraint(4-cycle 없음) 보장; 최소거리 하한 `≥γ_min+1`.
5. Masking: 저밀도 `Z(γ,ρ)`를 곱해 CPM 일부를 0으로 만들어 더 sparse·큰 girth의 irregular/regular QC-LDPC 생성.
6. 부호성능: (4032,3708) QC가 PEG pseudo-random보다 0.2dB 우세; (65280,32640) rate1/2 irregular가 DVB-S2 BCH+LDPC 대비 0.15dB 우세, threshold에서 0.5dB.
7. Error-floor: irregular (4032,3276) 코드가 FPGA scaled min-sum 디코더로 BER 1e-11·BLER 4e-9까지 error-floor 없음 확인.
8. Full-array subclass: 큰 column weight(예 63)로 최소거리 큼(≥64), OSMLGD로 31오류까지 정정, SPA 5~10 iter로 빠른 수렴.
9. 저복잡도 디코더: IRB-MLGD(binary message-passing, orthogonal check-sum+정수 reliability, 8-bit 양자화) — SPA 대비 XOR/정수덧셈만으로 SPA와 0.65dB 격차, ZF/JZSC-WBF보다 ~1dB 우세·병렬 복호 가능(Table IV 복잡도 비교).
10. 한계: 성능은 대부분 AWGN+SPA/FFT-QSPA 시뮬; length/lifting 크기가 `q^m-1`에 묶임; NAND soft-read 비용·HW throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| binary QC-LDPC H-matrix(RD/RC-constrained, masking) 구성 | `ecc_top.cpp` `Load_PCM()` | 중: 고rate binary QC-LDPC 부호원으로 검토 가능하나 circulant 크기 q^m-1이 z=32와 불일치 |
| girth≥6 / min-dist 하한 보증 구조 | 대응 없음(부호설계 이론) | error-floor 개선 근거지만 코드 대응 없음 |
| IRB-MLGD binary message-passing 복호 | `decoder.cpp` `LDPC_Decoder()` (min-sum 이터레이션 루프) | 중: min-sum 대체 저복잡도 디코더 후보이나 hard-read라 성능 0.65dB 손해 |
| scaled min-sum FPGA 디코더(error-floor 실측) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 중: 우리 min-sum과 동류, scaling factor 참고 가능 |
| 8-bit reliability 양자화 | `channel.cpp` + `ecc_data.h` `PARAM_LLR` | 낮음: soft-4bit+ 급, NAND read 비용과 불일치 |

적용 가치: **중간** — 고rate·저error-floor binary QC-LDPC라는 방향성은 Prime ECC와 정합하나, 부호 lifting 크기가 유한체 차수에 묶여 z=32 구조로의 이식·재검증 부담이 크고 성능 검증이 AWGN 중심이다.
