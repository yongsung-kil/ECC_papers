### B. 알고리즘 요약

1. 대상은 AWGN 채널의 binary LDPC(802.11n rate-1/2·5/6 blocklength 1944, Margulis 2640, rate-1/2 blocklength-648 코드군)로, flooding/LBP(layered) 대비 스케줄링을 개선한다.
2. 문제: LBP 등 순차 스케줄은 고정 순서라 trapping-set에서 특정 degree-2 check를 먼저 갱신해 오류를 강화, error-floor를 남긴다.
3. 핵심 지표는 잔차 `r(m) = |m_new − m_old|`(LLR 차의 절댓값)로, 큰 잔차 = 아직 수렴 안 된 그래프 영역.
4. RBP(Residual BP): 가장 큰 `r(m_c→v)` 메시지만 우선 갱신하는 greedy 방식. 소수 업데이트로 빠르나 "어려운" 오류에 취약.
5. NS(Node-wise Scheduling): check node 전체의 c→v 메시지를 동시 갱신, 노드 메트릭 `α_c = max r(m_c→v_b)`가 최대인 check를 동적 선택 → RBP보다 덜 greedy, 정답 도달률↑.
6. ANS(Approximate NS): 잔차 계산만 min-sum 근사(`r̃`)로 수행(실제 전파는 full sum-product 식 (1)(2) 사용) → 복잡도 대폭↓, NS와 성능 거의 동일.
7. trapping-set 극복: 오류 variable node에 연결된 induced-subgraph의 degree-1 check를 순차 갱신하여 LBP가 못 푸는 near-codeword를 해소.
8. 단블록 대책: LBP→ANS 혼합(F-LBP/ANS: 고정 ξ회 LBP 후 전환, A-LBP/ANS: unsatisfied check ≤ ζ일 때 전환)으로 non-ML undetected·myopic error 억제.
9. 구현: LC-ANS(두 최소 신뢰도 edge의 잔차 2개만 합산)로 메트릭 계산 축소, P-ANS(상위 p개 check 동시 갱신, p=81)로 부분 병렬화.
10. 결과: 802.11n 코드에서 200 LBP-iter 등가 업데이트로 flooding/LBP가 못 닿는 FER 도달, Margulis 코드 error-floor 개선; 다만 HW 미설계·gatecount/throughput 수치 없음.
