### B. 알고리즘 요약

1. 대상 부호: 단부호·중간길이 LDPC 예제들 — Tanner (155,64,20), EG (255,175,17), Gallager (204,102,8), PEG (256,128,17), PG cyclic (341,205,16), BPSK/AWGN.
2. 풀려는 문제: BP 단독은 stopping/trapping set으로 고SNR에서 error-floor 발생, OSD 단독은 order `i`가 커야 해 복잡도 급증.
3. 모델: ML 복호는 수신벡터와 최대 cross-correlation `Ymax` 코드워드 탐색; BP 출력을 OSD-i 입력으로 재활용하는 하이브리드 구조.
4. 핵심 기법: BP `N` iteration 후 출력 `r'`을 그대로 넘기지 않고 conditioning한 `f'`을 OSD-i에 투입.
5. 핵심 식: conditioned 출력 `f'_m = ±r_m`(부호만 BP hard-decision으로 뒤집음)으로 `Ymax=Σ|r_j|`를 원 수신벡터와 동일하게 유지(식 5·6).
6. 확장: `f'` 기반 재인코딩 코드워드 `c'`의 cross-correlation을 OSD-i 탐색의 lower bound로 써 탐색크기 축소.
7. 구현 디테일: 신뢰도순 permutation + LRB부터 Gaussian elimination으로 full-rank generator matrix 구성(비독립 bit은 skip), MRB/LRB 재정렬.
8. 분석: 표준 soft 출력은 반복될수록 LRB↔MRB bit swap으로 MRB error 증가(Tanner code 4dB, 106 벡터 실험); conditioning은 부호만 바꿔 이 열화를 방지하고 MRB error 수를 최소화.
9. 결과: 제안 decoder+OSD-1이 OSD-2 단독보다 우수하며 모든 예제에서 error-floor 제거, OSD-1 단독 대비 3dB 이상서 1.5dB 이득(Gallager).
10. 한계: 시뮬만, HW 미설계, OSD 자체가 full-precision soft·Gaussian elimination 요구라 HW 비용 큼, iteration 감소 아님(BP 후 추가 후처리로 latency 악화).
