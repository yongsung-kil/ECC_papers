### B. 알고리즘 요약

1. 목표: 3GPP-HSDPA, DVB-SH, 802.16e, 802.11n의 Turbo·LDPC를 모두 지원하는 단일코어 flexible 디코더를 ASIC로 설계.
2. 문제: 기존 IP 다중집적/이중코어/ASIP 방식은 면적·전력·throughput 중 하나 이상에서 열위(특히 ASIP는 클록 상승으로 전력 손해).
3. 핵심: `BCJR` 알고리즘을 공통 복호엔진으로 채택 — Turbo 복호와 LDPC의 `TDMP`(layered) 복호를 동일 SISO 하드웨어로 공유.
4. LDPC를 2-state trellis check-node 제약으로 보고 BCJR로 처리, check node degree ≤22라 sliding-window 없이 수행 가능(8 check-node 병렬).
5. Turbo는 codeword 길이가 커 sliding-window 필수, `TS-SP`/`BM-SP`/`BM-DP` 세 스케줄 비교.
6. 메모리 최소화 위해 W<150 구간에서 `TS-SP`(single-port, 3배 느리지만 메모리 최소) 선택.
7. 아키텍처: N개 data-path(각 BCJR SISO), inter-trellis(N codeword 병렬) 또는 intra-trellis 모드; N=12로 전 표준 throughput 충족.
8. 유연성: State-Metric Array(8 SMU)와 Extrinsic Processor를 max/max* 트리 + 순열망(permutation network)으로 표준별 재구성.
9. 결과: 45nm, 150MHz, 0.9mm2, 86.1mW(메모리 63%), FER=10^-4에서 IL<0.25dB; ASIP [10] 대비 전력 57% 절감.
10. 한계: NAND용이 아닌 무선/위성 통신 표준 타깃, 순수 HW 아키텍처 논문(부호설계·정정능력 개선 없음), min-sum 아닌 BCJR 기반.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BCJR/TDMP layered LDPC 복호 | 대응 없음 | Prime ECC는 min-sum 기반 — BCJR trellis 복호와 알고리즘 상이 |
| 순열망(circulant cyclic rotation) | `GT.cpp` `Make_GT_HW()` | 회전/MUX 라우팅 개념만 유사, 구조 이질 |
| N data-path 병렬(inter/intra-trellis) | 대응 없음 | z=32 row-병렬 구조와 정합 안 됨 |
| SISO extrinsic 갱신 | `decoder.cpp` `LDPC_Decoder()` | 역할만 대응, max/max* 트리는 min-sum과 무관 |

적용 가치: **낮음** — 무선통신 다중표준 Turbo/LDPC 공용 BCJR ASIC로, NAND binary QC-LDPC min-sum 엔진과 알고리즘·부호·타깃이 모두 달라 떼어 쓸 요소가 없음.
