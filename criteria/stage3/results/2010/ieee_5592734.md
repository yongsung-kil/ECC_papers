### B. 알고리즘 요약

1. 대상은 IRA(irregular repeat-accumulate) 계열 systematic 부호, rate 1/2, length 10,000, sum-product(BP) 복호, AWGN 채널.
2. 문제: threshold 최적화된 IRA는 accumulator가 요구하는 다수 weight-2 parity node 때문에 고SNR error floor가 나쁨.
3. 모델: repetition→interleaver Π→combiner→inner code(accumulator) 직렬연결, Tanner graph상 systematic edge/parity edge 구분, parity edge degree `λ_p(x)=ρ_p(x)=ηx+(1-η)x²`.
4. 핵심 기법: inner code(accumulator)를 일반화 — 출력의 `3η/(2+η)`는 과거입력 2개 함수, `(2-2η)/(2+η)`는 3개 함수로 하는 GIRA(generalized IRA).
5. 식 의미: 파라미터 `η`가 weight-2 parity node 비율을 조절 → η 낮추면 degree-2 node 감소로 error floor 개선, threshold는 소폭 하락(trade-off).
6. 확장: (η,s)-GIRA — inner conv code의 `1/(1+D+D^(ψ+1))`에서 `D^(ψ+1)`를 (s+1)번째 출력마다 on/off 스위칭, weight-3 parity node를 간격 s로 분산.
7. 부수: 이로써 inner code 최소 cycle 길이가 `6+4s`로 증가(6-cycle 회피), 인코딩은 길이 ψ shift register + parity당 1 mod-2 덧셈으로 저복잡도 유지. IRA=(1,0)-GIRA, w3IRA=(η,0)-GIRA로 재해석.
8. 최적화: rate r·η 고정 후 differential evolution으로 stability 조건(Lemma 1) 만족하는 λ,ρ degree 분포를 threshold 최대화로 탐색(예: (0.772,1)-GIRA는 capacity 0.244dB 이내).
9. 결과(Fig.4): rate-1/2 length-10k 랜덤 인터리버 시뮬 — IRA 대비 λ2=0/w3IRA/(η,s)-GIRA 순으로 error floor 개선, s=1은 SNR 2dB서 8×10^7 블록 중 20오류만 검출(BER~10^-9대).
10. 한계: AWGN 시뮬만·HW 미설계, IRA 특화(accumulator), turbo 복호 시 BCJR 복잡도 급증(sum-product로만 저복잡), interleaver/외부코드 개선은 별도.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sum-product(BP) 복호 | decoder.cpp LDPC_Decoder() | Prime ECC는 min-sum 근사 사용, full sum-product는 미보유(대응 없음에 가까움) |
| error-floor 겨냥 부호구조(weight-2 node 감소, cycle length↑) | ecc_top.cpp Load_PCM() | H-matrix 설계 철학은 참고 가능하나 QC 구조 아닌 IRA accumulator 특화, 직접 이식 불가 |
| IRA accumulator 인코딩 | encoder.cpp (dual-diagonal) | Prime ECC dual-diagonal 인코더와 개념 유사하나 GIRA 스위칭 구조는 별개 |
```

적용 가치: **낮음** — error-floor를 weight-2 node 감소·cycle 확대로 잡는 관점은 참고되나, IRA/RA 특화 부호설계라 Prime ECC 고rate QC-LDPC + min-sum 파이프라인에 직접 이식은 어려움.
