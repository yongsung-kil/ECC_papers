### B. 알고리즘 요약

1. 시스템: AWGN 채널, 이진 선형부호(예제는 tail-biting convolutional code `(12,6,3)`, memory 1)를 임계값 복호.
2. 문제: 반복 복호는 soft 정보 교환에 부동소수 연산이 필요해 고속·저전력 구현이 어려움 → 아날로그 복호로 power/speed·면적 개선 시도.
3. 채널 모델: antipodal 변조, `L_int(c_i)=L_ch(y_i)=2y_i/σ^2` 로 intrinsic L-value 정의.
4. 핵심 기법: 각 코드심볼의 L-value를 반복 갱신 `L^(t+1)(c_i)=L_int(c_i)+w0·Σ_{j∈A_i} e_ij^(t)`, extrinsic `e_ij`는 tanh 곱의 로그(BP식 CN 연산).
5. 핵심 매핑: `v_m=tanh(L(c_m)/2)`로 두면 임계값 복호식이 고차 재귀신경망의 운동방정식과 일치.
6. 확장: 신경망 뉴런 수 N = 코드워드 길이 n, 뉴런 내부상태 = L-value, 증폭기 비선형은 `tanh(./2)`.
7. 구현 디테일: 뉴런 미분방정식 `C·dL/dt = -L/R + T·L_int + w0·T·Σ e_ij`, Euler 근사로 이산화(Δt→0이면 아날로그).
8. 최적화: 가중치 `w0`를 스윕해 최적값 선택(예제서 w0=1이 최적).
9. 결과: 아날로그 복호 성능이 디지털 대응 및 block ML 복호와 거의 동일한 BER (Fig.6).
10. 한계: 소형 부호 시뮬만, 실제 아날로그 칩·긴 부호 미검증, 긴 부호는 뉴런 수 증가로 복잡도 급증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 반복 임계값 복호 알고리즘 | 대응 없음 | Prime ECC는 min-sum 기반 BP 복호, 임계값(majority-logic) 복호 아님 |
| tanh 곱 extrinsic 연산 | 대응 없음 | Prime ECC는 full-tanh Sum-Product이 아닌 min-sum 근사 사용 |
| 아날로그 신경망 뉴런 구조 | 대응 없음 | 디지털 bit-exact 시뮬레이터로 아날로그 회로 개념과 무관 |

적용 가치: **낮음** — 대상 부호가 tail-biting convolutional code이고 복호도 아날로그 신경망 기반 임계값 복호라 NAND binary QC-LDPC 디지털 HW에 이식할 요소가 없음.
