### B. 알고리즘 요약

1. 대상은 BSC 상 SPA로 복호하는 regular column-weight-3 LDPC(예: `(530,373)`, `(810,569)`, `(3165,2554)`, rate `0.7~0.8`, girth 8) 저복잡도 부호.
2. 문제: 유한길이에서 girth만 키우면 rate 손실이 크고, error-floor를 지배하는 가장 유해한 trapping set을 girth로는 제거 못함.
3. 핵심 가정: all-zero codeword 전송·출력대칭 채널에서 fixed set이 error-floor 원인, Gallager A/B의 실패 패턴이 SPA 실패와 위상적으로 연관.
4. 핵심 기법 1단: Gallager A/B용 Trapping Set Ontology(TSO)로 `(a,b)` trapping set과 successor/parent 위상관계 DB화, Tanner graph에서 작은 trapping set을 확장해 큰 것을 탐색.
5. 핵심 식: 상대유해도를 성공복호율 `χ1(ε), χ2(ε)`(식 (2),(3)) 차이로 정량화 — `χ2-χ1` 클수록 해당 successor가 유해.
6. 핵심 기법 2단: `(4,4)`에서 파생되는 `(6,2){1}`, `(8,2)`, `(5,3){2}` 등을 유해도 순으로 판정하고, 가장 유해한 subgraph를 금지제약 `τ`로 지정.
7. 구현 디테일: Latin square/quasigroup(`GF(q)`) 기반 permutation array `H=f̄(U)`를 PEG류 progressive check-and-select-or-disregard로 girth·금지 subgraph 조건 만족하게 구성.
8. 최적화 절차: test Tanner graph에 특정 subgraph를 포함/배제시켜 복호 실패를 반복 실험, 유해 trapping set 목록 확정 후 부호 구성.
9. 결과: `(6,2)`·`(8,2)` 회피 부호가 동일 파라미터 PEG/lattice/array-masking QC 부호 대비 error-floor를 낮춤(FER 곡선), weight-4 오류 보장정정 충분조건(Conjecture 1) 제시.
10. 한계: HW 미설계, SPA는 유해도 추론이 Gallager A/B 기반(미증명 가정), BSC hard-decision 채널, column-weight-3 저rate 특화.
