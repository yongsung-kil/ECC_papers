### B. 알고리즘 요약

1. 대상: (J,2J) regular protograph 기반 terminated LDPC convolutional code (LDPC-CC), 대표 파라미터 `ms=2, J=3, K=6, L=20`, rate `RL=0.45`, `M=512` 확장으로 `n=20480`, girth 12.
2. 문제: 상관 erasure 채널(단일버스트 SBC, Gilbert-Elliott GEC)에서 burst 정정력을 높이면서 BEC 성능도 유지하는 부호 설계.
3. 모델: protograph base matrix `B`를 두 다항식 `p(x),q(x)`로 표현(`(2i+1)`열↔`x^i p(x)`), copy-and-permute로 대형 Tanner 그래프 생성, terminated 시 QC block code로도 볼 수 있음.
4. 핵심 기법: protograph stopping set이 담기는 최소 연속열 수 `sBmin`을 정의하고, MTBL(최대허용 burst 길이) `Δmax`를 `sBmin`으로 bound.
5. 핵심 식: `Δmax ≥ M(sBmin−2)+1`, `Δmax ≤ M·sBmin−1` - stopping set 구조로 burst 정정 한계를 직접 규정.
6. 구성규칙(Prop.1): `p(x)=(J−1)+x^ms`, `q(x)=(J−1)+x`로 `sBmin=2ms` 달성, memory `ms`에 선형 증가. Prop.2로 `sBmin≤2ms` 상한(최대) 증명.
7. 디코더 2종: BP(SPA, 전체 codeword 수신 후 복호→고latency), Windowed Decoder(WD, `W·J`행×`W·K`열 sliding window로 연속복호, latency는 BP의 `w=W/L`배).
8. 채널: BEC(독립 erasure ε), SBC(단일 burst), GEC(good/bad 2상태 Markov, `ε=b/(b+g)`, `Δ=1/g`), 성능척도는 Singleton bound(MDS) 대비 CER/SER.
9. 결과: 코드 A/B의 MTBL 각 1023/1751 (B가 >70% 증가, rate 희생 없음), 작은 window(W=3,5)에서 B가 A보다 우수, (J,2J) LDPC-CC는 MDS의 MTBL 도달 불가 증명.
10. 한계: HW 미설계·erasure 채널 시뮬만, convolutional 구조라 Singleton bound와 큰 gap(far-from-optimal), AWGN/soft-decision 미고려.
