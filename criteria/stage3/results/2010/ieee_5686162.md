### B. 알고리즘 요약

1. 대상: 이진 선형블록/LDPC 코드의 ML 연판정 복호. (256,128,17) PEG-LDPC, (104,52,20) LQR 코드로 검증.
2. 문제: OSD-i는 order i가 커질수록 후보 codeword `Σ C(k,j)`가 폭증 — 탐색비용을 줄이면서 near-ML 달성 필요.
3. 모델: 정렬 수신벡터 `ỹ`에서 (n-k) less-reliable basis=parity subtree, k more-reliable basis(MRB)=information subtree로 분할한 bi-directional code tree.
4. 핵심기법: BESCT — 각 codeword를 tree branch로 보고 ensemble branch `B_i^en = g_i`(생성행렬 행)의 XOR로 branch를 한 스텝에 성장(`B_{i+1}=B_i ⊕ B_j^en`).
5. 핵심식: cost `cst(B_i)=Σ(2b_{i,j}-1)ỹ_j`를 `Mag(y)=Σ|y_j|`에 가장 가깝게 만드는 branch 탐색 — cross-correlation 최대화가 ML 판정.
6. 확장: 두 adaptive threshold — cost threshold(현 최고 cost 갱신)와 direction threshold `δ(Mag,cst)`로 growing-forward(V+)/backward(V-) 방향 제어.
7. 구현 디테일: MRB 서브트리 내로 탐색 제한, ensemble branch 사전계산으로 불필요한 branch 확장 회피.
8. 최적화: 방향 임계 비교(`δ ≥ 2|ỹ_{k-j-1}|`)로 V+/V- 전환, 잠재 codeword 집합 F에서 최대 cost branch 선정.
9. 결과: PEG-LDPC(256,128,17)에서 FER<1e-4시 OSD-3 대비 2dB+, OSD-1 대비 4dB 이득. 정보길이<30·오류<4일 때 OSD 대비 복잡도 10배↓.
10. 한계: HW 미설계, 시뮬만, 정보길이/오류수 커지면 복잡도 폭증하여 고rate 장코드에 부적합.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BESCT code-tree 탐색복호 | 대응 없음 | OSD/Dorsch 계열 탐색복호로 min-sum 반복복호와 근본적으로 다름, 이식 불가 |
| adaptive cost/direction threshold | 대응 없음 | Prime ECC의 iteration별 LLR threshold와 개념·목적 모두 상이 |
| ensemble branch(생성행렬 기반) | 대응 없음 | 생성행렬 기반 codeword 열거, Prime ECC는 PCM 기반 LLR message passing |

적용 가치: **낮음** — OSD 계열 near-ML 탐색복호는 복잡도가 실시간 NAND HW와 양립 불가하고, min-sum 파이프라인/LLR 테이블에 이식할 알고리즘 요소가 없음.
