### B. 알고리즘 요약

1. 대상: 고rate FG-LDPC(EG-LDPC) 소블록, 예시 (255,175) Type-I 2-D EG-LDPC, BI-AWGN/BPSK 채널.
2. 문제: IMP(BP)는 소블록·중고 SNR에서 성능 부족, 표준 내부점 복호(Wadayama [6])는 FG-LDPC의 무거운 행가중치(`wr`) 탓에 제약 수 `M·2^(wr-1)`로 계산 불가능.
3. LP 완화: ML 복호 `min γ^T c s.t. c∈C`를 fundamental polytope `P`(parity+box 제약)로 완화, LLR 벡터 `γ`가 목적함수.
4. 핵심기법1: 로그배리어 `φ(x)`를 근사 `φ̃(x)`로 대체 — 체크노드 m마다 최대위반 제약 `σm`(식13) 하나만 남김(Corollary 4.3), 목적함수 평가를 `O(N·2^wr)`→`O(N·wr)`로 축소.
5. 의미: 정리 4.1(한 parity 부등식 위반 시 나머지는 strict 만족)로 위반 후보 1개만 추적하면 충분 → 브루트포스 부분집합 열거 제거.
6. 핵심기법2: gradient/Hessian도 각 m의 최대기여항만 포함해 근사(식14~18), 계산량 `O(N·wr)`로 축소; Newton centering(outer `t=μt`, inner Newton) 반복.
7. 하이브리드: IPD로 search point 생성→양자화·syndrome 확인, 실패 시 LLR `γi=log((1-xi)/xi)` 계산 후 MBF(multiple bit flipping)로 복호 이어감(Algorithm 3), outer iteration 반복.
8. 최적화: outer 최대 5, inner 최대 20, MBF 최대 20, IDBP 200 iteration으로 비교.
9. 결과: 근사 IPD가 표준 IPD와 거의 동일 성능, 표준 대비 실행시간 약 1/10; 하이브리드는 BER 10^-6에서 IDBP 대비 약 0.2 dB 이득, 고SNR에서 평균 outer iteration 1회.
10. 한계: 시뮬만(HW 미설계), 소블록 전용, Newton 기반이라 연산 무거움, FG-LDPC(non-QC) 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IPD/하이브리드 복호 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 대응은 되나 내부점(Newton) 방식이라 min-sum 루프와 알고리즘 자체가 이질적 |
| LLR/search point → soft 정보 | `channel.cpp` `Set_LLR_Th()` | search point를 LLR로 변환하는 부분만 개념적 접점, 직접 이식 어려움 |
| syndrome 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 논문은 syndrome `s=z·H^T` 기반 종료, Prime은 CRC 기반 — 개념 유사하나 구현 상이 |
| 내부점 Newton gradient/Hessian | 대응 없음 | Prime ECC에 내부점법 모듈 없음 |

적용 가치: **낮음** — 내부점(LP) 기반 복호는 min-sum HW 파이프라인의 Prime ECC 3.1과 연산 구조가 근본적으로 달라 이식 부담이 크고, 소블록 FG-LDPC 전용이라 고rate 대블록 NAND QC-LDPC에 실익이 없음.
