### B. 알고리즘 요약

1. 채널은 nonergodic Rayleigh block-fading(BF): 길이 `N`을 `n_c` fading 블록으로 나눠 블록당 독립 fading gain, rate ~1/2 (`n_c=2`) 기준.
2. 문제: 랜덤/불규칙 LDPC는 ergodic AWGN엔 우수하나 BF에선 full-diversity 미확보 → outage limit 도달 실패. blockwise MDS(Singleton 한계 달성)가 필요조건.
3. 핵심 파라미터: blockwise Hamming distance `δ`(비영 weight 블록 최소수)로 diversity order 결정, coding gain은 minimum blockwise Hamming weight로 결정.
4. ML 설계: `H=[H1 H2]`에서 `H1,H2` 모두 full-rank면 full diversity(Prop.1). 그러나 iterative decoding에선 pseudo-codeword가 `H1`/`H2` 지지집합에 국한돼 diversity 붕괴 → ML 설계 부적합(Fig.5).
5. 해법 rootcheck 도입: 어떤 비트 `x`(fading `α_i`)의 이웃 비트가 전부 다른 fading `α_j`로 전송되는 checknode → 한 블록이 erase돼도 정보비트 복구.
6. root-LDPC 구조: 정보비트를 fading별 2클래스로 나눠 전부 rootcheck에 연결, `H`에 identity 블록 2개 + all-zero 2개 + sparse(weight 2/3) 4개 배치(Fig.10). parity비트는 보호 안 함.
7. irregular 확장: nonroot checknode 연결 edge에 `(λ(x),ρ(x))` degree 분포 적용 — diversity는 rootcheck가 유지하므로 불변, coding gain만 향상.
8. 6개 evolution tree로 log-ratio density evolution 유도(Prop.8): fading pair별 정보비트 BER 수렴 여부 판정, Density Evolution Outage(DEO) 확률로 WER 하한 추정.
9. 결과: (3,6) regular root-LDPC가 block-length 무관하게 outage 근접, irregular rate-1/2 예제는 outage limit로부터 2dB 이내(BF scale상 ergodic 0.2dB 상당), parallel turbo와 대등.
10. 한계: HW 미설계, 무선 BF 채널 전용, parity비트는 diversity 1(불균등 보호), ML은 비실용(genie-aided로만 시뮬).
