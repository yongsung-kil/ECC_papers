### B. 알고리즘 요약

1. 대상: medium/short length LDPC (예: (1024,512,3,6) regular PEG-LDPC), BP 후단에 OSD(order-3) 재처리를 캐스케이드한 soft-decision ML 복호.
2. 문제: 표준 OSD는 `k` MRIPs 내 bit-flip을 등확률로 수행 → order-3에서 후보 22,370,049개로 재부호화·상관불일치 계산이 폭증.
3. 모델: 수신 LLR `r`에서 hard-decision `z` 획득, 상관불일치 `λ(r,c)=Σ_{l∈D1(c)} r_l` 최소화하는 error pattern `e'`로 `c'=e'+z`가 ML 부호어.
4. 핵심 기법: 심볼 신뢰도 기반 **부등확률** error pattern 선택 - 신뢰도 높은 상위 `t`개 MRIP는 flip 대상에서 제외, 나머지 `k-t`개는 신뢰도로 그룹화.
5. 의미: 신뢰도 높은 비트는 오류일 확률이 낮으므로 flip 후보에서 빼면 재처리 후보수가 급감.
6. 구성 예: `k=512`, `t=128` 고정, 나머지 384개를 B1(128)·B2(256)으로 분할, weight 3 error pattern을 (B1 1비트 + B2 2비트) 및 (B2 3비트)로 제한.
7. MRIP 획득은 수신 심볼 순서로 `G` 열 치환 후 elementary row operation(Gaussian elimination)으로 독립 `k`열 식별.
8. `t`는 채널상태(SNR)에 따라 조정 - SNR 높으면 크게, 낮으면 작게. threshold 방식도 가능.
9. 결과: (1024,512) 코드, AWGN/BPSK, BP 최대 50 iteration + order-3 OSD에서 후보 7,072,641개로 **68.38% 감소**, FER=1E-3에서 성능저하 0.1dB.
10. 한계: HW 미설계·시뮬만, ML급 재처리(고정밀 LLR·재부호화·행연산) 자체가 고비용, medium/short 코드 전제.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| OSD 재처리(ML) / error pattern 생성 | 대응 없음 | Prime ECC는 min-sum 반복복호 기반, OSD 후처리 모듈 없음 |
| 심볼 신뢰도 기반 후보 축소 | decoder.cpp Get_VNU_Table_Idx() (개념적 LLR 활용만) | 신뢰도 활용 개념만 유사, 알고리즘 구조는 이질적 |

```

마지막 적용 가치: **낮음** - BP 후단 OSD(ML) 재처리는 고정밀 soft-LLR·Gaussian elimination·재부호화 기반으로 NAND용 binary QC-LDPC min-sum HW 파이프라인과 구조적으로 맞지 않고, Prime ECC에 대응 모듈이 없음.
