### B. 알고리즘 요약

1. 대상은 Polar code (`x = uG_N`, `G_N = B_N G_2^⊗n`), block length `N=2^n`, rate R은 정보비트 index set `I` 선택으로 결정.
2. 문제: Polar code의 parity-check matrix `H`는 high-density라 Feldman의 fundamental polytope `Q̄` 위 LP 복호가 blocklength 증가 시 오류확률 1로 실패(Lemma 2).
3. 모델: B-DMC(BEC/BSC/BAWGNC), LLR `γ_i(y)=ln P(y|0)/P(y|1)`, ML 복호를 선형목적 최소화 + parity 제약으로 정식화.
4. 핵심 기법: Arıkan의 sparse factor graph(각 check node가 최대 3변수 연결)에서 `O(N log N)` 보조변수 `a_{*,*}`로 새 polytope `P` 정의 — 각 check node local minimal polytope 교집합.
5. 식 의미: 3-neighbor check마다 `τ(s1)≤τ(s2)+τ(s3)` 등 + `Στ≤2` 부등식으로 정수 극점 보장, 병합그래프가 보조변수 공유로 제약 강화.
6. 확장: projection `P̄`가 fundamental polytope `Q̄`보다 provably tight (`P̄ ⊆ Q̄`, Theorem 1), N=4 rate 1/4 예제로 strict inclusion 입증.
7. 부수: polytope tightening — factor graph의 `T_i` 순열(n! 중 O(n) adjacent) 추가 hyperplane, lift-and-project로 codeword polytope 수렴.
8. 성질: ML-certificate property(정수해면 ML codeword 보증), BEC서 LP over P ≡ BP on sparse graph (Theorem 2) → capacity 달성.
9. 결과: N=2^10 BEC(ε=0.5)서 SC/LP over P/tightened P/MAP 비교 시뮬(Fig.3, LP trials=1000, MOSEK solver), tightened P가 MAP에 근접.
10. 한계: BSC·AWGN 등 non-BEC서 선형 목적 LP over P 불충분(min-sum 근사가 root bit서 suboptimal), HW 미설계, LDPC 아님.
