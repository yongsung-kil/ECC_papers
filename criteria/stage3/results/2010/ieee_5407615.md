> 총평: degree distribution을 threshold가 아닌 "복호 iteration 수" 기준으로 최적화해 동일 SNR에서 복잡도를 크게 줄이는 부호설계 방법론으로, iteration/latency 관점은 매력적이나 랜덤 irregular·긴 블록·SPA 가정이라 Prime ECC의 고정 고rate QC-LDPC + min-sum HW로의 직접 이식성은 낮다.

### B. 알고리즘 요약

1. 대상: 임의의 binary-input memoryless symmetric(BMS) 채널, 병렬 업데이트 sum-product 복호, rate-1/2 예제, 랜덤 구성 irregular LDPC, 블록 `n=1e5`.
2. 문제: threshold-optimized 부호는 성능은 좋으나 수렴에 과다한 iteration(=메시지 수)이 필요해 복잡도가 큼; 복잡도를 명시적으로 넣은 부호설계가 필요.
3. 핵심 척도: 복호 복잡도 `K = N·E/(Rn)`로 iteration 수 `N` × edge 수 `E`에 비례, `K = N(1-R)/(R·Σρ_i/i)`로 정리 → `N·d̄_c` 최소화 문제로 환원.
4. iteration 추정: 메시지 오류율 궤적 `p(k)=f(p(k-1))`(EXIT chart)에서 국소 기울기 `α(p)=f(p)/p`를 이용, `N ≈ ∫_{p_t}^{p_0} dp / (p·ln(p/f(p)))`로 반복횟수를 미분가능·정확하게 추정.
5. 이 적분이 왜 필요한가: `N`을 degree 분포 `λ_i`의 미분가능 함수로 표현해 연속 최적화(convex program)로 부호를 설계할 수 있게 함.
6. 확장/이론: `f(p)=Σλ_i f_i(p)`(elementary EXIT 합)로 분해; Theorem 1은 `f(p)≥e^{-2}p` 영역에서 목적함수가 `λ_i`에 대해 볼록임을 보장(위반 시 실험상 국소최소 유일).
7. 구현 디테일: 고정 `ρ(x)`(집중분포/정규 check degree) 가정, `‖λ-λ̄‖_∞<ε` 신뢰영역으로 elementary EXIT chart 정확도 유지하며 반복 재최적화; 정보비트 BER `p_b`를 node-perspective 분포로 목표 `p_t`에 매핑.
8. 최적화: threshold-optimized 분포를 초기값으로 (17)을 반복 풀어 목표 rate·복잡도에서 최소 복잡도 degree 분포 `λ(x)` 도출(예 `C150`,`C400`,`C800`).
9. 결과: `p_b=1e-6`에서 threshold-optimized 대비 iteration/복잡도 41%(C400), 56%(C800), 44%(C150) 감소; degree-2 비율 감소로 error floor도 나타나지 않음.
10. 한계: HW 미설계, 랜덤 비구조 irregular·긴 블록·SPA(tanh) 가정, 짧은 블록에선 이득이 threshold-optimized와 유사(DE 가정 붕괴).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| degree distribution 최적화(부호구조 설계) | ecc_top.cpp Load_PCM() | Prime ECC는 고정 QC-LDPC H-matrix라 degree 분포 이식은 H-matrix 재설계·재검증 필요(부담 큼) |
| iteration 수 최소화(복호 수렴 가속) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 목표(iteration/latency 감소)는 일치하나, 본 방법은 부호설계로 달성하며 SPA 가정 |
| 조기종료/iteration 예산 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 복잡도=iteration 관점이 Prime ECC의 조기종료 스케줄과 개념적으로만 연관 |
| sum-product(tanh) CN 연산 | decoder.cpp CNU_Update_New_Mag() (min-sum) | 본 논문은 full-tanh SPA 기반 → Prime ECC min-sum HW와 연산 모델 불일치(대응 부분적) |
```

적용 가치: **낮음** — "iteration 최소화 부호설계"라는 목표는 NAND latency와 맞닿아 매력적이지만, 랜덤 irregular·긴 블록·SPA 가정의 이론/시뮬 방법론이라 Prime ECC의 고정 고rate QC-LDPC + min-sum HW에 직접 떼어 쓰기 어렵고, 시사점(고degree 집중·degree-2 축소로 iteration↓·error-floor 회피)만 참고 가치가 있다.
