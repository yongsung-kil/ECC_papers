### B. 알고리즘 요약

1. 대상은 AWGN 채널의 (3,6)-regular random binary LDPC 코드(rate 1/2, length 20~504)의 LP decoding.
2. 문제: Feldman의 일반 LP 복호는 제약/변수 수가 방대해 실용적으로 풀기 어렵다(fundamental polytope `P` 위 `min γ^T x`).
3. 모델: ML 복호를 LP로 완화, polytope `P = ∩ conv(C_j)`의 vertex가 pseudocodeword; 정수 vertex면 ML certificate.
4. 핵심 기법 1: dual LP를 재구성해 constraint가 'block angular' 구조를 갖게 하고 revised simplex method 적용(역행렬 명시계산 회피).
5. 핵심 기법 2: Dantzig-Wolfe 분해 - master problem 1개 + check node당 subproblem `m`개, 각 subproblem은 convex polyhedral cone(`K_j = {y: D_j^T y ≤ 0}`, "local parity-check cone") 위 LP.
6. 각 iteration에서 master가 dual vector `x`를 주면 subproblem이 reduced cost `(-b_j - A_j x)^T v_l^j`로 새 column(extreme ray) 생성 → column generation.
7. 부수 트릭: local parity-check cone의 extreme ray를 대칭성(Theorem 10)으로 미리 열거(`ρ≥4`면 `2^ρ + 2^{ρ-1}`개)하여 subproblem을 실제로 풀지 않고 reduced cost 계산.
8. 저장: 같은 degree check는 extreme ray 집합 공유, regular 코드는 1그룹만 저장 → 저장량 `≈ n×n`(revised simplex `M×M`의 8.2%).
9. 결과: (3,6)-regular 코드에서 D-W 분해가 revised simplex 대비 iteration 4~5배 감소, 저장 8.2%, 연산비 `r_C < 1`; SNR↑ 시 integral 출력 많아 iteration 감소.
10. 한계: HW 미설계, 정정성능은 표준 LP 출력과 동일(개선 아님), tanh 없는 단순연산이나 복잡도는 여전히 code length 비선형.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP decoding (simplex + D-W 분해) 복호 루프 | 대응 없음 | LP-relaxation 복호로 min-sum 메시지패싱(`LDPC_Decoder()`)과 이질적 알고리즘 |
| local parity-check cone / extreme ray 열거 | 대응 없음 | polytope/cone 기반 최적화, Prime ECC의 CN/VN 연산에 대응 레이어 없음 |
| ML certificate(정수 vertex) 조기종료 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | 개념적 조기종료 유사하나 판정 메커니즘(정수성 vs CRC)이 달라 재사용 불가 |

적용 가치: **낮음** — LP 복호는 NAND용 min-sum QC-LDPC 디코더와 근본적으로 다른 패러다임이고 HW 미설계·성능개선 부재로 Prime ECC 이식 가치가 낮다.
