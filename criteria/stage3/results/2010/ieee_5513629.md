### B. 알고리즘 요약

1. 대상: 이진 선형코드 및 짧은 LDPC — rate 1/2, `n=100`(random/(3,6)/(5,10)), (3,6) LDPC `n=504/1008`(PEG), AWGN·antipodal.
2. 문제: 짧은 블록에서 표준 SP는 ML과 격차·error-floor; 기존 조작채널 반복복호[7-9]는 긴 LDPC엔 부적합.
3. 모델: parity-check H를 row echelon form으로 만들어 각 행의 incremental support `SΔ(hi)=supp(hi)\∪_{j>i}SΔ(hj)`로 code tree 구성.
4. 핵심기법: 여러 code tree(신뢰도 기반 열치환으로 생성)를 독립적으로 sweep 탐색; 노드 score = 부분코드워드 log-likelihood `log p(y|x)`, 동일 depth 노드끼리만 비교.
5. 핵심식: 트리 생성용 위치확률 `p(l)=γe^{|α log p(yl|xl=0)/p(yl|xl=1)|}` (3) — 고신뢰 비트를 트리 뿌리 근처에 배치해 분기 축소.
6. 확장(LDPC용): 희소성 보존 row echelon화(마지막 m행은 H0 sparsity 유지), 사전계산 트리 중 고신뢰 비트가 뿌리 근처인 것 선택, score는 완화코드 상한 `max_{x∈C':xI=vI} log p(y|x)` (8).
7. 부수 트릭: multitree-aided SP — SP 실패 시 일부 비트를 tree search 결정으로 고정/flip 후 채널정보 교체해 SP 재시작; 최단순형은 single-bit-flip(고신뢰인데 오판 의심 비트 1개 고정).
8. 절차: unsatisfied check를 loglikelihood로 정렬해 각 check에서 least-reliable 2비트를 후보로, 최대 시도수까지; multitree-aided는 depth-limited(m) 트리 다수로 부분코드워드 리스트 생성.
9. 결과: `n=100` random/LDPC에서 다중트리(10트리×200 sweep)로 near-ML WER 달성; (3,6) `n=504/1008`에서 SP(50 iter) 대비 single-bit-flip 재시도(최대 100~200 위치)만으로도 WER 대폭 개선, multitree-aided가 추가 이득.
10. 한계: HW 미설계·시뮬만, 짧은 저rate 코드·실수 LLR 전제, 다중 트리 저장(메모리↑)·다중 SP 재시도(평균/최악 복잡도·latency↑).
