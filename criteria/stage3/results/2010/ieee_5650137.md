### B. 알고리즘 요약

1. 대상은 `(J,K,L)` regular QC-LDPC (L=소수, circulant shift 기반), 부분병렬 디코더로 block row당 memory bank 1개 할당해 접근충돌 회피.
2. 문제: 전통 MSA 디코더는 check-to-variable(`r`)와 variable-to-check(`q`) 메시지를 별도 메모리에 모두 저장해 면적이 큼.
3. 채널/성능 모델은 다루지 않고, 순수 메모리 구조·복원 정확도만 논함 (BER 무관).
4. 핵심: 한 check node의 모든 `L(rji)`가 min1, min2, min1_index, 각 edge 부호비트로 결정된다는 min-sum 성질을 이용해 이를 **efficient chunk** 하나로 압축 저장.
5. chunk = `{min1, min2, min1 index, 부호비트열, 부호비트 XOR합}` — variable-to-check 메시지는 저장하지 않음.
6. **predictor**: VNFU가 `L(qij)`를 갱신할 때 2's-SM으로 부호/크기 분리 후 `βij`를 min1/min2와 비교해 memory2의 chunk를 즉시(다음 iteration용) 갱신 (식 11).
7. **recover unit**: 복호(column) 과정에서 chunk를 읽어 해당 edge가 min1_index면 min2, 아니면 min1을 MUX로 출력하고 XOR로 부호 복원 후 2's-complement 변환해 `L(rji)` 재생성.
8. Normalized MSA(`k=0.75`) 정규화 팩터를 복원 식에 반영해 MSA 대비 성능 손실(≈0.5dB) 보완.
9. 결과: 메모리 비트수 (4,36,128) 기준 전통 258048 → 제안 91132 (약 65% 절감), [6] 대비 약 56% 절감. gatecount/throughput/BER 수치는 없음.
10. 한계: 실 합성·FPGA·BER 검증 없이 메모리 비트 카운트 분석에 그치며, predictor의 순차 갱신 의존성과 정정성능 등가 여부 미검증.
