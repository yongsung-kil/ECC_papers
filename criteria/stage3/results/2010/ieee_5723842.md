### B. 알고리즘 요약

1. 시스템: partially-parallel LDPC 복호기용 구조화 QC-LDPC 행렬 구성, 예제는 `1/2` rate `(3,6)` regular 1296-bit(WLAN).
2. 문제: 비구조 랜덤 행렬은 BER 좋으나 partially-parallel HW 구현이 복잡(인터커넥트 저장 메모리 필요), 순수 QC는 girth 제약으로 성능 열세.
3. 모델: BPSK/AWGN, Min-Sum 복호 4-bit 양자화, 최대 iteration 10.
4. 핵심 기법: 3-Level HQC 계층 구성 — Level-1 Core matrix(rate/regularity 유지), Level-2 순환시프트 identity `L`(크기 N)에 순환시프트 Permuted matrix `Rx`(크기 R) 배열, Level-3 Base matrix `I`(크기 P, 병렬화 인자).
5. 핵심 아이디어: Core matrix 각 행(layer)마다 서로 다른 Permuted matrix 조합 삽입해 "가상 랜덤성" 부여 → 랜덤 행렬에 근접한 성능.
6. 확장: N/R/P 파라미터 조정으로 다양한 코드길이/rate 생성(WiMAX P=16, WLAN P=18, DVB-S2 P=27).
7. 구현 디테일: MatLab 소프트웨어 모델로 구성, girth>4 필터링 후 BER/FER/avg iter 시뮬로 선별.
8. 최적화: girth<4 행렬 제거하고 permuted 구조·순환시프트 변경 반복해 랜덤 대비 성능 근접 행렬 저장.
9. 결과: 3L-HQC가 2L-HQC 대비 BER 10^-5서 0.3dB 우수, PEG 랜덤 대비 0.1dB 미만 손실, avg iteration은 2L-HQC보다 소폭 감소.
10. 한계: HW 미구현(시뮬만), gate count/throughput 수치 없음, multi-standard 무선 지향으로 NAND 고rate·error-floor 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3L-HQC QC-LDPC H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 QC-LDPC(base+circulant shift) 로드, 구조 개념은 유사하나 특정 H-matrix 고정 |
| Min-Sum 복호(4bit 양자화) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC 이미 min-sum 보유(중복), 단 4bit 양자화는 soft-4bit+로 Prime ECC(2~3bit) 초과 |
| 병렬화 인자(Base matrix 크기 P) | 대응 없음 | Prime ECC는 z_sb=32 고정 lifting, P 가변 구조와 정합 필요 |
| 계층별 permuted 부행렬(라우팅 단순화) | `GT.cpp` `Make_GT_HW()` (간접) | 메모리/라우팅 절감 취지는 유사하나 HQC 특유 addressing과 GT는 다른 접근 |

적용 가치: **중간** — QC-LDPC 부호설계 유연성/랜덤성 확보 아이디어는 참고 가치 있으나, Prime ECC는 특정 고rate QC-LDPC와 z=32 고정 구조라 HQC 계층 구성을 이식하려면 H-matrix 전면 재검증 필요.
