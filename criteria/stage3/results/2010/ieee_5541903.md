### B. 알고리즘 요약

1. 대상 부호: IEEE 802.16e QC-LDPC, code length `1024`, rate `1/2`, BPSK/AWGN.
2. 풀려는 문제: BP는 fixed schedule이라 수렴 느림; RBP(residual BP)는 빠르나 greediness로 새 error 생성·복잡도 높음.
3. 모델: 노드 간 메시지 `m_k`, 갱신 전후 차이의 크기로 residual `r(m_k)=||m'-m_k||` 정의(수렴 안 된 메시지 우선 갱신).
4. 핵심 기법(RBP): 최대 residual 메시지를 먼저 갱신하고 잔차 재계산·정렬(reorder) 반복.
5. 핵심 기법(VC-RBP): 잔차를 check→variable(RBP)이 아닌 `variable→check` 메시지의 갱신 전후로 계산.
6. 의미: RBP는 단일 check 식 기반 최대 잔차라 신뢰도 낮은 VN으로 먼저 전파되어 새 error 유발; VC-RBP는 모든 check 식 기반이라 덜 greedy하고 trapping set을 더 효과적으로 해소.
7. 구현 디테일: VC-RBP는 잔차 시퀀스 저장·재정렬을 생략(Algorithm 2가 line 수 감소)해 RBP 대비 복잡도 저감.
8. 학습/최적화: 없음(휴리스틱 스케줄링).
9. 결과: 10 iteration에서 VC-RBP가 RBP 대비 BER 1e-4에서 약 0.28dB 이득; operation reduction은 SNR 5dB에서 BP 대비 45.2%(RBP는 26.2%).
10. 한계: MATLAB 시뮬만, HW 미설계, informed dynamic scheduling은 순차성이 강해 병렬 디코더 이식 시 재설계 필요.
