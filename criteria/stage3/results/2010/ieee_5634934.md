### B. 알고리즘 요약

1. 대상 시스템은 N=8개 NAND 플래시(채널)로 구성된 SSD, 각 페이지는 정보 `k×b`bit + 검사 `r×b`bit(`k=2048, r=64, b=8`).
2. 문제: 고밀도 플래시는 BER이 높고 셀 소거횟수(SLC 100K/MLC 10K)가 제한되어, 쓰기가 특정 칩에 몰리면 수명이 단축된다.
3. 기존 wear-leveling은 칩 **내부** 셀 단위라 칩 간 쓰기 편중을 못 막는다는 점을 지적.
4. 핵심 기법1: 페이지 내부는 random bit error 정정 코드(BCH, `n=2108, r=60, t=5`), 페이지 간(칩 간)에는 single-erasure 정정용 XOR parity(`D_{N-1}=D_0⊕...⊕D_{N-2}` over GF(2)) 적용하는 2-level ECC.
5. 이 parity 페이지가 한 칩(페이지) 소실을 복원 → 칩 하나를 wear-leveling 시 write 생략해도 읽기 시 erasure 정정으로 복원.
6. 핵심 기법2: 소거횟수 편차(max-min)가 임계 `α`를 넘으면 최다 소거 칩에는 write를 건너뛰어 칩 간 소거를 평준화; `α`는 소거 누적에 따라 5→25로 증가(Table II).
7. 구현 디테일: 소거횟수가 1K 초과 시 wear-leveling 중단하고 정정능력을 우선 확보.
8. 학습/최적화 절차 없음(정책 파라미터 `α` 테이블만).
9. 결과: 칩 간 wear-leveling으로 소거횟수 분포 편차 감소(Fig.8), 수명은 conventional method2보다 길고 BER은 method2보다 크되 일반 storage BER `10^-13~10^-16` 범위 유지.
10. 한계: HW 미설계, 시뮬/이론 분석뿐이며 throughput·delay 미평가(future work). LDPC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 페이지 내 BCH(t=5) 정정 | 대응 없음 | Prime ECC는 QC-LDPC 전용, BCH 미대응 |
| 페이지 간 XOR parity(single erasure) | 대응 없음 | 칩 간 시스템 레벨 코드, 디코더 무관 |
| 칩 간 wear-leveling 정책(α 테이블) | 대응 없음 | FTL/컨트롤러 정책, ECC 코어 밖 |

적용 가치: **낮음** — BCH+XOR 시스템 코드 및 wear-leveling 정책 논문으로 binary QC-LDPC 부호설계/디코더/HW에 이식할 요소가 없다.
