### B. 알고리즘 요약

1. 대상: turbo-like(LDPC/Turbo) 병렬 디코더에서 `P`개 PE가 `B=P`개 memory bank에 동시 접근할 때 발생하는 collision 문제.
2. 문제: 인터리빙 법칙(permutation)에 따라 여러 PE가 같은 memory block을 동시 read/write하면 충돌 → throughput 저하. 표준 준수하려면 permutation을 못 바꿈.
3. 기존 방식 한계: node-colouring 그래프 채색은 clique 크기만큼 bank 필요(예제서 3 bank로 충분한데 5-colouring 요구), simulated-annealing 방식[18]은 종료시점 예측불가·네트워크 미최적화.
4. 핵심 아이디어: 각 데이터 `ei`의 read와 write 접근을 **분리**하여 서로 다른 bank에 배정 → clique 제약 완화.
5. 자료구조: `MAP` 행렬, 각 데이터에 (read bank `bj`, write bank `bk`) 두 셀 배정.
6. 제약: MAP의 각 column에서 read/write 각각 memory를 1회만 사용, 데이터의 첫 read는 마지막 write와 같은 bank.
7. 알고리즘: column 순차 처리, 미배정 셀마다 사용가능 bank 리스트 생성(목표 steering component 우선 정렬), 유효매핑 있으면 첫 원소 배정.
8. 충돌 해소: 매핑 불가 시 가장 가까운 과거 column의 데이터 `D` 선택 → write 교환 → 재귀 전파(swap propagation).
9. 결과: pedagogical 예제(3 PE, 6 데이터)에서 충돌없는 in-place 매핑 도출 및 인터커넥션 제어 상태머신 정보 제공, barrel-shifter 등 regular permutation 유도 시 네트워크 최적화.
10. 한계: HW 미구현, 정량 실측/합성 결과 없음, 부호설계·복호성능과 무관한 architecture 계층 방법론.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 인터리버 메모리 충돌 매핑 | 대응 없음 | Prime ECC는 QC-LDPC circulant 기반으로 별도 인터리버 permutation 계층이 없음 |
| 병렬 memory bank 라우팅 | 대응 없음 (HW 모델은 z=32 row 병렬/GT) | turbo-style 인터리버 매핑은 QC-LDPC의 barrel-shift 라우팅과 구조 상이 |

적용 가치: **낮음** — turbo/LDPC 병렬 인터리버의 memory bank 충돌 회피용 architecture 방법론으로, Prime ECC의 binary QC-LDPC 디코더/부호설계 어느 계층에도 직접 대응하지 않는다.
