### B. 알고리즘 요약

1. 대상 시스템은 자기기록(HDD) perpendicular read channel, 4KB 섹터, PW50/T=2.3, 잡음 90% 미디어 jitter + 10% AWGN, 10-tap equalizer + 3-tap noise predictor + 8-state SOVA.
2. 문제: LDPC는 soft-decision 확률 기반 복호라 긴 burst error(write-in/erasure/TA)에 매우 취약해 LDPC-only는 종종 완전 실패.
3. 핵심 가정: burst 위치만 알면 해당 soft 입력 magnitude를 0으로 erasure해 LDPC 복호 실패율을 크게 낮출 수 있다.
4. 핵심 기법: outer BCH codeword를 8개 세그먼트로 분할, 7개는 inner LDPC, 1개는 burst 내성이 강한 inner RS로 부호화하고 모든 inner codeword를 bit-by-bit interleaving.
5. 복호 흐름: 먼저 RS를 복호, 성공 시 입출력 비교로 burst 위치를 식별→interleaving 덕분에 다른 LDPC codeword의 해당 위치를 erasure.
6. 확장(erasure burst): 감쇠계수 `α`를 MMSE(직교원리 `α̂ = E[rx]/E[x²]`)로 추정, LDPC hard-decision 출력을 원신호 `x` 근사로 사용해 재검출(redetection) 수행.
7. 구현 트릭: 이미 성공 복호된 세그먼트의 known bit로 SOVA trellis branch를 pruning해 재검출 연산량 절감, LDPC soft 출력을 a priori로 재투입.
8. 부호 스펙 예: inner LDPC는 (3,54)/(3,39)-regular QC-LDPC(circulant size 82/118, 4-cycle-free), rate 17/18(4428,4184)·12/13(4602,4250); overall rate 13/14, 8/9.
9. 결과: erasure burst에서 RS-only 대비 약 0.5~1.3 dB SER 이득, write-in burst에서도 이득; LDPC 복호는 32 internal iteration, 검출기-복호기 global iteration 없음.
10. 한계: 순수 시뮬레이션, HW/gate count 미평가(정성적 논의만), TA burst에서는 이상적 front-end 가정 시 LDPC-only가 더 우수, HDD 채널 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| burst 위치 파악 후 soft 입력 erasure | `channel.cpp` `Set_LLR_Th()`, `Set_R_Offset()` | 개념적 유사(신뢰도 0 처리)이나 HDD burst 전용, 이식 시 재설계 필요 |
| inner LDPC 복호 자체 | `decoder.cpp` `LDPC_Decoder()` | 표준 soft-decision 복호로 개선 없음(중복) |
| RS/BCH 외부·내부 부호, interleaving, SOVA 재검출 | 대응 없음 | Prime ECC는 순수 binary QC-LDPC로 concatenation/SOVA 계층 부재 |

적용 가치: **낮음** — HDD read channel용 concatenation·SOVA 재검출 시스템 논문으로, NAND binary QC-LDPC 디코더/부호에 떼어 쓸 이식 가능한 요소가 사실상 없음.
