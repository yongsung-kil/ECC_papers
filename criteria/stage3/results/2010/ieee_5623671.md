### B. 알고리즘 요약

1. 시스템: M-SC-MPC(다중 직렬연접 다중 패리티체크) LDPC의 파생 부호설계, AWGN/BPSK, SPA(LLR) 복호, block length `n=1632`, rate 1/2·2/3·3/4.
2. 문제: M-SC-MPC 행렬은 length-4 cycle은 없으나 length-6 cycle이 많아 복호기가 near-ideal로 동작하지 못함.
3. 핵심가정: 서로소·증가순 `ri` 값이면 `Σn ≤ r1r2+Σri` 조건에서 length-4 cycle 자유(식 2).
4. 핵심기법 1단: 수정 PEG를 블록단위로 실행 — 각 `ri×ri` identity 블록을 임의의 circulant permutation `Ip`로 치환(마지막 블록만 systematic 유지).
5. 핵심식 의미: circulant 치환 = 각 MPC 인코더 입력 정보비트의 순열(interleaver 삽입)과 등가, 구조(QC 유사) 보존.
6. 핵심기법 2단: 각 반복에서 length-4 cycle 발생수 `g=0`인 shift `p`만 선택해 삽입, local cycle을 6→8 이상으로 증가 → P-SC-MPC.
7. 부수 트릭: iM-SC-MPC(블록 cancellation로 irregular화)와도 비교, 열가중치 `<dv>` 낮추면서 L6(길이6 cycle 노드수) 감소.
8. 학습/최적화: PEG greedy edge 배치로 cycle 길이 최적화, C++ 구현.
9. 결과: rate별로 P-SC-MPC가 M-SC-MPC·iM-SC-MPC보다 L6 최소(예 R=1/2: 1283→999), BER에서 IEEE 802.16e QC 대비 comparable~우수, waterfall·error-floor 동시 개선.
10. 한계: HW 미설계·시뮬뿐, gatecount/throughput 없음, 저~중rate(0.5~0.75) 무선 표준 타깃, high-rate NAND 미검증.

### C. Prime ECC 관련 모듈 핀포인트

부호설계가 cycle 길이 증가로 error-floor/정정능력에 직접 기여하므로 `대상=both`.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 블록단위 수정 PEG로 circulant shift 선택 (H-matrix 구성) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 우리도 QC(base+circulant shift) 구조, cycle 최소화 shift 선택 아이디어 참고 가능하나 부호 고정으로 재검증 부담 |
| SPA(LLR) 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | 우리는 min-sum 근사 사용, 논문은 full SPA — 알고리즘 상이 |
| short-cycle 제거로 error-floor 개선 | [decoder.cpp](../Prime_ECC_3.1_Claude) (복호 성능에 반영), 부호구조는 `Load_PCM()` | girth 개선이 error-floor에 미치는 효과는 우리 부호 재설계 시 참고 |
| M-SC-MPC 직렬연접 구조 | 대응 없음 | 우리는 dual-diagonal 인코딩, SC-MPC 연접 구조 아님 |

적용 가치: **중간** — circulant permutation으로 girth를 늘리는 PEG 변형은 QC-LDPC 부호설계 관점에서 참고할 만하나, 특정 SC-MPC 직렬연접 구조에 종속되고 저~중rate·시뮬 수준이라 우리 고정 high-rate H-matrix에 직접 이식은 제한적이다.
