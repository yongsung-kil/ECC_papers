### B. 알고리즘 요약

1. col-weight-3 LDPC 4종(Tanner n=155 R=0.4, QC n=768 R=0.75 dmin=12, MacKay n=4085 R=0.82, array n=1503 R=0.668)을 BSC에서 hard-decision 복호.
2. 문제: BP는 유한길이·저잡음(error-floor) 영역에서 trapping set 때문에 급격히 열화, 기존 양자화 복호는 "BP 근사"가 목표라 BP를 넘지 못함.
3. 모델: 메시지 비트가 belief가 아니라 노드의 국소 이웃(사이클/트래핑셋) 구조를 반영하도록 설계, all-zero 전송·대칭복호 가정.
4. 핵심 기법: 2-bit 메시지 = [이웃정보 1bit | 변수값 1bit]. CN은 1비트 AND·2비트 XOR, VN은 incoming 2메시지로 국소구조 추론해 사이클 내 오류비트가 외부 correct 메시지에 기대 정정.
5. 식/규칙 의미: VN/CN 업데이트를 Boolean 진리표(Table I/II)로 정의 — '10'/'01' 같은 특정 메시지 인스턴스가 사이클 내 노드를 정상화하는 핵심 트리거.
6. 확장(3-bit): [이웃정보 2bit | 값 1bit], 8-레벨 메시지로 (5,3) trapping set 등 사이클보다 복잡한 국소구조 포착, 2-bit가 실패하는 패턴 정정.
7. 설계 절차: "trapping set ontology"(트래핑셋 계층)로 다수의 유해 부분그래프를 골라 최소 비트로 정정하는 룰을 도출, critical number 증가·최소 iteration 수렴을 기준으로 선정.
8. 이론적 근거: isolation assumption(부분그래프를 그래프에서 고립된 것으로 분석) — BP/min-sum은 dense 그래프에서 수렴이 느려 isolation이 깨지기 전 못 고치지만, 3-bit는 빠르게 수렴해 정정.
9. 결과: 4개 부호 모두 error-floor에서 3-bit가 BP·min-sum 초과(FER 곡선 slope 개선); Tanner code는 5-error 전부 보증(LP 복호도 실패하는 것), n=768 QC는 4-error 무실패.
10. 한계: HW 미설계, col-weight-3·BSC 한정, 룰이 trapping-set/부호구조 종속(범용성은 3-bit가 어느정도 확보하나 rate 중간대·soft 확장은 미래과제).
