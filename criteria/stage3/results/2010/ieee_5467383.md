### B. 알고리즘 요약

1. 시스템: 2-D 자기기록(TDMR), grain 1~수개에 1 채널비트 기록해 ~10 Tbit/in^2 목표, page 256x256 bit cell, LDPC rate `R`=27/64~45/64.
2. 문제: grain 랜덤성으로 일부 비트가 아예 기록 안 됨(write error) + spatially-varying 2-D 간섭(ISI/ITI)을 강력 ECC로 극복.
3. 채널모델: random Voronoi grain 모델을 랜덤 offset/jitter로 확장, 출력 `r_k = sum h*a + n` (spatially-varying PSF, AWGN, position jitter 포함).
4. 핵심기법1: 2-D LMMSE equalizer `(5)`로 bit cell-grain 랜덤 offset을 반영해 등화 MSE 최소화(기존 2-D ZFE 개선).
5. 핵심식: 등화 잔차 `e = z - a`를 written/unwritten 비트로 나눠 Gaussian-mixture(GM)로 근사 — unwritten은 평균 0, written은 평균 유의.
6. 핵심기법2: GM 조건부 pdf `f(z|a)`에서 LLR `(8)`을 산출, 이를 soft-input LDPC 복호 입력으로 사용(단일 Gaussian보다 실측 pdf에 근접).
7. 복호: LDPC는 기성 소프트웨어([7] R. Neal) 사용 — 부호구성·복호 알고리즘 자체는 논문 신규 기여 아님.
8. 파라미터: grain/bit 밀도 `lambda`=1.0~1.5, write error rate 0.16~0.025, 등화 후 BER 0.117~0.0525.
9. 결과: `lambda`=1.3 + rate 41/64에서 BER~10^-? 달성, 0.493 user bits/grain = 9.86 Tbit/in^2; noise/jitter 포함 시 0.45 bits/grain(~9 Tbit/in^2).
10. 한계: 실HW·gate 설계 없음(시뮬만), read/write head 한계 등 추가 impairment는 미반영; 자기기록 특화 채널.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft-input LDPC 복호 | `decoder.cpp` `LDPC_Decoder()` | 기성 SW LDPC 재사용, 알고리즘 신규성 없어 이식 요소 아님 |
| GM 잔차→LLR 산출 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | soft LLR 추출 개념은 유사하나 TDMR write-error/2-D 채널 전제라 직접 재사용 불가 |
| 2-D LMMSE 등화 | 대응 없음 | 자기기록 채널 등화, NAND에 무관 |

적용 가치: **낮음** — TDMR 자기기록 채널 특화 신호처리로, LDPC는 기성 재사용이라 Prime ECC 부호/디코더에 떼어 쓸 신규 요소가 없음.
