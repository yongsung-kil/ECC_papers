### B. 알고리즘 요약

1. Euclidean Geometry(EG) 기반 binary QC-LDPC (13299,11285), rate ~0.85 부호를 구성한다.
2. 이 부호는 min-sum(max 30 iter, AWGN)에서 BER 1e-10 부근에 error-floor가 발생한다.
3. error-floor의 원인을 trapping set으로 규정하고, `(a,b)` trapping set (a variable nodes, b odd-degree checks)을 정의한다.
4. 디코딩 실패 프레임의 오류 비트 위치로 부분 행렬 `H_z`를 뽑고, a개 열을 선택해 `H_z_TS`를 만들어 (a,b) trapping set을 탐색한다.
5. 시뮬레이션으로 이 부호가 두 종류의 고정 구조 (4,4) trapping set(총 17개 프레임 패턴)을 가짐을 확인한다 (TABLE I).
6. 이 (4,4) 패턴은 cyclic type의 고정 패턴이라 프레임당 오류가 4비트로 작다는 점에 착안한다.
7. error-floor 저감 3가지 방법(BP 알고리즘 개선, coding method 개선, concatenated code) 중 concatenation을 선택한다.
8. 외부 부호로 rate 0.99의 shortened BCH (3860,3824) (3비트 정정)를 사용한다.
9. 32개 BCH를 11개 LDPC (13299,11285)에 concatenate하는 구조로 trapping set 오류를 외부에서 정정한다.
10. 결과: 3.81dB에서 BER 7e-11, 3.84dB에서 4e-12; 확률 계산으로 error-floor를 1e-15 이하로 추정, 100Gbps 광통신 OTU4에 적용 가능하다고 주장.
11. 한계: HW 미설계, 시뮬/추정뿐이며 특정 EG 부호의 고정 패턴에 강하게 의존한다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EG QC-LDPC H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | 부호구조 로드 지점이나 Prime ECC는 고정 QC-LDPC라 EG 구성 이식 부담 큼 |
| trapping set / 수렴실패 추적 | `corner.cpp` | 수렴실패 프레임 추적 관점에서 error 패턴 분석에 참고 가능 |
| 외부 concatenation(BCH) 정정 | 대응 없음 | Prime ECC는 LDPC 단독, 외부 대수부호 concatenation 계층 부재 |
| min-sum 반복 복호 | `decoder.cpp` `LDPC_Decoder()` | 논문의 기저 디코더와 동일 min-sum이나 개선 대상 아님 |

적용 가치: **낮음** — error-floor를 외부 BCH concatenation으로 덮는 접근은 Prime ECC의 LDPC 단독 구조에 새 외부부호 계층을 요구하고, 개선이 특정 고정 trapping set 패턴에만 국한된다.
