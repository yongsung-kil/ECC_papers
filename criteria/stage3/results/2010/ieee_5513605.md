### B. 알고리즘 요약

1. base Tanner graph `G`를 `N`번 복사하고 각 edge에 순환 순열(circulant shift)을 배정해 `N`-lifting → quasi-cyclic 부호 `C̃` 생성.
2. 문제: 유한길이 LDPC의 error floor는 dominant trapping set(짧은 사이클로 구성)이 유발하므로 이를 없애야 함.
3. 채널/디코더별 dominant trapping set은 이미 알고 있다고 가정 (BSC/Gallager A/B 기준).
4. 핵심: 길이 `ℓ`인 base cycle `c`가 lifted graph에서 `N/k`개의 길이 `kℓ` 사이클로 사상됨 (Theorem 1). 여기서 `k=O(c)`는 순열지수의 위수.
5. cycle의 순열지수 `d = Σ(-1)^i d_{i+1} mod N`(식 2)이 nonzero이면 `O(c)>1`, 즉 inverse image가 원래보다 엄격히 긴 사이클만 됨 (Corollary 1).
6. Intentional Edge Swapping(IES): dominant trapping set의 모든 cycle `c`에 대해 `O(c)>1`이 되도록 최소 개수의 edge에 nonzero 순열지수를 greedy하게 배정.
7. trapping set을 critical number 오름차순 정렬 후 하나씩 처리, sequential하게 `Z_{1→N}` 값을 시험해 사이클 제거되는 인덱스 채택.
8. `N`을 키우면 알파벳 공간이 커져 더 많은 trapping set 제거 가능, degree distribution·인코더/디코더 구조·rate 보존(full-rank이면 rate 유지).
9. 결과: (155,64) Tanner code에서 `N`=2로 FER slope 3→4, `N`=5로 slope 5; (504,252) code는 `N`=6로 slope 5까지, random lifting(slope 3~4)보다 월등, BIAWGN+min-sum에서도 floor 없이 10^-8까지.
10. 한계: HW 미설계, 시뮬만, trapping set 사전 지식 필요, block length가 `N`배로 증가.
