### B. 알고리즘 요약

1. 방사선 유발 MBU를 막기 위한 메모리 ECC로, EG-LDPC와 Hamming을 물리 layout으로 섞은 `Mix codes`를 제안.
2. 순수 EG-LDPC는 redundancy가 과도(예: `(127,64,7)`는 63 패리티)하고 512bit 같은 긴 데이터는 처리 불가한 점이 문제.
3. 방사선 실험상 한 이벤트의 오류 interval은 3bit 이하라는 가정을 이용, layout으로 오류가 동시에 같은 부호를 치지 않게 배치.
4. EG-LDPC 부호를 여러 section으로 나누고 각 interval에 Hamming 부호를 삽입, `n/2 >= j` 조건이면 Mix 코드 성립.
5. layout 배치로 Hamming이 보호하는 bit는 동시에 MBU를 맞지 않으므로 Mix 코드 정정능력은 EG-LDPC와 동일하게 유지됨.
6. 저 redundancy 방식: `(84,64,2)` = `(63,48,2)` EG-LDPC + `(21,16,1)` Hamming → `(127,64,7)` 대비 redundancy 68% 감소.
7. 긴 데이터 방식: `(582,512,4)` = `(511,448,4)` EG-LDPC + `(71,64,1)` Hamming으로 EG-LDPC 단독 불가 데이터 보호.
8. 복호는 EG-LDPC의 majority-logic decoder(반복 없음)와 Hamming을 병렬 구현.
9. Verilog 구현 후 fault injection 100만회, SMIC 0.18um 합성; `(84,64,2)` Mix는 EG-LDPC 대비 area 29%/delay 42%/power 31% 수준.
10. 한계: 순수 코드워드 구성법 + SRAM MBU 타깃, iterative message-passing·soft LLR·NAND 채널 요소가 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| EG-LDPC + Hamming 코드워드 구성 | 대응 없음 | Prime ECC는 QC-LDPC(base+circulant) 고정, EG-LDPC 구성법과 무관 |
| majority-logic decoder | 대응 없음 | min-sum iterative 복호(`decoder.cpp` `LDPC_Decoder()`)와 원리 상이 |
| layout 기반 오류 interval 제어 | 대응 없음 | 반도체 물리 layout 이슈로 코드베이스 범위 밖 |

적용 가치: **낮음** — majority-logic EG-LDPC + Hamming의 SRAM MBU 대책이라 NAND binary QC-LDPC min-sum 엔진에 떼어 쓸 요소가 없음.
