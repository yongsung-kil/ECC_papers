### B. 알고리즘 요약

1. 시스템: HDD 32k-bit(4KB) 섹터용 부호를 rate 8/9 고정, (36864,32768) 32k-bit vs (4608,4096) 4k-bit QC-LDPC 및 32k/4k RS 비교.
2. 문제: 긴 부호가 coding gain 크나 복잡도·수렴속도 대가 → 32k binary LDPC가 여러 부호안 중 어디에 위치하는지 성능-복잡도 정량화.
3. 채널: AWGN과 PMRC(transition jitter 80%, GPR target len4 + equalizer len11, SOVA/turbo eq), 채널밀도 1.4.
4. 부호설계: PEG 알고리즘 + QC 제약, 32k는 circulant size 32/column weight 5, 4k는 circulant size 4/column weight 5, 복호는 log-domain(min-sum 근사 가능) 최대 20 iteration.
5. AWGN 결과: 32k LDPC가 4k LDPC 대비 0.7dB, 32k RS 대비 2.2dB, 4k RS 대비 2.7dB gain @ BLER 1e-3.
6. MRC 결과: 32k LDPC가 32k RS/4k LDPC 대비 0.8dB, 4k RS 대비 1.6dB gain, 채널밀도 증가 시 gain 감소.
7. 복잡도: bit당 iteration당 연산량은 column weight에 비례하며 4k·32k 동일(같은 column weight일 때), 전체 연산은 iteration 수에 비례.
8. iteration: 32k 부호가 4k보다 더 많은 turbo iteration 필요(MacKay 추정 log log N), layered decoding이 iteration 제한 시 해법으로 제시.
9. 설계 여지: (36864,32768)에 대해 PEG-QC로 circulant size별 달성 가능 girth/column weight 표 제시, girth-6에서 column weight 최대 13까지 가능(4k에선 불가).
10. 한계: HW 미설계, 순수 시뮬 비교, 자기기록 채널·turbo eq 전제, 새 알고리즘 없음.
