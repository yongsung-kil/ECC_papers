### B. 알고리즘 요약

1. 시스템: PR(partial-response) 채널 + high-rate LDPC. binary는 PEG로 만든 (4551,4096) col-weight 4, rate≈0.9; nonbinary는 GF(16) (1138,1024) col-weight 3(4096 user bits).
2. 문제: BP 최적성은 채널 intrinsic 정보 독립 가정이 필요한데 PR 채널의 BCJR 검출기 출력 LLR은 서로 상관돼 첫 iteration부터 BP가 suboptimal.
3. 모델: 채널메시지 상관은 인접 소범위에서만 유의하고 lag 증가 시 소멸(autocorrelation으로 검증), 상관을 `p`개 연속비트 subblock의 결합분포로 표현(causal `c` + anti-causal `ac`).
4. 핵심기법(IBP): 관심 비트 APP를 subblock `Sk` 내 결합분포 `P(Sk)`로 확장(식1), check-to-bit는 기존 BP와 동일하되 bit-to-check 메시지에 상관항을 추가 반영(식2/5). 모든 LLR 독립이면 첫 항이 0이 되어 표준 BP로 환원.
5. 수식 의미: BP가 무시하던 채널메시지 간 종속성을 subblock joint distribution으로 주입해 coding gain 확보.
6. 확장(IQBP): nonbinary(GF(q)) BP로 확장, code symbol 단위 상관을 overlapped subblock 분포로 반영, row-step은 FFT 구현.
7. 구현 디테일(OBBD): BCJR 확장인 OBBD로 overlapped subblock 분포 `P(Sk)`를 검출기가 1회(또는 global iter마다) 생성, IBP 내부에서는 갱신 안 함; 경계는 짧아진 subblock으로 boundary management.
8. 채널: EPR4(정수계수, AWGN)와 equalized GPR PMRC(hyperbolic tangent transition, 90% jitter+10% 전자잡음), 사용자밀도 저(0.87)/고(1.22).
9. 결과: turbo eq 없이 IBP가 BP 대비 최대 0.7 dB(EPR4)/0.6 dB(고밀도 PMRC) SER 이득; 상관 약한 채널선 이득 작음; IQBP는 QBP 대비 0.1 dB 초과. binary+IBP가 nonbinary+QBP에 근접.
10. 한계: turbo equalization 도입 시 IBP 이득이 BP와 동등해져 소멸; OBBD의 subblock 분포 계산이 subblock 길이 `p`에 시간·공간 복잡도 지수증가; HW 미설계, 시뮬만.
