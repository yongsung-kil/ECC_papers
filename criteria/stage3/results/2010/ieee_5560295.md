### B. 알고리즘 요약

1. 대상: WiMAX(802.16e) 114개 모드 전부 지원하는 QC-LDPC 디코더 칩, 대표 코드 `(2304, 1/2)`, base matrix `M×N`, sub-matrix `b×b`.
2. 문제: 기존 QC-LDPC 디코더는 CNU/VNU 두 phase가 완전 오버랩되지 못해 compactness·전력효율이 부족.
3. 스케줄: `TDMP`(turbo-decoding message passing) 흐름 채택 -> TPMP 대비 수렴속도·성능 우위.
4. 핵심 HW: dual-path 처리로 각 phase가 non-zero sub-matrix를 block row-wise로 2개씩 스캔, 두 phase를 완전 오버랩.
5. 파이프라인: symmetrical 6-stage pipeline. 각 phase는 4-stage, CNU의 시작 stage가 VNU의 끝 stage와 오버랩(2-port sum RAM 이용).
6. 충돌 해소: reading conflict 제거 위해 3가지 전처리 - block column interleaving, block row interleaving, non-zero sub-matrix reordering.
7. 메모리 공유: sum RAM이 variable node sum과 extrinsic message(FIFO)를 함께 저장 -> 11,520 memory bit 절약, 4-port로 bank tag `TB=n%4`/addr `AB=n>>2` 사용.
8. bypass: 특정 block column은 sum RAM 접근 우회, VNU->CNU로 갱신 sum을 직접 전달.
9. 결과: iteration당 48~54 cycle, sum RAM 접근 24.3~48.8% 감소; 214 MHz·10 iter에서 847~955 Mb/s, 397 mW, 42 pJ/bit/iter; 6-bit 양자화 손실 <0.22 dB.
10. 한계: 알고리즘(정정능력) 개선 없음, WiMAX 규격 전용, NAND 고rate/soft-read 비용 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP 이터레이션/CNU·VNU 스케줄 | `decoder.cpp` `LDPC_Decoder()` | 두 phase overlap 스케줄 아이디어는 Prime 6-stage pipeline과 개념 유사, 참고 가능 |
| min-sum 계열 CNU 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문은 정규화 min-sum류 CNU(구체 미상), Prime 이미 보유 -> 중복 |
| sum RAM 파티션/bank tag/bypass | 대응 없음 | Prime HW 메모리 모델과 직접 매핑 어려움(RTL 메모리 구조 세부 미노출) |
| 양자화(6-bit) | `channel.cpp` + `ecc_data.h` `PARAM_LLR` | soft LLR 양자화 대응하나 NAND read 비용 고려 필요 |

적용 가치: **낮음** — 성능이 아닌 WiMAX 특화 메모리/스루풋 HW 최적화라 Prime의 z=32/HCU/GT 구조와 정합 부담이 크고 알고리즘 신규성이 없음.
