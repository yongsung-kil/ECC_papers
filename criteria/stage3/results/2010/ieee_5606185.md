### B. 알고리즘 요약

1. 대상: irregular LDPC를 λ(x)(symbol node)·ρ(x)(check node) degree 분포로 정의, PEG로 large-girth Tanner graph 구성; 실험은 n=10^5, rate R=0.5.
2. 문제: 원 PEG는 symbol node degree 시퀀스만 반영하고 check node degree 분포 ρ(x)를 무시 → 복잡한 ρ(x) 코드족에서 준수도 저하.
3. 핵심 기법 (FCD 기준): 원 PEG의 "최소 partial degree check node 선택" 대신 "final-partial degree 차가 최대인 check node 선택" `f(ci)=d(ci)-dk(ci)` (남은 socket 최다) 사용.
4. compliance 지표: pre-set ρj와 실제 ρ*j 차의 합 `η = Σ|ρj-ρ*j|`로 ρ-준수도 정량화(Table I: 원 PEG 1.49 대비 FCD 0.02 수준으로 대폭 개선).
5. 의미: FCD는 "socket" 개념 기반으로 목표 check degree 분포에 근접시켜 irregular 코드 성능(특히 waterfall) 향상.
6. 확장 트릭 (zig-zag): 2-degree symbol node의 첫 edge는 기존에 연결된 check node로만 제한 → 2-degree VN 사이클 회피로 error-floor 개선.
7. relaxed edge selection: 후보 앙상블 N^l에 free edge check node가 없으면 N^(l-1)에서 탐색 → 국소 사이클 길이를 희생해 ρ-준수도 η 개선.
8. 학습/최적화: 별도 학습 없음; λ(x)/ρ(x)는 density evolution 기반 기존 최적화 다항식([2],[4])을 그대로 입력으로 사용.
9. 결과: AWGN·BSC 시뮬(BP, 최대 2000 iter)에서 5개 PEG 변형 비교; 제안법이 waterfall 우수하며 낮은 error-floor 유지, ρ-준수도 최대 ~70배 개선.
10. 한계: HW 미설계, gate/throughput 없음; QC 구조 아님(랜덤 PEG), 저rate(0.5) 단일 예제, rate-compatible 응용 가능성만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG + FCD 기반 H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | 우리는 고정 QC base+shift 로드; PEG 랜덤 구성과 구조 상이, 직접 대응 없음 |
| irregular degree 분포 (λ/ρ) 설계 | 대응 없음 | Prime ECC는 고정 QC-LDPC(VN degree 17 등) 사용, 분포 재설계 레이어 부재 |
| 2-degree zig-zag / error-floor 구조 | 대응 없음 | dual-diagonal 인코딩(`encoder.cpp`)과 개념 근접하나 부호설계 재검증 부담 큼 |
| BP(sum-product) 복호 | `decoder.cpp` `LDPC_Decoder()` (min-sum) | 논문은 full BP 사용, 우리는 min-sum — 알고리즘 계열 다름 |

적용 가치: **낮음** — 순수 irregular(비-QC) PEG 코드 구성법으로 Prime ECC의 고정 binary QC-LDPC 부호설계 레이어(재검증 부담 큼)에 닿지 않고 HW 설계도 없어 이식 실익이 작음.
