### B. 알고리즘 요약

1. 대상은 IEEE 802.11n QC-LDPC(length `648~1944`, rate `1/2`,`5/6`, AWGN), TDMP(turbo-decoding message-passing) + SISO(수정 BCJR) 커널 기반 layered 복호.
2. 문제: 매 iteration 후 codeblock 유효성 확인용 syndrome check(`H·cᵀ=0`)가 전용 메모리/유닛과 별도 시간을 소모해 latency·면적 낭비.
3. 핵심 관찰: LLR magnitude는 고SNR에서 단조 수렴, 저SNR에선 초기 변동 → 최종 결과 전에 부분 판정 가능(참고 [6][7]).
4. 핵심 기법(Algorithm 1): row `i`를 복호한 직후 그 parity-check `PCi`를 hard-decision(sign)의 ⊕로 즉시 평가; `PCi=1`(불만족)이면 나머지 row 검증을 즉시 disable, 마지막 row까지 모두 만족하면 valid로 halt.
5. QC-LDPC/architecture-aware 코드 구조상 `P`개 row를 병렬 처리 가능, iteration 사이 syndrome 지연이 best-case(초기 row 실패 시)에 소거됨.
6. 부작용 분석: 순차평가라 식(1)과 불일치 가능 — 모든 PC 만족 후 마지막 배치 실패 시 false alarm(undetected error), 마지막 iteration에 1개 PC 실패 시 miss(불필요 재전송). PH = 1−(PFA+PM).
7. 회복 방안: on-the-fly 최종 syndrome를 다음 codeblock 첫 iteration 동안 full-syndrome로 validate(또는 외부 CRC)해 성능 손실 완전 회복(Fig.5의 validated 곡선).
8. HW 임팩트: canonical TDMP 디코더에서 전용 syndrome 유닛 제거, 각 processing unit에 소량 회로 추가; 65nm 합성 결과 BCJR PU 대비 area overhead 단 0.65%.
9. 결과: syndrome check가 iteration당 태스크의 35% 차지 가정, Amdahl 법칙 상한 1.53배; rate 1/2에서 최대 성능의 84~96% 달성 → throughput 1.28~1.48배 향상(저SNR에서 이득 최대).
10. 한계: iteration 수 감소는 아니며(iteration 간 latency만 단축), false alarm/miss가 저SNR에서 저확률 발생하므로 validate 회로 replication과 hard-decision 메모리 유지가 필요, 어플리케이션별 평가 요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome check / parity 검증 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | Prime의 iteration 루프에서 row별 partial syndrome 산출·조기중단 로직 삽입 지점 |
| 조기종료(codeblock validity) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()`, [full_CRC.cpp](../Prime_ECC_3.1_Claude) | Prime은 partial(4 termination)/full CRC로 조기종료 보유; on-the-fly syndrome는 이와 상보적 latency 최적화 |
| row 병렬 처리(P rows) HW | 디코더 HW 모델 (z=32, HCU, GT) | z=32 row 병렬 단위가 논문의 P-row 병렬 syndrome 평가와 정합 가능 |

적용 가치: **중간** — iteration 간 syndrome 지연을 없애 latency/throughput을 개선하고 area overhead가 0.65%로 낮아 Prime의 관심사(latency·HW 효율)와 부합하나, iteration 수는 줄지 않고 Prime은 이미 partial/full CRC 조기종료를 보유하므로 상보적 참고 수준이다. false alarm/miss 회복을 위한 validate 로직 설계가 추가 부담.
