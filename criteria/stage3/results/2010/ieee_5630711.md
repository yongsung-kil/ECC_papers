### B. 알고리즘 요약

1. 대상: Chinese DTTB(DTMB) 표준 QC-LDPC 디코더, 대표 `0.8-rate` 코드 (H = 11행×59열, circulant `127×127`, 코드길이 `59×127=7493`).
2. 알고리즘: normalized min-sum. 초기화 `Pn=-2yn/σ²`, horizontal update `Rmn=∏sgn·min|Qmn'|`, vertical update `Qmn=Pn+ΣRm'n`.
3. 문제: 고전 semi-parallel 구조에서 HPU(check)와 VPU(variable)가 데이터 종속으로 동시 동작 불가 -> 절반 시간 idle, 자원 활용률 저하. 행 weight가 크면 HPU 입력수 증가로 자원 폭증.
4. HPU 비교: min 연산을 direct scheme(2-input 비교기 트리, `qt=log2(t)·t-2` 초선형 증가) vs index scheme(`{min,sub-min,index}` 계산, `q't≈3t-3` 선형 증가) 비교.
5. 결론: 입력수 7 부근에서 동등, 그 이상은 index scheme가 면적 효율적.
6. foldable HPU: index scheme 기반, 한 행 메시지를 `k`개로 나눠 직렬 입력. 중간 결과 `{min,sub-min,index}`를 RRAMT에 저장, Minboth 모듈로 병합 -> min 연산 모듈 복잡도 `1/k`.
7. splitting-matrix: H를 열 기준 `k`개 서브행렬로 분할(열/행 weight 균등 유지). VPU가 서브행렬을 직렬 처리하며 이전 서브행렬의 h-process를 HPU가 동시 수행 -> 첫/마지막 clock 제외 HPU·VPU 동시 동작.
8. 자원식: 효율 `E=1/((VPU+HPU logic)×clocks)`, 개선비 `e=En/Ec=2k/(k+1)` -> k 클수록 유리하나 라우팅 오버헤드로 실무상 적정 k 필요.
9. 결과(k=2, DTTB 0.8-rate): logic -36.9%, register -45.9%, memory +7.2%, max freq 54->66.7 MHz(+23.5%), max iter 30, iteration당 3×127 clock.
10. 한계: FPGA 자원 지표만, 공정/gate·throughput 수치 없음, 정정능력/BER 개선 없음, min-sum은 기존 기법.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized min-sum horizontal/vertical update | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime min-sum 이미 보유 -> 중복, 신규성 없음 |
| foldable HPU (min/sub-min/index 직렬 병합) | `decoder.cpp` `CNU_Update_New_Mag()` | min1/min2 기반 CN 연산과 동일 개념, HW 자원 재사용 아이디어만 참고 |
| splitting-matrix 스케줄(HPU/VPU 동시) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 파이프라인/스케줄 층에 대응하나 Prime은 이미 6-stage pipeline 보유 |
| H-matrix 열분할/로드 | `ecc_top.cpp` `Load_PCM()` | 부호 종속 분할 -> 특정 QC-LDPC 재검증 부담 |

적용 가치: **낮음** — 핵심 알고리즘(min-sum)이 Prime 보유 기법과 중복이고, 성능이 아닌 FPGA 자원효율 개선이라 z=32/HCU 고정 구조에 이식 이득이 작다.
