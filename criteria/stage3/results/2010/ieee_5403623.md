### B. 알고리즘 요약

1. 대상: rate-1/2, `n=2304`, 확장인자 `z=96`, base matrix 12x24 QC-LDPC (binary, circulant permutation `P`).
2. 문제: parity 부호화를 위한 기존 dual-diagonal ALT 구조가 거의 완전 결정적이라 부호 설계 자유도가 낮아 에러율 개선 여지가 적음.
3. 핵심: parity 부분 `H2`(m×m)를 근사 하삼각(ALT) 형태로 두되, non-zero sub-block의 약 절반은 upper diagonal 위에 고정, 나머지 절반은 하삼각(trapezium) 영역 어디든 규칙 하에 배치(반결정적).
4. 첫 열은 weight 3(세번째·마지막 sub-block이 동일 `P_y`, 두번째 마지막은 `I`), 이후 열은 upper diagonal `I` + 하나의 하부 non-zero, 마지막 두 열은 4개 `I`로 dual-diagonal 마감.
5. 열 가중치는 dual-diagonal과 동일하게 유지(첫 열만 3, 나머지 2)해 복호 degree 분포 보존.
6. 인코딩: `Hc^T=0`에서 행별 방정식 (4)~(9)로 `p1`부터 재귀 계산, 마지막 세 행 합으로 `p0` 산출 후 `p_{mb-1}`,`p_{mb-2}` 역산 → dual-diagonal과 유사 선형복잡도.
7. 행 가중치는 `Hb1`,`Hb2` 각각 비균일이나 전체 `Hb`는 근사균일(6 또는 7)로 유지되도록 데이터부 행 가중치를 상보 설계.
8. A code는 802.16e와 동일 열가중치·행가중치 분포(순서만 상이), B code는 열가중치 분포를 약간 변경.
9. 결과: AWGN, BP 복호(최대 80 iteration)에서 A code가 802.16e를 상회, B code가 A code보다 추가 우수(BLER/BER, 최소 100 block error까지 시뮬).
10. 한계: rate-1/2·AWGN 시뮬만, HW/throughput/gatecount 무기재, NAND 고rate로의 확장 검증 없음.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ALT(근사 하삼각) parity 구조 + 재귀 인코딩 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC는 dual-diagonal 인코더 고정; ALT 구조는 부호 PCM 설계 자체를 바꿔야 이식 가능 |
| base matrix + circulant P 부호구조 | ecc_top.cpp Load_PCM() | H-matrix 로드 계층에 새 QC 구조 반영 필요(고정 QC-LDPC 재검증 부담) |
| 부호부 dual-update 열 재배치 | encoder.cpp Column_Reordering_Dual_Update() | parity 열 배치 자유도가 dual-update 스케줄과 정합해야 함 |
```

적용 가치: **중간** — dual-diagonal 대비 유사 복잡도로 에러율을 낮추는 부호설계 아이디어라 관심 대상이나, Prime ECC의 고정 고rate QC-LDPC를 재설계·재검증해야 하고 rate-1/2 AWGN 예시뿐이라 직접 이식 부담이 큼.
