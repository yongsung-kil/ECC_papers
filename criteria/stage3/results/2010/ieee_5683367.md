### B. 알고리즘 요약

1. 시스템: 우주 방사선 환경 메모리(COTS, NAND 포함) 보호, scrubbing으로 ECC 출력 재기록, rate 8/9(및 4/5) 부호.
2. 문제: 기존 메모리 ECC는 SECDED로 3비트 이상 오류 불가, 짧은(<1K) LDPC 설계난이도·장부호 복잡도로 메모리엔 LDPC가 미채택.
3. 핵심모델: soft error(transient, 정정가능)와 hard error(permanent=erasure)를 시간의존 전이확률로 정의, cascade 등가성으로 `p(T),q(T),r(T)` 미분방정식 해 도출.
4. 핵심기법1(채널→LLR): 채널파라미터로 메모리 출력 LLR 유도, 대칭채널서 `Lch(y)=±log(p(Ts)/r(Ts))`, hard error(erasure)는 `Lch=0`.
5. 핵심식 의미: `p(Ts)/r(Ts)=tanh(λTs)`로 hard-error rate `λe`가 상쇄되어 대칭채널 LLR이 `λe`에 무관 → soft LLR이 soft error rate에만 의존.
6. 핵심기법2(부호): PEG LDPC(col-weight 3/5, girth6)와 JPL protograph AR4JA(rate4/5) 두 구성을 RS/BCH와 동일 rate/length로 비교.
7. 구현디테일: LDPC는 sum-product 또는 min-sum 복호, min-sum은 채널파라미터 민감도 낮은 저복잡 근사.
8. 최적화: pseudo-decoding(2e+f≤2t) 기반 RS/BCH 블록오류율 해석 계산, LDPC는 Monte Carlo.
9. 결과: (4608,4096) LDPC-SPA가 RS/BCH·LDPC-min-sum 대비 큰 이득, SPA-min-sum 격차는 부호길이↑일수록 커짐(AWGN 0.4dB@BER10^-6), 짧은 1~2kbit에선 이득 축소.
10. 한계: HW 미설계·시뮬만, 우주 방사선/scrubbing 특화 채널, hard error를 erasure로 단순화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 채널파라미터→soft LLR 유도(식20/21) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 방사선 메모리 채널 LLR 유도 방식, NAND read LLR 테이블 설계에 개념 참고 가능 |
| hard error=erasure(LLR 0) 처리 | `channel.cpp` / LLR 테이블 | erasure를 LLR 0으로 넣는 방식, 우리 stuck-bit/불량셀 모델에 응용 여지 |
| sum-product vs min-sum 복호 | `decoder.cpp` `LDPC_Decoder()` | 우리는 min-sum 보유, SPA 이득 비교는 참고용(full-tanh SPA는 대응 없음) |
| PEG/AR4JA protograph 부호 | `ecc_top.cpp` `Load_PCM()` | protograph/circulant 부호설계, 우리 고정 QC-LDPC로 이식 시 재검증 부담 큼 |

적용 가치: **중간** — 메모리 채널의 soft-LLR 유도와 erasure 처리, LDPC의 RS/BCH 대비 우위 실증은 참고가치가 있으나 방사선 scrubbing 특화 채널이라 NAND read 물리로의 직접 이식은 제한적.
