### B. 알고리즘 요약

1. 대상은 부호가 아니라 그래프 분석 도구: bipartite Tanner 그래프에서 길이 `g, g+2, ..., 2g-2` short cycle 개수를 세는 분산 message-passing 알고리즘 (예제는 rate-1/2 LDPC 4종, n=1008~20000).
2. 문제: finite-length LDPC 성능은 short cycle 수/분포에 강하게 의존하나 기존 계수법(Halford-Chugg `O(gn^3)`, Fan-Xiao `O(m^k n)`)은 큰 그래프·긴 cycle에서 계산·메모리 부담 과다.
3. 모델: 각 노드가 인접 edge로 monomial 메시지를 곱셈 전파(extrinsic), monomial을 지수 벡터로 표현해 곱셈을 벡터 덧셈으로 환원.
4. 핵심기법: 노드 v의 각 edge에 서로 다른 변수 `X1..Xdv`를 초기 전파, iteration k에서 v로 되돌아온 메시지 지수합으로 v를 지나는 2k-cycle 수 계수 (Lemma 3, 식 2).
5. 핵심식: `N_2k^v = (Σj ex(m_I,(v;ej)))/2` - 초기변수 제외한 수신 monomial의 지수합/2가 v 통과 cycle 수 (non-cycle closed walk는 최단 2g라 k≤g-1에서 자동 배제).
6. 확장: 전체 그래프는 한 partition의 모든 노드에 적용 후 `N_2k=(Σ N_2k^u)/k` (식 3), 계수 끝난 노드 비활성화로 k중복 제거.
7. 트릭: 병렬 구현 - 한 partition 노드 동시 초기화, iteration g/2에서 최초 non-one 수신으로 girth g와 `N_g` 동시 계산.
8. 복잡도: 정수 가감만 사용, 계산 `O(g|E|^2)`, 메모리 `O(dmax|E|)` - sparse 그래프(|E|<n^{3/2})에서 `O(gn^3)`·`O(n^2)` 메모리 대비 유리.
9. 결과: (8000,4000) 3-6 regular LDPC에서 Halford-Chugg 대비 30배 이상 빠르고 메모리 ~600배 절감, Code D는 기존법이 out-of-memory인 반면 계수 성공 (Table I-III, 2.2GHz/8GB).
10. 한계: 복호 알고리즘이 아닌 오프라인 그래프 분석 도구, HW 미설계, girth 6 이상 가정 하 g~2g-2 범위만 계수.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Tanner 그래프 short cycle 계수 (부호품질 진단) | `ecc_top.cpp` `Load_PCM()` (분석 대상 H-matrix 공급) | Prime ECC H-matrix의 cycle 분포 오프라인 진단에 이 도구 적용 가능 |
| short cycle ↔ error-floor 상관 | `corner.cpp` (수렴실패 추적, 간접) | trapping-set/error-floor 원인 cycle 식별 보조로 간접 연관 |
| message-passing 복호 자체 | 대응 없음 | 논문 message-passing은 cycle 계수용, min-sum 복호와 무관 |

적용 가치: **중간** - 복호 이식 대상은 아니나, Prime ECC QC-LDPC H-matrix 설계·error-floor 진단 시 short cycle 분포를 저비용으로 계수하는 오프라인 툴로 재사용 가치가 있다.
