### B. 알고리즘 요약

1. 대상: 임의 Tanner graph(구조/랜덤)의 dominant trapping set 탐색 — error-floor 추정 및 low-error-floor 부호설계 보조.
2. 문제: 크기 `t`, 길이 `n` 구조의 brute-force는 `C(n,t)`로 폭발(예 `t=10,n=1000`에서 ~2.6×10^23), 기존 탐색은 길이 ~500·크기 ~11까지만 실용적.
3. 핵심 관찰: 모든 (elementary) trapping set은 최소 하나의 짧은 cycle을 포함(Lemma 5,6,7,8) → 짧은 cycle을 초기 입력으로 사용.
4. 정의: `(a,b)` trapping set = `|s|=a`, unsatisfied check 수 `|O(s)|=b`; elementary = 모든 check degree ≤2; dominant = 주어진 `a`에 최소 `b`.
5. 확장 원리(Lemma 2~4): 크기 `a_i`에서 `a_{i+1}`로 확장 시 새 변수노드는 `O(s)`에만 연결되고, 두 개 연결이면 cycle 없음·한 개 연결이면 lollipop walk 1개(정확히 1 cycle).
6. Algorithm 1: 입력 TS `t_i`에 대해 `F(t_i)`와 이웃 제거 후, `O(t_i)`의 각 check에서 최대 `imax=k-|t_i|` layer까지 최단경로/lollipop walk를 찾아 확장, `|O(t')|≤T`인 것만 출력.
7. 재귀: 현재 단계 출력을 다음 단계 입력으로 사용해 크기 증가 방향으로 반복.
8. 초기집합: girth `g` 및 `g+2`(밀한 그래프) 또는 `g,g+2,g+4`(성긴 그래프)의 짧은 cycle 열거.
9. 결과: Tanner(155,64)에서 (8,2)등 dominant TS를 1분 내 탐색(4-bit min-sum, AWGN 6.5dB에서 오류패턴 90%가 (8,2)); Margulis(2640,1320)에서 (12,4)/(14,4) TS를 55분(impulse법 8.2시간 대비)에 탐색; 최대 길이 ~10,000 부호 처리 가능.
10. 한계: 부호구성/복호 자체 개선 없는 진단 도구, HW 미설계, 복잡도는 그래프 밀도(degree 분포)에 민감, threshold `T` 선택 필요.
