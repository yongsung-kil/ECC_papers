> 총평: layered 복호에서 조기종료로 인한 BER 손실을 제거하면서 빠른 termination을 얻는 알고리즘 레벨 기법 — Prime ECC의 CRC 조기종료와 목적이 같고 디코더 루프에 얹기 쉬워 이식성은 높으나, 우리는 이미 CRC 기반 조기종료 보유라 신규성은 제한적.

### B. 알고리즘 요약

1. 대상: QC-LDPC(WiMAX 2304, DVB-S2 64800), rate 1/2, BPSK/AWGN, 8-6bit 고정소수점, normalized min-sum(factor 0.75) layered 복호.
2. 문제: layered 복호는 한 iteration 안에서 bit-LLR이 여러 번 갱신되어, 기존 조기종료(ET)들이 BER 성능 열화를 유발함.
3. 기존 HDA-ET: 연속 두 iteration의 hard-decision codeword가 같으면 종료 — parity check 안 하므로 BER 보장 불가.
4. 기존 CSC-ET: 현재 iteration에서 layer별로 순차 parity 만족 확인 — 임시 hard-decision을 쓰므로 "모든 식 동시 만족" 기준과 불일치.
5. CSC 결함: 변수노드 `Vk`가 layer j 처리 중 1→0으로 바뀌면 앞서 만족했던 check(i)가 다시 불만족 → 오종료 → BER 열화.
6. 제안 LSC-ET: (i+1)번째 iteration에서 i번째 iteration이 만든 hard-decision으로 layer별 parity check, 한 iteration 내 모든 식 만족 시에만 종료.
7. 핵심: APP 메시지마다 hard-decision bit 1개 추가, 각 layer에서 "마지막 갱신(last-updated)" 위치의 hard-decision만 최신값으로 유지.
8. 구현: APP 메모리와 permuter의 데이터폭을 1bit 확장해 hard-decision을 PE에 전달, DVB-S2 기준 메모리 오버헤드 ≤3%.
9. 결과: BER는 이상적(non-ET)과 거의 동일하게 유지하면서 1/AIT(평균 iteration 역수, throughput ∝ 1/AIT) 개선, DVB-S2에서 특히 빠른 종료.
10. 한계: FPGA/ASIC 합성 없이 시뮬만, gate count/throughput 절대치 미제시, extra error floor는 없다고 주장.
