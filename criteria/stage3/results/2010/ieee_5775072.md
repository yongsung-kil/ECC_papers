### B. 알고리즘 요약

1. 시스템: IEEE 802.11n WLAN (3,6)-regular LDPC, code length 3072bit, rate 1/2, BPSK/AWGN, iteration 5~10.
2. 문제: partial-parallel 디코더의 concurrent 스케줄은 row/column 연산 효율 저하·메모리 2배, [10]의 accelerated 스케줄은 매 row 연산 후 3개 column 연산을 순차 수행해 중복 발생.
3. 가정: QC parity-check matrix(identity 우측 시프트)로 각 row 연산 시작점을 골라 같은 column 위치에서 처리 가능.
4. 핵심 기법1: i1,i2,i3 row 연산을 동시에 수행하되 시작 row를 조정해, 한 번의 column 연산으로 β'(jq(1..3)) 갱신 → [10] 대비 불필요한 `(3-1)×3` β 업데이트 제거.
5. 핵심 식: row `α(x,y)=∏sign(β)·f(Σf(|β|))`, column `β(x,y)=λy×Σα(x',y)` — β를 다음 row 연산에서 실제로 쓰일 때만 갱신.
6. 확장: row 연산 순서를 이용해 현재 column의 β 갱신 필요 여부 판단 → β 기록횟수를 [10]의 1/4로 축소.
7. 구현 디테일: row 연산 7-stage(CFU: β읽기/set, |β|, 1st/2nd min, select, sign 부여), column 연산 5-stage(BFU: α읽기/set, Σα, λ 가산); read/write 충돌 회피 위해 stage 분할.
8. 결과: MATLAB로 [10]과 BER 거의 동일 확인, IBM 90nm 합성 시 gate 95,873, throughput 203Mbps(iter10), column 연산 25%↓, 정규화 전력 27%↓.
9. 검증: Synopsys Design Compiler 합성, Table I에 [10](TSMC 0.18um) 대비 스케일링 보정 비교.
10. 한계: 정정성능 향상 없음(전력·연산만), (3,6)-regular·802.11n 특정 부호 전용, NAND 미언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| row/column 연산 스케줄 (β 업데이트 제거) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 스케줄링 최적화 → 이식 레이어(디코더 루프)와 직접 대응 |
| CFU min-sum (1st/2nd min + sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum과 동일 |
| BFU column 연산 (Σα + λ) | `decoder.cpp` `VN_Cal_HD()` 등 | 표준 VN 처리, 중복 |
| 메모리 접근 축소 (β 선택적 기록) | `decoder.cpp` `LDPC_Decoder()` 메모리 스케줄 | 우리 z=32/6-stage pipeline과 정합 검토 필요 |

적용 가치: **중간** — 정정성능은 동등하나 β 업데이트·메모리 접근을 줄이는 스케줄 아이디어가 우리 디코더 루프의 전력/메모리 최적화에 참고될 수 있음. 단 (3,6)-regular 전용이라 우리 고rate 비대칭 degree(HCU) 구조 적용엔 재설계 필요.
