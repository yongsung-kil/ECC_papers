### B. 알고리즘 요약

1. 대상은 `γ`-left-regular LDPC 부호와 left-regular·right-uniform GLDPC(Tanner) 부호, BSC 상 hard-decision bit-flipping(serial/parallel) 복호.
2. 문제: 반복 복호의 "보장(worst-case) 정정능력"을 랜덤그래프·앙상블 논증 없이, 쉽게 계산되는 부호 파라미터(column weight `γ`, girth `g`)만으로 상·하한을 구한다.
3. 핵심 모델: Sipser-Spielman expander 논증 — Tanner graph가 `(γ,α, δ)` expander면 병렬 bit-flipping이 특정 비율의 오류를 정정(Fact 1).
4. 핵심 기법(하한): Moore bound를 이용해 최소 `3/4` 팽창을 보장하는 variable node 집합 크기를 `γ`,`g`의 함수로 유도(Lemma 4, Theorem 1) — 정정능력이 girth에 지수적으로 증가.
5. 핵심 식 의미: reduced/edge-vertex incidence/augmented graph 변환으로 subgraph의 평균 degree를 Moore bound에 넣어 팽창 하한을 얻음.
6. 핵심 기법(상한): 복호 실패를 fixed set/trapping set으로 특성화(Theorem 2 필요충분조건), cage graph의 edge-vertex incidence augmented graph가 potential fixed set을 구성 → fixed set 크기가 상한(Theorem 3).
7. 부수 디테일: Table I에 `γ`=3~7, girth별 정정능력 상·하한 수치표 제시; Example 1로 하한이 tight한 경우(oscillation trapping set) 예시.
8. 학습/최적화: 없음(순수 조합론·그래프 이론 증명).
9. 결과(확장): regular GLDPC에서 병렬 bit-flipping이 good expander면 code length에 선형 개수의 worst-case 오류 정정 가능함을 증명(Theorem 4~7), 필요 팽창을 `γ`와 component code 정정능력 `t`의 함수로 도출.
10. 한계: 유도 하한이 실제 부호 성능 대비 매우 느슨(girth가 code length에 로그로만 증가 → 선형 정정 불가 증명 못 함), HW·soft·시뮬 전무한 순수 이론.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping hard-decision 복호 | 대응 없음 (Prime ECC는 min-sum soft: [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`) | 복호기 클래스(bit-flip vs min-sum)가 달라 직접 대응 없음 |
| trapping set / fixed set error-floor 특성화 | 대응 없음 (근접: 수렴실패 추적 [corner.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md)) | error-floor 진단 관점에서 정성적 참고 가능성만, 알고리즘 이식 아님 |
| girth·column weight 기반 부호설계 지표 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` (H-matrix) | 부호 설계 시 girth 지표는 일반론이나, 본 논문은 bit-flip 전용 이론 상·하한이라 QC-LDPC 설계에 직접 적용 안 됨 |
| GLDPC/component code | 대응 없음 | Prime ECC는 순수 binary LDPC, GLDPC(component block code) 미지원 |

적용 가치: **낮음** — trapping/fixed set과 error-floor 관계라는 개념적 프레임은 참고 가치가 있으나, bit-flipping·BSC 전용 순수 이론 상·하한이며 HW·soft 복호·NAND와 무관해 Prime ECC에 떼어 이식할 알고리즘이 없다.
