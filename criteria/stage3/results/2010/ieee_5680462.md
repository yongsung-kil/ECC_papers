> 총평: 고정소수점 offset-min-sum layered(TDMP) QC-LDPC 디코더를 stream processor에 올려 8bit saturation을 extrinsic hard-limit로 잡고 스케줄 재정렬·integrated parity check로 throughput을 높인 구현 논문으로, min-sum·QC-LDPC·layered라 Prime ECC와 결이 맞지만 stream processor 특화라 이식성 중.

### B. 알고리즘 요약

1. 대상은 64×64 circulant 부분행렬로 구성된 (2432,1216) rate-1/2 QC-LDPC(repeat-accumulate, Robertson cage)를 fixed-point stream processor(Storm-1)로 복호하는 것이다.
2. 문제: 저전력 embedded stream processor는 fixed-point만 지원하고, 최고 throughput의 8bit 모드는 posterior saturation으로 error 성능이 나빠진다.
3. 알고리즘은 offset-min-sum을 적용한 TDMP(layered/turbo-decoding message passing): H의 row block을 순차 복호하고 앞 row의 extrinsic을 뒤 row의 prior로 사용한다.
4. 핵심 갱신식: prior `ρ=γ(Ii)-λi`, `λij = (∏ sgn ρk)·max(min_{k≠j}|ρk| - η, 0)`, posterior `γ(Ii)=ρ+λi` (offset η=0.125, scaling ξ=0.7).
5. offset-min-sum TDMP는 10 iter로 min-sum 대비 0.25dB↑, SPA 20 iter와 동등(SNR 2dB 평균 6.2 iter vs SPA 11 iter).
6. fixed-point 분석: 16bit는 float 대비 0.03dB 손실뿐이나, 8bit는 iter 11~20에서 posterior saturation이 subiteration 간 전파돼 성능이 나빠진다(sequential TDMP가 flooding SPA보다 심각).
7. 완화 트릭: subiteration당 extrinsic update의 최대값을 hard-limit(8bit 정수 0~127에서 20~30)로 제한하면 16bit급 성능 회복.
8. HW 매핑: circulant 크기 64 → 8bit로 lane당 4 row씩 64 concurrent row update로 16 lane 완전활용(부분병렬); posterior는 lane 간 교환 필요.
9. throughput 트릭: (a) 데이터 의존 row가 연속되지 않게 스케줄 reorder → iter당 시간 7%↓/throughput 최대 7.7%↑, (b) integrated parity check(row 갱신 직후 parity 확인, inter-lane 교환 제거) → SNR 2dB에서 iter 6.50→7.85로 늘지만 throughput 28.5→35.2Mbps(23.5%↑).
10. 한계: Storm-1(130nm) stream processor 특화 실측이라 ASIC/RTL 디코더가 아니고, error floor·정정능력 자체 개선은 없다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset-min-sum layered(TDMP) 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum과 동일 계열, offset항 추가 여부만 차이 |
| 8bit fixed-point saturation·extrinsic hard-limit | `decoder.cpp` `VN_Cal_HD()` 등 + `ecc_data.h` `PARAM_LLR` | 양자화 magnitude 테이블/clamp 설계에 직접 참고 |
| iteration별 스케줄·row reorder | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, Dual-Update 스케줄 | 데이터 의존 완화 재정렬, Dual-Update(even/odd) 스케줄과 유사 개념 |
| integrated parity check 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | row별 즉시 parity 확인 vs partial-CRC termination point 비교 검토 |
| circulant 병렬 lane 매핑 | 디코더 HW 모델(z=32) | z_sb=32 row 병렬과 개념 유사하나 stream processor lane 구조는 상이 |

적용 가치: **중간** — min-sum·layered·QC-LDPC·fixed-point 양자화·조기종료 등 Prime ECC와 다루는 레이어가 정확히 겹쳐 offset/hard-limit·스케줄 아이디어는 참고 가치가 있으나, 이미 보유한 기법(min-sum/dual-update/partial-CRC)과 중복이 많고 stream processor 특화 구현이라 그대로 이식할 것은 적다.
