> 총평: NAND의 ICI 완화용 constrained(LOCO) coding 논문으로 binary LDPC ECC 파이프라인 밖 계층이라 Prime ECC 이식성은 하.

### B. 알고리즘 요약

1. 대상 시스템: q≥4 레벨/셀 MLC/TLC(q=8) NAND Flash. LDPC ECC의 앞단에서 ICI 유발 write 패턴을 제거하는 constrained coding.
2. 문제: 기존 capacity-achieving constrained code는 모든 페이지를 함께 처리해야 해 페이지 분리 불가·접근속도 저하. 페이지별 독립 read 유지가 목표.
3. 모델: 고-저-고 레벨 패턴 집합 `Lq = {β1 µ β1 | 0≤µ<min(β1,β1)}`가 ICI 주요 원인(ML 기반 특성화로 확정).
4. 핵심 기법1(binary RR): RAGM(recursive alternate Gray mapping)으로 레벨↔비트 매핑 후, 최좌측 페이지에서만 `{000,010}` 금지 → interleaved RLL (0,1) 제약. 나머지 페이지는 무코딩.
5. 부호화: binary LOCO code `RC_2^m`을 lexicographic index 규칙 `g(c)` (식 10)로 encode/decode — adder 하나로 구현되는 저복잡도.
6. 핵심 기법2(4-ary RR): 두 최좌측 페이지에 GF(4) LOCO code `RC_4^m` 적용, 완화 집합 `Lq' ⊂ Lq` 금지로 rate loss 축소.
7. 구현 트릭: 4-ary는 2-symbol bridging(정보비트 2개 동시 인코딩)으로 동일 rate를 짧은 length에서 달성, 복잡도·error propagation 감소.
8. 용량 분석: 1D binary RR 용량 `≈(log2 q−0.3058)/log2 q`, 4-ary RR `≈(log2 q−0.2282)/log2 q`; capacity-achieving.
9. 결과: 상용 1X-nm TLC 실측 — BER 2×10⁻³ 기준 1D binary RR-LOCO(wordline) 수명 약 57%(≈1800 P/E) 향상, 3×10⁻³에서 79%(≈3700 P/E) 향상.
10. 한계: HW 미설계, LDPC 디코딩/부호설계 자체는 다루지 않음(향후 결합만 언급). 초기 P/E 구간에서는 uncoded 대비 crossover(성능 역전) 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RR-LOCO constrained coding (LDPC 앞단 page-level 제약) | 대응 없음 | ECC가 아닌 pre-ECC 계층, Prime ECC 부호/디코더와 무관 |
| RAGM Gray mapping / 페이지 분리 | 대응 없음 | 채널·매핑 계층, `channel.cpp`의 LLR/threshold와도 직접 대응 없음 |
| LDPC decoder로 데이터 전달(언급만) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 논문은 LDPC 입력을 만들 뿐 디코더 개선 기여 없음 |

적용 가치: **낮음** — Prime ECC는 binary QC-LDPC ECC 엔진인데 본 논문은 ICI 완화용 constrained coding(LOCO)으로 부호설계/디코더/HW 어느 레이어도 이식 대상이 아니다.
