### B. 알고리즘 요약

1. 대상은 5G 표준의 QC-LDPC 디코더 HW로, PE(processing element)와 RAM 뱅크 간 interleaving을 수행하는 인터커넥트 네트워크 설계 문제.
2. 문제: 5G QC-LDPC는 lifting size(expansion factor) `m`이 2~384 사이 51개 값을 가지며, 여러 표준(LTE-A/WiFi/DVB)의 서로 다른 프레임을 동시에 다른 permutation으로 처리해야 하는데, 기존 barrel shifter/QSN/EBS는 단일 프레임 또는 제한된 병렬성만 지원.
3. 핵심 가정: N×N Back-to-Back Butterfly Network(B2BN, `N=2^n`)는 Benes network와 동등한 표현력으로 N! 개의 모든 permutation을 수행 가능하나 제어신호(무충돌 경로) 생성이 어려움.
4. 핵심 기법: 입력 `xi`에서 출력 `zj`로 가는 모든 가능 경로 집합 `Txi→zj`(길이 `2n-1`)를 정의하고, 이를 행렬 `A`(크기 N×2n)로 표현하되 "같은 열의 원소는 서로 달라야 함"(MUX 충돌 방지) 제약을 부여.
5. 핵심 식: 제어신호 행렬 `B`는 `A`의 인접 열 비교(`bij = 0 if a_i,j+1=a_i,j else 1`)로 계산(Algorithm 2) — MUX가 직접입력/간접입력 중 무엇을 선택할지 결정.
6. 확장/모듈화: N×N-B2BN에서 2N×2N-B2BN으로 확장하는 재귀적 경로 생성 알고리즘(Algorithm 3)을 제시해 임의 크기로 스케일업 가능.
7. 부수 구현: 실제 경로 탐색은 제약충족문제(CSP)로 정식화해 Gecode 솔버로 오프라인에 풀고, 결과 제어신호를 ROM에 저장(런타임 재계산 없음).
8. 학습/최적화 절차: Gecode 제약해결기로 permutation별 오프라인 탐색(별도 알고리즘 학습은 아님).
9. 결과(Table I): 512×512-B2BN 기준 MUX 개수가 QSN 대비 약 0.72~0.92배, EBS 대비 약 0.44~0.67배 수준의 오버헤드로 다중 프레임 병렬 permutation 지원(단, QSN/EBS는 애초에 다중 프레임 처리 불가/제한적).
10. 한계: 순수 아키텍처/이론 제안으로 합성·FPGA 검증 없음(이론 단계), 오프라인 솔버 의존(런타임 실시간 재구성 알고리즘은 future work로 언급).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PE-RAM 인터커넥트/셔플 네트워크 (barrel shifter 대체) | 대응 없음 | Prime ECC는 z_sb=32 고정 lifting의 row 단위 병렬 구조로 가변 크기 permutation 네트워크 자체가 불필요 |
| 다중 프레임·다중 lifting-size 동시 permutation | 대응 없음 | Prime ECC는 단일 H-matrix/PCM 기반 고정 구조(`ecc_top.cpp` `Load_PCM()`)이며 5G 스타일 multi-standard 유연성 미보유 |
| 무충돌 경로 생성을 위한 CSP/제약해결기 오프라인 설계 | 대응 없음 | Prime ECC HW 라우팅(GT.cpp MUX 기반 edge 선택)과 목적은 유사하나(자원 충돌 회피) 방법론(CSP solver)은 무관 |
| 제어신호 ROM 저장(오프라인 계산, 런타임 무연산) | `GT.cpp` `Make_GT_HW()` | GT의 MUX 기반 edge 선택 테이블과 "오프라인 계산 후 HW에 고정 테이블로 저장"이라는 설계 철학은 유사 |

> 적용 가치: 낮음 — Prime ECC는 고정 lifting(z_sb=32) 단일 코드 구조라 가변 크기·다중 프레임 인터커넥트 문제 자체가 발생하지 않으며, 이식 시 구조적 재설계 부담이 이득 대비 큼.
