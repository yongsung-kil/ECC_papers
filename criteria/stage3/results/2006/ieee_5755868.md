### B. 알고리즘 요약

1. AWGN 채널, BPSK, irregular LDPC 코드(rate `R=0.5`, check-regular `dc=9~10`, block length `N=10^5`)를 대상으로 Sum-Min(min-sum) 근사 알고리즘의 성능 개선을 다룸.
2. 문제: Sum-Min은 Sum-Product 대비 regular 코드에서 약 0.5dB, irregular 코드에서는 최대 1.0dB 성능 손실 발생 — check node 출력(minimum)이 정확한 LLR이 아니기 때문.
3. 핵심 가정: EXIT 함수는 bijective mapping에 불변이므로, minimum을 올바른 LLR로 변환하는 일대일 사상(post-processing)이 존재한다고 가정.
4. 핵심 기법 1단: minimum `L_SMA`를 관측값으로 재해석하여 사후 LLR `L_mn = f(L_SMA; d_c, p(·))`를 계산하는 최적 후처리 함수를 유도(정규 코드용 식 (2), irregular 확장은 입력 메시지 밀도가 variable node degree마다 다름을 고려한 벡터 밀도 `p(·)` 버전).
5. 핵심 식의 의미: 정규 코드는 모든 입력 메시지 밀도가 동일해 단일 lookup table로 구현 가능하지만, irregular 코드는 check node/iteration마다 함수가 달라져 정확한 구현이 사실상 불가능(조합 폭발).
6. 확장/근사: 평균 입력 메시지 밀도로 단순화하고, 최적 비선형 함수를 최소제곱 근사로 iteration별 선형 스케일 팩터 `α` (식 (3)) 또는 오프셋 `β` (식 (4))로 대체.
7. 구현 디테일: irregular 코드는 EXIT 곡선 사이 "narrow tunnel" 특성 때문에 정규 코드처럼 단일 고정 팩터를 쓰면 decoder가 도중에 멈춤 → iteration마다 다른 α 시퀀스 필요(고정 α=0.66/0.72 비교 실험으로 입증).
8. 학습/최적화 절차: 가중 MMSE(weighted minimum mean squared error)로 각 iteration의 최적 α, β를 사전 계산.
9. 결과: 정규 (3,6) 코드는 고정 α=0.80으로 Sum-Product 대비 gap 0.05dB; irregular 코드(check-regular dc=9)는 iteration별 가변 α로 gap 0.15dB (고정 α는 error floor 발생 또는 threshold 악화).
10. 한계: 순수 시뮬레이션/이론 검증(N=10^5, cycle 영향 배제 목적)이며 HW 설계 없음, 최적 비선형 함수 자체는 계산 불가능해 근사(선형/오프셋)로만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum check node 후처리 스케일링/오프셋 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Min-Sum magnitude 계산부에 스케일/오프셋 팩터 적용 가능 지점이나 이미 유사 보정 로직 보유 가능성 |
| iteration별 가변 스케일 팩터 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR threshold 테이블과 개념적으로 유사(iteration 구간별 파라미터) |
| irregular code degree distribution 최적화 | `ecc_top.cpp` `Load_PCM()` | irregular 부호 구조는 H-matrix 로드 시점 문제, Prime ECC는 고정 QC-LDPC라 재검증 부담 큼 |

적용 가치: **중간** — min-sum 후처리 개선은 Prime ECC의 관심 영역(§5 "관심 높음: min-sum 성능 개선")에 해당하나, 구체 기법(iteration별 스케일 팩터)이 이미 적응형 LLR 테이블로 상당 부분 커버될 가능성이 높고 irregular code 자체는 Prime ECC 구조(고정 QC-LDPC)와 거리가 있음.
