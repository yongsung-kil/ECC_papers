### B. 알고리즘 요약

1. 대상 채널은 자기기록 PR(partial response) 채널(EPR4)이며, LDPC(rate 8/9, length 4617~4923) 또는 TPC(rate 0.935, 899→960 bits)를 평가.
2. 반복복호 코드의 매우 낮은 BER(~10^-10) 영역 성능을 C 시뮬레이션으로 확인하려면 수개월이 걸려, 이를 FPGA로 가속하는 것이 목적.
3. 채널모델은 AWGN 기반 EPR4 convolution + turbo equalization(채널검출-복호 반복).
4. LDPC 디코더는 재구성 가능(reconfigurable) 구조로, H-matrix를 소형 규칙 서브행렬로 분할 가능한 임의의 LDPC 코드에 적용 가능.
5. 소프트 채널검출기로 bidirectional SOVA를 채택(MAP/Log-Map 대비 정확도-복잡도 절충), 6비트(4:2 포맷) 메시지 양자화 사용.
6. 조기종료: syndrome check 만족 시 복호 중단(throughput 향상).
7. TPC 인코더는 shift register + XOR array로 단순 구현, 1/(1+D) precoder로 거리 스펙트럼 개선 시도.
8. 별도 학습/최적화 절차 없음(파라미터화된 설계를 CAD 툴로 코드별 합성).
9. 결과: array code rate 8/9, length 4617에서 2.7 Gbps/iteration throughput, 2시간 내 BER 10^-10까지 도달. TPC+RS 결합은 동일 redundancy 대비 coding gain 없음(marginal).
10. 한계: 디코더 알고리즘 자체는 표준 SPA/min-sum 계열로 신규성 없음(범용 평가 플랫폼이 기여점), NAND가 아닌 HDD 자기기록 채널 전용, 부호설계 세부사항(H-matrix 생성법) 본문에 미기재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 재구성가능 LDPC 디코더(범용 H-matrix 지원) | `decoder.cpp` `LDPC_Decoder()` | 구조적 유사성 있으나 알고리즘 신규 기여 없음, 평가용 플랫폼일 뿐 |
| syndrome 기반 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념은 겹치나 syndrome check 방식으로 CRC와 상이 |
| bidirectional SOVA 채널검출기 | 대응 없음 | Prime ECC는 채널검출기(트렐리스 기반) 모듈 없음, NAND read/LLR 양자화와 무관 |
| TPC 인코더/디코더 | 대응 없음 | LDPC 전용 코드베이스, TPC 미지원 |

> 적용 가치: 낮음 — HDD 자기기록 채널 대상 FPGA 평가 플랫폼이며 디코더 알고리즘 자체의 개선이 없어 NAND binary QC-LDPC 이식 가치 낮음.
