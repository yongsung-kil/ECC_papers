### B. 알고리즘 요약

1. 대상: 고rate QC-LDPC 코드(예: (4608,4096) rate-8/9 regular, variable degree 3/4), H matrix는 circulant permutation 서브매트릭스로 구성.
2. 문제: 대형 LDPC 코드는 메모리가 하드웨어 전체를 지배하는데, 기존 partially parallel 설계는 extrinsic message 저장에 큰 메모리가 필요함.
3. 채널/알고리즘 가정: modified Min-Sum 알고리즘(BP 근사, 채널 파라미터 불필요) 사용, check-to-variable message를 압축형으로 표현.
4. 핵심 기법: check-to-variable message `R_cv^(k)`를 (최소크기, 두 번째 최소크기, 최소값 위치 인덱스, 부호들) 4개 요소로 압축 저장하는 rearranged decoding procedure - 식 `R_cv^(k) = α × min_{n∈N(c)\v}|S_n^(k-1) - R_n^(k-1)| × sign(...)`.
5. 이 압축표현으로 column sum `S_v^(k)` 갱신과 R 메시지 재생성을 분리해 R-memory/S-memory/I-memory로 나누고, 전통 설계(258k bits) 대비 R/S/I-memory 총합을 164k bits로 절감(약 36%).
6. 확장: H matrix가 weight-2 circulant(2개 순환행렬 중첩)를 일부 포함하는 일반적 경우, block column을 재배치하고 weight-1+zero 서브매트릭스로 분해해 동일 병렬 처리 기법을 적용(CPUa/CPUb, 2개 VPU, merge unit 사용).
7. 구현 디테일: p-parallel 처리를 위해 인접 p개 row/column 메시지를 한 메모리 엔트리에 저장, barrel shifter 기반 data scheduling/reverse scheduling unit을 저비용 W:1 멀티플렉서 스위칭 블록으로 최적화(기존 cyclic shifter 대비 하드웨어/지연 절감).
8. 별도 학습/최적화 절차 없음(아키텍처·회로 설계 논문).
9. 결과: (4608,4096) (4,36)-regular QC-LDPC, 6-bit 양자화, dual-port memory 가정 시 메모리 258k bits → 164k bits(약 36% 절감, 논문상 "30% 이상"). 정정성능(BER/FER)은 배경 설명(Fig.1)일 뿐 본 제안 구조의 성능 개선 수치는 아님.
10. 한계: FPGA/ASIC 합성 결과나 throughput 실측치 제시 없이 메모리 크기 계산 위주의 이론적 제안, weight-2 서브매트릭스 비중이 큰 H-matrix에는 부적합.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified Min-Sum 압축 메시지(min1/min2+sign+index) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2 기반 min-sum을 사용 - 메시지 압축·재생성 구조가 직접 대응 |
| R/S/I-memory 분리 및 메모리 절감 구조 | 디코더 HW 모델 (z=32, HCU, GT) | 메모리 뱅크/라우팅 최적화 아이디어로 참고 가능, 구조 정합 필요 |
| data scheduling / reverse scheduling unit (barrel shifter, W:1 MUX) | `GT.cpp` `Make_GT_HW()` | circulant shift 기반 데이터 정렬 로직으로 edge 선택/라우팅과 유사한 목적 |
| weight-2 circulant 분해(CPUa/CPUb, merge unit) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | HCU(비대칭 degree column 2-cycle 처리)와 유사한 문제의식 - 이종 degree 서브매트릭스 처리 |
| H-matrix 구조(고rate QC-LDPC, circulant) | `ecc_top.cpp` `Load_PCM()` | Prime ECC의 base+shift QC-LDPC 구조와 유사, 고rate라는 점도 부합 |

> 적용 가치: 중간 | Min-Sum 압축 메시지 표현과 메모리 절감 구조는 Prime ECC가 이미 유사 개념(min1/min2, HCU)을 보유해 중복도가 있으나, data scheduling unit의 저비용 MUX 기반 circulant shift 최적화는 GT/메모리 라우팅 개선에 참고 가치가 있음. 다만 실측/합성 검증이 없어 신뢰도는 제한적.
