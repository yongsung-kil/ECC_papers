### B. 알고리즘 요약

1. Density Evolution(DE)은 LDPC 코드의 belief propagation 복호 성능을 앙상블 수준에서 예측하는 설계 도구이며, 기존 exit-chart 등 근사법 대신 DE를 정확하고 빠르게 계산하는 것이 목표.
2. DE는 variable domain(`R ∪ {+∞}`의 LLR 컨볼루션)과 check domain(`GF[2]×[0,∞]`의 컨볼루션) 두 종류의 컨볼루션 연산이 필요하며 기존 방법은 각 pairwise 컨볼루션마다 forward FFT 2회+inverse FFT 1회가 필요해 aliasing 문제와 계산비용이 큼.
3. 핵심 가정: 대칭 채널(BSC/AWGNC) 하 all-zero 코드워드 전송 가정, LLR 밀도 `f(x)`의 대칭성(`f(-x)=e^{-x}f(x)`) 활용.
4. 핵심 기법: `f(1+iw)` (Laplace 변환 유사 functional)를 직접 다뤄 variable node 업데이트 시 tail probability mass가 지수적으로 작아지도록 만들어 aliasing 없이 pairwise 컨볼루션을 연쇄 수행, variable node 업데이트가 끝날 때만 FFT 1회로 실도메인 전환.
5. 핵심 근사(AppBP): check node에서 `log coth(z/2)` 대신 `2e^{-|z|}`로 근사(정리 1로 상한 증명), 이 근사로 check-to-variable 메시지 크기가 축소되지만 계산은 pointwise 곱셈으로 단순화됨.
6. BP(정확) 버전은 비선형 변환 `Ψ(x)=exp(-2e^{-x})` (정리 2)을 통해 AppBP 컨볼루션 구조를 재사용, check node 업데이트도 결국 컨볼루션 형태로 환원.
7. 이산화(discretization) 세부: LLR 샘플 간격 `δ`, 주파수 스텝 `w=0.6/δ`로 설정(δ=1/2,1/4,1/8,1/16에서 검증), 역변환 커널 `Λ(x)`를 `∫x²Λ(x)dx` 최소화하는 선형계획 문제로 설계.
8. 최적화 절차: 역양자화 커널 `Λ`의 최적 설계가 선형계획법(LP) 문제로 공식화되며, `δ→0`일 때 A가 full density evolution으로 수렴함을 증명.
9. 결과: (3,6)-regular 코드에 대해 BSC/AWGN 채널의 BP/AppBP threshold를 channel capacity 단위로 계산(표1), δ=1/16에서 threshold 오차가 수 mdB 이내이며 AppBP-BP 격차는 약 0.01dB.
10. 한계: 순수 asymptotic(n→∞) 앙상블 설계 도구이며, 실제 유한장 코드 디코더/HW 구현이나 시뮬레이션 결과가 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Density Evolution 계산 기법 | 대응 없음 | Prime ECC는 특정 QC-LDPC H-matrix가 고정된 시뮬레이터로, 앙상블 설계용 DE 도구를 사용하지 않음 |
| AppBP 근사(log coth 대신 2e^-|z|) | 대응 없음 | Prime ECC는 min-sum 근사를 이미 사용하며 이 논문의 exp 기반 근사와 다른 계열 |

> 적용 가치: **낮음** — 코드 앙상블의 asymptotic threshold를 계산하는 설계 단계 수치해석 도구이며, Prime ECC의 실제 디코더 알고리즘이나 HW 구조에 이식할 요소가 없음.
