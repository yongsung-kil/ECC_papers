### B. 알고리즘 요약

1. 시스템: 자기기록(perpendicular magnetic recording) 채널, NPSOVA(채널 APP 디코더) + LDPC 반복복호 구조, 코드 길이 `N=1309`, 서브행렬 크기 `q=88`.
2. 문제: 기존 array code(단일 순환시프트 행렬로 구성)는 ISI 채널에서 인접 비트가 강하게 상관되어, 다중 비트 오류가 패리티체크 행렬을 통해 쉽게 연결되며 이는 4-cycle처럼 작용하는 "pseudo-cycle"을 만들어 반복복호 성능을 저하시킴.
3. 채널 모델: white Gaussian noise + media noise(jitter/fluctuation/DC erase noise, media noise 비중 90%)를 포함한 PMR(perpendicular magnetic recording) 채널, 정규화 기록밀도 `Dc=1.5`.
4. 핵심 기법: 패리티체크 행렬 `H=[H1 | H2]`에서 서브행렬 `H1`을 파생 행렬 `B`에 무작위에 가까운 열 순열(quasi-cyclic, shift `s>1`)로 인터리빙하여 "Random Interleaved Array (RIA) code" 구성.
5. 핵심 식 의미: 단일 순환시프트가 아닌 인터리빙된 순열구조를 사용해 4-cycle 자체는 여전히 존재(girth 4)하지만, pseudo-cycle의 개수를 줄여 반복복호 시 extrinsic information 갱신 왜곡을 완화.
6. 확장: RIA code를 rate-16/17 QMTR(quasi-MTR, k=3 제약) 코드와 reverse concatenation(RS-ECC 포함)하여 채널의 긴 오류 이벤트(long error event)를 원천 제거.
7. 구현 디테일: 첫 행의 1 위치를 신중히 선택(예: q=88일 때 {0,11,33})하여 역행렬이 sparse가 되도록 해 저복잡도 인코딩 가능.
8. 최적화 절차: 별도 학습/최적화 없음 (수동 파라미터 선택 및 pseudo-cycle 개수 비교를 통한 설계 검증).
9. 결과: SER 10^-5에서 PRML+HR-RLL 대비 1.2dB 이득, RA+array code 대비 error-floor 크게 개선, girth 4임에도 pseudo-cycle 수 감소로 BER 개선 확인.
10. 한계: 시뮬레이션(자기기록 채널)만 수행, HW 미설계, MTR/RS-ECC와 결합된 특정 시스템 전제로 일반 NAND 채널에는 직접 적용 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RIA code 구성(무작위 인터리빙 QC array code) | `ecc_top.cpp` `Load_PCM()` | 자기기록 채널 전용 pseudo-cycle 대책으로 NAND H-matrix 설계와 목적이 다름 |
| pseudo-cycle 분석(4-cycle 카운트) | 대응 없음 | Prime ECC는 QC-LDPC H-matrix가 이미 고정되어 있어 girth/pseudo-cycle 재설계 여지 낮음 |
| QMTR/RS-ECC reverse concatenation | 대응 없음 | 자기기록 채널 특유의 RLL/MTR 코딩으로 NAND 컨트롤러 구조와 무관 |

적용 가치: 낮음 — 자기기록 채널 특유의 pseudo-cycle 문제와 MTR/RS-ECC 결합 시스템으로, NAND binary QC-LDPC 부호설계에 이식할 요소가 거의 없음.
