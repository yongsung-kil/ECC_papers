### B. 알고리즘 요약

1. Irregular LDPC 코드(M×N parity check matrix H)를 Tanner-graph로 표현, IN(정보노드)/PN(패리티노드)/CN(체크노드)으로 분리.
2. 기존 PEG(edge-by-edge) 또는 permuted identity matrix 방식은 HW 구현 효율성과 통신성능을 동시에 만족시키기 어려움.
3. 병렬화 계수 `P`개 단위로 노드를 묶어 HW 매핑 효율을 높이는 것을 목표로 함.
4. 핵심기법: 2-Vector(2V) 표현 - degree allocation vector `DA`와 cyclic shift vector `CS`로 parity check matrix 전체를 기술. `Π(i) = Ψ(DA(mod(i,A)))*P + mod(i/A - Φ(CS(mod(i,A))), P)`.
5. `Ψ`는 relative prime(RP) interleaver로 서로 다른 IN 서브셋이 하나의 parity grouping에 참여하도록 보장, `Φ`는 girth 최대화를 위한 cyclic shift 최적화.
6. 2V-PEG 알고리즘: 한 번에 P개 edge를 배치(girth histogram `GS` 평균 최대화)하여 기존 PEG 대비 계산량을 N/P로 축소.
7. 인코더는 puncturing 기반 type1(반복+parity grouping+2-state RSC accumulator)/type2(동일 accumulator+puncturing) 두 구조의 duality를 활용, hybrid-ARQ에 적합.
8. 최적화 절차: Ψ 고정 후 Φ만 각 위치별로 P가지 shift값 중 girth histogram 평균이 최대인 값을 그리디 탐색.
9. 결과: K=1600, R=0.33~0.8 AWGN FER 시뮬 제시, 0.13µm ASIC에서 decoder 3.957mm², encoder 0.121mm², throughput 최대 522Mbps(R=9/10, 10회 반복)~63Mbps(R=1/4, 50회 반복).
10. 한계: 통신 표준(DVB-S2, WiMAX, hybrid-ARQ) 타깃 설계로 NAND 채널모델·soft-read LLR 양자화 언급 없음, 시뮬은 AWGN 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2V(DA/CS) Tanner-graph 구성, 2V-PEG 알고리즘 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 레이어에 대응하나 QC-LDPC 고정 구조 재검증 부담 큼 |
| P개 단위 병렬 노드 매핑(functional unit, message RAM) | 디코더 HW 모델 (z=32 병렬, HCU) | 병렬화 단위 개념은 유사하나 GT/HCU와 직접 매핑 어려움 |
| puncturing 기반 hybrid-ARQ 인코더(type1/type2 duality) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | NAND ECC는 hybrid-ARQ 재전송 미사용, 대응 낮음 |
| shuffling 기반 permutation network | `GT.cpp` `Make_GT_HW()` | edge 라우팅 개념 유사하나 목적(girth 최적화 vs edge 축소) 상이 |

> 적용 가치: 낮음 - 통신 표준(hybrid-ARQ) 타깃 irregular LDPC 부호구성법으로, NAND binary QC-LDPC 고정 구조와 직접 연계되는 디코더/HW 기법(min-sum, iteration 감소 등)은 다루지 않음.
