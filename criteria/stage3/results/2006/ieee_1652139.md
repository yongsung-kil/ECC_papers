> 총평: LDPC 기반 Tornado Code를 디스크 archival storage의 erasure coding(RAID 대체)에 적용한 시스템/실험 논문으로, NAND binary LDPC ECC 디코더·부호설계와는 채널 모델과 복호 알고리즘이 전혀 다름.

### B. 알고리즘 요약

1. 시스템: 96-node cascaded bipartite LDPC 그래프(Tornado Code) 기반 archival storage, 각 노드는 디스크/사이트에 대응, 인코딩 시 원본과 동일 크기의 parity 생성(overhead ~50%, storage rate 0.5).
2. 문제: RAID/mirroring 등 기존 parity 기반 방식은 소수 디스크 동시 실패에도 데이터 손실이 발생하며 replication은 erasure coding 대비 낮은 fault tolerance를 제공.
3. 모델: 디스크(노드) 소실을 binary erasure로 간주, cascaded bipartite 그래프에서 각 right(parity) 노드는 정확히 1개의 missing left 노드만 있을 때 peeling 방식으로 복구 가능.
4. 핵심 기법: 96-node 그래프에 대해 (96 choose k) 조합 전수조사(k=1~6)로 worst-case failure(그래프가 복구 불가능해지는 최소 소실 노드 수)를 탐지, k가 큰 구간은 무작위 샘플링(최대 3400만 케이스)으로 재구성 실패율 추정.
5. 핵심 절차의 의미: closed set(소수 left 노드가 소수 right 노드에만 연결된 폐쇄 집합)이 그래프 실패의 원인이므로 이를 탐지·제거하면 fault tolerance가 향상됨.
6. 확장 기법: 실패에 관여한 critical left node의 연결을 실패에 관여하지 않은 right node로 재배선하는 수동/피드백 기반 그래프 조정(graph adjustment)으로 first-failure 노드 수를 4→5로 증가.
7. 부수 디테일: 그래프 마지막 두 단계는 left node 집합을 공유하도록 구성(Typhoon 방식 차용), degree 분포는 edge 단위로 설계 후 노드 수 맞춤을 위한 수치적 constant multiplier 적용.
8. 최적화 절차: 그리드서치가 아닌 실패 패턴 기반 반복 조정(defect detection → 실패 사례 분석 → 재배선 → 재검증)이며 별도 DE/GA/NN 없음.
9. 결과: 조정된 최적 Tornado 그래프는 first failure 5노드(96개 중), 96-disk 시스템 이론적 실패확률이 mirrored 4.79e-3 대비 Tornado 5.86e-10~1.35e-9로 대폭 개선; 복수 사이트에 상보적 그래프 사용 시 first failure가 4→17~19 노드로 확장.
10. 한계: 순수 시뮬레이션(전수조사+몬테카를로)뿐이며 인코딩/디코딩 throughput, HW, 실제 스토리지 프로토타입 미구현; 채널이 disk-level erasure(존재/부재)이지 NAND 채널의 bit-flip/soft-LLR과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Tornado Code 그래프 구성(cascaded bipartite, degree distribution) | 대응 없음 | binary erasure code 구성법으로 QC-LDPC binary error-correction 부호설계와 목적·구조가 다름 |
| Peeling 복호(1개 missing left node일 때만 복구) | 대응 없음 | Prime ECC는 min-sum 기반 soft/hard iterative message-passing이며 peeling 알고리즘 자체가 없음 |
| Worst-case failure 조합 탐색·그래프 조정 | 대응 없음 | H-matrix 최적화 절차와 유사한 발상이나 erasure channel 전용이라 `ecc_top.cpp Load_PCM()`과 목적이 다름(코드율/최소거리 최적화 아님) |
| 다중 사이트 상보 그래프(distributed reliability) | 대응 없음 | 시스템 레벨(스토리지 아키텍처) 개념으로 ECC 코드베이스 범주 밖 |

> 적용 가치: 낮음 - erasure code(디스크 소실) 문제이며 binary error-correction(bit-flip) LDPC와 채널 모델·복호 알고리즘이 근본적으로 다르고, HW/soft-decision/min-sum 등 Prime ECC의 어떤 모듈과도 직접 대응되지 않음.
