### B. 알고리즘 요약

1. 채널: AWGN, binary linear code(LDPC) ML 디코딩을 정수계획법(IP) 문제로 정식화.
2. 문제의식: 기존 SP(sum-product) 디코더는 짧은 길이/특정 regular 구조 LDPC에서 수렴 실패·성능 저하, 휴리스틱이라 분석 어려움.
3. 배경 모델: LP relaxation 기반 LP decoder가 ML-certificate 속성(ML 코드워드 출력 또는 오류 선언)을 가짐을 활용.
4. 핵심 기법 1단(Multistage LP): branch-and-bound 방식으로 LP 디코더가 오류를 보고하면, 신뢰도 낮은 비트 집합 `S`를 골라 재귀적으로 `2^|S|`개 LP를 추가로 풀어 pseudocodeword 그룹을 확장.
5. 핵심 식: pseudocodeword 그룹의 목적값이 특정 조건을 만족하면 ML 코드워드로 판정(Proposition 1/2) — 추가 LP 풀이로 결과를 정제하는 근거.
6. 핵심 기법 2단(Box-constrained QP): 원래 LP 제약을 Lagrange dual factor `α`로 목적함수에 흡수해 `min (1/2)x^T Q x + c^T x`, `x∈[0,1]^n` 형태의 QP로 변환.
7. 부수 트릭: QP를 1차원 minimization 문제 `n`개로 분해해 PSOR(projected successive overrelaxation, 순차) 및 parallel gradient projection(병렬) 알고리즘으로 선형시간 해결, LDPC 희소성 활용해 연산 축소.
8. 최적화 절차: PSOR은 weight factor `ω≤2`에서 수렴(Proposition 5), parallel 방법은 `β<2/ρ(Q)`(스펙트럴 반경 기반) 조건에서 수렴.
9. 결과: length=1024 product code에서 QP(PSOR)가 7 iteration만에 수렴, iteration당 SP 대비 덧셈 연산만 필요(곱셈·tanh/arctanh lookup 불필요)해 연산량 감소; OVD 디코더가 SP보다 waterfall 영역에서 더 가파르게 하강.
10. 한계: 시뮬레이션(MATLAB/COIN-OR)만 수행, HW 미설계, 긴 길이 LDPC에서는 LP 계열이 SP보다 열세.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP/QP 기반 디코딩 알고리즘 전체 | 대응 없음 | min-sum 기반 `decoder.cpp` `LDPC_Decoder()`와 알고리즘 패러다임 자체가 다름(최적화 solver 기반) |
| box-constrained QP 반복 갱신 | 대응 없음 | 반복 연산이지만 CN/VN 메시지 패싱이 아닌 QP 좌표하강/투영 연산 |
| product code (5-dim SPC) 실험 | 대응 없음 | QC-LDPC 구조가 아님, 부호설계와 무관 |

> 적용 가치: 낮음. LP/QP relaxation 기반 디코딩 패러다임은 Prime ECC의 min-sum HW 아키텍처와 근본적으로 이질적이며, 검증도 소규모 시뮬레이션에 그침.
