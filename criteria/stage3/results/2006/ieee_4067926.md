### B. 알고리즘 요약

1. AWGN 채널, `x∈{-1,1}` 이진 입력, LDPC 코드의 Tanner graph 상 message-passing 기반 Density Evolution(DE) 분석 대상.
2. 기존 Discretized DE(DDE)는 정확하지만 복잡도 `O(2^2k)`로 코드 최적화 cost function으로 쓰기엔 너무 무거움; Gaussian approximation은 낮은 SNR·고차수 variable node에서 부정확.
3. variable node LLR 메시지 분포가 근사적으로 Gaussian이며 consistency 조건(`p(-x)=e^-x p(x)`)을 만족한다고 가정.
4. Gram-Charlier 급수의 홀수항을 제거한 `p_Lv(lv) = (1/√4πβ) e^{-(lv-β)²/4β} Σ a_k lv^{2k}` 형태로 variable node 메시지 분포를 모델링, 미지수 `[β, a]`를 Newton-Raphson으로 첫 N개 모멘트에 매칭.
5. 모멘트 매칭이 필요한 이유: 정확한 pdf 계산은 닫힌 해가 없으므로, 모멘트만 맞춰 EXIT chart(Message Error Probability, MEP)를 근사 산출.
6. check node LLR 모멘트 `ω_n`은 tanh-rule 기반 변수 `u=tanh(l_c/2)`의 Taylor 전개로 계산, variable node 모멘트는 Gauss-Hermite quadrature로 적분.
7. 부수 트릭: 4개 모멘트만으로도 충분한 정확도를 얻음(수치실험 확인), 복잡도는 `O(n³)` (n=모멘트 수)로 DDE의 `O(2^2k)`보다 낮음.
8. 알고리즘은 8단계 반복 루프: 모멘트 계산→Newton-Raphson 파라미터 갱신→MEP 계산→tanh 기댓값 갱신→check node 모멘트 갱신, 최대 반복수까지 순환.
9. 결과: (3,6) regular R=1/2 코드에서 threshold 오차 Gaussian approx 0.06dB → 제안법 0.02dB; irregular R=1/3 코드에서 0.25dB → 0.082dB로 개선.
10. 한계: 순수 이론적 분석/설계 도구(threshold 추정)이며 실제 디코더 구현, HW, 검증(FPGA/ASIC) 전혀 없음. 특정 코드 앙상블(2개 예제)에서만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Density Evolution 근사(EXIT chart 계산) | 대응 없음 | Prime ECC는 실시간 HW 디코더 시뮬레이터로 코드 설계용 DE 도구는 포함하지 않음 |
| tanh-rule check node 모멘트 | 대응 없음 | Sum-Product의 tanh 자체를 다루나 Prime ECC는 이미 min-sum으로 근사해 사용 중이라 무관 |
