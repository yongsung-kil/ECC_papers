### B. 알고리즘 요약

1. 시스템: categorized(multi-edge-type) LDPC 코드 3종(R=1/3, IRA, R=1/2), interleaved length 5120, OFDM 기반 무선 이동통신 채널(AWGN + 18-path Rayleigh 페이딩, Jakes model, fd=20Hz).
2. 문제: irregular LDPC는 성능 좋으나 프레임 길이/차수가 커 모바일 기기엔 복잡도 과다. 이를 줄이려 sum-product 대신 min-sum(Max-Log 근사)을 쓰면 error floor 발생.
3. 채널 가정: 수신 신호를 노이즈 분산 `σ²`으로 정규화하지 않고 `y·h*`(수신신호×페이딩 켤레)를 그대로 사용 - 채널 상태 정보(CSI) 불필요.
4. 핵심 기법: edge type(카테고리) `i`마다 서로 다른 계수 `mcoef(i)`를 variable-node LLR 갱신식(식 3/4)에 곱하는 multi-coefficient 방법. 표준 정규화 BP(단일 계수)와 달리 카테고리별 개별 계수 사용.
5. 핵심 식: check node는 min-sum 근사(`U(i)=min|V|·Πsgn`, 식 2·10), variable node는 `V(i)(k)=Uini,n + mcoef(i)·U(i)(k) + Σj mcoef(j)·U(j)(k)`(식 4) - 카테고리별 계수가 근사 오차를 category-aware하게 보정.
6. 확장: Max-Log 도메인 Gaussian Approximation(GA)으로 카테고리별 평균/분산(`mi→mcoef(i)·mi`, `σ²i→mcoef(i)²·σ²i`, 식 11)을 추적해 최적 계수를 사전 추정.
7. 부수 트릭: 계수값이 "1"인 카테고리는 곱셈 생략 가능(복잡도 절감), single-coefficient 정규화 BP는 이 방법의 특수(카테고리=1개) 경우로 포함.
8. 학습/최적화: GA로 대략적 계수를 먼저 구하고(rough estimation) Monte Carlo 시뮬레이션으로 GA 결과 주변을 fine-tuning하는 2단계 절차.
9. 결과: R=1/3에서 다중계수(1.0, 0.58, 0.69) 적용 시 단일계수(0.81) 대비 error floor 소멸, 페이딩 채널에서 이상적 BP(완전 CSI) 대비 성능 근접, 단일계수 대비 R=1/3에서 3dB, R=1/2에서 1dB 이득.
10. 한계: 표준 랜덤 degree irregular LDPC엔 적용 불가(categorized 구조 필수), HW 구현·게이트카운트 없음, 무선 다중경로 페이딩/OFDM 환경 전용 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum(Max-Log) CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 카테고리별 계수 개념을 CN magnitude 계산에 결합 가능성 있음 |
| category(edge type)별 정규화 계수(mcoef) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` (양자화 테이블) | Prime ECC의 iteration별 LLR threshold 테이블과 유사한 위치이나 "category별" 축은 없음, 확장 시 테이블 차원 추가 필요 |
| GA 기반 계수 오프라인 설계 | `local_opt.cpp` (LLR 자동최적화) | 오프라인 파라미터 튜닝이라는 점에서 유사 목적의 모듈 존재 |
| OFDM/다중경로 페이딩 채널 모델 | 대응 없음 | Prime ECC는 NAND 채널(AWGN/RBER/fixed-error)만 지원, 무선 페이딩과 무관 |

> 적용 가치: 중간 — min-sum error-floor 개선이라는 목표는 Prime ECC의 핵심 관심사와 일치하나, categorized Tanner graph 구조 및 GA 기반 오프라인 설계 절차를 QC-LDPC 구조에 재정의해야 이식 가능.
