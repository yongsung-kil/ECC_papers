### B. 알고리즘 요약

1. 시스템은 BIAWGN 채널 + BPSK, 짧은 (n=64, k=44, rate 0.6875) irregular LDPC(PEG 구성)를 대상.
2. IDBP(BP/SPA)는 최고 성능이나 반복마다 실수 연산(덧셈/곱셈/지수/로그)이 많아 복잡도가 크다는 문제를 해결하려 함.
3. 고SNR에서는 최우도 codeword가 수신벡터 주변 sphere 내에 높은 확률로 존재한다는 가정을 사용.
4. reliability vector `ai = ln(P(ci=1|yi)/P(ci=0|yi))`를 계산하고 신뢰도가 낮은 위치부터 test error pattern `e(r)`을 순차 생성, `zH ⊕ e(r)`을 Gallager BF(수정판)로 하드디코딩.
5. optimality test criterion(OTC, 식 6): correlation discrepancy `I(y,c) ≤ Σ|a_{Do(c)}|`이면 더 나은 codeword가 없음을 보장, 이 조건 만족 시 조기 종료.
6. Step 4~5에서 unsatisfied parity check 수가 최대인 bit 집합 B를 flip하는 것이 BF 변형의 핵심(다수결 기반 flip).
7. 파라미터 δ(design parameter)로 이미 생성된 codeword와의 Hamming distance 최소값을 제어, 최대 candidate 수 `Nc`로 조기 종료.
8. 별도 학습/최적화 없음. 코드 자체는 PEG 알고리즘으로 사전 구성.
9. n=64 코드에서 `Nc=8` 이상은 성능 향상이 미미했고, IWBF/WBF/MWBF 대비 우수하며 IDBP와 유사한 BER/FER을 훨씬 낮은 복잡도로 달성.
10. 복잡도가 수신 시퀀스 상태에 확률적으로 의존(깨끗한 신호는 candidate 적음)하고, 검증은 시뮬레이션뿐이며 HW 구현이나 긴 코드에서의 확장성은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager BF 변형 하드디코딩 | 대응 없음 | Prime ECC는 min-sum soft 디코더 중심이라 BF 계열 하드디코딩 알고리즘 자체는 대응 모듈 없음 |
| reliability 기반 test pattern 생성/list decoding | 대응 없음 | 가변 latency의 list-decoding 구조는 고정 파이프라인 HW 디코더와 근본적으로 불일치 |
| optimality test criterion(조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료라는 목적만 유사, 구현 방식(상관 판별 vs CRC)은 무관 |

적용 가치: 낮음 — 짧은 코드 전용 list-BF 디코딩으로 정정능력 개선이 아니라 복잡도-성능 트레이드오프 기법이며, 가변 복잡도/latency가 NAND HW 파이프라인 요구와 맞지 않음.
