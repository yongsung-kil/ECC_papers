> 총평: 순수 부호설계·이론 논문으로 girth-cycle을 지배하는 선형방정식 기반 shortening 기법을 통해 array LDPC의 error-floor/waterfall 성능을 개선하지만, HW 미설계이며 대상 부호가 저rate(0.5 내외)·저밀도 array code로 Prime ECC의 고rate QC-LDPC 구조와는 상당한 차이가 있어 부호설계 아이디어(girth 최적화) 수준의 참고만 가능.

### B. 알고리즘 요약

1. 대상은 array code(quasi-cyclic LDPC의 부분클래스): parity-check 행렬이 q×q circulant permutation 행렬 블록들로 구성되며, block-row label 수열 `a0,...,a_{r-1}`로 정의(등차수열이면 PAC, 아니면 IAC). column-weight `r`(=3 또는 4), 길이 `q·s`(s=유지할 block-column 수), rate `≥1-r/s`.
2. 랜덤 구조 LDPC는 인코딩 복잡도·저장공간이 크고, 대수적(algebraic) 구조 LDPC(array code 등)는 이 문제를 해결하지만 girth가 짧아 성능(특히 error-floor)이 저하되는 문제를 다룸.
3. 채널은 BPSK+AWGN, belief-propagation(sum-product) 반복복호(최대 30~50 iteration) 가정.
4. 핵심 기법: Tanner graph의 length-`2k` 사이클 존재 여부는 block-row/column 인덱스에 대한 동차 선형방정식 `Σc_i·ν_i ≡ 0 (mod q)`(식 15)로 지배됨을 증명(Theorem 1). 특정 column-weight(r=3,4)에서 길이 6/8/10 사이클을 지배하는 방정식들을 전수 나열(Table I~IV).
5. shortening된 코드의 block-column label 수열이 이 cycle-governing 방정식들의 "proper solution"을 포함하지 않도록(non-averaging/Sidon sequence 등) 열을 선별하면 해당 길이의 사이클이 완전히 제거되어 girth를 8, 10, 12까지 늘릴 수 있음.
6. rate penalty 분석: 남길 수 있는 최대 열 개수 `s(q,Ω)`는 Ramsey 이론(Roth's theorem, Behrend 방법)으로 상/하한이 유도되며, q에 대해 sublinear하게 증가(예: girth-8 PAC는 `O(q/log log q)`) → q가 커질수록 shortening rate penalty가 심해짐.
7. 구현 기법: 탐욕(greedy) 알고리즘으로 방정식 해를 피하는 정수열 생성(복잡도 `O(Lq^{M+1})`, L=방정식 개수, M=변수 최대개수), 대안으로 Bosznay 방법 기반 조밀 수열 구성 후 위반 원소 제거(expurgation).
8. 별도 학습/최적화 절차는 없으며 조합론적 탐색(greedy search)만 사용.
9. 결과: girth가 클수록 BER 곡선이 랜덤 구조 LDPC(MacKay-Davey)에 근접, girth-10 IAC(r=4, length 10456)는 50 million block·30 iteration 시뮬레이션에서 SNR 2.5dB에서 오류 미관측(BER<10^-9 추정); 길이 7288 IACr4g8이 유클리드기하 부호(길이 8190) 대비 동일 BER 달성에 필요한 SNR을 5.82dB→2dB로 대폭 개선.
10. 한계: 순수 이론+시뮬레이션 논문으로 HW 구현·게이트수·throughput 등 전혀 없음; 대상 부호가 rate 0.42~0.56의 저rate array code(row/col weight 3~4)로 NAND ECC급 고rate(0.85~0.95) QC-LDPC와는 파라미터 영역이 상이해 직접 이식보다는 girth 최적화 방법론 차용 수준.

### C. Prime ECC 관련 모듈 핀포인트

| cycle-governing 방정식 기반 girth 최적화 shortening | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 선택 단계에서 girth 개선 열 선별 기법으로 참고 가능하나 현재 코드 구조 자체 재검증 필요(이식 난이도 높음) |
| array code(circulant permutation 블록) 구조 | `ecc_data.h` `PCM_b` (base+shift 구조) | QC-LDPC circulant 구조라는 공통점은 있으나 논문은 row/col weight 3~4의 매우 저밀도 array code로 Prime ECC의 degree-17 base matrix와 상이 |
| BP(sum-product) 반복복호 성능 비교 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Min-Sum(근사) 기반이라 본 논문의 full sum-product 결과를 그대로 적용하기 어려움 |
| 부호 rate/error-floor 트레이드오프 분석 | 대응 없음 | 순수 조합론적 분석으로 대응 모듈 없음 |

> 적용 가치: 낮음 — 저rate·저밀도 array code의 girth 최적화 이론이며, HW 미설계에다 Prime ECC의 특정 고rate QC-LDPC(H-matrix 고정) 구조에 재적용하려면 부호설계 전체를 재검증해야 하는 부담이 큼.
