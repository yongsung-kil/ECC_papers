### B. 알고리즘 요약

1. 대상: irregular LDPC 코드(IN degree distribution `p3=0.52, p4=0.24, p5=0.08, p9=0.04, p31=0.12`), N=1600, R=0.125~0.5(puncturing으로 rate 조절), AWGN 채널, Log-MAP 기반 디코딩.
2. 문제: 표준 two-phase message passing(먼저 모든 VN, 다음 모든 CN 갱신)은 수렴이 느려 DVB-S2 표준에서도 최대 50 iteration 필요.
3. 핵심 관찰: staircase(band-diagonal) parity check matrix를 가진 선형 인코딩 가능 LDPC는 parity node가 zigzag 연결을 갖고, 이는 2-state RSC(누산기) 인코더 + puncturing unit과 동일 출력을 낸다는 이중성(duality)을 이용.
4. 핵심 기법: 이 인코더 이중성을 디코더에 반영해, 여러 개의 check node를 하나의 punctured 2-state trellis section(크기 `Z`)으로 묶어 Log-MAP 알고리즘으로 joint MAP 확률을 계산하는 sectionalized message passing(SMP).
5. 핵심 식: VN 업데이트 `Λk = Λch + Σ Λi` (표준 LLR합), extrinsic 정보 `Λext = Λ − ΛIN`(turbo decoding 방식과 동일한 원리로 자기확인 방지).
6. 구조적 트릭: parity 트렐리스의 종결(termination) 조건과 puncturing 조건이 서로 dual임을 이용해, 별도 forward/backward acquisition phase 없이 파티 노드 정보가 들어올 때마다 순방향/역방향 recursion을 시작 - windowing 오버헤드 회피.
7. 파라미터: section size `Z`를 IN edge들이 서로 다른 IN에 연결되도록 설정하면(예: Z=8, Z=70) true MAP에 근접, `Z=α`(grouping과 동일)이면 기존 turbo MP로 환원.
8. 별도 학습/최적화 없음(PEG로 구성된 코드 사용, 알고리즘 자체는 고정 스케줄링).
9. 결과: R=0.5에서 SMP(30 iter)가 two-phase MP(200 iter) 성능에 도달, R=0.125에서 two-phase/turbo MP는 10 iteration으로 수렴 시작도 못하나 SMP는 수렴. error floor는 SMP < turbo MP < two-phase MP 순으로 낮음.
10. 한계: staircase(zigzag) 구조를 갖는 코드에만 적용 가능(DVB-S2, IRA 계열), HW 구현은 제시하지 않고 병렬 XMAP 아키텍처가 "필요하다"고만 언급, 실제 게이트/throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sectionalized message passing 스케줄링(joint CN 처리) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 구조를 바꿔야 하므로 이식 시 대규모 리팩터링 필요, 아이디어 참고 수준 |
| staircase(zigzag) parity 구조 및 punctured trellis 인코딩 | `encoder.cpp` `Generate_PCM_encoding()`, `Column_Reordering_Dual_Update()` | Prime ECC는 dual-diagonal 인코딩을 이미 사용하나 QC-LDPC(circulant shift) 기반이라 이 논문의 zigzag 구조와 정합 필요 |
| extrinsic information 계산(turbo 원리) | `decoder.cpp` `C2V_Cal_New_Sgn()` 등 CN 갱신 로직 | min-sum 기반 Prime ECC와 Log-MAP 기반 본 논문은 연산 방식이 달라 직접 대응 어려움 |
| 조기 error floor 저감(joint CN 계산) | 대응 없음 | Prime ECC는 partial/full CRC 조기종료만 보유, error-floor 대책은 별도 |

> 적용 가치: 중간 — 수렴속도/error-floor 동시 개선이라는 목표는 Prime ECC의 관심사와 일치하나, staircase 구조 의존적 트렐리스 기법이라 QC-LDPC 기반 min-sum 디코더에 이식하려면 구조적 재설계가 필요.
