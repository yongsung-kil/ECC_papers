### B. 알고리즘 요약

1. DVB-S2 표준 QC-LDPC 코드(N=64800, rate 1/2, sub-matrix 360×360 기반 partitioned 구조) 대상 하드웨어 디코더 설계.
2. 기존 Flooding Schedule(FS)은 반복당 갱신 빈도가 낮아 수렴이 느리다는 문제를 해결하려 함.
3. 채널모델은 AWGN 가정(Es/N0 기준 시뮬레이션), Corrected Min-Sum([11] 기반) 디코딩 알고리즘 사용.
4. Horizontal Shuffle Schedule(HSS, 레이어드 스케줄링)을 적용: 체크노드별로 `u_j` 메시지를 먼저 갱신 후 즉시 해당 체크노드에 연결된 `v_j`를 갱신, 같은 반복 내 이미 갱신된 메시지를 다음 체크노드 계산에 즉시 반영.
5. 핵심 식: `S_i = LLR_i + Σu_k` (bitnode 합), `v_j = S_i - u_j`로 checknode 갱신에 필요한 값 도출 — HSS에서는 이 `S_i`를 최신 값으로 유지해 반복당 갱신 횟수를 bitnode degree만큼 늘림.
6. DVB-S2 파티션 행렬 특성상 여러 프로세서가 동일 `S_i`를 동시에 읽고 쓰는 메모리 충돌(data conflict) 문제가 발생.
7. 이를 해결하기 위해 2단계 업데이트 기법 제안: `S_i`를 두 프로세서가 순차적으로 읽어 각각 부분합을 계산 후 별도 메모리에 기록, 세 번째 프로세서가 필요 시 두 부분합을 합산해 최신 `S_i` 재계산(식 8, 9) — no-operation cycle 없이 완전 병렬 유지.
8. 메시지 저장 최적화: checknode당 min1/min2 절대값+sign+index만 저장(21비트, 기존 48비트 대비 56% 절감).
9. 결과: 25 iteration만으로 FS 기반 기존 연구([8])와 동등한 오류정정 성능 달성, 300MHz@360프로세서 기준 throughput 591Mbps, 실리콘 면적 11mm²@90nm(ST 90nm 공정, [8]의 15.8mm² 대비 적음, [10]의 4.1mm² 대비는 병렬도 차이로 더 큼).
10. 한계: 특정 DVB-S2 표준 파티션 행렬 구조에 종속된 스케줄링/메모리 설계이며, 범용 QC-LDPC로의 일반화나 min-sum 알고리즘 자체의 개선은 없음(기존 [11] 기법 재사용).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Horizontal Shuffle Schedule(레이어드 스케줄링) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC의 Dual-Update(even/odd 교대 스케줄)와 유사 개념이나 이미 보유한 기법으로 신규성 낮음 |
| Corrected Min-Sum (offset) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 Min-Sum 보유, offset correction은 참고 가능하나 세부 파라미터 미상 |
| Si 2단계 업데이트(메모리 충돌 회피) | 대응 없음 | Prime ECC는 z_sb=32 row 단위 병렬 구조로 메모리 충돌 문제 자체가 다름 |
| checknode 메시지 압축 저장(21비트) | `decoder.cpp` `C2V_Cal_New_Sgn()` | min1/min2+sign 저장 방식은 유사 개념, 이미 최적화된 것으로 추정 |

적용 가치: `낮음` — 스케줄링과 메모리 충돌 회피는 Prime ECC의 기존 아키텍처(z=32, HCU, Dual-Update)와 전제가 다르고, 핵심 디코딩 알고리즘 자체는 이미 보유한 Min-Sum 변형 수준.
