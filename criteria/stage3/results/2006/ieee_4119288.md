### B. 알고리즘 요약 (10줄 내외)

1. AWGN 채널 기반 (regular/irregular) QC-LDPC 부호 구성, rate 0.65~0.88, length 1030~4160 예제 다수(sum-product 디코딩으로 시뮬레이션).
2. 문제의식: 기존 구조적 LDPC(geometry/RS 기반)는 code length·rate 선택 유연성이 부족하고, iterative decoding 성능은 stopping distance `s(H)`(BEC 성능 지표, error-floor와 상관)에 의존.
3. Berlekamp-Justesen(B-J) 부호(길이 `q+1`인 q-ary MDS 순환부호, `q=2^m`)를 base code로 채택 — Reed-Solomon 기반 LDPC(Djurdjevic 등, IEEE 802.3 표준)의 일반화.
4. 구성법 1: `q`-tuple 치환으로 `H^(1)(γ,ρ,u)` 생성 (`γ,ρ,u` 파라미터로 regular/irregular, 길이/rate 조절), 구성법 2: `(q-1)`-tuple 치환으로 `H^(2)(γ,ρ,u)` 생성.
5. 핵심 정리: girth>4 조건 하에서 stopping distance `s(H)`가 minimum distance `d`의 기존 최량 하한(`γ+1` 또는 `γ+2`(짝수 γ))보다 작지 않음을 증명(Theorem 3~5) — error-floor를 이론적으로 보증.
6. 부호는 permutation matrix 블록으로 구성되는 QC 구조(circulant/치환행렬 기반)라 인코더/디코더 HW 재사용에 유리한 대수적 구조.
7. 특수 사례로 `C(1)(6,32,0)` = 실제 IEEE 802.3 10G BASE-T 표준 (2048,1723) R-S 기반 LDPC와 동일함을 확인 — 기존 표준 부호를 포함하는 일반화 프레임워크.
8. 최적화 절차 없음(closed-form 대수적 구성, DE/GA 미사용).
9. 결과: (1056,813) rate 0.7699 코드가 BER 10^-5에서 Shannon limit 대비 1.73dB, MacKay 랜덤 코드 대비 동등 성능이나 error-floor(10^-6)를 보이지 않음; puncturing(u=0)이 rate 향상시켜도 성능 저하 미미.
10. 한계: HW 설계·복잡도 분석 전무, AWGN sum-product 시뮬레이션만 수행, NAND/SSD 채널 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| B-J 기반 H-matrix 구성 (H^(1), H^(2)) | `ecc_top.cpp` `Load_PCM()` | 새로운 QC-LDPC 부호구조 후보이나 Prime ECC는 특정 H-matrix 고정이라 재검증 부담 큼 |
| Permutation matrix 블록 구조 (QC 특성) | `ecc_data.h` `PCM_b` (base+circulant shift) | 구조적 유사성 있음(순환/치환 행렬 기반)이나 파라미터 체계가 다름 |
| Stopping distance 하한 증명(error-floor 이론적 보증) | 대응 없음 | Prime ECC 코드베이스는 시뮬레이터이며 부호이론적 증명 절차 없음 |

적용 가치: **중간** — QC-LDPC 구조적 부호설계로 error-floor 개선을 이론적으로 보장하는 기법이지만, Prime ECC는 이미 고정된 H-matrix를 사용하므로 새 부호 채택은 높은 재검증 비용(§4 "부호설계" 이식난이도=높음)을 수반.
