### B. 알고리즘 요약

1. 시스템은 regular/irregular LDPC, AWGN 채널, BPSK 변조, LLR 도메인 BP(Belief Propagation) 기반 iterative decoding을 가정.
2. 기존 Min-Sum(MS)은 check-node 업데이트에서 tanh 기반 정확 연산 대신 min/sign만 사용해 성능 저하 발생 - Normalized MS(NMS, 곱연산 `α`)와 Offset MS(OMS, 뺄셈 `β`)로 보정하나 `β`가 고정 상수라 출력 크기를 반영 못함.
3. 채널모델: AWGN, `y_n = x_n + z_n`, `L_n0 = 2y_n/σ²` 초기 LLR.
4. 핵심 기법: 제안하는 Adaptive OMS(AOMS)는 offset `β_t = η·Δ·⌊min|L|/Δ⌋`로, check-node 최소 출력 크기에 비례한 적응형 보정값 사용. `η=1-α`, `Δ=0.5` 설정.
5. 구현 간소화를 위해 index `t = ⌊2·min|L|⌋`로 8단계 lookup table(`β_t = 0.1t, t∈[1,7]`; `β_t=0.8, t≥8`) 사용, `α=0.8`은 (3,6) regular LDPC density evolution으로 결정.
6. 확장: irregular LDPC(rate 0.5, length 10k, `λ(x)=0.277x+0.283x²+0.440x⁸`, `ρ(x)=0.016x⁵+0.852x⁶+0.132x⁷`)에도 동일 기법 적용해 검증.
7. HW 친화 디테일: 기존 MS(2 min 탐색, `2d_c-3`회 비교)에 좌측 시프트 1회 + 길이 8 LUT만 추가 - NMS/OMS의 곱셈/덧셈 1회와 비슷한 수준의 낮은 오버헤드.
8. 별도 학습 절차 없음, `α` 값만 density evolution([12] 참조)으로 사전 결정.
9. 결과: (8k,4k) regular code에서 AOMS가 NMS 대비 0.025dB(BER=1e-4), BP 대비 0.02dB(BER<1e-5) 이내로 근접. Irregular (10k,5k) code에서 AOMS가 OMS 대비 0.05dB, NMS 대비 0.1dB 이득(BER=1e-3), BP 대비 0.05dB 이내.
10. 한계: 시뮬레이션(BER/BLER 곡선)만 제시, HW 게이트/타이밍/처리율 등 구현 결과 없음; 반복횟수 감소 효과는 언급 없음(최대 100 iteration 고정).
