### B. 알고리즘 요약

1. 시스템: rate 1/2 irregular LDPC 두 개를 turbo 구조로 병렬 연접한 NPCGC(New Parallel Concatenated Gallager Code), QPSK 변조, AWGN 채널(`Y = (2x-1)Es + n`).
2. 문제의식: 기존 PCGC(Behairy)는 서로 다른 degree distribution의 무작위(random) 구성 irregular LDPC 두 개를 병렬 연접하는데, 이는 girth가 보장되지 않아 성능이 제한적.
3. 핵심 가정: 두 성분 코드가 "동일한(same)" 우수 degree distribution 쌍을 갖고 PEG로 구성되면 girth가 개선되어 더 나은 성능을 낸다는 가정.
4. 핵심 기법: PEG(Progressive Edge-Growth) 알고리즘으로 symbol node별로 국소 girth를 최대화하며 Tanner graph 간선을 순차 구성, 파라티 체크 행렬 `H=[H1,H2]`(H2는 weight-2 열만) 구성.
5. 핵심 식: 시스템 생성행렬 `G=[I | H1^T H2^-T]` (H2는 이중대각 구조라 역행렬 계산이 간단 - Gaussian elimination 대신 간이 인코딩).
6. 확장: 두 성분 LDPC 디코더(sum-product/BP)를 turbo 구조로 교대로 반복(super iteration)하며 사후확률(LLR)을 서로 사전확률로 주고받음, LDPC 자체 내장 인터리버로 별도 인터리버 불요.
7. 부수 트릭: 각 v-node 차수 p, c-node 차수 q의 개수를 이론식 `N_v(p)=n·λ_p/p / ∫λ(x)dx`, `N_c(q)=m·ρ_q/q / ∫ρ(x)dx`으로 산출 후 H1을 PEG로 구성.
8. 학습/최적화 절차: 별도 학습 없음 - PEG 그래프 구성이 유일한 최적화 단계.
9. 결과: BER=10^-5에서 turbo codes/irregular LDPC/PCGC 대비 약 0.2dB 이득, 인코딩 복잡도는 PCGC 대비(H2 이중대각 구조 활용) 감소, 디코딩 복잡도는 turbo code 대비(BCJR 대신 SP 알고리즘) 감소.
10. 한계: AWGN·QPSK 시뮬레이션에만 국한, HW 구현/양자화/FPGA 검증 전무, 코드 길이 수치가 텍스트 추출 손상으로 확인 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Turbo 구조 병렬 연접(두 LDPC 디코더 교대 반복) | 대응 없음 | Prime ECC는 단일 QC-LDPC 디코더 구조로 turbo-concatenation 개념 없음 |
| PEG 기반 irregular LDPC 부호 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 방식이 다르며(랜덤 irregular vs QC base+shift) 직접 대응 어려움 |
| 이중대각(H2) 기반 간이 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코딩을 채택하고 있어 개념적으로는 유사하나 구조가 다른 부호 |
| Sum-product(BP) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 SP와는 다른 근사 |

적용 가치: `낮음` - AWGN 채널·비-바이너리 QC 구조가 아닌 순수 irregular LDPC 이론 논문이며, turbo-concatenation은 NAND ECC HW 구조와 무관하고 대상도 code-design(정정능력 개선 근거 미제시)이라 이식 가치가 낮음.
