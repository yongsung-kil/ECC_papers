> 총평: LDPC error floor의 원인인 stopping set/최소거리를 빠르게 찾는 오프라인 분석 도구로서 부호 검증에는 유용하나, Prime ECC는 이미 고정된 QC-LDPC 구조를 사용하므로 부호 재설계 관점의 이식 가치는 낮음.

### B. 알고리즘 요약

1. 대상 코드: irregular LDPC, `n=1000`, `R=1/2`, 다양한 variable node degree distribution(예: `λ(x)=0.283x+0.281x^2+0.436x^8`), regular 코드(`λ(x)=x^2, ρ(x)=x^5`)도 비교 대상.
2. 문제: BEC에서는 small stopping set이, AWGN에서는 small trapping set과 최소 해밍 거리 코드워드가 error floor를 결정하지만, 최소거리 계산은 NP-hard(다항시간 불가, Vardy 증명)이므로 근사 탐색 필요.
3. 핵심 가정: stopping set(모든 이웃 check node가 최소 2번 연결되는 variable node 부분집합)과 distance set(모든 이웃이 짝수번 연결)의 개념을 이용, distance set은 곧 코드워드에 대응.
4. 핵심 기법: 기존 error impulse 방법(2개 impulse 사용)을 개선 — 모든 `(j,l)` variable node 쌍에 대해 해당 두 비트에 강한 음의 reliability를 주입하고, sig-min(min-sum) 근사 BP를 `I`회 반복 후 threshold `t`를 -1부터 T까지 늘려가며 erasure decoding으로 stopping set을 탐색.
5. 핵심 식: 탐색된 stopping set 크기 분포 `S(x)=Σ S_j x^j`와 코드워드 해밍 가중치 분포 `W(x)`를 이용해 BEC 점근 FER `P_E(E)≈Σ_{j=Smin}^{Smin+u} S_j·E^j` (식 2), AWGN 점근 FER `P_E(Eb/N0)≈0.5·Σ W_i·erfc(...)` (식 1)로 계산.
6. 확장: 3개 impulse를 쓰면 degree-2 variable node 없는 랜덤 구성 코드에서 더 많은 codeword/stopping set을 찾을 수 있으나 복잡도가 `O(n^4)`로 증가(기본은 `O(n^3)`).
7. 구현 디테일: sig-min(=min-sum) 근사를 사용해 정수 변수로만 구현 가능하도록 하여 탐색 알고리즘 자체의 복잡도를 낮춤; erasure decoding 서브루틴은 매우 저복잡도.
8. 검증 절차: PEG(progressive edge growth) 알고리즘으로 구성한 코드 2종 + stopping set 회피 개선 PEG로 구성한 코드 1종을 비교, 기존 알고리즘([3],[8]) 대비 더 많은 저가중치 codeword를 더 적은 iteration(10회)으로 발견, 실행시간은 기존 대비 약 100배 빠름.
9. 결과: BEC/AWGN 시뮬레이션에서 계산된 점근 bound가 실제 FER curve의 error floor 영역과 잘 일치(코드1) 또는 하한으로 작동(코드2,3); regular 코드는 최소거리 `d≥36`으로 추정, 시뮬 구간에서 error floor 미관측.
10. 한계: 모든 small stopping set을 보장적으로 다 찾는다는 증명은 없음(휴리스틱), 순수 오프라인 분석·시뮬레이션 도구로 HW 구현이나 온라인 디코딩 알고리즘 개선이 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stopping set / 최소거리 탐색(오프라인 부호 분석) | `ecc_top.cpp` `Load_PCM()` | H-matrix 검증·error floor 사전 분석 도구로 활용 가능하나 Prime ECC는 이미 고정 QC-LDPC 사용 |
| sig-min(min-sum) 근사 BP 반복 | `decoder.cpp` `LDPC_Decoder()` | 탐색 알고리즘 내부에 sig-min BP를 활용하나 목적이 부호 분석이지 디코더 개선 아님 |
| erasure decoding 서브루틴 | 대응 없음 | Prime ECC는 BEC 전용 erasure decoder 모듈 없음(NAND는 주로 AWGN/RBER 모델) |

적용 가치: **낮음** — 부호 구성 단계의 오프라인 분석 도구로, Prime ECC의 이미 고정된 QC-LDPC H-matrix 재설계 없이는 활용 가치가 제한적이며 디코더/HW 개선과 무관.
