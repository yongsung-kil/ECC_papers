> 총평: Layered decoding과 정규화 Min-Sum을 결합해 check-to-bit 메시지 저장을 (E+n)*Nb 수준으로 줄인 메모리 효율 디코더 알고리즘 제안 논문. 신규 정정능력 개선은 없고 메모리/스케줄링 최적화가 핵심.

### B. 알고리즘 요약

1. 대상 부호는 구조화 QC-LDPC(rate 2/3, n=1944 및 rate 0.9149, n=2209/648), 채널은 AWGN, 반복복호 SPA 기반.
2. 문제: 기존 SPA/M-SPA 디코더는 check-to-bit 메시지 `Rij`, `Qji`를 모두 저장해야 해 메모리 부담이 크고, 고속 병렬 구현에서 메모리 접근 충돌도 문제.
3. 핵심 가정: check-node 갱신 직후 새 extrinsic 정보를 후속 check-node가 즉시 사용하는 layered schedule(L-SPA)이 non-overlapping parity-check 서브블록 구조(cyclic shift identity 서브블록)를 가진 LDPC에 적용 가능.
4. 핵심 기법: Normalized Min-Sum(N-MS)을 layered schedule에 결합한 `L-N-MS`(Box 4). check-node 출력 시 최소값 `R_i1`, 두 번째 최소값 `R_i2`, 최소 위치 `M(i)`, 부호 `S_ij`만 저장.
5. 핵심 식: `Q/α := Q - (Q >> s)`(Eq5, s=2, α≈1.333)로 정규화를 곱셈 대신 shift-뺄셈으로 근사해 고정소수점 구현 단순화.
6. 메모리 요구량 공식: L-MS = `n*Nb + 2*nc*(Nb-1) + E + ceil(log2(dc))` bit, 기존 SPA `2*E*Nb` 대비 대폭 축소.
7. 부수 트릭: switch-bar를 부호(sign)에만 적용하면 되고 magnitude는 재정렬 불필요 → 벡터화 구조 라우팅 단순화 언급(구현은 안 함).
8. 최적화 절차: 정규화 계수 α는 FER=1e-2 기준 시뮬레이션으로 최적화(rate 2/3: α=1.35, rate 0.9149: α=1.3).
9. 결과: SPA 대비 메모리 70%(rate 2/3), 76%(rate 0.9149) 절감, 25 iteration에서 L-N-MS가 SPA와 동등하거나 근소 우위 성능. (8,4) 고정소수점 손실 0.05~0.4dB.
10. 한계: HW(RTL/게이트카운트/throughput) 설계 없이 알고리즘·메모리량 시뮬레이션 분석에 그침. 완전 병렬 구현 시 layered 방식의 직렬성 이슈는 논의만 되고 미해결.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered scheduling (L-SPA) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 Dual-Update(even/odd 교대) 스케줄 보유, layered와 유사 계열이나 완전 동일하지 않아 참고 가치 있음 |
| Normalized Min-Sum + memory-efficient 저장(min1/min2/sign/position) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min1/min2 기반 min-sum은 이미 보유, 메모리 저장 최적화 세부는 참고 가능 |
| 정규화 계수 shift 근사(Eq5) | `decoder.cpp` `Get_VNU_*`, `ecc_data.h` `PARAM_LLR` | LLR 양자화 테이블 대비 단순 shift 근사이므로 참고 수준 |
| 메모리 라우팅(switch-bar sign-only) | HW arch(z=32 등) | 코드상 구체 대응 모듈 없음 → 대응 없음에 가까움 |

> 적용 가치: 낮음 | Prime ECC는 이미 min-sum, Dual-Update 스케줄, LLR 테이블 기반 양자화를 보유하고 있어 본 논문의 layered+N-MS 메모리 절감 기법은 상당 부분 중복 기여이며, HW 설계·게이트카운트 검증도 없어 이식 우선순위 낮음.
