### B. 알고리즘 요약

1. 채널은 AWGN, BPSK 신호, 반복 BP(belief propagation) 복호를 사용하는 doubly GLDPC 부호 설계 논문이다.
2. 기존 GLDPC는 check node만 subcode(예: Hamming/BCH)로 대체했는데, 본 논문은 variable node도 subcode(SPC, repetition 등)로 대체하는 구조를 제안한다.
3. Tanner graph의 super variable node/super check node는 각각 subcode이며, base matrix `H=[Hmn]`를 row expansion(서브코드 parity check 열벡터로 치환) → column expansion(서브코드 generator 행벡터로 치환) 2단계로 전개해 전체 parity check 행렬을 얻는다.
4. 반복 복호는 MAP(APP) 기반 horizontal step(super check node, 식 (3))과 vertical step(super variable node, 식 (4))의 LLR 메시지 교환으로 진행되며, `Wn(i,p)` 부호 판정 후 `bˆ(i)H^T=0` 이면 종료한다.
5. 서브코드 EXIT 곡선은 SPC/repetition은 닫힌 해, 나머지는 Monte Carlo 시뮬로 얻어 super variable/check EXIT 곡선을 `λi`(edge 비율) 가중합(식 (7),(8))으로 결합한다.
6. EXIT curve-fitting(계면 정합) 기법으로 variable/check 곡선이 서로 맞물리도록 `λi` 분포를 설계하여 문턱값(threshold)을 최소화한다.
7. Convex한 서브코드 EXIT 곡선((6,5) SPC 등)을 variable node에 도입하면 concave한 repetition 곡선보다 check node 곡선(예: Hamming)과 더 잘 맞아 threshold가 개선된다.
8. 최적화는 EXIT chart curve matching(수동/반복적 파라미터 탐색)이며 별도 GA/NN은 사용하지 않는다.
9. rate-3/4 length-100000 부호 C1은 threshold 1.9dB로 동일 rate irregular LDPC(PEG 구성, threshold 2.0dB) 대비 개선, error floor 영역에서도 우수. rate-7/15 length-7650 부호 C2는 threshold 0.3dB로 동일 rate (2,15) GLDPC(threshold 0.88dB) 대비 0.58dB 개선, BER 시뮬에서 0.35dB 이득.
10. HW 구현/합성 없음, 이론 및 시뮬레이션(AWGN, BPSK)에 국한되며 서브코드는 소형 대수코드(≤ (6,5))로 제한되어 있어 실용 하드웨어 복잡도는 미평가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 부호구성(doubly GLDPC base matrix, subcode 배치) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC 고정 구조와 무관한 대수적 서브코드 기반 구성으로 이식 난이도 높음 |
| BP 반복복호(horizontal/vertical step, MAP) | `decoder.cpp` `LDPC_Decoder()` | min-sum이 아닌 full MAP/APP 서브코드 복호로 CN/VN 연산 자체가 다름, 대응 없음 |
| EXIT chart 기반 degree/서브코드 최적화 | 대응 없음 | Prime ECC는 고정 H-matrix 사용, EXIT 최적화 파이프라인 없음 |

> 적용 가치: 낮음 — non-QC-LDPC 대수적 서브코드 구조 기반 이론적 부호 설계이며, NAND/SSD 타깃이 아니고 Prime ECC의 QC-LDPC + min-sum HW 구조와 접점이 거의 없음.
