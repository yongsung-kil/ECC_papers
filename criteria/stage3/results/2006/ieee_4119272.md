### B. 알고리즘 요약

1. 대상은 일반 binary LDPC 복호(중국 HDTV QC-LDPC, DVB-S2, MacKay 부호 등 다양한 코드에 적용), AWGN 채널 가정.
2. 기존 min-sum은 매 iteration마다 horizontal scan(CN 갱신)과 vertical scan(VN 갱신) 2단계가 필요해 속도·메모리·배선 비용이 큼.
3. 채널모델은 표준 AWGN + BPSK, 초기 LLR `Z_n(0) = 2*y_n/σ²`로 정의.
4. 핵심 재정식화: 기존 vertical scan 식 `Z_{mn}^(k) = Z_n^(k) + Σ L_{m'n}^(k)`을 `Z_{mn}^(k) = Z_n^(k) - L_{mn}^(k)`로 바꿔, CN 메시지 `L_{mn}^(k)`를 이전 값 `Z_n^(k-1)`, `L_{mn}^(k-1)`로부터 직접 계산해 두 scan을 하나의 horizontal scan으로 병합.
5. 이 재정식화는 원 min-sum과 수학적으로 완전 동치이며, CN→VN 주소지정만 필요해 VN→CN 주소지정을 제거함으로써 메모리·배선 절반 절감을 주장.
6. 추가 단순화: 동일 check node의 모든 `L_{mn}` 값은 최소/2번째최소 절대값(`A_m`, `B_m`) 두 가지뿐이므로, 이 두 값과 최소값의 위치(`ñ_m`) 및 부호(`s_{mn}`)만 저장하면 메모리 셀 수를 `n·|M(n)|`개에서 대폭 축소.
7. normalized/offset min-sum으로의 확장도 제시: `A_m`, `B_m`에 스케일링(`λ_k`) 또는 오프셋(`β`) 적용.
8. 별도 학습/최적화 절차 없음(구조적 재정식화).
9. C 언어 소프트웨어 구현 기준 원본 min-sum 대비 2배 이상 속도 향상을 보고(정성적, 구체 iteration/BER 수치 없음).
10. 한계: HW 구현·검증은 "현재 검증 중"이라고만 언급, 실제 회로면적/throughput 수치 없음. BER/정정능력 개선 주장 없음(속도·메모리만 개선).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2-scan → 1-scan 병합 (horizontal scan 통합) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC도 C2V/V2C 파이프라인 구조이나 이미 min1/min2 기반 CNU 연산을 사용해 재정식화 이득이 제한적 |
| CN 메시지 min/2nd-min 값 압축 저장 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min1/min2 방식 보유(§3 기재)와 사실상 동일 아이디어로 중복 기여 |
| normalized/offset min-sum 확장 | `decoder.cpp` `LDPC_Decoder()` 반복 루프 | 이미 보유한 정규화/오프셋 기법과 중복 |

> 적용 가치: 낮음 — 제시된 단일-scan 재정식화 및 min/2nd-min 압축 저장 아이디어는 Prime ECC가 이미 구현한 min-sum(min1/min2 기반 CNU) 구조와 본질적으로 동일하며, HW 검증 없이 SW 속도 비교만 제시되어 이식 시 추가 이득이 낮음.
