### B. 알고리즘 요약

1. 시스템은 (8176,7156) QC-LDPC 부모 코드(1022×8176 parity-check, 16개 511×511 right-circulant 서브행렬, 행/열당 2개의 1)를 shortening한 rate-7/8 (8158,7136) 코드, 0.25um CMOS 5-metal 공정 ASIC.
2. 기존 shift-register 기반 두 단계(two-stage, TS) 인코더는 8176 flip-flop이 필요해 면적·전력 비효율적이며, 목표는 flip-flop/게이트 수를 최소화하면서 고속(860Mb/s)을 달성하는 것.
3. 부모 코드의 parity-check 행렬 `H=[A|B]`에서 message에 대응하는 서브행렬만 사용해 (8176,7154) subcode의 generator matrix `G`를 유도, 랭크 손실(1022→1020)로 2개 미지수를 0으로 고정해 해결.
4. 핵심 기법은 generator polynomial reconstruction: 28개 511×511 circulant의 generator polynomial 전부를 저장하는 대신, 두 개의 14×511 multiplexer 기반 ROM에 첫 행 계수만 저장하고 필요 시 parity register를 left-shift하여 재구성.
5. `y = mG`(식 1) 계산 시, 511-bit 경계마다 shift 방향을 전환하는 parity register 제어로 별도의 1022-bit generator polynomial 레지스터(두 개)를 완전히 제거(Fig.3→Fig.4 개선).
6. 부수 기법: 16-to-21bit packer(21이 7의 배수라 MAC 크기 최소화), partial product multiplication을 사용하는 511×21bit MAC을 tree 구조로 재설계해 논리 depth를 6단계로 단축(critical path 개선).
7. Parity register는 halt/load-shift/reset/output-shift 5가지 기능을 겸용해 별도의 1022-bit 출력 레지스터를 없애고, 최상위 80비트만 shadow register로 유지.
8. 별도 학습/최적화 절차 없음(수작업 논리 설계 + Synopsys Design Compiler 합성).
9. 결과: 총 2230 flip-flop(8K 인코더는 1492개)로 SRAA-1(2044FF,15330gate)·SRAA-2(7154FF,67961gate)·TS(8176FF,50078gate) 대비 최소 FF, 게이트 수는 30690@0.25um, 511 clock cycle에 860Mb/s(54MHz clock) 달성, error floor는 10^-10 이하로 보장되는 코드 선택.
10. 한계: 특정 (8176,7156) QC 부모 코드의 16×511×511 circulant 구조에 강결합된 설계이며, 511이 7의 배수가 아니라서 발생하는 packing/unpacking 오버헤드가 존재(저자도 다른 서브행렬 크기였으면 더 단순해졌을 것이라 언급). 디코더는 다루지 않음(인코더 전용).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| generator polynomial reconstruction, parity register functional sharing | `encoder.cpp` `LDPC_encoder_4KB()` | 인코더 회로 면적/FF 절감 기법으로, dual-diagonal 인코딩과는 다른 방식(shift-register MAC 기반)이나 레지스터 재사용 아이디어는 참고 가능 |
| 부호 PCM 생성(shortening, submatrix 선택) | `encoder.cpp` `Generate_PCM_encoding()` | shortening을 통한 부호 파라미터 조정 기법 참고 가능하나 부모 QC 구조 자체가 다름 |
| circulant 기반 encoding matrix 구조(H-matrix) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC circulant 구조라는 점은 공통되나 511×511 특정 구조로 이식성 낮음 |
| MAC tree 구조 최적화(critical path 단축) | 대응 없음 | Prime ECC 인코더 세부 회로 최적화 기법에 대한 명시적 대응 모듈 없음 |

> 적용 가치: 낮음 — 특정 QC-LDPC 부모 코드(8176,7156)에 강하게 결합된 인코더 회로 설계이며, Prime ECC는 이미 고정된 dual-diagonal 인코딩 구조를 사용해 이식 시 인코더 알고리즘 자체 교체가 필요.
