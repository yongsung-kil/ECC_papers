> 총평: hard-decision majority-based 디코딩(Gallager B 계열)의 스위치 스케줄을 EXIT chart로 최적화해 유한길이 코드 성능과 채널 오추정 강건성을 개선한 이론 논문. Prime ECC는 soft-decision min-sum 기반이라 알고리즘 계열 자체의 직접 이식은 제한적이나, "스케줄링/threshold 적응" 아이디어는 간접 참고 가치가 있음.

### B. 알고리즘 요약

1. 대상은 (dv, dc)-regular LDPC 코드를 BSC 채널(AWGN+BPSK+1-bit quantizer로 모델링)에서 hard-decision iterative 디코딩하는 majority-based(MB) 알고리즘 계열.
2. 기존 Gallager 알고리즘 B(GB)는 무한길이 코드에서 EXIT chart 기반 최적 스위치(각 iteration마다 최적 MB 알고리즘 order로 전환)를 사용하지만, 유한길이 코드에서는 채널 파라미터 과소추정에 민감하고 성능이 저하되는 문제가 있음.
3. 각 iteration의 traversing message 평균 오류확률 전이를 EXIT 함수 `h_b(x)`로 모델링하고, order b의 MB 알고리즘 간 switching point `s_ij`, fixed point `q_i-, q_i+`를 정의해 EXIT chart를 조각별(piecewise)로 분석.
4. 핵심 정의: 스위치 스케줄 `(b1,b2,...,bn,...)`이 `b_n ≥ b_n+1` (전환 순서가 항상 낮은 order로만 이동)이면 normalized switch scheme.
5. `q_i+`(second characteristic interval의 우측 fixed point)가 첫 번째 특성구간(first characteristic interval)들과 달리 서로 겹친다는 것이 핵심 관찰 — 이는 GB의 최적 스위치 외에도 수렴하는 스위치 스케줄이 다수 존재함을 의미.
6. 두 가지 구체적 스킴: (1) Expanded optimal switch scheme — 원래 최적 스위치 `b_n`을 확장 인자 `N_e`로 `b'_n = b_⌈n/Ne⌉`처럼 각 order 유지 구간을 늘림. (2) TC(tail-cut) optimal switch scheme — 특정 iteration `l`(단, `q_l+1- = 0`) 이후로는 order를 `b_l`로 고정해 저차(order 낮은) MB 알고리즘으로의 전환을 아예 차단.
7. 수렴 조건은 Theorem 4~6에서 EXIT function의 fixed point 조건(`q_i+1- < q_i+`, `q_l+1- = 0`)으로 증명, 별도 학습/최적화 절차는 없고 닫힌형 EXIT 분석만 사용.
8. 결과: (4,9) QC 코드(length 1629)에서 3-expanded 스킴이 GB 대비 BER 10^-6에서 약 0.5dB 개선, (6,8)-regular PEG 코드(length 4000)에서 TC optimal이 GB 대비 0.6dB, MB order-1 대비 0.2dB 개선. 채널 90% 과소추정 조건에서 expanded 스킴들이 열화폭을 크게 줄임(1-expanded 큰 열화 → 3/4-expanded 0.1dB 미만).
9. iteration 수는 오히려 GB보다 다소 증가(예: GB 15회 vs 2-expanded 21회, 무한길이 (4,9) 코드 기준)하여 수렴 속도는 느려지는 트레이드오프가 있음.
10. 한계: regular 코드 + hard-decision MB(Gallager B) 계열에 한정, HW 미설계·시뮬레이션만 존재, irregular 코드로의 확장은 future work로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 스위치 스케줄(iteration별 알고리즘 order 전환) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 개념적으로 iteration별 LLR/threshold 테이블 적응과 유사하나, 대상 알고리즘이 majority-based hard-decision이라 min-sum 파라미터 테이블과 직접 대응은 아님 |
| Majority-based(Gallager B) 하드 디코딩 알고리즘 | 대응 없음 | Prime ECC는 min-sum(soft-decision) 기반이며 hard-1bit 전용 MB 알고리즘 자체는 미보유 |
| EXIT chart 기반 threshold 분석 | 대응 없음 | 부호/파라미터 설계 이론 도구로 특정 HW 모듈과 대응되지 않음 |
| 채널 오추정 강건성 | `channel.cpp` `Set_LLR_Th()` | LLR threshold 설정의 강건성이라는 문제의식은 유사하나 이 논문은 hard-decision 문맥이라 직접 재사용 어려움 |

적용 가치: **중간** — hard-decision majority-based 알고리즘 계열이라 Prime ECC의 min-sum soft-decision 디코더와 알고리즘 자체는 직접 이식 불가하지만, "스케줄을 낮은 order로 단조 전환"하고 "채널 파라미터 오추정에 강건한 스케줄 설계"라는 아이디어는 적응형 LLR 테이블 설계에 참고 가능.
