### B. 알고리즘 요약

1. AWGN 채널에서 rate-compatible LDPC를 위해 하나의 mother code(저rate)를 puncturing하여 다양한 rate를 얻는 문제를 다룸.
2. 기존 Ha et al.[8]의 puncturing 분포 최적화는 variable/check node degree 분포만 고려해 코드 구조(구성 알고리즘)를 반영하지 못함.
3. 핵심 가정: 무한 길이(cycle-free) LDPC 코드 앙상블, all-zero codeword 전송, BPSK+AWGN 채널.
4. 핵심 기법: check node를 `d(cj)` 벡터로 세분화한 **generalized check node degree distribution** `ρg(x) = Σ ρ_dc x^dc`를 정의해 구성 알고리즘별 코드 구조 차이를 반영.
5. 이 일반화 분포를 이용해 discretized density evolution을 확장, 각 코드(Random/Zigzag/PEG)에 특화된 puncturing 분포 `π(0)(x)`를 최적화(constraint: 개선된 threshold σ*).
6. 세 가지 구성법(Random: 완전 무작위, Zigzag: degree-2 노드 지그재그 연결, PEG: Progressive Edge-Growth)에 대해 각각 다른 최적 puncturing 분포 도출.
7. check node 확률질량함수 계산은 forward-backward 알고리즘으로 근사(전체 조합 대신 순차 convolution).
8. 최적화 절차: threshold를 높이는 방향으로 π(0)(x)를 반복적으로 미세 조정(제약: puncturing 비율 p(0) 유지).
9. 결과: PEG 코드에서 기존 대비 약 0.12dB, Zigzag 0.13dB, Random 0.4dB(BER 10^-3) 성능 향상.
10. 한계: 무한장 코드 가정, HW 미설계, 순수 시뮬레이션/이론 수준이며 고정 rate NAND ECC와 무관한 rate-adaptive 통신 시나리오.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Puncturing 분포 최적화(rate-compatible) | 대응 없음 | Prime ECC는 고정 rate QC-LDPC로 puncturing 미사용 |
| Generalized density evolution (코드 구조 반영) | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 단계와 개념적으로만 연관, 구현 대응 없음 |
