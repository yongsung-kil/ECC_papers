### B. 알고리즘 요약

1. 시스템은 유한 블록길이(최대 수천 심볼) irregular LDPC이며 목표는 하나의 mother code(rate 1/2)를 puncturing해 rate 0.5~0.9 범위를 지원하는 rate-compatible 구조.
2. 기존 eIRA(extended IRA) 코드는 고rate puncturing에서 성능이 크게 저하되는 문제가 있어 이를 개선하려 함.
3. 핵심 가정은 채널이 AWGN이고, puncture된 노드는 복호기에서 erasure로 취급된다는 표준 puncturing 모델.
4. 핵심 기법은 non-systematic part(H2)를 `k-SR(k-step recoverable) matrix`들의 결합으로 구성: `H = [H1 | H2]`, degree-2 노드를 H2에 배치.
5. k-SR 노드는 정확히 k회의 반복복호 후 erasure에서 복구 가능하며, 낮은 SR 노드가 많을수록 puncturing 성능이 좋다는 [Ha et al.] 기준을 만족하도록 H2를 설계.
6. 확장으로 H2를 하삼각행렬(`LM`) 형태로 재배열해 shift-register/switch/XOR만으로 인코딩(`P = H1^T · H2^-T`)을 구현, 또는 반복 erasure decoder로 (d+1) iteration 내 인코딩 가능.
7. 부수 트릭: Lemma 2/3으로 H2가 cycle-free임을 증명(길이-2s cycle이 존재할 수 없음)해 좋은 코드 특성 보장, degree-2 노드를 모두 non-systematic bit에 배정.
8. H1(systematic part)의 degree distribution은 density evolution 근사값을 사용하고 girth 개선 알고리즘([15],[16])으로 구성.
9. N=1024, rate 0.9에서 eIRA 대비 BER 10^-5에서 1.1dB, 일반 irregular LDPC 대비 최선의 intentional puncturing 기준 1.2dB, random puncturing 기준 2.8dB(rate 0.8) 개선.
10. 한계로 HW 구현은 다루지 않고(shift-register 인코더는 개념도만) 시뮬레이션 결과만 제시하며, non-systematic part의 degree-2 비중이 높아 minimum distance 저하로 인한 error floor 위험을 별도 기법으로 완화해야 함을 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| k-SR matrix 기반 non-systematic 구조 설계 | `ecc_top.cpp` `Load_PCM()` | 부호설계(H-matrix) 변경 요구, 특정 QC-LDPC 고정된 Prime ECC 구조에 적용 시 재검증 부담 큼 |
| shift-register 기반 선형시간 인코더 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 인코더와 유사 계열(하삼각 구조)이나 목적(rate-compatible puncturing)이 상이 |
| rate-compatible puncturing 알고리즘 | 대응 없음 | Prime ECC는 고정 rate 단일 코드 구조라 puncturing 메커니즘 자체가 대응 모듈 없음 |
| cycle-free non-systematic 구조(Lemma 2/3) | `ecc_top.cpp` `Load_PCM()` | H-matrix girth/cycle 설계 관점에서 참고 가능하나 코드 재설계 필요 |

적용 가치: 낮음 — rate-compatible puncturing/ARQ 목적의 부호구성법으로 NAND의 고정rate 요구와 맞지 않고, HW 구현·검증도 없어 Prime ECC 이식 우선순위가 낮음.
