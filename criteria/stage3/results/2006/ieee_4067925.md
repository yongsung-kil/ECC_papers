### B. 알고리즘 요약

1. 대상은 (j,k) regular QC-LDPC(SFT 코드, block length m×k, rate≥1-j/k), AWGN+BPSK, 반복복호는 sum-product(SP) 계열.
2. 문제: 표준 SP는 완전병렬(VNU/CNU 각각 동시)이라 throughput은 좋으나 수렴이 느림; turbo 디코딩(Mansour&Shanbhag)은 H를 J개 block row(서브코드)로 나눠 순차 갱신해 수렴은 빠르나 J개 서브코드를 직렬 처리해 throughput이 J에 반비례로 제한됨.
3. 핵심 가정: QC-LDPC의 H는 순환치환 identity 블록으로 구성되며, 첫 block row `H1(=H̃0)`는 각 열에 1이 정확히 1개뿐인 특수 구조를 가짐.
4. 핵심 기법(TSP): H를 `H1`(1개 block row)과 `H2`(나머지 j-1개 block row) 두 서브코드로만 분할 — VNU/CNU를 `V→C1→V→C2` 순으로 진행 후 `C2→V`로 마무리, turbo 효과를 2개 서브코드 사이에서만 활용.
5. 핵심 식: `q(n)_{l,i} = λi + Σ_{l'∈Q2i} r(n-1)_{l',i}` (C1 갱신), 이어서 C1의 새 메시지를 즉시 `q̃(n)_{l,i}`(식 6)에 반영해 C2 갱신에 사용 — C1의 extrinsic LLR을 저장할 필요가 없어 block row 1개분 메모리 절감.
6. 확장(SSP): 같은 아이디어를 column-wise로 적용한 shuffled-sum-product(SSP) — `H`를 첫 block column `H̄1`과 나머지 `H̄2`로 분할, C→V1→C→V2→C 순서.
7. 구현 디테일: TSP의 계산복잡도는 SP와 동일(식 5+7 = 식 2, 식 4+6 = 식 1), turbo처럼 O(j²) 증가 없음.
8. 학습/최적화 절차 없음(고정 스케줄, 파라미터 튜닝 불필요).
9. 결과: (211,3,5)/(421,4,7)/(281,5,8) SFT 코드에서 TSP/SSP가 SP 대비 평균 iteration 10.1~22.4% 감소; turbo 대비 TSP는 throughput 최대 약 45% 향상((281,5,8) 코드, j=5 기준), 메모리 절감은 turbo와 동일하게 1/j.
10. 한계: 시뮬레이션(BER, 평균 iteration, throughput 추정)만 제시, HW/FPGA/ASIC 구현·gatecount 없음; low-rate(0.375~0.43) SFT 코드 대상이라 NAND 고rate 코드로의 직접 검증은 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2-supercode 분할 turbo-sum-product 스케줄 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 내 메시지 패싱 순서 변경 기법이라 디코더 루프에 직접 적용 가능, 낮은 난이도 |
| CNU/VNU 갱신 순서 변경으로 메모리 절감 | `decoder.cpp` `CNUJ_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 서브코드 분할 스케줄이 기존 Dual-Update(even/odd parity 교대)와 유사한 계열, 중복 여부 확인 필요 |
| 서브코드 간 extrinsic 정보 즉시 전달(메모리 미저장) | `ecc_data.h` PARAM_DEC 관련 메모리 구조 | 1개 block row분 LLR 저장 생략 아이디어, HW 메모리 설계에 참고 가능 |

> 적용 가치: 중간 — 스케줄링만으로 SP 대비 수렴속도·메모리를 동시에 개선하는 아이디어는 디코더에 이식 난이도 낮으나, Prime ECC가 이미 Dual-Update 스케줄을 보유하고 있어 개념적 중복 가능성이 있어 재확인 필요.
