### B. 알고리즘 요약

1. 대상은 base-matrix 기반 구조적 QC-LDPC(circulant shift identity sub-matrix), layered belief-propagation(L-BPA) 복호를 사용하는 block-serial 디코더.
2. 기존 L-BPA는 layer 간 의존성(현재 layer의 Write가 끝나야 다음 layer의 Read가 시작) 때문에 block-serial 구현 시 latency가 큰 문제.
3. 제안 기법은 새 C2V 메시지와 이전 C2V 메시지의 차이(`ΔL(xj)`)를 LLR에 더하는 방식으로 LLR-update를 재정의해, 다음 layer의 Read/Process를 현재 layer의 Write와 오버랩(파이프라이닝) 가능하게 함.
4. 수정된 스케줄: `M(civj[q]) = ψ^-1(Σ ψ(L(xj')[q-1,l-1] - civj'[q-1]))`, LLR은 `L(xj)[q,l] = L(xj)[q-1,l-1] + ΔL(xj)[q,l]`로 갱신.
5. 이 재정의로 V2C 계산은 old C2V를 최신 LLR에서 빼는 형태로 단순화되지만, CNP에 LLR 조정치(ΔL) 계산을 위한 subtract 연산이 하나 추가됨.
6. Latency 모델: 파이프라인 없는 경우 `Γ_LBPA = Σ(2ρ_l + P - 1)`, 파이프라인 적용 시 `Γ_pipeline = L×ρ_max + P - 1`로, layer 수가 많거나 check-node degree가 클수록 개선폭이 커짐.
7. Min-sum 근사(`ψ^-1(ψ(x)+ψ(y)) ≈ sign(x)sign(y)min(x,y)`)를 이용해 C2V 메시지가 MIN 또는 MIN2 값만 가진다는 성질을 활용, accumulate-and-subtract 방식으로 CNP 연산량을 `ρ(ρ-1)`에서 `2ρ`로 축소.
8. C2V 메모리는 sign/magnitude 분리 저장(MIN, MIN2, sign, MIN index)으로 8-bit 정밀도 기준 rate-1/2(ρ_max=8) 약 60%, rate-5/6(ρ_max=20) 약 75% 절감.
9. 결과: rate-1/2(N=1536), rate-5/6(N=2400) 802.16e 부호에서 14-pipeline stage, 10 iteration 기준 latency 2x-3x 개선, BLER 성능 열화는 미미(대안 layer 순서 스케줄링 "Pipeline-2"로 원래 L-BPA 수준 성능 회복).
10. 한계: gatecount/실제 throughput 등 HW 합성 결과 없이 이론적 latency 분석과 시뮬레이션(BLER)만 제시, ASIC/FPGA 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| L-BPA 파이프라인 스케줄(layer 간 오버랩) | `decoder.cpp` `LDPC_Decoder()` | 현재 pipeline 6-stage 구조에 layer-간 파이프라인 스케줄 추가 이식 가능성, HCU와의 상호작용 검토 필요 |
| min-sum CNP accumulate-subtract 최적화 | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum 코어는 이미 보유, accumulate-subtract 구조는 중복 기여일 수 있음 |
| C2V 메모리 sign/magnitude 압축(MIN/MIN2 저장) | `decoder.cpp` `C2V_Cal_New_Sgn()`, `decoder.h` | C2V 메모리 절감 기법으로 HW 메모리 최적화에 참고 가능 |
| mirror LLR memory | 대응 없음 | Prime ECC 프로파일에 명시된 대응 모듈 없음 |

> 적용 가치: 중간 — 디코더 latency 개선 기법(layer 파이프라이닝)은 min-sum/HCU와 별개 레이어라 이식 난이도는 낮으나, 정정능력 개선이 아닌 순수 latency/HW 아키텍처 최적화로 Prime ECC의 우선순위(성능 개선)와는 결이 다름.
