### B. 알고리즘 요약

1. 시스템은 EPR4 partial-response 자기기록 채널, PEG-QC-LDPC 코드 (4608, 4096), rate 8/9, column weight 4, `(0,k)` RLL 제약(k=5~7) 결합.
2. 기존 Vasic-Pedagani의 deliberate bit-flipping 방식은 flip 비트 수가 통제되지 않아 LDPC 최소거리를 매우 크게 요구, 본 논문은 flip 수를 줄이는 제어법을 제안.
3. 채널 모델은 EPR4 + AWGN, RLL 제약은 `(0,k)`: 연속된 두 "1" 사이 "0"의 개수 상한.
4. 핵심 기법: LDPC 코드워드를 LFSR 생성 의사난수열과 XOR한 후 RLL 위반 비트를 flip. flip 수가 임계값을 넘으면 다른 LFSR seed로 재시도(최대 시도횟수 제한), 최소 flip 수의 시퀀스를 채택.
5. 핵심 식: `P(n_flip)`는 XOR 시도횟수 `L`에 따라 `P_L(n) = [P_1(n)]^L`(식 13)로 감소 — 시도 횟수를 늘릴수록 flip 비트 수를 LDPC 정정 능력 이내로 억제 가능.
6. 확장: 복호측은 LFSR seed를 별도 짧은 block code(EG 기반)로 보호해 복원 후, 수신 LLR의 부호를 XOR하여 LDPC SPA 복호기에 입력하는 결합 반복 검출/복호(iterative APP detector + SPA) 구조.
7. 구현 디테일: LFSR 13비트 레지스터, XOR 게이트 2개, 소형 threshold 비교기/버퍼만 필요해 RLL 전용 복호기(모듈레이션 코드 디코더) 자체를 제거.
8. 최적화 절차 없음(수치 확률 분석 + 시뮬레이션으로 XORing 시도횟수-flip수 트레이드오프 도출).
9. 결과: (0,7) RLL에서 XORing 없이 2dB 이상 손실이던 것이 20회 시도 시 손실 0.15dB(BER 10^-5)로 개선, 기존 전통적 RLL 인코딩(rate 32/33) 대비도 우수(이상적 RLL 복호 가정 하에서도).
10. 한계: HDD 자기기록 PR채널 전용, RLL 제약 자체가 NAND에는 존재하지 않는 변조코드 문제, HW 합성/게이트카운트 등 실제 구현 검증 없음(개념적 복잡도 분석뿐).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC SPA 복호 (결합 검출/복호 루프) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 SPA와 다르고, RLL 관련 XOR/역-XOR 전처리는 대응 모듈 없음 |
| PEG-QC-LDPC 부호 구성 (4608,4096, col weight 4) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC라는 부호종류만 겹치고, RLL 결합 목적의 구성법은 NAND에 무관 |
| LFSR 기반 bit-flipping/RLL 인코딩 | 대응 없음 | NAND는 RLL 변조제약이 없어 해당 기능 자체가 불필요 |
| LFSR seed 보호용 블록코드(EG 63,37) | 대응 없음 | Prime ECC의 인코더(`encoder.cpp`)는 dual-diagonal 고정, 별도 짧은 블록코드 보호 개념 없음 |

> 적용 가치: 낮음 — RLL 제약은 자기/광기록 채널 특유의 문제로 NAND 플래시 채널에는 해당하지 않아 이식 가치가 낮음.
