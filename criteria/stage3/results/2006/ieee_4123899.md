### B. 알고리즘 요약

1. 대상 코드는 base matrix(54x100, girth 6, C={2,3,4}, R={5,6})를 density evolution으로 구성 후 S=32/64배 확장한 QC-LDPC (length 3200/6400, rate 0.46).
2. 기존 sum-product 기반 FPGA 디코더는 bit-to-check/check-to-bit 메시지를 모두 저장해야 해 메모리 오버헤드가 크다는 문제를 해결하고자 함.
3. 채널은 BPSK + AWGN을 가정, 표준 시뮬레이션 환경.
4. 핵심 기법은 Single Parity Check Trellis(SPCT) 표현 기반 modified turbo decoding: check node를 (r,r-1)-SPC 코드의 trellis로 변환, Log-MAP의 `Q(x,y)=max(x,y)-max(x+y,0)` 근사(Jacobian log의 principal part)만 하드웨어로 구현하고 보정항 `δq(x,y)`는 생략.
5. `Δαk = Q(Δαk-1, Pk)`, `ΔΒk-1 = Q(Βk, Pk)`, `Λk = Q(Δαk-1, Βk)` 식으로 posterior LLR을 갱신, bit-to-check/check-to-bit 메시지를 단일 posterior LLR(γ)로 병합해 메모리 절감.
6. QC 구조 덕분에 block row 내 (r,r-1)-SPC trellis들을 S개 MDU(Message Decoding Unit)로 완전 병렬 처리 가능.
7. 하드웨어 구현은 5bit 고정소수점 해상도를 채택(성능-면적 균형), barrel shifter 기반 cyclic interleaver/deinterleaver로 셔플 네트워크 구성.
8. 별도 학습/최적화 절차 없음(density evolution은 코드 구성 단계에만 사용).
9. length 3200 코드에서 메모리 총 71Kb(72214bit) 사용, 기존 FPGA 구현 대비 BRAM 수 절감(16 vs 20~102), Q(x,y) 근사로 인한 성능 열화는 최대 0.5dB.
10. 알고리즘이 sum-product/min-sum이 아닌 turbo-style이라 일반 LDPC 디코더 아키텍처와 이질적이며, 인코더/HW ASIC 합성 결과 없음(FPGA만).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified turbo(Log-MAP SISO) 디코딩 알고리즘 전체 | 대응 없음 | Prime ECC는 min-sum 기반이며 turbo-style Log-MAP과 알고리즘 패러다임이 다름 |
| QC-LDPC 구조(base matrix+circulant shift) | `ecc_top.cpp` `Load_PCM()` | 구조적 유사성은 있으나 H-matrix 자체는 논문 고유 구성으로 이식 난이도 높음 |
| cyclic interleaver/deinterleaver (barrel shifter) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | QC-LDPC의 circulant shift 라우팅 개념은 유사하나 Prime ECC는 z_sb=32 기반 행 병렬 구조로 별도 구현 |
| 메모리 절감을 위한 메시지 병합(γ 단일 저장) | 대응 없음 | Prime ECC는 min-sum의 C2V/V2C 메시지 구조를 사용, turbo 메시지 병합 기법과 불일치 |

적용 가치: 낮음 — turbo-style SISO 디코딩은 Prime ECC의 min-sum QC-LDPC 아키텍처와 근본적으로 다른 패러다임이라 이식 난이도가 높고, 2006년 FPGA 결과만 있어 최신 HW 대비 비교우위도 불명확.
