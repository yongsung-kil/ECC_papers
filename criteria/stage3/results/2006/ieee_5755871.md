### B. 알고리즘 요약

1. Turbo/LDPC 디코딩은 모두 factor graph 상의 (iterative) sum-product 알고리즘의 인스턴스로 통합 설명됨.
2. 문제: loopy factor graph에서 표준 iterative sum-product는 반복 스케줄에 따라 수렴 속도가 다르며, LDPC의 표준(parallel, flooding) 스케줄은 turbo 디코더보다 느리게 수렴.
3. 핵심 가정: 채널은 BPSK+AWGN, 코드는 랜덤 regular LDPC(N=2000, R=1/2, column weight 3).
4. 핵심 기법: factor graph를 loop-free sub-graph들 `S1..Sk`로 분할(divide-and-conquer). 각 sub-graph는 입력벡터 `I`, 상태 `X`로 분리연산자 `⊖`(식 6: `R = I ⊖ D(X)`)와 결합연산자 `⊗`(식 9: `O = O1 ⊗ O2 ⊖ I`)를 이용해 다른 sub-graph 결과를 a priori로 받아 classical(loop-free) sum-product 실행.
5. 식 (6),(7)은 이전 iteration에 자신이 생성한 정보를 입력에서 분리해 이중 카운팅을 막는 상태 갱신 규칙.
6. sub-graph 연결 방식은 cascade/parallel 2종과 partitioning은 all-accessible/non-accessible/trivial 3종으로 조합 가능(4가지 실험: CAS-NON, PAR-NON, CAS-ALL, PAR-ALL).
7. PAR-NON은 표준 LDPC flooding 스케줄과 동일, CAS-NON은 [12]의 turbo-style LDPC 디코더 구조와 동일함을 이론적으로 증명.
8. 복잡도: 한 iteration의 총 edge 연산량은 sub-graph 분할과 무관하게 항상 원 factor graph의 edge 수와 동일함을 증명(공정 비교 근거).
9. 결과: CAS-ALL이 PAR-NON(표준) 대비 iteration 55% 감소, all-accessible이 non-accessible 대비 약 10% 추가 감소, cascade가 parallel 대비 50% 감소. FER-SNR 곡선은 4가지 방식 모두 거의 동일(정확도 손실 없음).
10. 한계: 순수 알고리즘/시뮬 수준(HW 설계 없음), 작은 랜덤 regular code 1종만 검증, QC-LDPC/구조적 부호나 min-sum 근사·양자화에 대한 논의 없음.
