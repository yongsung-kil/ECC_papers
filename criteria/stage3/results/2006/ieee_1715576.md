### B. 알고리즘 요약

1. Regular/irregular LDPC의 check node 복호에서 min-sum 근사(`L_out = min|L_in|`)는 sum-product 대비 성능 손실이 있어 이를 correction factor로 보정.
2. 기존 보정법(normalized BP, offset BP, modified min-sum)은 degree distribution별 파라미터 최적화(density evolution/시뮬레이션)가 필요해 multirate 시스템에 비효율적.
3. 일반 correction factor는 두 입력 LLR의 합·차의 exponential 함수로 매우 복잡(LUT 다수 필요)하여 sum-product 대비 이점이 없음.
4. 큰 등가 크기 입력 LLR에서 correction factor의 최대값이 `ln2 / d_c` (d_c=check node degree)로 수렴함을 이용해 degree-matched 근사를 유도.
5. Two-step 근사: 최소/차소 LLR(`L1, L2`) 차이에 따라 2개 threshold로 `ln2/d_c` 또는 그 절반을 `L1`에서 차감(식 8).
6. One-step 근사: threshold 1개만 사용해 `L1` 크기만으로 보정량 결정(식 9), 구현 복잡도를 추가로 축소.
7. Density evolution(threshold, SNR 기준)으로 regular (3,6), (3,16), irregular(rate 0.5) 세 분포에 대해 근사 성능을 검증.
8. Finite precision 시뮬로 6비트(1 sign+5 magnitude) 양자화 시 부동소수점 대비 근사손실 거의 없음을 확인.
9. 90nm CMOS VHDL 합성 결과, one-step degree-matched check node는 sum-product 대비 면적 41%로 근사 최적 성능 달성(min-sum은 29%로 가장 작으나 0.4dB 손실).
10. 한계: 단일 degree-6 check node 셀의 ASIC 합성 결과만 제시, 전체 디코더 통합/throughput/전력 등은 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Check node degree-matched correction factor (one-step/two-step) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산에 correction 로직 추가로 이식 가능, waterfall 성능 개선 기대 |
| Density evolution 기반 threshold 결정 | 대응 없음 | 코드베이스에 density evolution 모듈 없음, 오프라인 설계 단계에 해당 |
| 4/6-bit sign-magnitude 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 기존 LLR 양자화 테이블과 정합성 검토 필요 |
| Check node ASIC 면적 비교(90nm) | 대응 없음 | Prime ECC는 특정 공정 비의존 시뮬레이터, 면적 수치 직접 대응 불가 |

적용 가치: **높음** — min-sum CN 연산부에 저비용으로 삽입 가능한 degree-matched correction으로, irregular 코드에서도 파라미터 재최적화 없이 waterfall 성능을 sum-product 수준으로 회복할 수 있어 기존 min-sum 구조와 중복도가 낮고 개선 여지가 명확함.
