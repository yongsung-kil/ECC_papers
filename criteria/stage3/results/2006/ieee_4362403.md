### B. 알고리즘 요약

1. 대상은 길이 n(8 이상 짝수)의 이진 선형 블록코드 C=(n,k,4), 최소 Hamming distance 4를 갖는 코드.
2. 기존 LDPC는 BER 성능은 좋으나 생성행렬(G)이 밀도 높고 불규칙하여 인코더 구현이 어려움 — 이를 해결하고자 함.
3. 핵심 가정은 채널 모델 없이 순수 대수적 코드 구성(대상은 encoder/generator matrix 설계).
4. 핵심 기법은 두 성분 생성행렬 `G1=[1 1]`, `G2=[1 0]`을 기하학적으로 배치해 전체 생성행렬 G를 구성하는 "Geometric Construction(GC)" 기법.
5. 코드 크기 식 `C=(n, n-⌈log2 n+1⌉, 4)`로 full information rate distance-4 코드를 생성함을 정의.
6. 배치 규칙은 그룹 E1(G1 두 개 순환배치), E2 이후(G2 네 개씩 간격을 2배씩 늘려가며 순환배치)로 구성.
7. Theorem 1(가중치 4의 배수·직교 생성행렬 → self-orthogonal) 을 이용해 Theorem 2(GC 코드가 항상 distance-4)를 증명.
8. 별도의 학습/최적화 절차 없음 — 순수 조합적/대수적 구성.
9. 결과는 성능 수치(BER, iteration 등) 없이 "extended Hamming/Reed-Muller 코드를 포함하는 최적 distance-4 코드 생성"이라는 구성적 결론만 제시.
10. 한계: 시뮬레이션·HW 설계 전무, distance-4로 고정되어 있어 NAND용 고rate LDPC의 정정능력·error-floor와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GC 코드 생성행렬 구성법 | 대응 없음 | distance-4 고정 블록코드로 QC-LDPC H-matrix 구성과 무관 |
| quasi-cyclic 성분 구조 | `ecc_top.cpp` `Load_PCM()` | 이름만 quasi-cyclic 공유, 실제로는 LDPC 아닌 별개 코드 계열이라 대응 어려움 |

적용 가치: 낮음 — LDPC가 아닌 고정 최소거리(d=4) 대수적 블록코드 구성법으로 반복복호·soft-decision 개념이 없어 Prime ECC 이식과 무관.
