### B. 알고리즘 요약

1. 대상은 AWGN/BEC 채널에서 짧은 block length(수천 bit)의 regular/irregular LDPC mother code를 puncturing해 rate-compatible 부호 계열을 만드는 문제.
2. 기존 연구는 무한 길이 가정 하 좋은 puncturing 분포를 제시했으나, 짧은 길이에서는 무작위(random) puncturing이 심각한 성능 손실 및 stopping set 문제를 유발.
3. 핵심 아이디어: puncture된 variable node가 반복(iteration)을 통해 unpunctured node로부터 메시지를 받아 "복구(recovery)"되는 과정을 정의하고, k번째 반복에서 복구되는 노드를 k-SR(k-step-recoverable)로 분류.
4. Grouping 알고리즘: 1-SR 노드 그룹 `G1`을 최대화한 뒤 2-SR 그룹 `G2`, ... 순으로 채워 puncture 순서 결정. recovery-error probability `Pe(v)`가 작은 노드(=survived check node의 recovery tree 크기 `S(v)`가 작은 노드)를 우선 선택.
5. `Pe(v) = (1/2)(1 - Ψ(v,ε))`, `Ψ(v,ε)`는 BEC erasure 확률 ε 기반 재귀식으로, 작은 `S(v)`를 가진 노드일수록 복구 신뢰도가 높다는 것을 뒷받침.
6. Sorting 알고리즘: 같은 그룹 내에서도 survived check node 수(`cw_max`)가 많고 dead check node가 적은 노드를 우선 puncture하여 rate-compatible 순서를 세밀화.
7. 구현 디테일: 4-cycle loop를 피한 regular 구조 mother code, irregular code는 PEG(Progressive Edge Growth) 알고리즘으로 parity-check matrix 설계.
8. 최적화 절차는 없음(GA/NN 등 학습 없음) — 결정론적 그래프 탐색(그룹화+정렬) 알고리즘.
9. 결과: block length 1024, rate 0.8에서 intentional puncturing이 random 대비 최대 3dB Eb/N0 이득, block length 4096에서는 1.6dB 이득(block length가 커질수록 random puncturing 변동성 감소).
10. 한계: HW 설계 없음, AWGN/BEC 시뮬레이션뿐, ARQ/rate-adaptive 시나리오 전용으로 NAND 고정 rate ECC와 직접 관련성 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Puncturing 기반 rate-compatible 부호 설계 | 대응 없음 | Prime ECC는 고정 H-matrix 기반 단일 rate 부호이며 puncturing을 통한 rate 적응 기법을 사용하지 않음 |
| Recovery tree / k-SR 그룹화 알고리즘 | 대응 없음 | 부호 구성 단계의 이론적 puncturing 패턴 설계로 디코더 실행 로직과 무관 |
| PEG 기반 irregular H-matrix 설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계라는 점에서만 관련(이식 난이도 높음, §4 참조) |

> 적용 가치: 낮음 — puncturing을 통한 rate-compatible 부호 설계는 NAND 고정rate binary QC-LDPC ECC의 이식 대상과 목적이 다르며, decoder/HW 관련 기여가 전혀 없음.
