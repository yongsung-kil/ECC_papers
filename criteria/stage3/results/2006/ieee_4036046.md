> 총평: protograph 기반 QC-LDPC 계열 부호를 낮은 threshold와 선형 minimum distance(=낮은 error-floor)로 동시 설계하는 정석적 구성법으로, 신규 부호설계 시 참고 가치는 있으나 Prime ECC의 고정 H-matrix 대체는 재검증 부담이 큼.

### B. 알고리즘 요약

1. protograph 기반 LDPC 코드(BI-AWGN 채널, RA/IRA/ARA 계열)를 대상으로, rate 1/2~4/5의 코드 family를 구성. 목표는 낮은 iterative decoding threshold와 블록크기 n에 선형비례하는 minimum distance(`d_min≈nδ_min`)를 동시 달성.
2. 문제: 기존 RA/IRA/ARA 코드는 threshold는 낮지만 minimum distance가 sublinear로 증가해 error-floor가 높음. 반대로 (3,6) regular LDPC는 선형 d_min(δ_min=0.023)이지만 threshold가 높음(1.11dB). 두 특성을 동시에 얻는 방법이 필요.
3. 채널 모델은 BI-AWGN, reciprocal channel approximation(RCA)로 density evolution을 근사(Chung의 self-inverting 함수 `R(x)=C^-1(1-C(x))`, `s`,`r`를 변수/체크노드에서 각각 가산).
4. 핵심 기법: degree≥3 변수노드만 갖는 고rate protograph(선형 d_min 보장됨)에서 시작해, check node를 2개(`m1+1`,`m2+1`)로 분할하고 새 degree-2 변수노드로 연결하는 "check node splitting"을 반복 적용해 저rate 코드를 생성.
5. 핵심 식: 분할된 두 체크의 partial weight enumerator 곱의 합 `A^c_{w1,...,wm}=Σ_l (A_{w1,...,wm1,l}/N_l)(A_{wm1+1,...,wm,l})`(식 1) — 이 관계로 분할 후에도 평균 weight enumerator(따라서 선형 d_min 성질)가 보존됨을 증명.
6. 확장: threshold를 더 낮추기 위해 precoding(degree-1 노드를 별도 체크로 소거) 또는 base protograph에 매우 높은 degree 노드 1개를 포함시키는 두 방법 사용. 예: rate-1/2 AR4JA는 precoding으로 threshold 0.64dB, δ_min=0.015 달성.
7. 일반화 정리(Theorem/Lemma): degree-2 서브그래프의 연결요소마다 "check 개수 = degree-2 노드 개수+1" 조건(check node splitting condition)을 만족하면, 그 protograph는 선형 d_min을 갖는 조상 protograph로부터 노드분할로 구성 가능함(역도 성립).
8. 학습/최적화: ACE 알고리즘으로 protograph를 lifting(circulant 선택)해 실제 코드 생성, 별도 DE/GA 없음.
9. 결과: rate 1/2~4/5 AR4JA family(k=4096, JPL FPGA 디코더)에서 BER/FER이 예측 threshold와 부합, random code Gallager bound 대비 우수한 waterfall 성능 확인(구체 수치는 그래프 상 제시, 본문 서술 수치는 threshold dB값만: 1/2=0.628, 2/3=1.450, 4/5=2.413 dB 등).
10. 한계: 실제 gate-level HW 설계나 처리량/복잡도 분석 없음(단순 FPGA 기능검증), 고rate(0.9 이상) 예제 미제시, degree-2 노드 비율 등 세부 설계는 예제 의존적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph/H-matrix 구성(check node splitting) | `ecc_top.cpp` `Load_PCM()` | 신규 부호 설계 시 base matrix 구성 방법론으로 참고 가능하나 Prime ECC는 고정 QC-LDPC(N_b=129) 구조라 재검증 부담 큼 |
| minimum distance 선형증가(error-floor 개선) 설계 | 대응 없음 | Prime ECC 코드베이스에는 부호설계 단계의 d_min 보장 로직 없음(고정 H-matrix 로드만) |
| reciprocal channel approximation(density evolution 근사) | 대응 없음 | 부호설계 단계 threshold 계산 도구, 디코더/채널 모듈과 무관 |
| ACE lifting(circulant 선택) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC circulant 구조 생성과 개념적으로 유사하나 실존 함수 대응 없음 |

적용 가치: 중간 — 부호설계(protograph→QC-LDPC) 단계에서 threshold와 error-floor를 동시 개선하는 체계적 방법론은 신규 H-matrix 설계 시 참고 가치가 있으나, Prime ECC의 기존 고rate 고정 부호를 직접 대체하기보다는 향후 부호 재설계 시 이론적 지침으로 활용 가능.
