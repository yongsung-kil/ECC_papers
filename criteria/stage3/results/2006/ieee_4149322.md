### B. 알고리즘 요약

1. Matrix-Filled(MF) LDPC 부호는 mother matrix `M(H)`(m×n 이진 행렬)의 0/1을 각각 `l×l` 영행렬/cyclic shift matrix `P^i`(식 1~3)로 치환해 `lm×ln` 검사행렬 `H`를 구성하는 QC-LDPC 계열. 부호 rate `R=(dc-dv)/dc`(식 4, dc=row weight, dv=column weight).
2. 짧은 block length에서 검사행렬 구성 지침이 부족한 문제(Richardson의 asymptotic density evolution은 cycle-free 가정)와 저장 메모리·인코딩 복잡도를 줄이는 문제를 다룸.
3. AWGN 채널, BPSK 변조, sum-product 복호(SPA)를 가정. girth(최단 cycle 길이)가 클수록 독립 iteration 수와 최소거리가 커져 성능이 향상된다는 전제(식 5: `M<g≤M+1`, M=독립 iteration 수).
4. mother matrix의 girth `g_o`를 알 때, MF-matrix의 girth `g_f`는 cycle-bypass의 row-transformation 횟수 `t`(`g_o/2`의 배수)에 의해 `g_f = t×g_o/2 × g_o/2 → g_f=2t`(식 6)로 결정됨을 분석.
5. Proposition 1~3으로 하한 `g_f_min = g_o`(식 7, shift 값이 모두 1인 경우 block-cycle 불변), 상한 `g_f_max = l×g_o`(식 8, block-cycle 간 겹침 없을 때), 겹침 p점 존재 시 `g_f_max = 2×(g_o1+g_o2-p)`(식 9)를 유도.
6. Conditional-filling 구성: (1) mother matrix 행/열 위치 전수 탐색 → (2) girth `g` 집합을 제외한 shift 값 `{1~l}`에서 g개 원소 선택 → (3) mother matrix의 '1'을 선택된 shift 값으로 치환 → (4) 나머지 0/1을 영행렬/순환-shift 항등행렬로 최종 치환.
7. 부수 트릭: `245×343` 기하학적(geometrical) mother matrix, `l=5` cyclic shift matrix 사용, column weight 5·row weight 7 유지.
8. 별도 학습/최적화(DE, GA 등) 없이 결정론적 conditional-filling 알고리즘으로 구성.
9. 결과: mother matrix는 length-6 cycle 2058개(=girth); conditional-filling으로 얻은 `1225×1715` MF-matrix는 length-6 cycle 30개, length-30 cycle 10260개로 감소(random-filling은 최대 10290개 length-6 cycle). 단, rate가 0.335로 낮아 random-filling 코드(rate 0.443~0.455)가 BER>10^-3 구간에서는 오히려 우세하고, conditional-filling 코드는 BER<10^-5까지 error floor가 나타나지 않음.
10. 한계: mother matrix가 full rank가 아니라 실제 rate가 목표보다 낮아짐, HW 미설계·시뮬레이션뿐, 저rate(0.335) 예제 위주로 NAND 요구 고rate 영역에서의 효과는 미검증.
