### B. 알고리즘 요약

1. 채널: AWGN, 복호: soft iterative Sum-Product/quantized approximate-min 디코더(1 sign bit + 5 quantization bit), 코드: regular (96,48), (2000,1000) LDPC.
2. 문제: 초저BER(10^-14~10^-20) 영역은 표준 Monte Carlo(MC) 시뮬레이션으로 평가 시 천문학적 loop 수가 필요해 사실상 불가능.
3. 가정: 디코더 실패는 대부분 코드의 minimum distance가 아니라 `trapping set`(디코더가 절대 빠져나오지 못하는 에러 비트 조합)에 의해 지배됨.
4. 핵심 기법: Importance Sampling(IS)의 mean translation 방식을 trapping set 비트에 집중 적용 — 노이즈 평균을 trapping set bit 쪽으로 편향(bias)시켜 디코더 실패를 인위적으로 자주 유발.
5. 핵심 식: `w(x)=f(x)/f*(x)`(unbiasing weight), `Pr_IS = (1/Ns)Σ I_E(xi)w(xi)` — biased 시뮬레이션 결과를 원래 분포로 역보정해 참 BER 추정.
6. 확장: trapping set 탐색을 위해 bipartite graph의 cycle-6(길이 6인 사이클) 조합을 그래프 탐색으로 찾고, IS 시뮬레이션으로 실제 디코더 실패를 유발하는지 검증하는 2단계 절차 제안.
7. 구현 디테일: 동일 크기·구조의 trapping set은 같은 클래스로 묶어 클래스당 1개만 시뮬레이션(BER 값이 클래스 내 근사 동일함을 실험으로 확인) → 계산량을 클래스 수에 비례하도록 축소.
8. 최적화 절차: 그래프 탐색(cycle 열거) + IS bias 값(±1) 적용 후 고정 loop 수(Ls=1000) 시뮬레이션으로 실제 trapping set 여부 검증하는 반복 절차.
9. 결과: (96,48) 코드에서 BER 10^-14까지 시뮬레이션 gain 8.3×10^11배, (2000,1000) 코드에서 BER 10^-20까지 gain 5.3×10^14배 달성, MC 결과와 거의 일치.
10. 한계: 순수 성능평가(시뮬레이션 방법론) 논문으로 디코더 알고리즘 개선이나 HW 설계는 전혀 다루지 않으며, cycle-4가 없는 코드에 한정된 실험.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Importance Sampling 기반 초저BER 시뮬레이션 가속 | 대응 없음 | Prime ECC는 bit-exact HW 시뮬레이터이며 IS 기반 통계적 성능평가 가속 기법에 대응하는 모듈이 없음 |
| trapping set 탐색(cycle-6 그래프 탐색) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 분석 맥락에서만 개념적으로 참고 가능 |

> 적용 가치: 낮음 — 디코더/부호/HW 개선이 아닌 시뮬레이션 방법론이며, Prime ECC의 목표(부호설계→디코더→HW 이식)와 직접 관련 없음.
