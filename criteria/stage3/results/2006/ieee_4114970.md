### B. 알고리즘 요약

1. 대상은 자기기록채널(magnetic recording, EPR4+AWGN)용 high-rate regular QC-LDPC, ASIC 예시는 length=4608, rate=8/9 (4x36 circulant, circulant size p=128).
2. Reed-Solomon을 LDPC로 대체하려는 흐름에서, 매우 낮은 sector error rate(~10^-9~10^-12)를 시뮬레이션만으로 검증하기 어려운 문제를 FPGA 하드웨어 시뮬레이터로 해결.
3. 채널모델은 EPR4 신호 + AWGN, Max-Log-MAP(sliding window) 검출기와 QC-LDPC 디코더의 반복 검출-복호(iterative detection/decoding) 구조.
4. 핵심 기법1(부호설계): column weight `w=4`인 랜덤 구성 regular QC-LDPC가 girth-4 cycle을 배제하면 sector error rate 10^-9까지 error-floor 없이 동작함을 실증, circulant size `p`를 상대적으로 크게 하고 circulant weight를 낮게 유지하는 것이 error-floor 억제에 유리하다는 경험적 설계 가이드라인 제시.
5. 핵심 기법2(디코더 아키텍처): 기존 partially-parallel 구조[5]는 병렬도가 `1/p`에 비례해 p가 크면 throughput이 제한되는 문제를, `p`를 divisor `v`로 나누어(`h=p/v`) 그룹당 h개 CNU/VNU를 배치함으로써 병렬도를 h배 향상.
6. 메모리 fabric: 각 non-zero circulant의 p개 메시지를 w개 DMMB(dual-port)에 저장하고, barrel shifter로 순환 시프트 정렬을 수행해 매 클록마다 올바른 VNU/CNU 그룹에 메시지를 전달하는 구조(1 iteration = 2v clock cycle: check-node phase v cycle + variable-node phase v cycle).
7. 부수 디테일: 유한정밀도 설계로 AWGN 6bit, Max-Log-MAP path metric 9bit/soft output 6bit, LDPC 디코더 내부 메시지/soft output 6bit(soft-4bit+에 해당).
8. 별도 학습 절차 없음 — circulant size/weight 조합은 대규모 FPGA 시뮬레이션(Altera Stratix-II 180 2장 사용)으로 경험적 guideline 도출.
9. 결과: ASIC(0.13um, 2M gate, 49mm², 1.7W@225MHz)에서 글로벌 iteration 없이 16 internal iteration 시 throughput 1.8Gbps 달성, 검출기-디코더 피드백 4회(글로벌 iteration) 사용 시 360Mbps로 저하.
10. 한계: 본문 텍스트가 OCR 손상이 심해 세부 수식·일부 표 수치(예: circulant weight 정확값)는 손상되어 있으며, 저throughput 모드(글로벌 iteration)에서는 성능 이득 대비 throughput 저하 폭이 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 부호구성 가이드라인(circulant size/weight, error-floor-free 설계) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 파라미터 설계 지침으로 참고 가능하나 특정 QC-LDPC 고정 이식은 재검증 부담 큼(§4 표: 부호설계는 이식난이도 "높음") |
| p→v 단위 병렬화 조정(h=p/v배) 디코더 아키텍처 | `decoder.cpp` `LDPC_Decoder()`, HCU/pipeline 구조 | 우리 코드는 z_sb=32 고정 lifting 병렬 단위를 사용 — 가변 병렬도(v) 개념은 HW arch 이식 난이도 "중"에 해당, 구조 정합 검토 필요 |
| DMMB(dual-port memory)+barrel shifter 기반 메시지 라우팅 | `decoder.cpp` (HW pipeline), `ecc_data.h` (PCM 구조체) | 메모리 라우팅 기법 자체는 참고 가치 있으나 대응 함수 특정 어려움 |
| Max-Log-MAP 검출기(반복 검출-복호) | 대응 없음 | Prime ECC는 순수 LDPC 디코더로 채널검출기(Max-Log-MAP) 통합 반복 구조는 범위 밖 |

> 적용 가치: 중간 — error-floor-free QC-LDPC 설계 가이드라인과 가변 병렬도(h=p/v) 디코더 아키텍처는 참고 가치가 있으나, 특정 QC-LDPC 구조 재검증 부담과 검출기 통합 반복 구조가 우리 범위 밖이라 직접 이식보다는 설계 참고 수준.
