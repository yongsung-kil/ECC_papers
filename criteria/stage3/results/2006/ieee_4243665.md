> 총평: 부호 구성 단계에서 인터커넥트(배선) 길이를 윈도우로 제약해 라우팅 혼잡을 줄이는 부호설계 기법으로, Prime ECC의 고정 QC-LDPC H-matrix에 재적용하기엔 부호 재설계 부담이 크고 이식 난이도가 높음.

### B. 알고리즘 요약

1. (3,6)-regular LDPC 코드(rate 1/2, N=1024 bit node) 대상, 완전 병렬(fully-parallel) 디코더 구현을 전제로 인터커넥트 효율적 부호 설계 기법 제안.
2. 기존 무작위 생성 LDPC는 배선 길이가 길어져 로드 커패시턴스 증가, 지연·전력 증가, 라우팅 혼잡으로 칩 활용률 저하(선행연구 기준 50%) 문제 존재.
3. bit node와 check node를 2차원 그리드(N×M bit node, N×M/2 check node)에 배치한다는 레이아웃 모델을 가정.
4. 핵심 기법: 각 bit node의 연결을 지정된 윈도우 `(R, C)`(행 R, 열 C) 범위 내 check node로 제한 — 예제에서 (16,15) 윈도우 사용, 배선 길이를 8행×8열 이내로 제한.
5. 핵심 식은 별도 수식이 아닌 절차적 알고리즘(Fig. 4): assigned check node 기준 Horizontal/Vertical Domain 내에서 후보 check node 집합을 결정 — 윈도우를 벗어나는 연결을 원천 배제해 배선 최대 길이를 보장.
6. 동시에 girth 제약(4-loop free, 6-loop free, minimized 8-loop)을 결합해 무작위 연결이 짧은 사이클을 만들지 않도록 검사(Fig. 1, Fig. 2 알고리즘으로 4/6-loop 검출 후 해당 edge 제거).
7. 부수 트릭: check node/variable node 연산을 dummy single gate로 단순화한 generic VHDL 모델을 사용해 순수 인터커넥트 복잡도만 비교(Xilinx Spartan3 XC3S5000 FPGA 합성).
8. 학습/최적화 절차 없음 — 각 구성(4L/6L/M8L/WM8L)마다 무작위로 5개 코드 생성 후 최상 성능 코드 선택.
9. 결과: WM8L(윈도우+최소 8-loop)이 M8L 대비 FER 성능 약 0.1dB 저하되나, 지연 11.879ns(M8L 13.096ns 대비 개선), FPGA 슬라이스 2527(M8L 2562 대비 소폭 감소), 라우팅 혼잡 시각적으로 감소.
10. 한계: N=1024 소형 (3,6)-regular 코드에 국한된 실험, 실제 min-sum/sum-product 연산 없는 dummy gate 모델이라 실제 디코더의 gatecount/throughput과는 괴리, NAND용 고rate QC-LDPC로의 확장성 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 윈도우 제약 기반 H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base matrix+circulant shift 방식의 특정 QC-LDPC 고정 구조라 재구성 난이도 높음 |
| girth(4/6-loop) 제약 검출 알고리즘 | 대응 없음 | 부호 구성 단계 알고리즘으로 디코더/인코더 런타임과 무관, 대응 모듈 없음 |
| 인터커넥트/라우팅 혼잡 감소 | 대응 없음 | Prime ECC는 z_sb=32 row 단위 병렬 및 HCU 구조로 배선 모델이 상이, 직접 대응 없음 |

적용 가치: `낮음` — 소형 regular LDPC의 레이아웃 제약 기반 구성법으로, Prime ECC의 고정 QC-LDPC base matrix 구조에 직접 이식하기 어렵고 디코더/인코더 알고리즘 개선과도 무관.
