### B. 알고리즘 요약

1. 대상 코드는 regular Margulis [2640, 1320] LDPC (`dv=3`, `dc=6`), AWGN 채널, 표준 sum-product(BP) 복호.
2. Error-floor 영역에서 프레임 오류 대부분이 (12,4) 또는 (14,4) elementary point trapping set에 갇혀 발생하는 문제를 다룸.
3. Trapping set `(a,b)`는 `H`의 `a`개 열의 projection 중 홀수-weight 행이 `b`개인 부분집합으로 정의.
4. 핵심 기법: 표준 parity-check matrix에 redundant row(중복 parity-check 방정식) 1개를 추가하여 unsatisfied check 수를 늘려 디코더가 trapping set을 벗어나게 유도.
5. redundant row가 trapping set의 12개(또는 확장된 14개) 변수 모두에 대해 projection weight 1을 가져야 정정 성공 확률이 높음(단순히 unsatisfied check만 늘리면 오히려 더 큰 trapping set으로 전이).
6. 두 가지 탐색법: (1) genie-aided random search - 오류 변수 사전지식으로 조건을 만족하는 선형결합 탐색, (2) structured search(S1/S2/S3) - unsatisfied check 위치만으로 trapping set의 basic/expansion 변수·체크 구조를 이용해 후보 redundant row를 조합.
7. structured search는 (12,4)→(14,4) 확장 시 basic trapping set과 expansion 변수/체크의 연결 패턴(2가지 configuration)을 이용해 기존 parity-check 행들의 선형결합만으로 후보를 생성(부가 연산 저비용).
8. Section V에서 Lovász Local Lemma 기반으로 redundant row 개수 `m`과 trapping set 파라미터 `(a,b)` 사이의 관계를 규명하는 확률적 존재 정리(Theorem 5.2) 제시 - 특정 조건을 만족하면 trapping-set-free한 parity-check matrix가 존재.
9. 결과: 15개 오류 프레임 중 대부분이 redundant row 1개 추가만으로 정정 성공(반복 횟수 13~28회), waterfall 영역 성능에는 영향 없음을 시뮬레이션으로 확인.
10. 한계: 특정 코드(Margulis)의 특정 trapping set 형태(point, elementary, (12,4)/(14,4))에 국한된 case study이며, redundant row 추가 시 디코더를 재시작해야 하는 2-pass 구조(recovery 시점 검출 필요), HW 구현/복잡도 분석은 없음.
