### B. 알고리즘 요약

1. Binary erasure channel(BEC, 소거확률 `ε`)을 통한 전송에서 degree distribution 쌍 `(λ, ρ)`로 정의되는 표준 irregular LDPC ensemble의 유한길이 블록/비트 소거확률을 해석적으로 근사.
2. 문제: 기존 covariance evolution(scaling law의 정확한 파라미터 계산법)은 ODE 시스템 크기가 최대 check 차수 제곱에 비례해 수백~수천 차원이 되어 최적화 반복에 쓰기엔 너무 느림.
3. 채널/모델: BP decoding은 peeling algorithm으로 모델링(잔여 그래프에서 degree-1 check node 제거를 반복), density evolution으로 `r1(y)`(잔여 degree-1 check 비율) 추적.
4. 핵심 기법 1단: 오차확률을 대규모 소거(waterfall, `P_W ≈ Q((√n(ε*-ε))/α)`)와 소규모 소거(error floor, minimal stopping set 합)로 분리하고, waterfall의 scaling parameter `α`를 covariance evolution을 풀지 않고 닫힌 형태 수식(Lemma 1)으로 직접 유도.
5. `α`는 density evolution 임계점 `y*`, `x*=ε*λ(y*)` 근방에서의 message variance `V*`(Lemma 3, 4)로부터 유도되며, 이 variance는 유한 반복수 `V(ℓ)`의 재귀식 극한으로 계산.
6. error floor는 minimal stopping set 개수의 점근적 Poisson 분포(Lemma 2)를 이용해 `A_s`(size-s stopping set 기대개수)를 로그변환한 `Ã_s`로 union bound를 구성.
7. 부수 트릭: 두 근사(waterfall+floor)를 합쳐 전체 곡선 `P(n,λ,ρ,ε)`(식 17,18)을 구성, 각 SNR/ε에서 한 항만 지배적이라 union bound가 tight.
8. 최적화 절차: 해석적 근사의 gradient(∂P/∂λi, ∂P/∂ρi)를 이용한 선형계획법(LP1: rate 증가, LP2: 목표오차 이하로 낮춤) 반복으로 degree distribution을 로컬 최적화, 1회 반복이 수 초, 전체 수 분.
9. 결과: n=5000 예제에서 rate 0.2029 → 0.41065로 최적화, 시뮬레이션과 해석적 근사가 거의 완벽히 일치(Fig.3), LP1 스텝당 초 단위 소요.
10. 한계: BEC 채널 전용(AWGN/일반 채널 확장은 향후 과제로 언급), BP는 peeling algorithm 모델, HW 미설계, ensemble 수준 최적화라 특정 QC-LDPC 구조(protograph 등)로의 확장도 future work.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| degree distribution(λ,ρ) 최적화 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 설계 레이어에 대응되나 BEC 전용 scaling law라 binary AWGN/RBER QC-LDPC로 직접 이전 불가 |
| BEC peeling-algorithm decoding 모델 | 대응 없음 | Prime ECC는 min-sum LLR 기반 디코더(`decoder.cpp`)이며 소거채널 peeling과 알고리즘 성격이 다름 |
| error floor(minimal stopping set) 해석적 근사 | 대응 없음 | trapping-set이 아닌 stopping-set(소거채널 특유 개념) 분석으로 binary hard/soft LLR 디코더에는 직접 적용 안 됨 |

> 적용 가치: **낮음** — BEC 채널 전용 ensemble 최적화 이론으로 채널 모델과 디코더(peeling)가 Prime ECC의 AWGN/RBER min-sum QC-LDPC와 근본적으로 달라 이식 요소가 거의 없음.
