### B. 알고리즘 요약

1. LDPC-CC는 무한(주기적 시변) parity-check 행렬 `H^T`로 정의되는 convolutional 코드로, memory `m_s`와 주기 `T_s`를 가지며 rate b/c(예제는 rate-1/2, 1/3) 코드.
2. 유한 길이 데이터 프레임에 적용하려면 인코더를 all-zero state로 되돌리는 termination이 필요한데, 기존 방법은 phase별 termination 행렬 `F` 저장량이 `O(m_s^3)`로 과도하거나 padding으로 rate 손실 발생.
3. 인코더 상태는 최근 `m_s`개 정보비트/패리티비트와 phase `φ`로 정의되며, termination sequence `x^T = F_{φ,L} s^T`는 상태벡터의 선형결합.
4. 제안 기법(all-phase termination)은 매 state transition마다 termination bit 하나씩을 반복 계산해 임의의 phase에서 시작해도 padding 없이 종료 가능.
5. 핵심 증명: Gaussian elimination의 reduced row echelon form에서 자유변수(free variable)를 모두 0으로 고정하면, 매 단계 계산된 해가 원래 경로(original path)에서 벗어나는 detour가 발생하지 않음을 증명.
6. 저장 요구량이 `O(m_s^2)`(Ts × 2m_s, 각 termination 행렬의 첫 행만 저장)로 기존 대비 대폭 감소, L에 무관.
7. HW 구현: shift register, mux, XOR 게이트 기반 인코더 구조에 termination matrix 곱셈을 추가하는 형태로 설계 (Fig. 4).
8. 별도 학습/최적화 절차 없음 (구성적/증명 기반 기법).
9. 결과: ms=128~2048 LDPC-CC에서 termination 실패 없음을 시뮬로 확인; ms=128 기준 termination length L=134(≈ms); ASIC 합성 결과 인코더가 반복 디코더 대비 매우 작아 throughput 병목 아님(수치 미기재).
10. 한계: LDPC-CC(무한/시변 convolutional 구조)에 특화된 기법으로 QC-LDPC block code에는 직접 적용 불가; error performance/BER은 본 논문 범위 밖(참고문헌([4],[5])에서 다룸).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-CC 인코더 termination | 대응 없음 | Prime ECC 3.1은 QC-LDPC block code로 convolutional termination 개념 자체가 없음 |
| dual-diagonal 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()` | 부호 구조가 달라 참조만 가능, 직접 대응 없음 |
| shift-register 기반 encoder HW | 대응 없음 | Prime ECC 인코더는 dual-diagonal 고정 구조로 shift-register FSM 구조와 무관 |

적용 가치: 낮음 — LDPC-CC(convolutional code)는 Prime ECC 3.1의 QC-LDPC block code와 부호 패러다임 자체가 달라 인코더 termination 기법을 이식할 대상 모듈이 없음.
