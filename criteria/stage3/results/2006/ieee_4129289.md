### B. 알고리즘 요약

1. 대상은 LDPC 코드의 Tanner graph(parity-check 행렬 H, m행 n열)에서 발생하는 짧은 cycle(4/6/8/10-cycle) 개수를 세는 문제.
2. 기존에는 cycle 개수를 정확히 세는 체계적 방법이 부족했고, girth(최단 cycle 길이)만으로는 BER 성능을 예측하기 어렵다는 문제의식.
3. 별도의 채널/노이즈 모델 정의 없음 (H-matrix 구조 분석 + AWGN 채널에서 sum-product 디코딩 시뮬레이션으로 검증).
4. 핵심 기법: H에서 `Cm2`, `Cm3`, `Cm4`, `Cm5` 개의 row 조합을 각각 취해, 각 row에 1,2,4,8,16의 가중치를 부여한 뒤 합산한 "sum row"에서 특정 값(예: 4-cycle은 "3")이 나타나는 열 개수(`num_3` 등)를 세어 cycle 수 `p(i)`를 산출, 전체 조합에 대해 합산.
5. 예: 4-cycle 개수는 `p(i) = C(num_3, 2)`이고, 전체 4-cycle 수는 모든 2-row 조합에 대한 `p(i)`의 합 — 두 행에 동시에 1이 있는 열 쌍의 개수가 곧 그 두 행이 만드는 4-cycle 개수라는 조합적 관찰.
6. 8-cycle/10-cycle은 4행/5행 조합에서 여러 num_x 값들의 곱의 합으로 확장(예: `num_3×num_12×num_5×num_10 + ...`), 4-cycle이 없다고 가정하고 계산.
7. 별도의 근사/양자화/HW 변형 트릭은 없음 — 순수 조합 계산(row 가중치 합산 후 특정 정수값 카운팅) 절차.
8. 학습/최적화 절차 없음 (알고리즘은 결정적 카운팅).
9. 결과: MacKay 랜덤 LDPC(96,48)와 Fossorier 방식 QC-LDPC(108,54) 비교 — QC-LDPC가 girth 면에서 우수(6-cycle 0개 vs 191개)하지만 BER 성능은 random LDPC가 더 우수함을 시뮬레이션으로 확인.
10. 한계: 코드 길이 96/108의 소규모 예제뿐이며, 디코더 알고리즘·HW 설계·부호 최적화 기법은 전혀 제시하지 않는 순수 분석/카운팅 방법론 논문.
