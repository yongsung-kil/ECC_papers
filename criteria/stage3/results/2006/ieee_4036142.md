### B. 알고리즘 요약

1. 시스템: HARQ(증분 재전송)용 rate-compatible protograph LDPC 부호군, rate 범위 0.1~0.9, payload 1K/4K.
2. 문제: 기존 구조적(protograph-level) puncturing은 가능한 rate 집합이 매우 작고, 이미 punctured(hidden) node가 있어 추가 puncturing 시 stopping set이 형성되어 BP가 시작조차 못함.
3. 가정: AWGN 채널 + erasure 채널 모두에서 좋은 threshold를 갖도록 설계, DE(density evolution)로 asymptotic 성능 분석.
4. 핵심 기법 1단: `Algorithm 1` — base protograph `B`(rate `R_B`)를 α배 복제 후 PEG로 연결, β개 systematic 입력노드를 pruning하여 저rate protograph `L`(rate `R_L`) 생성. `β = αK(R_B-R_L)/(R_B(1-R_L))`.
5. 핵심 식 의미: pruning 후 그래프가 `Bα`의 부분그래프이므로 girth가 최소 `Bα`의 girth 이상 보장 → error floor 낮춤에 유리.
6. 핵심 기법 2단: 저rate 부모부호에서 고rate로 가는 puncturing 3종 — (a) Regular-Irregular puncturing(`Algorithm 2`, 저복잡도), (b) Random Puncturing Search(`Algorithm 3`, DE 기반 SNR 임계값 탐색 greedy), (c) Progressive Node Puncturing(PNP, `Algorithm 4/5`, 체크노드당 punctured 이웃 수 최소화+연결성 최대화로 error floor 개선).
7. 부수: puncturing score `S(c)`를 2-hop support tree 확장으로 정의해 체크노드 연결성 지표화(PNP-II).
8. 최적화: DE로 각 rate 후보의 AWGN/erasure threshold를 이분탐색.
9. 결과: 채널용량 대비 0.5~0.8dB 이내 threshold, 3GPP turbo 코드 대비 FER 10^-2에서 약 0.3dB 이득, throughput 약 200% 개선(연산량 관점).
10. 한계: HW 설계 전무, 순수 이론(DE)+FER 시뮬, HARQ/무선 통신 타깃이라 NAND 채널 특성과 무관, 가변 rate(0.1~0.9) 프레임워크로 NAND의 단일 고정 고rate 설계와 성격이 다름.
