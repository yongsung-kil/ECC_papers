> 총평: SDR 다중 프로세서 상호연결망(MIN)의 라우팅 충돌을 줄이기 위한 code-design 제약 조건 논문으로, NAND 고전 binary QC-LDPC HW(고정 배럴시프터 기반) 이식성은 낮음.

### B. 알고리즘 요약

1. SDR(software defined radio) 플랫폼은 다수의 processing unit(PU)과 storage unit(SU)이 multi-stage interconnection network(MIN, 예: butterfly, perfect shuffle)로 연결된 구조.
2. LDPC 부호를 MIN 구조 고려 없이 그대로 매핑하면 라우팅 충돌이 발생해 디코딩 처리량이 저하되는 문제를 해결하려 함.
3. 채널은 AWGN + BPSK 변조를 가정하고, 부호는 `Hb` (mb×nb block matrix, 원소는 Z×Z circulant shift 또는 0)의 QC-LDPC.
4. 입력 노드 X에 대해 stage i에서 같은 2x2 스위치를 공유하는 이웃 그룹 `Gi(X)`를 정의하고, exclusive node set `Ei(X) = {X}∪G0(X)∪...∪Gi(X)`를 도입.
5. 핵심 제약: 동일 행(row)에서 같은 ENS에 속한 두 개 이상의 '1'이 있으면 라우팅 충돌 발생 → 각 행에서 ENS당 최대 1개의 '1'만 허용하도록 `Hb` 구성.
6. 추가로 인코딩 효율(lower-triangular 구조)을 위해 `nb ≤ N`, `dc ≤ N/2 + d'min`, `dv_max ≤ mb - dv_min` 등의 열/행 가중치 제약을 유도.
7. 구현 디테일: degree distribution은 기존 density evolution 온라인 툴로 asymptotic 최적화 후 정수 라운딩(borrow-from-asymptotic), Hb 생성은 무작위 배치 후 ENS 제약을 만족하도록 열/행 교환(swap) 반복.
8. 최적화 절차는 기존 DE 결과를 그대로 사용하고, 코드 구성 단계(step v)에서 MIN 제약만 추가 적용(별도 학습 없음).
9. 결과: rate 3/4, nb=32, mb=8, Z=97 butterfly MIN 32점 예제에서 랜덤 Hb 대비 충돌 4.56%→0%, 슬로우다운 34%→0%; BER은 waterfall 영역 0.2dB 이내 열화, error floor는 오히려 더 낮음.
10. 한계: 특정 SDR 멀티프로세서+MIN 아키텍처 전제(HW 실제 설계 없음, 시뮬레이션뿐), NAND ECC의 고정 배럴시프터/HCU 구조와는 매핑 모델이 상이해 직접 이식 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Hb (QC-LDPC block matrix) 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 단계에 해당하나 MIN 특화 제약은 Prime ECC 아키텍처와 무관 |
| 라우팅 충돌 회피(ENS 제약) | 대응 없음 | Prime ECC는 고정 z=32 배럴시프터/HCU 기반이라 MIN형 상호연결망 충돌 문제 자체가 없음 |
| degree distribution 최적화 | 대응 없음 | density evolution 기반 asymptotic 설계로 코드 자체 최적화이며 디코더/HW 모듈과 직접 대응 없음 |
| 병렬 디코딩 스케줄(row-based layered BP) | `decoder.cpp` `LDPC_Decoder()` | 레이어드 BP 스케줄 개념은 일반적이나 이 논문의 기여는 아님 |

적용 가치: **낮음** — SDR 특화 다중 프로세서/MIN 상호연결망 아키텍처를 전제로 한 code-design 제약 기법으로, NAND ECC의 고정된 QC-LDPC HW 구조(z=32 배럴시프터)와 매핑 모델이 근본적으로 달라 이식 실익이 없음.
