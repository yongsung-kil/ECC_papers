### B. 알고리즘 요약

1. AWGN(BI-AWGN) 채널, protograph 기반 QC-LDPC 코드로 fixed block length(`n=5792`, `circulant size=181`)에서 rate `1/2, 5/8, 3/4, 7/8`을 하나의 protograph에서 파생.
2. 문제의식: 기존 RA/IRA/ARA 계열은 threshold는 낮지만 minimum distance가 sublinear해 error-floor가 높음. Degree-2 variable node를 쓰지 않고 degree≥3만 사용하는 것이 linear minimum distance 성장의 충분조건(必要조건은 아님)임을 이용.
3. 핵심 가정: protograph(소규모 Tanner graph)를 "copy-and-permute"로 N배 확장(lifting)해 원하는 크기의 derived graph/코드를 생성, 동일 rate·degree 분포 유지.
4. 핵심 기법 1단: rate 1/2 protograph를 degree-3 노드 다수 + 고차수 노드 1개(`node degree 16~18`)로 비대칭 연결 설계 → RCA(reciprocal channel approximation, `s+r` additive parameter, `R(x)=C^-1(1-C(x))`)로 iterative decoding threshold 계산.
5. 핵심 식(RCA): 변수노드에서 `s_e` 합산, 체크노드에서 `r_e` 합산 후 `R(·)` 변환 반복 → threshold는 모든 메시지가 무한 성장하는 최소 `s_chan`; 이는 density evolution의 빠른 근사(오차 0.005dB 이내)로 계산량을 줄여줌.
6. 핵심 기법 2단(rate-compatible 확장): degree-2 non-transmitted(punctured) node로 check node들을 결합(constraint combining)해 더 높은 rate 코드를 생성 - 모든 constraint를 하나로 합치면 최고 rate.
7. 부수 트릭: 8-node/11-node protograph를 progressive edge growth(4배 lifting) + ACE 알고리즘으로 circulant(크기 181) 위치 선정해 다중 병렬엣지 제거, 결과 H-matrix는 QC 형태(순환치환 `x^i`)로 명시(Table I).
8. 최적화 절차: 소규모 protograph에 대해 hand-selected search로 노드 연결을 비대칭화해 threshold를 1.102dB→0.618dB(8노드)까지, 12노드/degree18 구성에서는 0.544dB까지 낮춤.
9. 결과: rate 1/2 협상 threshold 0.618dB(Shannon 대비 gap 0.43dB); rate 1/2,5/8,3/4 코드는 FER 3×10^-7까지 error-floor 없음, rate 7/8은 trapping set으로 인해 10^-6 부근 error-floor 발생(FPGA 시뮬, JPL Universal Decoder for Sparse Codes).
10. 한계: HW 게이트카운트/throughput/파이프라인 분석 없이 FPGA는 BER/FER 정확도 검증용으로만 사용; degree≥3 제약은 우주/위성 통신급 rate 1/2~7/8 코드에 최적화되어 NAND ECC의 초고rate(~0.9)와는 다른 설계점.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Protograph 기반 QC-LDPC H-matrix 구성(degree≥3 제약, ACE 기반 circulant 선정) | `ecc_top.cpp` `Load_PCM()` | 부호구조 설계 단계에 해당하나 Prime ECC는 특정 QC-LDPC(base 129 col-block, VN degree 17)로 고정되어 재검증 부담이 큼 |
| Degree-2 non-transmitted node를 통한 rate-compatible 확장(constraint combining) | `ecc_top.cpp` `Load_PCM()` | rate-compatible 부호설계 아이디어이나 Prime ECC는 단일 고정 rate 구조라 직접 적용 어려움 |
| BP 기반 반복복호(RCA/density evolution은 설계용, 실제 복호는 표준 sum-product 암시) | `decoder.cpp` `LDPC_Decoder()` | 복호기 자체는 표준 BP로 min-sum과는 다르나 이터레이션 루프 구조는 대응 |
| rate 7/8에서의 trapping set으로 인한 error-floor | 대응 없음 | 특정 trapping set 대책(post-processing 등)은 본문에 제시되지 않음 |

적용 가치: `중간` — degree≥3 제약과 rate-compatible protograph 설계 방법론은 고전 QC-LDPC 부호설계에 참고할 수 있으나, Prime ECC는 이미 고정된 특정 QC-LDPC 구조를 사용하므로 새 H-matrix 재설계에는 높은 검증 비용이 따름.
