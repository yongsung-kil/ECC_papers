### B. 알고리즘 요약

1. 시스템: AWGN + BPSK 무선 채널, regular QC-LDPC 코드, rate 0.73~0.9, 길이 3645~6860 예제.
2. 문제: random LDPC는 우수한 성능이나 parity-check matrix의 비영 위치 지정에 큰 정보량이 필요해 복잡도가 높음. 구조화된 QC-LDPC는 shift-register 기반 저복잡도 인코딩이 가능하나 기존 클래스(Array, BIBD)는 설계 자유도가 제한적.
3. 핵심 가정: GF(2)상의 permutation matrix들의 집합 `Q`와 유한체 `F_q` (`q=p^n`) 사이에 bijective map `M: F_q → Q`가 존재.
4. 핵심 기법: `P(a)`(순열행렬)들의 tensor product로 큰 H-matrix `Θ`를 구성 (`Θ = [[P(a_i·a_j)]]`, 식 (3)/(7)), 임의의 row/column 부분집합을 선택해 원하는 row weight `ρ`, column weight `γ`를 갖는 H-matrix 생성.
5. 핵심 식 의미: Theorem 1의 nc4-matrix 조건(부분행렬이 모두 nc4면 전체도 nc4)으로 girth≥6을 보장, 즉 4-cycle 없는 QC-LDPC를 파라미터 몇 개(`p`, `n`, 선택 row/col 수)만으로 생성 가능.
6. 확장: prime field 기반 구성(§III-b)에서는 순환 shift 행렬 `A^c`를 이용해 동일한 nc4 특성을 갖는 대안 구성법 제시, Array LDPC는 이 구성법의 특수 case(식 (8)).
7. 부수 트릭: 특정 파라미터(`p=2`, 낮은 column weight)를 선택하면 인코딩/디코딩 복잡도를 더 낮출 수 있으나 성능 저하 trade-off 존재.
8. 학습/최적화 절차 없음 — closed-form 파라미터 기반 구성(GA/DE 등 최적화 불필요).
9. 결과: (6760,6084)/(6250,5625)/(6860,6174) 등에서 permutation LDPC가 동일 길이의 random LDPC(MacKay code-6)와 bER/BLER이 유사하거나 저error율 구간에서 소폭 우수. (3645,2673) 코드에서 iteration 4/8/16/50 비교로 반복 횟수 증가에 따른 성능/복잡도 trade-off 제시.
10. 한계: HW 설계 없음, 시뮬레이션(sum-product, 최대 50 iteration)뿐. 무선 채널(AWGN/BPSK) 대상이며 NAND read channel 특화 요소 없음. Error-floor 분석이나 trapping set 등 NAND 저rate 저에러 영역 특화 논의 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 일반화 QC-LDPC 구성법(permutation matrix tensor product) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 이미 base matrix+circulant shift 기반 QC-LDPC(`PCM_b`)를 사용 중 — 새 구성법 채택 시 H-matrix 재설계·재검증 필요, 이식 난이도 높음 |
| girth≥6 보장(nc4-matrix 조건, Theorem 1) | 대응 없음 | H-matrix 설계 시 이론적 참고 가능하나 Prime ECC 코드베이스에 구현된 구성 절차 없음 |
| shift-register 기반 저복잡도 인코딩 | `encoder.cpp` `Generate_PCM_encoding()` | QC-LDPC 순환 구조는 개념적으로 유사하나 Prime ECC는 dual-diagonal 인코딩 고정 방식이라 직접 대응은 아님 |

> 적용 가치: 중간 — QC-LDPC 구성법 자체는 Prime ECC와 부호 종류가 일치하나, 부호설계 레이어 이식은 난이도가 높고(H-matrix 전면 재검증 필요) 성능 개선 폭도 random LDPC 대비 제한적이라 우선순위는 높지 않음.
