> 총평: 무선통신 재전송(ARQ) 프로토콜에 LDPC LLR 기반 재전송 비트 선택을 결합한 시스템 레벨 스킴으로, decoder/encoder/부호설계 어느 것도 개선하지 않아 NAND ECC 이식 가치가 없음.

### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 채널에서 (1792,3,7) rate 4/7, (1057,3,13) rate 10/13 regular LDPC 코드를 사용하는 HARQ(Hybrid ARQ) 프로토콜.
2. 문제: 기존 RB-HARQ(Reliability-Based HARQ)는 디코더 출력 LLR 크기 순으로 비트별 재전송 인덱스를 피드백하므로 피드백 비트수가 `log2(N) x l`로 큼.
3. 가정: 이상적인(무손실) 피드백 채널을 가정, decoding 실패 시에만 재전송 발생.
4. 핵심 기법: RB-HARQ(row base)는 개별 비트 대신 parity check matrix의 "row"를 선택 단위로 사용 - 신뢰도 낮은 비트를 포함하는 row를 골라 그 row에 속한 (k-1)개 비트를 함께 재전송 요청.
5. 핵심 식: 선택 기준은 `min_m |z_i| / R_m` (R_m = 이미 선택된 비트 수 반영한 남은 미선택 비트 수)로, row당 재전송 효율을 정규화해 비교.
6. 피드백 비트수 비교: RB-HARQ는 `log2(N) x l`, row base는 `log2(M) x l/k`로 고rate·row weight k가 클수록 절감폭이 커짐.
7. 부수: soft decision 기반 LLR magnitude ranking을 사용하나 별도 양자화/HW 근사 언급 없음.
8. 학습/최적화 절차 없음(경험적 시뮬레이션 비교뿐).
9. 결과: (1792,3,7) 코드에서 재전송 2회 시 RB-HARQ가 no-ARQ 대비 BER=1e-4에서 약 0.8dB, row base는 약 0.5dB 이득; 피드백 비트는 1100→150 bit로 감소.
10. 한계: 알고리즘/HW 설계 없음, 순수 시뮬레이션(AWGN, ideal feedback channel) 기반이며 NAND에는 재전송 개념이 없어 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ARQ 재전송/피드백 프로토콜 | 대응 없음 | Prime ECC는 저장 채널 ECC 시뮬레이터로 재전송/피드백 프로토콜 개념 자체가 없음 |
| LLR 기반 비트 신뢰도 랭킹 | `decoder.cpp` `VN_Cal_HD()` 등 (LLR 산출 자체는 존재) | 랭킹·재전송 목적이 아니라 디코딩용이므로 간접적 관련성만 있음 |
| parity check matrix row 구조 활용 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체는 대응되나 사용 목적(피드백 절감)이 상이 |
