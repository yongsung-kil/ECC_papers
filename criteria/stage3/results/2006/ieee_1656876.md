### B. 알고리즘 요약

1. 대상은 irregular LDPC (Tanner graph: VN degree 분포 `f[j,...,2]`, CN degree `g[k,k-1]`), DVB-S2(rate 1/4~9/10, 길이 64800bit), 4MORE(rate 4/5, 길이 1000/2000/3000bit), 커스텀 1mm² 디코더(rate 1/4~4/5, 길이 200~1000bit) 3개 사례를 다룸.
2. 문제의식: 기존 LDPC 디코더 HW 구현들이 특정 code subclass 또는 단일 코드에만 최적화되어 있어(예: [6]), 코드율·블록길이 유연성(service flexibility)을 지원하는 범용 아키텍처가 부족함.
3. 핵심 모델: VN 업데이트는 `λ_k = λ_ch + Σ_{l≠k} λ_l`(식2), CN 최적 업데이트는 `tanh(λ_k/2) = Π_{l≠k} tanh(λ_l/2)`(식3, Sum-Product), 서브옵티멀은 Min-Sum(`λ_k = Π sign(λ_i)·min|λ_i|`, MSF=0.75)과 λ-3-Min(최소 3개 값 사용 + 보정항)을 병행 사용.
4. 핵심 기법: 설계공간(Table 1)에서 LDPC code/algorithm/iteration/quantization/parallelism이 area·throughput·통신성능·유연성과 서로 얽혀 있음을 정리 — 특히 code rate 증가 시 irregularity 감소(→성능 저하), throughput 증가, 필요 iteration 감소, 단순 알고리즘 적용 가능이라는 트레이드오프 사슬을 규명.
5. 아키텍처 템플릿(Fig.2): partly-parallel, CN 처리는 clock당 1 message로 직렬화(메모리 충돌 최소화·유연성 확보), 두 개의 barrel shifter로 message exchange network 구성, 두 개의 Sum RAM을 iteration마다 interleave.
6. 부수 트릭: Message RAM/Sum RAM/Channel RAM 분리 구조, Code Vector(코드별 barrel shifter 제어 시퀀스)를 하드코딩 또는 런타임 재로딩 가능하게 해 permutation-matrix 기반 임의 LDPC 코드를 지원.
7. 학습/최적화 절차 없음(설계공간 탐구 + 3개 설계 사례의 ASIC/FPGA 합성 실험).
8. 결과 수치: DVB-S2 설계 21.398mm²@0.13µm/270MHz, 6bit 양자화, λ-3-Min 50 iteration, throughput 157~542Mbps; 4MORE(FPGA XC4VLX100) Min-Sum+MSF, 10 iteration, 131~180Mbps; 1mm² 디코더 2종(throughput판 4bit/Min-Sum/10iter/163~324Mbps, 통신성능판 6bit/λ-3-Min/40iter/15~47Mbps), 960bit·rate1/2에서 UMTS Turbo Code와 동등 성능 달성.
9. 한계: 코드율/블록길이 유연성이 핵심 기여이나 통신·방송 표준(DVB-S2, 4MORE, UMTS) 타깃이라 NAND의 고정 고rate·대용량 요구와는 설계 목표가 다름; 알고리즘 자체(Min-Sum, λ-3-Min)는 이미 잘 알려진 기법.
10. 전반적으로 실측/ASIC합성 수준의 신뢰도 높은 HW 설계 사례연구이나, 신규 디코딩 알고리즘이나 error-floor 개선 기법은 제시하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum / λ-3-Min CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 Min-Sum 기반이며 λ-3-Min은 3-min 근사로 min1/min2 확장형에 해당, 이미 보유 개념과 유사 |
| barrel shifter 기반 message exchange network | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 MUX 기반 edge 선택과 유사한 역할(메시지 라우팅 HW), 직접 대응은 아님 |
| 가변 code rate/block length 지원 아키텍처 | `ecc_top.cpp` `Load_PCM()` | 다중 H-matrix/코드 지원 구조와 관련되나 Prime ECC는 특정 QC-LDPC 고정이라 이식 난이도 높음 |
| 병렬 CN Unit / VN Unit + RAM 분리 구조 | 디코더 HW 모델 (z=32, HCU, GT) | partly-parallel 처리 개념은 Prime ECC의 lifting 기반 병렬화(z_sb=32)와 유사한 설계 철학이나 구조는 상이 |

> 적용 가치: **중간** — 알고리즘(Min-Sum/λ-3-Min) 자체는 이미 보유 중이나, 코드율/블록길이 유연성을 지원하는 재사용 가능 디코더 아키텍처 설계 철학과 설계공간 트레이드오프 표(Table 1)는 참고 가치가 있음.
