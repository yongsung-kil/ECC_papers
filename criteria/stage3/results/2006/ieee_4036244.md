### B. 알고리즘 요약

1. 작은 protograph(변수/체크노드 소수)를 copy-and-permute하여 얻은 irregular LDPC block code 구성법을 convolutional code로 확장한다.
2. 기존 regular LDPC convolutional code(unwrapping 방식)는 irregular protograph 구조를 살리지 못해 성능이 제한된다는 문제를 해결하려 함.
3. protograph의 edge를 unpermutable(고정, solid line)과 permutable(dashed line)로 나누고, 후자만 copy 간 permute하여 loop-free encoding module을 구성(`식 (2)` `H^T_np`).
4. protograph를 `ms`번 copy한 뒤 대각 블록 하단 서브행렬을 unwrap하고 주기 `T=ms+1`로 반복해 semi-infinite syndrome former matrix `H^T`(rate `R=b/c`)를 생성.
5. `H^T`의 마지막 `(c-b)`행을 항등행렬로 두어 정보비트 `u_t`로부터 shift-register 기반 systematic 인코딩이 가능(`인코더 그림 5`, 메모리 `ms·c+b`).
6. 부호 구성 시 cycle 회피를 위해 수정된 Progressive Edge-Growth(PEG) 알고리즘을 사용하며 unpermutable edge를 먼저 연결 후 나머지를 PEG로 연결.
7. 실험은 9-VN/5-check protograph(rate 1/2, threshold σ*=0.9683, `Eb/N0=0.2798dB`, discretized DE 11-bit 양자화)를 사용.
8. 별도 학습절차 없음(구성적 설계, PEG 기반 결정론적 edge 배치).
9. BER 10^-5에서 동일 processor 복잡도의 block LDPC(n=2160) 대비 1.4dB, n=10800 대비 0.2dB 이득; ms=239는 waterfall은 좋으나 free distance가 작아 error floor 발생.
10. HW 구현은 다루지 않으며(시뮬 전용), n=108000 대형 block LDPC보다는 약간 열위이고 error floor 원인 규명은 향후 과제로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph 기반 semi-infinite H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 block QC-LDPC이며 convolutional(semi-infinite) 구조 미지원 |
| pipeline decoding(convolutional 특성) | `decoder.cpp` `LDPC_Decoder()` | 개념적 pipeline은 유사하나 SC-LDPC 고유의 sliding-window 디코딩과는 다름 |
| shift-register 기반 systematic 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()` | dual-diagonal 인코더와 구조적으로 상이, 직접 대응 어려움 |

적용 가치: 낮음 — protograph 기반 SC-LDPC 구성법은 semi-infinite parity-check matrix와 pipeline decoding을 전제로 하여, block 기반 고정 QC-LDPC인 Prime ECC 3.1 구조와 근본적으로 다름.
