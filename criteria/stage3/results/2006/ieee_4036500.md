> 총평: 무선 hybrid ARQ 특화 rate-compatible IRA 코드 설계로 NAND의 고정-rate 단일 QC-LDPC ECC 구조와 목적/제약이 근본적으로 달라 이식 가치가 낮음.

### B. 알고리즘 요약

1. Rate-compatible(RC) 코드는 상위 rate 코드워드가 하위 rate 코드워드에 내포되는 구조로, hybrid ARQ에서 재전송 시 추가 parity bit만 보내 incremental redundancy를 구현하며 하나의 encoder/decoder 쌍만으로 다중 rate를 지원.
2. 기존 puncturing(고rate) 또는 extending(저rate) 단독 방식은 mother code가 극단적 rate를 가져야 넓은 rate 범위를 커버할 수 있어 유한길이 설계가 어려움 → 본 논문은 puncturing+extending을 결합한 hybrid 설계로 중간 rate(R=0.5) mother IRA 코드에서 출발.
3. IRA(Irregular Repeat Accumulate) 코드는 정보비트를 irregular repeat 후 인터리빙, accumulator로 parity 생성하는 systematic LDPC 특수구조(`λi`=degree-i 정보노드 엣지비율, `a`=accumulator 그룹factor로 rate 결정).
4. Puncturing: k-step recoverable(kSR) 노드 개념을 이용해, IRA 특유의 parity 노드가 지그재그 형태로 연결된 특성을 활용, 연속 J개 펑처 노드 중 최대 recoverable step이 `⌈J/2⌉`이 되도록 균등 배치하는 "deliberate puncturing" 후 역순으로 un-puncture하여 rate-compatibility 확보 (Algorithm 1).
5. Extending: 기존 degree-2 parity 확장 대신 degree-1 parity bit를 섞어 확장하는 신규 기법 제안, degree-1/degree-2 비율(`π_q,a`)을 density evolution(Gaussian approximation)으로 최적화(식 1-5).
6. 핵심 식: `h_q(s,r,r̃) = Σλi·h_{q,i}(s,r,r̃)`는 반복복호의 extrinsic message 갱신 함수로, rate-compatible 제약(식 2-3) 하에서 각 확장 단계의 목표 Eb/N0를 최소화하도록 `{λq,i}, aq, π*_q`를 최적화.
7. 실무적 단순화: 정보노드 degree를 인접한 두 값(`λq,i`, `λq,i-1`)만 쓰도록 제한하고 그룹factor `a`의 최댓값을 제한해 사이클 제어가 쉬운 유한길이 코드로 구현.
8. 학습/최적화: density evolution 기반 profile 최적화(6절 설명은 없으나 5절에 명시된 finite-length 설계 전략[8] 적용).
9. 결과: K=512 mother code(R=0.5)에서 puncturing으로 R=0.6~0.95, extending으로 R=0.1~0.4까지 FER 성능 우수; hybrid ARQ throughput에서 SNR>3dB 구간에 기존 RCPT/RC-LDPC 대비 우위, cutoff-rate 크로스오버 지점을 η=0.82→0.94로 개선.
10. 한계: 무선 패킷 통신(AWGN/block fading, hybrid ARQ) 대상 시뮬레이션뿐이며 HW 구현 없음; NAND ECC처럼 고정 단일 rate로 운용되는 시스템에는 rate-compatible 구조 자체가 불필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| rate-compatible puncturing/extending | 대응 없음 | Prime ECC는 고정 rate 단일 H-matrix(`PCM_b`) 구조이며 ARQ 재전송에 따른 동적 rate 변경 개념이 없음 |
| IRA(accumulator 기반 인코딩) | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코딩으로 IRA의 accumulator 구조와 유사점이 있으나 코드 자체가 QC-LDPC 고정이라 재설계 부담 큼 |
| density evolution 기반 profile 최적화 | `ecc_top.cpp` `Load_PCM()` | 부호설계 이론 참고 가능하나 rate-compatible 목적함수 자체가 NAND ECC와 무관 |

적용 가치: `낮음` — hybrid ARQ를 위한 rate-compatible 코드 설계는 NAND ECC의 고정-rate 단일 코드 운용 방식과 목적이 근본적으로 달라 이식 필요성이 낮음.
