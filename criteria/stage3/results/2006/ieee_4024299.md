### B. 알고리즘 요약

1. Base code(tail-biting trellis, 길이 m)와 bipartite graph(λ 차수분포)로 코드를 구성하는 일반 프레임워크를 제시하며, LDPC·turbo-code를 특수 사례로 포함.
2. 기존 고성능 반복복호 코드(irregular LDPC/turbo)는 Shannon 한계에 근접할수록 최소거리가 작아져 error-floor가 높아지는 문제가 있음.
3. 채널 모델은 AWGN(및 erasure channel 이론분석)이며, LLR은 평균 `µ`, 분산 `2µ`인 Gaussian으로 가정.
4. 핵심 기법은 "codewords of partial weight 2 그래프" `G=(V,E)`를 정의해, 이 그래프의 사이클이 코드워드 weight에 대응함을 증명(Fact 1)하고 사이클을 선형 크기로 제한.
5. `C - R = λ̄(1-λ1)A`: erasure 채널에서 capacity gap이 base code EXIT chart와 variable node EXIT chart 사이 면적 `A`에 비례 — degree distribution 최적화 근거.
6. Family (A)~(E) 다섯 종류 base code(2/3/4-state trellis 변형, degree-1 section 포함)로 rate 1/4~1/2 코드 구성.
7. EXIT chart curve-fitting으로 degree distribution `Λ(x)` 최적화(예: family B `Λ(x)=0.5x+0.182x²+0.069x¹²+0.249x¹³`).
8. 최적화 절차는 EXIT chart 기반 degree distribution 탐색(GA/DE 유사, ten Brink 방식).
9. Family-B(rate 1/2, n=16000)가 동일 rate/length duo-binary turbo code 대비 WER 2·10⁻⁴에서 0.6dB 개선, error-floor 없이 BLER 10⁻⁶까지 확인.
10. HW 미설계, 순수 시뮬레이션 기반이며 AWGN/erasure 채널 가정에 한정, NAND 특유 채널·soft read 비용 고려 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Tail-biting trellis base code 구성 | 대응 없음 | Prime ECC는 QC-LDPC(base+circulant) 고정 구조로 trellis 기반 base code와 이질적 |
| EXIT chart 기반 degree distribution 최적화 | 대응 없음 | H-matrix/degree 설계는 `ecc_top.cpp` `Load_PCM()` 영역이나, 본 논문은 trellis 특화 최적화라 이식 난이도 높음 |
| Codewords of partial weight 2 그래프(사이클 제어) | 대응 없음 | error-floor 대책이나 QC-LDPC circulant 구조와 결합 방식이 불명확 |

적용 가치: 낮음 — turbo-code 대체용 코드설계 이론 논문으로 QC-LDPC 부호구조·HW 구현과 직접 연결점이 없음.
