> 총평: LDPC 코드의 minimum weight/distance 계산을 NP 문제에서 generator matrix 벡터 분석으로 대체하는 순수 이론 논문으로, 디코더/HW 이식 요소가 없어 Prime ECC 적용 가치 낮음.

### B. 알고리즘 요약

1. 대상: (N,j,k) 랜덤/quasi-regular LDPC 코드(rate `R=1-j/k`), AWGN 채널 가정.
2. 문제: LDPC 코드의 minimum weight `w_min`/minimum distance `d_min` 계산은 NP-complete이며 기존엔 확률적 탐색(probabilistic search)에 의존해 유한 탐색 시 정확한 값을 놓칠 수 있음.
3. 핵심 가정: parity check matrix `H`가 girth 4를 가지지 않아야 이하 정리들이 성립.
4. 핵심 기법: `H=[A B]`에서 generator matrix `G=[I, A^-1 B]` 구성 후, 코드벡터 `c_n = Σ u_n,i * g_i` (`g_i`는 G의 행벡터)로 표현.
5. 핵심 정리: `w_min(g) ≤ w_min(C)`, `w_min(g) ≤ d_min(C)`, `d_min(g) ≤ d_min(C)` — 코드셋 C 전체(2^M-1개) 대신 G의 M개 행벡터만 분석하면 하한을 얻을 수 있음 (girth-4-free 조건 하에서).
6. 부수 이론: `Θ = H^T H`의 대각선 외 원소가 모두 1이면 girth 4 없음(Theorem 2), 이를 전제 조건으로 사용.
7. 구성 절차: 4단계 랜덤 LDPC 구성(균등 랜덤 배치→redundant row 제거→even column parity 보정→length-4 cycle 제거, 최대 10 pass).
8. 학습/최적화 절차: 별도 최적화 없음, 랜덤 구성 후 이론적 하한을 계산하는 사후 분석.
9. 결과: (906,3,6) 코드에서 `w_min(g)=152 < w_min(C)≤190`, `d_min(g)=152/142 < d_min(C)≤193`. BER 시뮬(iteration 20회, AWGN)로 girth-4-free이지만 distance가 큰 코드가 양호한 BER을 보임을 확인.
10. 한계: 이론적 하한 제시일 뿐 부호 구성 최적화 알고리즘이 아니며, 디코더 알고리즘·HW 설계 요소 전무. girth-4-free 조건 하에서만 정리가 성립하는 제한적 적용범위.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Generator matrix 기반 weight/distance 하한 계산 | 대응 없음 | Prime ECC는 고정 H-matrix(`PCM_b`)를 사용하는 시뮬레이터로, 코드 설계 단계의 이론적 distance 분석 도구는 대응 모듈 없음 |
| girth-4 검사(Θ=H^T H) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 시 참고 가능한 구조적 성질이나 논문은 검증 절차만 제공, 구현 대응 없음 |
| 랜덤 LDPC 구성 절차 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정 구조, 랜덤 구성법과 무관 |

> 적용 가치: 낮음 — 순수 수학적 distance 하한 분석 논문으로 디코더 알고리즘, HW 구조, 양자화 등 Prime ECC 이식 대상 레이어 어디에도 해당하지 않음.
