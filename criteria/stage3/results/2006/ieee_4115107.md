### B. 알고리즘 요약

1. IEEE 802.11n 표준의 블록구조 irregular QC-LDPC 12종(길이 648/1296/1944, rate 1/2·2/3·3/4·5/6)을 지원하는 조인트 인코더/디코더.
2. 기존 아키텍처들은 단일 길이/rate만 지원해 유연성이 부족하다는 문제를 해결.
3. H2가 결정론적 dual-diagonal 유사 구조를 가져 `H2^-1H1` 고비용 계산 없이 재귀적 인코딩 가능(barrel shifter+XOR+register).
4. 디코더는 layered belief propagation(LBP) 사용, `L(qmj) = L(qj) - Rmj` 로 갱신하며 표준 BP 대비 수렴 2배 빠름.
5. check node 갱신은 modified min-sum 근사 `Rmj ≈ sign(...) * max(min|L(qmj)| - β, 0)` 로 tanh 비선형 함수의 HW 부담을 제거.
6. S(27/54/81)개 DFU(Decoding Function Unit)가 병렬 동작, flexible permuter로 APP/Check 듀얼포트 메모리에 라우팅.
7. 2-stage pipeline 구조 제안: 다음 layer가 이전 layer 업데이트 완료 전에 읽기 시작, 단 랜덤 non-zero 위치로 인한 read-write hazard 발생 시 stall 삽입.
8. 별도 학습/최적화 절차 없음(고정 표준 부호 사용).
9. 결과: 비파이프라인 90K게이트/77,760bit RAM, 파이프라인 195K게이트로 면적 증가하나 처리량 1.5~2배(SNR 2.2dB에서 1Gbps, 1.0dB에서 150Mbps) 향상.
10. 한계: FPGA/WARP 무선 실측 테스트는 "진행 중"으로만 언급, 본문에 구체 결과 없음. 802.11n 표준 부호 고정이라 NAND 채널 특화 설계는 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered BP + modified min-sum CN 갱신 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 min-sum 계열 기법과 유사, 신규 기여 낮음 |
| 재귀적 dual-diagonal 유사 인코딩(barrel shifter 기반) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 인코딩 구조 자체는 이미 보유, 재귀식 구현 참고 가능 |
| 2-stage pipeline + memory hazard stall 처리 | `decoder.cpp` (pipeline 6-stage 관련부) | HW pipeline 충돌 회피 아이디어는 참고 가치 있으나 Prime ECC는 이미 6-stage pipeline 보유 |
| flexible permuter + DFU 병렬 배치(S=27/54/81 가변) | 대응 없음 | 다중 code length 지원용 가변 병렬도, Prime ECC는 z_sb=32 고정 |

> 적용 가치: 낮음 | 802.11n 무선 표준 부호 기반이며 Prime ECC가 이미 보유한 layered min-sum/dual-diagonal/pipeline 기법과 중복도가 높고, NAND 채널 특화 요소가 없음.
