### B. 알고리즘 요약

1. VN phase를 제거하고 CN update만 순차 수행하는 staggered decoding(Yeo et al.)을 기반으로 함. 채널은 AWGN, 변조 BPSK.
2. staggered decoding은 누적 metric `μi`가 무한 발산해 소수 iteration 이후 BER floor가 나타나는 문제가 있음.
3. 이를 해결하기 위해 누적합을 leakage가 있는 가중합으로 대체: `μi[k] = hi·μi[k-1] + gi·εij[k]`.
4. 계수 `hi`, `gi`는 VN degree `dVi`와 직전 parity check 결과 `pj[k-1]`에 따라 table-기반으로 결정(표 I).
5. `hi`는 통상 `(1-1/dVi)`로 설정해 BP의 marginalization을 평균적으로 재현하며, `dVi=3`인 경우 flag 기반 재초기화/누적 전환 로직을 사용.
6. `gi`는 기본 1이나, 검증된 CN(`pj[k]=0`)이면서 `dVi>3`인 경우만 1/2로 감쇠(demagnification)해 과도한 성장 억제.
7. `dVi<3`인 경우는 정확한 log-BP와 동일(`μi[k]=εij[k]`)해 추가 메모리 불필요.
8. 채널 a priori LLR을 CN update 이전 metric에 더해(식 5,6) VN 없이도 traditional VN 방정식과 유사하게 동작.
9. WWiSE code(N=1944, R=1/2)에서 BP 대비 BER 10^-8까지 0.2dB 손실, throughput은 iteration time이 절반이라 약 2배; 5bit 양자화 시 gap이 0.05dB로 축소.
10. VLSI로 검증됨(0.18um, 1830 gates)이나 코드 독립적 일반화는 두 코드(WWiSE, MacKay 4095.738.4.102)만 검증했고 error floor 자체는 BP보다 열위.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 단일 phase CN 스케줄링(VN 제거, 누적 metric) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Dual-Update와 유사한 스케줄 아이디어이나 VN phase 완전 제거는 구조 변경 필요 |
| leakage 계수 hi/gi 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration/degree별 LLR threshold 테이블과 유사한 형태로 대응 가능 |
| 메시지 5bit 양자화 | `channel.cpp` `Set_LLR_Th()` | 양자화 비트수 절감 관점에서 참고 |
| VLSI 노드 프로세서(SPNP) 게이트 카운트 | `decoder.cpp` (CNU/VNU 통합 구조) | 메모리 절감 목적의 HW 구조로 참고 가치 낮음 |

적용 가치: 중간 — VN phase 제거로 인한 메모리 절감 아이디어는 참고할 만하나 정정능력 개선이 아니고 BP 대비 error floor 열위라 Prime ECC의 min-sum 구조에 직접 이식하기엔 재검증 부담이 있음.
