### B. 알고리즘 요약

1. BEC(Binary Erasure Channel) 상 반복복호 실패는 stopping set(잔여 소거 비트 집합)에 의해 결정되며, 짧은 사이클을 제거하면 작은 stopping set 발생 확률이 줄어듬.
2. 기존 PEG/ACE-minimization 알고리즘은 girth나 ACE(근사 EMD)를 간접 지표로 최적화해 실제 block error probability와 어긋나는 선택을 할 수 있음이 문제.
3. minimal cycle set을 `(d, τ)` (d=크기, τ=extrinsic check node 수)로 정의하고, 이 조합이 stopping set에 기여하는 확률 `P_cyc(d,τ)`을 union bound로 유도(식 1-3).
4. 핵심 식: `P_cyc(d,τ)` 상한이 d와 τ가 작을수록 커짐 → 이를 기준으로 cycle set 간 부분순서 `(d',τ') < (d,τ)`를 정의해 "minimal cycle set"만 추적하면 연산 가능한 수준으로 단순화.
5. 그래프 전체의 기대 block error probability를 각 variable node의 minimal cycle set spectrum `W_v` 합으로 상한 유도(정리 4.1, 식 5-8), 이는 긴 몬테카를로 시뮬레이션 없이 코드 우열 비교에 사용 가능.
6. EMPEG 알고리즘: PEG처럼 엣지를 순차 생성하되, 각 후보 엣지가 만드는 e-cycle set들의 block error probability 기여도를 계산해 기여도가 최소인 엣지를 선택하는 greedy 구성.
7. e-cycle set 열거는 variable node에서 트리를 전개해 최단·최소 EMD 경로(minimal path)만 추적하는 방식으로 근사(식 9-10).
8. 부수 트릭: 동일 기여도의 후보가 여럿이면 degree가 최소인 check node를 무작위 선택해 right-regular(균등 check degree) 구조를 유지; lifted graph(QC-LDPC, lifting factor Z)로 확장 가능(상세는 별도 논문).
9. 결과: length-512 앙상블에서 EMPEG가 PEG/ACE-conditioned 대비 error floor 개선; lifting factor Z=24 (N=576,M=288) 조건에서 IEEE 802.16e 표준 코드 대비 5e8 블록 시뮬에서 오류 4건만 검출, AWGN에서 BLER 10^-7 기준 0.5dB coding gain 개선.
10. 한계: 주 최적화 목표가 BEC 채널이며 AWGN은 부가 검증(1개 데이터 포인트)만 제공; HW 구현·복잡도 분석 없음, 순수 부호구성 이론+시뮬레이션.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EMPEG 부호 구성(H-matrix 설계) | `ecc_top.cpp` `Load_PCM()` | 새로운 QC-LDPC H-matrix 설계 기법이라 기존 고정 PCM 교체·재검증 필요, 이식 난이도 높음 |
| lifted graph(QC-LDPC) 확장 | `ecc_data.h` `PCM_b`(base+shift) | Prime ECC도 QC-LDPC 구조라 개념적으로는 유사하나 구체 구성 알고리즘은 별도 논문 참조 필요 |
| stopping-set/error-floor 분석 | 대응 없음 | Prime ECC는 error-floor 원인 분석용 모듈이 없어 부호설계 단계 참고자료로만 활용 가능 |

적용 가치: `중간` — QC-LDPC 부호설계로 error floor를 직접 낮추는 기법이라 개념적 이식 가치는 있으나, 채널모델(BEC)과 검증 채널(AWGN)이 NAND의 read-channel과 다르고 H-matrix 전면 재설계가 필요해 난이도가 높음.
