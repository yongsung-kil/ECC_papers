### B. 알고리즘 요약

1. 채널 모델: impulse noise 채널을 BEC(erasure 확률 `ε`)와 BSC(cross-over 확률 `α`)의 직렬 결합으로 모델링.
2. 기존 방식(예: Ardakani et al.)은 soft-decision sum-product로 성능은 좋으나 복잡도가 높고 유한 길이에서 error floor 발생.
3. 본 논문은 hard-decision 기반으로 복잡도를 낮추면서 impulse noise에 대응하는 디코더를 제안.
4. 수신값을 매핑: 0→1, 1→-1, erasure→0으로 3-state 표현 후 Gallager B를 변형.
5. Check node 갱신: `m(e) = Π m(e')` (BSC의 XOR 대신 곱 연산으로 erasure 상태 반영).
6. Variable node 갱신: `m(e) = sgn(r(v) + Σ m(e'))`, sgn 함수로 다수결 대신 부호 판정 수행.
7. 부수 트릭: erasure를 3번째 상태(0)로 명시적 인코딩하여 sum-product의 3-state 양자화로 취급.
8. 학습/최적화 절차 없음 (구성법 아님, 기존 랜덤/QC 코드 그대로 사용).
9. 결과: MacKay 코드(길이 96~1008, rate 1/2), Tanner(155,64), QC(900,450) 대상 시뮬레이션. RS(255,223/191) 대비 낮은 BSC error probability에서는 열세이나 복잡도는 크게 낮음(추정 24k gates vs RS 115.5k~161.6k gates).
10. 한계: HW 미설계, 시뮬레이션뿐. 채널 모델이 DSL/전력선 impulse noise 특화로 NAND read channel(AWGN/RBER 근사)과 다름. 부호 자체는 기존 MacKay/Tanner/QC 코드 재사용(신규 부호설계 아님).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-state hard-decision 디코딩(Gallager B 변형) | `decoder.cpp` `LDPC_Decoder()` (HD 계열) | Prime ECC의 hard-1bit(HD) 경로와 개념적으로 유사하나 erasure 3번째 상태는 현 HD 파이프라인에 없음 |
| Check/Variable node 갱신 (sgn 기반) | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 알고리즘 구조는 참조 가능하나 impulse noise 특화 erasure 처리로 직접 대응 어려움 |
| BEC+BSC 채널 모델 | `channel.cpp` | NAND의 AWGN/RBER/fixed-error 모델과 상이, 대응 없음 |

> 적용 가치: 낮음 — NAND read channel과 다른 impulse noise(DSL/전력선) 특화 모델이며, hard-decision 알고리즘 자체도 표준 Gallager B의 소규모 변형 수준으로 Prime ECC가 이미 보유한 min-sum/HD 계열 대비 기여가 제한적.
