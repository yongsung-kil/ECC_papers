### B. 알고리즘 요약

1. 목표 코드: 1020-bit, rate-1/2 regular LDPC (H는 `510×1020`, row weight 6, column weight 3, density≈0.006), fully-parallel 아키텍처 전제.
2. 문제: fully-parallel 디코더(VN 1020개+CN 510개 개별 유닛)는 배선 혼잡(routing congestion)이 칩 면적의 50%를 차지([2] Blanksby 2002 기준)하는 병목.
3. 채널/모델은 명시적 서술 없이 표준 LDPC BP 디코딩(soft decision, 4-bit 등급) 전제.
4. 핵심 기법 1단 - Wiring-efficient 코드 설계: H-matrix의 1을 대각선 부근에 클러스터링해 interconnection height(수평 배선 라인 수)를 최소화. 시작 행은 3개의 1을 좌상단에 배치 후 매 행마다 2-position cyclic shift, 나머지 1은 무작위 순열 벡터로 대각선 좌우에 근접 배치.
5. 결과 코드의 interconnection height는 690 wire-width(변수→체크 400 + 체크→변수 290)로, 기존 [4]의 1208(각 604) 대비 대폭 감소하면서도 [2](Blanksby)와 동등한 BER/BLER 성능 유지(`Eb/N0` 곡선 상 [4]의 0.7dB 손실 없음).
6. 핵심 기법 2단 - One-way broadcasting([1] 기법 채용): inverse check-node 연산을 check-node에서 대응 variable-node로 이동시켜 check-to-variable 배선 수를 추가 HW 비용 없이 40% 절감(check-node당 출력 메시지 1개로 통합). Two-way broadcasting은 추가 메모리/역연산기 필요해 채택하지 않음.
7. 부수 트릭 - Unwrapping(파이프라이닝): 메시지를 다시 VN으로 되돌리는 대신 디코더를 물리적으로 복제(replica)해 메시지를 "앞으로" 전달, `Niterations = R·L` (R=replica 수, L=재순환 횟수), `T=(fclock/L)·n`로 처리량 계산.
8. 별도 학습/최적화 알고리즘 없음(그래프/코드 설계는 결정론적 절차 + 무작위 순열 배치).
9. 결과: one-way broadcasting으로 클럭 100MHz 가정 시 L=10(50 iteration이면 R=5 replica)으로 10Gbps 달성. [2] 대비 면적은 3배 미만 증가로 추정되나 속도는 10배.
10. 한계: 면적/처리량 수치는 실제 합성이 아닌 [2] 수치 기반 이론적 추정치. FPGA/ASIC 실측 없음(순수 시뮬레이션+추정). fully-parallel 구조 자체를 전제해 z=32 등 sub-block 병렬 구조에는 적용 방식이 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 대각선 클러스터링 H-matrix 설계 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base matrix+circulant shift 기반 QC-LDPC로 고정되어 있어, fully-parallel 전제의 배선-최적 랜덤 regular H-matrix 재설계는 이식 난이도 높음 |
| One-way broadcasting (inverse check-node 연산 이동) | `decoder.cpp` `C2V_Cal()` | 이미 min-sum + sign XOR 방식으로 CN 출력을 압축 전달하고 있어 개념적으로 유사, 신규 기여 낮음 |
| Unwrapping(디코더 replica 파이프라이닝) | 디코더 HW 모델 (pipeline 6-stage) | Prime ECC는 이미 6-stage bit-exact pipeline을 갖고 있으나, 반복(iteration) 전체를 replica로 펼치는 방식은 별개 개념(면적 대 처리량 트레이드) |
| Fully-parallel VN/CN 개별 유닛 배선 | 대응 없음 | Prime ECC는 z=32 row 단위 병렬 구조로 fully-parallel(1020+510 유닛) 아키텍처 자체가 대응 없음 |

적용 가치: 낮음 - fully-parallel 아키텍처의 배선 혼잡 완화라는 특정 문제를 다루며, 정정성능 개선도 없고 Prime ECC의 QC-LDPC/z=32 병렬 구조와 설계 전제가 달라 이식 가치가 제한적.
