### B. 알고리즘 요약

1. 시스템은 QC-LDPC 코드, AWGN 채널, BPSK 변조, sum-product 알고리즘(반복 100회) 디코딩을 가정, 시뮬은 mother matrix 16×32(최대 variable-node degree 9), rate 0.5, `L=2^(i+3)` (i=0..6, 즉 L=8~512, 코드길이 32L=256~16384)로 확장.
2. 풀려는 문제: 하나의 QC-LDPC 코드(작은 lifting size `L0`)로부터 girth를 잃지 않으면서 더 큰 길이(`L1=qL0`)의 QC-LDPC 코드를 효율적으로 구성하는 방법이 부족했음 - 기존 Miladinovic-Fossorier 재귀법[10]은 mother matrix 크기가 매 단계 커지는 단점.
3. 핵심 가정/모델: 부호는 exponent matrix `E(H)=(a_ij)` (circulant permutation shift 값)로 표현되고, mother matrix `M(H)`(0/1 패턴)는 lifting 전후 고정.
4. 핵심 기법 1단: **Floor-lifting** - 각 `a_ij`를 `b_ij = q·a_ij + c_i` (`0≤c_i<q`)로 확장하되, 후보 중 girth를 최대화·최소 cycle 개수를 최소화하는 `c_i`를 그리디하게 선택(Step2~5, degree-오름차순 순회).
5. 핵심 식: Theorem 2 - block-cycle exponent chain의 `r_0·Σ(-1)^i a_i ≡ 0 mod L0`와 `r_1·Σ(-1)^i b_i ≡ 0 mod L1`에서 `r_1≥r_0`가 항상 성립함을 증명, 즉 lifting 후 girth `2l·r`은 감소하지 않음(Proposition 1과 결합).
6. 핵심 기법 2단: **Modulo-lifting** - `b_ij`를 `a_ij, a_ij+L0, ..., a_ij+(q-1)L0` 중에서 동일한 그리디 기준으로 선택하는 대안 절차(Step2만 다름, 이하 동일).
7. 재귀 적용으로 `C0→C1→...→Cs` 코드 시퀀스 생성 가능하며, 최종 `E(Hs)`와 각 `L_k`만 저장하면 모든 중간 코드 `H_k`를 `floor(L_k/L_s · E(Hs))` 또는 `E(Hs) mod L_k` 연산으로 즉시 복원 가능(저장 효율).
8. 별도 학습 절차 없음, 그리디 선택(girth 최대화, 최단 cycle 개수 최소화) 기준을 각 lifting 단계에 적용.
9. 결과: (3,6) regular QC-LDPC 예제에서 `L0=7→L1=21→L2=525`로 lifting해 girth 12 달성(이론상 이 mother matrix의 최대 girth); 시뮬 결과 랜덤(PEG) LDPC 대비 FER 성능 저하가 거의 없고 floor/modulo lifting 간 성능 차이도 거의 없음.
10. 한계: 순수 이론/시뮬 검증(FER 곡선)뿐이며 HW 설계·게이트 구현 없음; lifting은 동일 mother matrix를 유지한 채 배수 확장만 가능해 임의 rate/length 설계 자유도는 제한적.

| Girth 보존 exponent 선택(floor/modulo-lifting) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 다루는 요소로 특정 QC-LDPC 고정인 Prime ECC에 새 lifting size 코드를 파생하려는 경우에만 유효 |
| Sum-product 디코더(비교군) | `decoder.cpp` `LDPC_Decoder()` | 본 논문은 sum-product로 시뮬만 하며 min-sum/HW 특화 요소 없음 - 대응 없음 수준 |

> 적용 가치: 중간 — Prime ECC가 QC-LDPC(base matrix+circulant shift) 구조이므로 부호 확장/파생 시 girth 보존 lifting 기법은 참고 가치가 있으나, H-matrix 자체를 바꾸는 작업이라 이식 난이도(§4 "부호설계" 행)가 높고 재검증 부담이 큼.
