### B. 알고리즘 요약

1. 시스템: 길이 2048bit, rate 0.5인 (3,6)-regular architecture-aware(AA-)LDPC 코드를 16×32 block-row/column(64×64 순열행렬 서브블록) 구조로 설계, BPSK+AWGN 채널.
2. 풀려는 문제: 표준 parallel LDPC 디코더는 인터커넥트 복잡도가 블록길이에 선형 증가해 긴 코드에서 라우팅 네트워크가 성능/면적 병목이 됨(참조 [12]는 전체 면적의 50%가 네트워크).
3. 핵심 가정/모델: AA-LDPC의 parity-check matrix를 16개 constituent code(블록행)의 연쇄로 보고, 이를 serially concatenated turbo code처럼 처리.
4. 핵심 기법: TDMP(turbo-decoding message-passing) 알고리즘 — 16 sub-iteration마다 하나의 constituent code를 SISO(BCJR 기반) 디코더로 갱신, extrinsic `μ`/posterior `λ` 메시지를 dual extrinsic 원칙으로 관리.
5. 핵심 식: max-quartet 함수 `f(a,b,c,d)=max(a,b,c,d)+correction` 근사로 두 개 logsum 연산 차를 근사, Gallager 원식 대비 lookup-table 불필요·양자화 잡음에 강건.
6. 확장: 코드rate는 block row puncturing으로 8/16~14/16까지 1/16 단위 튜닝 가능(하드웨어 재사용, rate 9/16에서 성능손실 약 2 orders of magnitude BER@2.6dB).
7. 부수 트릭: 4-bit 2's complement soft message, 6-stage programmable dual bi-directional shuffle-exchange 네트워크(면적 sqrt(N) 스케일), 128개 4-bit 메시지 병렬 처리(64 QUAD-SISO 모듈).
8. 학습/최적화 절차 없음(고정 설계, DE/GA 미사용); 대신 Matlab/Simulink 고수준 모델로 양자화 분석 및 면적-BER 트레이드오프 튜닝.
9. 결과: 0.18um 6금속 CMOS, 14.3mm², 787mW@125MHz/1.8V, 640Mb/s@10 iteration(최대 1.6Gb/s@4 iteration, 2.6dB), TPMP 대비 50% 적은 iteration으로 수렴, 메모리 51680bit(비교 대상 대비 75% 절감).
10. 한계: (3,6)-regular rate-0.5 코드 기준 설계로 NAND ECC의 고rate(~0.9+) 요구와 다르며, wireless/자기기록/광통신 타깃; 부호구조(AA-LDPC 순열행렬)가 Prime ECC의 QC-LDPC circulant shift와 유사하나 완전 동일하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP 알고리즘(SISO 기반 turbo-style message-passing) | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 스케줄링 대안으로 참고 가능하나 min-sum과 알고리즘 계열이 달라(BCJR/SISO) 직접 이식은 어려움 |
| max-quartet Q-function 근사(lookup-table-free) | `decoder.cpp` `CNU_Update_New_Mag()` | CN 연산 근사 기법으로 min-sum 대체 후보지만 성능개선 폭(0.05dB) 제한적 |
| 프로그래머블 셔플-익스체인지 라우팅 네트워크 | 디코더 HW 모델(GT.cpp `Make_GT_HW()`와 유사 영역) | AA-LDPC 순열구조 기반 네트워크로, Prime ECC의 QC circulant shift 라우팅과 컨셉 유사하나 재설계 필요 |
| Dual extrinsic 메시지 관리(μ/λ 분리 저장) | `decoder.cpp` `C2V_Cal()` | 메시지 저장 최적화 아이디어로 부분 참고 가능 |
| Base matrix + permutation 프로그래머블 구조 | `ecc_top.cpp` `Load_PCM()` | 코드 재구성 유연성 참고용, 부호설계 자체는 이식 난이도 높음 |

> 적용 가치: 중간 — TDMP/turbo-style 스케줄링과 lookup-table-free 근사, 네트워크 라우팅 면적효율 아이디어는 참고가치가 있으나, 부호rate·타깃 도메인이 NAND와 달라 직접 이식보다는 설계 참고 수준.
