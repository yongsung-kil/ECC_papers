> 총평: horizontal scheduling + 개선된 forward-backward 방식으로 min-sum CN 처리를 가속화하는 스케줄링/구현 기법으로, HW ASIC이 아닌 DSP 소프트웨어 대상이지만 스케줄링 아이디어 자체는 디코더 이터레이션 루프에 참고 가능.

### B. 알고리즘 요약

1. 시스템: (504,252) regular(CN weight 6), (1908,1696) regular(CN weight 36), (1057,813) irregular(CN weight 17/18) LDPC 코드, AWGN+BPSK 채널, min-sum 알고리즘 기반.
2. 문제: 전통적 flooding schedule은 병렬 처리 전제라 순차 처리하는 DSP에서 비효율적이며, cycle이 있는 실제 코드에서 최적 성능을 못 냄.
3. 핵심 가정: horizontal(row-serial) scheduling을 채택 — CN을 하나씩 순차 처리하며 직전까지 갱신된 최신 메시지를 즉시 사용해 수렴 가속.
4. 핵심 기법 1단: horizontal scheduling min-sum — CN 갱신 시 `Lr'_mn`(새 메시지)과 `Lr_mn`(이전 메시지) 차이만큼 사후 LLR `LQ_n`을 즉시 갱신(식 14), BN마다 `LQ_n` 하나만 메모리에 유지.
5. 핵심 식: `Lq_nm = LQ_n - Lr_mn`(식 8)로 BN→CN 메시지를 별도 저장 없이 `LQ_n`과 `Lr_mn`만으로 재구성 — 메모리 사용량과 주소접근 부담 절감.
6. 핵심 기법 2단: forward-backward CN 업데이트를 3단계(forward/backward/updating)에서 2단계로 병합 — backward 진행 방향을 반전시켜 backward와 updating을 동시에 계산(식 19~21), 처리시간 절반 절감.
7. 부수 트릭: 32bit 버전은 `(Lr_mn, LQ_n index)` 쌍을 연속 메모리에 패킹해 순차 접근, 16bit 버전은 SIMD로 두 쌍을 동시 처리(패킹 순서 재배치).
8. 학습/최적화 절차: 없음(알고리즘/구현 최적화, 학습 기반 아님).
9. 결과: 32bit 대비 16bit(SIMD) 버전에서 (1057,813)/(1908,1696) 코드 기준 iteration당 CPU cycle 약 1.3~1.4배 감소, forward/backward/updating 단계 모두 6~9 CPU cycle 수준으로 최소화.
10. 한계: ASIC/FPGA가 아닌 범용 고정소수점 DSP(TMS320C6416) 소프트웨어 구현 결과이며, BER/error-floor 등 정정성능 데이터 없이 CPU cycle(처리속도) 수치만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Horizontal scheduling (row-serial, 즉시갱신) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update(even/odd 교대) 스케줄과 유사 계열의 스케줄링 아이디어이나 완전병렬 HW 구조와는 결이 다름 |
| CN min-sum 연산 (`min1/min2`, sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum과 동일 계열 연산, 신규 기여 아님 |
| Forward-backward 2단계 병합 (CN 내부 최소값 재계산 최적화) | `decoder.cpp` `C2V_Cal()` | CN 내부 min1/min2 계산 최적화 아이디어로 참고 가능하나, DSP 순차처리 전용 기법이라 완전병렬 HW pipeline에는 이식 난이도 중 |
| SIMD 16/32bit 벡터화 (DSP 명령어 최적화) | 대응 없음 | Prime ECC는 RTL bit-exact HW 시뮬레이터로 DSP 명령어 벡터화 개념 자체가 대응 없음 |

> 적용 가치: 중간 — horizontal scheduling과 CN 내부 forward-backward 병합 아이디어는 이터레이션 가속 관점에서 참고할 만하나, 대상이 범용 DSP 소프트웨어 구현이라 Prime ECC의 완전병렬 HW pipeline(z=32, HCU, GT)에 직접 이식하기보다는 개념적 참고 수준.
