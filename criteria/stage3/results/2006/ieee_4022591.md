> 총평: zigzag/dual-diagonal 부호 계열에서 rate-compatible puncturing 위치를 이론적으로 최적화하는 구성법 논문으로, HW 구현이나 디코더 개선은 다루지 않으며 Prime ECC의 이미 고정된 puncturing 구조(punct_col_num=2)와는 목적이 달라 이식 가치가 제한적.

### B. 알고리즘 요약

1. Zigzag parity 구조를 갖는 fast-encodable LDPC 코드(Z-LDPC, dual-diagonal 유사구조)에서 parity bit puncturing으로 rate 1/3~2/3의 rate-compatible 코드를 구성.
2. 문제: 유한길이 코드에서 puncturing 위치가 성능(waterfall)에 큰 영향을 주는데, 기존 [11]의 k-SR(reliability recovery) 기준은 최적 패턴이 유일하지 않아 추가 기준 필요.
3. 핵심 가정: 동일 punctured parity node에 연결된 두 check node는 병합 가능(Theorem 1), 이를 이용해 punctured parity node를 제거한 effective Tanner graph(eTanner)를 구성.
4. 핵심 기법: eTanner graph의 check node degree 분포가 punctured code 성능을 결정하며, `Criterion 1`: puncturing 후 check node degree 종류를 최소화(가능하면 연속된 정수 2종)해야 함.
5. `φ(k)=dc(k)+dc(k+1)`(degree metric)을 최소화하는 parity bit를 순차적으로 puncture하는 탐욕 탐색 알고리즘(Step 0~7) 제안, 동률 시 2차 metric `φ2(k)`로 재선택.
6. Theorem 4: 모든 punctured parity node의 reliability를 복구하는 최소 iteration 수는 `⌈m/nup⌉/2`이며, 제안 알고리즘이 이를 달성함을 증명(dual-diagonal 구조에서 unpunctured node를 규칙적으로 배치).
7. 부수 기법: check node 배치(Criterion 2)도 함께 최적화 - degree ρ, ρ+1을 최대한 균등 분산 배치.
8. 최적화 절차: 탐욕적 search algorithm(Step 0~7)으로 목표 rate별 puncturing set Γi를 순차 계산, mother code degree distribution은 기존 EXIT/DE 최적화(참고문헌 [4]) 결과 사용.
9. 결과: rate 1/3 mother code(length 1056, info 1056)를 rate 0.4/0.5로 puncturing 시 제안 패턴이 threshold(σ*) 1.060/0.933으로 기존 최적 패턴(1.042/0.909) 대비 개선, 3GPP rate-compatible Turbo code 대비 FER 우수(rate 1/3,1/2,2/3에서).
10. 한계: HW 구현/합성 없음, dual-diagonal(zigzag) 구조 특화라 일반 QC-LDPC 구성에는 이론(check node merging)만 재사용 가능, decoder 알고리즘 자체는 표준 BP 사용(개선 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Rate-compatible puncturing 위치 최적화 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 `punct_col_num=2` 고정 puncturing 사용 중이나 이 논문은 다중 rate 지원용 가변 puncturing 설계로 목적 상이 |
| eTanner graph / check node merging 이론 | 대응 없음 | 순수 이론적 그래프 분석, 코드 모듈 대응 없음 |
| Dual-diagonal zigzag parity 구조 | `encoder.cpp` `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코딩 사용하나 이 논문은 puncturing 설계가 핵심이라 인코더 로직 자체는 무관 |
| Degree distribution 최적화(mother code) | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 단계 참고 가능하나 이식 난이도 높음(특정 QC-LDPC 고정) |

적용 가치: **낮음** — 순수 부호설계(puncturing 패턴) 이론 논문으로 HW/디코더 구현이 전혀 없고, Prime ECC는 이미 고정 rate·고정 puncturing 구조를 사용하므로 rate-compatible 가변 puncturing 설계 자체의 이식 필요성이 낮음. 다만 H-matrix 재설계 시 check node degree 집중화 이론은 참고 가능.
