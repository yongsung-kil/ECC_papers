### B. 알고리즘 요약

1. 시스템: 1024-bit, rate-1/2 regular LDPC 코드(1024 VN, 512 CN), belief-propagation(message-passing) 기반 완전병렬 아키텍처.
2. 풀려는 문제: 완전병렬 LDPC 디코더는 배선(interconnect)이 로직보다 면적/전력을 더 많이 소비하고, 라우팅 혼잡으로 로직 영역 활용률이 50%에 불과한 문제.
3. 핵심 가정/모델: 배선 문제를 알고리즘이 아닌 물리적 3차원 집적(3D IC)으로 해결한다는 접근.
4. 핵심 기법: MIT Lincoln Lab의 3-tier wafer bonding FDSOI 0.18um 공정으로 VN/CN 컴퓨팅 노드를 3개 층(tier)에 분산 배치, 5um 미세 3D via로 층간 연결.
5. 데이터패스는 16-way 병렬 3단 파이프라인으로 구성, 클록 128MHz × 16 = 2Gbps throughput 달성(`128MHz*16=2Gb/s`).
6. 확장: 3D 배치 도구(in-house)로 면적/라우팅밀도/총배선길이/3D-via 수를 최소화하는 목표로 CNU/VNU를 3개 tier에 배치.
7. 부수 트릭: 3D 설계 흐름은 상용 2D 설계흐름(합성→P&R→DRC/LVS)에 3D 특화 도구(placement, routing, buffer insertion, CVS 체크)를 결합해 자동화.
8. 학습/최적화 절차 없음(물리설계 흐름 논문).
9. 결과: 2D 대비 총배선길이 182.42m→67.42m, 면적 290.35mm²→119.56mm², 전력 750mW→430mW, power-delay-area product 10배 이상(2.5×31.75×13.125) 개선. BER-SNR 시뮬로 10^-5까지 확인(그래프만, 본문 수치 없음).
10. 한계: 알고리즘/부호설계 개선 없이 순수 3D 공정 물리설계 논문이며, 특수 3D FDSOI 웨이퍼본딩 공정 전제라 일반 2D ASIC(Prime ECC 타깃)에는 직접 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 완전병렬 belief-propagation 디코더 아키텍처 | [decoder.cpp](decoder.cpp) `LDPC_Decoder()` | Prime ECC는 z=32 부분병렬+HCU 구조로 완전병렬이 아니며 이미 min-sum 채택, 알고리즘 대응 없음 |
| 3D IC 웨이퍼본딩 공정, 3D via 배치/라우팅 도구 | 대응 없음 | Prime ECC는 2D ASIC/RTL 시뮬레이터로 3D 공정 대상이 아님 |
| 16-way 파이프라인 데이터패스 | 디코더 HW 모델 (pipeline 6-stage) | 병렬 파이프라인 개념만 유사, 구체적 구조는 대응 없음 |

> 적용 가치: 낮음 — 3D IC 공정이라는 물리설계 차원의 개선으로, Prime ECC가 다루는 2D ASIC 알고리즘/아키텍처 레이어와는 근본적으로 다른 층위의 기여.
