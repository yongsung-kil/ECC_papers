### B. 알고리즘 요약

1. Rate-1/2, block length n=1008 irregular LDPC 코드를 PEG(Progressive Edge-Growth) 알고리즘으로 구성, 최대 symbol-node degree 15.
2. 기존 PEG는 girth 최대화만 추구하나, 실제로는 girth histogram(특히 degree-2 symbol node 관련)이 성능에 더 중요함이 문제의식.
3. Density evolution으로 최적화된 irregular 코드는 degree-2 symbol node 비중이 크며(예제에서 47.7%), 이들이 짧은 사이클에 관여할 때 EMD(extrinsic message degree)가 낮아 채널 잡음에 취약하다는 것이 핵심 가정.
4. 핵심 기법: edge (ci, sj) 배치 시, 새로 형성되는 최단 사이클에 관여하는 최저차수 symbol node의 최대 다중도 `nj,i`와 전체 다중도 `Nj,i`를 최소화하는 후보 check node ci를 선택.
5. `min_ci nj,i` (식1), `min_ci Nj,i` (식2) — 다중도를 최소화함으로써 최저차수 노드가 짧은 사이클에 반복적으로 얽히는 것을 방지, 이는 trapping set 크기 증가로 이어져 error floor 개선.
6. Spreading subgraph를 depth별로 한정 확장(중복 vertex/edge 배제)하는 방식으로 수정하여, 최저차수 노드의 다중도를 재귀적으로 효율 계산.
7. 첫 edge `Es0j` 선택에 따라 결과가 달라지므로 Ntry개의 후보 Es0j를 시도하여 최소 nj,i, Nj,i를 주는 것을 채택.
8. 별도 학습/최적화 절차 없음(기존 Richardson-Shokrollahi-Urbanke 최적 degree distribution을 그대로 사용).
9. 결과: degree-2 노드 평균 local girth 6.02→7.18로 개선, 최소 trapping set 크기 증가(예: PEG의 (9,1)→제안법의 (13,1)), error floor 영역에서 성능 향상, water-fall 영역은 소폭 개선.
10. 한계: 소규모 block length(1008) 단일 예제만 검증, HW/복잡도/throughput 분석 전혀 없음, BP 디코더 기준(min-sum 아님)이며 NAND 채널 가정 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG 기반 H-matrix/Tanner graph 구성 | `ecc_top.cpp` `Load_PCM()` | 부호설계 자체이나 Prime ECC는 QC-LDPC 고정 구조라 PEG류 irregular 랜덤 구성법은 직접 대응 어려움 |
| girth/trapping set 기반 error-floor 대책 | 대응 없음 | Prime ECC는 min-sum 디코더·HCU 등 HW 최적화 위주이며 부호 자체의 girth 재설계는 범위 밖 |
| BP(sum-product) 디코딩 | 대응 없음 | Prime ECC는 min-sum 기반, 논문은 tanh 기반 BP 사용 |

적용 가치: **낮음** — 순수 랜덤 irregular LDPC 구성법(PEG) 논문으로 QC-LDPC 구조가 고정된 Prime ECC 3.1에는 이식 난이도가 매우 높고(§4 표에서 "부호설계=높음"), HW/디코더 개선 요소가 없음.
