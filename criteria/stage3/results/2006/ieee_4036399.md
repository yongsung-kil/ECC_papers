### B. 알고리즘 요약

1. LDPC 반복 복호는 메시지-패싱 알고리즘(sum-product, min-sum 등)으로 수행되며, 노드마다 다른 알고리즘을 쓸 수 있는 "irregular decoder" 개념을 다룸.
2. 기존 연구는 code 설계 시 모든 노드가 같은 알고리즘을 쓴다고 가정했는데, 본 논문은 알고리즘 자체를 노드별로 다르게 하는 자유도를 복잡도 최적화에 활용하고자 함.
3. Check-side와 variable-side에 각각 M, N개의 서로 다른 update rule을 배정하는 확장된 degree distribution `Pij`, `Λij`를 정의.
4. 여러 알파벳(메시지 타입)이 섞여도 추적 가능하도록 universal check-to/from-variable alphabet을 도입해 density evolution을 2개의 pdf 추적으로 단순화.
5. 1-D EXIT chart 표현 `f(p,P,Λ) = Σ Λij·fij(p,P)`으로 code-decoder 쌍의 수렴 거동을 선형 결합으로 표현, Λ를 선형계획법(LP)으로 최적화 가능.
6. HW 복잡도 최적화(식 10-11)와 전체(iteration 포함) 복잡도 최적화(식 12, 필요 iteration 수 `L_p0^pt` 근사식 18) 두 가지 목적함수를 정의.
7. 부수 트릭: Λ 변화량을 제한하는 각도 제약(cosθ≈1)을 추가해 LP를 반복 적용하며 fij를 갱신하는 iterative LP 절차 사용.
8. 최적화 절차: check-side 분포 P는 탐색(search-based, steepest descent), variable-side 분포 Λ는 LP로 구함.
9. 결과: rate-1/2 BIPLGC 예제에서 σ=0.86 조건에서 irregular decoder가 규칙적 min-sum/sum-product보다 낮은 복잡도 달성; 채널 추정 오차 예제에서 irregular decoder(P61=0.8, P62=0.2)가 10^-6 오차율을 87 iteration만에 달성하며 규칙적 sum-product/min-sum이 모두 실패하는 조건에서도 강건함을 보임.
10. 한계: 완전 이론/시뮬레이션 결과뿐, HW 구현 없음. Fully-parallel decoder를 가정한 복잡도 모델이며 serial decoder에는 부적합하다고 저자가 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| irregular decoder(노드별 다른 알고리즘) | 대응 없음 | Prime ECC는 단일 min-sum 알고리즘으로 고정된 CNU/VNU 구조라 노드별 이종 알고리즘 스케줄링 개념이 없음 |
| min-sum/sum-product 근사 관계 | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC가 이미 min-sum 근사를 사용 중이므로 이론적 배경만 겹침 |
| irregular code degree distribution 최적화 | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 관점에서 참고 가능하나 코드 자체는 고정 구조라 재설계 부담 큼 |

적용 가치: `낮음` — 노드별 이종 알고리즘 혼용이라는 핵심 아이디어가 Prime ECC의 균일 min-sum HW 파이프라인과 구조적으로 맞지 않고, 검증도 이론 수준에 그침.
