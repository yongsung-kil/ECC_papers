### B. 알고리즘 요약

1. QC-LDPC, H=[HS|HP] 구조, circulant permutation sub-matrix `I(C_j,l)` 기반, 예시로 rate 1/2, length 2048, q=128.
2. 기존 triangular형/dual-diagonal형 parity 구조는 error floor가 각각 10^-6, 10^-7 수준으로 높음 — 이를 더 낮추면서 낮은 인코딩 복잡도도 유지하려는 문제.
3. AWGN 채널, BPSK 변조 가정, iterative belief-propagation(최대 200 iteration) 디코딩.
4. HP의 우하단에 4개 sub-matrix(`I`, `I(q-1)`, `I`, `I`)로 구성된 dual-diagonal 블록을 배치해 전체 H를 triangular+dual-diagonal 형태로 구성.
5. 짧은 사이클(2i-cycle) 회피 조건 `sum(C_jk,lk - C_jk+1,lk) = 0 mod q`을 만족하도록 shifting coefficient 랜덤 선택.
6. degree 2 이하 컬럼은 HS에 배치 금지, 최소거리 확대를 위해 HP를 삼각+dual-diagonal 형태로 구성.
7. shifting coefficient는 저차수(degree 2,3) 컬럼부터 순차적으로 짧은/중간 길이 사이클을 피하도록 선택 후 고차수 컬럼으로 확장.
8. degree distribution(`λ2=0.375, λ3=0.5, λ8=0.125`) 및 shifting coefficient는 시뮬레이션 기반 수작업 최적화(자동 최적화 알고리즘 명시 없음).
9. 결과: 제안 코드 error floor 10^-9 이하(triangular 10^-6, dual-diagonal 10^-7 대비 개선), 최소거리 상한 22(triangular 13, dual-diagonal 14 대비 향상), 인코더 912 XOR gate로 100MHz에서 12.8Gbps.
10. 한계: 디코더는 시뮬레이션만(HW 미설계), 최소거리는 상한값(upper-bound)일 뿐 정확한 최소거리 아님, 특정 예제(q=128, rate 1/2)에서만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| triangular+dual-diagonal H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` + H-matrix | Prime ECC도 dual-diagonal(shuffled dual-diag) 인코딩 구조를 이미 보유, 특정 QC-LDPC 고정이라 재구성 시 재검증 부담 큼 |
| 선형 인코딩 (부호 PCM 생성) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 인코딩 구조가 이미 유사하게 구현되어 있어 개념적 참고 가치 |
| 짧은 사이클 회피 shifting coefficient 설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 단계 참고, 코드 자체 재설계 필요 |
| error-floor 개선(디코더 알고리즘) | 대응 없음 | 디코더 알고리즘 변경 없음(표준 BP), 순수 부호구조 개선 |

> 적용 가치: 중간 — error-floor 대폭 개선과 선형 인코딩 복잡도 유지라는 매력적 특성이 있으나, Prime ECC는 이미 특정 QC-LDPC H-matrix로 고정되어 있어 부호구조 교체는 이식 난이도가 높음(§4 기준 "높음"에 해당).
