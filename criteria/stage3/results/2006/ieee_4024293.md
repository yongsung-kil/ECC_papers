### B. 알고리즘 요약

1. 시스템: N→∞ 코드워드 비트가 K개의 독립 서브채널(파라미터 `p_j`, 비중 `γ_j`)로 나뉘어 전송되는 non-uniform 채널(OFDM 주파수 bin, MIMO 채널 쌍, 네트워크 다중 경로 등).
2. 문제: 기존 LDPC 설계는 전체 코드워드가 동일 채널 파라미터를 갖는다고 가정하나, 서브채널마다 품질이 다르면 이 가정이 깨져 성능 저하.
3. 핵심 가정: 채널 파라미터는 설계 시점에 완전히 알려져 있고(고정 전송 플랫폼), 디코더는 one-dimensional 분석(단일 파라미터로 message error rate 추적) 가능.
4. 핵심 기법: variable node degree distribution을 서브채널별로 분해한 `Λ_i^(j)`(degree i, 서브채널 j에 배정된 edge 비율)를 설계변수로 정의, `pout = Σ_ij Λ_i^(j) p_out,i^(j)(pin)` 관계식 사용.
5. 핵심 식 의미: rate `R=1-Σρ_i/i / Σλ_j/j`가 설계변수의 선형함수이므로, ρ 고정 시 rate 최대화는 `Σ Λ_i^(j)/i` 최대화라는 LP로 환원됨(수렴보장 제약도 선형).
6. 확장: `p*_0`(초기 메시지 오류율)가 `Λ_i^(j)`에 종속되는 문제를 반복적 LP(round마다 p*_0 갱신)로 해결.
7. 부수 트릭: multi-dimensional 디코딩 알고리즘(one-dimensional 근사 불가 시)에는 `Λ`와 `Λ*`(이전 라운드) 벡터 간 각도 제약(`⟨Λ,Λ*⟩≥‖Λ‖‖Λ*‖(1-ε)`)을 추가해 LP 근사 유지.
8. 최적화 절차: density evolution을 이용해 `p_out,i^(j)(pin)`을 계산하고 LP solver로 `Λ_i^(j)` 도출, 검증 후 필요시 재반복(예제에서 1~3라운드).
9. 결과: 2-BEC 예제(p1=0.2,p2=0.4)에서 rate 0.6902(채널 capacity 0.7의 98.6%) 달성; interleaved 채널 설계 대비 동일 max-degree에서 더 높은 rate 확보 (예: 0.6902 vs 0.6844).
10. 한계: 이론/시뮬레이션(density evolution) 단계로 HW 미설계, N→∞ 점근적 가정, 단일 균일채널(NAND read channel)에는 적용되지 않는 문제 설정.
