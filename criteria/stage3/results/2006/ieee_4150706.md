### B. 알고리즘 요약

1. AWGN 채널, BPSK 변조를 가정하고 variable node와 check node를 모두 일반 서브코드(Hamming/SPC/repetition 등)로 대체한 "doubly generalized LDPC(DGLDPC)" 코드 클래스를 제안(rate 예제: `7/15`, `1/2`, length: `7650`, `1536`).
2. 기존 GLDPC는 check node만 일반화(SPC→Hamming/BCH/RS)했는데, variable node까지 다양화하지 않아 설계 유연성이 제한된다는 문제를 지적.
3. Tanner graph를 super variable node/super check node로 표현, base matrix `H`를 row expansion(체크 서브코드 PCM 치환) + column expansion(변수 서브코드 생성행렬 치환) 2단계로 확장해 실제 parity-check matrix 생성.
4. 핵심 기법 1단: super variable/check node 단위 BP(sum-product) 반복복호. LLR `U`(check→variable), `V`(variable→check) 메시지, 고차수 서브코드는 MAP(trellis 기반) 복호로 extrinsic LLR 계산(식 3~7).
5. 핵심 기법 2단(EXIT chart 최적화): information combining 개념을 이용해 SPC/repetition/특정 서브코드(`(L,2)` 코드 등)에 대한 EXIT 함수 `Iext`의 폐형식(closed-form) 유도(식 11~17) → variable node degree profile `λi`를 curve matching으로 최적화해 임계점(threshold) 개선.
6. `(6,5)` SPC + `(15,11)` Hamming 조합 코드(C1)는 rate-7/15 length-7650에서 임계값 0.3dB로, 동일 rate `(2,15)` GLDPC 대비 0.58dB 개선.
7. 부수 기법: 고차수 서브코드는 연산량 절감을 위해 trellis 기반 MAP(또는 Max-Log-MAP) 복호를 채택.
8. 최적화 절차: EXIT curve matching으로 각 서브코드 혼합 비율 `λ1..λ4` 결정(예: `0.425/0.075/0.075/0.425`).
9. 결과: C1은 동일 rate GLDPC 대비 BER 약 0.35dB 개선(Fig.9), C2(rate-1/2, length-1536)는 GF(16) `(2,4)`-LDPC 대비 WER 10^-6에서 약 1dB 개선(주로 error-floor 저감), 두 코드의 check node 연산 복잡도는 유사(`O(15×16)` vs `O(16×16)`).
10. 한계: 이론/시뮬레이션 수준(HW 미설계), variable node에도 비-repetition 서브코드를 쓰는 구조라 표준 binary LDPC와 인코딩/디코딩 구조가 크게 다르며 복잡도·throughput 실측 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Super check node를 Hamming/BCH 서브코드로 대체(체크 일반화) | 대응 없음 | Prime ECC는 표준 SPC 기반 QC-LDPC parity 구조만 지원, 서브코드 치환 개념 없음 |
| Super variable node를 SPC/repetition 서브코드로 대체(변수 일반화) | 대응 없음 | binary QC-LDPC의 단순 variable node(단일 비트) 구조와 근본적으로 다름 |
| BP 기반 반복복호(super node message passing) | `decoder.cpp` `LDPC_Decoder()` | 반복 메시지 패싱이라는 큰 틀은 유사하나 서브코드 MAP 복호는 Prime ECC의 min-sum CN/VN 연산과 대응 없음 |
| EXIT chart 기반 degree profile 최적화 | 대응 없음 | Prime ECC는 고정 QC-LDPC H-matrix(`Load_PCM()`)를 사용, EXIT 기반 재설계 파이프라인 없음 |

적용 가치: `낮음` — Variable/check 노드 모두를 일반 서브코드로 대체하는 DGLDPC 구조는 표준 binary QC-LDPC와 근본적으로 다른 부호 클래스이며, Prime ECC 3.1의 고정 QC-LDPC 아키텍처(단순 SPC 기반 parity, repetition variable)와 이식 접점이 거의 없음.
