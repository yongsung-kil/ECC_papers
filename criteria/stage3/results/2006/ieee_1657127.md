### B. 알고리즘 요약

1. 대상은 DVB-S2 표준의 QC-LDPC 코드(360×360 블록 구조, `N=64800`, rate 1/4~9/10, `Km`은 4~13 범위).
2. 기존 fully-parallel(360개 유닛, 800Mbps+)은 90Mbps 요구사양 대비 과설계이고, block-level parallel([5],[8])은 360개 데이터패스가 고정되어 스케일 다운이 비효율적인 문제를 다룸.
3. UMP-BP(min-sum 계열) 알고리즘 사용, check-node 메시지 `Amn`은 `sign XOR`과 `running min/one-but-minimum/index` 압축 벡터로 저장(`2*5+log2(Km)+Km` 비트, 기존 `(1+5)*Km` 대비 약 2배 압축).
4. Gauss-Seidel(=layered/shuffled) 반복: symbol-node 값 `An`을 즉시 갱신해 메모리 절반 절감 + 반복횟수 약 2배 감소(Yeo et al. 기법 채용, 단 BER 저하를 막기 위해 식(5)의 두번째 항은 생략하지 않음).
5. 핵심 스케일링 트릭: 코드 고유의 360 블록 병렬성 중 임의의 divider `D`(예: D=45)만큼만 데이터패스를 인스턴스화 -> 면적은 처리량에 선형 비례, barrel shifter 크기(360→D)는 제곱으로 감소.
6. 메모리 파티셔닝: 360개 symbol-node를 D개 word로 그룹화(간격 `360/D`), barrel shifter로 회전시켜 항상 동일한 D개 병렬 check-node 계산이 가능하도록 정렬.
7. 부수 기법: 5-bit magnitude(6-bit soft 포함), fudge-factor(정규화 상수)로 min-sum 성능을 BP 수준까지 개선 가능함을 언급(직접 적용은 안 함).
8. 별도 학습 절차 없음. 스케줄은 정적(static)으로, 연속된 두 parity 방정식이 symbol-node를 공유하지 않도록 사전 배치.
9. 결과: D=45 데이터패스로 90Mbps 처리량 달성, 90nm CMOS 합성 시 총 4.1mm2(기존 [5],[8] 대비 면적 2.7배/최대 3배 절감), 메모리는 압축 덕에 약 절반.
10. 한계: BER 시뮬레이션은 수개월 소요되어 <10^-9 레벨 검증은 FPGA 매핑으로 별도 진행 중(논문 시점 미완료). 정정성능 개선 자체를 목표로 한 논문이 아니며 순수 면적/구현 효율화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 압축 메시지 벡터(running min/one-but-min/index/sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Min-Sum의 min1/min2 계산과 사실상 동일한 아이디어로 이미 보유한 기법과 중복 |
| Gauss-Seidel(layered) 스케줄링 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 옵션) | Dual-Update의 parity even/odd 교대 스케줄과 유사한 계열의 조기 갱신 기법, 참고 가치 있으나 신규성 낮음 |
| D-개 데이터패스 스케일러블 sub-block 병렬화 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z=32 고정 병렬이라 가변 D 스케일링 개념은 구조적으로 다름, 이식 시 재설계 필요 |
| barrel shifter 기반 sub-block shuffling | `GT.cpp` `Make_GT_HW()` | MUX 기반 edge 선택과 유사한 라우팅 최적화 개념이나 barrel shifter 자체는 대응 없음 |
| H-matrix 360×360 QC 블록 구조 | `ecc_top.cpp` `Load_PCM()` | DVB-S2 전용 코드 구조로 Prime ECC 자체 H-matrix와 직접 호환 없음 |

적용 가치: 중간 - 메시지 압축과 스케일러블 데이터패스는 이미 유사 기법을 보유했거나 z=32 고정 구조와 맞지 않아 직접 이식은 어렵지만, "면적-처리량 선형 스케일링" 설계 사상은 HW 최적화 참고 가치 있음.
