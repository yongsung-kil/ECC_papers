### B. 알고리즘 요약

1. 대상은 고전 선형블록코드([24,12,8] extended Golay, [31,21,5]/[63,39,9] BCH)로, 원래 H가 고밀도(high-density)라 표준 Tanner graph 반복복호 성능이 낮다는 문제를 다룸.
2. 동일 코드를 나타내는 여러 패리티검사행렬(H1, H2, ...)로 구성한 "redundant Tanner graph"의 반복 SISO 복호를 통합 프레임워크로 정리 — 기존 Jiang&Narayanan(cyclic 코드 전용), Kothiyal ABP(임의 블록코드지만 고복잡도)를 일반화.
3. 채널모델은 표준 AWGN + BPSK(binary antipodal), 복호는 `-log` 도메인(min-sum류)에서 수행한다고 명시.
4. 핵심 기법(RRD, Algorithm 1): 매 inner iteration마다 부호의 순열군 `Per(C)`에서 무작위 순열 `θ`를 뽑아 소프트정보 벡터에 적용한 뒤 동일 base H로 Tanner graph 복호를 수행하고, 누적 소프트정보 `s ← s + s'`로 결합.
5. damping coefficient `α`를 outer iteration마다 `α0`에서 1에 가깝게 증가시켜(`α ← α0+(1-α0)*i3/(I3-1)`) local minima 회피 — 사실상 반복적 랜덤 재정렬(random re-permutation) + soft combining으로 고밀도 H의 짧은 사이클 영향을 희석하는 원리.
6. 초기 H 선택도 성능에 큰 영향을 미쳐, Algorithm 2(cycle-reduction row operation)로 Tanner graph의 4-cycle/6-cycle 수를 그리디하게 줄인 H'를 사용(예: Golay 코드에서 4-cycle 1551→295개로 감소, BER 약 1.75dB 개선).
7. 랜덤 순열군 원소 생성은 Celler et al.의 product-replacement 알고리즘(Algorithm 3, 그룹연산 1회로 저비용 생성)을 재사용.
8. 별도 학습(DE/GA/NN) 절차 없음 — 순열군은 코드의 automorphism 구조(예: BCH의 `σm(j)` 생성원)에서 대수적으로 유도.
9. 결과: extended Golay 코드에서 최적 trellis SISO 대비 약 0.3dB 열화(BER 1e-5), [31,21,5] BCH에서 0.25dB 열화, 기존 algebraic HIHO 대비 1.5~1.75dB 개선. 반복은 outer(I3=20)×inner(I2=30~50)로 최대 수백 회.
10. 한계: 순수 시뮬레이션(AWGN, 소형 블록코드)뿐이며 HW 설계·복잡도 분석 없음. 대상 코드가 QC-LDPC가 아닌 고밀도 H 대수적 코드(Golay/BCH)라 NAND 저밀도 QC-LDPC 이식과는 목적이 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum(`-log` 도메인) 기반 반복 SISO 복호 루프 | `decoder.cpp` `LDPC_Decoder()` | 반복 message-passing 구조는 유사하나 대상 부호(고밀도 H, redundant Tanner graph)가 달라 직접 이식 불가 |
| 랜덤 순열 기반 다중 H 앙상블 복호 | 대응 없음 | Prime ECC는 고정 QC-LDPC 단일 H·단일 스케줄 구조로 순열군 앙상블 기법 없음 |

> 적용 가치: 낮음 — 고밀도 H를 갖는 대수적 블록코드(Golay/BCH) 전용 기법으로, 이미 저밀도 QC-LDPC를 사용하는 Prime ECC의 문제 상황(고밀도→저밀도 근사) 자체가 발생하지 않아 이식 필요성이 없음.
