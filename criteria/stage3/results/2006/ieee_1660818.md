### B. 알고리즘 요약

1. Regular LDPC 코드, 선택 가능 rate {7/10, 6/9, 5/8, 4/7, 3/6, 2/5, 1/4}, code length 최대 1024, 0.1µm 공정.
2. 풀려는 문제: 기존 구조화(structured) LDPC 디코더는 고정 rate/length/parity 구조라 코딩 요구가 가변적인 응용(딥스페이스 통신, 소프트 라디오 등)에 대응 불가.
3. 핵심 가정: BP(belief propagation) 표준 알고리즘, AWGN 등 특정 채널 모델 언급 없이 순수 아키텍처/타이밍 설계에 집중.
4. 핵심 기법: Check-node Unit(CU)과 Variable-node Unit(VU)을 각각 중앙에 배치하고, 그 사이를 Switch-box Unit(SU) 배열로 완전 연결해 임의의 H-matrix를 필드 프로그래밍으로 구현.
5. 각 SU는 5-bit down-counter를 내장해 미리 로드된 사이클 값에서 0에 도달하면 해당 VU-CU 연결을 활성화하는 방식으로 메시지 전달 타이밍을 제어.
6. 메시지 직렬화: bit node가 보내는 여러 메시지를 병렬 버스가 아닌 단일 버스로 순차 전송해 라우팅 네트워크/다이 크기 축소.
7. 부수 트릭: 5단 OR-tree로 수직 배선을 분할해 배선 지연 최소화, VU-CU/CU-VU 전송을 동시 수행하는 2-frame 파이프라이닝.
8. 학습/최적화 절차 없음(순수 아키텍처/회로 설계).
9. 결과: TSMC 0.1µm 공정에서 clock 주기 660ps(60ps+600ps), rate 1/2 length 1024 코드 20 iteration 기준 처리량 2.44Gbps, 전력 7W(다이 12mm×8mm).
10. 한계: BER/정정성능 데이터 전혀 없음(순수 HW throughput/power/programmability 논문), 완전병렬 구조로 확장성(긴 코드)에는 SU 수정이 필요하다고만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Switch-box 기반 완전 재구성형 VN-CN 인터커넥트 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 base matrix+circulant shift 구조라 완전 프로그래머블 스위치 패브릭과는 개념이 다름 |
| 필드 프로그래머블 rate/length 전환 | 대응 없음 | Prime ECC는 z_sb=32 lifting 및 특정 H-matrix 고정, 런타임 rate 재구성 기능 없음 |
| 파이프라인/메시지 직렬화 배선 최적화 | 디코더 HW 모델 (z=32, HCU, GT) | 배선 지연 최적화 아이디어는 일반적이나 완전병렬 vs partial-parallel 구조 차이로 직접 대응 어려움 |

> 적용 가치: 낮음 | 완전병렬·완전 재구성형 아키텍처로 특정 QC-LDPC ECC(Prime ECC)와 설계 목표가 상이하고, 정정성능 데이터도 없어 이식 가치가 낮음.
