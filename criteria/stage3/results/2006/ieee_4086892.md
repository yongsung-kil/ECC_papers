### B. 알고리즘 요약

1. Regular LDPC는 variable/check node degree가 균일(`{d_v, d_c}`)한 부호로, {3,6}은 낮은 threshold(우수한 waterfall)로 널리 연구되었으나 {4,8}은 threshold가 약 0.25dB 나빠 덜 연구됨.
2. 풀려는 문제: {4,8} 코드가 고SNR 영역(error floor)에서 {3,6}보다 우수하다는 추측이 있었으나, 기존 Monte Carlo로는 초저오류율 영역 검증이 불가능했음.
3. Importance sampling(IS) 기반 저오류율 탐색 기법(별도 논문 [4])을 적용해 {4,8} 코드의 error floor를 실제로 정량화.
4. 핵심 정리: check node degree `d_c`가 짝수인 코드는 홀수 `b`(불만족 check 수)의 trapping set(a,b)이 존재하지 않음(Theorem 1) — H 열의 모드-2 합이 항상 짝수이기 때문.
5. 이 성질로 {4,8}은 {3,6}에서 흔한 저가중치 (a,1) TS(예: (5,1), (9,1))가 원천적으로 배제되어 error floor의 주원인이 줄어듦.
6. Trapping set 대칭성(Theorem 3): `d_v` 모두 짝수인 코드는 (a,b) TS 분포가 a에 대해 대칭.
7. 부수 구성법: girth-constrained random edge connection 절차에 확률적 edge 배정(`p_i ∝ d_ci - d_cur_i`)과 girth 위반 시 백트래킹을 결합, girth 6→8 전환 지점에서 4-cycle만 회피하도록 완화해 6-cycle 수를 축소(5000→1200 미만).
8. 학습/최적화 절차: girth 최적화 랜덤 탐색(GA/DE 아님, greedy probabilistic construction), dominant TS는 IS 3-step 절차로 탐색·검증.
9. 결과: (1200,600) {4,8} 코드가 {3,6} 대비 waterfall에서 0.25dB 열화되지만, 3.75~4.0dB에서 10억 프레임 무오류 등 압도적으로 낮은 error floor 달성. (504,252) 코드는 dominant (10,4) TS로 억제.
10. 한계: encoding은 {3,6} 대비 효율이 떨어지고(비직선시간), HW 구현·양자화·throughput은 전혀 다루지 않으며 순수 이론/시뮬레이션(부동소수점 추정)에 머무름.
