### B. 알고리즘 요약

1. (3,6)-regular LDPC, block length 2304, code rate 1/2, block-structured parity check matrix (3×6개의 384×384 shifted identity sub-block).
2. Row(check-node)와 column(bit-node) 연산을 동시에(concurrently) 수행하면 기존 serial 방식 대비 메모리가 2배 필요해지는 문제를 해결하고자 함.
3. 6bit symbol LLR을 입력으로 받는 modified min-sum 디코더를 가정.
4. Check-node Function Unit(CFU)이 실제 alpha(메시지) 값 대신 `minimum`, `2nd minimum`(각 5bit), 이를 선택하기 위한 flag-bit들, sign-bit들만 산출·저장하는 방식으로 메모리 사용량 절감.
5. 저장 정보(min/2nd-min/flag/sign)로부터 Bit-node Function Unit(BFU)이 beta 값을 재구성해 column 연산을 수행 — 실제 메시지 배열 전체를 저장할 필요가 없어짐.
6. CFU 36개(12×3), BFU 72개(12×6)를 병렬 배치해 그룹별 12개 row/column 연산을 동시 처리.
7. 모든 SRAM은 dual-port로 구성해 연속 디코딩 지원; 입력 SRAM은 2개 블록(현재 디코딩 중 블록 + 다음 블록)을 동시에 보관.
8. 별도 학습/최적화 절차 없음(shift value는 heuristic search로 사전 결정).
9. 147MHz 동작 시 throughput 530Mb/s(참고문헌 대비 127Mb/s에서 개선), gate count 206,343, chip size 36mm², 전력 3.6W(10 iteration 기준).
10. (3,6)-regular rate 1/2 코드 고정 설계이며, 정정능력·error-floor 관련 데이터는 제시하지 않음(순수 처리량/면적 관점 HW 논문).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CFU의 min/2nd-min/flag/sign 저장 방식(메모리 절감) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 메시지 저장 최적화 아이디어로 참고 가능하나 이미 유사 기법 보유 |
| row/column 동시 연산 스케줄링 | `decoder.cpp` `LDPC_Decoder()` | Dual-Update와 유사한 개념, 신규성 낮음 |
| (3,6)-regular block-structured PCM | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조라 regular 코드 자체 이식 난이도 높음 |

> 적용 가치: 낮음 — 메모리 절감 기법이 Prime ECC의 min-sum/HCU 구조와 상당 부분 중복되고, regular LDPC 특유의 구조라 QC-LDPC 이식에는 큰 이득이 없음.
