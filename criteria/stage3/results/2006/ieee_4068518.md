### B. 알고리즘 요약

1. `(3,6)` regular LDPC(inner, rate 0.8008)와 BCH(63,51)(outer, rate 51/63) 직렬연접, 전체 code rate 0.6483, block size 1416 bit, AWGN+BPSK 채널.
2. medium-length LDPC 코드는 Tanner graph의 short loop 때문에 순수 BP 반복복호가 ML 성능에 못 미치는 문제를 해결하고자 함.
3. 채널은 AWGN, LDPC BP 표준 message-passing 모델(`q`, `r` LAPPR 메시지) 사용.
4. 핵심 기법: inner LDPC가 고정 iteration까지 BP 복호 후 실패 시, BP 소프트출력(`q_n` from 식(3))을 reliability로 사용해 outer BCH에 대해 Chase-2 알고리즘으로 소프트결정 복호 수행.
5. Chase 알고리즘은 최소신뢰비트(LRB) `p`개를 골라 `2^p` 테스트패턴을 생성, `min W_a(Z_m)`(식 5)로 최소 아날로그 가중치 오류패턴을 찾음 - 신뢰도가 낮은 비트만 뒤집어 후보 codeword 집합 생성.
6. BCH 소프트출력 `L(y'_j)`(식 6, AWGN 분산 `σ²`로 스케일)를 다시 inner LDPC의 `q_n` 초기값으로 주입(식 7,8)해 다음 BP 라운드 반복 - inner/outer 간 extrinsic 정보 교환.
7. 부수 트릭: Gray code 기반 테스트패턴 재정렬로 low-complexity fast Chase 구현, erasure 위치와 hard-decoded 위치를 분리 처리해 소프트출력 계산 단순화.
8. 별도 학습/최적화 절차 없음 (반복 스케줄만 설계: inner max iteration=10, outer round=5 또는 10, Chase `p`=3,4,5로 스윕).
9. 결과: `p=4`, outer round=5일 때 순수 LDPC 대비 0.3dB 코딩이득 (BER 곡선 기준, Eb/N0 vs BER).
10. 한계: HW 미설계, 순수 시뮬레이션(BER 곡선)만 제시. Outer round·Chase test pattern 수(`2^p`)에 따라 복잡도가 급증하며, error-floor 영역 데이터는 제시되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC BP 반복복호 (inner) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | BP 기반이나 Prime ECC는 min-sum 사용, 알고리즘 계열 상이 |
| BCH Chase-2 SISO 복호 (outer) | 대응 없음 | BCH 알고리즘 자체가 Prime ECC(단일 LDPC 기반)에 없음 |
| LDPC-BCH 간 소프트정보(extrinsic) 교환 | 대응 없음 | 연접부호 구조 자체가 Prime ECC 아키텍처에 없음 |
| 조기종료 (Hx=0 검사) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 유효 codeword 검사 개념은 유사하나 구현 방식(패리티 검사 vs CRC) 다름 |

> 적용 가치: 낮음 | LDPC-BCH 직렬연접 구조와 Chase 알고리즘은 Prime ECC의 단일 QC-LDPC binary 디코더 구조와 근본적으로 다르며(outer BCH 블록 필요), 개선폭(0.3dB)도 크지 않고 복잡도 증가가 상당함.
