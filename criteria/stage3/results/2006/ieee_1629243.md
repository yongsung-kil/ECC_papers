### B. 알고리즘 요약

1. (1024,3,6) regular LDPC, 0.18um CMOS, 부분병렬 아키텍처 대상.
2. 기존 fully-parallel 구조는 배선(interconnection)이 복잡하고, serial 구조는 wide-input mux가 필요해 두 방식 모두 비효율적.
3. VTC(variable-to-check)/CTV(check-to-variable) 메시지 `r_mn`, `q_mn` 대신 row summation 값 `R_m`, column summation 값 `Q_n`만 두 데이터 흐름(check-flow, variable-flow) 사이에 교환하는 loosely-coupled 구조를 제안.
4. 각 PU가 summation 값으로부터 개별 메시지를 복원하는 추가 연산(중복 연산)을 포함하도록 (13)~(20) 식으로 재구성, 대신 인터커넥트 단순화.
5. 중간값 `R_m`, `Q_n`을 레지스터 대신 PU 하나만 접근하는 local memory(dual-port SRAM)에 저장해 면적 절감.
6. 부분병렬 구조에서 각 PU가 여러 행/열을 담당하도록 확장, VTC/CTV 오버랩 처리로 지연시간 단축.
7. row/column 순서를 정하는 스케줄링 알고리즘(CC, CTC 정의 기반 그리디 매트릭스 permutation)으로 미처리 구간(empty slot)을 최대화해 사이클 단축.
8. 학습/최적화 절차 없음 — 그리디 휴리스틱 스케줄링(각 스텝에서 최대 CTC/CC 행 선택).
9. 결과: 사이클/iteration 평균 36.7% 절감(스케줄링), parallelism(32,64) 구현 시 10.08mm2@0.18um, 200MHz, 약 1Gbps throughput (Blanksby 대비 약 1/5 면적).
10. 한계: 정규(regular) 코드 및 특정 parallelism 조합에서만 검증, error-floor/waterfall 등 정정성능 개선은 다루지 않음(순수 HW 구조 논문), dual-port SRAM 최소 word수(32) 제약으로 parallelism 확장에 한계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| row/column summation 기반 loosely-coupled 메시지 교환 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 min-sum + HCU 기반 유사 최적화 보유, 개념적 참고만 가능 |
| 부분병렬 아키텍처 및 로컬 메모리 접근 | 디코더 HW 모델 (z=32, HCU, GT) | HW 병렬화/메모리 구조가 이미 다른 방식(z_sb=32 lifting)으로 구현되어 있어 직접 이식보다는 참고용 |
| matrix permutation 기반 스케줄링 알고리즘 | `GT.cpp` `Make_GT_HW()` | 엣지/스케줄 최적화 관점에서 유사하나 non-QC 일반 LDPC 대상이라 QC 구조에는 대응 어려움 |
| 인코딩 | 대응 없음 | 논문은 인코더 다루지 않음 |

> 적용 가치: 낮음 — HW 배선/면적 최적화가 핵심이나 Prime ECC는 이미 QC-LDPC 고정 lifting(z=32) 기반 HCU/GT로 유사한 문제를 해결했고, 본 논문의 일반(비-QC) LDPC 대상 스케줄링은 구조적으로 이식 난이도가 높음.
