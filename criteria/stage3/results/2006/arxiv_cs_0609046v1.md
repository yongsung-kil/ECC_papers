### B. 알고리즘 요약

1. 유한 길이(n≈500) binary LDPC 코드의 BEC용 stopping set과 일반 채널용 (k-out) trapping set을 정의하고, 이들의 최소 크기(minimal stopping/trapping distance) 결정 문제가 NP-complete임을 증명.
2. 기존 방법(density evolution, importance sampling, instanton analysis)은 근사적이며, 소형 error-prone 패턴을 놓쳐 error-floor 예측이 부정확한 문제를 해결하고자 함.
3. all-zero codeword 가정 하 BEC 반복복호를 Boolean AND(변수노드)/OR(체크노드) 함수로 모델링하는 "Boolean expression framework" 사용.
4. Tree-based narrowing search(Algorithm 3): 디코딩 트리를 점진적으로 확장하며 Rule 0(독립 가정 DE-like 평가), Rule 1(단순 완화), Rule 2(pivoting rule로 공통 결정변수 처리)를 적용해 BER 상한 `UBi`를 구성.
5. `UBi ≥ E{fi(y)} = pi`이며, 트리 크기를 늘릴수록(t→t0) 후보집합 `Xt`가 실제 stopping set 집합 `Xxi`로 수렴(Theorem 4, 최적 leaf-finding 모듈 가정).
6. 확장: k-out trapping set(체크노드 degree-1이 k개 남는 패턴)으로 일반화해 non-BEC 채널의 error-floor 원인도 동일 알고리즘(k-TSE)으로 탐색.
7. 효율화 트릭: Rule 3(0 곱 가지치기), Rule 4(퇴화 pivoting), leaf-finding 모듈 설계, brute-force와 결합한 composite 접근(C-UB).
8. 별도 논문([18])의 "code annealing" 코드 최적화에서 이 exhaustion 결과를 목적함수로 사용해 코드를 재구성.
9. 결과: n=512 (3,6) 코드에서 stopping set 크기 ≤13, trapping set 크기 ≤11까지 완전탐색 성공(brute force 대비 2.5×10^25배 절감); 최적화 코드는 랜덤 코드 대비 FER 약 10,000배 개선(ǫ=0.3에서).
10. 한계: 알고리즘 자체가 NP-complete이라 n≈500~1000 규모로 제한되며, 순수 소프트웨어 시뮬레이션 기반 오프라인 코드 설계 도구로 디코더/HW 구현은 다루지 않음.
