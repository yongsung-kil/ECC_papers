### B. 알고리즘 요약

1. 대상은 blockwise 구조 QC-LDPC (IEEE 802.11n WWiSE 코드, `N=1944`, sub-block size `B=27`, rate 1/2/2/3/3/4/5/6), min-sum(M-min*) 기반 log-domain BP 디코딩.
2. 문제: 일부 코드는 두 circulant sub-block이 동일 위치(같은 VN/CN 연결, 다른 rotation)에서 겹쳐(column weight>1) layer 내에서 서로를 참조하는 순환 의존성이 생겨 정식 layered decoding(Hocevar 방식)이 불가능 — 이를 "non-layered" 코드로 정의.
3. 겹침 정도를 `η = N_overlap_block / N_total_block`로 정량화, layered 근사의 적용 가능성 척도로 사용.
4. 핵심 기법: soft-output(SO) 업데이트식 `y_n^(q)[k] = y_n^(q)[k-1] - μ_{m,n}^(q-1) + μ_{m,n}^(q)`(식 9)을 두 가지로 근사 — (a) SO-based: 마지막 갱신값으로 덮어쓰기, (b) δ-based: `Δμ_{m,n}^(q) = μ_{m,n}^(q) - μ_{m,n}^(q-1)`를 계산해 SO에 누적(식 10, 11).
5. δ 방식은 겹치는 sub-block들의 부분 업데이트를 합산 방식으로 결합해 병렬 처리 시 정보 손실을 줄이는 것이 핵심 — SO 덮어쓰기 방식보다 견고함.
6. 확장: layer 폭을 자연 구조(B=27) 이상으로(`L=k·B`) 늘려도 δ-메커니즘이 강건함을 시뮬로 확인 (η>30%에서도 성능 저하 미미).
7. HW 구현 디테일: δ-processor는 dual-port SO 메모리(포트A read, 포트B read-modify-write)를 2fclk로 구동해 delay-line 없이 실시간 처리; VHDL로 두 아키텍처(SO-based, δ-based) 설계, 0.18μm CMOS로 합성.
8. 최적화 절차: macro-row(=B=27행 집합) 순서를 재배열(reordering)해 η를 최소화하는 층(layer) 시퀀싱을 적용 — 학습이 아닌 결정론적 재배치.
9. 결과: 5-bit quantization 기준 δ-layered(L=27)는 flooding 대비 iteration 수 약 절반, throughput은 4배(정규화 latency 기준); ASIC 합성 결과 δ/SO-processor 오버헤드는 check node core의 22~24%(gate), VN processor 생략으로 41~43% 전체 gate 절감.
10. 한계: 정정능력(BER/FER waterfall) 자체의 개선은 없음(오히려 SO-based 방식은 0.1dB 손실), 검증 코드가 무선(802.11n) 특화이며 QC-LDPC circulant 구조가 NAND ECC와 다를 수 있음, error-floor 영역은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered(horizontal shuffle) 스케줄링 및 δ-based 근사 업데이트 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프의 메시지 갱신 스케줄(Dual-Update와 유사 계열)에 δ 누적 방식 이식 가능성 검토 대상 |
| SO(soft output) 메모리 관리, 겹침 블록 처리 | `decoder.cpp` `VN_Cal_HD()` 등, HCU 로직 | Prime ECC의 HCU(비대칭 degree column 2-cycle 처리)와 개념적으로 유사한 "겹침 처리" 문제이나 실제 겹침 여부는 H-matrix 종속 |
| min-sum(M-min*) 연산 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 Prime ECC가 Min-Sum 보유 — 중복 기여 |
| 조기종료(iteration 감소 측정) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 논문의 iteration 절반 감소 효과와 비교축으로 참고 가능하나 직접 대응 아님 |
```

> 적용 가치: 중간 — 이미 보유한 Dual-Update와 유사한 계열의 스케줄링 개선이라 완전히 새로운 기여는 아니지만, "겹치는 circulant block" 구조에서의 δ 누적 방식은 Prime ECC의 HCU/GT와 결합 검토 시 iteration/latency 개선에 참고 가능.
