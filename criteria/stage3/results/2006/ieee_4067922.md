> 총평: {3,6} regular LDPC의 trapping set을 국소 그래프 구조로 효율 탐색해 error floor를 수 자릿수 낮추는 부호 설계 기법이나, HW/디코더 알고리즘 자체 개선은 없고 부호 구성(오프라인 설계 단계)에 국한.

### B. 알고리즘 요약

1. 대상은 {3,6} regular LDPC (moderate length, 최대 수천 bit), AWGN 채널에서 MPA(message passing algorithm) 디코더 사용.
2. 문제: 고 SNR 영역 error floor를 지배하는 dominant error event(trapping set, TS)를 일반 코드에서 brute-force 없이 찾기 어려움.
3. TS는 `(a,b)` 쌍(가중치 a, unsatisfied check 수 b)으로 정의되며 elementary TS를 가정.
4. 핵심 기법: variable node를 root로 한 국소 Tanner graph tree의 1차 계층에서 얻어지는 4-bit 조합 집합 `S2` (`|S2| = n(dc-1)dv`)만을 대상으로 다중비트 "error impulse" `ε`를 주입해 MPA 디코더가 어디로 수렴하는지 관찰.
5. `S1`(전체 4-bit 조합, `C(n,4)`) 대비 `S2`가 훨씬 작아 탐색공간을 `O(n^4)`에서 `O(n(dc-1)dv)`로 축소 (Theorem 1: g≥6인 {3,6} 코드의 모든 dominant TS는 S2의 4-bit 조합을 최소 하나 포함).
6. 확장: 더 긴 코드는 4-bit로 부족하여 tree를 한 단계 더 전개한 10-bit 조합(`(dc-1)dv·(dc-1)dv(dv-1)`개)으로 탐색.
7. 구현 디테일: error impulse 크기 ε, 최대 iteration(30), 나머지 n-4 비트 스케일 계수, 디코더 입력 SNR(Eb/N0=4dB)을 파라미터로 지정.
8. 설계 절차: 탐색으로 candidate PCM의 TS 스펙트럼을 확인, 무작위 PCM 생성-탐색-재시도를 반복하여 낮은 error floor를 갖는 부호를 선택 (girth 8 우선).
9. 결과: (96,48) 코드 전수 탐색 1분 이내(AMD Athlon); (204,102) 새 설계는 MacKay 코드 대비 error floor 약 2자릿수 개선; (1008,504) 새 설계는 5~6자릿수 개선 추정.
10. 한계: 순수 시뮬레이션/설계 도구 수준으로 HW 구현 없음; 긴 코드일수록 탐색 비트수 증가로 비효율.

| trapping set 탐색(다중비트 error impulse) 자체 | 대응 없음 | 오프라인 부호 설계/분석 도구이며 디코더 HW 루프와 무관 |
| MPA 디코더를 이용한 TS 검증 | `decoder.cpp` `LDPC_Decoder()` | 탐색시 사용하는 MPA와 동일 계열이나 이 논문은 디코더 알고리즘 자체를 개선하지 않음 |

> 적용 가치: 낮음 — regular {3,6} 부호에 특화된 오프라인 error floor 설계·검증 기법으로, Prime ECC의 고정 QC-LDPC 구조에 직접 이식하기보다는 참고자료 수준.
