### B. 알고리즘 요약

1. 대상 문제는 통신 시스템 역공학: 관찰자가 부호(스크램블러/인코더 포함)를 모르는 상태에서, 노이즈가 낀 코드워드 스트림 `X`(M×n 행렬)만으로 이진 선형블록코드 `C`의 dual code `C⊥`(즉 parity-check 관계식)를 복원하는 것.
2. 기존 접근(Valembois)은 순수 통계적 가설검정(저Hamming weight 후보 `h`가 `hX^T`의 weight 분포로 `C⊥` 소속 여부 판별)뿐이었는데, 본 논문은 여기에 반복 디코딩 피드백을 결합해 더 효율적으로 만드는 것이 목표.
3. 채널모델은 binary symmetric channel(cross-over probability `τ`)이며, dual code 원소 `h`의 weight `wt(h)`가 낮을수록 `hX^T`의 weight가 통계적으로 더 뚜렷하게 낮아진다는 성질(Proposition 1/Corollary 1)을 이용.
4. 핵심 기법 1단: low-weight codeword 탐색(FWLW, Canteaut-Chabaud 방식 information-set 기반 무작위 탐색)으로 `C⊥` 후보 패리티검사식 `h`들을 수집, 길이-4 미만 짧은 사이클을 형성하는 식은 제거.
5. 핵심 식: 각 후보 식 `h`가 실제 `C⊥`에 속할 사후확률 `Pr[h∈C⊥|wt(hX^T)=w]`를 베이즈 정리(식 (1))로 계산 — 코드 exact 여부가 불확실하므로 "확률적 패리티검사식"이라는 개념 도입.
6. 핵심 기법 2단: Gallager의 원 반복복호 알고리즘을 확장하여 각 패리티검사식이 참일 확률 `p`를 반영한 "가중 패리티검사식 기반 반복복호"(section III-D)로 노이즈 코드워드의 오류를 정정하면서 동시에 식들의 신뢰도 `p_{e,i}`를 갱신.
7. 부수 트릭: `⊗` 연산자(두 독립 확률원 결합, `u⊗v=uv/(uv+(1-u)(1-v))`)로 관측정보 `Obs`와 각 식에서 온 extrinsic 정보 `Extr_e`를 결합 — SPA의 벨리프 결합과 동일한 형태.
8. 학습/최적화 절차: FWLW 탐색과 반복복호를 교대로 수행하는 outer loop(복호로 오류율을 낮춘 뒤 다시 low-weight word 탐색을 반복해 새 식을 추가)로 재구성 정확도를 점진적으로 향상.
9. 결과(시뮬): 길이 100 LDPC(3,6) 코드는 `τ=0.03`에서도 수 분 내 재구성 성공(perr가 0.03→0.0012까지 감소), 길이 1000 코드는 `τ≤0.002` 수준의 매우 낮은 채널오류율에서만 성공. 랜덤 코드는 LDPC보다 재구성이 훨씬 어려움.
10. 한계: 순수 알고리즘/소프트웨어 시뮬레이션(Pentium 4)뿐이며 HW 설계 없음. 대상 자체가 "코드 미지" 리버스엔지니어링(주로 신호정보/보안 응용)이라 부호가 이미 설계·고정된 NAND ECC 문제와 목적이 근본적으로 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 미지 코드의 dual code(H) 통계적 복원 | 대응 없음 | Prime ECC는 H-matrix가 이미 설계·고정되어 있어 코드 복원 문제 자체가 발생하지 않음 |
| 가중 패리티검사식 기반 Gallager 반복복호 | `decoder.cpp` `LDPC_Decoder()` | belief 결합(⊗) 구조가 SPA와 유사하나, "식의 신뢰도 자체가 불확실"한 상황 전용이라 확정된 H를 쓰는 Prime ECC 디코더와 목적이 다름 |

> 적용 가치: 낮음 — 코드가 미지인 리버스엔지니어링/보안 문제 전용 알고리즘으로, H가 이미 확정된 NAND binary QC-LDPC 설계·복호 파이프라인에 이식할 대상이 없음.
