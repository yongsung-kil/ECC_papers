### B. 알고리즘 요약

1. (3,6)-regular LDPC, 코드길이 3072bit, block-structured PCM(M×N sub-block, 각 b×b shift 행렬) 기반 partially-parallel 디코더 대상.
2. 기존 accelerated message-passing 스케줄(Ref. [8])은 α/β 메시지를 각각 dual-port memory에 저장해 buffer word수가 classical schedule 대비 2배로 커지는 문제.
3. row 연산으로 갱신된 α는 같은 클록에 3개 column 연산에 즉시 사용되어 row/column 연산이 동시 수행되는 것이 핵심 가정.
4. 제안 1: 파이프라인을 read/write가 동일 클록에 겹치지 않도록 7단계(row)/3단계(column)로 분할, single-port memory로 구현 가능하게 함.
5. 제안 2: sub-block 간 shift값 차이 `sp,q - sp',q'`을 이용해 FIFO 버퍼 길이 `kp(α,q)`, `kp(β,q)`를 식 (3),(4)로 계산, α/β를 shift-register FIFO로 대체.
6. 한 column block(M=3)의 총 버퍼워드는 `3b-6`으로, 기존 accelerated/concurrent 스케줄의 `2×3×b` 대비 절반 이하.
7. 워드수 16 미만 FIFO는 shift register로, 이상은 memory bank로 구현하는 하이브리드 구현 디테일.
8. 별도 학습/최적화 절차 없음 (수식 기반 정적 설계).
9. 결과: FPGA(Virtex-II XC2V6000) 구현에서 메모리 리소스 30%, 전체 slice 15% 절감(19,189→16,494 slices), throughput 102Mbps, coding gain은 Ref.[8]과 동일(5.40dB@BER=1e-5).
10. 한계: (3,6)-regular 코드 한정 실험, 정정성능 자체는 개선 없음(기존 accelerated schedule과 동일), 최신 QC-LDPC 고rate 구조로의 일반화는 본문에서 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| single-port memory 파이프라인 분할 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 내 read/write 타이밍 설계에 참고 가능 |
| FIFO 기반 α/β 버퍼링 | 대응 없음 | Prime ECC는 z_sb=32 QC 구조로 shift-diff FIFO 설계가 직접 이식되지 않음 |
| CFU min1/min2 계산 (min-sum 근사) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 보유한 min-sum과 동일 기법, 신규성 낮음 |
| 메모리 뱅크 수/면적 절감 | 디코더 HW 모델 (z=32, HCU, GT) | 메모리 최적화 아이디어로만 참고 가능 |

> 적용 가치: 낮음 — 정정성능 개선 없이 (3,6)-regular 코드의 메모리 절감 HW 기법이며, Prime ECC는 이미 HCU/GT로 유사한 메모리·면적 최적화를 보유해 중복성이 큼.
