> 총평: UMP-BP(min-sum 계열)의 bit node process에 이전 iteration의 extrinsic LLR을 조건부로 가산하는 저복잡도 oscillation 완화 기법으로, 순수 알고리즘 수정이라 decoder.cpp에 국한된 이식이 가능하나 short/middle length 코드에서만 검증되어 고rate NAND급 QC-LDPC에서의 효과는 미상.

### B. 알고리즘 요약

1. 대상은 regular LDPC 부호((504,252), (1008,504), rate 1/2), AWGN 채널, BPSK 변조, LLR-BP와 저복잡도 UMP-BP(=min-sum 계열) 두 디코딩 알고리즘 모두에 적용.
2. Short/middle length 코드에서는 H-matrix의 cycle로 인해 일부 bit의 a posteriori LLR 크기가 iteration마다 진동(oscillation)하며, 이것이 high Eb/N0 영역에서 지배적 오류 요인.
3. 핵심 정의: check node `m`에서 bit node `n`으로의 ex-LLR `z_mn`의 부호가 현재 iteration과 이전 iteration에서 다르면(`sgn(z_mn^t) ≠ sgn(z_mn^{t-1})`) 이를 oscillating으로 간주(식 1).
4. 핵심 기법: 표준 bit node process(`z_mn = F_n + Σ L_m'n`, 식 8)를 수행한 후, oscillating으로 판정되면 현재 ex-LLR과 이전 ex-LLR을 더한 값을 새 ex-LLR로 치환(식 9), oscillating이 아니면 그대로 사용.
5. 가산 결과 새 ex-LLR의 크기는 두 이전 값보다 작아지고 부호는 더 큰 크기를 가진 쪽을 따르며, check node process(LLR-BP는 tanh 기반 식4, UMP-BP는 min 기반 식5)는 기존과 동일하게 유지.
6. 부수 트릭: UMP-BP는 초기 LLR로 `y_n`(노이즈 분산 추정 불필요)을 그대로 사용하는 저복잡도 방식이며, check node에서 곱셈 대신 min 연산만 사용.
7. 학습/최적화 절차 없음(density evolution 등 별도 최적화 없이 단순 규칙 기반 수정).
8. 결과: (504,252), (1008,504) 코드에서 제안 알고리즘이 기존 LLR-BP/UMP-BP 및 normalized/offset BP(α=1.25, β=0.15) 대비 BER/BLER 개선, 특히 modified UMP-BP가 conventional LLR-BP보다도 우수한 BLER 달성. 평균 반복 횟수는 거의 동등(iteration 수 감소 효과는 없음, 일부 저Eb/N0에서 오히려 증가).
9. 복잡도 증가는 크지 않다고 주장(추가 덧셈 연산만, tanh/exponential 대비 미미)하나 gatecount 등 정량 수치는 없음.
10. 한계: HW 구현/합성 전혀 없이 시뮬레이션(최대 1000 iteration)만 수행, 코드 길이가 길어질수록(1008 코드) 개선폭이 감소함을 자체 확인(short cycle 감소 때문).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit node process에서 ex-LLR oscillation 탐지 및 조건부 가산 | `decoder.cpp` `VN_Cal_HD()` 등 VN 처리 로직 | VN 갱신 단계에 이전 iteration ex-LLR 저장 및 부호 비교 로직 추가로 이식 가능 |
| UMP-BP(min 기반) check node process | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 코어와 동일 계열이라 gating 로직만 추가하면 결합 가능 |
| iteration별 이전 ex-LLR 저장(추가 메모리) | `ecc_data.h` `PARAM_LLR` | 이전 iteration의 V2C 메시지를 별도 저장해야 하며 메모리 구조 확장 필요 |

> 적용 가치: 중간 — UMP-BP(min-sum 계열)에 적용된 저복잡도 oscillation 완화 기법은 decoder.cpp의 VN 처리 로직에 조건부 가산만 추가하면 되어 구조적 이식 난이도는 낮지만, short/middle length 코드에서만 검증되어 고rate NAND급 QC-LDPC(z=32, 129 col-block)에서 동일 효과가 있을지는 불확실.
