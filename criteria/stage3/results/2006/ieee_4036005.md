### B. 알고리즘 요약

1. 대상 코드는 projective geometry PG(2,2^s)로 구성된 LDPC 코드 `Cs`, column weight `γ=2^s+1`, girth≥6, BSC(전이확률 `α`) 채널을 가정.
2. 기존 연구는 이상적(perfect) 게이트로 구성된 디코더 성능만 다뤘으나, 실제 회로는 결함(faulty) 게이트로 구성되어 회로 자체가 오류를 유발할 수 있다는 문제를 다룸.
3. check node는 `(γ-1)`-input faulty XOR 게이트로, variable node는 "golden"(무결함) majority-logic 게이트로 구성된다고 가정. XOR 게이트는 확률 `ε`로 독립적으로 출력을 반전.
4. 핵심 기법은 message-passing 방식 one-step majority decoder에서, variable node v가 받는 잘못된 메시지 개수 k의 평균 확률 `b_{i,k}`를 채널 오류 개수 i와 게이트 실패 개수의 조합으로 계산(Theorem 1, 2).
5. `Pv(α,ε) = Σ_i Σ_{k>γ/2} P^i_{v,k}`: v가 절반 초과의 잘못된 메시지를 받으면 miscorrection 발생, 이 확률의 합이 비트오류확률.
6. PG 코드 특유의 조합구조(변수-체크 이웃이 모든 다른 변수노드를 정확히 한 번씩 포함)를 이용해 파티션(partition) 기반으로 계산을 단순화(Lemma 2~4).
7. 실제 계산에서는 낮은 α 영역에서 오류 개수 i가 큰 항을 생략하는 근사(예: 길이 73 코드는 i≤9만 고려)로 계산량 절감.
8. 별도의 학습/최적화 절차 없음 — 순수 조합론적 확률 계산.
9. 결과: PG(2,2), PG(2,4), PG(2,8), PG(2,16) 4개 코드에 대해 ε=10^-2~10^-6 범위에서 BER 곡선 제시. 결함 확률 ε=10^-3에서도 성능 열화가 작다는 결론(정성적, 수치는 그래프에만 있어 본문 서술 없음).
10. 한계: iterative(multi-step) message-passing LDPC 디코더가 아닌 one-step majority-logic 디코더에 한정, HW 구현/합성 없음, 순수 이론/수학적 분석뿐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| one-step majority-logic 디코딩 | 대응 없음 | Prime ECC는 min-sum 기반 iterative message-passing만 사용, majority-logic 디코더 없음 |
| faulty gate 신뢰성 해석 | 대응 없음 | Prime ECC 프로파일은 HW 결함 모델을 다루지 않음 (bit-exact 시뮬레이터로 게이트 결함 미고려) |
| PG 코드 구성 | `ecc_top.cpp` `Load_PCM()` | PG 코드는 특정 QC-LDPC 구조와 무관한 별도 부호족, 참고 가치만 있음 |

적용 가치: 낮음 — one-step majority-logic 디코더 및 faulty-gate 신뢰성 해석은 Prime ECC의 min-sum QC-LDPC 아키텍처와 근본적으로 다른 영역.
