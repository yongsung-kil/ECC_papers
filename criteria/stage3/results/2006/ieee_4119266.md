### B. 알고리즘 요약

1. 채널은 AWGN(BPSK), 코드는 (8000,4000) regular (3,6) LDPC이며 표준 BP 대비 반복 횟수 감소가 목표.
2. 표준 BP는 매 반복에서 모든 check-to-bit 메시지를 이전 반복의 bit-to-check 메시지만으로 갱신하여 수렴이 느림.
3. Shuffled BP는 이미 갱신된 일부 symbol node의 메시지를 같은 반복 내에서 즉시 재사용해 수렴을 앞당김.
4. 제안 기법(improved shuffled BP)은 한 check node `m`에서 갱신된 메시지를 현재 symbol node `n`뿐 아니라 `m`에 연결된 다른 모든 symbol node `n'`의 vertical step에도 즉시 반영.
5. 식 (7)~(8): horizontal step은 shuffled BP와 같은 `tanh` 기반 min-sum 계열 갱신이나, vertical step 합 `Σ_{m'∈M(n)} λ` 에 방금 갱신된 `m`의 메시지를 포함해 갱신 시점을 앞당김.
6. 기존 shuffled BP(4,5,6)는 갱신 메시지를 현재 symbol node 하나에만 전파하는 반면, 제안 기법은 해당 check node에 연결된 모든 symbol node로 즉시 전파(핵심 차이).
7. 알고리즘 구조상 완전 직렬(serial) 처리이며 HW 구현이나 양자화·근사에 대한 설명은 없음.
8. 별도 학습/최적화 절차 없음(고정 스케줄링 변경만).
9. WER 10^-4·최대 20 iteration 기준 shuffled BP가 standard BP보다 0.2dB, 제안 기법이 shuffled BP보다 추가로 0.2dB 개선; BER 10^-7에서도 0.2dB 개선.
10. 단일 코드(8000,4000)·AWGN 채널의 시뮬레이션만 존재하며 HW 설계, 복잡도, 병렬화 분석은 전무.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 개선된 shuffled BP 스케줄링(check→모든 인접 symbol 즉시 전파) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 Dual-Update(홀짝 교대 스케줄)와 유사한 스케줄링 최적화 계열, 알고리즘 위치는 일치 |
| horizontal/vertical step tanh 기반 메시지 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min-sum 근사(비-tanh) 사용, 본 논문은 sum-product 계열이라 직접 대응 약함 |
| HW/양자화/파이프라인 관련 논의 | 대응 없음 | 본문에 HW 설계 요소 없음 |

> 적용 가치: 낮음 — sum-product 기반 스케줄링 개선이며 Prime ECC는 이미 min-sum + Dual-Update 스케줄을 보유해 중복 기여, HW/복잡도 검증도 전무.
