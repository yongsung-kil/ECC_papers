### B. 알고리즘 요약

1. 목표는 medium-size(수 kb) LDPC 코드의 HW 효율적 인코더 설계이며, `H=[Hs Hp]` 구조(`Hp`는 dual-diagonal 삼각행렬)로 parity를 `p^T = Hp^-1 v^T`(선형 시간)로 계산.
2. 기존 dual-diagonal(eIRA) 인코더는 parity bit를 순차(serial) 계산해 latency가 parity bit 수 `M`에 비례하는 문제가 있음.
3. 채널모델은 AWGN(QPSK), rate 1/2 및 punctured rate 3/5, 2/3 코드로 시뮬레이션.
4. 핵심 기법은 두 개의 eIRA 코드를 결합한 신규 parity check 행렬 `H=[[H1,A1,Hp,0],[A2,H2,0,Hp]]`(`A1,A2`는 link matrix)로, 정보워드를 `c1,c2` 두 블록으로 나눠 parity `p1,p2`를 동시(병렬) 계산.
5. `p1^T = Hp^-1(H1c1^T + A1c2^T)`, `p2^T = Hp^-1(H2c2^T + A2c1^T)` — link matrix가 두 블록 간 정보를 교환해 concatenation 성능 저하를 방지.
6. 확장 기법으로 puncturing 규칙(`Rc ≤ R ≤ 2Rc/(Rc+1)`, parity bit를 alternate 패턴으로 천공)을 제시, 인접한 두 parity bit가 동시에 천공되지 않도록 제약.
7. 부수 트릭: 길이-4 사이클을 완전탐색으로 제거하는 2단계 알고리즘(전체 파라미터 집합에서 4-cycle 없는 `Hi` 선택 → link matrix 포함 global graph에서도 4-cycle 없는 조합 선택), Pi-rotation 행렬을 constituent code 예시로 사용.
8. 최적화 절차는 error impulse method 기반 weight distribution 계산(`Aw,h` 조합식)으로 최소거리(distance profile) 극대화 파라미터 탐색.
9. 결과: K=60byte rate 1/2 코드에서 제안 구조가 표준 Pi-rotation 코드와 유사한 BER 성능을 유지하면서 인코딩 latency를 2배 절감(throughput 2배 개선 가능).
10. 한계는 AWGN·무선 통신(수백~1천 bit) 대상 시뮬레이션뿐이며 HW 합성/게이트카운트 등 실제 구현 검증이 없고, 디코더 측 이점은 "향후 과제"로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Dual-diagonal parity 계산 구조(`Hp`, 미분인코더) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코더 보유(§3) — 본 논문의 병렬 2-block 구조는 인코더 latency 개선 아이디어로 참고 가능 |
| Link matrix 기반 병렬 인코딩(2-way 분할) | `encoder.cpp` `Column_Reordering_Dual_Update()` | dual-update column 재배치와 유사 개념(병렬 처리 단위 분할)이나 정확한 대응은 불명확 |
| 짧은 사이클(length-4) 제거 알고리즘 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 시 사이클 제거는 표준 설계 규칙과 유사, 특별한 신규 이식 요소는 적음 |
| Puncturing 패턴 설계 | 대응 없음 | Prime ECC는 고정 rate QC-LDPC로 puncturing 스킴 미보유 |

적용 가치: 중간 — dual-diagonal 인코더를 이미 보유한 Prime ECC 입장에서 병렬 2-block latency 개선 구조는 참고할 만하나, 검증 수준이 시뮬뿐이고 코드 길이/rate가 NAND ECC와 상당히 다름.
