### B. 알고리즘 요약

1. DMB-T 표준의 (7493,3048/6096) irregular QC-LDPC(rate 0.4/0.6/0.8), AWGN+BPSK 채널을 대상으로 함.
2. 기존 min-sum은 CNU에서 최소 LLR 하나만 사용해 저장 메시지는 줄지만 성능 손실이 큼 — 이를 개선하려는 문제.
3. Φ(x)=-ln[tanh(x/2)] 기반 CNU 연산에서 `Φ⁻¹[ΣΦ(x)]`는 최소 크기 입력값에 지배됨을 관찰.
4. Version-A: Vk(j)(LLR 크기 최소 k개 부분집합) 소속 여부에 따라 R_ji를 V(j)\i(전체) 또는 V(j)(전체 포함)로 계산 분기, 저장 메시지는 k+1개로 축소.
5. Version-B: CNU 연산 범위 자체를 Vk(j)로 제한(식 4,5)하여 저장뿐 아니라 연산 복잡도까지 k개 수준으로 축소.
6. Offset 버전(식 7~9): 실험적으로 min-sum/version-B가 LLR을 과대추정, version-A는 과소추정하는 경향(Fig.1)을 보정하기 위해 α, β offset을 추가.
7. 최적 offset 선택 시(Fig.5) float LLR-BP 대비 0.1dB 이내로 성능 근접, 오히려 초기 수렴은 더 빠름.
8. offset 값은 density evolution 또는 시뮬레이션으로 탐색(실험에서는 시뮬레이션 기반 탐색만 수행).
9. 고정소수점 시뮬(Fig.6)에서 error floor 관찰, dynamic range가 클수록 floor 낮아짐; (7493,6096) rate-0.8 코드에서 check node memory 71% 절감, 전체 메모리 51.53% 절감(TABLE I).
10. 파이프라인/직렬 check node 아키텍처(Fig.7,8) 제안했으나 gate count·throughput 등 정량 HW 합성 결과는 없음 — 시뮬레이션 수준 검증에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Partial-min CNU (k-min 부분집합 기반 min-sum 변형) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 대비 정확도-메모리 트레이드오프를 조절하는 대안 CNU 로직으로 참고 가능 |
| check node 메시지 메모리 절감 구조 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 저장 메시지 수 축소 아이디어는 HW 메모리 최적화 관점에서 참고 가치 있음 |
| offset 보정(α, β) | `ecc_data.h` `PARAM_LLR` | 기존 normalized/offset min-sum과 유사한 보정 방식, Prime ECC의 LLR 테이블 튜닝과 유사 개념 |
| 파이프라인 직렬/병렬 check node 아키텍처 | 디코더 HW 모델 (z=32, HCU, GT) | 구체적 gate count 없어 직접 대응은 어려움 |

> 적용 가치: 중간 — min-sum 대비 메모리 절감 아이디어는 참고 가치가 있으나, offset min-sum 계열은 Prime ECC가 이미 유사 보유 기법(§3)과 개념적으로 중복되고 HW 합성 검증이 없어 실이식 판단엔 추가 검증 필요.
