### B. 알고리즘 요약

1. 대상은 finite-geometry(Euclidean/projective geometry 기반) LDPC 코드, 예시로 type-I 2-D (1023,781) EG-LDPC, rate 0.763, cyclic/quasi-cyclic 구조로 선형시간 인코딩 가능.
2. 기존 bit-flipping(BF)/weighted-BF는 실수 연산이 필요하거나 신뢰도 추정이 부정확하고, 무한루프(loop-infinite)에 빠지면 이를 검출·회피하는 기존 방법은 O(n²) 복잡도로 HW 구현이 어렵다는 문제.
3. 채널은 BPSK+AWGN, 수신 신호는 hard-decision `Z = HT`로 syndrome 계산(`S = ZHᵀ`).
4. 핵심 기법 1단: 각 변수 `v`에 대해 unsatisfied constraint 개수 `U(v)`와, 최댓값 기반 파라미터 `T(v) = Σ R(c')·Umax(c'/v)`를 정의해 신뢰도를 정량화(`R(c)=1`이면 satisfied, `-1`이면 unsatisfied).
5. `U(v)`가 같은 변수들 사이에서는 `T(v)`가 작을수록 더 unreliable하다고 판단 — 두 파라미터를 큐(`Q0...Qd`)로 관리해 최소신뢰 비트를 찾아 flip.
6. 핵심 기법 2단(hybrid): 먼저 BF를 적은 고정 iteration(예: 5~10회)만 수행해 찾기 쉬운 오류를 정정하고, 그 출력을 majority-logic(MLG) 디코더로 최종 처리 — loop-infinite에 빠지지 않도록 BF 단계를 짧게 제한.
7. 부수 트릭: 실수(real) 연산이 전혀 없는 정수 기반 알고리즘이라 저복잡도 HW 구현에 적합하다고 주장(구체 HW 설계는 없음).
8. 별도의 학습/최적화 절차 없음(고정 파라미터/휴리스틱 규칙).
9. 결과: BER 10⁻⁴에서 WBF 대비 0.3dB 개선, SPA(sum-product) 대비 1dB 이내로 근접; hybrid BF iteration을 10회로 제한할 때 저SNR(2~3dB)에서 최적 성능.
10. 한계: finite-geometry LDPC(girth-6, 대형 column/row weight) 특유의 좋은 수렴성에 의존한 설계이며 일반 QC-LDPC로의 이식성은 본문에서 다루지 않음; HW 합성/게이트카운트/처리량 등 실제 구현 검증은 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| hard-decision BF 디코딩 단계 | `decoder.cpp` `VN_Cal_HD()` | 유사한 HD 경로가 있으나 본 논문은 min-sum이 아닌 별도 BF 알고리즘 |
| U(v)/T(v) 신뢰도(최소신뢰 비트 탐색) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | min-sum LLR 기반 신뢰도와 다른 방식(unsatisfied-check 카운트 기반)이라 대체 관점의 참고 가능 |
| MLG(majority-logic) 최종 디코딩 단계 | 대응 없음 | Prime ECC는 min-sum 기반이며 MLG hard 디코더 구조는 보유하지 않음 |
| 조기종료(전체 syndrome=0 시 정지) | `partial_CRC.cpp`/`full_CRC.cpp` | 개념적으로 유사(early termination)하나 구현 방식(CRC vs syndrome check)은 다름 |
```

적용 가치: 낮음 — hard-1bit 전용이고 Prime ECC는 이미 soft min-sum + partial CRC 조기종료를 보유해 중복 기여이며, MLG/finite-geometry 특유 구조는 Prime ECC의 QC-LDPC와 직접 대응되지 않음.
