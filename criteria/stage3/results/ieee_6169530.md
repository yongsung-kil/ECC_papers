### B. 알고리즘 요약

1. 대상 부호는 `4000×8000`, `10000×20000` (CN수×BN수, N=8000/20000-bit) binary (N,K) LDPC, rate=0.5, DVB-S2/WiMAX급 통신용 부호.
2. 풀려는 문제: LDPC 복호는 연산집약·불규칙 메모리 접근이 커서 many-core 병렬화가 필요하나, 캐시 일관성 없는 대규모 many-core에서 스케일링이 어렵다.
3. 채널/모델: BPSK + AWGN(`yi = xi + ni`, 평균0 분산`σ²`), 확률영역 Sum-Product Algorithm(SPA/belief propagation).
4. 핵심 기법: SPA를 Horizontal(Kernel1: `CNm→BNn`, `rmn(0)=½+½∏(1−2qnm(1))`) / Vertical(Kernel2: `BNn→CNm`, `qnm=knm·pn·∏rmn`) 두 커널로 나눠 각 코어가 코드워드 일부를 분담.
5. 식 의미: 반복 message-passing으로 posterior `Qn(1)`을 갱신, `ĉHᵀ=0` 또는 최대반복 `I` 도달 시 종료(hard decision).
6. 확장: 3가지 병렬모델 - Distributed(private off-chip + RCCE 메시지), Shared(off-chip 공유메모리), Multi-codeword(코어당 독립 코드워드).
7. 구현 디테일: RCCE API의 bcast/send-recv/fastbcast(MPB 직접접근) 3종 브로드캐스트, HBN/HCN 두 데이터스트림으로 H 저장.
8. 학습/최적화: 없음 (알고리즘 자체 변경 없음).
9. 결과: Distributed는 통신시간이 95%+로 스케일 실패, Shared는 uncacheable 오버헤드로 4코어에서 포화, Multi-codeword는 거의 선형(32코어 상대 throughput 27.8).
10. 한계: SW 병렬화 연구로 BER/정정능력 개선 없음, HW 미설계, NAND ECC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA 복호 알고리즘 (Kernel1/Kernel2) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 낮음 - Prime ECC는 full-tanh SPA가 아닌 min-sum 사용, 논문은 부동소수점 SPA를 그대로 병렬화 |
| Horizontal/Vertical processing | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 낮음 - 연산 자체가 아니라 many-core 매핑이 초점 |
| 병렬 실행 모델 (분산/공유/멀티코드워드) | 대응 없음 | Prime ECC는 z=32 row 병렬 HW 모델, 범용 CPU many-core 병렬화와 무관 |
| 메시지/메모리 라우팅 (MPB, off-chip) | 대응 없음 | SCC 특정 메모리 계층, NAND 컨트롤러 HW와 무관 |

적용 가치: **낮음** - 알고리즘·부호 개선 없이 SPA를 특정 실험 many-core 프로세서에 병렬 매핑한 연구라 NAND binary QC-LDPC HW 디코더에 이식할 요소가 없음.
