### B. 알고리즘 요약

1. NASA용으로 설계된 (8176,7156) EG-based regular QC-LDPC 코드(rate=0.875, column weight 4, row weight 32), `H`는 2x16 배열의 511x511 circulant submatrix(각 column/row weight 2)로 구성.
2. 기존 Sum-Product Algorithm(SPA)은 variable-to-check와 check-to-variable 갱신 단계 간 연산량 불균형으로 데이터패스가 비대칭적이라 critical path가 늘어나는 문제.
3. 채널/디코딩 모델은 표준 SPA를 재정식화한 min-sum 유사 형태: `Rcv = ∏sign(Lcv)·φ(Σφ(Lcv))`, `Lv = Σ(sign(Rmv)φ(Rmv))-φ2r_v`로 check/variable node 갱신, tentative decoding 포함.
4. 핵심 기법 1단: 기존 1개 CPU/VPU가 매 클록 1 row/column씩 처리하던 구조를 2배 병렬화(`2x16=32 CPU`, `2x2=4 VPU`)해 submatrix당 4개 soft message를 동시 접근.
5. 핵심 식의 의미: circulant가 2개의 superimposed cyclically-shifted identity로 구성되므로 even/odd row·column 인덱스로 메모리를 분리(MEM_E/MEM_0)하면 충돌 없이 병렬 접근 가능.
6. 확장 기법: memory partitioning + 3가지 data switching scheme(odd/even/zero shifting offset별)으로 CPU_E/CPU_0, VPU_0/VPU_1에 올바른 데이터를 라우팅, anti-overlap state로 파이프라인 충돌 회피.
7. 부수 트릭: 2-step non-uniform quantization — 균등양자화(7:4)로 계산된 soft message를 6-bit 비균등 양자화로 변환해 메모리에 저장(연산부는 균등양자화 유지), φ(x) 값이 x≥4.875에서 모두 0인 특성 활용.
8. 별도 학습/최적화 절차 없음(HW 아키텍처+양자화 설계); retiming으로 controller critical path 단축.
9. 결과: 15 iteration 기준 최대 (information) decoding throughput 172Mbps(non-uniform 6-bit), clock 192.6MHz, Virtex II-6000에서 27460 slices/81%, Block RAM 88% 사용; 기존 유사 플랫폼 대비 iteration당 처리량 2배 이상.
10. 한계: FPGA 합성 결과만 제시(ASIC 미검증), 정정성능(BER/FER) 개선은 목표가 아니며 double-precision 대비 성능 유지만 확인(그림 6/7 수치는 본문 서술 없어 미상); 이중병렬(2x)로만 제한, 고차 병렬화는 향후과제로 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN/VN 처리 유닛(CPU/VPU), min-sum 유사 재정식화 SPA | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | check/variable node 갱신 로직 구조가 유사하며 이미 min-sum 방식을 보유해 신규성은 낮음 |
| circulant even/odd 메모리 뱅킹 + 데이터 스위칭 | `decoder.cpp` `VN_Cal_HD()` 등 VNU 관련, HCU 처리 로직 | 병렬 접근 충돌 회피를 위한 메모리 파티셔닝 아이디어로 참고 가치 있음 |
| 6-bit non-uniform quantization(soft message 메모리 절감) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 테이블 기반 양자화와 목적이 유사, 비균등 양자화 변환 기법 이식 검토 가치 |
| 이중병렬(2x) 확장 partially-parallel 아키텍처 | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC는 z_sb=32 고정 row 병렬, 이 논문의 submatrix 내부 2x 확장 병렬화는 구조가 달라 중 난이도 |

> 적용 가치: 중간 | non-uniform quantization을 통한 soft message 메모리 절감과 circulant 서브뱅킹 데이터 스위칭 아이디어는 참고 가치가 있으나, CN/VN 코어 알고리즘 자체는 Prime ECC가 이미 보유한 min-sum 계열과 중복도가 높음.
