### B. 알고리즘 요약

1. 대상: MLC/TLC NAND flash의 BICM(bit-interleaved coded modulation) 구조, `(j,k)`-regular binary LDPC, hard-decision decoding.
2. 문제: MLC에서 MSB/LSB 페이지의 채널 오류율이 다르고(`b1<b2`) 오류의 96%가 single-bit인데, 무작위 bit 배치는 이 비대칭을 활용하지 못함.
3. 모델: MSB/LSB 초기 오류확률 `b1`,`b2`를 각각 부여, 4-PAM Gray map + AWGN(`b1≈0.5Q(1/σ)`, `b2≈Q(1/σ)`).
4. 핵심기법 1단: check node를 `T(α,β)`(α=MSB이웃, β=LSB이웃) 타입으로 정의하고, 타입 비율 `g`를 두어 Gallager A/B DE로 `b2` threshold 계산.
5. 핵심식: `q(t)=(1-(1-2p_M)^x1 (1-2p_L)^x2)/2` — check→variable 오류확률이 MSB/LSB 이웃 수에 지수적으로 의존.
6. 핵심기법 2단: 2타입(또는 3~4타입) 조합에서 `g=1/2`, `T(1,k-1)`/`T(k-1,1)` 극불균형이 최적 threshold; MSB 이웃 다수인 구성이 BICM 평균 대비 우수.
7. 구현 디테일: non-binary(GF(4)) LDPC의 binary image graph를 이용해 GF(4) edge label로 원하는 binary check 타입/차수분포를 유도, binary decoder로 복호.
8. 최적화: Table III처럼 edge label set별 MSB/LSB 차수분포·check 타입을 나열해 Gallager A/B threshold 비교로 우수 label 선정.
9. 결과: (3,6)-regular에서 `T(1,5)/T(5,1)` σ threshold 0.6189(BICM 0.6172), edge label `{1,1,1,A2,A2,A2}`가 binary image에서 최고 성능.
10. 한계: DE threshold(이론)뿐, HW/BER 시뮬 없음. binary image가 결국 non-binary GF(4) 부호이며, cycle-free 가정 근사.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MSB/LSB 비대칭 check 연결 부호구조 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 열배치에 MSB/LSB 편향 반영 시도 시 닿으나 재설계 부담 큼 |
| GF(4) binary image / non-binary 구성 | 대응 없음 | Prime ECC는 binary 전용 |
| Gallager A/B hard-decision DE 분석 | 대응 없음 | Prime ECC는 min-sum soft/HD, Gallager A/B 미사용 |
| soft-read 없이 hard-1bit 판정 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()` | HD 채널만 부분 대응, 핵심 기여와 무관 |

적용 가치: **낮음** — 핵심 산출물이 GF(4) non-binary 부호이고 binary QC-LDPC와 부호구조가 달라 이식 시 H-matrix 전면 재설계·재검증이 필요하다.
