### B. 알고리즘 요약

1. 대상은 400Gbit/s 이상 광통신 soft-decision FEC용 SC-LDPC(=LDPC convolutional), parity-check `H_SC` 크기 `(L+μ)m × Ln`, chunk 길이 n.
2. 문제: SC-LDPC가 이론상 capacity-achieving이나 실제 제약(작은 memory μ) 하에서 capacity에 접근하려 할 때 나타나는 pitfall(특히 burst error) 규명.
3. 부호 rate `R = 1 − (1+μ/L)(m/n)`; 큰 L이 고rate·고SE, 작은 μ가 rate loss·HW resource 최소화 - 본 논문은 unit-memory `μ=1` 고정.
4. 핵심 설계: H0, H1의 "1" 분포로 성능 제어 - uniform coupling vs non-uniform coupling; variable-regular(`dv,i = dv`) 채택.
5. Density evolution으로 wave-like threshold와 macro-convergence threshold 분리 분석 - `dv=5`에서 Shannon 한계에 0.1dB 미만 접근하나 두 threshold 사이 critical region이 0.8dB로 확대.
6. windowed decoder(window w, chunk (w+μ)n): window 내 I iteration 후 한 chunk shift, 최좌측 chunk가 잔여오류 0이 되도록; wave보다 빨리 shift하면 decoding stall.
7. 구현 디테일: proto-matrix B0(1/0 엔트리)·B1(1/2 엔트리)을 480×480 permutation(30×30 순환 shift identity 중첩)으로 lifting, chunk size n=4800; 비교용 uniform dv=4는 960×960.
8. 최적화/평가: FPGA 기반 windowed decoder emulator로 대량 error event 통계 확보(dv=5, w=9, I=2/window), stall 발생 전 성공 codeword 수 Z의 pdf 추정, post-FEC BER `≈ (1/2) p_pre-FEC/ν`.
9. 결과: dv=5, L=99, w=13에서 25% OH SC-LDPC가 block LDPC(dv=3,4) 대비 우수; 예 Es/N0=1.96dB, L=24에서 평균 4.6·10^10 error-free bit 후 평균 길이 57600 burst 발생.
10. 한계: HW는 FPGA emulator(ASIC/실측 아님), 광통신 특화(offset/scaled min-sum, GMI 기반), SC-LDPC·windowed decoding 전용으로 burst error 결론은 block 코드에 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| unit-memory SC-LDPC 부호설계(H0/H1 coupling, protomatrix lifting) | `ecc_top.cpp` `Load_PCM()` (H-matrix) | Prime ECC는 고정 block QC-LDPC라 SC/convolutional 구조 부호설계 접점 없음(이식 난이도 높음) |
| offset/scaled min-sum 복호 | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum 자체는 보유; offset/scaled factor는 광통신 특화, 별도 이득 불분명 |
| windowed decoder / wave-like 수렴 | 대응 없음 | Prime ECC는 block decoder, windowed/sliding 구조 없음 |
| FPGA emulator 기반 rare-event 통계 | 대응 없음 | Prime ECC는 bit-exact C++ 시뮬 |

적용 가치: **낮음** - 광통신용 SC-LDPC 부호설계와 windowed decoder burst error 분석이 핵심이라 block QC-LDPC 기반 Prime ECC에 떼어 쓸 접점이 거의 없음.
