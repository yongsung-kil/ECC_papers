### B. 알고리즘 요약

1. 대상 시스템은 BICM-ID(bit-interleaved coded modulation with iterative decoding): irregular repetition code(outer, rate `r_out`) → 인터리버 Π → rate-2 inner encoder → rate-1 recursive encoder → M-ary FQAM 변조의 직렬 결합 구조.
2. 문제의식: 하드웨어 친화적인 QC(quasi-cyclic) 구조 인터리버를 쓰면 저-weight 메시지 벡터가 저-weight 코드워드 b를 만들어 error-floor가 크게 악화됨(반면 random 인터리버는 문제 없음).
3. 채널/모델: AWGN 채널, 16-ary FQAM(4-FSK+4-QAM) 변조, 인터리버는 `µ×ν` 순열행렬 Π(각 `Q×Q` 서브블록이 circulant permutation matrix 또는 zero matrix)로 정의.
4. 핵심 기법 1단: Proposition 1/2로 QC 인터리버 사용 시 weight-1/weight-2 메시지가 만드는 코드워드 b의 최소 weight를 `dv,min` 또는 인접 CPM 열간 거리 `δ=min|j1-j2|`로 해석적으로 유도, weight-2 codeword 발생 확률이 `(m-1)²/m²`임을 도출.
5. 이 식의 의미: QC 구조에서 저-weight 코드워드가 필연적으로 자주 발생하는 이유(CPM 열 간 정렬 구조)를 수식으로 정량화해 error-floor 원인을 특정.
6. 핵심 기법 2단(제안): quasi-cyclic double interleaver(QCDI) — Π의 row를 `j'=µ_Q·γ+φ` (φ=⌊j/Q⌋, γ=j-Qφ, 즉 j를 Q로 나눈 몫/나머지) 순서로 재배치해서 접근. QC 구조(각 서브행렬이 circulant shift)는 그대로 유지하면서 low-weight 코드워드 생성 패턴만 깨뜨림.
7. 부수 트릭: 순열 연산 `c⊛Π`를 열 방향 배치 후 행 방향 판독으로 정의(Definition 1)해 HW 친화적 매핑 유지, row read-order만 바꾸므로 회로/메모리 구조 변경 최소.
8. 최적화/학습 절차는 없음(closed-form row-permutation 규칙, GA/DE/NN 없음). 외부 코드 설계엔 PEG(progressive edge growth) 알고리즘을 비교군(random interleaver) 생성에만 사용.
9. 결과: k=5040, rate=0.5, circulant size Q=120 설정에서 QC 인터리버는 저-message-weight 오류 이벤트가 random 대비 크게 많고 높은 error-floor를 보였으나(수치 그래프에만 존재), QCDI 적용 시 random 인터리버와 거의 동일한 WER/BER 성능으로 개선(정성적 확인, 정확 dB/BER 개선폭은 그래프에만 있어 본문 서술 수치 없음).
10. 한계: 순수 알고리즘/시뮬레이션 논문으로 HW 합성/구현 결과 없음, LDPC 부호가 아닌 BICM-ID(FQAM 변조 결합) 전용 구조, 특정 degree distribution·circulant size 예제 하나로만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC 인터리버 저-weight codeword 원인 분석(Proposition 1/2) | 대응 없음 | BICM 인터리버 대상 분석이며 LDPC PCM 구조가 아님 |
| QCDI(row 순서 재배치로 QC 구조 유지+error-floor 개선) | `ecc_top.cpp` `Load_PCM()` | H-matrix/circulant 배치 설계에 아이디어 차원 참고 가능하나 실제 LDPC PCM 검증 안 됨, 이식 난이도 높음 |
| PEG 기반 random 비교군 생성 | 대응 없음 | 비교 목적일 뿐 제안 기법 아님 |

적용 가치: 낮음 — BICM-ID 인터리버 설계 문제이며 LDPC parity-check matrix가 아니라 Prime ECC 모듈에 직접 대응되는 요소가 없음. QC 구조의 row-순서 조정으로 저-weight 구조를 피한다는 아이디어만 참고 가능한 수준.
