### B. 알고리즘 요약

1. 대상 채널은 1553B 차량용 버스(이진 삭제 채널 BEC), QC-LDPC 코드를 예시로 사용하나 구체적 rate/length는 본문에 명시되지 않음(코드워드 샘플 1440개=16codeword×90 node unit이라는 실험 설정만 서술).
2. 풀려는 문제: LDPC 복호/생성 시 데이터 저장량이 많고 증분 리던던시 정량 기준이 없어 기존 표준(2G 등)에서 채택되지 못한 한계를 신경망+병렬처리로 보완하려 함.
3. 핵심 가정: BEC 채널에서 수신 비트 `yi∈{0,1,E}`(E=삭제 심볼), `pr(ci=b|yi)` 확률모델을 sum-product 디코딩의 초기값으로 사용.
4. 1단 기법(baseline): 표준 sum-product(BP) 알고리즘을 OpenMP 기반 다중코어 병렬 구조(x86)로 실행해 check node(rmn)·bit node(qmn) 갱신을 병렬화.
5. 핵심 식: check/bit node 업데이트 식 `(4)~(7)`은 OCR 손상으로 정확한 수식 재현 불가 — 표준 BP의 통계적 신뢰도 전파 형태로 서술됨.
6. 2단 기법(제안): convolutional layer + output buffer layer로 구성된 신경망 디코더, 뉴런 상태 갱신식 `ui(t+1)=sgn(Σ ωij·uj(t) - θ)`(8)로 학습된 가중치 `ωij`와 임계값 `θ`를 반복 갱신.
7. 학습 절차: 코드워드 샘플 `X1...XN`을 학습셋으로 사용, `ωij`는 해밍거리 기반 학습 규칙(9)로 결정, buffer layer 출력이 정확히 1개 뉴런만 non-zero일 때 복호 성공으로 판정.
8. 부수 트릭: DSP 프로세서 + Qt/Eigen 라이브러리로 소프트웨어 플랫폼 구성(GPU 등 전용 HW 설계 없음), soft-decision 확률정보를 입력단에 추가하면 iteration 감소 가능하다고 언급(구체 수치 없음).
9. 결과: 1553B 버스의 12종 고장타입 분류에서 SNR 증가에 따른 오류율 감소 경향이 sum-product와 유사, 적은 하드웨어 자원으로 긴 코드워드에서도 양호한 정확도라고 주장(구체 BER/FER 수치는 그림에만 있고 본문 서술 없음).
10. 한계: 1kbit 초과 시 convolutional layer 구조가 복잡해지고 메모리 소요가 커짐, HW 합성/게이트카운트 없이 소프트웨어 시뮬레이션 수준, NAND/저장매체 실험 전무.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 신경망 기반 디코더(convolutional+buffer layer) | 대응 없음 | Prime ECC는 min-sum 기반 bit-exact HW 시뮬레이터로 NN 전면 대체 디코더는 프로파일 §5 기준 이식 대상 아님 |
| sum-product(BP) 병렬 디코딩 baseline | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 표준 BP는 이미 Min-Sum으로 대체 보유, 중복 기여로 참고 가치 낮음 |
| QC-LDPC 부호구조(순환치환행렬 기반) | `ecc_top.cpp` `Load_PCM()` | 부호 형태만 QC-LDPC로 언급되나 구체 파라미터(rate/length) 부재로 실질 비교 불가 |

> 적용 가치: 낮음 — NN 기반 디코더는 Prime ECC의 min-sum bit-exact HW 구조와 이질적이며 대응 모듈이 없고, BEC/차량버스 채널로 NAND read 채널(AWGN/RBER)과도 무관, 수치적 근거(rate/length/BER)가 본문에 부재해 이식 판단 자체가 어려움.
