### B. 알고리즘 요약

1. 대상 부호: WiMAX(802.16e) QC-LDPC(base matrix + 순환 shift identity), codeword length 2304, rate R=1/2·2/3A·2/3B·3/4A·3/4B·5/6 전부 지원, 96x96 permutation, R=1/2는 12 layer x 96 row.
2. 문제: bit-node message finite wordlength가 디코더 면적·메모리·라우팅·연산 유닛 복잡도를 크게 좌우 → wordlength 축소가 목표.
3. 채널/모델: AWGN + BPSK, LLR `(4)`로 초기화.
4. 핵심 기법 1(디코더): normalized BP-based(min-sum에 곱셈 correction factor `λ<1`) `(5)`. offset(가산) `(6)`보다 성능 우수하나 곱셈 필요.
5. 곱셈 회피: `λ=0.75` 또는 `0.875`면 곱셈을 shift+add로 단순화(offset과 동일 복잡도). 단 0.75는 R=1/2에서 BER 10^-6·FER 10^-3 error-floor 발생 → `0.875` 선택.
6. 핵심 기법 2(스케줄): layered decoding `(7)(8)(9)` — CN 갱신 즉시 연결 VN 갱신, 2-phase BP 대비 latency 약 50% 감소, 성능 손실 없음.
7. wordlength 설계: `(9)`에서 max column weight 6(R=1/2 등)이라 `Q'_j`는 최대 7개 합 → 원칙상 입력(5bit) + 3bit = 8bit 필요.
8. 핵심 관찰: VN 메시지 통계분포(Fig.1)상 대부분 (-63,63) 범위 → 7bit로 충분. 이유는 (a) 합산 항의 부호 상쇄, (b) 각 row 연산마다 magnitude가 1/8(=0.875 factor) 감소.
9. 결과(시뮬, 100,000 codeword): input LLR 5bit(5:2), bit-node 7bit, 0.875 factor로 6개 rate 모두 BER 10^-6·FER 10^-5까지 error-floor 없음; 평균 iteration 5/4.9/4.74. 8bit 대비 저SNR 미세차, 고SNR 무차. [12] 대비 1bit 절감으로 면적 최소 10% 절약, R=1/2 성능 향상·나머지 <0.2dB 손실.
10. 한계: 시뮬만(RTL 합성·gatecount·throughput·실측 없음), factor·wordlength가 WiMAX 특정 rate/길이 통계에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized min-sum(0.875 배수 shift-add) CN 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 높음: Prime ECC는 scaling factor 코드상 미명시 → 0.875 shift-add 도입 가치 |
| layered(row 즉시갱신) decoding schedule | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 중: Dual-Update(even/odd 교대)와 비교, layered 도입 여지 |
| VN 메시지 분포 통계 기반 wordlength 절감 | `decoder.cpp` `VN_Cal_HD/2SD/3SD`, `ecc_data.h` `PARAM_LLR` | 높음: magnitude 양자화 테이블 bit-width 최적화에 직접 참고 |
| 입력 LLR 5bit(5:2) 양자화 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 중: soft-read LLR 양자화 폭 조정 참고(NAND read 비용은 별도) |
| QC-LDPC base matrix + 순환 shift 구조 | `ecc_top.cpp` `Load_PCM()` | 중: 부호구조 동종(QC-LDPC)이라 이식 정합 용이 |

적용 가치: **중간** — QC-LDPC용 normalized min-sum(0.875 shift-add)와 VN 분포 기반 1bit wordlength 절감은 Prime ECC 양자화 테이블 폭 최적화에 저비용으로 참고 가능하나, 시뮬뿐이고 정정능력 자체 향상은 없음.
