### B. 알고리즘 요약 (10줄 내외)

1. 시스템은 regular (96,48) LDPC(rate 1/2, check node degree k=6) + AWGN 채널을 대상으로 소프트 정보를 오실레이터 초기 위상으로 인코딩.
2. 기존 오실레이터/이징 컴퓨팅(ring oscillator 기반)은 2/3-body 결합, 아날로그 저항성 커플링, binary 초기화만 지원해 LDPC 같은 복잡한 조합최적화(soft decoding)에 부적합했다는 문제를 다룸.
3. 핵심 가정은 LDPC 디코딩을 에너지 함수 `E_LDPC = Σ_i (1 - Π_{j,Hij≠0} s_j)`의 최소화 문제로 보고, 이를 결합 오실레이터의 물리적 이완(relaxation) 과정으로 직접 구현할 수 있다는 것.
4. 핵심 기법은 H-matrix의 6-input XOR 게이트(parity check)로 오실레이터를 그룹핑하는 6-body 상호작용 매핑이며, twin-capacitor relaxation oscillator(RXO) 코어가 계산 노드.
5. 각 오실레이터의 순간 주파수가 연관된 parity check 위반 개수 합에 따라 변조되어(FDAC 4-level feedback), 시스템 전체가 연속시간으로 최소 에너지 상태를 향해 수렴 — 모든 parity check가 만족되면(위상차 0/180도) 수렴 완료.
6. 확장으로 3-bit soft DAC(SDAC)를 통해 수신 신뢰도(soft information)를 오실레이터의 초기 위상으로 정밀 설정(soft decoding), binary 초기화 대비 수렴 정확도·처리량을 크게 개선.
7. 부수 트릭으로 digital crossbar(twin crossbar architecture)를 이용해 임의의 H-matrix 결선을 구성 가능하게 했고, DLL 기반 6-phase 샘플링 시스템으로 비동기 배열 상태를 노이즈 없이 읽어냄.
8. 학습/최적화 절차 없음(물리적 아날로그 이완 과정 자체가 최적화 solver, 별도 학습 불필요).
9. 결과: 28nm CMOS, 0.21mm² 코어면적, FER 1.38×10⁻⁵ / BER 1.25×10⁻⁶ @7dB SNR, 에너지효율 5.26pJ/bit(최신 BP 디코더 대비 1.5~11배 우수), TTS(time-to-solution) 89ns, 수렴율 99.90%, 저SNR 영역에서 BP 대비 FER/BER 1~2 자릿수 우수.
10. 한계: 검증 코드가 (96,48)로 매우 짧고, one-shot 아날로그 컴퓨팅이라 iteration 개념 자체가 없어 디지털 min-sum과 직접 비교 어려우며, 6-body XOR 크로스바가 코드 길이·check degree 증가 시 스케일링 방식이 불명확.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 오실레이터 기반 continuous-time 디코딩 전체 | 대응 없음 | Prime ECC는 디지털 bit-exact C++ HW 시뮬레이터(min-sum message-passing)로, 아날로그 물리 이완 컴퓨팅과 패러다임이 완전히 다름 |
| soft information 초기 위상 인코딩 | `channel.cpp` `Set_LLR_Th()` | soft-read 개념적 대응은 되나 구현 방식(전압 DAC vs LLR 양자화 테이블)이 이질적 |
| H-matrix → XOR 크로스바 매핑 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드라는 개념만 유사, 아날로그 크로스바 라우팅은 대응 없음 |
| 6-body parity check 상호작용 | 대응 없음 | Prime ECC의 CN 연산(min-sum)과 무관한 물리적 에너지 최소화 방식 |

> 적용 가치: 낮음 — 아날로그 오실레이터 기반 컴퓨팅은 Prime ECC의 디지털 HW(RTL) bit-exact 시뮬레이터 구조와 근본적으로 다른 컴퓨팅 패러다임이며, 검증 코드도 (96,48)로 짧아 NAND용 고rate QC-LDPC로의 이식 가능성 판단 근거가 부족함.
