### B. 알고리즘 요약

1. 대상: IEEE 802.11n WiMAX/WiFi QC-LDPC, FEC rate-1/2, H-matrix는 81×81 circulant permutation `pi`(항등행렬을 좌로 i shift) 블록 구성. 본문 예시 rate 1/2·6966 connected node.
2. 문제: LDPC 디코더의 고throughput 구현이 필요하나 ASIC/FPGA는 배포주기 길고 기능 고정 → 유연·확장·저비용 소프트웨어 대안으로 GPU 탐색.
3. 알고리즘: Min-Sum Algorithm(MSA), belief propagation의 CN 처리를 min 근사로 단순화. horizontal(CN->BN)·vertical(BN->CN) 두 처리 블록.
4. CN 갱신 `Lr_mn = min|Lq|·∏sign(Lq)` (식1), BN 갱신 `Lq_nm = Lp_n + Σ Lr_mn` (식2), 최종 hard decision `Q_n = Lp_n + Σ Lr_mn` (식3).
5. GPU 매핑: CUDA, 다수 codeword를 global memory에 동시 전송(kernel 초기화 비용 상쇄), thread당 단일 codeword 생애주기(Algorithm 1).
6. 최적화1: H-matrix를 CCS(Compressed Column Storage) 4벡터(rowDegree/rowIndex/columnDegree/columnIndex)로 압축(rate-1/2에서 원소의 0.67%), 읽기전용 constant memory 배치.
7. 최적화2: 입력을 coalesced로 shared memory에 적재, column/row 처리를 contiguous read/write로 수행, HCN 배열로 접근 패턴 관리.
8. 학습/최적화: 없음(별도 offline 최적화 없음). 조기종료 미사용(공정 비교 위해).
9. 결과: NVIDIA TITAN X(Pascal, 3584 core)에서 peak 1.276Gb/s, Matlab MSA와 거의 동일 BER, 5 iteration이면 저 Eb/No에서 충분. BCH concatenation 시 300Mbps+.
10. 한계: 알고리즘 기여 없음(표준 MSA), GPU 소프트웨어 가속뿐, NAND read/soft-quant 무관, error-floor/waterfall 개선 없음.
