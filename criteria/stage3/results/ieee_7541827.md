> 총평: LDPC 디코더의 정수 lookup-table 기반 노드연산(soft-info 양자화 최적화) 아이디어는 NAND LLR 설계에 참고 가치가 있으나, 논문 본체는 무선 fading 채널 receiver 전체 체인 설계이고 디코더 자체는 타 논문 재사용.

### B. 알고리즘 요약

1. 시스템은 Rayleigh flat-fading 채널 위 length `Nv=104` (dv=3, dc=5) regular LDPC 코드, rate≈0.4, BPSK 변조.
2. 기존 belief propagation 수신기는 LLR(실수) 연산이 필요해 고정밀 신호처리 하드웨어가 요구되는 문제를 해결하려 함.
3. Information Bottleneck(IB) 방법으로 관측 변수 `Y`를 압축변수 `T`로 매핑하되 관련변수 `X`와의 상호정보량 `I(T;X)`을 최대 보존(식 (1)) — 압축은 결정론적 lookup table `t=f(y)`로 구현.
4. 핵심 기법은 "Information Bottleneck Graph": factor graph의 각 노드(ADC, 채널추정, 검출, 복호)를 IB 방법으로 설계된 압축 매핑(trapezoid 기호)으로 재구성.
5. ADC는 `f^q: R -> T^q` 로 연속값을 `|T^q|`개의 정수로 매핑(양자화), 채널추정은 `p(ĥ_r|r_r^p)` lookup table, 검출은 `p(d_r|v_r)` lookup table로 각각 관련정보를 최대 보존하도록 설계.
6. 최종 LDPC 복호는 `p(s,t)`를 이산 메모리없는 채널 출력으로 보고, 별도 논문([3][4][6])의 discrete density evolution 기반 discrete LDPC decoder(모든 노드연산이 lookup table)를 그대로 사용.
7. 부수 트릭: 모든 서브시스템에서 `|T|=|T^q|=16`(4bit 정수)로 통일해 word length 고정, 전체 시스템이 4bit 정수 연산만 사용.
8. 학습 절차는 sequential Information Bottleneck 알고리즘(Blahut-Arimoto 유사 반복)으로 각 lookup table을 오프라인에서 사전 생성.
9. 결과: 4bit 완전 이산 수신기가 double-precision belief propagation 대비 약 0.4dB 손실만 발생, 4bit고정소수점 MMSE채널추정+double BP보다도 우수, 전체 lookup table 메모리 37.9kB(LDPC decoder가 98.94% 차지).
10. 한계: LDPC 디코더 자체의 상세 설계는 본 논문 범위 밖(타 논문 인용), Rayleigh fading 채널 전용 설계이며 HW(RTL/ASIC) 미구현, 시뮬레이션만 수행.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 정수 lookup table 기반 노드연산(discrete decoder) | `decoder.cpp` `CNU_Update_New_Mag()`, `decoder.h` `Get_VNU_*` | Prime ECC의 min-sum 근사와 달리 IB 최적화된 lookup table 노드연산 개념은 LLR 테이블 설계에 참고 가능하나 코드 자체는 min-sum 유지 |
| soft-info 압축(ADC/양자화) 설계 | `channel.cpp` `Set_LLR_Th()` | 채널 양자화 threshold 최적화 관점에서 간접 참고 가능 |
| Rayleigh fading 채널추정 | 대응 없음 | NAND는 fading 채널 모델 아님 |

> 적용 가치: 중간 — LDPC 디코더 노드연산을 정보이론적으로 최적화된 lookup table로 대체하는 아이디어는 NAND LLR 설계에 참고할 만하나, 본 논문은 디코더 자체를 직접 설계하지 않고 무선 채널 전체 체인이 핵심이라 이식 대상이 제한적.
