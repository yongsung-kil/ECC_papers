### B. 알고리즘 요약

1. 대상은 QC-LDPC의 Tanner graph `G=(U∪W,E)`에서 짧은 cycle의 **위치를 찾아 제거(girth 증가)**하는 오프라인 알고리즘.
2. 기존 TBA[8]는 trellis `2k` 섹션에서 길이-`2k` cycle을 찾음(복잡도 `O(n·du^k·dw^k)`).
3. 핵심 아이디어: 길이-`2k` cycle = 길이-`k` 부분경로 2개의 concatenation(하나는 mirror image)이므로, **`k` 섹션 trellis에서 `PEP(i,j,k)`(partial elementary path) 쌍**만 찾으면 됨 → **ITBA**.
4. 두 path가 시작/끝 노드 외 공유노드 없으면 길이-`2k` cycle 하나. 복잡도 `O(n·du^⌊k/2⌋·dw^⌊k/2⌋)`로 지수부 절반 감소, 메모리도 대폭 감소.
5. cycle 식별 후 노드 **deactivation**(노드·인접 edge 제거)으로 그래프가 점점 작아져 실질 복잡도는 상한보다 낮아짐. 병렬 구현 가능하나 그 경우 deactivation 불가.
6. **QC-ITBA**: `H=[A_{s,r}]` (CPM/ZM 배열)의 준순환성으로 한 column-block당 대표 VN 하나만 조사 → 복잡도 `1/p`. row-block cyclic 구조면 추가 `1/L`.
7. **cycle-breaking = masking**: `Hmask=Z⊗H`로 특정 CPM을 ZM으로 치환해 edge 제거, QC 구조 유지. `Nmask,column/row` 상한으로 정규성·연결성 보존.
8. 알고리즘: cycle-per-CPM 계수 → 최다 cycle 통과 column-block·CPM 선택 → 해당 CPM masking → girth `g→g+2`, 반복 적용.
9. 결과(Example 2): (2640,1323) girth-6 부호를 masking으로 girth 8~16의 9개 부호 생성; C2~C4,C6는 기준부호 성능 동등, **C7~C9는 girth가 크지만 min거리/degree 악화로 매우 높은 error-floor**.
10. 한계: 부호설계 계층 도구(디코더·HW 무설계), AWGN MSA 50-iter 시뮬만, girth가 성능의 유일 지표 아님을 자체 경고.
