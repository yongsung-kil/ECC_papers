### B. 알고리즘 요약

1. 시스템: Binary Erasure Channel(BEC)에서 finite-length scaling을 적용한 SC-LDPC(공간결합 LDPC) 복호를 VLSI/VHDL로 구현. polar BP decoder 대안으로 LDPC 채택 주장.
2. 문제: polar BP decoder(및 CSFG/quarter-way scheduling 기반)는 iterative 특성상 iteration·latency·에너지가 선형 증가하고 VN 종류가 많아 설계 복잡도가 큼.
3. 채널 모델: BEC - `X(t)∈{0,1}`, 출력 `Y(t)∈{0,1,∆}`(∆=erasure), erasure 확률 `ε`, 균등입력에서 용량 `C=1-ε`. (soft LLR 아님, erasure 판정 기반)
4. 부호 구성1 - randomly constructed `(l,r,L)u` SC-LDPC: VN degree `l`, CN degree `r`, `L`개 결합, 대형 `L`에서 rate `≈1-l/r`.
5. 부호 구성2 - protograph-based `(l,r,L)` SC-LDPC: 예 `(3,6,3)`, base matrix B를 NxN permutation으로 lifting, 길이 `N=kLN`, rate `r=1-(L+l-1)/kL`.
6. 복호1 - Sequential Peeling Decoder(SPD): residual graph에서 degree-1 CN을 하나씩 골라 연결 VN을 결정·제거(식 (1),(2)의 메시지 규칙).
7. 복호2 - Parallel Peeling Decoder(PPD): 매 스텝 가용한 모든 degree-1 CN을 동시 해결(SPD와 결과 동일, stopping set 이론으로 등가 증명).
8. 복호3 - Belief-sum-product Propagation(BPD): VN을 채널 출력으로 초기화 후 반복 메시지 전달, XOR 게이트 수가 SPD/PPD보다 적다고 서술.
9. 결과: 8-bit 예제로 syndrome 기반 정정 시연(VHDL, ModelSim 6.4b, Xilinx ISE 13.2, Spartan-3E `xc3s500e`). Device util - slice 1/4656, 4-in LUT 1/9312, IOB 35/232(15%). throughput/BER 수치 없음.
10. 한계: BEC(erasure-only) 가정, polar/peeling/BP 개념 혼재, 텍스트가 심하게 손상(OCR/영문)되어 정량 근거 빈약, HW도 8-bit 예제 수준.
