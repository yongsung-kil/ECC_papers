### B. 알고리즘 요약

1. 대상: 통신표준(DVB, WiFi, WiMax)용 LDPC, 시뮬 코드는 PEG `(N=1008,M=504)`와 QC-EG `(N=8176,M=7156)`, 최대 iteration 100.
2. 문제: BP(SPA)는 near-Shannon 성능이나 복잡도가 커 자원제약 응용에 부담, BF는 단순하나 성능 열세 → 절충 필요.
3. 배경식: check node에서 두 LLR의 합은 `L(A2)=sign(L1)sign(L2)min(|L1|,|L2|)+g(L1,L2)`; MSA는 뒤 보정항 `g()`를 버림.
4. a-min*: min-sum과 SPA의 절충으로 box-plus 연산자 `min*(L1,L2)=min(|L1|,|L2|)+g(L1,L2)`를 사용, check당 두 개의 outgoing 메시지(최소신뢰도 노드용 1 + 나머지용 1)만 계산.
5. 핵심식 의미: check node update를 최소 두 신뢰도 값만으로 근사해 dr개 입력 전체 조합 계산을 회피.
6. generalized a-min*: (6)/(7) 두 update식의 box-plus 연산자를 `+1`, `+2`로 분리 허용 → a-min*는 `+1=+2`인 특수경우. 서로 다른 연산자로 복잡도-성능 조합 다양화.
7. a-min*-min: (7)의 `+2`를 단순 min(1회 memory access)로 치환 → check당 `(dr-2)`개 `+1` 연산 + 1 memory access만 필요, a-min*보다 저복잡도.
8. 결과: PEG/EG 코드에서 BER 성능이 BP·a-min*에 근접, MSA와 a-min* 중간 수준; 큰 SNR에서 BP 대비 수렴 열세이나 실용 BER은 충분(수치는 그림에만).
9. 한계: HW 미설계, gate count/throughput 없음, NAND 채널·soft-read 미고려, 성능 개선이 아닌 복잡도 절감 목적.
