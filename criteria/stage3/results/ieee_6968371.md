### B. 알고리즘 요약

1. 대상: grayscale n×n 이미지 전송 시스템, LDPC + BPSK, AWGN 채널; 512×512 예시에서 `G` 512×1024(rate 1/2), 데이터벡터 512b.
2. 문제: 이미지 크기별로 고정 행렬 사용 시 비효율 → 입력 이미지 크기에 맞춰 G/H를 동적 생성해 처리속도 개선.
3. 핵심기법 1(adaptive matrix): rows M / cols N / OnesPerColumn 입력으로 sparse H 랜덤 생성, uniform/non-uniform 분포 선택, reshape+sparse 함수 사용.
4. short-cycle 제거: 행 쌍 `H(i,:)⊕H(j,:)` XOR로 겹침 count 계산, 임계 초과 시 bit flip/rearrange로 length-4 cycle 제거.
5. 인코딩: 이미지 8n2 bit → [8n×n] 데이터벡터, `G`([n×2n], row-transform echelon로 H에서 유도)와 곱해 [8n×2n] 코드벡터 → BPSK.
6. 핵심기법 2(복호): syndrome `c'H'^T` 계산, 0이면 정상; 비0이면 Bit-Flipping(BF) - CN 다수결로 bit node flip 반복, 또는 BP(marginal prob `qij/Qi`) 확률적 복호.
7. BP 식: check-to-bit extrinsic `rji`, bit-to-check `qij(b)=Kij·Pi·∏rj'i(b)`, hard decision prob<0.5→0.
8. 결과(처리시간): 128×128~2048×2048 이미지별 행렬생성/인코딩 시간 표(예: 512×512 인코딩 2.5s), 인코딩 크기 표.
9. 결과(BER): 1-iteration 복호로 SNR별 PER/BER, `>10dB`에서 BER 0(오류 없음), 저SNR은 iteration 증가로 개선 가능 언급.
10. 한계: HW 미설계, 1-iteration BF 실험뿐, 구조화(QC) 아닌 랜덤 sparse 행렬, error-floor 언급 없음, 이미지 특화.
