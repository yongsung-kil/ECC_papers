### B. 알고리즘 요약

1. 대상은 irregular LDPC(일부 QC-LDPC 포함, DTMB 방송 표준용) min-sum 복호를 범용 멀티코어/GPU에서 소프트웨어로 병렬화하는 것.
2. 문제: VLSI HW 디코더는 유연성·확장성이 없어 코드 변경 시 아키텍처 전면 교체가 필요하므로 범용 PC 플랫폼 대안을 평가.
3. 채널은 AWGN, BPSK 변조 가정, 소프트 입력 `y`로부터 반복 message passing 수행.
4. 핵심 알고리즘 MSA: horizontal processing에서 CN→BN 메시지 `Rmn = (∏sgn(Qmn))·min(abs(Qmn))` (식 3), vertical processing에서 BN→CN `Qmn` 갱신 (식 4).
5. 두 processing 블록은 nested loop이며 loop-carried dependency가 없어 병렬화 가능(핵심 관찰).
6. Tanner graph edge를 선형 배열의 edge 구조체(bit/check node, Rmn, Qmn 포함)로 표현해 연속 메모리에 배치→병렬 접근.
7. OpenMP 방식: `#pragma omp parallel for`로 horizontal/vertical/tentative 단계를 x86 멀티코어에서 스레드 분할, 한 번에 코드워드 1개 복호.
8. CUDA 방식: thread-per-edge로 edge마다 lightweight thread 할당(블록당 256 thread), global memory에 데이터 복사 후 kernel 실행(DLP).
9. 결과: 6개 행렬 실측, GPU가 최대 speedup ~180배(matrix F, 67840 edges), throughput 최대 ~34.59 Mbit/s; OpenMP는 8코어에서 speedup ~4배에 그침.
10. 한계: 복호 알고리즘·정정능력 개선 없음, x86 멀티코어는 실시간 throughput 미달, HW(RTL) 미설계, non-NAND 방송(DTMB) 코드 대상.
