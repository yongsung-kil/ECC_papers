> 총평: LDPC 디코더/부호가 아닌 3-D NAND FTL 계층의 read-level 기록·예측(ML) 최적화라 Prime ECC 코드베이스에 떼어 쓸 요소 없음.

### B. 알고리즘 요약

1. 대상은 LDPC ECC를 채택한 3-D CT NAND flash SSD로, read 실패 시 read voltage 수(NRL, 1~7)를 늘리는 soft-decision read-retry가 read latency를 최대 10배 늘리는 문제를 다룬다.
2. 문제: 기존 NRL 기록(page 단위 LALDPC, chunk 단위 PS-read)은 stale NRL 또는 고RBER page 지배로 oracle 대비 27~55% 성능 저하.
3. 채널·신뢰성 모델: 3-D flash의 interlayer/intralayer process variation(PV)과 process similarity(PS)를 이용, RBER은 layer 위치·P/E·retention에 의존한다고 모델링.
4. 핵심 1단 AR2-win: block을 인접 layer 묶음 chunk로 분할, 최근 `N`(기본 10)개 read의 다수결에 맞는 NRL을 기록해 고RBER page의 지배를 막고 `NRLnew`를 impact factor `α`로 매끄럽게 갱신.
5. 갱신식은 `α, NRLold, NRLcurrent, R`로 새 NRL을 점진 조정(예: 4→4.25), 급격한 fluctuation 억제.
6. 핵심 2단 AR2-pre: 접근 빈도 낮아 stale NRL을 가진 chunk에 대해 이웃 chunk의 RBER로부터 MLP(hidden 2층, tanh)로 다음 read의 NRL을 예측.
7. 구현 디테일: MLP 입력은 program voltage/pulse, P/E, retention, 각 state 비율·RBER 7개 특징(각 4b), RBER label은 LDPC decoder의 오류 bit 수에서 추출.
8. 학습/관리: 유사 SSD batch로 초기 학습 후 device별 재학습(idle time), retention time 그룹별로 모델 공유해 storage/compute overhead 최소화.
9. 결과: SSDsim + MSR Cambridge trace에서 AR2가 LALDPC 대비 47%, PS-read 대비 19% read latency 감소, 87% read가 최적 NRL 사용, MLP 예측 정확도 94%.
10. 한계: 실제 HW 미구현(SSDsim 시뮬), LDPC 디코더/부호 자체는 미변경, vendor-only NAND 명령 부재로 신뢰성 모델을 확장해 데이터 합성.

| MLP 기반 NRL/RBER 예측, FTL read-retry 스케줄 | 대응 없음 | Prime ECC는 디코더 bit-exact 시뮬로 컨트롤러 펌웨어/FTL 계층 미포함 |
