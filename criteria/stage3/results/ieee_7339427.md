### B. 알고리즘 요약

1. 대상은 protograph에서 lifting factor `N`으로 구성한 SC-LDPC 앙상블 3종: `(l,r,L)`-regular, SC-RA `RA(q,L)`, SC-ARJA, chain length `L`, 위치당 변수노드 `M=nu·N`, rate 0.4~0.5.
2. 문제: 기존 SC-LDPC 비교는 asymptotic(BP threshold, min-distance)만 봐서 실제 유한길이 waterfall 성능 차이를 놓침.
3. 모델: BEC(소거확률 ε) 전송 + peeling decoder(PD), BP와 성능 동등, 잔여그래프의 통계적 진화를 미분방정식으로 추적.
4. 핵심기법: PD 중 deg-1 check node 수 과정 `c1(τ)`의 기대값 `ĉ1(τ)`을 미분방정식 (8)(9)로 계산, critical phase에서 `ĉ1(*)≈γ(ε*−ε)`.
5. 핵심식: waterfall 성능이 `α=γ/√δ̃1(*)`에 지수적으로 의존 — `α`가 클수록 임계상 deg-1 check node 여유가 커져 오류율 급감.
6. 확장: 분산 `δ1(*)`을 완전 공분산 방정식 대신 단일 PD step 분산 `δ̃1`로 근사(계산량 |Fc|² → |Fv|+|Fc|).
7. 구현 디테일: c1(τ)를 Ornstein-Uhlenbeck 과정으로 모델링, 시간 공분산 `φ1∝e^{-θ|ζ-τ|}`, block error `P*` 식 (26)으로 예측.
8. 학습/최적화: γ, δ̃1, θ, τ*를 각 앙상블에 대해 수치적분(Euler)으로 산출.
9. 결과: RA(6,100)이 (4,8,100)보다 threshold·rate가 나쁨에도 유한길이 성능 우수(α 최대), M=2000~8000 시뮬레이션으로 확인, protograph가 unstructured 대비 γ 약 1.25배.
10. 한계: BEC·peeling decoder 전제의 순수 이론, HW 미설계, AWGN은 정성적 확장만 언급, non-binary/GLDPC는 future work.
