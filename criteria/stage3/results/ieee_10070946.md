## B. 알고리즘 요약

1. 시스템: QLC(4bit/cell) 3D NAND SSD, WL당 LSB/CSB/MSB/TSB 4페이지, gray-code GC(A,B,C,D)로 16레벨 매핑(A+B+C+D=15), LDPC ECC는 페이지당 72bit/1KiB 정정.
2. 문제: 고정 gray-code는 read/write 성능·신뢰도 요구를 동시에 못 맞춤(UGC=빠른 쓰기·느린 읽기, BGC=반대).
3. 모델: 페이지 read latency `tR = nSENSE * tSENSE`, 총 read `tREAD=(1+nRETRY)*(tR+tDMA+tLDPC)`; TSP(2m,2n)-GC 호환조건 A+B<2m 등.
4. 핵심기법1: MGC — 컨트롤러에 다중 encoding table + 다중 TSP(4,16)/(16,16)를 넣고 블록단위로 UGC/BGC 선택, GCB 비트맵으로 추적.
5. 의미: 빠른 프로그래밍(UGC+TSP4,16)과 좋은 읽기·신뢰도(BGC+TSP16,16) 장점을 상황별로 취함.
6. 핵심기법2: MGC-FTL — (a) 접근특성 기반 중재(read-only→BGC, write-only→UGC, hot-read 데이터 변환), (b) 신뢰도단계(Young/Middle/Old) 인지 중재.
7. 구현: encoding table 16엔트리, mapping table 오버헤드 flash면적 0.07% 미만, GC 전환 비용 대비 read 이득 휴리스틱으로 threshold 결정.
8. 신뢰도모델: 실측 RBER 기반 retry 모델(식3,4), 재시도마다 RBER δ=20% 감소, 초과 시 LDPC soft-decision 활성.
9. 결과: SSDsim(3D QLC 확장)에서 MGC/MGC_Stage가 UGC/BGC/UGC_RLV 대비 read/write·lifetime 개선(정확 수치는 그림, 본문 확정치 미상).
10. 한계: LDPC는 파라미터(정정능력·지연)만 가진 고정 블록으로, 부호설계·디코더 알고리즘·HW 기여 없음; 순수 시뮬(SSDsim), RTL/ASIC 없음.
