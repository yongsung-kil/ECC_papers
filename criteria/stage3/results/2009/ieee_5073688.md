### B. 알고리즘 요약

1. LDPC 코드를 SPA(soft, 초기 고정 반복)와 IERRWBF(hard, bit-flipping) 두 단계로 순차 복호하는 기존 two-stage hybrid 알고리즘을 개선.
2. 기존 IERRWBF는 반복이 진행될수록 성능 개선 없이 variable/check node 연산에 시간을 낭비하는 문제가 있음.
3. 채널은 AWGN과 Rayleigh fading을 결합한 무선 통신 채널 모델(BPSK 변조) 가정.
4. 제안 기법: check node 단계에 조건 분기를 추가해, iteration `i-2`와 `i`의 syndrome `s_m1`, `s_m2`가 동일하면 error-sum `E_n`도 동일해져 디코딩된 벡터가 반복(oscillation)됨을 검출하고 즉시 종료.
5. 핵심 관찰: `T_m`과 수신벡터 `r_n`이 반복 내내 상수이므로 syndrome이 2회 주기로 반복되면 벡터도 영구히 반복된다는 점을 이용.
6. 구현: 디코딩된 벡터와 syndrome을 3-entry 레지스터(shift register)에 저장해 매 iteration마다 2단계 전과 비교.
7. variable node step, decision step, initialization step은 원본 IERRWBF와 동일하게 유지, check node step만 수정.
8. 별도 학습/최적화 절차 없음 (규칙 기반 조기종료).
9. 결과: (250,125,3), (2000,1000,3) regular LDPC(rate 1/2)에서 SPA에 근접한 BER 성능을 유지하며 SNR=5dB 기준 iteration 수 55→14, 58→5로 대폭 감소.
10. 한계: HW 구현/합성 없음(MATLAB 시뮬만), 무선 페이딩 채널 특화, irregular 코드로의 확장은 future work로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| two-stage hybrid(SPA+IERRWBF) 조기종료 아이디어 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 목적(조기종료)은 유사하나 syndrome 반복 비교 방식으로 CRC 기반 대체 불가 |
| IERRWBF bit-flipping 알고리즘 자체 | 대응 없음 | Prime ECC는 min-sum 기반 soft decoder이며 bit-flipping hard decoder 미보유 |
| SPA(soft-decision) 단계 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 근사를 사용하므로 full-tanh SPA는 직접 대응 없음 |

> 적용 가치: 낮음 — bit-flipping 계열 hard decoder의 조기종료 트릭이며, Prime ECC의 min-sum soft decoder 구조와 알고리즘 계열이 달라 이식 난이도가 높고 정정능력 개선도 없음.
