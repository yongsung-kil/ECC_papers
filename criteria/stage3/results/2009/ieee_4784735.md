### B. 알고리즘 요약

1. 시스템: IEEE 802.11n MIMO-OFDM WLAN, 4x4 안테나, 20MHz, QC-LDPC codeword length `n=648/1296/1944`, rate `R=1/2,2/3,3/4,5/6` 12조합.
2. 문제: 필수인 BCC(Viterbi) 대비 옵션인 LDPC의 성능이득을 시뮬로 검증하고 HW 구현 실현성 확인.
3. 채널모델: AWGN 및 802.11n TGn 채널모델 B(MIMO 소형사무실 모델).
4. 디코딩: 표준 min-sum 알고리즘, 초기화 `H_MN=1, β_MN=0` 후 row(수평) 처리로 `α_MN = sign(λ_N+β_MN) * min|λ_N+β_MN|` 계산.
5. column(수직) 처리로 `β_MN = Σ α_MN` 갱신, LLR `λ_N = 2y_N/σ^2` (AWGN 기준).
6. codeword 잠정판정 `ĉ_N`을 부호합(`Σα_MN`)과 `λ_N` 합의 부호로 결정, `Hĉ^T=0`이면 종료 아니면 반복(최대 iteration까지).
7. HW: partially-parallel 구조, 내부데이터 16-bit fixed point(부호1+정수11+소수4), 4-stage pipeline MUL 유닛(CSA+CPA).
8. HPU 108개(row 연산, 3-to-2/4-to-2 비교기로 최소값 2개 탐색+XOR tree 부호결정), VPU 216개(column 연산, CSA tree+2-stage CPA).
9. 결과: MCS-31(rate 5/6)에서 AWGN 3.2dB, TGn-B 채널 6dB 성능이득; iteration 15회가 30회와 동등성능으로 최적; 0.13µm 합성 시 총면적 69,467,024um^2, clock 500MHz, throughput ≤250Mbps.
10. 한계: 표준 규격 그대로의 min-sum 구현·성능비교 보고서로 알고리즘적 개선(스케줄링, error-floor 대책 등) 없음; memory 면적이 전체의 대부분 차지하나 최적화 기법 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 디코딩 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 표준 min-sum과 동일 기법, 신규성 없음 |
| row(수평) 처리 (α_MN 계산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 표준 CN 연산과 동일 |
| column(수직) 처리 (β_MN 갱신) | `decoder.cpp` `VN_Cal_HD()` 등 | 표준 VN 연산과 동일 |
| partially-parallel HW / HPU·VPU 구조 | 디코더 HW 모델 (z=32, HCU, GT) | 802.11n 전용 12-조합 구조로 QC-LDPC 아키텍처 상이, 참고 수준 |
| 16-bit fixed point 양자화 | `channel.cpp` + LLR 테이블 (`PARAM_LLR`) | soft-4bit+ 수준으로 NAND read 비용 대비 과도, 이식 가치 낮음 |

> 적용 가치: 낮음 - 802.11n 표준 QC-LDPC에 대한 통상적 min-sum ASIC 구현·성능평가로 Prime ECC가 이미 보유한 기법과 중복되며 신규 디코더/부호설계 기여가 없음.
