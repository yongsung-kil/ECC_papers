### B. 알고리즘 요약

1. 대상: cyclic (4095,3367) EG-LDPC (2D 유클리드 기하 EG(2,2^6)), 열/행 가중 64, min distance 65, 크기<65 trapping set 없음; QC 형태(63×63 CPM)로 변환, 총 262080 edge.
2. 문제: SPA는 최고성능이나 HW 복잡; 저비용·고속·soft 미가용(storage) 환경 모두를 하나의 디코더로 커버하고자 함.
3. 3개 호환 알고리즘 통합: `OS-MLGD`(1-iteration 다수결), `SRBI-MLGD`(soft 신뢰도 기반), `HRBI-MLGD`(hard 신뢰도 기반) — 초기화·신뢰도 척도만 다르고 반복 스텝 동일.
4. 핵심 연산: 신드롬 `s(k)=z(k)H^T`, 정수합 `E_j(k)=Σ(1-2(s_i⊕z_j))`, 신뢰도 갱신 `R_j(k+1)=R_j(k)+E_j(k)` 후 부호로 hard-decision.
5. 모든 모드가 **1-bit 메시지 + XOR + 정수덧셈**만 사용 → CNU가 64-bit XOR 게이트로 축소(SPA CNU 대비 대폭 단순), 배선(interconnect) 복잡도·클록 개선.
6. soft 입력은 2^b 레벨 균일 양자화(∆=0.015625, 8-bit: 정수2+소수6)로 신뢰도 `q_j` 표현; HRBI/OS는 ±γ hard 초기화.
7. Memory sharing: 같은 offset의 CPM들을 넓은 워드로 묶어 하나의 block RAM에 매핑(1-bit라 더 많이 압축) → 288개뿐인 block RAM에 4160 CPM 수용.
8. 2 codeword 동시복호: block RAM depth(512)의 미사용 영역과 true-dual-port를 활용, VNU·CNU가 서로 다른 codeword를 처리 → waiting-time 계산 불필요, throughput 2배.
9. 결과: Virtex-4 LX160에서 191MHz, 12857 slices, 153 block RAM, 1.9Gbps, BER 10^-11까지 하루내 측정; SRBI-MLGD는 kmax=5에서 4.9dB시 평균 2.0 iteration.
10. 한계: SRBI-MLGD가 SPA 대비 약 0.6dB 손실, EG 고열가중 부호 특화, memory sharing 등은 FPGA block RAM 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 반복 복호 알고리즘(MLGD 3모드) | [decoder.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 다수결(MLGD) 계열로 min-sum과 연산이 달라 루프 골격만 유사, 알고리즘 교체 필요 |
| CN 갱신(XOR/신드롬) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min1/min2 magnitude, 논문은 64-bit XOR → 직접 대응 아님 |
| hard-decision 모드(HRBI/OS) | `decoder.cpp` `VN_Cal_HD()` | HD 경로와 개념 대응(soft 미가용 시 사용) |
| soft 신뢰도 양자화 | `channel.cpp` `Set_LLR_Th()`, `Set_R_Offset()` | 2^b 균일 양자화 vs LLR 테이블, 접점 있음 |
| QC H-matrix/CPM 로드 | `ecc_top.cpp` `Load_PCM()` | QC 구조 로드 대응 |
| memory sharing / 2 codeword 동시복호 | 대응 없음 | FPGA block RAM 특화 트릭, 고정 z=32 파이프라인 모델과 무관 |

적용 가치: **낮음** — MLGD 계열 + 유한기하 고열가중 부호에 종속되어 degree-17 min-sum QC-LDPC 엔진에 알고리즘이 맞지 않고, HW 최적화도 FPGA block RAM 특화다. 다만 hard/soft 겸용·2 codeword throughput 아이디어는 참고할 만하다.
