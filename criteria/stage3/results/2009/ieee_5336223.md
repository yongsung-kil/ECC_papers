> 총평: QC-LDPC offset min-sum 디코더의 serial-parallel HW 아키텍처와 quantization(Q5.2, offset=0.5) 설계는 Prime ECC와 유사 계열이라 참고 가치가 있으나, 핵심 기여인 VOS/RPR 저전력 기법은 NAND ECC 맥락과 거리가 있어 이식성은 중간 수준.

### B. 알고리즘 요약

1. IEEE 802.11n 표준의 12종 QC-LDPC 코드(rate 1/2, 2/3, 3/4, 5/6; codeword length 648/1296/1944, submatrix size 27/54/81)를 단일 유연 디코더로 지원.
2. 고처리량(최대 600Mbps~2.54Gbps) 요구를 만족하면서도 저전력을 달성하는 QC-LDPC 디코더 HW 설계가 목표.
3. Offset min-sum 알고리즘(MSA) 사용: check node update `λ'_{n→m} = sgn(∏λ) × max{min|λ|-β, 0}`(식 2), offset β=0.5(Q5.2 표현상 이진 10)로 SPA 대비 근사오차를 보정.
4. "Serial-parallel" 아키텍처: BPU(bit node processing, 최소값 2개 탐색)는 직렬화, CPU(check node processing, 합산)는 병렬화 — 기존 "parallel-serial" 대비 처리 유닛 단순화·메모리 오버헤드 감소.
5. Delayed offset calibration: check magnitude를 offset 보정 없이 저장했다가 bit node update 마지막 단계에서 보정 적용 — MIN32/check RAM 크기 절감, 정정성능 저하 없음.
6. Triple-bank 메모리로 parallel factor P(3/9/27) 확장 시 non-aligned memory access를 단일 사이클에 처리(정렬 HW로 두 뱅크 데이터 결합·shift).
7. Word-length 최적화: 정수부 5bit(부호 포함)+소수부 2bit(Q5.2)가 조기 error-floor 없이 최적, offset=0.5 적용 시 BER 10^-6에서 약 0.4dB SNR 이득(부동소수점 SPA 대비 고SNR에서 우세).
8. 저전력 전략 두 가지: (A) 완화된 main path + 엄격 타이밍 제약된 RPR path, VOS(Vdd=1.0V) 적용, 연속 디코딩 실패=타이밍 에러로 간주해 RPR 활성화; (B) 이완된 RPR path 상시 사용, 실패 시에만 고정밀 main path 가동.
9. 결과: 90nm IBM 공정에서 90MHz~476MHz 클록, 설계 B 기준 typical 조건에서 35% 전력 절감(25.6mW → 16.7mW 수준), 정정성능은 Q5.0 대비 offset-Q5.2가 0.4~0.5dB 우수, error floor는 BER 10^-9까지 미관측.
10. 한계: 저전력 전략(VOS/RPR)이 채널·PVT 조건 통계에 의존적이라 최적 설계 선택이 워크로드에 좌우되며, 정정성능 개선은 offset calibration에 의한 것으로 새로운 디코딩 알고리즘 자체의 혁신은 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset min-sum (CNU/VNU update, offset β 보정) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum + magnitude 양자화 테이블 기반이라 offset calibration 적용 여부 비교 참고 가능하나 이미 유사 기법 보유 |
| Q5.2 양자화 word-length 설계, delayed offset calibration | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC의 iteration별 LLR threshold 테이블과 유사한 목적, word-length 튜닝 참고 가치 있음 |
| Triple-bank 메모리 unaligned access 해결 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z_sb=32 고정 병렬단위이며 triple-bank 개념은 구조가 상이해 직접 대응 어려움 |
| VOS/RPR 저전력 기법 (예비경로, 전압 오버스케일링) | 대응 없음 | Prime ECC 프로파일에 저전력/타이밍 예비경로 개념이 없음 |
| 연속 디코딩 실패 감지 → RPR 활성화 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 디코딩 실패 감지 메커니즘 자체는 조기종료 CRC와 유사한 착안점이나 목적(저전력 전환 vs 조기종료)이 다름 |

적용 가치: **중간** — offset min-sum·양자화 설계는 Prime ECC와 유사 계열로 word-length/offset 튜닝 참고 가치가 있으나, 핵심 기여인 VOS/RPR 저전력 기법은 무선 통신向 설계로 NAND ECC 이식 시 근본적 재구성이 필요함.
