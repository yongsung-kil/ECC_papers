### B. 알고리즘 요약

1. 대상은 이진대칭채널(BSC) 상에서 hard-decision 디코딩(bit-flipping, Gallager A/B)을 사용하는 left-regular LDPC 코드의 이론적 오류정정 능력.
2. 기존 asymptotic 성능 분석(density evolution)은 잘 이해되어 있으나, 유한장(finite-length) 코드의 guaranteed 오류정정 능력은 expander graph 논증으로만 다뤄져 왔고 특히 dv=3(column-weight-three) 코드에는 적용 불가하다는 문제를 다룸.
3. 핵심 모델은 Tanner graph의 girth `g(G)`, column weight `dv`, expansion factor `(dv,dc,α,β)` expander 정의를 통한 오류 패턴 분석.
4. 핵심 기법은 girth와 expansion을 연결하는 정리로, `dv≥4`-left-regular 그래프에서 girth `g=2g'`일 때 크기 `k<n0(dv/2, g')`인 모든 변수노드 집합이 `(3/4)dv` 이상 팽창함을 증명.
5. 핵심 식 `n0(d,g)`은 girth 함수로, 이 값이 최소거리(및 최소 stopping set 크기)의 하한이 되어 ML 디코딩 하의 오류정정능력 `⌊n0(dv,g')-1⌋/2`을 도출 — 오류정정능력이 girth에 지수적으로 증가함을 보임.
6. 확장 개념으로 fixed set/trapping set을 정의(bit-flipping 디코더가 정지·진동하는 변수노드 집합), dv=3 코드는 cycle이 있으면 무조건 fixed set이 되어 선형 개수 오류정정이 원천적으로 불가능함을 증명.
7. 부수 논의로 cage graph(예: Peterson graph, (3,5)-cage)를 이용해 potential fixed set의 예시를 구성.
8. 별도 학습/최적화 절차 없음 — 순수 조합론적 증명/논증(work-in-progress로 2-bit 디코더가 dv=3 코드의 cycle 오류를 극복할 가능성만 제시).
9. 결과는 수치 시뮬레이션이 아닌 정리(theorem) 형태 — 예: girth≥10인 column-weight-3 코드의 Gallager A는 최대 g/2 iteration 내 최대 g/2-1개 오류를 정정 가능함을 증명.
10. 한계: 이론/증명 전용 논문으로 실제 코드 구성·시뮬레이션·HW 결과 없음, dv=3 코드에 대한 실용적 해법(2-bit 디코더)은 미완성(work in progress)으로 남겨둠.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| hard-decision bit-flipping/Gallager A 디코딩 | `decoder.cpp` `VN_Cal_HD()` | Prime ECC의 HD(hard-decision) 모드와 개념적으로 관련되나 본 논문은 min-sum이 아닌 순수 bit-flipping 이론 |
| trapping set 회피 코드 구성 | `ecc_top.cpp` `Load_PCM()` | error-floor 대책과 관련된 개념이지만 구체적 구성법 제시 없이 이론적 존재 논증뿐 |
| girth/expansion 기반 정정능력 하한 | 대응 없음 | 순수 조합론적 하한 증명으로 실제 HW/코드 모듈에 직접 대응 없음 |

> 적용 가치: 낮음 — 실제 코드 구성법이나 디코더 알고리즘 제시 없이 이론적 하한만 증명하는 서베이성 논문이라 Prime ECC 이식 대상이 아님.
