### B. 알고리즘 요약

1. (1057, 813) cyclic PG-LDPC 부호(row/column weight 33)를 AWGN+BPSK 채널에서 4-bit 양자화로 복호.
2. SPA는 고weight 부호에서 연결 복잡도가 급증해 구현이 사실상 불가능하고, BF는 구현은 쉽지만 정정성능이 SPA 대비 크게 열화됨.
3. 각 variable/check node는 `b`-bit 양자화, sign-magnitude 표현, 세 개의 flipping threshold(`δ1f, δ2f, δ3f`)로 flip 강도를 4단계(strong flip/weak flip/maintain/strengthen)로 결정.
4. 핵심 기법은 SBF(Soft Bit-Flipping): syndrome 계산 후 각 check node weight `α_m = Σ|y_n|`를 구하고, 각 variable node의 flipping function `f_n = Σ sgn(...)·α_m - |y_n|`을 계산해 threshold와 비교.
5. flip 강도별 update 식(`sgn(y_n)·(2.5·max(1,|y_n|)-2)` 등)은 SPA의 reliability 전파를 근사하면서 BF 수준의 단순 연산만 사용.
6. HW 구조로 VPU(variable node processing unit, 33-input XOR+adder)와 CPU(check node processing unit)를 결합, 이후 SNPU(Shared Node Processing Unit)로 두 유닛의 modulo-2/ordinary summation 하드웨어를 공유해 면적 절감.
7. 2-stage pipelining을 SNPU의 Wallace tree-CLA adder 중간에 삽입해 critical path(7.97ns→최적화) 단축, threshold adaptation 모듈도 병렬 factor에 맞춰 pipeline.
8. 별도 학습/최적화 절차는 없음(고정 threshold, 구조적 HW 최적화만).
9. 결과: parallel factor 64에서 SPA 대비 0.6dB 열화(FER 10^-4), 면적 2.5mm2, 처리량 6.5Gbps(직렬 대비 87배), SNPU로 면적 9.1% 절감.
10. 한계: FPGA/실측 아님(ASIC합성/시뮬 결과), 고weight 정규 PG-LDPC 특화 구조로 NAND용 irregular QC-LDPC로 일반화 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SBF 알고리즘 전체(iterative flipping) | `decoder.cpp` `LDPC_Decoder()` | min-sum이 아닌 대안 반복복호 알고리즘, 참고용 |
| VPU/CPU node processing(변형 min-sum 유사 연산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 33-input 고weight 연산이라 z=32 lifting 구조와 직접 대응 어려움 |
| SNPU 공유 하드웨어(면적절감) | 대응 없음 | Prime ECC엔 노드 유닛 공유 개념 없음, 참고 가치만 |
| threshold adaptation | `ecc_data.h` `PARAM_LLR` | LLR/threshold 테이블 개념과 유사하나 알고리즘 상이 |
| pipelining 2-stage | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC도 6-stage pipeline 보유, 설계사상 유사 |

적용 가치: 낮음 — high-weight regular PG-LDPC 전용 bit-flipping 변형으로 Prime ECC의 QC-LDPC min-sum 구조와 부호/알고리즘 모두 이질적.
