### B. 알고리즘 요약

1. 정규 (J,K) LDPC convolutional code(LDPCC)의 무한 협약 프로토그래프 base matrix `B[−∞,∞]`를 특정 시점 `t=1`부터 `L`까지 종료(termination)하여 유한 블록 프로토그래프 `B[1,L]`를 구성한다.
2. 문제의식: 표준 regular (J,K) 블록 코드는 짧은 parity-check(작은 K)를 쓰려면 J도 작아져 threshold가 낮아지는 trade-off가 있다.
3. 채널 모델은 BEC(erasure probability `ε`)이며, BP 복호의 density evolution을 `p^(i)`(V2C erasure 확률), `q^(i)`(C2V erasure 확률) 재귀식 (2),(3)으로 기술한다.
4. 핵심 구성: 정규 (J,K) 프로토그래프의 all-one base matrix `B`를 성분행렬 `B0,...,Bms`로 "edge spreading"하여 협약 프로토그래프를 만들고(식 10, `ΣBi=B`), 이를 길이 `L`로 종료한다.
5. 종료된 프로토그래프는 시작/끝 부분 check node degree가 K보다 작아지며, L이 커질수록 이 "강한 노드" 비율이 사라지지만 threshold 이득은 감소하지 않음(design rate 식 9, 11).
6. Theorem 1: sliding window 갱신 스케줄에서 window 시작점의 message probability가 특정 값 `ε0`에 도달하면 이후 모든 시점에서도 도달함을 증명(threshold가 L에 무관하게 유지되는 근거).
7. 구현 디테일: edge spreading은 유일하지 않으며, B0의 각 row weight를 낮게(최소 degree 2) 설계할수록 좋은 threshold를 얻는 경험적 지침을 제시.
8. Example 1 (3,6)-LDPCC: L=10에서 threshold `ε*=0.505`(R10=2/5, Shannon `εsh=0.6`), L→∞에서 `ε*=0.488`(R∞=1/2, `εsh=0.5`)로 capacity에 근접.
9. Example 3 (AR4JA 비정규 프로토그래프 확장): 원 코드 threshold `ε*=0.4387`(RB=1/2) → edge spreading+termination 후 `ε*=0.4996`(Shannon `εsh=0.5`)로 큰 개선.
10. 한계: 전 분석이 BEC에 국한되며 AWGN 등 실채널 검증 없음, HW/실측 전무, 순수 이론/구성법(threshold 분석)으로 실제 decoder 복호성능(BER, iteration) 수치 실험 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 협약(convolutional) 프로토그래프 구성/termination | 대응 없음 | Prime ECC는 QC-LDPC(block code) 고정 구조이며 SC-LDPC convolutional 구성 미지원 |
| Edge spreading을 통한 base matrix 분할 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 설계 관점에서만 간접 연관, 실제 이식은 재검증 부담 큼(§4 "높음") |
| BEC density evolution threshold 분석 | 대응 없음 | Prime ECC 채널은 AWGN/RBER/fixed-error(`channel.cpp`)이며 BEC 미지원, 이론적 threshold 분석 도구 부재 |

> 적용 가치: 낮음 — BEC 전용 이론적 SC-LDPC 구성법으로 binary QC-LDPC 고정 구조인 Prime ECC 3.1의 부호설계 레이어와 직접 대응되지 않고, HW/디코더 개선 요소가 전혀 없음.
