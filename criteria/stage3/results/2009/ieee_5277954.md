### B. 알고리즘 요약

1. 시스템: DVB-S2 위성방송 표준의 IRA(Irregular Repeat-Accumulate)-LDPC 코드, rate 1/4~9/10, 정보/패리티 비트노드 구분 구조.
2. 문제: 기존 partially-parallel 아키텍처(Kienle 255Mbps, Dielissen 90Mbps)는 처리량이 낮거나 메모리 사용이 비효율적.
3. 핵심 관찰: 행렬 H의 순환 구조 특성식 `α_{r+q×i,(c+i) mod 360+360×(c mod 360)} = α_{r,c}` (식2), `β_{(r+i) mod (n-k),(c+i) mod (n-k)} = β_{r,c}` (식3)에 의해 360개 비트노드/체크노드를 그룹으로 클러스터링 가능.
4. 노드 그룹을 각각 BFM(Bitnode Functional Module), CFM(Checknode Functional Module)로 구현, 360개 노드가 1개 functional module을 공유하며 순차 처리(365 clock cycle).
5. 핵심 기법: shared local memory 구조 — clustered Tanner graph의 edge 수만큼 메모리 뱅크를 두고, rising edge에 read, falling edge에 store를 수행해 동일 클록 내 read/store 동시 처리, 메모리 충돌 회피.
6. Shuffling network가 matrix H로부터 라우팅 정보를 추출해 계산된 메시지를 올바른 메모리 뱅크로 전달.
7. 구현 디테일: CFM 입력단에서 2's complement를 signed-magnitude로 변환해 LUT 엔트리·adder word length 감소(면적 절감), BFM은 5-stage pipelined 16-adder 구조.
8. 최적화 절차: 최대 반복횟수 25회는 exhaustive computer simulation으로 결정, 별도 학습 알고리즘(DE/GA/NN) 없음.
9. 결과: 90nm 합성, 300MHz 동작, throughput 520Mbps (Kienle 대비 +104%, Dielissen 대비 +479%), 면적 증가는 각각 +9%/+195%에 그침.
10. 한계: DVB-S2 IRA-LDPC 구조(360 고정 클러스터 크기)에 특화되어 있고, 정정성능/error-floor 개선은 다루지 않음(순수 처리량·메모리 아키텍처 논문).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Shared local memory (read/store 타이밍 분리로 충돌 회피) | 디코더 HW 모델 (z=32, HCU, GT) | 메모리 뱅크 충돌 회피 아이디어는 Prime ECC의 병렬 메모리 라우팅 최적화에 참고 가능 |
| 360-노드 클러스터링 (BFM/CFM) | `decoder.cpp` `LDPC_Decoder()` | IRA 특유의 360-그룹 구조로 QC-LDPC의 z=32 lifting 단위와는 상이, 직접 대응 어려움 |
| 2's complement → signed-magnitude 변환 (CFM 면적 절감) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 sign/magnitude 분리 방식(min1/min2+sign XOR) 채택 중, 신규성 낮음 |
| Shuffling network (matrix H 기반 라우팅) | `GT.cpp` `Make_GT_HW()` | 엣지 라우팅 개념은 Graph Thinning과 유사하나 목적(충돌회피 vs 처리 edge 축소)이 다름 |

> 적용 가치: 중간 — 정정성능 기여는 없으나 read/store 타이밍 분리를 통한 메모리 충돌 회피 기법은 Prime ECC의 HW 메모리 구조 최적화에 아이디어 차원에서 참고할 수 있음.
