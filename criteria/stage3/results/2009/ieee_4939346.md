### B. 알고리즘 요약

1. 대상은 array code 기반 QC-LDPC 부호로, exponent matrix `E(H) = (aij)`, `aij = i·j mod L` (L은 소수)로 정의됨.
2. Array code는 girth 6이며, 행/열을 일부만 남기는 shortening으로 girth를 키울 수 있으나 코드 길이가 짧아지는 트레이드오프가 존재.
3. 이 논문은 shortened array code의 짧아진 길이를 girth 손실 없이 다시 늘리는 문제와, 지정된 girth를 갖는 부호를 쉽게 설계하는 문제를 다룸.
4. 핵심 기법은 CRT(Chinese Remainder Theorem): `L = L1·L2·...·Ls` (서로 다른 소수)로 두고 각 컴포넌트 부호 `Cl`의 지수행렬 `E(Hl)`을 `aij = Σ alij·Al·L̄l mod L`로 결합해 새 지수행렬 `E(H)`를 생성 (`L̄l = L/Ll`, `Al·L̄l ≡ 1 mod Ll`).
5. Lemma 3.1에 의해 결합된 부호의 girth `g ≥ max{g1,...,gs}` (컴포넌트 부호 중 최대 girth 이상 보장).
6. 확장 기법(Sec. IV): girth `g1`의 shortened array code `H1`을 다른 소수들과 CRT 결합해 girth를 유지한 채 크기를 `mL1×nL1`에서 `mL×nL`로 확장.
7. 지정 girth 설계 기법(Sec. V): 행/열을 신중히 삭제해 cycle 조건(식 3, 4)을 위배하지 않는 컴포넌트 부호 `H̄1`을 만들고, 나머지 컴포넌트는 랜덤/임의 구성 후 CRT로 결합.
8. 별도의 학습/최적화 절차 없음. 컴포넌트 지수행렬 선택은 수작업/랜덤 탐색이며 greedy computer search로 실제 girth를 사후 확인.
9. Example 1: girth 8 유지하며 31×31→217×217 확장 (3×7 블록). Example 2: length 2065, girth 12 부호 구성 (목표 10보다 큰 girth 획득). 두 경우 모두 array code/shortened array code 대비 BER 개선을 시뮬로 제시.
10. 한계: 순수 이론/시뮬 구성법이며 디코더 알고리즘, HW, 양자화 등에 대한 언급 전혀 없음. component code로 array/shortened array만 다뤄 일반 QC-LDPC 확장 가능성은 미검증.
