### B. 알고리즘 요약

1. 대상: AWGN 채널 binary LDPC의 LP(linear programming) 복호. 테스트 부호 (3,6)-regular LDPC, len 96, rate 1/2.
2. 문제: 중·단블록 LDPC에서 SP 복호는 성능이 나쁘고, Feldman의 LP 복호는 ML certificate(정수해=ML codeword / 분수해=실패)를 주지만 고잡음 영역에서 SP보다 약간 열세.
3. 모델: ML 복호 `min γ^T x, x∈conv(C)`를 fundamental polytope `P=∩conv(Cj)`로 완화(식 6,7). 최적해가 정수 꼭짓점이면 ML codeword, 분수 pseudocodeword면 실패 선언.
4. 핵심 기법: LP가 분수해(실패)를 낼 때, 특정 check node `cj`의 single-parity-check 부호 `Cj-`의 각 configuration을 등식제약으로 고정한 `2^(γ-1)`개 LP(식 8) 생성·풀이(MLP).
5. 핵심 정리: GLP 중 최적해가 (1)정수이고 (2)목적값이 그룹 내 최소이면 반드시 ML codeword (Theorem 1). 결과 `x_MLP`도 ML certificate 보유(Remark 2).
6. Lemma 4: 초평면으로 자른 `P'`의 꼭짓점 집합은 `P`의 꼭짓점 집합의 부분 → pseudocodeword 정합성 유지.
7. check node 선택 기준: 분수 성분 집합 `M`과 이웃 `N(j)` 교집합 최대인 `j*=argmax|N(j)∩M|`(식 9) 휴리스틱.
8. 복잡도: 최악 `2^(γ-1)+1`개 LP 풀이. LDPC는 γ 작아 증가폭이 크지 않다고 주장.
9. 결과: len96 (3,6)-LDPC에서 WER=10^-2일 때 MLP가 LP·SP 대비 약 0.2dB 이득(LP·SP는 3.7dB 이상서 유사).
10. 한계: HW 미설계, 시뮬(AWGN)뿐, 짧은 rate1/2 부호 1개만; LP relaxation 패러다임이라 min-sum HW와 무관.
