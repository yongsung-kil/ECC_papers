> 총평: irregular QC-LDPC의 shortening 위치를 girth가 아닌 degree distribution/extrinsic-information 기준으로 선택해 임계값(threshold)을 낮추는 순수 부호 구성법. NAND ECC의 다양한 length/rate 요구에 대응하는 shortening 설계에 참고 가능.

### B. 알고리즘 요약

1. 대상 부호는 irregular QC-LDPC, base matrix `Hb`(M×N, 각 원소는 z×z circulant 또는 0), 예시로 802.16e (2304,1152, rate 0.5)와 802.11n (1944,972, rate 0.5) 코드 사용.
2. 풀려는 문제: shortening(특정 variable node에 무한 신뢰도 부여, 인코더/디코더가 위치를 공유)으로 더 짧은 length·낮은 rate 부호를 얻을 때, irregular 코드에서는 기존 regular 코드용 girth 최대화 기준이 성능에 안 맞음(irregular는 degree distribution이 성능을 지배).
3. 채널모델은 BPSK-modulated AWGN, 디코딩은 belief propagation(BP), 최대 100 iteration.
4. 핵심 기법: variable node(컬럼) 선택 시 extrinsic-sum `E(j) = Σ_{i∈Λj}(ρ(i)−1)` (자신이 연결된 각 check node degree −1의 합)을 정의, 낮은 degree column 후보 중 `E(j)`가 최대인 컬럼을 shortening 대상으로 선택.
5. `E(j)`가 큰 컬럼을 고르는 이유: 컬럼 제거 시 연결된 check node degree가 `1/ρ(i)` 비율로 줄어드는데, 이미 degree 큰 check node를 건드리면 그 감소율이 작아 전체 그래프의 degree profile 손상을 최소화.
6. 확장(동점 처리): extrinsic-sum이 같은 후보가 여럿이면 PEXIT(protograph EXIT) 분석으로 각 후보 제거 시 예상 iterative decoding threshold(`Eb/N0`)를 추정해 가장 낮은 threshold를 주는 컬럼 선택.
7. 구현 디테일: QC-LDPC는 protograph의 특수사례이므로 base matrix 단위(전체 순환치환 확장 없이)로 PEXIT 분석 가능해 계산복잡도 절감; regular 코드나 non-QC 코드에는 알고리즘 변형 필요(density evolution 대체, 계산량 증가)라고 명시.
8. 최적화 절차: Step0~Step6의 greedy 반복 탐색(최소 degree 후보 선정 → extrinsic-sum 계산 → 최대값 선택 → 동점시 PEXIT → 그래프 업데이트 → 목표 shortening 수 `T` 도달 시 종료).
9. 결과: (2304,1152) 코드에서 제안 패턴(1,2,5,11) threshold 0.534dB(girth 6, 무작위 패턴 대비 최저) vs 무작위 패턴 최대 0.844dB; (1944,972) 코드도 유사하게 제안 패턴이 최저 threshold(0.391dB) 확보. FER 시뮬레이션에서도 제안 패턴이 유의미한 성능 이득을 보임. girth가 가장 작아도(6) threshold가 가장 낮아, girth가 irregular 코드 성능의 지배 요인이 아님을 확인.
10. 한계: 순수 부호설계(shortening 위치 선정) 알고리즘으로 HW 구현·복잡도·throughput 분석 없음, 검증은 PEXIT 이론분석+시뮬레이션(FER)뿐이며 실측/FPGA/ASIC 없음, 802.16e/802.11n의 특정 base matrix 구조(dual-diagonal parity part)에 최적화되어 일반 QC-LDPC로의 확장은 density evolution 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Shortening 위치 탐색(degree/extrinsic-sum 기반) | `ecc_top.cpp` `Load_PCM()` | base matrix(`PCM_b`)에서 특정 col-block을 shortening하는 정책 설계에 참고 가능. 다만 Prime ECC는 이미 puncture(`punct_col_num=2`) 구조 보유, shortening은 별도 미구현 기능 |
| PEXIT 기반 threshold 추정 | 대응 없음 | Prime ECC는 이론적 threshold 분석 도구 없이 시뮬레이션 기반 BER 평가만 수행, 새 분석 레이어 필요 |
| Base matrix degree distribution 분석 | `ecc_data.h`(`PCM_b`, VN degree 17) | Prime ECC의 고정 degree 구조(129 col-block, VN degree 17)에 이 알고리즘을 적용해 rate/length variant 생성 시 참고 가능 |

> 적용 가치: 중간 | HW/디코더 구현이 아닌 순수 부호설계(shortening 위치 선정) 알고리즘이라 이식 난이도는 있으나(§4 "부호설계=높음"), NAND ECC가 다양한 length/rate 요구에 대응해야 한다면 Prime ECC 고정 base matrix 기반 shortening 변형 설계에 직접 참고할 수 있는 실용적 기법.
