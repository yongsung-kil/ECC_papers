### B. 알고리즘 요약

1. QC-LDPC 코드(length 2304, rate 1/2, block size 96) 대상 고속 FPGA 디코더, 채널 LLR은 7비트 soft 입력.
2. 기존 SPA/Min-Sum의 row-column 2-phase 반복은 수렴이 느리고 clock 소모가 커서 고속화에 한계.
3. Min-Sum 근사(anti-tanh 대신 min, scale α·offset β 적용)를 기반으로 하되, layered decoding으로 스케줄링만 변경.
4. 핵심 기법 1단(layered decoding): row/column을 분리하지 않고 `u_ij = U_j - v_ij`, `v'_ij = α·sign(·)·max(min|u_ik|-β,0)`, `U'_j = u_ij + v'_ij` 로 한 번에 갱신하는 joint row-column processor.
5. row processor 결과를 column processor가 즉시 사용하므로(값 갱신 즉시 재사용) 동일 iteration 수 대비 수렴이 2배 빠르고, 1-phase라 iteration당 clock이 기존 대비 절반 — 이론상 4배 속도 이득.
6. 핵심 기법 2단(코드 구성): PEG 알고리즘에 "홀수 인덱스 변수노드는 짝수 인덱스 체크노드만, 짝수는 홀수만" 연결하는 제약을 추가해 인접 처리 행(row) 간 교차(intersection)를 없애 병렬도(parallel degree) 향상 — 예시로 8x8 QC 블록에서 [1 6 3 8 5 2 7 4] 무교차 순열 도출.
7. 구현 디테일: 교차 없는 열(column)의 채널 정보를 같은 RAM에 저장해 RAM 블록 수를 최대 row degree(7)까지 축소; joint processor는 3 clock에 7개 값 중 min/2nd-min 계산, RAM read~write 전체 8 clock; 병렬도 12(block size 96 기준).
8. 별도 학습 절차 없음(PEG 알고리즘에 결정론적 제약만 추가), 제약 유무 성능 비교는 SPA/50 iteration 시뮬로 검증.
9. 결과: 단일 디코더 155MHz, 8 iteration 기준 232.5Mbps; 4-디코더 그룹 시 128MHz, 768Mbps(ALUTs 48%, RAM 11% 사용, Altera Stratix2 EP2S180); 제약 코드와 무제약 코드 BER은 거의 동일.
10. 한계: layered decoding 자체는 선행연구([9]) 기반 재구성이며, gatecount/ASIC 검증 없이 FPGA 자원(ALUT/FF/RAM) 수치만 제시, 성능 비교는 시뮬 BER 그래프에만 있고 본문 수치 서술은 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered decoding (joint row-column 1-phase) | `decoder.cpp` `LDPC_Decoder()`, `Get_VNU_Table_Idx()` | Prime ECC는 Dual-Update(even/odd 교대) 스케줄 보유 - layered 방식과 유사 계열이라 이식 시 iteration 루프 재구성 필요, 난이도 낮음~중 |
| PEG + odd/even check-node 제약 코드 구성 | `ecc_top.cpp` `Load_PCM()` | 부호설계(H-matrix 구성) 영역으로 Prime ECC 고정 QC-LDPC 특성상 이식 난이도 높음 |
| 무교차 채널정보 RAM 그룹핑 | `channel.cpp` `Set_R_Offset()` | LLR 저장 메모리 충돌 회피 기법으로 채널/메모리 설계 참고 가능 |
| joint processor min/2nd-min 3-clock 연산 | `decoder.cpp` `CNU_Update_New_Mag()` | CNU의 min1/min2 연산과 직접 대응, HW 파이프라인 참고 가치 있음 |
| scale α·offset β Min-Sum 근사 | `decoder.cpp` `CNU_Update_New_Mag()`, `ecc_data.h` `PARAM_LLR` | 이미 Prime ECC가 magnitude 양자화 테이블 기반으로 보유(중복 기여) |

> 적용 가치: 중간 | layered decoding에 의한 iteration/clock 절반 감소는 실질적 latency 이득으로 이식 가치 있으나, Prime ECC는 이미 유사 계열(Dual-Update)을 보유하고 있어 완전 신규 기여는 아니며 PEG 코드 구성 제약은 부호설계 영역이라 이식 난이도가 높음.
