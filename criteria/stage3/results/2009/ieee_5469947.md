### B. 알고리즘 요약

1. 대상은 통신/방송/스토리지 표준(10GBASE-T, DVB-S2, 802.11n/16e, WPAN)에 쓰이는 binary LDPC 디코더 HW 전반의 서베이.
2. 배경 문제: 낮은 error floor, 라우팅 혼잡 감소, 작은 면적, 저전력, 다중 code length/rate 재구성 요구를 동시에 만족하기 어렵다.
3. 복호 기본: Sum-Product(SPA)와 저복잡도 Min-Sum, 그리고 scaling factor `S_MS`를 곱한 `MinSum Normalized`(식 1: `α_ij = S_MS × sign곱 × min|β|`), VN 갱신 `β_ij = λ_j + Σα`(식 2), early termination으로 평균 반복/전력 절감.
4. 아키텍처: full-parallel(최고 throughput·큰 면적·라우팅 혼잡) vs partial-parallel(SRAM/주소생성 필요, 규칙적 구조 유리) 비교, 연도·기술노드별 throughput/energy/area/edge 수 추이 제시.
5. 연구동향 1 - error floor: trapping set이 원인, 부호구성 개선·RS/BCH concatenation·2단(post-processing: message biasing/bit flipping) 복호로 저감.
6. 연구동향 2 - 재구성: 랜덤 구조 대신 QC-LDPC(block-structured, `b×b` 순환치환 부분행렬)로 partial-parallel 주소생성 효율화, layer/TDMP overlapped 처리로 throughput 향상.
7. 연구동향 3 - 라우팅 혼잡: `Split-Row`가 PCM을 columnwise `Spn` 분할해 체크노드를 거의 독립 병렬 계산, 분할 간 통신은 sign 1비트뿐(식 3, scale `S_SR`) → 0.3~0.7 dB 손실.
8. `Split-Row Threshold`: 1비트 global threshold 신호로 잃은 min() 정보를 부분 복구, MinSum Normalized 대비 0.07 dB 손실로 대폭 축소하면서 고분할 full-parallel 실현.
9. 결과: 10GBASE-T Spn=16, 65nm CMOS full-parallel 디코더가 195MHz@1.3V에서 평균 92.8 Gbps(early term), 0.7V에서 worst 6.5 Gbps·62mW·9.5 pJ/bit, 면적 4.84mm2 — MinSum Normalized 대비 4.1× 작고 3.3× 빠르며 4.8× 에너지효율, 손실 0.22 dB@BER 1e-9.
10. 한계/전망: 미세공정 global wire 지연 심화로 라우팅이 병목, 스토리지용은 BER 1e-15·>5Gbps 요구 — 새 알고리즘/아키텍처 필요.
