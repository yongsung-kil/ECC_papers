### B. 알고리즘 요약

1. 대상은 (6,32) (2048,1723) regular LDPC, AWGN+BPSK 채널, 표준 2-phase (check node update / variable node update) 메시지 패싱 디코더.
2. 기존 MinSum/SPA 디코더는 큰 row weight(32)에서 check node로 모든 variable node 메시지가 모여야 해 global interconnect 복잡도가 급증, 대형 회로/저속의 원인이 됨.
3. Split-Row는 check node 처리를 `Sp` 개 파티션으로 나눠 각 파티션이 로컬 최소값만으로 독립 연산, 파티션 간엔 최소 정보(1비트 sign 등)만 교환 — 단 0.5~0.7dB 성능 손실 발생.
4. Split-Row Threshold는 파티션마다 로컬 최소값이 임계값 `T` 미만인지 나타내는 1비트 `Threshold_en` 신호를 인접 파티션에 전달, 자기 로컬 최소값과 `T` 중 최솟값을 취해 실제 최소값과의 편차를 줄임.
5. Check node 출력은 로컬 최소값·`Threshold_en`·정규화 인자(0.2~0.4, split 레벨에 따라 경험적으로 결정)로 계산되며, `Threshold_en`이 하나라도 high면 `T`를 사용해 출력 갱신.
6. Multi-Split-Row Threshold는 이 threshold 신호 전달을 임의 개수 파티션(Sp2~Sp16)까지 확장, variable node당 최소 2개 이상만 있으면 파티셔닝 제약이 없음(원 Multi-Split-Row는 특정 split 이상에서 성능 붕괴).
7. HW 구현 디테일: check node 프로세서 critical path는 comparator depth 기반 threshold propagate path(Path1)와 원래 로컬 check/variable path(Path2) 중 worst case로 결정, 파티션 수가 클수록 Path1이 지배적.
8. 별도 학습/최적화 절차는 없고 정규화 인자·threshold 값은 시뮬레이션으로 경험적 튜닝.
9. 결과: Split-16 Threshold는 65nm CMOS에서 3.8mm^2, 100MHz, 15 iteration에서 13.8Gbps, SPA 대비 0.28dB, MinSum Normalized 대비 0.22dB 열화; MinSum Normalized 대비 면적 4.8배 축소, throughput 6배 향상.
10. 한계: 단일 (2048,1723) regular 코드에서만 검증, HW 합성/레이아웃까지 진행했으나 정정능력 자체는 개선하지 않고 오직 HW 면적/속도/interconnect 트레이드오프 개선에 집중.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Split-Row Threshold check node 파티셔닝 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | check node 연산을 파티션 단위로 나누고 threshold 1비트만 교환하는 방식이 CN 업데이트 로직 변형에 참고 가능 |
| MinSum 정규화 인자 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 이미 min-sum 기반이라 정규화 계수 튜닝 방식만 참고 대상 |
| 파티션 간 wire/interconnect 축소 | GT.cpp `Make_GT_HW()` | Graph Thinning과 유사한 edge/wire 축소 목적이나 방식(threshold 신호 전달)은 다름 |
| VN 처리 | (변경 없음, 대응 없음) | 논문 자체가 VN 로직은 불변이라 명시 |

> 적용 가치: 중간 | 이미 min-sum·HCU·GT 등으로 interconnect를 다루고 있어 중복 기여 소지가 크지만, 파티션 간 1비트 threshold 신호 전달 아이디어는 CN 업데이트 로직의 저비용 근사 기법으로 참고할 가치가 있음.
