### B. 알고리즘 요약

1. 시스템: array LDPC 코드(`p`=67, `dv`=5, `dc`=25 예제, rate=1-dv/dc=0.8, length=p·dc=1675), AWGN 채널, BPSK 변조, offset min-sum 기반 layered(turbo) decoding 사용.
2. 문제: 기존 고정-iteration 디코더는 최악 케이스(itmax)를 항상 가정해 클럭/전압을 고정하므로, 대부분 프레임에서 조기 수렴 후 유휴 전력을 낭비함. 기존 fading-channel용 DVFS 예측(check-error 수 기반)은 AWGN 채널에서는 초기 check-error와 필요 iteration 수의 상관관계가 약해 그대로 적용 불가.
3. 핵심 가정: 3번의 sub-iteration을 거치고 나면 남은 check-error 수와 총 필요 iteration 수 사이 상관관계가 회복됨(Fig.4, Eb/N0=3.7dB에서 실증).
4. 핵심 기법: 매 프레임마다 pre-processing unit에서 고정 전압으로 3 sub-iteration을 수행 후 check error 수(`cHT`)를 4개 구간(`cErrThre1~3`)으로 분류, 구간별로 최대 sub-iteration 수 `C[1:4]`(예: 48/32/24/19)를 설정하고 이에 맞춰 디코더 주파수(165/110/82.5/66MHz)를 결정.
5. 핵심 식 의미: `Q_{l,n}=P_n - R_{l,n}`(check node 재사용 시 subtraction), `R_{l,n}=f(Q)` (offset min-sum), layered decoding의 각 sub-iteration은 H matrix의 한 block-row(layer)만 갱신하며 즉시 다음 layer 계산에 반영되어 표준 TPMP 대비 iteration 수 최대 50% 감소(선행연구 [4] 인용).
6. 확장: 결정된 iteration 예산에 맞게 클럭 분주기(clock divider)로 fdec를 낮추고, 가변 전압 공급 회로(variable VDDL, [17] 기반)로 VDD를 1.0~1.5V 범위에서 조정하는 DVFS controller 구성.
7. 구현 디테일: CNU는 compressed check-node message(M1, -M1, M2, -M2 + index)로 메모리 50~80% 절감(선행 아키텍처 [7]-[9] 재사용). pre-processing unit(고정 VDD)과 processing unit(가변 VDDL) 사이에 프레임 버퍼(4-frame) 삽입해 전압 전환 지연(25us/0.5V) 보완.
8. 최적화 절차: check-error threshold `cErrThre[1:3]`, 최대 sub-iteration `C[1:4]`는 특정 SNR(Eb/N0=3.7dB)에서 실험적으로 결정.
9. 결과: TSMC 0.13um 공정, 175MHz@1.5V. DVFS controller 면적은 전체 디코더(1.1mm²)의 약 1%(0.1×0.13mm²), 전력 오버헤드 10mW(디코더 전체 ~200mW 대비). Eb/N0=3.4dB에서 디코딩 에너지 최대 30% 절감, 4.0dB에서도 21% 절감. BER 성능 열화는 0.05dB 이내.
10. 한계: AWGN·array LDPC 특정 코드 파라미터(dv=5,dc=25,p=67)로만 검증. 정정능력이나 error-floor 개선과는 무관하며 순수 저전력 목적. 프레임 버퍼·가변전압 회로 등 아날로그 요소가 추가되어 NAND 컨트롤러 HW에 그대로 이식하려면 검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum layered decoding, check-node/variable-node 처리 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC도 min-sum+layered(Dual-Update) 구조를 이미 보유, 중복 기여 |
| iteration 조기 예측(초기 3 sub-iteration 기반 check-error 분류) 및 클럭/전압 스케일링 | 대응 없음 | Prime ECC는 조기종료(Partial CRC)는 있으나 iteration 예측 기반 DVFS(클럭/전압 동적 조정) 로직은 없음 |
| compressed check-node message(FS: M1/M2/index) 메모리 절감 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 기존 min-sum 메모리 압축 기법과 유사, 이미 보유 개념에 가까움 |
| 조기종료 판정(parity check 만족 여부, xˆHT) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념은 유사하나 본 논문은 CRC가 아닌 parity check 자체로 판정 |

> 적용 가치: 중간 — min-sum/layered decoding 자체는 Prime ECC가 이미 보유한 기법이라 중복이지만, 프레임 초기 iteration 결과로 남은 반복 예산을 예측해 클럭/전압을 동적 조정하는 아이디어는 NAND 컨트롤러의 저전력 설계에 참고 가능(단, SSD ECC는 정정능력이 우선이라 BER 열화 감수 여부는 별도 판단 필요).
