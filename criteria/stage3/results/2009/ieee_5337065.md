### B. 알고리즘 요약

1. LDPC `H=[A|B]`, `(n-k)×n`, rate `R=1-t/n`(`t=n-k`); 자기기록(partial response dicode) 채널, 대표 length ~4096, rate 0.89, GF(2)·GF(4).
2. 문제: parity `p^T=(B^-1)A u^T`는 대형행렬 Galois 역행렬이 존재불확실하고 저장비용 과다 → 실용적이지 않음.
3. 핵심: B를 QC 프로토타입으로 설계 — `s×s` permutation matrix 타일 + 소수 null circulant 삽입으로 full-rank(`gfrank(B)=t`) 보장, block 단위 on-the-fly 해결.
4. 조건: `gfrank(H)=gfrank(B)=t=n-k`, `girth(H)>4`, `t`는 `s`의 정수배 → on-the-fly 인코더 존재 보장.
5. Theorem 1: `gfrank(B)=t`가 on-the-fly 인코더 구성의 필요충분조건(부록 LUP 분해로 증명).
6. 인코딩: `γ=A u^T`를 `t/s`등분, `b_i p^T=γ_i`를 GF 상 순차(forward/backward substitution)로 `s`-bit 블록 단위 해결(사전 프로그래밍 가능).
7. 확장: circulant의 '1'을 `GF(4)` 원소(`α`)로 치환해 finite-field 프로토타입 인코더 구성.
8. 복잡도(Table I): generator 저장 `0~[0,js]`(기존 `k/(s(n-k))` 대비 대폭↓), XOR 연산 `δc̄² R/(1-R)`, 병렬도 `s`로 linear time.
9. 결과: GF(2)/GF(4) 프로토타입 P1·P2가 good regular array code와 거의 동일 성능; Olcer-Eleftheriou upper-triangular는 weight-1 column 탓 error floor. avg column weight>3.8, parity prototype column weight≥2 권장.
10. 한계: HW 미합성(복잡도/병렬도 분석만), 성능은 자기기록 채널 sim(SOVA+turbo eq, 4 outer/25 inner iter, 7-bit LLR)으로만 검증, 인코더 자체는 정정 향상 아닌 "무열화"가 목표.
