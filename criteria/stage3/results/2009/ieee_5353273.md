> 총평: residual 기반 동적 LBP 스케줄(EDS-LBP)로 낮은 복잡도에 error-floor(trapping-set)·수렴 개선을 얻지만, 데이터 의존 순차 스케줄이라 파이프라인 HW 이식성은 낮음.

### B. 알고리즘 요약

1. 이진 [N,M] LDPC, IEEE 802.16e rate `1/2`, block-length `576`/`1152`, BIAWGN 채널·BPSK.
2. 문제: 표준 LBP는 고정 스케줄이고 RBP/NWRBP는 residual 계산 복잡도가 큼 → 낮은 복잡도로 수렴·error 성능 개선.
3. 모델: BP 메시지(V→C 식1, C→V 식2 full-SP `2artanh(∏tanh)`, 판정 식3), residual=갱신 전후 메시지 차.
4. 핵심 기법: EDS-LBP — 각 동적 iteration에서 최대 상대 residual `r(mk)`를 갖는 변수노드에 연결된 check node들만 선택해 LBP 수행.
5. 핵심 식: `r(mk)=(f(mk)−mk)/(f(mk)+mk)` (f=갱신 후 LLR, mk=갱신 전 LLR); 큰 residual=미수렴 → 먼저 전파하면 수렴 가속.
6. lazy schedule(신뢰도 높은 노드 갱신 스킵) + NWRBP(check node 출력 동시 생성) 특성 결합, extra 복잡도는 NWRBP보다 훨씬 낮음.
7. 절차: 모든 `r(mj)` 계산 → `argmax` 변수노드 선택 → 이웃 check node 메시지 생성·전파·residual 갱신 → 선택 노드 residual 0 → 정지규칙까지 반복.
8. RBP의 trapping-set 정복 능력을 medium~high SNR에서 활용 → error floor 개선.
9. 결과: 802.16e 576/1152, max iter 5~30, EDS-LBP가 표준 LBP·NWRBP 대비 수렴이 빠르고 medium~high SNR FER를 크게 개선.
10. 한계: HW 미설계·시뮬만; 데이터 의존 순차 스케줄로 파이프라인 HW 부적합; 짧은 802.16e 부호만 검증.

| C→V full-SP(tanh) | 대응 없음 | Prime ECC는 min-sum, full-`tanh` Sum-Product 미사용 |
| trapping-set/error-floor 정복 | `corner.cpp` (수렴실패 추적) | 개념적 연관, 직접 대응 낮음 |

적용 가치: **낮음~중간** — error-floor(trapping-set) 개선 아이디어는 참고가치가 있으나, min-sum이 아닌 SP 기반이고 데이터 의존 순차 스케줄이라 Prime ECC의 z=32 파이프라인 layered HW에 직접 이식이 어렵다.
