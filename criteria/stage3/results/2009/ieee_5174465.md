### B. 알고리즘 요약

1. DVB-S2 표준 IRA-LDPC 부호(N=64800, 11개 rate 1/4~9/10) 디코더 HW 설계, parity check matrix H=[A|B] (A: 정보비트용 sparse, B: staircase lower-triangular).
2. 큰 블록길이(N=64800)에서 fully-parallel 디코더는 비현실적이므로, partially-parallel이면서 메모리 충돌/병목을 줄이는 구조가 필요.
3. 표준 인코딩 규칙 `p_j = p_j ⊕ i_m`, `j = {x+q×(m mod 360)} mod (N-K)`에 의해 matrix A가 주기 M=360의 cyclic-shift 패턴을 가짐(핵심 관찰).
4. 이 주기성을 이용해 동일 인덱스로 연결된 360개 노드를 하나의 그룹으로 클러스터링, 그룹당 1개 functional module(BFM/CFM) 배정(rate 1/2 예시: 90 BFM, 90 CFM).
5. 노드 클러스터링으로 K×(N-K)개의 개별 노드 대신 90+90개 functional module만 필요해 하드웨어 재사용 극대화.
6. 공유 메모리 2모듈(check-to-bit, bit-to-check) 각 90 bank 구성, cyclic-shift는 ROM 주소 증분으로 구현(뱅크당 자료 위치를 인덱스 시프트로 계산).
7. BFM 입력 수(8 또는 3)와 메모리 뱅크 출력 수(5)의 불일치를 해소하기 위해 전용 라인 450개의 셔플링 네트워크를 메모리-FM 사이에 배치.
8. 학습/최적화 절차 없음 (구조적 HW 설계).
9. 결과: 90nm 합성 기준 면적 4.76mm², throughput 277Mbps로 기존 대비([11]) 면적 -52%, throughput +9%.
10. 한계: 정정성능(BER/FER) 결과 없이 순수 면적·처리량만 보고, 디코딩 알고리즘(min-sum/SPA, 양자화) 세부 미기재, DVB-S2 특정 M=360 주기 구조에 종속된 설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 노드 클러스터링/공유 functional module | `decoder.cpp` `LDPC_Decoder()` | 병렬화 단위(z=32) 설계와 유사한 아이디어이나 큰 N(64800)·M=360 전용 구조라 직접 이식 어려움 |
| 공유 메모리 뱅크 + 셔플링 네트워크 | 디코더 HW 모델 (z=32, HCU, GT) | 메모리 충돌 회피 아이디어는 참고 가능하나 우리 구조(row 단위 병렬)와 매핑 재검증 필요 |
| H-matrix cyclic-shift 주기성 활용 | `ecc_top.cpp` `Load_PCM()` | 부호구조가 DVB-S2 고정, QC-LDPC 일반 개념과는 부합하나 특정 매크로 구조 이식은 높은 난이도 |
