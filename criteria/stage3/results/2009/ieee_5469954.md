### B. 알고리즘 요약

1. 시스템: WiMax rate-1/2 2304-bit QC-LDPC(dual-diagonal parity)를 mother code로 채택, mother rate와 1 사이 임의 rate 지원.
2. 문제: 여러 rate별 전용 부호를 모두 저장하면 메모리·스위칭망 부담 큼 → 하나의 mother code puncturing으로 rate-compatible화.
3. 이론 기반: `k-SR`(k-step recoverable) variable node — punctured node가 BEC에서 k번째 iteration에 복구, `k` 작을수록 신뢰↑.
4. puncturing 절차: (1) punctured block 선택 후 (2) block 내 bit 선택. 1-SR group `G1`부터 최대화, 이후 `G2`... 순으로 저인덱스 group 우선 puncture.
5. block 선택 핵심: survived-check 수가 많을수록 성능↑ → weight-3 vertical submatrix를 puncturing block에 포함(scheme 3)하면 이웃 check가 많아 성능 유리.
6. dual-diagonal zigzag 구조 덕분에 96-bit 단위 12 block의 k-SR 그룹화가 용이, rate 12/23~1까지 표(Table II)로 punctured block index 지정.
7. 디코더: Parallel Layered Decoding Architecture(PLDA) — layer마다 전용 CNU, 모든 layer 동시 처리·layer 내 serial, permutation 값 내림차순으로 message 전달.
8. HW 트릭: layer 간 permutation 차를 offset으로 조정해 최소 5 clock 확보 → CNU를 4-stage pipeline split, critical path 단축·최대주파수↑. puncturing은 punctured parity LLR을 0으로 patch(LLR initialization).
9. 결과: punctured code가 dedicated WiMax code 대비 BER 10^-5에서 <0.2dB 손실. FPGA(XC2V8000) 최대 61.3MHz, 28Mbps@25iter / 70Mbps@10iter, slices 63%·FF 73%·LUT 60%·RAM 30% 절감.
10. 한계: FPGA 검증뿐(ASIC 미완), soft-read 양자화 비트 미명시, rate-compat 목적이 시변 무선채널용이라 NAND와 직접 정합 낮음.
