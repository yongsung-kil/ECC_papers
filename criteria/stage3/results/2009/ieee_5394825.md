### B. 알고리즘 요약

1. 대상/채널: BSC 상 iterative 복호(Gallager A/B, bit-flipping), column-weight-three girth-8 및 (3,6) regular LDPC(예: Margulis length 2640), error-floor 영역.
2. 문제: 유한길이 LDPC의 error floor는 trapping set 등 조합 구조에서 기인하나 BSC/AWGNC에서의 특성화가 부분적으로만 이해됨.
3. 정의: `(a,b)` trapping set = 최종적으로 정정되지 않는 변수노드 `a`개와 유도부분그래프의 홀수차수 체크노드 `b`개; critical number `m(T)` = 복호기가 T에 빠지는 데 필요한 최소 초기 오류 변수노드 수.
4. 핵심 개념: critical number가 작을수록(그리고 inducing set이 많을수록) trapping set이 더 harmful — error-floor slope에 직결.
5. 표현: trapping set을 line-and-point incidence structure(변수노드=line, 홀수차수 체크노드=black point)로 표현해 graph coloring 문제와 연결.
6. Ontology: subgraph isomorphism 기반 parent-child 관계로 trapping set DB 구성 — child가 parent보다 harmful; `(5,3)` 회피만으로 자식 `(6,2)/(6,0)/(7,1)` 동시 회피.
7. 열거: g-cycle을 `(a,a)` parent로 두고 tree-based 탐색 → trapping set 열거 복잡도가 코드길이 `n`에 선형(예: `(4,4)` 탐색 `300n`).
8. 응용1(예측): dominant trapping set의 cardinality를 세어 error floor를 semi-analytic 예측(Margulis 2640 (3,6) BSC 예측이 시뮬과 정합).
9. 응용2(구성): critical number 낮은 harmful TS를 PEG 수정으로 회피 → FER slope `3→4` 개선, 3-error 정정 보증(Tanner graph cover로 TS 제거도 가능).
10. 한계: HW 미설계, BSC/Gallager-A·column-weight-three 중심, 타 복호/채널로의 ontology 확장은 future work.
