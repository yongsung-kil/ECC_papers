> 총평: rate-compatible punctured irregular LDPC(E2RC) 부호의 EXIT chart 기반 설계·분석 논문으로, 순수 부호구성 이론 기여이며 NAND 고정rate binary QC-LDPC 이식과는 목적·부호구조가 달라 적용가치는 낮음.

### B. 알고리즘 요약

1. 대상은 E2RC(Efficiently-Encodable Rate-Compatible) irregular LDPC 부호, `H=[H1|H2]`에서 H2(parity part)는 결정론적 lower-triangular 구조(degree-1/2 노드), H1(systematic part)은 랜덤 인터리버로 연결된 비구조 부분. BIAWGN 채널.
2. 풀려는 문제: 기존 E2RC 설계는 H2 구조를 고려하지 않고 degree distribution을 정했는데, 본 논문은 H2 구조까지 반영해 최적 degree distribution을 찾고, 여러 rate에서 동시에 좋은 성능(공동 최적화)을 내는 rate-compatible 부호를 설계.
3. 핵심 가정: log-domain BP 디코딩에서 메시지가 Gaussian이라 가정(EXIT chart 표준 가정), `IE,v(IA,v) = J(√((dv-1)[J^-1(IA,v)]^2 + σ_ch,v^2))`.
4. 핵심 기법 1단: 구조화 parity부(H2, protograph 표현)의 EXIT 함수를 Monte Carlo 시뮬 없이 고정점 반복(check node update Eq.4 ↔ variable node update Eq.3)으로 해석적으로 계산 - `IE_ci,e`, `IE_vi,e` 수렴까지 반복(threshold=1e-6).
5. 핵심 식 의미: 기존엔 EXIT 곡선 한 점당 대규모 Monte Carlo가 필요해 시간이 많이 걸렸는데(예: IRA 코드 AWGN 175886초), 제안 고정점 반복법은 동일 정확도(평균오차 <2.7e-3)를 0.5~3.7초로 단축.
6. 핵심 기법 2단: 계산된 구조화부 EXIT 함수를 이용해 비구조부(H1) degree distribution `{λ_dv}`를 capacity gap 최소화 선형계획법(LP)으로 최적화(zero-error 제약: `IE,unS > IA,S` 전 구간).
7. 확장: 여러 code rate `R`(펑처링 레벨)에 대해 동시에 capacity gap을 최소화하는 joint optimization 알고리즘(g를 점증시키며 LP 반복) 제안 - 기존 문헌에 없던 시도라 주장.
8. 최적화 절차: LP를 이산화된 `IA`에 대해 풀고, 역함수 `IA,S(IE,S)`는 선형보간으로 근사(정확도 위해 (IA,IE) 쌍 1만개 필요).
9. 결과: 설계 코드(block length 16384)는 원 E2RC 대비 rate 0.5~0.89 전 구간에서 capacity gap을 균일하게 0.3dB 이하로 유지(joint-optimized code 2는 0.25~0.30dB, 기존 code 1은 0.21~0.41dB로 불균일).
10. 한계: 순수 이론/시뮬 결과뿐(HW 미설계), rate-compatible puncturing이 목적이라 단일 고정 rate ECC에는 직접 적용 어려움, 코드 구조 자체가 degree-1/2 parity node를 포함하는 IRA류 protograph로 QC-LDPC와 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| E2RC/IRA 부호구조 (H2 lower-triangular, degree-1/2 parity) | 대응 없음 | Prime ECC는 QC-LDPC(base+circulant shift) 구조이며 E2RC/IRA류 구조와 무관 |
| H-matrix/degree distribution 설계 | `ecc_top.cpp` `Load_PCM()` | 개념적으로만 관련(부호 로드 위치) - 실제 degree distribution 최적화 로직은 Prime ECC에 없음 |
| rate-compatible puncturing | 대응 없음 | Prime ECC는 고정 H-matrix(matrix_sel) 기반이며 가변 rate puncturing 체계 없음 |
| EXIT chart 기반 해석적 설계 도구 | 대응 없음 | 오프라인 설계 단계 도구로, 런타임 코드베이스에는 대응 모듈 없음 |

> 적용 가치: 낮음 — rate-compatible punctured irregular LDPC 설계 이론으로, Prime ECC 3.1의 고정 rate binary QC-LDPC 구조·목적과 근본적으로 다르며 코드 재사용 여지가 거의 없음.
