### B. 알고리즘 요약

1. 시스템: 이진 LDPC, BSC(binary symmetric channel) 채널, 길이 1000~1908, rate 1/2~8/9(QC-LDPC 포함) 다양한 코드로 검증.
2. 문제: MILP 디코더는 ML에 근접하나 매우 느림, BP는 빠르지만 error-floor에서 성능 열화 - 둘을 결합해 평균은 빠르고 성능은 강하게.
3. 구조: Algorithm E(quantized BP, 1단) → 실패 시 standard BP(2단) → 실패 시 MILP(3단, 이진 제약 순차 추가)의 E-BP-MILP 파이프라인, 앞단이 valid codeword를 못 찾을 때만 다음 단 활성화.
4. Algorithm E 핵심식: check-to-variable `u_mn^(i) = Π v_m'n^(i-1)`(부호 곱), variable belief `v_n^(i) = sgn(w^(i)·y_n + Σu_m'n^(i))`, 메시지값은 {-1,0,+1} 양자화.
5. 식 의미: 메시지를 3-level로 양자화해 real-valued BP 대비 연산을 극도로 단순화(HW min-sum보다도 더 단순한 부호기반 sgn 연산).
6. 확장(active-set Algorithm E): 이전 iteration에서 메시지가 변하지 않은 노드는 업데이트를 skip, active set(A_v, A_c)만 갱신 - 낮은 crossover probability에서 대부분 메시지가 불변이므로 계산량 급감.
7. MILP 단: LP relaxation의 non-binary pseudo-codeword에 대해 0.5에 가장 가까운 심볼에 이진 제약 `b_n*∈{0,1}` 순차 추가, GLPK 솔버(Python)로 재실행.
8. 학습/최적화 없음, 대신 처리시간 모델 `T_E(1+WER_E·T_BP/T_E+WER_(E-BP)·T_MILP/T_E)`로 평균 throughput 근사.
9. 결과: length-1908 rate-8/9 코드에서 crossover prob 0.001 시 E-BP-MILP(10)이 BP 대비 WER 4자릿수 개선, WER 10^-10까지 시뮬 가능; active-set은 표준 Algorithm E 대비 update 수 최대 약 15배 감소(check updates 222→15).
10. 한계: MILP은 Python+GLPK 소프트웨어 솔버 기반으로 순수 시뮬레이션이며 HW 미설계, 최악 케이스 처리시간은 MILP 단에 의해 좌우되어 실시간 시스템엔 부적합.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| E-BP-MILP 다단 폴백 구조 | `decoder.cpp` `LDPC_Decoder()` | 조기종료 후 fallback 디코더로 확장하는 개념은 있으나 MILP 자체는 HW 부적합 |
| Algorithm E (3-level 양자화 BP) | `decoder.cpp` `VN_Cal_HD()` | hard-decision 계열 quantized 메시지 처리 개념과 유사하나 Prime ECC는 min-sum 기반이라 직접 대응 아님 |
| active-set 업데이트 스킵 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 저 BER에서 미변경 노드 스킵 아이디어는 min-sum 루프에 조건부 최적화로 이식 가능성 있음(난이도 중) |
| MILP 이진 제약 추가 (ML 근접) | 대응 없음 | 소프트웨어 LP 솔버 기반으로 실시간 HW 디코더와 이질적 |

> 적용 가치: 중간 — MILP 단 자체는 이식 불가하나, active-set 방식의 미변경 노드 업데이트 스킵은 저 BER 구간에서 Prime ECC 디코더 이터레이션 루프의 동적 전력/연산 절감에 참고할 만함.
