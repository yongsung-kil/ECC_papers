### B. 알고리즘 요약

1. 목표는 임의의 (N,M) LDPC 코드를 지원하는 유연한 멀티프로세서 LDPC 디코더를 Network-on-Chip(NOC) 인터커넥트로 구축하는 것이다.
2. 기존 NOC 기반 LDPC 디코더([12])는 면적 오버헤드가 크고 N=1024 정도의 중소형 코드만 지원 가능했던 한계가 있었다.
3. 디코딩은 표준 Two Phases Message Passing(TPMP), min-sum 근사(`Ri[j] = δij · min|Qβ[i]|`)를 사용하며 노드 연산과 네트워크 전달이 교대로 이루어진다.
4. 각 Processing Element(PE, 총 P=16)는 VN/CN 역할을 겸하고, VN/CN을 P개 PE에 분배하는 문제는 그래프 파티셔닝(cutset 최소화)으로 formalize된다.
5. `C = {eij ∈ E | VNi∈Pm, CNj∈Pn, m≠n}`, cutset 크기 최소화가 네트워크 전송 메시지 수를 최소화하며, METIS 기반 GP 방식이 Ordered/Random 대비 약 50% 메시지 절감을 달성한다.
6. 두 NOC 아키텍처를 비교: binary de Bruijn(온라인 동적 라우팅, deflection routing, 최대 latency log2N)와 ZONOC(오프라인 정적 라우팅, Control Memory에 사전계산된 라우팅 저장, payload만 전송해 오버헤드 제거).
7. 부수 구현: 메시지는 6-bit로 표현, PE는 클록당 최대 8메시지(2개 degree-4 VN + 1개 degree-8 CN) 처리.
8. 별도 학습 절차 없음(구조적/휴리스틱 설계, GP는 METIS 툴 사용).
9. ZONOC 802.16e 최대 코드 매핑 시 125kgates 로직, 3.7mm²@130nm, net throughput 56Mbps(10 iteration), de Bruijn 90nm에서 0.65mm² 면적으로 기존 NOC 기반 디코더([12]) 대비 NTAR 10배 이상 개선.
10. 한계: net throughput이 48~63Mbps 수준으로 낮고(코드 특성 미활용), 순수 유연성(다중 코드 지원)에 초점을 둔 아키텍처라 특정 코드 최적화 디코더 대비 성능 열위이며 NAND ECC급 Gbps throughput과는 거리가 멈.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 멀티코드 유연 NOC 인터커넥트 | 대응 없음 | Prime ECC는 고정 단일 QC-LDPC(z=32 row-parallel) 구조로 코드 간 재구성 개념이 없음 |
| PE 간 메시지 라우팅/그래프 파티셔닝 | `decoder.cpp` `LDPC_Decoder()` (파이프라인/HCU 구조) | 개념적으로 병렬 메시지 전달과 관련되나 Prime ECC는 고정 z=32 row-parallel 구조로 범용 NOC 라우팅이 불필요 |
| min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()` | 표준 min-sum으로 이미 보유, 중복 개념 |

> 적용 가치: 낮음 — 다중 LDPC 코드 지원용 재구성 가능 인터커넥트가 핵심이나, Prime ECC는 특정 QC-LDPC 하나에 최적화된 고정 HW 구조(z=32, HCU, GT)를 쓰므로 이식 근거가 약함.
