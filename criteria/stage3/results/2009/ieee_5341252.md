### B. 알고리즘 요약

1. 부호는 modified array LDPC(MAC, Eleftheriou-Olcer)와 interleaved modified array LDPC(IMAC, Singhaudom)를 기반으로 한 quasi-cyclic 구조의 irregular LDPC.
2. 기존 array 계열 구성법은 상삼각 부분에 "1"이 많아 인코딩이 복잡하거나, block length가 길 때만 성능이 좋다는 한계를 지적.
3. 채널은 AWGN, 디코더는 표준 Log-domain Sum-Product(BP) 5단계 알고리즘 사용 (교과서적 `L(qij)`, `L(rji)` 갱신식).
4. 제안 기법은 MAC/IMAC 행렬에 matrix transposition과 row/column interchanging을 적용해 상삼각의 "1" 개수를 줄이고 cycle-4를 회피.
5. 결과 행렬은 `H = [I2 α3; 0 α α]` (식 16) 형태로 단순화되어 인코딩 복잡도 감소를 의도.
6. j(column weight), k(row-related), p(circulant 크기) 파라미터 조합으로 short(c=255~522)/medium(c=1147~2021)/long(c=4087~4183) block 세 그룹 실험.
7. rate는 `R=1-(j/k)`로 계산되며 medium block에서 최대 0.930까지 달성.
8. 최적화/학습 절차 없음 (수작업 행렬 변환 규칙만 제시).
9. medium block(c=2021, R=0.930)에서 기존 QC-LDPC(Rakibul et al., R=0.5)보다 낮은 rate 코드 대비 약간 열세하나 rate가 거의 2배 높음. iteration 5/10/20 비교 시 10회 이후 개선 미미.
10. long block(4096bit)에서는 IMAC 대비 성능이 열세하며, HW 설계·양자화·복잡도 분석이 전혀 없는 순수 시뮬레이션 논문.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Parity check matrix 구성법 (array/IMAC 변형) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC 고정이라 새 H-matrix 구성법 이식 난이도 높음 |
| Log-domain Sum-Product 디코더 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 이미 Min-Sum 사용, 표준 SP는 중복·퇴보 기법 |

> 적용 가치: 낮음 — 표준 array-LDPC 구성법 변형과 교과서적 BP 디코더뿐이며 NAND/HW 요소가 전혀 없어 이식 가치가 낮음.
