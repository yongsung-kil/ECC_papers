### B. 알고리즘 요약

1. 대상 코드는 IEEE 802.3an 10GBase-T 표준의 (2048,1723) rate≈0.84 QC-LDPC 코드.
2. 기존 G-matrix 곱셈법은 저장공간(ROM)이 크고, RU(Richardson-Urbanke)법은 critical path가 길고 column swap 시 비시스터매틱이 되는 문제가 있음.
3. H-matrix를 approximate lower triangular form(`[A B T; C D E]`)으로 변환 후, `p1`(패리티 첫 부분)은 G-matrix 곱으로, `p2`는 RU의 back-substitution으로 계산하는 Hybrid 구조를 제안.
4. `p1 = -φ^{-1}(-CA^{-1}s + Es)` 식은 G-matrix로 미리 계산해두어 T가 singular해도 역행렬 불필요.
5. `p2`는 T가 하삼각(lower triangular)이므로 back-substitution으로 순차 계산 (`Ap1 + Bp2 + Cs = 0` 형태의 해).
6. L-parallel back-substitution 시 critical path가 L에 비례해 증가하는 문제를 look-ahead 기법(수식 재조합, 1비트 추가 저장)으로 완화.
7. ROM 구현을 PLA(NAND-NOR)식과 shift-register식 두 가지로 비교, shift-register가 병렬도가 높을 때 전력 효율이 더 좋음을 분석.
8. 별도 학습/최적화 절차 없음 (고정 H-matrix 기반 아키텍처 설계).
9. 결과: G-matrix법 대비 게이트 수 74%, ROM 저장량 10%(추가 HW로 52488bit까지 축소 가능), 전력 약 82% 절감, critical path는 유사~소폭 개선.
10. 한계: 특정 (2048,1723) 코드에 특화된 사례 연구이며 65-90nm ASIC 실제 합성 결과는 미확보(분석적 전력 모델만 사용), 디코더 설계는 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Hybrid(G-matrix+RU) 인코딩 구조 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코더를 사용하며 본 논문의 G/RU 혼합 back-substitution 구조와는 다른 방식 |
| H-matrix lower-triangular 변환, column swap 회피 | `ecc_top.cpp` `Load_PCM()` | 부호 PCM 구조가 이미 고정되어 있어 이식 난이도 높음 |
| ROM(PLA/shift-register) 저장구조 최적화 | 대응 없음 | Prime ECC 시뮬레이터는 클록레벨 bit-exact 모델이나 인코더 ROM 저장구조를 별도 최적화 대상으로 다루지 않음 |

적용 가치: `낮음` — 순수 인코더 면적/전력 최적화이며 디코더 정정성능과 무관하고, Prime ECC의 인코더 구조(dual-diagonal, 고정 H-matrix)와 접근 방식이 달라 이식 난이도(§4 "인코더: 중~높음")가 큼.
