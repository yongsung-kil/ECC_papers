### B. 알고리즘 요약

1. 대상은 (6,32)-regular (2048,1723) RS-LDPC 및 (5,47)-regular (2209,1978) array-based LDPC, binary-input AWGN, sum-product/min-sum message passing을 layered 구조 4/6/8-bit wordlength로 FPGA(Xilinx Virtex-II Pro, BEE2) 구현.
2. 문제: 실용 LDPC 디코더는 낮은 BER에서 error floor가 존재하고, SW 시뮬은 속도 부족으로 error floor 영역을 관측·분석하기 어려움.
3. 채널 모델: AWGN, all-zero codeword 가정, BPSK({0,1}→{1,-1}); prior LLR 통계를 수집.
4. 핵심 기법(HW emulation): FPGA로 SW 대비 수 자릿수 빠르게 복호(480 Mb/s@100MHz)해 실패 시 noise realization과 iteration별 soft-decision을 block RAM에 기록, Matlab reference로 재현·분석.
5. error 분류: implementation-induced oscillation error(양자화 동역학) vs code-intrinsic absorbing error(local minimum) — 후자가 error floor 지배.
6. absorbing set 정의: `(a,b)` set은 크기 a의 VN 부분집합 D로 odd-degree 이웃 check가 b개, 각 원소가 O(D)보다 F\O(D)에 더 많이 연결; 저weight·고 a/b(안정) set이 error floor 지배.
7. 개선 트릭 1: two-domain 양자화(HW 증가 없음)로 dominant absorbing set을 (4,8)/(5,9)→(8,6)/(8,8) 등 고weight로 이동시켜 floor 완화; min-sum 근사도 saturation 회피로 floor 개선.
8. 개선 트릭 2(post-processing): absorbing set 비트는 satisfied check 2개(오검출)+unsatisfied 1개에 연결 — satisfied check 메시지 신뢰도↓, unsatisfied↑로 작은 섭동을 주어 비트 복구, prior LLR 분포(중심이 0 약간 아래)가 소량 섭동으로 충분함을 확인.
9. 결과: (2048,1723)에서 4-bit modified min-sum baseline의 error floor가 post-processing(10 iter) 후 제거, 실패당 평균 error weight가 고SNR에서 8로 수렴((8,8) absorbing set 지배 확인), 참고 chip[18]은 post-processing 포함 47 Gb/s near-ML.
10. 한계: post-processing 상세 규칙/파라미터는 [17], 정량 gate/ASIC 없음(FPGA resource만), 대상 부호가 NAND QC-LDPC와 다른 통신·스토리지용 구조.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| post-processing(satisfied/unsatisfied check 메시지 섭동) | decoder.cpp LDPC_Decoder() | 이터레이션 루프 뒤 error-floor 대책 후처리로 삽입 가능, min-sum 메시지 신뢰도 조정 |
| CN 메시지 신뢰도 조정(min-sum) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal() | satisfied/unsatisfied check magnitude 섭동을 CN 연산에서 구현 |
| two-domain 양자화 개선 | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | 양자화/LLR 테이블 재설계로 absorbing set 이동(단 Prime ECC는 2~3bit) |
| absorbing set/수렴실패 error 포착·분석 | corner.cpp | 수렴실패 추적과 개념적 대응(error trace/absorbing set 통계) |
| FPGA HW emulation 플랫폼(BEE2) | 대응 없음 | Prime ECC는 C++ bit-exact 시뮬로 FPGA emulation 층과 상이 |
```

적용 가치: 높음 — absorbing set 기반 error-floor 분석과 check 메시지 섭동 post-processing은 Prime ECC min-sum 디코더의 error floor를 낮추는 데 직접 관계된 NAND ECC 핵심 관심사이나, 정확한 섭동 규칙·파라미터는 [17]에서 보완 필요.
