### B. 알고리즘 요약

1. 대상: (N=504, M=252) regular LDPC, `dv=3`, `dc=6`, rate 1/2, BPSK, offset BP(`α=1.15`), 최대 100 iteration.
2. 문제: PEG는 girth(최단 cycle 길이)만 키우고 최단 cycle의 **개수**는 무시 → 개수가 많으면 성능 열화.
3. 핵심 개념 `Polynomial of Cycle(PC)`: Tanner graph를 특정 variable node에서 트리로 전개, 각 node의 PC를 재귀 계산해 cycle의 길이·개수·분포를 다항식으로 표현.
4. PC 규칙: vertex의 초기 PC=1, son node PC = (부모 PC 합)×`x`. node가 다시 자기 자신으로 나타나면 PC = `hk·x^k` → local girth `k`, 최단 cycle 개수 `hk`.
5. 의미: PC의 지수 = 최단 cycle 길이, 계수 = 그 개수 → 두 지표를 동시에 정량 비교 가능(PEG는 개수 구분 불가).
6. PEGPC 절차: PEG처럼 edge를 순차 추가하되, 새 edge가 만드는 cycle의 (1) 길이 최대화(PC 지수 최대 check 선택) 후 동률이면 (2) 개수 최소화(PC 계수 최소 check 선택).
7. 예시: 후보 check c1,c4,c5의 PC가 3x^5,3x^5,2x^5 → 지수 동일(길이 6)이나 계수 최소인 c5 선택(길이-6 cycle 3개 대신 2개).
8. 결과(girth): PEGPC/PEG 모두 girth 8, 평균 local girth 8.079 vs 8.004 — girth 자체 개선은 미미.
9. 결과(cycle 개수·BER): 최단 cycle 수 844→398(52.8%↓), BER 10^-5에서 PEG 대비 0.1dB·random 대비 0.45dB 개선, 고SNR 영역에서 특히 우수.
10. 한계: HW 미설계·시뮬만, non-QC 비구조 regular 단부호, 고rate/QC 구조 미검증.
