### B. 알고리즘 요약

1. 대상: LDPC 반복복호에서 메시지 전달 순서(schedule)가 수렴속도에 미치는 영향을 분석, protograph(lifted graph) 기반 코드에 적용 가능한 일반화된 serial 스케줄 제안.
2. 문제: 표준 flooding 스케줄은 비효율적(가장 느린 스케줄)이며, 기존 serial-V 스케줄은 flooding 대비 2배 가속에 그침. 더 큰 가속을 얻는 구조적 스케줄 설계가 목표.
3. 핵심 가정: 코드는 protograph를 `Z`배 lifting해 생성(QC 유사 구조), 그래프는 depth-d subtree로 분할 가능하도록 특별히 설계됨.
4. 핵심 기법: Serial-Vd 스케줄 - 각 변수노드 v에 대해 거리 `d` 이내의 layers graph `L(G,v,d)`를 구성하고, 그 서브트리에 대해 최적 tree schedule을 순서대로 이어붙임 (`d=1`이면 기존 serial-V와 동일).
5. 핵심 식: 조합론적 가속인자 `η_l^(σ) = E[D_flood(l)] / E[D^(σ)(l)]`(computation tree 깊이 비율)이며, `η_0 = E[V]`(서브트리 내 경로 길이 기대값)로 수렴. 평균 노드 차수가 작을수록 가속인자가 커짐.
6. 확장: 확률론적 가속인자를 Density Evolution(DE)으로 별도 계산해, 조합론적 예측과 실제 수렴속도(반복횟수) 간 일치를 검증(Conjecture III-.5, 임계값 근처에서 두 값 수렴).
7. 부수 디테일: Serial-Vd를 지원하려면 depth-d subtree partition이 가능한 특수 protograph 구조가 필요 (일반 임의 그래프는 d=1만 가능).
8. 최적화 절차: DE 알고리즘을 lifted graph 앙상블에 적용해 (3,4)-regular 등 여러 앙상블의 점근적 가속인자 한계를 계산.
9. 결과: length 44032, rate 1/4 (3,4)-regular LDPC(128-lifting)에서 BEC 상 Serial-V1은 약 2배, Serial-V6는 약 3.1배 가속(flooding 대비), 이론 예측과 시뮬레이션 일치. 저차수 irregular 앙상블일수록 더 큰 가속 잠재.
10. 한계: 분석은 주로 BEC(erasure channel)+BP 기준이며 HW 설계·gatecount·soft quantization 논의 없음. protograph 특화 구조 요구로 임의 QC-LDPC 부호에 직접 적용 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 스케줄링(메시지 전달 순서) 전체 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 내 노드 갱신 순서 제어와 직접 관련, Dual-Update 스케줄과 비교 대상 |
| Dual-Update(even/odd 교대) 스케줄 | `decoder.cpp` `C2V_Cal()` | 이미 구현된 스케줄링 옵션과 개념적으로 유사(다른 형태의 layered/serial 스케줄) - 중복 기여 가능성 |
| iteration별 수렴 가속 분석(DE) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration 수 감소가 LLR 테이블 크기·조기종료 시점에 영향 |
| protograph/lifted graph 구조 설계 | `ecc_top.cpp` `Load_PCM()` | 특정 depth-d subtree partition 요구 구조는 현재 H-matrix와 별개 설계 필요 |

> 적용 가치: 중간 — iteration/latency 감소라는 Prime ECC 관심 영역과 직결되나, protograph 특화 구조 요구와 BEC/BP 전제 분석이라는 제약으로 min-sum HW 스케줄에 바로 이식하기보다 개념 검증 후 재설계가 필요.
