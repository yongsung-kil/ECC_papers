### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 3GPP HSPA/LTE 등 무선 incremental-redundancy HARQ를 위한 protograph 기반 구조적 LDPC 코드 패밀리, 정보블록크기 고정(1k/4k payload)으로 rate 0.1~0.9 범위, AWGN 채널과 binary erasure channel(BEC) 동시 고려.
2. 풀려는 문제: 기존 rate-compatible 구성법은 정보블록 크기가 가변이거나(HARQ 부적합), structured puncturing 시 stopping set이 생겨 반복복호가 시작조차 안 되는 문제, random puncturing은 고rate에서 심한 error floor 발생.
3. 핵심 가정: HARQ 재전송 특성상 초기 전송은 고rate 코드(early termination 목표)이고 재전송이 누적될수록 저rate가 되므로, 하나의 저rate base code에서 상위rate 코드들을 모두 punctured version으로 nest시켜야 단일 디코더로 전체 패밀리를 처리 가능.
4. 핵심 기법 1단(base code 설계): rate-1/2 ARCA(accumulate repeat check accumulate) protograph `B`를 `α`번 복제 후 PEG(progressive edge growth) 알고리즘으로 girth 최대화한 `B_α`를 만들고, `β = αK(R_B-R_L)/(R_B(1-R_L))`개의 입력 노드를 pruning해 목표 rate `R_L` protograph `L`을 구성.
5. 핵심 식의 의미: girth가 클수록 stopping distance가 커져 puncturing에 강건한 base code가 되며, systematic input node pruning(information shortening)으로 저rate 코드를 상위rate 코드와 동일 encoder로 생성 가능.
6. 핵심 기법 2단(puncturing): Progressive Node Puncturing(PNP, Algorithm 2) — 각 punctured variable node 후보에 대해 이웃 check의 "punctured neighbor 개수"(`F(c)`), `G(v)`(단일-puncture 이웃 check 수), `H(v)`(이웃 check의 punctured 총합)를 계산해 A) 단일-puncture check를 최대화, B) 다중-puncture check 연결을 최소화하는 순서로 노드를 한 번에 하나씩 선택 천공.
7. 부수 트릭: puncturing score `S(c)=Σ F(c')` (2-hop support tree 내 punctured 이웃 총합)로 여러 랜덤 시드 결과 중 최적 패턴 선택(criterion C), structured(protograph-level) puncturing은 stopping set 유발로 배제.
8. 학습/최적화 절차: girth·threshold 최적화는 density evolution(Gaussian approximation, Chung's DE) 기반 asymptotic 분석 + 다수 랜덤 시드 중 최선 선택(simulated annealing 가능 언급).
9. 결과: rate 1/2~1/6 base code가 AWGN capacity 대비 0.14~0.7dB 이내, PNP로 구성한 rate 0.8 코드가 capacity 대비 0.86dB(3GPP turbo 코드 대비 FER 10^-2에서 약 0.3dB 이득 및 더 낮은 error floor) vs random puncturing은 2.4dB 열위.
10. 한계: HW 설계 없이 시뮬레이션(AWGN/BEC, density evolution)만 수행되었고, 단일 고정 rate가 아닌 nested rate-compatible 패밀리(HARQ 재전송) 자체가 목적이라 NAND ECC의 고정 고rate 요구와 응용 성격이 다름.

