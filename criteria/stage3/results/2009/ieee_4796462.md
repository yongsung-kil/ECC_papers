### B. 알고리즘 요약

1. Mobile WiMAX 19-mode QC-LDPC 코드(n=576~2304, k=288~1152, rate 다양) 지원 디코더 칩 설계.
2. 기존 설계는 BNU/CNU 연산이 순차적이라 하드웨어 이용률이 50%에 그치고 decoding latency가 큼.
3. base matrix만 reorder(전체 PCM 재정렬 아님)하면 1의 위치가 대각선 인근에 모여 BNU/CNU 겹침 연산이 쉬워진다는 [6] 알고리즘을 채택.
4. Min-Sum Algorithm(MSA) 기반 디코더에서 BNU(bit node unit)와 CNU(check node unit) 연산을 overlap시켜 hardware utilization을 50%→75%로, latency를 31.25% 감소.
5. 새로운 early termination: 연속 두 iteration의 hard-decision 결과가 같으면 조기 종료, 다르면 최대 iteration 도달 여부 확인 — SNR별 iteration 수를 동적 조정.
6. 저장소는 단일 중앙 메모리 대신 24개 single-port + 76개 two-port 총 100개 분산 레지스터 뱅크로 구성해 데이터 접근 병렬성과 저전력 확보.
7. Address Generation Unit(AGU)이 Counter Unit(CU) 신호를 받아 BNU/CNU/Output Buffer/ETU 주소를 생성하는 reconfigurable architecture로 19-mode(블록사이즈)를 저장소·연산유닛 수정 없이 지원.
8. 100개 메모리 뱅크를 10x10 2-D checkerboard 배치로 라우팅 복잡도와 칩 면적 절감.
9. 결과: TSMC 0.13um, 420K gate, die 8.29mm2(core 4.45mm2), 83.3MHz@52mW, throughput 2~8 iteration에 따라 30.3~111.1Mbps; 0.85V 저전압 시 41.67MHz/16mW.
10. 한계: 성능(BER)은 MSA와 "거의 동일" 수준으로 제시될 뿐 정정능력 자체의 개선은 아니며, 특정 QC base matrix 구조(reorder 가능한)를 전제로 함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| overlapped BNU/CNU 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | iteration 루프 내 파이프라인 스케줄과 개념적으로 유사하나 base matrix reorder 전제라 Prime ECC 고정 QC 구조에 그대로 적용은 어려움 |
| 새 early termination(연속 2회 HD 일치) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료를 이미 보유, 본 논문의 HD 비교 방식은 대안적 저비용 판정 로직으로 참고 가능 |
| 분산 메모리뱅크/checkerboard 배치 | HW arch (병렬화, 메모리, 라우팅, pipeline) | 메모리 라우팅 최적화는 Prime ECC HW 구조와 정합 필요성 커서 이식 난이도 중 |
| reconfigurable multi-mode(AGU) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 단일 QC-LDPC 고정 설계라 멀티모드 재구성 개념 자체의 대응 없음 |
| Min-Sum 디코더 알고리즘 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 Min-Sum 보유, 본 논문은 알고리즘 자체 개선이 아닌 HW 스케줄링 개선이라 중복 |

> 적용 가치: 낮음 | Min-Sum 성능 개선이 아닌 멀티모드 WiMAX HW 구현 최적화(면적/전력/스케줄링)이며 Prime ECC는 단일 고정 QC-LDPC라 멀티모드 재구성 기법의 실효성이 낮음.
