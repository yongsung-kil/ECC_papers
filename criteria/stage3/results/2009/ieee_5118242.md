### B. 알고리즘 요약

1. Partially-parallel QC-LDPC 디코더는 CNU(check node unit)와 VNU(variable node unit)가 서로의 결과를 기다리는 의존성 때문에 하드웨어 이용률(HUE)이 기본 50% 수준으로 낮음.
2. 기존 OMP(overlapped message passing)나 휴리스틱 재정렬은 일반 LDPC 부호에 적용 불가하거나 base matrix의 quasi-cyclic 성질을 깨뜨려 구현이 어려움.
3. base matrix의 행(check node)·열(variable node) 처리 순서(`t_sci`, `t_svi`)를 결정하는 문제를 반복당 소요 clock cycle `L`을 최소화하는 정수계획 문제로 정식화(자원 제약 + intra/inter-iteration 의존성 제약).
4. 핵심 기법은 simulated annealing으로 행/열 쌍을 무작위로 swap하며 비용함수 `L`을 최소화하는 스케줄을 탐색 — QC-LDPC의 quasi-cyclic 성질(base matrix만 재정렬)을 유지.
5. `L`(디코딩 1 iteration당 clock cycle 수)이 작을수록 HUE가 높아지며, 식 `HUE = (실제 유효 처리 cycle)/L` 관계로 스케줄 품질을 정량화.
6. 확장 기법으로 sub-matrix decomposition을 제안: circulant sub-matrix(크기 q)를 인수 DF로 더 작은 sub-matrix(크기 q/DF)들로 쪼개 base matrix를 더 크고 sparse하게 만들어 스케줄링 탐색 공간을 확대.
7. decomposition factor가 클수록 HUE는 개선되지만 base matrix의 non-null entry 수가 늘어 memory bank 및 address generator 수가 증가하는 hardware cost trade-off 존재.
8. 학습/최적화 절차는 simulated annealing(비용함수=L, move=행/열 순서 swap) 하나이며, 탐색이 느린 큰 decomposed 행렬은 기존 휴리스틱[10] 알고리즘으로 대체.
9. 결과: IEEE 802.16e rate-1/2 부호에서 기존 대비 HUE 13.3~20%p 개선, decomposition 적용 시 프로세서 유닛 많은 구성에서도 HUE 최대 50%p까지 개선(예: rate 1/2, CNU=12/VNU=24에서 0.5→0.92~1.0).
10. 한계: 스케줄은 오프라인 계산이라 HW 오버헤드 없다고 주장하나 실제 ASIC/FPGA 합성·실측 없이 이론적 스케줄링 결과(HUE 수치)만 제시, 정정성능(BER/FER)에는 영향 없음(순수 처리량/지연 최적화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNU/VNU 스케줄링 순서 최적화 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 내 CN/VN 처리 순서·타이밍 스케줄과 직결, HUE 개선 아이디어 이식 가능 |
| simulated annealing 기반 스케줄 탐색 | 대응 없음 | 오프라인 설계 도구 단계로, 런타임 디코더 코드에 대응 모듈 없음 |
| sub-matrix decomposition(base matrix 분해) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 바꾸는 접근이라 특정 QC-LDPC 고정된 Prime ECC엔 재검증 부담 큼 |
| memory bank/address generator 증가 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC의 z=32 병렬단위·HCU와 상충 가능성, 정합 검토 필요 |
| Dual-Update(even/odd 교대) 유사 개념과 비교 | `decoder.cpp` Dual-Update 스케줄 옵션 | 기존 보유 조기 스케줄링 기법과 중복 여부 검토 필요 |

적용 가치: 중간 — 정정성능과 무관한 순수 HW 스케줄링/유틸리제이션 최적화라 Prime ECC의 처리량·지연 개선(특히 CNU/VNU idle time 감소)엔 참고 가치가 있으나, 부호구조 변경(sub-matrix decomposition)은 고정 QC-LDPC 구조상 이식 난이도가 높음.
