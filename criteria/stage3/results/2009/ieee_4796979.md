### B. 알고리즘 요약

1. (3,L) QC-LDPC, p×p circulant permutation matrix 기반 H-matrix, block length `n=Lp`.
2. 기존 random construction은 girth≥g를 만족할 때까지 반복 탐색해야 하므로 복잡도가 exhaustive search 수준(`p^3L`)으로 과도함.
3. cycle 존재 조건은 `sum(p_jk,lk - p_jk+1,lk) = 0 mod p` (식 3)로, 인접 행 간 shift 값 차이(incidence matrix `D`)로 재정의 가능.
4. 핵심 기법: Progressive-Block Growth(PBG) — H의 첫 행은 무제약 생성, 이후 행은 `D(i,j)`를 행별·좌에서 우로 순차 생성하며 매 스텝 candidate pool `S = GF(p) - E`에서만 값을 뽑아 길이 4/6 cycle을 원천 배제.
5. LEMMA 1(길이 4 방지)·LEMMA 2(길이 6 방지)로 exclusion pool `E`를 미리 계산 — `a=(b+c) mod p` 형태의 short/long edge 관계식이 핵심(식 6~9).
6. 확장 없음(단일 단계 알고리즘).
7. 부수 트릭: cycle length 6는 long edge 1개 + short edge 2개 조합으로만 검사하도록 case를 2가지(Fact1/Fact2)로 축소해 계산량을 줄임.
8. 최적화 절차 없음(GA/DE 등 미사용, 결정론적 순차 생성 + 매 스텝 무작위 선택).
9. 결과: PBG는 성공/실패 시 모두 검사 cycle 수가 `L^2` 이하로 random construction(`L^2(L-1)`, 매트릭스 수 `p^3L`)보다 현저히 낮음. BER 10^-5 기준 N=504에서 Mackay 랜덤코드 대비 0.3dB, N=1008에서 0.1dB, PG-LDPC(R≈0.77) 대비 0.1dB 개선.
10. 한계: (3,L) 즉 CN 차수 J=3 코드로 한정, BP decoding 시뮬(AWGN, max iter 100)만 수행, HW/복잡도(gatecount) 미검증, rate/length가 예제별 상이(0.5~0.77, N=504~1066)라 고rate(~0.9) 영역 확장성 불명.
