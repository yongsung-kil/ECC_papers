### B. 알고리즘 요약

1. 대상: 임의(unstructured 포함) 고rate binary LDPC의 부분병렬 디코더 HW, 채널 AWGN, Offset Min-Sum 복호.
2. 문제: 특정 코드류(QC 등)에 묶이지 않는 범용 디코더는 메시지 전달망(IN)이 복잡·고비용 → 유연성과 throughput/area 트레이드오프.
3. 관찰: 고rate 코드는 VN 수 >> CN 수, VN degree `j`가 작음 → 전달망 단순화 여지.
4. 핵심기법 VTC: VN은 모든 연결 CN에 동일값 전송(식(2)) → `E` 대신 `N` 사이클; `p>j`면 `q=p/j`개 VN을 충돌 없이 한 팩으로 전달(message packing) → `N/q`.
5. 팩 탐색: `p`개 VNP를 `j`그룹으로 나누고, forward-checking(자유도 최소 노드 선택) 휴리스틱을 얹은 Backtracking으로 충돌 없는 팩 구성.
6. 전달시간 식: `T ≈ [1-(1-j/p)ηp]N` (식(4)), `ηp`=packing efficiency → 병렬도 `p`·최대 VN degree `j`·효율 `ηp`에 의존.
7. 핵심기법 CTV: CN 메시지는 min-sum상 2개 magnitude뿐 → (min, 2nd-min, min index, sign pattern) row-pack 한 워드로 압축(식[3] 방식)해 한 사이클 전달.
8. 구현 디테일: VNP 입력에 FIFO(depth 16)로 불균형 row 처리 흡수, 2단 MUX 전달망(p/j개 j:1 + p개 p/j:1 MUX).
9. 결과: 코드 I~VI에서 VTC 오버헤드 14.2~70.8%, CTV 0~69.8%; Virtex4 합성 시 [5][6] 대비 slices(192@p16)·control bit 최소.
10. 한계: 알고리즘(정정성능·iteration) 불변, 순수 라우팅/area 최적화; ASIC gatecount·throughput 미제시, 유연성 대가로 시간 오버헤드.
