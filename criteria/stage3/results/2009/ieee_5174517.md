> 총평: protograph 코드설계(선형 최소거리 성장 보장 구성법)와 check-node의 정확한 extrinsic 계산(Min* 함수, 3(dc-2) 최적 연산)이 핵심이며, 특히 constraint-node 연산 최적화 및 저정밀도 근사(BP-Based/Approximate-Min*-BP)는 디코더 HW 최적화 관점에서 Prime ECC min-sum 대비 참고 가치가 있음.

### B. 알고리즘 요약

1. Protograph 기반 LDPC 코드(작은 base graph를 copy-and-permute로 확장, circulant 치환 시 QC-LDPC와 유사)를 다루며, rate-compatible 계열(rate 1/2~7/8, AR4JA 등) 구성을 목표로 함.
2. 문제의식: 기존 protograph 코드(RA/IRA/ARA)는 capacity-approaching threshold는 얻지만 최소거리가 sublinear로 성장해 error floor에 취약함.
3. 핵심 가정: 채널은 BEC 또는 BI-AWGN, reciprocal channel approximation(RCA)으로 density evolution을 단일 실수 파라미터 `s`(SNR), `r`(reciprocal)로 근사.
4. 핵심 기법 1단: 점근적 weight enumerator `r(δ) = Σ a_c(δ_i) - Σ(q_j^v-1)H(δ_j)` 로 코드 앙상블의 최소거리 성장률 δ_min을 계산, degree-3 이상 노드로만 구성하면 선형 성장 보장.
5. 핵심 식 의미: check-node splitting(degree-qic 체크를 두 체크+degree-2 변수로 분할)을 반복해 threshold를 낮추면서도 선형 최소거리 성장 조건(check node splitting condition)을 만족시키는 구성 정리(Lemma/Theorem) 제시.
6. 핵심 기법 2단: precoding(degree-1 노드 도입, accumulator 전단 부착)으로 threshold를 추가 개선한 AR4JA 코드 계열 구성.
7. 부수 트릭: constraint-node 연산을 `Λ_BP*` 대신 `Min*(v1,v2) = min(|v1|,|v2|) + ln(1+e^-(|v1|+|v2|)) - ln(1+e^-||v1|-|v2||)` 로 재정식화, HW 친화적 single/double-line 근사(Fig.19)로 combinational logic 구현. 확장 메시지 계산은 `A*(dc)` 알고리즘으로 3(dc-2) 연산에 최적화(이론적 하한과 일치).
8. 추가 근사: BP-Based(최소/차소값만 사용, ~1dB 손실), Offset-BP-Based(0.1~0.2dB 손실), Approximate-Min*-BP(<0.05dB 손실, SNR 추정 필요) 등 저복잡도 constraint-node 근사 계열.
9. 결과: rate-1/2 AR4JA(k=4096) FPGA 시뮬에서 capacity 대비 gap 0.44dB(rate 1/2)~0.36dB(rate 7/8), n=5792 코드는 FER 3×10^-7까지 error floor 없음(단 rate-7/8은 10^-6대에서 trapping-set 유발 에러 관측).
10. 한계: 우주통신(DSN) 응용 타깃, 시뮬/FPGA 검증(JPL Universal Decoder)뿐 ASIC 게이트카운트·throughput 수치는 본문에 없음; H-matrix가 protograph 재설계 전제라 기존 고정 QC-LDPC 구조엔 통째 이식 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min*/Λ_BP* constraint-node 정확 연산 및 A*(dc) 최적 스케줄 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum(min1/min2 근사) 대비 정확도 높은 CN 갱신 방식으로 waterfall 성능 개선 검토 가능 |
| BP-Based / Approximate-Min*-BP 저복잡도 근사 | `decoder.cpp` `C2V_Cal()` | 기존 min-sum과 유사한 계열의 대안 근사 - 중복 기여 가능성, 성능/복잡도 트레이드오프 비교 필요 |
| Protograph 기반 H-matrix 구성(check-node splitting, precoding) | `ecc_top.cpp` `Load_PCM()` | 코드설계 레이어, 고정된 QC-LDPC base matrix 재설계는 난이도 높음(이식 어려움) |
| Circulant-shift 기반 HW 디코더 아키텍처(edge memory + cyclic shift) | 디코더 HW 모델 (z=32 병렬단위) | Prime ECC의 QC-LDPC row 병렬 구조와 개념적으로 유사, 직접 코드 이식보다는 아키텍처 참고 수준 |

> 적용 가치: 중간 — constraint-node의 정확한 Min* 연산과 최적 스케줄(A*(dc))은 디코더 정확도/복잡도 균형 관점에서 참고할 만하나, protograph 코드설계 자체는 Prime ECC의 고정 QC-LDPC 구조와 이질적이라 이식 난이도가 높음.
