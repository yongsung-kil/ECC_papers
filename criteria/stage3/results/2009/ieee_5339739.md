### B. 알고리즘 요약 (10줄 내외)

1. 대상은 고rate(`R≥0.85`) irregular binary LDPC 부호로, BPSK 변조 후 AWGN 채널에서 SPA(sum-product algorithm, 최대 20회 반복) 디코딩을 가정.
2. 풀려는 문제는 LDPC 코드의 인코딩 복잡도가 높다는 점: 기존 array LDPC(ALDPC)나 modified array codes(MAC)는 인코딩을 위해 Gaussian elimination으로 생성행렬을 구해야 하거나 패리티비트를 재귀적으로 계산해야 함.
3. 별도의 채널/간섭 모델은 없으며 표준 BPSK+AWGN, iterative decoding(SPA) 프레임워크를 그대로 사용.
4. 핵심 기법은 ALDPC의 parity-check matrix `H_A`(파라미터 `p`(소수), `q`, `j`로 정의되는 순환치환행렬 블록 구조, 식 4)에 identity matrix `I_A`를 이어붙여 `H_S = [I_A | H_A]` (식 8) 형태의 systematic parity-check matrix를 구성. 이때 생성행렬은 `G_S = [H_A^T | I_B]` (식 9)로 sparse하게 얻어져 LDGM(low-density generator matrix) 코드가 됨.
5. `H_S`가 이미 systematic 형태이므로 생성행렬 계산 없이 `c=mG` 없이도 모든 패리티비트를 동시에(병렬로) 구할 수 있어 인코딩이 `(1-R)(q-1)n`회 XOR 연산으로 linear-time 수행됨(반면 MAC 코드는 재귀적 계산 필요).
6. 확장으로 `j>1`을 선택하면 열(column)마다 non-zero 원소 수가 달라져 irregular LDPC가 되며, `H_S` 구조는 identity matrix 결합에도 불구하고 girth-4 사이클이 없고 `H_A`와 동일 girth를 유지(길이-4 short cycle 없음이 성능 유지의 근거).
7. 구현 디테일로 행렬 밀도(density) `D_H`, `D_G`를 식(11)-(12)로 정의해 제안 코드가 ALDPC/MAC보다 밀도가 낮음을 정량적으로 제시(예: n=4489에서 제안코드 `D_G`는 ALDPC 대비 약 95.1% 감소).
8. 별도 학습/최적화 절차는 없음(파라미터 `p,q,j` 선택에 의한 결정론적 구성법).
9. 결과: (529,462) 코드 R≈0.87에서 MAC 코드 대비 SNR>5.85dB에서 약 0.25dB 성능 열위; (2209,2021) R≈0.91에서 약 0.15dB 열위; (4489,4221) R≈0.94에서는 격차가 더 줄어듦(수치 본문 미기재). 인코딩 XOR 연산 수는 전 구간에서 MAC 코드보다 적음(Table I, 예: n=2209에서 7896 vs 8118).
10. 한계: 디코더 자체는 표준 SPA를 그대로 사용하며 개선이 없고(정정능력 향상 없음, 오히려 소폭 저하), HW 구현/합성 결과 없이 시뮬레이션(BER/SNR 곡선)에만 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Array LDPC 기반 parity-check/generator matrix 구성(H-matrix 설계) | `ecc_top.cpp` `Load_PCM()` | 부호설계 층위로 이식 난이도 "높음"에 해당, 현재 고정된 QC-LDPC 구조 재설계 필요 |
| identity-concatenation 기반 저복잡도 systematic 인코딩(생성행렬 불요, 병렬 패리티 계산) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 현재 dual-diagonal 인코딩과 다른 방식의 저복잡도 인코딩 구조로 참고 가능하나 교체 시 재검증 부담 큼 |
| girth-4 없는 구조 보장 기법 | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 시 cycle 방지 기법으로 참고 가능한 수준 |
| 디코더(SPA) | 대응 없음 | 표준 sum-product 그대로 사용, Prime ECC의 Min-Sum 대비 개선점 없음 |

> 적용 가치: 낮음 — 정정능력 개선이 없고(오히려 MAC 대비 소폭 열위) 부호구조 자체를 재설계해야 하는 "높음" 난이도 항목이라, 인코딩 XOR 수 절감이라는 이득만으로는 Prime ECC의 검증된 dual-diagonal 인코더를 교체할 유인이 크지 않음.
