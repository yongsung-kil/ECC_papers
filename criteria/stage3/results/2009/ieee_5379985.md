### B. 알고리즘 요약

1. 대상은 무선 time-varying 채널의 다양한 rate 요구를 위한 rate-compatible(RC) LDPC 부호족, mother code rate `Ro=0.5`, 블록길이 `n=18432`, AWGN 소프트 반복복호.
2. 문제: 기존 RC-LDPC는 열(비트) 조작(puncturing/extending/nulling) 위주인데, puncturing만으론 고rate에서 punctured 비트가 많아 반복복호가 마비되고, extending은 단순 인코더 확보가 어려움.
3. 핵심 착안: 열(=codeword 비트)은 그대로 두고 parity-check matrix의 행(=check)만 조작 — 저rate는 행 추가, 고rate는 행 제거, 따라서 모든 rate에서 블록길이 불변.
4. 인코딩: zigzag 패턴을 미리 넣은 PEG(progressive edge growth)로 mother code를 구성, 우측 redundant 노드에 zigzag 엣지를 고정해 근사 하삼각(최우열 weight 2) 구조로 linear-time 인코딩 달성.
5. 행 추가 시 새 check 노드도 zigzag로 연결하고 modified PEG로 정보 노드 엣지를 큰 girth로 갱신; 행 제거는 하단 행부터 제거해 linear-time 인코딩 구조 유지(= parity puncturing 등가).
6. mother code 최적화: constrained density evolution(Gaussian approximation + linear program)으로 각 목표 rate의 degree 분포를 구함 — 균일한 row weight가 좋은 threshold를 줌을 Mean/Jensen 부등식으로 유도.
7. 정보비트 열 weight 배정: greedy 알고리즘 `Wj = argmax` (목표 분포와 실현 분포의 최대 불일치 지점)으로, 행 추가로 정보→parity로 기능이 바뀌는 열들의 성능을 보정.
8. 복호: 저rate 부호(행 추가)는 원 행을 모두 포함하므로 mother decoder로 그대로 복호 가능, 고rate 부호(행 제거)는 puncturing처럼 처리 → 단일 decoder가 항상 고rate로 동작해 면적·전력 절감.
9. 결과: rate 8/19~8/10 부호족의 AWGN BER가 잘 동작하며, rate 8/19~8/16 구간에서 기존 extending 방식보다 우수; 이 구성 기반 type-II hybrid ARQ 스킴도 제안.
10. 한계: HW 미설계·시뮬레이션뿐, error-floor/절대 정정능력 분석 없음, 무선 ARQ 특화라 NAND 고정 고rate 코드와 목적 상이.
