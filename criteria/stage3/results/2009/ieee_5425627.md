### B. 알고리즘 요약

1. 대상은 rate-compatible LDPC(무선 HARQ/IEEE 802.16m)로, mother code를 puncturing해 다중 rate 자식부호를 얻는 문제; 예시는 802.16e Block-LDPC(mother rate 1/2, `(960,480)`)와 `(3,6)` PEG code(길이 1024, rate 0.5).
2. 풀려는 문제: 주어진 mother code에서 target rate의 좋은 puncturing 패턴을 낮은 복잡도로 선별(후보수 `C(n, x)`가 지수적으로 큼).
3. 핵심 모델: puncturing = Tanner graph에서 variable node 제거 → 이웃 check 식들을 subtraction으로 `k-1`개로 합치는 등가 복호행렬 변환 `Ψ`.
4. 핵심 기법 1단: GADE(Gaussian Approximation Density Evolution)로 punctured 부호의 degree 분포를 1차원 재귀 `p_l=q(s,p_{l-1})`(식6)로 해석, 수렴속도로 FER 예측.
5. 핵심 식: cost function `f(λ̂_i1, ρ̂) = log λ̂_i1 + Σρ̂_j log(j-1) - c0·î1`(식9)로 후보 패턴의 기대 성능을 정량화(row+column degree 모두 반영).
6. 최적화: `min f` (식10)로 변환된 부호의 optimized degree 분포를 선택.
7. 탐색 알고리즘: (a) generic LDPC용 best-first **stack 알고리즘**(evaluation `e(k)=f(k)+h(k)`, heuristic `h(k)=-c1(p-k)`), (b) 표준 Block-LDPC용 **non-greedy exhaustive 랭킹**(block 단위, 우선순위 지정 + rate-compatible 포함관계 `p1⊆p2⊆...⊆pk` 보장).
8. 설계 이점: puncturing 패턴 설계와 인코딩 행렬 설계를 분리 → 인코더 항상 존재, irregular 부호에서 high-degree bit을 정보비트로 배정해 unequal protection 가능.
9. 결과: cost 값이 실제 FER와 잘 일치(802.16e 2/3B, 3/4B 전수 시뮬), k-SR 알고리즘[4] 대비 rate 2/3·5/6에서 Eb/N0 0.25~0.3dB 이득, 3/4는 동등.
10. 한계: 이론(GADE)+BP 시뮬만, HW/gate 없음, rate-compatibility·HARQ 지향으로 고정rate NAND엔 직접 필요성 낮음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |

| puncturing 패턴 / H-matrix subtraction 변환 | ecc_top.cpp Load_PCM() | puncture는 H-matrix/부호구조 층, Prime ECC는 고정 QC-LDPC(punct_col=2 이미 존재) 로드 |
| GADE cost function(degree 분포 평가) | 대응 없음 | Prime ECC엔 offline DE/GADE 부호설계 도구 미포함 |
| BP(soft) 복호 baseline | decoder.cpp LDPC_Decoder() | min-sum이 BP의 HW 근사이나 본 논문은 BP를 성능검증 도구로만 사용 |
```

적용 가치: **낮음** - 무선 HARQ rate-compatibility를 위한 offline puncturing 설계기법이라 고정rate NAND ECC엔 직접 쓸 곳이 좁고, Prime ECC의 punct_col 선택 참고 정도의 간접 가치만 있다.
