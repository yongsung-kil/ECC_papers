> 총평: Min-Sum에 보정항을 추가하는 알고리즘 자체는 이미 널리 알려진 기법(min-sum 근사 보정)이며 Prime ECC가 이미 min-sum/scaling 기반 보유, 코드 구조(1008,3,6 regular)도 NAND 고rate와 거리가 있어 이식 가치는 제한적.

### B. 알고리즘 요약

1. 대상 코드: `(1008, 3, 6)` regular LDPC(rate 1/2, column weight 3, row weight 6), Gaussian(AWGN) 채널, BPSK 가정.
2. 문제의식: sum-product(BP) 알고리즘은 성능은 좋으나 `tanh`/`2tanh^-1` 연산 복잡도가 높고, 이를 단순화한 Min-Sum-Plus는 고 SNR에서 성능 저하가 크다는 한계.
3. 핵심 가정: 체크노드 업데이트를 `min` 근사로 대체하되, 두 LLR의 합 근사 오차를 보정하는 correction factor `s(B)`를 추가로 더함(식 5).
4. 핵심 기법(MSPCF, Min-Sum-Plus-Correction-Factor): 체크노드 갱신 `C_{i,j}(t+1) = [Π α_{j,k}(t)]·(min_{k} β_{i,k}(t) + s(B_{i,j}(t)))`(식 5), correction factor `s(B)`는 `max(5-|ΔB|/8, 0) - max(5-4|ΔB|,0)/8` 형태의 LUT 근사(식 6).
5. 식(5)의 의미: `ln(e^L1+e^L2)=max(L1,L2)+ln(1+e^-|L1-L2|)`의 두 번째 항(오차항)을 LUT로 근사해 min-sum의 손실을 보정, 순수 min-sum 대비 고 SNR 영역에서 성능 열화를 줄임.
6. HW 구조(2단): 6-module FPGA 구조(입력제어/연산상태제어/체크노드갱신/비트노드갱신/데이터저장/행렬체크) + 6채널을 한 그룹으로 묶어 24채널을 4개 그룹으로 순차 처리하는 부분 병렬(part-parallel) 스케줄.
7. 구현 디테일: 채널 입력 3bit LLR을 직병렬 변환 후 1008bit로 재구성, BFU(Bit Function Unit, wb=24개, 각 42bit 처리)는 LUT 기반 덧셈+오버플로 처리, CFU(Check Function Unit, wc=12개)는 최소값 탐색+부호 XOR+correction factor LUT 3단계로 구성.
8. 학습/최적화 절차 없음 — 알고리즘 성능은 Matlab 시뮬레이션(최대 반복 30회)으로 Min-Sum-Plus vs MSPCF 비교, HW 성능은 Quartus 5.1 합성 결과 비교.
9. 결과(표1): all-parallel 대비 part-parallel 구조가 logic resource를 MSPCF 기준 18,996→11,830로 절감하면서 최고 클록 174MHz→128MHz(최대 디코딩 속도 128Mbit/s로 요약)로 낮아짐, MSPCF는 Min-Sum-Plus 대비 고 SNR에서 BER 성능 우수(정량 수치는 그림에만 존재).
10. 한계: 단일 (1008,3,6) regular 코드에만 실험, correction factor의 정량적 BER 개선폭·gate count 상세는 본문에 없고 그림에만 존재(→ 규칙상 미상 처리), soft 정보는 3bit LLR 입력 기준이나 보정항 계산상 4bit 이상 폭 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MSPCF 디코딩 알고리즘 (min-sum + correction factor LUT) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min1/min2 기반 Min-Sum을 보유, correction factor(오차 보정 LUT) 추가는 CN 연산부에 이식 가능하나 min-sum 대비 순증 이득 검증 필요 |
| CFU 구조 (min 탐색+부호 XOR+보정 LUT) | `decoder.cpp` `C2V_Cal()` | CN 업데이트 파이프라인과 기능적으로 유사, LUT 추가 삽입 지점으로 참고 가능 |
| BFU 구조 (VN 갱신, LUT 기반) | `decoder.cpp` `VN_Cal_HD()` 등 | VN 갱신 로직과 대응되나 특별한 개선 기법(양자화 등) 없이 표준 합산뿐 |
| part-parallel 스케줄(6채널 그룹×4) | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC는 이미 z=32 lifting 기반 행 병렬 구조를 가지고 있어 이 논문의 저병렬 방식은 구조적으로 열위, 참고 가치 낮음 |

적용 가치: 낮음 — correction factor는 Prime ECC가 이미 보유한 min-sum 계열 기법과 중복되는 개선이고(§3 "이미 보유한 기법" 목록에 해당), 코드 구조·병렬화 수준도 NAND ECC 대비 단순하여 실질 이식 이득이 크지 않음.
