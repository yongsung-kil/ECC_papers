### B. 알고리즘 요약

1. IEEE 802.15.3 WPAN 표준의 block-based QC-LDPC 코드(길이 576, sub-matrix 크기 18, rate 1/2·3/4·7/8) 디코더 설계.
2. 짧은 코드워드(576bit)로 최대 3.6Gbps를 내면서 3개 rate를 모두 지원하는 유연한 HW가 필요.
3. rate 7/8 기본 parity-check matrix를 row-split하여 3/4, 1/2을 파생시키는 구조(irregular H, 일부 sub-matrix 블록 비움).
4. modified Min-Sum 알고리즘: `Rcv = α × sign(∏Lvc) × min|Lvc|`, α≈0.75 스케일링 팩터로 SPA 근사.
5. Lvc(variable-to-check)는 `Rmv`의 합 + intrinsic message `Iv`로 계산(표준 VN update).
6. 고속 아키텍처: 72열×4-parallelism로 8 clock/iteration, CNU가 min/2nd-min/index/sign만 저장(압축)해 메모리 절감.
7. flexible CNU: rate별로 c1/c2/c3 비교모듈 조합을 바꿔 1/2/4행 처리, 4×4 switch로 row split 패턴 라우팅.
8. 학습/최적화 절차 없음 (고정 알고리즘의 HW 구현).
9. 결과: 고속형 500MHz·65nm에서 3.6Gbps@iter=8, 320k gate; 저비용형은 200MHz에서 360Mbps@iter=10, 150k gate(CNU 연산부만 911→148 gate로 축소).
10. 한계: 정정성능 개선 없음(고정 표준 코드의 순수 HW 구현), 576bit 짧은 코드 전용이라 NAND용 긴 QC-LDPC로 구조 일반화 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified Min-Sum CNU 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum과 동일 원리, 신규성 없음 |
| 압축 row 저장(min/2nd-min/index/sign) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 유사 압축 저장 방식 기존 구현과 중복 |
| 다중 code rate용 flexible CNU/switch | `ecc_top.cpp` `Load_PCM()` | 특정 표준(576, 3-rate row-split) 전용 구조라 이식 난해 |
| VNU 처리(고정 combinational) | `decoder.cpp` `VN_Cal_HD()` 등 | 표준적 VN update, 특이점 없음 |

적용 가치: **낮음** — 부호는 특정 무선표준(576bit, 3-rate) 고정이고 알고리즘(min-sum, 압축 저장)도 Prime ECC가 이미 보유해 정정능력/HW 면에서 새로 이식할 요소가 없음.
