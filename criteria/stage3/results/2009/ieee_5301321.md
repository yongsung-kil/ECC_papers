### B. 알고리즘 요약

1. DVB-T2는 enhanced IRA-LDPC(rate 6종/7종, N=16200 또는 64800)를 채용, H=[Hs, Hp]에서 Hp는 dual-diagonal 하삼각행렬로 재귀적 parity 계산 가능.
2. 문제: DVB-T2 원본 H_org가 QC 패턴을 보이지 않아 QC-LDPC 전용 하드웨어 구조를 그대로 쓸 수 없음.
3. 채널모델: AWGN + BPSK, 고차변조(QPSK/16/64/256QAM)와 결합해 사용되는 표준 규격.
4. 핵심 기법 1단: Hp_org에 대해 열을 `mod Q` 치환, 이어서 `[Hs_org, Hp_new]`에 대해 행을 `mod Q` 치환하여 서브블록 폭 360의 QC 유사 H_new 생성, 이후 1행 서브블록 보정으로 최종 H_QC 도출.
5. H_QC는 H_new와 완전히 동치가 아니므로 patiy-check 계산 시 코드워드 x도 동일 행 permutation을 적용(x_new*H_new=0)하고 1행 데이터 접근을 특수 설계해야 등가성 보장.
6. 핵심 기법 2단: Layered(TDMP) 업데이트 - 한 행(sub-block row) check-node 갱신 직후 즉시 variable-node 정보를 갱신해 다음 행이 최신 데이터를 사용, 평균 반복횟수를 TPMP 대비 약 50% 감소.
7. 부수 트릭: Rcv 갱신에 Normalized Min-Sum(NMS) 사용(`Rcv = K·min(...)`, K는 density evolution으로 결정), Lcv는 저장 없이 `Lv-Rcv`로 계산해 메모리 절감. 병렬 MDU 수를 표준 서브블록 크기 360 대신 180으로 축소(barrel-shifter 폭도 180으로 축소, 복잡도 1/4), 그 대신 Lv/Rcv 저장 깊이를 2배로 하여 홀/짝 분할 저장. 3-stage 파이프라인 적용.
8. 학습/최적화: NMS 정규화 계수 K는 density evolution 이론으로 산출(구체적 값은 본문 미기재).
9. 결과: Stratix II EP2S130 FPGA 구현, ALUTs 41526(39%), Registers 15864(15%), Memory 4.58Mbits(67%), 최대 clock 91.47MHz, 최대 반복 43회 가능. 병렬비 180 채택 시 기존 360-parallel 대비 시스템 복잡도 50% 이상 절감. NMS는 BP 대비 0.2dB 미만 성능 열화.
10. 한계: BER/FER 곡선은 그림(Fig.2)에만 제시되어 본문 수치 서술 없음, 실제 throughput(Gbps) 수치 미기재, 정정능력 자체의 개선이 아닌 순수 복잡도/latency 트레이드오프 연구.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Normalized Min-Sum (NMS) Rcv 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 NMS 정규화 계수 아이디어는 min-sum 성능 개선 참고 가능 |
| Layered(TDMP) 디코딩 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 Dual-Update(even/odd 교대)와 유사한 계열의 조기수렴 스케줄링, 개념적으로 이미 보유 |
| 병렬 MDU 수 축소(360→180) + barrel-shifter 폭 절감 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z_sb=32 고정 병렬단위라 구조는 다르나 병렬도-면적 트레이드오프 설계 사고는 참고 가능 |
| H_org→QC 유사구조 변환(row/col permutation) | `ecc_top.cpp` `Load_PCM()` | DVB-T2 표준 특화 permutation이라 Prime ECC의 고정 QC-LDPC 구조에 직접 이식은 난이도 높음 |

> 적용 가치: 중간 — NMS와 layered 스케줄 자체는 Prime ECC가 이미 보유한 기법과 유사하나, 병렬비 축소(360→180)를 통한 면적-지연 트레이드오프 설계 방식은 HW 최적화 참고 가치 있음.
