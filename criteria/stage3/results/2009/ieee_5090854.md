### B. 알고리즘 요약

1. CCSDS C2 표준의 (8176,7156) QC-LDPC 코드(2×16 array of 511×511 circulant, row weight 32, col weight 4), rate≈0.875, near-earth 우주통신 응용.
2. LDPC 디코딩의 고복잡도(iteration당 32k+ 메시지 갱신)를 FPGA에서 저비용/고속 두 요구사항 모두 만족시키는 것이 목표.
3. 채널/노이즈 모델은 별도 명시 없이 표준 BP/SP 메시지패싱 프레임워크(LLR 기반) 가정.
4. check node 갱신은 `cb_{m→n} = g(bc_{n0→m},...,bc_{ndc-1→m})`, `g(x,y) = sign(x)sign(y)Min(|x|,|y|)/α` — normalization factor `α`로 sign-min(min-sum) 근사의 성능 저하를 보정.
5. bit node 갱신은 `bc_{n→mi} = u_n + Σ_{mj∈M(n)} cb_{mj→ni} - cb_{mi→ni}` (표준 message-passing 갱신식).
6. generic parallel 아키텍처: controller + I/O memory + multi-block message memory + CN/BN 처리 유닛 다수 인스턴스, QC 구조의 regularity를 이용해 16 BN(/2 CN)을 동시 처리.
7. HW 친화 트릭: 동일 base 아키텍처에서 processing block/memory word 크기만 조정해 low-cost(Cyclone II, 16BN 병렬)와 high-speed(Stratix II, 자원 4배로 throughput 8배) 두 버전 생성 — genericity를 통한 재사용.
8. 별도 학습/최적화 없음, correction factor `α`는 참고문헌[4]의 density evolution 결과를 참조해 고정.
9. 결과: 18 iteration이 최적 trade-off로 채택, 200MHz 클록에서 low-cost 70Mbps/high-speed 560Mbps, CCSDS 기존 FPGA 결과([2]) 대비 BER/PER 0.05dB 개선(동일 성능을 50회 대신 18회 iteration으로 달성), error floor 없음.
10. 한계: 코드가 CCSDS 표준으로 고정(부호설계 자체는 논문 기여 아님), scaling factor의 구체 값/유도과정 미기재, ASIC 합성 아닌 FPGA 매핑까지만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sign-min(normalized min-sum) + correction factor | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2+sign XOR 기반 min-sum이므로 normalization factor 적용 여부/값 비교가 직접 참고 가능 |
| QC-LDPC circulant 기반 병렬 처리(16BN 동시) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | row/circulant 단위 병렬 구조가 Prime ECC의 `z_sb=32` lifting 병렬 처리와 개념적으로 유사 |
| generic 아키텍처(자원-throughput trade-off, 동일 IP 재사용) | 디코더 HW 모델(z=32, HCU, GT) | 저비용/고속 두 버전을 하나의 파라미터화된 설계로 구현하는 아이디어는 참고 가능하나 Prime ECC는 이미 고정 HW 구조 |
| CCSDS 고정 H-matrix(8176,7156) 부호구조 | `ecc_top.cpp` `Load_PCM()` | 표준 고정 코드라 이식 난이도 높음(H-matrix 자체 교체 필요) |
| iteration수-throughput trade-off 테이블화(10/18/50) | 대응 없음(설계 파라미터 스터디) | Prime ECC의 iteration 파라미터 튜닝 참고자료로 활용 가능한 수준 |

> 적용 가치: 중간 — normalized min-sum + QC-LDPC 병렬 아키텍처라는 점에서 Prime ECC와 구조적 유사성이 있으나, 부호가 표준 고정이고 알고리즘 자체(sign-min 보정)는 이미 잘 알려진 기법이라 신규성 있는 이식 요소는 제한적.
