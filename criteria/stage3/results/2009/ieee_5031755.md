> 총평: 코액시얼 홀로그래픽 광저장 시스템에 기성 regular LDPC(min-sum)를 적용해 기록밀도를 높인 광학/저장시스템 실험 논문으로, LDPC 부호·디코더 자체의 신규 기여가 없어 NAND ECC 이식 가치가 없음.

### B. 알고리즘 요약

1. 시스템은 코액시얼 홀로그래픽 저장 장치(파장 407nm, NA=0.85, photopolymer 매체)로, LDPC는 저장 채널의 오류정정에 사용됨.
2. 풀려는 문제는 홀로그래픽 저장의 기록밀도(bit/in.2)를 높이는 것이며, 신호처리(LDPC+변조코드) 도입으로 SNR 여유를 확보해 밀도를 높임.
3. 채널 모델은 홀로그래피 특유의 interpixel interference(IPI)와 SNR(식 1: `SNR=20log10((μ1-μ0)/√(σ1²+σ0²))`)로 정의되는 아날로그 픽셀 검출 채널.
4. 사용 LDPC는 코드워드 길이 4947, 코드율 50%, column weight 3, row weight 6인 regular LDPC이며 min-sum 알고리즘으로 복호.
5. 변조코드 `E(n,m,k)`(n픽셀 중 m개 밝음, 코드율 k/n)를 LDPC와 결합해 white rate(밝은 픽셀 비율)를 낮춰 IPI에 의한 SNR 열화를 완화.
6. E(16,3,8) sparse code가 white rate가 가장 낮아 최고 SNR(14.1dB, random 대비 +4.6dB)을 달성, LDPC 유무 비교에서 waterfall 도입으로 threshold SNR을 2.3dB 개선(7.0dB→4.7dB 부근).
7. 부수적으로 LLR 정의 차이가 코드별 SNRdec 차이의 주요 원인이라 언급(LLR 최적화 여지 있음, 구체 수식 없음).
8. 별도 학습/최적화 절차는 없고, 렌즈(L1-L4) 재설계, 페이지 데이터 용량 5% 증가, aperture 크기(×2.0→×1.6 Nyquist)를 실험적으로 최적화.
9. 결과: E(16,3,8)+광학 최적화 조합으로 raw data density 415 Gbit/in.2(약 180GB, SNR=4.9dB error-free) 달성, 기존 무변조 랜덤코드(135 Gbit/in.2) 대비 대폭 개선.
10. 한계: LDPC 부호/디코더는 표준 기성 regular code + min-sum 그대로 사용, 신규 부호설계나 HW 구현 없음. 개선은 전적으로 변조코드와 광학계 파라미터 튜닝에 의함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| regular LDPC + min-sum 복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 min-sum 계열이나 본 논문은 표준 min-sum을 그대로 사용해 기여 없음(중복) |
| 홀로그래픽 채널/변조코드(IPI 완화) | 대응 없음 | NAND read 채널과 무관한 광학 저장 특유 문제 |

> 적용 가치: 낮음 — LDPC 부호·복호는 표준 기성 구성을 그대로 사용했고 논문의 실질 기여는 광저장 시스템(변조코드, 광학계) 최적화에 있어 NAND binary LDPC ECC 이식과 무관함.
