### B. 알고리즘 요약

1. LDPC의 Tanner graph에서 variable node(repetition code)와 check node(SPC code)를 모두 더 일반적인 컴포넌트 코드(예: `(dv,kv)` Hamming, simplex, SPC)로 대체한 doubly generalized LDPC(DGLDPC) 코드 구조를 제안, base matrix `H`의 row/column expansion으로 실제 parity check matrix 구성.
2. 기존 GLDPC(check node만 일반화)나 molecular code는 성능/유연성이 제한적이었고, DGLDPC는 표준 LDPC와 더 가까운 구조를 유지하면서 variable node도 일반화해 다양성을 확대.
3. AWGN 채널, BPSK 신호, iterative belief propagation(각 컴포넌트 코드는 MAP/Max-Log-MAP 서브디코더) 가정.
4. 핵심 기법: EXIT chart 기반 성능 분석. variable/check 노드의 extrinsic mutual information 함수 `I_V(I_U, Eb/N0, R)`, `I_U(I_V, dc)`를 information combining(닫힌 형태, 식 (1)~(17)) 또는 BEC EXIT 함수 변환(식 (18)~(20))으로 유도.
5. 식 (6),(8)의 unnormalized split information function `e_g,h`는 컴포넌트 코드의 dual code 구조에서 유도되는 BEC EXIT 값으로, AWGN 채널의 저rate 컴포넌트 코드 EXIT를 근사하는 데 필수.
6. 확장: differential evolution(DE)으로 variable/check 노드 타입 비율(`λ_i`, `ρ_j`) 최적화하여 threshold를 개선(예: rate-1/2에서 DGLDPC1이 LDPC보다 0.2dB 개선).
7. 구현 디테일: 복잡도 제어를 위해 쉽게 복호되는 컴포넌트 코드(Hamming, SPC) 위주로 선택하거나, 복잡한 컴포넌트 코드는 낮은 비율만 사용.
8. 최적화 절차: EXIT chart curve-fitting + differential evolution으로 degree distribution 최적화(BEC 최적화 기법의 AWGN 확장).
9. 결과: 길이 10^6 rate-1/2 코드에서 BER 10^-5 부근 DGLDPC1이 LDPC 대비 0.2dB 개선 및 더 낮은 error floor; rate-3/4 DGLDPC2/3도 유사 개선.
10. 한계: 순수 시뮬레이션/이론 분석(HW 미설계), AWGN/BPSK 채널 전제, 컴포넌트 코드가 Hamming/simplex 등이라 서브디코더 복잡도가 표준 LDPC 대비 증가(soft MAP decoding 필요).
