### B. 알고리즘 요약

1. 대상: IEEE 802.16e/802.11n/DVB-S2/DTMB 등 멀티표준 block-LDPC. 구조형(QC-LDPC, AA-LDPC)·비구조형(IRA/HS-LDPC) 모두 지원, block size `576~2304`, sub-matrix `z=24~96`, rate 1/2~5/6.
2. 문제: 기존 디코더는 특정 표준·특정 부호구조에 종속(고throughput이나 code-dependent)이거나, code-independent지만 면적효율·throughput이 낮음.
3. 모델: block-LDPC PCM은 seed matrix의 각 '1'을 `z×z` cyclic-shift identity(offset `s`), '0'을 zero matrix로 확장. 비구조형은 superimposed cyclic-shift identity가 존재해 layered(TDMP) 시 data-collision 유발.
4. 핵심 기법: collision을 피하려 layered TDMP 대신 **rearranged TPMP + Normalized Min-Sum(NMS)** 채택 — check node 갱신 후 variable node 갱신을 2-phase로 분리.
5. 핵심 식: `E = S - R`, `R = NMS(E)`, `S = In + ΣR` — extrinsic message를 (min, 2nd-min, index, sign)로 압축 저장해 메모리 절감.
6. 핵심 HW: **Reconfigurable Pipelined Processing Unit(RPPU)** — RCS(cyclic shifter)+DRU+FAU+MSU에 3개의 2-to-1 MUX로 row-update(CPU) / column-update(VPU) 모드를 시분할 전환.
7. 구현 디테일: row/col을 순차 처리하므로 In/Sn/Rev 메모리를 dual-port 대신 **single-port**로 구성(면적·전력↓), critical path 축소 위해 6-stage pipeline(read/write 포함).
8. 제어: Central Controller의 12-state FSM이 DI-Mem에 저장된 decoding instruction(RUS/CNU/CUS/VNU 등)을 fetch해 code-independent 동작.
9. 결과: SMIC 0.18um CMOS 합성 13.7mm², 110MHz, 98Mbps(15 iter); FPGA(Stratix II) 56.6Mbps@73MHz(rate-1/2 2304-bit). [2]/[7] 대비 유연성 우위, 면적은 다소 큼.
10. 한계: 정정성능(BER) 개선 없음(NMS 그대로), 시분할 PU로 throughput은 전용구조 대비 competitive 수준, 실측 아닌 합성/layout까지, 멀티표준 유연성이 핵심이라 단일부호 응용엔 이득 작음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TPMP + Normalized Min-Sum 복호 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 min-sum(min1/min2+sign) 보유 — 중복 기여 |
| 압축 extrinsic(min,2nd-min,index,sign) 저장 | `decoder.cpp` `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2/sign 기반, 개념 동일 |
| 재구성 row/col 겸용 PU + 단일포트 메모리 | 대응 없음 | Prime ECC는 z=32 고정 파이프라인·dual-update 스케줄, 시분할 겸용 PU 미채택 |
| block-LDPC z-확장(cyclic shift) 로드 | `ecc_top.cpp` `Load_PCM()` | QC 구조 로드 대응은 있으나 멀티표준 재구성은 불요 |

적용 가치: **낮음** — 복호 알고리즘·파이프라인은 이미 보유 기법과 중복이고, 신규인 재구성 PU/단일포트 메모리는 멀티표준 유연성을 위한 것이라 단일 고정 QC-LDPC를 쓰는 Prime ECC엔 실익이 작다.
