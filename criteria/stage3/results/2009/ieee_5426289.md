### B. 알고리즘 요약

1. 대상: BEC(binary erasure channel) 상의 LDPC 부호를 ML 복호. 예시부호는 `(2048,1024)` PEG 기반 systematic IRA, rate 0.5.
2. 문제: IT(iterative) 복호는 stopping set 때문에 waterfall 손실·error-floor가 생겨 ML 대비 열화. ML은 stopping set을 풀지만 `O(n^3)` GE라 장부호에서 비현실적.
3. 기존 접근[16]: `H_K` (erased columns 부분행렬)를 삼각화→zero matrix→GE 3단계 구조적 GE로 sparse성을 활용, GE 대상 `R_L`을 축소.
4. 지배 비용: 최종 pivot 수 `α`가 `R_L` 크기이자 GE 비용의 제곱 인자를 지배(`(χn/α)^2` 만큼 절감). 따라서 `α` 최소화가 목표.
5. 핵심 기여: 삼각화가 막힐 때 어떤 column을 reference variable(pivot)로 뽑을지 정하는 pivoting 알고리즘 4종.
6. MCW: B의 최대 weight column 선택(γ=1)으로 weight-1 row를 많이 생성.
7. MAW: 최소 weight row에 연결된 column들을 pivot으로(항상 언블록 보장, γ≥1).
8. MC: bipartite graph의 maximal component에서 weight-2 row를 골라 pivot(q-1 스텝 언블록 보장).
9. SVN: 신규 개념 super variable node(key node 포함 서브그래프)를 growth phase로 병합 후 최대 degree 노드를 pivot(γ=1).
10. 결과: SVN이 최우수(MCW 대비 평균 pivot 4~5개, MC 대비 ~1개 절감). MCW SW 구현이 상용 PC에서 1.5 Gbps 이상, ETS-VIII 위성 필드 트라이얼 검증. pivoting은 복잡도만 줄이고 ML 정정성능은 brute-force GE와 동일(불변).
