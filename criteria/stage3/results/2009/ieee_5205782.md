> 총평: stochastic decoding 계열의 binary vector message passing 디코더에 syndrome-information 기반 상태적응(W2L 매핑, 메시지 길이)을 결합해 sub-iteration 수를 줄인 알고리즘 논문, min-sum/soft-LLR 기반 Prime ECC와는 메시지 표현 자체가 달라 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: regular (dv,de)=(3,6) LDPC, N=1000, AWGN+BPSK, BVMPA(binary vector message passing algorithm) 기반 - 메시지가 실수 LLR이 아닌 길이Q의 binary vector `b∈{0,1}^Q`.
2. 문제: 기존 BVMPA는 iteration 번호로 W2L(weight-to-LLR) 매핑을 고정 파라미터화하나, 실제 디코딩 경로가 density evolution 예측과 어긋나 매핑이 최적이 아님.
3. 핵심 가정: vector message의 Hamming weight `W`만 정보를 담고(위치 무관), `p(W|X=0)` 분포로 L-value 변환.
4. 핵심 기법: check node의 syndrome bit `S=X1⊕...⊕Xdc=0`에 대응하는 syndrome message weight 분포 `p_Ws(w)`로부터 decoder state(= mutual information) `Is = H((p_Ws(w)+p_Ws(Q-w))/2) - H(p_Ws(w))`를 정의, 데이터 없이 실시간 계산 가능.
5. 핵심 식 의미: 기존엔 iteration 번호로 W2L 매핑을 선택했으나, `Is`(신드롬 메시지의 실측 품질)로 선택하면 실제 디코더 상태를 더 정확히 반영.
6. 확장 2단: 벡터 길이 `Qi`도 상태 적응적으로 가변, 총 sub-iteration `ΣQi`를 최소화하는 trellis 기반 탐색(quantized `Is`를 trellis state로 사용, 동일 상태 중 최저 cost 경로만 유지).
7. 부수 트릭: `Is` 미개선 시 조기종료(10 iteration 동안 미개선이면 중단), lookup table 크기 축소(iteration당 1행 대신 quantized `Is`당 1행, 20 quantization level로 충분).
8. 최적화 절차: density evolution으로 W2L 매핑, 메시지 길이 시퀀스를 사전 계산.
9. 결과: BER=10^-6 근처에서 평균 약 50 sub-iteration만으로 수렴(고정 길이 대비 sub-iteration 수 크게 감소), 가변 길이도 BER 소폭 열화만으로 sub-iteration 추가 감소.
10. 한계: (3,6) 소형 코드 시뮬만 수행, HW 미설계, binary-vector/stochastic 메시지 패러다임 특유의 구조로 실제 LLR 기반 디코더 이식은 재설계 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| binary vector message passing (전체 알고리즘) | 대응 없음 | Prime ECC는 min-sum 기반 magnitude/sign LLR 메시지 구조로 stochastic binary-vector 방식과 이질적 |
| syndrome information 기반 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 목적은 유사하나 계산 방식(mutual information vs CRC)이 전혀 다름 |
| W2L 적응형 lookup table | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 LLR 테이블 개념은 있으나 여기선 vector-weight 기반이라 직접 대응 어려움 |

> 적용 가치: 낮음 — binary vector/stochastic message passing 자체가 Prime ECC의 min-sum LLR 기반 구조와 근본적으로 다른 패러다임이라 직접 이식할 요소가 거의 없음.
