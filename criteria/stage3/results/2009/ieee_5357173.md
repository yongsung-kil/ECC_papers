### B. 알고리즘 요약

1. 대상: binary QC-LDPC(base matrix 6×12, extending factor≤128, 예 `(1536,768)`), AWGN 채널, Normalized Min-Sum(NMSA, 정규화 factor `0.875`).
2. 문제: 기존 재구성형 ASIC/ASIP는 고정 다중모드만 지원 - 새 H-matrix 요구 시 재설계 필요, 채널적응 차세대 시스템에 부적합.
3. 프로그래머빌리티: `Step 0`(setup)에서 스위치 메모리 SCM(채널값)·SMM(상호정보)로 1의 위치·rate·길이를 실시간 다운로드, `mb*nb` 클록으로 임의 QC 행렬 구성.
4. 기법1 DGC(Divided-Group Comparison): 12-input CNU 최소값 비교를 3 stage(4그룹×3신호→그룹간→개별)로 분할해 `log3N` 복잡도로 critical-path 단축, 각 stage=가산기1+MUX1.
5. 기법2 AWA(Adaptive Wordlength Assignment): 초기 `(3-bit int, 2-bit frac)`[-4,3.75]에서 기준 충족 시 `(4-bit int, 1-bit frac)`[-8,7.5]로 전환 - LLR 절반↑ 개수가 codeword 95%(Check_Num) 초과 시 발동.
6. 기법3 EETS(Efficient Early Termination): AWA 발동 후, 모든 decoded bit의 correction distance(SV-CV)가 초기 dynamic range 1/2 초과면 조기종료(Early_flag).
7. HW 구성: CNU/BNU/WDU/ETU 4종 유닛 + SCM/SMM 스위치 메모리 뱅크(부분병렬).
8. 최적화 절차: 없음(경험적 임계 Check_Num=95%, threshold=DR 1/4·1/2).
9. 결과: `(1536,768)` 8-iter에서 AWA로 coding gain `0.13dB@1e-5`, `0.42dB@1e-6`; 칩 TSMC 0.13um, `358.4K` gate, `4.94mm2`, `125MHz`, `86Mbps`, `58mW`(0.9V시 40.5MHz/10.2mW), 2~8 iteration.
10. 한계: coding gain은 H-matrix 종속(범위 밖 논의), 프로그래머빌리티가 면적/전력 오버헤드 유발.
