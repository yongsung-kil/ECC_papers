### B. 알고리즘 요약

1. DMB-TH용 QC-LDPC 디코더, rate `0.4/0.6/0.8`, code length `7493`bit, submatrix `127×127`, `59` col-block, VN degree 6/7·11/12·25/26.
2. 문제: 듀얼포트 메모리는 면적·전력·라우팅 비용이 커서, 단일포트 메모리만으로 고throughput 디코더를 구현하는 것이 목표.
3. 알고리즘: TDMP(layered) + normalized MS. sub-iteration마다 한 check group을 갱신한 뒤 즉시 VN을 갱신(TDMP로 반복수 최대 50%↓, 성능저하 없음).
4. 핵심 식: `Rmn = Ω·sign(Q'n)·min|Q'n|` (Ω=정규화 factor `0.75`), 이어 `Qn = Σ Rmn`으로 VN 갱신.
5. 정규화 factor `0.75`는 min-sum의 과대추정 오차를 보정하여 SP 대비 성능저하를 막는 역할.
6. CVPU(check&variable node process unit): check/VN 갱신을 단일 사이클 파이프라인으로 재사용·재구성(가산기·R3·data-prep 공유) → 127개 CVPU에서 면적 절감.
7. min1/min2 값과 index만 저장해 check message `Rm`을 압축 저장(비트 절감).
8. SS-CS 셔플 네트워크: 순환·고정크기(127×127) 성질을 이용, `log2N` 스테이지 2×1 mux로 라우팅 지연 단축(Benes/Banyan 대비 구성 단순).
9. 메모리 지배(memory-dominated) 제어로 단일포트 R/W 충돌 회피; check update `d+2`, VN update `2d+2` cycle, 반복당 `r(3d+4)` cycle.
10. 결과: Xilinx XC4VLX160, 임계경로 9.617ns, 104MHz; 반복당 throughput ~`890/847/863`Mbps(rate 0.4/0.6/0.8).
11. 한계: 반복당 throughput만 제시, BER 성능곡선·ASIC gate count 없음, 특정 표준 부호에 특화.
