> 총평: 다중 표준(802.16e/802.11n)의 가변 submatrix 크기를 하나의 barrel-shifter 기반 네트워크로 지원하는 HW 라우팅 구조 논문으로, ASIC 합성까지 검증됐지만 정정성능 개선이 아닌 순수 인터커넥트 면적/유연성 최적화이며 Prime ECC처럼 고정 z를 쓰는 코드베이스에는 적용 동기가 제한적.

### B. 알고리즘 요약

1. 시스템은 IEEE 802.16e(19가지 submatrix 크기 z=24~96, step 4)와 802.11n(3가지 z=27/54/81) QC-LDPC 코드를 하나의 디코더 칩으로 지원, base matrix `Hb`(mb×nb, nb=24)를 z×z circulant/zero 블록으로 확장한 구조.
2. 문제는 다양한 submatrix 크기 z(114+12 모드)를 지원하려면 고정 크기 crossbar(Benes network 등)나 self-routing-bit(SRB) 삽입 방식이 필요한데, 이는 라우팅 결정 규칙이 복잡하거나 게이트 수가 커짐.
3. 핵심 모델은 z-symbol barrel shifter를 zmax-symbol barrel shifter로 대체하고, shift amount `p`와 목표 크기 `z`만으로 출력 메시지를 선택하는 규칙.
4. 핵심 기법(SRN, Shift-Routing Network) 3단 구조: 1단 zmax-symbol barrel shifter(소스 메시지 shift), 2단 두 개의 "routing decision data" 생성(`(zmax-z+1)`~`zmax`번째와 `1`~`z`번째 shifted message), 3단 shift amount `p`와 `zmax/2` 비교로 두 후보군 중 최종 출력 선택.
5. 핵심 식: subiteration cycle `k_sub = 1 + nrow/2`, iteration당 latency `T_iter = k_sub × (t_init + 2·n_col + t_wb)`, throughput `= (n_col × z × f_clk) / (T_iter × l)` — SRB 삽입이나 별도 제어회로 없이 barrel shifter만으로 임의 z를 처리해 라우팅 결정 로직을 단순화.
6. 확장/2단 기법 없음 — SRN 단일 네트워크를 CNP(check-node processor)-BNP(bit-node processor) 간 공유 메시지 경로에 적용, 두 개의 SRN을 병렬 배치해 병렬도 "2" 확보.
7. 구현 디테일: sign-magnitude(SM)→2's complement(TC) 변환, 4-stage subiteration(memory prefetch, CNP/BNP switch, CNP/BNP operation, message update), memory prefetch로 메모리 접근 latency 은닉.
8. 학습/최적화 절차 없음 — 규칙 기반 라우팅 회로 설계.
9. 결과: 90nm 1P9M 공정에서 6.22mm², 300MHz 포스트레이아웃, rate-5/6 2304-bit 코드 min-sum 20 iteration 기준 212Mb/s(802.16e)/178Mb/s(802.11n, 1944-bit), 357K 로직게이트+590K 메모리, 528mW@1.0V; SRN 단독은 27.1K게이트(self-routing network 40.1K, m-way duplicated network 52.3K 대비 절감).
10. 한계: min-sum 알고리즘/정정성능 자체는 기존 그대로이며 순수 HW 인터커넥트(라우팅) 최적화에 한정, error-floor/waterfall 등 디코딩 품질 개선 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Shift-Routing Network(메시지 라우팅, barrel shifter 기반 CNP-BNP 간 shift/switch) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | C2V/V2C 메시지 전달 경로의 shift-routing 구조로, Prime ECC의 z_sb=32 고정 row 병렬 처리와 개념적으로 유사하나 논문은 가변 z(24~96) 지원이 핵심이라 직접 대응은 제한적 |
| Graph Thinning(HW edge/MUX 씨닝)과의 비교 대상 | `GT.cpp` `Make_GT_HW()` | 둘 다 MUX/셔플 기반 HW 메시지 전달 최적화 계열이나 SRN은 multimode submatrix 크기 대응이 목적, GT는 edge 축소가 목적으로 최적화 축이 다름 |
| CNP/BNP 4-stage subiteration, memory prefetch | `decoder.cpp` pipeline 6-stage 구조 | 파이프라인·메모리 프리페치 개념은 유사하나 구체적 스테이지 구성은 다름 |

> 적용 가치: 낮음 — Prime ECC는 z_sb=32 고정 병렬 단위의 단일 부호구조를 쓰므로, 가변 submatrix 크기(19+3종)를 위한 SRN의 multimode 라우팅 유연성은 직접적 필요성이 낮고, 정정성능 개선도 없는 순수 인터커넥트 면적 최적화임.
