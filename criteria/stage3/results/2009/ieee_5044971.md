### B. 알고리즘 요약

1. 시스템: LDPC 코드의 Tanner graph `T` 및 그 유한 커버(finite cover) `T̃`를 다루는 순수 그래프이론 프레임워크, 특정 부호/rate/length 없음.
2. 문제: min-sum/sum-product 반복복호기의 실제 동작(computation tree)과 이론적으로 분석하기 쉬운 graph cover pseudocodeword 이론 사이의 간극을 규명.
3. 핵심 가정: Wiberg(1996)의 결과 - min-sum m회 반복 후 출력은 depth-m computation tree 상의 최소비용 configuration과 동일하다는 사실을 출발점으로 함.
4. 핵심 기법: computation tree configuration `(c,R)`이 항상 어떤 finite graph cover `T̃`의 configuration `(c̃,T̃)`으로 실현(embed) 가능함을 증명 (Proposition 2.1, Theorem 2.3/2.4/2.6).
5. 핵심 식: 최소 row weight ≥2인 패리티체크행렬 `H`에 대해 `T`의 computation tree configuration은 `4M`-cover(정리 2.4) 또는 더 타이트한 `(3M(c)+N(c))`-cover(정리 2.6)로 실현되며, 이때 대응 normalized pseudocodeword는 `(1/2,...,1/2)` 근방 값을 가짐.
6. 확장: cycle code(row degree 2)의 경우 결과가 더 깔끔해짐(Theorem 2.3, cover degree `M` 또는 `2M`).
7. 부수: cover degree 상한을 좁히기 위해 configuration별 check node 복제 개수 `M(c)`, `N(c)`(값 1 인접 여부로 구분)를 정의해 Theorem 2.4보다 개선된 bound 제공.
8. 최적화 절차: 없음 (수학적 증명 및 예제 검증뿐).
9. 결과: 정량적 실험 결과 없음. 예제(Example 3.1~3.4)를 통해 정리의 tightness를 그림으로 보임(예: 12-cover 필요하던 경우가 실제로는 5-cover로 충분함을 보임).
10. 한계: 순수 이론 논문으로 시뮬레이션/BER/HW 결과 전무, 특정 부호 클래스(예제)에 국한된 소규모 검증뿐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| computation tree / graph cover pseudocodeword 이론 | 대응 없음 | Prime ECC는 bit-exact HW 시뮬레이터로 pseudocodeword 이론적 분석 프레임워크를 사용하지 않음 |
| min-sum 알고리즘의 이론적 동작(Wiberg 특성화) | `decoder.cpp` `LDPC_Decoder()` | min-sum 자체는 이미 구현되어 있으나 본 논문의 pseudocodeword 이론은 error-floor 원인 분석 참고자료 수준 |

> 적용 가치: 낮음 — 순수 조합론/그래프이론 증명 논문으로 코드 구성, HW, 시뮬레이션 요소가 전혀 없어 이식할 구체적 알고리즘이 없음.
