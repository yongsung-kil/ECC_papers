### B. 알고리즘 요약

1. 시스템: LDPC 부호화된 신호가 PR(partial-response) 채널(임펄스응답 `h(D)`, 메모리 길이 `m`)을 통과, AWGN으로 오염된 채널 출력 `r=y+z`를 수신, 실험은 PR4 채널(`[1,0,-1]`)·(1908,rate 0.89) 랜덤 LDPC 코드 사용.
2. 문제: 기존 turbo-equalizer(SISO 검출기 BCJR/SOVA + SISO LDPC 디코더 간 반복 교환)는 채널정보와 코드정보를 번갈아 쓸 뿐 동시에 쓰지 못해 성능 한계.
3. 핵심 가정: 코드가 채널 입력에 제약을 가하면 그 제약이 noiseless 채널 출력 심볼 시퀀스에도 제약을 유도한다는 관찰 — 이를 이용해 디코더가 채널 입력이 아닌 채널 출력 심볼의 APP를 추정하도록 재구성.
4. 핵심 기법: 채널 트렐리스를 state-free 이분그래프 `G`(symbol node `y_i` ↔ channel node `s_i`)로 표현하고, 여기에 LDPC parity check node를 추가한 tripartite 그래프에서 sum-product 알고리즘(SPA)을 수행하는 joint decoding 알고리즘.
5. 핵심 식: check node에서 심볼 APP 계산이 기존 `tanh` 연산 대신 `p(y_k|r_i, C_i)`를 BCJR로 계산하는 "expanded code trellis"(check node의 degree `w_r`에 대해 상태전이 엣지를 `2^m`개로 확장) 연산으로 대체됨 — 채널 메모리 정보를 체크노드 연산에 직접 반영.
6. 확장: symbol node 간 독립성이 깨질 경우(실험 코드의 212개 체크 중 100개가 비독립 쌍 포함) 근사 계산으로 처리, 무수정 적용 가능함을 실증.
7. 구현 디테일: 체크노드 연산 복잡도는 `(3+2^m)·2·w_r`개 저장, `(1+5·2^(m-2))·8·w_r` 곱셈, `(1+2^(m+1))·2·w_r` 덧셈 필요(BCJR 기반이라 tanh 대비 훨씬 무거움).
8. 학습/최적화 절차 없음(고정 알고리즘, 파라미터 튜닝만).
9. 결과: SNR 5.4dB에서 joint decoder BER이 turbo-equalizer 대비 약 1자릿수(10배) 개선, iteration은 60회 소요(turbo-equalizer는 global 5×internal 3=15회) — 즉 성능 향상은 iteration/복잡도 증가와 맞바꿈.
10. 한계: PR(ISI) 채널 전용 기법이며 HW 미설계, 시뮬레이션만 수행, 코드 독립성 가정이 깨질 때 근사 계산 사용, 후속 저널판에서 다양한 채널/rate 결과 예고.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| joint channel-code message-passing (PR/ISI 채널 결합) | 대응 없음 | Prime ECC의 `channel.cpp`는 AWGN/RBER/fixed-error만 지원하며 ISI/PR 채널 모델 부재 |
| check node의 tanh → BCJR 대체 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 개념적으로 CN 연산부에 닿지만 PR 채널 전제가 없으면 무의미하며 오히려 min-sum보다 훨씬 무거운 연산으로 역행 |

> 적용 가치: 낮음 — ISI/PR 채널 특화 기법으로 NAND의 memoryless 채널 모델과 근본적으로 불일치하며 복잡도도 대폭 증가시켜 이식 실익 없음.
