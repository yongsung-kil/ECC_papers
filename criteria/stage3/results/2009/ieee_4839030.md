### B. 알고리즘 요약 (10줄 내외)

1. 시스템은 systematic LDPC 부호 `x=(s,p)`(정보비트 s, 패리티비트 p), 파리티체크행렬 `H=[A|B]`로 `sA^T + pB^T = 0`을 만족해야 하며 목표는 sum-product 디코더 회로를 인코딩에도 재사용하는 것.
2. 풀려는 문제: 기존 인코딩은 삼각화(back-substitution)나 (quasi-)cyclic shift-register 방식이라 직렬 계산이라 지연이 부호길이에 선형 비례하고 디코더와 별도 회로가 필요해 라우팅 면적을 추가로 씀.
3. 핵심 가정: 인코딩을 `pB^T=sA^T` 선형시스템 풀이(=행렬 역변환)로 보고, 이를 GF(2) 상에서 반복적으로 근사하는 고전 Jacobi 방법으로 치환.
4. 핵심 기법 1단: Jacobi iteration `x_{k+1} = D^{-1}(D-B)x_k + D^{-1}c` (GF(2) 특유화, `B`를 D(대각)+오프대각으로 분해)을 factor graph 상의 message-passing으로 재해석 — 결정론적으로 유한 스텝(`B^m=0`) 내 수렴하는 것이 특징(실수 spectral radius 조건과 달리 상삼각이면 항상 수렴, Theorem 2).
5. 핵심 식은 iterative encodability constraint `(B+D)^m=0`이며, 이를 만족하는 circulant matrix(다항식 `b(x)`)를 4-cycle 방지(overlap constraint, separation class)와 함께 설계(Algorithm 2)하면 `m`회 반복으로 인코딩 완료.
6. 확장 기법 2단(Type-II): 원형(circulant) 구조가 만드는 반복 그래프 패턴이 낮은 확장성(spectral gap)·낮은 pseudo-weight 근접부호워드를 유발함을 분석 후, Tanner의 재귀 구성(Kronecker product 템플릿+랜덤 성분)으로 그래프 확장성을 개선한 Type-II 부호를 제안.
7. 부수 트릭: 인코더가 디코더와 동일 회로/라우팅을 시분할 공유하므로 검증 시 라우팅 오류를 저비용 인코더 시뮬레이션으로 조기 발견 가능; bidirectional soft-logic gate count(BSGC)를 복잡도 지표로 사용.
8. 최적화 절차는 없음(알고리즘적 구성) — 최소거리 상/하한(Theorem 8), stopping set 분석, finite graph-cover 기반 pseudo-codeword 분석으로 구조적 약점을 사후 진단.
9. 결과: n=4096, rate≈0.75 Type-I 부호는 랜덤 벤치마크 대비 AWGN/BEC에서 error floor(BER~1e-6~1e-8대) 발생; column weight를 5로 올리거나 Type-II(재귀) 구성 시 랜덤 부호와 유사 성능(floor 미관측)으로 개선. 고rate(r≈0.6~0.763) 구성에서 EG-LDPC/quasi-cyclic 대비 0.08~0.26dB 열위.
10. 한계: HW 실합성 없음(BSGC로 근사 추정만), 시뮬레이션(AWGN/BEC)에 국한, Type-I 구조는 근본적으로 error-floor 취약(그래프 반복패턴), 부호 rate/길이가 예제별로 다양해 단일 확정 곤란.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Jacobi 반복 기반 인코딩(디코더 회로 재사용) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코더(부호용 PCM `PCM_b_enc` 별도)를 쓰므로, 디코더 재사용형 인코딩으로 교체 시 인코더 모듈 전면 재설계 필요(이식 난이도 중~높음) |
| 4-cycle-free circulant 부호 구성(overlap/separation class 제약) | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 제약 조건으로만 참고 가능, Prime ECC는 특정 QC-LDPC 고정이라 구조 교체 부담 큼 |
| stopping set / pseudo-codeword(그래프 확장성) 기반 error-floor 진단 | 대응 없음 | 정성적 분석 기법일 뿐 코드 모듈로 실존 대응 없음 |
| 인코더-디코더 회로 시분할 공유(면적 절감) | `decoder.cpp` `LDPC_Decoder()` / `encoder.cpp` (개념적 연계) | HW arch 재구조화 아이디어로만 참고, 현재 시뮬레이터는 별도 encoder/decoder 모듈 구조라 직접 대응 없음 |
```

적용 가치: `중간` — 부호설계(4-cycle-free 구성) 및 인코더 HW 면적 절감 아이디어는 참고할 가치가 있으나, Type-I 구조의 error-floor 취약성과 Prime ECC의 고정 dual-diagonal 인코더/QC-LDPC 구조와의 정합 문제로 직접 이식은 재설계 수준의 작업이 필요.
