### B. 알고리즘 요약

1. 시스템: DVB-S2 표준 IRA LDPC, normal frame 64800비트(단문 16200비트도 지원), rate 1/4~9/10(11종), AWGN 채널, BPSK.
2. 기존 SMP(flooding) 기반 BP/Min-Sum/Offset Min-Sum은 반복당 수렴이 느리고, 기존 저복잡도 설계([3][4][5])는 메모리 최적화나 병렬도 절충에서 한계.
3. 핵심 가정: check node degree가 (첫 노드 제외) 균일하다는 IRA 구조 특성을 이용해 check-node 중심 처리로 제어 복잡도 절감.
4. 핵심 기법: RMP(Row Message Passing) 스케줄 — variable node를 대응 check node 갱신 직후 즉시 갱신하여 같은 iteration 내에서 업데이트된 정보를 재사용, RMP 1회가 SMP 다중 iteration에 상당하는 효과.
5. 핵심 식: `Λ(k)_{mn} = sign · max(min_{n'∈S(m)\n}|λ_{n'}-Λ(k-1)_{mn'}| - β, 0)` (offset min-sum의 check-node 갱신식), offset β=0.5로 저SNR 성능 열화(순수 min-sum 단점)를 보정.
6. 확장: 메모리 구조를 코드rate/frame length 전반에 걸쳐 최적화 — RAM V(variable LLR), RAM C(check message: min/2nd-min/sign/index)를 파라미터 q(rate별 circulant 크기), Dc(check degree)에 따라 rate group별로 다르게 row당 1~2 check node 배치.
7. 구현 디테일: 6bit(sign+3int+2dec) 균일양자화 대신 RAM C는 5bit(min/2nd-min), RAM V는 8bit(λn)로 비균일 양자화 채택(저SNR 성능 개선), barrel shifter 3-stage 파이프라인, RAM V는 dual-port+read bypass로 conflict 회피.
8. 학습/최적화 절차 없음(고정 알고리즘, 파라미터 β는 시뮬레이션으로 선정).
9. 결과: RMP-Min-Sum이 동일 iteration수(15)에서 SMP-BP 대비 BER 10^-6에서 0.6dB 이득, 고정소수점 디코더는 부동소수점 RMP-Min-Sum 대비 0.1dB 손실, ASIC 합성 결과 90nm 9.6mm², 320MHz, rate 9/10에서 net throughput 998Mbps(최소 rate 1/4에서 181Mbps), 메모리 1.68Mb.
10. 한계: DVB-S2 표준(21종 rate/frame length 조합)에 특화된 구조라 범용성보다 표준 준수 목적, error-floor 영역 분석/수치는 본문에 없음(waterfall 영역 BER만 제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RMP(Row Message Passing) 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, Dual-Update(odd/even 교대) 옵션 | 즉시 갱신 정보 재사용으로 수렴 가속하는 스케줄링 개선 아이디어로, 기존 Dual-Update와 비교·결합 검토 가치 있음 |
| Offset Min-Sum CN 갱신 (min1/min2 + offset β) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 offset 파라미터 튜닝 방식 직접 비교 가능, 다만 offset 기법은 이미 표준적 |
| rate별 RAM V/C 메모리 구조 최적화(row당 check node 1~2개 배치) | `decoder.cpp` HCU(Half Column Update), `ecc_data.h` PCM 구조 | 다중 rate 지원을 위한 메모리 재구성 기법이나 Prime ECC는 고정 QC-LDPC 구조라 직접 이식 난이도 중간 |
| 비균일 양자화(RAM C 5bit, RAM V 8bit) | `ecc_data.h` `PARAM_LLR`, `channel.cpp` LLR 양자화 테이블 | soft-info 비트폭 설계 참고 가능, NAND soft-read 비용과 직결되는 4bit+ 영역이라 이식성 중간 |
| barrel shifter 기반 360-way interconnect | 대응 없음 | Prime ECC z_sb=32 대비 훨씬 큰 병렬도(360)의 인터커넥트로 구조 자체가 상이 |

> 적용 가치: 중간 — DVB-S2 특화 구조지만 min-sum 계열 스케줄링(RMP) 및 rate별 메모리 최적화 기법이 Prime ECC 디코더 아키텍처와 직접 비교·참고 가능한 수준.
