### B. 알고리즘 요약

1. 대상 채널/표준: IEEE 802.11n WLAN, QC-LDPC 코드 3종 base matrix(rate 1/2, (1944,972)/(1296,648)/(648,324)), min-sum algorithm(MSA) 기반 디코더.
2. 문제: 3개의 서로 다른 base matrix를 하나의 하드웨어로 멀티모드 지원 시 메모리뱅크 재사용을 위해 단순 MUX 삽입은 면적/배선 오버헤드가 큼.
3. 핵심 기법 1: Matrix Reordering — 두 base matrix `HA`, `HB` 간 행/열의 "Diversity"(공통 위치가 아닌 원소 수)를 정의하고, `Diversity-Minimized Row/Column Reordering(DMRR/DMCR)`으로 공통 메모리뱅크 위치를 최대화.
4. 핵심 식: `Minimize z = ΣΣ x_row,i,j · D_row,i,j` (job assignment 문제로 정식화, 헝가리안 알고리즘으로 다항시간 해결).
5. 의미: 완전탐색 O(n!) 대신 O(n3)으로 최적 재정렬을 찾아 3개 base matrix(802.11n 실제 사례)를 0.07초에 해결(완전탐색 726.5초 대비).
6. 핵심 기법 2: SEMD(Single-Entry-Multiple-Data) — 메모리 한 엔트리에 m개 연속 데이터를 묶어 저장(entry 수 p→p/m), 클록당 m개 데이터 처리로 처리량 m배 향상.
7. 구현 디테일: SEMD 적용 시 shift value의 나머지 r에 따라 데이터 정렬용 DSC(Data Switching Circuit) 3종이 필요하며, 멀티모드에서는 이를 통합한 R-DSC(5개 추가 MUX)로 재구성.
8. 최적화 절차: 헝가리안 알고리즘(단계별 행/열 축소·마킹·반복)을 이용한 diversity table 최소화.
9. 결과: TSMC 0.13um 기준 트리플모드 디코더 1.99mm2@172.4MHz(단일모드 3개 합 대비 면적 60.57% 감소), R-DSC 면적 오버헤드 평균 66.23%, MUX 34.38% 감소.
10. 한계: rate 1/2만 구현(다른 rate 미검증), 알고리즘 자체의 오류정정 성능 개선 없음(순수 재사용 아키텍처), non-802.11n 코드로의 일반화는 방법론만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 멀티모드 base matrix 재정렬(DMRR/DMCR) | 대응 없음 | Prime ECC는 단일 고정 QC-LDPC(`Load_PCM()`)로 멀티모드 재구성 대상이 아님 |
| SEMD 메모리 스킴(엔트리당 m개 데이터) | 대응 없음 | HCU(`decoder.cpp`)와 유사한 처리량 향상 방향이나 실존 대응 함수 없음 |
| 디코딩 알고리즘(MSA) | `decoder.cpp` `LDPC_Decoder()` | 표준 min-sum이며 이미 보유 기법, 개선 없음 |

> 적용 가치: 낮음 — 멀티모드 재구성용 하드웨어 재사용 기법으로, 단일 고정 QC-LDPC를 사용하는 NAND Prime ECC 3.1과는 문제 설정 자체가 다르며 정정능력/처리량 개선도 없음.
