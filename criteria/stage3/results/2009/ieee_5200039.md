### B. 알고리즘 요약

1. 대상은 중국 CMMB(모바일 멀티미디어 방송) 표준의 rate-1/2, 길이 9216-bit QC-LDPC 부호이며, H 행렬은 18×36 base matrix에 서브행렬 크기 `l=256`인 circulant 확장(대부분 weight-1, 1개 서브행렬만 weight-2).
2. 풀려는 문제: 긴 코드길이(9216bit) LDPC 디코더는 반복 메시지 저장용 메모리가 칩 면적/전력을 지배하므로, 방송·모바일 단말에 맞는 면적/전력 효율적 디코더 구조가 필요.
3. 핵심 가정/모델: BP 대신 Min-Sum Algorithm(MSA)으로 check node 연산(`r_mn[k] = sign(...) × min|q_mn[k]|`)을 근사해 복잡도를 낮추고, Layered Decoding Algorithm(LDA)으로 층별 즉시 갱신(`q_{j,n}=Λ_n−r_{j,n}`, `Λ_n=q_{j,n}+r_{j,n}`)해 수렴 가속.
4. CTV 메시지 압축: 한 행(row weight 6)의 CTV 메시지 6개를 개별 저장하지 않고 (최소값, 2번째최소값, 최소값 위치 인덱스, 전체 부호 6비트)만 저장 → row당 `18m`bit에서 `13m`bit로 약 28% 메모리 절감(quantized bitwidth 3bit 가정).
5. 반복 메시지 양자화: 부동소수점 시뮬 결과(BER 1e-7 @ SNR 2.5dB, CMMB 요구 BER 1e-4 초과 달성) 대비 여러 고정소수점 스킴을 비교, 정수부 2bit+부호 1bit(총 3bit)가 CMMB BER 요구를 만족하는 최소 비트폭으로 채택.
6. Interconnect 최적화: QC-LDPC의 서브행렬 구조를 이용해 큰 `m×n` read/write switching network를 `mb×nb`(18×36) → row weight 6을 활용해 `36×6` 네트워크로 축소, 실제로는 4개의 6:1 mux + 2개의 7:1 mux(Read), 대응 demux(Write)만으로 구현.
7. 부수 트릭: Recovery block으로 압축된 CTV에서 개별 메시지를 복원, APP Memory Bank에서 채널 초기값(intrinsic info) 로드 및 반복 갱신 값 저장을 분리 관리.
8. 학습/최적화 절차 없음 (CMMB 표준 부호를 그대로 사용, girth 확대된 기존 최적화 코드 채택).
9. 결과: 90nm 1.0V CMOS 합성 결과 면적 2.6mm², 60MHz 동작 시 48Mbps(5회 반복)에 전력 34mW, 최대주파수 181MHz 시 최대 145Mbps; 기존 DVB-S2급 디코더(F. Kienle: 22.7mm², P. Urard: 15.8mm²/700mW) 대비 면적·전력 대폭 절감.
10. 한계: rate-1/2 9216-bit 코드 1종만 구현·검증했고 ASIC 합성 결과이나 실측(chip measurement) 데이터는 없으며, 정정능력 자체의 개선(waterfall/error-floor)은 다루지 않고 순수 HW 면적/전력 최적화에 집중.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum CN 연산 (min1/min2 + sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 동일한 min-sum(min1/min2+sign XOR) 방식을 보유해 중복 기여, 새로운 이득 없음 |
| Layered Decoding(층별 즉시 갱신) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 Dual-Update(parity even/odd 교대) 스케줄과 유사 개념, 직접 대응은 불명확 |
| CTV 메시지 압축 저장(min/2nd-min/idx/sign) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_DEC` | 메모리 절감 기법으로 HW 메모리 구조 이식 시 참고 가능하나 코드 접근 없이 세부 대응 불명 |
| 3-bit(2정수+1부호) LLR 양자화 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | Prime ECC의 soft-2~3bit 지원과 일치하는 양자화 비트폭 참고 가치 있음 |
| QC-LDPC 구조 기반 read/write network(mux/demux) 축소 | `decoder.cpp` pipeline (col_M1/Z/P1~P4), `GT.cpp` `Make_GT_HW()` | 서브행렬 구조 이용한 인터커넥트 축소는 Prime ECC의 Graph Thinning(MUX 기반 edge 선택)과 유사한 방향성 |

> 적용 가치: 중간 — binary QC-LDPC decoder HW라는 점에서 직접 관련되나 min-sum/layered decoding은 Prime ECC가 이미 보유한 기법과 중복되고, CTV 압축·양자화·interconnect mux 축소는 메모리/면적 최적화 참고 자료로서 제한적 가치.
