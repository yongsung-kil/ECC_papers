### B. 알고리즘 요약

1. BSC/AWGNC 채널에서 BP, Gallager A/B(hard-decision), LP(linear programming) 디코더의 디코딩 실패를 통일적으로 분석하는 instanton 기법을 다룸.
2. 문제의식: 유한 길이 코드의 error floor 영역은 Monte-Carlo 시뮬로 도달 불가능한 매우 낮은 확률 영역이라 해석적 방법이 필요.
3. Instanton은 zero-codeword와 pseudo-codeword 사이 경계(error surface)에서 디코딩 실패를 유발하는 가장 확률 높은 노이즈 구성으로 정의됨.
4. AWGNC에서 instanton은 `yˆ`의 L2 노름 `Σyˆi²`을 최소화하는 실패 노이즈로, amoeba 최적화 또는 pseudo-codeword search(PCS)로 탐색.
5. FER은 `FER(s) ≈ Σ_β N_β P_s(yˆ_β)` (β=최상위 k개 instanton, N_β=대칭군에 의한 중복도)로 근사되며, instanton weight가 error floor 구간 FER 곡선의 기울기를 결정.
6. BSC에서는 LP 디코더용 ISA(instanton search algorithm)가 다항시간 내 수렴이 보장되는 형태로 제시됨 (median pseudo-codeword 반복 축소).
7. 서로 다른 채널/디코더의 instanton이 동일한 위상 구조(예: (5,3) trapping set)를 공유한다는 경험적 관찰이 핵심 통찰.
8. 코드 설계 절차: 기존 코드(PEG 등)에서 최악 instanton 구조를 찾고, forbidden-subgraph 회피 PEG 변형 알고리즘으로 해당 구조를 배제한 새 코드를 재구성 반복.
9. [155,64,20] Tanner code 대비 새 코드는 Gallager A 디코더에서 error floor 기울기가 3→4로 개선(critical number 증가), LP/BSC에서 최소 instanton weight 5→6으로 증가.
10. 한계: 소형 예제 코드(길이 155)로만 검증, HW 설계 없음, AWGNC 결과는 실험적 관찰이며 엄밀한 증명은 BSC에 국한.
