> 총평: 파이프라인 레이어드 디코더의 메모리 충돌(cutting-edge) 문제를 행렬 재정렬(스플릿)과 GA 기반 레이어 스케줄링으로 해결하는 아키텍처 레벨 기법으로, 정정능력과는 무관하며 Prime ECC의 병렬/스케줄링 구조와는 이질적이나 파이프라인 충돌 회피 아이디어는 참고 가치 있음.

### B. 알고리즘 요약

1. DVB-T2 QC-LDPC 코드(base matrix m×n, P=360 circulant shift, 예: rate 2/3 short-frame (15,45) base → (5400,16200) 확장)를 대상으로 한 layered(horizontal) belief propagation 디코더.
2. 레이어드 디코딩을 파이프라이닝하면 처리량이 2배가 되지만, 동일 SO(soft-output) 변수를 연속 서브이터레이션에서 읽고 쓰는 "cutting-edge" 메모리 충돌이 발생.
3. 충돌은 두 종류: `i+1` 충돌(연속 레이어 간 공유 VN)과 CN 지연 `E`로 인한 `i+2` 충돌.
4. 기여 1 - 그룹 스플릿: `S`개로 분할, `a(i) = (i mod S)*Ps + floor(i/S)`로 행/열을 재배치해 병렬도 `P`를 `P/S`로 낮추는 대신 base matrix를 sparser하게 재구성해 충돌 후보군 자체를 줄임.
5. 충돌 수는 `c(i,j) = |vgCgi ∩ vgCgj|`(식 8)이며, 레이어 순서 `π`에 대한 총 비용 `Cit(π) = Σ c(π(i),π(i+1)) + c(π(mg),π(1))`(식 9)를 최소화하는 문제로 정식화.
6. 기여 2 - 스케줄링: 위 최소화 문제를 완전그래프 위 최소비용 해밀턴 사이클, 즉 외판원 문제(TSP)로 등가 변환하고, `i+2` 충돌까지 반영한 가중 비용함수(식 11, 계수 `α`로 i+1 우선)를 유전 알고리즘(GA)으로 풀이.
7. GA는 초기 개체군 1000, 세대수 1000 설정, 2-라운드 토너먼트 선택 + 2점 교차 + 랜덤 스왑 뮤테이션 구조.
8. 학습/최적화 절차: DVB-T2 전체 code rate(1/2~5/6)와 short/long frame, 분할수 `Ps`(=360,180,...,40) 조합에 대해 GA로 무충돌(또는 i+1만 무충돌) 스케줄을 탐색.
9. 결과: `Ps=40`에서는 모든 code rate·frame type에서 i+1, i+2 충돌 모두 없는 스케줄을 발견(Table I/II), 해 탐색은 표준 데스크톱에서 15분 이내.
10. 한계: HW 구현·면적·실측 throughput 평가는 미수행(future work로 명시), 순수 스케줄링/알고리즘 레벨 결과만 제시하며 정정성능(BER)에는 영향 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 레이어드 디코더 파이프라인 메모리 충돌 스케줄링 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC의 min-sum flooding/dual-update 스케줄과 구조가 달라(레이어드 아님) 직접 대응 약함 |
| 행렬 스플릿(그룹 재배치)을 통한 병렬도-충돌 트레이드오프 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 재배치 아이디어이나 Prime ECC는 QC base+circulant 고정 구조라 이식 난이도 높음 |
| GA 기반 레이어 순서 최적화 | 대응 없음 | Prime ECC에 레이어드 스케줄링 개념 자체가 없음 |
| SO(soft-output) 값 갱신/CNP 아키텍처 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 근사 CNP라는 점은 유사하나 레이어드 vs Prime ECC의 흐름 방식 상이 |

> 적용 가치: 낮음 — 레이어드 디코더 특유의 파이프라인 메모리 충돌 문제이며 Prime ECC는 flooding/dual-update 기반 구조로 스케줄링 모델 자체가 다르고, 정정성능 개선도 없어 이식 실익이 낮음.
