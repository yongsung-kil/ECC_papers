### B. 알고리즘 요약

1. (c,t)-regular QC-LDPC의 parity check matrix `Hqc`는 b×b 순환/영행렬 서브블록 `Aij`로 구성, 길이 `N=c×t×b`.
2. 목표: 기존 부분병렬 디코더(HUE 50%)나 부분 overlap 디코더(memory access 충돌로 완전 overlap 불가)의 한계를 극복해 HUE~100%와 완전 overlap을 동시 달성.
3. 채널/노이즈 모델 서술 없음 (아키텍처/스케줄링 논문, BER 결과 없음).
4. 핵심 기법: CNUG(check node process unit group)를 전통적 c개 대신 1개만 두고, c개 check node group을 순차 처리하여 메모리 접근 충돌을 원천 차단.
5. 스케줄: `Address_CNU`/`Shift_num`을 `count=mod(hij,v)`, `Shift_num=floor(hij/v)`로 초기화, 매 사이클 barrel shifter를 회전시켜 서로 다른 VNUG-CNUG 간 메시지를 정렬.
6. 확장: 이전 iteration의 check-to-variable 메시지를 다음 iteration 내내 사용 가능하도록 `MAij`(현재 iteration용) + `MBij`(참조용) 이중 메모리 구조를 추가.
7. 구현 디테일: 한 그룹(b개 노드) 갱신에 `(v+1)` clock, 전체 iteration에 `c×(v+1)` clock 소요; throughput은 `(t/c - 1)×p / [(p/b+1)×I] × f_clock`로 추정.
8. 별도 학습/최적화 절차 없음. PEG(progressive edge growth) 기반 H-matrix 구성 시 variable node group 간 성능 불균형을 줄이기 위해 "전체 그룹을 하나로 보고 edge를 순차 선택"하는 스케줄링 변형만 제안.
9. 결과(정량): CNU/barrel shifter는 `(c-1)/c`, counter는 `(cxt-2t)/(cxt)` 비율로 감소(이론적 산출, 시뮬/합성 없음); 대신 메모리 공간은 2배 증가.
10. 한계: 순수 아키텍처/이론 설계로 FPGA/ASIC 합성 결과 없음, BER/정정성능 데이터 전무, c>2일 때 throughput 저하 언급.
