### B. 알고리즘 요약

1. 대상은 이진 선형 LDPC 코드의 Tanner 그래프이며, 채널은 BEC(binary erasure channel)로 가정, 목표는 크기 `w` 이하의 모든 stopping set을 정확히(exhaustive) 열거하는 것.
2. 기존 방법([15] Wang et al.)은 비효율적이라, 저자들은 이전에 제안한 turbo stopping set 열거 알고리즘([28],[29])을 LDPC 매트릭스용으로 확장.
3. 문제는 constraint set(포함/제외/미결정 위치 집합) 위에서의 branch-and-bound 탐색으로 정식화되며, 목표는 Hamming weight `w(x)` 최소인 유효 벡터를 찾는 것.
4. 핵심 알고리즘 SSE(Stopping Set Enumeration)는 branch 단계(미결정 위치를 골라 두 개의 constraint set으로 분기)와 bound 단계(하한 `lb(S)` 계산 후 pruning)로 구성.
5. bound 단계는 active row(가중치 1 row) 인덱스 집합의 최소 커버 문제를 LP relaxation(`(2)`식, simplex method)으로 풀어 하한을 계산 — 정수해가 나오면 tight.
6. branch 단계는 LP 해에서 `x_i > 0`인 미결정 위치 중 unconstrained 이웃 check node 수 `d_i`가 최대인 위치를 lexicographic order로 선택.
7. 확장 iterative decoding(BEC BP 변형)을 통해 constraint set에서 유도되는 벡터를 미리 채워 탐색 공간을 줄임; QC 구조가 있으면 automorphism/circulant shift로 초기 constraint set을 줄여 속도 향상.
8. 별도 절차 없이 완전 탐색이며, 학습/최적화 단계는 없음(단, LP를 반복적으로 품).
9. (155,64) Tanner 코드에서 크기 18 이하 stopping set 전량을 데스크톱 PC로 약 1분에 탐색; 802.16e LDPC 코드의 최소 stopping set 크기·중복도를 정확히 산출.
10. HW 미설계, 순수 시뮬레이션/조합 탐색 알고리즘이며 BEC 채널 가정 하에서만 성능 개선 실험(FER)을 수행 — 다른 채널(AWGN/RBER)이나 min-sum 디코더와의 관계는 다루지 않음.
