> 총평: QC-LDPC의 circulant shift 값을 짧은 유해 cycle(낮은 EMD) 제거 기준으로 탐색 배치하는 순수 부호설계 기법으로, HW 구조(블록 순환 시프트)는 이미 Prime ECC와 동일 계열이나 이 논문 자체는 알고리즘/시뮬 수준에 그침.

### B. 알고리즘 요약

1. Binary-input AWGN 채널, QC-LDPC(구조화 LDPC, HW 파이프라인 친화) 부호를 대상으로 하며 H를 P×P circulant(순열) submatrix들의 D×L 블록 배열로 정의.
2. 풀려는 문제: structured(QC) LDPC는 partially-parallel HW 디코더에 필수지만 "좋은" 그래프 특성(짧은 cycle 최소화)을 갖는 구성법 연구가 부족.
3. 핵심 가정: 짧은 cycle 중 EMD(Extrinsic Message Degree, 외부연결도)가 낮은 것이 Trapping Set/Stopping Set을 유발해 error-floor를 악화시킨다는 그래프 이론적 모델.
4. 핵심 기법: seed graph(작은 D×L seed matrix)의 각 edge에 순환 시프트 값 `s_e`를 할당하는 expansion 단계에서, covering graph의 cycle 존재 조건 `Σ±s_e ≡ 0 mod P` (Theorem 1)을 피하도록 컴퓨터 탐색으로 `s_e` 결정.
5. 핵심 식 의미: 이 합동식이 0이 되지 않도록 `s_e`를 조정하면 seed graph의 특정 closed walk가 covering graph에서 실제 cycle을 형성하지 않게 되어, 짧고 저EMD인 유해 구조를 제거.
6. 확장: 길이 4 cycle과 EMD=0 closed walk(Linearly Dependent Set 유발)를 최우선 순위로 두고, 나머지는 `k/2 + 1.1×EMD` 휴리스틱 점수로 정렬한 리스트(LCW) 순서로 순차 처리.
7. 구현 디테일: 각 edge마다 제약행렬 `CON_ei`를 유지하며 시프트값을 1씩 증가시키는 다단계 탐색(Stage I/II), 연속 실패(cycle not removed) 1000회에서 탐색 종료.
8. 최적화 절차: seed matrix 자체는 Improved PEG로 구성 후, 본 논문의 컴퓨터 탐색(그리디 증분 탐색)으로 shift 값만 최적화 — DE/GA 아님, 규칙 기반 탐색.
9. 결과: N=273(R=0.52), N=1056(R=0.77), N=2048(R=0.5, irregular) 코드에서 무작위 shift 대비 FER/BER 유의미 개선, MacKay 비구조화 코드와 동등 이상 성능(단 MacKay 코드는 VLSI 비친화).
10. 한계: 부호 구성(오프라인 설계) 알고리즘으로 디코더 HW 설계·복잡도·throughput 데이터 없음, 성능평가는 AWGN+BP 시뮬레이션뿐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC seed matrix + circulant shift expansion | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 base matrix+circulant shift(`PCM_b`) 구조이므로 새 H-matrix 설계 시 이 탐색법으로 shift 값 생성 가능 |
| Trapping Set/Stopping Set(저EMD 짧은 cycle) 제거 기준 | 대응 없음 | Prime ECC 코드베이스엔 그래프 사이클 분석/생성 모듈이 없음(고정 H-matrix 로드만) |
| Tanner graph EMD 기반 우선순위 탐색 | 대응 없음 | 부호 생성 툴 자체가 코드베이스에 없어 신규 개발 필요 |

> 적용 가치: **중간** — Prime ECC와 동일한 QC-LDPC(base+circulant) 구조라 부호설계 단계 이식은 가능하나, `Load_PCM()`은 완성된 H-matrix를 읽기만 하고 생성 로직이 없어 이 알고리즘을 넣으려면 별도 오프라인 설계 툴 개발이 필요(§4 "부호설계" 항목 = 이식 난이도 높음과 일치).
