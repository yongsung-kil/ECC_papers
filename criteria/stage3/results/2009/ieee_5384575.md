### B. 알고리즘 요약

1. 대상: length 2304·rate 1/2 LDPC(layered 구조, 아키텍처의 Rotator·Permutation ROM으로 보아 circulant/QC 기반), AWGN+BPSK, 최대 30 iteration.
2. 문제: LDPC 복호의 방대한 메시지 연산·메모리 접근이 전력을 지배(메모리 접근이 전력의 95%) → 총 연산 수를 줄여 저전력화.
3. 기반: layered decoding(H를 M개 layer로 분할, layer 순차 처리로 1 iteration 완성) + min-sum(`Φ` 대신 min, 보상계수 α<1로 성능손실 완화, 식(5)-(6)).
4. 핵심 기법: lazy scheduling — 총정보 `|L(Q)|`가 임계 `Tv` 초과(고신뢰)하면 해당 variable node를 sleep시켜 in/out 메시지 갱신 중단.
5. sleep 시 스킵: 총정보 갱신(6)·check 정보 갱신(5) 생략, bit 정보 계산(3)은 S(4)에 L(r)이 필요하므로 완전 생략 대신 (7)로 단순화(check 정보 메모리 read 1회 절약).
6. error-floor 대책: `Tv`가 너무 낮으면 오결정으로 error floor 발생 → sleeping node를 주기 `iu`마다 강제 wake-up해 전체 갱신, `exec(Tv,L(Q),iu)`가 노드별 참여 여부 결정.
7. HW 변경: variable node status vector(부호길이만큼 register), decision unit(`L(Q)>Tv` 판정), get bit information unit; sleep 시 check/channel 정보 RAM read/write 우회.
8. 효과(정성): sleep 노드에서 RAM 접근 ~75% 절감, 낮은 HW 오버헤드로 switching activity·전력 감소.
9. 결과: `R-L-MSA(Tv=10, iu=5)`가 variable node 연산을 약 60% 감소; 연산량 표에서 SNR 9dB일 때 MSA 대비 59.3% 감소(성능손실 작음).
10. 한계: 단일 부호·시뮬만, 합성/gate count/throughput 없음, lazy scheduling은 Levin[8] 재적용, 원고 수식·서술 손상 심함.
