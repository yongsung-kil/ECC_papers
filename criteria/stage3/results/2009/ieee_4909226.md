### B. 알고리즘 요약

1. (J,L,m)-regular QC-LDPC의 exponent matrix `E(H)`를 순환행렬 `P^{c_ij}`로 확장하는 표준 QC-LDPC 정의, girth≥6 조건은 `Δj1,j2(l1) - Δj1,j2(l2) ≢ 0 (mod m)` (Theorem 1).
2. 기존 array code(Fan)는 circulant 크기가 소수여야 하고, 수정판(Abematsu)은 임의 크기 지원하나 short cycle이 남는 한계 존재.
3. AWGN 채널 가정, sum-product 알고리즘 50 iteration 디코딩으로 시뮬레이션.
4. 핵심 기법: `Z*_m`의 부분군 `⟨σ⟩` (order `δm`)로 `Z*_m`을 ε개의 서로소 coset `[τs]=τsσ`로 분할, coset leader `τ1,...,τv`를 이용해 지수행렬 `E(H1)`을 구성(triple `(σ,m,S)`가 "matching" 조건 만족 시).
5. 핵심 식 `E(H1(σ,m,S,u,(τ1,...,τv))) = [τ1 E_S; ...; τu E_S; -τu+1 F_S; ...; -τv F_S]` (식4) — matching 조건이 성립하면 Theorem 2에 의해 girth≥6이 대수적으로 증명됨.
6. 확장(수정 구성): m이 소수일 때 Corollary 4의 `E(H2)`, m이 비소수일 때 Theorem 5로 확장 — coset leader 간 차이가 `Z*_m`에 속하면 girth≥6 보장.
7. 구현 디테일: column weight<3이면 sum-product가 잘 동작하지 않으므로, 조건이 맞으면 all-zero row vector를 추가해 column weight 3을 만드는 트릭 사용.
8. 별도 학습/최적화 절차 없음 (완전 대수적/폐쇄형 구성).
9. 결과: 예시(m=119, σ=38 또는 19)로 길이 n=2856, rate 5/6(=0.83), column weight 4인 3개 코드(eg-1,2,3) 구성 — MacKay 랜덤 코드와 BER/FER이 비슷하며 FER=10⁻⁴까지 뚜렷한 error floor 없음.
10. 한계: HW 미설계, 순수 시뮬레이션(단일 파라미터셋), "랜덤과 동등" 수준일 뿐 성능 개선 주장 없음.
