### B. 알고리즘 요약

1. 대상: 유한길이 binary LDPC(regular/irregular)의 waterfall 영역 BLER/BER를 BIOS 채널(BEC/BSC/BIAWGN)에서 예측하는 분석법.
2. 문제: DE/EXIT는 무한길이 가정이라 유한길이에 부정확; scaling law[8]는 정확하나 `α,β` 파라미터 피팅 필요(BEC 외 채널은 경험적 피팅).
3. 핵심 발상: 유한길이 codeword 전송 중 관측 채널품질이 기대값 주변에서 변동 → 단일 파라미터 random variable로 모델링, "관측채널이 threshold보다 나쁘면 복호 실패"(outage 원리).
4. 채널 파라미터: 관측 오류율 `P_obs`(식3, N샘플 오류수/N) 또는 관측 용량 `C_obs`(식2); 큰 N에서 CLT로 Gaussian 근사.
5. 분포: `P_obs ~ N(p0, p0(1-p0)/N)`, `C_obs ~ N(μ_C, Var[Z]/N)` → N 커질수록 분산 감소(μ는 N 무관).
6. BLER 추정: threshold `θ*`의 `p_th`/`c_th` 계산 후 `P_B = Q((p_th-μ_Pobs)/σ_Pobs)`(식5) 또는 `Q((μ_Cobs-c_th)/σ_Cobs)`(식4).
7. 복호 알고리즘 효과는 오직 decoding threshold를 통해서만 반영 → sum-product/min-sum/Gallager A 등에 공통 적용, 피팅 불필요.
8. BER 도출: EXIT chart가 닫히는 `p_stop`에서 `q_stop` 계산, Gaussian approximation `φ(x)`로 오류비트비율 `α` 산출(식7) → `Pb = α·PB`.
9. 결과: (3,6)-regular(BSC/BIAWGN), rate-1/2 irregular(N=10^4)에서 시뮬 대비 오차 <0.1 dB(N>2500); `P_obs`가 `C_obs`보다 정확(디코더는 용량보다 오류 수에 민감).
10. 한계: waterfall 전용, cycle 무시 → 짧은 코드·저BER(error-floor)에서 부정확; HW·정정성능 개선 없음, decoding threshold(DE) 사전 필요.
