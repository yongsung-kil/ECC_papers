### B. 알고리즘 요약

1. AWGN 채널, BPSK 변조, rate-1/2 length-500 PEG 생성 LDPC 코드에 대한 sequential(순차) BP 디코딩 문제를 다룸.
2. 기존 flooding 스케줄 대비 순차 업데이트가 수렴속도를 높인다는 선행연구를 바탕으로, 두 순차기법 RBP와 NWRBP를 결합해 둘의 장점만 취하는 것이 목표.
3. RBP는 residual `r(m) = m - m_old`가 가장 큰 메시지 하나만 매번 갱신(greedy)하여 초반 수렴이 빠르지만 반복이 늘면 flooding보다 성능이 나빠짐.
4. NWRBP는 residual이 가장 큰 메시지가 속한 check node의 모든 check-to-variable 메시지를 한꺼번에 갱신, 점근성능은 좋으나 초반 수렴이 RBP보다 느림.
5. 제안 기법은 1단계(RBP)로 시작해 특정 조건에서 2단계(NWRBP)로 전환하는 two-staged IDS. 전환은 미해결 check node의 이웃 변수노드 집합인 "suspicious node" 개수 `S_l`이 두 번 연속 `S_{l-1}/S_{l-2} ≥ S_l/S_{l-1}`(식 3)을 만족하면 발생(l≥4).
6. 전환 시 NWRBP는 residual을 0으로 리셋하지 않고 RBP에서 계산된 residual을 그대로 이어받아 시작, 추가 연산은 매 iteration마다 `S_l` 계산(선형 시간)뿐.
7. 특별한 양자화/근사 기법 언급 없음(순수 알고리즘/스케줄링 레벨).
8. 별도 학습/최적화 절차 없음(전환조건은 실험적으로 결정된 규칙식).
9. 결과: 전환은 평균 5.1 iteration(분산 1.1)에서 발생, 제안 기법은 iteration 7까지 RBP와 동등, 이후 RBP·NWRBP 모두를 능가하는 WER 성능(30 iteration 기준).
10. 한계: 단일 코드(500,1/2) AWGN 시뮬뿐, HW 구현/복잡도 분석 없음, RBP/NWRBP 자체가 완전 순차(메시지 1개씩 정렬 후 갱신)라 병렬 HW 친화적이지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RBP/NWRBP 순차 스케줄링, 전환조건 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 Dual-Update(even/odd 교대)는 부분병렬 스케줄이라 완전 순차 residual 기반 스케줄과 설계철학이 다름 |
| suspicious node 집합 S_l 계산 | 대응 없음 | 조기종료용 unsatisfied check 카운트와 유사하나 Partial/Full CRC와는 다른 메커니즘 |
| min-sum/tanh 근사 관련 | `decoder.cpp` `CNU_Update_New_Mag()` | 논문은 표준 tanh 기반 SPA 사용, min-sum 변형 아님 |
```

적용 가치: 낮음 — 완전 순차(residual sort 필요) 스케줄링은 Prime ECC의 병렬 HW 파이프라인(z=32, HCU) 구조와 근본적으로 충돌하며, 개선폭도 iteration 수 기준의 시뮬 결과뿐.
