### B. 알고리즘 요약

1. 목표: array LDPC code (2209, 2021), (4,47)-regular QC-LDPC를 AWGN 채널에서 시뮬레이션하여 error floor(BER<10^-11)를 빠르게 관측.
2. 문제: 소프트웨어(C/Matlab) 시뮬레이션은 BER 10^-7 수준 도달에도 10시간이 걸려 error floor 관측이 비현실적.
3. 채널 모델: AWGN 채널, 노이즈는 Box-Muller 방법으로 하드웨어 LUT 기반 생성.
4. 디코딩 알고리즘: modified Min-Sum, `Rcv = α × ∏sign(Lvc) × min|Lvc|` (식 1), scaling factor `α≈0.75`.
5. 변수노드 갱신: `Lvc = Σ Rmv + Iv` (식 2), 채널 고유 메시지와 이웃 check-to-variable 메시지 합산.
6. 인코더: QC-LDPC의 quasi-cyclic 생성행렬 G 구조를 이용해 shift-register 기반 serial 인코더 설계, 43×188비트만 저장.
7. 디코더 HW 구조: VPU 47개, CPU 188개를 4개 그룹으로 나눠 그룹 내 체인 통신(CPU-i ↔ CPU-(i±2))으로 엣지 패턴 복잡도 완화, [13] 논문의 shift decoder 아키텍처 재사용.
8. 최적화 절차 없음(DE/GA/NN 미사용) — 기존 코드/디코더 구조를 그대로 FPGA에 매핑.
9. 결과: throughput 120Mbps, 10시간 내 BER 10^-11 도달(소프트웨어 대비 약 1000배 빠름).
10. 한계: 새로운 디코딩/부호설계 기여 없음, 순수 시뮬레이션 가속 도구; NAND 채널/soft-read 모델 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Modified Min-Sum 디코딩 (식 1,2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 표준 min-sum과 동일 개념, 신규 기여 없음 |
| array LDPC QC-LDPC 인코더 (shift-register) | `encoder.cpp` `LDPC_encoder_4KB()` | 표준 QC 인코딩 구조, 신규성 낮음 |
| AWGN Box-Muller 노이즈 생성기 | `channel.cpp` | NAND read 채널(RBER/fixed-error)과 무관, 대응 약함 |
| VPU/CPU 체인 통신 파이프라인 구조 | `decoder.cpp` (HW pipeline 관련) | Prime ECC의 HCU/GT와 유사한 목적이나 array LDPC 전용 특화 구조 |

> 적용 가치: 낮음 — 알고리즘/부호설계 신규 기여가 없고 시뮬레이션 가속용 FPGA 플랫폼 구축에 초점이 맞춰져 있어 Prime ECC 이식 관점에서 참고할 요소가 거의 없음.
