> 총평: Min-Sum 알고리즘의 FPGA 구현·검증 사례이나 (8,4) Hamming code에 국한되고 float 기반 메시지 표현이라 NAND binary QC-LDPC로의 직접 이식 가치는 낮음. min-sum 자체는 Prime ECC에 이미 구현되어 있어 중복.

### B. 알고리즘 요약

1. 시스템: (8,4) extended Hamming code, `n=8, k=4, m=4` 패리티체크, 최소 rate `0.5`, Tanner graph는 check node degree `dc=4`, variable node degree 3(정보비트)/1(패리티비트), AWGN 채널 + BPSK.
2. 문제: 기존 complete BP(sum-product) 디코더의 check-node update가 `tanh`/`tanh^-1` 연산을 포함해 HW 구현이 어려움 → Min-Sum으로 근사하고 이를 FPGA로 가속.
3. 핵심 가정: 채널 LLR `Z_v = 2r_t/σ²` (AWGN + BPSK, equiprobable 입력 가정).
4. 핵심 기법 1단: Min-Sum 알고리즘 - Sum-step `m_{v,c}^(l) = Σ_{c'≠c} m_{c',v}^(l-1) + Z_v`(식 3), Min-step `m_{c,v}^(l) = (Π sign(m_{v',c})) · min|m_{v',c}|`(식 5)로 tanh 근사 대체.
5. 핵심 식의 의미: Min-step은 곱셈/tanh 없이 최솟값과 부호(XOR)만으로 check-node 메시지를 계산해 HW 면적/복잡도를 크게 낮춤.
6. 확장: Min-step을 floating-point 곱셈기 없이 구현 - abs value는 부호비트 마스킹(`x & 0x7fffffff`, 식 8), 출력 부호는 입력 부호들의 XOR(`sign_o = sign_a⊕sign_b⊕sign_c`, 식 9)로 계산.
7. 부수 트릭: 코드워드 심볼을 IEEE-754 32비트 단정밀도 float로 표현, FPGA 외부 SRAM(128비트 폭)에 4-element 벡터 단위로 저장해 메모리 대역폭 활용.
8. 최적화 절차: 없음(고정 알고리즘 구현), 반복횟수 `p=20`으로 고정(성능이 이 이상에서 포화).
9. 결과: FPGA(Xilinx Virtex-4 LX200, 100MHz)가 최적화 SW(-O3) 대비 15배 속도 향상(SW 652ms vs HW 44ms), FER 10^-4에서 uncoded BPSK 대비 1.6dB 코딩이득, HW/SW BER·FER 결과 일치로 설계 검증. LUT 사용률 35%, FlipFlop 33%, 곱셈기 0% 사용.
10. 한계: 매우 짧은 (8,4) block code(실전 NAND ECC급 QC-LDPC와 무관), IEEE-754 float 메시지 표현으로 저비용 양자화 논의 없음, gate count(공정 기준 환산치) 미보고, throughput(Gbps) 수치 미보고.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum check-node update (min1/min2 + sign XOR) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 핵심 기법과 완전히 동일한 개념(min-sum, sign XOR)이라 중복 |
| Min-step의 곱셈기 없는 abs/sign 비트연산 구현 | `decoder.cpp` `LDPC_Decoder()` | HW 면적 절감 트릭이나 Prime ECC는 이미 magnitude 양자화 테이블 기반이라 float 마스킹 기법 자체는 적용 불필요 |
| 완전병렬 V-node/C-node update (fine-grained parallelism) | 디코더 HW 모델(z=32 병렬) | Prime ECC가 이미 row 단위 병렬 구조(z_sb=32)를 보유해 개념적으로 이미 구현됨 |
| FPGA 검증/HW-SW BER 일치 방법론 | `TV_Dec_Inter/TV_Dec` | 검증 방법론 참고 가치는 있으나 알고리즘적 기여는 아님 |

> 적용 가치: 낮음 — Min-Sum 자체는 Prime ECC에 이미 구현된 핵심 기법이며, (8,4) Hamming code FPGA 구현 사례는 QC-LDPC 부호설계·HW 아키텍처 관점에서 새로운 이식 요소를 제공하지 않음.
