### B. 알고리즘 요약

1. 대상은 generic QC-LDPC 코드(예: (3456,1728), (3,6)-regular, rate 0.5, PEG로 구성)이며 표준 min-sum 기반 layered decoding을 사용한다.
2. 기존 고throughput LDPC 디코더는 shuffle network(인터커넥트)의 실리콘 면적·전력 소모가 크거나, array-LDPC 등 특정 구조에만 최적화되어 범용성이 떨어진다는 문제를 해결하려 한다.
3. 핵심 가정은 QC-LDPC의 파리티체크 행렬이 p×p circulant submatrix들의 배열이라는 구조이다.
4. 핵심 기법 1: row permutation으로 원본 행렬 `H`를 블록열마다 단일 shift 오프셋을 갖는 형태 `H_P`로 변환해, shuffle network를 1-level 2:1 MUX 기반 data shifter만으로 구현 가능하게 함.
5. Layered min-sum 갱신식은 `L^k_{c→v,t} = L^k_{v,(t-1)} - R^{k-1}_{c→v,t}`, `R^k_{c→v,t} = (Π sign) · min|L^k_{c→n,t}|`, `L^k_{v,t} = L^k_{c→v,t} + R^k_{c→v,t}`이며, layer간 데이터 의존성이 파이프라이닝을 막는다는 점이 문제의 핵심이다.
6. 핵심 기법 2: 근사 layered decoding — `R^{k-1}_{c→v,(t-1)} + R^k_{c→v,(t-1)}`의 증분을 다음 iteration에서 반영하는 근사식 `L^k_{c→v,t} = L^k_{v,(t-1-P)} - R^{k-1}_{c→v,t}`(P=파이프라인 단수, 예제에서 P=2)를 도입해 CNU에 파이프라인 스테이지를 삽입 가능.
7. 부수 트릭: CNU 연산을 compare-and-swap + 2단 pseudo rank order filter(PROF) + 스케일링 팩터(β=3/4) 보정으로 구성해 critical path를 3-adder+4-MUX 수준으로 최적화, R_cv 메시지는 compressed form(최소값·차소값·인덱스·부호)으로 저장.
8. 별도 학습/최적화 절차 없음(구조적 HW 설계).
9. 결과: 15 iteration 기준 250MHz(0.13μm CMOS 추정)에서 클록당 12 cycle/iteration으로 약 4.7Gb/s throughput 추정, 표준 layered decoding 대비 약 0.05dB 성능 손실(TPMP 대비는 0.2dB 손실보다 적음).
10. 한계: 실제 ASIC 제작·합성·실측 없이 추정치 기반이며([6] 대비 gate count 비교는 있으나 rough comparison), iteration 수·SNR 성능 개선(정정능력) 자체를 다루는 논문이 아니라 순수 HW throughput/복잡도 최적화에 집중.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Row permutation 기반 shuffle network 단순화 | `decoder.cpp` `LDPC_Decoder()`, HW arch(z=32 lifting/HCU 구조) | QC circulant shift 정렬을 통한 라우팅 단순화 아이디어는 Prime ECC의 병렬 row 처리 구조에 참고 가능(이식 난이도 "중") |
| Approximate layered decoding (파이프라인 근사) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, Dual-Update 스케줄 | layer간 의존성을 근사해 파이프라이닝하는 기법은 Prime ECC의 6-stage pipeline·Dual-Update(odd/even 교대) 스케줄과 비교 가능한 접근으로 이식 난이도 낮음 |
| CNU min-sum 연산 최적화 (compare-and-swap + PROF + scaling) | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum min1/min2 계산 구조로 Prime ECC의 CNU 연산과 직접 대응 |
| Data shifter (1-level MUX 기반 cyclic shift) | HW arch (z=32 병렬 row 단위) | 회로 최적화 기법으로 Prime ECC HW 모델과 정합 검토 필요(이식 난이도 "중") |

적용 가치: `높음` — min-sum 기반 layered decoding의 HW 파이프라이닝·라우팅 최적화로 디코더 알고리즘/스케줄링 레이어에 해당해 이식 난이도가 낮고, Prime ECC의 pipeline·HCU·Dual-Update 설계와 직접 비교할 가치가 있다.
