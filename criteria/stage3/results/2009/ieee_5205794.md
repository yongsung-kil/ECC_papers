### B. 알고리즘 요약

1. (3,6)-regular LDPC, 매우 짧은 길이(N=6, 7, 40) 코드를 대상으로 BIAWGN 채널에서 실험.
2. 기존 SP/MS 반복 디코더는 cycle이 있는 Tanner graph에서 성능 해석이 어렵고 stopping/trapping set, pseudocodeword 등 근사 모델만 존재 — 특정 코드에 대한 정확한 성능 상한 계산이 불가능한 문제를 해결하려 함.
3. Wiberg의 computation tree(SP/MS의 정확한 동작 모델) 개념을 채택, `deviation`(root=1이 되는 최소 valid assignment 집합) 단위로 성능 분석.
4. 핵심 기법: 각 variable node를 root로 하는 Extrinsic Tree(ET)를 반복적으로 구성 — check node 추가 시 최소 deviation weight가 증가하거나 동일 weight의 multiplicity가 감소하면 유지, 아니면 제거.
5. 노드가 트리에 중복 등장(copy)하면 LLR cost `λ_k`를 copy 수 `b_i`로 나눠 균등 분배(`λ_i/b_i`) — deviation weight `Σ(a_i/b_i)`를 codeword Hamming weight와 일치시키기 위한 스케일링.
6. ET 디코딩은 MS와 동일한 check/variable node 연산(min-sum)을 리프에서 루트까지 트리 위에서 1회만 수행(non-iterative), 최종 root 메시지 부호로 hard decision.
7. 부수 기법(코드 설계): full-rank parity-check matrix에 redundant row(선형결합)를 추가하면 MS 성능은 오히려 나빠질 수 있는 반면 ET 성능은 향상됨을 실험으로 보임(H_7x6 사례에서 min deviation weight가 codeword min weight와 일치, ML 근접).
8. 학습/최적화 절차는 없음(트리 구성은 deviation weight 기준 greedy 알고리즘).
9. 결과: (3,6)-regular N=40 코드에서 ET가 SNR 8dB 이상에서 MS(400 iteration) 대비 낮은 BER, 성능 상한이 시뮬레이션과 8dB 이상에서 근접.
10. 한계: 저자가 명시하듯 메모리 복잡도가 `N x A`(A≥N) 이상, 즉 O(N^2) 이상으로 스케일링 나쁨. HW 설계 없음, 매우 짧은 코드(N≤40)로만 검증되어 NAND급 실용 길이로 확장 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ET 디코딩(non-iterative tree decoder) | `decoder.cpp` `LDPC_Decoder()` | 반복 구조 자체가 다른(non-iterative tree) 접근으로 기존 min-sum 이터레이션 루프와 구조적 이질성 큼, 대체 어려움 |
| check/variable node min-sum 연산 재사용 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일한 min-sum 연산을 트리 위에서 재사용한다는 점만 공통, HW 관점 이득 없음 |
| redundant row 추가를 통한 부호 설계 개선 | `ecc_top.cpp` `Load_PCM()` | H-matrix 재설계·검증 부담이 크고 짧은 코드에서만 검증되어 고rate QC-LDPC 이식 근거 약함 |
| 트리 메모리 저장/파싱 | 대응 없음 | N^2급 메모리 요구는 NAND 컨트롤러 HW 제약과 정면 배치 |

> 적용 가치: 낮음 — 개념(deviation 기반 정확한 성능 상한)은 흥미롭지만 메모리/복잡도 문제로 실용 부호 길이 확장성이 없고, 저자 스스로도 향후 과제로 남긴 미해결 문제가 큼.
