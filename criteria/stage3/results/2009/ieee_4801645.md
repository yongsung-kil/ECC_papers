### B. 알고리즘 요약

1. WiMAX(IEEE 802.16e) QC-LDPC 디코더는 19가지 모드(코드 블록 576~2304)를 지원해야 하며, CNU-VNU 간 데이터 교환에 가변 크기 cyclic shift가 필요.
2. 기존 방식은 각 (IN, SN) 조합별 제어신호를 LUT에 저장하거나 직렬로 순차 계산해 서브매트릭스 크기가 클수록 비효율적.
3. Benes network `B(n)`(`N=2^n` 입출력)은 재귀적으로 두 개의 `N/2` 크기 서브 Benes network로 분해 가능하다는 구조적 성질을 이용.
4. 핵심 알고리즘(Algorithm 1)은 입력 포트 수 `IN`과 순환 시프트 값 `SN`의 최하위 비트 조합 4가지 경우에 따라 첫/마지막 스테이지 스위치 상태 `a_i`, `b_i`를 병렬로 결정하고, 하위 Benes network에 넘길 `(IN/2, SN/2)` 등 파라미터 쌍을 생성.
5. 이 재귀적 파라미터 전달로 한 스테이지의 모든 제어비트를 병렬 생성할 수 있어, 기존 방식(`T_DECODER+(K/2)·T_OR+...`) 대비 critical path가 짧아짐(`T_DEC+T_NOT+T_XOR+T_OR+T_MUX`).
6. 컨트롤러는 Parameter Generator(PG, n-1단 Div 모듈)와 Control Signal Generator(CSG, Signal Generator 모듈)로 구성되며 각각 재귀적 트리 구조로 구현.
7. WiMAX 적용 시 128x128 Benes network에서 신호가 지나지 않는 "unused switch"와 제어값이 항상 고정인 "reformed switch"를 시뮬레이션으로 식별해 제거, 4x4 서브네트워크 단위 추가 최적화로 128 포트를 96 포트, 11스테이지로 축소.
8. 별도 학습/최적화 절차 없음(결정론적 조합논리 설계 및 시뮬레이션 기반 스위치 제거).
9. SMIC 0.18um 합성 결과 전체 shuffle network 면적 0.160mm2(0.13um에서 0.080mm2), 기존 대비 면적 26.6%~71.1% 절감, 컨트롤러만 16000 gate.
10. 디코더의 정정성능(BER, iteration)에는 영향 없는 순수 인터커넥트 HW 설계이며, 특정 표준(WiMAX) 서브매트릭스 크기 집합에 최적화되어 일반화 시 재설계 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Benes network 기반 shuffle network 및 barrel shifter 컨트롤러 | 대응 없음 | Prime ECC 모듈 지도에 barrel shifter/permutation network 컨트롤러 항목이 없음 |
| CNU-VNU 간 데이터 라우팅(가변 cyclic shift) | 대응 없음 (참고: `decoder.cpp` 이터레이션 루프가 z=32 고정 병렬 구조 사용) | z=32 고정 lifting으로 가변 서브매트릭스 크기 대응 이슈가 Prime ECC에는 없어 직접 대응 모듈 부재 |

> 적용 가치: 낮음 — 디코더 정정성능과 무관한 순수 라우팅 HW 최적화이며, Prime ECC는 고정 z=32 구조라 가변 서브매트릭스 셔플 네트워크의 필요성 자체가 낮음.
