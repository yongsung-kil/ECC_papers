### B. 알고리즘 요약

1. 대상 시스템은 열차-지상(train-ground) 무선 통신망으로, probe vehicle이 수집한 실시간 속도·위치·상태 정보를 지상국으로 전송한다.
2. 고속 이동/다중경로 페이딩·Doppler shift로 인한 무선 채널 오류를 신뢰성 있게 극복하는 것이 문제.
3. 채널 모델은 AWGN이며 4QAM 변조를 가정.
4. 핵심 기법은 LDPC 기반 TMR(3중 모듈 중복) 구조: 3개 채널(A/B/C)이 각각 독립 LDPC 복호를 수행 후 TSC(totally self checking)/TMR voter로 다수결 결합.
5. 디코더는 기존 문헌(TDMP, turbo decoding message using BCJR)에서 가져온 architecture-aware LDPC 디코더 구조를 그대로 채택.
6. Check node 갱신에는 normalized min-sum 알고리즘을 사용, 레이어드(layered) 스케줄링 기반 message scheduler 채택.
7. 인코딩 체인은 BCH 외부코드 + LDPC 내부코드의 concatenated 구조(kbch→BCH→kldpc→LDPC→nldpc).
8. 별도의 학습/최적화 절차 없음(기존 알고리즘 재사용).
9. 결과: 단일/이중 채널 대비 3중 중복(ABC) 구성이 BER/FER 개선, 표준 대비 평균 50% 적은 iteration으로 수렴, 중~고 SNR에서 한 자릿수(order of magnitude) coding gain 개선을 보고.
10. 한계: LDPC 자체의 신규 부호설계나 디코더 알고리즘 개선이 아닌 기존 기법의 시스템 레벨 응용이며, HW 게이트카운트·throughput 등 정량적 HW 검증 수치는 본문에 없음(시뮬레이션 수준).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP 기반 디코더 아키텍처 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 이터레이션 디코더 구조이나 논문은 기존 문헌 인용뿐, 신규 기여 없음 |
| normalized min-sum CN 갱신 | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC가 이미 min-sum 보유, 논문은 표준 기법 재사용이라 중복 |
| TMR/TSC voter (3중 중복) | 대응 없음 | Prime ECC는 단일 디코더 시뮬레이터로 시스템 레벨 중복 구조 미해당 |
| BCH-LDPC concatenated 코딩 체인 | 대응 없음 | Prime ECC는 단일 QC-LDPC ECC 엔진, 외부 BCH 코드 미보유 |
```

적용 가치: `낮음` — LDPC를 철도 무선 시스템의 구성요소로 응용한 시스템 논문이며, 부호설계·디코더 알고리즘 자체의 신규 기여가 없고 이미 Prime ECC가 보유한 표준 기법(min-sum, 이터레이션 디코딩)만 재사용.
