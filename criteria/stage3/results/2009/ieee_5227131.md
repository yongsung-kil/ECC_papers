### B. 알고리즘 요약

1. 대상은 IEEE 802.11n/802.16e 표준의 QC-LDPC(structured LDPC) 코드로, `Z×Z` circulant sub-matrix로 구성된 H를 사용하며 `mb×nb` block 구조(`nb=24`, `mb∈{4,6,8,12}`).
2. 기존 연구들은 처리 병렬도/면적 트레이드오프에만 집중하고 메모리 서브시스템(디코더 에너지의 대부분을 차지) 설계 탐색은 누락되어 있다는 문제의식.
3. 디코딩 알고리즘은 Mansour의 TDMP(turbo-decoding message-passing)를 채택: check node를 순차 평가하며 `γ`(posterior)와 `λ`(extrinsic) 두 메모리만 사용.
4. 행 처리: `ρ = γ(Ii) − λi`로 prior 계산 → `Λ = SISO(ρ)`(Min-Sum) → `λi←Λ`, `γ(Ii)←ρ+Λ`로 갱신. 2-phase 대비 iteration 최대 50% 감소.
5. Min-Sum 특성상 각 행마다 `ci`개 메시지 대신 부호(sign) `ci`개 + magnitude 2개(1st/2nd min) + index만 저장하면 되어 8bit 메시지 기준 메모리 최대 65% 절감.
6. 메모리 설계를 micro-organization(shuffling unit으로 P개 처리유닛에 샘플 분배)과 macro-organization(block-column을 뱅크에 배치)으로 이원화.
7. macro-organization은 block-column 간 동시접근 충돌을 conflict graph로 모델링하고 그래프 컬러링(NP-complete, offline 해결)으로 최소 뱅크 수(`χ(G)`)를 산출해 뱅크 배치.
8. Synopsys Primetime + in-house 툴로 65nm CMOS, Vdd=1.32V에서 λ/γ 메모리 파티션별 면적·에너지 탐색(예: γ-memory `[192x48]x2`, λ-memory `[96x24]x4`).
9. 결과: memory area 0.211mm², use case R=1/2,N=648 기준 memory energy 275.71nJ로 기존 대비([9] 590.81nJ) 개선, 에너지의 85% 이상이 메모리에서 소비됨을 확인.
10. 한계: ASIC 합성 결과이나 무선 표준(11n/16e) 전용 TDMP 스케줄에 특화되어 있고, 디코딩 알고리즘 자체(Min-Sum) 개선은 없음; NAND용 부호/스케줄과 직접 호환 불확실.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP 메시지 메모리(γ/λ) 뱅크 분할·conflict graph 배치 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | HW 메모리 뱅킹 아이디어는 이터레이션 루프 메모리 접근 패턴과 유사하나 스케줄(TDMP vs Prime의 min-sum/HCU)이 달라 직접 대응 어려움 |
| Min-Sum sign+2magnitude 압축 저장 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 Prime ECC도 min1/min2+sign 방식 사용 중(§3), 중복 기여 |
| H-matrix(shift+column) ROM 저장 | `ecc_top.cpp` `Load_PCM()` | 구조 유사(QC-LDPC PCM 저장)하나 802.11n/16e 고정 행렬로 이식 불가 |
| 802.11n/16e 표준 코드 자체 | 대응 없음 | Prime ECC는 자체 H-matrix 사용, 표준 코드와 무관 |

적용 가치: 낮음 — 메모리 뱅킹 최적화는 흥미로우나 무선 표준 전용 TDMP 아키텍처이며 Prime ECC의 z=32/HCU 구조와 스케줄이 상이해 직접 이식보다는 참고 수준.
