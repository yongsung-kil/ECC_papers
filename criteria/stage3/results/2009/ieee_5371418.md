### B. 알고리즘 요약

1. 대상: narrow-sense `(N,K)` RS code over `GF(2^m)`, binary image expansion `Hb`((n-k)×n), BPSK over AWGN, soft-decision 반복복호.
2. 문제: RS는 HDPC(고밀도 parity-check)라 짧은 cycle 다수로 BP 직접 적용이 부적합, 기존 대수적 HDD는 soft 정보 미활용으로 성능 손실.
3. 배경(ABP): 매 iteration마다 `|L(ci)|` 오름차순 정렬 후 신뢰도 낮은 열부터 Gaussian elimination을 수행해 저신뢰 비트가 Tanner graph에 1회만 연결되게 adapt한 뒤 BP 실행(식 8~11, damping `α`).
4. ABP 한계: 그래프 adaptation(특히 GE)이 계산량이 커 HW 구현 불가.
5. 배경(Chase): dmin 기반 채널측정 soft 정보로 `p=⌈dmin/2⌉` 최저신뢰 위치의 test pattern을 만들어 최소 analog weight ML 선택(Chase-II).
6. 핵심 기법(ABP-Chase): 각 ABP iteration에 간소화 Chase 디코더를 cascade해 조기종료 기준으로 사용 — Chase가 valid codeword를 찾으면 즉시 종료, ABP 앞뒤로도 Chase 실행.
7. 간소화: `p`를 `(N-K)` 비트로 축소(원래 `(N-K)m`보다 훨씬 작음), ML 대신 parity-check 만족 "valid" codeword 출력(ML과 ~99% 일치)으로 고속화.
8. 결과: RS(31,25)는 max 5 iter로 ABP-BM(5 iter) 대비 FER `10^-5`에서 0.3dB, RS(63,55) 0.1~0.2dB, RS(255,239) 0.2dB 개선, HDD 대비 ≥2dB.
9. 평균 iteration: TABLE I~III에서 고SNR일수록 ABP-Chase가 ABP-BM보다 적은 평균 iteration으로 수렴(예 RS(31,25) 4dB에서 1.110→0.448).
10. 한계: HW 미설계·시뮬만, ABP의 GE 복잡도 자체는 미해결, 비이진 RS 전용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ABP 반복복호(GE+BP) | decoder.cpp LDPC_Decoder() | 반복복호 루프는 대응되나 매 iteration GE·비이진 RS라 Prime ECC min-sum과 구조 상이 |
| BP tanh CN 연산(식 8) | decoder.cpp CNU_Update_New_Mag() | Prime ECC는 tanh 대신 min-sum 근사라 full-tanh BP는 프로파일상 "대응 없음" 예시 |
| Chase 조기종료 기준 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 조기종료 개념은 유사하나 Prime ECC는 CRC 기반이고 Chase(대수 HDD)와 무관 |
| RS/GF(2^m) 부호구조·GE adaptation | 대응 없음 | Prime ECC는 고정 binary QC-LDPC로 비이진 RS·parity-check adaptation 개념 없음 |
```

적용 가치: 낮음 — 비이진 RS soft-decision ABP+Chase로, 고정 binary QC-LDPC·min-sum을 쓰는 Prime ECC(NAND)에 떼어 쓸 요소가 없다. 유일 시사점은 "조기종료 기준 개선이 평균 iteration을 줄인다"는 정성적 방향뿐.
