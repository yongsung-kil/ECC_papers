### B. 알고리즘 요약

1. 시스템: 통신용 채널 디코더 ASIP, 두 부호(Viterbi 대상 convolutional code rate 1/2 (171,133) 및 tail-biting LDPC convolutional code rate `R=0.4`, block length `NB=3200`)를 하나의 IP코어로 처리.
2. 문제: SDR(software-defined radio) 환경에서 여러 표준(CC/TC/LDPC)을 각각 별도 데이터패스로 구현하면 재사용성이 낮음(선행연구 [3][4]는 CC/TC와 LDPC 경로를 분리).
3. 핵심 가정: Viterbi(BMC/SMC/TB)와 Offset Min-Sum LDPC(CN/VN update)의 연산이 add/sub/compare 중심으로 유사하다는 점에 착안해 하나의 데이터패스로 통합.
4. 핵심 기법 1단: Offset Min-Sum CN 업데이트 `L_mn = Π sign(L_n'm) · max(min|L_n'm| - β, 0)`(β는 정규화 오프셋), VN 업데이트 `L_nm = y_n + Σ L_r'm`.
5. 핵심 식의 의미: CN/VN 연산과 Viterbi ACS(`Λk(i) = min(Λj(i-1)+λjk(i))`)가 공통적으로 compare/add/sub 연산을 요구하므로, 이를 처리하는 2단계 FU(stage1: Min Search/Accumulate, stage2: Compare&Select/Intrinsics Subtraction)를 공유 설계.
6. 핵심 기법 2단: Traceback(TB)만은 연산량 대비 메모리 접근이 많아 별도 FU(TBU)로 분리, 나머지(ALU)는 CC/LDPC 겸용 SIMD 배열(64-fold parallelism)로 구성.
7. 부수 트릭: LLR buffer size 16(최대 16 CN/VN 값), 5bit 양자화, Tanner graph 연결은 barrel shifter(shuffle unit)로 구현, Viterbi trellis 연결은 8/16/32/64-state 간 전환 가능한 programmable interconnect.
8. 최적화 절차: 별도 학습/DE 없음. STA(Synchronous Transfer Architecture) 저전력 설계 패러다임 기반 VLIW 프로그래밍 구조로 구현.
9. 결과: 130nm 공정 200MHz에서 Viterbi 30Mbit/s(SNR 4.25dB, BER=1e-4), LDPC(tail-biting SC-LDPC, 10 iteration) 53Mbit/s, 면적 745kGates·전력 130mW, 로직 재사용률 91%(LDPC 전용 대비 VA 추가 비용 18%).
10. 한계: LDPC 처리량이 선행연구([4] FlexiChaP) 대비 다소 낮고, LDPC는 tail-biting convolutional 구조 전용이라 QC-LDPC block code로의 일반화는 미검증. 정정능력 개선이 아닌 하드웨어 재사용/유연성이 목적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset Min-Sum CN/VN 업데이트 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 offset min-sum과 개념적으로 동일(중복 기여, 개선 없음) |
| Viterbi(CC)-LDPC 겸용 연산유닛 통합(ASIP) | 대응 없음 | Prime ECC는 LDPC 전용 bit-exact 시뮬레이터로 CC/Viterbi 겸용 아키텍처 대상 아님 |
| Tanner graph 연결용 barrel shifter(shuffle) | `GT.cpp` `Make_GT_HW()` | 목적(엣지 라우팅 단순화)은 유사하나 실존 대응 함수는 QC-LDPC 특화 MUX 구조 |

> 적용 가치: 낮음 — CC(Viterbi)와 tail-biting SC-LDPC를 하나의 ASIP으로 통합하는 재사용성 위주 논문으로, Prime ECC의 정정성능/처리량 개선과는 목적이 다르고 부호구조도 QC-LDPC block code와 상이함.
