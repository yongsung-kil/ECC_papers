### B. 알고리즘 요약

1. 대상은 memoryless binary erasure channel(BEC)에서 좋은 성능을 갖도록 설계된 LDPC parity-check matrix(정규/불규칙, IRA, GeIRA, PEG, 유한기하 등 임의 구조)이며, 여기에 single burst erasure 정정 능력을 추가하는 것이 목표.
2. 기존 2-step 방식(burst correction form 행렬 + quadratic complexity 정정 후 반복복호)은 긴 코드에서 계산비용이 크므로, 반복복호(linear complexity)만으로 독립오류·burst erasure를 동시에 정정하는 single-step 코드 설계가 필요.
3. 핵심 개념: stopping set — 유도 부분그래프의 모든 check node가 최소 2개 연결을 갖는 variable node 집합. `Lmax`(maximum guaranteed resolvable burst length)는 반복복호로 항상 정정 가능한 burst 길이의 최댓값이며 `Lmax = μ(G)-1`(stopping set의 최소 span).
4. 핵심 기법: pivot 개념 — stopping set `S`에서 값을 알면 반복복호로 `S` 전체를 복구할 수 있는 variable node `V`. Lemma 2/3로 pivot의 필요·충분조건(차수-2 check node 경유 인접 pivot)을 정의하고 효율적 pivot 탐색 알고리즘 제시.
5. 핵심 정리(Theorem 2): `Lmax+1` 길이의 정정불가 burst에 포함된 maximal stopping set은 그 burst의 첫/마지막 variable node를 pivot으로 가짐 → 이 pivot들의 span을 늘리면 정정 가능해짐.
6. 확장(PSS: Pivot Searching and Swapping 알고리즘): 정정 불가능한 burst마다 임의로 선택한 pivot을 다른 variable node와 column permutation으로 교체해 pivot span을 `Lmax+1` 초과로 확장, 이를 반복해 `Lmax`를 점진 증가시킴. degree distribution과 BEC 성능은 불변.
7. 구현 디테일: column permutation만 수행(sparsity·연결구조 불변) → BEC 성능 완전 보존. IRA/GeIRA류는 permutation으로 double-diagonal 인코딩 구조가 깨지므로 systematic 부분만 permute하는 대안도 제시(단, 달성 `Lmax`는 낮아짐).
8. 최적화 절차: greedy 반복 탐색 — 실패 횟수가 `Fmax`에 도달할 때까지 pivot swap 시도, 성공 시 burst 길이 목표를 1씩 증가.
9. 결과: (2640,1320) Margulis 코드는 `Lmax`가 이론추정치 `p*n`(1135)을 초과(1133→1135 이상)하는 근사-최적 코드 획득; (4608,4033) rate-0.8752 코드는 기존 QC 구성(Lmax=375) 대비 PSS 적용 후 425로 개선. 알고리즘 복잡도는 `n`에 대해 quadratic(반복복호가 `O(n)`이고 이를 `O(n)`회 수행).
10. 한계: BEC(erasure) 채널 전용이며 hard-decision bit-flip이나 soft LLR 기반 AWGN/NAND 채널에는 적용되지 않음; HW 구현·복호기 설계 없음(순수 오프라인 코드 최적화 알고리즘), 실측/FPGA 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PSS 알고리즘(stopping set pivot 기반 column permutation) | `ecc_top.cpp` `Load_PCM()` | H-matrix 열 순서 최적화이나, Prime ECC는 이미 고정 QC-LDPC 구조(z=32 circulant)를 사용해 임의 column permutation 적용이 구조적으로 제약됨 |
| stopping set / erasure 반복복호 | 대응 없음 | Prime ECC 채널 모델(`channel.cpp`)은 AWGN/RBER/fixed-error 패턴이며 BEC(erasure) 모델을 다루지 않음 |
| burst error 정정 능력 지표(Lmax) | 대응 없음 | NAND read 채널은 일반적으로 random bit error 모델이며 본 논문의 burst erasure 시나리오(연속 소실)와 직접 대응되지 않음 |

적용 가치: `낮음` — BEC(erasure) 채널 전용 column-permutation 최적화 기법으로, Prime ECC의 QC-LDPC 고정 구조·binary error(비erasure) 채널 모델과 정합되지 않아 이식 가치가 낮음.
