### B. 알고리즘 요약

1. 대상은 일반 binary (ir)regular LDPC 코드(rate/length 특정 없음), 채널·부호 스펙은 미제시.
2. 문제의식: LDPC는 무작위 구성이라 전통적 통신이론 분석이 어렵고, 통계역학(Ising spin/spin-glass) 유사성을 이용하면 code ensemble 성질을 exact하게 분석 가능.
3. 핵심 모델: 코드를 spin system으로 보고 `Sj=(-1)^sj`, Hamiltonian `H(S|J) = -Σ Jμ·Πsj - (F/β)Σsj`로 파싱크 제약(coupling `Jμ`)과 채널 노이즈(external field `F`, flip rate `f`로 결정)를 표현.
4. 핵심 기법: TAP(Thouless-Anderson-Palmer) 근사가 BP 알고리즘과 등가임(Kabashima et al.)을 이용해 LLR-BP를 유도 - `xμj`,`yμj`는 각각 variable→check, check→variable LLR.
5. 핵심 식 `f(t) = -ln tanh(|t/2|)`이 자기역함수(`f(f(t))=t`)라는 성질을 이용해 check node 갱신을 `|2βyμj| = f(f(|2βJμ|) + Σf(|2βxμl|))`로 정리(식 19) - 곱 형태 연산을 덧셈 형태로 근사 가능케 함.
6. 확장(BP-Based): `f(t)`가 `t`가 최소일 때 최대값을 갖는 근사적 성질을 이용해 식(19)를 `min(|2βJμ|,|2βxμl|)`로 근사(식 21) → check node 갱신이 최소값 선택으로 단순화(식 22, 표준 min-sum과 동형).
7. 부수 트릭: sign/magnitude 분리 처리(`sign(βyμj)=Π sign(Jμxμl)`), LLR 표현으로 곱셈을 덧셈으로 대체(구현 이점 언급뿐, 실제 구현 없음).
8. 학습/최적화 절차 없음 - 순수 해석적 유도.
9. 결과: 수치적 결과 전혀 없음(BER/iteration/gate count 등 일체 미제시), 결론에서 "differences via simulation"을 향후 과제로 제시.
10. 한계: 이론 유도에 그치고 시뮬레이션·HW·성능 비교가 전무하며, 유도된 BP-Based 알고리즘이 기존 min-sum과 본질적으로 동일해 신규성이 낮음.
