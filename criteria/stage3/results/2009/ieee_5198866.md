> 총평: min-sum(UMP-BP) 정규화 스케일링 팩터를 exhaustive search 없이 이론적으로(consistency condition) 구하는 방법론으로, 알고리즘 설계 단계에서 LLR 파라미터 튜닝 비용을 줄일 수 있어 이식성은 중간 수준.

### B. 알고리즘 요약

1. 대상은 turbo code와 LDPC code 두 iterative decoder 예시, LDPC는 regular rate 1/2 Gallager 코드 (504,252)/(1008,504), 체크노드 degree ρ=6, AWGN+BPSK 채널.
2. 하드웨어 효율을 위해 BP(sum-product)의 비선형 log-sum-exp를 max-log 근사(`Lout = max_i Lin,i`, 즉 UMP-BP/min-sum)로 단순화하면 L-value가 "consistency condition"(`log(p(λ|c=1)/p(λ|c=0)) = λ`)을 깨뜨려 성능 저하 발생.
3. 핵심 가정: 손상된 L-value의 조건부 pdf `p_L(λ|c)`를 히스토그램으로 추정할 수 있고, 이로부터 정합 매핑 함수를 구성해 참 L-value로 복원 가능.
4. 핵심 기법: bijective 매핑 `f(λ) = log(p_L(λ|c=1)/p_L(λ|c=0))`(Theorem 1, Eq.6)를 적용하면 결과 L-value가 다시 consistency condition을 만족함을 증명.
5. 이 매핑 f(λ)를 실제 히스토그램에 적용해보면 거의 선형(`f(λ) ≈ α·λ`)임이 관찰되어, 기존에 경험적으로만 쓰이던 "linear scaling factor α"의 이론적 근거를 제공.
6. 확장: 이 방법은 iteration마다 반복 적용 가능하나, 저자들은 첫 iteration에서 구한 α가 이후 iteration에도 거의 동일한 BER 성능을 보임을 시뮬로 확인 - 1회 계산으로 충분.
7. 구현 절차 3단계: (1) L-unit 출력 L-value 히스토그램 수집 → (2) f(λ) 계산 → (3) 디코더에 스케일링 factor α로 반영.
8. 최적화 절차는 별도 학습 없이 히스토그램 기반 해석적 계산(exhaustive grid search 대체)이 핵심.
9. 결과: (504,252)/(1008,504) LDPC 코드에서 제안 방법(UMP-BP-CO)이 기존 Chen&Fossorier 방법(UMP-BP-CF)보다 BER 우수, optimum(exhaustive search) 값에 근접. 구체 dB 수치는 그래프에만 있어 본문 서술 없음(미상).
10. 한계: 소규모 regular LDPC 코드로만 검증(QC-LDPC 아님), HW 구현/합성 없음, iteration 수는 최대 1000회의 순수 시뮬레이션 결과.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum(UMP-BP) L-value 스케일링/보정 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC는 이미 iteration별 LLR threshold 테이블을 보유 - 본 논문의 해석적 도출법을 테이블 설계/튜닝 절차에 참고 가능 |
| CN min-sum 연산 자체 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 스케일링 팩터가 적용되는 대상 |
| histogram 기반 pdf 추정 절차 | 대응 없음 | Prime ECC 코드에 튜닝용 오프라인 도구로는 없음(설계 프로세스 참고용) |
| 채널/soft-read LLR 양자화 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | consistency condition 개념이 LLR 양자화 테이블 최적화에 이론적 근거 제공 가능 |

> 적용 가치: 중간 — 알고리즘 자체는 표준 min-sum 정규화 팩터 튜닝 이론이라 신기법은 아니지만, Prime ECC가 이미 보유한 iteration별 LLR 테이블(PARAM_LLR)의 설계/최적화 근거로 참고할 여지가 있음. 단, QC-LDPC 대규모 코드·NAND 채널 미검증이라 직접 이식보다는 설계 프로세스 참고 수준.
