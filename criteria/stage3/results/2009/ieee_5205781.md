### B. 알고리즘 요약

1. 대상: regular binary (N,K) (dv,de) LDPC, MacKay (504,252) dv=3, de=6 코드, AWGN+BPSK 채널.
2. 문제: fully-parallel BP는 HW 복잡, 기존 group horizontal shuffled BP(GHSBP)는 균등 분할로 check node가 그룹당 1회만 속해 message 재사용 효율이 낮음.
3. 새 그룹핑: `Ym(Pnew) = W(m)`으로 정의된 그룹(check node가 여러 그룹에 중복 소속 가능)을 사용, W(m)은 check m과 2-hop 이내로 연결된 check 집합.
4. Average Check Number(ACN) `Ec(P) = (1/G)Σ Nc·dc/|Ngi|`로 그룹핑 품질을 정량화, 새 그룹핑이 기존/랜덤 대비 ACN이 더 큼(표I,II).
5. 디코딩 스케줄: 직전 처리 그룹과 연결 변수노드 교집합이 최대인 그룹을 다음 순서로 선택 `O(j)=argmax|U(O(j-1))∩U(l)|`.
6. 확장: min-sum의 정규화 계수를 `α = 1 - β·Zmin/Zsum`로 자기조정(M-NMSA), Zmin/Zsum은 check node에 모인 |Zn'→m|의 최소/합.
7. 근거: 두 세트 X,Y가 같은 최소값을 가질 때 합이 작은 쪽이 실제 φ 함수값이 더 작다는 부등식(13)을 이용해 min만 쓰는 MSA/NMSA보다 정보량을 더 반영.
8. 별도 학습 절차 없음 (β=1.1, α=0.8 등은 경험적으로 설정).
9. 결과: 동일 복잡도에서 표준 BP 대비 FER=10^-4에서 약 0.23dB, GHSBP(Nc=12) 대비 FER=10^-5에서 약 0.1dB 이득. New-M-NMSA는 New-NMSA 대비 FER~5×10^-5에서 약 0.12dB 이득.
10. 한계: 소형 regular 코드(504,252) 시뮬만 수행, HW 구현/게이트카운트 없음, girth>4 가정에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check node 그룹핑/스케줄링(GHSBP 개선) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 QC-LDPC z=32 row 병렬 구조라 check-serial 그룹 스케줄과 정합 낮음 |
| M-NMSA 자기조정 정규화 min-sum | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산에 Zsum 기반 보정계수 추가 이식 가능, 계산 낮음 |
| LLR 초기화/soft 메시지 | `channel.cpp` `Set_LLR_Th()` | 채널 모델은 AWGN 소프트값, NAND soft-read 양자화와 직접 대응 아님 |

> 적용 가치: 중간 — M-NMSA의 Zsum 기반 자기조정 보정계수는 기존 min-sum CN 연산에 낮은 난이도로 이식 가능하나, 그룹핑/스케줄 부분은 Prime ECC의 완전병렬 QC-LDPC HW 구조와 정합이 낮음.
