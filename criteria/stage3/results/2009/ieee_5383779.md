### B. 알고리즘 요약

1. 대상은 short(~1000bit)/medium(~2000bit) block length의 irregular QC-LDPC 부호이며, rate 0.5~0.73 범위에서 array 계열 LDPC(H matrix) 설계를 다룸.
2. 문제: 기존 random LDPC는 성능은 좋으나 H-matrix가 복잡하고, structured(array) LDPC는 encoder/decoder 구현이 간단하지만 성능 개선 여지가 있음.
3. 기존 기법인 Fan의 array code `H(p,j,k)`(순환치환행렬 α 기반), Eleftheriou/Olcer의 MAC(상삼각 구조), Singhaudom의 IMAC(quasi-cyclic shift ω 추가)를 배경으로 함.
4. 핵심 기법: IMAC의 좌측 절반을 그대로 사용하고, 우측 절반은 좌측을 수평 flip한 뒤 quasi-cyclic shift(`ω`)를 추가로 적용해 만든 새로운 H-matrix (식 (11)).
5. 최종 H는 `jp × kp` 크기의 상삼각 구조이며 코드율은 `R = 1 − (j/k)` (p는 소수, j ≤ k ≤ p)로 IMAC과 동일 공식이지만 1의 개수(밀도)를 줄임.
6. 부가 트릭: 우측 절반 생성 시 "cyclic shift", "quasi-cyclic shift", "flip+cyclic", "flip+quasi-cyclic" 4가지 옵션 중 flip+quasi-cyclic(옵션 3의 확장)이 가장 성능이 좋다고 별도 논문[13]에서 확인 후 채택.
7. 최적화 절차는 별도 없음 — 선행 연구[13] 결과를 바탕으로 옵션을 고정 선택하는 방식.
8. 인코딩은 H를 LU 분해(`H=LU`) 후 forward/backward substitution으로 parity 계산, 디코딩은 log-domain sum-product(belief propagation).
9. 결과: rate 0.563 제안 부호가 기존 regular LDPC(rate 0.5)보다 SNR<6.5dB에서 우수, 기존 MAC/IMAC(rate 0.563) 대비 SNR<9dB에서 우수(BER=10⁻⁴ 근방 waterfall 영역, Fig.2~3).
10. 한계: long block length에서는 기존 IMAC 성능을 따라잡지 못함, HW 미설계·시뮬레이션뿐, 반복횟수(5/10/20)에 따른 iteration-성능 트레이드오프만 정성적으로 언급.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix 설계(quasi-cyclic array 기반 irregular QC-LDPC) | ecc_top.cpp Load_PCM() | 저rate(0.5~0.73) short/medium block 구성법으로 Prime ECC의 고rate(~0.9) QC-LDPC와 rate대가 크게 달라 직접 재사용 어려움 |
| 인코딩(LU 분해 기반) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC는 dual-diagonal 고정 인코딩을 사용하므로 LU 분해 방식과 구조가 다름 |
| log-domain SPA 디코딩 | decoder.cpp LDPC_Decoder() | 표준 SPA 언급뿐 Min-Sum 변형이나 HW 최적화 요소 없음 |
```

적용 가치: 낮음 — 저rate(0.5~0.73) short/medium block irregular QC-LDPC H-matrix 구성법으로, Prime ECC의 고rate(~0.9) 고정 구조와 rate·용도가 상이하며 부호설계 이식은 원래 난이도가 높은 영역(§4)인 데다 실험 규모도 작아 실질적 이식 가치가 낮음.
