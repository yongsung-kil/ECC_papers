### B. 알고리즘 요약

1. 시스템: WiMedia 1.5 UWB 표준(멀티밴드 OFDM), structured QC-LDPC(`P=30` circulant), 8개 코드(N=1200/1320, rate 1/2~8/11), 264MHz 클록, air throughput 최대 1.41Gbps, latency 요구치 0.9375µs 이하.
2. 문제: 기존 layered 디코더는 클록당 submatrix 1개(=P edge)만 처리하나, WiMedia의 소블록(1200/1320bit)·높은 throughput·낮은 latency 요구를 맞추려면 클록당 다중 submatrix 처리가 필수(코드 설계로 `P`를 늘리기엔 소블록에서 성능 저하).
3. 채널 모델: IEEE 802.15.3a UWB 채널(CM1~CM4), QPSK/DCM/16QAM 변조, SystemC 기반 시뮬레이션 체인.
4. 핵심 기법: normalized Min-Sum(`α=0.875`, 최소 두 magnitude 사용)과 TDMP(turbo decoding message passing, 즉 layered/shuffled decoding)를 결합, `λ_{m→n} = α × sgn(...) × min(|λ_{n'→m}|)`로 CN 메시지 계산.
5. 식의 의미: layered decoding은 같은 iteration 내 중간결과를 즉시 반영해 수렴 속도를 최대 50% 향상시켜 iteration 수를 줄이고 그만큼 throughput/latency를 개선.
6. 핵심 확장(HW): 3-slot 아키텍처 — 각 슬롯이 submatrix 1개(30 edge)를 처리, 3슬롯 동시 동작으로 클록당 90 edge(=3 submatrix) 처리. Variable node 연산은 별도 FU 없이 30개 Check Node Block(CNB) 내부에서 처리.
7. 구현 디테일: APP RAM을 3분할해 슬롯별로 열 인덱스를 3으로 나눈 나머지/몫으로 매핑, 코드가 stall-free하도록(동일 APP 주소 재접근 시 이전 쓰기 완료 후에만 접근) 설계됨. 입력 quantization 5bit, APP quantization 7bit.
8. 별도 학습/최적화 절차 없음(고정 표준 코드 기반 HW 아키텍처 설계).
9. 결과: 65nm 저전력 공정 합성, 전체 면적 0.51mm² (메모리 76.5%, 로직 23.5%), N=1200/rate=1/2 코드에서 5 iteration·219 clock cycle로 payload throughput 1.44Gbps, latency 0.83µs; 동일 rate 3/4 convolutional 코드 대비 coding gain 4~6dB.
10. 한계: 정정능력 자체를 개선하는 논문이 아니라 순수 HW 아키텍처/합성 결과이며, WiMedia 표준의 고정 구조화 코드(P=30, N=1200/1320)에 특화되어 타 코드로의 일반화는 별도 검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized Min-Sum + layered(TDMP) decoding | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 Min-Sum·iterative message passing과 동일 계열 기법으로 신규성 낮음(중복 기여) |
| 3-slot 병렬(클록당 다중 submatrix 처리) 아키텍처 | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC의 z=32 row 단위 병렬과 다른 축(slot 기반 submatrix 병렬)의 처리량 확장 아이디어로, HCU/GT와의 정합 여부는 재검토 필요 |
| stall-free APP RAM 주소/슬롯 매핑 설계 | `decoder.cpp` HCU 관련 로직, `ecc_data.h` PCM 구조체 | 메모리 뱅크 충돌 회피 스케줄링 아이디어는 HW arch 레이어에 참고 가능하나 코드가 다름(WiMedia 전용 P=30) |
| 인코딩(구조화 코드 기반) | `encoder.cpp` `LDPC_encoder_4KB()` | 본 논문은 인코더 구조 미설계(디코더 전용), 직접 대응 없음 |

> 적용 가치: 중간 — 알고리즘(Min-Sum/layered) 자체는 이미 보유해 중복이나, 소블록·저latency 조건에서 클록당 다중 submatrix 처리하는 HW 병렬화 아이디어(slot 분할, stall-free 스케줄링)는 throughput 개선 참고 가치 있음.
