### B. 알고리즘 요약

1. 채널: BPSK+AWGN, regular (γ,ρ) LDPC 부호. hard-decision `zn`과 soft 채널값 `yn` 모두 사용하는 WBF 계열 디코더 대상.
2. 문제: WBF는 저복잡도이나 BP 대비 성능 손실이 크고, 기존 WBF 변형들은 경험적으로 설계되어 BP와의 이론적 연계가 불명확.
3. 핵심 관찰: min-sum/BP 디코딩의 1차 반복 후 LLR `L(cn|y)`이 flipping metric `En`과 `L(cn|y) ∝ (1-2zn)En` 관계를 가짐 (식 11).
4. min-sum 기반: `wn,m = min_{i∈N(m)\n} |yi|` (식 3)으로 `En = Σ(2sm-1)wn,m - |yn|` (식 4) 계산, `α` 곱하면 기존 IMWBF와 동일해짐(식 5, NMS 대응).
5. BP 기반(신규): `wn,m = Φ⁻¹[Σ_j Φ(|Lj|) - Φ(|Ln|)]` (식 9)를 사용한 NBP-WBF/OBP-WBF 제안, `Φ(x)=log((e^x+1)/(e^x-1))`.
6. 확장: OMS 유사 offset 버전(식 6, `max{wn,m-β,0}`)도 min-sum/BP 양쪽에 대칭적으로 유도 가능.
7. 병렬화: 기존 제안한 PWBF(parallel WBF) 프레임워크에 새 flipping function을 적용, 반복별 threshold 벡터 `Fth[k]`로 동시에 여러 비트 flip.
8. Threshold 최적화: 두 가지 시뮬레이션 기반 절차(Procedure 1: 단순, Procedure 2: 반복적 개선) 제안, 최적 threshold는 iteration마다 감소하는 패턴을 이용.
9. 결과: (273,191) FG-LDPC에서 NBP-WBF가 IMWBF보다 우수, (8192,1024) rate-7/8 QC-LDPC에서 LP-WBF/SZJ-LP-WBF가 IMWBF를 고SNR에서 능가.
10. 한계: 복잡도상 NBP-WBF는 N번의 실수 곱셈 + lookup table 필요(IMWBF는 비교연산만)해 실용성 낮음; 순수 시뮬레이션 검증, HW 설계 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 기반 WBF flipping function (식 3~5) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min-sum CN 연산과 유사 구조이나 이미 min-sum을 보유하여 중복 |
| 병렬 threshold 반복 스케줄 (Procedure 1/2) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | iteration별 threshold 개념이 `PARAM_LLR` 테이블 구조와 유사하나 WBF 자체가 hard-decision 계열이라 대응은 약함 |
| BP 기반 Φ/Φ⁻¹ lookup table (NBP-WBF) | `decoder.cpp` `VN_Cal_HD()` 등 | soft LLR lookup 기반 처리로 대응 가능하나 Prime ECC는 min-sum 기반이라 Φ 함수 미보유, 대응 없음에 가까움 |

> 적용 가치: 낮음 — Prime ECC는 이미 min-sum 계열 디코더를 보유하고 있어 본 논문의 핵심 기여(IMWBF=NMS와 등가성 증명, BP-WBF 변형)는 대부분 중복 기여이며, WBF는 hard-decision 계열로 Prime ECC의 soft-decision min-sum 구조와 이식 정합성이 낮음.
