### B. 알고리즘 요약

1. 목적: perpendicular magnetic recording 채널에서 LDPC 부호의 매우 낮은 BER(~10^-12~10^-15) 영역 error floor를 실측 특성화.
2. 문제: 소프트웨어 Monte Carlo 시뮬레이션이나 기존 단일 FPGA 에뮬레이터로는 이런 저 BER 통계 확보에 수 주~수개월 소요.
3. 채널 모델: 9-coefficient FIR step response 기반 perpendicular magnetic recording, AWGN + transition jitter noise 포함.
4. 디코더: SOVA(soft-output Viterbi, 8-state, 7-bit precision)로 소프트 정보 생성 후 `min-sum` LLR 기반 LDPC 디코더로 25 local + 2 global iteration 수행. 기존 알고리즘 그대로 사용, 개선 없음.
5. 가속 기법 1단: 채널 등화기(11-tap FIR)와 recording channel 임계경로를 5-stage 파이프라인으로 분할해 클록 20MHz→100MHz로 단일 코어 처리량 5배 향상.
6. 가속 기법 2단: leapfrog 알고리즘으로 긴 의사난수 시퀀스를 독립 서브시퀀스로 분할, BEE2 보드 3대·FPGA 15개에 걸쳐 LDPC 코어 27개를 완전 독립 병렬 실행.
7. HW: Virtex-II Pro FPGA, PowerPC 405 임베디드 코어로 제어소프트웨어 자체를 온칩 이동(PCI 버스 병목 제거).
8. 최적화/학습 절차 없음(순수 아키텍처/파이프라이닝 기법).
9. 결과: 이전 최고 FPGA 에뮬레이터 대비 수백~1000배 속도향상 달성, rate 8/9 length 4923 QC-LDPC 부호의 BER를 최초로 ~10^-14 근방까지 특성화.
10. 한계: 특정 자기기록 채널·특정 부호 1종에 대한 실험뿐이며, 디코더 알고리즘이나 부호 자체의 개선은 없음(순수 시뮬레이션 가속 인프라).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-sum LDPC 디코더 (기존 알고리즘 사용) | `decoder.cpp` `LDPC_Decoder()` | 이미 보유한 기법과 동일, 신규 기여 없음 |
| FPGA 파이프라이닝/멀티코어 에뮬레이션 가속 | 대응 없음 | Prime ECC는 bit-exact C++ 시뮬레이터로 검증 목적 상이(HW 클록 레벨 모사이지 error floor 측정 가속기가 아님) |
| SOVA 소프트 출력 채널 모델 (자기기록 특화) | 대응 없음 | AWGN/RBER/fixed-error 채널과 무관한 자기기록 PR 채널 전용 |

> 적용 가치: 낮음 — NAND ECC 부호설계/디코더 알고리즘 개선이 아닌 자기기록 채널 전용 error floor 측정용 FPGA 에뮬레이션 가속 인프라 논문이라 이식 요소가 없음.
