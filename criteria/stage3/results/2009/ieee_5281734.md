### B. 알고리즘 요약

1. Binary-input AWGN 채널에서 regular LDPC (d_v,d_c) 코드를 대상으로 하며, 메시지 알파벳 A={+1,-1}인 순수 binary message-passing 디코더를 제안.
2. 기존 hard-decision(Gallager A/B, majority-based) 알고리즘은 배선/전력에 유리하나 BP 대비 수 dB 성능 열세인 문제를 해결하고자 함.
3. Successive Relaxation(SR) 아이디어를 차용해, 각 반복에서 이전 상태 대비 "차등"으로만 갱신하는 메모리 기반 구조를 도입.
4. 엣지마다 q-bit 메모리 `B_vi,cj`를 두고 수신값 `Q(y_i)`로 초기화, 매 반복 check-to-variable 이진 메시지 합 `Σm_ck→vi`을 더해 차등 갱신: `B(ℓ) = B(ℓ-1) + Σ m_ck→vi`.
5. 이 메모리의 부호(sign)가 다음 반복의 variable-to-check 이진 메시지가 되어, 소프트 정보를 이진 통신으로 근사(quantization step Δ, clip threshold c_th).
6. 변형(MDD-BMP)으로 엣지 대신 variable node 하나당 메모리 1개만 두는 저메모리 버전도 제시 (성능은 저하되나 FG 코드에서는 손실 작음).
7. 메모리 업데이트를 `sgn(Σm)`으로 단순화한 경량 변형(식 6)도 제시, count up/down 연산으로 구현 가능하나 평균 iteration이 최대 3배 증가.
8. 최적화 절차 없음(별도 학습/DE 없이 Monte Carlo 시뮬로 q, Δ* 튜닝).
9. 결과: random (4000,2000)(4,8) 코드에서 BP 대비 1.4dB, FG 코드(1023,781)/(4095,3367)에서 BP 대비 0.5~0.75dB 이내 성능, MB 대비 최대 3dB 개선.
10. 한계: HW(RTL/합성) 설계 없이 순수 알고리즘·Monte Carlo 시뮬 결과뿐이며, 평균 iteration 수가 BP·MB보다 대체로 많고 저차수 irregular 코드에서 error flare 발생.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 이진 메시지 전달 디코딩 전체 구조 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 구조는 대응되나 min-sum 대신 완전히 다른 이진 차등 갱신 방식이라 대체 개념 |
| 엣지별 q-bit soft 메모리 및 양자화 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | LLR 테이블 대신 엣지 메모리를 두는 방식이라 구조적으로 다름 |
| check-to-variable 이진(XOR) 메시지 | `decoder.cpp` `C2V_Cal_New_Sgn()` | sign만 전달하는 점은 유사하나 min-sum magnitude 연산 없음 |
| 하드결정/조기종료(H*z=0) | `partial_CRC.cpp`, `full_CRC.cpp` | 대응 없음 (본 논문은 parity check 직접 검사, CRC 아님) |

> 적용 가치: **낮음** — Min-Sum(soft magnitude 기반) 대비 완전히 다른 이진 메시지 패러다임이라 기존 decoder.cpp 구조에 이식하려면 아키텍처 재설계가 필요하고, 성능도 Min-Sum류 대비 우위가 불명확하며 HW 검증도 없음.
