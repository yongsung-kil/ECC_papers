### B. 알고리즘 요약

1. 시스템: UWB(WiMedia) 통신용 예시 코드워드, QC-LDPC irregular, `n=1200`, `m=600`, `z=30`(rate 1/2, 640Mb/s), layered decoding 구조.
2. 문제: 기존 조기종료(inherent parity check 전량 일치 확인, 또는 [3]의 pre-check)는 waterfall 영역에서 최적 대비 성능이 떨어짐.
3. 핵심 관찰: 디코딩 실패 코드워드의 비트오류 분포가 불균일 — variable node degree(order)가 낮을수록, 특히 parity node(주로 order 2)일수록 오류 확률이 높음. 5 iteration 후 분포는 크게 치우치고 20 iteration 후엔 균형화.
4. 핵심 기법: `Partial Parity Check` — 코드워드별로 가장 신뢰도 높은(즉 order가 상대적으로 높은) parity node 부분집합(예시에서 앞쪽 30개 parity bit, order 3)을 선정해 checksum처럼 사용, 디코딩된 data node(DN)로 재계산한 parity와 비교해 일치 시 조기종료.
5. 핵심 식/의미: 별도 수식 없이 `Cmax`, `Dmax`, `Pmax`(check/data/parity node 최대 degree)로 노드 신뢰도를 정성적으로 판별 — order 낮은 노드는 correction이 덜 누적되어 오류 잔존 확률이 큼.
6. 비교 대상: inherent termination, unimplementable genie(전체 코드워드 정답 가정), unimplementable partial genie(DN만 정답 가정), VNR 방법([7]) 등과 iteration count/오류율 곡선 비교 — partial parity 방식이 partial genie(이상적 하한)에 근접.
7. 구현 디테일: 선택된 parity 그룹의 재계산은 Check Node Processor(CNP) 출력에서 z-bit 단위 XOR로 free byproduct처럼 얻어짐, 최저비용 구현 시 1 iteration 시간 소요로 회로 면적 영향 미미.
8. 별도 학습/최적화 절차 없음(코드워드별 order 분포 관찰 기반 정적 설계).
9. 결과: waterfall 영역에서 inherent termination 대비 iteration count 상당폭 감소, partial genie(이상적 하한)과 사실상 동일한 곡선.
10. 한계: 단일 예시 코드워드 시뮬 결과이며 최악 latency는 개선 안됨(최대 iteration 상한으로만 제어 가능), 여러 조기종료 기법 결합 실제 구현 상세는 향후 과제로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Partial parity 기반 조기종료(checksum 없이 parity node 신뢰도로 판별) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 유사한 "부분 확인 후 조기종료" 위치이나, 본 논문은 CRC가 아닌 parity node 신뢰도 기반이라 구현 방식은 다름(비교 대조 가치) |
| CNP 출력에서 z-bit 단위 parity XOR 재계산(byproduct) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 내 병렬 CN 연산 구조와 동일 맥락, parity 재계산 삽입 지점 참고 가능 |
| VN/PN degree 기반 오류 신뢰도 분석 | `decoder.cpp` `VN_Cal_HD()` 등 | VN 처리 단계에서 degree별 신뢰도 가중 아이디어 참고 가능(직접 대응 모듈은 없음) |

> 적용 가치: 중간 — 조기종료(latency/iteration 절감) 아이디어는 Prime ECC의 Partial CRC와 목적이 겹치나 메커니즘이 달라 대체재라기보다 보완적 비교 검토 대상.
