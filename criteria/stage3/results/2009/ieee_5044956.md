### B. 알고리즘 요약

1. 대상은 protograph 기반 정규 (ms,J,K) LDPC convolutional code(SC-LDPC) 앙상블이며, AWGN 채널에서 near-codeword/trapping set이 error-floor를 지배한다는 배경 하에 분석한다.
2. 문제의식: LDPC convolutional code가 block code와 동등한 capacity-approaching 성능을 내지만, error-floor 예측에 필요한 trapping set 성장률이 알려지지 않았다.
3. 채널/모델 가정: near-codeword(대부분의 parity check를 만족하는 시퀀스)와 Tanner graph 상의 trapping set 존재를 error-floor의 원인으로 가정(Richardson 방법론 확장).
4. 핵심 기법: `(a,b)` general trapping set(변수노드 a개가 만드는 서브그래프에서 홀수차수 check node b개)을 정의하고, 각 protograph check node에 auxiliary flag 변수노드를 추가한 modified protograph에 대해 two-part weight enumerator `A_{a,b}`(식 3)를 계산.
5. 핵심 식: 점근적 trapping set spectral shape function `r(α,β)=limsup (1/n)ln A_{a,b}`(식 4)의 첫 zero-crossing이 `Δ`-trapping set growth rate `δts(Δ)`를 정의하며, 이는 error-floor 발생 지점을 예측하는 지표다.
6. 확장 기법: unterminated convolutional code 자체는 직접 계산이 어려워, unwrapping factor `Λ`를 갖는 tail-biting convolutional code 앙상블(유한 block code로 취급 가능)의 growth rate `δ_ts^(Λ)(Δ)`로 하한을 구성(Theorem 1, 식 6,8).
7. 부수 기법: `gcd(nc,nv)=1`인 경우를 위해 protograph parity-check matrix를 nonuniform cutting vector `ξ`로 분할하여 convolutional code로 변환(Section III-A).
8. 결과 도출 절차: (3,6)-regular(rate 1/2), (3,4)/(3,5)/(3,9)-regular 등 여러 앙상블에 대해 zero-contour curve(`δts(Δ)` vs `Δ`)를 계산하는 수치 예제 제시(Table 1, Fig. 5~8).
9. 결과: convolutional code의 `δccts(Δ)` 하한이 대응하는 block code 앙상블의 `δ_ts^(1)(Δ)`보다 항상 크게 나와(예: (3,6) rate 1/2, Δ=0에서 block 0.023 대비 convolutional 하한이 더 큼), SC-LDPC가 block code보다 좋은 error-floor 성능을 가질 것으로 예측.
10. 한계: 순수 앙상블 평균 이론 분석(block length→∞ 점근), 실제 codeword 시뮬레이션/BER 곡선 없음, degree-1/2 check node로 구성된 elementary trapping set 분석은 별도 연구로 남김, HW 구현 전무.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping set / error-floor 성장률 이론 분석 | 대응 없음 | 점근적 앙상블 이론 분석 도구로, Prime ECC의 bit-exact 시뮬레이터 구조와 대응하는 모듈 없음 |
| protograph 기반 SC-LDPC convolutional code 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 설계 관점에서만 간접 연관, Prime ECC는 QC-LDPC(block) 고정 구조로 이식 난이도 "높음"(§4) |
| tail-biting termination | 대응 없음 | Prime ECC는 convolutional/tail-biting 구조 미지원 |

> 적용 가치: 낮음 — SC-LDPC 앙상블의 점근적 trapping set(error-floor) 성장률을 계산하는 순수 이론 분석으로, 실제 디코더/HW 구현이나 binary QC-LDPC 고정 구조인 Prime ECC 3.1에 이식할 구체적 알고리즘·구조가 없음.
