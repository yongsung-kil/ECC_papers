### B. 알고리즘 요약

1. 시스템: CMMB(중국 모바일 방송) LDPC 디코더, codeword `N=9216`, 두 rate(1/2 (3,6) regular, 3/4 (3,12) regular) 지원.
2. 문제: rate별 H-matrix가 달라 rate마다 별도 디코더는 낭비 → 하나의 재구성형 디코더로 multi-rate 지원 필요, 큰 sparse H-matrix 저장 메모리도 부담.
3. 모델: 채널 LLR 입력, soft-decision message passing (belief propagation).
4. 기법1(디코딩): Modified UMP-BP — CN에서 `Lmn=(∏sign(Zn'm))·min|Zn'm|·α` (min-sum + scaling α), BN에서 `Znm=LLR(Pn)+ΣLm'n`.
5. 의미: UMP-BP는 LLR-BP 대비 ~1dB 열세이나 곱셈→덧셈/min으로 복잡도 대폭 감소, α scaling으로 BER를 LLR-BP에 근접.
6. 기법2(AGU): CMMB 코드는 HS 구조(non-QC)로 18/9행마다 36bit cyclic-shift → `H'` 서브행렬(18x6/9x12)만 저장하고 나머지 주소는 AGU가 `Hm,n=3×(H'+36×qu(m/18)) mod N + I` 로 생성.
7. 기법3(Index matrix): bit node별 등장 index를 18x6x2bit I-matrix로만 저장해 주소계산 register overhead 제거 → H저장 0.39%, I-matrix도 0.39%로 축소.
8. HW: partial-parallel 16 BNU + 16 CNU + 16 shared memory(충돌 없는 병렬 read/write), rate별 reconfigurable BNU/CNU(rate-3/4는 12값을 2 cycle 접근).
9. 결과: 0.18μm 합성 455K NAND2, 221,184bit 메모리, 185MHz; throughput rate-1/2 14.32Mbps·rate-3/4 26.97Mbps(20 iter), 기존 대비 H저장 메모리 0.39%.
10. 한계: 방송용 non-QC 특정 부호, throughput 수십 Mbps(NAND급 아님), 정정성능 개선 없음, 합성 수준(실측 없음).
