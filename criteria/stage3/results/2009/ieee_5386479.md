### B. 알고리즘 요약

1. 시스템: 생체 secure biometric(fuzzy vault)을 Slepian-Wolf coding으로 구현, feature 벡터를 irregular LDPC로 syndrome 부호화(rate `R=0.2~0.35`, `N=150` VN).
2. 문제: 기존 [7] 방식은 feature bit별 신뢰도 차이를 ECC가 무시 → FRR 높음(11%), 보안(NBS)-robustness(FRR) tradeoff 개선 필요.
3. 모델: 같은 사용자 두 측정치의 bit `a_{i,j}` vs `b_j`는 crossover `p_{i,j}`의 BSC로 연결, 신뢰도 `R=log((1-p)/p)`.
4. 기법1(bit 재배치): enrollment 다중 샘플로 `p_{i,j}` 추정 → `p` 증가순(신뢰도 감소순)으로 feature bit 재정렬 후 최고신뢰 bit을 최고차수 VN에 배치.
5. 의미: 고차수 VN이 BP에서 더 많은 check 정보를 받아 정확히 복호되므로, 신뢰 bit을 여기 두면 신뢰 정보가 iteration 초반에 빠르게 전파.
6. 기법2(soft init): VN별 초기 LLR `L_{i,j}=±R_{i,j}` (probe bit 부호), 저장부담 완화 위해 사용자 평균 `R̄_j` 1세트(N개)만 device에 저장.
7. 구현: `R̄_j`는 bit이 0/1일 가능성 정보를 누설하지 않음(부호는 복호시 probe로 결정) → 보안 유지.
8. 기법3(shuffled BP): VN/CN을 병렬 아닌 serial 갱신 → 같은 성능에 iteration 감소 또는 같은 iteration에 성능 향상.
9. 결과: 지문 DB(1035명×15샘플, 400 cuboid 중 robust 150개) 실험서 FRR 11%→3.3%(shuffled BP), FAR<수e-6, SAR<0.06%; rate 0.2→0.35시 NBS 30→53, FRR 3.3%→7%.
10. 한계: 생체 채널·Slepian-Wolf 특화, HW 미설계, 짧은 부호/저rate, error-floor 미논의.
