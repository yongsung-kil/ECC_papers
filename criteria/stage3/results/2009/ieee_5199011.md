### B. 알고리즘 요약

1. 대상: block length 1008, rate 0.5의 irregular LDPC 3종(PEG-only-DE15, PEG-ACSE-DE15, PEG-only-DE10), AWGN 채널, message-passing 디코더(최대 50 iteration).
2. 문제: 고SNR에서 BLER이 극도로 낮아(10^-10 이하) Monte Carlo(MC) 시뮬레이션으로는 평가에 비현실적인 시간이 소요됨.
3. 배경 모델: 에러 발생은 주로 trapping set(TS)의 induced connected subgraph(ICS)에 기인하며, `[w;u;e]` 라벨(w=TS 크기, u=홀수연결 체크노드 수, e=cycle indicator `e = Σ(di-2) - u`)로 분류.
4. 핵심 기법 1단: "detrimental primary TS-ICS(PTS-ICS)"를 3단계로 탐색 — (1) ACE(approximate cycle extrinsic message degree) 기반 short cycle 탐색, (2) 발견된 단일 cycle에 error impulse(진폭 `ε`)를 주입해 다중 cycle PTS-ICS를 유도, (3) 디코더 실패 시 hard-decision 시퀀스에서 최소 hamming weight 벡터로 PTS-ICS 추출.
5. 핵심 식: importance sampling 가중치 `ω(x*) = f(x*)/f*(x*)`, mean translation으로 편향분포 중심 `μ`를 오차영역 경계로 이동시켜 희귀 오류사건 발생빈도를 인위적으로 높임.
6. 핵심 기법 2단: 같은 `[w;u;e]` 라벨의 PTS-ICS를 그룹화한 뒤, 대표 원소만 IS로 1차 추정(rough) → 오차확률이 큰 그룹만 전체 원소를 IS로 재추정(fine)해 전체 BLER 합산.
7. 부수 디테일: `μ=1`(u=0인 경우), `u>0`인 경우 이분탐색(bisection, `μ∈(1, 3.2)`)으로 최적 편향중심 탐색.
8. 최적화 절차 없음(구성적 탐색+통계적 추정이며 학습 기반 아님).
9. 결과: SNR=5.8dB에서 MC 대비 최대 3.92×10^9배 시뮬레이션 속도 향상(NMC 3.51×10^15 vs NIS 9.07×10^5), BLER 10^-14까지 추정 가능.
10. 한계: 순수 시뮬레이션/통계 방법론이며 부호설계나 디코더 알고리즘 자체의 개선이 아님; HW 미설계; regular 코드에서 제안된 원기법을 irregular로 확장한 이론적 평가 도구.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping-set 기반 BLER 평가(IS) | 대응 없음 | Prime ECC는 bit-exact HW 시뮬레이터로 MC 기반 검증만 수행하며 IS 가속 평가 기법 없음 |
| PTS-ICS 탐색/분류 | 대응 없음 | 부호구조 분석 도구이며 코드베이스 모듈과 직접 대응 없음 |
