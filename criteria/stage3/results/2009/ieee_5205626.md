### B. 알고리즘 요약

1. 대상 채널은 co-channel interference로 인한 impulsive noise를 modeling한 ε-mixture pdf(Contaminated Gaussian/Laplacian/Cauchy) `f_n(x) = (1-ε)/√(2πσ1²) e^(-x²/2σ1²) + εH`.
2. 기존 Gaussian LLR(GLLR)을 그대로 쓰면 이런 노이즈에서 성능 저하가 크다는 문제를 다룸.
3. 노이즈 종류별로 최적 LLR(CGLLR, CLLLR, CCLLR)을 유도하고, density evolution으로 regular (3,6) LDPC code의 threshold를 비교.
4. 실제로는 노이즈 pdf의 4개 파라미터(H, σ1, σ_nb, ε) 추정이 어려우므로, σ1만 안다고 가정하고 파라미터 불필요한 robust LLR을 연구.
5. 기존 Parametric-Cauchy LLR(PCLLR, 식 11)과 Generalised Likelihood Ratio(GLR, 식 12)를 검토, GLR은 σ1 재도입한 Parametric GLLR(PGLLR, 식 13)로 개선.
6. 논문이 새로 제안한 PGLLR-2(식 14)는 σ1을 분모/분자에 더하는 방식으로 PGLLR 대비 0.1~0.3dB 개선.
7. 디코더(BP) 자체는 수정하지 않고 입력 LLR 계산식만 교체하는 구조 - 알고리즘/HW 변경 없음.
8. 최적화 절차 없음(닫힌 형태 LLR 유도 + threshold 비교만 수행).
9. 결과: robust LLR들이 최적 LLR 대비 0.5~1dB 이내 threshold, PGLLR-2가 PGLLR보다 근소 우세.
10. 한계: BER 시뮬 대신 density evolution 위주(계산량 회피 목적), LDPC convolutional code 하나의 앙상블만 검증, HW/구현 이슈 미다룸.
