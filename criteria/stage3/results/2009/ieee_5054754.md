### B. 알고리즘 요약

1. 시스템은 BIAWGN 채널, (6,3)-regular LDPC 코드(예제: N=40, K=20)를 대상으로 한다.
2. 기존 SP/MS 반복복호는 Tanner graph에 사이클이 있으면 이론적 성능 분석(bound)이 어렵다는 문제를 해결하려 한다.
3. Wiberg의 computation tree 모델을 확장해, 각 variable node마다 별도의 "extrinsic tree"를 사전에 구성한다.
4. Extrinsic tree 구성 규칙: 어떤 check node와 그 자식 variable node들은, 그 variable node가 트리의 이전 레벨에 이미 등장한 적이 없을 때만 추가한다 (deviation의 최소 weight를 최대화하려는 휴리스틱).
5. 트리에 variable node의 복제본이 여러 개 생기면 LLR 비용 `λi`를 복제본 수만큼 균등 분배(`ai/bi` 스케일링)하여, deviation 확률이 codeword의 Hamming weight로 귀결되도록 한다.
6. 복호는 트리의 leaf에서 root까지 한 방향으로만 min-sum 연산(`mf = min|mv|·sign`, `mv = λ + Σmf`)을 수행하는 비반복(non-iterative) 방식이다.
7. 최종 root node 메시지 `mvroot`의 부호로 root variable의 비트값을 hard decision.
8. 트리 구성은 코드마다 1회만 수행하면 되고, decoding 연산은 트리 깊이만큼의 클록으로 병렬화 가능하다고 언급(HW 설계는 없음).
9. (40,20) (6,3)-regular 코드에서 ET decoding은 2회 반복 MS보다 high SNR에서 우수했고, deviation enumeration으로 계산한 상한이 8dB 이상에서 시뮬레이션과 근접했다.
10. 한계: 코드 길이가 커지면 extrinsic tree 크기와 deviation enumeration이 지수적으로 커져 실용 코드에 적용하기 어렵고, HW 구현/합성 검증이 전혀 없다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Extrinsic tree 기반 non-iterative min-sum 복호 | 대응 없음 | Prime ECC는 iterative message-passing(`decoder.cpp` `LDPC_Decoder()`) 구조이며, 트리 재구성 기반 비반복 복호 방식은 구조적으로 대응되지 않음 |
| Deviation enumeration 기반 성능 상한 | 대응 없음 | 이론적 성능 분석 도구로, HW 시뮬레이터인 Prime ECC의 어떤 모듈과도 직접 대응되지 않음 |
| CN min-sum 메시지 계산 (`mf`) | `decoder.cpp` `CNU_Update_New_Mag()` | 연산식(min1/sign) 자체는 Prime ECC의 min-sum과 유사하나 스케줄링 구조가 다름 |

적용 가치: `낮음` — 짧은 regular 코드의 이론적 트리 복호/상한 분석이며, NAND용 고rate QC-LDPC HW 파이프라인 구조와 접점이 없다.
