### B. 알고리즘 요약

1. 대상은 저rate(1/3, 1/4) irregular LDPC 부호이며, 파리티체크 행렬 `H = [H1 Htr]`에서 `Htr`(파리티부)은 간단한 인코딩을 위해 echelon form(주대각선+부대각선)으로 고정.
2. 문제: 저rate 부호는 최적 degree distribution상 degree-2 컬럼 비율이 낮아(예: 1/3-rate에서 약 0.5) `Htr`에 degree>2 컬럼이 다수 포함되며, 기존 PEG 등 greedy 설계는 girth만 고려해 국소 그래프 구조(짧은 cycle 분포)를 놓침.
3. 핵심 가정: 성능 저하는 girth(최소 cycle) 단일값보다 cycle '분포'(다수의 짧은 cycle)에 더 민감하다는 가정 하에 cycle distribution을 설계 지표로 채택.
4. 핵심 기법: echelon form에 새로 추가되는 entry가 만드는 cycle을 Class-1(신규 entry+대각선만), Class-2(신규 entry+기존 entry 1개+대각선), Class-3(신규 entry 2개+대각선, 최대 기존 entry 2개까지)로 분류, 각 클래스의 cycle 길이를 `Δr`, `Δc`(행/열 인덱스 차) 기반 폐형 정리(Theorem 1~9)로 계산.
5. 측정함수 `M(Ec) = Σλ1 + Σλ2 + Σλ3`, 각 `λi = D^(-L/2)` (지수형, `D`는 설계 파라미터=10) - 짧은 cycle에 더 큰 페널티를 부여하는 지표.
6. 컬럼별 entry 배치를 trellis의 노드/경로로 표현, 각 컬럼에서 가능한 배치 조합을 상태로 하는 trellis search(dynamic programming, PEG류의 2단계 완화 중 1단계만 적용)로 전역 최적 경로 선택.
7. structured(QC-type) 코드로 확장: characteristic matrix `CH`(shift 값 포함)에서 cycle 길이를 `L_actual = LB/gcd(S,B)`로 환산(`S`=alternating sum), block size `B`가 커질수록 trellis 길이가 `B`배 축소되어 계산량 감소.
8. 학습/최적화 절차: density evolution으로 미리 얻은 degree distribution(`λ_{1/3}(x)`, `ρ_{1/3}(x)` 등)을 고정한 뒤 trellis search로 실제 edge 배치만 최적화(구조 최적화, 확률분포 최적화 아님).
9. 결과: AWGN 채널·100회 반복 기준 제안 부호가 PEG 대비 낮은 BER(정성적 개선, 구체 dB 수치는 그림에만 존재), 설계 계산시간은 구조화(B=32) 시 non-structured 대비 수백~수천 배 단축(Table I: (2048,512) 코드 86080s→87s).
10. 한계: 순수 시뮬레이션(AWGN, BPSK)뿐이며 HW 미설계, 저rate(1/3~1/4)·짧은~중간 길이(512~2048) 부호에 국한, error-floor 개선은 별도 measure 도입 가능성만 언급되고 본 논문에서 검증되지 않음.
