### B. 알고리즘 요약

1. 대상 부호는 IEEE 802.3an 표준의 (2048, 1723) rate 0.8413 regular (6,32) LDPC 코드, AWGN 채널.
2. Error floor는 BER 10^-12 수준에서 발생해 일반 시뮬레이션으로는 관측 불가능한 영역이라 해석적 예측이 필요.
3. 핵심 가정은 dominant trapping set이 absorption set(다수 이웃이 짝수 번 연결된 variable node 집합)이며, message-passing이 여기서 fixed point에 갇힌다는 것.
4. (a,b) absorption set의 동역학을 선형대수로 모델링: 출력 메시지 벡터 `x`, 입력 `y`에 대해 `y=Cx`(check node permutation), `x_{i+1}=VCx_i+λ+extrinsic` 반복식.
5. 스펙트럴 정리로 `(VC)^i λ → μ_max^i (λ^T v_max) v_max` 수렴하며, (8,8) set의 최대 고유값 `μ_max = d_v - 2 = 4`가 "gain"으로 error floor 기울기를 결정.
6. 이 gain과 extrinsic 평균 LLR(density evolution으로 계산)을 이용해 absorption set 오류확률 `P_AS = Q(f(a,b))` 폐형 식 (식 4) 도출.
7. 부수 보정: check node tanh 합을 Taylor 근사로 "check node gain"으로 치환, extrinsic 신호 극성 반전 확률(polarity reversal)까지 보정항으로 추가.
8. 학습/최적화는 없고 대신 (a,b) absorption set 존재/개수를 조합적 탐색(Theorem 1)과 위상 축소(topology reduction)로 도출 — (8,8) set 14,272개 확인.
9. 결과: 해석적 error floor 곡선이 FPGA 하드웨어 시뮬레이션 및 importance sampling과 Eb/N0=5~7.5dB 구간에서 잘 일치, BER 10^-14 이하까지 예측 가능.
10. 한계: 특정 코드의 특정 absorption set 토폴로지에 국한된 해석이며 새로운 디코더/부호 설계 기법이 아니라 사후 분석 도구이고, HW 설계는 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| absorption/trapping set 동역학 분석, error floor 예측식 | 대응 없음 | Prime ECC는 시뮬레이터이며 이런 해석적 error floor 예측 모듈이 없음; 참고용 분석 방법론일 뿐 |
| 특정 (8,8) absorption set 토폴로지 탐색 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 분석 시 유사 토폴로지 탐색이 필요하면 참고 가능하나 코드 내 기능 대응 없음 |
| min-sum/check node 근사(tanh 대체) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min-sum 근사 보유, 논문의 tanh 근사는 이론적 보정항이라 직접 이식 불필요 |

> 적용 가치: **낮음** — 새로운 디코딩/부호설계 기법이 아니라 기존 코드의 error floor를 해석적으로 예측하는 이론적 분석 도구이며 Prime ECC에 이식할 알고리즘적 요소가 없음.
