### B. 알고리즘 요약 (10줄 내외)

1. 대상: BEC(이진소거채널) 위에서 반복(BP/peeling) 복호되는 일반 LDPC 코드. 성능은 Tanner graph의 stopping set 구조로 결정됨.
2. 문제의식: minimum distance `d(C)`와 달리 stopping distance `s(H)`는 특정 패리티체크행렬 `H`의 구조에 종속되며 `s(H) ≤ d(C)`. 주어진 `H`에 대해 `s(H)=d(C)`가 되도록 만드는 최소 중복 행수(stopping redundancy `p(C)`)를 정확히 계산하는 실용 알고리즘이 그동안 없었음(지수적 시간복잡도).
3. 핵심 정의: stopping set `S`는 `H`의 부분행렬 `H_S`에 weight-1 행이 없는 열 집합; 벡터 `r`이 `x_S·r=1`을 만족하면 `S`를 "커버"하여 stopping set이 아니게 됨.
4. 핵심 기법: SS multiplicity `α(j) = Σ e_j·x_S` (index j가 몇 개의 stopping set에 속하는지) 계산 후, `α(j)` 내림차순으로 열을 재배열하고 가우스 소거로 `[I_m | Z]` 형태를 얻어 `Z`를 `A`,`B` 두 부분행렬로 분할.
5. 핵심 식: 열 부분집합 `L` (크기 σ)을 무작위 선택, `A`,`B`의 행 조합 중 `L` 위치가 일치하는 쌍(`a_iA|L = b_iB|L`)을 찾아 저weight 후보 dependent row `r'`을 생성(Canteaut의 최소weight 코드워드 탐색 알고리즘을 stopping set 커버링에 응용).
6. 반복 구조: 생성된 후보 벡터 중 가장 많은 stopping set을 커버하는 `r`을 선택해 추가하고, 커버된 stopping set을 집합 `X`에서 제거 → `X`가 빌 때까지 반복(외부 루프 `t_max`회 반복해 최소해 탐색).
7. 부수 트릭: `t≥2`일 때 열 하나를 무작위로 스왑하여 매 시행마다 다른 순열 `π` 생성, 국소 최적에 갇히지 않도록 함. 파라미터 `σ`(열 부분집합 크기)와 `t_max`는 코드별로 실험적으로 설정.
8. 학습/탐색 절차: 별도 학습 없음. Monte-Carlo식 확률적 탐색을 `t_max`회 반복해 dependent row 수의 최솟값(`ℓ_min`)을 추적.
9. 결과(수치): (155,64) Tanner code는 `p(C)=113`(원래 redundancy 91 대비 +22행), (504,252) MacKay code는 `p(C)=255`(+3행), (1008,504) MacKay code는 `p(C)=505`(+1행); 계산 시간은 각각 767초/43초/143초(Core2Duo 2.67GHz). 증강된 H로 BEC 블록오류율이 저erasure 영역에서 개선됨을 그림으로 확인.
10. 한계: BEC 전용 stopping-set 분석(실제 AWGN/soft LLR min-sum 채널 미검증), 대상 코드가 소형·저rate(155~1008, rate 0.41~0.5) regular LDPC, HW 구현·양자화·throughput 등 전혀 다루지 않음.
