### B. 알고리즘 요약

1. 시스템: base matrix `P`(J×L, 원소는 shift 값 `p_{j,l}`, 0≤p_{j,l}<p)로 정의되는 QC-LDPC 부호, circulant permutation matrix `I(p_{j,l})`로 확장한 parity-check matrix `H`.
2. 문제: 기존 사이클 제거법은 QC 구조를 보존하지 못해 인코딩 복잡도가 늘거나([9]), 파라미터(특히 code length) 선택 유연성이 부족함(특히 non-binary QC-LDPC).
3. 가정: QC-LDPC에서 한 사이클의 "인접 사이클"(adjacent nonzero element로 정의)들이 같은 길이의 사이클 그룹(cycle-group)을 이루며, 그룹 내 임의의 한 사이클이 그룹 전체를 대표(characterize)한다는 QC 고유 성질을 이용.
4. 핵심 기법: 체크노드 `r`의 neighborhood tree를 구성해 cycle-path를 탐색하고, 각 행 섹션(section)마다 새로 등장하는 사이클만 모은 집합 `C(1),...,C(J)`의 합집합 `C`로 전체 사이클(그룹 대표)을 중복 없이 탐색.
5. 핵심 식: 보조 이진행렬 `M`(c×s, c=사이클 수, s=circulant 행렬 수)을 만들어 각 반복에서 열가중치(column weight)가 가장 큰 열(가장 많은 사이클이 지나가는 circulant 행렬)을 찾아 해당 circulant 행렬을 통째로 0-행렬로 치환(삭제)하고 관련 사이클을 M에서 제거, `M`이 all-zero가 될 때까지 반복 — 삭제하는 circulant 행렬 수를 최소화해 최소거리(및 error-floor) 저하를 억제.
6. 확장: 여러 개의 서로 다른 QC parity-check matrix `H1,...,Hn`(같은 행 수, 같은 circulant 크기)을 옆으로 이어붙여 `H=[H1 H2 ... Hn]`를 만든 뒤, 이 알고리즘으로 짧은 사이클(주로 length-4)을 제거해 더 긴/유연한 non-binary QC-LDPC 부호를 구성.
7. 부수 디테일: 탐색 복잡도를 `Q(2l)`(제안 알고리즘)와 `Q'(2l)`(기존 [9]/[10] 방식)로 수식 유도해 비교, 예시(p=63,J=4,L=8,l=2)에서 제안 알고리즘이 약 165배 저복잡도.
8. 학습/최적화: 별도 학습 없음. 결정론적 그리디(탐욕적 column-weight 최대 삭제) 알고리즘.
9. 결과: (4113,2288) rate-0.556 이진 QC-LDPC를 girth 8→10으로 개선해 BER 10⁻⁶에서 약 0.4dB 이득; 16-ary (360,240) 부호는 girth 6→8로 1.25dB 이득이며 동일 rate의 IEEE 802.16e 이진 (1440,960) 부호 대비 0.15dB(BER)/0.2dB(FER) 우위.
10. 한계: 부호 구성(설계) 단계에만 적용되며 디코더/HW 변경 없음, 검증은 AWGN 채널 SPA/FFT-QSPA 시뮬레이션(최대 반복 50회)뿐, NAND 채널·soft-read 양자화 관련 언급 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC base matrix / circulant shift 구조 | `ecc_data.h` `PCM_b`, `ecc_top.cpp` `Load_PCM()` | Prime ECC도 base+shift QC-LDPC 구조라 사이클 제거 알고리즘이 동일 표현 위에서 동작 가능 |
| 저복잡도 cycle removal (girth 확대) | `ecc_top.cpp` `Load_PCM()` (H-matrix 생성/로드 단계) | H-matrix 재설계 시 참고 가능하나 코드는 특정 QC-LDPC 고정이라 재검증 부담 큼 |
| non-binary QC-LDPC 결합 구성(Section IV) | (대응 없음) | Prime ECC는 binary 전용, non-binary 결합 기법은 적용 대상 아님 |
| SPA/FFT-QSPA 디코딩 성능 검증 | `decoder.cpp` `LDPC_Decoder()` | 검증에 쓰인 디코더 자체는 Prime ECC의 min-sum과 다른 계열(Sum-Product), 디코더 이식 대상 아님 |

적용 가치: **중간** — QC-LDPC 구조를 보존하는 저복잡도 girth 확대 기법은 H-matrix 설계 단계 이식 이론상 가능하나, Prime ECC는 특정 H-matrix가 고정되어 있어 재검증 부담이 크고(§4 이식 난이도 "높음" 항목) HW/디코더 이득은 없음.
