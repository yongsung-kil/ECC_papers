### B. 알고리즘 요약

1. 시스템: star-topology WSN 중앙노드 FEC 복호, 짧은 패킷; rate-1/2 binary **SFCC-N4**(systematic feed-forward convolutional, no 4-cycle) 부호, AWGN + BPSK, length 200.
2. 문제: WSN에서 지연·성능이 전력보다 중요할 때 delay·complexity·performance trade-off를 user 요구에 맞춤; BP/iterative는 메모리·지연 커서 delay-intolerant에 부적합.
3. 모델: SFCC-N4를 quasi-regular (2N,J,2J) LDPC로 간주(주기적 sparse H, 홀/짝 column weight `J1,J2`), 트렐리스와 Tanner graph 이중표현.
4. 핵심기법 1단: **modified serial min-sum** — variable node를 순차 1개씩 갱신, check당 최소 두 값 `γ1,γ2`와 갱신값 `γu` 3개 비교로 min 추적, **단 1 iteration**만 실행해 실시간 연속출력.
5. 의미: serial 스케줄이 parallel min-sum보다 수렴 빠르고 저장량 적음; 1회 iteration으로 `L(qnm)` 재저장 불필요.
6. 핵심기법 2단: 1단 출력이 valid codeword 아니면 **adaptive list decoder** — 트렐리스에서 경로 이탈 시점 `t`부터 `t'`까지 M-algorithm으로 `Mp` survivor path 확장, 최대 metric 경로 복구.
7. 구현 디테일: 경로가 `dL` 내 두 번 깨지면 구간 병합해 재복호 중복 제거; TSHD 총 지연 = `L + dL`, 추가메모리 미미(주기적 H).
8. 복잡도: TSHD = `N(10J+⌈log2 2J⌉−2)+θ(7Mp−1)`, θ=재복호 트렐리스 stage 수, `ρ=θ/N`; SMAD=`N(7Mp−1)`.
9. 결과(시뮬): hard-decision, BER 10⁻⁴에서 Mp=8 시 SMAD 대비 0.4dB 우수 + 연산 33.4% 절감, Mp=32 시 동일성능 + 74.6% 절감; `dL>3L`은 성능이득 없음.
10. 한계: HW 미설계(시뮬만), hard-decision·짧은 프레임 WSN 특화, 컨볼루셔널 트렐리스 구조 의존.
