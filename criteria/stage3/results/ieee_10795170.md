### B. 알고리즘 요약

1. 대상: 이진 선형 코드 전반(RLC, eBCH, product code 등) — 특정 부호 구조에 무관한 GRAND/GCD 계열 디코더.
2. 문제: 기존 SI(soft-input) 디코더 대부분은 정확한 blockwise/bitwise soft output(SO)을 생성하지 못함(Forney의 근사는 부정확).
3. 모델: 채널 출력 `Rn`, 노이즈 `Nn`에 대해 `Yn = Cn ⊕ Nn`, GRAND는 노이즈 추정치 `zn,1, zn,2, ...`를 likelihood 내림차순으로 순차 질의(guess)해 코드북 소속 여부(`yn ⊖ zn,q ∈ C`)를 검사.
4. 핵심 기법: GCD(Guessing Codeword Decoding)를 GRAND 관점으로 재해석 — 정보비트 `k`개에 대한 질의 하나가 `2^(n-k)`개의 확장 질의를 대표한다고 보고, 그 그룹 likelihood 합을 이미 계산된 값으로 활용.
5. 핵심 식: 식(6),(7) — 식별된 코드워드들의 likelihood 합과 "리스트 밖(미식별)" 확률 `p(C\L|rn) = 1 - Σp(cn,j|rn)`을 결합해 blockwise SO 근사, 이를 통해 비트별 LLR(식 8) 도출.
6. 확장: 종료조건(식 5)을 Λ개의 최우도 코드워드까지 찾도록 완화하면 리스트 크기 Λ의 list decoder로 확장 가능.
7. 구현 트릭: 별도 dynamic memory 불필요한 ORBGRAND의 1-line 질의 순서(GCD-ORB)를 사용해 HW 친화적 순서로 질의 생성.
8. 최적화 절차 없음(학습/DE/GA 등 미사용, closed-form 근사식 기반).
9. 결과: RLC/eBCH에서 SO 예측 BLER가 실측과 정확히 일치(Λ≥2), Forney 근사 대비 크게 개선. eBCH product code (256,121)/(1024,676)/(4096,3249)에서 SO-GCD-ORB 기반 turbo 복호가 ML 성능(TUB) 근접.
10. 한계: HW 미설계·시뮬레이션 결과만 제시, 대상 코드가 LDPC가 아니며 min-sum 기반 반복복호와 무관한 별개 패러다임(guessing/query 기반).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GRAND/GCD guessing 기반 디코딩 전체 | 대응 없음 | Prime ECC는 min-sum 기반 message-passing 디코더로, 노이즈 패턴 순차 질의 방식과 구조적으로 무관 |
| soft output(SO) LLR 계산(식 8) | 대응 없음 | LLR 산출 방식이 채널 통계 기반이 아닌 guessing likelihood 누적 방식이라 `channel.cpp`의 LLR 양자화 테이블과 무관 |
| SISO 반복 복호(turbo product code) | 대응 없음 | Prime ECC는 QC-LDPC 단일 코드 구조이며 product code(행/열 반복복호) 아님 |

적용 가치: **낮음** — 대상 코드(RLC/eBCH/product code)와 디코딩 패러다임(guessing/query 기반)이 Prime ECC의 QC-LDPC min-sum 구조와 근본적으로 달라 이식 가능한 요소가 없음.
