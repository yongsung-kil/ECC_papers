### B. 알고리즘 요약

1. 시스템: BI-AWGN 채널, protograph 기반 QC-LDPC, rate-1/2, 정보길이 약 `1024` bit (2단 PEG lifting: ×4 후 ×64 또는 ×85).
2. 문제: 기존 protograph는 decoding threshold만 최적화 -> 좋은 성능에 큰 반복횟수 필요 -> 복잡도·전력 제약 응용에서 부적합.
3. 모델: iterative decoding threshold(무한 blocklength에서 신뢰디코딩 가능한 최소 채널품질)에 "미리 정한 반복횟수" 차원을 추가.
4. 핵심기법: PEXIT chart를 반복횟수 `I=20`으로 고정해 threshold를 계산하고, 그 조건에서 threshold 최소화하는 proto-matrix 탐색.
5. 식 의미: 반복횟수 제한 threshold = "20 iteration 안에 수렴 가능한 최소 Eb/No" -> delay-sensitive 응용의 실제 성능 지표.
6. 확장: punctured(7×4, degree-1 + high-degree punctured node)와 non-punctured(8×4) 두 구조 모두에 적용.
7. 구현 디테일: linear minimum distance growth 보장 위해 앞 4열 edge 합 >=3, 병렬 edge 최대 3 제약으로 탐색공간 축소.
8. 최적화: 32개(또는 16개 축소) proto-matrix 원소 위 search space reduction 후 단순 탐색.
9. 결과: Code-3(punctured)은 Code-1 대비 약 0.5 dB, Code-4(non-punctured)는 Code-2 대비 0.23 dB / Code-1 대비 0.72 dB FER 이득(20 iteration, FER=4e-4). error floor 미관측(FER 1e-4까지).
10. 한계: HW·디코더 미설계, 시뮬만, rate-1/2 AWGN 한정, gate count/throughput 없음.
