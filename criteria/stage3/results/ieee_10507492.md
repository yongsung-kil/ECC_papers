### B. 알고리즘 요약

1. 대상 부호: IEEE 802.3an regular LDPC (`N=2048`, `R=0.8413`)와 PEGReg504×1008 (`N=1008`, `R=0.5`), AWGN + BPSK.
2. 문제: 원본 SBF는 threshold sequence `θ(ℓ)`가 작을 때 non-error bit까지 flip해 error propagation 발생, error-floor 성능 저하.
3. 모델: 수신 `y = ĉ + z`, syndrome `s = Hx mod 2`, SBF 에너지 `Ek = Σ_{i∈M(k)} s_i` (채널정보 무시).
4. 핵심 기법 1단: flipping을 `SSBF ∩ S`로 제약. `S`는 narrowed flipping set으로 반드시 `Serr ⊆ S` 필요, `|S|`가 짧을수록 성능↑.
5. `S` 구성: `S1` = `|y|` 최소 top-`T1` (unreliable) bit + `S2` = 나머지에서 랜덤 `T2`개 → `S = S1 ∪ S2`.
6. 근거식: 한 trial에서 `Serr ⊆ S` 확률 `p`를 일반화 이항계수(gamma 함수)로 계산 (식 5,7), `γ` trial 누적 `Pγ = 1-(1-p)^γ`.
7. 파라미터: `(T1,T2)`는 `p`와 `T1+T2` 최소화 절충으로 선택 (예 code2 `(400,200)`, code1 `(300,400)`), `γ`=trial 상한.
8. 2단 구조: 1단은 원본 SBF, 미수렴 시 2단에서 `S`로 제약하고 threshold sequence는 첫 번째만 사용해 복잡도 절감.
9. 결과: code2에서 BER=10⁻⁶ 기준 `γ=50`으로 원본 SBF 대비 1.2 dB 이득, code1은 약 0.7 dB; 단 S-NGDBF보다는 0.25~0.3 dB 열위.
10. 한계: HW 미설계, 시뮬만, 평균복잡도 저SNR에서 최대 8배, 저SNR에서 unreliable bit 선별 무력화.
