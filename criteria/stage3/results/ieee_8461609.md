> 총평: NAND TLC 직접 타깃의 shuffle-IDD 스케줄러·structure interleaver로 메모리·throughput을 실측급으로 개선한 논문이나, IDD(반복 복조-복호) 전제와 8-PAM coded modulation 채널이 Prime ECC의 단발 hard/soft-read 구조와 달라 스케줄링 아이디어만 부분 이식 가능.

### B. 알고리즘 요약

1. 시스템: TLC NAND(8-PAM, 비-Gray 매핑) coded modulation, rate-0.9 (18396,16644) 매우 희소 LDPC(블록열 구조, circulant size `Z`), NMS 복호.
2. 문제: 비-Gray 8-PAM용 희소 LDPC로 IDD 시 복호기-복조기 간 latency·hardware idle이 커져 non-IDD 대비 throughput/면적효율 열위.
3. 기법 1단: shuffle-based(vertical scheduling) 복호를 활용, PCM을 G개 블록열로 나눠 블록열 순차로 V2C/C2V(NMS: `R=α·S·min{...}`, 0<α<1) 갱신 → extrinsic `Le`를 실시간 전달 가능.
4. 기법 2단: HW친화 structure interleaver — 한 8-PAM 심볼의 3비트를 연속 3개 블록열(`b_j,z, b_{j+1,z}, b_{j+2,z}`)에 배치, 복조기가 3블록열 `Le`만 받으면 `Lc` 재계산 가능.
5. 의미: 랜덤 interleaver 대신 구조화하여 복호기가 전체 블록열 `Le`를 다 계산해 넘길 필요 없이 3열 단위 파이프라인 → 1-codeword 처리 실현, idle 최소화.
6. 부가 트릭: 블록열을 degree 내림차순 배치해 C2V recover/APP adder vs V2C calculator 간 HW conflict(idle) 회피.
7. 인터페이스: `Le`를 전 블록열 저장 대신 3열 buffer로 대체, demodulator↔decoder 메모리 bank 최적화.
8. 양자화: 5비트(non-IDD) / 6비트(IDD), 90nm TSMC 1P9M, 190MHz, FO4 27.8ps.
9. 결과: throughput 1555 Mbps(layer-IDD 1100 대비 +40%), gate count 1888K, HW효율 292.19 Mbps/mm² (+41.6%), Le/Lc 메모리 53.5% 절감(1028K→478K), BER 0.5dB 개선.
10. 한계: 8-PAM IDD 전제·특정 비-Gray 매핑 부호([5]) 종속, iteration 감소는 아니며 반복 복조 자체가 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| shuffle(vertical) 블록열 스케줄링 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 블록열 순차 스케줄은 Prime ECC의 z=32 row-병렬 스케줄과 상이, 재구성 필요 |
| NMS C2V/V2C 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | min-sum 동일 계열이라 이미 보유(중복 기여) |
| structure interleaver / Le·Lc 버퍼 | 대응 없음 (Prime ECC는 단발 read LLR, IDD 복조 루프 없음) | 8-PAM 심볼-블록열 매핑 인터리버는 우리 구조에 없음 |
| 8-PAM 비-Gray coded modulation 채널 | `channel.cpp` `Set_LLR_Th()`, `Set_R_Offset()` (확장 필요) | Prime ECC는 셀-비트 LLR을 직접 산출, 심볼 단위 IDD 채널모델 부재 |
| block column degree 정렬(HW conflict 회피) | `GT.cpp` `Make_GT_HW()` (개념적 유사) | edge/열 스케줄 최적화 아이디어는 GT와 유사하나 대상 다름 |

적용 가치: **중간** — NAND TLC 직접 타깃이며 메모리·throughput 개선이 실측급이지만, 핵심 이득이 IDD(반복 복조-복호)+8-PAM coded modulation 전제에서 나오므로 단발 read 기반 Prime ECC엔 스케줄링·인터리버 아이디어만 부분 적용 가능하다.
