### B. 알고리즘 요약

1. (576,432) LDPC code, rate 3/4 (IEEE 802.16e), AWGN 채널, BPSK 변조, 32-layer(15 iteration) 신경망 구조.
2. 기존 NOMS(Neural Offset Min-Sum)는 각 엣지별 독립적 offset 파라미터만 학습, CN의 실제 입력 특성을 반영하지 못해 학습이 최적화되지 않는 한계.
3. 채널은 AWGN, 노이즈는 표준 가우시안 가정, 별도 NAND/비대칭 노이즈 모델 없음.
4. 핵심 기법: OMS의 CN→VN 메시지 갱신식(`u^t_{c,v} = ReLU(min|u^{t-1}_{v,c}| - β^t_{c,v}) * sign(...)`)에서 offset `β^t_{c,v}`를 CN 최소 입력값 기반 적응형 벡터로 대체.
5. 핵심 식(13): `β^t_{c,v} = (1 - α^t/2) * (min_{v'∈M(c)\v} |u^t_{v',c}|)/2` — offset을 CN 입력 크기에 비례시켜 반복마다 하나의 학습 파라미터 `α^t`만 필요.
6. NOMS는 엣지별(check node 수에 비례) 파라미터가 필요하지만, NAOMS는 iteration당 1개 파라미터만 학습 → 파라미터 수 대폭 감소.
7. Cross-entropy loss, mini-batch SGD(batch=120), Adam optimizer(lr=0.001), SNR 0~5dB 구간에서 학습 데이터 생성.
8. 학습/최적화: 신경망 학습 기반(training으로 α^t 결정), 별도 DE/GA 없음.
9. 결과: SNR=5dB에서 BER 10^-8 달성, NOMS 대비 약 0.8dB 개선. 연산량은 ADD 감소, MUL 소폭 증가로 "약간의 복잡도 증가".
10. 한계: HW 미설계(FPGA는 future work로만 언급), AWGN 시뮬레이션뿐, 부호구조(Basic Graph) 변경 시 일반화 성능 저하(모델 기반 파라미터가 Tanner graph edge에 종속).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum offset 갱신 (NAOMS β^t 학습) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum offset 값을 신경망 학습으로 대체하는 아이디어는 대응되나, 학습 기반 파라미터 도입은 기존 정적 offset 테이블 구조와 이질적 |
| VN 메시지 갱신 | `decoder.cpp` `VN_Cal_HD()` 등 | 식(3) VN 갱신 자체는 표준 min-sum과 동일, 별도 개선 없음 |
| 신경망 학습(파라미터 최적화) | 대응 없음 | Prime ECC는 bit-exact HW 시뮬레이터로 신경망 학습 파이프라인 부재 |

> 적용 가치: 낮음 — 신경망 학습 기반 offset 최적화는 Prime ECC의 정적 LLR/offset 테이블 구조와 이질적이며, AWGN·비NAND 부호(802.16e)에서만 검증되어 이식 시 재학습·재검증 부담이 큼.
