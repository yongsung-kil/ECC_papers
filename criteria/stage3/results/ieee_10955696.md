### B. 알고리즘 요약 (10줄 내외)

1. DNA 데이터 저장: LDPC(outer code, rate `Ro=k/n`)와 marker code(MC) 또는 embedded marker code(EMC, inner code)를 직렬연결(concatenated coding)해 IDS(insertion/deletion/substitution) 채널 오류를 정정.
2. 문제: DNA 합성·나노포어 시퀀싱 시 동기화 오류(삽입/삭제)와 치환 오류가 동시 발생, 복제 용이성으로 얻는 다중 출력 서열(M개)을 효과적으로 결합하지 못하면 복호 성능이 저하됨.
3. 채널 모델: 각 입력 base 심볼에 삽입확률 `pi`, 삭제확률 `pd`, 치환확률 `pt·ps`를 부여한 IDS 채널 + nanopore 특화 치환 전이확률(`p1=β, p2=3β, p3≈0`).
4. SPM(Segmented Progressive Matching) 알고리즘: 긴 서열을 m개로 분할 후 Levenshtein distance 기반 progressive matching으로 다중 출력 서열의 합의 서열(consensus sequence)을 추출, 복잡도를 PM 대비 선형(m배) 절감.
5. SDH(Synchronous Decoding based on HMM): 오프셋 변수 `Ok`를 은닉상태로 하는 HMM forward-backward(식 9~11)로 base 심볼의 APP를 계산 후 LLR로 변환(식 12~13), NMS 외부복호기에 전달.
6. IED(Iterative External Decoding, EMC용): 내부(동기화) 복호와 ENMS 외부복호를 최대 `Jmax`회 반복 교환(피드백 LLR `fj=L_total,j - Lj`)해 APP를 정교화.
7. ENMS(Embedded NMS): marker bit(`j∈D(m)`)에 대해 check/variable node 갱신과 total LLR 계산을 생략, NMS 대비 `CM(ω+Cρ)`만큼 연산 절감.
8. 실험: QC-LDPC(4096,4544) rate 0.9, itermax=20, marker DS=4nt/DE=18nt, 저장효율 `RMC=0.74`, `REMC=0.72`.
9. 결과: SPM이 PM 대비 M=10에서 BER 64.2% 개선(M>3에서 SPM 우세), 기존 연구 대비 BER 21.72~99.75% 감소, IED가 SDH 대비 BER 2자릿수 낮음(β=0.04에서 각각 98%/54% 개선).
10. 한계: IED는 반복마다 복잡도가 지수적으로 증가, LDPC 자체는 표준 NMS의 marker-bit 특화 변형(ENMS)에 그치며 DNA nanopore 채널·동기화 오류에 특화되어 NAND 채널과 근본적으로 다름.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS/ENMS 외부 LDPC 디코딩 (min-sum 변형) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 정규화 min-sum 자체는 이미 보유 기법과 유사하나 marker-bit 갱신 생략은 NAND에 대응 없음 |
| SPM/SDH/IED (IDS 채널 동기화, HMM) | 대응 없음 | NAND는 삽입/삭제가 없는 고정 위치 채널이라 동기화 복호 개념이 적용되지 않음 |
| LLR 계산 (식 2, 12, 13) | `channel.cpp` `Set_LLR_Th()` | soft LLR 산출 개념은 유사하나 DNA base 심볼 특화 매핑이라 직접 이식 불가 |
```

> 적용 가치: 낮음 - LDPC 외부복호가 표준 NMS 변형 수준이고 핵심 기여(SPM/SDH/IED)는 DNA IDS 채널 특화 동기화 문제 해결이라 NAND binary QC-LDPC ECC와 문제 도메인이 근본적으로 다르다.
