### B. 알고리즘 요약

1. 대상은 CMMB 표준 rate 1/2 LDPC (K=4608, N=9216, row weight 6/column weight 3 regular), AWGN·BPSK, normalized min-sum(계수 0.8) 시뮬.
2. 풀려는 문제: 기존 LBP(layered BP)는 check node를 행 순서대로 비동기 갱신해 먼저 처리된 check node의 신뢰도가 떨어져 후속 bit node에 악영향.
3. 배경 기법1(layered): H를 행 기준 layer로 나눠 상위 layer의 갱신된 check 정보를 하위 layer가 즉시 사용 → 수렴 가속 (식 (1),(2)).
4. 배경 기법2(dynamic): residual `R=|m_new−m_old|`이 가장 큰 check node를 우선 갱신·전파 (식 (7),(8)) → informed scheduling.
5. 제안 핵심: layered의 즉시반영과 dynamic의 순서갱신을 결합하되, residual 대신 새 신뢰도 지표로 check node를 정렬.
6. 신뢰도 정의: `r(ck) = min|Lvj| + sub-min|Lvj|` (연결 bit node LLR 절댓값의 최소+두번째 최소, 식 (12)).
7. 절차: 매 iteration 시작 시 모든 check node를 `r(ck)` 큰 순으로 정렬 → 그 순서로 식(1)로 갱신·식(2)로 연결 bit node 즉시 갱신 → 판정·종료조건.
8. 의도: 신뢰도 높은 check node를 먼저 갱신해 BER 높은 노드가 후속 bit node에 주는 오염을 억제.
9. 결과: 동일 BER에서 기존 layered 대비 SNR 약 0.4~0.5dB 개선, 동일 BER에서 iteration 수 감소(수렴속도 향상), max iteration 2~3의 저iteration 조건 시뮬.
10. 한계: HW 미설계, 매 iteration 전 check node 정렬 필요, regular CMMB code·AWGN float 시뮬만, gate count/throughput 미제시.
