### B. 알고리즘 요약

1. AWGN 채널 대상, rate 5/6·length 98304 CC-LDPC 계열 부호(모든 비교부호 동일 rate/length)를 FPGA로 구현·평가한다.
2. 기존 QC-LDPC는 CPM 기반 대수적 구성이라 짧은 길이에서 큰 girth 보장이 어렵고 고SNR에서 error floor가 남는 문제를 푼다.
3. 기존 CC-QC-LDPC(`[25]`)는 K개 동일 QC-LDPC 서브코드를 순환 결합(`Hcc`, 식 (5))해 높은 throughput과 단순 디코더를 얻는 구조다.
4. 핵심 기법: 서브코드 `Hs=[Hl Hm Hr]`의 CPM을 `z×z` 랜덤 순열행렬로 치환해 RP-CC-LDPC를 구성(`J=4, L=28, W=4, K=4, z=1024`).
5. 이를 base matrix `Bs=[1_4x4 1_4x20 1_4x4]`의 protograph로 보고, 각 1을 랜덤 순열행렬로 lifting해 `Bcc`(식 (7))를 만든다.
6. 부호 cycle 조건은 base matrix 원소 차의 mod-z 합=0(식 (3))으로 판정하며, 랜덤 순열이 short-cycle 회피에 자유도를 준다.
7. 비교부호 A/B는 fast hill-climbing(`[21]`)으로 4×24 QC-LDPC를 girth-8(regular)/girth-10(irregular)로 최적화해 만든다.
8. 디코더는 K개 동일 서브디코더 + global controller, 인접 서브디코더 간 C2V/V2C 메시지 전달, 4-bit 양자화·100MHz FPGA.
9. 결과: RP-CC-LDPC가 CC-QC-LDPC 대비 BER 10^-13에서 0.05dB 개선, error floor 미관측. 그러나 병렬도 32→4, throughput 1.55→0.182Gbps.
10. 한계: 랜덤 순열행렬 저장으로 메모리 대폭 증가·병렬화 붕괴, 낮은 Eb/N0에서는 regular/irregular QC-LDPC에 오히려 열세.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CC-LDPC 부호구조/랜덤 순열행렬 lifting | ecc_top.cpp Load_PCM() | H-matrix 구조 전면 상이(랜덤 순열) — QC 고정 구조와 비호환, 이식 부담 큼 |
| K 서브디코더 순환결합 디코더 | decoder.cpp LDPC_Decoder() | 구조적으로 다른 다중 서브디코더 아키텍처, 대응 어려움 |
| 4-bit 양자화 메시지패싱 | decoder.cpp VN_Cal_HD() 등 | 양자화 수준만 유사, 알고리즘 이득 없음 |
| CPM cycle 조건 (식 3) 기반 girth 최적화 | 대응 없음 | Prime ECC는 QC-LDPC 고정, hill-climbing 구성법 미보유이나 랜덤 순열은 방향 상반 |
```

적용 가치: **낮음** — 랜덤 순열행렬은 QC 구조의 병렬화/저메모리 이점을 정면으로 파괴해 NAND용 binary QC-LDPC 엔진 방향과 상반된다.
