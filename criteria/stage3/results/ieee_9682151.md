### B. 알고리즘 요약

1. TB(transport block)는 여러 CB(code block)로 구성되며, 각 CB는 systematic LDPC(`C[n,k]`); mother code로 (3,6)-regular PEG, WiMAX(802.16e), 5G-NR(QC raptor-like, `BG2`, lifting `Z`) 사용.
2. 문제: LTE/5G에서 CB 하나만 오류나도 TB/CBG 전체 재전송이라 자원 낭비 → TBER/CBER 개선 필요.
3. 채널은 BPSK AWGN. 두 공간결합 기법을 통합: PIC(정보 공유)와 PS(패리티 중첩).
4. PIC: 인접 CB 간 정보비트 일부(`coupling fraction β, 0≤β≤0.5`)를 공유·1회만 전송(puncturing), 경계 CB에 dummy bit로 a priori 정보 주입.
5. PS: 이전 코드워드 패리티 일부(`superposition fraction α`)를 선택행렬 `S=[O;I]`로 다음 CB에 XOR 중첩; 본 논문은 패리티에만 적용.
6. 통합 PIC-PS-LDPC: `α=0`이면 PIC-LDPC, `β=0`이면 PS-LDPC로 환원되는 상위집합; `α,β`를 2차원 탐색으로 최적화.
7. 인코딩(Alg.1): `v_ℓ = z_ℓ + z_{ℓ-1}S`로 패리티 중첩, `c_ℓ=(u_ℓ,u^c_ℓ,v_ℓ)` 구성, 양 끝 dummy block으로 종단.
8. 복호(Alg.2): high-level normal graph 위 sliding-window(window `d`) message passing; 각 층은 `+/S/=/LDPC/P/S` 노드, `LDPC` 노드는 SPA(SISO) `Imax` iteration, global iteration `Jmax`, forward/backward recursion + successive cancellation.
9. 결과: 5G-NR rate-1/2 len-1280(`Z=64`) 기준 PIC-LDPC 대비 0.4dB, 5G-NR 대비 0.6dB(CBER=1e-4) 이득; WiMAX 대비 0.9dB(TBER=1e-2); 고SNR에서 평균 iteration은 벤치마크로 수렴.
10. 한계: HW 미설계 시뮬만, SW/TB 복호로 지연·복잡도 증가, 5G-NR HARQ TB 전송에 특화된 다중 CB 결합.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| mother LDPC SISO 복호(SPA) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | mother code 복호는 우리 min-sum으로 대체 가능하나 SPA(full-tanh) 가정이라 직접 대응 아님 |
| PIC/PS 사슬결합·중첩 인코딩 | 대응 없음 | 우리 인코더는 단일 4KB dual-diagonal(`encoder.cpp`)로 CB 간 결합 개념 없음 |
| sliding-window 다층 message passing | 대응 없음 | 단일 코드워드 반복복호라 TB 층간 스케줄 없음 |
| 패리티 선택행렬 `S` 중첩 | 대응 없음 | NAND 단일 페이지 ECC와 무관 |

적용 가치: **낮음** — TB-HARQ 다중 CB 사슬결합 부호로, mother code(우리 QC-LDPC) 위에 결합층을 얹는 방식이라 단일 페이지 NAND ECC에는 지연·복잡도만 늘고 떼어 쓸 요소가 없음.
