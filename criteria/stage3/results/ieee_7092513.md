### B. 알고리즘 요약

1. 문제: 고throughput을 위한 병렬 turbo/LDPC 디코더는 interleaving law상 같은 메모리 뱅크 동시 접근 시 memory conflict 발생.
2. 목표: 데이터 원소가 처리 전체에서 같은 메모리 위치를 유지하는 in-place 무충돌 매핑을 임의 응용에 대해 최적 아키텍처로 구성.
3. 기존 한계: LDPC-like 문제는 P=B 뱅크로는 MRMW(multi-read multi-write) 매핑이 강제됨([13]), turbo만 in-place 가능([15]).
4. 핵심 아이디어: 뱅크 수를 P보다 늘려(P+1) in-place 확보, Vizing 정리(simple graph는 P+1 색으로 edge coloring 가능) 활용.
5. 절차 1단: 메모리 접근 스케줄 행렬을 conflict graph `G`로 변환(edge=데이터 원소, node=두 시간 인스턴스).
6. 절차 2단: uncolored edge 선택 후 인접 edge와 색 충돌 없으면 채색; 불가 시 dedicated 절차.
7. 부수 트릭: Vizing fan(누락색 순서열) 구성 → case1(fan 내 색 무반복: 색 교환) / case2(반복: path 추적·loop 검출 후 색 교환)로 충돌 edge 채색.
8. 결과 산정: 면적은 90nm STMicroelectronics NAND gate 등가로 측정, 제어부+메모리가 면적의 ~95% 차지(PE·네트워크 제외).
9. 결과: NB-LDPC 코드 대상 block length 48~788, P=4/6/16 실험에서 [18] 대비 최대 15%, [13] 대비 최대 40% 면적 절감.
10. 한계: simple graph에만 유효(multigraph용 in-place edge coloring 미해결), 대상이 NB-LDPC/turbo interleaver로 부호설계·복호 알고리즘과 무관.

### C. Prime ECC 관련 모듈 핀포인트

대상이 `other`(인터리버 메모리 매핑 EDA 방법론, 디코더 알고리즘 아님)이므로 C섹션 성능표는 전부 N/A. 모듈 매핑은 참고용:

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 병렬 메모리 접근/충돌 매핑 | 대응 없음 | Prime ECC는 z=32 고정 QC-LDPC circulant 구조로 interleaver 자유설계 대상 아님 |
| HW 메모리/라우팅 아키텍처 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) HW 모델(z=32, HCU, GT) | 병렬 HW 개념만 접점, 매핑 알고리즘은 무관 |
| 그래프 edge coloring | 대응 없음 | Prime ECC에 그래프 컬러링 기반 매핑 미보유 |

적용 가치: **낮음** — NB-LDPC/turbo interleaver의 메모리 충돌 해소용 EDA 도구로, 고정 circulant QC-LDPC(Prime ECC)의 디코더·부호설계·soft-read에 이식할 접점이 없음.
