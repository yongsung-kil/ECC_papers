### B. 알고리즘 요약

1. WiMAX irregular (1056,528, dv={2,3,6}, dc={6,7}) 및 10GBASE-T regular (2048,1723, dv=6, dc=32) LDPC, AWGN+BPSK 채널 대상.
2. 문제: 기존 stochastic decoder는 채널수신확률 `pi`만으로 고정된 stochastic bit stream을 생성해 randomness 부족 시 hold state에 갇혀 성능·latency 저하.
3. 채널/모델: 각 메시지를 Bernoulli 확률스트림(비트열)으로 표현, 1 decoding cycle(DC)마다 Tanner graph 각 edge에서 1비트 교환.
4. 핵심 기법: `Conditional Stochastic Decoding(CSD)` — 변수노드 `vi`에 연결된 모든 체크노드가 만족되면 다음 채널입력 비트 `Bi`를 연결된 체크노드 extrinsic bit들의 다수결(majority)로 설정, 하나라도 불만족이면 기존처럼 `pi` 기반 stochastic number generator로 생성.
5. 핵심 식 `Pc = Pa(1-Pb) + (1-Pa)Pb` (체크노드 XOR 연산의 확률적 표현) — 만족된 체크노드의 extrinsic bit가 신뢰도 높은 정보를 담고 있어 이를 재사용하는 근거.
6. 확장: degree-3 노드 연산을 기본으로 하고 고차수 노드는 degree-3 서브그래프 조합으로 구성.
7. 구현 디테일: variable node hold-state 방지를 위해 기존 EM(Edge Memory)/TFM 방식과 비교, CSD는 별도 메모리 없이 만족된 체크노드의 extrinsic bit를 직접 활용.
8. 최적화 절차: 없음(알고리즘 규칙 기반, 학습 없음).
9. 결과: (1056,528) 코드에서 SNR 3.75dB 기준 EM 대비 BER 절반, latency 70%(즉 30% 감소); expected DC는 EM 대비 최대 20% 이상 감소; HARQ 적용 시 FER 0.4dB 개선, latency 70% 수준.
10. 한계: HW 미설계(게이트/면적/전력 없음), latency는 DC(decoding cycle) 단위로만 측정되어 실제 클록/시간 환산 없음, 고차수 노드(dc=32)에서는 개선폭이 작아짐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 체크노드 만족 시 채널입력 bit 재생성(CSD 규칙) | 대응 없음 | Prime ECC는 min-sum LLR 메시지 기반이며 Bernoulli bit-stream 방식 자체가 없음 |
| 반복 디코딩 루프 (DC 단위) | `decoder.cpp` `LDPC_Decoder()` | 반복적 메시지 교환이라는 개념은 유사하나 내부 연산(min-sum vs stochastic)은 이질적 |
| 조기 종료(codeword test cHT=0) | `partial_CRC.cpp` `full_CRC.cpp` | 파리티 체크 기반 조기종료 개념은 유사하나 stochastic 특유 termination이라 코드 재사용 불가 |

적용 가치: **낮음** — latency 개선이라는 관심 방향은 일치하나, 완전히 다른 디코딩 패러다임(stochastic computation)이라 Prime ECC의 min-sum HW 구조에 알고리즘 자체를 이식할 수 없음.
