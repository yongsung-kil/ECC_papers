### B. 알고리즘 요약

1. 시스템: anytime 채널부호 - 소스가 매 시각 K비트 블록 생성, 인과적(causal) 인코더가 M비트로 부호화, rate `R=K/M`(예제 R=1/2), BEC(소실확률 `ε`) 전송.
2. 문제: P-LDPCC는 시간 진행에 따라 check node degree가 증가해 성능이 메시지 인덱스에 종속. SC-LDPCC는 인덱스 독립이나 유한지연 성능이 P-LDPCC보다 나쁨.
3. 모델: anytime 부호는 복호오류가 지연 d에 대해 지수적으로 감소해야 함, delay exponent `α = lim log Pe(i,i+d)/d`로 정의.
4. 핵심 기법: base matrix `B[1,t]`를 수정해 정보노드 `VI(i)`용 check node를 class `gi(U)`(다항분포)와 `gi(E)`(지수분포) 두 부류로 분할, `Dj,i` 연결수로 규정.
5. 핵심 식: `gi(U)`는 `dU`개 균등선택(다항), `gi(E)`는 `dE`개를 `Pr(d)=e^{-λd}(1-e^{-λ})/(1-e^{-λγ})` 지수분포로 연결 - 고정 degree로 인덱스 독립성 확보.
6. 확장: parity node `VP(i)`를 `C(i)`와 `VP(i-1)`에 각각 연결(C5)해 short cycle 제거 및 parity 복호 개선.
7. 구현 디테일: P-RA는 P-LDPCC(irregular)와 달리 regular 부호로, 각 노드 연결 패턴이 동일해져 점근적으로 인덱스 독립.
8. 학습/최적화: 별도 학습 없음, density evolution(DE) 방정식 (5)-(13)로 점근 거동 해석.
9. 결과: BEC(M=40, ε=0.44)에서 P-RA가 SC-LDPCC보다 유한지연 성능 우수, P-LDPCC와 동등하며 큰 인덱스에서 우월. DE와 시뮬 delay exponent 일치(예 0.1152 vs 0.1169).
10. 한계: HW 무설계, BEC 소실채널·anytime 실시간 제어 가정에 국한, 시뮬만.
