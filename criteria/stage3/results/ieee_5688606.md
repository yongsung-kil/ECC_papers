### B. 알고리즘 요약

1. 시스템: CMMB(중국 모바일 멀티미디어 방송)용 regular LDPC 디코더, 부호길이 `N=9216`, rate 0.5(4608×9216)와 0.75(2304×9216) 두 가지.
2. 문제: fully-parallel은 라우팅/게이트 폭증, fully-serial은 throughput 부족 → semi-parallel로 절충하되 메모리를 줄이는 것이 목표.
3. 복호 알고리즘: TDMP(=layered decoding, Mansour/Shanbhag), check/variable 노드를 동시 갱신해 iteration 수를 최대 50%까지 감소, TPMP 대비 2배 throughput.
4. CN 갱신은 offset min-sum: `Λ = ∏sign(ρ)·max(min|ρ|-β, 0)`, rate 0.5에서 `β=0.40`.
5. 메모리 압축 핵심: 식(3)의 min-sum은 행의 첫째 최솟값·둘째 최솟값·첫째 최솟값 인덱스 3값만 있으면 전체 check 메시지 복원 가능 → 전체 저장 대신 3값만 저장.
6. rate 재사용: check RAM 1라인(50bit)이 rate 0.5에서는 2행, rate 0.75에서는 1행을 담아 하드웨어 증설 없이 두 rate 공유.
7. 양자화: offset min-sum 8-bit 양자화, floating point 대비 SNR 손실 0.1dB 미만.
8. 병렬도: rate 0.5는 `p=32`, rate 0.75는 `p=16` 병렬 채널, 16개 process engine.
9. 결과: Stratix II FPGA 구현, TPMP 대비 메모리 57.3% 절감(식5), `MAX_it=8`(실효 7)에서 throughput 41.2Mbps(식6), CMMB 대역폭(22.2Mbps)의 약 2배.
10. 한계: 성능(BER/정정능력) 개선 없음(메모리 절감 목적), CMMB 특정 규격 H-matrix 종속, throughput이 NAND ECC급(Gbps)에 크게 못 미침.
