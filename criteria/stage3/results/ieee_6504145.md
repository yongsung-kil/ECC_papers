### B. 알고리즘 요약

1. 시스템: 대용량 stand-alone STT-MRAM, MTJ 저항 분포 기반 binary-input 비대칭 무기억 채널. dual-ECC 스킴에서 BCH(hard) 대신 LDPC(soft) 대체 검토.
2. 문제: STT-MRAM은 read/write 오류로 저항 spread가 넓어짐. worst-case Hamming/BCH는 저장밀도 손실 큼 → soft-decision LDPC로 정정능력↑ + 양자화 비트 최소화 필요.
3. 채널모델: 저·고 저항상태를 근사 Gaussian(`μ0=2.06kΩ, μ1=4.13kΩ`)으로 모델링, `σ0/μ0`를 키워 write/read 실패 spread 반영.
4. 핵심기법1: q-bit 양자기로 연속 채널을 `L=2^q` 출력 DMC로 변환, 상호정보 `I(X;Y)=H(Y)-H(Y|X)` 계산하여 채널용량 `C=max I(X;Y)` 도출.
5. 핵심식 의미: 양자기 경계 `t1=μ0+ασ0`, `t_{L-1}=μ1-βσ1`를 `α,β,q`로 파라미터화하고, 용량 최대화로 최적 경계 탐색 → 저에러율 영역까지 커버(시뮬만으로 불가한 영역).
6. 핵심기법2: 부호는 EG(3,2^2) 3D Euclidean Geometry로 (336,285) EG-LDPC (rate 0.848, `d_min≥5`) 구성, BCH(292,256,4)와 rate/length 정합.
7. 구현 디테일: RB-MS(reliability-based min-sum)는 양자화 구간 인덱스 `j`를 입력받아 정수/논리연산만 사용, extrinsic 정보를 check node에 사용해 성능 개선.
8. 최적화: 수치탐색으로 `q, α, β` 결정. 예: `σ0/μ0=0.17, q=3`에서 `α=1, β=1.6`일 때 `C=0.9677` 최대.
9. 결과: 3-bit capacity-max 양자기+RB-MS로 BCH 대비 BER 약 2자릿수 감소, BER=10^-6에서 tolerable spread 2% 향상. 평균 iteration 1.9~5.4회(spread 종속), Imax=5로 충분.
10. 한계: STT-MRAM 비대칭 채널 특화, HW 미구현(시뮬만), gate count/throughput 없음, 양자화 비트 절감 초점.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| capacity-maximizing 양자기 설계 (α,β,q) | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold/LLR 양자화 설계에 직결, 다만 채널 통계는 NAND용 재설계 필요 |
| iteration별/양자화 인덱스 → LLR | [ecc_data.h](../Prime_ECC_3.1_Claude) `PARAM_LLR`, [decoder.cpp](../Prime_ECC_3.1_Claude) `Get_VNU_Table_Idx()` | 양자 인덱스 기반 정수 메시지 전달과 유사, LLR 테이블 참조 방식 대응 |
| RB-MS check node (min1/min2) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 Min-Sum과 부분 중복(RB-MS는 extrinsic 사용 변형) |
| EG(3,2^2) 부호 구조 로드 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | EG-LDPC는 QC 구조 아님 → H-matrix 로드만 가능, 부호설계 이식성 낮음 |

적용 가치: **중간** — capacity-maximizing 양자기 설계 원리는 NAND soft-read LLR 최적화(`channel.cpp`)에 이식 가치가 있으나, EG-LDPC 부호와 RB-MS 복호는 우리 QC-LDPC/Min-Sum과 구조가 다르거나 중복이며 STT-MRAM 채널 특화라 재검증 부담이 크다.
