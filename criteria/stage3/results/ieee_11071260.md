### B. 알고리즘 요약

1. 시스템은 CPM(연속위상변조) 파형과 결합된 QC-LDPC 부호로, rate 1/2, 2/3, 4/5의 protomatrix 기반 코드를 설계함.
2. 기존 CPM용 LDPC 설계([10],[11])는 protomatrix 일부 컬럼을 사전 고정하고 나머지만 PEXIT로 탐색해 decoding threshold가 최적이 아니었음.
3. 채널은 AWGN이며, CPM 변조기·복조기가 LDPC 인/디코더와 SISO(soft-input soft-output) 모듈로 global iteration 루프에서 결합됨.
4. 핵심 기법은 `Algorithm 1` 컬럼별 완전탐색 PEXIT 최적화로, 모든 protomatrix 컬럼을 순차적으로 최적화하여 decoding threshold `(Es/N0)*`를 낮춤.
5. 선형 최소거리 성장 보장을 위해 `dv(j) ≥ 3` (모든 변수노드 차수 3 이상) 제약을 추가해 capacity 대비 1dB 이내를 유지하면서 error floor를 방지함.
6. 2단계 lifting(pre-lift 2 또는 4배 + 2차 ACE 알고리즘 기반 lifting)을 적용해 최종 H 행렬의 최소거리·girth를 증가시킴.
7. 조기 종료·병렬화를 위해 SIMD 128-bit 벡터(16-bit fixed-point, P=8)로 LDPC/SISO를 병렬 구현함.
8. 최적화 절차는 PEXIT 기반 컬럼탐색(교과서 EXIT 확장)이며, DE/GA/NN은 사용하지 않음.
9. 결과: 18개 LDPC-CPM 코드 조합에서 coding gain 7.1~10.9dB, BER은 10^-8 이하까지 error floor 없이 도달함.
10. 한계: 소프트웨어 SIMD 시뮬레이션만 수행했고(RTL/FPGA/ASIC 없음), CPM 변조 특화 설계라 일반 binary LDPC나 NAND 채널에는 직접 적용 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protomatrix PEXIT 최적화 (Algorithm 1) | `ecc_top.cpp` `Load_PCM()` | 부호구조 자체를 새로 설계하는 것으로 기존 고정 QC-LDPC와 직접 대응 안 됨 |
| 2단계 lifting / check-node splitting | 대응 없음 | Prime ECC는 고정 H-matrix 사용, 부호 재설계 절차 없음 |
| CPM SISO / global iteration 결합 디코딩 | 대응 없음 | CPM 복조 특화 구조로 NAND 채널 디코더와 무관 |
| SIMD 병렬 소프트웨어 디코더 | `decoder.cpp` `LDPC_Decoder()` | 병렬화 개념만 참고 가능, HW arch(z=32, HCU)와는 결이 다름 |

> 적용 가치: 낮음 — CPM 통신 채널 특화 부호 구성법이며 NAND binary LDPC 디코더/HW 구조와 직접적 접점이 없음.
