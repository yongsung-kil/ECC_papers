### B. 알고리즘 요약 (10줄 내외)

1. rate-0.531, 길이 2048bit irregular LDPC(Code43, 열차수 불규칙/행차수 규칙) 및 (3,6) regular LDPC를 대상으로, AWGN+BPSK 채널에서 log-BP(로그영역 sum-product) 디코딩.
2. 완전병렬 구현은 처리량은 높으나(1Gb/s ASIC, 54Mb/s FPGA) 복잡한 pseudorandom 라우팅망으로 코드 변경·확장이 불가능한 문제를 해결.
3. 채널 모델: `y = x + n` (BPSK+AWGN), 데이터노드 LLR `L_ch`가 디코딩 입력.
4. 핵심 기법: 하드웨어 제약 LDPC(HC-LDPC) 부호구성 — block-diagonal 구조로 `N`개 파티션을 만든 뒤, interleaver tag 기반으로 일부 edge를 파티션 간 이동시켜 randomness 확보.
5. 인터리버 메모리 크기는 `(파티션당 edge수) × N`으로, 파티션 수만큼 나누어 저장 가능(전체 edge 저장 대비 절감).
6. 디코더 아키텍처: 파티션마다 독립 PU(DP+CP)와 로컬 메모리 배치, sequential scheduling(flooding) + 두 개의 동일 interconnection network(circular shift)로 파티션 간 통신, 완전순차 처리 대비 처리시간을 파티션 수(N)로 나눔.
7. 고정소수점 8bit(Q4.4, 정수4bit+소수4bit) 양자화, log-BP의 두 비선형함수(φ 함수류)에 최적 log-base를 discrete DE로 탐색해 양자화 손실 보상.
8. 최적화 절차: 이산 밀도진화(discrete DE)로 코딩정밀도별 최적 log-base 결정, DE 문턱값으로 코드 성능 사전평가.
9. 결과: rate-0.531/2048bit 4PU 구현이 APEX20Ke FPGA에서 978LC(PU)+71LC(interconnect)+542LC(순차로직), 164224 memory bit, 최대 37.89MHz로 1Mb/s 처리량. 파티션(병렬도) 증가시 error-floor는 유지되며 반복시간 단축(예: SNR 2dB에서 16PU는 5회, 2PU는 15회로 성능 포화).
10. 한계: log-BP(비-min-sum) 알고리즘 기반, 처리량이 1Mb/s로 최신 NAND ECC 요구 대비 매우 낮고, 완전병렬 대비 속도는 파티션 수에 비례해서만 개선(당시 FPGA 자원 제약).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 부분병렬 스케일러블 아키텍처(파티션 분할+PU) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 z=32 row 단위 병렬 처리를 하나, 이 논문은 코드 자체를 파티션 단위로 구성해 유연성 확보 — 개념적 참고 수준 |
| interleaver/interconnection network (edge 라우팅) | HW arch (병렬화, 메모리, 라우팅) — 디코더 HW 모델(z=32, HCU, GT) | Prime ECC는 circulant shift 기반 고정 QC-LDPC라 이 논문의 랜덤 interleaver 방식과 구조적으로 다름 |
| H-matrix 구성(HC-LDPC, 파티션 block-diagonal) | `ecc_top.cpp` `Load_PCM()` | 부호 자체가 다른 구성법(HC-LDPC vs QC-LDPC)이라 이식 난이도 높음 |
| log-BP 유한정밀도 최적 log-base 탐색(DE) | `channel.cpp` `Set_LLR_Th()` | LLR 양자화 최적화라는 목적은 유사하나 Prime ECC는 min-sum이라 log-base 개념 자체가 대응 없음 |

> 적용 가치: 중간 — 부분병렬 파티션 아키텍처와 메모리/라우팅 절감 아이디어는 참고할 수 있으나, HC-LDPC 부호구조와 log-BP 알고리즘은 Prime ECC의 QC-LDPC+min-sum 구조와 직접 호환되지 않음.
