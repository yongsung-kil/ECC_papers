### B. 알고리즘 요약

1. Gbps급 무선전송용 QC-LDPC 부호 설계, 다중 code rate(예: 4/5, 2/3, 1/2)를 하나의 하드웨어로 지원하는 것이 목표.
2. 기존 방식(802.11n/16e)은 rate별로 별도 설계·별도 codeword length 고정이라 정보워드 길이가 가변적이 되어 rate compatibility 및 type-II hybrid ARQ 미지원 문제.
3. 정보워드 길이 k를 고정하고, mother code(최고 rate, 예: 4/5)의 systematic block `H_s(1)`을 순차적으로 row-split하여 낮은 rate 코드의 `H_s(i)`를 도출.
4. Row-splitting 함수 `f_{n,m}(A)`: 원 행렬 A의 non-zero entry를 m개로 균등 분할, `sum f_{n,m}(A) = A`, 서로 겹치지 않음(식 4,5,6).
5. Parity block `Hp = [hp | H̄p]`는 dual-diagonal 행렬 `H̄p`(대각+우측대각 identity)과 weight-3 열벡터 `hp`(첫행/마지막행/중앙행에 shift value a,b)로 구성, a=b=0으로 설정 시 인코딩 효율 이점(별도 논문[12] 참조).
6. Memory access time이 Gbps 디코더의 병목이라는 문제의식으로, LLR을 memory/bus 대신 register 직접 배선(interleaving)으로 처리하는 것을 전제로 하고, 이를 위해 rate별 하드웨어 재사용이 필요하다는 동기 제시.
7. Design example: rate 3/4 mother code를 4개 block matrix(C12,C11,C10,P1)로 표현, splitting function `g_i^k`로 shift값 x와 split index y를 조합해 하위 rate 행렬 생성.
8. 시뮬레이션: rate 2/3, 1/2 코드를 AWGN+BPSK 채널에서 최대 50 iteration, block error 50개 검출시 종료 조건으로 BLER 측정.
9. 결과: 제안 코드(2/3, 1/2 rate)가 802.16e 표준 코드와 comparable한 BLER 성능을 보임(Fig.2, 수치 곡선은 본문 서술 없이 그림 참조).
10. 한계: girth/에러성능 최적화를 위한 shift value 선택 알고리즘 상세 미기재, HW 구현/합성 결과 없음(순수 시뮬레이션 수준).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Row-splitting mother-code 부호설계 (다중 rate 지원) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 rate H-matrix 사용, 다중 rate 지원 목적 자체가 적용 대상과 맞지 않음 |
| dual-diagonal parity block 구조 | `encoder.cpp` `Generate_PCM_encoding()` | 이미 dual-diagonal 인코더 보유, 개념적으로 유사하나 신규 기여 없음 |
| register 직접 배선 기반 interleaving (memory access 병목 회피) | 대응 없음 | 논문에 구체 HW 구현 없이 아이디어 수준 언급뿐 |

적용 가치: 낮음 - 무선통신 표준(802.11n/16e)의 다중 rate 지원이 목적인 부호설계 방법론으로, NAND ECC의 고정 rate 요구사항과 부합하지 않고 HW 구현/성능 데이터도 없음.
