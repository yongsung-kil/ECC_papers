> 총평: 기존 표준 QC-LDPC(IEEE 802.3ca) 코드를 대규모 FPGA 어레이로 실시간 실측한 벤치마크 논문으로 신규 알고리즘 기여가 없어 이식 가치가 낮다.

### B. 알고리즘 요약

1. 대상은 IEEE 802.3ca 100G-EPON Task Force가 채택한 25G/50G-PON QC-LDPC 코드(rate 0.849, mother code rate 0.826), BPSK 변조 AWGN 채널.
2. 문제의식: 기존 FPGA 검증([2])은 처리량 약 0.1Gbps에 그쳐 완전 부하 상태의 error floor를 배제하지 못함.
3. 채널 모델: Box-Muller Gaussian variate generator로 32-lane 비상관 AWGN 노이즈를 생성해 SNR을 0.005dB 해상도로 정밀 제어.
4. 핵심 기법: 75개 Xilinx UltraScale XCVU125 FPGA(75MHz)로 QC LDPC 인코더/디코더를 병렬 구현. 수신단은 6-bit LLR soft decision, 32-parallel VNU/CNU가 revised min-sum 알고리즘의 layered decoding을 수행.
5. 핵심 수치: 75MHz 클록 x 32-parallel 구조로 디코딩 속도 90Gbps 달성 — 병렬화 규모가 실측 시간을 대폭 단축(`BER=10^-12`를 40분 내 달성, 기존 [2] 대비 10일→40분).
6. 부수 기법: parity puncturing 및 zero-padding 삭제 모듈로 rate 0.849/0.826 코드 모두 지원, shortening/puncturing LLR 처리 포함.
7. 결과: 출력 BER 10^-12 기준 입력 BER은 rate 0.849 HD 1.08e-2, rate 0.826 HD 1.16e-2, rate 0.849 SD 1.97e-2, rate 0.826 SD 2.86e-2. SD는 HD 대비 rate 0.849에서 0.95dB, rate 0.826(mother code)에서 1.54dB 코딩이득. BER 10^-15까지 31시간 실측, error floor 미관측.
8. 한계: 알고리즘/부호설계 신규성 없음(표준코드 채용, min-sum도 기존 기법), gatecount 등 HW 자원 수치는 그림(Fig.2)에만 있어 본문 서술로는 미상, NAND 채널·타깃 언급 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 병렬 디코딩(VNU/CNU, layered, revised min-sum) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 구조적으로 유사(QC-LDPC min-sum layered)하나 이미 Prime ECC가 보유한 기법이라 신규 기여 아님 |
| 32-parallel VNU/CNU 병렬 아키텍처 | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC의 `z_sb=32` lifting/병렬 구조와 유사 규모지만 논문은 세부 아키텍처 서술 없음 |
| soft-read 6-bit LLR 양자화 | `channel.cpp` `Set_LLR_Th()` | Prime ECC는 최대 soft-2~3bit(2SD/3SD) 지원, 6-bit는 soft-4bit+로 미지원 영역 참고용 |
| 대용량 FPGA 실측 방법론(90Gbps, error floor 확인) | 대응 없음 | 방법론적 참고는 가능하나 Prime ECC 모듈에 직접 대응되는 구현 요소 없음 |

> 적용 가치: 낮음 — 표준 QC-LDPC의 실측 벤치마크일 뿐 새 디코딩·부호설계 기법이 없고, Prime ECC가 이미 보유한 min-sum/layered/병렬 구조와 중복되어 이식 대상이 아니다.
