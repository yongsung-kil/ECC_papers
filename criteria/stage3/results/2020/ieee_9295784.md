### B. 알고리즘 요약

1. IEEE 802.16e 표준 기반 irregular binary QC-LDPC, rate 3/4, length 576/1152, BPSK/AWGN 채널에서 시뮬레이션.
2. Min-Sum 계열(LNMSA) 복호에서 관측되는 error floor(고SNR 영역에서 BER 곡선 기울기 급감)를 trapping set 문제로 규정하고, 구조를 몰라도 해결 가능한 딥러닝 기반 2단계 후처리(DLAPPS)를 제안.
3. 채널 모델은 표준 AWGN(BPSK, noise variance N0/2); 1단계 복호는 스케일링 팩터 `α=0.75`의 Layered Normalized Min-Sum(LNMSA), 최대 반복 50회.
4. 1단계에서 2회 iteration 간격으로 8세트 LLR + 초기/최종 LLR을 더해 총 10세트의 LLR로부터 평균·분산을 계산, 총 12개 특징을 변수노드별로 추출해 DNN(비교로 CNN/LSTM도 실험, 성능 차이 미미)에 입력해 Type I(정확) / Type II(오류) 노드로 분류.
5. 클래스 불균형 보정을 위해 가중 교차엔트로피 손실 `L = -[w1*y*log(ŷ) + w0*(1-y)*log(1-ŷ)]`을 사용, 오류 샘플 기여도를 강화.
6. Type I 노드 상위 `p=5`개는 LLR 크기 순으로 정렬해 신뢰도를 `γ=1.5` 배 증폭(정보 강화); Type II 노드는 LLR 절댓값 오름차순(의심도 높은 순)으로 하나씩 flip 후 LNMSA 재복호, 성공 시 종료·실패 시 다음 노드로 순차 진행.
7. 재복호는 최대 반복 `i_max2=50`, flip 시도 횟수는 Type II 노드 수(v2) 이하로 반복.
8. 학습은 Pytorch, learning rate 1e-4~(본문 표기 손상), batch 2048, 200 epoch로 지도학습(오프라인 사전학습).
9. 결과: (576,432) 코드에서 FER 10⁻⁵ 기준 LNMSA 대비 약 1.6dB 이득, (1152,864) 코드에서 약 1.1dB 이득; 복잡도는 `i_max2 × v2`만큼 증가하나 "slightly" 증가로 서술.
10. 한계: HW 구현/합성 없이 순수 소프트웨어 시뮬레이션(CPU+Pytorch)뿐이며, 모든 오류가 항상 정정되는 것은 아니고(트랩핑셋이 크고 복잡하면 실패), NN 추론과 다회 재복호로 인한 지연 증가는 정량화되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LNMSA 1단계 복호 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` (Dual-Update 옵션) | Prime ECC는 이미 Min-Sum + Dual-Update(layered 유사) 구조 보유, 신규 기여 아님 |
| 조기종료/재복호 트리거 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 오류 프레임 판정(H×cT≠0) 자체는 기존 CRC 조기종료 로직과 유사하나, 후속 DNN 분류·플립 루프는 대응 없음 |
| DNN 기반 노드 분류·비트플립 후처리 | 대응 없음 | Prime ECC는 NN 기반 후처리/비트플립 모듈이 없어 신규 개발 필요 |
| LLR 이력 기반 특징 추출(반복별 LLR 저장) | 대응 없음 | 반복별 LLR을 여러 세트 저장·통계 처리하는 구조는 기존 코드에 없음 |

> 적용 가치: 중간 | error-floor 저감이라는 목표는 Prime ECC의 관심 영역과 일치하지만, DNN 학습/추론 파이프라인과 다회 재복호 오버헤드가 기존 HW bit-exact 시뮬레이터 구조에 없어 상당한 신규 설계(추론 HW, latency 예산 검증)가 필요하다.
