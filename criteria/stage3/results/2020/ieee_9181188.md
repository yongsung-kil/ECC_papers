### B. 알고리즘 요약

1. 5G-NR eMBB 표준 BG1 base-graph(46×68, z=384) 기반 QC-LDPC, code rate 1/3, codelength 26112 bits.
2. 5G-NR 표준 준수 LDPC 디코더 HW 구현 사례가 전무하여 이를 최초로 제시하는 것이 목표.
3. AWGN 채널, soft-demodulator에서 생성된 LLR을 입력으로 사용.
4. offset min-sum decoding + layered scheduling(Algorithm 1): CN 갱신 시 `αm,n = min|xm,n| - offset`, `βm,n = max(αm,n, 0)`.
5. `z̃ = xm,n + ỹm,n` 식으로 VN 갱신된 belief를 합산 후 하드결정(`ĉn ≥ 0 → 0`)하고 `Ĉ·H^T = 0`이면 조기종료.
6. 384개 병렬 CVCPU(Combined Check-node & Variable-node Processing Unit)가 layer별로 19-LLR-combination을 동시 처리하는 fully-parallel 구조.
7. CE-LLR(32-bit, min1/min2/sign/index 압축 포맷)로 extrinsic LLR을 압축 저장(IntM)해 반복 간 메모리 절감; mVG-19 tree 구조로 min1/min2 탐색.
8. 별도 학습/최적화 절차 없음(고정 offset min-sum, 표준 BG1 사용).
9. Q=7bit, 10 iteration에서 Eb/N0=1dB에 BER 10^-6, FPGA(Virtex Ultrascale+) 102.45MHz에서 처리량 2.9Gbps(기존 대비 최대 20배).
10. HW 구조가 5G-NR 대용량 코드(26112bit, z=384) 전용으로 설계되어 NAND ECC 스케일과 크게 다르며, 정정능력 개선이나 error-floor 개선은 다루지 않음(순수 디코더 아키텍처/처리량 논문).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 계열 기법과 동일 알고리즘, 신규성 낮음 |
| layered scheduling | `decoder.cpp` `LDPC_Decoder()` | 이미 Dual-Update 등 레이어 스케줄링 보유, 참고용 |
| CE-LLR 압축 저장(min1/min2/sign/index) | `decoder.cpp` `Get_VNU_*`, `ecc_data.h` `PARAM_LLR` | 메모리 절감 아이디어로 부분 참고 가능 |
| fully-parallel 384-CVCPU 구조 | 대응 없음 | Prime ECC는 z=32 기반 row-병렬 구조로 스케일·아키텍처 상이 |
| BG1 H-matrix / 5G-NR 표준 부호 | `ecc_top.cpp` `Load_PCM()` | NAND 타깃 아님, 표준 특화 구조라 이식 난이도 높음 |

적용 가치: 낮음-중간 | 알고리즘(offset min-sum, layered)은 Prime ECC가 이미 보유하고 있고, HW 구조는 5G-NR 대용량/완전병렬 특화로 NAND 스케일과 맞지 않아 직접 이식 가치는 제한적. CE-LLR 압축 저장 기법만 참고 가치 있음.
