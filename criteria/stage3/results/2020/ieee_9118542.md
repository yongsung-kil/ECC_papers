### B. 알고리즘 요약

1. 코드 스펙: CCSDS 근지구 표준 type-II QC-LDPC, `(8176,7154)` (실질 rate=0.875), parity-check matrix는 `2×16` 배열의 `511×511` weight-2 circulant, row weight 32, column weight 4.
2. 문제의식: 위성 온보드 채널부호로서 고신뢰·고속이면서 FPGA 자원 소모가 적은 codec 구현이 필요.
3. 부호 구조: type-II QC-LDPC는 circulant마다 `1`이 2개뿐(weight-2)이라 각 circulant을 두 개의 initial phase(IP) 정수쌍으로 표현 가능 → decoder 저장소는 64개 IP만 있으면 됨.
4. 핵심 기법(디코더): min-sum(MS) 알고리즘, 6개 모듈(timing-control, control-signal-gen, input-storage, vertical operation, horizontal operation&decision, data-output)로 구성.
5. 핵심 연산: vertical op에서 `Q = a1+a2+a3+a4+p1`(complement code 덧셈), `b_i=Q-a_i`; horizontal op에서 sign은 31개 값 XOR, magnitude는 min/2nd-min 탐색 후 scaling factor 곱(shift&add로 근사).
6. 조기종료: 1×8176 codeword 벡터와 parity-check matrix 각 행의 내적(XOR 기반)이 all-zero면 반복 종료, 최대 10 iteration 고정.
7. HW 디테일: 16-column 파이프라인 병렬 처리로 vertical op 511 clock, horizontal op 280 clock/iteration; blockram 구성은 8개 dual-port(1022×16, 입력저장) + 64개 dual-port(511×8) + 2개 single-port(511×16, 판정 저장).
8. 학습/최적화 절차: 없음(고정 알고리즘 구현, GA/DE/NN 미사용).
9. 결과: XC4VLX200 FPGA에서 encoder 287MHz@810 slices/15 BRAM, decoder 244MHz@10481 slices/74 BRAM; coding gain은 BER 10⁻⁷에서 약 7.0dB, BER 10⁻⁸에서 Shannon limit 대비 1.56dB.
10. 한계: 위성 채널(AWGN, BERT 테스트) 대상, 10회 고정 iteration(적응형 없음), 연판정 비트수·LLR 양자화 세부사항 미기술, 실측 throughput(Gbps) 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum decoder(vertical/horizontal op, sign+min/2min) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 연산 구조 자체는 이미 보유 기법과 동일 계열이나, scaling factor·pipeline 세부 매핑 방식은 참고 가능 |
| 16-column 파이프라인 병렬 vertical operation | 디코더 HW 모델 (z=32, HCU, GT) | z_sb=32 병렬 단위와 유사한 열 단위 병렬화 개념이나 규모(16 vs 32)·구조는 상이 |
| 조기종료(parity 벡터 all-zero 검사) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료, 본 논문은 syndrome(parity check) 직접 계산 기반 — 방식이 달라 대응이 정확하지 않음 |
| type-II QC-LDPC circulant(weight-2, IP 저장) 구조 | `ecc_top.cpp` `Load_PCM()` | 특정 위성 표준 부호구조로 고정, NAND 고rate 요구와 rate 정합성 낮음(0.875) |
| 인코더(shift register 기반 systematic circulant 인코딩) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal이 아닌 systematic circulant 방식으로 구조가 다름 |

> 적용 가치: 중간 — min-sum 디코더의 HW 파이프라인·메모리 매핑 세부사항은 참고할 만하나, 부호 자체(rate 0.875, length 8176, 위성 채널 특화)와 조기종료 방식이 Prime ECC와 상이해 직접 이식보다는 아키텍처 참고용.
