### B. 알고리즘 요약

1. 시스템: QC-LDPC 디코더의 PE(Processing Element)-RAM 간 데이터 전송을 담당하는 인터커넥션 네트워크(interconnection network, IN). 5G 표준 QC-LDPC의 lifting size 집합 `L={2,...,320,352,384}`(51개 값) 대상.
2. 문제: 기존 barrel shifter/QSN 기반 IN은 한 번에 한 프레임(단일 lifting size)만 순환시프트 가능해, 서로 다른 길이·순열 제약을 가진 여러 프레임을 동시 병렬 처리(예: 5G 다중 코드 블록 동시 디코딩)하지 못함.
3. 핵심 가정: `L`의 각 원소의 최대 홀수 약수(g.o.d, greatest odd divider) 중 최댓값을 `a_odd`로 정의(5G의 경우 `a_odd=15`)하여 기본 블록 크기로 사용.
4. 핵심 기법: `a_odd`-GBS(Generic Barrel Shifter) 코어 블록을 기본 단위로 하고, 이를 2배씩 복제하며 "trading stage"(MUX 기반 전환 단)를 추가해 `N=2^n·a_odd-FPCN`(Fully Parallel Circular-shift rotation Network)으로 확장 — `n`회 반복 시 `N-FPCN`은 서로 다른 길이의 벡터 `q`개를 동시에 순환시프트 가능.
5. 핵심 식: MUX 개수 `γ_FPCN = γ_TS + 2^n·γ_GBS = n·N + 2^n·γ_GBS`, 스테이지 수 `δ_FPCN = log2(N)+1` — 기존 Beneš 네트워크(`γ_Benes`) 대비 MUX 수를 낮추면서도 프레임 병렬성(`ν`)을 유지.
6. 확장: 입력 벡터 할당 알고리즘(Algorithm 2/3)으로 각 벡터 `U_k`(길이 `M_k=2^{i_k}·j_k`)를 `2^{i_k}`개의 `a_odd`-GBS 블록에 분산 배치.
7. 부수 트릭: 제어신호(trading stage/GBS 회전량)는 오프라인 계산 후 ROM(Read Address Selection, RAS 블록)에 저장, 런타임엔 `q, M, P`로 주소만 계산.
8. 학습/최적화 절차 없음(결정론적 조합적 설계).
9. 결과: 5G 480-FPCN이 Xilinx Virtex4 FPGA 합성에서 GBS 설계([15]) 대비 MUX 5728개→(대안[16]) 4800개, 기존 [11]의 barrel shifter 12중 복제(17280 MUX) 대비 9216 MUX로 8064개 절감(약 47% 절감), Beneš 네트워크 대비 최대 66% MUX 절감.
10. 한계: 순수 라우팅/인터커넥션 네트워크 HW 설계이며 LDPC 복호 알고리즘(min-sum 등)이나 부호설계와는 무관. 실측 throughput/gate count(공정 미명시) 없이 MUX 개수·LUT/Slice 기준 합성 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PE-RAM 인터커넥션 네트워크(순환시프트 라우팅) | `GT.cpp` `Make_GT_HW()` | Prime ECC의 MUX 기반 edge 선택(Graph Thinning)과 유사한 MUX 절감 라우팅 아이디어, 다만 GT는 클록당 처리 edge 축소 목적으로 목적이 다름 |
| 다중 lifting size 동시 병렬 처리(FPCN) | 대응 없음 | Prime ECC는 `z_sb=32` 고정 단일 circulant 구조, 다중 lifting size 프레임 동시 처리 기능 없음 |
| barrel shifter 기반 circulant shift 연산 | `decoder.cpp` `LDPC_Decoder()`(row 단위 병렬 처리 내 shift 연산) | QC-LDPC circulant shift 자체는 공통 개념이나 Prime ECC는 고정 z 단일 구조라 FPCN의 가변 길이 대응 불필요 |
| 제어신호 오프라인 생성+ROM 저장 | `ecc_data.h` `PARAM_LLR`(유사한 오프라인 테이블 저장 패턴) | 설계 철학(오프라인 계산+런타임 조회)은 유사하나 대상 데이터가 다름(LLR 테이블 vs 라우팅 제어신호) |

적용 가치: 낮음 — 5G 다중 프레임 동시 병렬 라우팅이라는 목적 자체가 Prime ECC의 고정 z=32 단일 QC-LDPC 구조에는 불필요하며, 복호 알고리즘 개선도 아니어서 직접 이식 동기가 약함. 다만 MUX 절감형 라우팅 설계 아이디어는 GT 모듈과 비교 참고 가치 있음.
