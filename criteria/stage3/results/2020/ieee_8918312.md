### B. 알고리즘 요약

1. 시스템: binary cyclic code `C(n,k)`(예: BCH), AWGN 채널 + BPSK 변조. 기존 parity-check matrix `H`는 length-4 cycle이 많아 BP 복호에 부적합.
2. 문제: 기존 PTA(parity-check transformation algorithm)는 MDS 부호에서만 행렬 역변환(Gaussian-Jordan)이 보장되는데, non-MDS인 BCH 등 binary cyclic code에서는 역변환 불가(비가역 행렬) 시 복호 실패.
3. 핵심 가정: 채널 reliability(Euclidean distance 기반 정규화 APP 행렬 `β`)를 이용해 매 iteration마다 `H`의 열을 permutation만으로 재정렬, 역행렬 계산 단계를 완전히 제거.
4. 핵심 기법: reliability 내림차순 index `I`로 `H`를 열순열하여 `H^ι` 생성 후 행 연산으로 systematic form `H_s=[I_(n-k)|P]`, 다시 `I`로 재정렬해 최종 변환행렬 `Ĥ` 도출(식 9~13).
5. 핵심 식: 갱신은 `β` 값에 damping factor `δ`를 가감(`p_j=0`이면 `+δ`, 아니면 `-δ`)하는 방식으로 ABP의 sum-product extrinsic 계산(`L_ext(c_i)=2·tanh⁻¹(Π tanh(L(c_p)/2))`, 식 14)을 대체해 연산량을 줄임.
6. 확장: 정지조건은 모든 parity check `p_j=b̄Ĥ^T=0`을 만족하거나 최대 iteration `N` 도달(Algorithm 1).
7. 부수 트릭: damping factor `δ=0.1` 선택(시뮬로 iteration수·BER 트레이드오프 확인, δ=0.5는 최소 iteration이나 최악 BER).
8. 최적화 절차: 없음(학습 기반 아님), δ값은 실험적으로 고정.
9. 결과: BCH(31,16) R=0.5161에서 ML 대비 0.5dB 이내(BER 10⁻³), PTA 대비 2.7dB, ABP(20,1) 대비 1.7dB 이득(BER 10⁻⁴); BCH(255,231) 고rate(R=0.906)에서도 mRRD/PBP와 동등 성능을 더 적은 iteration/복잡도로 달성.
10. 한계: LDPC가 아닌 대수적 cyclic code(BCH) 전용 기법이며 sparse H 구조·Tanner graph 기반 BP와 무관. HW 미설계, 시뮬(AWGN, BPSK)뿐이며 Big-O 복잡도 분석만 있고 gatecount/throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Iteration마다 parity-check matrix 열 permutation/재구성 | 대응 없음 | Prime ECC는 고정 QC-LDPC H-matrix(`PCM_b`)를 사용, iteration별 행렬 재구성 개념 자체가 없음 |
| Reliability 기반 damping factor 갱신(sum-product 대체) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` | Prime ECC는 이미 min-sum(§3)으로 sum-product `tanh`를 근사 완료, GPT의 갱신방식은 non-LDPC 전용이라 대응 안 됨 |
| BCH 부호 구조/생성다항식 기반 parity-check | 대응 없음 | Prime ECC는 binary QC-LDPC 전용(§5 필터: non-LDPC 기법은 이식성 낮음) |
| Worst-case 시간복잡도(Big-O) 분석 | 대응 없음 | HW 클록레벨 복잡도(gatecount)와 무관한 알고리즘적 분석 |

적용 가치: **낮음** — LDPC가 아닌 binary cyclic code(BCH) 전용 반복 SD 복호 기법으로, Prime ECC의 QC-LDPC 고정 H-matrix·min-sum 구조와 부호·복호 패러다임이 근본적으로 달라 이식할 요소가 없다.
