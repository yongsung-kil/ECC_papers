### B. 알고리즘 요약

1. 시스템: 고성능 SSD의 다중 NAND 채널 구조. 각 채널을 큐(queue)로 모델링한 fork-join queuing 시스템, 페이지는 inner LDPC(1KB, rate 0.8947, PEG 생성)로 보호, 그 위에 `(n,k)` MDS outer RS 부호(layered coding)를 추가.
2. 문제: progressive sensing/soft-decision LDPC 복호는 여러 번의 재감지(re-sensing)로 read latency가 크게 증가하며, 페이지/블록/다이 단위 NAND element failure는 inner ECC로 정정 불가능해 별도 대책이 필요.
3. 채널/failure 모델: (a) read request failure(=LDPC 복호 실패로 재감지 유발, 감지 레벨 `i`마다 `t_i = t_sen-ref + t_xfer + t_dec + (i-1)(t_sen+t_xfer+t_dec)`), (b) NAND element failure(블록/다이/칩 단위 하드 실패, 별도 재구성 필요).
4. 핵심 기법: `(n,k)` MDS-outer-coded layered coding + fork-join 큐 모델을 확장해 read failure와 NAND element failure를 동시에 반영, split-merge 열화모델로 평균 read access time 상한(Theorem 5/9/11)을 유도.
5. 핵심 식: read access time 분포 `p_j=Pr[X=x_j]`는 각 감지 레벨의 성공확률 `P_fail,i`과 균등분포 dispersion 계수 `a`로 정의되며, `k`th order statistic 기반 M/G/1 큐의 Pollaczek-Khinchin 공식으로 평균 latency 상한을 도출 — 코딩 오버헤드(`n-k`)와 latency 트레이드오프를 정량화.
6. 확장: NAND element failure 대응 정책 2종 — Instantaneous Repair(IR, preemptive-resume 우선순위, 실패 즉시 복구)와 Postponed Repair(PR, non-preemptive, 읽기 큐가 빌 때까지 복구 지연).
7. 부수 디테일: 실측 파라미터(25nm MLC NAND, `t_sen-ref=t_sen=96μs`, `t_xfer=5μs`, `t_dec=8μs`, `t_prog=785.5μs`)를 사용한 정규화 latency 시뮬레이션.
8. 최적화 절차 없음(해석적 상한 유도 + 수치 검증), LDPC 자체는 PEG 알고리즘으로 생성된 고정 코드로 파라미터 튜닝 대상이 아님.
9. 결과: outer RS parity node 1개 이상 추가 시 RBER<0.006 영역에서 read access time 최대 약 70% 감소, NAND element failure 존재 시(IR 정책) RBER 0.006~0.008에서 약 45%(Case I)/40%(Case II) 감소. PR 정책은 추가 노드 4개 이상일 때 IR보다 우수.
10. 한계: LDPC 코드는 고정 스펙(1KB, rate 0.8947)의 inner ECC로 취급되며 본 논문은 이를 개선하지 않음. 순수 해석적/시뮬레이션 연구로 HW 구현이나 실측 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| outer RS erasure code(layered coding) | 대응 없음 | Prime ECC는 단일 inner LDPC ECC 엔진 시뮬레이터, outer erasure code 계층 없음 |
| queuing 기반 read access time 상한 분석 | 대응 없음 | 시스템/컨트롤러 레벨 latency 모델링, 코드/디코더 시뮬레이터와 무관 |
| progressive sensing/재시도 정책(IR/PR) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 소프트 리드 재시도 개념은 유사하나 본 논문은 재시도 발생 후 상위 정책(복구 스케줄링)만 다룸, 채널 파라미터 자체는 변경 없음 |
| LDPC inner code (PEG, rate 0.8947) | `ecc_top.cpp` `Load_PCM()` | 논문에서 고정 파라미터로만 사용, 부호 개선 기여 없음 |

적용 가치: 낮음 — LDPC 부호설계·디코더 알고리즘·HW를 다루지 않고 SSD 시스템 레벨의 outer erasure coding과 큐잉 지연 분석에 집중하여 Prime ECC(단일 LDPC 엔진) 모듈과 직접 대응되는 요소가 거의 없음.
