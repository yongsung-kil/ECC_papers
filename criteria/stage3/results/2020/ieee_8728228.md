> 총평: NAND soft-decision LDPC의 LLR 코드북을 채널 사전지식 없이 syndrome weight 기반으로 blind 추정하는 기법으로, decoder 자체보다는 channel/LLR 테이블 레이어에 이식 여지가 있으나 알고리즘·검증 모두 이론/시뮬 수준.

### B. 알고리즘 요약

1. NAND MLC/SLC flash, 논리 페이지별로 개별 LDPC 부호화(로직 페이지 분리로 읽기 latency 절감), 실험 부호는 `(9216, 8192)` regular LDPC, girth 6, rate≈0.889.
2. 문제: 채널(온도, ICI, retention 등)이 시간에 따라 변해 사전계산된 LLR LUT가 무효화되므로 LLR을 주기적으로 재추정해야 함.
3. 채널 모델: `Vth = V + W + I + R (+P)`로 프로그래밍 부정확성 P, ICI I, RTN/retention noise R을 각각 uniform/proportional/Laplace+Gaussian으로 모델링.
4. 핵심 기법 1단: hard-decision boundary 추정을 위해 여러 기준전압으로 읽고 syndrome weight `w(s)`로 raw BER을 추정 `1-p̂e = (1-2w(s)/Nc)^(1/dc)`, BER 최소화 지점을 `t*`로 선택 (Algorithm 1).
5. 핵심 식 의미: syndrome weight만으로 채널 통계 없이 BER을 blind 추정 가능 (LDPC parity 제약을 채널 추정기로 재활용).
6. 핵심 기법 2단: soft-decision boundary는 mutual information `I(X;Y)=1-Σhb(εj)P(Y∈Bj)`를 최대화하도록 추정, DMC 모델의 quaternary output으로 근사.
7. 부수 트릭: 추정 LLR magnitude를 `LLRmax`로 clip해 소수 오류 bin에서의 과도한 신뢰도 방지, 최외곽 bin(B1,B4)은 항상 `LLRmax` 부여.
8. 반복 정밀화: 디코딩 실패 시 불확실 영역을 세분화하여 재추정(B2∪B3 → B2,B3 분리) 하는 방식으로 점진적 정밀도 상승.
9. 결과: 제안 기법이 기존 blind 적응 기법[8]과 유사 PER 성능을 더 낮은 계산복잡도로 달성(O(2^m·L·Nc) vs O(Nc·Nv)).
10. 한계: HW 미설계, 시뮬레이션 결과만 제시(FPGA/ASIC 없음), regular LDPC 한 가지 코드에 대해서만 검증, 채널 모델은 특정 논문[6] 파라미터에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| blind LLR/threshold 추정 알고리즘 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold/LLR 테이블 생성 로직과 같은 레이어이나, syndrome 기반 blind 추정은 현재 미구현 |
| iteration별 LLR 재추정 | `ecc_data.h` `PARAM_LLR` | LLR 테이블 구조 자체가 아니라 값을 채우는 별도 채널추정 모듈이 필요, 대응 없음에 가까움 |
| syndrome weight 기반 BER 추정 | `decoder.cpp` `LDPC_Decoder()` | 디코더 내부에서 syndrome은 계산되나 이를 채널 추정에 재사용하는 로직은 대응 없음 |

> 적용 가치: 낮음 — Prime ECC 3.1은 이미 iteration별 LLR threshold 테이블(§3 적응형 LLR)을 보유하고 있으며, 이 논문은 그 테이블 값을 채널 사전지식 없이 blind로 채우는 상위 레벨 채널추정 기법이라 디코더/HW 코어와 직접 겹치지 않고 이식 시 별도 모듈 신설이 필요.
