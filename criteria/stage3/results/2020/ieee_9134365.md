### B. 알고리즘 요약

1. 대상은 802.11n 표준 구조(`Hb = [Hb1 Hb2]`, `dc_max = dc_min+1`, `dv_min=2`)를 따르는 irregular QC-LDPC, WiFi N=648 코드에 적용.
2. 기존 PEG/PEG-ACE는 girth 최적화 위주라 error-floor에 큰 영향을 못 주므로, cycle의 Extrinsic Message Degree(EMD, `γ = Σ(dv_i - 2)`)를 최대화해 iterative BP decoding의 잡음 내성을 높이려 함.
3. 핵심 가정은 낮은 EMD를 갖는 짧은 cycle(4/6/8/10)이 error floor의 주요 원인이라는 selective cycle conditioning 관점.
4. 핵심 기법은 열 단위 forward-backward PEG-like 탐색: 각 column의 원소별로 Fossorier의 cycle 존재 조건식 `Σ(pik,jk − pik+1,jk) = 0 mod z`을 이용해 shift 값 후보를 예측.
5. 길이 8/10 cycle은 base matrix 내 단독 cycle이 아니라 두 개의 짧은 cycle(4-4 또는 4-6) 쌍으로 구성되는 "route"로 분해해, route가 lifted matrix에서 실제 cycle이 되는 조건과 그 EMD를 짧은 cycle들의 base 방정식으로부터 계산.
6. route 조건은 예측이 어려워 guess-and-try 방식(가능한 shift 값 각각에 대해 route 조건 재확인)을 결합, 사후(a posteriori) 테스트 없이 원하는 girth/EMD 특성의 행렬을 직접 구성.
7. 열 내 비영원소 배치는 row/column weight 분포 균형을 맞추도록 배치하며, 탐색은 1시간 제한 내에서 짧은 cycle부터 순차적으로 EMD 최대화.
8. 별도 학습 절차는 없음(탐색 기반 구성, 시간제한 내 만족해가 없으면 이전 열로 백트래킹).
9. 결과: WiFi N=648, rate 1/2·2/3·3/4·5/6 코드에서 최소 EMD를 표준 대비 대폭 향상(예: R=2/3, length-6 cycle EMD 3→9), BER 시뮬(BP+LUT box-plus, max iteration 50)에서 R=1/2를 제외한 코드가 고SNR(error-floor) 영역에서 표준 WiFi 코드 대비 우위.
10. 한계: R=1/2에서는 성능 개선 없음(본문 명시), HW 구현·gatecount·throughput 데이터 전무, N=648 단일 length에서만 검증, iteration 횟수 감소 여부는 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EMD 기반 irregular QC-LDPC base matrix 구성 | `ecc_top.cpp` `Load_PCM()` | 새 H-matrix를 오프라인 생성 후 로드하는 지점만 대응, 구성 알고리즘 자체는 §4 기준 "부호설계=높음 난이도"로 이식 어려움 |
| cycle/route EMD 계산 및 최대화 | 대응 없음 | Prime ECC 코드베이스에 오프라인 부호 설계용 EMD 분석 모듈 없음 |
| BER 시뮬레이션(BP+LUT box-plus) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 논문의 box-plus(Sum-Product 계열) LUT 근사와는 알고리즘이 다름, 참고 수준 |

> 적용 가치: 낮음 — error-floor 개선이라는 목적은 Prime ECC 관심사(§5)와 부합하지만, 특정 QC-LDPC 구조 고정 이식이라 재검증 부담이 크고 HW/throughput 데이터가 전무해 즉시 적용 근거가 부족함.
