### B. 알고리즘 요약

1. Rate-compatible ECC는 동일 정보차원 `k`에서 길이가 다른 `M`개 코드 `C1≺C2≺...≺CM`(`N1<N2<...<NM`)로, 상위 rate 코드의 심볼이 하위 rate 코드에 포함되는 구조(`E_Ci(u) = π[Ni](E_Ci+1(u))`); NAND flash P/E 사이클 증가에 따른 RBER 상승에 맞춰 ECC 강도를 단계적으로 강화하는 데 사용.
2. 기존 puncturing 방식(저rate 기반 코드에서 패리티 일부 폐기)은 디코더가 항상 최저rate 코드의 전체 Tanner graph 크기로 동작해야 하는 비효율이 있어, 본 논문은 extending 방식(고rate 코드에 패리티 추가)을 채택.
3. 핵심 아이디어: 선형부호 `C`를 부분부호 `C'`의 coset들로 분할하면 각 coset은 `C`보다 강한 정정능력을 가지며, codeword의 coset 식별자(=syndrome)를 다음 레벨의 부가 부호(auxiliary code)로 보호해 계층적으로 결합(`Construction 1`).
4. Construction 1: nested 부호 `C1^M ⊂ ... ⊂ C1^1 = C1`와 각 레벨의 nested auxiliary 부호 `A_i^j`를 이용, syndrome `s_i = c1 H_{C1^i|C1^{i-1}}^T`를 다음 레벨에서 재귀적으로 부호화(`a_i^i = E_{A_i^i}(s_i, Λ_i^2,...,Λ_i^{i-1})`).
5. 핵심 정리(Theorem 5): 구성된 코드 `Ci`의 최소거리는 component 부호 `di`와 auxiliary 부호 `δ_i^j ≥ d_j - d_{i-1}` 조건 하에 `Di = di`로 결정 — coset 구조가 정정능력 보증의 핵심.
6. 복호(Algorithm 1)는 최저rate 코드부터 역순으로 진행하며, 각 단계 coset decoder가 성공하면 올바른 syndrome을 얻어 상위 레벨을 복호; 오류/소거 정정 조건은 `2t_i+τ_i ≤ δ_i^M-1`(식 6).
7. 구성이 rate `R_i < C(W_i)`인 nested capacity-achieving component 부호(예: polar code)를 쓰면 전체 계열이 QMS 채널족에 대해 capacity-achieving함을 증명(Theorem 12), 기존 HHM polar rate-compatible 구성과 동치(Theorem 13).
8. 별도 학습/최적화 없음 — 대수적 구성(BCH/RS/LDPC/polar 어떤 선형부호도 component로 사용 가능)이며 Construction 2/3는 auxiliary 부호의 nested 제약을 없애 인코딩/디코딩 복잡도를 낮춘 변형.
9. 결과: MLC flash 실측 오류 데이터 기반 평가에서 2-level BCH(rate 0.94→0.90)는 P/E 수명을 lower page 약 3500회, upper page 약 2000회 연장; 2-level LDPC(rate 0.94→0.90, SPA 100회 반복)는 lower page 약 4000회, upper page 약 3000회 연장.
10. 한계: 알고리즘/HW 설계 전무, LDPC 사용 예는 4-cycle-free RS 기반 regular LDPC(Prime ECC의 QC-LDPC와 무관)이며, MLC flash 성능평가도 실측 오류 위치를 재생한 소프트웨어 시뮬(SPA, floating-point)로 HW 친화적 검증(FPGA/ASIC) 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 다단 rate-compatible 부호 구성(coset/syndrome) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 단일 rate QC-LDPC 고정 구조로, 계층적 rate 확장을 이식하려면 auxiliary PCM 및 부가 인코더/디코더 계층 신규 추가 필요 |
| 계층별 coset 디코더(Algorithm 1) | `decoder.cpp` `LDPC_Decoder()` | 별도 디코더가 아니라 다단 순차 디코딩 오케스트레이션이라 현 반복 루프 구조와 상이, 대응 관계 약함 |
| Syndrome 계산/패리티 생성 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 부가 syndrome 부호화가 dual-diagonal 인코더 구조와 다른 별도 부호이므로 직접 재사용 어려움 |

적용 가치: `낮음` — NAND flash 수명 관점의 다단 rate-compatible 부호 이론(대수적 구성+실측 성능평가)이지만 Prime ECC 3.1의 단일 고정 rate QC-LDPC 아키텍처와 근본적으로 다른 시스템 레벨 설계이며, 디코더/인코더 모듈에 부분 이식하기엔 구조 변경 범위가 큼.
