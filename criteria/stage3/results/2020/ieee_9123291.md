> 총평: DVB-S2 TDMP 디코더의 메모리 뱅크/스케줄 최적화로 정정성능 변화 없이 throughput만 높인 순수 HW 구현 논문. binary QC-LDPC이나 부호rate/구조가 표준 종속적이라 이식성은 중간 수준.

### B. 알고리즘 요약

1. DVB-S2 QC-LDPC(eIRA 구성, 360-circulant), 짧은 프레임 n=16200, rate 2/3(k=10800)/4/5 대상, normalized Min-Sum + TDMP(layered) 스케줄링 사용.
2. 기존 TDMP 디코더는 체크노드 업데이트 시 메모리에서 `dc`(행 가중치)개 데이터를 순차 읽고 쓰는데, 파이프라인화하면 인접 업데이트 간 메모리 뱅크 충돌이 발생해 최신 변수노드 메시지를 못 쓰고 성능이 저하됨.
3. 정보비트 메모리 유닛 할당을 Graph Coloring Problem(GCP)으로 정식화: 충돌하는 정보비트 쌍을 간선으로 하는 그래프에서 최소 색수 `k`(=뱅크 수)로 채색.
4. 15-parallel 그룹화 후 720개 정점을 두 서브그래프(각 360개, 간선 360*10 / 360*3)로 분리해 유전 알고리즘(룰렛 선택, PMX crossover, 랜덤 구간 mutation)으로 GCP를 근사 해결.
5. 체크노드 업데이트 순서는 Traveling Salesman Problem(TSP)으로 정식화: 거리 `d(i,j)=|V(ci)∩V(cj)|`(공유 변수노드 수)를 최소화하는 순열을 탐색해 ε 간격 충돌 제거.
6. H행렬의 순환 대칭성(`d(15n+i,15n+j)=d(15(n+1)+i,15(n+1)+j)`)을 이용해 그룹별 대표 정점 하나만 탐색 후 휴리스틱+백트래킹으로 나머지 순서를 결정, 남는 소수 충돌은 파이프라인 버블 삽입으로 흡수.
7. 최적화된 뱅크 배치(LQM 3그룹, LQP 2그룹) 테이블을 사전 계산해 저장, 런타임엔 테이블 참조만으로 매 클록 다른 뱅크에서 1클록 read/write 가능.
8. 유전 알고리즘/휴리스틱 탐색은 오프라인 전처리로만 수행되며 하드웨어 디코딩 알고리즘 자체(Min-Sum 수식)는 변경 없음.
9. Stratix V FPGA(182MHz) 구현 결과 rate 2/3에서 2.05Gbps, rate 4/5에서 2.73Gbps, 총 메모리 1.93Mb로 기존 문헌 대비 throughput 대비 메모리 효율 개선, BER은 이론 대비 약 1dB 손실(성능 저하 없음 주장).
10. 정정 성능 개선은 없음(무손실 재현이 목표), 코드rate별 개별 설계가 필요해 all-rate 통합 디코더는 미해결 과제로 남김, FPGA 검증만 수행.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 메모리 뱅크 할당 최적화(GCP) | 대응 없음 | Prime ECC는 z_sb=32 고정 lifting 기반 병렬 구조이며 오프라인 GCP식 뱅크 할당 최적화 로직은 프로파일에 명시 없음 |
| 체크노드 업데이트 순서 스케줄링(TSP) | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 MUX 기반 edge 선택과 유사한 목적(HW 접근 충돌/edge 처리 축소)이나 구현 방식은 상이 |
| TDMP/layered 스케줄 자체 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 Dual-Update 옵션으로 유사 스케줄링 보유 |
| 파이프라인 버블/타이밍 조정 | `decoder.cpp` pipeline 6-stage 구조 | 이미 bit-exact pipeline 모델 존재, 세부 버블 삽입 로직은 대응 없음 |

적용 가치: 낮음 — DVB-S2 표준 특유의 360-circulant/rate 구조 종속적 메모리 뱅크 최적화이며 정정성능 개선이 없는 순수 HW 자원/타이밍 최적화라 Prime ECC 이식 우선순위는 낮음.
