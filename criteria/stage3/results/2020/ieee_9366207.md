### B. 알고리즘 요약

1. 대상 코드: `RM(r,m)` 이진 Reed-Muller 코드(길이 `n=2^m`, 차원 `K=Σ(m choose i), i=0..r`), 시뮬레이션은 RM(1,5), RM(1,6), RM(2,7) 사용(길이 32/64/128).
2. 문제: 기존 majority-logic 디코딩은 hard-decision 방식이지만 성능이 제한적이고, 기존 soft-decision 디코더들은 복잡도가 높음 - 저복잡도 hard-decision 개선이 목표.
3. 채널 모델: AWGN 채널의 soft 수신 시퀀스 `y`를 hard-decision `v`로 변환 후 처리(hard decoding 관점에서는 BSC로 간주).
4. 핵심 기법: conventional majority decoding의 이진 덧셈(vote 합산)을 LDPC min-sum 연산으로 대체 - `û_{j,k} = sgn(φ_l) × min|φ_l|` (l∈S_k)로 각 정보비트의 vote 신뢰도 계산.
5. 핵심 식: reliability measure `R_j = Σ_k û_{j,k}` (모든 vote의 합) - 다수결(majority) 대신 신뢰도 크기로 정보비트 판정, degree γ→γ-1로 내려가며 재귀적으로 φ_l 갱신.
6. 확장(2단): 매 iteration마다 재구성된 codeword와 초기 hard-decision 시퀀스 간 차이 `D_l`을 계산해 가장 큰 차이를 갖는 위치 `l*`를 한 비트만 flip(iterative bit-flipping), 이를 반복.
7. 부수 트릭: 조기 종료(termination) - 매 iteration마다 갱신된 시퀀스가 유효 codeword가 되면(`D_l=0` for all l) 즉시 종료해 복잡도 절감.
8. 학습/최적화 절차: 없음(고정 알고리즘, 반복 횟수만 `I_max=100`로 설정).
9. 결과: majority decoding 대비 BLER 10⁻⁶(RM(1,5))에서 0.34dB, 10⁻⁵(RM(1,6))에서 0.45dB, 10⁻⁴(RM(2,7))에서 0.55dB 이득; 복잡도는 majority decoding 대비 `I_max`배.
10. 한계: HW 구현 없음, RM 코드 특화 구조(votes/degree 계층)로 일반 LDPC Tanner graph에 직접 적용 불가, 순수 시뮬레이션 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-sum 기반 reliability measure(vote 신뢰도 계산) | `decoder.cpp` `CNU_Update_New_Mag()` | Min-sum의 min 연산 아이디어는 유사하나 RM 코드의 vote 구조 전용이라 QC-LDPC C2V 갱신에 직접 대응 안 됨 |
| Iterative bit-flipping(1비트씩 flip) | 대응 없음 | Prime ECC는 min-sum message-passing 디코더이며 단일비트 flip 방식의 BF 알고리즘이 프로파일에 없음 |
| Hard-decision(BSC) 디코딩 | `decoder.cpp` `VN_Cal_HD()` | Prime ECC도 HD 모드를 지원하나 본 논문은 RM 코드 전용 majority-vote 구조라 이식 난이도 높음 |
| 조기 종료(유효 codeword 시 stop) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 유사하나 본 논문은 CRC가 아닌 codeword 재구성 비교 방식으로 상이 |

적용 가치: `낮음` — Reed-Muller 코드 전용 majority-logic 구조에 기반한 디코더로, QC-LDPC 기반 Prime ECC의 Tanner graph 디코딩 구조와 근본적으로 다르며 이식 시 전면 재설계가 필요함.
