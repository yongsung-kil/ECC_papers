### B. 알고리즘 요약

1. 대상 시스템은 단거리(short-reach) 광전송으로 802.3, OIF 400ZR, ITU 50G PON 등 표준용 FEC를 FPGA 기반으로 비교·검증한다.
2. 목표 BER은 10^-15 수준까지 요구되어 FPGA 에뮬레이션으로 저BER 영역 성능을 확인해야 한다는 문제의식.
3. 비교 대상 코드는 6.7% 오버헤드 staircase code, 14.8% 오버헤드 CSHC(staircase+extended Hamming(128,119) SD 디코딩), QC-LDPC(HD, shortening/puncturing 적용), 25% LDPC-CC(+outer BCH).
4. staircase code는 510×512-bit 서브블록(Bi) 단위 feed-forward 인코딩, unidirectional/bidirectional 반복복호 옵션을 갖고 디코딩 버퍼 크기(Bi 블록 개수)가 error floor에 영향.
5. 결과: bidirectional 복호는 error floor가 심하고, unidirectional 복호는 버퍼 7블록에서 10^-15 이상 BER에서 error floor 없음.
6. CSHC의 eHamming SD 복호는 고정소수점 스케일 팩터 최적화가 핵심이며, 최적값은 2.0으로 확인(정보손실 최소화).
7. QC-LDPC는 shortening/puncturing으로 오버헤드를 조정하며 4종 구성 모두 HD 복호에서 10^-13 BER까지 error floor 없이 waterfall 성능 확보.
8. 18.5% QC-LDPC에 SD 복호를 적용하면 입력 BER 임계값이 1.1e-2에서 2.4e-2로 개선되고 10^-15까지 waterfall 유지.
9. 25% LDPC-CC는 FPGA 에뮬레이션에서 10^-12 부근 error flare가 관측되어 outer BCH 코드(오버헤드 26.3%로 증가)로 제거.
10. 한계: FPGA 에뮬레이션 기반 성능 비교에 그치고, gatecount/throughput/디코더 세부 알고리즘(min-sum 변형, 스케줄링 등)에 대한 수치·서술이 본문에 없음(그림에만 존재).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC HD 디코딩(shortening/puncturing) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 QC-LDPC HD 지원하나 본문에 알고리즘 세부 없어 참조 수준 낮음 |
| SD 복호 고정소수점 스케일 팩터 최적화(eHamming) | `channel.cpp` `Set_LLR_Th()` | eHamming/staircase 계열로 LDPC와 무관, 대응 약함 |
| staircase code / LDPC-CC | 대응 없음 | Prime ECC는 QC-LDPC 전용, staircase·convolutional 코드 미지원 |

> 적용 가치: 낮음 - 광통신 특화 FEC 개관 논문으로 NAND 이식에 직접 활용 가능한 알고리즘적 디테일이 부족.
