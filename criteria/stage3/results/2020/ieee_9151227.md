### B. 알고리즘 요약

1. 대상 문제: 희소 선형계 `Ax=y` (F=R 또는 GF(2))를 structured Gaussian elimination(SGE)으로 빠르게 푸는 것 (LDPC/LT/raptor 코드의 ML erasure decoding, 정수 인수분해 등에 응용).
2. 기존 방식 한계: SGE의 삼각화(triangulation) 단계에서 maximum component(MC, 가장 큰 연결 성분)를 찾기 위해 Tanner graph 전체를 DFS로 매번 탐색해야 해 계산 병목이 됨.
3. 핵심 모델: `A`는 `m x n` 희소행렬, `θ`는 비영 원소 수. degree-2 row(2개 column과만 연결)를 통해 column들이 "component"로 연결됨을 정의(Definition 1).
4. 핵심 기법: DFS 대신 disjoint-set data structure(DSDS, union-find)를 사용해 degree-2 row가 나올 때마다 해당 두 column에 `UNION(v,v')`을 적용, 각 rooted tree가 하나의 component에 대응하도록 유지.
5. 핵심 연산: `FIND(v)`(path-compression 포함)와 `UNION(u,v)`(union-by-rank 포함)로 `O(α(n)·nf)` 시간에 component 관리 (`α`는 역 Ackermann 함수, 사실상 상수).
6. 전체 삼각화 복잡도: DSDS 사용 시 `O(α(n)m + nγ + θ)`로 기존 DFS 기반 `O(θγ)` 대비 수 배 낮음 (`γ`는 삼각화 후 남는 dense 부분 크기).
7. 삼각화 이후 처리: `(2)`를 `I/0, R1/R2` 형태로 변환 후 `R2 x2 = y2`를 conventional GE로 풀고 back substitution — 전체 `O(θγ + θl + γ^3 + γ^2 l)`.
8. 구현 디테일: bit-string 32비트를 정수 하나로 압축해 XOR 연산 가속(C++ 구현), F=GF(2) 특화.
9. 결과: rate-0.5 irregular LDPC(m=5000,n=10000,l=3200, BEC)와 LT code(n=1024)에서 pe≥0.45 구간 등에서 DSDS가 DFS 대비 수 배(several times) 빠른 실행시간 확인(시뮬 그래프 기반, 정확한 배수는 본문 수치 미제시).
10. 한계: 대상이 BEC 상의 erasure decoding(선형대수 기반 ML decoding)이며, HW 설계·min-sum 반복복호와 무관. 순수 알고리즘/자료구조 논문(시뮬만, ASIC/FPGA 없음).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MC 기반 triangulation / SGE (erasure ML decoding) | 대응 없음 | Prime ECC는 min-sum BP 반복복호 기반이며 Gaussian elimination 기반 ML decoding을 사용하지 않음 |
| disjoint-set 자료구조(DSDS) | 대응 없음 | H-matrix 구조 관리·인코딩 어디에도 union-find 기반 component 탐색이 쓰이지 않음 |
| 부호 PCM 생성/구조 (LDPC/LT 코드 예시) | ecc_top.cpp Load_PCM() | 원문은 QC-LDPC PEG 알고리즘으로 생성된 PCM을 예시로만 사용, 부호설계 기법 자체는 아님 |
```

적용 가치: **낮음** — 알고리즘 대상(BEC 소거 채널, ML erasure decoding)과 최적화 대상(component 탐색 자료구조)이 Prime ECC의 min-sum 기반 binary QC-LDPC 반복복호 파이프라인과 근본적으로 다른 계열이라 이식 지점이 없음.
