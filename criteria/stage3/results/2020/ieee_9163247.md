### B. 알고리즘 요약

1. 대상: 5G PBRL(protograph-based raptor-like) LDPC 코드, `H=[[Hcore,Hex],[0,I]]` 구조, K∈[40,8448], R∈[1/5,8/9], AWGN 채널, QPSK.
2. 문제: 5G LDPC는 degree-1 VN이 많아(특히 저rate) 기존 MSA/NMSA가 SPA 대비 성능 저하가 크고(R=1/5에서 SPA 대비 최대 0.8dB), short code에서는 BP 자체가 ML 대비 성능 갭 존재.
3. 핵심 관찰: degree-1 VN이 보내는 C2V 메시지는 크기가 상한(`|L(vn̂)|`)으로 bound되어(식 7) 신뢰도가 낮고, 실패 복호 시 V2C 메시지의 부호 flip 비율이 saturate됨(Fig.1).
4. 핵심 기법1 (2D-SC MSA): Hcore에서 오는 V2C 메시지에 partial self-correction(PSC) 적용 — 현재/이전 iteration 부호가 다르면 0으로 소거(`Z_n^(i)=0 if sign flip`, 식 6), 동시에 Hex로 가는 V2C 메시지에 스케일링factor `β>1`로 증폭(식 8), C2V는 NMSA와 동일 `α`로 정규화(식 5).
5. 핵심 식 의미: PSC는 신뢰 낮은(부호 흔들리는) 메시지를 죽여 오류 전파를 막고, β 증폭은 degree-1 VN에 인접한 CN이 만드는 bound-limited C2V 메시지의 신뢰도를 다음 iteration에서 보정.
6. 핵심 기법2 (EQML): 초기 BP 실패 시 reprocessing 수행 — Edge-Wise Selection(EWS, 식 9: 각 edge의 부호flip 횟수 `w(v_n)=Σw_{k,n}` 최대인 VN을 saturate 대상으로 선택)으로 불신뢰 VN 선택, Partial Pruning Stopping(PPS)으로 유효 코드워드 발견된 sub-branch는 조기 종료.
7. 구현 디테일: LUT 불필요(SPA/A-Min*/Adj-MSA 대비), α=0.7125(NMSA), (α,β)=(0.75,1.25) 5G 코드 전역 공통 파라미터로 최적화; PSC 메모리 오버헤드는 `O(M·Ncore)`(H 전체 아닌 Hcore 크기에만 비례).
8. 최적화 절차: (α,β)는 2D-SC MSA와 SPA 간 FER의 MMSE를 최소화하도록 그리드서치.
9. 결과: 2D-SC MSA는 SPA 대비 0.3dB 이내(N=8448) FER 근접; EQML(jmax=6)은 PPV bound 대비 0.4dB 이내, avg iteration 15%↓·decoding resource 최대 25~81%↓(OSD 대비 jmax=4에서 19%만 필요).
10. 한계: 순수 알고리즘/시뮬(AWGN, floating-point) 검증뿐 HW 미설계, 5G PBRL 특유 Hcore/Hex 구조·degree-1 VN 다수 가정에 특화(NAND QC-LDPC는 일반적으로 degree-1 VN 구조가 다름), EQML worst-case 복잡도는 jmax에 지수적.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2D-SC MSA (PSC + message amplification) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 기존 min-sum 연산에 부호flip 기반 V2C 소거·증폭을 추가하는 형태로 CN/VN 업데이트 로직에 이식 가능 |
| iteration별 스케일링 factor (α, β) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 이미 iteration별 LLR threshold 테이블 보유 - α/β 파라미터 테이블 확장으로 적용 가능 |
| EWS 노드선택 (edge-wise sign flip 카운트) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 카운터 추가로 구현 가능하나 Prime ECC는 조기종료 CRC 기반이라 reprocessing 자체 구조는 신규 설계 필요 |
| PPS 정지규칙 / EQML reprocessing | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 조기종료 개념은 유사하나 EQML은 LLR saturation 재시도 방식으로 Partial/Full CRC와는 메커니즘이 다름 - 대응 없음에 가까움 |
| Hcore/Hex 2단 부호구조(PBRL) | ecc_top.cpp Load_PCM() | Prime ECC의 고정 QC-LDPC PCM_b와 구조가 다른 5G 특화 rate-compatible 구조 - 이식 난이도 높음 |
```

적용 가치: **높음** — 2D-SC MSA의 self-correction+메시지 증폭은 기존 min-sum 파이프라인(decoder.cpp)에 큰 구조 변경 없이 이식 가능하고, waterfall 영역 정정능력·avg iteration 개선이 정량적으로 검증되어 Prime ECC 성능 개선에 직접 참고 가치가 있음.
