### B. 알고리즘 요약

1. TI-SC-LDPC-CC(Time-Invariant Spatially-Coupled LDPC Convolutional Code)는 `m × n` 지수행렬 `B`(base matrix + syndrome former memory order `mh`)로 정의되며, 대상은 single-edge protograph 기반 부호(QC-LDPC 포함).
2. 기존 PEG류 탐색 기반 방법은 파리티체크행렬 위에서 직접 탐색해야 하므로 복잡했으나, 본 논문은 그래프 이론적 접근으로 elementary trapping set(ETS)을 특징짓는 edge-coloring technique(ECT)을 제안.
3. Tanner graph의 ETS는 degree 1 또는 2 check-node만 갖는 `(a,b)` trapping set으로 정의되며, 이를 degree-2 check-node를 edge로 갖는 "VN graph"로 축약해 분석.
4. 핵심 정리: VN graph에 ETS가 존재하려면 `m`-edge-coloring이 가능해야 함(Vizing's theorem `Δ(G) ≤ X ≤ Δ(G)+1` 활용). 특정 색상 조합의 삼각형(triangle)·k-cycle을 지수행렬 `B`의 특정 행(row) 조합에서 회피하면 해당 ETS가 TG에서 제거됨.
5. 식 (1) `Σ ±(b_{m_i n_i} - b_{m_{i+1} n_{i+1}}) = 0`은 TG의 2k-cycle이 지수행렬 `B`의 어떤 행 조합에서 발생하는지를 규정 — 이 값이 반복되면 해당 cycle이 존재.
6. Theorem 3~7로 column weight `m=3,4,5`, girth `g=6,8`인 경우 각각 회피해야 할 특정 6-cycle/8-cycle 조합(예: `6-cycle{1,2,3}`)을 제시해 최소 ETS 크기를 `m+1`에서 `m+2`로 증가시킴.
7. 부수 기법: 8-cycle 회피를 위해 `B`의 2×r(r=2~4) 부분행렬에서 식(1) 우변 절댓값 집합 `A(i1,i2)`의 중복/교집합 여부만 검사하면 되므로 탐색 없이 대수적 조건으로 환원.
8. 별도의 학습/최적화 절차 없음 — 순수 정리 증명 기반 구성법이며 Table I에 최소 `mh`, `N`을 만족하는 지수행렬 예시 제공.
9. 결과: `(3,6)`-regular QC-LDPC(N=31)와 `(3,9)`-regular QC-LDPC(N=85) 비교 시뮬(sum-product, 50회 반복)에서 제안 코드 C2가 기존 최량 코드 C1과 대등~근소 우세, C4(g=6, ETS-free)가 C3(g=8)를 크게 능가.
10. 한계: 이론적 구성법으로 디코더/HW 설계 전무, column weight 3~5·girth 6/8·특정 single-edge protograph 구조에 한정되며 rate/length 등 실용 NAND 부호 스펙과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 지수행렬/H-matrix 구성(ETS-free 조건) | `ecc_top.cpp` `Load_PCM()` | Prime ECC의 고정 QC-LDPC PCM 구조를 새 구성법으로 대체하려면 재검증 필요 |
| Elementary trapping set 회피(error-floor 대책) | `decoder.cpp` `LDPC_Decoder()` (간접) | 부호 자체의 ETS 제거이므로 디코더 알고리즘과는 별개, 부호설계 단계에서만 유효 |
| SC-LDPC convolutional 구조 | 대응 없음 | Prime ECC는 QC-LDPC block code이며 SC-LDPC-CC(convolutional) 구조 자체는 미지원 |

적용 가치: `낮음` — SC-LDPC convolutional code의 이론적 trapping-set 회피 구성법으로, Prime ECC 3.1의 고정 QC-LDPC block 구조·디코더 알고리즘과 직접 접점이 없고 부호설계 재검증 부담이 큼.
