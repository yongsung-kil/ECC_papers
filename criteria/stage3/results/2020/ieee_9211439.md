### B. 알고리즘 요약

1. 대상은 8/9 code rate, 4kB message 단위 LDPC를 사용하는 3-D CT NAND TLC SSD. sensing level 수(1=hard, ≥2=soft)로 LLR 정밀도가 결정된다.
2. 문제: 3-D NAND는 layer-to-layer 공정편차로 하위층 rBER이 상위층 대비 최대 21배 높아, 하위층 페이지 디코딩에 다수 sensing level(최대 7단)이 필요해 read 지연이 커진다.
3. 채널/노이즈 모델: retention loss와 layer 위치·cell-bit significance(MSB/CSB/LSB)에 따른 비대칭 rBER (Vth shift 기반).
4. 핵심 기법: "error diluting" — 상위층(강한) 페이지와 하위층(약한) 페이지 각각을 절반(subpage)으로 나눠 `virtual page`로 재조합(`X0+Y0`, `X1+Y1`)한 뒤 하나의 LDPC 코드워드로 인코딩/디코딩.
5. 강한 페이지의 고신뢰 LLR이 약한 페이지의 평균 rBER을 낮춰, 약한 페이지 쪽 sensing level을 기존 7단에서 3단까지 줄일 수 있음(Table I).
6. 확장: layer 편차뿐 아니라 page-type(MSB/CSB/LSB) 편차까지 고려한 "variation-aware subpage striping"(하위층 MSB ↔ 상위층 LSB 페어링)으로 효과 극대화.
7. 구현 디테일: 두 물리 페이지 간 4개 subpage 인터리빙, cache reading(순차 접근용 512-page LRU 캐시)과 snap reading(partial-page read로 소용량 요청 처리)을 병행.
8. 최적화 절차 없음(경험적 페어링 규칙 + retry table로 sensing level 점증 재시도).
9. 결과: read throughput 29% 향상, read latency 43% 감소(Baseline 대비), sensing level 총량 평균 24%(최대 40%) 절감.
10. 한계: LDPC 알고리즘/부호 자체는 미변경(블랙박스), FTL 소프트웨어 정책 논문으로 HW 미설계, DiskSim 기반 시뮬레이션만 수행.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 가상 페이지 인터리빙(subpage striping) | 대응 없음 | FTL/컨트롤러 데이터 배치 정책으로 decoder.cpp/encoder.cpp 범위 밖 |
| sensing level 결정/LLR 재사용 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold 개념은 유사하나 논문은 페이지 페어링으로 강도를 낮추는 시스템 정책, 코드 자체 변경 없음 |
| retry table(점증 재시도) | 대응 없음 | 디코더 내부 알고리즘이 아닌 FTL read retry 정책 |
| LDPC 디코딩 자체 | `decoder.cpp` `LDPC_Decoder()` | 논문은 기성 LDPC 모듈을 블랙박스로 사용, 알고리즘 변경 없음 |

적용 가치: **낮음** — 이 논문은 LDPC 부호설계/디코더 알고리즘을 개선하지 않고 SSD FTL의 페이지 배치·read 캐싱 정책만 다루므로 Prime ECC(디코더/인코더 코어 시뮬레이터)에 직접 이식할 요소가 없음.
