> 총평: 방송(DTTB) 채널 다양성에 대한 universal 성능을 노린 spatially-coupled QC-LDPC 구성법으로, NAND 고전 binary LDPC와는 채널·목적이 달라 이식성은 낮고, 부호구성(H-matrix) 레이어 이식은 재검증 부담이 큼.

### B. 알고리즘 요약

1. DTTB(디지털 지상파 방송) 시스템 대상, QC-SC-LDPC 부호를 FEC로 제안. 예시 부호는 uncoupled rate `Rtrue=2/3`, base matrix `Bunc=[7 3 3]` (1×3), coupling length `L=40`, smoothing parameter `w=6`, `nunc=2304`.
2. 문제: 기존 LDPC는 특정 변조/채널 시나리오에 맞춰 최적화되어 다른 시나리오(변조 차수, 채널조건, 수신기 타입 변경 시) 성능 저하 또는 복호 실패 발생.
3. 핵심 가정: coupling length `L`, smoothing `w`, `nunc`, iteration `Iiter`가 유한한 실용 제약 하에서도 threshold saturation(→MAP threshold 근접)을 유지해야 함. 채널은 BI-AWGN과 2-user Gaussian MAC(GMAC) 두 종류로 검증.
4. 핵심 기법 1단: base matrix 설계 시 동일한 uncoupled code를 반복 결합(edge spreading pattern `Bpattern=[B0^T,...,Bw^T]^T`)하여 시간불변 periodic 구조를 만들고, windowed decoder 구현을 단순화.
5. 핵심 식 의미: design rate `Rdesign = Rtrue - w*Munc/(L*Nunc)`로 rate loss가 정의되며, `wremain` row-removal로 loss를 `wremain*Munc/(L*Nunc)`까지 축소(터미네이션에 따른 불가피한 손실 완화).
6. 핵심 기법 2단: two-step lifting(pre-lifting: 소규모 일반 permutation으로 설계 자유도 확보 + 길이-4 cycle 제거, post-lifting: circulant permutation + ACE 기반 short cycle 제거)로 QC 구조와 유한장 성능을 동시 확보.
7. 부수 트릭: 최소 variable-node degree를 3으로 유지해 linear minimum distance growth property를 보장(error-floor 개선), post-lifting에서 모든 length-4 cycle 제거.
8. 최적화 절차: BI-AWGN(density evolution+RCA)과 GMAC(DE-EXIT)에서 threshold gap을 계산해 edge spreading pattern을 탐색, 두 채널 모두에서 threshold gap이 작은 패턴만 채택.
9. 결과: 구성된 부호가 BI-AWGN에서 Shannon limit 대비 0.07dB, 2-user GMAC에서 0.08dB 이내 threshold 달성. ATSC 3.0 rate-2/3 코드(scheme A) 대비 FER=10⁻³ 기준 유사~우수하나, decoding iteration은 windowed decoding 사용해도 비트당 100회로 scheme A/B(50회) 대비 약 2배 복잡도.
10. 한계: 실제 HW(windowed decoder) 구현/합성 없이 시뮬레이션(BER/FER)만 제시, 방송채널(AWGN/Rayleigh/GMAC) 전용이라 NAND 채널 특성과 무관, iteration/complexity가 기존 대비 오히려 증가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-SC-LDPC base matrix / edge spreading 구성법 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조를 SC 형태로 바꾸는 것은 코드 전면 재설계 수준, 이식 난이도 높음 |
| two-step lifting (pre-lifting + circulant post-lifting) | `ecc_top.cpp` `Load_PCM()` | 현재 Prime ECC는 고정 QC-LDPC(base+circulant shift)이며 SC 확장을 위한 lifting 절차는 대응 없음 |
| windowed decoding (SC 특유 복호 스케줄) | `decoder.cpp` `LDPC_Decoder()` | 표준 flooding/dual-update 스케줄과 다른 구조로, 파이프라인 재설계 필요 - 대응 없음에 가까움 |
| ACE 기반 length-4/length-6 cycle 제거 | `ecc_top.cpp` `Load_PCM()` | H-matrix 생성 단계의 부호설계 기법으로 대응 없음 |

> 적용 가치: 낮음 — SC-LDPC는 Prime ECC의 고정 QC-LDPC(non-coupled) 구조와 근본적으로 다른 부호 클래스이며, 방송채널 특화 universal 설계 목적이 NAND의 rate/length 고정 요구와 맞지 않아 부호설계 레이어 이식 난이도가 매우 높음.
