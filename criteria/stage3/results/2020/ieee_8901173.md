### B. 알고리즘 요약

1. Protograph 기반 QC-LDPC 코드(fully-connected base graph의 cyclic N-lifting)를 대상으로 error-floor를 낮추는 부호 구성법을 제안.
2. 기존 PEG/ACE 등 구성법은 error-floor를 지배하는 leafless elementary trapping set(LETS)을 충분히 제거하지 못하거나 탐색이 비효율적이었음.
3. LETS는 `(a, b)` 클래스(변수노드 a개, 홀수차수 검사노드 b개)로 분류되며 QC 구조 자체가 특정 LETS 구조의 존재를 원천 배제함을 edge coloring으로 증명(`Gˆ(S)`의 chromatic index `χ(Gˆ(S)) > m`이면 그 ETS는 존재 불가).
4. 목표 LETS 집합 `L` 중 다른 구조의 자식이 아닌 최소 집합 `Lt`만 골라, dpl(dot/path/lollipop) 확장의 backward recursion으로 `Lt`를 생성하는 out-of-range 부모 구조만 탐색(Algorithm 1).
5. `Lt`의 최소성 덕분에 기존 [13] exhaustive dpl 탐색 대비 탐색 대상 구조 수와 필요한 확장 종류가 대폭 감소(예: r4 범위에서 `|Lt|=74` vs `|L|=392`).
6. 탐색을 크기 `a` 순으로 layer화하여, 앞 layer에서 목표 LETS 인스턴스가 하나라도 발견되면 즉시 종료(조기 판정)해 메모리·시간 추가 절감.
7. 부호 구성은 exponent matrix `P`의 열을 왼쪽부터 greedy random search로 채우며, 매 열마다 girth 제약과 `L` 내 LETS 부재를 layered 탐색으로 검증, 실패 시 backtrack.
8. 학습/최적화 절차 없음(무작위 greedy search + 백트래킹, 시간제한 3시간/lifting degree).
9. 결과: 3×5/3×6 base graph에서 기존 [10] 대비 동일 girth·trapping-set 제약 하 lifting degree N을 41→26, 63→41로 축소, 탐색 실행시간도 [13] exhaustive 대비 최대 한 자릿수 이상 단축, FER 곡선상 error-floor 영역에서 Tanner code/[9]/[29] 대비 우수.
10. 한계: 시뮬레이션(5-bit min-sum, AWGN)뿐이며 HW 설계 없음, fully-connected regular base graph에 한정, rate/length가 예제별로 상이해 단일 확정값 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC H-matrix(exponent matrix) 구성/trapping set 회피 설계 | `ecc_top.cpp` `Load_PCM()` | 논문의 부호 구성 산출물(exponent matrix)이 Prime ECC의 `PCM_b` 대응 위치에 로드되나, 특정 base graph·degree 고정이라 재설계·재검증 부담 큼 |
| LETS(trapping set) 탐색 알고리즘 자체 | 대응 없음 | Prime ECC는 오프라인 부호설계 도구를 포함하지 않음(디코더/인코더 시뮬레이터) |
| 5-bit min-sum 디코더(시뮬레이션용) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문은 기존 min-sum을 검증용으로만 사용, 알고리즘 개선 아님 |

> 적용 가치: 중간 | 부호설계(H-matrix) 층위 기법이라 이식 난이도는 높으나, 완전한 오프라인 도구로서 Prime ECC의 고rate QC-LDPC 재설계 시 error-floor 개선 참고자료로는 활용 가능.
