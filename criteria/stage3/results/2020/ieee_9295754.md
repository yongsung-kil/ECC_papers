### B. 알고리즘 요약

1. (N,K) 이진 LDPC, BPSK/AWGN, IEEE 802.16e 기반 base matrix를 24배/48배 확장해 rate 0.5/0.66, length 576/1152 코드로 시뮬레이션.
2. BP는 성능은 좋으나 `tanh`/`tanh⁻¹` 연산이 복잡하고, MS는 연산은 단순하나 `min(tanh(|x|))` 근사로 인한 성능 손실이 있어 두 알고리즘의 장점을 절충하려는 문제 제기.
3. 채널 모델은 AWGN, 채널 LLR은 `L(0)(qij) = 4yi/N0`로 초기화.
4. 매 반복 직전 변수노드 LLR 절댓값이 구간 `(0,a]`에 속하는 비율 `R = n1/n`을 계산하고, 임계값 `b`와 비교해 `R ≥ b`이면 BP 식(2)(3)(4), 아니면 MS 식(5)(3)(4)을 그 반복에 적용.
5. 핵심 근거: `tanh(|x|) ≤ min(tanh(|x|))`이며 LLR이 작은 노드 비율이 여전히 존재하는 구간에서 MS 근사 오차가 커지므로, 그 구간 비율이 높을 때만 정밀한 BP 연산을 쓴다.
6. 파라미터군 `(a,b)`는 rate에만 의존하고 code length와는 무관함을 실험적으로 확인(r=0.5: a/b=22.2, r=0.66: a/b=42 근방 관계식 제시).
7. 극한 케이스: `a=0,b=0`이면 순수 MS로, `a=∞,b=∀`이면 순수 BP로 수렴하는 일반화된 프레임워크.
8. 파라미터 튜닝은 Monte Carlo 방식으로 (a,b) 조합을 탐색해 BP/MS 서브반복 횟수가 고SNR에서 거의 같아지도록 최적화.
9. 결과: BER=3×10⁻⁵ 기준 BP 대비 평균 0.08dB 이내 성능 손실, MS 대비 약 0.2dB 개선; (576,288)/(1152,576) 코드에서 곱셈 연산 약53.3%, 나눗셈 연산 약51.1% 감소.
10. 한계: HW 구현/합성 없음, AWGN 시뮬레이션에 국한, 정정능력 자체는 BP 대비 개선이 아니라 동등 수준 유지에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP/MS 하이브리드 반복 선택 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 이미 Min-Sum 기반이며 full-tanh BP를 보유하지 않아 하이브리드 구조 이식 시 BP 경로 신규 구현 필요 |
| CN 갱신(BP식/MS식) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | MS 부분은 기존 보유 기법과 중복, BP 부분만 신규 |
| LLR 구간 비율 기반 threshold 판단 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 반복별 LLR 테이블 개념과 유사하나 본 논문은 알고리즘 선택 기준이라 구조적 대응은 부분적 |

> 적용 가치: 낮음 | Prime ECC는 이미 Min-Sum 기반 HW이며 full-tanh BP 연산은 채택하지 않는 구조라, BP/MS 스위칭 기법은 오히려 회로 복잡도를 늘리는 방향이라 이식 실익이 적음.
