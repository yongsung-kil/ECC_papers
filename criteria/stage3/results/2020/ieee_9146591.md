### B. 알고리즘 요약

1. 대상은 racetrack memory(RM)의 position error(PE) 채널: 트랙 이동 오차로 인해 삽입(sticky-insertion)·삭제(deletion) 오류가 다중 read port(arp개)에서 동시에 발생하는 채널.
2. 기존 반복(iterative) MAP검출+LDPC복호 방식은 BCJR 기반 검출기의 연산복잡도가 높아, 검출을 1회만 수행하는 비반복(non-iterative) 시나리오에서 성능이 크게 저하되는 문제를 해결하고자 함.
3. 채널 모델: PE 채널은 확률 `pid`로 삽입/삭제가 동시에 모든 RP에서 발생하는 hidden Markov model 상태(state drift `st`)로 표현되며, MAP검출은 BCJR forward-backward 알고리즘으로 APP-LLR을 계산.
4. 핵심 기법 1단: outer irregular LDPC code와 inner 2-D-marker code(사전 정의된 pilot bit 시퀀스, arp개 subblock에 규칙적 간격 `d(j)`로 삽입)를 직렬연결(concatenation), 전체 코드율 `r = rin·rout`.
5. 핵심 식: BCJR-once bound `Conce`(식 9, 채널입력과 APP-LLR 간 평균 상호정보량)로 비반복 시나리오에서의 달성 가능 정보율을 측정, marker 구조/간격을 이 지표 기준으로 최적화.
6. 확장: regular 2-D-marker 설계를 위한 4단계 ad hoc 규칙(Rule 1~4) - 후보 inner rate 선택 → 후보 rate tuple 집합 R → 후보 interval tuple 집합 D → Conce 최대화하는 interval tuple 선택.
7. 부수 트릭: outer LDPC는 `λ'(0)ρ'(1) < 1` 조건(선형 최소거리 증가 특성, error floor 억제)을 만족하는 irregular ensemble로 제한, 최대차수 `dv=20, dc=50`로 설계.
8. 최적화 절차: 수정된 density evolution(1회 MAP검출만 반영)과 기존 code optimization 기법[19]을 결합해 목표 PE확률 `pi*_d`(zero-error decoding threshold)를 최대화하는 degree distribution을 탐색.
9. 결과(시뮬레이션, `nL=5k/50k`): arp=4일 때 BER=10⁻⁵ 기준 PE 허용확률이 arp=1 대비 약 4.7배(7.6×10⁻³→3.6×10⁻²) 개선, BER=10⁻⁵까지 error floor 관측 안 됨(최소거리 증가 특성 덕분).
10. 한계: HW 구현/합성 전혀 없음(순수 알고리즘/시뮬), outer 복호는 표준 log-domain SPA(반복 100회)로 min-sum이나 HW 최적화 요소 없음, RM 고유의 다중 read-port 동시 삽입/삭제 채널 모델이라 NAND flash 채널(독립 bit-flip)과 직접 대응되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Outer irregular LDPC 부호 구성/degree distribution 설계 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조라 irregular ensemble 설계 자체가 적용 난이도 높음(§4 "부호설계=높음"에 해당) |
| MAP검출(BCJR)/삽입-삭제 채널 모델 | 대응 없음 | Prime ECC는 AWGN/RBER/fixed-error 채널만 지원, 삽입/삭제 동기화 오류 모델 없음 |
| 2-D-marker code(pilot bit 삽입) | 대응 없음 | NAND ECC 파이프라인에 marker 삽입 개념 자체가 없음 |
| outer LDPC log-domain SPA 복호 | `decoder.cpp` `LDPC_Decoder()` | 표준 SPA로 이미 Prime ECC는 더 HW효율적인 min-sum 사용 중, 이식 실익 없음 |

> 적용 가치: 낮음 — 채널 모델(삽입/삭제 동기화 오류)과 문제 설정(비반복 검출-복호)이 NAND binary LDPC ECC의 bit-flip 채널·decoder HW 설계와 근본적으로 달라, Prime ECC 3.1에 이식할 요소가 거의 없음.
