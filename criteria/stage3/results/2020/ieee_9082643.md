### B. 알고리즘 요약

1. 5G NR LDPC(BG1/BG2 base graph)는 파리티체크행렬 첫 2블록컬럼(시스템 비트)을 펑처링해 송신하지 않는 구조적 특징이 있어, 이 펑처링이 layered BP의 수렴 속도를 느리게 만드는 문제를 다룬다.
2. 기존 layered BP(LBP)는 무작위 또는 범용 고정 순서로 layer를 갱신해 NR LDPC의 펑처링 구조를 활용하지 못하며, 정보이론적 informed dynamic scheduling(RBP, NW-RBP)은 성능은 좋으나 계산량이 크다는 문제가 있다.
3. 채널모델은 AWGN + QAM 변조이며, C2V 메시지는 `m_{ci→vj} = 2·atanh(Σ tanh(L_{vb→ci}/2))`(식 1, sum-product) 또는 min-sum 근사식(식 2)로 계산된다.
4. 첫 번째 기법 LPHD(Least-Punctured Highest-Degree)는 펑처링된 변수노드가 1개뿐인 check node(펑처링 변수노드 2개면 메시지가 0이 되어 무의미)를 최우선 갱신(LP)하고, 동률이면 row-weight(차수)가 높은 layer를 우선(HD) 처리하는 고정 스케줄이다.
5. 핵심 아이디어는 "펑처링된 심볼이 빨리 복원될수록 메시지 전파가 유효해진다"는 것으로, 별도 추가 연산 없이 layer 순서만 재배열해 수렴을 가속한다.
6. 두 번째 기법 RB-LBP(Residual-Based LBP)는 매 반복 전 C2V residual `r_{ci→vj} = m^pre_{ci→vj} - m_{ci→vj}`(식 6)을 min-sum 근사(식 2)로 미리 계산해 각 layer의 최대 residual을 구하고, 이 값의 내림차순으로 layer 순서 `Q`를 동적으로 재배열한다.
7. 구현 디테일로 residual 사전계산은 정밀도가 필요 없어 저복잡도 min-sum(식 2)을 쓰고, 실제 C2V 갱신은 sum-product(식 1)로 수행해 정밀도 손실을 최소화하는 하이브리드 근사를 사용한다.
8. 별도 학습/최적화 절차는 없으며, 복잡도는 해석적으로 유도된다: RB-LBP는 정렬 대신 최대값 탐색만 필요해 비교연산 `E+M(M-3)/2 ≈ M²/2`로 RBP(`≈d_c²M²`)/NW-RBP(`≈d_cM²`)보다 훨씬 적다(Table I, 식 7-9).
9. 결과로 length-9024, rate-0.468 시뮬레이션에서 LPHD는 기존 LBP 대비 0.5dB, RB-LBP는 0.65dB 이득(BER 10^-5 기준)을 달성했고, RB-LBP는 NW-RBP 대비 근소한 성능 차이로 훨씬 낮은 복잡도를 보였다.
10. 한계는 HW 구현이 없고 순수 시뮬레이션(AWGN)에 그치며, 5G NR 특유의 base graph 펑처링 구조에 특화되어 있어 다른 코드 구조에서의 일반화는 검증되지 않았다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered BP 스케줄링(LPHD, RB-LBP) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 내 layer/column 처리 순서를 결정하는 스케줄러이므로 동일 루프 구조에 스케줄 로직만 추가 이식 가능 |
| C2V residual 기반 동적 재배열 | `decoder.cpp` `C2V_Cal_New_Sgn()` | C2V 메시지 계산 자체는 CN 갱신 로직과 동일 위치, residual 비교값 산출을 위한 추가 버퍼/로직 필요 |
| 펑처링 컬럼(punct_col) 우선처리(LP) | `ecc_data.h`(`N_b`, `punct_col_num=2`), HCU 로직(`decoder.cpp`) | Prime ECC도 `punct_col_num=2`로 펑처링 구조를 이미 가지므로, LP 아이디어(펑처 변수노드 1개인 check 우선)를 HCU 처리 순서와 결합 검토 가능 |
| Dual-Update(odd/even 교대 스케줄) | `decoder.cpp` Dual-Update 옵션 | 기존 보유 스케줄링 기법과 개념적으로 유사(스케줄 순서 최적화)하여 중복 여부 확인 필요 |

> 적용 가치: 중간 — Prime ECC가 이미 펑처링(punct_col_num=2) 및 Dual-Update 스케줄 옵션을 보유하고 있어 개념적 중복이 있지만, LPHD/RB-LBP의 "펑처 우선 + residual 기반 동적 재배열"은 저복잡도로 수렴속도(iteration/latency)를 개선하는 구체적 스케줄 규칙이라 decoder.cpp 이터레이션 루프에 추가 이식을 검토할 가치가 있음.
