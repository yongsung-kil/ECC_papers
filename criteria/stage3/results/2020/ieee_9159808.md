> 총평: Row-layered OMS 디코더의 파이프라인 깊이를 데이터 의존성 완화(Δ-update)로 늘려 EDP를 최대 75% 줄인 HW 아키텍처 논문으로, 정정성능 개선이 아닌 타이밍/전력 최적화 기법이라 이식 가치는 중간 수준.

### B. 알고리즘 요약

1. QC-LDPC 3종(5GNR CR=1/3·Z=32, 802.11n CR=1/2·Z=27, C9-Z18 CR=1/2·Z=18)에 대해 row-layered offset min-sum(OMS) 디코더를 설계.
2. 문제: 밀집한 irregular 코드(5GNR, 802.11n)는 layer 간 데이터 의존성으로 파이프라인을 깊게 못 넣어 clock 속도가 제한됨.
3. 채널은 BPSK+AWGN, `satQ(x)` 포화 양자화로 `Qµ`-bit belief 계산.
4. 핵심 기법: `∆-update` — CN→VN 메시지 `λ`의 갱신값과 이전값 차이 `∆i = λj(t→)i − λj(t→−1)i`만 계산해 `Λi ← satQΛ(∆i + Λi)`로 belief sum을 갱신, 동시 충돌 VN만 스케줄을 늦춰 처리.
5. 기존 (3)식(직접 Λi 갱신) 대신 차분 갱신을 씀으로써 read-after-write 충돌 없이 깊은 파이프라인 레지스터 삽입 가능.
6. HW 구조: `DCMAX` Shift-∆-Mem × `ZMAX` CNPE 병렬, 2-min search는 sort-merge 트리, `dDEC = d2-MIN + 2×dshift + 2`로 총 파이프라인 깊이 결정.
7. 구현 파라미터: MAX_DC=26, ZMAX=32, itmax=25(ET 포함), QΛ=8, Qλ∈{5,6,7} 비교, β=1 offset.
8. 최적화는 Monte-Carlo로 waterfall/error-floor 영역별 belief scaling `α` 튜닝(학습 절차라기보다 파라미터 탐색).
9. 결과: dDEC를 5→13으로 늘리면 clock 주파수 거의 2배, 대신 cell count +21%·power ×2.75; dDEC=7에서 EDP는 5GNR -75%, 802.11n -62%, C9-Z18 -23% 감소.
10. 한계: dDEC≥9부터는 EDP 개선이 미미(면적/전력 대비 이득 감소), 정정성능(FER) 자체의 개선은 없고 오히려 iteration 수가 늘어나는 트레이드오프 구조.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Δ-update 기반 layer 간 파이프라인 완화 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 스케줄링 구조를 바꾸는 기법으로 대응 위치는 명확하나 우리 구조(HCU, Dual-Update)와의 정합 재검증 필요 |
| CN min-sum(OMS, offset β) 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum 계열 보유, offset 파라미터 β=1은 참고 가능 |
| VN belief sum 양자화(Qλ, QΛ) | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | soft quantization 폭(5~7bit) 비교 참고용, NAND read 비용과 직결 |
| 파이프라인 depth vs EDP 트레이드오프 분석 | HW arch(디코더 HW 모델, z=32, HCU, GT) | 우리 z_sb=32 lifting과 유사 규모, 파이프라인 삽입 전략 참고 가능하나 RTL 재설계 필요 |
| 채널/벨리프 스케일링 α 최적화 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | waterfall/error-floor 별 α 최적화 아이디어는 LLR 테이블 튜닝과 유사 |

적용 가치: `중간` — Δ-update는 파이프라인 깊이 확장을 위한 스케줄링 완화 기법으로 우리 디코더 구조(HCU, Dual-Update)에 개념적으로 이식 가능하나, 정정성능 개선이 아닌 타이밍/전력 최적화이며 RTL 재설계 부담이 크다.
