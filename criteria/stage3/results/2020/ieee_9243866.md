### B. 알고리즘 요약

1. 대상은 방송 시스템용 QC-SC-LDPC 코드로, 결합길이(coupling length) `L=40`, smoothing parameter `w=6`, lifting factor `Z=768`, 최종 부호길이 `n_SC=92160`, uncoupled code rate `R_true=2/3`, 최종 rate `R_design=13/20`(=0.65).
2. 문제의식은, SC-LDPC는 채널 보편적 capacity-approaching 성능을 위해 긴 coupling length가 필요한데, 전체 full BP decoding은 하드웨어 자원·지연을 크게 증가시킨다는 점.
3. sliding window decoding은 check-node 중심 window(크기 `Nw`, layer 단위)로 SC-LDPC의 convolutional 구조를 이용해 지연·복잡도를 `L`에 무관하게 만듦; window는 forward로만 이동(HW 구현 단순화).
4. 핵심 기법 1: decoding complexity 제약(`Nw*I=30~60`, layer-scheduled decoding) 하에서 SNR threshold를 최소화하는 최적 window size가 존재함을 시뮬레이션으로 규명 (`Nw=16`이 최적).
5. 복잡도 정의식 `I_bar = (Σ_{j=1}^{MSC} I_j) / MSC` (layer별 horizontal processing 연산 횟수 평균)로 sliding window와 conventional 방식을 공정 비교.
6. 핵심 기법 2: 조기종료(ET) 조건 2종 - (a) 현재 window 내 반복 조기중단(첫 3개 layer의 parity-check가 각각 3,3,2회 연속 만족 & LLR 부호 불변), (b) window 전체 조기종료(미만족 parity-check 비율이 임계값 `p=0.0625` 초과 시).
7. 기존 PC-based ET의 약점(LLR이 0 근처인 변수노드의 부호 잦은 변화로 불안정한 parity 만족)을 "연속 반복 만족 + 부호 불변" 조건으로 보완.
8. 별도 학습/최적화 절차 없음(시뮬레이션 기반 파라미터 탐색).
9. 결과: 최적 window size로 SNR threshold 개선, ET 적용 시 복잡도 저·고 SNR 영역에서 60%까지 감소(No ET 대비), SNR 손실은 0.025dB 이하; conventional full BP 대비 복잡도 1/11 수준, SNR 손실 0.12dB 이하로 BLER `10^-1~10^-3` 구간에서 성능 거의 유지.
10. 한계: BI-AWGN 채널 시뮬레이션에 한정, HW 미설계, SC-LDPC 고유의 window sliding 구조 전제(QC-LDPC block code에는 직접 적용 불가).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sliding window decoding (SC-LDPC 구조 특화) | 대응 없음 | Prime ECC는 QC-LDPC block code로 convolutional/coupled 구조가 없어 window sliding 개념 자체가 적용 불가 |
| layer-scheduled decoding | `decoder.cpp` `LDPC_Decoder()` | 레이어 단위 스케줄링 개념은 유사하나 논문은 SC 구조 전제로 직접 이식 어려움 |
| 조기종료(ET) - parity-check 연속 만족 기반 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료라는 목적은 동일하나 판정 기준(CRC vs. parity-check 연속만족·LLR 부호불변)이 상이해 개념 참고 수준 |

적용 가치: **낮음** — SC-LDPC의 convolutional 구조에 특화된 window decoding 최적화이며, Prime ECC는 고정 크기 QC-LDPC block code라 구조적 전제가 맞지 않음. 조기종료 아이디어만 개념적으로 참고 가능하나 이미 Partial CRC 기법을 보유하고 있어 중복.
