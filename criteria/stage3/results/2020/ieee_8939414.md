### B. 알고리즘 요약

1. 대상: `GF(2^p)` 위에서 정의된 non-binary LDPC(NB-LDPC) 부호, BI-AWGN 채널, 시뮬레이션 부호는 rate 1/2~0.89, 길이 32~1998 심볼(`GF(2^3)`, `GF(2^4)`).
2. 문제: 기존 QSPA(Q-ary sum-product)는 복잡도 `O(q^2)`로 과도하며, FFT-QSPA(`O(q log q)`)도 여전히 non-binary 연산(확률벡터 permutation/convolution)이 필요해 HW 구현 부담이 큼.
3. 핵심 가정/모델: `K=GF(2^p)`의 가법 부분군 `H`(order `2^{p-1}`)의 proper coset `C_H`를 이용, 심볼이 특정 coset에 속할 확률을 이진 확률변수로 재정의하면 check node의 convolution 연산이 binary LDPC의 XOR 연산(`p_out = p1(1-p2)+(1-p1)p2`)과 동일해짐.
4. 핵심 기법: 하나의 `GF(2^p)` 노드를 `(2^p-1)`개 binary 노드(각 coset 대응)로 확장하는 새로운 binary Tanner graph 변환("graph expansion"). 추가로 coset 확률 간 의존성을 포착하기 위해 `(2^p-1,p)` 로컬 부호(coset representation, 심플렉스 부호의 쌍대부호)의 parity-check로 `(2^p-p-1)`개의 local check node를 삽입.
5. 핵심 식: local check 행렬(`H_local`)의 girth를 6 이상으로 만들도록 row 조작(예: `(7,3)` 코드에서 row weight 3, girth≥6 확보)하여 message passing에 적합하게 함; global/local check node를 구분해 가중치 `w_l`(local), `β`(초기 추정 스케일)로 보정.
6. 확장 알고리즘 CPbD(Coset Probability based Decoding): SPA(sum-product)를 binary 확장 그래프 위에서 수행하되, 전송되지 않는 `(2^p-p-1)`개 parity bit는 채널 LLR로부터 tanh 기반 계산식(식 7,11)으로 초기 추정.
7. 부수 트릭: min-sum 근사(CPbD-MS) 적용 시 정규화 인자 `δ`(0<δ≤1) 사용, `w_l=1/2^r` 형태로 두면 스케일링이 단순 bit-shift로 구현 가능(HW 친화).
8. 최적화 절차: `β`, `w_l`, `δ` 등 하이퍼파라미터는 코드별 시뮬레이션으로 결정(학습 알고리즘 없음, grid search 수준).
9. 결과: rate 1/2 `GF(2^3)` 부호(C1)에서 CPbD-M이 QSPA 대비 FER 10^-4 기준 0.3dB 이내 손실, min-max 대비 0.1dB 우위; CPbD-MS는 QSPA 대비 iteration당 decoding latency 33~80% 감소(코드별); check node 복잡도 `O(2^p)`로 QSPA류(`O(2^{2p})`, `O(2^p p)`) 대비 크게 절감.
10. 한계: non-binary LDPC 전용 알고리즘(binary LDPC에는 원천적으로 적용 대상 아님), HW 구현은 "적합함"만 논의하고 실제 합성/FPGA 검증 없음, 메모리 이득은 없음("do not offer any advantage in memory"으로 본문 명시)을 저자가 인정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NB-LDPC(GF(2^p)) Tanner graph → binary graph 확장 | 대응 없음 | Prime ECC는 처음부터 binary QC-LDPC이므로 non-binary→binary 변환 자체가 불필요 |
| Coset probability 기반 binary check node 연산 | `decoder.cpp` `C2V_Cal()`, `CNU_Update_New_Mag()` | 최종 결과가 binary LDPC check node 연산(XOR 기반)으로 수렴한다는 점은 개념적으로 유사하나, 실제로는 non-binary 심볼 문제를 binary로 "변환"하는 절차이므로 이미 binary인 Prime ECC엔 대응할 대상이 없음 |
| min-sum 근사 + 정규화 인자 δ (CPbD-MS) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 Prime ECC가 min-sum + 정규화 계열을 보유(§3), 중복 기여로 추천도 낮음 |
| 로컬 코드(girth 확보) 삽입 | 대응 없음 | non-binary 심볼 표현의 의존성 포착용 특수 구조, binary LDPC 설계와 무관 |

> 적용 가치: 낮음 — non-binary LDPC 전용 복잡도 절감 기법으로, 이미 binary QC-LDPC인 Prime ECC 3.1에는 적용 대상 자체가 없음(도메인 불일치).
