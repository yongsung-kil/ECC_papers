### B. 알고리즘 요약 (10줄 내외)

1. 대상 시스템은 차세대 ITU-T DSL(MGFAST) 표준에 채택된 LDPC-coded modulation(LCM) 방식으로, M-QAM 심볼의 일부 비트(Bc)만 rate `R_LDPC=K/N` QC-LDPC로 부호화하고 나머지(Bu)는 비부호화 상태로 영역 매핑을 통해 유클리드 거리 이득만 얻는다.
2. 기존에는 LCM 성능 평가가 Monte Carlo 시뮬레이션에만 의존해 코드/스킴 파라미터가 많을 때 시스템 설계가 어렵다는 문제를 다룬다.
3. 채널은 binary-input AWGN이며, 코딩/비코딩 비트 각각을 등가 binary-input continuous-output bit channel로 모델링하는 것이 핵심 가정이다.
4. 핵심 기법은 두 갈래: (1) 코딩·비코딩 비트 위치별 hard-decision 오류율의 폐형식(complementary error function 기반) 유도, (2) 이를 등가 잡음분산 `σ²_n,equiv`으로 변환해 표준 EXIT 분석에 대입.
5. 핵심 식 `σ²_n,equiv(b, Eb/N0) = 1/(2·erfc⁻¹(2·P_cb(b, Eb/N0)))²` — 비트채널별 오류확률을 등가 AWGN 잡음분산으로 환산해 EXIT VN/CN mutual information 업데이트에 사용.
6. 확장 기법으로 composite EXIT chart `I_E,V(I_A,V, Eb/N0) = Σ_b Σ_i (λ_i · I_E,V(I_A,V, d_v,i, σ²_n,equiv))`를 구성해 코드의 디코딩 임계값(threshold)을 해석적으로 도출.
7. 부수 트릭으로 mutual information 근사(`I_ch ≈ I_E,V(0, Eb/N0)`, code rate>0.5에서 유효)를 이용해 BPSK 시뮬레이션 결과 1회만으로 LCM 유한장 성능을 준해석적으로 예측(semi-analytical finite-length predictor).
8. 최적화/학습 절차는 없음 — 순수 해석적 유도와 시뮬레이션 검증(수치 적분·closed-form 식).
9. 결과: N=12000, Bc=4, 256-QAM 등 조건에서 EXIT 임계값이 실제 시뮬 성능과 잘 일치, LCM vs BICM/TCM 비교에서 concatenated RS-TCM 대비 BER=10⁻⁷에서 약 2dB 개선, 짧은 블록(N=600)에서 MI 기반 예측이 완전 시뮬과 거의 일치(N=48000에서는 약 0.1dB 갭).
10. 한계: 순수 이론/시뮬 분석 논문으로 HW 설계 없음, DSL 유선 채널(AWGN) 가정, 디코더는 표준 log-domain SPA(최대 50 iteration) 그대로 사용해 LDPC 알고리즘 자체의 개선은 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LCM 다중레벨 변조/비트매핑 | 대응 없음 | Prime ECC는 NAND binary read/write 채널 대상이며 M-QAM 변조·비트매핑 개념 자체가 없음 |
| EXIT chart 기반 임계값 해석 | 대응 없음 | 이론적 부호 설계/분석 도구이며 Prime ECC의 특정 H-matrix(`ecc_top.cpp` `Load_PCM()`)에 대한 재설계 절차가 아님 |
| 표준 QC-LDPC dual-diagonal 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()` | 인코딩 구조(dual-diagonal)는 Prime ECC와 유사한 개념이나 이미 보유 기법이라 신규 기여 없음 |
```

적용 가치: **낮음** — DSL coded-modulation 성능 해석 논문으로 NAND binary LDPC 디코더/부호설계 개선과 무관하며, LDPC 코드 자체는 표준 QC-LDPC를 그대로 사용.
