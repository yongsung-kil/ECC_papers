### B. 알고리즘 요약

1. 대상 매체는 RRAM(저항변화메모리), reset hard breakdown 오류가 축적되어 LDPC soft decision CBER과 반복횟수를 크게 악화시킴.
2. 기존 LDPC(가우시안형 오류 가정)의 대칭 LLR 스킴은 RRAM의 비대칭(매우 낮은 저항 방향으로 치우친) reset hard error 분포에 부적합함.
3. 채널 모델: reset/set retention error는 대칭 가우시안이나, reset hard breakdown error는 최저(가장 신뢰도 높은 음의) LLR 영역에 집중되는 별도 분포로 모델링.
4. 기법1 improved-LLR: reset hard error 분포에 맞춘 비대칭 LLR 값 `X`(최적값 −4)를 재매핑해 min-sum 소프트 디코딩 초기 LLR로 사용.
5. 기법2 RCRF(reset-check-reverse-flag): 유저데이터를 서브섹션(128비트 단위)으로 나누고 flag bit 1개로 섹션 내 reset 오류를 최소 1개 정정(user data 전체 반전 방식), overhead는 `LDPC rate / (LDPC rate + Subsection size)`(식 1).
6. RCRF+LDPC 결합: reset-all-bits → 오류 위치 sensing → flag 반전/데이터 반전 정정 → systematic LDPC 인코딩 → 1회 set program만 수행(프로그램 latency 증가 없음).
7. 구현 디테일: 128-bit/subsection 선택 시 array overhead 0.69%, 최대 30 iteration 제한 min-sum 디코더 사용, girth 4/6 회피 H-matrix.
8. 별도 학습/최적화 절차는 없음(X값 스윕 실험으로 최적 −4 도출).
9. 결과: hard decision CBER 50%+ 개선, soft decision CBER 1KB/0.91-rate QC-LDPC에서 69%, 2KB/0.89-rate Mackey code에서 51% 개선, iteration 감소.
10. 한계: 시뮬레이션만(HW 미설계), waterfall 영역 초반에서 시뮬 중단(error floor 미검증), RRAM reset hard breakdown 특유 오류 모델에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| improved-LLR (비대칭 LLR 재매핑) | `channel.cpp` `Set_LLR_Th()` | NAND는 대칭 오류모델 가정이 일반적이라 직접 대응 낮으나 LLR threshold 테이블 구조는 유사 |
| RCRF (flag bit 기반 reset 오류 사전정정) | 대응 없음 | RRAM reset-all-bits 특화 전처리로 NAND 프로그램/read 구조와 무관 |
| LDPC min-sum 디코딩 자체 | `decoder.cpp` `LDPC_Decoder()` | 논문은 표준 min-sum을 그대로 사용, 개선 기여 없음 |

> 적용 가치: 낮음 — RRAM 고유 reset hard breakdown 모델 기반 전처리(concatenation) 기법으로 LDPC 부호/디코더 알고리즘 자체 개선이 아니며 NAND 채널 모델과 직접 대응되지 않음.
