### B. 알고리즘 요약

1. 시스템: 5G NR QC-LDPC, irregular base graph 1 코드 (16128, 8448), rate 22/42, layered decoding, partially-parallel 아키텍처(대량의 serial CNU 사용).
2. 문제: 표준 OMS는 CNU마다 min1/min2를 모두 계산해야 하며, serial CNU(메시지를 순차 입력받는 구조)에서는 두 comparator가 필요해 자원 소모가 큼.
3. 핵심 가정: 5G NR 코드는 check node degree(`dc=3~19`)가 매우 다양한 irregular 구조이며, subminimum-minimum 차이는 degree가 클수록 작아진다는 관찰(고정점 시뮬레이션으로 확인).
4. 핵심 기법 1단: single-minimum OMS(smOMS/vwsmOMS)로 second-minimum을 `M = min + w`(weight parameter)로 근사해 comparator 1개만 사용.
5. 핵심 식: `Mc2v = max(M − β, 0) × sgn(∏Mv'2c)`, 여기서 `M`은 `Nmin=1`일 때 `min+w`, 아니면 그대로 `min` — subminimum 계산 없이 offset 적용.
6. 핵심 기법 2단(확장): check node degree별로 weight `w`를 다르게 설정하는 EvwsmOMS — 고차수 노드는 작은 `w`, 저차수 노드는 큰 `w`(`dc=3`: `0.5+0.4·it`, `dc=19`: `0.25·it`), layer 단위로 이미 외부 제어되는 `w` 파라미터라 추가 HW 비용 없음.
7. 구현 디테일: `w−β`를 미리 계산해 저장, output 단에서 뺄셈 대신 부호 있는 덧셈만 수행(overflow 시 saturate)해 addition/subtraction 연산 1개로 축소. 6-bit 메시지, 8-bit LLR 양자화.
8. 학습/최적화: 고정점 Monte Carlo 시뮬레이션으로 각 dc 그룹의 subminimum-minimum 평균 차이를 측정해 선형 weight 함수(`Table 1`)를 수동 튜닝.
9. 결과: EvwsmOMS는 vwsmOMS 대비 BLER 유의미하게 개선(원 OMS 대비는 여전히 열세), FPGA 자원(384 CNU 기준) OMS 대비 CLB -35%, LUT -11%, FF -25%.
10. 한계: 원본 OMS 대비 SNR 손실이 여전히 상당함(waterfall 구간 열화), throughput 수치 미보고, 정정능력(error-floor) 개선은 다루지 않음(오히려 근사로 인한 손실 트레이드오프).

적용 가치: **높음** — CNU에서 second-minimum 계산을 생략하고 degree별 weight로 근사하는 기법은 이미 min1/min2 구조를 가진 Prime ECC decoder.cpp의 CNU 로직에 낮은 난이도로 이식 가능하며 게이트 절감 효과가 실측(FPGA)됨. 단, waterfall 성능 손실 트레이드오프를 NAND 채널에서 재검증 필요.
