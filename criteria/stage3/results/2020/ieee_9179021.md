### B. 알고리즘 요약

1. 대상은 IEEE 802.16(WiMAX), DVB-S2X, 5G NR의 고도로 불규칙(irregular)한 QC-LDPC 코드에 대한 row-layered offset min-sum 디코더 (LLR 8bit, 메시지 6bit 양자화).
2. 파이프라인 layered 디코더는 메모리 접근 해저드(data hazard)로 stall cycle이 필요해 처리량이 저하되고, 특히 5G NR처럼 인접 레이어 간 check node weight(CNW) 차이가 크면(최대 dc,max-dc,min=16 cycle) 손실이 큰 문제를 해결하려 함.
3. 채널/노이즈 모델은 표준 AWGN + QPSK, offset min-sum(`Mc2v = max(min|Mv2c|-β,0)×sgn`) 근사를 사용.
4. 핵심 기법은 hybrid schedule: 메모리 충돌(오래된 LLR 읽기) 발생 시 layered schedule처럼 대기하지 않고 outdated LLR을 읽어 flooding 방식으로 처리(`LLRvlk = LLRvli + Mc2v` 식 13~16), 이후 최신 LLR이 준비되면 buffer memory에 double-write.
5. 핵심 식 `LLRvlk = LLRvlj + Mc'2v`(식 16)는 지연된 check node 기여분을 즉시 반영해 residue 기반 선행 연구([33])와 달리 LLR 갱신을 지연시키지 않고도 stall 없이 진행.
6. 확장 기법으로 CNU 내부에 decoupling FIFO 버퍼를 삽입해 layer 간 CNW 변화로 인한 stall도 완전 제거(필요 depth ≤ `dc,max/dc,min`).
7. HW 구현 디테일: LLR RAM 더블버퍼링(디코딩 메모리+버퍼 메모리 분리), cyclic shifter는 LLR이 아닌 메시지만 shift(비트폭 절감), WiMAX/5G NR용 2단(prerotator+QSN) 유연 shifter와 DVB-S2X용 고정 3단 shifter 별도 설계.
8. 학습/최적화 절차: 레이어·VNG(variable node group) 처리 순서를 유전 알고리즘(GA, TSP류 문제)으로 오프라인 재배열해 outdated LLR read 횟수(=SNR 손실 원인)를 최소화 — recombination(부분 벡터 교차)+mutation.
9. 결과: Xilinx ZCU111(RF-SoC) FPGA 구현, 13 pipeline stage 기준 5G NR 4.92Gbps(10 iteration), 동일 SNR 성능 기준 기존 layered 디코더 대비 throughput 30.8~109.1% 향상, GA 최적화로 up-to-date LLR read 25~120% 개선, enhanced hybrid schedule의 SNR 손실은 대부분 0.1dB 이하(고rate·저iteration 예외 시 0.2dB+).
10. 한계: hybrid schedule은 근본적으로 SNR 성능을 소폭 저하시키므로(고밀도 base graph·고파이프라인 단수일 때 더 심함) 추가 iteration으로 보정이 필요하며, ASIC 합성이 아닌 FPGA 구현만 제시(게이트카운트/공정 수치 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| hybrid schedule (layered↔flooding 전환, stall cycle 제거) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 파이프라인 데이터 해저드 해소 기법으로, Prime ECC 파이프라인(col_M1/Z/P1~P4) stall 최소화에 직접 참고 가능 |
| CNU decoupling FIFO (CNW 변화 대응) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | irregular 코드의 CN degree 변화 대응 버퍼링 구조로, Prime ECC의 HCU(비대칭 degree col 처리)와 목적상 유사·보완 가능 |
| LLR 더블버퍼링(decoding/buffer memory 분리) | 대응 없음 (Prime ECC HW 파이프라인 구조 참조 필요) | 코드 프로파일에 명시된 메모리 구조 정보 부족으로 정확한 대응 불가 — "대응 없음"으로 처리 |
| GA 기반 PCM/VNG 오프라인 재배열 최적화 | `ecc_top.cpp` `Load_PCM()` | 부호 구조(H-matrix) 로드 시 스케줄 최적화라는 개념적 접점은 있으나 Prime ECC는 고정 base matrix라 직접 재배열 적용은 제한적 |
| 메시지만 cyclic shift(LLR 대신, 비트폭 절감) | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD) | HW 면적 절감 트릭으로 soft-info 폭이 큰 2SD/3SD 처리에 참고 가치 있음 |

적용 가치: 중간 — hybrid schedule과 decoupling FIFO는 파이프라인 stall 제거라는 명확한 HW 이득을 실측(FPGA)으로 입증했으나, Prime ECC는 이미 Dual-Update 스케줄을 보유하고 있어 중복 가능성이 있고 GA 기반 재배열은 고정 base matrix 구조상 적용 난이도가 있음.
