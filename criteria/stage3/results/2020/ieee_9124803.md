> 총평: girth 확대를 통한 QC-SC-LDPC 구성법 논문으로 정정능력에 대한 직접 기여(대상=both)로 보기엔 본문 수치 근거가 부족해 code-design으로 분류했고, NAND 관련성도 낮아 이식가치는 제한적.

### B. 알고리즘 요약

1. 시스템: rate `R=(n-m)/n` binary time-varying QC-SC-LDPC. 시뮬레이션은 QPSK 변조 + AWGN 채널, sliding-window layered(Log-BP) 디코딩 사용.
2. 문제: 기존 protograph 기반 SC-LDPC 구성법([6])은 code rate가 `(n-1)/n`으로 제한되고, 4-cycle 회피를 위한 syndrome former memory `μ` 하한이 명확하지 않았음.
3. 핵심 가정: base protomatrix `B` (크기 `(μ+1)m×n`)를 반복 시프트해 만든 convolutional protomatrix `Bconv`에서 4-cycle 존재 여부는 열 내 "1" 원소 간 거리값(distance parameter) `B_{i,k}^{(j)}`의 중복 여부로 판정 가능.
4. 핵심 기법 1단: 4-cycle-free 조건(`B_{i1,k1}^{(j1)} ≠ B_{i2,k2}^{(j2)}`, 식 12)을 만족하는 base protomatrix를 backtracking search(Algorithm 1)로 구성, code rate 제한 없이 girth-6 protograph를 얻음.
5. 핵심 식: 4-cycle-free 최소 `μ` 하한 `μ ≥ ⌈n·C(dv,2)/m⌉` (식 14) — 열의 "1" 쌍 개수(`C(dv,2)`)만큼 distinct distance가 필요하다는 조합론적 필요조건.
6. 핵심 기법 2단: base protomatrix를 QC lifting(circulant shift, lifting factor `S`)하여 실제 QC-SC-LDPC 행렬 생성, girth profile(GP)·sliding-window girth(SWG) 최적화로 girth-8까지 확대. 시간가변(period `T`) 행렬을 regrouping(식 5)하여 시간불변 QC-SC-LDPC로 재구성해 encoding/windowed decoding 동기화.
7. 구현 디테일: 첫 행을 모두 "1"로 고정해 탐색 공간을 `5.5×10^3`에서 `1.3×10^10`으로 늘리되 열 재정렬 불필요하게 함; systematic encoding 위해 첫 `m`열을 뒤로 재배치.
8. 최적화 절차: backtracking search(Algorithm 1)로 후보 protomatrix 생성 → row degree 편차 최소인 것 선택 → QC lifting 시 GP/SWG 만족할 때까지 반복, 불만족 시 제약 완화.
9. 결과: S1(girth-8) vs S2(girth-6) 코드가 고SNR에서 error floor 개선(그림만, 수치는 정성적 서술); 기존 문헌 [2] 대비 약 0.1dB 이득(단, FPGA 양자화 손실 차이 기인); dv=4 QC-SC-LDPC가 동일 iteration에서 dv=3 BC-LDPC 대비 0.11~0.34dB 이득, 복잡도는 BC-LDPC의 약 89.8%.
10. 한계: 순수 알고리즘/시뮬 수준(HW 미설계), 채널은 AWGN/QPSK 일반 통신 가정으로 NAND 특화 요소 전혀 없음, 정량적 BER/FER 값은 대부분 그림에만 존재해 본문 서술 수치는 제한적(gain %/dB만 텍스트로 확인 가능).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-6/8 protograph 구성 알고리즘(backtracking) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체가 다른 방식으로 생성되므로 로드 위치만 대응, 구성 로직은 대응 없음 |
| QC lifting / circulant shift 기반 QC-SC-LDPC 행렬 생성 | 대응 없음 | Prime ECC는 고정된 QC-LDPC(`PCM_b`) 구조이며 spatially-coupled 구조 자체가 없음 |
| sliding-window layered(Log-BP) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | SC-LDPC 특유의 windowed decoding으로, Prime ECC의 block 단위 디코딩과는 구조가 다름 |

적용 가치: **낮음** — spatially-coupled 부호 구성법 자체는 Prime ECC의 고정 QC-LDPC 구조와 근본적으로 다르고(재검증 부담 큼), NAND/SSD 타깃 언급이 없으며 정정능력 개선의 정량 근거도 본문에 부족.
