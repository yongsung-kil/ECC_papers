> 총평: DVB-S2 위성통신용 BCH-LDPC concatenated 인코더의 VCM(가변부호율) 대응 FPGA 구현으로, 인코더 자원 재사용·병렬화 기법은 있으나 Prime ECC가 이미 보유한 dual-diagonal 인코딩과 개념적으로 중복되어 이식 가치는 제한적.

### B. 알고리즘 요약 (10줄 내외)

1. DVB-S2 표준의 BCH(외부코드)+IRA-LDPC(내부코드) concatenated FEC를 LEO 위성-지상국 VCM(가변부호변조, 코드rate `2/3`~`4/5`, 프레임길이 `n=16200`) 시스템용으로 인코더 코어 하나에 통합 구현.
2. 문제: 기존 CCM(고정 부호변조)은 최악 링크버짓 기준 설계로 링크자원 낭비; VCM은 다양한 code rate/모듈레이션을 지원해야 하는데 하드웨어 자원 재사용과 고속 인코딩을 동시에 만족하는 인코더가 부족.
3. IRA-LDPC의 parity-check matrix `H=[A|B]`(B는 계단형 하삼각행렬)를 이용, parity bit는 `pr = Sr ⊕ p_{r-1}` 재귀식(식7)으로 계산되며 `Sr = ⊕ i_z (z∈IN(r))`는 정보비트 그룹(360개 단위)의 XOR합.
4. 핵심 기법: FASPR(Fast Accumulate Semi-Parallel Recursive) 인코딩 - D개 병렬 정보비트가 들어올 때마다 관련 `Sr`을 갱신(`Sr = Sr ⊕ ic`)하고, `Sr`을 bit-inversion으로 단순화(`Sr=Sr̄` when `ic=1`)해 XOR 대신 반전 연산만 사용해 HW 오버헤드 절감.
5. `Sr`을 D열 분산 RAM 행렬(식10)로 저장, 정보비트 입력 완료 후 D개 parity bit를 자연순서(natural order)로 한 번에 산출(식11) - 기존 [14]는 non-natural order 출력이라 별도 인터리버 필요했던 것을 개선.
6. 병렬 BCH 인코더는 state-space 변환(`R(t+D)=F^D·(R(t)+M(t+D))`) 기반, D는 변조차수에 따라 결정(8PSK: D=3, 16APSK: D=4), 재구성 가능한 조합논리로 다양한 VCM 모드의 생성다항식 `F^D`를 MATLAB으로 사전계산.
7. 부수 트릭: dual-port RAM + distributed storage(v·D개 Sr RAM)로 latency 축소, ROM에 parity-check 초기주소 저장, 계층적 저장관리로 VCM 모드별 적절한 ROM 행 선택.
8. 별도 학습/최적화 절차 없음(고정 표준 코드 파라미터 기반 하드웨어 설계).
9. 결과: Xilinx XC7K325t FPGA에서 clock 389.5MHz, 인코딩 throughput 최대 `1.19Gb/s` 달성; CCM 대비 VCM 시스템 전체 데이터처리량 30.9% 향상; 기존 [9] 대비 LUT 41% 절감(단일모드 대비 3개 VCM 모드 지원).
10. 한계: 인코더 전용 설계로 디코더/복호 성능(오류정정능력, error-floor, iteration) 관련 내용 전무; SRAM 기반 FPGA라 방사선 환경 단일이벤트 대비 scrubbing+TMR 별도 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IRA-LDPC dual-diagonal(B행렬) 기반 recursive parity 계산 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코딩을 이미 보유해 개념적으로 중복되는 접근 |
| FASPR 병렬 accumulate(D-bit 단위 Sr 갱신) | `encoder.cpp` `Column_Reordering_Dual_Update()` | 병렬 처리·컬럼 재배치 아이디어는 관련 있으나 목적(다중 VCM rate 호환)이 상이 |
| VCM 모드별 재구성 가능 컨트롤러/ROM 주소테이블 | `ecc_top.cpp` `Load_PCM()` | 다중 부호rate/length 지원을 위한 H-matrix/주소 로드 구조와 유사한 개념이나 실존 대응은 아님 |
| BCH 외부코드 인코더 | 대응 없음 | Prime ECC는 BCH concatenation 없음 |

적용 가치: **낮음** — 인코더 하드웨어 자원 재사용·병렬화 기법은 흥미로우나 Prime ECC가 이미 dual-diagonal 인코딩을 보유하고 있어 개념 중복이며, 디코더/정정성능과 무관한 순수 인코더 논문이라 핵심 관심사(min-sum 성능, error-floor, throughput/면적) 대비 이식 가치가 낮음.
