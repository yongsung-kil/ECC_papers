> 총평: 자기기록/광기록 시스템의 run-length-limited·dc-free 변조부호와 LDPC를 결합하는 concatenation 구조 제안 논문으로, LDPC 자체 부호설계·디코더·HW 개선이 없고 NAND에는 존재하지 않는 채널 제약(run-length, DC 성분) 문제를 다뤄 이식 가치가 낮음.

### B. 알고리즘 요약

1. 대상 시스템은 자기/광 기록 채널(AWGN 가정)이며, ECC(LDPC)와 constrained code(run-length/DC 제약)를 함께 사용하는 recording system 구조. LDPC 부호 자체는 기존 문헌([9])의 (4182,4592) 및 (4428,4920)/(4096,4344) 코드를 그대로 사용.
2. 풀려는 문제: 기존 concatenation 방식은 복잡하거나(다중 constrained encoder), reverse concatenation 시 parity 구간의 modulation 복호에서 에러가 LDPC 디코더로 전파(error propagation)되는 문제.
3. 채널 모델: AWGN, `SNR = 10log10(Vp^2/σn^2 · R)`(R=전체 코드율), constraint 파라미터 k=1(nibble 내 최대 연속 제약).
4. 핵심 기법 1단: nibble replacement 기반 k-constrained 부호화 — all-0 nibble을 pivot nibble로 치환(주소+값을 pivot bit에 인코딩), all-1 nibble은 조건부 flip.
5. 구조: user 부분은 (1023,1024) NRE로, LDPC 인코더 출력의 parity 부분은 별도 (31,32) NRE로 각각 constrained 부호화 후 멀티플렉싱(reverse concatenation).
6. 핵심 기법 2단(확장): dc-free를 위해 user part는 guided scrambling(GS, 다항식 `g(x)`로 스크램블 후 최소 RDS 후보 선택), parity part는 (4,6) balanced modulation block code로 dc 억제.
7. 부수 트릭: (4,6) balanced code는 블록 단위 독립 인코딩이라 error propagation이 전혀 없음(최소 해밍거리 ≥2), 수신 시 Euclidean distance로 검출.
8. 학습/최적화 절차 없음 — 구조적 설계와 시뮬레이션 비교만 수행.
9. 결과: k-constrained LDPC 설계가 기존 RLL5051-LDPC([9]) 대비 코드율 0.9075(vs 0.8929)로 BER 개선; dc-free 설계는 DCRLL5054-LDPC 대비 BER 10⁻⁴에서 0.4~0.7dB 이득, PSD null −50dB 이상.
10. 한계: LDPC 디코더는 표준 sum-product 그대로 사용(개선 없음), HW 미설계·시뮬뿐, run-length/DC 제약이 있는 자기·광 기록 채널 전용 문제로 NAND read channel에는 해당 제약이 존재하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| k-constrained/dc-free 변조부호와 LDPC concatenation 구조 | 대응 없음 | Prime ECC는 NAND 컨트롤러 ECC 엔진으로 run-length/DC 제약 변조부호 계층이 존재하지 않음 |
| LDPC sum-product 디코더(기존 [9] 재사용, 개선 없음) | decoder.cpp `LDPC_Decoder()` | 표준 디코딩 알고리즘으로 이미 Prime ECC가 Min-Sum 버전을 보유, 신규 기여 없음 |
```

적용 가치: `낮음` — LDPC 부호/디코더 개선이 아니라 자기·광 기록 특유의 run-length/DC 제약 변조부호와의 concatenation 구조 설계이며, NAND에는 해당 채널 제약이 존재하지 않아 이식 대상이 아님.
