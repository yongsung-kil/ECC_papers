### B. 알고리즘 요약

1. 대상은 circulant-based(protograph) spatially-coupled(SC) LDPC 부호(`γ`=열가중치≥3, `κ`=행가중치, memory `m`, 결합수 `L`), PR(partial-response, 1-D 자기기록) 채널용, binary(GF(2))·non-binary(GF(4)) 모두 다룸, rate ≈0.73~0.81.
2. 기존 AWGN/Flash 채널용 OO-CPO 기법은 detrimental object(`(3,3(γ-2))` UAS)가 단일 패턴이지만, PR채널은 detector-decoder looping과 채널 메모리로 balanced absorbing set(BAS)이 지배적이며 `(4,4(γ-2))` UAS/UTS가 공통 서브구조라 기존 기법을 그대로 못 씀.
3. 채널 가정: 1-D MR PR 채널(equalization target `[8 14 2]`), BCJR 검출기(PDNP 기반) + FFT-QSPA LDPC 디코더, global iteration(검출-복호 루프) 10회, local iteration 20회.
4. 핵심 기법 1단(OO, optimal overlap): protograph 상에서 `(4,4(γ-2))` object를 만드는 9가지 서로 다른 패턴(`P1~P9`, 2~4 CN/VN 조합)을 정의하고, overlap parameter `t_{i1,...,iμ}` 기반으로 패턴 가중합 `F_sum = Σ β_P F_P`를 최소화하는 이산 최적화 문제를 풂.
5. 핵심 식 `(25)`: `f_c1,v1+f_c2,v2+f_c3,v3+f_c4,v4 ≡ f_c1,v2+f_c2,v3+f_c3,v4+f_c4,v1 (mod z)` — 이 합동식을 만족하는 circulant power 조합이 실제 length-8 cycle(=UAS)로 전개되므로, 이를 피하도록 파워를 조정.
6. 핵심 기법 2단(CPO, circulant power optimizer): OO로 정해진 partitioning 위에서 circulant power를 반복적으로 조정해 활성(active) cycle-8 후보 수를 최소화하는 휴리스틱 알고리즘(Algorithm 1), 내부연결(internal connection, 식 `(26)`,`(27)`)까지 고려해 원치 않는 저차 object 생성도 회피.
7. 부수 트릭: non-binary(GF(4)) 코드는 WCM(weight consistency matrix) 프레임워크로 추가 edge-weight 최적화, balanced partitioning 제약으로 특정 component matrix에 cycle이 몰리는 것을 방지.
8. 학습/최적화 절차: 이산 최적화(정수계획적 조합 탐색) + 그리디/휴리스틱 반복(Step 29 heuristic 선택)으로 circulant power 배정, GA/NN 등 학습기법은 없음.
9. 결과: `(4,4)` UAS 수를 uncoupled 대비 85~92%, 기존 CV기법 대비 61~72% 감소. FER 기준 기존 SOTA 대비 최대 약 3자리수 개선, 1.1dB SNR 이득. 동일 length/rate block code 대비도 최대 1.8자리수 개선(0.4dB), waterfall 영역서도 0.25dB threshold 이득.
10. 한계: PR 채널(자기기록) 특화 설계로 NAND flash 채널모델과 다름, HW(RTL/ASIC) 설계·복잡도·throughput 분석 전혀 없음, 검증은 시뮬레이션(FER~1e-9)까지만이며 실제 FPGA/ASIC 실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Protograph 기반 SC 부호의 H-matrix 구성(OO partitioning) | `ecc_top.cpp` `Load_PCM()` | 구조는 QC-LDPC와 유사(protograph+circulant)하나 partitioning/coupling 구조가 Prime ECC의 고정 QC-LDPC와 근본적으로 다름, H-matrix 재설계 필요 |
| (4,4(γ-2)) balanced absorbing set 최소화(circulant power 최적화) | 대응 없음 | error-floor 개선 목적은 Prime ECC 관심사와 일치하나, circulant power(shift) 최적화는 코드 생성 단계 개념으로 decoder.cpp 런타임과 무관, H-matrix 설계 시점에만 적용 가능 |
| WCM 프레임워크(non-binary edge weight 최적화) | 대응 없음 | GF(4) non-binary 전용 기법으로 Prime ECC는 binary 전용이라 직접 적용 불가 |
| PR채널 detector-decoder looping(global iteration) | `channel.cpp` | Prime ECC 채널모델(AWGN/RBER/fixed-error)과 PR 채널 특성이 달라 참고만 가능, 직접 이식 대상 아님 |

적용 가치: 중간 — SC 부호 자체 및 non-binary 확장은 Prime ECC(binary QC-LDPC) 구조와 거리가 있어 직접 이식은 어렵지만(§4 표 "부호설계=높음 난이도"), absorbing-set을 채널 특성에 맞춰 조합적으로 최소화하는 설계 철학과 patterns 분석 방법론은 향후 H-matrix 재설계 시 error-floor 개선 아이디어로 참고 가치가 있음.
