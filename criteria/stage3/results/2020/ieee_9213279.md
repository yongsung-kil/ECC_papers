### B. 알고리즘 요약

1. 대상 부호는 custom QC-LDPC (16200, 9000), circulant size `z=90`, DVB-S2 (16200,9000) 대비 row/col weight가 각각 12, 7로 더 큼.
2. 기존 GPU LDPC 디코더는 massively-codeword 병렬처리로 높은 throughput을 얻지만 codeword 패킷을 모아야 해서 지연시간이 수백 ms까지 증가, SDR(1ms 이하 요구)에는 부적합.
3. 채널 모델은 AWGN + BPSK, 디코딩 알고리즘은 normalized Min-Sum Algorithm(MSA), `Imax=20`.
4. 핵심 기법은 V2C/C2V 메시지를 원본 그대로가 아니라 구성요소(첫/두번째 최소 C2V 값, 최소 위치 인덱스, 부호)로 저장해 메모리 사용량을 줄이고 필요시 재계산.
5. 핵심 식: `Lj→i = αi→j · (submin(βi→j) if Li→j is min V2C, else min(βi→j))`, `Li→j = LAPP_i − Lj→i` — 메시지를 압축 표현에서 즉석 복원해 메모리 접근을 줄임.
6. CUDA 커널 4종(초기화, early termination/syndrome, APP update, hard decision)으로 디코더를 분할, `NCW × zG` 2차원 thread block grid 사용.
7. 구현 디테일: 각 순환행렬(circulant)당 z=90 thread지만 CUDA 요구상 32의 배수 필요해 96 thread 실행(6개는 idle), APP는 atomic addition으로 합산.
8. 별도 학습/최적화 절차 없음 (알고리즘은 표준 MSA, 최적화는 메모리 배치 설계 위주).
9. 결과: NVIDIA RTX 2080에서 latency 0.269~0.745ms, throughput 60.2~130.5Mbps 달성. 기존 codeword-parallel 전용 디코더(수백 ms) 대비 지연 대폭 감소.
10. 한계: 정정능력/error-floor 개선은 다루지 않음(순수 구현/지연시간 최적화), GPU 소프트웨어 구현으로 ASIC/FPGA HW 설계 없음, throughput은 기존 고throughput GPU 디코더 대비 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 알고리즘 (표준 MSA) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 표준 min-sum과 동일 개념, 신규 기여 없음 |
| C2V/V2C 메시지 압축 저장(min1/min2/부호/인덱스) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | GPU 메모리 대역폭 절감 기법으로 ASIC 온칩 메모리 설계와는 목적이 다름(HW 라우팅은 이미 z=32 기반 고정) |
| early termination (syndrome=0 체크) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념적으로 유사하나 논문은 syndrome 기반, Prime ECC는 CRC 기반으로 방식 상이 |
| QC-LDPC H-matrix (base+circulant, z=90) | `ecc_top.cpp` `Load_PCM()` | 특정 통신용 부호구조로 NAND rate/length와 무관, 대응 낮음 |
| GPU thread/block 병렬화 구조 | 대응 없음 | Prime ECC는 RTL bit-exact HW 시뮬레이터로 GPU 아키텍처 개념 자체가 대응 없음 |

적용 가치: **낮음** — 표준 min-sum을 GPU 소프트웨어 환경에서 메모리 최적화한 통신용(SDR) 논문으로, NAND ASIC ECC의 정정능력·HW 아키텍처 개선에 기여하는 신규 기법이 없음.
