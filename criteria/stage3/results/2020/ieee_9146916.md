### B. 알고리즘 요약

1. 대상은 binary quantum stabilizer code([[N,K]] code, Pauli 연산자 I/X/Y/Z 기반), 이는 GF(4) 위의 additive code로 취급되어 quaternary BP(BP4)로 복호됨.
2. 문제: 기존 BP4는 check-node당 O(q²) 복잡도로 binary BP2 대비 16배 느리고, Tanner graph의 짧은 사이클(특히 4-cycle)로 성능 열화.
3. 채널 모델: independent depolarizing channel, Pauli 오류확률 `p=(1-ε, ε/3, ε/3, ε/3)`.
4. 핵심 기법: 오류 syndrome이 binary라는 관찰에 기반해 δ-rule(likelihood difference)로 single-valued message를 전달하는 refined BP4(Algorithm 3)를 제안, 기존 BP4와 동일 출력을 내면서 check-node 효율 16배 개선.
5. 핵심 식: `d_mn = q⁰_mn - q¹_mn` (variable→check), `δ_mn = (-1)^zm Π d_mn` (check→variable) — 다치 메시지를 이진 스칼라로 축약.
6. 확장: serial schedule(변수노드 순차 갱신, Algorithm 4)이 parallel schedule 대비 사이클 많은 그래프에서 수렴성·error floor를 크게 개선([[5,1]] code는 parallel 발산·serial 수렴).
7. 부수 기법: message normalization(파라미터 α_c, α_v)과 offset(β)을 single-valued message에 적용해 error floor 추가 개선, α_v가 성능은 좋으나 복잡도 증가.
8. 최적화 절차 없음(학습 기반 아님), 시뮬레이션 기반 파라미터 스윕(α, β 값 탐색)만 수행.
9. 결과: [[256,32]] 및 [[800,400]] quantum bicycle code에서 normalization 적용 시 error floor 대폭 개선, serial BP4는 parallel 대비 평균 반복횟수 절반.
10. 한계: 전량 시뮬레이션(HW 미설계), quaternary/quantum 특화 알고리즘이라 binary classical LDPC로의 직접 적용은 이미 알려진 기법(§2 classical BP2)의 재탕 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Serial(shuffled) schedule | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 스케줄) | Prime ECC의 Dual-Update와 유사한 개념이나 이미 보유, 신규 기여 아님 |
| Message normalization(α) / offset(β) | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum scaling factor와 개념적으로 유사, 이미 min-sum 근사에 내재 |
| δ-rule / quaternary BP4 refinement | 대응 없음 | GF(4) quantum stabilizer 전용, binary QC-LDPC와 무관 |
| 조기종료(수렴판정 SUCCESS/FAIL) | `partial_CRC.cpp` `full_CRC.cpp` | 개념적 유사(신드롬 매칭 시 종료)이나 quantum 특화라 직접 이식 불가 |

> 적용 가치: **낮음** — 핵심 기여(quaternary BP 복잡도 절감)는 quantum code 전용이며, binary LDPC에 이식 가능해 보이는 요소(serial schedule, normalization)는 Prime ECC가 이미 보유한 기법과 개념적으로 중복.
