> 총평: polar code 전용 bit-flipping 재시도 기법으로, LDPC bipartite graph를 차용했으나 대상 코드·CRC 재인코딩 구조가 Prime ECC(binary QC-LDPC)와 근본적으로 달라 이식 가치 낮음.

### B. 알고리즘 요약

1. 대상은 polar code (N=256~1024, rate 0.5, 24-bit CRC 결합)이며, 채널은 AWGN + BPSK.
2. SC/SCL 디코딩은 처리량이 낮아, 병렬 처리 가능한 BP 디코딩을 사용하되 factor graph를 pruning해 LDPC와 유사한 희소 bipartite graph(L-BP)로 변환(선행연구[13] 기반).
3. 문제: 기존 L-BP는 BP보다 성능이 열세(~0.1dB 손실)하여 실사용에 부적합.
4. 핵심 기법: L-BP 1차 디코딩 후 CRC 실패 시, 신뢰도 낮은 VN 집합 Γ를 골라 LLR을 `Lv_init = +∞ (Lv<0) / −∞ (Lv≥0)`로 강제 플립한 뒤 재디코딩을 최대 Tmax회 반복(L-BPF, Fig. 2).
5. VN 식별법 2종: (a) CNI - 불만족 CN에 연결된 VN 집합 ΓCNI 사용(LDPC trapping-set 개념 응용), (b) HI - CNI가 stopping set(패리티는 만족하나 오류인 경우)에서 무력화되므로 출력 LLR 절대값이 작은 VN(ΓLLR)과 결합.
6. HI 방식은 |ΓCNI|가 Tmax보다 크면 ΓCNI 중 LLR 절대값 작은 순 Tmax개, 작으면 ΓLLR을 보충해 크기 Tmax 고정(Algorithm 1).
7. 부수 디테일: stopping set 발생 빈도를 (256,128) 코드에서 실측(Fig. 3, 고SNR에서 약 50%)하여 CNI 단독의 한계를 정량 제시.
8. 별도 학습/최적화 절차 없음(파라미터 Tmax는 그리드 실험으로 10/20 비교).
9. 결과: HI, Tmax=20에서 (1024,512) 코드가 FER=1e-4 기준 L-BP 대비 0.33dB 이득, lower bound에 근접. 평균 iteration은 고SNR에서 L-BP 수준(약 10~15회)으로 수렴하나 저SNR에서는 추가 라운드로 증가.
10. 한계: 순수 시뮬레이션(이론/시뮬 단계)이며 HW 구현·복잡도(gate/throughput) 분석 없음. 대상이 polar code이며 재인코딩(encoder 재호출)이 매 flip 시도마다 필요해 latency 부담 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| L-BP bipartite graph 메시지 전달 (BP 디코딩) | 대응 없음 | Prime ECC는 min-sum 기반이며 polar code용 BP 메시지 전달 구조 자체를 보유하지 않음 |
| CNI 기반 불만족 CN → 오류 VN 식별(trapping-set 유사) | `decoder.cpp` `LDPC_Decoder()` (조기종료용 syndrome 계산) | syndrome/불만족 check 개념은 공유하나 flip 재시도 로직은 대응 없음 |
| CRC 기반 조기종료/재시도 트리거 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 목적은 유사하나 L-BPF의 "실패 시 재시도" 루프는 Prime ECC 미보유 |
| 비신뢰 VN 재추정(LLR ±∞ 강제) | 대응 없음 | Prime ECC decoder.cpp의 LLR 테이블/양자화 갱신 로직과 개념적 유사성만 있음, 실제 대응 함수 없음 |

> 적용 가치: 낮음 — polar code 전용 알고리즘이며 binary QC-LDPC의 min-sum 파이프라인에 이식할 요소가 거의 없음.
