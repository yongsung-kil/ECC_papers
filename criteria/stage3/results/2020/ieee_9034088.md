### B. 알고리즘 요약

1. ATSC 3.0 LDPC는 length 16200(short)/64800(long), rate 2/15~13/15의 QC 구조(base q=360)이며 Type-A(raptor-like)와 Type-B(IRA) 두 구조를 가짐.
2. 기존 NMSA/OMSA/VS-NMSA는 PCM 전체에 단일(또는 iteration별 단일) 보정 factor `α`를 적용해 서로 다른 서브행렬(edge type)의 수렴 속도 차이를 무시하는 한계가 있음.
3. Protograph 기반 Gaussian Approximation DE(P-GADE)로 관찰하면 같은 PCM 안에서도 edge type(b,d)별 LLR 평균 수렴 속도가 서로 다름(예: Code A의 `m(u_{2,2})`가 가장 빠르게 수렴).
4. 제안 기법(MET-NMSA)은 check/variable node type 쌍 `(b,d)`마다 별도 정규화 factor `α_{b(m),d(n)}`을 곱함: `U_{mn} = α_{b(m),d(n)} · sgn(...) · min|V_{m'n'}|` (식 10).
5. Type-A는 3개 edge type((1,1),(2,1),(2,2)), Type-B는 3개 edge type((1,1),(1,2),(1,3))으로 분류해 각각 별도 factor를 둠(그림/표 구조 기반 분류).
6. 최적 factor는 MET-DE(수정된 density evolution)로 탐색: check-node step은 MSA처럼 type별로 edge 전체를 하나로 묶어 처리(식 12~14)하고, edge type별 PDF를 스케일링(식 15)해 PDF 진화를 추적.
7. Search space `(0,1]`, step 0.01로 factor를 탐색해 표로 정리(ST1/ST2/ST3 = α 값들).
8. 추가 저장 복잡도는 3(edge type 수), NMSA/OMSA는 1, VS-NMSA는 50(iteration별)로 MET-NMSA가 VS-NMSA보다 오히려 저장 비용이 낮음.
9. FER=10^-4 기준 NMSA 대비 최대 1.53dB, OMSA 대비 0.21dB, VS-NMSA 대비 0.35dB 개선. 이론(DE) 임계값과 실제 시뮬 gap은 0.79dB 이내로 일치.
10. HW 구현/합성 결과는 없고 AWGN·QPSK/64QAM 시뮬로만 검증됨. 특정 표준(ATSC 3.0) PCM 구조에 종속적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MET-NMSA (edge-type별 정규화 factor) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 갱신에 곱해지는 scaling factor를 단일값 대신 edge/degree-type별로 세분화하는 방식으로 대응 가능 |
| iteration별 LLR 테이블 참조 구조 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존에도 iteration 구간별 LLR threshold 테이블을 두고 있어 유사한 구조 확장 여지 존재 |
| MET-DE (factor 탐색용 density evolution) | 대응 없음 | Prime ECC는 오프라인 factor 최적화 도구를 프로파일 상 명시하지 않음(local_opt.cpp가 LLR 자동최적화이나 DE 기반은 아님) |
| Type-A/B PCM 구조(raptor-like/IRA) 분류 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 시 서브행렬 구조 구분이 필요하나 Prime ECC는 특정 QC-LDPC 고정 구조라 재설계 부담 큼 |

적용 가치: **중간** — edge-type별 min-sum scaling factor 세분화 아이디어는 `decoder.cpp`의 CN 갱신 로직에 낮은 난이도로 이식 가능하나, ATSC 3.0 방송채널 전제·HW 미검증·factor 탐색을 위한 별도 DE 툴체인 필요가 걸림돌.
