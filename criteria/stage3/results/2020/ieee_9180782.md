### B. 알고리즘 요약

1. IEEE 802.11n/ac/ax(WiFi) 표준의 QC-LDPC 부호(rate 1/2, 2/3, 3/4, 5/6, frame length 648/1296/1944, sub-block size Z=27/54/84)를 대상으로 layered(turbo) 디코딩 스케줄 기반 VLSI 디코더를 설계.
2. 문제: 최대 Z(84)에 맞춰 barrel shifter/역shifter를 설계하면 짧은 프레임에서 면적이 과도하게 낭비되고, CNU 입력수도 base-matrix 컬럼수(24)에 맞춰져 있어 실제 최대 row-degree(22)보다 커 비효율적.
3. Check node 갱신은 offset min-sum 근사 `Lcv = max(min|Qv'c| - β, 0) * sign(Qv'c)` (β는 경험적 오프셋 상수)를 사용.
4. 핵심 기법 1: 긴 프레임(ZL=3×ZS 등)을 최소 프레임 barrel shifter(크기 ZS)로 여러 사이클에 나눠 회전시키는 `mc, ms`(수정 사이클/shift값) 생성 규칙 `mc=(cn+r) mod ct` 제시 — barrel shifter 개수를 48→그대로 유지하되 크기를 최소화.
5. 이 방식은 처리량을 긴 프레임에 대해서만 낮추되(최소 프레임 처리율로 제한), 면적을 크게 절감하는 트레이드오프.
6. 핵심 기법 2: LLRMemory에서 base-matrix 컬럼(24개) 중 실제 최대 row-degree(22개)만 필요하므로, 소규모 mux로 출력라인을 24→22로 축소해 adder/subtractor/barrel shifter/CNU 입력수를 줄임.
7. 부수 기법: 초기 LLR을 `Pv(0)=rv/σ²`로 수정(기존 `2rv/σ²`에서 계수 2 제거)해 CheckNodeUnits/CNMem 비트폭을 5비트→4비트로 절감, 그 결과 saturated subtractor 22개를 unsaturated subtractor로 대체해 비교/클리핑 연산 제거.
8. 학습/최적화 절차 없음(고정 규칙 기반 HW 최적화).
9. 결과: 40nm CMOS 합성 결과 면적 0.71mm², 클럭 562MHz, throughput 3.8~11.4Gbps(rate별), 에너지효율 12.5pJ/bit — 기존 최고 대비 throughput/area 13.4Gbps/mm², 에너지효율 12.5pJ/bit로 최우수. BER curve는 Imax=8, β=1로 시뮬레이션됨(Fig.6, Fig.7).
10. 한계: 정정성능(BER/error-floor) 개선 기법은 전혀 없고 순수 HW 최적화(면적/전력/throughput)에 집중; 802.11 특유의 3-frame-length 배수 구조에 특화되어 일반 QC-LDPC로의 일반화는 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 보유한 표준 min-sum 기법과 동일 계열, 중복 기여 |
| layered decoding 스케줄 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Dual-Update(even/odd 교대) 방식을 이미 보유, layered와 유사한 조기수렴 목적이나 스케줄 방식 다름 |
| barrel shifter 멀티사이클 공유 기법 | 디코더 HW 모델(z=32, pipeline 6-stage) | Prime ECC는 z_sb=32 고정 단일 병렬단위이며 다중 프레임 길이 지원 구조가 아니라 직접 대응 없음. 다만 GT(`GT.cpp` `Make_GT_HW()`)의 MUX 기반 edge 선택과 유사한 HW 절감 아이디어 |
| 최대 row-degree 기반 mux 축소 | `decoder.h` `Get_VNU_*`, HCU(`decoder.cpp` Half Column Update) | Prime ECC는 이미 비대칭 degree column(127,128)을 HCU로 2-cycle 분할 처리하는 유사 최적화를 보유 — 개념적으로 중복되나 mux 축소 방식 자체는 참고 가치 있음 |
| LLR 비트폭 절감(계수 2 제거) | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | Prime ECC는 이미 HD/2SD/3SD 별도 LLR 테이블을 사용 중이라 비트폭 최적화 아이디어는 참고 가능하나 직접 이식 시 코드 검증 필요 |

> 적용 가치: 중간 — 정정성능 개선은 없으나 barrel shifter 공유, mux 축소, 비트폭 절감 등 HW 면적/전력 최적화 기법이 Prime ECC의 디코더 HW 모델(pipeline, GT, HCU)과 유사한 방향성을 가지므로 구현 디테일 참고 가치는 있음.
