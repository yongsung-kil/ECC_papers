### B. 알고리즘 요약

1. 대상은 `(dv, dc)`-regular QC-LDPC의 base matrix `P`(size `dv x dc`), circulant permutation `I(p_i,j)`로 구성된 parity-check matrix `H`, girth 8을 목표로 함.
2. 기존 확률적 방법(Simulated Annealing, Hill-Climbing, Guess-and-test)은 최소 lifting factor `z_min` 존재 여부를 보장할 수 없다는 문제를 해결하려 함.
3. Fossorier의 cycle 존재 조건식(`Σ(p_ik,jk - p_ik+1,jk) = 0 mod z`, `2 ≤ l ≤ g/2-1`)을 기반으로 girth 조건을 판정.
4. 핵심 기법은 PEG 유사 forward-backward exhaustive search(SES): 두 번째 열부터 마지막 열까지 원소별로 가능한 shift 값 리스트를 유지하며 순방향/역방향으로 이동.
5. 각 원소에 대해 "허용되지 않는 상대/절대 shift 값 리스트"를 실시간 갱신하여 cycle을 만드는 값을 제외한 뒤 가장 작은 가용값을 선택(`p1,1 = min(p_i,j)` 등 정규화 제약식 (3)-(7), (10)으로 탐색 공간 축소).
6. 탐색은 마지막 원소까지 값을 채우면 성공(해 존재), 두 번째 열 두 번째 원소에서 더 이상 값이 없으면 실패(해 부존재)로 종료 — 확률적 방법과 달리 존재/부존재를 확정적으로 판별.
7. 탐색 공간 축소를 위해 대칭성 제약(행/열 재배열로 오름차순 정렬)과 6-cycle 방지를 위한 shift 값 중복 금지 조건을 결합, 조합 수를 이론상 최대 4자리수 이상 감소시킴(예: (3,6) matrix z=18: 2.0×10^12 → 2.7×10^8).
8. 별도 학습/최적화 절차는 없음(exhaustive search 기반, 파라미터 튜닝 없음).
9. 결과: (3,4)~(3,12), (4,5)~(4,12), (5,6)~(5,12) 행렬에서 기존 대비 동일하거나 더 작은 circulant size 발견, 예: (3,12) z=42(기존 대비 최소), (4,6) z=21; 탐색 시간도 기존 SA 대비 대폭 단축(z=49 탐색: 164초→1초).
10. 한계: 복호 성능(BER/FER) 시뮬레이션 전혀 없음, HW 구현 없음, 단일 스레드 구현으로 큰 행렬은 최대 24시간 소요, girth 10/12로의 확장은 future work로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-8 regular QC-LDPC base matrix 구성/탐색 | `ecc_top.cpp` `Load_PCM()` | 논문은 새 H-matrix를 오프라인 생성하는 도구이며, 생성된 base matrix를 로드하는 지점만 대응(구성 알고리즘 자체는 이식 대상 아님) |
| 최소 lifting factor 결정 | 대응 없음 | Prime ECC는 z_sb=32 고정 병렬 단위를 사용하며 lifting factor 최소화 문제와 무관 |
| girth 8 cycle 판정식 | 대응 없음 | H-matrix 설계 시점의 오프라인 이론 조건, 런타임 디코더/인코더 코드에 대응 없음 |

> 적용 가치: 낮음 — 순수 오프라인 코드 구성 알고리즘으로 Prime ECC는 특정 QC-LDPC 구조가 이미 고정되어 있어 재검증 부담이 크고(§4 "부호설계=높음 난이도"), 복호 성능/HW 실험이 전무해 이식 근거가 부족함.
