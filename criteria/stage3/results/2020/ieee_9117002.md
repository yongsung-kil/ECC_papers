### B. 알고리즘 요약

1. 대상은 일반 binary linear code(LDPC 포함)의 fundamental polytope 기반 ML(최대우도) 디코딩 문제; AWGN+BPSK 채널, `(96,48)` MacKay, `(575,288)` IEEE 802.16e, `(2640,1320)` MacKay 코드로 검증.
2. 기존 LP 디코더는 error floor가 낮고 이론적 보장이 강하지만 계산복잡도가 높고 저 SNR에서 성능이 떨어지는 문제, 특히 ADMM 기반 디코더의 check polytope projection(CPP) 연산이 가장 시간 소모적임.
3. 채널모델: memoryless binary-input symmetric-output channel, LLR `v_i = log(Pr(y_i|x_i=0)/Pr(y_i|x_i=1))`.
4. 핵심 기법1: PDD(penalty dual decomposition) 프레임워크로 `min v^T x s.t. P_j x ∈ PP_dj` 문제를 보조변수 `{x̂, z}`와 dual variable `{y,w,η}`로 재구성해 outer loop(dual/penalty 갱신)-inner loop(BCD로 x/z/x̂ closed-form 갱신) 2중 루프로 해결.
5. 핵심 식(12): `z_j^{k+1} = Π_PPdj(P_j x^k + y_j/μ_m)`가 CPP(check polytope projection) 연산이며 각 반복마다 필요해 전체 지연을 좌우함.
6. 핵심 기법2(NCPP): 기존 ICPP(반복적 CPP, 정렬 불필요)의 반복횟수를 줄이기 위해 3층 MLP(CPP-net, `SinAct` 활성함수 사용)로 초기 추정치 `s̃`를 예측 후 잔여 반복만 수행.
7. 구현 디테일: CPP-net 가중치를 `{0,±1,±2^k}`로 양자화해 곱셈을 비트 시프트로 대체(예: C1 코드 가중치 `[0,0,0,-2,0,0;...]`), `SinAct`는 LUT로 구현 가능.
8. 학습: PDD 디코딩 중 수집한 `(v_p, ŝ_p)` 샘플로 CPP-net을 TensorFlow/Adam으로 학습, loss는 `(ŝ_p-s̃_p) + κ‖ŝ_p-s̃_p‖²` (초기 추정치가 더 크도록 유도).
9. 결과: BLER에서 ADMM-2 대비 C1/C2/C3 각 0.3dB/0.1dB/0.08dB 개선(BLER=10⁻⁴), NCPP는 ICPP 대비 평균 반복횟수 거의 절반으로 감소.
10. 한계: 순수 시뮬레이션(HW 미설계), 짧은/중간 길이 코드만 검증, LP/ADMM/PDD는 min-sum과 반복 구조·수학적 기반이 전혀 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PDD/ADMM 기반 ML 디코딩 전체 | 대응 없음 | Prime ECC는 min-sum 기반 메시지 패싱만 지원, LP relaxation 디코더 구조 자체가 없음 |
| CPP(check polytope projection) 연산 | 대응 없음 | min-sum의 CN 연산(min1/min2)과 수학적으로 무관한 정렬/투영 연산 |
| NCPP의 MLP 근사(가중치 양자화, SinAct LUT) | 대응 없음 | NN 디코더 보조 기법으로, Prime ECC 프로파일상 "NN/딥러닝 디코더"는 대응 없음으로 명시됨 |

> 적용 가치: 낮음 — LP/ADMM/PDD 기반 ML 디코딩과 신경망 보조 CPP 근사로, Prime ECC의 min-sum QC-LDPC 디코더 구조와 패러다임이 근본적으로 달라 이식 불가.
