### B. 알고리즘 요약 (10줄 내외)

1. 5G NR LDPC는 core LDPC 부분과 LDGM(확장, degree-1 VN) 부분이 결합된 protograph 기반 부호(BG1: `Nv≤68`, BG2: `Nv≤52`, lifting factor `Zc≤384`)로 puncturing을 통해 rate `R=K/(Nv-2)`를 유연하게 지원.
2. 기존 2D EXIT chart 분석은 degree-1 확장 VN 때문에 `Ive=1`로 수렴하지 못해 NR LDPC의 실제 낮은 BLER 동작과 모순되는 문제가 있음(closed tunnel인데 실제로는 잘 복호됨).
3. 이를 해결하기 위해 factor graph를 core VN/CN, extension VN/CN 4개 그룹으로 재구성하여 MI 교환을 분리 모델링.
4. 핵심 기법: core/extension 그룹별 EXIT 함수 `Ivec(Ivac,Ivae,IH)`, `Ivee(Ivac,Ivae,IH)`, `Icec(Icac)`, `Icee(Icae,IH)`를 정의하는 3D EXIT chart 제안(식 (3)~(6)).
5. LDGM part(extension)는 채널 SNR에 직접 영향받아 몇 회 iteration 내 조기 수렴하고 이후 기여가 없어짐 - 이 관찰이 스케줄링 설계의 근거.
6. 확장 노드 수렴 후 core VN/extension CN/extension VN을 결합한 "combined VN decoder"로 취급하는 2D EXIT projection도 추가 제안(계산량 감소, 시각화 단순화).
7. HW 친화적 구현 디테일: iteration 활성화 순서를 오프라인에서 (SNR, rate)별 LUT로 미리 계산 - extension CN이 완전수렴치의 99.99%에 도달하면 이후 iteration에서 비활성화(core-only mode).
8. 별도 학습 절차 없음 - LUT는 3D EXIT chart 오프라인 시뮬레이션으로 채움. 99%/99.9%/99.99% threshold 비교 실험으로 99.99%가 최적 trade-off로 선정.
9. 결과: 동일 복잡도에서 flooding 디코더 대비 약 0.2~0.3dB BLER 이득, 동일 BLER 기준 약 14~23% 복잡도(ACS 연산 수) 감소. 단, 고rate(R=2/3)·저SNR에서는 이득이 줄어들거나 error floor 우려.
10. 한계: 순수 시뮬레이션(AWGN/Rayleigh fading)뿐 HW 구현 없음, 5G NR 특유의 core+LDGM 구조 전제(고전 binary QC-LDPC와 부호구조 상이)이며 SPA(sum-product) 기준 분석 - min-sum 변형 적용은 future work로 언급만 됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| iteration별 컴포넌트 활성화순서 최적화(조기 CN 비활성화로 복잡도 절감) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | EXIT 궤적 대신 다른 기준(LLR threshold 등)이지만 "일부 노드 조기 비활성화로 연산 절감"이라는 스케줄링 아이디어는 Dual-Update 스케줄과 유사한 층위에서 참고 가능 |
| SNR/rate별 활성화 스케줄 LUT | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration 구간별 파라미터 테이블 구조와 개념적으로 유사(적응형 LLR 테이블과 같은 층위) |
| core LDPC + LDGM 이종 부호 구조 / degree-1 확장 VN | 대응 없음 | Prime ECC는 순수 QC-LDPC(base+circulant)만 지원, LDGM 확장 구조 없음 |
| 3D/2D EXIT chart 분석 기법 자체 | 대응 없음 | 코드베이스는 bit-exact 시뮬레이터로 EXIT 이론 분석 도구 없음 |

> 적용 가치: **중간** — 부호구조(core+LDGM)는 이식 불가하나, "EXIT 수렴 관찰 기반 조기 컴포넌트 비활성화로 평균 iteration/복잡도 절감"이라는 스케줄링 원리는 Prime ECC의 min-sum 디코더 이터레이션 루프에 개념적으로 참고할 여지가 있음.
