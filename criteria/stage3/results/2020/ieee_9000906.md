### B. 알고리즘 요약

1. MLC NAND(2bit/cell) 채널, programming/RTN/data-retention noise로 인한 threshold voltage shift(offset)를 다룸. LDPC 부호는 rate 0.93 regular(5,69) N=8832,K=8196 및 rate 0.90 irregular N=4544,K=4096 두 종류로 검증.
2. 기존 read threshold 설계(MMI, CR 등)는 채널의 정확한 모델/파라미터를 요구하나, data retention offset은 예측 불가하여 실채널에서 성능 저하.
3. 채널은 programming noise + RTN + retention noise 혼합 Gaussian mixture로 가정(§II), CCI는 predistortion으로 이미 제거되었다고 가정.
4. 1단: GRU 2층 RNN이 readback voltage 시퀀스 `v`로부터 심볼 soft 추정 `x̃`를 산출(비지도 채널모델 없이 LDPC 디코더가 유효 복호한 codeword를 학습 라벨로 사용).
5. 핵심 식: hard threshold `{a1*,a2*,a3*} = argmin d(X̄,X̂)`(RNN 출력과 threshold 적용 hard 추정 간 Hamming distance 최소화), DP로 O(m²n) 해결.
6. 2단: RNN이 도출한 3-level threshold 주변에 6-level soft threshold `b*_{2i-1}=a_i*-W_i/2, b*_{2i}=a_i*+W_i/2`를 정의, 정수 기반 reliability mapping(Table II)으로 채널지식 없이 LLR 근사.
7. RNN은 idle 상태에서만 주기적으로 활성화(수개월 1회)하여 read latency/전력 절감 - "RNNA dynamic threshold detector"로 명명.
8. W1,W2,W3 최적화는 DDE(discrete density evolution, NMS 기반) + differential evolution(global opt, cost=Pe(l))으로 LDPC 복호성능 기준 수행.
9. 결과: RNNA quantizer의 채널 MI가 완전 채널지식 가정한 MMI quantizer에 근접, LDPC-coded BER도 MMI 대비 유사 성능 확보(P/E cycle mismatch ≤2000, retention mismatch ≤3개월까지 강건).
10. 한계: 알고리즘/시뮬레이션 수준 검증뿐(FPGA/ASIC 없음), 디코더는 표준 NMS 그대로 사용, RNN 자체의 HW 구현 비용 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft-read threshold(6-level) 설계 및 LLR 정수 매핑 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read 비용 절감과 직결되는 threshold/LLR 테이블 설계 아이디어로 참고 가능 |
| RNN 기반 hard threshold 동적 추정 | 대응 없음 | Prime ECC는 NN 기반 채널 추정 모듈 없음, 이식 시 신규 컴포넌트 필요 |
| LDPC 부호 특화 DE+차분진화 threshold 최적화 | `ecc_data.h` `PARAM_LLR` (iteration별 LLR 테이블) | 디코더 알고리즘 변경 없이 LLR 테이블 값 산출 방법론만 참고 가능 |
| 디코딩 알고리즘(NMS) | `decoder.cpp` `LDPC_Decoder()` | 논문은 표준 NMS 그대로 사용, 신규 기여 없음(이미 보유 기법과 중복) |

> 적용 가치: 중간 — 디코더 알고리즘 개선은 없으나, 채널 offset에 강건한 soft-read threshold/LLR 재설계 아이디어는 Prime ECC의 `channel.cpp` LLR threshold 테이블 갱신 전략에 참고할 여지가 있음. 단 RNN 자체 HW 이식은 별도 검토 필요.
