### B. 알고리즘 요약

1. 대상: 3D charge-trap TLC NAND flash (LSB/CSB/MSB 3개 논리 페이지), 실측 code rate 0.80/0.89, normalized min-sum 디코딩(Matlab 시뮬).
2. 문제: 3D NAND는 P/E 사이클·retention 증가에 따라 RBER이 커져 LDPC 디코딩 반복횟수·지연이 증가.
3. 핵심 가정/모델: 실제 FPGA 테스트 플랫폼(NVMe/PCIe)에서 P/E 0~6000, retention 1~4주 조건으로 LSB/CSB/MSB 페이지별 1→0, 0→1 bit-flip 에러율을 실측.
4. 관측1(비대칭성): LSB 페이지의 1→0 에러율이 CSB/MSB 및 다른 방향(0→1) 대비 압도적으로 높고(P/E 6000, retention 1주에서 1.2×10⁻², 4주에서 1.6×10⁻²) P/E·retention 증가에 따라 급격히 상승; 나머지 조합은 대부분 2×10⁻³ 미만.
5. 관측2(temporal locality): 인접 retention 기간 간 반복적 에러 위치 비율이 LSB에서 특히 높음(최대 90%, P/E 5000~6000 부근), CSB/MSB보다 뚜렷.
6. 핵심 기법1(BeLDPC 인코딩, Algorithm 1): P/E·retention이 임계값(thr1, thr2) 초과 시 LSB는 저rate `R1`, CSB/MSB는 고rate `R2`로 인코딩하여 반복횟수 감소를 유도.
7. 핵심 기법2(BeLDPC 디코딩, Algorithm 2): LSB 페이지는 hard-decision 후 bit=1일 때 `L=M`, bit=0일 때 `L=-N`(M>N, 양의 정수)으로 비대칭 LLR 가중치 부여(1→0 에러가 지배적이므로); CSB/MSB는 대칭 가중치(`±Q`, `±P`) 사용, 이후 normalized min-sum으로 디코딩.
8. 결과: rate 0.89 대비 rate 0.80(LSB, 저rate)이 반복횟수 더 적고 완만히 증가; rate 0.89는 P/E>5000·retention 3~4주 부근에서 반복횟수 급증, P/E=6000에서 정정 실패(최대 반복 도달).
9. LLR 가중치 실험(Fig.4): (M,N) 조합(예: (4,-1))별로 반복횟수 변화, retention 1주·rate 0.89 및 retention 2주·rate 0.80 조건에서 적절한 LLR 파라미터 선택 시 LSB 반복횟수 감소 확인.
10. 한계: 실제 부호(H-matrix/QC 구조) 상세 미기재, HW 구현 없이 Matlab 시뮬레이션 수준, rate 전환 임계값·LLR 파라미터 최적화 절차(탐색법) 불명확, 부호length 미기재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 페이지(LSB/CSB/MSB)별 adaptive code rate 선택 | `ecc_top.cpp` `Load_PCM()` | RBER/retention 조건에 따라 다른 rate의 H-matrix를 선택 적용하는 아이디어로 참고 가능, 실제 부호 전환 메커니즘 구현 필요 |
| 비대칭 LLR 가중치(1→0 에러 지배적 반영) | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | NAND 실측 비대칭 에러 특성을 LLR 초기값/threshold에 반영하는 기법으로 직접 이식 가능성 높음, soft-read 비용 절감과 연결 |
| normalized min-sum 디코딩 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min-sum 계열 보유(scaling/normalized factor는 코드상 명시 미확인) - 개념 중복이나 비대칭 가중치 결합은 신규 |
| temporal locality(반복 에러 위치) 활용 | 대응 없음 | 본 논문은 인코딩/LLR 설계에만 활용, 위치 정보 기반 디코더 후처리(예: bit-flipping 보조)는 미제안 - Prime ECC에 직접 대응 모듈 없음 |

적용 가치: **높음** — NAND flash 실측 비대칭 비트에러 특성을 LLR 초기값/adaptive rate로 직접 반영하는 기법은 Prime ECC의 `channel.cpp` LLR 테이블 설계 및 `ecc_top.cpp`의 H-matrix 선택 로직에 낮은 난이도로 이식 가능하며 NAND 직접 타깃 연구로 관련성이 매우 높음.
