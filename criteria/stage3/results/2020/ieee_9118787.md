> 총평: Layered LDPC 디코더의 single-port SRAM 메모리 접근 시 발생하는 파이프라인 hazard(RAW)를 오프라인 알고리즘(그래프 컬러링+유전 TSP+in-layer fine-tuning)으로 해소해 하드웨어 사용효율과 iteration duration을 개선하는 순수 HW 아키텍처/스케줄링 논문으로, NAND flash LDPC 디코더 문헌([5] Ho et al. 2016)을 직접 인용하며 QC-LDPC 병렬 디코더 공통 문제를 다뤄 이식 관점에서 참고 가치가 높음.

### B. 알고리즘 요약

1. 시스템: QC-LDPC 코드(base graph `B`, expansion factor `z`)의 layered scheduling decoder, `z`개 처리유닛(processing unit)이 base matrix 한 행(=check-node)씩 처리.
2. 문제의식: single-port SRAM 기반 AP-LLR(A-Posteriori LLR) 메모리는 iteration당 `dv(n)`회 읽히므로, 서로 다른 처리유닛이 같은 메모리 뱅크의 메시지를 동시 요청하면 충돌(stall)이 발생 — 이를 회피하는 오프라인 메시지 매핑·스케줄링이 필요.
3. 핵심 모델: layered decoding에서 각 layer는 check-node `m`에 대응, check-node 메시지와 채널 LLR을 합산한 AP-LLR을 저장해 `variable-node read = AP-LLR read - 이전 check-node 기여분 subtract`로 처리.
4. 핵심 기법 1단(메모리 매핑): variable-node coloring 문제로 추상화(hypergraph edge coloring), 수정된 Misra-Gries 휴리스틱(다항시간) + 유전 알고리즘(GA) 후처리로 뱅크 배정.
5. 핵심 기법 2단(layer 순서): check-node를 layer에 매핑하는 순서 `σ`를 유전 알고리즘 기반 Traveling Salesman Problem(TSP) 풀이로 결정, 이후 in-layer fine-tuning으로 각 AP-LLR read 클록 사이클(`ψ`)을 최종 확정.
6. 핵심 식: `rdF_P(m_last,n) = max{rdF(m_last,c), wb(m_prev,n)+1}` (RAW 회피를 위한 최초 가능 읽기 시각), `rdF(m,c) = max_idx(r_idx - idx + 1)` (color별 layer 시작 클록 결정).
7. 메트릭: Hardware Usage Efficiency `HUE(G,σ) = Σ dc(m) / (I_G · n_banks) × 100` — 유효 AP-LLR 읽기 접근 비율.
8. 최적화 절차: 오프라인(설계시간) 알고리즘으로 Misra-Gries + GA coloring → GA-TSP layer 순서 → Algorithm 1(order_layer) fine-tuning(3단계 파이프라인, 모두 사전 계산되어 ROM에 저장).
9. 결과: in-order write-back 정책 기준 WiMAX rate 3/4(length 2304) 코드에서 최소 latency(2 clock) 조건 시 HUE 25~66% 개선, DVB-S2X rate 140/180(length 64800) 코드에서 10~47% 개선(모두 최적화 미적용 대비).
10. 한계: ASIC/FPGA 합성이나 실측 없이 시뮬레이션 기반 HUE/레지스터 파일 수 평가에 그침, 정정능력(BER/error-floor)에는 영향 없는 순수 HW 스케줄링 최적화이며 GA 런타임(최대 2시간)이 오프라인 비용으로 소요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered decoding, check-node/layer 순서(σ), Dual-Update 유사 스케줄링 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 옵션) | Prime ECC의 parity even/odd 교대 스케줄과 본 논문의 layer 순서 최적화는 같은 파이프라인 hazard 회피 목적으로 직접 비교·이식 검토 가치 높음 |
| AP-LLR 메모리 뱅크 매핑(variable-node coloring), single-port SRAM 충돌 회피 | 디코더 HW 모델 (z=32, HCU, GT) | z_sb=32 병렬 처리유닛 구조에서 동일한 메모리 뱅크 충돌 문제가 발생할 가능성이 있어, 본 논문의 오프라인 coloring 알고리즘을 HCU/GT의 뱅크 설계 검토에 참고 가능 |
| in-layer fine-tuning(RAW hazard 회피 read 스케줄, Algorithm 1) | `decoder.cpp` `C2V_Cal()`, pipeline 6-stage(col_M1/Z/P1~P4) | 6-stage pipeline에서의 read-after-write 처리 방식과 직접 비교 대상 — 이식 시 파이프라인 스톨 감소 효과 기대 |
| HUE 메트릭(메모리 접근 효율) | `ecc_top.cpp` (통계 수집) | 새로운 평가지표로 Prime ECC HW 효율 분석에 참고 가능하나 기존 통계 모듈에 대응 없음 |

> 적용 가치: 높음 — decoder.cpp의 파이프라인/Dual-Update 스케줄링과 직접 맞닿는 메모리 접근 hazard 최소화 기법으로, min-sum 알고리즘 자체는 건드리지 않으면서 순수 HW 효율(latency/throughput)을 개선하는 이식 난이도 낮은 레이어(디코더 이터레이션 루프)에 해당.
