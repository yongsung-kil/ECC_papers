### B. 알고리즘 요약

1. AWGN 채널, BPSK 변조, quantized(5-bit uniform, Δ=0.15, max=2.25) MSA 기반 LDPC 디코더 대상.
2. MSA는 error floor에서 AMSA/OMSA보다 낫지만 waterfall에서 열세; AMSA/OMSA는 반대로 waterfall은 좋으나 error floor가 높아 trade-off 존재.
3. 핵심 가정: error floor는 trapping set/absorbing set 같은 problematic graphical object가 주원인이며, 해당 object 내부 check node는 고신뢰 LLR로 빠르게 성장하지 못해 구분 가능.
4. check node cj가 이웃 VN 중 최소 크기 `min|Vij| = M1,j ≤ τ`이면 cj가 problematic object 내부일 가능성이 높다고 보고 그때만 attenuation(TAMSA, `α≤1`) 또는 offset(TOMSA, `β>0`)을 선택적으로 적용, 그 외에는 순수 MSA(식 2) 그대로 사용.
5. 식 (7)/(8): `M1,j > τ`이면 CN LLR을 그대로 max까지 키워 신뢰 정보로 활용하고, `M1,j ≤ τ`이면 attenuate/offset하여 problematic object 내 오류 LLR의 과도한 성장(오판 강화)을 억제.
6. τ=max이면 TAMSA=AMSA, α=1(또는 τ=0)이면 TAMSA=MSA와 동일 — 즉 AMSA/MSA를 포함하는 일반화된 알고리즘.
7. 파라미터 선택은 code simulation 대신 [18]의 code-independent 성능추정(주어진 problematic object에 대한 FER lower bound)으로 (α, τ) contour를 그려 최적점 탐색(예: (5,3) absorbing set에 최적 (α=0.65, τ=1), waterfall-error floor 균형점 (α=0.8, τ=1.5)).
8. 학습 절차 없음(closed-form/estimation 기반 파라미터 튜닝). dominant problematic object는 시뮬레이션 오류 프레임 통계에서 식별.
9. 결과: (8000,4000) LDPC-BC에서 MSA/AMSA/OMSA 대비 error floor 1~2 orders 개선(waterfall 동일 유지); (155,64) Tanner code에서 약 1 order; SC-LDPCC 적용 시 최대 5 orders(3dB 이상) 개선. Iteration 수는 AMSA와 거의 동일(저SNR에서 MSA 대비 감소, 고SNR에서 수렴).
10. 한계: HW 구현/게이트카운트 없음(시뮬 전용), 임의 code에 대해 dominant problematic object를 알아야 최적 파라미터 도출 가능(모르면 검색 필요), 검증 코드는 (8000,4000) random-like, PEG, QC-Tanner(155,64) 등 일반 LDPC이며 NAND 특화 실험 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TAMSA/TOMSA check node 임계값 비교·선택적 attenuation/offset | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | M1,j 계산 후 τ 비교·mux 추가로 삽입 가능한 CN 연산 확장 지점 |
| M1,j/M2,j 최소값 탐색(기존 min-sum 구조 재사용) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 M1,j를 사용 중이므로 τ 비교 로직만 추가하면 됨 |
| iteration별 attenuation/offset 파라미터(α, τ, β) 튜닝 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 적응형 LLR 테이블 구조에 τ 임계값 필드 추가로 확장 가능 |
| error floor용 problematic object 기반 파라미터 추정(오프라인) | 대응 없음 | 코드에 별도 offline 최적화 도구 없음(local_opt.cpp는 LLR 자동최적화이나 object 기반 방법론과는 다름) |

적용 가치: **높음** — 기존 min-sum CN 유닛에 threshold 비교기 1개 + mux만 추가하면 되는 저비용 변형으로, error-floor 개선이 필요한 Prime ECC의 min-sum 경로에 바로 실험 가능.
