### B. 알고리즘 요약

1. 시스템은 Gaussian wiretap channel(Wyner 모델)로, 송신자 Alice가 정당 수신자 Bob(양호한 AWGN 채널)과 도청자 Eve(열악한 AWGN 채널)에게 동시에 신호를 전송하는 구조이며 BPSK 변조를 사용한다.
2. 풀려는 문제: 기존 punctured LDPC 보안 스킴(message bit 천공)은 security gap(Bob과 Eve의 필요 SNR 차이)이 크고, scrambling 기반 대안은 계산복잡도(`O(wk+k²)`)가 높다.
3. 핵심 가정: Eve의 채널 조건이 Bob보다 항상 나쁘며(wiretap 채널 정의), 이 비대칭성을 이용해 Eve 쪽에서만 오류가 전파되도록 유도한다.
4. 핵심 기법: `ρ`개 연속 systematic LDPC 코드워드를 partial coupling — i번째 코드워드의 parity bit 일부 `c_i^{p,c}`를 `(i-1)`번째 코드워드의 message bit `c_{i-1}^m`와 XOR: `c̃_i^p = c_i^{p,c} ⊕ c_{i-1}^m`.
5. 핵심 식: 디코더가 얻은 message bit LLR을 다음 코드워드 복조 시 a priori 정보로 피드백(`L_A(c_i^m)`)해 파이프라인 방식으로 codeword-by-codeword 반복복호를 수행, 식 (3)으로 XOR 결합 비트의 채널 LLR을 계산.
6. 확장: coupled 계수 `α=k/r`, coupling 개수 `ρ`를 조절해 Eve 쪽 오류 전파 범위(SNR_E,max)를 키우는 반면 Bob 쪽 SNR 손실은 최소화.
7. 부수 트릭: 표준 BP(belief propagation) LDPC 디코더를 그대로 사용하며, XOR coupling만 추가되어 계산복잡도는 `O(k)`로 scrambling 대비 낮음.
8. 최적화 절차 없음 (DE/GA/NN 미사용). 이론적 equivocation rate 유도(식 4~6, mutual information security 개념 이용)로 보안 이득을 분석.
9. 결과: rate-0.25, length-3940 irregular LDPC에서 ρ=20일 때 security gap이 punctured scheme 대비 약 4.3dB 개선(9.13dB→4.8dB), Bob 쪽 SNR 손실은 0.08dB만 증가.
10. 한계: 이론분석+시뮬레이션뿐이며 HW 미설계. 목적이 정정능력 향상이 아닌 도청자 성능 저하(보안)이므로 NAND ECC의 waterfall/error-floor 개선과 무관한 별개 응용 분야.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Partial coupling (코드워드 간 XOR로 parity-message 결합) | 대응 없음 | 보안 목적의 의도적 오류 전파 기법으로, Prime ECC의 정정 목적 구조와 정반대 |
| Codeword-by-codeword 파이프라인 반복복호 + a priori 피드백 | `decoder.cpp` `LDPC_Decoder()` | 표준 BP 디코더 자체는 대응되나, 코드워드 간 피드백 구조는 Prime ECC 단일 코드워드 디코딩과 무관 |
| Equivocation rate/security gap 분석 | 대응 없음 | 물리계층 보안 지표로 Prime ECC의 BER/FER/iteration 성능지표와 무관 |

> 적용 가치: 낮음 — LDPC를 물리계층 보안(도청 방지)에 응용한 스킴으로 목적이 NAND ECC의 정정능력 향상과 정반대이며 이식할 요소가 없음.
