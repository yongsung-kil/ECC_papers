> 총평: 5G NR QC-LDPC 대상 parallel layered + loosely coupled 디코더 HW 구조와 적응형 정규화 min-sum으로, 이미 min-sum/HCU 계열을 보유한 Prime ECC 대비 인터레이어 네트워크 설계와 정규화 상수 테이블은 참고 가치가 있으나 부호설계(5G BG2 특화)는 이식 난이도 높음.

### B. 알고리즘 요약

1. 대상은 5G NR 표준의 QC-LDPC(Base Graph 2, BG2), 단일 lifting factor `z=384`, coded block size `N_coded=384*52=19968`, average check node degree 4.7(최대 10).
2. 문제의식은 5G NR eMBB 요구사항(20Gbps, 4ms latency)을 만족하며 51종 lifting factor를 모두 지원하는 유연하고 확장성 있는 디코더 HW 아키텍처가 필요하다는 점; 기존 layered decoder는 layer 순차 처리로 iteration duration이 길다는 한계.
3. 핵심 가정: BPA(Belief Propagation)의 근사인 Min-Sum(MSA)에 정규화 상수를 채널조건·check node조건별로 다르게 적용하는 ANMSA(Adaptive Normalized MSA, ref[6][12])를 채택.
4. 핵심 기법 1단: Parallel Layered Decoder(PLD, ref[8]) - 모든 레이어를 병렬 처리하는 Layer Processing Unit(LPU) `NL`개, 사전계산된 offset으로 인터레이어 간섭을 제거하는 Interconnection Network(IN) 구조.
5. Loosely Coupled Algorithm(LCA, ref[9]) 적용 - 명시적 variable-to-check 메시지 계산·저장 없이 이전 이터레이션의 check node 출력 메시지만으로 암묵적 V2C 메시지를 복원, LPU당 RAM 2개(check message + APLLR)만 사용.
6. 정규화 인자 적용을 위한 추가 곱셈기(multiplier)는 6-bit 폭으로 LUT 1500개(전체 4% 미만)만 소모하면서 BER 10^-5 이하에서 약 0.2dB 코딩이득 제공.
7. HW 친화 디테일: CNPU 파이프라인 스테이지 1단, N-input 2-min sorting net, LPU 42개+HardDecAndSave 10개 구성; 작은 z 값에서는 check message memory 갱신 대기로 인해 디코더를 수 클록 정지시켜야 하는 한계.
8. 별도 학습/최적화 절차 없음(RTL/VHDL 구현 + Vivado synthesis 3가지 전략 비교: AreaOptimized, PerfOptimized, AlternateRoutability).
9. 결과: FPGA(XCKU060) synthesis에서 LUT 32896, FF 24543, BRAM 137.5(약 12~13% 활용), 클록 71MHz, 10 iteration에 3840 cycle 소요, throughput 369.2Mbit/s(coded)/71Mbit/s(info); 기존 비교 설계 대비([10] 5Mbit/s, [11] 87Mbit/s) throughput 우위.
10. 한계: FPGA 검증뿐(최종 목표는 ASIC 65/45nm, 500MHz이나 미실측), BER 성능 곡선 수치는 그림/참조논문([12])에만 있고 본문 서술은 "0.2dB 코딩이득"만 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Adaptive Normalized Min-Sum (채널/CN 조건별 정규화 상수) | `decoder.cpp` `CNU_Update_New_Mag()`, `ecc_data.h` `PARAM_LLR` | Prime ECC는 min-sum + magnitude 양자화 테이블 기반이나 scaling factor가 코드상 명시 미확인 — 적응형 정규화 상수 테이블 도입 시 참고 가능 |
| Parallel Layered Decoder + Loosely Coupled Algorithm (인터레이어 네트워크) | `decoder.cpp` `LDPC_Decoder()`, `decoder.h` `Get_VNU_*` | Prime ECC도 row 단위 병렬(z_sb=32) 구조이나 LCA의 암묵적 V2C 복원 방식은 현재 구조와 결이 다름 — HW arch 재설계 필요 |
| N-input 2-min sorting net (CNU) | `decoder.cpp` `C2V_Cal_New_Sgn()` | min1/min2 계산 방식은 이미 유사 로직 보유, 정렬망 구조 세부만 참고 가능 |
| 5G BG2 H-matrix / lifting factor 구조 | `ecc_top.cpp` `Load_PCM()` | 5G 표준 특화 코드 구조로 NAND용 고rate QC-LDPC와 별개 — 이식 난이도 높음 |

적용 가치: **중간** — 이미 min-sum/HCU 계열을 보유한 Prime ECC 대비 완전히 새로운 기법은 아니나, 가변 lifting factor를 지원하는 인터레이어 네트워크 설계와 적은 자원(LUT 4% 미만)으로 코딩이득을 얻는 적응형 정규화 상수 테이블 아이디어는 HW 구조 개선 참고 가치가 있음.
