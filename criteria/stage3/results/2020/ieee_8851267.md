### B. 알고리즘 요약

1. 이진 LDPC(regular/irregular) 대상, BI-AWGN 채널, 실험 부호는 rate 1/2~0.84, length N=2048(10GBASE-T)~64800(DVB-S2-like) 다양.
2. 문제: 기존 min-sum/BP의 고정밀 LLR(5~7bit) 대신 LUT 기반 finite-alphabet 디코딩은 저복잡도이나 기존 연구는 regular 코드·저차수에 국한되어 irregular LDPC에 적용 불가.
3. 핵심 가정: 메시지 label을 임의 유한집합으로 일반화하고, discrete message label과 density evolution 기반 LLR 값을 연결하는 대칭성(symmetry) 개념 정의.
4. 핵심 기법 1단: `Φ(y)=z`로 정의되는 정보량 최대화 quantizer(LUT)를 VN/CN 업데이트마다 설계, `Φ* = argmax I(Φ(y);x)` (식 35)로 상호정보량을 최대화하는 대칭·연속영역 quantizer를 동적계획법(O(M³))으로 구함.
5. 핵심 식 의미: LUT 출력 label의 LLR `Λ(μ)`가 입력 message tuple LLR들의 min~max 사이에 bound됨(Theorem 1) → LUT decoding = "quantized BP"로 해석 가능, min-sum의 대안적 일반화.
6. 핵심 기법 2단: irregular 코드를 위해 VN/CN degree distribution `λ(μ),ρ(μ)`을 함께 고려하는 joint LUT 설계(식 39,40)로 개별 degree별 설계보다 우수한 threshold 확보.
7. 부수 트릭: `min-LUT`(CN 업데이트만 min 연산으로 대체, 식 37), LUT tree(고차수 노드의 LUT를 저차원 LUT로 nesting, 최대 ~2000배 복잡도 절감), LUT reuse(반복별 상이한 LUT를 특정 구간에서 재사용), 메시지 알파벳 downsizing(반복 진행에 따라 |M| 축소).
8. 학습/최적화 절차: BP용 stability bound(Richardson-Urbanke)를 LUT decoding으로 확장한 Lemma 2 기반 hill-climbing으로 LUT decoding 최적화 degree distribution `C_LUT` 설계.
9. 결과: 4bit min-LUT irregular 코드가 floating-point min-sum/BP를 능가; DVB-S2급(N=64800) 비교에서 4bit LUT가 5bit/4bit fixed-point MS 대비 0.5dB 성능 우위 및 error floor 개선. 별도 인용 VLSI 구현([26])은 588Gbps 10GBASE-T 디코더로 면적/에너지효율 우수성 확인(본 논문 결론에서 언급, 본문 수치 아님).
10. 한계: 본 논문 자체는 시뮬레이션/DE threshold 분석뿐이며 HW 합성·ASIC 결과는 별도 논문 인용에 그침. LUT는 iteration마다 별도 테이블 필요해 unrolled/serial 아키텍처 없이는 구현 복잡도가 커짐(reuse로 완화하나 트레이드오프 존재).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN LUT 업데이트 (min-sum 대체) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 근사를 정보이론 최적 LUT로 교체하는 직접 대응 지점, 이식 난이도 낮음 |
| min-LUT 알고리즘 (CN은 min 유지, VN만 LUT) | `decoder.cpp` `CNU_Update_New_Mag()` | 기존 min-sum CN 구조를 그대로 유지하며 VN LUT만 추가하는 하이브리드로 점진 이식 가능 |
| VN 처리 / 양자화 LUT | `decoder.cpp` `VN_Cal_HD()`, `decoder.h` `Get_VNU_*` | 기존 HD/2SD/3SD 양자화 테이블을 4bit+ LUT 기반으로 확장 가능한 지점 |
| iteration별 LUT 테이블 (LUT reuse 포함) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 적응형 LLR threshold 테이블 구조와 개념적으로 유사, LUT reuse 기법으로 저장 오버헤드 절감 아이디어 이식 가능 |
| irregular degree distribution 최적화 (LUT용) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(고정 degree 17 등) 사용 중이므로 degree distribution 재설계는 부호구조 변경 수준, 이식 난이도 높음 |
| LUT tree (nested LUT, 고차수 노드 복잡도 절감) | `decoder.cpp` `LDPC_Decoder()` | 파이프라인 내 고차수 CN/VN 처리에 적용 가능한 HW 최적화 아이디어 |

> 적용 가치: 높음 — min-sum을 완전 대체하거나 하이브리드(min-LUT)로 결합 가능한 CN/VN 업데이트 대안으로 3~4bit 저해상도에서 floating-point 성능을 넘어서는 결과가 waterfall/error-floor 모두에 이득이 있어, Prime ECC decoder.cpp의 CNU/VNU 코어 교체 후보로 정독 가치가 큼.
