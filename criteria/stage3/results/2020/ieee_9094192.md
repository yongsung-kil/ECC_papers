### B. 알고리즘 요약

1. 시스템은 (n,k) binary LDPC(irregular, n=96/192, CRC-8 concatenated), BPSK/AWGN, NMS(normalized min-sum, scaling factor 0.78, Imax=20)를 컴포넌트 BP 디코더로 사용.
2. 짧은 부호는 short cycle/trapping set으로 인해 BP 디코딩이 최적 성능에서 열화되는 문제가 있어, 실패 시(syndrome/CRC 불일치) 수신신호에 perturbation을 가해 재디코딩하는 MBPDP(multi-round BP decoding with perturbation) 계열을 일반화된 프레임워크(Algorithm 1)로 정리.
3. 핵심 가정: perturbation 대상 좌표집합(perturbation space) `R`은 신뢰도 낮은 심볼 순으로 선택하고, 매 라운드 BP 재수행(최대 `T`회) 후 CRC 통과 시 종료.
4. 제안 기법 "impulsive perturbation": 선택된 τ개 심볼에만 최대값 `A_max`을 원래 부호와 반대 방향으로 부여(`r_π(i) = -sgn(r_π(i))A_max`, pattern p의 Hamming weight 1비트만 변경), 나머지는 원신호 유지 — perturbation pattern 탐색공간을 이진 패턴 탐색으로 단순화.
5. 핵심 식: 최대 perturbation round `T`가 주어지면 패턴 집합 `P`는 Hamming weight가 작은 순서(가장 작은 `s`, 식 1: `Σ_{i=1}^{s-1} C(τ,i) ≤ T ≤ Σ_{i=1}^{s} C(τ,i)`)로 나열되며, τ는 최대로(`τ=min(T,n)`) 설정할수록 성능이 좋음 — OSD(ordered statistics decoding)의 Hamming sphere 탐색에서 착안.
6. 확장 기법(perturbation space 선택 기준) "nSMV": 각 iteration에서 외인성 LLR과 사후 LLR의 부호가 불일치한 횟수 `m_j = |{i | l^e_i(j)·l_i(j) < 0}|`를 계산해 값이 클수록(신뢰도 낮음) perturbation 우선 대상으로 선정, 동률 시 intrinsic LLR magnitude로 2차 판별.
7. 부수 트릭: perturbation space가 static(사전 결정 후 고정)이면 병렬 구현이 가능하나, dynamic(라운드마다 갱신)은 추가 연산과 병렬화 불가 트레이드오프가 있음 — 제안 기법은 static이라 병렬 처리 가능.
8. 별도 학습/최적화 절차(DE, GA 등) 없음. τ, T, s는 식(1) 기반 설계 규칙으로 결정.
9. 결과(96,48) 부호: 기존 plain NMS 대비 BLER=10^-4에서 T=14 시 0.7dB, T=62 시 1.1dB 이득. nSMV 기준이 기존 nSCV 대비 0.12dB(96,48)/0.07dB(192,96) 추가 이득. IAugBP·Gaussian perturbation 대비 대형 T에서 우위, 복잡도는 5~15% 낮음(Eb/N0 1.5~3.0dB 구간).
10. 한계: 짧은 부호(n≤192)·AWGN·URLLC 맥락에 한정되어 검증되었고, HW 합성/gatecount 없음. NAND 특유의 비대칭 채널·고rate 대형 부호 적용성은 논문 범위 밖.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP/min-sum 디코딩 실패 시 재시도 루프(MBPDP) | `decoder.cpp` `LDPC_Decoder()` | 조기종료(CRC) 실패 후 재디코딩 라운드를 추가하는 구조로 이터레이션 루프 바깥에 후처리 단계 삽입 가능 |
| CRC 기반 성공/실패 판정 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 기존에도 Partial/Full CRC로 조기종료 판정을 하고 있어 실패 시 perturbation 트리거로 재사용 가능 |
| impulsive perturbation (소수 심볼 부호반전+최대값) | `channel.cpp` (LLR 재계산) 또는 `decoder.cpp` VN 초기화부 | 특정 VN의 초기 LLR을 재설정하는 방식으로 대응되나 Prime ECC 프로파일에 perturbation 트리거 기능은 명시되어 있지 않음 |
| nSMV 신뢰도 기준(외인성/사후 LLR 부호 불일치) | `decoder.cpp` `Get_VNU_Table_Idx()` 부근 (VN LLR 추적) | iteration별 VN LLR 값을 추적하는 기존 구조를 확장하면 부호 불일치 카운트 계산 가능 |

적용 가치: **중간** — 조기종료 실패 후 재시도라는 개념은 Prime ECC의 CRC 조기종료 구조와 자연스럽게 결합 가능하나, static perturbation space를 미리 계산해두는 저장/제어 로직 추가가 필요하고 short-code/URLLC 전제(고rate 대형 NAND 부호와 다름)의 유효성은 별도 검증 필요.
