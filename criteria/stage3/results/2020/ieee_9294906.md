### B. 알고리즘 요약

1. 대상: DVB-S2/CCSDS 계열 satellite optical down-link, 10Gbps 목표. 자체 설계한 regular QC-LDPC `(N=16384, M=4096, Z=256, dc=12, dv=3)` 및 비교용 DVB-S2x `(64800, 29160, R=11/20)`, CCSDS `(22528, 6144, R=4/5)` 코드 사용.
2. 문제: 기존 연구는 1Gbps 이하 저throughput 대역에서만 HW/SW 비교가 이뤄졌고, 10Gbps급 고throughput 환경에서 전력·비용까지 포함한 공정 비교 데이터가 없었음.
3. 채널/모델: AWGN 채널, 단방향(재전송 불가) 광링크라 error floor `BER<10^-10` 요구, frame interleaving 15ms 내 latency 무시 가능해야 함.
4. 핵심 기법: 표준 layered QC-LDPC 디코더 아키텍처(Fig.2 - VN/메시지 메모리 + interleaver + Z개 병렬 처리유닛)를 Xilinx Zynq FPGA와 Intel Xeon Gold 6148(40코어) 멀티코어 각각에 구현해 동일 코드로 비교.
5. Offset Min-Sum(OMS) 근사로 sum-product의 계산 복잡도 절감(`Q8,6`/`Q8,8` 고정소수점 양자화, float 대비 BER/FER 0.05dB 차이).
6. SW 구현은 [4]의 프레임 단위(one-frame-at-a-time) 방식을 채택, AVX2/AVX512 SIMD로 병렬화(코어당 8-bit 값 32~64개 동시처리).
7. 부수 디테일 없음(양자화 외 추가 근사 없음, 10 decoding iteration 고정).
8. 학습/최적화 절차 없음.
9. 결과: FPGA는 11.65Gbps@13.8W@7.7µs latency, 멀티코어는 11.25Gbps@343W@60µs latency 달성 — SW는 전력 30배, latency 7~20배 더 소모, FPGA 대비 비용은 1.5배(P2 $7500 vs P1 $5200).
10. 한계: 새로운 디코더 알고리즘/부호 구조 제안 없음(기존 OMS·layered 구조 재사용), 순수 구현/비교 연구.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset Min-Sum 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC도 이미 min-sum 계열 보유, 논문은 표준 OMS 재사용뿐이라 중복 기여 |
| QC-LDPC 병렬 처리유닛(Z=256) 아키텍처 | 디코더 HW 모델(z=32) | 병렬 단위(z)가 다르나 구조적 유사성 있음, 다만 알고리즘 개선 없음 |
| 고정소수점 양자화(Q8,6/Q8,8) | `channel.cpp` LLR 테이블(`PARAM_LLR`) | 표준 양자화 비교일 뿐 신규 기법 없음 |
| HW vs SW 구현 성능/전력/비용 비교 | 대응 없음 | Prime ECC는 HW 시뮬레이터로, SW/멀티코어 구현 비교는 판정 범위 밖 |

적용 가치: **낮음** — 새 디코더 알고리즘이나 부호설계 기여가 없고, 기존 OMS 기반 QC-LDPC HW/SW 구현을 공정 비교한 시스템 연구로 위성광통신 특화이며 이미 Prime ECC가 보유한 기법(min-sum, 양자화 LLR)과 중복됨.
