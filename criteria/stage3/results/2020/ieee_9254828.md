> 총평: LDPC 컨볼루션 코드(LDPC-CC) 전용 pipeline 디코더 구조 논문으로 stopping rule과 weighting factor 아이디어는 min-sum 보정과 유사하나, 대상 부호 자체가 block QC-LDPC와 다른 컨볼루션 코드 특유 아키텍처(FIFO, 인접 프로세서 이격)라 Prime ECC(block QC-LDPC)로 직접 이식하기 어려움.

### B. 알고리즘 요약

1. 대상 코드는 rate 1/2, (ms,3,6) regular LDPC-CC(convolutional code, memory length ms=128/256/512bit), AWGN+BPSK 채널.
2. 기존 compact pipeline decoder는 OVA(on-demand variable node activation) 대비 초기지연/저장공간은 줄였지만 복잡도는 동일하고, stopping rule 단독으로는 지연/저장공간 절감 효과가 없었다는 문제를 지적.
3. 채널은 AWGN, 인코더 레지스터 초기값은 0, LLR 초기화는 `I(n) = 4R·r(n)·Eb/N0`.
4. 핵심 기법: compact decoder(인접 프로세서 간격 `S = ms/2+1`)에 프로세서별 카운터 기반 stopping rule을 추가 — parity-check 만족이 ms회 연속되면 해당 프로세서를 sleep 모드로 정지.
5. CNU 갱신식에 weighting factor(`α`=correction factor, `β`=offset factor)를 추가하여 min-sum 근사의 message 진폭을 보정, parity-check node 업데이트로 인한 오차 전파를 완화(식 (8)(9), 시뮬로 α=0.8, β=0.4가 최적).
6. VNU는 기존 OVA와 동일한 on-demand 갱신 방식 유지, 프로세서 N에서만 hard decision 수행.
7. 구현 디테일: 프로세서당 FIFO 레지스터 (J+1)개, 각 FIFO 길이 `2(ms+1)`; compact 구조 채택 시 저장공간 `(N·S+ms+1-S)·2(J+1)`, 초기지연 `N·S+ms+1-S` 비트로 OVA 대비 약 50% 감소.
8. 별도 학습/최적화 절차 없음, α/β는 MATLAB 시뮬레이션 스윕으로 결정(그리드서치 성격).
9. 결과: BER=10^-4에서 OVA 대비 약 0.3dB 이득, (128,3,6) 코드에서 2.4dB SNR 시 평균 iteration이 10(최대)에서 6으로 감소, 초기지연/저장공간 약 50% 절감.
10. 한계: HW(RTL/FPGA) 미설계, MATLAB 시뮬레이션뿐이며 gatecount/throughput 미제시. 대상 부호가 LDPC-CC(convolutional)로 QC-LDPC block code와 구조가 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNU weighting factor(correction/offset) 보정 | [decoder.cpp](Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` | min-sum 보정 계수 아이디어는 개념상 유사하나 LDPC-CC 전용 수식이라 직접 이식은 재검증 필요 |
| stopping rule(프로세서별 카운터) | [decoder.cpp](Prime_ECC_3.1_Claude) 이터레이션 루프, [partial_CRC.cpp](Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | Prime ECC는 이미 CRC 기반 조기종료를 보유하므로 중복 기여, LDPC-CC의 개별 프로세서 sleep 방식은 block code 구조와 불일치 |
| pipeline 프로세서(FIFO 기반, 인접거리 S) 구조 | 대응 없음 | LDPC-CC 특유의 convolutional 구조로 QC-LDPC block decoder(z=32 병렬)와 근본적으로 다름 |

> 적용 가치: 낮음. weighting factor와 조기종료 아이디어의 방향성은 참고할 만하나, 대상 코드가 컨볼루션 코드 전용 pipeline 아키텍처이고 Prime ECC는 이미 유사 기능(조기종료, min-sum 보정)을 보유하고 있어 중복이며 이식 시 재설계 부담이 큼.
