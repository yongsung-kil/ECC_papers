### B. 알고리즘 요약

1. 통신용 RC(rate-compatible) QC-LDPC 코드 설계 문제. rate 4/5(최고rate 부모 코드, AR4JA)에서 1/3까지 단계적으로 낮은 rate 자식 코드를 파생.
2. 기존 extension 단독 또는 splitting 단독 구성법은 특정 rate 구간에서 decoding threshold나 그래프 연결성 중 한쪽만 우수해 유연성이 부족함.
3. 채널은 AWGN, 변조는 BPSK, 복호는 Sum-Product algorithm(SPA) 가정.
4. 핵심 기법: 매 rate 단계마다 extension 후보 `C_E`(threshold `T_E`)와 splitting 후보 `C_S`(threshold `T_S`)를 각각 만들고, 두 threshold 차이가 `ΔT`(경험적 임계값, 본문 예시 0.25dB)보다 크면 낮은 threshold 후보를 직접 선택.
5. threshold 차이가 `ΔT` 이내면 그래프 연결성 지표 `A2CE`(shortest cycle들의 ACE 값 평균, `A2CE = (Σ ACE6(j))/NC6`)를 비교해 부모 코드 타입과 번갈아가며(alternating) 선택 — splitting은 `A2CE_S > A2CE_i`(부모보다 개선) 조건을 추가로 요구.
6. 부모 코드 타입에 따라 extension/splitting을 번갈아 선택함으로써 short cycle 수 급증을 억제.
7. multi-edge 기반 base matrix는 P-EXIT 차트로 threshold 산출 후 single-edge base matrix로 lift, 이후 RS-LDPC 코드를 마스크로 써서 z×z circulant permutation으로 QC화(4-cycle 원천 제거).
8. 최적화 절차: 각 rate 단계마다 Algorithm 1(그리디 규칙 기반 선택)을 반복 적용하는 구성 알고리즘이며, DE/GA/NN 등의 학습 절차는 사용하지 않음.
9. 결과: 정보길이 k=960 기준 rate 2/3, 1/2, 1/3에서 기존 pure extension[3][4], pure splitting[5] 및 5G 표준 코드보다 우수하거나 동등한 FER 성능 확보(구체 dB/FER 수치는 그래프에만 존재해 본문 서술로는 미상).
10. 한계: HW 설계 없음(순수 시뮬), 짧은/중간 길이 코드(k=960, 4106)에 한정된 검증이며, 장문 코드로의 확장 시 threshold-연결성 가중치 재조정이 필요하다고 저자가 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RC-LDPC 부호 구성(extension/splitting 선택 알고리즘) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 rate QC-LDPC이며 rate-compatible 가변 구성 로직 자체는 미지원, H-matrix 로드 지점만 대응 |
| A2CE 그래프 연결성 메트릭 | 대응 없음 | 코드 설계/선택 단계 오프라인 메트릭으로 디코더/인코더 런타임과 무관 |
| QC-LDPC circulant/RS-LDPC 마스킹 | `ecc_top.cpp` `Load_PCM()` | PCM_b(base+shift) 구조는 유사하나 특정 base matrix 고정이라 재검증 부담 큼 |
| SPA(Sum-Product) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Min-Sum 근사만 지원, 논문은 SPA 성능 비교용으로만 사용 |

적용 가치: **낮음** — 가변 rate 통신 표준(5G 비교 포함)용 부호 구성법이며, Prime ECC는 고정 고rate NAND ECC로 부호설계 이식은 난이도 최고(§4)이고 본 논문은 이미 보유하지 않은 새로운 디코더/HW 기법을 제공하지 않음.
