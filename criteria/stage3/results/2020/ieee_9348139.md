### B. 알고리즘 요약

1. IEEE 802.11 표준 rate-1/2, length-1296 비정규 LDPC 코드, 채널은 BPSK/AWGN 기반 symmetric BI-DMC.
2. 기존 MIM-QBP·Min-LUT는 degree-2 VN 비율이 큰 코드(802.11: 45.8%)에서 성능 저하 문제 발생.
3. 메시지를 외부(라우팅용, 저비트폭 `m`)와 내부(연산용, 고비트폭 `nc`/`nv`) 두 종류로 분리하는 것이 핵심 모델.
4. RCQ = Reconstruction `R(·)`(외부→내부 매핑) + Computation `F(·)`(BP의 tanh 또는 Min-Sum) + Quantization `Q(·)`(내부→외부, 2^m−1개 threshold).
5. `RCQ(m, nc, nv)` 3-튜플로 정밀도 표기; ms-RCQ는 `m=nc`이고 반복에 따라 `Rc`가 불변이라는 성질을 이용해 LUT 1개로 클러스터형 Min-Sum 구현 가능.
6. 비균일 양자화기 설계에 Hierarchical Dynamic Quantization(HDQ) 도입: STS(Sequential Thresholds Searching) 알고리즘으로 bit-level별 threshold를 순차 결정, 복잡도 `O(mM)`.
7. 체크/변수노드 결합확률(p.m.f.) 계산 시 카디널리티 폭발을 막기 위해 One-Step-Annealing(OSA) 근사 병합(LLR 차이 `< ls`인 항목 병합)을 삽입.
8. MIM-DDE(비균일 양자화 기반 Discrete Density Evolution)로 반복별 p.m.f.를 추적해 `Qc,Qv,Rc,Rv`를 이터레이션마다 재설계.
9. 4-bit ms-RCQ(4,4,∞)가 저 Eb/N0에서 floating BP 대비 0.1dB 이내, 고 Eb/N0(FER≤1e-5)에서는 오히려 BP·Min-Sum을 능가(trapping set 완화 효과).
10. HW 미설계, IEEE 802.11 특정 코드·시뮬 결과에 국한되며 nc/nv 고정소수점 구현의 실제 오버헤드는 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 외부/내부 메시지 분리 비트폭 설계, 반복별 LLR threshold 재설계 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 적응형 LLR 테이블 구조와 유사한 개념으로 확장 가능 |
| ms-RCQ 체크노드 min-sum 연산 및 LUT화 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 변형이므로 이식 난이도 낮음 |
| VN 양자화(soft-info) 재설계 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | soft-2~3bit/4bit+ 채널 재설계와 관련 |
| channel soft-read threshold(HDQ 결과) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 비균일 threshold 설계 기법으로 대응 가능 |
| 부호구조(802.11 특정 irregular LDPC) | `ecc_top.cpp` `Load_PCM()` | 본 논문은 부호설계 자체를 제안하지 않고 기존 표준 코드 사용 → 대응 없음(구조 재설계 불필요) |

적용 가치: **중간** — min-sum 체크노드 및 반복별 LLR/threshold 재설계 아이디어는 이식 난이도가 낮은 레이어에 해당하나, HW 미설계·무선 표준 코드 기준 시뮬로만 검증되어 NAND 직접 적용 전 재검증 부담이 있음.
