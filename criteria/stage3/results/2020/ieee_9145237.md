### B. 알고리즘 요약

1. 대상은 IEEE 802.16e 표준 (576, 432) binary QC-LDPC(rate 3/4), AWGN 채널, BPSK 변조.
2. 기존 conventional NMS(CNMS)는 모든 반복에서 고정된 단일 correction factor `α`를 사용해 성능이 제한적이고, 기존 딥러닝 디코더(BP unrolling)는 short code에만 적용 가능해 long code 확장이 어렵다는 문제를 다룸.
3. 핵심 가정은 반복 복호의 CN↔VN 메시지 전달 과정을 feedforward 신경망으로 해석(model-driven unfolding)해 파라미터(correction factor)만 학습 가능하게 하는 것.
4. 핵심 기법 1단(NNMS): CN-to-VN 메시지 계산에 edge별 개별 `α_ij`(식 6), VN-to-CN 메시지에 edge별 개별 `β_ji`(식 7)를 부여, `(N,k)` 코드를 `2*l_max+2`개 레이어(CN층 l_max개 + VN층 l_max개 + 입출력층)로 unroll.
5. 핵심 식은 `L(l)(rji) = Π sgn(...) · min(α^(l-1)_i'j · |L^(l-1)(qi'j)|)` — 기존 min-sum의 고정 α를 edge·iteration별 학습 파라미터로 대체해 min 근사 오차를 보정.
6. 핵심 기법 2단(SNNMS, 확장): edge별로 다르던 α, β를 같은 레이어 내에서 공유(`α^(l-1)`, `β^(l)` 스칼라 1개씩)해 곱셈기 수를 줄이고 원래 LDPC 구조(H-matrix)를 유지.
7. 부수 트릭: 출력 뉴런 활성함수로 학습 시 sigmoid `σ(x)=(1+e^-x)^-1`, 테스트 시 sign 함수로 전환; loss는 cross-entropy(식 9).
8. 학습 절차: TensorFlow, mini-batch gradient descent(batch=120), SNR 0~5dB 혼합 학습 데이터, Adam optimizer(lr=0.001) — 여러 SNR 혼합 학습이 단일 SNR 학습보다 강건함을 실험으로 확인.
9. 결과: NNMS는 CNMS 대비 BER 1.5dB 개선(더 적은 iteration으로), 32-layer(15 iteration) 기준 SNNMS가 NNMS 대비 BER^10^-7에서 0.4dB 추가 개선; SNNMS는 MUL 연산 수가 `T`(반복수)로 NNMS(`Σaj(aj-2)T + Σbi(bi-2)T`) 대비 대폭 감소(Table I).
10. 한계: 부동소수점 학습/추론 기반이며 fixed-point 디코더 및 FPGA 가속은 future work로만 제시, 실제 HW 합성·gatecount·throughput 측정 없음, 단일 코드(576,432) rate 3/4에서만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CN 연산에 학습된 correction factor 적용 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 magnitude 양자화 테이블 기반 min-sum(§3)을 사용하므로, 학습된 α를 iteration별 스케일 팩터로 대체 적용 가능성 있음(단, 학습 파이프라인 부재) |
| VN-to-CN 메시지 계산 β 팩터 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | 개념적으로 대응되나 코드는 HD/2SD/3SD 이산 양자화 방식이라 연속값 β 학습 결과를 그대로 이식하려면 재양자화 필요 |
| iteration별 correction factor(SNNMS의 layer-shared α,β) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC의 기존 "iteration 구간별 LLR threshold 테이블(적응형 LLR)"과 유사한 목적(반복별 파라미터 차등화)으로 개념적 중복 있음, 완전히 새로운 기여는 아님 |
| BP unrolling / 신경망 학습 구조 자체 | 대응 없음 | NN 학습 프레임워크(TensorFlow)는 C++ bit-exact 시뮬레이터 구조에 대응 없음 |

> 적용 가치: 중간 — min-sum 성능 개선(waterfall/error-floor, iteration 감소)이라는 목적은 Prime ECC 관심사(§5)와 직접 부합하고 학습 결과(스칼라 correction factor)만 추출하면 재현 가능성이 있으나, Prime ECC는 이미 적응형 LLR 테이블을 보유(§3)해 기여 중복이 있고 fixed-point/양자화·HW 검증이 부재해 즉시 이식은 어려움.
