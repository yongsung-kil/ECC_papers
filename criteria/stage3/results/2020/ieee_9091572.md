### B. 알고리즘 요약 (10줄 내외)

1. 5G 표준의 protograph-based raptor-like(PBRL) QC-LDPC(정보비트 `K=1032`, rate `Rc=1/3~4/5`) 부호를 BI-AWGN 채널에서 복호하는 문제를 다룸.
2. 저정밀도(3~6bit) 메시지 패싱은 성능 저하가 크고, box-plus 연산 근사(min-sum/normalized/offset-min-sum)만으로는 거친 양자화에서 한계가 있다는 문제의식.
3. Information Bottleneck(IB) 방법으로 `max I(X;T)` (관련정보 X, 관측 Y, 압축 클러스터 T)를 만족하는 결정적 매핑 `t=f(y)`을 설계, LLR을 전혀 쓰지 않고 정수 클러스터 인덱스 `t_in/t_out`만 교환.
4. 체크노드/변수노드 연산을 discrete density evolution으로 사전 설계한 LUT(`p(t_out|t_in,d)`)로 대체, 노드 degree `d`별 결합분포 `p(x,t_in|d)`를 기반으로 최적화.
5. 핵심 문제: 클러스터 인덱스는 노드 degree에 독립적이나 그 의미 `p(x|t_out,d)`는 degree에 종속 → "message alignment"(명시적/암시적 두 방식)로 이 불일치를 IB 문제로 재정식화해 해결.
6. PBRL 부호 특유의 puncturing(HRC 상시 펑처링 노드 + degree-1 IRC 노드의 rate별 가변 펑처링)을 채널유도/체크노드유도 두 경우로 나눠 message alignment에 통합, 짝수 클러스터 수로도 처리.
7. Rate-compatible 설계: 최고 rate(`Ri=4/5`)용 LUT를 기반으로 낮은 rate의 추가 degree LUT만 덧붙여 재사용(Proposition 1: rate가 낮을수록 유효 최대 degree 증가), 단일 LUT 세트로 전 rate 범위 커버.
8. 체크노드는 대안으로 min-sum-inspired 매핑(클러스터 인덱스 기반, 반복마다 불변)으로 대체하는 하이브리드도 검토.
9. 결과: 4bit IB 디코더가 6bit normalized-min-sum을 능가하고 double-precision BP 대비 0.2dB 이내 성능, 3bit도 4bit offset-min-sum과 동등; 하이브리드(체크노드 min-sum)는 offset-min-sum 대비 FER 10⁻⁴에서 0.6dB 개선; 테이블 재사용 시 메모리 요구량 체크노드 1/3, 변수노드 약 1/1.8로 감소.
10. 한계: 순수 알고리즘/시뮬레이션 논문으로 FPGA/ASIC 등 실제 HW 합성·gatecount·throughput 결과 없음(참고문헌 [16]의 588Gb/s FAID 결과만 인용); LUT 오프라인 설계 비용과 iteration별 테이블 저장 오버헤드는 정량화 안 됨.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IB 기반 LUT로 min-sum 대체 (체크노드 연산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산을 IB LUT로 대체하는 아이디어는 이식 가능하나 클러스터 인덱스 체계 재설계 필요 |
| 변수노드 LUT 처리 / 3~5bit 클러스터 인덱스 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | soft-2~3bit 대응 VN 처리부와 유사 위치, LLR 대신 클러스터 인덱스 사용은 구조 변경 큼 |
| iteration별 LUT(테이블 변화) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블과 유사한 위치이나 IB는 완전히 다른 설계 방식(density evolution 기반) |
| Puncturing 처리(PBRL 구조) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC 고정이라 PBRL식 가변 rate/puncturing 구조 이식은 난이도 높음 |
| Rate-compatible 다중 rate LUT 재사용 | 대응 없음 | Prime ECC는 단일 고rate 부호 고정, 다중 rate 지원 구조 없음 |
```

**적용 가치**: `중간` — min-sum을 대체하는 거친 양자화(3~4bit) LUT 디코더 아이디어는 soft-read 비용 절감 관점에서 참고 가치가 있으나, LLR 기반이 아닌 클러스터 인덱스 패러다임 전환이 필요해 Prime ECC 구조에 직접 이식하기보다는 개념적 참고에 그침.
