### B. 알고리즘 요약

1. 대상 부호는 information length k=20의 초단문 부호 4종(n=32,64,128,256): Brouwer-Grassl 최적 선형블록부호, Gaussian approximation 기반 Polar code, 5G eMBB Base Graph 2 MET QC-LDPC(protograph), AWGN+BPSK 채널.
2. 문제의식: 짧은 부호에서 Belief Propagation(BP) 계열 soft decision 복호기가 trapping set/soliton으로 인해 유한블록길이 Polyanskiy 이론 한계 대비 성능 gap이 큼(BLER 10⁻²에서 1dB 이상, 10⁻⁵에서 2dB 이상).
3. 핵심 가정/모델: LDPC BP 복호 문제를 Sourlas의 spin glass(Edwards-Anderson/Sherrington-Kirkpatrick Hamiltonian) 환원과 Mezard의 replica symmetry breaking(cavity method)으로 위상학적으로 재해석, 코드의 cycle이 만드는 "curvature"가 복호 오류의 원인이라는 관점 제시.
4. 핵심 기법: Nachmani-Wolf의 hyper-graph neural network 복호기(BP unfolding에 학습 가능한 가중치 `θg`, `θf` 추가)를 채택 - variable node 갱신식(6)을 hyper-network `g`(가중치는 network `f`가 입력 LLR로부터 산출)로 대체.
5. 핵심 식: check node 갱신(식 8)에서 `arctanh`를 `2m+1`차 Taylor 급수로 근사(`q(x)=Σ x^(2m+1)/(2m+1)`), Nishimori temperature T=1 근방에서 magnetization 함수를 평탄화해 학습 안정성(loss landscape uncertainty 완화)을 개선.
6. 확장: waterfall(paramagnetic, symmetric) 영역과 error-floor(glass, symmetry-breaking) 영역을 구분하고, 대칭 조건이 성립하는 waterfall 영역에서는 zero-codeword만으로 `f`,`g` 네트워크 학습이 충분하다고 설명.
7. 부수 구현 디테일: hyper network `f`,`g`는 4-layer(128 뉴런)/2-layer(16 뉴런) 구조, bias 없이 tanh 활성화 사용. 비교군으로 flooding scheduler BP(50 iter), Afterburn Saturated Min-Sum, OSD(order dmin/4), Successive Cancellation List(size 32) 디코더 사용.
8. 학습/최적화 절차: 본문에 hyperparameter network 학습 세부 절차(손실함수, epoch, 데이터셋 크기 등)는 기술되지 않음(참조논문 [42] 구조 인용 수준).
9. 결과: OSD, Hyper-graph decoder, 대형 리스트 SCL이 Polyanskiy 유한블록길이 한계에 가장 근접. 정량적 BER/BLER 수치는 본문 서술에 없고 Fig.4~7 그래프로만 제시되어 본문 근거로는 확인 불가.
10. 한계: 극단적으로 짧은 코드(n≤256, k=20)에서만 검증, HW 구현/합성 전혀 없음, 뉴럴넷 학습 절차·수렴성에 대한 구체적 서술 부족, 제시된 개선(정정능력/iteration/latency)에 대한 본문 수치 근거 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Hyper-graph neural network 복호기(학습 가중치 BP) | 대응 없음 | Prime ECC 프로파일 §6에 "NN/딥러닝 디코더는 대응 없음"으로 명시된 항목 |
| arctanh Taylor 근사(check node 갱신) | 대응 없음 | Prime ECC는 이미 min-sum(§3, tanh를 min으로 근사)을 채택해 sum-product 계열 arctanh 연산 자체가 없음 |
| Afterburn Saturated Min-Sum(비교군, NN 아님) | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum 변형 스케줄러라는 점에서 개념적 유사성은 있으나 본문에 알고리즘 세부가 서술되지 않아(인용 [19]) 이식 판단 근거 부족 |
| MET QC-LDPC(5G BG2 protograph) 부호구조 | `ecc_top.cpp` `Load_PCM()` | protograph 기반 부호 구성이나 5G 표준 특화 구조라 NAND 고정 QC-LDPC와 직접 대응 어려움 |
| OSD, SCL(Polar) 비교 디코더 | 대응 없음 | 대상이 binary LDPC decoder가 아닌 타 부호 계열 |

> 적용 가치: 낮음 — 핵심 기여가 NN 기반 복호기와 초단문 부호 이론 분석에 집중되어 있어, binary QC-LDPC HW 시뮬레이터인 Prime ECC 3.1에 이식 가능한 요소가 거의 없음(NN 디코더는 프로파일상 "대응 없음"으로 명시된 항목).
