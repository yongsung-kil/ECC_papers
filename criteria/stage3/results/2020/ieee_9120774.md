### B. 알고리즘 요약

1. 시스템: 5G NR LDPC 코드(protograph 기반, base graph 2), AWGN+BPSK 채널, 정보길이 `K=260,520`, 코드율 `R=1/2,1/3,1/4`.
2. 문제: 기존 조기종료 스킴은 decodable case를 빨리 끝내는 syndrome test(순수 all-zero 검사)는 있지만, undecodable(정정 실패) case를 예측해 불필요한 반복을 줄이는 스킴은 별도 필요.
3. 핵심 가정: `l`번째 iteration의 syndrome `s(l)`과 `(l-T)`번째의 `s(l-T)`을 비교, 두 반복에서 동시에 불만족인 parity check 개수 `N_PC^(l) = s(l)·(s(l-T))'` (식 4)이 undecodable case에서는 0보다 큰 값에 수렴하고 decodable case에서는 0으로 수렴한다는 관측.
4. 핵심 기법(VUPC-T): `N_PC^(l) > N_th`이면 undecodable로 판정하고 즉시 종료 (Algorithm 1). 기존 syndrome test(all-zero 검사)와 병행 사용.
5. 핵심 식: 임계값 `N_th`는 PEXIT chart의 mutual information 재귀식(식 6, 9)으로 유도한 parity check 불만족 확률 `P_upc^(l)(i)`(식 10)을 이용해 `N_th = Σ Z·P_upc^(1)(s)·P_upc^(1+T)(s) + 1` (식 11)로 offline 1회 계산.
6. 확장(VUPPC-T): 전체 parity check 대신 일부(예: 1/4)만 관찰해 메모리(`M·T` bit)와 연산량을 절감. 관찰 대상은 두 반복에서 불만족 확률이 높은 parity check type 우선 선택.
7. 부수 트릭: 식(4)의 AND+덧셈 연산만으로 계산 가능해 HW 친화적, threshold는 offline 1회 유도라 온라인 추가 연산 없음.
8. 최적화 절차: PEXIT chart mutual information 반복 계산(Gaussian 근사)으로 threshold를 오프라인 유도 (Density-Evolution 계열 분석).
9. 결과: VUPPC-T가 CER(코드워드 에러율) 성능 저하 거의 없이 평균 iteration을 CMM/CSVP 대비 더 빨리(고에러율 영역에서 가장 먼저) 줄임. 계산 복잡도는 CSVP보다 낮음(부분 관찰 덕분), 코드율이 낮을수록/parity check 수가 많을수록 이득 커짐(정성적 도표, 정확 수치는 그래프에만 존재).
10. 한계: FPGA/ASIC 등 HW 구현 없이 시뮬레이션(AWGN/BPSK)뿐. threshold가 특정 protograph(NR LDPC BG2) 구조 기반이라 다른 코드 구조에는 재유도 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 변동 기반 undecodable 조기종료(VUPC-T/VUPPC-T) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 기존 CRC 기반 조기종료와 목적(불필요 iteration 절감)이 유사하나 판정 근거(syndrome unsatisfied count 변동)가 달라 보완적으로 검토 가능 |
| syndrome(parity check) 계산 및 저장 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 매 iteration syndrome을 계산하는 기존 루프에 `s(l)` 저장/비교 로직 추가 필요 |
| PEXIT chart 기반 threshold 오프라인 유도 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블처럼 offline 유도된 파라미터를 런타임에 참조하는 구조와 유사 |
| 부분 관찰(VUPPC-T) 메모리 절감 | `decoder.h` `Get_VNU_*` | 전체 parity check 대신 일부만 저장하는 메모리 절감 아이디어가 HCU 등 자원 절감 기법과 유사한 맥락 |

적용 가치: **중간** - Prime ECC가 이미 Partial CRC 조기종료를 보유해 완전히 새로운 기여는 아니지만, syndrome 변동 기반 undecodable 예측은 다른 판정 축이라 CRC와 병행 적용 시 추가 iteration 절감 여지가 있고 PEXIT 기반 threshold 유도 방법론은 참고할 만함.
