> 총평: TLC 셀 상태 bit-labeling으로 read threshold 수를 소폭 늘려 페이지 용량 불균형(spare area)을 줄이는 채널/저장 레이어 논문으로, LDPC 부호설계나 디코더 이식 요소가 없어 Prime ECC 3.1 이식 가치는 낮음.

### B. 알고리즘 요약

1. TLC(3bit/cell) NAND 플래시에서 MSB/CSB/LSB 3개 페이지의 page capacity `C_l`이 bit-labeling에 따라 달라지고, 이 중 최소값 `C_min`이 필요 redundancy(spare area)를 결정.
2. 기존 Gray code labeling은 랜덤 페이지 단위 read에 유리(각 페이지 독립 디코딩 가능)하지만 페이지간 capacity 불균형이 커 spare area 낭비(4.1%+) 발생.
3. 채널 모델: 측정된 threshold voltage 히스토그램을 정규분포+지수분포 결합의 piecewise 함수(§III 식 3~6, 파라미터 σ,μ,λ,x_i)로 피팅해 각 상태의 조건부 pdf `f(x|L_i)` 추정.
4. 핵심 기법: 첫 페이지(MSB)만 독립 디코딩하고 나머지 두 페이지(CSB, LSB)는 멀티레벨코딩처럼 2단 joint 처리하는 새 labeling(`proposed code 1/2`) 제안.
5. Joint 처리 시 2번째 레벨은 1번째 비트값을 알고 있다는 정보를 활용해 절반의 상태만 구분하면 되므로 오류확률이 낮아짐(예: r1,3, r3,5만 사용).
6. 인접 threshold 재사용 트릭: 근접한 reference(예: r1,5 ≈ r2,3)를 공유해 read threshold 개수를 줄임(17→10, 9로 절감).
7. 비교 대상: Gray code 1/2/3(7 threshold), max-distance labeling(17 threshold, Ungerboeck 멀티레벨코딩), 제안 code 1/2(9~10 threshold), soft-read 버전(1bit/2bit soft, 21/35 threshold).
8. 최적화/학습 절차 없음 — 측정 데이터 기반 파라미터 추정(regression으로 지수 tail λ 추정) 및 조합 설계.
9. 결과: EOL 기준 spare area가 Gray(4.20%) 대비 proposed code 2는 3.25%(hard), 1bit soft 사용 시 2.00%까지 감소(같은 조건 Gray 1bit soft는 2.46%).
10. 한계: 페이지 joint 처리로 랜덤 read latency 증가, LDPC/ECC 디코더 자체는 다루지 않고 채널·라벨링 레이어에 국한, HW 구현 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-labeling / 페이지 용량 균형화 | 대응 없음 | 부호설계·디코더가 아닌 셀 상태-비트 매핑 문제로 Prime ECC PCM/디코더 어느 것에도 대응하지 않음 |
| soft-read threshold(1bit/2bit soft) 비교 | `channel.cpp` `Set_R_Offset()` `Set_LLR_Th()` | soft-read 비용 개념은 유사하나 이 논문은 threshold 설계가 아닌 labeling 문제 |

적용 가치: 낮음 — LDPC 부호설계/디코더 요소가 전혀 없고 NAND 채널·페이지 구성 레이어 문제로, Prime ECC 코드베이스와 접점이 없음.
