### B. 알고리즘 요약

1. Binary symmetric channel(BSC, crossover probability `p`)에서 regular binary LDPC 코드(Tanner code (155,64) dv=3, QC-LDPC (1296,972) dv=4)의 linear FAID(finite alphabet iterative decoder) 설계.
2. 기존 FAID 설계는 density evolution 기반 trial-and-error로 CN/VN 갱신규칙을 탐색해야 했고, 기존 NN 디코더는 AWGN(연속값) 전용이라 BSC(이산 채널값)에는 비효율적이었음.
3. 채널 모델: BSC 이진 대칭 채널, bipolar mapping `z_i=(-1)^{y_i}C`, 메시지 알파벳 `M={0,±L1,...,±Ls}`(s≤2^(b-1)-1).
4. 핵심 기법: linear FAID를 K개 hidden layer로 unfold한 recurrent QNN(RQNN)으로 구성, VN 갱신 `Φ(z_i,n_i)=Q(Σm+ω_i z_i)`, CN 갱신은 min-sum과 동일(`Ψ(m_j)=sgn·min`), 가중치·바이어스·알파벳 레벨을 전 iteration 공유.
5. `Γ(u,x)=(1/N)Σ(x_i-x̂_i)^2` (BER 자체를 loss function으로 사용, soft-decision loss 대신 hard-decision 기반)로 학습 — 기존 AWGN용 soft loss가 BSC에서 수렴 안 되는 문제를 해결.
6. 확장: BER/quantizer의 gradient가 거의 everywhere 0이 되는 문제를 straight-through estimator(STE, `∂h_M/∂x`, `∂h_sgn/∂x`)로 대체해 backprop 가능하게 함, 알파벳 레벨 `M`과 RQNN 파라미터(채널값 C, 가중치, 바이어스)를 ADAM으로 공동학습(joint training).
7. 구현 디테일: 학습 후 `Φ_RQNN`, `Υ_d`를 (dv-1)/dv차원 LUT로 변환해 3-bit 정밀도로 HW 구현 가능(대칭성 이용해 절반만 저장), 학습 데이터는 waterfall 영역 crossover probability `p`로 heuristic 선택.
8. 학습 절차: ADAM optimizer + mini-batch 기반 joint training(Algorithm 1), 초기값은 quantized min-sum 디코더에서 시작.
9. 결과: QC-LDPC(1296,972)에서 p=0.004 시 MS 대비 FER 4자릿수, OMS 대비 1자릿수, SPA 대비 1/3 이하 개선; 동일 FER(<10^-4) 달성에 SPA 30회 대비 15회로 수렴, BER 10^-8까지 error-floor 미관측(SPA는 10^-7에서 floor).
10. 한계: BSC·regular column-weight(3~4) 코드에 한정된 시뮬레이션 결과뿐이며 실제 HW 설계·gate count·throughput은 제시되지 않음(LUT 개념 검증만).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-bit linear FAID LUT(VN 메시지 갱신) | `decoder.cpp` `VN_Cal_HD()` 등(HD/2SD/3SD), `decoder.h` `Get_VNU_*` | soft-2~3bit 양자화 LUT 구조가 Prime ECC의 VN 양자화 처리와 직접 대응 |
| CN min-sum 연산(Ψ, sign·min) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | FAID의 CN 갱신이 기존 min-sum과 동일하여 그대로 대응 |
| iteration별 LLR/알파벳 레벨 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 학습된 M 레벨/threshold를 LLR 테이블 대체 후보로 검토 가능 |
| BSC 채널·soft-read 양자화 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | BSC는 hard-read(1bit)와 유사, NAND soft-read(2SD/3SD)로 확장 시 재검증 필요 |
| RQNN 학습(오프라인, ADAM+STE) | 대응 없음 | Prime ECC는 학습 파이프라인 미보유, LUT만 이식 가능(학습 자체는 오프라인 도구 별도 필요) |

> 적용 가치: 중간 — 3-bit LUT 기반 FAID가 min-sum 대비 waterfall 개선·수렴 가속·낮은 error-floor를 보이고 HW 오버헤드도 작다고 주장하나, BSC/regular 코드 한정 시뮬레이션이라 Prime ECC의 QC-LDPC/soft-read 구조로의 직접 이식엔 재검증이 필요.
