> 총평: turbo equalization/ISI 채널 특화 spatial coupling 연구로 NAND 고전 binary LDPC 이식 관점에서는 관련성이 낮음.

### B. 알고리즘 요약

1. Rate `Rc=1/2`, length `N=5120`의 5G LDPC(BG2, lifting 256) 또는 convolutional code를 컴포넌트 부호로 사용, BPSK 변조 후 5-tap ISI 채널(`h=[0.277,0.46,0.688,0.46,0.277]`) + AWGN 통과.
2. 기존 turbo equalization은 약한 부호(좋은 waterfall/나쁜 error-floor)와 강한 부호(반대) 사이 trade-off가 존재하며, 이는 BP 디코딩 방식(joint MAP 아님)의 한계임을 EXIT chart로 설명.
3. 핵심 가정: SC-LDPC의 threshold saturation(결합 앙상블 BP threshold → 비결합 앙상블 MAP threshold 근접) 성질이 이 trade-off 회피에 적용 가능.
4. 3가지 결합(coupling) 방식 제안: (a) encoder 출력-채널 간 memory `m` 결합, (b) protograph 기반 SC-LDPC 부호 자체 결합(base matrix `B=B0+B1`), (c) (a)+(b) 동시 적용.
5. 핵심 식: 결합 시스템 rate `R=Rc(1−m/L)`, chain 길이 `L`이 커질수록 원래 rate `Rc`에 수렴하며 known bit들이 BP 성능 개선에 기여.
6. window decoding(`W≥m+1`)으로 전체 chain을 기다리지 않고 블록 단위 복호, latency 감소 목적이나 블록당 반복 방문 횟수가 window size(5)만큼 증가.
7. 부수 트릭: base matrix를 두 서브행렬(B0, B1)로 분할하는 protograph coupling 기법(regular (3,6) 코드 예시).
8. 별도 학습/최적화 절차 없음(코드 구성은 기존 문헌([17]) 방식 채용, DE/GA/NN 미사용).
9. 결과: MAP equalizer 기준 SC-LDPC coupling이 채널 coupling 단독 대비 약 2dB 추가 이득(총 4dB, i.u.d capacity 한계 대비 1dB gap), MMSE equalizer는 SC-LDPC coupling으로 약 8.5dB 이득.
10. 한계: HW 미설계, 순수 시뮬 기반이며 ISI/turbo equalization 채널 특화 결과이고 window decoding으로 인해 실효 반복 횟수(및 latency)가 5배 증가하는 복잡도 비용 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC protograph coupling(부호구조) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조라 SC 결합 도입은 부호설계 레벨 재검증 필요(대응은 있으나 난이도 높음) |
| turbo equalization(encoder-channel coupling) | 대응 없음 | Prime ECC는 ISI 채널/equalizer 모델 자체가 없어 대응 없음 |
| window decoding(지연 제어) | 대응 없음 | Prime ECC의 조기종료는 partial CRC 기반이며 window decoding과 메커니즘이 다름 |
| MAP/MMSE equalizer | 대응 없음 | NAND read 채널에는 equalizer 개념 없음 |

적용 가치: **낮음** — ISI/turbo equalization 채널 특화 연구로 NAND 고전 binary LDPC의 채널 모델과 근본적으로 다르며, Prime ECC 구조에 대응되는 모듈이 거의 없음.
