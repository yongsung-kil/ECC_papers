### B. 알고리즘 요약

1. 대상: 3GPP 5G NR 데이터채널용 QC-LDPC, base graph BG2(42×52, 4×14 core), lifting Z에 따라 다양한 N(832~3120), rate `rBG2=10/52=0.1923`(비펑처링 기준).
2. 문제: 실용적 quantizer + iterative decoder(MSA)에서 absorbing set(AS)에 의한 error floor가 발생하는데, 5G NR 코드의 AS를 어떻게 효율적으로 찾을지가 미해결.
3. 채널/모델: BPSK-modulated AWGN, `d.f` 균일 quantizer(q=d+f+1 비트), 최대 200 iteration의 MSA 디코더 가정.
4. 핵심 기법: BG2가 dense core(4×14)와 extension part로 나뉘는 구조를 이용, core 내 특정 열(해당 열이 core 밖에 더 많은 1을 가짐)을 골라 그 열과 연결된 core-내부/외부 행·열을 추적하여 elementary absorbing set(EAS)을 구성하는 절차를 제시.
5. 핵심 식: `(a,b)` AS는 `|D|=a`, `O(D)`(odd-degree neighbor check) 크기 `b`; elementary는 `O(D)` 전부 degree-1, `E(D)` 전부 degree-2인 경우 - 이 성질을 core column 조합으로 역산.
6. 확장: core variable node 1개짜리 EAS(11종, Table I) → 2개짜리 EAS(Z=32 기준 다수, Table II)로 확장하며, lifting 값 Z에 따라 동일 column 조합도 다른 크기의 EAS를 생성함을 관찰(예: 컬럼 9,13 조합이 Z=32면 (12,3), Z=16이면 (12,1)).
7. 부수 관찰: quantizer의 dynamic range가 EAS 구성을 좌우(예: 5-bit 2.2→3.1로 range 확장 시 저weight AS 제거), precision(비트수)은 동일 range면 구성에 거의 영향 없음.
8. 최적화 절차 없음(탐색적/조합적 열거, 학습 기법 아님).
9. 결과: puncturing만 하면 waterfall 저하+error floor 소폭 개선, shortening(F=64)만 하면 error floor 거의 제거(~10⁻⁶), 둘 다 적용하면 10⁻⁸까지 error floor 미관측.
10. 한계: 5G NR BG2 특정 구조에 특화된 분석(다른 code에 일반화 불명확), HW 구현/합성 없음, elementary AS 이상(4개 이상 core column 개입 사례)은 수작업 카탈로그화가 어려워 별도 문헌([14] 석사논문) 참조로 넘김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Absorbing set(error-floor 원인) 탐색 방법론 | 대응 없음 | Prime ECC는 고정 H-matrix(`Load_PCM()`)를 사용하며 AS 탐색·설계 도구가 프로파일에 명시되지 않음 |
| Quantizer dynamic range와 error floor 관계 | `channel.cpp` `Set_LLR_Th()` | LLR 양자화 threshold/dynamic range 설계 시 참고 가능하나 논문은 이론적 관찰 수준 |
| Puncturing/shortening을 통한 error floor 개선 | 대응 없음 | Prime ECC 인코더는 dual-diagonal 고정 구조로 puncture/shorten 정책이 프로파일에 없음 |
| Min-sum 디코더(MSA) 사용 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 min-sum 기반이나 본 논문은 디코더 알고리즘 자체를 개선하지 않음(관찰용 도구) |

적용 가치: `낮음` — 5G NR 저rate 통신 코드의 error-floor 원인 분석 논문으로 디코더/HW 개선 기법이 없고, Prime ECC의 고정 QC-LDPC 구조에 AS 탐색 방법론을 이식하려면 별도의 재설계·재검증이 필요함.
