### B. 알고리즘 요약

1. 3D NAND(MLC 2bit/cell 기준, TLC 확장 가능), LDPC 채널단 soft-sensing 최적화 논문으로 부호 자체 rate/length는 다루지 않음(기존 채택된 LDPC 부호에 적용).
2. 문제: 기존 대칭(symmetric) sensing level 배치는 최악 상태의 오류율을 기준으로 모든 state pair에 동일하게 많은 sensing level을 배치해 불필요한 read latency를 유발.
3. 핵심 가정: 오류에 3가지 비대칭성 존재 — (a) state 쌍 간 RBER 차이(inter-state, 최대 10배 이상), (b) 좌/우 shift 비대칭(intra-state, retention→left-shift vs P/E cycling·interference→right-shift), (c) 3D 블록 내 레이어(워드라인)간 RBER 편차(cross-layer, 블록 간 공통 패턴).
4. 핵심 기법: 3개 서브모듈로 sensing level을 상태쌍별·좌우별·레이어별 비대칭 배치 — inter-state는 RBER 높은 state pair에 우선 추가(`if ek<ek+1 then Nk≤Nk+1`), intra-state는 P/E cycle 임계값 `T_PE` 기준 좌/우 우선순위 결정, cross-layer는 레이어별 시작 sensing level 수(n0)를 stage 테이블로 미리 설정.
5. 핵심 식: `Nk = Nkl + 1 + Nkr`(식 2), `ekl > ekr → Nkl ≥ Nkr`(식 3) — 비대칭 오류율에 비례해 sensing level을 좌우 불균등 배치함으로써 총 sensing level 수를 감소.
6. 확장: 3개 기법(inter-state, intra-state, cross-layer)의 결합 적용, cross-layer는 블록별 stage 번호를 기록·업데이트하는 상태기계 방식.
7. 구현 디테일: 실측 대신 flash error model(P/E cycling, retention Gaussian, RTN 지수분포)로 RBER 계산, 오버헤드는 상수 테이블(<16B) + cross-layer variation table(예: 6 stage×66 layer×4bit=198B) + 블록당 stage 3bit.
8. 별도 학습 절차 없음(경험적 error model 기반 stage 테이블 사전 계산).
9. 결과: inter-state 단독 13.6%, intra-state 단독 17.5%, cross-layer 단독 최대 23.3%(stage 6) read latency 감소; 3종 결합 시 stage에 따라 4.7~44.1%, 최대 45% read latency 감소(DiskSim, MSR trace 기반).
10. 한계: LDPC 디코딩 알고리즘/부호구조 자체는 불변(순수 채널 sensing 정책), 시뮬레이션(DiskSim)뿐 실칩 HW 미설계, error model 파라미터가 특정 공정·칩(Micron B16A, SK Hynix 3D-V4) 실측치 기반이라 타 제조사 일반화는 불확실.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| inter/intra-state 비대칭 sensing level 배치 | [channel.cpp](../../../Prime_ECC_3.1_Claude) `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold 위치 결정 로직에 비대칭 배치 정책 결합 가능 |
| 레이어별(워드라인) 적응적 시작 sensing level(n0) | `channel.cpp` (soft-read 양자화 threshold 설정부) | 워드라인 인덱스별 파라미터화 가능하나 Prime ECC 프로파일엔 레이어 개념 없음 — 상위 컨트롤러 정책 필요 |
| LDPC decoding fail 시 sensing level 반복 증가(progressive read-retry) | `ecc_top.cpp` (시뮬 오케스트레이션, 통계) | 조기종료·재시도 루프와 유사하나 CRC 기반 조기종료(`partial_CRC.cpp`)와는 다른 채널단 반복 |
| 부호설계/디코더 알고리즘 자체 | 대응 없음 | 논문이 건드리지 않는 영역 |

> 적용 가치: **중간** — decoder 알고리즘과 무관하지만 soft-read LLR 입력 품질(threshold 위치)을 개선하는 채널단 기법으로 `channel.cpp`의 LLR threshold 설정 로직에 정책적으로 결합 가능. NAND 실측 오류 비대칭성 활용이라는 방향성은 Prime ECC의 NAND 타깃과 직접 부합.
