### B. 알고리즘 요약

1. (J,K)-regular LDPC 코드(design rate `R=1-m/n`)의 sequential(node-wise) BP 디코더 스케줄링을 다룸, 대상 코드는 (3,6)-regular 및 (3,7)-array-based 코드, block length n=196.
2. 기존 node-wise scheduling(NS, [4])은 매 스텝 모든 CN residual `r_{m_a→v} = |m'_{a→v} - m_{a→v}|`(식(1))을 실시간 계산해야 해 연산량이 flooding보다 큼 — 이 residual 계산 비용을 줄이는 것이 목표.
3. CN 스케줄링을 Markov Decision Process로 모델링: Tanner graph를 m-armed slot machine(각 CN=arm)으로 보고, 상태는 CN별 quantized soft syndrome `ŝ(j)=Σ H_{j,i} L̂(i)`(M-level scalar quantizer `g_M`), reward는 스케줄된 CN의 최대 residual `R_a=max_{v∈N(a)} r_{m_a→v}`.
4. 정책 추정 기법 2가지: (a) Gittins Index(GI) — 각 CN을 독립 bandit으로 가정, 선형 복잡도, 식(2)/(6); (b) Q-learning — 분포 가정 없음, 상태공간이 CN 수에 지수적으로 증가하는 문제를 해결하기 위해 CN을 크기 z 클러스터로 묶는 clustered Q-learning 제안(식(7),(8)).
5. 핵심 식: `Q_{ℓ+1}(s_u,a_u)=(1-α)Q_ℓ(s_u,a_u)+α[R_ℓ(s_u,a_u,f(s_u,a_u))+β·max Q_ℓ(f(s_u,a_u),a_u')]` — 클러스터별 상태·행동공간으로 학습 복잡도를 완화.
6. (3,3) absorbing set(ABS) 관점의 정성적 분석: sequential 스케줄링이 flooding 대비 (3,3) ABS로 인한 오류를 더 잘 정정한다고 주장(에러 전파 순서 논거).
7. 구현: 학습은 오프라인(runtime 아님), 학습된 정책 π_G(GI, MAB-NS-1) 또는 π_Q(clustered Q-learning, MAB-NS-2)을 Algorithm 1 Step 12의 CN 선택에 적용.
8. 학습 하이퍼파라미터: α=0.1, β=0.9, ε=0.6, cluster size z=7, M=4-level quantization, ℓ_max=25, 학습 데이터 |L|=2.5×10^7((3,6)-regular)/5×10^7((3,7)-AB).
9. 결과: MAB-NS-2가 BER·평균 메시지 전달량 모두에서 flooding/NS/MAB-NS-1보다 우수(Table I, 예: SNR=0dB에서 (3,6)-regular 메시지수 flooding 16070 대비 MAB-NS-2 280), AB 코드처럼 구조화된 코드에서는 이득이 줄어듦.
10. 한계: block length 약 200의 짧은 코드에서만 검증(Q-learning 런타임 복잡도 제약), 향후 과제로 긴 코드 확장을 명시, HW 구현·게이트카운트·throughput 분석 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN 스케줄링 정책(MAB-NS, sequential node-wise) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 flooding 기반 C2V→V2C 스케줄(Dual-Update로 even/odd 교대)이며, 논문의 학습 기반 sequential/랜덤 순서 스케줄링과 구조가 상이 — 이터레이션 루프 로직 교체 필요 |
| CN residual 기반 reward, quantized soft syndrome | `channel.cpp` `Set_LLR_Th()` | LLR 양자화(M-level)라는 개념은 대응되나, 논문의 residual/syndrome 학습 파이프라인 자체는 대응 없음 |
| Gittins index / Q-learning 오프라인 학습 | 대응 없음 | Prime ECC에는 RL 기반 정책 학습 모듈이 없음(고정 LLR threshold 테이블만 존재) |

적용 가치: 중간 — sequential 스케줄링으로 iteration/메시지 전달량 감소라는 목표는 Prime ECC의 관심사(avg iteration·latency)와 일치하나, 학습 기반 정책은 실시간 하드웨어 디코더에 내장하기 어렵고 짧은 코드에서만 검증되어 고rate/긴 QC-LDPC로의 이식 검증 부담이 큼.
