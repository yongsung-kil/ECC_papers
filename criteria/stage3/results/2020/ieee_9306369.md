### B. 알고리즘 요약

1. 대상은 5G NR 채널코드(3GPP TS 38.212) irregular QC-LDPC(BG1/BG2 base graph, `Zc` 확장계수), AWGN 채널 가정.
2. 풀려는 문제: check matrix 종류/크기가 표준마다 달라 매번 전용 디코더를 설계하는 비용이 크므로, 임의 check matrix에 대해 디코더 아키텍처를 자동 생성하는 프레임워크가 필요.
3. 채널모델은 AWGN, LLR `λn = ln(2yn/σ²)` (식 4)로 표준적 계산.
4. 핵심 기법: min-sum 근사 `αmn = (Π sign(βmn´)) · min|βmn´|` (식 9)로 sum-product의 `tanh` 연산을 제거해 HW 친화적으로 근사.
5. 식 (9)는 row processing(CN 갱신)의 핵심으로, 최소 절댓값과 부호 XOR만으로 tanh 없이 근사 메시지를 계산해 HW 복잡도를 낮춤.
6. 확장: 컬럼처리 모듈(β 갱신, 식 13)까지 포함한 전체 아키텍처를 given check matrix H로부터 RTL(Verilog-HDL) 자동 생성.
7. 구현 디테일: SRAM α/β를 각각 N/M 뱅크로 분할해 메모리 충돌 방지, 뱅크별 read/write 스케줄을 사전 계산해 ROM에 저장; 5G BG 구조상 `wc=1`인 컬럼은 β값이 반복 내내 불변이므로 해당 컬럼처리 회로를 제거(BG1 44Zc개, BG2 38Zc개 감소).
8. 별도 학습/최적화 절차 없음(구조 기반 규칙적 회로 축소만).
9. 결과: Zc=20(정보비트 약120)에서 3.7M gates@0.13um(축소 미적용), 회로면적 절감률 약2%(축소 적용 시 8,097~80,886 gates 감소, 미미), BG2 기준 최대 throughput 468Mbps(I=10 iteration).
10. 한계: 정보비트 길이가 실제 5G 실용 규모로 커지면 게이트 수가 과다해 구현 어려움 저자 스스로 인정, 제안 면적 축소 기법은 논리합성이 이미 자동 최적화해 효과가 기대보다 작음(~2%), HW 실측(FPGA/ASIC 테이프아웃) 없이 합성 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum row/column processing (식 6~13) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 알고리즘과 동일 근사이며 신규성 없음 |
| SRAM α/β 뱅크 분할 및 사전 스케줄링(메모리 충돌 방지) | 디코더 HW 모델(z=32, HCU, GT), `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 z=32 병렬 처리에서 유사한 메모리 라우팅 이슈를 다루므로 뱅킹/스케줄 아이디어는 참고 가능하나 표에 명시된 대응 함수 없음 → 대응 없음에 가까움 |
| 임의 check matrix용 RTL 자동생성 프레임워크 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(base+shift) 고정 구조라 범용 자동생성 도구 자체는 이식 난이도 높음(§4 표: 부호설계 이식 난이도 "높음") |

적용 가치: 낮음 — min-sum은 중복 기여이고, 5G 표준 전용 자동생성 툴과 미미한 면적 절감은 Prime ECC의 고정 QC-LDPC HW 구조에 직접 이식하기 어려움.
