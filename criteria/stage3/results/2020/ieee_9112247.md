### B. 알고리즘 요약

1. Protograph 기반 spatially-coupled(SC) LDPC 코드 설계 문제. Sliding window(SW) 디코딩으로 capacity-approaching 성능과 낮은 latency를 동시에 달성하려는 목적.
2. 기존 QC-SC-LDPC 구성법은 최적화 기반 short cycle 최소화에 의존해 coupling width `ω`가 2 이하로 제한됨(복잡도 문제). 큰 `ω`가 고정 latency 대비 더 나은 성능을 준다는 사실이 동기.
3. 핵심 가정: girth(최단 사이클 길이)는 lifted graph가 base graph의 girth보다 작을 수 없다(Lemma 1). base matrix `B`의 protograph coupling으로 SC base matrix `B_SC`를 구성.
4. 핵심 기법 1단(Stage 1/2 설계): `B_SC`를 `ω+1`개 component matrix `B_0,...,B_ω`로 분해하고, representative block `B_R`과 constituent block `B_C` 내 4-cycle을 제거하도록 초기화(Design Rule 1) 후 check-and-flip 절차(Design Rule 2)로 나머지 4-cycle 제거해 girth≥6 보장.
5. 핵심 식: `θ3-θ9 = (θ2-θ5)+(θ4-θ7) mod M`(Fossorier 조건, 식10) — circulant shift 값 `θ`를 선택해 6-cycle이 lifted graph에 생기지 않도록 하는 조건.
6. 핵심 기법 2단(확장): girth-6 protograph를 QC(circulant) lifting하며 Fossorier 조건으로 6-cycle까지 제거해 최종 girth≥8 `H_SC` 획득(non-periodic/periodic 두 방식 제시).
7. 부수 트릭: `nv-nc≥nc`(rate≥1/2)일 때 `Ξ` 행렬의 max column weight를 1로 낮춰 4-cycle 방지를 단순화; periodic lifting은 첫 `ω+1`열만 설계 후 반복해 복잡도 절감.
8. 최적화 절차: DE/GA/NN 없음. 휴리스틱 flip 기반 설계(Design Rule 1/2) + 최소 필요 `ω`는 exhaustive search로 표(Table I) 도출.
9. 결과: (nc,nv)=(3,7), M=7, L=60, rate=0.56, length=2940bit 코드에서 girth 8(설계) vs girth 6(OO-CPO 비교대상)로 BER 10^-8에서 1.3dB 개선.
10. 한계: HW 미설계(순수 알고리즘/시뮬), BP 디코딩 threshold 분석은 BEC 채널 기준(waterfall 성능은 girth 설계와 무관하다고 명시), SC(convolutional) 구조 전용이라 block code로의 직접 이식 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 4-cycle/6-cycle 제거 protograph 설계(Design Rule 1/2) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 이미 4-cycle-free RS-LDPC 마스킹 QC-LDPC 사용, SC(convolutional) 구조가 아니라서 설계 방법론 자체 이식 어려움 |
| Fossorier 조건 기반 QC circulant lifting | `ecc_top.cpp` `Load_PCM()` | circulant shift 설계 원리는 개념적으로 유사하나 Prime ECC는 이미 특정 base matrix에 고정된 block code, 부호설계 이식 난이도 최고(§4) |
| Sliding window 디코딩 | 대응 없음 | SC-LDPC 전용 디코딩 스케줄로 Prime ECC의 block code 이터레이션 루프(`decoder.cpp` `LDPC_Decoder()`)와 구조적으로 대응 없음 |
| BP 디코딩 threshold 분석(BEC) | 대응 없음 | 이론 분석 도구로 코드/HW 대응 없음 |

적용 가치: **낮음** — SC-LDPC(convolutional) 전용 부호설계·girth 최적화 이론이며, Prime ECC는 고정 QC-LDPC block code로 부호 구조가 근본적으로 다르고(sliding window vs block decoding) 이식 시 재검증 부담이 매우 큼.
