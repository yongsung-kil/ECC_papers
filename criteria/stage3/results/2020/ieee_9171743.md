### B. 알고리즘 요약

1. 대상은 flash memory(및 DVB-S2) 데이터 신뢰성 확보로, LDPC 단독은 waterfall 성능은 좋지만 error floor가 flash 요구 저오류율을 만족 못한다는 문제의식.
2. 해결책으로 BCH(outer)와 LDPC(inner)를 직렬(serial) 대신 **병렬(parallel) β-branch 구조**로 concatenate: 입력비트를 β그룹으로 나눠 각각 BCH 인코딩 후 합쳐 LDPC 인코딩.
3. LDPC는 column weight 3의 규칙적 parity-check 행렬 `H=[A|I_{n-k}]`을 사용하며 (93,186), (155,310), (217,434), (310,620) 4종 코드길이로 실험(모두 rate 0.5).
4. BCH는 (26,31) 고정 코드를 β=3,5,7,10개 병렬 분기로 사용, LDPC 코드길이는 β와 매칭되도록 선택.
5. LDPC 복호는 bit-flipping 또는 belief propagation(BP) 알고리즘, 최대 iteration=5로 설정하여 AWGN 채널에서 시뮬레이션.
6. 복호 순서: 수신 비트 -> LDPC 복호 -> β그룹 분할 -> 각 그룹 BCH 복호(Berlekamp 알고리즘 + Chien search로 오류위치다항식 근 계산).
7. 결과(BER=10^-3 기준): 3-branch는 단일 (93,186) LDPC 대비 0.3dB, 5-branch는 0.15dB, 7-branch는 0.07dB, 10-branch는 0.06dB 개선.
8. 7-branch와 10-branch 간 개선폭은 0.01dB로 미미해 HW 자원 대비 이득이 낮다고 판단, 7 parallel BCH를 최종 선택.
9. 결론: 병렬 BCH 분기 수가 늘수록 BER 개선되나 HW 자원 소모도 증가하는 트레이드오프이며, error floor 완화가 확인됨.
10. 한계: HW 설계·gatecount·throughput 없이 시뮬레이션(AWGN, 5 iteration)에만 그치고, 코드가 저rate(0.5)·소형(n≤620)이라 NAND 고rate QC-LDPC(~0.9)와 스펙 차이가 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC bit-flipping/BP 디코딩 루프 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반 soft/hard 복호를 이미 보유, 단순 BP/bit-flipping은 이미 상회하는 기법 |
| BCH outer code 병렬 concatenation(error floor 개선) | 대응 없음 | Prime ECC는 CRC 기반 조기종료(`partial_CRC.cpp`)만 있고 BCH concatenation 구조 없음 - 신규 아이디어이나 저rate 소형코드 검증이라 직접 적용성 낮음 |
| 조기종료(iteration 상한 5) | `decoder.cpp` 이터레이션 루프 | 단순 max-iteration 종료로 Prime ECC의 partial CRC 조기종료보다 기초적 |

> 적용 가치: 낮음 - BCH-LDPC 병렬 concatenation 아이디어는 error-floor 대책으로 참고할 만하나, rate 0.5·소형 코드·HW 미설계 시뮬레이션 수준이라 고rate QC-LDPC 이식 시 재검증 부담이 큼.
