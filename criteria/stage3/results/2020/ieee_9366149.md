### B. 알고리즘 요약

1. AWGN 채널을 통해 전송된 rate-1/2 PEGReg504x1008 (n=1008) irregular LDPC 코드에 대한 Bit-Flipping 계열 디코더를 다룸.
2. 기존 GDBF/NGDBF는 휴리스틱 기반 성능 개선에 의존하며 이론적 프레임워크가 부재하다는 문제를 제기.
3. 채널 메시지 샘플 오류와 syndrome 정보 간의 조건부 확률을 통계적으로 정량화하는 것을 핵심 가정으로 삼음.
4. 3차원 행렬 MIPFM을 인덱스 `(xq, p, ℓ)` (xq=하이포시스×양자화값, p=syndrome, ℓ=iteration)로 구성, `P(r,s) = E(r,s)/H(r,s)`로 각 bin의 플립 확률을 정의.
5. `Ek = xk*yk + Σsi` 형태의 GDBF 목적함수를 확장해 syndrome 기반 오류 확률을 매핑, 확률적으로 비트를 플립하는 방식.
6. 확장 기법으로 NGDBF의 inversion function을 이용해 초기 PFM을 구성하고, 이후 반복마다 `PF(x_{h+1}) = Nh/(Nh+1)*PF(xh) + e_{h+1}/(Nh+1)` 베이지안 갱신으로 확률을 튜닝.
7. Q=8비트 균일양자화, Ymax=2.5 클리핑, T=100 iteration, 200,000 프레임으로 MIPFM 사전 구축 후 2000 프레임으로 시뮬레이션.
8. 학습/최적화는 θ, η(NGDBF inversion 파라미터)를 SNR별 empirical search로 결정(표 제공), 바이어싱 파라미터 B=10도 경험적 결정.
9. 결과: PLMLBF는 SNR 3.75~4dB에서 M-NGDBF(threshold adaptation 없음) 대비 성능 우위, threshold adaptation 있는 M-NGDBF와는 0.25dB 이내로 근접. SNR 4dB 이후 trapping set로 error-floor 진입.
10. 한계: HW 미설계, 순수 시뮬레이션(단일 코드·소규모 프레임 검증), trapping set 대응 휴리스틱 부재로 고SNR 영역 성능 열위.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PLMLBF Bit-Flipping 디코딩 전체 | 대응 없음 | Prime ECC는 Min-Sum 기반(`decoder.cpp` `LDPC_Decoder()`)이라 BF 계열 알고리즘 자체가 대응 없음 |
| MIPFM 확률 룩업테이블(soft 정보 활용) | `channel.cpp` `Set_LLR_Th()` | LLR threshold 테이블 구조와 개념적으로 유사하나 BF 전용 구조라 직접 이식 불가 |
| syndrome 기반 조건부 확률 계산 | 대응 없음 | Prime ECC는 min-sum 메시지패싱 방식이라 syndrome 조건부 확률 테이블 개념이 없음 |

> 적용 가치: 낮음 — Bit-Flipping 계열 알고리즘으로 Prime ECC의 Min-Sum 디코더 구조와 근본적으로 다른 계열이며, HW 미설계·단일 소형 코드 시뮬레이션에 그쳐 이식 실익이 낮음.
