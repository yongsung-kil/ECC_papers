### B. 알고리즘 요약

1. Polar code (N=1024, K=512, systematic encoding), AWGN 채널, encoding factor graph를 LDPC-like sparse decoding graph로 프루닝하여 변환 (선행연구 [8] 기반).
2. 기존 Arıkan BP는 encoding factor graph 위에서만 동작해 SCL 대비 성능이 열세이고, parity-check 기반 그래프는 dense/short-cycle이라 BP가 실패하는 문제를 해결하려 함.
3. LDPC-like sparse graph 위에서 메시지 업데이트 순서(스케줄)가 Polar 복호 성능에 LDPC보다 더 큰 영향을 준다는 관찰을 핵심 가정으로 삼음.
4. Modified NW-RBP(node-wise residual BP): 잔차 `r(c→v) = |rnew(c→v) − rold(c→v)|`가 가장 큰 edge를 포함하는 CN을 선택해 업데이트, `mc→v`는 `φ(x) = −log(tanh(x/2))` 함수를 사용한 SPA 식(식(1)) 기반.
5. 식(1)(C2V)·식(2)(V2C, `mv→c = Livni + Σ mc′→v`)는 표준 SPA 메시지 전달이며, 잔차 기반 스케줄이 수렴 속도와 최종 BER을 동시에 개선.
6. Schedule diversity 확장: 첫 시도는 LBP, 이후 최대 `Imax=10`번까지 잔차 상위 `Γ=5`개 CN 중 무작위 선택으로 서로 다른 디코딩 결과를 생성해 다양성 확보.
7. 부수 트릭: 동일 잔차값이 같은 CN에 여러 개 몰릴 경우 후보 CN 풀을 Γ개로 유지하도록 탐색 범위를 확장.
8. 별도 학습/최적화 절차는 없음(휴리스틱 스케줄링 규칙).
9. 결과: 고SNR에서 SCL(L=8,32) 대비 낮은 연산복잡도, BER은 고SNR 영역에서 SCL 대비 약 0.2dB 열세.
10. 한계: HW 미설계, 시뮬레이션(AWGN)만 수행, Polar code 전용(LDPC 부호 자체 대상 아님), 정확 gate/throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Modified NW-RBP 잔차 기반 CN 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 스케줄 다양화 아이디어는 원리상 참조 가능하나 대상이 Polar이며 잔차 계산 오버헤드가 커 우리 min-sum HW 구조와 정합 낮음 |
| SPA 기반 C2V 메시지(φ 함수, tanh) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 min-sum(HW 친화) 사용, full-SPA/tanh는 대응 없음 |
| Schedule diversity(다중 시도 무작위 스케줄) | 대응 없음 | 반복 복호 재시도 구조로 latency/HW 오버헤드 커 현재 아키텍처에 없음 |
| LDPC-like sparse graph 변환(Polar→LDPC) | 대응 없음 | 부호 자체가 Polar이므로 QC-LDPC PCM 로드(`Load_PCM()`)와 무관 |

적용 가치: **낮음** — 대상 부호가 Polar이고 LDPC는 그래프 구조 차용에 불과하며, 이미 보유한 min-sum/dual-update 스케줄링과 중복되거나 대응 모듈이 없음.
