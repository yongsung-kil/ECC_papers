### B. 알고리즘 요약

1. 대상은 IEEE 802.3an (2048,1723) 및 802.15.3c (672,336)/(672,588) regular LDPC 코드의 stochastic(확률적) SPA 디코더 — 채널은 unary bit-stream(Bernoulli 확률)으로 LLR 대신 확률값 `In`, `Pm→n`, `Qn→m`을 곱셈 없이 AND/OR/MUX 논리로 근사 계산.
2. 기존 fully-parallel MSA 디코더는 배선 혼잡(routing congestion)으로 면적·클록이 저하되는 반면, 기존 stochastic 디코더는 확률 트레이서(EM/TFM/MTFM)의 면적 비용이 크고(EM이 전체 면적 60%+) 3-3 입력 그룹핑 구조가 부정확했던 문제를 해결.
3. 핵심 가정은 stochastic logic gate의 상관관계(SCC, stochastic computation correlation)가 출력 정확도를 좌우한다는 것 — 저자는 VN의 degree-2 sub-node가 correlation divider(CORDIV)와 동일 구조임을 SCC 분석(`SCC(VN)=+1`)으로 규명.
4. 핵심 기법은 기존 3-3 입력 그룹핑을 5-1 그룹핑(degree-6 VN: 5개 C2V bit 합산 후 IM 기반 재랜덤화, +1개 채널 bit)으로 대체해 division 단계를 줄이고 확률 트레이서(probability tracer) 크기를 8/64bit EM에서 소형 up/down saturated counter(4-bit)로 축소.
5. 상관관계 식 `Q = PXPY / (PXPY+(1-PX)(1-PY))`(수식 7) 분석으로 "정정 계산은 부정확하다 → 중간 division 단계를 줄여야 정확도 상승"이라는 설계 지침 도출.
6. 확장 기법으로 무편향(unbiased) 확률적 시퀀스 생성기를 제안, 소형 카운터(3-bit)에서 발생하는 편향(bias≈1/8)을 MSB 처리로 제거.
7. HW 디테일: SBF(soft bit-flipping) prior-processing(초기 5 DC 동안 활성화)과 C2V 기반 hard-decision 갱신을 결합해 디코딩 전 구간에서 latency 단축, re-decoding으로 error-floor 완화.
8. 별도 학습 절차 없음(설계 파라미터는 시뮬레이션 기반 trade-off 분석 — 4-bit tracer/1-bit hard-decision 선택).
9. 결과: TSMC 65nm 합성 시 (2048,1723) 코드 2.34mm², 970MHz, ADC 17.1(4.4dB)/4.3(5.5dB), throughput 116.17Gbps(4.4dB)/461.99Gbps(5.5dB); 기존 최고성능 stochastic 디코더[24] 대비 ADC 42.3%로 감소, throughput 113.5%/378.7% 개선.
10. 한계: stochastic computing 특유의 unary bit-stream 표현이라 LLR/min-sum 기반 구조와 완전히 다른 패러다임이며, 정정성능 손실 방지에는 성공했으나 여전히 MSA 대비 절대 지연 사이클 수가 많고(최대 700 DC), 순수 시뮬레이션·합성 검증만(FPGA/실측 없음, IEEE 802.3an 표준 채널 가정).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic SPA VN/CN 연산 전체 | 대응 없음 | Prime ECC는 min-sum LLR 기반이며 stochastic bit-stream 연산과 자료구조가 근본적으로 달라 대응 모듈 없음 |
| 확률 트레이서(EM/TFM/MTFM/counter) | 대응 없음 | Prime ECC의 LLR 테이블(`ecc_data.h` `PARAM_LLR`)과 목적은 유사(정보 보존)하나 구현 원리가 상이 |
| SBF prior-processing / C2V 기반 hard-decision 가속 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 조기 반복 가속이라는 개념은 Prime ECC의 조기종료(Partial CRC)·적응형 LLR과 목적상 유사, 구현은 이식 불가 |
| BF post-processing / re-decoding (error-floor 대책) | `decoder.cpp` `LDPC_Decoder()` | error-floor 완화 목적의 재시도 개념만 참고 가능, 알고리즘 자체는 min-sum 구조에 적용 불가 |

적용 가치: 낮음 — stochastic computing 패러다임 전체가 Prime ECC의 min-sum/LLR 기반 아키텍처와 이질적이라 회로·알고리즘 직접 이식은 불가하며, "상관관계 분석을 통한 조기 가속" 같은 설계 아이디어만 참고 수준.
