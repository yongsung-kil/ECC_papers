### B. 알고리즘 요약

1. QC-LDPC 부호 `H`는 mL×nL 크기이며 circulant permutation matrix `P_ast`(shift 지수 `ast`)와 zero matrix `P_-1`로 구성된다.
2. 기존 lifting 방식(modulo-lifting)은 girth와 최단 cycle 개수만 줄이려 하나, 항상 성능 개선으로 이어지지 않는다는 문제를 지적한다.
3. 제안 기법 LPMP는 PMP(parallel vector message passing) 알고리즘으로 얻은 cycle 정보를 활용해 새 지표 Cycles Metric(`CM`)을 정의한다.
4. `CM(vj,ci) = Σ_{l=2}^{g-1} α^{l-2} · N_{j,i}(2l)`로, edge `E_j,i`를 지나는 길이 `2l` cycle 개수 `N_{j,i}(2l)`를 가중합한다(`0<α<1`, 짧은 cycle에 더 큰 가중치).
5. 전체 Tanner graph의 `CM = Σ_j Σ_{ci∈C(vj)} M(vj,ci)`이며 CM이 작을수록 BER 성능이 좋다.
6. lifting 절차: 기존 부호 `C0(H0)`의 exponent matrix `E(H0)={ast}`에서 컬럼 가중치가 낮은 순으로, 가능한 `bst = ast + k·Z0` (`k=0..q-1`, q=lifting 배수) 후보 각각에 대해 `CM_k`를 계산하고 최소값을 선택해 새 부호 `C1(H1)`을 구성한다.
7. HW 친화 트릭이나 양자화 관련 서술은 없음(부호 구성 알고리즘에 국한).
8. 학습/최적화는 CM 최소화를 위한 greedy 탐색(컬럼 가중치 순 순차 대입)이며 DE/GA/NN은 사용하지 않는다.
9. 결과: IEEE 802.11ad Code (648,486, rate 0.75)를 2배·3배 lifting 시 기존 대비 각각 0.2dB, 0.4dB BER 개선; Code2 (2304,384)는 2배·3배 모두 약 0.2dB 개선.
10. 한계: PON/광통신 채널(AWGN) 시뮬레이션뿐이며 HW 설계·복잡도·throughput 분석 없음, decoder 변경 없이 순수 구성법 비교임.
