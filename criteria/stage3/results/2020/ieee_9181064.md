### B. 알고리즘 요약

1. 5G NR(TS 38.212) QC-LDPC 부호(base graph BG1: mb=46,nb=68 / BG2: mb=42,nb=52, lifting factor Z=2~384, 최대 (25344,8448) rate 1/3 부호)를 vRAN(가상화 무선접속망) 환경의 GPU에서 디코딩.
2. 문제: 5G NR은 51가지 lifting factor와 2개 base graph, 8개 set index로 매우 다양한 코드율/블록길이를 지원해야 하며, HW(FPGA/ASIC) 고정 디코더로는 이 유연성 대응이 어려움. eMBB(고처리량)/URLLC(저지연)/mMTC(다중접속)라는 상충 요구도 존재.
3. 채널/디코딩 모델: 표준 min-sum algorithm(MSA)의 two-min 근사(scaled) 기반 offset/scaling 변형을 사용, SPA 대비 낮은 복잡도.
4. 핵심 기법: CN 커널(각 스레드가 H의 한 row 담당, two-min 기반 메시지 계산)과 VN 커널(각 스레드가 H의 한 column 담당, APP LLR 갱신)로 분리해 GPU CUDA 블록/스레드에 매핑. 다중 코드워드를 매크로코드워드(MCW)로 묶어 병렬 처리.
5. `Hx=0`을 만족하는 유효 코드워드 검증이 기본 원리이며, 여러 CW를 하나의 CUDA 블록 내에서 동시 평가하고 여러 MCW를 배치 전송해 블록 단위 병렬성 확보.
6. 부수 최적화: LLR 워드길이를 8비트 char로 축소(데이터 전송시간 최대 4배 절감), hard-decision 결과를 비트 단위로 패킹, 다중 CUDA stream으로 연산-전송 오버랩, H 행렬을 row-major/column-major 압축본으로 constant memory에 캐싱.
7. 학습/최적화 절차 없음(고정 알고리즘의 소프트웨어/HW 매핑 최적화).
8. 결과: (25344,8448) 부호·5 iteration 기준 BER은 float/half 정밀도와 8비트 char가 근사(char에서 소폭 열화); throughput은 TITAN RTX에서 char 데이터타입 사용 시 3964Mbps(전송시간 포함), latency는 최적 설정 시 87µs(OAI 기존 CPU 구현 178µs 대비 51% 감소, 이때 throughput 290Mbps로 저하).
9. Open Air Interface(OAI) NR 소프트웨어 스택에 통합해 vRAN 컨테이너화 가능성 검증, CRC 정합 확인.
10. 한계: GPU 기반 소프트웨어 구현으로 ASIC/FPGA 게이트카운트·전력 등 HW 합성 결과 없음; 정정성능 개선 기법 없이 표준 MSA 그대로 사용; 5G 통신(eMBB/URLLC/mMTC) 특화 요구사항 기반이라 NAND 저장장치 맥락과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| two-min 기반 scaled min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 보유한 Min-Sum과 동일 계열 알고리즘, 신규 기여 아님 |
| CN/VN 커널 분리 및 GPU SIMT 병렬화 | 대응 없음 | Prime ECC는 bit-exact HW(RTL) 시뮬레이터로 GPU/CUDA 소프트웨어 병렬화 구조 자체가 적용 대상이 아님 |
| 8비트 LLR 양자화 | `channel.cpp` + LLR 테이블(`PARAM_LLR`) | 양자화 비트수 절감이라는 방향성은 유사하나 Prime ECC는 이미 HD/2SD/3SD 등 NAND read 비용 기준의 별도 양자화 체계를 보유, 5G 통신용 8bit 양자화 결과는 직접 참고가치 낮음 |
| 5G NR base graph/lifting factor 유연성 지원 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(z_sb=32, N_b=129) 고정 구조라 다중 lifting factor 지원 개념은 대응 없음 |

> 적용 가치: 낮음 — GPU 소프트웨어 플랫폼과 5G 통신 표준(vRAN)이라는 맥락이 NAND ASIC 기반 Prime ECC와 근본적으로 다르고, 알고리즘 자체도 이미 보유한 min-sum 변형이라 이식 가치가 거의 없음.
