### B. 알고리즘 요약

1. 시스템: rate 1/2~1/3 turbo code(RSC 두 개, 인터리버), AWGN 채널, BPSK/16-QAM, K=40~140.
2. 문제: 기존 max-log-MAP 알고리즘은 log 항 생략으로 성능 열화, 기존 data-driven DNN 디코더(FC-DNN, RNN/LSTM)는 학습 파라미터 과다·긴 부호에서 수렴 실패.
3. 핵심 기법: max-log-MAP의 반복 구조를 그대로 M개 "DNN decoding unit"으로 unroll(model-driven), 각 유닛은 forward/backward recursion(`ᾱ_k(s)`, `β̄_k(s)`) 및 γ 계산에 학습가능 가중치 삽입.
4. 핵심 식: extrinsic LLR `Le(uk)=L(uk|y)-y_k^s-L(uk)` (식 6)에 가중치 `w_e1,w_e2,w_e3` 적용(식 20); 가중치=1이면 원 max-log-MAP과 동일 성능 보장.
5. 손실함수: `Loss = (1/K)Σ(L_M(uk|y) - L^T_log-MAP(uk|y))^2` (식 21) - T iteration log-MAP을 교사로 삼아 gradient vanishing(sigmoid 포화) 문제 회피.
6. 확장: 가중치 히스토그램 분석 후 GW(감마)·PLW(posterior LLR) 가중치를 제거하고 ELW(extrinsic LLR)만 남긴 TurboNet+로 가지치기, 파라미터 수 150MK → 6MK로 축소.
7. 학습 절차: SGD/ADAM, 학습 SNR 탐색(NVE, 경험식 (24)) + early stopping 기반 훈련전략으로 overfitting 방지.
8. 결과: TurboNet+가 max-log-MAP 대비 BER 0.3~0.5dB 이득, log-MAP(6회 반복) 근접 성능을 log-MAP 대비 약 20배 빠른 단일 forward pass로 달성; 5G RaPro USRP 기반 OTA 실측(4개 실내 시나리오)에서도 강건성 확인.
9. 추가 결과: neuralBCJR/DeepTurbo(LSTM 기반 data-driven) 대비 고SNR에서 약 1.5dB 우위.
10. 한계: GPU 학습만 다루고 ASIC/FPGA HW 합성 없음, LDPC가 아닌 turbo code(trellis/BCJR 구조) 전용이라 QC-LDPC Tanner graph 기반 min-sum 디코더에 알고리즘 구조 자체는 이식 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| max-log-MAP 학습가능 가중치(TurboNet/TurboNet+) | 대응 없음 | Turbo code trellis/BCJR 구조 전용, Prime ECC는 LDPC Tanner graph(min-sum) 구조라 알고리즘 골격 자체가 다름 |
| Extrinsic LLR 스케일링(ELW, 식 20) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 개념적으로 min-sum CN 메시지에 학습된 offset/scale factor를 넣는 아이디어(참고문헌 [28] neural offset min-sum)와 유사하나 본 논문은 turbo code 대상이라 직접 대응 아님 |
| 반복구조 unroll + 지도학습(log-MAP teacher) | `decoder.cpp` `LDPC_Decoder()` | LDPC iterative message passing에 유사 방법론(iteration별 학습가능 파라미터) 적용 가능성은 있으나 본 논문 자체는 LDPC 미다룸 |
| OTA 실측(5G USRP-RIO) | 대응 없음 | NAND 채널이 아닌 무선(OFDM/QPSK) 환경, Prime ECC 채널 모델과 무관 |

> 적용 가치: 낮음 — Turbo code trellis 구조 전용 DNN 디코더로 LDPC Tanner graph/min-sum 구조에 알고리즘이 직접 이식되지 않으며, "학습가능 가중치로 반복 디코더 개선"이라는 방법론적 아이디어만 참고 가치가 있음(다만 관련 원조 논문 [27][28][29]이 LDPC weighted-BP/neural min-sum을 직접 다루므로 그쪽이 더 적합).
