### B. 알고리즘 요약

1. Rate 8/9, length 9216bit QC-LDPC(AA-LDPC, architecture-aware) 코드, 16×144 base matrix를 64×64 서브행렬로 확장(row weight 27, column weight 3).
2. 목표: 4-line loop(4각 사이클) 없는 규칙 LDPC용 H행렬을 임의 code length/row weight/column weight 조합으로 생성하는 방법 제안.
3. H행렬 생성 절차: base matrix 랜덤 생성 → 64×64 sub-matrix 위치에 순환이동 단위행렬 삽입 → shift 값 랜덤 선택 → 4-line loop 검사, 위반 시 재시도.
4. 디코더는 TDMP(turbo-decoding message-passing) 알고리즘 사용: `Γ=δ,Λ=0`으로 초기화 후 `ρk=Γk−Λk`(층별 변수노드 메시지) → `Λk=Φ⁻¹(Σj≠k Φ(ρj))`(외부메시지) → `Γ=ρ+Λ`(사후메시지 갱신) 반복.
5. 체크노드 외부메시지 계산은 Log-MAP(BCJR) grid로 수행: `q(x,y)=ln(1+e^(x+y))−ln(e^x+e^y)`, forward `α`/backward `β` 재귀 계산 후 `Λ(i)=q(α_{i-1},β_{i+1})`.
6. Log-MAP의 지수/로그 연산을 Jacobi 로그 근사로 단순화: `q(x,y)=max(x,y)−max(x+y,0)` (덧셈/뺄셈만 필요, HW 친화).
7. HW 구현 디테일: scoreboard(Scboard)로 TDMP 초기화와 1차 iteration 병합 시 발생하는 데이터 충돌 해결, 64개 SISO 처리유닛 병렬(블록행 단위), 외부메시지 6bit·사후메시지 8bit saturated addition 저장으로 메모리 60% 절감.
8. 별도 학습/최적화 절차 없음(H행렬 생성은 규칙 기반 랜덤 탐색, DE/GA/NN 미사용). Chisel 고수준 언어로 설계 후 Verilog 생성, UVM으로 10,000개 랜덤 벡터 검증.
9. 결과: 55nm ASIC 기준 1.96mm² 면적, 290MHz 클록, 10 iteration에서 450Mbps throughput (FPGA도 동일 throughput, 기존 [7] 대비 클록·throughput 우위).
10. 한계: BER/FER 등 정정성능 데이터 없음(순수 처리량/면적 비교), Log-MAP(sum-product 계열)이라 Prime ECC의 min-sum과 알고리즘 계열이 다르고, 정정능력 개선 여부는 판단 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP + Log-MAP(Jacobi 근사) 디코딩 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | sum-product 계열 근사(min-sum과 다른 계열)이나 이터레이션 루프 구조 참고 가능 |
| scoreboard 기반 초기화-1st iteration 데이터 충돌 해결 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update(짝/홀 교대 스케줄)와 유사한 문제의식, 병합 가능성 |
| 외부/사후메시지 6bit/8bit saturated addition 저장 | `decoder.h` `Get_VNU_*`, `ecc_data.h` PARAM_DEC | 메시지 비트폭 절감 기법으로 참고 가능 |
| AA-LDPC(QC-LDPC) H행렬 생성(4-line loop 회피) | `ecc_top.cpp` `Load_PCM()` | 부호설계 레이어, 이식 난이도 높음(Prime ECC는 특정 H-matrix 고정) |
| Chisel/Verilog HW 구조(SISO 64-way, pipeline) | 디코더 HW 모델(z=32, HCU, GT) | 병렬화 단위(64-way)가 Prime ECC의 z_sb=32와 규모가 달라 직접 대응 어려움 |

적용 가치: **중간** — scoreboard 기반 파이프라인 충돌 해결과 메시지 비트폭 절감(6/8bit) 기법은 참고할 만하나, 핵심 디코딩 알고리즘(TDMP+Log-MAP)이 Prime ECC의 min-sum 계열과 다르고 정정성능 검증이 전무해 직접 이식보다는 설계 참고 수준.
