### B. 알고리즘 요약

1. 대상은 자기기록매체(MR)용 protograph 기반 QC-LDPC 코드 4종(IARA, 2-D-P1/P2, Nested high-rate ISI, RCOP), rate 4/5, codeword 길이 약 4140~5175 bytes.
2. 기존 MR LDPC 연구는 부호설계/디코더 이론에 집중되어 고속 인코더 HW 구현이 부족하다는 문제를 다룸.
3. 채널 모델은 논문 대상이 아니며(참조 문헌의 partial-response/1D·2D ISI 채널), 이 논문은 순수 인코더 HW 설계에 집중.
4. 핵심 기법은 저자들의 기존 CCSDS LDPC 인코더 아키텍처([9])를 재사용: 부호를 `c = [s p]`로 분할하고 `pT = H2^-1 H1 sT`로 파리티 계산.
5. `H1`은 희소행렬이라 매 사이클 `rLm`비트를 조합논리로 계산해 LFSR에 공급, `H2^-1`은 조밀 circulant 배열이라 LFSR 구조로 처리(`Lm`이 병렬도 파라미터).
6. 각 코드의 protograph 노드 재배열로 `H2^-1`의 특이 구조(0/단위행렬 블록)를 활용해 곱셈 연산량을 축소 (예: IARA는 첫 x비트 생략, 2-D-P/RCOP/Nested ISI는 `p2 = [u0...u(xm-1)]`로 직접 대입).
7. 인코더는 AMBA AXI4-Stream 인터페이스의 FPGA IP core로 구현, 시프트레지스터+순열/합산 모듈+LFSR 조합.
8. 별도 학습/최적화 절차 없음(부호설계는 선행 문헌의 EXIT chart/AWD 기반 구성 재사용).
9. Zynq XC7Z045-2 SoC 합성 결과: IARA 25.28 Gbps @395MHz(51138 LUTs), 2-D-P/RCOP 22.02 Gbps @367MHz(39007 LUTs), Nested ISI 22.50 Gbps @375MHz(38082 LUTs).
10. 한계: 인코더만 다루며 디코더/정정성능은 미설계, 검증은 FPGA 동작검증(GNU/Octave golden model 비교)까지이며 ASIC 합성 결과 없음, MR 채널 특화라 NAND 직접 적용성은 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LFSR 기반 partitioned-H 인코딩 (`pT = H2^-1 H1 sT`) | [encoder.cpp](Prime_ECC_3.1_Claude) `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코딩이라 구조가 다르나 QC 인코더 병렬화 아이디어는 참고 가능 |
| protograph 노드 재배열로 H2^-1 단순화 | [encoder.cpp](Prime_ECC_3.1_Claude) `Column_Reordering_Dual_Update()` | 코드 구조가 다른 특정 부호 고정 설계라 직접 이식은 어려움 |
| H-matrix/부호 구조(IARA, 2-D-P, RCOP 등) | [ecc_top.cpp](Prime_ECC_3.1_Claude) `Load_PCM()` | MR 채널 특화 부호로 NAND용과 무관 |

> 적용 가치: 낮음. 인코더 HW 최적화 아이디어는 Prime ECC의 dual-diagonal 인코더와 목적이 겹치나, 대상 코드가 MR 특화 protograph이고 디코더/정정성능 기여가 전혀 없어 직접 이식 가치는 제한적.
