### B. 알고리즘 요약 (10줄 내외)

1. TLC NAND flash (n=3bit/cell), 128 wordline/block, 16KB/wordline, regular LDPC rate=0.9, column degree 5, min-sum decoder, 최대 50 iteration으로 시스템 모델 구성.
2. 기존 read reference voltage 추정은 retention time을 정확히 알아야 하나, 컨트롤러는 온도 민감성으로 인해 정확한 retention time을 알 수 없음이 문제.
3. 채널 손상은 P/E cycle과 retention time에 의한 threshold voltage distribution 왜곡(RTN + retention noise)으로 모델링.
4. 단일 wordline 센싱·디코딩으로 얻은 4가지 feature `{FER(φ), avg iteration(ω), RBER(η), BFR(γ)}`를 각 페이지타입(MSB/CSB/LSB)별로 산출, 총 (4n+1)차원(P/E cycle 포함) feature vector `x` 구성.
5. 3가지 추정기 제안: k-NN(`l* = argmax_l Σ I(λ(z),l)`), NC(centroid 최근접 `argmin||y-x̄_l||²`), PR(`m`차 다항식 회귀, `w=(X^T X)^{-1}X^T v_label`).
6. 저장 오버헤드/연산복잡도는 k-NN이 `MKd`로 가장 크고, NC/PR은 M(클래스당 샘플수)에 무관해 훨씬 작음.
7. Wrapper 방식 반복 feature selection 알고리즘(Algorithm 1)으로 MSE 개선폭이 fidelity parameter `δ` 이하가 될 때까지 차원 `d`를 늘려 최소 차원 결정(실험상 d*=2).
8. 64-stack TLC V-NAND 에뮬레이션 채널모델(실측 정합도 93%)로 학습셋 구축, off-line 학습 + on-line 추정(단일 wordline 센싱만 필요).
9. 결과: d=2 이상에서 optimal sensing 대비 near-optimal FER 달성, read-retry(NWL=1) 대비 latency 약 2.98배 개선, valley-tracking 대비 1.5~2배 낮은 latency.
10. 한계: 실제 NAND 칩이 아닌 에뮬레이션 채널 모델 기반 시뮬레이션뿐이며, LDPC 디코더/부호 자체는 개선 대상이 아니라 feature 추출용 도구로만 사용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum LDPC 디코딩(feature 추출용) | `decoder.cpp` `LDPC_Decoder()` | 디코더 알고리즘 개선이 아닌 부산물(iteration count, FER) 획득 용도 |
| read reference voltage 추정 (k-NN/NC/PR) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold 결정 로직과 개념적으로 유사하나 ML 기반 추정 자체는 대응 모듈 없음 |
| retention time 미지 상황 대응 | 대응 없음 | Prime ECC 프로파일에 retention-time 추정/read voltage 캘리브레이션 모듈 없음 |
