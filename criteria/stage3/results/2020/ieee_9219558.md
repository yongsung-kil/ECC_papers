### B. 알고리즘 요약

1. 대상은 5G NR 표준 QC-LDPC(BG1, `mb×nb`=46×68 base graph, lifting `Z`≤384), 코드레이트 1/3~5/6, 길이 2088~2160, GPU(RTX 2080Ti) 소프트웨어 구현.
2. 5G NR의 넓은 rate/length 범위와 20Gbps급 요구 처리량을 GPU로 고속 디코딩하려는 문제(기존 GPU 디코더는 WiFi/WiMax 등 짧은 고정 code에 한정).
3. AWGN 채널, QPSK 변조, Eb/N0=3dB 가정으로 성능 측정.
4. 레이어드 min-sum(scaled MSA, `α`=0.75)을 Z개 스레드로 병렬화, layer별로 `Lij`, `Rij` 갱신(`L_new = L_old + R_new`).
5. min1/min2(`f`,`s`) two-pass(global/local pass) 방식으로 `O(d_i^2)` 복잡도를 선형화하고 nested loop 제거, 부호 곱셈을 XOR 기반 sign 갱신으로 대체.
6. Multi-chunk decoding으로 블록당 `Np×Z` 스레드 할당해 다중 코드블록 동시 디코딩.
7. 2단계 양자화: LLR은 4bit(대역폭·오버플로 방지), 중간값 L/R은 8bit로 양자화하고 8bit 단위 data packing으로 CPU-GPU 전송 오버헤드 절감.
8. 학습/최적화 절차 없음. 대신 SM 자원(register/shared mem/thread/block) 제약식(`(18)`,`(19)`)으로 최적 thread 배치를 코드rate별로 도출.
9. 결과: (2080,1760) r=5/6에서 1.385Gbps 처리량, data packing으로 15%+ 개선, multi-chunk로 40%+ 개선(vs 순차 대비).
10. 한계: GPU 소프트웨어 구현으로 ASIC/FPGA HW 설계 없음, early termination 미적용 상태로 측정, 5G NR 표준 코드 고정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 레이어드 min-sum 디코딩(scaled MSA) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 min-sum 알고리즘과 동일 계열, GPU 병렬화만 신규 |
| min1/min2 two-pass 계산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 로직 자체는 기존 CNU 연산과 유사, 새 기여 아님 |
| 4bit/8bit 2단계 양자화 | `channel.cpp` `Set_LLR_Th()` | soft-read 비트폭 설계 참고 가능하나 GPU 대역폭 최적화 목적으로 방향 다름 |
| GPU 스레드/메모리 배치 최적화 | 대응 없음 | Prime ECC는 RTL HW 시뮬레이터로 GPU 병렬화 개념 자체가 무관 |

적용 가치: 낮음 — 5G NR 표준 코드용 GPU 소프트웨어 디코더로, NAND binary QC-LDPC HW(RTL) 이식 경로가 없고 알고리즘(min-sum/레이어드 디코딩)도 이미 Prime ECC에 구현되어 있어 중복 기여.
