### B. 알고리즘 요약

1. polar code(N=512/1024, R=0.5/0.65, 24-bit CRC) factor graph 기반 belief propagation(BP) 디코딩을 다루며 채널은 AWGN+QPSK.
2. 기존 BP는 factor graph의 cycle/stopping set으로 인해 error floor가 발생해 CA-SCL 대비 성능이 열세이고, 이를 개선하는 기존 multi-trellis BP(FP-BP, 전체 factor graph permutation)는 permutation마다 중간 변수노드 값을 버려 반복수(`Nit,reset`)가 크게 필요.
3. 채널/에러 모델: false-converged/oscillation/unconverged 3종 오류로 분류, 오실레이션은 factor graph의 짧은 cycle(girth=12)에서 기인.
4. 핵심 기법(PP-BP, Algorithm 1): factor graph 전체가 아닌 `ρrange`(2~`Prange`) 레벨 폭, `ρlevel`(1~`Plevel`) 시작 레벨의 subgraph만 무작위 선택해 permutation, 나머지 변수노드 LLR은 보존.
5. 핵심 식: `Nit,reset = max(Nmin, Nzero/D)`, `Nzero = 2^(ρlevel+ρrange-1) × (ρrange-1)` — 변경된 변수노드 수에 비례해 다음 permutation까지 반복수를 동적 조절, permutation을 더 자주 수행하게 함.
6. boxplus 연산은 `sign(x)sign(y)min(|x|,|y|)`(non-scaled min-sum)로 근사해 구현 복잡도 절감(polar BP의 PE 연산).
7. HW 구현 관점 언급: PP-BP는 FP-BP와 동일 permutation 하드웨어 재사용 가능, 추가 연산은 `Nit,reset` 계산뿐(정성적 언급, 수치 미제시).
8. 학습/최적화 절차 없음 — `Prange`, `Plevel`, `D`, `Nmin`은 실험적으로 결정된 고정 설계 파라미터(Table I).
9. 결과: N=1024, R=0.5, FER=1e-6에서 FP-BP(전체 permutation) 대비 0.25dB 이득 및 반복수(latency) 감소 동반.
10. 한계: polar code 전용(LDPC 아님), HW 합성/FPGA 검증 없이 시뮬레이션뿐, R=0.65(고rate)에서는 오히려 noise-injection 기법(NA-BP)이 더 우수해 범용성 제한적.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar code factor graph BP 디코딩 전체 | 대응 없음 | 프로파일상 turbo/polar는 명시적으로 대응 없음 항목 |
| subgraph 부분 permutation을 통한 정보 보존형 스케줄링 아이디어 | `decoder.cpp` `LDPC_Decoder()` | 개념적으로 layered/shuffled 스케줄과 유사할 수 있으나 polar factor graph 구조 전제라 직접 적용 불가, 참고 수준 |
| non-scaled min-sum(boxplus 근사) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 이미 min-sum(min1/min2+sign XOR)을 보유해 중복 기여 |

적용 가치: `낮음` — polar code 전용 BP/factor-graph 기법으로 Prime ECC의 binary QC-LDPC Tanner-graph min-sum 디코더와 부호·그래프 구조가 근본적으로 달라 직접 이식 불가.
