### B. 알고리즘 요약

1. 시스템: (3,6) 규칙 LDPC(`R=1/2`, `n=8000`, `k=4000`, DB명 8000.4000.3.483), BPSK/AWGN, 2/3/4-bit 양자화.
2. 문제: 기존 LUT 기반 복호기는 채널 출력 양자화기와 LUT 복호기를 **분리** 설계 → 저해상도에서 성능 손실.
3. 모델: all-zero 무한길이 codeword 가정 하 discretized density evolution으로 CN/VN LLR 분포 추적.
4. 핵심기법: CN은 `f(L1,L2)=2tanh⁻¹(tanh(L1/2)tanh(L2/2))`, VN은 `g(L1,L2)=L1+L2`를 2-input LUT 사슬로 근사하고 각 단마다 양자화 경계 `τ`로 `2^q` 레벨로 매핑.
5. 핵심식: `Perr ≈ Σ p_L(lmax)[Li]` (식10) — 최종 iteration LLR 분포의 오류확률을 최소화 목표로 설정.
6. 확장: 채널 경계 `τ_ch`, CN 경계 `τ_CN`, VN 경계 `τ_VN`를 공동 최적화 `{τ_ch*,τ_CN*,τ_VN*}=argmin Perr` (식24).
7. 구현 디테일: 설계 시 real-value 매핑함수 사용, 실복호에서는 정수 LUT(`Lq...int ∈{0..2^q-1}`)로 대체.
8. 최적화: DE 기반으로 오류확률 최소가 되도록 모든 매핑함수 경계를 탐색.
9. 결과: IBM[8] 대비 waterfall에서 우위 — 이론상 2/3/4-bit에서 각 0.241/0.065/0.023 dB, 2-bit 시뮬 gain 0.05 dB.
10. 한계: HW 미설계, 규칙(3,6) rate 0.5 부호·AWGN 한정, gate/throughput/메모리 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 채널 출력 양자화기 경계 최적화 | channel.cpp Set_R_Offset(), Set_LLR_Th() | soft-read LLR threshold 설계와 직접 대응 |
| CN/VN LUT 매핑 (min-sum 대체) | decoder.cpp CNU_Update_New_Mag(), VN_Cal_HD() 등 | min-sum을 LUT로 치환하는 접근, 구조 재검증 필요 |
| iteration별 양자화 매핑 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | iteration 구간별 LLR 테이블 개념과 유사 |
| DE 기반 파라미터 자동 최적화 | local_opt.cpp (LLR 자동최적화) | 오류확률 최소화 파라미터 탐색과 유사 |
```

적용 가치: **중간** — 채널 양자화기+LUT 공동 최적화로 저해상도 soft-read 성능을 끌어올리는 아이디어는 NAND 2~3bit 연판정 LLR 설계에 참고되지만, min-sum을 LUT로 전면 대체·(3,6) 규칙부호·AWGN 가정이라 Prime ECC의 고rate QC-LDPC/min-sum 구조에 그대로 적용하려면 재설계 부담이 크다.
