### B. 알고리즘 요약

1. 대상 코드: rate-0.5 (2640,1320) Margulis 코드와 rate-0.3 (640,192) QC 코드, 둘 다 SPA(sum-product) 플로팅포인트 디코더 기준, AWGN 채널.
2. 문제: LDPC 반복복호는 저가중치 trapping set(고정 또는 진동하는 subgraph)에 갇혀 error floor가 발생, 기존 코드설계/스케줄링 개선만으로는 한계.
3. 가정: trapping set은 사전 시뮬레이션·importance sampling(`Pr{ξT}` 식 (2))으로 식별 가능하며, 특정 코드의 지배적 trapping set(예: Margulis (12,4)/(14,4), QC (5,5)/(5,7))을 표적으로 디코더를 커스터마이징.
4. 기법1 - bi-mode syndrome-erasure decoder: 1차 SPA 복호 후 syndrome weight가 알려진 trapping set과 일치하면 2차 모드로 전환, 미충족 CN에 연결된 VN들을 레벨별 트리(`L1~L4`)로 소거(erasure) 처리해 순수 BEC로 만든 뒤 반복 소거복호로 해결.
5. 핵심 절차: Margulis (12,4)/(14,4) 구분을 위해 공통 level-3 CN을 갖는 두 VN(`va1, va2`)를 flip하는 보조 알고리즘(Algorithm 1)으로 352개 erasure를 생성, 3~4 iteration 내 복구.
6. 기법2 - concatenation+bit-pinning: outer BCH 코드로 trapping set의 특정 비트를 정정 후 LDPC 디코더에 해당 LLR을 최대값으로 pin, 4개 BCH(132,124)/(77,70)/(55,49)/(33,28) 조합 시 code rate loss 0.086dB로 최소화.
7. 기법3 - G-LDPC decoder: 미충족 CN들을 combined constraint(GCP, generalized-constraint processor)로 묶어 BCJR 알고리즘으로 처리, G-LDPC III는 trapping set 쌍의 CN을 결합(660 GCP, 2-SPC)해 Margulis에서 FER~10⁻⁸까지 floor 미관측.
8. 학습/탐색 절차: two-step importance sampling으로 G-LDPC 디코더의 새 trapping set을 재탐색(Step a: 편향 −2 인가 후 실패패턴 기록, Step b: Gaussian bias로 안정 패턴 확정).
9. 결과: bi-mode는 Margulis에서 FER~10⁻⁸까지 floor 미관측(SPA 대비 수 자릿수 개선), G-LDPC III는 SPA 대비 복잡도 증가 200% 이하로 유사 성능.
10. 한계: 순수 시뮬레이션(floating-point SPA) 검증뿐 HW 미설계, trapping set은 디코더 구현(quantization 등)에 종속적이라 실제 min-sum/양자화 디코더에서는 재식별 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bi-mode syndrome-erasure post-processing (2차 모드 erasure decoding) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 내 syndrome 기반 조건 분기 및 erasure(LLR=0) 처리 추가로 이식 가능성 있음 |
| bit-pinning (LLR 최대값 고정) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | outer code 없이 pinning 개념(LLR 강제 고정)만 부분 이식 가능, outer BCH 자체는 대응 없음 |
| G-LDPC / GCP (BCJR 기반 constraint 결합) | 대응 없음 | Prime ECC는 min-sum 기반 SPC 구조 고정, BCJR 트렐리스 결합 구조는 근본적으로 다른 디코더 아키텍처 |
| trapping set 식별용 importance sampling | `ecc_top.cpp` (시뮬 오케스트레이션) | 오프라인 분석 도구 성격, 코드 자체 수정 없이 별도 스크립트로 병행 가능 |

> 적용 가치: 중간 — bi-mode erasure decoding은 원리상 이식 난이도가 낮고 error-floor 개선 효과가 크지만, G-LDPC(BCJR)는 Prime ECC의 min-sum 아키텍처와 구조적으로 맞지 않아 부분 이식만 가능.
