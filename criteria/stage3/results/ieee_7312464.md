### B. 알고리즘 요약

1. 대상은 QC-LDPC 블록/convolutional 부호와 GF(2^m) 기반 non-binary QC-LDPC 부호로, 다항식 parity-check 행렬 `H(D)`(monomial 원소 `D^wij`)로 통일 기술한다.
2. 목표는 주어진 length/rate 하에서 BP 복호 시 FER을 최소화하는 `H*`를 찾는 것 (`H* = argmin_H FER(H, SNR)`), 이를 위해 girth·girth profile·sliding-window girth(SWG) 등 조합적 파라미터를 탐색 기준으로 사용.
3. 채널 모델은 기본적으로 AWGN(BPSK)이며, 4절에서는 Faster-Than-Nyquist(FTN) 신호로 인한 제어된 ISI 채널도 최적화 루프에 포함시킨다.
4. 핵심 기법은 2단계 탐색: (1) base matrix `B`를 열 단위로 무작위 생성하며 공통위치수 `C1`, 공통삼중항수 `C2`, girth `C3`, ACE류 지표 `C4` 순으로 스코어링, (2) 이미 정해진 base matrix에 대해 degree matrix `W`(voltage 라벨)를 girth·SWG 제약을 만족하도록 탐색(Algorithm 2, 3).
5. girth 계산은 voltage graph 기법으로 가속화되며, 전체 탐색 복잡도는 `|T| ~ [(Jmax-1)(Kmax-1)]^(3log(c-b+2))`로 base matrix 크기에만 의존하고 lifting factor `M`(코드 길이)에는 무관함이 이론적으로 증명됨(정리 1: `g ≤ 4(c-b+1)` 또는 `g ≤ 8(c-b-1)`).
6. 낮은 인코딩 복잡도를 위해 parity-check 행렬을 bidiagonal 구조(`H_bd(D)`)로 제한하며, 이는 인코딩 복잡도를 코드 길이에 선형으로 유지(식 27-38의 재귀식으로 인코딩).
7. non-binary 확장은 base matrix를 GF(q) 원소(companion matrix)로 라벨링하며, "row code" 뱅크에서 최소거리 4 이상인 후보를 사전 구축해 탐색 복잡도를 낮춘다.
8. 최적화는 조합 지표(girth 등)로 후보를 예비 스크리닝한 뒤, 소수 후보에 대해 Monte-Carlo BP 시뮬레이션(FER 추정)으로 최종 선정하는 그리디 방식.
9. 결과: WiMAX/DVB-S2 표준 대비 개선된 QC-LDPC(R=1/2, n=576~2304에서 뚜렷한 SNR 이득; n=64800, R=1/2에서 DVB-S2 대비 0.1~0.15dB 이득), FTN 신호에 최적화된 부호는 DVB-S2 표준 대비 최대 0.6dB, 채널-loop 최적화로 추가 0.1~0.2dB 이득.
10. 한계: 순수 부호구성 논문으로 디코더 HW/구현은 다루지 않으며(BP 복호만 시뮬레이션), FER 기반 exhaustive-ish 탐색은 계산비용이 커 실전 재현/재검증 부담이 크다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth/SWG 기반 base matrix·degree matrix(H-matrix) 탐색 알고리즘 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 단계에 이론적으로 대응되나 Prime ECC는 QC-LDPC 구조 고정이라 이식 난이도 높음 |
| bidiagonal 구조 기반 저복잡도 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 인코딩과 개념적으로 유사하나 논문은 부호설계 관점, Prime ECC는 이미 구현 완료 |
| non-binary(GF(2^m)) LDPC 이진 이미지 구성 | 대응 없음 | Prime ECC는 binary LDPC 전용, non-binary 기법은 이식성 낮음 |
| FTN 채널 모델을 최적화 루프에 포함 | 대응 없음 | NAND 채널(RBER/fixed-error) 특화가 아닌 통신 채널(AWGN, ISI) 대상 |
| convolutional LDPC(sliding window decoding) | 대응 없음 | Prime ECC는 QC-LDPC 블록 부호 전용, convolutional 구조 미보유 |

적용 가치: **낮음** — 순수 QC-LDPC/convolutional/non-binary 부호 구성·girth 최적화 이론이며 HW/디코더 설계가 없고, Prime ECC는 이미 고정된 QC-LDPC 구조를 사용하므로 H-matrix 재설계는 이식 난이도가 가장 높은 영역에 해당.
