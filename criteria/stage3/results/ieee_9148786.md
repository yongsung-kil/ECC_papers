### B. 알고리즘 요약

1. (n,k,d) 블록코드(Hamming (7,4,3), extended Hamming (8,4,4), (10,5,4) parity check, CWC (10,5,4,5) 비선형 부호, 연접부호 2종)를 AWGN 채널에서 디코딩.
2. 기존 single-label 신경망 디코더는 출력노드가 `2^k`로 지수 증가해 큰 k에 비현실적이라는 문제를 해결하고자 함.
3. 채널 모델: AWGN, 심볼 `{-1,+1}` BPSK 매핑, noise variance `N0/2`.
4. 핵심 기법: multi-label classification 디코더 — 출력층 노드 수를 `2^k`가 아닌 `k`로 설정, 각 정보비트를 독립적으로 Sigmoid 출력.
5. 손실함수는 binary cross-entropy(`E(a,x) = -Σ[a·log(x)+(1-a)·log(1-x)]`)로 각 출력비트를 개별 최적화 → BER은 최적에 근접하나 BLER은 열화.
6. 확장: 연접부호(inner+outer parity-check code)도 동일 구조의 신경망으로 end-to-end 학습.
7. 구현 디테일: hidden layer는 ReLU, output layer는 Sigmoid, feed-forward 구조, 2 hidden layer가 1 hidden layer보다 항상 우수.
8. 학습: SNR_T=0dB에서 지도학습(무작위 메시지 생성 → 부호화 → BPSK 변조 → AWGN 추가), binary cross-entropy 최소화.
9. 결과: multi-label 디코더는 BER 기준 ML-SD에 근접하나 BLER은 열세, hidden node 수는 single-label 대비 초선형 증가(예: (7,4,3)에서 7→28개), 연접부호에서는 natural decoding(내부 ML+외부 다항시간)보다 BLER 우수.
10. 한계: 극소형 블록코드(n=7~20)만 검증, HW 설계 없음, LDPC/QC-LDPC 미포함, BLER이 ML-SD 대비 근본적으로 열등한 구조적 한계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| multi-label 신경망 분류 디코더 | 대응 없음 | NN/딥러닝 디코더는 프로파일상 대응 없음 항목으로 명시됨 |
| Hamming/parity-check/CWC 부호 | 대응 없음 | binary QC-LDPC 전용 코드베이스이며 non-QC-LDPC 블록코드 미지원 |
| 연접부호(inner+outer) 디코딩 | 대응 없음 | Prime ECC는 단일 QC-LDPC 복호기 구조로 연접 구조 없음 |

적용 가치: `낮음` — 신경망 기반 소형 블록코드(비-LDPC) 디코더로, Prime ECC의 binary QC-LDPC min-sum HW 구조와 알고리즘·부호 모두 무관.
