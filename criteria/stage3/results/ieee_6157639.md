### B. 알고리즘 요약

1. 대상은 (491,3,6) 시변 LDPC 컨볼루셔널 코드(LDPC-CC), 주기 3, rate 1/2~5/6(puncturing), IEEE 802.16m(WiMAX) 스펙, `d_v=3`/`d_c=6` regular.
2. 문제: LDPC-CC 디코더는 지연·전력이 크고 처리량이 수백 Mb/s로 LDPC 블록코드 대비 열세.
3. 채널은 AWGN/BPSK, 6-bit LLR 입력 양자화(6,2), NMS scaling factor 0.875.
4. 알고리즘 최적화: on-demand variable node activation(OVA) 스케줄링을 채널값 은닉(OVA-CC)으로 변형 — VN-to-CN 메시지에 채널값을 숨겨 채널값 저장 FIFO 제거.
5. 핵심식: `(7)(8)`의 두 VN-to-CN 메시지가 공통항을 공유 → 이전 CN-to-VN 메시지 빼고 최신 것 더해 갱신, 채널값 불필요.
6. 효과: OVA 25 iter ≈ flooding 50 iter, 수렴 2배 가속 + FIFO 약 17% 축소, 5 processor로 log-BP 10 processor 성능 달성(latency 절반).
7. 노드 최적화: folding(factor 12)로 CNU/VNU 복제·shift register 접기 → 처리량 = 12×clock, 12.4% HW 오버헤드로 12배 throughput.
8. 비트 최적화: sub-VNU retiming으로 critical path 단축·메시지 bit-width 축소(20% 저장 감소).
9. hybrid-partitioned FIFO: 긴 연속 구간을 2-port 메모리 뱅크로 병합, 나머지는 레지스터 → clock buffer 54% 감소로 저전력.
10. 결과: 90nm CMOS, 2.24mm² 코어, 2.37Gb/s@198MHz/1.2V/284mW, 0.8V로 스케일 시 1.58Gb/s/90.2mW.
