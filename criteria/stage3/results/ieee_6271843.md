### B. 알고리즘 요약

1. 대상 부호는 rate-1/2 `(491, 3, 6)` time-varying LDPC convolutional code (`ms=491`, column weight 3, row weight 6), BPSK/AWGN 채널.
2. 기존 log-BP LDPC-CC 디코더는 파이프라인 프로세서 `I`개당 latency `δ*I`, storage `M*I`로 메모리·지연이 과대 → 이를 줄이는 것이 목표.
3. Stochastic decoding: LLR을 Bernoulli 확률 스트림으로 변환, CN 갱신은 XOR, VN 곱셈/나눗셈은 AND gate + J-K flip-flop으로 구현 (식 (5),(6)).
4. 핵심 아이디어: layered decoding처럼 매 decoding cycle(DC)마다 CN·VN 메시지를 즉시 갱신해 최신 메시지를 활용, 수렴속도 향상 (Algorithm 1).
5. Latching 문제 대책으로 NDS(noise-dependent scaling, `α/Y` 계수, 여기서 0.6)와 tracking forecast memory(TFM) 기반 probability tracking 채택.
6. 슬라이딩 윈도우 `L`의 boundary effect로 virtual edge 메시지가 소실되어 BER이 uncoded보다도 나빠지는 문제 발생.
7. 이를 3-phase로 보상: Initial phase는 채널 LLR 스트림으로, Normal phase는 CN→VN 메시지로, Terminating phase는 decoded bit 정보를 담은 TFM 출력으로 register 갱신.
8. VN 결정은 counter `cntk`를 메시지·`R(t)`에 따라 ±1, 최종 `cntL≥0`이면 1로 판정.
9. 결과: `(491,3,6)` 부호에서 log-BP(10 processors) 대비 동일 SNR에서 더 나은 BER, latency 60% 감소, storage 40% 감소 (L=2000, 8-bit 입력양자화, 6-bit counter).
10. 한계: convolutional 부호·stochastic 방식 한정, error-floor·gate count·throughput 수치 없이 시뮬 위주 검증.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|

| 즉시 갱신 스케줄(layered 유사) 이터레이션 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 개념적 유사(dual-update 이미 보유), 그러나 stochastic이라 직접 대응 아님 |
| NDS / 채널 LLR 스케일링 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | soft-info 스케일 개념만 느슨히 대응 |
