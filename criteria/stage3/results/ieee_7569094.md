> 총평: NAND QC-LDPC의 단일 layer parity violation 수로 rate penalty 없이 on-die BER를 추정하는 저면적(28nm 0.015mm2) HW — 디코더는 아니나 soft-read/조기종료 트리거·메모리관리에 직접 활용 가능해 이식가치 중간.

### B. 알고리즘 요약

1. 대상 시스템은 NAND flash 컨트롤러의 QC-LDPC ECC(regular, row weight `K`, column weight `C`), MLC/TLC page 단위 read.
2. 문제: ECC 복호 전에 page/codeword BER를 알면 메모리관리(soft-read 트리거, programming error 완화, SLC→MLC 컴팩션)에 유리한데, EDC 방식은 parity overhead(rate penalty) 발생.
3. 핵심아이디어: QC-LDPC의 각 codeword bit은 한 layer당 정확히 1개 parity check에만 참여 → 단일 layer의 parity violation 수를 입력 bit error 수의 척도로 사용, rate penalty 없음.
4. 핵심기법: `BEREST = α × (Parity Violations / Sample Size)`, scalar `α≥1`은 짝수 오류 미검출로 인한 과소추정을 보정(binomial 분포 가정 시 해석적으로 최적 α 계산).
5. 식 의미: parity violation 수는 실제 오류수의 하한(짝수개 오류 미검출)이므로 α로 스케일해 실제 BER에 근접.
6. HW 친화 변형: barrel shifter를 die에 두지 않기 위해 write path에 interleaver를 넣어 한 layer의 parity check 참여 bit들을 연속 배치 → read path에서 K bit씩 serial XOR 후 누적. scalar 곱은 right shift+add로 구현(fractional 3-bit 양자화로 near-optimal).
7. 구현: 28nm, 333MHz, 0.015mm2 (worst corner), 64bit/cycle 입력, 최대 2 circulant/cycle parity 계산, 128bit alignment buffer, zero-pad 길이로 rate 조정.
8. 응용: (a) preread page 중 BEREST>BERTH만 컨트롤러 전송해 HRE 완화, (b) hard info 스트리밍 중 BER 추정해 hard 실패 예상 시 soft info 자동 준비(worst latency↓), (c) SLC→MLC 컴팩션 시 고오류 page만 선별 전송.
9. 결과: K=50, C=4/5/6 → rate 0.92/0.90/0.88, BER 2e-3·4e-3에서 sample size 2/4/16KB로 PDF 분석(sample 클수록 정확), throughput 2.6GB/s(ONFI 1.6GB/s 초과), 단일 사이클 latency 3ns.
10. 한계: 디코더가 아니라 pre-ECC 추정기라 정정능력 자체는 무개선, 짝수개 오류·정확 오류수 미검출로 하한만 제공, less-structured LDPC로 확장 시 중복 카운트로 정확도↓.
