### B. 알고리즘 요약

1. 시스템: AWGN 채널, 초단 linear/non-linear block code((7,4) Hamming, parity-check, BCH, constant-weight, ZTCC), rate `R=k/n`, 대상 `10≤k≤1000` 5G URLLC.
2. 문제: NBP 등 기존 NN 디코더는 iteration 고정·복잡도 과대, BP는 short-blocklength/HDPC에서 성능 저조 - 저지연·저에너지 NN 디코더 필요.
3. 모델: message `m∈{0,1}^k`→codeword `c=mG`→BPSK(±1)→AWGN `y=s+w`, 목표는 ML-SD(최우 연판정) 성능 근접.
4. 핵심 기법 1단: single-label 디코더 - 수신 `y`를 2^k codeword 중 하나로 분류(SoftMax+categorical CE), hidden layer 1개·노드수 `N1h≥n`이면 ML-SD BLER 달성(BLER 최적).
5. 핵심 식 의미: `al=σ(Wl·al-1+bl)` feed-forward, 각 hidden layer 노드가 blocklength `n` 미만이면 정보 손실로 ML-SD 미달(Hanin 함수근사 한계).
6. 핵심 기법 2단: multi-label 디코더 - 출력노드를 `k`로 축소(Sigmoid+binary CE), 노드 수는 k에 선형이나 BER만 최적화되고 BLER는 ML-SD와 격차 발생, hidden layer/노드 증가 필요.
7. 부수 기법: concatenated 구조 - inner code를 multi-label로, outer code(RS/parity-check)를 single-label로 복호해 multi-label의 잔여 block error 정정.
8. 최적화 절차: `(Eb/N0)train`(0~4.5dB)에서 지도학습, 복잡도는 곱셈수 `nmul (5)`·활성함수수 `ng (6)`·weight수로 산정.
9. 결과: single-label은 hidden 1층 `n`노드로 ML-SD 근접; concatenated는 multi-label 대비 노드수 소폭 감소하면서 BLER 개선, n 클수록 이점 큼; BP는 HDPC에서 열세, BCH는 BM이 우세.
10. 한계: HW 미설계·시뮬만, k 크면(예: ZTCC k=64) Viterbi보다 열세이고 학습·일반화 곤란, AWGN/Rayleigh 무선 채널·초단 부호 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| single/multi-label NN 디코더 전체 | 대응 없음 | 프로파일 예시대로 NN/딥러닝 디코더는 대응 모듈 없음 |
| ML-SD 벤치마크 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 우리는 반복 min-sum, full-search ML-SD와 알고리즘 상이 |
| concatenated inner/outer 부호구조 | 대응 없음 | RS/parity-check 연접, QC-LDPC 부호구조와 무관 |
| BER/BLER 성능 지표 | 대응 없음 | 평가지표, 코드 모듈 아님 |

적용 가치: 낮음 - NN 전면대체 디코더로 초단 블록부호(비-LDPC, 무선 URLLC)에 특화되어 NAND binary QC-LDPC min-sum 파이프라인에 떼어낼 요소가 없다.
