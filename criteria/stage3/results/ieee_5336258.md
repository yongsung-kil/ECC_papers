### B. 알고리즘 요약

1. 대상은 QC-LDPC 기반 min-sum column-layered(=shuffled) 디코딩으로, 코드워드를 K개 block column layer `C0...C_{K-1}`로 나눠 horizontal/vertical step을 순차 수행.
2. 기존 min-sum column-layered 디코딩은 check node당 최소/2번째 최소값을 매 층마다 재계산해야 해 비교 연산량이 큼(`d_c 관련 comparisons` 다수).
3. 각 check node는 이웃 변수노드로부터 온 메시지 크기 중 최소값 `m1`, 2번째 최소값 `m2`(개선판은 `m3`까지)와 그 인덱스 `j1,j2(,j3)`를 초기화 시 저장.
4. 매 반복(iteration)에서 새로 갱신되는 `Q_ij` 메시지 크기 `m`, 인덱스 `j`에 대해 `(j==j1 or j==j2) and (m>m2)`(개선판은 `m3` 기준) 조건으로 Process A(전체 재탐색, 비용 큼) 또는 Process B(캐시된 값 재사용, 비용 작음)를 선택 실행하는 selective computation을 적용.
5. Process B는 3회 비교, Process A는 `d_c`회 비교가 필요하며, `PR = N_B/N_A`(Process B 대비 A 실행 비율)가 클수록 절감 효과가 커짐 — `m3`을 추가로 캐싱해 Process A 발생 확률을 더 낮춘 것이 개선판의 핵심.
6. 개선판(modified selective computation)은 3번째 최소값 `m3`을 함께 유지함으로써 원판(selective computation, [8] 방식)보다 PR을 높여 Process B 비중을 늘림.
7. 부수 트릭: 각 check node를 독립 처리 단위(CNU)로 두고, Process B를 마친 CNU는 해당 layer 갱신이 끝날 때까지 deactivate하여 대기시키는 behavior model 제안(HW 회로 구현은 없음).
8. 별도 학습/최적화 절차 없음(수학적 유도 및 시뮬레이션 기반 검증).
9. 결과: WiMax 계열 코드에서 성능 열화 거의 없이 원 min-sum column-layered 대비 비교 연산 81.1~90.9% 절감(BER 10^-6 기준 Process B 비중 90.5~93.2%); (2048,1723) RS-LDPC(check node degree 32)에서는 선행연구([8]) 대비 성능 열화가 없음을 확인. 소프트웨어 디코딩 속도는 원 shuffled 디코딩 대비 5~10배.
10. 한계: throughput 증대에는 기여하지 않는다고 저자가 명시(모든 CNU가 동시에 Process B를 수행한다는 보장이 없어 고속 병렬 HW엔 부적합), HW 합성/gatecount 없이 시뮬레이션 및 개념적 behavior model 수준에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|

| 디코딩 이터레이션 루프(column-layered 스케줄) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 논문의 shuffled/column-layered 스케줄이 Dual-Update 스케줄과 유사한 층 단위 갱신 구조 |
| CNU 대기/deactivate behavior model | 대응 없음 | HW pipeline·edge 선택(GT)과는 다른 소프트웨어적 개념으로 RTL 대응 모델 없음 |

적용 가치: **중간** — CN 연산량 절감 기법 자체는 min-sum 디코더 코어에 낮은 난이도로 이식 가능하나, throughput/HW 이득이 저자 스스로 부정되어 병렬 HW인 Prime ECC 파이프라인에는 전력/면적 관점 외 이득이 제한적.
