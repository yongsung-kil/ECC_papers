### B. 알고리즘 요약

1. 대상: partial-parallel layered LDPC 디코더, AA-LDPC(QC-LDPC, base matrix + cyclic-shift identity), IEEE 802.11n rate 5/6 sub-block 81 예시, 802.16e/DVB-S2(180 layer)도 적용.
2. 문제: LDPC 디코더 전력의 주요분은 메모리 접근(특히 posterior reliability 저장 Channel RAM) → 접근량을 최적으로 줄이는 것이 목표.
3. 핵심 가정: layered(horizontal) 스케줄에서 연속 layer가 같은 column에 non-null이면 Add-array 결과를 Channel RAM 저장 없이 다음 layer로 직접 전달(memory bypassing) 가능.
4. 핵심 기법 1단: Channel RAM의 read/write 순서를 decouple → 연속 2 layer뿐 아니라 3 layer까지 확장해 bypass 가능한 모든 overlapped column 실현, idle cycle 최소화.
5. 핵심 식: layered 갱신 `L(qmn)=L(qn)-R(mn)`, min-sum 근사로 CN 출력은 최소 2개 magnitude+index+sign만 저장(dc 대신 2).
6. 핵심 기법 2단: bypass 총량은 layer 복호 순서에 의존 → 최적 순서 탐색을 TSP(완전그래프, 간선비용=layer간 overlap 수)로 정식화(NP-hard).
7. 최적화 절차: 소형부호는 brute-force, 대형부호(DVB-S2 180 layer)는 simulated annealing(layer 쌍 swap, cost=overlapped column 수)으로 근사 최적 순서 획득.
8. 아키텍처: read/write decouple로 Channel RAM+intermediate FIFO를 4-port 단일 SRAM으로 병합(면적↓), 수신 5bit 양자화·soft output 6bit, two-output SISO, 81 SISO 병렬, 최대 15 iteration.
9. 결과: 802.11n에서 Channel RAM 메모리접근 12.9~19.3%↓(802.16e 65.8~98.8%, DVB-S2 30.9~65.9% bypass 가능), idle cycle 1.0~13.2%↓, 전체 에너지 13.2~25.9%↓(TSMC 0.18µm, 250MHz, 1.8V).
10. 한계: 정정능력은 불변(순수 전력/면적/latency 최적화), layer 순서 SA는 오프라인 전처리 필요, 무선 표준 타깃(NAND 미언급), gate count/throughput 본문 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered 복호 이터레이션/스케줄 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 교대 스케줄) | Prime ECC도 layered/dual-update — memory-bypass는 스케줄 layer 순서 최적화로 접목 여지 |
| min-sum(two-output) CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유(min1/min2) — 기법 중복 |
| Channel RAM read/write decouple + 메모리 병합 | 대응 없음 (HW 메모리 모델) | Prime ECC HW 모델(z=32, pipeline)의 메모리 배치 최적화 참고 대상, 직접 함수 대응 없음 |
| soft output 6bit 양자화 | `channel.cpp` + `ecc_data.h` `PARAM_LLR` | Prime ECC LLR 테이블(HD/2SD/3SD)과 정합 확인 필요 |
| edge/MUX 기반 라우팅 축소 | `GT.cpp` `Make_GT_HW()` | 관점 유사(클록당 접근 축소)나 기법(memory-bypass vs graph-thinning) 상이 |

적용 가치: **중간** — 알고리즘(min-sum/layered)은 이미 보유하나, Channel RAM 접근을 최적 layer 순서 + read/write decoupling으로 줄이는 HW 저전력 기법은 Prime ECC의 메모리/라우팅 layer 면적·전력 최적화에 참고할 만하다(정정능력 개선은 없음).
