### B. 알고리즘 요약

1. 시스템은 outer binary (turbo) LDPC 디코더가 bit-detector와 반복(iteration)하는 "bit turbo-receiver"(iterative demapper), AWGN 채널·고차변조/MIMO/ISI/다중사용자 시나리오, 예시 rate `Rc=0.5`.
2. 문제: bit-interfered 채널에서 detector EXIT chart가 flat이 아니면(Gray 아닌 labeling) 기존 `S=0` 설계 LDPC는 최적성을 잃어 rate loss 발생.
3. 모델: detector/decoder의 (G)EXIT chart를 조건부 엔트로피로 정의, 대부분 거의 선형 거동을 보인다고 가정.
4. 핵심 기법: EXIT chart의 slope `S = 양끝 값 차이`를 rate·복잡도와 나란한 새 설계 파라미터로 도입, detector slope와 매칭시켜 최적 수신 성능 달성.
5. threshold `theta(lambda,rho,S) = min_x[E_I(x) - S*E_O(x)]`를 정의(EBP EXIT의 직선 `e_o=e_I-b` 최소 shift), `S=0`이면 통상 BP threshold와 일치.
6. 확장: quasi-regular(QR)/right-QR(RQR) LDPC 클래스 도입, density `eta`(평균 변수차수 `dv`) 단일 파라미터로 복잡도 표현.
7. 구현 트릭: 수정 stability condition `lim E_I(x)=(1-Rc)-S/2`로 최소 density QR 코드에서 출발(차수 2·3만).
8. 최적화: Algorithm 1 - 좌차수 다항식에 차수 `d=2+i*epsilon` 항을 greedy로 추가하며 threshold 개선될 때만 계수 증가, density `eta_T` 상한 내에서 최적 `lambda**(x)` 반환.
9. 결과: `Rc=0.5`에서 slope `S=0.2`·density `2.95`로 code efficiency 98.5%, slope `S=0`·density `3.3`으로 97%; slope 클수록 목표효율 대비 density(복잡도) 감소.
10. 한계: 순수 EXIT/BEC 이론 설계, HW 미설계·시뮬 BER 없음, hard/soft read 무관하며 NAND 응용 밖.
