### B. 알고리즘 요약

1. 대상: 유한길이 (variable-regular) LDPC 부호가 AWGN 고SNR에서 겪는 error floor의 원인 구조인 trapping set을 그래프이론으로 분석 (부호 rate/length 미지정, `dv=3/4/5` regular 가정).
2. 풀려는 문제: 같은 `(a,b)` 라도 trapping set마다 error floor 기여도가 다른데 기존 gain 지표는 이를 구분 못함.
3. 정의: `T(a,b)` = `a`개 variable node, 유도 부분그래프에 홀수차수 check node `b`개; elementary = degree-1/2 check만.
4. 1단(열거): Theorem 1로 `(a,b)` 존재 필요조건 `(dv+1)a - C(a,2) ≤ b ≤ a(dv-2)` 유도 후 그래프 툴 `nauty`로 모든 비동형 elementary trapping set 생성 (girth 제약별 개수 표).
5. 식 의미: 내부 edge 수 `(a·dv - b)/2` 가 완전그래프 edge 수 이하/연결성 이상이어야 함에서 경계가 나옴.
6. 2단(protograph 판정): trapping set을 단순 protograph에서 lifting으로 만들 수 있는지를 edge coloring으로 판정 - `χ'(T)=J` (class 1) 이면 등장 가능, 아니면 어떤 lift에서도 불가.
7. Theorem 2/3: Vizing 정리로 `χ'(T)` 는 `Δ(T)` 또는 `Δ(T)+1`, 완전연결 `J×L` protograph에서는 `χ'(T)=J` 여야 등장.
8. 3단(유해성): trapping set의 선형시스템 모델 `x_l = ḡ_l A x_{l-1} + Bλ + B_ex λ_l^(ex)` 에서 A의 최대고유값 `r`(=gain, 점근적 `dv-1-b/a`)과 좌고유벡터 `w1` 사용.
9. 측정치: trappability `t(T) = r·Σ|w1i| / Σ_{i∈Iex}|w1i|` - intra/extrinsic 연결 모두 반영해 동일 `(a,b)` trapping set 간 유해도 차등 (예: 네 개의 `(6,2)` set 중 T_D가 `t=6.39`로 최유해).
10. 한계: 순수 이론(HW·디코더·부호구성 알고리즘 없음), AWGN·regular·min-sum gain 가정, 실제 부호 BER 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping set 열거/판정 (부호구조 분석) | 대응 없음 | Prime ECC는 고정 QC-LDPC 로드(ecc_top.cpp Load_PCM())만 있고 trapping set 분석/스크리닝 도구 없음 |
| error floor 유해성 측정치 (trappability) | 대응 없음 | 디코더/인코더 실행모듈과 직접 연결되는 함수 없음 - 부호설계 오프라인 지표 |
| 선형시스템 모델(min-sum gain) | decoder.cpp LDPC_Decoder() (개념적 참조만) | 실제 min-sum은 있으나 이 논문은 알고리즘이 아닌 분석식이라 코드 대응 아님 |
```

적용 가치: **낮음** - Prime ECC 부호가 이미 고정된 QC-LDPC이고, 이 논문은 이식할 디코더/HW/부호구성 알고리즘이 아니라 오프라인 조합론 분석 도구라 즉시 활용 경로가 없음.
