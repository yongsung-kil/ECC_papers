### B. 알고리즘 요약

1. 대상 부호는 IEEE 802.11n QC-LDPC, code rate 1/2, block length 648~1944, sub-matrix(circulant) 크기 27/54/81의 가변 block-size.
2. 문제: 여러 표준의 code length/rate 다양성을 커버하면서 area·power·throughput을 동시에 만족하는 디코더가 필요.
3. 복호 알고리즘은 layered decoding인 `TDMP`(turbo decoding message passing) + `SMSA`(Scaling Min-Sum Algorithm).
4. TDMP는 parity-check matrix를 non-overlapping 수평 layer로 나눠 각 layer를 sub-code로 순차 복호, 앞 layer의 extrinsic을 다음 layer 입력에 결합(layered schedule).
5. 각 column이 group 내 weight≤1이므로 VN 업데이트 시 직전 iteration의 `L(c→v)`를 subtrahend, 신규 입력을 addend로 하는 a posteriori 갱신 형태로 정리(Algorithm 2).
6. 아키텍처는 DRU(data route unit)/VNPU/CNPU/LLR CU/FIFO array 5개 컴포넌트를 4-stage pipeline으로 구성, retiming으로 pipeline DFF 축소.
7. DRU는 27입력 DWU + 3 FIFO + SFU로 QC circulant 순환을 off-line 재계산해 단일 라우팅으로 27/54/81 sub-matrix를 모두 처리(메모리 유닛 없음).
8. CNPU는 각 layer에서 최대 LLR·2nd LLR·최대값 주소만 저장(min-sum 구조), CMP4x2(=CMP2x2 2개)로 fully-parallel 비교하여 critical path 단축.
9. 90nm low-k multi-Vth CMOS 합성 결과: 450MHz, 349.48K gate, 168.76mW, 1.215Gbps @ max iteration 10, multi-Vth로 critical path는 고속셀·비critical은 저누설셀 사용.
10. 한계: rate 1/2 802.11n 전용, BER/정정능력 시뮬 결과 없음, soft 양자화 비트/scaling factor 미기재.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SMSA (Scaling Min-Sum) CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 기반 동일 계열이나 Prime ECC는 이미 min-sum 보유(중복) |
| TDMP layered schedule | decoder.cpp LDPC_Decoder() 이터레이션 루프 | layered 스케줄은 Prime ECC의 Dual-Update와 유사 개념, 신규성 낮음 |
| CNPU (min1/min2+주소 저장) | decoder.cpp CNU_Update_New_Mag() | min1/min2 저장 방식 이미 보유 |
| DRU 단일 라우팅 / memory-less | GT.cpp Make_GT_HW() (edge/MUX) | 라우팅 최적화는 GT와 결이 유사하나 802.11n circulant 종속 |
| 가변 block-size(z=27/54/81) | 대응 없음 (Prime ECC는 z_sb=32 고정) | Prime ECC는 고정 lifting이라 직접 대응 없음 |
```

적용 가치: **낮음** — TDMP/SMSA/min1·min2 저장은 Prime ECC가 이미 보유한 기법이고, 유일한 기여인 memory-less 단일 라우팅이 802.11n QC 구조에 강하게 종속되어 z=32 고정 구조에 이식 이점이 적음.
