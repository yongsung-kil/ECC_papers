### B. 알고리즘 요약

1. 대상은 AWGN + BPSK 상의 (2304,1152) WiMAX QC-LDPC, αMS(normalized min-sum) layered 복호 후 실패 코드워드를 처리.
2. 문제: LDPC의 error-floor(작은 trapping set 기인)를 기존 후처리기(리스트 소거[7], backtracking[8])는 표준 BP보다 높은 복잡도로 해결 → HW 부적합.
3. 핵심 관찰: 실패 코드워드에서 오류 심볼의 신뢰도 `|Li|`(누적 LLR)가 정상 심볼보다 작다(Fig.1) → 저신뢰 심볼만 소거 대상.
4. 코드워드 복구를 BEC 상 소거 복호로 환원: `uI(e) HI(e)^T = sIc(e)` (식1), L개 소거에 대한 L차 선형계.
5. H가 sparse이므로 `HI(e)`에 비영 원소 1개뿐인 행이 다수 → 해당 소거값을 직접 결정(단일 미지수). 이것이 복잡도를 크게 낮추는 핵심.
6. 소거 선택 2방식: Method1(신뢰도 정렬 후 하위 `ρN`개), Method2(`|Li|≤σ`, `Lmax`로 상한) — Method2는 정렬 제거로 HW 친화.
7. 미해결 소거는 Solution1(uˆ 값 그대로 대입, 저복잡도, HW 채택) 또는 Solution2(플립 리스트 + 최소 유클리드 거리)로 처리.
8. HW(ESPP-B): AFU가 병렬로 소거 인덱스 수집, SCU가 syndrome를 `s ⊕ sI(e)`로 분할 계산, shifter로 행렬곱 대체, sparsity로 LUT 크기를 code-length(2304)→row-weight(7)로 축소.
9. 결과: αMS 대비 error-floor 영역 FER 대폭 개선, suboptimal scaling(α=0.75)에서 개선폭 더 큼, TSMC 90nm 합성, 후처리기 면적은 완전 LDPC 디코더[9]보다 훨씬 작음, 최장 latency 706 cycle(x=64, Lmax=50).
10. 한계: 단일 코드 예시 위주, 실패 코드워드에 한해서만 동작하나 CUU/RU 반복으로 latency 추가 발생, gatecount/throughput 정량치 본문 서술 부재.
