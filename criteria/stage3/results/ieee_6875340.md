### B. 알고리즘 요약

1. 대상: mp=4, rate 1/2~9/10, punctured variable node 없는 protograph 기반 LDPC 부호군, 정보 블록길이 약 16384 bit, AWGN + sum-product 복호.
2. 문제: AR4JA/NND 등 용량접근 protograph는 punctured(precode) VN에 의존하는데, 이는 복호 수렴을 늦춰 저iteration(예 20회) 환경에서 (3,6)-regular보다 WER가 나쁨 → storage 등 지연민감 앱에 부적합.
3. 모델: base matrix `B∈(Z+)^{mp×np}`를 2단 lifting(q1 이진 → q2 순환 permutation)으로 QC parity-check 생성, 설계rate `R=(np-mp)/(np-up)`.
4. 핵심 기법1: differential evolution(DE)으로 mother base matrix 탐색, cost는 PEXIT(500 iter) threshold `δ(X)`, mutation `Ms=Φ(Br1+0.5(Br2-Br3))` + crossover(`pc=0.88`) + selection(조건 지표 `I(X)`).
5. 핵심 식 의미: threshold를 cost로 써서 특정 iteration이 아닌 넓은 iteration 범위에서 잘 동작하는 부호를 목표.
6. 핵심 기법2: partial brute-force search로 컬럼 부분집합 `S=argmax δ([U|A])I(...)` 전수탐색 (ep≤2, mp·nu≤16에서 가능)해 DE 결과 국소 미세조정.
7. 제약: VN degree≥3(선형 최소거리 성장), max VN degree≤8, weight-2 컬럼 1개 허용(threshold 개선), `ep=3`으로 병렬 edge 제한(short cycle 억제).
8. 고rate 확장: lengthening으로 저rate에서 `B_{(n+1)/(n+2)}=[En|B_{n/(n+2)... }]` 순차 확장(En은 partial brute-force, ep≤2). finite-length은 PEG로 2단 lifting.
9. 결과: rate 1/2 threshold 0.501dB(NND 0.395보다 높으나 punctured 없음), 전 rate Shannon gap 0.188~0.314dB. 20 iteration 제한 시 NND 대비 0.1~0.4dB 우수, R4JA와 대등.
10. 한계: HW 미설계·시뮬만; 200 iteration에선 NND보다 0.1dB 열세; rate 7/8은 소형 trapping set로 WER 10^-5 error floor 관측.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph base matrix + 2단 lifting 부호구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 base+circulant shift QC-LDPC 로드 - protograph 부호를 H-matrix로 교체 가능하나 고정 구조 재검증 부담 |
| DE/PEXIT threshold 부호탐색 | 대응 없음 | Prime ECC에 부호 최적화·threshold 탐색 루틴 없음(오프라인 설계 단계) |
| puncture 없는 저iteration 수렴 개선 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 부호가 avg iteration/latency를 줄여 복호 루프 조기수렴에 기여(간접) |
| QC(circulant) lifting | `ecc_data.h` `PCM_b`(base+shift) | Prime ECC의 z=32 circulant 구조와 개념 정합 |

적용 가치: **중간** — HW 미설계·부호 재검증 부담으로 직접 이식은 어렵지만, "punctured VN 제거로 저iteration 수렴 개선(storage 타깃)"이라는 방향성은 Prime ECC의 avg iteration/latency 절감 관심사와 정합해 부호설계 참고가치가 있다.
