### B. 알고리즘 요약

1. 시스템: outer nonbinary LDPC(GF(2^8)) + inner marker code 연접부호, IDS(삽입/삭제/치환) 채널 대상 (BPM 등 고밀도 저장).
2. 문제: 마커 디코더의 심볼레벨 동기화(SLS)는 심볼크기 `bS` 커질수록 BER↓이나 시간복잡도가 `bS`에 지수적으로 증가해 `bS≥4` 비실용.
3. 채널모델: Davey-MacKay IDS 채널 — 입력비트마다 삽입확률 `Pi`, 삭제 `Pd`, 전송 `Pt=1-Pi-Pd`, 치환 `Ps`, 최대삽입길이 `lI`로 유한화.
4. 핵심기법: forward-backward(FB) 알고리즘의 심볼확률 `g(x;y)=Pr(y|x)`를 채널·부호 파라미터로 사전계산하여 룩업테이블 `G(x,y)`에 저장, 재귀계산을 테이블조회로 대체.
5. 식 의미: `Pr(r|vij=x)=Σ α(t,s)G(x,[r])β(t+bS,s')` — FB 확률에 테이블값 곱해 심볼 우도 산출, 무거운 재귀 `g` 계산 제거.
6. 확장: reduced 테이블 `GR` — 수신길이 `|y|`를 `bS-ωd ≤ |y| ≤ bS+ωi`로 제한해 엔트리 수 축소.
7. 구현 디테일: 단일경로(SP)와 다중경로(MP, inner-outer 반복복호) 모두 적용, MP는 외부 SPA의 extrinsic를 심볼단위로 FB에 피드백.
8. 최적화: 외부 LDPC는 ultra-sparse NB-LDPC, FFT-SPA 복호.
9. 결과: `bS`=1,2,4,8 평가, 심볼크기 클수록 BER↓; `bS`=8 naive 대비 테이블조회 4만배 이상 빠름(Case1); `GR` 엔트리 `bS`=8/ω=2에서 약 50만개.
10. 한계: HW 미설계·SW 시뮬만, IDS/BPM 전용 채널 가정, 메모리(50만 엔트리)는 FFT-SPA NB-LDPC 수준으로 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SLS FB 알고리즘 / 마커 동기화 | 대응 없음 | IDS 동기화 디코더, Prime ECC에 대응 모듈 없음 |
| nonbinary LDPC(FFT-SPA) 외부코드 | 대응 없음 | Prime ECC는 binary min-sum 전용, NB-LDPC 미지원 |
| 확률 룩업테이블 `G(x,y)` | 대응 없음 | IDS 채널 심볼확률용, LLR 테이블([ecc_data.h](../Prime_ECC_3.1_Claude))과 성격 다름 |

적용 가치: **낮음** — nonbinary LDPC 외부코드 + IDS 마커 동기화라는 완전히 다른 문제영역으로, NAND binary QC-LDPC 디코더/부호설계에 이식할 요소가 없음.
