### B. 알고리즘 요약

1. MLC NAND Flash, cell당 2-bit(LSB/MSB), 각 bit는 독립 페이지로 인코딩되는 시스템을 모델링.
2. P/E cycle 증가에 따라 write voltage 분포가 왜곡되어 페이지 실패가 불가피해지는데, 기존 연구(Wong et al. [5])는 read threshold만 최적화하고 write voltage와 read threshold의 공동 최적화는 부족.
3. 채널 모델: write voltage `X`가 signal-dependent Gaussian noise `Z(X)`로 왜곡되어 `Y`를 생성, `Z(X)`는 두 개의 분산(un-programmed cell은 큰 분산, 그 외 cell은 작은 분산)을 갖는 two-variance Gaussian으로 근사.
4. 핵심 기법: 중간 두 write voltage 레벨 `μ01, μ00`과 read threshold를 조인트로 조정해 페이지별 mutual information의 최솟값을 최대화. 세 가지 목적함수 `I_Initial`, `I_Enhanced`, `I_Soft` 각각에 대해 `min(I_LSB, I_MSB)`를 최대화.
5. `I_Initial = I(X_LSB; Y_LSB)`처럼 채널 입력-출력 간 MI로 정의하며, 이 값이 클수록 initial/enhanced read 시점의 LDPC 복호 성공률과 상관관계를 가짐(수식 (1)~(4)).
6. 확장으로 progressive read 구조(initial read 실패 시 enhanced-precision read 추가)의 두 단계 MI를 각각 최적화하는 두 가지 변형 목적함수를 비교.
7. 최적화는 MATLAB `fminsearchcon`(Nelder-Mead Simplex 기반 direct search)으로 다차원 탐색, 매 100 P/E cycle 단위로 lookup table화하여 오프라인 사전 계산.
8. 학습 절차 없음(그리드/직접탐색 기반 수치 최적화), 채널 파라미터는 실측 Flash 특성화 데이터에서 가져옴([14]).
9. 결과: `I_Initial` 기준 최적화 시 initial-read lifetime이 Wong et al. 대비 약 100 P/E cycle(FER 10⁻⁶ 기준 1300→1400) 개선, enhanced-read lifetime도 약 210 cycle 개선(1300→1510).
10. 한계: 순수 시뮬레이션(이론+MATLAB+LDPC FER 시뮬)이며 HW 설계 없음. LDPC 부호(rate 8/9, n=16200)는 기존 고정 코드를 검증 도구로만 사용, 부호구조나 디코더 알고리즘 자체의 개선은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| write voltage / read threshold 최적화 | 대응 없음 | Prime ECC는 채널 write voltage 설계를 다루지 않음(NAND controller 레벨 밖) |
| soft-read LLR threshold (progressive read) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | LLR threshold 설계 관점에서 개념적 유사성은 있으나 본 논문은 write voltage 공동최적화가 핵심으로 직접 대응 어려움 |
| LDPC 부호(rate 8/9, protograph) | `ecc_top.cpp` `Load_PCM()` | 검증용 고정 코드로만 쓰여 부호설계 기법 자체의 이식 요소 없음 |

> 적용 가치: 낮음 — write voltage 레벨 설계는 Prime ECC(디코더/인코더 시뮬레이터) 범위 밖이며, LDPC는 검증 도구로만 사용되어 이식 가능한 ECC 기법이 없음.
