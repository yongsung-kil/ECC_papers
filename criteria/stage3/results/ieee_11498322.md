### B. 알고리즘 요약 (10줄 내외)

1. 시스템은 OLED 기반 VLC(OOK 변조) 링크에서 시스템틱 QC-LDPC (1024,512) rate 1/2 코드(w̄c=3)를 BP(최대 50 iteration)로 복호.
2. 문제는 잔여 비트오류가 텍스트에서 불균형하게 문자 손상을 유발하며, 기존 LLM 텍스트 후처리는 디코더의 soft reliability와 parity 제약을 버리는 한계가 있음.
3. 핵심 가정은 post-slicing 채널을 측정된 crossover 확률 `p`의 BSC로 모델링, 초기 LLR `Lc_j = log((1-p)/p)*(1-2y_j)`.
4. 핵심 기법은 1차 BP 후 syndrome 가중치 `w_s = H*ĉ mod 2`로 잔여오류 바이트 수 `n_e = w_s/w̄c`를 추정, 오류확률 `p_j` 상위 바이트를 마스킹.
5. 마스킹된 문자를 LLM에 다중 샘플링(S회) 질의하여 다수결 `â_j`와 투표비율 `φ_j`를 얻고, `Λ_j = log(φ_j/(1-φ_j))`로 pseudo-LLR 변환.
6. 확장으로 채널 LLR과 LLM LLR의 스케일 불일치를 보정하는 적응형 가중치 `α = clip(β·|L_LDPC|/|L_LLM|, α_min, α_max)`를 도입, 신뢰도 낮으면(`φ_j<θ` 또는 `α<1e-3`) 2차 패스 생략.
7. 구현 디테일: pseudo-LLR은 시스템 비트(데이터) 위치에만 주입하고 parity 위치는 0으로 두어 Tanner-graph 잡음 유입 방지, 2차 BP는 동일 파라미터로 재실행.
8. 별도 학습 절차 없음(LLM은 사전학습 4-bit/8-bit 양자화 Qwen3-4B-Instruct를 블랙박스로 사용).
9. 결과: p=0.005에서 BER 8.5e-5→6.0e-5(30% 개선), p=0.009에서 26% 개선, harm rate(성능 악화) p≤0.005에서 3.4% 미만; 4-bit/8-bit 양자화 성능차 없음.
10. 한계: LLM 추론이 지배적 지연(4-bit 기준 최대 48.3초)로 latency 크게 악화, 텍스트/ASCII 소스 구조에 완전히 의존하며 부호설계나 디코더 알고리즘 자체의 개선이 아닌 응용 계층 후처리.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP(min-sum 계열 아닌 belief propagation) LDPC 디코딩 | decoder.cpp LDPC_Decoder() | 표준 BP 디코더 자체는 이미 보유한 min-sum 계열과 유사하나 부호/디코더 개선이 아닌 외부 LLM 융합이 핵심 |
| syndrome 계산 기반 오류 위치 추정 | decoder.cpp (조기종료용 syndrome check와 유사 개념) | 조기종료(CRC)와 유사한 syndrome 활용이나 목적이 텍스트 마스킹이라 무관 |
| 2차 BP 패스에 pseudo-LLR 주입 | channel.cpp Set_LLR_Th() | LLR 도메인 융합 방식은 유사하나 소스는 LLM 텍스트 추론이라 NAND 채널 LLR과 무관 |
| LLM 기반 텍스트 오류 정정 | 대응 없음 | Prime ECC는 범용 비트스트림 ECC로 텍스트 시맨틱 정보를 활용하지 않음 |
```

적용 가치: **낮음** — 핵심 기여가 텍스트 소스 특화 LLM 사후보정이며 LDPC 부호/디코더 자체 개선이 아니라 Prime ECC(범용 NAND 데이터 ECC)에 이식할 요소 없음.
