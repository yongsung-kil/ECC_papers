> 총평: 5G LDPC(QC-LDPC BG1/BG2 layered min-sum)와 polar를 하나의 add-cmp-add RDU로 통합한 재구성형 HW 논문으로, LDPC layered decoding·파이프라인 기법은 참고가치가 있으나 폴라 겸용·5G 특화라 NAND 이식성은 중간.

### B. 알고리즘 요약

1. 대상은 5G R15 QC-LDPC(BG1 `46×68`, BG2 `42×52`, lifting `Z=2~384`)와 polar(`N=2^n`) 겸용 복호 HW; LLR 6bit 고정소수점, AWGN/BPSK.
2. 풀려는 문제: 5G eMBB는 data plane=LDPC, control plane=polar를 모두 요구하나 각각 전용 디코더는 면적·전력 낭비 → 표준호환 단일 재구성 디코더 필요.
3. 접근: 두 부호 모두 BP 기반임에 착안, LDPC의 CN/VN 갱신과 polar PE 갱신을 **add-comparison-add(ADD-CMPa-ADD)** 공통 연산 패턴으로 재정렬해 기본 유닛을 통일.
4. 핵심 기법: polar PE는 min-sum 근사 `g(a,b)≈sign·min(|a|,|b|)`로 3 ADD+3 CMPa로 분해; LDPC는 layered decoding으로 CN 1개+연결 VN을 `M_v→c=L-M_c→v`, `M_c→v=∏sign·min|·|`, `L=M_v→c+M_c→v` (Eqn.12)로 fine-grained 갱신.
5. 통합 근거: LDPC의 check node 유닛(8 ADD+6 CMPa)이 polar PE 2개로 분해 가능 → 동일 기본 구조로 재구성, ADD 75~100%·CMPa 100% 재사용.
6. RDU 구조: 5-stage 완전 파이프라인(stage1/5=ADD array, stage2~4=CMPa), VLIW 방식 off-line 제어어로 최대 `Nj=11`까지 check node 처리, 최대 20개 decoding pattern 지원.
7. 병렬화: RDU를 SIMD RDU array로 묶어 intra-codeword(같은 lifting block) + inter-codeword(다른 codeword) 병렬; RDU 간 직접통신 없이 mini switching bus로 memory bank 연결.
8. 재구성 스케줄(Algorithm 1): `N_RDU=I×J`로 codeword length·Z·모드에 따라 array 수 동적 결정, BG1 22~45행 등 비겹침 2 layer 동시처리로 iteration당 latency 감축.
9. 결과(FPGA Stratix IV + ASIC ZTE 55nm): RDU 재사용 LDPC 82.67%/polar 99.94%; polar(1024, R=1/2) 1.23Gbps·2.23Gbps/mm2, LDPC(2912, R=5/26) 594Mbps·1.08Gbps/mm2, 재구성 오버헤드 ALUT +11.3%.
10. 한계: 정정성능 개선 없음(HW효율/면적 목표), 핵심가치인 polar 겸용은 NAND(LDPC 단일)엔 불필요, 5G BG1/BG2 lifting 구조와 6bit LLR에 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC layered min-sum CN 갱신(∏sign·min) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 min-sum CN 연산으로 직접 대응, 참고가치 |
| VN/LLR 갱신 (Eqn.12 layered) | `decoder.cpp` `VN_Cal_HD()`, `LDPC_Decoder()` 이터레이션 루프 | layered 스케줄 개념 대응(Prime ECC는 dual-update 보유) |
| QC-LDPC lifting(Z) H-matrix 로드 | `ecc_top.cpp` `Load_PCM()` | QC 구조 로드 대응, 단 5G BG와 부호 상이 |
| RDU 파이프라인(5-stage ADD-CMPa) HW arch | 디코더 HW 모델(z=32, pipeline 6-stage), `GT.cpp` `Make_GT_HW()` | 파이프라인/병렬 유닛 개념 접점, 구조 정합 필요 |
| polar PE / 재구성 로직 | 대응 없음 | Prime ECC는 LDPC 전용, polar 겸용 불필요 |

적용 가치: **중간** — LDPC layered min-sum과 파이프라인 RDU 병렬화 기법은 Prime ECC 디코더 HW와 계열이 같아 arch 참고가치가 있으나, 핵심 신규성인 polar 겸용 재구성은 NAND에 불필요하고 정정성능 향상도 없어 이식 이득이 제한적.
