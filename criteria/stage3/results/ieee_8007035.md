### B. 알고리즘 요약

1. 시스템: (N,K) RS 부호 over GF(2^m)의 binary image(고밀도 패리티행렬 `Hb`, size `(n-k)×n`), BPSK/AWGN, 예: (255,239), (63,55), (127,121/107).
2. 문제: RS의 고밀도 패리티행렬에 BP 직접 적용 시 몇 iteration 만에 메시지 상관 급증해 수렴 저하, 기존 ABP는 flooding이라 LRVN이 iteration당 1회만 갱신됨.
3. 모델: 신뢰도 낮은 bit(LRVN)를 systematic part(degree-1)로 매핑해 영향 최소화하되, 그만큼 이들이 갱신될 기회도 줄어드는 trade-off.
4. 핵심기법1: ADP(adaptive parity-check) - 매 iteration LLR 크기로 VN 정렬 후 elementary row operation으로 `Hb`의 앞 `(n-k)`열을 항등행렬화(LRVN=degree-1).
5. 핵심식: C2V residual `rC2V(i,j)=|m_pre - m_ci→vj|` (식 참조) - 갱신 전후 C2V 차이로 메시지 중요도(importance) 측정.
6. 핵심기법2: double-polling RBP(DP-RBP) - VN 폴링 안에 check node 폴링 삽입, 동일 CN 내에서만 residual 비교(공정성), LRVN을 iteration당 여러 번 갱신.
7. 트릭: damping coefficient 제거, iteration마다 누적 LLR을 채널값으로 reset해 상관을 단일 iteration 내로 국한; V2C는 식(4)로 해당 CN 메시지 제외.
8. 최적화절차: 하드결정 후 만족 못하면 기존 non-binary 대수적 HDD로 최종 복호(하이브리드).
9. 결과: (255,239)에서 ML bound 대비 0.8dB, 기존 최고 soft-decision RS 대비 0.1dB 이득; (31,25)/(63,55)는 near-ML(단 최대 4000 iteration 필요).
10. 한계: HW 미설계, 부동소수 시뮬만, RS non-binary·고밀도 대상, 수천 iteration + 매회 Gaussian 소거로 복잡도/latency 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA/BP 메시지 갱신(V2C·C2V) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 우리는 min-sum, 논문은 full tanh SPA + 고밀도 그래프라 직접 대응 약함 |
| ADP 적응형 패리티행렬(row op) | 대응 없음 | 우리 부호는 고정 QC-LDPC PCM(`Load_PCM()`), 매 iteration 행렬 재구성 없음 |
| residual 기반 IDS 스케줄링(DP-RBP) | 대응 없음 | 우리는 flooding/dual-update 고정 스케줄, dynamic scheduling 미지원 |
| non-binary 대수적 HDD 후단 | 대응 없음 | RS 전용, binary LDPC 엔진과 무관 |

적용 가치: **낮음** — RS non-binary 부호의 고밀도 binary image에 대한 적응형 행렬+동적 스케줄링 디코더로, 우리 sparse binary QC-LDPC min-sum 엔진과 정합되지 않는다. residual IDS 개념만 원거리 참고 가능.
