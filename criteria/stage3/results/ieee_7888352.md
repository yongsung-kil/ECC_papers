### B. 알고리즘 요약

1. 시스템: WSN(Wireless Sensor Network) AWGN 채널, BPSK 변조, outer=QC-LDPC(324×648, code rate 1/2~5/6), inner=convolutional code(constraint length 3/4/5/7, code rate 1/2)의 serial concatenation 구조.
2. 문제: 고 coding-gain과 디코더 복잡도 사이 trade-off를 완화하면서 Turbo code 특유의 error floor와 burst error 문제를 동시에 해결.
3. 핵심 가정/모델: AWGN 채널, 저전력·저활동 WSN 노드를 겨냥해 저 code rate로 송신전력을 낮추고 프레임 길이 324bit 선택.
4. 핵심 기법 1: Truncated-Iteration Layered-Decoding QC-LDPC(TILD-LDPC) — 기존 layered decoding QC-LDPC([9] 인용)에 반복횟수를 5회로 truncation하여 복잡도 절감.
5. 핵심 관찰: iteration>5에서는 BER 개선이 미미한 반면 복잡도만 증가하므로 5-iteration이 최적점(문헌 대비 8회, 10회 이상 대비 절감).
6. 핵심 기법 2: TILD-LDPC(outer, "stronger code") + low-constraint(CL3) convolutional/Viterbi soft-decision decoder(inner, "weaker code") 조합이 최고 BER 성능(시나리오 01)을 보임.
7. 부수 트릭: TILD-LDPC와 convolutional code 사이에 random interleaver를 삽입해 Viterbi decoder에서 발생하는 burst error를 완전히 제거(이미지 전송 시뮬레이션으로 시각 검증).
8. 최적화 절차: CL(constraint length) 3/4/5/7 비교 시뮬레이션으로 CL3가 CL7 대비 약 1300 equivalent addition 절감하면서도 code rate 1/2, 2/3에서 동등 이상 BER 성능 확인.
9. 결과: 제안 방식이 code rate 1/4에서 BER=1e-3 기준 8.6dB, 1e-4 기준 9.9dB coding gain 달성(문헌 [1][5][6][2] 대비 우수), error floor 없음.
10. 한계: MATLAB 시뮬레이션에 국한, HW 구현/gatecount/throughput 전무, QC-LDPC 자체 알고리즘(layered decoding, offset min-sum)은 기존 문헌 인용이며 논문 고유 기여는 iteration truncation·concatenation 조합·interleaver 설계에 한정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TILD-LDPC iteration truncation(5회) | decoder.cpp LDPC_Decoder() | 조기종료 개념과 유사하나 고정 반복횟수 truncation이며 Prime ECC의 CRC 기반 조기종료(partial_CRC.cpp)와는 방식이 다름 |
| QC-LDPC layered decoding, offset min-sum | decoder.cpp CNU_Update_New_Mag() | Prime ECC도 min-sum 계열 사용하나 layered scheduling 세부는 대응 없음, 기존 보유 기법과 중복 |
| Convolutional/Viterbi 직렬 연접, random interleaver | 대응 없음 | Prime ECC는 단일 QC-LDPC 구조이며 concatenation·interleaver 아키텍처 없음 |
```

> 적용 가치: **낮음** — QC-LDPC 요소는 이미 Prime ECC가 보유한 기법과 중복되고, 논문 고유 기여(convolutional 연접, interleaver, WSN 전력효율)는 NAND 단일 부호 구조와 무관.
