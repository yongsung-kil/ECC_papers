### B. 알고리즘 요약

1. BEC 및 BI-AWGN 채널에서 R=0.5, N=10,000의 (G)LDPC 랜덤 앙상블 코드 설계.
2. 기존 EXIT chart 기반 설계는 무한 반복(threshold) 최적화만 다뤄 실제 유한 반복(imax) 환경에서 성능이 부족한 문제를 해결.
3. 채널 모델: BEC(소거확률 ε) 또는 BI-AWGN(Eb/N0), belief propagation 반복 복호 가정.
4. 핵심 기법: EXIT 함수의 "교차 안 함" 조건 대신, `imax` 반복 후 출력 extrinsic information이 임계값 `ξ`를 넘도록 요구하는 "iteration-constrained threshold" `ε*` 정의.
5. 이 수정된 EXIT 조건으로 differential evolution(변이/교차/선택)을 수행해 목표 rate R을 만족하는 degree distribution pair `(λ, ρ)`를 탐색.
6. 확장: SPC 대신 (7,4)/(15,11) Hamming code 등 일반 선형블록코드를 CN에 배치하는 GLDPC 앙상블도 동일 절차로 최적화 (`ρ(x)`에 CN type별 항 포함).
7. 부수 관찰: `imax`가 작을수록 degree-2 VN 비율(`λ2`)이 급격히 감소하고, 성장률(growth rate) 곡선의 초기 기울기 `λ'(0)ρ'(1) < 1` 조건을 만족해 최소거리 특성이 개선됨.
8. 학습 절차: differential evolution (Fig.1 알고리즘) - population 크기 Np=70, 목표 rate 제약 하 반복적 벡터 갱신.
9. 결과: imax=10 최적화 코드(Code A/B/E)는 imax=10에서 무한반복 최적화 코드(Code C/D)보다 우수한 waterfall, (1024,512) 코드에서 최소거리 58 vs 44로 개선.
10. 한계: HW 미설계, 순수 시뮬레이션(랜덤 앙상블에서 무작위 edge 연결), QC-LDPC 구조·NAND 채널 모델 전혀 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EXIT chart + differential evolution 기반 앙상블(H-matrix) 설계 | `ecc_top.cpp` `Load_PCM()` | 랜덤 irregular LDPC 앙상블 설계로 특정 QC-LDPC 고정 구조 재검증 부담 커 이식 난이도 높음 |
| GLDPC (Hamming code at CN) | 대응 없음 | Prime ECC는 SPC 기반 QC-LDPC이며 일반 선형블록코드 CN 구조 미지원 |
| iteration-constrained EXIT 임계값 설계 | 대응 없음 | 부호설계 단계 이론 도구로 디코더 런타임 모듈과 무관 |

> 적용 가치: 낮음 - 랜덤 irregular LDPC/GLDPC 앙상블의 이론적 설계 기법으로 QC-LDPC 고정 구조인 Prime ECC 3.1에 직접 이식하기 어려움.
