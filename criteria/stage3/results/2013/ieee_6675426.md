### B. 알고리즘 요약

1. 대상은 LDPC Block code가 아닌 LDPC Convolutional Code(`LDPC-CC`)로, 연속/streaming 전송에 적합한 semi-infinite 부호를 다룬다.
2. 기존 LDPC-CC 구성(랜덤 [5], 대수적 [6])은 하드웨어 복잡도가 크다는 문제를 단순·규칙적 구조로 개선하려 한다.
3. 채널은 AWGN, 입력은 LLR 벡터 `LLRk = Lc*yk + L(xk)`, `Lc=4a·Es/N0`.
4. 핵심 기법: `(ms,J,K)` 저밀도 행렬을 4개 단위행렬 `I1~I4`를 이어붙인 `(4,2,2)` 시작에서 대각 아래를 잘라 우하단에 붙이고(welding) 0행 삽입·1 추가로 primitive matrix `H`를 만든다.
5. 이 syndrome-former 형태 `H^T`는 rate `R=b/c`, memory `M`, period `T`로 `H`를 반복·시프트해 large matrix (15)를 생성한다(시프트량 `(c-b)` 배).
6. 인코딩은 syndrome former 관계식으로 과거 code symbol에서 현재 parity를 얻으며 b=1, c=2 예제로 systematic 인코더 구현.
7. 디코딩은 Tanner graph 상 message passing: check node box-plus(`min-sum`/`tanh`), variable node 합산 `mv→f = m1+m2+m3+x`.
8. sub-graph(길이 `M+1`) 단위로 vertical/horizontal 연산을 sliding window로 수행, 여러 프로세서 pipeline으로 가속 가능.
9. 결과: rate 1/2 `(256,(2)3,5)` 부호, `I=50` iter, AWGN에서 동일 복잡도 LDPC-BC 대비 약 1.4dB(BER 1e-5), 기존 LDPC-CC [9] 대비 약 0.2dB 개선. SPA가 min-sum보다 우수.
10. 한계: HW 미설계(gatecount/throughput 없음), 시뮬만, NAND 블록 부호가 아닌 convolutional 대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-CC 구성법(welding·시프트로 H 생성) | `ecc_top.cpp` `Load_PCM()` (부분) | convolutional H 구조라 블록 QC-LDPC PCM 로드와 정합 안 됨 |
| message passing 디코딩(box-plus, min-sum) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | sliding-window 스케줄이라 블록 이터레이션 루프와 구조 상이 |
| variable node 합산/LLR | `decoder.cpp` `VN_Cal_HD()` 등 | 교과서 수준 SPA/min-sum, 이미 보유 |
| SPA(tanh) box-plus | 대응 없음 | Prime ECC는 min-sum 근사만 사용 |

적용 가치: **낮음** — convolutional 부호·sliding-window 디코딩으로 NAND 블록 QC-LDPC와 구조가 다르고, 기여가 표준 min-sum/SPA 및 특정 구성법에 국한된다.
