### B. 알고리즘 요약

1. 시스템: IEEE 802.15.3c WPAN, code length `672`, 4개 code rate(1/2, 5/8, 3/4, 7/8) 지원 QC-LDPC 디코더 ASIC.
2. 문제: 고병렬 LDPC 복호에서 처리노드 간 permutation network(PN)가 라우팅 혼잡·로직 오버헤드·저전력효율의 주범.
3. 핵심기법 1: 표준 PCM 대신 **modified-PCM**을 설계 - virtual sub-block을 삽입해 row-split, macro-block(MB) cyclic shift 규칙성을 강화.
4. 규칙성 덕분에 pre-PN/post-PN 각 3-stage가 모두 **고정 패턴 배선(wire)만**으로 구현되어 게이트·MUX가 전혀 없음.
5. 데이터플로우: 672 APP 메시지(32 column block)를 21개 macro-layer processing engine(MLPE)이 병렬 처리, 레지스터로 메모리 대역폭 확보.
6. 각 macro-layer 갱신 후 post-PN이 MB를 1 sub-block씩 cyclic shift하여 다음 macro-layer도 동일 pre-PN 재사용 → 2개 PN을 16 macro-layer가 공유.
7. MLPE 내 32 temp/APP updater를 4 group으로 나눠 code-rate별 구성(binary-tree min sorter도 동일), virtual sub-block은 message bypass + MSG active LUT로 처리.
8. 복호 알고리즘: **Normalized Min-Sum** (scaling factor 0.75), frame-level pipelining으로 임계경로 개선(레지스터 2배).
9. 결과: 65nm LVT CMOS 실측, core 430.8K gate(96.8K는 pipeline 레지스터), 6-bit 고정소수, 1.2V/400MHz/10-iter에서 6.72Gb/s, 8.0pJ/bit/iter, chip density 73.3%→86.3%.
10. 한계: 특정 표준 PCM에 맞춰진 순열 규칙성 - 다른 QC-LDPC 구조로 이식 시 modified-PCM 재설계 필요, error-floor/정정능력 개선은 목표 아님.
