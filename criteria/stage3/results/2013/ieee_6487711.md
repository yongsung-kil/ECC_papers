### B. 알고리즘 요약

1. 대상: NAND+ReRAM 하이브리드 통합 스토리지(USSS), ABER(ECC 후 BER<1e-15 기준 raw BER)를 32× 향상; 40bit/1KB ECC 가정.
2. 문제: server/array/SSD/NAND 계층마다 컨트롤러가 독립 최적화되어 big-data 복제·중복·오버헤드 발생, 신뢰도 비효율.
3. 모델: program disturb는 lower/upper page별 error 방향(1→0, 0→1)이 다르고, retention BER은 page 번호(source-line 거리)에 의존한다는 측정 관찰을 활용.
4. 기법1(RM, reverse-mirroring): primary NAND의 고BER cell 데이터를 mirrored NAND의 저BER cell에 배치(lower/upper·page번호 교차), read 시 오류 적은 쪽 선택 → PD/DR BER 각 69%/41% 감소. ReRAM이 재정렬 write 버퍼.
5. 기법2(ERS, error-reduction synthesis): RM로 primary/mirror의 dominant error 방향이 반대가 되게 mirror를 bit-flip 저장, read 시 mismatch를 error 방향 경향으로 정정 → BER 91%↓(12× ABER).
6. 기법3(page-RAID): NAND string(BL 방향) XOR block-parity를 ReRAM에 저장(chip당 1 parity page), ECC fail 시 나머지 page XOR로 복구; ReRAM 고내구성으로 잦은 갱신 가능 → ABER +45%.
7. 기법4(EM, error-masking): 발생한 error 위치를 error-location table에 기록·run-length 압축(BER 1%면 16%로)해 ReRAM 저장, read 시 masking → BER 67%↓(3× ABER).
8. 구현 디테일: ReRAM parity buffer 요구 내구성 255×4000≈1e6 cycle, 필요 용량은 NAND의 0.1% 미만.
9. 결과: 종합 시 ABER 32× 향상 = W/E endurance 4.2× 또는 retention 34× 연장; 2X/1Xnm NAND 실측.
10. 한계: LDPC/부호·디코더 기법이 아니라 시스템 리던던시·컨트롤러 아키텍처; ReRAM 하이브리드 하드웨어 전제; LDPC는 병용 가능하다고만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| reverse-mirroring / error-reduction synthesis | 대응 없음 | 스토리지 컨트롤러·RAID·mirroring 층, ECC 시뮬 범위 밖 |
| page-RAID (BL XOR parity) | 대응 없음 | outer RAID 리던던시, LDPC 부호와 직교 |
| error-masking (error-location table) | 대응 없음 | 컨트롤러 후처리, 디코더 무관 |
