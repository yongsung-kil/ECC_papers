### B. 알고리즘 요약

1. 시스템: IEEE 802.16e(WiMAX) QC-LDPC, rate 1/2, length 2304 (code length 576:96:2304 가변 지원), 130nm ASIC 합성.
2. 문제: LDPC 디코더 throughput = `Freq*BlockLen/(CyclesPerIter*NumIter)` — iteration당 클록 수를 줄여 throughput을 높이려 함.
3. 모델: min-sum 메시지 전달. `α_ij`=CN→VN, `β_ij`=VN→CN, `λ_j`=채널 LLR, 보정계수 `S_MS`.
4. 핵심기법 (overlapped layered): H를 L개 layer로 분할, 1st layer 처리 중 다음 layer의 row 처리와 이전 layer의 column 처리를 동시 수행 → iteration당 클록 수 감소.
5. CN 갱신: `α_ij = S_MS · Π sign(β) · min_{j'≠j}|β_ij'|`, Min1/Min2 + sign + Min1 position만 저장해 메모리 절감.
6. VN 갱신: `β_ij = LQ_j - α_ij`, posterior `LQ_j = λ_j + Σ α_ij`, layer별 부분합 `L_i^l = L_i^{l-1} + α_ij` 누적.
7. 곱셈기 축소 트릭: Min1/Min2 대신 `S_MS·Min1`, `S_MS·Min2`를 저장 → CN/VN에서 곱셈기 2→1개, 성능도 개선.
8. 양자화: 채널 LLR 7bit(부호1bit 포함), Min1/Min2 7bit, overflow 방지 위해 LQ 8bit.
9. HW: z개 CNU + z개 VNU 동시 동작, RAM 3개(채널 LLR / posterior LLR / CN→VN 합). layer당 Wr(=6 or 7) 클록.
10. 결과: 100MHz, 10 iteration에서 throughput 183Mb/s, 면적 6.9mm2, 전력 242mW — 추가 RAM으로 면적 증가하나 [11][12] 대비 throughput 우위. 한계: 시뮬 BER 미보고, 저rate WiMAX 특화.
