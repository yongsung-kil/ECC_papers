### B. 알고리즘 요약

1. 대상: QC-LDPC block code(base matrix `4x24`, expansion factor)에서 unwrapping으로 유도한 rate-5/6 QC-LDPC convolutional code(LDPCCC). 광통신 저error-floor 타깃.
2. 문제: 고rate LDPCCC 디코더에서 CN↔VN 연결이 많아 switch network의 라우팅/HW 복잡도가 병목.
3. 핵심 가정: AWGN 채널, LLR 메시지, girth 8인 sub-matrix 선택(block code 대비 girth 보존).
4. 핵심 기법 1: generic PU 대신 dedicated `Block Processing Unit(BPU)` 사용 → switch network 완전 제거. 노드-레벨+iteration-레벨 부분병렬.
5. 핵심 식: CN 업데이트 식 `(4)`/(7), quantized SPA. 입력 pairing으로 extrinsic 계산.
6. 핵심 기법 2: 4-bit quantized Sum-Product(`QSPA`) — full LUT(`2^(4·d_c)`) 대신 LUT tree(각 `2^8`, 총 `3(d_c-2)`개)로 CN 복잡도를 `O(d_c)` 선형화.
7. 구현 디테일: sign-magnitude ↔ 2's complement 변환, dynamic message storage(메시지 종류가 시간에 따라 동적 변화), 단순 counter 주소 제어(QC 구조 덕).
8. 최적화: DE(density evolution)/differential evolution 기반 양자화 스텝 → float SPA 대비 0.1dB 손실.
9. 결과: Altera Stratix IV FPGA, 100MHz에서 2.0Gb/s, 3.55dB에서 BER<10^-? (본문 손상). block code 대비 0.1~0.2dB gain, [9] 대비 0.8~1dB coding gain + error floor 없음.
10. 한계: convolutional code 전용(block QC-LDPC와 구조 상이), gate count 수치 본문 서술 없음(표만), min-sum 아닌 QSPA로 Prime ECC(min-sum)와 알고리즘도 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 디코딩 알고리즘(QSPA CN update) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum, 논문은 4bit LUT-tree QSPA로 CN 연산 방식 상이(직접 이식 부적합) |
| 4-bit 양자화 / LLR | `channel.cpp` + `ecc_data.h` `PARAM_LLR` | 4bit soft는 Prime ECC(hard/2SD/3SD, soft-4bit+ 미지원)와 불일치 |
| BPU 파이프라인/메모리 결합 | 대응 없음 | LDPCCC 고유 처리단위·메모리 결합 - block 디코더(z=32 pipeline) 구조와 미대응 |
| switch network 제거/라우팅 | `GT.cpp` `Make_GT_HW()` | 둘 다 라우팅 축소 목적이나 GT(MUX edge선택) ≠ BPU(convolutional) 접근 상이 |

적용 가치: 낮음 — LDPC convolutional code 전용 아키텍처로 Prime ECC의 block binary QC-LDPC(min-sum, z=32 pipeline)와 코드구조·알고리즘·soft 폭 모두 불일치. dynamic memory/BPU 개념은 참고만.
