### B. 알고리즘 요약

1. 시스템: 플래시 메모리용 concatenated 부호 — 외부 β개 동일 BCH(systematic) + 내부 QC-LDPC, BPSK/AWGN, rate 0.8355~0.971, QC-LDPC length 4590~12700.
2. 문제: MLC 플래시에서 BCH만으로 부족해 QC-LDPC 도입 시 trapping-set에 의한 error floor(BER~10^-5) 발생, concatenation은 rate loss로 waterfall 성능 저하.
3. 모델: SPA(sum-product) 입력 `R_i=2y_i/σ²`, (a,b) trapping set = a개 변수노드·b개 홀수차수 이웃 검사노드, dominant trapping set은 주어진 a에 최소 b를 갖고 error-floor 지배 원인.
4. 핵심 기법 1(BCH 선택): 컴퓨터 탐색으로 dominant trapping set의 최대 a인 `a*`를 구하고, 정정능력 `t≥a*`인 BCH를 외부부호로 선택 → 한 BCH 블록에 몰린 잔여오류도 정정.
5. 의미: `t≥a*`면 내부 QC-LDPC가 못 고친 trapping-set 오류를 외부 BCH가 흡수해 error floor 제거(예: a*=4에 (255,223,4) BCH 성공, (255,239,1) 실패).
6. 핵심 기법 2(outer iteration): rate loss 보상 위해 내·외부 반복 — 성공 디코딩된 BCH 결과의 신뢰비트 인덱스 집합 `S`를 SPA에 고정 피드백.
7. 구현 디테일: Refined SPA에서 `i∈S`면 `R_i=2(1-2d̂_i)/σ²`로 고정, `L_i,j`·`D_i`도 고정해 S 밖 비트 정정 유도, 외부 BCH는 소프트 재설계 없이 하드결정 디코더 그대로 유지.
8. 절차: raw readout → 외부 BCH가 systematic 부분 먼저 디코딩, 전부 성공이면 종료; 실패 시 내부 QC-LDPC(SPA) 기동 후 Refined SPA로 outer iteration 반복(성공 BCH 수 불변 or 최대 outer iter 도달까지).
9. 결과: (4590,3835) a*=4에 (255,223,4) BCH로 error floor 소멸; (12700,12328)+세 BCH 모두 outer iteration(최대 2회)으로 QC-LDPC 단독 이하 BER 달성, 복잡도 증가 미미; 동rate 단일 BCH(12700,11531)보다 우수.
10. 한계: HW 미설계·시뮬만, dominant trapping-set 탐색이 부호별 선행 필요, concatenation rate loss는 잔존.
