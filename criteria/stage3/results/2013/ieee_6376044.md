### B. 알고리즘 요약

1. 대상: circulant permutation sub-matrix + zero sub-matrix로 구성된 layered(QC) LDPC, 예제 코드 `PEGReg504x1008`(rate `1/2`, regular (3,6)) 및 irregular, AWGN/BPSK.
2. 문제: SP(sum-product)는 성능 우수하나 복잡도↑ throughput↓; 기존 BF/GDBF는 block 전체 min 연산이 필요해 `log2(N)` 비교단으로 ASIC critical path 지연.
3. 기법 배경: GDBF inversion function `Δk^GD(x)=xk·yk + Σ_{i∈M(k)} Π_{j∈N(i)} xj` (hard decision + soft value 상관항).
4. 핵심 기법: TATBF — bit-local 음수 threshold `λk`와 대조, `Δ<λ`면 bit flip 후 관련 check 방정식 즉시 갱신, 아니면 `λk=θλk`로 스케일 → global min 불필요.
5. turbo 원리: TDMP(layered)처럼 방금 갱신된 값을 같은 iteration 내 재사용, `D=M/Z` layer를 순차 처리, 각 layer의 `Z` row 동시 복호 → 평균 iteration 대폭 감소.
6. 확장(ITATBF): 목적함수 `f(x)=Σ xi·yi + Σ_i Π_{j∈N(i)} xj`의 local maximum 탈출 위해 perturbation(`λk=θ2+α`, `α`~N(0,σ)) 후 1 iteration 수행하는 escape 절차.
7. 파라미터: ITATBF `λ0=-20, θ=0.9, θ2=0.1, σ=1.2`, 최대 iteration 100.
8. HW 친화 포인트: block-wise min 제거로 critical path 단축, 목적함수 계산의 `O(log2(N))` 지연을 부분집합 합산으로 회피.
9. 결과: ITATBF가 모든 BF 중 최소 iteration + 최고 정정성능, MGDBF+escape 대비 평균 iteration 적음; TATBF는 ATBF/single-GDBF 대비 iteration 유의미 감소(그림상 약 절반).
10. 한계: BF 계열이라 SP 대비 BER gap, irregular 코드에서 error-floor 접근, HW 미구현(gatecount/throughput 수치 없음), MGDBF 대비 약 0.5dB 열세 구간 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping 복호 알고리즘 전체(TATBF) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 이터레이션 루프; BF로 전면 대체는 성능상 부적합, 알고리즘 컨셉만 참조 |
| block-wise min 제거 / bit-local threshold | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC min-sum의 min1/min2 연산 critical path 관점과 대비 참고 |
| layered(turbo) 스케줄, layer당 Z row 동시 처리 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 row 단위 병렬(z_sb=32)/Dual-Update 스케줄과 정합 검토 가능 |
| iteration별 threshold 스케일 `λk=θλk` | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR/threshold 테이블 개념과 유사, 단 BF용이라 직접 대응은 약함 |

적용 가치: **중간** — min-free critical path 단축과 layered/turbo 조기수렴 아이디어는 유용하나, min-sum 기반 Prime ECC를 BF로 대체하면 정정성능 손실이 커 부분 참고 수준.
