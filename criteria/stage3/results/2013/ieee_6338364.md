### B. 알고리즘 요약

1. 대상은 array-of-circulant QC-LDPC로, 각 nonzero submatrix가 K개 cyclic-shifted identity의 superposition(circulant weight `K≥1`)인 코드; CMMB rate 1/2·3/4, block length 9216bit, Z=256.
2. 문제: 전통 layered 디코더는 K=1만 지원, K>1이면 한 layer 내 같은 variable node에 여러 check node가 연결되어 APP LLR 갱신 충돌 발생.
3. 기존 해법([12]~[15])은 특정 코드 구조 전용이거나 병렬도 저하·layer 재정렬 필요로 범용성 부족.
4. 핵심: block-serial scheduling — parity check matrix를 layer별, 각 Z×Z submatrix를 병렬단위로 처리, 각 block을 1 clock에 처리해 latency가 K와 무관.
5. LLR combining 식 `Ln = Ln + Σ_{j}(R'_{mj,n} − R_{mj,n})`로 K개 check node 기여를 합산해 충돌 해소.
6. 디코딩은 scaled min-sum(scaling factor S=0.75), submatrix당 첫/둘째 최소값 `Am,Bm`을 두 번 scan(Q 계산 → LLR 갱신)으로 갱신.
7. HW 트릭: R-Mem은 min-sum 압축(first/second min + 위치 + sign)으로 저장, memory 구조가 K에 무관, permuter를 first/second scan 시분할로 면적 절감.
8. min-finder 유닛은 K에 따라 회로 변경(K=2는 CMP 5개+MUX 4개), K=3 일반화 가능.
9. 결과: CMMB용 65nm TSMC 합성 시 core area 3.9mm², 600MHz에서 15 iter로 1.1Gb/s throughput 달성(기존 대비 higher throughput).
10. 한계: LLR combining 부작용으로 K=2,3,4 시 0.1dB 미만 성능 저하, error-floor/정정능력 개선 목적 아님, 무선 방송(CMMB/DVB-S2) 타깃.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| block-serial layered 디코딩 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | K>1 circulant 지원 스케줄링; Prime ECC가 K=1이면 부분 참고만 |
| scaled min-sum CN 연산(0.75) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 압축저장(min1/min2/pos/sign)은 Prime ECC와 동일 계열 |
| permuter 시분할(row shift) | GT.cpp Make_GT_HW() | HW edge/라우팅 최적화 관점 부분 관련 |
| Z×Z 병렬 CNP / LLR-Mem 구조 | 디코더 HW 모델(z=32) | Z=256 vs z=32 정합 필요 |
```

적용 가치: 낮음 — min-sum 압축저장은 Prime ECC가 이미 보유, 논문 고유 기여인 K>1 circulant 지원은 Prime ECC가 K=1 QC-LDPC 전제로 보이면 불필요.
