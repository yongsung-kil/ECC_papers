> 총평: one-step majority-logic decoding의 조기종료(early termination) 아이디어는 Prime ECC의 partial CRC 조기종료와 개념적으로 유사하나, 코드 구조(EG-LDPC, N=15, serial shift-register, hard-1bit)와 정정 알고리즘(majority-logic vs min-sum) 자체가 근본적으로 달라 직접 이식 가치는 낮음.

### B. 알고리즘 요약

1. 시스템: (15, 7, 5) EG-LDPC 코드(rate `7/15≈0.47`)를 이용한 memory(SRAM 등) soft-error/SEU 대응 ECC, systematic generator matrix `G=[I:X]`.
2. 문제: 기존 one-step majority-logic decoder(MLD)는 error 유무와 무관하게 항상 `N+2` 사이클(코드 길이만큼) 소요되어 error-free 워드에서도 메모리 read latency가 큼.
3. 핵심 모델: cyclic shift register + XOR 행렬로 계산한 parity check sum `{Bj}`의 다수결(majority) 값으로 각 비트 정정 여부를 결정하는 Type-II one-step ML decoding.
4. 핵심 기법: 제안하는 MLDD는 기존 MLD와 동일한 알고리즘을 쓰되, 처음 3사이클 동안의 `{Bj}` 결과가 모두 0이면(휴리스틱 근거: "최대 4비트 오류는 3사이클 내 검출된다"는 가정 [12]) 즉시 error-free로 판정해 조기 종료.
5. 핵심 회로: control logic unit(2개 shift register + OR게이트 2단으로 3사이클 결과 결합)과 output tri-state buffer로 조기종료 시 디코딩을 건너뛰고 바로 출력.
6. 오류 있는 경우: 기존과 동일하게 전체 `N+2` 사이클 디코딩을 수행(조기종료 실패 시 페널티는 3사이클 추가에 불과, `N+5`).
7. 부수 구현: 인코더는 (15,7,5) EG-LDPC의 정보비트 복사 + XOR parity 계산 구조(Fig. 5, 6)로 별도 설계.
8. 학습/최적화 절차 없음(고정 구조 설계, exhaustive/random 오류 조합 시뮬레이션으로 3사이클 검출 가설만 검증).
9. 결과: Xilinx Spartan 3E-XC3S500E FPGA 합성, error-free 코드워드에서 기존 MLD(3033ns) 대비 MLDD(1934ns)로 약 1100ns speedup, 총 delay 12.578ns vs 14.006ns, 전력소모 47mW vs 49mW(약 4% 절감).
10. 한계: code length 15로 매우 작음, hard-1bit majority-logic decoding으로 NAND용 min-sum 기반 QC-LDPC와 알고리즘·구조 모두 상이, SRAM/nanoscale memory SEU 대응이 목적이며 NAND flash 실험/타깃 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 조기종료(3사이클 내 error-free 판정) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 조기종료 아이디어 유사(Partial CRC의 termination point와 유사한 목적) — 단 majority-logic 기반이라 실 구현 이식은 불가 |
| One-step majority-logic decoding (XOR 행렬 + 다수결) | 대응 없음 | Prime ECC는 min-sum 기반 iterative message passing이며 majority-logic 디코더 구조 자체가 없음 |
| EG-LDPC 부호/인코더 구조 | `encoder.cpp` `LDPC_encoder_4KB()` | 대응 없음에 가까움 — EG-LDPC(15,7,5)는 Prime ECC의 QC-LDPC dual-diagonal 인코딩과 무관 |

> 적용 가치: 낮음 — 조기종료 개념 자체는 Prime ECC에 이미 Partial CRC로 구현되어 있고, 코드 구조(EG-LDPC, 극소 length, majority-logic hard decision)가 NAND용 QC-LDPC min-sum 디코더와 근본적으로 달라 이식 실익이 없음.
