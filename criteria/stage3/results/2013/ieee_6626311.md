### B. 알고리즘 요약

1. 대상: regular `(J,L)` binary QC-LDPC, PCM은 P×P 순환치환행렬(CPM) `I(p_{j,l})` 블록으로 구성, exponent matrix `E`로 완전 결정.
2. 문제: QC-LDPC는 HW 친화적이나 girth가 낮아 성능 저하 → short cycle 없는 구조적 구성이 필요.
3. 핵심가정: exponent를 변수 분리형 `p_{r,c}=f(r,c)=g(r)h(c)`로 제한(독립 row-column 매핑), 한 함수를 고정하면 나머지를 저비용 탐색.
4. 핵심기법: g,h가 서로 다른 non-negative 정수열이면 4-cycle 없음, girth≥8 필요충분조건은 `(gj−gi)(hm−hl)≠(gk−gi)(hn−hl)` (Lemma 1).
5. 핵심식: Theorem 1 — 위 조건 하에 `P>pmax=gmax·hmax`인 임의 CPM 크기 P에서 girth≥8 유지(LHS의 절댓값 상·하한 증명).
6. 확장: h가 초항0·공차1 등차수열일 때만 GCDg8과 동일, 그 외 임의 h에는 GCDg8 미적용 → IRCMS가 GCDg8을 일반화.
7. 트릭: mask matrix `M(J,L,R,C)`로 CPM 블록을 선택적 제거해 더 sparse한 PCM 생성(4·6-cycle 유지).
8. 알고리즘: g를 {0,1}에서 시작해 girth-8 위반 시 후보 증가시키는 sieve, 복잡도 `O(J³L³)`(GCDg8 `O(J³)`보다 높음).
9. 결과: (3,6) rate0.5 len3000, (3,5) rate0.4 len5000 두 예제, AWGN/BPSK/BP 100 iter에서 BER이 GCDg8·PEG QC-LDPC와 동등.
10. 한계: HW 미설계, AWGN 시뮬뿐, 성능은 PEG와 comparable(우위 아님), 정정능력/error-floor 정량 보증 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-8 exponent matrix(IRCMS) 구성 | ecc_top.cpp Load_PCM() | 낮음: 결과 H-matrix를 로드할 수는 있으나 구성 알고리즘 자체는 대응 모듈 없음, Prime ECC는 고정 base+shift QC-LDPC |
| CPM/exponent matrix 구조 | ecc_data.h PCM_b (base+shift) | 낮음: QC 구조 개념은 공유하나 mask/독립매핑 방식은 미대응 |
| mask matrix sparsification | 대응 없음 | Prime ECC PCM은 고정, mask 기반 재구성 미지원 |
```

마지막 한 문장 **적용 가치**: 낮음 — binary QC-LDPC라는 부호종류는 Prime ECC와 일치하나, 이미 고정된 고rate H-matrix를 girth-8 IRCMS로 교체할 실익이 없고 성능도 PEG와 동등 수준에 그쳐 재검증 부담만 크다.
