### B. 알고리즘 요약

1. 시스템: NVM 스토리지용 rate-0.9 4489-bit(4489,4087) mirror-paradigm RS-based QC-LDPC, 6×67 base matrix, TPMP(two-phase message passing) partial-parallel 디코더, 6-bit 균일양자화(sign 1 + int 2 + frac 3), N_ITR 최대 10.
2. 문제: QC-LDPC 디코더 throughput 효율 `TEFF = f_MAX·L_CODE/(t_OCD·N_ITR)`에서 f_MAX·t_OCD가 병목 - min/sub-min 산출용 cascaded tree comparator가 critical path 지연·면적 증가.
3. 모델: Box-Muller AWGN 채널 HW Monte Carlo, syndrome detector 기반 종료, TPMP 스케줄.
4. 기법1 (dual-shift cyclic generation): 기존 circulant 생성은 한 방향 1-shift/clock로 CS clock 소요 → 좌/우 양방향 동시 shift로 H-matrix 복제 clock을 절반(CS=11 → 11clock→6clock)으로 축소, 병렬 thread 2배.
5. 의미: circulant 복제 지연이 t_OCD의 큰 부분이므로 dual-shift로 반감 → throughput 2배 잠재력(단 서브모듈 유닛·counter·메모리도 2배).
6. 기법2 (stochastic-detection of local minima): 6-bit 유한정밀 하에 local minima는 이진패턴 유한집합으로 축소, threshold 0.5 미만 집합 `℘={"000000","000001","000010","000011"}` 존재여부만 검사 → bit-wise 비교 대신 패턴매칭으로 min/sub-min 탐색을 log(row weight)에서 1 clock으로 축소.
7. 구현 디테일: cascaded comparator 제거로 critical path·logic 감소, mirror-paradigm으로 dual-shift가 유발하는 address generation counter 증가를 상쇄.
8. 학습/최적화: 없음(구조적 HW 최적화).
9. 결과: Virtex-6 ML-605 FPGA, 대비 디코더들 대비 t_OCD=70 최소·f_MAX=257.3MHz 최고, Max TEFF 16.5Gb/s(약 300% 향상), routing complexity Medium, BER 손실 <5%, ideal MATLAB 대비 0.2dB 이내.
10. 한계: FPGA(LUT/slice)만·ASIC gatecount/전력 미보고, RS-based mirror-paradigm 부호에 종속, 정정능력 개선 아닌 throughput/latency 개선, dual-shift는 면적 증가 수반.
