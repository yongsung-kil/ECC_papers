### B. 알고리즘 요약

1. 대상: binary (N,K) LDPC, BPSK/AWGN. 실험 부호는 Mackay (504,252) regular(Code1)와 IEEE 802.16 (576,288) irregular(Code2).
2. 문제: 표준 BP는 trapping set 때문에 high SNR에서 error floor가 남아 저장/광통신 요구(극저 오율)를 못 맞춤.
3. 관찰: trapping set 중 초기 `|un|` 랭킹이 큰 bad node로만 구성된 집합은 잘 안 깨진다 → 이를 stable trapping set `Vs`로 정의.
4. 핵심 지표: 반복 중 결정메시지 `εn = un + Σ umn`. `Vs`의 bit node는 홀수/짝수차수 체크노드에서 옳고 그른 메시지를 동시에 받아 `|εn|` 랭킹이 `|un|` 랭킹 대비 크게 하락.
5. 라벨링 규칙: `rank(|un|)>α` 이고 어떤 iteration에서 `rank(|εn|)<β` (α<β)인 노드를 flipping set `Γ(n)`에 넣음.
6. Set-breaking: 최초 BP 실패 시 `un = −sgn(un)·IMax`로 flip 후 BP 재시작. `Γ(n)` 내 노드를 하나씩 시도해 하나만 옳게 flip되면 set 전체가 연쇄적으로 풀림(Fig.1의 (4,4) set 예).
7. 최적 iteration 선택: 미충족 체크노드 최소인 iteration(`min(wk)`)의 결정메시지가 trapping set 노드 위치추정에 최적 → 그 시점 hard-decision `zα`를 참조/최종출력으로 사용.
8. 파라미터 탐색: coarse step p=50, fine step q=5로 (α,β)를 격자 탐색, BER와 재시도횟수 T의 tradeoff로 선택. Code1=(150,395), Code2=(100,400).
9. 결과: error floor 저감이 Augmented BP(Alg1)와 유사 성능이면서 재시도 훨씬 적음 — Code1 3.6dB에서 복잡도 Alg1의 1/10, 1.8dB에서 4.41회(vs Alg2 34.84회).
10. 한계: HW 미설계, 시뮬만. AWGN 채널·중저rate 부호 전제, re-decoding 반복으로 최악 latency 증가, α/β 이론모델 없이 코드별 대량 시뮬 튜닝.
