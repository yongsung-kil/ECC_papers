### B. 알고리즘 요약

1. 시스템: rate `r0=1/2`, length `n0=1024`인 systematic LDPC mother code(regular 및 irregular)에서 parity bit만 puncturing하여 rate `r1<...<rm`의 RC-LDPC 코드열 생성.
2. 문제: 기존 puncturing 기법([2]-[5])은 waterfall/error-floor 동시 최적화가 부족하고 특정 rate 범위에서 성능 저하.
3. 채널/모델: AWGN + BPSK, sum-product(BP) 디코더, 최대 50 iteration.
4. 핵심 기법: 각 rate 단계마다 puncture할 bit 수 `Nk = ceil(n0(rk-r0)/rk)`를 (1)식으로 계산 후 한 bit씩 순차 puncturing.
5. 선택 절차 1단계: check node 인접 punctured bit 수 `F(c)`가 최소인 check 중 recovery tree 내 unpunctured node 수 `U(c)`가 최소인 check를 후보(`CandidateCheck`)로, 그 인접 variable node를 `CandidateVar`로 선정.
6. 선택 절차 2단계(계층적 tie-break): `H(v)`(경계 punctured 이웃 수) 최소 → `d(v)`(차수) 최소 → `K(v)`(이웃 check의 recovery tree unpunctured 노드 합) 최소 → 마지막으로 ACE score spectrum(짧고 low-ACE 사이클 참여도) 최소 순으로 좁혀 최종 선택.
7. ACE score spectrum: cycle length `ℓ`, ACE값 `η≤ηmax`인 사이클에 variable node `v`가 참여하는 횟수를 `ℓ`별로 tuple화하여 비교(`ηmax≈4`, `ℓmax≈14~22` 사용).
8. 학습/최적화 절차 없음 (결정론적 그리디 알고리즘, Algorithm 1로 명시).
9. 결과: (3,6)-regular 및 PEG 기반 irregular(length 1024) 코드에서 rate 0.6~0.9까지 기존 4개 기법 대비 waterfall 이상 및 error-floor 우수, 특히 고rate(0.9)에서 우위 확인(FER 곡선, 수치 gate/throughput 없음).
10. 한계: HW 구현 없음, 이론/시뮬(AWGN, length 1024)에 한정, NAND 채널·soft-read 특성 미고려.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Rate-compatible puncturing 알고리즘(bit 선택) | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 단계와 관련되나 puncturing은 Prime ECC의 고정 QC-LDPC 구조에 부재 개념 |
| ACE score spectrum 기반 사이클 회피 | 대응 없음 | Prime ECC는 puncturing 대신 고정 PCM 사용, cycle 기반 puncturing 선택 로직 없음 |
| Rate-compatible 코드열 생성 | 대응 없음 | 단일 고rate QC-LDPC 고정, multi-rate 지원 구조 없음 |

> 적용 가치: 낮음 — Prime ECC는 단일 고정 rate QC-LDPC 구조로 rate-compatible puncturing 자체가 설계 목적에 없고, 부호구성 변경(H-matrix 재검증 부담)까지 요구되어 이식 실익이 낮음.
