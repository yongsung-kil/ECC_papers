### B. 알고리즘 요약

1. 시스템: IEEE 802.11n QC-LDPC, code rate `1/2`, code length `648` (info `324`), base matrix `Z=27/54/81` circulant shift, BPSK+AWGN 채널, partial-parallel 디코더.
2. 문제: partial-parallel LDPC 디코더는 area는 작으나 routing 혼잡·긴 iteration latency, 조기종료도 최초 iteration에서 CNU/BNU 1회는 반드시 수행해야 함.
3. 모델: 수신 `y_i`, noise variance `σ²`, 초기 LLR `L_Bi = -2y_i/σ²` (Eq.4), min-sum-correct 알고리즘(Jacobian log 근사, Eq.6).
4. 기법1 (SNR processor): SNR은 `1/σ`에 비례하므로 입력 `y_i`에 SNR을 곱해 bit-node를 최적 초기화. 곱셈은 shift+add + SNR mapping table(Tab.I) look-up으로 대체.
5. 의미: `L_Bi`의 `1/σ²` 항을 SNR으로 정밀 초기화 → 적은 iteration에서 BER 개선(iteration 많아지면 개선폭 감소).
6. 기법2 (CNU/BNU 재배치): 원래 Init→CNU→BNU 순서를 BNU→CNU로 재배치, BNU에서 hard-decision과 초기화를 동시 수행 → 고SNR 시 CNU 없이 조기종료, 성능 불변·routing 공유.
7. 구현 디테일: 조기종료 비교/hard-decision 생성 시점을 2~3 step 앞당김(Tab.II), init routing을 후속 CNU와 공유해 congestion 완화.
8. 기법3 (latency reduction): PCM 행·열 재배치로 3개 CNU/BNU 그룹 분할, BNU1↔CNU3, BNU3↔CNU1이 비겹침 → 동시 실행으로 iteration당 clock 1/3 절감(Fig.10).
9. 결과: BER=10⁻⁴에서 SNR 정보로 0.3dB 개선(8 iter), fixed-point 4int/4dec, TSMC 90nm 674K gate, 5.3mm² die, 151MHz, throughput 37~319Mbps.
10. 한계: rate-1/2 무선 표준 전용, SNR을 파일럿으로 안다는 전제, error-floor·NAND soft-read 미다룸, 시뮬+합성만(실측 없음).
