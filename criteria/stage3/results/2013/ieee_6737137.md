### B. 알고리즘 요약

1. 시스템: WiMAX(2304,1152) rate 1/2, WiFi(1944,972) QC-LDPC(mb=12, nb=24, Z=96/81), NVIDIA GTX TITAN(Kepler, 2688 core) GPU, 메시지 32bit float.
2. 문제: 기존 GPU LDPC 디코더는 다수 codeword 묶음으로 throughput은 올리나 latency가 커 실제 SDR에 부적합, throughput·latency 동시 개선 필요.
3. 모델: scaled Min-Sum(MSA), CN 갱신 `Rmn = α·sign_prod·min|Q|`, `α=0.75`, VN 갱신 `Ln += (Rnew-Rold)`, `Qmn = Ln - Rnew`.
4. 핵심 기법: two-min algorithm(TMA) — CN당 min1/min2/idx/sign_prod 4항만 저장해 `Rmn` 복원, sign은 char bitmap `sign_bm`으로 압축 저장.
5. 식 의미: FBA 대비 CS 연산 `M(ωr-1)`로 감소(TABLE I)하나 MSA와 동일 정정성능 — 순수 복잡도/메모리 접근 최적화.
6. 병렬화: CNP/VNP를 별도 커널, multi-codeword(NCW×NMCW), grid=(mb,NMCW), block=(Z,NCW), Rmn·Ln만 저장하고 Qmn on-the-fly 계산.
7. 메모리 최적화: 완전 coalesced 접근(shared memory 캐시로 shift로 인한 misalign 해소), page-locked 메모리(+15%), warp(32) 정렬용 dummy thread 패딩(+20%).
8. latency 최적화: 비동기 memory copy로 커널과 전송 overlap(+17%), depth-first 멀티스트림 스케줄(최대 32 concurrent stream, Hyper-Q)로 latency 축소.
9. 결과: 단일 GPU WiMAX 316.07Mbps@10iter, latency 0.207~1.266ms(62.5~304Mbps), 4-GPU 집계 1.25Gbps@10iter.
10. 한계: HW(RTL/ASIC) 미설계 GPU 소프트웨어, 알고리즘 정정능력·부호설계 개선 없음, 32bit float은 NAND 저bit read 비용과 정반대.
