### B. 알고리즘 요약

1. 시스템: MLC NAND flash용 hybrid hard-decision/soft-decision QC-LDPC 디코더. 코드 rate `8/9`, info length `2KB`, column weight `λ=4,5,6`, circulant `p×p`.
2. 문제: soft-decision은 성능 좋으나 반복 셀 센싱으로 read latency·throughput 손해. P/E cycling 낮아 RBER 낮을 때는 고throughput의 hard-decision BF 디코더가 유리 → BF의 HW 비용·평균 iteration을 줄여야 함.
3. 모델: syndrome `S=Z×H^T`, 각 비트의 unsatisfied 정도 `U(i)=Σ_{q∈W(i)} S(q)` (≤ column weight `λ`), 플리핑 임계 `δ=u_max`.
4. 핵심기법(FDBF): 글로벌 비교 전에 이전 iteration의 `u_max-1`(첫 iteration은 `λ-1`)로 `δ`를 사전 설정. `U(i)>δ`면 즉시 플립, `U(i)==δ`면 1비트 `E(i)`에만 저장 → 비교 완료 후 `δ==u_max` 확인 후 반영.
5. 핵심식: `U(i)` 저장을 3비트→1비트(`E(i)`)로 축소. 전통 BF는 `δ`를 전역비교 끝까지 몰라 축소 불가했던 것을 사전예측으로 해결.
6. 2단 트릭: `U(i)`를 `λ,λ-1,λ-2,λ-3` 4값으로 제한하고 각 값을 one-hot 4-wire로 표현 → 글로벌 비교를 logic AND(1-wire algebra)로 대체, `(λ≥δ)=λ6+δ̄6(λ5+δ̄5(λ4+δ̄4·λ3))` 형태.
7. HW: codeword(register array), syndrome(FF array), shift logic는 banyan network로 행렬-벡터 곱 라우팅 축소, flipping logic, syndrome check FSM. `λ=6`는 CSA 6→2 압축으로 `U(i)` 계산.
8. 최적화 절차: 별도 학습 없음 (결정론적 HW 알고리즘 변형).
9. 결과: TSMC 65nm ASIC 합성, parallel 512bit, `λ=4`. 총 면적 0.833→0.497mm2(40.4%↓, memory 53.8%↓), throughput ~10%↑, decoding failure rate는 전통 BF와 동등(성능 손실 없음). column weight 4에서 iteration당 77 clock, 200MHz.
10. 한계: hard-decision BF 경로 전용(정정성능은 soft-decision 디코더가 담당), throughput 절대 Gbps 미명시, error-floor/waterfall 정정력 개선 아님.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|

| U(i) 임계값 사전예측·1비트 저장 | `decoder.cpp` VN 처리 계열 | 메모리 절감 아이디어이나 min-sum magnitude 양자화와 구조 다름 |

| banyan network shift 라우팅 | `GT.cpp` `Make_GT_HW()` | 둘 다 HW edge/permutation 라우팅 축소 지향(접근 상이) |
| soft-3bit / hard-1bit 경로 전환 | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD) | Prime ECC도 HD/2SD/3SD 경로 보유 — hybrid 개념 정합 |

적용 가치: **중간** — NAND hybrid HD/SD의 hard-decision 경로 면적/throughput 개선으로 타깃은 정확히 일치하나, Prime ECC는 이미 HD를 min-sum으로 처리하므로 BF를 별도 저비용 HD 경로로 추가할지에 따라 가치가 갈린다(정정성능 향상은 아님).
