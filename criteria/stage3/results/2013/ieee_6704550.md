### B. 알고리즘 요약

1. 시스템: binary (j,k)-regular LDPC, AWGN/BPSK, min-sum 디코딩. 대상 부호 (96,50) Gallager, (82,41) BIBD, (1536,768)·(155,62) QC-LDPC. XILINX Spartan-6 FPGA 구현.
2. 문제: 기존 QC-LDPC 디코더는 CPM을 block RAM에 매핑하는데 CPM 수 >> RAM 수이면 매핑 불가, 랜덤/IRA 부호(DVB-S2)엔 부적합하며 flexible 디코더는 비용이 큼.
3. 모델: 표준 min-sum — `L(rji)=(∏sign)·min|L(qij')|`, `L(qij)=L(cj)+Σ L(rji')` (교과서 근사, related factor 튜닝).
4. 핵심 기법(메시지 패킹): VN→CN, CN→VN 메시지를 Extrinsic RAM에 배치하고 Address RAM으로 CNU read/write alignment를 관리해 dual-port RAM에서 충돌 없이 접근.
5. 식 의미: CNU 동작 시 read/write 순서를 Address RAM이 지정해 계산유닛 간 memory conflict를 제거 → block RAM 수 최소화가 목표.
6. 핵심 기법(row grouping): 같은 위치에 `1`이 겹치는 행을 아래로 밀어 γ개(=column weight) 그룹으로 분할, 그룹 내 CNU 병렬 수행. 겹침 심하면(BIBD 25그룹) serial 구조로 fallback.
7. 구현 디테일: invalid 위치에 flag 비트로 무효 메시지 표시, VNU는 3-cycle pipeline(6 VNU), RAM 클록을 global(≤100MHz) 대비 2~4배로 올려 throughput 증가.
8. 최적화: 없음(구조 설계뿐, 학습 없음).
9. 결과: (1536,768) QC-LDPC에서 block RAM 36→9개(1/4)로 줄이며 throughput 92.4→100 Mbps 유지. Titer=768, CNU 6/VNU 12.
10. 한계: 짧고 저rate인 부호만 실증, throughput 100Mbps급으로 낮음, 정정능력 개선 아님(메모리 최적화), 고rate NAND QC-LDPC와 결이 다름.
