### B. 알고리즘 요약

1. IEEE 802.11n block-structured QC-LDPC(Z×Z circulant/null 서브행렬, Z=expander factor)의 인코더+디코더를 GPP(SDR)에 구현.
2. 인코더: H를 A/B/T/E/F 블록으로 분해, `p1=E*T^-1*A+F...`, `p2=T^-1*(A*S^T+B*p1)` (식6~7) — 5개 블록행렬만 LUT로 사전저장(총 A 1182B, F 198B, B/T^-1/E 26/204/26B로 L2 cache 적재).
3. 인코딩은 circulant shift-down + xor로 환원, 128bit SIMD 1명령이 16 연산 동시 처리.
4. 디코더: layered(TDMP) + offset min-sum(OMS) MP. layered는 layer마다 message 갱신해 TPMP 대비 수렴 개선, SIMD data-level 병렬에 적합.
5. 양자화: `(q,Δ)` uniform, 1bit sign + (q-1)bit magnitude, 알파벳 크기 `2^q-1`, 채널값 `yn=min(floor(|yn|/Δ+0.5),T)`.
6. MIN phase: `tn = qn(shifted) - r^{i-1}`에서 OMS value-reuse로 각 entry의 min1·min2만 계산, sign 벡터 s 유지.
7. SEL phase: `sel(m1,m2,v,n)`로 min1/min2 선택 후 OMS(공통 offset `E`)로 새 r 계산, `qn`은 shift-down한 `tn+r` 합.
8. clipping: Q-value 대신 R-message clipping(식8)으로 overflow 방지, `_mm_adds_epi8`/`_mm_subs_epi8` saturating 명령으로 자동 처리.
9. 병렬도 `P=Z/ceil(Z/16)`, Z=81/54/27에서 P=13.5; Z=81 layer를 16-row 6그룹으로 분할 병렬.
10. 결과: Core i7-2600K 단일코어, 8bit fixed vs float 미미한 손실, layered 5 iter ≈ TPMP 10 iter, 디코딩 throughput 30~56Mbps(rate 1/2~5/6), HW gatecount 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered(TDMP) 디코딩 스케줄 | decoder.cpp LDPC_Decoder() | Prime는 dual-update(even/odd) 스케줄, layered 개념 비교·병존 검토 |
| offset min-sum min1/min2 value-reuse | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime min-sum과 동일 계열, OMS offset E 추가는 소폭 개선 여지 |
| 8bit fixed-point 양자화/R-message clipping | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | Prime 2~3bit LLR 테이블과 폭 상이, clipping 전략 참고 |
| QC-LDPC circulant shift 인코딩(A/B/T/E/F 분해) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime는 dual-diagonal, 802.11n 분해법은 다른 QC 구조 |

```

적용 가치: **중간** — binary QC-LDPC의 layered+OMS와 R-message clipping·SIMD 병렬화 아이디어는 참고할 만하나, HW 미설계이고 802.11n wireless 부호(rate 0.5~5/6, 8bit soft)라 Prime 고rate NAND QC-LDPC로는 알고리즘 개념 차용 수준에 그침.
