### B. 알고리즘 요약

1. 시스템: turbo(double/single binary)와 LDPC를 겸용하는 재구성 가능 디코더, WiMAX/WiFi/CMMB/DTMB/HPAV/LTE/DVB-RCS 등 다표준 지원, BPSK/AWGN.
2. 문제: 표준 간·내 코드 전환 시 복호를 멈춰야 하는 reconfiguration 문제를 최초로 정형화하고 중단 없는 전환 기법 제시.
3. 모델: LDPC는 layered normalized min-sum(6), turbo는 BCJR(Max-Log-MAP), 둘 다 LLR 기반 반복복호이며 PE(processing element)에 작업 분할.
4. 핵심기법 1단(NoC): 각 노드에 PE+RE(routing element, 4×4 crossbar), Kautz 22-node topology, SSP-FL 결정적 라우팅(Floyd-Warshall off-line 사전계산).
5. 핵심식 의미: 한 iteration의 clock cycle 수가 PE가 읽는 memory location 수에 비례 → reconfiguration memory 크기가 전환 시간 결정.
6. 핵심기법 2단(reconfiguration): reconfiguration memory를 circular buffer(SCC/ECC/SFC/EFC 포인터)로 관리, 5단계 phase로 현재/차기 코드 복호와 설정 로딩을 overlap, 소/대 코드 case 분리.
7. 구현 디테일: min-sum은 CN당 2개 magnitude + 2-bit index만 저장해 필요 bit 21.2% 절감; LDPC 채널 LLR 양자화 7→6bit로 축소(성능손실 <0.05dB).
8. 최적화 절차: METIS graph coloring으로 H-matrix를 topology에 분할, mesh/toroidal/spidergon/Kautz 등 비교해 Kautz 선정.
9. 결과: TSMC 90nm post-layout, 구현 A는 WiMAX/HPAV/DVB-RCS 지원 2.75mm², 87.8mW(WiMAX LDPC)~101.5mW(turbo); reconfiguration 확률 0.25~0.3%에서 BER 손실 무시가능.
10. 한계: 정정성능 개선 없음(유연성·면적 최적화 목적), 무선 다표준 특화, NAND read 비용/soft-2~3bit 고려 없음.
