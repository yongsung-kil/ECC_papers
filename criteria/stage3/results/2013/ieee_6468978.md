### B. 알고리즘 요약

1. 시스템: rate `R=2/3` 등 고정 rate mother LDPC code에 `Kd`개 dummy bit(값 0으로 고정, 송신 전 puncture)를 삽입해 다양한 effective rate `Reff = Keff/(Keff+M)`를 얻는 rate-compatible 방식.
2. 문제: 짧은/중간 블록길이에서 Tanner graph의 short cycle이 message-passing 성능 저하의 주요 원인이며, 기존 dummy bit 삽입 방식([10],[11])은 cycle을 크게 줄이지 못함.
3. 채널/모델: AWGN + BPSK + Belief Propagation(BP) 디코딩, dummy bit는 수신측에서 완전히 알려진 값(LLR=+∞)으로 취급.
4. 핵심 기법: 각 정보노드 `i`를 지나는 길이 `l`의 cycle 수 `C_i^l`을 계산해, 현재 최단 girth `g_I`를 지나는 cycle 수가 가장 많은 노드부터 순서대로 dummy bit 위치로 선택(Algorithm 1, Breaking Cycles).
5. 의미: dummy 노드는 디코더 관점에서 그래프에서 완전히 제거되는 것과 동일(관련 edge 모두 제거)하므로, cycle을 가장 많이 지나는 노드를 없애면 effective graph의 short cycle이 크게 줄어 성능이 개선됨.
6. 확장: 계산복잡도 `O(N^3)` 문제를 해결하기 위해 cycle 수를 1회만 계산 후 선택하는 단순화 버전 Algorithm 2 제안.
7. 구현 디테일: [18]의 message-passing 기반 cycle counting 알고리즘(`O(gE^2)`)을 노드별 cycle count로 변형해 재사용.
8. 최적화 절차 없음(별도 학습/DE 없음, PEG mother code는 기존 구성 그대로 사용).
9. 결과: rate 1/2 PEG(N=2304) mother code에서 `Kd=768`(Reff=1/4) 시 길이-6 cycle을 기존 대비 약 90% 감소, FER 기준 기존 방법 대비 최대 1.5dB 이득, error rate 10^-4에서 최적화 PEG 코드 대비도 waterfall 구간 근접.
10. 한계: 이론/시뮬레이션만(HW 미설계), 무선통신 RC(rate-compatible ARQ) 시나리오 전용 설계로 고정 rate NAND ECC 맥락과는 목적이 다르며 error floor가 남음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Dummy bit insertion에 의한 rate-matching (multi-rate 지원) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 단일 고rate QC-LDPC 고정 설계로 다중 rate 지원 목적이 없어 관련성 낮음 |
| Short cycle 카운팅 기반 열 선택 알고리즘 | 대응 없음 | 부호구성 알고리즘으로 코드 자체가 특정 QC-LDPC 고정이라 이식 난이도 높음 |
| BP(sum-product) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 BP 자체는 대응 없음, 단 이터레이션 루프 구조만 유사 |

적용 가치: 낮음 — 무선통신 rate-compatible 시나리오를 위한 부호구성 최적화로, 단일 고정 rate NAND binary QC-LDPC 구조와 목적/전제가 달라 이식 실익이 적음.
