> 총평: Hamming 성분부호 기반 generalized LDPC의 rate-compatible puncturing 설계로, SPC check 기반 binary QC-LDPC를 쓰는 NAND ECC와는 부호구조·용도(HARQ)가 달라 이식성 낮음.

### B. 알고리즘 요약

1. 대상은 efficiently-encodable generalized LDPC(EE-GLDPC): check node에 SPC 대신 Hamming(15,11) 등 선형 성분부호(super check node, SCN)를 쓰는 systematic 부호, blocklength 1024~1095, rate 0.467~0.9.
2. 문제: 전통 sGLDPC는 고rate 설계가 어렵고(rate constraint), GLDPC용 효율적 puncturing 알고리즘 연구가 거의 없음.
3. 핵심 개념: erasure decoder로 부호화 시 punctured VN이 l회 iteration 후 복구되면 `l-SR` 노드, 전체 복구 iteration 최대값 `L`을 최소화하는 것이 목표.
4. 핵심 기법: 그리디 depth-first puncturing - 매 스텝 후보 VN을 하나 punctured했을 때 (1)`L` 최소, (2)조기 iteration에 더 많이 복구되는 가중비용 `W=Σ2^(i-1)|Gi|` 최소, (3)deficiency 벡터 `T̃` 평균, (4)`Var(T̃)` 최소 순의 4기준으로 최적 노드 선택.
5. 의미: SCN은 최대 `(n-k)` erasure까지 복구하나 erasure가 많을수록 extrinsic 신뢰도가 급락(EXIT 곡선), 따라서 deficiency `τ`를 낮고 균일하게 분산시켜야 복구 신뢰도 확보.
6. 확장: hGLDPC(SCN+SPC 혼합)에는 EXIT 곡선 아래 면적 `A=∫IE(IA,τ)dIA`로 SCN/SPC 정정능력을 점수화, MI 기반 기준(3)(4)로 강한 SCN을 우선.
7. 구현 디테일: 성분부호 SCN 복호에 Johansson-Zigangirov MAP decoder 사용, Imax=200.
8. 최적화: hGLDPC 앙상블은 EXIT curve-fitting + differential evolution으로 프로파일 도출(threshold Eb/N0=0.56 dB), PEG로 Tanner 그래프 생성.
9. 결과: systematic puncturing이 random 대비 BER=10^-5에서 R=0.8일 때 약 0.8 dB, R=0.9에서 약 1.1 dB 이득. GLDPC는 EERC/IRA-LDPC보다 error floor 우수(R=0.5, BER=10^-7에서 0.8 dB 이득)하나 고rate에서 decoding threshold 손실.
10. 한계: HW 미설계, 부동소수 시뮬만, non-SPC 성분부호 구조라 SPC-check binary QC-LDPC와 이질적, HARQ 지향.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC(super check node) 부호구조 | `ecc_top.cpp` `Load_PCM()` | 대응 없음 수준 - Prime ECC는 SPC-check binary QC-LDPC 고정, SCN(Hamming) 성분부호 미지원 |
| systematic puncturing / rate-compatible 설계 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | NAND ECC는 고정rate·비HARQ, puncturing 파이프라인 자체가 없음 |
| erasure 기반 반복 인코딩(l-SR) | 대응 없음 | Prime ECC 인코더는 dual-diagonal 직접 인코딩, erasure decoder 인코딩 아님 |
| error floor 개선(EXIT/deficiency 분석) | `decoder.cpp` `LDPC_Decoder()` | error floor 관심은 공유하나 SCN 성분부호 전제라 QC-LDPC로 직접 전이 불가 |

적용 가치: **낮음** - generalized LDPC의 puncturing 기반 rate-compatible 설계는 SPC-check binary QC-LDPC + 고정rate NAND ECC와 부호구조·용도가 근본적으로 달라 떼어 쓸 여지가 거의 없음.
