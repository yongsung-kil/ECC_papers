### B. 알고리즘 요약

1. 대상은 spatially coupled(LDPC convolutional) 부호, `(3,6)`-regular protograph를 edge spreading으로 결합해 termination length `L`로 확장, AWGN에서 lifting factor `N=500`, 블록길이 `1000`(rate 0.5).
2. 문제: MAP threshold 도달에는 큰 `L`이 필요하나 이는 복잡도·구조적 latency를 키움. 슬라이딩 윈도우(크기 `W`)로 latency/복잡도를 `L` 무관하게 만들지만, 윈도우 내 uniform 업데이트는 기대만큼 복잡도가 줄지 않음.
3. 관찰: 윈도우 내 target 노드(`w=1`) 외 노드(`w>1`)는 iteration 진행에도 BER가 거의 변하지 않음(density evolution, BEC `ε=0.48`).
4. 핵심 기법: 각 위치 `w`의 output LLR로 soft BER 추정 `Pb(w)`를 계산, 직전 대비 개선폭이 파라미터 `θ`를 넘는 위치만 `updateList[w]=true`로 다음 iteration에 갱신(Algorithm 1).
5. 갱신식은 표준 SPA - CN은 `2 tanh^-1(∏ tanh(Lv,cj))`, VN은 `Lch(k)+Σ Lcl,vk`, output LLR `Lout(k)=Lch(k)+Σ Lcl,vk`.
6. 확장: exhaustive computer search 스케줄이 주기적이라는 점에 착안, 파라미터 `PR` iteration 동안 비활성 노드를 강제 갱신하는 주기성 추가(Algorithm 2)로 컴퓨터 서치와 동등 성능·복잡도 달성.
7. 변형: serial 업데이트 버전(Algorithm 3) - CN 갱신 전 이웃 VN 메시지를 먼저 생성, 위치 `w`의 CN은 동시 갱신 가능해 부분 병렬화.
8. 학습/최적화: 별도 학습 없음, `θ`는 최저 복잡도를 주는 값을 실험적으로 결정.
9. 결과: 비균일 parallel 윈도우가 균일 대비 35~40% 복잡도 절감, serial 비균일은 최대 50% 절감. BER 성능 손실은 무시할 수준.
10. 한계: HW 미설계, BEC density evolution + AWGN 시뮬만, convolutional(SC) 구조·윈도우 디코딩 전용이라 블록 코드로 직접 이식 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 비균일 노드 on/off 스케줄 (updateList) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | 낮음 - 윈도우 디코딩 전제라 블록 루프에 직접 대응 안 됨 |
| soft BER 추정 / output LLR 기반 조기 종료 | [decoder.cpp](../Prime_ECC_3.1_Claude) `Get_VNU_Table_Idx()`, [partial_CRC.cpp](../Prime_ECC_3.1_Claude) | 낮음 - Prime은 이미 partial/full CRC 조기종료 보유 |
| CN 갱신 (full tanh SPA) | 대응 없음 | Prime은 min-sum 사용, full-tanh SPA는 미대응 |
| 슬라이딩 윈도우 / 메모리 read access | 대응 없음 | Prime은 블록 QC-LDPC로 SC 윈도우 구조 없음 |

적용 가치: **낮음** - SC-LDPC convolutional 윈도우 디코딩 복잡도 최적화라 블록 binary QC-LDPC(Prime ECC)의 부호구조·디코더 루프와 정합되지 않는다.
