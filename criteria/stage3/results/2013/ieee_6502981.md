### B. 알고리즘 요약

1. 대상: 순열행렬 배열 기반 **구조적(QC) LDPC** 앙상블 `C_n(d_v, d_c, g)`, 사례연구는 `C_n(3, d_c, 8)` (column-weight 3, girth 8), dmin≤16.
2. 문제: dmin 계산은 NP-hard이고 error impulse/Stern/branch&cut 등 기존 저중량 codeword 탐색은 부분집합의 topology를 고려 못하며 weight>12에서 exhaustive 탐색이 비실용적.
3. 핵심 관점: codeword = `(a,0)` trapping set (`supp(x)`가 홀수차수 CN 0개인 유도 subgraph) → codeword 탐색을 작은 trapping set 탐색으로 분해.
4. 핵심 도구: **Trapping Set Ontology(TSO)** — trapping set 간 parent-child(predecessor/successor) 관계 DB로 개별 exhaustive 탐색을 대체.
5. 확장연산 3종: `EO1`(두 TS 합집합→`(c,d)`), `EO2`(`(a,b)→(a+1,b-1)`, 홀수 CN 2개 공유 VN 추가), `EO3`(`(a,b)→(a+1,b-3)`, 홀수 CN 3개 공유 VN 추가).
6. 구조 활용: 순열행렬 기반이라 CN에 **coloring**(색 `f:C→K`) 부여 가능 → 가능한 codeword 후보 수 상한, 탐색 복잡도 대폭 감소.
7. weight-14 사례: Lemma 1,2로 degree-6 CN·3개의 degree-4 CN 불가 증명 → codeword를 (elementary / degree-4 CN 1개 / 2개) 3범주로 분류 후 각 범주를 유한 단계로 전수 탐색(Alg.1~4).
8. 시작점: 모든 codeword가 8-cycle(=`(4,4)` TS) 포함 → (4,4)에서 시작해 확장연산 반복.
9. 결과(이론): weight-14 codeword를 유한 단계로 전수 열거 가능함을 증명, [28] 구성법과 결합 시 weight w까지 회피해 dmin ≥ w+2 보증. 저중량 codeword multiplicity도 산출.
10. 한계: **HW/디코더 미설계, 순수 구성법 이론**, `d_v=3`·girth-8 특화, 다른 column-weight/girth는 해당 TSO가 있어야 하고 복잡도 급증.
