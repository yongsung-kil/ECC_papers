### B. 알고리즘 요약

1. 시스템은 두 source 노드 A,B가 공통 destination D로 전송하며 complex AWGN + block Rayleigh fading 하에서 협력(coded cooperation)으로 diversity를 얻는 구조다.
2. 문제: repetition 기반 협력은 SNR accumulation뿐이라 outage가 높아, 파트너에게 추가 parity를 relay하는 information accumulation(coded cooperation)이 필요하고 이에 rate-compatible 부호가 적합하다.
3. 부호: rate-compatible protograph 블록부호(RCPBRL, RC-PN-PBRL, RC root-LDPC)에 `edge spreading`으로 SC 버전 생성 — base matrix `B`를 `1+ms`개 component `Bi`로 분해해 시간 `t..t+ms`로 edge 확산.
4. 종단 base matrix `B[1,L]`는 diagonal 구조이며 design rate `R = 1-(c-b)(L+ms)/(cL)`이 `L→∞`에서 `b/c`로 수렴(종단 rate loss 소멸).
5. RC는 고rate mother code에 parity를 "extending"으로 추가해 저rate로 확장, 협력수준 `β = N2/N`, spectral efficiency `η = K/N`, full-diversity 필요조건 `η ≤ min{β,1-β}`.
6. 수신 상태: destination D는 A/B의 협력/비협력 모드 조합에 따라 S0~S3 4상태로 복호하고, 협력 상태(S1,S3)에서는 MRC로 diversity 획득.
7. threshold 분석: reciprocal channel approximation(RCA) density evolution으로 BP threshold θ 계산, outage를 capacity-outage/code-outage 두 방식으로 평가.
8. windowed decoding(WD): SC부호의 convolutional 구조를 이용해 크기 `W` 윈도우만 갱신하며 slide → latency 감소(성능 소폭 손실), `W=10`이 전체 그래프 복호(W=L+ms=101)와 거의 동일.
9. 결과: threshold saturation으로 SC부호가 블록부호 대비 outage 10^-2에서 0.54~0.60 dB 이득, capacity 대비 2.5% 이내; `L=100`이 `L→∞`와 사실상 동일.
10. 한계: 무선 block-fading 협력통신 특화, 저rate(η≈1/3), HW 미설계, AWGN+fading 시뮬만.
