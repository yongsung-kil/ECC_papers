> 총평: MSA flooding 디코더를 GPU 클러스터/MPI로 SPMD 병렬화해 error-floor BER 시뮬을 가속하는 시뮬레이션 인프라 논문으로, 디코더 알고리즘·부호설계·NAND HW를 바꾸지 않아 Prime ECC 이식가치는 낮음.

### B. 알고리즘 요약

1. 대상 시스템: LDPC BER Monte Carlo 시뮬레이터를 GPU 클러스터에서 실행, 사례부호는 DVB-S2 `N=64800, rate 1/2`, QPSK/AWGN.
2. 문제: error-floor 영역(BER 1e-13~1e-15)의 정확한 BER 곡선 특성화에 방대한 비트 시뮬이 필요해 CPU로는 수개월~수년 소요.
3. 디코더 모델: 표준 Min-Sum SPA(`L(r_mn)=∏sign·min|L(q)|`, `L(q_nm)`, a-posteriori `L(Q_n)`) flooding 스케줄, 부호 자체·알고리즘은 교과서 그대로.
4. 핵심 접근: 인코더/모듈레이터 생략(all-zero codeword가 임의 선형부호에 속함), AWGN 채널이 LLR 직접 초기화, `T_i=N×Words/T_kernel`로 처리량 측정.
5. GPU 병렬화(핵심): flooding으로 BN 처리 N thread, CN 처리 M thread를 thread-per-node로 배정, 노드 간 통신 없이 register에서 계산.
6. 데이터 병렬 트릭: LLR을 fixed-point 8bit로 잡아 16 codeword를 128bit 벡터로 묶어 메모리-연산 오버랩 극대화(granularity 확대).
7. 난수/통계: XORWOW PRNG로 thread별 비중첩 시퀀스 skip-ahead, 통계 모듈은 16bit 정수 Hamming weight reduction으로 비트·프레임 오류 집계, max 50 iteration.
8. 클러스터 실행: GPU마다 독립 BER 시뮬(SPMD), MPI로 codeword를 P 분할, master는 로깅만 담당해 네트워크 통신 최소화(Infiniband/Ethernet 무관 유사 성능).
9. 결과: 1+16 노드(Tesla M2050) 클러스터에서 최대 32 GPU 사용 시 speedup 13.49(이론 16 대비), SNR2.0dB에서 평균 17.42 iteration, BER 1e-13을 수일 내 도달; 디코더가 전체 시간 약 2/3, 통계 30.76%.
10. 한계: 부호설계·디코더 알고리즘 개선이 아니라 시뮬 가속만, ITU-T G.709 BER 1e-15엔 32 GPU 초과 필요, NAND HW·정정성능과 무관.
