### B. 알고리즘 요약

1. 시스템: 저탐지(LPD) DSSS 수중음향 통신, short packet(rate-0.95 weight-2 regular LDPC, `k=95→n=100`, 198-chip DSSS), AWGN + 40kHz 샘플 수중채널.
2. 풀려는 문제: short block regime에서 BP는 short cycle 민감성으로 성능 보증이 어려움 → LP 복호가 대안(성능 보증·ML-certificate).
3. LP 복호 정식화: 각 code bit을 `č_i, č_i^- = 1-č_i` 양변수로 표현, parity 제약을 짝수합으로 완화(Eq.1~3), 목적함수는 LLR `γ` 가중합(Eq.4). 해는 polytope 꼭짓점(ML codeword 또는 pseudo-codeword).
4. 핵심 기여(SISO 확장): LP 해 `x̌(k)`를 참 soft output `x̄`의 양자화 관측으로 모델링, code polytope 기반 uniform 분포를 Gaussian으로 근사해 soft-info `x̃(k)` 추출.
5. 엔트로피 스케일링: 반복마다 출력 엔트로피가 감소한다고 보고 스칼라 `α(k)`로 `α(k)x̌(k)`의 엔트로피를 이전 iteration과 맞춰 `x̄(k)` 영역 경계 결정.
6. soft output 분포: 평균 `μ(k)=(1+α(k))/2 · x̌(k)`(Eq.5), 공분산 `V(k)=Σ(k)+A(k)diag(σ_L², σ_W²,...)A(k)^T`(Eq.6) — 양자화 잡음 방향별 분산 `σ_L`(x̌ 방향, 감소)·`σ_W`(고정).
7. 반복 수신기: 추출 soft-info를 MMSE 채널추정에 되먹임 → 스프레딩 시퀀스를 채널추정과 convolve 후 상관·복호를 수렴까지 반복.
8. 비교 수신기: SISO BP, DFE+standard LP, uncoded(공정 비교 위해 95 bit를 220-chip 확산) 3종.
9. 결과: first pass에선 BP가 1.5dB 우세하나, 10 iteration 후 SISO LP 수신기가 다른 방식 대비 1dB 우세(BP의 short-cycle 수렴 문제로 큰 오차에 강함).
10. 한계: 수중음향 특화 시뮬만, HW 미설계, LP soft-info 추출법 형식화·개선은 future work.
