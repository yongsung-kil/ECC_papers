### B. 알고리즘 요약

1. 대상: Turbo/LDPC 병렬 디코더의 하드웨어 interleaver(permutation network) 설계 방법론. 특정 부호/rate/length가 아니라 임의 interleaving law + 병렬도 P를 입력으로 받음.
2. 문제: 병렬 디코더는 메모리 뱅크 분할 시 여러 PE가 동일 뱅크에 동시 접근하는 memory access conflict 발생. 기존 방법은 network 또는 controller(ROM) 비용 최적화를 동시에 못함.
3. 모델: memory access schedule을 행=PE, 열=time step 매트릭스로 표현. 각 data의 lifetime `Γ(vi)=[τmin,τmax]`(write~consecutive read)으로 충돌 정의.
4. 핵심기법: `Address Conflict Graph (ACG)` — 정점=data access, 간선=lifetime 겹침(같은 뱅크서 다른 주소 필요). 뱅크당 ACG를 만들어 충돌 최소화.
5. 3단 flow: Bank Mapping(뱅크 배정, ACG 생성) → Address Mapping(주소 배정) → Code Generation(VHDL).
6. Bank mapping은 매핑 제약 준수 + ACG 충돌 최소화를 함께 고려(충돌 적을수록 이후 최적화 여지 큼), 못 풀면 register로 postpone.
7. Address mapping: 가장 제약많은 정점 우선 선택(`VertexValue`=degree+이미배정된 인접수), 메트릭 `w(v,@ki)=NbrOcc(@ki)-PropConf`로 뱅크 간 addressing regularity 최대화.
8. Regularity 최대화로 공통 addressing sequence를 단일 ROM으로 공유 → memory controller(ROM) 수·크기 축소.
9. 결과: Xilinx Virtex5/6 합성. HSPA/UWB/WiMAX 테스트케이스에서 기존[13]/[5] 대비 slice 면적 3.6~42% 감소(평균 24.5%), throughput 저하 없음(critical path 불변).
10. 한계: 부호설계·복호 알고리즘 무관한 순수 HW 인터리버/메모리 매핑 방법론, 결과는 통신 표준 대상, NAND QC-LDPC 대상 아님.

### C. Prime ECC 관련 모듈 핀포인트

대상이 other(디코더 성능 아님)이므로 C섹션 디코더 성능 항목은 N/A. 모듈 핀포인트만 기재.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 병렬 interleaver 메모리 매핑/충돌회피 | 대응 없음 | Prime ECC는 z=32 고정 QC-LDPC HW 매핑, 임의 interleaving 매핑 문제 아님 |
| 메모리/라우팅 구조 | 디코더 HW 모델(z=32, GT) | 개념적으로 라우팅 관련이나 ACG 방법론은 Prime ECC 고정구조에 직접 대응 없음 |
| controller ROM 압축 | 대응 없음 | Prime ECC 스케줄은 고정 base matrix 기반, 동적 매핑 최적화 대상 아님 |

적용 가치: **낮음** — 알고리즘·부호설계·정정성능 기여가 전혀 없는 순수 병렬 interleaver 메모리 매핑 방법론이며, Prime ECC의 고정 z=32 QC-LDPC HW 구조와 문제 정의가 다르다.
