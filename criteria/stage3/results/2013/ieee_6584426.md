> 총평: WiMAX 802.16e 전 rate 지원 block-circulant LDPC 디코더의 표준적 FPGA 구현·실측 테스트베드로, NAND와 무관하고 이미 보유한 min-sum류(λ-min) 대비 새 기여가 없어 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: IEEE 802.16e(WiMAX) LDPC 디코더, block length 2304bit, code rate `1/2, 2/3A/B, 3/4A/B, 5/6` 전부 지원, BPSK 20MS/s, AWGN(AWG 잡음원).
2. 문제: 표준의 각 code rate PCM이 대용량 메모리를 요구해 HW 실현이 복잡 - 면적/throughput/성능 트레이드오프 필요.
3. 모델: block-circulant(QC) 성질을 활용해 permutation 번호를 VNP에 수집, 24개 VNP + 단일 CNP + shuffle network 구조.
4. 핵심기법: `λ-min` 디코딩 알고리즘 (FPGA 친화 min-sum계 BP 근사) 채택, ControlSignal이 ROM에 저장된 PCM에 따라 VNP·shuffle network 제어.
5. 구현: OutputSyncArray가 VNP 처리 완료·출력 준비를 동기화, DCM으로 100→80MHz 클록 관리, 12bit soft 입력.
6. 테스트베드: VSG(Agilent E4438C) 신호원 + AWG 잡음원 + AD9432 A/D + Xilinx ML501(XC5VLX50) FPGA로 실시간 RF-베이스밴드 측정.
7. 데이터 취득: 심볼당 12bit 샘플, 2048 symbol/cycle, 80MHz, UART 115.2kbps 제어.
8. 결과(면적): Virtex-5 XC5VLX50에서 slice reg 3086, LUT 13555, BRAM 15, occupied slice 4446.
9. 결과(성능): 10 iteration·100MHz에서 throughput 9.6~27.91Mbps, latency 0.08~0.24ms (rate별).
10. 한계: error floor·에너지 미고려, throughput 수십 Mbps로 낮음, BER 곡선은 그림(Fig.5)에만 존재, 학술 테스트베드 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| λ-min 디코딩 알고리즘 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum계 근사로 Prime ECC의 min1/min2와 유사 - 이미 보유, 신규성 없음 |
| VNP/CNP + shuffle network 구조 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` (z=32 병렬, GT) | 병렬 구조 개념은 겹치나 Prime ECC의 pipeline/HCU/GT가 더 정교 |
| ROM 저장 PCM / block-circulant 로드 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 로드 위치 대응, 그러나 WiMAX PCM은 Prime ECC 고정 QC와 다름 |
| PCM 접근용 ControlSignal/ROM 라우팅 | 대응 없음 | FPGA 특화 제어 로직, Prime ECC 모듈과 직접 대응 없음 |

적용 가치: **낮음** - block-circulant QC-LDPC와 λ-min은 Prime ECC가 이미 보유한 QC 구조·min-sum계와 중복되며, WiMAX 저throughput 학술 FPGA 테스트베드라 NAND ECC로 떼어올 신규 기법이 없음.
