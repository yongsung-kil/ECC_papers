> 총평: bi-diagonal(저인코딩복잡도) binary QC-LDPC의 부호설계 조합최적화 기법으로 소프트 BP FER/BER를 개선하며, 부호구성 절차 자체는 Prime ECC H-matrix 설계에 참고 가치가 있으나 무선·단블록 타깃이라 고rate NAND용 재검증 부담이 커 이식성 중.

### B. 알고리즘 요약

1. 대상은 tailbiting(TB) 부모 convolutional 부호로 표현되는 binary QC-LDPC block code `(Mc, Mb)`이며, polynomial PCM `H(D)`의 각 원소는 monomial `D^wij`(degree matrix `W`)로 표기된다.
2. 문제: 유한 길이(short/moderate)에서 점근 최적 degree distribution이 최적이 아니며, 무선용으로 turbo 대비 저rate·단블록에서 열세라 더 좋은 QC-LDPC가 필요하다.
3. 모델: BPSK+AWGN(SNR/bit)에서 `H* = argmin FER(H,SNR)`을 목표로 하되 인코딩/디코딩 복잡도 제약을 둔다.
4. 핵심 기법: 탐색을 (1) base matrix `B` + column degree distribution(CDD) `λs` 선택, (2) 선택된 `B`의 최적 labeling(degree matrix `W`) 선택의 두 독립 단계로 분리한다.
5. 저인코딩 복잡도를 위해 `H(D) = [Hbd(D) | h0(D) | H(c-b)×b(D)]` 형태로 parity부를 bi-diagonal + 3-nonzero column `h0`로 고정 → `n2 ≥ c-b-1` 등 CDD 일부 고정.
6. base matrix 선택 기준: 열 간 공통위치 `C1`, 3-중복 triple `C2`, 부분그래프 girth `C3`, 최단사이클 참여열 가중치합(ACE 유사) `C4`, 최소 ACE 사이클 다중도 `C5`를 순차 적용.
7. degree matrix `W`(labeling) 선택 기준: Tanner graph girth `g ≥ gt`, sliding-window girth `gw ≥ gwt`를 만족하도록 열별 무작위 원소 `{1..M}` 선택.
8. 최적화 절차: CDD → base matrix → column → labeling을 중첩 루프(Fig.1)로 탐색, 후보 `W` 약 1000개에 대해 목표 SNR FER를 몬테카를로 추정해 최종 승자 선택.
9. 결과: `R=1/2,3/4`, `n=576/2304`, `c=24/32/48`에서 50-iter BP로 WiMAX 802.16 QC-LDPC 대비 FER/BER 유의 개선, base 16×32 부호가 [5] 및 turbo 대비 우세(짧은 길이·저rate에서 이득 큼). 최적 labeling은 girth profile이 같으면 FER 분산이 거의 없음을 histogram으로 확인.
10. 한계: HW 미설계, 시뮬(AWGN)뿐이며 gate count/throughput 없음. 무선(WiMAX) 단블록·중저rate 타깃.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| base matrix + labeling(monomial shift) QC-LDPC PCM 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC의 base+circulant shift(`PCM_b`) 구조와 개념 정합, H-matrix 설계 참고 |
| bi-diagonal parity부(저인코딩 복잡도) | `encoder.cpp` `Generate_PCM_encoding()`, `LDPC_encoder_4KB()` | Prime ECC 인코더가 dual-diagonal이라 bi-diagonal 구성 아이디어와 유사 |
| girth/girth-profile/sliding-window girth 기반 사이클 회피 | 대응 없음 | Prime ECC 프로파일에 부호설계 사이클 최적화 모듈 명시 없음 |
| FER/BER 몬테카를로 평가 (BP 50-iter) | `channel.cpp`(AWGN), `decoder.cpp` `LDPC_Decoder()` | 성능 평가 파이프라인은 대응되나 알고리즘 자체는 부호설계 |

적용 가치: 중간 — bi-diagonal binary QC-LDPC의 base+labeling 조합최적화 절차는 Prime ECC의 H-matrix 설계(`Load_PCM`)·인코더(dual-diagonal)에 개념적으로 이식 가능하나, 무선 단블록·중저rate 타깃이라 고rate NAND ECC용 재탐색·재검증 부담이 크다.
