### B. 알고리즘 요약

1. 대상: 45nm 4b/cell ONO CT Flash를 6b/cell(23-level)로 부스트, page size 1KB, R=0.875 QC-LDPC ECC 사용.
2. 문제: 6b/cell은 인접 분포 간격이 좁아 RBER이 커지고, program disturb·retention·온도로 VTH 분포가 drift하여 sensing level 고정 시 정정 실패.
3. 모델: ED(error-detection) scheme으로 i번째 sensing level `VREF_i` 아래 셀 수 `Ni`를 저장, read 시 `Ni,measured`와 비교해 drift 방향·sub-optimal level 추정.
4. 핵심기법1(DI-SS 엔진): moving-read로 sub-optimal sensing level을 찾고, 그 근처 2회 추가 sensing으로 soft-2bit 정보 획득(직전 결과를 buffer에 갱신해 실제 1회 추가 sensing만 필요).
5. 의미: 정확한 LLR(비균일 양자화) 확보가 LDPC 성능·구현비용을 좌우하므로 drift 보정된 sensing이 핵심.
6. 핵심기법2(확장): 필요 시 추가 2회 sensing으로 soft-3bit 제공, Read-Retry로 RTN 기반 error pattern 변화를 이용해 정정 회피.
7. 부호레벨 트릭: 새 23-level 1-3-3 mapping으로 high/middle/low page RBER 균형 → low page RBER 20%↓, worst-case random read latency 25%↓.
8. 디코더 HW: shuffle decoding으로 수렴 2배 가속, CNU 1172개 + VNU 293개, 90nm에서 935K gate, 200MHz, 평균 8 iteration.
9. 결과: soft-3bit R=0.875 LDPC가 10년 retention까지 전 패턴 정정(R=0.85 BCH+ED는 초기부터 실패); DI-SS 적용 시 평균 iteration 27% 감소; soft-2bit로 80%+ 패턴 커버.
10. 한계: ISSCC 다이제스트 수준이라 부호 구성·shuffle 디코더·LLR 양자화 테이블 상세 부재; 특정 CT Flash 공정 타깃, NAND 일반화는 저자도 "믿는다" 수준 언급.
