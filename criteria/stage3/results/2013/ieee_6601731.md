> 총평: NAND에 직접 타깃한 실측 논문으로 read cycle을 대폭 줄이는 테이블 기반 LLR 생성이 채널/LLR 모듈에 이식 여지가 있으나, LDPC 복호 알고리즘·HW 자체는 손대지 않아 이식은 채널 계층에 국한된다.

### B. 알고리즘 요약

1. 시스템: sub-20nm MLC(2-bit/cell) NAND SSD, LDPC ECC codeword length `8100 Byte`, code rate `9/10`(user 7290B) 및 `2/3`(user 5400B), 채널은 NAND retention/program-disturb 실측.
2. 문제: soft-decision LDPC는 정밀 LLR을 위해 최대 21개 read(`V_ref`)가 필요해 read time이 과도(예: 2b/cell 21회≈1050us) - 신뢰성은 높지만 read latency가 수용 불가.
3. 모델: BER를 W/E cycle `N_W/E`, retention time `t_DR`, 이웃셀(inter-cell coupling) 데이터의 함수로 보고, 출하 전 측정으로 만든 prerecorded 테이블(`t_DR`, `N_W/E`, EP 테이블)로 표현.
4. 핵심기법(EP-LDPC): 하드리드 3회(BCH와 동일 read cycle)만 수행 후, lower page의 "1" 개수 `N_1_read`와 테이블로 BER를 예측하고 이 예측 BER에서 각 비트 LLR을 산출(8-step read flow).
5. 식 의미: `N_1_read`가 프로그램 시점 `N_1_write` 대비 감소/증가하는지로 program-disturb vs data-retention error mode를 판별(식3), 이를 통해 read 반복 없이 `V_th` shift 방향을 추정.
6. 핵심기법 2(inter-cell): EP 테이블이 상/하/좌/우 이웃셀 데이터별 target-cell error 수를 저장해 CER를 이웃 의존 가중(식9,10)으로 보정 - 이웃 4셀 고려가 좌우 2셀만보다 신뢰성 우수.
7. 구현 디테일: EP 테이블 크기 약 195KB(512GB SSD 대비 무시가능), `N_1_write`는 3중화 저장, LDPC codec 아키텍처는 기존과 동일(추가 로직은 NAND controller 내 단순 게이트).
8. 학습/최적화: EP 테이블은 출하 전 write/erase stress + 실제 read error 카운트로 training(Fig.11c), 칩 1개 측정 테이블로도 변이 내성 확보.
9. 결과(실측): sequential read cycle 1/7, acceptable retention time 최대 11배(rate 2/3)·2.8배(rate 9/10), acceptable BER 3.7배, 추가로 PDRP/DRRP 에러복구 펄스가 program-disturb·retention error를 76%·56% 저감.
10. 한계: LDPC 복호/HW 성능은 논문 범위 밖(codec 불변), 기여가 LLR 생성(테이블·채널모델)에 국한되고 EP 테이블은 칩별 측정·retention time 추정에 의존(칩간 20% 편차).
