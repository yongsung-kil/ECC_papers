### B. 알고리즘 요약

1. 대상은 다중표준(IEEE 802.16e/802.11n) **QC-LDPC** 디코더로, base matrix `Hb`(Nb=24열)를 확장인자 `Z=N/Nb`로 펼친 H를 GPU에서 복호(rate 1/2, length 576~2304).
2. 문제: ASIC은 유연성이 없고 CPU는 느리며, 기존 GPU 구현은 대부분 TPMP 스케줄이라 수렴이 느리다.
3. 채널 모델: BPSK 변조 + AWGN, 수신열 `y`, VC 메시지 `Li,j`·CV 메시지 `Ri,j`.
4. 핵심 기법 **Min-Sum 기반 column-layered(CL) 디코딩**(Algorithm 1): VN을 layer로 나눠 순차 처리, `Ri,jl = sign(...)·min_{j'∈Vi\j}|Li,j'|`, VN 갱신 `Li,jl = α·Σ Ri',jl + yj`.
5. CL은 같은 iteration 내 갱신값을 즉시 반영해 TPMP 대비 약 2배 빠른 수렴(5 iter ≈ TPMP 10 iter).
6. Ri,j sign/magnitude를 `mi`(sign,min,secmin 3변수)의 two-step(step-a 제거·step-b 정렬) 갱신으로 단순화 [6].
7. 병렬화: sub-matrix 내 edge가 같은 행에 없음을 이용, H를 Nb layer(각 Z열)로 분할해 layer 내 Z개 column을 block.x로 병렬, grid.x로 여러 codeword 동시 복호.
8. GPU 최적화 5종: 압축행렬(HVN/HCN/RowNum/ColNum)+texture memory, sign/min/secmin을 shared memory·register 배치, 8bit codeword 4개를 32bit로 packing, 2D thread configuration(NY=min(NBR,NBS)/NBB), 8-stream async 전송(memcpy 33.6%→6.0%).
9. 결과: GTX580에서 peak throughput **712Mbps**(CPU 2.6Mbps 대비 ~240~273배, 기존 최속 GPU 대비 3.0배), 8bit 고정소수점이 부동소수점과 근접한 BER, ASIC과 비견.
10. 한계: HW(RTL) 미설계·GPU 실측만, 최적화가 CUDA 아키텍처 종속, 부호정정능력 개선이 아니라 처리량/구현 최적화.
