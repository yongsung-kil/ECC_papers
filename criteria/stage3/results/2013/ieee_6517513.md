### B. 알고리즘 요약

1. 대상: 병렬 turbo/LDPC 디코더 HW에서 여러 processing element(PE)가 같은 메모리 뱅크에 동시 접근할 때 생기는 memory conflict 문제.
2. 문제: interleaver로 데이터가 섞여 병렬화 시 충돌 발생 → latency·HW 비용 증가; 기존 heuristic(simulated annealing 등)은 종료시점 예측 불가·비다항.
3. 모델: 자연순서/인터리브순서 data access 행렬(row=PE, col=time)을 정의; LDPC는 VN degree만큼 데이터가 2회 초과 접근되어 turbo보다 복잡.
4. 핵심기법: access 행렬을 tripartite graph로 만든 뒤 data node를 합쳐 `Δ`-regular bipartite graph로 변환, **bipartite edge coloring**으로 각 색=메모리뱅크 할당.
5. 의미: regular bipartite graph의 최소 edge coloring은 색수=최대차수 `Δ`이며 항상 존재 → 최소 뱅크수로 충돌없는 매핑이 항상 가능함을 증명(Lemma 1/2).
6. 확장: LDPC는 read/write time node를 분리한 "Double Memory Mapping"으로, 마지막 write 뱅크=첫 read 뱅크 제약을 만족시킴.
7. 알고리즘 디테일: Euler partitioning(Gabow)으로 `2k`-regular를 두 `k`-regular로 분할, 홀수차수는 perfect matching(Schrijver) 먼저 제거하는 divide-and-conquer.
8. 복잡도: turbo `O(...)`, LDPC 매핑을 다항시간에 계산(본문 지수 손상으로 정확식 미상).
9. 결과: HSPA turbo에서 CPU 매핑시간 최대 43배, random Gallager LDPC(serial/부분병렬)에서 최대 1950배 단축; 결과 HW 면적은 기존[25][27][28]과 동일(90nm, NAND-gate 등가).
10. 한계: 부호설계·복호성능과 무관; 무선표준용 flexible 디코더 대상이며 error correction 성능 지표 없음.
