### B. 알고리즘 요약

1. 대상: `(7807,7177)` QC-LDPC(rate≈0.92, `L=211, r=dc=37, dv=3`), BSC(cross-over `α`), 15 iteration, flash memory·광통신 초저에러율 응용 지향.
2. 문제: BP/min-sum은 trapping set로 error floor 발생, 기존 저감법은 다중 시도+고비용 postprocessing 또는 복잡한 VN 처리를 요구.
3. 핵심 기법: FAID - 메시지를 `2s+1`개 레벨 알파벳(7-level=3bit)으로 제한, VNU가 채널값 `y`와 c-to-v 메시지로부터 사전정의 Boolean map `Φv`(Table I)로 v-to-c 산출.
4. 식 의미: min-sum VNU는 `Φv=y+Σmj`(식1)로 5-bit 가산기+saturation 필요하나, FAID의 대칭 Boolean map은 Karnaugh 축약으로 단순 게이트만 사용 → 면적↓.
5. CNU: FAID/min-sum 공유하는 min 연산(식2), Min1·Min2·부호곱 S·Min1 인덱스 I 4개만 저장.
6. HW 트릭1: bit-serial CNU([10] 기반) - MSB부터 입력, flag/OR/AND로 Min1을 `w-1` 클럭에, 2라운드로 Min2를 정확히 계산(min-sum [10]의 Min2=Min1+1 근사가 error floor 유발하는 것을 회피).
7. HW 트릭2: unique zero detector(UZD)로 Min1 중복 여부 판별, VNU를 시분할(time-multiplexed)로 적게 두어 CNU 속도에 정합 → VNU 유휴 제거.
8. 스케줄링: 2 데이터블록 인터리빙, 열을 `2w-1` 그룹으로 나눠 barrel-shift permutation, `L·t/(2w-1)` VNU만 사용.
9. 결과(180nm 합성): FAID 36.1mm²(min-sum [10] 69.3mm² 대비 52%), 동일 throughput 33.3Gbps, 동일 latency 180클럭, error-floor 영역에서 5-bit min-sum·float BP보다 우수.
10. 한계: BSC/dv=3 regular 전제, iteration 감소 아님, 다중 FAID map 복잡도 저감은 향후과제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FAID Boolean map VNU / 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | VNU 처리를 map 기반 3-bit로 대체하는 이식 지점, LLR 테이블 방식과 상충 검토 필요 |
| bit-serial CNU(Min1/Min2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 우리 min-sum CNU와 동일 min1/min2 연산이라 정합 양호 |
| iteration별 알파벳 레벨 매핑 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블을 FAID map으로 치환/병행 가능성 |
| 인터리빙·permutation(barrel shift) 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | HW 유틸 스케줄이나 z=32 파이프라인(HCU/GT)과 정합 필요 |
| Boolean map edge/MUX 축약 | `GT.cpp` `Make_GT_HW()` | permutation MUX 개념 근접(직접 대응은 아님) |

적용 가치: **중간** — Boolean-map VNU로 3-bit에서 error-floor를 개선하며 면적을 반감시키는 아이디어는 Prime ECC의 min-sum VNU/LLR 테이블 대체 후보로 가치가 있으나, BSC/dv=3 전제와 부호별 map 재최적화·soft-read 확장 부담이 있다.
