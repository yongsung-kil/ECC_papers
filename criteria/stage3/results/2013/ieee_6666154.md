### B. 알고리즘 요약

1. 부호는 binary LDPC(길이 `1944` rate-1/2 WiFi 802.11), rate-3/4 length-1296 punctured code도 대상; AWGN 채널.
2. 문제: RBP는 초기 수렴은 빠르나 몇 iteration 후 개선이 멈춰 수렴 error-rate가 flooding/LBP보다 나쁘고, punctured code는 LBP에서 수렴이 느리다.
3. 핵심 기법 T-RBP: 두 residual을 결합 — relative message residual `r(vj)` (식(5), 이전·현재 iteration의 posterior LLR)로 VN 선택 후, 그 이웃 CN마다 C2V residual `rC2V` (식(3), 현재·다음 iteration C2V 차)로 최대 edge를 갱신.
4. 의미: 세 연속 iteration(이전/현재/다음) 정보를 함께 써서 가장 중요한 메시지를 더 정밀하게 지목.
5. pre-computation은 식(2) tanh 대신 min-sum 근사 식(4) `sgn·min`으로 생성해 residual 계산 복잡도를 낮춤.
6. 확장 S-RBP: 전체 그래프가 아닌 단일 CN 내 C2V residual만 비교해 갱신, 비교 횟수를 RBP의 약 `1/M`로 축소.
7. 두 알고리즘 모두 상단 CN에서 시작해 최대 residual edge 갱신 후 이웃 V2C 전파, stopping rule 미충족 시 반복.
8. 복잡도 표(Table I): T-RBP 연산량 = EDS-LBP + S-RBP 합, S-RBP는 residual generation/comparison이 최소.
9. 결과(시뮬): T-RBP가 모든 기존 IDS 대비 최고 FER, 특히 고SNR에서 이득 확대; S-RBP는 NW-RBP 수준 성능을 훨씬 낮은 복잡도로 달성. punctured code 수렴속도·FER 크게 개선.
10. 한계: HW 미설계(시뮬만), full BP(tanh/atanh) 순차 IDS 스케줄, gate count/throughput 미보고.
