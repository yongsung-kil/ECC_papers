### B. 알고리즘 요약

1. 시스템: rate 1/2, `252 × 504` parity-check matrix(PEG 구성) LDPC 코드를 대상으로 한 디코더 구현 연구.
2. 문제: LDPC 반복 디코딩 복잡도가 코드 크기에 따라 커져 병렬 구현이 어려운데, FPGA/GPU/SDR은 설계시간·HW크기 비용이 큼.
3. 플랫폼: 오픈소스 MPSoC 프레임워크 `HeMPS`(HERMES NoC + Plasma MIPS급 프로세서 2D mesh, DMA, message passing)를 활용.
4. 디코딩 알고리즘: `Reduced Minimum Sum Algorithm (RMSA)` — min-sum을 재배열해 메모리 footprint를 줄인 변형(성능은 다소 저하). LLR 도메인에서 동작.
5. CN 업데이트: `Λ(c→v) = ∏ sign(·) × min|Λ(v)−Λ(c→v)|` (식 2), VN 업데이트 `Λv = Λ(0) + Σ Λ(c→v)` (식 3).
6. 매핑: master 프로세서가 초기화·VN·판정(step 1,3,4)과 scatter/gather 담당, slave들이 check node 그룹을 분담(star형). CN 계산이 서로 독립임을 이용.
7. 구현 디테일: PE는 100MHz Plasma IP, 페이지 16KB, 메시지 128byte 제한, 최대 30 iteration 고정. LLR은 memory page에 하드코딩.
8. 실험: 순차 1PE 대비 3~10PE(1 master + slave) 7개 시나리오, SystemC 시뮬. 비교용으로 데스크톱 MPI 구현도 측정.
9. 결과: HeMPS 5 프로세서에서 speedup 최대 1.25배(throughput 4.03kbps), 이후 통신비용으로 성능 저하. MPI는 8PE에서 2.09배.
10. 한계: 순수 SW 시뮬(HW 미설계), throughput kbps급, 부호설계·양자화·error-floor 기여 없음, soft-read/NAND 무관.
