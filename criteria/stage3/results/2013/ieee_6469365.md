### B. 알고리즘 요약

1. 시스템: 16-QAM BICM-ID(반복 검출·복호) 프론트엔드와 결합된 terminated convolutional(=spatially coupled) protograph LDPC 코드, rate `R_L = 1-((L+m_s)/L)(M/N)`.
2. 문제: 전통적 LDPC 설계는 검출기(detector)의 EXIT 특성에 맞춰 최적화해야 하며, 검출기 특성이 변하면(채널/매핑 변경) 성능이 급락함.
3. 가정: LLR은 AWGN 채널 후 Gaussian 분포를 따르며 평균 `μ`만으로 추적(Gaussian Approximation, `μ=σ²/2`).
4. 핵심 기법: Protograph 기반 spatially coupled 코드(`B_L` 행렬, replication factor `L`)를 검출기 EXIT 함수 `f_D`와 결합한 density evolution 확장.
5. 핵심 식: check node update `μ_{vj←ci} = φ⁻¹(1-∏(1-φ(μ)))`, variable node update는 검출기 출력 `μ_{D→vj}`를 매 iteration마다 반영.
6. 확장: 비대칭 protograph 구조(예: `B_r^(2)=[3 2 3 2 | 1 2 1 2]`)를 쓰면 termination이 좌우 비대칭이 되어 "decoding wave"가 한 방향으로만 전파, 매핑 무관 universal 성능 확보.
7. 안정조건 보완: `λ(0)·ρ(1) < exp[J⁻¹(f_D(1))]²/8`을 최악 검출기(flat EXIT) 가정으로 일반화(식 3)해 degree-2 VN 비율 제한.
8. 최적화 절차: 별도 학습 없음, 코드는 수작업 protograph 설계 + Gaussian Approx DE로 threshold 계산.
9. 결과: Code C2(비대칭 protograph)는 Gray/SP/M16a 매핑 간 threshold 차이가 0.03~0.17dB로 거의 없음(Code C1은 최대 0.56dB 차이); 단 필요 iteration 수는 L=50에서 최대 ~1500회로 급증.
10. 한계: HW 미설계, 순수 시뮬/DE 검증뿐, AWGN+16-QAM 특화, iteration 수가 실용적이지 않아 스케줄링 최적화가 별도 필요(본 논문 범위 밖).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Protograph 기반 spatially coupled 부호 구성 | 대응 없음 | Prime ECC는 고정 QC-LDPC(`ecc_top.cpp` `Load_PCM()`) 구조이며 convolutional/spatially coupled 부호는 미보유 |
| BICM-ID 반복 검출-복호 결합 | 대응 없음 | NAND 채널은 변조/검출 프론트엔드가 없어 해당 없음 |
| Gaussian Approximation Density Evolution | 대응 없음 | Prime ECC는 시뮬레이터로 DE 기반 부호 설계 도구를 포함하지 않음 |

적용 가치: **낮음** — 무선 변조/검출 결합 이론(BICM-ID, EXIT/DE) 중심이며 NAND binary QC-LDPC의 부호설계·디코더·HW 어느 레이어에도 대응되는 요소가 없음.
