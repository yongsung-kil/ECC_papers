### B. 알고리즘 요약

1. 시스템: AWGN wire-tap 채널(Alice→Bob 정상, Eve 도청), 길이 `n=1024`, rate≈3/4 LDPC 5종 비교.
2. 문제: LDPC를 systematic으로 쓰면 security gap이 매우 큼(~18dB), 이를 줄이는 것이 목표.
3. 지표: security gap `Sg = (Eb/N0)_B - (Eb/N0)_E` [dB], Bob은 `PeB≤10^-5`, Eve는 `PeE≥0.4` 조건.
4. 핵심 기법: 구조적 LDPC 구성법 4종(QC-LDPC, M-SC-MPC, interleaved product, PDC-LDPC)을 PEG 벤치마크와 비교.
5. QC-LDPC: 식 (3)의 단일 circulant 블록 행 `H=[H0|...|Hn0-1]`, 여기선 4개의 256x256 circulant, row/col weight 5, random difference family로 short cycle 제거.
6. M-SC-MPC: M개 단순 MPC 성분코드 직렬연접(여기 M=5, r1..r5=45,47,52,53,59), lower-triangular PCM으로 인코딩 매우 단순.
7. 부수 트릭: scrambling(dense kxk 행렬)으로 Eve 오류를 프레임 전체에 확산 → security gap 축소. L개 프레임 연접 scrambling으로 추가 개선.
8. 복호: LLR-SPA(floating-point) BP 디코딩만 사용, HW/양자화 없음.
9. 결과: systematic 시 Sg≈17~18dB, perfect scrambling 시 PEG=1.62dB / M-SC-MPC=1.63dB, L=3 시 PEG=M-SC-MPC=1.46dB. M-SC-MPC가 PEG와 대등하거나 더 좋음.
10. 한계: 시뮬만, HW 미설계, 목적이 물리계층 보안이라 NAND ECC 정정능력·error-floor·throughput과 무관.
