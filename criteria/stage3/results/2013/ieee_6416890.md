### B. 알고리즘 요약

1. 시스템: WiMAX 802.16e irregular QC-LDPC (base matrix `24(1-R)×24`, rate 1/2·2/3A, length 2304), BAWGN 채널.
2. 문제: 멀티미디어 전송의 패킷화·헤더에 존재하는 redundancy를 물리계층 FEC가 활용하지 못함 → BER/threshold 개선 여지.
3. 핵심가정: MPEG null packet(188B, 대부분 0x00), service information 등 "deterministic bits"가 송수신단에 예측가능하게 존재.
4. 핵심기법1: deterministic bit는 해당 variable node에 무한신뢰(`V = ±∞`) 부여 = H에서 그 column 제거(shortening) → `(n,k)`가 `(n-p,k-p)` sub-code로 됨.
5. 핵심식: check node `U_ij = 2tanh⁻¹(∏ tanh(V/2))`, bit node `V_ij = U_ch + ΣU`; deterministic 시 `V=±∞`, 실제로는 column 삭제.
6. 핵심기법2: 재귀적 shortening `H_{p+q}=(H_p)_q` — 한 번에 한 column씩 제거해 mother matrix 체인 생성, 가변 길이 deterministic bit에 정합.
7. 구현 디테일: 송신단에서 controllable bit을 frame/FEC buffer에 재배치(repositioning)해 shortening 이득 최대화; QC 성질 유지 위해 base matrix 레벨에서 shortening.
8. 최적화: 각 column의 Eb/N0 threshold를 EXIT-chart로 계산 → 가장 낮은 것부터 제거(Algorithm 1), 복잡도 `O(p(2n-p+1)/2)`로 brute-force `O(n!/p!)`보다 낮음.
9. 결과: 4 column 삭제 시 recursive shortening이 brute-force에 근접, MES·random shortening보다 낮은 FER/threshold (rate 1/2·2/3A length 2304 시뮬).
10. 한계: HW 미설계·시뮬만, 이득이 WiMAX 상위계층 redundancy에 의존, 상위계층 피드백 반복으로 지연 발생.
