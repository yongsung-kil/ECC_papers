### B. 알고리즘 요약

1. 대상은 `J×K` 원형행렬(`e×e` circulant) 배열로 주어진 대수적 QC-LDPC(예: (992,750), (4032,3304)), BPSK/AWGN 채널.
2. 문제: MS/layered-MS 디코더는 전체 `Hqc,perm`을 써서 CN unit·wire·메모리가 커진다.
3. 행/열 치환 `πrow, πcol`으로 `Hqc`를 블록순환 구조 `Hqc,perm`(식 (1))으로 바꾼다 — 각 row block은 위 block을 `K` 우측순환한 것.
4. 핵심: 전체 대신 첫 부분행렬 `Hqc,perm,0`(top `l` row block, `Jl×n`) **하나만** 사용해 revolving(순환) 방식으로 복호.
5. 매 iteration 후 reliability vector를 좌측 `lK` 순환 이동 → `Hqc,perm,0`만으로 `Hqc,perm,i` 기반 iteration과 등가.
6. Revolving MS(RMS): 식 (3)/(4)로 `Fj = Rj + λatten·Σ sign·min|R|`, VN은 모든 인접 CN에 동일 메시지 전달(VN unit 단순화).
7. 트릭: `f` iteration마다 채널정보 `y`를 재주입(식 (4))해 오차성능 유지, hard-decision `z`의 syndrome=0이면 조기 종료.
8. 각 iteration 종료 시 CN 메시지 메모리 전부 해제 가능 → 메모리 추가 절감.
9. 결과: CN unit·wire를 MS 대비 32.26%, RN 덧셈 36%, 6bit 양자화 시 메모리 최대 89% 절감, 성능손실 0.1~0.12dB 이내.
10. 한계: HW 실설계 없음(복잡도는 분석적 카운트), regular 대수적 QC/cyclic에 효과적이나 irregular엔 성능 저하.
