### B. 알고리즘 요약

1. 시스템: BPSK/AWGN, `(N,K)(dv,dc)`-regular LDPC. 검증은 2D EG-LDPC (4095,3367)(Rc≈0.82)와 (1023,781)(Rc≈0.76).
2. 문제: 표준 WBF/MWBF/IMWBF는 iteration당 1비트만 flip해 느리고, GDBF/GRSBF는 flipping 임계 파라미터를 경험적 사전 튜닝해야 함.
3. 목표: iteration당 여러 비트를 flip하는 adaptive weighted multibit-flipping(AWMBF)로 사전 최적화 없이 ANI를 줄이고 성능 개선.
4. flipping function: IMWBF의 `E_n = Σ(2s_m−1)φ_{m,n} − α|y_n|` (식5) 신뢰도 프로파일 사용 (`φ_{m,n}=min_{i∈N(m)\n}|y_i|`).
5. LRP 정의: 신뢰도 프로파일 `E`에서 임계 `E_th` 이상인 위치를 least reliable position 집합 `LRP(E)={n | E_n ≥ E_th}` (식7)로 정의.
6. 적응 임계: `E_th = E_max(1 − Σs_m/M)` (식8) — 만족된 syndrome 비율이 높을수록 window를 넓혀 flip 대상을 늘림. 코드 rate별 실험 튜닝 불필요.
7. 병렬 flip: LRP에 속한 비트들을 한 iteration에서 동시에(병렬) 뒤집어 iteration당 다중 오류 정정 확률↑.
8. 절차: init(φ 계산)→syndrome 검사→E 프로파일(식5)→E_th(식8)→LRP flip(병렬)→k++ 반복(kmax까지).
9. 결과: (4095,3367)에서 BER=1e-5 기준 IMWBF 대비 0.9dB, GRSBF 대비 0.7dB gain(kmax 10 vs 50), SP와는 0.6dB 차. (1023,781)에서 ANI를 60% 이상 감소(Eb/N0 3.6dB 이상서 5 iter 미만).
10. 한계: hard-decision BF 계열이라 soft BP/MS와 계열 다름, EG-LDPC(finite geometry) 특화, HW 미설계, throughput/gate 정보 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| AWMBF 디코딩 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 우리는 soft min-sum message passing, BF 경로 자체가 없음 |
| flipping function/신뢰도 | 대응 없음 | Prime ECC에 WBF/신뢰도프로파일 계산 없음 |
| 적응 임계 E_th (syndrome 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 우리는 syndrome/CRC를 조기종료에만 사용, 임계 flip과 목적 다름 |
| 병렬 비트 flip | `GT.cpp` `Make_GT_HW()` | 둘 다 병렬화지만 GT는 edge MUX, BF 병렬 flip과 무관 |
| 입력 hard-decision z | `input.cpp`, `channel.cpp` | hard-1bit 입력은 지원하나 이후 처리 경로가 min-sum |

적용 가치: **낮음** — Prime ECC는 soft min-sum(HD/2SD/3SD message passing) 기반인데 본 논문은 별개 계열인 hard-decision multibit-flipping이고 EG-LDPC regular 코드 특화라, 고rate QC-LDPC NAND 디코더에 떼어 붙일 공통 모듈이 사실상 없다.
