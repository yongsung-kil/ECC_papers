> 총평: FPGA block RAM의 미사용 대역폭(약 78%)을 vectorization+folding으로 회수해 QC-LDPC NMS 디코더 throughput을 올리는 HW 논문. 메모리 병목 아이디어는 유효하나 FPGA 특유 구조 종속이고 정량결과·정정능력 개선이 없어 우리 ASIC 구조 이식가치는 낮음.

### B. 알고리즘 요약

1. 대상: `(3,6)`-regular **QC-LDPC** (rate 1/2), CPM 256x256 배열, FPGA(Xilinx Virtex-4) 구현.
2. 문제: LDPC 디코더 throughput은 **메모리 대역폭**에 제약; 통상 FPGA는 메시지 하나를 block RAM 워드 1개에 저장해 대역폭의 ~22%만 사용(~78% 낭비).
3. 모델: NMS(normalized min-sum) 메시지 6~8bit, block RAM은 aspect ratio(예 512x36, 1Kx18 …) 재구성 가능.
4. 핵심 기법1 **vectorization**: 한 메모리 워드에 K개 메시지 패킹(폭 활용) → `(γ,ρ)` 코드에 `Kγ` CNU, `Kρ` VNU, `(γ+1)×ρ` block RAM, throughput 최대 K배.
5. 부수 문제: 다중 메시지 동시접근으로 메모리 충돌 발생 → 적절한 packing scheme과 message alignment unit 필요.
6. 핵심 기법2 **folding**: block RAM의 남는 depth에 여러 CPM을 같은 물리 block RAM에 매핑(greedy heuristic)해 라우팅·CNU/VNU 복잡도 축소(단 다중 cycle 처리).
7. 직교성: vectorization(폭)과 folding(깊이)은 orthogonal → vectorized-folded 디코더로 결합 가능.
8. 구현 트릭: double buffering(CNU/VNU 접근패턴 맞춤 복제, 같은 block RAM 재사용), 새 packing strategy, 저복잡도 sequential alignment unit.
9. 결과: DN8000K10PSX(Virtex-4 2개) FPGA 플랫폼에서 AWGN all-zero codeword 디코딩 검증, 다중 표준용 유연 디코더 구현 가능 언급(정량 throughput/gate 수치는 본문 서술에 없음).
10. 한계: **FPGA block RAM 특성 종속**, 정량 성능수치·정정능력 개선 없음, 부호설계/디코더 알고리즘 자체는 표준 NMS 그대로.
