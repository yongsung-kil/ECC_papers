### B. 알고리즘 요약

1. 시스템: half-duplex 단일/다중 relay 채널, BI-AWGN, source-relay/source-destination 두 SNR에서 두 rate로 동시 동작하는 부호와 서브부호를 설계.
2. 문제: 기존 relay LDPC는 고정 채널조건에 painstaking 최적화되어 재사용·인코딩·rate적응이 어려움.
3. 핵심가정: decode-forward(DF) 프로토콜, `SN R_SR > SN R_SD`, 릴레이가 Layer2 비트를 k2 parity로 보호해 목적지가 제거 가능.
4. 핵심기법1(bilayer lengthened): rate-1/2 protograph(`H1/2`, 7 VN/4 CN)에 3 VN씩 추가해 `R=(n+1)/(n+2)` 고rate 부호족 생성, `H_SR1=[H_SD1 H_e]`.
5. 최소거리: rows 2-4 열합을 3 이상으로 유지해 최소거리의 선형 성장(error-floor 회피) 보장.
6. 핵심기법2(bilayer expurgated): rate-3/4 protograph에 CN을 추가해 `R=(13-4-n)/(13-1)` 저rate 부호 생성, degree-1 VN이 threshold 개선.
7. 구현디테일: parallel edge 최대 2로 제한해 탐색공간 축소, PEXIT를 cost function으로 threshold 최적화.
8. 리프팅: protograph를 PEG로 factor 4 리프팅해 parallel edge 제거 후, 2차 circulant 리프팅으로 목표 block length 도달.
9. 결과: rate>2/3에서 capacity gap 0.1dB 이내, WER 10^-6까지 error floor 미관측, 기존 최적화 부호와 0.1dB 이내 성능이면서 rate-compatible·저인코딩 이점.
10. 한계: 릴레이 채널 종단성능(α,β 슬라이스, 다중relay) 특화, HW 미설계·시뮬만, single-link NAND ECC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| bilayer protograph 부호설계 (H-matrix 구성) | `ecc_top.cpp` `Load_PCM()` | 부호구조 로드 지점은 있으나 릴레이 다층부호 구조와 정합 불가 |
| protograph 리프팅(PEG/circulant) → QC 구성 | 대응 없음 | Prime ECC는 특정 고정 QC-LDPC, 릴레이 nested 부호와 무관 |
| 표준 message-passing 복호 (max iter 200) | `decoder.cpp` `LDPC_Decoder()` | 논문은 알고리즘 개선 없이 표준 디코더 사용, 신규 기여 없음 |
