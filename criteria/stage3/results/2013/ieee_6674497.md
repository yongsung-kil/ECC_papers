### B. 알고리즘 요약

1. 대상: IEEE 802.11ad(60GHz WLAN), code rate 13/16, block length 672, 64-QAM 환경의 min-sum LDPC 디코딩.
2. 문제: 기존 부분/완전병렬 디코더는 반복적 데이터 교환으로 VN↔CN 양방향 배선이 겹쳐 라우팅 혼잡·낮은 area utilization(~50%)과 높은 latency 발생.
3. 핵심기법: 반복 디코딩 루프를 `unroll`하여 각 iteration마다 전용 HW를 인스턴스화하고 파이프라인화 → 클록당 1블록 처리.
4. 데이터 흐름이 단방향(iteration i → i+1)이 되어 반대방향 네트워크 중첩이 사라지고 배선이 단순화됨(structured routing).
5. min-sum: `sgn`은 부호 XOR(식3), magnitude는 `γ×min`(식4, ESF γ 스케일링), VN은 채널값+extrinsic 누산(식5).
6. 채널값은 모든 iteration에서 필요하므로 파이프라인 레지스터에 저장.
7. VN: degree 최대 4, adder tree로 병렬 누산 후 2C2SM 변환. CN: 16입력에서 min1/min2를 트리형 sort+CS4 유닛으로 탐색, 출력단은 MUX 3개로 단순화.
8. 4bit 양자화 채택(6bit 대비 BER 손실 있음), 9 iteration 언롤(그 이상 성능 개선 미미).
9. 결과: 65nm ASIC, 257MHz(WC), 12.09mm2, 160.8Gbit/s, 5.36W, area eff 13.6 Gbit/s/mm2 (기존 대비 area eff 1.3~8배).
10. 한계: two-phase 스케줄만 가능(layered 불가), 단일 rate/length 고정으로 flexibility 없음, 언롤로 면적 大.
