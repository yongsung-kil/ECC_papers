### B. 알고리즘 요약

1. 대상 부호: `dv=2` regular structured QC-LDPC, `H`는 circulant permutation matrix로 구성(product method, Fossorier [2]), girth-4 없음.
2. `H`는 `2p×np` 행렬로 `E^{s_{i,j}}` 블록 배열, `s_{i,j}=ij mod p` (`i=0,1`, `0≤j≤n-1`, `p`는 소수).
3. 풀려는 문제: random LDPC은 효율 인코딩이 없고, QC-LDPC도 generator matrix 별도 구성이 필요 → 이를 회피할 systematic form 제시.
4. rank 유도: 상단 p행을 하단에 modulo-2 가산해 `H_t`로 변환, 부분행렬 rank가 `p-1`이므로 `H`의 rank는 `2p-1` → parity-check bit 수 `= 2p-1`, rate `= 1-(2p-1)/(np)`.
5. 열 치환으로 `H`를 `H̃`로 재배열, 부호어를 `p×n` codeword matrix `C=(c_{k,j})`로 표현.
6. 두 제약: row constrain `Σ_j c_{k,j}=0 mod 2`, diagonal constrain `Σ_j c_{(k+j)p,j}=0 mod 2` → 2차원 single parity product code 형태.
7. parity-check bit 위치를 특정 열/대각으로 고정(마지막 열 + 특정 대각 원소)하면 systematic code가 됨.
8. Theorem 1: `k=p-n` 대각 제약은 나머지 제약 만족 시 자동 충족 → parity bit 수 `p+(n-1)+(p-n)=2p-1` 일치.
9. 인코딩: row/diagonal 제약을 교대로 적용해 XOR로 parity bit를 재귀적 직렬 계산, 복잡도는 parity bit 수에 선형.
10. 인코더를 `H̃`의 Tanner graph 위에서 구현 가능 → 디코더와 실리콘 면적 공유. HW 설계/시뮬/BER 결과는 없음(예제 p=5,n=3, length 15만 제시).
