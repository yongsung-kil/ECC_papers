### B. 알고리즘 요약

1. 대상: binary QC-LDPC 인코더, `H_b`는 `z×z` circulant permutation matrix `P^{a_ij}` 블록으로 구성(예제 IEEE 802.11n rate 5/6, `n=1944`, `z=81`).
2. 풀려는 문제: Richardson-Urbanke 일반 인코딩은 `φ=(E·T⁻¹·B+D)`의 역행렬이 non-sparse → 복잡도 높음.
3. B-LDPC(Myung et al. [4]) 인코딩 채택: `φ`를 identity 또는 circulant로 선택해 선형복잡도, `p1=φ⁻¹(E·T⁻¹·A·sᵀ+C·sᵀ)`, `p2`는 `T·p2ᵀ=A·sᵀ+B·p1ᵀ`로 back-substitution.
4. 핵심 기법: GF(2) 상수행렬 곱 `q=L·xᵀ`을 최적화 — `H_b`가 거의 불변이므로 상수로 간주.
5. circulant 곱을 z-bit cyclic shifter로 구현하는 대신, 신호를 올바른 LUT(XOR 연산자)에 직접 배선(hard-wire).
6. 결과적으로 cyclic-shifter, block-memory(ROM) 불필요 → gate 수·자원 감소, 최대 필요자원이 완전 예측가능.
7. FPGA 재구성성을 이용해 다양한 code rate/length는 재프로그래밍으로 지원(유연성 유지).
8. HW 검증: syndrome check `H·cᵀ=0`(Eq.3)으로 인코딩 회로 정당성 확인.
9. 결과(Virtex5 xc5vlx20t, 802.11n 5/6 n=1944): 290MHz, 4 clock cycles, throughput 117.45Gbps, 1782 LUTs, 2187 XOR, block-mem/barrel-shifter 0 → 기존 대비 고속·저면적.
10. 한계: 인코더만 다루고 복호/정정성능 논의 없음, LUT hard-wire는 FPGA 재구성 특화(ASIC 고정구조와 상이), 목표 응용은 long-haul optical.
