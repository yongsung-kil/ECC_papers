### B. 알고리즘 요약

1. 대상은 CMMB(중국 모바일 멀티미디어 방송) 표준의 LDPC 부호로, code rate `0.5`/`0.75`, codeword length `9216`, 정보어 길이 `4608`/`6912`.
2. 풀려는 문제: LDPC 인코딩에서 `A⁻¹`가 Gaussian elimination 후 dense해져 대규모 연산·저장이 필요한 문제를 회피.
3. 채널/noise 모델은 정의하지 않고 인코딩 연산량만 다루는 순수 인코더 논문.
4. 기존 Neal의 LU 분해(`A = L·U`)를 사용해 forward substitution으로 `z`, backward substitution으로 parity `c`를 구함(dense matrix 곱 회피).
5. 개선안: 행 치환 `P`와 열 치환 `Q`를 한 분해 과정에서 동시 적용(`P·A·Q = L·U`)해 `L`, `U`를 더 희소화.
6. 각 단계에서 행가중치×열가중치 곱 `D1`(대각 후보 열)과 `D2`(대각 후보 행)를 비교, 더 작은 쪽으로 행 또는 열 치환을 선택해 Gaussian elimination이 채우는 1의 수를 최소화.
7. HW 친화 근거: CMMB는 인코딩 후 정보어·parity를 interleave하므로 열 치환이 이미 재정렬 패턴에 흡수되어 추가 저장·연산 비용이 없다고 주장.
8. `L`, `U`는 사전 컴퓨터 전처리로 메모리에 저장, 인코더는 읽기만 하므로 분해에 HW를 쓰지 않음.
9. 결과: classic LU 대비 forward/backward substitution의 총 반복 연산이 rate 0.5에서 23.2%, 0.75에서 24.4% 감소(Table I), 추가 열 치환·HW 비용 없음.
10. 한계: HW 설계·throughput·gatecount 없음, 시뮬 수준 연산량 비교뿐이며 CMMB 전용 비-QC HS 부호에 특화.
