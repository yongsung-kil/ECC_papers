### B. 알고리즘 요약

1. 대상은 BSC 상 column-weight-three(dv=3) binary LDPC 복호로, girth-8 QC-LDPC(2388,1793)rate0.751·(504,252)PEG rate0.5·(5184,4322)QC rate0.833 및 (155,64) Tanner 코드로 검증.
2. 문제: BP/quantized-BP는 loopy graph의 trapping set 때문에 error-floor가 발생, DE-threshold 기반 양자화 디코더는 finite-length·고SNR에서 최적 보장 없음.
3. 핵심: FAID `D=(M,Y,Φv,Φc)` — 메시지가 유한 알파벳 `M={±L1,±L2,±L3,0}`(7-level=3bit), LLR/확률의 양자화가 아님.
4. VNU 맵 `Φv(yi,m1,...,m_{dv-1})=Q(Σmj + ωi·yi)`, 여기서 `ωi=Ω(·)` (LT면 Ω=1, NLT면 비선형)로, 같은 메시지 합이라도 다른 출력 가능(BP·min-sum과 차별화). Φc는 min-sum과 동일한 sign·min 연산.
5. 핵심: Φv를 대칭 LUT(2D array `[li,j]`, symmetric plane partition과 전단사)로 표현, harmful subgraph에서 오류정정 능력(guaranteed error correction)을 최대화하도록 맵 설계.
6. 확장: noisy trapping set / noisy critical number vector(NCNV) — TS를 isolation assumption 하 2 iter 뒤 degree-1 check로 initialization vector Θ 주입해 이웃 영향을 근사, FAID의 오류정정력 평가.
7. 트릭: decoder domination(n˜)·cost function `Cn˜`으로 good/bad FAID 집합(Fg/Fb) 대비 후보 FAID 집합 Fc 선별, TS는 Trapping Set Ontology(TSO)에서 선택.
8. 학습/최적화: DE-threshold를 선별에 쓰지 않고 TSO 기반 semi-heuristic 선별(code-specific 아님), 7-level 후보 31개 도출(NI=6, b=2).
9. 결과: 3-bit FAID가 floating-point BP를 FER 10⁻⁵부터 능가, error-floor slope 개선; (155,64)에서 FAID는 5-error 전부 정정하나 BP는 일부 5-error 실패, BP보다 낮은 복잡도·메모리.
10. 한계: column-weight-3·BSC(hard-decision)·girth-8 한정, HW(RTL/합성) 미설계, waterfall은 BP 대비 소폭 손실.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FAID VNU 맵(Φv, LUT 기반) | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | VNU를 min-sum 대신 3bit LUT 맵으로 교체하는 지점 |
| iteration별 메시지 처리 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | 반복 message-passing 골격 재사용 |
| CN min-sum(Φc, sign·min) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Φc가 min-sum과 동일, 그대로 재사용 |
| 메시지 양자화(3bit 유한알파벳) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | LLR 테이블을 FAID 알파벳 LUT로 치환 검토 |
| BSC/hard-read 채널 | channel.cpp Set_R_Offset(), Set_LLR_Th() | hard-1bit(HD) 입력과 정합 |
```

적용 가치: 중간 — VNU 맵 교체 자체는 decoder.cpp에 이식 쉽고 error-floor·저복잡도 이득이 매력적이나, col-weight-3/BSC 전제라 Prime ECC의 고rate·VN degree 17 QC-LDPC에 맞춘 맵 재설계·재검증이 선행 필요.
