### B. 알고리즘 요약

1. QC-LDPC(base+circulant shift) 다표준(ITU G.hn, 802.11n WiFi, 802.16e WiMAX) 133코드, rate 0.5~5/6, 최대 (8640,4320) 대상 shuffled 디코더 ASIC.
2. 문제: shuffled MSA는 CN마다 3개 최소값을 저장해야 해 저rate 부호에서 CN 메모리(디코더 면적 지배)가 커져 다표준화에 불리.
3. 핵심기법1 DOMSA(Dynamic OMSA): C2V 크기별 gap이 커지는 특성을 반영, 임계 `β`로 V2C 값에 따라 두 보정계수 `δ1/δ2`를 동적 선택해 양자화비트 감소 없이 성능 유지.
4. 핵심식 (6),(7): C2V = min − (선택된 offset), offset은 threshold `β` 비교로 `δ1` 또는 `δ2` 결정 -> MSA-SPA magnitude gap 보정.
5. 핵심기법2 인공(artificial) 최소값: 실제 3rd min 대신 `2nd min + Δ`(offset)로 대체해 CN당 2개 최소값·인덱스만 저장, 부호 sign만 추가 저장.
6. 결과 BER: (8640,4320)에서 artificial 3rd min이 actual 3rd min과 거의 동일, 5-bit DOMSA+artificial이 6-bit OMSA+actual과 근접 -> 양자화비트·메모리 절감.
7. 핵심기법3 VN 그룹분할: 각 그룹이 각 block column에서 1열씩 취하도록 구성, block-wise 하향 순환시프트로 barrel shifter 없이 read/write 카운터만으로 CN 메모리 접근.
8. FIFO기반 CN 메모리: shift index 차이를 FIFO 길이로 매핑, offset 조정 알고리즘으로 FIFO 길이를 균일화해 pipeline hazard 방지·CNU 통일(G.hn 82->164 FIFO/CNU).
9. 구현: 90nm CMOS, 400MHz, 코어 5.529mm², 48 VNU/164 CNU 병렬, 정보 throughput 1.956 Gbps, single-port CV 메모리·two-port CN FIFO.
10. 한계: waterfall 수렴속도 개선(shuffled 2배)이나 error-floor/정정능력 개선 아님, length-1 FIFO 존재 시 BER 소폭 손실, 다표준 유연성 위주.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DOMSA 2보정계수 CN 연산 | decoder.cpp CNU_Update_New_Mag() | 우리 min-sum magnitude 테이블에 threshold별 2-offset 확장 가능 |
| VN 처리 / 양자화 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | artificial-min·양자화비트 절감이 VNU 양자화에 대응 |
| iteration별 LLR/offset 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | DOMSA offset 선택이 우리 iteration별 LLR 테이블 개념과 유사 |
| shuffled 그룹분할·CN메모리 라우팅 | decoder HW 모델 (z=32, HCU, GT) | z=32 row-parallel/GT와 구조 상이, FIFO 메모리 정합 필요 |
| H-matrix / QC 구조 로드 | ecc_top.cpp Load_PCM() | QC base+circulant 로드 대응 |
```

적용 가치: 중간 - DOMSA(2계수 offset)와 artificial-min의 CN 메모리·양자화 절감 아이디어는 우리 min-sum LLR 테이블에 부분 이식 검토 가치가 있으나, FIFO/shuffled 그룹분할 아키텍처는 z=32 row-parallel + GT 구조와 상이해 HW 재사용성은 낮음.
