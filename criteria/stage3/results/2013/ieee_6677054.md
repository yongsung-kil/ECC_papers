### B. 알고리즘 요약

1. 대상은 random-like LDPC 부호의 구성법으로, PEG 알고리즘 복잡도 `O(nm)`를 줄이는 PEG-CRT를 유연 길이로 확장한다.
2. 문제: PEG-CRT는 부호길이가 `n = n_b·∏L_p`(서로소 곱)로 고정되어 PEG처럼 임의 길이를 지원하지 못함.
3. 채널/디코딩 모델 명시 없음(BER 곡선은 BPSK AWGN 가정), 핵심은 부호 구성 자체.
4. 핵심 기법: 작은 base matrix `H_b`(m_b×n_b)를 PEG로 만들고 CRT로 `H`(m×n, m≫m_b)로 확장, 각 `b_ij=1`을 `L^2`개 원소로 `L`개 서로 다른 행·열에 배치.
5. CRT 인덱스 `I_t = (iB_L + Σ r_tp·Ã_p m_b L̄_p) mod m` (식 13)로 행·열 인덱스를 계산해 girth를 줄이지 않고 확장(Theorem 3.1: `g ≥ g_b`).
6. LC 확장(핵심 2단): base 행/열을 split(식 21,22)→CRT 확장→combine(식 23,24)하여 `n̄_b`가 `L_p`와 서로소가 아니어도 되는 유연 길이 지원.
7. 부수 트릭: remainder matrix `R`을 preprocessing에서 한 번만 만들고 permutation matrix `P_V, P_Q`를 곱해 재사용.
8. 이론 보증: split/combine이 girth를 줄이지 않고(Theorem 3.2 `g≥g_b`) degree distribution도 보존(Theorem 3.3).
9. 결과: rate-1/2, degree-3 및 최적화 degree분포 예제(길이 502→3000/7500), LC-PEG-CRT가 동일 길이 PEG 부호와 유사 BER을 훨씬 낮은 구성 복잡도로 달성.
10. 한계: HW 미설계·구성 복잡도만 정성 논의, 시뮬 BER만, random-like(QC 고정 아님)로 NAND circulant 구조와 상이.

### C. Prime ECC 관련 모듈 핀포인트

대상이 code-design이나 girth 보존이 error 성능에 간접 기여하므로 참고용으로 기재. 디코더 성능 항목은 부호구성만 다뤄 N/A.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG+CRT 부호 구성 / H-matrix 생성 | `ecc_top.cpp` `Load_PCM()` | 부호 구성법이나 Prime ECC는 고정 QC-LDPC(base+circulant shift)로 재설계·재검증 부담 큼 |
| girth·degree분포 보존 확장 | 대응 없음 | Prime ECC는 부호구조 고정, 온라인 구성 없음 |
| CRT 인덱스 기반 circulant 확장 | 대응 없음 | circulant shift(`PCM_b`)와 개념 유사하나 CRT 특유 서로소 제약이 z=32 lifting과 미정합 |

적용 가치: **낮음** — 유연 길이 random-like LDPC 구성 알고리즘으로, 고정 QC-LDPC(z=32, 4KB)를 쓰는 Prime ECC에는 부호 재설계·재검증 부담만 크고 직접 이득이 없다.
