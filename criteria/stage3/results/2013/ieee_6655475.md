### B. 알고리즘 요약

1. 대상 부호는 IEEE 1901 PLC 표준의 **LDPC-CC**(convolutional), rate 1/2~4/5, memory `M`=215~226, constraint length `CL`=432~1135.
2. 기존 BP(Brief Propagation) 복호는 수렴 느리고 변수노드당 intrinsic 1 + prior `Wr`개 저장으로 메모리 과다, OVA 스케줄도 여전히 메모리 큼.
3. 핵심 기법: LDPC-BC의 **layered decoding**을 LDPC-CC에 도입 — 각 check node를 1개 layer로 보고 갱신 결과를 즉시 다음 변수노드에 반영.
4. check node 연산은 **Normalized Min-Sum(NMS)** 사용, 정규화 계수 `α`=0.75(float/fixed 동일).
5. 스텝: 초기화 `Lv=Iv, Zc→v=0` → CNU `Lv→c=Lv_old−Zc→v_old`, `Zc→v=α·(sgn곱)·min|Ln→c|` → VNU `Lv_new=Lv→c+Zc→v_new` → 전 프로세서 전파 반복 → hard decision.
6. 변수노드당 **posterior 1개만 저장**(prior는 소수라 레지스터), extrinsic은 min1/min2 + min1 위치 + 부호들 + 부호곱만 저장해 메모리 대폭 축소.
7. 양자화: posterior/prior 8bit(1 sign, 4 int, 3 frac), extrinsic 6bit; extrinsic 저장 depth = 5+5+4+15+1 = 30bit.
8. VLSI: memory-based 아키텍처, 10 processor + 9 extrinsic memory block, 5-stage pipeline, dual-port RAM(check당 3read/3write).
9. 결과: layered가 절반 processor로 BP보다 우수한 BER; SMIC 130nm에서 333.3Mb/s@200MHz, core 3.55mm2, 262mW@rate4/5, 총 메모리 152,760bit.
10. 한계: convolutional code 전용 아키텍처, block QC-LDPC 아님; error-floor/정정능력 개선 주장 없음.
