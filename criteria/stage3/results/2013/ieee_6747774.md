### B. 알고리즘 요약

1. 대상: CMMB 표준 구조화 binary LDPC, 길이 9216, 두 rate(0.5: row6/col3, 0.75: row12/col3), systematic code.
2. 문제: 일반 선형블록 인코딩은 `O(N²)` 복잡도, LU 분해로 선형화해도 순차 iteration 때문에 풀파이프라인 불가 → throughput 병목.
3. 모델: `H=[A B]`, parity `pT=A⁻¹·B·sT`, 전치 pivoting(최소 행 무게)으로 `P·A=L·U` 분해(식 5), `pT=U⁻¹[L⁻¹(P·B)sT]`(식 6).
4. 핵심 기법: LU 분해는 Matlab 전처리, 인코더는 행렬-벡터곱 + forward/backward iteration(둘 다 L,U의 1개수만큼 XOR)만 수행.
5. 의미: forward iteration을 행렬곱 전부 끝나기 전에 버퍼시간 뒤 시작 가능 → 단계간 지연 은닉.
6. 확장(핵심): SIMD 구조 — 1개 SCU가 다중 CPU(Coding PU)에 같은 명령 broadcast, 서로 다른 데이터 벡터 병렬처리로 공간 병렬성 확보.
7. 구현 디테일: 희소행렬 B/L/U는 1의 행/열 위치+flag로 저장(MMC), RAM read/write collision 시 bypass register 우회 파이프라인 유지.
8. 최적화: full pipeline + 다단 Ping-Pong 버퍼, CPU 5개 구성, 100MHz 동작.
9. 결과: Altera Cyclone III EP3C120 합성, payload throughput 31.86Mbit/s(rate0.5)/52.37Mbit/s(rate0.75) — 참조 multi-pipeline(16.10/33.34) 대비 향상, LE 2228·저장 1,291,654bit.
10. 한계: CMMB는 QC-LDPC 아님(범용 인코딩만 가능), 방송용 저throughput(Mbit/s급), 복호성능·에러정정과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LU 분해 기반 parity 계산 | `encoder.cpp` `Generate_PCM_encoding()` | 낮음 — Prime ECC는 dual-diagonal 인코딩 고정, LU 방식 아님 |
| SIMD 병렬 인코딩 구조 | `encoder.cpp` `LDPC_encoder_4KB()` | 낮음 — non-QC 범용 인코더 병렬화, QC 구조와 정합 안 됨 |
| 희소행렬 저장/파이프라인 | 대응 없음 | Prime ECC는 QC circulant 기반, LU L/U 저장구조 불필요 |

적용 가치: **낮음** — CMMB(방송용) non-QC LDPC의 범용 LU 인코더를 SIMD로 병렬화한 HW 논문으로, QC-LDPC dual-diagonal 인코딩이 고정된 Prime ECC와 인코딩 원리·부호구조가 달라 이식 가치가 낮다.
