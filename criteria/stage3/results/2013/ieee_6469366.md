> 총평: UWB turbo equalization 환경에서 protograph LDPC convolutional code의 설계 파라미터(m_s, L) 영향을 EXIT/DE/BER 시뮬로 연구한 파라미터 스터디로, NAND binary QC-LDPC 이식 요소가 없음.

### B. 알고리즘 요약

1. 시스템: 단일 안테나 UWB 통신, 블록 페이딩 다중경로 채널(exponential PDP), turbo equalizer(soft-equalizer ↔ LDPC decoder 반복 교환) 구조.
2. 문제: AWGN에 최적화된 LDPC는 ISI가 커지면 성능 저하되므로, protograph 기반 LDPC convolutional code(PG-LDPCCC)의 설계 파라미터(m_s, L) 영향을 규명해야 함.
3. 가정: 채널은 CIR `h=[h(0)...h(T-1)]`인 block-fading 다중경로 + AWGN, exponential PDP `h(i)=(exp(-(i+1)/10))/10`.
4. 핵심 기법: base matrix `B`를 `m_s+1`개의 partial base matrix `B_i`로 분해해 무한 band matrix로 배열 후 termination length `L`에서 절단(`B[0,L-1]`), rate `R_CC=1-((L+m_s)/L)(1-R_P)`.
5. 핵심 식: EXIT 함수는 density evolution(protograph, pdf 전파)으로 계산, 상호정보량 `I(L(V);V)=1-E[log2(1+exp^-L(V))]`(식 6)로 a-priori/extrinsic MI 산출.
6. 확장: syndrome former memory `m_s` 증가 시 EXIT 곡선이 낮은 a-priori 구간에서 extrinsic 정보를 더 많이 생성(ISI 채널에 유리), `L` 증가는 상대적으로 영향이 작음.
7. 부수 트릭: protograph 양 끝단 변수노드는 연결수가 적어 더 강하게 보호되며 이 노드들이 반복복호의 "bootstrap" 역할(양끝→중앙 전파).
8. 최적화 절차: PEG(permutation P=100) 및 ACE+PEG cascade(P=1792) 알고리즘으로 lifting; 별도 학습(NN/GA) 없음.
9. 결과: ARJA convolutional code가 block code 대비 0.5~1.0dB 개선, RCPC(R=1/2) 대비 BER<1e-4 구간에서 우수; `m_s` 증가가 성능에 더 지배적, `L` 증가는 rate loss 감소하나 code length 증가.
10. 한계: HW 미설계, 순수 EXIT/DE + Monte-Carlo 시뮬 검증뿐, UWB ISI 채널 특화 파라미터 스터디로 정정능력 향상 자체를 제안하는 논문이 아님.
