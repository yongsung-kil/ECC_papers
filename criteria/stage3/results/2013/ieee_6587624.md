### B. 알고리즘 요약

1. 대상은 임의 BMS 채널(BSC, BI-AWGN)의 block/convolutional LDPC 복호로, moderate length(n=128~2560), (3,6)-regular 및 irregular(λ=1/6x+5/6x³, ρ=x⁵), IEEE 802.11 rate-1/2 포함.
2. 문제: finite-length LDPC는 그래프 loop 때문에 BP bitwise marginal이 열화되어 waterfall·error-floor 성능이 제한됨.
3. 핵심: EP(Expectation Propagation)로 BP를 일반화, tree-factorized family `Ftree`에서 true posterior와의 inclusive KL `D_KL(p(V|y)||q(V))` 최소화(moment matching).
4. 각 parity factor `Cj(V)`를 tree 구조를 공유하는 `Wj(V)`로 치환하고, 병렬 iteration으로 A) 보조함수 `fm(V)=Cm(V)q\m(V)` 계산 B) KL 최소화 marginal 매칭 C) factor 갱신을 반복.
5. `q(V)=Π qi(Vi|Vpi)`는 cycle-free Markov tree로, pairwise marginal이 loop의 상호정보를 포착해 BP가 무시한 정보를 회복.
6. 트리 구성(§III): BP가 stuck되면 llr이 0 근처 진동하는 변수와 unsatisfied check 연결 변수를 후보(VTS)로 모아, seed부터 loop 닫힐 때까지 짧은 chain을 이어붙임(평균 chain 길이 약 5, O(rmax)).
7. 구현 디테일: cycle-free graph는 BP처럼 O(rmax) 선형비용, cycle이 있으면 Pearl cutset conditioning으로 marginal 계산(O(rmax·2^{d-1}), d=loop 내 변수 수).
8. 학습/최적화 없음(순수 추론 알고리즘), word별로 다른 트리/포레스트를 동적 생성.
9. 결과: (3,6)-regular AWGN에서 BER=10⁻⁴ 근처 waterfall gain 0.2~0.3dB, error-floor는 약 1자릿수 개선; 전체 복잡도는 BP의 약 1.1~1.2배(n=256/128).
10. 한계: HW 미설계, moderate length 시뮬만, Pearl cutset은 최악 시 지수복잡도, GBP 대비 sparse graph 우위 주장은 이론적.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TEP 메시지패싱 iteration 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | BP 대체 추론이나 min-sum 아닌 KL/moment matching 기반이라 정합 어려움 |
| 트리/포레스트 동적 구성(word별) | 대응 없음 | 고정 파이프라인·GT 구조에 word별 가변 스케줄 없음 |
| Pearl cutset marginal 계산 | 대응 없음 | 지수복잡도 조건부 marginal, HW 모델 없음 |
| soft-info/likelihood p(yi|Vi) | channel.cpp Set_R_Offset(), Set_LLR_Th() | LLR 입력만 부분 관련 |
```

적용 가치: 낮음 — waterfall/error-floor 동시 개선은 매력적이나, word별 동적 트리 + Pearl cutset(지수복잡도)이 min-sum 기반 고정 HW 파이프라인과 근본적으로 상충.
