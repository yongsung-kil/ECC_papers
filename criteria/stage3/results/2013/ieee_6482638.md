### B. 알고리즘 요약

1. 대상: 임의 protograph(특히 one-sided spatially-coupled) 구조를 유지하는 binary QC-LDPC 부호설계. HW 친화(circulant) 유지.
2. 문제: waterfall 최적 protograph는 multi-edge를 가져 단순 lifting 시 inevitable short cycle(6/8-cycle)이 생겨 error-floor 유발.
3. 핵심 표현: HQC(hierarchical QC) LDPC - circulant의 circulant을 다변수 다항식 `H(x_1..x_L)`(식 (21))과 라벨 트리로 표현, 각 level이 graph cover에 대응.
4. cycle 판정: HQC의 길이-`2m` cycle 필요충분조건은 각 변수별 교대합 `sum ≡ 0 (mod z_j)`가 모든 level에서 동시 성립(Theorem 1, 식 (30)).
5. 핵심 기법 1: hill-climbing girth 최대화(Algorithm 1/2) - cost vector 기반 greedy 라벨 조정으로 non-inevitable cycle 제거. PEG/guess-and-test보다 짧은 length·빠른 시간.
6. 핵심 기법 2: squashing - HQC 계층구조를 접어 inevitable cycle을 제거, 계층성은 사라지나 원 protograph 대응 QC-LDPC 유지(계산 trivial).
7. 설계 절차: protograph → 2-level HQC lifting → hill-climbing(non-inevitable 제거) → squash(inevitable 제거) → 고girth QC-LDPC.
8. 결과 부호: girth-10 rate-0.45 len-8000, girth-10 rate len-24000, girth-8 rate-0.7 len-28000(모두 one-sided spatially-coupled protograph).
9. 성능: Gallager-B/BSC 시뮬에서 동일 protograph girth-6 대비 error-floor 대폭 감소(girth-10은 WER~10^-? 까지 floor 없음), var-degree 4 girth-8은 관측 floor 없음.
10. 한계: 부호설계뿐 HW 미설계, Gallager-B/BSC 시뮬만(min-sum/AWGN·gate count 없음), 고rate girth-10은 block length 과대(rate-0.8 girth-10은 ~수만 필요).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 고girth QC-LDPC 부호구조 생성(HQC+squashing) | `ecc_top.cpp` `Load_PCM()` | girth 재최적화로 error-floor 낮추는 부호설계 - Prime ECC는 H-matrix 고정이라 재검증 부담 큼 |
| girth→error-floor(trapping-set) 개선 | 대응 없음 | 정정능력/error-floor 개선을 부호구조로 달성, decoder 모듈 직접 대응 아님 |
| hill-climbing / squashing 알고리즘 | 대응 없음 | 오프라인 부호설계 도구, 런타임 디코더 코드와 무관 |
| Gallager-B 복호 시뮬 | 대응 없음 | Prime ECC는 min-sum, 논문 시뮬은 Gallager-B(BSC) |

적용 가치: 중간 — waterfall 유지+girth 최대화로 error-floor를 낮추는 부호설계 방법론은 Prime ECC 고rate QC-LDPC의 error-floor 개선에 원리적으로 유용하나, 고정 H-matrix 재설계·재검증 부담과 min-sum/AWGN 미검증으로 즉시 이식은 제한적.
