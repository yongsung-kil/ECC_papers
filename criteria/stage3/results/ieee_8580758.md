### B. 알고리즘 요약

1. 시스템: 단(短)패킷 M2M/저지연 통신용 QC-LDPC block code와 time-invariant SC-LDPC convolutional code(SC-LDPC-CC), BPSK.
2. 문제: 자원제약 환경에서 저지연·저복잡도가 필수. girth를 크게(6~12) 유지하면서 blocklength(block) / constraint length(CC)를 최소화해야 함.
3. 핵심 가정: CPM 기반 QC-LDPC, exponent matrix `P`의 `p_{i,j}`로 표현, 길이 `2k` cycle 존재조건 식 `sum(p_{m_i n_i}-p_{m_i n_{i+1}})=0 mod N`.
4. 핵심 기법: SMC(sequentially multiplied columns) 가정 - exponent matrix `P_SMC = [0 | P1 | γ2⊗P1 | ... | γ_{n-1}⊗P1]`, `P1` 첫 열을 정하고 나머지 열은 `γj`(mod N 곱)로 순차 생성.
5. 핵심 명제: 서브행렬 `0P1`이 길이 λ까지 strictly avoidable cycle이 없으면, 충분히 큰 N과 적절한 `γj` 선택 시 전체 부호도 girth λ 보장(Proposition 1).
6. 확장: block↔SC-LDPC-CC 연결 - syndrome former matrix `Hs`를 다항식 `H(x)` 표현, monomial code의 exponent matrix `P`로 mh(memory order) 도출.
7. 구현 디테일: `γj`를 최소로 찾는 pseudocode([7] Alg.1) 사용, m=3,4 / n=4..12로 girth 10,12 부호 대량 생성(Table I~VIII).
8. latency/복잡도: block은 `Λ_BP = nN`, SC-CC는 SW 디코딩 `Λ_SW = α(mh+1)a`. N·mh 축소가 latency·per-output-bit 복잡도 직접 감소.
9. 결과: 기존 최소부호 대비 N 비율 `Θ_N=0.47`, mh 비율 `Θ_mh=0.23`까지 축소. BER=10^-4에서 성능손실 `Δdb`는 매우 작음(0.005~0.05).
10. 한계: 순수 구성법(디코더/HW 미설계), 시뮬(Monte Carlo BER/BLER)만, 예제 rate 낮음(0.2~0.75), NAND 고rate·soft-read 미고려.
