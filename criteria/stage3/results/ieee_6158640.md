> 총평: NAND 직접 타깃의 quasi-EZ-NAND 시스템 아키텍처 논문으로 LDPC는 기성 min-sum QC-LDPC를 그대로 쓸 뿐, 디코더·부호 이식 대상은 아니지만 soft-read/progressive sensing 개념은 참고할 만하다.

### B. 알고리즘 요약

1. 대상 시스템은 4KB user page당 regular rate-`9/10` QC-LDPC(parity-check column weight 4), min-sum 디코딩 최대 8 iteration, soft-decision.
2. 풀려는 문제: 강력한 ECC/DSP는 fine-grained sensing을 요구해 flash-to-controller 전송량이 급증, SSD 속도를 떨어뜨림. EZ-NAND(칩 내장 ECC)는 해결하나 실리콘 비용 과다.
3. 핵심 모델: P/E cycling(RTN `σr∝N`, detrapping `μd,σd²∝N^0.5/N^0.6·ln(1+t/t0)`)과 cell-to-cell interference(coupling ratio) 기반 근사 NAND threshold-voltage 분포 모델(식 (1)~(8)).
4. 핵심 기법: quasi-EZ-NAND — 각 flash chip에 약한 ECC/DSP(1-bit 입력 LDPC), 컨트롤러에 full-strength(5-bit 입력 LDPC), 칩이 실패할 때만 컨트롤러가 개입하는 계층적 분산.
5. 근거: NAND raw BER은 P/E cycling에 따라 점진 악화하고 full-strength ECC는 최악(수명한계)에 맞춰 설계되므로 대부분 기간엔 약한 ECC로 충분 → 대부분 error-free처럼 동작.
6. postcompensation DSP: 간섭 셀 threshold shift로 간섭 강도를 추정·차감. `(m+n)-sensing`(현 WL m-bit, 인접 WL n-bit; 최대 5+4)로 sensing 정밀도 표기.
7. 부수 트릭: P/E-cycling-aware progressive memory sensing — 낮은 정밀도로 시작해 LDPC 실패 시에만 정밀도 상향, P/E cycling 수에 따라 초기 sensing config를 2000 cycle마다 조정.
8. 평가: DiskSim 기반 trace 시뮬(Postmark/Finance/Trace1) + finite-precision C 시뮬로 PER 산출(Table 1: 5-bit, Table 2: 1-bit).
9. 결과: quasi-EZ-NAND는 EZ-NAND와 거의 동일 속도(평균 응답시간 90%+ 감소), 65nm ASIC에서 EZ-NAND 대비 실리콘 비용 최대 44% 절감(채널당 16칩), 5-bit 디코더 2Gbps에 1.47mm².
10. 한계: LDPC 디코더·부호 자체 개선은 없고(기성 min-sum QC-LDPC 재사용), 기여는 시스템 레벨 ECC/DSP 배치 전략과 sensing 스케줄에 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum QC-LDPC 디코더 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 우리와 동일 min-sum QC-LDPC, 이미 보유 기법이라 신규 이식분 없음 |
| soft-read (m+n)-sensing / LLR 정밀도 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold·양자화 비용 모델은 참고 가능(단 4bit+ sensing은 우리 미지원) |
| progressive sensing / 조기종료 개념 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | fail-and-retry 개념은 유사, 그러나 논문은 sensing 정밀도 상향이 대상 |
| 계층적 ECC/DSP 배치(quasi-EZ-NAND) | 대응 없음 | 시스템 아키텍처 레벨, 코드베이스 대응 모듈 없음 |

적용 가치: **낮음** — LDPC 디코더·부호는 우리가 이미 보유한 min-sum QC-LDPC이고, 핵심 기여는 시스템 계층배치 전략이라 Prime ECC 이식 대상이 아니다. soft-read/progressive sensing 비용 모델만 참고 여지가 있다.
