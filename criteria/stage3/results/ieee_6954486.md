### B. 알고리즘 요약

1. 대상: BPSK/256QAM AWGN 채널의 irregular LDPC (DVB-T2 rate 1/2, length 64800 check-regular / length 10032 check-irregular).
2. 문제: min-sum이 check node에서 LLR 크기를 과대평가 → NMS(고정 α)/OMS(고정 β)로 보정하나, iteration과 check-node degree 양방향 변수 스케일링은 파라미터 폭발로 미해결.
3. 핵심 관찰: MS 근사 오차는 iteration이 진행될수록(LLR 신뢰↑) 줄고, check-node degree `dc`가 클수록 커진다 → 스케일 `αj(i)`는 `i`와 `dj` 모두에 의존해야 함.
4. 핵심 기법: 각 degree·iteration에 대해 generalized mutual information(GMI) `I(U~c; B)`를 최대화하는 스케일 `αj(i)`를 1D 탐색(greedy, 이전 iteration 값 고정).
5. 핵심 식: `αj(i)=argmin_α ∫ P_U~j(i)(l) log2(1+e^{-αl}) dl` (식31) — GMI 최대화가 mismatch LLR 정확도 최대화와 등가.
6. 확장: 검사노드 degree 무시 시 `αj(i)≡α(i)`가 되어 iteration만 의존하는 간이판 I-GMI, degree까지 반영하면 ID-GMI.
7. 실무 트릭: DE로 얻은 `αj(i)`는 실코드와 mismatch → Monte Carlo(식32)로 훈련 LLR에서 직접 `αj(i)` 산출, `αj(i)`는 (αj(i-1),1] 내 non-decreasing.
8. 학습 절차: Eb/N0를 BP threshold 근처로 두고 Nblk(10^3~5×10^3) 훈련 블록의 iteration별 MS 출력 LLR로 스케일 시퀀스를 평균.
9. 결과: 64800 code BPSK에서 I-EXIT-P/NMS 대비 0.15dB, 256QAM에서 0.2~0.3dB 개선(BP와 0.1~0.2dB차); 10032 code는 BP와 0.05dB 이내, error-floor 10^-8까지 없음.
10. 한계: HW 미설계, 시뮬만, 스케일 테이블 오프라인 사전계산 필요, degree 다수인 통신용 부호 전제(NAND 고rate 단일 degree와 상이).
