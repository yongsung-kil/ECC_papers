### B. 알고리즘 요약

1. 시스템은 5G NR 물리계층: 데이터채널 protograph QC-LDPC(BG1 rate up to 8/9, 8448bit / BG2 rate 1/5, 3840bit), 제어채널 polar. D2D/URLLC 대상.
2. 문제: LDPC/polar 복호기의 error performance·복잡도·latency를 ML로 개선하려는 최신 기법들을 비교·정리하고 gap을 찾는다.
3. LDPC 측 리뷰 대상: (a) DL 기반 통합 Polar-LDPC 디코더[17], (b) Q-learning node-wise scheduling[18], (c) RL+MCTS 기반 LDPC 부호구성[19].
4. [17] DNN이 입력에 코딩타입 indicator를 concat, hidden ReLU/output sigmoid로 메시지비트 추정. BER 10^-2에서 conventional BP LDPC 대비 0.8dB gain, throughput 6자리수 개선 주장(수치는 원저).
5. [18] check node를 cluster로 묶어 Q-learning으로 node-wise scheduling, message-passing 복잡도·전력 절감. 변형 QR/QO 중 QR이 최고 성능(최대 2dB gain, 본문 서술).
6. [19] RL+MCTS로 LDPC edge growth 최적 경로 탐색, PEG 대비 유연·저복잡. QC-LDPC로 검증, 성능은 PEG 수준.
7. Polar 측: SDAE-enhanced polar[25], ML 기반 multi-flips SC[26], RL-aided BP(RL-CABP)[27] 리뷰.
8. 학습: 각 기법이 DNN/Q-learning/RL/LSTM을 Monte Carlo 시뮬 데이터로 학습(구체 하이퍼파라미터는 원저).
9. 결과: 비교분석에서 LDPC는 Scheme2(Q-learning)가 최대 2dB로 최고, polar는 SDAE/RL-BP가 ~1dB gain. 다만 복잡도 개선은 어느 기법도 없다고 지적.
10. 한계: 자체 구현·HW 설계 없음, 전부 인용 요약이며 gap 표(Table I/II)와 두 하이브리드 스킴은 future work 제안 수준.

| Q-learning node-wise scheduling(복호 스케줄) | `decoder.cpp` `LDPC_Decoder()` | 이터레이션/스케줄 루프에 개념적 대응, 단 ML 에이전트라 직접 이식 곤란 |
| normalized/offset MS 언급 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 보유한 min-sum과 중복 |
| ML 기반 조기종료(CRC 예측) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 개념 대응하나 DNN 예측기는 대응 없음 |
| DNN/RL 통합 디코더 전면 대체 | 대응 없음 | NN 디코더는 Prime ECC 대상 외 |
| RL+MCTS LDPC 부호구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드에 닿으나 QC-LDPC 고정이라 재구성 부담 큼 |
