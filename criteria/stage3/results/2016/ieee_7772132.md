### B. 알고리즘 요약

1. 대상은 time-invariant LDPC convolutional codes(SC-LDPC의 특수족), `c×a` 다항식 parity-check `H(x)`와 syndrome former memory order `m_h`로 특성화.
2. 문제: 기존은 좋은 QC-LDPC block code를 unwrap해 LDPC-CC를 얻는데, 이는 주어진 성능에 대해 최소 `m_h`(=낮은 복잡도·저지연)를 주지 못함.
3. 핵심 도구: symbolic graph를 일반화한 symbolic hyper-graph 도입 - weight `w>2` 엔트리를 `w` inner-state를 갖는 hyper-node로, weight-3 컬럼(3 monomial 또는 monomial+binomial)을 hyper-edge로 표현.
4. 그래프의 닫힌 경로의 edge 라벨 합이 0이면 Tanner graph의 cycle에 대응; edge 라벨 `b = (q_i − q_j)c + (i − j)`.
5. row-column 제약(length-4 cycle 회피)을 만족시키는 `m_h` 하한: 정규코드 `a·C(w,2) ≤ m_h·c + C(c,2)(2m_h+1)`(Lemma III.1), 비정규 확장(Lemma III.2).
6. horizontal/vertical separation 개념으로 unavoidable cycle·codeword를 분류, free distance `d`의 상한을 유도(예: 특정 구조에서 `d=18`, 또는 order-4 hyper-node로 `d=24`, `m_h=14`).
7. 부수: girth 12(c=2)·girth 10(c=3) monomial code에 필요한 최소 `m_h`를 rate별로 하한(식 9,10)과 heuristic search로 대비 - 문헌 대비 대폭 개선(예 rate 3/6에서 38 vs 85).
8. 성능화 트릭: symbol "1"을 크기 `Q` circulant permutation, "0"을 zero block으로 확장(scale factor Q)해 밀도를 낮추고 DE threshold에 접근.
9. 결과: rate 1/2(c=3,a=6) monomial code(`d=24`, `m_h=38`)의 Q=1,5,7,9 확장 BER 시뮬(AWGN/BPSK) - Q 증가 시 DE threshold 접근, 문헌 최적 정규코드[5] 및 WiMAX block code 대비 작은 constraint length로 동등/우수 성능.
10. 한계: HW 미설계, AWGN/BPSK 시뮬만, time-invariant LDPC-CC polynomial 부호설계 이론 전용(block QC-LDPC·NAND 무관).
