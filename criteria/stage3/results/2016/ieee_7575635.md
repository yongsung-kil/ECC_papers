> 총평: variable-regular LDPC의 error-floor 원인인 LETS(trapping set)를 simple cycle로부터 최소 확장으로 특성화·완전열거하는 오프라인 분석 알고리즘으로, NAND용 부호선정/floor 예측 도구로는 참고 가치가 있으나 디코더·HW 런타임 이식과는 무관하다.

### B. 알고리즘 요약

1. 대상: variable-regular LDPC(`dv≥3`, `g>4`)의 leafless elementary trapping set(LETS) — error-floor의 주 원인구조 — 를 완전(exhaustive) 열거. 코드 C1~C11 (rate 0.5~0.87, `n`=155~20000, `dv`=3~5).
2. 문제: 기존 LSS(dot)-기반 탐색[17]은 girth보다 훨씬 긴 cycle 열거가 필요, 그 multiplicity가 급증해 실용상 불가(수천만 인스턴스 저장).
3. 표현: LETS를 normal graph(degree-1 check 제거, degree-2 check를 간선으로 치환)로 표현. `(a,b)` = 변수노드 수 a, 불만족 check 수 b. `b = adv - 2e` (`e`=normal graph 간선 수, Lemma 1).
4. 기법1(핵심): 3종 그래프 확장 정의 — `dot`(depth-one tree, 노드1개 추가), `path`(개/폐경로 `pam`), `lollipop`(`lomc`, cycle+path). 각각 `(a,b)` 클래스 이동을 Proposition 1/3/4로 산정.
5. 핵심정리: Theorem 1 — 모든 LETS는 simple cycle에서 dot/path/lollipop 조합으로 생성 가능. Lemma 5 — minimal 특성화는 이 3종 확장만 필요(그 외 불필요).
6. 기법2(dpl 특성화): simple cycle만을 prime으로 삼는 forest 구성. Algorithm 1이 각 `(a,b)` 클래스에 필요한 확장집합 `EX`와 최대 cycle 크기 `K`를 산출, Theorem 2로 `K`·`bmax'`가 최소임을 보장.
7. 탐색 디테일: Algorithm 2가 simple cycle 인스턴스 열거(Routine 5) 후 dot/path/lollipop search(Routine 6/7/8)로 확장. 관심범위 내 구조만 탐색(대형 b 클래스 회피)해 복잡도↓.
8. 복잡도: cycle 열거 `O(n(dv·dc)^k)`, dot 확장 `O(N_{a,b} b^2 dc^2)`, path `O(N_{a,b} b^2 (dv·dc)^m)`, lollipop `O(N_{a,b} Nc bc (dv·dc)^d)`.
9. 결과: dot(LSS) 대비 runtime·memory 최대 3자릿수 절감. 예: C4(`n`=20000)를 dpl은 s6까지만 열거해 9분, dot은 s10/s11 필요로 1일+8GB 초과. FAS 탐색[21]의 5~48시간 대비 42분 이내.
10. 한계: 순수 그래프이론 분석 알고리즘(2.4GHz 데스크톱 검증), 디코더·HW·양자화 미포함. variable-regular 전용(irregular 미지원), error-floor 예측/부호선정 후단 도구.

### C. Prime ECC 관련 모듈 핀포인트

대상이 `code-design`이므로 C섹션 디코더 성능표는 전부 N/A. 참고용 모듈 대응:

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LETS/trapping-set 열거로 error-floor 원인 식별 | ecc_top.cpp Load_PCM() | 우리 고정 H-matrix의 floor 유발구조 사전분석·부호선정에 참고 가능(런타임 아님) |
| 수렴실패 구조 추적 | corner.cpp | 논문은 오프라인 구조열거, 우리 corner.cpp는 런타임 수렴실패 추적 — 목적 유사하나 방법 다름 |
| dot/path/lollipop 확장 탐색 알고리즘 | 대응 없음 | 부호분석 전용 그래프 알고리즘, 디코더/인코더 런타임 모듈 없음 |
```

적용 가치: **중간** — NAND ECC의 핵심 관심사인 error-floor를 유발하는 trapping-set을 완전 열거하는 강력한 오프라인 도구로, 우리 고정 QC-LDPC의 floor 원인분석·부호선정 검증에 참고 가치가 있으나, 디코더/HW/양자화 이식 대상은 아니며 variable-regular 가정이 우리 degree-17 구조와 정합 확인 필요.
