### B. 알고리즘 요약

1. 시스템: MLC NAND 기반 enterprise SSD, code rate 9/10, codeword 8KByte, 16채널 컨트롤러.
2. 문제: soft-decoding LDPC는 21 VRef(714us)로 read latency가 커 read-intensive workload에서 SSD 성능을 최대 70% 저하시킴.
3. 모델: 애플리케이션을 write-hot/cold, read/write-intensive 4분류(Fig.2)하고 W/E cycle에 따라 신뢰도가 변하는 것으로 가정.
4. 핵심 기법(AOA-ECC): 두 advanced LDPC를 조합 - Quick-LDPC(1~2 VRef, 40-62us)와 EP-LDPC w/o upper/lower cells(3 VRef, 102us).
5. 선택 규칙: read-intensive는 Quick-LDPC 우선, W/E cycle 증가 시 EP-LDPC로 전환; write-intensive는 EP-LDPC 상시 적용.
6. 근거: EP-LDPC는 Quick-LDPC 대비 acceptable W/E cycle 22% 높음, Quick-LDPC는 read time 61% 짧음.
7. 성능 모델: total read time = ECC latency x cell read 수 (`ECC latency = cell read time + ECC decoding time`).
8. 구현: 두 advanced LDPC가 동일 encoder/decoder 회로 공유 -> 회로 오버헤드 없음, BCH 제거로 decoder gate 25% 감소(629K->470K).
9. 결과: write-intensive(web_1) 1.5배, read-intensive(proj_3) 3배 성능 향상, reliability 57% 향상, decoder cost 25% 감소.
10. 한계: 디코더 알고리즘/부호 구조는 참조[8]에 의존하고 본 논문은 transaction-level 시뮬만 수행, HW 미설계.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ECC 선택 firmware 정책 | 대응 없음 | Prime ECC는 단일 QC-LDPC 엔진, 다중 LDPC 선택 상위 정책 계층 없음 |
| soft-read VRef/LLR 추정 | channel.cpp Set_R_Offset(), Set_LLR_Th() | 개념적 연관뿐, Quick/EP-LDPC 구체 LLR 추정식은 본문에 없음 |
| LDPC decoder(sum-product) | decoder.cpp LDPC_Decoder() | 논문은 반복 sum-product를 blackbox로만 참조 |
```
