### B. 알고리즘 요약

1. BI-AWGN 채널, moderate-length high-rate QC-IRA 부호(2592bit, rate 0.833, col weight 4, girth 6, 16×96 array of 27×27 circulant)를 constituent code로 사용.
2. 문제: high-rate LDPC의 error-floor가 실사용을 막음 → 소량 redundancy로 error-floor를 낮추려 함.
3. 인코딩: `m`개 constituent codeword를 각각 permute(`Πi`) 후 XOR로 integrated user data `us`(중간 SPC parity) 생성, `ps = Hp^-1·Hu·us`로 shared parity 생성해 부착.
4. 핵심 아이디어: trapping set에 잡힌 에러 bit를 인터리버로 shuffle → 인터리브된 bit가 trapping set 밖이면 재복호 성공("trapping set 탈출").
5. 2-stage 복호: 1단은 각 `ci`를 순차 BP 복호, 실패 시 2단에서 shared parity로 실패 codeword 재복호(대형 `Hm` 전체 BP 대신 저복잡도).
6. 알고리즘 A: 성공한 user data로 `us\i` 생성 후 SPC erasure 복호로 대체 LLR `π̃i` 만들어 `Πici`를 재BP. A+는 `Hi,s`(더 큰 그래프)로 error-floor 시작부를 추가 개선.
7. 알고리즘 B: 두 constituent 동시 실패 시 `Hi,j,s`(5616bit, rate 0.769)로 재복호해 waterfall 하단까지 개선. 알고리즘 D는 오검출(incorrect decoding) 처리.
8. 인터리버: random 부호엔 1-bit cyclic-shift로 충분하나 QC-LDPC는 circulant-size(27bit) cyclic-shift가 적합(순수 spreading보다 trapping set 탈출 확률이 핵심).
9. 결과: circulant-size cyclic-shift 인터리버가 random 인터리버 수준의 error-floor 개선을 단순하게 달성, BCH 외부부호 SCC-II/SCC-SPC보다 우수. non-shared 대비 waterfall 0.2dB 손실 대신 error-floor 대폭 개선.
10. 한계: HW 미설계·시뮬만, 2-stage soft-BP 재복호로 SCC 대비 메모리↑, 3개 이상 constituent 실패 시 방법 무력.
