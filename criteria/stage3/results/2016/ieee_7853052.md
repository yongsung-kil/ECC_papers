### B. 알고리즘 요약

1. 대상: (1008,504) rate-1/2 MacKay LDPC, AWGN+BPSK, hard-decision 벡터 `z`에 soft 정보 `y` 결합한 GDBF 복호.
2. 문제: GDBF는 성능/복잡도 균형이 좋으나 무한 decoding loop(local maximum)에 빠져 max iteration 도달 시 복호 실패.
3. 기존: [13] Liu는 flip된 bit 위치를 기록·비교해 loop 검출·excluding set으로 방지 → 추가 연산/저장 부담.
4. 핵심 기법: syndrome weight `wt(s)` 변화로 loop 검출 — 성공 복호는 weight 감소 과정이므로 weight 증가를 loop로 간주.
5. flipping function `Ek = zk*yk + Σ(1-2si)` (식5)로 bit 신뢰도 측정, single-bit는 최소 `Ek` bit를 flip.
6. 탈출 메커니즘: loop 검출 시 GDBF flipping 대신 원래 BF flipping function `E'n = Σ(1-2sm)`(식6)으로 syndrome weight를 가장 크게 줄이는 bit 선택.
7. excluding set `B`(size 임계 Δ=2)로 두 flipping function 간 대형 loop 형성 방지, `|B|≥Δ`이면 B를 비우고 Δ 증가.
8. 학습/최적화 절차 없음(순수 규칙 기반).
9. 결과: EGDBF가 GDBF 대비 BER 10^-5에서 약 1dB 코딩 이득, NGDBF 대비 0.1dB 손실이나 복잡도 더 낮음(iteration당 N+1 비교 추가).
10. 한계: 단일 부호·rate 0.5 시뮬만, HW 미설계, multi-bit 케이스 미다룸, 고rate NAND 부호 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBF/BF flipping function 이터레이션 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum MP 기반이라 BF 계열 이터레이션과 알고리즘 상이 |
| syndrome weight 기반 loop/조기종료 판정 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 개념은 유사하나 Prime ECC는 CRC 기반, syndrome weight 미사용 |
| soft 정보 `zk*yk` 결합 | `channel.cpp` `Set_LLR_Th()` | soft-read LLR와 개념 접점이나 GDBF 전용 |

적용 가치: **낮음** — BF/GDBF 계열은 min-sum 기반 Prime ECC와 디코더 구조가 달라 이식 시 엔진 교체 수준이고, 검증도 저rate 단일 부호 시뮬에 그침.
