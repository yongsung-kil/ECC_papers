> 총평: SMS+WBF post-processing으로 short code 빠른 수렴·1.4~27Gbps 확장 throughput을 FPGA로 검증했으나 저rate·random 부호·flooding 전용이라 고rate QC-LDPC Prime ECC 이식성은 제한적.

### B. 알고리즘 요약

1. 대상: (512,256) rate-1/2 random regular (3,6) LDPC, BAWGN+BPSK, flooding(완전병렬) 스케줄, short code에서 고throughput 목표.
2. 문제: 긴 부호는 성능 좋으나 HW 복잡·라우팅 난이, short code는 throughput 높으나 정정능력 부족 — 성능/throughput/복잡도 trade-off 해결.
3. 핵심 기법: WBFSMS — Scaled Min-Sum(`SMS`, check node에 scaling factor `a`)로 노드를 steady state로 수렴시킨 뒤 WBF를 post-processing으로 적용해 오류정정 강화.
4. SMS check node 식 `E_new = a·(∏ sign)·min|L|`, VN 식 `L = uCh + Σ L_ij`, 경판정 후 syndrome null이면 조기종료.
5. WBF는 syndrome `S=y·H^T`와 error weight `Wij=Σ Sj·Hij`, `ei=Σ Wij`로 최대 `ei` bit flip — SMS 결과 오류를 보정.
6. cyclic mode: SMS·WBF 출력이 일치하는 노드의 LLR을 증가시켜(식8) 재복호, node deactivation의 error-floor 문제 회피.
7. HW 트릭: WBF를 XOR+permutation+decoder만으로 1클럭에 구현, scaling을 2 MSB 감산으로 근사해 로직 21→5(76% 감축), VNU 6→5 adder(16.6% 감축).
8. 학습/최적화 절차 없음(scaling factor는 시뮬로 결정).
9. 결과: 2.5dB에서 BER 10^-6·FER 10^-4·1.4Gbps, 6dB에서 27Gbps(250MHz), fast convergence 계열 대비 우수(단 serial [13] 제외이나 throughput 20배).
10. 한계: short random (3,6) 부호 전용, gatecount@공정·전력 수치 본문 미기재(표 손상), QC-LDPC pipelining 아닌 flooding 전제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Scaled Min-Sum CN 연산(scaling factor a) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이나 scaling factor 명시 미확인 — scaling HW 트릭 참고 여지 |
| WBF post-processing 이터레이션 | `decoder.cpp` `LDPC_Decoder()` | BF 계열 post-processing은 Prime ECC min-sum 루프와 이질적 |
| syndrome checker 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 목적 유사하나 Prime ECC는 CRC 기반 |
| flooding 완전병렬·라우팅/VNU HW | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC는 z=32 row 병렬 구조라 flooding random 부호와 정합 낮음 |

적용 가치: **낮음** — SMS+WBF 통합·flooding 아키텍처는 고rate QC-LDPC(z=32) Prime ECC와 부호·스케줄이 달라 직접 이식 어렵고, 참고할 만한 건 scaling 2-MSB 감산 등 국소 HW 절감 트릭뿐.
