### B. 알고리즘 요약

1. 대상은 표준 convolutional code(CC) `C(q,p,m)`, rate 1/2·2/3, block length `N=1000`, 무선/turbo 문맥.
2. 문제: Viterbi(VA)/BCJR 복호는 memory 블록수 `m`에 대해 복잡도가 지수적(2^m states)으로 증가.
3. 핵심 관찰: CC의 D-domain 생성행렬 `G(D)`→패리티행렬 `H(D)`를 extended invariant factor 알고리즘으로 얻고 binary `Hbin`으로 전개하면 sparse 행렬이 됨.
4. `Hbin`은 edge block `[Hm...H1 H0]`의 순환 shift로 구성되며, sparse하므로 LDPC용 message-passing(MP) 복호가 적용 가능.
5. 핵심 주장: MP(OMS) 복호 복잡도는 `I·q(m+1)(N-K)`로 `m`에 대해 선형 증가 → VA의 지수 증가 대비 큰 절감.
6. 복호기: BP의 실용 근사인 Offset Min-Sum(OMS)을 채택(VA와 동일한 add/compare 기반이라 공정 비교 가능).
7. 트릭: Table II/III의 표준 CC는 girth-4를 가져 MP에 불리 → 생성다항식을 약간 수정해 length-4 cycle 제거(Table IV)로 성능 개선.
8. 학습/최적화 절차 없음.
9. 결과: rate 2/3은 OMS≈VA 성능, rate 1/2는 큰 m에서 VA 우세하나 girth-4 제거 후 m=10까지 VA에 근접; 복잡도 격차는 최대 1 order; OMS soft-output으로 turbo 복호도 가능.
10. 한계: HW 미설계·시뮬만, MP는 block-length만큼 latency 추가, 대상이 CC/turbo로 NAND LDPC와 직접 무관.
