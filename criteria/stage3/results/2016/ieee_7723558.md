### B. 알고리즘 요약

1. 대상: 임의 QC-LDPC(base matrix B, R×C=12×24, 확장인자 Z)에 대응하는 layered Min-Sum 디코더 HW 아키텍처. (3,6)정규 (648,1296)과 WiMAX rate-1/2 (1152,2304) 두 부호로 검증.
2. 문제: layered QC-LDPC 디코더에서 pipeline은 base matrix 제약·수렴속도 저하를 유발 - pipeline 없이 HW 자원 재사용으로 면적↓·throughput↑을 달성하려 함.
3. 알고리즘: 표준 layered MS - `α_{m,n}=γ̃_n-β_{m,n}`(VNU), `β_{m,n}=∏sign·min|α^SAT|`(CNU), `γ̃_n=α+β`(AP-LLR), 레이어별 순차 갱신 후 syndrome check.
4. 기여1(VNU/AP-LLR 병합): VNU는 감산기, AP-LLR은 가산기라는 차이만 있으므로 `sel` 신호로 carry-in(C0=sel̄)과 XOR을 제어하는 수정 RCA 하나로 두 기능 병합.
5. 기여2(CNU 2회 실행): CNU가 min1·indx_min1만 계산(1클록), 두번째 클록에 indx_min1 입력을 최대값으로 세팅해 재실행하여 min2 계산. TS 트리구조(FMIG/IG 블록) 기반.
6. 기여3(대칭 2단 분할): 1클록=read+VNU모드+CNU(min1), 2클록=CNU(min2)+AP-LLR모드+write. 두 클록 부하 대칭 → 최적 클록주파수.
7. 압축 β-message: signs+min1+min2+indx_min1만 저장(Fig.2)해 β memory 축소. γ̃은 레지스터, β는 RAM.
8. 양자화: 입력 LLR·β는 q bit, AP-LLR γ̃은 q̃ bit(q<q̃), 포화 가산기 사용. 실험은 q̃=6, q=6/7.
9. 결과: 65nm ASIC 합성 - 최대주파수 baseline 대비 ×2.25((3,6)), ×3(WiMAX). 면적 baseline 대비 24.2% 감소((3,6)), 동일타이밍시 25.26%/13.64% 감소. NTAR 27.9 Gbps/mm2/iter(WiMAX), 75(정규).
10. 한계: 순수 HW 아키텍처 최적화 - BER/정정능력/iteration 개선 없음, 확장인자 Z는 run-time 고정, iter=20 고정.
