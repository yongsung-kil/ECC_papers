### B. 알고리즘 요약

1. 대상은 일반 `[n,k,d]` binary linear code의 parity-check matrix(PCM) 이론 (특정 rate/length 없음).
2. 문제: BP 디코딩 실패를 유발하는 `(a,b)`-trapping set을 없애려면 PCM에 몇 개의 redundant row가 필요한가 = trapping redundancy `T_{a,b}(C)`.
3. 기존 최강 상한(Laendner et al. [15])은 Lovász Local Lemma에 의존하나, 상호독립 가정이 성립하지 않아 엄밀성 결함, 또 작은 `a`(`a≤⌊(d-1)/2⌋`)에만 적용됨.
4. 본 논문은 LLL 없이 `1≤a≤d-1` 전 범위에 적용되는 더 sharp한 상한을 제시 (Theorem 3.1).
5. 핵심 도구: dual code `C⊥`의 전 codeword가 `OA(2^{n-k},n,2,d-1)` orthogonal array (Prop 2.1), 즉 임의 `a≤d-1` 비트가 완전 랜덤처럼 보임.
6. `t×n` 랜덤 행렬 `H_t`(rows drawn from `C⊥`)에서 rank 분포(Lemma 3.2)와 odd-weight 행 수 기대값 `E(X_M)`를 결합해 `E(Z_t)` 계산.
7. `T_{a,b}(C) ≤ min_t {t + ⌊E(Z_t)⌋}`로 collective trapping redundancy 상한 유도 (linearity of expectation).
8. Corollary 3.3: 좌변 부등식 <1이면 `T_{a,b}(C)=n-k` (자명 하한 달성) — Gilbert-Varshamov 이하 코드는 지수적으로 빠르게 `n-k` 도달.
9. 결과: Margulis code(n=2640,k=1320), `a≤14,b=5`에서 상한이 자명 하한 1320과 정확히 일치 (기존 LLL 상한 1394~1464 대비 tight, Table I).
10. 한계: 존재성만 보장, sparse·구조 제약 반영 안 함, 실제 PCM 구성 알고리즘 없음, HW·시뮬 없음, 연판정/디코더 성능 무관.
