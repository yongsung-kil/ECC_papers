> 총평: dual-diagonal QC-LDPC의 q-parallel 인코더 HW(42Gbps FPGA)는 Prime ECC 인코더와 같은 계열이나 이미 보유 기법이고 RC 부분은 NAND에 불필요해 참고용 수준.

### B. 알고리즘 요약

1. `J×(K+J)` 서브행렬 배열로 된 mother matrix `H`를 정의, 우측 parity 부분은 dual-diagonal 고정, 각 서브행렬은 `q×q` shifted identity `I(C_j,k)` 또는 null.
2. 문제: puncturing 기반 RC LDPC는 차수분포를 훼손해 고rate daughter code 성능이 나빠짐.
3. 기법(부호): systematic part의 non-null 위치·시프트계수 `C_m,n`을 short-cycle 회피 조건 `Σ(C_mk,nk − C_mk+1,nk)=0 mod q` 하에 배치, density evolution으로 column-degree 분포 최적화.
4. 인코딩: `Hx^T=0`에서 `p1^T=Σ h1,k s_k^T`, `p_j^T=Σ h_j,k s_k^T + p_{j-1}^T` (dual-diagonal 재귀) → linear-time 인코딩.
5. shifted identity 곱은 `x_j,k = s_k`를 `C_j,k`만큼 좌순환 시프트로 대체(곱셈 제거) → XOR로만 parity 계산.
6. q-parallel 아키텍처: `q`개 XOR processor(degree ≤ 6, 예제 w_j=6)를 병렬로 돌려 한 codeword를 `J` 클록에 인코딩.
7. HW: 8쌍 RAM-SYS/RAM-PARITY(block RAM), 24개 좌순환 시프터(시프트계수는 ROM 상수), 72개 XOR processor(6입력 1출력).
8. 전송기법: mother code(RL=0.5)에서 `p_J`만 보내 daughter rate `R_H=K/(K+1)=0.96` 달성, zero-filling 복호(미전송 bit LLR에 작은 양수 `U_j=x̄_n/100` 부여)로 단일 H로 mother/daughter 모두 복호.
9. 결과: 72-parallel 인코더가 Virtex-5에서 300MHz 시 42Gbps(최대 600MHz), FF 72/LUT 144(<0.1%), rate 0.5~0.96 7종에서 FER=1e-5까지 error floor 없음.
10. 한계: encoder HW만 FPGA 실증(복호는 시뮬), 부호길이 1800 고정 예제, RC/재전송(ARQ) 문맥으로 NAND 고정rate 시나리오와 목적 상이.

| shifted-identity 곱 → 좌순환 시프트+XOR | [encoder.cpp](../Prime_ECC_3.1_Claude) `Generate_PCM_encoding()` | QC circulant 시프트 인코딩 원리 동일, HW 병렬화 아이디어만 참고 |
