### B. 알고리즘 요약

1. 대상: (3,6) regular QC-LDPC, 길이 1296bit, base matrix 12×24, circulant `z=54`, MS 복호.
2. 문제: partially-parallel flooded 디코더에서 병렬도 `p`를 올리면 멀티포트 메모리와 다중 barrel shifter가 필요해 메모리·라우팅 비용이 지배적으로 증가.
3. 관찰: 최신 FPGA에서 메모리/BRAM 동작주파수(~600MHz)가 처리유닛(150-250MHz)보다 훨씬 높다는 점을 활용.
4. 핵심기법: 두 클록 도메인 사용 — 메모리 인터페이스와 barrel shifter 라우팅에 고클록 `fH`, 처리유닛에 저클록 `fL`, `fH = R·fL` (정수배).
5. 의미: 고클록 도메인이 시분할로 단일포트 메모리/소수 barrel shifter를 재사용해 멀티포트·다중 라우팅을 에뮬레이션 → 배선·MUX 수 감소로 라우팅 혼잡 완화.
6. flooding 스케줄 MS: CNU `β=∏sign(α)·min|α|`, VNU `α=γ+Σβ`, APP `γ̃=γ+Σβ`, 출력 `sign(γ̃)`.
7. 구현 디테일: `R=3`일 때 MC는 read/write 각 1포트 메모리 + barrel shifter 2개로, 병렬도 3 단일클록(메모리 3R/3W + barrel shifter 6개)과 동일 정정성능을 훨씬 적은 메모리/라우팅으로 구현.
8. 두 디코더를 Verilog로 구현, Xilinx ISE 14.7 / Virtex-7 VLX485T 합성, SystemVerilog+C++ 모델로 검증.
9. 결과: 단일클록 serial=1976 slice/10 BRAM/280Mbps(20 iter), MC(fH 450/fL 150MHz)=3763 slice/10 BRAM/353Mbps → throughput +25%, 메모리 오버헤드 0, 로직 +80%.
10. 한계: rate-0.5 (3,6) 정규부호·짧은 부호길이 FPGA 합성만, BER/정정성능·고rate 부호·ASIC 미평가, 로직 면적 증가가 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| flooding MS CNU/VNU 이터레이션 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | MS 연산 자체는 이미 보유, 스케줄만 flooding |
| barrel shifter 라우팅 / MUX edge 선택 | `GT.cpp` `Make_GT_HW()` | 라우팅 축소라는 목표는 GT(그래프 씨닝)와 유사, 접근은 상이 |
| dual-clock 메모리 포트 에뮬레이션 | 대응 없음 (HW clock-domain 트릭, C++ bit-exact 시뮬 범위 밖) | 알고리즘 무관, RTL 설계 이슈 |
| 부호구조(circulant, base matrix) 로드 | `ecc_top.cpp` `Load_PCM()` | z=54 vs 우리 z=32, 부호구조 상이 |

적용 가치: **중간** — 메모리/라우팅 병목 절감이라는 방향은 우리 관심사와 맞지만, dual-clock은 RTL 클록 도메인 설계 트릭이라 bit-exact C++ 시뮬레이터에는 직접 이식할 대상이 없고 정정성능 기여도 없음.
