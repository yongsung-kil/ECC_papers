### B. 알고리즘 요약

1. 대상: 랜덤/구조 irregular·regular LDPC (예제 A: `N=504,M=252` PEG, B: `N=2048` IEEE 802.3an), AWGN+BPSK, LLR clipping ±200.
2. 문제: 고SNR error-floor 영역 BER를 importance sampling(IS)으로 구하는 데 시간이 오래 걸리고 랜덤 코드에는 구조적 선형모델([3][4])이 안 맞음.
3. 모델: min-sum 3단계 중 VN처리(2)·사후LLR(3)는 선형, CN처리(4)만 비선형 → CN을 선형화하면 전체를 `R_l=A·Q_{l-1}`, `Q_l=B·R_l+L_ch`, `L_l=C·R_l+L_ch` 로 표현.
4. 핵심 기법: 비선형 CN 행렬 `A(e,em)=∏sgn(V_j')`를 구하려면 최소절댓값 메시지를 보내는 VN(`jm`)을 특정해야 함.
5. 의미: 고SNR에서는 소수의 active dominant TS의 VN만 저신뢰도라 최소메시지 원천이 됨 → TS로 `jm` 후보를 좁힘.
6. active TS 식별: 개별 오류확률 `Pek=1/(1+e^{|Lch_k|})`가 평균오류 상한 `H(X|Y)/2`를 넘으면 그 VN을 vulnerable로 판정, TS의 모든 VN이 vulnerable이면 active(식(20)).
7. A 계산(Algorithm 1): 각 edge에 대해 `S∩Va`가 1개면 `A=1`, 2개면 채널LLR 작은 쪽을 `jm`으로 잡고 `A=-1`, 없으면 extrinsic 정보량 `Ei` 최소 VN 선택.
8. BER 추정: 최종 iteration CNU 메시지 `Mcv`의 평균/분산으로 사후LLR 통계(식23,24) → `P_i=Q(...)` → `BER=(1/N)ΣP_i` (100개 active TS 수집까지 반복).
9. 결과: 코드 A는 선형모델 15 iter, 코드 B는 7 iter로 IS와 유사한 error-floor BER 곡선 재현(약간 과대추정), 계산시간은 IS보다 크게 단축.
10. 한계: 디코더/부호 개선 없음, HW 설계 없음, 시뮬만이며 error-floor 예측 정확도가 [4]보다 낮음.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |

| active TS/error-floor BER 예측 | corner.cpp (수렴실패 추적) | 개념적으로만 근접, 예측기법은 코드 미대응 |
| 채널 LLR/vulnerable 판정 | channel.cpp Set_LLR_Th() | LLR 기반 판정 대응하나 논문 목적은 BER예측 |
```
