### B. 알고리즘 요약

1. 시스템: regular QC-LDPC (N, 열가중치 λ, 행가중치 ρ), BPSK/AWGN 채널로 실험 (구체 수치는 .txt 손상).
2. 문제: ASIC은 고정되어 빠른 알고리즘 변화·다표준 호환에 대응 불가, DSP는 유연하나 면적·복잡도 큼 -> 유연성과 ASIC급 성능을 겸비한 재구성 아키텍처 필요.
3. 알고리즘: Normalized Min-Sum(NMS)를 채택(정확도와 HW 비용 균형). 초기화 `L(qij)=LMi+yi`, CN 처리(min + sign), VN 처리, 종료판정(H·codeword) 단계로 구성.
4. 핵심기법: NMS의 각 단계 DFG를 분석해 coarse-grained reconfigurable array(RaSP-D)에 매핑하는 매핑 전략 제안.
5. 구조: RaSP-D = 두 동형 RCA region(각 8x8 PE 추정, 표기 손상) + 도메인간 동기제어/구성제어/공유메모리.
6. fusion-step routing: crossbar 기반 인접 PE 완전연결 + adjacent-column 연결(고정->가변 pipeline stage 변환) + stepped 연결(역삼각 DFG 데이터 흐름 대응).
7. 구현 디테일: 고정소수점 PE(16bit 추정, sign bit + fixed-point), Input/Output FIFO·상수/임시 레지스터로 NMS 데이터접근 특성에 맞춘 I/O 인터페이스.
8. 최적화: 매핑 전략으로 초기화·CN·VN 단계 배열 이용률 100%, 판정 단계 75% 등, 총 평균 이용률 up to (수치 손상).
9. 결과: TSMC 65nm 합성, peak throughput >1 Gbps로 LTE-A 충족, 4가지 code rate + code length 범위 지원, ASIC(1.5Gbps)/CGRA 대비 더 넓은 rate 범위·유연성.
10. 한계: BER 개선 아님(HW 매핑 논문), iteration 감소 없음, .txt 추출 손상으로 정량 수치 다수 불명, NAND 특화 아님(LTE-A 대상).

| VN 처리 / 고정소수점 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` 등 | 고정소수점 PE 개념은 공통, 매핑 전략 자체는 CGRA 특화라 무관 |
| fusion-step routing / RCA 배열 매핑 | 대응 없음 | Prime ECC는 z=32 고정 pipeline(HCU/GT) HW 모델, 재구성 배열 라우팅과 목적 상이 |
| 반복/종료 판정(H·codeword) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` 루프 / `partial_CRC.cpp` | 종료판정 개념 공통이나 CGRA 매핑은 이식 대상 아님 |
