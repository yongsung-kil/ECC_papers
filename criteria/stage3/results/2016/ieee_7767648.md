### B. 알고리즘 요약

1. 대상은 광섬유 통신용 QC-LDPC 컨볼루션 코드(SC-LDPC), 프로토그래프 `(J,K,L,M)` = 최대 열가중 J, 행가중 K, 결합길이 L, lifting M.
2. 문제: windowed decoding(WD) 복잡도를 낮추려 반복수를 제한하면 성능이 떨어지는데, 기존은 naive round-robin/flooding 스케줄만 사용.
3. 저지연·저메모리 WD에서 window 크기 W, 매 sliding마다 `W·M`개 CN을 활성화, 이를 M 단위 layer로 분할해 layered 스케줄링.
4. 핵심: window 내 W개 layer의 갱신 순서(irregular decoding order)를 최적화해 추가 복잡도 없이 수렴 속도 향상.
5. 최적화 지표는 P-EXIT chart로 계산한 proto-CN 갱신 시 extrinsic mutual information 평균 증가량.
6. 스케줄 탐색공간이 지수적으로 크므로 M-algorithm(트리 탐색 pruning, survivor 500개)으로 유망 후보만 유지.
7. 확장: window당 반복수 I=1,2,3와 window 크기 W=3,6,12 영향을 분석 - 1-iteration·소형 window에서 이득 최대.
8. 흥미롭게도 irregular 코드가 regular 코드를 항상 능가하지 못함 - irregular 스케줄이 regular 코드에서도 weight irregularity처럼 작동.
9. 결과: threshold 기준 최대 70% 복잡도 감소 또는 동일 복잡도에서 1.9dB threshold 개선; 유한길이 girth-8 LDPC-CC(4,20,20,384)에서 1-iteration WD 시 flooding 대비 BER 10^-8에서 0.95dB, round-robin 대비 0.34dB 이득.
10. 한계: HW 미설계(스케줄링 분석·시뮬만), 광통신 SC-LDPC 특화, block QC-LDPC 미대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| layer 갱신 순서(스케줄링) 최적화 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 block-layered(dual-update even/odd 교대)라 window 슬라이딩 스케줄 개념이 다름 |
| P-EXIT 기반 스케줄 설계 | 대응 없음 | Prime ECC에 오프라인 스케줄 최적화 툴 없음 |
| windowed decoding(SC-LDPC) | 대응 없음 | Prime ECC는 block QC-LDPC, WD 구조 없음 |

적용 가치: **낮음** - SC-LDPC windowed decoding 전용 스케줄링이라 block QC-LDPC 기반 Prime ECC에 직접 이식할 구조적 접점이 없음.
