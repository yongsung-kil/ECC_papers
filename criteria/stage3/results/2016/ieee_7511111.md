> 총평: min-sum VN-update에 비전사(non-surjective) LUT framing function을 적용해 메시지 비트폭 축소와 복호 성능 개선을 동시에 얻는 기법으로, decoder.cpp의 LLR 테이블/양자화 레이어에 낮은 난이도로 이식 검토 가치가 있음.

### B. 알고리즘 요약

1. WiMAX irregular LDPC (rate 1/2, N=2304, λ(x)=0.2895x+0.3158x²+0.3947x⁵, VN degree∈{2,3,6})를 대상으로 binary-input AWGN 채널에서 q=4bit(Q=7) Min-Sum 계열 디코더를 다룸.
2. 기존 FAID는 BSC·regular 코드 한정이었으나, 이 논문은 AWGN 등 임의 채널과 irregular 코드로 확장.
3. VN-update 결과에 saturation 후 framing function `F`를 적용: `Φv(γ,m)=F(sM(γ+Σmj))`, F는 `[F(0),...,F(Q)]` LUT로 정의.
4. F가 non-surjective(치역이 M의 진부분집합)이면 weight `W=|Im(F)|`이 되고, 메시지는 `w=⌈log2(W)⌉+1` bit로만 표현 가능(Proposition 6) — 이것이 NS-FAID.
5. F는 비감소·기함수 조건(Prop.2)만 만족하면 되며, Offset-MS(`F(x)=sgn(x)max(|x|-λ,0)`)·Partially-OMS 등 기존 기법이 NS-FAID의 특수 사례로 통합됨.
6. VN-degree별로 다른 Fdv(irregular NS-FAID)를 적용, CN-message 비트 절감을 위해 `Im(Fdv)⊆Im(Fdv')` 포함관계 제약을 추가 부여(layered architecture 대응).
7. 특정 image set(예: {0,±1,±3,±7})을 쓰면 CN의 min 연산이 비교기 트리 없이 AND 게이트만으로 구현 가능해짐.
8. Density Evolution으로 목표 BER η=10⁻⁶ 기준 전체 1,097,797개 조합의 η-threshold를 계산해 각 (w2,w3,w6) 앙상블별 최적 F 탐색, 채널 gain factor μ도 공동 최적화.
9. NS-FAID-433(index17)이 MS 대비 최대 0.36dB 이득 + interconnect 8.88% 절감, NS-FAID-332(index22)는 성능 저하 0.05dB로 interconnect 29.93%/메모리 25% 절감 확인(Monte-Carlo로 0.31dB 이득 재확인).
10. HW 구현 결과는 미제시(향후 과제), rate 0.5 WiMAX 코드 한정 검증이며 고rate NAND급 코드에서의 결과는 미상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN-update framing function(LUT) | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | VN 출력에 비전사 LUT를 추가 적용하는 방식으로 이식 검토 가능 |
| iteration/degree별 LLR·양자화 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | degree별 다른 framing function을 두는 구조가 기존 LLR 테이블 구조와 유사 |
| CN min 연산 AND-게이트 단순화 | `decoder.cpp` `CNU_Update_New_Mag()` | 특정 image set 채택 시 비교기 대신 AND 로직 적용 검토 가능 |
| 메시지 비트폭 축소(메모리/interconnect) | 디코더 HW 모델(z=32 등) | Prime ECC의 메시지 저장 폭 설계에 참고 가능하나 HW arch 자체는 논문에 없음(중간 난이도) |
| Density Evolution 최적화 절차 | 대응 없음 | Prime ECC는 고정 H-matrix/LLR 테이블 사용, DE 기반 자동 탐색 도구 없음 |

적용 가치: 중간 — VN-update LUT 축소 아이디어는 낮은 난이도로 이식 가능하나, 검증이 이론/시뮬 수준이고 rate 0.5 코드 기준이라 NAND 고rate 환경에서의 재검증이 필요함.
