### B. 알고리즘 요약

1. 대상 채널: symbol-pair read 채널을 read length `D`의 **symbol-tuple 오류채널**로 일반화(`r_j`가 `x_{j-D+1..j}`에 의존, D=1이면 BSC, D=2면 symbol-pair). 고밀도 저장 응용 상정.
2. 문제: symbol-pair 부호는 대수적 구성·복호만 연구됐고, symbol-pair(tuple) 채널 위 LDPC의 반복복호는 미보고.
3. 채널 확률모델: `p(r|x)=∏ p(r_j|a_j)p(a_j|x_{j-D+1..j})`, 오류확률 `p`, 오류 시 `p/(2^D-1)`로 균등(식 1).
4. 핵심: 사후확률 `p(x|r)`를 factor graph로 인수분해(식 3)하고 **sum-product**로 주변화하는 메시지패싱 디코더 유도 — 검은 factor node가 `Σ p(r_j|a_j)p(a_j|x_{j-D+1})` 계산.
5. 문제점: `D≥3`이면 black factor node와 variable node 사이 **작은 루프**가 생겨 SP 근사가 부정확 → 보조변수 chain `p(a|x)=∏ p(a_j|a_{j-1},x_j)`로 재인수분해(식 4)해 루프 완화.
6. 사용 부호: `(dl,dr)` regular LDPC(랜덤 그래프) 및 **spatially-coupled** 부호(base matrix H(dl,dr,L), lifting M, band 구조).
7. SC 부호 rate: `R=1-dl/dr - (dl-1)/(kL)`, coupling length L↑이면 block rate `1-dl/dr`에 근접(rate-loss는 [12]로 완화).
8. 무게분포 분석: `(dl,dr)` LDPC의 D-symbol-tuple code 평균 Hamming 무게분포 `A_D(w)`를 emulator 함수 `g_D(x1,x2)=1-x1+x1(1+x2)^D`의 계수로 표현(식 5), Burshtein-Miller 방식으로 점근 분석(Lemma 1).
9. 결과(시뮬): (3,6) LDPC(n=131072)와 (3,6,16) SC 부호(M=4096)를 D=2 tuple 채널에서 제안 디코더로 복호, symbol-tuple 오류확률 p≈0.17~0.24 구간 BER/BLER 곡선 제시(SC가 waterfall 우수).
10. 한계: 채널 용량·구체 gate/throughput 미도출(future work), HW 미설계, symbol-tuple 채널이 표준 NAND read채널과 상이.
