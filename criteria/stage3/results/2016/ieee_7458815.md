### B. 알고리즘 요약

1. AWGN/BPSK 채널, sum-product 50 iteration 가정. 정보길이 고정(576/1248/1872 bit), rate-compatible 코드군 `{R1>R2>...>RJ}` 구성.
2. 풀려는 문제: 기존 RC-LDPC(puncturing은 고rate 성능열화, Kite코드는 random-like라 HW 곤란) 대비 구조적이면서 넓은 rate 범위 성능 확보.
3. 핵심 프레임: superposition 구성 - base matrix `B` + masking matrix `Z` + expansion(CPM 치환).
4. 대수기반 base matrix: `GF(q)`(q=L+1 소수)에서 `B=[α^{im}-α^{jn}]` 형태로 M×N base matrix 생성, 각 원소는 `α^{cm,n}`.
5. Masking: `Bmask = Z⊗B`, `Z`의 0/1로 엣지 선택. 이후 각 원소를 (q-1)×(q-1) CPM으로 치환(expansion)해 QC 구조 획득.
6. 코드구조: Raptor-like 팩터그래프. check bit을 Part1(최고rate)+Part2(추가 parity)로 분할, `H_RC-LDPC`는 하삼각 `[M_j | I]` 블록형.
7. 핵심 기법(Algorithm 1): P-EXIT chart 기반 masking 최적화 - `Z1`에서 컬럼을 순차 puncture하며 decoding threshold(`PEXIT(Znow)`)가 최소가 되는 컬럼을 찾아 `Z2..ZJ` 결정.
8. CPM 크기 `L` 결정: `L+1`이 소수이고 `L+1 > max(M1×J, N1)` 조건 만족.
9. 결과: Example2에서 BER 10^-6 부근 Kite코드 대비 우수, Example3에서 WiMax LDPC와 고rate 유사 성능+저rate 확장 가능.
10. 한계: waterfall 영역만 시뮬 검증, error-floor/HW 구현/throughput 미보고, RC 특화라 단일 고rate NAND ECC와 목적 상이.
