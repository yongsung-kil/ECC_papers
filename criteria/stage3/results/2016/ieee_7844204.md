### B. 알고리즘 요약

1. 시스템: pre-5G 무선용 `R=4/5` 시변 `(215,3,6)` LDPC 컨볼루셔널 부호(LDPC-CC), AWGN/BPSK 채널.
2. 문제: LDPC-BC는 라우팅 복잡도·가변 프레임 길이 문제, LDPC-CC 디코더는 긴 FIFO가 면적·latency 병목.
3. 핵심 가정: LDPC-CC의 Tanner 그래프는 반무한(semi-infinite), 메모리 `ms` 기반 슬라이딩 윈도우 프로세서 `I`개를 직렬 연결해 `I` iteration 수행.
4. 핵심 기법 1: Overlapped 아키텍처 - 두 인접 processor를 한 그룹으로 묶고 check node 갱신 순서를 바꿔 overlapped 영역에서 저장소 요소 공유.
5. 핵심 식: shift register 길이를 `2(ms+1)`에서 `ms+S+1`로 축소, `(ms+1-S)/(2(ms+1))%` 저장소 절감(`S`=processor 헤드로부터 거리).
6. 핵심 기법 2: OVA(on-demand variable node activation) 스케줄링 - VNU를 `dv`개 SVNU로 분할해 LLR이 CN에 들어가는 위치에서 즉시 갱신, 수렴 가속.
7. 부수 트릭: min-sum 기반 CNU(`dv=6`)에 partial SVNU를 파이프라인 단에 삽입해 unready message로 인한 stall 회피, folding으로 병렬화.
8. 부수 트릭2: Hybrid-partitioned FIFO - register 기반에서 shift-only 긴 구간(길이 12/16/20)을 memory bank로 치환, post-LLR의 약 40%를 12개 2-port memory로.
9. 결과: 65nm UMC 실측 7.72 Gb/s@322MHz, 1.19 mm2, 410.5 mW, 8.75 pJ/bit/proc, 1014.4k gate.
10. 한계: LDPC-CC 전용 아키텍처로 블록 부호와 무관, 정정능력 개선이 아닌 순수 HW 효율 개선.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CNU (`dv=6`) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 자체는 대응하나 이미 보유, CC용 파이프라인 CNU 구조는 부적합 |
| OVA/SVNU 스케줄링 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | 스케줄링 개념은 닿으나 CC의 슬라이딩 윈도우 전제라 이식 불가 |
| Overlapped FIFO / hybrid FIFO | 대응 없음 | CC 파이프라인 저장소 구조로 블록 QC-LDPC 메모리 모델과 무관 |
| LDPC-CC 부호구조 | 대응 없음 | Prime ECC는 binary QC-LDPC-BC 고정 |

적용 가치: **낮음** — 무선용 LDPC 컨볼루셔널 부호 전용 HW 최적화로, NAND용 블록 QC-LDPC ECC의 부호설계·디코더·메모리 구조와 정합되지 않음.
