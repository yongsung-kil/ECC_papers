### B. 알고리즘 요약

1. 대상: DVB-T2/S2/C2의 직렬연접 BCH(외부)+LDPC(내부) 인코더, frame 16200/64800 bit, 11개 code rate(1/4~9/10) 모두 지원하는 재구성형 HW.
2. 문제: 각 mode마다 BCH 생성다항식·LDPC 패리티행렬이 달라 모든 mode 호환되는 병렬 인코더 설계가 어렵고, 직렬 LFSR은 serial-in/out 병목.
3. BCH 병렬화: remainder register 상태점화식 `R(T+w)=F^w·[R(T)+M(T+w)]`을 수학적 귀납으로 유도, `w`bit 동시처리(실험은 `w=8`, `F^8` 사용).
4. 가변 정정능력: 최대차수 `n_max`로 register/`F`를 설계, 낮은 차수 `g(x)`는 우측 zero-padding `g_new(x)=g(x)·x^(n_max-n)`으로 동일 HW 재사용.
5. BCH는 GF(2^16)(normal)/GF(2^14)(short), 정정능력 `t={8,10,12}`.
6. LDPC: DVB의 IRA(Irregular Repeat-Accumulate) 코드, `H=[Hu Hp]`에서 Hp는 staircase 하삼각.
7. LDPC 저장압축: 정보노드를 M=360 연속비트 disjoint 그룹으로 나눠 첫 노드의 check index만 저장, i번째 비트 index는 `[c_j+(i-1)q] mod (n-k)`로 계산.
8. LDPC 병렬 인코더: qmax=135용 SRAM 하나만 설계, ROM에 code별 index 저장 후 rate로 선택, 8bit씩 GF(2) 누적 후 accumulate로 패리티 계산(식 17).
9. 결과: XC7K325T FPGA DVB-T2 모듈레이터에 통합, 노이즈 주입 시 LBER < 1e-8(TV analyzer 실측), 정정 정상 확인.
10. 한계: 인코더만 다룸(디코더/BER 곡선/gate count/throughput 정량치 없음), DVB IRA 코드 구조 전용.
