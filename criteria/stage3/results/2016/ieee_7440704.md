### B. 알고리즘 요약

1. 시스템: QC protograph LDPC 부호 구성, base matrix `m×n`을 lift size `L`로 copy-and-permute, 예제 `256×512`→`16384×32768`(rate 1/2, N=32768), `6×12`→`3600×7200`(rate 1/2, N=7200), BPSK/AWGN.
2. 문제: 기존엔 PEG로 base matrix 먼저 만든 뒤 permutation shift 결정에 별도 cycle search가 필요 - 부호 길이 증가 시 계산량 급증.
3. 핵심 기법: PEG의 cycle-search·girth 극대화 특성을 활용해 base graph 구성과 permutation shift 결정을 **동시** 수행(PSD-PEG).
4. 핵심 이론: lifted graph의 cycle 길이는 base cycle의 permutation shift 합 `Sρ`와 `L`의 관계로 결정 - `r·(Sρ - p(il,j0)) ≡ 0 mod L`의 최소 `r`에서 길이 `2r(l+1)` cycle 발생(Theorem 3.3).
5. shift 배정 규칙: 새 cycle이 생기는 case에서 `(Sρ - p(il,j0)) ≠ 0 mod L`이 되도록 shift를 골라 lifted girth를 키움(식 (6)).
6. 확장/효율화: subgraph를 depth 0→l로 펼치며 permutation shift 누적 `Sρt`(식 (7))을 계산해 역추적 없이 O(1) 갱신 - 확장 트리가 작아 계산 복잡도↓.
7. 구현 디테일: P 행렬 전체 `∞` 초기화, edge마다 case 1/2는 랜덤 shift, case 3은 식 (6) 만족 shift 배정(Algorithm 1).
8. 학습/최적화: 별도 학습 없음 - greedy PEG + 누적식 기반 결정론적 shift 선택.
9. 결과: Ex1 `dsmax=10`에서 PEG-QC 대비 FER 10^-4에서 약 0.2 dB 이득; Ex2에서 제안 C1이 multiple-edge protograph C2/비최적 C3/PEG C4보다 우수.
10. 한계: BER/FER 시뮬만(AWGN/BPSK), HW·게이트카운트·throughput 없음, error-floor 정량 미제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC base matrix + circulant shift 구성 | ecc_top.cpp Load_PCM() | Prime ECC의 PCM_b(base+shift) 생성 방식과 직접 대응, girth 최적화 shift 배정에 적용 가능 |
| permutation shift 동시 결정(PSD-PEG) | ecc_top.cpp Load_PCM() (H-matrix 로드) | 부호구조 재설계 - 특정 QC 부호 고정이라 재검증 부담 |
| girth 개선 → waterfall 성능 | decoder.cpp LDPC_Decoder() (부호 girth가 수렴 좌우) | 부호구조가 iterative 수렴/정정능력에 기여, 디코더 자체 변경 아님 |
```

마지막 적용 가치: **중간** - QC-LDPC의 large-girth 구성·shift 최적화는 Prime ECC 부호설계 레이어에 직접 닿지만, 프로파일 §4대로 특정 QC 부호가 고정되어 재검증 부담이 크고, girth 이득도 waterfall 0.2 dB 수준으로 제한적.
