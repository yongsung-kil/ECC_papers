### B. 알고리즘 요약

1. 부호: protograph 기반 (3,6)-regular SC-LDPC, base matrix `B=[3,3]`, rate `R=1/2`, AWGN 채널, 스트리밍용 블록 연속출력.
2. 문제: 기존 SC-LDPC 연구는 전체 프레임 BER/FER만 봤으나, 윈도우복호가 블록단위로 출력하므로 블록당 BLER가 관심대상.
3. 구성: edge-spreading으로 `B=B0+...+Bw`(coupling width `w`) 분할 → convolutional base matrix, constraint length `nu = cM(w+1)`, lifting factor `M`.
4. 복호: sliding window decoder(윈도우 span `W`), target `cM` 심볼을 윈도우 이동마다 복호 → latency·memory = 윈도우크기 `cMW`(전체 flooding보다 훨씬 작음).
5. 핵심 관찰1: `nu` 고정하고 `w`↑·`M`↓ 트레이드오프 — 새 edge spreading(w=3,4 [1 1;0 1;1 0;1 1] 등) 제안, `[2 1;1 2]`가 w=1 최적 threshold.
6. 핵심 관찰2: BLER는 `M`보다 `w`(memory)에 더 좋게 스케일 (regular SC-LDPC free distance가 `nu`에 선형증가), BLER 1e-5까지 error floor 미관측.
7. 외부부호: 각 블록을 [n,k,t] 외부 BCH로 보호(hard-decision BDD), case(1) M=dn / (2) M=n / (3) n=pM, 소 `t`로 rate손실 최소화.
8. 외부부호 효과: (4,2) absorbing set이 (3,6) 부호 주 오류원이라 오류블록에 최대 4비트 오류 가능 → t=1/2 외부부호로 error-floor 개선, 외부 CRC는 조기종료 stopping-rule로도 사용 가능.
9. 결과: 동일 블록크기 LDPC-BC 대비 큰 coding gain(N 작을수록↑), (3,6) SC-LDPC threshold 0.46dB vs LDPC-BC 1.11dB; [500,491,1] shortened BCH로 error-floor↓, waterfall 0.079dB 손실.
10. 한계: HW 미설계·시뮬만, 윈도우복호는 초기 수블록 지연·메모리 요구, convolutional 구조라 블록 QC-LDPC와 상이.
