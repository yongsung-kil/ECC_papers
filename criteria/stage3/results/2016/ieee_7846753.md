### B. 알고리즘 요약

1. 시스템: 5G 무선용 HARQ 링크(BS↔UE 양방향), LDPC mother code rate `R=1/2`, packet `k=1152`, IEEE 802.16(2012) QC-LDPC, 코드워드 `n=2304`, Xilinx Kintex-7 FPGA.
2. 문제: 5G PHY는 초고throughput·저latency 요구인데 HDL 기반 개발은 사이클이 길고 VLSI 전문성 필요 → 빠른 실세계 프로토타이핑 필요.
3. 핵심 접근 - HLS: LabVIEW Communications의 HLS 컴파일러가 loop/array/feedback로 쓴 untimed dataflow를 RTL HDL로 자동 변환(timing estimation, pipelining, loop unrolling, memory inference).
4. HARQ Type-I(Chase combining): 실패 packet을 버퍼링해 재전송본과 soft-combine `B_i=B_{i-1}+RV_i` 후 재복호.
5. HARQ Type-II(IR): systematic 부호를 puncturing해 서로 다른 rate의 RV 생성, 재전송마다 parity 증분 결합 `B[σ(j)]=RV_i(j)`.
6. Rate adaptation: [15]의 puncturing 패턴으로 4개 RV(RV0 4/5, RV1 3/4, RV2 2/3, mother 1/2) 생성 (Table I).
7. 서브시스템 분류: bit-manipulation(stateless: LDPC/repetition enc·dec)과 protocol-sensitive(stateful FSM: HARQ controller, RV gen/combiner) → HLS가 양쪽 모두 수정 용이하게 함.
8. LDPC 디코더: 참조[6]의 QC-LDPC 디코더 사용(1x 비파이프라인 337Mb/s, 2x 파이프라인 608Mb/s), 상세 아키텍처는 본문 밖.
9. 결과: FPGA 자원 54% Slice 사용, VHDL 생성 5min, in-HW 시뮬로 CPU 대비 4x 속도(10k frame/40 SNR: 4.28min→1.02min), HARQ로 residual FER 개선(대신 throughput 감소).
10. 한계: 기여가 HLS 개발방법론·시스템 통합에 있고 LDPC 부호/디코더 알고리즘 자체 개선 없음, soft 양자화·gatecount 등 미기재, 5G 무선 타깃.

### C. Prime ECC 관련 모듈 핀포인트

대상이 other(HLS 개발방법론/HARQ 시스템)이므로 C섹션 디코더 성능 항목은 전부 N/A.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 디코더(참조[6], 파이프라인) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` (약함) | 본문에 알고리즘 상세 없음, HW 성능표만 → 이식 정보 없음 |
| Type-II IR puncturing rate adaptation | 대응 없음 | Prime ECC는 고정 rate QC-LDPC, IR-HARQ rate 가변 미대상 |
| HLS 기반 RTL 생성 워크플로 | 대응 없음 | Prime ECC는 bit-exact C++ 시뮬레이터, HLS 개발방법론과 무관 |
