### B. 알고리즘 요약

1. 대상 부호: `(3,6)-regular` LDPC, rate 1/2, codeword length 32~1024 (예: (512,256)), Tanner 그래프 기반.
2. 문제: 표준 반복복호는 연산량이 커 최대복호시간 내 처리 어려움 → OpenCL HLS로 저설계비용 병렬구현 탐색.
3. 알고리즘: `min-sum` belief propagation. VN은 2's complement로 채널측정+3 입력메시지 합산 후 각 입력 빼서 Out 생성(sign-magnitude 변환).
4. CN: parity(전체 sign XOR) + first/second min 탐색. 최소로 선택된 입력엔 2nd min, 나머지엔 1st min 반환.
5. FPGA arch 1(Global Memory interleaver): 2-kernel(VN/CN 분리), off-chip SDRAM 통신 → long stall, 동기화 부담. 조기종료 가능하나 메모리접근 과다.
6. FPGA arch 2(Local Memory interleaver): VN+CN 단일 kernel, single work-group, on-chip block RAM에 interleaver 배치. `n=CN 수` work-item, 각 1 CN + 2 VN 처리.
7. HW 친화 변형: SIMD vectorization(vector len 1/2/4/8/16), branch 제거→bit-mask select, `char`(8bit) LLR가 최적. min 계산은 tournament(comparator tree) 구조.
8. 최적화: vector width↑ 시 throughput↑, 16-codeword vector가 최고. code length 256+ 에서 plateau(pipeline stall 추정).
9. 결과: Stratix V GX A7 FPGA에서 length-1024 (3,6) 부호 5.12Mbps. 동일 코드 GPU 1.28Mbps, CPU 2.35Mbps. 고정 32 iteration.
10. 한계: 알고리즘 신규성 없음, 산업용 디코딩 rate 대비 매우 낮음, custom HW 성능 미달, cross-platform이나 메모리 제약으로 GPU/CPU에서 큰 부호 실행 불가.
