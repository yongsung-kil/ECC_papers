### B. 알고리즘 요약

1. 대상: `(dv,dc)=(3,6)`-regular LDPC ensemble(rate 0.5), memoryless BEC(소거확률 `ϵ`), erasure belief propagation 복호.
2. 풀려는 문제: 로직 게이트·flip-flop이 transient fault를 겪는 fault BP 디코더의 점근 오류확률을 DE로 해석.
3. Fault 모델: BP 메시지 `{0,1,e}`를 message encoder로 이진화(`φ:0→00,1→11,e→01`)해 FF 저장, 각 비트가 확률 `α`로 독립 flip(von Neumann 모델), decoder `ψ`로 복원.
4. 중간노드 조건확률 `Q(x̂|x)` 행렬(Eq.4)로 fault 효과를 포착; VN이 모순(0과 1 동시수신) 시 erasure `e` 출력하도록 규칙 수정.
5. DE 방정식: CN 출력 `q_i(0)= [(p'_i(0)+p'_i(1))^{dc-1}+(p'_i(0)-p'_i(1))^{dc-1}]/2` (Eq.7), VN 출력 `p_{i+1}` (Eq.9,10), 점근오류 `γ(ϵ,α)=lim(1-s_i(0))`.
6. 핵심 관찰: `α>0`이면 fault-free와 달리 `γ`가 양의 값으로 수렴(0 불가), 각 곡선은 특정 `ϵ`에서 급격한 수직점프(fault BP threshold).
7. 예: `α=10^{-3}`에서 `ϵ=0.36207` 경계로 오류확률이 `>10^{-1}` vs `<10^{-3}`로 분기(비선형 동역학 bifurcation).
8. 개선 1(majority voting): N개 병렬 디코더 다수결, `γ_maj≥(1-s(0))^2` 하한(N배 회로).
9. 개선 2(message encoding): BP 메시지를 길이 `n` 이진코드로 보호(Eq.13,14), `n=4`가 majority voting(N=2)보다 threshold 우수(동일 회로규모 비교, Fig.5).
10. 한계: 무한길이 DE 이론만, HW/실측 없음, BEC·erasure BP 특수 설정으로 NAND(BER·soft) 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| erasure BP 복호 (BEC) | 대응 없음 (Prime ECC는 BER/AWGN·min-sum) | 채널·알고리즘 상이 |
| CN/VN DE 업데이트 | 대응 없음 | erasure 알파벳 `{0,1,e}` DE, 코드 미반영 |
| message encoding(FF 보호) | 대응 없음 | fault-tolerant 인코딩, Prime ECC 신뢰 HW 가정과 무관 |
| majority voting 병렬 디코더 | 대응 없음 | 중복 디코더 다수결, 미채택 |

적용 가치: **낮음** — BEC·erasure BP·불신뢰 FF 대상 DE 이론으로, NAND binary QC-LDPC의 min-sum·soft-read 기반 Prime ECC 3.1과 접점 없음.
