### B. 알고리즘 요약

1. 대상 부호는 1π-rotation LDPC로 `rate=0.5`, `H=[Hp|Hd]`이며 `Hp`는 4m×4m dual-diagonal, `Hd`는 16개 m×m sub-matrix로 구성.
2. 문제: 완전 랜덤 LDPC는 성능은 좋으나 스위칭 네트워크·라우팅으로 회로자원 소모가 큼 → 특수 구조로 자원 절감 필요.
3. 1π-rotation은 단일 permutation vector `πA`로 전체 부호 정의(πB/πC/πD는 πA의 90°/180°/270° 회전), triplet `[m,a,b]=[246,31,154]`로 (1968,984) 부호 생성.
4. 핵심 기법: `HdCd` 곱을 sub-matrix 곱으로 분해하고, permutation을 스위칭 네트워크 대신 ROM/RAM의 **write/read address mapping**으로 구현(complement `m+1-perm` 사용).
5. πA/πB/πC/πD 각각 forward/reverse × complement/original × write/read 규칙으로 매핑(Table 1)하여 스위칭 네트워크 제거 → 회로 크기 감소.
6. 복호는 정규화 min-sum: `Rcv=α·Scv·min|Lnc|` (식3-4), VN 갱신은 BP식(식5-6). scaling `α=1`로 고정.
7. 양자화: `G=|Vref|/|max(Vsig)|`, 별도 scaling `γ=4` 도입, soft input을 6-bit signed로 설정.
8. 아키텍처: 수신 심볼을 8개 RAM slice(각 246×6bit)에 적재, Hp branch는 단순 shifting, Hd branch는 perm mapping, VN은 single/dual/quad-tiered, ping-pong·layer별 스케줄로 버퍼 재사용.
9. 결과: Virtex4 FPGA에서 Logic elements 2034(vs [10] 3237), Memory bits 77597(vs 137579), throughput 31.4 Mb/s, iteration limit 10, QC-LDPC 대비 comparable BER.
10. 한계: 1π-rotation 특수 구조 전용, rate 1/2 고정, throughput 낮음(31.4 Mb/s), gate count 공정 데이터·error-floor 분석 없음.

적용 가치: **낮음** — 1π-rotation(rate 1/2, permutation 회전) 전용 LUT 매핑이며 복호 알고리즘(min-sum)은 이미 보유, 고rate binary QC-LDPC인 Prime ECC와 부호구조가 근본적으로 달라 이식 실익이 없음.
