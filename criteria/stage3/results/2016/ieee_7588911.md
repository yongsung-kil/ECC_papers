### B. 알고리즘 요약

1. 대상: 빅데이터 분산스토리지(Hadoop/HDFS)의 노드 소실 대비 erasure code. `(m,n)` systematic 부호, 예제 rate ≈ `10/14`, base code (kb,nb)=(10,15) 계열, n=14/42/63/84.
2. 문제: RS/MDS 부호는 n 증가 시 복잡도가 `O(n²)`로 증가, LRC/regenerating code도 인코딩 복잡·다중소실 복구비용 큼. 신뢰도를 높이려 n을 키우면 계산·repair traffic이 급증.
3. 관찰: LDPC erasure 부호의 인코딩/복구 비용은 m이 아니라 PCM의 row weight `β`로 결정. 따라서 row weight를 고정하면 n을 키워도 비용 불변.
4. 핵심 기법: `expanCodes` — LDPC convolutional code를 base LDPC block code로부터 유도. Step1 base code 정의(min-distance↑, low row weight `Hb`), Step2 column permutation으로 `Hsb` 생성(데이터/패리티 열 interlacing).
5. 의미: JFZ unwrapping은 row/column weight를 바꾸지 않으므로, 확장해도 single-repair 비용이 base code와 동일 유지.
6. 핵심 기법 2단: Step3 JFZ unwrapping으로 `Hsb=Hu+Hl` 분할(가위 cut, c-b 단위 시프트), Step4 반복인자 `rep`로 `Hrep`를 대각 stacking해 크기 `rep·kb × rep·nb`로 확장(redundancy rate 불변).
7. 복구/인코딩: recovery-equation 기반 복구로 m보다 적은 chunk만 수집. 최대 XOR 연산 `XOR_max{Hrep}=XOR_max{Hb}`, single-repair traffic `β-1`로 rep에 무관하게 일정.
8. 결과(신뢰도): 동일 failure rate에서 n이 클수록 견딜 수 있는 erasure 비율 증가. rep 증가로 신뢰도 향상하며 복잡도/공간효율 유지.
9. 결과(Hadoop 실측): 48 data node HDFS 클러스터, 64MB chunk. RS 대비 인코딩 지연 15.76→8.5분(약 46%↓), 복구 지연 6.37→4.26분(약 33%↓). 같은 family 내 지연 유사.
10. 한계: HW 미설계, erasure(노드 소실) 채널 전제·XOR erasure decoding, soft-decision/BER 성능 없음. base code 개선은 future work.
