### B. 알고리즘 요약

1. 대상: 유한길이 SC-LDPC 부호 (체인길이 `L`, coupling width `W`, component block (N-K)×N), 광통신 지향; 예제 `K/N=1/2`, 길이 ~10000/32640bit, rate 0.4~0.8.
2. 문제: 체인 termination이 rate loss를 유발, 길이를 늘리면 완화되나 구현 복잡도 증가 -> 제한된 길이에서 성능 개선 필요.
3. 아이디어: coupling 강도를 부호 내에서 비균일하게 - component block `(l,w)`마다 서로 다른 regular/irregular degree distribution `v_lw(x)` 부여.
4. 핵심기법: BEC용 수정 density evolution으로 CN/VN 소거확률 전파(식 7,8) 계산, BP threshold `ε_th`를 산출(식 10).
5. 식 의미: `ε_th`는 반복 `I_max`·소거확률 한계 `p_lim`에서 수렴 가능한 최대 초기 소거확률 -> 부호 성능 지표.
6. 최적화: `v_lw(x)` 집합을 individual, `ε_th`를 fitness로 GA 실행; 대칭조건 `v_lw=v_{L-l+1,W-w+1}`로 자유도 축소, `h_lw`는 거의 regular로 고정.
7. 결과 구조: 최적해가 codeword 양끝은 고차 irregular, 중앙(4<=l<=7)은 degree-1 regular인 bathtub형 -> 양끝 신뢰정보를 중앙으로 전파.
8. 부호생성: 최적 degree distribution을 만족하도록 PEG로 실제 H행렬 생성.
9. 결과: BEC/AWGN 시뮬에서 비균일 SC-LDPC가 uniform regular/irregular 대비 waterfall 개선(BER>1e-4까지 우위); 광통신 rate 0.8 비교(C1 len 32640, 평균 bit degree ~4.3, layered decoding 25 iter)에서 요구 Eb/N0 감소.
10. 한계: 순수 부호구성법·시뮬만(HW 없음), SC-LDPC 전용이며 low-BER(<1e-6)에선 slope 완만화로 우위 상실 우려.
