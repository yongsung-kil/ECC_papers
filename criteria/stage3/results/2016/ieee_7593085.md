### B. 알고리즘 요약

1. 대상은 protograph 기반 Raptor-like(PBRL) rate-compatible LDPC, BI-AWGNC, 정보비트 `k=16384`, 설계 rate `8/i` (i∈[10,16], 약 0.5~0.8).
2. 문제: inter-frame/multi-hop 통신에서 디코더가 증분(redundancy) 도착 순서를 예측 못 하며, 순서에 따라 각 rate의 부호가 달라져 성능이 변동함.
3. 목표: 모든 증분 순서에 대해 균일하게 좋은 "universal for any increment ordering(UIO)" 부호 설계.
4. 핵심기법: PBRL protomatrix `[HHRC; HIRC]` 구조에서 IRC(degree-1 노드) 행을 조합해 각 rate 앙상블을 형성, protograph threshold(RCA)를 설계 척도로 사용.
5. 척도의 의미: RCA는 density evolution의 1차원 근사로 각 순서 조합의 threshold를 계산해 순서 간 성능 격차를 정량화.
6. 설계 metric 1: 각 rate에서 모든 순서의 threshold 격차(Max-Min)를 최소화 — `HIRC` 행을 "교환 대칭"이 되도록 구성해 격차 0 dB 달성(식 4).
7. 설계 metric 2: threshold 격차 대신 각 rate 평균 threshold 최소화 → inter-frame에서 조기 복호로 throughput 최대화 (행 추가 시 상위 3 앙상블 유지 탐색, 729개 중 선택).
8. 구현 디테일: quasi-cyclic, 1차 lifting factor 4(PEG), 2차 512(C-PEG), ACE 알고리즘 병용; 시뮬은 full-precision flooding LLR-domain BP 최대 200 iteration.
9. 결과: metric 1은 원 PBRL 대비 최대 0.4 dB threshold 페널티지만 순서 무관 격차를 0.698→0.01 dB로 축소; metric 2는 각 SNR에서 최적 throughput에 근접.
10. 한계: 부호구성/threshold 이론·시뮬만, HW 미설계, 두 metric이 상충, 짧은 블록길이는 향후 과제.
