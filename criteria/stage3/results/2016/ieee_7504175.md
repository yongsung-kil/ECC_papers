> 총평: multi-rate QC-LDPC를 상관제약으로 설계해 cyclic-shift 라우팅과 layered 파이프라인을 단순화한 HW지향 부호설계 논문. 아이디어(인접행 remainder 정합→pipeline overlap, 열별 shift 집합 제한→고정 shift network)는 참고가치가 있으나 부호 자체는 무선용이라 직접 이식은 아니다.

### B. 알고리즘 요약

1. 대상은 codeword 길이 `672`, rate `1/2, 5/8, 3/4, 13/16`의 structured(QC) LDPC, base matrix `mb×nb`, expand factor `z`(예 `z=42`), circulant `P` shift 구조.
2. 문제: 표준(11n/11ad)은 rate마다 base matrix가 독립 → multi-rate 디코더 라우팅 복잡, 또 인접행 원소가 무상관이라 layered 디코딩에서 직렬처리("off-line")로 파이프라인 저해.
3. 이를 해결하려 4개 rate 행렬 간·인접행 간 "상관제약(correlation limitation)"을 부여해 공유 하드웨어를 가능케 함.
4. Feature 2.1 nested cycle-4: rate 3/4 base matrix의 length-4 short cycle이 타 rate에도 50%+ 공유 → cycle-4 제거를 한 행렬에서만 보증하면 전체 적용.
5. Feature 2.2 same-set: 같은 열의 non-`-1` 원소가 최대 4원소 집합에서만 선택 → cyclic shift 값이 열당 4종뿐, banyan 대신 "고정 shift network 4개 + 4:1 선택망"으로 대체.
6. Feature 2.3 girth 정합: systematic bit 열 무게 > parity bit 열 무게 → systematic VN girth ≤ parity VN girth로 성능/설계 균형.
7. Feature 2.4 quasi-same-remainder: 한 열 인접 원소쌍이 대부분 `z`로 나눈 나머지가 같음 → LLR을 remainder별 word로 저장, 인접 layer의 W와 다음 layer R이 다른 word라 동시 수행 → clock 절감.
8. Feature 2.5 첫 원소 0: 각 열 첫 non-`-1` 원소=0 → iteration 종료 즉시 posterior LLR 산출(CRC 조기종료), 인접차분만 shift하면 CNU 후 역 cyclic-shift 생략→intra-vector 라우팅 절반.
9. 결과: 디코딩은 layered normalized min-sum(scale 0.85, 최대 20 iter), BLER은 11ad와 동등~5/8에서 0.1dB 개선. 라우팅 복잡도 banyan 대비 대폭 감소(l=42: complexity 4608→252), throughput 약 20~33%(word 많으면 최대 2배) 향상.
10. 한계: HW는 게이트/MUX 상대비교와 clock 절감 논증뿐(FPGA/ASIC 합성·실측 없음), 대상이 Wi-Fi/5G 무선, NAND 특유 고rate·soft-read 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 상관제약 base matrix / QC 부호구조 설계 | `ecc_top.cpp` `Load_PCM()` (H-matrix 로드) | 부호설계 레이어, 특정 QC-LDPC 고정이라 재설계·재검증 부담 큼(이식난이도 높음) |
| 열별 4원소 same-set → 고정 cyclic-shift + 선택망 | `GT.cpp` `Make_GT_HW()` (MUX 기반 edge/shift 선택) | shift 선택을 MUX로 축소하는 발상이 GT의 edge MUX 테이블과 개념 유사, 참고가치 |
| quasi-same-remainder 인접행 정합 → layered W/R overlap | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프(dual-update 스케줄) | Prime ECC의 parity even/odd dual-update 교대 스케줄과 유사 목적(pipeline stall 완화), 부분 참고 |
| 첫 원소 0 → iteration 종료시 posterior LLR/CRC 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 이미 Partial/Full CRC 조기종료 보유 → 중복 |
| 역 cyclic-shift 생략(인접차분 shift) | `decoder.cpp` (VN/CN 처리) | HW 라우팅 최적화 발상, 우리 z=32 구조 정합 필요(중) |

적용 가치: **중간** — 부호 자체는 무선용(rate 0.5~0.81, N=672)이라 직접 이식 대상은 아니지만, "인접행 remainder 정합으로 layered W/R overlap"·"열별 shift 집합 제한으로 라우팅 축소" 설계 원리는 Prime ECC의 dual-update 스케줄·GT MUX 구조에 참고할 만하다.
