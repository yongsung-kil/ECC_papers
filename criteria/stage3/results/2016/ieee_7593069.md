### B. 알고리즘 요약

1. 대상: 고rate·중간길이 binary QC-LDPC (`R=13/16, N=672, Z=42` / `R=5/6, N=2304, Z=96`), AWGN·BPSK, BP 복호(max iter 50).
2. 문제: 단일가중 circulant(SQC, CPM 1개)은 열당 CPM 수가 VN 최대 degree를 제한 → 고rate에서 좋은 degree 분포 불가.
3. 핵심 아이디어: 하나의 circulant 블록 `h_{j,l}(x)`에 여러 CPM을 겹쳐 넣는 multi-weight circulant(MQC)로 dv를 6/8까지 상향.
4. 짧은 사이클 조건: 두 single-weight circulant(shift `P1,P2`)가 겹쳐 만든 4-cycle은 `P2-P1=Z/2`일 때만 발생(Prop.1) → `Z` 홀수면 4-cycle 없음(Cor.1); 3개 이상 겹치면 girth≤6 → 본 논문은 2겹만 사용.
5. 구성 알고리즘: 원래 EMPEG(error minimization PEG, BEC 기반 block error 최소화)를 MQC 지원하도록 일반화 — protograph에서 VN-CN 간 edge 최대 2개.
6. 절차: 각 VN 소켓에 대해 모든 lift layer `q`의 경로 열거, shift `p`별 `P_LB(G,e^p,E)` 계산, 4-cycle 유발 `p`는 +∞로 폐기, 최소 error인 shift 선택(우측 shift에 맞춰 식(14) GCD 수정).
7. 구현 트릭: 식(1)의 `A(s-d,ℓ)` 항을 재귀적으로 사전계산(matrix `A`,`Atmp` 누적)하여 EMPEG 계산 단순화.
8. 분석: density evolution threshold — MQC-LDPC1 2.474 dB < SQC-LDPC1 2.593 dB (dv=6 덕분); error impulse법으로 Hamming/stopping-set 분포 추정(max iter 10, thr 50).
9. 결과: MQC가 SQC 및 802.11ad/802.16e 대비 waterfall·error-floor 모두 개선; MQC-LDPC1은 802.11ad 대비 BLER 1e-6에서 0.5 dB 이상 coding gain; 최소 stopping-set 크기 11→12 증가.
10. 한계: 순수 부호구성·시뮬만, HW/throughput/gate 미기술, NAND·soft-read 미고려, 802 표준 부호 대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MQC 구성 / H-matrix(base matrix) 생성 | `ecc_top.cpp` `Load_PCM()` | 새 PCM 구조(multi-weight) 로드·재검증 필요, 이식 난이도 높음 |
| 일반화 EMPEG로 부호 구조 설계 | 대응 없음 | 오프라인 부호설계 도구, 런타임 코드 대응 없음 |
| 인코딩 유지(802.16e BDD 구조) | `encoder.cpp` `LDPC_encoder_4KB()` | dual-diagonal 인코딩과 유사(BDD 유지), 구조 정합 확인 필요 |
| BP/sum-product 복호 | `decoder.cpp` `LDPC_Decoder()` | 부호만 교체하면 기존 min-sum 복호 재사용 가능 |

적용 가치: **중간** — 고rate QC-LDPC의 degree/stopping-set 분포를 올려 waterfall·error-floor를 동시에 개선하는 부호설계 기법이라 Prime ECC의 `Load_PCM()` H-matrix 재설계에 참고 가치가 있으나, multi-weight circulant는 우리 z=32·고정 QC 구조·HCU 파이프라인과의 정합 및 전면 재검증 부담이 크고 HW·NAND 검증이 없다.
