### B. 알고리즘 요약

1. 대상: 고throughput 병렬복호를 위한 대girth(g=12) LDPC 부호 구성. 예제 rate 0.5, length 4800, 열가중 `J=3`.
2. 문제: 부분병렬 복호 시 서로 다른 프로세서가 같은 메모리에 접근하는 memory collision이 throughput을 지배함. 대girth 부호는 병렬화가 어려움.
3. 표현: PCM을 `J×K`개의 `L×L` 블록으로 분할(각 블록은 행·열당 nonzero 1개 = regular sparseness), 블록의 nonzero 위치를 접근순서로 해석한 simplified matrix `H` 사용.
4. 충돌회피: 각 행에서 정수(접근순서)가 최대 1회만 등장하는 row uniqueness `H(b,a)≠H(b,a')` 제약으로 프로세서 간 동시 동일메모리 접근을 방지.
5. cycle 조건: 길이 `2n` cycle을 access difference 합 `Σ H^{ai,ai+1}_{bi} = 0`(식 8)으로 표현, girth `g` 만족은 이 합이 0이 되지 않는 부등식으로 환원.
6. 선형화: 모든 cycle-check 부등식을 행렬곱 `PD=C`(C의 각 원소 ≠0, 식 9)로 정식화. `P`는 cycle checking 행렬, `D`는 entry 차분 벡터.
7. 핵심 기법: `P`를 Smith normal form `UPV=E`로 분해해 좌역행렬 `P'=VE'U` 도출, `D`를 독립 열벡터 `B=εP'`의 정수 선형결합 `D=Σ k_i B(i) (mod L)`으로 구성(식 13).
8. rank 제약: `r=w`(full column rank)일 때만 해 존재, `r<w`면 구성 실패로 보고 초기 entry 재배치(재배치는 드물다고 보고).
9. 절차: 첫 블록행·블록열을 cycle 없이 초기화 후, 블록을 순차 추가하며 rank 검증→P' 도출→k 결정→미지 entry 계산(Step1~8) 반복.
10. 결과: AWGN에서 랜덤 QC-LDPC 대비 BER 1e-7에서 약 0.1dB 개선·error floor 하강. 구성 성공률 7.85e-4 vs 랜덤 2.73e-5. HW 미설계·시뮬만.
