> 총평: min-sum LLR 연산 자체를 정수 LUT로 대체해 4bit로 BP 근접(0.1dB, Min-Sum 능가)하는 저복잡도 디코더로, Prime ECC의 min-sum/양자화 레이어를 직접 겨냥한 매력적 대안. 단 오프라인 매핑 생성·error-floor 튜닝·고rate QC 재검증이 이식 관문.

### B. 알고리즘 요약

1. 시스템: BPSK/AWGN에서 regular LDPC(dv,dc), 채널출력을 상호정보 보존 양자화기로 unsigned integer `ych`로 변환, 디코더 메시지도 정수 `Ydec` 사용(`|Ydec|=|Ych|=16`, 4bit).
2. 문제: BP는 CN에서 LLR의 비선형 함수 평가가 필요해 고해상도 실수 데이터패스·HW가 병목.
3. 핵심 기법: 모든 노드연산을 정수 message-mapping `f:(Ydec)^M→Ydec`로 대체, 각 노드는 입력 벡터 `y`를 bit `bM`과의 상호정보가 최대가 되는 출력 정수로 클러스터링.
4. 분해: 전체 매핑을 2-입력 부분연산 `tl=gl(yl)`의 직렬로 분할해 LUT 크기를 `|Ydec|^M`에서 `(M-1)|Ydec|²`로 축소(식 1,2).
5. 핵심 결과: global function `Γ`(CN=parity δ, VN=equality δ)에 상태변수 `xl`를 도입해 cycle-free factor graph(=trellis)로 인수분해(식 5,7,8).
6. 이론 기여: 부분연산의 message `μγl→xl`가 sum-product update rule에서 그대로 유도됨을 보여, discrete density evolution이 SP algorithm과 연결됨을 증명(식 12~14가 Kurkoski [3]와 일치).
7. 압축: `p(xl,yl)`를 classifier `p(tl|yl)`(=`gl`)로 `I(Tl,Xl)` 최대화하며 `p(xl,tl)`로 압축, 매핑을 iteration마다 재귀 생성(식 15,16).
8. 학습/최적화: 매핑은 특정 `σn²` 하나에서 오프라인 1회 생성(Information Bottleneck으로 classifier 구성), 수신기는 채널 양자화기만 실제 noise에 맞춤.
9. 결과: length-104 (3,5) rate-0.4 코드, imax=50, 4bit에서 quantized-BP 대비 0.1dB 손실, Min-Sum 능가; 매핑 생성 σn²가 너무 크면 error-floor 발생.
10. 한계: HW 미설계(정수 arch 주장만), 짧은 랜덤코드·저rate만 시뮬, H의 cycle이 discrete DE에 미모델링되어 σn² 수동튜닝 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 노드연산 전체를 정수 LUT로 대체(디코딩 알고리즘) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | min-sum CN/VN 연산을 message-mapping LUT로 교체하는 대체 디코더 |
| CN/VN 정수 매핑(min-sum 대체) | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` 등 | min-sum magnitude/sign 연산을 LUT lookup으로 치환 |
| iteration별 message-mapping 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 LLR 테이블을 iteration별 LUT로 확장/대체 |
| 채널 상호정보 보존 양자화기 | `channel.cpp` `Set_LLR_Th()` | soft-read 양자화·threshold 설계에 정보이론적 최적 양자화 적용 |
| discrete DE 매핑 생성(오프라인) | `local_opt.cpp` (LLR 자동최적화) | 매핑/테이블 오프라인 생성 파이프라인에 대응 가능 |

적용 가치: **중간** — min-sum+양자화 레이어(decoder.cpp)를 정수 LUT 디코더로 대체해 면적/전력 절감과 waterfall 성능(BP 근접)을 동시에 노릴 수 있어 관심도는 높으나, 고rate QC-LDPC(z=32) 구조에서의 매핑 재설계·LUT 메모리·error-floor 튜닝 재검증 부담이 커 이식성은 중.
