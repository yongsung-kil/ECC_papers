### B. 알고리즘 요약

1. 대상: LDPC 복호(NMSA)의 긴 SW 시뮬 시간을 8코어 TI DSP(TMS320C6678, Keystone C66x, 1GHz)에서 OpenMP로 병렬화. 검증부호는 DVB-S2, length `16200`, rate `8/9`.
2. 문제: LDPC 반복 복호의 SW 시뮬 시간이 매우 길어 다양한 파라미터 검증이 느림. 멀티코어로 가속 필요.
3. 모델: AWGN 채널(평균0, 분산 `σ²`), LLR `λ_i=2y_i/σ²` 초기화.
4. 핵심 기법: Normalized Min-Sum(NMSA) — CN 처리 `R_ji = α·(min|Q_i'j|)·Π sgn(Q_i'j)`, BN 처리 `Q_ij=LLR_i−R_ji`, 소프트출력 `LLR_i=λ_i+ΣR_ji`, syndrome check로 조기종료.
5. 의미: 정규화 계수 `α`로 min-sum의 SPA 대비 성능열화를 보정(HW 복잡도↓).
6. 병렬화: PCM을 non-overlapping strip으로 분할 — check-to-bit는 수평 stripe, bit-to-check는 수직 stripe로 코어별 배분. `#pragma omp parallel for`로 c2b/b2c/syndrome 블록 병렬화.
7. 구현 디테일: extrinsic 메시지는 non-cached shared MSMC 메모리, 상수는 cached L1D. 캐시 coherency 회피 위해 shared var를 non-cached alias에 배치. 상수는 `.L2vars` 섹션 배치·64B align.
8. 최적화: 최대 컴파일러 최적화, UIA로 벤치마크 계측, 코어 1~8 스케일 측정.
9. 결과: 8코어에서 iteration당 6015us→947us, 6.35~6.4x speedup. check node 처리가 bit node보다 확장성 우수. 단 실시간 throughput에는 미달.
10. 한계: HW 미설계, 알고리즘 신규성 없음(NMSA 그대로), SW 시뮬 가속 목적, 정정능력/BER 개선 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Normalized Min-Sum CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 알고리즘은 동일 계열이나 이미 보유(min-sum) — 신규 기여 없음 |
| BN 처리/소프트출력 | decoder.cpp VN_Cal_HD() 등 | 표준 VN 갱신, 이미 보유 |
| PCM strip 병렬 분할 | 대응 없음 | SW 스레드 분할이라 z=32 row 병렬 HW 구조와 무관 |
| non-cached shared 메모리 배치 | 대응 없음 | DSP OpenMP 런타임 특화, RTL과 무관 |
| syndrome 조기종료 | partial_CRC.cpp / full_CRC.cpp | 우리는 CRC 기반 조기종료 사용 — 개념만 유사 |
```

적용 가치: **낮음** — 기여가 멀티코어 DSP SW 병렬화(NMSA는 이미 보유)에 국한되어 NAND용 QC-LDPC RTL/부호설계에 떼어 쓸 요소가 없음. bit-exact SW 시뮬 자체 가속에만 부분 참고 가능.
