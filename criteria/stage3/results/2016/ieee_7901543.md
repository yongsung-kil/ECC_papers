### B. 알고리즘 요약

1. 시스템: time-invariant LDPC convolutional(LDPCC = SC-LDPC 특수계열), semi-infinite parity-check matrix, 각 블록 `Hi`는 `c×a` 이진행렬, rate `R=(a-c)/a`.
2. 문제: LDPCC는 보통 QC-LDPC 블록코드에서 unwrapping으로 만드나 syndrome former constraint length `vs`가 커짐 - encoding/decoding 복잡도가 `vs`에 선형 비례.
3. 핵심 기법: QC 블록코드 제약 없이 syndrome former 행렬 `Hs`를 직접 설계 → 더 작은 constraint length 달성, `Hs↔H(x)` 변환으로 기존 연구와 비교.
4. 핵심 표현: `Hs` 각 행의 1들 간 차이 `δ_{i,j}`와 시작/끝 레벨(`ls=j mod c`, `le=(j+δ) mod c`)로 수치 표현.
5. 핵심 조건: local cycle은 `δ_{i1,j1}±δ_{i2,j2}±...=0`일 때 발생(길이 `2l`), 레벨 정합 규칙으로 short cycle 탐색을 효율화.
6. 이론 하한: girth `g=6`, `g=8`에 대해 minimum constraint length `Lh` 하한 유도(예: `w=2,g=8`이면 `Lh≥2a`, 일반식 (7)/(11)).
7. 검증: `g=6,8`은 exhaustive search, 큰 `g`는 Monte Carlo로 하한 접근 코드 탐색.
8. 결과(구성): 기존 [8] 코드(mh=85, Lh=258)를 mh=38·Lh=117로, [15] 코드(mh=185, Lh=558)를 mh=52·Lh=159로 대폭 축소.
9. 결과(성능): `w=3,a=9,c=3,g=8,vs=1143` LDPCC 코드가 동일 rate(2/3)·length 2304 WiMax 블록 LDPC 대비 BER 10^-5에서 약 0.3dB gain(BPSK AWGN).
10. 한계: 순수 부호구성·이론 하한 위주, HW 미설계, NAND 무관(SC-LDPC convolutional + window 복호), 성능은 소수 예제만.

### C. Prime ECC 관련 모듈 핀포인트

대상이 code-design(순수 구성법·이론)이므로 디코더 성능 섹션은 N/A. 부호구조 자체가 SC-LDPC convolutional로 Prime ECC의 고정 QC-LDPC block과 이질적.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome former 행렬 직접 설계 (SC-LDPC 구성) | 대응 없음 | Prime ECC는 QC-LDPC block(N_b=129, z=32) 고정, convolutional/spatially-coupled 구조 미지원 |
| local cycle(girth) 탐색·constraint length 하한 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` (간접) | girth 최적화 개념은 공유하나 SC-LDPC 전용 δ-표현이라 block QC로 직접 이식 불가 |
| window sliding 복호 | 대응 없음 | Prime은 block 단위 min-sum 복호, convolutional window 디코딩 아님 |

적용 가치: **낮음** - SC-LDPC convolutional 코드의 constraint length 최소화 구성·이론 하한으로, Prime ECC의 고정 QC-LDPC block ECC 파이프라인과 부호구조·복호방식이 근본적으로 달라 이식 가치가 낮음.
