### B. 알고리즘 요약

1. 시스템: `(γ,ρ)`-regular binary LDPC, BSC(crossover `α`), column-weight-3 위주(Tanner (155,64), iRISC(1296,972), LS, Margulis(2640,1320)), rate 0.5~0.75.
2. 문제: hard-decision Gallager A/B·BF는 단순하지만 trapping set 때문에 error-floor가 높음; SPA/FAID는 정정력 좋으나 복잡도 과다.
3. 모델: 변수노드가 이웃 위상을 모르는 것이 오판 원인 → depth-one computation tree `Ti`에서 변수노드끼리 직접 메시지 aggregate.
4. 핵심기법1: unsatisfied check 수 `U(vi)`로 "probably correct" `ψ(vi)` 판정, criterion `Υ(Ti\j)`로 flip 후보 결정, 제한적 flip 규칙(3-step)으로 trapping set을 깸.
5. 핵심식: `Υ(Ti\j)=0`이면 `vi`를 `VI`에서 제외; `γ` unsatisfied면 즉시 flip → 오정정 확률 최소화하며 corrupt만 flip.
6. 핵심기법2: 하이브리드 `D1` = Gallager B + Algorithm 1 병렬 구성(`D'`, `D''`), 시간다중화로 실제 1블록만 구현 가능.
7. 구현 디테일: 모든 연산을 2개의 Boolean 함수 `F`(XOR), `G`(AND)로 표현, 4-iteration cycle(0/3 mod 4에서만 flip)로 표준 parallel message-passing HW에 매핑.
8. 오정정 분석: girth-8 column-weight-3에서 isolation condition 하 모든 weight-3 error 정정, (16,5) trapping set만 예외 (Prop.1).
9. 결과: Tanner(155,64)에서 FER<10^-7에서 SPA 제외 모든 디코더 능가; 여러 부호에서 error-floor 영역 1자릿수 개선; 복잡도 `CA=γρ+7γ-2γ/ρ+1`로 Gallager B의 ~2배.
10. 한계: HW 미설계·시뮬만, hard-1bit/BSC/column-weight-3 특화, 총 122 iteration(latency↑), 정보저장 오버헤드는 복잡도 분석서 제외.
