### B. 알고리즘 요약

1. 대상은 rate 1/2 time-invariant `SC-LDPC`(convolutional) 부호로, 반무한 대각 parity-check 행렬을 window size `W`로 잘라 복호한다(실험: `mcc=23`, `W=35`, `NW=840`, `MW=420`).
2. 문제: 기존 window decoder 비교는 structural latency만 봤고 iteration에 비례하는 processing latency를 무시했다(throughput 직결).
3. 채널/모델: AWGN, LLR 기반 belief propagation(`tanh` sum-product 식 (5)(6)).
4. 핵심 기법: `Imax` 이터레이션을 파이프라인 `Imax` 스테이지로 언롤해 completely parallel화, 각 스테이지가 n번째 이터레이션을 담당하고 코드블록이 매 `Tp`마다 출입.
5. 핵심 난점: 블록부호용 언롤 아키텍처([8])와 달리 window decoder는 `mcc`개 기복호 블록의 memory read가 필요 → 중간(미완) LLR만 활용 가능.
6. Memory Management: 각 스테이지가 `mcc`개 메모리를 pipe register 지연(`Tp`,`2Tp`...)으로 읽는 규칙을 식 (8)로 정식화(target 값 write/read).
7. 확장(Circular Pipeline): 마지막 스테이지의 계산 결과 `y[t+1,t+3]`를 첫 스테이지로 피드백해 채널값 초기화를 대체, 성능 손실 축소.
8. time-invariant 필수(Remark 3): 모든 스테이지의 VN2CN/CN2VN 네트워크가 동일해야 함.
9. 결과: latency가 block번호 의존 (10)에서 W-무관 고정 `Tp` (12)로 감소; BER은 classical WD 대비 고SNR에서 0.1dB 손실, 저SNR 개선 미미(Fig.6).
10. 한계: Verilog 시뮬만(FPGA는 future work), gatecount·throughput 수치 미보고, SC-LDPC 전용·이터레이션당 HW 복제로 면적↑.

| 윈도우 memory read(mcc 블록) 관리 | 대응 없음 | SC-LDPC convolutional 메모리 구조로 QC 블록부호에 대응 없음 |

적용 가치: **낮음** — SC-LDPC convolutional 윈도우 디코더 전용 아키텍처이고 full-tanh sum-product 기반이라, min-sum·QC-LDPC 블록의 Prime ECC와 부호·알고리즘 모두 정합하지 않는다.
