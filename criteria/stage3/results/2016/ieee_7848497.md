### B. 알고리즘 요약

1. 시스템: regular `(N,K)` LDPC, 실험은 code length `816`, rate `1/2`, AWGN 채널, Xilinx ZYNQ zc7z020(dual ARM A9 + Artix-7 FPGA).
2. 문제: LDPC 디코더 HW 설계는 복잡·시간 소요, HLS(SDSoC)로 C/C++에서 HW-SW 협업설계를 빠르게 생성하려 함.
3. 핵심 가정: min-sum 알고리즘(SPA 대비 곱셈을 비교·덧셈으로 근사), max iteration `50`.
4. 핵심 기법 1: 디코딩을 init/CNP/VNP/PC 4단계로 나눠 알고리즘 특성·데이터 의존성에 따라 SW(ARM) 또는 FPGA(PL) 매핑 선택.
5. 핵심 기법 2: FPGA 단계에 loop unrolling(`#pragma HLS unroll factor=24`)과 loop pipelining(`#pragma HLS PIPELINE II=1`) pragma 적용.
6. 부수 트릭: 내부 루프 수동 unroll, function in-lining, 메모리 접근 패턴 변경으로 iteration간 의존성 제거.
7. 부수 트릭2: 전 연산 fixed-point화, 스케일링 팩터 `1/16` 선택(overflow 없이 BER 손실 최소).
8. 결과: 최적화 SDSoC 디코더가 순수 SW 대비 최대 `4.41`배, 비최적 SDSoC 대비 `9.89`배 가속, 저 SNR에서 가속비 큼.
9. 결과2: fixed-point화로 BER 평균 `1.15%` 저하(무시 가능), block RAM 28.9%·FF 36.9%·LUT 37.4% 절감.
10. 한계: FPGA 동작주파수(142.86MHz)가 ARM(666MHz)보다 낮아 데이터 전송 오버헤드 존재, throughput/gate 절대수치 미제시.
