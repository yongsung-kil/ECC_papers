### B. 알고리즘 요약

1. 시스템: (n,k) LDPC 부호화 후 ISI 채널(`y(t) = Σ f_l x(t-l) + w(t)`, AWGN `w(t)`)을 통과, 코드길이 CWL 64/128/512, rate 0.5~0.9.
2. 문제: 기존 IDD(iterative detector-decoder, BCJR+SPA)는 검출/복호 지연이 크고 최적 ML 디코더와 성능 격차 존재.
3. 채널 모델: ISI 채널 impulse response `f=[1 2 1]/√6`, AWGN 부가.
4. 1단계: 일반 Viterbi 알고리즘(VA)으로 trellis 상 최소 유클리드 거리 경로인 MMS(minimum metric sequence)를 산출, survivor 리스트를 초기화.
5. 2단계 핵심: JVDD와 달리 모든 노드가 아닌 **parity check 제약에 관여하는 노드에서만** survivor path를 2개로 분할(splitting), `H`의 각 행(parity check)마다 순차적으로 syndrome(`ĉh_i^T=0`) 검사를 수행하는 multi-pass 구조.
6. 각 parity check 노드는 최초 등장 시 한 번만 split(중복 split 방지), threshold 파라미터로 survivor metric 상한을 제한해 복잡도 관리.
7. 부호 구성: JVDD 전용 VGGDLD(variable-gradient Gaussian distribution linear diagonal) 코드, 파라미터 ρ(대각선 오프셋, 0이 최적)와 τ(대각선 오프셋, 최적화 대상)로 열가중치 wc 균일 구성.
8. 결과: MP-JVDD가 JVDD 대비 survivor 수(복잡도) 대폭 감소, 동일 SNR·rate에서 FER 성능은 JVDD와 유사(성능 손실 없음), IDD 대비 약 2dB 이득.
9. 한계: multi-pass 구조로 인해 알고리즘 latency는 오히려 증가(논문 결론에서 명시), HW 미설계·시뮬레이션뿐, VGGDLD라는 특수 코드 클래스 전제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MP-JVDD trellis 기반 joint detector/decoder | 대응 없음 | min-sum BP 반복 디코더(`decoder.cpp`)와 근본적으로 다른 Viterbi/trellis 계열 알고리즘 |
| VGGDLD 부호 구성(대각선 gradient 구조) | 대응 없음 | QC-LDPC(base+circulant) 구조와 무관한 JVDD 전용 특수 코드 |
| ISI 채널 모델 | 대응 없음 | NAND 채널(AWGN/RBER/fixed-error)과 무관 |

> 적용 가치: 낮음 — trellis/Viterbi 기반 joint detection-decoding으로 ISI 채널(자기기록/무선) 대상이며, NAND binary QC-LDPC의 min-sum BP 디코더 구조와 이식 접점이 없음.
