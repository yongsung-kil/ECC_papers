### B. 알고리즘 요약

1. 대상: self-orthogonal codes (SOC), 예 `g(x)=1+X+X^4+X^6`, `R=1/2`, `d=5`, `n=26`; 시뮬은 `n=20748,d=9`와 branch 할당 SOC `n=31824,R=8/16,d=17`.
2. 문제: SOC의 표준 복호기 MTD(multithreshold decoder)는 선형복잡도지만 coding gain이 min-sum보다 ~1~1.5dB 낮음.
3. 관찰: SOC의 parity-check matrix `H=[P:I]`가 low-density → LDPC용 min-sum decoder 적용 가능(Tanner graph).
4. 핵심기법: 결합 디코더 — 초기 몇 iteration은 min-sum(고잡음 내성), 이후 MTD로 전환.
5. min-sum이 첫 iteration에서 높은 노이즈를 다루므로 MTD 단독보다 낮은 SNR에서 수렴 시작.
6. 복잡도 식: MTD `Nmtd≈(l+1)(d+2)`, min-sum `Nms=l(8d-6)+(d-1)` per bit → min-sum이 MTD의 6~7배 연산.
7. branch 할당 SOC에서는 dimension 큰 branch 8 check를 초기·전 iteration에서 제외해 복잡도 추가 절감.
8. 학습/최적화 없음 (고정 스케줄, min-sum iteration 수만 튜닝).
9. 결과: `n=20748` SOC에서 min-sum(9)+MTD(30)이 MTD 대비 ~0.75dB gain(복잡도 2배); `n=31824`에서 min-sum(9)+MTD(26)이 최대 1.1dB gain(복잡도 2~3배), min-sum 단독 대비 0.3~0.4dB 손실.
10. 한계: AWGN/BPSK 무선 채널 가정, HW 미설계, 복잡도 정성비율뿐, error-floor/iteration 감소 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum 복호 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 min-sum 보유 — 중복 기여 |
| SOC/MTD 하이브리드 스케줄 | 대응 없음 | MTD는 shift-register majority 복호로 QC-LDPC 이터레이션과 무관 |
| SOC 부호구조(H=[P:I]) | 대응 없음 | 자기직교 convolutional 부호로 binary QC-LDPC 아님 |

적용 가치: **낮음** — min-sum은 이미 보유했고, 나머지는 무선통신용 SOC/MTD 하이브리드로 NAND QC-LDPC 디코더에 이식할 신규 요소가 없다.
