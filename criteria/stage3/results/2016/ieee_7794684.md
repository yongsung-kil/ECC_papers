### B. 알고리즘 요약

1. 시스템: rate-compatible punctured LDPC, mother code regular `(3,6)` rate `R0=1/2`, 목표 `R1=2/3`, AWGN/BPSK, soft BP; 예 n=360→n1=270, n=1620→n1=1215.
2. 문제: rate 유연성(adaptive/IR-HARQ)을 encoder/decoder 쌍 증설 없이 puncturing으로 얻되, 어떤 bit를 puncture할지 패턴 설계가 핵심.
3. 모델: puncture = Tanner graph에서 variable node 제거 = degree-k bit가 관여한 k개 check 방정식을 modulo-2로 소거해 equivalent punctured H 도출; 디코더는 punctured LLR을 0으로 초기화.
4. 핵심 기법 - DE의 Gaussian approximation(Chung et al.[8]): 메시지 밀도를 Gaussian으로 근사, degree-i variable node 평균 `m_{v,i}=m_{u0}+(i-1)m_u`.
5. mean update `t^(l)=f(s,t^(l-1))`, 여기서 `f(s,t)=Σρ_j f_j(s,t)`, `φ(x)`는 tanh 룰의 단조감소 함수.
6. 핵심 식: 소오류(큰 t)에서 `Δt=f(s,t)-t ≈ s + (i1-2)t - 4logλ_{i1} - 4Σρ_j log(j-1) + O(t^{-1})` — stability 조건과 연결, error-floor 거동 좌우.
7. 설계 기준: `Δt` 최대화 → i1(첫 비영 λ 지수) 우선 최대화, 동률이면 `4logλ_{i1}+4Σρ_j log(j-1)` 최소화 (품질평가 함수).
8. 패턴 생성(genetic/greedy): NP개 랜덤 패턴 생성→equivalent H·degree분포 계산→품질함수 평가→상위 NS 선택→자주 1인 NB 위치를 유지(L)하며 목표 weight까지 progressive 반복.
9. 결과: 제안 puncturing이 랜덤 puncturing 대비 coding gain 우위, native rate-2/3 부호에 근접 (BER/BLER 곡선은 그림만, 수치 없음); 고SNR에서 이득 큼(식(6)이 저오류 근사이므로).
10. 한계: HW 미설계, 무선/IR-HARQ 지향, 강한 구조 부호(802.11n)엔 랜덤 대비 개선 극미, non-QC/저rate 검증뿐.
