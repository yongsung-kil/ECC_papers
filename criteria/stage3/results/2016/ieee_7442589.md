### B. 알고리즘 요약

1. 대상: GPGPU(NVIDIA GTX 470, Fermi)에서 regular/irregular LDPC를 모두 처리하는 통합 병렬 디코딩 설계. 부호는 IEEE 802.15.3c `(672,336)`, 802.11n `(1944,972)`, 합성 `(1024,512)`/`(4896,2448)` 등 통신표준 행렬.
2. 문제: 기존 GPGPU 설계는 특정 LDPC 행렬(regular 또는 QC)에 커스터마이즈되어, 다른 행렬 지원 시 재작업/재설계 필요.
3. 스케줄링: `TPMP`(Two-Phase Message-Passing) 채택 — CN 커널/BN 커널을 분리해 phase마다 한 번씩 수행, codeword당·codeword간 병렬성 노출.
4. 기법1(node-based, NBTGL): 노드 1개당 스레드 1개. `Address Translation Array(ATA)` = `LxcBegin/LxcSize/LxcStripe` 3배열로 노드별 edge 수가 달라도 유연 인덱싱, 접근을 coalesce.
5. 최적화: `Node Grouping` — edge 수가 같은 노드를 정렬·클러스터링해 warp 내 스레드가 동일 edge 수를 갖게 하여 memory coalescing 향상.
6. 기법2(edge-based, EBS): edge 1개당 스레드 1개. `NextLxcIndex` 배열로 같은 노드 edge를 연결, 노드보다 edge가 많아 concurrent codeword가 적을 때 병렬성↑.
7. 구현 디테일: EBS는 공유데이터를 Fermi shared memory(16KB)에 적재(`Lxc_SharedMem` 등), 같은 노드의 모든 edge를 동일 thread block에 배치.
8. 데이터 footprint: node-based는 스레드당 12원소 접근, edge-based는 6원소로 축소(단 스레드 수는 2배 이상).
9. 결과: CPU 대비 최대 134.56x 속도향상, 최대 throughput 80.25 Mbps(256 concurrent codeword). concurrent codeword 적으면 EBS가 44~14% 우세, 많으면 NBTGL 우세.
10. 한계: 정정능력(BER) 개선 없음(고정 iteration 비교), HW(ASIC/RTL) 미설계, single-precision float, host-device memcpy가 전체시간 40~52% 차지.
