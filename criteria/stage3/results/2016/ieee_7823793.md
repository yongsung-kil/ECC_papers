### B. 알고리즘 요약

1. 대상: 2-bit MLC NAND SSD. ECC로 BCH(baseline)와 LDPC(strong)를 혼용, 부호 rate/length·디코더 구조는 논문에서 정의하지 않음(블랙박스).
2. 문제: ISPP의 step voltage `ΔVpp`를 키우면 write는 빨라지나 RBER↑ → 강한 ECC(LDPC) 필요, 그러나 LDPC read latency는 BCH 대비 ~120% 증가해 read 성능 저하.
3. 핵심 아이디어: logical page를 write-only / read-only / overlapped 3분류하고 페이지별로 write step과 ECC(BCH/LDPC)를 차등 적용.
4. 분리기: multiple bloom filter를 read/write BF pair로 확장(4 pair, 2 hash, BF당 2048bit, decay 512req, recency weight 2/1.5/1/0.5)해 read/write hotness 계산.
5. 규칙: write-only는 low-cost write+LDPC, overlapped는 `write_hotness - read_hotness > m`이면 LDPC 아니면 BCH.
6. rewrite: LDPC 페이지의 read hotness가 write hotness보다 크고 차이가 `n` 초과면 normal-cost write+BCH로 재기록해 read latency 회복.
7. 파라미터 탐색: `m`은 -4~4에서 -1이 최적, `n`은 0~4에서 3이 최적.
8. 오버헤드: 블룸필터 2KB(용량 무관), ECC-mode bit 페이지당 1bit(64GB/8KB → 1MB), firmware 부담 미미.
9. 결과: Disksim SSD 확장 + 15개 실 트레이스(MSR/OLTP)에서 write/read 응답시간 평균 48%/11% 감소, AGCR 대비 write 4%·read 11% 개선, erase 증가 <2%.
10. 한계: 순수 시뮬(Disksim), latency는 표 상수값 가정([16] 기반 추정), LDPC 부호/디코더/HW 설계 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC/BCH 차등 적용 스케줄링(블룸필터, FTL) | 대응 없음 | Prime ECC는 부호/디코더 bit-exact 시뮬 - FTL/워크로드 계층 없음 |
| ISPP write step vs RBER 트레이드오프 | 대응 없음 | 채널 RBER 주입은 있으나 write step 정책은 범위 밖 |
| LDPC read latency 상수 가정 | 대응 없음 | 디코더 latency 실측이 아니라 표 상수 |

적용 가치: 낮음 - LDPC를 블랙박스로만 다루는 SSD 스케줄링 논문으로 Prime ECC의 부호설계/디코더/HW 어느 레이어에도 이식할 알고리즘 요소가 없음.
