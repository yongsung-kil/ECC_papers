### B. 알고리즘 요약

1. protograph LDPC 코드는 base matrix `B` (|C|×|V|, 원소는 다중edge 개수)로 정의되며, copy-and-permute(lifting) 연산으로 임의 길이의 코드로 확장됨. rate는 `1 - |C|/|V|`.
2. 기존 연구는 bit-error threshold(무한장 asymptotic) 위주였고, degree-2 bit node가 포함된 protograph는 block-error threshold 보증이 없었으며, 랜덤 샘플링 대신 결정론적 구성이 부재했던 문제를 해결.
3. 채널 모델: binary erasure channel(BEC, 파라미터 `ε`)과 binary-input symmetric channel(BIAWGN 포함, Bhattacharyya parameter `B_t` 사용).
4. 핵심 이론(Theorem 1): protograph의 degree-2 서브그래프가 cycle-free이고 모든 degree-2 변수노드가 degree≥3 노드에 연결되면, edge별 erasure 확률 `x_t(i) = O(exp(-β2^(αt)))`로 double-exponential 감소.
5. 이 성질로 large-girth(`girth ~ c log n`) lifted graph 수열에서 block-error 확률 `P_B(n) = O(n·exp(-βn^(cα)))`가 되어, block-error threshold = bit-error threshold가 성립(기존엔 미보증).
6. 안정성 조건(Theorem 2): gradient matrix `∇f`의 Frobenius 정규형 고유값 분석으로, degree-2 서브그래프 구조(cycle-free/비중첩/최대 edge수 `r_max`)에 따라 안정 영역 `ε < 1`, `ε < 1/r_max` 등을 도출.
7. 구현 트릭: König 정리 기반 edge-coloring + node-splitting으로 임의 |E|-regular large-girth 그래프(LPS graph `X_{p,q}`, `D(m,q)` graph)를 protograph에 lifting해 girth를 보존하며 결정론적 구성.
8. 최적화: differential evolution(mutation/crossover/selection, `pc=0.88`, 최대 6000세대)으로 base matrix 탐색, degree-2 노드 유발 cycle은 crossover 시 강제 재배정하여 tree 구조 유지.
9. 결과: rate-1/2 4×8 protograph에서 BEC threshold 0.479(용량 근접), 16×32에서 0.4952(rate-1/2 용량 0.5 대비), BIAWGN에서도 AR4JA/DVB-S2/WIMAX 대비 우수한 threshold; 시뮬레이션(길이 4056~90000)에서 표준 message-passing 디코딩으로 우수한 BER/BLER 곡선 확인.
10. 한계: 순수 asymptotic/threshold 이론 및 무한장 가정 기반 구성이며, 실제 디코더 근사(min-sum 등)·HW 구현·복잡도·NAND 특화 채널(비대칭 오류, soft-read) 분석이 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph base matrix / large-girth 결정론적 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC H-matrix가 고정된 구조이며, 이 논문의 protograph 자체 설계 이론을 이식하려면 H-matrix 전면 재설계·재검증이 필요(난이도 높음) |
| Density Evolution 기반 block-error threshold 이론 | 대응 없음 | Prime ECC 프로파일에 asymptotic threshold 설계/분석 도구 자체가 없음 |
| BEC/BIAWGN 채널 모델, 표준 message-passing(tanh) 디코딩 | `channel.cpp`, `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 AWGN/RBER/fixed-error 채널과 min-sum 디코더만 지원하며, 이 논문은 BEC 위주 이론 분석 + 표준 sum-product 시뮬로 직접 대응은 약함 |

적용 가치: 낮음 — protograph의 block-error threshold를 보증하는 결정론적 대형-girth 구성 이론이며, 표준 message-passing 시뮬 검증뿐이라 min-sum HW 디코더나 QC-LDPC 고정 구조인 Prime ECC와의 접점이 거의 없음.
