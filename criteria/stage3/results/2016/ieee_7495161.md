### B. 알고리즘 요약

1. 시스템: IEEE 802.3an RS-LDPC `(2048,1723)` rate-0.84 binary code, puncturing으로 rate를 최대 0.907까지 상향, 완전병렬 FPGA/ASIC 디코더.
2. 문제: hard-decision BF 디코더는 zero-reliability(punctured) 심볼을 다루지 못해 puncturing 하에서 심한 error floor 발생.
3. 모델: punctured 위치는 디코더가 알고 있고 erasure `e`로 초기화, check node 하나에 punctured bit이 최소가 되도록 위치 선정.
4. 핵심 기법1(ED 전처리): erasure decoding으로 단일 erasure를 갖는 check에서 even parity로 해당 bit 확정, majority로 해소, 최대 2 iteration.
5. 핵심 식/의미: check node 추정 `El(j,k)=XOR_{i in Mj\k} d_i` - 단일 erasure면 나머지 bit의 짝수 패리티를 만족하는 값으로 복원.
6. 핵심 기법2(ED->BF 결합): ED 출력을 BF(Gallager B, `delta_k=floor(Dv/2)`) 입력으로 주고, reliability-aware BF에는 해소된 erasure에 사전정의 신뢰도 `R` 부여(R은 시뮬로 결정).
7. 구현 트릭: bit당 3값(0/1/e)을 2-bit("00/01/10")로 인코딩, half-adder/OR/XOR 트리로 노드 구성; puncturing 후 최대 rate 0.907 가정 시 variable node를 punctured bit용 150개만 인스턴스화해 복잡도 절감.
8. HW 공유(SHEDBF): ED와 BF가 동일 네트워크·유사 노드를 써서 MUX로 두 모드 공유, BF 노드 HW를 ED 자원에 흡수(Common CN/VN).
9. 결과: (2048,1723)에서 BER 1e-5 총 throughput 48.42(EDBF)/50.21(SHEDBF) Gbps, avg 2.27 iteration, SHEDBF 4.46mm²@90nm·809K NAND gate로 선행 min-sum 대비 면적 최대 2.1배 작고 throughput 우위.
10. 한계: 통신 표준 부호 대상 hard-decision BF, NAND soft-read 트랙과 상이; optimal R 미확정, BER 곡선은 그림에만 있어 정량 floor 수치 본문 부재.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BF/Gallager-B iterative 복호 루프 | decoder.cpp LDPC_Decoder(), C2V_Cal() | Prime ECC는 min-sum 기반 - BF는 성격 다른 대체 알고리즘, 직접 병합보다 참고 |
| CN parity/XOR 연산 | decoder.cpp C2V_Cal_New_Sgn() | BF check node의 XOR 트리와 개념 유사(sign/parity) |
| VN majority/flip 갱신 | decoder.cpp VN_Cal_HD() | hard-decision VN 갱신과 대응, 양자화 없이 majority |
| erasure(punctured) 전처리 | 대응 없음 | Prime ECC punct_col은 존재하나 별도 erasure decoder 전처리 단계 없음 |
| 완전병렬 저면적 HW 공유(SHEDBF) | GT.cpp Make_GT_HW() | edge/자원 축소 아이디어와 목적 유사(면적 절감), 구조는 다름 |
```

적용 가치: 중간 - binary LDPC 디코더의 rate 유연성(puncturing)과 저면적 완전병렬 아이디어는 유용하나, Prime ECC의 soft min-sum 파이프라인과 계열이 달라 직접 이식보다는 puncturing 지원·erasure 전처리 개념 참고에 가치.
