### B. 알고리즘 요약

1. 대상은 protograph 기반 SC-LDPC 부호(base protograph를 `L`번 복제·coupling, coupling width `w`), 예시는 block length 32000, rate 0.45.
2. 문제: SC-LDPC은 threshold saturation으로 capacity에 근접하나 유한 길이에서는 trapping set으로 error floor가 남으며, 특히 windowed decoder 기준으로 해로운 trapping set을 제거해야 한다.
3. 모델: (a,b)-trapping set(변수노드 a개, 홀수차수 체크노드 b개), critical number(최소 inducing set 크기), spread(σ2(v)>σ1(v)인 변수노드 수)로 harmfulness 정의.
4. 핵심기법(Algorithm I): base graph의 trapping set 우선순위 리스트 `P`를 받아, 해로운 subgraph의 edge를 뒤쪽 position 복사본으로 spread하고, 같은 체크노드 공유 edge를 freeze하여 later position에서 trapping set 재구성을 방지.
5. 핵심 아이디어: cycle에 포함된 edge를 우선 spread하면 girth 개선과 다수 trapping set 동시 제거 효과가 크다.
6. 확장(Ranking): trapping set의 parent/child 관계로 poset `P(G)` 구성 — parent를 제거하면 down-set(children) 전부 제거되므로 parent 우선 제거가 더 효율적.
7. 구현 디테일: down-set 격자 `J(P(G))`에서 maximal element를 제거하여 원하는 critical number 이하 trapping set 집합을 최소 개입으로 제거.
8. 우선순위 리스트는 (a)critical number, (b)spread, (c)poset maximal element, (d)join down-set의 maximal element 중 선택.
9. 결과: BSC/Gallager A, rate 0.45, length 32000, `L=20`, lifting `m=100`에서 Algorithm I이 random edge-spreading 대비 error floor를 2자릿수 이상 개선.
10. 한계: HW 미설계, BSC/Gallager A 시뮬만, SC-LDPC/windowed decoder 특화 구성 이론, lifting 최적화는 후속 논문으로 미룸.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-protograph edge-spreading 부호 구성 | `ecc_top.cpp` `Load_PCM()` (H-matrix/PCM 로드) | 낮음: Prime ECC는 고정 block QC-LDPC를 로드만 하며 SC/protograph 구성 파이프라인 없음 |
| trapping set 기반 error-floor 대책 | 대응 없음 | Prime ECC에 부호설계 단계의 trapping set 제거 로직 부재 |
| windowed decoder | 대응 없음 | Prime ECC는 전체 block 디코딩(min-sum, HCU, pipeline)으로 sliding window 미지원 |

적용 가치: **낮음** — SC-LDPC convolutional + windowed decoder 전용 부호구성 이론으로, 고정 block binary QC-LDPC인 Prime ECC의 부호구조·디코딩 방식과 정합되지 않는다.
