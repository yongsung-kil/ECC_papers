### B. 알고리즘 요약

1. 시스템: SC-LDPC 앙상블 `(l,r,L)=(3,6,L)`, base matrix를 lifting factor `z`로 확장, 채널은 BEC(erasure probability `ε`).
2. 문제: SC-LDPC는 uncoupled block LDPC 대비 code rate가 `ΔR=1/(2L)`만큼 손실(rate-loss)되며, 이를 줄이려 추가 variable node를 경계 check node에 붙이면 finite-length BLER이 오히려 저하됨.
3. 핵심 관찰: 기존 연구([11])는 BP threshold 불변 제약만으로 degree distribution `λ(x)`를 최적화했으나, 점근 성능(BP threshold)이 같아도 유한길이 성능은 저하될 수 있음.
4. 핵심 기법 1단: 추가 variable node 전용 지표인 local BLER `P_λ`와 local threshold `ε_λ`를 정의하고, scaling law(`P≈Q(√M·α(ε_SC-ε)/√ε(1-ε))` 형태, 식(7))로 목표 local BLER `P*=(k-1)P*/(1-P*)`(식 8)를 도출.
5. 의미: local BLER이 main region BLER보다 충분히 작아야(`P_λ(x) ≪ P_(3,6,L)`) 전체 finite-length 성능이 유지됨(식 6).
6. 핵심 기법 2단: differential evolution(Algorithm 1)으로 local threshold 제약을 만족하는 `λ(x)`를 탐색하고, expected graph evolution 기반 `r1(l)(ε)`의 초기 구간 local minimum을 금지하는 "local minimum constraint"를 추가해 trade-off 개선.
7. 부수 트릭: `λ(x)` 대신 protograph 표현 base matrix `B`(degree-1 variable node 포함 precoding + puncturing)로 확장(Algorithm 2)하여 하드웨어 친화적 구조 확보, 목표 rate는 puncturing fraction `β`로 미세 조정.
8. 최적화 절차: differential evolution(DE/rand/1/bin, per-vector dither) 2종(λ(x)용, B용) 반복 수행, 목표 rate·local threshold별로 별도 최적화.
9. 결과: (3,6,L,B) 앙상블은 rate-loss를 53% 이상(`γ>0.53`) 완화하면서 (3,6,L) 대비 BLER 성능 저하 없음(동일 rate 비교 시 오히려 성능 개선, Fig. 8·10).
10. 한계: BEC 채널·windowed BP 시뮬레이션 및 이론적 scaling law 분석뿐이며 HW 구현·복잡도·throughput 분석 없음. SC-LDPC 고유 boundary 구조에 특화되어 일반 QC-LDPC 블록 코드로 직접 확장 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호구성(추가 variable node degree distribution, protograph base matrix 최적화) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | Prime ECC는 고정 QC-LDPC H-matrix이며 SC-LDPC 자체를 채택하지 않아 구조적으로 대응 없음 |
| windowed BP decoding / DE 기반 이터레이션 | 대응 없음 | SC-LDPC 전용 슬라이딩 윈도우 복호로 Prime ECC의 iterative min-sum 구조와 무관 |
| puncturing을 통한 code rate 매칭 | 대응 없음 | Prime ECC는 인코더(`encoder.cpp`)가 dual-diagonal 고정, puncturing 기법 없음 |
| BEC 채널 모델 | [channel.cpp](../Prime_ECC_3.1_Claude) | Prime ECC는 AWGN/RBER/fixed-error 채널만 지원, BEC 이론 분석 결과는 직접 이식 불가 |

적용 가치: **낮음** — SC-LDPC 고유의 rate-loss 완화용 부호구성법으로, binary QC-LDPC 고정 구조의 Prime ECC 3.1에는 부호 패러다임 자체가 달라 이식할 레이어가 없다.
