### B. 알고리즘 요약

1. 대상: IEEE 802.3an(10GBASE-T)의 RS 기반 (2048,1723) regular LDPC(VN degree 6, PN degree 32, rate≈0.841), 완전병렬 stochastic 디코더 ASIC.
2. 문제: stochastic 디코딩은 각 VN 나가는 edge마다 재랜덤화 유닛(EM/TFM)이 필요해 유닛 수=graph edge 수(12288개)라 면적이 병목.
3. 모델: 채널확률을 Bernoulli 스트림으로 변환, VN=SPA equality, PN=XOR parity를 비트직렬로 수행. hold(입력 불일치)/nonhold 상태로 latch-up 방지.
4. 핵심 기법: TFM은 successive relaxation으로 스트림 이동평균확률 `p`를 추정(`(1)`: nonhold 시 `p(t+1)=(1-β)p(t)+β·b`, β=relaxation coef).
5. MTFM: VN당 edge별 TFM 대신 **VN당 MTFM 1개**만 두고, 나가는 regenerative bit들의 **다수결**을 입력으로 사용 → (2048,1723)에서 재랜덤화 유닛 12288→2048개로 축소.
6. MTFM 갱신은 regenerative bit 수가 threshold `Ts` 초과 시에만, majority threshold `Tm`(보통 절반)로 출력 결정. degree≥4에서 다수결 근사 정확.
7. HW 트릭: β를 2의 음승으로 두어 곱셈→시프트, adder/subtractor 1개로 축소한 reduced-complexity TFM/MTFM. bit-serial·counter-based 근사 구조도 제시.
8. 시스템: 6-bit 입력→7-bit 확률(LUT), 11-bit MTFM, flooding schedule, split-row 방식 32 VN block 파티션, LFSR 분산 난수엔진, syndrome 조기종료(최대 400 clock).
9. 결과: 90nm CMOS 6.38mm2, 500MHz, 최대 61.3Gb/s, BER 4e-13@5.15dB, error-floor 미관측(down to 1e-13). MTFM VN 면적은 64-bit EM VN의 19%. redecoding+postprocessing으로 (8,8) absorbing set error-floor 억제.
10. 한계: stochastic 비트직렬 패러다임(min-sum/SPA 아님), soft-4bit+ 입력, latency 800ns로 타 (2048,1723) 디코더보다 큼, NAND 무관(10GbE 이더넷).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stochastic VN/PN 비트직렬 디코딩 | 대응 없음 (Prime ECC는 min-sum message-passing) | 패러다임 상이, 재사용 불가 |
| syndrome 기반 조기종료 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/) / [full_CRC.cpp](../Prime_ECC_3.1_Claude/) | 우리는 CRC 조기종료 보유 — 개념만 유사 |
| redecoding/postprocessing error-floor 대책 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` 루프 (개념 참고) | (8,8) absorbing set 억제 아이디어만 참고 |
| split-row 파티션·비동기 파이프라인 라우팅 | 대응 없음 (우리는 z=32·GT·pipeline 6-stage) | 라우팅 완화 개념만 참고 |

적용 가치: **낮음** — stochastic 디코딩은 Prime ECC의 min-sum message-passing과 근본적으로 다른 계산 패러다임이며 soft-4bit+ 입력·이더넷 부호 타깃이라, error-floor 억제용 redecoding 개념 정도만 간접 참고 가능하고 직접 이식은 불가하다.
