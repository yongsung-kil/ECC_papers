### B. 알고리즘 요약

1. 5G NR eMBB 시나리오의 데이터채널(QC-LDPC, BG1/BG2, lifting Z=2~384, rate 1/3~8/9 및 1/5~2/3)과 제어채널(Polar, N=2^7~2^9)을 하나의 HW 디코더로 통합.
2. 기존 멀티모드 디코더는 처리량과 면적 효율 사이 트레이드오프가 심하고 통합 방법론이 부재하다는 문제를 해결.
3. LDPC/Polar 모두 factor-graph 기반 BP(belief propagation) 디코딩이라는 공통점을 가정하고, `degree`(노드 복잡도)와 `parallelism`(인스턴스 수)이라는 두 속성으로 두 코드의 그래프 구조를 통일하는 하드웨어 서술언어(HSPL) 표현을 사용.
4. 핵심 기법 1단: unified processing node(UCPE/UVPE) — DeCOMU, configurable adder, MSCU(min-sum computation unit), FIFO, adder stack 5개 서브블록으로 LDPC CN/VN 연산(offset min-sum `φ(β)=sgn·max(min|β|-θ,0)`)과 Polar L/R 전파 연산을 공유 로직으로 재구성.
5. MSCU가 mode 신호(0=LDPC, 1=Polar)에 따라 BAR/CROSS 스위치 상태를 바꿔 동일 비교기 하드웨어를 두 코드 모두에 재사용하는 것이 핵심 재사용 트릭.
6. 핵심 기법 2단: 512×512 combined Banyan network 기반 configurable permutation network로 LDPC의 circular shift와 Polar의 shuffle/reverse-shuffle L/R propagation을 모두 지원, control signal을 modulo 연산 없이 비트 시프트/절단으로 생성해 critical path 단축.
7. 부수 트릭: multi-frame processing으로 노드 병렬도 `P=max(Zmax, Nmax/2)=384`를 채널별 병목 없이 활용, 6-bit 고정소수점 양자화(2 fractional bit) 공용.
8. 별도 학습/최적화 절차 없음 — 결정론적 HW 아키텍처 설계 및 스케줄링 규칙(메모리 접근 충돌셋 `ρ`) 기반.
9. 40nm CMOS 합성 결과 1.64mm² 면적, 500MHz, 135.83mW, LDPC모드 21.87Gbps/Polar모드 10.24Gbps 처리량으로 기존 SOA 멀티모드 디코더 대비 최대 33.64배 처리량, 5.98배 면적효율 달성.
10. 정정능력/오류마루 개선은 다루지 않음(표준 OMS 알고리즘 그대로 사용), Polar 코드 지원 로직이 구조의 상당 부분을 차지해 순수 binary QC-LDPC 전용 코드베이스에는 기여분 재사용성이 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC CN offset min-sum 연산(MSCU) | `decoder.cpp` `CNU_Update_New_Mag()` | 동일 계열(min-sum 근사)이나 이미 보유, Polar 겸용 스위치 로직은 불필요 |
| block-parallel layer 디코딩 스케줄링 | `decoder.cpp` `LDPC_Decoder()` | layer 단위 처리 자체는 Prime ECC와 유사하나 5G BG1/BG2 파라미터화 로직 특화 |
| configurable Banyan permutation network(circular shift) | 대응 없음 | Prime ECC는 QC-LDPC 고정 shift 구조(z_sb=32)로 범용 network 불필요 |
| Polar L/R propagation, shuffle shift | 대응 없음 | Prime ECC는 binary LDPC 전용, Polar 코드 미지원 |
| 6-bit soft quantization | `channel.cpp` `Set_LLR_Th()` | 이미 2SD/3SD 등 자체 양자화 테이블 보유, 참고 수준 |

> 적용 가치: 낮음 — LDPC/Polar 멀티모드 통합 및 HW 면적·처리량 효율화가 핵심 기여이나 Prime ECC는 단일 QC-LDPC 전용 구조로 멀티코드 통합 니즈가 없고, 디코딩 알고리즘 자체는 표준 OMS 재사용 수준.
