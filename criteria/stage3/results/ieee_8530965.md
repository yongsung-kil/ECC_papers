### B. 알고리즘 요약

1. code rate `1/2`, 길이 `1612` PEG 생성 irregular LDPC (VN 분포 `λ(x)=0.3524x+0.2665x²+0.3812x⁷`, CN 분포 `ρ(x)=0.5673x⁵+0.4327x⁶`), OFDM(128-pt IFFT, CP 32) 시스템.
2. 문제: NMS/ANMS는 irregular code에서 error-floor가 높고 수렴이 느림. FER 성능도 저조.
3. 채널: BPSK + OFDM 멀티캐리어, ISI 방지용 cyclic prefix. 명시적 noise 모델은 AWGN 가정.
4. 핵심 기법: ANMS의 message suppression(`if hᵀx≡1 mod2 then β=η·β`)에 joint row-column 스케줄을 결합. 각 행 처리 직후 곧바로 VN LLR과 extrinsic을 갱신(수직위상을 수평위상에 병합).
5. 갱신식 (9)~(12): `α=γ-β`, 정규화 min-sum `β=μ·∏sgn·min`, suppression, `γ=α+β`를 M개 행에 대해 순차 수행. 곧바로 정보가 전파돼 수렴 가속.
6. 확장: 두 개의 정규화 인자(`μ=0.85`, `η=0.88`) 사용. QC-LDPC의 자연스러운 degree-decreasing 행순서 활용을 제안.
7. 구현 트릭: H 행을 가중치 내림차순으로 재배열 → 고가중 check node 먼저 처리해 저가중 노드에 신뢰정보 제공. 인코딩은 Gaussian elimination(행 가중 보존).
8. 학습/최적화: 별도 학습 없음, 정규화·suppression 인자는 수동 설정.
9. 결과: `Eb/N0=2.75dB`에서 BER<10⁻⁵ 도달에 JANMS 8 iter vs ANMS 16 iter (절반). error-floor에서 FER이 sum-product와 근사, 재배열 후 추가 개선.
10. 한계: HW 미설계, 순차 행처리 기반이라 병렬화 언급 없음, OFDM 단일 채널·단일 code 검증.
