### B. 알고리즘 요약 (10줄 내외)

1. 시스템: QC-LDPC (n=4096, k=2048, rate 1/2), base matrix 16×32 블록, 서브행렬 block size `b=64`, 총 check matrix 1024×2048, row weight 8/column weight 4, 8-bit 양자화.
2. 문제: 고정 정규화 계수(fixed NMSA)는 디코딩 성능이 낮고, 기존 partially-parallel 구조는 처리량이 낮거나 저장자원 소모가 큼.
3. 채널 모델: AWGN, 변조 후 가우시안 잡음 추가, MATLAB 시뮬레이션으로 인코딩·복조 후 FPGA 디코더 입력.
4. 핵심 기법 1: iteration 횟수 `n`에 따라 정규화 계수를 4단계(`n<1`, `n=2,3,4` 구간별 `1/2*1/4`, `1/2n*1` 조합)로 변화시키는 variable normalization factor(식 (7)); 곱셈을 shift-add로 구현.
5. 핵심 식: check node 갱신 `L(k)(rji) = min(|L(k-1)(qi'j)|) * normalization_factor * sign product` — normalization factor 조정이 min-sum 근사오차 보정에 직결.
6. 핵심 기법 2: HW 구조 - CNU/VNU 개수를 block size `b`와 동일하게 배치하는 intra-layer 완전병렬(fully intra-layer-parallel) 구조, 전통 partially-parallel 대비 CNU 16배·VNU 8배 병렬도 증가.
7. 부수 트릭: 변수노드/체크노드 data cache unit을 하나의 memory module(64개 memory, 각 16행)로 시분할(time-division) 공용화하여 저장공간 절감; H-matrix의 0 서브행렬은 저장 생략.
8. 학습/최적화 절차 없음(정규화계수는 참고문헌[7] 방식을 변형한 고정 스케줄).
9. 결과: 고정계수(a=0.5, 0.75) 대비 BER 성능 개선(문구상 0.2dB), 기존 partially-parallel(reference[16]) 대비 LUT 13%↑·BRAM(저장자원) 38%↓, throughput 391→402.5 Mb/s(14.5M 증가), FPGA는 MATLAB 대비 0.05dB 성능손실.
10. 한계: 단일 rate(1/2)·단일 length(4096)만 검증, ASIC 미검증(FPGA만), iteration 최대 15로 고정, 정규화계수 4구간 설계의 이론적 도출 근거는 본문에 상세히 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 가변 정규화 계수(iteration별 NMSA scaling) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 min-sum 기반이나 scaling/normalized factor가 코드상 명시 미확인 상태 — iteration별 가변 계수 아이디어를 CNU magnitude 갱신에 직접 참고 가능 |
| shift-add 기반 정규화 곱셈 HW 구현 | `decoder.cpp` `CNU_Update_New_Mag()` | 곱셈기 대신 shift-add로 자원 절감하는 방식은 HW 최적화 기법으로 이식 용이 |
| intra-layer 완전병렬 CNU/VNU 구조 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 이미 z_sb=32 row 단위 병렬 구조 보유 - 완전병렬 확장은 구조 정합 필요(중간 난이도) |
| node data cache 시분할 공용화(VN/CN cache 통합) | `decoder.h` `Get_VNU_*` (개념적 참고) | 저장자원 절감 아이디어는 HW arch 레벨 참고 가능, Prime ECC 메모리 구조와 정합 필요 |
| QC-LDPC H-matrix 서브행렬(0-block) 저장 생략 | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 QC 구조라 유사한 sparse 저장 기법 이미 적용 가능성 있음, 중복 기여 여지 |
```

> 적용 가치: 중간 — min-sum 정규화 계수 가변화와 HW 자원 절감형 완전병렬 구조는 Prime ECC 디코더/메모리 레이어에 개념적으로 이식 가능하나, 성능개선폭(0.2dB)이 크지 않고 검증이 단일 rate·FPGA 시뮬 수준이라 우선순위는 중간.
