### B. 알고리즘 요약

1. 시스템: rate-0.5 regular (3,6) LDPC(길이 10000, girth 8)로 부호화한 데이터를 `sqrt(N)×sqrt(N)` 비트 행렬로 배치, 2D ISI(인접트랙+다운트랙 간섭) + AWGN 자기기록 채널로 전송.
2. 문제: 2D 채널의 BCJR은 폭에 지수적, MCMC는 메모리 큼; graph 기반 검출기는 병렬친화적이나 채널 check node 연산이 check degree `q`에 대해 `O(2^q)`로 지수적.
3. 채널모델: 2D ISI 채널행렬 `H`(2×2) + guard band, factor graph에 code graph(LDPC)와 channel graph(ISI 의존성)를 함께 표현.
4. 핵심기법: 채널 check node의 APP LLR 계산을 전체 `2^q` 벡터 대신 Chase 디코딩 후보리스트 `Y`로 제한. 신뢰도 낮은 `p`개 least-reliable bit만 perturb.
5. 핵심식: APP LLR을 후보집합 `Y=Y1∪Y2` 위에서만 근사(식 7); 유한 LLR 보장 위해 각 비트위치에 반대부호 후보 2개가 존재하도록 best-vector 갱신(식 10).
6. 확장: 후보수를 정수 `p`로 조절 → 복잡도 `O(2^p·q)` 수준으로 낮추고 성능/복잡도 tradeoff 제공.
7. 구현: guard bit로 섹터 격리·검출 종단, 3계층(LDPC bit/LDPC check/channel check) 위 4단계 message-passing 의사코드.
8. 최적화: 없음(학습 없음), overall graph 반복(최대 50회) + graph 검출 반복(최대 8회) 스케줄.
9. 결과: `p` 선택으로 full-complexity graph 수신기 대비 0.2 dB 이내 손실에서 채널 check node 복잡도 대폭 절감, row/column BCJR 대비 성능 근접, 수렴 iteration 유사.
10. 한계: 시뮬만, HW 미설계, 대상이 2D 자기기록 채널 검출기(LDPC 코드/디코더 자체는 표준 그대로).
