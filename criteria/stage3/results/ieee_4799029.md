### B. 알고리즘 요약

1. 이진 (N,K) LDPC 코드, BI-AWGN 채널, BPSK 변조. 하드결정 초기 시퀀스 `z(0)`와 그 신뢰도 `|L_i| = |4y_i/N0|`를 이용하는 bit-flipping 계열 디코더.
2. 기존 BF/WBF/IWBF/BP류 대비 낮은 복잡도로 우수한 성능을 내는 디코더가 목표(APP/BP는 성능 좋으나 복잡도 높음, BF는 복잡도 낮으나 성능 열세).
3. 핵심 가정: 저밀도 패리티검사행렬에서는 오류 개수가 증가할수록(최소거리의 절반을 넘는 범위까지는) 평균적으로 신드롬 가중치도 증가한다는 성질(Bossert-Hergert 알고리즘 기반).
4. Algorithm I(hard): 매 iteration마다 각 위치 i에 대해 `S_i(k) = wt(z(k-1) + u_i·H^T)`(그 비트를 뒤집었을 때의 신드롬 가중치)를 계산, 가장 작은 `S_i(k)`를 갖는 위치 `j(k) = argmin_i S_i(k)`를 1비트만 플립.
5. Algorithm II(soft)는 `S_i(k) = wt(z(k-1)+u_i·H^T) + α·w_c·|L_i|`로 신뢰도 항을 더해, 신뢰도 높은 비트는 덜 뒤집히게 함(`α`는 SNR·코드별 최적화 필요한 가중치 계수).
6. 소프트 확장은 이진 연산(XOR 기반 신드롬) 외에 심볼당 실수 덧셈 1회만 추가되어 복잡도 증가가 미미함.
7. 루프 검출(Loop Detection): `j(k)=j(k-1)`이면 무한루프 진입을 감지, 이전에 루프를 유발한 위치들을 제외 리스트 `B`에 넣고 차순위 최소 `S_i(k)` 위치를 선택해 회피.
8. 학습 절차 없음. `α`는 코드별로 BER 10^-5 지점에서 시뮬레이션으로 최적값을 찾아 고정 사용(SNR 전 구간에 큰 성능저하 없이 적용 가능함을 확인).
9. 결과: Algorithm I은 하드결정 BF보다 최대 0.5dB 개선(코드별 상이), Algorithm II+LD는 IWBF 대비 0.1dB 개선, BP(IDBP) 대비 0.53dB 열세지만 복잡도(연산당 2N-1 real add + N·wc·wr XOR)가 훨씬 낮음.
10. 한계: 매 iteration 1비트만 순차적으로 플립(argmin 탐색 필요)하여 병렬화가 본질적으로 어려움, HW 구현/게이트카운트 없음, α의 SNR·코드 의존성이 실사용 시 튜닝 부담.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 1-bit flipping 디코딩(Algorithm I/II), 신드롬 가중치 argmin 탐색 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | min-sum 기반 message-passing과 계열이 달라(BF는 신드롬 가중치 기반 그리디 탐색) 직접 대체는 아니지만 결정 로직 삽입 위치는 동일 |
| 소프트 확장(채널 신뢰도 |L_i| 가중 덧셈) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-2~3bit LLR 활용 개념은 Prime ECC의 2SD/3SD 양자화 테이블과 유사한 층위 |
| 루프 검출/방지(제외 리스트 B) | 대응 없음 | Prime ECC는 min-sum+CRC 조기종료 구조로 별도 루프검출 메커니즘 없음 |
```

적용 가치: 낮음 — bit-flipping 계열은 min-sum message-passing과 근본 알고리즘이 다르고(매 iteration 1비트 순차 플립), Prime ECC는 이미 min-sum 기반 병렬 파이프라인을 보유해 이 논문 기법 자체를 이식할 유인이 적음.
