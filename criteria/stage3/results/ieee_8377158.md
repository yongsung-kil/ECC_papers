### B. 알고리즘 요약

1. 시스템: 5G 제어채널용 polar code `P(2048,1024)`(rate `R=1/2`, `N=2^n`)를 AWGN에서 iterative BP로 복호, SCL 대비 병렬화 가능한 대안 제시.
2. 문제: 유한길이 polar의 BP는 SC보다 낫지만 SCL-ML 대비 열세 → 부호구조 수정 없이 BP 성능을 SCL 수준으로 끌어올리는 것이 목표.
3. 모델: polar 인코딩 그래프의 `log2(N)` stage를 permute하면 동일 부호로 `(log2 N)!`개 factor graph 존재(N=1024면 10!>4×10^8) → 각 그래프는 loop/stopping set 구조가 달라짐.
4. 핵심 기법(multi-trellis BP, Alg.1): 한 permutation에서 BP를 `Nit,max`까지 돌려 실패하면 채널 `Lch`와 frozen 정보를 새 permutation으로 넘겨 최대 `qmax`개 그래프까지 순차 재시도.
5. 핵심 식: PE 갱신은 `f(L1,L2)=boxplus`(`log((1+e^{x+y})/(e^x+e^y))`)로 L/R 메시지 전파, permutation마다 stopping set이 셔플되어 특정 noise에 유리한 그래프가 존재.
6. 확장: 조기종료 조건 3종 - practical(`û·G=x̂`), perfect-knowledge(`û=u`, 하한 bound), CRC-aided(외부 CRC-32 만족 시 정지).
7. 구현 디테일: permutation 전환 시 내부 L/R 메시지는 clear(공유 가능 메시지 재사용은 future work), 5G 표준 polar+CRC와 호환.
8. 학습/최적화: 없음(그래프 permutation은 random schedule), 향후 LLR 기반 수렴 metric으로 무의미 permutation 스킵 제안.
9. 결과: perfect-knowledge 종료 시 SCL(L=32) 성능 근접·초과, CRC-aided는 고SNR에서 plain polar SCL 초과, error floor 영역 stopping set 완화 → 알려진 iterative polar 디코더 중 최고 성능(단 SCL-CRC엔 미달).
10. 한계: naive 구현은 복잡도·latency 큼(qmax만큼 순차), HW 미설계, polar 전용 permutation이라 LDPC로 직접 이전 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar factor graph BP 복호 | 대응 없음 | Prime ECC는 QC-LDPC Tanner graph min-sum, polar 인코딩 그래프 BP와 부호부류 상이 |
| factor graph permutation(multi-trellis) | 대응 없음 | polar stage 치환 개념은 LDPC Tanner graph에 존재하지 않음 |
| CRC 기반 조기종료 | `full_CRC.cpp`, `partial_CRC.cpp`(개념적 유사) | 조기종료용 CRC는 Prime ECC도 보유(Partial/Full CRC), 신규성 없음 |
| stopping-set 셔플로 error-floor 완화 | 대응 없음 | polar 그래프 permutation 특유 기법, QC-LDPC에 직접 이전 불가 |

적용 가치: **낮음** — polar code 전용 BP 복호(그래프 permutation) 기법으로 NAND용 QC-LDPC min-sum과 부호부류·그래프구조가 근본적으로 달라 이식 대상이 없다(CRC 조기종료는 이미 보유).
