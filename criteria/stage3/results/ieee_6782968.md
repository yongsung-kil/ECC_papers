### B. 알고리즘 요약

1. 시스템: rate-compatible(RC) QC-LDPC, mother matrix `M×(K+M)` 서브행렬 배열, 서브블록 `q=64`, systematic bit `1152` 고정, BPSK+AWGN, BP 디코더 최대 50 iteration.
2. 문제: dual-diagonal+puncturing으로 만든 선형인코딩 RC-LDPC는 고rate로 puncture 시 error-floor가 높음(small stopping/trapping set 때문).
3. 모델: error-floor는 Tanner graph의 소형 stopping/trapping set 기인, ACE(Approximate Cycle Extrinsic) message degree = `Σ(d_j-2)`로 cycle·degree·stopping set 관계를 근사.
4. 기법1 (선형 인코딩): parity part를 deterministic 형태(identity/shifted-identity/null만)로 고정 → parity `p_m`을 Eq.(4)-(9)의 XOR 누적으로 선형시간 인코딩.
5. 의미: `Hx^T=0`에서 parity가 결정적 대각 구조라 몇 개 XOR 게이트만으로 순차 인코딩 가능, generator 저장 불필요.
6. 기법2 (layered density evolution): mother matrix를 L layer로 분할, top layer `d_max`부터 하위 layer로 `d`씩 degree 증가시키며 각 layer별 DE로 최적 degree distribution 탐색 → 넓은 rate 범위(R_H~R_L) waterfall threshold 최적화.
7. 기법3 (layered node-connection, LNC): degree distribution 기반으로 서브행렬 위치/shift 계수를 rightmost(저차수)→leftmost(고차수) 순으로 배치, short cycle에 최고차수 column 포함시켜 ACE↑, Eq.(10) 조건으로 2i-cycle 회피해 girth↑.
8. 최적화 절차: DE로 degree distribution 확정 후 LNC로 girth/ACE 동시 증대, 예: 36×54 array(M=36,K=18), L=11 layer, `d_max=4`, `d=1`.
9. 결과: FER=10⁻⁵까지 error-floor 관측 안 됨, 최대 1920 rate 지원, rate 1/3에서 3G(3GPP2/802.20) 대비 FER=10⁻⁴에서 0.5dB gain, Shannon limit 대비 0.4dB 근접.
10. 한계: HW 미설계, 시뮬만, rate-compatible ARQ 무선 시나리오 전용, gate count/throughput/latency 수치 없음.
