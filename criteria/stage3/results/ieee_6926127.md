### B. 알고리즘 요약

1. 시스템: 라디에이션 soft-error(MCU/SEU) 대비 메모리 ECC. difference-set cyclic codes(DSCC, LDPC의 majority-logic 복호 가능 부분군), 예시 code length 72 (Table.1은 .txt에서 손상).
2. 풀려는 문제: 기존 plain MLD는 code length 만큼(예: 72비트→72 cycle) 복호시간이 크고, SFD(syndrome fault detector)는 code 크기 증가 시 면적·복잡도가 급증.
3. 모델: BSC 가정, `P(y|x) = (1-p)^(n-d) * p^d` (d=Hamming distance, p=BSC 오류확률)로 MLE 채널 모델링.
4. 핵심기법: majority-logic decoding(MLDD) — 각 variable node가 참여하는 unsatisfied syndrome(check) 수를 세는 flipping function, threshold 초과 시 hard-decision buffer 비트 flip.
5. 핵심 아이디어: bit node가 받은 메시지 다수가 수신값과 다르면 flip; check node는 modulo-2 합이 0이면 만족. 이는 hard-decision message-passing.
6. 확장(제안): 2차원 modulo-sum(modified Benes network) 오류검출기 — 데이터를 k개 m비트 세그먼트로 나눠 ones-complement 합 후 보수, 검출기 면적을 code 크기와 무관하게 설계.
7. 구현: `H=[P^T|I]`, `G=[I|P]` Gauss-Jordan 구성, `C·H^t=0` 유효성 검사, 유효 codeword 조기종료.
8. 학습/최적화: 없음(결정론적 majority-logic + threshold flip).
9. 결과(HDL 합성): 최대 주파수 `433.088MHz`(period 2.309ns), combinational path delay `10.381ns`; MLDD 면적 오버헤드는 plain MLD 대비 `10.16%~0.43%`로 SFD(`25.4%~294.94%`)보다 크게 절감.
10. 한계: hard-decision majority-logic 전용, DSCC 특정 부호, 절대 gatecount·rate·throughput 부재, 대상은 SRAM급 라디에이션 오류(NAND flash 아님).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| majority-logic 복호(MLDD, flipping) | 대응 없음 | Prime ECC는 min-sum message-passing, majority-logic 아님 |
| DSCC 부호구조/PCM 구성 | ecc_top.cpp Load_PCM() (부분) | difference-set cyclic, QC circulant 아님 → 이식 부담 |
| 조기종료(C·H^t=0 유효성) | partial_CRC.cpp / full_CRC.cpp | 개념 유사하나 Prime ECC는 CRC 기반, 방식 상이 |
| 2D modulo-sum 오류검출기(HW) | 대응 없음 | Benes network 검출기, min-sum 디코더 구조와 무관 |
```

적용 가치: **낮음** — 라디에이션 메모리 오류 대비 DSCC majority-logic 복호기(hard-decision)를 2D modulo-sum으로 경량화한 것으로, Prime ECC의 min-sum binary QC-LDPC(soft 지원, NAND) 디코더와 부호종류·복호원리가 달라 떼어 쓸 요소가 없다.
