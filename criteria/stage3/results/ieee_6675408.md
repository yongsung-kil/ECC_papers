### B. 알고리즘 요약

1. 대상: time-invariant LDPC convolutional codes(LDPCC). 인코더는 `b` bit 입력을 `Gi`로 `c` coded bit로 부호화, delay block `M`개면 constraint length `b(M+1)`, memory `bM`.
2. 문제: LDPCC의 memory size(delay 수), encoder block size(sub-matrix 크기), constraint length가 각각 성능에 어떻게 기여하는지 정량 비교.
3. 부호구조: H는 `M+1`개 sub-matrix `H0..HM`(각 `(c-b)×c`)를 대각으로 반복. 설계는 반복단위 `H'`에만 집중.
4. 구성법: monomial matrix `H(D)`(원소 `D^q` 또는 0, 최대차수=M)를 QC-LDPC와 유사하게 구성, `D^i` 위치에 1을 넣어 각 `Hi` 생성.
5. 핵심기여1: matrix expansion operation `⊛` 정의. `A⊛B=C`, `c_ij = b_ij · a_{⌊(i-1)/l⌋+1, ⌊(j-1)/l⌋+1}`로 monomial matrix를 l배 확장.
6. 확장 목적: degree distribution 유지한 채 encoder block size(sub-matrix)를 키우기 위해 sparse monomial matrix를 QC-LDPC 매트릭스 `HQ`(sub-block l×l)로 확장.
7. 실험 셋업: 모든 H girth 8, BP 복호 max 100 iteration.
8. 실험1(memory): delay block 10 vs 30 → 30 delay 부호가 항상 우수(memory 클수록 성능↑ 확인).
9. 실험2(block size): 3×6 vs 6×12 sub-matrix(고정 memory) → 6×12가 더 우수. 실험3(constraint length): constraint length 63 vs 66 → 유사 성능(constraint length 유사시 성능 유사 추정).
10. 한계: 순수 부호구성·시뮬 비교 연구, HW 미설계, rate 0.43~0.48 저rate, expansion법이 모든 HQ에서 좋은 성능 보장 못함.
