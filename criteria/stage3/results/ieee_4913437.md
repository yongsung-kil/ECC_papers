### B. 알고리즘 요약

1. 시스템은 전력선 통신(PLC) 채널 위에서 OFDM 전송을 사용하며, irregular QC-LDPC 코드를 rate 1/2, 1/3, 2/3, 3/4, 4/5로 구성.
2. 기존 sum-product(soft) 및 bit-flipping(hard) 디코딩 알고리즘이 PLC의 impulsive 잡음 환경에서 성능/복잡도 트레이드오프가 있어, 이를 개선하는 hybrid 기법을 제안.
3. 채널 모델은 Zimmermann의 multipath 주파수 모델, 잡음 모델은 Middleton's class A(배경+impulsive 잡음, `A=0.05`, `Γ=0.1`)를 사용.
4. 제안 bit-flipping 변형은 매 라운드 여러 비트 대신 unsatisfied check 수가 최대인 비트 `1개만` 플립하여 잘못된 비트 반전 위험을 줄임.
5. Hybrid 스킴은 먼저 sum-product(반복 15회)를 적용, 실패 시 변형 bit-flipping(반복 85회)을 순차 적용해 두 알고리즘의 장점을 결합.
6. 부호는 upper-triangular 형태의 QC-LDPC 생성 행렬 `H(q,j,k)`(식 5)로 구성하여 인코딩을 용이하게 함, rate = `1 - j/k`.
7. Sum-product 메시지는 로그도메인 대신 선형 도메인으로 계산해 복잡도를 낮추되, PLC 환경에 맞춰 수신 심볼의 분산을 추정해 입력값으로 사용.
8. 별도의 학습/최적화 절차 없음(고정 파라미터 시뮬레이션).
9. 결과: BER-Eb/N0 곡선에서 hybrid 기법이 sum-product 단독(반복 20회), bit-flipping 단독(반복 150회)보다 우수한 성능을 보였다고 서술(수치는 그래프에만 존재, 본문 수치 없음).
10. 한계: 시뮬레이션뿐이며 HW 설계·복잡도 분석 없음, PLC 특화 채널/잡음 모델에 한정되어 NAND 채널 가정과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-product(비-log domain) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum(로그도메인) 기반이라 알고리즘 종류 자체가 다름, 참고 수준 |
| 변형 bit-flipping(1비트씩 플립) | 대응 없음 | Prime ECC는 min-sum 계열만 사용, hard-decision bit-flipping 미보유하나 이미 HD 모드 지원 중이라 중복 |
| Hybrid(soft→hard 순차) 디코딩 | 대응 없음 | 조기종료(Partial/Full CRC)와 유사한 발상이나 알고리즘 전환 방식은 다름 |
| QC-LDPC 인코딩(upper-triangular H) | `encoder.cpp` `Generate_PCM_encoding()` | 부호 구성법이 dual-diagonal과 다른 triangular 방식, 이식 난이도 높음 |

> 적용 가치: 낮음 — PLC 채널 특화 시뮬레이션이며 NAND 관련성, HW 설계, 정량적 성능 개선 근거가 모두 부족함.
