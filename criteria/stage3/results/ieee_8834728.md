### B. 알고리즘 요약

1. 대상 시스템은 TLC/QLC 3D NAND SSD, LDPC ECC를 기본 채용하며 read는 hard→soft(최대 7 sensing level) 재시도 구조.
2. 문제: 신뢰도 저하 시 LDPC가 여러 번 retry(soft sensing)를 해 decoding latency가 급증, 95th+ tail latency로 QoS 악화(최대 21~25배).
3. 모델: read latency = sensing + transfer + decoding(8us 고정), sensing level당 총 latency를 Table 1로 고정(108~618us).
4. 핵심 기법 1단: `threshold based refresh` - 페이지 read 시 retry 수 `R` 획득, `R>R2`면 즉시 refresh.
5. 보조 조건: `R>R1 && F>T`(접근 빈도)이면 성능용 refresh. 기본 `R1=3, R2=5, T=2`.
6. 핵심 기법 2단: `periodic refresh` - 주기적으로 전 데이터를 검사해 `R>R2`인 극단 latency 데이터를 refresh(첫 접근 tail 회피).
7. 구현 트릭: 블록 첫 페이지만 read해 임계 미만이면 블록 스킵, benefit bit로 무익 refresh 회피(비용 절감).
8. 오버헤드: FTL 매핑 엔트리에 access counter 2bit + benefit bit 1bit만 추가(무시할 수준).
9. 결과: SSDsim + MSR 8개 워크로드에서 tail latency 대폭 개선, 평균 read Du et al. 대비 15.4% 추가 단축, write 증가 ~2% 이하.
10. 한계: 디코더/부호 알고리즘 자체는 미변경(LDPC를 retry-count 소스로만 사용), 순수 시뮬(SSDsim), HW 미설계.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| retry 횟수 기반 refresh 트리거 | 대응 없음 | FTL/컨트롤러 정책 - 디코더 코드 외부 |
| soft sensing level 재시도 | channel.cpp Set_R_Offset(), Set_LLR_Th() | 간접 - 재시도 자체는 soft-read 비용이나 논문은 이를 개선 안 함 |
| LDPC 디코딩 (블랙박스) | decoder.cpp LDPC_Decoder() | 대응 없음 - 논문은 디코더 내부 미변경 |
```
