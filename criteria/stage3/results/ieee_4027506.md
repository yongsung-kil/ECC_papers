### B. 알고리즘 요약

1. 시스템: (3,6)-regular QC-LDPC, code length 11520bit, rate 1/2, 1920x1920 크기 identity-shift 서브블록(3x6 sub-block) 구조.
2. 풀려는 문제: 기존 long code-length LDPC 디코더는 throughput이 200Mb/s 미만이었고, row/column operation을 동시에 수행하려면 메모리량이 크게 늘어나는 문제.
3. 핵심 가정: partially-parallel individual memory 구조(row/column 연산 동시 수행, shared memory 대비 처리량 2배)를 기반으로 함.
4. 핵심 기법 1단: modified min-sum 알고리즘의 row(check-node) 연산이 `alpha_mn = sign(prod) x min(|beta|; n'∈A(m)\n)` 형태로 각 행마다 실제로는 min1, min2 절대값 두 개만 필요함에 착안, row operation 출력을 `(min1, min2, location, sign)`만 저장(축소 저장).
5. 핵심 식: 저장 비트수는 `roundup(log2(Wr))`(위치 인덱싱) 비트만 추가로 필요 - `Wr`은 row-weight(=6).
6. 핵심 기법 2단: 서브블록의 순환 shift 값에 따라 alpha/beta가 계산된 후 다음 사용 시점까지만 저장하면 되므로 "shifting-out used data" - 예시(10x10 서브블록)로 alpha는 1개, beta는 7개만 저장하면 충분함을 타이밍 다이어그램으로 증명.
7. 부수 트릭/구현: 6-bit 양자화 LLR 입력, CFU(check-node function unit, row-weight=6) 90개(30x3), BFU(bit-node function unit, column-weight=3) 90개(30x3) 병렬 배치, dual-port SRAM으로 입력 코드 2블록 버퍼링(파이프라인).
8. 학습/최적화 절차: 없음(shift 값은 heuristic search로 사전 결정된 고정 부호).
9. 결과: 두 메모리 절감 기법으로 memory requirement 55% 감소, ASIC(TSMC 0.18um) 합성 결과 gate count 2,073K(그중 temp registers 62.7%), 69MHz에서 20회 반복 시 throughput 312Mb/s; FPGA 구현에서는 50MHz, 225Mb/s.
10. 한계: (3,6) regular 고정 코드에서만 검증, BER/waterfall/error-floor 등 정정성능 수치는 본문에 없음(순수 HW 구조·throughput 논문), 조기종료 없이 고정 20회 반복.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Row operation 출력 축소(min1/min2/location/sign만 저장) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산 자체가 min1/min2 방식이라 Prime ECC 구조와 직접 대응, 레지스터 저장 축소 기법을 참고 가능 |
| Shifting-out used data (alpha/beta 임시 저장 최소화) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 메모리 스케줄링 최적화로 HCU/pipeline 구조에 이식 검토 가능 |
| Individual memory architecture (row/col 동시 수행) | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC의 pipeline 6-stage 구조와 개념적으로 유사, 직접 대응 모듈 특정은 어려움 |
| CFU/BFU 병렬 배치 구조 | `decoder.cpp` `VN_Cal_HD()` 등, `GT.cpp` `Make_GT_HW()` | 병렬 유닛 배치 자체는 Prime ECC z=32 구조와 유사하나 세부 대응 낮음 |

적용 가치: `중간` - min-sum 디코더에서 row operation 결과를 min1/min2/location/sign으로만 저장하고 사용 완료 데이터를 즉시 shift-out하는 메모리 절감 기법은 Prime ECC decoder.cpp의 메모리/레지스터 사용량을 줄이는 데 직접 참고할 만함(단, Prime ECC는 이미 HCU/GT로 유사한 최적화를 갖추고 있어 중복 가능성 있음).
