### B. 알고리즘 요약

1. Factor Graph(변수노드+팩터노드, bipartite) 솔버를 위한 HDL 자동 생성 프레임워크 — Python 생성 스크립트 + Verilog 템플릿(태그 삽입 방식)으로 벤더 무관 HDL 산출(현재 Xilinx Virtex-6 ML605 타깃).
2. 솔버는 graph structure / factor computation / iterative algorithm / termination 4개 기준으로 정의, 이 파라미터로 프로그래머블하게 HW 생성.
3. 지원 알고리즘 2종: Sum-Product(N-domain, 16-bit float 확률), Min-Sum(1-domain, N-bit signed integer LLR — binary decoder용).
4. 변수노드 M-S: `L = log(P(0)/P(1))` LLR 저장, 메시지 `q_{n→m}=γ_n + Σ_{m'∈M_{n,m}} r_{m'→n}`, belief는 전체 합의 sign으로 hard decision(식 3~5).
5. 팩터노드 M-S(LDPC): `r_{m→n} = (Π sign) · min|q_{n'→m}|` — 표준 min-sum check node(식 6). 팩터노드는 application-specific.
6. 그래프는 graph definition file로 노드 연결/모듈명 지정 → 완전병렬(fully-parallel) 인스턴스화. termination 모듈은 개발자가 수동 작성.
7. 응용1(LDPC basket): `N∈{120,200,400,800,1008,1156}`, rate 1/2, (3,6)-regular 코드 37개 생성, PEG/LDPC-codes 소프트웨어로 코드 구성, Virtex6에서 BER/Iavg/throughput 측정.
8. 응용2(802.11n): irregular block-circulant LDPC, length 648, rate {1/2,2/3,3/4,5/6} — 3개만 구현 성공(R=5/6은 routing congestion으로 P&R 실패).
9. 결과(Table1, N=1008, quant 4-3 hyb): 100 MHz, Eb/N0 3.4 dB @ BER 1e-6, avg iter 5.6, throughput 6.6 Gbps, LUT 53,748(참조 [8]보다 면적 효율↑, throughput은 낮음). 응용3으로 4x4 Sudoku 솔버도 생성.
10. 한계: LDPC 알고리즘 신규성 없음(교과서 SP/MS), 완전병렬만 지원(pipelined/partially-parallel은 future work), 대형·고rate 코드는 라우팅 혼잡으로 구현 실패.
