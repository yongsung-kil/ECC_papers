### B. 알고리즘 요약

1. 시스템: AWGN + BPSK, rate `R=1/2` binary LDPC **convolutional** code, 파라미터 `(M,3(2),5)`, memory `M=129,257,...,4097`.
2. 문제: 블록 LDPC보다 구조가 규칙적이고, pipeline 디코더로 처리 가속 가능한 convolutional 버전을 만들자.
3. 모델: 시변 주기적(period `T=M-1`) 저밀도 parity-check 행렬 `H^T`, semi-infinite band 구조.
4. 부호 구성: `(4,2,2)` 행렬을 대각선으로 잘라 재배치→interleave→행 추가로 `(5,3(2),5)` 시변 PCM 생성, 무한 반복. 정보열 3 ones / parity 2 ones.
5. 인코더: dual-diagonal형 systematic, `c`개 length `M+1` shift-register 병렬 + 시변 연결로 구현(Fig.4).
6. 디코더 핵심: check-sum extrinsic를 최소 reliability 항으로 근사 `sgn·prod × min|l|` (min-sum형, 식 6/28), 각 iteration은 해당 parity-set 제외(독립정보 유지).
7. HW 트릭: **I-processor pipeline** — I개 프로세서를 `2(M+1)` register 간격으로 배치, `J+1`개 FIFO를 통해 streaming으로 vertical/horizontal step 수행.
8. 분석: 양자화(q=4, 9레벨) AWGN에서 J-fold convolution으로 BER의 asymptotic 분포 수치계산, `(N,3,6)`이 최적.
9. 결과: memory 4097에서 BER `1e-5`를 capacity 약 1 dB 이내로 달성, memory-10 Viterbi급 복잡도. 동일 길이 block부호보다 우수.
10. 한계: HW 미설계(SW 시뮬), gate/throughput 없음, convolutional streaming 구조라 latency(디코딩 delay `M(I+1)-1`) 큼.
