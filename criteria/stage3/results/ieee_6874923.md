### B. 알고리즘 요약

1. 대상: column-weight-3, girth-8 permutation-based(QC) LDPC를 global code로 두고, 일부 single parity check를 2-error correcting component code의 super check로 대체한 check-hybrid GLDPC.
2. 문제: 반복 디코더 실패의 원인인 `trapping set`(특히 (4,4),(5,3),(7,3))을 rate 손실을 최소화하며 제거.
3. 채널/디코더 모델: BSC, all-zero codeword 전송 가정, `Parallel Bit Flipping(PBF)` 알고리즘 기반(고속·저복잡도).
4. 핵심 기법: global LDPC의 트랩핑셋 지식을 이용해 "임계(critical) degree-2 체크"만 골라 super check로 변환 → 해당 트랩핑셋을 harmless로 만듦.
5. 핵심 개념: super check는 BDD(bounded distance decoding, `t≤(d-1)/2`)로 codeword를 찾으면 틀린 VN에만 flip 메시지 전송 → 사이클을 끊음.
6. 최적화 목표: critical set의 최소 크기 = `splitting number` `s(a,b)(H)`를 최소화해 rate-loss 축소.
7. 알고리즘: Algorithm 2로 트랩핑셋 T에서 critical set을 탐색(degree-1 체크 1개+degree-2 체크 2개에 연결된 VN 우선 처리).
8. 상계: `s(a,b)(H)≤2p`(모든 VN이 super check 2개 연결 시 모든 elementary TS 제거), 특정 조건(마지막 2p행 girth 12)에서 `s(4,4)(H)≤p`.
9. 결과: 이론적 상계와 fixed set 조건(Theorem 1) 제시. BER/시뮬 수치는 본문에 없음.
10. 한계: 이론(BSC/PBF)만, HW 미설계, 실제 성능곡선·rate 수치 없음, component code 도입으로 rate 손실 불가피.
