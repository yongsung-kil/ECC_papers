### B. 알고리즘 요약

1. 대상: IEEE 802.3ca EPON irregular LDPC `N=17664`, `R=0.826`, BPSK/AWGN, 최대 12 iteration. LLR은 4bit 양자화, 디코더 내부 메시지는 `q={2,3,4}`bit(알파벳 4/8/16).
2. 문제: 저복잡도 Sign-Preserving Min-Sum(SP-MS)을 EPON 부호에 쓰면 `q=2,3`에서 심한 error floor(BER≈1e-3, 1e-6).
3. 원인 모델: 채널 LLR(4bit)과 내부 메시지(2~3bit) 알파벳 mismatch로 저차(degree-3) VN이 나쁜 채널 LLR에서 못 벗어남. degree-3 VN이 전체 ≈72%.
4. 핵심기법: SP-MS VN update에 iteration별 가중치 `w_n^(l)` 도입 - `m = Ψ(I_n + w·(μ/2) + Σ_ext m_c→v)`. degree-3 VN에만 `w≠1` 적용.
5. 식 의미: 반복이 진행될수록 incoming 메시지 신뢰도가 커지므로 채널 LLR 대비 메시지 magnitude 비중을 iteration에 따라 키워 mismatch 탈출을 유도.
6. 확장: 고차 VN은 incoming 메시지가 많아 자체적으로 탈출 가능 -> 가중치 불필요, degree-3에만 적용해 복잡도·α 최적화 부담↓.
7. 구현 디테일: 곱셈 회피 위해 가중치를 최대 3개 power-of-2 합(3bit)으로 제한, 첫 4 iteration은 `w=1`이라 나머지 8 iteration만 shift-and-add.
8. 학습/최적화: random candidate weight vector 집합에서 최저 BER/FER 선택, `0<w_n^(l)≤w_n^(l+1)` 단조 제약, SNR 2.8/3.1dB에서 최적화(Tab.1).
9. 결과: degree-3 VN에 3bit 가중치만으로 error floor를 3자릿수 이상 감소(FER 기준 최대 ×6.3e3), q=3 weighted가 BP에 근접.
10. 한계: HW 미설계(shift-and-add 제안만), 가중치는 특정 부호·SNR에 사후 튜닝, floor 근본 원인은 미규명(후속연구 필요).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SP-MS VN update + iteration별 가중치 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | VN 처리 루프에 저차 VN 가중 로직 삽입 가능(직접) |
| iteration별 가중치 테이블(shift-and-add) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | iteration 구간별 LLR/가중 테이블과 동일 축, 이식 용이 |
| min-sum CN 연산(min1/min2+sign) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유, 논문은 CN 변경 없음 |
| error-floor 대책(저차 VN) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | error-floor 개선이 관심 대상(추천도↑) |
```

적용 가치: **높음** - binary min-sum 계열의 error-floor를 iteration별 가중치(shift-and-add, HW친화)로 낮추는 기법으로, Prime ECC의 VN 처리·iteration별 LLR 테이블 구조에 그대로 얹기 좋다. 다만 Prime의 QC-LDPC는 VN degree 17 중심이라 저차 VN 이슈 존재 여부 확인 필요.
