### B. 알고리즘 요약 (10줄 내외)

1. QC-LDPC 부호는 `H=[Hs Hp]` 구조이며 `Hs`는 circulant permutation(shift) 블록들로 구성된 sparse base matrix, `Hp`는 accumulator(IRA류) triangular 행렬로 인코더를 단순화.
2. 기존 랜덤 계수 선택 기반 설계는 short cycle(특히 4-length)을 회피하기 어렵고, 특히 base matrix가 dense한 고rate 코드나 `z`(expand factor)가 작을 때 문제가 심각.
3. 채널/노이즈 모델은 표준 AWGN이며, 핵심 가정은 shift 계수 `δ(i,j)`가 코드 그래프의 cycle 구조를 완전히 결정한다는 것.
4. 핵심 기법: 각 순환 shift 행렬의 non-null 원소 위치를 기하학적 좌표(x,y)로 표현하고, 4개 shift 블록이 이루는 사이클 조건을 `(δ_A0-δ_A1+δ_A2-δ_A3) mod z = 0`으로 도출(식 16).
5. 이를 `2l`-length cycle로 일반화한 식 `Σ(-1)^i δ_i mod z = 0`(식 17)이 사이클 존재 조건이며, 이 식으로 특정 shift 계수 `δ_j`를 목표 cycle length(TCL)를 만족하도록 순차적으로 결정하는 PEG류 점진적 알고리즘을 제안.
6. 확장: `γ_j`(나머지 계수들의 부호교대합)와 `z`가 서로소가 되도록 `δ_j`를 고르면 cycle length가 최대화됨(식 20-21).
7. minimum distance는 weight-2 정보워드의 accumulator RTZ(Return-To-Zero) 시퀀스 출력 weight를 분석해 `d_min ≤ 2+m·q_min`(`q_min`=Hs 최소 variable degree, `m`=Hs row-block 수) 상계를 유도.
8. 별도 학습/최적화 절차 없음(순차적 조합 탐색으로 base matrix S를 채우는 결정론적 설계 알고리즘).
9. 결과: rate=1/2, N=672 코드에서 제안 알고리즘으로 최소 8-length cycle 보장 코드가 무작위 생성 코드(4-length cycle 존재, 최저 성능) 대비 waterfall 구간에서 BER 우세(도 5, ~1dB 수준 개선 관찰).
10. 한계: 작은 rate=1/2 코드 1개 사례로만 검증(HW 미설계, 시뮬뿐), Hp가 특정 accumulator 구조로 고정되어 고rate·NAND 특화 QC-LDPC로의 일반화는 별도 검증 필요.
