### B. 알고리즘 요약

1. 대상: rate-compatible QC-LDPC (expansion factor z, base Tanner graph 복제); low-rate mother code를 puncturing해 고rate 파생(WiMAX rate-1/2·2/3a → 최대 rate 6/7, 8/9).
2. 문제: puncture된 VN을 prior LLR=0으로 두는 표준 복호는 불필요 연산 다수 + 수렴 느림.
3. 핵심 기법 (check-node merging): punctured VN과 purged CN을 정의, purged CN의 edge를 다른 이웃에 병합 후 두 노드 제거 → merged PCM `HP = HA + BA·BB^-1·HB`.
4. 복호는 generalized offset min-sum: `αnm = γn + Σβm'n`, `βmn = (Πsign)·max(min|αn'm|-c, 0)`, `λn = γn + Σβmn` (c: 양의 normalization 상수).
5. merging은 평균 iteration을 크게 줄이면서 BLER 성능도 개선(waterfall 향상), 단 CN degree 편차가 커짐(최대 degree 증가).
6. 구조 (check-serial): CFU가 degree 편차를 순차 처리로 흡수; edge index 5bit로 최대 CN degree 32 지원.
7. 병렬 data congestion 문제: merge로 layer 내 VN 독립성이 깨져 같은 bit-sum이 여러 check에 쓰임 → CFU가 갱신값 대신 bit-sum "차분"을 계산.
8. bit-sum update unit: programmable feedback shift register로 여러 소스의 갱신을 누적, old bit-sum에 차분 가산.
9. 구현 디테일: bit-sum 8-bit(24개 memory), min-sum은 min1/min2 5-bit magnitude + 5-bit index + parity(16bit) 저장, c2v는 6-bit 2's complement, 24×24 cyclic shifter로 라우팅.
10. 한계: 시뮬(BLER/avg-iter 곡선)만 제시, gate count·throughput·합성 결과 없음; puncturing 전제 하 알고리즘.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum 복호 (min1/min2) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유 min-sum과 동일 계열, 중복 |
| check-node merging (수렴 가속 스케줄) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | dual-update 대안 스케줄; puncturing 없인 적용 애매 |
| punctured VN / merged PCM 생성 | `ecc_top.cpp` `Load_PCM()`, `encoder.cpp` `Generate_PCM_encoding()` | rate-compatible PCM 관리 필요, 우리 부호는 고정 |
| bit-sum 차분 / min-sum memory (16bit) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_DEC` | VN 갱신·메시지 메모리 관리 대응 |
| cyclic shifter 라우팅 (QC 병렬) | `GT.cpp` `Make_GT_HW()` (근사 대응) | QC circulant shift 라우팅과 개념 유사 |

적용 가치: **낮음** — offset min-sum은 이미 보유, 신규성인 check-node merging은 puncturing 기반 rate-compatibility가 전제라 단일 고정 고rate NAND QC-LDPC에는 직접 이득이 불명확.
