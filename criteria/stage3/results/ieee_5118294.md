### B. 알고리즘 요약

1. IEEE 802.15 WPAN 표준의 QC-LDPC 부호(코드워드 길이 576, sub-matrix size 18, rate 1/2·3/4·7/8)를 대상으로 1.5Gbps 이상 요구사항을 충족하는 디코더 설계.
2. 짧은 코드워드 길이에서 높은 throughput 요구가 문제이며, 기존 partly-parallel 구조는 이 요구를 충족하기 어려움.
3. rate 7/8(32×4 sub-matrix)의 각 행을 2번, 4번 split해 rate 3/4, 1/2를 파생하는 구조(H matrix는 일부 블록이 비어 irregular)를 채널 가정으로 사용.
4. 핵심 기법은 modified Min-Sum: check-to-variable 메시지 `R_cv = α·∏sign(L_vc)·min|L_vc|` (α≈0.75 scaling factor), variable-to-check 메시지 `L_vc = ΣR_mv + I_v`.
5. 위 식은 Sum-Product의 근사로 계산복잡도를 낮추고 SNR 정보 불필요하다는 게 의미이며, 6-bit 양자화(부호1+정수2+소수3)로 SPA와 유사한 성능을 확보.
6. 72 VNU + 72 CNU를 완전 병렬 배치해 한 iteration을 8 clock cycle(파이프라인 미포함)에 처리하는 고병렬 스케줄, CNU는 4개 메시지/cycle 처리하는 flexible 구조로 3개 rate 모두 지원.
7. 부수 트릭: row process 결과를 min/2nd-min magnitude+index+sign만 압축 저장(`[4]` 기법 재사용)해 메시지 메모리 요구량을 크게 절감, sign은 32-entry FIFO 레지스터로 별도 관리.
8. 별도 DE/GA 학습 절차는 없음(구조적 스케줄·CNU 설계); critical path에 2-stage pipeline을 삽입해 500MHz 합성(1 iteration=10 cycle로 증가).
9. 결과: 65nm 공정에서 320k gate, 2.8Gbps throughput(max iteration 10), 기존 partly-parallel[8](640Mbps, 14.3mm2) 대비 면적 대비 효율 우수(0.4mm2급).
10. 한계: 정정성능 개선 기법 없음(표준 min-sum 성능 확인 수준), ASIC 합성 결과만 제시(FPGA/실측 없음), rate 가변 구조가 802.15 특유의 row-split 패턴에 강하게 결합되어 일반화 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Modified Min-Sum (min/2nd-min + sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2+sign XOR 기반 min-sum, 알고리즘 동일 계열(중복 기여) |
| VNU 메시지 갱신 | `decoder.cpp` `VN_Cal_HD()` 등 | 표준적 VN 처리로 이미 보유 |
| 압축 메시지 저장(min/2nd-min/index/sign) | `decoder.h` `Get_VNU_*` | 메모리 절감 기법, Prime ECC 메시지 저장 방식과 비교 가치 있음 |
| flexible CNU(3-rate 지원, 4x4 switch, row-split) | `ecc_top.cpp` `Load_PCM()` | 다중 rate 지원 구조는 Prime ECC의 단일 고정 H-matrix와 달라 이식 난이도 높음 |
| 8→10 cycle/iteration 고병렬 스케줄 + 2-stage pipeline | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC의 6-stage pipeline과 설계사상 유사, 스케줄링 참고 가능 |

적용 가치: 낮음 — min-sum, 압축 메시지 저장, pipelining 등 핵심 기법이 Prime ECC에 이미 구현되어 있어 중복 기여이며, 유일한 차별점(3-rate flexible CNU)은 802.15 특유 row-split 구조에 결합되어 이식 난이도가 높음.
