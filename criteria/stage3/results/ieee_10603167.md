### B. 알고리즘 요약

1. 5G NR 데이터채널용 QC-LDPC 표준(base graph BG1/BG2, 서브행렬 크기 Z=2~384)에 대한 부분병렬 인코더 설계.
2. 완전병렬 구현은 하드웨어 자원 소모가 과도하고, 직렬 처리는 처리량이 낮아 두 방식의 절충이 필요한 문제.
3. 부호는 forward substitution만으로 인코딩 가능한 5G LDPC 특수 구조(H를 A,B,C,D,I 블록으로 분해)를 전제.
4. 핵심 기법은 core parity bit 계산식 `λ_j^T = Σ a_{j,l} i_l^T`을 통한 4개 core parity bit(`P_c,1~P_c,4`) 산출 후 나머지 parity bit를 순환시프트(CS) 모듈로 계산.
5. 식 (5) `P_c,1=Σλ_j, P_c,2=λ_1+P_c,1, ...`는 GF(2) XOR 연산만으로 core parity를 빠르게 계산하기 위한 재귀식.
6. base matrix 크기 Z에 따라 병렬도를 가변(Z>192: 1행/cycle, 96<Z≤192: 2행/cycle, Z≤96: 4행/cycle)하는 적응형 CS 활용 구조.
7. PCM 전체 대신 서브행렬 배치정보를 양자화해 ROM에 저장, 파이프라인 구조로 인코딩 효율화.
8. 유전 알고리즘(GA)으로 큰 base matrix(Z>192)의 인코딩 스케줄(멀티 CS 배치)을 최적화, `N_CPC`(코드워드당 클록수) 최소화.
9. Altera Cyclone IV E FPGA(EP4CE10F17C8N, fmax=402MHz)에서 BG1 peak throughput 17.68Gbps, BG2 14.84Gbps 달성, HUE는 완전병렬 대비 약 3배, 직렬 대비 20% 향상. GA로 4CS 스케줄 시 BG1 클록수 180→106(32.1%), GA 추가 시 60% 개선.
10. 5G NR 표준 전용 인코더로 디코더/정정성능은 다루지 않으며, NAND read soft-info나 error-floor와 무관한 순수 인코더 하드웨어 최적화 연구.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| forward substitution 기반 core parity 계산 | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코딩 고정 구조라 5G BG 전용 forward substitution 알고리즘과는 부호 구조 자체가 달라 직접 이식 난이도 높음 |
| GA 기반 CS(cyclic shifter) 스케줄 최적화 | `encoder.cpp` `Column_Reordering_Dual_Update()` | 스케줄 최적화 아이디어는 참고 가능하나 5G BG 특유의 CPM 배치 최적화라 일반화 난이도 중~높음 |
| PCM 서브행렬 양자화 ROM 저장 | `ecc_top.cpp` `Load_PCM()` | 메모리 절감 아이디어는 유사하나 5G 표준 행렬 구조 전용 |
```

적용 가치: **낮음** — 5G NR 표준 base graph 전용 인코더 하드웨어 최적화로 NAND ECC 코드 구조·채널과 무관하며, 디코더/정정성능 개선이 없는 순수 인코더 처리량 연구.
