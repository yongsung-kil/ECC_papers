### B. 알고리즘 요약

1. 대상: Gallager 이진 `(n,q,r)` regular LDPC(열 weight `q`, 행 weight `r`, rate `R≥1-q/r`)를 sequential decoding으로 복호.
2. 문제: 표준 sequential decoding은 계산량 기댓값이 유한하려면 rate `<R0`(computational cutoff rate) 필요 — `R0` 위에서 동작시키려는 시도.
3. 모델: 주로 BEC(binary erasure channel), 보조로 BSC. 채널출력 `y`로 parity-check 순서를 결정.
4. 핵심기법1(unpruned codeword tree): parity-check 순서 리스트로 부분지정 n-tuple 트리를 생성, 각 parity check가 한 level. LDPC 희소성 덕에 fanout이 `2^{r-1}` 이하로 제한.
5. 핵심식: level별 노드수 `N_i`, MNE 상한 `M_i = 2^{a_i} M_{i-1}`, `a_i = e_i + δ_i - 1` — `e_i`는 해당 level n-set의 new erasure 수, 트리 성장을 정량화.
6. 핵심기법2(SDR/MNE 순서화): 비신뢰 심볼(erasure)을 트리 뒤쪽으로 밀어 트리 성장·오검색을 줄이는 greedy heuristic `E(C)=Σ f(a)`; BEC용 MNE는 new erasure 수 최소화.
7. 구현 디테일: MNE + stack algorithm 결합, 효율 구현은 `O(qn)` 연산·`O(qn)` 메모리; 코드 제약(두 parity set이 1개 초과 공유 금지).
8. 최적화 절차: 없음(수신값 기반 그리디 순서 선택뿐, 학습 없음).
9. 결과: (396,5,6) 등에서 MNE가 random ordering 대비 우수. rate 0.7 (1000,3,10) 부호는 `ε_MNE≈0.245 > ε_SD=0.2311`로 R0 초과 가능성 시사(Table I).
10. 한계: 시뮬만, HW·양자화 없음. BSC에서는 성능 부진, 트리가 앞쪽에서 빨리 성장하는 구조적 제약.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |

| MNE parity-check 재배열(스케줄) | 대응 없음 | Prime ECC 스케줄은 dual-update(even/odd)로, tree 순서화와 무관 |
| unpruned codeword tree / R0 분석 | 대응 없음 | 순수 계산량 이론 |
```

마지막에 한 문장으로 **적용 가치**: `낮음` — sequential/stack tree-search 복호는 Prime ECC의 iterative min-sum message passing과 계열이 달라 떼어 쓸 알고리즘·구조가 없다.
