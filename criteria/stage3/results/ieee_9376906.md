### B. 알고리즘 요약

1. 대상 시스템은 CV-QKD 정보조정용 side-information(SI) 압축, AWGN·BPSK 채널, 초저 SNR(-0.6dB) 동작이 요구된다.
2. 부호는 QC-LDPC(base matrix `Hb` + circulant offset, 확장인자 `q`), irregular, 길이 `L`=262144/349952, rate 0.430/0.115.
3. 풀려는 문제: 기존 FPGA 디코더는 CV-QKD가 요구하는 초저 SNR 성능·초장부호를 만족 못함.
4. 핵심 기법 1단: `SI-LBP`(side-information layered belief-propagation) = layered sum-product 스케줄로, min-sum보다 성능 좋은 SPA를 채택.
5. 노드 갱신식 `M_ji = LLR - E_ji`, `E_ji = (1-s_j)·Πsgn·φ(Σφ(|M|))`, `LLR = M_ji + E_ji`; `φ(x)=-log tanh(|x|/2)`.
6. layered 스케줄은 V2C 메시지 저장이 불필요해 초장부호 저장공간을 절반으로 줄이고 반복수를 줄여 throughput↑.
7. 구현 트릭: 고정소수점 (1,5,13) 균일 양자화(LLR·노드메시지 공용), `φ(x)`를 2차함수 `ax²+bx+c` 비균일 4구간 piecewise 근사(DSP slice 16개/NPU).
8. HW 아키텍처: partial-parallel(`p=q=64`) + 파이프라인(층·노드 간 빈 클록 없음), read&write 충돌 방지용 PCM 구성 제약(인접 비영원소 간격 확대).
9. 결과: Virtex-7 XC7VX690T에서 SNR=1.0dB시 throughput 108.64/70.32 Mb/s, 최소 성공 SNR -0.6dB, 고정소수점-부동소수점 BER차 <10⁻⁴.
10. 한계: throughput이 이전 연구 대비 낮음(초장부호 탓), rate-adaptive 미지원, CV-QKD 특화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered SPA 디코딩 루프(SI-LBP) | decoder.cpp LDPC_Decoder(), C2V_Cal() | Prime는 min-sum 기반, full-SPA·φ(x)는 대응 없음에 가까움 |
| φ(x)=-log tanh 2차 piecewise 근사 | 대응 없음 | Prime는 tanh 대신 min1/min2 근사 사용(§3), 중복/비호환 |
| 고정소수점 양자화(1,5,13) | decoder.cpp VN_Cal_HD() 등, ecc_data.h PARAM_LLR | magnitude 양자화 테이블과 개념 대응하나 bit폭 다름 |
| partial-parallel(p=q) 파이프라인 | 디코더 HW 모델(z=32, pipeline) | z=32 vs q=64 구조 상이, 정합 부담 |
| 충돌방지 PCM 구성 제약 | ecc_top.cpp Load_PCM() | Prime는 특정 QC-LDPC 고정, 재구성 부담 큼 |
```

적용 가치: **낮음** — full-SPA φ(x) 근사는 Prime의 min-sum 대비 HW 부담 크고, 초저rate·초장부호 CV-QKD 스펙이 NAND 고rate ECC와 정반대라 떼어 쓸 요소가 없음.
