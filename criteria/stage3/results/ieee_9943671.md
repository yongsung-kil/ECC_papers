### B. 알고리즘 요약

1. 대상은 protograph 기반 GLDPC 부호로, 일부 SPC check node를 GC(generalized constraint) node로 대체(doping)한 구조다.
2. 문제: PEG lifting의 greedy 특성상 나중에 처리되는 VN에 local cycle이 몰려 부호 성능이 불균형해진다.
3. 관찰: GC node에 연결된 `doped VN`은 강한 component code(예 (7,4) Hamming)로 보호되어 local cycle에 강건하고, `undoped VN`은 취약하다.
4. 핵심 기법: 취약한 undoped VN을 PEG에서 먼저 lifting(적은 cycle 생성), 강건한 doped VN을 나중에 lifting해 전체 cycle 분포를 균형화한다.
5. 실험: 동일 base matrix (1)에서 doped VN 위치만 앞/뒤로 바꾼 두 base matrix (2)로 `C1`(former), `C2`(latter) 부호 집합 생성.
6. 파라미터: lifting factor `z = 15, 30`, tie girth 시 CN 랜덤 선택, randomness 보정 위해 case당 50개 부호 생성해 평균 비교.
7. 디코딩: GC node는 ML(APP) 디코딩, 전체는 iterative decoder. 채널은 BEC, AWGN.
8. 결과(BEC): `ε=0.6`에서 C2(z=30)가 C1 대비 15배 낮은 FER.
9. 결과(AWGN): FER `10^-4`에서 C2(z=15)가 C1 대비 약 0.5dB 이득. 부호 간 편차도 C2가 3배 이내(C1은 400배)로 훨씬 안정적.
10. 한계: 2쪽 분량, HW 미설계, GLDPC 전용 개념(doping)에 의존, 정확한 rate/length·복잡도 미기재.
