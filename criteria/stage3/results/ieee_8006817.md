### B. 알고리즘 요약

1. 시스템: QPSK/AWGN 광전송, 내부 soft-decision LDGM + 외부 hard-decision staircase의 concatenated 부호, 예제 rate `5/6`(및 20/23, 4/5).
2. 문제: soft-decision LDPC/SC-LDPC는 NCG는 높지만 디코더 복잡도(operations/bit)·전력·발열이 크고 error-floor를 후처리/외부부호로 낮춰야 함.
3. 모델: EXIT chart로 메시지 LLR 평균 `μ`의 iteration 전이함수 `μo=g(μi)`, 안정 고정점 `μf`로 부호족 특성 분석.
4. 핵심기법1: 내부 LDGM은 "error-reducing code"로 완전정정 대신 BER만 충분히 낮춰 외부 staircase의 입력 BER(`psc`) 조건을 만족.
5. 핵심식: 복잡도 score `η = E·I/K + P` (E=graph edge수, I=최대 iteration, K=정보비트, P=staircase 연산). operations/bit에 비례.
6. 핵심기법2: 주어진 SNR·rate에서 `ηi=EI/K` 최소화하는 LDGM ensemble `ρ(x)`를 convex 최적화로 도출(변수노드 degree는 Poisson `α` 가정).
7. 트릭: iteration 수 `J` 근사식(3)을 LDGM용으로 수정(check message 초기값 0, 부호 반전)하여 `ηi≈αJ/Rsc`로 계산.
8. 최적화절차: 3중 loop(SNR × staircase표 × `α`) 검색 + U에 대한 binary search로 Pareto frontier(NCG vs η) 특성화.
9. 결과: rate 5/6에서 MS 디코딩 시 동일 NCG에 대해 기존 대비 η를 최대 57%(flooding), layered 대비 25% 절감.
10. 한계: HW 미설계, 부동소수 BP/MS 시뮬만, η는 flooding 병렬 가정으로 latency 미반영, 광통신 채널 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 내부 LDGM soft-decision 디코딩(BP/MS) | `decoder.cpp` `LDPC_Decoder()` | LDGM은 generator-matrix 구조로 우리 QC-LDPC PCM과 다름, 직접 대응 약함 |
| 외부 staircase(BCH component) 디코딩 | 대응 없음 | Prime ECC는 binary QC-LDPC 단독, concatenation 미지원 |
| 복잡도 score η(edge·iteration 기반) | 대응 없음 | HW 면적/전력 지표는 우리 gatecount/pipeline 모델과 별개 |
| EXIT 기반 ensemble 설계 | `ecc_top.cpp` `Load_PCM()` | 부호설계 레이어이나 고정 QC-LDPC라 재설계 부담 큼 |

적용 가치: **낮음** — 광통신용 LDGM+staircase concatenation·복잡도 최적화로, NAND binary QC-LDPC 부호·디코더 구조와 정합되지 않는다.
