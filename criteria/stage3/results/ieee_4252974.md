### B. 알고리즘 요약

1. 대상은 regular time-invariant LDPC convolutional code(LDPCCC), rate `R=b/c`, semi-infinite syndrome former matrix `H^T`로 정의, 시뮬레이션 코드 파라미터(rate/length) 미상.
2. LDPC block code는 message-passing 병렬 구현 시 interleaving에 따른 메모리 충돌·라우팅 복잡도가 문제이며, LDPCCC는 그래프의 locality/regularity로 이를 완화하려는 시도.
3. 디코딩 그래프 연결이 시간불변(critical distance `ms+1`)이라는 가정 하에 3D(time `t`, depth `d`, iteration `i`) 신호처리로 모델링.
4. graph-level(iteration/time/depth 분할), node-level(SIMD 벡터 연산), operation-level 3단 병렬화 전략 제시, 이 논문은 graph-level (a)(b)와 node-level에 집중.
5. node-level 병렬화는 processing window 길이 `pt`의 SIMD 연산, check/variable node 연산 시 벡터 정렬 불일치를 shuffle-network(cyclic rotation, 회전수 `pt+1`)로 해결.
6. 확장은 VLIW(instruction-level) + SIMD(data-level) 결합 프로세서 아키텍처(CALU/VALU, AGU, 이중포트 벡터 메모리).
7. 구현 디테일: 이중포트 RAM으로 load/store 파이프라이닝 시 약 1.7배 속도 향상, throughput 식 `T=(K-J)·pt / (tCN·J + tVN·K)`.
8. 별도 학습/최적화 절차 없음(파라미터 pt 스윕으로 throughput-area trade-off 분석).
9. 결과: 250MHz 클록에서 1Gbps 달성에 필요한 병렬도는 `100≤pt≤220`, code-specific shuffle-network가 full shuffle 대비 면적 2.4~3.8배 절감(pt=64/32 기준).
10. 한계: 정정성능(BER/FER) 결과 전무, 실리콘 프로토타입 미완성(합성/place-route만), QC-LDPC block code와 부호 구조가 근본적으로 다름(convolutional).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SIMD 벡터 기반 CN/VN 병렬 연산(pt-parallel) | 디코더 HW 모델 (z=32, HCU, GT) | z=32 lifting 병렬 구조와 개념적으로 유사하나 convolutional 전용 |
| shuffle-network(cyclic rotation, 메모리 정렬) | `GT.cpp` `Make_GT_HW()` | circulant shift 라우팅과 유사 개념이나 코드 구조 상이 |
| VLIW+SIMD 프로그래머블 프로세서 아키텍처 | 대응 없음 | Prime ECC는 고정 파이프라인 bit-exact 시뮬레이터, 프로그래머블 프로세서 개념 자체가 대응 없음 |
| LDPCCC H(D) polynomial parity-check 구조 | `ecc_top.cpp` `Load_PCM()` | QC-LDPC의 base+circulant 구조와 다른 convolutional 형식, 이식 난이도 높음 |
| throughput/latency/memory 해석 모델(식 3~7) | 대응 없음 | 수식적 성능 모델로 코드베이스 대응 모듈 없음 |

> 적용 가치: 낮음 — LDPC convolutional code 전용 아키텍처로 부호 형식이 Prime ECC의 QC-LDPC block code와 근본적으로 다르고, 정정성능 개선 요소가 없어 순수 HW 병렬화 아이디어(shuffle-network 라우팅 절감) 정도만 참고 가능.
