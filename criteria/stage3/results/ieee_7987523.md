### B. 알고리즘 요약

1. 대상 부호: array QC-LDPC, base matrix `B`를 확장계수 `z`로 lifting, `N = z × ncol`; 실험은 regular rate 3/4, dv=3, N=1296(z=108, 3 layer).
2. 문제: layered MS 디코더에서 barrel shifter(가변 shift)와 AP-LLR 메시지 메모리가 면적·throughput 병목 → 수십 Gbps 달성이 어렵다.
3. 핵심 기법1(layer unrolling): 각 layer를 별도 HW 블록으로 펼치고 layer 사이에 pipeline register 삽입, 한 layer 처리 = 1 clock cycle.
4. 핵심 식: layered Min-Sum `α_{m,n}=γ_n−β_{m,n}`, `β̃_{m,n}=∏sign·min|α|`, `γ̃_n=α_{m,n}+β̃_{m,n}` (AP-LLR `γ` 갱신).
5. 데이터 해저드: 다중 codeword 동시 처리(`nl`개)로 pipeline stall 없이 층을 채워 처리량을 layer 수만큼 증대.
6. 라우팅: layer간 shift량을 hard-wire로 고정해 가변 shifter 제거, layer0 입력만 2-to-1 MUX로 새 codeword LLR 선택.
7. 메모리: β는 SRL(shift register)에 compressed 포맷(sign, min1, min2, index)으로 저장; AP-LLR은 층간 pipeline flip-flop으로 대체(별도 메모리 제거).
8. MS 변형: SCMS/SCMS:v2(erasure 저장 없는 memory-friendly 버전) 지원, waterfall 개선하나 별 추가 이득은 부호정정보다 스케줄 효율에 집중.
9. 결과(FPGA Virtex-7): MS(2,4) 197MHz에서 62.8Gbps, MS(4,6) 34.3Gbps; 4 iteration/FER<1e-4, [10] 대비 3~4x, [14] 대비 50x throughput, BRAM·DSP 0.
10. 한계: FPGA LUT-SRL 조직에 크게 의존, full-layer/array code(소수 layer) 전제, 정정능력·iteration 개선은 없고 ASIC 이득은 미확인.
