### B. 알고리즘 요약 (10줄 내외)

1. 대상: left-regular(variable-regular) LDPC, variable node degree `dl=3,4,5,6`, girth `g=6,8`; 검증 부호는 TSF `(155,64)`, PEGReg `252×504`.
2. 풀려는 문제: error-floor의 주범인 elementary trapping set(ETS, 유발 check node degree 1/2뿐)의 그래프 구조를 부호별로 보증적으로 열거·특성화.
3. `(a,b)` ETS 정의 = variable node `a`개, odd-degree(unsatisfied) check node `b`개; 작은 `a,b`가 dominant.
4. 1단(구조 열거): normal graph 표현(degree-1 CN 제거, degree-2 CN을 edge로 치환)으로 nauty 그래프동형 프로그램의 탐색공간을 대폭 축소(예 `(6,6)`: 5.4천만→11 그래프).
5. 핵심 개념: layered superset(LSS) — ETS `S`가 짧은 cycle `C`에서 시작해 한 노드씩 늘어나는 중첩열 `C=S(0)⊂...⊂S=S(a-α)`로 도달 가능.
6. 2단(탐색 알고리즘): Routine 1(`OneExpansion`)이 `O2(S)`(unsatisfied CN에 2개 이상 연결·satisfied CN 무연결 노드)만 추가 → Algorithm 1이 short cycle에서 크기 `k`까지 확장.
7. 복잡도: 한 스텝 `O(b·dr)`, 메모리 `O(a·b·dr)` (dr=check degree).
8. 핵심 결과: 임의 `dl,g`에 대해 `a<α, b<β`인 모든 dominant ETS class가 short cycle의 LSS임을 증명 → cycle 시작 보증탐색 성립(각 class별 필요 cycle 길이 표 제공).
9. 검증: TSF(155,64)는 4-bit min-sum, SNR 6.5dB error-floor에서 300 block error 전부가 표의 ETS(주로 465개 `(8,2)`); PEG252×504는 3-bit min-sum, SNR 5.5dB에서 150중 149가 표의 ETS. Algorithm 1이 기존 exhaustive 대비 대폭 빠름(7h→10min급).
10. 한계: HW 미설계, ETS 분석/탐색 툴에 한정(디코더·부호 자체 개선 아님); 일부 구조(NA)는 어떤 cycle의 LSS도 아니어서 미포착, irregular 확장 시 구조 폭증.
