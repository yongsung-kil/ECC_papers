### B. 알고리즘 요약

1. 시스템: 코히어런스 구간 N비트의 블록페이딩 채널로 LN 부호비트를 전송, SC-LDPC(길이 LN, L개 결합 LDPC 블록)로 부호화. 대상 rate R=1/2, 2/3, 3/4.
2. 문제: 기존 root-LDPC 등은 특정 d(대개 d=2)에만 설계되어 페이딩 계수 수가 바뀌면 잘 동작하지 않음. 임의 d를 목표로 하는 체계적 설계법 부재.
3. 채널 모델: `Yi = aj*xi + ni`, BPSK, 레일리 페이딩 계수 `aj`(블록 단위 고정), AWGN.
4. 핵심 기법: base matrix `B`(예 [3,3])에서 memory constraint `mcc`를 갖는 convolutional protograph `Pc`를 구성, component matrix `B0..Bmcc`로 edge spreading 결정.
5. 핵심 식: 트렐리스 인접행렬 `A`의 L제곱으로 Hamming weight enumerator `W(x)=[A^L]0,0` 계산, 최소차수 지수가 `dmin`=diversity d.
6. 확장(핵심): 반복복호 다이버시티 `dIT`를 blockwise minimum stopping distance `s^B_min`로 정의하고, 이를 `d^B_min`까지 끌어올리는 재귀 알고리즘.
7. 재귀 절차: (Split) `Bmcc`를 최소 edge로 두 파트로 분할, (Add) all-zero 행렬 삽입으로 2블록 소실을 1블록 문제로 환원, 반복하여 목표 `dIT` 달성.
8. 학습/최적화: 별도 NN/GA 없음, 조합적 재귀 구성. 초기 옵션(Option 1/2) 선택으로 edge 소모 최적화.
9. 결과: (3,6)-regular에서 dIT=6 (mcc=9), (4,8)에서 dIT=8 (mcc=15) 달성. Table III: mcc=9일 때 추정 dIT=10, latency 10N.
10. 한계: HW 무설계, density evolution outage(DEO) 이론 상한만 제시, 블록페이딩 무선 채널 가정에 국한.

|---|---|---|

| 다이버시티 차수 목표 설계 | 대응 없음 | 블록페이딩 다이버시티는 NAND AWGN/RBER 채널에 개념적 대응 없음 |
| dmin/stopping set 분석 | 대응 없음 | 이론 분석 도구, 코드베이스에 해당 모듈 없음 |
