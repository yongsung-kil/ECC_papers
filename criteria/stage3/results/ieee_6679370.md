### B. 알고리즘 요약

1. BSC(교차확률 `ε`) 채널 + (dv,dc)-regular LDPC; 시뮬 코드는 (3,6)/(5,10)-regular, length 204·2640.
2. 문제: 나노스케일 소자의 unreliable HW에서 게이트 transient 오류와 메모리 stuck-at(permanent) 오류가 Gallager B 복호기 성능을 얼마나 떨어뜨리는가.
3. 모델: VN/CN 연산 flip 확률 `σv,σc`(transient), 입력·V2C·C2V 메모리별 stuck-at-0/1 비대칭 확률 `γ,α,β`로 오류전파를 비대칭 채널로 모델링.
4. 핵심 기법 1단: 코드워드 상대가중치 `w`에 의존하는 type-0/type-1 메시지별 exact density evolution 유도(식 4), 비트오류 `p_x^(ℓ)` 재귀식.
5. 핵심 식 의미: 비대칭 stuck-at 때문에 all-zero 코드워드가 대표성을 잃어 `w` 파라미터를 채널에 흡수한 근사 DE `p_app`를 별도 유도.
6. 확장: residual error `pr ≈ ᾱw+σv` (dv>3), 즉 VN transient·V2C permanent 오류가 잔차의 주 기여.
7. 구현 디테일: 패리티검사식과 기존 HW를 재사용해 추가회로 최소로 defective 메모리셀 위치 탐지(2·dv 추가 iteration, J셀만 검사하는 저오버헤드 변형).
8. 최적화: VN voting threshold 최적값이 소조건에서 ⌈dv/2⌉임을 증명, march test 병용으로 false alarm 감소.
9. 결과: 검출스킴이 라운드마다 미발견 permanent 오류의 약 절반을 제거, residual error를 transient rate(5e-4) 아래인 ~1e-5까지 하강; 이론과 시뮬 일치.
10. 한계: HW(RTL/ASIC) 미설계·게이트수/throughput 없음, 1-bit hard Gallager B 전용, 결함 하드웨어라는 특수 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager B majority-voting 복호 루프 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 Gallager B(hard-voting)와 알고리즘 상이, 직접 대응 아님 |
| 패리티검사 재사용 검출·정정 | 대응 없음 | 결함 메모리셀 자가검출은 신뢰 HW 전제인 Prime ECC 범위 밖 |
| stuck-at/transient 결함 HW 모델 | 대응 없음 | `channel.cpp`는 채널 오류 주입용, 디코더 내부 HW 고장 모델 미보유 |

적용 가치: **낮음** — 결함 하드웨어에서의 hard-decision Gallager B 신뢰성 분석이 주제로, soft min-sum·신뢰 HW 전제의 NAND binary QC-LDPC 코드베이스에 떼어 쓸 요소가 없다.
