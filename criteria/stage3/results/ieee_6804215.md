### B. 알고리즘 요약

1. 시스템: (3,ρ,8) permutation/QC-LDPC를 global code, 2-error correcting BCH를 component code로 하는 check-hybrid GLDPC, BSC 채널, 예제 길이 1335·9703.
2. 문제: 반복 디코더 실패의 원인인 small trapping set(short cycle 결합)을 제거하되, 기존 GLDPC의 무작위 super check 치환에 따른 큰 rate loss를 줄인다.
3. 모델: all-zero codeword 전송 가정, `T(a,b)` trapping set(변수노드 a개, odd-degree check b개), PBF/Gallager B 디코더 실패 구조 분석.
4. 핵심기법1: trapping set의 degree-2 single check 중 선택된 것만 super check로 치환 → 각 super check에서 bounded distance decoding(BDD)로 trapping set 내 오류 정정.
5. 핵심식: rate 하한 `R ≥ 1 − γ/ρ − κλ(1−r)` (`κ`=치환 super check 수, `λ=ρ/N`), rate loss를 정량화해 최소화 목표 설정.
6. 핵심기법2: critical set(치환하면 trapping set이 harmless가 되는 최소 check 집합)과 splitting number `s_(a,b)(H)` 정의, Algorithm 2로 cycle 분할(super check를 degree-1 single check 2개로 split)해 탐색.
7. 트릭: subdivision 이론으로 (7,5)=(6,4) subdivision 등에서 splitting number 동일함을 보여 계산 없이 재사용, s(4,4)=1·s(5,3)=2 등 상한 도출.
8. 이론: permutation matrix 크기 p 기준 `s_(a,b)(H) ≤ 2p`(변수노드당 super check 2개 시 모든 elementary trapping set 제거), (4,4)는 girth-12 조건 시 s ≤ p (Theorem 2), 결과 fixed set 특성화.
9. 결과: (3,15,8) global+BCH(15,7) 및 (3,31,8) global+BCH(31,21)로 CH89/CH178/CH313/CH626 구성, Gallager B 하에서 BSC error-floor 개선(수치는 그림, 미상).
10. 한계: HW 미설계, BSC/hard-decision(Gallager B) 시뮬만, GLDPC(BCH BDD) 구조 변경 + rate loss, guaranteed error correction 미완.
