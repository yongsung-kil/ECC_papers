### B. 알고리즘 요약 (10줄 내외)

1. 대상: NR 표준 (704,352) QC-LDPC, `rate=0.5`, base-graph 1, `lifting Z=16`, 최대 iteration 5, BPSK/AWGN (원래 (736,352)에서 앞 2 VN puncture).
2. 문제: offset-min-sum(OMS)의 offset 값 `wij`가 해석적 최적해가 없음 - 이를 딥러닝으로 최적화.
3. 기존 한계: trellis 기반 NN([13],[14])은 hidden node 수가 Tanner edge 수와 같아 복잡도 매우 높고 flooding만 지원.
4. 핵심 기법: OMS의 CN update `αij = (∏sign)·max(min|β|−wij, 0)` (식2)에서 error function J를 back-propagation으로 offset에 대해 직접 미분.
5. 핵심 식: `dJ/dwij = −sign(αij)·Δα_ij` (식4-5), Δα/Δβ 메시지가 forward BP의 V2C/C2V와 동형으로 전파(식6-10).
6. 확장: layered 스케줄 지원 - 인과성 제약(`k1<i`이면 같은 iteration의 최신 `α` 사용)으로 gradient 식만 수정(식12-16).
7. 구현 디테일: offset 제약 3단계 - edge/iteration별(EL개), CN/iteration별(mL개), 전체 1개(conventional OMS).
8. 학습: all-zero codeword를 AWGN 통과→BP 복호, 중간 V2C/C2V 메시지 저장, batch 100으로 gradient descent (GPU 불필요).
9. 결과: BCH(15,11) 3-iter 기준 trellis 800 edge vs 제안 160 edge, NR BG1 Z=384는 6×10^6 vs 6×10^5 edge; offset 최적화로 BLER 유의미 개선. cost는 BER엔 cross-entropy, BLER엔 p-norm.
10. 한계: HW 미설계, 시뮬만, offset은 오프라인 학습 산출물(런타임 NN 아님), 성능은 학습 데이터/SNR 의존.

| iteration/CN별 offset 테이블 | [ecc_data.h](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `PARAM_LLR`, [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Get_VNU_Table_Idx()` | 학습된 offset을 iteration 구간별 테이블로 적재 가능 (기존 LLR 테이블과 유사 구조) |
| 이터레이션 루프/스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | Dual-Update(layered류) 스케줄과 offset 학습 정합 확인 필요 |
| back-propagation 학습기 | 대응 없음 | 학습은 오프라인 별도 도구, 코드베이스엔 산출된 offset 테이블만 이식 |

적용 가치: **중간** - 학습된 offset 테이블만 떼어 Prime ECC min-sum에 적용하면 waterfall 성능 개선 여지가 있으나, Prime ECC가 offset-min-sum이 아닐 수 있고 오프라인 학습 파이프라인·CN별 테이블 저장비용 검증이 필요.
