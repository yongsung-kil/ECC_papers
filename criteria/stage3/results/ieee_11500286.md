### B. 알고리즘 요약

1. 시스템: CV-QKD(연속변수 양자키분배)의 information reconciliation(IR) 단계에서 극저 SNR 채널 오류를 정정하는 QC-LDPC 디코더, ATSC 3.0 규격 부호(길이 `N=64800`, rate `3/15`) 사용.
2. 문제: layered belief propagation(L-BP) 방식은 인접 층(row)이 동일 변수노드(메모리 주소)를 공유해 현재 층이 이전 층의 write 완료를 기다려야 하는 read-write hazard가 발생, 이를 막으려 idle cycle을 삽입하면 처리량이 저하됨.
3. 핵심 가정/모델: 표준 layered decoder 구조(PCM RAM/LLR RAM/Lc2v RAM 3종 메모리) 위에서, quasi-cyclic PCM의 `q×q` circulant/zero 서브행렬 구조를 활용해 주소 충돌 패턴을 사전에 분석 가능하다고 가정.
4. 핵심 기법: base matrix `H`를 전처리해 정규화 행렬 `H'`(Extraction/Offset/Address Matrix)로 변환한 뒤, 주소가 충돌하지 않는 row들을 재정렬(row reordering)하여 그룹핑하고 해당 데이터를 merge하여 저장하는 cross-layer scheduling.
5. 핵심 식: LLR 갱신은 표준 min-sum 계열 layered decoding 식(`L(rli,jk) = sgn(...)·Ψ(Σ Ψ(L(qlj',ki)))`, `Ψ(x)=-log(tanh(|x|/2))`)을 그대로 사용하며, 이 식 자체가 아니라 이를 계산하는 스케줄 순서를 재배열하는 것이 핵심.
6. 확장: 충돌 없는 row 그룹들을 병렬로 read/update함으로써 전통적 층간 순차 접근에서 발생하는 대기 사이클을 완전히 제거.
7. 구현 디테일: `Ψ(x)` 함수는 lookup table로 양자화(19-bit 고정소수점), sign 곱은 XOR 모듈, 누산은 Sigma 모듈로 구현; 메모리는 주소 스위칭만으로 read/write 관리.
8. 최적화 절차: 학습·최적화 알고리즘은 없음(row reordering은 PCM 구조 분석 기반 결정적 전처리).
9. 결과: 클록 250MHz에서 throughput 100.5Mbps 달성, 크로스-레이어 스케줄링 미적용 baseline 대비 약 20% 처리량 향상, FER 성능은 최적화 전후 거의 동일(그림5, 성능 저하 없음).
10. 한계: FPGA(Virtex-7) 구현·검증에 그치고 ASIC 합성 결과 없음, 저rate(3/15)·긴 길이(64800) CV-QKD 특화 코드라 NAND 고rate 짧은/중간 길이 코드로 그대로 이식 시 자원(LUT/FF) 규모 재검증 필요, min-sum 자체의 정정성능 개선은 다루지 않음(순수 스케줄링/메모리 최적화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered decoding 층간 메모리 충돌 해소 (row reordering + data merging) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC도 layered/pipeline 구조의 CN/VN 업데이트 스케줄을 가지므로 base matrix row 재배열을 통한 충돌 회피 아이디어가 파이프라인 스테이지 설계에 직접 참고 가능 |
| Ψ(x) 함수 lookup table 양자화 | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum 근사 및 magnitude 양자화 테이블 방식이 Prime ECC의 기존 min-sum 구현과 유사한 접근으로 신규성은 낮으나 검증 가능 |
| Extraction/Offset/Address Matrix 기반 PCM 전처리 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 시 주소/오프셋 사전계산 방식이 유사한 전처리 개념으로 참고 가능 |
| HW pipeline/병렬 처리 구조 | `decoder.cpp` pipeline(col_M1/Z/P1~P4), `GT.cpp` `Make_GT_HW()` | Graph Thinning의 edge 선택과 유사하게 충돌 없는 row/edge 그룹을 선정하는 아이디어가 겹침 |

적용 가치: **중간** — layered decoding의 메모리 충돌을 알고리즘 변경 없이 스케줄링(row reordering/merging)만으로 해소해 처리량을 개선하는 HW 아키텍처 기법으로, decoder 알고리즘 자체의 정정성능 기여는 없지만 Prime ECC의 pipeline/HCU/Graph Thinning 스케줄링 최적화에 실질적으로 참고할 수 있음.
