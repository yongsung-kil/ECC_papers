### B. 알고리즘 요약

1. 대상: LDPC, repeat-accumulate(RA), turbo-like 코드를 포괄하는 sparse parity-check matrix(SPCM) 코드의 반복 복호.
2. 표준 two-phase(LDPC-like) 스케줄은 variable/check 두 라운드로 나뉘어 같은 이터레이션 내에서 새로 계산된 메시지를 즉시 활용하지 못해 수렴이 느리고 메시지 두 종류(check/variable) 저장으로 메모리 소모가 큼.
3. 채널모델은 AWGN+BPSK, 표준 sum-product 메시지패싱 프레임워크 가정.
4. TDMP(Turbo-Decoding Message-Passing): parity-check 행렬 H의 행(row, parity-check 식)을 순차 처리하며, 각 행마다 posterior message 벡터 `r`과 extrinsic message 벡터 `e_i`만 유지(메시지 타입 1종으로 통합).
5. 각 row 처리는 4단계(Read: `r`,`e_i` 읽기 → Subtract: 사전확률 `p_i = r - e_i` 생성(extrinsic 원칙 유지) → Decode: SISO(식 4, 또는 BCJR)로 새 `e_i` 계산 → Write-back: `e_i` 갱신 및 `r += (new e_i - old e_i)`)로 구성, 최신 메시지를 같은 이터레이션 내 다음 row 계산에 즉시 반영.
6. 병렬화판(P-TDMP): H를 겹치지 않는 row-band(cluster) 구조의 architecture-aware SPCM(AA-SPCM) 코드로 설계해 여러 row를 동시에 SISO 디코더 다수로 처리, Benes망 같은 프로그래머블 permuter로 클러스터 단위 라우팅.
7. 메시지 계산은 `φ(x)=-log(tanh(|x|/2))` 대신 근사식 `Φ`(간단한 max 연산 기반, 식 7)로 대체해 단순 로직게이트로 구현 가능, 정확도 손실은 무시할 수준(그림3, ideal 대비).
8. 코드 설계는 girth가 큰 그래프(Ramanujan graph, cage graph)로부터 base Tanner graph를 구성 후 확장(expansion)하는 절차 사용, EXIT chart로 수렴 분석.
9. 결과: rate-1/3 length 3456 RA 코드에서 두 배 빠른 수렴(예: 1dB에서 20 vs 10 iteration), 고SNR에서 BER 약 10배(1자릿수) 개선, (3,6) regular 코드 기준 메모리 절감 약 43.75%.
10. 한계: 알고리즘/시뮬레이션 수준 검증뿐, ASIC/FPGA 합성 결과 없음(단, 데이터플로우 그래프 및 Benes망 트랜지스터 수 추정치만 제시); P-TDMP는 특정 구조화 코드(AA-SPCM) 전제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Row-serial(layered) 스케줄, read-subtract-decode-writeback | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 이미 보유한 Dual-Update(홀짝 교대 스케줄)와 유사 계열의 layered 스케줄 원조 개념, 직접 비교 가능 |
| 단일 메시지 타입(posterior) 통합, 메모리 절감 | `decoder.cpp`, `ecc_data.h` `PARAM_DEC` | 메시지 저장 구조 최적화 아이디어로 메모리 라우팅 개선 참고 가능 |
| Φ-함수(간단 max 근사) 메시지 계산 | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum과 유사하게 tanh 근사를 단순연산으로 대체하는 개념, 이미 Min-Sum 채택 중이라 중복 기여 |
| AA-SPCM 코드 설계(girth 기반 그래프 확장), Benes network permuter | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체 변경으로 이식 난이도 높음, 랜덤 permuter 대신 구조화 permuter 아이디어는 GT.cpp와 결이 다름 |

> 적용 가치: 높음 — layered 스케줄/단일 메시지 통합을 통한 수렴속도·메모리 절감 원리는 디코더 이터레이션 구조 개선에 직접 참고할 수 있는 근본 기법(2006년 시점 layered decoding의 대표 원조 논문).
