### B. 알고리즘 요약

1. 대상: IoT 엣지 에너지효율 ECC. 예제로 regular LDPC `(N=8, M=4, dv=2, dc=4)` 8비트 코드워드, 확장 예시로 `N=1296, rate=0.5`.
2. 문제: 뉴로모픽(SNN) 아키텍처가 ML 외 결정론적 알고리즘(ECC)엔 미탐색 → 하드결정 반복복호를 뉴런모델로 저전력 실행하려는 시도.
3. 모델: TrueNorth 계열 LIF(leak-integrate-fire) 뉴런, axon-neuron crossbar, tick 동기 spike({0,1}) 실행. RANC(오픈소스 시뮬+FPGA 에뮬)로 구현.
4. 기법1(GaB 매핑): Gallager B 하드결정 알고리즘의 CN suggestion `c_mv_n = (Σ_{t∈S(cm)\vn} v_t c_m) mod 2`, VN majority vote `(2)`, syndrome check `Hx^T mod 2=0`를 8개 뉴로모픽 코어(Input/VNU/CNU/Iter/Parity/Syndrome/OR/Output)로 완전병렬 매핑.
5. 핵심난점: XOR는 선형분리 불가라 LIF 뉴런 1개로 불가 → 기존엔 2입력 XOR에 3뉴런·2틱 필요.
6. 기법2(XOR-integrated 뉴런블록): 뉴런블록에 op_select 1비트+MUX를 추가해 LIF/XOR 선택, XOR를 뉴런 1개·1틱으로 처리. baseline LIF는 XOR에 `NNeurons=M(dc²+dc+1)`, 개선형은 `M(dc+1)`로 선형화.
7. 부수: register/버퍼(피드백 루프), 3·4입력 majority(가중치·leak로 tie 처리), AND/OR/NOR도 뉴런 매핑.
8. 최적화(없음): 학습 불필요, 결정론적 매핑. serial Python GaB와 tick-by-tick 일치 검증.
9. 결과: 288워드 데이터셋 tick 32.57% 감소(87556->59042), 자원 +0.62%(LUT)/전력 +1.74%로 미미, 에너지 약 31% 절감(5x5). spike-count 기준 baseline 대비 2.8배 에너지 절감.
10. 한계: 8비트 toy code로 실 부호크기 미확장, RANC는 event-driven 아님이라 ASIC 대비 공정 에너지 직접비교 불가, 정정능력은 baseline과 동일(에너지/실행시간만 개선), NAND/SSD 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GaB 하드결정 디코딩 루프(뉴로모픽 매핑) | decoder.cpp LDPC_Decoder() | 알고리즘 계열(GaB bit-flipping)·계산모델(SNN)이 min-sum과 달라 프레임만 대응, 이식 아님 |
| CN mod-2/XOR·majority-vote 뉴런 매핑 | 대응 없음 | Prime ECC는 min-sum(min1/min2+sign) CN, GaB majority/XOR 뉴런 로직은 대응 없음 |
| XOR-integrated 뉴런블록(SNN HW) | 대응 없음 | 뉴로모픽 spike HW로 z=32 QC-LDPC RTL 모델과 무관 |
| syndrome check(Hx^T mod 2) | full_CRC.cpp / partial_CRC.cpp | Prime ECC는 CRC 조기종료, 본 논문은 syndrome NOR로 방식 상이 |
| 뉴로모픽 코어 mesh 라우팅/tick 스케줄 | 대응 없음(NN/딥러닝 디코더 계열) | 프로파일상 NN/뉴로모픽 디코더는 대응 없음으로 분류 |
```

적용 가치: **낮음** — GaB는 binary LDPC 하드결정이라 개념적 접점은 있으나, 구현 전체가 뉴로모픽 SNN(TrueNorth/RANC) 패러다임이라 NAND용 고전 min-sum QC-LDPC RTL(Prime ECC)에 떼어낼 부분이 없음. IoT 엣지 타깃으로 NAND 무관.
