### B. 알고리즘 요약

1. 시스템: 고rate(0.878~0.97) binary 부호(fair-density parity-check, FDPC), 대상 채널은 BEC 및 B-AWGN, 초고속 mmWave/THz/광통신 등 high-SNR·저지연 요구 시나리오 타깃.
2. 문제: 기존 LDPC(5G-LDPC 등)는 sparse 고정 degree 구조라 moderate rate에 최적화되어 있고, 초고rate(>0.87) 영역에서 polar-CRC/BCH 대비 성능 여지가 있음.
3. 핵심 가정: base matrix `Hb`는 길이 `4t`의 모든 이진 열벡터 중 Hamming weight 2이며 두 non-zero 인덱스 차이가 홀수인 것들로 구성(`n=4t²`), 모든 행 weight `2t`, `Hb`의 rank는 `4t-1`, `d_min(Cb)=4`.
4. 핵심 기법 1단(부호 구성): order-s FDPC 부호는 `Hb`의 서로 다른 permutation `π1,...,π_{s-1}`을 열 방향 적용 후 수직으로 쌓아 `H = [Hb; π1(Hb); ...; π_{s-1}(Hb)]` 구성(식 1), `r=4st`; 본 논문은 주로 order-2 사용.
5. 핵심 식의 의미: 저weight codeword 개수의 기댓값 `E{Aw}`를 조합론적으로 유도(예 `E{A4}<3/2`, `E{A6}<20`)해 minimum distance를 짧은 shortening(1~20bit)으로 쉽게 늘릴 수 있음을 보이고, 이를 통해 union-type 근사 ML bound(식 10, 12)를 도출.
6. 핵심 기법 2단(MP-PL 디코더): plain min-sum 대비 성능을 높이기 위해 weighted min-sum message-passing(스케일 `β=0.05`, 누적 LLR 업데이트)과 progressive list(PL) decoding을 결합 — min-sum이 `λit`(=4) iteration 내 미수렴 시 가장 정보량 낮은 미해결 erasure/변수노드를 골라 0/1 두 경로로 분기, 최대 list size `L=2^l`(l=16)까지 확장.
7. 부수 트릭: BEC용은 erasure 기반 path-splitting(최소 non-zero row weight 기준), 일반 잡음 채널용은 unsatisfied parity-check 개수(deficiency) 기반 path-splitting; list decoding은 min-sum 실패시에만 드물게 활성화(평균 list size ≈1.63 @ SNR 5.25dB).
8. 최적화 절차: 학습 없음, 순수 조합론적 구성(결정적 base matrix) + random permutation 앙상블 평균 분석.
9. 결과: (1023,898) FDPC, plain min-sum이 5G-LDPC(BP) 대비 B-AWGN에서 약 0.5dB 개선, error floor는 5G-LDPC의 BLER≈10⁻⁵ 대비 FDPC는 10⁻⁶ 이하로 지연; MP-PL은 저SNR에서 polar-CRC(SCL, CRC=8,L=32) 대비 BLER<10⁻⁵에서 평균 50배 낮은 복잡도, 800배 빠른 latency; n=16384, rate≈0.97 FDPC는 BCH(HDD) 대비 약 1dB 개선.
10. 한계: 전부 시뮬레이션/이론(ML 근사 bound)이며 HW 미설계, 복잡도/latency는 O(n), O(log n) 점근적 추정치와 이론적 게이트 연산 가정(µc,µa,µd 등)에 기반해 실제 회로 구현 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FDPC base matrix 결정적 구성(H-matrix, order-s 구조) | `ecc_top.cpp` `Load_PCM()` | 고rate binary 부호구조 대체 후보이나 Prime ECC는 QC-LDPC(circulant) 고정이라 재검증 부담 큼(프로파일 §4 "높음" 난이도) |
| Weighted min-sum message-passing (스케일 β, 누적 LLR) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 변형이라 CN/VN 업데이트 로직에 직접 대응, 이식 난이도 낮음 |
| Progressive List(PL) decoding (path-splitting, list 확장) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC에 list decoding 없음, min-sum 실패 시 후처리로 추가 가능한 신규 모듈 |
| 조기 성공 판정(parity-check 충족 여부) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념적으로 유사(조기종료), 다만 CRC가 아닌 parity 직접 검증 |
| Iteration별 soft-info 누적 (yi 업데이트) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 채널 독립적 누적 LLR 방식이 non-coherent 상황에 유리하다는 설계 아이디어로 LLR 테이블 설계에 참고 가능 |

> 적용 가치: **높음** — 고rate(0.87~0.97) binary 부호 구성법과 weighted min-sum + list decoding 결합이 min-sum 성능 개선/error-floor 완화라는 프로파일 관심 영역에 직접 부합하며, 특히 부호설계와 디코더 개선을 모두 포함해 정독 가치가 큼.
