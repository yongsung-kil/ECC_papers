> 총평: column-weight-2 QC-LDPC의 girth를 이론적 최댓값까지 구성하는 결정론적 알고리즘으로 학술적 가치는 있으나, degree-2 VN 구조 자체가 NAND 고rate binary LDPC(Prime ECC 규모/구조)와 궤를 달리해 이식 여지가 낮음.

### B. 알고리즘 요약

1. 대상은 column-weight-2, row-weight `j`인 regular QC-LDPC 코드 4종(Type-I: row-weight 4, Type-II: row-weight 3, Type-III1/III2: 가변 고rate)이며, 순환치환행렬(circulant permutation matrix) `I_s`로 구성된 cylinder structure graph(CSG) 기반.
2. 풀려는 문제: column-weight-2 LDPC는 encoding/decoding 복잡도가 낮고 부분응답채널에서 유리하지만 girth가 작으면 성능이 떨어짐. 기존 Zhang-Moura slope 기법은 Type-I/II girth를 각각 16/20으로만 달성했는데, 저자들은 이론적 상한(girth 16/24)을 증명하고 이를 달성하는 알고리즘을 제시.
3. 핵심 가정/모델: Tanner graph의 대칭구조를 "slope" `s = i2 - i1`(두 인접 vertex-set 간 edge의 index 차)로 표현하고, admissible slope pair(ASP) 개념으로 그래프의 cycle 구조를 modular arithmetic(mod N) 방정식으로 환원.
4. 핵심 기법 1단: Type-III cycle(벨트형이 아닌 cycle)을 정수열 `T=(t_v,...,t_{v+w})`가 만족하는 합동식 `Σ t_i s_i ≡ Σ t'_i s'_i (mod N)`으로 특성화(Lemma 2, Corollary 1)하여 girth를 결정.
5. 핵심 식의 의미: 이 합동식의 최소 길이 해가 곧 최소 Type-III cycle 길이(=girth의 절반 관련량)이므로, 식을 만족하는 slope 집합을 배제/탐색하면 목표 girth를 보장하는 slope 시퀀스를 구성 가능.
6. 확장(2단): Type-I/II의 girth 상한 증명을 일반화해 고rate Type-III1(rate `(l-3)/(l-1)`), Type-III2(rate `(l-2)/(l-1)`) 코드를 완전 cylinder structure graph(CCSG, 모든 vertex-set 쌍이 연결)로 구성, girth 상한 20/16 도출.
7. 부수 트릭: `g8-algorithm`(및 `g12-algorithm`)이라는 결정론적 그리디+백트래킹 탐색으로 slope 집합 `A_1,...,A_l`을 순차 구성, 실패 시 이전 단계로 되돌아가 재탐색(low-complexity, RPEG 대비 빠름).
8. 학습/최적화 절차: 무작위 탐색이 아닌 결정론적 조합 탐색(그리디+백트래킹)이며 GA/DE/NN 없음.
9. 결과: rate-0.71 Type-III1 코드에서 길이 308/728(girth 10/12)로 RPEG 기반(길이 385/840) 대비 짧은 길이 달성; AWGN 채널 시뮬레이션(sum-product, iteration 10, block 5000)에서 girth 증가에 따라 BER 개선, random-like/algebraic 코드 대비 우수한 waterfall 성능 확인.
10. 한계: column-weight 2 구조 특유의 한계(Gallager: 최소거리가 길이에 로그적으로만 증가)로 고rate NAND ECC급 강한 정정능력엔 부적합, HW 구현/복잡도/soft-info 양자화 논의 전무, error-floor 분석 없음(BER 곡선은 waterfall 영역 중심).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Slope 기반 QC-LDPC H-matrix 구성(girth 최대화) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 고rate QC-LDPC(N_b=129, VN degree 17)로 고정, column-weight-2 저degree 구조 대체는 재검증 부담이 큰 근본적 부호 교체에 해당 |
| LDPC convolutional code(Type-I/II 대응) | 대응 없음 | Prime ECC는 block QC-LDPC만 사용, convolutional 부호 개념 없음 |
| Sum-product 디코더(시뮬용) | `decoder.cpp` `LDPC_Decoder()` | 논문은 표준 sum-product만 사용(교과서 수준), Prime ECC의 min-sum과 직접 비교 대상 아님 |

적용 가치: `낮음` — column-weight-2라는 특수 저degree 구조에 국한된 girth 최적화 구성법으로, NAND용 고rate binary QC-LDPC(Prime ECC의 VN degree 17 구조)와는 근본적으로 다른 부호 클래스이며 HW/soft-info/복잡도 관점의 논의도 없음.
