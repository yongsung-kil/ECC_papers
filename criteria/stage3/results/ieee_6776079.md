### B. 알고리즘 요약

1. 대상 시스템: IEEE 802.11n WLAN용 QC-LDPC 디코더, 예시 부호는 length `1944`, rate `1/2`(rate 1/2,2/3,3/4,5/6 다중모드 지원).
2. 풀려는 문제: 802.11n PCM의 심한 data dependency 때문에 dual-path fully-overlapped 구조가 부적합 → single-path partially-parallel로 클록·전력·면적 개선 필요.
3. 기반 알고리즘: turbo-decoding message-passing(TDMP)/layered MSA, CNU+VNU+permutation network(PN)+check node message memory+LLR memory 구성.
4. 핵심 기법 1(retiming): PN이 CNU 앞에 직결되면 파이프라인 단 지연이 불균형 → PN을 LLR memory 앞으로 이동하는 circuit retiming으로 critical path 약 11% 단축.
5. 핵심 기법 2(message bypassing): VNU의 posterior LLR을 LLR memory 우회로 CNU에 직접 bypass하여 첫 row 및 적절한 phase에서 memory read/store 충돌 회피.
6. 부수 효과: LLR memory 접근 횟수 감소로 디코딩 시간 및 전력 동시 절감(register file 3뱅크 구조).
7. 양자화: 소프트웨어 floating-point 대비 HW는 8-bit fixed point 연산.
8. 결과(성능): BER 곡선상 8-bit fixed가 floating과 근접(acceptable), 정정능력 개선 주장 없음.
9. 결과(HW): 90nm CMOS, 3.67 mm², 250MHz에서 171mW, throughput 672Mbps(rate 1/2), 반복 1~7. 선행연구 [4] 대비 클록 11%↑, 디코딩시간 3%↓.
10. 한계: 부호설계·정정능력 기여 없음, 802.11n 표준·TDMP 스케줄 종속, gate count 미기재.

```
| PN retiming / 파이프라인 재배치 | 대응 없음 | 우리 파이프라인은 6-stage 고정(col_M1/Z/P1~P4), 802.11n PN 최적화 무관 |
| LLR memory bypassing(read/store 충돌 회피) | 대응 없음 | HW 메모리 마이크로아키텍처 최적화, 우리 z=32 구조와 미정합 |

```

적용 가치: **낮음** — 802.11n TDMP 디코더의 HW 파이프라인/메모리 마이크로아키텍처 최적화로, 정정능력·부호설계·soft-read 기여가 없어 NAND binary QC-LDPC에 떼어낼 알맹이가 없음.
