### B. 알고리즘 요약

1. 시스템: protograph 기반 SC-LDPC(convolutional LDPC), AWGN 채널, coupling length `L=500`, base matrix `Bt=[3 3]`, memory `mcc=2`.
2. 문제: SC-LDPC 윈도우 복호의 flooding vs serial 스케줄 성능/수렴을 이론적으로 정량 비교할 도구 부재.
3. 모델: edge spreading으로 시간 t의 VN 엣지를 t+i(i=0..mcc) CN에 분산, 윈도우 크기 `W ≥ mcc+1`.
4. 핵심 기법: protograph-based EXIT(P-EXIT) 차트를 윈도우 복호에 적용, VN→CN 외재정보 `I_Ev`와 CN→VN `I_Ec`의 MI를 엣지별 추적.
5. 식 의미: `σ_ch^2 = 8R·Eb/N0`로 채널 신뢰도 반영, MI가 1로 수렴하는 최소 Eb/N0를 복호 threshold `(Eb/N0)*`로 정의.
6. 확장: serial(shuffled) 스케줄은 각 VN 업데이트 전 연결 CN의 최신 LLR를 먼저 요청(Table 2), flooding은 전 노드 동시 갱신.
7. 구현 디테일: J(·)/J^-1(·)는 Gaussian approximation 다항식 근사, cumulative MI `I_CMI`와 평균 `I_CMI_avg`로 threshold 판정.
8. 학습/최적화: 없음 (해석적 EXIT 반복 계산).
9. 결과: W=8에서 threshold serial 0.674dB vs flooding 0.683dB; Eb/N0=0.7dB에서 iteration flooding 33→serial 19(W=8), 16→9(W=10), 약 45% 감소, 성능 손실 없음.
10. 한계: 무한 codeword 가정 이론 분석뿐, HW 미설계, SC-LDPC 특화, NAND/실측 없음.
