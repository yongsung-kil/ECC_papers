### B. 알고리즘 요약

1. 대상은 IEEE 802.3an 10GBase-T에 채택된 RS-based (2048,1723) (6,32) regular LDPC 부호(GF(64) 기반 (64,2,63) 확장 RS 코드로부터 6x32 subarray 선택해 구성).
2. 문제의식은 완전병렬 구현은 배선혼잡(interconnect가 칩면적의 절반 이상 차지 가능)을, 부분병렬은 메모리 접근충돌·저장량 문제를 야기하므로, RS-based LDPC 특유의 대수적 구조를 이용해 이를 해결하려는 것.
3. 핵심 가정/모델: parity-check 행렬 Hrs는 GF(q) coset 구조로 생성되며, 특수행/특수열을 제외한 (q-1)x(q-1) 부분이 circulant(각 행이 위 행의 우측 순환시프트)라는 property 4가 저복잡도 디코더 설계의 핵심.
4. 표준 min-sum: CNU가 `R_cv = α × Π sign(L_vn) × min|L_vn|`(scaling factor α≈0.75), VNU가 `L_vc = ΣR_mv + I_v` 계산 (식 1, 2).
5. circulant 성질을 이용해 CNU-VNU 연결을 고정 shift 규칙으로 배선(CNU i는 항상 특정 VNU 하나와만 연결), row-processing을 dc단계로 분할해 column process와 인터리빙하여 iteration당 dc 사이클로 완료.
6. 확장 기법(2단): (a) 4배 병렬 레벨 설계 - 4개 block column 동시 처리로 4q개 VNU 인스턴스화, CNU가 4개 메시지 동시처리해 iteration당 dc/4+2 사이클; (b) 비연속 열 선택(jump>1) 및 마지막 특수열 선택 시 MUX 추가로 아키텍처 확장.
7. 부수 트릭: CNU당 압축 메시지 저장(minimum, 2nd-minimum, minimum 위치, sign bit만 저장)으로 고rate LDPC 디코더에서 약 70% 메모리 절감(ref [14] 인용).
8. 학습/최적화 절차 없음 - 순수 대수적 구성(coset partition)과 HW 아키텍처 매핑.
9. 결과: 기본설계와 4배병렬설계 두 가지를 90nm CMOS로 합성, (2048,1723) (6,32) 코드에서 82만 게이트로 10Gbps 달성, 4배병렬 설계는 기본 대비 약 3배 속도에 2배 하드웨어, throughput-to-area 비율에서 기존 완전병렬·부분병렬 디코더 대비 우수.
10. 한계: 정정능력/BER 곡선 관련 수치는 본문에 없음(HW 아키텍처·게이트수·throughput 위주), 비연속 열 선택 확장은 기본(1배 병렬) 아키텍처에만 적용 가능하고 고병렬 설계와는 호환 불가로 명시, RS-based 부호의 대수적 shift 성질에 종속적이라 일반 QC-LDPC로 확장 시 재설계 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum CNU 연산(식 1, min/2nd-min/sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2+sign XOR 기반 min-sum이라 알고리즘 형태 자체는 이미 보유, 압축 저장 방식 비교 참고 가능 |
| VNU 처리(식 2) | `decoder.cpp` `VN_Cal_HD()` 등 | 표준 VN 업데이트 형태로 이미 보유한 기법과 중복 |
| circulant shift 기반 고정 CNU-VNU 배선 | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 QC-LDPC circulant shift(PCM_b) 구조를 이미 사용 중이나, 본 논문은 RS 대수적 coset 유래 shift라 부호구조(H-matrix) 재설계에 해당해 이식 난이도 높음 |
| 압축 메시지 저장(min/2nd-min/index/sign only) | `decoder.h` `Get_VNU_*` | 메모리 절감 기법으로 Prime ECC의 기존 magnitude 양자화 테이블과 비교해 추가 절감 여지 검토 가능, 이식 난이도 낮음~중 |
| 4배 병렬 레벨 확장(MUX 기반) | `decoder.cpp` `LDPC_Decoder()` pipeline 구조 | Prime ECC의 z=32 row 병렬·HCU와 목적은 유사하나 본 논문 방식은 RS 대수 구조 전용 MUX 스킴이라 직접 이식보다는 개념 참고 수준 |
| 비연속 열/특수열 선택 MUX 확장 | `GT.cpp` `Make_GT_HW()` | HW MUX 기반 edge 선택이라는 점에서 Graph Thinning과 유사한 발상이나 대상 부호구조가 달라 직접 대응은 아님 |

적용 가치: **중간** — 정정능력 개선 요소는 없고(대상=decoder, C섹션 전부 N/A) 부호구조가 RS 대수적 구성에 특화되어 있어 부호설계 레이어 이식은 어렵지만, min-sum 압축 메시지 저장 기법과 HW 면적/throughput 벤치마크(82만 게이트@90nm, 10Gbps)는 Prime ECC의 디코더 HW 최적화 참고 자료로 가치 있음.
