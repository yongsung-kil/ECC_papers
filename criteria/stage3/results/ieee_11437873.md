### B. 알고리즘 요약

1. 시스템: AWGN·BPSK 채널, binary LDPC 두 부호 — `C1`=PEGReg504×1008 (3,6)-regular, `C2`=array-based QC-LDPC 124×961 (4,31)-regular.
2. 문제: BF 복호는 저복잡도이나 BP보다 성능 낮음, GDBF는 비볼록 목적함수의 local minima에 갇힘. NGDBF가 Gaussian 잡음으로 탈출하나 HW에서 Gaussian 난수 생성·SNR추정·scaling factor 튜닝이 부담.
3. 모델: GDBF 목적함수 `f(x)=Σ x_k y_k + Σ s_i` (채널상관 + 미충족 parity 벌점), inversion function `E_k = x_k y_k + Σ_{i∈M(k)} s_i`.
4. 핵심기법: inversion function에 균일잡음을 더한 `E_k = x_k y_k + Σ s_i + u_k`, `u_k ~ U[-1,1]`. `E_k<θ`인 비트를 플립.
5. 핵심식: CDF 분석상 소분산(η²N0/2≈0.25) Gaussian 잡음의 CDF가 `U[-1,1]` CDF와 근사 → Gaussian을 균일잡음으로 대체 정당화.
6. 확장: 적응형 threshold `θ_k(t+1)=λθ_k(t)` (미플립 시 축소, `λ=0.98`).
7. 구현: `u_k = 2d_k-1`로 [0,1] 균일난수 선형변환, `d_k`는 primitive 다항식 기반 최대길이 LFSR(stage 3~5)로 생성 — Gaussian 난수의 비선형연산·SNR추정·scaling factor 불필요.
8. 학습/최적화: 별도 학습 없음, threshold(C1 -0.6, C2 -0.7)·λ만 고정 설정.
9. 결과: SNR>3dB에서 제안 복호기가 표준 GDBF 상회, 고SNR에서 Gaussian NGDBF와 동등(C2는 소폭 상회), stage-3 LFSR로도 근접 성능. NGDBF들의 평균 반복수(ANI)가 GDBF보다 낮아 빠른 수렴.
10. 한계: HW 미설계(MATLAB 시뮬뿐), gate count·throughput 없음, BF 계열이라 BP/min-sum 절대성능엔 미달, error-floor 영역 미보고.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBF/NGDBF 복호 이터레이션 | `decoder.cpp` `LDPC_Decoder()` | BF 계열 복호로 min-sum 루프와 다른 알고리즘, 대체 시 전면 교체 필요 |
| inversion function E_k(채널상관+syndrome) | `decoder.cpp` `C2V_Cal()` / `CNU_Update_New_Mag()` | CN 연산 대신 syndrome 합 사용, 개념적 대응만 |
| 적응형 threshold θ | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 임계 테이블과 유사 개념(적응형 임계) |
| LFSR 균일난수 생성 | `random.cpp` (XOR25 난수) | 난수 소스 대응, 잡음주입은 우리 복호기에 없음 |
| syndrome `s_i` 계산 | `decoder.cpp` `LDPC_Decoder()` | parity-check 계산 공통 |

마지막에 한 문장으로 **적용 가치**: `중간` — binary LDPC용 저복잡도·HW친화 복호 아이디어(LFSR 균일잡음, SNR-free)로 참고 가치는 있으나, Prime ECC는 min-sum 기반이라 BF 계열 대체는 성능 트레이드오프가 크고 HW 검증도 없어 직접 이식보다는 조기수렴·저면적 참고용.
