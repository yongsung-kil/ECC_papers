### B. 알고리즘 요약

1. 시스템: block-permutation 구조 QC-LDPC, `(1524,762)` 코드(rate `0.5`), AWGN 채널 + hard-decision 복조(그래서 hard 디코더 적용).
2. 문제: soft BP는 정확하나 복잡, 순수 hard bit-flip은 단순하나 에러확률 손해 - "hard 수준 복잡도로 soft 근접 성능"의 절충 디코더가 필요.
3. 모델: `(n,γ,ρ)` LDPC, girth≥6(4-cycle 없음), 최소거리 작지만 반복 디코딩으로 정정성능 확보.
4. 기법1(기본 bit-flip): `S=rH^T`(GF(2)), `f=SH`(정수환), `f≥b`(임계)인 위치를 반전, `b=max{f}`가 가장 단순한 선택.
5. 핵심아이디어: 반전임계 `b`가 입력 noise 수준에 의존 - noise 높으면 임계 높게, iteration 진행으로 noise 감소하면 임계 낮춰 에러확률 개선 -> iteration별 동적 임계.
6. 기법2(임계 스케줄): 특정 `b` 선택 시 bit-flip이 majority-logic와 일치(입력에러 `≤γ/2`면 1 iteration 정정 보장)한다는 성질을 절충점으로 활용.
7. 임계 산출: 채널 noise로 입력 평균에러 `t_in(1)` 추정 -> 1 iteration 후 출력에러 `t_out(1)`을 `b` 조정으로 최소화 -> `t_in(2)=t_out(1)`로 다음 iteration 반복(최대 iter <10).
8. 최적화: greedy 국소최적화로 전역 최적 임계 스케줄에 근사(전역 달성 증명은 없음, 향후 과제).
9. 결과: `(1524,762)` 코드 5 iteration에서 동적 multi-threshold가 single-threshold 대비 BER/FER 감소(구체 dB 이득은 그림에만, 본문 수치 미상).
10. 한계: 시뮬만, HW 미설계, hard-decision, greedy 경험적 임계(전역최적 보장 없음), 위치별 임계 변화는 미탐구.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flip 디코딩 루프 | decoder.cpp LDPC_Decoder() | 낮음 - Prime ECC는 min-sum soft이고 논문은 hard bit-flip이라 계열 상이 |
| iteration별 동적 반전임계 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 낮음 - Prime ECC는 이미 iteration별 적응형 LLR threshold 테이블 보유(중복 기여) |
| block-permutation QC-LDPC 구조 | ecc_top.cpp Load_PCM() | 낮음 - QC 구조 유사하나 rate 0.5로 NAND 고rate와 불일치 |
```

적용 가치: **낮음** - hard-decision bit-flip 동적 임계 아이디어는 Prime ECC가 이미 보유한 soft min-sum + iteration별 적응형 LLR threshold와 중복되어 신규 이식가치가 낮음.
