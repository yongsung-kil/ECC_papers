### B. 알고리즘 요약

1. 시스템: WIMEDIA UWB(OFDM 기반, 480Mbit/s급) 표준의 convolutional code(K=7)를 LDPC로 대체하는 검토, codeword size ~1200bit, rate 3/4 프레이밍 제약.
2. 풀려는 문제: 기존 CC는 고rate(3/4)에서 diversity gain이 낮아 성능 부족, LDPC로 대체 시 표준(WiMax/WiFi) 코드는 area/throughput 제약(latency 2µs, throughput 480Mbit/s)을 만족하기 어려움.
3. 핵심 가정: IEEE 802.15.3a 채널모델(CM1~CM4, LoS/NLoS UWB 다중경로), 두 채널(CM1/CM2)만 480Mbit/s 고속 전송에 적합해 시뮬 대상.
4. 핵심 기법 1단: Min-Sum + message scaling factor(MSF=0.75) 기반 디코더를 기본으로, Ultra-Sparse(U-S) LDPC 코드를 신규 설계 — `f[2,3]={1/4,3/4}`, `g[11]={1}`, 1200bit/3300edge(밀도 0.92%), 2V-PEG 알고리즘으로 H-matrix 생성.
5. 핵심 식: VN 업데이트 `λk = λch + Σλl` (식 2), CN은 Min-Sum `λk = sign(Πλl)·min|λl|` (식 3) — 표준 근사 min-sum, U-S 코드의 핵심은 dv_max를 3으로 낮춰 edge 수/access conflict를 최소화한 것.
6. 핵심 기법 2단: Layered decoding(TDMP, Mansour/Hocevar) 아키텍처를 U-S 코드에 적용 — CN 부분집합 처리 후 즉시 VN에 반영해 수렴 속도 향상, dv_max=3이라 layered 스케줄의 latency 제약(CN 처리 지연 dc 사이클) 충족 가능(표준 코드는 dv_max 4~6이라 불가능).
7. 부수 트릭: layered 아키텍처는 channel RAM에 9bit a-posteriori 정보를 직접 저장해 sum RAM 2개(two-phase 구조)를 생략, permutation network를 단일 30-port 9bit shifter로 축소(표준 2-phase는 50+port 6bit ×2).
8. 학습/최적화 절차: 2V-PEG(Progressive Edge Growth 변형) 알고리즘으로 degree distribution `f[2,3]` 하 H-matrix 최적화, 이론적으로 density evolution 대비 최대 0.5dB 손실 예상되나 실측 채널에서는 무시 가능.
9. 결과: 65nm@528MHz 합성 결과 U-S 코드는 area 0.207mm²(logic+mem)로 표준 WiMax/WiFi 코드(0.475~0.485mm²) 대비 55%+ 절감, throughput 530Mbit/s·latency 1.70µs 달성, PER 10⁻³ 기준 통신성능은 CC 대비 3~4dB 개선되며 표준 코드들과 거의 동일.
10. 한계: 시뮬은 PER 10⁻³까지만 검증(더 낮은 PER에서 U-S 코드의 error-floor가 더 높을 것으로 예비 관측·이론적으로 시사됨), UWB 표준 프레이밍(1200bit, R=3/4 고정)에 특화된 코드설계라 범용성 낮음, 다른 rate(1/3,1/2,5/8)는 future work.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 저밀도(dv_max≤3) QC 유사 LDPC 부호설계 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(base+circulant) 고정 구조라 dv_max=3 초저밀도 코드로 교체는 재검증 부담 큼(이식난이도 높음) |
| Min-Sum + MSF(message scaling factor) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC도 min-sum 기반이라 MSF 적용 여부 참고 가능, 다만 코드상 scaling factor 명시 미확인 상태와 비교 필요 |
| Layered decoding(TDMP) 스케줄 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 옵션과 유사 개념) | Prime ECC는 Dual-Update(even/odd 교대)를 이미 보유, layered scheduling은 추가 수렴속도 이득 가능성 있으나 회로 구조 상이 |
| 단일 shifter 기반 permutation network(area 절감) | GT.cpp(Graph Thinning), decoder.cpp pipeline 라우팅 | 저밀도 코드 특유의 라우팅 단순화로, Prime ECC의 고정 QC 구조에는 직접 적용 어려움 |

> 적용 가치: 낮음 — UWB 표준(1200bit, R=3/4) 특화 code-design이며 Prime ECC는 이미 다른 고정 QC-LDPC 구조를 사용하므로 코드 자체 이식은 어렵고, layered decoding·저밀도 그래프에 의한 area/latency 절감이라는 아이디어만 참고 가치.
