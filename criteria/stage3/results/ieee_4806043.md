### B. 알고리즘 요약

1. 대상은 IEEE 802.11n/16e 무선 LAN/MAN QC-LDPC(`(24,96,1/2)` 등), 확장인자 z 27/54/81(LAN)·최대 96(MAN), rate 1/2~5/6, AWGN, 최대 iteration 30.
2. 문제: rate/length/pattern을 on-the-fly로 바꾸는 flexi-mode 디코더는 메모리·순열망(PN)이 면적·전력 병목.
3. 기본 알고리즘: TDMP(turbo-decoding message passing, 층상복호) — 최신 posterior로 다음 subiteration 수행해 수렴 가속, LUT 없이 채널파라미터 독립.
4. 핵심 기법 1: mTDMP — SISO의 forward-backward 재귀에서 correction term을 min1/min2(`M1`,`M2`) 기반 forward 연산(비교+덧셈+간단보정)으로 단순화, 식 (11).
5. 결과: mTDMP는 원 TDMP 대비 0.1 dB 미만 손실, (6) 대비 복잡도 50%+ 감소, OMS보다 덧셈 1개만 더 쓰고 채널/부호 파라미터 독립.
6. 핵심 기법 2: 구조적 메모리 — base matrix 최적화로 인접 subiteration 간 최대상관 정렬 → PMEM 계층화(레지스터파일+소형레지스터)로 접근 40%+ 감소, EMEM은 read/write 버퍼링으로 접근 최적화.
7. 부수 기법: SISO cell 게이팅(확장인자 z에 따라 동적 활성/비활성), 로그시프터 기반 저복잡도 PPN(programmable permutation network)으로 permutation size·확장인자 지원.
8. 핵심 기법 3: ESS-SPC 조기종료 — satisfied parity check 관찰로 진동/stuck하는 미복호 iteration을 조기종료, 저SNR(1.2 dB 이하)에서 평균 iteration 40%+ 감소.
9. 결과 수치: SMIC 180nm 칩 10.12 mm2, 198k gate, 100 MHz, NTh 4.4 Mbps~1.45 Gbps, mTDMP+양자화+ESS로 [4] 대비 0.3 dB BER 개선, NEE 410 pJ/(bit·iter).
10. 양자화: posterior 6-bit, extrinsic 8-bit(압축형 M1/M2/loc/sign 저장)로 성능손실 0.1 dB 미만.
11. 한계: 부호는 무선 표준 QC-LDPC(rate 0.5~0.83, length ~2304)이며 복호골격이 층상(TDMP)이라 우리 flooding min-sum과 스케줄 상이.
