### B. 알고리즘 요약

1. 시스템: sub-20nm(3X/4Xnm) MLC(2 bit/cell) NAND SSD, 실측 SATA 시스템. 기존 BCH는 3 Vref, 기존 LDPC는 21 Vref 필요(read 1050μs @50μs/cell).
2. 문제: 정확한 VTH 정보를 위해 Vref 수를 늘리면 sequential read cycle 급증 → fast read와 high reliability를 동시 달성 불가.
3. 핵심가정: FG-FG capacitive coupling·data retention·program disturb가 동시 발생하며 VTH shift는 `NW/E`·`TRetention`에 강하게 종속.
4. 핵심기법1(EP-LDPC): 3 Vref만 사용하되 VTH+inter-cell coupling+`NW/E`+`TRetention`을 사전기록 테이블로 참조해 각 데이터 BER를 예측, 예측 BER를 LDPC 디코더에 전달.
5. 핵심식/절차: 프로그램 시 초기 lower page의 "0" 개수 `N"initial`를 유저데이터에 추가 저장, read 후 `N"measured`와의 차로 `BER_Lower=|N"measured-N"initial|/NP` 추정.
6. 핵심기법2(retention/coupling 추정): `BER_Lower`+`NW/E`+TRetention table로 retention time 추정 → EP table로 이웃셀 데이터 의존 target BER 추정.
7. 부수기법(error-recovery): 디코딩 실패 시 error mode 판별 후, program-disturb 우세면 PDRP(CG 20V, 전자 de-trap→VTH↓), retention 우세면 DRRP(FG 전자 주입→VTH↑) 펄스 인가 후 재디코딩.
8. 최적화: 출하 전 test 공정에서 TRetention/NW/E/EP table을 flash에 사전 기록.
9. 결과(실측): SSD lifetime CR=2/3에서 11배, CR=9/10에서 2.8배 연장, read 7배 빠름. PDRP로 program-disturb error 76%, 500회 DRRP로 retention BER 56% 복구.
10. 한계: LDPC 부호구조/디코더 알고리즘 세부·gate/throughput 미공개, 소자레벨 recovery 펄스·특정 공정 종속, 테이블 사전측정 필요.
