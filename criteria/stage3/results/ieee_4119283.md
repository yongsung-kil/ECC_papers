### B. 알고리즘 요약

1. 시스템은 양방향 통신에서 고용량 무잡음 피드백 채널을 가정, 데이터 채널은 잡음이 있는 memoryless 채널(AWGN)이며 대상 코드는 RSC 기반 punctured Turbo 코드(LDPC 아님).
2. 풀려는 문제: 기존 ARQ는 ACK/NAK만 사용하나, 고용량 피드백 채널을 활용해 비신뢰 비트 인덱스를 보내면 더 효율적인 증분 재전송이 가능하다는 것.
3. 핵심 가정: 수신측이 BCJR 디코더의 log-APP ratio `λ_i = log(p(u_i=0|Y)/p(u_i=1|Y))`로 비신뢰 비트 집합 `Z(t)`를 계산해 피드백.
4. 핵심 기법 1단: 1단계에서 RSC 인코더-1로 `X(1)`(시스템릭+펑처드 패리티) 전송, 수신측이 `λ(1)`과 `Z(1)` 계산 후 피드백.
5. 핵심 식: 2단계 인터리빙 함수 `π(i, Z(1), b)`를 정의해, weight-2 입력패턴 최소거리 `b`를 고려해 `Z(1)`에 속한 인덱스 쌍 간 거리가 `b`가 되지 않도록 설계(BCJR 디코더-2의 사전확률 신뢰도 저하 방지).
6. 핵심 기법 2단: 펑처링 규칙도 `Z(1)`에 기반해 비신뢰 비트에 대응하는 패리티 비트를 우선 전송하도록 적응적으로 설계.
7. 부수 트릭: RSC 인코더/BCJR 디코더 자체는 수정하지 않고 인터리버·펑처링 파라미터만 동적으로 바꿔 구현 용이성을 확보.
8. 학습/최적화 절차: 별도 학습 없음, weight-2 입력패턴 분석에 기반한 휴리스틱 설계.
9. 결과: Code A(interleaver 300, K/P=3), Code B(interleaver 400, K/P=4) 시뮬레이션에서 제안 Procedure 1/2가 동일 RSC 고정 Turbo 코드 대비 BER 개선, 적응형 인터리버(Procedure 2)가 랜덤 인터리버(Procedure 1)보다 우수.
10. 한계: Turbo 코드/BCJR 전용이며 LDPC 적용은 결론에서 가능성만 언급(수식·실험 없음), 고용량 피드백 채널 전제가 NAND ECC(단방향 저장) 상황과 맞지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 피드백 기반 적응형 인터리빙/펑처링 (Turbo/BCJR) | 대응 없음 | Turbo 코드 전용 스킴이며 Prime ECC는 QC-LDPC 고정 부호·비피드백 디코더 |
| 비신뢰 비트 인덱스 기반 재전송 | 대응 없음 | NAND ECC는 read-retry 등 별도 메커니즘이며 논문의 피드백 채널 모델과 불일치 |

> 적용 가치: 낮음 — Turbo 코드+피드백 채널 기반 ARQ 스킴으로 LDPC/NAND ECC와 부호 종류·시스템 모델이 근본적으로 달라 이식 근거가 없음.
