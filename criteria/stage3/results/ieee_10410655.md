### B. 알고리즘 요약

1. DTMB-A(중국 2세대 지상파 방송 표준) 저-SNR 클라우드 전송 시나리오를 지원하기 위한 저rate(1/3, 1/4) LDPC 부호 설계, 기존 DTMB-A는 rate 1/2·2/3·5/6, length 15360/61440만 지원.
2. 기존 ATSC 3.0/5G-NR의 Raptor-Like 구조는 rate-compatible이지만 길이가 rate마다 달라지거나(ATSC 3.0) HW 복잡도가 커, 고정 길이+rate-compatible 새 구조가 필요.
3. 채널모델은 AWGN 및 Rayleigh fading(QPSK), BP(layered schedule, max 25 iteration) 표준 복호.
4. 핵심 구조: 여러 rate `R1<...<RT`의 검사행렬을 하나의 fusion matrix에 중첩(nest), 각 rate가 Raptor-Like 구조(`H=[[A,D],[C,E],[0,I]]`)를 유지하며 코드길이 `N`은 고정. QC lifting factor도 `Zj=Z0·2^j` 형태로 중첩(modulo lifting).
5. 기존 설계는 waterfall 영역 asymptotic threshold(DE)만 기준으로 삼았지만, 본 논문은 error-floor에 직결되는 **최소거리 상한**(minimum distance upper bound)을 base matrix 설계의 주 기준으로 채택.
6. Base matrix를 블록 `B̂_L,1...B̂_L,P` 단위로 progressive block extension하며, 각 블록 확장 시 모든 nested rate의 최소거리 상한 곱 `d_p = Π dmin,upp(B_partial,t,p)` 이 임계값을 넘는 후보만 유지(트리 기반 후보 유지).
7. Lifting 설계도 동일하게 tree-structured progressive block extension(이전 연구[15] 확장, edge 대신 block 단위)로 수행하며, 개선된 cycle classification(girth `g`, 1-CN 개수 `η` 기반 weight `w_g,η`) + 최소거리 테스트(ANC 알고리즘) + BLER 평가 3단계로 후보 가지치기.
8. 최적화 절차: MET-DE로 degree distribution 결정 → progressive block extension(최소거리 우선) → tree 기반 lifting(cycle/최소거리/BLER 3단계 pruning).
9. 결과: rate 1/3, 1/4 구성 부호가 ATSC 3.0 Type-A 대비 N≈15k에서 뚜렷한 성능 우위, N≈60k에서도 동등 이상이며 error-floor(undetected error) 미관측; rate 1/2 nested 부분행렬도 기존 DTMB-A rate-1/2 대비 SNR 0.3dB 이내 격차.
10. 한계: HW 미설계(순수 부호설계+BP 시뮬), 저rate(1/4~1/3) 방송 채널(AWGN/Rayleigh) 대상이라 NAND 고rate(~0.9) binary QC-LDPC로 직접 재사용 시 rate·채널·최소거리 요구조건이 판이해 재검증 부담이 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Raptor-Like rate-compatible fusion matrix + QC lifting 구조 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 대상이나 특정 QC-LDPC 고정 구조라 이식 난이도 높음(프로파일 §4) |
| 최소거리 상한 기반 progressive block extension 구성법 | 대응 없음 | 부호설계 단계 오프라인 최적화 도구로 코드 내 대응 모듈 없음(설계 툴체인 영역) |
| 개선된 cycle classification(girth/1-CN) + ANC 최소거리 테스트 | 대응 없음 | error-floor 대책이 부호설계 단계(H-matrix 최적화)에서 이루어져 `decoder.cpp`의 런타임 알고리즘과 무관 |

> 적용 가치: 낮음 — 저rate 방송용 부호설계 기법으로 NAND 고rate QC-LDPC 구조·채널과 응용 영역이 달라 코드 직접 이식보다는 최소거리 최적화 "설계 아이디어"만 참고 가치.
