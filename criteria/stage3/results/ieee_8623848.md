### B. 알고리즘 요약

1. 대상 시스템: 매우 긴 블록·고rate LDPC를 쓰는 스토리지(NAND) 메모리, 테스트 부호는 `dv=3, dc=48, N=8192, M=512, R=0.9375` regular LDPC.
2. 문제: soft-decision 복호기는 정정능력은 좋으나 soft-info 센싱·연산이 무거워 read latency가 길고 메모리 수명을 깎음.
3. 접근: 최근 PPBF(확률적 병렬 bit-flipping) hard-decision 복호기를 스토리지용으로 변형한 NS-PPBF 제안.
4. 핵심기법: 각 VN이 energy `E_n(k)` = (불만족 CN 수) + (`v_n(k) ⊕ y_n`)를 계산, `E_n=1`일 때 Bernoulli 확률 `p_{E_n}`로 flip.
5. 핵심 변형: `E_n(k)=1`이고 `v_n(k)=y_n`인 경우에만 flip하도록 조건 추가 → 정답 codeword 수렴 시 모든 VN이 자동으로 flip 정지.
6. 효과: 이 정지 조건 덕에 global syndrome-check(SC)를 쓰지 않고도 수렴 시 값 유지 → SC 모듈 제거로 임계경로 단축.
7. 확률벡터 `p=(0, 0.0001, 0.05, 0.9, 1.0)`, RG가 VN당 p1/p2/p3 확률의 3 랜덤신호 생성, FD 블록의 MUX5to1이 energy로 flip 결정.
8. HW: Part1(D-FF, y·v 저장), Part2(dc-input XOR CNU), Part3(energy 계산 EC) 3부 구조, 완전병렬, 한 iteration = 1 클록.
9. 결과: `f_max=500MHz`(PPBF 400, GDBF 200), Itmax=300에서 13.65Gbps(PPBF 10.92, MS 5.1), 면적 2.151mm2@90nm(PPBF 2.367, MS 2.825).
10. 한계: hard-decision이라 Raw BER 10^-4에서 MS 대비 0.8 decade 성능 열세, error-floor 정량 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NS-PPBF 복호 루프(energy·flip) | decoder.cpp LDPC_Decoder() | min-sum 이터레이션 루프를 BF 방식으로 대체하는 실험 위치, 다만 알고리즘 계열 상이 |
| VN energy·flip 처리 | decoder.cpp VN_Cal_HD() | hard-decision 경로의 VN 갱신 대응(개념적) |
| SC-free 국소 정지 | partial_CRC.cpp / full_CRC.cpp | Prime ECC는 CRC 조기종료 보유 - NS-PPBF는 SC 자체를 제거하므로 서로 다른 정지 철학 |
| 확률적 flip 랜덤신호 | random.cpp | XOR25 난수기와 대응(Bernoulli RG) |
| 완전병렬 HW·CNU XOR | GT.cpp Make_GT_HW() | Prime ECC의 GT/z=32 부분병렬과 구조 상이(NS-PPBF는 full-parallel) |
```

적용 가치: **낮음** - hard-decision BF 계열은 Prime ECC의 min-sum + CRC 조기종료 파이프라인과 알고리즘 축이 달라 직접 이식은 어렵고, min-sum 대비 정정능력 열세로 대체 유인 낮음(단, 초저지연 fast-read 옵션 아이디어로만 참고).
