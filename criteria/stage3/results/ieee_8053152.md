### B. 알고리즘 요약

1. 타깃 시스템: SSD 컨트롤러(ARM+PCIe VC709 FPGA), NAND Flash용 ECC. rate-0.94, length-35840 (35840,32768) QC-LDPC.
2. 문제: SSD storage density 상승으로 RBER 증가 → 고throughput·고정정능력 LDPC 디코더 필요, 기존 min-sum 근사는 성능 열화.
3. 채널/모델: NAND read LLR `I_n` 입력, AWGN 기반 FER 평가(SNR 6.08dB 지점), RBER `3×10^-3` 요구.
4. 핵심기법1(layered NMS): 계층적(hierarchical) 메시지 스케줄로 각 행 결과를 즉시 VN 갱신에 반영 → 수렴속도·throughput 향상. `L_nm=Y_n-C_mn`, `C_mn=ρ·∏sgn·min|L_km|`, `Y_n=L_nm+C_mn`.
5. 핵심식 의미: 정규화 계수 `ρ`가 min-sum 과대추정을 보상 (식4), ρ가 클수록 LLR 신뢰↑ 가정.
6. 핵심기법2(적응형 ρ): iteration마다 error bit 수 `ES_j` 카운트 → `ES_j` 크면(LLR 비신뢰) ρ 작게, 3구간 적응(선행연구 [8]의 2구간보다 세분).
7. 구현 디테일: `L_nm` 6-bit signed, `C_mn` 4-bit signed 양자화. 512×512 sub-matrix 단위 cyclic shifter로 정렬.
8. 최적화: Matlab에서 각 ρ 구간별 데이터 수집해 FER/iteration 기반 ρ 테이블 결정(NN/DE 아님).
9. 결과: 적응형 ρ로 avg iteration 약 26% 감소, throughput 약 35%↑, SNR 6.08dB. 최종 throughput 1.97Gb/s @100MHz, 5 iter. LUT 98561/FF 101809/BRAM 210/DSP 0.
10. 한계: gatecount(ASIC) 없음(FPGA 자원만), soft-read bit수·정확 채널 모델 불명, ρ 구간 튜닝이 부호 종속.
