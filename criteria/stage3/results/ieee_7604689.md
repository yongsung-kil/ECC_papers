### B. 알고리즘 요약

1. 시스템: CCSDS 우주통신 데이터링크 계층용 QC-LDPC 인코더. 두 부호군 - deep-space AR4JA(9종, 블록길이 1024/4096/16384, rate 1/2·2/3·4/5)와 near-earth C2(circulant 511bit).
2. 문제: 기존 인코더는 DVB-S2/IEEE 802.11 등 타 표준 최적화라 CCSDS 부호(RU법에서 `g=m`으로 dense φ 큼)에 비효율. RU법 이점 없음.
3. 접근: `G = [I_k | W]` systematic-circulant 생성행렬의 직접 곱셈법(`c = sG`)을 채택하고 QC 구조의 병렬성을 활용.
4. 핵심기법: recursive convolutional encoder(RCE)를 병렬화. function generator `f_i`가 circulant 첫 행 1비트를 표현, `Lm` 비트를 한 클록에 병렬 출력.
5. 두 종류 병렬성: `Lm`(연속 비트 병렬) × `La`(다중 circulant 병렬). 총 병렬도 `Lm×La`를 컴퓨터 버스폭에 맞춰 2의 거듭제곱으로 설정.
6. La 증가는 function generator 단순화(자원 절감)하나 입력 버퍼(double buffer, `2·m·La`bit)와 latency `(La-1)(m/Lm)/La` 증가 유발.
7. C2 특례: circulant 크기 511(2의 거듭제곱 아님)로 경계 불일치 문제. 매 511bit마다 zero 삽입 대신 circulant 마지막 행 선택으로 RCE 복잡도 증가 없이 처리, La=1 고정.
8. 스트리밍: AMBA AXI-Stream으로 입력 프레임 전체 버퍼 없이 파이프라인 인코딩. 채널코딩 외 동기화(ASM)·랜덤화까지 통합.
9. 결과: Virtex XC5VLX/Virtex-7 FPGA 구현, 기존 문헌/시판 IP 대비 throughput 우위(정확 수치는 표 손상). AR4JA는 Lm×La=32 전 조합 구현.
10. 한계: 순수 인코더로 복호·BER·정정능력 분석 전무. NAND ECC 무관, 우주통신 프로토콜(프레이밍/랜덤화) 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC systematic-circulant 인코딩 | [encoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 인코딩 고정, 본 논문은 direct G-multiplication - 방식 상이 |
| Lm×La 병렬 RCE / function generator | 대응 없음 | HW 인코더 병렬화 개념이나 Prime ECC 구조(z=32, dual-diag)와 정합 안 됨 |
| CCSDS 프레이밍/랜덤화/동기화 | 대응 없음 | NAND ECC와 무관한 통신 프로토콜 계층 |

적용 가치: **낮음** — 순수 CCSDS 우주통신 QC-LDPC 인코더 FPGA 아키텍처로, Prime ECC의 복호기·부호설계·NAND soft-read 어느 것과도 접점이 없음.
