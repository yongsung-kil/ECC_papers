### B. 알고리즘 요약

1. 대상 시스템: 데이터센터 분산스토리지, block을 stripe로 코딩; 노드 실패를 BEC 상의 variable node erasure로 모델링, 비교대상은 RS·LRC·piggybacked-RS.
2. 문제: MDS(RS)는 복구 시 k blocks 다운로드로 repair-bandwidth 오버헤드가 큼(예 (14,10)RS는 10x); 낮은 redundancy와 낮은 repair-BW 동시 달성 필요.
3. 핵심 관찰: LDPC의 VN은 소수 CN에만 연결 → LRC 유사 local repair, repair-BW가 code length에 무관.
4. repair-BW 공식화: `RBW_LDPC = Σ dc,i(dc,i-1) / E` (식 1), CN degree 분포로 결정.
5. Lemma 1: 고정 edge 수 E에서 Cauchy-Schwarz로 CN-regular가 repair-BW를 최소화(`RBWmin = dc-1 = dv/(1-R) - 1`), dv=2로 고정.
6. 신뢰도 모델: 연속시간 Markov 모델로 MTTDL 평가, 조건부 생존확률 `pi = qi+1/qi`를 erasure-channel 복호 시뮬로 추정.
7. MTTDL 일반식(Lemma 2): m parity에 대해 `MTTDL ∝ μ^m / f(...)`, pi를 키우면 MTTDL 증가; p0=1이 가장 지배적.
8. Proposition 1: stopping number s가 클수록 더 많은 pi가 1이 됨 → dv=2일 때 stopping number = girth/2이므로 girth를 키우는 PEG 설계가 신뢰도 향상.
9. 결과: rate 1/2·2/3·3/4에서 QC-PEG LDPC(dv=2)가 (14,10)/(15,10) RS 대비 repair-BW 25~50% 감소하면서 더 높은 MTTDL, length 증가로 repair-BW 불변 상태로 신뢰도만 향상.
10. 한계: HW 미설계, BER/soft-decoding 아닌 BEC/노드 erasure·MTTDL 도메인, VN-irregular·non-binary는 future work.
