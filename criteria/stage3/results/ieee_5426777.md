### B. 알고리즘 요약

1. 시스템: 임의 LDPC(regular/irregular, cyclic/non-cyclic) 범용 디코더의 상호연결망을 다룸. rate/length 미지정, toy 예제는 6 VN × 4 CN(col weight 2, row weight 3).
2. 문제: LDPC 디코더 인터커넥트는 코드 종속·충돌 발생 → 코드마다 tailor-made. 범용·충돌 없는 재구성 가능 아키텍처가 목표.
3. 핵심 도구: Perfect Difference Network(PDN) — perfect difference set 기반, 노드 수 `N=δ²+δ+1`, 각 노드 차수 `2δ`, 직경 2(임의 노드 쌍 2-hop). all-to-all personalized 통신이 `2δ` 스텝에 수행.
4. 알고리즘: 표준 SPA(belief propagation). CN 갱신 `φ(x)=-log tanh(x/2)` + sign(식 3), VN/codeword test(식 4-6). 알고리즘 자체는 변형 없음.
5. 매핑: PDN 프로세서 `μ(k)`에 VN(`μn(j)→vj`) 및 CN(`μm(i)→ci`)을 one-to-one(또는 다중) 매핑. 각 프로세서는 자기 이웃주소 집합 `Aj`,`Bi`만 보유(전역 그래프 불필요).
6. 통신: iteration마다 `μn→μm`(Step1) personalized 패킷 전송 → CN 계산(Step2) → `μm→μn`(Step3) → codeword test & VN 갱신(Step4). PDN이 충돌 없이 라우팅.
7. 구현: shuffle network를 PDN으로 대체, memory bank + scheduler(round-robin 또는 code-specific heuristic)로 재구성. PDN은 전용 HW 상정.
8. 확장: Tanner graph는 factor graph의 특수 경우 → 분산 추론/센서망 처리에도 적용 가능하다고 주장.
9. 결과: 정량 결과 없음. 6×4 toy 예제의 메시지 패킷 스케줄 표(Table I/II)만 제시. BER/gate/throughput 수치 전무.
10. 한계: RTL·합성·시뮬 결과 없음(개념/이론). N=δ²+δ+1 노드 수 제약, 알고리즘은 표준 SPA, NAND/soft-read 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PDN 기반 VN/CN 인터커넥트·shuffle 대체 | [decoder.cpp](../Prime_ECC_3.1_Claude) HW 모델(z=32 row 병렬 라우팅) | 우리 QC-LDPC는 circulant shift로 라우팅 확정 — 범용 PDN 이점 희석, 정합성 낮음 |
| 메모리 뱅크·conflict-free 접근 | `decoder.cpp` HW 메모리 모델(GT edge 선택 포함) | 충돌 회피 목표는 공통이나 우리는 GT/HCU로 이미 대응 |
| 표준 SPA CN 갱신 `φ`=-log tanh | `decoder.cpp` `CNU_Update_New_Mag()` | 우리는 min-sum 근사 사용 — full-tanh SPA는 대응 없음(면적↑) |

적용 가치: **낮음** — 임의 LDPC용 범용 인터커넥트 토폴로지 제안이나, Prime ECC는 특정 QC-LDPC에 최적화된 circulant 라우팅을 이미 보유하고 알고리즘도 표준 SPA(우리는 min-sum)라 이식 이득이 없고 HW 검증도 부재.
