### B. 알고리즘 요약

1. 대상은 protograph 기반 SC-LDPC(QC/PEG): 대표 코드 ensemble A(`(3,6)`), `L=16`, `M=512`, `n=16384`, rate `R=0.4375`, girth≥10. ensemble B/C(rate 0.46875)도 검증.
2. 풀려는 문제: SC-LDPC windowed decoder(WD)의 약점 - 작은 윈도우 `W`에서 성능 저하, 오류 전파, error-floor, 그리고 윈도우가 PCM 경계를 벗어날 때의 비효율.
3. 모델: BI-AWGN, RCA(reciprocal channel approximation) 기반 density evolution으로 각 윈도우 위치 `p`의 noise threshold `σ*_p` 계산.
4. 핵심 기법 1(조기종료): `p=L-W+1`에서 윈도우 슬라이딩을 멈추고 윈도우 내 전 심볼을 target으로 처리, 수직 차원을 `(W+w)Jg·M`으로 확장(Alg.1). 종단측 저차수 CN 이점 조기 확보.
5. 핵심 식: 프레임 latency `τ^F_iWD = TF + Σ(W/L)TD` (식9) - 슬라이딩 중단으로 conventional WD 대비 latency 감소.
6. 핵심 기법 2(메시지 재사용): 이전 target 심볼의 output LLR 대신 extrinsic LLR(Alg.2 line7)을 엣지 메모리에 유지해 다음 윈도우로 전달 - 오류 전파·메시지 독립성 위반 완화.
7. 핵심 기법 3(증폭): 이전 윈도우 parity 만족 시 재사용 엣지 메시지를 `α`배 증폭(Alg.4, iWD-M) - 우측 저차수 VN의 error-floor 기여 억제.
8. 최적화 절차: RCA-DE로 threshold 사전결정(δ=15), `α*≈2.1` 경험적 최적화(성능/복잡도 트레이드오프는 `α≤α*`에서 양호).
9. 결과: W=6, BER 1e-3에서 WD 대비 약 0.17dB 이득; W=6, 1.8dB에서 VN 처리 42.5% 감소; α 적용 시 ensemble C error-floor 대폭 하락. CN 처리는 full-tanh SPA(Alg.2 line6).
10. 한계: 순수 Monte Carlo 시뮬(HW 미설계), gate/throughput 없음, SC-LDPC 슬라이딩 윈도우 전용 구조.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC windowed decoding / 윈도우 슬라이딩 | 대응 없음 | Prime ECC는 block QC-LDPC 전프레임 디코딩(`decoder.cpp` `LDPC_Decoder()`)이라 슬라이딩 윈도우 개념 부재 |
| 엣지 메시지 재사용/증폭 초기화 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 반복 루프 (개념 유사만) | 반복간 메시지 유지 아이디어는 유사하나 윈도우 경계 종속이라 직접 이식 불가 |
| CN 처리(full-tanh SPA) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` (min-sum) | Prime ECC는 min-sum 근사 사용 - full-tanh SPA와 연산 계보 다름 |
| RCA-DE threshold 분석 | 대응 없음 | 이론 분석 도구로 시뮬레이터 코드에 대응 모듈 없음 |

적용 가치: **낮음** - SC-LDPC 슬라이딩 윈도우 디코더 전용 기법이라 block QC-LDPC 전프레임 min-sum 구조의 Prime ECC에는 구조적 이식 지점이 없다.
