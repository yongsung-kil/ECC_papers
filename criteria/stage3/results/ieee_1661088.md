> 총평: NAND와 무관한 phase-noise 채널의 위상 추정(particle filter) 논문으로 LDPC는 부수적으로만 등장하여 이식 가치 없음.

### B. 알고리즘 요약

1. 채널모델: `r_k = c_k e^{jθ_k} + n_k`, θ_k는 random walk 위상잡음(`θ_k=(θ_{k-1}+Δ_k) mod 2π`), M-PSK 변조에 (3,6)-regular LDPC(length 4000) 부호화.
2. 문제: phase와 data를 동시추정은 intractable, EM은 프레임 내 위상이 일정해야만 효율적 → 강한 위상잡음(σ∆=6)에서 반복 추정 필요.
3. 채널코더(LDPC/turbo)와 위상 추정기 간 message-passing을 factor graph(그림1)로 모델링, forward-backward 노드 활성화 스케줄 사용.
4. 핵심 기법: 위상공간을 particle filter로 이산화(`p_f(θ_k) ≈ Σ w̃_k^(i) δ(θ_k-θ_k^(i))`)해 phase discretization(트렐리스 L개 위상, 복잡도 O(L²)) 대비 복잡도를 O(N_part)로 낮춤.
5. 세 가지 추정 변형: MMSE phase estimation, forward-only phase pdf(FOPP), forward-backward phase pdf(FBPP, `p_u(θ_k)≈½(p_f+p_b)`로 계산량 O(K·N_part)로 단순화).
6. resampling 단계로 particle degeneracy(유효 particle 수 N_eff < 임계값) 완화.
7. 시뮬 설정: QPSK, pilot 20심볼당 1개, global iteration 10회 × LDPC 내부 iteration 20회.
8. 결과: N_part=64 FBPP가 L=20 discretization과 유사 성능, L=32(사실상 최대) 대비 BER=1e-3에서 약 0.4dB 열화이나 branch metric 계산은 64 vs 400으로 대폭 감소.
9. N_part=16과 256 FBPP 간 성능차 0.25dB에 불과해 저복잡도로도 준수한 성능.
10. 한계: HW 미설계, 순수 시뮬 결과뿐이며 무선 phase-noisy AWGN 채널 특화, LDPC 부호 자체는 표준 규칙 부호로 개선 대상이 아님.
