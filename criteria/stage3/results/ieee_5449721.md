### B. 알고리즘 요약 (10줄 내외)

1. 대상: 802.16e/802.11n/DVB-S2 계열 LDPC 코드, 평가된 6개 코드는 length 324~2640, row weight 6~12(rate 0.5~0.9대 추정).
2. 문제: 기존 P-TDMP(Parallel-TDMP)는 posterior memory 컨텐션 때문에 한 번에 `S`(AA-LDPC 서브행렬 크기)개 행만 병렬 디코딩 가능해 처리량이 제한됨.
3. 가정: 코드 구조(H-matrix) 수정 없이, TDMP의 posterior 메시지 write 접근만 스케줄링으로 제어.
4. 핵심 기법: TDMP 행별 갱신을 이분그래프(bipartite graph, check node/variable node = vertex, time slot = edge)의 **edge colouring 문제**로 변환.
5. 핵심 식: `λi(t+1) = φ⁻¹(Σφ(ρ(t)))`, `ρ(t) = γ(t) - λi(t)`; posterior 갱신 `γ(t+1) = ρ(t) + λi(t+1)` — 최신 계산값을 즉시 반영해 수렴 가속(TDMP 고유 특성).
6. 제안 스케줄링(Algorithm 1): 무작위 노드부터 시작해 미착색 edge에 순차적으로 유일 색상을 배정하는 근사 최적 edge-colouring, König 정리상 최적 색상 수 = 최대 노드 차수 `d`.
7. 부수 트릭: 최적 edge-colouring(NP에 가까운 탐색)은 시간이 오래 걸려(4시간+ 실패) 실용성이 없어, 제안 알고리즘은 최적 대비 색상 수 +2 이내로 근사하면서 수 분 내 처리.
8. throughput 모델: `θ = bfL·PR/(cT·M)` (b=메시지 비트, f=클록, L=length, c=row weight, M=총 행 수, PR=병렬 처리 행 수, T=행당 클록 사이클).
9. 결과: 제안 알고리즘의 edge colouring이 그리디(Matgraph) 대비 색상 수 25~50% 적고, P-TDMP 대비 처리량 약 7~15배 향상(6개 코드에서 6.9~15.0배).
10. 한계: 순수 스케줄링/이론 분석뿐이며 실제 HW 합성·면적·전력 결과 없음, 코드 구조는 AA-LDPC(순열 항등행렬) 가정에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP(레이어드) 디코딩, 최신값 즉시 반영 갱신 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC도 layered/Dual-Update 스케줄 사용, 개념적으로 대응되는 반복 구조 |
| Dual-Update(parity even/odd 교대) 스케줄 | `decoder.cpp` (Dual-Update 옵션) | 컨텐션 회피 목적이 유사하나 본 논문은 그래프 컬러링 기반 일반화된 스케줄러로 이식 시 추가 검토 필요 |
| posterior memory 컨텐션 회피(edge colouring 스케줄) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 (스케줄링 로직) | 병렬 처리 행 수를 늘리는 스케줄 최적화로 디코더 루프 재설계에 참고 가능(이식 난이도 낮음~중, 프로파일 §4 기준) |
| AA-LDPC(순열 항등행렬 기반) 부호 구조 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 바꾸는 요소는 아니며 컨텐션-프리 특성을 활용하는 전제조건, 부호설계 변경은 아님 |

> 적용 가치: 중간 — 코드 구조 변경 없이 순수 스케줄링으로 병렬도/처리량을 높이는 기법이라 decoder.cpp 이터레이션 루프의 스케줄 재설계에 참고할 수 있으나, HW 구현·정량적 면적/전력 검증이 없어 직접 이식 전 재검증 필요.
