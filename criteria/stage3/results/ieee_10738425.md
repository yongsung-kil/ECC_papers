### B. 알고리즘 요약

1. 채널은 BPSK+AWGN, LLR `l_v = 2y_v/σ²`. BP 디코딩은 VN→CN 메시지 `u^t_{v,c}`, CN→VN 메시지 `u^t_{c,v}`(식(2)(3), tanh/tanh⁻¹ 기반) 반복.
2. NBP는 CN 갱신식(3)에 학습 가중치 `α^{(t,v-1)}_c`를 곱해(식(6)) 성능을 높이지만, 가중치·메시지 저장량과 tanh 계산량이 급증하는 문제.
3. CENBP(선행연구[9])는 cyclic code의 shift-invariant 구조를 이용해 가중치를 홀수/짝수 레이어로 공유, 메모리는 줄이지만 곱셈·덧셈 수는 오히려 NBP보다 증가.
4. 핵심 기법: 학습 가중치 텐서 `α`와 메시지 텐서 `u`를 4차 텐서로 reshape 후 Tensor Ring(TR) 분해 `A = A1·A2·A3`(식(13)(14))로 저장/연산량을 지수(`O(n^D)`)에서 선형(`O(DnR²)`)으로 축소.
5. TR 분해된 저rank 인수텐서가 원 텐서를 근사 대체(식(15)(16))하며, tanh/tanh⁻¹/cascade multiplication 각각에도 TR 압축 적용 가능(Type I~III, 식(18)~(24)).
6. CENBP 확장: 홀수 레이어의 학습 가중치는 shift-invariant 구조 보존을 위해 그대로 두고, 메시지 `L_j`, `x^{[s-1]}`만 TR 분해(식(27)~(29))해 시간복잡도·메모리 동시 절감 (TR-CENBP).
7. 부수 기법: piece-wise 함수로 tanh 근사 시 고차항(`x²` 등)을 추가로 TR 분해해 HW 구현 복잡도 추가 절감(식(30)~(34)).
8. 학습 절차: cross-entropy loss + Adam optimizer, learning rate 0.001, minibatch 120, iteration T=5(12-layer 신경망 등가), TensorFlow 2.0 시뮬.
9. 결과: BCH/LDPC(800,400) 등에서 학습 가중치 100~2000배, 메시지 수 최대 2000배, 곱셈 연산 50~600배 감소하면서 BER/FER 성능 저하 거의 없음. 단, TR 압축을 학습 가중치 없이 순수 BP에 직접 적용하면(TR-BP) 성능이 크게 저하.
10. 한계: 전부 시뮬레이션 결과(Tensorflow)뿐이며 HW 구현/합성 없음. non-binary 신경망 프레임워크로 이진 min-sum HW와 직접 비교 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NBP/CENBP 신경망 디코더 전체 | 대응 없음 | Prime ECC는 non-NN min-sum 기반, 학습 가중치 개념 없음 |
| tanh/tanh⁻¹ 기반 CN 갱신 | 대응 없음 (Prime ECC는 min-sum으로 tanh 대체) | Prime ECC가 이미 min-sum 근사로 tanh 회피, 압축 대상 자체가 없음 |
| TR 분해 기반 메모리 압축 아이디어 | 대응 없음 | LLR 테이블(`ecc_data.h` `PARAM_LLR`)과 개념적 유사성은 있으나 논문 기법은 신경망 가중치 전용 |

> 적용 가치: 낮음 - non-binary/신경망(tanh, 학습 가중치) 기반 NBP·CENBP 압축 기법으로 Prime ECC의 min-sum 이진 HW 구조와 근본적으로 이질적이며, 순수 시뮬레이션 결과뿐 HW 검증 없음.
