### B. 알고리즘 요약

1. WiMax(IEEE 802.16e) QC-LDPC, 6가지 code rate(1/2~5/6), sub-matrix 크기 24×24~96×96, code length 576~2304 bit.
2. 전통적 OMS는 모든 VN을 갱신한 뒤에야 CN 갱신을 시작해 수렴이 느림 — 이를 개선하려는 문제.
3. AWGN 채널, BPSK 변조를 가정.
4. Log-BP → Min-Sum → Normalized MS(계수 `α`) → Offset MS(계수 `β`) 순으로 근사 단순화, `L(hmn) = sgn(·)·max{min|L(gmn')|-β, 0}`.
5. `β`는 code rate별로 사전 결정된 상수(예: rate 1/2에서 `β=0.15`)이며 Density Evolution/Gaussian approximation에서 유도.
6. 본 논문의 핵심 확장: layered(부분병렬) 스케줄링 — 12개 sub-matrix 단위로 CN 갱신을 12-way parallel, VN 갱신을 24-way parallel로 분할해 서브-이터레이션 단위로 메시지를 즉시 갱신.
7. 매 서브-이터레이션마다 한 row/column만 새 메시지로 갱신된 행렬을 다음 단계로 즉시 전달(freshest-message 원리).
8. 별도 학습/최적화 절차 없음 (β는 기존 문헌값 사용).
9. 결과: 2304-bit 코드에서 BER=10⁻³ 기준 layered OMS가 traditional OMS 대비 SNR 약 0.05dB 개선. NMS는 MS 대비 BER=10⁻² 기준 0.4dB 개선.
10. 한계: HW 구현/합성 없음, 순수 시뮬레이션 결과만 제시. Gate count/throughput/latency 수치 없음.
