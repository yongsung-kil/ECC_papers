### B. 알고리즘 요약 (10줄 내외)

1. Inner 코드는 SC-LDPC(협 LDPC-(588,518), coupling width `w=1`, lifting `Z=14`, girth 6), outer 코드는 BCH-(511,493,2) product code, 전체 rate `R=82%`(inner 88%×outer 93%).
2. 5G-NR LDPC는 최대 블록길이(26112)에서 메모리 오버헤드가 커 디코더 면적/라우팅이 병목이 되고 throughput이 ~20Gbps로 제한되는데, SC-LDPC의 windowed BP로 이를 해결하면서도 error-floor 문제(trapping set)를 product code로 보완하려는 것이 목적.
3. AWGN 채널 가정, SC-LDPC coupling length `L=n`으로 설정해 2D codeword 구조(product code와 동형)를 자연스럽게 형성.
4. 핵심 기법 1단: SC-LDPC 프로토그래프를 `[B0,...,Bw]`로 edge-spreading해 대각 계단형 base matrix `B_SC`(식 1) 구성, code rate `R_SC = R - w·mb/(L·nb)`(식 2).
5. 핵심 식: 인코딩 시 `pt = H0'·cΨt,: + Σ Hi·ct-i`(식 6)로 이전 블록들과의 결합을 통해 parity 생성 — convolutional 구조가 MAP threshold 달성의 근거.
6. 핵심 기법 2단: Phase-I windowed BP(window `W`)로 SC-LDPC를 sliding window 디코딩해 BER을 10^-3 수준까지 낮춘 뒤, Phase-II iBDD(iterative bounded distance decoding, 식 4)로 row/column 반복 디코딩해 잔여 오류를 error-floor까지 제거(Algorithm 1), 두 단계 사이 피드백 없이 파이프라인 가능.
7. 부수 트릭: SC-LDPC 서브블록들이 동일 프로토타입이라 유사 오류패턴이 정사각형으로 뭉치는 문제를, diagonal interleaver `Ψ`(식 5, Fig.6)로 흩어 stopping set 발생을 억제.
8. 학습/최적화 절차는 없음(수동 프로토그래프 설계, 4-cycle 없는 girth-6 `B_SC` 직접 구성).
9. 결과: BER 10^-8에서 기존 product code(Chase-Pyndiah p=6) 대비 이득 확보, OSD-SDD(최적 근사) 대비 0.13dB 차이(W=20); error-floor는 stopping-set 확률식(식 8)으로 10^-15 이하로 semi-analytical 예측.
10. 한계: HW 미설계, 시뮬레이션(및 semi-analytical 추정)뿐이고 최대 반복횟수(iBDD)의 실측 미제공, product code 추가로 시스템 복잡도 상승(codebase 공개: github.com/yuqren/pc-sc-ldpc).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 프로토그래프 base matrix 설계(식 1,2) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base+circulant) 구조라 SC-LDPC의 convolutional coupling은 부호구조 차원에서 다름(높은 이식 난이도) |
| Windowed BP 디코딩(Algorithm 1 Phase-I) | `decoder.cpp` `LDPC_Decoder()` | 슬라이딩 윈도우 기반 스트리밍 디코딩 개념은 Prime ECC의 블록단위 반복 루프와 구조가 달라 이식 난이도 중간 |
| Diagonal interleaver를 통한 stopping-set/error-pattern 분산 | 대응 없음 | Trapping-set/error-floor 대응 아이디어라는 점에서 관심 대상이나, 코드 표에 대응 모듈 없음(개념적으로 error-floor 대책이므로 향후 검토 가치) |
| Error-floor의 stopping-set 기반 semi-analytical 예측(식 8) | 대응 없음 | 정정능력/error-floor 분석 도구이나 코드 내 대응 모듈 없음 |
| iBDD outer 디코딩(BCH product code) | 대응 없음 | Prime ECC는 순수 LDPC 단일 코드 구조로 product code concatenation 자체가 대상 밖 |

> 적용 가치: 중간 — SC-LDPC+product code 결합 자체는 이식 난이도가 높지만, error-floor를 trapping-set/stopping-set 관점에서 억제하는 인터리빙 아이디어와 semi-analytical 예측 방법론은 Prime ECC의 error-floor 개선 검토 시 참고 가치가 있음.
