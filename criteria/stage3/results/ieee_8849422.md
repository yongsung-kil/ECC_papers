### B. 알고리즘 요약

1. 대상은 binary QC-LDPC: 패리티행렬이 `m×m` circulant permutation matrix(CPM) 블록으로 구성, `(I,J,m)` 행렬 구조.
2. 문제: regular QC-LDPC는 (1) randomness 부족(각 CPM이 `Zm` 원소 하나로 결정)으로 waterfall 열화, (2) girth 이론 상한 12로 error-floor 열화.
3. 핵심 기법: 각 permutation matrix를 CPM 대신 linear permutation polynomial `R(x)=ax+b` (`a`,`b∈Zm`)로 정의한 linear permutation matrix로 대체 → LPP code.
4. bijection 조건: `R(x)=ax+b`가 `Zm`을 permute하려면 `a`와 `m`이 coprime(Thm.4), 역함수 `R^-1(x)=a^-1 x - a^-1 b`(Thm.2).
5. cycle 조건: 2L edge가 cycle을 이루지 않을 필요충분조건을 `(A,B)`-edge 형태 `Ax0-B≠0`으로 유도(Thm.3), 최종적으로 `gcd(A,m)≠1` 그리고 `B≠0 (mod gcd(A,m))`(Thm.5)으로 x0 없이 판정.
6. girth 하한: `m`이 prime이면 CPM 아닌 행렬은 girth-4 cycle 유발(Thm.6) → `m` non-prime 필요; girth>4는 `m≥J`, girth>6은 `m>(I-1)(J-1)`.
7. 구성: 주어진 `N,I,J,g`에 대해 `m=N/J`로 두고 `a_{i,j}`(coprime), `b_{i,j}`를 랜덤 선택 후 Thm.5 조건 검사, 실패 시 재추첨.
8. 결과(girth): CPM QC-LDPC는 girth≤12지만 LPP는 이를 돌파 — (3,3,114)로 girth>12, girth와 `m` 증가 관계 표로 제시(g=22일 때 m=32768).
9. 결과(성능): (3,4,906) N=3624 rate~0.25 LPP는 waterfall에서 random 부호 능가·WER 1e-6까지 error-floor 없음; (3,6,762) N=4572 rate~0.5는 random/QC와 대등하며 WER 1e-5까지 floor 없음.
10. 한계: HW 미설계(시뮬만), AWGN/BPSK 가정, 고girth 요구 시 `m` 폭증, LPP가 CPM이 아니므로 QC 특유의 shift-register 인코딩·순환 라우팅 이점 유지 여부 불명.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LPP 기반 H-matrix 부호구성 (ax+b permutation matrix) | ecc_top.cpp Load_PCM() | Prime ECC의 base+shift QC-LDPC(PCM_b) H-matrix를 LPP로 재설계하는 지점, error-floor/waterfall 개선 후보 |
| QC 인코딩 (shift-register 선형시간) | encoder.cpp Generate_PCM_encoding() | CPM 전제의 dual-diagonal QC 인코딩이 LPP 도입 시 유지 가능한지 검증 필요 |
| girth/cycle 개선 → 복호성능 | decoder.cpp LDPC_Decoder() | 부호 자체 개선이라 디코더 min-sum 루프는 그대로 재사용 가능 |
```

적용 가치: **중간** — Prime ECC가 쓰는 binary QC-LDPC의 girth-12 한계를 부호설계로 돌파해 waterfall+error-floor를 동시에 개선하는 점은 매력적이나, H-matrix 재설계(`Load_PCM()`)와 CPM 전제 QC 인코딩/라우팅 재검증 부담이 커 이식 난이도는 프로파일 §4의 "부호설계=높음" 범주에 걸린다.
