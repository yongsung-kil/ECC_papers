### B. 알고리즘 요약

1. 대상은 column-weight 2인 quasi-cyclic LDPC 코드로, rate는 `1-2/k`(k=row weight)로 결정되며 구체적 length는 실험 표(예: 105×210, 3120×6240 등)로 다양하게 제시.
2. 풀려는 문제: 기존 bit-filling 알고리즘([9])은 girth/rate 최적화는 되지만 row-column 연결구조가 불규칙해 HW 구현이 어려움 → quasi-cyclic 구조로 개선하려는 문제.
3. 채널 모델은 AWGN + BPSK를 사용해 성능 시뮬레이션.
4. 핵심 기법은 modified bit-filling: 코드의 row를 2개 이상의 동일 크기 그룹으로 나누고, 각 컬럼(column-weight=2)이 서로 다른 그룹의 두 row를 연결하되 최소 거리(distance) 조건을 만족하는 row를 순차 탐색해 연결.
5. 핵심 규칙: 한 그룹의 첫 row가 연결되면 나머지 row들은 동일한 shift로 상대적으로 연결되어(quasi-cyclic 구조, 식 없음, 절차적 규칙), girth 위반 시 재탐색.
6. 그룹 수를 2개로 하면 최대 girth 12(bipartite graph 특성상 짝수 girth만 가능), 그룹 수를 3개 이상으로 늘리면 girth 14, 16, 18, 20, 24까지 확장 가능.
7. 구현 디테일: 알고리즘 복잡도는 `O(M^2)`(M=row 수), row 그룹을 HW의 processing node에 매핑하는 구조 제안(그룹 간 interconnection network, Fig.5).
8. 별도 학습/최적화 절차 없음(결정론적 순차 탐색 알고리즘).
9. 결과: girth가 높을수록 BER 성능이 개선됨을 확인(예: girth 20 코드가 girth 12보다 우수), 25 iteration 기준 시뮬레이션.
10. 한계: HW는 개념적 블록 다이어그램(Fig.5)만 제시, 실제 게이트/타이밍 설계 없음. column-weight 2는 code size에 대해 성능이 로그적으로만 개선되는 근본적 한계(본문 인용).

### C. Prime ECC 관련 모듈 핀포인트

| row-group을 processing node에 매핑하는 HW 구상 | 대응 없음 | 개념적 블록도 수준(Fig.5)이며 Prime ECC의 z=32 병렬 구조·HCU와 무관 |

적용 가치: 낮음 — column-weight 2 LDPC는 magnetic recording/partial-response channel 특화 저복잡도 코드로, NAND 고전 binary LDPC(고차수 QC-LDPC)와 근본 구조가 달라 Prime ECC에 이식할 요소가 거의 없음.
