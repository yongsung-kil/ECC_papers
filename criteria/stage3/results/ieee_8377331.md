> 총평: LP(ADMM l2) 복호기의 FPGA 실현성 실증 논문이나, Prime ECC가 이미 쓰는 min-sum 대비 면적·throughput이 크게 열위여서 NAND ECC 이식가치 낮음.

### B. 알고리즘 요약

1. 시스템: 정규 Gallager LDPC(dv=3, dc=6), rate r=(N-M)/N, N=96~9216, AWGN/BPSK 채널, LLR `γ_i=log(P(r|x=0)/P(r|x=1))`.
2. 문제: Feldman의 LP 복호는 BP(SPA/MS) 대비 FER은 좋으나 solver 기반이라 HW 구현이 어려웠음.
3. 핵심 기법: Barman et al.의 ADMM 재정식화 + Liu의 penalized ADMM `l2` (목적식에 `-α||x-0.5||^2` 벌점), check polytope `P_dc`로의 Euclidean projection이 CN 연산의 핵심.
4. 스케줄링: flooding 대신 수평 계층(horizontal layered) 스케줄로 계산량 ~절반, 메모리 30% 절감 → BP류처럼 수렴 가속.
5. 알고리즘 구조(Algorithm 1): 3단계 φ1(V→C 메시지 `T_ij`, 입력 projection `P_ji`) → φ2(Euclidean projection `w=Π_Pdc(P)`) → φ3(`z`,`λ` 갱신·VN update).
6. HW 아키텍처: NISC(No Instruction Set Computing) 기반 부분병렬, P개 CNU + Mu개 메모리, 두 Benes network로 H-matrix에 따른 interleave/deinterleave 라우팅, 오프라인 충돌 해소.
7. CNU: 3단 파이프라인 데이터패스, edge당 1클록 처리, 2dc-cycle latency로 VN 공유 충돌 회피, projection이 CNU 로직의 40% 차지.
8. 양자화: fixed-point Q12.8(Ti), Q10.8(z), Q11.8(λ), projection 입력 10비트(fractional 6) — NMS(Ti 8b/msg 6b)보다 큼.
9. 결과(Virtex6 FPGA): ADMM l2가 NMS 대비 LUT 6~7.5배·FF 5.5배, throughput 0.68~12.54 Mbps(NMS의 0.5~0.7배), 단 저SNR FER은 NMS보다 우수.
10. 한계: 나눗셈에 DSP 사용, 주파수 저하(대부분 100MHz 미달), throughput Mbps급으로 NAND용 Gbps급과 격차 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM l2 CN 연산(Euclidean projection) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal()` (대체 대상) | Prime ECC는 min-sum CN이며, projection 기반 CN으로 교체 시 면적 급증하여 이식 부적합 |
| VN update / z·λ 메시지 저장 | `decoder.cpp` `VN_Cal_HD()` 등 | ADMM은 VN당 메시지 2개(z,λ) 저장으로 메모리 2배, Prime ECC 구조와 비정합 |
| 수평 계층 스케줄링 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 계층 스케줄 개념만 유사, ADMM 특유 3단계는 대응 없음 |
| 고정소수 양자화 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC는 2~3bit 테이블, ADMM은 Q12.8급으로 폭 상이 |
| Benes network 라우팅 | 대응 없음 (Prime ECC는 z=32 QC circulant shift 기반) | QC 구조 라우팅과 다름 |

적용 가치: **낮음** — ADMM/LP 복호는 정정성능은 좋으나 Euclidean projection·나눗셈·2배 메시지 저장으로 min-sum 기반 Prime ECC 대비 면적·throughput 열위, NAND HW 예산과 맞지 않는다.
