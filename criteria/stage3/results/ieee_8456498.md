> 총평: binary QC-LDPC exponent matrix의 사이클 최소화 대수적 구성법으로 부호설계 관점에서 참고 가치는 있으나 다중 lifting/5G 특화라 Prime ECC 직접 이식성은 중.

### B. 알고리즘 요약

1. 대상은 binary QC-LDPC(`H`=CPM/ZM 배열, exponent matrix `P`의 shift value 설계), rate 1/5~8/9, 정보길이 100~8448, lifting size `Z=a·2^j`.
2. 풀려는 문제: 5G LDPC는 단일 exponent matrix가 다중 lifting size를 동시 지원해야 하는데 기존은 greedy search+대규모 시뮬에 의존해 복잡도가 크다.
3. 핵심 가정: 반복복호 성능은 Tanner graph의 짧은 사이클 구조와 직결(girth≥6, length-4 사이클 회피가 핵심).
4. 핵심 기법 1단: WANC 메트릭 `W(P)=Σ_i ω_Z(i)·Σ_j ω_c(j)·N(i,j)` — lifting size별·사이클 길이별 가중치로 다중 lifting에서의 사이클 수를 평균 정규화.
5. 핵심 식 의미: 식(5) `Σ(p_{iz,jz}-p_{iz+1,jz})≡0 (mod Z)`가 length-2d 사이클 존재 조건 → 이를 최소화하도록 shift value 선택.
6. 확장: circulant 계수표 `Tcc`를 유한체 `Fq` 원소로 대수적 구성(식 11~16), Latin square 성질로 행/열 내 원소가 서로 달라 사이클 제약 자동 충족.
7. 구현 디테일: 5G base matrix를 A/D/O/E/I 서브행렬로 나눠 첫 (kb+4) 열만 독립 설계(kernel Ab + Eb 행별 탐색)로 탐색 복잡도 축소.
8. 최적화 절차: Algorithm 2/3/4로 `Tcc`에서 행 선택→`W` 최소화 반복 탐색(Nsearch), base matrix는 P-EXIT/PEG로 사전 설계.
9. 결과: 설계 부호가 5G 표준·CCSDS 표준 부호와 BLER 10^-4~10^-6까지 거의 동등, semi-random/random-select 대비 우월. 설계된 B5G,2 exponent matrix가 3GPP 채택. CCSDS는 단일 exponent matrix로 K=1024/4096 동시 지원(저장복잡도 이득).
10. 한계: HW 미설계, 성능은 표준코드와 동등(우월 아님), 사이클 분석이 단일 lifting 기준이라 다중 lifting은 WANC로만 근사.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC exponent matrix / shift value 설계 | `ecc_top.cpp` `Load_PCM()` (H-matrix/PCM 로드) | base+shift 구조 동일, 사이클 최소 shift 선택이 부호설계 단계에 대응 |
| WANC 사이클 최소화 구성법 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정, 재설계 도구 없음 |
| dual-diagonal/bidiagonal parity 구조(D 서브행렬) | `encoder.cpp` `Generate_PCM_encoding()` | 논문의 bidiagonal D와 dual-diagonal 인코딩 구조가 유사 |

적용 가치: **중간** — binary QC-LDPC 부호설계(사이클/girth 최소 shift 선택)라는 코어 문제는 Prime ECC의 H-matrix 설계와 정합하나, 다중 lifting size·5G/CCSDS 특화 요건이라 단일 고정 lifting(z=32) 코드베이스에는 방법론 참고 수준.
