### B. 알고리즘 요약

1. 시스템: 일반 통신 모델(소스→LDPC 인코더→채널→디코더)에서 표준 선형 블록 부호로서의 binary LDPC를 다룸, QC 구조 없이 임의 parity-check matrix `H(m x n)` 기반.
2. 풀려는 문제: LDPC 인코더를 Verilog HDL로 구현해 FPGA 상에서 전력/면적을 실측·비교(선행 연구 대비 시스템 통합·오류정정 성능 평가가 아닌 하드웨어 구현 자체에 초점).
3. 핵심 가정: binary LDPC만 고려(non-binary는 배제), 매우 작은 예제 행렬((3,6), (8,12), (6,8), (10,12))로 개념 검증.
4. 핵심 기법: generator matrix 곱셈 방식의 straightforward 선형 인코딩. `c = xG` 형태(명시적 G 유도식은 본문에 없음), 유효성 검증은 syndrome `Z = cH^T = 0`.
5. 핵심 식의 의미: 인코더가 message `x`(길이 k)에서 codeword `c`(길이 n, rate `R=k/n`)를 생성 후 `H`로 syndrome 검사해 유효성 확인 - 표준 선형 블록부호 인코딩 절차.
6. 확장 기법: 없음(디코더 미포함, 순수 인코더만).
7. 구현 디테일: 회로는 MUX·XOR gate 기반 조합 로직(Fig.4), Xilinx Vivado로 합성, Artix-7(28nm)과 DE10-Lite(55nm) 두 FPGA에서 전력 비교.
8. 학습/최적화 절차: 없음.
9. 결과: 3비트 입력→6비트 코드워드, 32비트 입력→64비트 코드워드 등 시뮬레이션 검증. 전력은 rate 1/2~5/6 구간에서 0.075~0.077W(28nm), 55nm 대비 28nm이 전력(0.075W vs 0.106W)·I/O전력 모두 낮음. 자원 사용 LUT 3%, FF 1%(5/6 rate 기준).
10. 한계: 디코더 구현·복호 성능(BER 등) 전혀 다루지 않음, 코드 크기가 극소규모(최대 10x12)로 실용 QC-LDPC(수백~수천 bit) 규모와 무관, 구조화된 QC/protograph 설계 없이 임의 소규모 H-matrix 예제에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| generator matrix 곱셈 기반 선형 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 구조 인코딩(부호 PCM 기반)이라 본 논문의 임의 G-matrix 곱셈 방식과는 구조가 다름, 개념적 참고 수준 |
| 소규모 예제 H-matrix((3,6) 등) | `ecc_top.cpp` `Load_PCM()` | 규모·구조 모두 실제 QC-LDPC와 무관해 대응 없음에 가까움 |
| MUX/XOR 조합회로 기반 인코더 회로 | `encoder.cpp` `Generate_PCM_encoding()` | 개념적 유사성만 있을 뿐 QC-LDPC dual-diagonal 인코더 구조와 직접 대응되지 않음 |
| 디코더 알고리즘 | 대응 없음 | 본 논문은 디코더를 다루지 않음 |

> 적용 가치: `낮음` — 소규모 교과서 예제 수준의 선형 LDPC 인코더 구현으로 QC-LDPC 구조나 복호 알고리즘 기여가 없어 Prime ECC에 이식할 요소가 사실상 없음.
