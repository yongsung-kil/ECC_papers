### B. 알고리즘 요약

1. 대상은 QC-LDPC로, base matrix에 circulant permutation matrix `P^i`(size `L×L`)를 배치해 `mL×nL` PCM `H`를 만든다.
2. 문제: PEG는 girth 큰 부호를 주지만 QC 구조가 아니라 인코딩이 어렵다. QC 구조로 linear-time 인코딩을 얻으려 한다.
3. 먼저 PEG 알고리즘으로 binary basic matrix `M(H)`를 구성한다.
4. basic matrix의 ring(basic ring)이 확장 후 `H`의 ring으로 전이되는지를 shift difference로 분석: 수평/수직 shift 차 `Δa`의 합이 `kL`이면 `2g`-length basic ring이 `H`에서 `L`개의 ring으로 확장된다(Conclusion 1).
5. 즉 `Σ_H(Δa)`, `Σ_v(Δa)`가 `kL`이 아니도록 exponent를 조정하면 해당 길이의 ring이 생기지 않는다(식 6,7).
6. Theorem 2: `2l`-length basic ring의 exponent chain에 대해 `r·Σ(-1)^i a_i = 0 mod L`을 만드는 최소 `r`로 확장 ring 길이(`>=2lr`)를 통제.
7. 구성 절차: basic matrix 생성→4/6(옵션 8)-length ring 탐색→식(7) 위배 시 exponent를 `+1 mod L`씩 조정→Theorem 1,2 충족까지 반복→circulant로 확장.
8. 별도 학습/최적화 없음(그래프 조합적 규칙 기반).
9. 결과: rate 1/2, row weight 6, col weight 3, girth 8인 (1152,576),(4000,2000) 부호를 BP 복호로 시뮬; BER가 PEG 부호와 유사하고 Gallager/Mackay보다 우수.
10. 한계: HW 미설계, 시뮬만, 정정능력 이득 없이 인코딩 복잡도만 낮춤. block-PEG는 long code 지향(L,m을 크게 잡아야 함).
