> 총평: IEEE802.16e 12종 rate/length를 EPROM 파라미터로 지원하는 programmable serial 디코더로, extrinsic memory를 2-message A-min 근사로 크게 줄이고 FPGA 160MHz 합성 결과까지 제시했으나 완전병렬 고속 NAND ECC보다는 저면적/유연성 지향 설계라 이식 시 구조 재설계가 필요함.

### B. 알고리즘 요약

1. IEEE802.16e 기반 block-structured irregular QC-LDPC, rate 1/2·2/3·3/4, length 576/1152/1728/2304, AWGN 채널.
2. 문제: block-serial 구현은 하드웨어 자원을 줄이지만 지연이 커져 throughput이 낮아지고, min-sum은 0.1dB BER 손실이 있음. 다양한 rate/length를 하나의 디코더로 지원(재합성 없이)하는 것도 과제.
3. 가정/모델: 각 row(check node)를 하나의 layer로 취급해 layered shuffled BP(LSBPA)를 serial 모드로 변형.
4. 핵심 기법 1단(LSBPA): `L(qji)[k] = L(Qj)[k,r-1] - L(rij)[k-1]` (식1), check node 갱신 `L(rij)[k]` (식2, `φ(x)=log(tanh(x/2))` 사용), `L(Qj)[k,r] = L(qji)[k] + L(rij)[k]` (식3)로 표준 BP 대비 수렴이 약 2배(논문 인용: 70% 더) 빠름.
5. 핵심 기법 2단(수정 A-min): `M1(i)=Σφ(L(qji)[k])`, `M2(i)=M1(i)-φ(min|L(qji)|)` 정의 후, 최소 edge는 `L(rij)=-φ⁻¹[M2(i)]`, 나머지 edge는 `L(rij)=-φ⁻¹[M1(i)]`로 근사 — check node degree와 무관하게 단 2개의 extrinsic message만 저장.
6. 이 근사로 LSBPA+A-min은 LSBPA(근사 없음)와 거의 동일한 BER을 유지하면서 min-sum(0.1dB 손실) 대비 우수한 트레이드오프를 달성.
7. 구현 디테일: 3-stage pipeline(read/process/write)로 read-after-write 충돌 회피, CNPU는 accumulator+subtractor로 add-tree 대체해 φ(x) 연산 유닛 수 축소, greedy comparator(GCMP)로 최소값·인덱스 탐색, 8-bit fixed-point message width 채택(7/8/9bit 비교 후 8bit 선정).
8. 별도 학습/최적화 절차 없음(고정 알고리즘, 파라미터는 EPROM에 rate/length별로 사전 저장해 재합성 없이 전환).
9. 결과: extrinsic memory 68.4%(rate 1/2)~85.9%(rate 3/4) 절감(rate가 높을수록 절감폭 증가). FPGA(Stratix-II EP2S60F672C5) 합성 160MHz, 총 257 ALUT / 69518 RAM bit / 182 REG. Max iteration 10/15/25 지원.
10. 한계: 완전병렬이 아닌 block-serial 구조로 처리량이 병렬 대비 제한적. Throughput 수치(Gbps)는 본문에 없음. 정정능력 개선(waterfall/error-floor) 관련 언급 없이 메모리/자원 효율화에 집중.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| A-min 근사(2-message만 저장) CN 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 계열 대체 근사로 CN 연산부에 대응, 개념 중복(이미 min-sum 보유) |
| LSBPA(layered shuffled BP) 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update(odd/even 교대) 스케줄과 유사한 layered 스케줄 아이디어 |
| Programmable EPROM 기반 multi-rate/length 컨트롤러 | `ecc_top.cpp` `Load_PCM()` | H-matrix/파라미터 동적 전환 아이디어로 대응 가능하나 Prime ECC는 특정 QC-LDPC 고정 |
| 3-stage pipeline(read/process/write) | `decoder.cpp` pipeline 6-stage(col_M1/Z/P1~P4) | Prime ECC가 이미 더 세분화된 pipeline 보유, 개념 중복 |
| Lr/LQ RAM 주소생성기 | `channel.cpp`, `ecc_data.h` PCM 구조체 | 메모리 라우팅 설계 참고 가치 있으나 코드 직접 대응 없음 |

적용 가치: `중간` — 메모리 절감형 A-min CNU와 multi-rate programmable 구조는 참고 가치가 있으나, min-sum/layered-schedule 개념이 이미 보유 기법과 겹치고 완전병렬이 아닌 serial 구조라 고throughput NAND ECC에는 구조적 재설계 부담이 큼.
