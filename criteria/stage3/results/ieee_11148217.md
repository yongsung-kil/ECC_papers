> 총평: SC-LDPC의 6-cycle을 GCD 프레임워크로 완전 제거해 error-floor 관련 BER을 개선하는 부호 구성법으로, 정정능력 개선이 부호설계 단계에서 나오므로 대상은 both이나 디코더/HW 자체를 다루지 않고 SC-LDPC 특유의 convolutional coupling 구조라 Prime ECC(block QC-LDPC)에 직접 이식하기는 어려움.

### B. 알고리즘 요약

1. QC-SC-LDPC는 block protograph `B`를 `w+1`개 component matrix로 분해하고 시간축으로 결합(coupling width `w`)한 convolutional 구조이며, 코드율은 `R_SLC = 1-(L+w)p/Lq`.
2. 기존 explicit 구성법(edge spreading, "good sequence" 기반 spreading matrix `Ms`)은 4-cycle은 제거하지만 6-cycle은 남아 error-floor를 악화시키는 문제가 있음.
3. 채널은 SW(sliding window) 디코딩을 사용하는 BPSK+AWGN 가정, window size는 무한대(W→∞)로 시뮬.
4. 핵심 기법 1단: GCD 프레임워크로 만든 exponent matrix `Mg = S2^T·S1`(오름차순 시퀀스 `S2`가 GCD 제약 `(s_c-s_a)/gcd(s_c-s_a,s_b-s_a) ≥ q` 만족)를 spreading matrix `Ms`의 component matrix `M_c^e`의 non-negative 원소 자리에 대입해 값을 부여.
5. 핵심 식: Theorem 2 - `E^L_SC(Ms,Mg)`가 6-cycle-free ⇔ `Mg`가 6-cycle-free(식 6의 Lemma 2 조건). 즉 3×3 submatrix 조건 검사를 spreading matrix 전체가 아닌 exponent matrix `Mg`로 축소해 6-cycle 제거를 증명 가능하게 함.
6. QC lifting: 각 component matrix 원소를 circulant permutation matrix(CPM)로 치환해 최종 QC-SC-LDPC PCM `H`(식 7) 생성.
7. 부수 트릭: 6-cycle 카운트를 위한 전용 탐색 알고리즘(Algorithm 2) - coupling 구조의 반복성을 이용해 처음 `w`, `w+1` block만 탐색하고 `N=N0+(N1-N0)(L-w)`로 전체 6-cycle 수를 대수적으로 계산, 전수탐색 대비 훨씬 빠름(표 I: 기존 good-sequence 방식 대비 6-cycle 수 1000+개→0개, 탐색시간도 단축).
8. 별도 학습/최적화는 없음(폐형 GCD 시퀀스 탐색 + Algorithm 1의 파라미터 p,q,R*,n*로부터 L,P 역산).
9. 결과: (1860,653) rate 0.35, (4560,2168) rate 0.475, (3348,1397) rate 0.42 세 예제에서 기존 구성법[5],[15] 대비 BER/BLER 개선 확인(예: Fig.4에서 SNR>1.75dB부터 우위), 6-cycle 수를 0으로 완전 제거.
10. 한계: 디코더는 표준 SW-BP(무한 window)를 그대로 사용해 알고리즘/HW 개선 없음, coupling width가 message recovery latency에 미치는 영향은 언급만 되고 정량 분석은 미흡, 시뮬(AWGN)만으로 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GCD 기반 exponent matrix / spreading matrix 구성(girth-8 SC-LDPC) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 대체하는 구성법이나 Prime ECC는 특정 고정 QC-LDPC(block, non-coupled)를 사용해 재검증 부담 큼, 게다가 SC-LDPC 구조 자체가 Prime ECC와 다름 |
| 6-cycle 탐색 알고리즘(Algorithm 2) | 대응 없음 | 부호 생성 단계 오프라인 도구, 런타임 디코더와 무관 |
| sliding-window BP 디코딩(무한 window, 50 iter) | `decoder.cpp` `LDPC_Decoder()` | 개념적으로 iterative decoding 루프이나 SC-LDPC 특유의 window 기반 스케줄로, block LDPC인 Prime ECC의 이터레이션 루프와 구조가 다름 |

> 적용 가치: 낮음 — error-floor 개선이라는 목표는 Prime ECC의 관심사(§5)와 부합하지만, SC-LDPC(convolutional coupling) 구조 자체가 Prime ECC의 block QC-LDPC와 근본적으로 다르고 순수 부호 구성법(디코더/HW 무관)이라 이식 난이도가 높음.
