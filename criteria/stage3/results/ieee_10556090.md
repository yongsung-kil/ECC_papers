### B. 알고리즘 요약

1. 시스템: LED 기반 imaging MIMO 광무선통신(OWC), 8개 광변조기(LED) 송신 + 16개 APD 수신, direct detection으로 LED-APD 쌍(서브채널) 선택, 옥외 50~100m 거리 실험.
2. 문제: 무부호화 시 100m는 에러프리이나 그보다 짧은 거리(예: 90m)에서 광학계 정렬 불완전으로 서브채널별 BER 편차가 크게 발생(LED별 최대 6×10⁻²), 균일 신뢰도 가정 디코딩은 성능이 나쁨.
3. 핵심 가정/모델: 각 서브채널(m번째 LED-nm번째 APD)은 시불변 이진대칭채널(BSC)로 모델링되며, 사전 훈련열로 측정한 서브채널별 `Pe+`, `Pe-`(0/1 반전 확률)가 시간에 걸쳐 균일함을 실측으로 확인.
4. 핵심 기법: 부호화측은 Gallager 구성법(`N`, row weight `wr`, column weight `wc`)의 표준 랜덤 LDPC 부호를 사용, 복호측은 각 서브채널의 사전측정 BER을 신뢰도(likelihood)로 반영한 MAP 기반 sum-product 알고리즘으로 가중 복호.
5. 핵심 식: 부호화 효율 평가를 위해 부호화 전/후 채널용량 `C0 = Σ{1-h(pm)}`(서브채널별), `Cc = 8R{1-h(p)}`(부호화 후 단일 등가 BSC)를 비교, `h(P)=-Plog2P-(1-P)log2(1-P)` — Shannon limit 대비 실제 코드 성능 갭 평가용.
6. 확장: 없음 (단일 sum-product 디코딩 기법).
7. 부수 트릭: 균일 신뢰도(서브채널 구분 없음) 가정과 비교해 서브채널별 BER 가중이 성능상 우수함을 실증 — 채널별 비대칭 에러율을 LLR 초기값에 반영하는 것이 핵심 트릭.
8. 학습/최적화 절차: 없음 (코드 파라미터는 시뮬레이션 grid search로 rate/length/wr/wc 조합 중 최고 성능 선정).
9. 결과: 무부호화 시 90m 평균 BER 4.42×10⁻²였으나 rate 0.6 코드로 에러프리 달성; rate 0.9(70m), rate 0.95(80m)에서도 에러프리; 90m에서도 rate 0.8 이상에서 Shannon limit에 근접.
10. 한계: 부호설계는 표준 Gallager 랜덤 구성법 그대로이고 디코더도 표준 sum-product(HW 미설계, gate/throughput 없음); 검증은 실제 옥외 광 MIMO 실측 데이터 기반이나 채널이 광무선(BSC 근사) 특유의 모델로 NAND read channel과 근본적으로 다름.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 서브채널별 BER 가중 sum-product 디코딩 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 채널(서브채널)별 사전측정 에러율을 LLR에 반영하는 아이디어는 NAND 워드라인/셀별 soft-read threshold 설정과 개념적으로 유사 |
| Gallager 구성법 랜덤 LDPC (H-matrix) | `ecc_top.cpp` `Load_PCM()` | 표준 랜덤 구성법으로 QC 구조 아님, Prime ECC의 QC-LDPC(base+circulant)와 직접 대응 안 됨 |
| sum-product(BP) 디코딩 | 대응 없음 | Prime ECC는 min-sum 기반(HW 면적/전력 절감), full sum-product(tanh)는 미보유 기법으로 이식 시 재설계 필요 |
```

적용 가치: `낮음` — 부호구조(Gallager 랜덤)·디코더(full sum-product)가 Prime ECC의 QC-LDPC min-sum 구조와 상이하고, 서브채널 가중 개념만 참고 가치가 있으나 이미 Prime ECC도 채널 기반 LLR threshold를 보유.
