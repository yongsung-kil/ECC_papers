### B. 알고리즘 요약

1. 대상: 시간불변 rate `R=b/c` LDPC convolutional code(LDPC-CC), syndrome former memory `ms`, 예시는 `(21,3,5)` Tanner LDPC-CC (rate 2/5=4/10).
2. 문제: 시변채널 대응 rate-compatible 부호가 필요하나, LDPC-CC의 distance spectrum은 큰 `ms`에서 정확 계산 불가하고 iterative 복호는 short cycle·trapping set에 민감.
3. 핵심: mother code에서 주기 `P` 천공패턴 `a` (P×c 이진행렬)로 부호율을 `R=(P·b)/(P·c−l)`로 조절, nested 패턴으로 rate-compatible 계열 생성.
4. 기준1 (recoverability): 천공노드의 m-step-recoverable(m-SR) 등급 최소화, ∞-SR(복구불가) 노드 회피 → 열거벡터 `Ec_ai`.
5. 기준2 (CPCTS): completely-punctured cycle trapping set(cycle의 w/2 변수노드 전부 천공) 개수 최소화 → `Eτ_ai`, error-floor 억제.
6. 기준3 (short cycle): 짧은 cycle에 포함된 천공 변수노드 수 최소화 → `Eb_ai`, waterfall 수렴 개선.
7. 알고리즘1: Step1(∞-SR 최소+recoverability)→Step2(CPCTS 최소)→Step3(short-cycle 천공노드 최소) 순차 필터로 최적 패턴 선택.
8. cycle enumerator: `(21,3,5)` 코드에서 `R8=11, R10=62, R12=351`, girth 8; cycle당 고유 cycle trapping set 대응.
9. 결과: rate 4/9~4/5 계열 획득, 시뮬(BPSK/AWGN, 50 iter sum-product)로 선택 패턴 우월 확인; 고rate(4/5)는 ∞-SR 불가피 → P를 4로 늘리면 R=8/10에서 BER 10^-3 기준 약 2.1dB gain·error floor 제거.
10. 한계: HW 미설계, 시뮬뿐, distance spectrum 대체 지표는 계산가능하나 LDPC-CC(스트리밍 구조) 전용이라 block QC-LDPC와 이질적.
