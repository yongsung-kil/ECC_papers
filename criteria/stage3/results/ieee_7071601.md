### B. 알고리즘 요약

1. 4-cycle free random LDPC 코드(4095,738,4,102, rate 0.82, quasi-regular)를 BPSK/AWGN 채널에서 시뮬레이션.
2. 기존 mod-MS(Min-Sum) 디코더의 CNP(Check Node Processor)가 여전히 전체 복잡도를 지배하는 문제를 해결하려 함.
3. CN 크기 업데이트에 modified Min-Sum(mod-MS) 근사 `M-min*(|a|,|b|) ≈ min(|a|,|b|) - log((1+e^-d)/(1+e^-d))`, `d=||a|-|b||`를 채택.
4. 기존 2-output 규칙[13]을 P+1-output으로 일반화: `ℓ_ij = M-min*_{k∈N(i)\{j}}(|μ_ik|)` (j가 최소신뢰도 P개 집합에 속하면 전용값, 아니면 전체 집합에 대한 공통값).
5. 이 식은 CN이 전달하는 서로 다른 magnitude 값의 개수를 노드 차수(dCN)와 무관하게 P+1개로 고정해 저장·연산량을 줄이는 것이 핵심.
6. P=1(2-output)일 때 입력을 정렬해 최소값 하나만 레지스터로 유지하는 2-way MUX 구조로 단순화(Fig.3).
7. 부호(sign) 업데이트는 별도 CN Sign Processor에서 XOR 기반 exact rule로 수행, magnitude와 분리.
8. 유한정밀도(5bit vs 6/7bit) 양자화 실험에서 P=1(2-output) 근사가 오히려 mod-MS 대비 양자화 손실을 줄임(0.25dB→0.2dB).
9. 결과: P=1 CNP가 865 gates(dCN.max=30, Nm=6bit)로 기존 forward-backward 구조[16] 대비 2~6.5배 절감, DVB-S2 대비 노드프로세서 복잡도 약 59% 절감, 최대 클록 200MHz 이상.
10. 한계: random code 기반 시뮬레이션이며 P>3에서는 비효율적, VLSI 검증은 CNP 단독이고 실제 QC-LDPC나 SSD 채널 특화 검증은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| mod-MS 기반 CN magnitude 근사(M-min*) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min-sum 계열 보유, 근사 방식만 다름 |
| P+1-output 압축 CN 프로세서 | `decoder.cpp` `C2V_Cal()`, `C2V_Cal_New_Sgn()` | 출력 압축 아이디어는 GT.cpp 방식과 유사한 목적, 직접 대응 없음 |
| 부호(sign) 업데이트 XOR | `decoder.cpp` `C2V_Cal_New_Sgn()` | 이미 동일 원리 구현 |
| 압축 메시지 저장(라우팅 절감) | `GT.cpp` `Make_GT_HW()` | 유사 목적(edge/메시지 압축)의 HW 최적화 |

> 적용 가치: 낮음 — mod-MS·min-sum 기반 CNP 면적 절감 기법이나 우리 코드가 이미 min-sum, HCU, GT로 유사 목적을 달성하고 있어 중복 기여.
