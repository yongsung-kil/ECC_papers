### B. 알고리즘 요약

1. 시스템: QC 구조 SC-LDPC(circulant size `z`, base `Π`=all-one `γ×κ`), 파라미터 `(γ,κ,m,z,L)`; 플래시 실험은 `(4,24,6,17,40)`, length 14450, rate~0.75.
2. 문제: SC 부호는 memory `m`이 클수록 성능↑지만 유한길이 고memory 부호를 조합적으로 최적화(OO-CPO)하면 계산량이 폭증 — 이 병목을 깬다.
3. 핵심가정: 유한길이 성능은 Tanner graph 내 detrimental object(short cycle, absorbing set) 개수가 지배. cycle 조건은 lifting `L(ik,jk)` mod z 로 판정(Lemma 1).
4. 핵심기법 1 — GRADE: partitioning matrix의 edge 분포 `p`(각 component matrix로 갈 '1'의 확률)를 변수로, protograph 내 기대 cycle/object 수를 coupling polynomial `f(X)=Σpi X^ai`의 상수항으로 표현, gradient descent로 국소최적 `p` 도출.
5. 핵심식: cycle-6 기대확률 `P6=[f^3(X)f^3(X^-1)]_0`, cycle-8은 6개 구조 가중합(Thm 2). 최적성 필요조건은 `[f^3(X)f^2(X^-1)]_ai=c0`(Lagrangian).
6. 핵심기법 2 — 일반화(GRADE): 임의 object를 cycle basis(fundamental cycle 집합)의 characteristic polynomial로 표현, absorbing set을 이루는 연쇄사이클(2-1-2/2-1-3/2-2-2/3-1-3)까지 미세최적화.
7. 핵심기법 3 — AO: GRADE로 얻은 `p` 근방으로 탐색공간을 제한한 semi-greedy로 partitioning matrix 탐색(GD/UNF/TC codes), 이어 CPO로 lifting 최적화. 복잡도 `O(Kmγκ dv^2 dc^3)`로 OO 대비 완만 증가.
8. 학습/최적화: gradient descent(edge 분포) + 조합 탐색(partitioning/lifting), NN 없음.
9. 결과: Flash(NLM, sub-20nm MLC, 3 read, GF(4), FFT-QSPA 50iter) 채널에서 GD/TC가 UNF 대비 object 40%↓, RBER 0.0235에서 UBER 약 2 orders 이득, error floor 없음. (3,17) GD는 cycle-8 29%↓.
10. 한계: HW 미설계(시뮬만), 부호가 SC(convolutional)+주로 GF(4) non-binary+windowed/FFT-QSPA decoding 전제, 순수 binary block QC-LDPC 대상이 아님.
