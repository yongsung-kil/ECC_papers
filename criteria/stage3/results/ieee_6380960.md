### B. 알고리즘 요약

1. 대상: short-block irregular LDPC (수정 array code 기반), rate 0.7~0.8, block length 513~1020, AWGN 채널.
2. 문제: 랜덤 H 생성은 계산량이 크고, CRT/SC-array 방식은 short-block에서 성능 저하 + 구성 복잡도 높음.
3. 아이디어: H를 다수의 `p×p` sub-permutation(순환 시프트) 블록으로 보고, 각 블록의 시프트량을 `z×z` magic square의 숫자들로 정함(랜덤 대신 결정론적 절차).
4. 파라미터 `j,k,p` (`j<k≤p`) 정의, `λ=(j·k)-(j(j+1)/2)-(k-1) ≤ z²` 조건으로 필요한 서로 다른 숫자 개수 확보.
5. 매직스퀘어 크기는 `p/2 < z² < p` 범위에서 그룹 1~4(Agrippa/odd/doubly-even/singly-even)의 3~9차 사용.
6. 시프트 배치 방법 3종: Method I(원본 숫자·위치 그대로), Method II(1부터 순서대로 독립 선택), Method III(절반은 magic square, 절반은 정규분포 되게 배치).
7. `p` 이상 숫자는 modulo p로 접고, 이미 쓴 숫자는 가장 가까운 미사용 숫자로 대체(중복 방지).
8. 27개 H 행렬을 동일 조건(length/rate/iter/z)에서 개별 시뮬 비교.
9. 결과: length 513에서 Method III + iteration 30이 최적, BER 10⁻⁷ @ SNR 5dB(rate 0.8) 달성, MAC[6]/CRT[8] 대비 우수(특히 5dB).
10. 한계: 순수 구성법으로 HW 미설계, AWGN 시뮬뿐, error-floor·복호기 구조 논의 없음.
