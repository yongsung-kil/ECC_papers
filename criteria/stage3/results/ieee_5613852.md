### B. 알고리즘 요약

1. 대상 시스템: AWGN 채널, `c=uG`, LLR `Ai=(2/σ²)yi`, `(dv,dF)`-regular binary LDPC, 예제 `H[20,10]` (3,6)-regular.
2. 문제: MS/SP 반복 디코더의 error floor는 low-weight trapping/absorbing set이 지배하는데, Monte Carlo로는 관측이 계산적으로 불가능하고 상한도 없다.
3. 핵심 개념: computation tree 위의 deviation(각 check가 변수노드 0 또는 2개에 인접, root 포함)의 negative cost가 root 에러의 필요조건. `P(vi=1) ≤ Σ P(G(δ)<0)`, `G(δ)=Σ Ai`.
4. 관계 규명: stopping set은 그 변수노드만으로 deviation 형성 가능; trapping/absorbing set은 내부에 stopping set이 있을 때만 가능.
5. absorbing set의 deviation 내 A-노드 수 하한 `|δ_A| ≥ 1 + dA Σ(dA-1)^(i-1)`, 전체 `|δ| = 1 + dv Σ(dv-1)^(i-1)`.
6. Trapping set 탐색(Algorithm IV.1): 변수노드에 LLR 0/1을 부여하고 MS를 e회 돌려 최소-weight deviation을 가장 많이 줄이는 노드를 하나씩 추가, syndrome으로 stop/trap/abs/full-abs 분류.
7. deviation weight 계산: stopping set으로부터 방향성 Tanner graph를 만들고 multiplicity 점화식 `mi = A·mi-1 − B·mi-2`(홀수 레벨)로 각 노드 복제수를 구함.
8. weight 정의: cost가 정규분포 `N(Σa_vi, σ²Σa_vi)`를 재척도한 평균 `(Σa_vi)²/Σ(a_vi)²`.
9. 결과: `H[20,10]`에서 30개 trapping set 발견, 시뮬 non-codeword 에러의 82%가 이들과 일치. (2,2) fully-absorbing set의 deviation weight가 3.8784 < 최소거리 4 → ML 코드워드 에러보다 잦은 MS 에러 필요조건 존재 입증.
10. 한계: N=20 예제뿐, HW/실코드 없음, 계산량이 iteration 수에 지수적, 개선책이 아닌 분석 도구.
