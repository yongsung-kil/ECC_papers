> 총평: layered scheduling + NMS로 iteration 절반(15->5), reconfigurable sorter와 라우팅 pre-coding으로 4-rate 공유·MUX 64% 절감한 고throughput 실측칩. 스케줄/NMS는 우리와 중복, 라우팅 최적화 아이디어만 부분 참고 가치.

### B. 알고리즘 요약

1. 대상: IEEE 802.15.3c 4종 irregular QC-LDPC (`N=672`, rate `0.5/0.625/0.75/0.875`, `z=21` circulant), H를 4개 layer(H0~H3)로 수평 분할.
2. 문제: 고throughput 요구 + 4-rate 지원 HW 오버헤드 + irregular 부호의 복잡한 라우팅.
3. 핵심기법1 (row-based layered scheduling + NMS): 행을 `G=4` layer로 나눠 sub-iteration마다 layer별 순차 갱신, 갱신정보 빠른 재활용으로 수렴 가속.
4. NMS 갱신식: `rmn = ∏sign(qn'm)·min|qn'm|·β` (`β=0.75` normalized factor로 min-sum 근사오차 보정).
5. VN/사후 LLR: `qnm = Pn + Σrm'n`, `Zn = Pn + Σrm'n` — layer 순차로 최신 message 전파해 반복수 감소.
6. 핵심기법2 (reconfigurable 8/16/32-input sorter): 32-sorter = 8-sorter 4개 합성, 행차수 8/16/32 비호환을 단일 HW로 통합.
7. 라우팅 최적화: sorter input reallocation(12-to-1 MUX->4-to-1) + pre-coding routing network(C-to-V coarse 선택)으로 MUX 입력수 177408->63168 (64% 절감).
8. 학습절차: AWGN/16-QAM 시뮬로 β=0.75, 6-bit(4정수+2소수) 양자화 결정.
9. 결과: 65nm 1P10M CMOS 실측, 5.79Gbps@5iter, 723k gate(디코더 647k), 62.4pJ/bit, 3.7Gbps/mm2 (5iter이 표준BP 15iter와 유사 BER).
10. 한계: 정정능력 개선 없음(수렴만 가속), 특정 표준부호 4종 종속, 라우팅 절감이 부호구조 의존, soft-read 비용 고려 없음.
