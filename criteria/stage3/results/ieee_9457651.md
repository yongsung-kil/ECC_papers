### B. 알고리즘 요약

1. 대상은 (3,6)-regular protograph 기반 SC-LDPC 코드 체인(edge-spreading, 결합폭 `m=2`, lifting `M`), sliding window decoding(SWD, window `W`).
2. 문제: 저지연 위해 `W`를 작게 하면 저SNR에서 드물지만 심각한 디코더 오류전파(error propagation)가 발생, 특히 큰 프레임에서 성능 붕괴.
3. 모델: 디코더를 3-state 마르코프(`Sre0`, `Sre1`, `Sbe`)로 모델링, 진입확률 `p`와 burst 탈출확률 `r`로 BLER 표현.
4. 핵심식: 점근 BLER `PBL = (rp+p^2)/(r+rp+p^2)` (식 3), `r=0`이면 `PBL=1`(무한 전파), `r`↑이면 BLER↓.
5. 유한 L 분석: round-trip 시간 `x=1/q+1/r`(`q≈2p^2`)로 프레임 길이 `L` 의존 BLER 도출(식 5), 3개 case로 구분.
6. 핵심기법: 적응형 VN doping — 최근 블록 평균 LLR 크기 `Lt`가 임계 `η` 이하로 `Nr`회 연속 실패하면 피드백채널로 도핑 요청, 다음 VN 블록을 known값으로 고정.
7. 적응형 doping의 탈출확률 `r = 2/(W+2Nr-4)`, 주기적 doping은 `r=2/s`(간격 `s`)로 대응.
8. 이점: 주기적 doping 대비 필요할 때만 도핑점 삽입 → rate loss 최소화하며 오류전파 즉시 절단.
9. 결과(시뮬): (3,6)-regular, `M=2000`, `W=12`, `L=1000`, AWGN/BPSK에서 적응형이 undoped·주기적 doping보다 BLER 우수(예: 강코드 점근 `PBL` undoped=1 → periodic 0.0196 → adaptive 0.0108).
10. 한계: HW 미설계, 시뮬/이론 수준, 노이즈리스 피드백채널·스트리밍 SC-LDPC 전제로 NAND ECC와 거리 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC/SWD 부호 구조 및 doping | 대응 없음 | Prime ECC는 블록기반 QC-LDPC로 SC-LDPC/sliding window·doping 개념 자체가 부재 |
| 오류전파 3-state 모델 / BLER 분석 | 대응 없음 | 스트리밍 SWD 전용 분석, 블록 디코더에 직접 대응 없음 |
| 적응형 VN doping (피드백 요청) | 대응 없음 | 노이즈리스 피드백채널 전제, Prime ECC 파이프라인에 대응 모듈 없음 |

적용 가치: **낮음** — SC-LDPC 스트리밍 SWD의 오류전파 완화 기법으로 블록기반 NAND binary QC-LDPC와 코드 구조·전제가 근본적으로 달라 떼어 쓸 요소가 없음.
