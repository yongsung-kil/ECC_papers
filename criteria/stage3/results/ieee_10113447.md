> 총평: QC-LDPC base matrix에서 임의 길이 route를 트리 탐색으로 열거하는 부호구성 전처리 도구로, 이론 단계이며 디코더/HW 기여가 없어 Prime ECC 이식성은 낮다.

### B. 알고리즘 요약

1. 대상은 protograph lifting 기반 binary QC-LDPC(base matrix + z×z CPM), rate `R=k/n`, 예시 CCSDS R=1/2 부호; circulant weight `w_i,j>1`(평행 edge) 지원.
2. 문제: QC-LDPC 구성 전 base graph route 목록이 필요한데, 기존 short-cycle 결합법은 길이>10이거나 평행 edge가 있으면 조합 폭발·미포착 구조 발생.
3. 모델: route `r`(같은 VN에서 시작·종료하는 edge 수열)가 리프트 그래프의 cycle로 사상되며, `Σ(-1)^k·s_k = 0 mod z`(Fossorier[2])로 연결.
4. 핵심기법1: 각 VN에서 원하는 길이의 절반 깊이 트리를 구성하고, 서로 다른 edge로 뻗은 sub-tree의 branch를 결합해 route 생성(같은 종단 노드+다른 도달 edge 조건).
5. 핵심 식: `Σ_{k=0}^{g-1}(-1)^k s_k = 0 mod z` — route가 lift 후 cycle이 될 조건이자 저장 대상(순서 무관, 부호만 필요).
6. 핵심기법2: VN을 순차 처리(v0 route 찾고 연결 삭제→v1…→마지막 열)해 전체 route 완전 열거, 평행 edge 때문에 마지막 열도 탐색.
7. 부수트릭: half-route를 종단 노드·도달 edge별로 분할 저장해 조건 검사 없이 결합; edge 번호(2B) 또는 "turn"(1B) 표현으로 메모리/속도 trade-off.
8. 부수트릭2: route를 양/음 edge 집합으로 정렬 저장 → 공통 edge 제거(짧은 route 중복 배제), 양=음이면 lift로 못 없애는 balanced cycle 조기 검출, 동일 route 복제 제거.
9. 결과: 다양한 부호에서 길이 20까지 route 열거 성공, CCSDS R=1/2 부호의 cycle 길이 4~20 min/max EMD 값 산출(Table I).
10. 한계: HW/디코더/BER 없는 순수 구성 전처리, 리프트 그래프 자체의 cycle 분포·EMD-spectre 산출은 향후 과제로 남김.
