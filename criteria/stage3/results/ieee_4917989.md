### B. 알고리즘 요약

1. AWGN 채널, BPSK 매핑 하에서 sum-product 디코딩되는 irregular LDPC 코드 앙상블(degree distribution `λ(x)`, `ρ(x)`)의 decoding threshold를 계산하는 문제를 다룸.
2. 기존 Density Evolution(DE)은 무한차원 메시지 밀도 계산이 필요해 복잡하고, Chung의 Gaussian Approximation(GA1)은 `φ` 함수 기반 평균 갱신을 쓰며, Lehmann&Maggio의 GA3는 오차확률 직접 사상을 쓰는데 정확도가 낮음.
3. 핵심 가정: 중심극한정리에 의해 반복 디코딩 메시지는 Gaussian(또는 Gaussian mixture)으로 근사되며, symmetry 조건 `2m = σ²`을 만족.
4. 핵심 기법: `φ` 함수 대신 Gallager의 원 결과와 Q함수를 결합해 check-to-variable 메시지 평균 `m_u^(l)`을 오류확률 `P_e^(l)`로부터 직접 유도(식 6~9), 재귀식 `m_u^(l) = Σρ_j·φ⁻¹(...)` (식 10)로 평균을 갱신.
5. 핵심 식: `P_e^(l) = Q(m_v^(l)/√2)`(식 7)는 variable-node 메시지 평균에서 오류확률을 직접 계산해 φ 함수 호출 없이 반복 갱신 가능하게 함 — 이것이 GA1 대비 연산속도 이점의 근거.
6. threshold `s*`는 `sup{s>0 : lim_{l→∞} p_l(s)=0}`로 정의되며, 단조성 분석(f, h, g 함수)으로 수렴/발산 조건을 규명(식 13~15).
7. 부수 트릭: φ(x) = Q(√(x/2)) 함수를 정의해 재귀식을 간결하게 표현, GA1의 tanh 기반 φ 함수 호출을 없애 실행속도 향상.
8. 학습/최적화 절차 없음(닫힌형 재귀식 기반 수치 계산).
9. 결과: regular 코드 9종·irregular 코드 4종(A~D)에서 DE 대비 threshold 오차(Error2)가 GA1(Error1)보다 크지만 GA3(Error3)보다는 작음(예: irregular code D는 Error2=0.3670dB vs Error3=0.6663dB); rate 1/2 irregular 코드 예시로 threshold `s*=2.3558` 도출.
10. 한계: 순수 이론적 threshold 계산 도구로 시뮬레이션/FPGA/HW 검증 없음, 특정 유한장 코드가 아닌 무한장 앙상블 근사이며 정확도가 기존 GA1보다 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Degree distribution 기반 threshold 계산 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 H-matrix(QC-LDPC) 사용, 앙상블 최적화 단계는 이미 지난 상태라 이 도구는 부호 설계 초기 단계에만 관련 |
| Gaussian Approximation 재귀식(φ, Q함수) | 대응 없음 | 디코더 알고리즘이 아닌 이론 해석 도구, decoder.cpp의 실제 min-sum 연산과 무관 |
| Sum-product 메시지 정의(LLR 갱신식 1,2) | `decoder.cpp` `LDPC_Decoder()` | Sum-product(tanh 기반)는 Prime ECC의 min-sum과 다른 알고리즘이나 반복 구조는 참조 가능 |

> 적용 가치: 낮음 — 특정 부호의 디코더/HW 구현이 아닌, degree distribution 최적화 단계에서 쓰이는 순수 이론적 threshold 근사 도구이며 Prime ECC는 이미 고정된 QC-LDPC 구조를 사용해 직접 적용 지점이 없음.
