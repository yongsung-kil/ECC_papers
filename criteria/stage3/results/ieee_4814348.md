### B. 알고리즘 요약

1. 대상은 AWGN 채널에서 BPSK 변조된 (m,j,k) 정규 QC-LDPC 부호(circulant identity submatrix로 구성된 H), rate `R ≥ 1-j/k`, partly-parallel 디코더 아키텍처 기준 메모리 `jkm` unit 필요.
2. 기존 SP(sum-product) 알고리즘은 메모리 소모가 크고, turbo/shuffled 디코딩은 메모리는 절감하지만 서브행렬 개수 `J`(또는 `k`)만큼 직렬 sub-iteration이 생겨 처리량이 제한됨.
3. 핵심 가정: H를 row-wise로 정확히 2개 서브행렬 `H=[H1^T H2^T]^T`로 분할하며, `H1`은 각 열에 1이 정확히 1개(turbo와 동일한 메모리 절감 제약).
4. 핵심 기법(TSP): 매 iteration을 2개 sub-iteration으로 고정 - `V→C1, C1→V, V→C2` 후 `C2→V`. `H1`으로 정의된 supercode `C1`은 `C2`로부터만 정보를 받고, `C1`의 새 정보가 즉시 `C2`로 전달되어 저장할 필요가 없어 `km` unit 메모리 절감.
5. CNU 갱신식 `r_{l,i}^{(n)} = Π sgn(q) · Ψ^{-1}(Σ Ψ(|q|))`, `Ψ(x)=ln((e^x+1)/(e^x-1))`(표준 SP check node update이나 스케줄만 2단 turbo형으로 변경).
6. SSP(shuffled 버전)는 H를 column-wise 2개 그룹으로 분할, `jm` unit 메모리 절감, 각 iteration이 `C→V1, V1→C, C→V2` 후 `V2→C`로 진행.
7. 일반 LDPC 부호에도 확장 가능: row permutation으로 `H1`이 열당 1을 최대한 갖도록 재배열 후 2분할(turbo/shuffled는 `J`/`G`가 커야 해 일반 LDPC엔 부적합하나 TSP/SSP는 2분할만 필요).
8. 별도 학습/최적화 절차 없음(고정 스케줄), EXIT chart로 SP/turbo/shuffled/TSP/SSP 수렴 거동을 비교.
9. 결과: (211,3,5)/(421,4,7)/(281,5,8) SFT 부호에서 TSP는 SP 대비 평균 iteration 23~33% 감소(turbo는 46~48% 감소이나 sub-iteration 수가 많아 처리량 열위), TSP throughput은 turbo 대비 20~72% 높음(가정: 동일 clock period); TSP/SSP 모두 SP 대비 BER 성능 개선.
10. 한계: 회로 구현/합성 없이 알고리즘·시뮬레이션·EXIT 분석 수준이며, threshold(waterfall 점근 성능)는 SP와 동일(개선 없음), 실제 clock period 차이는 미반영 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TSP/SSP 2-분할 스케줄링(메시지 패싱 순서 변경) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 스케줄 변경만으로 수렴속도 개선 가능해 이식 난이도 낮음 |
| Dual-Update(even/odd 교대 스케줄) 옵션과의 비교 | `decoder.cpp` (Dual-Update 스케줄 로직) | 이미 보유한 스케줄링 기법과 개념적 유사성 있어 중복 기여 여부 확인 필요 |
| CNU/VNU 갱신식(표준 SP, tanh 기반 `Ψ`) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min-sum 근사 사용, 논문의 full-tanh SP식 자체는 이식 시 근사 필요 |
| 메모리 절감(H 분할로 extrinsic info 저장량 축소) | `decoder.h` `Get_VNU_*`, LLR 저장 구조 | HW 메모리 라우팅 관점에서 참고 가능 |

> 적용 가치: **중간** — 스케줄링 아이디어 자체는 이식 난이도가 낮으나 Prime ECC가 이미 Dual-Update 스케줄과 min-sum 근사를 보유하고 있어 중복 기여 여부를 먼저 확인해야 함.
