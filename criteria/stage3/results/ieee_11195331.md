### B. 알고리즘 요약 (10줄 내외)

1. AWGN(BPSK) 채널, time-invariant SC-LDPC 코드(rate 3/4, 2/3) 대상, sliding window decoding(SWD)으로 긴 codeword를 부분적으로 반복 BP 복호.
2. 풀려는 문제: SWD에서 한 윈도우의 복호 오류가 코드 메모리(`ms`)로 인해 이후 윈도우들로 전파(error propagation)되는 문제, 기존 LLR 평균 임계값(`η`) 기반 검출은 임계값 튜닝이 어렵고 계산 오버헤드가 큼.
3. 핵심 가정: 윈도우 내 parity-check 방정식이 최대 반복횟수(`Imax`)까지 만족되지 않으면(즉 수렴 실패) 오류전파 가능성이 높다는 side-channel적 관찰.
4. 핵심 기법(Algorithm 2): 반복 `I`가 임계값 `ι`(=`Imax`로 설정)에 도달하고 parity-check 미충족 시 `flag_iter`를 세우고, 다음 윈도우에서 read-only 심볼의 LLR을 clipping 값(±L)으로 고정하는 variable-node doping을 적용.
5. 핵심 아이디어: 별도 LLR 임계값 튜닝 없이 "이미 알고 있는" `Imax` 도달 여부만으로 트리거하므로 추가 비교연산·복잡도가 거의 없음(기존 SWD 대비 미미한 오버헤드).
6. 검증 방법: confusion matrix(TPR/FPR/TNR/FNR/Accuracy)로 LLR 기반(η=1,10,50) 대비 제안 기법(iteration 기반)의 오류전파 검출 정확도 비교.
7. 부수 트릭: LLR 클리핑(`±L`)을 이용한 variable-node doping을 완화기법으로 채택했으나 check-node doping, window-size 조정 등 다른 완화기법과도 결합 가능.
8. 결과(수치): Code CA(rate 3/4, girth 6)에서 제안기법 Accuracy 0.9976(vs LLR η=10의 0.9901), Code CB(rate 2/3, `ms`=12)에서 Accuracy 0.9998(vs LLR η=10의 0.9994). BER 곡선(Fig.3,4)에서도 제안 기법이 no-doping/systematic-doping 대비 최선의 오류율.
9. 학습/최적화 절차 없음(사전 정의 threshold=Imax 사용, 별도 튜닝 불필요가 장점).
10. 한계: 낮은 Eb/N0에서는 rate loss가 커질 수 있음, TI(time-invariant) 코드 위주 검증(time-varying 코드에도 적용 가능하다고만 언급, 상세 결과 없음), HW 구현/게이트 비용 분석 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 반복횟수(iteration count) 기반 오류전파/수렴실패 트리거(Algorithm 2) | `decoder.cpp` `LDPC_Decoder()` | 반복 루프 내 조기 이상탐지 아이디어로 디코더 이터레이션 루프에 개념적으로 대응 가능 |
| parity-check 미충족 시 조기 플래그(수렴 판정) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료(CRC) 모듈과 유사한 "수렴 판정" 개념이나 본 논문은 오류전파 완화(doping)가 목적이라 조기종료와는 반대 방향 |
| variable-node doping(LLR clipping)을 통한 오류전파 완화 | 대응 없음 | SC-LDPC 특유의 sliding window/코드 메모리(`ms`) 구조 기반 기법으로 block code(Prime ECC)에는 직접 대응 없음 |
```

적용 가치: `중간` — "반복횟수 소진=이상 신호"라는 저복잡도 검출 아이디어는 개념적으로 참고 가치가 있으나, 본 기법 자체(SC-LDPC doping)는 block code 구조인 Prime ECC와 직접 대응되지 않음.
