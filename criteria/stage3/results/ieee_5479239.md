> 총평: 표준 (3,6) Min-Sum LDPC 디코더의 Impulse C 기반 FPGA 구현 소개(툴플로우 논문), 알고리즘/부호 신규성 없어 Prime ECC 이식 가치 없음.

### B. 알고리즘 요약

1. 대상: rate 0.5, block 2500-bit `(3,6)-regular` LDPC 코드, 125×250 sparse parity-check matrix (본문엔 125*250 기재).
2. 문제: 전통적 HDL 대비 더 빠르고 편한 LDPC 디코더 HW 설계 방법을 원함 → Impulse C(C→hardware) 사용.
3. 알고리즘: 표준 `MSA`(Min-Sum Algorithm)를 채택 — VN↔CN 메시지 전달, look-up table·복잡연산 회피로 HW 친화.
4. 설계흐름: Impulse C의 communicating sequential process 모델(Producer/DeProcessor/Consumer stream)로 기술 → C analysis, loop unrolling, C-to-HW compiler로 VHDL top/bottom 모듈 자동 생성.
5. HW 구조: 디코더는 총 195-state FSM으로 구현; 데이터는 250-bit 블록을 10회 read; `CO PIPELINE`으로 loop pipelining.
6. 구현결과: Xilinx Virtex2 XC2V2000, 50MHz, 최대 10 iteration에서 throughput 10Mbps, power 367mW, 5493 LUT 사용.
7. 비교: 전통 HDL 디코더 대비 logic cell 수·HW 규모가 상대적으로 작다고 정성 주장(수치 대조는 미상).
8. 응용 관점: 3G 이동통신 등 최대 10Mbps면 충분한 시스템 타깃이라 낮은 throughput 정당화.
9. 결과 요지: C 언어로 embedded HW를 빠르게 프로토타이핑하는 설계 방법론 제시가 핵심.
10. 한계: LDPC 알고리즘/부호설계 신규성 전무(표준 MSA·표준 (3,6)코드); BER 성능·정정능력 데이터 없음, throughput 매우 낮음.
