### B. 알고리즘 요약

1. 대상은 fully-parallel **stochastic LDPC 디코더**로, 채널 확률 `Pch`를 Bernoulli 시퀀스(N비트 중 `1`의 수 `m = Pch × N`)로 변환해 bit-serial 반복 처리한다.
2. 문제: 기존 stochastic 디코더는 변수노드가 0 또는 random 상태로 시작해 수렴이 느리고 latency가 크며, 특히 짧은 코드에서 성능이 나쁘다.
3. check node(dc=3) 출력확률 `Pc = Pa(1−Pb)+Pb(1−Pa)`, variable node는 `Pc = PaPb / (PaPb+(1−Pa)(1−Pb))` 형태로 계산.
4. 입력이 불일치(hold/disagreement state)하면 ASMBG(advanced stochastic method bit generation, 여기선 Delayed Stochastic DS)로 출력 비트를 생성한다.
5. 핵심 기법 **CSS(Controlled Start-up Stochastic)**: 첫 decoding cycle에서만 변수노드를 채널확률 1비트 코딩값으로 초기화 — `if Pch ≥ 0.5 then Pc(1)=1 else 0`.
6. HW 구현: 변수노드 processor 출력과 `Pch`의 MSB 사이에 **2-to-1 MUX** 하나만 추가, 첫 cycle에 `Pch(k-1)` 전달·이후 processor 출력 전달(식 5,6).
7. 이로써 (2048,1723) 코드 기준 Eb/N0=5dB에서 변수노드의 98%가 올바른 비트로 시작해 초기 random 상태를 회피한다.
8. 검증: (48,24) LDPC를 Xilinx Virtex-6 VLX240T FPGA에 CSS 유·무로 구현, ASMBG는 DS 방식 사용.
9. 결과: Eb/N0=4.5dB에서 BER 100배 개선·디코딩 속도 7배, 평균 iteration은 Eb/N0=5dB에서 약 1.06 DC. (48,24) CSS가 (2048,1723) 6bit OMSA를 3.9dB 이하, 6bit SPA를 4.3dB 이하에서 상회.
10. 한계: stochastic bit-serial 패러다임, 짧은 코드만 실증, 높은 SNR에서는 긴 코드 필요, gate count·throughput 수치 미제시.
