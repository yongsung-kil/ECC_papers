### B. 알고리즘 요약

1. 시스템: MLC(2bit/cell, 4상태) NAND 채널, rate-0.90 (4544,4096) QC-LDPC + BP 복호(max iter 25), 32-level 센싱(상태당 8-level).
2. 문제: cell-to-cell interference(CCI)가 threshold voltage를 왜곡, 기존 검출은 정확도 위해 다중 센싱 필요 → detection latency 급증, 사후보상은 overcompensation 발생.
3. 채널 모델: erase/program 상태 Gaussian 분포, CCI 강도 `I=ΔVn·γn` (수직 `γy`, 대각 `γxy`), 추가로 retention/RTN/read-disturb를 zero-mean Gaussian `z`로 근사.
4. CSAD 핵심: threshold voltage 분포를 상태별 sub-window로 나누고 각 상태의 실제 분포에 맞춰 sensing reference를 비균일 배치.
5. 핵심 식: `ΔVp = (Vref|S| − Vref|S|−1)/wp` (상태 교차영역 기준 reference 간격), 셀 센싱값은 인접 reference 중간값 `(Vref_k+Vref_{k+1})/2`로 근사.
6. NIA-CSAD 확장: 순차 read 시나리오에서 이미 성공 복호된 (i+1)-WL 셀의 복호어(사후정보) `V˜`로 i-WL의 CCI를 재추정.
7. CCI 상쇄: `Ī(i,j)=γxy(V˜−Ve)+γy(V˜−Ve)+γxy(V˜−Ve)` 후 `V→(i,j)=V(i,j)−Ī(i,j)`로 overcompensation 억제.
8. LLR 계산: Gaussian Kernel density `fp(V)`로 `L(bi)=log(...)` 산출 후 BP 복호 수행.
9. 결과: BER=1e-5에서 Post-Comp 대비 CSAD +0.8dB, NIA-CSAD +2.0dB 이득; 복잡도는 9N add·4N mul·3N read(Post-comp 6N/3N/2N 대비 소폭 증가).
10. 한계: HW 미설계·시뮬만, 순차 read 및 이웃 WL 성공복호 가정 의존, LDPC 디코더/부호 자체 개선 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 비균일 센싱 양자화 / read reference 조정 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold·offset 설정부와 개념적 대응 (양자화 레벨 재배치) |
| Gaussian Kernel 기반 LLR 산출 | `channel.cpp` + LLR 테이블 (`PARAM_LLR`) | 초기 LLR 생성 단계에 대응, 다만 논문은 32-level(soft-4bit+)로 Prime ECC 지원범위 초과 |
| 이웃 WL 사후정보 CCI 상쇄 (NIA) | 대응 없음 | 컨트롤러 read 스케줄/채널 전처리 레이어, 디코더 코어 밖 |
| BP 복호 (벤치마크용) | `decoder.cpp` `LDPC_Decoder()` | 논문은 표준 BP를 그대로 사용, 개선 없음 |

적용 가치: **낮음** — NAND 직접 타깃이나 개선 지점이 채널 검출/센싱 양자화(전처리)에 국한되고 HW 미설계·soft-4bit+ 요구로 Prime ECC 디코더/부호 코어에 떼어 쓸 요소가 적다.
