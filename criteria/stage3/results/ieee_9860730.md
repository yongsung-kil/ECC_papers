### B. 알고리즘 요약 (10줄 내외)

1. 대상: IEEE 802.11n WLAN, `(n=648, k=540)`, `R=0.833`, 4bit 양자화, 5회 언롤 iteration, flooding 스케줄, 28nm CMOS.
2. 문제: 언롤드 LDPC 디코더의 최고 throughput 추구 — 알고리즘이 아니라 순수 HW(RTL/합성/floorplan) 최적화로 임계경로 단축이 목표.
3. 기법: 표준 offset min-sum 디코딩만 사용(패리티 행렬·복호 스킴 미변경). 모든 개선은 RTL 코드 레벨.
4. Step1(언롤 아키텍처): iteration 간 교환 신호 전부에 추가 flip-flop 삽입 → floorplan 툴이 타이밍 여유로 각 iteration 디코더를 물리 배치 유연화(합성단엔 면적만 증가, floorplan 후 클록 이득).
5. Step2(7단 파이프라인): 기존 2~4단 대비 단일 iteration을 7-stage 파이프라인화, 7 frame 동시 처리, iteration latency 7 cycle.
6. 노드 처리: `Var. to check` 계산을 2 clock으로 분할(U2 negate 후 soft 값 가산), sign-magnitude 변환 후 min-search, (648,576)에서 CN당 22값 이진 비교트리 `ceil(log2 22)=5` 단계로 2 clock에 최소값 탐색.
7. 최적화 방법론: floorplan 툴의 타이밍 리포트로 임계경로를 정밀 추정 후 FF 위치 결정(수백 시간 합성/floorplan 반복). core 크기를 10→7→반복적 0.5mm^2 축소.
8. BER: 참조는 Matlab float SPA(50 iter). 4bit·5iter가 면적/BER 절충; 양자화 3~6bit, iteration 4~8 스윕 곡선 제시(신규 알고리즘 이득 없음).
9. 결과: 1218 Gbps coded(1015 Gbps info), 5.49mm^2, fmax 1880MHz(worst-case), latency 19.68ns, area efficiency 222 Gbps/mm^2, energy 12.74 pJ/b@4dB — [8] 대비 throughput 약 3배.
10. 한계: 알고리즘 신규성 없음, 완전 언롤(iteration HW 5중 복제)로 면적·전력 최적화 미고려, 특정 통신 코드·짧은 길이 고정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 표준 offset min-sum CN/VN 연산 | decoder.cpp LDPC_Decoder(), CNU_Update_New_Mag() | 우리가 이미 보유한 min-sum과 동일 알고리즘 (중복, 이득 없음) |
| min-search 비교트리 / sign-magnitude | decoder.cpp C2V_Cal_New_Sgn() | 표준 min1/min2 sign XOR와 동일 개념 |
| 7단 파이프라인 / 언롤 HW 아키텍처 | 대응 없음 | 우리는 부분병렬 6-stage pipeline(z=32), 완전 언롤/flooding 아님 — 구조 이질 |
| floorplan 기반 FF 삽입 타이밍 최적화 | 대응 없음 | RTL/물리설계 방법론이라 C++ bit-exact 시뮬과 무관 |
```

적용 가치: **낮음** — 알고리즘은 우리가 이미 보유한 표준 min-sum이고, 기여의 본질이 완전 언롤드 통신 디코더의 RTL/floorplan 파이프라인 최적화라 NAND용 부분병렬 QC-LDPC bit-exact 시뮬(Prime ECC)에 떼어낼 요소가 없다.
