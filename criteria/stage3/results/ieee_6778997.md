### B. 알고리즘 요약

1. 시스템: QC-LDPC, base matrix `Hb=[Hb1|Hb2]`를 확장인자 `z`로 lifting, BPSK/AWGN, LLR BP 복호 20 iteration. rate 1/7~5/8, length 42~2304.
2. 문제: 기존 QC-LDPC(802.16e girth6, ACE 등)는 length-4 cycle 제거에 computer search가 필요하고 code length/rate 유연성이 낮으며, dual-diagonal의 degree-2 VN이 error-floor를 높임.
3. 핵심 기법 1단: Slide Rectangular Window(SRW) — 산술수열 `hij = hi1+(i-1)(ct+(j-1))`로 정보부 `Hb1`을 구성해 length-4 cycle을 검색 없이 제거, `z ≥ (mb-1)rt+(kb-1)(ct+mb-1)` 조건 만족.
4. 유연성: `mb`, `kb`만 조정하면 행/열 삭제만으로 rate/length가 재구성 없이 바뀜(최소길이 42, step 4).
5. 부수 기법: optimal diagonal method — `Hb1`의 특정 대각 원소를 -1(zero matrix)로 치환해 regular→irregular degree 분포로 만들어 BER 개선(degree>6에서 특히 효과).
6. 핵심 기법 2단: quasi tri-diagonal 패리티부 `Hb2` — 세 번째 대각선 위치 `r(1)`(≥3행)를 유연 조정해 degree-2 VN을 부분 제거, error-floor 완화(dual/기존 tri-diagonal 대비 유연).
7. 인코딩: 세그먼트 길이 `z`로 `HcT=0`에서 `p_i`를 순차 계산하는 fast encoding — dual-diagonal 대비 `p1` 계산량(곱셈 nz/2 vs n²/4)이 작아 인코딩 복잡도 감소.
8. 결과: BER 1e-4 도달 SNR이 tri-diagonal 2.05dB vs dual-diagonal 2.4dB(코딩게인 ~0.35dB), 길이 클수록 이득 증가; 길이 960/1152에서 802.16e에 근접, 고SNR·장코드에서는 초과.
9. 비교: 짧은 길이 576에서는 SRW-tri가 802.16e보다 BER 손실.
10. 한계: HW 미설계, AWGN 시뮬만, 무선(adaptive transmission) 타깃, error-floor 정량 floor 수치·gate count 미제시.
