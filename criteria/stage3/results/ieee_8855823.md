### B. 알고리즘 요약

1. 대상: BPSK/AWGN 채널의 binary irregular LDPC(IEEE 802.16e, (576,432)/(1152,864), rate 0.25), 최대 10 iteration.
2. 문제: MSA는 CN 메시지를 과대추정하고, 2bit 저해상 양자화에서 clipping이 early error-floor를 유발.
3. 모델: VN 메시지에 clipping `clip_γ(l)=sgn(l)·min(|l|,γ)`와 균일 양자화 `Q_{q,γ}`(스텝 `Δ=2γ/(2^q-1)`) 동시 적용.
4. 핵심 기법: per-iteration 적응형 clipping 임계 - hard-decision 후 미충족 체크 비율 `(1/M)·H·x̂^T`을 계산해 구간별로 `γ1/γ2/γ3` 선택(식 14).
5. 핵심 식 의미: unsatisfied check 수가 코드워드 오류 수와 상관되므로, BER이 낮아질수록(오류↓) 더 큰 clipping 임계를 써 clipping 왜곡·양자화 잡음을 줄임.
6. 확장: clipping 인자 개수를 3→4개로 늘리면 error-floor를 추가로 낮춤(복잡도 미미).
7. 구현 디테일: CN 메시지도 q-bit라 CN 양자화는 생략, HW 친화 위해 규칙을 단순 임계 비교로 설계, 스케일링 인자 시퀀스 저장 불필요(clipping 인자 3~4개만).
8. 학습/최적화: 없음 - clipping 임계 `[γ1,γ2,γ3]`와 결정 임계 `[τ1,τ2]=[10,20]`는 부호별 수동 설정.
9. 결과: 2bit에서 code I는 최적 고정 clipping 대비 BER 10^-7에서 ~0.2dB, code II는 10^-6에서 ~0.5dB 개선, 무한정밀 SPA에 근접.
10. 한계: HW 미설계(gatecount/throughput 없음), AWGN 시뮬만, 임계 부호별 튜닝, NAND read/soft-info 비용 미고려.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum 연산(min1/sign) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 높음 - 논문 MSA CN 처리와 동일 구조, 이미 보유 |
| VN 메시지 clipping+양자화 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | 높음 - clipping은 VN magnitude 양자화 테이블에 흡수 가능 |
| per-iteration 적응형 clipping 임계 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 중 - 논문은 unsatisfied-check로 임계 선택, Prime은 iteration 구간별 LLR 테이블(선택 소스가 다름) |
| unsatisfied-check(신드롬) 카운트 | partial_CRC.cpp partial_crc_HW_T4() (조기종료용) | 중 - 신드롬 계산은 있으나 clipping 임계 선택엔 미사용 |
```

적용 가치: **중간** - min-sum VN 양자화/clipping은 Prime ECC의 LLR 테이블 구조에 흡수 가능하나, Prime은 이미 iteration 구간별 적응형 LLR을 보유(중복)하고 논문 기법은 unsatisfied-check 기반이라 선택 소스만 다르다. error-floor 개선 아이디어(임계를 iteration/오류량으로 키움)는 참고 가치 있음.
