### B. 알고리즘 요약

1. 대상은 유한체 `Fq`의 두 부분집합 `S1,S2`로 만든 binary QC-LDPC(CPM-dispersion), (3,6)/(4,8)-regular, 예제 `(186,100)`/`(1016,508~511)`, `rate≈1/2`, AWGN·NMSA-0.75 복호.
2. 문제: 두 부분집합을 어떻게 고르면 성능이 좋아지는지 불명확하고, exponent matrix·lifting 크기 증가 시 탐색 복잡도가 폭증.
3. 모델: exponent matrix `P=[p_{k,t}]`가 Tanner 그래프의 girth·cycle distribution·min distance를 결정하며, 동형(isomorphic) 부호는 이 구조특성이 동일 → 성능도 유사.
4. 핵심기법: Case1 `M(η)=[ηα^{ik}+α^{jt}]`(식3, lifting `q-1`), Case2 prime field `P=[ik·jt mod q]`(식5, lifting `q`) 두 구성으로 유한체 기반 QC-LDPC 생성.
5. 식 의미: 그래프 동형사상(permutation/addition/multiplicity rule)으로 동형 부호를 정규형(식6/7)으로 환원 → 비동형(non-isomorphic) 대표만 남겨 탐색공간 축소.
6. 확장: Prop.1/2로 같은 `P`를 만드는 pair 수(`q`, `q-1`)와 비동형 개수 `N1*`,`N2*`를 유도해 단순 exhaustive search 설계(식8 `d*` 최소화).
7. 트릭: 비동형 부호들의 cycle distribution `C*(x)`를 열거해 girth 최대·단주기 최소 부호 선택, 이후 masking(-1 삽입)으로 (4,8)→(3,6) 부호 파생.
8. 학습/최적화: 없음(대수적 구성 + 그래프 탐색). NMSA-0.75, 50 iteration 시뮬로 검증.
9. 결과: Ceg1이 임의집합 Carb1 대비 0.65dB, PEG 부호 대비 0.4dB(BER 1e-7) 우수; Ceg2,mask가 WiMAX·5G 표준 부호보다 낮은 error-floor·빠른 수렴(BLER 1e-4~1e-5 이하).
10. 한계: HW 미설계·시뮬만, 짧은 부호(rate≈1/2)만, gatecount/throughput 없음, 부호구성법이라 특정 코드베이스 H-matrix 교체가 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 유한체 기반 QC-LDPC exponent matrix 구성 | ecc_top.cpp Load_PCM() | H-matrix/부호구조 로드에 대응하나 Prime ECC는 고정 QC-LDPC라 교체 부담 큼 |
| CPM-dispersion / base+shift 구조 | ecc_data.h PCM_b (base+shift) | QC-LDPC base+circulant 표현과 개념 일치 |
| masking으로 부호 PCM 파생 | encoder.cpp Generate_PCM_encoding() | 부호용 PCM 생성 흐름과 접점 (구성법은 상이) |
| girth/cycle 최적화 → error-floor 개선 | 대응 없음 | Prime ECC에 부호설계 탐색 로직 없음(H-matrix 고정) |
```

적용 가치: **중간** — girth↑·error-floor↓의 고rate QC-LDPC 부호설계 방향은 관심영역이나, Prime ECC는 특정 QC-LDPC를 고정 사용하고 부호 교체 시 디코더·인코더 재검증 부담이 크며, 본 논문 예제는 짧은 rate≈1/2 부호라 NAND 고rate(~0.9)로의 직접 적용에 재설계·재검증이 필요.
