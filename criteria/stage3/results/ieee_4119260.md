### B. 알고리즘 요약

1. 짧은 블록길이(n<10^3) LDPC 코드의 error floor를 낮추기 위한 code annealing(CA) 구성법과, 이를 cyclic lifting(CL)과 결합한 유한코드 앙상블 분석.
2. 기존 PEG 등 무작위 탐색 구성법은 girth/extrinsic cycle degree 등 간접 지표를 최적화해 짧은 코드에서 error floor 보장이 약함.
3. binary erasure channel(BEC) 가정, error floor는 최소 stopping set 크기 `D_stp`(stopping distance)와 그 개수 `M_s`로 결정(`M_s × ε^D_stp`가 asymptote).
4. CA 알고리즘: 임의의 두 edge를 swap하여 `(D_stp, M_s)`가 개선되면 채택, 아니면 롤백 — girth 대신 `D_stp`를 직접 목적함수로 사용하는 최초의 효율적 구성법.
5. cyclic lifting의 stopping set 생존확률 `E{|S_L,1|} = O(K^-(0.5#E-#V+0.5#C_odd))`, suppressing weight `W_sup = 0.5#E - #V + 0.5#C_odd`가 클수록 lifting 후 error floor가 낮아짐(Theorem 1, Proposition 1).
6. degree-2 variable node가 많은 base code는 `W_sup=0`이 되어 lifting 효과가 없음 → degree augmentation(DA)으로 임시 보조 노드쌍을 추가해 CA 수행 후 제거하는 방법 도입.
7. 구현 디테일: edge swap trial은 병렬 계산 가능(sequential 불필요), stopping set exhaustion 알고리즘([11] 인용)으로 `D_stp, M_s` 계산.
8. 최적화 절차 자체가 본 알고리즘(CA는 일종의 local search/greedy edge-rearrangement, 정식 학습이 아님).
9. 결과: n=128 regular(3,6) 코드에서 K=32 lifting 시 FER 6~4자릿수(10^6~10^4배) 개선, n=576 irregular 코드(62.5% degree-2)에서 FER 4×10^-6 @ ε=0.3.
10. 한계: BEC 전용 분석(AWGN/일반 채널의 trapping set 확장은 "현재 연구 중"으로만 언급), HW 미설계, 시뮬레이션(이론+수치)만 제공.
