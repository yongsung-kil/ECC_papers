### B. 알고리즘 요약

1. 대상은 짧은/중간 길이 CRC-LDPC 코드([96,37+11], [255,159+16])이며 BPSK/AWGN 채널을 가정, URLLC 등 고신뢰성 요구 상황을 겨냥.
2. 기존 BP는 코드 구조 내 메시지 의존성으로 짧은 코드에서 성능 한계가 있고, OSD는 ML에 근접하나 코드길이·비율이 나빠질수록 복잡도가 급증하는 문제가 있음.
3. 핵심 가정: BP 오류(`E_B`)는 주로 trapping/stopping set에 기인하고 PEPOSD 오류(`E_P`)는 사전설정된 error pattern(EP) 범위에 정답이 없거나 ML 오류인 경우로, 두 오류의 원인이 달라 교집합 `C=E_B∩E_P` 비율이 SNR 증가에 따라 급감(SNR 2.0dB에서 23%→5.0dB에서 1.6%).
4. 핵심 기법: BP를 먼저 시도해 실패(Condition 1: `t≥t_max` 또는 Condition 2: hard decision이 `b_max`회 연속 불변인 trapping)하면 PEPOSD로 전환하는 2단 하이브리드 디코더(Algorithm 1).
5. PEPOSD는 신뢰도 기반 permutation `λ1`, 선형독립 systematic bit 선택 `λ2`로 벡터/생성행렬을 재정렬 후, index weight(IW)·Hamming weight(HW)로 미리 구성된 EP들을 순차 테스트해 CRC를 통과하는 결과를 큐(길이 `δ`)에 저장, 최종적으로 Euclidean distance 계열 metric `ω=Σĉ_i·y_i`가 최대인 결과를 선택.
6. 확장: "switch plan"(수신벡터 그대로 PEPOSD 입력)과 "soft plan"(BP 마지막 반복의 soft value를 PEPOSD 입력)의 2가지 결합 방식 제안 — switch는 고SNR, soft는 저SNR에서 우수.
7. 부수 트릭: trapping iteration 조건(Condition 2)으로 BP의 무의미한 반복을 조기 중단해 불필요 연산을 skip.
8. 별도 학습/최적화 절차는 없음(EP는 IW/HW 기준으로 사전 구성, OSD 계열 표준 방식).
9. 결과: [96,37+11] 코드에서 BP(50)-PEPOSD(δ=1)가 BP 대비 1.2dB, δ=2가 1.5dB 이득, PEPOSD 단독 대비 고SNR에서 0.3~0.6dB 추가 이득이면서 평균 디코딩 시간은 PEPOSD 단독보다 크게 감소; [255,159+16] 코드에서는 고SNR에서 BP 대비 0.1~0.3dB 이득.
10. 한계: HW 구현·게이트카운트 없음(순수 시뮬), 짧은 CRC-LDPC 코드(최대 255bit)에서만 검증되었고 PEPOSD의 Gaussian elimination/permutation 비용은 코드가 길어질수록 감소하지 않는 구조적 한계가 있다고 저자도 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 실패 시 조건부 전환(trapping/max-iter) | `decoder.cpp` `LDPC_Decoder()` | 반복 루프의 종료 조건(조기종료) 로직과 유사한 위치이나, 전환 대상이 별도 OSD 모듈이라 Prime ECC엔 대응 모듈 없음 |
| trapping iteration 조기중단(hard decision 불변 검출) | `partial_CRC.cpp` `partial_crc_HW_T4()` | CRC 기반 조기종료 개념과 유사하나, hard decision 불변 검출은 별도 로직 추가 필요 |
| PEPOSD(OSD 계열 post-processing) | 대응 없음 | Prime ECC는 min-sum 기반 message-passing 디코더만 보유, OSD/Gaussian elimination 기반 post-processing 모듈 없음 |
| soft value 전달(BP→OSD) | 대응 없음 | Prime ECC 디코더는 min-sum LLR을 외부 post-processor로 넘기는 인터페이스가 없음 |

> 적용 가치: 낮음 | BP 실패 시 error-floor 개선을 위한 post-processing 전환 아이디어 자체는 유효하나, PEPOSD가 OSD 계열(Gaussian elimination, permutation, EP 큐 관리)로 Prime ECC의 min-sum HW 아키텍처와 이질적이며 짧은 코드 전제라 NAND 긴 QC-LDPC로 이식 시 복잡도·검증 부담이 큼.
