### B. 알고리즘 요약

1. CV-QKD 정보재조정(information reconciliation)에 쓰이는 저rate(`R=0.01`, `R=0.1`) protograph 기반 TBP-LDPC 코드를 AWGN 채널에서 500회 반복 복호.
2. MET-LDPC는 대형 block length·고iteration이 필요해 처리량이 낮고, 표준 scaled MSA는 저용량(저SNR) 양자채널에서 SPA 대비 성능 열화가 크다는 문제를 해결하려 함.
3. box-plus 연산의 SPA CN update `L1⊞L2 = α1α2[min(β1,β2) + s(β1,β2)]`에서 보정항 `s(β1,β2)`를 Taylor 전개·기하급수 근사로 유도해 `α1α2·min(β1,β2)·tanh(max(β1,β2)/2)` 형태의 scaled MSA로 재표현.
4. 제안 기법(ID-MSA)은 CN update `Lti = ∏sign(Lk)·βm·cti(βm′)`에서 두 번째로 작은 LLR `βm′`에 조건부인 iteration-dependent 스케일링 계수 `cti(βm′)`를 사용.
5. `cti(βm′)`는 EXIT 챠트 분석으로 얻은 edge별 LLR 분포 `fℓt(x)`의 조건부 기댓값으로 계산, iteration·edge-type(11종 또는 8종)별 LUT를 사전 구성.
6. LUT 크기 축소를 위해 K-means로 압축하여 최종 2600개 entry LUT 사용 (`βm′`는 32단계 양자화).
7. 복잡도 측면에서 기존 3-LLR 기반 lambda-min 방식(테이블 lookup 2회) 대비 ID-MSA는 lookup 1회로 계산량이 더 적음.
8. 결과: R=0.01 코드에서 SPA 대비 MSA는 5.2dB 열화이나 ID-MSA는 0.059dB 갭까지 축소(FER=0.1 기준), R=0.1 코드에서도 0.068dB 갭.
9. 한계: 시뮬레이션(이론적 EXIT 분석 + BER 곡선)뿐이며 HW 구현·gatecount·throughput 언급 없음. 초저rate 특수 코드(TBP-LDPC) 대상으로 일반 NAND ECC 대비 rate/channel 격차 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| scaled min-sum CN update 보정(iteration-dependent scaling LUT) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 스케일링 아이디어는 CN 연산부에 대응되나, 계수 유도가 초저rate protograph EXIT 분석 기반이라 재설계 필요 |
| iteration별 LUT 개념 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 이미 iteration 구간별 LLR threshold 테이블 보유(§3), 개념적으로 중복 |
| 채널/soft LLR 분포 가정(AWGN 저SNR) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | NAND RBER 채널과 통계적 성격 상이 |

적용 가치: **낮음** — 스케일링 계수 유도 아이디어는 흥미로우나 초저rate MET/TBP-LDPC·양자채널 전제이며, Prime ECC는 이미 유사한 iteration별 LLR 테이블 기법을 보유하고 있어 중복도가 높음.
