### B. 알고리즘 요약

1. 대상 문제: ADMM-based LP 복호에서 각 체크노드·매 iteration마다 필요한 parity polytope `P_d,even`로의 유클리드 사영 `z = Π_{P_d,even}(x)` (`d`=체크 degree).
2. 기존 방식은 사영을 위해 정렬(sort)이 필수였고, 정렬이 HW 구현의 주 병목이 됨 → 정렬 없는 알고리즘이 필요.
3. 핵심 모델: even/odd 두 parity polytope를 함께 정의하고, cut-search algorithm(Zhang-Siegel)으로 위반된 forbidden-set 부등식 `θ^T w ≤ p` 하나를 찾음(최대 1개만 위반됨).
4. 사영이 폴리토프 밖이면 초평면 `θ^T w = p`로의 직교사영 `v = x - ((θ^T x - p)/d)θ`를 계산.
5. 핵심 정리(Thm 3): `v_i > 1 ∧ θ_i=1 → z_i=1`, `v_i < 0 ∧ θ_i=-1 → z_i=0` 으로 성분을 확정 고정 가능(기하학적 모순 논증으로 증명).
6. Thm 5: `v ∉ [0,1]^d`이면 위 조건을 만족하는 성분이 적어도 1개 존재 → 매 iteration 최소 1성분 고정.
7. 재귀 구조: 성분 하나를 0/1로 고정하면 나머지는 `P_{d-1,odd}`/`P_{d-1,even}`로의 저차원 사영 문제로 축소, cut-search 결과(θ,p)를 재사용해 재계산 회피.
8. 구현(Algorithm 1): 확정된 성분을 벡터 앞으로 swap하고 인덱스 벡터 Q로 추적, 문제 크기 `l`이 매번 감소, worst-case `O(d^2)`.
9. 결과: d=2~50, 각 d당 [-5,5]^d 백만 벡터 시뮬. state-of-the-art 대비 low-complexity 연산 최대 36%, 총 연산 최대 37% 감소. d=10/30에서 while-loop 평균 3.7/4.9회.
10. 한계: 순수 연산량 카운트만 제시(실제 HW 미구현), BER/정정성능·부호 스펙 없음, ADMM-LP 복호 프레임 전용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP parity polytope 사영 | 대응 없음 | Prime ECC는 min-sum message-passing 복호이며 LP/ADMM 사영 연산 자체가 없음 |
| 체크노드 처리(사영이 CN 연산 대체) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime ECC CN은 min1/min2+sign, 논문의 폴리토프 사영과 연산 성격이 상이 |
| 반복 복호 루프 | decoder.cpp LDPC_Decoder() | 반복 구조는 유사하나 갱신 규칙(ADMM vs min-sum)이 근본적으로 다름 |
```
