### B. 알고리즘 요약

1. 시스템: AWG 기반 데이터센터 네트워크/PON용 FPGA rate-adaptive QC-LDPC FEC, PAM-2(OOK), rate 0.7/0.75/0.8, block size `b=1129`.
2. 문제: worst-case용 고정부호는 비효율. 채널조건에 따라 code rate를 frame-by-frame 적응시켜 대역·지연·전력 최적화 필요.
3. 핵심 가정: raw BER을 sync correlator 직후 실시간 측정하여 채널상태 추정, 임계 초과 시 rate 전환.
4. 핵심 기법1: 고girth `(J,K)`-regular QC-LDPC를 circulant permutation matrix `I(p_{i,j})`로 구성, CIDS로 shift값 탐색.
5. 핵심 식: Tanner bound상 최소거리는 girth에 지수비례 → high-girth로 waterfall·error-floor 동시 개선.
6. 핵심 기법2: shortening의 early error-floor 문제 회피 위해 low-rate high-girth (3,10 girth-10) 부호를 먼저 설계 후 5 block column 추가로 (3,15) girth-8 rate-0.8 부모부호로 확장.
7. 구현 트릭: encoder는 systematic-circulant generator matrix `G_QC`의 첫 행만 memory 저장, SRAA(shift-register-adder-accumulator)로 인코딩.
8. 디코딩: layered scaled min-sum, scaling factor 0.75, max iteration 12, VNP=sum/CNP=min-finding, APP memory 갱신.
9. 결과: FPGA 실측, error floor 미관측(post-FEC BER 10^-12까지), raw BER 3.7e-2까지 정정, throughput 1.0Gb/s(250MHz clock).
10. 한계: 광통신(AWG DCN) 특화, throughput·gatecount 미보고, 부호 세부(shift값) 미공개, soft 정보 양자화 비트수 미명시.
