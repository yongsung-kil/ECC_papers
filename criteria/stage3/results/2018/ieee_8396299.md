### B. 알고리즘 요약

1. 대상: polar code `P(N,k)`, `G=F⊗n`(`F=[[1,0],[1,1]]`), 실험 `P(2048,1024)`, rate 0.5, 5G 제어채널 채널부호.
2. 문제: SCL은 ML 근접하나 직렬구조로 latency·throughput 불리, hard-output이라 iterative detection 부적합; 기존 BP는 SCL보다 성능 낮음.
3. 모델: encoding factor graph(n+1 stage, stage당 N node) 위 message passing, 채널 LLR `Lch,i`로 마지막 stage 초기화.
4. 핵심기법: factor graph stage 순열 `Π=randperm(n)` → `sj=2^(πj−1)` 로 `n!`개 동일 인코딩 그래프 생성, 각기 다른 loop 구조.
5. 의미: 입력(codeword+noise)마다 최적 그래프가 달라 다중 그래프가 다양성 제공 → iterative 디코딩 성능↑.
6. 확장(BPL): L개 병렬 독립 BP 디코더(각기 다른 permuted graph, cyclic shift 우선) 후 `x̂_BPL=argmin‖y−x̂i‖` 로 최소 유클리드거리 codeword 선택.
7. 트릭: G-matrix 기반 조기종료(`x̂=û·G`), 중간 sorting 불필요(끝에서 1회만), stage-combining으로 latency/메모리 절감.
8. 부호설계: RM-polar hybrid 구성(row weight≤d는 frozen, 나머지는 Bhattacharyya 최소 k개 정보비트)로 최소거리↑, `P_RM(2048,1024,16)`.
9. 결과: BPL(L=32,Nit=200)이 SCL(L=32) 성능 근접, 고SNR(3.2dB)에선 L=5로 SCL(L=32) 수준·1.01×10⁶ PE update, CRC-aided SCL 대비 0.5dB 이내.
10. 한계: HW 미설계, 시뮬만, naive 복잡도 높음(`O(L·N·logN)`), CRC/PC 비호환, polar code 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP message passing 디코딩 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | 개념상 iterative MP이나 polar encoding-graph 기반이라 우리 PCM 기반 min-sum과 구조 상이 |
| permuted factor graph 앙상블 디코딩 | 대응 없음 | LDPC Tanner graph 순열 개념은 코드베이스에 없음 |
| polar code / RM-polar 구성 | 대응 없음 | Prime ECC는 binary QC-LDPC 고정, polar/RM 미지원 |
| 유클리드거리 candidate 선택 / list | 대응 없음 | list 디코딩·후보선택 로직 없음 |

적용 가치: **낮음** — polar code용 병렬 BP list 디코더로 부호종류(polar)와 타깃(5G 무선)이 Prime ECC(NAND binary QC-LDPC)와 근본적으로 달라 직접 이식 불가. permuted-graph 앙상블 다양성 아이디어만 개념적 참고 가치.
