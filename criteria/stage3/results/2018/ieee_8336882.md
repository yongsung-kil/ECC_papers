### B. 알고리즘 요약

1. 시스템/채널: 5G URLLC용 저rate LDPC, BPSK/AWGN, 정보길이 `K=256`(short)~10240, rate `1/3~1/12`, 최대 iteration `I=20`으로 BP 복호.
2. 문제: 기존 저rate protograph(RU/AR4A/QLF)는 rate-compatible 구조 부족 + punctured node로 저iteration 수렴이 느려 URLLC(1ms/BLER 10^-5) 부적합.
3. 핵심 구조: raptor-like protomatrix `P=[[P_HRC,O],[P_IRC,I]]` - 고rate 코어 `P_HRC`(rate-2/3, col-deg 3, row-deg 9)와 저rate 확장 `P_IRC`(M×3).
4. 핵심 아이디어: RU protograph를 일반화(GRU)하되 punctured column을 제거 → 저iteration에서 수렴 가속, HRC edge 밀도도 RU(row당 15) 대비 낮춤(row당 9).
5. 최적화 변수: `P_IRC`의 연결차수 벡터 `a=(a_0..a_{M-1})`, `b=(b_0..b_{N-1})`, 범위 `a∈[1,u]`, `b∈[0,v]`로 탐색공간 제한.
6. 최적화 절차: 행 단위 incremental - 각 행에서 `u(v+1)` 조합 생성, 제한 iteration PEXIT threshold로 평가, 상위 `Q`개 protomatrix 유지(u=3,v=1,Q=10).
7. rate-compatible 재귀 정의 `a_{w+1}=(a_w, ea_w)`로 rate `R_w=1/(3+w)`를 점진 확장, 최저 R_9=1/12.
8. 부호 구성: two-step lifting + 단순화 EMPEG(오류확률 추정 생략, 최대 cycle length·EMD·최소 cycle 수 기준 edge 선택)로 shift 값 확정.
9. 결과: K=256 short length에서 RU 대비 우수, QLF 대비 BLER 10^-4에서 0.2 dB 개선; rate-1/3 GRU는 I=5 iteration으로 BLER 10^-4 달성(QLF/AR4A는 8+ 필요).
10. 한계: 시뮬만(HW/합성/throughput 없음), 저rate 무선 특화, hard/soft read 무관, 저길이 K=256 중심.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph/protomatrix 부호 설계 (lifting) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 로드 지점이나, Prime ECC는 고정 고rate QC-LDPC라 저rate protograph 이식은 재설계·재검증 부담 큼 |
| rate-compatible 저rate 확장 (IRC/diagonal) | 대응 없음 | Prime ECC는 dual-diagonal 인코딩([encoder.cpp] `LDPC_encoder_4KB()`) 고정, rate-compatible 확장 개념 없음 |
| PEXIT / EMPEG 오프라인 최적화 | 대응 없음 | 코드베이스는 bit-exact 시뮬레이터로 부호 최적화 도구 미포함 |

적용 가치: 낮음 - rate 영역(1/12~1/3)과 용도(무선 URLLC)가 NAND 고rate ECC와 정반대이고 HW 미설계라, punctured node 제거 설계 철학의 개념적 참고 외 직접 이식 대상 아님.
