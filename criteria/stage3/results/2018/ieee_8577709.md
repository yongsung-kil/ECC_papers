### B. 알고리즘 요약

1. 대상: 임의 비특이 구조의 QC-LDPC 부호 인코딩. H는 `c×t` 배열의 `b×b` circulant로 구성(예: CDR 시스템 `b=256`, rate `1/4,1/3,1/2,3/4`).
2. 문제: 고속·저복잡도 인코더 필요. G 기반보다 sparse H 기반이 유리하나, 기존 RU 방식은 H를 6개 submatrix로 쪼개 무결성을 훼손.
3. 핵심 아이디어: RU의 5개 작은 sparse submatrix를 하나의 큰 sparse 행렬 `X`(=H에서 far-right diagonal의 항등행렬만 제거한 `H'`)로 대체해 인코딩을 3-phase로 축약.
4. Phase 1: `px=0`으로 두고 back-substitution으로 `q(=py), x` 계산 (식 `(6)`).
5. Phase 2: dense circulant 배열 `Φ`에 대해 vector-dense-matrix 곱셈기로 `px = xΦ^T` 계산 (식 `(4)`).
6. Phase 3: 같은 back-substitution 회로로 `py` 재계산 (`x` 불필요), 최종 parity `p=(px,py)` 확정.
7. Back-substitution 회로: LUT(ROM)에 블록열번호·shift값 저장, RAM에 `v` 저장, barrel shifter로 `si,k`만큼 좌회전 후 Acc1 누산. Phase1/3이 동일 회로 공유.
8. Path latency: barrel shifter가 `τ=⌈log2 b⌉` 파이프라인 단, 목적지 주소 생성기에 `τ` 사이클 지연 보상, `k≥i'` 충돌 시 0 입력 처리.
9. 곱셈기: RLA-IV 변형 - LUT를 circulant 첫 R행으로 확장해 병렬도 R배(CDR에서 `R=4`), `⌈b/R⌉` 사이클 소요.
10. 결과: CDR에서 `[6]` 대비 속도 약 6.7~11.9배, flip-flop/XOR/read-addr-gen은 2.4~4.5% 수준으로 감소. 단 메모리는 7.9배 더 소모. 실측 없이 clock cycle·자원 bit 이론 산정.
