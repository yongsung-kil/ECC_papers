> 총평: layered 스케줄의 데이터 의존성을 Δ-update로 완화해 병렬+파이프라인을 동시 활용하는 아키텍처 아이디어가 핵심. NAND 고rate·z=32 정합과 실제 HW 검증이 미해결이라 개념 참고 수준.

### B. 알고리즘 요약

1. 시스템: 임의 QC-LDPC 지원 고병렬 디코더, protograph 기반 부호(예제 N=2592, M=1296, rate 0.5), BPSK/AWGN, OMS.
2. 문제: 고throughput 3대 전략(sequential schedule로 iteration 절반↓, 병렬 PU, 파이프라인)을 동시 사용 불가 — sequential/layered 스케줄이 layer 간 계산 overlap을 막음.
3. 핵심기법1(아키텍처): `Δ-update` — PU가 갱신된 VN sum(Λ') 대신 차이 `Δ=Λ'−Λ`를 계산해 임의 스케줄에서도 정확성 유지, 노드별로 스케줄 선택 가능 → 병렬+파이프라인 동시 사용.
4. 대가: VN 메모리 위치 고정 필요 → read-side 외 write-side shifter 1개 추가(Δ는 비트폭 작아 소형), 파이프라인 깊이 확대로 상쇄.
5. 메모리 접근: 임의 B-VN subset 병렬 접근 위해 B-VN을 K≥dc,max 뱅크로 분할(뱅크 내 동시접근 불필요 조건), K를 그래프 컬러링(VN-only graph)으로 최소화.
6. 핵심기법2(부호설계): first-lifting PEG를 `modified PEG`로 교체 — 색 조건(color condition, |Lv∪Lc|<K)을 추가해 컬러 수 K를 부호 구성 단계에서 직접 최소화, 메모리 관리 단순화.
7. 스케줄 최적화: T-stage 파이프라인이 동시처리하는 B-CN 쌍의 무시된 의존성 가중치 `wi,j=|Vci∩Vcj|` 합을 최소화하는 B-CN 순서를 randomized greedy로 탐색.
8. 학습/최적화: protograph는 differential evolution으로 threshold 최소화(MS=2,NS=4, Si,j≤3, dc,max=7), two-step lifting(Z1=36, Z2=18)로 girth 8 QC 부호.
9. 결과: 6bit 양자화, offset=1, 25 iter. C8/C9 유사 성능(C7은 girth 4로 열화). C8을 T=20 파이프라인으로 latency 3.2배(=20/5·20/25) 감소, WiFi 802.11n rate-1/2에도 3.2배 적용.
10. 한계: HW 미합성(ideal pipeline speedup 가정), 정정능력 향상 아님(latency만), 예제 부호 소형·rate 0.5.
