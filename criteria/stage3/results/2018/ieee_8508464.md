### B. 알고리즘 요약

1. 대상: 자기기록(MR/HDD)용 `SC`(spatially-coupled) 부호, block code의 `H`를 성분행렬 `H_0..H_m`으로 분할 후 `L`개 복제·coupling하여 `H_SC` 구성 (memory `m`, coupling length `L`).
2. 문제: HDD 섹션마다 SNR이 달라 기존 ECC는 최저 SNR 섹션 기준 설계 -> 고 SNR 섹션에 불필요한 redundancy 발생.
3. 모델: 각 section을 `SNR_i = SNR_abs + ΔSNR_i`인 독립 AWGN 채널로 모델링(section index `i`), underlying block code 길이 = 한 section 길이, `L`이 span하는 section 수 결정.
4. 핵심기법 1 (부호설계): minimum-overlap(MO) partitioning으로 성분행렬 분할 -> 파생 SC 그래프의 detrimental object(trapping/absorbing set) 수 감소, memory `m≥1`로 확장.
5. 핵심식/원리: memory `m` 클수록 하나의 `CN`이 `m`개 연속 section의 VN 메시지를 받아 섹션 간 협력(cooperation) 증가 -> SNR 변동 완화.
6. 핵심기법 2 (interleaving): SC codeword를 `L·(m+...)` chunk로 나눠 연속 chunk에서 하나씩 뽑아 재배열하는 channel-aware interleaving -> 대부분의 CN이 모든 `L` 신뢰도 레벨에서 균등하게 메시지 수신.
7. 구현 특성: interleaving 복잡도가 성분행렬 수에 반비례, `m=0`(uncoupled) 시 regular interleaving과 동일 복잡도.
8. 최적화: 없음(구성법·시뮬 기반).
9. 결과: array-based Block Code 대비 SC Code가 error-floor 영역에서 약 2자릿수(orders of magnitude) 개선, 최적화 interleaving 및 큰 memory에서 추가 개선.
10. 한계: HW 미설계, 시뮬(BER)만, 정확한 rate/length/memory 수치는 그림·손상 텍스트에 있어 본문 확정 불가, 자기기록 SNR 변동 채널 특화(NAND 무관).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC 부호 minimum-overlap partitioning 구성 | `ecc_top.cpp` `Load_PCM()` | 낮음 - Prime ECC는 고정 QC-LDPC(비-SC), H-matrix 재설계 부담 큼 |
| channel-aware interleaving | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 낮음 - 개념 차용은 가능하나 SC 구조 전제, NAND 균일 채널과 부정합 |
| error-floor 개선(섹션 협력) | `decoder.cpp` `LDPC_Decoder()` | 낮음 - SC coupling 기반이라 blockwise 디코더에 직접 대응 없음 |
| section별 SNR 변동 AWGN 모델 | 대응 없음 | 자기기록 HDD 채널 특화 |

적용 가치: **낮음** - SC 부호설계 + 자기기록 SNR 변동 인터리빙이라 NAND binary QC-LDPC 코드베이스에 구조적 접점이 없고, 차용 가능한 것은 "채널 편차 인식 interleaving" 개념 정도.
