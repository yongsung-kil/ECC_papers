### B. 알고리즘 요약

1. 시스템: WiMAX 802.16m rate `1/2` LDPC 컨볼루션 부호(LDPC-CC, memory `ms=491`)를 AWGN 채널에서 `P=20` iteration으로 복호, streaming 데이터 대상.
2. 문제: Feldman의 LP 복호는 BP급 성능이나 메모리·연산량이 커 HW 부적합 → ADMM으로 반복화한 뒤 LDPC-CC의 pipeline 구조에 맞춰 재정식화 필요.
3. 모델: 반무한 syndrome-former `H^T`를 `(ms+1)` 코드블록 크기의 독립 window로 분할, 각 window에 processor 하나씩 할당해 P iteration을 완전병렬 파이프라인화.
4. 핵심 기법: horizontal-layered(HL) 스케줄 기반 ADMM-LP-CC(Alg.1) - CNA에서 `w=ρ·lv+(1-ρ)z-λ`, `z=Π_{Pd}[w]`(parity polytope Euclidean projection), `λ=z-w` 갱신.
5. 핵심 식: 목적함수 `v·γ^T` 최소화 + augmented Lagrangian(penalty `µ`), z는 check polytope로, v는 [0,1] hypercube로 사영 → LP 완화해를 반복적으로 근사.
6. 확장: penalized ADMM-l2(penalty `α`) 버전 추가로 MS-FP 대비 BER 개선(단 SPA 대비 최대 0.2 dB gap).
7. 구현 트릭: skipping projection을 `∆w=ρ(lv-z^{t-1})`로 재유도해 과거 `w` 저장 없이 무의미 projection 스킵 → 기존[1] 대비 메모리 `m` 절감, add/sub 12.5~37.5% 절감.
8. 양자화: Lv 8-bit, CN-to-VN 6-bit, z/λ/w 별도 fixed-point 배분, 상수(`δ,ρ,µ`)를 2의 거듭제곱 분모로 shift 처리.
9. 결과: FX-Q3(Lv 8b)로 FP 대비 BER 손실 약 0.05 dB, processor 메모리 30% 절감, Q1 대비 메모리 약 절반, 산술연산 23%·메모리접근 28% 절감.
10. 한계: HW 합성/실측 gate·throughput 수치 없음(시뮬만), convolutional 부호+ADMM projection 특화, 무선 표준(802.16m) 타깃.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP CN 갱신(Euclidean projection) | `decoder.cpp` `CNU_Update_New_Mag()`(대응 부분적) | Prime ECC는 min-sum CN이라 projection 기반 CNU와 알고리즘 상이, 직접 대응 없음 |
| pipeline processor(P iteration 완전병렬) | 대응 없음 | Prime ECC는 QC-LDPC block용 z=32 row 병렬 pipeline로 CC window 파이프라인과 구조 다름 |
| 고정소수점 양자화 스킴(Lv/z/λ/w) | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 양자화 개념은 유사하나 ADMM 변수(z,λ,w) 전용이라 min-sum LLR 테이블과 무관 |
| skipping projection | 대응 없음 | min-sum엔 Euclidean projection 자체가 없음 |

적용 가치: **낮음** — LDPC 컨볼루션 부호 + ADMM-LP projection 복호는 Prime ECC의 QC-LDPC block + min-sum과 부호구조·복호원리가 모두 달라 이식 대상이 없다.
