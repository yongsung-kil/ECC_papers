> 총평: LDPC를 소비하는 시스템 계층(압축+FTL)에서 RBER을 줄여 read latency를 개선하는 방식 — 부호설계·디코더와 직교하여 Prime ECC 이식 대상 아님.

### B. 알고리즘 요약

1. 시스템: NAND flash(2ch×2chip×4plane, 64×4KB-page/block), LDPC ECC(user `k`=4KB, redundancy `n-k`=512B, rate≈0.889), soft-decision read.
2. 문제: 높은 RBER 데이터는 LDPC가 여러 reference voltage로 반복 sensing해야 해 read latency가 증폭됨(`RL=α·N+β·log(N+1)`, `N`=sensing level).
3. 모델: page 단위 무손실 압축으로 user data를 `k·cr`로 줄이고 남은 `k(1-cr)`를 known data(all-`1`)로 채워 ECC 인코딩 — 디코딩 전 known 비트 복원.
4. 핵심기법: 압축 codeword의 RBER `RBER2 = RBER1·(1 - (k-k·cr)/n)` `(2)` — 알려진 비트만큼 실효 에러비트 감소로 낮은 sensing level 사용 가능.
5. 식 의미: RBER↓ ⇒ 필요 sensing level `N`↓ ⇒ sensing+transfer latency↓, read당 이득.
6. 확장(선택 압축): 모든 page 압축은 write 성능 저하 → read 빈도·압축률 둘 다 고려한 Double Queue(Q1=좋은 압축률, Q2=보통, LRU 관리) 제안.
7. 구현 디테일: 압축률 임계 `CT1=0.55`, `CT2=0.925`, entropy로 압축률 예측, idle time/write hit 시 압축 후 재기록(compression flag를 FTL 매핑에 추가).
8. 학습/최적화: 없음(휴리스틱 큐 관리), DQ 길이·μ·σ 민감도 실험.
9. 결과: SSDSim 트레이스 시뮬에서 평균 read latency 42% 감소, 압축 page <2%, write 성능 영향 없음(CL=68µs, DL=16µs @4KB).
10. 한계: HW 설계·합성 없음(트레이스 시뮬), ECC 알고리즘/부호 무변경, 압축 가능 데이터에 한정, 여분 압축이 wearing 소폭 증가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 무손실 압축 + Double Queue 선택 (FTL/컨트롤러) | 대응 없음 | ECC 엔진 외부 시스템 계층 — Prime ECC 시뮬레이터에 대응 모듈 없음 |
| known data 채움으로 RBER 감소 | 대응 없음 | 부호 인코딩 입력 전처리, `encoder.cpp`와 별개 상위 로직 |
| RBER→sensing level→read latency 모델(식1) | [channel.cpp](../Prime_ECC_3.1_Claude/) `Set_R_Offset()`, `Set_LLR_Th()` | RBER/soft-read 주입 개념만 접점, 알고리즘 이식 아님 |
| LDPC 디코딩 자체 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 논문은 디코더를 블랙박스로 소비, 변경 없음 |

적용 가치: **낮음** — LDPC를 상위에서 소비하는 시스템/FTL 최적화로, 부호설계·디코더·HW 어느 레이어도 건드리지 않아 Prime ECC에 이식할 알고리즘 요소가 없다.
