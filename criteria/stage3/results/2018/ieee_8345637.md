> 총평: 디코딩 전에 base matrix로 VN group 갱신순서를 오프라인 결정하는 저비용 스케줄링. 디코더 코어 로직 무변경으로 iteration만 줄여 Prime ECC layered/shuffled 스케줄에 참고 가치.

### B. 알고리즘 요약

1. 대상: protograph 기반 QC-LDPC (802.11ad N=672 R=1/2,5/8,3/4; 802.11n N=1296 R=2/3, N=1944 R=1/2,3/4), AWGN+BPSK, normalized min-sum(scaling 0.75), `Imax=10`.
2. 문제: shuffled(VN-wise serial) decoding에서 VN group 갱신순서가 수렴속도를 좌우. 기존 dynamic scheduling은 residual 계산으로 추가 복잡도, edge-wise는 iteration당 시간이 edge 수배로 큼.
3. 핵심모델: PEXIT chart의 상호정보량 `J(σ)`로 메시지 신뢰도 측정. 채널 상호정보 `Ich(j)=J(√(8R·Eb/N0))`, C2V/V2C 상호정보를 base matrix `b_{i,j}`로 반복 갱신(식 2,3).
4. 핵심기법: "가장 신뢰도 높은 정보를 여러 CN type에 전파할 VN type을 먼저 갱신". objective `S(j)=Σ_i b_{i,j}·I^E_{V2C}(i,j)` (식 5) 최대인 `j_max`를 순서로 확정(식 6), G에서 제거하며 `Nb`회 반복(Algorithm 1).
5. 의미: `S(j)`에 degree가 두 번 반영(`b_{i,j}` 가중 + V2C 계산 내부)되어, IAPP/ΣIC2V보다 high-degree 노드 우선순위 오정렬 확률을 낮춤.
6. 초기화: `I_{V2C}=Ich=J(√(8R·(Eb/N0)_th))` (decoding threshold 사용), `IC2V=0`.
7. 구현 특성: 스케줄은 반복 디코딩 전에 오프라인으로 결정되므로 디코더 자체에 추가 복잡도 없음. degree 기반 grouping 불필요.
8. 학습/최적화: 없음(닫힌형 상호정보 계산만).
9. 결과: 정규화 평균 iteration이 Original/CW/IFS 대비 감소, SNR 높을수록 격차 확대. FER 곡선은 모든 스케줄이 유사(정정능력 동일). PEXIT 분석에서 min IAPP가 IFS보다 빠르게 증가.
10. 한계: 순수 알고리즘(HW 미설계), 무선 802.11 코드 대상, group 수 증가 시 throughput 저하, 802.11ad R=3/4에선 CW/IFS와 순서 동일.
