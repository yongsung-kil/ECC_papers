### B. 알고리즘 요약

1. 대상은 DVB-S2 표준 binary LDPC(codeword 16200/64800 bit, 21개 rate 중 6개 선택: 11/45, 2/5, 2/3, 1/4, 1/2, 3/4), Xilinx Artix-7 FPGA 구현.
2. 문제: 저가 FPGA에서 DVB-S2 인코더/디코더를 파라미터화(rate·frame 길이 가변)하여 수용 가능한 throughput·자원으로 구현.
3. 채널: soft demodulator가 127 이산레벨 LLR 출력(-63~+63, 0=불확실) - 사실상 ~7bit soft.
4. 핵심기법 - min-sum 복호: `log[tanh(x/2)]`를 min 합산으로 근사해 HW 자원 절감, 표준 2-step(CN 전체→BN 전체)을 1-step(CN 계산 즉시 해당 BN 갱신)으로 대체 [3].
5. 의미: 즉시갱신(1-step)으로 iteration당 처리시간 단축 → 복호 속도 향상.
6. HW 구조: 90개 FU(functional unit) 부분병렬, 각 FU는 현재 check 비트확률의 majority 갱신 수행, 90 RAM block 메모리 모델.
7. 구현 디테일: DVB-S2 H의 360x360 dual-diagonal 서브행렬이 병렬 check 간 메모리 read/write 충돌 유발 → H 재배치로 dual-diagonal 제거 [4], FU 데이터 파이프라인으로 iteration 처리시간 추가 단축.
8. (학습/최적화 절차 없음)
9. 결과: Artix-7 xc7a200 합성, 230 MHz까지 동작·logic cell 31% 점유, 110 MHz 기준 throughput 4.24~17.9 Mbit/s(rate·길이별), 1~50 iteration, erasure 최대 37.5%까지 복호 가능(N=64800, r=1/4).
10. 한계: DVB-S2 특정 부호 대상, 정정능력 자체 개선은 없음(표준 부호 구현), throughput은 Mbit/s급으로 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CN 근사(log-tanh→min합) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 낮음 - Prime ECC 기보유 min-sum과 동일 |
| 즉시갱신(1-step) 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 낮음 - 스케줄 변형, Prime ECC는 Dual-Update 보유 |
| dual-diagonal 제거·H 재배치(병렬충돌 해소) | `encoder.cpp` `Column_Reordering_Dual_Update()` | 낮음 - 개념 유사하나 DVB-S2 부호 전용 |
| 90 FU 부분병렬 / RAM 메모리 모델 | 대응 없음 | 낮음 - Prime ECC는 z=32 행병렬로 구조 상이 |
| soft demodulator 127레벨 LLR | `channel.cpp` `Set_LLR_Th()` | 낮음 - Prime ECC는 최대 soft-2~3bit, 7bit 미지원 |

적용 가치: **낮음** - min-sum·즉시갱신·부분병렬·dual-diagonal 재배치가 모두 Prime ECC 기보유 기법이고, DVB-S2 부호구조·7bit soft·Mbit/s throughput 모두 NAND QC-LDPC 맥락과 어긋난다.
