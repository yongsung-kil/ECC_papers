### B. 알고리즘 요약

1. 대상: binary LDPC (`[2640,1320]` Margulis, `[2304,768]` WiMAX), AWGN 채널, LP(선형계획) 디코딩 계열. LLR `γ_i=log(P(y_i|0)/P(y_i|1))`.
2. 문제: BP/MS는 근사 기반이라 고SNR error floor 존재. Feldman LP 디코딩은 error floor를 낮추나, penalized-LP는 penalty 파라미터 `α` 선택이 성능·효율을 크게 좌우하고 SNR별 튜닝이 어렵다.
3. 핵심모델: binary 제약 `x∈{0,1}^N`을 box와 ℓp-sphere 교집합으로 **정확** 재정식화 `x∈[0,1]^N ∩ {‖x−½·1‖_p^p = N/2^p}` (식 2), penalty 파라미터 불필요.
4. 핵심식: ℓp-box 디코딩 문제(식 3) = `min γ^T x s.t. box + ℓp-sphere + Pj x∈PP_d`. `p=2`로 고정해 sphere projection을 단순화.
5. 의미: penalized-LP는 최적해가 원 문제와 달라질 수 있으나, ℓp-box 재정식화는 원 binary 디코딩과 등가라 전역 최적해가 일치.
6. 솔버(ℓ2-box ADMM): 보조변수 y(box), z(codeword polytope)를 도입해 분리, augmented Lagrangian `L_{μ1,μ2}`로 x/y/z/λ 교대 갱신(Algorithm 1). x-update는 `P=Σ Pj^T Pj`가 대각(`|N(i)|`)이라 분산·병렬 처리 가능.
7. 구현: y-update는 sphere projection의 닫힌형 해, z-update는 codeword polytope projection `Π_{PP_d}`. 종료조건은 `‖Pj x−zj‖_∞<ε` 및 `‖x−y‖_∞<ε`.
8. 학습/최적화: ADMM 파라미터 `μ1,μ2`만 존재하며 `[0,200]×[10,200]`에서 성능 둔감(파라미터프리 강조).
9. 결과: Margulis에서 SNR 1.8~2.4 구간 BP error floor 대비, WiMAX에서 SNR 4~4.4 구간 MS error floor 대비 WER 최저 달성. 2-penalized ADMM 대비 추가 CPU time <0.008s.
10. 한계: 순수 알고리즘(HW 미설계), 반복 최적화(max iter 1000)라 min-sum 대비 latency 큼, Margulis/WiMAX 특정 코드 실험, gate count/throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM LP 디코딩 전체 | 대응 없음 | Prime ECC는 min-sum message-passing, LP/ADMM 최적화 디코더 자체가 없음 |
| error-floor 저감(pseudocodeword 억제) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 개념적 목표(error-floor)만 공유, 알고리즘 이식 아님 |
| codeword polytope projection Π_{PP_d} | 대응 없음 | min-sum CN 연산(`CNU_Update_New_Mag()`)과 수학적 구조 상이 |
| LLR 벡터 γ 입력 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | LLR 소스만 대응, 이후 처리 완전 상이 |

적용 가치: **낮음** — full-LP/ADMM 최적화 디코더로 Prime ECC의 min-sum HW 파이프라인과 계산 구조가 근본적으로 달라 이식 불가. error-floor 저감이라는 목표만 참고 대상이다.
