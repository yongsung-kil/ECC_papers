### B. 알고리즘 요약

1. 대상은 병렬 edge(최대 2개)를 갖는 protograph LDPC(PTG-LDPC)로, base matrix `B`를 lifting 크기 `L`로 확장해 QC 형태의 parity-check matrix `H`를 얻는다.
2. 문제: 기존 그래프기반 lifting 구성은 원하는 cycle 분포를 얻기 위해 대규모 컴퓨터 탐색이 필요해 설계 복잡도가 높다.
3. 4-cycle 회피의 필요조건을 exponent matrix `P`의 원소 `p_{i,j}`(circulant shift) 관점에서 6개 case(1×1, 1×2, 2×1, 2×2 부분행렬 등)로 유도한다.
4. 핵심 case 조건: 각 부분행렬의 shift 원소들이 `mod L`로 특정 합/차 등식을 만족하지 않아야 4-cycle이 없다(Proposition 1).
5. 이 조건식은 병렬 edge를 가진 protograph의 shift 값이 galois field 원소로부터 유도될 때 4-cycle 부재를 보장한다.
6. 알고리즘: base matrix `B`를 두 component base matrix `B_a + B_b = B`로 분해하고, GF(q)의 세 disjoint subset `S1,S2,S3`로 component exponent matrix `P_a`,`P_b`를 대수적으로 구성.
7. `S3`는 4-cycle 조건 만족을 위해 원소를 하나씩 탐색하는 Searching Algorithm(Algorithm 1)으로 결정(탐색 규모는 lifting factor보다 작음).
8. 최종 exponent matrix `P`는 `P_a`,`P_b`를 조건 (10)/(11)로 결합해 얻고, expansion으로 `H` 생성. lifting `L = q-1`.
9. 결과: (8128,7113) rate≈0.875, (672,546), (144,72) 예제 부호가 [15] PTG-LDPC·CCSDS·5G 표준부호와 유사 성능, IEEE 802.11ad 대비 BLER 10^-4에서 약 0.25dB coding gain.
10. 한계: AWGN/BPSK sum-product 시뮬만 제시, HW/디코더 설계·error-floor 분석 없음, 성능이 기존 대비 대등(뚜렷한 우위 없음).
