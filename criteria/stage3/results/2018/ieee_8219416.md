### B. 알고리즘 요약

1. 대상: 정규 QC-LDPC 코드 (`(dv,dc)=(3,6)` R=0.5 N=1296, `(4,8)` R=0.5, `(4,34)` R=0.88 N=9520), 채널은 BSC (crossover α), hard-decision 디코딩.
2. 문제: PGDBF는 GDBF 대비 큰 성능이득(soft-decision 근접)을 주지만, 확률적 신호 생성기(random generator)가 필수라 HW 오버헤드(레지스터 8배, LUT 1.64배)가 발생.
3. 핵심식: 각 VN은 에너지 `E_ij = (v_ij ⊕ y_ij) + Σ c_ab`를 계산, 최대에너지(`E_max`)이고 난수 `R_ij=1`일 때만 flip. `R`은 `Pr(R=1)=p0`.
4. 핵심기법(VNSA): QC 순환구조를 이용해 iteration마다 VN 메시지 메모리 B와 채널값 C를 base column 내에서 1칸 cyclic shift(`B_{i,(j%Z+1)} = Bu_{i,j}`). 결과 디코딩은 generic 구조와 동일하나 한 VN이 매 iteration 다른 VNU에서 처리됨.
5. 의미: 서로 다른 동작(f1=flip, f2=hold)을 한 VNU에 다 넣는 대신, type-1 VNU(p0 비율)와 type-2 VNU(1-p0 비율)로 물리적 분산 → 난수생성기 불필요.
6. VNSA-PGDBF: type-1은 conventional VNU에서 AND게이트 제거(=GDBF VNU), type-2는 XOR2 제거하고 현재값 그대로 전달. `S=Z`인 CSTS-PGDBF와 동등 동작.
7. VNSA-IM-PGDBF(imprecise): type-2를 type-3(에너지 계산·회로 전부 제거)로 대체, Maximum Finder가 `p0·N`개 후보만 탐색 → MF도 단순화. "부정확한 최대값"이 오히려 성능 향상.
8. 복잡도 효율 `ξ = (p0·C1+(1-p0)·C2)/C12`, p0=0.7에서 ξ≥0.7 → VNU 자원 최대 30% 절감.
9. 결과(ASIC, SAED 90nm): VNSA-PGDBF는 GDBF보다 약 6.68~11% 낮음, VNSA-IM-PGDBF는 GDBF보다 15~18% 낮은 복잡도, fmax 400MHz(GDBF 385, CSTS 357), 1 iter=1 clock. dv3R050N1296에서 FER 1e-8까지 이론 PGDBF에 근접. MS 대비 면적 4.7배 작음.
10. 한계: BSC hard-decision 전용, gate count/throughput 절대수치는 표에만 있어 본문 미상. dv4 및 R=0.88 고rate에서 VNSA-IM은 error floor 발생, 전력은 GDBF보다 높음(76~82mW).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PGDBF 디코딩 루프 (에너지·flip) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | BF 계열은 min-sum과 알고리즘 상이, 이터레이션 루프 골격만 대응 |
| VN flip / hard-decision 처리 | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` | HD 경로에 BF flip 로직 삽입 관점 대응 |
| VNSA 메시지 cyclic shift (z 단위) | 대응 없음 | Prime ECC는 z=32 shift가 CN연산용, VNSA식 iteration별 VN 메모리 shift 개념 없음 |
| 확률적 flip 대체(난수생성기 제거) | 대응 없음 | Prime ECC min-sum에는 확률적 요소 자체가 없음 |

적용 가치: **낮음** — hard-decision BF 계열로 Prime ECC의 soft min-sum과 알고리즘 축이 다르고, VNSA는 BSC·정규코드 특화라 고rate soft NAND 파이프라인에 직접 이식하기 어렵다(저전력 백업 디코더 컨셉으로만 참고).
