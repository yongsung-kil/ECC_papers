### B. 알고리즘 요약

1. 시스템: 프로토그래프 기반 SC-LDPC 부호(`nc×nv` base matrix B, lifting factor N), 윈도우 복호기(window size W), rate R=1-nc/nv, 예제 R=1/2·3/4.
2. 문제: 큰 coupling length L에서만 threshold saturation이 나타나 latency/복잡도가 커지며, 윈도우 내 degree-1 VN이 성능을 떨어뜨려 큰 W가 필요함.
3. 모델: BEC(점근 해석) / BI-AWGNC(유한길이), Protograph-EXIT로 WD threshold `ε_WD(B,W,δ)` 산출(δ=10^-12).
4. 핵심 기법: base matrix B를 변수로 WD threshold를 최대화하고 평균 VN degree `dv`를 최소화하는 다목적 최적화 (식 4), 제약: `bmax`, degree≥2, 체계적 저복잡도 인코딩 제약 `B0=[B0'|I]`.
5. 의미: WD threshold↑는 정정성능(convergence speed) 개선, `dv`↓는 복호 복잡도 `C=W·Imax·dv` 감소 → 두 목표가 상충하므로 Pareto front를 찾음.
6. 확장: memory `ms`, 윈도우 크기 W를 파라미터로 스윕(ms=1~6, W=ms+4) 하며 Pareto front 변화 분석.
7. 구현 디테일: 정수 base matrix라 mutation 후 반올림, feasible-only selection으로 penalty 함수 생략.
8. 최적화 절차: SHADE(success-history adaptive DE) 기반, NP=H=50, Gmax=700 세대, 다목적 확장(Zhang-Sanderson).
9. 결과: 최적 프로토그래프 P1(R=1/2, σ_WD=0.961), P2(R=3/4, σ_WD=0.667). R=3/4에서 WiMAX 블록부호와 유사 복잡도로 경쟁, R=1/2는 0.2dB gap.
10. 한계: HW 미설계, 유한길이는 PEG lift(비-QC)로만 검증, NAND용 hard/soft-read·error-floor 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 프로토그래프/base matrix 구성 | 대응 없음 (Prime ECC는 QC-LDPC 블록부호, `ecc_top.cpp` `Load_PCM()`는 고정 H-matrix) | SC 컨볼루셔널 구조라 QC 블록부호로 직접 이식 불가 |
| 윈도우 복호기(WD) | 대응 없음 (`decoder.cpp` `LDPC_Decoder()`는 블록 이터레이션 루프) | 슬라이딩 윈도우 스케줄이 블록 복호기와 상이 |
| DE/SHADE 프로토그래프 최적화 | 대응 없음 (`local_opt.cpp`는 LLR 자동최적화 전용) | 부호구조 최적화 대상 자체가 다름 |

적용 가치: **낮음** — SC-LDPC 윈도우 복호 전용 이론 논문으로 NAND용 binary QC-LDPC 블록부호/min-sum 복호기와 구조적 접점이 없다.
