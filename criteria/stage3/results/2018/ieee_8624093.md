### B. 알고리즘 요약

1. 대상은 그래프 기반 부호(LDPC/RS/BCH)의 랜덤화 디코딩 일반론으로, 특정 코드 rate/length·채널을 고정하지 않은 개념 논문.
2. 문제의식: cycle이 있는 Tanner graph에서 메시지 의존성이 생겨 BP가 sub-optimal이 되고 ML 도달이 어려움.
3. 랜덤화 디코더를 static(한 디코딩 사이클 동안 random 파라미터 고정)과 dynamic(사이클 내부에서 random 연산 교체)으로 분류.
4. static 예: mismatch 해석, dithered decoding, stochastic Chase — 입력을 랜덤화 후 재디코딩해 ML 후보를 채널출력과의 Euclidean 거리로 선택.
5. 제안: hard-decision Gallager-A 기반 "ideal stochastic decoder" — 각 VN이 채널출력 Bernoulli 열에 대응하는 random bit를 방송하고, 모든 입력이 일치할 때만 출력.
6. 이 ideal stochastic decoder는 iteration→∞에서 stochastic 도메인의 ML 디코딩을 provably 구현(단, 성공확률이 유효 codeword 발견확률과 같아 비실용적).
7. dynamic 랜덤화(random sign flip, random timing violation, probabilistic scheduling)는 cycle을 통신채널로 보아 `H(X|Y)>0`로 만들어 cycle 영향을 완화.
8. 정보이론 해석: randomness가 무한대면 cycle이 완전 무력화(용량 0)되나 유용한 정보 흐름도 끊겨 성능이 악화 → 최적 randomness 양이 존재한다는 가설.
9. 결과: static randomized 일부가 ML 도달 가능함을 논증, dynamic은 waterfall 개선 여지가 있으나 ML 보장은 없음(정량 시뮬·HW 없음).
10. 한계: latency가 매우 커 deep-space/손상 저장매체 복원 등 지연 무관 응용에 한정된다고 저자 스스로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 랜덤화 message-passing 디코딩 루프 | `decoder.cpp` `LDPC_Decoder()` | 개념상 이터레이션 루프에 randomness 삽입 가능하나 우리 결정론적 min-sum과 상충 |
| dithered/random sign flip (CN sign) | `decoder.cpp` `C2V_Cal_New_Sgn()` | CN sign에 random flip을 넣는 아이디어와 대응하나 논문상 non-ML로 판명 |
| stochastic/Gallager-A hard-decision | 대응 없음 | Prime은 min-sum 기반 soft/HD 디코더로 stochastic computation 미보유 |
| re-decoding / random 입력 반복 | 대응 없음 | Prime에 재디코딩·random 입력 생성 경로 없음 |
