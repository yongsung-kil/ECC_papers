### B. 알고리즘 요약

1. 대상: M×N all-one base matrix에서 각 "1"을 CPM/RPM으로 lifting한 QC-LDPC의 잠재 cycle 수를 추정하는 순수 조합 문제.
2. 문제: 기존 tree 방법은 base matrix가 커지면 트리 저장·중복 제거 비용이 폭증해 실용성이 떨어짐.
3. 핵심 정의: 길이 `2l` closed path를 행-인덱스 벡터 `I`와 열-인덱스 벡터 `J`로 표현, `(m,n)-CP`는 서로 다른 m행 n열을 지나는 경로.
4. 예비 함수 `G(u,v)`: u개 연속 슬롯에 v개 서로 다른 digit을 (인접 슬롯은 서로 다르게) 배치하는 경우의 수 (Theorem 1).
5. `G'(u,v)`: 여기에 "마지막 슬롯이 첫 슬롯과 달라야 한다"는 조건을 추가한 변형 (Theorem 2), row/col-index 조합 수 `R(l,m)=G'(l,m)`, `C(l,n)=G'(l,n)`에 사용.
6. 중복 제거 1: 첫 행에서 시작하지 않는 CP 제거 → `R'(l,m,t)` (t = 첫 행을 지나는 경로 수, t=1,2만 고려).
7. 중복 제거 2: cyclic shift/reverse로 같은 CP가 되는 transform-equivalent(2t-1 중복) / transform-exact 쌍을 Theorem 3으로 분류·상쇄.
8. 최종식: `Dsum(l,m,n,t) = [R'(l,m,t)C(l,n) + Dex(l,m,n,t)] / 2t`, 이를 (m,n) 및 조합계수 `C_M^m C_N^n`로 합산해 `S(l,M,N)` 도출.
9. 결과: 길이 4/6/8/10에 대한 폐형식 제공(예: 5×5 base matrix 길이-10 CP = 104,040개), tree 방법 exhaustive search와 정확히 일치 검증.
10. 한계: 개수 "추정"만 제공하고 HW 미설계·복호성능 평가 없음, 실제 cycle 회피는 lifting 시 CPM 선택에 달려 있어 이 개수는 상한/노력량 지표에 불과.
