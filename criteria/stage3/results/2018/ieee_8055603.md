### B. 알고리즘 요약

1. 대상은 rate-1/2 binary LDPC 2종: Margulis regular `C1`(N=2640, d=6)과 IEEE 802.16e irregular `C2`(N=576, d=6/7).
2. 풀려는 문제: ADMM LP 복호의 지배적 연산인 Euclidean projection `ΠPP_d`를 LUT로 대체하면 빠르나, LUT 메모리가 과대(최대 100만 유닛+)해 실용성 저하.
3. 모델: memoryless binary-input symmetric channel, LLR `γ`, check polytope `PP_d`(짝수 패리티 이진벡터 convex hull)로의 projection.
4. 핵심 기법: projection 입력을 componentwise 양자화 후 LUT 참조. 입력 분포가 근사 Gaussian이라는 관찰에 기반해 균등 대신 비균등 양자화 도입.
5. 핵심 식: MMSE 규칙 `Qopt = argmin MSE(U, Q0)` (식 7,8) - 양자화 자체가 아니라 projection 출력 왜곡을 최소화하는 kernal set 탐색.
6. 확장: check degree 분포를 반영해 입력 샘플 union `U`의 비율을 degree 분포에 맞춤(`|Ud|/|U|`).
7. 구현 디테일: 범위 `[a,b]`를 precision `τ`로 이산화 후 exhaustive search로 kernal set `Q0` 선정(식 5,6). L 짝/홀 별 대칭 레벨 구성.
8. 최적화 절차: Algorithm 1, waterfall SNR에서 최적화(C1 2.0dB, C2 3.0dB), `|U|=5×10^6`; 대안으로 dynamic programming/differential evolution 언급.
9. 결과: C1에서 N71이 U81 수준 성능을 절반 메모리로 달성; U110 대비 0.05dB 손실로 메모리 81.5% 절감(30030→5544). C2는 N92가 U15 근접, 메모리 94% 절감(1046520→63063). 평균 iteration도 감소.
10. 한계: 순수 시뮬(HW 미설계), ADMM/LP 복호 전용, gate count/throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM Euclidean projection LUT | 대응 없음 | Prime ECC는 min-sum CN 연산(`decoder.cpp` `CNU_Update_New_Mag()`)이며 projection polytope 개념 없음 |
| projection 입력 비균등 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` / `ecc_data.h` `PARAM_LLR` (개념 유사만) | LLR 양자화 테이블과 목적은 유사하나 대상 연산이 달라 직접 이식 불가 |
| 평균 iteration 감소 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 루프 | 수렴 개선 효과는 있으나 ADMM 특유로 min-sum에 전이 불가 |

적용 가치: **낮음** - ADMM/LP 복호 전용 projection LUT 최적화라 min-sum 기반 Prime ECC와 알고리즘 계보가 달라 재사용 지점이 없다.
