> 총평: NAND retention 특성으로 hard read만으로 신뢰비트 활용하는 아이디어는 직접 관련·매력적이나, 우리는 min-sum(soft) 기반이라 BF 채택은 아키텍처 전환이 필요.

### B. 알고리즘 요약

1. 대상: 2bit/cell MLC NAND, regular LDPC `R=0.75`, `N=1296`; retention 오류만 가정(단일 단계 인접 상태 천이).
2. 문제: soft-decision LDPC는 정정력 좋지만 soft-sensing이 read latency를 크게 늘림 → hard read만으로 유사 성능을 원함.
3. 채널 모델: raw BER `α`의 crossover(`Pr(y_n≠x_n)=α`), retention은 항상 낮은 전하 방향의 단방향 전이.
4. 핵심 관찰: retention은 단방향이라 특정 read 값의 일부 비트는 error-free — 예: `01`은 MSB·LSB 모두 신뢰, `00`은 MSB 신뢰, `10`은 LSB, `11`은 MSB 신뢰.
5. 전처리(pre-processing): 셀의 (MSB,LSB)로부터 신뢰벡터 `I=(I_1..I_N)` 생성(단순 조합회로, XOR 기반, 오버헤드 무시가능).
6. BF 기본: CN은 parity XOR(`c_m`), VN은 energy `E_n = (v_n⊕y_n) + Σ c_m` 계산, 최대 energy VN을 flip(GDBF), PGDBF는 확률신호 `R_n`로 flip 게이팅.
7. 적응(A-GDBF/A-PGDBF): flip은 해당 VN이 unreliable(`I_n=0`)일 때만 수행 → 신뢰(정답) VN이 잘못 flip되어 발생하는 진동/트래핑셋 실패를 회피.
8. 예시: trapping set (5,3)에서 GDBF는 항상 실패하나 A-GDBF는 신뢰정보로 75% 정정.
9. 결과: raw BER `2e-3`에서 A-PGDBF BER `8e-10`, A-GDBF `4e-8`; 전처리 미사용 대비 A-GDBF 0.5dB·A-PGDBF 0.2dB 이득; BF는 1 iter=1 clock, MS는 layer당 2 clock(4 layer)이라 BF가 MS 대비 약 4배 빠름.
10. 한계: N=1296 소규모·regular 코드 시뮬만, soft-sensing 회피의 latency 이득은 정량화 안 함, MS(4bit LLR/6bit AP-LLR)보다 정정능력은 여전히 낮음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BF VN energy/flip 처리 | decoder.cpp VN_Cal_HD() (HD 경로) | 우리 HD는 min-sum 기반, BF flip 로직은 신규 — VNU 대체 필요 |
| CN parity XOR | decoder.cpp CNU_Update_New_Mag()/C2V_Cal_New_Sgn() | BF의 binary CN은 sign만 → 부분 대응 |
| 신뢰비트 판정(retention 기반 I) | channel.cpp Set_R_Offset()/Set_LLR_Th() | soft-read threshold 대신 hard 신뢰마스크 주입점으로 활용 가능 |
| 조기종료(syndrome=0) | partial_CRC.cpp/full_CRC.cpp | BF는 CN 만족 검사로 종료, 우리 CRC 종료와 개념 유사 |
| PGDBF 확률신호 생성 | random.cpp (XOR25 난수) | 난수원 재사용 가능 |
| MLC MSB/LSB 신뢰 전처리 회로 | 대응 없음 | 우리 채널에 MLC 비트매핑/retention 모델 없음 |
```

적용 가치: 중간 — retention 단방향성으로 hard read 신뢰비트를 디코더에 주입하는 개념은 NAND 직접 관련이나, Prime ECC는 min-sum(soft) 기반이라 BF 전면 채택보다는 "신뢰비트를 LLR 초기값/고정"으로 반영하는 부분 이식이 현실적.
