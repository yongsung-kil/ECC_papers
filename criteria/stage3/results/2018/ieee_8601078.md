### B. 알고리즘 요약

1. 시스템: STT-MRAM용 고rate·단(短)코드 protograph LDPC, 정보어 길이 520 bit, rate `R=0.91/0.83/0.72`, 3-bit capacity-maximizing 양자화.
2. 문제: 기존 BCH·EG-LDPC는 worst-case raw BER 기준 고정 rate라 저장밀도 낭비, die별 raw BER 다양성(diversity) 대응 불가.
3. 채널 모델: STT-MRAM은 본질적으로 비대칭 → i.i.d. 채널 어댑터로 대칭화 후 PEXIT 적용(write error rate `1e-4`, 정규화 저항 spread `V0/P0` 변화로 process variation 반영).
4. 핵심 기법 1단: symmetrized 채널에 `PEXIT` 알고리즘으로 복호 threshold 측정.
5. 핵심 식 의미: 단코드에서는 threshold(PEXIT)와 AWE(error floor)의 의미가 장코드만큼 크지 않아, 둘로 후보 L개를 뽑고 실제 시뮬 BER로 최종 선택하는 3중 결합 지침 사용.
6. 핵심 기법 2단: `AWE`(asymptotic weight distribution)로 error-floor 최적화.
7. 구현 디테일: rate `R=10/(11+n)` 형태로 daughter code(rate-0.91)에 variable/check node를 동수로 추가하는 code extension → 단일 base matrix 구조 유지.
8. 최적화 절차: L개 후보 생성(threshold+AWE 우수) → waterfall·error-floor 양 영역 실측 BER로 최종 선택.
9. 결과: 제안 RCP-LDPC가 3개 rate 모두에서 AR4JA 대비 BER 우수, 평균 iteration 수도 더 빠른 수렴(Fig.4).
10. 한계: STT-MRAM 채널 특화, HW 미설계, 시뮬만, 컨퍼런스 short paper라 세부 base matrix 미공개.
