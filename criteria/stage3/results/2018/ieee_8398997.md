### B. 알고리즘 요약

1. 대상은 QC-LDPC 코드로, base matrix `H_b`에 circulant shift를 적용해 확장한 sparse PCM `H`를 사용한다.
2. 문제의식: LDPC 디코더는 PCM 저장 메모리 부담과 flexibility 확보가 어렵고, high-throughput HW 구현이 난제라는 점을 배경으로 든다.
3. 채널/noise 모델은 별도 정의 없이 일반 LLR 기반 message passing 전제만 서술.
4. 핵심 기법: Variable Node Decoder를 VMEM(Block RAM)→MUX→VNPU→De-MUX→CMEM 파이프로 구성하고, 반복 message passing으로 iterative 복호.
5. VNPU는 intrinsic LLR(채널)과 a priori LLR들의 합으로 a posteriori LLR을 계산(자기 edge 제외 합).
6. Interleaver는 Barrel Shifter로 구현, 1 clock에 n-bit shift/rotate하여 QC circulant permutation을 수행.
7. Check Node Decoder Memory(CMEM)는 VN 결과를 Block RAM에 저장, CN이 row당 1 clock에 처리한다고 언급.
8. 학습/최적화 절차 없음(고정 HW 블록 나열).
9. 결과: Altera Quartus II로 각 블록(Block RAM, MUX, De-MUX, Barrel Shifter)의 파형 예시만 제시, 정량 성능(BER/throughput/area) 없음.
10. 한계: 부호 rate/length·CN 갱신 알고리즘(min-sum/SPA) 미명시, 실측 성능·복잡도 수치 전무, 통합 디코더 검증 미제시.
