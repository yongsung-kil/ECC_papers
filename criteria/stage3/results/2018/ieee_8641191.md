### B. 알고리즘 요약

1. 시스템: AWGN 채널, RCM(rate compatible modulation, 중첩부호변조) 복조. 소스비트 `N=400`, 가중치집합 `{±1,±1,±2,±4}`, 심볼수 `M=200/400`.
2. 문제: RCM 복조의 기존 BP는 flooding(병렬) 방식이라 현재 iteration의 최신 메시지를 못 써서 수렴이 느리고 error floor가 높다.
3. 모델: 매핑행렬 `Q`(M×N sparse, 행당 L≤10 nonzero)로 `y=Q·x`, bipartite graph의 bit node(BN)/symbol node(SN) 간 message passing.
4. 핵심기법 1 (GS-BP): 매핑행렬을 `K_G` 그룹으로 열(vertical) 분할, 그룹별로 순차 갱신하여 앞 그룹의 최신 B2S 메시지를 뒤 그룹 S2B 갱신에 즉시 사용.
5. 의미: RCM은 채널조건별 사용 심볼수(행수)가 변해 row 분할(LBP)이 복잡 → column 분할 선택.
6. 핵심기법 2 (BS-RBP): residual BP 응용. S2B 잔차 대신 계산이 싼 bit-to-symbol(B2S) 잔차를 우선갱신 지표로 사용(식 8: `r(m_k)=||f_k(m)-m_k||`).
7. 구현 디테일: S2B(식 3) convolution 부담을 [4]의 table-lookup + max 근사(식 6~7)로 완화.
8. 결과: BER=10^-2에서 GS-BP·BS-RBP가 conventional BP 대비 각 ~7dB, ~5.5dB 이득. Es/No=20dB에서 수렴 iteration 9→6.
9. 그룹수 ↑ 성능 ↑이나 ≥50이면 포화. RBP 계열은 greedy라 수렴 후 FER은 GS-BP보다 낮음(열등).
10. 한계: 시뮬레이션만, HW 미설계, 대상이 LDPC 복호가 아닌 RCM 복조(symbol node가 convolution 연산).
