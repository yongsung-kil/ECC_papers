### B. 알고리즘 요약

1. 대상 시스템은 NAND flash SSD(8채널×4칩×4plane, 4KB 페이지, page-mapping FTL); LDPC를 기본 ECC로 채택하나 코드 스펙은 미제시.
2. 문제: ISPP `DVpp`(program step size)를 키우면 write cost↓지만 RBER↑ → read시 필요 reference voltage 수 `N`↑ → read cost↑. 기존 기법은 물리특성만 이용, 워크로드 접근특성 미활용.
3. 모델: write cost `WC(DVpp)=g/DVpp`, read cost `RC(N)=a·N + b·⌈log(N+1)⌉`, 제약 `RBER(DVpp) < CBER_LDPC(N)`.
4. 핵심기법1(성능): 페이지를 read-only/write-only/interleaved로 분류(history window로 식별) 후, read-only엔 high-cost write→low-cost read, write-only엔 low-cost write를 적용.
5. 식 의미: LDPC 정정능력 `CBER_LDPC(N)` 안에서만 `DVpp`를 키울 수 있어 read/write cost의 안전한 tradeoff 경계를 규정.
6. 핵심기법2(수명): wearing `we = ζ·Vp`(최대 threshold voltage 비례). write-only 데이터를 reduced threshold(reduced wearing) write로 프로그램, write cost는 동일 유지.
7. 구현 디테일: FTL 매핑 엔트리에 access history + 1bit high-cost write tag 추가, idle-time re-write queue로 read-only 페이지를 재프로그램.
8. 최적화: history window size=2로 storage overhead와 식별 정확도 trade-off; busy면 low-cost write, idle면 reduced-wearing write 선택하는 결합 방식.
9. 결과: HCW/LCR로 read 성능 54%↑, LCW/HCR로 write 성능 26%↑; R+W read 48%↑(vs Li et al.), R+L wearing 약 20% 감소, re-write overhead <1%.
10. 한계: trace-driven 시뮬(SSDSim)만, LDPC 디코더·부호·HW 설계 없음. ECC는 read cost 제약 조건으로만 추상화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ISPP step size / threshold voltage 조절 (물리 write) | 대응 없음 | Prime ECC는 컨트롤러 내 복호 시뮬레이터로 NAND 물리 프로그래밍/wearing 미모델 |
| RBER↔reference voltage N↔read cost 모델 | 대응 없음 | soft-read 비용의 시스템 모델일 뿐 LLR 양자화 설계 아님(개념적으로만 `channel.cpp` 인접) |
| FTL access characteristic 식별 / re-write queue | 대응 없음 | FTL/스케줄러 계층, ECC 코어 밖 |
| CBER_LDPC(N) 정정능력 경계 | 대응 없음 | LDPC를 블랙박스로 사용, 디코더/부호 내부 미접촉 |
