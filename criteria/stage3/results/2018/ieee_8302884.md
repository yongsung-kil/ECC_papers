### B. 알고리즘 요약

1. 대상은 BSC(hard-decision) 상의 (dv,dc) regular/QC-LDPC. 대표 예: `(3,27)` QC-LDPC, rate `0.889`, `N=9216`, QC-block 256, message 1KB — NAND SSD용 hard-read ECC를 명시적으로 상정.
2. 문제: 기존 WBFA류는 채널 soft 신뢰도 `wm=min|yn|`에 의존 → BSC에선 soft 정보가 없어 그냥 BFA로 퇴화, 특히 column weight 작으면 성능 열화.
3. 모델: BSC crossover 확률 `α`. bit-node는 bit-decision `zn`과 1비트 reliability `rn`, check-node는 syndrome `sm`과 1비트 reliability `γmn`를 주고받음.
4. 핵심 기법: check reliability `γmn = OR_{n'≠n} r_{n'}` (한 개라도 unreliable이면 unreliable로 표시, MSA의 min 동작 모사), flipping function `En = Σ_m (2sm-1)·w(γmn)`.
5. 의미: soft 채널값 대신 디코딩 진행 중 생성한 reliability 비트로 가중치를 부여 → BSC에서도 WBFA식 가중 flipping 가능.
6. 확장/트릭: flip된 비트는 부호 반전 `En=-En`으로 reliable 표시; reliability function `Dn=En(ℓ)+En(ℓ-1)`(직전 2 iter 합)로 보수적으로 `rn` 갱신.
7. 구현 디테일: 파라미터 4개 `ws,ww,θ1,θ2`. `ww=1` 정규화로 `θ1=dv-3`, `ws=2dv-1` 유도(R1,R2 부등식). `θ2`는 crossover별 BER 최소화로 오프라인 결정, `α̂`는 unsatisfied check 수로 추정.
8. 학습/최적화: closed-form 분포 불가로 `θ2`를 Table I처럼 crossover별 시뮬로 튜닝.
9. 결과: `(3,27)` rate0.889 코드에서 `α=1e-3`일 때 경쟁 알고리즘 대비 4자리수↑ 개선; `(6,54)` PEG 코드에서 WBFA 대비 3자리수↑, scaled MSA에 근접; rate0.5 코드에서도 일관 우위.
10. 한계: HW 미설계, 시뮬만(FPGA/ASIC 없음), `θ2` 오프라인 튜닝 의존, non-binary 확장·DE 분석은 future work.
