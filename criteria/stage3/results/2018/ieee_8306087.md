### B. 알고리즘 요약

1. 시스템: implant-to-implant WBAN(IEEE 802.15.6), log-normal shadowing 경로손실 채널(PL0=35.04dB, n=6.26, σ=8.18dB, capsule endoscope), BPSK.
2. 문제: WBAN 센서는 배터리·복잡도 제약이 커 soft-decision BP는 과도한 HW/전력을 소비, hard BF는 성능 부족 → 둘의 절충 필요.
3. 부호: 소형 regular LDPC 두 개 — PEG `(504,252)` rate 0.5, Gallager `(204,102)` rate 0.5, 최대 50 iteration(Min-Sum은 10).
4. 배경 기법: BF→WBF→MWBF→RRWBF→IERRWBF→MIERRWBF 계열 weighted bit-flipping과 bootstrap 삽입한 BMIERRWBF를 정리.
5. 기존 한계: BMIERRWBF는 unreliable bit 판별용 threshold `β`를 오프라인 계산하는데, 변동 심한 WBAN 채널에서 부정확·오버헤드.
6. 핵심 기법(제안 MBMIERRWBF): `β` 대신 syndrome `s=zH^T`의 non-zero 비트로 unreliable check node를 직접 식별하고, 각 unreliable check에 연결된 최소 soft-value variable node를 bootstrap `(17)`로 갱신.
7. 즉 채널 상태 정보/사전 threshold 없이 unreliable bit를 추출 → BMIERRWBF의 comparator 제거로 복잡도 소폭 감소.
8. 복잡도: `O((2Mρ+4Nγ)+Nh(Mρ+Nγ))`, M'<M·N'<N (unreliable node 수)로 full soft-decision보다 적음.
9. 결과: 제안 알고리즘이 하이브리드 중 최고 BER(min-sum 근접), 최소 iteration 수, 최고 throughput(큰 H에서), 최속 수렴 달성. BER 10^-3 @ Eb/No=26dB 요구 충족.
10. 한계: HW 미설계(gatecount 없음), 시뮬만, WBAN shadowing 채널·소형 regular 부호 전제, bit-flipping 계열이라 정정능력 상한이 BP 대비 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 unreliable node 식별 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | syndrome 계산은 공통이나, 제안은 bit-flipping 계열이라 우리 min-sum 이터레이션 구조와 상이 |
| bootstrap soft-value 갱신(min 연산) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()` | min 연산 자체는 유사하나 bootstrap의 unreliable bit 재추정은 우리 스케줄과 미대응 |
| 오실레이션 기반 조기종료(3-cell shift register) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/) `partial_crc_HW_T4()` | 우리는 이미 partial/full CRC 조기종료 보유 — 중복 |
| soft-value threshold 처리 | [channel.cpp](../Prime_ECC_3.1_Claude/) `Set_LLR_Th()` | soft-read threshold 개념은 닿으나 WBAN raw soft-value 전제라 NAND read 비용 모델과 상이 |

**적용 가치**: 낮음 — bit-flipping 계열 하이브리드 디코더로 채널(WBAN shadowing)·부호(소형 regular)·복호(BF)가 모두 우리 고rate QC-LDPC min-sum과 달라, threshold-free bootstrap 아이디어 외에는 직접 이식 여지가 거의 없음.
