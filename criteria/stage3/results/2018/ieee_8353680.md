### B. 알고리즘 요약

1. 대상은 3D-TLC NAND(Upper/Middle/Lower 3페이지, VTH 8-state)로, data-retention 시 상위 VTH-state("G"등)의 BER이 급증하는 것이 문제.
2. 기존 AEP-LDPC는 블록 평균BER 테이블로 각 비트 예측BER를 정하고 이로부터 `LLR = ln((1-BER)/BER)`를 계산해 복호하나, 실제 오류비트에 낮은 BER이 배정되면 정정력이 떨어짐.
3. 핵심 관찰: 오류비트의 예측BER이 50%에 가까울수록(불확실성 최대) LDPC가 더 잘 정정 → 오류비트에 높은 BER을 부여해야 함.
4. Asymmetric Coding(AC): flag bit로 code unit을 flip/no-flip 결정해 VTH 분포를 변조(3D-TLC에선 AC7이 "F"-state 증가로 최적), majority="1"/minority="0" 편향을 만듦.
5. HED(Horizontal Error Detection): code unit(NCL) 내 minority 개수 `Nmin`이 절반 초과면 오류 존재 → `Ndet = Nmin - NCL/2` 비트를 오류로 검출(program 시 minority ≤ 절반 보장 사용).
6. step2에서 검출된 minority 비트의 예측BER를 갱신: `Ndet=1`이면 총합 100%, `Ndet=2`이면 200%가 되도록 재분배(예: 0.60%→65%), LLR 재계산에 반영.
7. 단독 HED는 효과 작음(Fig.8): upper page는 "0→1"이 지배적인데 "0"이 minority가 아니어서 검출률 낮아(30%) upper page에서 정정 실패.
8. V-LDPC(Vertical-LDPC): EDC와 유사한 코딩순서로 3페이지에 걸쳐 검출오류를 분산·균등화해 최악 페이지(upper) 검출률을 끌어올림.
9. 결과(실측): XEE(=HED+V-LDPC)로 conventional AEP-LDPC 대비 data-retention lifetime +230%, acceptable BER +90%, 허용 W/E cycle 2.5배, 동일 조건에서 iteration 수 감소.
10. 한계: LDPC 부호/디코더 알고리즘 자체 설계가 아니라 AC 데이터매핑+오류검출로 LLR 사전확률을 조정하는 상위계층 기법이며, HW 설계·gatecount·throughput·부호 rate/length 미제시(측정 위주).
