### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 상 binary LDPC, bit-flipping(BF) 계열 디코더(MacKay code1 PEGReg504×1008 등), MS 대비 저복잡도.
2. 문제: GDBF의 noisy 변형 NGDBF는 weight `ω`만 곱해 각 partial syndrome `s_i`가 k번째 VN에 긍/부정 영향인지 구분 못해 inversion function 신뢰도 저하.
3. 모델: ML 디코딩 `x_ML=argmax Σx_k y_k`, objective `f(x)=Σx_k y_k + Σs_i`(두번째 항 penalty로 valid codeword 강제).
4. 핵심기법1: inversion function `E_k = x_k y_k + Σ_{i∈M(k)} a_{ki} s_i + q_k`로 각 syndrome에 adjustment factor `a_{ki}` 추가(A-NGDBF).
5. 핵심식: `a_{ki}=λ1 (∃k'∈N(i)\k, E_{k'}^{T-1}<θ1), else λ2` — 직전 iteration `E`로 error node 판정해 부정 영향 syndrome을 억제.
6. 핵심기법2: multi-bit용 적응 inversion threshold `θ2 = η/2 - (η/2m)Σs_i` (`η<0`), 미충족 syndrome 비율에 따라 threshold 자동 조정.
7. 구현트릭: iteration 시작 시 `a_{ki}`를 일괄 계산해 VN별 반복 비교(`n_k`회) 회피, flip 후 `E_k^T=-E_k^T`로 다음 iteration에서 non-error 처리.
8. 복잡도: iteration당 `nd_v+n` 실수 덧셈, `m(d_c-1)` XOR, `2n` 곱셈, `2n-1` 비교(S-A-NGDBF) — factorization으로 곱셈 2회.
9. 결과: code1에서 S-A-NGDBF가 BER=1e-5에서 S-NGDBF 대비 0.5dB, S-GDBF 대비 1.2dB 이득; M-A-NGDBF는 MNI=50에서 BER 1e-5(M-NGDBF는 90 iter 필요), 수렴 최고.
10. 한계: AWGN·MacKay 부호만, 파라미터(θ1,η,λ1,λ2) 부호별 실험 튜닝, HW 미설계, NAND soft-read 미검증.
