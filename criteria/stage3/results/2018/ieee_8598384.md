### B. 알고리즘 요약

1. 대상은 polar code `(N,K)`, 실험은 `(64,32)`, `(512,256)`, `rate=1/2`, AWGN 채널.
2. SC/SCL 대비 병렬성 좋은 BP 디코더를 쓰되, BP의 `h(a,b)=ln((1+ab)/(a+b))` 연산이 무거운 문제를 min-sum 근사로 완화.
3. 채널: AWGN, LLR 초기화 `L=ln(P(y|x=0)/P(y|x=1))`, frozen bit는 `R=+∞`.
4. 기법1 OMS: `ho(a,b,β)=sign(a)sign(b)·max(min(|a|,|b|)-β,0)`, node·iteration별 offset `β`를 다르게 적용.
5. offset `β`는 곱셈 없이 덧셈만 필요 → MSMS(곱셈 scaling) 대비 HW 친화적.
6. 기법2 SOMS: scaling `α`와 offset `β`를 함께 적용, `L=α·ho(...)` - 성능↑ 복잡도↑.
7. HW 친화 변형: `ho`를 `gReLU(min(|a|,|b|)-β)`로 재작성해 factor graph를 그대로 DNN feed-forward로 전개(식 12/13).
8. 학습: factor graph를 T iteration DNN으로 펼치고 iteration별 `α,β`를 mini-batch SGD(Adam, `η=0.001`, batch 120)로 cross-entropy loss 최소화, all-zero codeword+AWGN으로 데이터 생성.
9. 결과: OMS는 MSMS와 유사 BER, SOMS는 모든 기존 BP 계열 능가(고 SNR에서 뚜렷), 4~5 epoch만에 파라미터 수렴, iteration=5.
10. 한계: HW 미설계(future work), 순수 시뮬(Python/TensorFlow), polar code 전용 구조, 추가 계수 저장 메모리 필요.
