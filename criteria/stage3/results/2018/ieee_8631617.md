### B. 알고리즘 요약

1. 대상: (1024,512) polar code, BPSK, AWGN. BP factor-graph 위에서 LLR 메시지를 좌우 양방향(`L`,`R`)으로 반복 갱신하는 복호(min-sum 근사 `g(x,y)≈α·sgn(x)sgn(y)min(|x|,|y|)`).
2. 문제: 고정 `Imax` 반복은 유연성 부족·고복잡도, 기존 guessing 기반 조기종료는 BCB·메시지 스케줄을 바꿔 HW-비친화적.
3. PLA 조기종료: Stage 2의 `L2` 메시지가 최근 두 iteration에서 안정화(`mse<eps`)되면 종료 → 좌측노드 판정을 매 iteration 반복(Step iii)하지 않아 latency·중복연산 절감.
4. 핵심 식: `mse = |L2^(t) - L2^(t-1)|^2` 최대값이 `eps` 미만이면 수렴 판정 → 최좌 노드 부호판정 반복 제거.
5. TB-LLR belief enhancing: 노드 belief가 임계 `T` 초과/미만이면 `±∞`로 극단화 → 수렴 미완 key belief를 조기 확정해 오판정 감소(0.35dB@BER=1e-3).
6. SD-CRC belief enhancing: 프레임을 정보비트 K/4씩 4 segment로 분할, 각 segment에 CRC-4 적용해 통과 segment의 비트를 `±∞`로 고정(0.2dB 이득).
7. 구현 디테일: min-sum 근사 `g` 사용, segment 분할은 균등이 아니라 정보비트(집합 A) 기준 K/4 카운트로 경계 결정.
8. HW: BP decoder core + control module(terminating/enhancing unit) + instruction module 구성의 일반 아키텍처 제안(fixed/flexible 구조, mode·Vpre·Imax 명령어 인코딩).
9. 결과: SNR=3.5dB에서 iteration LSA 82%·PLA 73% 감소(평균 iter 5.4/8.0), BER/FER 손실 없음, belief enhancing으로 0.2~0.35dB 이득.
10. 한계: HW는 블록도 수준(RTL/VLSI·gatecount·throughput 미제시, future work), 기법 전체가 polar code 극화·frozen-bit 구조에 종속.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PLA/LSA 조기종료 (LLR 안정성 기반) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | Prime ECC는 CRC 기반 조기종료를 이미 보유; 논문의 LLR-안정성 종료는 polar BP factor-graph 전용이라 직접 이식 불가 |
| SD-CRC segment 조기종료 | partial_CRC.cpp (4 termination point) | 개념적 유사(부분 CRC)하나 polar 정보비트 분할 규칙에 종속 |
| BP min-sum 근사 g(x,y) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min 근사 형태는 유사하나 polar factor-graph 갱신식이라 LDPC Tanner graph에 대응 안 됨 |
| belief enhancing (LLR 극단화) | 대응 없음 | Prime ECC 적응형 LLR 테이블과 목적 상이(polar frozen-bit 수렴 촉진) |
```
