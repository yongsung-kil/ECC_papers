### B. 알고리즘 요약

1. 시스템: 자기기록 채널용 concatenation 부호 — LDPC(4428,4920) + DC-free k-제약 변조부호, AWGN 채널 시뮬.
2. 문제: 기록시스템은 ECC와 제약부호(run-length/spectrum)를 결합해야 하며, 고rate·저복잡도·낮은 error propagation·낮은 error floor를 동시에 요구.
3. 채널 모델: AWGN, 수신신호를 복호 후 k-제약 해제·descramble로 원본 복원.
4. 핵심 기법 1단: nibble replacement 기반 k-제약 부호화 — `(n-1)` user bit을 pivot nibble+`(L-1)` nibble로 분할, all-0 nibble을 pivot nibble로 치환하고 주소·값을 저장.
5. 핵심 식 의미: nibble flipping — `Ui`가 all-1이고 직전 nibble 마지막 bit `Ui-1(q)=1`이면 nibble 전체를 flip해 최대 run-length `k` 보장.
6. 핵심 기법 2단: user part는 guided scrambling(GS)으로 `2^p` scramble 후보 중 최소 RDS 코드워드 선택해 DC 억제.
7. 구현 디테일: parity part는 parity-NRE(31,32) k-제약 + (4,6) balance code(4bit→6bit 매핑 테이블)로 처리, 복호는 Euclidean distance 기반.
8. 최적화 절차: GS redundant bit 길이 `p`로 spectrum null 깊이 조절.
9. 결과: spectrum null > -50 dB 확보, 선행기법 DCRLL5054-LDPC 대비 BER `1e-4`에서 약 0.4 dB 이득.
10. 한계: 자기기록 채널 특화(DC-free/run-length 제약), HW 미설계, 시뮬만, LDPC 디코더·부호설계 자체 개선은 아님.
