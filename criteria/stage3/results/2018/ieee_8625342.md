### B. 알고리즘 요약

1. 시스템: block fading 채널, BPSK, (3,6)-regular LDPC(`Nv=8000`, rate 0.5), 블록당 pilot `P=3` + codeword `M=32` 심볼, `q=5`bit ADC.
2. 문제: 고정밀 ADC/부동소수 수신기는 복잡도·전력이 크고, fixed-point 근사는 성능 손실이 크다.
3. 모델: ADC 출력을 표현값이 아닌 unsigned integer `z_k`로 두고, 상호정보 `I(T;X)`를 최대화하는 압축 mapping `t=f(y)`만 사용(machine learning 유래 Information Bottleneck).
4. 핵심 기법: 채널추정·검출·복호 모든 신호처리를 산술 없이 정적 lookup table 조회로 대체. KL-means IB 알고리즘으로 오프라인 설계.
5. 핵심 식: mapping은 `p(t|y)`가 `I(T;X)`를 카디널리티 제약하에 최대화 — 표현값이 아닌 "관련 정보량"을 보존.
6. 확장(opened node): P입력 lookup을 2입력 lookup 직렬로 분해해 테이블 크기를 `2^{q·P}`(지수) → `(P-1)·2^{2q}`(선형)로 축소(예: 10^9 → 5120 엔트리).
7. 구현 디테일: 결정피드백에서 신뢰도 높은 `N_FB` 비트를 골라 ADC 정수를 부호에 따라 mirror(식 4) 후 feedback 채널추정 lookup에 투입, `iFB`회 반복.
8. 최적화: 모든 테이블은 `Eb/N0=5.65dB` 단일 조건에서 offline 생성해 mismatched 사용(재설계 불필요).
9. 결과: 총 lookup 메모리 <0.3MB(최대 8bit 정수), 제안 수신기 BER이 double precision 대비 waterfall에서 ~0.2dB 손실, 8bit fixed-point(Q5.3) 대비 우위, iFB=5에서 double과 거의 동일.
10. 한계: 무선 fading 수신기 시뮬만, HW 미설계, LDPC 디코더는 [4] 재사용(본 논문 신규 아님), 채널추정이 주 기여.

```
| IB LDPC 디코더(정수 lookup, [4] 재사용) | decoder.cpp VN_Cal_HD() 양자화 / ecc_data.h PARAM_LLR | 우리는 min-sum+LLR테이블, lookup 전면대체는 구조 상이 |
| 채널추정/검출 자체 | 대응 없음 (Prime ECC는 채널추정 블록 없음) | NAND 컨트롤러엔 fading 채널추정 단계 부재 |
```
