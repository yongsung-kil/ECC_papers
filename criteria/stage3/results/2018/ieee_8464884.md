### B. 알고리즘 요약

1. 대상: binary linear block code(주로 dense BCH blocklength 63, RS binary image length 60/155)의 ML 디코딩. BPSK/binary memoryless 채널, FER Monte-Carlo.
2. 문제: ML 디코딩은 NP-hard로 blocklength 증가 시 지수복잡도 — 대형 부호의 최적 성능(FER) 시뮬이 수십 시간 소요.
3. 가정: 동일 부호 `C`의 서로 다른 parity-check matrix는 kernel이 같아 등가 — H의 1 개수(density)를 줄여도 부호는 불변, ML 최적성만 보존.
4. 핵심기법: (MinOnes) — H의 전체 1 개수 `Σ Hij`를 최소화하는 문제를 정수계획(IP)으로 정식화, `O(n³)` 제약. NP-hard임을 minimum distance 문제 환원으로 증명.
5. 식 의미: H의 각 행을 dual code `C⊥`의 basis로 보고, `Hold`의 각 행이 새 H 행들의 span에 속하도록(등가성) 제약 → 1이 최소인 등가 H 탐색.
6. 확장: lower bound — `n−k`개 최소 non-zero dual codeword weight 합을 IP(CodeDistDual)+forbidden-set 부등식으로 계산해 휴리스틱 최적성 검증.
7. 구현 디테일: row-wise 휴리스틱(Algorithm 1) — 각 행 i에 다른 행들을 F2 가산(xli)해 1을 최소화하는 서브 IP `(Pi)`를 Gurobi(12s/IP 제한)로 순차 해결.
8. 학습/최적화: Gurobi(MIPFocus=1, Heuristics=0.5), 별도 학습 없음.
9. 결과: 1 개수 최대 50% 감소, 여러 BCH/RS에서 lower bound와 일치(최적 입증), ML(branch-and-bound/LP/Gurobi) 런타임 최대 81% 단축((63,39) BCH 37h40m→8h20m), (155,115) RS ML FER 최초 산출.
10. 한계: HW 미설계, dense 대수부호 ML 시뮬 가속 목적이며 LDPC엔 추가 최적화 불가(본문 명시), BP/min-sum 성능 영향은 미평가(향후과제).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix 1-최소화(등가 PCM 전처리) | `ecc_top.cpp` `Load_PCM()` | Prime은 sparse QC-LDPC 고정 로드 — 이미 sparse라 dense부호용 최소화 대상 아님 |
| ML(branch-and-bound/LP) 디코더 | 대응 없음 | Prime은 min-sum iterative BP, ML/LP 디코더 아님 |
| dual codeword 최소거리 IP | 대응 없음 | 부호설계/디코더 최적화 도구 아닌 오프라인 분석 |

적용 가치: **낮음** — dense BCH/RS의 ML 시뮬 런타임 단축용 오프라인 전처리이며 본문이 LDPC엔 추가 최적화 불가함을 명시. NAND sparse QC-LDPC + min-sum 디코더와 부호·복호방식 모두 달라 이식 가치 없음.
