### B. 알고리즘 요약

1. 대상: AWGN+BPSK 채널의 일반 binary LDPC(IEEE 802.16e rate 1/2 (360/960/2304), CCSDS rate 1/2·2/3·4/5), SPA(BP) 복호.
2. 문제: flooding은 수렴 느리고 error-floor 높으며, residual 기반 IDS(VC-RBP)는 성능은 좋으나 모든 edge residual 계산으로 `ρMN` 비교 연산 → HW 복잡도 과다.
3. 가정: 채널에서 얻은 사전 LLR `L_n(0)=2y_n/σ²`의 절댓값(ABS-LLR-CH)이 VN 신뢰도를 근사 표현한다.
4. 핵심기법: `Lvalue_threshold`를 최소 ABS-LLR-CH로 초기화하고, `|L_n(0)| ≤ threshold`인 저신뢰 VN만 골라 LBP식 layered 업데이트를 수행.
5. 식 의미: residual `r=|L(l)(Q)−L(l-1)(Q)|` 대신 채널 고정값 `|L_n(0)|`을 기준으로 써서, iteration마다 재계산 없이 N번 비교만으로 노드 선택.
6. 확장: layered stepping value `∂>0`로 매 iteration threshold를 증가 → 신뢰도 낮은 VN이 먼저·더 많은 iteration을 받는 계층(layer) 형성.
7. 구현 디테일: 초기 iteration에서만 VN 불평등 처리, threshold가 max ABS-LLR-CH에 도달하면 전 VN이 동등 업데이트.
8. 학습/최적화: 별도 학습 없음(고정 기준·threshold 스케줄).
9. 결과: BER 10⁻⁵에서 flooding 대비 코딩이득 약 0.15~0.35dB, VC-RBP 대비 유사 성능이나 비교연산 `ρMN→N`으로 대폭 감소, (960,480)에서 ~15 iteration 수렴.
10. 한계: HW 미설계, min-sum 아닌 SPA 전제, 결과가 그림 기반 시뮬만이고 gate/throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ABS-LLR-CH 기반 VN 선택·layered 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 노드 선택 스케줄을 루프에 삽입 가능하나 순차 선택이라 z=32 병렬과 충돌 |
| 채널 사전 LLR `L_n(0)` 활용 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 이미 iteration별 LLR threshold 테이블 보유 — 개념 부분 중복 |
| VN 업데이트 | `decoder.cpp` `VN_Cal_HD()` 등 | 선택된 VN만 업데이트하도록 스케줄 변경 대상 |

적용 가치: **낮음** — 채널 LLR 기반 동적 스케줄 아이디어는 유용하나, edge/노드 단위 순차 선택이 z=32 row병렬·pipeline HW와 근본적으로 상충하고 SPA 전제라 min-sum 코드에 그대로 이식 불가.
