### B. 알고리즘 요약

1. 대상은 network coding용 **nested protograph LDPC** — `12×16` base matrix를 `M=305`로 lifting해 `3660×4880` PCM 생성, global rate `R=0.5`, subcode `R1=R2=0.375`.
2. 문제: 좋은 waterfall에도 불구하고 1.5~2.0 dB부터 error-floor가 발생(BER `1e-5`~`1e-6`에서 flatten), 광통신·데이터 저장 등 저 BER 요구 앱에 부적합.
3. 채널/복호 모델: BPSK on binary-input AWGN, `sum-product(SPA)`, 최대 100 iteration.
4. 핵심 기법: 시뮬레이션으로 dominant error location sequence `E={322,522,582,1969,2055,2061}`를 식별 — 이는 `(6,0) absorbing set`(Hamming weight 6, 12-cycle)에 대응.
5. 이 object가 SPA의 오류 정정을 막으며(BEC에선 stopping set), 고SNR에서 수집 오류의 23~93%를 차지 → 제거 대상 확정.
6. 핵심 기법 2단: **graph shortening** — AS에 속한 variable node 1개(예: 582)와 인접 edge를 모든 nested 부호에서 제거해 object를 '깨뜨림', rate는 `0.5→0.4999`로 극소 감소.
7. 구현 디테일: 노드 제거 시 degree-1 노드나 더 작은 신규 AS 생성을 피하도록 주의 — subcode마다 관여 check node가 다름.
8. 다단계 확장: 2차로 `(5,1) AS` `E2={490,1368,2866,3825,4235}` 제거(node 4235), rate `→0.4998`. 필요 시 반복 적용 가능.
9. 결과: 1차 수정으로 2~3 dB error-floor 영역에서 global·subcode 모두 BER/FER 유의미 개선, rate 손실은 극소. 2차는 개선폭 작음(dominant error 다중도 2~9%).
10. 한계: HW 미설계, 시뮬만, AWGN·SPA 전제, nested 구조·highly irregular 코드에 특화(structured regular array code엔 효용 적다고 명시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dominant absorbing set 식별(시뮬 기반 오류 위치 수집) | [corner.cpp](../Prime_ECC_3.1_Claude/) (수렴실패 추적) | 수렴 실패 error location 추적에 활용 가능, 그러나 AS 분류 로직은 미보유 |
| variable node 제거(그래프 shortening)로 H-matrix 수정 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | H-matrix 로드 단계에서 shortening 반영 가능하나 QC 구조 고정 재검증 부담 |
| nested/protograph 부호 구성 | 대응 없음 | Prime ECC는 단일 고rate QC-LDPC(부호 고정), nested 개념 미대응 |
| SPA 복호 성능(error-floor) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 우리는 min-sum(SPA 아님) — error-floor 지표 비교 가능하나 복호 알고리즘 상이 |

**적용 가치**: 낮음 — absorbing set 제거로 error-floor를 낮추는 발상은 참고할 만하나, nested·protograph·SPA 특화이고 고정된 QC-LDPC H-matrix를 shortening하는 것은 재검증 부담이 커서 Prime ECC로의 직접 이식 가치는 제한적.
