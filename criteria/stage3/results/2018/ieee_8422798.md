### B. 알고리즘 요약

1. 시스템: 920MHz ISM 대역 uplink-only STAR IoT 무선, GPS로 시간·주파수 캘리브레이션, π/2 BPSK(6.35kHz) + linear chirp 확산(154kHz 점유).
2. 문제: 소량 데이터(102bit)를 초장거리·초저SNR·짧은 전송시간(<0.4s)으로 신뢰 전송, 듀티사이클 규제 만족.
3. 프레임: payload 102bit + device ID 24bit → 24bit CRC 붙여 150bit, rate-1/4 LDPC parity 450bit → 600bit Sub-frame, 3회 반복 + 600bit Sync → 2400bit Frame.
4. LDPC 설계: AWGN 최적화 rate 1/4, `N=600`, degree spectra 최적화, PCM 내 short girth 제거 후 5만+ 부호 실현 BER 시뮬로 최종 선택.
5. 부호 비교: IEEE802.15.4k CC(1/2), Turbo(UMTS/LTE) 대비 최대 coding gain, 실사용역에서 error floor 없음.
6. 채널추정: 균등분포 Sync 심볼을 FFT→저주파·대진폭 성분 선택→역FFT로 `m(t)` 추정, 위상왜곡 `φ(t)` 보정.
7. TX: GPS 좌표를 payload로, 1PPS로 TCXO 주파수오차(±1ppm)를 PLL로 보정해 반송파오차 ±20Hz(±0.02ppm) 달성, 20mW 송신.
8. RX: GPS 기반 pseudo-random 함수(pf1,pf2)로 정확 timing·주파수 추출, SDR로 4 Frame·3 Sub-frame 누적 후 LDPC 복호.
9. 결과: 감도 -144dBm(4-Frame 결합, link budget 165dB), 도쿄 도심 20km 대부분·최대 53km 복호, 190km/h Shinkansen 이동체 복호 성공.
10. 한계: HW 디코더 미상세, 부호설계는 개요 수준만, 무선 특화 초저rate로 NAND ECC 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| rate-1/4 LDPC 부호설계 (degree spectra, girth) | 대응 없음 | Prime ECC는 고rate 특정 QC-LDPC 고정, 저rate 무선 부호와 무관 |
| LDPC 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 논문에 복호 알고리즘 서술 없음, 대응 불가 |
| Sync 기반 채널추정/위상보정 | 대응 없음 | 무선 물리계층 처리, ECC 시뮬레이터 무관 |
| chirp/π2-BPSK 변조 | 대응 없음 | 무선 변조, ECC 무관 |

적용 가치: **낮음** — 무선 물리계층 시스템 논문으로 부호설계도 고수준 개요뿐이고 초저rate·초단페이로드 동작점이 NAND 고rate QC-LDPC ECC와 정반대라 이식 실익 없음.
