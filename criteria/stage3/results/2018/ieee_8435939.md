### B. 알고리즘 요약

1. 대상은 protograph LDPC 및 DGLDPC(variable/check 노드가 임의 선형 component code를 강제) 부호, 채널은 BEC와 BIAWGN, rate 1/3~1/10 저rate near-capacity 설계.
2. 문제: 저rate에서 near-capacity 부호는 degree-1 VN 비중이 커야 하는데, 기존 block-error threshold 조건(참조 [7])은 degree-1 VN을 불허해 저rate에서 capacity와 gap이 큼.
3. 모델: girth-`g` lifted 그래프에서 protograph density evolution이 iteration `⌊(g-4)/4⌋`까지 정확하다는 가정, edge별 erasure 확률 `x_e^t`(VN→CN), `y_e^t`(CN→VN) 벡터 재귀.
4. 핵심기법1: `x^t = max_e x_e^t`가 `t`에 대해 double-exponential(`O(exp(-β2^{αt}))`)로 떨어지는 edge 집합을 찾는 subgraph `RED(G)` 추출(Algorithm 1: degree-1 VN·degree-2 cycle 재귀 제거).
5. 식 의미: `P_B ≤ n·x^t = O(n·exp(-βn^α))` — bit-error가 double-exp fall하면 block-error threshold가 bit-error threshold와 같아짐(Theorem 1, 2).
6. 핵심기법2(확장): DGLDPC로 일반화 — component code의 최소거리 `d`로 `E1`(d=1),`E2`(d=2 cycle) 정의(Lemma 3,4); degree-2 subgraph에 cycle이 있어도 등식 조건 성립 가능.
7. 구현 디테일: 5G 표준 프로토그래프(rate-1/5 42×52, rate-1/3 46×68)와 PBRL이 본 조건을 만족함을 검증, AR4A는 불만족.
8. 최적화: differential evolution(모집단 |C||V|개 base matrix, mutation factor 0.5, `pc=0.88`, 최대 6000세대)로 block-error threshold 최대화, generalized 노드엔 (7,4) Hamming/dual 사용.
9. 결과: rate-1/8 BEC threshold 0.866(capacity gap 0.009), rate-1/3 BIAWGN 64000-length가 5G 대비 threshold 0.1dB·FER 우위(FER=1e-3에서 rate-1/5 gap 0.463dB vs 5G 0.513dB).
10. 한계: 무한 blocklength DE 이론 + BER/FER 시뮬만, HW·양자화·soft-read 없음; 저rate 통신/보안(wiretap)·5G 대상.
