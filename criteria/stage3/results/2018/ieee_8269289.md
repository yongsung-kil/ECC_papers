### B. 알고리즘 요약

1. 성격: 단일 알고리즘이 아닌 자기기록(MR) 시스템용 LDPC 부호설계·복호·점근분석 방법론을 총정리한 서베이/튜토리얼.
2. 시스템: LDPC 인코더 → BPSK → MR 채널(ISI+AWGN) → turbo 수신기(내부 BCJR/SOVA 검출기 + 외부 log-BP 복호기)로 soft-info 반복교환.
3. 채널 모델: 1차원 OD-ISI(부분응답 `H(D)=(1-D)(1+D)^{κ-1}`)와 2차원 TD-ISI(예 4Tb/in² 3×3 CIR 행렬), memory length가 클수록 밀도↑·ISI↑.
4. 성능척도: 채널용량 대신 SIR(symmetric information rate, i.i.d. 입력 MI)을 Monte-Carlo로 추정해 신뢰저장 하한 설정.
5. 부호 구성: PEG/ACE 기반 H 생성, 선형인코딩 QC-LDPC, protograph(PEXIT), RA, SC(spatially-coupled), 그리고 이들의 non-binary(GF(q)) 변형까지 정리.
6. 부호설계 핵심: DE·EXIT로 복호 threshold, AWD로 linear-minimum-distance(=낮은 error-floor)를 보장하는 부호를 MR 채널에 맞춰 최적화.
7. 검출·복호 알고리즘: BCJR(MAP) 검출기와 저복잡도 변형(GA-IRCSDFA 등), 외부는 log-BP/min-sum, joint detection-and-decoding 스킴 열거.
8. Anti-ISI: precoding은 long-constraint LDPC엔 무효라 불필요, turbo detection(터보 등화)이 핵심(uncoded 대비 ~4~6dB gain).
9. 결과 예시: 4Tb/in² TD-ISI에서 R=8/9, N=4608, non-binary(GF(16)) 및 최적화 protograph/TB-2 부호의 BER 비교(Fig.20) 등 인용 결과 제시.
10. 한계·미래: HW 구현 연구 극소수([82],[185]), polar 부호 대안, NAND flash용 LDPC 설계는 "infancy"라며 future direction으로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ISI+turbo detection(BCJR) 채널/검출 | 대응 없음 | Prime ECC는 AWGN/RBER 채널, ISI·turbo 등화 미지원 |
| QC-LDPC/protograph 부호 구성 (DE/EXIT/AWD) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 설계 층이나 MR·ISI 특화라 이식 부담 큼 |
| non-binary(GF(q)) LDPC/SC 변형 | 대응 없음 | Prime ECC는 binary 전용 — 비바이너리 기법 이식성 하 |
| log-BP/min-sum 외부 복호기 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 교과서 수준 재사용, 서베이 고유 기여 아님 |

적용 가치: **낮음** — 자기기록 ISI 채널·turbo detection·non-binary 부호가 중심이라 NAND binary QC-LDPC(부호설계→디코더→HW)에 떼어 쓸 신규 기법이 없고, 필요 시 서베이가 가리키는 개별 원논문(고rate QC-LDPC FPGA 등)을 별도 검토하는 편이 낫다.
