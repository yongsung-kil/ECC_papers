### B. 알고리즘 요약

1. 시스템: 광 optical recording 채널 + RLL(0,3) 제약, PEG-LDPC rate `0.65`, 길이 `4608/11520/10240`, MAP equalizer + LDPC iterative decoding(`Uo=5`, `Ui=3~5`).
2. 문제: RLL encoder의 rate loss를 피하려 codeword 일부 bit를 deliberate flipping으로 강제, read 측에서 LDPC 정정력으로 flip 제거해야 하나 최적 code 미상.
3. 모델: write측 flipping error가 랜덤 codeword에 의존(symbol-dependent 채널) → all-zero codeword 가정 불가, Monte Carlo로 첫 iteration pdf 측정.
4. 핵심기법1: min-sum 기반 DE 유도. flip error 존재/비존재 메시지를 `Vf`/`Vnf`로 분리해 `φ+f`, `φ-f`, `φ+nf`, `φ-nf` 정의.
5. 핵심식: check node pdf `QL(u)(l)`을 negative value의 홀/짝수 조합으로 계산(식3,4) — flip/non-flip 혼재 채널에서 CN 출력 분포 근사.
6. 확장: 2nd iteration 이후 flip/non-flip 부분 병합, 동일 min-sum DE로 단일화(식7,8).
7. 구현 디테일: variable node는 FFT/IFFT로 수치 계산(식5), CN은 lookup 대신 min-sum으로 단순화.
8. 최적화: differential evolution(Storn-Price)로 degree 분포 global 최적 탐색, 2-level UEP(`Ivweak=[2,3]`, `Ivstrong=[4,5,6,7,...]`, α=0.5, X=50).
9. 결과: `VND=[2,5,7]` δ=[0.5,0.499,0.001](Code 6)가 `[2,5]`(Code 2)보다 우수, check weight-10 유지 + weight-7 추가로 high-SNR BER 개선, error floor 제거.
10. 한계: 광 recording/RLL flipping 특화 채널, HW 미설계, 시뮬/이론뿐, .txt 수식 손상.
