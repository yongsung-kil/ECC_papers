### B. 알고리즘 요약

1. 대상은 DVB-S2 LDPC (`N`=16200, 64800, rate 1/4~9/10), circulant 크기 `S`=360.
2. 문제: 기존 GPU 디코더는 inter-codeword 병렬로 고정 iteration 수행 → 유효 codeword 도달 후에도 불필요 연산, 조기종료 불가.
3. 핵심: DVB-S2 PCM은 QC 구조가 아니므로 (a) 첫 행 제외 행 permute, (b) parity 열을 360 간격으로 grouping해 QC 구조(circulant/superimposed circulant)로 변환.
4. 인코딩: systematic 코드에서 `p = -H_p^{-1} H_i m`을 "interleaved cumulative sum" 구조로 풀어 GPU 병렬화(3개 커널: computeGamma, transposeGamma, cumSum).
5. circulant는 32bit word(행 degree/열 위치/우측 rotation)로 constant memory 저장.
6. 디코딩: intra-codeword 병렬, 한 codeword씩 offset min-sum + TDMP(layered) 스케줄로 처리.
7. HW 친화 트릭: `λ`를 fp16(half)로 저장, 연산은 fp32; branchless min-mag 계산으로 warp divergence 제거; superimposed circulant는 중복분(`L_m`) 별도 감산.
8. 조기종료: 매 iteration 후 checksum 검사, early-termination flag/checksum을 volatile __shared__로 스레드 간 공유.
9. 결과: 인코더 multi-Gbps, 디코더 rate≥1/2에서 BER<1e-9시 500Mbps~1Gbps, `Eb/N0` 0.1dB당 약 100Mbps throughput 증가(조기종료 효과). min-sum offset 0.125.
10. 한계: GPU(Titan Xp) 전용, RTL/ASIC 설계 없음, 저rate 코드는 BCH 병행 필요(error floor inflection).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 연산(min1/min2) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유 min-sum과 동일 범주, 신규성 낮음 |
| TDMP(layered) 스케줄 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | Dual-Update 유사, 스케줄 아이디어만 참고 |
| 조기종료(checksum) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 우리는 CRC 기반, 논문은 parity checksum — 개념만 겹침 |
| QC PCM 재배열/circulant 로드 | ecc_top.cpp Load_PCM() | DVB-S2 전용 변환이라 우리 고정 QC-LDPC에 무관 |
| superimposed circulant 중복 감산 | 대응 없음 | DVB-S2 특유 구조, 우리 base matrix에 없음 |
| GPU 커널/fp16 저장 | 대응 없음 | HW(RTL) bit-exact 모사와 무관 |
```

적용 가치: 낮음 — 알고리즘 요소(offset min-sum/TDMP/조기종료)는 이미 보유, 구현은 GPU/DVB-S2 특화라 NAND binary QC-LDPC HW에 떼어 쓸 것이 사실상 없음.
