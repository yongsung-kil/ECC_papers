### B. 알고리즘 요약

1. 802.16e/802.11n QC-LDPC 코드(N=648~2304, rate 1/2~5/6, Z=27~96) 대상 FPGA 디코더 생성 사례연구.
2. 문제: 기존 HLS 기반 LDPC 디코더는 manycore(OpenCL/CUDA, GPU 유래) 기술을 재사용해 HW 복잡도가 과도.
3. 제안: multicore(x86 SIMD) 지향 소프트웨어 기술이 HW 생성에 더 적합함을 실증 - Vivado HLS로 RTL 자동 생성.
4. 디코더 알고리즘은 horizontal layered(TDMP) Min-Sum (`Algorithm 1`), CN 연산에서 `|Lmn|=min` sign XOR.
5. 병렬화 1: inter-frame - Q 프레임을 병렬 복호, 단일 저비용 컨트롤러, throughput이 Q에 선형 비례하나 메모리 비용 큼.
6. 병렬화 2: intra-frame - QC 구조가 보장하는 Z개 독립 CN을 병렬 처리, 배럴시프터 인터리버 필요, N/Z에 따라 latency 가변.
7. HLS 변환: SystemC 캡슐화, 동적→정적 메모리, x86 SIMD 명령 라이브러리(~300줄) 자작, pipeline pragma 삽입.
8. 양자화: VN 8bit, CN2VN 메시지 6bit로 floating-point 근접 성능.
9. 결과: intra-frame 26~124 Mbps(Virtex-7, 10 layered iter), 관련 HLS 연구 대비 HW 복잡도 2~10배 낮고 iteration 2배 적음.
10. 한계: 통신표준(WiFi/WiMAX) 대상, 부호설계·정정능력 개선 없음, NAND/SSD 무관, throughput도 Gbps 미달.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| horizontal layered Min-Sum 디코딩 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Min-Sum은 이미 보유, 중복 |
| intra-frame(QC Z-병렬) HW arch | 디코더 HW 모델 (z=32) | 병렬화 개념 유사하나 HLS 생성 방식은 우리 RTL 구조와 무관 |
| 8b/6b 양자화 | [decoder.cpp](../Prime_ECC_3.1_Claude/) VN/LLR 테이블 | 양자화 개념 참고 가능하나 우리 HD/2SD/3SD 테이블과 다름 |
| HLS 툴체인 / SystemC 생성 | 대응 없음 | Prime ECC는 bit-exact C++ 시뮬, HLS 방법론 무관 |

적용 가치: **낮음** - Min-Sum/layered/QC 병렬화는 이미 보유한 표준 기법이고, 논문 핵심은 SDR용 HLS 자동생성 방법론이라 NAND ECC 부호설계·정정성능에 이식할 요소가 없다.
