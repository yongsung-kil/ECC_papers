### B. 알고리즘 요약

1. 대상: binary cyclic/BCH 부호(예 [63,·], [255,206], LDPC 아님), BPSK/AWGN 채널, locality(LRC: 각 좌표가 저가중 local parity로 복구)를 복호 개선에 활용.
2. 문제: ML 복호는 길이 증가에 지수 복잡, OSD/trellis는 준최적 — locality를 주입해 성능↑·복잡도↓.
3. 접근1 (locality-aware OSD): OSD 시작 전에 local parity-check만으로 BP 1라운드 `Λ^(1)=2 tanh^{-1}(∏ tanh(Λ/2))+Λ^(0)`를 돌려 신뢰도(reliability)를 갱신 후 OSD 재처리.
4. Quick-Look ML(QL-MLD): disjoint local code(단일 parity-check 또는 middle code)를 독립 ML 복호, 합쳐 유효 codeword면 조기종료 — 중고SNR에서 ML을 자주 반환(예 n=63,nl=7: 3.5dB에서 82% 성공).
5. 확장: hierarchical locality/availability(t개 orthogonal parity), overlapping locality는 GDL(junction tree message passing)로 LLR 계산.
6. 접근2 (trellis decoding): cyclic 부호의 zeros에 locality train을 추가해 subcode `C_new`를 만들어 특정 레벨의 state complexity `s_i=log_q|V_i|`를 감소.
7. 이론: 부호의 zeros만으로 state complexity profile 상계(Theorem 14)를 유도, Vardy-Beery 결과를 hierarchical locality로 일반화.
8. 부호 구성: `(r+1)|n` zero 패턴이 r-all-symbol locality를 부여(Theorem 8), 예 [255,192] r=16 부호를 zeros 추가로 생성.
9. 결과: locality-aware order-0 OSD가 conventional order-2 OSD를 고SNR에서 능가(Fig.2,3,5), QL-MLD로 평균 복잡도·지연 감소(연산량 real ops로 정량화).
10. 한계: HW/FPGA 미설계, throughput/gatecount 없음, 시뮬레이션(연산량 카운트)뿐, LDPC 부호 대상 아님.
