### B. 알고리즘 요약

1. 대상 부호는 `(1024,512)` polar code, rate 0.5, 5G 채널 대상 BP 디코딩.
2. 문제: polar BP는 만족 error-rate 위해 대량 iteration(최대 80) 필요 → 복잡도 과다, 조기종료 필요.
3. polar는 생성행렬 `G_N = B_N G_2^⊗n`로 정의되어 패리티체크행렬이 없음 → syndrome 기반 종료 불가가 원 문제.
4. 핵심: frozen bit를 all-zero로 두고 정보열만 남긴 `G(k×n)`를 Gaussian elimination으로 `[P | I]` systematic화, 그로부터 `H = [I | P^T]` 유도.
5. 매 iteration마다 경판정 codeword `x̂ = sign(L+R)`에 대해 `syndrome = x̂ H^T` 계산.
6. syndrome이 all-zero면 legal codeword로 간주 → iteration 즉시 종료, `x̂`에서 `û` 역산.
7. 부수 효과: 종료 없이 max까지 돌면 수렴 후 오히려 오류로 이탈하는 경우가 있어, 조기종료가 BER도 개선.
8. all-zero syndrome이 전송 codeword를 보증하진 않음(undetectable error) — 그러나 시뮬상 그 확률이 오이탈 확률보다 훨씬 낮음.
9. 결과: max_iter=80, Eb/N0=3dB에서 평균 iteration 80→약 23, 연산량 약 70% 감소. Case 3(H-matrix 오판이면서 원BP 정답)은 0건.
10. 한계: HW 미설계, 시뮬만, polar 전용이며 gate/throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 조기종료 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | LDPC는 이미 CRC 조기종료 보유 — polar용 H유도는 불필요 |
| BP iteration 루프 | decoder.cpp LDPC_Decoder() | polar BP는 factor-graph PE 구조로 LDPC min-sum과 상이, 직접 대응 아님 |
| 경판정 codeword 판정 | 대응 없음 | polar의 x̂=sign(L+R) 산출은 LDPC와 무관 |
```

적용 가치: **낮음** — polar code 전용 기법이며 LDPC는 이미 CRC 기반 조기종료를 보유하고 부호구조도 달라 이식 이득이 없음.
