> 총평: SC-LDPC 윈도우 디코더의 stall 회피용 적응형 스케줄링으로, block-based binary QC-LDPC인 Prime ECC엔 구조 전제가 달라 이식 가치 낮음.

### B. 알고리즘 요약

1. 시스템: unit-memory SC-LDPC (`R≈0.8`, `dv=5,dc=25`, sub-matrix `H0/H1` 각 `960×4800`, replication `L=99`), AWGN, 윈도우 디코딩(window size `w=9`, iteration `I=3`).
2. 문제: full BP는 SC-LDPC에서 iteration·latency 과다 → 윈도우 디코딩으로 완화하나, 윈도우를 미완성 위치에서 shift하면 이후 블록이 연쇄 실패하는 decoder stall(버스트 에러) 발생.
3. 모델: BER가 spatial position을 따라 파도처럼 수렴 — stall은 wave-threshold(2.117dB)와 macro-threshold(2.896dB) 사이 SNR 구간에서만 발생, DE 분석엔 안 나타남.
4. 기법 핵심: stall 검출(parity-check 기반 또는 LLR 기반 soft-BER `Pe,est=1/K·Σ 1/(1+exp(|Lk|))`)을 window 내 3번째 spatial position에서 평가해 robust하게 판단.
5. 의미: 첫 position은 대개 error-free라 검출 신뢰도가 낮고, 3번째 position이 임박한 stall에 민감.
6. 적응형 디코더 3종: (a) AID — stall 검출 시 iteration을 `Imin~Imax`로 증가, (b) WSD — window를 `nb`만큼 뒤로 shift, (c) WTD(Wave Tracking) — wave 위치 따라 window를 전후로 이동해 error-free 위치 갱신 최소화.
7. 구현 디테일: CUDA/GPU 시뮬 프레임워크로 2,000개 stall codeword test-set 생성(SNRb=2.86dB), stall 검출 threshold `δBER=1e-7`.
8. 실험: 특정 block(pos.10)에 인공 noise 주입해 stall 유발 — foresightful stall 예측이 retrospective 검출을 유의하게 능가하지 않음(예측은 오히려 iteration을 더 써 평균 복잡도↑).
9. 결과: Wave Tracking이 최적 BER + 최저 평균 복잡도(2.64·w), fixed I=4 대비 평균 복잡도 34% 감소하면서 BER 성능 유지.
10. 한계: SC-LDPC 전용, 광통신/streaming 타깃, HW 미설계, block-based 코드엔 window/wave/stall 개념 자체가 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 윈도우 디코딩 window shift/stall 스케줄 | [decoder.cpp](Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | 낮음: Prime ECC는 block-based QC-LDPC라 window/spatial position 개념 부재 |
| LLR 기반 soft-BER 조기종료 판단 | [partial_CRC.cpp](Prime_ECC_3.1_Claude) `partial_crc_HW_T4()`, [full_CRC.cpp](Prime_ECC_3.1_Claude) | 낮음: 우리는 이미 CRC 기반 조기종료 보유 — soft-BER 추정은 대체 아이디어일 뿐 |
| Wave Tracking / SC-LDPC 구조 | 대응 없음 | SC-LDPC convolutional 구조는 Prime ECC 부호설계와 무관 |

적용 가치: **낮음** — 기법의 핵심(window shift·wave tracking·stall 회피)이 SC-LDPC의 spatial coupling 구조에 종속되어, block-based binary QC-LDPC ECC 엔진인 Prime ECC엔 떼어낼 재사용 단위가 없다.
