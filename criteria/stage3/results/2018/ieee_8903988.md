### B. 알고리즘 요약

1. 대상 부호: WiMAX 표준 irregular LDPC, rate `1/2`, block length `N=2304` (`K=1152`), AWGN 채널, soft 수신 심볼 입력.
2. 문제: 다중 error-correction factor를 쓰는 min-sum이 VN degree 분포에 맞지 않게 갱신되면 수렴 속도 저하·magnitude 과대추정·error-floor 상승.
3. 초기화: 각 bit node를 `Λv=2rv/σ²`로 세팅, syndrome `Sv=Σ h·ĉ`로 하드결정 벡터 산출(Step 1).
4. CN 갱신(Step 2): 기존 offset min-sum에 추가 scaling factor `δ`를 곱한 형태 `α = ψ·max(δ·min|Λ| − λ, 0)`, 부호는 `ψ = Π sgn(Λ)`.
5. 의미: `δ`(adaptive scaling)는 "가장 불안정한 bit"가 생길 때만 활성화되어 unreliable bit를 reliable로 전환, 평시엔 unity.
6. threshold 분기: syndrome=0 경로에서는 adaptive threshold offset `ξ`를 사용, `ξ`는 stifling factor `Φ`와 threshold `T` 비교로 적응 선택(식 6).
7. VN 갱신(Step 3): 노드를 converged/non-converged로 분류(sparseness 이용), adaptive variable offset `θ`를 적용해 `βv` 갱신(식 7,8).
8. 조기종료(Step 4): 하드결정 `ĉ`가 `ĉ·Hᵀ=0`이면 종료, 아니면 Step2~3을 max iteration까지 반복.
9. 결과: iteration limit 12에서 BER 10⁻⁶ 기준 DOMSA 대비 0.16dB, I-POMSA 대비 0.1dB, RCMSA 대비 0.08dB coding gain, BPA와는 0.02dB 차; waterfall에서 평균 iteration 감소.
10. 한계: HW 미설계, 시뮬만, gate count/throughput 무보고, 그림에만 있는 BER/iteration 곡선 수치는 본문 서술 외 미상.
