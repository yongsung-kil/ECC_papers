### B. 알고리즘 요약

1. 대상: One-Step Majority-Logic Decodable(OSMLD) 블록부호를 BIBD/PBIBD(2-design) incidence matrix로 구성, BPSK/AWGN 채널.
2. 문제: 기존 OSMLD 부호가 커버하지 못하는 rate·length 조합의 부호를 새로 확보.
3. 구성 소스: 사영기하 `PG(2,q)`에서 유도한 unital design `2-(m³+1, m+1, 1)`와 oval design을 블록 디자인으로 사용.
4. 핵심기법: 디자인의 `v×b` incidence matrix `N`을 부호의 parity-check/generator 구조에 매핑, systematic class-1(`H=[N,Ib]`)·class-2(dual)·non-systematic 3종 부호족 생성.
5. 부호 파라미터: unital은 `n=m⁴+m²+1`, `d=r*+1=m²+1`; min distance가 디자인 파라미터 `r*,k*`로 해석적으로 결정됨.
6. 복호: Massey threshold decoding의 SISO 확장인 Iterative Threshold Decoding(ITD) 사용, dual code의 J개 orthogonal 벡터로 check-sum 구성.
7. 복호식: 각 심볼 `en`에 orthogonal한 J개 check-sum의 다수결(majority vote)로 오류심볼 추정, HIHO는 zero-threshold 하드판정.
8. 부호표: unital(Table I)·oval(Table II)로 `[21,9,5,R=0.428]`~`[10101,9100,12,0.90]` 등 다수 예제 제시(rate 0.099~0.94).
9. 결과: 최소 1000블록·200잔류오류 몬테카를로로 BER 10⁻⁶까지; class-2가 class-1보다 우수, BER 10⁻⁵에서 약 1 dB 손실 관찰.
10. 한계: HW 미설계, AWGN/BPSK 이론·시뮬만, NAND/저장 채널 무관, min-sum/BP 아닌 majority-logic 복호.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| incidence matrix로 H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 지점은 있으나 QC circulant 아닌 조합디자인 구조라 정합 안 됨 |
| Iterative Threshold Decoding(majority-logic) | `decoder.cpp` `LDPC_Decoder()` | 우리 디코더는 min-sum BP, threshold/majority 복호와 알고리즘 상이 |
| orthogonal check-sum / dual code 복호 | 대응 없음 | min-sum·partial-CRC 조기종료 구조와 무관 |

적용 가치: **낮음** - OSMLD/threshold decoding은 binary QC-LDPC+min-sum 파이프라인과 부호족·복호기 모두 달라 직접 이식 요소가 없다. 조합 디자인 구성법의 orthogonality 개념만 참고 수준.
