### B. 알고리즘 요약

1. 시스템: 32×32 large MU-MIMO uplink, Rayleigh flat-fading, 4~256-QAM, LDPC(802.11n, `rate=0.75`) 부호화, `y=Hs+n`.
2. 문제: SD는 ML 최적이나 dense 성상·다수 스트림에서 복잡도·latency 폭증, soft LLR 추출 비용 큼; MMSE/LAS는 저복잡도지만 성능 열위.
3. 모델: AWGN noise 통계 기반, `Θ` = 최소거리 `Dmin`만큼 떨어진 오검출 후보 벡터 집합.
4. Antipodal 검출: depth-first SD + 통계적 pruning. 레이어별 pruning 경계 `rp(l)=σ²·Fχ⁻²(1−β)` (χ² CDF 역함수, `β`=pruning 파라미터).
5. 의미: PD가 `rp(l)`보다 크면 가지치기 - 단 하나의 해만 통과하면 "고신뢰", 아니면 erasure로 처리(Antipodal 출력).
6. `β` 조정으로 erasure 확률 `PEr`을 임계 `τE` 이하로 맞춰 디코더 오류내성과 동기화; latency는 `αNt` 방문노드로 모델링, 복잡도 `O(αNt²)`.
7. Antipodal 디코더: 검출기 출력 d∈{0,1,e}에 특화된 BP 변형, 내부 연산을 binary addition만으로 단순화.
8. Erasure 복원: parity check `Ve=Σ Vi`로 resolvable check node 순회하며 erased 비트 복원(리스트 `Le` 반복).
9. 비트정정: 단일 비트오류 가정(`Pτ≈B·τ`), exposed/hidden 오류 구분 - doubtful 노드를 erasure로 바꿔 재복원 시도로 hidden 오류 위치 탐색.
10. 결과: 32×32 64-QAM에서 soft-MMSE 대비 9dB 전력 절감, throughput 56% 향상; 한계 - HW 미설계, 시뮬만, MIMO erasure 채널에 강결합.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 기반 erasure 복원/비트정정 디코더 | decoder.cpp LDPC_Decoder() | 낮음: erasure(3값) 입력 전제, min-sum LLR 파이프라인과 구조 상이 |
| SD 통계적 pruning 검출기 | 대응 없음 | MIMO 검출 단계로 NAND 채널에 대응 요소 없음 |
| parity check 기반 erasure 해결 | decoder.cpp C2V_Cal() | 낮음: BEC-style 복원으로 우리 iterative MP 스케줄과 목적 다름 |
```

적용 가치: **낮음** - MIMO SD 검출기 출력(erasure)에 강결합된 디코더로, NAND binary LDPC의 LLR 기반 min-sum 파이프라인에 떼어 쓸 요소가 사실상 없음.
