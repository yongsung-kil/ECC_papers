### B. 알고리즘 요약

1. 대상은 광통신(BICM + QAM 4~1024-ary) SD-FEC용 QC-LDPC convolutional code(=spatially-coupled), protograph `(J,K,L,M)`로 표기(J=최대 열무게, K=최대 행무게, L=coupling termination 길이, M=QC lifting).
2. 문제: irregular LDPC 최적 degree 분포가 target overhead에 종속되어 한 부호가 여러 overhead(16QAM에서 27~189%)에 scalable하지 않음.
3. 저지연 windowed decoding(WD, window size `W`)을 사용, 창 안의 `W·M` CN을 M 크기 layer로 나눠 스케줄링(layer 크기·window stride = M).
4. 핵심 기법1: 창 내부 `W`개 layer의 irregular 복호 순서를 P-EXIT(protograph EXIT) 분석으로 greedy 최적화해 수렴속도 개선(추가 복잡도 없음); reverse RR가 고overhead에서 거의 최적.
5. 핵심 기법2: shortening(frozen info 비트 전송생략, overhead↑)·puncturing(parity 생략, overhead↓)할 proto-VN을 scheduling과 공동으로 greedy tree search 최적화.
6. P-EXIT 초기화: shorten proto-VN은 mutual info=1(error-free), punctured proto-VN은 0(unreliable)로 두고 FEC limit 최소화.
7. HW 친화성: `LDPC-CC(J,K)`는 `LDPC-CC(J,K+J)`의 uniform shortening으로 얻어져 인코딩/디코딩 HW 수정 없이 rate 조정 가능(가장 낮은 overhead의 mother code 재사용).
8. 결과: joint 최적화가 RR+uniform shortening 대비 0.4dB 이상 이득(W=6,I=2), W=10에서 Shannon limit 0.5dB 이내, overhead 5~110% 전역 scalable.
9. puncturing은 punctured VN에서 mutual info가 0이 되어 shortening보다 성능 낮고, truncated LDPC-CC가 non-truncated 대비 rate loss 완화로 우수.
10. 한계: 성능이 P-EXIT threshold(이론값)뿐이고 BER 시뮬/HW/gatecount 없음, 광통신 windowed SC-LDPC 특화라 block QC-LDPC와 구조 상이.
