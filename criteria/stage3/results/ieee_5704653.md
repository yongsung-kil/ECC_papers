> 총평: 802.16e IRA QC-LDPC용 dual-diagonal 재귀 누적 XOR 인코더(1-clock latency). 우리 인코더도 dual-diagonal 기반이라 개념 유사, 신규성은 register 재활용 스케줄 수준, 이식성 중.

### B. 알고리즘 요약

1. 대상: IEEE 802.16e QC-LDPC, base matrix `B=[B(1),B(2)]`를 expansion factor `n`(24~96)로 z×z shifted-identity 확장, 예시 codeword 2304-bit rate-2/3B.
2. 문제: LDPC 인코딩은 행렬곱으로 code length에 quadratic 복잡도 → HW 부담. 기존 [8] parity 예측·정정 방식을 재귀 누적으로 개선.
3. 구조 가정: `B(2)`가 dual-diagonal 형태 → `HeT = H(1)aT + H(2)rT = 0`에서 parity를 순차 재귀로 유도 가능.
4. 핵심 기법: 시작 parity 벡터 `r0 = Σ H(1)aT` 계산 후, `1≤i<m-1`에서 재귀식 `r_{i+1}=(βi+ri+r0)` 또는 `(βi+ri)` (`B(2)`의 shift 부호에 따라)로 각 parity 블록 산출.
5. 의미: `r0`를 output으로 내보내는 동시에 별도 register에 저장 → 다음 permuted 연산에 재사용, 매트릭스 실저장 없이 XOR 함수 전개로 계산.
6. 마지막 parity: `r_{m-1}=β_{m-1}+perm[d]r0`; HW 효율 위해 `β_{m-1}`은 명시 계산 생략.
7. 구현 디테일: XCH shift register(메시지 96-DFF cyclic 저장) + shift register β/acc 블록, MUX A로 `a*`/`ai` 누적 선택. rate/n에 따라 permutation tap 위치(Pd) 가변.
8. 처리 흐름: 첫 1536 clock에 메시지 적재 → cyclic shift로 전환 → `a*`가 H(1) non-zero tap 합산해 `r0` 출력 → 96 clock마다 `r1,r2,...` 순차 산출.
9. 결과: Design Vision 합성, critical path 약 0.44ns(안정적), total cell area 15k~85k(rate·expansion 선형 증가), 1-clock latency 실시간 동작.
10. 한계: 802.16e base matrix 종속, 공정·throughput 미기재, 디코더/정정능력과 무관한 순수 인코딩 아키텍처.
