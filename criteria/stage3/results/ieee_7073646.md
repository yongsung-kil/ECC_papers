### B. 알고리즘 요약

1. 대상은 binary erasure channel(BEC) 상에서 반복 메시지 전달 복호를 사용하는 LDPC 부호이며, 구조적 결함(stopping set)이 error floor의 주원인.
2. 기존 stopping set 제거 기법(그리디, 휴리스틱)은 최적성 보장이 없고 4-cycle을 새로 유발하거나 새 row의 weight가 과도하게 커지는 문제가 있음.
3. 채널 모델은 BEC(소거확률 `ε`)이며, 복호 실패 확률은 stopping set의 크기 분포(`Di`, dead-end 개수)에 의해 결정된다고 가정.
4. 핵심 기법은 parity-check matrix `H`와 stopping set 리스트 `L`이 주어졌을 때, 리스트 내 최대 개수의 stopping set을 제거하는 새 parity-check row `r`을 찾는 ILP 정식화(ILP1).
5. 지시변수 `δl`을 각 stopping set `Sl`에 대응시켜 `δl=1 ↔ Σ_{i∈Sl} r_i = 1` 조건을 선형부등식(1)-(5)로 표현, 목적함수는 `max Σδl`.
6. 확장으로 redundant row 제약(mod-2 조건, 식 12), row weight 제한(`Σr_j ≤ w_max`), 4-cycle 회피(`Hr ≤ 1`), stopping set 크기 가중(`(ε/(1-ε))^{|Si|}`)을 추가 가능.
7. 부수 트릭으로 knapsack cover inequality를 이용한 LP relaxation 강화(cutting plane)로 ILP 풀이 시간 단축.
8. 최적화 절차는 CPLEX/GUROBI 등 상용 ILP solver로 직접 최적해(또는 near-optimal, timeout 시 incumbent) 탐색.
9. 결과: Golay code(24,12,8)에서 stopping set 110개 중 60개를 단일 row로 제거, 여러 LDPC 예제에서 기존 휴리스틱([8]) 대비 동등하거나 우수한 BLER 개선 확인.
10. 한계: BEC 전용(soft-decision/AWGN 미다룸), 장문 LDPC(대형 H, 큰 L)에서는 ILP 변수 수 폭증으로 실용적 계산시간 확보 어려움, HW 설계 없음.
