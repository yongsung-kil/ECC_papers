### B. 알고리즘 요약

1. 시스템: `L×U`개 GPU SM 중 일부는 정확연산(`I0`), 일부는 근사연산(`I1`); SM 출력 간 비트 일치확률 `P(Yi=Yj)=r`로 상관 모델링.
2. 문제: 근사연산 오류를 에너지 절감 유지하며 정정 — 정확출력과의 상관을 이용해 "복호 문제"로 변환.
3. 모델: 근사출력을 `Ỹi=Yi+n`의 memoryless Gaussian 채널(ACC)로, parity 상관을 별도 PCC 채널로 분리한 parallel-channel 코딩 프레임.
4. 핵심: systematic 코드 `[Y0 P1(Y0)]`을 가정하되 parity `P1(Y0)`를 다른 정확출력 `Y1`의 parity `P1(Y1)`로 대체(imaginary encoder) → 상관을 parity로 활용.
5. 제약: systematic 구조상 `H=[P I]`로 degree-1 VN 다수 발생, check node degree `m`은 균일·사전고정(PCC 용량 `1-H(0.5-(1-2r)^m)`이 `m` 종속이므로).
6. 코드: two-channel(ACC/PCC)·two-edge-type semi-regular LDPC를 `μ(r,x)`, `ν(x)` 다항식(식 5)으로 표현, VN degree `dv`만 조정.
7. 안정성: multi-edge-type 프레임([10])을 수정, degree-1 VN을 적절 배치하면 "simple cascade"가 되어 stability matrix가 0 → 완전복호 가능(Proposition 1).
8. 분석: protograph 기반 modified PEXIT(Algorithm 1)와 density evolution `al=λ[a0,ρ(al-1)]`로 Gaussian·BEC 채널에서 `dv` 증가 시 `Iapp→1` 증명(Prop 2,3).
9. 결과(시뮬): `r=0.15`, `dc=2`, length 10000~13000에서 실제 noise variance가 설계값의 2배여도 error prob `<10^-3` — robust.
10. 한계: 이론+시뮬만, HW 미설계, GPU 근사연산 도메인 특화, rate 명시 없음.
