### B. 알고리즘 요약

1. WiMAX 표준 (2304, 1152) Block-LDPC(QC-LDPC) 코드를 대상으로 layered MPD (LMPD-ICM, identical core matrix 사용) 디코더를 다룬다.
2. 기존 LMPD-ICM은 매 라운드 모든 레이어(P=96)를 순차 처리해 복잡도가 크다는 문제를 해결하려 한다.
3. Check-node 관점에서 레이어별 신뢰도를 판단: 레이어 `p`가 `Hp·v̂p^T = 0`을 `Nat`번 연속 만족하면 신뢰 가능(reliable)으로 간주.
4. 동적 스킵(DS): 신뢰 가능한 레이어의 디코딩을 스킵. 동적 강화(DE): 매 라운드 끝에 `H·v̂^T`로 만족되지 않은 레이어 집합 `Ω`를 재처리(최대 `NR`회).
5. DS는 error floor를 유발하지만 DE는 BER 열화 없이 복잡도(ANPL)를 줄인다는 것이 핵심 관찰.
6. Core matrix `Hc`를 이용한 조기종료: 각 레이어(=시간 순간) 디코딩 종료 시점마다 parity check를 수행, 연속 `λ` 개(≥P) 레이어가 유효(valid)하면 디코딩 종료 - 기존 전체 PCM 체크(PC-H) 대비 HW 자원을 P배 절감.
7. DE(NR=1)와 제안 조기종료 결합 알고리즘: 정상 단계에서 연속 유효 레이어 카운터 `c`를 유지하고, 재처리 단계(Step 7)에서는 카운터를 갱신하지 않아 조기종료가 발생하지 않도록 함.
8. 별도 학습/최적화 절차 없음(파라미터 `Nat`, `NR`, `λ`는 시뮬레이션으로 튜닝).
9. 결과: SNR=2dB, LM=9600 기준 DE(NR=2) 사용 시 ANPL 23.7% 감소(719.91→549.25), BER은 열화 없이 4.4×10⁻⁷ 수준 유지; DS 단독 사용 시 ANPL 최대 24.3% 감소하나 BER 열화(2.0×10⁻⁶) 발생.
10. 한계: HW 구현/합성 결과 없음(순수 시뮬레이션), 단일 코드(WiMAX 2304,1152)만 검증, 정정능력 자체는 개선하지 않고 복잡도/latency만 절감.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 동적 레이어 스케줄링(skip/redo, DS/DE) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update 스케줄과 유사한 레이어 처리 순서 제어 영역에 참고 가능 |
| Core-matrix 기반 조기종료(λ consecutive valid layers) | `partial_CRC.cpp` `partial_crc_HW_T4()` | Partial CRC의 termination point 아이디어와 유사한 부분 parity 체크 기반 조기종료 |
| Layered MPD(LMPD-ICM) 자체 스케줄 | `decoder.cpp` `C2V_Cal()` | 이터레이션 루프의 레이어 순회 방식에 대응 |

> 적용 가치: 중간 — 조기종료·스케줄링 아이디어는 이미 Prime ECC에 유사 기능(Partial CRC, Dual-Update)이 있어 중복성이 있으나, layer-level 신뢰도 기반 재처리(DE) 방식은 참고할 만함.
