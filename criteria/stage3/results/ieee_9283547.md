### B. 알고리즘 요약 (10줄 내외)

1. 대상 시스템은 플래시 기반 SSD(3D XPoint/Z-NAND급 저지연 디바이스)의 FTL 내 inline 데이터 중복제거(deduplication)로, LDPC/BCH 코드 자체가 아니라 스토리지 I/O 경로 최적화가 목적이다.
2. 풀려는 문제는 기존 SHA/MD5 기반 fingerprint 생성이 계산·메모리 오버헤드가 크고, 저지연 플래시(3D XPoint/Z-NAND)에서는 이 해시 연산 지연이 쓰기 지연보다 커지는 병목이 된다는 점.
3. 핵심 가정은 NAND 페이지마다 이미 저장되는 ECC(BCH 또는 LDPC) 값이 데이터 청크의 고유성을 어느 정도 반영한다는 것 — ECC 자체를 해시처럼 재사용.
4. 핵심 기법(EaD)은 ECC 값 기반 Bloom filter + Blake2 Index Table(BIT) 2단계 필터로 중복 후보를 빠르게 걸러내고, 후보만 실제 플래시에서 읽어 byte-by-byte 비교로 확정하는 구조.
5. 핵심 식/구조 의미: ECC의 hash collision 확률이 SHA-1 대비 거의 0(`BCH(4224,4096,8)` 기준 미측정)이라는 실험적 관찰이 ECC를 fingerprint로 대체 사용할 수 있는 근거.
6. 확장 기법으로 두 계층 간접 매핑(Primary/Secondary Mapping Table, PMT/SMT)을 두어 hash collision 시에도 실제 데이터는 안전하게 저장하고 garbage collection 역참조를 단순화.
7. 구현 디테일: ECC 코드(BCH 104바이트/4KB)를 직접 fingerprint로 쓰지 않고 Blake2(8바이트, 3.08 cycle/byte)로 재해싱해 메모리 오버헤드를 추가로 절감.
8. 학습/최적화 절차 없음 — 시스템 설계 및 SSDSim 기반 이벤트 시뮬레이션으로 평가.
9. 결과: 4종 실트레이스(Homes/Webs/Mails/Hadoop)에서 SHA-1 대비 평균 I/O 지연 1.92배, 샘플링 기반 대비 1.86배 개선, 메모리 소비는 각각 평균 35.0%/21.9% 절감, 중복제거율은 SHA-1과 거의 동일(false positive 0%).
10. 한계: NAND ECC를 정정용이 아닌 fingerprint로 전용(轉用)하는 시스템 논문으로 LDPC 디코더/부호설계 개선이 전혀 없고, 실제 실험은 LDPC가 아닌 BCH 코드 기준(LDPC는 배경 언급뿐)이며 HW 미설계, SSDSim 시뮬레이션만 수행.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ECC 값을 fingerprint로 재사용하는 dedup 알고리즘 | 대응 없음 | Prime ECC는 부호정정 시뮬레이터이며 FTL/dedup 시스템 계층 개념이 없음 |
| ECC-based Bloom filter / Blake2 Index Table | 대응 없음 | 스토리지 인덱싱 구조로 LDPC 디코더/인코더 알고리즘과 무관 |
| BCH(4224,4096,8) 실험 코드 | 대응 없음 | Prime ECC는 binary QC-LDPC 전용이며 BCH는 판정 대상 외(§5 판정 필터: binary LDPC 전용) |
```

적용 가치: **낮음** — LDPC/ECC를 정정 목적이 아닌 데이터 중복제거 fingerprint로 전용하는 스토리지 시스템 논문이라 Prime ECC(디코더/부호설계) 어느 모듈에도 대응하지 않음.
