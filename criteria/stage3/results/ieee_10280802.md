### B. 알고리즘 요약

1. 시스템: QC-LDPC, 예제 (2700,1352) code length 2700 rate 0.5, BPSK+AWGN, soft-decision 입력, 5G NR 등 다중 길이/rate 지원 목표.
2. 문제: 일반 LDPC는 H-matrix 저장·읽기 어렵고 부호화 복잡도 큼 → 구조적 QC-LDPC로 저장/주소지정 완화 필요, 가변 길이/rate 요구.
3. 구성기법(부호설계): Fibonacci-Lucas 수열(1,3,4,7,11,18,...)을 삼각배치 후 45도 회전(triangular rotation)해 지수행렬 `E`(0×) 구성, 행/열 수와 확장인자 P로 길이·rate 가변.
4. 확장: 지수행렬 원소값만큼 U×U 단위행렬을 순환시프트해 검사행렬 H(0U×U) 생성; 꼬리원소에 파라미터 k(T) 더해 최대원소를 확장차원 P에 근접시킴.
5. girth 보증: Fibonacci-Lucas 정리로 식(12)(4-ring 없음)·식(15)(6-ring 없음) 유도 — p가 지수행렬 최대원소보다 크면 4/6-ring 미발생 → error correction 향상.
6. 핵심기법(디코더): LNMS = Layered + NMS, CN update `E=∏sign·γ·min|L|` (식17, correction factor `γ`), layered로 갱신된 사후확률을 다음 layer에 즉시 사용.
7. 구현 디테일: correction factor `γ=0.75`가 최적(시뮬), min/sub-min을 부분병렬로 그룹 계산; layered는 layer간 직렬·layer내 병렬.
8. HW 구조: interlayer pipeline로 read/compute/return sub-process를 겹쳐 1 iteration 지연을 약 절반 단축, throughput 향상(NMS와 HW 자원 소비 동일).
9. 결과: LNMS는 NMS와 BER 동등하되 iteration 절반(LNMS 10 vs NMS 20)로 동일 성능; (2700,1352) BER=10^-5에서 50 iteration SNR≈1.95dB, 50회 이후 개선 미미해 iteration 50 선택; MS 대비 BP는 ~0.7dB 우수, NMS는 BP 대비 ~0.25dB gap.
10. 한계: HW는 layered/pipeline 구조도만 제시하고 gatecount·throughput·공정 수치 없음, Matlab 시뮬만, rate-0.5 무선 타깃.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS(min-sum+correction) CN 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 min-sum 보유 — NMS의 γ correction factor는 magnitude 테이블에 반영 검토 가능 |
| Layered(수렴가속) 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | layered 스케줄은 Prime ECC의 Dual-Update와 유사군, iteration 감소 목적 정합 |
| interlayer pipeline HW | 디코더 HW 모델(z=32, pipeline 6-stage) | Prime ECC 6-stage pipeline과 개념 대응하나 구조 정합 필요 |
| Fibonacci-Lucas H-matrix 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 부호구조 레이어에 닿으나 Prime ECC는 고정 고rate QC-LDPC라 재검증 부담 큼 |
| soft-decision LLR 입력 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | soft 2~3bit 입력 정합 |

적용 가치: **중간** — QC-LDPC+layered NMS+pipeline 방향은 Prime ECC와 정합되나, layered/NMS는 이미 유사 기법 보유(Dual-Update/min-sum)이고 부호구성은 rate-0.5 무선 타깃이라 고rate NAND 적용엔 재검증이 필요하다.
