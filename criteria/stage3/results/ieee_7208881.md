### B. 알고리즘 요약

1. 대상은 NAND 플래시용 고rate ECC로, `(18900,17010)` rate `0.9` binary QC-LDPC, submatrix size `z=63`, column degree `dv=4 or 5`, row degree `dc≈50`.
2. 문제: Latin-square QC-LDPC는 성능은 좋으나 submatrix가 커서 HW에서 큰 barrel shifter·심한 routing congestion 유발 → NAND 컨트롤러에 부적합.
3. 채널/모델: AWGN + 실측 2Xnm 3-bit/cell NAND RBER, hard(1bit)/soft-2bit 채널값을 LLR로 매핑.
4. 핵심 기법 1단(부호구성): Latin square로 작은 base matrix(`6x60`, GF(2^6)) 생성 후, `array dispersion`(expand factor 5)으로 diagonal-like 구조 유지하며 `30x300` submatrix 대형 부호로 확장 → submatrix를 z=63으로 작게 유지, cycle-4 없음(RC-constraint 보존).
5. 핵심 기법 2단(최적화): base matrix에 `masking`(submatrix를 0으로 치환)을 적용해 short cycle·small trapping set 제거 → error-floor 억제; FPGA로 ~500개 masking 후보를 2주간 탐색해 최저 BER 선택(column degree≥4 유지).
6. 디코딩: normalized min-sum VN-centric sequential scheduling(`NMS-VSS`, α scaling), CN당 min-and-index(1st/2nd min) + global sign, 조기종료·max iter 20.
7. HW 트릭: diagonal 구조를 이용한 `hybrid register-memory storage`(6 register block + single-port SRAM @2x clock) → SRAM 대역폭을 5/6로 축소, min-and-index 18bit; 6 CNU block으로 global routing 회피, 4-bit 양자화 Q(4,2).
8. 최적화: dynamic sensing(hard 우선, 실패 시 2nd bit 획득) 채택, P/E cycle별 hard/soft-2bit 디코더 사용비율 시뮬(10,000 패턴).
9. 결과: UMC 90nm postlayout `1.58Gb/s @6 iter, 166.6MHz, gate 520k, 2.56mm2`; error-floor를 BER `1e-12`까지 미관측(unmasked는 1e-10), soft-2bit이 hard 대비 1dB 개선, Toggle DDR1.0/ONFI2.3 충족.
10. 한계: soft-2bit까지만, masking 검색이 코드 고정형(재구성 부담), 성능 이득의 핵심은 부호구성·저장구조이며 min-sum 알고리즘 자체 개선은 아님.
