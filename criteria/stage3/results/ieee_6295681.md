> 총평: high-rate QC-LDPC의 라우팅(switch network)을 fixed-wire 1-stage로 단순화해 41Gbps 달성한 HW 논문 — 정정능력 이득은 없고 라우팅/throughput 관점 참고용.

### B. 알고리즘 요약

1. 대상: (2048,1723) RS-LDPC (10GBASE-T용), (6,32)-regular, row weight 32/col weight 6, rate≈0.841. BPSK/AWGN.
2. 문제: RS-LDPC는 girth≥6·MDS로 성능 좋지만 random PCM이라 switch network(Benes/Banyan) HW 복잡도·라우팅 지연이 큼.
3. 핵심 기법1(부호설계): codeword `c`를 symbol location vector `z(c)`로 전개 후 cyclic shift 반복해 layer 생성 → QC-RS-LDPC. layer1을 submatrix 크기 배수로 다시 cyclic shift해 concat → block-circulant(BC) PCM.
4. 구조 보존: RS-LDPC 두 성질(같은 layer 내 1-공유 없음, 다른 layer 간 1-공유 ≤1)을 유지해 girth·거리 특성 손실 없음.
5. 핵심 기법2(HW): BC PCM은 각 layer가 동일 cyclic permutation 수 집합(순서만 다름) → switch network를 input control block + fixed-wire block만으로 구성(1 stage).
6. SN 이득: Banyan 64 subnet ≈582K gates·7 stage 대비 제안 SN ≈103K gates·1 stage·critical path 1.4ns, 제어신호 메모리 불필요(clock 제어).
7. 복호: offset min-sum(β=1.25) + layered decoding, 4-stage pipeline(iteration당 총 9 stage, 6 layer). 양자화 `(q,f)=(6,2)`.
8. 파리티체크로 조기종료: `ĉ×Hᵀ=0`이면 종료, 아니면 반복(최대 8 iter).
9. 결과: BER는 기존 RS-LDPC/QC-RS-LDPC와 거의 동일(성능 저하 없음, 10⁻⁷서 Shannon 한계 2.04dB). 합성 1.3M gates@90nm·450MHz·throughput 41Gbps·efficiency 31.54 Gb/s/Mgate로 [14]~[16] 대비 우위.
10. 한계: 특정 (2048,1723) 부호 전용, 정정능력 개선 아님(라우팅/throughput 개선). soft LLR 6bit 양자화 가정, NAND 채널 미검증.
