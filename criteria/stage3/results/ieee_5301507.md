### B. 알고리즘 요약

1. 대상은 CMMB(China Mobile Multimedia Broadcasting) 표준의 binary LDPC 코드(9216, rate 1/2·3/4), H는 cyclic이나 quasi-cyclic은 아님.
2. 기존 Richardson의 ALT(Approximately Lower Triangular) 인코딩은 H를 여러 부분행렬로 쪼개면서 원래의 cyclic·균등 row weight 특성을 소실시켜 저장 효율이 떨어짐.
3. 제안 기법은 partial trapezium 인코딩(반복적으로 unknown parity bit 1개만 갖는 행을 찾아 순차적으로 해를 구하는 방식)을 ALT와 결합한 "mixed partial trapezium and ALT form"을 사용.
4. H를 `V'HM'`로 치환 후 최상단 `d`개 행은 partial trapezium 형태(사다리꼴), 나머지 `r-d`행에 ALT를 적용, parity vector를 `Z0', Z1', Z2'` 세 블록으로 분할.
5. `Z1'`은 기존 ALT 방식(`Q'^-1` dense 소행렬 이용)으로, `Z0', Z2'`는 partial trapezium(순차 전진대입)으로 계산 — 즉 처음/끝 구간은 통짜 행렬 연산 없이 H 원본 행을 그대로 재사용.
6. 이 결합 구조 덕분에 부분 행렬을 분리 저장할 필요 없이 H(또는 순열된 H)의 cyclic 구조·균등 row weight를 그대로 활용해 메모리 절감.
7. 이진 부호의 경우 sparse 행렬은 0이 아닌 원소 위치만 저장(값은 생략 가능), dense한 `Q'^-1`만 전체 값 저장.
8. 별도 학습/최적화 절차 없음(그리디 permutation으로 `g`, `d` 결정, 상세 알고리즘은 원 ALT 논문[7] 인용).
9. FPGA(Cyclone III EP3C55) 구현 결과: 기존 ALT 대비 M9K RAM 147→56블록(약 62% 절감), logic cell은 1969→3107로 증가; 100MHz에서 encoding rate 15.71Mbps(rate 1/2), 29.542Mbps(rate 3/4)로 CMMB 규격(10.852/16.243Mbps) 충족.
10. 인코더만 다루며 디코더/정정성능과 무관, CMMB 특화 cyclic(비-QC) 구조 가정이라 일반 QC-LDPC로의 일반화는 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ALT/partial trapezium 인코딩 절차 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코딩이라 알고리즘 자체는 다르지만 부호용 PCM 생성 로직과 대응 위치는 유사 |
| H-matrix cyclic 구조 활용 저장 전략 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(circulant shift) 구조라 이미 유사한 압축 저장을 적용 중, 추가 이득 낮음 |
| 인코더 FPGA 자원(RAM/logic) 최적화 | `encoder.cpp` `Column_Reordering_Dual_Update()` | 메모리 절감 아이디어는 참고 가능하나 코드 구조 상이 |

적용 가치: **낮음** — CMMB 특화 cyclic(비-QC) LDPC 인코더용 메모리 최적화로, QC-LDPC dual-diagonal 고정 구조인 Prime ECC 인코더에 이식하려면 근본적 재설계가 필요함.
