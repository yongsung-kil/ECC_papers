### B. 알고리즘 요약

1. 대상: NAND flash 컨트롤러용 QC-LDPC Codec IP, (9088, 8192) 코드(rate≈0.90), parity part는 dual-diagonal 구조.
2. 문제: soft-decision은 다중 read로 throughput 저하·HW 비효율 → hard-decision 정정성능을 BCH급으로 끌어올리는 것이 실용화 관건.
3. 인코더: dual-diagonal parity 구조로 O(N) 인코딩, 병렬도(dual/single/serialized data-path)로 area-throughput trade-off (67.5k@800MB/s vs 18k@100MB/s).
4. 디코더 구조: flooding schedule의 column-by-column 아키텍처 → variable node memory 부재가 핵심 장점(고rate·장부호에서 메모리 효율).
5. 메시지 처리: VNPU가 초기 LLR + CNPU 메시지 합산, CNPU는 `min-sum` 연산, R 모듈에서 QC 순환 rotate, 2to7/7to2 MUX로 라우팅.
6. CNPU 메모리: 1st/2nd min, 1st min 위치, 부호 총합은 register, edge sign은 SRAM(Check Mem)에 분리 저장 → Check Mem이 전체 면적 44%.
7. 개선기법1: 1st iteration의 LLR은 신뢰도 낮으므로 작은 절대값(=1) 부여, 이후 iteration은 큰 값(=4) → 평균 iteration 감소(수렴 가속).
8. 개선기법2: offset min-sum과 normalized min-sum을 절충한 경험적 매핑테이블로 sum-product 성능에 근접(TABLE II).
9. 결과: 45nm 700k gate, 200MHz에서 2.5E-3 raw BER까지 400MB/s 보장, 평균 iteration 3.8(2.5E-3 BER 시), 142 col/145 cycle 처리.
10. 한계: BER 곡선·매핑테이블 계수가 경험적이고 정량 그래프는 텍스트에 없음, hard-decision 전용(soft는 보조 언급뿐).
