### B. 알고리즘 요약

1. 시스템: IEEE 802.11aj LDPC, rate `5/8`, length `672` bits, BPSK/AWGN 채널. 완전병렬 stochastic 디코더 대상.
2. 문제: stochastic LDPC 디코더는 VN(J-K FF)이 두 입력 불일치 시 hold(latching) 상태에 빠져 정보 갱신 효율·수렴이 느림.
3. 모델: 확률(stochastic) 비트스트림으로 정보를 표현, VN은 J-K FF, CN은 XOR 게이트로 구현.
4. 기존 대책: Tehrani의 re-randomization 유닛 - edge memory(`EM`)/internal memory(`IM`) 시프트레지스터로 non-latch 상태 비트를 저장·랜덤 재선택.
5. 핵심 기법: dual updating EM 구조 - 채널정보와 EM 갱신을 두 개의 독립 인에이블 신호(`U1`,`U2`)와 이중 시퀀스(`ch_in1/2`,`CN_out1/2`)로 동시 처리.
6. 규칙: 두 입력이 같으면 두 갱신비트로 EM 갱신, 하나만 같으면 한 갱신비트, 둘 다 다르면 bucket shift 후 EM에서 랜덤 출력.
7. 효과 논리: 두 신호가 EM만 공유하고 나머지 완전 독립 → EM 갱신 효율 상승 → 수렴 가속.
8. HW: SMIC 65nm, Synopsys DC 합성 + IC compiler 배치.
9. 결과: `BER=2×10^-4`에서 dual이 single 대비 `0.37dB` 개선, 디코딩 사이클 100~200 집중(single은 150~400), gate `314K→404K`(+28.7%).
10. 한계: 저SNR에서는 노이즈로 dual 갱신 오차 심화되어 오히려 성능 열화, throughput 절대값 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stochastic VN/CN(J-K FF/XOR) 디코딩 | 대응 없음 | Prime ECC는 Min-Sum 정수연산, stochastic 연산 미보유 |
| edge memory 재랜덤화(latching 완화) | 대응 없음 | stochastic 전용 구조로 Min-Sum 이터레이션 루프([decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`)에 대응 개념 없음 |
| 이중 병렬 업데이트 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 개념적 유사(Prime ECC의 Dual-Update parity 스케줄)이나 원리 상이, 직접 이식 불가 |

적용 가치: **낮음** — stochastic 연산 패러다임 전용 기법으로 Min-Sum 기반 binary QC-LDPC ECC에 떼어 쓸 코드 요소가 없음.
