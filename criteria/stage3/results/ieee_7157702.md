> 총평: HDD/SSD 저장장치를 명시적으로 타깃한 concatenated LDPC(4KB C1 + 트랙단위 outer parity C2) 구조로, 낮은 오버헤드로 SNR 이득을 얻는 아이디어는 NAND ECC의 outer code/RAID-유사 보호 레이어로 참고 가치가 있으나 본문이 짧아 세부 설계·HW 영향은 확인 불가.

### B. 알고리즘 요약

1. 대상 시스템은 HDD/SSD 등 데이터 저장장치의 iterative soft-decoding LDPC ECC이며, 재전송이 없어 error floor는 10^-12 이하가 요구됨. 4KB 섹터 포맷으로 코드워드 크기가 4096→32768 유저비트로 증가.
2. 문제: 4KB를 넘어서는 LDPC 블록 크기 확대는 추가 SNR 이득을 주지만 실리콘 면적·전력 비용이 매우 커짐. 이를 대체할 저비용 SNR 이득 방안이 필요.
3. 핵심 가정: 채널은 HDD 매체 노이즈(80%)+정상 전자 노이즈(20%), 채널 밀도 T50/T=1.0.
4. 핵심 기법: LDPC 베이스 코드 C1(4KB)과 별도 parity code C2(트랙 전체, 예: 400섹터 중 399 유저섹터+1 parity섹터)를 결합한 concatenated 구조. 유저데이터는 RLL 인코딩→LDPC C1 인코딩→C2 systematic 인코딩(패리티 생성)→그 패리티를 다시 C1로 인코딩해 parity 섹터로 저장.
5. 복호는 각 섹터를 C1로 개별 iterative soft decoding 시도, 실패 섹터가 있으면 C2를 디코딩해 실패 섹터에 추가 soft information을 생성, C1이 이를 이용해 재시도(C1-C2 간 extrinsic information 교환하는 iterative 구조).
6. 확장/설계 트레이드오프: C1은 상대적으로 강한 LDPC(전체 concatenated 디코딩은 드물게 발생)로, C2는 단순 parity check code(single parity check)로 선택해 인코딩/복호 복잡도를 낮춤.
7. 구현 디테일: 트랙(400섹터) 단위 outer code, 추가 오버헤드 0.0025%(패리티 섹터 1개), 섹터 단위 read-modify-write 필요.
8. 별도 학습/최적화 절차 없음(구조적 설계).
9. 결과: 무제한 버퍼·처리시간 가정 시 4KB LDPC 단독 대비 약 0.7dB SNR 이득; 버퍼를 6개 실패섹터로 제한한 실제적 상황에서는 섹터 실패율 1e-5 기준 이득이 약 0.3dB로 감소.
10. 한계: HW 미설계, C1/C2 세부 스펙 비공개, outer code 활용 시 지연시간·메모리 제약이 실제 이득을 크게 낮춤, 순수 이론/시뮬레이션 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| C1(4KB LDPC) 단일 섹터 iterative soft decoding | `decoder.cpp` `LDPC_Decoder()` | Prime ECC의 기본 디코딩 루프와 동일 구조(4KB LDPC), 그대로 C1 역할 대응 가능 |
| C2(outer parity code) 조기 실패 시 추가 soft info 생성 및 C1 재시도 | 대응 없음 | 현재 Prime ECC에는 트랙/섹터 단위 outer concatenation 계층이 없음. 별도 아키텍처 레이어 신설 필요 |
| 실패섹터 버퍼링에 따른 latency/메모리 트레이드오프 | 대응 없음 | HW arch(메모리) 관련이나 논문 자체가 outer-code 시스템 레벨 설계로 decoder.cpp 단위 모듈과 직접 대응 안 됨 |

> 적용 가치: 중간 — 4KB LDPC를 그대로 C1로 재사용하는 concatenated outer-code 아이디어는 시스템 레벨에서 참고 가능하나, Prime ECC는 섹터 단위 디코더이며 트랙 단위 outer parity 계층이 없어 이식에는 새로운 아키텍처 설계가 필요.
