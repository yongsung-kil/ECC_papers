### B. 알고리즘 요약

1. 시스템: AWGN+BPSK 채널에서 rate-compatible(RC) short LDPC 코드 `C(n,k)`, LLR `L_i=2y_i/σ²`, 실험은 non-binary LDPC C(128,64)와 5G RC 코드(k=45, 48≤n≤138).
2. 문제: short block-length에서 BP는 Tanner graph의 짧은 cycle 때문에 sub-optimal이고, 기존 OSD/BP-OSD는 ML급 성능에 여전히 고복잡도.
3. 핵심 가정: girth `g`인 LDPC는 `g/2`번째 반복 전까지 메시지가 독립이며, 이후 반복 출력 LLR은 신뢰도가 낮아짐(short cycle 영향).
4. 핵심 기법1: ordered likelihood decoder(OLD) - TEP likelihood `R(e)=Σα_j e_j`를 최도-정렬된 reliability curve `α_B`의 구간별 선형근사(`r_j=s_d j+I_d`, 세그먼트 D개)로 모델링, TEP를 확률 내림차순(=R 오름차순)으로 직접 생성.
5. 핵심 식: `Pr(e)∝exp(-Σα_j e_j)`이므로 TEP 생성 순서를 근사식 (9)로 정렬해 OSD처럼 조합 전수탐색 없이 순차 생성.
6. 핵심 기법2(MB-OLD): BP 반복 `l=0,...,g/2-1`의 출력 LLR을 `L'=Σβ_l L^(l)` 가중합(β_l은 density evolution 기반 정규화 정확도)으로 결합해 OLD 입력으로 사용.
7. 부수 트릭: stopping criterion(SC)으로 `D*=D_MRB*+D_LRB*`를 추정해 최적 TEP 발견 시 조기 종료(식 16~18), 최대 TEP 수로 복잡도 상한 설정.
8. 최적화 절차: girth `g`와 iteration 수 `T_max`는 사전 분석(density evolution)으로 결정, 최적 반복수 이후 OLD 활성화.
9. 결과: C(128,64) NB-LDPC에서 MB-OLD가 OSD(order=3)/mBP-OSD/BP-OSD2007 대비 각각 0.35/0.45/0.65dB 개선(BLER=1e-4), 5G RC 코드에서도 NA bound에 근접하며 낮은 복잡도.
10. 한계: HW 구현/합성 없음(시뮬만), 실험은 short block-length(n≤138) 및 non-binary/5G 코드 중심이라 NAND급 대형 binary QC-LDPC로의 복잡도 확장성 미검증, Gaussian elimination 기반 OSD 구조 특성상 대형 코드에서 비용 증가 우려.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 반복 후 OSD 계열 concatenated 디코딩(post-processing) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | min-sum 수렴 실패 시 error-floor 대책으로 이식 검토 가능 |
| girth 기반 반복별 LLR 가중합 결합 | `decoder.cpp` `Get_VNU_Table_Idx()`, `ecc_data.h` `PARAM_LLR` | 반복 구간별 LLR 테이블 조정 아이디어와 유사하나 목적(신뢰도 가중합)이 다름 |
| stopping criterion(조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 목적은 유사(조기종료)하나 메커니즘은 CRC가 아닌 WHD 추정 기반 - 직접 대응 아님 |
| Gaussian elimination 기반 TEP/OSD 재정렬 | 대응 없음 | Prime ECC는 min-sum 기반 iterative decoder이며 OSD류 GE 연산 모듈 없음 |

> 적용 가치: **중간** — BP 실패 시 post-processing으로 OSD 계열을 붙이는 아이디어는 error-floor 개선에 참고 가치가 있으나, GE 기반 구조·short/non-binary 코드 중심 검증이라 대형 binary QC-LDPC HW 이식은 추가 검증 필요.
