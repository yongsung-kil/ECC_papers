### B. 알고리즘 요약

1. (3,6)-regular SC-LDPC 코드(protograph 기반, coupling width m, rate R≈0.497~0.499)의 저지연 sliding window decoding(SWD, window size W)을 다룸.
2. 짧은 W에서 SWD 사용 시 드물지만 심각한 decoder error propagation(블록 오류가 연쇄적으로 후속 블록까지 전파)이 발생해 BLER이 1로 수렴할 수 있다는 문제를 다룸.
3. 핵심 가정: error propagation은 blk 디코딩 오류 시 잘못된 LLR이 다음 블록 디코딩에 부정적 영향을 주며 연쇄되는 현상, 스트리밍(무종단) 응용에서 특히 치명적.
4. 핵심 기법 1(VN doping): 커플드 체인 내 일부 변수노드(VN)를 "0"으로 고정(fix)해, 인접 CN을 사실상 저차수로 만들어 CN doping과 동일 효과를 얻되 디코딩 윈도우 형태 변경이 불필요. Design rate `R_{L,doped} = 1 - (L+m)/(L-d) * (1-R)`.
5. 핵심 식의 의미: 고정 VN이 이웃 노드에 완벽히 신뢰할 수 있는 정보를 전달, 그래프 종단(termination)을 에뮬레이트하여 에러 전파를 국소적으로 차단.
6. 핵심 기법 2(window extension): 관측 구간 τ 내 블록들의 평균 LLR 크기 `Λ̄_i = (1/2M)Σ|Λ_i^j|`가 임계값 θ보다 작으면(잠재적 error propagation 징후) 윈도우 크기 W를 2 time-unit씩 동적으로 확장(Winit→Wmax).
7. 부수 트릭: window 확장 폭(+2)은 성능-복잡도 트레이드오프 실험으로 결정된 경험적 값.
8. 최적화 절차 없음(코드 설계+휴리스틱 임계값 기반, 학습 기법 아님).
9. 결과: VN doping은 CN doping과 동등하게 BER 약 2자릿수, BLER 약 1자릿수 개선(전형적 SNR 영역). Window extension은 W=9에서 W=18(미확장) 대비 동등 성능을 유지하며 평균 latency 약 1/3 감소.
10. 한계: SC-LDPC(컨볼루션) 특유의 sliding window decoding 문제에 국한, HW 미설계·순수 시뮬레이션, block code인 Prime ECC와 코드 구조 자체가 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC sliding window decoding 전체 | 대응 없음 | Prime ECC는 QC-LDPC block code(`ecc_top.cpp` `Load_PCM()`)이며 windowed/streaming 디코딩 구조 자체가 없음 |
| VN doping(부호설계, 알려진 비트 고정) | 대응 없음 | H-matrix/protograph 구성법 변경이라 프로파일 §4 "부호설계: 높음(난이도)"에 해당, 게다가 SC-LDPC 전용 개념이라 QC-LDPC엔 대응 없음 |
| Window extension(동적 윈도우 크기, LLR 평균 기반 판단) | `decoder.cpp` `LDPC_Decoder()` | 이터레이션/스케줄 제어라는 점에서 개념적으로만 유사하나 sliding window 자체가 없는 Prime ECC 구조엔 직접 적용 불가 |

> 적용 가치: 낮음 — SC-LDPC 컨볼루션 코드의 sliding window decoding 고유 문제 해결책으로, block code 기반 QC-LDPC인 Prime ECC에는 코드 구조 차이로 인해 이식 경로가 사실상 없음.
