### B. 알고리즘 요약

1. 3GPP-LTE turbo, 802.11n/802.16e QC-LDPC, 64-state convolutional code(CC) 3종을 하나의 trellis application-specific instruction-set processor(TASIP)로 통합 디코딩.
2. 문제: 각 표준마다 별도 FEC 디코더를 두면 면적/전력이 과도하며, 기존 duo-mode 솔루션은 기능 커버리지 또는 처리량이 부족.
3. 세 알고리즘(turbo BCJR, LDPC layered decoding, Viterbi)이 공통적으로 forward-backward recursion(FBR) 구조를 갖는다는 점을 이용해 통합 커널 설계.
4. 상태 `S`는 이전 스텝의 `S0=S/2`, `S1=S/2+Ts/2`, `S`(LDPC용)와 연결되는 8-state 병렬 trellis 구조. 순방향 메트릭 `α(S)`, 역방향 `β(S)`, 갱신함수 `f(x,y)`가 알고리즘별로 다르게 정의(LDPC는 `f(x,y)=sign(x)·sign(y)·min{|x|,|y|}`로 scaled min-sum 근사, λ=0.75).
5. `f(x,y)` 형태가 세 알고리즘 간 유사하여(min-sum류) 하나의 datapath(FRU)로 하드웨어 공유 가능.
6. SIMD 아키텍처: SISO 12개, Global Memory System(GMS), Read/Write Permutation Network(RPN/WPN), Address Transform Unit(ATU)으로 구성. LDPC는 base matrix 행(layer) 단위로 8개 sublayer씩 SISO에 매핑.
7. HW 친화 트릭: LDPC a posteriori 메시지 5비트 양자화, LDPC layer 재배열 및 원소 재스케줄링으로 read-after-write 충돌 최소화(처리량 최대 60% 개선). 데이터는 8비트 기본 양자화.
8. 별도 학습/최적화 절차 없음(고정 아키텍처 설계).
9. 결과: 65nm 합성 시 면적 2.12mm², 1135k-gates, 200MHz에서 LDPC 533Mb/s, turbo 186Mb/s, CC 225Mb/s. 메모리 비트 최대 41% 절감.
10. 한계: LDPC 부분은 신규 디코딩 알고리즘이 아니라 기존 layered min-sum을 재사용한 것이며, 5비트 양자화로 인해 error floor가 상대적으로 높음(~10⁻⁶). 논문 기여의 핵심은 멀티표준 간 하드웨어 재사용 구조이지 LDPC 자체 성능 개선이 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC layered decoding + scaled min-sum | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 기법(min-sum, layered)과 동일 개념, 신규 이식 가치 없음 |
| turbo/CC 공유 FBR 커널, SISO 아키텍처 | 대응 없음 | Prime ECC는 binary QC-LDPC 전용이며 turbo/CC 멀티모드 공유 구조는 대상 외 |
| LDPC a posteriori 5비트 양자화 | `channel.cpp` `Set_LLR_Th()` | 양자화 비트폭 설계 참고 가능하나 성능 저하(0.18dB) 트레이드오프가 있어 이식 매력 낮음 |
| GMS 메모리 뱅크/충돌 회피(layer 재배열) | 대응 없음(HW arch 레벨) | Prime ECC 고유 HW 구조(z=32, HCU)와 상이해 직접 이식 어려움 |

> 적용 가치: 낮음 — LDPC 알고리즘은 이미 보유 기법과 중복이고, 논문 핵심(멀티표준 트렐리스 공유)은 binary QC-LDPC 전용인 Prime ECC의 이식 대상과 맞지 않음.
