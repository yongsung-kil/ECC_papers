### B. 알고리즘 요약

1. 대상: 표준 QC-LDPC(CCSDS, 802.11n, 5G), short block-length(N `128`~`648`), rate `0.5`(rate-half)~`0.83`(648,540), 코드는 protograph를 lifting factor `Z`로 확장.
2. 문제: short-length에서 BP는 ML 대비 성능 gap이 크나 QC-LDPC는 decoder equivariance 때문에 automorphism 앙상블 디코딩(AED) 이득이 원천 차단됨.
3. 핵심 관찰: QC-LDPC의 quasi-cyclic 열 대칭이 automorphism group `Q_Z`를 만들지만, 동일한 행 대칭이 BP 디코더에 흡수(absorbed)되어 permuted 디코딩이 원본과 동일 결과.
4. 핵심기법: 코드는 그대로 두고 수신단 Tanner graph `H̃`만 비quasi-cyclic으로 변형해 디코더 대칭을 `Q_Z`보다 작게 깨뜨림(부호·인코더 불변).
5. 핵심식: AED 후보 `ĉ_j = π_j^{-1}(Dec(π_j(y)))`를 `L`개 permutation으로 생성, ML 기준 `argmax P(ĉ_j|y)`(BI-AWGN에서 유클리드 거리)로 최종 선택.
6. H̃ 생성 3방법: (1) row operation(행 더하기), (2) auxiliary check 추가(overcomplete), (3) check 제거(undercomplete). undercomplete가 구현 최단순(한 check 비활성)이라 채택, 원본 H로 code membership 검증.
7. 구현 디테일: 모든 변형은 lifted full H에 적용(protograph 아님), 전 permutation `L=Z` 사용, 표준 flooding SPA 및 layered 디코더 모두 적용 가능.
8. 최적화 절차: 별도 학습 없음, check 선택은 랜덤(과잉 check degree 11 예) - 최적화 시 더 큰 이득 기대.
9. 결과: 5G/CCSDS rate-half에서 BLER `1e-3~1e-4`에 `0.2~0.3dB` 이득, avg latency는 조기종료 시 AED 3.4 iter vs SBP 28.5 iter(4dB), (128,64) CCSDS는 AED 32 iter가 단일 BP 256 iter와 동등.
10. 한계: HW 미설계, min-sum 아닌 full SPA로 비교(공정성 목적), short·저rate 표준부호만 검증, 고rate·장부호 이득 불명.
