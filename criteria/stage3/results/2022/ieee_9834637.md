### B. 알고리즘 요약

1. 대상: protograph 기반 SC-LDPC 코드(cyclic M-lifting), memory `m`, coupling length `L`; block 부호 base matrix `Bblock`을 edge spreading `B`로 SC 부호 `HSC`로 변환.
2. 문제: SC-LDPC의 error floor는 trapping set→short cycle에서 기인하므로, 유한길이 SC 부호의 short cycle 다중도(multiplicity)를 정량화하고자 함.
3. 모델: base graph의 TBC(tailless backtrackless closed) walk가 lifting 후 사이클이 됨(Lemma 1); spreading 후 TBC walk가 살아남는 조건은 차분합 `ΣΔBi(j,j')=0` (Lemma 2).
4. 핵심 기법1: spreading 행렬 원소 `Bi,j`를 `[0,m]` 균등분포 i.i.d로 두고 차분 `ΔBi(j1,j2)=Bi,j1-Bi,j2`의 분포 `Pr(ΔB=z)=(m+1-|z|)/(m+1)^2` 유도(Lemma 4).
5. 핵심 식: 두 차분의 합 분포(Lemma 5)를 다항식으로 구해, 사이클 존재를 `k=0` 확률로 계산.
6. 핵심 기법2(정리): 4-cycle `P4=(2m^2+4m+3)/3(m+1)^3`, 6-cycle `P6=(11m^4+...)/20(m+1)^5`, 8-cycle은 구조별(Fig.2 a~i) 분기 확률.
7. 트릭: 8-cycle은 TBC walk 구조 유형(Fig.2)마다 확률이 달라 상·하한으로 bound.
8. 일반화(Theorem 4 / Lemma 6): 임의 길이 ℓ TBC walk가 SC 부호에 남을 확률은 `O(1/m)` - 길이와 무관.
9. 결과(Remark 1): memory `m`을 키우면 모든 TBC walk(사이클) 존재확률이 단조 감소 → error floor 개선의 이론적 근거.
10. 한계: 순수 확률 해석(이론), 시뮬/HW/디코더/구체 부호 산출물 없음, block QC-LDPC가 아닌 SC 구성 대상.
