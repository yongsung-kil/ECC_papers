### B. 알고리즘 요약

1. 시스템: 3D TLC NAND SSD(256GB, 8ch, P/E 1000), LDPC를 기본 ECC로 사용, SSDsim 시뮬레이터·MSR 8 workload.
2. 문제: wear leveling이 모든 블록을 균등 마모 → 전체 신뢰도 동시 저하 → LDPC read-retry 급증으로 read latency가 마모 20~80%에서 3.8x~6.1x 증가.
3. 모델: LDPC read latency는 셀 신뢰도와 강상관 (신뢰도 낮으면 sensing voltage/retry 증가). retry 수는 마모 stage(new/middle/old)별 [11] 데이터로 주입.
4. 핵심기법: plane 내부를 저마모영역 `LWA`(hot-read 데이터, read 서비스)와 고마모영역 `HWA`(write 서비스)로 물리 분할(block offset 1변수).
5. 의미: read 대상 데이터를 신뢰도 높은 블록에 모아 LDPC retry를 줄여 read latency를 낮춤.
6. 확장: hot-read 식별 후 HWA→LWA 마이그레이션, 임계 `NR`(read 횟수)을 마모 stage별 5/4/3로 적응 조절.
7. 구현: 페이지별 read counter(mapping table +3bit), LWA full 시 FIFO victim으로 LWA→HWA 반환, 마이그레이션은 GC/WL/host-update에 편승해 저비용.
8. 수명 최적화: LWA 크기 제한(hot-read는 소량, 10% 데이터가 4회+ read), HWA 마모율 80% 상한, 초과 시 write를 LWA로 유도.
9. 결과: old stage에서 SWL 대비 read latency 45%(PWL 대비 33%) 감소, 마이그레이션 오버헤드 0.22~0.42%, TBW(수명) 영향 무시할 수준.
10. 한계: locality 없는 workload(STG_0)엔 효과 미미, HW 미설계, LDPC 코덱 자체는 손대지 않음(시스템 FTL 기법).

| LDPC read-retry / sensing voltage 단계 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` (soft-read threshold) | 개념적 연결만: retry↔soft-read 비용, 알고리즘 이식 아님 |
| LDPC 복호 자체 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 논문이 디코더를 변경하지 않음 → 이식 요소 없음 |

적용 가치: **낮음** — NAND read latency를 FTL 차등 마모/데이터 배치로 개선하는 시스템 레벨 기법으로, Prime ECC(부호·디코더·HW codec)에 떼어낼 알고리즘이 없다.
