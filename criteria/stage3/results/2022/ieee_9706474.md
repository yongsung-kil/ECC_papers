### B. 알고리즘 요약

1. 대상은 layered 스케줄 QC-LDPC(예: `(9472,8192)`, rate 0.8649, VN degree 4, CN degree 29/30, 10 layer)로 flash 컨트롤러급 고rate 부호.
2. 문제: 저비트폭 균일 양자화 Min-Sum은 성능 열화가 크고, LUT 기반(FAID/IB/Min-IB) 디코더는 노드 degree·iteration 수에 비례해 LUT 메모리가 폭증.
3. 핵심: 각 노드에 Reconstruction `R(·)`(b_e→b_i bit LLR 복원) → Computation `F(·)`(min 또는 boxplus) → Quantization `Q(·)`(b_i→b_e bit) 3모듈을 붙여 외부 메시지는 저비트폭, 내부 연산은 고정밀로 수행.
4. `Q(·)`는 부호비트+크기 대칭 양자화(임계 `τ_j`), `R(·)`는 단조 크기복원. msRCQ는 CN이 min이라 `R_c/Q_c` 불필요 → `(b_e, b_v)`만으로 정의.
5. 파라미터 설계는 discrete density evolution로 각 iteration 메시지 PMF를 추적, HDQ(hierarchical dynamic quantization, mutual-info 최대화 greedy + golden-section search, 복잡도 `O(2^b log(γ) log(B))`)로 임계·복원값 산출.
6. 확장: layered 스케줄에서 layer마다 메시지 PMF가 달라 단일 `R^(t)/Q^(t)`가 부정합(Representation Mismatch) → layer-specific `R^(t,r)/Q^(t,r)`을 base matrix `Hp` 기반 layer-specific HDQ DE로 설계.
7. FPGA 구현 3방식(Lookup/Broadcast/Dribble): Broadcast는 파라미터를 중앙 저장 후 VN에 배선, 크기 복원은 4x1 MUX·양자화는 thermometer→binary 디코더로 구현.
8. 결과1: IEEE 802.11 부호에서 4-bit msRCQ가 IB/Min-IB보다 낮은 error floor + BP 상회, LUT 저장 4-bit IB 14.43k/Min-IB 10.24k bit 대비 msRCQ는 135~165 bit로 대폭 절감.
9. 결과2: `(9472,8192)`에서 layer-specific msRCQ(3,8)이 5-bit OMS(5,7) 대비 LUT·routed net >10%, register >6% 감소, 성능은 동등(FER 1e-8 근처 ±0.02dB), Zynq UltraScale+ 500MHz 충족.
10. 한계: gatecount/throughput 절대치 본문 미제시(FPGA 자원 상대비교), AWGN 채널 시뮬 중심으로 실제 NAND soft-read 채널 검증은 미포함.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min 기반 CN 연산 (msRCQ) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리 min-sum CN에 그대로 대응 |
| VN 처리 + R/Q 양자화·복원 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | 저비트폭 외부메시지 + 고정밀 내부연산 이식 지점 |
| iteration/layer별 R,Q 파라미터 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 적응형 LLR 테이블을 RCQ 임계/복원값으로 확장 |
| 채널 LLR 양자화(HDQ) | channel.cpp Set_R_Offset(), Set_LLR_Th() | 채널 관측 비균일 양자화 임계 설계에 대응 |
| QC-LDPC layered 구조 로드 | ecc_top.cpp Load_PCM() | base matrix Hp 기반 layer 분할과 정합 |
```

적용 가치: **높음** — 이미 보유한 Min-Sum + 적응형 LLR 테이블 구조에 layer/iteration별 비균일 R/Q만 얹으면 되어 이식 난이도 낮고, 저비트폭에서 자원·error-floor 동시 개선이라 Prime ECC 관심영역과 직결.
