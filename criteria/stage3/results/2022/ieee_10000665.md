### B. 알고리즘 요약

1. 시스템: binary LDPC(BPSK+AWGN), flooding schedule, regular 코드 2종(R=0.84 dv6/dc32/N2048, R=0.5 dv4/dc8/N7648).
2. 문제: 2~3bit 초저해상도 메시지 전달 시 기존 유한알파벳 디코더는 VN 양자화가 CN 동작을 무시해 성능 손실.
3. 관찰: CN(min 근사) 출력 신뢰도는 입력 중 최저 신뢰도에 지배 → 고신뢰 입력은 출력에 거의 영향 없음.
4. 핵심기법: VN 압축연산 `Qv`를 `max I(B;Tv)`(기존) 대신 `max I(X;Tc)`(CN 출력 상호정보) 기준으로 최적화(check-node-aware).
5. CN: box-plus `(2)`의 correction term 무시한 최소근사 `(3)` `t=sgn·sgn·min(|t1|,|t2|)` 사용.
6. VN: computational-domain 구조 + 균일양자화 `Qv(y)=sgn(y)·min(|y|/2^r+1, 2^(w-1))` `(6)`, 단일파라미터(∆,r) 최적화.
7. HW 친화: 균일양자화는 r-bit right shift + clipping으로 소수 게이트로 구현, 2-bit CN은 XOR+OR 2게이트(3-bit는 9게이트).
8. 최적화: 이산 density evolution으로 `p(b,tch),p(b,tc)` 추적하며 (∆,r) grid search로 `max I(X;Tc)`.
9. 결과: 2-bit에서 0.16~0.2dB 이득, 수렴 iteration 약 1/3, 2-bit CN이 게이트 2/9·지연 1/4, 2-bit aware가 32-bit BP 능가.
10. 한계: 시뮬만(HW는 게이트회로 비교 수준), 2-bit에서 에러플로어 약간 상승, non-QC regular 코드 대상.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN 양자화(computational domain, 균일양자화 Qv) | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | VN 압축·양자화 재설계 지점 직접 대응 |
| iteration별 LLR/translation 테이블(φch,φc) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | φ 테이블 값을 CN-aware 최적화로 교체 |
| CN 최소근사(box-plus min) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 min-sum 보유 — CN은 그대로, VN만 손봄 |
| soft-read 2/3bit 양자화 | channel.cpp Set_LLR_Th(), Set_R_Offset() | 2~3bit 메시지 해상도 정합 |
```

적용 가치: **높음** — Prime ECC가 이미 보유한 min-sum/2~3bit soft를 그대로 두고 VN LLR 테이블만 CN-aware 기준으로 재최적화하면 추가 HW 비용 없이 waterfall·iteration 개선 여지가 있어 이식 난이도가 낮다.
