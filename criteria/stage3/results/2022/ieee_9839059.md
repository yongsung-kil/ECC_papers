> 총평: binary QC(protograph) LDPC의 error-floor를 부호구조 소폭 수정(하삼각 에지추가)으로 낮추는 실용적 아이디어지만, PBRL/degree-1 VN 구조 의존이라 고rate NAND QC-LDPC 이식엔 재설계 필요.

### B. 알고리즘 요약

1. 시스템/코드: protograph 기반 raptor-like(PBRL) LDPC, cyclic lifting으로 QC-LDPC화. 5G NR base graph 2(42×52 protomatrix, 앞 2 VN puncture), z=80, info 800/code 4000, R=1/5(~1/3), QPSK+AWGN, SPA 최대 50 iter.
2. 문제: PBRL은 IRC의 degree-1 VN 다수로 waterfall/thr는 좋지만 error floor가 높음. URLLC/xURLLC는 BLER≪10^-5 요구 → 낮은 error floor 필요.
3. 핵심아이디어: PBRL protomatrix `P=[[PHRC,O],[PIRC,I]]`의 확장부 I(하삼각)에 소수 에지 추가. 기존 PHRC/PIRC 에지 불변 → 인코더/디코더에 로직 소량만 추가, rate-compatibility·원부호 호환 유지.
4. 핵심기법1(원리): 하삼각에 추가한 에지가 degree-1 VN `v_i^(2)`→`c_i^(2)` 메시지를 채널값+추가 extrinsic로 강화, 이를 통해 HRC의 약한 VN(`v3^(1)`,`v6^(1)` 등) 신뢰도 boost → error floor 저감.
5. 지표: HRC VN별 subblock BER `m_v`를 신뢰도 지표로 시뮬로 측정, 불안정 VN에 에지 집중.
6. 알고리즘: Alg1 전체 절차 / Alg2 V(tgt)(총 에러의 1/2 기여까지 최불안정 VN 수집)·V(sel) 구성 / Alg3 각 v의 추가 check node 위치 `Cv(sel)` 결정.
7. 제약(HW/성능 친화): check node당 추가 에지 최대 1개, 하삼각 내 사이클 방지, protograph 4-cycle 회피(불가피 시 두 VN metric 합 최소화). 추가 에지 lifting은 EMPEG로 cycle length·extrinsic degree 최대화.
8. 학습/최적화: 별도 NN 없음. subblock BER 시뮬 기반 greedy 선택.
9. 결과: n(e)=4,5,6,8 에지 추가 시 원 5G NR 부호 대비 error-floor 영역 개선(BLER<10^-5에서 우위), waterfall은 소폭 악화(waterfall↔floor tradeoff). R=1/5 설계가 R=1/4,1/3에도 유효.
10. 한계: HW 미설계·gatecount/throughput 미제시, PBRL degree-1 VN 구조·rate-compatible 확장부에 의존, 지표를 시뮬로 얻어야 함.

### C. Prime ECC 관련 모듈 핀포인트

대상이 both(부호설계가 error-floor 직접 개선). 아래 모듈 대응.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 에지 추가로 H-matrix/protograph 수정 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 변경 → 부호설계 레이어, Prime ECC는 고정 QC-LDPC라 재검증 부담 큼 |
| 인코더 확장부 변경 | [encoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 에지 추가가 부호 PCM 변경 유발 → 인코딩 재생성 필요 |
| SPA 반복 복호(에지 추가된 그래프) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | Prime ECC는 min-sum, 논문은 SPA(full sum-product) — 알고리즘 상이 |
| 추가 에지 lifting(EMPEG, 4-cycle 회피) | 대응 없음 | Prime ECC엔 그래프 구성/EMPEG 도구 없음(고정 base+shift) |
| subblock BER 지표(시뮬 기반) | [channel.cpp](../Prime_ECC_3.1_Claude) (RBER/fixed-error) | error 주입 시뮬로 VN별 BER 측정은 가능하나 별도 계측 파이프 필요 |

적용 가치: **중간** — error-floor 저감이라는 관심 주제이고 binary QC-LDPC 대상이라 방향은 맞으나, PBRL degree-1 VN·rate-compatible 확장부에 원리가 묶여 있어 Prime ECC의 고rate 고정 QC-LDPC(punct_col 2개, VN degree 17)에 그대로 옮기려면 부호 재설계·재검증이 필요.
