### B. 알고리즘 요약

1. 시스템: TLC(3bit/cell) Flash, 8-PAM constellation, 각 bit(B1/B2/B3)이 독립 page로 인코딩(low-latency), 모든 page 동일 rate 제약. AWGN 모델 `Y=X+Z, Z~N(0,N)`.
2. 문제: independent encoding + equal-rate 제약 하에 ESEL(등간격·등확률) constellation은 page별 MI가 불균등 -> 최약 bit-channel이 전체 rate 제한 (`3·minj(Ij) < I(X;Y)`).
3. 모델: TLC를 3-user MAC로 해석, 각 bit `Bj`의 `I(Bj;Y)`를 독립 계산, cell PMF `p(xi)=∏P(Bj=bj(i))`.
4. 핵심 기법(Alg.1): dynamic-assignment Blahut-Arimoto(DAB) 변형 - equally-likely 유지하며 write-level 위치만 이동. 최소 MI page 식별 후 대칭쌍 이동방향 `D̃=el−er`, line search로 `λ*` 최적화, 전력제약 위해 대각행렬 `d̃`로 스케일.
5. 핵심 식: `max min I(Bj;Y)` s.t. 전력제약 `Σp(xi)xi²=EESEL`, 대칭 `xk=−x(M−1−k)`, 제로평균 - 최약 페이지 MI를 올려 세 rate 균등화.
6. 확장(Alg.2): 위치+확률 동시 최적화. `p2,p3`를 gradient descent(`△p=−∇f, α=0.01`)로 조정, 전력제약 위해 constellation 재스케일. equally-likely 아닌 점 허용.
7. 구현 디테일: MATLAB `integral()`로 MI, `fminbnd` line search. 대부분 SNR에서 50 iteration 미만 수렴.
8. (학습절차 해당 없음 - closed-form 최적화)
9. 결과: equal-rate constraint loss 거의 0(전 SNR), independent encoding loss는 15dB 이상서 0, shaping loss만 10~22dB에서 최대 ~0.15bit. 최약 page rate 유의 개선.
10. 한계: LDPC 부호/디코더 미포함, 순수 MI 시뮬(HW 없음), 최적 constellation은 probabilistic amplitude shaping(PAS) 코딩 필요, single AWGN 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| write-level constellation MI 최적화 | 대응 없음 | write-voltage shaping은 부호/디코더 레이어 아님, Prime ECC 범위 밖 |
| soft-read threshold / 채널 모델 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | AWGN/read threshold 개념은 간접 접점이나 write-level 최적화는 미대응 |
| page별 bit-channel MI | 대응 없음 | Prime ECC는 고정 QC-LDPC 부호로 page-rate 균등 최적화 대상 아님 |

적용 가치: **낮음** - NAND 직접 타깃이나 개선 레이어가 write-voltage constellation shaping(코드모듈레이션)이라 Prime ECC의 부호설계·디코더·HW에 떼어 쓸 요소 없음.
