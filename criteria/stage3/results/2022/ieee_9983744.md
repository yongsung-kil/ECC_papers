### B. 알고리즘 요약

1. 시스템: 5G NR/WiMAX/DVB-S2X용 QC-LDPC codec의 FPGA HW; 인코더는 모든 5G NR 부호, 디코더는 임의 QC-LDPC/QC-like 부호 지원. 최적화 지표는 HUE(throughput/area).
2. 문제: 유연성(flexibility)과 고throughput을 동시에 얻기 어렵고, 고병렬화는 HW 자원·유연성을 희생. 부분병렬 구조에서 HUE 극대화가 목표.
3. 인코더: 5G NR은 forward-substitution 기반, circulant 크기 `Zc`의 순환시프트+XOR이 지배적. 큰 shift를 작은 shift 다수로 분할 가능한 성질로 CSN(cyclic shifter network)을 재구성 — 큰 Zc는 단일 대형 CSN, 작은 Zc는 다수 소형 CSN으로 동작해 짧은 부호에서 병렬 처리.
4. 인코더 최적화: PCM circulant 행 순서를 GA로 재배열해 동시에 시프트되는 정보비트 그룹 최대화 → 짧은 부호 encoding throughput 최대 183% 향상.
5. 디코더: layered 스케줄(circulant 행 단위 처리)로 flooding 대비 수렴 빠름. VNU/CNU + LLR RAM + Mc→v RAM + 순환시프터 구조(Fig.1).
6. 문제(디코더): heavy pipeline은 클록↑ 가능하나 data hazard 발생 — 직전 layer의 LLR 갱신 전 사용 금지라 stall cycle 삽입 필요 → throughput↓.
7. 핵심 기법 hybrid schedule: hazard 시 stall 없이 old(out-of-date) LLR을 그대로 읽고 CN 기여를 다르게 전파 → conflict 시 flooding, 그 외 layered처럼 동작. 임의 pipeline stage 수 허용.
8. 최적화: out-of-date LLR read 수를 최소화하도록 PCM 처리순서를 GA로 재배열 → EC 성능 손실 거의 무시할 수준; 필요 시 iteration 1~수회 추가로 보상.
9. 결과: 동일 EC 성능에서 hybrid가 conventional layered 대비 30~110% 빠름. FPGA에서 WiMAX 1.77Gbps, DVB-S2X 4.32Gbps, 5G NR 4.92Gbps(10 iteration).
10. 한계: TELFOR 확장초록 형태라 알고리즘 상세식·gatecount·rate/length 미기재, EC 성능 손실을 iteration 추가로 상쇄하는 경우 존재.
