### B. 알고리즘 요약

1. 문제 설정: 메시지패싱 노드(CN/VN)에서 입력 `x=(x1..xn)`으로부터 각 출력 `yj`를 `xj` 제외한 이진트리로 계산하는 구조 `S`의 complexity(내부노드 수)·latency(최장경로) 최적화.
2. 대표 예: min-sum CN 연산 `yj = min_{i≠j} xi`. 기존 forward-backward 구조(Fig.1)는 complexity `3n-6`, latency `n-2`.
3. 기여1(complexity-optimal): 임의 구조의 최소 complexity가 `3n-6`임을 함수 `f`/Lemma1로 증명, 이때 최소 latency는 `δ+⌈log(n-2^δ)⌉`, `δ=⌊log(n/2)⌋`.
4. T-tree 도입: complexity-optimal 구조 집합 `S_n^co`를 내부노드 degree-3인 트리(T-tree) `T_n`과 전단사(bijection) `h`로 등가 기술 → latency 분석 단순화(Theorem 3).
5. 기여2(latency-optimal): 최소 latency가 `⌈log(n-1)⌉`이고, `n=2^k+1`일 때 그 구조의 최소 complexity는 `n·log(n-1)` (Theorem 5,6).
6. 두 최적을 동시 만족하는 구조는 `n=3,4,6`에서만 존재(Corollary 1).
7. 기여3(트레이드오프): 주어진 `(n,τ)`, `τ≥⌈log(n-1)⌉`에 대해 latency≤τ이면서 complexity가 최소라 추정되는 구조 `S_{n,τ}`를 Construction 3/4로 구성. 부트리 재사용으로 중간결과 공유.
8. 알고리즘 복잡도: 구성법(Algorithm 1, DP)은 `O(n^3 log^2 n)` 시간, 결과 complexity `≤ n⌈log n⌉-2` (보통 훨씬 작음, Table I).
9. 결과: 802.11n LDPC(CN degree 7,8) 예시 - 기존 구조 대비 동일 complexity에서 latency 5→4(deg7), 6→4(deg8)로 감소하는 구조 존재.
10. 한계: 순수 이론(그래프 구성/증명)뿐, 실제 comparator/mux 비트폭·지연·HW 합성은 범위 밖; min-sum 정정성능 자체는 불변; 최소성은 Conjecture 1로 미증명.
