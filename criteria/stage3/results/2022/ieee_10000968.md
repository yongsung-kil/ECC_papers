### B. 알고리즘 요약

1. 시스템: SSD, [18432, 16640] QC-LDPC(rate≈0.903) + [32,31] RAID-5, virtual 3-bit-over MLC NAND, AWGN.
2. 문제: ECC-only 및 기존 ECC-RAID-5는 stripe에 UECC가 2개 이상이면 XOR 복구가 불가 -> 정정능력 한계.
3. 모델: 각 데이터는 2-bit SD(HD+reliability)로 표현(TABLE I: strong/weak 0/1), 3-bit SD 디코딩 채널 가정.
4. 핵심기법1(data checking): stripe 내 타깃 제외 데이터로 check HD = `⊕ x_HD^(l)` (식1), check SD = `min_l x_SD^(l)` (식2).
5. 식 의미: check SD의 element-wise min은 LDPC CN의 min-sum과 동형 - stripe 데이터의 최소 신뢰도를 전파.
6. 핵심기법2(data regenerating): 타깃 2-bit SD + check 2-bit SD -> 새 3-bit SD(HD/SD/TD) 생성, exhaustive search로 얻은 irregular 매핑 룰 사용(기존 실패한 regular 룰과 다르게).
7. 구현 트릭: min(식2)의 HW 부담 회피 위해 bit-wise AND(Alg1)/조건부 compare(Alg2)로 등가 변환, latency 최적화 위해 loop 순서 상이.
8. 최적화: regenerating 룰을 ECC 3-bit SD 디코더 기준 exhaustive search로 결정(ECC/RAID 종류별 재최적화 필요).
9. 결과: NTRBER 기준 UBER 1e-15에서 ECC-only 대비 15%, ECC-RAID5 대비 7% 이득(T=31); T=4는 14%/6%. productized SSD 실측 시연.
10. 한계: 디코더/인코더 무변경(동일 엔진 재사용), gate count/throughput 무기재, 개선은 RAID 시스템 레이어 한정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check SD의 min 연산 | decoder.cpp CNU_Update_New_Mag() (개념적 유사) | min-sum과 동형이나 이미 보유 기법, 이식 가치 없음 |
| RAID-5 XOR 기반 data checking/regenerating | 대응 없음 | Prime ECC는 단일 코드워드 디코더, RAID 시스템 레이어는 범위 밖 |
| 3-bit SD 채널/양자화 | channel.cpp Set_LLR_Th() (부분) | soft-read 표현은 있으나 논문 기여의 핵심 아님 |
```

적용 가치: **낮음** — LDPC 디코더/부호를 전혀 건드리지 않고 RAID 시스템 레이어에서 SD 데이터를 재가공하는 스킴이라, min 연산 동형성 외에 Prime ECC로 이식할 요소가 없다.
