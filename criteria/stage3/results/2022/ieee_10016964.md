### B. 알고리즘 요약

1. 시스템: outer 정정 LDPC(`Router≈0.85`, `N=2600`)와 inner error-reducing LDPC(`Rinner≈0.29`) 사이에 permutation/interleaver를 둔 직렬 concatenation, 전체 `R≈0.25`.
2. 문제: 5G 저rate LDPC 단독 대비 FER 곡선의 급경사(sharp slope)와 낮은 SNR 성능을 얻고 싶음.
3. 채널: AWGN + BPSK 가정.
4. 핵심 기법(인코딩): info → outer 인코딩 → permutation `π` → inner 인코딩. 병렬 concatenation의 permutation을 직렬에 차용한 woven형 구조.
5. 핵심 아이디어: inner code를 forward error correction이 아니라 **error reducing**(bit error 감소)로 사용해 outer 디코더의 superchannel을 개선.
6. 복호: woven code 복호(Kreshchuk et al.) 차용. inner/outer 모두 soft-in soft-out. inner 입력을 `[π(outer info) | ext_outer]`로 구성, `-=D_in(...)`, `+,=D_out(...)`을 iteration 간 extrinsic 교환.
7. inner 최적화: 초기 protograph(5G LDPC, 24x34, `Rinner≈0.29`)에서 랜덤 위치 flip → ACE 확장 → concatenated FER 시뮬 → target FER의 SNR 개선 시 채택. 150 iteration 수행.
8. 최적화 결과: protograph에 unit diagonal 형성 + 밀도 감소 → LDGM(low-density generator matrix) 성격. 단, error-floor는 상승.
9. 결과: `N=4420, R≈1/4`에서 5G LDPC(5120,1280) 대비 FER 0.01에서 약 0.2dB 이득, 직렬 concat 특성상 최소거리=성분부호 최소거리 곱으로 FER 급경사. iterative decoder 10회 + inner/outer BP 각 5회(총 50 BP).
10. 한계: HW 미설계, throughput/gate 무보고, 저rate 무선 대상, inner code error-floor 상승.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| inner/outer LDPC BP 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 단일 LDPC BP는 대응되나 woven iterative 교환 스케줄은 우리 구조에 없음 |
| protograph 최적화(flip+ACE 확장) | `ecc_top.cpp` `Load_PCM()` / H-matrix | 우리 QC-LDPC는 고정 H, concatenated protograph 재설계는 이식성 높음(부담 큼) |
| interleaver/permutation between codes | 대응 없음 | concatenation 구조 자체가 Prime ECC에 없음 |
| soft extrinsic 교환(iterative) | 대응 없음 | 단일부호 min-sum 파이프라인이라 turbo형 extrinsic 루프 없음 |

적용 가치: **낮음** — 이중 LDPC concatenation + interleaver + woven iterative 복호는 우리의 단일 고rate binary QC-LDPC min-sum 엔진과 아키텍처가 근본적으로 다르고, 저rate·error-floor 상승·HW 미설계로 NAND ECC 요건과 상충.
