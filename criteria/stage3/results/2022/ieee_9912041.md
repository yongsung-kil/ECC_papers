> 총평: min-sum flooding 디코더의 CNU를 MB(min1/min2/sgn)와 LVC-B로 쪼개 열간 의존성을 없앤 저latency HW 기법으로, Prime의 min-sum 파이프라인 스케줄 개선 아이디어로는 참고가치가 있으나 저rate 무선 코드·부동소수 대비 손실이 있어 이식은 중.

### B. 알고리즘 요약

1. 시스템: IEEE 802.11ad(WiGig) QC-LDPC, base matrix `B∈C^{8×16}`, expansion factor `z=42`, mother rate 1/2, block length 672bit(16×42), BPSK/AWGN, 7-bit 양자화(Q4.3).
2. 문제: flooding min-sum을 단일 CVPU로 처리하면 column-slice 간 min 비교가 순차 의존이라 latency가 열 수에 선형 증가.
3. 모델: 표준 min-sum(Algorithm 1) — `mcv = (∏sign)·min|Lvc|`, `Lv=Σmcv+Qv`, `Lvc=Lv-mcv`, syndrome=0 조기종료.
4. 핵심 기법: CNU를 둘로 분리. min-block(MB)이 행별 `sgn`(식1), `Min1`(식2), `Min2`(식3, Min1 인덱스 제외)만 먼저 계산.
5. 핵심 식: `mcv = sgn_i·sign(Lvc)·α`, `α=Min2 (|Lvc|=Min1일 때) else Min1` (식4,5) — index 값 없이 Min1/Min2만으로 mcv 산출.
6. 확장: 나머지 mcv/Lvc/Lv 계산은 LVC-Block(LVC-B)이 담당. MB는 1클록에 전부 병렬, LVC-B는 추가 1클록 → column 간 의존 제거.
7. 구현 트릭: rejection-based Min1/Min2(4×2-MIN, 3×2-MIN, index unit 제거), sign-magnitude/2's-complement 변환기, VN당 최대 4 CN으로 interconnect 축소, 4-stage 파이프라인.
8. 최적화: 없음(학습/DE 없음). RCU 8개·CCU 16개가 각 z=42 MB/LVC-B로 완전 병렬.
9. 결과: iteration당 4 clock, 217MHz@130nm, core 6.02mm2, 3.6Gbps(10 iter), scaled 36Gbps/iter로 SOTA 대비 +85.64% throughput·+41.02% 에너지효율; BER 10^-5 @1.2dB(10 iter), 7bit 양자화 손실 0.6dB@BER 10^-3.
10. 한계: rate 1/2·672bit 무선 코드 한정, error-floor/정정능력 개선 없음, min-sum α 근사로 부동소수 대비 손실.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MB의 Min1/Min2/sgn 계산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime도 min1/min2+sign XOR 방식 — 동일 연산의 HW 분리 스케줄 |
| mcv=α(Min1/Min2) LVC-B 계산 | decoder.cpp C2V_Cal(), VN_Cal_HD() 등 | C2V/VN 갱신 지점과 대응 |
| iteration당 4-clock 파이프라인/열 병렬 | decoder HW 모델(z=32, pipeline 6-stage) | Prime은 6-stage — MB/LVC 분리 스케줄과 정합 검토 대상 |
| 7-bit LLR 양자화 | ecc_data.h PARAM_LLR, channel.cpp Set_LLR_Th() | Prime은 HD/2SD/3SD — 7bit 무선 양자화는 직접 이식 아님 |
| flooding schedule | 대응 없음 | Prime은 dual-update(even/odd) 교대 — flooding 자체는 별개 |
```

적용 가치: **중간** — CNU를 min-block/LVC-block으로 분리해 열간 의존을 없애는 파이프라인 재구성 아이디어는 Prime min-sum 디코더의 클록당 처리량 개선 관점에서 참고할 만하나, z/스케줄/rate/양자화가 모두 다르고 정정능력 이득은 없어 직접 이득으로 옮기려면 재정합 부담이 크다.
