> 총평: 자기 부호(SC-LDPC)와 BCJR 검출기를 window 단위로 함께 도는 ISI 채널 turbo equalization 최적화로, NAND binary QC-LDPC 이식 관점에서는 관련성 낮음.

### B. 알고리즘 요약

1. 채널: ISI(자기기록/BPMR) `HISI=[1 1]`(PR1), `[1 2 1]`(PR2) + AWGN; 부호는 protograph SC-LDPC, medium-rate `R=0.490`(B=[1 1]), high-rate `R=0.796`(B=[1 1 1 1 1]), coupling `L=100`, 총길이 `nvLN=20,000`bit.
2. 문제: 기존 turbo equalization(C-TE)은 SC-LDPC 디코더만 window로 돌고 BCJR 검출기는 전체 블록을 끝내야 시작 → BCJR이 `L` 트렐리스 전체 branch metric 저장(메모리·지연 큼).
3. 핵심 기법: **SW-TE(sliding-window turbo equalization)** — BCJR 검출기와 SC-LDPC 디코더를 **모두 window 크기 `W` 안에서** 반복 메시지 교환, 위치 `t`의 갱신 LLR을 즉시 다음 위치 초기값으로 사용하며 window slide.
4. 메모리: BCJR branch-metric 저장이 `M_bm=2^(D+1)·W`로 C-TE의 `2^(D+1)·L`보다 작아짐(`W<L`).
5. 분석 도구: **P-EXIT chart**를 SW-TE용으로 수정(변수노드별 a priori MI를 따로 추적)해 asymptotic decoding threshold 계산.
6. 결과(threshold): SW-TE가 C-TE보다 항상 낮은(우수) decoding threshold, medium-rate PR2에서 `IterTURBO=6`일 때 약 0.87 dB 개선. 단 branch-metric 갱신수 `U1b-mnode`가 크게 증가(24→227.5 등).
7. 복잡도 절감 기법1 **irregular-updating**: MI가 포화된 변수노드(연속 turbo iter에서 개선 없음, 비 `βUB`)의 branch metric 갱신을 다음 iter에서 생략.
8. 기법2 **dynamic-shifting**: 1번 변수노드와 비슷한 신뢰도(비 `αLB`)인 노드를 한 번에 출력해 window를 여러 위치(`s`) 건너뜀 → 불필요 decoding position 제거.
9. 기법3 두 기법 결합(standard shift `s=1`이면 irregular-updating 미적용 규칙). 결과: `U1b-mnode`를 2배 이상(~54~80%) 감소, BER 손실 없음(threshold 손실 ~0.05~0.08 dB), high-rate PR2에서 coding gain 0.25 dB.
10. 한계: 전 과정 시뮬(컴퓨터)뿐 HW 미설계, ISI 채널 + SC-LDPC + BCJR turbo equalization 특화, sum-product/BCJR 기반이라 NAND 무-ISI binary QC-LDPC와 채널·구조 정합 안 됨.
