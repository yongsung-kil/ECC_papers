### B. 알고리즘 요약

1. 대상은 BSC(crossover `α`)상의 (N,K) LDPC, regular/irregular 모두, 정수연산 기반 hard-decision 비트플립 디코더.
2. 문제: 유한길이 LDPC의 trapping set 탓에 BP가 ML 한계에 못 미치고 error floor 발생; FAID는 column-weight-3에 한정, 고차수/irregular엔 설계 난이도 큼.
3. 기반 알고리즘 GDBF-w/M: 각 반복에서 역에너지 `E(vi)=a(xi⊕yi)+b·Σ_{j∈P(vi)} sj − m[τi]` 계산 후 최대치 비트 flip (식1,2), momentum 벡터 `m`(비증가 정수열)로 국소최적 탈출.
4. 핵심 기법 diversity: 성분 디코더 `Dect(zt,Lt,at,bt,mt)`를 순차 연결, restart 스위치 `rt∈{0,1}`로 이전 출력 재사용 or 채널벡터 재시작 선택 (AD-GDBF).
5. 파라미터 `rt,at,bt,mt`를 GA(유전 알고리즘)로 최적화; 결정론적(RNG 불필요)이며 regular 코드는 정수연산만으로 구현.
6. 학습 절차(Algorithm 1): Dec0로 Monte-Carlo하여 오정정 패턴집합 `E(0)` 수집 → 라운드별 성분 디코더를 GA fitness(정정 패턴수 최대화)로 학습, `Nlow/Nup` 임계로 overfitting 방지·패턴집합 확장.
7. 구현 디테일: 곱셈 불필요(×2,×4는 shift, ×3은 x+2x 가산 `β`개), N−1 magnitude comparator + N equality comparator.
8. 최적화는 exhaustive search 대신 GA (구성 수 ~10^20로 전수탐색 불가).
9. 결과: Tanner(N=155,R=0.4), iRISC QC(N=1296,R=0.5), 802.3an(N=2048,R=0.84), 5GNR irregular(N=1276,R=0.75)에서 (P)GDBF-w/M·ISBF 초과, BP error-floor를 1자릿수 낮춤(post-processor로 L=1000). FAID가 못 푸는 weight-6 패턴을 210 iter에 정정.
10. 한계: BSC(hard) 전제, HW 미설계, 최대 반복 수백~1000으로 latency 큼, 성능이 학습 error-pattern 집합 품질에 의존.
