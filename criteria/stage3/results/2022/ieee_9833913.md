### B. 알고리즘 요약

1. 시스템: 5G NR PUSCH, protograph LDPC(BG1/BG2), lifting size `Zc`, 첫 2Zc bit puncturing, AWGN/TDL-A, `Im=10` iteration.
2. 문제: NR BG의 degree-1 VN 다수로 min-sum이 BP 대비 성능 열위 → HW 저복잡 유지하며 min-sum 정정성능 개선 필요.
3. 모델: layered NMS(LNMS) 기반, CN→VN 메시지 `Rmj = αi·∏sign·min`, VN 갱신 `Li(cj)=Li(qmj)+Rmj`.
4. 핵심기법1: TB 내 filler bit(항상 0, 위치 기지) 이웃 `Ñ(m)`을 min·sign 연산에서 제외 → 잘못된 belief 전파 억제(식 8), hard decision도 filler는 0 강제(식 9).
5. 핵심 식: filler LLR이 이상적으로 `→∞`여야 하나 채널로 유한값 → 이를 제외해 오류 전파 감소.
6. 핵심기법2: iteration별 normalization factor `αi`를 단일 스칼라가 아닌 전 iteration 동시 최적화.
7. 구현 디테일: runtime 곱셈 수는 LNMS와 동일, 메모리만 `O(1)→O(Im)` 증가(α가 αi로), α는 2의 거듭제곱 아님 → shift 대체 불가.
8. 학습: forward-propagation DNN(hidden layer = 1 iteration), 목표는 (8)(9) 적용한 LBP 출력 LLR, MSE loss(정보비트만), Adam(lr=0.001), 10000 TB 80:20, 특정 SNR(10% BLER)에서 학습.
9. 결과: LNMS 대비 0.3~1.9dB, 타 DNN 디코더[10] 대비 0.3~1.7dB BLER 개선(AWGN), fading에서 0.25~1.8dB, 곱셈기 자원 추가 없음.
10. 한계: filler-bit 트릭이 5G TB 구조 전용, DNN은 MCS·SNR별 학습 필요, HW 전체 설계 아님(복잡도 논의만).
