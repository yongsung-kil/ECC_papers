> 총평: MS의 over-estimation을 LLR 분포 구간별 정규화/오프셋으로 채널·변조 적응 보정하는 min-sum 개선안으로, Prime ECC min-sum 디코더에 알고리즘 이식 가능하나 실수 LLR·6구간 분류의 양자화/HW 재검증 부담이 있어 이식성 중.

### B. 알고리즘 요약

1. 시스템: ATSC 3.0 방송용 LDPC(irregular, RL/IRA 구조), length 16200/64800, rate 2/15~8/15. AWGN/Rayleigh, QPSK/64QAM, 최대 iteration 50.
2. 문제: MS가 BP의 `tanh` horizontal step을 min으로 근사하며 over-estimation 발생. NMS/OMS/VMS의 보정계수 `α`,`β`는 AWGN+QPSK에서 최적화돼 타 채널·변조로 옮기면 성능 손실.
3. 모델: CN→VN 메시지 `U_ij`의 MS값과 BP값 차이를 over-estimation value `OEV=D_ij`(오프셋용), 비율을 over-estimation rate `OER=R_ij`(정규화용)로 정의. `α=1/R`, `β=D`.
4. 핵심 기법 PMS: `OEV/OER`가 iteration이 아니라 LLR 분포에 의해 결정됨을 이용. `tanh(|x|)` 곡선을 두 분할점 `LLRP1`,`LLRP2`로 part1/2/3 구분.
5. 핵심 식: part1은 OEV(D) 안정, part3은 OER(R) 안정 → 구간별로 정규화/오프셋 중 안정한 쪽을 선택해 채널·변조 무관 단일 계수셋 확보.
6. 확장: horizontal step에서 min `MIN`과 second-min `SEC`의 part 위치 조합으로 6개 LLR 분포 `k=1..6` 분류, 각 `k`별 `α_k`,`β_k` 적용(식32/33).
7. HW 친화 단순화 SPMS: `k=1~4`는 오프셋만, `k=5,6`은 정규화만 사용해 계수선택 복잡도 O(n²)→O(n). Taylor 전개로 `LLRP1≈1`,`LLRP2≈5`(δ=0.03,θ=5%) 상수화.
8. 최적화: 각 `k` 집합 `S_k`에서 `α_k`,`β_k`를 offline arg min(식34/39)으로 도출, 한 부호당 한 번만 계산.
9. 결과: FER=1e-4에서 NMS 대비 최대 1.61dB, OMS 0.24dB, VMS 0.36dB 이득(AWGN QPSK). BP와 격차 대부분 <0.2dB. 64QAM/Rayleigh에서도 계수 재최적화 없이 성능 유지, NMS 대비 error-floor 개선. iteration 20으로 제한 시 상대 이득 더 큼(동일 SNR에서 iteration·전력 절감).
10. 한계: HW 미설계(비교기 수만 분석, gate/throughput 없음), 실수 LLR 가정, 방송 채널(AWGN/Rayleigh) 특화로 NAND soft-read 양자화 정합은 미검증.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN horizontal step min-sum 보정(α/β 구간 적응) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 높음 - 우리 min1/min2(min·2nd-min) 구조에 α_k/β_k 구간분류 직접 대응 |
| min/2nd-min(MIN,SEC) 기반 6구간 LLR 분류 | decoder.cpp CNU_Update_New_Mag() (min1/min2 산출) | 높음 - min1/min2 이미 계산하므로 분류 로직만 추가 |
| iteration별/구간별 계수 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 중 - 우리 iteration별 LLR 테이블 자리에 (α_k,β_k) 매핑 가능 |
| LLR 분포 임계 LLRP1/LLRP2 & 양자화 | channel.cpp Set_LLR_Th() | 중 - NAND soft-2~3bit 양자화 격자에 임계값 재정합 필요 |
| BP full-tanh 참조식 | 대응 없음 | 무관 - 우리는 min-sum 전용, BP는 미구현 |
```

적용 가치: **중간** - CN min-sum 보정계수를 개선하는 정확한 대응 모듈(`CNU_Update_New_Mag`)이 있고 min1/min2를 이미 계산하므로 알고리즘 이식 경로는 명확하나, 실수 LLR·6구간 분류를 NAND 2~3bit 양자화·HW 면적으로 옮기는 재검증 부담과 방송 채널 특화 계수가 걸림.
