### B. 알고리즘 요약

1. 대상: TLC NAND flash(셀당 3bit, 8-ASK/AWGN 등가), MSB/CSB/LSB 3페이지, Gray code 매핑. 코드는 DVB-S2류 irregular LDPC, `R=0.83`(n=9870,k=8192)·`R=0.85`(n=38556,k=32768).
2. 문제: page-wise read/encode는 각 페이지에서 일부 reference voltage만 써서 mutual information을 잃고, worst-case page(예: CSB) 용량이 전체 rate `R<min_i C̃_i`를 제한.
3. 모델: 셀 전압분포를 등분산 Gaussian(unipolar 8-ASK)로 모델링, page capacity `C̃_i = I(X_i;Ỹ_i)`, 전체 capacity `C=ΣC_i` (chain rule).
4. 핵심기법: cell-wise read - 셀의 7개 reference를 모두 읽어 BICM demapper가 bit별 LLR `L(x_i)=log Σ_{Ci(0)}P(y|x)/Σ_{Ci(1)}P(y|x)`를 joint 계산.
5. 식 의미: joint LLR은 같은 셀의 다른 비트 정보를 활용해 page-wise 대비 추가 정보(capacity gap)를 회수 -> soft-read 없이 신뢰도 향상.
6. 확장: max-log 근사 `L(x_i)≈log max_{Ci(0)}P/max_{Ci(1)}P`, 출력값이 8개뿐이라 단순 lookup table로 구현.
7. 구현 디테일: enhanced page-wise(enh. pw) - 우선 page-wise로 시도하고 실패 시에만 다른 페이지를 읽어 joint LLR로 재복호(progressive read류).
8. 학습/최적화: 없음(정보이론 분석 + BP 복호, 최대 10 iteration).
9. 결과: hard-decision cell-wise joint 처리가 best-case page-wise보다도 WER 우수; 1소프트비트 soft-input이 hard를 추가로 능가(정성 곡선, 수치 미기재).
10. 한계: HW 미설계·시뮬만, cell-wise는 전 페이지 동시 read로 random read latency↑, binary code만(향후 q-nary 제안).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| joint/cell-wise LLR 디매핑(max-log lookup) | channel.cpp Set_LLR_Th(), Set_R_Offset() | soft-read threshold/LLR 양자화 설계에 매핑 |
| BP 복호(최대 10 iter) | decoder.cpp LDPC_Decoder() | 표준 message passing, 신규성 없음(min-sum과 별개) |
| irregular LDPC(DVB-S2류) 부호 | ecc_top.cpp Load_PCM() | Prime은 QC-LDPC 고정, 부호 자체는 이식 부적합 |
| page/cell capacity 분석 | 대응 없음 | 정보이론 분석, 코드 대응 없음 |
```

적용 가치: **중간** - joint LLR demapper 아이디어는 NAND soft-read 비용 절감 관점에서 LLR 테이블 설계에 참고되나, 부호/디코더 자체는 표준 수준이라 직접 이식할 신규 알고리즘은 없다.
