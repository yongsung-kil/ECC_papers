### B. 알고리즘 요약

1. 대상은 시간불변(time-invariant) SC-LDPC 부호로, 5G 표준 LDPC의 Raptor-like 확장구조(kernel `[AD]` + 확장 `[EI]`)를 참조해 rate-compatible 하게 설계.
2. 문제: SC-LDPC는 커플링 체인 길이로 행렬 크기가 통제 불가라 전체 행렬 저장 시 HW 자원 낭비·복호 복잡도 상승.
3. 채널 모델: AWGN + BPSK, UWA/무선 이동통신 시나리오 가정.
4. 핵심 기법: 슬라이딩 윈도우 + 계층(layered) min-sum. 윈도우를 `L`개 layer로 나누고 각 layer `K`행을 직교화하여 layer 내 동시 min-sum 수행.
5. CN 업데이트 식 `Lm,n(k) = sgn(Z) × min|Z|` (식3)로 min1/min2 비교 기반 근사.
6. VN 업데이트 `Zmn(k) = Zn(0) + ΣLmn(k)` (식4), 사후 LLR `Zn(k) = Zn(0) + ΣLm(kn)` (식5).
7. 조기종료: 판정 codeword를 `H`와 modulo-2 곱해 0이면 해당 layer 복호 종료, `T=T+1`로 다음 layer 진행.
8. HW: control/storage/calculation 3모듈, row-parallel 계층 스케줄. 8입력 데이터를 4입력 비교기로 pairwise 비교해 min/2nd-min 추출.
9. 결과: rate 1/3, info 192bit, coupling chain 100, max iter 40, clock 300MHz에서 3120 cycle/입력, throughput 18.46Mbps, BER 10^-4 수준. BP/LLR-BP 대비 성능 유사~소폭 개선.
10. 한계: gate/LUT count 미보고, 무선/저rate 특화, throughput가 NAND ECC급 대비 극히 낮음.
