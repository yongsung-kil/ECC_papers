### B. 알고리즘 요약

1. 대상은 protograph 기반 GLDPC 부호를 이진소거채널(BEC)에서 설계, 실험은 rate 1/2 (48000,24000)과 rate 2/3 (45000,30000) 부호.
2. 문제: 기존 protograph doping은 CN(SPC 노드)을 component code로 통째 교체 → doping granularity가 component 길이 `µ`로 커서 중~고rate 최적화가 어려움.
3. 핵심가정: GLDPC = 일부 SPC 노드를 최소거리 큰 선형부호(component/GC 노드)로 대체해 최소거리 증대, 채널은 memoryless BEC.
4. 핵심기법1(partial doping): protograph를 lifting factor `N`으로 lift한 뒤 단일 protograph VN에서 복제된 `N`개 VN에 `N/µ`개 GC 노드를 붙임 → doping granularity=1, `µ`와 무관하게 특정 VN만 보호.
5. 핵심식: PD-GLDPC rate `R = 1 - {nc + |X|·(µ-κ)/µ}/nv`, doping ratio `ν = xβ/(xβ+ncN)`; GC 노드가 단일 protograph VN에만 연결돼 평균화 EXIT가 정확해짐.
6. 핵심기법2: GC 노드 EXIT를 단일 a priori mutual info로 닫힌형태(식(2))로 계산 → 빠른 PEXIT 최적화 가능.
7. 구현 디테일: component code는 (7,4)/(15,11) Hamming, GC 노드는 ML(가우스소거) 복호, partial doping + puncturing(precoding 유사)으로 rate 보상.
8. 학습/최적화: differential evolution(g=6000, Np=10·ncnv, F=0.5, pc=0.88)로 BEC threshold 최적화, typical minimum distance 조건(Thm 1: undoped degree-2 VN에 cycle 없음)을 제약으로 사용.
9. 결과: 최적 PD-GLDPC가 doping ratio 0.024로 GLDPC[17](ν=0.4) 대비 coding gain 0.0079, AR4JA 대비 rate 1/2·2/3에서 0.0477·0.032 이득, FER 10^-4까지 error-floor 없음, 평균 VN degree 80~87%로 감소.
10. 한계: BEC 전용(AWGN은 future work), HW 미설계, GC 노드 ML복호로 복잡도·redundancy 증가, 중간rate 위주.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph GLDPC 부호구조/PCM | `ecc_top.cpp` `Load_PCM()` | 우리는 고정 binary QC-LDPC(z=32)라 GC 노드 추가형 protograph 구조 이식 부담 큼 |
| partial doping/puncturing 부호설계 | 대응 없음 | GC 노드(Hamming) 도핑은 우리 SPC-only QC-LDPC 부호설계에 대응 개념 없음 |
| GC 노드 ML(가우스소거) 복호 | 대응 없음 | 우리 디코더는 min-sum 반복복호(`decoder.cpp`), GC 노드 ML복호 미지원 |
| PEXIT/DE 부호최적화 | 대응 없음 | Prime ECC에 EXIT/DE 기반 부호최적화 파이프라인 부재 |

적용 가치: **낮음** — BEC 소거채널·비-SPC(GC 노드) ML복호에 기반한 중간rate GLDPC 부호설계로, AWGN/soft-read 기반 NAND binary QC-LDPC(min-sum HW) Prime ECC로의 직접 이식 경로가 없음. error-floor 없는 부호설계 관점의 참고 가치만 미미하게 존재.
