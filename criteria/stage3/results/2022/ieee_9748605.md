### B. 알고리즘 요약

1. 시스템: 50G PON 상향(US) 링크, G.hsp(G.9804) 표준의 (17280,14592) binary LDPC(=(17664,14592) mother code에서 384bit puncturing), rate 0.844 default.
2. 문제: US burst mode는 ONU별 채널 편차가 크나 표준 FEC는 worst-case 고정 → 여유 채널 ONU의 throughput 손실.
3. 모델: BSC 가정, output BER=1E-12 목표로 input BER(=coding gain)을 평가.
4. 핵심기법: 동일 mother code에 서로 다른 `shortening`(정보비트 0 고정)과 `puncturing`(parity 열 제거)을 적용해 rate/length가 다른 변종 부호군 생성.
5. 의미: 인코더/디코더 재설계 없이 rate 조절만으로 throughput vs margin(BER) 연속 trade-off 확보.
6. 확장: burst 내 부분별(선두 codeword 저rate, 말미 codeword 별도 puncturing)로 다른 부호 적용 가능성 제시.
7. 구현 디테일: puncturing은 최대 12열 중 7열까지 제한(과도 puncturing 시 성능 급락), length 하한 ~8700bit(sustained processing) 권고.
8. 결과: rate 0.891 (16384,14592) 부호는 default 대비 throughput +5.4%(평균 +7.5%), OCG 0.65dB 손실; rate 0.733 (11520,8448)은 +0.59dB margin(throughput -13%).
9. 디코더: layered min-sum, 14 iteration으로 시뮬레이션.
10. 한계: HW 미설계, BSC 가정, 광통신/PON 표준 특화로 NAND 관련 신규 기법 부재.
