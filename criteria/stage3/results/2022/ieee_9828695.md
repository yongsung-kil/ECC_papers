### B. 알고리즘 요약

1. 대상은 ATSC 3.0 방송 물리계층 L1 signaling, length-16200 QC-LDPC(rate 3/15, 6/15), 7개 보호 모드, AWGN.
2. 문제: payload용 잘 설계된 LDPC를 짧은 signaling bit `k`에 그대로 쓰면 극심한 information shortening·parity puncturing이 불가피해 L1 성능이 물리계층 병목이 됨.
3. 모델: QC-LDPC를 base matrix + exponent matrix(CPM shift)로 표현, ATSC 3.0은 lifting size 360 고정.
4. 핵심기법: 5G LDPC식 modulo lifting 도입 — 동일 base matrix·shortening/puncturing 패턴은 유지하되, `k`에 맞춰 shortened bit이 최소가 되도록 lifting size를 360의 약수 중에서 선택.
5. 식 의미: 작은 lifting size `Z`에 대해 exponent를 `mod Z`로 변환(예: Z=60이면 원 exponent mod 60)해 동일 base graph로 짧은 부호를 생성.
6. 확장: 7개 보호 모드 전부에 동일 패턴 적용, non-contiguous lifting size 한계로 잔여 shortening/puncturing만 보완.
7. 구현 디테일: shortening/puncturing 패턴이 base matrix column 단위로 정의되어 lifting size가 바뀌어도 그대로 재사용 가능.
8. 결과: L1-Detail mode3에서 shortening/puncturing 비율 대폭 감소(예 k=600시 5.2%/6.2% vs ATSC 88.1%/88.3%), BLER=10⁻²에서 SNR 이득 최대 0.9dB(k=600), 복잡도는 lifting size 비례라 k=600시 8배 감소.
9. 디코딩은 SPA 50 iteration 가정, 신규 복호 알고리즘 없음.
10. 한계: HW 미설계, exponent matrix 미최적화(joint 최적화는 향후 과제), 방송 signaling 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| base matrix + exponent(lifting) 부호 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | H-matrix 로드 대응, 그러나 특정 QC-LDPC 고정이라 재구성 부담 큼 |
| modulo lifting으로 가변 length 부호 생성 | 대응 없음 | Prime ECC는 고정 lifting `z_sb=32`, 가변 length 미지원 |
| shortening / parity puncturing 패턴 | 대응 없음 | 프로파일에 shortening/puncturing 모듈 없음 (`punct_col_num=2` 정도만) |
| SPA 50 iter 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | Prime ECC는 min-sum 기반, 논문은 표준 SPA로 신규성 없음 |

적용 가치: 낮음 — 방송 L1 signaling의 짧은 가변 length 부호를 modulo lifting으로 만드는 구성법으로, Prime ECC의 고정 z=32·고rate·고정 length binary QC-LDPC 환경과 정합되지 않고 복호 기여도 없다.
