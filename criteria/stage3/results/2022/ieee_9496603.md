### B. 알고리즘 요약

1. 대상: NAND SSD의 periodic data refresh 오버헤드 문제 (SSDsim 트레이스 시뮬, 128GB MLC, read/prog/erase = 95/700µs/3.5ms).
2. 문제: retention 오류 방지용 refresh가 잦아지며 write amplification·read 방해로 성능/수명 저하.
3. 핵심 관찰: refresh 후 원본 데이터는 ECC 정정능력을 넘긴 "uncorrectable data"가 되지만, 근사연산 앱(이미지/영상)은 이를 재사용(UDR) 가능.
4. 핵심 기법 RFR(Remapping-Free Refresh): 전체 페이지를 옮기지 않고, 정정 후 **enhanced ECC parity bit만** 계산해 전용 ECC block에 remap → refresh write 대폭 감소.
5. Enhanced ECC 모델: 기본 ECC로 binary BCH `(n,k,t)`, `n=2^m-1`, `k≥2^m-m·t` 채택. UBER-RBER 관계식과 신뢰성 모델 `log(RBER)=(a·PE+b)·log(RET)+c·PE+d`로 필요 parity 수 산출 (LDPC는 각주에서 "적용 가능"만 언급).
6. AR-RFR: 페이지를 approx/precise read hotness로 4분류(AP_hot/P_cold/A_cold/AP_cold), 카운터 임계 `Tpread=1`,`Taread=3`로 RFR / RBR / RBR+UDR 선택.
7. 구현 디테일: A_READ(ECC 우회 근사읽기, 80µs), P_READ_RF(데이터+parity 2회 읽기, 210µs) 신규 커맨드, PMT/EMT/UMT 3중 매핑 테이블.
8. PV 관리: strong/weak page 구분해 refresh mode별 재배치, TBER=2.2e-3 초과 시 UDR 비활성화.
9. 결과: FCR 대비 refresh latency 36.19% 감소, I/O throughput 21.39% 향상, 에너지 24.70% 절감.
10. 한계: 순수 FTL/시스템 레벨 최적화 - LDPC 디코더/부호 설계 기여 없음, HW ECC 엔진 미설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RFR/AR-RFR refresh 스케줄링 | 대응 없음 | FTL/컨트롤러 레벨, ECC 시뮬레이터 범위 밖 |
| enhanced BCH (n,k,t) parity 모델 | 대응 없음 | binary QC-LDPC min-sum 디코더와 무관 |
| A_READ/P_READ_RF 커맨드, PMT/EMT/UMT | 대응 없음 | SSD 인터페이스/매핑 테이블, 디코더 무관 |

적용 가치: **낮음** - refresh/FTL/GC 시스템 최적화라 Prime ECC의 부호설계→디코더→HW 어느 레이어에도 닿지 않음. ECC는 교과서 BCH 모델로만 쓰여 이식할 알고리즘 없음.
