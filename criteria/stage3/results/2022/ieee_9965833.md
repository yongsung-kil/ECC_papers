### B. 알고리즘 요약

1. 시스템: 고rate(R=3/4) irregular QC-GLDPC, 길이 ≈2000bit, BPSK/AWGN, sum-product BP 복호(50 iter, 50 frame error 수집).
2. 문제: 고rate 영역에서 저복잡도로 이진 LDPC를 능가하려 함 — GLDPC는 base LDPC의 nonzero를 constituent 부호 열로 치환하는 방식.
3. 구조: base 이진 LDPC `Hb`(row-regular, 행 weight `nc`)의 1을 constituent 부호 `Hc(rc×nc)` 열로 치환 → `R=1-Jav(1-Rc)` (식 1). constituent는 최소거리 2(redundant 3~4개)로 제한해 MAP 복호 복잡도 `~2^rc` 억제.
4. 핵심 1: NB LDPC 대비 — NB는 binary image 상 복잡도 `~2^{2m}`(FHT로 `m2^m`), GLDPC는 constituent trellis MAP `~2^rc`로 더 유리.
5. 핵심 식: 앙상블 평균 스펙트럼 `E{An,w}` (식 5)를 Poltyrev tangential-sphere bound에 대입해 유한길이 ML 복호 오류확률 random coding bound 도출(BP FER의 하계로 해석).
6. 핵심 2(주기여): base 그래프의 cycle이 라벨링 후에도 남으려면 edge 라벨 합이 0이어야 함(식 9) → 라벨을 골라 남는 cycle 수를 최소화.
7. 알고리즘: 길이 g cycle 방정식 행렬 `Eg` 구성 → 무작위 permutation `P`로 각 행에 constituent 열 배정 → 남는 cycle 수 N(기준 C1) 또는 만족 subequation 수(C2)로 평가 → best 갱신(Algorithm 1).
8. 부수: base 매트릭스 예 25×300(λ=125/300 x+150/300 x²+25/300 x⁷, Jav=3.0, lift 7), 31×496(Jav=2.875, lift 4); error floor 회피 위해 고weight column 추가.
9. 결과: 최적화된 GLDPC/NB LDPC 모두 WiFi·5G binary LDPC 대비 낮은 FER 달성, GLDPC는 이진 LLR 교환만 필요해 복호 복잡도 이점.
10. 한계: HW 미설계, 시뮬(BP)뿐, GLDPC라는 별도 부호클래스(constituent MAP 복호 필요) 전제 — 고전 binary min-sum LDPC와 직접 호환 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC base matrix / H-matrix 구성·라벨링 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | GLDPC는 constituent 부호를 얹은 별도 PCM 구조라 특정 binary QC-LDPC 고정인 Prime ECC와 부정합(이식 높음/부담 큼) |
| BP + constituent MAP(trellis) 복호 | 대응 없음 | Prime ECC 디코더는 SPC 기반 binary min-sum이며 constituent block code MAP 복호 경로 없음 |
| ML random coding bound (Poltyrev) | 대응 없음 | 부호설계 평가 해석도구로 Prime ECC 코드베이스에 대응 모듈 없음 |
| cycle 제거용 라벨링 최적화 | 대응 없음 | girth 개선 아이디어이나 constituent 부호 라벨링 전제라 min-sum PCM에 직접 대응 함수 없음 |

적용 가치: **낮음** — constituent 블록부호 기반 GLDPC라는 별도 부호클래스·별도 복호(BP+MAP)를 요구하므로 Prime ECC의 고전 binary QC-LDPC + min-sum + dual-diagonal 파이프라인에 떼어 쓸 요소가 사실상 없다.
