### B. 알고리즘 요약

1. 대상은 5G NR QC-LDPC(BG1/BG2 protograph, lifting size `Z`), layered BP 복호로 각 layer(PCM 부분행렬)를 순차 갱신.
2. 문제: layer 처리순서 `Φ`가 수렴속도/성능을 좌우하나 기존 스케줄은 heuristic(check node degree 기준)이며 rate마다 별도 최적화 필요.
3. 채널 모델은 BI-AWGNC, 성능지표는 iterative decoding threshold `s*(Φ)` (성공 복호되는 최악 SNR).
4. 핵심기법 1단 — layered RCA-DE(Algorithm 1): 기존 flooding용 protograph RCA-DE를 layer 순차 연산에 맞게 개조, C(s)+C(r)=1 관계의 reciprocal energy `R(s)=C^-1(1-C(s))`로 V→C/C→V 메시지를 변환.
5. AP-SNR `s_i`를 incoming 메시지 합으로, V-to-C `p_i→j = s_i - q_j→i`로 계산해 layer별 즉시 갱신 반영(수렴 가속).
6. 핵심기법 2단 — nested scheduling: 단일 mother sequence `Φ_m`을 만들고, 임의 `m' ≤ m`에 대해 `m'`보다 작은 원소만 순서대로 읽어 target sequence 도출(모든 rate를 하나의 시퀀스로 커버).
7. 구현 트릭: mother PCM의 행을 `Φ` 순서로 미리 permute해 메모리에 고정 → rate별 스케줄 재설정 비용 0, 행 permute는 부호 불변.
8. 최적화 절차: 전수탐색((m')! ≈ 5.6×10^57)은 불가 → length-4 최적(4!=24)에서 시작해 원소 하나씩 삽입하는 greedy(BG1 기준 1,095회 RCA-DE 평가)로 `Φ*_46` 도출, 저차수 check node 우선 처리가 유리.
9. 결과: BG1, rate {1/3,1/2,2/3}, K=1200~8400, Imax=10, layered sum-product에서 natural order 대비 1% BLER 기준 최대 0.1442 dB 이득 + 평균 iteration 최대 0.98 감소, 기존 [7][8] 스케줄도 상회.
10. 한계: HW 미설계(시뮬만), 5G NR 통신 부호 전용 rate·length, offset min-sum에서도 유사 이득이나 본문 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered 스케줄(layer 처리순서 Φ) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC 복호 이터레이션의 row/block 처리순서에 직접 대응 가능 |
| 행-permuted PCM 프리컨피그 | `ecc_top.cpp` `Load_PCM()` | mother PCM 행 재배열을 로드 단계에 반영 가능(부호 불변) |
| layered RCA-DE 분석도구 | 대응 없음 | 코드베이스에 DE 분석기 없음, 스케줄 오프라인 산출용(런타임 무관) |
| avg iteration 감소 | `partial_CRC.cpp` `partial_crc_HW_T4()` 조기종료와 시너지 | 수렴 가속→조기종료 시점 앞당김 |

적용 가치: **중간** — 부호·min-sum 코어를 건드리지 않고 layer 처리순서만 재배열해 waterfall·평균 iteration을 동시 개선하는 저비용 기법이라 binary QC-LDPC 디코더에 이식 여지가 있으나, Prime ECC의 자체 H-matrix로 layered RCA-DE를 오프라인 재실행해야 하고 Prime ECC가 layered(row-serial) 스케줄을 쓰는지 확인이 필요하다.
