### B. 알고리즘 요약

1. 코드: `(dv,dc)=(3,6)` regular QC-LDPC, `N=1296`, `R=0.5`, base matrix 6×12, CPM 크기 108×108 (자체 PEG-like 구성).
2. 문제: ISBF가 BF 계열 중 최고 FER이나 수렴이 느림 → 수렴 가속·성능 개선 필요. 채널은 BSC(crossover `α`), hard-info만 사용.
3. 모델: `Si`=i번째 VN에 연결된 unsatisfied CN 수, 에너지 `Ei = ci⊕yi + Si` (클수록 unreliable).
4. 기법1 LISBF: 전체 VN을 `L=12` layer로 분할(각 layer=base matrix 1열=108bit), layer 내 병렬·layer 간 직렬 처리로 정보전파 가속·HW 재사용.
5. flipping 규칙: `Si≥T`면 확률 `p0`로, `Ei≥T`면 확률 `p1(≤p0)`로 flip; threshold `T`는 전체 layer threshold `Tl`의 최대값(이전 iter 기반 → 임계경로 단축).
6. 기법2 forward-backward: `T` 계산을 forward `T`/backward `T`의 max로 분해, layer당 complexity/latency O(1)로 정확 계산.
7. 기법3 two-max(ALISBF): layer threshold 상위 2개만 pair 벡터 `A`로 유지·on-the-fly 갱신, `T=A11`로 근사 계산(저장부담↓, 저정밀).
8. flip 시 갱신: `ci=ci⊕1`, `Ei=dv+1-Ei`, 연결된 CN syndrome `sj=sj⊕1`.
9. 결과: `Imax=50`, `p0=0.75,p1=0.45`. ISBF 대비 FER 최대 1자리 감소, 평균 iteration 약 40% 감소. LISBF≈ALISBF.
10. 한계: HW 미설계(gate/throughput 없음), BSC·hard-decision BF 전용, code 1종(N=1296)만 검증.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BF 복호 이터레이션(LISBF) | decoder.cpp LDPC_Decoder() | Prime ECC는 min-sum이라 BF 알고리즘 자체는 교체가 아닌 참고 수준 |
| column layered 스케줄(layer 병렬/직렬) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | dual-update(parity 교대 스케줄)와 유사 개념 - 부분 중복 |
| two-max/forward-backward threshold 계산 | decoder.cpp CNU_Update_New_Mag() | min-sum의 min1/min2(two-min)와 유사한 최대2값 추적 - 이미 보유 기법과 근접 |
| 에너지/syndrome 기반 flip | 대응 없음 | Prime ECC는 LLR 기반 soft min-sum, hard-BF 에너지 개념 부재 |
```

적용 가치: **낮음** - column-layered 스케줄과 two-max 아이디어는 참고 가치가 있으나 Prime ECC의 min-sum/dual-update/min1-min2와 상당 중복이고, 논문 알고리즘(hard BF, BSC)은 soft min-sum 파이프라인에 직접 이식 불가.
