### B. 알고리즘 요약

1. 시스템: `(N,K)` binary LDPC + CRC-8, BPSK/AWGN. 주 대상은 5G NR `(104,20)` Raptor-like(RL) LDPC, 보조로 CCSDS `(128,64)` irregular.
2. 문제: 짧은 LDPC의 유한길이 성능 한계를 후처리 섭동 복호(MBPD-P)로 개선하되, 섭동 심볼 선택 기준이 휴리스틱(nSMV)이라 RL 코드의 degree-1 VN을 못 잡음.
3. 모델: 수신신호 `r=x+z`, LLR `l=2r/σ²`. IP는 소수 심볼에 반대부호 임펄스 `-sgn(r_π(i))A`를 더하는 섭동.
4. 핵심기법1: MLP `MLPθ(r)`가 입력 `r`을 받아 각 심볼별 "one-bit 섭동 성공확률" `b̂`를 출력, 상위 `T`개 인덱스를 섭동공간 `R`로 선택.
5. 식 의미: `R={π(i)|0≤i≤T-1}`, `b̂_π(i)≥b̂_π(i+1)` — NN 순위로 nSMV 휴리스틱을 대체해 코드구조 적응성 확보.
6. 확장: 라벨 `b`(성공한 one-bit 위치 표시)의 Hamming weight `wH(b)`로 디코딩 다양성 정의.
7. 트릭: class imbalance 대응 weighted MSE loss `L=(w/N)Σ(b_i-b̂_i)²`, `w=wH(b)⁻¹`로 저다양성 데이터 학습 촉진.
8. 학습: Alg.1로 full one-bit IP 탐색 라벨 생성, 고→저 Eb/N0 순차 지도학습(MLP 2N/N 은닉, ReLU, ADAM, 500 epoch).
9. 결과: `(104,20)` RL에서 nSMV+intrinsic 대비 BLER 10⁻³서 0.35dB(T=10)·0.45dB(T=40) 이득, 평균 섭동 라운드 43%(T=40) 감소.
10. 한계: 시뮬만, HW 미설계, non-RL은 약간 열화(soft-syndrome 입력 추가로 보완). one-bit 섭동만 고려.

```
| NN 기반 섭동 심볼 선택 | 대응 없음 | NN/딥러닝 디코더 요소, Prime ECC에 없음 |

| 섭동/임펄스 노이즈 주입 | 대응 없음 | 후처리 섭동은 우리 채널/디코더 구조에 없는 별도 루프 |
```
