### B. 알고리즘 요약

1. 대상은 TB(transport block) 기반 전송으로 하나의 TB가 `L`개 CB(code block)로 구성, 각 CB는 rate-1/2 (3,6)-regular LDPC(길이 1024) 또는 IEEE 802.16e LDPC(1152)로 독립 부호화, AWGN+BPSK.
2. 문제: TB/CBG 단위 HARQ에서 CB 하나만 오류나도 전체 재전송 → 자원 낭비. 기존 GC-LDPC는 global parity 추가로 rate가 k/n→(k−δ)/n로 손실되고 component encoder를 재사용 못함.
3. 모델: global parity `f = v·H1^T`를 코드워드에 담아야 하나, L개 component 코드워드 병치는 `v·H1^T=0`을 일반적으로 만족하지 않음 (BIOS 채널 가정).
4. 핵심: global parity bit `f`를 payload에 free-ride 코딩으로 얹어 전송 — 전송코드워드 `c = v + g`, 여기서 `g = f·Gf`는 free-ride 부호(RM 기반)로 생성, 추가 전력·대역 불요.
5. 핵심 식: free-ride 생성행렬은 `Gf·H^T = G_RM·Π`(Π는 interleaver)를 풀어 구하며, H^T의 행이 전체공간을 span하므로 해가 존재 → payload에 직교적으로 extra bit 삽입.
6. 확장: global parity-check H1은 Gallager 구성(행 무게 wL, `H1=[AΠ0 … AΠ_{L−1}]`)으로 만들어 implicit GC-LDPC를 정의, encoder는 component LDPC encoder를 그대로 병렬 재사용.
7. 복호(2단계): 1단계 free-ride 디코더로 `f̂` 추정 후 `g` 효과 제거, 2단계 normal graph 위에서 local LDPC(H)·global LDPC(H1) SISO(SPA) flooding, 이미 성립한 CB(green node)는 LLR 고정·복호 중단.
8. 부수 트릭: 중·고 SNR에서 오류 CB 수 `Ne`가 작다는 점(예: L=8에서 Pr{Ne>3}≈1e-3~1e-5)을 이용, `Ne ≥ L/2`면 Jmax/2 이후 조기종료.
9. 결과: (3,6)-regular component 대비 WER 1e-4에서 최대 0.7dB, IEEE 802.16e component 대비 FER 1e-3에서 0.5dB coding gain, rate 손실 없음, 평균 iteration은 component와 유사(때로 더 적음).
10. 한계: HW 미설계(SPA 시뮬만), TB 다중 CB 결합·HARQ 전제, AWGN 채널, gatecount/throughput 미제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| component LDPC SPA 복호 | decoder.cpp LDPC_Decoder() | 우리는 Min-Sum이라 SISO SPA 그대로는 아니나 local 복호 위치는 동일 |
| global coupling H1 / GC 부호구조 | ecc_top.cpp Load_PCM() | 단일 PCM 고정 구조라 다중블록 global check 결합 미지원 |
| component encoder 병렬 재사용 | encoder.cpp LDPC_encoder_4KB() | encoder는 dual-diagonal 단일부호 전용, TB 결합 인코딩 아님 |
| free-ride(RM) 추가비트 삽입/복호 | 대응 없음 | Prime ECC에 free-ride/RM 오버레이 개념 없음 |
| 2단계 global 복호 스케줄 | 대응 없음 | 다중 CB normal-graph 복호 스케줄 미보유 |
```

적용 가치: **낮음** — 여러 code block을 하나의 TB로 결합하는 5G/LTE HARQ 시나리오 전제라, 단일 page를 독립 복호하는 NAND binary QC-LDPC ECC 구조와 정합하지 않고 free-ride/global-coupling 계층이 우리 코드에 대응 모듈이 없다.
