### B. 알고리즘 요약

1. 대상: IEEE 1901 PLC 표준 LDPC-CC(convolutional), `Tp=3`, `dv=3`, `dc=b·dv`, `b∈{2,3,4,5}`, `ms=215/226`, systematic `e=1`, layered 스케줄, `Np=10` PE, window `l=228`.
2. 문제: 기존 1901 layered 디코더는 AM(AP-LLR memory) bandwidth 제약으로 layer 하나 처리에 3 cycle이 걸려 throughput이 병목.
3. Storage 개선(1): `Reg-based` — shift register로 AP-LLR 저장, delay term factor로 입출력 포트 위치 결정, cycle당 `3·Np` LLR 확보로 bandwidth 3배.
4. Storage 개선(2): `CMem-based` — 주소를 mod `m`으로 분할해 충돌 없이 병렬 read(Theorem 1: 조건2 만족 시 `m` 뱅크로 분할, `n=m=3`), depth `l/3` CMB 3그룹으로 bandwidth 3배, layer당 1cycle.
5. 식 의미: 1901 delay polynomial의 DT factor 차가 `%3`으로 서로 다른 잉여류를 가져 3-way 뱅킹이 무충돌로 성립.
6. Arithmetic 개선: layered 갱신은 표준 NMS `Q=α·∏sgn·min|P|`(`α=0.75`), `Qcn→vi` zero-init 특성을 이용해 PE0의 1st stage 모듈 전부 제거.
7. C2V 압축: `b=5`에서 NMin1/NMin2·index·부호를 29-bit 압축 저장, PAS(부호 곱)는 계산 가능하므로 미저장 → 저장량 감소, SM2TC/TC2SM/Neg 변환 모듈로 복원.
8. Control 개선: `l=228`이 3으로 나뉘어 PE간 메모리 접근이 동기 → global controller 하나로 통합, 76-ary down counter 공유로 counter 수 22.2% 감소.
9. 결과: SMIC 55nm, `wL=wP=8/wQ=6` 양자화(1/2-rate에서 BER 손실 0.2dB 이내), max throughput `363.6×4` Mbps, [11] 대비 2.45배, area efficiency Reg 25.4%·CMem 84.5% 향상, storage 150.7Kb.
10. 한계: 대상이 IEEE 1901 LDPC 컨볼루셔널 부호 특화, 알고리즘 신규성 없음(표준 NMS), block QC-LDPC와 데이터 흐름 구조가 근본적으로 다름.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN NMS 갱신(min1/min2·부호) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 낮음 - 우리가 이미 보유한 표준 min-sum과 동일, 신규성 없음 |
| C2V 압축 저장(NMin1/NMin2·index·부호, PAS 미저장) | decoder.cpp C2V_Cal() (메시지 저장부) | 낮음 - block 디코더 메시지 버퍼에 압축 아이디어 참고 정도 |
| CMem 주소 mod 분할(bandwidth 3배) | 대응 없음 | 낮음 - LDPC-CC sliding-window FIFO 전용, block layered 뱅킹과 구조 상이 |
| H-matrix(semi-infinite convolutional) | ecc_top.cpp Load_PCM() | 대응 없음 - Prime ECC는 유한 base+shift QC-LDPC |
```

적용 가치: 낮음 — 대상이 convolutional LDPC(semi-infinite H, sliding FIFO)로 Prime ECC의 유한 block QC-LDPC와 구조가 달라 storage/control 최적화가 그대로 옮겨지지 않고, 디코딩 알고리즘은 이미 보유한 표준 NMS라 중복.
