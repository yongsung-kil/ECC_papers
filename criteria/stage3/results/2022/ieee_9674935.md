> 총평: min-sum CN의 first-two-minima 탐색기를 one-hot 출력으로 재설계해 비교기를 없애고 다단 pipeline으로 multi-frame throughput을 올린 순수 HW 최적화 - Prime의 min-sum CNU에 직접 대응하고 이식 난이도 낮으나, 이미 min-sum 보유라 기여가 CNU 회로 최적화로 국한되고 정정성능 이득은 없음.

### B. 알고리즘 요약

1. 대상: IEEE 802.11 QC-LDPC 4개 rate(648~1944 길이 추정), BPSK/AWGN, (6,2) 고정소수점 양자화(정수4+소수2).
2. 문제: min-sum CN 연산의 핵심인 first-min/second-min과 min1 위치 탐색이 LDPC 디코더 복잡도 병목 - 기존 rejection 기법[5]은 min1을 binary index로 출력.
3. 핵심 관찰: CN이 이웃 VN들의 |Z_j,l| 중 min1/min2와 min1 위치만 알면 CTV `L_j,l`은 그 위치엔 min2, 나머진 min1로 결정됨 (min-sum 근사).
4. 핵심 기법: 3x2-MIN·4x2-MIN 기본모듈을 수정해 min1 위치를 `one-hot 시퀀스`로 출력 - flag 비트(a~f)에서 Karnaugh map으로 one-hot 비트 I0~I3 도출.
5. 식 의미: one-hot 비트를 CTV 계산 MUX의 select로 직접 사용 → 기존 binary index를 select로 변환하던 추가 비교기(ρ개)를 제거.
6. 확장: 큰 입력(ρ=5~16)은 기본모듈 log2(ρ)-1 레벨 트리로 구성, 지연 `2(⌈log2 ρ⌉-1)+1` 클록.
7. 구현 디테일: 다단 트리의 각 레벨은 이전 레벨이 idle → 매 클록 새 입력 투입 가능한 pipeline; multiplicity factor로 동시 처리 프레임 수 지정, APP/CTV/VTC flip-flop을 factor만큼 복제.
8. 최적화 절차: 별도 학습 없음 - RTL(ModelSim) 검증 후 Synopsys DC 합성, Cadence Innovus floorplan.
9. 결과: 28nm CMOS, Jmax=5, (6,2)양자화; multiplicity 1→4→8로 throughput 대폭↑·에너지효율 거의 불변·면적 증가율은 throughput보다 완만; critical path 0.3~0.4ns(2.5~3.33GHz), 기존 대비 clock 3배·core area 최소. (구체 수치는 Table4/5 표에만 존재→미상)
10. 한계: 정정능력/BER 개선 없음(양자화 손실 ~0.1dB), power/energy는 프레임 추적·복제 회로로 근소 열위, 무선통신 표준코드 대상(NAND 미검증).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN first-two-minima + one-hot 위치 탐색 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime min-sum CNU(min1/min2+sign XOR)에 직접 대응, one-hot MUX 구동으로 대체 검토 가능 |
| min-sum CTV 계산 전체 / 이터레이션 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()`, `C2V_Cal()` | 알고리즘 동일(min-sum), one-hot는 회로 최적화 레이어 |
| layered(LD) 셔플 스케줄 | 대응 없음 | Prime는 Dual-Update(parity even/odd 교대) 채택, 본 논문 셔플 LD와 별개 |
| multi-frame pipeline / flip-flop 복제 | 대응 없음 | Prime pipeline 6-stage(col_M1/Z/P1~P4)는 단일프레임 - multi-frame 복제 구조 미보유 |
| VTC 2's-complement↔sign-magnitude 변환 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` 등 | 양자화/부호처리 인접, 형식변환은 구현 디테일 |

적용 가치: 중간 - one-hot rejection CNU는 Prime의 min-sum CNU 면적을 줄일 여지가 있어 이식 난이도 낮음. 다만 Prime은 이미 min-sum·pipeline 보유라 기여가 CNU 회로 미세최적화로 한정되고 정정성능·iteration 이득은 없음.
