### B. 알고리즘 요약

1. 시스템: 바이너리 LDPC, AWGN/BPSK, IEEE 802.11n length-1296 rate-compatible 부호(`Rc`=2/3,3/4,5/6), `Imax`=30.
2. 풀려는 문제: LUT 기반 min-sum 디코더는 code rate·node degree·iteration마다 별도 LUT 필요 → HW 메모리 과다.
3. 모델: `(qm,qv)` MIM-QMS 디코더(qm=교환메시지, qv>qm=APP), 이산 AWGN 채널서 pmf 추적(MIM-DE).
4. 핵심 기법(1단): modified MIM-DE — 여러 target 부호의 **joint degree distribution** `ρ̃(ζ),θ̃(ζ)`로 rate 공용 iteration-varying LUT 구성.
5. 핵심 식: CN update `Φc(r)=f⁻¹(∏ sgn(f(rk))·min|f(rk)|)` (min-sum), VN update `Φv(l,s)=φch(l)+Σφv(sk)` — MI 최대화 양자화(DP로 threshold `Γv`).
6. 핵심 기법(2단): LUT 최적화 — 인접 iteration LUT 유사성 활용, discrepancy `d(y(t),y(t'))`와 merge function으로 그룹 평균 LUT 재사용.
7. 구현: reconstruction/calculation/quantization 3함수 구조, N=2^qm(reconstruction)/2^qm−1(threshold) 엔트리, 4-bit(qm=4) 기준.
8. 최적화: greedy search로 discrepancy threshold `Δd*` 결정(exhaustive), MI `Q(k)≥Q*` 만족 최대 그룹화(Algorithm 1).
9. 결과: LUT 메모리 rate-compatible LUT 디코더[11] 대비 최대 94.92%, rate-specific 대비 45.37% 절감; NMS 대비 최대 0.3dB 우수, BP(∞) 대비 0.15dB 이내.
10. 한계: Monte-Carlo 시뮬만, HW(RTL) 미설계(DSP/SDR SW 메모리 분석), target 부호 선택 systematic 방법 미해결(open problem).
