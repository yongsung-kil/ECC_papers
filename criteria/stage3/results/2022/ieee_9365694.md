### B. 알고리즘 요약

1. 시스템: LDPC 기반 고밀도·저가 NAND SSD(TLC/QLC, 3D). SSDsim 8ch×8chip×4plane, block당 64×4KB page. LDPC 부호 rate/length는 미명시.
2. 문제: LDPC는 확률기반 ECC라 신뢰도 낮을 때 read retry(soft sensing) 다회 발생 → decoding latency 급증 → tail latency 악화(QoS 저하).
3. 핵심 관찰: retry 수가 일정 한계 초과 시 평균 read latency 상승, 더 높은 한계 초과 시 극단적 tail latency 발생. retry 수를 데이터 신뢰도 proxy로 사용.
4. 기법1 threshold refresh: 두 retry 임계 `R1`(성능, 기본 3), `R2`(tail, 기본 5)와 접근빈도 `T`(=2). `R>R2`면 즉시 refresh, `R>R1 && F>T`면 성능용 refresh.
5. 기법2 periodic refresh: 블록 첫 wordline만 읽어 신뢰도 추정 → 임계 초과 시 블록 valid page refresh. benefit bit(마지막 refresh 후 접근여부)로 불필요 refresh 회피.
6. 기법3 dual-ECC: weak-ECC(초기, 저비용) / strong-ECC(고오류·저빈도 cold 데이터) 이원화. `R>R2` 데이터를 strong-ECC로 재부호화하여 refresh 횟수(=periodic 비용 γ) 절감. ECC 강도는 refresh 주기(3일 vs 1개월)로 표현.
7. 구현 오버헤드: mapping entry에 access counter 2bit + ECC remark 1bit(총 3bit)만 추가.
8. 학습/최적화: 없음(수학적 benefit 모델 식(1)~(3)로 각 scheme 이득 정식화).
9. 결과: MSR/SSD 10개 워크로드에서 tail latency 대폭 개선, dual-ECC가 Threshold+Periodic 대비 refresh 오버헤드 평균 84.2% 감소, 평균 read 성능 6.2% 개선. write 성능 영향 없음.
10. 한계: HW 미설계, LDPC 디코더/부호구조 자체 미개입, write-intensive 시나리오는 future work.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC retry count → 신뢰도 신호 | decoder.cpp LDPC_Decoder() (반복/실패 카운트) | 낮음 - 우리 디코더는 retry 수 산출 가능하나 refresh는 FTL 상위 계층으로 코드베이스 밖 |
| soft sensing level / read latency 모델 | channel.cpp Set_R_Offset(), Set_LLR_Th() (soft-read threshold) | 낮음 - read 비용 캘리브레이션 참고용, 알고리즘 이식 아님 |
| dual-ECC(weak/strong) 선택 | 대응 없음 | 무관 - FTL 정책, 우리 단일 QC-LDPC 엔진 밖 |
| refresh / FTL mapping / GC | 대응 없음 | 무관 - 컨트롤러 firmware 레이어 |
```

적용 가치: **낮음** - NAND 직접 타깃이지만 개선 레이어가 FTL/refresh/스케줄링이라 Prime ECC(부호→디코더→HW) 경계 밖이다. retry-latency 수치표만 read-cost 모델 참고 소재.
