### B. 알고리즘 요약

1. 시스템: binary QC-LDPC(BPSK+AWGN), base matrix 7×72, circulant Z=256(즉 M=1792, N=18432), VN degree 4, CN degree 41/42, rate 0.9028(플래시 후보).
2. 문제: 기존 layered FAID들[9~11]은 교환 메시지 pmf를 정확히 추적 못 하거나 채널·메시지 정밀도 제약이 있어 성능 손실.
3. 모델: 이산 DMC로 채널 양자화, 코드비트 X 조건부 pmf를 다항식 `PA|x(θ)`로 표현해 추적.
4. 핵심기법: layered min-sum에 VN 3단(reconstruction `φch,φv` → calculation 가감산 → quantization `Γv`) 추가한 MIM-QLMS.
5. 핵심 아이디어: protograph 기반 MIM-DE(PMIM-DE)로 각 iteration·각 edge별 LUT를 pmf 정확 추적하며 설계(`(5)` `PR=quan(PB,Γv)` 등).
6. CN: min 연산 `(1)` `Sj=f^-1(∏sgn·min|f(Rk)|)` 유지(min-sum), C2V/V2C를 LLR 비례 값으로 정렬.
7. 채널 LUT: `φch(l)=round(η·log(PL|X(l|0)/PL|X(l|1)))` `(2)`, η는 iteration 불변·qv 오버플로 방지로 최적화.
8. 최적화: edge-specific(Tmax·NB개 LUT)를 2단계 반복으로 통합해 code-specific 4개 LUT(φch,φv,Γch,Γv)로 축소, 메모리 최대 Tmax·NB배 절감.
9. 결과: (3,3,7) code-specific이 (3,3,7) RCQ 대비 FER 최대 0.06dB↑·평균 iteration 최대 10%↓, (4,4,8)은 (4,4,7)LNMS보다 0.05dB↑, floating LBP에 0.04dB 근접.
10. 한계: HW 미구현(시뮬만), 이득 폭 작고 LUT 설계에 오프라인 PMIM-DE·DP 비용, 단일 코드(rate 0.9028)만 평가.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered min-sum CN 연산 (min, sign) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유 min-sun과 동일 계열 — LUT만 추가 |
| VN reconstruction/quantization LUT(φv,Γv), iteration별 LLR | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx(), VN_Cal_HD() 등 | code-specific 4-LUT를 LLR 테이블 자리에 매핑 |
| 채널 LUT φch, DMC 채널 양자화 | channel.cpp Set_LLR_Th(), Set_R_Offset() | soft-read 3/4bit 양자화 테이블 대응 |
| QC-LDPC/protograph 구조(base+circulant) | ecc_top.cpp Load_PCM() | Prime ECC의 base+shift QC 구조와 동형 |
```

적용 가치: **높음** — Prime ECC가 이미 min-sum과 iteration별 LLR 테이블을 가지므로, code-specific 4-LUT(φch,φv,Γch,Γv)를 LLR 테이블 위치에 이식하면 소량 메모리로 waterfall·iteration 개선을 노릴 수 있고 대상 코드가 고rate QC-LDPC(플래시)라 정합성이 높다.
