### B. 알고리즘 요약

1. 시스템: 5G NR QC-LDPC 디코더, base graph BG1(rate 1/3~8/9, info 292~8448b)/BG2(rate 1/5~2/3), lifting size `Z=a*2^j` 51종(2~384), raptor-like puncturing.
2. 문제: 5G는 lifting size 51종·긴 codeword로 기존 full-parallel HW는 자원 과다, 51종 지원 실패 → 실용적 면적효율 디코더 필요.
3. 알고리즘: row-layered **Offset Min-Sum(OMS)**, `Rcv=S*max(min|Qvc|-β,0)`, layered로 수렴 약 2배 빠름(flooding 대비).
4. 핵심기법 - 부분병렬: full parallelism 대신 병렬인자 `P=32`(power-of-two)로 DFU 수 최소화하며 20Gbps peak 충족, 처리량식 `T=Z*Nb*R/(Iavg*Mb*CLKlay*TCLK)`.
5. 신규 메모리 접근 스케줄링: layer를 `K=max{k|k divides Z, k≤P}`개 subset으로 분할해 stride scheduling의 "Z가 P 배수여야 함" 제약 제거 → data access 문제 해소.
6. data alignment: 유효한 K개 값만 cyclic-shift하는 **QSN(QC-LDPC Shift Network)**(left/right shift + merge)로 정렬 문제 해결.
7. 구현 디테일: 8-bit 고정소수점(offset 0.0011 binary), 27 APP memory bank + 1 shared bank, mask-flag로 27중 19 path 선택(대형 MUX 제거), MIN TREE(C1M1/C1M2)로 1st/2nd min.
8. 부수 트릭: syndrome checker 조기종료, 첫 iteration 첫 layer skip(punctured part로 min=0)으로 고rate throughput↑, transpose memory(12x32)로 codeword 재배열.
9. 결과: 65nm CMOS, 500MHz, core area 5.74mm^2, **21.78Gbps**, [14] 대비 45% 작은 면적, 8-bit로 double-precision과 성능 거의 동일(BG1 max iter 10).
10. 한계: 5G 다중 lifting size 지원이 핵심 가치인데 NAND는 단일 고정부호라 스케줄링 니즈 자체가 낮음, error-floor/정정능력 개선 없음(순수 HW/스케줄).
