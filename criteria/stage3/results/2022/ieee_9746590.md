> 총평: NAND retention 실측으로 2-bit soft-read offset을 MMI 클러스터링 최적화하는 채널/LLR 계층 기법. 디코더·부호 미변경이나 Prime ECC soft-read threshold 설계에 직접 참고가치.

### B. 알고리즘 요약

1. 시스템은 3D TLC NAND(8상태 ER,A~G) SSD, 기성 2K LDPC 디코더에 2-bit soft-read 입력. 블록당 `L×F×3×B`=2688bit, 168 page.
2. 문제: SoC의 soft-read register 제약으로 offset pair가 4개(K=4)로 한정, 벤더/SoC가 주는 AWGN 기반 기본 offset은 열화(retention) 채널에 부적합.
3. 모델: data retention(F-N tunneling 전자누설)로 Vth 분포 좌시프트, 고상태(E,F,G) 시프트 심함. 3KPE·85°C·83~372h(=55°C 3개월~1.75년) 실측.
4. 핵심기법: MMI(maximum mutual information)로 각 layer·7 read 경계의 최적 soft-read offset(hard-read 기준 좌/우 offset) 산출 → 채널용량 최대화 양자화 경계.
5. 핵심식: 채널용량이 code rate보다 크면 신뢰전송 가능 → MMI가 code rate 초과(hard-read MI>0.94) 지점은 이득 미미.
6. 데이터 전처리: 우측 tilt(비정상, hard-read MI>0.94) 점 제거 - 1-bit 디코더로 성공 복호되어 soft-read 이득 없음을 확인 후 배제.
7. 핵심기법 2단: MMI 기반 클러스터링(Alg.1) - K-means(Euclidean, K=4)로 초기화 후, 이웃 확장집합에서 총 MI 최대 대표점 갱신, MI 최저 10점을 재배정(over-fitting 방지).
8. 학습/최적화: 비지도 K-means + MI 목적 재할당 반복(MaxIter).
9. 결과: 3KPE DR228h@85°C에서 기본 대비 평균 +73 error-bit tolerance, Lower/Middle/Upper page FER=1e-1에서 약 80/80/60 EBC 개선, 16 die 전반 codeword pass 향상.
10. 한계: HW 미설계, 디코더·부호는 기성 2K LDPC, 특정 벤더 3D TLC·retention 조건 종속(실측 기반).
