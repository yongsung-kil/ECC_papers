> 총평: 5G NR/WiMAX/WLAN용 QC-LDPC layered 디코더의 forward/backward 병렬화 시 발생하는 read-after-write 메모리 충돌을 matrix reordering과 이중포트 대신 병렬 메모리서브시스템으로 해소한 HW 최적화로, scaled-min-sum·layered 등 우리와 공유 요소가 많아 메모리 충돌회피 아이디어는 참고 가치가 있으나 정정성능 개선은 없다.

### B. 알고리즘 요약

1. 대상은 5G NR/WiMAX/WLAN QC-LDPC 디코더(base matrix+lifting factor Z), NR code length ~100~25344, rate 1/5~5/6, BG1(max row weight 19, 46 rows)/BG2(10, 42 rows). SIMD ASIP 구조.
2. 문제: 기존 설계는 forward/backward 동시 연산 위해 single-port 주파수배증(frequency-doubling) SRAM 사용 → 메모리 클록이 시스템 클록의 2배를 요구해 클록 상한을 지수적으로 제약.
3. 스케줄: layered decoding — H를 `Z×NZ` 크기 R개 non-overlap layer로 분할, 각 layer(base matrix 한 행) 갱신 후 즉시 VN 갱신(수렴 가속).
4. 노드 갱신: scaled-min-sum, `f(x,y)=λ·sign(x)·sign(y)·min(|x|,|y|)`, 승산보정계수 `λ`(scale factor `δ=0.75`). forward `α=f(α,γ)`, backward `β=f(γ,β)` 재귀.
5. 핵심 식: 사후정보 `ρ=γ+Λ`, `Λ=δ·f(α,β)`. 5bit `γ` overflow 방지 위해 saturation 후 보상 `Λ=sat(γ+Λ)-γ`로 재계산(음수 시 우측 2bit shift ≠ /4 → /4 사용).
6. HW 기법 1(충돌 최적화): forward(현재 layer)+backward(이전 layer) 병렬 시 write-back이 입력 역순이라 심각한 RAW 충돌 → matrix reordering 알고리즘으로 인접 layer 동일 컬럼 non-zero 충돌 사전 회피(HW 수정 없이 base matrix search table만 교체).
7. HW 기법 2(메모리서브시스템): forward/backward 병렬 시 연산유닛과 메모리유닛의 idle 상보성 활용, dual-port 대신 Z개 RAM bank로 분산 저장(n번째 세그먼트를 addr n-1로 동시 fetch) → 주파수배증 SRAM 제거.
8. 부수: 임의 lifting factor 회전에 double barrel shifter, 입력버퍼 ping-pong 2-set SRAM으로 연속 스트림 디코딩.
9. 결과: SMIC 28nm, 400MHz에서 max throughput 1984Mbps(48 SISO 병렬, 이전 설계 대비 3.8배), 회로 최적화로 시스템 클록 상한 344MHz→1.1GHz(최대 5456Mbps 가능), gate count 1716K, 5bit 양자화 시 error floor ~1e-6, 4.5dB에서 수렴.
10. 한계: 정정성능(BER) 개선 없이 throughput만 향상, ASIP/SDR 지향 명령어셋 구조로 고정 RTL과 이질적, error floor 5bit 양자화 한계 존재.
