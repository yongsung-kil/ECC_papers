### B. 알고리즘 요약

1. 시스템: 블록체인 light node가 **data availability(DA) 공격**을 탐지해야 하는 Coded Merkle Tree(CMT). 각 layer를 rate `R=0.5` systematic LDPC로 인코딩, short length(`n=128~200`) 대상.
2. 문제: peeling decoder는 **stopping set**이 숨겨지면 복호 실패 → 악의 노드가 최소 stopping set을 숨겨 DA공격 성공. short length에서 random LDPC ensemble은 최소 stopping set 보장 불가.
3. 모델: adversary를 weak/medium/strong 3종으로 정의(stopping set 탐색 능력별). 목표는 sample size 대비 light node 탐지 실패확률 `Pf` 최소화.
4. 핵심 기법 1단 (weak): **EC-PEG** — PEG의 CN 선택 시 cycle 분포 `ζg`의 **엔트로피 `H(α_gc)`를 최소화**하여 stopping set을 소수 VN에 집중시킴(dv-regular, `dv≥3`).
5. 핵심 식 의미: 균일분포는 엔트로피 높고 집중분포는 낮음 → 엔트로피 최소화 = stopping set 집중 → greedy sampling으로 탐지확률↑.
6. 핵심 기법 2단 (medium/strong): **LC-PEG**(LP 제약 PEG) + **LP-sampling** co-design — 작은 stopping set 지식으로 LP를 풀어 최악 stopping set의 탐지실패를 최소화.
7. 구현 디테일: CMT 여러 layer 간 parity-check 행렬 열을 정렬(Algorithm 2)해 base layer greedy sampling이 Merkle proof 통해 상위 layer에도 전파되도록 함.
8. 최적화: greedy set 탐색은 cycle(길이 `gmax`) 열거로 `O(n^(gmax/2))`, stopping set 탐색은 ILP로 short length에서만 tractable.
9. 결과: EC-PEG/LC-PEG가 원 PEG·MC-PEG 대비 세 adversary 모두에서 탐지 실패확률 `Pf` 감소(시뮬).
10. 한계: DA공격 탐지용 erasure/peeling 부호설계, soft-decision·HW 없음, short length 전용(long length 확장 미보장).
