### B. 알고리즘 요약

1. 시스템: 이진 선형 블록부호 `C[n,k]`, BPSK+AWGN, LLR `r_j = 2y_j/σ²`; 실험은 rate-1/2 (3,6) LDPC `C[128,64]`, `C[64,32]` 및 `C[128,64]` eBCH.
2. 문제: 기존 OSD는 order-`t`에서 후보 리스트 크기가 `Σ C(k,i)`로 폭증(최악 `O(k^t)`), URLLC용 저지연 near-ML 복호에 부적합.
3. 핵심 아이디어: MRB로 `k`개가 아닌 `k+δ`개 최신뢰 위치(확장 MRB)를 선택. `δ>0`이면 이들이 선형종속이라 국소 제약이 자연히 부여됨.
4. 패리티 검사행렬을 가우스 소거로 `H=[[I,P1],[0,P2]]` 형태로 변형; 후보는 `P2·cR^T=0`을 만족해야 하고 `cL=cR·P1^T`.
5. 제약 `P2`가 최대 `2^δ` 상태 trellis를 정의 → 이 위에서 SLVA(serial list Viterbi)로 후보 코드워드 리스트를 우도 내림차순 생성.
6. 조기종료: `γ(cR^(j))+γ(zL) < α(γ(copt)+γ(z))`이면 중단(`γ(cR^(j))+γ(zL)`가 미탐색 후보 우도의 상한). `α`는 오프라인 학습 임계값.
7. δ=0이면 conventional OSD(=FPT-OSD, flipping pattern tree 탐색)로 환원됨.
8. 복잡도: 정렬 `O(n log n)` + 가우스소거 + 재부호화 `l_ave·O((k+δ)(n-k-δ))` + SLVA `O(2^δ(k+δ)+l_ave(k+δ))`.
9. 결과: `C[128,64]`에서 conventional OSD 대비 평균 테스트 패턴 약 530배(δ=8,α=0.6), `C[64,32]`에서 약 9200배 감소하며 FER 성능 손실 무시할 수준; PB-OSD보다도 테스트 패턴 적음.
10. 한계: HW 미설계·시뮬만, 짧은 부호(n≤128) 전용, FER(near-ML) 자체 개선이 아니라 복잡도 절감이 기여, soft 실수 LLR 기반.

| 조기종료(우도 상한 기준) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | Prime ECC는 CRC 기반 조기종료라 기준 자체가 다름 |
