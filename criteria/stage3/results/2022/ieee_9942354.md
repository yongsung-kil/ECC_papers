### B. 알고리즘 요약

1. 대상은 short LDPC(rate 1/2): Code-1 = length 64 regular(VN deg3/CN deg6, PEG, girth6), Code-2 = length 128 CCSDS 코드, BI-AWGN 채널.
2. short code에서는 absorbing set(만족 CN이 불만족 CN보다 많은 오류 VN 집합) 크기가 정정오류수와 비슷해 waterfall 성능을 크게 떨어뜨림.
3. 핵심1: absorbing set을 backtracking+DFS 그래프탐색으로 전수 열거하고 extended type `ν-(ω,ε,Pc)`로 세분류(brute-force `C(128,8)≈2^40` 대비 38분).
4. 각 오류 클래스별로 truncated Gaussian으로 해당 absorbing set에만 오류를 심은 training set을 만들어 특화 BP-RNN 디코더를 학습(diversity).
5. BP-RNN은 tanh 기반 BP의 메시지에 학습가중치 `w`를 곱한 변형(식(1)-(4)), data-pass 가중치를 `w_{n→m}` 하나로 단순화해 기존 BP HW 재사용 가능.
6. 손실은 all-zero 가정하의 average binary cross-entropy(식(5)), multi-loss는 cycle-reduced/CN-regular 코드에서 이득 없어 미사용.
7. 핵심2: 상호보완성 기준(식(9))으로 `J`개 특화 디코더 중 `Z=10`개만 선택, parallel/serial 아키텍처로 실행(parallel은 worst-case latency 감소, serial은 1개 HW 재사용).
8. 핵심3: 모든 BP-RNN이 codeword를 못 내면 각 디코더 soft-output에 저차 OSD(w≤1) post-processing 적용 후 ML rule로 선택 - cross-entropy가 신뢰 VN 오류확률을 낮춰 OSD와 시너지.
9. 결과(FER=10^-4): D10이 BP(Itest=25) 대비 약 0.4dB, D10-OSD-1이 BP-OSD-1 대비 0.43dB(Code-1)/0.72dB(Code-2) 이득, D10-OSD-2는 ML 대비 0.2dB 이내.
10. 한계: HW 미설계(complexity는 CN-update 카운트로만 논의), 시뮬(FER)만, rate 1/2 short code 전용, gate/throughput 수치 없음.

| VN/후험 LLR 처리 | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` 등 | soft LLR 처리 위치 대응, 학습가중치·cross-entropy 손실은 미보유 |
| absorbing set 열거/분류, diversity 학습 | 대응 없음 | Prime ECC는 단일 고정 min-sum 디코더, NN diversity 미보유 |
| OSD post-processing(가우스소거/ML rule) | 대응 없음 | Prime ECC 조기종료는 CRC(`partial_crc_HW_T4()`)이며 OSD 계열 후처리 없음 |

적용 가치: **낮음** - tanh BP + 학습가중치 + OSD 후처리 + 다중 디코더 diversity가 Prime ECC의 단일 min-sum/CRC 조기종료 고rate QC-LDPC 구조와 정합되지 않는다.
