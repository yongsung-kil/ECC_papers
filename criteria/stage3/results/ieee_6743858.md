> 총평: LDPC의 Gallager update rule을 Polar SC 복호에 차용하되 piecewise-linear로 근사한 복잡도 절감 논문 — 대상 부호가 Polar이고 성능 열화만 있어 NAND binary QC-LDPC 이식 가치 없음.

### B. 알고리즘 요약

1. 시스템/코드: Polar code `(256,128)`, rate 0.5, AWGN+BPSK, 정보집합 `A`는 Tal-Vardy법(Eb/N0=2dB)으로 선택.
2. 문제: Arikan SC 복호는 노드 갱신에 초월함수(hyperbolic 등)를 써 HW 구현 복잡 → core operation 복잡도 절감 필요.
3. 채널/모델: SC를 LLR 도메인 message-passing으로 보고, `f(La,Lb)=2 tanh⁻¹(tanh(La/2)tanh(Lb/2))`가 LDPC SPA와 동형임에 착안.
4. 핵심기법 1: LDPC BP의 Gallager form을 SC f-노드에 도입 — `f_node = sign(La)sign(Lb)·f(f(La)+f(Lb))`, 여기서 `f(x)=log((e^x+1)/(e^x−1))`은 involution(`f(f(x))=x`).
5. 식 의미: Gallager form은 tanh 대신 involution transform + 덧셈으로 SPA를 재구성하나 log/exp/division이 남아 HW 부담.
6. 핵심기법 2: involution `f(x)`를 piecewise-linear(7구간, 예 `−12.875x+4.9` 등)로 근사해 곱셈1·덧셈1만 사용 → SC(Piece-wise approx.).
7. 구현 디테일: Table I에서 core operation 함수 사용횟수 비교 — Arikan(hyperbolic×3), Gallager(log1/exp2/div1), Piecewise(mult1/add1).
8. 최적화: 근사함수는 원함수와 오차 최소가 되도록 구간별 A+C*x 계수 설계(5구간 대비 7구간이 더 정밀).
9. 결과: SC(Gallager)는 Arikan과 거의 동일 성능, SC(Piecewise)는 BER 10⁻⁵에서 약 0.5~0.7dB 열화(FER 10⁻³ 기준 0.7dB), BER 10⁻⁶까지 error floor 없음.
10. 한계: 시뮬만·HW 미설계, 성능-복잡도 tradeoff로 BER 열화, 대상이 Polar SC 복호라 LDPC 부호와 직접 무관.
