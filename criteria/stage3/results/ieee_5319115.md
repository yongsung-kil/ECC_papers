### B. 알고리즘 요약

1. (3,6) regular LDPC 코드를 대상으로 BPSK 변조, AWGN 채널에서 전통적 log-BP(로그영역 BP, LLR 기반) 디코딩을 기준선으로 사용.
2. 문제의식: 표준 BP는 곱셈이 많아 HW 구현에 부적합해 log-BP로 단순화했지만, flooding 방식(모든 check node를 먼저 갱신 후 variable node 전부 갱신)은 반복 수렴이 느려 iteration 수와 HW 자원 소모가 크다.
3. 채널/노이즈 모델은 표준 AWGN, 별도의 특수 간섭 모델 없음.
4. 핵심 기법: layered-iterative 갱신 — check node 메시지 `L(hmn)`의 한 행이 갱신되면 그 결과를 즉시 대응 variable node 메시지 `L(gnm)`의 열 갱신에 반영(식 9~10)하고, 그 갱신된 열이 다음 행의 check node 갱신에 바로 쓰이는 방식(식 11~12)으로 순차 갱신한다.
5. 핵심 식은 표준 log-BP의 check-node 갱신(`L(hmn) = Σ...`식 (5)/(9)/(12), tanh 근사 `f(x)=ln((exp(x)+1)/(exp(x)-1))` 기반)과 variable-node 갱신(식 (6)/(10)/(11))을 그대로 쓰되, 갱신 순서만 flooding에서 layered(행별 즉시 반영)로 변경한 것.
6. 별도 2단 확장 기법 없음(단일 스케줄링 변경).
7. 구현 디테일: 로그영역 연산(곱셈→덧셈 변환)은 기존 log-BP와 동일하게 유지, HW 자원 절감은 iteration 감소를 통해서만 간접적으로 주장(직접 gatecount 측정 없음).
8. 별도 학습/최적화 절차 없음.
9. 결과: improved layered log-BP 6회 iteration이 traditional log-BP 10회 iteration과 유사한 BER 성능을 보였고(BER 1e-2에서 SNR 약 0.2dB 이득), 동일 10회 iteration 비교 시 BER 1e-2에서 SNR 약 0.25dB 개선.
10. 한계: HW 미설계, (3,6) regular 코드 1종만 시뮬레이션되어 QC-LDPC·고rate 코드로의 일반화는 검증되지 않았고, 제안 방식이 이미 표준적으로 알려진 layered(shuffled) BP decoding과 본질적으로 동일해 신규성이 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered-iterative 갱신(행 갱신 직후 즉시 열 반영) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 이미 iterative message passing(C2V→V2C) 및 Dual-Update 스케줄링을 보유, 개념 중복 |
| check-node 갱신 (log-BP, tanh 기반) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min-sum(§3)으로 이미 근사 사용, sum-product 기반 log-BP는 HW 관점에서 더 무거워 대응 없음에 가까움 |
| iteration 수 감소를 통한 HW 자원 절감 주장 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` 이터레이션 루프 | 정량적 gatecount 근거 없이 정성적 주장뿐이라 실질 이식 가치 낮음 |

> 적용 가치: 낮음 — layered scheduling 자체는 Prime ECC가 이미 보유(§3 iterative message passing, Dual-Update)하고 있어 중복 기여이며, sum-product 계열 log-BP는 Prime ECC의 min-sum 구조와도 맞지 않음.
