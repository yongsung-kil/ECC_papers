### B. 알고리즘 요약

1. 시스템/코드: BSC(crossover `p0`), (3,6)-regular LDPC, 유한정밀 MS 디코더 — 메시지 `q=4`bit(`Q=7`), APP `q̃=5`bit(`Q̃=15`); finite-length `N=1008`, `N=10000`(MacKay).
2. 문제: 나노공정 소자의 신뢰성 저하로 디코더 로직(adder/comparator/XOR)이 불안정 → 불신뢰 소자 위 MS 거동을 정량 분석 필요.
3. 에러 모델: noiseless 연산 출력 `v`에 error set `E`에서 뽑은 `e`를 bitwise XOR 주입. adder는 full-depth(부호비트까지)와 sign-preserving 두 모델, 주입확률 `pa`; comparator 뒤집힘 `pc`, XOR 뒤집힘 `px`.
4. 핵심 알고리즘: Noisy-MS(Algorithm 1) — CN은 부호 `xpr`(noisy XOR)와 크기 `mpr`(noisy min), VN은 nested noisy adder `apr`로 `q̃`bit 합 후 `q`bit 포화.
5. 핵심 관찰: `dv=3`이라 무잡음 MS의 APP 분포 `C(l)`이 짝수값에만 지지되며 이 "gap"이 DE fixed-point attractor를 유발.
6. Noise-aided 효과: adder 노이즈가 gap을 채워 attractor 탈출을 가능케 함 → noiseless 대비 정정능력 증가(comparator/XOR 노이즈는 gap을 못 채워 효과 없음).
7. 새 임계 개념: noisy 디코더는 `Pe(∞)`가 0으로 수렴 못하므로 useful region, η-threshold, discontinuity line = functional threshold `p0(DL)`(무잡음 threshold의 일반화) 정의.
8. Scale factor 최적화: 유한정밀에선 채널 scale `μ`가 성능 좌우, `μ∈{1,3,5}`에서 noisy가 noiseless보다 우수, 최적 `μ=6`; Proposition 1: `Pe(∞) ≥ pa/(2Q̃)`.
9. 결과: `μ=6`에서 noisy MS BER가 BP에 근접, `pa=10⁻⁴`에서 `Pe(∞)≈3.33e-6`; 조기종료(syndrome check) 유무로 error-floor(`~3.33e-5`) 발생/소멸.
10. 한계: BSC(hard) 전용, HW 미합성(에러 확률 파라미터만), 이득이 (3,6)-regular의 message-alphabet 미사용 특성에 의존, latency는 수백~수천 iteration으로 악화.
