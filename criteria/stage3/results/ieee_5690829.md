> 총평: majority-logic 계열에 감쇠계수(0.5=시프트)로 정련한 신뢰도 척도를 반복 갱신하는 hard/hybrid 디코더로, error floor 제거와 빠른 수렴(5 iter≈50 iter)이 매력적이나 min-sum 기반이 아닌 reliability-based majority-logic이라 Prime ECC 파이프라인과 알고리즘 성격이 달라 이식은 중간.

### B. 알고리즘 요약

1. 시스템: `(γ,ρ)`-regular binary LDPC, BI-AWGN + BPSK, RC-constraint 만족. 검증 부호는 (255,175) EG-LDPC와 Mackay (408,204).
2. 문제: hard-decision(BF)은 채널정보 버려 성능손실, soft(SPA)는 복잡하고 고 SNR에서 error floor 발생 → 성능/복잡도 절충하는 reliability-based hybrid 필요.
3. 기반: OSMLG(one-step majority-logic) 복호, 비트 `z_j`에 직교(orthogonal) check-sum `S_j`를 두고 다수결 결정함수 `Λ_j=Σ(2s_i-1)`로 flip 여부 결정.
4. 신뢰도 척도: check-sum `s_i`의 신뢰도를 `φ_i,j = min{|y_l| : l∈N_i\j}` (자기 비트 제외 최소 크기)로 정의, 식(5)의 min 근사 활용.
5. 핵심 아이디어: 식(5)의 근사가 실제값 `log((1-r_i)/r_i)`보다 크므로 감쇠계수 `catten (0<catten<1)`을 곱해 보정 → 더 정확한 가중 외부정보.
6. 갱신식: `ψ_j = Σ(2σ_i,j -1)·catten·φ_i,j`, 여기서 `σ_i,j = s_i + z_j`. `catten=0.5`면 시프트 레지스터로 구현 가능(추가복잡도 최소).
7. 반복 절차: `R_j^(0)=y_j`(채널 크기로 초기화) → 신드롬 계산 → `ψ_j` 계산 → `R_j^(k+1)=R_j^(k)+ψ_j^(k)` 갱신 → 부호 `R_j` 부호로 hard-decision 재형성, `s=0`이거나 `Imax` 도달 시 종료.
8. 복잡도(Table I): 반복당 이진연산 `(ρ-1)m` XOR, `γn` XOR로 σ, `(γ-1)n` 실수곱셈으로 ψ, `n` 실수덧셈·부호검사; SPA보다 훨씬 가볍고 WIML/IML/IMWBF와 비교.
9. 결과: (255,175)에서 SPA/IMWBF/IML/WIML 능가, WIML은 BER 10^-5에서 error floor인데 AIML은 10^-8까지 floor 없음; 5회 반복이 50회와 거의 동일 성능; (408,204)에서 SPA는 10^-7 부근 floor, AIML은 10^-8까지 floor 없음.
10. 한계: 시뮬레이션만(HW 미설계), 소규모 EG/Mackay 부호에서만 검증, 실수 곱셈 포함(정수 min-sum 파이프라인과 성격 상이), throughput/gatecount 수치 없음.
