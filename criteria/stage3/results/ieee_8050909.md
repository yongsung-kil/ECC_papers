### B. 알고리즘 요약 (10줄 내외)

1. 대상: fully-parallel LDPC 디코더의 라우팅 복잡도를 줄이는 **Split-Row** 아키텍처 (check node를 `Ns`개 split으로 분할).
2. 문제: split마다 local minimum만 알기에 global min을 놓쳐 BER 열화가 발생하며, split 비대칭이 성능·HW를 악화시킨다.
3. 모델: 정규화 min-sum, split별 CN 갱신식 `α_ij:Spn = S_SR × sign(β) × min|β|` (식 2), `S_SR`은 split용 정규화 인자.
4. 핵심 지표: 행 `i`의 split `j` 가중 불균형 `δ(i,j) = W(i,j) − Wr/Ns`, split 전체 불균형 `Δ(j) = Σ_i |δ(i,j)|` (식 3,4).
5. 대칭 조건 `∀j, Δ(j)=0` - split들이 동일 가중(=동일 interconnect 수)을 가질 때 최적.
6. 핵심 기법: **heuristic smart column-permutation** (Table I 4단계: 불균형행렬 H̄ 구성 → column별 Swap Index → 최대 컬럼 탐색 → swap)으로 `Δ(j)` 최소화. HW 오버헤드 없음, girth 불변.
7. QC-LDPC 확장: circulant을 깨지 않도록 `N/Z` 단위 **column-bundle permutation**으로 일반화 (Fig.5). EG-LDPC는 본래 대칭이라 불필요.
8. 오류성능 분석: 잘못된 `S_SR` 곱의 기대값 `E[α_e]`이 `δ(i,1)=0`에서 최소화됨을 증명 (식 6,7) → 대칭이 BER 개선.
9. 결과: IEEE 802.11n(648,324)에서 BER=1e-3에 0.5dB 개선; 802.3an(2048,1723) 합성(lsi_10k)에서 levels-of-logic 11.7%↓, critical path 10.2%↓, compile time 2.92×↑.
10. 한계: fully-parallel Split-Row 전용 기법, 통신표준(Ethernet/Wi-Fi) 코드 대상, gate count/throughput 미제시, NAND soft-read 무관.
