### B. 알고리즘 요약

1. IEEE 802.11n 예제 기준 blocklength `n=1944`, sub-matrix 크기 `p=27` 또는 `p=54`, mother code(rate 0)에서 rate `1/2, 2/3, 3/4, 5/6`의 effective code를 생성하는 구조화 LDPC 다중 rate 부호.
2. 기존 puncturing/shortening 기반 다중 rate 지원은 blocklength가 실질적으로 줄어들어 특히 최고/최저 rate에서 성능 저하가 크다는 문제를 해결하려 함.
3. mother code(최저 rate, parity-check matrix `H`)의 row(또는 sub-matrix row)들을 결합(합산)하면 check node 개수가 줄어 rate가 높아지되 blocklength(column 수)는 불변이라는 것이 핵심 아이디어.
4. Strict Row-Combining(SRC): 결합될 check node들이 공통 variable-node 이웃을 갖지 않도록 mother code를 설계해 edge 추가/삭제 없이 row만 합산; Row Combining with Edge Variation(RCEV): 결합 시 edge를 소수 추가/삭제해 rate별로 variable-node degree distribution을 다르게 최적화.
5. 구조화(structured) 버전은 sub-matrix(cyclic shift identity `Si`) 블록 구조를 유지해 병렬 디코더 구현이 가능하고, `H2` 부분을 하위삼각(lower/block-lower-triangular) 구조로 유지해 back-substitution 기반 저복잡도 인코딩을 지원.
6. graph-conditioning: ACE 알고리즘(`dACE`, `ηACE`)과 stopping-set 회피 메트릭(`βc, βp` 기반 `dSS, γc, γp`)으로 각 rate 별 column을 순차 생성해 small cycle/stopping set을 최소화, error floor를 낮춤.
7. SRC의 한계는 모든 rate가 동일한 variable-node degree distribution을 공유해 저rate 코드의 degree-2 노드 수가 이론 최적치보다 부족한 점; RCEV는 rate별로 edge를 조정해 이를 완화.
8. 설계 절차는 Algorithm 1(SRC)/Algorithm 2(RCEV): rate·blocklength·sub-matrix 크기 선택 → degree distribution 선택 → row 결합 방식 선택 → column별 랜덤 생성+graph-constraint 검증 반복(설계 최적화, DE/GA는 아님).
9. 결과: blocklength 1944, 15 iteration 기준 SRC가 802.11n 표준 stand-alone 코드 대비 근접 성능(0.1dB 손실); 50 iteration에서는 RCEV가 SRC보다 저rate에서 우수하며 802.11n에 근접; 동일 mother blocklength 기준 punctured code 대비 rate 9/10에서 성능 격차 존재(Fig. 5,6).
10. 한계: 순수 이론/시뮬레이션(AWGN, flooding decoder)뿐이며 HW 구현·gatecount·throughput 수치 없음, row-combining 제약이 LDPC 설계 자유도를 제한.

| back-substitution 기반 저복잡도 인코딩(block-lower-triangular H2) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 고정 인코더와 다른 구조라 재검증 부담 큼 |
| ACE/stopping-set graph-conditioning (error floor 개선) | `ecc_top.cpp` `Load_PCM()` | 부호설계 단계 기법으로 참고 가치 있으나 H-matrix 재설계 필요 |
| 구조화 sub-matrix 기반 병렬 디코더 아키텍처 | 디코더 HW 모델 (z=32 등) | Prime ECC도 이미 circulant/lifting 기반 병렬화를 사용해 개념적으로 유사 |

> 적용 가치: 낮음 | 다중 rate 지원이라는 논문의 핵심 목적이 Prime ECC의 단일 고정 rate NAND ECC 설계 목표와 부합하지 않으며, HW 구현도 없어 이식 시 재검증 부담이 큼.
