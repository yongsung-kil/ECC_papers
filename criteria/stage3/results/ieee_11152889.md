### B. 알고리즘 요약

1. UAV-to-UAV MISO(Nt≤4 안테나, 단일 수신 안테나) 통신 시스템에서 LDPC(irregular, rate 0.5, length 1296)로 부호화된 데이터를 BPSK/QPSK 변조·Rice fading 채널로 전송.
2. 기존 MAP 기반 MISO 검출은 안테나수·변조차수에 지수적으로 증가하는 NP-hard 문제이고, MAP LDPC 디코딩도 tanh/tanh⁻¹ 연산이 필요해 UAV의 제한된 자원(소형·저전력)에 부적합.
3. 채널모델은 단일경로 Rice fading(`h = √(k_l/(k_l+1))h_LoS + √(1/(k_l+1))h_NLoS`)과 AWGN.
4. 핵심 기법 1단: MAP MISO 검출기를 QNN(3-hidden layer FC, 은닉층 뉴런수 `15N_tM`)으로 근사, quantizer step size `s`까지 학습(STE로 rounding 미분 불가 문제 해결, 식(7)~(9)), 8-bit 양자화.
5. 핵심 기법 2단: LDPC 디코더는 offset min-sum(OMS) — `p_{ji}^{(k)} = sgn(Πq) · [min|q| − β(k)]^+` (식17), 곱셈/tanh 없이 adder+XOR+max/min만 사용.
6. β(k)는 iteration마다 감소하는 정수 시퀀스로 설계(`[14,14,14,14,14,12,12,10,10,8]`)해 min-sum의 과대추정 편향(overestimation)을 보정하면서 정수 연산만으로 정확도 손실 없음.
7. 부수 트릭: QNN 입력·가중치·bias 모두 uniform quantizer(식7)로 b=8bit 양자화, ReLU도 양자화 버전 사용, 전체 파이프라인이 FLOAT32→INT8로 전환되어 연산량 대폭 감소(Table I/II: MAP은 곱셈·tanh 필요, 제안 OMS는 곱셈 0).
8. 학습 절차: QNN을 MAP 검출기 출력(extrinsic LLR)을 타깃으로 MSE loss(식10)로 SGD/RMSProp 학습(lr=0.0001, batch=50, epoch≤20, early-stop threshold `e_m=1e-4`).
9. 결과: QNN 기반 검출기는 MAP 대비 BER=0.1에서 최대 0.2dB 손실; OMS 기반 LDPC 디코더는 MAP 대비 BER=1e-5에서 최대 0.5dB 손실이나 기존 min-sum보다 0.2~0.5dB 우수; 전체 LBW 파이프라인은 serial MAP 대비 약 0.5dB 손실로 8-bit(vs 32-bit MAP) 연산.
10. 한계: HW(RTL/FPGA/ASIC) 구현 없이 알고리즘 시뮬레이션 및 연산량 표(Table I/II) 분석에 그침, LDPC 자체 알고리즘(OMS)은 기존 문헌 재사용이며 신규성은 QNN 검출기와의 결합·8bit 양자화 조합에 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset Min-Sum (iteration별 가변 정수 β) | `decoder.cpp` `CNU_Update_New_Mag()`, `ecc_data.h` `PARAM_LLR` | Prime ECC의 min-sum도 iteration별 LLR threshold 테이블을 이미 사용 중이라 개념 유사, offset 값 결정 방식(정수 스케줄)은 참고 가치 있음 |
| 8-bit INT8 고정소수점 전 파이프라인 양자화 | `channel.cpp` `Set_LLR_Th()` | soft-read 양자화 비용과 직결되나 Prime ECC는 이미 2SD/3SD(2~3bit)급 저비트 양자화를 다루므로 8bit는 상대적으로 coarse, 직접 이식보다는 비교참고 |
| Layered decoding 스케줄 | `decoder.cpp` `LDPC_Decoder()` | Dual-Update(odd/even 교대)와 유사한 스케줄링 개념, 구체 계층화 순서는 본문에 미기재 |
| QNN 기반 MISO 검출기 | 대응 없음 | Prime ECC는 채널검출기(MISO/MIMO) 모듈이 없으며 NAND 채널과 무관 |
| 조기종료(신드롬 Hc=0) | `partial_CRC.cpp` / `full_CRC.cpp` | 표준 syndrome check 조기종료로 Full CRC와 유사, 신규성 없음 |

적용 가치: **중간** — LDPC 디코더 자체(OMS)는 표준 기법 재사용이라 직접 이식 가치는 낮으나, 8-bit INT8 고정소수점화와 iteration별 정수 offset 스케줄 설계는 저비트폭 soft-read 환경(NAND)에서 양자화 전략 참고 자료로 간접적 가치가 있음. MISO 검출기(QNN) 부분은 NAND와 무관.
