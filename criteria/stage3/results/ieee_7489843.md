### B. 알고리즘 요약

1. QC-LDPC 부호는 base matrix `B`(mxn)와 shift matrix `P`(mxn)로 표현되며, `B_i,j=1`이면 `P_i,j=S_i,j`만큼 순환이동된 pxp 단위행렬, `B_i,j=0`이면 영행렬로 확장(`p`=extension coefficient).
2. 풀려는 문제: 기존 random search 기반 shift matrix 구성법([5])은 girth를 키울수록 탐색 시간이 급증하고 큰 girth를 못 찾을 수 있으며, quadratic congruence/arithmetic sequence 기반 기법([7][9])은 length-4 cycle만 제거하고 length-6 이상은 다루지 않는 한계가 있음.
3. 핵심 가정: base matrix `B`의 block cycle(길이 `2t`)이 확장 후 H에서 몇 개·몇 길이의 cycle로 나타나는지는 `S1 = T mod p` (`T`=block cycle 상의 shift값 교대합)로 결정되며, `S1=0`이면 block cycle이 그대로 p번 복사되어 short cycle이 다량 생성됨.
4. 핵심 기법: `B_i,j=1`인 위치에 `S_i,j = i·j` (`1≤i≤m, 1≤j≤n`)로 shift matrix를 결정적으로 설정하고, `(S_i,j)_max = mn < p` 조건을 부여.
5. 핵심 식의 의미: length-4 cycle에서 `T = a1·b1`(`a1,b1`은 cycle 꼭짓점 간 인덱스 차), length-6 cycle에서 `T = a1(b1+b2)+a2b2` — 두 값 모두 cycle이 감싸는 영역의 "넓이" `S`와 같아 `S>0`이면 항상 `S1=T=S≠0`이 성립, 즉 length-4/6 block cycle의 복사(`S1=0`)를 원천 차단.
6. 확장: `(S1,p)≠1`인 최악의 경우에도 length-4 block cycle이 p/2개의 length-8 cycle로만 확장되어 girth가 최소 8 보장되며, `p`가 소수면 `(S1,p)=1`이 항상 성립해 block cycle이 단 하나의 length `2tp` cycle로만 확장(성능 극대화).
7. 부수 트릭: base matrix `B`는 기존 PEG(Progressive Edge-Growth) 알고리즘으로 구성(그대로 재사용), shift matrix만 새 결정적 수식으로 대체 — random search 불필요.
8. 학습/최적화 절차 없음(폐형 수식 기반 구성).
9. 결과: N=1414, R=0.5, m=7,n=14,p=101(소수) 코드(girth≥8)가 동일 조건 girth-4/6 코드 대비 BER=10^-5에서 약 0.1dB 성능 향상, 구성 시간 대비 random search girth-8 코드는 3.48×10^6 ms 탐색 후에도 못 찾음(제안법은 즉시 구성).
10. 한계: 표준 BP 디코딩(max iter 100)만으로 waterfall 개선을 확인했고 error-floor·HW 구현·대형 코드(N 수천 이상)에서의 검증은 없음.
