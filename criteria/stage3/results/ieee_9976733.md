> 총평: DVB-S2X 긴 부호(64800) QC-LDPC의 layered min-sum을 360-병렬코어와 BRAM 저장 최적화로 고throughput 구현, 알고리즘은 표준기법이라 메모리·병렬 아키텍처 참고용으로 이식성 중.

### B. 알고리즘 요약

1. 시스템: DVB-S2X 위성표준 QC-LDPC 디코더, normal frame length `64800`, 전 code rate 지원(1/4~5/6 등), Kintex-7 XC7k325 FPGA, soft-in/hard-out.
2. 문제: DVB-S2X는 codeword가 길고 PCM이 커서 전통 LDPC 디코더의 자원·메모리 부담이 큼 - 고throughput+저자원 동시 달성이 과제.
3. 부호 변환: DVB-S2X PCM을 QC(quasi-cyclic) 형태로 변환해 HW 친화 구조로 만듦.
4. 디코더: horizontal layered MSA - SO(soft-output)를 채널 LLR로 초기화, VTC `Q_mn = Λ_n - r_mn` (식5), 레이어 순차 갱신.
5. CTV 갱신: `r_mn = ∏sign · α · min|Q|` (식4), BP 대비 성능손실 보상 위해 normalizing coefficient `α` 사용(normalized MSA).
6. 병렬 구조: 360개 병렬 디코더 코어가 VTC / Min-Submin / CTV·APP 계산 담당, Min/Submin 블록은 dr(행 weight) 클록 동안 최소·차소·최소index 산출.
7. 메모리 최적화: APP는 true dual-port BRAM(36bit×180addr), APP 1값=9bit → 셀당 4값 → 360 APP 저장에 90 BRAM; CTV는 최대 주소 660(rate 5/6에서 row weight 22 × 30 layer).
8. 종료조건: 최대 iteration 도달 또는 `Hx=0` 신드롬 만족 시 조기종료, `Λn>0`이면 비트 0.
9. 결과: 64APSK rate 5/6에서 493Mbps(최대), QPSK rate 1/4에서 150Mbps(최소), clock 250MHz, LUT 115864 / BRAM 180 / DSP 0, iteration당 최대 10Gbps 도달 주장.
10. 한계: 정정능력(BER/error-floor) 개선은 없고 표준부호 고정, throughput/자원 최적화가 핵심 - soft LLR 9bit로 NAND read 비용 관점 아님, 통신(위성)채널 가정.
