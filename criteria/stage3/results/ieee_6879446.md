### B. 알고리즘 요약

1. 대상 시스템은 SSD가 HDD 캐시로 쓰이는 대규모 스토리지, 19nm TLC NAND(page 9kB=73,728bit, user 8kB, parity 1kB), raw BER 최대 `1e-2`.
2. 문제: LDPC 디코딩 latency가 입력 오류수에 비례 → BER 높으면 다중 sensing + iteration 증가로 read latency 악화.
3. 핵심 관찰 "error locality": 같은 page를 erase 전까지 두 번 읽으면 오류 위치의 약 `70~80%`가 동일 (Read1→Read2 559/725≈77%).
4. 핵심 기법 EC-Cache: 최근 읽은 page의 검출된 오류(위치+valid value)를 DRAM 해시테이블에 캐싱, 다음 read 시 LDPC 이전에 raw data에 merge하여 오류를 미리 정정.
5. 오류 위치 저장을 `8-bit offset coding`(절대 17bit 대신 상대거리)으로 압축, dummy entry `(255,1)`로 255 초과 간격 처리, line size 1kB(=page의 1/9)로 고정 → 데이터캐시 대비 8배 line 수.
6. 적응형 할당: SDH(stack distance histogram) + shadow cache로 워크로드 locality 추정, `Benefit=HitRate_Data + a·HitRate_EC`(a=0.4) 메트릭으로 epoch마다 DRAM을 데이터캐시/EC-Cache에 way 단위로 재분배.
7. 구현 디테일: assisting unit이 raw buffer와 정정된 output buffer 비교로 새 EC-entry 생성, 주 HW 오버헤드는 codeword 크기 raw buffer RAM.
8. 검증: 19nm TLC/25nm MLC 실측 error locality, 외부 LDPC 시뮬 프레임워크(9kb codeword, 922b parity)로 sensing/latency 모델, SSDSim+DiskSim으로 MSRC 워크로드 시스템 시뮬.
9. 결과: hit 시 오류 60%+ 사전 제거 → BER 0.015에서도 평균 1 sensing, read 성능 최대 `2.6x`(static)/`2.2x`(adaptive) 향상.
10. 한계: LDPC 디코더/부호는 블랙박스, 알고리즘·부호설계·디코더 HW 설계 없음. NAND read 시스템/컨트롤러 캐시 레벨 최적화에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EC-Cache DRAM 오류 캐시 / offset coding | 대응 없음 | 컨트롤러/펌웨어 레벨 - 디코더 코어 밖 |
| 적응형 DRAM 할당 (SDH, epoch) | 대응 없음 | 시스템 캐시 정책 - 코드베이스 범위 밖 |
| LDPC 입력 오류 사전 제거 → sensing/latency 감소 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` (간접) | 디코더는 소비자일 뿐, merge 전처리는 채널/입력 밖에서 수행 |
| soft-sensing(다중 read reference voltage) LLR | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` (간접) | 개념적 연결뿐, 논문은 LLR 설계를 다루지 않음 |

적용 가치: **낮음** - 디코더/부호/HW 어디에도 이식할 알고리즘이 없고, LDPC를 블랙박스로 다루는 SSD 컨트롤러 캐시 아키텍처라 Prime ECC 코어와 접점이 없다.
