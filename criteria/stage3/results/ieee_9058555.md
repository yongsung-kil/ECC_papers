> 총평: EQML 재처리(EWS+PPS)는 BP 실패 후 pseudo-codeword 제거로 정정능력을 높이는 알고리즘적 아이디어로 Prime ECC post-processing에 참고가치가 있으나, 논문 자체는 GPU 가속에 초점이라 HW 이식은 별도 설계가 필요하다.

### B. 알고리즘 요약

1. 시스템: WiMAX 802.16e (576,288) irregular LDPC(rate 1/2, length 576), AWGN 채널, GeForce GTX 1060 GPU 시뮬.
2. 문제: EQML은 BP보다 BER 우수하나 재처리 구조로 디코딩시간이 BP 대비 크게 증가 → GPU 병렬화로 속도만 회복.
3. 모델: LLR 기반 BP, `L(qij)=2yi/σ²` 초기화, CN update `φ(x)=log((e^x+1)/(e^x-1))`, VN update, APP hard decision.
4. 핵심기법 1단 (EQML 재처리): BP 실패 시 EWS(edge-wise selection)로 부호비트별 sign-flip 수 `w(vi)=Σ w_{l,i}`가 최대인 VN을 선택·포화 후 saturated 시퀀스를 재-BP.
5. 핵심 식: `vs=argmax w(vi)`로 가장 불안정한 노드를 고르고 동률 시 APP 최소 노드를 선택해 포화(pseudo-codeword 탈출).
6. 확장 (PPS): partial pruning stop으로 `TF=2^{jmax+1}-2` 초기화 후 수렴 서브브랜치를 조기 가지치기, `jc=jmax` 또는 `TF=0`에서 재처리 종료.
7. 구현 디테일: irregular H를 7개 1D 배열(V2C/C2V/rowdegree/coldegree/colnum/rowsum/colsum)로 압축저장, BP를 4 Kernel(VN/median/CN/APP)로 재정렬해 uncoalesced write 회피.
8. 병렬화: constant/shared memory 사용, 재처리 스테이지가 커질수록 단일→다중 codeword(2,4,8...)로 늘어나 `N×Ncodeword` 스레드로 intra+inter-codeword 동시 복호.
9. 결과: GPU가 CPU 대비 약 2자릿수 가속, 저SNR에서 speedup 최대 70(jmax=5), 성능 저하 없이 EQML이 BP보다 나은 정정능력 유지.
10. 한계: HW(ASIC/FPGA) 미설계, GPU 전용 구현, 짧은 부호(576)·rate 0.5·WiMAX 예시만.
