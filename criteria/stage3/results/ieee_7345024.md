> 총평: min-sum 산술 연산을 상호정보량 최적화된 LUT로 치환해 3bit 메시지로 5bit급 성능을 내는 finite-alphabet 디코더로, fully unrolled 아키텍처에서 면적 8%↓·속도 64%↑를 달성했으나 NAND向 검증은 없고 unrolled 구조 특화라 이식 시 구조적 재해석이 필요.

### B. 알고리즘 요약

1. IEEE 802.3an (6,32)-regular LDPC 코드(N=2048, rate 13/16=0.8125), BI-AWGN 채널, fully unrolled 구조(iteration마다 별도 VN/CN 인스턴스 N·I개).
2. 문제: 기존 min-sum은 4~7bit 양자화가 필요하고 저해상도는 error-floor에서 성능 손실이 큼.
3. 가정: 메시지 알파벳 `M`은 실수가 아닌 유한 label 집합, 채널은 대칭(symmetric) BI-AWGN.
4. 핵심 기법: VN update rule `Φv`을 산술 덧셈 대신, CN-to-VN 메시지 분포 `p(μ|x)`와 채널 LLR 분포로부터 상호정보량 `I(m;x)`을 최대화하는 LUT로 설계(`Φv(i)_MI = argmax I(Q(L,μ̄);x)`).
5. 핵심 식(12): 각 iteration `i`마다 이전 분포를 이용해 지역 최적 LUT를 재귀적으로 구성, 메시지가 iteration마다 다른 표현(label)을 가지며 암묵적으로 동적 범위 확장 효과.
6. CN update는 기존 min-sum 규칙(6)을 그대로 유지 — LUT 출력이 정렬 가능하고 대칭 채널에서 부호가 label로부터 바로 추론되기 때문(식 16).
7. HW 디테일: VN을 `dv`개의 LUT-tree(계층 구조, Fig.3a)로 구현, LUT 공유로 자원 절감; decision node는 단일 LUT-tree로 이진 출력.
8. 학습/설계 절차: cycle-free Tanner graph 가정 하에 식(8)(9)(11)(13)을 iteration별로 반복 적용해 각 단계 LUT를 사전 계산(density evolution 유사 절차, SNR-specific 설계).
9. 결과: Qch=4bit·Qmsg=3bit로 fixed-point 5bit 성능과 동등한 FER; TSMC 90nm 합성 시 adder 기반 대비 면적 8%↓, 주파수 64%↑, throughput 1665Gbps(24% area efficiency 73%↑), 배선 약 40% 감소.
10. 한계: fully unrolled(비반복 재사용) 아키텍처 전용 설계이며, 배치·라우팅 미반영 합성 결과일 뿐이고 NAND/SSD 관련 실험은 없음(일반 통신 표준 코드로만 검증).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VN update LUT (mutual information 기반) | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD) | 산술 min-sum VN 연산을 LUT로 치환하는 아이디어는 이식 가능하나 fully unrolled 특화라 재설계 필요 |
| CN update (기존 min-sum 유지) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN 구조는 그대로 유지되므로 직접 대응 |
| iteration별 LLR/메시지 표현 변화 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 이미 iteration별 LLR 테이블 보유, LUT 재구성 개념과 유사하나 label 기반 표현은 새로운 접근 |
| fully unrolled 파이프라인 (iteration별 별도 HW) | 대응 없음 | Prime ECC는 iteration 재사용(반복) 구조로 unrolled와 근본적으로 다른 HW 철학 |
| sign-magnitude 기반 CN sorting(2-min) | `decoder.cpp` `C2V_Cal_New_Sgn()`, `decoder.h` `Get_VNU_*` | 이미 유사한 min/sub-min 정렬 구조 보유 |

> 적용 가치: **중간** — LUT 기반 저비트폭 VN update라는 핵심 아이디어는 soft-read 비용 절감 관점에서 의미 있으나(연판정 3bit로 5bit급 성능), fully unrolled 전용 검증이라 반복재사용형 Prime ECC 구조에 이식하려면 iteration별 LUT 저장 오버헤드 재평가와 NAND 채널 재설계가 필요.
