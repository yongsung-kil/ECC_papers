### B. 알고리즘 요약

1. 길이 `n=2640` Margulis LDPC code, binary linear code에 대한 LP 복호를 HW로 구현하는 것이 목표. min-sum과 면적/비트폭 경쟁이 동기.
2. BP/min-sum은 고SNR error-floor가 있어 고신뢰 응용에 부적합. LP 복호는 error-floor가 거의 없고 ML certificate 성질을 가짐.
3. ML 복호를 LP로 완화(Feldman), 1-penalty 함수 `α||x-0.5||_1`로 비정수 vertex를 회피해 waterfall 개선(Liu-Draper).
4. ADMM으로 LP를 분해, primal `x` / auxiliary `z` / dual `λ` 를 round-robin 갱신(식 8). C2V 메시지는 `z_j`, `λ_j`, V2C 메시지는 `x`.
5. 최대 병목은 `d`차원 parity-polytope 투영 `Π_PPd(·)`. Wasson은 simplex 투영으로 변환해 FPGA 구현했으나 면적·비트폭이 큼.
6. 핵심 관찰: 입력을 균일 양자화 `Q(v)`(식 9)한 뒤 PPd 투영 출력 알파벳들의 합집합 `Z=∪Z_i`의 카디널리티가 개별 합보다 훨씬 작음(Observation 3).
7. C2V 비트폭 = `⌈log2|Z|⌉`. degree-6 CN에서 `|Z|`이 43~123으로 작아 6~7bit로 표현 가능(기존 [12]는 10bit).
8. Algorithm 2로 수정: dual 갱신을 check→variable로 이동, 스칼라 LUT/LUT⁻¹ 추가, V2C에 균일 양자화 적용해 알파벳을 투영 입출력에 정렬.
9. 결과: 길이 2640 Margulis, 1000 iter 시뮬. curve 3 파라미터(`μ=3.9, α=0.8`)로 double precision 대비 근소한 손실, 평균 메시지 비트폭 5 달성.
10. 한계: HW 미구현(FPGA 목표만), 파라미터 민감(curve 1,2는 2.1dB부터 error-floor), Margulis code 단일 검증.
