### B. 알고리즘 요약

1. 시스템: WiMAX QC-LDPC, rate-1/2, length-576(및 2304), AWGN+BPSK, BP(belief propagation) 소프트 디코딩, 최대 iteration 5~50.
2. 문제: IDS(informed dynamic scheduling)는 최대 residual edge를 우선 갱신하는 greedy 전략인데, 자원 배분 불균형으로 성능 저하.
3. 핵심관찰: "update-relayed trend" - 갱신된 메시지로 residual 재계산된 edge가 다음에도 갱신될 확률이 높고, 이 경향이 cycle 경로를 따라 전파되어 소수 edge가 갱신 기회를 독점(greedy update cycle).
4. 기법(RSPUR): 전체 edge가 아니라 `n_p`개 check node로 구성된 좁은 update range `S`에서만 최대 C2V residual을 탐색·갱신.
5. 핵심원리: range를 좁히면 그 안에 포함되는 cycle 수가 급감 -> update-relayed 전파가 억제되고 탐색 복잡도도 감소.
6. 절차: `S`에 `n_p`개 check node를 무작위 선택 -> range 내 `r_{cm->vn}` 계산 -> 최대 residual `m_{cmax->vmax}` 갱신·전파 -> V2C 갱신 -> 매번 `S` 재선택(모든 CN 동일 기회).
7. 메시지식: C2V는 full `tanh`/`tanh^-1` BP식 (1), V2C는 채널 LLR `C_vj` + 이웃 C2V 합 (2), residual (3) 기반 우선순위.
8. range 튜닝: test frame(예: all-zero codeword) 전송으로 update-error-rate·BER 비교해 최적 range 결정(576->4 CN, 2304->8 CN); range>18이면 greedy 상태로 성능 악화.
9. 결과: FER `1e-5`에서 OV-RBP 대비 `0.12dB` 이득, SNR 3.5dB에서 처리량 LQRD 대비 약 `52%` 향상(CPU 기준, HW 아님).
10. 한계: 시뮬(CPU)만·HW 미설계, full-tanh BP+순차 스케줄, update range 오프라인 튜닝 필요, throughput은 "성공프레임/시간" 소프트웨어 지표.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IDS/RSPUR 스케줄링 (edge 선택 순서) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 중 - 스케줄 계층은 디코더 루프에 닿으나 Prime ECC는 flooding+Dual-Update 고정, 순차 IDS 삽입은 pipeline과 상충 |
| C2V residual 계산/갱신 | decoder.cpp C2V_Cal(), CNU_Update_New_Mag() | 중 - CN 갱신부에 해당하나 논문은 full tanh, Prime ECC는 min-sum이라 residual 정의 재정합 필요 |
| full tanh Sum-Product 메시지식 | 대응 없음 | Prime ECC는 min-sum 근사, full-tanh SP는 미지원(프로파일 §6 대응 없음 명시) |
```

적용 가치: **중간** - greedy update cycle 억제라는 스케줄링 통찰은 error-floor/수렴 개선에 유효하나, full-tanh BP·순차 스케줄이 min-sum 완전병렬 pipeline인 Prime ECC와 아키텍처가 달라 직접 이식엔 재설계 부담.
