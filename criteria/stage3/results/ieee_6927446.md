### B. 알고리즘 요약

1. 시스템: SDR 플랫폼용 고처리량 재구성 FEC 프로세서, Turbo/QC-LDPC/Convolutional Code 3종을 SW/HW 공동 재구성으로 지원.
2. 문제: 표준·알고리즘이 불확정한 SDR에서 전용 디코더는 유연성 부족, FPGA 재구성은 시간 과다. 빠른 재구성 가능한 다중알고리즘 가속기 필요.
3. 핵심 커널: forward-backward-recursion(FBR)을 통합 계산 커널로 선택, 8-state trellis 하나로 Turbo(BCJR)/CC(Viterbi)/LDPC(layered) 모두 표현.
4. 커널 함수: `f(x,y)`를 알고리즘별로 정의 - LD는 `sign·min` min-sum형, BCJR는 `max{x,y}`, Viterbi는 `min,idxmin`으로 통합 PE에서 스위칭.
5. LDPC 매핑: (N,M) QC-LDPC를 base matrix(J×K) shift로 확장, 한 row=layer가 Z개 sub-layer로 구성, 비영 sub-matrix chain을 cycle별 처리하며 식(7)(8)(9)의 α/β/Λ 업데이트.
6. 병렬 구조: Multiple-Memory-Multiple-Datapath - P개 MBG와 P개 PE 병렬, ICB(Inter-Connection-Bar)가 cyclic-shifter/crossbar로 permutation 담당, AGU가 접근 패턴 생성.
7. 구현 디테일: 채널 데이터 5-bit 양자화(MBG당 40bit=8×5bit), ICB 내 cyclic-shifter로 sub-matrix shift 처리.
8. ASIP: 명령어가 1개 trellis recursion step 제어, 구성 레지스터(CRF)+명령어 내 config bit로 모드 전환, HW loop/instruction repeat로 SW 루프 가속.
9. 결과: Stratix IV에서 80MHz, 802.11n LDPC 230Mbps(6 iters), Turbo 96.8Mbps, 저가형 Cyclone III는 46717 LE·63MHz·608mW.
10. 한계: NAND 무관 무선표준 대상, error-floor/정정능력 개선 없음, LDPC는 통합 trellis로 우회 구현되어 전용 min-sum 대비 최적화 아님.
