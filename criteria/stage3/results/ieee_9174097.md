### B. 알고리즘 요약

1. 짧은 선형블록코드(RM, CCSDS LDPC length 128 rate 0.5)를 대상으로 overcomplete parity-check matrix `Hoc` 기반 weighted BP(WBP) 디코딩을 다룬다.
2. 기존 BP는 dense/short-cycle이 많은 H에서 성능이 제한되고, 표준 WBP도 어떤 H를 쓰느냐에 성능이 크게 좌우된다는 문제를 다룬다.
3. Tanner graph의 CN 방향 가중치 `w_c^(ℓ)` 크기를 CN의 디코딩 기여도 지표로 간주하는 것이 핵심 가정이다.
4. CN 단위로 weight를 tie한 modified WBP를 학습(Adam/Tensorflow)한 뒤, 가장 작은 `w_c^(ℓ)`를 갖는 CN을 순차적으로 제거(pruning)한다.
5. CN update는 `λ_c→v = 2 w_c^(ℓ) tanh^{-1}(∏ tanh(λ_v→c/2))`로, weight 크기 0에 가까우면 해당 parity-check가 사실상 무의미해져 제거 가능함을 의미한다.
6. iteration마다 서로 다른 H_ℓ을 허용해, 최종 디코더는 iteration별로 다른 parity-check 집합을 사용한다(재학습-pruning 반복).
7. weight 사용 여부에 따라 D1(tied weight 사용)/D2(weight 없음, 순수 pruned H)/D3(untied weight)로 3종 디코더를 구성해 복잡도-성능 trade-off를 제공.
8. 학습은 soft bit-error-rate 기반 multi-loss(η로 중간층 기여도 조절)로 진행되며 all-zero codeword 가정(대칭성) 활용.
9. RM(3,7)에서 D1이 ML 대비 0.27dB, CCSDS LDPC(128,0.5)에서 1.5dB(기존 BP 대비 0.5~0.6dB 개선), 동일 CN 평가 수 대비 MBBP·기존 WBP를 능가.
10. HW 미설계(순수 시뮬레이션), 복잡도 지표는 tanh 연산 횟수(CN evaluation 수)로만 간접 측정, 학습 비용·수렴성은 코드 길이 커질수록 문제(RM(3,7)은 random subset 필요).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN 가중치 기반 pruning(overcomplete H iteration별 축소) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 iteration마다 바꾸는 구조라 고정 QC-LDPC PCM 로드 방식과 근본적으로 다름 |
| WBP CN/VN update (tanh 기반) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum(§3) 채택, full-tanh Sum-Product 기반 weight 학습은 대응 없음 |
| iteration별 다른 parity-check 집합 사용 | 대응 없음 | HW pipeline·HCU는 고정 PCM 전제로 설계됨 |
| soft bit-error-rate multi-loss 학습(Adam/Tensorflow) | 대응 없음 | 오프라인 신경망 학습 절차이며 HW 디코더 알고리즘과 무관 |

> 적용 가치: 낮음 — non-binary 학습 기반 오프라인 최적화, 저rate 표준코드(RM/CCSDS) 실험 중심이며 iteration별 가변 PCM은 Prime ECC의 고정 QC-LDPC PCM·HW pipeline 구조와 정합되지 않음.
