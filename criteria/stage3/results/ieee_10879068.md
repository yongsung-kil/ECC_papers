### B. 알고리즘 요약

1. 3D NAND(MLC 기준, 최대 200+ layer) TLC/QLC를 대상으로, LDPC 대신 systematic shortened polar code를 ECC로 사용하는 RaPC(RBER-aware Polar Coding)를 제안. codeword length N=8192(2^13)를 4608bit로 shortening, code rate 0.889로 LDPC(rate 0.889, length 4608)와 동일 조건 비교.
2. 풀려는 문제: LDPC는 반복 디코딩 구조상 RBER 증가 시 iteration 수 증가로 디코딩 지연이 커짐. Polar code는 SC/SCL 디코딩 구조가 고정적이라 RBER 변화에 지연이 덜 민감하지만, 특정 RBER 이하에서는 LDPC보다 정정 성능이 열세.
3. 채널 모델: [26]의 3D NAND retention-loss 채널 모델(P/E cycle, retention time 함수의 Vth 분포 drift), nonuniform sensing으로 overlap 영역에 read reference voltage 집중 배치해 LLR 산출(식 2,3).
4. 핵심 기법: SCL(Successive Cancellation List) 디코딩의 list length `L`을 RBER 구간에 따라 동적으로 전환(1→2→4→…→32). RBER이 낮으면 SC(list=1)로 충분, RBER이 상승하면 완전 정정을 위해 list를 키우고, 이후 polar code가 LDPC 곡선의 변화율보다 완만해 교차점 이후엔 list를 다시 줄여 지연을 낮춤.
5. 핵심 식: PM(path metric, 식 7,8)으로 상위 L개 경로만 유지 — list 크기가 정정능력·저장·지연 3자 트레이드오프의 유일한 조정 변수.
6. 확장: CA-SCL(CRC-aided, 정정성능 우위) 외에 Fast-SCL(Rate-0/Rate-1/Rep/SPC 특수노드 조기 판정으로 지연 대폭 감소, 정정성능 소폭 손실) 두 디코더를 병행 제공해 사용자가 목적에 따라 선택.
7. 부수 트릭: systematic polar coding(식 10~12)으로 nonsystematic 대비 BER 개선, list 상한을 32로 제한(64 이상은 정정 이득 대비 저장 오버헤드 배증이 부적절).
8. 학습/최적화 절차 없음(list 전환 임계 RBER은 시뮬레이션으로 사전 결정, DE/GA/NN 미사용).
9. 결과: RBER > 9.5×10^-3(BER 기준, CA-SCL)에서 RaPC가 LDPC 대비 정정성능 우위(CA-SCL 평균 정정능력 LDPC 대비 약 6배, Fast-SCL 약 4.5배); 디코딩 지연은 RBER 6×10^-3~1.2×10^-2 구간에서 LDPC 대비 CA-SCL 약 0.2배, Fast-SCL 약 0.1배; FEMU 시스템 시뮬 기준 대역폭 최대 40% 개선.
10. 한계: MATLAB/FEMU 시뮬뿐 HW 미구현, LDPC는 min-sum 기준 비교이나 Prime ECC와 다른 부호 패밀리(polar)라 이식 시 디코더 구조(SC 트리 기반) 자체를 새로 만들어야 함, list 커질수록 공간 오버헤드 급증(list=16에서 64KB).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SCL 디코딩(list length 동적 조정) | 대응 없음 | polar code 전용 SC 트리 디코딩 구조이며, Prime ECC의 min-sum 반복 메시지 패싱과 부호체계·디코더 구조가 근본적으로 다름 |
| RBER 인지 동적 파라미터(list length) 조정 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 개념적 유사성만(신뢰도/에러율에 따라 파라미터 테이블 전환) — 실제 파라미터(list vs LLR threshold)와 메커니즘은 무관 |
| nonuniform sensing / LLR 산출(식 2,3) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | LLR 산출 방식은 유사한 memory-sensing 개념이나 polar 전용 LLR 수식이라 그대로 이식 불가 |
| 부호 자체(polar code 구조) | 대응 없음 | Prime ECC는 binary QC-LDPC 전용, non-LDPC 부호는 §5 필터 기준 이식성 하 |

적용 가치: `낮음` — polar code 기반 대안 ECC 제안으로 Prime ECC(binary QC-LDPC)와 부호 패밀리 자체가 다름. RBER 구간별 동적 파라미터 전환이라는 설계 아이디어만 개념적으로 참고 가능한 수준.
