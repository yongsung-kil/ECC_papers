### B. 알고리즘 요약

1. 대상: `(l,r)` regular LDPC code 앙상블(bit degree `l`, check degree `r`), Tanner graph `G=(V,C,F)`, 유한길이 아닌 `n→∞` 점근 영역.
2. 문제: LDPC error-floor는 trapping set으로 설명되나, 유한정밀 반복복호에서 실제 오류는 그 부분집합인 absorbing set(특히 BF에서 안정적인 fully absorbing set)이 더 정확히 기술.
3. 정의: `(a,b)` absorbing set은 각 bit node가 불만족 check(`O(D)`)보다 만족 check에 더 많이 연결(`σi>l/2`)된 크기 `a` 집합, fully absorbing set은 집합 밖 bit node까지 동일 조건(`μk>l/2`).
4. 핵심 기법: Milenkovic-Soljanin-Whiting [1]의 trapping set enumeration을 확장, PCM 전치 `H^T`를 `M1|M2|M3` 부분행렬로 분해해 행/열 weight 제약 하 이진행렬 개수를 점근적으로 계산.
5. 핵심 식: `p^{l,r}_{a,b,n}` (평균 absorbing set 수)의 정규화 로그 점근분포 `e_{l,r}(θ,λ)=lim (1/n)log p`, `θ=a/n`, `λ=b/n` — 앙상블에서 특정 크기 결함구조가 지수적으로 몇 개 존재하는지 지수를 준다.
6. 확장(Theorem 3): fully absorbing set은 집합 외부 노드 제약 `μk` 추가, `M3`을 `M31|M32`로 재분해해 enumeration.
7. 단순화: bit degree `l=3,4`인 elementary (fully) absorbing set에 대해 엔트로피 함수 `Hb` 기반 닫힌형 공식(Cor.1~4) 유도, Stirling·binomial 근사 사용.
8. 존재 한계: `λ/θ≤1` 및 `λ≤1-R`(설계 rate `R`)에서만 absorbing set 존재; `r≥9`면 지수적으로 다수, `r<9`이면 작은 `θ,λ`에서 subexponential.
9. 결과: `θ,λ`가 작고 rate 중간일 때 absorbing≈fully absorbing≈trapping set; rate↑·`λ/θ→1`이면 trapping set과 absorbing set 분포 괴리 급증 → trapping set만으로는 error-floor 예측 부정확.
10. 구조부호 함의: RAA(rate 1/3), (3,6) protograph 부호가 random regular 앙상블 대비 좋은(또는 동등) absorbing set 점근특성을 가짐을 시사.
11. 한계: 전부 이론/점근, 특정 유한길이 부호·HW·복호기 미설계, 구체 rate/length 없음.
