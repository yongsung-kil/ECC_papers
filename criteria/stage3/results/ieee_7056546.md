### B. 알고리즘 요약

1. 배경: flash controller의 기본 ECC인 BCH는 고 RBER 환경에서 과도한 오버헤드(rate<0.8) 요구, 이에 대안으로 낮은 error-floor를 갖는 algebraic QC-LDPC가 대두. QC 구조(circulant 배열)는 routing/메모리 복잡도를 줄여준다는 점을 서론에서 명시.
2. 문제: QC-LDPC의 parity-check matrix `H_qc`(식 5, c×t circulant 배열)가 non-full-rank(중복 parity-check row 포함)인 경우, Gaussian elimination 기반 생성행렬은 dense/non-circulant가 되어 인코더 구현이 복잡해지고, 기존 Li et al.[5] 방식은 non-systematic 코드워드(메시지가 codeword에 직접 나타나지 않음)를 만들어 실사용상 불편.
3. 기존 대안(Zeng et al. 특허)의 한계: remainder matrix를 0으로 대체해 systematic화하지만 `m−r`(m=rank 부족분)만큼 메시지 비트를 잃고 그만큼 parity를 더 부담해 정정능력이 저하됨.
4. 제안 기법(2-phase): phase 1은 `H_qc`를 rank `r` 기준으로 재구성한 `H^(1)_qc = [M^(1) D^(1)]`(식 10)에 대해 row echelon 형태 `D^(2)`를 구해(식 11) generator `G_qc = [I P_qc]`(식 16)를 얻고, l개의 SRAA(shift-register-adder-accumulator) 회로로 "extended parity" `p^(1)`을 생성(식 17).
5. 핵심 식: `D^(2)`의 rank가 `r < lb`이므로 `p^(1)`에는 `lb−r`개의 "dimension-free bit"가 존재(식 17~23) — 이 여유 차원이 phase 2에서 메시지 `a(2)`를 끼워 넣을 자리가 됨.
6. 핵심 기법 2단(phase 2): `D^(2)`의 row echelon 형태에서 0-대각원소 위치 `z_i`를 찾아, 해당 위치의 parity bit를 `f_{z_i}∈{0,1}` 선택으로 원하는 메시지 비트 `a(2)`와 일치시킴(식 24~26) → 손실 없이 메시지를 parity 영역 안에 piecewise로 삽입.
7. 구현 디테일: phase 2는 phase 1과 동일한 l개 SRAA 회로를 재사용(레지스터 B를 A의 left-feedback shift로 대체해 절반 절감), 최종 인코더는 `l·b` flip-flop + `l·b` AND-gate + `l·b` XOR-gate + wire-table로 구성(Fig. 3).
8. 학습/최적화 절차: 없음 (닫힌 형태의 선형대수적 유도, row echelon 변환 기반).
9. 결과: 정보비트 손실이나 추가 parity 부담 없이(기존 Zeng 특허 대비) systematic(piecewise) 인코딩 달성, SRAA 레지스터를 원안 대비 절반으로 절감했다고 서술(정량적 게이트/면적 수치는 본문에 없음).
10. 한계: HW 구현은 회로 구성요소 나열 수준이며 FPGA/ASIC 합성 결과나 throughput/latency 수치 없음, 디코더 성능(정정능력)에는 영향 없는 순수 인코더측 논문.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| non-full-rank QC-LDPC systematic encoding (SRAA 2-phase) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코더 구조를 사용하므로 이 논문의 SRAA 기반 대안은 구조적으로 다르지만, non-full-rank PCM 대응이라는 문제의식은 `Generate_PCM_encoding()`과 관련 |
| H-matrix rank/circulant 재구성 (H^(1), H^(2) 유도) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드/분석 단계와 관련되나 Prime ECC는 특정 H-matrix 고정이라 재검증 부담 큼 |
| Column Reordering 없이 parity 영역에 메시지 삽입 | `encoder.cpp` `Column_Reordering_Dual_Update()` | 목적(부호 구조 유지하며 encoding 효율화)은 유사하나 구체 메커니즘(dual-update)과는 다른 접근 |

> 적용 가치: 중간 - flash controller ECC를 배경으로 명시한 non-full-rank QC-LDPC 인코더 이론이나, Prime ECC는 이미 고정된 dual-diagonal 인코더 구조를 사용해 인코더 교체는 재검증 부담이 크고(§4 "인코더: 중~높음"), 디코더/정정성능과는 무관.
