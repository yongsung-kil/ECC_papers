### B. 알고리즘 요약

1. 통신 채널(Wi-Fi/5G 가정), regular LDPC `(24,12)` 코드(정보 12비트→코드워드 24비트), 12×24 sparse parity-check matrix H, rate 0.5.
2. 문제: SPA(sum-product)는 곱셈/나눗셈/지수 연산으로 하드웨어 비용이 크므로, 저전력 FPGA/ASIC용으로 min-sum 근사 디코더의 완전한 RTL→합성 워크플로우를 시연.
3. 채널 모델: 단일/이중 비트플립 오류를 주입하는 controlled bit-flip 채널(Wi-Fi 스타일 왜곡 모사), 정밀 확률모델 없음.
4. 핵심 기법: 표준 Min-Sum 알고리즘 — CNU는 `min1, min2`와 sign XOR 계산, VNU는 채널 LLR과 인접 CNU 메시지 누적 후 extrinsic 정보 생성.
5. 핵심 식: 고정 magnitude LLR 양자화 `수신비트 0→+8, 1→-8` (부호=비트결정, 크기=고정 신뢰도) — 곱셈기·정규화 회로 완전 제거가 목적.
6. 확장 없음(normalized min-sum도 검토했으나 스케일링 팩터/곱셈기 필요해 채택하지 않고 기본 min-sum 유지).
7. 구현 디테일: CNU/VNU 완전 파이프라인(레지스터로 critical path 절단), syndrome 기반 조기종료(`H·c^T=0`), re-encode & compare 검증 방식.
8. 별도 학습/최적화 절차 없음(고정 파라미터 RTL 구현).
9. 결과: FPGA에서 92 LUT(0.22%), 전력 0.086W(동적 <0.004W); ASIC(Cadence Genus)에서 12477 cells, 114620.951µm², 총 전력 0.00414628W, 임계경로 지연 2.882ns@100MHz, 10개 테스트케이스 100% 성공.
10. 한계: 코드 크기(24,12)가 극소형으로 확장성 논의만 있고 대형 코드 실측 없음, NAND/SSD 관련 언급 전혀 없이 Wi-Fi/5G/IoT 타깃, 정정능력(waterfall/error-floor) 비교나 개선 주장 없이 순수 구현 검증 논문.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum CNU(min1/min2, sign XOR) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 보유한 표준 min-sum과 동일 원리, 중복 기여(추천도 낮음) |
| VNU 메시지 누적/양자화 | `decoder.cpp` `VN_Cal_HD()`, `decoder.h` `Get_VNU_*` | 표준 VN 업데이트, 이미 구현됨 |
| syndrome 기반 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 이미 Partial/Full CRC로 더 정교하게 구현됨 |
| 고정 magnitude LLR 양자화(+8/-8) | `channel.cpp` `Set_LLR_Th()` | Prime ECC는 iteration별 LLR threshold 테이블(HD/2SD/3SD)을 이미 보유, 이 논문은 단일 고정값 수준으로 단순 |
| 파이프라인 RTL(레지스터 삽입) | 디코더 HW 모델 (pipeline 6-stage) | Prime ECC가 이미 pipeline bit-exact 모델 보유, 규모/복잡도 차이 큼 |
| Wi-Fi/5G 채널 모델·타깃 | 대응 없음 | NAND/RBER 채널이 아닌 통신 채널 특화, 대응 없음 |

> 적용 가치: 낮음 — 표준 min-sum/syndrome 조기종료의 소규모 교육용 RTL 구현이며 Prime ECC가 이미 더 정교한 형태(HCU, GT, adaptive LLR, partial CRC)로 보유한 기법과 중복되고 NAND 특화 요소가 전무함.
