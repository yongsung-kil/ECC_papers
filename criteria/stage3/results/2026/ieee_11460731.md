> 총평: 5G/6G NR LDPC에 대한 GNN(MLP/GRU/LSTM) 기반 학습형 디코더 제안으로 NAND binary QC-LDPC HW 이식과는 거리가 멀다.

### B. 알고리즘 요약

1. 5G NR BG1/BG2 기반 QC-LDPC(예: (648,324), 코드rate 0.5/0.75)에 대해 AWGN·Rayleigh TDL-A 채널에서 GNN 기반 디코더를 평가.
2. 기존 BP/MS/NMS는 노이즈·페이딩·비선형 환경에서 성능 저하 및 error floor 문제가 있어 적응형 디코더가 필요하다는 문제의식.
3. Tanner graph의 variable/check node를 그래프로 보고 메시지패싱 함수를 학습 가능하게 대체하는 모델(채널 노이즈는 AWGN/Rayleigh로 가정).
4. 메시지 업데이트 `m_{i→j}^{(t+1)} = m^{(t)}(h_i^{(t)}, h_j^{(t)}, e_ij)`, 노드 업데이트 `h_i^{(t+1)} = u^{(t)}(h_i^{(t)}, Σ_{j∈N(i)} m_{j→i}^{(t+1)})`로 고정된 BP 규칙 대신 학습 파라미터 함수 사용.
5. `m^{(t)}`, `u^{(t)}`는 MLP(`y=σ(Wx+b)`) 또는 GRU/LSTM 게이트 함수로 구현 - 채널 메모리 효과를 반영하기 위함.
6. GRU는 update/reset 2게이트만으로 LSTM(forget/input/output 3게이트) 대비 경량화된 대안으로 제시.
7. 초기값은 LLR(`h_i^{(0)}=LLR(v_i)`), 최종 판정은 `h_i^{(T)}`의 부호로 hard-decision.
8. 학습: Adam optimizer(lr 0.001), batch 64 codeword, 200~500 episode, hidden dim 64~128, binary cross-entropy loss.
9. 결과: BER 10^-4 기준 256QAM 0.5~0.8dB, 1024QAM 1~1.2dB 이득. BP 대비 FLOP/latency 최대 60% 절감(BP 350MFLOP/4.0ms → GNN 130MFLOP/1.3ms).
10. 한계: MATLAB 시뮬뿐으로 HW 미설계, 5G NR 채널(AWGN/Rayleigh) 가정에 한정, 학습형 디코더라 NAND 고전 binary LDPC HW 이식과는 패러다임이 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GNN 기반 메시지패싱 디코더 전체 | 대응 없음 | NN 전면 대체 디코더로 프로파일 §6 "대응 없음 예시"에 명시된 범주 |
| LLR 초기화 | `channel.cpp` `Set_LLR_Th()` | LLR 개념 자체는 겹치나 학습형 초기화는 이식 불가 |
| 5G NR BG1/BG2 QC-LDPC 구조 | `ecc_top.cpp` `Load_PCM()` | 5G 표준 부호구조로 NAND 전용 고정 H-matrix와 무관 |

> 적용 가치: 낮음 — NN 기반 학습형 디코더로 Prime ECC의 bit-exact HW 시뮬레이터(min-sum 기반)와 패러다임이 다르고, 5G/6G 채널·표준 코드에 한정되어 NAND binary LDPC 이식성이 없음.
