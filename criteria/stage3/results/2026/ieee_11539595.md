### B. 알고리즘 요약

1. 3D TLC NAND, 4KB 데이터 단위, QC-LDPC 베이스(column weight 4, row weight 36, CPM 1024)에서 파생된 sub-LDPC 2종으로 구성, 서브코드 rate 0.94.
2. 문제: hot read data(자주 읽히고 retention 짧은 데이터)에서 기존 hard/soft 하이브리드 LDPC 디코딩이 soft 디코딩 트리거 시 큰 latency 급증을 유발.
3. 가정: hot read data는 짧은 retention·낮은 PEC로 RBER이 낮게(≤5×10⁻³) 유지되는 특성을 실측(FPGA, 160 layer 3D TLC) 근거로 활용.
4. 핵심 기법: TD-LDPC — 정보행렬을 `r`행 `c`열로 배치하고 각각 서브코드 `LDPC1(n1,j1,k1)`(행), `LDPC2(n2,j2,k2)`(열)로 이중 인코딩. rate `R = K/N` (식 3)로 계산.
5. 디코딩은 각 서브코드에 layered normalized min-sum(LNMS) 적용, `r·H^T=0` 검증 후 행 디코딩→열 디코딩→(열 결과를 새 행 디코딩 입력으로) 반복 스캐닝(Algorithm 1, `CycleNum`회 한도).
6. 확장 설계: TD-LDPC-2ways(행/열 동일 서브코드, CPM 256)와 TD-LDPC-4ways(CPM 128, 4행4열)로 병렬도·CPM 크기를 조절.
7. 구현 디테일: VDLDPC 상위 구조는 hot/cold 데이터 패턴 분류기 + ECC 스킴 선택기로, hot 데이터에만 TD-LDPC 적용, cold 데이터는 기존 1차원 LDPC 유지.
8. 별도 학습 절차 없음(구성적 설계, 파라미터는 실험적으로 CPM/row·col 크기 조정).
9. 결과: TD-LDPC-4ways는 UBER를 RBER 5×10⁻³에서 10⁻⁶까지 낮추고(LDPC-hard 대비 개선, LDPC-soft에는 못 미침), soft LDPC 대비 지연 최대 9배, hard LDPC 대비 2배 감소. 시스템 레벨(FEMU)에서 read response time 최대 12.0%(평균 5.8%) 개선.
10. 한계: RBER 5×10⁻³ 이상에서는 정정능력이 soft LDPC보다 약함(적용 범위 제한적), FPGA는 RBER 측정용이며 TD-LDPC 자체는 순수 시뮬레이션(FEMU/수치 시뮬)만으로 검증, HW 미설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TD-LDPC 2차원 부호 구성(행/열 sub-LDPC, H-matrix 이원화) | `ecc_top.cpp` `Load_PCM()` | 기존 단일 QC-LDPC PCM_b 구조를 2개 서브코드로 분할해야 하므로 부호설계 전면 재작업 필요 (이식 난이도 높음) |
| 행-열 반복 스캐닝 디코딩(Algorithm 1) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 반복 제어 로직 자체는 기존 이터레이션 루프 확장으로 이식 가능하나 서브코드 2개 동시 관리 필요 |
| LNMS(layered normalized min-sum) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | normalized min-sum은 기존 min-sum과 유사 계열, 정규화 계수 도입 정도로 낮은 난이도 이식 가능 |
| hot/cold 데이터 분류 + ECC 스킴 선택기 | 대응 없음 | FTL/컨트롤러 정책 레이어로 Prime ECC(디코더 시뮬레이터) 범위 밖 |
| 인코딩(행/열 이중 인코딩) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 현재 dual-diagonal 고정 인코더 구조와 다른 2D 인코딩 방식으로 재설계 필요 (이식 난이도 중~높음) |

> 적용 가치: 낮음 — 정정능력과 latency를 동시에 개선하는 흥미로운 결과이나, 이식하려면 Prime ECC의 단일 QC-LDPC 부호구조 자체를 2차원 서브코드 체계로 전면 재설계해야 해 이식 난이도가 매우 높음(§4 "높음" 레이어).
