### B. 알고리즘 요약

1. Polar code (N,K)=(1024, 488+24 CRC)를 systematic encoding 후 pruning하여 LDPC-like sparse factor graph(SFG, N'=2335, M'=1823, CRC 통합 시 M'=1847)로 변환해 BP(SPA/offset min-sum)로 복호.
2. 기존 LDPC 스케줄링(Flooding/LBP/RBP/NWRBP/VCRBP)을 그대로 적용하면 node-wise IDS가 일반 LDPC와 달리 오히려 SSS(LBP)보다 성능이 나쁜 현상을 발견 - SFG에 4-cycle 없이 다수의 6-cycle 존재, 다수의 hidden VN(VNH, 채널 LLR 없음)이 원인.
3. VNH가 0-값 V2C 메시지를 계속 전파하는 "zero propagation" 문제를 분석하고, 이를 반영해 CN을 매 반복 1회만 갱신하도록 제약한 NWRBPI/VCRBPI를 제시.
4. 제안 알고리즘 LBP-PUS: C2V 잔차 `r(m)=|f(m)-m|`의 CN별 최대값 `r_ci,max`가 임계값 `μ` 초과인 CN 집합 Q를 잔차 내림차순 정렬해 LBP 방식으로 일괄 갱신 (Algorithm 1).
5. `μ=0`(M1)과 `μ=μ1`(비영값 rci,max 평균, M2)을 비교 - M2가 수렴속도·오류성능 모두 개선, LBP·Flooding 대비 BER 10⁻⁵에서 약 0.156 dB 이득.
6. Shuffled LBP-PUS는 V2C 잔차 `r̃_ci,max` 기준으로 Q를 정렬(Algorithm 2)하는 변형.
7. CRC 검사식을 추가 CN으로 SFG에 통합(integrated SFG)해 조기종료 기준을 강화하되, 수렴 저하를 막기 위해 매 Q 생성마다 CRC CN은 최고 잔차값 1개만 포함하도록 수정 - 원 SFG 대비 BER 10⁻⁵에서 0.245~0.375 dB 이득.
8. 정렬 대신 무작위 갱신 순서(Random update)를 도입해 하드웨어 정렬 비용을 낮추고, schedule diversity(매 디코딩 시도마다 다른 갱신 순서) + bit-flipping(ω개 최소 LLR 노드 값 반전) 결합(Algorithm 3)으로 최대 Dmax회 재시도.
9. 결과: Dmax=3000일 때 CA-SCL(L=32)과 comparable FER 달성, CN 업데이트 수 기준 계산복잡도는 CA-SCL(L=32) 대비 낮음. 5G 표준 코드(R=501/1024) 대비 FastSOL/Soft List보다 FER 10⁻⁴에서 약 0.1 dB 이득.
10. 한계: 대상이 polar code이며 SPA를 offset min-sum으로 근사(오프셋 값은 SNR·CN degree별 조정, 구체 수식 본문 미기재), 전부 시뮬레이션 결과이며 HW 구현·gatecount·throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LBP-PUS/Shuffled LBP-PUS 스케줄링(잔차 기반 CN 선택 후 일괄 갱신) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 반복 스케줄 개선 아이디어로 참고 가능하나 Dual-Update(odd/even 교대)와는 다른 방식이라 재설계 필요 |
| offset min-sum 근사(SPA 대체) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min-sum 채택 - 중복 기여 |
| CRC를 SFG에 CN으로 통합해 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 이미 Partial/Full CRC 조기종료 보유 - CRC를 파리티 그래프에 CN으로 편입하는 방식은 새로우나 QC-LDPC 구조와 결합 難 |
| Random update / schedule diversity + bit-flipping 재시도 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 대응 없음(현재 코드는 다중 attempt 재시도 구조 아님) - 참고용 아이디어 |
| Polar-to-LDPC-like SFG 변환/pruning, VNH 개념 | 대응 없음 | polar code 고유 구조로 QC-LDPC와 무관 |

> 적용 가치: 낮음 — 대상 부호가 polar code이고 부호설계·HW 이식 요소가 없으며, 스케줄링 아이디어도 Prime ECC가 이미 보유한 min-sum/Dual-Update/조기종료와 상당 부분 중복.
