> 총평: Staircase code(BCH 기반) 전용 weakly-hybrid HD/SD 디코딩 논문으로, LDPC 부호·min-sum 구조와는 직접 관련이 없으나 "채널 LLR을 1회만 사용해 신뢰도 마킹 후 HD 디코더의 오검출 방지·비트플리핑에 활용"하는 저비용 soft-aided 개념 자체는 일반적 참고 가치가 있어 이식성은 중간 수준으로 평가.

### B. 알고리즘 요약

1. Staircase code(SCC, BCH 기반 component code, 대역폭 W×W 블록의 sliding-window BDD 반복복호)를 2/8-PAM + AWGN 채널에서 다룸, 부호율 `R=1-U/W=2kc/nc-1` (예제: BCH(256,239,2) R≈0.934, BCH(254,230,3) R=0.811).
2. 풀려는 문제: SD 디코딩(LDPC/polar 등)은 큰 이득을 주지만 복잡도·전력 부담이 크고, 순수 HD-BDD는 저복잡도지만 HD Shannon limit과 큰 성능 격차 - 기존 SABM/iSABM 계열보다 더 나은 저비용 hybrid 필요.
3. 채널 모델: 수신 LLR `λ_k`를 식 (1)/(2)로 계산(2-PAM: `λ=2γy`), BDD 출력은 성공/오검출(miscorrection)/실패 3가지로 구분(`d(r,c)≤tc` 등 식 (5)).
4. 핵심 기법: MLBM(Multi-Level Bit Marking) - 채널 수신 시 LLR 절대값 `|λ_k|`을 임계값 `δ1>δ2>δ3`으로 비교해 4단계(HRB/RB/UB/HUB, 식 (6))로 마킹 후 LLR 자체는 폐기(저장 불필요) - "weakly-hybrid"의 핵심.
5. 마킹된 HRB로 BDD 성공 결과의 오검출을 탐지(검출된 오류위치가 HRB와 충돌하면 오검출로 판정, Algorithm 1)하고, 오검출/실패 시 HUB→UB 순 우선순위(R0⊆R1⊆R2, Algorithm 2)로 비트플리핑 후 BDD 재시도.
6. 부수 기법: 디코딩 상태(성공/오검출/실패) 및 인접 블록 상태에 따라 매 라운드 마크비트를 갱신(Algorithm 3, MBU)해 반복 진행 중 마킹 정확도를 유지 - GMDD와 달리 오류 개수를 BDD 실패 여부로 미리 알고 있어 추가 BDD 1회만으로 대부분 처리.
7. 파라미터 최적화: 임계값 δ1/δ2/δ3, 위치임계값 K_MD, HRB 신뢰한도 T_HRB, 반복별 플립비트수 f_failure를 수치 시뮬레이션 + LLR PDF 기반 이론적 예측(식 (7)(8))으로 도출, 이론값과 시뮬레이션값이 거의 일치.
8. 학습/최적화 절차: Density-evolution류 학습은 없고 그리드서치식 수치 최적화 + closed-form 이론 예측 병행.
9. 결과: BCH(256,239,2)/2-PAM에서 iterative BDD 대비 0.97 dB, iSABM 대비 0.31 dB 이득(BER 10⁻⁷); 8-PAM에서는 1.22 dB(HD-BDD 대비); 대가로 메모리 약 2.3배 증가, 추가 BDD로 인한 RCI(상대 복잡도 증가)는 SNR 6.3dB에서 약 31%(부분BDD 포함) ~ 80%+(완전BDD만 기준).
10. 한계: FPGA/ASIC 구현 없음(향후 과제로 명시), 전부 시뮬레이션이며 대상이 BCH 기반 staircase/product code이지 LDPC가 아님, HUB 마킹 정확도가 40% 미만으로 비트플리핑 효율 제한적임을 저자가 스스로 지적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LLR 1회 사용 후 폐기하는 다단계 신뢰도 마킹(HRB/RB/UB/HUB) | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | Prime ECC의 iteration별 LLR threshold 테이블(적응형 LLR)과 개념적으로 유사하나, 목적이 "오검출 탐지+비트플리핑"용 정적 1회 마킹이라는 점이 다름 |
| 우선순위 기반 비트플리핑(오검출/실패에 따라 R0/R1/R2 차등) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 대응 없음(현재 min-sum 반복복호는 별도의 bit-flip post-processing 단계를 갖지 않음) - trapping-set 대응 확장 시 참고 가능 |
| Marked Bit Update(반복마다 신뢰도 등급 갱신) | `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 테이블 인덱스 갱신과 유사한 반복적 상태갱신 개념이나 BDD 성공/실패 상태 기반이라 min-sum 메시지 갱신과 메커니즘이 다름 |
| BDD 기반 component decoding, sliding window 구조 | 대응 없음 | Prime ECC는 QC-LDPC 전체를 한 코드워드로 병렬 처리, BCH 기반 product/staircase 구조 자체가 대응 없음 |
| 메모리 오버헤드 분석(2-bit 마킹 + 상태행렬) | `ecc_data.h` (PCM/PARAM_DEC 구조체) | SRAM 예산 고려 관점만 참고, 구체 구조는 무관 |

> 적용 가치: 중간 — 대상 부호(BCH 기반 staircase code, BDD 반복복호)는 Prime ECC의 QC-LDPC min-sum 구조와 이질적이라 직접 이식 불가하지만, "채널 soft 정보를 1회만 사용해 저비용으로 HD 계열 디코더 성능을 개선"하는 weakly-hybrid 설계 철학과 우선순위 기반 비트플리핑 아이디어는 향후 min-sum 디코더의 error-floor/trapping-set 대응 post-processing 설계에 참고 가치가 있음.
