### B. 알고리즘 요약

1. 시스템: SSD QC-LDPC (N=36864, M=4096, q=512), column weight 4, rate 0.89, 4KiB 정보비트+512B 패리티.
2. 문제: 기존 QC-LDPC 구성(GF 기반, masking 기반)이 girth/cycle/trapping set을 동시 최적화하지 못해 대용량 SSD에서 신뢰성 부족.
3. 핵심 가정: exponent matrix `A`(크기 M/q × N/q, 원소 `[0,q-1]` 또는 `∞`)로 QC-LDPC를 표현, `∞`는 Tanner graph edge 제거(masking).
4. 핵심 기법 1단: 컬럼 단위 탐색으로 girth `g`, g-cycle 수 `rg`, chord 수 `tg`를 동시 개선하는 exponent matrix 최적화(Step1 초기화 → Step2 컬럼별 masking enumeration/column optimization 반복).
5. 우선순위: girth 최우선, 그 다음 cycle-first 또는 chord-first 정책으로 `rg`/`tg` 비교해 더 나은 컬럼 채택.
6. 핵심 기법 2단: smart decoding — hard decoding latency `th(RBER)=tRh+tDMAh+tECCh`, soft decoding latency `ts(RBER)`, 기대 latency `T=th+P(RBER)×ts`를 비교해 크로스포인트 이하에서만 hard decoding 수행.
7. 구현: MQSim에 error injector(published error model + AWGN), encoder/decoder(HD=Gradient Descent Bit-Flipping, SD=min-sum) 통합, 3.9K LoC.
8. 최적화 절차: 탐색 기반(greedy column-wise search), 별도 DE/GA/NN 없음.
9. 결과: girth 8 달성(기존 6 대비), decoding failure rate 최대 93% 감소, read response latency 최대 47.5% 감소.
10. 한계: 탐색 시간 330s(오프라인 전용), HW 미설계, MQSim 시뮬레이션 기반 검증뿐, RBER 0.0045(hard)/0.015(soft) 이상에서는 이득 미미.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| exponent matrix masking 탐색(girth/cycle/chord 최적화) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 바꾸는 구성법이라 고정 QC-LDPC를 쓰는 Prime ECC와는 재검증 부담이 큰 영역 |
| smart decoding(hard decoding 선택적 스킵) | `decoder.cpp` `LDPC_Decoder()` | 반복 루프 진입 여부를 RBER 기반으로 결정하는 정책이며 조기종료(`partial_CRC.cpp`)와는 별개의 상위 레벨 스킵 로직 |
| RBER 기반 latency 모델/크로스포인트 산출 | `channel.cpp` `Set_R_Offset()` | RBER 추정치를 활용하는 정책이라 채널 모델과 연계 가능 |
| hard decoding(Gradient Descent Bit-Flipping) | `decoder.cpp` `VN_Cal_HD()` | 이미 보유한 HD 알고리즘과 다른 계열(GDBF)이나 본 논문의 핵심 기여는 아님 |

적용 가치: 중간 — masking 탐색 기반 부호구성은 이식 난이도가 높지만, smart decoding(RBER 기반 hard-decoding skip) 정책은 별도 모듈로 상대적으로 낮은 난이도로 적용 검토 가능.
