### B. 알고리즘 요약

1. 시스템: overloaded SCMA(사용자 수 J > 자원 수 K, 예 4×6, 12×18), 각 사용자 LDPC(rate 1/2, length 2400)로 부호화 후 SCMA 심볼로 매핑, IDD(반복 검출·복호) 수신기 사용.
2. 문제: 기존 IDD는 한 outer iteration 내에서 사용자 디코더가 병렬 동작해 서로의 복호결과를 공유하지 못하고, 이미 복호 성공한 사용자도 계속 처리되어 연산 낭비.
3. 핵심 가정: AWGN/Rayleigh/TDL-B 페이딩 채널, MPA(message passing algorithm) 기반 SCMA 검출기 + LDPC BP 디코더 조합.
4. 핵심 기법 1: R-SDD(Ranked Successive Detection and Decoding) — 사용자를 순차적으로 검출·복호하며, 앞선 사용자의 복호결과를 뒤 사용자가 즉시 활용. 평균상호정보량 `AMI_p,j = E[f_I(L_p,j)]`로 사용자 신뢰도를 매 outer iteration마다 재계산해 신뢰도 낮은 사용자를 뒤로 재배치.
5. 핵심 식: `AMI_p,j`가 a-posteriori LLR의 절댓값에만 의존(부호 비트값 불필요)하므로 실제 송신 비트 없이도 신뢰도 추정 가능 — 이 성질로 온라인 재정렬이 가능해짐.
6. 확장 기법 2: FG(factor-graph) 인터리버 — VN degree 대신 AMI로 VN 신뢰도를 평가해, 서로 다른 신뢰도의 VN이 같은 SCMA 심볼에 매핑되도록 그룹별 cyclic shift 인터리빙.
7. 부수 트릭: MPA 검출기에 LLR 변화량 기반 조기종료(`τ_MPA=1`)를 적용해 검출 복잡도 절감.
8. 최적화 절차: modified P-EXIT(outer iteration까지 포함한 MI 추적)로 protograph 결정 임계값을 계산, 이를 기반으로 exhaustive search로 MU-1(2×4), MU-2(3×6) base matrix 설계 — precoding 구조 배제, degree-2 VN 비율 50%까지 허용, VN마다 다른 edge distribution 부여.
9. 결과: AWGN 채널 BER 1e-5에서 R-SDD가 기존 IDD 대비 0.30~0.55dB 이득(Tdec별), MU-2 코드가 기존 최고 대비 0.3dB 이상 개선, 동일 Tdec에서 tanh 연산 감소(반복수 감소로 인한 복잡도 저감).
10. 한계: 순차(SIC형) 구조로 인해 latency가 오히려 증가함을 저자가 인정(§V), 코드 설계 원칙이 다중사용자 outer-iteration 존재를 전제로 하며 HW 구현·합성 없이 시뮬레이션(AWGN/Rayleigh/TDL-B)에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| R-SDD 순차 검출·복호 스케줄링 | 대응 없음 | 다중사용자 MUD/SCMA 검출기 자체가 Prime ECC(단일채널 NAND ECC)에 존재하지 않음 |
| protograph 설계 원칙(precoding 회피, degree-2 비율↑, edge distribution 다양화) | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 지점과 대응되나, SCMA outer-iteration 전제 설계라 NAND 단일 디코딩 루프에 그대로 적용 시 error floor 재현 우려 |
| MPA 조기종료(LLR 변화량 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 조기종료 아이디어는 유사하나, SCMA 검출기(MPA) 전용이라 LDPC 디코더 자체의 조기종료(CRC)와는 메커니즘 다름 |
| BP 디코더(tanh 기반) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum(Min1/Min2) 기반이라 tanh 전면 사용 Sum-Product와는 이미 다른 알고리즘 |

> 적용 가치: 낮음 — 핵심 기여(R-SDD, FG 인터리버)가 다중사용자 SCMA 반복수신 구조에 종속되어 NAND 단일채널 ECC에 대응점이 없고, protograph 설계 원칙도 outer-iteration 전제가 달라 직접 이식 시 위험.
