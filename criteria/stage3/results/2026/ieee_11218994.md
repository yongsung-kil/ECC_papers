### B. 알고리즘 요약

1. 시스템: 2-user Gaussian Multiple Access Channel(GMAC), `y = P1*x[1] + P2*x[2] + z`로 두 사용자 신호가 중첩 수신됨.
2. 문제: 채널 품질이 다른 두 사용자에게 rate-diverse 부호(H1, H2 = [H1; He])를 할당하는 RDJD(rate-diverse joint decoding)에 대한 부호 설계 연구가 전무했음.
3. 채널/모델 가정: 강한 사용자 1은 rate `R1` (PCM `H1`), 약한 사용자 2는 rate `R2` (PCM `H2` = H1 위에 잔차행렬 He 추가), AWGN 및 quasi-static Rayleigh fading 채널.
4. 핵심 기법: joint user decoder(JUD, 공통행렬 Hc로 joint codeword `v[1,2]` 복호)와 residual user decoder(RUD, He로 `v[2]` 복호)가 반복적으로 정보를 교환.
5. 핵심 식: RDJP-EXIT은 non-binary(F2^2) PEXIT 분석을 확장해 JU/RU 사이 MI(`Iv[es]`, `Is[cv]` 등)를 반복 갱신하며 decoding threshold를 예측.
6. 확장: 대칭 채널 구현을 위해 랜덤 시퀀스 `t1, t2`(±1)를 곱해 all-zero codeword 가정 하 threshold 추정을 단순화.
7. 구현 디테일: base matrix 원소 최대값을 3으로 제한(탐색 단순화), 예외적으로 degree-2 다수 + 1개 large-degree VN 허용, PEG 알고리즘으로 목표 길이까지 lifting.
8. 최적화 절차: RDJP-EXIT 기반 decoding threshold 최소화 기준으로 base matrix를 탐색(설계 원칙 4단계: check/variable node 수 결정 → degree-2 비율/large-degree VN 포함 → 나머지 degree≥3 → threshold 최저값 선택).
9. 결과: AR4JA 대비 0.8~1dB, irregular LDPC 대비 0.4dB 이득(BER 기준); capacity 대비 0.062~0.074 bits/channel 이내로 근접.
10. 한계: 무선 다중접속(GMAC) 채널 전용 설계이며 HW 구현 없이 시뮬레이션 결과만 제시, non-binary(F2^2) 기반 분석이라 단일 사용자 binary LDPC ECC로의 직접 적용 불가.

| JUD/RUD 반복 복호 구조 | 대응 없음 | 단일 사용자 binary LDPC 디코더(`decoder.cpp`)와 무관한 2-user 복호 프레임워크 |
| non-binary PEXIT / F2^2 메시지 모델 | 대응 없음 | Prime ECC는 binary LDPC 전용, non-binary 기법은 이식성 낮음 |
