### B. 알고리즘 요약 (10줄 내외)

1. 6G URLLC 환경에서 채널 손상이 burst 형태로 발생하며 LDPC BP 디코더가 burst length가 코드 girth/lifting factor를 초과하면 성능이 열화됨을 지적하는 서베이 논문.
2. 기존 서베이들이 코드 구조, 학습기반 디코딩, 하드웨어 양자화를 개별적으로 다뤄온 것과 달리 이를 하나의 프레임워크로 통합.
3. 채널 모델은 Gilbert-Elliott(GE) Markov, SHMM(PLC 임펄스), Fisher-Snedecor fading 등을 burst 통계 추상화로 채택.
4. LDPC 코드 계열 3종(SC-LDPC, QC-LDPC, protograph-LDPC) 비교: SC-LDPC는 sliding-window decoding으로 wave-like 복호, `Bmax` 100~150 symbol, BCR>0.90(Lb≤80); QC-LDPC는 회로 라우팅 효율은 높으나 `Bmax` 60~80, BCR 0.70~0.80로 상대적으로 취약(circulant 위상정렬 burst에 취약).
5. `Bmax = max{Lb : BLER(Lb) < 1e-5}` (burst coverage 평가지표), `M ∈ [0,1]^Nv`(burst 영향 soft mask)가 핵심 정의 수식.
6. 신경망 보조 모듈: CNN/Transformer로 burst 영향 영역 마스크 생성 → 선택적 damping/재복호, RNN(GRU/LSTM)으로 syndrome 기반 반복 스케줄링 조정, RL/GNN으로 interleaver·lifting 구조 최적화.
7. 하드웨어 배치를 위해 quantization-aware training(QAT) + mixed-precision(예: 6-bit extrinsic/4-bit intrinsic)을 적용해 FPGA/ASIC에서 저전력·저면적 구현.
8. 별도의 학습/최적화 절차는 논문 자체 수행이 아니라 인용 문헌들의 방법(QAT, RL, GA 등)을 정리한 것.
9. 결과는 인용 문헌 재인용 수치: QC-LDPC FPGA 2.1Gbps(Deng et al.), ASIC 4.5Gbps(Yang et al.); QAT 6-bit에서 BCR ≥95% 유지.
10. 한계: 이 논문 자체는 실험/제안 알고리즘이 없는 리뷰이며, 표준화된 burst 데이터셋 부재, 미지 burst 분포 일반화 한계, 코드-신경망 공동설계 부재, 양자화 무시로 인한 비현실적 전력 추정 등을 향후 과제로 지적.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Deep-learning 기반 burst 검출/스케줄링(CNN/RNN/RL) | 대응 없음 | Prime ECC는 NN 기반 보조모듈 미보유, binary QC-LDPC min-sum 시뮬레이터로 구조 상이 |
| SC-LDPC/protograph 코드 구조 비교 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정, SC-LDPC/protograph 미지원 |
| QC-LDPC 회로 기반 계층복호 | `decoder.cpp` `LDPC_Decoder()` | QC-LDPC 회로 구조는 Prime ECC와 유사하나 논문은 서베이라 구체 알고리즘 이식 요소 없음 |
| QAT/mixed-precision 양자화 | `channel.cpp` + `PARAM_LLR` | LLR 양자화 개념은 유사 영역이나 논문은 NN 가중치 양자화 중심으로 직접 대응 어려움 |
```

적용 가치: 낮음 — 본 논문은 6G 무선 burst 채널 대상 리뷰로, 자체 알고리즘·수치 검증이 없어 NAND binary QC-LDPC 이식 관점에서 실질 기여가 없음.
