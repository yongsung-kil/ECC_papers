### B. 알고리즘 요약

1. 3D NAND 채널, QC-LDPC 코드 `n=18656, R=0.9`, min-sum 알고리즘(MSA), 최대 반복 30회, 최대 9회 read-retry로 소프트 LLR 획득.
2. 문제: 기존 "1회 하드 디코딩 + 다수 소프트 재시도" 고정 파이프라인은 저/중 RBER 상황에서도 불필요한 read-retry를 유발해 latency를 악화시킴.
3. 채널모델: Gaussian Mixture Model 기반 임계전압 분포, P/E cycle(1k/3k/5k)·retention(6/9개월) 별 RBER 10⁻⁴~2×10⁻² 시뮬레이션.
4. 핵심 기법: 첫 하드디코딩 직후 계산되는 syndrome sum `SS = wh(HyT mod 2)` (Hamming weight)를 채널 상태(RBER) 대리지표로 사용.
5. `SS ≤ TL`이면 E-Mode(하드 디코딩만), `TL < SS ≤ TH`이면 B-Mode(2차 하드 디코딩 재시도), `SS > TH`이면 곧바로 P-Mode(소프트 디코딩)로 진입 — 불필요한 중간 단계 생략.
6. 임계값 `TL=280, TH=350`은 오프라인 실측(2300 프레임)으로 SS-RBER 상관곡선을 얻어 결정 (RBER 5.61×10⁻³/7.67×10⁻³ 지점 대응).
7. HW 오버헤드 최소화: SS는 기존 디코더의 조기종료 검사용 Hamming weight adder-tree를 재사용, 추가로 TL/TH 저장 레지스터 2개와 소규모 제어 로직만 필요.
8. 별도 학습/최적화 절차 없음 (임계값은 통계적 오프라인 튜닝).
9. 결과: RBER<1.4×10⁻²에서 기존 대비 read-retry 33.3% 감소, 고RBER에서 11.1~20% 감소, 기존 방식 대비 24.4%, DHD 대비 16.1% retry 감소.
10. 한계: 순수 시뮬레이션(FPGA/ASIC 미검증), 정정능력(에러플로어/waterfall) 자체 개선은 없고 read-retry 스케줄링 최적화에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Syndrome sum 기반 모드 선택(E/B/P-Mode) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료용 syndrome/CRC 계산 로직과 유사 레이어, read-retry 제어 확장으로 이식 가능 |
| Hamming weight adder-tree (SS 계산) | `decoder.cpp` `LDPC_Decoder()` 반복 루프 내 종료판정 | 기존 조기종료 판정 로직에 부가 가능 |
| 적응형 LLR/재시도 임계값(TL, TH) | `ecc_data.h` `PARAM_LLR`, `channel.cpp` `Set_LLR_Th()` | iteration별 LLR threshold 테이블과 유사한 적응형 임계값 설계 |
| 소프트 디코딩 read-retry 트리거 로직 | `channel.cpp` `Set_R_Offset()` | soft-read 전환 시점 결정 로직에 대응 |
| Min-sum 알고리즘 자체 | `decoder.cpp` `CNU_Update_New_Mag()` | 표준 MSA 그대로 사용, 변형 없음(중복 기여, 이식 가치 낮음) |

> 적용 가치: 중간 — read-retry/모드 선택 제어 로직은 Prime ECC의 조기종료·적응형 LLR 레이어에 낮은 난이도로 이식 가능하나, 디코더 자체 성능(정정능력) 개선은 없어 가치가 제한적임.
