### B. 알고리즘 요약

1. 시스템은 AWGN 채널에서 PAS 기반 CCDM + LDPC 부호화 변조(예: 16-QAM)이며, 예제 부호는 (576,288) rate-0.5 WiMax LDPC와 (272,191) rate-0.7 FG-LDPC.
2. 짧은 LDPC 부호는 trapping set으로 인한 error-floor가 발생하고, shaping decoder는 오류정정 능력이 없어 LDPC decoder 오류가 심각한 오류 전파를 일으킴.
3. 원 디코더 실패는 최소 UCN(unsatisfied check node) 집합 `T0`로 식별하며, `|T0|=0`인 near-codeword는 CCDM codeword의 weight constraint(`ωs`) 위반 여부로 추가 검출.
4. 핵심 기법: 각 UCN을 이중 확장(4-layer tree)하여 신뢰도가 낮은 인접 VN들을 후보 집합 `Us1`로 구성(`Us1=∪(Vi(2)∪Vi(4))`); `T0=0`이면 전체 LLR을 정렬해 하위 `Nmax1`개 VN을 `Us2`로 선택.
5. 후보 VN의 하드결정 비트 `us`와 신뢰도 오름차순 정렬 벡터 `Rs`를 이용해, 개별 비트 revise(`N1`개)와 쌍(pair) 단위 revise(`N2`개)로 candidate input message vector(CIMV) `Rc(i)`를 생성.
6. 개별 revise 식: `Rc(i,ψi) = -1·sgn(Rψi)·|Lmax|·η1`; 쌍 revise 식: `Rc(i,αj) = sgn(Rαj)·|Lmax|·η2` (η2>η1로 과추정/의사코드워드 방지 및 수렴 가속).
7. 원 복호·재복호 모두 min-sum algorithm(MSA) 사용, 생성된 N1+N2개 CIMV를 순차 재복호하며 유효 코드워드(및 CCDM weight 만족) 발견 시 종료.
8. 파라미터는 실험적으로 고정: τ=10, Nmax1=15, Nmax2=35, η1=1.5, η2=1.75, Imax1=50, Imax2=20(재복호당).
9. 결과: C1(576,288)에서 FER 10^-6 기준 1st-BTD 대비 0.27dB, BP-LED 대비 0.55dB 개선, waterfall 영역에서는 MSA/SPA와 평균 iteration 동일; Table I에서 1st-BTD 대비 Navg/ΔIavg/runtime 모두 낮음.
10. 한계: CCDM/PAS 결합 시스템 전제(순수 binary LDPC 단독 적용 시 weight constraint 기반 근접코드워드 검출 불가), MATLAB 시뮬레이션만 수행, HW 설계·게이트카운트 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MSA 기반 원 복호/재복호 루프 | `decoder.cpp` `LDPC_Decoder()` | 재복호를 위한 반복 루프 재호출 구조와 유사 |
| UCN(unsatisfied check node) 식별 | `decoder.cpp` (parity check 판정 로직, 명시적 함수명 없음) | UCN 집합 계산은 기존 조기종료용 신드롬 체크와 결이 다름 - 대응 없음 |
| LLR 국소 revise(post-processing) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 특정 VN LLR을 강제로 flip/scale하는 방식은 기존 iteration별 LLR 테이블과 다른 후처리 단계 - 부분 관련 |
| CCDM weight constraint 검출 | 대응 없음 | Prime ECC는 CCDM/PAS 미지원, NAND 시스템에는 해당 개념 없음 |
| 조기종료(재복호 성공 시 exit) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 유효 코드워드 판정 후 종료라는 개념적 유사성만 있음 |

> 적용 가치: 낮음 | CCDM/PAS 고차변조 시스템에 강하게 결합된 post-processing이며, UCN 이중확장 자체는 순수 binary LDPC에 이식 가능하나 근접코드워드 검출(핵심 기여 중 하나)은 CCDM 없이는 성립하지 않고 재복호 오버헤드로 latency 악화가 우려됨.
