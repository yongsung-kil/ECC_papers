### B. 알고리즘 요약

1. Surface/color code(quantum stabilizer code, planar surface `[[2d²-2d+1,1,d]]`, distance d)를 Tanner graph로 표현, X/Z-type 두 종류 stabilizer로 구성된 채널에서 depolarizing noise 하 디코딩 문제를 다룸.
2. 기존 BP-OSD는 근최적 논리오류율을 내지만 OSD의 Gaussian Elimination(GE)이 `O(n³)` 시간·`O(n²)` 메모리에 직렬적이라 low-latency 양자오류정정에 부적합.
3. NMS 변형의 quantum BP를 전처리로 사용, 정규화계수 `α`로 check-to-variable 메시지 `μ_j→i = (-1)^sj α sgn(...) min(...)`를 계산해 사후 LLR을 산출(식 (해당 절 수식)).
4. 핵심 기법: GE 기반 OSD를 GE-free `LC-GCD`(Locally-Constrained Guessing Codeword Decoding)로 교체 - 고정된 systematic 기저에서 신뢰도 상위 `δ`개 위치 집합 A로 trellis를 구성해 SLVA(List Viterbi)로 TEP를 탐색.
5. `BP-LCGCD+α`: 단일 α 대신 리스트 `{α_t}_{t=1}^L`로 다중 BP를 수행해 서로 다른 사후 LLR 후보를 생성하고, Phase I(단일 α0 BP) 실패 시 Phase II(다중 α BP + LC-GCD)로 진행하는 2단계 구조(Algorithm 1).
6. 복잡도: 평균 복잡도 `O(nT1,avg) + η1[O(LnT2,avg) + η2 O(L'(ℓavg k n + 2δ(k+δ)))]`, low physical error rate(η1≈10⁻⁴)에서는 Phase I(순수 BP) 비용에 근접.
7. 부수 디테일: HeTnc(인코딩 행렬)로 candidate 재구성, 최소 weight candidate를 최종 추정치로 채택; classical LC-GCD를 quantum 이진 syndrome 형식(H=[Hx|Hz])으로 확장.
8. 최적화 절차 없음(학습/DE/GA 미사용), 파라미터는 δ=8, L=16, L'=8로 실험적 설정.
9. 결과: surface code `[[145,1,9]]`에서 BP-LCGCD가 BP·MWPM 대비 크게 우수, BP-LCGCD+α는 BP-OSD와 거의 동일한 논리오류율·threshold(약 18%) 달성. 6.6.6 hexagonal color code(d=5,7,9)에서도 BP-OSD 대비 comparable~약간 우수.
10. 한계: 전부 Monte Carlo 시뮬레이션(HW 미설계, gatecount/throughput 없음), worst-case 복잡도는 code dimension k에 지수적으로 커져 고rate QLDPC에는 비현실적이라고 저자가 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GE-free LC-GCD 후처리(로컬제약 TEP 탐색) | 대응 없음 | Prime ECC는 OSD/GE 계열 후처리를 사용하지 않음(min-sum 반복복호 + CRC 조기종료 구조) |
| NMS 기반 quantum BP(정규화계수 α, C2V/V2C 갱신) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 형식적으로 min-sum 계열이나 X/Y/Z Pauli 3종 LLR과 syndrome 부호(`(-1)^sj`) 처리로 binary LDPC와 메시지 구조 자체가 다름 |
| 리스트 정규화계수(α list) 다중 BP 실행 | 대응 없음 | Prime ECC는 단일 스케일/오프셋 LLR 테이블(`PARAM_LLR`) 사용, 다중 α 병렬 시도 구조 없음 |
| 복잡도 분석(Phase I/II 평균 비용 모델) | 대응 없음 | 참고용 방법론일 뿐 코드 이식 대상 아님 |

> 적용 가치: 낮음 — quantum stabilizer code 전용 디코더로 부호구조(Pauli X/Y/Z, stabilizer matrix)와 후처리 방식(OSD/LC-GCD)이 NAND binary QC-LDPC의 min-sum 반복복호 프레임과 근본적으로 다르며, Prime ECC가 OSD 계열을 채택하고 있지 않아 직접 이식 지점이 없음.
