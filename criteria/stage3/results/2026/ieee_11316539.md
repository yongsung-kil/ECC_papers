### B. 알고리즘 요약

1. 시스템: DNA 기반 데이터 저장, 각 oligo(≈250 base)에 index bits(시퀀스 식별용) + data bits를 넣고 binary-to-DNA 변환(00→A, 01→C, 10→G, 11→T) 후 합성/저장/PCR/샘플링/시퀀싱을 거쳐 복원.
2. 문제: 기존 2-D(inter/intra-oligo) 코딩은 index bits와 data bits를 단일 코드(single-layer)로 함께 보호해 index 오류가 잘못된 클러스터링을 유발, 이를 위한 index/data 최적 redundancy 배분 방법이 부재했음.
3. 채널 모델: base-level 오류(삽입/삭제/치환, 확률 `Ps=0.57Pe`)와 sequence-level 오류(카피 손실, 확률 `Pl=0.43Pe`)로 구분, DESP 시뮬레이션 파이프라인으로 모델링.
4. 핵심 기법: two-layer BCH(TL-BCH) intra-oligo 코딩 — index `zi`를 BCH `C1`(redundancy `r1`)로, data `vi`를 BCH `C2`(redundancy `r2`)로 별도 인코딩해 `wi=(ti, si)` 구성, outer는 5G LDPC `C0`로 inter-oligo(sequence-level) 오류 보호.
5. 핵심 식: 유한장 정규근사(normal approximation) `k2 = n2(1-h(δ)) - sqrt(n2*δ(1-δ))*log((1-δ)/δ)*Q^-1(ε2) + ...`로 data bits BER `ε2`를 예측, BCH 정정능력 `t1 ≈ r1/⌈log(n1+1)⌉`로 index FER `ε1` 예측.
6. 확장: 총 redundancy `r=r1+r2` 고정 시 mis-clustering이 majority voting에 미치는 영향까지 반영한 최종 intra-oligo FER `ε=1-(1-P̃s)^k2` 유도, 이를 최소화하는 `(r1,r2)` 배분을 탐색.
7. 구현 디테일: 클러스터 내 다수결(voting score) 산출 후 LLR로 변환해 outer 5G LDPC SPA 디코더 입력으로 사용, LLR은 threshold `T=5`로 클리핑.
8. 최적화 절차: 주어진 `r`에서 해석적 FER 식(Eq.16)을 이용해 `(r1,r2)` 격자 탐색으로 최적 배분 결정(예: r≈112일 때 (15,99) 최적).
9. 결과: 동일 총 redundancy·sequencing depth에서 two-layer(TL-BCH) 코딩이 single-layer(SL-BCH) 대비 concatenated 코드 FER을 유의하게 낮춤(그림14), 특히 높은 sequencing depth에서 이득 증가.
10. 한계: HW 미설계, DESP 기반 시뮬레이션만 수행, outer LDPC는 표준 5G NR BG1 코드를 그대로 사용(자체 부호설계·디코더 개선 없음), DNA 특유의 IDS 채널·시퀀싱 깊이 기반 majority voting 구조에 특화되어 NAND read 채널과 이질적.

| TL-BCH index/data 분리 보호 | 대응 없음 | BCH 기반 intra-oligo 코딩으로 Prime ECC의 binary QC-LDPC 디코더/인코더 구조와 무관 |
| majority voting 기반 soft LLR 생성 | `channel.cpp` `Set_LLR_Th()` | 클러스터 다수결로 LLR을 만드는 방식이 개념적으로 soft-read LLR 양자화와 유사하나 DNA 특유 clustering 구조 의존이 커 직접 이식 어려움 |

적용 가치: **낮음** — outer LDPC는 표준 5G 규격을 그대로 사용하고 핵심 기여(TL-BCH index/data 분리, redundancy 최적화)는 DNA IDS 채널과 majority voting 구조에 특화되어 있어 NAND binary QC-LDPC ECC로 이식할 요소가 거의 없음.
