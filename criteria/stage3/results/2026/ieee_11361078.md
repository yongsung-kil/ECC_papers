### B. 알고리즘 요약

1. 시스템: QC-LDPC(CCSDS C2, 511×511 submatrix) 부호를 FPGA(Xilinx Virtex-7)에서 복호, BPSK 2Gbps 위성 광통신(FSO) 링크 대상.
2. 문제: 고차수(degree-32) check node의 min1/min2 탐색에 정렬망(sorting network) 기반 CNU가 과다한 comparator/MUX를 소모해 critical path와 자원 사용량을 늘림; 기존 병렬 아키텍처는 고정 블록 크기로 인한 파이프라인 정체와 낮은 BRAM 활용률 문제도 있음.
3. 핵심 가정: normalized min-sum(NMSA) 알고리즘, AWGN 채널 LLR 초기화 `L_v(0)=2y_i/σ²`.
4. 핵심 기법 1단: check node 입력 M개를 N개 그룹으로 분할해 그룹 내 로컬 최솟값을 독립 계산 후, 트리 비교로 전역 min1을, N개 로컬 최솟값 중 두 번째로 작은 값으로 min2를 근사(min2_approx) — 정확한 global min2 탐색을 생략.
5. 핵심 식/의미: 제안 방식은 `M+N-3` comparator·`M+N-4` MUX만 필요(기존 트리 소터는 `2M-3`·`3M-4`) — 그룹 수 N을 늘릴수록 comparator/MUX는 줄지만 근사 오차(ROP 열화)는 커지는 트레이드오프.
6. 핵심 기법 2단: CCSDS C2 서브매트릭스(511×511)에 맞춰 BRAM depth를 512로 설정해 511/512=99.8% 활용률 달성, 두 개 서브그룹으로 프레임을 병렬 저장해 CNU/VNU가 비동기 스케줄링(핸드셰이크 기반, CNU가 VNU보다 2cycle 느려 VNU에 2 wait cycle 삽입)으로 충돌 없이 접근.
7. 부수 디테일: 12-bit ADC 기준 총 워드길이 16비트 여유 내에서 fixed-point 폭 튜닝(fractional 1bit, 총 6bit `fix6_1`로 최종 채택), normalization factor `α=0.75`.
8. 최적화 절차: 그룹 수(2/4/8/16) 스윕으로 ROP 열화 대 자원절감 트레이드오프 실험 → 8그룹이 balance point로 채택(comparator 39%, MUX 52% 절감, 0.02dB 손실).
9. 결과: 350MHz 클럭, 10-iteration으로 peak throughput 2.87Gbps, 80.4 Mbps/kLUT(기존 [14] 대비 25% 향상), ROP=-53dBm@2Gbps BPSK. 단, latency는 9,447 cycle로 [14] 대비 명시적으로 더 높음(처리량 우선 설계).
10. 한계: 근사 min2로 인한 성능 열화(≤0.02dB)가 상존, CCSDS C2 특정 부호(degree-32, 511×511)에 최적화되어 다른 코드 구조로 일반화 시 그룹 파라미터 재튜닝 필요, latency 트레이드오프가 존재.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum 연산(정규화 min-sum) | decoder.cpp `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2 기반 min-sum CNU 사용, 그룹화+계층비교 근사 기법을 CNU 탐색 로직에 직접 적용 검토 가능 |
| 그룹화 최소값탐색(2S1/2S2/4S2 커스텀 소터) | decoder.cpp `CNU_Update_New_Mag()` 내부 comparator tree | degree 17(HCU 비대칭 col 포함) 구조에서도 comparator/MUX 절감 아이디어 이식 가능 - Prime ECC degree가 CCSDS C2(32)보다 작아 절감 폭은 재검증 필요 |
| 다중프레임 병렬 BRAM 관리(compact-packing, CNU/VNU 비동기 스케줄링) | 디코더 HW 모델(z=32, HCU, GT), decoder.cpp pipeline 6-stage | Prime ECC의 z_sb=32 row-병렬 구조와 메모리 정합 재설계 필요하나 BRAM 활용률 개선 아이디어는 직접 참고 가치 있음 |
| fixed-point LLR 폭 최적화(fix6_1) | channel.cpp LLR 테이블(`PARAM_LLR`) | Prime ECC의 HD/2SD/3SD LLR 양자화 설계와 유사한 워드폭 튜닝 방법론, 참고 가치 있음 |
| CCSDS C2 H-matrix(511×511, degree-32) 구조 | ecc_top.cpp `Load_PCM()` | 특정 위성통신 표준 부호 고정, Prime ECC 부호구조와 직접 대응 안 됨 |
```

적용 가치: **높음** — CNU 최소값탐색의 comparator/MUX 절감(그룹화+계층비교) 기법과 프레임병렬 BRAM 최적화 전략은 Prime ECC의 `decoder.cpp` CNU 로직과 메모리 구조에 직접 이식을 검토할 만한 HW 면적/자원 절감 기법임 (단, degree/코드 구조 차이로 파라미터 재튜닝 필요).
