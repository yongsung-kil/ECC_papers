### B. 알고리즘 요약

1. 시스템은 MLC NAND flash(셀당 2비트, MSB/LSB 각각 독립 LDPC 부호화, Gray 매핑)이며 예제 코드는 (4544,4096) irregular LDPC, rate≈0.9014, 디코더는 SPA(max iter 10).
2. 문제: 기존 학습 기반 read threshold 설계는 근사 LLR만 생성하거나(정수 레벨 매핑, posterior 평균 등) threshold 개수가 제한적(HDD 3개/SDD 6개)이라 유연한 SDD가 어려움.
3. 채널 모델: programming noise, data-retention noise(retention time `T`에 따라 평균 `µrs`·표준편차 `σrs` 변화), wear-out noise가 결합된 Gaussian mixture 전압분포를 학습데이터 생성용으로만 가정, 실제 추론 시 PDF 불필요.
4. 핵심 기법 1단: BiGRU(2-layer, LayerNorm+Dropout+FC) 기반 LLR 추정기. 참 LLR 레이블이 없으므로 `x=1-2u`(bipolar label)를 사용, 관측 상호정보량(OMI) `I(L̄;X)≈1-(1/N)Σlog2(1+e^{-x_n·L̄_n})`을 최대화하는 방향으로 손실함수 `L=(1/B)Σlog2(1+e^{-x_n·L_n})`를 최소화.
5. 이 손실은 채널 비대칭성을 반영한 MI 하한 추정치이며, 참 LLR 없이도 DNN 출력이 실제 LLR에 근접하도록 강제하는 것이 핵심(참 LLR ±10 이상에서는 근사 오차 존재하나 BER에 영향 미미).
6. 핵심 기법 2단: BiGRU 출력 LLR을 이용해 read threshold `{a1,...,a_{q-1}}`를 OMI 최대화로 설계. `Iobs` 최대화가 비용함수 `M(a_i,a_{i+1})=Σlog2(1+e^{-x·L̂i})` 최소화로 등가 변환되며, 구간별 대표 LLR `L̂i*`는 MSE 최소화로 구간 평균값으로 결정.
7. DP 최적화: 전압공간을 p개 구간(예 p=1000)으로 이산화 후 exhaustive search `O(p^{q-1})` 대신 DP로 `O(p^2 q)`에 최적 threshold 탐색, level 수 증가 시에도 복잡도가 `p^2`에 선형 스케일.
8. 학습 절차: 배치크기 100, 학습률 0.001, dropout 0.1, 500개 LDPC 코드워드 분량 전압 샘플로 학습하며 시스템 idle 시 또는 LDPC 디코딩 실패 시 주기적(예: 1주) 재학습으로 캘리브레이션 드리프트 추적.
9. 결과: q=7(MSB)/q=10(LSB) threshold에서 완전 채널지식 MMI와 유사한 MI 달성, (4544,4096) LDPC SPA 코드 BER도 MMI 근접; 비-Gaussian(Beta분포) 채널에서는 MMI보다 오히려 우수(MMI의 Gaussian 가정 붕괴 시에도 데이터기반이라 강건).
10. 한계: BiGRU 추론 자체는 블록 단위로 매 순간 수행하기엔 계산량이 커 "read threshold 설계"라는 오프라인 캘리브레이션 용도로만 사용, HW 구현·게이트카운트·throughput 실측 없음(저자도 향후 과제로 명시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BiGRU 기반 채널 무지식 LLR 추정 + OMI 최대화 read threshold 설계 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold/LLR 양자화 설계 레이어에 직접 대응, 기존 고정 threshold 테이블을 데이터기반 가변 설계로 대체 가능 |
| iteration별 LLR 테이블 | `ecc_data.h` `PARAM_LLR` | 본 논문의 구간별 대표 LLR `L̂i*` 계산 결과를 LLR 테이블 값으로 대체 이식 가능 |
| retention time에 따른 채널 파라미터 변화 대응(주기적 재학습) | `channel.cpp` (RBER/fixed-error 패턴 갱신 로직) | 개념적으로 유사하나 실시간 재학습 자체는 Prime ECC 시뮬레이터 범위 밖 |
| LDPC 디코딩 자체(SPA) | `decoder.cpp` `LDPC_Decoder()` | 본 논문은 디코더 알고리즘을 변경하지 않고 벤치마크로만 사용 |

> 적용 가치: 높음 | NAND flash 채널을 직접 타깃으로 하며 soft-read LLR/threshold 설계(Prime ECC의 `channel.cpp` LLR 테이블 대응 레이어)에 데이터기반 방식으로 채널 미지·비대칭·비-Gaussian 조건에서도 강건한 성능을 제공, 이식 난이도도 프로파일 §4에서 "낮음~중"으로 분류된 레이어에 해당.
