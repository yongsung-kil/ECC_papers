### B. 알고리즘 요약

1. 시스템은 BI-AWGN 채널에서 짧은 블록길이 코드(BCH, CRC-polar)를 IR-HARQ로 전송하는 URLLC 시나리오. 최대 `M`회 재전송, 각 라운드 코드율 `R_m=k/n_m`은 mother code 펑처링으로 생성.
2. 표준 HARQ는 매 라운드 전체(근사-ML, OSD) 디코딩 후 CRC로 성공/실패를 판정하는데, 이 전체 디코딩 자체가 계산 오버헤드와 지연을 유발해 URLLC 요구(지연<1ms, 신뢰도>99.9999%)에 부적합하다는 문제.
3. 핵심 가정: 디코딩 실패 사건의 단조성(`Prob{E_{m-1}|E_m}≈1`)을 이용해 PER/복잡도/처리량을 재귀식으로 근사(정규근사 기반 유한블록길이 PER `ε_m` 활용).
4. 핵심 기법(ET-HARQ): 디코딩 전에 "디코딩 가능 여부"를 예측하는 조기종료(ET) 단계를 추가, 예측이 "불가능"이면 전체 디코딩 없이 즉시 NACK. confusion matrix `α_m`(decodable을 정확히 판별), `β_m`(undecodable을 정확히 판별)으로 성능 모델링.
5. 핵심 식: PER `ζ_E`, 평균복잡도 `K_E`, 처리량 `η_E=k_I(1-ζ_E)/(K_E T_b)`를 α,β,CRC 오검출확률 `p_ud=2^{-k_C}`의 함수로 유도(Lemma 4/5). Selective ET-HARQ는 마지막 라운드만 항상 전체 디코딩(`α_M=1,β_M=0`)해 신뢰도 손실을 방지.
6. BP-NN 조기종료(실제 구현): 수신 LLR에 소수 iteration BP(LDPC 표준 메시지패싱, 식 23-25)를 적용해 LLR 궤적을 정제하고, 이 궤적의 1차/2차 모멘트+information density를 특징으로 NN(4-layer, 64 neuron, residual, ReLU)이 decodable/undecodable을 이진분류.
7. 부수 트릭: BP는 실제 디코딩이 아니라 NN 입력 특징 정제용으로만 사용(BCH는 Tanner graph에 short cycle이 많아 BP 자체 디코딩엔 부적합). NN은 15K 파라미터, 20KB/모델로 코드 길이별 개별 학습.
8. 학습: SNR별 개별 학습, decodable/undecodable 균형 데이터셋(SNR당 10^5 샘플), Adam(lr 1e-4) + early stopping.
9. 결과: (128,64) eBCH에서 예측정확도 70~80%, PB-OSD 대비 지연 51ms(0dB SNR, PB-OSD 단독 64-122ms 대비 개선이나 고SNR에서 5.5ms 고정 오버헤드로 역전 발생), selective ET-HARQ는 PER 손실 없이 최대 79% 처리량 이득.
10. 한계: HW(FPGA/ASIC) 미설계(가능성만 논의), 대상이 BCH/CRC-polar이며 LDPC는 BP 특징추출 배경 설명에만 등장(실제 디코딩 대상 아님), NN은 코드 길이·SNR별 개별 재학습 필요(확장성 제한을 GNN/meta-learning으로 향후 과제화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP iteration 궤적 기반 디코딩가능성 예측(조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적 유사성(조기 판별로 불필요한 반복/디코딩 회피)은 있으나, Prime ECC는 CRC 검사 기반이고 본 논문은 디코딩 이전 NN 예측이라 직접 대응 아님 |
| HARQ 재전송 스케줄링(선택적 재전송) | 대응 없음 | Prime ECC는 NAND 컨트롤러 내 ECC 디코더 시뮬레이터로 상위 재전송 프로토콜(ARQ/HARQ) 계층 미포함 |
| BCH/OSD/CRC-polar 디코딩 | 대응 없음 | Prime ECC는 binary QC-LDPC min-sum 전용, OSD/BCJR/SCL 등 타 부호군 디코더 미구현 |
| BP LLR 궤적 통계(NN 입력 특징) | `decoder.cpp` `LDPC_Decoder()` (LLR 관련) | LDPC BP 자체는 겹치나, 목적이 "디코딩 여부 예측"이라는 상이한 태스크로 이식 시 재설계 필요 |

> 적용 가치: 낮음 — HARQ 재전송 프로토콜 계층의 디코딩-전 조기종료 기법으로, BCH/CRC-polar 짧은 블록 부호를 대상으로 하며 NAND binary QC-LDPC ECC의 부호설계·디코더·HW 자체를 개선하지 않음.
