### B. 알고리즘 요약

1. 대상은 (n,k) 이진 선형 확장 cyclic code(eBCH 등), AWGN+BPSK 채널에서 OSD 계열 디코딩을 다룸(LDPC 아님).
2. 기존 DD-OSD(derivative decoding based on OSD)는 저차수 order로도 고차수 OSD 대비 성능이 좋으나, local MLD(l-MLD) 오류로 인해 디코딩이 특정 상태에 갇히는 "trap" 문제가 있음.
3. 핵심 가정: 원 코드의 cyclic derivative ascendant(cDA) `A(D(C))`에 속하지만 원 코드 `C`에는 속하지 않는 추정치로 수렴하면 trap 발생(Theorem 1).
4. 핵심 기법 1단: 패리티체크 `H`, `HA`로 `Hĉ^T=0` 및 `HA ĉ^T=0` 검사해 trap 여부를 검출.
5. 핵심 기법 2단: trap 검출 시 control band 내 **저신뢰도** 비트를 인자 `λ(0<λ<1)`로 스케일링하는 perturbation을 적용해 재복호(Function 1, Algorithm 1 PDD-OSD).
6. 핵심 식: 유효 perturbation 조건 `max{i0,...,i_{s-1}} ≤ n/2+kM` (Lemma 1) - min-sum 근사 `L_αβi = sign·sign·min(|L|,|L|)` 기반 도출.
7. 이터레이션 구조는 inner loop(DD-OSD 반복)와 outer loop(perturbation 재시도) 이중 루프, 최대 `tmax`/`wmax` 회.
8. 최적화 절차: 별도 학습 없음, perturbation 인덱스 집합·λ는 실험적으로 선정(Gaussian approximation 기반 오차확률 이론 분석 포함).
9. 결과: (128,36) eBCH에서 PDD-OSD order-1이 OSD order-4 대비 1.3dB, DD-OSD 대비 약 1.2dB 이득. FLOPs는 DD-OSD 대비 평균은 근소 증가, worst-case는 (tmax+1)배.
10. 한계: 순수 시뮬레이션(이론+AWGN), HW 미설계, 대상이 LDPC가 아닌 cyclic/BCH/polar 코드의 OSD/SC 계열 디코더로 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| OSD 기반 디코딩 전체 | 대응 없음 | Prime ECC는 min-sum 반복 LDPC 디코더이며 OSD 계열 알고리즘 미보유 |
| min-sum 근사 (derivative LLR 계산) | `decoder.cpp` `CNU_Update_New_Mag()` | 식 형태만 유사(sign·min), 적용 맥락(OSD 재인코딩 vs CN 업데이트) 상이 |
| perturbation(비트 신뢰도 스케일링) | 대응 없음 | trapping-set 회피 개념은 유사하나 OSD 재인코딩 프레임워크 전용 |
| cDA 널스페이스 기반 트랩 검출 | 대응 없음 | cyclic code 구조(Mattson-Solomon, derivative) 특유 성질에 의존 |

> 적용 가치: 낮음 - 대상 코드 클래스(OSD 기반 cyclic/BCH code)가 Prime ECC의 QC-LDPC min-sum 구조와 근본적으로 다르고, 이식 가능한 공통 모듈이 없음.
