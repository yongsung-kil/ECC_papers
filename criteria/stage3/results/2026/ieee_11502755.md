### B. 알고리즘 요약

1. 시스템: 6G IoT 게이트웨이향 AWGN/Rayleigh/Rician 채널, Turbo 또는 LDPC/Polar 부호, QPSK, 프레임 길이 1024bit.
2. 문제: 기존 Turbo/LDPC/Polar 디코더는 고정 반복횟수·정적 파라미터라 동적 SNR/간섭 환경에서 에너지·성능 비효율.
3. 채널모델: `y(t) = h(t)*x(t) + n(t)`, h(t)는 AWGN 또는 Rayleigh/Rician 페이딩, n(t)는 분산 σ² AWGN.
4. 핵심 기법 1단 (NN-SDM): 얕은 3계층 MLP(32-16-1)가 SNR/노이즈분산/초기 LLR을 입력받아 향상된 LLR을 출력, Log-MAP 확률계산을 데이터기반 추정으로 대체.
5. 핵심 기법 2단 (RL-AC): 상태(SNR, 현재 BER, iteration index), 행동(반복횟수 증가/감소/유지)의 MDP로 모델링, 보상 `r = -BER - α*E_iter`.
6. 학습/최적화: Q-learning 업데이트 `Q(s,a) ← Q(s,a) + η[r + γ*max_a' Q(s',a') - Q(s,a)]`, 학습률 0.001, 채널조건당 500 에피소드.
7. 결합 운영: NN이 매 반복마다 soft-decision을 점진 개선, RL 에이전트가 수렴을 위한 반복횟수를 결정하는 루프 구조.
8. 구현: MATLAB R2024b + Python 3.12(TensorFlow 2.16)로 소프트웨어 시뮬레이션만 수행.
9. 결과: AWGN에서 고정반복 Turbo 대비 수렴 31% 향상, Rayleigh SNR=8dB에서 BER 27% 개선, 프레임당 에너지 25% 절감, 지연 2.4ms→1.7ms, 평균 반복 8→6→5(Conventional/NN-Assisted/Hybrid).
10. 한계: 시뮬레이션 검증만 수행, FPGA/ASIC HW 구현 없음. 심각한 채널 변화 시 RL 정책 재학습 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NN-SDM (LLR 보정) | 대응 없음 | NN 기반 소프트정보 추정은 Prime ECC의 min-sum 기반 LLR 갱신 구조와 근본적으로 상이 |
| RL-AC (반복횟수 적응 제어) | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프에 개념적으로 대응하나, Prime ECC는 이미 Partial CRC 조기종료를 보유해 중복 |
| 채널/SNR 기반 적응 | `channel.cpp` `Set_LLR_Th()` | 채널 상태 기반 LLR threshold 조정 개념은 유사하나 무선 페이딩 채널 특화로 NAND 채널모델과 불일치 |
| Turbo/Polar 부호 자체 | 대응 없음 | binary QC-LDPC 전용 코드베이스와 부호 종류 자체가 다름 |

적용 가치: **낮음** — NN/RL 기반 무선 6G IoT 적응형 디코딩 개념으로, NAND binary QC-LDPC HW 시뮬레이터 구조(고정 파이프라인, bit-exact 검증)와 이질적이며 시뮬레이션 전용 검증이라 이식 시 재검증 부담이 큼.
