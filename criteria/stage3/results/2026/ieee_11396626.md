### B. 알고리즘 요약

1. TLC NAND(3-bit/cell), payload 1kB, code rate 0.9의 binary LDPC(HD 최대 50회, SD 최대 250회 반복)를 채널 모델로 사용.
2. 기존 SD read는 균등 간격 RRV + 고정 LLR을 쓰지만, threshold voltage 분포가 PV·RTN·retention에 따라 변하면 비효율적이라는 문제를 제기.
3. threshold voltage 분포는 `p(v|x) = pVth,x * pr * pt` (Gaussian 초기분포, Laplacian RTN, Gaussian retention shift의 컨볼루션)로 모델링.
4. RRV(양자화 경계) 최적화는 입력 심볼 `X`와 양자화 출력 `Z` 간 상호정보량(MI) `I(X;Z)`를 최대화하는 문제로 공식화(`max I(X;Z)` s.t. `q_z < q_{z+1}`), FER 직접 최적화는 비미분성 때문에 MI를 대리목표로 사용.
5. 복잡도 절감을 위해 인접 두 전압분포만 고려하는 pair-wise(SLC 근사) 최적화로 quantizer 개수를 줄임(TLC 2-bit SD: 50개→8개 transition prob).
6. LLR 매핑 `L(z) = log(분자 확률합/분모 확률합)`을 LSB/CSB/MSB 페이지별로 정의(식 18~20), 추정된 분포와 최적화된 quantizer를 바탕으로 산출.
7. 실제 분포는 컨트롤러가 직접 못 보므로, 실용 입력 특징 `x=[P/E cycle N, WL index w, on-cell ratio r]`로 분포(µ,σ)를 추정하는 다층 NN 회귀모델을 사용.
8. chip간 PV 대응을 위해 MAML 계열(CAVIA)을 적용: 공유 파라미터 θ는 offline meta-training, chip별 소량 pilot 샘플로 context 변수 φ만 SGD 업데이트(온라인 적응, 1회성).
9. 결과: 기존 ML(NN/RF) 대비 추정 오차 34.3% 감소, 5개 칩 평균 FER을 기존 2-bit SD 대비 19.49%로 감소; 단일 NN 추론 지연 약 38.35µs.
10. 한계: 시뮬레이션(합성 threshold voltage 모델)만 검증, HW 미설계, LDPC 디코더 알고리즘 자체는 변경하지 않고 채널단(RRV/LLR)만 최적화하며 파이프라인 구조상 추정 오차가 후단으로 전파됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RRV 비균등 양자화 / MI 기반 quantizer 설계 | `channel.cpp` `Set_R_Offset()` | soft-read 임계값 설계 아이디어가 채널 모델링 레이어와 대응 |
| 페이지별(LSB/CSB/MSB) LLR 매핑 함수 | `channel.cpp` `Set_LLR_Th()` | LLR threshold 테이블 설계에 참고 가능 |
| NN 기반 threshold voltage 분포 추정, MAML(CAVIA) | 대응 없음 | Prime ECC는 채널 파라미터를 NN으로 추정하는 모듈이 없음 (컨트롤러 레벨 ML, 디코더 외부) |
| LDPC message-passing 디코딩 자체 | `decoder.cpp` `LDPC_Decoder()` | 논문은 디코더를 변경하지 않고 입력 LLR만 개선하므로 간접 관련 |

> 적용 가치: 중간 — 디코더/부호 구조를 건드리지 않아 이식 난이도는 낮지만, NN·메타러닝 기반 채널 추정은 Prime ECC의 시뮬레이터 범위(C++ bit-exact 디코더) 밖의 컨트롤러 ML 로직이라 직접 이식보다는 soft-read LLR 설계 시 참고 자료로서의 가치.
