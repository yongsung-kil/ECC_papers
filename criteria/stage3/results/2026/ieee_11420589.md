### B. 알고리즘 요약

1. 대상 시스템은 3D NAND In-Flash Computing(IFC) 기반 LLM(LLaMA3-8B 등) 추론 가속기로, 로직 다이 면적·전력 제약 때문에 LDPC/BCH 대신 경량 Hamming code(SEC) 채택을 목표로 함.
2. 문제: 기존 SoTA(Lincoln-BCH, AiF-BCH)는 로직 다이 면적의 68~80%를 ECC 회로가 차지해 PE(연산 유닛) 배치를 제약; LDPC는 정정력은 강하나 면적이 더 커서 IFC 온다이 구조에 부적합.
3. 채널/오류 모델: TLC/SLC NAND의 RBER(1e-5~5e-4)에 랜덤 bit-flip 오류를 주입해 LLM weight(BF16/FP16/INT8)의 정확도(perplexity, MMLU)에 미치는 영향을 실험적으로 특성화.
4. 핵심 기법 1단(bit 중요도): 지수부 상위 비트일수록, 그리고 0-dominant 비트(대부분 0인 위치)일수록 오류에 민감하다는 관찰을 `S_bit(i) = P0(i) × 2^(2i-7)` (지수부), `S_bit(15)=2`(부호비트) 스코어로 정량화.
5. 핵심 식의 의미: 0→1 flip이 1→0 flip보다 훨씬 치명적(실험상 227,000배 perplexity 악화)이라는 비대칭성을 반영해 비트별 보호 우선순위를 매김.
6. 핵심 기법 2단(layer 중요도): residual connection의 입출력 코사인 유사도 `C = cos(x^(l), x^(l+1))`로 `S_layer = 1-C`를 정의, 유사도가 낮을수록(변환 기여가 클수록) 중요 레이어로 간주.
7. 부수 트릭: bit-group(t=3 for float, t=2 for int)과 layer 기준으로 데이터를 SEU(stratified ECC unit)로 분할하고, bit-우선 랭킹이 layer-우선 랭킹보다 성능이 좋음을 실험적으로 확인; TLC의 LSB/CSB/MSB 서브페이지 BER 차이를 이용해 중요 SEU를 저BER(LSB) 서브페이지에 배치.
8. 최적화 절차: greedy 알고리즘으로 중요도 순 SEU에 하나씩 Hamming 보호를 추가하며 WikiText2 perplexity/MMLU가 임계값(0.5%) 이내로 유지될 때까지 반복.
9. 결과: LLaMA3-8B on TLC에서 Lincoln(BCH) 대비 면적 275배, 전력 899배 감소, code rate 95.5%(+6%p), throughput 6.4Gbps 유지, 정확도 저하 0.5% 이내.
10. 한계: 대상 코드가 LDPC가 아닌 Hamming(SEC)이며 hard-decision 전용; LLM weight라는 특정 데이터 유형(오류 허용도가 높은 부동소수점/양자화 가중치)에 최적화되어 있어 일반 NAND 데이터 저장에는 그대로 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Hamming code(SEC) 부호 자체 | 대응 없음 | Prime ECC는 QC-LDPC 전용, Hamming code 모듈 없음 |
| TLC LSB/CSB/MSB 서브페이지별 BER 차이를 이용한 데이터 배치 | `channel.cpp` | 서브페이지별 신뢰도 차이 개념은 channel 모델링과 유사하나 이 논문은 부호가 아닌 "어떤 데이터를 어디에 저장할지"의 배치 정책 |
| 중요도 기반 불균등 오류 보호(bit/layer significance 기반 선택적 ECC) | 대응 없음 | Prime ECC는 codeword 내 VN degree 차등(HCU 등)은 있으나, 상위 응용(LLM weight) 데이터 중요도 기반 정책과는 무관 |
| H-matrix/부호구조, LDPC 디코더 전체 | `ecc_top.cpp` `Load_PCM()`, `decoder.cpp` `LDPC_Decoder()` | 본 논문이 오히려 LDPC를 배제하려는 논문이라 이 모듈들과 직접적 접점 없음 |

> 적용 가치: 낮음 — 부호 알고리즘 자체(Hamming, hard-decision)가 Prime ECC의 QC-LDPC/soft-decision과 이질적이며, 논문의 핵심 아이디어인 "상위 데이터 중요도 기반 선택적 보호"는 응용 계층(LLM weight) 정책이라 코드/디코더 모듈 이식 대상이 아님.
