### B. 알고리즘 요약

1. ZNS 기반 ReRAM crossbar 어레이에서 sneak-path(SP) 간섭으로 발생하는 비대칭(Z-channel+AWGN) 채널에 irregular LDPC(코드rate 0.8~0.87, length 4096)를 적용.
2. 기존 규칙적 LDPC는 비균일 간섭에 취약하고, 여러 데이터 타입을 여러 subarray에 걸쳐 저장하는 across-array 구조에서는 subarray별로 SP 확률/노이즈가 달라 "sum channel"이 비균일해지는 문제를 다룸.
3. 채널 모델: 각 cell의 저항은 SP 발생확률 `ε_ij`와 표준편차 `σ`로 특징지어지는 비대칭 채널(HR/LR + SP간섭 + AWGN)로 가정.
4. 핵심 기법 1단: 계산 가능한 비대칭 discrete density evolution(ADDE-BP) - 균등양자화기 `W(x)`로 메시지를 양자화하고, CN 갱신에 `R2` 연산자(`R2(a,b)=2tanh⁻¹(tanh(a/2)tanh(b/2))`) 기반 discrete 컨볼루션 사용.
5. VN 메시지밀도 갱신 `P_z^(l)(v)=P_z^(0)(v)⊗Q_z^(l)(u)^⊗(dv-1)`은 0-code/1-code 밀도를 각각 추적해야 하는 비대칭 채널 특유의 확장.
6. 확장 기법 2단(ADDE-BP-NON): subarray(subchannel)별로 서로 다른 "subdegree distribution" `λ^(t)(x)`를 설계하고 edge 비율 가중평균으로 합산 sum channel의 밀도를 구성 - VN을 겪는 간섭 강도별로 분류해 비균일 면역성 확보.
7. 구현 디테일: 코드 탐색은 differential evolution(초기화-변이-교차-선택)으로 degree distribution 최적화, threshold는 최소 간섭강도(σ_min)를 기준으로 비교. "pseudo-regular code"(서브채널당 단일 degree)로 설계 복잡도-성능 트레이드오프 완화.
8. 최적화 절차: differential evolution + ADDE-BP-NON 기반 threshold 평가를 반복하여 최적 degree distribution 탐색.
9. 결과: fully irregular code가 pseudo-regular 대비 BER 약 1자릿수, MacKay 코드 대비 최소 2자릿수 개선. across-array 수 T 증가 시 다양성 이득 증가, 최대 check-node degree 감소로 HW 구현 복잡도 절감.
10. 한계: asymptotic BER(density evolution 이론)와 유한장 시뮬레이션 검증에 그치며 실제 HW/FPGA 구현 없음, ReRAM sneak-path 채널 특화 가정으로 NAND read 채널과 직접 매핑 안 됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| irregular LDPC degree distribution 설계 (ADDE-BP-NON) | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 설계 레이어, 특정 QC-LDPC 고정된 현재 구조 대비 이식 난이도 높음 |
| 비대칭 discrete density evolution (R2 연산자, 양자화) | 대응 없음 | 부호설계 단계의 이론 도구로 디코더 실행시간 모듈에는 직접 대응 없음 |
| BP 반복 디코딩 자체 | `decoder.cpp` `LDPC_Decoder()` | 표준 BP 기반 검증이며 Prime ECC의 min-sum 근사와는 다른 알고리즘(성능 벤치마크용) |
| 비균일 채널 모델(subarray별 σ, ε) | `channel.cpp` | ReRAM 특화 채널 모델로 NAND RBER 채널과 개념적 유사성만 있음 |

> 적용 가치: 중간 — 부호설계(density evolution 기반 irregular LDPC) 자체는 이식 난이도가 높은 레이어이나, 비균일/비대칭 채널에서 degree distribution을 subchannel별로 최적화하는 아이디어는 NAND의 서로 다른 wordline/셀 타입별 비대칭 오류 특성에 참고할 여지가 있음.
