### B. 알고리즘 요약

1. 시스템은 BB84 기반 DV-QKD이며, 채널은 Binary Symmetric Channel(BSC)로 모델링, QBER `p`로 특성화.
2. 기존 고정 rate LDPC 조정은 순간 채널 변화(QBER 변동)에 대응 못해 SKR 저하 및 정보 누설 증가.
3. 채널 LLR은 `Lch = (1-2y)ln((1-e)/e)`로 계산, 신경망이 `Lcal = fθ(Lraw, e, σ²)`로 보정.
4. 핵심 기법: QBER 기반 code-switching 모듈이 LDPC rate를 동적 선택(고노이즈=저rate), Neural-BP 디코더가 가중치 `w0, wc'`를 학습해 V2C 메시지를 `mv→c = w0·L̂LRv + Σ wc'·mc'→v`로 갱신.
5. 이 가중 업데이트는 기존 BP의 균등 메시지 합산 대신 신뢰도 기반 가중을 부여해 수렴 개선.
6. SKR 산식: `K = R[1 - fEC·H2(Q) - H2(Q)]`, 정보 누설 `leakEC = fEC·n·H2(Q)`, FER 반영 시 `Keff = K(1-FER)`.
7. 신경망은 48-뉴런 feed-forward MLP(PyTorch)로 BER 예측 회귀모델, Adam optimizer로 학습.
8. 학습 목표는 조정 효율 `β = (H(X)-leakEC)/H(X)` 극대화.
9. 결과: QuNetSim 시뮬(512-bit 키, 에러확률<0.05)에서 정규화 SKR이 QBER<2%에서 기존 고정rate LDPC(0.75) 대비 NAAR 0.85, QBER 8%에서 0.48→0.52~0.68로 개선.
10. 한계: 순수 시뮬레이션(QuNetSim)뿐, H-matrix/부호 구조/HW 구현 전무, 향후 FPGA/ASIC 구현은 future work로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Neural-BP 가중 메시지 업데이트 | 대응 없음 | Prime ECC는 min-sum 기반 고정 HW 디코더로 학습형 가중치 갱신 구조 없음 |
| QBER 기반 적응형 LDPC rate 선택 | 대응 없음 | Prime ECC는 고정 H-matrix(`Load_PCM()`) 사용, 런타임 rate switching 미지원 |
| 채널 LLR 계산(BSC) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | BSC 대신 NAND RBER 채널이지만 LLR 산출 개념은 유사 |
| BP 디코더(Tanner graph 반복) | `decoder.cpp` `LDPC_Decoder()` | 본 논문은 Sum-Product 계열 BP이며 Prime ECC는 min-sum이라 알고리즘 자체는 다름 |

> 적용 가치: 낮음 - QKD 응용 특화, 부호 스펙/HW 정보 부재, neural 가중치 학습형 BP는 Prime ECC의 고정 min-sum 구조와 이질적.
