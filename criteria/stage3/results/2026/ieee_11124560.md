### B. 알고리즘 요약

1. 3D TLC NAND flash, QC-LDPC(18656,16748) 부호(baseline rate 0.9), layer min-sum 디코더, 6-bit quantizer, max iteration 30으로 시뮬레이션.
2. 3D 적층 구조 특유의 interlayer interference로 layer마다 RBER 편차가 크고 불규칙해, 고정 rate(0.9) LDPC만으로는 read retry가 자주 발동해 latency가 크게 증가하는 문제.
3. RBER은 P/E cycle, retention time, layer 번호의 함수로 가정(FG TLC 실측 플랫폼 PCIe 테스트 데이터 기반, LSB/CSB/MSB 페이지별 분리).
4. 핵심 기법 1단(LVLDPC): 4-layer BP(backpropagation) 신경망으로 (P/E cycle, retention time, layer) → RBER 매핑을 회귀(R²=0.90/0.97/0.93, 평균 정확도 93%), RBER을 구간별로 나눠 5개 code rate(0.9/0.85/0.8/0.75/0.7)와 매칭.
5. RBER 임계값 `L[i]`와 code rate `C[i]` 매핑 테이블(Table I)로, `RBER < L[i]`인 code rate를 선택(Algorithm 1). rate 변환은 shortening 기법(정보비트 `S_x` 만큼 0 고정 후 제외, rate=`(S_k-S_x)/(S_n-S_x)`)으로 단일 H-matrix(0.9)만 저장해 추가 오버헤드 없이 구현.
6. 확장 기법 2단(eLVLDPC): 고RBER 구간에서 rate 종류를 5개→3개(0.9/0.8/0.7)로 단순화해 반복횟수를 추가로 19.1% 감소, 정정 가능 RBER 임계를 1.56×10⁻²→2.68×10⁻²로 확장.
7. 구현 디테일: shortening된 비트는 논리값 1로 고정, HW bit-width 한계 내 최대 LLR 부여(known message 처리). 0.8 rate의 경우 코드워드 앞/뒤 두 구간을 병렬 파이프라인으로 독립 shortening 가능.
8. 학습/최적화: momentum gradient descent로 BP 신경망 학습(sample 550개/page type, train 70%/test 30%), 오프라인 실행 후 code rate 설정 테이블 생성(저장 오버헤드 1KB).
9. 결과: RBER<1.56×10⁻²에서 read retry 불필요(voltage 9회→3회, 67% 절감), 전체 iteration 67% 감소, 저장 오버헤드 1.15%(eLVLDPC 1.57%)에 불과. QLC 확장 시 iteration 19.7%(LVLDPC)/24.7%(eLVLDPC) 감소.
10. 한계: HW/FPGA 구현 없이 시뮬레이션(및 오프라인 RBER 실측 데이터) 기반, code rate 낮출수록 decoder throughput 저하(유효 저장공간 감소) 트레이드오프 존재, RBER 예측 신경망은 대략적 추정 수준(정밀도보다 프레임워크 검증 목적).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RBER 기반 code rate 동적 선택 (Algorithm 1) | `decoder.cpp` `LDPC_Decoder()` | iteration 루프 진입 전 rate 선택 로직으로 디코더 오케스트레이션에 삽입 가능 |
| shortening 기반 rate 변환 (단일 H-matrix 재사용) | `encoder.cpp` `Generate_PCM_encoding()`, `ecc_top.cpp` `Load_PCM()` | 단일 PCM에서 shortened bit를 known-LLR로 고정하는 방식으로 인코더/PCM 로드 로직과 연계 |
| shortened bit에 최대 LLR 고정 | `channel.cpp` `Set_LLR_Th()` | LLR 테이블 조작으로 구현 가능한 이식 난이도 낮은 부분 |
| layer/RBER 예측용 BP 신경망 | 대응 없음 | Prime ECC 시뮬레이터 외부의 controller-level 정책 모듈, 코드 내 대응 없음 |
| read retry 횟수 감소 메커니즘 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료·재시도 판정과 연계되는 조기종료 로직에 참고 가능 |

> 적용 가치: 높음 — shortening 기반 code rate 적응은 단일 H-matrix만 유지하며 구현 가능해 이식 난이도가 낮고, NAND 3D 적층 구조의 layer별 RBER 편차 대응이라는 목표가 Prime ECC의 실제 타깃과 정확히 일치함.
