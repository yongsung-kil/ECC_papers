### B. 알고리즘 요약

1. 시스템: terminated LDPC-CC(convolutional), rate `R=b/c`(예제 R=0.5), C1~C4 부호(dv 3~4, dc 6~9), AWGN·BPSK.
2. 문제: BP는 tanh 연산 비용이 크고 MS 근사는 성능 저하; LDPC-CC에 LP 복호 적용은 미검증.
3. 모델: ML 복호를 LP로 완화하고 parity check polytope `Pdj`(짝수 무게 이진벡터 convex hull) 제약으로 표현.
4. 핵심 기법: ADMM 분해로 LP를 MP형 반복 복호로 변환 — VNU는 `[0,1]` 투영, CNU는 `Pdj` Euclidean 투영으로 `z`-replica·`λ` 갱신.
5. 핵심 식: 증강 Lagrangian `Lμ(v,z,λ)` `(3)`과 l2 penalty `g(v)=-(v-0.5)^2`로 pseudocodeword 억제.
6. 확장: over-relaxation `ρ`, penalty `α`, penalty parameter `μ`로 수렴/성능 조정 (μ=3.0, ρ=1.9, α=0.8).
7. 트릭: CN이 edge당 `z`·`λ` 두 메시지를 전달 — 메모리 `2(m+N)`로 MP(`2m+N`)보다 큼.
8. 복잡도: CNU는 tanh 대신 투영 1회(BP와 유사, MS보다 큼), VNU는 곱셈·비교 추가로 약간 큼.
9. 결과: ADMM-l2가 BP 대비 BER 1e-5 이하에서 0.4~0.9dB 이득, error-floor 미발생(BP·MS는 발생); MS 대비 iteration 42~61% 절감이나 waterfall에서 BP보다 1.5배 느림.
10. 한계: HW 미설계·시뮬만, convolutional 부호 대상, 메모리·latency 증가, penalty 파라미터 튜닝 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP 반복 복호(VNU/CNU) | `decoder.cpp` `LDPC_Decoder()` | 반복 루프 위치는 같으나 min-sum이 아닌 LP 투영 기반이라 알고리즘 이질 |
| CNU parity-polytope Euclidean 투영 | 대응 없음 | Prime의 `CNU_Update_New_Mag()`(min-sum)와 연산 원리 상이 |
| LDPC-CC(convolutional) 부호구조 | 대응 없음 | Prime은 QC-LDPC block 고정, 컨볼루셔널 부호 미지원 |
| z-replica/λ 이중 메시지 메모리 | 대응 없음 | Prime 메시지 메모리 모델과 불일치, 저장량 증가 |

적용 가치: **낮음** — LDPC-CC 대상 ADMM-LP 복호는 error-floor 이점이 있으나 Prime의 min-sum QC-LDPC HW와 알고리즘·부호구조·메모리 구조가 모두 이질적이다.
