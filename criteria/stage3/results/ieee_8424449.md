> 총평: circulant 기반(QC) SC-LDPC의 error-floor 지향 분할·인터리빙 설계로 트래핑/흡수집합 억제 아이디어는 참고가치가 있으나, SC 결합구조와 자기기록 채널 특화라 Prime ECC block QC-LDPC로의 직접 이식성은 낮다.

### B. 알고리즘 요약

1. 시스템: circulant-based(CB=QC) LDPC를 underlying block code로 쓰는 spatially-coupled 부호, 자기기록(HDD) SNR 변동 채널 대상. Block Code 1 `(γ=3,κ=17,z=17)` rate ≈0.824, SC Code `L=30` 길이 8670, SC Code 3 `(γ=6,κ=29,z=61,m=2,L=6)` 길이 10614 rate ≈0.724.
2. 문제: HDD 연속 섹터가 서로 다른 SNR을 겪음 → 섹터별 독립 block code는 최저 SNR 기준 과잉 redundancy·성능 저하.
3. 모델: `N`개 AWGN 섹션, 섹션 SNR을 자기상관 벡터 `a`(WD 실측)로 결합한 상관 SNR 변동 모델(식: `ΔSNR_s = a^T u`).
4. 핵심 기법 1: minimum-overlap(MO) 분할을 column weight `γ=2(m+1)`(즉 γ=4,6)으로 확장 — 성분행렬 간 최대 오버랩 `t`를 최소화(Theorem 1: `tmin=⌈κ/⌊γ/2⌋⌉`)하여 cycle·trapping·absorbing set 발생 억제.
5. 식 의미: 오버랩이 그래프의 유해 조합구조를 만들므로 이를 최소화하는 balanced partitioning이 error-floor를 낮춘다; lifting 후 CPO(circulant power optimizer)로 circulant power를 추가 최적화.
6. 핵심 기법 2: SC 부호가 `(m+1)` 연속 섹션에 걸친 CN으로 신뢰도 높은 VN이 낮은 VN을 도와 SNR 변동에 강인(memory `m`↑일수록 강인).
7. 부수 트릭: SC 전용 저비용 인터리빙 — codeword를 `μopt=L²/(m+1)` chunk로 분할(Claim 1)하여 각 CN이 모든 신뢰도 레벨에서 균등한 메시지를 받게 함(기존 regular 인터리빙 `L²` chunk 대비 저복잡).
8. 최적화: MO 분할 후보 탐색 or 단일 MO+CPO 휴리스틱으로 유해 객체 수 최소화.
9. 결과: SC Code가 uncoupled block 대비 error-floor에서 1~2 order 개선, SC Code 3은 `SNR=4.1 dB`에서 BER `10^-10`, SC 부호의 평균 수렴 iteration이 block 대비 일관되게 적음(latency는 comparable).
10. 한계: 시뮬레이션만, HW 미설계, gate/throughput 미보고, SC 결합구조·자기기록 SNR변동 채널에 특화.
