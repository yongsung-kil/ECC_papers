### B. 알고리즘 요약

1. 대상 부호는 짧은 blocklength의 대수적(algebraic) 선형블록코드 — extended Golay `[24,12,8]`, extended quadratic residue `[48,24,12]`(EQR48), `[104,52,20]`(EQR104), 채널은 AWGN.
2. 문제의식: 이런 고전 대수부호는 parity-check matrix H의 weight가 높아 Tanner graph에 short cycle이 많아 표준 SPA(sum-product algorithm) 성능이 나쁨.
3. 핵심 가정: 동일 부호 C에 대해 서로 다른(structurally distinct) parity-check matrix가 다수 존재하며, 이들을 번갈아 쓰면(diversity decoding) 국소 최적점(fixed point)을 탈출해 SPA 성능이 개선됨.
4. 핵심 기법: Edge Local Complementation(ELC) — Tanner graph의 edge `(u,v)`를 하나 골라 그 국소 이웃(`N_u`, `N_v`) 사이의 edge를 상보(complement)시켜 H를 행 덧셈(row addition)으로 변형하되 부호 C 자체는 보존. 기존 전역적 permutation decoding(SPA-PD, `Aut(C)` 필요)과 달리 완전히 국소적(local) 연산.
5. 핵심 식: SPA-ELC 알고리즘(Algorithm 2)은 `I1` flooding iteration마다 임의 edge `p`개를 뽑아 ELC 적용 후 새 edge의 메시지를 damping factor `α`로 초기화(`μ_v'→u' ← (x̂_v'-L_v')α + L_v'`) — damping은 ELC로 새로 생긴 edge에만 국한 적용.
6. 확장: damping 없는 버전 `SPA-ELC(p,I1,T)`(α0=1)도 별도 평가 — 단순히 flooding iteration 사이에 무작위 ELC를 끼워 넣는 형태.
7. 구현 디테일: 코드는 사전에 weight·4-cycle 수를 최소화하는 휴리스틱으로 전처리(Table I); ELC 1회 연산 복잡도는 `O(|N_u||N_v|)`로 SPA 1 iteration보다 훨씬 작음.
8. 최적화 절차: 파라미터 `p`(ELC 적용 횟수)는 EQR48 코드에서 경험적으로 결정.
9. 결과: Golay `[24,12,8]`에서 SPA-ELC가 표준 SPA 대비 FER·평균 SPA 메시지 수 모두 개선, SPA-PD와 유사한 FER 성능을 (약간의 복잡도 페널티로) 국소 연산만으로 달성; damping 없는 SPA-ELC는 SPA-PD에 근접한 복잡도를 약간의 FER 손실로 달성.
10. 한계: MLD(최대우도복호) 대비 gap이 존재하며 HW 구현·gatecount 없음, 대상이 short algebraic block code(24~104bit)로 대규모 sparse QC-LDPC(NAND ECC)와는 그래프 밀도·크기가 근본적으로 다름. 매 iteration 그래프 구조가 동적으로 변해 고정 파이프라인 HW에 부적합.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA(sum-product, flooding iteration) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum(HW 근사) 기반이라 full-SPA(tanh)와 다름 — 대응은 있으나 알고리즘 계열이 다름 |
| ELC 기반 동적 Tanner graph 변형(diversity decoding) | 대응 없음 | 매 iteration H-matrix 구조를 바꾸는 접근으로 고정 QC-LDPC PCM(`PCM_b`) 기반 HW 파이프라인과 근본적으로 불일치 |
| 부호 구조(short algebraic block code, Golay/QR) | `ecc_top.cpp` `Load_PCM()` | 대상 부호 자체가 QC-LDPC가 아니므로 대응 없음(non-QC-LDPC) |

> 적용 가치: 낮음 | short algebraic block code 전용 기법이며 매 iteration 그래프 구조를 바꾸는 방식이 고정 QC-LDPC HW 파이프라인과 근본적으로 맞지 않음.
