### B. 알고리즘 요약

1. 대상: 중국 디지털TV 지상파방송(DTMB) 표준의 QC-LDPC 코드 3종(rate 0.4/0.6/0.8), block size `t=127`, column 수 59×127로 동일.
2. 문제: 세 rate를 각각 단일-rate 디코더로 구현하면 하드웨어 자원(로직/메모리) 중복이 크므로 재사용 구조가 필요.
3. 채널 모델: AWGN, BPSK 변조, zero-mean noise variance `σ²`.
4. 디코딩 알고리즘: 표준 min-sum(MS) 기반 BP 근사. Initialization `Qn=2y_n/σ²`, Horizontal update `R_mn = (Π sign(Q))·min|Q|`, Vertical update `Q_n=P_n+ΣR_mn`.
5. 구조 단순화: vertical update와 tentative decoding(판정) 단계를 결합해 4단계(Init/Horizontal/Vertical/Decision)를 3단계로 축소, 제어 단순화.
6. Multi-rate 재사용: 세 코드의 column degree가 {3,4,7,11,16}으로 공통되어 VPU(vertical processing unit)를 rate-switch로 재사용(VPU3~VPU16 5종), row degree는 rate별 상이해 J-HPU(joint horizontal PU)로 여러 HPU를 결합.
7. 구현 디테일: improved VPU는 rate-choice switch 입력을 추가해 메모리 유닛은 296→332개로 증가하지만 routing unit 2개를 절감.
8. 최적화 절차: 없음(자동 최적화 아님, 수작업 구조 설계).
9. 결과: multi-rate 디코더 LC 사용량 38331 vs 단일 3개 디코더 합 76966(절반 이하), 처리량 60.8MHz clock, BER 3×10⁻⁶에서 SNR 성능은 단일-rate 디코더와 동등.
10. 한계: FPGA 검증만(ASIC 미검증), 디코딩 알고리즘 자체 개선 없음, DTMB 특정 3-rate 코드 구조에 특화된 설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-sum 디코딩(horizontal/vertical update) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 표준 min-sum으로 Prime ECC가 이미 보유, 개선 없음 |
| VPU 재사용(column degree별) | `decoder.cpp` `VN_Cal_HD()` 등 | Prime ECC는 단일 고rate 부호 고정이라 multi-rate VPU 공유 개념 자체가 대응 없음 |
| HW 자원 공유(multi-rate 아키텍처) | 대응 없음 | Prime ECC는 multi-rate 지원 목적이 아님 |
| 메모리/라우팅 최적화 | 디코더 HW 모델(z=32, HCU) | 자원 재사용 아이디어는 참고 가능하나 구조가 상이 |

> 적용 가치: 낮음 — 표준 min-sum 알고리즘에 multi-rate 자원 공유라는 FPGA 구현 트릭만 있고, Prime ECC는 단일 고rate QC-LDPC 고정이라 multi-rate 재사용 구조 자체가 필요하지 않음.
