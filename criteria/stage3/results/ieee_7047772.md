### B. 알고리즘 요약

1. DVB-T2/C2/NGH의 BICM(FEC+변조+비트인터리버) 구조에서, Q=360 circulant 기반 QC-LDPC(IRA 구조) 코드와 고차 QAM(16~256/1024/4096)을 결합해 전송.
2. 문제: 고차 변조 시 하나의 QAM 심볼에 동일 check node에 연결된 2개 이상의 codeword bit이 실릴 수 있음 - 이를 "less-capable check node"(LCN)라 정의, 심각한 fading/erasure 채널에서 성능 저하 유발.
3. 채널 모델: Rayleigh faded erasure channel, `y_i = h_i e_i x_i + z_i` (h_i는 페이딩, e_i는 소거 지시자, z_i는 AWGN).
4. 기존 해법(column-twist interleaver)은 별도 저장/연산 오버헤드 필요. 제안 기법은 LDPC 코드의 순열 등가성(permutation equivalence)을 이용, parity check matrix H의 column을 재배열해 새로운 부호기술 `Ŝ_{n,r}`을 생성.
5. Inter-block permutation: 메시지부 블록(크기 (n−k)×Q) 단위로 블록째 순서 교환, `ĥ_{π(p)Q+r} = h_j` (식 1).
6. Intra-block permutation: 블록 내부에서 Q개 열을 circular shift, `ĥ_{pQ+u} = h_j`, `u=(j mod Q + t_p) mod Q` (식 2) - 자유도가 커서 탐색이 더 쉬움.
7. 부수 디테일: 부호기술 재배열은 인코더 변경 없이(QC/IRA 구조 유지) 순수 column-index 재기술만으로 수행, Algorithm 1(탐색적 offset 결정)로 모든 Nc(변조 조합)에 대해 LCN-free를 만족할 때까지 반복.
8. 최적화 절차: brute-force/trial-and-error 탐색으로 offset `t_i` 결정 (별도 학습 없음, 단순 탐색 알고리즘).
9. 결과: n=64800, r=3/4 코드에서 16/64/256-QAM 전 구간에서 column-twist interleaver 사용 시와 동일한 BER 곡선 달성(성능 저하 없음), Fig.5.
10. 한계: 시뮬레이션(BER 곡선)만 제시, HW 설계·게이트카운트·throughput 언급 없음. DVB 표준의 특정 QC-LDPC H-matrix에 종속적인 방법.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Block permutation (inter/intra-block, LCN 회피) | 대응 없음 | BICM 고차변조 인터리빙 문제로, NAND read channel(변조/심볼 개념 없음)에는 대상 자체가 없음 |
| QC-LDPC H-matrix column 재배열 | `ecc_top.cpp` `Load_PCM()` | 형식적으로만 H-matrix 재배열이라는 점에서 유사하나, 목적(LCN 제거)이 Prime ECC에 무의미 |

> 적용 가치: 낮음 - BICM/고차변조 특화 문제(LCN 회피)이며 LDPC 정정능력 자체는 불변이므로 NAND binary LDPC ECC 이식 요소 없음.
