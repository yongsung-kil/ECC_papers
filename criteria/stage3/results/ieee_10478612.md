> 총평: informed dynamic scheduling(IDS) 계열 BP 디코딩의 복잡도·수렴속도 개선 알고리즘으로 waterfall 영역 성능·iteration 감소 효과는 있으나, 비동기 edge-단위 스케줄링이 전제되어 Prime ECC의 동기식 row-parallel HW 구조로의 이식은 재설계 부담이 큼.

### B. 알고리즘 요약 (10줄 내외)

1. IEEE802.16e 표준 기반 irregular LDPC 코드 (576,288)/(576,144)/(1152,576)/(2016,1008) 및 PEG 구성 regular (2016,1008) 코드, AWGN + BPSK 채널.
2. 표준 BP는 고정 순서로 전체 노드를 갱신해 수렴이 느리고, 기존 informed dynamic schedule(IDS) 계열(RBP/NW-RBP/SVNF-RBP/RD-RBP)은 residual 비교 비용이 `O(E)`로 크거나 특정 edge 집합이 갱신을 독점하는 탐욕성(greediness) 문제가 있음.
3. 핵심 가정: C2V residual `r_{ci→vj} = |m_new - m_old|`(식 4)이 클수록 우선 갱신할 가치가 큰 정보로 간주하되, 과도한 탐욕성은 일부 variable node의 정보를 묵살시킴.
4. 핵심 기법: 매 갱신마다 전체 edge를 비교하는 대신 두 개의 소형 리스트 `L0`(이전 잔여분 상위), `L1`(신규 계산분 상위, `Size(L1) = 최소 cycle 길이 - 1`)로 구성된 잔여 리스트 `L`만 비교 대상으로 삼는 List-based RBP(L-RBP).
5. `L`에서 최대 residual edge를 골라 C2V 갱신 -> V2C 전파 -> 연결 edge들의 residual 재계산 -> `L`을 부분 갱신하는 사이클(Algorithm 1)로, 안정화된 edge를 리스트에서 배제해 반복적으로 소모되는 비교 자원을 줄임.
6. 확장: 중복 갱신되는 edge의 residual을 decay 식 `r = β^n × |Δm|`(식 5, `β=0.9`, RD-RBP[7] 방식 차용)로 감쇠시켜 특정 edge 그룹의 독점(탐욕성)을 추가로 억제.
7. 구현 디테일: `Size(L1)`은 코드의 최단 cycle 길이에 연동, `Size(L0) = Size(L) - Size(L1)`, `Size(L)`은 하한 `Size(L1)+1` ~ 상한 `E` 범위에서 튜닝(논문은 `E/128` 채택).
8. 파라미터 탐색: `Size(L)`을 스윕하며 BER/FER과 수렴속도의 trade-off를 시뮬레이션으로 관찰해 최적값 결정(그리드서치 성격, 별도 학습 없음).
9. 결과: (2016,1008) irregular 코드에서 BER 10^-5 기준 RD-RBP 대비 약 0.13dB 이득, regular 코드에서 BER 10^-4 기준 RBP 대비 0.25dB·NW-RBP 대비 0.2dB·RD-RBP 대비 0.12dB 이득; 비교 연산량이 `d_v≈3` 기준 NW-RBP의 약 1/2, RBP/RD-RBP의 약 1/11.
10. 한계: HW 설계·합성 없이 알고리즘/시뮬레이션 수준에 그치고, 비동기(edge-serial) 갱신 스케줄이 전제되어 병렬 HW 구현 시 스케줄러 오버헤드가 별도로 필요하며 short-length·high-rate 코드에서는 복잡도 이점이 줄어든다고 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| C2V residual 기반 informed dynamic scheduling(L-RBP) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 동일 디코더 루프 계층이지만 Prime ECC는 동기식 row-parallel(z=32) 갱신, 논문은 edge-serial 비동기 갱신이라 스케줄링 모델 자체가 상이 |
| CN 갱신 시 사용하는 full sum-product `tanh` 연산(식 3) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min-sum(§3)으로 tanh를 근사·대체했으므로 이 부분은 직접 이식 대상 아님(참고용) |
| Dual-Update(even/odd 교대) 스케줄 | Prime ECC `Dual-Update` 옵션 | 논문의 리스트 기반 우선순위 스케줄은 Prime ECC의 고정 교대 스케줄과 목적은 유사(수렴 가속)하나 구현 방식은 이질적 |
| 잔여 리스트 크기 결정에 쓰이는 최단 cycle 길이 분석 | `ecc_top.cpp` `Load_PCM()` | H-matrix girth 분석 자체는 이식 가능하나 QC-LDPC 고정 구조에서 큰 의미 없음 |

적용 가치: **중간** — waterfall 영역 성능·iteration 감소 효과는 실질적이나, 비동기 edge-serial informed scheduling이 전제되어 Prime ECC의 동기식 row-parallel(z=32) min-sum 파이프라인에 이식하려면 스케줄러 재설계가 필요함.
