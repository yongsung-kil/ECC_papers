> 총평: NAND MLC 셀 내 비트 의존성을 min-sum에 additional belief로 결합해 hard-decision 성능·throughput·에너지를 크게 개선한 직접 타깃 논문 — Prime ECC 복호 루프에 이식 가치 높음.

### B. 알고리즘 요약

1. 시스템: MLC(2bit/cell, Gray) NAND용 min-sum QC-LDPC, rate `8/9`, length `2kB`, column weight `4/5`; throughput 평가엔 length `18432` rate-8/9 디코더(300MHz, 256 병렬, 72 cyc/iter) 사용.
2. 문제: soft-sensing은 read latency를 키워 throughput을 떨어뜨림 → hard-decision 성공률을 높여 soft 호출 빈도를 줄이려 함.
3. 모델/관찰: MLC 셀은 upper page BER ≫ lower page(9k P/E에서 최대 76% 차), 같은 셀 두 비트 간 데이터 의존성 존재(Gray 매핑).
4. 핵심기법1 (intracell data placement interleaving): 한 codeword가 같은 셀의 upper/lower page 비트를 함께 포함하도록 subblock 단위(디코더 병렬폭의 절반)로 두 page를 교대 배치.
5. 핵심기법2 (data dependence aware min-sum): 같은 셀 짝 비트 `c`의 `Rc,dc`로부터 추가 belief `Ki`를 산출해 V2C 메시지(step3)와 경판정(step4)에 가산.
6. 핵심식: `Ki = αRi (Rc<0&dc<0, lower)` / `0.5αRi (Rc·dc<0, lower)` / `0.5α|Ri| (Rc·dc<0, upper)` / `0 (else)`, `α=0.75` — 짝 비트가 신뢰 가능할 때 자기 LLR 신뢰도를 강화.
7. 디테일: 4-bit 내부정밀도, 256 병렬, 추가 belief 유닛은 단순 조합회로(`M=αRi`, `M=0.5αRi` MUX); flooding/layered 스케줄 모두 적용, 파일 비연속 시 회로 비활성화로 일반 min-sum 동작.
8. 학습/최적화: sub-22nm MLC 칩 raw BER 실측(128블록, 10k P/E), Flashsim FTL 시뮬로 연속 page 접근율 99.9%+ 검증, Ki 민감도 분석.
9. 결과: hard-decision throughput 최대 +84.6%(flooding)/62.8%(layered), 에너지 -33.2%, 평균 iteration·decoding failure rate 감소; 추가 belief 모듈 6592게이트/9216μm²@TSMC 65nm로 area overhead <0.2%.
10. 한계: upper/lower page 동시 read 필요라 대용량 read-only 멀티미디어 파일에 한정, MLC 2bit/cell·Gray 매핑 특성에 종속, throughput 절대값(Gbps) 미제시.
