> 총평: (3,6) regular LDPC의 log-domain sum-product 디코더를 tree-topology 개선·8bit 고정소수점으로 FPGA(10Mbps) 구현한 교과서적 HW 설계 논문으로, Prime ECC의 min-sum QC-LDPC와 정합성·이득 모두 낮음.

### B. 알고리즘 요약

1. 대상 부호는 기하구성법으로 설계된 girth-8 `(1008, 3, 6)` regular LDPC (rate 0.5), 채널은 노이즈 오염된 likelihood-ratio 입력을 가정한다.
2. 풀려는 문제: tree-topology check-node 갱신이 core 연산 호출 수와 클록 사이클이 많아 HW 구현이 비효율적이다.
3. check-node 갱신의 두 core 연산은 `f(U⊕V')`와 `L(U⊕V)`이며, tree 구조에서 (3,6) 부호는 5회 `f`, 6회 `L`, 4 clock이 필요하다.
4. 개선안: verification-node 갱신에 `f(U⊕V')` core 연산을 2개만 포함시키고 total core 연산을 12회로 재배치, 클록을 4→3으로 축소한다.
5. reduction 연산 병렬화로 single-point check 갱신을 9회 `f(U⊕V')` + 4 clock으로 처리한다.
6. `L(U⊕V)` core 연산을 제거해 양자화 대상은 `g(x)=log(1+e^-|x|)` 함수와 입력 `2y/σ²`만 남는다.
7. 양자화: 8bit 고정소수점(부호 1bit + 정수 3bit + 소수 4bit), 값 범위 ±7.9375, 초기·중간 정보에 동일 표준 적용해 별도 처리 불필요.
8. HW 구조: 입력전처리 + variable-node 갱신(폭 Wv=36, L=28) + check-node 갱신(폭 Wc=18, L=28) + interleaving + 판정출력, 부분병렬. lookup(lpm_rom)으로 `g(x)` 저장.
9. 결과: Stratix II EP2S60F484C3에서 logic 67%, memory 15%, 최고 처리율 10Mbps, Matlab과 시뮬레이션 일치.
10. 한계: 정정성능 개선 없음(구현 효율만), throughput 10Mbps로 NAND급에 크게 미달, BER/gate-count 정량치 미제시, 특정 기하구성 (3,6) 부호에 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check-node 갱신 core 연산 축소 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime은 min-sum 사용, 논문은 log-SP `f`/`L` core 연산 축소 → 알고리즘 계열 상이, 이식 이득 낮음 |
| variable-node 부분병렬 갱신 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` | Prime은 z=32 row 병렬, 논문은 Wv=36 단순 부분병렬 → 구조 상이 |
| 8bit 고정소수점 양자화 | [ecc_data.h](../Prime_ECC_3.1_Claude/) `PARAM_LLR` | Prime은 iteration별 LLR 테이블 보유, 단일 8bit 균일양자화는 이미 열위 |
| lookup(lpm_rom) `g(x)` 테이블 | 대응 없음 | log-SP 전용 correction 함수, min-sum엔 불필요 |

적용 가치: **낮음** — log-domain sum-product 계열의 tree-topology 개선이라 min-sum 기반 Prime ECC와 알고리즘 정합성이 낮고, 정정성능·throughput 이득이 없다.
