> 총평: NAND flash controller의 copyback 기반 내부 데이터 마이그레이션(IDM) 정책을 위한 Markov 상태전이 예측 모델로, ECC(LDPC)는 마이그레이션 트리거 판단의 배경 요소일 뿐 부호설계·디코더 알고리즘 기여가 없어 Prime ECC 이식 대상이 아님.

### B. 알고리즘 요약

1. 시스템: 3D TLC/MLC NAND flash SSD의 copyback(동일 plane 내 페이지 직접 이전) 명령을 이용한 내부 데이터 마이그레이션(IDM)에서, ECC 한계를 넘기 전 최대 copyback 허용 횟수를 예측하는 문제.
2. 문제: 기존 기법은 (1) copyback 반복 시 페이지 종류(LSB/CSB/MSB)별 오류누적이 비대칭인 점, (2) 디코딩 전에는 RBER을 알 수 없어 EDM(외부 마이그레이션) 없이는 copyback 임계값 판단이 불가능한 점을 다루지 못함.
3. 가정: 셀 상태 전이확률(read reference voltage 오검출)을 4x4 Markov 전이행렬 `P`로 모델링, 동일 plane 내 블록은 wear-leveling으로 PE cycle이 유사해 전이확률도 유사하다고 가정.
4. 핵심 기법: 초기 상태분포 `S(0)`에서 k회 copyback 후 분포 `S(k)=S(0)·P^k`를 예측(식 4)하고, 실측 RBER 대신 KL divergence `β = Σ S_i(0) log(S_i(0)/S_i(k))`를 사용해 두 분포 차이를 정량화(식 6).
5. `β`가 사전 결정된 임계값을 초과하면 copyback 한계 도달로 판정해 EDM으로 전환 — RBER을 직접 몰라도 β를 대리 지표(proxy)로 사용하는 것이 핵심 아이디어.
6. 확장(ECCPM): PE cycle 실시간 기록이 없는 경우를 위해, 지수항을 추가한 개선 β식(식 9, `+γe^{S4(0)}`)으로 페이지 타입별(LSB/MSB) 임계값을 정렬하고, 오프라인으로 여러 PE 구간별 전이행렬 집합 `B={P̂_1,...,P̂_M}`을 미리 계산.
7. 구현 디테일: GC 시점에 실제 워드라인 상태분포를 EDM으로 측정해 집합 `B` 중 최소 적합오차(least absolute error) 행렬을 투표(voting) 방식으로 선정, 이후 해당 행렬을 재사용(Algorithm 1, 2).
8. 학습/최적화 절차: 상태전이확률 `P̂_ij`를 최소화 문제(식 7,8, `min Q`, `P̂_ii>δ=0.92`)로 오프라인 적합.
9. 결과: FEMU(FTL 시뮬레이터) 기반 실험에서 기존 copyback-GC(TCB-GC) 대비 랜덤 write 워크로드 지연시간 최대 22.8~24.9% 감소, write amplification은 유의미한 악화 없음. LDPC(2048,2304) 부호는 MLC 오류주입 시뮬레이션에 사용된 기존 표준 부호(참고문헌 [31][32])로만 인용.
10. 한계: ECC/LDPC 알고리즘 자체는 제안 대상이 아니며 기존 공개 모델을 오류 시뮬레이션 도구로만 사용, HW 설계 없음(FTL/GC 정책 레벨 알고리즘), NAND controller 펌웨어 로직에 국한.

| LDPC(2048,2304) 오류주입 시뮬레이션 | 대응 없음 | 논문 내 배경 도구일 뿐 부호설계·디코딩 알고리즘 기여 없음 |
| RBER 사전 미지 상태에서의 ECC 여력 추정 | `channel.cpp` (RBER 정의부와 개념적 연관) | 직접 대응 함수 없음, ECC 용량 초과 여부 판단이라는 개념만 인접 |
