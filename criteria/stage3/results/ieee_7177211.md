### B. 알고리즘 요약

1. HINOC 2.0(동축케이블 기반 차세대 방송망) 물리계층 요구사항: code length 1920, code rate 9/10, 1Gbps 이상 throughput.
2. 기존 표준(802.16e, DVB-S2/T2)의 LDPC 코드/디코더는 1Gbps급 고속 throughput을 지원하지 못해 새 설계 필요.
3. Structured Irregular Repeat-Accumulate(S-IRA) 구조 기반 QC 서브행렬(`P_{i,j}`, 크기 `q×q`)로 파티티 검사행렬 `H=[H1, ΠP]` 구성, 서브행렬 크기 `q=24`가 병렬 factor.
4. 디코더는 Ping-Pong RAM 기반 병렬 구조: `X = N*4/(w*T1)` 로 최대 iteration 수 산출(`N=1920, w=16, T1=56`), 전통적 2-RAM 구조 대비 5-RAM 구조로 X=8 확보.
5. 제안하는 advanced Ping-Pong RAM 구조는 5개 RAM + 4개 decoder core로 RAM R이 x번째 채워지면 `D=(R+x+2)%4+1` 코어에 데이터 전달, 기존 5-core-5-RAM 구조 대비 코어 1개(자원 23%) 절감.
6. 추가로 "additional iteration scheme" 제안: HINOC 2.0은 양방향 통신이라 LLR 스트림이 불연속적이므로, 다음 RAM이 채워지기까지 남는 시간을 활용해 최대 iteration(8)을 넘어 추가 반복(최대 50) 수행.
7. throughput 산출식 `T = f_clock * w * 0.9`, `f_clock=130MHz` 추정 시 bit-width `w=16`으로 1Gbps 이상 확보.
8. 별도 학습/최적화 절차 없음 (구조적 설계 + degree distribution 수동 설정: `λ(x)=0.5503x^3+0.3904x^2+0.0592x+0.0002`, `ρ(x)=0.3732x^32+0.6268x^33`).
9. 결과: 802.16e 코드 대비 Shannon distance 기준 0.1dB 개선, FPGA Stratix V 145MHz, 2Gbps throughput, 자원 23% 절감(코어 1개분), iteration 8→16 시 큰 성능 향상.
10. 한계: BP(Belief Propagation) 디코딩만 시뮬레이션(min-sum 등 HW 친화 근사 없음), NAND가 아닌 방송/케이블 통신망 표준 대상, error-floor 분석 없음(waterfall 영역만 제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| S-IRA 구조 QC-LDPC 부호 설계 | `ecc_top.cpp` `Load_PCM()` | 특정 방송망 표준용 rate 9/10 부호로 NAND 부호 구조와 별개, 재검증 부담 큼 |
| Ping-Pong RAM 병렬 디코더 구조 | 대응 없음 | Prime ECC는 z=32 row 병렬 구조로 이미 별도 HW 아키텍처 보유, RAM 스케줄링 방식 상이 |
| Additional iteration scheme (유휴시간 추가반복) | `decoder.cpp` `LDPC_Decoder()` | 유휴 사이클에 반복 추가한다는 개념은 있으나 NAND read 스케줄과는 다른 문맥(스트리밍 불연속성 기반) |
| BP 디코딩 (min-sum 아님) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 이미 min-sum 보유, BP는 이식 대상 아님 |

> 적용 가치: 낮음 — HINOC 2.0 방송망 표준 전용 부호/구조이며 NAND와 무관, min-sum이 아닌 BP 기반 시뮬레이션이라 HW 이식 가치도 낮음.
