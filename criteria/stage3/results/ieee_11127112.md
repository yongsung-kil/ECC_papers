### B. 알고리즘 요약

1. 시스템: SMR HDD, 4 Tbits/inch² 밀도, (30,3)-regular LDPC 코드(길이 `36990`, column weight 3, row weight 30), 128/130 RLL 인코딩과 결합.
2. 채널: PR1 타깃 등화, TD-FIR로 ITI 억제, APP 디코더와 SP 디코더 간 turbo equalization 반복.
3. 문제: 종전 신경망 기반 LLR 강조 기법에서 GA로 threshold/weight를 탐색했으나, 적합도 함수가 신뢰도 낮은 LLR까지 강조해 성능 개선이 제한적.
4. 핵심 기법: 새 적합도 함수 `f_δl^(1) = -log10(Nte/Nt) - Cδl·log10(1-Nt/Nall)` 로 강조 정확도와 강조 확률을 동시에 반영.
5. 식의 의미: 첫 항은 오강조 비율(선택 정확도), 둘째 항은 강조되지 않은 LLR 비율에 계수 `Cδl`을 곱해 강조 개수 증가를 유도 - 두 목표를 하나의 스칼라로 결합.
6. `Cδl`은 세대마다 GA로 함께 진화하며 약 0.965로 수렴(11세대 이후), 선택 정확도와 강조 확률의 균형점을 자동 탐색.
7. 신경망은 decoding target bit과 인접 bit의 `ln γkn`을 입력으로 강조 여부(THγd 기준)와 강조 가중치(Wγd)를 결정, 4가지 decoding pattern(d=0~3)별 파라미터 별도 관리.
8. 최적화 절차: GA로 THγd, Wγd, Cδl을 1차원 chromosome으로 묶어 selection/crossover/mutation, elitist preserving selection 적용, 20세대 진화.
9. 결과: 제안 적합도 함수 적용 시 isp=20, SNRS=21.0dB에서 iglobal=22로 error-free 도달(기존 대비 반복 횟수 감소).
10. 한계: 시뮬레이션 단계뿐이고 HW 설계 없음, HDD SMR 특화 채널/코드 구조로 다른 도메인 일반화는 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 신경망 기반 LLR 강조 SP 디코딩 | 대응 없음 | Sum-Product 전체(Maxlog-MAP, trellis 기반) 알고리즘으로 Prime ECC의 min-sum 구조와 근본적으로 다름 |
| GA 기반 파라미터 탐색 | 대응 없음 | Prime ECC의 LLR threshold 최적화는 `local_opt.cpp`가 있으나 GA 방식 자체는 대응 없음 |
| (30,3)-regular LDPC 부호구조 | `ecc_top.cpp` `Load_PCM()` | regular LDPC이며 Prime ECC는 QC-LDPC 고정 구조라 참조성 낮음 |

적용 가치: `낮음` - HDD SMR 채널 특화 신경망+SP+GA 복합 기법으로 binary QC-LDPC/min-sum 기반 NAND ECC와 알고리즘·채널 모델이 근본적으로 달라 이식 요소가 거의 없음.
