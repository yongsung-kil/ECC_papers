### B. 알고리즘 요약

1. 대상: irregular LDPC 코드, PEG(progressive edge-growth) 알고리즘으로 구성된 block length 1008, rate 0.5 코드(MacKay's `PEGirReg504x1008`)를 BEC(binary erasure channel) 상에서 평가.
2. 문제: 유한장 LDPC 코드의 error floor는 작은 stopping set(BEC 기준, 이웃이 모두 2개 이상 연결된 VN 집합)이 지배, 기존 PEG/ACE 등 구성법은 stopping set 존재를 완전히 배제하지 못함.
3. 가정: Tanner 그래프에 새 check node `c_a`를 추가해 특정 VN들에 엣지를 연결하면, 원래의 stopping set은 제거되거나 그대로 유지될 뿐 새로운 stopping set은 생기지 않는다는 성질(BEC 한정)을 이용.
4. 핵심 기법: SSE(Stopping Set Eliminating) 알고리즘 - 소형 stopping set 목록 `S`에서 빈도가 가장 높은 VN을 골라 새 check node에 연결, 해당 VN이 포함된 stopping set을 모두 소거된 집합 `SE`로 옮기고 반복.
5. 핵심 식: 점근적 FER `P_F(ε) ≈ Σ s_j ε^j` (`j=s_min..s_min+u`, 식 (2))로 stopping distance `s_min` 개선 효과를 평가.
6. 부수 트릭: 새 check node의 degree를 원래 그래프의 지배적 check node degree와 맞춰 코드 구조 유지, degree 초과 시(`j≥d_c`) 새 check node를 추가로 생성.
7. rate 손실 보정: 동일 rate 유지를 위해 처음부터 rate가 약간 높은 코드(C4, rate 0.503)를 구성한 뒤 check node 3개를 추가해 원래 rate(0.5)로 맞추는 방법(C3) 제시.
8. 학습/탐색 없음 - 결정론적 그리디 알고리즘. stopping set 탐색은 기존 알고리즘(Richter, [10])을 재사용(belief propagation + erasure decoding 기반 근사).
9. 결과: check node 2개 추가로 stopping distance 상한이 13→22로 개선, rate 손실 0.002에 그침. BEC 상 FER이 원래 코드 대비 약 2자릿수(order of magnitude) 개선.
10. 한계: BEC 채널 전용 이론/시뮬로 AWGN 등 실채널 미검증, 저자 스스로 4-cycle(및 trapping set) 유발 가능성을 한계로 언급, high-rate 코드에서는 개선 효과가 제한적이라고 명시.
