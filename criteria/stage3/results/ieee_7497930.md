### B. 알고리즘 요약

1. CCSDS (2048,1024) LDPC 코드(rate 0.5)를 AWGN 채널·BPSK·5-bit quantization으로 시뮬레이션.
2. 문제: 기존 informed dynamic schedule(RBP, VC-RBP)과 lazy schedule(LS-LBP 등)은 각각 수렴 가속에 한계가 있고, "다음에 갱신할 메시지"를 잘 고르는 순서 결정이 어려움.
3. 가정: variable node의 LLR 부호가 업데이트 전후로 바뀌면 그 노드는 "unreliable"(에러 가능성 높음)로 정의.
4. 핵심 기법 1단 (Lazy schedule): girth 기반 업데이트 확률 `p_n = f(g_n, g_max)`(식 4)로 갱신 대상 variable node 집합 `S_j`를 확률적으로 선택, 짧은 cycle(girth 작음)의 악영향을 지연시킴.
5. 핵심 기법 2단 (새 IDS): `S_j` 내에서 unreliable variable node를 우선으로, variable-to-check 잔차(residual) `r(m_{v→c})`(식 5)가 가장 큰 메시지를 먼저 전파하도록 선택(unreliable + 최대 residual 조합).
6. 선택된 variable node의 unreliable 여부와 residual을 기준으로 먼저 incoming check-to-variable 메시지를 갱신 후 해당 variable-to-check 메시지를 전파하는 순서로 스케줄링(알고리즘 7단계 절차).
7. 공정 비교를 위해 "equivalent iteration" `I_eq = round(2*Δupd/E + 1/2)`(식 6, `Δupd`=갱신 메시지 수, `E`=전체 edge 수)로 반복 횟수를 정규화.
8. 별도의 학습/최적화 절차 없음(GA/NN 없음); 스케줄링 규칙은 결정론적 heuristic.
9. 결과: VC-RBP 대비 BER=10⁻⁶ 지점에서 0.16dB coding gain, 약 25 iteration까지 더 빠른 수렴, flooding/LBP/VC-RBP 대비 적은 iteration에서 더 나은 BER/FER.
10. 한계: AWGN 채널·CCSDS 코드에 대한 시뮬레이션만 제공, HW 구현/gatecount/throughput 분석 없음; NAND 채널이나 QC-LDPC 구조 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Informed dynamic scheduling (unreliable node + residual 기반 갱신 순서) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 디코더 스케줄링 로직에 이식 가능한 위치이나 Prime ECC는 현재 flooding형 C2V→V2C + Dual-Update(odd/even) 구조로 완전 순차(residual 기반) 스케줄과 정합 필요 |
| Lazy schedule (girth 기반 확률적 variable 선택) | 대응 없음 | Prime ECC 모듈 지도에 girth/cycle 기반 스케줄링 대응 요소 없음 |
| Equivalent iteration 정규화 (`I_eq`) | 대응 없음 | Prime ECC의 iteration 카운트 방식과 별개의 평가 지표로 코드 이식 대상 아님 |
| Residual 계산 및 저장 | `decoder.cpp` `C2V_Cal()`, `Get_VNU_Table_Idx()` | VN 메시지 갱신 경로에 residual 추적 로직 추가 이식 가능하나 추가 메모리/비교 연산 필요 |

> 적용 가치: 중간 — waterfall 영역 오차성능·수렴속도 개선 기법으로 Prime ECC 디코더 이터레이션 루프에 개념적으로 이식 가능하나, 완전 순차(sequential) residual 기반 스케줄링은 Prime ECC의 병렬 HW 구조(z=32, HCU, GT)와 정합에 추가 설계가 필요.
