### B. 알고리즘 요약

1. 대상은 스트리밍 데이터 전송용 SC-LDPC(spatially coupled LDPC, protograph 기반 convolutional 구조) 부호; AWGN+BPSK 채널, 시뮬 코드는 길이 48384/19845, coupling width `ω`, lifting factor `M` 등으로 구성.
2. SC-LDPC는 BP 기반 sliding window decoding(SWD)으로 낮은 지연을 얻지만 SWD가 burst error를 유발하고 error propagation(윈도우 간 실패 전파) 문제가 있음.
3. 핵심 가정: RS 코드(outer, `F_{2^p}` 상의 MDS 코드)를 SC-LDPC(inner)와 concatenate하면 concatenated code의 최소 해밍거리가 커져 error floor를 낮출 수 있음.
4. 핵심 기법1(부호 구성): inner SC-LDPC를 locally systematic 인코딩(`V_t=[U_t, V_t^(1)]`)으로 설계, 유한 길이를 위해 partial syndrome `P_L` 기반 termination tail(길이 `τ=ω+1`)을 도입.
5. 핵심 식: `V_t H_0^T(t) + ... + V_{t-ω} H_ω^T(t) = 0`(식4)이 로컬 systematic 인코딩 조건, termination은 `F_0(L)`(정칙 정방행렬)의 역행렬로 tail을 계산(`V^(0)_[L,L+τ-1] = Ω_0·F_0^{-1}(L)`).
6. 핵심 기법2(디코딩): SWD-BM 디코딩 — window 크기 `W`로 log-BP SWD가 inner code를 복호, 결과를 outer BM(Berlekamp-Massey) 디코더가 복호해 성공 시 해당 심볼 LLR을 결정론적 값으로 되먹임(feedback).
7. 구현 디테일: outer code 선택에 따라 3가지 케이스(`L_I=L_II`, `<`, `>`)로 분류, 매칭되는(Case1) 경우가 최적 성능; window당 100회 log-BP iteration 수행.
8. 별도 학습 절차 없음(부호 구성은 protograph coupling + Fossorier 조건 기반 결정적 설계, girth ≥8 보장).
9. 결과: `(63,59)` RS 외부코드 결합 시 BER 10⁻⁸까지 error floor 미관측, RS-CC 대비 flooding schedule decoding(FSD, W=L)에서 BER 10⁻⁶에서 1.1dB 코딩이득, 단 FSD 복잡도는 `O(L²N_v²)`로 RS-CC의 `O(ΓLN_v)`보다 높음.
10. 한계: 순수 시뮬레이션(HW 미설계), SC-LDPC 스트리밍 구조 특유의 window decoding 전제, outer RS/BM 추가로 시스템 복잡도·지연 트레이드오프 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호 구성(protograph coupling, termination) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 QC-LDPC(블록) H-matrix 로드 구조로, convolutional/coupled 부호 설계·termination 로직 자체가 대응 없음에 가까움 |
| BP 기반 sliding window decoding(SWD) | 대응 없음 | Prime ECC 디코더는 블록 전체를 한 번에 처리하는 min-sum 이터레이션 루프로, 윈도우 슬라이딩 구조가 없음 |
| outer RS(BM 디코딩) 피드백 | 대응 없음 | Prime ECC는 조기종료용 CRC(`partial_CRC.cpp`)만 있고 외부 RS/BM concatenation 구조는 없음 |

> 적용 가치: 낮음 — SC-LDPC(convolutional 부호) + RS 외부코드 concatenation·sliding window 디코딩으로 Prime ECC의 QC-LDPC 블록 부호·min-sum 파이프라인과 구조적으로 이질적.
