### B. 알고리즘 요약

1. WIMEDIA UWB(MB-OFDM, 802.15.3a CM1~CM4 채널) 표준의 기존 convolutional code(K=7, rate 1/3~3/4)를 대체하는 QC-LDPC 부호, 고정 block length N=1200bit, rate R=3/4(고처리량용 N=9600bit 변형도 제시).
2. 문제: 기존 CC는 고rate(3/4)에서 diversity gain이 제한적이라 480Mbit/s 이상 고속·장거리 지원에 성능 부족, LDPC로 대체해 throughput 1.1~1.6Gbit/s, latency 2us 이하, PER<10^-3 목표.
3. 채널모델: IEEE 802.15.3a CM1(LOS, 0~4m)/CM2(NLOS, 0~4m), MMSE equalizer + ideal LLR 가정.
4. 핵심 기법 1단(부호설계): permuted `I×I`(I=30) identity matrix 기반 QC 구조에 modified PEG(progressive edge growth) 알고리즘으로 girth 최대화(평균 girth g=7.6, 대부분 cycle 길이 8), variable node degree distribution `f[3,2]={3/4,1/4}`, check node degree `g[11]={1}`인 초저밀도(ultra-sparse) 부호 채택.
5. 핵심 식: 체크노드 LLR 업데이트는 min-sum `Λ_k = sign(Π Λ_l)·min|Λ_l|`에 message scaling factor(MSF) 0.75를 곱해 성능 보정(Sum-Product 근사 오차 완화).
6. 핵심 기법 2단(디코더 아키텍처): layered decoding(TDMP 계열, Hocevar 방식) 채택 - CN 서브셋을 순차 처리하며 갱신된 메시지를 즉시 VN에 반영해 수렴속도 향상, VN 처리는 CN block(CNB) 내 FIFO로 흡수.
7. 부수 트릭/구현: 채널/extrinsic 정보 6bit 양자화, a-posteriori 정보(A-RAM) 9bit; permutation은 barrel shifter로 구현(I=30 병렬도, functional unit은 직렬 처리로 rate/length 유연성 확보); 조밀한 degree distribution(f[6,3,2])이 0.5dB 성능 이득이 있지만 면적 2배·layered latency 제약 위반이라 포기.
8. 학습/최적화 절차: modified PEG 알고리즘으로 identity matrix 배치/순열 최적화(별도 논문[14] 참조), density evolution은 언급만.
9. 결과: CM1 채널에서 LDPC(1200bit, 6iter)가 CC 대비 PER=10^-3에서 최대 2.5dB 이득(9600bit/10iter는 추가 2dB, 총 4.5dB), 1200bit 디코더 면적 0.189mm^2/705Mbit/s/1.70us(65nm@528MHz), 9600bit 디코더 0.606mm^2/1.14Gbit/s/8.37us, 인코더는 디코더 대비 7~10배 작고 최대 9.27Gbit/s.
10. 한계: rate 3/4 고정(다른 rate 미검증), UWB 특화 채널(CM1/CM2)만 시뮬레이션, HW 면적 절감을 위해 성능(최대 0.5dB) 희생한 트레이드오프.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG 기반 초저밀도 QC-LDPC 부호구조(H-matrix) | `ecc_top.cpp` `Load_PCM()` | 특정 rate 3/4·저밀도 degree 고정 부호라 Prime ECC 특정 H-matrix 대체는 재검증 부담 큼 |
| min-sum + message scaling factor(MSF=0.75) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min-sum CN 연산에 고정 스케일링 팩터 곱을 추가하는 간단한 이식 후보 |
| layered decoding(TDMP) 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 Dual-Update(even/odd 교대) 스케줄 보유, layered 방식과는 결이 다르나 수렴속도 개선 아이디어로 참고 가능 |
| barrel shifter 기반 permutation network (인코더/디코더 공용) | `encoder.cpp` `Generate_PCM_encoding()`, `decoder.cpp` pipeline 구조 | QC circulant shift 구현의 표준적 방식으로 Prime ECC z_sb=32 shift 구조와 유사, 신규 이식 요소는 제한적 |
| 6bit/9bit 양자화 파라미터 | `channel.cpp` `Set_LLR_Th()` | Prime ECC의 HD/2SD/3SD보다 세밀한 soft-4bit+ 양자화로, NAND read cost 트레이드오프 검토 참고자료 |

적용 가치: `중간` — 부호설계(PEG, 특정 rate/degree)는 이식 난이도가 높아 직접 재사용은 어렵지만, MSF 적용 min-sum과 layered decoding 스케줄링 아이디어는 Prime ECC 디코더 개선에 참고할 만함.
