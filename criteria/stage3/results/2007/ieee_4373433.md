> 총평: LDPC 디코더의 serial/layered scheduling이 flooding 대비 iteration을 절반으로 줄인다는 것을 계산 트리 성장 속도와 density evolution으로 엄밀히 증명한 이론 논문. HW 구현이나 특정 부호 설계는 없지만 스케줄링 개선의 이론적 근거로서 이식 가치가 있음.

### B. 알고리즘 요약 (10줄 내외)

1. 대상은 이진 LDPC 코드의 message-passing 반복복호 스케줄이며, 채널은 BEC 및 AWGN(Gaussian) 모두에서 분석.
2. 표준 flooding 스케줄(모든 VN 갱신 후 모든 CN 갱신)은 iteration당 정보 전파가 느려 수렴에 필요한 iteration 수가 많다는 문제를 다룸.
3. 핵심 가정은 cycle-free(tree-like) 국소 그래프 가정 하의 density evolution(DE) 표준 가정과 대칭 채널.
4. 핵심 기법은 serial-V(변수노드 순차 갱신)/serial-C(체크노드 순차 갱신) 스케줄이며, 두 phase(V→C, C→V)를 노드 단위로 interleave.
5. 계산트리(computation tree) 깊이가 flooding에서는 iteration당 `+1`씩 균등 성장하지만 serial 스케줄에서는 leaf별로 최소 `+2`씩 성장함(Eulerian number 기반 `Proposition III-5`)을 증명, 이것이 2배속 수렴의 근거.
6. 확장 기법으로 subset 수 `S`개로 나눠 순차 갱신하는 semi-serial 스케줄을 정의, `S`가 커질수록 serial 스케줄에 근접(`Proposition III-8`).
7. 구현 디테일: BEC에서는 `DE` 재귀식(`Theorem IV-A.1`, `Theorem IV-B.2`)으로 semi-serial/pseudo-serial 스케줄의 소거확률 진화를 계산, `p*(ε)` 고정점 분석.
8. 최적화/분석 절차는 DE 알고리즘(점근적 code ensemble 분석) + Azuma 부등식 기반 집중정리(concentration theorem)로 유한 길이 코드에서의 편차를 이론적으로 bound.
9. 결과: BEC/AWGN 두 채널의 (3,4), (3,6) 등 regular LDPC ensemble에서 semi-serial/serial 스케줄이 threshold 근방에서 flooding 대비 약 절반의 iteration으로 수렴(시뮬 Fig.1, DE Fig.6).
10. 한계: 순수 이론/시뮬 분석이며 HW 구현·gate count·throughput 수치 없음, 정정능력(BER curve) 자체의 개선은 다루지 않고 오직 수렴속도(iteration 수)만 분석함.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| serial-V/serial-C / semi-serial 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 Dual-Update(even/odd 교대) 스케줄과 같은 계열의 layered/serial scheduling 이론적 근거 |
| 계산트리 성장 속도 이론(2배속 수렴) | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 감소 근거로 참고 가능하나 코드 변경 없이 이론적 justification 수준 |
| DE 기반 수렴 분석(BEC/AWGN) | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 DE 점근분석 도구 자체는 없음 (참고 지표로만 활용 가능) |
```

적용 가치: **중간** — Prime ECC가 이미 Dual-Update 스케줄을 보유하고 있어 직접적인 알고리즘 이식 여지는 적으나, serial/semi-serial 스케줄의 서브셋 수(S)별 수렴 속도 트레이드오프 이론은 기존 Dual-Update 스케줄 최적화(서브셋 세분화 등)에 참고할 수 있음.
