### B. 알고리즘 요약

1. 시스템: IEEE 802.11n QC-LDPC(AA-LDPC) 3개 codeword 길이(648/1296/1944), 4개 rate(1/2,2/3,3/4,5/6), block size `Bi=27/54/81`.
2. 문제: block-serial 구조는 면적/전력에 유리하나 pipeline 시 layer 간 SO(soft output) 의존성으로 idle cycle이 생겨 decoding latency가 크게 증가.
3. 모델: 수평(check-centric) layered decoding(HLD), 2-output modified min-sum(mod-MS, `P=1`)으로 c2v 메시지 계산, `Φ(x)=-log(tanh(x/2))` 근사 대신 min 연산 사용.
4. 핵심 기법 1단: UOP(Unconstrained Output Processor) — 이전/다음 layer와 공유하는 SO 집합(`A_k`,`B_k`,`E_k`,`F_k`)을 정의해, 겹치는 메시지는 늦게 입력·빨리 출력하도록 CNU의 입출력 순서를 재배치.
5. 핵심 식: 최소 idle cycle `I_k = L_SO - (d_{k+1} - α_{k+1})`(식4), 감소차수(`Δd_k`)에 따른 추가 idle 고려해 `I_k = max{I_k, Δd_k, 0}`(식5), `L_SO`는 SO 데이터패스 latency(4~6 clock).
6. 핵심 기법 2단: 전체 idle cycle 합 `C(p)=ΣI_k`(식6)을 최소화하는 layer 처리 순서 `p̂ = argmin_p C(p)`를 탐색(최적 sequence of layers).
7. 부수 트릭: v2c 메모리를 dual-port 2뱅크(A/B)로 분할, 읽기 충돌 해소를 위해 일부 메시지를 두 뱅크에 중복 저장(redundancy 최대 33%); c2v 5bit, SO 7bit 고정소수점(FER 10^-4 목표).
8. 최적화 절차: cycle-true 모델 기반 오프라인 시뮬레이션으로 layer 순서 및 뱅크 중복도를 탐색(학습 아님, 결정론적 스케줄링 최적화).
9. 결과: idle cycle을 최적화 전 대비 20~46% 절감(예: N=648,r=1/2에서 Iorig=1236→I=95), 65nm 합성 0.48mm^2, 168~185mW, throughput 136~355Mbps.
10. 한계: 특정 802.11n H-matrix(prototype matrix) 구조에 맞춘 스케줄링으로 임의의 QC-LDPC에 자동 일반화되는지 불명, error-floor/정정능력 개선은 다루지 않음(순수 latency/HW 최적화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 수평 layered decoding + 2-output mod-MS(min-sum) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 min-sum/iterative message passing 보유, 스케줄 최적화만 차별점 |
| UOP 기반 idle cycle 최소화(입출력 순서 재배치) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, pipeline 6-stage 구조 | 직접 대응: pipeline stage 간 데이터 의존성 해소 기법으로 이식 난이도 낮음 |
| v2c 메모리 dual-bank + 중복 저장 최적화 | `channel.cpp`/`decoder.h` 관련 LLR/메시지 메모리 구조 (프로파일에 구체 함수명 없음) | 메모리 라우팅/뱅크 충돌 해소 기법, HW arch 레벨로 이식 난이도 중 |
| Graph Thinning(HW edge 선택)과 유사한 목적의 스케줄링 | `GT.cpp` `Make_GT_HW()` | 목적(클록당 idle/처리 edge 최소화)이 유사하나 구현 메커니즘은 다름 |

적용 가치: **높음** — layered decoding의 idle cycle 최소화(UOP)와 메모리 뱅크 최적화는 Prime ECC의 pipeline/HCU 구조에 직접 참고 가능한 latency 개선 기법.
