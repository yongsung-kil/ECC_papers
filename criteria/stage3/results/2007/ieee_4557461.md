### B. 알고리즘 요약

1. 시스템: regular `(γ,ρ)` LDPC 코드, BPSK+AWGN 채널, rate `Rc=K/N`, FG-LDPC 및 QC-LDPC(GF 기반 multi-fold dispersion 구성) 코드 실험.
2. 문제: 표준 WBF decoding은 BP 대비 성능 손실이 크고, 기존 WBF 변형들은 휴리스틱하게 제안되어 이론적 근거가 부족.
3. 핵심 가정: message-passing decoding의 1차 반복 로그우도 `L(c_n|y)`와 WBF의 flipping metric `E_n`이 `L(c_n|y) = β(1-2z_n)E_n` 관계로 연결됨(첫 반복에서만 성립).
4. 핵심 기법 1: min-sum decoding의 쌍대 WBF가 기존 IMWBF(Jiang et al.)와 동일함을 증명, `W_{n,m} = min_{i∈N(m)\n} |y_i|` 기반 flipping function `E_n = Σ_m (2s_m-1)W_{n,m} - α|y_n|`.
5. 핵심 식 의미: normalized/offset min-sum(NMS/OMS)의 보정 인자(곱/합)가 WBF의 flipping function에도 동일하게 대응되어, message-passing 계열 개선 기법(NMS, OMS)을 WBF로 자동 이식 가능함을 보임.
6. 확장: BP decoding의 쌍대 WBF(NBP-WBF/OBP-WBF)를 새로 유도, IMWBF 대비 성능 우위를 시뮬로 확인. 단 LP-WBF(고rate·large row-weight 코드에 강함)의 쌍대 message-passing decoder는 성능이 나빠 대칭성이 항상 성립하진 않음.
7. 부수 트릭: 병렬 구현(PWBF) — 각 unsatisfied check node가 가장 unreliable한 variable node 1개에만 "flip" 신호 전송, variable node는 신호를 누적한 flip-decision 값 `F_n`이 반복별 임계값 `F_th(k)`를 넘으면 병렬로 flip.
8. 최적화 절차: `F_th(k)` 임계값 벡터는 시뮬레이션으로 튜닝(학습/DE 아님, 수동/경험적 탐색).
9. 결과: (273,191) FG-LDPC에서 병렬 PWBF는 5 iteration만에 직렬 50-iteration 성능에 근접(약 10배 반복 감소); (1023,781)에서도 유사 경향. Rate-7/8 (4,32) QC-LDPC에서 LP-WBF가 IMWBF보다 고SNR에서 우세.
10. 한계: HW 설계 없음(순수 알고리즘/시뮬), 임계값 벡터 최적화 방법 미확립("open"으로 명시), regular LDPC 위주 분석.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum ↔ WBF 쌍대관계, NMS/OMS 대응 flipping function | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min-sum CN 연산과 동일 수학적 뿌리를 공유하나, WBF는 저정밀 hard-decision 대안으로 이미 보유한 soft min-sum과 중복·하위호환 관계 |
| 병렬 WBF(PWBF) 구현 프레임워크, 반복별 flip 임계값 `F_th(k)` | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, `ecc_data.h` `PARAM_LLR` | 반복별 threshold 테이블 구조가 Prime ECC의 iteration별 LLR threshold 테이블(`PARAM_LLR`)과 유사한 스케줄링 개념이나, WBF는 hard 판정 기반이라 soft LLR 갱신과 직접 대응 안 됨 |
| VN 처리 / 양자화(hard-decision 기반) | `decoder.cpp` `VN_Cal_HD()` | 대상이 hard-1bit 판정이라 Prime ECC의 HD 모드와 유사 |

> 적용 가치: 중간 — min-sum과의 쌍대관계 및 병렬 flip 임계값 프레임워크는 저복잡도 HD 디코더 설계에 참고 가치가 있으나, Prime ECC는 이미 soft min-sum(HD/2SD/3SD)을 보유하고 있어 WBF는 대체로 하위 호환적 대안에 그침.
