> 총평: full BP(sum-product, tanh) 알고리즘의 check node 연산에서 edge(bit node) 수를 적응적으로 줄여 복잡도를 낮추는 기법으로, Prime ECC 3.1은 이미 min-sum을 사용 중이라 직접적 중복기여이며 이식 가치는 제한적.

### B. 알고리즘 요약

1. 대상 코드: (816, 408) regular LDPC (VN degree 3, CN degree 6), AWGN+BPSK 채널, 표준 LLR-BP(log domain sum-product) 디코딩.
2. 문제: 표준 BP의 check node update `L(rji)=2·tanh^-1(∏tanh(L(qij)/2))`가 매 edge마다 tanh/tanh^-1 연산이 필요해 복잡도가 높음, min-sum은 복잡도는 낮지만 성능저하 큼.
3. 관찰: 신뢰도(LLR magnitude)가 매우 높은 bit node는 tanh 값이 1에 근접해 곱셈 결과에 거의 기여하지 않으므로 생략 가능(단 부호(sign)는 유지 필요).
4. 핵심 기법 1 (Algorithm 1): magnitude가 threshold `T`보다 큰 bit node는 CN update 계산에서 제외(부호만 반영), 모두 T 이상이면 min-sum처럼 최소값 사용.
5. 핵심 식: 총 사용 bit node 수 `(N-K)×(ρ-1-N_d,avg)` (BP는 `(N-K)×(ρ-1)`, min-sum은 CN당 1개) — threshold T=0이면 min-sum, T=1이면 표준 BP와 동일.
6. 핵심 기법 2 (Algorithm 2): 매 iteration마다 사용할 최소-LLR bit node 개수를 1개(min-sum)에서 시작해 parity check 실패 시마다 1씩 증가, 최종적으로 전체 노드 사용(표준 BP)까지 점증.
7. 구현 디테일: 두 알고리즘 모두 매 iteration 모든 bit/check node를 갱신(값 전달 생략이 아닌 연산 입력 edge 수만 축소)하며, forced convergence(노드 자체 skip)나 early detection[8]과는 다른 방식.
8. 학습/최적화 절차: 없음 (threshold T는 수동 설정, DE/GA 없음).
9. 결과: 표준 BP는 iteration당 12240 bit node 연산 필요 vs min-sum 2448개; 제안 알고리즘은 성능손실 거의 없이 사용 bit node 수를 크게 감소, 특히 고SNR에서 감소폭이 큼(저SNR에서는 이득 작음).
10. 한계: HW 구현/합성 없음, 소형 regular 코드(816,408) 단일 사례만 시뮬레이션, min-sum 대비가 아닌 full BP 대비 절감이라 이미 min-sum 사용 중인 시스템엔 적용 의미가 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Check node update의 선택적 edge 사용(Algorithm 1/2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min-sum(1~2개 최소값만 사용)이라 본 기법의 목표(BP→선택적 edge)와 중복·대체 관계, 신규 기여 미미 |
| full-tanh Sum-Product 대비 복잡도 절감 | 대응 없음(Prime ECC는 애초 tanh 미사용) | 프로파일 §5 "full-tanh Sum-Product"는 대응 없음으로 명시된 항목과 일치 |
| VN/CN 처리 구조 | `decoder.cpp` `VN_Cal_HD()` 등 | 일반적 BP 구조 대응이나 본 논문의 threshold 기반 edge selection은 min-sum 구조에 이식 시 재설계 필요 |

> 적용 가치: 낮음 — Prime ECC 3.1은 이미 min-sum(최소 근사)을 채택해 tanh 연산 자체가 없으므로, 본 논문이 해결하는 "BP의 tanh 복잡도"는 이미 해소된 문제이며 min-sum 대비 추가 이득은 불명확.
