> 총평: LDPC 코드의 trapping/stopping set 최소 크기 근사가 NP-hard임을 증명하는 순수 이론 논문으로, 부호설계나 디코더 알고리즘 제안이 없어 NAND ECC 이식 관점에서는 적용 대상이 아님.

### B. 알고리즘 요약

1. 대상은 일반 선형 블록코드 및 LDPC 코드의 Tanner graph `G=(L∪R,E)`, 채널별로 BEC(stopping set)/BSC(ZP-trapping set)/AWGN(trapping set) 실패 configuration을 정의.
2. 풀려는 문제: iterative decoder(BP/ER/ZP/product-sum)가 실패하는 최소 크기의 stopping/trapping set을 찾는 문제(`MINSTOP`, `MINTzp`, `MINT`, `MINTe`, `MINTc`)의 계산 복잡도를 규명.
3. 핵심 가정: 각 채널-디코더 쌍마다 stopping/trapping set의 조합적 정의(vertex degree parity 등)가 다르며, 이 정의들에 대해 근사 알고리즘 존재 여부를 다룸.
4. 핵심 기법: `MINSETCOVER`, `MAX3DMATCH`, `MINCODEWORD` 등 알려진 NP-hard 문제로부터 다항시간 환원(reduction)을 구성해 5개 trapping/stopping set 문제 전부가 NP-hard, 나아가 일부는 상수/로그 인자로도 근사불가임을 증명.
5. 핵심 식 예: `MINSTOP(G) = 2 + MINSETCOVER(S)`가 되도록 그래프 `G`를 구성 - stopping set 크기와 set cover 크기가 정확히 대응하도록 gadget(Zig, Or, Ballast) 설계.
6. 확장: BSC ZP-trapping set과 AWGN elementary trapping set(`MINTe`)에 대해서도 각각 좌-정규(left-regular) Tanner graph 구성과 3D-matching 환원을 별도로 제시.
7. 부수 기법: "Zig", "Or", "Ballast"라는 그래프 gadget을 도입해 원 코드의 최소 codeword 가중치 문제를 trapping set 문제로 정확히 매핑.
8. 학습/최적화 절차 없음 (순수 증명 논문).
9. 결과: 5개 문제 모두 NP-hard이며 `MINSTOP`은 `o(log n)` 근사불가, 나머지는 상수인자 근사불가(`O(1)`-hard); 이 결과는 sparse(LDPC) 코드 클래스에도 그대로 적용됨(Theorem 7~9).
10. 한계: 알고리즘/디코더/HW 제안 없음, 시뮬레이션 없음. 결론은 "trapping set enumeration 기반 error-floor 추정 기법이 정확한 worst-case 보장을 제공할 수 없다"는 이론적 함의뿐.
