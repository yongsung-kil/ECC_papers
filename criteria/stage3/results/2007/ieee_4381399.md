### B. 알고리즘 요약

1. 대상은 802.16e/3GPP 등 무선통신용 irregular LDPC 부호이며, 단일 mother code(R=0.5, N=1024 또는 R=0.3, N=1920)에서 puncturing으로 다양한 rate(최대 5/6~7/8)를 지원하는 rate-compatible 방식이다.
2. 기존 puncturing 기법([7] 등)은 복구 신뢰도(recovery reliability)만 최대화할 뿐 실제 복호 성능과의 상관성이 부족하고, puncturing 비율이 클 때 정지집합(stopping set) 형성 위험이 있다.
3. 채널은 AWGN, BPSK 변조, sum-product 복호(최대 50 iteration)를 가정한다.
4. 핵심 기법은 체크노드 c의 비용함수 `C(c) = Σ_{v∈Vc} s(v)`를 정의해 `max C(c)`를 최소화하는 노드 선택이며, `s(v)`는 pruned recovery tree `Tq(c)`에서 계산된 값이다.
5. `C(c)`가 작을수록 해당 체크노드가 SR(c)-step 후 제공하는 초기 신뢰도가 높아지므로, 모든 체크노드의 `max C(c)`를 최소화하면 puncture된 노드에 고르게 신뢰도를 분배할 수 있다.
6. 변수노드 비용함수는 `C(v) = r(v) + b(v)w^-1 + k(v)w^-2` (w=2n²)로 정의해 예약된 체크노드 수 r(v), 차수 b(v), 이웃 체크노드 비용합 k(v)를 우선순위대로 반영한다.
7. puncturing마다 정지집합 검사 알고리즘(Table I)을 실행해 영구 복구 불가능한 노드 발생을 방지하며, 필요 시 다음 후보로 대체한다.
8. mother code의 degree distribution은 density evolution(DE)과 progressive edge growth(PEG)로 사전 설계되었고, puncturing 알고리즘 자체는 최적화 학습 없이 결정론적 그리디 방식이다.
9. 결과: 제안 기법이 기존 [7] 대비 FER 성능 갭이 puncturing 비율 증가에 따라 최대 약 0.5dB(FER=10^-3) 개선되며, 고rate(0.9)에서도 error floor 없이 동작한다.
10. 한계: 특정 mother code(H-matrix) 구조나 HW 구현은 다루지 않고 시뮬레이션(FER)만 제시하며, puncturing이라는 특정 시나리오(가변 rate 지원)에 국한된 기법이다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Puncturing 노드 선택(cost function) | 대응 없음 | Prime ECC는 고정 rate QC-LDPC로 puncturing 기반 rate-compatible 구조를 사용하지 않음 |
| Pruned recovery tree / 정지집합 검사 | 대응 없음 | 부호 구성 단계 이론으로 디코더 실행 코드와 무관 |
| Mother code degree distribution (DE/PEG) | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 단계 참고 가능하나 Prime ECC는 특정 QC-LDPC 고정 구조 사용 |
| Sum-product 복호(비교 기준) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 알고리즘 자체는 무관, 성능 비교 기준일 뿐 |

적용 가치: **낮음** — rate-compatible puncturing이라는 무선통신 특화 시나리오의 부호 구성법 논문으로, NAND 고정-rate binary QC-LDPC ECC와 직접적 접점이 없음.
