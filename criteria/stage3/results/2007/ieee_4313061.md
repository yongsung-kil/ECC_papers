### B. 알고리즘 요약

1. 대상은 저장매체 오류가 아니라 메모리 회로 자체(레지스터·XOR·majority 게이트)가 transient fault를 갖는 상황에서의 정보 저장; (n,γ,ρ) regular binary LDPC 코드로 n비트 레지스터에 코드워드를 저장.
2. 문제의식: 기존 Taylor-Kuznetsov(TK) 방식은 무한 길이에서만 성립하는 존재성 증명이며, 저자들은 유한 길이 expander code 기반의 구성적(constructive) 방법을 제시.
3. 핵심 가정: adversarial(최악 경우) bit-flip 결함 모델 — 레지스터 결함률 `α_m`, XOR게이트 결함률 `α_⊕`, majority게이트 결함률 `α_γ` 각각 매 시행마다 독립적/일정 비율 이하로 발생.
4. 핵심 기법: Sipser-Spielman parallel bit-flipping을 변형한 `Algorithm A` — 매 시각 variable node는 이웃 check node로부터 받은 추정치(±1 XOR 다수결) 중 과반(`γ/2` 이상)이 다르면 값을 flip.
5. 핵심 정리(Theorem 1): `(γ,ρ,α,(3/4+ε)γ)`-expander 그래프 상에서 `α_m + γ(ρ-2)α_⊕ + α_γ < α(1+4ε)(4ε)/2` 조건이면 모든 시간에서 오류 비율이 유계로 유지되어 메모리 실패가 발생하지 않음을 증명.
6. 복잡도/중복도(redundancy) 계산: correcting circuit은 노드당 `γ(ρ-2)`개 2-input XOR게이트 + 1개 γ-input majority게이트, 전체 redundancy `R ≤ (1+D_γ+γ(ρ-2))/(1-γ/ρ)`로 rate 1/2가 최소 redundancy를 준다는 예시 도출.
7. 부수 트릭: check node 추정치를 만족/불만족 이진 신호 대신 실제 이웃값 추정치로 대체(Algorithm A)해 TK 대비 redundancy 절감.
8. 학습/최적화 절차 없음. 대신 explicit expander construction(Capalbo et al.) 존재성 이용, 수치적으로 상/하한(`α_total`) bound만 계산.
9. 결과: γ=9, γ=34에 대한 redundancy-ρ 곡선 및 α_total 상/하한 곡선(수치 그림, 본문 서술 수치 없음)만 제시. 정량적 throughput/gate count 실측 없음.
10. 한계: 순수 이론적 존재성/구성적 증명이며 HW 미설계, 실측 없음. 독립 결함 모델은 향후 연구로 남김, irregular LDPC·message-passing 디코딩으로의 확장도 future work.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| parallel bit-flipping (hard-decision) 디코딩 | 대응 없음 | Prime ECC는 min-sum(soft) 기반이며 순수 bit-flipping 알고리즘은 보유 기법 목록에 없음 |
| 결함 있는 논리회로 자체의 신뢰성 이론 | 대응 없음 | Prime ECC는 신뢰 가능한 HW를 가정한 저장매체 ECC 시뮬레이터로 회로 결함 허용은 스코프 밖 |
| expander graph 기반 H-matrix 구조 | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체와는 관련 있으나 QC-LDPC 고정 구조와 직접 연결되지 않음(간접) |

> 적용 가치: 낮음 — 회로 결함 허용(fault-tolerant computing) 이론이며 NAND 저장매체용 soft-decision min-sum ECC와 목적·디코더 방식이 근본적으로 다름.
