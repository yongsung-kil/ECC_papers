### B. 알고리즘 요약

1. 대상: rate-1/2 protograph 기반 LDPC, 입력 블록크기 `k=64,128,256` bit (위성 command/control 짧은 프레임용).
2. 문제: 기존 RA/IRA/ARA류 protograph 부호는 반복복호 임계값(threshold)은 낮지만 최소거리가 sublinear라 error floor가 높음.
3. 가정: degree-2 variable node를 배제하면 최소거리가 블록크기 `n`에 선형으로 증가한다는 성질(δmin>0)을 이용.
4. 핵심 기법: variable node degree 3~5인 2-node 소형 protograph를 기본 구조로 채택, `copy-and-permute`로 확장(lifting factor 4, 이후 16).
5. Lifting 시 progressive edge growth(PEG)와 ACE 알고리즘으로 circulant(크기 16) 위상을 선택해 병렬 edge 제거 및 girth 확보.
6. 반복복호 임계값 계산은 정확한 density evolution 대신 reciprocal channel approximation(RCA, 편차 <0.005dB)로 근사.
7. Degree(3,4) 프로토그래프 대비 degree(3,5)는 최소거리 12→14로 증가, undetected error rate 추가 감소.
8. 최적화 절차: 최소거리·stopping set 크기를 최대화하되 최대 variable node degree를 제한하는 조건 하에 설계(자동 최적화 알고리즘 명시는 없음, 수작업 protograph 선택 + PEG/ACE lifting).
9. 결과: k=64, n=128 부호의 `dmin=12`(degree 3,4) 또는 `14`(degree 3,5), 최대 undetected error rate `<3×10⁻⁵`(k=64); k=256에서 AR4JA·(512,256) 대비 waterfall은 0.25dB 열세지만 error floor는 우수.
10. 한계: HW 구현/합성 없이 시뮬레이션 결과만 제시, 위성 통신 채널(AWGN) 및 매우 짧은 블록길이에 특화되어 NAND 채널·codeword 크기와 상이.
