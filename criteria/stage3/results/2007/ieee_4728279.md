### B. 알고리즘 요약

1. 대상 문제: LDPC 코드 성능 평가는 이론이 없어 시뮬레이션에만 의존하는데, 긴 블록길이·많은 블록 수 시뮬레이션은 단일 머신에서 시간이 매우 오래 걸림.
2. 채널 모델: AWGN + BPSK 변조, 표준 컨벤션 (`s = sqrt(2*R*Eb/No)`).
3. 디코딩 알고리즘: 표준 sum-product algorithm(SPA/BP)을 교과서 그대로 사용 (check node update `L(rji)`, variable node update `L(qij)`, `L(Qi)` 추정, `phi(x)=log(tanh(x/2))` 근사 없음).
4. 핵심 기법: 알고리즘 개선이 아니라 "Simulation Controller + Processing Nodes" 구조의 분산 프로토콜을 설계. Controller가 H-matrix/SNR/iteration 파라미터를 다수의 유휴 네트워크 PC(최대 126대)에 배포하고 부분 결과를 주기적으로 수집.
5. 프로토콜: Stage1(노드 탐색·버전 확인), Stage2(파라미터 전송), Stage3(주기적 부분결과 수집, `t=T/n` 간격)로 3단계 controller 상태머신.
6. 부수 트릭: 전송 블록을 all-zero codeword로 고정하면 인코딩 단계 생략 가능해 시뮬레이션 가속.
7. 별도 학습/최적화 절차 없음 (순수 분산 실행 인프라).
8. 결과: length-1024, rate-1/2 코드, SNR=2.5dB, 1000만 블록 시뮬레이션이 단일 노드 14.5시간에서 126노드 사용 시 6분으로 단축 (노드 수에 거의 선형 비례).
9. 한계: 부호설계·디코더 알고리즘·HW 어느 것도 개선하지 않음. 순수 소프트웨어 시뮬레이션 가속 인프라이며 실제 ECC 성능/복잡도와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 표준 sum-product 디코딩 알고리즘 | 대응 없음 | Prime ECC는 이미 min-sum 기반이며 논문은 이보다 낮은 성능의 full-tanh SPA만 다룸, 새로운 기법 없음 |
| 분산 네트워크 시뮬레이션 프로토콜 | `mpi.cpp` | 코드 내 MPI stub과 개념적으로 유사한 "여러 노드에 시뮬 분산" 목적이나 논문은 TCP/IP 소켓 기반 자체 프로토콜로 실질적 이식 대상 아님 |

> 적용 가치: 낮음 — 부호설계·디코더·HW 어느 레이어도 개선하지 않는 시뮬레이션 인프라 논문으로 Prime ECC 알고리즘/HW 이식과 무관함.
