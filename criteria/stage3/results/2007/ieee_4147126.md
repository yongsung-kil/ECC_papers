### B. 알고리즘 요약

1. Rate-compatible array LDPC 부호(1a/1b식)는 prime `p`, check-node degree `k`, variable-node degree `j`로 정의되며 최좌측 p열/최상단 p행을 순차 puncturing해 rate 0.666~0.91을 지원.
2. 기존 TDMP는 BCJR 기반 CNU를 사용해 내부 FIFO가 많고 면적이 크다는 문제를 offset min-sum(OMS)으로 대체해 해결.
3. 채널은 BPSK+AWGN, soft LLR 기반 message passing(TPMP/TDMP) 사용.
4. 핵심 기법은 layered decoding(TDMP)에 OMS를 결합: check-node 메시지 `R_mn = δ_mn * max(κ_mn - β, 0)`, `κ_mn`은 이웃 Q메시지 중 최소값.
5. OMS의 value-reuse 성질(check-node가 M1, M2 두 값만 산출)로 partial state(PS)/final state(FS) 구조를 만들어 R 메모리를 50~90% 절감.
6. Block-serial CNU 마이크로아키텍처: PS 스테이지(k 클록)→FS 스테이지(2클록)→R select 스테이지의 3단 파이프라인.
7. 구현 디테일: cyclic shifter를 log2(p) 스테이지의 logarithmic shifter로 구성해 Omega network 대비 면적/배선 절감, 5-bit(R,Q)/6-bit(P) 균일 양자화 사용.
8. 최적화 절차 없음(density evolution으로 β=0.15는 선행 연구[3] 값 인용).
9. 결과: 총 메모리 82%(TPMP-SP 대비)/72%(TDMP-BCJR 대비) 절감, router 50% 감소, throughput 2배 증가, ASIC 130nm 5.9Gbps, FPGA 1.36~2.27Gbps.
10. 한계: array LDPC 전용 구조 의존, BER 성능 개선 자체는 없음(기존 TDMP와 동일 수렴, finite precision으로 인한 0.2dB 열화만 존재), 정정능력 향상은 목표가 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset min-sum CNU (PS/FS 2-value 재사용) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 PS/FS value-reuse 아이디어는 CNU 메모리 최적화에 참고 가능 |
| Layered decoding(TDMP) 스케줄 | `decoder.cpp` `LDPC_Decoder()` | 이미 Dual-Update 등 레이어드 스케줄 보유, 신규 기여는 아님 |
| Cyclic shifter 단일화(logarithmic shifter) | 대응 없음 | Prime ECC는 QC circulant shift 구조가 다르므로 array LDPC 전용 shifter 설계는 직접 대응 없음 |
| 5/6-bit 양자화, LLR threshold | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 양자화 비트수 설계 참고 가능하나 이미 자체 LLR 테이블 보유 |
| Array LDPC H-matrix 구성(1a/1b) | `ecc_top.cpp` `Load_PCM()` | 부호 자체가 array LDPC로 Prime ECC의 QC-LDPC base matrix와 상이, 재검증 부담 큼 |

적용 가치: 중간 — CNU의 value-reuse 메모리 절감 아이디어는 참고할 만하나 array LDPC 고유 구조와 이미 보유한 layered decoding·min-sum 기법과 중복도가 높아 신규성은 제한적.
