### B. 알고리즘 요약

1. IEEE 802.11n 표준의 AA-LDPC 코드(N=648/1296/1944, rate=1/2,2/3,3/4,5/6, block size B0=27/B1=54/B2=81)를 대상으로 한 layered(HLD) 디코더 설계.
2. 기존 block-serial HLD 구현은 파이프라인 해저드(SO가 갱신 전에 재사용됨)로 인해 idle cycle이 많이 필요해 throughput이 제한되는 문제.
3. HLD는 레이어(super-code) 단위로 순차 갱신하며 c2v 메시지 `Δ_m,n`와 soft-output `y_n`을 갱신하는 표준 layered BP.
4. 핵심 기법: Reversed Output Processor(ROP) - 레이어 k와 k-1의 공통 SO(`A_k`)를 마지막에 acquire하고 출력 순서를 뒤집어(LIFO 버퍼) 다음 레이어에 더 빨리 제공.
5. idle cycle 수 `I_k = max{I_k, Δd_k, 0}`이며, 여기서 `Δd_k = d_k - d_{k+1}`은 레이어 간 차수 차이 - 이 식으로 파이프라인 해저드와 차수 감소 문제를 동시에 해결.
6. 추가로 레이어 처리 순서(permutation)를 cost function `C(p) = Σ I_k`를 최소화하도록 완전탐색으로 최적화.
7. CN 코어는 2-output modified min-sum[12] 사용, v2c 메시지는 최대 CN 차수(dc,max=22) 크기의 LIFO 시프트레지스터로 버퍼링.
8. 코드 구조를 활용해 B0(=27) 단위 클러스터로 NP를 구성, 짧은/중간/긴 코드워드는 1/2/3개 클러스터로 처리(Multi-Size Circular-Shifting Network).
9. 결과: idle cycle 비율 0~14.5%(최적화 전 대비 최적화 후 약 2자릿수 감소), throughput 178~410 Mbps, 전력 235mW 이하, 65nm 합성.
10. 한계: 특정 표준(802.11n)의 고정 코드 구조에 최적화되어 일반 QC-LDPC로의 일반화는 제한적이며, 정정능력 개선은 다루지 않음(순수 스케줄링/HW 최적화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| HLD 이터레이션 루프, 레이어 처리 순서 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC도 layered(Dual-Update) 스케줄 사용, idle-cycle 최적화 아이디어는 파이프라인 스케줄링에 참고 가능 |
| ROP(v2c LIFO 버퍼, 출력 역전) | `decoder.cpp` `C2V_Cal()` | 파이프라인 해저드 해결 방식이 상이(HCU/GT와 다른 접근), 직접 이식보다는 참고용 |
| 2-output modified min-sum CN 코어 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 Prime ECC에 유사 min-sum 계열 보유, 중복 기여 |
| MS-CS 네트워크(circular-shifting) | (대응 없음) | Prime ECC의 z=32 lifting/shift 구조와 다른 rotation 방식, 직접 대응 모듈 없음 |
| 레이어 순서 최적화(cost function 완전탐색) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 시 레이어 순서 결정 로직에 참고 가능하나 실존 대응 함수 없음 |

> 적용 가치: 중간 - 레이어드 디코딩 idle cycle 최소화 아이디어(ROP, 레이어 순서 최적화)는 HW 파이프라인 latency 개선에 참고할 만하나, 802.11n 특정 코드 구조 종속적이고 정정능력 개선은 없어 우선순위는 높지 않음.
