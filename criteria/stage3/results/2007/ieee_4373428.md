### B. 알고리즘 요약 (10줄 내외)

1. 대상은 두 개의 identical recursive systematic convolutional encoder(RSC)를 pseudorandom interleaver로 병렬 연접한 turbo code이며, 채널은 binary erasure channel(BEC).
2. LDPC는 BEC에서 stopping set으로 복호 실패가 정확히 특성화되나, turbo code는 이런 특성화가 없었던 문제를 해결하고자 함.
3. 채널 모델은 단순 BEC(2-입력 3-출력 DMC, 소거확률 `ε`)이며 잡음/간섭은 고려하지 않음.
4. 핵심 기법은 "turbo stopping set" 정의: 두 constituent code의 부분부호 `C1'`, `C2'`(dimension>0)의 support set이 인터리버로 서로 매핑되는 집합.
5. 정리(Theorem 1): turbo decoding을 반복하다 더 이상 진행 못하면 남는 소거 위치 집합은 유일한 최대크기 turbo stopping set과 일치.
6. 확장 기법으로 "improved turbo decoding"을 제안: 미결정 systematic bit 하나를 추정(guess)하여 복호를 재시도, 불일치(inconsistency) 발생 시 값을 바꿔 재시도.
7. bit-position 선택은 forward/backward pass의 legal vertex 개수(`w_l(t)`)를 이용해 최대한 많은 위치를 동시에 해제할 수 있는 위치를 고름.
8. Turbo stopping set size enumerating function(TSSEF)을 uniform interleaver 가정 하에 constituent code의 SIRSEF로부터 유도, Garello et al.(2001) trellis 기반 알고리즘을 확장해 특정 인터리버의 소크기 turbo stopping set을 열거.
9. 결과: (155,64,18) turbo code 예제에서 크기 17짜리 turbo stopping set 존재(최소거리 18보다 작음); improved decoding은 near-ML 성능 달성.
10. 한계: HW 미설계, 시뮬/이론 분석뿐이며 turbo code(convolutional 기반) 전용 분석으로 LDPC 부호에는 직접 적용 안 됨(저자도 다른 turbo-like code로의 확장만 future work로 언급).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| turbo stopping set / turbo decoding 알고리즘 전체 | 대응 없음 | turbo code(BCJR 기반) 전용이며 QC-LDPC min-sum 디코더 구조와 무관 |
| BEC 채널 모델 | 대응 없음 | Prime ECC는 AWGN/RBER/fixed-error 채널이며 BEC 소거 모델 미지원 |
| improved decoding(bit guessing) | 대응 없음 | LDPC min-sum iteration 루프와 알고리즘 구조 자체가 다름(trellis forward-backward 기반) |
```

적용 가치: **낮음** — turbo code(convolutional 병렬연접) 전용 유한장 분석이며 NAND용 binary QC-LDPC의 부호설계/디코더/HW 어느 레이어와도 대응되지 않음.
