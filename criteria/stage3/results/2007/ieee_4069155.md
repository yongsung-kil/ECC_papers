### B. 알고리즘 요약

1. BIOSM(binary-input output-symmetric memoryless) 채널 상의 irregular LDPC 부호, 길이 `n≈1000~10000`, rate는 puncturing으로 `0.5~0.97` 범위까지 가변.
2. 표준 반복복호(Algorithm A)가 실패할 때 finite-length 코드의 성능을 개선하는 것이 목표 - 짧은 길이에서 stopping set에 걸려 복호 실패가 잦은 문제.
3. 채널을 BEC/BIAWGN 등 일반 BIOSM으로 모델링, `f(x)=+1 if x>=0, -1 if x<0`로 하드 판정 변환.
4. Algorithm D: 표준 복호 실패 시 unsatisfied check node에 가장 많이 연결된 변수노드(최대 5개)를 골라 LLR을 `+∞`/`-∞`로 강제(guess) 후 재복호, 성공할 때까지 값 조합을 바꿔 반복.
5. guess 대상 선택 기준은 인접 unsatisfied check node 수 `d_v`가 가장 큰 노드 - 정보 부족 영역을 우선 보정한다는 의미.
6. Punctured LDPC: 부호어 일부 비트를 전송하지 않고(LLR=0) 수신측에서 위치만 아는 상태로 복호, puncturing 비율이 임계값(threshold) `p*`를 넘으면 오류확률이 채널과 무관하게 0에서 멀어짐(threshold effect) 증명.
7. 부수 기법: BEC 전용 Algorithm B/C(guess를 stopping set 내에서만 수행)는 계산량이 선형/저복잡도로 효율적이나 non-BEC 채널로 일반화 불가.
8. 최적화 절차 없음 - degree distribution은 기존 앙상블(예: `(3,6)` regular, 특정 irregular 앙상블) 사용, random puncturing이 intentional puncturing과 점근적으로 동일 threshold임을 증명(Fact 1).
9. 결과: BIAWGN 길이 1000 코드에서 Algorithm D가 BER `10^-5`에서 Algorithm A 대비 0.35~0.5dB 이득, punctured half-rate 부호가 최대 rate 0.97까지 capacity 대비 0.7dB 이내 gap 달성(기존 [Ha-McLaughlin] 대비 개선).
10. 한계: HW 설계 전무, guess 재시도로 인한 worst-case 복호시간이 평균 대비 최대 38.4배로 증가(지연 예측 불가), 결과는 전부 시뮬레이션/점근적 앙상블 분석이며 특정 유한장 코드의 실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Algorithm D (실패시 변수노드 guess 후 재복호) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 조기종료/재시도 로직과 유사하나 guess 재시도는 worst-case latency 증폭이라 NAND HW에는 부적합 |
| Punctured/rate-compatible LDPC 설계 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC 고정 설계라 이식 난이도 높음, 대응 없음에 가까움 |
| BEC 전용 stopping-set 기반 Algorithm B/C | 대응 없음 | NAND 채널은 BSC/AWGN형이라 BEC 전용 기법은 직접 적용 불가 |

적용 가치: 낮음 - 이론적 threshold 증명과 BEC 특화 기법이 중심이라 NAND binary QC-LDPC HW 이식 시 직접 재사용 가능한 요소가 거의 없음.
