### B. 알고리즘 요약

1. Rate 0.8 array LDPC 부호(expansion factor `p=67`, check-node degree 25, variable-node degree 5, block length 1675)를 block-fading AWGN 채널에서 Offset-Min-Sum 레이어드(TDMP) 디코딩으로 시뮬레이션.
2. 문제: 기존 고정 반복(fixed-iteration, 예: 20회) 디코더는 데이터 프레임 품질과 무관하게 항상 최대 반복·최대 전압으로 동작해 에너지를 낭비함.
3. 핵심 관찰: 수신 프레임에서 하드결정 후 계산한 `cH^T`(check-error 개수)가 SNR에서 Gaussian 분포를 따르며, check-error 개수가 많을수록 필요 반복횟수(sub-iteration)가 통계적으로 증가.
4. 핵심 기법: check-error 개수에 대해 경험적 임계값 `Check_Err_Threshold(i)` (i=1,2,3)를 설정하고, 각 구간에 대응하는 고정 반복횟수 `num(j)`(예: 60/45/30/15)를 사전 할당하는 speculative 스케줄링.
5. 임계값 선정 기준: 각 check-error 구간의 평균 반복횟수 `E`와 표준편차 `σ`에 대해 `E+2σ` 지점을 반복횟수 상한으로 채택(예: 55, 42, 27, 16 → 클록 분주 편의상 60/45/30/15로 조정).
6. 반복횟수가 정해지면 이에 대응하는 클록 주파수(167/125/84/42MHz)와 전압을 4단계 중 선택, 프레임당 총 디코딩 시간이 일정(constant-time decoding)하도록 함.
7. 구현 디테일: `cH^T` 연산은 기존 디코더의 조기종료 판정 하드웨어를 재사용해 추가 오버헤드 없음; PLL 대신 clock divider로 주파수 전환, 별도 입력 버퍼 불필요.
8. 임계값·반복횟수 세트는 시뮬레이션 기반 경험적 튜닝(학습 알고리즘 아님).
9. 결과: TSMC 0.13μm 합성, 1.5V에서 175MHz 동작 확인; SNR 3.8~4.4dB에서 최대 37% 디코딩 에너지 절감(BER/FER 성능 저하 거의 없음, checkErr_threshold2=130, threshold3=115일 때 BER 1.4×10⁻⁵).
10. 한계: 무선 block-fading 채널 및 다중 전압 공급(DVFS) HW 전제, 정정능력 자체를 개선하지 않고 순수 전력관리이며 예측 오류 시 coding gain 손실 트레이드오프 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check-error 개수 기반 반복횟수 사전 예측 | `decoder.cpp` `LDPC_Decoder()` | 조기종료 판정용 syndrome 계산 로직과 유사 개념이나, 목적이 정정성능이 아닌 DVFS 전압/클록 스케줄링이라 우리 구조엔 직접 대응 없음 |
| DVFS 다중 전압/클록 스케줄링 | 대응 없음 | Prime ECC는 고정 클록 bit-exact HW 시뮬레이터로 전압 스케일링 개념 자체가 코드에 없음 |
| 조기종료(early termination) 판정 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 반복 감소 취지는 유사하나 본 논문은 CRC가 아닌 check-error 카운트 기반 사전예측이라 메커니즘이 다름 |

> 적용 가치: 낮음 — 무선 fading 채널·DVFS 다중전압 HW 전제의 저전력 스케줄링 기법으로 NAND 컨트롤러의 고정전압 ECC HW 구조와 목적·전제가 맞지 않음.
