### B. 알고리즘 요약

1. (J,K)-regular QC-LDPC 코드: 크기 m×m circulant permutation sub-matrix J×K개로 구성된 parity-check matrix `H`, rate `R ≥ 1-J/K`.
2. 문제: 기존 overlapped message passing(선행연구 [8])은 특정 QC-LDPC 구조에서만 throughput 이득이 제한되고, highly-parallel 아키텍처([14])에는 직접 결합이 어려움.
3. 채널/노이즈 모델은 별도 정의 없이 표준 BP(sum-product/min-sum) 반복 복호 프레임워크를 그대로 사용.
4. 핵심 기법: `H`를 재구성(`H̄`)해 첫 VNUG1과 CNUGJ의 갱신을 인접 iteration의 CNUG/VNUG와 block 단위로 부분 중첩(`m/h`, `m/h+1` 클록 스케줄).
5. `H̄`에서 첫 행/마지막 열 sub-matrix를 `I0`(단위행렬)로 고정해 VNUG1과 CNUGJ 연산이 항상 이웃 그룹과 겹칠 수 있도록 구조를 맞춤.
6. 확장: MUX1/MUX2 스위치로 CNUGJ와 VNUG1의 갱신 메시지를 메모리에 저장하지 않고 직접 다음 그룹(CNUGJ-1, VNUG2)으로 우회 전달.
7. 구현 디테일: h-parallel CNU/VNU 그룹(CNUG/VNUG)마다 Benes/Banyan 또는 특수 6×6 스위치 네트워크로 메시지 순서를 재정렬.
8. 별도 학습/최적화 절차 없음 (구조적 재배치 설계).
9. 결과: (3,5)-regular 예시에서 CNU 약 33%, VNU 약 20% 하드웨어 복잡도 절감, 메모리 접근량 100%→80%로 감소, 성능 저하 없음(주장).
10. 한계: 성능 저하 없음이라는 주장에 대한 시뮬레이션/BER 결과 없음, ASIC 합성이나 실제 gatecount 수치도 제시되지 않은 순수 아키텍처 제안 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNUG/VNUG 부분 중첩 스케줄링 | `decoder.cpp` `LDPC_Decoder()` (파이프라인 6-stage 구조) | 병렬 그룹 간 스케줄 겹침 아이디어가 파이프라인 스테이지 설계와 유사한 목적 |
| MUX 기반 메시지 우회(메모리 접근 절감) | `GT.cpp` `Make_GT_HW()` | 유사 목적(엣지/메시지 선택으로 처리량 절감)의 MUX 기반 HW 기법이 이미 존재 |
| 메모리 뱅크 접근 감소 | `decoder.cpp` (메모리 기반 메시지 저장 구조) | 메모리 접근 비율 절감 아이디어는 관련 있으나 코드 상 구체 대응 함수 불명 |
| h-parallel 스위치 네트워크(Benes/Banyan) | 대응 없음 | Prime ECC는 z=32 고정 row 병렬 구조로 임의 h-parallel 스위치 네트워크 개념 없음 |

> 적용 가치: 낮음 — (3,5)-regular 특정 예시의 이론적 HW 절감 제안이며 실제 검증(합성/시뮬)이 없고, Prime ECC는 이미 유사 목적의 GT/HCU/pipeline 최적화를 보유해 중복 기여도가 큼.
