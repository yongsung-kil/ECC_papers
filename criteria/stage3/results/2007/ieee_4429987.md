### B. 알고리즘 요약

1. 대상은 LDPC 컨볼루션 코드(LDPCCC): syndrome former 행렬 `H^T`가 대각·semi-infinite 구조, rate `R=b/c`, memory `ms`로 정의되는 시간불변(time-invariant) QC 기반 컨볼루션 부호.
2. 기존 LDPC 블록코드 디코더는 인터리버가 블록크기·서브블록에 종속돼 커지면 복잡해지는 문제가 있어, 그래프 규칙성을 활용한 유연한(programmable) 병렬 아키텍처가 필요.
3. 채널은 일반 AWGN 가정, 디코딩은 min-sum 기반 반복 메시지패싱(`mij`, `mji`)으로 수행.
4. 노드레벨 병렬화(node-level parallelization): 변수노드를 길이 `pt`의 겹치지 않는 processing window로 그룹화해 SIMD로 동시 처리, `pt=64`로 구현.
5. Check node update는 `mji = sign·min1/min2` 근사(식 3, 8): 최소값 2개(MIN1, MIN2) 비교로 1 cycle에 갱신.
6. 체크노드 연산 시 메모리 정렬이 어긋나 barrel shifter(3단 계층 구조)로 벡터 재정렬 필요, `S = log_D[H^T(D)] mod pt`로 회전량 결정.
7. 프로세서는 STA(synchronous transfer architecture) 기반, VLIW(127-bit) 명령어로 주소생성부/데이터패스부 병렬 동작, 8-bit soft-value 양자화.
8. 프로그램은 INITIAL/CHECKOP/BITOP 등 섹션으로 구성되며 필요 VLIW 수는 `N_VLIW = 4JK+3J+6K+32`로 코드 파라미터(J,K)에 따라 결정, 소프트웨어만 교체하면 다른 (ms,J,K) 코드도 지원.
9. UMC 130nm 합성 결과 200MHz, 1.2MGates, 7.83mm², 437mW@2.8dB(BER≈10^-3), throughput은 수백 Mbit/s 수준(정확 수치는 그림에만 존재).
10. 한계: 정규(regular) LDPCCC만 실증, 고정 반복횟수만 구현(조기종료 없음), 고차 코드용 확장성은 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum check node update (min1/min2) | `decoder.cpp` `CNU_Update_New_Mag()` | 알고리즘 개념은 동일(min-sum)이나 Prime ECC는 이미 보유한 기법으로 신규성 낮음 |
| SIMD 노드레벨 병렬화(processing window) | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 QC-LDPC lifting(z=32) 기반 병렬화로 구조가 다름, 직접 대응 없음 |
| barrel shifter 기반 vector 재정렬 | `GT.cpp` `Make_GT_HW()` | 개념적 유사성(edge 정렬/재배치)만 있고 실제 대응 없음 |
| LDPCCC 부호구조(syndrome former, semi-infinite) | `ecc_top.cpp` `Load_PCM()` | 부호 자체가 QC-LDPC 블록코드가 아니므로 대응 없음 |

> 적용 가치: 낮음 — 부호종류가 컨볼루션형(LDPCCC)으로 Prime ECC의 QC-LDPC 블록코드 구조와 근본적으로 다르고, HW 기법(SIMD, barrel shifter)도 무선통신용 범용 프로세서 아키텍처 지향이라 NAND ECC HW로 이식하기엔 재설계 부담이 큼.
