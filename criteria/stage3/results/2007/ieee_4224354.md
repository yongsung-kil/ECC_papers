### B. 알고리즘 요약

1. 유한 block length에서 큰 girth를 갖는 LDPC를 구성하는 PEG(Progressive Edge-Growth) 알고리즘 개선. rate-1/2 (1008,504) regular(`ds=3,dc=6`) 및 irregular(최대 symbol-node degree 11, density evolution 최적화 분포) 부호에 적용.
2. 문제: 표준 PEG는 symbol-node degree만 자유롭게 지정 가능하고 check-node degree는 "가능한 균일하게"만 시도할 뿐 엄격히 보장하지 않음(예: rate-1/2 (1008,504)에서 실제로는 degree 5,6,7 세 종류 혼재). 엄격히 집중된 check-node degree(`ρ(x)=ρx^d+(1-ρ)x^(d-1)`)가 수렴속도 최대화와 반복복호 HW 구현(엔트리 종류 최소화)에 유리함이 기존 연구([6])로 알려짐.
3. 채널/모델: AWGN, sum-product/iterative 반복복호(80 iterations), 표준 PEG 프레임워크(symbol node에서 tree 확장, girth 최대화 edge 선택) 위에서 동작.
4. 핵심 기법 1단 - free check-node degree: 각 check node `ci`의 free degree `d_free(ci) = d_max(ci) - d(ci)`(식 (1))를 정의하고, edge-selection 시 후보 집합 내에서 free degree가 최대인 check node를 우선 선택해 degree 편차를 자연스럽게 균등화.
5. 핵심 기법 2단 - maximum check-node degree exchange: 후보 집합 전원이 free degree=0이 되는 경우, 조건 `d_max(c̃i) > d_max(ci)=d(ci) ≥ d(c̃i)`(식 (2))를 만족하는 check node를 찾아 두 노드의 `d_max`를 맞바꿔(식 (3),(4)) girth 저하 없이 edge 배치를 계속함 — 전체 degree 분포는 불변, PEG 후반부에만 소량 발생해 오버헤드 작음.
6. 확장 기법 - girth histogram 개선: 기존 PEG의 "최소 index 또는 랜덤 선택" 대신, 후보 check node 중 local girth가 `2l+4`로 낮아지는 symbol node 개수 `N(ci,sj)`(식 (5))를 최소화하는 후보를 선택 — 실제 girth histogram을 재귀적으로 계산해 girth 열화를 최소화.
7. 구현 디테일: subgraph 확장을 depth `l`→`l+1`에서 직전 depth 노드만 확장하도록 수정(중복 방지), 이를 통해 girth histogram 평가를 저복잡도로 계산(trellis 기반 완전탐색 회피).
8. 학습/최적화 절차: irregular 부호의 symbol-node degree 분포는 별도 density evolution([5]) 결과를 그대로 사용, check-node degree는 본 알고리즘이 결정.
9. 결과: regular (1008,504)에서 평균 local girth 9.66→9.83로 소폭 개선, BER/BLER 소폭 개선. irregular (1008,504)에서 평균 local girth 6.22→7.20(약 1 증가), 고SNR(error-floor 영역)에서 BLER 유의하게 개선; 최소 trapping set 크기가 PEG (10,1)→제안 (16,1)로 커짐(undetected error 비율은 6.7%→21.9%로 증가하지만 trapping-set 기인 detected error가 크게 줄어 전체 성능 개선).
10. 한계: HW 미설계·순수 시뮬레이션(AWGN)뿐이며, 작은 block length(1008) rate-1/2 예제로만 검증, right-concentrated 분포 외 일반 분포(UEP 등)는 논의만 하고 실험 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix/부호구조 구성(PEG 기반, check-node degree 제어) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(base matrix+circulant shift) 고정 구조라 PEG류 무작위 edge-growth 구성법 자체를 직접 대체 이식하기엔 난이도 높음(프로파일 §4: 부호설계 이식난이도 "높음") |
| 엄격히 균일한 check-node degree (HW CN 처리 단순화) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN degree가 균일할수록 HW CNU 파이프라인의 엔트리 종류가 줄어 우리 min-sum CN 유닛 설계에도 유리한 방향과 일치 |
| trapping-set 크기 확대를 통한 error-floor 개선 | `decoder.cpp` `LDPC_Decoder()` (반복 루프의 error-floor 특성과 연관) | 부호설계로 error-floor를 개선하는 접근이라 대상=both, 디코더 알고리즘 자체 변경은 없어 decoder.cpp 로직 이식은 아니고 H-matrix 설계 참고 수준 |

> 적용 가치: 중간 — 특정 QC-LDPC 부호 자체를 이식하긴 어렵지만("높음" 난이도), check-node degree 균일화와 trapping-set 크기 확대라는 설계 원칙은 Prime ECC의 H-matrix 재설계/차기 부호 최적화 시 참고할 만한 일반 원리.
