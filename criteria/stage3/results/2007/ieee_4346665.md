### B. 알고리즘 요약

1. 시스템: 코드/인코더/디코더를 동시설계(concurrent design)하는 versatile LDPC codec, IPPM(irregularly partitioned permutation matrix) 기반 circulant QC-LDPC, AWGN 채널.
2. 문제: 기존 LDPC 디코더는 특정 코드에 아키텍처가 고정되어 rate/length 변경 시 재설계 필요, puncturing 기반 rate-compatible 시스템은 성능 열화 발생.
3. 부호 구성: macro-matrix(작은 base matrix)를 shifted-identity 서브행렬(크기 `p×p`, 확장계수 `p`)로 확장, ACE(approximate cycle EMD) 기준으로 shift값 선택해 저error-floor 보장, ALT(approximate lower triangular) 변환(gap=1)으로 `Richardson-Urbanke` 준선형 인코딩 지원.
4. 인코더: `H`를 ALT 형태로 변환 후 `p1=-φ^-1·(-ET^-1A+C)s^T`, `p2` 계산(식 2-3)으로 전체 인코딩을 macro-multiplication(cyclic shift+XOR 누산) 및 macro-forward-substitution으로 환원, dense `φ^-1` 곱셈만 예외.
5. 디코더: Mansour-Shanbhag의 TDMP(Turbo Decoding Message Passing) 기반, 각 파리티 행을 2-state trellis로 보고 BCJR형 SISO 메시지 처리, 핵심 근사함수는 max-quartet bivariate function `Q(x,y)`(식 11).
6. 확장/최적화: circulant IPPM 채택으로 인터리버를 barrel shifter(`shift`/`revshift` 모듈)로 대체, `Q(x,y)`를 principal part `q(x,y)`(식 13)로 단순화(LUT 사용량 대폭 절감, 성능손실 약 0.1dB@BER=1e-5, 10 iteration).
7. 부수 최적화: compound rotation으로 revshift 모듈 제거(회전값을 code memory에 미리 반영), multiframe interleaving(F1/F2 교차 처리)으로 파이프라인 idle을 제거해 MPU 활용률 극대화.
8. 코드 생성 절차: macro-matrix 무작위 채움 → ACE 기준 확장 → ALT 변환 → 역행렬 가능성 확인의 반복적 코드 합성 흐름(교과서적 GA/DE는 아니고 구조적 제약 기반 탐색).
9. 결과: Virtex-4에서 인코더 278MHz/614slices(~17k gate), 디코더 181MHz/10970slices(~300k gate), rate 0.33~0.75, length 6912 4종 코드에 대해 코드 RAM 교체만으로 동일 HW로 스위칭 가능함을 검증.
10. 한계: FPGA 검증뿐 ASIC 미검증, TDMP는 Prime ECC의 min-sum 스케줄과 다른 아키텍처(2-state trellis/BCJR 기반)라 디코더 코어 자체 이식은 재설계 수준, 정정능력/error-floor 개선을 직접 측정한 실험은 없음(코드 구성법 자체 성능만 제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| circulant 회전(barrel shifter, compound rotation) | `decoder.cpp` `LDPC_Decoder()`, `GT.cpp` `Make_GT_HW()` | Prime ECC의 z=32 lifting 기반 circulant shift 처리와 개념적으로 유사, compound rotation으로 revshift 제거하는 아이디어는 GT(Graph Thinning) MUX 절감과 유사한 방향성이나 실제 회로 대응은 확인 필요 |
| TDMP(BCJR형 SISO, max-quartet 근사) | 대응 없음 | Prime ECC는 min-sum(C2V_Cal 계열)만 사용, turbo-like trellis 기반 SISO는 다른 알고리즘 계열 |
| multiframe interleaving(파이프라인 idle 제거) | `decoder.cpp` pipeline 6-stage(col_M1/Z/P1~P4) | 파이프라인 유휴 시간 절감이라는 목표는 유사하나 멀티프레임 교차처리 방식은 Prime ECC HW 모델에 대응 없음 |
| IPPM 부호구성(rate/length 가변) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC 고정 채택, 동적 rate 스위칭은 대응 없음 |
| ALT 인코딩(Richardson-Urbanke 준선형) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 인코딩 고정 채택, ALT 방식은 대응 없음(다른 인코딩 방식) |

> 적용 가치: 중간 — circulant shift 최적화(compound rotation)와 파이프라인 유휴 제거 아이디어는 HW 효율 관점에서 참고할 만하나, 핵심 기여인 TDMP 디코더와 동적 코드 스위칭은 Prime ECC의 min-sum·고정 단일 코드 설계와 목적/구조가 달라 직접 이식은 어려움.
