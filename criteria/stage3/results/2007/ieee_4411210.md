### B. 알고리즘 요약

1. Binary-input AWGN 채널, regular (dv,dc) LDPC code, 테스트 코드는 rate 0.33~0.77, length 273~3000(랜덤 구성 2종 + finite-geometry 2종).
2. 문제의식: hard-decision(GA/GB/MB) 알고리즘은 배선·전력 이점은 크지만 soft-decision(BP/MS) 대비 성능 열화가 수 dB로 큼 — 이 격차를 줄이면서도 배선비용 이점을 유지하고자 함.
3. 채널 모델은 표준 BI-AWGN, 수신값을 `cth`로 클리핑 후 `2^q-1` 구간으로 균일 양자화(q-bit).
4. 핵심 기법: 엣지마다 q-bit reliability memory `B(vi,cj)`를 두고, check node는 단순 XOR(부호곱) `m_cj→vi = Π m_vk→cj`, variable node에서 `B_new = B_old + round(Σm_ck→vi / 2)`로 차동 업데이트, variable→check 메시지는 `sign(B)`만 전송(1-bit).
5. 핵심 식 `(2)`(메모리 업데이트)의 의미: soft 정보(reliability)는 메모리에만 보존하고 노드 간 전송은 1-bit로 제한 — VLSI 배선 수를 q배 줄이는 효과.
6. Hard-decision 최종판정은 `D_i = Σsign(B(vi,cj)) + sign(yi)`의 부호로 결정, 매 iteration 후 `zHᵀ=0` 체크로 조기종료.
7. 변형(MDD-BMP): 엣지별 메모리 대신 variable node당 메모리 1개만 사용 — 메모리 절감되나 랜덤코드 성능은 크게 저하, FG 코드는 저하폭 작음.
8. 별도 학습절차 없음(Monte Carlo 시뮬로 최적 quantization step Δ*를 코드별 결정: q=6~8).
9. 결과: hard-decision 대비 최대 약 3dB(랜덤코드)/1.5dB(FG코드) 개선, BP 대비 1.3dB/0.7dB 이내로 근접, 단 평균 iteration 수는 BP/MS보다 대체로 많거나 유사.
10. 한계: HW 미설계(순수 알고리즘·시뮬), SNR 추정 불필요라는 장점은 있으나 iteration 수 증가로 실제 latency 이득은 불확실, 코드별 최적 quantization step 및 q값을 사전 튜닝해야 함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| binary(1-bit) message-passing 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 min-sum(soft 2~3bit LLR) 대체 대안이나 성능 열화 있어 우선순위 낮음 |
| check node XOR 연산 | `decoder.cpp` `C2V_Cal_New_Sgn()` | 부호(sign)곱 XOR 연산은 이미 min-sum 내 부호 계산부와 유사 |
| 엣지별 reliability memory 갱신 | `decoder.h` `Get_VNU_*`, `ecc_data.h` `PARAM_LLR` | 신규 방식(differential update)으로 기존 LLR 테이블 구조와 상이, 대응 부분적 |
| soft quantization(q-bit) | `channel.cpp` `Set_LLR_Th()` | soft-2~3bit 지원 범위 내, 논문의 q=6~8은 Prime ECC 미지원 범위 |
| VLSI 배선/전력 절감 근거 | 디코더 HW 모델(z=32, HCU, GT) | HW 미설계 논문이라 직접 비교 불가, 개념적 참고만 가능 |

적용 가치: **중간** — 1-bit message-passing으로 배선 절감 아이디어는 흥미로우나 성능(waterfall)·iteration 수 모두 min-sum 대비 열위이며 HW 미검증이라 즉시 이식보다는 개념 참고 수준.
