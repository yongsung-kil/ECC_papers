### B. 알고리즘 요약

1. 대상은 [31,16,7]-BCH, [63,45,7]-BCH, [127,113,5]-BCH, [24,12,8]-Golay 등 rate>1/2, n<127의 대수적 블록코드, AWGN 채널.
2. 대수적 블록코드는 dual code의 최소거리 codeword로 만든 parity-check matrix도 밀도 20% 이상, Tanner graph girth 4로 표준 BP 디코딩에 부적합.
3. 채널은 AWGN, 안테나 심볼은 `c_v ∈ {-1,1}`인 이진 antipodal 신호로 가정.
4. 핵심 기법: cyclic group generator(cgg, dual code 최소 weight codeword의 cyclic shift 집합 대표원)로 서로 다른 `H_ℓ, ℓ=1..I`를 구성하고, `I`개의 BP 디코더를 병렬로 동시에 실행(Multiple-Bases Belief-Propagation, MBBP).
5. 각 디코더가 유효 codeword로 수렴하면 집합 `S`에 포함시키고, 여러 후보가 있으면 least metric selector(LMS)로 `argmin Σ(y_v - c_{s,v})^2`를 최소화하는 codeword를 최종 선택.
6. 전체 복잡도는 `I_total = i·l`(디코더당 iteration `i` × 디코더 수 `l`)로 정의, 동일 총 iteration 예산에서 디코더 수를 늘리는 편이 디코더당 iteration을 늘리는 것보다 성능 이득이 큼.
7. BEC용 redundant BP 아이디어를 AWGN에 그대로 적용하면 오히려 cycle 수 증가로 성능이 나빠져, 단일 큰 redundant 행렬 대신 여러 개의 작은 저밀도 행렬로 분할하는 방식을 채택.
8. 별도 학습/최적화 절차 없음(cgg 선택은 조합적 열거).
9. [63,45,7]-BCH에서 rate 0.71 부호를 5dB에서 BER 10^-5로 디코딩(union bound ML 대비 0.06dB 이내), [127,113,5]-BCH에서 표준 BP 대비 1dB 성능 개선과 동시에 iteration을 100→30으로 감소.
10. 한계: LDPC가 아닌 대수적 블록코드(BCH/Golay) 전용이며 n<127의 짧은 코드에만 검증, HW 설계 없이 시뮬레이션 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 디코딩 알고리즘(대수적 블록코드 대상) | 대응 없음 | Prime ECC는 QC-LDPC 전용이며 BCH/Golay 등 대수적 블록코드 디코딩 로직 없음 |
| 다중 H-matrix 병렬 디코더 + LMS 선택 | 대응 없음 | min-sum 기반 단일 H-matrix 구조와 근본적으로 다름 |
| cyclic group generator 기반 parity-check matrix 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 개념만 유사, 구성법 자체는 QC-LDPC circulant와 무관 |

> 적용 가치: 낮음 — 부호가 QC-LDPC가 아닌 BCH/Golay 대수적 블록코드이며, 다중 parity-check matrix 병렬 디코딩이라는 아이디어도 Prime ECC의 단일 QC-LDPC H-matrix 구조와 정합성이 낮음.
