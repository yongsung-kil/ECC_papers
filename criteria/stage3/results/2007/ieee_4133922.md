### B. 알고리즘 요약

1. 시스템은 AWGN 채널의 regular systematic LDPC 코드(예: (96,48), (504,252), ρ=6, γ=3, rate 1/2)를 대상으로 함.
2. 기존 BP는 성능은 좋으나 복잡도가 높고, reduced-complexity BP나 bit-flipping, linear programming(LP) 디코더는 성능-복잡도 트레이드오프에 한계가 있어 새로운 원리의 디코더를 제안.
3. 핵심 가정은 패리티검사를 이진체가 아닌 연속 box set `[0,1]^ρ`로 완화(convex relaxation)하는 것.
4. 핵심 기법은 GF(2) 덧셈과 동치인 `f(x1,x2)=x1+x2-2x1x2`를 nested 적용해 각 체크노드의 patiry-check를 `fj(x1,...,xρj)`로 표현하고, 전체 목적함수 `F=Σfj`를 정의.
5. Gradient Projection 반복식 `ĉl = PQ[ĉl-1 - α∇F]`(box set으로의 사영)로 채널 관측값 근방의 유효 코드워드를 탐색 — `α`는 수렴속도/정확도를 좌우하는 step parameter.
6. 확장으로 2차(second-order) parity check 표현(모든 first-order 페어의 mod-2 합)을 추가해 (2(ρ-1), (ρ-1)γ(γ-1)) 등가 코드를 구성, 짧은 cycle 유발에도 성능·수렴속도 개선.
7. 부수 구현 디테일: 각 변수노드 업데이트는 독립적/병렬 수행 가능(식 5의 clipping 연산), 연산량은 코드 길이에 sub-linear(이미 0/1로 수렴한 노드는 연산 생략).
8. 별도 학습/최적화 절차 없음(고정 step size `α` 사용, 값에 따라 수렴속도-성능 트레이드오프 실험적으로 확인).
9. 결과: (96,48) 코드에서 GP는 BP 대비 BER=1e-5에서 약 0.8~1dB 열세, LP 대비 0.8~1dB 우세, 10 iteration 이내 수렴; 2차 표현 적용 시 0.2~0.3dB 추가 이득; (504,252) 코드에서는 BP 대비 2.5dB 열세로 격차 확대.
10. 한계: 성능이 BP보다 뚜렷이 열세(특히 중간 길이 코드에서 격차 확대), HW 설계 없음(실수 곱셈/덧셈 기반), soft 채널 입력 전제로 NAND의 제한된 read 비트수 환경 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gradient Projection 반복 복호 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()` | Min-sum과 원리가 다른 신규 이터레이션 방식이나, 실수 연산·연속값 기반이라 정수 min-sum 파이프라인과 구조적으로 이질적, 이식 난이도 높음 |
| 변수노드 병렬 업데이트(box 사영) | `decoder.cpp` `VN_Cal_HD()` 등 | 완전 병렬 처리 아이디어는 유사하나 실수 gradient 연산은 HW 대응 없음 |
| 2차(denser) parity check 표현 | 대응 없음 | 코드 자체를 확장하는 표현이라 Prime ECC의 고정 H-matrix 구조와 부합하지 않음 |
| soft LLR 채널 입력 | `channel.cpp` `Set_R_Offset()` | 채널관측값 사용은 유사하나 GP는 확률(0~1) box 표현으로 LLR 양자화 테이블과 직접 대응 어려움 |

> 적용 가치: 낮음 — BP 대비 성능 열세인 신규 원리 디코더로, 실수 연산 기반이라 정수 min-sum HW 파이프라인에 이식하기 어렵고 짧은 코드에서만 검증됨.
