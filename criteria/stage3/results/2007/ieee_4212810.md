### B. 알고리즘 요약

1. 대상은 IEEE 802.16e 표준의 block-type LDPC(B-LDPC, QC-LDPC 일종) 부호, rate 1/2 mother code에서 puncturing으로 rate 2/3, 3/4 생성.
2. 문제: puncturing은 rate-compatible 코드를 얻는 간단한 방법이지만 성능 손실 발생, 기존 grouping/sorting 기법[3]은 B-LDPC 특유의 block 구조를 활용하지 못함.
3. 채널/모델: AWGN, BPSK, iterative BP(log-MAP layered decoding, 50 iteration) 디코딩 가정.
4. 핵심 기법: punctured variable node를 반복 후 non-zero LLR을 받도록 회복(recovered)시키는 l-SR(l-step recoverable) 노드 개념과 recovery tree 기반 grouping/sorting.
5. 핵심 식: `mu(v) = φ^-1[1-(1-φ(mu0))^S(v)]` — recovery tree 내 unpunctured node 수 S(v)가 커질수록 에러확률 Pe(v) 증가하므로 S(v) 최소화가 목표.
6. 확장(B-grouping/sorting): Lemma 1/Theorem 1로 base matrix Hb의 l-SR 노드/survived check node가 확장 시 factor z만큼 그대로 block 단위로 대응됨을 증명, 이에 따라 grouping/sorting을 base matrix에서만 수행.
7. 구현 디테일: base matrix 단위 puncturing으로 패턴 저장 메모리를 1/z로 절감, 코드워드 크기 무관하게 하나의 base 패턴 재사용 가능.
8. 최적화 절차: 없음(그룹/정렬 규칙 기반 결정론적 알고리즘, DE/GA 없음).
9. 결과: FER 10^-3에서 B-grouping/sorting이 random puncturing 대비 약 0.6dB 이득, 기존 grouping/sorting과는 성능 동등하면서 메모리·복잡도만 절감.
10. 한계: 순수 알고리즘/시뮬레이션 결과만 제시(HW 미설계), NAND가 아닌 무선통신(hybrid ARQ, AMC) rate-compatible 시나리오 대상.

> 적용 가치: 낮음 — rate-compatible puncturing은 NAND ECC의 고정rate 설계 목적과 맞지 않고, Prime ECC 3.1에 대응하는 모듈이 없음.
