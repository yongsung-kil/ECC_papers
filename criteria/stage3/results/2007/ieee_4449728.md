> 총평: QC-LDPC의 special class(Block-LDPC)에 lower-triangular형 irregular H를 적용하고 block matrix 크기를 조절해 BNU/CNU 기반 min-sum 디코더의 throughput-BER 트레이드오프를 시뮬로 분석한 하드웨어 아키텍처 논문. 정정능력 개선보다 처리속도/라우팅 복잡도 균형에 초점.

### B. 알고리즘 요약

1. Rate 1/2, 2/3, 5/6의 QC-LDPC special class(Block-LDPC, N=2304)를 AWGN 채널에서 min-sum algorithm(MSA)으로 반복 디코딩.
2. 표준 QC-LDPC는 부호화는 쉽지만 하드웨어 구현이 어려워, encoding·decoding 모두에 적합한 parity-check matrix 구조가 필요함.
3. 채널은 soft-demapping을 거친 AWGN 채널, LLR은 sign-magnitude 포맷으로 표현.
4. `H=[H_I|H_p]`로 분할하고 `H_p`를 거의 lower-triangular(대각 인접 두 개의 `p×p` identity block) 구조로 고정, `H_I`는 서로 다른 지수(`d1≠d2≠d3`)의 순환행렬 시퀀스로 girth를 증가시키는 구성.
5. Check node 갱신은 `Lr(r_{c→b}) = sign(∏Lr(q_{b'→c})) × min|Lr(q_{b'→c})|` (식 2), bit node 갱신은 `Lr(q_{b→c}) = Lr(p_b) + Σ Lr(r_{c'→b})` (식 3) — MSA의 표준 min-sum 근사.
6. Block matrix 크기 `L`을 12×12/48×48/96×96로 가변해 BNU(N=n/L개)·CNU(K=N×(1-R)개)·RF 자원과 동작 사이클(예: L=96에서 1632cycle, L=12에서 204cycle)을 조절.
7. 구현 디테일: LLR을 2's complement로 변환해 8-bit adder로 연산 후 6-bit로 clipping(overflow 방지), sign bit는 XOR, magnitude는 CMP-6로 min/2nd-min 산출.
8. 학습/최적화 절차 없음(고정 구성 규칙 기반 설계).
9. 결과: block matrix 96×96/48×48/12×12 간 BER/FER 성능 차이는 미미하나, L=12가 사이클 수를 대폭 줄여 throughput을 크게 향상; rate 1/2→2/3→5/6 순으로 SNR 이득이 각각 약 1dB, 2dB 손실.
10. 한계: 시뮬레이션(FER/BER 곡선)에만 근거, 실제 게이트카운트·공정·throughput 수치(Gbps)는 본문에 제시되지 않음(그림에만 존재).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum 기반 check/bit node 갱신(식 2~3) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min1/min2+sign XOR 구조와 동일 계열, 기법 자체는 이미 보유 |
| BNU/CNU 하드웨어 아키텍처, block matrix 크기 가변에 따른 처리속도 트레이드오프 | `decoder.cpp` `LDPC_Decoder()` (pipeline/HCU 등 HW 모델) | 병렬화 단위(z=32)와 유사한 개념이나 이 논문은 알고리즘보다 블록 크기 스윕 실험 성격 |
| 8-bit 연산 + 6-bit clipping LLR 양자화 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | soft LLR 비트폭 설계와 관련되나 NAND read 비용(soft-2~3bit) 대비 과도한 폭 |
| Lower-triangular 기반 irregular H 부호구성 | `ecc_top.cpp` `Load_PCM()` | 특정 QC-LDPC 고정 구조 재검증 필요, 이식 난이도 높음 |

> 적용 가치: 중간 — 디코더 핵심 알고리즘(min-sum)은 Prime ECC와 중복되어 신규성이 낮지만, block matrix 크기에 따른 라우팅/사이클 트레이드오프 데이터는 HW 아키텍처 설계 참고자료로 일부 가치 있음.
