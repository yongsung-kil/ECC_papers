### B. 알고리즘 요약

1. LDPC 디코더의 CNU(check node unit)는 Min-Sum 근사 `Lc→v = Π sign(Lv'→c) · min|Lv'→c|` (exclusion 연산)를 계산하며, 이 min/2nd-min 비교 연산이 임계경로.
2. 기존 conventional CNU[6]는 정렬로 min/2nd-min 산출 후 각 변수노드에 선택 분배(`2Nin-3`개 비교기, `Nin`개 멀티플렉서 필요), exclusive CNU[7]는 각 입력마다 자신을 제외한 min을 병렬 산출하나 6-input 전용 수작업 설계뿐.
3. 이 논문은 임의의 Nin에 대해 최단 지연 `⌈log2(Nin-1)⌉·T`와 최소 2-input 비교기 수를 보장하는 exclusive comparison을 자동 합성하는 체계적 알고리즘 제안.
4. 핵심 기법: 입력 원소들을 행렬(M)로 배치하고, 행/열 방향 공유 가능한 비교집합(H-V set, H set, H-C set)을 재귀적으로 구성하는 VHC(vertical-and-horizontally-cyclic) set construction 알고리즘.
5. H-V set은 `2^k`개 원소를 가진 교집합 형태 공유집합, 잔여(remnant) 원소는 H set 또는 경계 순환 결합인 H-C(horizontally-cyclic) set으로 처리하여 공유 극대화.
6. 확장: 동일 알고리즘의 2-input comparator를 2-input adder로 치환하면 VNU(variable node unit) 설계에도 그대로 적용 가능하다고 언급.
7. 구현 디테일: 8단계 알고리즘(초기화→H-V set 크기 갱신→H set 형성→H-C set 형성→row 반복→이진비교 실행)으로 정형화, 행렬 대칭성(대각선 blank, row 간 2원소 차이)을 이용해 계산량 축소.
8. 최적화 절차는 손으로 하기 힘든 긴 코드(large Nin)에 대해 자동으로 근최적해를 도출하는 것이 목적(학습이 아닌 결정적 알고리즘).
9. 결과(Table 2): Nin=6~14에 대해 필요 비교기 수 제시(예: Nin=7→18개), conventional 대비 최대 2배 미만 비교기로 동일 지연시간 `⌈log2(Nin-1)⌉T` 달성.
10. 한계: 특정 코드 rate/length 명시 없이 순수 CNU 하드웨어 아키텍처 최적화 이론이며, 실제 합성/칩 결과나 BER 성능 실험 없음(이론적 시간/면적 분석뿐).
