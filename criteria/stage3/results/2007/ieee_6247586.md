### B. 알고리즘 요약

1. 대상은 Chinese DTTB(지상파 디지털TV 방송) 표준의 QC-LDPC 코드(BCH 외부코드 + QC-LDPC 내부코드 concatenation 중 LDPC만 다룸), rate 0.4/0.6/0.8, codelength 7488 bits(b=127 고정, k/c만 변화).
2. 풀려는 문제: 무작위 컴퓨터 탐색으로 만든 LDPC는 구조가 없어 인코딩이 복잡(O(N^2)); 표준화된 QC-LDPC를 빠르게 인코딩할 저복잡도 HW 회로가 필요.
3. 핵심 가정: 부호는 2-정보심볼 RS 코드 구조 기반 구성으로 girth≥6(4-cycle 없음) 보장, 생성행렬 `G_qc`가 permutation submatrix들로만 구성(식 4).
4. 핵심 기법: permutation matrix는 첫 행/열(generator)만으로 전체를 특정할 수 있어 저장량이 크게 줄고, `c = a·G_qc`(식 5) 인코딩이 shift-register 기반 회로로 구현 가능.
5. 핵심 식: 인코딩 `c = aG_qc`이며, `G_ij`가 permutation matrix이므로 각 서브블록은 순환 shift register로 대체 가능 (하드웨어 저장/게이트 절감의 근거).
6. 확장: 3가지 회로 변형 - (a) serial encoding: 정보 비트를 직렬 입력, `(k+1)b` 클록, 2bc FF; (b) parallel encoding: 정보 비트 전체 병렬 입력·패리티 직렬 생성, `(c+1)b` 클록, 2kb FF; (c) partial-parallel: 절충형, `(k+1)b` 클록, `(2c+1)b` FF.
7. 구현 디테일: 최대 c=35, k=48 기준 실제 게이트 수 산출(serial: FF 8890/AND·XOR 4445; parallel: FF 12192/AND 6096; partial-parallel: FF 9017/AND 4445/XOR 35) - FPGA 구현 시 FF 수가 중요하므로 serial/partial-parallel이 유리하다고 결론.
8. 별도 학습/최적화 절차 없음(구성법은 RS 코드 이론에서 결정적으로 유도).
9. 결과: 3개 회로의 클록사이클·FF·게이트 수 정량 비교(Table I)만 제시, BER/디코딩 성능 실험 없음(인코더 전용 논문).
10. 한계: 실제 합성/타이밍 결과 없이 이론적 게이트 카운트만 제시, 디코더/정정성능 관련 내용 전무, 특정 표준(DTTB)의 고정 파라미터(b=127)에 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 저복잡도 인코딩 회로(serial/parallel/partial-parallel) | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코더를 이미 사용 중이나, permutation-matrix 기반 shift-register 인코딩은 대안적 회로 구조로 비교 참고 가치 있음 |
| 부호용 생성행렬/PCM 구성(RS 코드 기반) | `encoder.cpp` `Generate_PCM_encoding()` | 부호구성 자체가 RS 코드 기반으로 Prime ECC의 QC-LDPC 구성법과 달라 재검증 부담 큼(이식 난이도 높음) |
| H-matrix 구조(RC-constraint, girth≥6) | `ecc_top.cpp` `Load_PCM()` | 특정 부호구성 방법론으로 대응 없음(참고용) |

> 적용 가치: 중간 — 인코더 게이트카운트 트레이드오프 비교(serial/parallel/partial-parallel)는 HW 인코더 설계 시 참고할 만하나, 부호구성 자체(RS 기반)와 이미 Prime ECC가 보유한 dual-diagonal 인코딩 방식이 달라 직접 이식보다는 설계 아이디어 참고 수준.
