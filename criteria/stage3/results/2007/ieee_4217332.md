> 총평: super-code 분할 디코딩에서 degree-one 노드 제거로 outer-iteration·메모리접근·throughput을 약 2배 개선하는 부호설계 기법. 메모리 병목 완화 아이디어는 참고 가치 있으나 non-binary GF(193) 검증, SDR 아키텍처 특화로 이식은 재해석 필요.

### B. 알고리즘 요약

1. SDR(software defined radio) 멀티프로세서 플랫폼: local memory가 작은 PU에서 큰 block-length LDPC를 디코딩해야 하는 메모리 제약 문제.
2. 문제: 기존 super-code 기반 분할 디코딩([11])은 global memory 접근(outer iteration `Q2`)을 줄이면 BER이 저하됨.
3. 핵심 가정: LDPC를 `C`개의 super-code(부분 parity-check matrix `Hb_i`)로 나눠 inner iteration `Q1`(super-code 내부) / outer iteration `Q2`(super-code 간 정보교환)으로 2단 반복.
4. 핵심 기법: super-code 안에서 degree-one 변수노드는 inner iteration 동안 LLR이 갱신되지 않음(정리 2.1로 증명)을 지적, 부호설계 단계에서 exchange 연산으로 degree-one 노드를 제거(가능한 한 degree≥2).
5. 핵심 식: `L_n^(s,t+1) = L_n^(s,t) + E_{n,m}^(s,t+1)`이지만 degree-one이면 다음 inner 반복에서 `L_{n,m}^(s,t+1) = L_{n,m}^(s,t)`로 불변 → 오류정정 정체.
6. 확장: `Tmem`(전역메모리 접근 횟수)과 `Tdec`(디코딩 사이클)이 모두 `Q2`에 비례 → `Q2` 최소화가 throughput 개선의 핵심.
7. 부수 트릭: 하삼각 구조 유지를 위한 필수 degree-one 노드는 예외적으로 유지, 여러 pass로 점진적 제거.
8. 최적화 절차: exchange operation([10] 인용, 별도 논문) 반복 적용으로 degree-one 노드 최소화하는 code construction.
9. 결과: 동일 BER 성능 기준 outer-iteration 및 전역메모리 접근 횟수 약 2배 감소, throughput 약 2배 증가, local memory도 최대 9.7% 절감(Scheme-II 기준 BER손실 0.1dB 미만).
10. 한계: GF(193) non-binary irregular LDPC(rate 3/4)로 시뮬레이션, SDR 멀티프로세서 아키텍처(global/local memory, PU) 특화 가정, HW(RTL) 미설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| super-code 분할 디코딩 (inner/outer iteration) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 단일 파이프라인(row 단위 z=32 병렬) 구조로 super-code 개념 없음, 개념적 참고만 가능 |
| degree-one 노드 제거 부호설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성 단계 제약조건 추가에 해당, QC-LDPC 고정 구조 재설계 필요해 이식 난이도 높음 |
| 전역/로컬 메모리 접근 최소화 | 대응 없음 | Prime ECC 문서상 메모리/라우팅 최적화 대상 모듈 명시 없음(HCU/GT는 edge/cycle 축소이지 memory access 모델과 다름) |

> 적용 가치: 중간 — degree-one 노드가 반복 중 갱신되지 않는다는 통찰은 부호설계 시 유효 노드 활용을 극대화하는 일반 원칙으로 참고 가능하나, SDR super-code 구조·non-binary 검증이라 직접 이식은 재검증 부담이 큼.
