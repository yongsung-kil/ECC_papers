### B. 알고리즘 요약

1. 규칙적 (6,3) LDPC code, blocklength 1200, rate 1/2, PEG(progressive edge growth) 기반 부호 사용.
2. 목표: FPGA에서 완전병렬(fully-parallel) 구조로 최대 throughput 달성, 기존 설계 대비 하드웨어 오버헤드 없이 성능 개선.
3. 채널은 AWGN, antipodal 신호, soft-decision 입력 `L(ci) = yi/2σ²`로 스케일링.
4. 핵심 기법은 min-sum(MS) 알고리즘의 check node processor(CNP) 저복잡도 설계: degree-6 CN에서 comparator 12개만 사용(기존 24개 대비 절반 이하).
5. `L(rji) = (Π αij) · min(βij)` — CN 출력은 부호곱과 최소 크기값으로 근사, φ함수(tanh 기반) 대신 min 연산으로 SP 대비 약 0.5dB 열화.
6. Variable node processor(VNP)는 degree-3, 4-input adder 1개와 2-input subtractor 3개로 단순 구현.
7. 3-stage pipeline(input/decode/output) 구조, 3-bit precision((3,1) quantization, 8-level) 채택 — precision 1비트 증가마다 자원 사용 2배 증가하는 trade-off로 결정.
8. 별도 학습/최적화 절차 없음(PEG는 기존 구성법 인용).
9. 결과: Virtex-4 FPGA(xc4vlx200)에서 100MHz 클록, 10 iteration으로 throughput 6Gb/s 달성, slice 이용률 45%(라우팅 한계로 인해 자원 여유에도 더 긴 code 불가).
10. 한계: rate-1/2 저rate·1200 bit 짧은 code, ASIC 미검증(FPGA만), offset/scaling factor 등 성능개선 기법은 precision 한계로 미적용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CN 연산(저복잡도 comparator 구조) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum 보유, comparator 최적화만 참고 가능 |
| VN 처리 | `decoder.cpp` `VN_Cal_HD()` 등 | 단순 가산 구조로 이미 대응 |
| 3-bit soft quantization | `channel.cpp` `Set_LLR_Th()` | soft-2~3bit 이미 지원, 신규 기여 없음 |
| 완전병렬 FPGA 파이프라인 | 디코더 HW 모델(z=32, HCU, GT) | Prime ECC는 반병렬 구조라 완전병렬 설계는 구조 상이 |
| PEG 기반 부호 구성 | `ecc_top.cpp` `Load_PCM()` | regular code 구성법으로 QC-LDPC 아님, 대응 없음 |

적용 가치: **낮음** — min-sum, soft quantization 등 핵심 기법이 이미 보유 기술과 중복되고, 저rate regular code·완전병렬 FPGA 구조는 Prime ECC의 고rate QC-LDPC 반병렬 구조와 정합성이 낮음.
