> 총평: BP 실패 시 이전 iteration LLR을 저비용(shift 연산)으로 재활용하는 2단계 피드백 복호로 error floor를 0.1~0.4dB 개선했으나, HW 설계·게이트/타이밍 검증이 없어 실제 이식 시 재검증 부담이 있음.

### B. 알고리즘 요약

1. 대상 코드: Mackay(504,252) irregular, QC-LDPC(155,64), 802.16e QC-LDPC(576,288, rate 1/2), Finite Geometry(273,191)/(1057,813). AWGN + BPSK.
2. 문제: LDPC BP 복호의 error floor는 trapping set 내 오류 비트의 LLR이 iteration마다 진동(oscillate)하며 수렴하지 않아 발생.
3. 가정: trapping set 비트의 LLR은 반복 동안 진동하므로, 이전 iteration 결과를 현재 iteration에 피드백하면 진동을 억제해 일부 trapping set을 완화할 수 있음.
4. 핵심 기법(1단): 표준 LLR-BP로 먼저 복호, 결과가 parity check(신드롬 0)를 만족하면 즉시 종료.
5. 핵심 기법(2단, FBP): 1단 실패 시 check-to-variable 메시지를 `L'mn(k) = βL(k)mn + (1-β)L(k-1)mn` (one-order feedback, J=1)로 재계산해 재복호. `β`는 이전/현재 메시지 가중 결합계수.
6. HW 친화 포인트: `β=0.5`로 고정하면 곱셈이 1비트 레지스터 shift로 대체되어 FBP 1회 iteration 복잡도가 BP와 거의 동일.
7. 정지 기준은 기존 parity check syndrome=0 사용(별도 CRC 없음); 고rate/짧은 length 코드는 undetected error 비율이 커 CRC 등 대체 기준 필요하다고 논의.
8. 별도 학습/최적화 절차 없음 (고정 β=0.5, one-order feedback만 사용).
9. 결과: (504,252)에서 FER 1e-4 기준 약 0.1dB 이득, (155,64)에서 약 0.2dB, (273,191)/(1057,813)에서 각각 0.1dB/0.4dB. 오류 프레임의 약 30~65%가 FBP 단계에서 추가로 정정됨. 증분 복잡도는 고SNR에서 약 0.1%.
10. 한계: 실제 HW 구현/합성 없음(시뮬만). undetected error(오검출) 비율이 코드에 따라 최대 30%까지 존재해 정지 기준의 신뢰성 문제 지적. 부호설계 요소는 없고 순수 디코더 후처리 기법.
