### B. 알고리즘 요약

1. 짧은 regular LDPC 코드(TS code (576,1152), MF code (1225,1715))의 Tanner graph에서 girth(최소 사이클 길이) 및 girth+2 사이클 분포를 분석.
2. 문제의식: 짧은 코드는 사이클이 많아 conventional flooding-schedule BP가 최적이 아니며, 일부 bit node의 extrinsic LLR이 사이클로 인해 진동(oscillate)해 수렴을 방해.
3. 채널 모델은 표준 BPSK+AWGN, 기존 log-domain BP(합-곱, 식(1)-(7)) 그대로 사용.
4. 핵심 기법: bit node를 girth 사이클 개수 기준으로 "bit node location 1(cycle-girth 없는 노드), location 2(일정 개수 이하 사이클)"로 분류해 select updating schedule(BP-SU) 구성.
5. `h = SGmax/2 - SGmin/2` (single bit node girth 최대/최소 차)로 반복수 조건식 `(girth/2 + h + 1)×k ≤ iteration ≤ ... + (girth/2 -1)` 정의, 이 그룹 단위로 반복을 나눠 일부 노드만 순차 업데이트.
6. 3단계: `Maxiter/(girth/2+2)` 그룹으로 나누고, 각 그룹 내 iteration마다 location1 → location2 → 나머지 순으로 업데이트 노드를 선택적으로 정지/재개.
7. 부수 트릭: check node process는 변경 없음, bit node process만 스케줄링 변경 — 구현상 알고리즘 로직 최소 변경.
8. 최적화 절차 없음(학습 기반 아님) — cycle distribution 사전 분석(오프라인 카운팅)으로 location 1/2 결정.
9. 결과: TS code 기준 BP-SU는 BP 대비 그룹 반복당 bit node 업데이트 997회 감소(BP-PU는 208회 감소, BP는 6912회), BER 6×10⁻⁵에서 BP/BP-PU 대비 0.2dB 이상 개선. 단, 평균 반복 횟수는 BP보다 최대 2회 더 필요.
10. 한계: 검증은 AWGN 시뮬레이션뿐(HW/FPGA 없음), regular 코드 두 종류(TS/MF)에만 적용, girth 분포 사전계산이 필요해 코드 구조 변경 시 재계산 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP(sum-product) bit node 업데이트 스케줄링 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 BP 전제 스케줄 변경은 알고리즘 성격이 달라 직접 이식 어려움 |
| 조기 노드 업데이트 정지(select updating) | `decoder.cpp` (이터레이션 루프 내 VN 업데이트 로직) | 개념적으로 조기종료(Partial CRC)와 유사하나 node-level이라 대응 구조 다름 |
| girth/cycle distribution 사전분석 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 단계에서 cycle 분석 추가 가능하나 코드 자체 구현 없음, "대응 없음"에 가까움 |

> 적용 가치: 낮음 — BP(sum-product) 전제·regular 코드·소프트웨어 시뮬 수준이며, Prime ECC의 min-sum/QC-LDPC HW 구조와 알고리즘 기반이 달라 이식 난이도가 높고 실익이 제한적.
