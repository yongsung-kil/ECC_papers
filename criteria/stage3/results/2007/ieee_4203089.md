### B. 알고리즘 요약

1. 대상은 DVB-S2 규격의 64800-bit irregular LDPC 코드(rate 1/4~9/10 중 특히 rate 3/5 집중 분석), AWGN 채널, QPSK/BPSK 신호.
2. 문제: min-sum 알고리즘은 BP(sum-product)의 비선형 check node 연산 `tanh^-1(Π tanh(L_i/2))`을 `min` 연산으로 근사하는데, check node 출력 크기를 항상 과대추정(over-estimate)해 BP 대비 성능 손실(SNR degradation) 발생.
3. 기존 보정법(offset, constant scaling normalization, 2-D normalization[Zhang])은 고정 상수 인자를 쓰거나 degree별 상수만 다르게 적용해 반복(iteration)·SNR 변화에 적응하지 못함.
4. 핵심 기법: check node degree `d_c`의 입력 크기들을 order statistics로 정렬한 확률변수의 pdf/cdf를 유도하고, 가장 작은 두 입력크기(min1, min2에 해당하는 `m_1, m_2`)에 각각 다른 정규화 인자 `α(m_1), β(m_1,m_2)`를 곱해 check node 출력을 `α·m_1`, `β·m_2` 형태로 보정 (식 8~10).
5. 핵심 식 의미: `α, β`는 check node 입력들의 조건부 기댓값 함수로 정의되며(식 11~14), 이 인자들이 반복 회차·채널 SNR에 따라 커지는 경향을 밀도진화 유사 4단계 재귀 절차(bit node→check node 입력→check node 출력→bit node 입력, Step1~4)로 추적.
6. 확장: 매 반복마다 다른 정규화 인자를 쓰면 HW 비용이 크므로, 여러 반복 구간의 인자를 평균내어 사용하는 저복잡도 realization 3종 제안 — direct-mapping(2-D LUT), adaptive-K(고정 후보 집합에서 K개 인자 선택, single-β/double-β), annealing(특정 반복에서만 간헐적으로 정규화 적용해 전력·복잡도 절감).
7. 부수 구현 디테일: annealing 방식은 정규화를 매 반복 대신 `T` 간격으로만 적용, 이때 보정계수는 `T`회 반복 효과를 한번에 반영하도록 재유도(식 23~25). 정규화 인자는 shift-and-add로 구현 가능한 `{0,0.125,...}` 후보 집합으로 제한.
8. 학습/최적화 절차: 채널 SNR·코드 degree distribution이 주어지면 order-statistics 기반 이론식으로 정규화 인자를 사전 계산(오프라인), 반복별 인자를 시뮬레이션 기반 평균화해 최종 계수 확정 — GA/NN 등 학습 없음.
9. 결과: rate 3/5, 64800-bit 코드에서 double-β 방식이 평균 SNR 손실을 fixed-β(상수 정규화)의 0.5dB에서 0.2dB로 감소(BP-FP 대비), 목표 BER 1e-... 도달에 필요한 최대 반복횟수를 min-sum 50회 대비 12회로 단축(76% 반복감소), HW는 check node unit당 약 100게이트(0.13um) 추가로 min-sum 대비 약 5% 면적, 17% 지연 증가.
10. 한계: DVB-S2 irregular LDPC(long block, 64800bit) 특정 code에 최적화된 이론 유도로, degree distribution이 다른 코드(예: Prime ECC의 고정 QC-LDPC, N_b=129, VN degree 17)에는 정규화 인자를 재계산해야 함. 실측(칩 동작) 없이 합성(synthesis) 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum check node 동적 정규화(self-compensation) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min1/min2 기반 CN 연산에 α(m1), β(m1,m2) 정규화 인자를 곱하는 형태로 직접 적용 가능한 핵심 대상 모듈 |
| 반복(iteration)별 인자 조정, annealing 방식 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 이터레이션 구간별 LLR threshold 테이블 구조와 유사해 반복별 정규화 인자 테이블로 확장 이식 가능 |
| shift-and-add 기반 저복잡도 정규화 계수 구현 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | HW 친화적 계수 제한(2의 거듭제곱 근접값)이 Prime ECC의 양자화 테이블 기반 magnitude 처리와 정합 |
| VN 처리 / 양자화 | `decoder.cpp` `VN_Cal_HD()` 등 | 정규화는 CN magnitude 출력 단계에 적용되므로 VN 처리 자체와는 간접적 관련 |

> 적용 가치: 높음 — Prime ECC가 이미 보유한 min-sum + 반복별 LLR 테이블 구조에 최소 오버헤드(check node당 약 100게이트, 5% 면적)로 정정성능(SNR 손실 0.5→0.2dB)과 평균 반복횟수(최대 76% 감소)를 동시에 개선할 수 있는 실증된 HW 합성 결과 보유 기법.
