> 총평: EG-LDPC 대상 hard-decision BF/QWBF 계열의 shuffled·replica-shuffled 스케줄링으로 반복수를 대폭 줄인 논문으로, 스케줄링 아이디어(그룹 단위 병렬+순차 처리, replica 교환)는 min-sum 디코더에도 참고 가능하나 알고리즘 자체(BF/QWBF)는 Prime ECC의 soft min-sum과 이질적이라 이식성은 중간 수준.

### B. 알고리즘 요약

1. 대상 코드는 Euclidean Geometry(EG) 기반 cyclic regular LDPC, 예시로 (4095,3367), (16383,14179) 정사각 parity-check 행렬 사용.
2. 표준 BF/WBF 디코더는 수렴에 수십 회 반복이 필요해 고속 VLSI 구현에 부적합하다는 문제를 다룸.
3. 채널은 AWGN + BPSK, 수신값 `y`의 부호로 hard decision `z`를 얻고 `s = z·H^T`로 syndrome 계산.
4. 핵심 기법은 shuffled BF: 한 비트를 플립하면 즉시 관련 syndrome `{sm: m∈M(n)}`을 갱신해 다음 비트 판정에 반영하는 순차 갱신 방식.
5. QWBF는 각 비트 신뢰도를 1비트(`|yn|` vs 임계값 `Δ1`)로 양자화하고, 비트별 가중합 `En = Σ(2sm−1)|y|min−m`이 임계값 `Δ2`를 넘으면 플립 — 정렬·실수 저장 불필요.
6. 확장 기법은 replica-shuffled: 증가순/감소순 서브디코더 2개(또는 4개)가 병렬로 동작 후 매 반복마다 값 교환.
7. HW 친화를 위해 group-shuffled 방식 도입: N비트를 G개 그룹으로 나눠 그룹 내는 병렬, 그룹 간은 순차 처리(고정된 순서라 하드웨어 구현 용이).
8. 별도 학습/최적화 절차 없음(임계값 Δ1, Δ2는 경험적 설정).
9. 결과: G=16, Imax=2인 group-shuffled/replica-shuffled BF가 표준 BF의 Imax=10과 동등 이상 성능; QWBF가 BF 대비 BER 10⁻⁵에서 약 0.5dB 이득; importance sampling으로 BER 10⁻¹⁰~10⁻¹⁵ 구간까지 유의미한 error floor 없음 확인.
10. 한계: 순수 시뮬레이션(HW 미설계), hard-decision 계열 알고리즘이라 소프트 min-sum 대비 정정성능 한계, 임계값 최적화는 이론적 근거 없이 경험적으로 설정됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| shuffled/group-shuffled 갱신 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 스케줄링 순서 개념은 Dual-Update(even/odd 교대)와 유사한 방향성이나 알고리즘(BF/QWBF)이 상이해 직접 이식 어려움 |
| BF/QWBF 비트 플립 연산 | 대응 없음 | Prime ECC는 min-sum 기반 soft 디코더로 hard BF 알고리즘 자체는 대응 모듈 없음 |
| 1비트 신뢰도 양자화(QWBF) | `channel.cpp` `Set_LLR_Th()` | 1비트 threshold 개념이 LLR 양자화 threshold 설계와 유사한 방향이나 목적(BF flip 기준 vs LLR 테이블)이 다름 |
| replica 병렬 서브디코더 | 대응 없음 | Prime ECC는 replica 개념 없이 단일 파이프라인 구조 |
| importance sampling 기반 error floor 분석 | 대응 없음 | 코드/HW 모듈이 아닌 시뮬레이션 분석 기법 |

> 적용 가치: 낮음 — hard-decision BF/QWBF 계열 알고리즘 자체는 Prime ECC의 soft min-sum 구조와 이질적이며, HW 설계나 정량적 gatecount/throughput 데이터도 없어 참고 가치가 제한적.
