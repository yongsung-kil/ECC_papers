### B. 알고리즘 요약

1. 대상: finite-length LDPC 코드(QC/cyclic-lifted 코드 포함)의 error-floor를 유발하는 trapping set(TS) 구조를 체계적으로 정의·탐색·제거.
2. 문제: 기존에는 TS가 조작적 정의(near-codeword, pseudo-codeword 등)로만 다뤄져 이론적 분석/탐색 알고리즘이 부재.
3. 핵심 정의: `k-out TS`를 유도 부분그래프 내 degree-1 check node 개수가 정확히 `k`개인 변수노드 집합으로 순수 그래프 이론적으로 재정의(SS는 0-out TS).
4. 핵심 결과 1: 임의 `k`에 대해 minimal `k`-out trapping distance(`d_{t,k}`) 결정 문제가 NP-complete임을 stopping-set 결정 문제로부터의 reduction으로 증명(Theorem 1).
5. 핵심 기법: 기존 minimal stopping-set exhaustion 알고리즘 `MSSE(H,v)`를 building block으로 사용해 minimal `k`-out TS를 완전탐색하는 Algorithm 2 제안(n~500, k=0,1,2까지 실용적).
6. 확장(suppressing effect): cyclic lifting(base code에서 K배 lifted code 생성)에서 base-code TS가 lifted code에서도 TS로 살아남을 확률(`E{|T_L|}`)을 `f(K,d)`, `g(K,d)` 보조함수로 closed-form 유도(Theorem 2, 3).
7. 핵심 식: suppressing weight `W_sup = 0.5#E - #V + 0.5#C_odd,≥3`이 생존확률 `∝ K^(0.5k - W_sup)`의 지수를 결정 → base-code 최적화의 목적함수로 사용.
8. 최적화 절차: code annealing 알고리즘(선행연구 [11])으로 base code의 최소 `W_sup`를 최대화하여 n=512 CL 코드 구성.
9. 결과: 최적화된 코드(1-out TS distance 10)가 랜덤 코드/SS만 최적화한 코드 대비 AWGN error-floor 개선(Fig.1); 단 n=504 irregular PEG 코드와 비교 시 trapping-distance 최적화만으로는 PEG를 능가하지 못함(TS distance 8 vs PEG의 실질 우수 성능).
10. 한계: 순수 이론+시뮬(HW 미설계), 소형 n(≤512~2184)만 검증, TS distance 최대화가 곧 최적 성능을 보장하지 않음을 스스로 확인(추가 특성화 필요성 명시).

| cyclic lifting 기반 base-code 최적화(W_sup 목적함수)로 error-floor 개선 | `ecc_top.cpp` `Load_PCM()` | H-matrix(base+circulant shift) 구성 단계에 이론적으로 대응되나, Prime ECC은 이미 고정된 QC-LDPC H-matrix를 로드하는 구조로 이 논문의 최적화 절차 자체를 이식하려면 H-matrix 재설계 필요(난이도 높음) |
| trapping set exhaustion 알고리즘(디코더 무관) | 대응 없음 | 부호 설계/분석 단계 도구이며 디코더 실행 코드와 대응 없음 |

적용 가치: `중간` — QC-LDPC(cyclic lifting) 구조와 직접 관련되고 error-floor 개선이라는 Prime ECC의 관심사와 부합하나, 실제 이식은 새 H-matrix 설계·재검증이 필요한 부호설계 레이어(난이도 높음)에 해당하며 HW/디코더 코드에는 직접 닿지 않음.
