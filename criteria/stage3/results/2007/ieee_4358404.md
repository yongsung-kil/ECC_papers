> 총평: RAID/분산 스토리지 계층의 disk erasure-correcting code 재구성 기법으로, NAND flash 채널의 binary LDPC ECC(Prime ECC 3.1)와는 적용 계층과 정정 대상(디스크 vs 비트)이 달라 이식 관점에서 관련성이 낮음.

### B. 알고리즘 요약

1. 시스템: K개 information disk + M개 check disk로 구성된 분산 저장시스템, systematic parity-check matrix `H = [P_{M×K} | I_M]` 기반 low-density triple-erasure correcting code(Steiner code, additive-3 code) 사용.
2. 풀려는 문제: 기존 RAID형 시스템은 check disk와 별도로 spare disk(교체용 여분 디스크)가 필요해 중복 디스크 수가 많음.
3. 핵심 가정/모델: disk 고장은 disk controller가 100% 검출하며 controller 자체는 고장나지 않는다고 가정, 고장은 absorbing Markov chain으로 모델링.
4. 핵심 기법: 고장난 disk를 spare disk 대신 check disk로 대체 - s개 disk가 고장나면 s개 check disk를 정보 복구용으로 소모하고 남은 `M-s`개 check disk로 재구성된 `H^{M-s}_K` 행렬을 유지.
5. 핵심 식: 거리-3/거리-4 부호의 code length 상한 `N ≤ 2^{M-s-1}`(거리4), `N ≤ 2^{M-s}-1`(거리3)에 따라 재구성된 행렬이 triple/double/single-erasure correcting 중 어느 것이 되는지 결정, 열벡터 일부 bit를 반전(inversion)해 distinct/odd-weight 조건 재충족.
6. 확장: disk 고장이 누적될수록 정정능력이 triple→double→single로 점진적으로 저하되는 "graceful degradation" 구조.
7. 구현 디테일: 행렬 재구성 시 check bit 갱신을 최소화하는 최적 inversion 방법은 future work로 남김(미해결).
8. 학습/최적화 절차 없음 - 순수 조합론적 행렬 재구성 규칙.
9. 결과: K=120, M=21 예제에서 제안 시스템 MTTDL = 7.9×10^4 년 (disk MTTF 0.5×10^6시간, 유지보수주기 5,376시간), 동일 조건 기존 시스템(M+L=26 중복디스크) MTTDL = 5.8×10^1 년.
10. 한계: 이론/Markov chain 해석적 평가만 제시, throughput·latency·repair time 평가는 future work로 명시. 디스크 단위 소실(erasure) 정정 문제로 NAND 채널의 비트 단위 오류정정(LDPC BP/min-sum)과 근본적으로 다른 문제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Steiner/additive-3 code, parity-check matrix 재구성 | 대응 없음 | disk-array 소실정정 코드로 binary QC-LDPC 비트 오류정정과 무관 |
| MTTDL/Markov chain 신뢰성 평가 | 대응 없음 | 시스템 레벨 disk 신뢰성 지표, ECC 디코더/HW와 무관 |
| spare disk 없는 동적 재구성 아키텍처 | 대응 없음 | Prime ECC는 NAND 채널 ECC이며 disk-array 아키텍처 개념 자체가 대응 없음 |

> 적용 가치: 낮음 — RAID/분산 스토리지 disk erasure code로 NAND flash binary LDPC ECC와 적용 계층이 전혀 다르며 Prime ECC 코드베이스에 대응되는 모듈이 없음.
