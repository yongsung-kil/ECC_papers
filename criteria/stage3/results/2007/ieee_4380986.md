### B. 알고리즘 요약

1. UWB(WIMEDIA) OFDM 시스템에 rate 3/4, codeword 1200bit의 ultra-sparse QC-LDPC 코드를 제안, 기존 CC(rate 1/3~3/4, K=7 Viterbi) 대비 채널코딩 성능 비교.
2. 문제: LDPC가 CC보다 통신성능은 우수하나 구현 복잡도·에너지 소모가 크다고 여겨짐 - 실측으로 이를 검증.
3. 채널모델은 IEEE 802.15.3a UWB 채널(CM1~CM4), MMSE 등화 후 이상적 LLR 계산 가정.
4. 디코딩은 message passing 기반 min-sum(`Λk = sign(Λl)·min(|Λl|)`)에 message scaling factor(MSF) 0.75를 곱해 성능 보정.
5. 코드 degree 분포는 `f[3,2]={3/4,1/4}`(VN), `g[11]={1}`(CN)의 ultra-sparse 구조로 실리콘 면적을 최소화(denser 분포 대비 0.5dB 손실 감수).
6. Layered decoding(TDMP, Mansour/Hocevar 방식)으로 CN 서브셋 처리 직후 갱신 메시지를 즉시 반영, 수렴속도 향상.
7. Iteration control: VNR(모든 VN 중간결과 절댓값 합) 기반 stopping criterion - VNR이 2회 연속 정체/감소하면 중단, SNR별 VNRoff 임계값으로 성능손실 0.01dB 이하 유지.
8. 채널값/외부정보 6bit, a posteriori 정보 9bit 양자화, floating-point 대비 손실 0.2dB 이하.
9. 결과: waterfall 영역 평균 iteration 최대 5.5(32% 절감), non-convergence 영역 최대 45% 절감(8회→4.5회 감소), LDPC 코딩이득 CC 대비 4dB(PER 1e-3), 두 디코더 FPGA 에너지 소모는 거의 동일.
10. 한계: FPGA 실측치를 ASIC로 직접 전이 불가(상대비교만 타당 가정), 특정 UWB 채널·시스템 파라미터 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum + MSF 디코딩 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반, MSF 개념은 LLR 테이블 참조와 유사 목적 |
| Layered decoding | `decoder.cpp` `LDPC_Decoder()` | Prime ECC의 Dual-Update 스케줄과 유사 개념, 이미 보유 |
| VNR 기반 iteration control(조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 목적(조기종료로 iteration/에너지 절감)은 동일하나 Prime ECC는 CRC 기반, 본 논문은 VNR 기반으로 판정 로직 상이 |
| 6/9bit 양자화, LLR 채널값 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-quant 방식 참고 가능 |
| QC-LDPC ultra-sparse H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | 코드 자체는 UWB 특화 설계로 재검증 부담 큼 |

> 적용 가치: 낮음 | 이유: UWB 특화 코드/채널이며 min-sum, layered decoding, 조기종료 등 핵심 기법이 Prime ECC에 이미 구현되어 있어 신규성 낮음. VNR 기반 조기종료 아이디어만 참고 가치.
