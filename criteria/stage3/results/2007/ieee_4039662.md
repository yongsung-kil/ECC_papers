### B. 알고리즘 요약

1. 대상은 (N,K,d) 이진 선형 블록 부호(BCH, Goppa 코드 등)이며 BSC 채널에서 Gallager Algorithm-B의 iteration-1을 비직교 check sum 집합에 대해 모델링.
2. 기존 Gallager 모델은 check sum들이 서로 직교(intersection이 해당 비트 하나뿐)한다고 가정하나, 실제 BCH/Goppa 코드의 parity-check 행렬은 비직교적이라 기존 모델이 적용되지 않음.
3. 핵심 가정은 dual 코드의 weight profile이 이항분포(binomial distribution)에 근사한다는 것이며, check sum들은 길이 `N`, 평균 weight `W`로 독립적으로 무작위 생성된다고 가정.
4. 핵심 기법은 오류패턴 `p`(weight `w`)에 대해 check sum이 비트 `n`에 대해 잘못된 정보를 줄 확률 `q1`, `q2`(수식 (3),(4))를 유도하고 이를 임계값 `B`와 결합한 iteration-1 오류확률 `p1(w)`(수식 (5))로 정식화.
5. `p1 < p0`(수식 (2),(5))는 반복복호가 개선 방향으로 수렴하기 시작하는 필요조건이며, 이는 코딩이 아닌 암호시스템 보안성 평가 기준으로 사용됨.
6. 확장으로 irregular(2-weight) 모델(`W1`,`W2`, `N1`,`N2` 분할)을 도입해 unequal error protection(UEP) 효과로 특정 위치의 오류를 우선 정정하는 "error pattern trapping" 기법 제시.
7. 부수 기법으로 정규분포 근사(Gaussian approximation, 수식 (15)-(21))를 이용해 필요한 최소 check sum 개수 `Jmin`(Proposition 2.1, 2.2)을 폐형식으로 유도.
8. 별도의 학습 절차는 없으며, McEliece PKC의 (1024,524,102) 확장 Goppa 코드에 대한 실제 공격 사례로 검증.
9. 결과: (127,64,21) BCH 코드 시뮬레이션에서 `W=24`, `J=3264`, `t=8` 오류에 대해 최적 임계값 `B=1675`에서 1회 iteration 후 WER `~10^-1.5`; McEliece 공격에서 1000개 메시지에 대해 기존 information-set 공격(2^77.7 연산) 대비 제안 방식 2^73.4 연산으로 감소.
10. 한계: LDPC 코드는 McEliece 암호시스템에 보안 수준이 부족해 논문 프레임워크에서 명시적으로 제외됨. 대상이 암호분석(BCH/Goppa)이며 HW 구현이나 error-floor/waterfall 등 통신용 LDPC 디코더 성능 개선과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping 복호 (McEliece 공격용) | 대응 없음 | Prime ECC는 min-sum 기반 soft-decision 디코더이며 hard bit-flipping 알고리즘 및 암호분석 목적과 무관 |
| 비직교 check sum 모델링 | 대응 없음 | BCH/Goppa 코드의 dual 코드 weight profile 분석으로, QC-LDPC 부호설계(`Load_PCM()`)와 목적이 다름 |

> 적용 가치: 낮음 — McEliece 암호시스템 공격 분석 논문으로 LDPC ECC 자체를 다루지 않아 NAND binary QC-LDPC 이식 요소가 없음.
