> 총평: finite-length LDPC의 waterfall 성능을 사이클 무시하고 채널 변동성만으로 예측하는 해석적 분석기법으로, 부호설계나 디코더 개선이 아닌 순수 성능 예측 도구라 이식 대상이 아님.

### B. 알고리즘 요약

1. 대상은 finite-length regular/irregular LDPC 앙상블, 채널은 BEC/BSC(1차원 파라미터)와 BIAWGN(다차원)까지 포괄; 코드 예시는 (3,6)-regular, N=1024~100000 및 rate 1/2 irregular(`λ(x)=0.4x²+0.4x⁵+0.2x⁸`, `ρ(x)=x⁸`).
2. 기존 density evolution/Gaussian approximation은 무한길이 점근분석에 유효하지만, 유한길이 코드의 성능은 factor graph의 사이클·stopping set 때문에 예측이 어려웠음(BEC 조합론적 분석은 계산복잡도가 매우 큼).
3. 핵심 가정은 "복호 실패는 관측된 채널이 해당 코드의 복호임계값보다 나쁠 때 발생한다"는 것이며, 사이클 효과는 무시.
4. 핵심 기법: 코드워드마다 관측되는 채널 파라미터(관측 오류확률 `Pobs` 또는 관측 채널용량 `Cobs`)를 랜덤변수로 모델링, 유한 N에서의 분포(이항분포→가우시안 근사 `N(μ,σ²)`)를 해석적으로 도출.
5. 핵심 식: block error probability `PB(C(σ),N,λ,ρ) = 1 - F_Pobs(C(σ),N,p_th)` — 관측 채널이 무한길이 임계값 `p_th`보다 나쁠 확률로 블록 오류율을 근사.
6. 확장 기법: block error probability로부터 EXIT chart 분석(`φ(x)` Gaussian approximation, 식(16)(17))을 이용해 bit error probability까지 유도, 계수 `α`(복호 실패 시 잔여 비트오류율)를 도출.
7. 부수 트릭: 다차원 채널(BIAWGN)에서도 단일 파라미터(Pobs 또는 Cobs)로 근사 가능함을 실증, 파라미터 fitting이나 추가 curve-fitting 불필요(scaling law [8] 대비 단순함).
8. 최적화 절차는 없음(수치적분으로 mean/variance만 계산); 대신 코드 설계 가이드라인 2가지(목표 BLER 만족 rate 선택 (19), 목표 threshold 선택 (20))를 부가 제시.
9. 결과: (3,6)-regular, N=10⁴, BIAWGN에서 BER 10⁻⁵ 기준 예측-실측 gap 약 0.032dB; N≥50,000에서 예측오차 소실, N>2500에서 오차 0.1dB 이내; irregular 코드(N=10⁴)에서도 gap 약 0.035dB.
10. 한계: waterfall 영역에서만 검증(error-floor 미다룸), 순수 성능 예측/분석 도구로 부호설계나 디코더 알고리즘 개선을 제시하지 않으며 HW 구현은 전혀 다루지 않음; 고도로 최적화된 코드(EXIT chart가 여러 점에서 닫히는 경우)에는 정확도 저하.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| finite-length 성능 예측(waterfall) | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 직접 시뮬레이션하여 성능을 얻으므로 해석적 예측 도구가 불필요 |
| EXIT chart 기반 bit error probability 유도 | 대응 없음 | 알고리즘/HW 구현이 아닌 이론적 분석 기법으로 코드 모듈에 대응 없음 |
| 채널 파라미터 모델(Pobs, Cobs) | `channel.cpp` | AWGN/RBER 채널 시뮬 구조와 개념적 유사성은 있으나 본 논문은 통계적 예측 이론이라 코드 이식 대상 아님 |

> 적용 가치: 낮음 — 부호설계/디코더/HW 어느 것도 개선하지 않는 순수 성능 예측(해석적 분석) 이론이며 Prime ECC는 직접 시뮬레이션으로 성능을 확인하므로 실질적 이식 가치 없음.
