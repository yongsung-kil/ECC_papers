### B. 알고리즘 요약

1. 대상 부호: 규칙적(regular) 랜덤 LDPC (Gallager 예제 n=20, ρ=4, γ=3 및 시뮬레이션에서 n=3~2700, rate 0.23~0.67), 이진 소거 채널(BEC) 가정으로 인코딩 문제만 다룸(복호 성능과 무관).
2. 문제: LDPC는 PCM으로 정의되어 생성행렬(GM)이 sparse하지 않아 고전적 행렬곱 인코딩이 O((n-k)·k)로 고비용이며, 그래프기반 메시지패싱 인코더는 PCM에 stopping set이 있으면 동작하지 않음.
3. 가정: 정보비트는 알려짐, 패리티비트는 소거(erased)로 간주해 BEC 상의 BP 디코더를 인코더로 재사용하는 기존 접근([7])을 전제로 함.
4. 핵심 기법: "key set" 개념 도입 — 변수노드 부분집합 K가 존재해 V\K 내 최대 stopping set이 공집합이 되면, K 이외 비트(비-key 패리티 비트)는 메시지패싱으로 복구 가능.
5. 핵심 식/개념: `key set`은 그리디 알고리즘(Algorithm A~D, 최저차수 check node부터 순차 처리)으로 근사 탐색 — 최소 개수의 key parity bit만 알면 stopping set이 사라짐을 이용해 PCM에 소수의 새 행(패리티비트를 정보비트만의 함수로 표현)을 추가.
6. 확장: PCM 열 재배열(non-key 먼저, key 나중) 후 reduced echelon form을 구해 key-parity-bit을 정보비트의 함수로 산출, 불필요한(선형종속) check node는 삭제.
7. 부수 트릭: 추가되는 행은 sparse하지 않을 수 있으나 개수가 적어 전체 인코딩 복잡도는 blocksize에 거의 선형(O(n) 근접) — Gallager n=20 예제로 상세 단계(Table I) 시연.
8. 최적화 절차: "optimum" key set 탐색은 NP-hard급으로 어려워 그리디 근사(최저차수 check node 우선 처리) 사용.
9. 결과: 100회 무작위 생성 실험(Table II)에서 key parity bit 수는 전체 parity bit 대비 평균 약 2~34% 수준(예: n=2700, rate 0.67에서 평균 13.4개/약 2.16%)으로 상대적으로 작음을 시뮬레이션으로 확인.
10. 한계: 순수 code-design/encoder 레이어 기법으로 복호 성능(BER/error-floor)에는 기여하지 않음(BEC 소거채널 가정), HW 구현·게이트/throughput 검증 없음, QC-LDPC 등 구조화된 부호가 아닌 랜덤 regular LDPC로만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PCM 수정(행 추가/삭제)을 통한 stopping-set-free 인코딩 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 단계에 개입하는 code-design 레이어 기법으로 이식 난이도 높음 |
| 메시지패싱 기반 그래프 인코더 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 이미 dual-diagonal 고정 인코딩(`PCM_b_enc`)을 사용 중이라 대체 이식보다는 개념적 참고 수준 |
| key set 그리디 탐색(열 재배열, 불필요 check node 삭제) | `encoder.cpp` `Column_Reordering_Dual_Update()` | 열 재배열 개념은 유사하나 목적(stopping-set 제거 vs dual-update)이 달라 직접 대응 아님 |
| 복호 성능 관련 항목 | 대응 없음 | 논문이 BEC 소거채널 기반 인코딩 문제만 다루어 디코더/정정성능과 무관 |

> 적용 가치: 낮음 — 대상이 encoder이고 정정능력·error-floor와 무관하며(N/A), PCM 구조 자체를 수정하는 code-design 레이어 개입이라 이미 고정 QC-LDPC dual-diagonal 인코더를 보유한 Prime ECC에는 직접 이식보다는 참고 수준의 가치만 있음.
