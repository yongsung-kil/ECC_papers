### B. 알고리즘 요약

1. 대상: irregular/regular LDPC, rate-1/2 (1008,504) 예제 및 고rate(0.8~0.93대) 코드 구성. AWGN + BPSK + BP 디코딩(최대 80회 반복).
2. 문제: 기존 girth-conditioning(PEG 등)은 모든 심볼노드를 동등하게 처리해, 고차수 노드에서 girth 확보가 어렵고 저차수 노드(주 성능 기여자)의 girth 최적화가 충분치 않음.
3. 가정: error-floor는 주로 저가중치 codeword와 (특히 저차수 노드로 구성된) small trapping set에 기인.
4. 핵심 기법 1단: Tanner 그래프의 girth histogram 측도 `F(G) = Σ_{d,g} w_{d,g} N_{d,g}` 정의 (`N_{d,g}`=차수 d, local girth g인 심볼노드 수, `w_{d,g}`는 가중치).
5. 의미: 가중치 `w_{d,g}`를 저차수·저girth 조합에 크게 부여해, PEG의 edge 선택 시 저차수 노드의 girth를 우선 확보하도록 유도.
6. 확장: 가중치 선택 2가지 제안 - 일반형 `w_{d,g}=α^d β^g` (식2), 최소차수 노드만 겨냥하는 `w_{d,g}=-1 (d=d_min, g=g̃_min)` (식3).
7. 구현 디테일: PEG의 spreading subgraph를 변형해 candidate edge(s_j,c_i) 추가 시 girth histogram 변화를 근사 추정(모든 edge를 한 번씩만 확장); 고rate 구성 시 bit-filling 알고리즘과 결합.
8. 최적화 절차: 없음(학습 기반 아님) - 알고리즘적 edge 선택 휴리스틱(Algorithm 1).
9. 결과: irregular(1008,504) 코드에서 degree-2 노드 평균 local girth 6.0→7.0로 개선, error-floor 영역(2.7dB)에서 detected error 개선; 고rate 구성 시 extended bit-filling 대비 block length 최대 약 10% 증가(ds=4, m=444).
10. 한계: 시뮬레이션만 수행(HW 미설계), 특정 파라미터(ds,dc,g0) 조합에서만 검증, 다른 개선 PEG 변형([8])과 성능이 유사함을 저자 스스로 언급.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| trapping-set 크기 확대를 통한 error-floor 개선 | 대응 없음 | 디코더 알고리즘이 아닌 부호 구성 단계 개선이라 decoder.cpp 로직과 직접 연결 없음 |
| bit-filling 기반 고rate 구성 | `ecc_top.cpp` `Load_PCM()` | 고rate 부호(현재 ~0.9) 설계 시 참고 가능한 구성법 |
