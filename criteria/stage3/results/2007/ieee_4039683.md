### B. 알고리즘 요약

1. m차원 Euclidean geometry EG(m,p) (p는 소수)의 점·선·초평면 구조를 이용해 코드를 구성한다.
2. 기존 random-like LDPC는 컴퓨터 탐색 비용이 크고, 유한 길이에서 girth/degree distribution 최적화가 어렵다는 문제를 해결하고자 함.
3. 핵심 가정: geometry의 parallel bundle of (μ-1)-flats로 EG를 분해하고, flat들을 잇는 connecting line들의 incidence vector로 순열행렬을 구성.
4. Type-I/Type-II 두 방식으로 `c×c` 배열의 permutation matrix subarray `H(c,c)`를 뽑아 RC-constraint(두 행/열이 1을 최대 1개 공유)를 만족하는 `γ`-regular 행렬을 얻는다.
5. RC-constraint는 girth≥6과 최소거리 하한(`d_min ≥ γ+1`)을 동시에 보장하는 핵심 성질.
6. Masking 기법: 작은 masking matrix `Z`를 base matrix `H_b`에 성분별 곱(Hadamard-like)해 밀도를 낮추고, `Z`가 순열행렬로 구성되면 girth가 보존되며 irregular degree distribution도 구현 가능.
7. Irregular 구성 시 무한길이 최적 degree distribution을 유한길이에 맞게 조정(과도한 degree-2 노드를 상위 degree로 전환)해 error floor를 완화.
8. 최적화 절차는 별도 GA/DE 없이 masking matrix의 column/row weight 분포를 목표 degree distribution에 맞춰 컴퓨터 탐색으로 근사 구성.
9. 결과: (4096,3687) 코드가 SPA 디코딩 시 Shannon limit 대비 1.3dB, error floor는 BER<10^-13; (7680,6144) irregular 코드는 error floor 없이 BER 10^-10까지 확인.
10. 한계: 순수 시뮬레이션(이론+SPA 시뮬)만 제공, HW 구현/게이트카운트/throughput 전혀 없음. OSMLG/WBF는 대안 디코더로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Euclidean geometry decomposition 기반 base matrix 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 교체하는 수준으로, 기존 QC-LDPC 고정 구조와 재검증 부담 큼 |
| Masking 기법(irregular degree distribution 구현) | `ecc_top.cpp` `Load_PCM()` | 부호구조 설계 단계에 해당, 디코더/인코더 로직과는 무관 |
| SPA(sum-product) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 이미 Min-Sum 근사를 사용하므로 SPA 자체는 대응 없음(중복) |
| OSMLG / WBF hard-decision 디코딩 | 대응 없음 | Prime ECC는 min-sum 기반 soft 디코더 중심, 별도 majority-logic 디코더 없음 |

> 적용 가치: 낮음 — 순수 부호구성(geometry+masking) 논문으로 H-matrix 전면 교체가 필요하고, 디코더/HW 기법 기여가 없어 Prime ECC 3.1에 이식할 실질적 레이어가 마땅치 않음.
