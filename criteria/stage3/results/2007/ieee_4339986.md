### B. 알고리즘 요약

1. 대상은 L-step majority-logic decodable(LSMLD) 코드(예: (7,4) Hamming, (63,45) BCH, (1023,813) twofold EG code) — LDPC가 아닌 전통적 대수 코드.
2. 문제: BCH 등 전통적 코드는 parity check density가 높고 표준 Tanner graph에 4-cycle이 많아 BP 디코딩 성능이 나쁨.
3. 채널은 AWGN + BPSK 변조, 성능 비교 대상은 hard-decision decoding(HDD)과 표준 BP.
4. 핵심 기법: **L-step orthogonalization**으로 원래 parity-check 행렬 `H`를 GPC(generalized parity check) 행렬 `H~`로 변환, 이때 추가 variable node(채널 정보 없는 노드)가 도입됨.
5. 핵심 식/의미: 변환된 `H~`는 원래 `H`보다 sparse하고 4-cycle이 없어(2-step orthogonalization으로 보장) BP에 적합한 orthogonal Tanner graph를 형성 — 예제 (7,4) Hamming code에서 `H`(3×7) → `H~`(6×9)로 확장.
6. 핵심 기법 2단(BP-O 디코딩): 추가 variable node는 첫 반복에서만 다른 노드로부터 belief를 전달받고(자신은 channel LLR 없음), 2번째 반복부터는 표준 BP와 동일하게 동작.
7. 부수 디테일: 각 check node는 orthogonal Tanner graph에서 정확히 1개의 추가 variable node와 연결되도록 구성(2-step orthogonalization 한정 논의).
8. 학습/최적화 절차: 없음(고전 코드의 대수적 구조 기반 변환, DE/GA 없음).
9. 결과: (63,45) BCH code에서 BP-O가 HDD 대비 BER 1e-6에서 1.5dB 이득, union bound 대비 0.4dB 이내; (1023,813) EG code에서 hard-decision TSMLD 대비 2.2dB 이득(BER 2e-6), 동일 길이/rate의 무작위 LDPC와 동등 성능.
10. 한계: 코드 표현이 커짐(variable/check node 수 증가로 복잡도 상승), HW 설계 없음, 순수 시뮬레이션 결과뿐이며 LDPC 자체가 아닌 대수 코드 대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Orthogonal Tanner graph 변환(4-cycle 제거) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC로 이미 설계 단계에서 4-cycle을 회피하는 구조이므로 이식 난이도 높고 실익 낮음 |
| BP-O 디코딩(추가 variable node 처리) | 대응 없음 | BCH/EG 코드 전용 GPC 행렬 변환이며 QC-LDPC 디코더 구조와 무관 |
| BP 알고리즘 자체(표준 BP) | `decoder.cpp` `LDPC_Decoder()` | 이미 Prime ECC가 min-sum 기반 BP 변형을 보유해 중복 기여 |

> 적용 가치: 낮음 — BCH/EG code 등 non-LDPC 대수 코드에 BP를 적용하기 위한 Tanner graph 변환이 핵심이며, Prime ECC의 QC-LDPC 부호설계·디코더·HW 구조 어디에도 직접 대응되지 않음.
