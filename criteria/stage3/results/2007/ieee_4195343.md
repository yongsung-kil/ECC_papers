### B. 알고리즘 요약

1. 정규(regular) QC-LDPC 모부호 `H_QC`(circulant 배열, RC-constraint로 girth≥6 보장)를 기반으로 다중 rate 부호를 생성하는 구성법을 제안.
2. 문제의식: 기존 partially-parallel 디코더 아키텍처는 regular QC-LDPC에만 적용 가능해 irregular 부호에는 적용이 어렵다는 한계.
3. 채널 모델은 AWGN, BPSK 기반 BP(belief propagation) 표준 디코딩 가정.
4. 핵심 기법: 모부호 `H_QC`의 하단에 새 circulant 행 `C_m+1,i, ..., C_m+i,n-m+i`를 추가해 새 parity-check matrix `H_QC'`를 구성, 이때 원 codeword가 만족하던 관계는 확장 부호에서도 유지되어 모부호 디코더로 복호 가능.
5. 새 행의 weight는 원 row weight `p`와 동일하게 균일 배분(threshold 성능을 위해 uniform check-node degree가 유리하다는 stability condition `λ2 < λ2*` 이론 근거).
6. 확장 시 full row rank 및 RC-constraint(girth 4 방지) 유지를 제약조건으로 부여, 랜덤 생성 후 rank check·girth 평균·시뮬레이션으로 최적 후보 선택.
7. 인코딩은 systematic-circulant(SC) 형태의 생성행렬을 이용한 shift-register-adder-accumulator(SRAA) 회로로 구현, serial/parallel/two-stage 3가지 인코더 트레이드오프 제시.
8. 별도 학습/최적화 알고리즘 없음(랜덤 생성+rank/girth 검증 방식).
9. 결과: rate 8/16~8/19 부호군(길이 9216~21628)의 BER 시뮬레이션(AWGN, Es/No 1.2~1.8dB)에서 성능 유지 확인, 인코딩 복잡도는 parity bit 수(mc) 또는 code length(nc)에 선형 비례.
10. 한계: 디코더 자체는 기존 partially-parallel 구조(CNFU/VNFU 재매핑)를 그대로 재사용해 새 디코딩 알고리즘 기여 없음, HW 합성/FPGA 검증 없이 시뮬레이션 수준에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 모부호 행 확장을 통한 multi-rate 부호 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 다루므로 이식 난이도 높음, 참고 수준 |
| SRAA 기반 QC-LDPC 인코딩(serial/parallel/two-stage) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 인코딩 고정 구조라 SRAA 방식과 상이 |
| partially-parallel CNFU/VNFU 재사용(디코더 공유) | `decoder.cpp` `LDPC_Decoder()` | 표준 BP 기반 디코더로 Prime ECC의 min-sum 구조와 직접 대응되지 않음, 신규 기여 없음 |
| RC-constraint/girth 4 방지 설계 | 대응 없음 | H-matrix 설계 이론으로 코드 모듈 대응 없음 |

적용 가치: 낮음 — 순수 부호 구성법/인코더 회로 논문으로 디코더 알고리즘 개선이나 HW 최적화 기여가 없고, 표준 QC-LDPC 인코딩·디코딩 기법 재사용 수준.
