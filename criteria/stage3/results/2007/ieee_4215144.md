### B. 알고리즘 요약 (10줄 내외)

1. 대상은 LDPC 코드의 Tanner graph에서 정의되는 stopping set/stopping distance 계산 문제.
2. STOPPING SET(주어진 크기 t의 stopping set 존재 여부)과 STOPPING DISTANCE(크기 t 이하의 stopping set 존재 여부) 두 결정 문제를 다룸.
3. 기존에 stopping distance를 다항시간에 계산하는 알고리즘이 알려져 있지 않았고, 이 논문은 그것이 불가능함(NP-hard)을 증명.
4. VERTEX COVER 및 VERTEX COVER(=) 문제로부터 각각 STOPPING DISTANCE, STOPPING SET으로의 다항시간 many-one reduction을 구성.
5. 원래 그래프 `G=(V,E)` (|V|=n, |E|=m)로부터 vertex-edge incidence graph를 확장한 `G''`를 `O(mn)` 시간에 구성.
6. `G''`는 원본 incidence graph에 `m+1`개의 `V` 복사본(`L_1..L_{m+1}`, `R_2..R_{m+1}`)을 연결한 구조로, 특정 정점을 포함하는 stopping set이 반드시 관련 정점 집합 전체를 포함하도록 강제(Observation 1).
7. Lemma 2/3을 통해 `G''`의 stopping set 크기가 `t(m+1)+m` 꼴일 때만 원본 `G`의 vertex cover(크기 t)와 정확히 대응함을 증명.
8. 최적화/학습 절차 없음 (순수 조합론적 증명).
9. 결과: STOPPING SET, STOPPING DISTANCE 모두 NP-complete이며, 따라서 P≠NP이면 Tanner graph의 stopping distance를 다항시간에 계산하는 알고리즘은 존재하지 않음(Corollary 3).
10. 한계: 알고리즘/부호설계/HW 구현 전혀 없음, 시뮬레이션·수치실험도 없는 순수 이론(증명) 논문.
