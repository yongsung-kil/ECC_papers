### B. 알고리즘 요약

1. Full-parallel LDPC 디코더, (6,32) (2048,1723) QC-LDPC(10GBase-T 채택) 및 (6,72) (5256,4823) regular QC-LDPC 코드 대상.
2. 문제: full-parallel 디코더는 row/column processor 간 배선(interconnect)이 방대해 면적·전력 부담이 큼. 기존 Split-Row(2-way)는 이를 완화하지만 추가 절감 여지가 있음.
3. 핵심 가정: check node(row) 연산의 magnitude 근사(MinSum)는 파티션별로 나눠 계산해도 부호(sign) 정보만 파티션 간 공유하면 근사적으로 성립.
4. 핵심 기법: parity check matrix를 열방향으로 `Spn`개 파티션(`H_split-Spn`)으로 나누고, 각 파티션 내부에서 magnitude 연산을 독립 수행, sign 정보만 파티션 간 배선으로 교환(Multi-Split-Row).
5. 핵심 식: `α_ij^Spn = S × Π sign(L_j') × φ(Σ φ(|L_j'|))` (파티션 내부 합), magnitude가 SPA 대비 커지므로 scale factor `S<1`로 정규화.
6. 확장: sign wire 수는 `2(Spn-1)M`으로 전체 배선(`2bMWr + 2(Spn-1)M`) 대비 미미(Split-Row-4에서 2% 미만)해 대부분의 절감은 magnitude 배선에서 발생.
7. 부수 트릭: row processor 면적은 mux(`Wr/Spn`), XOR(`Wr/Spn+1`), comparator(`Wr(2Spn)-1`)로 파티션 수에 따라 선형~약간 빠르게 감소, column processor 면적은 불변.
8. 최적화 절차: 별도 학습 없음. scale factor `S`는 실험적 최적값 사용(SplitRow-4: S=0.2).
9. 결과: 0.18um CMOS 합성 결과, (2048,1723) 코드에서 Split-Row-4가 표준 MinSum 대비 면적 3.2배 축소, throughput 5.2배 증가(7.1Gbps, 15 iteration), BER 열화 0.65dB@BER 6×10^-7. Split-Row-12는 error floor 발생(~2×10^-6).
10. 한계: regular 고정 row-weight LDPC 전제, 파티션 수 증가에 따른 BER 저하 트레이드오프, irregular/비대칭 구조 코드(Prime ECC류)에 대한 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum row 연산 파티션 분할 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | magnitude 연산 분할 + sign 별도 처리 아이디어가 CN 업데이트 구조와 직접 맞닿음 |
| 파티션 간 sign wire만 교환하는 HW 라우팅 절감 | 디코더 HW 모델(z=32, HCU, GT) | GT(Graph Thinning)와 목적 유사(배선/edge 절감)하나 접근 방식(row split vs MUX 기반 edge 선택)은 상이 |
| scale factor 정규화 | `decoder.cpp` `Get_VNU_Table_Idx()` / LLR 테이블 | 기존 min-sum normalized/scaling factor와 유사 개념, 이미 보유 기법과 중복 가능성 |

> 적용 가치: 중간 — CN 연산 파티션 분할로 wire/area를 줄이는 아이디어는 HW 면적 개선에 참고 가치가 있으나, Prime ECC는 이미 z=32 row 단위 병렬 + GT로 배선을 축소한 상태라 중복 검토가 필요하고 regular 코드 전제도 재해석 필요.
