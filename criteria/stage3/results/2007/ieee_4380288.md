### B. 알고리즘 요약

1. Magnetic recording 채널 대상 rate-8/9 이상, 512B~4096B user data, column weight 4 regular-(4,36) QC-LDPC 코드.
2. 기존 min-sum 병렬 디코더는 V2C 메시지를 전부 저장해야 하고(H행렬의 1의 개수만큼), CN 처리에 명시적 sorter가 필요해 rate가 높을수록 면적 병목이 커짐.
3. min-sum은 CN 처리에서 `tanh` 기반 함수 대신 min1/min2만 사용하는 근사(식 (2))로, C2V 메시지가 최소·차소 두 값만 가지는 성질을 이용.
4. 제안하는 "Transformed Min-Sum Algorithm"(Algorithm 2)은 VN 처리와 CN 처리를 인터리빙해, 새로 갱신된 V2C 메시지가 저장 없이 바로 CN 처리 유닛에 흡수되도록 실행 순서를 재배치.
5. CN 처리를 순차화(serialize)하여 min1, min2, 신호(sign), 최소값을 제공한 VN 인덱스만 추적하면 되므로 명시적 sorter 하드웨어가 필요 없음.
6. 아키텍처는 병렬화 인자 `P`개의 VNU와 `m/P`개의 SCNU(serial check node processing unit)로 구성, 레지스터 배열 A/B의 이중 버퍼로 VNU/SCNU가 동시 동작(concurrent) 가능.
7. 4bit 양자화 사용, 4-bit이 error floor 회피에 충분함을 인용(6-bit 대비 0.2dB 손실).
8. 별도 학습/최적화 절차 없음 (H-matrix는 고정 구성, DE/GA 미사용).
9. 65nm ASIC 합성 결과 16 iteration 기준 2.1Gbps throughput, 4가지 코드워드 길이(512B~4096B)에서 각각 1.92mm2, 2.32mm2, 2.81mm2, 3.78mm2.
10. 순수 알고리즘 개선(정정능력)은 없고 HW 면적/처리량 트레이드오프에만 초점; 시뮬레이션이 아닌 합성 결과 기반이나 magnetic recording 채널 고정 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Transformed min-sum (V2C 미저장, VN/CN 인터리빙) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC도 이미 min-sum + C2V/V2C 파이프라인 구조 채택, 재배치 아이디어는 참고 수준 |
| CN 순차 처리(sorter-free, min1/min2 추적) | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min-sum 기반 min1/min2 연산 보유, 신규 기여도 낮음 |
| 4bit 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 이미 다양한 word-length 지원, 대응 가능하나 신규성 없음 |
| VNU/SCNU 파이프라인, 이중 레지스터 버퍼 | 디코더 HW 모델 (z=32, HCU, GT) | pipeline 구조 유사, 세부 정합 필요 |

적용 가치: 낮음 - 이미 Prime ECC가 min-sum + 파이프라인 구조를 보유하고 있어 알고리즘적 신규 기여가 크지 않으며, HW 재배치 아이디어만 참고 가능.
