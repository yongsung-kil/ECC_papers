### B. 알고리즘 요약

1. (2209,1978) array-based LDPC 코드(regular, column weight γ=5, p=47), BPSK+AWGN 채널, sum-product 디코딩(Φ 함수 기반) 대상.
2. 문제: 유한 wordlength(6-bit) quantization이 error floor를 지배하는 absorbing set 구성에 어떻게 영향을 주는지 규명 안 됨.
3. 가정: absorbing set은 부호 H행렬에 내재된 구조이며, 각 변수노드는 `p:q` (satisfied:unsatisfied check 연결 수) 형태로 분류, q가 작을수록 강한(strong) absorbing set.
4. 핵심 기법: 균일 4.2 quantization을 두 도메인(Domain A: 4.2, Domain B: 1.5)으로 분리해 Φ1/Φ2 함수 입출력 양쪽에서 정밀도를 다르게 할당하는 "adaptive quantization" — 동일 6-bit wordlength 유지.
5. `L_pr(x_i)=2y_i/σ²` — 채널 LLR 표현, 4.2 quantization은 correct prior의 76%를 clip하는 반면 incorrect prior는 0.071%만 clip해 비대칭적 손실 발생.
6. 확장: extrinsic message 포화(saturation) 레벨(Φ2(0))을 조정해 강한 absorbing set만 남도록 유도 — 낮은 SNR에선 낮은 포화레벨, error floor 영역에선 높은 포화레벨이 유리(SNR별 적응형 saturation).
7. 구현 디테일: 6.0/1.5 quantization(Domain A를 6.0로 확장)이 4.2/1.5보다 error floor를 더 낮춤 — correct prior의 과도한 clipping 방지가 핵심.
8. 최적화 절차 없음(경험적 도메인 분리 설계, DE/GA 미사용); 58개 오류 샘플 기반 empirical absorbing set 카운팅으로 검증.
9. 결과: 4.2 quantization에서 지배적이던 (4,8) absorbing set이 4.2/1.5 적응형에서 거의 제거되고 더 강한 (6,8)/(8,6)이 남음; 10 iteration만으로도 200 iteration 대비 성능 손실 미미(대부분 오류는 absorbing set, non-convergence 아님).
10. 한계: FPGA 에뮬레이션 기반이나 HW 아키텍처(게이트/throughput) 설계는 없음(순수 알고리즘/분석), array-code와 RS-LDPC 두 부호에서만 검증, min-sum 계열 확장은 future work로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| adaptive quantization (Φ 함수 도메인 분리, 4.2/1.5) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read LLR 양자화 threshold 설계에 직접 참고 가능 (Prime ECC soft-2~3bit 지원과 연관) |
| iteration별 extrinsic message saturation 적응 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC의 iteration별 LLR threshold 테이블과 개념적으로 유사 — 비교/개선 참고 가능 |
| absorbing set 기반 error-floor 분석 | `decoder.cpp` `LDPC_Decoder()` (min-sum 계열) | 이 논문은 sum-product(Φ함수) 기반이라 min-sum(Prime ECC)엔 직접 적용 불가, 개념(약한 absorbing set 억제)만 이식 가능 |
| 채널 prior 표현/clipping | `channel.cpp` | AWGN 채널 clip 비대칭 문제는 Prime ECC의 RBER/fixed-error 채널 모델과 다름 — 참고용 |

> 적용 가치: 높음 — soft-read LLR quantization 설계와 error-floor 억제라는 목표가 Prime ECC의 LLR 테이블/soft-quant 설계와 직결되며, 특히 균일 quantization 대비 비대칭·적응형 quantization으로 error floor를 낮춘다는 결론은 NAND soft-read threshold 최적화에 바로 응용 가능.
