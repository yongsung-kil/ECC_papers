> 총평: 표준(802.16e) QC-LDPC의 rate/length 가변 인코더를 FPGA에 파이프라인 구현한 것으로, 인코딩 알고리즘 자체는 Richardson-Urbanke류 dual-diagonal 근사이며 Prime ECC가 이미 채택한 접근과 유사해 신규성 낮음.

### B. 알고리즘 요약

1. 시스템: IEEE 802.16e 표준 QC-LDPC, base matrix `H_b`(m_b×n_b, n_b=24) 확장 factor `z=n/24`, code rate 1/2, 2/3, 3/4, 5/6, code length 576~2304.
2. 문제: 표준 LDPC 인코딩(`P = H_b2^-1 H_b1 S`)은 quadratic 복잡도 및 H_b2^-1의 dense화 문제로 하드웨어 구현이 비효율적.
3. 핵심 가정: base matrix `H_b`를 `H_b1`(systematic, size m_b×k_b)과 `H_b2`(parity, size m_b×m_b)로 분할, `H_b2`를 다시 `H_b2a`(홀수 weight 3-column, 상하단 동일 shift) + `H_b2b`(dual-diagonal 구조)로 분해.
4. 핵심 기법: `V = H_b1·S` 벡터 계산 후 `H_b2·P = V`를 순차적으로 풀어 parity bit 계산 (`p0 = v0+v1+...+v_{mb-1}` XOR, 이후 `p1=v0-c(0,0)p0` 등 재귀식).
5. 핵심 식 의미: dual-diagonal 구조 덕분에 `H_b2` 역행렬 계산 없이 XOR·circular shift 연산만으로 parity를 순차 도출 가능 → HW 면적 절감.
6. 확장: 두 단계(V 계산, P 계산)를 2-stage 파이프라인으로 겹쳐 처리해 처리량 향상 (첫 블록 P 계산 중 다음 블록 V 계산 동시 수행).
7. 부수 트릭: shift 값이 z/2보다 크면 우측, 작으면 좌측 circular shift로 최소 클록수 사용(z/2 클록/스텝); rate/length에 따라 base matrix shift 값을 스케일링(수식 9,10)해 재사용.
8. 학습/최적화 절차 없음(고정 표준 구조 기반 설계).
9. 결과: Stratix FPGA(EP1S25/80) 합성 결과 LE 3391~12727, throughput 3~6Gbps(I/O 직렬화 미고려), I/O 직렬화 고려 시 상수 422Mbps(선행연구 2.6Mbps 대비 개선).
10. 한계: FPGA 합성 결과만 제시(ASIC 미검증), 인코더만 다루며 디코더/정정능력 관련 내용 없음, IEEE 802.16e 고정 표준 구조 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-diagonal 기반 parity 계산 (H_b2a/H_b2b 분해) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal(shuffled 옵션 포함) 인코딩을 이미 채택 — 개념적으로 중복 |
| V=H_b1·S 벡터-벡터 곱 (circular shift 기반) | `encoder.cpp` `Generate_PCM_encoding()` | QC circulant shift 곱 연산으로 유사한 부호 PCM 처리 로직에 대응 |
| rate/length 가변(base matrix scaling) 설계 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(H-matrix) 고정 — 가변 rate/length 지원은 별도 재검증 필요 |
| 2-stage 인코딩 파이프라인 | `encoder.cpp` | 파이프라인 개념은 이미 pipeline 6-stage 구조로 반영됨 |

> 적용 가치: **낮음** — dual-diagonal 인코딩 자체는 Prime ECC가 이미 보유한 기법이며, 이 논문의 주된 기여(802.16e 표준 rate/length 가변성)는 Prime ECC의 특정 QC-LDPC 고정 구조와 맞지 않아 재검증 부담이 큼.
