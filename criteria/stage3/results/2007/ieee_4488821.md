### B. 알고리즘 요약

1. IEEE 802.11n 표준의 648-bit, rate-1/2 irregular LDPC 부호(12x24 sub-block, 각 27x27 circulant) 대상, partial-parallel 디코더 구조.
2. 기존 regular LDPC용 고효율 message-passing schedule(Ref.[8])을 irregular 코드에 그대로 적용하면 column weight가 균일하지 않아 불필요한 column 연산(β message 재계산)이 다수 발생해 면적·전력이 커지는 문제.
3. row 연산은 pipeline 방식으로 매 `wrMAX+1` clock마다 수행하고, 그 사이 column 연산(BFU)이 동시 진행되는 구조를 가정.
4. 핵심 기법 1단(Memory Access Reduction): 각 row 연산(OR1 등) 직전에 필요한 β message는 여러 번 갱신된 값 중 "가장 최근에 갱신된 것" 하나뿐이므로, 이 최신 값만 기록하고 나머지 중복 갱신은 생략 — 논문 예시에서 β 기록 수를 기존 대비 1/5로 축소.
5. 핵심 기법 2단(Column Operation Reduction): row 순서(예: OR4→OR1→OR2→OR3)와 각 sub-block의 shift 값 차이를 분석해, 특정 row 연산 전에 실제로 필요한 column 연산 개수(`Rj_wc`, update number)만 수행 — 기존 대비 약 14% column 연산 감소.
6. BFU(Bit-node Function Unit) 구성: column weight `wc`별로 필요한 BFU 개수를 `Kwc = ceil(Nwc / wrMAX)` 식으로 산정해 동일 column weight의 column 연산을 배열 공유(BFU array)로 처리, 국소 과부하/유휴 불균형 방지.
7. row 연산 시간은 최대 row weight(`wrMAX`) 기준으로 통일해 제어회로 단순화.
8. 별도 학습/최적화 절차 없음(스케줄링 규칙 기반 설계).
9. 결과: TSMC 0.18um 합성 결과, conventional 대비 memory 461kB→445kB, throughput 27Mbps(Itr_MAX=10)→54Mbps(Itr_MAX=5)로 2배 향상, 평균 iteration 수 감소(Fig.5), BER은 conventional과 동등.
10. 한계: 정정능력 자체 개선은 없고 스케줄 효율화에 국한, 특정 PCM(802.11n 648b) 구조에 맞춘 설계라 다른 코드에 일반화하려면 재분석 필요, 실측(칩) 검증 없이 합성 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| irregular LDPC용 개선 message-passing schedule (중복 column 연산 제거) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` 이터레이션 루프 | Prime ECC는 QC-LDPC 고정 degree(HCU 비대칭 col 2개 제외) 구조로 irregular 전반의 스케줄 재설계 필요성은 낮으나 스케줄링 최적화 아이디어 참고 가능 |
| column weight별 BFU 배열 공유 (HW 자원 최소화) | `decoder.cpp` `VN_Cal_HD()` 등 VN 처리 유닛 | 부분적 관련 — Prime ECC는 이미 HCU로 비대칭 degree column을 별도 처리하는 유사 개념 존재, 중복 기여 |
| 평균 iteration 감소 (스케줄 개선으로) | `decoder.cpp` 이터레이션 루프, `ecc_data.h` `PARAM_DEC` | 개념적으로 관련되나 본 논문은 iteration 수치보다 스케줄 효율(throughput)에 초점 |
| IEEE 802.11n 특정 H-matrix 구조 | `ecc_top.cpp` `Load_PCM()` | 대응 없음 — Prime ECC는 자체 QC-LDPC H-matrix 고정 사용 |

적용 가치: 낮음 — irregular LDPC 특유의 불균일 degree로 인한 중복 연산 제거가 핵심인데, Prime ECC는 이미 HCU 등으로 비대칭 degree를 처리하는 구조를 갖추고 있어 중복 기여이며 정정성능 개선도 없음.
