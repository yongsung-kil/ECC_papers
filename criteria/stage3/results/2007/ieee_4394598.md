> 총평: IEEE 802.16e 기반 QC-LDPC(B-LDPC)의 rate-compatible 확장(extending+puncturing) 구성법으로, linear-time 인코딩과 저 iteration에서의 FER 개선을 보이나 HW 미설계이며 HARQ 특화 목적이라 NAND ECC 이식성은 중간 수준.

### B. 알고리즘 요약

1. IEEE 802.16e 기반 Block-type QC-LDPC(circulant permutation 기반, base matrix `Hb`)를 대상으로, 하나의 인코더/디코더 쌍으로 rate 1/3~4/5를 지원하는 rate-compatible(RC) 코드 구성법 제안(HARQ 응용).
2. 문제: puncturing만으로 낮은 rate(높은 puncturing 비율)를 만들면 punctured 노드에 0 LLR을 주어 디코딩 수렴이 느려지고, iteration 수가 적을 때(<20) 성능이 unpunctured 코드보다 나빠짐.
3. 채널모델은 BPSK/AWGN, belief propagation, noiseless feedback·무한 버퍼 가정.
4. 핵심 기법: rate-1/2 mother code에 row/column을 padding하는 "extending"으로 낮은 rate 코드를 생성 - puncturing과 달리 필요한 parity bit만 생성해 불필요한 연산을 피함.
5. 핵심 식: extended block 구성 시 check node degree 분포를 `ρ(x)=Σρi x^(i-1)`로 최소 종류로 제한(Criterion 1), 정보부-확장 parity부 간 의존성은 대각 배치 circulant로 부여(Criterion 2,3).
6. Case 1~3(check node degree 4/5/6)로 확장 블록을 구성, DE 문턱값(threshold) Case1=1.1501 > Case2=1.1425 > Case3=1.1269로 Case1이 점근 성능 최우수.
7. 부수 트릭: 확장부 우상단은 0, 우하단은 Hamming weight-1 column+duo-diagonal matrix로 linear-time 인코딩 가능하게 구성, 확장 parity는 이전 parity와 독립적으로 생성/복호 가능.
8. 최적화 절차: 5단계 construction procedure(대각 배치 → check degree 결정 → 정보-parity 의존성 부여 → 최대 VN degree 보강 → girth=6 shift값 결정)로 구성, 별도 DE/GA 학습 없음.
9. 결과: 528 information bits 기준 rate 1/3에서 Case1이 가장 낮은 복잡도(덧셈 20768회, f(x) 연산 10912회, puncturing-only 대비 낮음)이면서 iteration 20/30/50 모두에서 puncturing-only 대비 FER 우수, 특히 고rate·저iteration에서 성능 이득 큼.
10. 한계: 하드웨어 설계·구현 없이 시뮬레이션(BP, 무한 버퍼, ideal feedback)만 수행, HARQ/ARQ 재전송 시스템을 전제로 한 rate-compatible 목적이라 NAND처럼 고정 rate 위주인 응용과는 목적이 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| B-LDPC base matrix + circulant 구조 (QC-LDPC) | `ecc_data.h` `PCM_b`, `ecc_top.cpp` `Load_PCM()` | Prime ECC도 동일한 base+circulant(shift) QC-LDPC 구조 사용 - 구조적으로 가장 유사 |
| duo-diagonal parity 구조(linear-time encoding) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC의 dual-diagonal 인코딩과 개념적으로 매우 유사, 직접 비교 가능 |
| extending에 의한 rate-compatible H-matrix 확장 | `ecc_top.cpp` `Load_PCM()` | rate 가변 자체는 NAND ECC에는 통상 불필요(고정 rate)하나 H-matrix 확장 아이디어는 참고 가능 |
| check node degree 분포 설계(Criterion 1~3) | 대응 없음(부호설계, H-matrix 고정) | 부호설계 단계 요소로 디코더 코드에는 직접 대응 없음 |
| f(x)=log((e^x+1)/(e^x-1)) 기반 sum-product check update | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 min-sum 근사(min1/min2) 사용, 논문은 exact sum-product 연산 가정으로 직접 대응은 아님 |

> 적용 가치: 중간 | 이유: QC-LDPC base+circulant 구조와 duo-diagonal 인코딩이 Prime ECC와 유사해 구조적 유사성은 있으나, HARQ용 rate-compatible 목적 자체가 NAND ECC(고정 rate)와 어긋나고 HW 설계·성능 검증이 없음.
