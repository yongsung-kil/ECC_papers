### B. 알고리즘 요약

1. 대상은 이진 선형 코드의 Tanner graph 표현(정규화 incidence matrix `A_p`)이며 BEC(binary erasure channel) 및 AWGN 채널에서의 반복 복호 성능과 연관.
2. 풀려는 문제는 Tanner graph의 vertex expansion 성질이 좋을수록 반복복호 성능이 좋다는 것이 알려져 있으나, 특정 그래프의 expansion을 직접 계산하기 어렵다는 점.
3. 핵심 가정은 이진 선형 코드의 parity-check(혹은 redundant parity-check) 행렬을 bipartite Tanner graph로 표현하고, 정규화 incidence matrix `A_p = [h_ij / sqrt(r_i c_j)]`의 고유값 스펙트럼을 분석.
4. 핵심 기법은 `A_p^T A_p`의 두 번째로 큰 고유값 `λ1`을 이용해 임의의 부분집합 `S`의 expansion `β(S) = |N(S)|/vol(S)`의 하한을 유도(`Theorem 2`).
5. 핵심 식 `Theorem 3, 4`: minimum/average variable expansion의 하한을 각각 유도, 이 값이 좋을수록(1에 가까울수록) 반복복호가 잘 수렴함을 시사.
6. 확장 기법으로 stopping distance(`s(H_p)`)의 두 하한(bit-oriented, parity-oriented, `Theorem 5`)을 유도해 Tanner의 기존 minimum distance 하한이 실제로는 stopping distance의 하한임을 규명.
7. difference-set code(type-I 2-D projective geometry LDPC)에 대해 중복 parity-check 행을 추가한 redundant parity-check matrix `H_n(z)`를 구성, stopping distance가 minimum distance와 같아지도록 하는 상한 `stopping redundancy ≤ n`을 증명(`Theorem 8, 9`).
8. 학습/최적화 절차 없음 (고유값 계산은 표준 선형대수, difference set은 조합론적으로 고정 구성).
9. 결과는 `[21,11,6]` difference-set code에서 redundant parity-check 행 수를 10→21로 늘리면 BEC에서 block error가 0.001→0.00047, AWGN에서 0.5dB 성능 개선(ML 대비 0.25dB 이내).
10. 한계는 순수 이론/시뮬레이션(BEC, AWGN 소규모 코드)이며 HW 설계 없음, redundant parity-check 추가는 rate 손실(추가 행 = 추가 제약이나 encoding rate 자체는 원 코드 유지)과 메모리/라우팅 오버헤드를 수반하나 본문에서 정량화하지 않음.

| Stopping distance/redundancy 개선을 위한 redundant parity-check 추가 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 확장 아이디어로 참고 가능하나 고정 QC-LDPC 구조 재검증 부담 큼(높은 이식 난이도) |
| BEC 기반 stopping set 분석 | 대응 없음 | Prime ECC는 AWGN/RBER 채널(`channel.cpp`) 대상, BEC 특화 분석은 직접 대응 없음 |
