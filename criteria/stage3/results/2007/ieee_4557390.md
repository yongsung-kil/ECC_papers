### B. 알고리즘 요약

1. QC LDPC 블록코드 `C^(r)_QC` (circulant size `r`, period `L`)를 다항식 parity-check 행렬 `H_QC(X)`로 표현.
2. 기존 QC 블록코드는 짧은 SNR 영역에서 waterfall 성능이 pseudo-codeword spectrum과 cycle 구조에 제약받는 문제가 있음.
3. `H_QC(X)`를 행/열 재정렬 후 대각선으로 자르고 이어붙이는 "unwrapping"으로 time-invariant convolutional code `H_conv(D)` 생성 (기존 [1],[7] 방법).
4. 본 논문의 핵심 기법: 행/열 재정렬 없이 직접 `J x L` 단위로 unwrapping하여 **time-varying** convolutional code 생성, syndrome former memory `m_s ≤ r-1`, 주기 `r`.
5. 핵심 결과식: `w_p,min(H_QC^(r)) ≤ w_p,min(H_conv)` — unwrapped code의 최소 pseudo-weight가 원본 QC 코드 이상임을 증명, 즉 pseudo-weight spectrum이 "얇아짐".
6. 추가로 unwrapped convolutional code의 임의의 cycle은 원본 QC 코드에서 같거나 더 짧은 길이의 cycle로 매핑됨을 증명(short cycle 감소).
7. Circulant size `r`을 다르게 선택하면 서로 다른 `m_s`, period를 갖는 time-varying code 계열을 얻을 수 있음(time-invariant는 `r`에 무관하게 동일 코드).
8. 별도 학습/최적화 절차 없음 (해석적 구성 + 시뮬레이션 검증).
9. [155,64] (3,5)-regular QC LDPC 예제에서 time-varying convolutional code가 BER 10^-6 기준 QC 블록코드 대비 1.9~2.8dB 이득, time-invariant 대비도 큰 개선.
10. 한계: sliding-window message-passing 디코더를 가정한 시뮬레이션뿐이며 HW 구현/복잡도 분석 없음, block code가 아닌 convolutional code 클래스로 구조 자체가 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC block → convolutional code unwrapping (부호구조 변환) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 block QC-LDPC이며 convolutional 구조 미지원, 이식 시 근본 재설계 필요 |
| sliding-window message-passing 디코더 | 대응 없음 | Prime ECC는 block 단위 iterative decoder(`decoder.cpp`)만 지원, convolutional sliding-window 구조 없음 |
| pseudo-weight/cycle 기반 부호 특성 분석 | 대응 없음 | 코드 구성 시점의 이론적 분석 도구이며 런타임 모듈과 무관 |

> 적용 가치: 낮음 — LDPC convolutional code 클래스 자체가 Prime ECC의 block QC-LDPC 구조와 호환되지 않아 이식 불가.
