### B. 알고리즘 요약

1. 대상은 BSC/BI-AWGN 채널 위의 asymptotically good expander code(정규 left-degree LDPC의 특수 사례)이며, exact ML decoding의 worst-case NP-hardness는 이미 알려져 있음.
2. 문제의식은 worst-case 복잡도 결과가 지나치게 비관적이므로, 특정 코드 클래스에서 "기대(expected) 복잡도" 관점의 다항시간 ML 디코딩이 가능한지를 규명하는 것.
3. ML 인증(ML certificate) 디코더 개념을 사용: 서브옵티멀 디코더가 실패를 스스로 감지하면 전수탐색(exhaustive search)으로 보완하는 방식으로, 기대복잡도 `E_NML(n) = N(n) + k(n)*Pe(n)*2^{nR}`.
4. 전처리 허용 시, Tanner graph가 `(αn, 3c/4)`-expander이면 minimum distance 사전계산 기반의 단순 순차 플립(simple sequential decoding) + Hamming distance 비교로 ML 인증 가능(Theorem 1, 2).
5. `Pe(n) ≤ 2^{-D(α/2‖p)n}`(Chernoff bound, KL divergence 기반)로 지수적 감소 시, 전체 기대복잡도가 다항시간이 됨.
6. 전처리 없이도 동작하는 신규 ML 인증 알고리즘 제안: 단순 플립 후 결과가 코드워드가 아니면 source-sink max-flow(Ford-Fulkerson) 인스턴스를 구성해 ML 여부를 검증, worst-case `O(n^2)`(기존 LP decoder의 `O(n^9)` 대비 개선).
7. max-flow 구성은 Feldman et al.의 LP 쌍대 실행가능점(dual feasible point) 증명에서 쓰인 균일 capacity 대신, 변수노드별로 최적화된 capacity(`A = ⌊c/2⌋+1`)를 사용하는 점이 차별점.
8. 별도의 학습/최적화 절차 없음(순수 그래프/조합론적 증명).
9. 결과: p가 0에 가까울수록 달성 가능한 rate region이 채널 용량(capacity)에 근접함(Lemma 9); ML decoding이 반복복호 대비 error exponent를 임의로 크게 개선 가능함(Lemma 5).
10. 한계: 실제 알고리즘 구현/시뮬레이션/HW 설계 전무, regular left-degree LDPC로 구성된 코드 클래스는 여전히 worst-case NP-hard임을 별도 증명(Section IV), 실용성보다 이론적 통찰 제공에 그침.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ML 인증 알고리즘 / max-flow 기반 exact ML decoding | 대응 없음 | Prime ECC는 min-sum 기반 근사(suboptimal) 디코더이며 exact ML/max-flow 구조 없음 |
| expander code 구성(전처리, minimum distance 계산) | 대응 없음 | QC-LDPC 고정 구조(`ecc_top.cpp` `Load_PCM()`)와 무관한 이론적 코드 클래스 |
| error floor 관련 언급 | `decoder.cpp` `LDPC_Decoder()` | error floor 현상 해소 가능성만 언급, 구체 알고리즘/수치 없어 관련성 낮음 |

> 적용 가치: 낮음 — 실용 알고리즘·HW·수치결과 없는 순수 복잡도 이론 논문으로 Prime ECC 이식 요소 전무.
