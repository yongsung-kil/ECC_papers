### B. 알고리즘 요약

1. 대상은 무선 통신(IEEE 802.11/802.16, DVB-S2) 채널코딩용 regular LDPC, AWGN 채널 가정.
2. 표준 sum-product(log-domain LLR) 복호 알고리즘을 그대로 사용, 새로운 복호 알고리즘 제안은 없음.
3. 핵심 문제는 가변 parity-check matrix/다중 code rate 지원 시 variable-check node 간 interconnect(배선) 면적이 커지는 것.
4. Crossbar switch(`N²` 복잡도) 대신 Benes network(`N/2 × (2log2N - 1)` 복잡도)를 스위치 블록으로 채택해 배선 복잡도를 낮춤.
5. Broadcasting 구조로 check/variable node 간 고정 배선(`2nγ`)을 `2n`으로 축소.
6. Partially parallel architecture(demux/mux로 노드 time-share)를 결합해 실제 배치되는 variable/check node 수 자체를 줄임.
7. 지원 code rate는 1/3, 2/3, 4/5, 5/8 등 다중 rate이며 입력 message select 신호로 전환.
8. 별도의 학습/최적화(DE, GA 등) 절차는 없음.
9. 결과: 0.18um 6-metal CMOS ASIC 구현, 656Mbps@200MHz, 1199K gate, 15.81mm² (비교 대상 S.H Kang 543K/10.08mm²/985Mbps 대비 면적·게이트 모두 큼).
10. 한계: 정정능력/error-floor 개선 없음, 제안 구조가 비교군보다 오히려 게이트 수·면적 크고 throughput 낮음, ASIC 합성 결과만 있고 BER 곡선 등 성능 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-Product 복호(표준 tanh 기반) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 이미 Min-Sum 근사 사용, 표준 Sum-Product는 중복·퇴보 기법 |
| Benes network 스위치 블록 | 대응 없음 | Prime ECC는 고정 QC-LDPC 구조(z=32 lifting)로 가변 PCM 스위칭 아키텍처 없음 |
| Broadcasting 배선 구조 | 대응 없음 | 표에 명시된 HW 모듈(GT, HCU)과 다른 배선 절감 기법으로 대응 없음 |
| Partially parallel architecture | 디코더 HW 모델(z=32, HCU, GT) | 개념적으로 유사하나 가변 PCM 지원 목적이라 이식 난이도 중~높음 |

> 적용 가치: 낮음 — 무선 다중 code rate 지원용 배선 절감 아키텍처로, NAND 고정 QC-LDPC 구조에는 실익 적고 복호 알고리즘 자체는 표준 Sum-Product라 Prime ECC 대비 퇴보.
