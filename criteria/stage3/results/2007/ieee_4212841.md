> 총평: 이미 보유한 조기종료(parity check) 기법을 전제로, 디코더 입력에 작은 버퍼를 추가해 packet간 iteration 편차를 평균화함으로써 면적 대비 throughput을 높이는 아키텍처 기법으로, NAND ECC에도 개념적으로 이식 가능하나 우선순위는 중간.

### B. 알고리즘 요약

1. 대상은 DVB-S2 위성방송 표준의 이너 LDPC(IRA, block size Q=360, N=64800) + 아우터 BCH 직렬연접 코드, partially-parallel SIMD 디코더.
2. 문제: 표준 HW 설계는 최대 iteration(worst-case, N_it,max)을 기준으로 자원을 고정하므로, 실제 packet마다 조기수렴(early convergence)해도 자원이 낭비됨.
3. 모델: 각 packet의 디코딩 시간 T_s = N_it·T_it을 i.i.d. 랜덤변수로 가정, 디코더를 D/G/1/1+L 큐(결정적 도착, 임의 서비스시간, 버퍼 L)로 모델링.
4. 핵심 기법: parity check `p_m(q) = ⊕_{n∈N_m} sign{λ_{m,n}}` 로 매 iteration 유효 코드워드 여부 검출하는 조기종료 규칙을 전제로, 입력단에 길이 L의 소버퍼를 추가해 packet 간 iteration 편차를 statistical multiplexing으로 흡수.
5. 핵심 식: overflow 확률 `P_L ≈ C·e^{-θ̂BΓ^-1}` (large deviation, θ̂는 `log E[e^{θ(Ts(n)-Γ^-1)}]=0`의 해) — 버퍼 크기 B와 목표 처리율 Γ 설계에 사용.
6. 확장: 반대로 목표 throughput을 고정하고 병렬 처리유닛 수를 K(<1)배로 줄여 칩 면적을 절감하는 scaled-down 설계도 동일 프레임워크로 제시.
7. 구현 디테일: 복잡도 S(L) = S_LDPC + L·S_IO (버퍼 1개당 면적 S_IO ≪ S_LDPC)로, 추가 HW 비용이 매우 작음.
8. 최적화 절차: 없음(Monte-Carlo 시뮬레이션 + large deviation 이론 검증만).
9. 결과: L=2 버퍼만으로 throughput 2배(γ(2)>2), 동일 throughput 기준 칩면적 41.2%(18.75mm²) 절감; 0.13um CMOS 기준 255Mbps 베이스라인.
10. 한계: 실제 회로 합성이 아닌 [8] 문헌치 재사용, ARQ 등 지연-민감 시스템엔 버퍼로 인한 추가 latency가 부적합할 수 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Parity-check 기반 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료를 이미 보유(파리티체크는 아니지만 유사 목적), 신규 기여 아님 |
| 입력 버퍼 + statistical multiplexing 아키텍처 | 대응 없음 | Prime ECC 3.1 프로파일에 packet 큐잉/버퍼 기반 throughput 최적화 언급 없음, 신규 구조 |
| 병렬 유닛 수 K배 축소 + 버퍼 보상 (scaled-down) | 디코더 HW 모델 (z=32, HCU, GT) | 병렬화/면적 트레이드오프 개념은 관련되나 Prime ECC의 z=32 고정 병렬구조와 직접 대응 안 됨 |

> 적용 가치: 중간 — 조기종료 자체는 이미 보유했지만, "packet간 iteration 편차를 입력버퍼로 statistical multiplexing"하는 아키텍처적 아이디어는 NAND read request 큐잉 구조에 맞게 재해석 시 면적 대비 throughput 개선에 참고할 만함.
