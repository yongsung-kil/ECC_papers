### B. 알고리즘 요약

1. LDPC Tanner graph에서 각 반복마다 variable node(VN)/check node(CN)가 LLR 메시지 `u`(VN→), `v`(→CN), `U`(CN→), `V`(→VN)를 주고받는 belief propagation 구조.
2. 문제: check node에서 degree `dc`개의 고유 extrinsic 메시지 `U_i`를 계산할 때 `Λ_lnBP(v)` 함수의 동적범위가 [0,∞]로 넓어 근사가 어려움.
3. Jacobian 로그 항등식을 두 번 적용해 `Λ_BP*(v1,v2) = sgn(v1)sgn(v2)Min*(v1,v2)` 형태의 **Min* 재귀**를 유도(근사 아님, 정확식).
4. 핵심 기법 1단: `Min*(v1,v2) = min(|v1|,|v2|) + ln(1+e^-(|v1|+|v2|)) - ln(1+e^-||v1|-|v2||)` — 함수값 범위가 [0,ln2]로 좁아 근사가 쉬움(single/double-line 근사 가능, Fig.1).
5. 역연산자가 없는 비선형 연산에서 `dc`개 출력을 `dc(dc-2)`가 아닌 **3(dc-2) 연산**으로 계산하는 최적 알고리즘 A*(dc)(오름차순/내림차순 누적 후 결합) 제시, 이 하한이 최적임을 증명.
6. 핵심 기법 2단(축소형): BP-Based(최소/차소값만 이용, 최대 1dB 손실, `Offset-BP-Based`로 0.1~0.2dB로 개선)와 Approximate-Min*-BP(가장 약한 edge만 정확 계산, 0.05dB 미만 손실, SNR 추정 필요) 두 저복잡도 근사.
7. 부수 구현: CNP는 sign-magnitude, VNP는 2's-complement 표현 사용, 두 표현 간 변환블록 필요, LUT 기반 `Λ_lnBP` 구현.
8. HW 아키텍처: (a) protograph 기반 structured 디코더(edge당 memory+cyclic shift, switch matrix 불필요), (b) Type-I universal 아키텍처(H를 N×M 서브행렬로 분할, row/column permutation으로 memory load balancing, 불규칙 코드는 degree별 정렬로 idle time 최소화), (c) Type-II universal 아키텍처(block column당 edge 수를 균등화해 idle time 원천 차단, `4L`개 RAM과 `L^2` 크로스바로 column↔row interleaving).
9. 결과: Approximate-Min*-BP는 Full-BP 대비 0.05dB 미만 손실(JPL FPGA 실측 확인), Type-II는 L=16까지 병렬화 달성, throughput ∝ `N(FL/2EI)`.
10. 한계: 실제 gate count/throughput 수치는 본문에 명시 안 됨(예제 코드도 소규모), Approximate-Min*-BP는 SNR 추정 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min*/Approximate-Min*-BP (check node extrinsic 계산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min-sum(0th-order 근사) 사용 중 — Min*/Approximate-Min*-BP는 그보다 정확한 대안으로 이식 시 min-sum 대체 검토 가능 |
| 3(dc-2) 최적 extrinsic 계산 알고리즘 | `decoder.cpp` `C2V_Cal()` | CN 연산 스케줄링 최적화와 직결, 낮은 이식 난이도 |
| Type-I/II universal decoder memory partitioning | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 QC-LDPC 고정 lifting(z_sb=32) 구조라 universal 아키텍처 자체보다는 memory balancing 아이디어만 참고 가능 |
| protograph 기반 structured 디코더(cyclic shift, no switch matrix) | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 QC-LDPC(circulant shift) 구조라 개념적으로 이미 유사 구현 보유 |
| sign-magnitude/2's-complement 변환 | `decoder.cpp` `VN_Cal_HD()` 등 | 이미 유사 표현 방식 사용 중, 대응 확인 필요 |

> 적용 가치: 높음 — check node 연산 최적화(Min*/Approximate-Min*-BP)가 Prime ECC의 min-sum 디코더 핵심 루프에 직접 대체 가능한 대안이며, 3(dc-2) 최적 스케줄링도 CN 연산 저복잡도화에 참고 가치가 있음.
