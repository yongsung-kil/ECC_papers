### B. 알고리즘 요약

1. 시스템: rate-1/2 (128,3,6) LDPC convolutional code(LDPC-CC), 180nm 1.8V CMOS ASIC, AWGN 채널(BPSK).
2. 문제: LDPC block code(LDPC-BC)는 스트리밍/가변길이 패킷 데이터에 부적합, LDPC-CC는 memory `m_s`가 block size보다 훨씬 작아 유리하지만 첫 ASIC 구현 사례가 없었음.
3. 부호 구성: 코드비트 `c_i`는 최근 `m_s+1`개 정보비트와 이전 코드비트로만 생성되는 시변 주기적 컨볼루션 구조, `H^T`는 lower-triangular로 인코딩 지연 0.
4. 복호: soft-decision message-passing algorithm(MPA), LLR을 프로세서 체인을 따라 단방향 전파. parity-check node는 min-sum 근사(`tanh` 대신), variable node는 덧셈.
5. 핵심 식: `q_2ˆ(i) = sgn(q1)·sgn(q2)·min(|q1|,|q2|)` 형태의 min-sum parity-check 연산(식 8-9), variable node는 입력 LLR들의 합(식 10).
6. HW 구조: 10개 동일 프로세서를 캐스케이드(파이프라인) 연결, 각 프로세서 = 1 parity-check node + 2 variable node, 프로세서 수가 LDPC-BC의 iteration 수에 대응.
7. 구현 디테일: LLR을 6비트(부호1 + 포화1 + 크기4)로 양자화, FIFO 딜레이라인으로 LLR 저장(8줄×140레지스터=6720 레지스터), on-chip LFSR 기반 근사 AWGN 채널 에뮬레이터(BIST용).
8. 최적화: 소프트웨어 시뮬레이션으로 LLR 크기 비트수(4비트가 SNR 3dB에서 적정) 및 필요 프로세서 수(30~50개가 이상적, 실리콘 제약으로 10개만 구현) 결정.
9. 결과: 175Mb/s 처리량, 1.3W 전력(1.8V), 3dB SNR에서 BER 0.0018, 기존 LDPC-BC 대비 net length 97% 감소(평균 3mm→0.1mm), 에너지 7.6nJ/decoded-bit(전압 스케일링 시 4.1nJ).
10. 한계: 파워 비효율적(register-intensive 설계, 시간 중심 합성으로 전력 미최적화), 프로세서 10개로 제한되어 완전한 BER 성능 미검증, LDPC-BC 대비 완전병렬화가 어려워 처리량 확장에 한계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum parity-check 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 근사를 이미 사용 중이라 개념적으로 중복, 다만 LDPC-CC의 시변 phase-matrix 구조는 QC circulant와 상이 |
| LLR 양자화(6bit, 4bit magnitude) | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | soft-4bit+ 양자화 설계는 참고 가능하나 이미 유사 파라미터 보유 |
| 캐스케이드 프로세서 파이프라인 구조 | 대응 없음 | Prime ECC는 QC-LDPC의 z=32 row-parallel + HCU 구조(`decoder.cpp` pipeline 6-stage)이며 LDPC-CC의 단방향 FIFO 캐스케이드와 아키텍처가 근본적으로 다름 |
| 부호구조(시변 phase matrix, lower-triangular H) | 대응 없음 | Prime ECC는 QC-LDPC base+circulant 구조(`ecc_top.cpp` `Load_PCM()`)로 컨볼루션 부호와 전혀 다른 부호족 |

> 적용 가치: 낮음 — min-sum/LLR 양자화는 이미 Prime ECC가 보유한 기법과 중복이고, LDPC-CC 고유의 컨볼루션 구조·파이프라인은 QC-LDPC block code 구조와 근본적으로 달라 이식 난이도가 높음.
