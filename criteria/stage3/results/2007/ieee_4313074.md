### B. 알고리즘 요약

1. 대상 코드는 IEEE 802.3an 10GBASE-T에 채택된 (2048,1723) Reed-Solomon 기반 LDPC(RS-LDPC), column degree 6, row degree 32, 64×64 순열 서브행렬 구조, girth≥6.
2. 문제: 유한 길이 LDPC의 low-FER 영역은 error-floor로 인해 Monte Carlo 시뮬레이션으로는 10⁻⁸ 이하 FER 추정에 10¹⁰ 이상의 trial이 필요해 비현실적.
3. 모델: AWGN + BPSK 채널, all-zero 코드워드 가정(대칭성), iterative message-passing 디코더가 non-codeword인 "absorbing set"에 흡수되어 실패한다고 정의.
4. absorbing set: 부분집합 `D⊂V`(크기 a)에서 unsatisfied check 이웃 집합 `O(D)`(크기 b)를 갖고 각 비트가 unsatisfied보다 satisfied check을 더 많이 갖는 구조; `(2048,1723)` 코드는 `(8,8)` absorbing set이 지배적(최소거리보다 작은 weight).
5. Mean-shift importance sampling: 원래 가우시안 밀도 `f`의 평균을 (8,8) absorbing set에 속한 8비트에서만 `μ≈1.2`만큼 이동한 `f_bias`로 샘플링, `w(x)=f(x)/f_bias(x)`로 재가중.
6. `μ≈1.2`(ML 경계 `μ=1`보다 약간 큼)를 경험적으로 선택하면 언더/오버바이어스 없이 디코더가 정답과 absorbing set에 비슷한 빈도로 도달, 이 값을 벗어나면(0.8 이하, 1.6 이상) 추정 정확도가 급격히 나빠짐.
7. 전체 오류확률은 `P[error]≈M·P(8,8;σ²)`로 근사(`M`=(8,8) absorbing set 총 개수, 조합적 제약으로 정확히 열거하여 `M=11,168`).
8. 학습/최적화 절차: 없음(사전 조합적 열거 + 단일 mean-shift 파라미터 튜닝뿐, DE/GA/NN 아님).
9. 결과: 6bit/9bit 양자화 디코더 각각에서 IS 예측과 FPGA 하드웨어 에뮬레이션 결과가 FER 10⁻¹²까지 슬로프 일치, naive MC 대비 분산 감소(simulation gain) 최대 3.12×10⁹배(9bit, SNR 5.6dB).
10. 한계: 정정능력을 개선하는 기법이 아니라 저FER 영역의 성능을 빠르게 "추정"하는 시뮬레이션 방법론이며, 6bit vs 9bit 양자화 비교는 클리핑 효과에 대한 정성적 설명에 그침(HW 설계 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| absorbing set 기반 error-floor 원인 분석 | 대응 없음 | Prime ECC 코드는 error-floor 원인(trapping/absorbing set) 분석 모듈이 없음, `decoder.cpp` 이터레이션 루프의 수렴 실패는 `corner.cpp`에서 추적만 함 |
| mean-shift importance sampling 저FER 추정 | `corner.cpp` | corner.cpp가 수렴실패 추적 용도이므로, 유사한 목적(저FER 영역 관측)의 시뮬레이션 가속 아이디어로 참고 가능하나 실제 IS 구현은 없음 |
| soft LLR 양자화(6bit/9bit) 클리핑 영향 | `channel.cpp` `Set_LLR_Th()` | LLR 양자화/threshold 설정과 직접 관련되나, Prime ECC는 hard-1bit/2SD/3SD만 지원해 9bit급 fine quantization은 지원 범위 밖 |

> 적용 가치: 중간 — 정정능력 개선 기법은 아니지만 error-floor 영역의 시뮬레이션 가속(importance sampling) 아이디어는 Prime ECC의 error-floor 검증 속도 개선에 참고할 수 있음.
