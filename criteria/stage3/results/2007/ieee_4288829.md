### B. 알고리즘 요약

1. 대상 부호: rate-1/2, length 1944 QC-LDPC(802.11n 서브구조, 54x54 submatrix) 및 802.11n rate-5/6 부호, AWGN 채널에서 floating-point 시뮬레이션.
2. 문제: 기존 flooding/SSS(sequential) 스케줄은 메시지 갱신 "순서"가 고정되어 trapping set에 취약하고 수렴이 느림.
3. 가정: 메시지의 residual(갱신 전후 값 차이, 식 3 `r(mk)=fk(m)-mk`)이 클수록 해당 그래프 영역이 수렴에서 멀다는 가정.
4. 핵심 기법 1단: RBP(Residual BP)는 residual이 가장 큰 메시지를 우선순위 큐로 관리해 먼저 갱신 — 초반 수렴은 빠르지만 "difficult error"(trapping set)에서 SSS보다 성능 열화.
5. 핵심 식: `r(mk)=fk(m)-mk` (식 3)로 각 메시지의 미수렴 정도를 정량화, 이를 큐 정렬 기준으로 사용.
6. 확장(핵심 기법 2단): node-wise RBP — 하나의 check node에서 나가는 모든 메시지를 동시에 갱신/전파(Algorithm 2)해 greedy함을 완화, trapping set의 degree-1/degree-2 구조를 순차적으로 해소.
7. 부수 트릭: 정확한 residual 대신 min-BP 근사(check node의 최소/차소 reliability만 이용, 식 4)로 계산량 감소한 ARBP/node-wise ARBP 제안 - 성능은 exact와 거의 동일.
8. 병렬화 확장: parallel node-wise ARBP - residual이 큰 p개 check node를 동시 갱신(p=54, submatrix 크기와 동일), 성능 저하는 미미.
9. 결과: node-wise ARBP는 18 iteration에서 SSS(LBP) 50 iteration과 동일 FER 달성; rate-5/6 고rate 부호에서도 SSS 대비 빠르고 낮은 FER; RBP는 4 iteration에서 SSS 13 iteration과 동일하나 19 iteration 이후 역전(교차).
10. 한계: 부동소수점 시뮬레이션뿐, HW 구현·게이트카운트·throughput 수치 없음; residual 계산·큐 재정렬로 인한 반복당 복잡도 증가는 구체적 구현별 trade-off로 남겨둠(future work); parallel 버전의 메모리 clash 등 HW 이슈는 명시적으로 미해결.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| node-wise (A)RBP 동적 스케줄링 (잔차 기반 check-node 갱신 순서) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 반복 스케줄링 로직 변경으로 min-sum 루프에 순서 결정 로직만 추가하면 이식 가능 |
| min-BP 근사 잔차 계산(식 4, 최소/차소 reliability 이용) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산과 동일한 min1/min2 구조를 residual 근사에 재사용 가능 |
| Dual-Update(even/odd 교대) 스케줄과의 비교 대상 | `decoder.cpp` Dual-Update 옵션 | 기존 보유 스케줄링(§3)과 성격이 유사해 중복 기여 가능성 검토 필요 |
| parallel node-wise ARBP (p개 check node 동시 갱신) | 디코더 HW 모델 (z=32, HCU, GT) | 병렬 처리 단위(p=54)가 우리 lifting size(z=32)와 다르며 메모리 clash 등 정합 검토 필요 |

> 적용 가치: 중간 — 메시지 생성 함수 변경 없이 갱신 순서만 잔차 기반으로 동적 결정하는 아이디어는 이식 난이도가 낮지만, Prime ECC는 이미 Dual-Update 등 자체 스케줄링을 보유하고 있어 중복 기여 여지가 있고 HW 검증(게이트/throughput)이 전무해 추가 검증 부담이 있음.
