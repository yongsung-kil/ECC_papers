### B. 알고리즘 요약

1. 대상은 short/medium length irregular LDPC 부호(예제: n=1000, rate 1/2, `λ(x)=0.2449x+0.20298x²+0.00055x³+0.17234x⁴+0.37923x¹⁴`, `ρ(x)=x⁷`), AWGN 채널에 표준 BP 디코딩 가정.
2. 짧은 길이 LDPC는 asymptotic density evolution 설계만으로는 성능이 나쁘고, 유한길이에서 문제되는 부분그래프(stopping set)를 제거하도록 그래프를 직접 설계해야 함.
3. 기존 ACE(Approximated Cycle EMD) constrained design[3]은 사이클 길이 `l`과 ACE값 `η`이 특정 임계 이하인 (l,η)-cycle을 제거하는 방식이나, 모든 성분에 동일한 `η_ACE`를 강제하는 "flat spectrum" 방식이라 유연성이 부족.
4. 핵심 기법: 각 사이클의 `ACE(C) = Σ_{v∈VC}(d(v)-2)` (변수노드 차수 합에서 유도)를 이용해, 코드 그래프 전체의 사이클 길이별 최소 ACE값 분포인 "ACE spectrum" `η(G(H)) = (η2, η4, ..., η2dmax)`을 정의.
5. ACE spectrum이 낮은(0에 가까운) 성분은 작은 stopping set 존재를 의미하며 error-floor 열화와 직결; `expanded ACE spectrum`(히스토그램 형태)으로 그래프 구조를 세밀히 진단.
6. 확장 기법(GACE): 기존 flat-spectrum ACE 설계를 일반화해 각 사이클 길이별 목표 `η2i`를 독립적으로 지정 가능하게 하고, 기존 Viterbi-like ACE 검출 알고리즘([3])을 그대로 재사용해 순차적으로 각 spectrum 성분을 최대화.
7. 부수 트릭: PEG(Progressive Edge Growth) 알고리즘과 결합한 변형(참조 [13],[14])도 언급되며, degree-2 변수노드가 짧은 사이클을 형성하는 문제를 별도로 지적.
8. 최적화 절차는 수치 최적화 도구가 아닌 순차적 그리디 탐색(각 성분을 설계시간이 과도해지기 전까지 최대로 밀어올림)으로 진행.
9. 결과: 예제(n=1000, R=1/2)에서 GACE spectrum을 (∞,∞,13,9,4)까지 개선하자 BER 10⁻⁸ 부근 error-floor가 기존 (5,3) ACE 코드 대비 크게 낮아짐(그래프 근거, 정확 수치는 도표에만 존재).
10. 한계: 시뮬레이션(AWGN, BP)만 수행, HW 설계·복잡도·throughput 미제시; girth/stopping-set 최소화 접근으로 low-weight trapping set 문제는 완전히 해결하지 못함(별도 대책 필요함을 저자도 인정).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ACE spectrum 기반 H-matrix(그래프) 설계/진단 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구성·검증 단계에 참고 가능하나 QC-LDPC circulant 구조에 맞춘 재설계 필요 |
| stopping set/error-floor 개선 목적 | 대응 없음 | 디코더가 아닌 부호설계(H-matrix 생성) 단계 기법이라 decoder.cpp에는 직접 대응 없음 |
| BP 디코딩 시뮬 검증 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이나 본 논문은 sum-product 계열 BP로 검증, 알고리즘 자체 이식 아님 |

> 적용 가치: 중간 — 디코더 알고리즘이 아닌 H-matrix 설계 단계의 error-floor 개선 기법으로, Prime ECC의 고정 QC-LDPC 구조에 직접 이식은 어렵지만 향후 부호 재설계 시 그래프 진단 도구로 참고 가치 있음.
