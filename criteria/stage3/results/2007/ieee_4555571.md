### B. 알고리즘 요약

1. QC-LDPC 코드는 c×c circulant 배열로 구성된 parity check matrix `H_QC`로 정의되며, RC(row-column) constraint를 만족하면 girth ≥ 6을 가짐.
2. 기존 방식은 고정 rate LDPC만 다루므로, 채널 상황에 따라 다양한 rate를 지원하려면 매번 새로운 코드/인코더가 필요해 비효율적임.
3. 최고 rate 모부호 `(N0, M0)`에서 시작해, 매 레벨마다 `Ml`개의 행/열을 padding하여 `L`단계에 걸쳐 낮은 rate 코드를 유도하는 확장(extending) 구조를 가정.
4. 확장된 parity check matrix는 상위-우측이 항상 0이 되도록 설계되어, 상위 rate 코드워드가 하위 rate 코드워드에 내포(embed)됨.
5. Systematic-circulant(SC) 형태의 생성행렬 `G_QC = [I, G(0)]`을 이용, `pj = s1*G1,j + s2*G2,j + ... + s(n-m)*G(n-m),j` (식 1)로 parity를 계산하며, 이는 shift-register-adder-accumulator(SRAA) 회로로 구현 가능.
6. 확장 시 각 레벨의 `G(l)`도 SC 형태를 유지하여, 추가된 parity bit들만 이전 인코딩 결과 위에 독립적으로 누적 계산 가능(직교적 확장).
7. 세 가지 인코더 구현: (a) 직렬 입력 - `(n-m)c` 사이클, (b) 병렬 입력-병렬 출력 - `mc` 사이클, (c) 병렬 입력-직렬 출력 - `c` 사이클(단, 하드웨어 최다 소요).
8. 학습/최적화 절차 없음(구성적 방법).
9. 결과는 정성적 게이트 수식(플립플롭/AND/XOR 게이트 개수를 rate·padding 크기의 함수로 표현)만 제시, 실측 gatecount/throughput 수치 없음.
10. 시뮬레이션이나 HW 합성 없이 이론적 회로 설계 및 게이트 수 계산식만 제시하는 수준에 그침(irregular QC-LDPC 확장은 언급만, 정량 결과 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 확장(rate-compatible) parity check matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 rate H-matrix 사용, 가변 rate 확장 구조는 미보유 |
| SC(systematic-circulant) 형태 생성행렬 기반 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 인코딩 고정 방식으로 구조가 다름 |
| SRAA 기반 shift-register 인코더 회로 | `encoder.cpp` | 인코더 HW 구조 변경에 해당, 이식 난이도 중~높음 |
| 직렬/병렬 인코더 전환(전력 절감 목적) | 대응 없음 | Prime ECC 인코더는 고정 구조로 rate별 전환 개념 없음 |
