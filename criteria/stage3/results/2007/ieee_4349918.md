### B. 알고리즘 요약

1. 대상은 IEEE 802.16e QC-LDPC(rate 1/2, N=1152/2304, base matrix `Hb`(Mb×Nb) + z×z circulant shift), 예: (2304,1152) 코드는 z=96, Mb=12, Nb=24.
2. 기존 SPA 기반 message-passing decoding(MPD)은 수렴에 많은 iteration이 필요해 throughput이 낮은 문제를 해결하려 함.
3. 채널은 BPSK+AWGN, 디코딩은 표준 sum-product(식 `Rij[k]=-Sij[k]Ψ(ΣΨ(|Qj'i[k]|))`) 기반.
4. 핵심 기법: H의 행을 z개 그룹(`Hl(i)`, i=0..z-1)으로 나누고 각 그룹의 non-zero column만 남긴 `Nl`×`Mb` 크기 서브코드 `Cl(i)`를 정의(모든 `Hl(i)`가 동일한 `Hl`을 가짐, `Nl≪N`). 서브코드들의 code bit index 집합 `Sl(i)`는 서로 겹치되 disjoint는 아님.
5. 핵심 식 의미: 겹치는 index 집합 덕분에 `Cl(0),Cl(1),...,Cl(z-1)`을 순차로 디코딩하며 서로 extrinsic 정보를 즉시 주고받아, 하나의 global iteration 내에서도 정보가 빠르게 갱신되어 수렴이 가속(shuffle-BP와 유사한 효과).
6. 확장: 서브코드당 반복수 `Nl`과 global iteration 수 `Ng`를 조합(`Nl×Ng=k` 고정)했을 때 `Nl=1`이 최적으로 나타남.
7. 구현 디테일: `Hl` 디코더는 `Nl=63`(2304,1152 코드 기준)로 짧아 fully-parallel 구현이 가능해 복잡한 배선 문제를 피함, 4-bit non-uniform quantized SPA 사용(5-bit min-sum과 성능 유사하나 메모리 절감).
8. 별도 학습/최적화 절차 없음(고정 QC 구조 기반 결정론적 서브코드 분해).
9. 결과: 제안 디코딩(Nl=1, Ng=30)이 기존 MPD(Ni=50)와 동등 BER 달성(iteration 최대 40% 절감); Table I에서 (2304,1152) 코드 기준 gate 157K, 메모리 30300bit, 총 clock cycle 2880으로 TDMP([15], gate 미상/메모리 60228bit/2880cycle 대비 유사 clock이나 메모리 절반) 및 overlapped decoder([14], gate 457K/1900cycle) 대비 우수.
10. 한계: 공정 노드/클록 주파수·실제 throughput(Gbps) 수치 미제시, 결과는 rate-1/2 802.16e 코드로만 검증되어 타 rate/부호구조 일반화는 추가 검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC를 z개 겹치는 서브블록으로 분해해 순차 디코딩(수렴 가속) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 Dual-Update(홀/짝 교대 스케줄)와 유사한 계열의 조기정보 갱신 기법으로 이식 시 스케줄링 로직 확장 가능 |
| MUvn(variable-to-check 메시지) 메모리 제거 | `decoder.h` `Get_VNU_*`, `ecc_data.h` PARAM_DEC | Prime ECC 메시지 저장 구조와 비교해 메모리 절감 아이디어 참고 가능, 실제 구현 시 col-block 파이프라인(z_sb=32)과의 정합 재검토 필요 |
| 4-bit non-uniform quantized SPA (메모리 절감) | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | LLR 양자화 테이블 설계에 참고 가능 |
| fully-parallel Hl 디코더(짧은 서브코드) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 이미 row 단위 병렬(z_sb=32) 구조이나, sub-block 분해를 통한 완전 병렬화 접근은 HCU와 다른 병렬화 축 |
| H-matrix 서브블록 분해(Hl 생성) | `ecc_top.cpp` `Load_PCM()` | Block-LDPC(Hb+circulant) 구조는 Prime ECC의 `PCM_b`와 유사한 base+shift 방식이라 개념적으로 대응 가능 |

적용 가치: **높음** — min-sum/HCU/Dual-Update와는 다른 축(서브블록 순차 디코딩+extrinsic 조기 전달)의 수렴 가속 기법이며, MUvn 메모리 완전 제거는 Prime ECC 메모리 구조 개선에 직접 참고 가치가 있음. 다만 Nl=1 최적 설정은 802.16e 특정 z/Hb 구조에 종속적이라 Prime ECC의 z_sb=32/HCU 비대칭 col 구조에 대한 재검증이 필요함.
