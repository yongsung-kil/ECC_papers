> 총평: protograph LDPC 앙상블의 stopping-set enumerator에 대한 점근적/조합론적 이론 분석 논문으로, 특정 부호나 디코더 구현이 아닌 앙상블 평균 성질(다항식 vs 지수 growth)을 다루어 Prime ECC의 어떤 모듈과도 직접 대응되지 않음.

### B. 알고리즘 요약

1. 대상은 protograph 기반 및 표준 LDPC 앙상블(코드워드 길이 n)로, BEC(binary erasure channel) 상의 iterative decoding 성능을 stopping set으로 분석.
2. 풀려는 문제: 기존 점근 분석([3],[4])은 코드길이에 선형으로 증가하는(linear growth) stopping set만 다뤄, 짧은 코드에서 지배적인 sublinear(로그 이하) growth stopping set의 유한장 거동을 설명하지 못함.
3. 핵심 가정: stopping set 크기 `v`가 코드길이 `n`에 대해 로그 이하로 증가하는 "category-P" 영역과, 선형으로 증가하는 "category-E" 영역을 구분.
4. 핵심 기법: 표준/protograph 앙상블의 정확한 stopping-set enumerator `s(n,v)`(식 (1),(2))에 상한/하한 결합을 적용해 다루기 쉬운 근사식(Theorem 2, 3) 도출.
5. 핵심 식: category-P에서 `s(n,v) ~ n^(-E)` (다항관계), 지배 지수 `E_s = ⌈d_v,min·v/2⌉ - v` (표준), `E_p = w*-v` (protograph, `w*`=stopping set에 연결된 체크노드 최대 수) — protograph 구조가 `E_p ≥ E_s`를 보장해 error-floor 개선.
6. 확장: 근사식이 유효한 `(v,n)` 영역을 Theorem 4로 정량화(오차항 A와 지배항 비교, 임계 코드길이 `N_0` 계산).
7. precoding 기법(체크노드+변수노드 1쌍 추가 후 puncture)을 protograph에 적용 시 stopping-set 특성 `(v'-e'+w*')`가 원 protograph 대비 개선됨을 증명(Theorem 5) → error-floor 성능이 저하되지 않음을 이론적으로 보장.
8. 학습/최적화 절차 없음 - 순수 조합론적 상한/하한 유도 및 수치 검증(exact enumerator 계산).
9. 결과 예시: regular (3,6) 표준 앙상블 vs protograph-P 앙상블 비교에서, 홀수 크기 v에 대해 protograph-P의 log-log 기울기가 표준 앙상블보다 1 작아 stopping set 수가 더 적음(Fig. 2); v=5일 때 근사가 유효한 최소 코드길이 `N_0=222`.
10. 한계: HW 미설계, 시뮬레이션이 아닌 이론적 앙상블 평균 분석(특정 QC-LDPC 인스턴스 아님), BEC 채널 가정으로 AWGN/NAND read-channel과는 직접 연결 안 됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stopping-set enumerator 근사(앙상블 평균) | 대응 없음 | Prime ECC는 특정 고정 QC-LDPC(PCM_b) 시뮬레이터로 앙상블 통계 분석 기능 없음 |
| Protograph 구조/precoding | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 설계 관점에서만 개념적 연관(높은 이식 난이도, 특정 QC 고정) |
| BEC 채널 모델 | `channel.cpp` | Prime ECC는 AWGN/RBER/fixed-error 채널만 지원, BEC 미지원 |

적용 가치: 낮음 — 특정 부호 인스턴스가 아닌 LDPC 앙상블의 점근적 통계 성질(BEC 기반)에 대한 순수 이론 분석으로, 디코더 알고리즘이나 HW 구현과 무관하며 이식 가능한 코드/모듈 요소가 없음.
