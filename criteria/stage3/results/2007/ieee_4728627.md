> 총평: 디코딩 실패 시 최종 결과 대신 failed-check-count 최소인 중간 iteration 결과를 반환하는 저비용 post-processing 아이디어로, 조기종료(CRC) 로직과 유사한 위치에 추가 가능해 Prime ECC 이식성이 높은 편.

### B. 알고리즘 요약

1. 시스템: rate 1/2 (1024,512) PEG(progressive edge-growth) irregular LDPC 코드, AWGN 채널, 표준 확률영역 BP(sum-product) 디코더.
2. 풀려는 문제: 최대 iteration에 도달해 디코딩이 실패(syndrome `Hc≠0`)하는 프레임의 경우, 최종 iteration 시점의 비트 오류 수가 최적이 아닐 수 있음 — 중간 iteration에 더 적은 오류를 가진 순간이 존재.
3. 핵심 관찰: 미수렴 프레임의 오류 패턴을 iteration에 따라 관찰하면 오실레이팅(주기적 큰 변동), 준일정, 랜덤형 3가지로 분류되며, 고SNR일수록 오실레이팅 패턴이 지배적(3dB에서 63%).
4. 핵심 기법: syndrome `Hc`의 비영 원소 수(`Check Errors`, 즉 실패한 parity check 개수)를 매 iteration 추적하여, 실제 비트오류 수는 모르지만(수신 불가) `Check Errors`와 강한 상관관계가 있음을 이용해 대리 지표로 사용.
5. 알고리즘 변형: 기존 `Loop: update check/variable nodes → compute c → compute Hc → until Hc=0 or max iter` 에서, 매 iteration마다 `Check Errors < Minimum`이면 `Minimum=Check Errors`, `m=c`로 최적 후보를 갱신하고, 종료 시 마지막 `c`가 아닌 저장된 `m`을 반환.
6. 부수 구현: 추가 저장 공간은 최소 check-error 값(스칼라)과 해당 시점의 비트벡터 `m`(코드워드 길이) 뿐으로 구조적으로 단순.
7. 학습/최적화 절차 없음(휴리스틱 규칙 기반, 별도 훈련 불필요).
8. 결과: (1024,512) PEG 코드에서 SNR 2.25~3.0dB 구간 BER이 기존 대비 최대 40% 감소, SNR이 높을수록(오실레이팅 비중이 클수록) 개선폭 증가.
9. 한계: 부동소수점 SPA 시뮬레이션에 한정되어 HW 구현·고정소수점/min-sum 환경에서의 검증 없음, iteration 수 자체는 줄지 않고(여전히 max iteration까지 실행) 오히려 매 iteration syndrome 계산과 후보 갱신이 필요해 실행시간은 늘 수 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 매 iteration syndrome(failed check 수) 추적 및 최소 후보 비트벡터 저장 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 내부에 최소-check-error 후보 추적 로직을 추가하는 형태로 이식 가능 |
| 조기종료/실패시 최적 후보 반환 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 기존 CRC 기반 조기종료(성공시 종료)와 달리 이 기법은 "실패 확정시 최적 중간결과 반환"이라 상호보완적 post-processing으로 추가 가능 |
| syndrome 계산 (Hc) | `decoder.cpp` `C2V_Cal()` | 이미 check node update 과정에서 유사 계산(parity 만족 여부) 수행 중이라 추가 비교기만으로 확장 가능 |

> 적용 가치: 중간 — 알고리즘 자체는 단순하고 조기종료 모듈 인근에 이식 난이도가 낮으나, 표준 SPA 기반 검증뿐이라 min-sum·고정소수점 환경에서 오실레이팅 패턴 재현 여부와 추가 레지스터/비교 로직의 HW 비용 검증이 선행되어야 함.
