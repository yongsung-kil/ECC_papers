> 총평: turbo/LDPC 부호의 역사적 개관과 JPL의 protograph 기반 LDPC 설계·FPGA 구현·정지규칙을 폭넓게 다루는 서베이 논문으로, 특정 신기법 하나를 이식하기보다는 protograph 설계 사상과 조기종료 규칙 개념 참고에 그침.

### B. 알고리즘 요약

1. 딥스페이스 통신(AWGN, 매우 낮은 rate 1/6~1/2 turbo, rate 1/2~0.8752 LDPC)을 위한 turbo/LDPC 부호의 JPL 개발사를 다루며, protograph 기반 AR4JA LDPC 부호군(n=1024~16384)이 핵심 사례다.
2. 기존 turbo 부호는 저rate에서 우수하나 고rate·고속 디코딩에서 복잡도가 크고, 비구조 LDPC는 그래프가 랜덤이라 HW 라우팅이 어렵다는 문제를 다룬다.
3. 채널은 AWGN, BPSK 변조를 가정하며, protograph를 T배 lifting하여 대형 코드를 구성(순환shift인 circulant 또는 PEG/ACE 최적화 순열 사용).
4. 핵심 기법은 AR4JA(Accumulate Repeat-4 Jagged-Accumulate) protograph 설계로, 일부 degree-2 변수노드를 고차수 노드로 대체해 최소거리를 블록길이에 선형으로 증가시킨다(참고 [44]).
5. 체크노드 갱신은 `min*(v1,v2)=sgn(v1)sgn(v2)(min(|v1|,|v2|)+correction term)`(식 3) 형태이며, 보정항은 lookup table/2-segment 선형근사로 단순화하거나 생략(최대 1.0dB 손실) 가능하다.
6. 확장(lifting)은 2단계로 수행: protograph를 T=4로 1차 lifting(PEG, parallel edge 제거) 후 circulant phase를 Viterbi-유사 알고리즘으로 ACE 변형을 최대화하도록 선택해 T=n/16 만큼 2차 lifting한다.
7. HW 구현으로 비구조 그래프용 "universal decoder"(macro-column/row 분할 + crossbar, L=16 파티션, ~10Mbps, Xilinx Virtex-II)와 circulant 구조 전용 고속 디코더(2~80Mbps, Xilinx FPGA)를 언급한다.
8. 부호 설계 최적화는 밀도진화(density evolution)의 근사인 reciprocal channel approximation(단일 파라미터 s로 SNR 진화 추적)으로 protograph 문턱값(threshold)을 계산해 protograph nonoptimality를 약 0.4dB까지 최소화했다.
9. 결과: AR4JA(n=8192,k=4096) 부호는 Eb/N0=1.35dB에서 평균 22.5회 iteration(최대 50~200회 필요 경우 존재)으로 WER<10^-6 달성, stopping rule 적용 시 작은 버퍼(5프레임)로도 평균 iteration 근처로 지연 감소 가능.
10. 한계: HW 구현 수치(gatecount, 정확 throughput)는 별도 참고문헌(IPN 42-168 등)에 있고 본문엔 범위(2~80Mbps)만 제시되며, 매우 저rate·긴 지연 허용 딥스페이스 환경 특화로 NAND의 고속·짧은 latency 요구와는 맥락이 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Protograph/circulant 기반 QC-LDPC 부호 구성(H-matrix 설계) | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 base matrix+circulant shift 구조라 개념적으로 유사하나 특정 QC-LDPC 고정이라 재설계는 이식 난이도 높음 |
| min* 연산(min-sum + 보정항) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 순수 min-sum(보정항 없음)이라 correction term 적용 여부 검토 가능하나 이미 유사 근사 보유 |
| Stopping rule(코드워드 조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC의 조기종료(Partial/Full CRC)와 개념적으로 동일한 목적, 신규 기법 아님 |
| Universal decoder(macro-column/row + crossbar 라우팅) | 디코더 HW 모델 (GT.cpp 등) | 비구조 그래프 대응 라우팅 기법으로 Prime ECC의 고정 QC 구조와는 다른 문제이나 라우팅 개념 참고 가능 |
| 가변 iteration + 입력버퍼(preemptive buffer control) | 대응 없음 | Prime ECC는 고정 iteration/조기종료 방식이며 프레임 도착시간 기반 버퍼 스케줄링은 다룬 바 없음 |

적용 가치: **낮음** — 딥스페이스 특화 서베이 논문으로 신규 알고리즘 제안이라기보다는 기존 JPL 성과(protograph, universal decoder, stopping rule) 개관이며, 개별 요소들은 이미 Prime ECC가 유사 형태(QC-LDPC, min-sum, CRC 조기종료)로 보유하고 있어 추가 이식 가치가 제한적.
