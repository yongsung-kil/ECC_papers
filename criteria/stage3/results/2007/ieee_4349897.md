### B. 알고리즘 요약

1. Array LDPC 코드는 `j,k,L` (row/column weight, submatrix 크기) 파라미터로 정의되며 rate 0.9167(11/12)~0.9333(14/15), 순환 shift identity submatrix로 구성(`H_A`).
2. 기존 array LDPC는 4-cycle을 없애려면 submatrix 크기 L이 소수(prime)여야 하는데, 이는 코드 길이가 L의 배수로만 제한되어 임의 길이 지원이 어렵고, 소수가 아닐 때는 dummy bit 삽입이나 puncturing이 필요해 rate/성능이 저하됨.
3. 채널 모델은 AWGN(BPSK) 및 UWB multiband OFDM CM3(IEEE 802.15.3a, 16QAM)이며 디코딩은 LLR-BP(max iteration 100).
4. 핵심 기법: 순환 shift 값을 `Psc(j,k) = (j-1)(k-1) + floor((j-1)(k-1)/L)`로 재정의(식 3)하여 기존 `(j-1)(k-1) mod L`(prime 전제) 대신 non-prime L에서도 중복 permutation 쌍(4-cycle 원인)을 줄임.
5. 식의 의미: 두 번째 항(`floor(...)`)이 "shift 방지" 보정항으로 작용해, 동일 permutation 번호가 같은 행 쌍에서 두 열 이상 겹치는 것을 억제함(첫/마지막 column 사이 잔여 4-cycle만 남김).
6. 확장 기법: 4-cycle을 추가로 제거하기 위해 행 번호에 따라 열 순서를 재배치하는 SC-array LDPC 최종 구성(`H_sc`, 식 4) 제안.
7. 부수 트릭: 순수 permutation index 재배치(추가 연산 없음)만으로 4-cycle을 제거해 디코더/인코더 구조 변경이 필요 없음.
8. 최적화 절차 없음(학습/DE/GA 미사용, 결정론적 permutation 공식).
9. 결과: R=11/12, 14/15 AWGN 시뮬에서 SC-array LDPC가 non-prime array LDPC 및 punctured array LDPC보다 우수한 BLER, prime array LDPC와는 거의 동일 성능; UWB CM3 16QAM 1158/1192Mbps 시스템에서도 non-prime array LDPC(error floor 발생) 대비 우수, dummy bit 없이 prime 코드와 동등 성능 달성.
10. 한계: HW 구현/디코더 복잡도 분석이 전혀 없고 순수 시뮬레이션(이론적 구성법) 검증에 그침; array LDPC 고유 구조에 특화되어 일반 QC-LDPC 설계로 일반화하려면 추가 검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| non-prime submatrix 크기에서 4-cycle 회피 permutation 식(`Psc`) | `ecc_top.cpp` `Load_PCM()` | H-matrix/PCM 구성 로직과 관련되나 array LDPC 전용 순환식으로 QC-LDPC base matrix 설계와는 별개 알고리즘, 참고 수준 |
| 임의 코드 길이 지원(dummy bit/puncturing 회피) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 이미 특정 QC-LDPC H-matrix(`matrix_sel`) 고정 사용, 부호길이 가변화 기법으로 참고 가능하나 재검증 부담 큼 |
| 디코딩 알고리즘(LLR-BP) | 대응 없음 | 논문은 표준 sum-product/BP만 사용하며 Prime ECC의 min-sum/HCU 등 HW 최적화 기법과 무관 |
| HW 구현/디코더 아키텍처 | 대응 없음 | 본문에 HW 설계·복잡도 언급 없음 |

적용 가치: **낮음** — 4-cycle 회피를 위한 순환 permutation 공식은 부호구성 아이디어로는 흥미로우나 Prime ECC는 이미 확정된 QC-LDPC base+circulant 구조를 사용하고 있어 H-matrix 재설계는 이식 난이도가 높고, 표준 BP 디코딩만 다뤄 디코더/HW 이식 요소가 없음.
