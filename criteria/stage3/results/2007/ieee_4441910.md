### B. 알고리즘 요약

1. 대상 부호: IEEE 802.11n 표준 QC 구조 irregular LDPC, 648bit, rate 1/2, 12x24 sub-block(27x27 circulant).
2. 문제: 기존 accelerated message-passing 스케줄([8], 원래 regular LDPC용)을 irregular LDPC에 그대로 적용하면 column weight가 불균일해 불필요한 column 연산·메모리 갱신이 발생.
3. 채널 모델: AWGN.
4. 핵심 기법 1: row 연산 후 P(β) 메시지 중 실제로 다음 row 연산에서 사용되는 "최신" 메시지만 기록하도록 메모리 접근 빈도를 축소.
5. 핵심 기법 2: column 연산 중 후속 row 연산 전에 값이 덮어써져 사용되지 않는 "redundant" column 연산을 스케줄 분석(shift 값 비교)으로 사전에 생략.
6. 핵심 식: `N_wc = sum(R_j,wc)` (column weight별 update 횟수 합), `K_wc = ceil(N_wc / Wr_MAX)`로 필요한 BFU(Bit node Function Unit) 개수 산출.
7. 구현 디테일: row weight가 상이(7,8)해도 최대 row weight(Wr_MAX=8) 기준 파이프라인 주기(Wr+1 clock)로 통일. column weight별(2,3,12) 전용 BFU 배열 공유로 15개 BFU 절감.
8. 최적화 절차 없음(휴리스틱 스케줄 분석, DE/GA 미사용).
9. 결과: column 연산 약 14% 감소, TSMC 0.18um 합성 시 면적 30% 감소, 전력 36% 절감, 동일 iteration(5)·동일 처리량(54Mbps)에서 BER 성능은 기존과 거의 동일.
10. 한계: 특정 648bit rate-1/2 표준 부호 한 종류로만 검증, 정정성능 개선 없음(순수 HW 비용 절감), FPGA/실측 없이 ASIC 합성 결과만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| message-passing 스케줄(불필요 column 연산 제거) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 내 메시지 갱신 스케줄 최적화 아이디어로 참고 가능 |
| 메모리 접근 빈도 축소(β 메시지) | `decoder.cpp` (HCU 관련 로직) | HCU가 이미 비대칭 degree column을 half-cycle 처리하는 유사 목적 대응 |
| BFU 배열 공유(면적 절감) | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 edge 선택 MUX와 목적(HW 면적 절감) 유사하나 접근 방식 상이 |
| 부호 자체(802.11n H-matrix) | `ecc_top.cpp` `Load_PCM()` | 대응 없음(다른 표준 부호, 이식 불필요) |

적용 가치: 낮음 | 이미 Prime ECC는 HCU/GT로 유사한 HW 스케줄 최적화를 보유하고 있으며, 본 논문은 특정 표준 부호에 국한된 스케줄 재배치 기법으로 정정성능 개선이 없어 중복성이 높음.
