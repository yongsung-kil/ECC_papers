### B. 알고리즘 요약

1. LDPC 디코딩은 factor graph 위 message-passing(BP)이며 check-to-variable/variable-to-check 메시지는 LLR, check node 갱신은 `m_ci→vj = 2·atanh(Π tanh(m_vb→ci/2))`(sum-product) 형태.
2. 문제: flooding(동시 갱신)이나 LBP(순차 고정 스케줄)는 trapping set(작은 odd-degree 이웃을 갖는 variable node 집합)에 취약해 error-floor를 유발.
3. 핵심 가정: check node별 "residual"(갱신 전후 메시지 차이의 norm)을 min-sum으로 근사한 `approximate residual`을 스케줄링 지표로 사용.
4. 핵심 기법 1단 - ANS(Approximate Node-wise Scheduling): 매 스텝 residual 최댓값 `α_c = argmax_u α_cu`인 check node를 골라 우선 갱신, trapping set의 degree-2 check가 degree-1로 바뀌며 순차적으로 오류 정정.
5. ANS의 문제(짧은 블록길이에서): non-ML undetected error(greedy하게 잘못된 방향으로 수렴, 유클리드 거리가 ML보다 먼 코드워드로 확정) + myopic error(length-4 cycle 밀집 구조에서 같은 몇 개 check node만 반복 갱신).
6. 핵심 기법 2단 - Mixed scheduling: F-LBP/ANS(고정 `ξ`회 LBP 후 ANS 전환), A-LBP/ANS(unsatisfied check 수가 임계값 `ζ` 이하로 떨어지면 ANS 전환) — LBP로 초기 greedy 오류 방지, ANS로 trapping set 해결.
7. 부수 기법 - LC-ANS(Low-Complexity ANS): 전체 check node residual 비교 대신, min-sum에서 이미 알고 있는 최소 신뢰도 2개 variable-to-check 메시지에 대응하는 residual 2개만 합산한 간이 지표 `α_c^LC` 사용, 계산량 대폭 감소.
8. 최적화 절차: `ξ`, `ζ` 파라미터는 Monte Carlo 시뮬레이션으로 튜닝(본 논문에서는 미세 최적화는 후속 연구로 언급, `ξ=35`, `ζ=5` 예시값 사용).
9. 결과: blocklength 648, rate-1/2 코드 5종에서 mixed 전략이 순수 ANS 대비 UFER를 낮추고(예: code A FER 5.8e-5→3.9e-6), blocklength-1944 코드에서 LC-ANS/A-LBP/ANS가 ANS와 유사한 성능을 더 낮은 복잡도로 달성.
10. 한계: 순수 시뮬레이션(AWGN, Monte Carlo) 결과이며 HW 구현/합성/throughput 분석은 없음, 동적 check-node 선택(정렬/최댓값 탐색)의 하드웨어 비용은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ANS/LC-ANS 동적 스케줄링(residual 기반 check node 선택) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프의 갱신 순서 결정 로직에 직접 개입 가능한 새 스케줄링 방식, min-sum 기반이라 CNU 연산과 호환 |
| min-sum 근사 residual 계산 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 min-sum(magnitude) 연산을 수행하므로 residual 근사값 산출에 필요한 데이터가 존재, 추가 비교/저장 로직만 필요 |
| Dual-Update(even/odd 스케줄) | `decoder.cpp`/Dual-Update 스케줄 | 논문의 LBP(순차)/flooding과 유사한 고정 스케줄 범주에 해당, mixed scheduling의 "LBP 단계"에 대응 가능하나 ANS 동적 선택은 현재 미보유 |
| trapping-set 대응 조기 완화 | 대응 없음 | Prime ECC는 partial/full CRC 조기종료만 있고 trapping-set 자체를 겨냥한 스케줄링 대응책은 없음 |

> 적용 가치: 높음 — min-sum 및 layered 스케줄을 이미 보유한 Prime ECC 디코더에 ANS/mixed scheduling을 이터레이션 순서 변경만으로 추가해 error-floor(trapping-set) 개선을 시도할 수 있으나, 동적 선택 로직의 HW 오버헤드(비교기/정렬 회로) 검증이 선행되어야 함.
