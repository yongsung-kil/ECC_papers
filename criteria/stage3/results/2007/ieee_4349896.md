> 총평: IRA류 구조 LDPC에 shuffle scheduling 설계규칙과 메모리 절감형 SPC 프로세서를 적용한 FPGA 디코더 아키텍처 논문으로, 스케줄링/메모리 절감 아이디어는 이식 가능하나 부호설계 자체는 QC-LDPC와 다른 IRA 계열이라 이식 난이도가 있음.

### B. 알고리즘 요약

1. 구조화 LDPC(IRA 유사) 코드: `H=[Hs Hp]`, Hs는 순환 shift identity block, Hp는 (비순환) shift triangular accumulator 구조.
2. 기존 방식은 코드 설계와 디코더 아키텍처를 분리 최적화해 하드웨어 비친화적 구조가 되는 문제를 해결하려 함.
3. 채널은 AWGN, 코드워드는 `x=[c p]`, `HxT=0` 제약.
4. 핵심 기법은 turbo-like 디코딩: 코드를 outer(SPC)와 inner(accumulator, 2-state trellis) 서브디코더로 분리해 각각 SPC decoding과 BCJR형 FBA(`Fj`, `Bj`, `mIO(j)`)로 처리.
5. 핵심 식 `g(a,b)=sign(a)sign(b)min(|a|,|b|)`는 min-sum 근사이며, SPC inward/outward 메시지 `miOI`, `mciv` 계산에 사용.
6. 설계 규칙: "시스템 비트는 한 decoding window 내에서 한 번만 사용"(`δ(i,j)=δ(k,j), ∀i≠k`)이 shuffle scheduling과 동일한 효과를 내어 수렴을 가속.
7. 구현 디테일: SPC outward 단계에서 2개 최소값(`|m1|`,`|m2|`)만 저장/비교하는 방식으로 메모리 약 70%(J가 클 때) 절감, 채널 양자화 4bit, mcj 양자화 5bit.
8. serial(window) scheduling을 semi-pipeline으로 확장해 forward/backward 재귀와 outward 체크를 겹쳐 처리(그림5, throughput 식 `Db=(2mk+α)/(Imax·fclk)`).
9. 결과: N=16320, R=1/2 예제에서 제안 스케줄링 5 iteration이 기존 스케줄링 7 iteration과 동등한 성능(throughput 40% 향상), FPGA 구현은 7 iteration에서 5.4Mbit/s(50MHz), fully pipelined 시 14.28Mbit/s.
10. 한계: ASIC 대비 FPGA는 클록이 낮아 throughput 직접비교 어렵고, fully pipelined 구조는 코드 설계 제약을 더 강화시켜 코드 선택이 어려워짐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| turbo-like min-sum(SPC+accumulator FBA) 디코딩 알고리즘 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 표준 min-sum C2V/V2C 구조로 IRA류 turbo-like 분리 디코딩과는 다른 스케줄이나 min-sum 근사 자체는 대응 |
| shuffle 기반 serial(window) scheduling 설계규칙 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 교대 스케줄) | Dual-Update가 유사한 조기 정보 갱신 개념을 이미 채택, 이식 시 중복 가능성 |
| SPC outward 2-comparison 메모리 절감(min1/min2 저장) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC의 min1/min2 기반 min-sum 연산과 개념적으로 거의 동일(이미 보유) |
| 채널/mcj 양자화(4bit/5bit) | `channel.cpp` `Set_LLR_Th()` | 양자화 비트수 설계 참고 가능하나 IRA 특유 파이프라인 종속 |
| IRA류 H-matrix(Hs+Hp triangular) 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC base+circulant 구조(`PCM_b`)로 IRA의 triangular accumulator 파트와 구조가 달라 대응 없음 |

적용 가치: **중간** — min-sum 근사와 메모리 절감형 비교 로직은 이미 Prime ECC가 유사 기법을 보유하고 있어 신규성은 낮으나, shuffle scheduling에 따른 수렴 가속 아이디어는 참고 가치가 있음. 다만 코드 자체가 IRA 계열이라 부호설계 이식은 높은 난이도.
