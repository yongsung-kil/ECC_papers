> 총평: 2007년 FPGA 기반 QC-LDPC(EG-LDPC) 디코더로, SPA 페이즈 균형화·부분병렬 처리율 선형 증가 아키텍처·비균등 양자화 메모리 절감이 핵심이나 min-sum이 아닌 Sum-Product 기반이고 특정 EG 구조에 특화되어 Prime ECC로의 직접 이식은 제한적, HW 최적화 아이디어(메모리 파티셔닝/스위칭망)만 참고 가치.

### B. 알고리즘 요약

1. (8176, 7154) EG(Euclidian geometry)-based QC-LDPC 코드(column weight 4, row weight 32, rate≈0.875, 2×16 배열의 511×511 서브행렬) 대상 Xilinx Virtex-II 6000 FPGA 디코더.
2. 기존 Sum-Product 디코더는 CPU(check-node unit)가 2개 LUT연산+합산을 갖는 반면 VPU(variable-node unit)는 합산 1개뿐이라 두 노드처리유닛의 critical path가 불균형해 클록속도가 CPU에 의해 제한되는 문제를 해결.
3. 표준 log-domain SPA: `R_cv = φ^-1(Σφ(Q_vc))`, `Q_vc = P_v + ΣR_cv` (φ는 tanh 기반 비선형 LUT 함수)를 사용, AWGN+BPSK 채널 가정.
4. 핵심 기법 1단: LUT 연산 1개를 CPU에서 VPU로 옮기는 변형 SPA(식 5-7)로 두 유닛 간 연산부하를 균형화, sign-magnitude↔2's complement 변환 블록도 제거해 critical path를 약 1회 다중비트 덧셈(5~6bit) 수준으로 단축(3-stage 파이프라인).
5. 핵심 기법 2단(enhanced partially-parallel): 메모리뱅크를 서브뱅크로 분할하거나 인접 행/열 데이터를 한 엔트리에 패키징(δ배속)하고 스위칭망(데이터 버퍼+멀티플렉서)으로 메모리 접근충돌을 해결해, 추가 메모리 없이 소량 HW만으로 처리율을 선형 증가.
6. EG-LDPC 서브행렬(2개 중첩 순환치환 행렬 구조)에 맞춰 짝수/홀수 행·열 주소로 메모리를 분할하고 전이그룹(transition group) 재배열 규칙으로 접근충돌을 제거하는 EG 특화 스위칭 구조를 설계.
7. 부수 트릭: 7bit uniform 양자화(BER 1e-9 달성 필요조건) 대신 균등→비균등 변환기(U2NU, 단순 조합논리)를 도입해 6bit로 축소하면서 성능 손실 없음을 시뮬로 확인(90G random bit 시뮬).
8. 학습/최적화 절차 없음(고정 알고리즘/아키텍처 설계).
9. 결과: Xilinx Virtex-II 6000 FPGA에서 클록 192.6MHz(비균등 양자화 기준), 15 iteration에서 source data throughput 172Mbps 달성 — 당시 타 FPGA 구현([11]-[13],[23]) 대비 최고 처리율.
10. 한계: 정정성능(BER) 개선이 아닌 처리율/클록속도/메모리 효율화가 목적이며 특정 EG-LDPC 서브행렬 구조(2-초과 중첩 순환치환)에 특화된 스위칭망 설계라 일반 QC-LDPC로 일반화 시 추가 설계 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 변형 SPA (CPU-VPU 연산부하 균형화) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum 기반이라 φ-LUT 균형화 기법 자체는 대응 없음, critical path 단축 아이디어만 참고 가능 |
| 부분병렬 메모리 파티셔닝/스위칭망 | 디코더 HW 모델(z=32, HCU) | Prime ECC도 z_sb=32 row 병렬 구조 보유, 메모리 접근충돌 해결 아이디어는 부분 대응 |
| 비균등 양자화(uniform→nonuniform 변환) | `channel.cpp` `Set_LLR_Th()` | Prime ECC의 LLR 양자화 테이블(PARAM_LLR)과 비교 가능한 메모리 절감 기법 |
| EG-LDPC H-matrix 구조(2-중첩 순환치환) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조라 EG 특유 서브행렬 구성은 대응 없음 |
| full-tanh Sum-Product(φ 함수) | (대응 없음) | Prime ECC는 min-sum 전용, SP 알고리즘 자체는 이식 대상 아님 |

> 적용 가치: 중간 — 부분병렬 메모리 파티셔닝/스위칭망과 비균등 양자화 절감 아이디어는 HW 효율화 관점에서 참고 가치가 있으나, 핵심 알고리즘(SPA φ-LUT 균형화)은 min-sum 기반 Prime ECC와 계열이 달라 직접 이식은 어려움.
