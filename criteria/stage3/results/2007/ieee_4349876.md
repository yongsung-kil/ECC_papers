### B. 알고리즘 요약

1. 대상은 LDPC convolutional code(LDPCCC)로, block code와 달리 스트리밍/가변길이 데이터에 적합한 rate `R=b/c` 부호.
2. 기존 LDPCCC는 encoder를 zero-state로 되돌리는 zero-tail 때문에 짧은 프레임에서 rate-loss가 발생하는 문제가 있음.
3. protograph(소규모 템플릿 Tanner graph)를 `X`번 복사·permutation하여 girth를 키우는 구성법을 tail-biting 방식에 적용해 rate-loss를 회피.
4. mother LDPCCC의 syndrome former `H^T_[0,∞]`를 protograph 기반 PEG 변형 알고리즘으로 채우고, unwrapping 절차로 주기 `T=ms+1`의 구조를 얻음.
5. tail-biting 코드는 syndrome former의 마지막 `ms`개 열을 wrap-around시켜 `H~^T_[0,N-1]` (식 (6))로 정의, 코드시퀀스는 `v~H~^T=0`을 만족.
6. 복호는 circular topology를 갖는 pipelined belief-propagation 디코더(circular pipeline decoder)를 사용, `I_P` 개의 동일 병렬 프로세서 `D_i`가 메모리 섹션을 나눠 처리.
7. rate-compatible 확장: protograph의 특정 타입 변수노드(8,9,10)에 0/무한 reliability를 할당해 하나의 디코더로 rate 1/2,5/8,3/4,7/8을 모두 지원.
8. 학습/최적화는 별도 없음 - PEG 알고리즘의 저차수 우선 edge 배치로 short cycle만 회피.
9. 결과: TB-LDPCCC는 동일 블록길이 LDPCBC와 유사한 BER 성능을 보이며 error floor는 약 10^-5 부근에서 공통 발생, 동일 성능 기준 하드웨어 복잡도는 TB-LDPCCC가 LDPCBC 대비 최소 2배 낮음(설계 노력 기준).
10. 한계: 이론적 분석 없이 시뮬레이션(AWGN, BPSK)만 수행, HW 미설계(pipeline 구조 개념만 제시), encoder는 향후 연구 과제로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph 기반 tail-biting 구성법 | 대응 없음 | LDPCCC는 convolutional 구조로 Prime ECC의 block QC-LDPC(`ecc_top.cpp` `Load_PCM()`)와 패러다임 불일치 |
| circular pipeline belief-propagation 디코더 | `decoder.cpp` `LDPC_Decoder()` | 파이프라인 개념은 유사하나 convolutional 특유 구조로 직접 이식 어려움 |
| rate-compatible puncturing(변수노드 타입별 0/무한 할당) | 대응 없음 | Prime ECC는 고정 rate 구조, puncturing 스킴 자체는 개념적 참고만 가능 |

> 적용 가치: 낮음 - LDPC convolutional code 특유 구성법으로 NAND용 고정 블록 binary QC-LDPC와 구조적으로 이질적이며, HW 미설계·시뮬뿐이라 즉시 참고할 요소가 적음.
