### B. 알고리즘 요약

1. DVB-S2 표준의 LDPC-IRA 코드(frame length 16200/64800, rate 1/4~9/10), M=360 주기성을 갖는 A행렬 구조와 staircase lower-triangular B행렬로 구성된 parity-check matrix H.
2. 문제: 기존 M-kernel 부분병렬 아키텍처([9])는 M=360개의 functional unit(FU)과 넓은 barrel shifter가 필요해 ASIC 면적이 22.74mm²(0.13um)에 달함.
3. 핵심 가정: DVB-S2 코드의 모든 코드가 공통 인수 M=360을 공유하며, ν_I 노드를 M개씩 그룹화한 pseudo-random 연결 구조(`(r+i×q) mod(n-k)`, `q=(n-k)/M`)를 가짐.
4. 핵심기법: M=L×N으로 인수분해해 C(c), R(r) 집합을 L개 서브그룹으로 subsampling, N개(= M/L)의 FU만으로 동일 아키텍처를 재사용하는 구조(`Cγ(c)`, `Rβ(r)`).
5. 핵심 식 `shift_γ(c) = (shift_0(c)+γ) div L`, `address_β(r) = (address_0(r)+q×wC×β) mod(q×wC×L)` — 기존 ROM(shift/address) 테이블을 재사용하며 새 인덱스를 즉석 계산해 ROM 크기 증가 없이 재구성.
6. 확장: barrel shifter 복잡도가 `O(N log2 N)`로 감소하고 FU 블록 하드웨어가 인수 L만큼 선형 축소, 라우팅 문제도 단순화됨.
7. 구현 디테일: 메시지 메모리 매핑 행렬 R을 L열로 reshape((7)식)해 기존 아키텍처의 매핑 스킴을 그대로 활용, 클록당 wj-3 idle cycle이 L배로 증가.
8. 별도 학습/최적화 절차 없음(구조적 서브샘플링 유도이며 GA/DE 등 사용 안 함).
9. 결과: N={45,90,180} FU 구성을 Xilinx Virtex-II Pro(XC2VP30/50/100)에 합성, 최소 90Mbps(mandatory 최소 throughput) 이상 달성, 180-FU 구성은 45-FU 대비 4배 throughput.
10. 한계: ASIC(전용 실리콘) 대신 FPGA 합성 검증만 수행했고, DVB-S2 표준의 특정 IRA 코드 계열(M=360 고정)에 특화된 구조로 일반 QC-LDPC로의 이식은 재검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| M-kernel 부분병렬 디코더를 인수 L로 서브샘플링 | 디코더 HW 모델 (z=32, HCU, GT) | 병렬화 레벨(z_sb) 조정과 유사한 개념이나 Prime ECC는 이미 z=32 고정 lifting 구조 채택 |
| 메시지 메모리 매핑/barrel shifter 재사용 (ROM 재계산) | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 MUX 기반 edge 선택과 유사 목적(라우팅/면적 절감)이나 구현 방식 상이 |
| DVB-S2 IRA H-matrix(A/B 구조, 360-periodicity) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC base+circulant 구조로 IRA staircase 구조와 직접 대응 없음 |
| FPGA 합성 검증 (면적/throughput 곡선) | 대응 없음 | Prime ECC는 bit-exact C++ 시뮬레이터로 RTL 합성 결과 자체는 참고 데이터에 불과 |

적용 가치: 중간 | 병렬화 인수(FU 개수)를 유연하게 조절하면서 기존 메모리 매핑을 재사용하는 면적-throughput trade-off 설계 아이디어는 Prime ECC의 HW 아키텍처 검토에 참고할 수 있으나, DVB-S2 IRA 코드 구조에 특화되어 있어 직접 이식은 어려움.
