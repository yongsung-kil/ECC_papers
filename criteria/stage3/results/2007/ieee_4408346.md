> 총평: 정정능력 개선이 아닌 high-rate LDPC에서 row weight가 클 때 CNU의 곱셈/비교 연산 수를 상수(2개)로 줄이는 HW 구조 최적화로, min-sum CNU HW 이식에 참고 가치 있음.

### B. 알고리즘 요약

1. IEEE 802.16e 등 high-rate(rate 3/4, 5/6) LDPC 코드에서 row weight `dc`가 클 경우(예: dc=14~20) Normalized Min-Sum CNU가 `dc`개의 곱셈 유닛을 필요로 해 하드웨어 복잡도가 큼.
2. 기존 정규 병렬 CNU 구현은 row weight에 비례해 곱셈(Normalized Min-Sum) 또는 비교/선택(Row Weight Matched Min-Sum) 유닛 수가 늘어나는 것이 문제.
3. 채널 모델은 AWGN, 표준 Log-BP 대비 Min-Sum 계열 근사 알고리즘의 성능/복잡도 트레이드오프를 다룸.
4. 핵심 기법: CNU 내부에서 최솟값 `Sm`과 두 번째 최솟값 `Sm'`을 최소/최대값 위치 인덱스 `IDXm`(및 이를 4-way 디코딩하는 `2-4 DEC`)로 산출한 뒤, 곱셈(정규화 factor `α`/scale factor `SF`)을 최솟값·2차 최솟값 두 개에만 적용.
5. 식 `Lmn = Π sign(Vmn') · α·Min(|Vmn'|)`(Normalized Min-Sum, 식 3)에서, 기존에는 row 내 모든 `Vmn`에 곱셈이 필요했으나 제안 구조는 min/2nd-min 값만 곱셈 대상으로 하고 나머지는 MUX로 선택.
6. Row Weight Matched Min-Sum(식 4, `CON=3ln(dc-1)/8`, `DF=ln(dc-1)/4` 기반 보정)도 동일 원리로 C/S(compare/select) 유닛을 4개에서 2개로 축소.
7. 구현 디테일: ABS(절대값) 유닛 1계층, MIN 유닛 2~3계층, AND/OR/MUX/2-4DEC 조합 로직 추가로 곱셈 유닛 수를 row weight와 무관하게 2개로 고정(IDX 표현에 2비트만 사용해 메모리 절감).
8. 학습/최적화 절차 없음(순수 하드웨어 구조 재설계).
9. 결과: 시뮬레이션상 Normalized Min-Sum, Row Weight Matched Min-Sum이 최적 Log-BP와 거의 동일한 성능(Min-Sum은 Log-BP 대비 약 0.5dB 열화), N=576, rate 3/4·5/6 코드에서 확인. 하드웨어 곱셈/비교 유닛 수는 row weight 무관 상수로 감소(구체 gate count 수치는 미제시).
10. 한계: 실제 ASIC/FPGA 합성 결과 없이 알고리즘·회로 구조 제안과 성능 시뮬레이션에 그침, 텍스트 추출 손상으로 회로도(Fig.5, 6) 세부 확인 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Normalized/Row Weight Matched Min-Sum CNU 재구조화(곱셈 유닛 축소) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 min-sum(min1/min2) 기반이라 최솟값·2차최솟값 산출 구조가 유사, 정규화 factor 적용 방식 최적화에 참고 가능 |
| 최솟값/2차최솟값 위치 인덱스(IDXm) 및 2-4 DEC 구조 | `decoder.cpp` `Get_VNU_Table_Idx()` | min1/min2 already 구현된 구조와 유사한 인덱싱 기법, 세부 회로 재설계 시 참고 |
| Row weight 종속 보정(dc별 CON/DF 상수) | `ecc_data.h` `PARAM_LLR` | LLR/보정 테이블 파라미터화 방식과 개념적으로 연관 |
| Log-BP/Max-Log-MAP 대비 성능 비교 | 대응 없음 | Prime ECC는 순수 min-sum 기반이며 tanh 기반 Sum-Product는 미보유(§3), 비교 대상일 뿐 이식 대상 아님 |

> 적용 가치: 중간 - 정정능력 개선은 아니지만 이미 보유한 min-sum CNU의 곱셈/정규화 연산을 row weight 무관 상수 개수로 줄이는 HW 최적화 아이디어는 고rate 코드의 CNU 게이트 절감에 참고할 만함.
