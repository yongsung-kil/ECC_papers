### B. 알고리즘 요약 (10줄 내외)

1. 대상: BSC 채널에서 hard-decision 반복복호기(GA, GB, MB0 등)로 복호되는 regular/irregular LDPC 코드, block length 200~4000.
2. 문제: 낮은 error rate(특히 error floor) 성능을 Monte Carlo 시뮬레이션으로 추정하려면 계산량이 지나치게 크거나 불가능함.
3. 채널/가정: BSC, crossover probability `ε`, 복호 알고리즘은 대칭이라 가정해 all-zero codeword 전송만 고려.
4. 핵심 개념: decoder가 정정 못 하는 최소 가중치 `J`와 그 개수 `|E_J|`를 알면 FER/BER를 해석적으로 근사 가능(선행연구 [9]의 식 (1),(2)).
5. 기존 한계: `J`, `|E_J|`를 구하려면 가중치 ≤J인 모든 오류패턴을 직접 열거해야 하며 복잡도가 `n^J`로 지수적 증가.
6. 핵심 관찰: 정정 실패하는 저가중치 오류패턴 대다수는 Tanner graph의 short cycle(길이 `l=g, g+2, …`, girth `g`)의 부분집합(subset)임.
7. 제안 알고리즘(Algorithm 1): 길이 `l`인 cycle 집합 `C_{l/2}`와 그 부분패턴(가중치 1~l/2)을 순차적으로 열거·복호 테스트하여 `J`, `|E_i|`를 점진적으로 갱신, 두 연속 추정치가 수렴하면 종료.
8. 추정식 개선: 새로운 `|E_i|_est` 산출 시 중복 카운팅 방지를 위해 이전 라운드에서 이미 포함된 부분집합 `E'`를 빼주는 보정식(식 (5)) 사용.
9. 결과: 복잡도 비율 `T`(직접열거 대비)가 실험한 8개 코드에서 최대 1.7×10^5 이상까지 감소, FER/BER 추정치는 Monte Carlo 시뮬 결과와 거의 일치.
10. 한계: 순수 소프트웨어 해석/추정 도구로 HW 설계 없음, hard-decision 알고리즘(GA/GB/MB0) 전용이며 min-sum 등 soft-decision 알고리즘에 대한 적용은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Error rate(FER/BER) 추정 알고리즘 | 대응 없음 | Prime ECC는 bit-exact HW 시뮬레이터이며 별도의 해석적 error-floor 추정 모듈은 없음 |
| hard-decision 반복복호(GA/GB/MB0) | 대응 없음 | Prime ECC는 min-sum 기반 soft-decision(HD/2SD/3SD) 디코더로 GA/GB/MB0과 알고리즘 상이 |
| Tanner graph cycle/trapping set 분석 | 대응 없음 | H-matrix 로드(`ecc_top.cpp` `Load_PCM()`)는 있으나 cycle enumeration 기반 error-floor 해석 기능은 없음 |
```

적용 가치: 낮음 — 알고리즘/부호/HW 개선이 아닌 오프라인 성능추정 방법론이며, 대상 복호 알고리즘도 Prime ECC의 min-sum 계열과 다름.
