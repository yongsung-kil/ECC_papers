### B. 알고리즘 요약

1. RCP-LDPC(rate-compatible punctured LDPC) 코드는 낮은 구현비용으로 다양한 rate를 지원하지만, punctured bit로 인해 BP 디코딩 수렴이 느려지는 문제가 있다.
2. 기존 layered BP는 unpunctured 코드에서만 layer 순서에 따른 성능 차이가 미미했으나, punctured 코드에서는 layer 선택이 성능에 큰 영향을 미친다.
3. Punctured variable node는 초기 LLR이 0이며, 이웃 check node로부터 처음 non-zero 메시지를 받는 시점을 `k-SR`(k-step recoverable)로 정의한다.
4. 제안 기법: `k-SR` variable node를 처음 회복시키는 check node(survived check node)를 layer `L_k`에 배정, 나머지는 `L_{K+1}`에 배정한다.
5. layer를 `L_1`부터 오름차순으로 순차 처리하면 첫 iteration(K subiteration) 내에 모든 punctured variable node가 회복되어 수렴이 빨라진다.
6. layer 배정은 가장 높은 code rate 기준으로 한 번 수행하며, 중간 rate들에서도 동일한 layering을 유지해 구현 복잡도를 추가시키지 않는다.
7. rate 0.5 mother code(길이 2000, degree distribution `λ(x)=0.30780x+0.27287x²+0.41933x⁶`, `ρ(x)=0.4x⁵+0.6x⁶`)를 puncturing해 rate 0.6~0.9 코드를 생성, 최대 15 iteration의 AWGN 시뮬레이션.
8. 결과: rate 0.9에서 BER 10⁻⁴ 기준 제안 layering이 random layering 대비 0.7dB, 기존 BP 대비 0.8dB 이득. 최대 iteration을 10으로 제한 시 이득은 각각 1.3dB, 1.5dB로 확대.
9. 3GPP-LTE 구조화 코드(길이 2080, rate 0.5→0.88 punctured)의 AWGN/fading 채널 검증에서도 fading 채널에서 random 대비 1dB, 기존 BP 대비 2dB 이득 확인.
10. 한계: 순수 시뮬레이션(이론/BER 곡선)만 제시, HW 구현·gatecount·throughput 분석 없음. puncturing이 없는 코드에는 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 디코딩 알고리즘(layered BP 스케줄링) | `decoder.cpp` `LDPC_Decoder()` | layer 순서 결정 로직은 이터레이션 루프의 CN 처리 순서에 관여 가능하나 puncturing 전제 |
| check node layer 배정 | 대응 없음 | Prime ECC는 puncturing 기반 rate-compatible 구조가 없어 대응 모듈 부재 |
| iteration별 회복 순서 | `ecc_data.h` `PARAM_LLR` | punctured bit LLR 0 초기화 개념은 LLR 테이블과 무관하게 별도 스케줄 설계 필요 |

> 적용 가치: 낮음 — puncturing 기반 rate-compatible 코드 전용 기법으로 Prime ECC 3.1의 고정 rate QC-LDPC 구조와 전제 조건이 다름.
