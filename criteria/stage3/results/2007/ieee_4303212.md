### B. 알고리즘 요약

1. 시스템: patterned perpendicular magnetic recording 채널(GMR 리드헤드, 3-D reciprocity 리플레이 모델), 저장밀도 1~1.6 Tbit/in².
2. 문제: 패터닝 공정 결함으로 island 크기/위치에 lithography jitter가 발생, 이는 replay 파형을 열화시켜 데이터 복구가 어려움. 기존 연구(Hu et al.)는 jitter를 고려하지 않음.
3. 채널 모델: island edge shift가 평균 0, 분산 `σ_p`인 절단 가우시안 분포를 따르는 jitter 노이즈 + AWGN.
4. 부호: irregular LDPC, zigzag parity 패턴으로 parity 부분 `H_p`를 구성해 선형시간 인코딩 달성, `H = [H_p | H_i]`로 파티션.
5. Tanner graph는 progressive-edge-growth(PEG) 알고리즘으로 girth 최대화하도록 구성.
6. 부호 스펙: rate 0.9, parity length 456, information length 4096, 전체 code length 4552.
7. 복호: sum-product algorithm(SPA), 채널 MAP 검출기로부터 APP를 받아 반복 복호, LDPC iteration 5회 + system iteration(MAP↔LDPC 피드백) 2회가 최적 타협점으로 확인됨.
8. 결과: jitter 없을 때 LDPC 적용으로 SNR 약 5.5 dB 개선, `σ_p=2.5nm` jitter에서도 SNR 이득 약 10 dB (코드 rate loss 보정 위해 밀도를 1.27 Tbit/in²로 증가).
9. 저장밀도를 1.6 Tbit/in²(island period 20nm)까지 늘려도 최대 `σ_p≈2nm` jitter까지 acceptable BER(`10^-5`) 달성.
10. 한계: 순수 시뮬레이션(MATLAB)이며 HW 구현 없음, 코드 구성법 자체는 기존 zigzag+PEG 표준 기법을 채널에 적용한 것으로 신규 디코더/부호 알고리즘 기여 없음, NAND 채널과 무관한 자기기록 특화 모델.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| zigzag parity + PEG 구성 LDPC | 대응 없음 | 표준 구성법으로 특정 QC 구조 아님, `ecc_top.cpp` H-matrix와 무관 |
| Sum-Product 반복복호(SPA) | 대응 없음 | Prime ECC는 min-sum 근사 사용(`decoder.cpp`), full-tanh SPA는 이미 보유 기법 대비 열위 |
| patterned media jitter 채널 모델 | 대응 없음 | NAND RBER/fixed-error 채널(`channel.cpp`)과 물리적으로 무관 |

> 적용 가치: 낮음 — 자기기록 특화 채널·표준 LDPC 구성·SPA 복호로 NAND binary QC-LDPC 이식 요소 없음.
