### B. 알고리즘 요약

1. 대상 부호는 Simple Full-length Array LDPC(SFA-LDPC) `C_A(p,4)`, p×p 순환치환행렬 P와 단위행렬 I로 구성된 `H_A(p,j)` (pj × p²) 검사행렬, rate `1-(pj+1)/p²`.
2. 풀려는 문제: 임의의 실용적 LDPC 코드의 정확한 최소거리(minimum distance)를 구하는 일반 알고리즘은 없고, `C_A(p,4)`의 정확한 최소거리도 기존 연구(Mittelholzer 상한 12, Yang 하한 10)로 8~12 범위에서만 알려져 있었음.
3. 핵심 가정: 코드워드 v가 `C_A(p,j)`에 속할 필요충분조건은 대응하는 support matrix `S_v`의 각 행이 "cancel-out condition"(각 값이 짝수 번 등장)을 만족하는 것(Corollary 2.4).
4. 핵심 기법: p>7인 모든 소수 p에 대해 10개 열로 구성된 특정 support matrix `S`(Fig.3, normal form)를 파라미터 `a,b,c,d,e,k1~k5`의 모듈로-p 연립방정식(식 1~10)으로 명시적으로 구성, 모든 행에서 cancel-out 조건이 성립함을 직접 검증.
5. 핵심 식: 예컨대 `4a≡1`, `2e≡-5 (mod p)` 등 각 파라미터를 유일해로 결정(p가 3 이하 소인수를 갖지 않는 소수이므로 modulo 역원이 항상 존재), 이 해로 만든 10-column support matrix가 항상 유효한 코드워드를 구성함을 보임.
6. 확장/추가 결과: p=5,7인 경우는 별도로 최소거리 8이 이미 알려져 있어(Yang 결과 인용), 전체적으로 `d(p,4)=8 (p=5,7)`, `d(p,4)=10 (p>7)`로 정리(Theorem 3.2).
7. 부수 트릭: 열벡터가 모두 다르면 weight 10인 코드워드, 일부가 겹치면 weight 10 이하인 코드워드가 되므로 어느 경우든 상한 10을 증명하는 논증 구조 사용.
8. 학습/최적화 절차 없음(순수 수학적 증명, 사전 실험적 관찰(참고문헌 [9])로 파라미터 패턴을 발견한 뒤 해석적으로 증명).
9. 결과: `d(p,4)=10` (p>7 소수) 확정, MacKay의 일반 상한 `(j+1)!=120`(j=4)보다 훨씬 타이트한 값 제공; 코드 길이 `p²`를 늘려도(p 증가) 최소거리가 10에서 더 이상 성장하지 않음을 증명.
10. 한계: 순수 이론/조합론 논문으로 디코딩 알고리즘·HW·시뮬레이션 전무; 저자 스스로 "p>7인 `C_A(p,4)`는 코드 길이만 늘어나고 최소거리는 그대로라 실용적 가치가 낮다"고 결론지음, j≥5 일반화는 추측 수준으로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Array LDPC 최소거리 증명 | 대응 없음 | 순수 수학적 증명으로 코드 구조 자체를 다루지 않는 시뮬레이터 성격상 대응 모듈 없음 |
| Array LDPC 검사행렬 `H_A(p,j)` (순환치환 array 구조) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드라는 개념적 유사성만 있고, base+circulant shift 방식의 QC-LDPC(`PCM_b`)와는 구성 방식이 다름 |
| 최소거리(minimum distance) 분석 | 대응 없음 | Prime ECC는 시뮬레이터로 최소거리 계산/분석 모듈 없음 |

> 적용 가치: 낮음 — 순수 이론적 최소거리 증명이며 디코더·HW·복호성능과 무관, array LDPC 구조 자체도 Prime ECC의 QC-LDPC 구성법과 결이 다름.
