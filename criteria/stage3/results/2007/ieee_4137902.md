### B. 알고리즘 요약

1. Binary Erasure Channel(BEC) 상 LDPC 코드를 message-passing(MP)으로 복호할 때, 복호 실패의 원인인 stopping set을 분석 대상으로 함.
2. 기존 improved decoding(Pishro-Nik & Fekri, "Algorithm A/C")은 무작위 bit-guessing으로 stopping set을 해소하지만 regular/near-regular ensemble에서 개선 폭이 작음.
3. `g`개 bit를 guess해서 완전 복호되는 stopping set을 "g-solvable"이라 정의하고, 잔여 parity-check 행렬 `H_S`가 특정 순열 하에서 upper-diagonal 구조를 가지면 필요충분조건임을 증명(Theorem 1).
4. "free number" `F_S`(최소 guess 수)의 하한(Lemma 2)과 상한(Lemma 3, 4)을 그래프 degree 기반으로 유도.
5. 핵심 관찰: 낮은 오류율에서 stopping set 주변 unsatisfied check node 중 상당수가 degree-2(bivalent)이며, 이 정보를 이용해 stopping set 내 bit node를 동치류(equivalence class)로 묶어 그래프를 축약(contraction)할 수 있음.
6. 제안 알고리즘(CMP, Contraction-based Message Passing): DFS로 bivalent check node 기반 연결요소를 찾아 대표 bit로 축약 → 대표 bit 중 N개(보통 1~2개) guess → 메시지 패싱 재개 → 이를 J회(보통 3회) 반복.
7. 부수 트릭: 원 parity-check 행렬에 길이 4/6 cycle의 check node 합으로 만든 auxiliary parity check row를 추가해 bivalent check node 수를 늘려 recovery 효율 향상.
8. 별도 학습/최적화 절차 없음(그래프 이론적 분석 및 알고리즘 설계).
9. 결과: 길이 1000 코드에서 기존 Algorithm C(g=6 guess) 대비 BER 10^-4에서 약 1자릿수(order of magnitude) 추가 개선, 표준 MP 대비 2~3자릿수 개선. 평균 실행시간은 표준 MP의 약 1.3~1.4배(J=3, N=2 파라미터).
10. 한계: BEC(erasure) 채널 전용 이론/시뮬레이션이며 AWGN/soft-decision 채널이나 HW 구현에 대한 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stopping-set 기반 후처리 복호(CMP, bit-guessing) | 대응 없음 | BEC 전용 stopping-set 개념으로 hard/soft-decision min-sum 디코더 구조와 매핑되지 않음 |
| 조기 실패(stopping set) 감지 후 재시도 아이디어 | `decoder.cpp` `LDPC_Decoder()` | 개념적으로 수렴 실패 시 후처리라는 점에서만 유사, 구체 알고리즘은 이식 불가 |
| bivalent(degree-2) check node 활용 | 대응 없음 | Prime ECC의 min-sum CN 연산과 무관한 그래프 조합론적 기법 |

> 적용 가치: 낮음 — BEC 채널 특유의 stopping-set/erasure 개념에 기반한 알고리즘으로 Prime ECC가 다루는 AWGN/RBER binary min-sum 디코더 구조에 개념적으로도 이식 경로가 뚜렷하지 않음.
