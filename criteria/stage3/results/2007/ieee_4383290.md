### B. 알고리즘 요약

1. 대상은 이진 규칙/불규칙 LDPC(및 RA 코드) 디코더로 예시는 (dv,dc)=(3,6) 규칙부호, N=2304~9216, R=0.5~2/3 다양한 기존 구현을 포괄한다.
2. 기존 문헌의 LDPC 디코더 아키텍처는 서로 비교·분류할 공통 프레임워크가 없어 설계자가 사양에 맞는 아키텍처를 고르기 어렵다는 문제를 다룬다.
3. 채널은 AWGN+BPSK, 복호는 belief propagation(BP)이며 스케줄은 flooding(FS/FS-P/FS-V), horizontal shuffle(HSS), vertical shuffle(VSS) 세 계열을 다룬다.
4. 핵심 기법은 node processor를 datapath(총합/trellis), interconnection 위치(4가지), update mode(grouped/straight spread/delayed spread)의 조합으로 정의하고, message-passing 전체는 병렬화 파라미터 `(P, α, β)`(체크/변수 프로세서 병렬 수, 처리 사이클)로 기술한다.
5. 처리능력 `Pc = d̄v·imax·D / (fclk·R)` [edges/cycle] 식으로 목표 throughput D, 최대 반복 imax, 코드율 R로부터 필요한 병렬 노드 프로세서 수를 산출한다.
6. VSS/HSS는 check-to-variable 메시지 `Em,n`을 부분 갱신(straight spread update)하여 반복당 수렴 속도를 높이는 확장이며, `Rm(i,J)` 재귀식(식 8-9)으로 부분합을 갱신한다.
7. 부수 디테일로 f(x)=-ln(tanh(|x|/2)) 기반 check-node 연산(Sum-Product), λ-min/Λ-min* 근사(참고문헌)로 실제 구현시 근사가 가능함을 언급한다.
8. 학습/최적화 절차는 없으며, 순수 아키텍처 파라미터 프레임워크(수식 유도 및 사례 분석)이다.
9. 결과: 프레임워크로 기존 3개 구현(Zhang&Parhi FPGA 54Mbit/s, Chen&Hocevar ASIC 376Mbit/s, Mansour ASIC 192Mbit/s)을 파라미터로 재기술하고, VSS용 신규 아키텍처(Fig.4)를 제안. 메모리 요구량은 HSS < VSS ≤ FS-P 순.
10. 한계: 신규 제안 아키텍처는 합성/실측 없이 개념 설계 및 메모리 비교(N+M+E vs N+E)만 제시하며 정정능력·throughput 수치는 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Shuffle schedule (HSS/VSS, straight spread update) | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 옵션) | Prime ECC의 Dual-Update(parity even/odd 교대)와 개념적으로 유사한 조기 갱신 스케줄, 세부 구현 비교 가치 있음 |
| check-node F 함수(tanh 기반 sum-product) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 min-sum 근사 사용, 논문의 exact 연산과 직접 대응은 아님 |
| 병렬화 파라미터(P, α, β)/interconnection 위치 | 디코더 HW 모델 (z=32, pipeline 6-stage) | Prime ECC의 고정 병렬구조(z_sb=32)와 개념 비교 가능하나 특정 architecture 이식은 아님 |
| 메모리 요구량 비교(N+E vs N+M+E) | 디코더 HW 모델 | 메모리/라우팅 설계 참고자료 수준 |

적용 가치: **중간** — 구체적 신기법이 아닌 기존 아키텍처 분류·비교 프레임워크이므로 직접 이식 대상은 아니지만, shuffle schedule 개념은 Prime ECC의 Dual-Update와 비교 참고 가치가 있음.
