### B. 알고리즘 요약

1. 목표 채널은 Binary Erasure Channel(BEC)이며, capacity-approaching LDPC 코드는 표준 랜덤 구성법에서는 error floor가 높다는 기존 결과([11],[12])가 문제.
2. 채널/모델 가정: BEC에서 복호 실패는 지워진 비트 집합이 stopping set을 포함할 때 발생 (`stopping set` = 모든 check node가 집합 내 variable node를 0개 또는 2개 이상 연결).
3. 핵심 기법: 작은 protograph `G0`(base graph)에서 시작해 `uniform random 2-lift`를 `M`번 반복 적용하여 최종 Tanner graph `G_M` 생성 (`G_k`는 `G_{k-1}`의 균등 랜덤 2-lift).
4. 핵심 식: `E[|Λ(G_{k+1},w)| | G_k] = Σ E[|B(S',w/w'-1)| | G_k]` — lift 후 작은 weight `w`의 stopping set 개수 기댓값을 base graph의 stopping set 분포로 재귀 표현.
5. 위 식의 의미: 결과 코드의 저weight stopping set은 대부분 base graph 내 "graph expansion 특성이 약한" stopping set에서 유래함을 규명 (Theorem 4.4의 지수 상한식으로 증명).
6. 부수 트릭: `Condition (A)` (β-vertex expansion, (η,γ)-uniform-edge-distribution, 평균 degree < d)를 만족하는 stopping set은 "strong expansion"으로 분류, lift 후에도 저weight stopping set을 잘 만들지 않음.
7. 설계 기준(파생 결과): protograph는 (a) 최소 stopping set weight가 커야 하고, (b) 약한 expansion 특성을 가진 저weight stopping set을 포함하지 않아야 함.
8. 최적화 절차: protograph는 exhaustive search로 구성(32 variable node, 16 check node), degree distribution은 density evolution([5])으로 근사(`λ(x)`, `ρ(x)` 다항식).
9. 결과: rate 0.5, length 8192 코드에서 제안 구성법이 표준 구성법 및 (3,6)-regular 코드 대비 error floor 영역에서 유의미하게 낮은 bit/block error probability 달성(capacity-approaching 유지).
10. 한계: BEC 채널 전용 이론/시뮬레이션이며, HW 설계나 AWGN/BSC 채널의 min-sum 디코더 적용 여부는 다루지 않음.
