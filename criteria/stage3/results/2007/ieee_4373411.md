### B. 알고리즘 요약

1. 배경: min-sum(MS)/sum-product(SP) 반복복호기는 finite-length LDPC의 Tanner graph에 사이클이 있을 때 최적(ML) 복호와 다르게 수렴하거나 실패할 수 있음 - 원인 규명이 핵심 과제.
2. 문제: 기존 `Koetter-Vontobel` 의 graph-covers-polytope 정의는 finite lift로 실현 가능한 pseudocodeword만 포착하고, MS 디코더의 실제 computation tree 상에서 발생하는 pseudocodeword(특히 nc-pseudocodeword)는 놓침.
3. 모델: `computation tree` `T(v,ℓ)`(임의 변수노드 `v`를 루트로 `ℓ`회 반복 전개) 상의 valid assignment로 codeword/pseudocodeword를 정의, BSC/BEC/AWGN 채널별 pseudocodeword 가중치 정의(`Definition 2.10`).
4. 핵심 기법: pseudocodeword를 `good`(모든 채널 조건에서 실제 codeword보다 cost 높음)과 `bad`(반대)로 분류(`Definition 2.8, 2.9`), bad pseudocodeword의 존재가 error-floor의 직접 원인.
5. 핵심 식: 정규 bipartite graph의 최소 pseudocodeword 가중치 하한 `Theorem 3.1`(girth `g`, 최소 좌측차수 기반) - 최소거리·최소 stopping set 크기와 동일한 형태의 하한 성립.
6. 확장: irreducible pseudocodeword의 최대 성분값 `t-value`(`Definition 3.2`) 정의, 이를 실현하는 최소 lift 차수 상한 제시(`Theorem 3.6`) - 그래프 구조만으로 pseudocodeword 분포 예측 가능.
7. 구현 디테일: `[7,4,3]` 해밍코드의 세 가지 Tanner graph 표현(A/B/C, redundant check node 개수 차이)을 비교 - 동일 코드라도 표현에 따라 pseudocodeword 가중치 분포가 크게 달라짐(`Theorem 6.1`: redundant check 추가는 lift-realizable pseudocodeword 개수를 줄임).
8. 최적화 절차: 명시적 알고리즘(GA/NN 등)은 없고, redundant parity check 추가를 통한 그래프 재설계가 유일한 개선 수단으로 제시됨.
9. 결과: representation B(order-1 redundant check 포함)는 BSC에서 MS 디코딩이 ML과 완전히 동일 성능 도달, representation A는 detected error(수렴 실패) 다수 발생 - 동일 코드라도 그래프 표현만으로 성능 차이 큼.
10. 한계: 길이 4~273의 소형 예제(반복코드, 해밍코드, 유한기하 코드)에 국한된 분석이며 실제 NAND급 대형 QC-LDPC에서의 redundant check 설계 절차·HW 비용은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-sum 디코더의 error-floor(수렴 실패) 원인 분석 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 min-sum 반복 루프에서 error-floor가 발생하는 근본 원인(pseudocodeword/trapping-set류 구조)과 직결 |
| H-matrix에 redundant check row 추가로 pseudocodeword 억제 | `ecc_top.cpp` `Load_PCM()` | 부호설계(H-matrix) 변경으로 error-floor를 개선하려는 접근 - 부호구조 재설계 영역, 이식 난이도 높음(§4 기준) |
| Tanner graph 표현/차수 구조 설계 | `ecc_data.h` (PCM 구조체) | 부호구조 설계 파라미터에 대응되나 실제 QC-LDPC 특유의 base+circulant 구조엔 직접 적용 어려움 |

적용 가치: **중간** — min-sum decoder의 error-floor 원인(pseudocodeword)을 이론적으로 이해하는 데 유용하나, 개선 수단(redundant check 추가)은 부호설계(H-matrix) 변경을 요구해 기존 QC-LDPC 고정 구조 재검증 부담이 큼.
