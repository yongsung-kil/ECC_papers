> 총평: OMS(offset min-sum)의 value-reuse 특성을 활용한 layered decoding(TDMP) 아키텍처로 메모리·배선·throughput 전방위 개선을 ASIC 합성까지 검증한 강력한 HW 이식 후보.

### B. 알고리즘 요약

1. 대상은 IEEE 802.11n block LDPC(QC-LDPC 유사, base matrix Nb=24, expansion factor z=27/54/81), 코드길이 648/1296/1944, rate 1/2·2/3·3/4·5/6.
2. 문제의식: 기존 TPMP(two-phase message passing) 기반 아키텍처는 배선 복잡도·메모리 사용량이 크고, TDMP(layered decoding)도 R메시지 전체 저장 시 메모리 낭비가 큼.
3. 채널/코드 모델: OMS(offset min-sum) 알고리즘 `Rmn = δmn·max(κmn − β, 0)`, β는 density evolution으로 산출.
4. 핵심 기법 1: OMS의 value-reuse 특성 — check node당 `Rmn`이 실제로 3개 값(±M1, ±M2, 부호)만 가짐을 이용해 serial CNU가 M1/M2/index/sign(compact "final state")만 저장.
5. 핵심 식: TDMP 서브 반복 `Q^l_{i,n} = P_n^{S(l,n)} − R^{l,i-1}_n`, `R_{l,n}=f(Q^l_{i,n})`, `P_n^{S(l,n)}` — 레이어(블록로우) 단위로 즉시 variable 갱신해 반복수 절감.
6. 핵심 기법 2: block-serial 스케줄링 + out-of-order R selection — 체크노드 degree가 레이어마다 7~21로 크게 달라 순서 재배열로 파이프라인 효율화, data-forwarding으로 불규칙성 처리.
7. 구현 디테일: 단일 M×M cyclic log-shifter(log2(M)단 mux)로 27/54/81 전체 지원, dual-access Q memory 뱅킹, FS memory word 20bit({M1,-M1,±M2,index}), PS memory 16bit.
8. 최적화 절차 없음(설계/스케줄링 기반, 학습 없음). 양자화: Q/R 5-bit uniform, P 6-bit uniform, 0.1dB 이내 손실.
9. 결과: TPMP 대비 메모리 63% 절감(SP/min-sum 대비), 다른 TDMP(BCJR/SP) 대비 55% 절감, 라우터 50% 감소, throughput 2배(반복수 절반). ASIC(0.13um) 99900 gates, 1.618Gbps@z=81, FPGA(Xilinx 2V8000) throughput 119~356Mbps.
10. 한계: 802.11n 표준 특정 base matrix에 최적화(범용 QC-LDPC로 일반화 시 재작업 필요), 정정능력 자체 개선 없음(HW 효율/throughput이 목적), 802.16e/802.22는 "minor changes"로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| OMS value-reuse serial CNU (M1/M2/index/sign 압축) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 기반 CNU 구조와 직접 대응, offset 방식 비교 가능 |
| TDMP(layered decoding) 레이어별 즉시 갱신 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 (Dual-Update 옵션과 유사 개념) | 기존 Dual-Update와 스케줄링 방식 비교 검토 대상 |
| 단일 cyclic log-shifter (27/54/81 통합) | `decoder.h` 관련 병렬 처리 구조 (z_sb=32 row 병렬) | Prime ECC는 z=32 고정, 논문의 multi-rate shifter 개념은 구조 차이로 직접 대응 어려움 |
| Q/R/P 균일 양자화(5/6-bit) 및 offset β | `ecc_data.h` `PARAM_LLR`, `channel.cpp` `Set_LLR_Th()` | 양자화 비트폭·offset 파라미터 튜닝 참고 가능 |
| out-of-order R selection / data-forwarding | `decoder.cpp` `LDPC_Decoder()` 파이프라인 스케줄 | HW 스케줄링 최적화 아이디어로 참고 가능, 직접 대응 모듈 없음 |

적용 가치: **높음** — offset min-sum + layered decoding 조합은 이미 Prime ECC가 채택한 min-sum/Dual-Update와 계열이 같아 메모리·배선 절감 기법을 구체적으로 비교·이식 검토할 가치가 큼. ASIC 게이트카운트/throughput 수치까지 확보되어 벤치마크 참고자료로 유용.
