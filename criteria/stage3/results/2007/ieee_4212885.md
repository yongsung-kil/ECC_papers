### B. 알고리즘 요약

1. 채널은 AWGN, BP 복호(최대 200 iteration), 대상은 irregular LDPC, block length 2000/3000, rate 1/2.
2. 기존 PEG/IPEG는 후보 check node가 여러 개일 때 ACE(근사 cycle EMD) 기준까지만 쓰고 동률이면 무작위로 선택해 stopping set 축소가 불충분.
3. Stopping set은 모든 인접 check node가 최소 2번 연결된 variable node 부분집합으로 정의되며, EMD(0)는 stopping set의 필요조건(`EMD=0`).
4. 제안 기법은 ACE 기준으로 동률인 candidate check node 집합 Φ에 대해, 각 후보를 root로 하는 tree-expanded subgraph의 column sum 1의 개수(EMD)를 계산해 EMD가 더 큰 check node를 선택.
5. EMD가 클수록 해당 variable node 집합이 stopping set이 될 가능성(=`EMD=0`)에서 멀어져 작은 stopping set 수가 줄어듦.
6. EMD까지 동률이면 subgraph의 variable node 수가 더 많은 쪽을 선택하고, 그래도 안 되면 무작위 선택(PEG/IPEG의 fallback 유지).
7. 부수 디테일 없음 — 순수 그래프 구성 알고리즘, 양자화/HW 변형 없음.
8. 최적화 절차: 표준 PEG 트리 확장 + degree distribution(λ1, λ2)은 [Richardson-Shokrollahi-Urbanke] 사전 설계값 사용, 알고리즘 자체는 결정론적 선택 규칙.
9. 결과: length 3000 코드 30개 평균 최소거리 IPEG 18.03 → 제안 19.93(최대 24→29), length 3000에서 error floor가 IPEG 및 Richter 구성법보다 낮음.
10. 한계: 시뮬레이션(AWGN, BP)만 수행, HW 설계 없음, block length 2000/3000의 소규모 예제로만 검증, QC-LDPC 제약 하 실험 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IPEG 개선 구성법(H-matrix 생성) | `ecc_top.cpp` `Load_PCM()` | H-matrix를 논문 알고리즘으로 새로 생성해야 하므로 재검증 부담 큰 대응 |
| EMD/ACE 기반 stopping set 회피 | 대응 없음 | Prime ECC는 QC-LDPC 고정 구조(base matrix+circulant)라 PEG류 무작위 구성 개념 자체가 없음 |
| BP 복호(성능 검증용) | `decoder.cpp` `LDPC_Decoder()` | 논문은 min-sum이 아닌 sum-product(BP) 사용, 검증 목적일 뿐 이식 대상 아님 |

> 적용 가치: 낮음 — 순수 code-design(H-matrix 구성) 논문으로 QC-LDPC 고정 구조인 Prime ECC와 정합이 어렵고, 부호설계 이식은 프로파일상 난이도가 가장 높은 영역임.
