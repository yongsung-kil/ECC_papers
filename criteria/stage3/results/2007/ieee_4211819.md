### B. 알고리즘 요약

1. 대상은 중국 지상파 DVB 표준(CDVB-T)의 QC-LDPC 코드: circulant 크기 `z=127`(메르센 소수), rate 0.4/0.6/0.8, codeword length 7493bit(=59×127), single-diagonal per circulant 구조.
2. 문제: circulant 크기가 소수이면 기존 sub-group parallel 아키텍처([4])는 병렬화 인자 D가 z의 약수여야 하므로 D=127(전체 병렬)만 가능해, 필요 throughput(45Mbps) 대비 실제 처리능력이 6.6배 과설계되고 λ-메모리 효율(23%)이 낮아 면적 낭비(3mm²@90nm)가 발생.
3. 채널/디코딩 알고리즘은 min-sum 기반이며 Gauss-Seidel(layered/shuffled/staggered) 방식 반복을 사용, check-node centric 처리로 `λ_n^i(m) = L_n + Σ_{U(n,m)}Λ + Σ_{R(n,m)}Λ`(식 5)를 갱신.
4. 핵심 기법 1단: circulant를 r개 sub-circulant로 나눠 D=⌈z/r⌉ data path만 사용하는 sub-group parallelism을 일반화하여, D가 z의 약수가 아니어도(fractional divider) 동작하게 하는 메모리 매핑 방법 제안.
5. 핵심 식/원리: 연속 sub-circulant가 회전각과 무관하게 메모리 한 행(row)을 공유한다는 성질(`consecutive rows modulo ⌈z/D⌉`)을 이용 — 이전 sub-circulant 처리 시 읽은 행을 FIFO에 저장해 다음 sub-circulant가 재사용, 새로 필요한 행 1개만 추가로 읽어 throughput 효율 `r/(r+1)`을 달성.
6. 핵심 기법 2단: 경계 조건에서 발생하는 "4-row-conflict"(m=z mod r, 충돌 각도 개수 `(m-1)(r-m-1)`)를 분석하고, 무충돌 조건(`m=r-1`, 즉 `r·D=z+1`)을 만족하는 D 후보(예: D=32, D=26)를 도출하거나 컴파일타임 스케줄링으로 우회.
7. 부수 트릭: 그룹 간 초기화 겹침(다음 그룹 초기화가 이전 그룹 출력 완료 전에 시작)으로 인한 메모리 충돌도 조기/후기 스케줄링으로 회피 가능함을 CDVB-T rate 0.8 사례(27개 중 15개 중첩)로 분석.
8. 별도 학습/최적화 절차 없음 — 모든 회전각(angle) 조합을 시뮬레이션해 충돌 패턴을 도출하는 설계공간 탐색.
9. 결과: D=127(전체병렬) 3.00mm² 대비 D=32는 1.12mm², D=26은 1.03mm²(충돌 없음 확인, 스케줄링으로 해결)로 약 3배 면적 절감(90nm 기준).
10. 한계: single-diagonal per circulant 구조에만 적용(DVB-S2처럼 circulant당 다중 diagonal이면 미적용), throughput 수치 및 정정성능은 다루지 않고 순수 면적/메모리 매핑 최적화에 국한되며 실제 throughput 실측치는 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| circulant/sub-circulant 병렬화 인자 선택 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z_sb=32 고정 row 단위 병렬이라 fractional divider 개념 자체는 직접 필요하지 않으나, 소수 크기 lifting을 쓰는 변형 부호 설계 시 참고 가능 |
| λ-메모리/Λ-메모리 매핑, barrel shifter 기반 permutation | 디코더 HW 모델 (메모리 라우팅) | 메모리 행 공유·FIFO 재사용 아이디어는 Prime ECC의 메모리 라우팅 최적화에 참고 가능(대응 코드 파일 미특정) |
| 4-row-conflict 스케줄링 회피 | `GT.cpp` `Make_GT_HW()` | 엣지/메모리 충돌 회피라는 목적은 유사하나 구체 기법(Graph Thinning MUX)과는 결이 다름 |
| min-sum + Gauss-Seidel(layered) 반복 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC의 Dual-Update 스케줄과 유사 계열(이미 보유, 중복) |

적용 가치: **중간** — 부호 자체(CDVB-T)는 NAND와 무관하지만, circulant 기반 QC-LDPC HW의 메모리/병렬화 면적 최적화 기법은 Prime ECC의 lifting/메모리 구조 재설계 시 참고할 만함.
