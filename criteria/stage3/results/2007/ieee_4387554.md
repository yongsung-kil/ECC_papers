### B. 알고리즘 요약

1. 대상은 (1024,512) regular LDPC 코드(dv=3, dc=6), AWGN+BPSK 채널, FPGA(Xilinx Virtex-4 XC4VLX200) 완전병렬 구현.
2. 문제: 완전병렬 LDPC 디코더는 인터리버 라우팅 혼잡이 심해 긴 실용 코드에서 구현이 어려움 — min-sum도 라우팅 문제를 완전히 해결하지 못함.
3. 핵심 가정: 확률을 Bernoulli 확률적 비트스트림({a_i})으로 표현하는 stochastic computation 모델을 LDPC 팩터그래프에 적용.
4. 핵심 기법: variable node는 `Pc = PaPb / (PaPb + (1-Pa)(1-Pb))`(식 1), check node(dc=3)는 `Pc = Pa(1-Pb) + Pb(1-Pa)`(식 2)로 단순 논리게이트(AND 등)로 구현.
5. 의미: 확률연산이 비트단위 논리연산으로 축소되어 노드당 복잡도와 엣지당 배선폭(1비트)이 극소화되어 라우팅 혼잡을 완화.
6. 확장 기법: Noise-Dependent-Scaling(NDS, `L'_i=(4α/Y)y_i`)으로 스위칭 활동 증대, Edge Memory(EM, M=64 shift register)로 hold-state 시 랜덤 재생성 비트를 사용해 latching(lock-up) 문제 해결.
7. HW 디테일: EM은 Virtex-4 4-input LUT를 SRL로 캐스케이드 구현(4개로 64bit), 6bit saturating up/down counter로 최종 hard-decision, LFSR 10개 기반 랜덤화 엔진으로 난수 공유.
8. 별도 학습/최적화 절차 없음(파라미터 α=3, Y=6, M=64는 선행연구[20] 기준 채택).
9. 결과: 32875/89088 slice(36%) 사용, 212MHz, BER 10^-6에서 SPA(32회 반복) 대비 0.1dB 손실, 평균 300 DC로 706Mbps 처리량; 기존 근사 min-sum FPGA 디코더 대비 면적 약 53% 절감.
10. 한계: FPGA 검증뿐(ASIC 미검증), 코드 길이가 (1024,512)로 제한적, stochastic 방식 특유의 latching/switching 활동 문제로 高SNR에서 별도 대책(NDS, EM) 필수, 정정성능 개선이 아닌 순수 HW 구현 효율화 논문.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stochastic variable/check node 연산(확률 비트스트림) | 대응 없음 | Prime ECC는 LLR 크기값 기반 min-sum이라 확률 비트스트림 인코딩 자체가 없음 |
| edge memory(latching 방지) | 대응 없음 | Prime ECC의 결정론적 메시지패싱에는 latching 문제가 존재하지 않아 개념 자체가 불필요 |
| 완전병렬 인터리버/라우팅 최적화 | `decoder.cpp` `LDPC_Decoder()` (구조적 참고만) | 라우팅 혼잡 완화 아이디어는 일반적이나 Prime ECC는 이미 z=32 GT/HCU 등 자체 방식으로 해결 |
| up/down counter 기반 hard-decision | 대응 없음 | Prime ECC는 LLR sign 기반 HD/2SD/3SD로 전혀 다른 메커니즘 |

> 적용 가치: 낮음 — stochastic computation은 min-sum/sum-product와 근본적으로 다른 디코딩 패러다임으로, Prime ECC의 결정론적 LLR 기반 HW 구조와 공유 가능한 모듈이 없음.
