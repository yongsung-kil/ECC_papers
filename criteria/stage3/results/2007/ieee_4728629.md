### B. 알고리즘 요약

1. (n, {k1, k2}, 1) block design은 G. C. Yang의 optical orthogonal code 구성법을 LDPC에 적용, 두 종류 base block(크기 k1=4, k2=3)을 결합.
2. n을 `n=18t+1` 형태 소수로 두고 GF(n)의 원시원소 `a`로 base block `B_i`(크기4), `T_i`(크기3)를 생성(식 4, 5).
3. 각 base block의 incidence vector로 circulant `Q_k1`, `Q_k2`를 만들고, 이를 나열해 parity check matrix `H_{l,m} = [Q_k1(1) ... Q_k1(l) Q_k2(1) ... Q_k2(m)]` 구성(식 8).
4. `H_{l,m}`의 null space가 길이 `N=(l+m)n`, rate `≈(l+m-1)/(l+m)`인 LDPC 부호가 되며, k1만 쓰면 regular, k1/k2를 섞으면 irregular 부호.
5. BIBD의 index `λ=1` 성질로 임의 두 열의 공통 1 개수가 최대 1개 → Tanner graph에 4-cycle 없음(girth≥6) 보장.
6. Halford-Chugg 알고리즘으로 girth g, g+2, g+4 길이의 short cycle 수(N6, N8, N10)를 세어 코드 품질을 정량 분석.
7. 실험: rate 1/2, N=362 코드(code1: col weight3/girth6, code2: col weight4)와 rate 3/4, N=724 코드(code3/4/5) 비교.
8. AWGN 채널, sum-product 반복복호(최대 50회)로 BER 시뮬레이션 수행(별도 부호화/복호 최적화 없음).
9. 결과: code1이 code2보다 BER 10^-3에서 0.35dB 우수(short cycle 적음), code5(irregular)가 code3(regular, short cycle 적음에도)보다 0.2dB 우수 — cycle 규칙성이 개수보다 중요함을 시사.
10. 한계: 시뮬뿐, HW 미설계, 특정 예제 block length/rate 조합만 제시, NAND 채널·soft-read 고려 없음.
