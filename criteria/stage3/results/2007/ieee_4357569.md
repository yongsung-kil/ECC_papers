### B. 알고리즘 요약

1. 시스템: rate `R=b/c` binary LDPC convolutional code와 동일 protograph 기반 LDPC block code를 비교, ARA/AR4JA protograph(rate 1/2) 사용.
2. 문제: LDPC convolutional code가 block code 대비 성능 이점(convolutional gain)이 있다고 알려져 있으나, 이 이점이 메모리/latency/복잡도 비용 대비 정당한지 불명확.
3. 핵심 가정: 두 코드가 동일 protograph kernel(AR4JA)에서 유도되어 동일 차수분포를 가지므로 공정 비교 가능.
4. 핵심 기법: block code의 parity-check 행렬 `H`를 대각선 방향으로 크기 `(c-b)×c` 단위로 자르고(cutting), 상단 대각 부분을 하단에 붙여(pasting) 무한 반복하는 "unwrapping" 절차로 시간가변 LDPC convolutional code `H_conv` 유도.
5. 핵심 식: convolutional 코드는 syndrome former memory `m_s`, 전체 constraint length `v_s=(m_s+1)c`를 가지며, 이 유한 폭 덕분에 sliding-window pipeline 디코더(`I_C`개 독립 프로세서 병렬)로 연속 복호 가능.
6. 확장/비교: block code는 `I_B` 반복에 대해 복잡도 `∝(1-R)·C_check + C_var)·I_B`, convolutional은 `I_C` 반복에 대해 동일 식 형태이나 관측/엣지 메모리는 각각 `n`, `I_C·v_s`로 구조가 다름.
7. 부수 트릭: convolutional 코드는 동일 protograph를 반복(`I_C`배) 확장하는 것만으로 추가 rate 손실 없이 약 1dB 성능 이득(convolutional gain) 획득.
8. 실험: AR4JAx4x128/x125로 block code 유도 후 unwrapping, `I_C=100` 반복 pipeline 복호와 `I_B≤200`(syndrome 기반 조기종료) block 복호를 BER=1e-7 기준 SNR에서 비교.
9. 결과: 동일 BER 성능 기준으로 block code가 관측/엣지 메모리 6배 적고, 연산량 2.57배 적고(반복수 `I_B=36.3` vs `I_C=100`), latency 최대 2배 적음 - convolutional의 성능 이득이 비용을 상쇄하지 못함.
10. 한계: 위성/JPL 우주통신 지향 protograph(AR4JA)이며 HW(FPGA/ASIC) 구현은 "현재 착수 중"이라 미완료, 저자 스스로도 실용성은 block code가 우위라 결론.
