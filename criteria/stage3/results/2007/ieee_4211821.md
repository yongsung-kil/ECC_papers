### B. 알고리즘 요약

1. 시스템: IEEE P802.16e/D7 표준 irregular LDPC, code length 576~2304, rate 1/2·2/3A·2/3B·3/4A·3/4B 5종 지원, WiMax 응용.
2. 문제: 기존 설계는 H matrix를 오프라인 계산해 저장하므로 단일 code length/rate만 지원 — 실시간 적응(real-time adaptive) 불가.
3. 핵심 구조: model matrix(mb×nb, 최대 24열)를 저장 후 spreading factor `z`로 확장해 base matrix, 이후 child matrix A/B/C/E/T를 실시간 생성.
4. 인코딩 식: `F=C*u^T`, `G=A*u^T`, `D=E*T^-1*G^T`, `P1=D+F`(VA), `M=B*P1^T`, `N=M+G`, forward substitution으로 `P2=T^-1*N^T`, 최종 codeword `c=[u P1 P2]`.
5. 메모리 절감: child matrix를 dense 저장 대신 1의 인덱스만 저장(기존 논문[8] 기법 재사용)해 316KB→16KB 약 20배 절감.
6. 파이프라인: 4-stage 파이프라인 인코더로 약 2.5배 속도 향상(메모리 오버헤드 증가와 트레이드오프).
7. HW 플랫폼: Reconfigurable Instruction Cell Architecture(RICA, coarse-grained 재구성형 fabric, 32-bit 연산셀, 8 R/W 메모리 인터페이스)에 ANSI-C로 구현.
8. 최적화 기법: 코드율별 전용 함수 분리(if문 제거), Vector_Add loop unrolling(4배), 조건분기를 멀티플렉싱으로 대체(jump cell 제거), intra/inter-module HW 파이프라이닝.
9. 결과: 최적화 전 3.5Mbps → 최적화 후 10~19Mbps(rate별) → HW 파이프라이닝 후 26~47Mbps → 멀티코어(64-cell fabric) 파이프라인 시 78Mbps.
10. 한계: 재구성형 프로세서(RICA) 아키텍처 특화 소프트웨어/명령어 수준 최적화이며, LDPC 부호설계나 디코딩 알고리즘 자체의 신규 기여는 없음(표준 인코딩 절차 그대로 구현).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| forward substitution 기반 dual-diagonal 유사 인코딩(A/B/C/E/T child matrix) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 표준 802.16e 인코딩 구조로 Prime ECC의 dual-diagonal 인코더와 개념적으로 유사하나 이미 보유한 기법, 신규성 없음 |
| H-matrix 실시간 생성(model matrix → base matrix → child matrix) | `ecc_top.cpp` `Load_PCM()` | 코드율/길이별 동적 재구성 개념이나 Prime ECC는 고정 QC-LDPC PCM 사용, 이식 난이도 높음 |
| RICA 재구성형 아키텍처 최적화(loop unrolling, jump-to-mux) | 대응 없음 | 범용 재구성형 프로세서 대상 소프트웨어 최적화로 Prime ECC HW RTL 시뮬레이터와 무관 |

> 적용 가치: 낮음 — WiMax 표준 인코더의 재구성형 프로세서 구현/처리량 최적화이며, 부호설계나 디코더 알고리즘 개선이 없고 대상 플랫폼도 Prime ECC의 고정 HW 구조와 근본적으로 다름.
