### B. 알고리즘 요약

1. 대상은 자기기록(magnetic recording) 등 area-constrained 응용의 regular LDPC, 대표 예시로 rate 8/9(dc=3, dr=27) 코드를 사용.
2. 문제: fully-parallel 구조는 배선 복잡도가 과도하고(예: 53.5mm² 중 절반이 배선), 기존 serial 구조(check-node centric, Yeo/Hocevar 방식)는 고rate 코드에서 메모리 요구량이 큼.
3. 표준 sum-product 알고리즘(`Q_ij`, `R_ji`, `φ(x)=-log tanh(x/2)`)을 그대로 사용하되 근사(min-sum) 없이 최적 성능을 유지.
4. 핵심 기법: check node 계산(식 2,3)을 `ρ=φ(|Q|)`, `σ=sign(Q)`의 running sum `P_j`, running product `S_j`로 분해해 bit node 처리와 동시에(on-the-fly) 갱신 (`type-1 update processing unit, UPU1`).
5. 핵심 식 `|R_ji| = φ(P_j - ρ_ij)`, `sign(R_ji) = S_j × σ_ij` (식 9,10) — 전체 체크노드 합을 미리 누적해두면 개별 엣지 메시지를 감산만으로 즉시 복원 가능해 별도 체크노드 프로세싱 유닛이 불필요.
6. 확장: 검출기(SISO detector)와 결합된 concatenated 시스템에서는 global iteration당 local iteration 1회만 수행한다는 가정 하에 FIFO 버퍼 크기를 `dc`배 축소(식 15-19).
7. 구현 디테일: check node 연산은 sign-magnitude 포맷, bit node 연산은 2's-complement 포맷 사용, 포맷 변환 로직 포함.
8. 별도 학습/최적화 절차(DE, GA 등)는 없음 - 순수 아키텍처 설계.
9. 결과: 메모리 요구량 `(2m+ndc)q` bit로 check-node centric의 `(2n+mdr)q` 대비 rate 8/9, dc=3 예시에서 29mq vs 45mq(약 50% 절감), concatenated 시스템에서는 추가 지연 없이 SISO 검출기 지연에 흡수됨.
10. 한계: 실제 ASIC/FPGA 합성 결과나 게이트 수·throughput 수치 없이 메모리 비트수·클록사이클 수의 이론적 비교만 제시, 정정능력(BER)은 표준 sum-product와 동일(기존 min-sum 근사 대비 우위만 언급, 정량 비교 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-Product 알고리즘(표준, φ 함수 기반) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 Min-Sum 근사 사용 중이라 표준 Sum-Product 자체는 성능은 높지만 이미 대체된 기법 |
| Check node running sum/product (P_j, S_j 누적) 기반 on-the-fly 메시지 계산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 메모리 절감 관점의 CN 업데이트 스케줄링 아이디어로 대응 가능성 있음 |
| Bit-node centric 메모리 절감 구조(고rate 코드에 유리) | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | 고rate QC-LDPC(Prime ECC도 고rate)에서 메모리 절감 아이디어 참고 가능 |
| Concatenated detector-decoder 지연 흡수 구조 | 대응 없음 | Prime ECC는 SISO 검출기 결합 구조 없음(NAND soft-read 채널 전용) |

> 적용 가치: 중간 — 알고리즘 자체(표준 sum-product)는 Prime ECC의 min-sum과 다르고 중복 기여는 아니지만, check node를 on-the-fly로 처리해 메모리를 절감하는 스케줄링/메모리 배치 아이디어는 고rate QC-LDPC 디코더의 HW 메모리 최적화에 참고할 가치가 있음.
