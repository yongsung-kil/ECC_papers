### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 12.4 Gb/s 광통신 OOK 채널, 3-bit soft-decision + Block Turbo Code(BTC) LSI, FEC 23.6% redundancy.
2. 문제: OSNR/색분산/편광모드분산/비선형성 변화로 최적 soft-decision 임계값(D1~D7)이 계속 변하여 성능 저하.
3. 채널 모델: 수신 신호(mark/space)를 비대칭 Gaussian 분포로 근사, LLR을 `LLR = ln(p(x|1)/p(x|0))`로 정의(식 1).
4. 핵심 기법: 4단계 임계값 자동 추적 - (1) hard-decision 임계값 D4를 mark/space 개수 균등화로 조정, (2) reference decider가 임계값을 스윕해 µ1(mark), µ0(space) 평균값 추정, (3) 나머지 6개 soft 임계값을 사전 계산값으로 산출, (4) 진폭 변화 감지 시 재계산.
5. 3-bit encoder의 truth table에서 "impossible output"(노이즈로 인한 비정상 패턴)을 정의하고, 이를 최확값으로 자동 정정하는 error filter를 encoder에 내장(표 1).
6. Monte Carlo 시뮬레이션 결과, error filter 적용 시 정정성능이 최소 0.5 dB 개선.
7. 실험 구성: 231-1 PRBS 신호를 9.95 Gb/s로 BTC 인코딩 후 12.4 Gb/s OOK 변조, EDFA로 AWGN 열화, 가변 분산발생기로 색분산 에뮬레이션.
8. 결과: 임계값 오프셋 수 mV 이내에서 BER 10-9 유지(displacement tolerance, Fig. 4).
9. LDPC 언급: PG(2,26)-LDPC(4161,3431) 코드가 NCG 11.27 dB 가능하다는 선행연구를 인용하며, 자체적으로 제안한 cyclic approximated δ-minimum 알고리즘으로 3-bit soft-decision만으로 Q=6.2 dB @ BER 10-6 확보 가능하다고 언급(참고문헌 [5] 인용, 본문 상세 없음).
10. 한계: 본 논문의 실험/회로 기여는 BTC 기반이며, LDPC 부분은 결론에서 향후 적용 가능성만 서술하고 별도 알고리즘·수치 검증은 제시하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 3-bit soft-decision 임계값 자동 추적 | `channel.cpp` `Set_LLR_Th()` | 소프트 임계값 개념은 유사하나 대상이 BTC이며 광통신 아날로그 임계값 추적 회로로 직접 이식 불가 |
| error filter (impossible output 정정) | 대응 없음 | Prime ECC는 QC-LDPC decoder 기반이며 BTC encoder의 3-bit truth table 정정 로직과 대응되는 모듈 없음 |
| LDPC 관련 언급(PG-LDPC, δ-minimum) | 대응 없음 | 본문에 알고리즘 상세가 없어 핀포인트 불가 |

> 적용 가치: 낮음 — 논문의 핵심 기여(BTC 임계값 트래킹, error filter)가 대상 코드(BTC)와 도메인(광통신 아날로그 회로)에서 Prime ECC의 binary QC-LDPC 디코더 구조와 접점이 없고, LDPC 관련 서술은 참고문헌 인용 수준에 그침.
