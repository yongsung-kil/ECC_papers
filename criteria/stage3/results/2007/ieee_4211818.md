### B. 알고리즘 요약

1. 대상 표준은 DVB-S2(길이 64800, rate 1/4~9/10), WiMax 802.16e(576~2304bit, rate 1/2~5/6), WiFi 802.11n(648~1944bit, rate 1/2~5/6)의 구조화된(accumulator-based, permuted identity submatrix) irregular QC 계열 LDPC 코드.
2. 문제: 표준마다 code length/rate/node degree가 크게 달라 완전 호환 디코더는 면적이 과도해지고, 특히 WiFi처럼 지연 예산(6μs 이하)이 빡빡한 경우 처리속도가 관건이다.
3. 두 가지 스케줄: 전통적 flooding(2-phase) vs layered(horizontal shuffle) 디코딩을 비교.
4. 핵심 기법 1단(layered decoding): CN 업데이트마다 SO 추정치 `y_n^(q)[k]`를 즉시 갱신·전파해 반복시간을 최대 50% 절감 — `y_n^(q)[k] = y_n^(q)[k-1] - μ_{m,n}^(q-1) + μ_{m,n}^(q)`.
5. CN 처리에서 tanh 기반 정확 연산 `Φ(x)=-log(tanh(x/2))` 대신 근사 연산자 `g(a,b)=min(a,b)-log(...)`를 사용, M-min*(λ개 최소 reliability 메시지만 정밀 계산)로 수치안정성과 복잡도를 동시에 개선.
6. 핵심 기법 2단: Adaptive Single Phase Decoding(ASPD) — VN마다 leakage 포함 러닝섬 단일 메트릭을 유지해 메시지 메모리를 Tanner graph edge 수가 아닌 codeword 길이에 비례하게 축소(메모리 60~80% 절감, IL 0.2~0.3dB).
7. HW 구현 디테일: 두 아키텍처(2-phase 방식은 sum RAM 2개 교대 사용, layered 방식은 채널RAM에 a-posteriori 값 저장해 permutation network 1개로 축소) 모두 serial 노드 프로세서 기반 partly-parallel 구조, barrel shifter로 permutation 구현, 6bit quantization.
8. 응용특화 코드 설계: OFDM-UWB용 Ultra-Sparse LDPC(rate 3/4, 9600bit)를 2V-PEG(edge growth) 기법으로 설계해 VN/CN degree를 최소화(26400 edges vs 통상 35000+), layered decoding과 결합.
9. 결과: DVB-S2 디코더 3.861mm²@65nm(400MHz)로 net throughput 60~708Mbps, WiMax 1.337mm²로 48~333Mbps, WiFi 1.023mm²로 54~281Mbps; Ultra-Sparse UWB 코드는 0.504mm²@500MHz로 net 1.08Gbps(≤10 iteration).
10. 한계: 표준 코드 자체가 무선/방송 특화라 NAND 고rate 코드에 직접 대응 안 되며, 6bit soft quantization 고정이라 hard-1bit/저bit 이식성은 별도 검증 필요, error-floor 등 정정성능 개선은 다루지 않음(순수 구현 복잡도/throughput 관점).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered decoding(horizontal shuffle) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Dual-Update(odd/even 교대) 스케줄 보유, layered 방식과 비교 시 반복시간 단축 아이디어 참고 가능 |
| CN min-sum 근사(M-min*, λ-min) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 이미 Min-Sum 보유(§3), M-min*의 보정항은 개선 여지 있으나 중복도 높음 |
| serial 노드 프로세서(가변 degree 지원) | `decoder.cpp` `VN_Cal_HD()` 등 | 이미 특정 QC 구조(z=32, degree 17) 고정이라 유연 degree 서포트는 재설계 부담 큼 |
| 2V-PEG 부호구성(Ultra-Sparse) | `ecc_top.cpp` `Load_PCM()` | 이식 난이도 높음(§4의 부호설계 항목), 참고용 |
| ASPD(단일 메트릭 메모리 절감) | 대응 없음 | Prime ECC는 표준 message-passing 메모리 구조, ASPD식 메모리 절감 기법은 대응 모듈 없음 |

적용 가치: **중간** — 무선/방송 표준 대상이라 부호 자체는 이식 곤란하지만 layered scheduling·node approximation 등 디코더 구현 기법은 참고 가치 있음.
