### B. 알고리즘 요약

1. 시스템: 자기기록 read channel, SOVA 소프트출력 트렐리스 검출기 + 길이-8136 rate-8/9 regular-(4,36) QC-LDPC, PR-shaping 등화 타깃 + AWGN 채널.
2. 문제: 기존 방식은 트렐리스 검출기 출력의 dominant error event 검출에 별도 parity code를 삽입해 code rate penalty가 발생하고, k-constraint(RLL) 강제도 별도 코드/역결합 구조가 필요.
3. 핵심 가정: LDPC parity check matrix가 행/열 치환으로 "각 행이 겹치지 않는 all-1's 서브벡터이고 전체 열을 커버하는" 행-부분행렬 구조를 갖는다는 약한 제약을 만족하면(QC-LDPC circulant 구조에서 자연히 성립), LDPC 코드워드 내에 명시적 single-parity 서브코드가 존재.
4. 핵심 기법 1단: 이 내재 single-parity 구조(예: 36비트 단위)를 이용해 하드 결정 출력에서 parity violation 발생 시 error event correlation으로 가장 유력한 dominant error event를 사전목록에서 검색(식 1) 후 해당 soft output 크기를 0으로 소거(erasure).
5. 핵심 식 (1): `argmax` 형태 event 목록 중 하드결정·채널함수와 가장 잘 부합하는 error event를 고르는 상관 검색 — 부가 parity bit 없이 코드 자체 구조에서 검출 가능케 함.
6. 핵심 기법 2단(확장): 동일 메커니즘을 deliberate bit-flipping 기반 k-constraint 강제에 확장 — parity violation 시 flipping 위치 하나만 있으면 이를 all-0 error event로 처리, 아니면 기존 dominant error 목록 유지.
7. 부수 디테일: 4-cycle free 랜덤 circulant 구성, 트렐리스 검출기-LDPC 디코더 간 반복(iteration) 없음, LDPC 자체는 최대 20회 내부 반복.
8. 학습/최적화 절차: 별도 없음(사전 시뮬레이션으로 dominant error event 목록을 미리 산출).
9. 결과: dominant error 검출만으로 BER 10^-5에서 0.1dB 이득. k=8/9 k-constraint 강제 시 0.2~0.4dB 이득(k를 9→8로 줄일수록 이득 증가).
10. 한계: 자기기록 채널·SOVA 트렐리스 검출 전용 시스템 기법, LDPC 디코더 알고리즘 자체 변경 없음(순수 후처리/시스템 레벨), NAND에는 대응 채널 모델 없음.

| dominant error event correlation 검출 | 대응 없음 | NAND에는 트렐리스 검출기·error event correlation 개념이 없어 대응 모듈 없음 |
| k-constraint(RLL) 강제용 deliberate bit-flipping | 대응 없음 | NAND ECC는 RLL 제약 대상이 아니므로 대응 없음 |

> 적용 가치: 낮음 — 자기기록 채널의 트렐리스 검출-LDPC 결합 시스템에 특화된 후처리(parity 기반 error event 검출, RLL k-constraint) 기법으로 NAND ECC 디코더/부호설계와 직접 연결점이 없음.
