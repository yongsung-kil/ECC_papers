> 총평: 임의의 코드/채널/디코딩 알고리즘에 보편 적용 가능한 trapping-set 제거용 그래프 커버 구성법으로, decoder 알고리즘 변경 없이 error-floor를 개선하나 코드 길이 증가와 hard-decision 검증 한정이라는 제약이 있어 이식 가치는 중간 수준.

### B. 알고리즘 요약

1. 대상은 regular LDPC(예: Tanner (155,64), Margulis (2640,1320), MacKay (1008,504) 등)이며 BSC 채널에서 Gallager B(hard-decision) 디코딩을 사용.
2. 풀려는 문제는 error floor: 반복 디코더가 고SNR(저 error probability `a`) 영역에서 FER 개선이 급격히 둔화되는 현상, 그 원인은 Tanner graph의 trapping set(near-codeword) 구조.
3. 핵심 정의는 `(a,b)` trapping set(H의 a개 열 projection에 홀수weight 행이 b개)과 `critical number k`(디코더가 해당 trapping set에 빠지기 위해 초기에 오류여야 하는 최소 변수노드 수); FER의 semilog 기울기는 최소 critical number `i`(instanton 크기)로 결정됨.
4. 핵심 기법: 코드 `C`(길이 `n`, `H` 행렬) 2개 복제본 `C1, C2`로 `H(2) = [[H,0],[0,H]]`를 만들고, dominant trapping set(critical number가 작은 것부터)에 속한 edge를 두 복제본 사이에서 swap하여 원래 trapping set을 제거.
5. 핵심 식: swap 후 `H(2) = [[H', B],[B, H']]` (`H'+B=H`)이며, `Theorem 1`은 이 조작이 rate를 `r(2) <= r`로 만듦(H가 full-rank면 `Corollary 1`에 의해 `r(2)=r`); `Theorem 2`는 최소거리 `d_min <= d_min(2) <= 2*d_min` 유지.
6. 알고리즘 절차(2단): trapping set을 critical number 순 정렬 → 아직 swap 안 된 edge 하나를 무작위로 swap → 해당 trapping set의 edge를 "freeze"하여 재도입 방지 → 목표 크기까지 반복.
7. 부수 트릭: edge swap을 무작위 대신 "가장 적은 trapping set에만 관여하는 edge" 우선으로 선택하면 효율 개선 가능(예: Margulis 코드는 1320개 (4,4) trapping set이 각각 고유 edge를 가져 결정론적 제거 가능); 3개 이상 복제본으로 확장해 더 높은 critical number의 trapping set도 제거 가능.
8. 별도 학습/최적화 절차는 없음(그래프 커버 기반 결정론적/무작위 edge swap 조합 구성).
9. 결과: Tanner(155,64)->(310,126)에서 dominant trapping set을 (5,3,k=3)에서 (4,4,k=4)로 상향(rate 0.4129->0.4065로 소폭 하락); Margulis(2640,1320)->(5280,2640, full-rank이므로 rate 유지 0.5)에서 (4,4) trapping set 완전 제거, dominant가 (5,5,k=5)로 개선; MacKay(504)->(1008)에서도 FER 개선 확인(그림 4~7).
10. 한계: Gallager B(hard-decision) 알고리즘·BSC 채널에서만 수치검증(soft-decision/AWGN·min-sum 적용은 제안뿐), HW 미설계, 코드 길이를 최소 2배로 늘려야 하는 오버헤드 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| trapping-set 제거용 H-matrix 구성(edge swap, graph cover) | `ecc_top.cpp` `Load_PCM()` | H-matrix/PCM 구조 변경에 해당하나 코드 길이 2배 확장이 전제되어 재검증 부담 큼(§4 "부호설계" 이식 난이도 "높음") |
| trapping set/critical number 기반 error-floor 원인 분석 | 대응 없음 | Prime ECC 코드베이스에 trapping-set 분석 모듈 없음(코드는 시뮬레이터일 뿐 부호구성 최적화 도구 아님) |
| Gallager B(hard-decision) 디코딩 결과 | `decoder.cpp` `VN_Cal_HD()` | Prime ECC의 hard-decision(HD) 디코딩 모드와 대응되나 min-sum(2SD/3SD) soft 디코딩에서의 유효성은 별도 검증 필요 |

> 적용 가치: 중간 — 디코더 알고리즘 변경 없이 순수 부호구조 수정만으로 error-floor를 낮추는 범용 기법이라 개념적으로 매력적이나, 코드 길이 2배 확장 전제와 hard-decision 검증 한정으로 Prime ECC의 고정 rate/length QC-LDPC 구조에 바로 적용하기는 어려움.
