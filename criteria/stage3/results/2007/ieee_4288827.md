### B. 알고리즘 요약

1. 대상 부호: IEEE 802.16e 표준 LDPC, code length 2304, code rate 2/3, AWGN+BPSK 채널에서 평가.
2. 문제: BP 기반 디코더는 Tanner graph 내 trapping set으로 인해 error floor(약 10^-6)가 발생하고, 표준 부호는 프레임 길이별 공용 H로 인해 trapping set을 완전히 배제하기 어려움.
3. 가정: trapping set 존재 시 잘못된 정보가 그래프 전체로 전파되는 것이 error floor의 주 원인이라는 기존 관찰([2]-[4])을 채택.
4. 핵심 기법: row processing과 column processing을 분리하지 않고, row 처리 도중 각 extrinsic message `v'ij`가 계산될 때마다 즉시 `U'j`를 갱신하는 joint row-column processor 도입, column processor 자체를 제거.
5. 핵심 식: `U'j = uij + v'ij` (식 13/17)로 열 degree `wcj`만큼 반복당 U_j를 여러 번 갱신 — column processor를 기다리지 않고 정보를 즉시 확산시켜 trapping set의 오류 전파를 억제.
6. 확장: log-BP 버전(식 11-13)과 min-sum 버전(식 14-17, 스케일 계수 α, β 포함) 두 가지 joint processor 제시.
7. 구현 디테일: φ(x)=log((e^x+1)/(e^x-1)) 함수를 6-bit LUT로 구현, tree 구조 가산기로 propagation time 단축, 파이프라이닝 적용.
8. 별도 학습/최적화 절차 없음(고정 H-matrix, 알고리즘적 스케줄링 변경만).
9. 결과: BER<10^-4 도달에 traditional 14회 vs joint 7회 반복; 최대 clock 365MHz; 게이트 수 7054(row+column)→6825(joint)로 3% 절감; decoding speed 11Mbps→90Mbps(8배); 50회 반복 시 error floor가 traditional 10^-6 부근 발생 vs joint는 10^-8까지 미관측.
10. 한계: min-sum 버전은 지면 제약으로 시뮬레이션 결과 미제시, 검증은 단일 표준 부호(802.16e, rate 2/3, length 2304)에 한정, scaling factor(α, β) 최적값에 대한 상세 논의 부족.
