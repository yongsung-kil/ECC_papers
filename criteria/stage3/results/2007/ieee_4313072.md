### B. 알고리즘 요약

1. 대상은 이진입력 memoryless symmetric(BMS) 채널(주로 biAWGN) 위의 irregular LDPC 앙상블, min-sum(MS) 복호.
2. 문제: BP는 degree distribution 최적화로 capacity에 근접 가능하지만, MS는 최적화해도 capacity와 갭이 크게 남는지 미규명.
3. 핵심 가정: 메시지 밀도 evolution을 all-zero codeword 가정 하에 추적, MS check-node 업데이트는 `L(t)uv = sign(...) * min(|L(t-1)_bc,u'v|)`로 BP의 tanh 대신 min 연산.
4. 핵심 기법 1단: Bhattacharyya 파라미터 `B(a)=∫a(x)e^-x dx`를 이용한 MS 안정조건 유도 - `B(ach)λ'(0)ρ'(1) < 1`이면 density가 error-free로 수렴 (BP와 동일 형태의 조건).
5. 핵심 식 의미: 이 조건은 초기 반복에서 에러 밀도가 특정 임계값 이하이면 반복적으로 감소함을 보장 - degree distribution 설계 제약조건으로 사용.
6. 핵심 기법 2단: EXIT chart 기반 선형계획법(LP)으로 λ(x), ρ(x)를 단계적으로 갱신하며 rate 최대화(constraint: bit-node EXIT curve가 check-node curve 위에 위치).
7. 부수 트릭: modified MS는 check node 출력에 상수 `1/α`를 곱하는 정규화(scaling factor) 적용 - α=1.25일 때 capacity gap이 MS 대비 감소.
8. 최적화 절차: 이전 스텝의 density evolution 결과를 입력으로 사용하는 반복적 EXIT-chart 근사 LP (Gaussian 가정 대신 실제 density 사용).
9. 결과: rate 0.3에서 capacity gap 1dB(MS), rate 0.9에서 0.4dB(MS); modified MS(α=1.25)는 0.75dB~0.2dB로 개선; BP용 rate 0.5 코드를 MS로 복호 시 MS전용 설계보다 threshold가 0.97dB 열세.
10. 한계: 무한 블록길이 asymptotic 분석(density evolution/EXIT chart)뿐이며 유한길이 코드 성능, HW 구현, 실제 양자화 효과는 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum 복호(check-to-bit min 연산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min-sum CN 연산과 동일 알고리즘 계열, 안정조건은 이론적 배경 참고용 |
| Modified MS scaling factor(1/α) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 magnitude 양자화 테이블과 유사 목적(BP 근사 보정)이나 정확한 정규화 팩터 적용 위치는 코드상 명시 미확인 |
| Degree distribution 최적화(코드 설계) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC(N_b=129, VN degree 17)로 고정되어 있어 degree distribution 자체를 바꾸려면 H-matrix 재검증 필요 |
| EXIT chart / density evolution 기반 설계 절차 | 대응 없음 | Prime ECC는 시뮬레이터이며 자체 부호설계 최적화 도구 없음 |

> 적용 가치: 중간 - Prime ECC의 min-sum 알고리즘과 동일 계열이라 안정조건과 scaling factor 아이디어는 참고 가치가 있으나, 부호설계(degree distribution) 자체를 바꾸는 것은 이식 난이도가 높음.
