### B. 알고리즘 요약

1. Gbps급 매우 고속 LDPC 디코더를 대상으로, QC-LDPC 부호(rate `r=k/n`, 서브블록 크기 `t`)의 systematic block `H_s`와 dual-diagonal parity block `H_p`로 구성된 `(nt,kt)` 코드.
2. 문제: 고속 전송 시 병목은 연산량이 아니라 중간 LLR 저장용 memory access time - single/dual port memory 대신 register 직결 wiring 인터리빙이 필요한데, 이 경우 rate마다 별도 설계된 코드는 디코더 HW 재사용이 안 됨.
3. 핵심 가정: 정보워드 길이 `k`를 고정하고 parity 길이 `n-k`만 rate에 따라 가변(HARQ incremental redundancy 호환 목적), 기존 row-combining(저rate에서 고rate 유도)과 반대로 고rate mother code `H_s^(1)`(예: rate 4/5)에서 시작해 순차적으로 row-splitting해 저rate 코드 유도.
4. row-splitting 함수 `f_{n,m}(A)`: 행렬 `A`를 `m`개로 분할하되 `Σf_{n,m}(A)=A`, 각 분할행렬 non-zero entry가 겹치지 않고, 대략 균등한 non-zero 개수를 갖도록 정의(식 (4),(5)).
5. splitting은 각 non-zero entry `xy`(x=shift값, y=분할용 인덱스)에 `y mod j` 값으로 어느 분할행렬에 속할지 결정하는 규칙(식 (6), `g_i^k`)을 따름 - shift값 자체는 기존 값 유지, 최적화는 base matrix의 shift value 선택으로만 수행.
6. parity block `H_p`는 dual-diagonal 구조(`[h_p|H_p bar]`)로 고정, weight-3 column vector `h_p`의 shift값 a,b는 0으로 설정해도 girth/성능 영향이 미미하며 오히려 효율적 인코딩(참고문헌 [12])에 유리.
7. 부수 트릭: shortening 기법을 병행해 rate/frame length에 추가 자유도 부여(MIMO 연접 대응).
8. 학습/최적화 절차는 없음 - splitting 규칙은 단순 규칙 기반, 최적화는 base matrix shift value 선택에 국한(본 논문에서는 주요 초점 아님으로 명시).
9. 결과: rate 2/3, 1/2 코드(codeword length 1368/1824, avg column degree ~3.02~3.36)를 802.16e 표준 코드와 BLER 비교, 두 rate 모두 802.16e 개별 설계 코드와 comparable 성능(비슷한 BLER curve).
10. 한계: HW 미설계(register-wiring 구조는 개념 제안뿐), 이론적 gate/throughput 분석 없음, splitting 함수의 최적성은 논문 범위 밖으로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 고정 information length 유지 row-splitting 다중 rate 설계 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 단일 고정 rate H-matrix 로드 구조라, 다중 rate 재사용 프레임워크 자체는 대응 없음이나 QC-LDPC prototype matrix 개념은 `PCM_b` 구조와 유사 |
| memory access가 Gbps 디코더 병목이라는 지적 및 register 직결(wiring) 인터리빙 제안 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC도 HW 파이프라인/라우팅 최적화(GT, HCU)를 이미 갖추고 있어 개념적으로 유사하나 구체 회로 대응 없음 |
| dual-diagonal parity block 구조(shift 0) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC 인코더도 dual-diagonal 구조를 채택하고 있어 구조적으로 가장 근접 |

> 적용 가치: 중간 - HARQ향 다중 rate 재사용 목적은 NAND 고정 rate 시나리오와 어긋나지만, QC-LDPC dual-diagonal 인코딩 구조 및 "메모리 접근이 Gbps 디코더 병목" 통찰은 Prime ECC HW 설계 방향과 부합.
