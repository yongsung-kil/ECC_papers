### B. 알고리즘 요약

1. 시스템: LDPC 부호화 BICM(bit-interleaved coded modulation), M-PAM(예: 8-PAM) Gray 매핑 constellation, MacKay (1008,504) irregular 부호(rate=1/2) 및 SC=12 circulant 기반 QC irregular (1008,504) 부호 예시.
2. 문제: 반복복호 부호는 error floor 예측이 어려움(union bound 불가). BEC와 달리 AWGN+고차변조에서는 stopping set 대신 trapping set 기반 예측이 필요하며, 고차변조는 all-zero codeword 가정이 불가하고 비트레벨별로 노이즈 방향/문턱이 다름.
3. 핵심 가정: `stable trapping set`(정의 1) — 마지막 η(=5)회 반복에서 하드결정이 변하지 않는 실패 비트집합 `T(y)`. 고SNR 영역에서 지배적 실패 메커니즘으로 실험적으로 확인.
4. 핵심 기법 1단: 2단계 예측 — (1) trapping set 후보 목록 수집(장기 시뮬 또는 기존 탐색기법), (2) 각 trapping set의 FER 기여도 `P(ξ_T)` 평가.
5. 핵심 식: 노이즈 벡터를 실패방향 `d̄`(worst-case representative 기반)로 분해 `(n1,...,n|T|) = γd̄ + Σδi b̄i` (식 1), `γ`에 대해 조건부 적분해 `P(ξ_T|A_T)` 산출(식 2) — 노이즈의 특정 방향 성분만 디코더 실패에 지배적으로 기여한다는 근사.
6. 핵심 기법 2단(확장): constellation point를 bit-level별 good/bad로 분류(Table I)해 `Ω_T`를 disjoint subset `Bi`로 분할, worst-case subset `B1` 기반 하한(식 5,7)과 second-worst subset 포함 상한(식 9) 도출.
7. 부수 트릭: 대표 constellation 심볼(worst-case representative)만 고정해 시뮬레이션하고, 나머지는 무작위 전송+coset 복호로 인코딩 복잡도 회피. circular shift 대칭성 이용해 trapping set을 class로 그룹화(Table III).
8. 학습/최적화 절차: interleaver 설계 알고리즘 — (1) 지배적 trapping set들 탐색, (2) 그 trapping set에 속한 variable node를 모두 신뢰도 높은 MSB 비트레벨로 재매핑.
9. 결과: MacKay(1008,504)+8-PAM에서 예측 상/하한이 시뮬 stable FER와 잘 부합(Fig.3). interleaver 재설계 후 σ²=0.0447에서 stable FER가 약 1자릿수(order of magnitude) 감소(Fig.4).
10. 한계: M-ary(8-PAM) BICM·Gray매핑 전제, HW 미설계, 시뮬 결과만 존재. 새 interleaver가 저SNR(waterfall) 영역엔 거의 영향 없고, 일부 non-dominant trapping set의 기여도가 오히려 커질 수 있어 지배적 set만 타겟팅 권장.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stable trapping set 식별·error floor 기여도 예측 | 대응 없음 (Prime ECC는 시뮬 기반 BER/FER 측정, 해석적 error floor 예측 모듈 없음) | 개념(trapping set 기반 error floor 원인 분석)은 error-floor 대책 설계 시 참고 가능 |
| bit-level 신뢰도 기반 interleaver 재매핑 | 대응 없음 (Prime ECC는 binary 1비트 심볼, M-ary constellation·bit-level 개념 없음) | M-ary BICM 전용 기법으로 binary NAND 채널엔 직접 대응 없음 |
| MSB/LSB 등 constellation bit-level 분류(Table I) | 대응 없음 | NAND soft-read LLR 양자화(`channel.cpp` `Set_LLR_Th()`)와 목적은 유사하나(신뢰도 차등화) 변조 기반이 아니라 이식 불가 |

적용 가치: **중간** — trapping set 기반 error floor 원인 규명·정량 예측이라는 방법론은 Prime ECC의 error-floor 개선 설계에 개념적으로 참고할 수 있으나, 핵심 기법(interleaver 재매핑)은 M-ary constellation 전용이라 binary NAND 채널에는 직접 이식 불가.
