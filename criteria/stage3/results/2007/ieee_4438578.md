### B. 알고리즘 요약

1. 시스템: 클러스터 스토리지(RSC 플랫폼)에서 파일을 데이터 스트라이프로 나눠 여러 storage node에 분산 저장, node crash/네트워크 장애에 대한 내결함성 확보가 목표.
2. 문제: 복제(RAID-1)는 저장 오버헤드가 크고 Reed-Solomon(RAID-5류)은 인코딩/디코딩 복잡도가 큼 → XOR 기반 small LDPC erasure code로 대체.
3. 핵심 가정/모델: `m` 데이터 노드, `n` 코딩 노드로 구성된 이분 그래프(bipartite graph), 각 코딩 블록은 연결된 데이터 블록들의 XOR parity(예: `r1=l1+l3+l4`).
4. 핵심 기법: rate `R=5/8`(`m=10,n=6`)로 Monte Carlo 방식 그래프 생성 알고리즘을 사용해 임의의 노드 3개 손실까지 견디는 그래프를 탐색.
5. 핵심 식: 가용성 `A_L(µ,m',n') = Σ_{i=m'}^{N} C(N,i)·µ^i·(1-µ)^{N-i}`(식1)로, host 가용성 `µ`와 rate/overhead 조합별 파일 재구성 확률을 평가.
6. 부수 트릭: overhead factor `f=o/m`(디코딩에 필요한 평균 블록 수/데이터 블록 수)로 rate를 R=5/8로 선택해 저장 오버헤드와 가용성의 트레이드오프를 맞춤(fmax=1.23).
7. 구현: 클라이언트가 정상 시 데이터 블록만으로 파일 재구성 시도, 실패 시 코드 그래프로 필요한 parity 블록을 추가 조회(node-level erasure decoding, bit-level 반복복호 아님).
8. 학습/최적화 절차: Monte Carlo 기법으로 특정 노드 손실 내성(3개 노드까지)을 만족하는 그래프를 탐색(도수분포 최적화 아님, 시스템 설계 차원).
9. 결과: degraded 상태에서 LDPC 스킴이 RAID-5 대비 읽기/쓰기 성능 우수(예: 5클라이언트에서 read 210MB/s vs 179MB/s), 정상 상태에서는 RAID-5가 더 많은 데이터 서버로 인해 read 대역폭 우세.
10. 한계: bit-level LDPC decoder/HW가 아닌 노드-블록 단위 erasure code 시스템 평가이며, 실험도 네트워크/디스크 I/O 벤치마크(XDD)에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 노드-레벨 LDPC erasure code (Monte Carlo 그래프 생성) | 대응 없음 | NAND binary LDPC(bit-level QC-LDPC)와 무관한 분산 스토리지 노드 erasure code |
| XOR parity 기반 인코딩/디코딩 | 대응 없음 | Prime ECC의 dual-diagonal 인코더/min-sum 디코더와 추상화 레벨이 다름(블록 단위 vs bit 단위) |
| 가용성 분석(availability formula) | 대응 없음 | 시스템 레벨 신뢰성 지표로 ECC decoder 성능 지표(BER/FER)와 무관 |

적용 가치: **낮음** — 분산 클러스터 스토리지 시스템 논문으로 NAND binary QC-LDPC 부호설계/디코더/HW 어느 레이어와도 직접 연결되지 않음.
