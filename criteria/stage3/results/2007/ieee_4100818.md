### B. 알고리즘 요약

1. Magnetic recording channel(EPR4+AWGN) 대상, high-rate(8/9 등) regular QC-LDPC 코드(길이 4608/8640)를 FPGA 하드웨어 시뮬레이터로 sector error rate 10^-9~10^-14 수준까지 검증.
2. 문제의식: 기존 소프트웨어 시뮬레이션은 10^-6~10^-7 수준까지만 도달 가능해 초저 오류율에서의 error-floor를 정확히 예측하기 어려움.
3. 채널 모델: EPR4 신호 + AWGN, max-log-MAP 검출기와 QC-LDPC 디코더 간 soft-output을 주고받는 turbo(global iteration) 구조를 가정.
4. 핵심 실험: column weight `wc=3`은 error floor에 취약, `wc=4`가 훨씬 강건함을 발견. circulant size `p`가 작으면 error floor 발생, `p`가 너무 크면 circulant weight `w>2` 필요해져 오히려 성능 저하.
5. 도출 가이드라인: circulant size는 상대적으로 크게, circulant weight는 1 또는 2로 유지 → 이 조건에서 rate 8/9~0.96 코드가 관측 가능한 범위(10^-10) 내 error-floor free.
6. 디코더 아키텍처 확장(2단): 기존 구조([10])는 병렬도가 circulant size `p`에 반비례했는데, `p`가 divisible한 `k`(=p/parallelism factor)로 그룹화해 병렬도를 `k`배 향상시키는 CNU/VNU 그룹 구조 제안.
7. 구현 디테일: decoding message memory(DMMB)/channel message memory(CMMB)를 dual-port로 분리하고 barrel shifter로 순환 permutation 라우팅을 처리하는 memory fabric 설계, deep pipeline + retiming으로 225MHz 달성.
8. 학습/최적화 절차: 없음(경험적 구조 파라미터 스윕을 통한 가이드라인 도출).
9. 결과: rate 8/9, length 4608 QC-LDPC로 2M gates, 320 memory macros, 49mm²@0.13um, 225MHz, 최악의 경우(16 iteration, global iteration 없음) 1.8Gb/s throughput(global iteration 4회 시 약 360Mb/s로 저하), RS 코드 대비 코딩 이득 우위 확인.
10. 한계: magnetic recording 채널(EPR4+AWGN) 및 soft-4bit+/6bit 정밀도 가정에 특화되어 있고, iterative detector-decoder turbo 구조 전제이며 NAND read 채널 특성(제한된 soft read count)은 고려되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| high-rate regular QC-LDPC error-floor free 구조 가이드라인(circulant size/weight) | `ecc_top.cpp` `Load_PCM()` | 부호설계 이식은 난이도 높음이나, error-floor 개선 가이드라인 자체는 참고 가치 있음 |
| p배 병렬 CNU/VNU 그룹 디코더 아키텍처 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC의 z_sb=32 row 단위 병렬 구조와 유사 개념(병렬도 확장), 비교 검토 가치 있음 |
| DMMB/CMMB 메모리 fabric + barrel shifter 라우팅 | `decoder.cpp` (HW 모델), `ecc_data.h` (PCM 구조체) | Prime ECC HCU/GT 구조와는 다른 방식의 메모리-CNU/VNU 매핑, HW arch 비교 대상 |
| max-log-MAP 검출기-디코더 turbo(global iteration) 결합 | 대응 없음 | Prime ECC는 NAND read 채널 기반으로 이런 turbo detector 결합 구조 없음 |
| ASIC 합성 결과(2M gates, 1.8Gb/s, 0.13um) | 대응 없음(참고용 벤치마크) | 실측 HW 벤치마크로서 비교 참고 가치 |

> 적용 가치: 중간 — error-floor free 부호설계 가이드라인과 병렬화 디코더 HW는 참고 가치가 있으나, 채널 모델(magnetic recording, turbo detection)과 연판정 비트수가 NAND 환경과 다르고 부호설계 이식 자체는 재검증 부담이 큼.
