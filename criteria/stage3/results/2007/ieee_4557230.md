### B. 알고리즘 요약

1. 이진 LDPC(irregular, rate 1/2, K=4032)를 Min-Sum으로 반복복호할 때, check/variable node의 처리 순서(스케줄링)가 성능에 영향을 미친다는 점에서 출발한다.
2. Wiberg의 computation tree 이론을 이용해, minimal deviation이 낮은 신뢰도(negative LLR) variable node의 복사본을 많이 포함할수록 디코딩 오류 확률이 커짐을 근거로 삼는다.
3. 핵심 가정: all-zero codeword 전송 가정 하에, unverified check node(패리티 불일치, `C_m=1`)는 그 이웃에 신뢰도 낮은 variable node가 있다는 신호로 사용한다.
4. depth-2 neighborhood reliability 정수값 함수 `f(m) = σ_m(d_max+1) + Σ_{n∈H(m)} σ_{m,n}` 정의 (`σ_m`=검증여부, `σ_{m,n}`=이웃 unverified check 존재 flag, `d_max`=최대 check degree).
5. `f(m)`이 작을수록 신뢰도 높은 check node로 판정, 값에 따라 check node를 reliability set `C_f`로 분류해 정렬 없이 그룹화(정수값 함수의 장점).
6. mixed scheduling(가변 병렬도)으로 `C_f` 집합을 reliability 순으로 직렬 처리하고 집합 내부는 병렬 처리; 처리된 check node의 이웃 variable node는 "iteration leaf" 플래그를 0으로 갱신.
7. "cutting back" 기법: 아직 미처리된(iteration leaf) variable node는 이전 iteration 메시지를 재전송해, minimal subtree가 항상 더 신뢰도 높은 descendant check node를 거치도록 강제.
8. 학습/최적화 절차 없음(closed-form 함수 정의 및 알고리즘적 스케줄 배치).
9. 결과: AWGN/QPSK, rate 1/2 irregular 이진 LDPC(K=4032, 200 iteration)에서 Min-Sum+정수 reliability가 고 SNR에서 Sum-Product 성능에 근접(그림 4). GF(8) 비이진 LDPC(K=2016 bits, 672 심볼)에도 동일 함수가 수정 없이 적용되어 유사한 개선 확인.
10. 한계: 200 iteration 기준 시뮬레이션 결과만 제시, HW 구현·gatecount·throughput 분석 없음. reliability 계산·집합 분류의 실제 HW 오버헤드 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum 디코딩(CN/VN 갱신, iteration 루프) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | neighborhood reliability를 삽입할 최상위 반복 루프 |
| CN min-sum 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | check node 신뢰도 계산 시 sign/magnitude 정보 재사용 가능 |
| check node 검증(parity) | 대응 없음 (조기종료 CRC와 별개) | `σ_m` 계산은 check node별 parity 값이 필요, 기존 조기종료용 CRC(`partial_CRC.cpp`)와는 목적이 다름 |
| 처리 순서(스케줄링) 변경 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 내 CN 처리 순서 | Dual-Update(even/odd 교대) 스케줄과 병행/대체 검토 필요 |

> 적용 가치: 중간 — Min-Sum waterfall 성능 개선 기법으로 관심 영역에 해당하나, reliability 계산·정렬 오버헤드의 HW 비용과 기존 pipeline(z=32, HCU, Dual-Update) 스케줄과의 정합성이 미검증.
