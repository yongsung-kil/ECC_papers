### B. 알고리즘 요약

1. 대상: 1024-bit rate-1/2 LDPC(1024 variable node, 512 check node) fully-parallel 디코더의 throughput-area 효율 극대화.
2. 문제: fully-parallel 구현은 최대 throughput을 내지만 실리콘 면적/배선혼잡/클록 스큐 문제로 code length가 커질수록 효율이 급격히 저하됨.
3. 핵심 가정: iterative decoding은 각 입력 메시지가 서로 독립적으로 디코딩되는 태스크이며 반복횟수(iteration count `M`)가 고정이라는 특성을 이용하면 피드백 루프도 파이프라이닝 가능.
4. 핵심 기법 1단: `C-slow retiming` - 루프/IO 레지스터를 `C`개 연속 레지스터로 교체해 서로 다른 독립 데이터 스트림 `C`개를 인터리빙 처리, 클록 주기를 `T' = Tlogic + TSU + TCKQ + Tskew`로 단축 (`Tlogic`은 원 combinational delay/pipeline depth).
5. 스케줄링 식: `Tload=(C-1)×M`, `T1st=Tload+M`, throughput은 `M/C` iteration당 1 메시지(비파이프라인 대비 `C`배 향상 잠재력); `M`이 `C`로 나누어떨어지면 `M/C` iteration마다 새 입력 스케줄.
6. 핵심 기법 2단: 위 동기식 C-slow 파이프라인을 `asynchronous micropipeline`(Sutherland 방식, request/acknowledgement handshake + C-element)으로 변환해 글로벌 클록 없이 지역적 데이터 전송, PVT 변이에 강건.
7. 구현 디테일: VNU-CNU 경로에 input MUX/DEMUX + counter로 C-slow 스케줄에 따른 select 신호 생성, message extension register로 `C`개 독립 스트림의 입력 메시지 보관.
8. 최적화 절차: Synopsys Design Compiler `pipeline_retime` 커맨드로 다양한 pipeline depth에 대해 합성, absolute cost function(`area×clock_period`) 및 relative cost function(면적 오버헤드 대비 지연 개선)으로 최적 depth 탐색(최소 비용은 초기 클록주기 8ns, depth 6).
9. 결과: 기존 발표된 동기식 구현(2Gbps, 클록주기 8ns) 대비 클록주기 2ns에서 throughput 4배, 면적 오버헤드는 약 80%(비동기 오버헤드는 CNU/VNU 대비 약 3.8%로 무시 가능).
10. 한계: 정정성능(BER/error-floor) 개선 없음(순수 타이밍/아키텍처 논문), 비동기 설계 검증은 pre-layout 시뮬레이션 수준이며 실측/ASIC 합성 후 공정변이 실증은 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| C-slow retiming(피드백 루프 파이프라이닝) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 파이프라이닝 개념은 Prime ECC의 6-stage pipeline과 유사 목적이나, 완전병렬 fully-parallel 아키텍처 전제로 z=32 반병렬 구조에는 직접 적용 어려움 |
| 비동기 micropipeline(clockless, handshake) | 대응 없음 | Prime ECC는 클록 레벨 bit-exact 동기식 시뮬레이터로 비동기 설계 패러다임 자체가 대응 없음 |
| VNU/CNU 표준 Log-BP(tanh 기반) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 Min-Sum 채택, full-BP/tanh는 구식 중복 기여 |
| pipeline depth 최적화(cost function 기반) | `decoder.cpp` pipeline(col_M1/Z/P1~P4) | 파이프라인 단수 최적화 방법론은 Prime ECC HW 모델에 참고 가능하나 fully-parallel 전용 |

적용 가치: 중간 — 알고리즘 개선은 없지만 iterative 피드백 루프의 파이프라이닝(C-slow retiming) 아이디어는 Prime ECC의 pipeline 구조 최적화에 참고할 여지가 있음. 다만 비동기(clockless) 설계와 fully-parallel 아키텍처 전제는 Prime ECC의 동기식 반병렬(z=32) bit-exact 모델과 이질적이라 직접 이식은 어려움.
