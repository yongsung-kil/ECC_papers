### B. 알고리즘 요약

1. 대상은 이진 소거 채널(Binary Erasure Channel, BEC)에서 선형 블록부호 `[n,k,d]`의 복호로, 각 비트는 확률 `ε`로 소거되고 `1-ε`로 정확히 수신됨.
2. 전통적 반복복호(iterative decoding)는 한 번에 방정식 1개(패리티체크 1개)만 풀어 미지수 1개를 구하는데, 이는 stopping set에 걸리면 실패 — 최적(exhaustive) 복호는 복잡도가 지수적으로 큼.
3. 핵심 가정/모델: 패리티체크행렬 `H`(rank `n-k`)의 부분집합 방정식을 이용해 소거된 비트를 순차적으로 복구하는 대수적(선형대수) 복호 절차.
4. 핵심 기법: order-p 일반화 반복복호 — 각 스텝에서 최대 `p`개의 방정식으로 `p`개 미지수를 동시에 푸는(≤p×p 가역행렬 역산과 동치) 절차. `p=1`이면 전통 반복복호, `p=n-k`이면 최적복호와 동일 성능.
5. 핵심 식: 소거불능확률 `P_UD^IT,p(H) = Σ_i D_i^(p) ε^i(1-ε)^{n-i}`(order-p dead-end set enumerator 기반)이며, `d ≥ s^(p) ≥ min{p+1,d}`로 order가 커질수록 stopping distance `s^(p)`가 최적 복호의 최소거리 `d`에 근접.
6. 구현 트릭: `t=1`부터 시작해 순차적으로 `t`를 늘려가며 방정식을 풀고, `t=3`인 경우는 사전계산된 XOR 조합식(`x1=b1+b2+b3` 등)으로 행렬 역산 없이 간단히 처리 가능.
7. 핵심 이론 결과(Theorem 1): Tanner 그래프 girth `g`에 대해, order를 늘려도 개선 없는 최대 order는 `p_min(g)=⌈3g/4⌉` — 즉 girth 6(LDPC 전형)이면 order 3까지는 무의미, order 4부터 개선 가능.
8. 학습/최적화 절차 없음 — 조합론적 증명(Lemma 1: 특정 구조의 `p×p` 이진행렬 `Q*`로 하한 달성 증명).
9. 결과: 수치실험 없음, girth-order 관계식 `p_min(g)=⌈3g/4⌉`이 유일한 정량적 결과(예: girth 6 LDPC에서 order 4 이상이 필요).
10. 한계: BEC 전용(소거만 다룸, soft-LLR/AWGN 미적용), HW/구현 복잡도 분석 없음, 시뮬레이션도 없는 순수 이론 논문이라 NAND 채널(BSC/AWGN형, min-sum 기반) 디코더에 직접 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| order-p 일반화 반복복호(가우스 소거 기반) | 대응 없음 | Prime ECC는 min-sum 기반 soft/hard 디코더이며 BEC 대수적 소거 복호 알고리즘 자체가 없음 |
| stopping set / girth 분석 | 대응 없음 | error-floor 원인 분석 이론으로 참고 가능하나 코드 내 구체 모듈 대응 없음 (H-matrix girth는 `ecc_top.cpp` `Load_PCM()`에서 로드하는 값이지 본 논문의 알고리즘과는 무관) |
| BEC 채널 모델 | `channel.cpp` | Prime ECC는 AWGN/RBER/fixed-error만 지원, BEC 소거 모델 미지원 |

> 적용 가치: 낮음 — BEC 전용 대수적 복호 이론으로 채널 모델과 복호 알고리즘 모두 Prime ECC(binary QC-LDPC, min-sum, BSC/AWGN형 채널)와 맞지 않음.
