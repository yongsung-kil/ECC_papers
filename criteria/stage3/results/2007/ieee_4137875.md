### B. 알고리즘 요약

1. 시스템: irregular LDPC 부호(N,K) 설계, degree-2/3/higher bit node 혼합 구조, 예시로 (1728,864), (4608,2304) 부호.
2. 문제: 무작위 생성 irregular LDPC는 DE 상 점근적 임계값이 좋아도 짧은/중간 길이에서 error floor 발생, degree-2·3 bit node로 구성된 저가중치 코드워드가 원인.
3. 핵심 가정: degree-2 bit node로만 이뤄진 사이클이나 degree-3 bit node 2개+degree-2 bit node로 이뤄진 서브그래프(Fig.1)를 flip하면 여전히 모든 parity check가 만족되어 저가중치 코드워드가 생성됨.
4. 핵심 기법: degree-2 bit node와 check node를 교대로 연결해 tail-biting 체인(loop)을 만들고, degree-3 이상 bit node를 loop에 연결할 때 두 check node 간 거리가 `d1` 이상 되도록 하는 제약(Constraint 1) 부과.
5. 핵심 식: 두 degree-3 bit node의 연결 check 인덱스 `c=(c1,c2,c3)`, `c'=(c1',c2',c3')` 간 `|ci-ci'| ≥ d3` (Constraint 2), 짧은 사이클 방지용 `|ci-ci'| ≥ d2`(Constraint 3, d2<d3) - 위반 시 weight `d3+2` 미만 코드워드 존재.
6. 확장: quasi-cyclic extension(작은 그래프 G0를 extension factor n배 복제 + offset value로 edge 연결)으로 위 제약을 만족하는 큰 그래프를 재귀적으로 생성, 탐색 복잡도를 `O(N3²)`에서 `O(N3'²)`로 축소.
7. 구현 디테일: encoding은 degree-2 bit를 parity bit로 고정하고 RA code의 rate-1 accumulator처럼 순차적으로 parity를 결정하는 선형시간 인코더(길이에 선형).
8. 최적화 절차: 별도 학습 없음, offset value를 제약 조건에 맞게 수동/탐색적으로 선택.
9. 결과: (1728,864) 부호에서 PEG 대비 error floor 1자릿수 이상 개선, undetected WER PEG 대비 약 2자릿수 낮음; ACE 대비 저WER 영역 마진 최대 0.4dB, (4608,2304)에서도 ACE 대비 weight profile 개선.
10. 한계: 최소거리 하한을 수학적으로 보장하지 않음(확률적 배제), degree-4 이상 조합의 저가중치 케이스는 다루지 않음, HW 구현/합성 없음, rate 0.5 중심 실험(고rate 미검증).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| quasi-cyclic extension 기반 저가중치 서브그래프 배제 구성법 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 바꾸는 제안이라 우리 고정 QC-LDPC(base+shift) 구조에 이식하려면 H-matrix 재설계·재검증 필요 |
| tail-biting loop + linear-time 인코더 | `encoder.cpp` `Generate_PCM_encoding()`, `Column_Reordering_Dual_Update()` | 우리는 dual-diagonal 인코더 고정, 이 논문의 tail-biting accumulator 구조는 인코더 재설계급 이식이라 난이도 높음 |
| degree-2/3 노드 제약 기반 error-floor 저감 | `decoder.cpp` `LDPC_Decoder()` | 디코더 알고리즘이 아닌 부호설계 단계 기여로 디코더 코드에는 직접 대응 없음 |
```

적용 가치: `중간` - error-floor 개선 아이디어(저가중치 서브그래프 배제)는 부호설계 원리로서 유효하나, 우리 코드의 고정 QC-LDPC H-matrix·인코더 구조 재설계가 필요해 이식 난이도가 프로파일 §4 기준 "높음" 레이어에 해당.
