### B. 알고리즘 요약

1. 대상은 Reed-Solomon 부호 `Cq(n,k)` (GF(2^q)) - RS 부호의 parity-check 행렬 `Hq`는 밀도가 높아 BP 복호에 부적합.
2. 문제: RS 코드의 이진 이미지 `Hq,2`도 여전히 조밀하고 4-cycle이 많아 BP(신뢰전파) 성능이 나쁨.
3. 핵심 가정: 채널은 AWGN, 입력은 채널 LLR `llri = log(Pr(vi=0|y)/Pr(vi=1|y))`.
4. 기법 1단: dual code의 이진 이미지에서 저가중치 codeword를 탐색(Algorithm 1/2)해 저밀도 이진 parity-check 행렬 `H~q,2`를 구성(예: C8(255,239) 밀도 35%).
5. 기법 2단: Bit-Slicing으로 고신뢰 비트(Sl)를 hard-decision 처리 후 그래프에서 제외, 저신뢰 비트(ASl)만 BP 수행.
6. redundant parity-check 집합 중 활성 변수(ASl) 연결수 `nci(ASl)`를 최소화하는 부분집합(CASl)을 greedy 선택해 사용.
7. BP(min-sum) 출력을 legacy 복호기(HDD 또는 LCC Chase)에 반복 적용하는 하이브리드 구조.
8. Error Recovery: 불만족 check의 최소 메시지 크기 `eci(j)`가 큰 순서로 검사집합 K(j)를 골라 일시적으로 메시지 전달을 차단(Algorithm 3).
9. 결과: C8(255,239)에서 HDD 대비 FER 10^-6에서 0.9dB, LCC(η=5) 사용 시 1.3dB 개선; K-V(100) 복호도 능가.
10. 한계: RS 코드 전용이며 그래프 구성이 코드별로 사전 계산 필요, HW 구현 관련 논의 없음(시뮬레이션 결과만 제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP(min-sum) 반복복호 프레임 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | RS를 대상으로 하지만 min-sum 기반 메시지 전달 구조는 유사, 다만 부호가 다름 |
| Bit-Slicing(고신뢰 비트 조기 확정) | 대응 없음 | Prime ECC는 고정 LLR 테이블 기반 VN 처리이며 동적 slicing 미보유 |
| 저밀도 이진 parity-check 구성 | `ecc_top.cpp` `Load_PCM()` | RS의 이진 이미지 구성법으로 QC-LDPC H-matrix 설계와 직접 대응 안됨 |
| Error Recovery(불만족 check 차단) | 대응 없음 | Prime ECC는 조기종료(CRC)만 있고 check-freezing 방식 없음 |

> 적용 가치: 낮음 - 대상 부호가 RS이고 binary QC-LDPC와 구조가 달라 직접 이식 불가하며, HW 미설계로 검증 신뢰도도 낮음.
