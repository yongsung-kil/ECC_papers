### B. 알고리즘 요약 (10줄 내외)

1. Polar (N=256, rate 1/2, BPSK, AWGN) 코드의 BP(Belief Propagation) 디코더를 대상으로 함.
2. BP 디코딩은 실수 연산(L, R 메시지)이 복잡해 HW 구현이 어렵다는 문제를 해결하고자 함.
3. Information bottleneck 방법(관측변수 `Y`, 관련변수 `X`, 압축변수 `T`)으로 실수 메시지를 부호없는 정수로 양자화, `I(T;X)` 최대화·`I(T;Y)` 최소화 목적함수(`L = I(T;Y) - βI(T;X)`, β→∞).
4. 기존 LDPC 대상 연구([11])는 check/variable node 연산을 lookup table로 대체했으나, Polar BP는 메시지 종류가 4개(L, R 좌우)이고 노드당 3개 메시지를 조합해 1개 메시지 생성하는 구조라 더 복잡.
5. 채널 출력 `y`를 joint pdf `p(x,y)` 기반 quantizer로 `|T|`개 정수로 이산화, `p(t|y)` 매핑에서 LLR = `log(Pr(x=0|t)/Pr(x=1|t))` 산출.
6. Polar 전용 확장으로 3개 입력 메시지의 joint pdf `p(b,y) = Σ p(b1,y1)p(b2,y2)p(b3,y3)` (제약: `b=b1⊕b2, b2=b3`)를 정의해 lookup table 생성.
7. Lookup table 설계용 최적 `Eb/N0`(design SNR)을 반복적으로 탐색, `I(T;X)`가 iteration 50에서 1에 근접하는 `Eb/N0=2.5dB`를 채택(3.0dB는 조기포화, 과소값은 정보손실).
8. 별도 학습 없이 quantizer가 오프라인으로 lookup table을 생성(그리디 merge 방식, [11] 인용).
9. `|T|=16`일 때 비양자화 BP 대비 BER 열화 무시할 수준, 계산은 곱셈 12회·덧셈 4회/블록에서 lookup table 참조로 대체(4비트 정수만 사용).
10. 한계: 알고리즘/시뮬레이션 수준만 제시, HW 설계·gatecount·throughput 수치 없음. Polar 전용이며 LDPC 이식은 검증되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Polar BP 디코딩 전체 | 대응 없음 | Prime ECC는 QC-LDPC 전용, Polar 코드 구조 자체가 대응 없음 |
| Information bottleneck 기반 메시지 양자화/lookup table | 대응 없음 | 개념적으로 `channel.cpp` `Set_LLR_Th()`의 LLR 양자화 테이블과 유사하나 논문은 IB 방법론이 LDPC가 아닌 Polar에 적용된 것이라 직접 대응 불가 |
| Channel output quantizer (`p(x,y)`→`p(t|y)`) | 대응 없음 | soft-read LLR 설계와 개념 근접하지만 본문이 Polar 채널모델 기준이라 이식 근거 부족 |

적용 가치: 낮음 — 대상 코드가 Polar이며 NAND binary QC-LDPC와 무관, IB 양자화 개념 자체는 참고문헌([11])이 LDPC 원조격이므로 그쪽 검토가 더 적절함.
