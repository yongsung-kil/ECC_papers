> 총평: circulant CPM을 통과하는 짧은 cycle 수 기준으로 successive하게 masking해 error-floor를 낮추는 binary QC-LDPC 부호설계법으로, NAND용 고rate QC-LDPC 부호설계에 개념 이식은 가능하나 H-matrix 교체·재검증 부담이 크다.

### B. 알고리즘 요약

1. 대상: AWGN + SPA(50 iter)에서 girth를 키우고 short cycle을 줄인 binary `(γ,ρ)`-regular/irregular QC-LDPC 부호설계. 예제: `(576,288)`, `(576,432)`, `(2640,1320)`.
2. 문제: 짧은 cycle이 iterative 복호 성능(특히 error-floor)을 저하 -> 기존 masking은 computer search로 방대(예: [10]은 65373+8970개 masked matrix 탐색).
3. 구성: `W`의 원소 `w_{i,j}=i·j (mod p)`를 `p×p` CPM `I(w_{i,j})`로 분산(array dispersion), `γ×p` CPM 배열 `A_disp(W)` 생성. p는 소수 제약을 임의 양의정수로 확장.
4. 부호형성: `A_disp(W)`에서 ρ개 column-CPM을 선택해 `γp×ρp` H 구성, null space가 길이 `pρ`인 `(γ,ρ)`-regular QC-LDPC.
5. 핵심식: 2k-cycle 존재조건 `Σ(h_{i_s,j_s} - h_{i_{s+1},j_s}) ≡ 0 (mod p)` — cycle이 CPM shift값과 p로 완전 결정됨.
6. Algorithm 1: 위 조건으로 짧은 cycle을 exhaustive search해 각 CPM의 "indication number" `n_{i,j}`(그 CPM을 통과하는 2k-cycle 수) 산출. 복잡도 `O(γ^k ρ^k)`.
7. Algorithm 2(masking): indication number가 가장 큰 CPM을 ZM으로 successive 치환, N이 all-zero면 k증가. regular면 목표 degree분포 도달 시, irregular면 threshold `Th(M)>Th(M0)`(protograph EXIT threshold) 개선 시 종료.
8. 결과 신뢰도: `(576,288)` 코드가 WiMAX-LDPC 대비 WER 3e-5에서 0.65dB, CCSDS/[4] 대비 WER 1e-6에서 0.15dB 이득, WER 4.3e-7까지 error floor 없음.
9. 효율: `(2640,1320)` (3,6)-regular 코드는 [10]과 성능 동등하면서 Algorithm 1을 8회만 수행해 훨씬 단순. 복잡도 `O(T γ^k ρ^k)` (T=masked CPM 수).
10. 한계: 순수 부호구성법으로 HW 설계·복잡도·throughput·양자화·NAND read 전혀 없음, 시뮬(SPA)만. rate/length가 예제별로 다름.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| array dispersion + masking으로 H 구성 | ecc_top.cpp Load_PCM() | 새 base+shift H-matrix를 만드는 부호설계법이나, Prime ECC는 특정 QC-LDPC 고정이라 교체·재검증 부담 큼 |
| CPM shift값 기반 cycle 판정(indication number) | 대응 없음 | 부호설계 단계 분석 도구로 디코더/인코더 런타임 모듈과 무관 |
| girth↑ / short-cycle↓로 error-floor 개선 | 대응 없음 | 성능은 H 구조 자체에 내재, decoder.cpp 런타임 처리 대상 아님 |
```

적용 가치: **중간** — binary QC-LDPC의 error-floor를 낮추는 부호설계 원리(cycle 통과수 기준 masking)는 우리 고rate QC-LDPC 설계에 개념적으로 참고 가능하나, 고정 H-matrix(`matrix_sel`) 교체는 이식난이도 높고(프로파일 §4 "높음") 우리 rate~0.9·z=32 대역에서 재현/재검증이 필요하다.
