### B. 알고리즘 요약

1. 시스템: Nt=4 송신/Nr=4 수신 안테나, 16-QAM, Rayleigh fading MIMO 채널, 3GPP 5G NR LDPC (`R=1/2`, `K=1024`)를 outer code로 사용하는 반복검출-복호(IDD) 구조.
2. 문제: 저복잡도 MIMO 검출기(SOCA, tree pruning 기반)가 출력하는 extrinsic LLR `c̃e`가 완전탐색 ML 대비 over/under-confident해 self-consistency가 떨어지고, 이것이 반복될수록 LDPC 디코딩 성능을 오히려 저하시킴.
3. 가정/모델: LLR 품질은 claimed MI(평균 크기 기반 추정치)와 measured MI(히스토그램 기반 실측치)의 격차로 정량화하며, 격차가 클수록 self-consistency가 낮음.
4. 핵심 기법: DNN(`L=5`, 은닉 3층×`dl=6` 노드)이 입력 `x=[Eb/N0, iinner-outer, raw LLR]` → 보정 LLR `c̄e`를 회귀. 학습은 offline, MSE loss `Loss(M)`로 가중치 `W^l`, bias `θ^l` 학습.
5. 핵심 식: 학습 라벨은 히스토그램 기반 최적 LLR `Lt = ln(P0t/P1t)` (bin별 0/1 비트 조건부 확률 비, bin width `β`는 Scott's rule로 결정) — SOCA가 못 만드는 진짜 사후확률 기반 LLR을 근사.
6. 확장: LLR clipping(`|c̃e|>Lmax` → `±Lmax`, `Lmax=6`)을 DNN 보정 전 전처리로 병행 적용해 극단값을 우선 억제.
7. 구현 디테일: 학습 데이터는 O=500 프레임×1000 LLR, 10개 Eb/N0 값×3개 iteration index 조합마다 별도 학습(15 epoch, 10~20분); 온라인 추론은 ReLU 활성화 순전파 1회.
8. 벤치마크: LUT 기반 보정(양자화 후 LUT lookup+선형보간), FS-Net(DL 기반 하드결정 후보 탐색) 대비 성능/복잡도/메모리 비교.
9. 결과: M1=4에서 BLER 10^-3 기준 conventional 대비 0.9dB 이득(복잡도 +16%), LUT 대비도 0.2dB 우위이며 메모리는 LUT(47,920B) 대비 97% 감소(920B). M1=8/16에서는 이득이 0.6dB/0.4dB로 축소.
10. 한계: 순수 시뮬레이션(BER/BLER)뿐 HW 미설계, 무선 Rayleigh fading MIMO 채널 전용 검출기(SOCA) LLR 보정이 목적이라 NAND read channel/LDPC 디코더 내부와는 직접 연관이 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DNN 기반 LLR 보정 (MIMO 검출기 출력) | 대응 없음 | Prime ECC는 NAND read channel LLR(soft-read threshold)을 다루며 MIMO 검출기가 없어 직접 대응 모듈 부재 |
| LLR clipping | `channel.cpp` `Set_LLR_Th()` | 클리핑 개념은 LLR threshold 테이블과 유사하나 목적(검출기 출력 보정 vs read 양자화)이 다름 |
| LDPC 디코더 (5G NR, log-sum-product) | 대응 없음 | 본 논문은 표준 outer code를 그대로 사용, 디코더 알고리즘 자체 개선 없음 |
```

적용 가치: `낮음` — LDPC 부호/디코더를 개선하지 않고 MIMO 검출기 LLR 보정이 핵심이라 NAND binary QC-LDPC ECC 이식 여지가 거의 없음.
