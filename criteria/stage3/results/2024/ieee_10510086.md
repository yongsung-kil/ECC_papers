### B. 알고리즘 요약 (10줄 내외)

1. 선형 블록코드(이진/삼진)의 parity-check/generator 행렬을 Kannan embedding으로 격자(lattice) 문제로 변환한다.
2. 목표는 NP-hard인 "최소 가중치 코드워드(최소 거리) 탐색"을 SVP(Shortest Vector Problem)로 치환해 자동사상·순환성 등 부호 대칭성 없이도 빠르게 푸는 것.
3. 채널/부호 모델은 없고 순수 조합론적 격자 최적화 문제로 정식화.
4. 핵심 절차: `BKZ`(block Korkin-Zolotarev) 격자 축약 → Gram-Schmidt 직교화 → `KFP`(Kannan-Finke-Post) branch-and-bound로 SVP 해 탐색.
5. 탐색 트리 각 노드가 부분해 `xi`에 대응하며, 상한 `A`(초기값 `A=(n+1)×r_max^2`) 이하 노름의 벡터만 남기는 부등식 시스템(식 3)으로 가지치기.
6. 가속을 위해 Gaussian/Linear/Piecewise/Step branch pruning, extreme pruning(cylinder intersection), 이중공간 탐색, 순열 삽입, NN 기반 Gaussian heuristic 등 다수 heuristic을 결합.
7. HW 친화 트릭은 없으며 대신 multi-core(NPTL)/GPU(CUDA warp-level) 병렬화로 QR 분해(Householder/Givens)를 가속.
8. 별도 학습 절차는 없음(참고문헌상 NN heuristic 언급뿐, 본 논문 주 기법 아님).
9. 결과: rate 0.5, length 1280 부호에서 가중치 212 코드워드 발견(4,147,201초), 별도 문제(weight 228)에서 MAGMA Brouwer-Zimmerman 대비 3172.9배, 벡터화·병렬화된 최고 구현 대비 899배 가속.
10. 한계: 실행시간이 수십분~수백만 초 규모로 실시간 디코딩과 무관한 오프라인 분석 도구이며, HW 미설계, 이진 QC-LDPC 부호설계/디코더 알고리즘 자체를 제시하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 격자기반 최소거리(SVP) 탐색 알고리즘 | 대응 없음 | 디코더/인코더 알고리즘이 아닌 오프라인 부호 특성(최소거리) 분석 도구 |
| Kannan embedding / BKZ / KFP branch-and-bound | 대응 없음 | Prime ECC는 고정 QC-LDPC H-matrix를 사용하며 격자 기반 최소거리 계산 모듈 없음 |
| GPU/멀티코어 병렬 SVP 탐색 | 대응 없음 | 디코더 HW 병렬화(z=32, HCU)와 무관한 별도 오프라인 계산 |
```

마지막에 한 문장으로 **적용 가치**: `낮음` — LDPC 디코더/인코더/HW 설계가 아닌 암호해석용 격자 SVP 탐색 논문으로 Prime ECC 3.1 코드 이식 대상 요소가 없음.
