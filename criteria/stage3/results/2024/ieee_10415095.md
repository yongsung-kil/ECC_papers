### B. 알고리즘 요약

1. 시스템: 이진 소거 채널(BEC) 위에서 protograph 기반 GLDPC 부호, base matrix 크기 `4×8`, `8×16`, `3×12` 등으로 중간~고rate(0.5~0.75) 설계.
2. 문제: 기존 GLDPC는 SPC 노드를 GC(generalized constraint) 노드로 교체하는 doping 방식이라 rate-loss가 커서 중~고rate에서 LDPC 대비 우위를 못 보임; 기존 PD-GLDPC[5]는 degree-1 VN 미고려·단일타입 컴포넌트코드(SCC)만 사용.
3. 핵심 가정: lifting 후 GC 노드를 추가하는 PD(partially-doped) 방식으로 rate-loss 최소화, 채널은 BEC(소거확률 `ϵ`).
4. 핵심 기법1: degree-1 VN + 다중타입 컴포넌트코드(MCC, Hamming/extended Hamming/extended Golay 5종)를 도입해 PD-GLDPC를 일반화. 부분적으로 doped된 VN 인덱스 집합 `X`.
5. 핵심 식: GC 노드의 EXIT 함수 `I_EC(c_g,v_g) = (1/n_g)Σ...`(식 1)로 PEXIT 분석 확장, RED(G) 부분그래프 조건 `|RED(G)| ≥ n-m`이면 bit-error threshold = block-error threshold(정리 1, equality condition 확장).
6. 핵심 기법2: rate matching을 위한 랜덤 puncturing으로 목표 rate `R* = 1-m/n` 달성, puncture 비율 `ρ_d = ρ×n/|X|`.
7. 부수 트릭: RED(G) 탐색 알고리즘(Alg.1)에서 undoped cycle/degree-1 VN 제거 + doped VN 제거 반복, CPEG(circulant progressive edge growth)로 4-cycle 회피.
8. 학습/최적화: differential evolution 알고리즘(세대수 G=10000, 후보 S=10mn)으로 base matrix 최적화, 5종 컴포넌트코드 조합 15가지 탐색.
9. 결과: 최적화된 PD-GLDPC(dv1,MCC)가 FER 10⁻⁴에서 기존 LDPC/GLDPC 대비 우수한 성능을 보이며 평균 VN degree가 낮아 비슷한 디코딩 복잡도(비교/덧셈 연산 수, ×10⁵ 단위) 유지.
10. 한계: BEC 전용 설계·시뮬레이션(AWGN 확장은 향후 과제로만 언급), GC노드는 MAP(Gaussian elimination) 디코딩 필요해 HW 구현 복잡도 별도 고려 필요, HW 설계 없음.
