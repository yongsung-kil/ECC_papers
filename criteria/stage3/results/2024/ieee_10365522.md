### B. 알고리즘 요약 (10줄 내외)

1. CCSDS Near-Earth 표준 C2 LDPC 코드(rate 7/8, `n=8176`) 기반 rate-compatible(shortening) 코덱, AWGN + BPSK 채널.
2. 기존 고정 길이(fixed-length) 전송은 shortening 시 무효 padding bit(`Ipad`)를 그대로 전송해 저rate에서 valid throughput이 급격히 감소하는 문제(`TPLoss`, 식 (3)).
3. 부호화·복호화 지연을 늘리지 않으면서 무효 padding bit를 제거하는 것이 목표.
4. 기법 1: parity check matrix H(2×16 블록)와 생성행렬 B(14×2 블록)를 블록 단위로 나누고, shortening 시 동일 번호 블록을 무효 블록으로 지정해 아예 전송하지 않는 Block Tag 기반 non-fixed-length 프레이밍.
5. Block Tag(`Tg0`=정보블록, `Tg1`=패리티블록, `Ntag=33`, N-modular redundancy로 오류정정)를 이용해 디코더가 프레임 길이를 스스로 판별.
6. 기법 2: Bit-Flipping(BF) 복호를 `V(t) = -V(t-1)` 갱신식(식 6a/6b)으로 재구성해, 매 iteration마다 전체 곱셈을 재계산하지 않고 이전 iteration과 1비트 차분만 갱신하는 single-bit decoding chain(3단: accumulation-argmin-flipping)으로 단축.
7. 이 단축된 iteration chain 덕분에 CNU는 최초 1회만 곱셈을 수행하고, 이후에는 단일 CNU + 다중 VNU 구조의 미니어처 TDM 멀티플렉서로 여러 프레임을 무간격 병렬 처리.
8. 별도 학습/최적화 절차 없음(고정 알고리즘 설계).
9. 결과: 인코더 TPLoss 1.22(기존 대비 최저), 디코더 TPLoss 1.22, TDM 구조가 기존 멀티플렉서 대비 LUT 22.9%·FF 34.2% 절감, EbNo>3dB에서 rate 0.313 무오류.
10. 한계: 복호 알고리즘이 hard-decision 1-bit Bit-Flipping에 한정되어 min-sum/soft-LLR 계열 고성능 디코더로 일반화되지 않고, 통신(AWGN/CCSDS) 프레이밍 문제에 특화되어 NAND 채널 특성(고정 페이지, read-retry 등)과 직접 관련 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Block Tag 기반 non-fixed-length 프레이밍 | 대응 없음 | NAND ECC는 고정 페이지 크기 프레이밍이라 통신용 가변 길이 전송 개념 자체가 부재 |
| Bit-Flipping 단축 iteration chain (식 6a/6b) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반 soft decoding이라 hard 1-bit BF 알고리즘과 근본적으로 다른 계열, 아이디어 참고만 가능 |
| TDM 멀티플렉서(단일 CNU + 다중 VNU) | `decoder.cpp` (pipeline 구조) | 다중 프레임 병렬 처리 개념은 Prime ECC의 pipeline/병렬화와 목적이 다름(멀티 코드워드 아님) |
| CCSDS C2 부호(QC-LDPC, rate 7/8) 구조 | `ecc_top.cpp` `Load_PCM()` | 특정 통신 표준 부호 고정, NAND용 H-matrix와 무관 |

적용 가치: **낮음** — 통신 표준(CCSDS) 대상 가변 길이 프레이밍/멀티플렉싱 최적화이며, 복호 알고리즘도 hard-decision BF 계열로 Prime ECC의 soft min-sum 디코더와 이질적이라 이식 요소가 거의 없음.
