### B. 알고리즘 요약

1. 시스템은 binary linear block code `C(N,K)`, AWGN 채널, BPSK 변조 가정, 대상은 uRLLC용 short LDPC(N=96, 128 등).
2. 기존 BP는 short LDPC의 짧은 사이클(short cycle) 때문에 MLD 대비 큰 성능 격차(예: (128,64) 코드에서 NA bound 대비 1.75dB)를 가지는 문제.
3. 핵심 가정은 Tanner graph의 girth `g`가 짧아 짧은 반복 후 메시지 상관(correlation)이 발생한다는 것.
4. 핵심 기법은 mBP-OSD: 원래 BP를 `Tmax` 반복 수행 후, 정지조건 불만족 시 offset 파라미터 `β`를 곱한 modified BP를 `α`번 추가 수행, `Mji = λi + β(Σ Eij)` (식 11).
5. 이 mBP는 코드워드 추정이 목적이 아니라 OSD 입력(MRP, most reliable positions)의 신뢰도를 높여 OSD 차수(order) `m`을 낮추는 것이 목적 — order-(m-1) mBP-OSD가 order-m OSD와 동등 성능.
6. `α`는 Tanner graph girth로 결정: `α = ⌈g/4⌉ + 1` (식 14), 짧은 반복에서 정지시켜 그래프 사이클에 의한 상관 누적을 방지.
7. 정지조건은 `xH^T=0` AND 가중 해밍거리 `Dx ≤ λ` (식 9~10) — short code는 `xH^T=0`만으로는 미검출 오류가 많아 `Dx` 조건 추가.
8. `β`는 실험적으로 결정(OSD order·SNR에 따라 최적값 다름, order-1/2/3에서 각각 0.65/0.6/0.5), density evolution 이론 분석은 범위 밖.
9. 결과: (128,64) CCSDS 코드에서 order-3 mBP-OSD가 order-4 OSD(근사 MLD) 성능을 달성하면서 디코딩 시간 30ms(order-3 OSD 200ms, order-4 OSD 3000ms 대비 대폭 감소).
10. 한계: HW 미설계, Matlab 시뮬(BP는 병렬화 미적용) 기반 decoding time 측정뿐이며, OSD의 Gaussian elimination·순열 연산이 짧은 코드(N≤128) 전제로 설계되어 NAND ECC급 대형 QC-LDPC로 확장 시 복잡도가 검증되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| modified BP (offset β 적용 message passing) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | offset 스케일링 아이디어는 min-sum CN 연산에 부분 이식 가능 (단, 논문은 tanh 기반 sum-product이며 Prime ECC는 min-sum 근사 사용) |
| 정지조건(stopping criterion, xH^T=0 & Dx≤λ) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 개념적으로 조기종료 아이디어와 유사하나 논문의 WHD 기반 조건은 CRC와 다른 방식 |
| OSD (Gaussian elimination, TEP 탐색) | 대응 없음 | short code 전용 near-MLD 후처리로 대규모 QC-LDPC HW 디코더 구조와 정합 없음 |
| iteration별 LLR 스케줄(β 값 조정) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 적응형 LLR 테이블 개념과 유사하나 논문은 offset 상수 하나만 사용 |
```

적용 가치: 중간 — OSD 자체는 이식 난이도가 높고(§4 "부호설계/OSD 후처리"에 대응 없음) short code 전용이지만, mBP의 offset 기반 extrinsic scaling과 WHD 기반 조기종료 아이디어는 기존 min-sum/조기종료 모듈 개선에 참고할 여지가 있음.
