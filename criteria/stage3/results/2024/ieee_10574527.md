### B. 알고리즘 요약 (10줄 내외)

1. 5G NR QC-LDPC(BGM-1, N=3808, base rate R=1/3, 확장 rate 2/5~8/9)를 BPSK-AWGN 채널에서 stochastic decoder(SD)로 디코딩하는 시스템.
2. 문제의식: 실제 채널은 필터링/샘플링/클록지터 등으로 인한 상관(colored) 잡음이 존재하는데, 기존 SD/BP 디코더는 백색잡음 가정이라 성능 저하되고, whitening은 긴 코드에 대해 행렬곱 부담이 큼.
3. 채널 모델: 잡음을 AR(1) 프로세스 `n_s = C·n_{s-1} + w_s`(상관계수 `C`)로 모델링, 공분산행렬은 Toeplitz 구조.
4. 핵심 기법 1단: SD가 1차 복호와 함께 채널잡음 추정치 `n̂`을 산출 → 1D CNN(5개 conv layer, ReLU)이 `n̂`을 입력받아 잡음의 자기상관 패턴을 특징으로 학습해 보정된 잡음 `n_c`를 출력.
5. 핵심 식: `ŷ = y - n_c = x + r_c`(잔여잡음 `r_c = n - n_c`)로 LLR을 갱신 후 SD 재실행 — 반복(iteration) 구조로 SD와 CNN을 순환.
6. 확장: SD-CNN을 여러 회 반복((SD-CNN)_1 → (SD-CNN)_4)하면 성능이 추가 개선되나 4회 이후 포화.
7. 부수 트릭: 커스텀 손실함수 `T = ||r_c||²/N + ρ(S² + (C-3)²/4)`로 잔여잡음의 왜도(S)·첨도(C)를 정규분포에 맞추도록 정규화, He-normal 초기화, Adam 옵티마이저.
8. 학습/최적화 절차: TensorFlow 기반, SNR 0~4dB 범위 데이터로 mini-batch(1200블록) 학습, 상관계수 C와 스케일 인자 ρ를 그리드 탐색(C=0.9일 때 ρ=0.1 최적).
9. 결과: N=3808, R=1/3, 강한 상관(C=0.9) 조건에서 BER 1e-6 기준 SD 대비 SNR 2.9dB 개선(3.5dB→0.6dB), 4회 반복 시 추가 0.5dB, custom loss가 MSE/MAE 대비 0.3dB 우수.
10. 한계: HW 구현/합성 없이 순수 소프트웨어(TensorFlow/MATLAB) 시뮬레이션뿐이며, 대상이 stochastic decoding(bit-stream 기반)이라 Prime ECC의 min-sum LLR 구조와 근본적으로 다르고, colored noise라는 특정 채널 가정에 한정됨.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic decoding (SVN/SCN, 비트스트림 comparator) | 대응 없음 | Prime ECC는 min-sum LLR 기반 디코더이며 stochastic bit-sequence 방식의 SVN/SCN 구조 자체가 없음 |
| CNN 기반 채널잡음(colored noise) 추정·보정 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 채널 잡음/LLR 보정이라는 목적은 유사하나, Prime ECC 채널모델은 AWGN/RBER/fixed-error이며 AR(1) 상관잡음·CNN 추론을 다루지 않아 직접 대응 없음 |
| 5G BGM-1 QC-LDPC 부호구조(rate matching) | `ecc_top.cpp` `Load_PCM()` | 표준 5G QC-LDPC 구성법으로, Prime ECC의 고정 H-matrix 로드 위치와 레이어만 유사, 실제 rate-matching 로직은 대응 없음 |
```

마지막에 한 문장으로 **적용 가치**: `낮음` — stochastic decoding 자체가 Prime ECC의 min-sum 아키텍처와 이질적이며, CNN 기반 colored-noise 보정도 NAND 채널(RBER/fixed-error 중심) 가정과 맞지 않아 이식 실효성이 낮음.
