### B. 알고리즘 요약

1. 대상은 SC-LDPC(spatially coupled LDPC) 코드로, syndrome former memory `ms`와 coupling length `t`를 갖는 QC 구조 PCM `HSC`를 사용하며 code rate `RSC`는 식 (2)로 결정.
2. 문제의식은 short cycle(특히 길이 4, 6)이 waterfall/error-floor 성능을 저하시키므로, girth 8을 보장하면서도 유연한 code rate를 갖는 SC-LDPC 구성법이 필요하다는 점.
3. 핵심 가정은 AWGN + BPSK 채널이며, 기존 GCD 기반 QC-LDPC 구성(3×3 submatrix constraint, girth 8 보장)을 SC-LDPC로 확장.
4. 핵심 기법은 FLRM(full-length row-multiplier) 지수행렬 `PFLRM`(J×L, `a0<a1<...<a_{J-1}`)을 4개 서브행렬(A1,A2*,B1,B2)로 truncate하고 A2*를 B2 아래로 shift한 뒤 T번 복제해 coupled 행렬 `PSC,T`를 구성(그림 1,2).
5. Lemma 1,2/Theorem 1로 row/column interleaving `π(P)`과 coupled 행렬 `PSC,T`가 원 행렬 P와 동일한 cycle 분포를 가짐을 증명 — 즉 girth 8이 coupling 후에도 유지됨을 보장하는 핵심 근거.
6. 확장 기법으로 CPM 크기 하한을 줄이는 탐색 알고리즘(Algorithm 1)을 제시: 초기값 `Z+ = (a_{J-1}-a0)(L-1)+1`에서 exhaustive search로 `Zmin`을 축소(예: (5,10) FLRM에서 208→143).
7. 부수 트릭으로 code rate 손실을 줄이기 위해 서브행렬 A1의 row 수 `n1`을 J 내에서 유연하게 배분하는 구조(식 10)를 사용.
8. 복호는 sliding window decoding(SWD, W=window size, s=sliding step)으로 수행하며 window 내에서만 표준 BP(식 11~13, `2tanh^-1(...)` 형태의 min-sum 아님, sum-product) 갱신.
9. 결과: W=8, code rate 0.45/0.625 기준 RS 구조 대비 BER 10^-6에서 1.05dB/1.0dB, BLER 10^-4에서 1.1dB/1.25dB 이득. 평균 iteration 수도 SNR 증가에 따라 더 빠르게 감소(그림 7).
10. 한계: HW 구현·gate count·throughput 분석 없음, sum-product(tanh) 기반 BP만 시뮬레이션되어 min-sum 근사·양자화 영향은 미평가, 검증 코드 rate가 0.45~0.625로 NAND 전형적 고rate(~0.9) 영역과 거리가 있음.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |

| CPM 크기 하한 탐색(Algorithm 1) | ecc_top.cpp `Load_PCM()` | QC 구조 circulant shift 최적화에 참고 가능하나 코드 자체는 별도 구현 필요 |
| Sliding window decoding(SWD) | 대응 없음 | Prime ECC는 block 단위 min-sum 디코더(decoder.cpp)로 SWD 개념 없음 |
| BP(sum-product) 반복 갱신(식 11~13) | decoder.cpp `LDPC_Decoder()`, `C2V_Cal()` | 알고리즘 갱신 위치는 대응되나 Prime ECC는 min-sum(HW용) 근사를 쓰므로 sum-product 자체는 직접 이식 불가 |
```

> 적용 가치: 중간 — girth-8 보장 QC 구성법 아이디어는 부호설계 참고 가치가 있으나, coupled 구조·저rate·SWD 특화로 재설계 부담이 커 즉시 이식은 어려움.
