### B. 알고리즘 요약

1. 대상은 QC-LDPC 코드(base-matrix `Hb`, CPM 크기 `Z`, row weight `ρm`, column weight `γn`)이며 5G-LDPC BG1(정보비트 길이 `22×Z`, Z=384)을 예시로 사용, AWGN+BPSK 채널.
2. 문제의식은 초고처리량·저지연 LDPC 디코더가 필요하나 기존 NMS 기반 구조는 블록 내 직렬 처리(`Z` clock/iteration)로 처리량이 블록크기·주파수에 종속되어 향상이 어렵다는 점.
3. 기반 알고리즘은 normalized min-sum(NMS): C2V `r_mn[k] = Π sign(q) × α × min|q|`(식 5), V2C `q_mn[k] = Σ r_m'n[k] + Q_n[0]`(식 6), 최종 `Q_n[k]`로 하드결정.
4. 1단계 기법은 synchronous partially parallel(SPP) NMS 구조: 한 iteration을 `Z` clock cycle로 나누고 `mb`개 CNPU와 `nb`개 VNPU가 매 clock마다 동시에 C2V/V2C를 갱신(식 8,9)해 기존 NMS 대비 iteration당 지연을 `2Z`→`Z`로 절반 단축.
5. 처리량 식 `η = 22×Z×f_clk / ((k_max+1)×τ_iter)`(식 10)에서 τ_iter=Z이므로 SPP 구조의 throughput은 블록길이에 무관하게 주파수·최대반복수로만 결정됨(짧은 블록에서는 향상 한계).
6. 2단계 핵심기법은 matrix splitting: base-matrix의 각 CPM 블록을 s^2개 소블록으로 분할하고, 순환구조가 깨진 부분은 padding(Q-RAM 쪽은 최댓값, R-RAM 쪽은 0으로 채움)해 min/합산 연산에 영향 없도록 함.
7. 구현 디테일: 분할 시 CPM shift값을 `y = x mod (Z/s)`로 재계산, RAM을 s^2등분해 QR-RAM 모듈로 묶고 CNPU/VNPU 수를 배가시켜 iteration당 지연을 `Z/s`로 단축, throughput `η = 22×s×f_clk/(k_max+1)`(식 11)로 s에 비례 증가.
8. 실장: Virtex-7 XC7VX690T FPGA, 6-bit 양자화, Z=384를 s=12 분할해 코어행렬 4×26 → 48×312(Z=32)로 확장, Block RAM 낭비 방지 위해 Distributed RAM 일부 사용.
9. 결과: 10 iteration·253MHz 기준 SPP decoder 0.506Gbps → Splitting(s=4) 2.024Gbps → Splitting(s=12) 6.072Gbps, 기존 PipeShift decoder(2.65Gbps) 대비 2배 이상 처리량을 더 적은 비-BlockRAM 자원으로 달성. s=12에서 자원은 SPP 대비 약 3~4배 증가에 그침(throughput은 12배).
10. 한계: BER/BLER 등 오류정정 성능 곡선 제시 없이 순수 HW 처리량/지연/자원 비교에 집중, 코드 rate·length 구체 수치는 본문 서술에 없음(BG1 예시만 언급).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS(min-sum) C2V/V2C 갱신(식 5~7) | decoder.cpp `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 알고리즘 구조 자체는 이미 보유(중복 기여) |
| SPP(synchronous partial parallel) 구조, C2V/V2C 동시 갱신 | decoder.cpp `LDPC_Decoder()` 이터레이션 루프 | 클록당 CN/VN 동시 처리 스케줄링은 Prime ECC의 pipeline 구조와 비교 참고 가능 |
| Matrix splitting(블록을 s^2개로 분할+padding) | decoder.cpp `LDPC_Decoder()`, HCU(비대칭 degree col 처리) | Prime ECC의 z_sb=32 고정 병렬단위를 세분화하는 아이디어로 HW throughput 개선에 직접 참고 가능(이식 난이도 낮음~중) |
| Q-RAM/R-RAM 분할 및 QR-RAM 모듈 재구성 | 디코더 HW 모델(z=32, GT, 메모리 라우팅) | 메모리 분할·라우팅 재설계 아이디어로 HW arch 레이어에 대응 |
| 6-bit 양자화 | ecc_data.h `PARAM_LLR`, decoder.cpp `Get_VNU_Table_Idx()` | LLR 비트폭 설계 참고용이나 Prime ECC는 HD/2SD/3SD 별도 테이블 사용 중이라 직접 대체는 아님 |
```

> 적용 가치: 높음 — 알고리즘(NMS) 자체는 이미 보유하고 있으나 matrix splitting을 통한 처리량 확장 기법은 Prime ECC의 고정 병렬단위(z_sb=32) 구조에 자원 대비 처리량 이득을 주는 실질적 HW arch 아이디어로 이식 난이도가 낮음.
