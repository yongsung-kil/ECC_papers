### B. 알고리즘 요약

1. 시스템: causal encoder가 시간 t마다 정보블록 스트림 m[1,t]를 코드워드 c_t로 매핑하는 anytime 통신(networked control 등 delay-sensitive 응용) 모델, BEC/BI-AWGN 채널.
2. 문제: 기존 anytime 코드 설계는 delay-exponent α(점근적 지표)만 최적화했으나, 유한길이에서는 "erasure floor(irreducible level)"가 성능을 지배하며 이는 protograph 설계로 개선 가능함을 지적.
3. 채널 모델: BEC(erasure probability ϵ) 및 BI-AWGN, anytime reliability 조건 `Pe(p,t) ≤ βe^(-αd)` (d=디코딩 지연).
4. P-anytime 코드: bc×bv protograph를 exponential coupling 분포 `Pr(k)=e^{-kλ}(1-e^{-λ})/(1-e^{-wλ})`로 spatial coupling하여 convolutional base matrix 구성.
5. 핵심 식: density evolution (5)-(6)으로 erasure probability 재귀 계산, α는 asymptotic decay line의 지수 감쇠율(식 9).
6. 확장 지표: anytime mean parameter γ — 매 iteration의 erased VN 감소량 r1_l을 정규화해 추적, γ=min r1_l — irreducible level(floor)을 예측.
7. 구현 디테일: threshold δ=10^-3 도달 시 디코딩 종료로 간주, 최대 지연 d_max=50까지 시뮬.
8. 최적화 절차: 다목적 차분진화(mutation-crossover-selection, Pareto 지배 기반)로 {-α, -γ} 동시 최소화하여 protograph base matrix B 탐색.
9. 결과: 제안 코드가 기존 대비 BER/BEC 성능 1~2 자릿수 개선, block length 40bit의 짧은 길이에서도 erasure floor를 10^-7까지 억제.
10. 한계: HW 미설계, 순수 시뮬레이션 기반이며 BEC/BI-AWGN 특화 이론 구성법으로 NAND read channel 모델과 무관.
