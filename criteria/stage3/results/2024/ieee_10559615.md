> 총평: Golomb ruler 기반 primitive polynomial로 rate-compatible LDPC를 구성하는 새로운 부호족(non-QC)으로 인코딩 단순성과 최소거리 이론 분석이 강점이나, Prime ECC의 QC-LDPC 구조와 근본적으로 다르고 정정능력 개선(대비 5G raptor-like) 없이 유사~열위 성능이라 직접 이식보다는 참고용 개념적 가치.

### B. 알고리즘 요약

1. 대상은 이진 LDPC 부호(비 QC), BPSK/AWGN 채널, LLR-SPA(최대 100~200 iteration) 디코딩, code rate 1/2~고rate(5G raptor-like 비교군 포함), block length는 예제별로 128~1024대(k=512 기준).
2. 풀려는 문제: 기존 RC-LDPC(rate-compatible)는 QC 구조나 별도 base matrix 설계가 필요해 복잡한데, 단일 parity-check 다항식만으로 puncturing/shortening 비트 단위 rate 적응이 가능한 저복잡도 대안을 원함.
3. 설계는 simplex code(단일 primitive polynomial `h(x)` 기반 순환부호)에서 출발, puncturing으로 rate를 높이고 필요시 shortening 병행.
4. 핵심 기법: `h(x)`의 계수 지지집합 `e`가 **Golomb ruler**(모든 pairwise 간격이 서로 다름)이면 parity-check 행렬의 Row-Column Constraint(길이-4 사이클 없음)가 필요충분조건으로 성립(Theorem 1).
5. 핵심 식은 평균 열가중치 `⟨wc⟩ = (1−R)wh` (식 1) — rate가 높을수록/wh가 작을수록 반복복호 복잡도가 낮아짐을 뜻함; 최소거리 상한은 separations `s_i`(연속 지수 간격) 크기 비교로 결정(Lemma 1~4, Corollary 1~3).
6. 확장: 부호는 pseudo-noise 시퀀스 `p`(길이 `2^k-1`) 위 슬라이딩 윈도우로 표현되며, 낮은 가중치 코드워드가 `p`의 특정 국소 구간(h*의 주변)에서만 발생한다는 것을 증명해 최소거리 예측 알고리즘화(전체 탐색 없이 근사 가능).
7. 부수 트릭: 인코더는 단일 shift-register 기반 O(n) 회로(QC-LDPC의 다중 shift-register보다 단순); 코드 생성은 Golomb ruler 탐색→primitive polynomial 검증 알고리즘(Section IV-C)으로 반자동화.
8. 최적화 절차는 GA/NN 없이 Golomb ruler 문헌표(최대 65000 marks) 참조 + 조합 탐색(polynomial primitivity test)만 사용.
9. 결과: k=512에서 5G raptor-like 코드 대비 Eb/N0 손실 항상 0.2dB 이내, 일부 저Eb/N0 구간에서는 BER 더 우수; 짧은 코드(n=128, R=1/2)에서는 AR3A/ARJA/CCSDS 대비 근소 열위이나 인코딩 단순성으로 상쇄 주장.
10. 한계: HW 미설계, gatecount/throughput 실측 없음, shortening 패턴이 랜덤 선택(최적화 여지 명시), non-QC 구조라 기존 QC-LDPC HW 파이프라인과 직접 호환 안 됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix/부호구조(단일 다항식 기반 PRC-LDPC) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base matrix+circulant, `PCM_b`)로 논문의 non-QC 대각구조와 근본적으로 다름, H-matrix 전체 재설계 필요(이식 난이도 "높음"에 해당) |
| rate-compatible puncturing/shortening | `ecc_top.cpp` `Load_PCM()`, `encoder.cpp` `Generate_PCM_encoding()` | Prime ECC는 이미 고정 punct_col_num=2의 특정 QC-LDPC로 다중 rate 지원 목적이 다름, 개념적 참고만 가능 |
| LLR-SPA 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Min-Sum 기반, 논문은 SPA 기준이라 디코더 알고리즘 자체 기여는 없음(대응 없음에 가까움) |
| 인코더(단일 shift-register, O(n)) | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 고정 인코더, 논문의 단일 다항식 인코더는 구조가 달라 직접 대응 어려움 |

적용 가치: **낮음** — non-QC 단일 다항식 기반 부호 구조라 Prime ECC의 QC-LDPC 아키텍처와 근본적으로 이질적이며, 정정능력 개선도 없어(5G 대비 유사~열위) 부호설계 관점의 개념적 참고 이상의 실용 이식 가치는 제한적.
