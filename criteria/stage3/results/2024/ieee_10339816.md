### B. 알고리즘 요약

1. 정보열을 길이 `k`의 `L`개 블록으로 나누고, basic code `C[n,k]`(반복코드/SPC코드/Hamming코드)로 각 블록을 부호화하는 block-oriented convolutional 구조.
2. 기존 BMST(비재귀)와 rBMST(재귀) 방식은 각각 waterfall/error-floor 트레이드오프와 큰 디코딩 지연 문제가 있어, 두 연결을 결합한 BiBMST(bidirectional)를 제안.
3. 채널 모델은 BEC(erasure) 및 AWGN+BPSK 두 가지로 분석.
4. 인코딩메모리 `m`에 대해 2m개의 인터리버 `Πi`를 사용, `z(t)=v(t)+Σw1(t-i)`, `c(t)=v(t)+Σ(w1(t-i)+w2(t-i))`로 비재귀+재귀 중첩 전송.
5. 생성행렬/검사행렬을 유도(`GBiBMST`, `HBiBMST`)했으며, m>1일 때 두 행렬 모두 조밀(dense)해짐 — SC-LDPC(희소 H)와 근본적으로 다른 구조.
6. PEXIT(protograph-EXIT) 차트로 BEC 상 반복복호 임계값을 유도, 노드 `+`/`=`의 전달함수(`fplu`, `feq`)를 정의해 반복 계산.
7. 슬라이딩 윈도우 디코더(delay `d`)와 entropy 기반 조기종료 사용, m=2가 유한길이 영역에서 최적으로 관찰됨.
8. 성능 한계 극복을 위해 연결계수 일부를 0으로 두는 PC-BiBMST(부분연결)를 제안하고 PEXIT로 연결패턴 최적화(초과탐색 22m 회피).
9. 결과: PC-BiBMST가 동일 디코딩 지연에서 rBMST, (3,6)/(4,8) SC-LDPC, SC-GLDPC 대비 waterfall+error-floor 동시 우위, 복잡도도 SC-LDPC와 유사~약간 낮음.
10. 한계: LDPC가 아닌 basic code(반복/SPC/Hamming) 기반 구성이며 HW 미설계, BEC/AWGN 시뮬·이론 위주로 실제 NAND 채널·양자화 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BiBMST/PC-BiBMST 부호구조 (basic code+비재귀/재귀 중첩) | 대응 없음 | binary QC-LDPC가 아닌 별개 코드 패밀리, `ecc_top.cpp Load_PCM()`과 구조적으로 무관 |
| 슬라이딩 윈도우 반복복호 + entropy 조기종료 | 대응 없음 | LDPC min-sum 계열이 아닌 `+`/`=`/`C` 노드 기반 peeling/BP형 복호, `decoder.cpp`와 알고리즘 불일치 |
| PEXIT 기반 연결패턴 최적화 | 대응 없음 | 부호설계 단계 최적화 도구이며 QC-LDPC H-matrix 설계와 무관 |

> 적용 가치: 낮음 — LDPC가 아닌 BMST 계열 코드의 새 변형으로, Prime ECC의 binary QC-LDPC 부호/디코더 구조와 근본적으로 다른 패러다임.
