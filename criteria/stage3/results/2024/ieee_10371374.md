### B. 알고리즘 요약

1. 3D MLC/TLC NAND, LDPC 부호율 8/9, codeword당 정보길이 2KB(페이지당 8 codeword), quasi-cyclic PCM(열가중치4/행가중치36) 사용.
2. 문제: retention time·P/E cycle 증가로 RBER이 상승해 hard-decoding iteration이 급증하고, 결국 soft-decoding(추가 read level) 필요해져 read latency가 악화.
3. 가정: threshold voltage drift 모델(Gaussian 상태분포)로 상태 간 천이 확률을 모델링, MSB 페이지의 상태 `10→00` 천이 비율이 유독 높다는 실험 관찰 기반.
4. 핵심 기법: 기존 hard-read voltage `Vm2`(상태 00/10 중간) 대신, 천이 전후 평균 `μs1a0`, `μ0sa0` 정보를 반영한 `Vp`(PEAR read voltage)를 식(4)로 계산해 재배치.
5. 식(4) `Vp = 2Vm2 - |μs1f0-μs1a0|/2 - |μs0f0-μs0a0|/2` — 상태분포 이동량 절반을 반영해 두 상태 간 오독 확률(Ep1+Ep2, 식 5~7)을 최소화하는 지점을 찾음.
6. 확장 없음(단일 read voltage 재배치가 전부, 상태 11/01 간 read voltage는 기존 값 유지).
7. 구현 디테일: LSB 페이지는 기존 hard-read 그대로, MSB 페이지만 retention/P/E cycle이 임계값을 넘을 때 PEAR read 적용(Algorithm 1).
8. 별도 학습/최적화 절차 없음(closed-form 식 계산).
9. 결과: 10K P/E cycle 기준 RBER이 2.0×10⁻²→1.5×10⁻³로 감소, MSB hard-decoding iteration이 LSB 수준으로 근접, soft-decoding(2HR+2SR/2HR+4SR) 회피로 decoding latency 최대 60μs→3μs 수준 개선.
10. 한계: 순수 시뮬레이션(HW 미설계), 디코더는 기존 normalized min-sum 그대로 사용, threshold voltage drift 모델([5])에 의존한 파라미터 산정으로 특정 공정/소자 가정에 종속.
