### B. 알고리즘 요약

1. 시스템은 6G immersive communication 목표 rate-compatible QC-SC-LDPC 코드(coupling length L=36, W=4, dv=5, rate 2/3~5/6 계열), lifting factor Z=768, BI-AWGN 채널.
2. 문제: 5G-NR LDPC의 highly irregular raptor-like base graph가 row-parallel HW 구현 시 라우팅 오버헤드를 유발해 고throughput(Tbps급) 확장에 한계.
3. 가정/모델: BI-AWGN 채널, check-ratio(현재 iteration에서 unsatisfied check node 비율)를 디코딩 상태 지표로 사용하는 모델.
4. 핵심 기법 1단: regular edge-spreading 패턴의 QC-SC-LDPC 구조(같은 row/column weight 유지)로 row-parallel 디코더에 최적화, nested base graph로 rate-compatibility(2/3~5/6) 확보. layered-scheduling + sliding window decoding(SWD, window size `W`, iteration `I_iter^win`)으로 복잡도 축소.
5. 핵심 식: throughput_row-parallel ∝ `(N/M)·(µb/µr)·dv` (block-parallel 대비), 그리고 SWD 적용 시 `µ_r^SWD < µ_r`로 라우팅망 단순화 → 최소 14배 throughput 이득.
6. 핵심 기법 2단(AQNMSA): CNU 입출력에는 저비트폭 `q_L=4`와 check-ratio 기반 적응형 양자화 구간 `T_cur`, VNU 입출력에는 고비트폭 `q_H=9`와 고정구간 `T0` 적용, iteration마다 check-ratio를 LUT 인덱스로 사용해 `(T, α)` 갱신.
7. 부수 트릭: LUT 크기 224bit(qH=9,qL=4,qchk=5,qT=4,qα=5), VNU/CNU 비대칭 비트폭으로 내부 데이터흐름 F=2·n_c·q·d_v를 최소화(F는 5G-NR SPA 대비 2배 수준).
8. 최적화 절차: multi-edge-type density evolution(MET-DE) 도구로 LUT의 (check-ratio, T/T0, α) 조합을 설계.
9. 결과: 4-bit AQNMSA가 고정 4-bit NMSA 대비 0.2dB 문턱 이득, float-point SPA 대비 0.3dB 손실에 그침, 5G-NR LDPC(SPA) 대비 오히려 더 나은 asymptotic threshold(2.61dB vs 2.69dB), row-parallel+SWD 구조로 총 30배 이상 throughput 이득.
10. 한계: 이론(MET-DE)+시뮬(BLER)만 존재, 실제 RTL/ASIC 합성이나 gatecount·실측 throughput 데이터 없음. 미래연구로 3-bit 이하 양자화 및 iteration 추가 감소를 명시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check-ratio 기반 적응형 양자화/정규화(AQNMSA) | `decoder.cpp` `CNU_Update_New_Mag()`, `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 임계값 테이블과 개념 직결 - check-ratio를 iteration 상태지표로 쓰는 LUT 갱신 로직 참고 가능 |
| VNU/CNU 비대칭 비트폭(qH=9, qL=4) | `decoder.cpp` `VN_Cal_HD()`, `decoder.h` `Get_VNU_*` | soft-2~3bit 저비트 양자화 설계가 Prime ECC 2SD/3SD 양자화 레벨과 비교 가능 |
| layered-scheduling + sliding window decoding | `decoder.cpp` `LDPC_Decoder()` | SC-LDPC 특유의 슬라이딩 윈도우는 Prime ECC QC-LDPC(비SC) 구조에 직접 대응 없음 |
| row-parallel 라우팅/shift network 설계 | 디코더 HW 모델(z=32, HCU, GT) `GT.cpp` `Make_GT_HW()` | 라우팅 단순화를 위한 regular edge-spreading 아이디어가 GT의 edge 축소와 목적 유사 |
| QC-SC-LDPC 부호구조(H-matrix, rate-compatible) | `ecc_top.cpp` `Load_PCM()` | 부호설계 레이어 - SC 구조 자체는 대응 없음, 부호 교체는 난이도 높음 |
```

적용 가치: `높음` — AQNMSA의 check-ratio 기반 적응형 비대칭 양자화·정규화 기법은 Prime ECC의 적응형 LLR 테이블 구조와 직접 대응되어 min-sum 디코더 레이어에 이식 시 저비트폭에서도 성능 개선(waterfall+error-floor 동시)이 기대됨. 다만 SC-LDPC 부호구조 자체의 이식은 별개로 난이도가 높음.
