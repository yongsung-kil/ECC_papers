### B. 알고리즘 요약

1. 6G 통신 표준용 통합 채널코드를 목표로, 단일 base matrix에서 rate `1/3~9/10`, length `32~2440`(HRC/IRC 조합, power tower로 최대 64800까지) 유연성을 갖는 QC-PBRL(protograph-based raptor-like) LDPC 코드 설계.
2. 문제: 5G LDPC는 punctured message bit(state variable)로 고rate를 달성하지만 디코더 수렴이 느리고 무오류 시에도 불필요한 복호가 필요해 에너지 비효율적, 짧은 길이에서는 polar SCL이 더 우수하나 BP처럼 반복횟수로 성능-복잡도 트레이드오프를 조절할 수 없음.
3. 부호는 `H = [[H_HRC, 0], [H_IRC, I]]` 구조(highest-rate code + incremental redundancy code), regular·4-cycle-free(RD-constraint, 식 1) HRC와 irregular 확장부 IRC로 구성, QC 리프팅(circulant shift `w_i,j`, factor Z).
4. 핵심 기법 1: HRC를 rank-deficient(`rank(H) ≤ mZ-m+1`, Theorem 1)로 설계해 QC 대칭성을 깨는 non-equivariant PCM을 만들고, 이를 통해 AED(automorphism ensemble decoding, `π_{d,Z}(i)` 식 2)를 지원, 짧은 길이에서 다중 병렬 디코딩 후보 중 ML-in-list로 선택.
5. rank-deficient 구조가 필요한 이유: QC PCM은 자체 QC 자동사상(automorphism)에 equivariant해 순열해도 같은 디코딩 결과가 나오므로, 여분 체크 행을 제거해 비대칭 PCM을 만들어야 AED의 다양한 디코딩 경로가 유효해짐.
6. 핵심 기법 2: HRC 설계는 Algorithm 1(progressive lifting design)로 열마다 RD-constraint를 만족하는 최소 shift값을 탐색, IRC 확장은 Algorithm 2(genetic algorithm)로 행별 base graph+shift값을 joint 최적화(fitness `f(W')=-Σ_k log P̂err`).
7. 부수 트릭: variable node puncturing을 전혀 사용하지 않아(수렴 느려짐 방지) 유한 iteration에서 성능 우수, 대신 점근적 성능은 소폭 희생; power tower(`W_tower = W + Σ Δ(i)·α·2^i`, 5G 방식 차용)로 여러 lifting size Z를 하나의 base matrix에서 modulo 연산만으로 지원.
8. 학습/최적화: GenAlg(population 160, elite 10, crossover 54, mutation 48, tournament selection)로 IRC 확장 행을 5G LDPC 성능 기준(`Γ_5G`) 대비 Monte-Carlo BLER로 순차 최적화(Table I 하이퍼파라미터).
9. 결과: R=1/2,N=957에서 5G LDPC 대비 약 0.4dB 개선; R≈9/10,N=8960에서 0.5dB 이상 개선(DVB-S2 대비 N=64800에서 0.3dB); 짧은 길이(N≈132,R=1/4)에서 BP-8+AED-6이 5G LDPC BP-48과 유사 성능이면서 지연 감소; 전반적으로 5G LDPC 대비 평균 반복횟수 감소(특히 고rate).
10. 한계: 순수 부호설계+BP 시뮬레이션 논문으로 HW 미구현, AED의 다중 병렬 디코딩 시도에 따른 HW 면적/전력 오버헤드는 미평가, 6G 무선통신(AWGN/BPSK) 채널 가정으로 NAND 채널 특성(비대칭 오류, RBER)은 미고려.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-PBRL 단일 base matrix, rank-deficient HRC 설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 자체를 대체하는 설계라 이식 난이도 높음(Prime ECC는 특정 QC-LDPC 고정) |
| Automorphism Ensemble Decoding (AED) | `decoder.cpp` `LDPC_Decoder()` | 다중 병렬 디코딩 시도 개념은 Prime ECC의 단일 디코더 이터레이션 루프와 구조가 달라 큰 개조 필요, 짧은 길이 최적화라 NAND 응용 rate/length와도 맥락 다름 |
| puncturing-free, 유한 iteration 최적화 설계원칙 | `ecc_top.cpp` `Load_PCM()` | 적은 iteration에서 성능 최적화라는 설계 철학은 Prime ECC의 부호설계 시 참고 가치 있으나 코드 레벨 이식 대상은 아님 |
| GenAlg 기반 확장부(IRC) 최적화 | 대응 없음 | 부호 구성법 최적화 절차로 Prime ECC 디코더/인코더 모듈과 직접 대응 없음 |
| power tower(다중 lifting size 지원) | `ecc_top.cpp` `Load_PCM()` | 유연 길이 지원 기법, Prime ECC는 고정 z=32 lifting이라 개념적 참고만 가능 |
```

적용 가치: **중간** — 코드/모듈 직접 이식 대상은 아니지만, rank-deficient QC 설계로 AED를 가능케 하는 방법론과 puncturing-free·유한 iteration 최적화 설계원칙은 향후 Prime ECC H-matrix 재설계 시 참고할 만한 아이디어.
