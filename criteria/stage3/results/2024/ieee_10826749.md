> 총평: girth-8 이상 QC-LDPC를 trapping-set 인지 SA 탐색으로 빠르게 구성하는 부호설계 기법으로, 기존 PEG/ACE 계열 대비 구성시간 단축과 고SNR 영역 BER 개선을 보이나 검증 규모가 작아 이식 시 재검증 부담이 있음.

### B. 알고리즘 요약

1. Regular QC-LDPC 부호(circulant permutation matrix 기반, exponent matrix `E`, 크기 `J×L`, circulant 크기 `P`)를 대상으로, girth 8 이상을 보장하며 trapping set이 적은 부호를 구성하는 문제.
2. 기존 PEG/ACE 계열 구성법은 girth-8 QC-LDPC 탐색에 수십 시간(예: `[15]` 41열에서 23h, 63열에서 75h) 소요되어 비효율적.
3. 핵심 가정: `(5,3)` trapping set이 3개의 8-cycle을 공유하는 구조에서 파생되며, 이 8-cycle 조합을 깨면 trapping set도 함께 제거된다는 상관관계를 이용.
4. 목적함수 `CheckCyclesSharing(E)`: exponent matrix `E`에서 검출된 8-cycle들 중 두 점 이상을 공유하는 cycle 쌍의 개수(`shared`)를 비용으로 최소화.
5. `Σ_{k=0}^{l-1}(p_{ik,jk}-p_{ik+1,jk}) = 0 mod P`(식 4) 위반 여부로 길이 `2l` cycle 존재를 판정, 이를 어기는 shift 값을 후보로 열별 순차 배정.
6. SA 수락확률 `Pr = e^{-ΔE/(kT)}`(Metropolis 기준)로 지역 최적해에 빠지지 않고 전역 최적 exponent matrix로 수렴.
7. 열(column) 단위 순차 배정: 각 원소마다 4/6-cycle 없는 유효 값을 시도, 실패 시 이전 열로 backtrack(`col ← col-1`).
8. 학습/최적화 절차: SA 자체(초기온도 `T`, 냉각률 `λ`, 반복횟수)가 최적화 루프. 예: `(L,P)=(5,27)`에서 `T=1000, λ=0.99, 50회` 등 예제별 상이한 하이퍼파라미터.
9. 결과: `(6,63)` 부호에서 기존 75h → 420s로 구성시간 대폭 단축, 동일/유사 8-cycle·10-cycle 수 대비 BER은 고SNR에서 더 우수(code rate 0.4/0.5/0.7 비교, BP 50회 반복 AWGN+BPSK 시뮬).
10. 한계: HW 미설계, 순수 시뮬레이션(이론적 girth 보장 + 시뮬 BER)이며 길이 1650에서도 SA 탐색시간이 9000s로 급증해 대형 NAND ECC급(수천~수만 bit) 확장성 불명확.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| exponent matrix(H-matrix/QC 구성) SA 탐색 알고리즘 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 이전 단계의 부호설계 도구로, 기존 고정 QC-LDPC(`PCM_b`) 대체/개선 설계 시 참고 가능하나 재검증 필요 |
| 8-cycle/trapping-set 상관관계 기반 girth 최적화 | 대응 없음 | Prime ECC는 이미 고정된 base matrix+shift 구조(`N_b=129`, VN degree 17)를 사용, girth 최적화 설계 로직 자체는 코드에 없음 |
| Trapping-set 감소를 통한 error-floor 개선 | `decoder.cpp` `LDPC_Decoder()` (간접 영향) | trapping set이 적은 H-matrix는 min-sum 반복복호의 error-floor 특성에 영향을 주나, 디코더 알고리즘 자체 변경은 아님 |

> 적용 가치: 중간 — QC-LDPC 부호설계(H-matrix 구성) 단계에서 girth-8 보장과 trapping set 감소를 통한 error-floor 개선 기법으로 이론적으로 유효하나, Prime ECC는 이미 고정된 특정 QC-LDPC 구조를 사용해 재검증 부담이 크고(§4 "부호설계: 이식 난이도 높음"), 검증 규모(길이 최대 1650, regular code)가 NAND ECC급과 격차가 있음.
