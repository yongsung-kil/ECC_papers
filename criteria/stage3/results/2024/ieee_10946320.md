### B. 알고리즘 요약

1. QC-LDPC 코드는 지수행렬 `E`와 circulant size `P`로 정의되며(`H`는 CPM `I(p_ij)` 블록으로 구성), 코드 성능은 Tanner graph의 girth(최단 사이클 길이)에 크게 좌우된다(4-cycle 존재 시 성능 저하).
2. 기존 explicit(수식 기반) 구성법은 큰 circulant size에서만 다양한 row weight를 지원하는 한계가 있어, 이를 개선해 column weight 6에서 작은 circulant size로 girth-8(4/6-cycle 없음)을 얻는 explicit 구성이 목표.
3. 핵심 가정: 지수행렬 `E = S2^T · S1`, `S1=[0,1,...,L-1]`(공차 1 등차수열), `S2`는 GCD 제약 `(a_k-a_i)/gcd(a_k-a_i, a_j-a_i) > L`(Lemma 2)을 만족하는 J개 정수열.
4. 핵심 기법: column weight 6에 대해 `S2 = [0, 1, 2L+1, 2L+3, 4L+3, 4L+4]`라는 구체적 정수열을 제시(Theorem 1)하고, circulant size `P ≥ (4L+4)(L-1)+1`이면 girth-8이 보장됨을 증명.
5. `P`를 더 작게 잡을 수 있는 경험적 최적값(Property 4)도 제시: `mod(L,6)` 값에 따라 `P = 2L²+3L+1`, `2L²+L`, `2L²+4L+4`, `2L²+2L+2` 중 하나 선택.
6. 증명은 Property 1~3(시퀀스 변환 S/R/D 규칙)과 Lemma 1~3(GCD 기반 4/6-cycle 부재 조건)을 이용해 15개(4-cycle)·20개(6-cycle) 케이스를 대칭성으로 축소 후 개별 증명.
7. 부수 트릭: 시퀀스 변환 규칙(S: 평행이동, R: 역순, D: 배율)으로 동형(같은 girth) 시퀀스를 만들어 증명 케이스 수를 줄임.
8. 학습/탐색 절차 없음 — computer search를 대체하는 것이 이 논문의 핵심 주장(수식만으로 explicit 결정).
9. 결과: L=12(C1, n=3900, k=2063, rate=0.529)와 L=11(C2, n=2783, k=1356, rate=0.4872)에서 BPSK/AWGN/SPA(50 iteration) 시뮬 시, 동일 circulant size의 search 기반 symmetric-structure girth-8 코드([4])보다 BER/BLER 성능이 유의하게 우수(그림 2, 3).
10. 한계: column weight 6, 특정 row 길이(L=6~200 경험식) 범위로 검증됐고, HW 구현·복잡도·throughput은 다루지 않음(순수 시뮬 기반 부호설계 논문); 코드 rate가 0.45~0.53 수준으로 NAND ECC가 요구하는 고rate 영역과는 차이가 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GCD 기반 explicit 지수행렬 구성(girth-8, 4/6-cycle 제거) | `ecc_top.cpp` `Load_PCM()` | H-matrix(PCM) 생성 로직과 대응되나 Prime ECC는 특정 QC-LDPC 고정(재검증 부담 큼, 프로파일 §4) |
| girth/cycle 최적화로 waterfall 성능 개선 | 대응 없음 | 디코더가 아닌 순수 부호설계 단계 기법으로 decoder.cpp에는 닿지 않음 |
| computer search 대체(explicit formula) | 대응 없음 | Prime ECC 코드베이스에 부호 탐색/생성 모듈 자체가 없음(고정 PCM 사용) |

> 적용 가치: 낮음. 부호설계(H-matrix 재구성) 레이어는 이식 난이도가 가장 높은 영역(프로파일 §4)이며, 예시 rate(~0.5)도 NAND 고rate 요구와 맞지 않아 직접 채택보다는 이론적 참고 수준.
