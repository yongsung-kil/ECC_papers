### B. 알고리즘 요약

1. 대상은 3D FG(floating-gate)/CT(charge-trap) TLC NAND flash, 각 워드라인이 LSB/CSB/MSB 3개 논리 페이지로 구성, Gray code 인코딩.
2. 문제: P/E 사이클 증가와 retention time 경과에 따라 Vth 분포가 좌측으로 shift·중첩되어 고정 RRV로는 RBER이 급증하고, read-retry 반복은 latency를 크게 늘림.
3. 핵심 가정: 데이터 scrambling(랜덤화)으로 각 상태(state)의 셀 개수가 기록 시점에 균등분포를 이룬다고 가정.
4. 핵심 기법(EDTVD): 기준 전압 `V_base` 대비 조정 전압 `V_adjust`를 두고, 좌우 셀 개수 차 `Δ = |N_initial - N_base*0.5|`를 계산해 이분탐색식으로 `V_adjust`를 갱신.
5. 핵심 식: 스텝 크기 `p = f(Δ)`(Δ가 `2.5%N_base` 초과 시 4, `0.5%~2.5%N_base`면 2, 이하면 1)로 가변 스텝을 적용해 정확도와 조정 횟수(latency)를 절충.
6. 확장: FG/CT 타입별로 페이지 순서(LSB→CSB→MSB)에 따라 순차적으로 여러 RRV(`Vref1~Vref7`)를 조정하며, 이전 페이지의 최적값을 다음 페이지 조정의 기준으로 재사용.
7. 구현 디테일: 최소 스텝 7.5mV, 스텝 조합 (4,2,1)이 조정 횟수 최소로 실험적 선정, offset 범위 [-1.28V, 1.27V].
8. 별도 학습/최적화 절차 없음(규칙 기반 반복 탐색), MATLAB 시뮬레이션 후 Nanocycler FPGA 테스트 플랫폼(NPlusT)에서 실측 검증.
9. 결과: 8-fixed-step 대비 RBER 49~52% 감소(FG/CT, P/E 5000~8000, retention 최대), 고정 read 대비 RBER 85.2% 감소, 2-fixed-step 대비 조정횟수 약 21% 감소.
10. 한계: LDPC 등 ECC 디코더/부호설계는 다루지 않고 배경 설명 수준으로만 언급(RBER 감소가 LDPC iteration 감소로 이어진다는 정성적 서술뿐, 정량 결과 없음), scrambling으로 상태 균등분포 가정이 깨지면 성능 저하 가능.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RRV(read reference voltage) 동적 조정 | 대응 없음 | Prime ECC는 soft-read LLR 양자화(`channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()`)를 다루지만 채널 측 실제 RRV 물리 탐색 자체는 ECC 엔진 범위 밖 |
| RBER 감소를 통한 read-retry 횟수 절감 | 대응 없음 | ECC 앞단 채널 최적화로 decoder.cpp/channel.cpp의 알고리즘 변경과 직접 연결되지 않음 |
| LDPC iteration/latency와의 정성적 관계 언급 | `channel.cpp` `Set_LLR_Th()` (참고용) | RBER이 낮아지면 LLR threshold 테이블 설계에 유리해질 수 있다는 간접 시사점뿐, 정량 이식 요소 없음 |
```

적용 가치: **낮음** — NAND 채널의 read voltage 최적화 논문으로 Prime ECC 3.1의 부호설계·디코더·인코더 코드에 직접 이식할 알고리즘 요소가 없으며, ECC 언급은 배경 설명 수준에 그침.
