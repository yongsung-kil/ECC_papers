### B. 알고리즘 요약

1. 대상: QC-LDPC 부호의 인코더 하드웨어 아키텍처. PCM은 CPM(Circulant Permutation Matrix) 배열, 우측이 bidiagonal 구조(5G/WiFi 802.11n/ac/ax 유사)를 가정.
2. 문제의식: 기존 RU(Richardson-Urbanke) 인코딩은 다단계 연산으로 hardware critical path가 길고 유연성이 떨어짐. LU분해 방식도 H가 singular일 때 불완전.
3. 핵심 가정: PCM은 좌측 임의 QC 구조 + 우측 결정론적 bidiagonal 구조(식 6)를 가진다고 가정.
4. 핵심 기법: forward substitution(FS) 인코딩. 전체 PCM 대신 CPM 위치·shift값만 ROM에 저장해 메모리 절감. 파리티 비트 `q`를 행별 순차 재귀식(식 7, `Hj,l*ul^T + qj^T + q(j+1)^T = 0`)으로 계산.
5. 식의 의미: 파리티 계산이 이전 행의 파리티에 의존하는 순차 재귀 구조라 저장공간은 줄지만 병렬성이 제한됨.
6. 핵심 기법 2단: Cyclic Shift(CS) 모듈 중심 하드웨어. 서브행렬 크기 M/2M에 따라 CS를 하나의 긴 shifter 또는 다수의 독립 shifter로 재구성해 짧은 코드워드에 대해 클록당 최대 4개 CPM을 병렬 처리.
7. 구현 디테일: dual-port input buffer로 인접 두 코드워드의 정보비트를 동시 관리(현재 인코딩용 절반 + 다음 코드워드 프리로드용 절반)해 인코딩 클록 사이클 절감.
8. 학습/최적화 절차: 없음(구조 설계·시뮬레이션 검증뿐).
9. 결과: 길이 2304, rate 1/2 예제로 FPGA/소프트웨어 인코딩 결과 일치 검증. xc7vx485tffg1157-1 FPGA 기준 Register 1205(vs 기존 1497), FF 3505(vs 6208)로 자원 절감(Table I).
10. 한계: 전체 throughput(Gbps) 수치 미제시, ASIC 합성 결과 없음, bidiagonal 우측 구조를 가정하므로 임의 PCM에는 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FS 기반 파리티 순차 계산 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코딩이라 FS 방식과 구조가 다름, 대체 아이디어로만 참고 가능 |
| Cyclic Shift(CS) 가변 병렬 모듈 | `encoder.cpp` (dual-diagonal 고정) | 부호 PCM 변경 시 shift 네트워크 재사용 아이디어는 참고 가능하나 코드 재검증 부담 큼 |
| PCM 로드/구조(base graph + shift) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC 구조 자체는 동일 계열이나 논문은 bidiagonal 특정 구조 전제 |
| dual-port input buffer 파이프라이닝 | `encoder.cpp`, `input.cpp/.h` | 버퍼링 기법은 일반적 HW 최적화로 이식 가능성 있음 |

적용 가치: **낮음** — 인코더 전용 HW 아키텍처 논문으로 Prime ECC의 dual-diagonal 인코더와 구조가 다르고 정정능력/디코더 성능과 무관하며, 표준 QC-LDPC 인코딩 최적화 수준(FS/CS)에 그침.
