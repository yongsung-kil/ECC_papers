### B. 알고리즘 요약

1. 시스템/코드 스펙: variable-regular QC-LDPC (γ=column weight=3 또는 4, girth 6 또는 8), circulant permutation matrix 기반 parity-check matrix `H`(lifting degree p).
2. 풀려는 문제: LDPC의 error floor는 Tanner graph 내 elementary trapping set(ETS, check node degree가 1 또는 2뿐인 induced subgraph)에서 기인하며, ETS의 비동형 구조를 (a,b) 값별로 전수 열거하는 것은 NP-hard라 계산복잡도가 매우 높음.
3. 핵심 가정/모델: variable-regular Tanner graph에서 ETS는 그 VN(variable node) graph와 1:1 대응하며, VN graph에서 특정 theta graph(θ(1,2,2), θ(2,2,2))의 존재 여부로 ETS 크기를 제약할 수 있다는 그래프이론적 관찰.
4. 핵심 기법 1단: Turán number `ex(n,H)`(부분그래프 H를 포함하지 않는 n-vertex 그래프의 최대 edge 수) 개념을 도입, θ(1,2,2)의 정확한 Turán number `ex(n,θ(1,2,2))=⌊n²/4⌋`를 귀납법으로 증명(Turán 정리의 C3 케이스와 동치).
5. 핵심 식: girth 6에서 두 6-cycle이 check node를 공유하지 않으면(=VN graph에 θ(1,2,2) 없음) `b ≥ aγ - a²/2`가 성립, 이는 기존 Amirzade의 하한 `b≥aγ-2a³/(4a-3)`보다 더 tight한 결과.
6. 핵심 기법 2단: girth 8에서는 θ(2,2,2)와 C3를 동시에 배제하는 그래프의 Turán number 상한 `ex(n,{C3,θ(2,2,2)}) ≤ n(√(8n-7)-1)/4`를 diameter≤3 논증으로 유도, 이를 통해 `b ≥ aγ - a(√(8a-7)-1)/2` 하한을 도출.
7. 구현 디테일: 두 8-cycle이 check node 1개 이하만 공유하도록 하는 QC-LDPC를 lifting degree p=35, 57의 (3,5)-regular / (3,6)-regular 부호로 random search를 통해 구성(식(2)의 cycle 존재 필요충분조건으로 8-cycle 전수 검사).
8. 학습/최적화 절차: 별도 최적화 없이 조건을 만족하는 circulant shift 값을 random search로 탐색(제안 조건을 만족하지 않으면 재탐색).
9. 결과: 제안 부호(N=175, 342)가 PEG 알고리즘·random lifting으로 만든 동일 (3,5)/(3,6)-regular 부호 대비 error floor 영역 FER 우위, girth 6 조건에서 계산된 최소 a값이 기존 컴퓨터 전수열거 결과와 일치.
10. 한계: SPA(sum-product) 디코더·AWGN/BPSK 채널의 소규모 시뮬(N=175, 342, 최대 20 iteration)에만 그침, HW 설계·대형 length(NAND급 수천~수만 bit) 검증 없음, 순수 이론적 그래프 조건 도출이 목적이라 실제 코드 탐색은 random search 의존.

> 적용 가치: 중간 — error-floor를 직접 낮추는 부호설계(그래프이론적 trapping-set 회피 조건) 자체는 Prime ECC 이식 난이도가 높은 레이어(H-matrix 고정 구조 재검증 부담)이나, 8-cycle 공유 제약이라는 명시적 설계 기준은 향후 고rate QC-LDPC의 error-floor 개선 시 참고할 이론적 가치가 있음.
