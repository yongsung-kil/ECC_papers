### B. 알고리즘 요약 (10줄 내외)

1. 대상은 5G NR LDPC 코드(base graph 기반 QC-LDPC), AWGN 및 Rayleigh fading 채널, 정보비트 260/520, rate 1/2~1/5.
2. 기존 신경망 디코더는 코드 길이에 비례해 크고 깊은 네트워크가 필요해 학습 지연·자원 소모가 커서 실시간 재학습에 부적합한 문제를 해결하려 함.
3. Self-corrected min-sum(SCMS) 알고리즘(기존 기법, 부호부호 반전 시 메시지를 신뢰불가로 판정해 소거)을 기반으로 함.
4. 핵심 기법: C2V/V2C/APP 계산식(식 2,3,4)에 4개의 학습 가능 인자 `α(l), β(l)`(C2V 정규화·오프셋), `α_ch(l), β_ch(l)`(채널 정규화·오프셋)를 iteration별로 도입한 NSCMS(Algorithm 1).
5. 핵심 식: C2V 메시지 `L_C2V = sgn(·)·max(α·min|L_V2C|+β, 0)`(식 2)로, ReLU형 클리핑을 학습 인자로 미세조정해 min-sum의 근사오차를 BP에 가깝게 보정.
6. 네트워크 구조는 `Imax`개의 얕은(shallow) 1-hidden-layer NN으로 분리(식 5,6,7)해 반복별로 독립 학습, 인자는 max iteration 이하로 재사용 가능.
7. 부수 트릭: self-correction 조건(부호 반전 시 소거)으로 일부 노드를 pruning처럼 제거해 학습 연산량 감소, NR 코드는 첫 `2Z`비트(lifting factor Z) puncture로 입력층 일부도 고정 소거.
8. 학습 절차: cross-entropy loss(식 8)로 Adam optimizer(lr=1e-3), 학습 데이터 1000개·배치 100, BLER 10^-1 근방 SNR에서 iteration별 순차(greedy) 학습.
9. 결과: NSCMS가 NMS/SCMS/LAMS 대비 BLER이 BP에 더 근접, rate가 낮고 length가 길수록 성능격차 확대, 평균 iteration 수도 감소(구체 수치는 그래프에만 존재).
10. 한계: 하드웨어 미설계(부호/오프셋 저장에 비트맵 메모리 필요하다는 언급뿐), 실시간 학습 복잡도는 노드 삭제 비율(2번째 iteration 0.1~0.15)로만 정성 분석, gate/throughput 실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum C2V 정규화/오프셋 인자 학습(식 2) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime ECC의 magnitude 양자화 테이블을 학습 기반 정규화/오프셋으로 대체 가능한 지점 |
| iteration별 적응형 인자(α(l), β(l), α_ch(l), β_ch(l)) | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 기존 iteration별 LLR threshold 테이블 구조와 동일 위치, 값 산출 방식(학습 vs 수동설계)만 차이 |
| self-correction(부호 반전 시 메시지 소거) | decoder.cpp VN_Cal_HD() 등 (HD/2SD/3SD) | 신규 기법, 메시지 신뢰도 소거 로직 추가 지점으로 대응 가능하나 Prime ECC는 미보유 |
| NN 학습 파이프라인(Adam, shallow NN, pruning) | 대응 없음 | Prime ECC는 C++ bit-exact 시뮬레이터로 오프라인 NN 학습 인프라 없음, 별도 오프라인 학습 도구 필요 |
```

적용 가치: `중간` - min-sum 프레임워크 내 정규화/오프셋 인자 최적화라는 점에서 decoder.cpp의 CNU/VNU 및 LLR 테이블과 구조적으로 맞닿아 있으나, 학습 인프라(NN 학습·재학습)가 별도로 필요하고 NR LDPC(저rate) 대상이라 NAND 고rate 코드로의 이전 검증이 없음.
