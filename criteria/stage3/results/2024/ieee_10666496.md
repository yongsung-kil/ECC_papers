### B. 알고리즘 요약

1. 대상 채널은 Gallager/Davey-Mackay ID(insertion/deletion) 채널에 BS(bit-flip) 또는 BI-AWGN 노이즈가 결합된 복합 채널, 컴포넌트 코드는 binary irregular LDPC(rate 0.48~0.52) 및 (3,6,l_sc)-regular SC 코드.
2. 문제: 기존 concatenated(외부 LDPC+내부 marker/trellis) 또는 standalone LDPC 스킴은 ID 오류와 일반 노이즈 오류 모두에 범용적으로 강인한 성능을 달성하지 못함(부호 프로파일이 ID/노이즈 중 하나에 특화됨).
3. 핵심 가정: 다중 코드워드(L개)를 동시에 전송하며, 이전에 복호된 코드워드의 정보를 다음 코드워드 검출의 사전확률로 재활용하는 순차적(chained) 구조.
4. 핵심 기법 1단(HI, Hierarchical Interleaving): 각 코드워드 `c_t`를 `m`개 서브블록으로 나누고, shift vector `T=(T1,...,Tm)`에 따라 서브블록을 시간축으로 재배치, 빈 자리는 송수신자가 공유하는 known bit로 패딩.
5. 핵심 식: 전송률 `Rtx = L/(L+Tmax) · R`, 지연 `θ=(Tmax+1)n`, 검출복잡도 `O(n(Tmax+1)|B|)` — Tmax가 클수록 AIR은 향상되나 latency/복잡도 증가(전통적 concatenated code의 고정 rate loss 회피).
6. 핵심 기법 2단(CR, Chained Recovery): BCJR 기반 MAP 검출을 non-iterative(single-pass)로 수행하되, 앞서 복호된 코드워드의 extrinsic LLR을 현재 코드워드 검출의 사전확률로 사용해 marker 코드 없이도 유사한 동기화 효과 획득.
7. 부수 기법: shift vector `T`를 exhaustive search 대신 simulated annealing(SA)으로 설계해 BCJR-once rate(AIR)을 최대화, `T_opt`가 stepwise vector 대비 더 높은 AIR 달성.
8. 최적화 절차: SA 알고리즘(Metropolis 기준 accept, 온도 `τ=γ^i τ_max`로 감소)으로 set-partitioning 형태의 shift vector 최적화 문제 해결.
9. 결과: 동일 검출복잡도에서 density evolution 기반 BP threshold가 표준 standalone 대비 크게 향상, 유한길이 시뮬레이션(BER)에서 error-floor 없이 sharp waterfall 곡선, 기존 스킴 대비 우수한 성능.
10. 한계: HW 미설계·순수 시뮬레이션(density evolution+수치 BER)뿐이며, ID/동기화 오류가 없는 순수 NAND bit-flip 채널에 대한 평가는 없음. 다중 코드워드 체인 구조로 구조적 latency가 오히려 증가(θ=(Tmax+1)n), min-sum 등 디코더 알고리즘 자체의 개선은 아님.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2단계 계층적 인터리빙(HI) | 대응 없음 | ID(삽입/삭제) 오류 동기화 대응을 위한 다중 코드워드 재배치로 Prime ECC의 단일 코드워드 처리 구조와 무관 |
| 체인 복구(CR) / BCJR MAP 검출 | 대응 없음 | ID 채널 전용 sync 검출기로 NAND 채널모델(AWGN/RBER/fixed-error)과 오류 유형 자체가 상이, channel.cpp에 대응 기능 없음 |
| irregular LDPC / SC 코드 부호설계(부호 프로파일) | `ecc_top.cpp` `Load_PCM()` (참고용) | H-matrix 로드 위치는 대응되나 이 논문의 degree distribution은 ID 채널 특화 설계로 Prime ECC의 고정 QC-LDPC 구조에 그대로 이식 불가 |
| Simulated Annealing 기반 shift vector 설계 | 대응 없음 | 순수 인터리빙 파라미터 최적화이며 Prime ECC 디코더/부호 파라미터와 대응 요소 없음 |
```

적용 가치: **낮음** — insertion/deletion 동기화 오류라는 NAND 표준 채널과 무관한 오류 유형을 다루며, LDPC는 컴포넌트 코드로만 활용될 뿐 디코더 알고리즘이나 QC-LDPC 부호구조 자체의 개선을 제안하지 않아 Prime ECC 이식 요소가 거의 없음.
