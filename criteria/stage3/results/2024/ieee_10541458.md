### B. 알고리즘 요약

1. 대상은 짧은 LDPC 코드((32,16), (96,48), (128,64))와 BCH 코드, AWGN+BPSK 채널에서 Neural BP(NBP) 디코더를 사용.
2. 기존 NBP는 부동소수점 메시지를 쓰는데, 고정소수점 quantization 적용 시 성능 손실과 bit-width 선택 문제가 남아있음.
3. VN/CN 메시지의 동적 범위가 iteration마다 달라진다는 가정 하에, iteration별로 다른 quantization bit-width를 할당하는 것이 목표.
4. 균일 quantizer `Q(q, Δ, x)`를 사용해 메시지를 `2^q-1`개 레벨의 alphabet으로 매핑, 학습 시엔 Straight-Through Estimator(STE)로 gradient를 근사.
5. K개의 후보 bit-width `q1,...,qK` 중 하나를 layer(iteration)별로 선택하는 확률 `pk`를 3-layer FC 신경망(bit prediction network)으로 예측.
6. 순전파는 argmax(`pk`)로 one-hot 선택, 역전파는 미분 가능한 Gumbel-Softmax(`τ`, Gumbel noise `πk`)로 근사.
7. 손실함수는 디코딩 손실 `L(Lo,c)` + 기준 bit `b_benchmark` 초과분 페널티 + bit predictor 파라미터 정규화 항의 가중합.
8. 학습: Adam(lr=0.001), NBP 네트워크와 bit predictor를 joint training, 5000 epoch, base quantizer는 5-bit 균일 quantizer, 후보 {4,5,6}.
9. 결과: LDPC(128,64)에서 평균 4.8bit로 고정 5-bit 균일 quantization과 유사한 BER 성능 달성, 최고 SNR 구간에서만 소폭 손실.
10. 한계: 짧은 코드·AWGN 채널·시뮬레이션에서만 검증, HW 미설계, NAND 채널/QC-LDPC 구조 미고려.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| iteration별 dynamic bit-width 할당 아이디어 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC도 iteration 구간별 LLR threshold 테이블을 쓰나 고정 테이블이며, 신경망 기반 동적 선택은 이질적 |
| CN/VN 메시지 quantization | `channel.cpp` `Set_LLR_Th()` | 균일 quantizer 개념은 참고 가능하나 NN 예측기는 미보유 |
| Neural BP 디코더 전체(가중치 학습) | 대응 없음 | Prime ECC는 min-sum 기반 고정 알고리즘, NN 전면 대체 구조 없음 |
| Gumbel-Softmax/STE 학습 절차 | 대응 없음 | 학습 기반 최적화 파이프라인 자체가 코드베이스에 없음 |

> 적용 가치: 낮음 — NN 기반 학습 파이프라인이 Prime ECC의 bit-exact HW 시뮬레이터 구조와 이질적이고, 짧은 코드 시뮬레이션 검증에 그쳐 고rate QC-LDPC로의 일반화 근거 부족.
