### B. 알고리즘 요약 (10줄 내외)

1. 시스템: NAND flash SSD, threshold voltage를 낮춰 P/E cycle 수명을 늘리는 대신 RBER과 read latency가 증가하는 트레이드오프를 다룸.
2. 문제: 낮은 threshold voltage는 수명(지속가능성)에는 유리하지만 RBER 증가로 LDPC 반복 횟수가 늘어 read latency가 악화됨.
3. 가정: RBER은 전압 분포의 겹침 적분(`RBER = ∫p(k)(x)dx`, 식 (3))으로 모델링되며, RBER 증가는 곧 ECC 디코딩 반복수 증가로 이어진다고 가정.
4. 핵심 기법: SSD를 표준 전압의 Region 1(read 최적화)과 저전압의 Region 2(write/lifetime 최적화)로 물리 분할하는 access pattern-aware FTL.
5. 데이터 배치: R_count/W_count 카운터와 지수 decay 함수 `D(t)=e^(-λt)`로 최근 접근 패턴에 가중치를 부여해 4가지 접근 패턴 유형으로 페이지를 분류.
6. 확장: R_count, W_count가 모두 높은 페이지는 DRAM 캐시에 저장하여 Region 2의 높은 read latency 영향을 완화.
7. 구현 디테일: 두 영역 간 wear 비율(1:1.5~1:2)을 모니터링해 초과 시 Region 2→Region 1으로 데이터 마이그레이션하는 wear leveling 전략 포함.
8. 최적화 절차: 별도 학습 없음, threshold(Read_Threshold, Write_Threshold)는 워크로드에 따라 동적 조정.
9. 결과: SSDSim 기반 시뮬(MSR Cambridge 6개 trace)에서 기존 대비 수명 8.7~20.3% 개선, 평균 latency 19.7~69.8% 감소.
10. 한계: LDPC는 RBER-latency 관계의 배경 모델로만 사용되며 실제 부호/디코더 설계·강화는 다루지 않음(향후 과제로 언급). 시뮬레이션(SSDSim)만 수행, HW 검증 없음.
