### B. 알고리즘 요약 (10줄 내외)

1. 대상은 3GPP LTE 표준 Turbo code(rate 1/3, K=800), Max-Log-MAP(MLM) 기반 두 component decoder가 반복적으로 extrinsic information을 교환하는 구조.
2. 문제: MLM 디코더는 forward recursion에서 계산한 state metric `α`를 backward recursion까지 LIFO 방식으로 저장해야 하며, 이 State Metric Cache(SMC)가 전체 하드웨어 면적/전력의 주요 병목이다.
3. 기존 SMC 축소 기법은 역방향 재계산(연산복잡도 증가) 또는 단순 bit-width 절감(성능 저하) 트레이드오프를 가진다.
4. 핵심 기법: Stochastic Quantization(SQ) - 고정밀 값 `x`를 저장 시 결정론적으로 반올림(RQ)하지 않고 인접한 두 양자화 레벨에 확률적으로 배분해 기댓값 관점에서 무편향(unbiased) 양자화를 구현.
5. 핵심 식 `Pxi = (x-x_l)/(x_l bar - x_l)`로 양자화 확률을 정해 `E(x_q)=x`를 만족시키며, 반복(iteration)이 진행될수록 양자화 오차가 상쇄되어 감소.
6. 무작위 소스로 Bernoulli/LFSR/Sobol 시퀀스를 비교했고, Sobol/LFSR이 8회 반복 내 unbiased 조건을 만족하며 BER 성능이 가장 우수.
7. 부수 기법으로 LDPC 디코딩에서 쓰이던 Successive Relaxation(SR) 업데이트(`t_ext = (1-η)·t-1_ext + η·δ·ext`)를 SQ-MLM에 적용해 추가 성능 개선(원래 Turbo 디코딩에는 무효과로 알려졌던 기법을 SQ 조합에서 재활용).
8. 최적화 절차: η, δ 파라미터를 low-SNR 영역(Eb/N0=0.9dB)에서 그리드서치로 탐색(η=0.8, δ=0.8).
9. 결과: 9비트 forward state metric을 SQ로 4비트까지 축소해도 float-point 대비 거의 성능 손실 없이, RQ 대비 약 0.1dB 이득. 130nm ASIC 합성 결과 SM/NLQ/WHT+NLQ 대비 최소 면적.
10. 한계: Turbo 코드 특유의 forward/backward recursive LIFO 캐시 구조에 특화되어 있고, LDPC는 SR 기법의 출처로만 인용(참고문헌 [5], [6])될 뿐 LDPC 부호/디코더 자체 개선은 없음. NAND/SSD 응용도 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic Quantization(SQ) forward state metric 압축 | 대응 없음 | Turbo 고유의 LIFO SMC 구조 대상으로, Prime ECC의 min-sum CN/VN 메시지 저장 구조와 근본적으로 다름 |
| SR(Successive Relaxation) extrinsic update | decoder.cpp `LDPC_Decoder()` (간접, 개념적) | SR 자체는 LDPC decoding에서 유래한 기법(§C 인용 [5])이나, 본 논문은 Prime ECC가 이미 채택하지 않은 damped 업데이트 방식이며 이터레이션 간 relaxation을 코드가 지원하는지 별도 확인 필요 |
| 랜덤소스 기반 확률적 양자화(RNG 설계) | channel.cpp `Set_LLR_Th()` (간접) | LLR 양자화 임계값 설계와 개념적으로 유사한 영역이나 본문은 Turbo state metric 대상, LDPC LLR 적용 검증 없음 |
```

적용 가치: **낮음** — Turbo 디코더 SMC 압축이 핵심이며 LDPC/NAND와 직접 연관이 없음. 확률적 양자화라는 아이디어 자체는 흥미롭지만 본문에 LDPC 적용 검증이 없어 이식 시 재검증 부담이 큼.
