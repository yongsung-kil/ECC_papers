> 총평: NAND ECC 직접 대상이 아닌 5G/6G 무선통신 채널코딩 서베이지만, LDPC 챕터(§VI)가 QC-LDPC 구성법·min-sum 계열 디코딩 변형·디코더 스케줄링/HW 아키텍처를 폭넓게 정리해 Prime ECC 개선 아이디어의 "지도" 역할로 참고 가치가 있음. 단, 본 논문 자체는 신규 기법이 아니라 기존 문헌들의 리뷰.

### B. 알고리즘 요약

1. 대상: 1G~5G 이동통신 채널코딩 전체(turbo/convolutional, LDPC, polar/RM) 리뷰 및 6G 요구사항(≥1Tb/s, <100μs latency, 7-nines 신뢰성) 대비 전망 서베이.
2. 풀려는 문제: 5G-A/6G에서 요구되는 초고속·초저지연·고신뢰 통신을 만족할 채널코딩 후보(LDPC/polar/turbo/기타) 정리 및 격차 분석.
3. LDPC 핵심(§VI): `H` 기반 `[N,K]` 부호, 앙상블(regular/irregular/protograph), 5G NR은 PBRL-LDPC 구조(`B_5G = [B_core B_ex; 0 I]`)로 BG1(rate 1/3~8/9)/BG2(rate 1/5~2/3) 2개 base matrix + 다중 lifting factor로 rate-compatibility 지원.
4. 핵심 디코딩 알고리즘: Sum-Product의 `Φ(x)=-log tanh(x/2)`를 min 근사한 Min-Sum, 성능보정을 위한 Normalized-MS/Offset-MS(`α`, `β` 스케일/오프셋), Self-Corrected MS(부호반전 메시지 소거), A-min*(재귀 Jacobian 근사), Adjusted-MS(min/max 조합).
5. 핵심 식의 의미: NMS/OMS는 MS의 과대추정 보정(`E_{i,j}=α·min(...)` 또는 `-β`)으로 SPA와의 성능 격차를 줄이되 에러플로어 위험 존재; A-min*/AdjMS는 SPA에 근접한 정확도-복잡도 균형.
6. 확장: QML(Quasi-ML) 디코딩으로 reprocessing(리스트 디코딩) 기반 EQML이 MS 대비 0.5~0.6dB, SPA 대비 0.2~0.3dB 개선; ML 대비 0.3dB 이내 근접.
7. HW/구현 디테일: 디코더 스케줄링 3종(flooding/layered/IDS), 아키텍처 3종(fully parallel/unrolled fully parallel/partially parallel), 처리량 모델 `T̂(H)=Proc(E(H))/E(H)·n_p·Z·f_max·R`.
8. 학습/최적화: NMS/OMS의 `α,β`는 Monte-Carlo 또는 DE로 결정; 딥러닝 기반 unfolding(NMS/OMS 파라미터 학습), 신경망 기반 FA 디코더(양자화 3bit로 0.3dB 이득), syndrome 기반 신경망 LLR 추정(플래시 컨트롤러 LDPC 사례 인용).
9. 결과(수치): 5G NR LDPC K=8448 rate 1/3~8/9 FER 비교, 1Tb/s throughput 사례(12-core, 최대 4 iteration), IEEE 802.11n (1944,1620) 코드에서 1.511GHz@7-stage pipeline로 2.937Gb/s 피크.
10. 한계: 서베이 논문 특성상 특정 HW 신규 설계·실측 없음(기존 문헌 인용치 정리), NAND/SSD 채널이 아닌 AWGN/BSC/BEC 무선채널 기준, 세부 표(Table 18)의 gatecount/공정 수치는 .txt 추출 손상으로 본문에서 직접 확인 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum 근사(`Φ` 함수 min 대체) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 채택한 기본 기법과 동일 원리(§3 "이미 보유"), 신규성 낮음 |
| Normalized/Offset MS, A-min*, AdjMS, SCMS 등 MS 변형 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 이식 난이도 낮음(프로파일 §4). Prime ECC는 magnitude 양자화 테이블 기반이라 정규화/오프셋 factor 대체 적용 여지 있음 |
| 디코더 스케줄링(flooding/layered/IDS) | `decoder.cpp` Dual-Update(odd/even 교대) 스케줄 | Prime ECC의 Dual-Update와 layered BP 개념이 유사, IDS(informed dynamic scheduling)는 신규 아이디어 |
| 5G NR PBRL-LDPC 부호 구조(BG1/BG2, dual-diagonal core) | `ecc_top.cpp` `Load_PCM()`, `encoder.cpp` (dual-diagonal 고정) | Prime ECC도 dual-diagonal 인코딩 채택(§3), 부호설계 자체 이식은 프로파일 §4 기준 "높음(어려움)" |
| QML/reprocessing 리스트 디코딩(EQML, OSD) | 대응 없음 | Prime ECC는 단일 min-sum 이터레이션 구조, 리스트 디코딩/재처리 로직 미보유 — error-floor 개선 후보로 검토 가치 있으나 코드 수정 범위 큼 |
| 머신러닝 기반 unfolding/신경망 디코더 | 대응 없음 | 프로파일 §6 "NN/딥러닝 디코더는 대응 없음"에 명시된 항목과 일치 |

> 적용 가치: 중간 — 신규 알고리즘 제안은 아니지만 min-sum 계열 변형(NMS/OMS/A-min*/AdjMS)과 디코더 스케줄링/HW 아키텍처 트레이드오프를 체계적으로 정리해, Prime ECC의 min-sum 튜닝이나 error-floor 개선(QML/reprocessing) 검토 시 참고 문헌 지도로 활용 가능.
