### B. 알고리즘 요약

1. 시스템: 50G-PON(passive optical network) 표준의 QC-LDPC, (17280,14592) 코드(마지막 384비트 puncturing), 12x69 exponent matrix, rate ≈ 0.844.
2. 문제: 긴 코드워드에서 완전병렬 아키텍처는 면적·wiring congestion이 심각, partially-parallel 구조가 필요하나 기존 layered decoder는 congestion overhead가 큼.
3. 핵심 가정/모델: 8-bit 양자화 LLR, layered scheduling(VNP/CNP/APP update 각 PCM row 단위), Z/2 row를 동시 처리하는 partially-parallel 구조.
4. 핵심 기법1(컬럼 재배열): exponent matrix를 그룹으로 분할해 그룹마다 row당 nonzero submatrix가 하나만 있도록 재배열, `read permutation`이 `23·Z` 후보 중 필요한 메시지만 그룹 독립적으로 선택 → MUX 이용률 50% 감소.
5. 핵심 식/의미: 최대 check-node degree `23`을 활용해 `69·Z` 후보 APP 메시지 중 `23·Z`만 read permutation으로 추출, congestion을 원천적으로 줄이는 구조.
6. 핵심 기법2(odd/even scheduling): Z/2 rows를 홀수/짝수 인덱스로 나눠 순차 처리하는 rotation network 설계, 임의 shift amount 지원하면서 rotator 입력 수 감소 → 기존 barrel shifter 대비 MUX 53% 감소.
7. 부수 디테일: rotated 결과를 restoration 없이 메모리에 그대로 저장하고, 이후 read rotator가 pre-rotated 상태를 반영해 rotation amount를 계산 → HW 비용 절감. CNP는 offset min-sum(OMS) 알고리즘 사용, C2V 메시지는 row별 분산 메모리로 저장.
8. 학습/최적화 절차 없음(구조적 재배열·스케줄링 최적화, DE/GA/NN 없음).
9. 결과: XCKU19P FPGA에서 172K FF/311K LUT, 87MHz, throughput 2.94Gbps(기존 [2] 대비 2.38배), UBER=10⁻¹²에서 RBER 0.023으로 기존 NMS 디코더와 동등한 정정성능 유지.
10. 한계: 특정 50G-PON PCM 구조(max degree 23, 12x69 exponent matrix)에 맞춘 재배열 기법이라 다른 부호 구조로 일반화 시 재설계 필요, 정정능력 개선 자체는 없음(HW 효율만 개선), FPGA 구현만 있고 ASIC 결과 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Partially-parallel layered scheduling (Z/2 row 동시처리) | 디코더 HW 모델 (z=32, HCU, GT) | 우리도 z_sb=32 lifting 기반 row 단위 병렬 처리 구조라 스케줄링 아이디어 참고 가능 |
| PCM 컬럼 재배열로 read permutation MUX 축소 | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 MUX 기반 edge 선택과 목적(congestion/MUX 절감)이 유사, 적용 시 GT 로직 재설계 필요 |
| odd/even scheduling 기반 rotation network | 디코더 HW 모델 (z=32, HCU, GT) | HCU가 이미 비대칭 degree column을 half-cycle 처리하는 방식과 유사한 문제의식, 직접 대응은 아님 |
| offset min-sum (OMS) CNP | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 계열 기법(§3)과 중복, 신규성 낮음 |
| C2V 메시지 row별 분산 메모리 저장 | `ecc_data.h` (PCM/PARAM_DEC 구조체) | 메모리 분산 저장 아이디어는 HW arch 레벨에서 참고 가능하나 우리 구조와 정합 필요 |
| rotated 결과 restoration 없이 저장 (pre-rotated 반영) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | HW 비용 절감 트릭으로 파이프라인 6-stage 구조에 참고 가능 |

> 적용 가치: 중간 — 정정성능 개선은 없지만 partially-parallel QC-LDPC 디코더의 MUX/congestion 절감 아키텍처 아이디어(컬럼 재배열, odd/even rotation scheduling)는 Prime ECC의 GT/HCU HW 최적화 레이어에 참고할 여지가 있음.
