> 총평: 5G irregular QC-LDPC의 layered FAID LUT를 NN으로 학습해 waterfall 성능을 개선하지만, min-sum 기반 Prime ECC와는 디코딩 알고리즘 계열(FAID vs. min-sum)이 달라 직접 이식보다는 LLR/양자화 테이블 최적화 아이디어 차원의 참고 가치.

### B. 알고리즘 요약

1. 대상: 5G BG1 기반 irregular QC-LDPC, rate 0.5/0.6875/0.786, length 4224/6528/11520, AWGN+BPSK 채널.
2. 문제: 기존 FAID(finite alphabet iterative decoder) LUT는 exhaustive search로 설계되어 규모가 커지면(irregular, layered) 최적화가 비효율적이고, 기존 layered LUT를 그대로 쓰면 L-NMS 대비 FER 10⁻³에서 0.5dB 이상 성능 손실 발생.
3. 가정: soft 채널 출력 LLR을 `wu=4bit`로 양자화 후 입력, VN 출력은 `wN=6bit`.
4. 핵심 기법: layered FAID의 갱신함수 `ϕv, ϕc, ϕl`를 unfolded NN(RQNN)으로 보고, 매핑함수 `F(x)`(quantization threshold `ηk` / level `Mk`로 정의된 LUT)를 학습 파라미터화.
5. 핵심 식: `F(x)=M0·ε(x-η0)+(M1-M0)·ε(x-η1)+(M2-M1)·ε(x-η2)`; threshold 선택을 one-hot화하고 `softmax(gi,j/ζ)`로 미분 가능하게 근사(온도 `ζ`를 1→작은 값으로 감소).
6. 확장: check node를 parity-check matrix의 row weight별로 분류해 반복(iteration)마다, CN 카테고리마다 별도 LUT 학습(rate 0.5/0.6875는 4개, 0.786은 3개).
7. 구현 디테일: STE(straight-through estimator)로 `round(·)`와 `step(·)`의 역전파를 각각 `softround`, `softstep`(tanh 기반)으로 대체; 전체 네트워크를 한 번에 unfold하지 않고 `Δ=2` iteration 단위로 부분(partial) unfolding해 학습(gradient vanishing/overfitting 방지).
8. 학습: mini-batch(batch size 20) + Adam, 총 60 batch(1200 샘플셋)로 수렴, loss는 hard-decision 대신 sigmoid(LLR) 기반 cross-entropy.
9. 결과: FAID3/FAID2 모두 기존 flooding 기반 최적화 FAID 대비 유의미한 FER 개선, FAID3(다중 LUT)는 양자화 L-NMS와 동등~우월(R=0.6875에서 초과); LUT 탐색 복잡도는 exhaustive 대비 최대 4~5 자릿수 감소(예: FAID3 LUT(x) exhaustive 4.82×10¹⁷ vs RQNN 8400·E).
10. 한계: 알고리즘/시뮬레이션 수준 논문으로 HW 구현·gatecount·throughput 언급 없음, decoding complexity는 학습된 LUT를 오프라인 적용하는 전제라 실제 HW 이득은 별도 검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|

| iteration별/CN row-weight별 LUT | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블 구조와 개념적 대응, 코드 자체는 FAID 전용 |
| layered(레이어드) 갱신 스케줄 | `decoder.cpp` `LDPC_Decoder()` | 이미 보유(Dual-Update 유사) 스케줄로 신규성 낮음 |
| min-sum(NMS) 비교 baseline | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 보유한 min-sum과 성능 비교 대상일 뿐, 이식 대상 아님 |
| RQNN 학습(오프라인) 자체 | 대응 없음 | 코드베이스에 NN 학습 파이프라인 없음 |

> 적용 가치: 낮음 — FAID는 min-sum과 다른 디코딩 계열이라 알고리즘 자체 이식은 어렵고, LUT/threshold 최적화라는 개념만 참고 가능.
