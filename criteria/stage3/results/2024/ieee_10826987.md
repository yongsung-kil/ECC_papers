### B. 알고리즘 요약

1. 대상은 short block LDPC(49,24) 코드를 AWGN+BPSK 채널에서 복호하는 세 가지 방식: BP, NBP(neural BP), ECCT(error correction code transformer).
2. 문제의식은 짧은 블록에서 BP 성능이 저하되고, ECCT가 최신 성능을 보이지만 계산복잡도·메모리 사용량이 과도하다는 점.
3. NBP는 BP의 반복 과정을 trellis로 unroll하고 각 메시지에 학습 가능한 가중치를 부여하는 모델.
4. ECCT는 채널출력 크기와 syndrome을 결합해 임베딩한 뒤 PCM 기반 마스크를 적용한 self-attention으로 비트 간 관계를 학습.
5. attention block 복잡도는 `O(N n^2 d)`, feedforward block은 `O(N n d^2)`(N=encoder layer 수, n=codeword 길이, d=임베딩 차원)로, 코드 길이·임베딩 차원에 따라 급격히 증가.
6. 모델 단순화 전략으로 I-ViT 기법의 integer-only quantization(FP32→INT8)을 ECCT에 적용해 봄.
7. BER 결과: L=5에서 NBP가 BP 대비 0.1dB 이득, L=50에서는 BP·NBP 성능 수렴, ECCT는 L=50에서 NBP 대비 0.2dB 추가 이득.
8. 복잡도 결과: masking으로 attention block 연산을 20%까지 줄여도 ECCT는 NBP(L=5) 대비 30배 많은 연산량 요구.
9. 파라미터 수: ECCT는 NBP(L=5) 대비 50배 많은 파라미터 필요, INT8 양자화로 25%까지 줄여도 여전히 메모리 부담 큼.
10. 한계: 실제 HW(FPGA/ASIC) 구현 없이 FLOPs·파라미터 수 기반 이론적 분석에 그치며, 향후 압축·HW 친화 설계가 필요하다고만 제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ECCT(transformer) 디코더 전체 | 대응 없음 | NN/딥러닝 디코더로 Prime ECC의 min-sum 구조와 완전히 상이 |
| NBP(neural BP) 디코더 | 대응 없음 | 학습형 weighted BP로 binary min-sum HW와 대응 안 됨 |
| INT8 양자화(모델 압축) | decoder.cpp `VN_Cal_HD()` 등 (HD/2SD/3SD 양자화) | 개념적으로 양자화 아이디어는 유사하나 대상이 NN weight/activation이라 직접 이식 불가 |
```

> 적용 가치: 낮음 — 신경망 기반 디코더의 이론적 복잡도 비교 논문으로, binary QC-LDPC HW(Prime ECC) 이식 대상 기법이 없음.
