### B. 알고리즘 요약

1. 시스템: BPSK 변조 후 correlated Gaussian noise 채널(`n = R^{1/2} n_t`, 상관계수 `θ=0.65`)을 거쳐 (576,432) WiMAX LDPC, (1998,1512) QC-LDPC를 NMS(정규화 인자 0.75, 10회 반복)로 복호.
2. 문제: 기존 denoiser(CNN 기반 노이즈 추정)는 error block 내 노드를 균일하게 처리해, 노이즈에 취약한 error node를 특별히 처리하지 못함.
3. 채널 모델: correlation matrix `R_{i,j} = θ^{|i-j|}`로 정의된 상관 잡음.
4. 핵심 기법: NMS 1차 복호 실패(`Hu^ ≠ 0`) 시, 추정 노이즈 `n^ = y - s^`를 CNN에 입력해 근사 노이즈 `n~`를 얻고 잔차 `r = n - n~`, 추정 오류패턴 `e^ = n^ - n~ = e + r`를 계산.
5. threshold discrimination: `|e^_i| ≥ T`이면 error node로 판정, 해당 노드의 채널 LLR을 0으로 zeroing(5G NR LDPC puncturing에서 착안); `|e^_i| < T`인 correct node는 denoising 처리.
6. 처리된 LLR을 입력으로 NMS 1회만 재디코딩(다회 반복 재디코딩 아님).
7. CNN 구조: 4-layer conv, filter `{10,4,4,16}`, feature map `{64,32,16,1}`, Xavier 초기화, MSE loss, Adam(lr=0.001), SNR 범위 [0, 3]dB에서 학습, 조기종료 적용.
8. 학습: 5,140,800 샘플 학습/210,000 테스트, batch 2800, epoch 최대 30000(500 epoch loss 정체 시 조기종료).
9. 결과: BLER=1e-1 기준 zeroing이 attenuation(0.9dB)·flipping(1dB) 대비 최대 1.3dB 이득; 기존 denoiser 대비 (576,432)에서 0.7dB, (1998,1512)에서 1.5dB 추가 이득, (1998,1512)에서 error floor(2e-2→BLER 1e-5까지 미관측)를 크게 낮춤. 연산 오버헤드는 subtraction/comparison/zeroing 각 1회뿐.
10. 한계: correlated Gaussian noise 채널만 가정, HW 구현·복잡도(gatecount/throughput) 미제시, 재디코딩은 NMS 1회로 한정된 시뮬레이션 결과만 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNN 기반 오류 노드 식별·zeroing post-processing | 대응 없음 | Prime ECC는 NN 기반 pre/post-processing 모듈 미보유 |
| threshold discrimination 후 채널 LLR 0 처리(puncturing 유사) | `channel.cpp` `Set_LLR_Th()` | LLR threshold 개념은 유사하나 CNN 추정치 기반 zeroing은 새로운 로직 필요 |
| NMS 재디코딩(1회) | `decoder.cpp` `LDPC_Decoder()` | 기존 min-sum 반복 루프 재사용 가능하나 외부 트리거(재디코딩 여부 판단) 별도 필요 |
| correlated noise 채널 모델 | `channel.cpp` | AWGN/RBER/fixed 패턴만 지원, correlated noise 모델 없음 |

> 적용 가치: 낮음 — CNN 학습·추론 인프라가 필요하고 correlated Gaussian noise 통신 채널 가정이라 NAND 채널 특성과 괴리가 크며, non-binary 구조 개선이 아닌 순수 알고리즘 wrapper라 이식 시 재검증 부담이 큼.
