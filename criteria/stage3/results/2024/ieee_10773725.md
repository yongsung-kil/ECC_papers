### B. 알고리즘 요약

1. 시스템: LDPC 인코더 - 인터리버 - CPM 변조기 구조로, 항공 텔레메트리에 쓰이는 3종 CPM(ARTM0=PCM/FM, ARTM1=SOQPSK-TG, ARTM2=ARTM CPM)에 매칭되는 QC-LDPC(rate 1/2, 2/3, 4/5) 코드를 설계.
2. 문제: IRIG-106 표준에 ARTM0/ARTM2용 현대적 capacity-approaching FEC가 부재하고, 기존 AR4JA 코드는 BPSK(등가 OQPSK)에 최적화되어 CPM의 비선형 변조 특성을 반영하지 못함.
3. 채널/모델: CPM 복조기를 binary-input AWGN 등가 채널로 보는 EXIT 프레임워크를 확장, CPM SISO 모듈의 EXIT 특성함수 `T_{Es/N0}[I_A]`를 Monte Carlo 시뮬레이션으로 사전 산출.
4. 핵심 기법: protomatrix EXIT(PEXIT) 알고리즘을 CPM용으로 수정 - `I_ch(j) = I_Es`(CPM SISO 출력)를 LDPC 디코더의 variable node 초기값으로 주입하는 방식으로 EXIT 반복.
5. 핵심 식: mutual information 함수 `J(σ)`와 그 역함수 `J⁻¹(I)`로 σ(L-value 표준편차) 도메인과 I(정보량) 도메인을 오가며 protomatrix의 디코딩 임계 SNR을 탐색.
6. 확장: 4-ary alphabet(`b_i,j ∈ {0,1,2,3}`) protomatrix를 column-wise greedy 탐색으로 최적화, 이후 2단계 lifting(1단계 factor 2 + ACE 알고리즘 기반 2단계)으로 최종 QC-LDPC 생성.
7. 구현 디테일: MB=4행 고정 protomatrix 포맷, variable node degree 3 이상 유지(degree-1/2 노드·puncture 미사용)로 linear minimum distance growth 보장, error floor 없음을 목표.
8. 최적화 절차: 후보 column을 SNR을 점차 낮춰가며 survivor set이 사라질 때까지 반복 평가하는 greedy column-wise PEXIT 탐색(GA/DE 아님, 단순 탐색).
9. 결과: coding gain(BER=1e-8 기준, 미부호화 대비) 7.1~10.9dB, capacity gap 0.55~1.35dB, error floor 없음(BER<1e-8까지 관찰).
10. 한계: HW 미설계, 시뮬레이션(BER/FER 곡선)만 존재, CPM 복조기-LDPC 반복(global iteration) 구조 전제로 일반 binary AWGN/NAND 채널과 근본적으로 다른 시스템.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CPM 전용 PEXIT protograph 설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 레이어에 대응되나, CPM 채널 특화 설계 기법으로 QC-LDPC 구조 재설계 필요 |
| CPM SISO ↔ LDPC 반복복호 (global iteration) | 대응 없음 | Prime ECC는 NAND read 채널을 직접 가정하며 변조기/복조기 SISO 반복 개념이 없음 |
| 2단계 lifting (ACE 알고리즘) | 대응 없음 | 부호 구성법 도구로 디코더 실행시간 모듈과 무관 |
| binary AWGN 채널 EXIT 분석 (J(σ) 함수) | `channel.cpp` | 채널 모델 관점에서만 개념적 유사성, RBER/fixed-error 패턴과는 다른 통신 채널 모델 |

> 적용 가치: 낮음 — CPM 변조·복조 반복 구조에 특화된 protograph LDPC 설계로, NAND 고전 binary LDPC의 부호설계·디코더·HW 어느 레이어에도 직접 이식할 요소가 없음.
