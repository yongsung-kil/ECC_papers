### B. 알고리즘 요약

1. 시스템: BPSK, AWGN+단일 burst 오류 결합 채널, regular QC-LDPC(7154,8176) 코드, normalized min-sum 디코더 사용.
2. 문제: burst error는 LDPC 디코더 성능을 크게 저하시켜 error floor를 유발하는데, 기존 burst 검출법은 디코딩 결과를 이용해 추가 지연이 발생.
3. 핵심 가정: burst 구간 `B={i|T≤i<T+L}`의 비트는 부호 반전(`(-1)^{v_i}`)되어 신뢰도가 사실상 무작위(0.5)가 됨.
4. 핵심 기법 1단: 수신 심볼을 hard-decision한 뒤 `H`로 syndrome `S=Hĉ^T`를 계산, VN별 unsatisfied check 수 `d_i`가 자신의 column weight `ρ_i`와 같으면(모든 연결 check 불만족) 오류 추정치 `e_i=1`로 플래그.
5. 핵심 식: 검출 통계량 `Ψ(y)=‖e‖_0`을 문턱값 `ζ`와 비교하는 이진 가설검정(`H0`: burst 없음 vs `H1`: burst 존재), `Ψ(y)`의 분포를 Binomial/Hypergeometric로 유도해 검출확률 `Pd`, 오경보확률 `Pfa`의 닫힌형 식을 도출.
6. 핵심 기법 2단: burst 위치 추정은 sliding window 비율 `Pe,W(i)`의 최대값 위치 `α`를 찾고 `0.5·Pe,W^max` 경계에서 `α_left`, `α_right`를 결정해 위치집합 `B̂_W`를 산출, window 크기 `W`를 반복적으로 축소(`W_{l+1}=W_l-ϑ·Ψ(y)`)하며 정밀화.
7. 부수 트릭: 검출된 burst 위치의 초기 LLR `L0(c_i)`를 0으로 강제(식 (8))해 디코더에 erasure로 전달, 오검출 완화를 위해 다중 window 크기를 독립 적용 후 합집합.
8. 별도 학습절차 없음(닫힌형 통계 모델 + 반복 탐색 알고리즘).
9. 결과: burst 검출은 완전 known-position 대비 약 1dB 성능 손실 내로 접근, 기존 CME(jamming 검출) 대비 저 JSR 구간에서 우수, 계산복잡도는 elementary AND/XOR/덧셈/비교 연산만 사용해 CME보다 낮음.
10. 한계: 단일 burst·AWGN 결합 채널만 다루고(다중 burst 미검증), 이론적 성능은 CN/VN 독립 및 무한 코드길이 가정 하에 유도되어 실제 유한장 코드와 괴리 존재, HW 구현 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 unsatisfied check 카운트 오류 추정 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 디코딩 이터레이션 진입 전 pre-processing으로 추가 가능한 구조 |
| LLR zero-forcing(erasure) 적용 | `channel.cpp` `Set_LLR_Th()` | LLR 테이블/threshold 설정 로직에 erasure 옵션 추가 관점에서 관련 |
| sliding window 기반 burst 위치 탐색 | 대응 없음 | NAND 채널엔 burst 개념이 없어 대응 모듈 부재 |
| detection/false-alarm 확률 닫힌형 분석 | 대응 없음 | 통계적 채널 모델링으로 HW/디코더 모듈과 직접 대응 없음 |

> 적용 가치: 중간 — burst error가 아닌 random error 위주인 NAND 채널에는 직접 적용 어렵지만, syndrome만으로 사전 오류패턴을 추정해 LLR을 조정하는 pre-processing 아이디어 자체는 decoder 앞단에 참고할 여지 있음.
