### B. 알고리즘 요약

1. 대상은 5G NR QC-LDPC(BG1/BG2), row-layered scheduling, normalized min-sum(NMS) 알고리즘 기반, partially-parallel 구조(DFU 32개).
2. 기존 min-tree 기반 CNU는 2's complement→signed-magnitude 변환과 비교기 트리로 인해 면적·지연이 크다는 문제를 해결하려 함.
3. V2C 메시지(5bit SM)를 절댓값 기반 thermometer code(TC, unary code)로 변환한다는 것이 핵심 가정.
4. 핵심 기법 1단: TC 기반 CNU - `AND` 연산만으로 첫 번째 최소값을, `XOR+NOR`로 위치 마스킹 후 다시 `AND`로 두 번째 최소값을 탐색(비교기 불필요).
5. 정규화(`α`)를 C2V 계산 후가 아니라 2C→TC 변환 시점에 결합(`R̂cv = S·min|α·Qv'c|`), TC 비트폭을 15bit→11~12bit로 축소해 추가 면적 절감.
6. 핵심 기법 2단: 인접 layer 간 두 번의 shift(roll-back + 다음 layer shift)를 상대 shift coefficient `ŝ=(s_b-s_a) mod Z`로 통합, QSN 2단→1단으로 축소.
7. 부수 기법: C2V memory를 two-port 대신 single-port로 구현(두 C2V 벡터를 한 워드로 결합해 매 클록 교대 접근), 면적 50%·전력 18%·지연 33% 절감.
8. 학습/최적화 절차 없음(고정 HW 아키텍처 설계).
9. 결과: 65nm CMOS, 500MHz, 0.63M gate, 23.47Gbps, CNU 복잡도 최대 36% 감소, QSN 면적 절반, FER 성능 열화 없음(baseline NMS와 거의 동일).
10. 한계: 정정성능(FER) 개선은 목표가 아니며 기존 NMS와 동등 유지가 목표, TC 변환기 자체는 SM 변환기보다 약 1.59배 복잡해 대신 CNU 논리 절감으로 상쇄.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Thermometer code 기반 min1/min2 탐색 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산의 비교기 로직 대체안 - 이식 난이도 낮음, HW 면적/지연 직결 |
| 정규화(normalization)를 TC 변환에 결합 | `decoder.cpp` `Get_VNU_Table_Idx()`, `ecc_data.h` `PARAM_LLR` | 스케일링 로직 위치 재배치 아이디어, 대응 가능 |
| 상대 shift coefficient 기반 단일 QSN | 대응 없음 (Prime ECC는 z_sb=32 row 병렬 방식으로 QSN/barrel-shifter 구조 상이) | 5G의 넓은 lifting size(2~384) 대응용 shift network 최적화로, 고정 z=32인 Prime ECC 구조와는 적용 맥락 다름 |
| Single-port C2V memory | `decoder.cpp` 메모리 참조 로직 (C2V 저장 관련) | 메모리 포트 절감 아이디어, HCU/GT와 별개로 메모리 뱅크 설계 참고 가능 |
| 부호구조(5G BG1/BG2, 51종 lifting size) | `ecc_top.cpp` `Load_PCM()` | 5G 표준 특화, 고정 QC-LDPC인 Prime ECC엔 재구성성 자체는 불필요 |

> 적용 가치: 높음 — CNU의 min-search를 비교기 대신 논리게이트(thermometer code)로 대체하는 아이디어는 성능 손실 없이 HW 면적·지연을 줄이는 순수 이득으로, Prime ECC의 CNU 로직(`CNU_Update_New_Mag`)에 바로 검토 가능.
