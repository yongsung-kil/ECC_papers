### B. 알고리즘 요약

1. 대상은 RAID-6류 (k+2,k) systematic MDS array code(erasure code)로, k개 원본 패킷(각 l비트)에서 2개 패리티 패킷 p, q를 XOR만으로 계산하는 구성.
2. 기존 RDP/EVENODD/X-code/P-code는 sub-packetization level `l`이 정해지면 최적 인코딩 복잡도(`2-2/k` XOR/bit)를 만족하는 k의 상한이 낮다는 한계가 있음.
3. 부호는 block generator matrix `Gb=[Kij]`로 정의되며 MDS 조건은 각 `Ki2`가 full rank이고 서로 다른 두 `Ki2` 차가 full rank인 것(랭크-메트릭 코드 조건).
4. 첫 구성 C1: `l=4`에서 15개의 4x4 이진 행렬 집합 `A1`으로 `k≤15`(이론적 최대) 달성, `k≤7` 부분코드는 `2-2/k` XOR/bit로 이론적 인코딩 하한 정확히 달성.
5. 둘째 구성 C2: `l=L-1`(L은 특정 소수)에서 순환치환행렬 합 집합 `A2`로 `k≤L(L-1)/2`까지 확장, `k=L-1` 또는 `2L-3`일 때 RDP코드와 일치하며 최적 인코딩 복잡도 달성.
6. 셋째 구성 C3: bit-permutation(치환행렬 V)을 이용해 C2를 확장, `l=2(L-1)`에서 `k≤L(L-1)`까지 확장, `k=2(2L-3)`에서도 최적 인코딩 복잡도 달성(기존 문헌에 없는 특성이라 주장).
7. 중간결과 재사용(reuse) 트릭으로 p 계산 시 생긴 부분합을 q 계산에 재사용해 XOR 수를 줄임(모든 3개 구성 공통 기법).
8. MDS 성질 증명은 BCH bound·이진 quadratic residue code의 최소 해밍거리, cyclic permutation matrix rank 이론에 의존(Lemma/Theorem, Appendix 증명).
9. 결과: C1 부분코드(k≤7)는 EVENODD/RDP 대비 update complexity 동등~우수, C2/C3는 update complexity가 EVENODD/RDP보다 약간 높음. 디코딩 복잡도는 L,k 증가에 따라 `2-2/k`로 점근.
10. 한계: 순수 이론적 구성·복잡도 증명 논문으로 시뮬레이션/HW 구현 없음. 채널모델·오류정정(BER) 개념이 아닌 소거(erasure) 복구 모델이라 LDPC의 soft-decision iterative decoding과 근본적으로 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MDS array code (RDP/EVENODD 확장) 구성 | 대응 없음 | binary QC-LDPC가 아닌 XOR 기반 erasure code(RAID-6)로 Prime ECC의 부호 모델과 무관 |
| 소거(erasure) 복구 디코딩 | 대응 없음 | Prime ECC는 soft/hard LLR 기반 iterative LDPC decoder(`decoder.cpp`)이며 소거정정 알고리즘이 아님 |
| 인코딩 복잡도(XOR 수) 최적화 | 대응 없음 | `encoder.cpp`의 dual-diagonal QC-LDPC 인코딩과 수학적 구조가 상이 |

> 적용 가치: 낮음 — RAID-6/분산스토리지용 XOR 기반 MDS array code(erasure code) 구성 논문으로 NAND binary QC-LDPC ECC의 부호설계·디코더·HW 어느 레이어와도 대응되지 않음.
