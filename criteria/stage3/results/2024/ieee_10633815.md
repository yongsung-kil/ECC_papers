### B. 알고리즘 요약

1. 시스템: IEEE 1901 표준(스마트그리드/EV/IoT용 전력선통신)에 규정된 3종 FEC(LDPC-CC rate 1/2~4/5, RS(255,239), RSCC)를 모두 지원하는 통합 재구성형 디코더. LDPC-CC는 시간가변 패리티검사행렬 H(주기 Tp=3, 메모리사이즈 ms)를 가지며 rate=(b-e)/b.
2. 문제: 기존 LDPC-CC 전용 디코더들은 표준의 일부 FEC 타입만 지원하며, AP-LLR/C2V 메모리가 전체 면적의 80% 이상을 차지해 면적 절감 여지가 큼.
3. 채널 모델: AWGN + BPSK 변조, 채널 LLR `I_vi` 입력.
4. 핵심 기법 1단: layered decoding 알고리즘 - V2C recovery(`P=L_old-Q_old`), CN update(정규화 min-sum, `β=3/4`로 `Q_new = β·sign(...)·min|P|`), AP-LLR update(`L_new=P+Q_new`)를 PE별로 순차 처리, N_p=10개 PE가 FIFO(AP-LLR/C2V)를 통해 레이어(CN)를 갱신.
5. 핵심 식의 의미: CN update의 정규화 factor β=3/4는 min-sum의 근사오차를 보정해 sum-product 대비 성능 저하를 줄임; Min1/Min2 finder가 CN 갱신에 필요한 최소값 2개와 인덱스를 트리구조로 산출.
6. 핵심 기법 2단(heterogeneous storage): PE 인덱스별 AP-LLR 절댓값 분포 관찰(저인덱스 PE는 값 범위가 작고, 고인덱스 PE는 정밀도 요구가 낮음)에 기반해 PE마다 다른 정수/소수 비트폭을 할당, 수치탐색으로 최적 비트폭 결정(Table III).
7. 부수 트릭: 압축 C2V 메시지 포맷(부호 15bit + Min1 index 4bit + Min1/Min2), PE 클래스 간 전환 시 DFC(data format conversion) 모듈로 비트폭 변환(면적 오버헤드 0.1% 미만); RS/CC 디코더와 메모리를 재구성 공유(reconfigurable Min Finder는 signed/unsigned 변환으로 LDPC-CC의 Min1Min2Finder와 CC의 Min1Finder를 공유).
8. 학습/최적화 절차: heterogeneous 비트폭은 수치적 탐색(numerical search)으로 결정, 별도 학습(NN/GA) 없음.
9. 결과: 55nm 공정 합성 결과, heterogeneous scheme 단독으로 면적효율 35% 개선(기존 flooding 대비 0.25dB 성능이득, quantized layered 대비 0.03dB 손실 이내); 재구성형 디코더 전체는 개별 디코더 합 대비 면적 28.2%(또는 25% 이상) 절감, 에너지효율 최대 11.04Gbits/J, 처리량 220Mbps(표준 요구치).
10. 한계: 대상 부호가 QC-LDPC block code가 아닌 LDPC-CC(convolutional)이며, 알고리즘은 표준 NMS 기반이라 정정성능 자체의 신규 개선은 없음(면적/전력 최적화가 주기여); throughput이 220Mbps로 낮고 latency 개선 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PE별 heterogeneous 비트폭 할당(AP-LLR/C2V 메모리 절감) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | LLR/메시지 양자화 테이블을 col-block 위치별로 세분화하는 아이디어와 유사, 이식 시 재검증 필요 |
| CN 정규화 min-sum(β=3/4) + Min1/Min2 finder | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 Min-Sum 계열(min1/min2) 보유, 정규화factor 값 자체는 참고 가능하나 중복 기여 |
| 압축 C2V 메시지 포맷(부호+Min1 index+Min1/Min2) | `decoder.cpp` `C2V_Cal()` | 메시지 압축저장 방식이 유사한 목적, 직접 대응 여부는 미확인 |
| RS/CC/LDPC-CC 간 재구성형 메모리·연산기 공유 | 대응 없음 | Prime ECC는 QC-LDPC 단일 부호 시뮬레이터로 RS/CC와의 재구성 공유 개념 없음 |
| LDPC-CC(convolutional, time-varying H) 부호구조 | `ecc_top.cpp` `Load_PCM()` | 부호 자체가 SC-LDPC 계열로 QC-LDPC(PCM_b)와 구조 상이, 부호설계 이식은 난이도 높음 |

> 적용 가치: 중간 - PE(또는 col-block)별 가변 비트폭 메모리 절감 아이디어는 Prime ECC의 LLR 테이블 구조에 참고할 만하나, 부호 자체가 LDPC-CC이고 표준 NMS 기반이라 정정성능 개선 기여는 없음.
