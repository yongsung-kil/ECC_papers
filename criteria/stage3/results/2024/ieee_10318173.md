### B. 알고리즘 요약

1. NAND Flash TLC 채널에서 GC-LDPC((21150,19050), local submatrix 3개) 코드로 hard/soft progressive decoding 및 read-retry 수행, RBER 0.003~0.017 범위.
2. 기존 FSGPD(고정 1-level 증가)/FMGPD(고정 m-level 증가)/DMGPD(인접 페이지 기반 동적 선택) 방식은 read-level 증가폭(granularity)이 실제 필요량과 안 맞아 불필요한 재복호나 과도한 read latency 유발.
3. GC-LDPC PCM `HGC`는 t개 `Hlocal`(대각 배치)과 `Hglobal`로 구성, general syndrome `S = HGC·C^T`가 t개 local parity check와 1개 global parity check로 분해됨(식 1,2).
4. 복호 실패 시 불만족 local parity check 개수 `L`이 적을수록 잔여 비트오류 `U`가 적다는 상관관계를 시뮬레이션으로 확인, `L`로 최적 read-level 증가폭 `PGL`을 선택(AGPD).
5. `L≤2`일 때는 `L`만으로 충분하지만 `L≥3`일 때 `U`의 분산이 커서, syndrome `S`의 비영원소 개수 `P`를 추가 지표로 사용해 `PGL,P`를 결정(개선판 IAGPD).
6. `P` 계산은 다수의 서로 다른 비트폭 adder와 9-bit comparator 2개로 구현, 기존 [8] 대비 decoder area 0.098% 추가·클록 1사이클 추가.
7. Algorithm 2(IAGPD): read level `i≤7` 범위에서 hard/soft decoding 후 `S, L, P` 계산, `S=0` 또는 `i=7`이면 종료, 아니면 `L`(≤2) 또는 `(L,P)`(≥3) 기반으로 `i`를 증분.
8. 결과: FSGPD/FMGPD(3종) 대비 flash read latency 평균 43%/21%/13%/15% 감소, DMGPD·AGPD 대비도 최대 15%/16% 추가 감소.
9. MQsim 기반 SSD 시뮬에서 read response time 최대 32%(FSGPD 대비) 감소, RBER·워크로드별 감소폭 상이(저RBER에서는 개선폭 작음).
10. 한계: GC-LDPC 특유 구조(local/global parity 분해) 전제이며, 정정능력·error-floor 자체 개선이 아니라 read-retry 스케줄링 최적화(latency 개선)에 국한. FPGA/ASIC 게이트카운트 상세치·throughput 언급 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 불만족 syndrome/parity check 기반 오류 추정(L, P) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 (syndrome 계산부) | 복호 실패 시 syndrome 상태로 재시도 판단하는 아이디어는 이터레이션 루프에 추가 가능하나, GC-LDPC local/global 분해 구조가 Prime ECC PCM_b와 상이해 직접 이식은 재설계 필요 |
| read-retry 시 read-level 증가폭 적응 선택 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold/양자화 관리부에 read-retry 정책으로 연동 가능 |
| 조기종료 판단(S=0) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념적으로 조기종료와 유사하나 이 논문은 CRC가 아닌 syndrome 직접 사용 |

적용 가치: **중간** — 정정능력 개선이 아닌 read-retry 스케줄링(latency) 최적화이며 NAND 직접 타깃이라는 점에서 실용적이나, GC-LDPC의 local/global parity 구조 의존성 때문에 Prime ECC(단일 QC-LDPC PCM)로 옮기려면 "local parity check" 대응 개념을 새로 정의해야 함.
