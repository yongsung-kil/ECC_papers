> 총평: 정보임베딩·상태채널(Gelfand-Pinsker)·Wyner-Ziv 문제를 위한 새로운 코드 클래스(WPC) 이론 논문으로, NAND binary LDPC 디코더/부호 이식과는 목적과 채널 모델이 근본적으로 달라 관련성이 낮음.

### B. 알고리즘 요약

1. 채널: 상태(state)가 인코더에만 non-causally 알려진 Gelfand-Pinsker 세팅, 비대칭 채널(예: Z-channel/flash memory), Wyner-Ziv 문제까지 통합 다루는 정보이론적 코드 설계.
2. 문제: 기존 nested linear code는 subcodebook을 0/미사용으로만 이분화해 상태에 최적 적응이 제한적이고, 비구조적 weighted codebook(Poisson functional rep.)은 효율적 인/디코딩이 불가능.
3. 핵심 가정: full-rank parity-check matrix `H`, 코드워드 bias `p`, 패리티 bias `q ∈ [0,1]^n`로 각 패리티체크 비트를 0/1로 고정하지 않고 "1일 확률"로 부여(`wq(u)=∏ q_i^{u_i}(1-q_i)^{1-u_i}`).
4. 핵심 기법: query function `f_H(p,q) = argmax_x w_p(x) w_q(xH^T)`로 MAP 유사 encoding/decoding, `q_i=0/1`이면 기존 nested linear code, `0<q_i<1`이면 "soft parity-check bit".
5. 핵심식: capacity 달성 조건 `E[H_b(Q)] = (1-H(X|S))/(1-R)` (정리 1) — 패리티 bias 분포 `P_Q`가 이 엔트로피 제약을 만족하면 임의의 (비)대칭 상태채널에서 capacity 달성.
6. 확장: threshold-linear 등 `P_Q` 선택 방식 여러 개 제시, 실험적으로 threshold-linear이 nested linear 대비 최대 27.3% 오류율 감소(n=20, binary-Hamming embedding).
7. Sparse 확장(§V): H를 sparse regular bipartite graph로 구성해 LDPC와 유사한 구조를 만들고, 패리티 비트 weight `q_j`를 가상의 fictitious codeword bit(LLR `log((1-q_j)/q_j)`)로 취급해 표준 벨리프 프로파게이션 업데이트식(19)(20)(22)에 통합.
8. 최적화: dense 코드는 brute-force 완전탐색(지수시간)으로 encoding/decoding, sparse 코드는 random encoding(8000 샘플 중 최댓값 선택) + BP 디코딩(5회 재시도).
9. 결과(n=1000 sparse, degree 11, BSC β=0.05): threshold-linear low-density WPC가 unweighted 대비 블록오류율 최대 26.7%(k=100) 감소.
10. 한계: dense 버전은 지수시간 알고리즘으로 n=20 초소블록에서만 검증, sparse 버전도 n=1000 Python 시뮬(pyldpc 기반)뿐이며 HW/실측 없음, 목표 채널이 상태채널·정보임베딩·Wyner-Ziv이지 NAND 저장 채널 최적화가 아님.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| WPC 부호 구성(weighted parity-check bit) | ecc_top.cpp Load_PCM() (부분 유사) | H-matrix 로드 개념은 있으나 "패리티 비트에 가중치 부여"라는 부호 정의 자체가 Prime ECC의 고정 binary QC-LDPC parity와 근본적으로 다름 |
| sparse WPC + belief propagation 디코딩 | decoder.cpp LDPC_Decoder(), C2V_Cal() (부분 유사) | BP 업데이트식(19)(20)은 표준 sum-product이나 fictitious bit로 weight를 LLR에 주입하는 방식은 Prime ECC의 min-sum 파이프라인에 대응 없음 |
| Gelfand-Pinsker/Wyner-Ziv 상태채널 모델 | 대응 없음 | Prime ECC는 AWGN/RBER/fixed-error NAND read 채널만 지원, 상태채널·정보임베딩 개념 자체가 없음 |
```

적용 가치: **낮음** — 정보이론적 capacity-achieving 코드 구성 논문으로 목적 채널(상태채널/비대칭채널/Wyner-Ziv)과 검증 방식(소블록 brute-force, Python BP 시뮬)이 NAND binary QC-LDPC 이식 관점과 거리가 멀다.
