### B. 알고리즘 요약

1. 시스템: QC-LDPC 인코더(FPGA, Xilinx Alveo U50, 8GB HBM), IEEE 802.16/5G-NR BG1/CCSDS 등 다양한 표준의 코드 패밀리 지원, 코드길이 768~2304bit, rate 1/2~5/6 테스트.
2. 문제: 기존 RTL 전문가 설계는 특정 코드/HW에 종속돼 유연성이 없고, 기존 HLS 기반 설계는 HLS-to-RTL 설계공간이 과도하게 커서 throughput·HW효율이 낮음(기존 VMM 기반 HLS 설계는 0.3~6Gbps, 20k~30k LUTs).
3. 핵심 가정: QC-LDPC 부호화는 GF(2) 위의 vector-matrix multiplication(VMM)으로 범용 처리 가능하며, 특정 코드에 최적화된 RU/LU 분해 대신 범용 VMM 구조를 채택해 코드 패밀리 전체(whole family)를 지원.
4. 핵심 기법 1단: Encode Block(EB) — LUT 레벨 최적화로 논리적 full connectivity 구현, 3개 입력 포트(정보비트/생성행렬 조각/0 또는 temp buffer)와 1개 출력 포트(XOR 연산 결과)를 갖는 기본 연산 단위.
5. 핵심 식/의미: LUT 사용량 `LUT_encoder = (NR·NC·NU·LUT_EB + WL·LUT_L + WS·LUT_S + α_LUT·LUT_control)·NM`, throughput `T_HEncode = NR·NC·NU·NM·T_Base` — NR(row)/NC(col)/NU(unit)/NM(module) 4개 파라미터로 자원-성능 트레이드오프를 선형 예측.
6. 핵심 기법 2단: EB를 head-to-tail 링 구조로 연결해 super column 방향 subword-encoding 파이프라인 구성(fan-in 방지), super row 방향은 데이터 의존성이 없어 EB 단순 복제로 병렬화, Encode Unit(EU) 간 병렬로 다중 코드워드 동시 인코딩.
7. 부수 트릭: DataLoad/DataSend Unit이 HBM과 AXI 인터페이스로 연결돼 fan-in/fan-out 문제를 완화, 3단 파이프라인(읽기-인코딩-쓰기)으로 Encode Module 구성, HBM pseudo channel별로 Load/Send Unit을 분리해 채널 병렬화.
8. 최적화 절차: 자원이용모델·성능모델 기반 "design space analyzer"가 LUT 제약·throughput 요구에 맞춰 NR/NC/NU/WL/WS/NM 파라미터 조합을 자동 탐색(수동 파라미터 튜닝 노동 회피). 모델 예측 오차는 throughput +16~39%, LUT +23~47%.
9. 결과: Xilinx U50에서 최대 throughput 922.66 Gbps 달성, 기존 대비 peak throughput 154.5배, HW효율 5.89배 향상. 5G-NR BG1/CCSDS 표준에서 각각 30.01/47.85 Gbps 실측.
10. 한계: 통신 표준(5G/802.16/CCSDS) 대상 대규모 데이터센터 FPGA(U50) 실험이며 NAND/SSD 타깃 실험 없음, 디코더는 다루지 않고 인코더만 대상, LUT 자원소모(최대 403k)가 매우 커 NAND 컨트롤러의 저전력·저면적 제약과는 상충.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VMM 기반 QC-LDPC 인코더 아키텍처(EB/EU) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 구조라 범용 VMM 인코더로 교체 시 구조 자체가 크게 달라짐(이식 난이도 중~높음) |
| 코드 패밀리 전체 지원(코드 가변성) | `ecc_top.cpp` `Load_PCM()` | 다중 코드 파라미터 대응이라는 설계 철학은 유사하나 Prime ECC는 특정 QC-LDPC 구조 고정 |
| Design space analyzer(자원-성능 모델) | 대응 없음 | Prime ECC 코드베이스에 HW 파라미터 자동탐색 도구 없음 |
| HBM 채널 병렬화, 파이프라인 스케줄링 | 디코더 HW 모델(z=32, pipeline 6-stage) | 파이프라이닝 개념은 유사하나 인코더 전용이며 메모리 구조(HBM vs NAND 컨트롤러 SRAM/DRAM)가 상이 |

적용 가치: `중간` — NAND 직접 타깃은 아니지만 QC-LDPC 인코더의 병렬화·파이프라이닝 설계 원리는 일반적으로 참고 가능하며, 특히 Prime ECC의 고정 dual-diagonal 인코더를 범용화하려는 경우 EB/EU 개념이 참고가 될 수 있음. 다만 자원 규모와 표준 종속성으로 직접 이식은 제한적.
