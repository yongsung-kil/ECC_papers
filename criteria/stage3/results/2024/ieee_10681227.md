> 총평: 디코더 알고리즘 변경 없이 read reference voltage(HDL) 최적화로 iteration·latency를 크게 줄이는 실측 기반 기법으로, Prime ECC의 channel/soft-read threshold 레이어에 부분 이식 가능.

### B. 알고리즘 요약

1. 대상: Micron B16A 3-D TLC NAND, LDPC code rate `8/9`, code length `18432`, min-sum hard-decision decoding, 최대 iteration 40.
2. 문제: RBER이 높아질수록 hard decision decoding에 필요한 iteration 수가 증가해 read latency가 커지지만, 실제 RBER은 사전에 알 수 없어 다수의 read retry가 필요.
3. 가정: decoding iteration 수 `Niter`는 RBER과 비례(`Niter ∝ RBER`)하며, iteration 수만으로 RBER을 간접 특성화할 수 있다는 실측 관찰에 기반.
4. 핵심 기법 1단: default HDL과 optimal HDL 간 offset `V_offset = |V_DHDL − V_OHDL|`을 정의하고, TLC의 7개 overlapping zone별로 iteration `Xi`와 offset `Yi`의 선형 피팅 함수 `Yi = δ·(ai·Xi + bi)` (Eq.3, δ=1/15) 를 오프라인 실측으로 구축.
5. 이 함수모델은 "1차 hard read로 얻은 iteration 수 → 최적 HDL offset 추정"을 가능하게 해 read retry 없이 2차 read로 성공률을 높임.
6. 확장 1(AWLD): 블록 내 모든 wordline을 읽어 각 wordline의 iteration으로 offset을 계산·재조정(정확하지만 a priori read 오버헤드 큼).
7. 확장 2(OWLLD): 블록 내 wordline 200~300번대가 특성이 유사함을 이용해 대표 wordline 1개만 읽어 블록 전체 offset을 추정(오버헤드 감소).
8. 확장 3(OPLLD): 한 wordline 내에서도 LSB 페이지(1-level read)의 iteration만으로 나머지 6개 overlap 영역의 offset을 재특성화(Eq.4, `Fj: Yj = αj·YLSB4 + βj`)해 랜덤 소량 read에 최적화, a priori read level을 6배 감소.
9. 결과: OPLLD가 iteration을 최대 94.63% 감소, 총 latency를 최대 66.89% 감소(LSB 기준), 저장 오버헤드는 56 bytes(256GB 칩 대비 56/2^38)로 매우 낮음.
10. 한계: 실제 HW(RTL) 구현 없이 ZYNQ-7000 SOC 테스트 플랫폼상 실측 검증뿐이며, 특정 Micron B16A TLC 칩 특성에 종속된 선형 피팅 모델이라 다른 공정/칩으로의 일반화는 추가 재보정이 필요.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|
| iteration-offset 함수모델 기반 HDL 조정 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | hard-decision read voltage(threshold) 조정 로직과 직접 대응 |
| iteration 수 기록/활용 | `decoder.cpp` `LDPC_Decoder()` | 디코더가 반환하는 iteration count를 offset 계산에 활용하는 구조와 연결 |

| min-sum 디코딩 방식 | `decoder.cpp` `CNU_Update_New_Mag()` | 논문은 기존 min-sum을 그대로 사용, 변형 없음(참고용) |

> 적용 가치: 중간 | 디코더 알고리즘 변경 없이 채널단 read voltage offset 추정 로직만 이식 가능하나, 이는 이미 Prime ECC가 갖춘 적응형 LLR/threshold 테이블과 기능적으로 겹치는 영역이라 부가가치는 제한적.
