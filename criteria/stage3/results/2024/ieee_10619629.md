> 총평: (v,w)-regular LDPC/MDPC 코드의 2-iteration bit-flipping 복호에 대한 DFR을 정밀하게 예측하는 확률모델 논문으로, post-quantum 암호(LEDAcrypt) 파라미터 설계가 목적이며 새로운 복호 알고리즘이나 HW 기법은 제안하지 않음.

### B. 알고리즘 요약

1. 대상은 `(v,w)-regular` 이진 코드(모든 행 가중치 `w`, 모든 열 가중치 `v`인 parity-check 행렬 `H`)로, LDPC(`v,w=O(log n)`)와 MDPC(`w=O(√(n log n))`)를 모두 포괄하며 특히 post-quantum 암호(BIKE, LEDAcrypt)의 QC-LDPC/QC-MDPC 코드를 타깃.
2. 풀려는 문제: 암호 시스템은 능동적 공격자 방어를 위해 DFR을 `2^-128` 이하로 보장해야 하는데, 이 수준은 Monte Carlo 시뮬레이션으로 직접 측정 불가능하므로 닫힌 형태(closed-form) DFR 추정식이 필요.
3. 채널/디코더 모델: Gallager의 2-iteration parallel bit-flipping 디코더를 그대로 사용(`upcj`=unsatisfied parity-check count, threshold `th`, `e¯j ← e¯j⊕1` if `upcj ≥ th`); 채널 자체는 순수 오류벡터 `e`(weight `t`)로 모델링.
4. 핵심 기법 1단: syndrome 가중치 분포 `Pr(Wt=y)`를 `r+1`상태 비균질 이산시간 Markov chain(`wp(l)=wp(l-1)·P(l)`)으로 모델링, 전이확률은 hypergeometric 분포 기반 flip 확률(`π^{l-1→l}_{flip,0→1}`, `π^{l-1→l}_{flip,1→0}`)로 유도.
5. 핵심 식: 1차 반복 후 오류 불일치 개수 `Pr(E(1)=d|Wt=y)`를 `pflip|0`, `pflip|1`(각 `upcj≥th`가 될 이항분포 확률)로부터 직접 counting argument로 계산 — 이는 기존 Markov chain 근사(Chaulet, Vasseur)보다 정확도가 개선됨.
6. 핵심 기법 2단: 2차 반복은 1차 결과 비트를 4개 클래스 `(a,b)=(ej, ej⊕e¯(1)_j)`로 분할해 각 클래스별 flip 확률(`pflip|00, pflip|01, pflip|10, pflip|11`)을 유도하고 최종 `DFR=1-Pr(E(2)=0)` 산출.
7. 부수 트릭: threshold `th`는 majority rule(`⌈(v+1)/2⌉`)을 사용하며, 기존 code-specific correction guarantee(Tillich의 `τ`)와 결합해 waterfall/floor 영역을 구분 추정 가능.
8. 학습/최적화 절차 없음(순수 확률모델 유도 + 수치검증); C 코드로 10^9 syndrome 샘플, 50k core-hour급 시뮬레이션으로 검증.
9. 결과: LEDAcrypt NIST category 1 파라미터에서 기존 추정(`2^-64`) 대비 실제 DFR을 `2^-134`~`2^-147`로 더 타이트하게(270배 이상 개선) 추정, syndrome weight 분포/1차반복 오류분포 모두 시뮬레이션과 잘 일치.
10. 한계: 새로운 복호 알고리즘이나 정정능력 개선이 아닌 기존 2-iteration bit-flipping의 확률적 성능 "예측" 도구일 뿐이며, HW 미설계, 순수 통계모델(waterfall은 신뢰 추정, floor는 보수적 추정에 그침).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping (upc-threshold) 디코딩 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum soft decoder가 주력이라 hard-decision bit-flipping은 별도 알고리즘 계열(대응 없음에 가까움) |
| syndrome/DFR 확률 모델링 | 대응 없음 | 순수 이론적 실패율 예측 도구로 HW/알고리즘 모듈에 대응 없음 |
| 2-iteration 병렬 threshold 갱신 | `decoder.cpp` `VN_Cal_HD()` | HD(hard-decision) 갱신 개념만 유사, threshold 기반 bit-flip 로직 자체는 min-sum과 다른 계열 |
| post-quantum 암호 DFR 보장 목적 | 대응 없음 | NAND ECC의 목적(random error 정정)과 달리 암호학적 부채널/키유출 방지가 목적 |

적용 가치: **낮음** — post-quantum 암호용 DFR 정밀 추정 이론이며 새로운 디코더/HW 기법이 아니어서 Prime ECC(soft-decision min-sum 기반 NAND ECC)에 이식할 실질적 요소가 없음. 다만 hard-decision bit-flipping 계열 자체가 Prime ECC의 HD 모드 성능 예측/failure rate 분석 방법론으로 참고 가치는 존재.
