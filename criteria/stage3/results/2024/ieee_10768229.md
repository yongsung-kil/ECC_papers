### B. 알고리즘 요약 (10줄 내외)

1. McEliece 포스트양자암호 시스템의 MDPC(medium-density parity-check) 부호 디코더 대상, `(n0, r, w)` 파라미터(n0=2~4, r=3079~32771, w=45~161)로 정의되는 무작위 대형 circulant 행렬 기반 secret key 구조.
2. 기존 row-layered min-sum 디코더[13]는 병렬도 p가 커질수록(예 p=64) 조건을 만족하는 secret key 탐색이 어려워지고 DFR(decoding failure rate)도 증가해 병렬성 확장에 한계가 있음.
3. row-layered scheduling: H를 L행 단위 layer로 나누고 각 layer 컬럼당 비영 원소 1개 제약(기존 기법[16])을 가정, 인접 비영 원소 간 최소거리 p 제약으로 p×p identity block 동적 분할.
4. 핵심 기법1: 동일 layer 내 identity block m1(>1)개를 동시 처리하는 다중블록 병렬처리 — a posteriori 갱신 `γ̃' = γ̃ - αv(k-1,f) + αv(k,f)`을 다중 block에 대해 병렬 수행, RAM U를 b개 뱅크로 분할해 메모리 충돌 최소화.
5. 이 식은 최신 c2v 메시지를 즉시 반영(row-layered)해 수렴을 가속하는 것이 핵심이며, m1·L개 항목을 한 클록에 읽으려면 뱅크 수 b가 병목이 됨.
6. 핵심 기법2(확장): 가변폭 large block 처리 — layer 크기 L>p로 설정, 컬럼당 다중 비영원소 발생 시 DFR 저하를 막기 위해 block을 세그먼트로 나눠 최신 c2v를 항상 반영하는 하이브리드 스케줄링 도입(DFR 저하 없음).
7. 부수 구현 디테일: CNU A(min1/min2/idx/sign 계산, 2-input 비교기 5개)와 CNU B(c2v 메시지 복원)로 분리, RAM I/M/S/U/T 5종 메모리뱅크 구조로 인덱스·메시지 관리.
8. 학습/최적화 절차 없음(순수 알고리즘/HW 설계, 무작위 생성 MDPC 코드 10개 평균으로 speedup 측정).
9. 결과: 예시 코드(n0=2, r=4801, w=45, p=32)에서 기존 대비 약 40~44% speedup(정규화 44.3~46.2배), 메모리 크기 5% 증가, 전체 면적(XOR게이트 환산) 6~9% 증가.
10. 한계: 이론/게이트 카운트 분석 수준(FPGA/ASIC 실측 없음), MDPC는 LDPC와 달리 error correction용이 아닌 암호 secret key 구조라 error-floor/waterfall 성능 개선 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| row-layered min-sum 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `Dual-Update`(parity even/odd 교대) | Prime ECC의 Dual-Update 스케줄과 유사 목적(최신 메시지 즉시 반영)이나 MDPC 전용 layer 구조라 직접 대응 어려움 |
| CNU min1/min2/sign 계산(Algorithm 1) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 동일한 min1/min2 기반 min-sum 구조 보유, 중복 기여 |
| 다중 identity block 동시처리(병렬 CNU 확장) | 디코더 HW 모델 (z=32, HCU) | Prime ECC는 이미 z=32 고정 병렬구조라 MDPC의 동적 p 확장 기법과 목적이 다름 |
| RAM U 뱅크 분할(메모리 충돌 최소화) | 디코더 HW 모델 (메모리/라우팅) | 메모리 뱅크 설계 아이디어는 참고 가능하나 MDPC 무작위 행렬 특성에 종속 |
| 부호구조(무작위 대형 circulant, secret key) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC 고정구조가 아닌 암호용 MDPC라 대응 없음 |

> 적용 가치: 낮음 — 부호 자체가 NAND 표준 QC-LDPC가 아닌 암호(McEliece)용 MDPC이며 목적(secret key 보안)도 다르나, row-layered min-sum 병렬 스케줄링과 CNU 다중입력 처리 아이디어는 HW 최적화 관점에서 제한적으로 참고 가능.
