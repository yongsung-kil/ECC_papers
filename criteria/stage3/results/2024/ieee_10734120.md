### B. 알고리즘 요약

1. 시스템: 5G NR QC-LDPC(BG1/BG2, base graph + 순환치환 shift matrix, lifting size Zc)를 AWGN·BPSK 채널에서 min-sum 알고리즘(MSA)으로 layered decoding.
2. 문제: 전통적 layered LDPC는 레이어 간 순차 의존성으로 iteration당 지연이 크고, 이를 병렬화한 LP-LDPC(layered parallel)는 동일 변수노드가 여러 레이어의 체크노드에 동시 연결될 때 LLR 갱신 충돌(message passing conflict)이 발생.
3. 핵심 가정: H-matrix를 Zc행 단위의 L개 레이어로 분할, 각 레이어는 shift matrix P의 열(column) 값을 기준으로 non-negative 값이 같은 행들을 그룹화하여 충돌 없는 다중레이어 집합 S를 구성(Algorithm 1).
4. 핵심 기법: MLP-LDPC는 집합 S1부터 순차적으로 처리하되, 각 집합 내부의 레이어들은 LP-LDPC처럼 완전 병렬 처리 — layered(순차)와 LP(완전병렬)의 절충.
5. 핵심 식: CN 갱신은 표준 min-sum `r_j,i = sgn(q_j,i')·min|q_j,i'|` (식 3/10) 그대로이며, 개선점은 레이어 스케줄링(집합 구성)뿐, 디코딩 수식 자체는 변경 없음.
6. 확장: Algorithm 2(MLP-LDPC 디코딩)는 집합 Sζ별로 레이어를 병렬 처리 후 APP(p_i) 값을 다음 집합으로 즉시 전달, 전체 H matrix에 대해 syndrome check(H·ĉ^T=0)로 조기종료.
7. 부수 구현: KyMLP-LDPC 시스템은 Kyber(post-quantum KEM, 512/768/1024 보안레벨)로 데이터를 사전 암호화한 뒤 비트변환→블록분할(K=256)→LDPC 인코딩→BPSK→MLP-LDPC 디코딩→복호화 파이프라인 구성(Algorithm 3).
8. 최적화 절차 없음(학습 기반 아님) — 레이어 그룹화는 결정론적 알고리즘(Algorithm 1)으로 오프라인에 1회 계산.
9. 결과: code rate 1/3~2/3에서 LP-LDPC 대비 레이어 충돌 수가 대폭 감소(예: R=1/3에서 11→4건), block당 평균 실행시간(AET)이 layered/LP-LDPC보다 낮고 BER도 개선(K=256, 15 iteration 상한, SNR 0.5~3.0dB).
10. 한계: 순수 소프트웨어 시뮬레이션(AET는 실행시간 측정치로 HW clock/gate 단위 아님), HW 파이프라인/합성 결과 없음, Kyber 결합부는 채널코딩과 독립적인 암호화 레이어로 ECC 이식과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Multi-layered set 스케줄링(레이어 충돌 최소화 병렬화) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 레이어/행 그룹을 non-conflict 집합으로 재배치하는 아이디어는 Prime ECC의 row 단위 병렬(z_sb=32) 스케줄링과 개념적으로 유사, 참고 가능 |
| CN min-sum 연산(표준 MSA, 변경 없음) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문에서 알고리즘 자체는 그대로 사용, 이미 보유 기법과 중복 |
| Graph Thinning / HCU 등 HW 특화 병렬 구조와의 결합 | `GT.cpp` `Make_GT_HW()` | 논문의 레이어 집합 구성이 Prime ECC의 MUX 기반 edge 선택과 유사한 목적(충돌·자원 최소화)이나 직접 매핑은 확인 안 됨 |
| Kyber 포스트양자암호 결합 | 대응 없음 | ECC 디코더/부호설계와 무관한 암호화 레이어 |

적용 가치: `중간` — 레이어드 디코딩의 병렬 스케줄링(충돌 회피 다중레이어 집합) 아이디어는 Prime ECC의 row-병렬/HCU 구조 설계에 참고할 만하나, 알고리즘 자체(min-sum)는 표준 그대로이고 HW 검증이 없어 직접적 게이트/throughput 이득은 확인 불가.
