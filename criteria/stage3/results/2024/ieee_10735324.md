### B. 알고리즘 요약 (10줄 내외)

1. 대상 채널: ternary dual-parameter barrier channel — 파라미터 `(p,q)`, downward error(비0→0)와 upward error(0→비0)만 존재하고 비0↔비0 전이는 없음.
2. 기존 방식(두 constituent code를 순차적으로 bounded-distance 디코딩)은 두 코드 간 상호 의존성을 활용하지 못해 성능이 제한됨.
3. 코드 구성: ternary 코드 `C = Θ ⊗ Λ` — indicator code `Θ`(rate `kI/n`)와 residual code `Λ`(rate `kR/n`) 두 개의 binary irregular LDPC로 구성, 각각 `ı(·)`, `ψ(·)` 매핑으로 정의.
4. 핵심 기법: 두 코드의 parity-check를 표현하는 bilayer Tanner graph(`U(1)`-indicator check, `U(2)`-변수노드 공유, `U(3)`-residual check) 위에서 fully-joint iterative message-passing decoder 설계 (Algorithm 1).
5. 핵심 식: 변수-체크 메시지에 상대 코드의 정보를 변환 삽입하는 함수 `T(1)(x)=-ln(1/2+1/2*e^-x)`(residual→indicator), `T(3)(x)=ln(1/3+2/3*e^x)`(indicator→residual) — 두 코드 간 상관관계를 LLR 도메인에서 교환.
6. 디코딩 순서는 indicator iteration `tI=6`회, residual iteration `tR=2`회를 번갈아 수행하는 고정 시퀀스(`s`), 총 30 iteration.
7. 초기 LLR은 채널 파라미터 `(p,q)` 기반 closed-form(Proposition 1)으로 설정, 조합적 편향 방지를 위해 표준편차 `0.05*m^(0)`의 랜덤 섭동 추가.
8. 별도 학습/최적화 절차 없음(파라미터는 실험적 그리드 탐색으로 tI/tR 고정).
9. 결과: n=192, 256 길이에서 제안 joint decoder가 기존 순차 2-step 디코더 대비 BLER 전 구간에서 우세, 목표 BLER 10^-4에서 더 높은 q(upward error율) 허용.
10. 한계: 순수 시뮬레이션(HW 미설계), ternary 비대칭 barrier 채널 전용 모델이라 alphabet 크기(Q>3) 일반화나 NAND의 binary/soft-read 채널 모델과는 직접 대응하지 않음.
