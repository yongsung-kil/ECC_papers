### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 5G URLLC/mMTC용 짧은 코드(N=208~520, rate 0.2~0.625) protograph 기반 raptor-like(PBRL) QC-LDPC, AWGN+BPSK 채널, log-BP 디코더(50 iteration).
2. 문제: 기존 5G eMBB용 PBRL LDPC(BG1/BG2)는 URLLC/mMTC의 짧은 transport block 크기에 최적화되지 않아 성능 열세.
3. 핵심 아이디어: PBRL 구조 `HPL=[[H1,O],[H2,I]]`의 core part H1을 quadratic residue(QR) 코드의 패리티체크행렬 기반 QC-LDPC(`H_QR`)로 확장 구성, girth≥6 정리(Theorem 3, 6, 7) 이론적으로 증명.
4. 핵심 기법 1단: degree distribution 최적화(`ν(r,x)`, `µ(x)`) 후 QR 기반 base matrix `B̃`에서 short-cycle(4/6/8-cycle) 최소화 서브행렬 선택 → masking algorithm(Algorithm 1)으로 EMD(extrinsic message degree) 기준 cycle 제거하며 목표 degree distribution 충족.
5. 핵심 식: cycle 존재조건 `Σ(w_is,js − w_is,js+1) ≡ 0 (mod q)` 기반으로 short-cycle을 계산해 최소화하는 서브행렬/마스킹 원소를 선택.
6. 핵심 기법 2단: `(κ,0)`~`(κ,2)` trapping set 개수가 최대인 열을 filler column으로 지정해 나머지 부분의 error-floor 특성 개선, punctured column은 전수탐색(Algorithm 2)으로 B1-B2 간 short-cycle 최소화.
7. 부수 트릭: base matrix B2는 기존 QC-PEG + multipath EMD 전략을 결합해 구성, 3GPP 규격상 filler/punctured column 위치 제약을 만족하도록 column permutation(`σ_fill`, `σ_punc`) 적용.
8. 최적화 절차: 학습이 아닌 조합적 탐색(전수 permutation 탐색으로 cycle 최소 조합 선택), 복잡도는 `C(b,b-c)` 규모로 가장 시간 소모적.
9. 결과: 제안 PBRL LDPC(Code I~IV, q=4,6,8,10)가 5G BG2 표준 코드 대비 BER=1e-5에서 최대 0.63dB 이득, 여러 code rate(0.2/0.4/0.625)에서도 우수.
10. 한계: HW 설계·복잡도/throughput 분석 없음, 순수 시뮬레이션(BLER/BER)만 수행, 짧은 5G URLLC 코드 규격(3GPP TS 38.212) 종속 설계로 NAND 고rate 코드에 직접 적용 불가.

| QR-QC-LDPC 기반 base matrix 구성 / girth-6 보장 | `ecc_top.cpp` `Load_PCM()` | 부호구조(H-matrix) 생성 단계로 대응되나 QR 코드 기반 구성법 자체는 이식 난이도 높음(프로파일 §4) |
| Masking 기법(EMD 기반 short-cycle 제거) | `ecc_top.cpp` `Load_PCM()` | H-matrix 설계 단계에 참고 가능하나 코드 구조 재검증 부담 큼 |
| Trapping-set 기반 filler column 선택(error-floor 개선) | `ecc_top.cpp` `Load_PCM()` (H-matrix 구조) | error-floor 대책 관점에서 부호설계 참고 가치 있음, 단 decoder.cpp 알고리즘과는 무관 |
| PEG 기반 B2 확장 구성 | `ecc_top.cpp` `Load_PCM()` | 표준 PEG 변형으로 재사용 수준의 구성법 |

적용 가치: `중간` — NAND 고rate binary QC-LDPC 부호설계 시 short-cycle/trapping-set 최소화 기법(masking, EMD, 전수탐색)은 참고 가능하나, HW/디코더 레이어 기여가 없고 5G 짧은 코드 규격에 특화되어 직접 이식은 제한적.
