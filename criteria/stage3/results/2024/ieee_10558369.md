### B. 알고리즘 요약

1. 대상 부호는 IEEE 802.15.3d THz 표준 QC-LDPC, coding rate `14/15`, H는 `96x1440`(column degree 3, row degree 45), 96x96 circulant submatrix로 구성되며 16-QAM 변조.
2. 문제의식은 기존 layered schedule이 고정된 레이어 순서로 갱신하다 보니 BER이 낮아진 후에도 이미 정정된 variable node를 불필요하게 갱신해 클럭 사이클을 낭비한다는 점.
3. 채널 모델은 AWGN + 16-QAM, sub-THz 대역(최대 100Gbps급) 고속 무선통신을 가정.
4. 핵심 기법은 partial syndrome-based dynamic decoding(PSDD): 레이어 `l`의 partial syndrome `S_l = D_hat_l ∘ H_l`(식 9)이 0이면 해당 레이어의 인접 variable node들이 이미 정정되었다고 보고 그 레이어 갱신을 스킵.
5. 이 partial syndrome은 어차피 layered 알고리즘도 계산하는 전체 syndrome의 부분집합이라 추가 HW 오버헤드가 미미하며, BER 성능 저하 없이(동일 max iteration, 4-layer 조건) 클럭 사이클만 절감한다는 것이 핵심 의의.
6. 확장으로 check node 처리에서 45개 입력 중 `m1,m2,m3`(1st~3rd minimum)을 찾는 병목을 해결하기 위해 divide-and-conquer 소팅(45→15→9→3 단계, 5x3-MIN/9x3-MIN 비교기)을 rejection-based tree 기법을 확장해 설계.
7. 구현 디테일은 6bit 고정소수점 양자화(부동소수점 대비 0.1dB 손실)와 3-frame multi-frame interleaved pipeline(3 pipeline stage, VNR 3개 각 1440노드x6bit=8640bit, register block 사용 SRAM 미사용)으로 고클럭 달성; PSDD shifter network가 partial syndrome에 따라 posteriori LLR을 24/48/72bit 단위로 circular shift.
8. 별도 학습/최적화 절차는 없음(스케줄링 알고리즘 및 HW 아키텍처 설계).
9. 결과: 16nm FinFET 칩 실측으로 throughput 128.5Gbps, coding gain 10^-8@SNR 18.1dB, 기존 [7](Layered, 유사 표준) 대비 약 1/4 전력, core area 78% 이상 감소, gate count 520K(비교 대상보다 작음), area efficiency 200.78Gbps/mm².
10. 한계: BER 성능 자체는 기존 layered 알고리즘과 "동일"(개선 아님, 사이클/전력만 절감)하며, IEEE 802.15.3d 특정 고정 H-matrix(rate 14/15, length 1440)에 최적화되어 일반화 검증은 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PSDD (partial syndrome 기반 레이어 스킵 dynamic 스케줄) | `decoder.cpp` `LDPC_Decoder()`, Dual-Update 스케줄 | 이터레이션 루프 내 레이어별 syndrome 재사용으로 불필요 갱신 스킵 아이디어는 Dual-Update 스케줄에 낮은 난이도로 이식 가능 |
| 3-min 탐색 divide-and-conquer 소팅 아키텍처 | `decoder.cpp` `CNU_Update_New_Mag()` | min1/min2(+min3) 탐색 로직 개선으로 CNU 연산부에 직접 대응, 높은 관심 영역(§5 min-sum 성능/HW 개선) |
| syndrome 계산/early termination | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념(전체 syndrome 0일 때 종료)은 이미 Partial/Full CRC로 보유, PSDD의 레이어별 부분 syndrome은 추가 세분화 아이디어 |
| multi-frame interleaved pipeline (3-frame) | `decoder.cpp` pipeline 6-stage 구조 | 기존 pipeline 구조와 병행 적용 가능성 있으나 정합 재검증 필요(중간 난이도) |
| 6bit LLR 고정소수점 양자화 | `channel.cpp` LLR 테이블(`PARAM_LLR`) | soft-4bit+ 영역으로 이미 보유한 양자화 체계와 유사, 참고용 |

> 적용 가치: 높음. min-sum 3-min 소팅 HW와 partial-syndrome 기반 레이어 스킵 스케줄링은 이미 보유한 min-sum/Dual-Update 스케줄과 직접 결합 가능한 낮은 난이도 영역이며, 128.5Gbps 실측(16nm) 검증까지 갖춰 참고가치가 크다.
