### B. 알고리즘 요약

1. 대상 부호는 LDPC CCSDS `(128, 64)` short code(rate 0.5), AWGN+BPSK 채널, normalized min-sum(NMS) `T=12` iteration 디코딩을 기본으로 사용.
2. short LDPC는 Tanner graph의 short cycle 때문에 NMS/BP가 ML 성능에 도달하지 못하는 gap이 존재, 이를 메우기 위해 NMS 실패 시에만 OSD(ordered statistics decoding)를 적용하는 하이브리드가 필요.
3. 핵심 가정은 NMS 디코딩 실패 이력(iteration별 a posteriori LLR trajectory)에 유의미한 정보가 남아 있다는 것.
4. 핵심 기법 1단: DIA(Decoding Information Aggregation)라는 4-layer CNN이 NMS 실패의 반복별 LLR 궤적(크기 `T+1`)을 입력받아 codeword bit의 재측정값 `y˜`를 산출.
5. 핵심 식: OSD 판정기준 `c̃ = argmin Σ 1(ri(2)≠ci)|yi(2)|` (식 1) — MRB(most reliable basis)를 정렬 후 최소 가중 Hamming distance를 갖는 TEP(test error pattern)를 선택.
6. 핵심 기법 2단: 전체 TEP를 통계(NMS 실패 데이터에서의 실제 오류패턴 분포)기반으로 정렬한 "decoding path"의 TEP block 순서 `bi`를 사전 구성(카테고리화+우선순위화).
7. 부수 트릭: SWA(Sliding Window-Assisted) 2-layer 신경망이 decoding path를 슬라이딩 윈도(`wd=5`)로 훑으며 soft margin `sm`에 따라 조기종료 여부를 판정(Algorithm 1).
8. 학습: DIA는 표준 CE loss, SWA는 클래스 불균형 보정 가중치 `αi`와 조기종료 오판 페널티 `βi=max(1,γ·1(...))`를 포함한 weighted CE loss로 Adam 최적화(20,000 step, 수 분 내 학습).
9. 결과: N-D-S(3,60,0.9)가 FER `10^-4`에서 NBP-D 대비 약 0.5dB, D10-OSD-2 대비 약 0.7dB 열위나 지연시간(ct) 면에서 SOTA 대비 크게 우세; NMS 복잡도는 iteration당 약 `2.88K FLOPs`.
10. HW 미설계, 시뮬(Python/TensorFlow)만 검증됐고 short block(N=128) 코드에 특화되어 있어 NAND용 장코드에서의 TEP 탐색 복잡도는 불명.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS(normalized min-sum) 실패 감지 후 후처리 트리거 | decoder.cpp `LDPC_Decoder()`, partial_CRC.cpp `partial_crc_HW_T4()` | 조기종료/실패판정 지점과 유사한 후크 위치, min-sum 자체는 이미 보유(§3) |
| DIA CNN(LLR 재측정) | decoder.cpp `Get_VNU_Table_Idx()`, ecc_data.h `PARAM_LLR` | 개념적으로 LLR 테이블 갱신과 유사하나 NN 기반이라 대응 없음(§5: NN 디코더는 대체로 대응 없음) |
| OSD(Gaussian elimination, MRB/LRB 기반 TEP 탐색) | 대응 없음 | Prime ECC는 min-sum 기반 반복 디코더로 Gaussian elimination 후처리 구조 없음 |
| SWA 조기종료 신경망 | partial_CRC.cpp `partial_crc_HW_T4()` | 조기종료라는 기능적 역할만 유사, NN 기반 구현은 대응 없음 |
```

> 적용 가치: `낮음` — 핵심 기여가 신경망 기반 OSD 후처리이며 short block code 및 NN 추론 의존이라 Prime ECC의 bit-exact min-sum HW 구조에 이식하려면 근본적 재설계가 필요.
