### B. 알고리즘 요약

1. CV-QKD 정보 재조정(information reconciliation)에서 BI-AWGN 합성 채널을 가정, TBP(type-based protograph)-LDPC 코드(rate `R=0.01`, `N=998400` / `R=0.1`, `N=128000`)를 사용.
2. 문제: 초저율 코드의 FPGA 구현 시 높은 정밀도(fractional bit) 요구로 메모리 사용량이 과다, 기존 2단계 디코더[9]는 정밀도를 줄이는 대신 반복 횟수가 늘어 throughput 저하.
3. 핵심 가정: degree-1 VN 비율이 매우 높은 초저율 코드(예 R=0.01에서 CN의 98.9%가 degree-1 VN에 연결)라는 특성을 이용해 CN 업데이트를 근사.
4. 핵심 기법 1단: box-plus 연산의 보정항 `s(γ1,γ2)`을 Taylor 전개+기하급수로 근사해 CN 업데이트를 `L[c←] ≈ (Πα_k)·γ_m·Π tanh(γ_ℓ/2)` (식 6)로 단순화 (`m`=최소 magnitude 인덱스).
5. 식(6)의 의미: 최소 magnitude 값 `γ_m`이 지배적이고 나머지는 tanh 스케일링 계수로만 작용 — 큰 LLR은 saturate되어 무시 가능.
6. 핵심 기법 2단(log-log domain SPA): LLR 자체가 아닌 `L˜=ln(|L|)`(log-LLR)과 부호 `α`를 메시지로 사용, CN/VN 업데이트를 log-sum/log-difference-exp 함수(`f±`)와 구간별 선형근사 `g(x)`(식 11, LUT 불필요)로 재구성.
7. 구현 디테일: 고정소수점 표현에 상수 오프셋 `b`(=5, 시뮬로 결정)를 더해 log-LLR의 부호 비트를 제거하고 LLR 부호는 별도 비트로 저장, `g(x)` 구간별 선형계수는 HW에서 비교기+게이트로 구현 가능.
8. 별도 학습 절차 없음(b값은 그리드서치 형태의 시뮬레이션으로 선택).
9. 결과: R=0.01 코드에서 fractional bit 6비트로 floating-point SPA 근접 성능(FP SPA 8비트 대비 saturation 회피), R=0.1 코드에서 fractional bit 4비트로 FP SPA 7비트보다 우수, 전체적으로 fractional bit width 25~40% 이상 절감, 복잡도는 SPA와 유사 수준.
10. 한계: 순수 알고리즘/시뮬 수준(FPGA 언급은 배경뿐 실제 HW 미설계), CV-QKD 초저율(R=0.01~0.1) MET-LDPC 전용 근사로 NAND용 고rate(~0.9) 코드에는 근사 전제(degree-1 VN 비율 높음)가 성립하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| log-LLR 메시지 표현(비트폭 절감) | `channel.cpp` `Set_LLR_Th()` (soft-read LLR 양자화 테이블) | 메시지 표현 정밀도 절감이라는 목표는 유사하나 log-domain 재구성은 Prime ECC의 magnitude 양자화 테이블 구조와 근본적으로 다른 접근 |
| CN 업데이트 근사(box-plus → min 기반, 식 6) | `decoder.cpp` `CNU_Update_New_Mag()` | 형태상 min-sum과 유사하나 원 알고리즘은 SPA(tanh 기반) 근사이며 이미 Prime ECC는 min-sum 사용 중이라 중복 기여 |
| 로그 도메인 VN 업데이트(log-sum-exp) | `decoder.cpp` `VN_Cal_HD()` 등 | HD/2SD/3SD 양자화 방식과 다른 패러다임(연속 log-LLR)이라 직접 대응 어려움 |
| 초저율 protograph 부호 구조 | `ecc_top.cpp` `Load_PCM()` | 고rate QC-LDPC 전용 구조와 무관, 대응 없음에 가까움 |

> 적용 가치: 낮음 — SPA(합/곱) 기반 초저율 MET-LDPC 전용 근사 기법으로, Prime ECC 3.1은 이미 min-sum 기반 고rate binary QC-LDPC 디코더를 사용 중이라 알고리즘 패러다임과 부호율 영역이 맞지 않음.
