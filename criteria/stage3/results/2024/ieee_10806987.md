### B. 알고리즘 요약 (10줄 내외)

1. 이진 선형블록부호 `C[N,K]`를 DMC 채널에 적용, 수신 LLR `r`에서 하드결정 `z`와 test error pattern(TEP) `e=z-v`를 정의.
2. 문제: ML 복호는 NP-hard이나, soft-weight `γ(e)=Σe_i|r_i|`가 최소인 TEP를 찾는 것과 등가(식 7) — 짧은 부호에서 실용적 근사가 가능함을 배경으로 함.
3. 두 guessing 계열을 구분: GND(GRAND류, 전체 `e∈F2^N` 공간을 순차 추측)와 GCD(정보집합 `e_R∈F2^K`만 추측 후 `e_L=s-e_R·P^T`로 재인코딩).
4. 핵심 정리(Theorem 1): GCD는 ML 알고리즘이며, guess 횟수가 GND 이하임을 injective mapping 논증으로 증명.
5. 조기종료 조건 `γ(e_R)≥γ_opt`가 GCD 효율의 핵심(더 이상 개선 불가 판정 시 즉시 종료).
6. OSD를 GCD의 특수 형태로 재해석: MRB(most reliable basis) 정렬로 `γ(e_R*)`를 최소화하고 `γ(e_R)` 증가 속도를 높여 조기종료 유도.
7. OSD 변형 3종 소개: LC-OSD(확장 MRB+SLVA로 guess 수 감소), ROSD(staircase generator matrix로 GE 복잡도 감소), Quasi-OSD(RS 부호에 Lagrange 보간으로 GE 대체).
8. 최적화/학습 절차 없음 — 이론 증명 + 시뮬레이션 검증(Monte-Carlo).
9. 결과: eBCH[128,64], RM[128,64], RS[26,23]_25 시뮬에서 GCD/LC-OSD/ROSD/Quasi-OSD가 기존 OSD/GND 대비 평균 TEP 수를 크게 절감(FER 10^-3 기준), 코드율이 높을수록 GCD-GND 격차는 축소.
10. 한계: 모두 짧은 block code(N≤130) 대상 시뮬뿐, HW 구현 없음, off-line GE를 전제로 하여 LDPC 반복복호(message-passing) 구조와는 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GCD/GND/OSD 계열 ML 근사 복호 | 대응 없음 | Prime ECC는 min-sum 반복 message-passing 기반 LDPC 디코더로, GE·재인코딩 기반 guessing 계열과 알고리즘 패러다임이 상이 |
| soft weight 기반 TEP 정렬 | `channel.cpp` `Set_LLR_Th()` | LLR 기반 신뢰도 순서화 개념은 유사하나 적용 방식(MRB 재인코딩 vs iterative C2V/V2C)이 달라 직접 이식 불가 |
| 조기종료(γ(e_R)≥γ_opt) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 조기종료라는 목적은 유사하나 판정 기준(soft weight vs CRC)이 전혀 달라 대응 없음에 가까움 |
```

> 적용 가치: **낮음** — 짧은 block code용 OSD/GRAND 계열 이론이며 Prime ECC의 QC-LDPC min-sum 반복복호 구조와 알고리즘 패러다임이 근본적으로 달라 이식 접점이 거의 없음.
