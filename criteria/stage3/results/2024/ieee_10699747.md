### B. 알고리즘 요약 (10줄 내외)

1. 시스템은 OAM-SK(궤도각운동량 편이변조) 자유공간 광통신으로, LG(Laguerre-Gaussian) 빔이 대기 난류를 통과한 뒤 CNN으로 복조되고 LDPC로 최종 오류정정.
2. 풀려는 문제는 대기 난류에 의한 OAM 모드 분산으로 복조 정확도가 저하되는 것 — 단순 CNN 분류 결과만 쓰면 채널 신뢰도 정보가 손실됨.
3. 채널 모델은 power spectrum inversion 기법으로 생성한 랜덤 위상 스크린(von Kármán 굴절률 스펙트럼 `Φn(κ)`) 기반 대기 난류 시뮬레이션.
4. 핵심 기법은 PCNN-NMS: CNN이 출력한 분류 확률 `p_i`(비트 i가 1일 확률)로부터 LLR을 `L_i = log((1-p_i)/p_i)` 형태로 계산해 NMS 디코더 입력으로 사용.
5. 이 LLR이 채널 추정 신뢰도를 반영해 기존 hard-decision CNN 출력보다 정교한 초기 메시지를 NMS 디코더에 제공.
6. NMS(Normalized Min-Sum) 업데이트식은 `r_j^(it) = L_i + Σ(min|r_k^(jt-1)|)·sign(Π r_k^(jt-1))·α` 형태로 정규화 계수 `α`를 이용해 min-sum 근사 오차를 보정.
7. CNN 구조는 3개 conv layer(5x5 kernel) + 3개 pooling layer + softmax, ReLU 활성화, Dropout(0.5), Adam optimizer, cross-entropy loss로 학습.
8. 학습 절차는 48,000장 광 세기 이미지(16개 codeword당 3,000장)로 CNN을 지도학습해 97.3% 분류 정확도 달성.
9. 결과는 무-LDPC 대비 및 단순 CNN-NMS 대비 PCNN-NMS가 최저 ABER 달성(예: Z=1000m, Cn²=1e-13에서 ABER 1.14e-06까지 감소).
10. 한계는 순수 시뮬레이션(HW 미설계)이며, 대기 난류·OAM 광통신이라는 특정 채널·복조 방식에 종속되어 NAND 등 타 채널로 일반화가 검증되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS(Normalized Min-Sum) 디코딩 업데이트식 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 Min-Sum(정규화 미확인) 기반이라 개념적으로 근접하나 정규화 factor 세부 비교 필요, 중복 기여 가능성 |
| CNN 확률 기반 LLR 계산(`L_i=log((1-p_i)/p_i)`) | `channel.cpp` `Set_LLR_Th()` | soft-info 생성 방식 자체는 대응 가능하나 입력이 CNN 복조 확률(광통신 특화)이라 NAND read 채널과 무관 |
| CNN 복조기(이미지 기반 OAM 모드 인식) | 대응 없음 | NAND read 파이프라인에 대응하는 개념 자체가 없음(광통신 전용) |
```

적용 가치: **낮음** — NMS 알고리즘 개념은 Prime ECC의 min-sum과 유사하지만 이미 보유 기법과 중복이고, CNN 기반 LLR 생성은 광통신 특화 복조 과정에 종속되어 NAND read 채널에 이식할 근거가 없음.
