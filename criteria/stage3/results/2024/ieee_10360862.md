### B. 알고리즘 요약

1. AWGN·BPSK 채널에서 다양한 block length/rate의 LDPC 코드(DVBS-2 (16200,7200), QC-LDPC (9472,8192), 5G PBRL (3096,1032) 등)를 대상으로 neural min-sum 계열 디코더를 설계.
2. 기존 neural min-sum(N-NMS/N-OMS)은 Tanner graph의 edge마다 별도 가중치를 학습해 파라미터 수가 edge 수에 비례, 긴 코드에는 학습 메모리·저장 부담이 과도하다는 문제.
3. C2V update `u_{ci→vj}^{(t)} = β_{(ci,vj)}^{(t)} · sign(...) · min(...)` 에서 가중치 `β`를 check node degree `dc`/variable node degree `dv`별로 공유하는 node-degree-based weight-sharing(N-2D-NMS/N-2D-OMS)을 제안, degree가 클수록 평균 가중치가 작아지는 상관관계를 실증.
4. weight-sharing type 0(edge별 개별)~8(전체 공유)까지 8종 스킴 정의, type 1/2(check+variable node degree 결합)가 파라미터 수를 edge 수 대비 수백~수만분의 1로 줄이면서 N-NMS와 동등 성능 확보.
5. 학습 시 발생하는 gradient explosion 문제(체크노드 degree 클수록 역전파 gradient 누적 과다)를 발견, `∂J/∂u_{ci→vj}` 계산을 posterior `lv_j^{(t)}`만으로 근사하는 posterior joint training 제안(식 10, 11)으로 greedy training과 동등 성능·더 낮은 시간복잡도 달성.
6. RCQ(reconstruction-computation-quantization) 저bit 디코더와 결합한 W-RCQ(weighted RCQ): 기존 RCQ가 layer·iteration마다 별도 양자화기/역양자화기(Q/R) 쌍을 요구하는 대신, degree-specific 학습 가중치로 조정하고 Q/R 쌍은 소수(2~3개)만 사용해 메모리·라우팅 부담 절감.
7. Q/R 함수는 power-function 기반 비균일 양자화(`τ_j = C·(j/(2^{bc-1}-1))^γ`)로 파라미터화, QNN(quantized neural network)을 straight-through estimator(STE)로 학습.
8. (9472,8192) QC-LDPC 코드에서 4-bit W-OMS-RCQ가 5-bit OMS와 동등 FER(0.01dB 갭)이면서 하드웨어 자원(LUT/BRAM/routed net)을 크게 절감(Xilinx Zynq UltraScale+ MPSoC FPGA 구현, 자원 수치는 논문 Table III, 본문 서술은 상대적 절감으로만 언급).
9. 5G PBRL 코드(k=1032, rate 1/3~8/9)에서 4-bit rate-specific W-NMS-RCQ가 6-bit OMS 대비 0.1~0.15dB FER 개선(목표 FER 10^-6), rate-compatible(단일 디코더로 전 rate 대응) type-5는 저rate에서 성능 저하.
10. 한계: NN 학습(오프라인 훈련) 및 QNN 파이프라인이 필수이며, 논문 자체는 위성/5G 표준 코드 중심으로 NAND 특유의 read-retry·RBER 채널 실험은 없음. gatecount/throughput 수치는 표(Table III)에만 있고 본문 서술엔 구체 수치 없어 미상 처리.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| degree-specific 학습 가중치(β, α)로 CN/VN update 보정 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 스케일링 계수를 degree별 학습값으로 대체하는 아이디어는 CN 연산부에 이식 가능하나 NN 훈련 인프라 신규 구축 필요 |
| VN 처리 / 저bit 양자화(W-RCQ) | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD), `decoder.h` `Get_VNU_*` | 3~4bit RCQ 양자화 아이디어가 soft-2~3bit 처리부와 연관되나 Prime ECC는 soft-4bit+ 미지원이라 적용 범위 제한 |
| iteration별 양자화기(Q/R) 소수 쌍 사용 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 이미 iteration 구간별 LLR 테이블 보유(§3), 개념적으로 유사·중복 가능성 |
| QC-LDPC (9472,8192) FPGA 구현 자원 절감 | 디코더 HW 모델(z=32, HCU, GT) | 병렬화/HW 구조는 Prime ECC와 다르나 자원 절감 목적은 유사 |

적용 가치: **중간** — degree-specific 학습 가중치와 저bit RCQ 결합은 min-sum 성능·HW 자원 개선에 관련성이 있고 QC-LDPC 실증까지 있으나, NN 훈련 파이프라인 신규 구축과 재훈련 비용이 진입장벽이며 Prime ECC가 이미 유사한 iteration별 LLR 테이블 기법을 보유해 일부 중복.
