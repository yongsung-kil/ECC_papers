### B. 알고리즘 요약

1. 시스템: 3D TLC NAND flash(8 state, 3bit/cell), read reference voltage(RRV: Va~Vg)로 LP/MP/UP 페이지 상태 판별.
2. 문제: retention/P-E cycle 증가로 threshold voltage distribution(TVD)이 이동해 default RRV로는 오독 발생 -> read-retry 및 LDPC soft decoding 유발, 특히 WL(word line)별 최적 RRV가 상이해 다중 조정 필요.
3. 핵심 가정/모델: 연속 두 번의 read에서 RTN(random telegraph noise)에 의해 무작위로 뒤집히는 bit 수(RFBC)가 RRV offset level(RRVOL)과 정규분포 형태로 상관관계를 가짐.
4. 핵심 기법: RFBC-RRVOL 변환 모델 `RFBC = a1 * exp(-((x-b1)/c1)^2)` (Formula 1)을 실측 데이터로 피팅, 역함수로 RRVOL 예측(Formula 2).
5. 의미: 한 번의 추가 read(XOR)만으로 RFBC를 구해 최적 RRV를 즉시 추정, 기존 on-line 방식(순차 trial-and-error)보다 read-retry 횟수 대폭 절감.
6. 확장 기법: RRV grouping(같은 page type 내 여러 RRV를 하나의 offset coefficient로 묶음) + WL extension scheme(같은 block 내 WL간 초기 RRV 차이는 retention에 무관하게 거의 일정하다는 관찰 기반, 한 WL 보정값으로 다른 WL 추정).
7. 구현 디테일: LDPC hard decoding 실패 또는 iteration 임계값 초과 시에만 트리거(Algorithm 1), FIFO로 블록당 최대 5세트 RRVOL 데이터 저장, 계수 16bit/차이값 8bit로 저장.
8. 학습/최적화 절차: 최소자승법(least squares)으로 6000개 샘플 블록에서 (a1,b1,c1) 사전 피팅(RMSE 6.197, R²≥0.997).
9. 결과: 저장 오버헤드 15.26KB, EOL RBER 0.080%, 1회 read 성공률 99.89%(sentinel 대비 read-retry 평균 626.2회→2.46회로 감소).
10. 한계: LDPC 부호/디코더는 완전 블랙박스(트리거 신호로만 사용), 특정 Micron B17A 3D CuA 플래시 실측 기반이라 타 공정/제조사 일반화 미검증, HW 구현(가속기) 설계 없음.
