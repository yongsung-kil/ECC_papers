### B. 알고리즘 요약

1. 대상은 protograph 기반 QC-LDPC 코드(base matrix `Bm×n`을 lifting factor `z=pq`로 2단계 lifting), 채널은 두 개의 서로 다른 `Eb/N0`를 갖는 non-uniform AWGN(sub-channel 우량 `Vg`/열등 `Vb`, 차이 `x=1,3,5dB`).
2. 기존 PEG/CC-PEG lifting 알고리즘은 균일 채널만 가정해 non-uniform 채널의 실제 신뢰도 편차를 반영하지 못해 decoding 성능(FER)이 저하됨.
3. GLDPC의 doped VN(신뢰도 높음)-undoped VN 개념을 non-uniform 채널의 우량/열등 sub-channel VN에 대응시켜 CC-PEG를 확장.
4. shift value 선택을 4단계 기준으로 순차 필터링: 열등VN만으로 구성된 최단 사이클 길이 `gb`, `gB` → concentrated CM `Mcc = Σ_{Vb}Mvs + α·Σ_{Vg}Mvs` (α=0.1) 최소화.
5. `Mcc`는 사이클 길이별 가중합 `Mvs,cr=Σβ^{l-2}N_{2l}^{s,r}` (β=0.01)로, 짧은 사이클이 열등 VN에 몰리지 않도록 억제.
6. greedy 완화를 위해 `Mcc` 최저 c개(`c=⌈δk⌉`, δ=0.1)를 후보로 남긴 뒤, PEXIT 분석 기반 APP-LLR MI가 낮은 VN에 가중치 `Wvs=γ^{h-1}`(γ=0.9)를 부여한 weighted ACE로 최종 shift 선택.
7. 부수 트릭: PMP(parallel vector message passing) 알고리즘으로 길이 `2g` 미만 사이클을 전수 탐색해 CM/ACE 계산.
8. 최적화 절차: δ, γ는 휴리스틱 실험으로 결정(δ=0.1, γ=0.9), c 평균값은 1.69~2.54로 계산복잡도 억제.
9. 결과: 기존 QC-PEG/QC-PMP/ACE-PEG/CC-PEG 대비 FER=10⁻⁴에서 0.1~0.5dB 이득, 코드당 lifting 시간 수 분.
10. 한계: N=480~840의 소형 protograph만 검증, HW 미설계, BP 디코딩(최대 400 iteration) 시뮬레이션만 수행, non-uniform 채널 위치가 고정이라는 가정.
