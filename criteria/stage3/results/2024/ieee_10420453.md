> 총평: layered LDPC 디코더의 파이프라인 conflict를 barrel shifter/RAM 우회로 해소하는 HW 스케줄링 기법으로, decoder HW 아키텍처 레이어에 이식 가능성이 높음.

### B. 알고리즘 요약

1. 대상은 DVB-S2 QC-LDPC, 길이 16200, code rate 11/15 및 2/3, Normalized Min-Sum(NMS, `α=0.875`) 기반 layered decoding, LLR 양자화 8/8/6bit(posterior/VN/CN).
2. 풀려는 문제는 pipelined layered 디코더에서 인접 layer의 BGM(base graph matrix) 비제로 원소가 겹칠 때 발생하는 read-after-write conflict로 인한 stall cycle과 성능 저하.
3. 채널 모델은 AWGN + BPSK(`y_i = x_i + w_i`), 시스템은 NMS check-node 갱신식 `R = α·sgn(ΠQ)·min|Q|` 사용.
4. 핵심 기법은 conflict 발생 시 LUG(posterior LLR update unit)의 출력을 barrel shifter 2개와 RAM을 거치지 않고 다음 layer의 VNG(variable node group)로 직접 전달하는 shortcut bypass 구조.
5. conflict-free read/write interval `ε = Dv + 2·Dbs + Dvng + Dlug + 2`를 shortcut 구조로 `Dv + Dvng + Dlug`로 단축 — barrel shifter 2개, RAM 접근 지연분을 제거.
6. 3가지 시나리오로 동적 스케줄링: (1) conflict 없음 → 통상 read/write, (2) conflict 발생 → shortcut bypass, (3) 불가피한 conflict → residual updating(식 `LLR = LLR_m - R_prev + R_new`)으로 patch.
7. offline BGM 재정렬을 TSP로 모델링해 GA(유전 알고리즘)로 conflict 발생을 최소화하는 layer 순서를 사전 탐색(첫 layer는 최대 row weight 배치로 stall 방지).
8. 학습/최적화 절차: GA 기반 오프라인 BGM 재정렬(온라인 학습 아님).
9. 결과: FER=10^-5에서 hybrid decoder 대비 0.25dB(10 iter) / 0.1dB(20 iter) 성능 이득, 20 iteration만에 수렴(hybrid는 25 iteration, 이상적 conflict-free 대비 3 iteration만 더 소요). Virtex-7 FPGA에서 LUT/FF/BRAM 각각 conventional 대비 10.2%/10.8%/7.5% 추가 소모.
10. 한계: DVB-S2 특정 rate(2/3, 11/15)·길이(16200) 실험만 수행, ASIC 합성 없이 FPGA 자원표만 제시, throughput은 정규화 값(Tnorm)만 제공되어 절대 Gbps 수치는 본문에 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered decoder pipeline conflict 해소 (shortcut bypass) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, `C2V_Cal()` | Prime ECC의 Dual-Update(홀짝 교대 스케줄)와 유사 목적(conflict/hazard 완화)의 스케줄링 기법으로 파이프라인 최적화에 참조 가능 |
| barrel shifter 우회, LLR 직접 전달 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | HW pipeline 6-stage 구조와 정합 필요하나 병목 지점(shift/RAM 접근) 최적화 아이디어로 참고 가능 |
| BGM 재정렬(GA 기반 TSP 최적화) | `ecc_top.cpp` `Load_PCM()` | 기존 QC-LDPC H-matrix 재배치이며 Prime ECC는 특정 H-matrix 고정이라 재검증 부담 큼 |
| residual updating (conflict patch) | `decoder.cpp` `LDPC_Decoder()` | 조기종료·이터레이션 로직과는 별개의 conflict patch 기법, 이식 시 파이프라인 재설계 필요 |

> 적용 가치: 중간 — layered 디코더의 pipeline conflict 완화라는 HW arch 레이어 기법으로 Prime ECC의 pipeline/Dual-Update 스케줄과 목적이 겹치나, 통합하려면 기존 GT/HCU 구조와의 정합 검증이 필요.
