### B. 알고리즘 요약

1. 대상 코드: CCSDS 표준 (128,64) block length, code rate 1/2 LDPC, 위성(deep space/telecommand) 통신용.
2. 문제: 장거리·저전력 우주통신 채널에서 신뢰성 있는 오류정정이 필요, 기존 turbo/BCH 대비 LDPC 채택.
3. 채널 모델: 수신 신호를 16bit(부호1, 정수2, 소수13) soft LLR로 양자화, AWGN 가정.
4. 핵심 기법: Normalized Min-Sum 알고리즘. 정규화 계수 `0.5`를 곱해 메시지 크기를 조절.
5. 핵심 식: extrinsic LLR을 min-sum 근사로 계산(`f(x)=|log(tanh(x/2))|`의 min 근사), 정규화로 LLR 포화 방지.
6. 구조: Check-Node(row, SPC 디코딩)와 Bit-Node(column, repetition 디코딩) 연산을 FSM 6-state(RESET/INITIALISE/ROW/COLUMN/ITERATION/OUTPUT)로 제어.
7. 구현 디테일: control path(FSM)와 data path(연산 유닛) 분리한 모듈러 설계, matrix multiplication으로 초기 belief 계산.
8. 최적화 절차 없음(고정 정규화 계수 0.5, MATLAB 시뮬로 사전 튜닝만 수행).
9. 결과: 9% 노이즈 주입 시 1회 반복 66.7%→10회 97.7%→15회 100% correction efficiency. 전체(128,64) 디코더 latency 715 clock cycle(71500ns @10MHz).
10. 한계: FPGA 실합성/검증은 4x7 축소 parity-check 예제(Check-Node 모듈만)에 한정, 병렬화·메모리 최적화·throughput 수치 없음, 향후 layered decoding 등은 future work로만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Normalized Min-Sum 디코딩 알고리즘 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 min-sum 계열 기법과 동일 수준, 신규성 없음 |
| Check-Node(SPC) 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 표준 min-sum CN 연산과 동일, 대응됨 |
| Bit-Node(repetition) 연산, LLR 양자화(16bit) | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | soft-4bit+급 양자화이나 NAND read 비용 관점 이식 실익 낮음 |
| 정규화 계수(0.5) 적용 | `ecc_data.h` `PARAM_LLR` | 고정 스칼라 정규화로 이미 보유한 적응형 LLR 테이블보다 단순 |
| FSM 기반 제어(RESET~OUTPUT) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | HW 스케줄링 개념은 유사하나 코드 구조상 이식 불필요 |
| 부호구조(64x128, CCSDS 표준) | `ecc_top.cpp` `Load_PCM()` | CCSDS 위성용 고정 H-matrix, NAND QC-LDPC와 무관 |

> 적용 가치: 낮음 — 표준 normalized min-sum의 교과서적 재구현으로 Prime ECC가 이미 보유한 기법을 넘어서는 신규 알고리즘·HW 최적화가 없고, 대상도 위성통신(비-NAND)임.
