### B. 알고리즘 요약

1. 시스템: 3D TLC NAND flash, LSB/CSB/MSB 3종 페이지, QC-LDPC `(18656, 16748)`(rate 0.9), layer min-sum 디코더, 최대 iteration 30, 6-bit quantizer.
2. 문제: 층간간섭(inter-layer interference)으로 3D TLC의 RBER이 층마다 크게 달라지는데 고정 rate(0.9) LDPC를 쓰면 고RBER 층에서 read-retry(soft read 전압 재인가)가 반복돼 지연이 급증.
3. 핵심 가정: RBER은 P/E cycle, retention time, inter-layer interference 3요소로 결정되며, 데이터 저장 전 미리 RBER 구간(6구간)을 code rate(0.9/0.85/0.8/0.75/0.7)의 교정한계 `L`과 매핑해둘 수 있다고 가정.
4. 핵심 기법: `shorten` 기법으로 디코더 입력 LLR 일부(`Sx`)를 0으로 고정해 정보비트 길이를 줄임 — rate = `(Sk − Sx)/(Sn − Sx)`. 단일 H-matrix(rate 0.9용)만 저장하고도 다중 rate를 지원.
5. 핵심 식/의미: 사전 측정한 RBER 임계값 `L[i]`(예: rate0.9→L=5.61×10⁻³, rate0.7→L=2.68×10⁻²)와 실측 RBER을 비교해 RBER<L[i]인 최고 rate `C[i]`를 선택(Algorithm 1) — rate가 높을수록 redundancy가 적으므로 가능한 한 높은 rate를 우선 선택.
6. 확장: 선택된 rate로 통상 layer min-sum 반복복호를 수행, `y·Hᵀ=0` 또는 최대 iteration 도달 시 종료.
7. 부수 트릭: shorten은 매트릭스를 다시 만들지 않고 정보비트 일부를 마스킹하는 방식이라 추가 저장 오버헤드가 사실상 없음(단일 encoder/decoder로 전 rate 처리).
8. 최적화 절차: 20000 프레임으로 각 code rate의 correctable RBER 상한(L)을 FER=10⁻⁴ 기준으로 사전 측정(오프라인 캘리브레이션, 실시간 학습 아님).
9. 결과: RBER<1.56×10⁻²에서는 read-retry 자체 불필요, 전체 iteration 수 67% 감소, RBER<2.51×10⁻²에서 read-retry 횟수 9회→2회로 감소, 공간 오버헤드는 1.15%에 불과.
10. 한계: 알고리즘/시뮬레이션 수준 검증(FPGA는 RBER 특성 측정용, LDPC 디코더 자체의 HW 구현·gatecount·throughput은 미제시)이며 shorten 위치·구현 세부는 본문에 상세 기술 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RBER 구간별 code rate 선택 정책(Algorithm 1) | `decoder.cpp` `LDPC_Decoder()` 진입 전 rate 결정 로직, `ecc_top.cpp` `Load_PCM()` | 단일 H-matrix에서 rate 스위칭하는 정책은 Prime ECC의 `matrix_sel` 기반 PCM 로드와 연계 가능 |
| shorten 기법(단일 H-matrix로 다중 rate 지원) | `encoder.cpp` `Generate_PCM_encoding()`, `decoder.cpp` VN 처리 | 정보비트 LLR 일부를 0-고정하는 것은 VN 초기화 단계에 삽입 가능 — 인코더/디코더 모두 영향 |
| layer min-sum 디코더 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC도 min-sum 기반이라 층별 rate만 바꿔 그대로 적용 가능 |
| read-retry 횟수(soft-read 전압 수) 감소 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read 임계값 재인가 정책과 직접 연동되는 지점 |

적용 가치: `높음` — NAND 직접 타깃, QC-LDPC 기반, 단일 H-matrix 재사용으로 오버헤드 거의 없이 read-retry/iteration을 크게 줄이는 실용적 정책 기법으로 Prime ECC의 채널·PCM 로드 로직에 바로 참조 가능.
