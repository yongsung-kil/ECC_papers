### B. 알고리즘 요약

1. 대상 부호는 CCSDS 131.0-B-5(Blue Book) 표준의 QC-LDPC로, `k={1024,4096,16384}`, `R={1/2,2/3,4/5}` 조합 총 9종(기존 정의 파라미터 기준)이며 base H-matrix는 `mb x nb` 블록 구조(예: R=8/11에서 `nb=11, mb=3`)로 lifting size `M`(128~8192)만큼 확장된다.
2. 문제의식은 CCSDS가 제공하는 code rate/length 조합(9~18종)이 Adaptive Coding and Modulation(ACM)에 쓰기엔 세분화가 부족하다는 것(Wi-Fi6는 12종, 5G는 51종 조합 제공).
3. 채널모델은 AWGN(위성통신 표준 관례), soft-decision LLR 기반 SISO 디코더(10-bit fixed-point 구현)를 전제로 한다.
4. 핵심 기법은 블록(`M`) 단위 shortening: 데이터워드 앞에 zero-fill `s`개 블록을 채워 인코딩 후 전송에서 제거하고, 수신측 SISO 디코더에는 해당 위치에 high-confidence-zero LLR(예: 10bit 구현에서 1023)을 삽입한다. rate는 `Rs=(k-s)/(n-s)`로 감소.
5. puncturing은 인코더 출력 중 parity 또는 data 비트를 `p`개 블록 단위로 단순 제거하고, 디코더에는 zero-confidence(=0) LLR을 삽입; rate는 parity puncture 시 `Rp=k/(n-p)`로 증가.
6. 확장 기법으로 두 변환을 조합해 원래 18개(정의상 9개 base rate x 미사용 rate 포함) 파라미터에서 31개를 추가 도출, 총 49개 조합으로 세분화된 ACM용 코드 세트를 구성.
7. 구현 디테일로 bitmap/packed-bit 저장을 이용해 메모리를 절감(예: k=4096, R=2/3 base 구성 B에서 인코더 49212B + 디코더 176128B = 총 225KiB로 5개 파라미터 세트를 동시 지원).
8. 별도의 학습/최적화 절차는 없음(표준 규격 기반 변환 룰만 적용).
9. 결과: R=8/11(k=4096, M=512) shortening 시 rate가 낮아질수록(s 증가) waterfall 성능이 개선되는 것을 확인했고, 처리량 비교에서 자체 구현(YALDPC, 16-thread MEX)이 114.72Mbps(1-thread 대비 개선)로 AFF3CT(31.69Mbps, 1-thread) 대비 우수하나 비교기준이 스레드수 등에서 혼재됨.
10. 한계: HW(ASIC/FPGA) 미설계, x86 CPU 소프트웨어 구현/MATLAB 시뮬에 국한되며, 짧은 논문(short paper) 특성상 puncturing 최적 컬럼 선택 등은 스코프 밖으로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| block 단위 shortening (zero-fill + high-confidence-zero LLR) | `channel.cpp` `Set_LLR_Th()` | LLR threshold/양자화 테이블에 high-confidence-zero 값을 별도 정의하면 이식 가능한 개념적 대응 |
| puncturing (zero-confidence LLR 삽입) | `channel.cpp` `Set_R_Offset()` | punctured 비트에 대한 LLR 초기화 로직과 개념적으로 유사 |
| H-matrix 블록(base matrix) 구조/lifting | `ecc_top.cpp` `Load_PCM()` | CCSDS 표준 특정 QC-LDPC 구조이며 Prime ECC 고정 QC-LDPC와 다름, 부호설계 자체 이식은 난이도 높음 |
| 메모리 절감(bitmap/packed bits) | `ecc_data.h` (PCM 등 구조체) | 소프트웨어 구현 최적화로 HW 시뮬레이터와 직접 대응 약함 |

> 적용 가치: 중간. shortening/puncturing을 통한 rate/length 유연화 아이디어는 일반 QC-LDPC 디코더의 LLR 초기화 로직에 개념적으로 이식 가능하나, HW 미설계·위성통신 AWGN 채널 전제·소프트웨어 처리량 비교 위주라 Prime ECC의 HW/NAND-read 비용 관점과는 거리가 있음.
