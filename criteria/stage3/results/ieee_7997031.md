### B. 알고리즘 요약

1. 시스템: 다채널 병렬 NAND 고성능 SSD, 각 채널은 inner LDPC로 페이지 보호, 채널마다 read/write 속도편차로 자체 큐 보유.
2. 문제: LDPC soft-decision은 강력하나 soft read를 위한 다중 sensing이 read latency를 크게 키움 → latency/overhead 트레이드오프를 실패까지 포함해 정량화.
3. 모델: 채널을 분산 스토리지 노드로 보고 `(n,k)` fork-join 큐잉, Poisson 도착률 `λ`, read failure(디코딩 실패→resensing)와 node failure(page/block/die 고장) 두 실패 모델링.
4. 계층부호: inner soft-decision LDPC 위에 노드간 outer MDS(RS) code를 추가, inner 디코딩 실패 시 outer RS로 복구해 재sensing 반복(latency) 회피.
5. 핵심식: progressive sensing latency `τi = τsen-ref + τhard + τdec + (i-1)(τsen+τsoft+τdec)` (식1), sensing level 증가마다 지연 누적.
6. read time을 순서통계량 `Xk:n`(가장 빠른 k개)로 표현, split-merge M/G/1 근사로 평균 접근시간 상한 유도.
7. node failure는 preemptive resume priority(class1=repair, class2=read)로 처리, 단일노드 고장 복구를 n-1 중 k copy로 모델(식9,13,16).
8. 학습/최적화: 없음(해석적 큐잉 상한 유도 중심).
9. 결과: inner 1-KB regular LDPC(rate 0.8947, PEG설계) + outer RS(k=16, n가변), RBER<0.006에서 outer RS 추가 시 평균 read access time 최대 ~70% 감소(node failure 포함 시 ~50~65%).
10. 한계: HW 미설계, 시뮬/해석 상한만, inner LDPC 디코더·부호 자체는 기존 것 사용(기여는 outer 계층+큐잉분석), 특정 25nm MLC 지연 파라미터 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| inner LDPC(1KB, rate 0.8947) 복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC가 이미 보유한 고rate binary LDPC 디코더, 논문은 이를 블랙박스로 재사용만 |
| progressive soft sensing / read latency | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read 비용은 우리 채널모델과 개념 유사하나 논문은 큐잉지연 파라미터일 뿐 알고리즘 미제공 |
| 채널간 outer MDS(RS) 계층부호 | 대응 없음 (Prime ECC는 단일 inner LDPC 엔진) | 컨트롤러/RAID 계층 사안, 코드베이스 범위 밖 |
| fork-join 큐잉 latency 분석 | 대응 없음 | 시스템레벨 성능해석, ECC 엔진과 무관 |

적용 가치: 낮음 — NAND read latency 관점은 유익하나 기여가 채널간 outer RS 계층부호와 큐잉 기반 read-access-time 해석(컨트롤러/시스템 레벨)이라, Prime ECC의 단일 inner LDPC 디코더/인코더에 이식할 알고리즘 요소는 없다.
