### B. 알고리즘 요약

1. Rate-compatible punctured LDPC 코드(단일 인코더/디코더로 여러 rate 지원)인 E2RC(efficiently-encodable rate-compatible) 코드의 체계적 설계·분석을 다룸. H=[H1|H2], H2는 결정적 lower-triangular 구조로 선형시간 인코딩 지원.
2. 문제의식: 기존 E2RC 구성은 H1(정보부)의 degree distribution 설계 기준이 없고, 특정 rate에서만 최적화되어 puncturing rate 전역 성능 보장이 안 되며, mother code rate에서 높은 error floor를 가짐.
3. 핵심 가정/모델: H2를 protograph 구조로 모델링, 노드를 k-step recoverable(k-SR, 반복복호 k회 만에 복구 가능)로 분류. BIAWGN 채널.
4. 핵심 기법 1단: EXIT chart 기반 설계 - H1(비구조부)과 H2(구조부)를 두 code component로 분리, `I_E,unS(I_A,unS,σ²_mch)=Σλ_dv·J(√((dv-1)[J⁻¹(I_A,unS)]²+σ²_mch))` (식 3)로 H1의 degree distribution `{λ_dv}`를 선형계획법(LP)으로 최적화.
5. 핵심 식 의미: multi-edge-type 기반으로 구조부 H2에 2|ER|+|EL|개 연립방정식(식 4~6)을 세워 EXIT 함수를 Monte-Carlo 없이 반복(iterative) 계산 - 기존 대비 수천 배 빠름(Table I: 3.7초 vs 24596초).
6. 핵심 기법 2단: protograph E2RC - check-splitting(degree-d 체크를 c1,c2로 분할하고 degree-2 변수 노드로 연결, Fig.7)을 반복 적용해 rate-compatible protograph 계열을 생성, c1이 새 노드 연결을 모두 상속하는 규칙으로 E2RC의 k-SR 구조를 보존(Appendix 증명).
7. 부수 트릭: H2를 짧은 protograph로 제한해 최대 check-degree를 낮춤으로써 원본 E2RC의 mother-code-rate error floor 문제를 해결.
8. 최적화 절차: 여러 puncturing rate에 대해 동시에 capacity gap `g`를 최소화하는 joint optimization(LP, "zero-error constraint"로 EXIT curve tunnel 보장) 알고리즘 제시.
9. 결과: semi-structured E2RC(code 2, block 16384bit)는 rate 0.5~0.889 전역에서 capacity gap 0.25~0.30dB로 균일, protograph E2RC(dv_max=20)는 gap 0.235~0.278dB, AR4JA 대비 공통 rate(1/2,2/3,4/5)에서 0.12~0.17dB 개선. 원본 E2RC의 error floor 문제 해소.
10. 한계: 순수 알고리즘/시뮬 레벨 코드설계 연구로 HW 구현·gatecount·throughput 미제시, rate-compatible puncturing이라는 통신 특화 목적(단일 mother code로 다중 rate) 때문에 고정 rate NAND ECC와 설계 목표가 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EXIT chart 기반 degree distribution 최적화 | 대응 없음 | Prime ECC는 고정 H-matrix 시뮬레이터로 코드설계(degree distribution 최적화) 단계 자체가 없음 |
| Protograph check-splitting을 통한 rate-compatible 코드 계열 생성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 지점과 개념적으로 연결되나, Prime ECC는 단일 고정 rate이라 rate-compatible 구조 자체가 대응 없음 |
| E2RC lower-triangular H2 기반 선형시간 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC의 dual-diagonal 인코딩과 유사한 계열의 결정적 인코딩 구조 - 개념적 참고 가능하나 직접 이식은 아님 |
| EXIT chart 기반 error floor 회피(짧은 protograph로 max check-degree 제한) | 대응 없음 | error-floor 대책이지만 코드설계(H-matrix) 단계 기법이라 디코더 실행 로직과 무관 |

> 적용 가치: 낮음 — rate-compatible puncturing이라는 통신 시스템 특화 목적의 코드설계 연구로, Prime ECC의 고정 rate QC-LDPC 구조나 디코더/HW 로직에 이식할 요소가 거의 없음.
