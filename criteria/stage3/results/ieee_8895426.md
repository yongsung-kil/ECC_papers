### B. 알고리즘 요약

1. 시스템: 우주 telecommand(TC) 업링크, CCSDS 표준 binary LDPC 2종 `LDPC(128,64)`·`LDPC(512,256)`, rate `Rc=1/2`, systematic, `Q×Q` circulant(Q=16/64) QC 구조.
2. 문제: 새 LDPC로 동작 SNR이 낮아져(Es/N0≈2dB) 획득·추적·프레임동기가 병목, O/B 디코더 복잡도·latency 제약.
3. 모델: residual carrier PCM(SP-L/NRZ-L), AWGN, 목표 CER `10^-5`.
4. 핵심기법1: 세 디코더군 비교 — iterative(SPA/MS/NMS), 비반복 MRB(ordered statistics), hybrid(NMS 먼저, 실패 시에만 MRB 호출).
5. MRB: k개 최신뢰비트로 `G*` 재구성(Gauss-Jordan)→후보 CW 인코딩→차수 i의 TEP(Hamming weight≤i) 전수→수신벡터와 유클리드 최소거리 CW 선택(i=k면 ML 등가).
6. 확장: hybrid는 NMS(저복잡, `Imax=50`)로 대부분 처리, detected error 시 MRB로 ML급 성능 회수 → LDPC(128,64) hybrid CER 10^-5 @ Es/N0≈0.7dB.
7. 구현 디테일: NMS 부분병렬 `v=(n-k)/Q=4` CPU, `t=n/Q=8` VPU, `v·t=32` MB. MRB Part1(GJ)은 SW, Part2(TEP평가 TEU 뱅크)는 HW 병렬. 양자화 3bit/6bit 평가.
8. 최적화: TEU 뱅크 공유 TEP generator + 사전인코딩으로 병렬 TEU 중복 제거, distance calculator를 c분할 부분병렬.
9. 결과: SoC(SW+DSP) breadboard 실측, NMS는 SW대비 ~0.1dB 손실, hybrid 최대 0.15dB차. Space FPGA(RTAX4000S) 자원 여유. 6bit 대비 3bit 양자화 ~0.2dB 손실.
10. 한계: MRB 400K TEP은 ~60s로 latency 요구(2s) 초과 → HW 필수, 심볼레이트 sps~수ksps급으로 NAND ECC 스케일과 상이.
