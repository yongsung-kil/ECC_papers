### B. 알고리즘 요약

1. 통신 채널(AWGN, QPSK) 대상 QC-LDPC 부호 구성법. 검증 코드 길이 N=190(K=76, rate≈0.4)과 N=2496(K=1872, rate≈0.75), (m,n) exponent matrix 크기는 예제별로 (5,6)~(6,24) 등 다양.
2. 풀려는 문제: 주어진 code length/rate에서 girth-8을 보장하면서 CPM size(순환치환행렬 크기 Z)를 최소화하는 exponent matrix를 낮은 탐색복잡도로 찾는 것 - 기존 GCD 기반 방법보다 작은 Z, 낮은 error floor를 목표.
3. 핵심 가정/모델: exponent matrix를 product-like 형태 `p_i,j = a_i b_j mod Z`로 제한, cycle 존재 조건 `Σ(p_iz,jz - p_iz+1,jz) ≡ 0 (mod Z)`을 2차원 좌표상의 직사각형 넓이(rectangular area, RA)로 재해석.
4. 핵심 기법: cycle-4/cycle-6 조건을 RA의 양의 넓이 `S+`와 음의 넓이 `S-`로 분해(`S_x,y = (a_{ix+1}-a_ix)(b_{jy+1}-b_jy)`), `S+ = S-`가 곧 짧은 cycle 존재 조건임을 증명(Theorem 1, 2).
5. 핵심 식 의미: RA 표현으로 cycle-4/6 판별을 정수 넓이 비교 문제로 단순화해 CPM size 하한(`Z ≥ (a_{m-1}-a_0)(b_{n-1}-b_0)+1` 등)을 직접 유도 가능하게 함.
6. 확장: search depth `L`을 도입한 semi-search 알고리즘(Alg.1, RA/DEPTH 함수)으로 탐색복잡도와 CPM size 최적화 사이 균형 조절, `L=0`이면 비탐색(closed-form), `L`이 클수록 CPM size는 작아지되 복잡도 증가.
7. 부수 트릭: row/column 교환(a, b 역할 교체), masking matrix(전연결 대신 일부 CPM을 zero-matrix로 대체)와의 결합으로 탐색복잡도·CPM size 동시 축소, enumeration으로 이론적 하한보다 더 작은 실제 최소 Z 탐색.
8. 최적화 절차: Alg.1의 이중 루프(외부 RA 탐색 + 내부 DEPTH 재귀)로 `a_{m-1}` 최소화, `g(n,L)`, `h(m,L)` 등 조합 함수로 탐색량을 사전 추정.
9. 결과: masking 결합 시 기존 GCD 계열([9],[10],[17],[18],[20]) 대비 CPM size 대폭 축소(예: M1 기준 Z4=19 vs 기존 150~217), N=190/2496 두 코드에서 SPA(max iter 50) 디코딩 시 GCD 방법 대비 더 낮은 BER/BLER error floor 확인.
10. 한계: 순수 대수적 구성·시뮬레이션(SPA, AWGN/QPSK)만 수행하고 HW 구현·min-sum·양자화·NAND 채널 전혀 다루지 않으며, girth-8 이상(girth-10/12) 확장이나 고rate(~0.9) 영역 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-8 exponent matrix(H-matrix) 구성법 | `ecc_top.cpp` `Load_PCM()` | 신규 H-matrix 설계법이나 Prime ECC는 특정 QC-LDPC 고정, 이식 시 전면 재검증 필요 |
| masking matrix 기반 CPM size 축소 | `ecc_top.cpp` `Load_PCM()` | base matrix 설계 단계에만 관련, 디코더/인코더 로직과 무관 |
| SPA 디코딩 성능 검증 | 대응 없음 | Prime ECC는 min-sum 기반이며 SPA(full-tanh Sum-Product)는 미보유 기법으로 직접 매핑 안 됨 |

> 적용 가치: **낮음** — H-matrix/exponent matrix 자체를 재설계하는 순수 부호 구성법으로 Prime ECC의 고정된 QC-LDPC 구조에 이식 시 재검증 부담이 가장 큰 영역이며, NAND 타깃도 아니고 디코더/HW 개선과 무관.
