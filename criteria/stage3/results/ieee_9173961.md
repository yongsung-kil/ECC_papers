### B. 알고리즘 요약

1. (3,6)-regular SC-LDPC 코드(협화합 폭 `m=2`, lifting `M`, 결합 체인 길이 `L`)를 base matrix `B=[3 3]`로부터 edge-spreading으로 구성, sliding window decoding(SWD)으로 복호.
2. 문제: SWD는 window 크기 `W`가 작을수록 latency는 줄지만, 한 블록 오류가 다음 블록 복호에 악영향을 미치는 오류 전파(error propagation)가 드물게 발생해 BLER을 심각하게 악화시킴.
3. 채널/모델: 디코더 상태를 랜덤오류상태 `Sre`(오류확률 `p`)와 오류전파상태 `Sep`(오류확률 1)로 나눈 2-state 마르코프 모델로 근사.
4. 핵심 기법: 결합 체인 중간에 reduced-degree check node(`CN doping`)를 규칙적으로 삽입해 긴 프레임을 짧은 sub-frame으로 분할, 오류전파가 sub-frame 경계에서 truncate되도록 유도.
5. 핵심 식: 평균 BLER `PBL = Σ PBL(τ)·q(1-q)^(τ-1) + p·(1-q)^Λ` — doping 없는 프레임 길이 `Λ`가 커질수록 BLER이 나빠짐을 보임(식 4), doping 적용 시 sub-frame별 `λj`로 분할해 재계산(식 7-8).
6. rate loss: doping된 CN 수 `γ`에 대해 `R_Λ,doped = 1 - (Λ+m)/Λ - γ(1-R)/(Λ·ηv)`로 정량화, full termination 대비 rate loss가 더 작음을 증명.
7. 부수 디테일: doping point에서는 window shift가 VN 기준 1 time unit, CN 기준 2 time unit으로 비대칭 이동(그림 4/5).
8. 최적화 없음 — doping point 개수/균등 배치는 해석적 모델(식 8)로 사전 예측, Monte-Carlo로 검증.
9. 결과: doping 1개소로 `PBL` 5.79e-2 → 3.44e-2, 3개소로 2.23e-2까지 개선(예제 3); 실제 시뮬(L=500, W=18)에서는 doped 코드가 undoped 대비 BER 최대 2자릿수, BLER 1자릿수 이상 개선.
10. 한계: SC-LDPC/스트리밍 특화(긴 프레임·연속 전송) 문제이며, HW 구현이나 정량 latency/throughput 분석 없이 순수 이론+시뮬 결과만 제시. 부호rate 값(0.498/0.497)은 예제 케이스에 한정된 특정값.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호 구성(edge spreading, base matrix) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(block code) 고정 구조이며 SC-LDPC convolutional 결합 구조는 대응 없음 |
| Sliding window decoding 스케줄 | `decoder.cpp` `LDPC_Decoder()` | window 기반 복호는 우리 layered/dual-update 스케줄과 근본적으로 다른 개념, 대응 없음 |
| Check node doping (오류 전파 차단) | 대응 없음 | 우리는 단일 block QC-LDPC로 프레임 간 연쇄 오류전파 개념 자체가 없음 |
| BLER 2-state 마르코프 분석 모델 | 대응 없음 | 순수 SC-LDPC용 통계 모델, block code 시뮬레이터(ecc_top.cpp 통계)와 직접 대응 없음 |

적용 가치: `낮음` — SC-LDPC 스트리밍/긴 프레임 sliding-window 오류전파 방지 기법으로, Prime ECC의 block 기반 QC-LDPC 구조와 코드 클래스 자체가 달라 이식 경로가 없음.
