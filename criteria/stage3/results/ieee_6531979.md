### B. 알고리즘 요약

1. 대상: 1X/2X/3Xnm MLC NAND SSD의 시스템 신뢰성, ECC로 EP-LDPC(error-prediction LDPC)와 ER(error-recovery) 검토, code rate `9/10`·`2/3`, codeword 1KByte.
2. 문제: scaling으로 cell-to-cell interference·HCI 열화 심화, 기존 BCH(20bit/1KB)는 한계. 정확 `VTH`를 위한 conventional LDPC는 read reference 21개로 read access time이 과대.
3. 핵심 가정: cell 오류를 program-disturb(PD, `VTH`↑)와 data-retention(DR, `VTH`↓) 두 모드로 분류, 사용이력(`N_W/E`, retention time, 온도)으로 BER가 결정된다고 모델링.
4. 핵심 기법 1: EP-LDPC — 사전기록 reliability table로 각 셀 BER를 추정. `Vref` 3개만으로 conventional LDPC 대비 7배 빠른 read, 추정 BER로 LDPC 정정 수행.
5. 핵심 식: overall BER `BER_Lower = |N"1"measured - N"1"initial| / N_P` (초기·read 데이터의 "1" 개수 차이로 페이지 BER 추정).
6. 핵심 기법 2: ER — `N"1"measured` vs `N"1"initial` 비교로 PD/DR 지배모드 판별, PDRP(program-inhibit)·DRRP(read voltage)로 전자 방출/주입해 물리적 오류 복구.
7. 구현 디테일: retention 중 `tCrossPoint`(PD=DR 시점)를 기준으로 PDRP/DRRP 선택, `N_W/E`·온도별 tCrossPoint 특성 측정.
8. 최적화 절차: 별도 학습 없음, 3세대 NAND 실측 데이터를 reliability table로 축적.
9. 결과: 1Xnm에 CR=9/10 EP-LDPC 적용 시 3Xnm+20bit BCH와 동등 신뢰성, CR=2/3 적용 시 2Xnm+44bit BCH를 크게 상회(block lifetime).
10. 한계: LDPC 부호·디코더 자체는 미설계, table 기반 BER 예측·물리적 error-recovery에 집중, HW 아키텍처 없음.
