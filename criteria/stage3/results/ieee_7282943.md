### B. 알고리즘 요약

1. Protograph 기반 Generalized LDPC(GLDPC) 코드(단일 parity-check 대신 임의의 선형 블록 부호를 constraint node로 사용)를 binary erasure channel(BEC)에서 분석.
2. 기존 LDPC finite-length 분석([7])은 일반 LDPC(SPC 제약)에 한정, GLDPC로의 확장이 미해결이었음.
3. 채널/모델: BEC, 각 constraint node는 연관된 component code(예: (7,4) Hamming)가 복호 가능한 erasure 패턴 목록으로 특성화.
4. 핵심 기법: peeling decoder(PD)를 일반화한 generalized peeling decoder(GPD) 정의 - decodable constraint node를 무작위로 제거하며 반복.
5. 핵심 식: 정규화된 degree distribution `ld(τ), rc(τ)`의 기대 진화를 나타내는 미분방정식 `∂l̂d/∂τ`, `∂r̂c/∂τ` (식 3,4) - 그래프가 반복에 따라 어떻게 변하는지 결정론적으로 근사.
6. 확장: 미분방정식 해로부터 GPD threshold `ε*`(임계 erasure율) 산출, 임계점에서의 decodable fraction `â(τ*)`와 분산으로 유한길이 블록 오류율을 scaling law `P_Block ~ Q(α√n(ε*-ε))`로 추정 (식 8).
7. 부수 디테일: direct/indirect constraint node 제거 확률 계산(식 5,6), edge type별 결합확률 사용.
8. 최적화/검증 없음(HW/학습 아님) - (2,7)-regular protograph에 (7,4) Hamming ML/제한거리 디코딩, SPC 혼합 등 3가지 사례로 예시 분석.
9. 결과: ML 디코딩 시 ε*≈0.7025, α=1.8024; 제한거리 디코딩 시 ε*≈0.5135, α=1.6719; Hamming+SPC 혼합(rate 0.4285) 시 ε*≈0.3625, α=1.5102 - 강한 component code일수록 threshold·scaling parameter 개선.
10. 한계: BEC 전용(AWGN/NAND read 채널 미다룸), HW 미설계, Monte Carlo로 분산 추정(완전 해석적 아님), spatially-coupled GLDPC 확장은 future work로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC 부호구조(protograph + component code) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 순수 binary LDPC(SPC 제약)로 GLDPC 구조 자체가 없어 관련성 약함 |
| Generalized Peeling Decoder(BEC 전용) | `decoder.cpp` `LDPC_Decoder()` | BEC 전용 알고리즘이라 min-sum 기반 실제 디코더와 알고리즘적으로 무관 |
| Finite-length scaling law(부호 최적화 방법론) | `ecc_top.cpp` `Load_PCM()` | 부호 최적화 시 참고 가능한 이론이나 채널모델 불일치로 직접 적용 불가 |
| (대응 없음) | - | non-binary/GLDPC 이론 전반이 Prime ECC의 binary QC-LDPC 범위 밖 |

> 적용 가치: 낮음 | BEC 전용 이론 분석이며 GLDPC(constraint node가 일반 블록부호) 구조 자체가 Prime ECC의 binary SPC 기반 QC-LDPC와 다르고, 디코딩도 min-sum이 아닌 ML/peeling 기반이라 직접 이식 대상이 없음.
