### B. 알고리즘 요약

1. 대상은 binary protograph 기반 LDPC 앙상블 `Cn(P)`(protograph `P`를 lifting factor `Q`로 복사·permute하여 `H` 생성).
2. 문제: 특정 부호의 trapping/absorbing/fully-absorbing set 완전열거는 중간 블록길이만 돼도 비현실적이라 앙상블 평균 enumerator로 우회한다.
3. 정의: `(a,b)` FAS는 각 VN이 unsatisfied CN보다 satisfied CN에 더 많이 연결된 trapping set, EFAS는 추가로 각 satisfied CN이 정확히 2개·각 unsatisfied CN이 1개 VN에 연결.
4. 핵심 기법(유한길이): edge weight 벡터 `(e,w,l)`를 도입, CN측 생성함수 `Ai`(식4)와 VN측 생성함수 `Bj`(식5)를 정의해 평균 FAS 수 `E[FAS(a,b)]`(Lemma 1, 식1)를 조합적으로 산출.
5. 의미: 각 VN이 "unsatisfied<satisfied CN" 제약을 만족해야 해 기존 TS enumerator 방식(flag VN)이 곧바로 안 되던 것을 생성함수로 해결.
6. 핵심 기법(점근): Hayman 공식을 `Ai`, `Bj`에 적용해 정규화 로그 점근분포 `GFAS(θ,γ)`(Theorem 1, 식7)를 도출, `a=θn, b=γn`.
7. 복잡도: 점근분포는 `6e+2` 방정식/미지수 계를 풀어야 하며, protograph의 parallel edge 대칭성(Corollary 1)으로 차원 축소(예제서 44→14).
8. EFAS는 `Ai` 생성함수만 교체(Remark 1)하면 동일 절차로 산출.
9. 결과: 위성 telecommand용 `B1=[3 4]`에서 FAS/EFAS 점근분포와 relative minimum FAS size 산출; `B2`(3×6 all-ones), `n=900`에서 Gallager-B 및 4-bit quantized min-sum(biAWGN) 100개 랜덤부호 FER이 dominant (2,2)·(4,2) FAS 기반 예측 error floor와 부합(7개는 저중량 codeword로 이탈).
10. 한계: 앙상블 평균 이론뿐(특정 부호·디코더·HW 산출물 없음), error floor 예측/부호선별 도구 성격.
