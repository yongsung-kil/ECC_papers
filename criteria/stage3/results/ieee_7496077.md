### B. 알고리즘 요약

1. 대상은 `(155,64)` Tanner pseudo-quasi-cyclic LDPC 코드, AWGN 채널(1.8 dB)에서 LP 디코딩 성능/속도 평가.
2. LP 디코더는 fractional 해가 나오면 ML 실패로 간주 → ALP는 이 fractional 해를 배제하는 redundant parity-check(유효 컷)를 반복 추가해 ML에 근접시킴.
3. 문제: 유효 컷을 만드는 사이클(cycle) 탐색이 어렵고, 기존 ILP 기반(IALP) 탐색은 시간이 오래 걸림.
4. 핵심 기법: degree `d(c)` 체크노드를 보조변수 `a1..a4`로 분해해 degree-3 노드 `(d(c)-2)`개로 재정의, 각 노드는 `2^2=4`개 컷만 필요.
5. 효과(Önerme 1): 표준 LP는 컷 수가 `2^(d(c)-1)`로 지수 증가하나, 보조변수 표현은 `4·(d(c)-2)`로 선형화 → local polytope 축소.
6. fractional 노드를 0F/2F/3F 세 클래스로 분류(Önerme 2), 각 클래스의 변수합 규칙(합=1, 상등 등)으로 유효 컷 조건을 판별.
7. 사이클 내 3F 노드 수가 적을수록 유효 부등식 생성이 쉬움을 증명 → 3F 노드 최소 사이클을 휴리스틱 탐색.
8. HALP 알고리즘: SDP/LP 모델 풀이 → 정수해면 종료 → 아니면 `y`개 3F 포함 사이클 탐색, `y` 증가, 유효 컷 추가 후 재풀이, 시간한계까지 반복.
9. 결과: HALP는 초기 iteration에서 목적함수 값을 IALP/ACG-ALP보다 빠르게 상승시킴(그림 2), 첫 유효 컷을 가장 빠르게 발견.
10. 한계: 일정 시점 이후 컷 발견이 정체돼 최종 BER은 ACG-ALP보다 나쁨 → 단독 디코더 부적합, 타 디코더와 병용 지향. HW 미설계, 소형 코드 1종 시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP/ALP 최적화 기반 디코딩(HALP) | 대응 없음 | Prime ECC는 min-sum 반복복호(`LDPC_Decoder()`)라 LP 디코더 패러다임과 불일치 |
| redundant parity-check(유효 컷) 생성 | 대응 없음 | PCM은 고정 QC-LDPC(`Load_PCM()`), 동적 컷 추가 개념 없음 |
| fractional 노드 분류/사이클 탐색 | 대응 없음 | min-sum message passing에 해당 구조 없음 |

적용 가치: **낮음** — LP 디코딩 전용 속도개선으로, min-sum 기반 NAND QC-LDPC 엔진에 떼어 쓸 접점이 없다.
