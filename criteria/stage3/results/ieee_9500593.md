### B. 알고리즘 요약

1. 시스템: BI-AWGN 채널, regular QC-LDPC(exponent matrix `P`, circulant size `Z`), rate 0.5~5/6, N=7648~18360, 4bit 이하 메시지 교환, 최대 8~10 iteration.
2. 문제: coarse quantization LDPC는 주로 flooding schedule로 연구됨 - 수렴이 빠른 layered schedule에 IB 기반 설계를 적용한 사례가 거의 없음.
3. layered schedule: full VN update 대신 layer별 partial update(`dv`개)로 각 layer가 개선된 신뢰도 메시지를 받아 수렴 속도 약 2배 향상(iteration 절반).
4. 핵심 기법(IB): node 연산을 `max p(t|s) I(X;T)`로 설계 - 관측 메시지를 압축하되 code bit `x` 관련 정보를 최대 보존, discrete density evolution로 메모리 내 분포까지 추적.
5. VN 설계1(LUT): 거대한 `|S|`(예 dv=4,4bit=65536)을 2-input LUT 연쇄로 분할, 각 LUT를 sequential IB로 경계 threshold 최적화(LLR 정렬 기반) - 서브구조 최적화라 손실 존재.
6. VN 설계2(computational domain): `s̃=L(x)+ΣL(yl|x)` 로 LLR 합산 후 threshold quantizer `Q(s̃)`, 정수 스케일 `φl`, w=8bit 내부폭이면 무손실 - 단일 압축이라 LUT 대비 MI 더 보존.
7. CN 설계: 2-input LUT / computational domain / min-approximation 3종, min-approx는 HW 복잡도 낮음.
8. 최적화: 오프라인 layered discrete density evolution + IB, 설계 `Eb/N0` 고정으로 초기 `p(x,ych)` 결정.
9. 결과: 4bit IB 디코더가 4bit OMSQ 대비 0.06~0.14dB 우위, min-approx CN이 오히려 error floor를 1~2 order 낮춤(C1 layered 10^-12), 3bit IB는 4bit OMSQ 대비 0.02~0.05dB 손실. GPU(PyOpenCL) 시뮬로 error floor 관찰.
10. 한계: fiber optical(R>0.8, floor<10^-15) 타깃, HW area/energy는 future work, layered는 중간결과 저장 추가 메모리 필요.
