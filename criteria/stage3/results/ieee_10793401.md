### B. 알고리즘 요약

1. biAWGN 채널에서 protograph 기반 이너 LDPC 코드(고정)와 외부 constant-composition distribution matcher(DM, CC 코드)를 직렬결합한 MacKay-Neal(MN) 코드; 블록길이 고정한 채 DM 파라미터 `ω`만 바꿔 rate 조정.
2. 기존 rate 적응 기법(puncturing/shortening/lengthening)은 pipeline 디코더가 요구하는 "단일 고정 Tanner graph" 제약과 상충하는 문제를 해결하려 함.
3. 핵심 가정은 non-systematic 이너 인코더 `G=H1^T H2^{-T}`가 조밀(dense)하여 결과 코드워드 비트 분포가 균일에 가깝다는 것과, DM 출력이 constant-composition(고정 Hamming weight `ωh`) 코드라는 것.
4. 비선형 결합코드 분석을 위해 equivalent parallel channel(EPC) 모델 도입: DM 제약을 BSC(교차확률 `ω`)로 근사해 이너 선형 LDPC 코드의 오류확률 분석으로 환원.
5. 핵심 식은 `R ≈ H_b(ω)·R_I`(rate), EPC 모델 하에서 BP 디코더는 표준 이터레이션 메시지 패싱을 이너 mother code Tanner graph 위에서 수행하며 punctured VN에 DM 사전확률 `Δ=ln[(1-ω)/ω]` 주입.
6. 확장으로 error-floor 분석: 이너 코드의 평균 input-output weight enumerator `Ā^IO_{a,b}`(Lemma 2)를 유도하고, 그 정규화 로그 점근형 `G(α,β)`(Theorem 1)의 부호(양/음)로 ensemble이 "bad"(error-floor 큼)/"good"(작음)인지 판별하는 기준을 제시.
7. 부수 트릭: 문턱값 계산은 빠른 PEXIT(Gaussian 근사) 분석을 우선 사용하되 저rate(ω 작음) 구간에서 부정확(최대 0.6dB 과소평가)하므로 정밀한 quantized DE(255단계 양자화)로 보정.
8. 최적화 절차: protograph base matrix를 worst-case loss(WCL, 목표 rate 구간에서 Shannon한계와의 최대 gap) 최소화 기준으로 differential evolution 탐색, good/bad ensemble 판별을 탐색 제약으로 추가해 나쁜 ensemble을 배제.
9. 결과: 예시 코드 family(rate 0.1~0.666)가 Shannon한계 대비 1dB 이내 문턱값 달성, good ensemble 선택(Example 9)이 bad ensemble(Example 8) 대비 저rate(R=1/5, 2/5)에서 FER 10^-5 부근까지 error-floor 없이 우수한 성능을 보임(블록길이 1800/9000 시뮬레이션).
10. 한계: HW 설계·복잡도·throughput 분석 없음(순수 이론+시뮬), BP soft 디코딩만 다루며 min-sum/HW pipeline 미검토, DM(arithmetic coding 기반) 자체가 추가 시스템 블록으로 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph 이너 LDPC 코드(H-matrix) 구성법 | `ecc_top.cpp` `Load_PCM()` | 부호설계 요소로서 대응되나 특정 QC-LDPC(PCM_b) 고정 구조 재검증 부담 큼(프로파일 §4 "높음" 난이도) |
| error-floor 판별용 input-output weight enumerator 기준 | 대응 없음 | Prime ECC 코드에 등가 분석 도구 없음. 개념적으로 error-floor 사전 스크리닝에 참고 가능 |
| BP(soft) 디코딩, Tanner graph 이터레이션 | `decoder.cpp` `LDPC_Decoder()` | 표준 BP이며 min-sum 근사·HCU·조기종료 등 Prime ECC 고유 최적화와는 무관 |
| distribution matcher(DM, rate 적응) | 대응 없음 | Prime ECC 인코더(`encoder.cpp`, dual-diagonal 고정)에 DM/CC코드 개념 없음 |

> 적용 가치: 중간 — error-floor를 사전에 판별하는 weight-enumerator 기준(good/bad ensemble)은 부호설계 이론으로 참고 가치가 있으나, DM 결합·BP soft 디코딩 중심 구조라 Prime ECC의 고정 QC-LDPC min-sum HW 디코더에 직접 이식하긴 어려움.
