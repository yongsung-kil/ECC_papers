### B. 알고리즘 요약

1. 대상은 girth-8, column weight `dv=4`인 fully-connected regular QC-LDPC(protograph를 circulant permutation matrix로 리프팅), rate `R=1-dv/dc` 범위 0.2~0.75, length `n=dc*z` 범위 305~46256.
2. 문제: girth 8 QC-LDPC는 harmful trapping set을 줄이지만, 기존 최적화(girth 8 + 모든 8-cycle 제거)는 필요 이상으로 큰 lifting factor `z`(circulant 크기)를 요구해 실용성이 떨어짐.
3. 핵심 가정: leafless elementary trapping set(LETS) 중 error-floor에 가장 유해한 것은 `(a,0)`(codeword)과 `(a,1)`(pseudo-codeword) 클래스이며, 8-cycle은 route type A/B/C로 분류 가능.
4. 핵심 기법: **Theorem 1** — dv=4 girth-8 프로토그래프에서 Type-A 8-cycle이 없으면 LETS(8,0)과 LETS(8,2)가 존재하지 않음을 증명(Type-B cycle까지 제거할 필요 없음).
5. 증명 방식은 LETS(7,4)의 조상 그래프를 Tanner graph node indexing 전수조사로 분석해, Type-A cycle 부재 조건 하에서 모든 가능한 인덱싱이 LETS(7,4) 생성 조건과 모순됨을 보임.
6. 확장: 기존 SES(Smart Exhaustive Search) 알고리즘[12]을 수정해 "no Type-A" 조건만 적용한 base matrix 탐색을 수행, "no Type-A,B"(기존[11] 기준) 대비 zmin을 비교.
7. 구현 디테일: 특별한 양자화·HW 변형 없음, 순수 조합론적 base-matrix(교환행렬 P) 탐색 알고리즘.
8. 최적화 절차: SES 알고리즘 기반 exhaustive/heuristic search로 dc=1~8에 대해 zmin과 base matrix를 표(Table I)로 제시.
9. 결과: dv=4,dc=16(R=0.75) 예제에서 Type-A만 제거 시 z=599(n=9584), Type-A,B 모두 제거 시 z=2891(n=46256)로 zmin이 약 5배 차이. BER은 pb=1e-7에서 기존 girth-8 무최적화 코드 대비 Eb/N0 약 0.25dB 개선(waterfall+error-floor 동시 관찰, BP decoder max 20 iteration).
10. 한계: dv=4 fully-connected 구조로 한정(parallel edge 없음), 실측/FPGA/ASIC 검증 없이 AWGN 채널 시뮬레이션(BP, 20회 반복)에 그침, 고rate에서 Type-B까지 제거하는 것은 zmin이 지나치게 커져 실용성 낮음.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| SES 기반 base matrix(circulant shift 값) 탐색·최적화 | 대응 없음 | 오프라인 부호 설계 도구로 실행 코드에 대응 모듈 없음, `PCM_b`(base+shift) 생성 시 참고 가능 |
| LETS(trapping set) 기반 error-floor 개선 | `decoder.cpp` `LDPC_Decoder()` (간접) | 디코더 알고리즘 변경 없이 부호 자체의 trapping set 제거로 error-floor 개선하는 접근이라 디코더 코드는 불변, H-matrix 재설계 시에만 효과 |

적용 가치: **중간** — HW/디코더 이식 요소는 없으나, error-floor 개선을 위한 girth-8 QC-LDPC H-matrix 설계 이론(Theorem 1)은 Prime ECC의 향후 부호 재설계 시 zmin 축소·trapping set 회피에 참고할 수 있는 이론적 근거 제공.
