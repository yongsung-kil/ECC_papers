### B. 알고리즘 요약

1. 대상: 5G NR QC-LDPC(BG2), rate 1/5·1/4·1/3, `N`=2600/2080/1560, `Z`=52, AWGN+BPSK, 4bit 양자화(`q`=4, APP `q̃`=6), layered schedule, `Tmax`=15.
2. 문제: 5G 코드는 고rate LDPC+LDGM 연접으로 degree-1 VN이 대량이라 유일 extrinsic 메시지에 민감, QMS/QOMS는 C2V magnitude 과대추정으로 성능 열화.
3. 모델: CN-update의 magnitude 계산만 BP(`logcoth`) 기준을 유지하고 나머지는 quantized 파이프라인 재사용.
4. 핵심 기법: BP CN-update `δ = f^{-1}(Σ f(|α|))` where `f(x)=logcoth(x/2)`를 LUT로 실현, offset 없는 정확 magnitude 계산.
5. 핵심 식 의미: `f`가 감소함수·involution(`f=f^{-1}`)이라 threshold 비교로 정수 매핑 가능 → offset 오차 제거.
6. 확장(분해매핑): 두 매핑 함수(Table I: `|α|→f(|α|)`, Table II: 합→최종 `δ`)로 분해해 테이블 크기를 `2^(q-1)*dc`에서 `2^q`로 축소.
7. 구현 트릭: Table II를 7개 threshold `L1>...>L7`의 binary search tree로 구현, 모든 분수를 정수화 → 단순 매핑+고정소수 덧셈만으로 CN-update.
8. 학습/최적화: 없음(결정적 threshold 설계). 채널 gain `μ`로 입력 양자화.
9. 결과: mismatch 확률이 QMS/QOMS보다 크게 낮고(일부 0), waterfall에서 QAMS(SOTA) 대비 약 0.5dB 개선, floating OMS 초과·BP 근접, 면적 3.586mm^2로 QMS(3.418) 대비 소폭 증가.
10. 한계: rate-1/4·1/3 FER=10^-5 부근 error-floor, HW는 [9] 아키텍처 재사용이라 상세 미제시, 5G 저rate 특화.
