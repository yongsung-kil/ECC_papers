### B. 알고리즘 요약

1. WiMax 802.16e 표준의 QC-LDPC 코드 6클래스(rate 1/2, 2/3A, 2/3B, 3/4A, 3/4B, 5/6), 서브행렬 크기 `z=24~96`(4단위), 코드워드 길이 `N=24z`(576~2304bit) 지원.
2. 문제: 다수의 rate/length/구조가 상이한 코드를 단일 하드웨어로 모두 처리해야 하는 유연성 요구.
3. 채널모델: AWGN, LLR 기반 belief propagation (BP) 메시지 패싱.
4. VN 업데이트: `λk = λch + Σλl - λk_old` (식 2), CN 업데이트는 저복잡도 λ-3-Min 알고리즘(식 3,4) 사용 - 3개 최소 크기값과 보정항 δ(x,z) 적용.
5. λ-3-Min은 Min-Sum보다 저rate 코드에서 성능 열화를 보정하며 넓은 rate 범위에 최적 절충안.
6. Layered decoding(TDMP, Hocevar 방식) 적용 시 CN 서브셋 처리 직후 갱신 메시지를 VN에 즉시 반영해 수렴속도 향상; rate 1/2, 2/3B 코드만 이 제약(latency dc cycle)을 만족하도록 설계됨.
7. HW: parallelism P=96(edge/cycle), z-by-z permuted identity 서브행렬을 유연 barrel shifter(19단 wrap-around)로 구현, 6bit 채널 입력/9bit 메시지 quantization.
8. 최적화 절차 없음(코드는 표준 고정, HW 아키텍처 설계만).
9. 결과: 0.13µm 공정 333MHz에서 면적 3.834mm2, throughput 83~619Mbps(코드별), latency 3.7~6.9µs; layered 코드만 지원 시 면적 3mm2 미만으로 축소 가능.
10. 한계: 유연성 확보가 목적이라 특정 고정 코드 최적화 대비 면적 오버헤드 크며, error-floor/정정능력 개선 기법은 다루지 않음(순수 아키텍처 논문).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| λ-3-Min CN 업데이트 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Min-Sum 계열 변형이나 이미 Min-Sum 보유, 중복 기여 |
| Layered decoding(TDMP) | `decoder.cpp` `LDPC_Decoder()` | Dual-Update와 유사 개념, 이미 보유 기법과 중복 |
| Barrel shifter 유연 permutation network | 대응 없음 | Prime ECC는 z=32 고정 lifting, multi-z 유연 네트워크 불필요 |
| Sum RAM / message RAM 구조 | `decoder.cpp` (pipeline 6-stage) | 유사 개념이나 코드 다중지원 특화라 직접 대응 약함 |

적용 가치: **낮음** — 표준 다중 코드 지원용 유연 아키텍처가 핵심이며, Prime ECC는 고정 단일 QC-LDPC 구조라 유연성 확보 기법의 이식 필요성이 낮고 CN/layered 기법도 이미 보유.
