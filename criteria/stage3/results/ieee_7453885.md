> 총평: GLDPC(Hamming component code) 부호설계로 수렴속도·BER을 개선하지만, SPC 대신 Hamming 코드 기반 체크노드와 MLD 복호를 요구하는 이질적 구조라 QC-LDPC/min-sum 기반 Prime ECC에는 이식성이 낮음.

### B. 알고리즘 요약

1. `(N,M,K,n,m,k)` Hamming-GLDPC 부호: 표준 LDPC의 SPC 체크노드를 `(7,4)` 등 Hamming 코드 체크노드(HCN)로 대체해 구성, 예시 `(147,126,21,7,3,4)` 부호(rate ≈0.121~0.143).
2. 표준 LDPC의 SPC 체크노드는 제약이 약해 수렴 속도·BER 성능에 한계가 있다는 문제의식에서 출발.
3. 채널 모델은 BEC(erasure prob. `q`)와 AWGN(BPSK) 두 가지를 사용해 이론(EXIT)과 시뮬레이션을 각각 수행.
4. 핵심 기법: PEG로 만든 표준 LDPC 체크행렬의 비영 원소들을 Hamming 코드 체크행렬의 열로 치환해 GLDPC 부호를 구성(VN은 REP, CN은 Hamming code).
5. Hamming 코드는 자체 EXIT 함수가 복잡해 dual code인 simplex code의 EXIT(`IE=(IA)^{h-1}(1-IA)^{n-h}`, 식(9))를 duality property(`I_E^⊥(IA)=1-IE(1-IA)`)로 유도해 우회 계산.
6. 확장으로 non-exhaustive search를 이용해 최적 edge degree 분포 `λ=[0.9463,0.0293,0.0144,0.01]`(`dv=[2,3,4,6]`)를 탐색, decoding threshold `q=0.7710` 도출.
7. AWGN에서 HCN 복호는 SPA 대신 MLD(maximum likelihood decoding)로 Hamming 제약을 직접 처리(부수 구현 디테일).
8. 최적화 절차: EXIT chart 기반 non-exhaustive search로 최적 VN degree 분포 탐색(GA/DE 유사 최적화는 아님, 논문 자체 탐색법).
9. 결과: `(147,21)` Hamming-GLDPC가 최대 반복 50회에서 표준 LDPC(최대 반복 50) 대비 BER 10^-3 기준 약 1.3dB 이득, 반복 5/10/20/30회로 줄여도 각각 0.2/0.5/0.8/1.1dB 이득 유지, 약 20회 반복 내 수렴, 평균 반복수도 표준 LDPC보다 낮음.
10. 한계: HW 설계 없음, 짧은 코드(147bit) 하나로만 검증, 실용적 복호 알고리즘은 "향후 과제"로 남김(SPA 대체할 효율적 HCN 복호기 미제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| H-matrix / 부호구조(Hamming 체크노드 GLDPC) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 SPC 기반 표준 QC-LDPC 고정 구조로, Hamming CN 기반 GLDPC는 이질적 부호 클래스라 이식 난이도 높음 |
| CN 복호(MLD 기반 Hamming 제약) | 대응 없음 | Prime ECC의 min-sum CNU(`CNU_Update_New_Mag()`)는 SPC 기반이며 Hamming code MLD 복호와 메커니즘이 다름 |
| iteration감소/수렴속도 개선 | `decoder.cpp` `LDPC_Decoder()` | 개선 효과는 부호설계(체크노드 강화)에서 기인하므로 디코더 루프 자체 변경 없이는 이식 불가 |

> 적용 가치: 낮음 — 체크노드를 Hamming code로 대체하는 GLDPC 부호설계·MLD 복호이며, Prime ECC의 SPC 기반 QC-LDPC + min-sum 구조와 근본적으로 다른 부호 클래스라 이식 난이도가 크고 실용 복호 알고리즘도 미제시.
