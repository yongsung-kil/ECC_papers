### B. 알고리즘 요약

1. 대상은 IEEE 802.11n/802.16e/802.15.3c/802.3an 등 최신 통신표준용 LDPC 디코더 가속기의 아키텍처 개발 방법론 (특정 부호/알고리즘 아님).
2. 문제: 기존 high-level synthesis는 처리유닛 micro-architecture 합성만 지원하고, LDPC 같은 massive 데이터 병렬 + 복잡한 메모리/통신 병목 문제를 통합적으로 다루지 못함.
3. LDPC 배경: PCM `H`(mxn) 기반 iterative Message Passing(MPA)로 VN(가산)·CN(log/tanh 또는 min 비교) 반복, LLR 교환 후 parity check 만족 시 종료.
4. 핵심 관찰: VNP/CNP 처리유닛은 fully-serial~fully-parallel 스펙트럼, macro-architecture(다수 PU)와 micro-architecture(PU 내부 병렬도)가 강하게 상호의존.
5. 핵심 기법: macro/micro-architecture와 computation/memory/communication 구조를 하나의 통합 아키텍처 합성 task로 보고 동시 결정.
6. quality-driven·model-based 접근: effectiveness/efficiency를 측정가능 파라미터로 모델링해 multi-objective decision model로 설계공간 탐색.
7. generic architecture template(재사용 가능한 파라미터화 VNP/CNP·메모리·인터커넥트·ROM(PCM)·HD/parity check unit 등) 인스턴스화로 설계 재사용+합성 혼합.
8. 설계 파라미터: 디코딩 알고리즘, intrinsic/extrinsic 메시지 bit-precision, 최대 iteration/정지기준, 동작주파수, PU 병렬도, PU 개수·노드매핑, 메모리 구조, 스위칭 네트워크(barrel/Benes/Omega).
9. 결과: 정량 수치 없음. "positive first impression"으로 짧은 시간 내 여러 후보 아키텍처 구성·분석 가능하다는 정성적 주장뿐.
10. 한계: 실제 구현·합성·성능수치 부재, 방법론(프로세스) 서술에 그침, 부호설계나 디코더 알고리즘 자체의 기여 없음.
