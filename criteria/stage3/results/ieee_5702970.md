> 총평: QC parity-check matrix에서 protograph LDPC의 generator matrix가 block-circulant가 됨을 증명하고, sub-matrix 첫 행만 저장하는 고속 인코딩을 제시. 인코더 저장·면적 절감 관점은 유용하나 Prime은 이미 dual-diagonal 인코딩을 보유해 이식가치는 중간.

### B. 알고리즘 요약

1. 대상: protograph(=multi-edge-type의 하위) LDPC, rate 2/5, 시뮬 부호 `N=1024` 리프팅으로 `(n=5120, k=2048)`.
2. 문제: 일반 generator matrix `G`는 dense·random 위치의 1이라 인코더가 모든 1을 저장 → 칩 면적 큼. 저장·인코딩 복잡도 절감 필요.
3. 정의: `l`차 unit circulate matrix `E_i`(단위행렬 우측 shift), quasi-cyclic matrix(zero+unit circulate 블록), block circulate matrix; QC는 block-circulate의 하위 클래스.
4. 핵심 성질(Property 1~5): unit/circulate/block-circulate 행렬의 곱·합 폐포성, 그리고 Property 5 — 비특이 QC matrix의 역행렬은 block-circulate matrix.
5. Lemma 1 / Theorem 1: `H=[Hb Ha]`가 QC이고 `Ha`(=P)가 비특이면 generator matrix `G=[I_ml | W]`, `W=(P^-1 Q)^T`가 block-circulate가 됨을 증명.
6. 핵심 의미: `P^-1`이 block-circulate(Property5)이므로 `W`도 block-circulate → `G`의 각 sub-matrix는 첫 행만 알면 나머지는 우측 shift로 복원.
7. 구성 절차: `H`에서 P(마지막 3N열)·Q(첫 2N열) 분리 → mod-2로 `W=(P^-1 Q)^T` 계산 → `G=[I_2N | W]`, 각 permutation을 4x4 QC 하위블록으로 확장.
8. 고속 인코딩: `c=m·G`, 정보벡터 `m`을 길이 N/4 블록 8개로 나눠 각 sub-matrix `W_i,j`의 첫 행만 저장·shift로 연산 → 저장공간·연산 감소, girth-4 없음(Theorem 2로 검증).
9. 결과: 제안 부호(length 5120)가 GB20600 표준 LDPC(length 7493)와 동일 rate·AWGN·동일 BP 복호에서 BER 1e-7 기준 약 0.6dB 우수, 더 짧아 인코딩 속도·구현복잡도 유리.
10. 한계: HW 미설계·이론+시뮬만, rate 2/5 저rate 예제라 NAND 고rate와 직접 부합 안 함, 정정능력은 복호 아닌 부호구조 부산물 수준.
