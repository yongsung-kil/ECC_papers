### B. 알고리즘 요약

1. 대상: 802.11n (1944,972) rate-0.5 QC-LDPC (base matrix + CPM/ZM), AWGN/BPSK, FER 1e-7·BER 1e-10까지 시뮬.
2. 문제: FPGA/ASIC 디코더는 고성능이나 비유연·고비용 - GPU의 대규모 병렬성으로 유연하면서 고throughput 디코더를 만들고 싶다.
3. 알고리즘: Normalized Min-Sum(NMSA), `L_c2v = α·ΠS·min`, flooding/layered 스케줄, 조기종료(ET) 적용.
4. 핵심기법1(flooding): V2C·C2V 두 phase 모두 스레드를 CN에 할당해 메모리 접근패턴 개선, barrier 대신 atomic 연산으로 동기화, 압축 base matrix 재배열로 동시쓰기 확률 저감.
5. 레지스터 저장: 각 스레드가 한 row만 처리하므로 V2C/C2V를 shared/global 대신 레지스터에 transformed 표현(`min1,min2,loc,sign product`)으로 저장 - shared memory 접근 감소, 중복연산 비용 감수.
6. 핵심기법2(layered): shared memory가 병목 - transformed 표현으로 8/16/32bit 양자화서 메모리 40/50/55% 감축, 블록당 SM 상주수 25개 상향.
7. 구현 트릭: 부호를 bitmap 저장, sign product를 순차 곱셈 대신 XOR 병렬로 계산해 명령수 축소, V2C 중간값을 APP용 shared에 재사용.
8. 결과: 정규화 throughput flooding 55%↑/layered 34%↑, ET 포함 4.77/3.67 Gb/s. 큰 부호(18360,16524, submatrix 306)선 layered가 flooding보다 36%↑.
9. 추가: 동일 flooding을 Virtex UltraScale 440 FPGA에도 구현해 76% 자원사용률로 최대 23.72Gb/s.
10. 한계: 디코딩 알고리즘(NMSA) 자체 신규성 없고 부호·정정능력 개선 없음, 순수 GPU 병렬/메모리 최적화라 HW 디코더 회로설계와 무관.

적용 가치: **낮음** - 대상이 GPU 소프트웨어이고 알고리즘(NMSA)·부호·정정능력 신규성이 없어, NAND용 bit-exact QC-LDPC HW 디코더(min-sum 이미 보유)에 떼어 쓸 요소가 사실상 없다.
