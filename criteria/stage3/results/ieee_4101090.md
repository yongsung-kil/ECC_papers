> 총평: QC-LDPC를 SPC trellis Log-MAP 근사(turbo-style)로 복호해 메모리를 크게 절감한 FPGA 아키텍처로, 정정능력 개선은 없고 HW 메모리 효율화 관점에서만 참고 가치가 있음.

### B. 알고리즘 요약

1. QC-LDPC(base matrix + S×S 순환치환 확장, girth-6, C={2,3,4}, R={5,6}, density evolution 기반 차수분포)를 대상으로 rate 0.46, length 3200(C1)/6400(C2) 코드 두 종을 구성.
2. 문제의식: 기존 FPGA sum-product 디코더는 bit-to-check/check-to-bit 메시지를 모두 저장해야 해 메모리 오버헤드가 큼.
3. 채널은 BPSK+AWGN. 각 parity-check 행을 (r,r-1) SPC(single parity-check) code의 trellis로 표현(Fig.2)하고 Log-MAP 알고리즘을 이 trellis에 맞춰 적용.
4. 핵심 기법: SISO(soft-in soft-out) message decoding unit(MDU)이 α/β forward-backward recursion으로 `Λ_k`(갱신 LLR)를 계산 — bit-to-check/check-to-bit 메시지 대신 posterior LLR(`γ = δ + Σλ_i`) 하나만 주고받아 메모리 절감.
5. 핵심 식: `q(x,y) = max(x,y)+ln(1-e^-|x-y|) - max(x+y,0)-ln(1-e^-|x+y|) = Q(x,y)+δq(x,y)`. HW에서는 보정항 `δq`를 생략하고 `Q(x,y)`만 구현("modified turbo decoding algorithm")해 throughput 확보(성능 저하는 C1 0.5dB, C2 0.4dB로 제한적).
6. 확장/구현 디테일: quad-port γ memory(동시 read/write로 처리량 2배)와 barrel shifter 기반 cyclic interleaver/deinterleaver(π memory의 shift값으로 1클록에 회전).
7. 양자화: 5비트 고정소수점이 성능-면적 균형점(4/5/6비트 비교, Fig.8)으로 채택.
8. 학습/최적화 절차 없음(구성 알고리즘은 [5] Hocevar 방법 재사용, density evolution으로 차수분포만 결정).
9. 결과: Xilinx Virtex-4(XC4VLX2512FF668)에서 clock 180MHz, throughput 223Mbps(1.24bits/cycle), 메모리 71Kb(72214bit)만 사용 — 비교 대상 문헌들 대비 최소 BRAM·2위 최소 logic로 throughput 최고.
10. 한계: 정정능력 개선이 목적이 아니라 메모리·throughput HW 최적화가 목적. 디코딩 알고리즘이 min-sum이 아닌 SPC-trellis Log-MAP 근사(turbo류) 방식이라 Prime ECC의 min-sum CN 연산 구조와 직접 대응되지 않음. 순수 시뮬레이션이 아닌 FPGA(RTL) 검증이나 정정능력 데이터는 waterfall 영역(Eb/No 1~2.6dB)만 제시, error-floor 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPC-trellis Log-MAP 기반 turbo-style 복호(Q(x,y) 근사) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum(CNU_Update_New_Mag) 기반이라 알고리즘 구조 자체가 다름 — 대체가 아닌 참고 수준 |
| quad-port γ memory (posterior LLR 통합 저장) | `decoder.cpp` LLR 테이블 참조부, `ecc_data.h` `PARAM_LLR` | 메시지 병합 저장 아이디어는 참고 가능하나 Prime ECC는 C2V/V2C 별도 메시지 구조 |
| barrel shifter 기반 cyclic interleaver/deinterleaver | `GT.cpp` `Make_GT_HW()` (edge 라우팅 관련) | QC 순환치환 라우팅이라는 점에서 개념적으로 유사, 직접 대응은 아님 |
| 5bit 고정소수점 양자화 선택 근거(4/5/6bit 비교) | `channel.cpp` `Set_LLR_Th()` | soft 양자화 비트수 트레이드오프 참고 가능(Prime ECC도 2SD/3SD 등 제한 비트 사용) |
| QC-LDPC base matrix + girth-6 구성(Hocevar 방법) | `ecc_top.cpp` `Load_PCM()` | 부호설계 자체이나 Prime ECC는 이미 고정 QC-LDPC 구조 보유, 재검증 부담 |

적용 가치: `중간` — 정정능력 개선은 없으나 메모리 절감형 메시지 표현(SISO 병합)과 barrel shifter 라우팅은 HW 면적/메모리 최적화 아이디어로 참고할 여지가 있음.
