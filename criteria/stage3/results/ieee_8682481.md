### B. 알고리즘 요약

1. 대상은 5G NR rate-compatible QC-LDPC(base graph, circulant `z`), rate `0.303~0.846`, layered message passing 복호(LMPD), Rayleigh fading 채널.
2. 문제: 5G PCM은 정보비트 앞 2 block column이 punctured라 zero-padding된 unreliable 노드 탓에 수렴에 더 많은 반복이 필요.
3. LD 스케줄: check-node degree(row weight)가 낮은 레이어를 먼저 복호하면 신뢰도 높은 a posteriori LLR을 먼저 확보해 평균 반복수 감소(비punctured rate 0.324에서 4dB시 15.6%↓).
4. OE 스케줄: punctured edge가 적은(one-punctured-edge) 레이어를 먼저 복호해 unreliable 노드 영향 축소; OE+LD 결합(OELD)으로 punctured rate 0.303에서 반복 17.6%↓, BER 0.125dB 개선.
5. CC-HARQ: 1차 복호에서 얻은 앞 2 punctured block column의 a posteriori LLR을 2차 복호에 재사용(`Prop.cc`)+LDOE 정렬로 Conv.cc 대비 반복 최대 36.27%↓, 추가 메모리는 system buffer 재활용.
6. IR-HARQ: 재전송 parity를 row weight 낮은 pure-row-orthogonal(PO) 부분부터 보내면 quasi-row-orthogonal(QO)보다 BER=10^-3에서 0.2dB 개선.
7. HW: conventional layered 디코더[12]에 LLR memory(CC)·concatenator(IR)+`384*2` adder/MUX와 스케줄 순서용 소형 ROM만 추가.
8. 검증: Virtex-7 FPGA, 46.5MHz 합성; LUT 224888 / FF pair 69335.
9. 결과: rate 0.303에서 1차 throughput +21.37%, CC-HARQ +56.9%, IR-HARQ +14.51%; throughput 최대 350.06 Mb/s.
10. 한계: 5G HARQ/puncturing 구조 특화, FPGA 합성만(ASIC 아님), gate count·부호길이(bit) 본문 서술 없음.
