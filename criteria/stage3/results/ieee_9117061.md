### B. 알고리즘 요약

1. 대상은 5G LDPC 표준(BG1/BG2, lifting size Z=2~384, 51종)의 QC-LDPC 디코더 내 cyclic shift(순열) 연산을 수행하는 permutation network.
2. 기존 barrel shifter는 다중 크기/다중 프레임 병렬 처리가 어렵고, 기존 variation Banyan network[10]는 Z≤6까지만 병렬 처리(최대 4프레임) 가능해 작은 lifting size에서 처리량이 저하되는 문제를 해결하려 함.
3. 채널/노이즈 모델은 다루지 않으며, 순수 조합논리 셔플 네트워크의 구조·타이밍 문제.
4. 핵심 기법은 Banyan network의 next-to-last stage 제어 신호(`Ci`)를 새로 설계해 `(3×2^n)×(3×2^n)` 네트워크를 최대 `2^n`개의 `3×3` 서브네트워크로 분할, 병렬 처리 하한 `Zl`을 6에서 3으로 낮춤(200% 병렬도 향상, Z=2,3에서).
5. 원 네트워크의 shift값 `SV1`과 bypass 네트워크의 `SV2=PMs-Z+SV1`을 병렬 계산해 MUX로 선택하는 구조(`Alg. 1/2`)로 blocking 문제를 회피.
6. 두 번째 기법은 shift value `SV`와 그 이진표현 `SVb`를 재귀적으로(비트 추가) 매핑하는 스킴으로, 제어모듈 내 모든 modulo 연산(`SV%(3×2^i)`)을 제거해 critical path 단축.
7. HW 구현은 두 개의 384×384 Banyan network + 768-to-384 MUX stage + shuffle module + 3개 제어모듈, sel 3bit로 8가지 병렬도 옵션(PMs=384~3) 지원.
8. 별도 학습/최적화 절차 없음 (순수 조합논리 설계, DE/GA/NN 미사용).
9. TSMC 90nm 합성 결과 660MHz(384×384), 96×96 비교에서 770MHz(기존[10] 대비 28.3% throughput 개선, Z≤12에서는 156~926% 개선); 게이트카운트는 표(Table III/IV)에만 있고 본문 서술 수치 없음.
10. 순수 상호연결 HW 구조 논문으로 디코딩 알고리즘(min-sum, error-floor 등)과 무관하며, 부호설계/정정능력 개선과는 완전히 별개 영역.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 순열(cyclic shift) 네트워크 전체 | 대응 없음 | Prime ECC는 z_sb=32 고정 row 단위 병렬 구조로 5G식 가변 lifting size 순열망 자체가 불필요 |
| Graph Thinning MUX 기반 edge 선택 | `GT.cpp` `Make_GT_HW()` | 본 논문의 MUX 기반 원본/bypass 선택 구조와 개념적으로 유사하나 목적(순열 vs edge 축소)이 달라 직접 대응은 아님 |
| H-matrix/PCM 로드 | `ecc_top.cpp` `Load_PCM()` | 5G BG1/BG2 lifting 구조와 Prime ECC의 base+circulant 구조가 개념적으로만 유사, 코드 재사용 불가 |

적용 가치: 낮음 — 5G 표준 다중 lifting size(가변 Z) 대응을 위한 범용 상호연결 HW로, NAND용 고정 z=32 QC-LDPC 구조에는 직접 이식 대상이 없음.
