### B. 알고리즘 요약

1. 시스템: Xilinx Vitis HLS로 layered semi-parallel QC-LDPC 디코더를 FPGA(Alveo U50)에 구현. 부호는 802.16e/CCSDS 표준, rate 0.5~0.9, expansion factor Z=64, iteration 5.
2. 문제: HLS 생성 QC-LDPC 디코더는 RTL 전문가 설계 대비 QoR 격차가 큼(H-LDPC는 RTL 대비 throughput 44%↓, HW 584%↑). HLS가 iteration/circuit 수준 디테일을 정확히 기술 못 함.
3. 접근: 세 가지 핵심 모듈의 behavior-description을 정교화(refine)해 HLS 결과 품질을 RTL급으로 개선.
4. 최적화1 - MSO(Multi-instance Sharing One): CNU 배열에서 공유부(CNU control)/비공유부(PS/FS/q_sign)를 명시 분리, 배열 멤버 클래스로 표현해 중복 함수·주소 회로 제거. CNU array LUT 13% 감소(1345->1165).
5. 최적화2 - 단일단(single-stage) 시프터: array_partition으로 barrel-shifter 구조 유도, HLS의 불필요한 bit-selection 회로 제거(LUT 8599->6627, delay 1->0 cycle).
6. 최적화3 - 다단(multi-stage/logarithmic) 시프터: `count` 비트별로 0 또는 2^i 상수 시프트를 단계 연쇄, 복잡도 `O(N^2)->O(N logN)`. LUT 8599->797(1 cycle 지연).
7. 최적화4 - invalid-element aware loop scheduling: base matrix의 '-1'(무효, 802.16e 1/2에서 ~70%) 원소를 compile 전에 제거·압축(c_matrix+c_index)해 pipeline loop iteration 수를 유효 원소만으로 축소.
8. 복호 알고리즘: 표준 layered message-passing min-sum(flood/layered 서술), min_finder + Rgenerate로 CNU 갱신. 알고리즘 자체는 신규성 없음.
9. 결과: H-LDPC 대비 HW 최대 56~66%↓, latency 최대 67%↓, throughput 최대 300~305%↑(209Mbps 단일DU). coarse-grained DU pipeline으로 48 DU까지 선형 확장, 최대 9.9Gbps(결론엔 9.6Gbps 병기).
10. 한계: 정정성능(BER/error-floor) 개선 없음. Vitis HLS 툴/pragma 종속 최적화라 알고리즘·부호설계 기여 부재, 통신 표준(802.16e/CCSDS) 부호 대상.
