### B. 알고리즘 요약

1. 시스템: 우주선(위성)용 방사선 내성 고용량 SSD "Pelican", 3D-NAND(TLC) + 커스텀 flash 컨트롤러, 목표 2TB / read 2000MB/s / 1200 TBW, NVMe over PCIe Gen4 x4.
2. 풀려는 문제: 위성 온보드 데이터 저장이 데이터 생성 속도를 못 따라가고, 기존 방사선 내성 저장장치는 저용량/대형, 상용 SSD는 우주환경 미대응.
3. 플랫폼: Xilinx Versal AI Edge SoC(7nm FinFET, ARM A72/R5 + TMR MicroBlaze + AI Engine/DSP + FPGA), 캐시/RAM에 Hamming(SECDED) ECC.
4. ECC 정책: NAND에 BCH 대신 LDPC 채택, hard-bit/soft-bit(다중 전압 read) 복호 가능, 블록별 P/E cycle(노화)에 따라 redundancy를 점진 상향 튜닝.
5. 핵심 서술: "Some work remains to identify the best code"로 구체 LDPC 부호·디코더·rate·length·양자화는 미확정(선택 미완).
6. 신뢰성 계층 1: 컨트롤러 코어 전부 TMR + temporal skew(코어별 명령 지연) + FPGA fabric 물리 배치 분산으로 SEU/SEFI 완화.
7. 신뢰성 계층 2: RAISE(die 간 parity) 확장 — 독립 NAND manager 2개 string으로 데이터 이중화, NAND 절반 고장까지 동작.
8. 하드웨어 보호: 방사선 내성 전원, SEL 감지 load switch, NAND 내부 charge-pump 취약점 회피용 외부 VPP 핀 사용.
9. 부가 기능: 연산 스토리지(computational storage) — Versal AI Engine/DSP로 드라이브 내 AI/ML in-situ 처리(GPU 대비 W당 최대 5배).
10. 상태/한계: NASA SBIR Phase I(TRL 2→3) 완료, 프로토타입 미제작, 구체 ECC 알고리즘·성능 실측치 없음(제품 아키텍처 개요 수준).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NAND LDPC ECC 채택(정책 언급만) | 대응 없음 | 부호설계/디코더 세부가 없어 매핑할 기법 자체가 없음 |
| hard-bit/soft-bit 다중 전압 read | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` (개념상) | soft-read 개념만 공유, 구현/양자화 세부 없음 |
| 블록별 redundancy 튜닝(노화 대응) | 대응 없음 | rate 적응 정책 개념뿐, 부호 구성법·전환 알고리즘 미기재 |
| TMR / RAISE / 시스템 신뢰성 | 대응 없음 | 컨트롤러 시스템 아키텍처 영역으로 ECC 코어와 무관 |

적용 가치: **낮음** — NAND SSD에 LDPC를 쓴다는 시스템 설계 개요일 뿐, 부호설계·디코더·양자화·HW ECC 세부가 없어 Prime ECC에 이식할 기법이 존재하지 않는다.
