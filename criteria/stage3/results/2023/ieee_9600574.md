### B. 알고리즘 요약

1. 시스템: MLC(4PAM, k=2) NAND 플래시를 BICM-ID로 모델링, D=2 mapping(Gray+anti-Gray) 혼합(IM), 부호는 rate-0.9 protograph, codeword length 4000.
2. 문제: 기존 BICM-ID는 outer iteration이 많아 복호 복잡도·지연이 큼. IM으로 수렴 가속하되 flash용 protograph 부호 분석·설계 도구가 부재.
3. 채널 모델: `Vth = vw+nu+np+nw+nr+nc` (ISPP/프로그램/RTN/retention/셀간간섭), 6 read-voltage로 4입력 7출력 DMC 양자화(3 erasure 영역).
4. 핵심기법 1 — EPEXIT: 기존 PEXIT를 확장해 검출기↔복호기 outer iteration, IM, threshold voltage 변동을 반영해 MI 진화(`IE,det=IA,dec`)를 추적, decoding threshold(dB) 산출.
5. 등가 SNR: Gaussian 근사로 `Eb/N0 = Σμu²/(4R·Σσu²)` 정의, PE cycle을 등가 SNR로 환산.
6. 핵심기법 2 — IMARA 부호설계: rate-0.9 base matrix(3×21)에 degree-1 precoding VN, 고차 punctured VN, degree-2 VN 제약을 걸고 EPEXIT exhaustive search로 최적 base matrix 도출, degree-3 VN 패턴 반복으로 고rate 확장(linear-min-distance-growth 보존).
7. 핵심기법 3 — read-voltage 최적화: voltage-region iterative gain 특성(dHam=2인 dominant-gain 영역 E2가 outer iteration으로 MI 크게 증가)을 이용, E2에만 hard-decision level 1개 추가(총 7 read voltage), border 재최적화(θ2=0.25).
8. 학습/최적화: EPEXIT 기반 exhaustive/bisection search로 base matrix 및 voltage entropy θ 탐색(NN 없음).
9. 결과: IMARA는 decoding threshold 11.405dB(capacity limit 11.301dB 대비 0.182dB gap 원문 표기), MMI 대비 ~1000 PE cycle 개선, IM-BICM-ID가 conventional BICM-ID 대비 평균 inner iteration 50~60% 감소.
10. 한계: HW 미설계(MATLAB 시뮬만), MLC 4PAM·BICM-ID·outer iteration 결합 구조 전제라 순수 binary QC-LDPC 복호 파이프라인엔 직접 적용 불가.
