### B. 알고리즘 요약

1. 시스템: 3-D charge-trap TLC NAND(YMTC 64 WL), QC-LDPC(`length=16383`, `rate=0.87`, ACE+stopping-set 구성), 최대 iteration 50.
2. 문제: TLC는 분포 마진이 좁아 read 횟수 증가로 read latency가 커지며, 기존 LLR 보정(LCA)은 추가 read 또는 다른 논리페이지의 정정된 codeword를 요구해 강건성·지연이 나쁨.
3. 모델: 기존 N-M 채널모델(i.i.d. 가정) 대신 논리페이지별 `2-M`(예: 2-input 7-output) 이산 무기억 채널모델을 세워 정확한 상호정보 계산.
4. 핵심 기법: LIAD - hard-decision과 soft-decision 사이에 "hard decision-soft decoding" 단계 삽입, page-buffer의 sensing 결과(D1 등 위치정보)를 추가 read 없이 전송해 region을 세분하고 LLR을 보정.
5. 의미: hard-decision에서 NXOR로 손실된 상태 위치(Region1 vs Region3 등)를 복원하면 region별 transition prob으로 `LLR=log(P_a/P_b)`를 정밀화 - 추가 read 비용 0.
6. 확장: sequential read 시 다른 논리페이지의 sensing 결과까지 활용(2-7 채널), region을 최대 8개로 세분해 성능 추가 향상; random read는 현재 페이지 센싱만으로 제한적 활용.
7. 판정 절차: MIC(상호정보)로 채널용량 계산 후 `MI>CR`이면 LCA로 LLR 갱신, 실패 시 conventional soft decoding으로 폴백(Algorithm 1/2/3).
8. 복호기: variable/check node 반복 갱신, CN 업데이트가 `sgn 곱 x min` 형태의 min-sum(Algorithm 3, 식 CN=Πsgn·min).
9. 결과: UBER 최대 99% 감소, decoding iteration 최대 80%(하위페이지 13~50%) 감소, SSDsim read latency 최대 53%(전체 read 기준 43%) 감소, EMAL 대비 5~10% 추가 개선.
10. 한계: HW(gatecount/throughput) 미설계, sequential read·중간 RBER 구간에 이득 집중(Systor 동시read 부하에선 열세), 3-D TLC 물리특성·특정 칩 데이터 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 2-M 채널모델 기반 region별 LLR 보정(LCA) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold/LLR 산출 지점과 직결, region 세분 LLR 이식 여지 큼 |
| iteration별/상황별 LLR 갱신 값 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 보정된 LLR을 VNU 테이블/파라미터로 반영 가능 |
| min-sum 복호기(CN=Πsgn·min) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 Min-Sum과 동일 계열(중복, 신규 기여 아님) |
| 조기종료(parity 만족 시 종료) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 이미 보유(중복) |
| QC-LDPC 부호구조 로드 | `ecc_top.cpp` `Load_PCM()` | rate 0.87/len 16383 고rate QC-LDPC로 우리 부호와 유사 계열이나 H-matrix는 고정 |
| MIC(상호정보) 기반 채널용량 판정 | 대응 없음 | Prime ECC에 상호정보 기반 read/전송 판정 로직 없음(신규 추가 필요) |

적용 가치: **중간** — 추가 read 없이 sensing 위치정보로 LLR을 개선하는 핵심 아이디어는 Prime ECC의 `channel.cpp` LLR/threshold와 `PARAM_LLR` 레이어에 이식 여지가 크지만(관심 높음: soft-read 비용 절감), 복호기 자체(min-sum/조기종료)는 이미 보유이고 2-M 채널모델·MIC 판정과 page-buffer 센싱 파이프라인은 신규 구현·특정 칩 재검증 부담이 있음.
