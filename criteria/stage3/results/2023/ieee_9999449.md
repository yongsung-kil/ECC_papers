### B. 알고리즘 요약

1. 시스템/부호: `(n,k)` RS 부호 `GF(2^p)`의 이진영상 `CBRS`(행)와 SPC(열)로 2D 곱부호 P1/P2 구성, rate `0.82~0.91`, 광통신(100G+ PON) 대상.
2. 문제: BTC의 SDD(Pyndiah)는 오버헤드>15%에서 성능 붕괴·복잡, LDPC BP는 큰 내부 데이터플로우로 수백 Gbps 구현 난제.
3. 모델: BPSK over AWGN 또는 ISI 채널, LLR `Li=yi`(AWGN)/SOVA(ISI), RS는 연속 비트오류 정정 강점으로 ISI에 유리.
4. 핵심기법1(2단 복호): local decoding(LD)=RS 성분을 BM-HDD로 복호 후 정지기준 만족 sub-block을 `freezing`(hard decision 고정), global decoding(GD)=행/열 반복복호.
5. 핵심식: ML metric `M(v̂)=Σ|Li|(v̂i⊕d̂i)` (식7), `M(v̂)<Mθ`(식8) 정지기준으로 BM-HDD miscorrection 회피.
6. 핵심기법2(ID/ID-NACA): 성분복호기 간 3진 메시지 `{-1,0,+1}`만 교환(식9~10), locking 완화 위해 인공 가우시안 잡음 주입(식11)으로 Chase 유사 복호(NACA).
7. 구현 트릭: 인공잡음은 오프라인 LUT(길이 65536, 393Kbit)로 저장, 양자화 `Qch=6`, 교환메시지 `Qex=2`(3진).
8. 최적화: `Mθ`,`σa` 파라미터 시뮬로 최적화, waterfall/error-floor 반해석적 추정법(FER 하한식16/23, PTS 기반 error-floor 식26~36) 제시.
9. 결과: ISI 채널서 BCH-BTC SDD 능가, MSA 대비 계산복잡도 1/7~1/10·데이터플로우 1/46, Iavg 0~2(저BER), 일부 부호 error-floor `<10^-19` 추정.
10. 한계: HW 미설계(계산복잡도·데이터플로우 해석만), RS 심볼복호(BM-HDD/Chien/GF 곱셈) 의존, 광통신 AWGN/ISI 특화, ISI error-floor 추정 미완.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RS 성분 BM-HDD / GF(2^p) 심볼복호 | 대응 없음 | Prime ECC는 binary QC-LDPC min-sum 전용, RS/비이진 심볼복호 경로 없음 |
| SPC 열복호 MSA(min-sum) 3진 메시지 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 연산 자체는 유사하나 SPC(degree 낮은 단일 parity) 열구조라 우리 QC-LDPC CN에 직접 대응 안됨 |
| freezing 기반 조기고정/조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적 유사(부분 조기종료)이나 이미 partial-CRC 4-point 보유, sub-block 단위 freezing은 곱부호 특유 |
| 인공잡음 LUT 섭동(NACA) | 대응 없음 | Chase 유사 섭동은 우리 min-sum 복호 스케줄과 무관 |
| 곱부호 구성(RS×SPC) H-matrix | 대응 없음 | 특정 binary QC-LDPC H-matrix 고정, 곱부호 구성 미대상 |

적용 가치: 낮음 — RS 이진영상+SPC 곱부호는 비이진 RS 심볼복호(BM-HDD)에 기반한 별개 부호족이라 NAND binary QC-LDPC min-sum 디코더에 떼어 이식할 핵심요소가 없으며, freezing 개념도 이미 보유한 partial-CRC 조기종료와 유사.
