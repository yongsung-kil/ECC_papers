### B. 알고리즘 요약

1. 대상 부호: IEEE 802.16e protograph LDPC, base graph 24 VN × (6~12) CN, `(N,K)`=(288,144)/(576,288)/(576,432), rate 0.5~0.75, AWGN+BPSK.
2. 문제: 기존 model-driven DL 디코더는 학습 파라미터 수가 code length `n`에 비례해 길이 증가 시 학습비용·복잡도가 급증.
3. 핵심 아이디어: Tanner graph 전체가 아니라 base matrix(고정 크기)만 학습 → 파라미터가 `n`이 아닌 base graph 크기에 종속.
4. 디코더: Offset Min-Sum을 unroll한 neural OMS. C2V `u = ReLU(min|u'|-β)·∏sign(u')` (식7), V2C `u=w·l+Σu'` (식8), 출력 `s=w·l+Σu` (식9).
5. 학습 대상: iteration별 offset `β_c,v^t`, weight `w_v,c^t`를 base graph edge에만 부여, 파라미터 공유 불필요.
6. bit decoding allocation: 학습된 `z0`개 sub-decoder를 병렬 연결, codeword를 `z0` 간격으로 분할 삽입 `BDi = {v_i, v_{i+z0}, ...}` (식10~12), `z0`=lifting/expansion factor(예 rate0.5 n/24).
7. 학습 설정: Gaussian approximation, training SNR 0~5 dB, learning rate 0.001, TensorFlow, `z0`=24.
8. 메모리: 학습 파라미터를 32bit float(또는 NOMS류 c=3bit 양자화)로 저장, 제안법은 `Σ 32·24·dv`로 code length 무관하게 일정(Table III).
9. 결과: SNNMS 대비 BER 최대 0.1 dB 우위, 단 NNMS/LLR BP보다는 열위, iteration 15(hidden 30층)에서 수렴 포화.
10. 한계: HW 미설계, 시뮬만, NN 학습 필수, 정정능력 자체 개선은 아니고 학습비용 절감이 핵심, 무선 표준 부호 대상.
