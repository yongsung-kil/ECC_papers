### B. 알고리즘 요약

1. 시스템: 2-user Gaussian MAC (`y = h1x1 + h2x2 + z`), 두 사용자가 같은 codeword 길이 `L`, 서로 다른 code rate(`R1≥R2`)로 송신, CSI는 송수신 모두 알고 있음.
2. 문제: 기존 NCMA/CFMA의 XOR-CD 달성율이 나쁜 채널에 묶여(`I(C;Y)≤min(C1,C2)`) 강한 사용자가 우수 채널을 못 살림.
3. 부호설계 RE법: 강한 사용자 PCM `H1` 아래에 cycle-4 없는 `He`를 덧붙여 약한 사용자 저rate `H2` 생성 (`C1⊇C2`).
4. 부호설계 RC법: `H2`의 일부 행을 `A`로 결합(`Hr=A·Hr`)해 고rate `H1` 구성, 두 부호가 공통 PCM `Hc` 공유.
5. 핵심식 `c1,2·Hc^T = 0`: joint codeword를 단일 디코더로 복호 가능하게 함, 잔여행렬 `Ha`로 `c2·Ha^T=0` 추가 검증.
6. 복호 RDJD: JUD가 `Hc`로 joint 심볼 `c1,2(i)∈{0,1}^2` 4-조합을 SPA로 동시복호, RUD가 `Ha`로 약한 사용자 `c2`를 별도복호해 extrinsic 교환.
7. 복호는 F2^2 상 nonbinary SPA와 등가, 4PAM/Z4 확장 시 G-SPA 사용, 복잡도 `O(q^2)`로 사용자수 지수 증가.
8. 최적화: EXIT chart로 JUD/RUD 수렴 tunnel 분석, degree-distribution·rate 배분 최적화, BER 예측식 (30).
9. 결과: 반복 RDJD가 NCMA/CFMA 대비 최대 1.0dB gain, imperfect CSI에서도 0.2dB 열화만 발생 (BER 10^-4~10^-5, 시뮬).
10. 한계: HW 미설계, 무선 2-user MAC 전용, nonbinary/다중접속 결합복호로 NAND 단일 binary LDPC와 구조적 무관.
