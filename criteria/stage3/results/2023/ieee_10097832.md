### B. 알고리즘 요약

1. 시스템: protograph-coded reference-reused DCSK (예: HE-DCSK), rate-0.5 AR4JA protograph 부호, codeword length `N=2560`, spreading factor `M=100`, BP 디코더.
2. 문제: reference-reused DCSK에서 전통 수신기는 "오염된" chaotic reference를 그대로 검출에 써 여러 information-bearing 신호가 주는 정보를 낭비, LLR 부정확.
3. 채널/모델: multipath Rayleigh fading + AWGN, ISI 무시(다중경로 지연 < chaotic 신호 지속), logistic map `x(j+1)=1-2x^2(j)` chaotic 생성.
4. 핵심 기법: WRE(weighted reference estimation) - reference `Ref_y`(weight 1)와 정보신호 `Msg_x2`(weight `LP(i)/Avg`)를 신뢰도 가중합으로 reference 추정 (식 4), `Avg=Σ|LP(i)|/N`.
5. 의미: 신뢰도 높은 fragment가 reference 추정을 지배해 후속 LLR 정확도를 높임; hard decision 잡음을 가중으로 완화.
6. 확장: iterative decoding(ID) - BP 디코더의 사후 LLR `LP(i)`(외적정보)를 WRE에 되먹여 reference 재추정 → LLR 재계산 반복 (Algorithm 1).
7. LLR 유도: WRE-ID HE-DCSK에서 `Zi` 평균·분산 재유도(식 7,8), Gaussian PDF(식 9)로 `LA(i)=2Zi·(EZ/varZ)` (식 10).
8. 분석: P-EXIT로 decoding threshold `ε` 산출; multipath Rayleigh에서 it=1에 전통 대비 0.75 dB threshold gain, 추정분산 `VE`(식 12)로 정확도 개선 확인.
9. 결과: BER=1e-5에서 AWGN 0.35 dB, multipath Rayleigh 0.8 dB(it=1) gain; it 1→4에서 소폭 추가(0.06~0.09 dB), it 4에서 수렴; SR/MC-DCSK로도 확장 gain.
10. 한계: HW 미설계, 시뮬만; gain 원천이 DCSK 특유 chaotic reference 재사용 구조라 일반 binary LDPC로 분리 불가; LDPC 디코더는 표준 BP 그대로.

### C. Prime ECC 관련 모듈 핀포인트

대상이 other(변조단 수신기/reference 추정)이며 부호설계·복호 알고리즘 기여가 없어 C섹션은 전부 N/A. 참고: 사용된 LDPC 디코더는 표준 BP로 Prime ECC의 min-sum HW 엔진과 대응하지 않음.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| WRE reference 추정 / DCSK 복조 | 대응 없음 | chaotic 변조 특유, ECC 코어와 무관 |
| BP 디코더 | 대응 없음 | 표준 BP(full-BP), Prime ECC는 min-sum이며 신규 기여 아님 |
| a priori LLR 계산 (변조 채널) | 대응 없음 | NAND soft-read LLR과 다른 chaos 채널 모델 |

적용 가치: 낮음 - 무선 chaos 통신 수신기 논문으로 LDPC는 표준 BP를 재사용할 뿐이며, 신규성인 WRE reference 추정은 NAND binary QC-LDPC ECC에 이식할 접점이 없음.
