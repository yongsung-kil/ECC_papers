> 총평: Gilbert-Elliott형 버스트+독립오류 채널을 겨냥한 짧은 block-permutation LDPC의 hard-decision 리스트 복호 연구로, NAND용 고rate soft min-sum QC-LDPC 엔진(Prime ECC)과는 채널모델·복호원리가 달라 이식가치 낮음.

### B. 알고리즘 요약

1. 대상은 짧은 binary LDPC(block-permutation `H`, PEG, 랜덤; 길이 44~100, rate≈0.3~0.5)로, 통신·저장 시스템에서 버스트와 독립오류를 함께 정정하는 문제.
2. 문제: 채널이 memory를 가져 good/bad 상태로 버스트가 생기는데, 기존 LDPC 복호는 독립오류 위주라 긴 버스트에서 성능 급락, 버스트+독립 동시정정 연구는 희소.
3. 모델: 길이 n 코드워드에 길이 b 단일 버스트(시작위치 균일분포)와 버스트 밖 t개 독립오류; 정정능력을 쌍 `(b_max, t_max)`로 정의(예: Code A (44,13), d0=10, t_max=4, b_max=10).
4. 핵심 1단(Alg.1): 윈도우 슬라이딩 단일버스트 복호 — 각 시작위치 i에서 부분행렬 `H_b,i`와 syndrome `S=yH^T`로 `decode_burst`([3]) 수행, 성공 시 반환.
5. 핵심 식: `e A^T = S`의 해를 윈도우 내에서 구해 `â=y+e`; 윈도우가 버스트보다 크면 정정능력 초과분도 일부 정정.
6. 핵심 2단(Alg.2): Chase형 리스트 확장 — 윈도우 밖 독립오류를 covering vector 집합 L로 덮어 각 후보를 단일버스트 복호에 투입, 유일 성공 시만 채택.
7. 리스트 크기 `L=Σ C(nG,i)`(nG=n-b)로 t 증가 시 급증 → majority-logic 원리로 축소.
8. 축소 휴리스틱: `f=yH^T`(십진 계산)를 신뢰도 지표로 보고 f 최대값 위치들만 covering에 사용(랜덤 대비 소폭 개선, 리스트 축소 시 오류확률 급증).
9. 결과: 버스트 밖 독립오류가 1개만 생겨도 정정가능 버스트 길이가 크게 감소; 고정 길이·rate에서 3-stripe(γ=3) 구성이 동시정정에 유리(단일버스트만은 γ=2 유리).
10. 한계: HW 미설계·시뮬만, hard-decision·짧은 부호 전제, 버스트 위치 미지 시 리스트 폭증(복잡도·latency 악화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 윈도우 버스트 복호 | 대응 없음 | Prime ECC는 iterative soft min-sum, syndrome-list 복호기 없음 |
| Chase형 리스트 복호 | 대응 없음 | 리스트/covering vector 개념 없음 |
| 벡터 f 신뢰도 휴리스틱 | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` | HD 신뢰도 착안점만 미미하게 유사, 구조 상이 |
| block-permutation 부호구조 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | QC-LDPC와 형태 유사하나 버스트 특화 소블록 구성이라 재설계 부담 |

적용 가치: **낮음** — 버스트 채널·hard-decision 리스트 복호·짧은 부호가 NAND용 soft min-sum 고rate QC-LDPC와 부합하지 않는다.
