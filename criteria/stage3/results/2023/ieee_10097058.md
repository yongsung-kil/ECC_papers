### B. 알고리즘 요약

1. 대상은 LTE Turbo 부호(두 RSC + 인터리버, rate 1/3, `G0=15,G1=13` octal)이며, 목표는 4G Turbo와 5G LDPC를 함께 처리하는 unified BP 디코더.
2. 문제: Turbo 부호의 PCM(`Hturbo`, 2K×3K)은 희소하지만 girth-4(4-cycle) 다수라 BP 복호 성능이 BCJR 대비 매우 나쁨(K=6144에서 FER 10⁻³ 기준 6.5 dB gap).
3. 부호 스킴이 고정되면 PCM 최적화는 construction이 아니라 고정 제약하 희소행렬 extraction 문제로 정식화.
4. 기법1 - 다항식 기반 MWPC: `Q(D)`를 곱해(`G1'=Q·G1, G0'=Q·G0`) parity 제약 불변으로 최소 weight parity 행 생성, greedy로 `Q(D)` 탐색.
5. `Q(D)=D²+D+1` 적용 시 4-cycle 6개 제거·6-cycle을 `2K-15`로 감소, K=400에서 FER 10⁻³ 기준 1.5 dB 이상 개선.
6. 기법2 - 다항식 기반 G-PC: 보조 VN(입력 LLR=0)을 도입하는 `A(D)=U·K(D)+Z·J(D)`로 parity식을 2개로 분할, 4-cycle 제거.
7. 두 기법 결합(G-MWPC): `{Q,K,J,L}(D)` 다항식 세트로 4/6-cycle 모두 제거하며 규칙적 구조 유지, 복잡도가 부호 길이와 무관.
8. cycle 계수는 Hashimoto edge 행렬 `E`의 trace(`tr(E^g')/2g'`)로 계산. 6개 후보(CASE-I~VI) 중 CASE-IV가 8/10-cycle 최소로 최선.
9. 결과: CASE-IV G-MWPC가 원본 BP 대비 FER 10⁻³에서 2 dB 이득·edge 수 40% 감소, 개선 BP가 원본 BP↔BCJR gap을 절반으로 축소(I=160에서 추가 0.5 dB), BP-list/BP-flip으로 BCJR 대비 0.4 dB 이내까지 접근.
10. 한계: 여전히 BCJR 대비 gap 존재, HW 미설계(향후 과제), soft/양자화·NAND read 무관, 대상이 Turbo 부호라 binary QC-LDPC 부호설계와 직접 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP(sum-product tanh) 메시지 갱신 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal() | Prime ECC는 min-sum 근사 사용, 논문은 full-tanh SP → 직접 대응 없음 |
| PCM cycle 제거·희소화(MWPC/G-PC) | ecc_top.cpp Load_PCM() | H-matrix 로드 지점이나 Turbo 부호 대상이라 QC-LDPC 구조와 불일치 |
| Turbo 트렐리스/RSC 부호구조 | 대응 없음 | Prime ECC는 binary QC-LDPC 전용 |
| BP-list / BP-flip 재시도 | 대응 없음 | Prime ECC의 조기종료(partial/full CRC)와 성격 다름 |
```

적용 가치: **낮음** — Turbo 부호를 BP로 복호하기 위한 PCM cycle 제거·희소화 기법으로, Prime ECC의 binary QC-LDPC 부호구조·min-sum 디코더와 정합하지 않으며 4-cycle 제거 아이디어 자체는 QC-LDPC 구성법에 이미 표준(girth 최적화).
