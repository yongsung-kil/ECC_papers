### B. 알고리즘 요약 (10줄 내외)

1. 시스템: IEEE 802.16e QC-LDPC(irregular, base matrix 24열) 기반, data bit block `8100`, code length `16200`(rate 1/2), rate node `1/2, 2/3, 3/4`.
2. 문제: 채널 상태에 맞춰 code rate를 유연히 바꾸는 rate-compatible LDPC(RC-LDPC)가 필요하나 기존 puncturing/shortening/padding은 인코더·디코더를 동시 수용 못 함.
3. 채널/모델: Gaussian white channel + 100Gbit/s QPSK 코히런트 광통신(1000km SMF, EDFA), MATLAB 오프라인 DSP.
4. 핵심 기법: **다중 검사행렬 기반 계단식(stepped) puncturing** — 목표 rate에 맞는 rate node 검사행렬을 먼저 선택 후 펑처링.
5. 펑처링 규칙: 유효 row weight 계산·정렬 후, row weight가 작은 행을 우선 펑처, 동일하면 랜덤 선택, 목표 rate 미달 시 최대 weight 행 중 최소 weight 행 선택.
6. 결과 rate: rate node 1/2 → 펑처 후 `3/5`, 2/3 → `7/10`, 3/4 → `4/5`(1/2에서는 802.16e로 4/5 도달 불가).
7. 부수 특성: 펑처 진행에 따라 parity check bit 갱신, 목표 부호율·프레임 구조 확보, 인코더/디코더 동시 수용 유연성.
8. 검증: MATLAB PRBS, frame 40, message bit 8100, BER 곡선 및 constellation(OSNR 10dB) 비교.
9. 결과: AWGN에서 랜덤 펑처 대비 HD-FEC threshold(3.8E-3)에서 `0.36~0.47dB` gain, 1000km 전송(rate 3/5, 50Gbaud)에서 `0.49dB` gain.
10. 한계: HW 미설계·시뮬만, 광통신/AWGN 특화, IEEE 802.16e 부호 의존, gate count/throughput 없음.
