### B. 알고리즘 요약

1. 대상: 3-D TLC CT NAND(BiCS), hot-data storage; LDPC ECC를 hard decision(min-sum)으로 사용, codeword user data 1kB.
2. 문제: 정적 ECC는 초기 수명엔 정정능력 낭비·후기엔 부족, dynamic LDPC는 정확한 수명 예측 필요하나 기존 ML 예측은 온칩 학습·대량 데이터·다수 파라미터로 오버헤드 큼.
3. 관찰: FPGA 테스트 플랫폼(NplusT)으로 측정한 FBC가 초기 read/P-E 단계와 중후기 단계 사이에 근사 선형 관계(인접 gear일수록 선형성 강함).
4. RD 예측: `FBCr_k = S(k)*FBCr_100 + I(k)`, S(k)·I(k)는 read time k의 이중 지수함수 `sum ai*exp(k*bi)` (long-term 식4, short-term 식5).
5. Endurance 예측: 동형으로 `FBCp_n = H(n)*FBCp_10 + P(n)`, H(n)·P(n)도 이중 지수 (식9 long, 식10 short).
6. short-term 모델이 long-term보다 정확: 샘플의 99.16%(RD)·97.63%(endurance)에서 정확도 손실 10% 이하.
7. STWS = EBPM(주기적 FBC 예측) + SLDPC(예측 FBC를 4개 사전저장 LDPC code의 정정한계 C와 비교해 다음 P/E 구간 code rate 선택).
8. 사용 code rate 4종: 0.9499/0.8935/0.8380/0.7823, max iteration 30, hard decision LLR은 경험값 -7/+7 할당.
9. 결과: 정적 code3(rate 0.838) 대비 초기 user data 2kB 절약(page당 17 vs 15 codeword), 후기 endurance 약 60% 연장, 예측모델 구축에 ~5 block/0.39s(ML은 1200 block/3.77s).
10. 한계: HW 미설계, 디코더/부호구조 변경 없음, 예측 파라미터는 chip type·환경별 재보정 필요, rate 4종 자체 설계법 미서술.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FBC 기반 수명 예측·dynamic rate 선택 | 대응 없음 | 컨트롤러 신뢰성 정책이라 코드베이스 이터레이션/부호 모듈과 대응 없음 |
| rate 다른 LDPC code 다수 로드/선택 | ecc_top.cpp Load_PCM() | H-matrix 로드 지점과 개념적 접점뿐(논문은 rate별 code 설계·구조 미서술) |
| hard decision(min-sum) 디코딩 | decoder.cpp LDPC_Decoder() | 논문은 디코더를 블랙박스로 사용, 내부 변경 없음 |
```

마지막으로 적용 가치: **낮음** — 예측 모델은 특정 chip 특성화 산물이라 이식 불가하고, dynamic rate 선택은 컨트롤러 정책이지 디코더/부호 알고리즘이 아니어서 Prime ECC 대상이 아니다.
