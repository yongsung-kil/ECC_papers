### B. 알고리즘 요약

1. 대상은 이진 선형블록부호 `C[n,k]`(예: LDPC `C1[128,64]`, eBCH `C2[128,64]/[32,16]`), BPSK+AWGN, LLR `r_i = 2y_i/σ²`.
2. 문제: 원조 OSD의 order-t 재처리는 시간복잡도 `O(k^t)`로 커, 성능손실 없이 복잡도를 줄이려 함.
3. LC-OSD는 MRI(most reliable independent) k비트가 아니라 `(k+δ)`개 most-reliable dependent 비트를 재처리(`δ≥0`).
4. `HΠ`를 Gaussian elimination해 `H=[I | P1; O | P2]` 형태로 만들고, `e=(eL,eR)`로 분해(`eL` 길이 `n-k-δ`, `eR` 길이 `k+δ`).
5. 제약식 `P2 eR^T = P2 zR^T`, `eL^T = zL^T + P1 zR^T + P1 eR^T` 로 `eR`만 정하면 `eL`·`e`가 유일 결정.
6. soft-weight `Γ(e)=⟨e,|r|⟩=Γ(eL)+Γ(eR)`; SLVA로 `P2` trellis 위에서 `Γ(eR)` 비감소 순으로 test error pattern 탐색.
7. Trivial 기준: `Γ(e_opt^(j)) ≤ Γ(eR^(j))`이면 ML임을 보장하며 종료(Prop.1,2).
8. Ideal 기준: `Γ(e_opt^(j)) ≤ Γ(eL_true)+Γ(eR^(j))` (실사용 불가) → `Γ(eL)`을 기대값으로 대체.
9. DAI: `E{Γ(eL)|r}=Σ (1+exp(|r_i|))^{-1}|r_i|` 사용; SAI: 이를 적분근사한 상수 `τ`로 `Γ(e_opt)≤τ+Γ(eR^(j))` 종료.
10. 결과: 원조 OSD 대비 평균 탐색수·복호시간 대폭 감소, 성능손실 미미(SNR↑일수록 복잡도↓). HW 미설계, n=128급 short code 시뮬만.
