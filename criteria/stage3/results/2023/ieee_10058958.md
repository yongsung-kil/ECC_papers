### B. 알고리즘 요약

1. 대상 부호: BPSK, rate `r=0.75`, length `N=576` LDPC (문헌 [17]과 동일 부호), iteration `T=5`(12-layer unfold NNMS 망).
2. 문제: 신경망 min-sum 디코더(NNMS+)는 학습 weight `A`를 edge별 도입해 성능은 좋으나 파라미터·곱셈 저장/연산 복잡도가 급증.
3. 기반 알고리즘: CN→VN 메시지 `Uc,v = sign(...)·min|A×U|`로 min-sum에 학습 weight 행렬 `A∈R^3` 곱을 추가한 NNMS+.
4. 핵심 기법 1: Tensor-Ring(TR) 분해 - weight 행렬 `A`와 메시지 행렬 `U`를 모두 4차 텐서화 후 3차원 코어 곱(`G1·G2·G3`, `I1·I2·I3`)으로 근사.
5. TR 의미: 공간복잡도 `O(dnr^2)`, 순환 차원 치환 불변성으로 TT 대비 rank 선택이 유연 → 최적 permutation 탐색 불필요.
6. 핵심 기법 2: weight sharing 2종 - temporal(iteration 간 weight 공유, 복잡도 `1/T`)과 spatial(같은 layer edge 간 공유, layer당 weight 1개).
7. Joint-way 압축: weight sharing 후 TR 분해(또는 역순)를 결합해 8가지 조합 구성; `TRwm-ssNNMS+`가 최고 압축률.
8. 근사 오차: 메시지 행렬 TR 분해는 Tanner graph를 변형; Frobenius norm `‖A−B‖F ≤ ε‖B‖F`로 오차 관리, rank 클수록 성능 저하.
9. 학습: cross-entropy loss (식 23), gradient descent, TensorFlow 2.0, batch=120.
10. 결과: `TRwm-ssNNMS+`는 NNMS+ 대비 파라미터 40,800배·곱셈 13,493배 축소, TT-NNMS+ 대비 파라미터 71배·곱셈 약 6배 축소, BER 성능 저하 거의 없음.
11. 한계: HW 미설계, 시뮬만; 학습 weight 도입한 NN 디코더가 전제라 비학습 min-sum HW에는 직접 적용 곤란; gate count/throughput/공정 정보 없음.
