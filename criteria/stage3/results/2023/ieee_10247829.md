### B. 알고리즘 요약

1. 대상: NN 기반 채널 디코더 Neural Belief Propagation(NBP)을 SRAM 디지털 in-memory computing(IMC)으로 가속하는 HW 플랫폼. code length 121/672/1056/2048 예시.
2. 문제: NBP는 BP를 unfold해 scaling parameter를 학습(정정성능↑)하지만 storage·연산 복잡도가 커 실시간/저전력 통신 HW에 부담.
3. 핵심 관찰: NBP 가중치 행렬 `W1,W2,W4`가 특수 sparsity 패턴(`W1`은 행당 non-zero 1개 → sparsity `(N-1)/N`)을 가져 이를 HW로 활용 가능.
4. 핵심 기법1: 5-step NBP dataflow를 IMC로 매핑 — Step1 structured sparse MVM(`ktv→c=W1 lv`), Step2 unstructured sparse MVM+accumulate(`qtv→c=W2 u`), Step3&4 min-sum+dot-product, Step5 soft output `stv=lv+W4 u`.
5. 핵심 회로: 6+2T(8T) compute bit-cell — 6T SRAM에 T1/T2 추가로 셀 내부 1-bit AND(dot-product) 수행, weight bit×IWL 입력.
6. 핵심 기법2: bit-serial multi-bit MVM — 행렬 transpose 저장 + IWL 공유로 열 병렬 dot-product, shift-accumulator로 multi-bit 정합.
7. 압축: GWCL(greedy weight compression & localization) — zero weight 제거 후 column index 저장, structured(`W1`)는 누산 불필요; EGWCL로 unstructured(`W2/W4`)는 row index 저장해 누산 대상 식별.
8. 구현 디테일: overflow detector, Mod-Counter, comparator로 흩어진 weight 누산; W3는 대형이라 register file 버퍼 + off-chip fetch.
9. 결과: TSMC 28nm 합성 기준 code length 1056에서 throughput 224Gb/s@4it, energy efficiency 1374.2Gb/s/W, power 0.163W로 기존 LDPC HW 대비 에너지효율 우위; SSP-MVM 29.56 TOPs/W.
10. 한계: 합성(post-synthesis)까지만이고 실측(post-silicon) 아님, BER/정정성능 곡선 수치 미제시, NAND 무관, NN 디코더 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Neural BP 디코더 전체(unfolded NN, 학습 가중치) | 대응 없음 | 프로파일 §6에서 NN/딥러닝 디코더는 명시적 "대응 없음" |
| min-sum 연산 (Step 3&4) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 보유(중복 기여) — 단 학습 weight 곱은 별개 |
| SRAM in-memory computing / sparse MVM HW | 대응 없음 | Prime ECC는 z=32 pipeline RTL 시뮬레이터, IMC 아키텍처와 HW 모델 상이 |
| GWCL/EGWCL 가중치 압축·라우팅 | 대응 없음 | NBP weight 전용 압축, QC-LDPC circulant 구조와 무관 |

적용 가치: **낮음** — 본 논문은 NN(neural-BP) 디코더를 위한 SRAM in-memory computing HW로, Prime ECC의 고전 min-sum binary QC-LDPC 알고리즘 및 z=32 pipeline HW 모델과 접점이 min-sum 명칭 외엔 없고 NAND 관련성도 낮다.
