### B. 알고리즘 요약

1. 시스템: 256GB 3D TLC NAND SSD(48층, 4채널×2칩), LDPC를 기본 ECC로 사용, read latency `T_READ=(Tsense+Ttransfer+Tdecode)*(N_RR+1)` (90/16/20 µs).
2. 문제: 고밀도 3D NAND에서 read-retry 횟수 `N_RR`가 커져(최악 25+회) read latency 급증, 기존 병렬read/SOML은 full-page read엔 bitline 제약으로 무력.
3. 모델: RBER를 `RBER=RBER_cycle(PE)+RBER_retention(RT)+RBER_readdisturb(RC)+RBER_variation(LC)` 오프라인 프로파일로 예측, Arrhenius로 온도 보정.
4. 핵심기법1(PRS): 들어온 read의 예측 RBER가 ECC 정정능력 초과 시 RRQ의 victim read와 block-decoder 제약 확인 후 병렬 read 노드 구성, 1-bit `PRmethod`로 PRR/APR 선택.
5. 핵심기법2(PRR): full-page read를 subread로 분할→bitline 상보 subread를 SOML read로 병렬, 성공 subread의 RRV를 저장·재사용해 동일 페이지 나머지 subread의 retry를 0으로.
6. 핵심기법3(APR): 동일 retry 특성 페이지들에 서로 다른 aggressive RRV(V_REF^i)를 병렬 인가하여 optimal RRV 탐색에 필요한 retry 횟수 자체를 감소.
7. 부수: 저장한 RRV는 DRAM/cap-backed RAM에 임시 보관(오버헤드 무시), power-off 시 OOB 메타로 복구, NAND 칩 변경 불필요(펌웨어 구현).
8. 최적화: 없음(모델 기반 예측+스케줄링 룰).
9. 결과: SSDSim 12+ 트레이스에서 PaRR가 baseline 대비 평균 I/O response 33.77% 단축(최대 81.81%), read-response 35.58%, aged SSD(case4)에서 최대 43.15%.
10. 한계: LDPC/ECC 디코더 자체 개선 없음, 성능은 시뮬레이터(SSDSim) 기반, high-density MLC/TLC에서 APR 효율 제한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SOML read/read-retry 병렬 스케줄링(PRR/APR) | 대응 없음 | FTL 펌웨어 레벨 스케줄링, Prime ECC 디코더/인코더와 무관 |
| RBER 예측모델·RRV 조정 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()` / `Set_LLR_Th()` | 채널/RBER 계층에 개념적 인접, 그러나 read-retry는 soft-read LLR 설계와 별개 |
| LDPC ECC 엔진(배경 언급) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | LDPC를 배경 ECC로만 언급, 디코더 기여 없음 |

적용 가치: **낮음** — NAND read 성능 시스템 최적화(FTL read-retry 병렬화)로 LDPC 부호설계→디코더→HW 어느 레이어에도 이식할 알고리즘 요소가 없음.
