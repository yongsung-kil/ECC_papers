### B. 알고리즘 요약

1. 시스템/부호: deep-space optical(Earth-to-Mars), rate-adaptive staircase LDPC(component code는 systematic LDPC, block length `N=128`, check `r=64`), 4-PPM 변조.
2. 문제: staircase의 HDD-SWD는 stall pattern으로 error floor, 기존 rate-adaptive staircase LDPC에 적합한 SDD가 부재.
3. 채널/모델: 태양 scintillation index `σI²`로 채널을 3상태로 구분 — `σI²<0.01` AWGN, `0.01<σI²<0.99` Gamma-Gamma 난류(식(2)), `σI²>0.99` outage(식(1)(3)).
4. 핵심 기법 1: SWD 구조를 rate-adaptive staircase에 맞게 변형 — 재부호화 행렬이 block보다 작아 target block을 첫 block이 아닌 둘째 block으로 이동(window size `w=4`).
5. 핵심 식: 세 종류 외재정보 교환(식(8)(9)(10)) — window 이동 시 직전 window 복제 block, 같은 outer iteration 내 인접 group, outer iteration 간 초기화로 a priori 값 갱신.
6. 핵심 기법 2: 내부 디코더로 SOSD(OSD의 soft-output 변형) 사용 — reliability 정렬 + Gaussian elimination + 재부호화로 codeword 추정하고 비트별 soft 정보 출력, order `Δ`.
7. 구현 디테일: 내부 iteration `Imax`, 외부 iteration `Jmax`, 스케일링 `μ1,μ2`(Algorithm 1), Memory A/B로 재부호화·a priori 값 저장, staircase 폭만큼 인코더/SOSD 병렬 배치.
8. 학습/최적화: 없음(학습 기반 아님), 파라미터(`m,Δ,Jmax`) 튜닝으로 BER-복잡도 trade-off 조정.
9. 결과: SOSD-SWD가 BP-SWD보다 낮은 BER(특히 높은 `σI²`), 세 채널 상태에서 파라미터 조정으로 BER 1e-4 달성, `Δ` 증가 시 성능↑·복잡도 급증.
10. 한계: HW 미설계·시뮬만, deep-space optical GG 채널 전용, gate count/throughput/rate 명시 없음.
