## B. 알고리즘 요약

1. 시스템: DNA 저장 outer code. 데이터행렬 `U∈F2^{k×w}`를 각 열마다 (n,k) 선형블록 ECC(시뮬은 (1296,1080) LDPC, IEEE 802.11n)로 부호화, 각 string(행)에 `a=log2(n)`bit 주소 부착.
2. 문제: 각 열을 독립 복호(independent decoding)하면 모든 열 복호 성공을 요구해 성능이 부족.
3. 모델: string 단위 채널 - 정확전송 `pc`, erasure `pe`, random substitution `ps`, 이후 무작위 shuffle. 전이확률식 (1), `pc > ps/(2^l-1)` 요구.
4. 핵심 기법1: 같은 주소 `i`를 가진 t개 행 중 j-bit=0인 t0로부터 bit `vi,j`의 soft info `mi,j` 유도(Proposition 1, 식(8)), hard info는 다수결(Corollary 1, 식(12)).
5. 핵심 식: `mi,j = P(vi,j=0|t,t0)/P(vi,j=1|t,t0)` - string 신뢰도를 열 복호 근거로 삼기 위한 LLR 유도.
6. 핵심 기법2: joint decoding(Algorithm 1) - 독립복호 결과 `Ṽ`와 각 행 `zi`의 Hamming 거리 `di`로 신뢰도 정렬, 가장 신뢰도 높은 행들을 correct로 나머지를 erasure로 보고 재복호.
7. 구현 디테일: 재복호는 erasure 채널상 선형시스템 해법 = SGE(structured Gaussian elimination = inactivation decoding), sparse H(LDPC)에 효율적.
8. 복잡도: 전체 `O(wθtmax + wn + n log2 n)`, `O(wθtmax)`(독립 열 복호)가 지배적.
9. 결과: (1296,1080) LDPC에서 joint decoding이 independent 대비 FER 3자리수 이상 감소. erasure error가 substitution보다 덜 해로움, LT코드 BFA[4] 대비 비교.
10. 한계: HW 미설계, 시뮬만, DNA 저장 채널(erasure/substitution/shuffle+주소) 전용 모델이라 채널 종속.

| BP 열 독립복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 내부 BP는 표준 재사용이나 Prime은 min-sum, 논문의 기여는 BP 밖(신뢰도 정렬+SGE) |

적용 가치: **낮음** — 핵심 기여(신뢰도 기반 string 선택 + SGE 재복호)가 DNA 저장의 erasure/substitution/shuffle 채널과 outer-code 구조에 특화되어, 단일 codeword를 iterative min-sum으로 복호하는 NAND binary QC-LDPC 복호기에는 이식 대상이 사실상 없다.
