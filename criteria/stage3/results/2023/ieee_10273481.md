### B. 알고리즘 요약

1. 대상: `nc × nv` protograph를 lifting factor `N`으로 확장한 regular/irregular **QC-LDPC** (block length `nvN`, 다항식 PCM `H=(pij(x))`).
2. 문제: 기존 PEG는 large girth 확보를 위해 symbol node에서 subgraph 확장 + 최원거리 CN 탐색이 필요해 복잡도 `O(mn)`.
3. 핵심 개념: 각 exponent `a`에 대해 **forbidden set** `F_{a}^{g,N}` 정의 — `a∈F`이면 `girth(H)≤g`가 되는 exponent 집합(girth 초과를 막는 QC 제약).
4. forbidden set 구성: girth>4/>6 조건을 msets(예 `{il-jl}`)의 원소가 서로 distinct해야 한다는 필요충분조건으로 환원해 계산.
5. Algorithm 1 (forward selection): column-by-column으로 `g∈{4,6,8,10}` 중 `F=[N]\FORBIDDEN(...)≠∅`인 최대 girth를 골라 `RANDOM(F)` 배정, 막히면 target girth를 2씩 감소.
6. Algorithm 2 (fixed girth): target girth 고정, `F=∅`이면 실패 선언 후 재시도(예: N=73, girth 12 구성).
7. 부수 트릭 (multiples of rows): 한 행의 exponent를 다른 행의 배수로 선택 → 고정 girth에서 실패 시도 횟수 대폭 감소(Table II).
8. Algorithm 3 (prioritization): forbidden set 크기가 `[N]` 미만이면서 가장 큰 exponent에 우선순위 부여 → short cycle 수 감소, large girth 확률 증가 (CCSDS protograph 예제).
9. 결과: N=64 CCSDS 예제에서 6-cycle 수 128/320/384로 기존 [1] 대비 감소; 복잡도 `O(nc·nv^2·N·logN)`(girth>4)로 저복잡도.
10. 한계: girth·cycle 수만 지표, BER/정정능력/HW 없음. rate/degree/length가 예제마다 다르고 복호기 연동 결과 부재.
