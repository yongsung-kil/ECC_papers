### B. 알고리즘 요약

1. AWGN·BPSK 채널에서 길이 `n=64` 단·중길이 블록부호(`[64,45]` BCH, `[64,45]`/`[64,21]` CA-Polar) 다중부호를 하나의 디코더로 통합 복호하는 문제 설정.
2. 기존 문제: 부호마다 별도 디코더를 두면 수신기 복잡도가 크고, 전체 codebook 학습형 NN은 "차원의 저주"로 짧은 부호에만 적용 가능.
3. 채널 모델: `y = x + z`, `z ~ N(0,σ²)`; 디코더 입력은 LLR `γ_i = 2y_i/σ²`.
4. 핵심 기법(syndrome-based, Bennatan et al. [8]): NN 입력을 `[|γ|, s̃]`로 정의해 전송 codeword를 채널 오류와 분리하고 NN `F`가 오류 `ẑ`만 추정 → overfitting 회피, codebook 시뮬 불필요.
5. soft syndrome `s̃_i = (min_{j∈M(i)}|γ_j|)·(∏sign(γ_j))`로 hard-decision 없이 syndrome 계산(Lugosch [16]).
6. 통합 확장: code type indicator `I`를 입력에 추가해 `d = [|γ|, s̃, I]` 사용(known/unknown 두 시나리오). 서로 다른 부호길이는 `n'−n`, `l'−l` 크기 zero-padding으로 입력 크기 고정.
7. NN 구조: Stacked-GRU(RNN), 층 `L=4`, timestep `T=5`, hidden size `5n=320`, 뒤에 단일 FC층으로 노이즈 추정. 복호는 `γ̂ = γ − sign(γ)⊙ẑ`.
8. 학습: BCE loss(식 7), Adam(lr `1e-3`→`1e-7`, reduce-on-plateau), `1e5` epoch, batch `2^13`, BER `5e-2~1e-4` 구간의 SNR을 균일 샘플링.
9. 결과: code indicator 사용 시 단일부호 학습 디코더 대비 BER `1e-3`에서 갭 <0.2 dB; indicator 사용이 미사용 대비 평균 0.15 dB 우수(Chase·SCL 고전 디코더와 대등).
10. 한계: LDPC를 명시적으로 배제(단·중길이 전용), HW 미설계, n=64 짧은부호·순수 NN 접근이라 NAND용 고전 binary QC-LDPC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome-based DNN(Stacked-GRU) 노이즈 추정기 | 대응 없음 | NN/딥러닝 디코더는 Prime ECC(min-sum HW) 구조 밖 |
| soft syndrome `s̃_i` (식 2) | 대응 없음 | min-sum CN 연산과 형태 유사하나 부호대상(BCH/Polar)·용도(NN 입력) 상이 |
| code type indicator 통합 복호 | 대응 없음 | 단일 QC-LDPC 고정 엔진과 무관 |
| BCH / CA-Polar 부호 | 대응 없음 | binary QC-LDPC 전용 코드베이스와 부호종류 불일치 |

적용 가치: **낮음** — NN 전면 대체 디코더이며 대상 부호가 BCH/Polar로 LDPC를 논문이 스스로 배제, Prime ECC의 어떤 모듈에도 매핑되지 않음.
