### B. 알고리즘 요약

1. 대상은 확률을 비트스트림 1비트 비율로 부호화하는 stochastic computing(SC) 회로이며, 응용 예로 LDPC의 belief-propagation(BP) 디코더를 든다.
2. 문제: SC 회로를 Verilog로 컴파일하는 오픈소스 도구 BitSAD.jl이 code-tracing에 쓰는 Ghost 라이브러리의 tape 자료구조가 대규모 회로에서 성능 병목.
3. 배경: BitSAD는 Julia 함수를 tape(call stack 기록)로 tracing한 뒤 tape 항목을 atomic SC 연산(AND=곱, OR 등)으로 치환해 Verilog 생성(Fig.2,3).
4. 병목 원인: Ghost의 tape가 동적 배열이라 확장 시 재할당, 중간 삽입/삭제 시 원소 이동으로 최악 선형 오버헤드 발생.
5. 핵심 기법 1: tape를 원형 doubly-linked-list로 교체해 head/tail 삽입·삭제를 상수시간화(원소 이동 제거).
6. 핵심 기법 2 - bookmarking: 일정 간격(예 1000)마다 노드 포인터를 두어 random access를 `O(interval)`로 근사(Fig.4), 인덱스 M은 bookmark 후 `M mod N` 노드만큼 이동.
7. 구현: Julia DataStructures의 MutableLinkedList를 확장, BitSAD API는 불변 유지.
8. 벤치마크: surface code(양자 오류정정) BP 디코더를 distance 3/5/10/15/20으로 컴파일(회로크기는 distance 제곱에 비례), Macbook Pro i5 2GHz/16GB.
9. 결과(Table1~3): distance 3에서 434.7s→0.642s, distance 15는 기존 불가→892.95s로 컴파일 가능, distance 3 메모리 97.815GB→약 130MB로 급감.
10. 한계: ECC 알고리즘/성능이 아닌 SC 회로 컴파일 도구 개선, distance 20은 1h timeout 초과, HW 정정성능·부호설계와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC 회로 Verilog 컴파일 tape 자료구조 최적화 | 대응 없음 | EDA 컴파일러 도구로 Prime ECC 코드/알고리즘 모듈과 무관 |
| stochastic BP LDPC 디코더 | 대응 없음 | Prime은 결정론적 min-sum HW 디코더, stochastic-computing BP는 미지향 |
| surface code (양자 오류정정) BP | 대응 없음 | NAND binary QC-LDPC 대상이 아님 |

적용 가치: **낮음** - SC 회로 EDA 컴파일러의 자료구조 최적화라 부호설계·복호·HW 어느 레이어에도 이식할 알고리즘 요소가 없음.
