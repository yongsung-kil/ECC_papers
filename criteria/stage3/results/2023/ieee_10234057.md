### B. 알고리즘 요약

1. 시스템: LEO 위성→지상국 FSO 다운링크, IM/DD OOK 변조, IR-HARQ(sliding window ARQ + RC-LDPC) 링크레벨 오류제어.
2. 문제: FSO 채널 coherence time(ms급)·Gbps 장부호에서 RS/convolutional puncturing 방식이 복잡도·고rate 성능 열화 → LDPC code extension으로 대체.
3. 채널모델: Gamma-Gamma 난류 페이딩 `ha`, 대기감쇠 `hl`, 빔퍼짐 손실 `hp`, `h=hl·ha·hp`, AWGN.
4. 핵심기법: code-extension으로 RC-LDPC 부호족 구성. 기저 PCM `HBR=[Hd Hp]`에 `i`행·열을 덧붙여 저rate `H(i)=[[HBR,0],[HIR^(i),I]]` 생성.
5. 식 의미: 확장 부호는 정보비트 `k` 고정, rate `k/(nBR+i)`로 낮아지며 상위rate 프레임에 증분 패리티만 append → nested 구조.
6. 인코딩: 계통형 `GBR=[I  Hd^T·Hp^-T]`, 내부 블록부호 `GI^(i)=[I  HIR^(i)^T]`, 프레임 `c(i)=c(0)·GI^(i)`.
7. HARQ 동작: rate족 `C1>...>CM`, 복호 실패 시 다음 버스트에 증분 패리티 전송, 성공/최대전송 M=3 도달 시 종료·폐기.
8. 복호: OOK LLR `2Pt(h^2-h·yj)/σn^2`로 iterative BP, 최대 20 iteration.
9. 결과: (1,2/3,1/2)·(1,1/2,1/3) 두 부호족, protograph 2회 lifting(factor 4·250), goodput 0.8Mb/burst 유지 시 순수 ARQ 대비 7dB SNR gain.
10. 한계: HW 미설계, 시뮬(Monte Carlo)뿐, FSO 위성 채널 특화. 복호는 표준 BP.
