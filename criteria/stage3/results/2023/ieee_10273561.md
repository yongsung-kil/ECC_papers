### B. 알고리즘 요약

1. 시스템: 무선/고throughput FEC용 SC-LDPC, (640,512) sub-block, ms=1, N=51328, R=0.798, BPSK/AWGN, NMS(scaling γ=0.75).
2. 문제: 고throughput LDPC 디코더는 block size·iteration 수가 실리콘 제약에 걸림; SC-LDPC + sliding window로 소블록 병렬 처리하되 전력이 여전히 병목.
3. 모델: SC-LDPC의 대각지배 PCM `H[1,L]`, layer = ms+1 submatrix 행, window를 layer별로 이동하며 P processor가 각 1 iteration만 수행(processor 수 = iteration 수 역할).
4. 핵심 기법1(적응 윈도우): 각 layer의 partial syndrome `zl` 검사, 현 window 첫 layer syndrome 실패 시 processor 1개 추가로 window 폭을 `ms+1`~`Pmax·(ms+1)`까지 monotonic 확장(Alg.1).
5. 식(제약): syndrome `zl=0`이라도 선행 layer 모두 `zi=0`일 때만 처리 제외 가능(coupling 의존성), 안 그러면 조기 error floor.
6. 핵심 기법2(HW): Pmax processor 파이프라인, enable 신호로 필요한 processor만 on-demand 활성화, 미사용 processor는 clock gating으로 비활성.
7. 구현 디테일: 각 processor는 [15]의 full-parallel VN-VN unit 1개, R-/L-APP·EXTR 메시지 파이프라인 전달, 첫/마지막 블록은 MUX 특수처리, HD bypass register로 미활성 시 우회.
8. 최적화: 성능 손실 없이 활성 processor 수만 SNR에 따라 줄임(고SNR에서 평균 2개까지 감소).
9. 결과: 22nm FD-SOI 합성 — area 3.43mm2(+8.2%), throughput 178.6Gbit/s, latency 60.9ns, 채널/EXTR 4bit·APP 6bit; 고SNR 에너지효율 최대 4배(10dB에서 73.2%) 개선, waterfall 성능은 fixed와 거의 동일, BER 10^-10까지 error floor 무.
10. 한계: 무선 SC-LDPC·layer-parallel 아키텍처 전제, fixed 대비 area·latency·throughput은 소폭 악화(절전 트레이드오프), NAND 채널 미고려.
