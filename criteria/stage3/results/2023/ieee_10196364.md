### B. 알고리즘 요약

1. 시스템: cyber-physical/mobile 통신용 non-binary(multivariate) LDPC, GF(4) 4-element 부호, length 400, (dc,dv)=(2,4), AWGN+BPSK, max iteration 20.
2. 문제: BP/EMS의 flooding·LBP 스케줄은 수렴/복잡도 trade-off가 나쁘고, trap-set에서 미수정 노드가 계속 남음.
3. 모델: multivariate LLR 벡터 `L(z)=(L[0..q-1])`로 message 표현, Tanner graph의 edge마다 non-binary 계수 `h_vc`로 message replacement 필요.
4. 기반 알고리즘(EMS): CN 갱신은 configuration set 기반 truncated max 연산 `Vc[β]=max Σ Upc[βp]` (식5), message vector를 내림차순 정렬해 상위 `nm`개만 유지해 계산 축소.
5. 핵심 식 의미: nm≪q일 때 CN 갱신 계산량이 크게 감소 - EMS의 핵심 복잡도 절감.
6. 핵심 기법(DCNS): check node의 신뢰도를 검사식 `Σ h_mn C_n = 0`(식9)과 임계 `min{u_n(C_n)}>θ`(식10)·연속 동일 판정 `C_n^i=C_n^{i-1}`(식11)로 판정, 신뢰 CN은 상태 `Scm=0`으로 두고 이후 갱신 skip.
7. trap-set 대책: 수렴했지만 여전히 이웃에 영향 주는 변수노드는 correction value α로 최적화 `U^i(β)=U^i(β)+α (β=C_n^{i-1})` (식12)하여 정답 심볼 신뢰도 강화.
8. 절차(Algorithm 1): iteration이 threshold k(=4, g>4→k>2) 도달 전엔 flooding처럼, 이후 신뢰 CN skip + 미수렴 VN을 α로 최적화. 전체 복잡도 `O(M·N)`, 정렬 `O(N^2)→O(N logN)`으로 축소.
9. 결과: 64F에서 LBP 대비 BER/FER 평균 48.3%/53.2% 감소, BP 대비 11.3%/13.2% 감소, waterfall 진입 SNR 3.9dB(flooding은 미진입). 단 256F에선 우위 소멸.
10. 한계: HW 미설계, gatecount/throughput 없음, non-binary GF(q) 전용, NAND 아닌 AWGN/CPS 채널 가정, 고차 유한체(256F)에서 이득 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EMS non-binary CN 갱신(config set, 식5) | 대응 없음 | Prime ECC는 binary Min-Sum, non-binary EMS 대응 모듈 없음 |
| 신뢰도 기반 check-node skip 스케줄링 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` 이터레이션 루프 | 스케줄 개념만 느슨히 대응(Dual-Update와 다름), 직접 이식 어려움 |
| trap-set 억제용 α 최적화(식12) | 대응 없음 | binary Min-Sum용 post-processing/error-floor 대책과 형태 상이 |
| GF(q) message replacement `h_vc` | 대응 없음 | binary 전용이라 non-binary edge 계수 처리 없음 |

적용 가치: 낮음 - non-binary EMS + AWGN/CPS 타깃이라 프로파일 §5 필터상 binary QC-LDPC 코드베이스에 이식성이 낮고, 신뢰도 기반 스케줄 아이디어만 개념적 참고 수준이다.
