### B. 알고리즘 요약

1. 시스템: AWGN/BPSK 채널, protograph 기반 QC-LDPC(lifting factor `Q=Q1·Q2`, `Q2=128`), `n=30720`, rate `2/5~9/10`. 타깃은 LEO 광위성 다운링크(≥10 Gbit/s, BER 1e-12).
2. 문제: 5G/DVB-S2 부호는 고/중 BER용이라 error floor·인코더 복잡도가 광통신 요구(초저 BER, 온보드 저전력 인코더)에 부족.
3. 모델: EXIT 분석으로 iterative decoding threshold(waterfall) 예측, permanent 기반 최소거리 상한 `dmin`(식 (1))으로 error-floor 예측.
4. 핵심 기법1(고rate protograph 탐색): ARA 구조 변형. base matrix `B0=[A E 0 D; C 0 I 0]`에서 D를 double-diagonal로 강제해 linear-time·병렬 인코딩 가능.
5. 식 의미: permanent 기반 상한 `dmin(B,P)`을 최대화하면 error floor가 낮은 짧은 부호를 얻음.
6. 핵심 기법2(저rate 확장): `B=[B0 0; M I]` 형태로 행/열을 순차 추가, 각 단계에서 `dmin` 상한 side-constraint 하에 threshold `η` 최소화(`B=argmin η`).
7. 구현 트릭: 채널 LLR 6bit, APP 9bit, 디코더 메시지 7bit 양자화; offset min-sum(OMS) 디코더, layered horizontal scheduling(수렴 약 2배 개선), pipelined barrel shifter permutation.
8. 최적화: threshold 급락 방지를 위해 목표 `dmin,t` 증분 ∆을 비감소 수열로 설정(Table I).
9. 결과: 용량 대비 0.3~0.4 dB gap, BER 1e-12까지 error floor 미관측. 인코더 자원(LUT/FF)이 DVB-S2/ARJA 대비 약 10배 절감, 인코더 코어당 ≥10 Gbit/s.
10. 한계: HW는 FPGA(place&route) 검증까지, ASIC 아님. 장블록·광통신·soft-4bit+ 타깃이라 NAND(4KB, 2~3bit) 직접 이식은 부적합.
