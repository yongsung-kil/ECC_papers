### B. 알고리즘 요약

1. 시스템: YMTC 3D 64-layer TLC NAND, QC-LDPC `code length=8192`, `rate=0.9`(ACE+stop-set 구성), 페이지당 8~16 코드워드, 초기 LLR은 상태 히스토그램식 (식 1).
2. 문제: 기존 LLR 보정은 블록 단위·word line 차이 무시·타 논리페이지 정답 의존이라 수명 말기 FER 개선 실패 → 낮은 UBER라도 read latency 감소 안 됨.
3. 관찰: 동일 물리페이지 내 코드워드는 RBER 유사, TLC 임계전압은 인접 상태로만 잘 천이(교차상태 오류 희박) → 정답 코드워드로 채널을 재평가 가능.
4. 수평 최적화 HLC(Alg.1): 논리페이지 내 정답 코드워드를 pre/post XOR해 0→1/1→0 RBER 산출, 그 RBER로 오류 코드워드 LLR 재계산(word line 차 반영).
5. 수직 최적화 VLC1(Alg.2): 3 논리페이지 동일 위치가 모두 정답인 코드워드로 8 상태 천이확률 추정 → 더 정확한 LLR.
6. 수직 최적화 VLC2(Alg.3): error-mode 기반, 교차해야 할 상태 수 `n`으로 신뢰도 5등급화, 가중치 `W(n)=LLR_lp/LLR_hp`(식 5,6)로 bit별 LLR 스케일.
7. 통합 ILD(Alg.4): 복호 실패 시 수평(즉시, `KHmax=20`) → 실패분에 대해 RBER 높은 순(upper→middle→lower) 수직(`KVmax=40`), 정답비율 `ccr`≥50%면 VLC1, 미만이면 `ccr·VLC1+(1−ccr)·VLC2` 혼합; 코어는 min-sum(식 CN/VN).
8. 채널: 실칩 read(±−1.5~6V, 25mV step, 2회 read)로 임계분포 수집, MQSim 시스템 시뮬로 read latency 평가(8ch×2plane, tR=32µs).
9. 결과: FER 최대 34~36%↓(수명 말기에도 유지), 평균 iteration 최대 45.6~63.4%↓, read latency 최대 65%↓(EMAL 대비 ≥10%↓); FPGA 200MHz에서 8192 코드워드 1회 최적화 1.3µs(read 시간의 <10%).
10. 한계: 디코더 코어 무변경(초기 LLR 보정만), 순차 read 가정(random read엔 수평만), UBER는 소폭 증가 가능.
