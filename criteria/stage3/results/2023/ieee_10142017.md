### B. 알고리즘 요약

1. 대상 시스템: 고밀도 저비용 QLC(4b/cell) 3D NAND SSD, 32채널/채널당 4칩, SSDsim 시뮬레이션(read 110us, write 2ms, erase 3.5ms).
2. 문제: read/write latency gap(최대 30배)으로 인한 write-interference-read(WIR)가 read 성능을 평균 57%(최대 98%) 악화.
3. 관찰: 서버 워크로드에서 접근의 대다수가 read-mostly(>=95% read) 또는 write-mostly로 명확히 분리 가능.
4. 기법1(데이터 배치): PU(parallel unit)들을 read area(N PU)와 write area(M PU)로 물리 분할하고, read-mostly 데이터는 read area, write-mostly는 write area에 배치.
5. 기법2(마이그레이션): 임계값 `Tr`/`Tw`로 접근특성이 바뀐 interleaved 데이터를 area 간 이동(연속 Tr회 read 시 write->read area 등).
6. 기법3(워크로드 기반 파티셔닝): `RV=nREAD*rSIZE`, `WV=nWRITE*wSIZE`를 강도 임계 `I0-I2`와 비교해 read/write area의 PU 수를 7단계(L0-L6)로 동적 조정(기본 16:16, offline 12:20).
7. 기법4(마모평준화): read area 최소 마모 PU와 write area 최대 마모 PU를 선택, hot-read 데이터만 백업 후 두 PU를 swap하는 hot-data driven WL(HRT 테이블 사용).
8. 오버헤드: 매핑엔트리에 5비트(area marker 1, op type 2, access counter 2) 추가, HRT 10MB로 저장 오버헤드 미미.
9. 결과: read 성능 평균 45% 개선(Placement 45%+Migration 17%, offline Partition 최대 96%), WL 시 read latency 39% 절감, WAF 1.2, migration 오버헤드 약 1.6%.
10. 한계: 순수 FTL/시스템 firmware 최적화이며 ECC/LDPC 부호설계·디코더와 무관(LDPC는 신뢰성 배경으로만 언급); 시뮬레이터 기반, 실 HW/실측 없음; interleaved·parallelism 민감 워크로드엔 효과 제한.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SSD PU read/write 파티셔닝·배치·마이그레이션 | 대응 없음 | FTL 시스템 레이어로 ECC 디코더/인코더/부호구조와 무관 |
| hot-data driven wear leveling | 대응 없음 | 마모평준화는 컨트롤러 FTL 관리로 ECC 범위 밖 |
| WIR 저감 I/O 스케줄링 | 대응 없음 | 채널/디코딩과 무관한 요청 스케줄링 |
```

적용 가치: **낮음** — 논문 전체가 FTL 레벨 read/write 간섭 저감·마모평준화로, Prime ECC의 부호설계->디코더->HW 어느 레이어에도 닿지 않음. LDPC는 배경 동기로만 인용.
