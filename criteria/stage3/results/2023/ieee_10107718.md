### B. 알고리즘 요약

1. 시스템: DNA storage - binary 데이터를 4염기(A/C/G/T) oligo(길이 100~300nt, 본문 `k=200nt`)로 변환, RL(homopolymer run `m`)·GC-content 제약 만족 목표.
2. 문제: constrained code와 error-correcting code의 단순 concat이 어려움(정정→제약 순은 오류 증폭, 역순은 parity가 제약 위반).
3. 핵심 기법 1(BIC): 미리 정한 binary RSP(run-stopping position)에 두 조건(COND.1 m연속 동일염기, COND.2 상위비트 동일) 만족 시 dummy 비트(RSB) 삽입해 long run 차단; 선형시간 `O(k)` 인코딩·병렬 디코딩.
4. 정보밀도 분석: RSB 삽입확률 `Pi(m)` 상·하한 유도(Lemma 1~4, Theorem 1~2), `m=3`에서 1.9748 bits/nt로 error-free 용량의 99.62% 달성.
5. 핵심 기법 2(B-BIC): modified Knuth balancing으로 상위비트 flip해 GC-content `[0.5-α,0.5+α]` 만족, `α≥1/14`면 추가 2nt만 필요, RL과 무손실 결합.
6. 오류정정 통합: 5G NR LDPC를 oligo 간 vertical(inter-oligo) 방향으로 인코딩 → substitution 오류뿐 아니라 insertion/deletion/dropout을 erasure로 정정; rate-compatible·가변 blocklength(20~8448) 활용.
7. 부수 트릭: parity oligo가 GC-content 위반 시 상위비트 flip("error insertion", `Nflip` 식 14)로 제약 만족, 삽입오류율은 채널오류 대비 매우 작음.
8. 복호: 각 vertical codeword를 LLR `L(x)=log((1-ps)/ps)` 기반 BP 디코더로 정정 후 RSB 삭제·B-BIC 디코딩으로 원본 복원(Algorithm 6).
9. 결과: `η=8448`, oligo 200nt, 403.46KB 입력을 rate 0.88 LDPC로 iBER<0.005 구간 PLER<1e-3 달성; 기존 VT-code 대비 정보밀도·정정능력 우수.
10. 한계: HW 미설계, 시뮬만; 채널은 BSC(substitution) + indel/dropout(erasure)로 NAND read와 상이; LDPC는 5G NR 표준을 off-the-shelf로 채용해 부호설계·디코더 개선 없음; gate count/throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

대상이 other(DNA storage constrained coding·부호 통합)이며 LDPC 부호설계·복호 알고리즘에 신규 기여가 없어 C섹션(디코더 성능)은 전부 N/A. 참고 대응관계만 기재:

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BIC/B-BIC constrained coding (RL/GC 제약) | 대응 없음 | DNA 생화학 제약 전용, NAND에 대응 개념 없음 |
| 5G NR LDPC 부호 채용 | 대응 없음 | 표준 5G NR 부호(Prime ECC는 고정 QC-LDPC), 신규 부호설계 아님 |
| BP 디코딩(BSC LLR) | 대응 없음 | 표준 full-BP, Prime ECC는 min-sum이며 기여 아님 |
| inter-oligo erasure/dropout 정정 | 대응 없음 | DNA plane 2D 구조 특유, NAND page 모델과 무관 |

적용 가치: 낮음 - DNA storage 특화 constrained coding 논문으로 LDPC는 5G NR 표준을 그대로 쓸 뿐이며, NAND binary QC-LDPC의 부호설계→디코더→HW 어느 레이어에도 떼어 이식할 실질 요소가 없음.
