### B. 알고리즘 요약

1. 대상 코드: IEEE 802.3an 10GbE 표준 regular LDPC `H` 384×2048, rate 0.8413, `dc=32`, `dv=6` (BSC/AWGN 채널).
2. 문제: 고SNR에서 trapping set(absorbing set)이 error floor를 지배하는데 기존 bit-flip 디코더는 이를 못 벗어남.
3. 동기(압축센싱 유추): 오류 `t`가 프레임 길이 대비 희소하면 syndrome `s=He`가 오류패턴을 거의 유일하게 특징지음 → 채널샘플 `y` 없이 syndrome만으로 복호 시도.
4. 핵심기법: SBF 에너지함수 `E_k(SBF)=Σ_{j∈M_k} s_j` (인접 non-satisfied check 수)만 사용, 채널항 제거한 GDBF 변형.
5. flip 규칙: `E_k > θ(ℓ)`이면 비트 flip, `0 ≤ θ(ℓ) ≤ dv`. threshold가 iteration마다 바뀌는 시변값(decoding key)이라는 점이 핵심.
6. decoder diversity: trapping set별 error-state 전이도를 분석해 상태 000(무오류)에 도달을 보증하는 θ 시퀀스(decoding key)를 exhaustive search로 도출(29개 topology).
7. 비고립 trapping set: θ=0에서 이웃으로 오류전파 → Heuristic 1(θ=0 뒤 비영값), Heuristic 2(θ=dv-1, dv-2 삽입), Heuristic 3(|y|≤γ 신뢰불가 비트만 SBF)로 억제.
8. HW: NGDBF symbol node에 mode 신호 1개 추가해 `q_k=0, y_k=0` 마스킹만으로 dual-function(오버헤드 최소).
9. 결과: NGDBF+SBF로 802.3an error floor를 10⁻⁶→10⁻⁸(≈2 orders), 최대 iteration 600→400으로 줄여 worst-case throughput 33% 개선.
10. 한계: bit-flipping 계열 전용 post-processing, decoding key는 코드별 사전탐색 필요, gate count·throughput 절대수치 본문 미제시.
