### B. 알고리즘 요약

1. 시스템: polar code `N=2^n` (64~1024), rate `K/N`=1/2, BPSK/AWGN, 5G 신뢰도 시퀀스로 subchannel 선택.
2. 풀려는 문제: BP 디코더는 병렬성으로 throughput 높으나 evolved LLR의 dynamic range가 넓어 균일 양자화 시 bit수(메모리)가 커짐.
3. 핵심 가정: 채널 LLR `Li~N(2(±1)/σ², 4/σ²)`, `2/σ²`를 상수로 뽑아내면 범위가 [-2,+2]로 압축되어 98% 값 포함.
4. 핵심 기법 1: base-2 로그 매핑 `Λi = sign(λi)·log2(|λi|+1)`(식 3)으로 evolved LLR 범위를 추가 압축 후 mid-tread 비균일 양자화.
5. 핵심 식 의미: 로그 압축으로 큰 LLR을 성기게, 작은 LLR을 촘촘히 표현해 적은 레벨로 넓은 범위를 커버(채널 9레벨/evolved 7레벨, 전체 범위 ±21.63).
6. 핵심 기법 2: min-sum `f(x,y)=α·sign(x)sign(y)min(|x|,|y|)` (scaled)로 CN 연산, G-matrix 기반 조기종료 적용.
7. HW 트릭: 로그 도메인의 비선형 덧/뺄셈을 실시간 계산 대신 사전계산 LUT+/LUT-로 대체, mag(a)를 16레벨 전체와 동시 연산 후 mag(b)로 index 선택(MUX).
8. 아키텍처: N/2 PE 병렬로 한 layer 갱신, dual-port RAM, layer간 고정 permutation buffer, 조기종료(hard decision→re-encoding→비교).
9. 결과(FPGA Xilinx ZC706): (1024,512)에서 throughput 1.36 Gbps, BRAM 71.5, 5bit 비균일이 8bit 균일 대비 메모리 37.5% 절감 + 고SNR BLER >0.1dB 개선, 동일 5bit 균일 대비 최대 0.4dB 개선.
10. 한계: polar code 전용, throughput 개선은 목표 아님, 전력은 다수 LUT 병렬로 증가, layer별 양자화 차등화는 future work.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar BP factor graph L/R 테이블 디코딩 | 대응 없음 (프로파일: turbo/polar 대응 없음) | polar (n+1)층 FG는 LDPC parity-check 그래프와 구조가 다름 |
| min-sum CN 연산 `α·sign·min` | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 연산 자체는 유사하나 Prime ECC는 이미 min-sum 보유(중복) |
| 로그 비균일 LLR 양자화 + LUT 매핑 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 개념상 iteration/soft-read LLR 테이블 설계에 참고 여지(구조는 재작업 필요) |
| 조기종료(G-matrix) | `partial_CRC.cpp` / `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료 보유(중복) |

적용 가치: **낮음** — polar 전용 BP FG 아키텍처라 직접 이식 불가하고 min-sum·조기종료는 이미 보유. "로그 압축 비균일 양자화" 개념만 NAND soft-read LLR 테이블에 아이디어 수준으로 참고 가능.
