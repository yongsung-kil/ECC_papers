### B. 알고리즘 요약

1. 시스템: protograph(`nP` VN, `mP` CN, base matrix `B`) copy-permute lifting으로 만든 GLDPC 앙상블, super-CN에 SPC 대신 선형블록부호(예: (6,3) shortened Hamming, (15,11) Hamming) 사용; 예제1 rate 1/6.
2. 문제: GLDPC가 waterfall/error-floor는 좋으나 반복복호 시 trapping/absorbing set이 성능 저하 유발 → 앙상블 평균 enumerator로 이를 정량화·설계에 활용.
3. 모델: BSC + parallel bit-flipping(PBF) 복호, super-CN은 bounded distance decoding(BDD), VN은 `n_v(f)>d_v/2`면 flip; 이 모델 하에서 TS/AS/FAS 정의(Def 1-3).
4. 핵심기법 1단: VN weight vector `ε`, edge weight vector `w(ε)` 도입, 유한길이 TS 개수 `E_P^TS(a,b)`를 생성함수 계수추출로 유도(식2, `A_i(x_i,y)`가 CN type별 satisfied/unsatisfied 기여).
5. 핵심 식: 점근분포 `G_P(θ,γ)=lim (1/n)ln E_P(θn,γn)`; `G_P^TS(θ,γ)`를 Lagrangian으로 최대화(식3-5)해 방정식계 해로 효율적 계산.
6. 핵심기법 2단: AS/FAS는 flip-message 캐리 여부를 나타내는 edge weight `e,w`(FAS는 `l` 추가)와 `B_j` 생성함수로 확장(식8-12), VWEF 및 MacWilliams(Lemma1)로 큰 `k_i` super-CN 계산 완화.
7. 구현 디테일: 상대 최소 ∆-set 크기 = `G_P(θ,∆θ)=0`의 첫 양의 zero-crossing → 방정식 하나 추가로 직접 계산, 앙상블 선별 필터로 사용.
8. 학습/최적화: 없음(폐형 방정식계 수치해).
9. 결과: 예제1(∆=0.005)에서 GLDPC `θ*_TS=0.1726` vs 정규(3,6) LDPC `0.0208`로 크게 우수; 예제2에서 (2,15) 정규 LDPC는 상대 최소 TS 크기 부재이나 Hamming super-CN GLDPC는 존재 → super-CN이 TS 특성 개선.
10. 한계: 순수 이론(방정식계 풀이만), HW·BER 시뮬 없음, GLDPC(super-CN)·PBF/BSC 전제.
