### B. 알고리즘 요약

1. 대상: post-quantum McEliece용 QC-MDPC (단일 행의 2~4개 랜덤 대형 circulant, 컬럼웨이트 w=45+), 예 `(n0,r,w)=(2,4801,45)` → N=n0·r=9602, rate 1/2, hard-decision 입력.
2. 문제: MDPC는 LDPC 대비 컬럼웨이트가 훨씬 커(3~4 vs 45+) LDPC decoder 단순화 기법이 부적용, 기존 MDPC는 성능 낮은 BF 위주.
3. 핵심 알고리즘: scaled Min-sum(Algorithm 1). c2v 합에 scalar α를 곱해 γ에 더함. 고컬럼웨이트 탓 α=0.25/0.5가 아니라 훨씬 작은 값 필요.
4. α 최적화: 최대 2개 nonzero digit(±1)로 표현해 곱셈을 adder/subtractor 1개로 구현, 시뮬로 `α=0.09375`가 최소 FER (α=0.25 대비 3자리 이상 FER 개선, 101 errors 기준).
5. 정밀도: c2v/v2c 4-bit magnitude+1-bit sign, C=9. 두 scalar 곱 모두 rounding(Fig.4) 적용이 truncation보다 우수(작은 α라 truncation 손실 큼).
6. 병렬 아키텍처: H의 nonzero entry만 컬럼 단위로 처리(다음 컬럼 index = 현재+1 mod r), CNU A(min1/min2/idx/s 갱신)+CNU B(c2v 도출), RAM I/M/S/C/T 구성.
7. 유연 저장: 랜덤 H의 불균등 nonzero 분포에 대해 2가지 row 분할·저장 방식 중 latency 짧은 쪽을 on-the-fly 선택 → 4-parallel에서 상위 5% 코드 평균 15.6% latency 감소.
8. 2-stage scaling: 긴 VNU feedback loop(예 33개 c2v 누적, 22 logic level)를 g개 group으로 나눠 α1 곱·rounding 후 α2 곱 → critical path 단축(step별 g/α1/α2 결정 절차, 예 g=17, α1=α2=0.375).
9. group re-balancing: 2-stage로 생기는 마지막 작은 group의 정밀도 손실을 group 크기 균등화(K-map 단순 logic)로 보완, latency penalty 없음.
10. 결과: GlobalFoundries 22FDX 합성에서 clock frequency 16% 향상(critical path 11→9 full adder), 실효 latency 14% 감소(iteration 약간 증가 반영), area overhead 미미. HW합성까지 검증, 실측(실칩)은 아님.
