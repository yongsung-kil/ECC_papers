### B. 알고리즘 요약

1. 대상은 block-permutation 구성 LDPC(base matrix `Hb`의 `tij`가 순환치환행렬 차수, 블록 `m×m`, `γ×ρ`)이며 복호 방식은 information-set decoding(ISD).
2. 문제: 독립 오류가 아니라 실제 채널의 버스트(burst, 길이 `b` 내 첫/마지막 비영 원소) 오류에서 정보집합 발견 확률을 평가.
3. 정보집합 `γ={j1<...<jk}`는 해당 위치 성분으로 부호어를 유일 복원, 생성행렬 G의 k개 열이 가역(rank=k)이면 정보집합; 버스트 위치를 제외한 폐구간(`k+Δ`)에서만 탐색해 복잡도 절감.
4. 핵심 파라미터 `Δ`: k개 연속 위치에 정보집합이 없을 수 있어 탐색구간을 `k+Δ`로 확장, Δ 최소화가 목표.
5. random 행렬 기준 full-rank 확률 `Q_Δ = ∏(1-1/2^i)`, Δ=0에서 `Q0≈0.29`, 평균 Δ≈1.61 (식 1~3).
6. block-permutation 구조 특이성: 연속 두 블록의 열 합이 항상 영벡터라, 탐색구간이 두 블록을 통째로 포함하면 항상 특이(singular)행렬 → 정보집합 없음.
7. rank(H) ≤ `mγ-(γ-1)` 이므로, 조건 `mγ-(γ-1) ≥ 3m-1` (식 6)에서 `γ ≥ (3m-2)/(m-1)` (식 7) 도출: γ≥4면 연속위치 정보집합 부재, γ=3은 특정 경우만, γ=2는 불가.
8. 실험 설정: `γ=[2,3,4]`, `ρ=6`, `m=11`, 대비용 random 행렬 `k=22,n=66`(비영 원소 확률 0.5).
9. 결과: random 행렬은 Δ 무관하게 확률 ~0.3 유지; block-permutation은 블록 경계에 peak, γ별로 최적 Δ 존재(γ=2는 Δ 증가 시 0 수렴, γ=4는 Δ 증가 시 상승, γ=3은 Δ=30에서 random에 수렴).
10. 한계: 순수 확률론적 분석(이론)으로 BER/HW/디코더 성능 미평가, ISD는 지수 복잡도, 버스트 채널 가정이라 NAND raw-error(랜덤 bit) 및 min-sum 반복복호와 무관.
