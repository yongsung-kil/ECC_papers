### B. 알고리즘 요약

1. 시스템: motif(16 nt 조각) 라이브러리 기반 DNA 저장 파이프라인 OA-DSM, 대형 블록 regular LDPC(블록 256,000 bit, 30% redundancy 예시) 사용.
2. 문제: motif는 30 bit를 16 nt에 매핑(연관배열)하므로 nt 단위 count로 LLR을 구하는 기존 방식(Chandak et al.)이 적용 불가.
3. 채널 모델: DNA 저장을 BSC(오류확률 `ϵ`) + Poisson 랜덤 샘플링으로 가정, insertion/deletion/substitution 오류 존재.
4. 핵심 식: LLR `= (k0-k1) log((1-ϵ)/ϵ)`, 여기서 `kb`는 비트가 b로 읽힌 횟수, `α = k0-k1`.
5. 의미: motif 단위 오류는 30 bit 패턴을 통째로 바꾸므로 per-position nt count 기반 `α` 산출이 부정확·과다연산.
6. 휴리스틱1: consensus 출력만 채널 출력으로 간주해 `α = ±1`로 고정(신뢰도 정보 없음).
7. 휴리스틱2: consensus motif와 정확히 일치하는 read 수를 세어 `α = ±(exact match count)` (고오류율에서 매칭 실패 위험).
8. 휴리스틱3: 정렬로 얻은 edit distance(Levenshtein) ≤ 임계값(15%) read 수를 count로 사용 → 최고 성능.
9. 결과: 시뮬(1~15% 오류율)에서 edit-distance LLR이 최저 coverage로 복구; wet-lab(1.2MB, 4340만 read, 평균 951× coverage)에서 완전 복구, 서브샘플 4× coverage까지 복구.
10. 한계: HW 설계 없음, DNA 저장 특유의 consensus/alignment에 의존, 디코더 자체는 표준 BP 사용.
