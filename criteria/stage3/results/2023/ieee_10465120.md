### B. 알고리즘 요약

1. 시스템은 binary linear block code `[64,45]` eBCH를 BPSK로 AWGN 채널에 전송, 입력은 LLR `γ_i = 2y_i/σ²`.
2. 기존 DNN 디코더는 hard-output 위주라 iterative/turbo 수신기에 필요한 soft-output 품질이 검증되지 않은 문제를 다룬다.
3. Bennatan(2018)의 syndrome-based 프레임워크를 계승하되 hard-decision 대신 soft syndrome을 사용.
4. soft syndrome `s̃_i = (∏ sign(γ_j))·min|γ_j|` (row i의 1 위치 집합 M(i))로 noise estimator 입력 `d=[|γ|, s̃]` 구성.
5. 잡음추정 NN F가 `ẑ=F(d)`를 내고 `γ̂ = γ − sign(γ)⊙ẑ`로 soft-output LLR을 추정(Algorithm 1).
6. F는 Stacked-GRU (hidden `5n`, 4 layers, 5 time-steps), 마지막 FC로 차원 축소.
7. 학습은 BCE loss로 시작 후 마지막 epoch에서 MAP 출력 `γ*` 기준 정규화(MSE/KL/moments) 추가하여 출력 분포를 MAP에 근접.
8. TPC(행/열 `[64,45]` eBCH)에 적용, 반복마다 extrinsic scale `α_c,α_r`를 trainable로 두고 fine-tune, LLR 폭증은 L1 batch-norm으로 억제.
9. 결과: soft-output DNN이 BER/FER에서 MAP에 근접하고 Chase-Pyndiah를 크게 상회(Fig.4,6,7).
10. 한계: HW 미설계, NN 파라미터 약 2.2·10⁶로 복잡도 큼, AWGN·eBCH/TPC 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome-based DNN(Stacked-GRU) 디코더 | 대응 없음 | NN 디코더는 Prime ECC(min-sum) 구조와 무관 |
| soft syndrome / soft-output LLR 추정 | 대응 없음 | non-LDPC eBCH 대상, LLR 테이블과 접점 없음 |
| TPC 반복 스케줄 / extrinsic scale | 대응 없음 | product code 반복이며 QC-LDPC 이터레이션과 다름 |

적용 가치: 낮음 — eBCH/TPC용 NN 디코더로 NAND binary QC-LDPC 및 Prime ECC 모듈에 대응이 전혀 없다.
