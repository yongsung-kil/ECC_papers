### B. 알고리즘 요약

1. 시스템: 고rate binary QC-LDPC 부호설계(통신/5G·IoT 지향), AWGN·BEC 채널, 예시 `[1849,0.91,4,43]`(a=43 odd), `[1892,0.91,4,43]`(a=44 even).
2. 문제: 기존 finite-field 기반(transversal design 등) QC-LDPC는 order가 소수거듭제곱으로 제한돼 code length/rate 유연성이 부족.
3. 핵심기법: order a의 순환군(mod a 덧셈)상 difference matrix `DM(3;a)`(a 홀수)와 difference covering array `DCA(3;a)`(a 짝수)로 4a×a² 형태 PCM(식 4,5)을 CPM `Pi` 블록으로 구성.
4. 핵심: PCM 각 블록의 shift가 `DM/DCA`의 원소로 결정되고 나머지 열은 순환시프트 → 저장은 `DM(3;a)` 한 배열만, 인코딩/복호 병렬성·저장효율 확보.
5. 구조: 홀수 a는 `(a²,4,a)` regular, 짝수 a는 한 부블록 제거로 `(a²-a,4,{a-1,a})` near-regular, 모두 RC-constraint 만족 → girth≥6(4-cycle 제거).
6. 이론보증: rank가 홀수 `4a-3`/짝수 `≤4a-6`로 rate `1-(4a-3)/a²`(홀), `≥1-(4a-6)/(a²-a)`(짝) 확정.
7. 최소거리/stopping distance: `3∤a`면 stopping distance≥8, `3∤a·5∤a·α=(a-1)/2`면 최소거리 10인 무한족 QC-LDPC 보증(error-floor 하락 직결).
8. 학습/최적화: 없음(조합론적 결정 구성), row/column 순열로 QC 표준형 `H*`(식 20) 유도.
9. 결과: AFF3CT로 SPA(AWGN)·NMS(BEC) 100 iter 시뮬, DM(3;43)(dmin 10)이 DCA(3;44)(dmin 8)보다 고SNR 우수, 기존 [12] 대비 BER/FER 개선, Array/TD/Lattice/SCB 동일파라미터 부호와 대등.
10. 한계: HW 미설계·시뮬만, 열가중치 wc=4 고정에 길이가 a²(-a) 이산값이라 임의 rate/length·degree profile 조정 불가.
