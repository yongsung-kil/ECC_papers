### B. 알고리즘 요약

1. 대상: moderate length binary LDPC (AB (3,5) n=500, 5G-NR [520,100] BG2 lift z=10, WRAN [384,256]), AWGN + BPSK, 표준 BP(soft SPA, 식2의 `2 atanh(prod tanh(m/2))`).
2. 문제: BP 업데이트 순서(스케줄)가 수렴/성능에 큰 영향, 기존 layered/NS/EDS-LBP/RBL-BP는 residual 계산 비용·error-floor(myopic error) 존재.
3. 모델: Tanner 그래프를 finite MDP로 보고 CN cluster 스케줄 문제를 액션 `A=[[ceil(m/z)]]`로 정의, cluster size `z<=2`(z=1 최선).
4. 기법1 (RELDEC): 새 state space = cluster별 hard-decision VN 비트열 `x_hat_a`(2^la 크기)로 축소, reward `Ra`=클러스터 비트 정답 비율(식5)로 Q-learning(식6) 스케줄 정책 학습.
5. 새 state 표현으로 action-value table이 max_a(2^la)×ceil(m/z)로 축소 → block length 500까지 학습 가능(이전 196 한계 극복), 매 iteration 모든 cluster 1회 스케줄로 error-floor 개선.
6. 기법2 (AM-RELDEC): meta-RL(MAML을 Q-learning에 적용), SNR 혼합으로 global Q 학습 후 관측 SNR에 pilot 소량(75, 심지어 7)으로 online 적응, local Q는 global 갱신에 재환류.
7. 구현 디테일: z=1은 explicit Q-table, z=2는 deep-Q-learning 사용; hyperparam α=0.1, β=0.9, ε=0.6, ℓmax=50, Imax=50(WRAN 5).
8. 학습: |L_hat|=15000(RELDEC), AM-RELDEC 100 meta-iteration(99 global+1 online), all-zero codeword로 학습(BP 대칭성).
9. 결과: flooding·random·NS·EDS-LBP·RBL-BP·5G VN-layered·hyper-network[29]보다 BER/FER 낮고 CN→VN 메시지 수도 평균 감소(residual 계산 불필요), AM-RELDEC이 RELDEC보다 소폭 우수.
10. 한계: HW 미설계·시뮬만, block length<=520 wireless/5G 타깃, z 클수록 Q-table 지수 증가(deep-Q 필요), NAND/QC-LDPC HW 무관, gate count/throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN 스케줄링(cluster/layered order) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 스케줄 개념은 Dual-Update(even/odd) 위치에 닿지만 RL 정책 자체는 대응 없음 |
| RL/meta-RL 학습 스케줄러 | 대응 없음 | NN/딥러닝·RL 디코더는 프로파일상 명시적 "대응 없음" |
| full tanh SPA CN 연산(식2) | decoder.cpp CNU_Update_New_Mag() (min-sum) | Prime ECC는 min-sum 근사이며 tanh SPA와 이질적 |
| BP 조기종료(H*x_hat=0) | full_CRC.cpp / partial_CRC.cpp | 종료조건 개념 접점뿐, 방식(parity check vs CRC) 상이 |
```

마지막으로 적용 가치: **낮음** — RL/meta-RL 기반 tanh SPA 스케줄러로 프로파일이 "대응 없음"으로 규정한 딥러닝/RL 디코더 계열이며, NAND용 고정 QC-LDPC min-sum HW 디코더와 알고리즘·검증(wireless AWGN, block<=520)이 맞지 않는다.
