### B. 알고리즘 요약

1. 시스템: MLC NAND(64GB, page-level FTL) 대상의 FTL 데이터 배치 전략이며 LDPC는 read latency 원인으로만 다뤄지고 부호 스펙(rate/length)은 명시 없음.
2. 문제: 높은 RBER에서 LDPC가 hard-sensing 실패 시 soft-sensing(다단계 read-retry)로 read latency가 급증 - 특히 write-once/read-multiple 서비스에서 QoS 저하.
3. 모델: retention time↑ 또는 P/E cycle↑ → RBER↑ (Fig.2) 라는 경험적 관계와, 데이터별 lifetime/access 패턴 차이를 가정.
4. 핵심 기법 1단(Access Feature Identification): `Read/Write Bloom Filter`로 LBA를 키로 read/write 각각 별도 bit array에 해시. false positive 근사식 `(1 - e^{-NK/M})^K` (M=엔트리수, N=원소수, K=해시함수수).
5. 핵심 식 의미: FP<2% 목표 시 M/N=8, K=6 필요 - RAM overhead와 정확도의 trade-off를 정량화.
6. 확장(3-level hotness): 실트레이스(Proxy0) 분석상 대다수 LBA가 구간당 read<3회이므로 Observation/Recency/Frequency 3단계(3-bit)로 hot 정도 판별, 주기적 decay.
7. 배치 트릭(Request-based Allocation): 데이터를 read/write hot·cold 4종으로 나눠 P/E cycle 기반 4개 파티션(young→old)에 배치 - read-hot은 RBER 낮은 young block에.
8. Migration: aggressive wear-leveling 대신 lazy migration - block이 migration threshold 근접 시에만 이동해 불필요한 live-page copy 회피.
9. 결과: baseline wear-leveling 대비 평균 read response time 최소 ~32% 감소, total live-page copying 최소 ~11% 감소(모두 정규화 값, 절대치 미상).
10. 한계: LDPC 부호/디코더/HW 미설계(블랙박스), trace-driven 시뮬만, gate count/throughput 등 HW 수치 없음.
