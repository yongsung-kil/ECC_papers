### B. 알고리즘 요약

1. 시스템: software-defined 통신용 GPU 기반 polar code BP 디코더, `(n,k)` polar, rate `0.5`, `n=512~4096`, BPSK/biAWGN.
2. 문제: ASIC/FPGA polar 디코더는 고속이나 유연성 부족, 기존 GPU BP 디코더[8]는 저장량·병렬화 한계로 throughput 낮음.
3. 모델: polar factor graph의 `m=log2 n` stage, stage당 `n/2` PE, 각 노드에 L(우→좌)/R(좌→우) LLR 메시지.
4. 핵심기법1(저장): `τ(Li)≺ω(Ri)≺τ(Ri)`(Property 1) 증명으로 L,R을 충돌 없이 한 행렬 M에 통합 저장 → `2n(m+1)`→`n(m+1)`로 50% 절감.
5. 핵심식: 메시지갱신 `g(a,b)=α·sgn(a)sgn(b)·min(|a|,|b|)` (정규화 min-sum), 통합저장 후 `M(i+1,2j-1)=g(M(i,j),M(i+1,2j)+M(i,j+n/2))`.
6. 핵심기법2(연산): `R(m+1,j)`는 갱신에 안 쓰임(Property 2) → 마지막 stage 제거, iteration당 계산 메시지 `2nm`→`n(2m-1)`.
7. 구현 디테일: 정규화계수 `α=0.9375`를 `α·z=z-(z>>4)` (뺄셈+shift)로 근사, 메시지 int8_t 8bit 저장, shared memory 활용.
8. 병렬화: `n≤2048`은 thread=PE, `n=4096`은 thread당 PE 2개로 stage 완전병렬, multi-stream+asyncMemcpy로 부하균형·전송 은닉.
9. 결과: `n=1024`에서 `3.8Gbps`, 정규화 throughput이 SOTA GPU BP[8] 대비 64.1~294.1% 향상(RTX 3090 실측), SCL 대비 BER 약 0.7dB 손실.
10. 한계: polar code 전용(LDPC 아님), GPU 플랫폼 특화(ASIC HW 아님), 조기종료 있으나 avg iteration 감소는 정정성능 기여 아님.
