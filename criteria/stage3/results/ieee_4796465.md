### B. 알고리즘 요약

1. IEEE 802.11n irregular QC 형태 LDPC 코드 (648,324), rate 1/2 대상 partially-parallel 디코더 설계.
2. 기존 message-passing 디코더는 row operation에서 갱신되는 β(m,n) 메시지를 열마다 최대 12개까지 저장해야 해 저장 면적이 크고, min/2nd-min 비교 연산에 다수 clock이 소요되는 문제.
3. 표준 Min-Sum 기반 채널/오류 모델(802.11n irregular LDPC의 row/column operation, 표준 message-passing 알고리즘)을 그대로 사용.
4. 핵심 기법(SDMP, sum-delta message passing): `sum(n) = λ(n) + Σα(m,n)` 하나만 저장하고, β(m,n)=sum(n)-α(m,n)으로 계산해 저장 메시지 종류를 β 대신 단일 sum으로 축소.
5. `sum'(n) = sum(n) + ΣΔα(m,n)` 로 갱신하며 D(n)(직전 계산 이후 갱신된 α의 행 집합)만 더해 덧셈 횟수를 |B(n)|에서 |D(n)|로 감소.
6. 확장: row operation에 binary sort 구조(상위/하위 트리로 min·2nd-min 동시 탐색)를 적용해 비교를 9 clock→2 clock으로 단축, column operation은 갱신된 α만 병렬 처리해 1 clock에 완료.
7. 구현 디테일: row op 5-stage, column op 3-stage 파이프라인으로 데이터플로우 구성; α는 메모리, sum/Δα는 레지스터에 저장(다중접근 문제 대응); 12 CFU + 24 BFU 병렬 배치.
8. 별도 학습/최적화 절차 없음(고정 HW 스케줄링 설계).
9. 결과: TSMC 0.18um, 200MHz, 1 iteration 32 clock, 5 iteration(SNR 3.0dB)으로 codeword correction, throughput 404Mbps(기존 대비 7.5배), gate count는 참조 대비 11%(vs [2])~39%(vs [3]) 절감, 저장 면적은 β 저장 대비 총 3,117,887 μm² 절감.
10. 한계: 정정 알고리즘 자체는 표준 min-sum 그대로이며 성능 개선이 아닌 메시지 저장/스케줄링 최적화이고, BER 성능은 C++ 시뮬로만 검증(그림에만 제시, 수치 본문 서술 없음), iteration 수는 5로 고정 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SDMP: β 대신 sum(n) 저장, Δα 누적 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | C2V/V2C 메시지 저장 방식 자체를 바꾸는 최적화라 이식 시 재검증 필요하나 개념적으로 메모리 절감 아이디어는 참고 가치 있음 |
| binary sort 기반 min/2nd-min 탐색 | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum의 min1/min2 계산 가속 구조로 Prime ECC CNU 연산과 직접 대응 |
| row/column 5-stage/3-stage 파이프라인 | 디코더 HW 모델(pipeline 6-stage) | Prime ECC도 pipeline 구조 보유, 스테이지 재설계 시 참고 가능하나 구조 상이 |
| α 메모리 / sum·Δα 레지스터 분리 저장 | `decoder.cpp` `LDPC_Decoder()`, `ecc_data.h` | 메모리 구조 최적화는 우리 구조와 정합 필요(HW arch 이식 난이도 중) |
| irregular VN/CN degree별 다중 연산모듈(row 2종, column 3종) | `decoder.cpp` `VN_Cal_HD()` 등 | Prime ECC는 특정 QC-LDPC(HCU로 비대칭 degree 처리) 고정이라 irregular 다중모듈 구조는 대응 부분적 |

> 적용 가치: 중간 | 정정성능 개선은 없으나 β 메시지 미저장(sum-delta 갱신)과 binary sort min-search는 Prime ECC의 CNU 메모리/연산 최적화에 실질적으로 참고할 만한 HW 기법.
