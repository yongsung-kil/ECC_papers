### B. 알고리즘 요약

1. 시스템: TLC 3D-NAND(16kB page = 4kB chunk 4개), lower/center/upper 페이지 구분, QC-LDPC ECC, PCIe NVMe 컨트롤러 특성화 장비.
2. 문제: 3D-NAND는 파라미터 공간·페이지/블록별 신뢰도 편차가 커 "judge-by-eye"로 최적 조건·code rate 찾기 불가능.
3. 데이터: 800+ 블록을 3k P/E cycle 후 read해 BER 측정(수개월 캠페인), worst BER 기준으로 ECC 정정능력 설계.
4. 핵심 기법: k-means 클러스터링으로 블록 내 유사 신뢰도 영역을 k=5로 분할, centroid 수렴까지 반복.
5. 문제점: 단순 k-means는 클러스터 분리가 불명확하고 컨트롤러에 구현하기엔 복잡도 과다.
6. 확장(제약 클러스터링): semi-supervised 제약(한 TLC 페이지의 chunk는 동일 클러스터, 페이지 인접성, lower/center/upper 동일 클러스터 공유)을 부과해 6개 클러스터로 분리, 펌웨어 구현 용이화.
7. 부호구조: QC-LDPC H-matrix가 circulant 배열이라 modular 인코딩·layered 복호가 가능해 code rate를 HW 구조 변경 없이 adaptive하게 조정 가능(식 1).
8. 최적화(식 2): 페이지 타입·클러스터별 worst BER에 FER=1e-4 유지 code rate 배정, 등가 code rate = Σ(n_j·CR_ij)/(3·WL).
9. 결과(실측): 단일 code rate 0.75(보수적) 대비 페이지 분리 시 0.87(16% 이득), 클러스터링까지 적용 시 등가 rate 0.93으로 24% user space 이득.
10. 한계: LDPC 디코더/알고리즘 자체 개선 아님, 오프라인 클러스터링 기반 code-rate 배정 방법론, code length·HW 미제시.
