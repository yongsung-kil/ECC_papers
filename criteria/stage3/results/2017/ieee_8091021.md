### B. 알고리즘 요약

1. 시스템: 802.11n QC-LDPC(`81×81` circulant, length `1944`, rate 1/2·3/4·5/6) 내부부호(inner) + BCH 외부부호(outer) concatenated, 16-QAM/AWGN, min-sum(MSA) 복호.
2. 문제: 광통신 등 초고속 링크에 ASIC/FPGA 대신 저비용·유연한 SW 솔루션으로 Gbps급 LDPC 복호 throughput 확보.
3. 구조: outer BCH 인코딩→inner LDPC 인코딩. 수신부는 LDPC 먼저 복호→BCH로 residual error 정리(error floor 낮춤).
4. LDPC 커널: horizontal(row) processing·vertical(column) processing·hard decision 3단계. 1 thread block = 1 codeword, row↔column 간 데이터 교환.
5. 핵심 최적화: H는 972 rows×1944 cols(rate 1/2). thread당 register 63개 제약(1944 thread면 register 초과)으로 block당 thread를 `243`으로 튜닝(SM occupancy·active block 최적).
6. 메모리: global memory→shared memory(SMEM, 96KB) coalesced 이동, thread당 4 row·8 column 처리(rate 1/2), hard decision은 max iter 도달 시 1회.
7. BCH outer: syndrome `S(x)` 생성→Berlekamp-Massey로 error locator `σ(x)`→Chien search로 정정 `C(x)=E(x)+R(x)`. 각 다항식 계수를 thread 1개가 계산.
8. 플랫폼: TITAN X(Pascal GP102, 3584 CUDA core, 12GB), CUDA 8.0, 32768개 codeword 동시 복호.
9. 결과(Table II): rate 3/4 10 iter서 peak `1.82 Gbps`(rate 1/2 5 iter 2.43Gbps). BER(Fig 7): GPU MSA는 Matlab BP보다 나쁘나 LDPC+BCH concatenation이 Matlab 성능/Shannon 한계 근접 회복.
10. 한계: GPU/CUDA 소프트웨어 구현으로 HW(RTL) 미설계, min-sum 성능손실을 BCH concatenation으로 보상, latency는 GPU batch 특성상 비교 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC min-sum(MSA) inner 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 우리도 min-sum 보유(중복) - GPU MSA 구현 자체는 이식 대상 아님 |
| horizontal/vertical(row/col) 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `C2V_Cal()`, `VN_Cal_HD()` | 메시지 패싱 개념만 동일, GPU thread mapping은 우리 z=32 row 병렬 HW와 무관 |
| BCH outer + LDPC concatenation(error-floor 정리) | 대응 없음 (Prime ECC는 CRC 조기종료만, BCH concat 미보유) | error-floor를 outer code로 정리하는 개념적 참고만 |
| GPU SMEM/thread 자원 최적화 | 대응 없음 (우리는 RTL bit-exact HW 모델) | GPU 병렬화는 NAND 컨트롤러 HW에 이식 불가 |
| 802.11n QC-LDPC(circulant) 로드 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | QC 구조는 유사하나 WiFi 부호는 우리 고rate NAND 부호와 다름 |

적용 가치: **낮음** - GPU/CUDA 소프트웨어 가속 논문으로 throughput·병렬화가 GPU 특화라 우리 HW 시뮬레이터/NAND 컨트롤러 RTL에 이식 불가. LDPC+BCH concatenation으로 error-floor를 정리한다는 개념적 시사만 있으나 우리는 CRC 조기종료 보유.
