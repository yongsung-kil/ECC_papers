### B. 알고리즘 요약

1. 시스템: regular QC-LDPC(circulant permutation matrix 기반), rate `0.5/0.75/0.9`, girth 6/8/10/12, BPSK+AWGN, LogSP 복호(20 iteration).
2. 문제: girth가 작으면(loop 4/6) 반복복호에 트랩이 생겨 성능 저하, 큰 girth를 간단히 얻는 구조적 설계가 필요.
3. 모델: 소수 `p` 크기 identity matrix를 우측 shift한 버전들을 concat, shift 값 `a`를 등비수열 `an = a1·γ^(n-1)`(식 1)로 결정, `γ`는 GF(p)의 root(예: GF(13) root 2,6,7,11).
4. 기법 1단: `a1=1`, `γ=GF(p) root`로 full-range matrix 생성(행 수 `j×k`), 첫 행의 shift 후 나머지 행은 shift된 버전.
5. 핵심 식: full-range matrix를 `j×j` sub-block으로 나눈 뒤 각 sub-block에서 girth-6을 유발하는 동일 column을 삭제 → loop 제거로 girth 상승.
6. 기법 2단: girth 8은 girth 6 설계 기반, girth 10/12는 같은 절차 반복(단 `p`가 커져야 함), sub-block append 개수로 code rate 조정.
7. 구현 트릭: column weight `wc=3` 위주, 최소거리 `dmin ≥ wc+1`, girth 못 키우면 column weight를 올려 최소거리 보전(단 PCM 크기 증가로 HW 불리).
8. 최적화: 학습 없음, girth·rate 만족까지 column 추가 반복(설계 절차 7단계).
9. 결과: rate 0.5에서 제안 QC(girth 8, (321,642))가 BIBD class II 대비 BER 1.25dB 개선, rate 0.75는 PEG와 유사·Tanner보다 우수, rate 0.9는 column weight 낮을수록 BER/FER 우수.
10. 한계: HW 미설계, 시뮬만, girth 12는 계산자원 한계로 시뮬 생략, gate count/throughput 없음, error-floor 정량 보증 없음.

적용 가치: **중간** — QC-LDPC circulant shift를 소수체 root로 정해 large girth를 얻는 구성법은 Prime의 base+shift 부호설계에 개념적으로 참고 가치가 있으나, column weight 3·regular 위주이고 Prime의 고정 z=32·고rate H-matrix에 맞추려면 재설계·재검증 부담이 있다.
