### B. 알고리즘 요약

1. 대상: base graph를 copy-and-permute(cyclic permutation)로 lifting한 QC protograph LDPC, rate-1/2, length 2304~32768(예제별 상이), BPSK/AWGN.
2. 문제: 기존 PEG로 QC 부호 구성 시 base graph 생성 후 permutation shift 결정을 위한 추가 cycle searching이 필요해 code length 증가 시 복잡도 급증(`O(nmL²)`).
3. 핵심 정의: lifted graph girth `g`는 base graph girth `gb`, lifting degree `L`, permutation shift `p(i,j)`로 결정. path permutation shift `Pρl = Σ(p(it,jt)-p(it,jt+1))`.
4. 핵심기법 1단: PSD-PEG - base graph 구성과 shift matrix `S` 결정을 동시에. tree spreading 중 shift 누적(식 15), edge 성립 시 조건 `(Pρl^cil - p(il,j0)) ≡ 0 mod L`(식 4) 만족하도록 shift 배정.
5. 핵심 식: Lemma 3.3 - 조건 (4) 만족 시 base cycle 2(l+1)이 lifted graph에서 `≥ 2·L1·(l+1)` cycle로 확장 (L1 = L의 최소 인수).
6. 핵심기법 2단: girth `ηgb` 달성을 위한 lifting degree 하한 `L > ⌈(dsmax/m)·((dsmax-1)^gb(dcmax-1)^gb -1)/((dsmax-1)(dcmax-1)-1)⌉` (식 11), η=2(L1=2) 또는 3.
7. 확장: nonbinary GF(q)로 확장 - full rank condition(식 21)으로 low-weight codeword 제거, β-multiplied circulant로 대체.
8. 복잡도: PSD-PEG는 shift 누적/선택이 L에 무관해 전체 `O(nm)`, PEG/PEG-QC의 `O(nmL²)` 대비 대폭 감소 (10000bit 구성: PEG 10108s vs PSD-PEG 10.2s).
9. 결과: girth 하한이 동일 length PEG 대비 훨씬 높고, FER에서 PEG-QC 대비 약 0.2dB@FER 10⁻⁵ 이득(Example 1), 802.16e/multi-edge QC 대비 우세.
10. 한계: 시뮬레이션(sum-product, max iter 50)만, HW 미설계, rate-1/2 AWGN·BPSK 가정, error-floor 정량 분석 없음.
