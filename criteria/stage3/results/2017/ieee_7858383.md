> 총평: MLC flash의 상태별/방향별 비대칭 에러를 활용해 sensing level을 비대칭 배치, LDPC read latency를 평균 54% 줄이는 soft-read 최적화. NAND 직접 타깃이나 LDPC 부호·디코더 자체는 손대지 않는 채널/센싱 레이어 기법.

### B. 알고리즘 요약

1. 대상: MLC(2bit/cell, 4-state) NAND flash + soft-decision LDPC, 인접 상태쌍당 최대 7 sensing level(총 21), soft 정보 log(N+1) bit.
2. 문제: 고RBER 데이터는 fine-grained soft sensing이 필요해 sensing latency + flash→controller 전송 latency가 큼 → LDPC 디코딩 지연.
3. 핵심 관찰: flash 에러는 두 비대칭 특성 — inter-state(상태쌍별 RBER 상이, S2/S3 >> S0/S1), intra-state(한 상태의 left-shift(retention 누설) vs right-shift(P/E·셀간간섭) 비대칭).
4. 기법1 inter-state: RBER 높은 쌍에 sensing level 더 배치. 관계식 `if e_k < e_{k+1} then N_k ≤ N_{k+1}` (1) — 고에러 영역 우선.
5. 기법2 intra-state: `N_k = N_kl + 1 + N_kr` (2), hard level 1개 기준 좌/우 비대칭 분배. `e_kl > e_kr → N_kl ≥ N_kr` (left-shift aware), 반대는 right-shift aware.
6. 의미: 같은 총 sensing level로 에러 많은 쪽에 집중 배치 → 동일 정정능력을 더 적은 level로 달성 → level·정보bit 감소.
7. 구현: 디코딩 실패 시 level을 반복 증가시키되 비대칭 순서로(Table I: N2→N1→N0 우선). 좌/우 방향은 P/E cycle 임계값 `T_PE`로 판정(초과 시 right 우선).
8. 결과 예: P/E 5K·retention 3년에서 대칭 21 level(5bit) → 비대칭 14 level(4bit)로 감소.
9. 성능: DiskSim SSD 모듈 + MSR trace 시뮬에서 read latency 평균 54% 감소(retention 길수록·read-intensive일수록 효과 큼).
10. 한계: LDPC 부호/디코더 미변경(순수 sensing 배치), 정정능력 향상 아닌 "유지", HW 미설계, error model 기반 시뮬, MLC 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 비대칭 sensing level 배치 / soft-read threshold | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 상태별·방향별 비대칭 threshold로 확장 가능(soft-read 비용 절감 핵심) |
| level→LLR 양자화 (log(N+1) bit) | `channel.cpp` + `ecc_data.h` `PARAM_LLR` | 비대칭 배치에 맞춘 LLR 테이블 조정 필요 |
| iteration별 progressive sensing 추가 | `decoder.cpp` `Get_VNU_Table_Idx()`, 적응형 LLR 테이블 | 실패 시 level 증가 로직과 iteration 구간 LLR 연동 |
| P/E cycle 기반 좌/우 우선순위 결정 | 대응 없음 | 컨트롤러 정책 레이어(코드베이스 외부) |

적용 가치: **중간** — NAND 직접 타깃이고 Prime ECC의 soft-read threshold(2SD/3SD)에 상태별·방향별 비대칭 배치를 얹어 read latency를 줄일 여지가 크나, Prime ECC는 soft-4bit+를 미지원하고 이 기법은 LDPC 부호/디코더 자체를 개선하지 않아 정정능력 이득은 없다.
