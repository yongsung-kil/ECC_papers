### B. 알고리즘 요약

1. 대상 소자는 DRAM 대체용 STT-MRAM으로, ECC는 ~10% redundancy + 매우 짧은 codeword + 10~30ns read latency 요구.
2. 문제: 기존 (71,64) Hamming HDD는 process variation 증가 시 error floor에 걸려 write error rate 감내폭이 부족.
3. 채널모델: write/read-disturb 비대칭 오류를 BAC로, read decision 오류를 Gaussian mixture channel(GMC)로 모델링해 cascaded BAC-GMC를 구성.
4. 이 cascaded 채널의 raw BER를 해석적으로 유도하고, 최적 hard sensing threshold와 dominant error event를 도출.
5. 채널 LLR을 유도해 SDD를 가능케 하고, 최적 ML decision rule과 ML metric을 도출 (기존 SED metric 대비 우수).
6. 복호: 1단은 HDD 수행, 성공 선언 시 즉시 출력 → 실패 시에만 2단 Chase-II 복호로 진행하는 하이브리드 구조.
7. Chase 복호에서 cascaded 채널 ML metric을 적용하고, 채널 LLR로 least-reliable Q-bit(Q=2) 위치 식별.
8. GMC 채널을 L=3 bit로 양자화하며, 2^L-1 threshold를 cascaded 채널 capacity 최대화로 결정.
9. 확장 Hamming의 추가 error-detection 능력을 활용해 일반 Hamming 대비 하이브리드 이득을 크게 확대.
10. 결과: (72,64) 확장 Hamming이 P1=1e-4, FER=1e-5에서 HDD 대비 σ0/µ0 2%+ 이득, error floor 극복. Chase는 HDD 실패 시(≈FER)만 활성 → σ0/µ0=8%에서 복잡도 증가 약 4%.
11. 한계: LDPC 아님(Hamming/BCH), HW 미설계, STT-MRAM 비대칭 채널 특화 가정.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| cascaded BAC-GMC 채널모델 / raw BER·sensing threshold 유도 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()`, `Set_LLR_Th()` | STT-MRAM 비대칭 채널 특화라 NAND AWGN/RBER 모델과 직접 대응은 약함 |
| 채널 LLR 유도 / GMC L=3bit 양자화(capacity 최대화 threshold) | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_LLR_Th()` | soft-read LLR threshold 설계 관점 일부 참고 가능 |
| HDD→Chase-II 2단 하이브리드 복호 | 대응 없음 | Prime ECC는 min-sum LDPC 복호기, Chase/block-code 복호 없음 |
| ML metric 기반 조기종료(HDD 성공 시 출력) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 조기종료 개념은 유사하나 CRC 기반이라 메커니즘 상이 |
| (72,64) 확장 Hamming 부호 | 대응 없음 | binary QC-LDPC 아님 |

적용 가치: 낮음 — STT-MRAM 비대칭 채널 특화 Hamming/Chase 하이브리드로 binary QC-LDPC 디코더/부호설계에 떼어 쓸 요소가 거의 없다. 다만 GMC LLR 양자화·threshold 유도 사고방식만 soft-read 설계에 간접 참고 가능.
