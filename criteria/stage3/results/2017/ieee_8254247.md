### B. 알고리즘 요약

1. 시스템: block length `N=2^n` polar code(생성행렬 `G_N=F^{⊗n}`), 예제 (256,128)/(1024,512), rate 0.5, 6-bit 양자화 BP 디코더, AWGN.
2. 문제: polar BP는 고throughput·저latency지만 error-correcting 성능이 SC/유사길이 LDPC BP보다 열등 - moderate~high SNR 성능을 개선하려 함.
3. 모델: BP 오류를 3종으로 분류 - unconverged(저SNR 지배), false-converged(잘못된 codeword 수렴), oscillation(factor graph loop로 hard decision 진동).
4. 핵심기법1: polar에 CRC를 concatenate(frozen bit 일부 재할당)해 99%+ 오류 검출 및 조기종료; 검출된 오류에만 조건부 사후처리.
5. 핵심식: cost function `C(x)=sqrt(∑(Mmax-xi)^2/|A|)` - 디코딩 soft decision과 all-zero codeword 간 정규화 평균거리로 섭동 효과 정량화.
6. 확장(오류유형별 3알고리즘): false-converged=신뢰bit 강화+불신뢰bit 섭동 후 BP cleanup(Alg.1); oscillation=stable bit 강화+unstable bit 섭동(Alg.2); unconverged=stable 강화+섭동 unroll(Alg.3).
7. 구현 디테일: frozen-set R메시지(`Ri`)를 `M0`만큼 bias, 실패 시 `c>1`로 점증 강화, `Mlimit` 상한; 정보bit를 방향성 있게 partial-freeze.
8. 학습/최적화: 파라미터(`M0,Mlimit,c`)는 고정값 하나로 두 부호·다양한 SNR에 공통 적용, 오류유형별 설계라 임의 polar code에 일반화 주장.
9. 결과: FER 1e-8에서 coding gain (256,128) 0.8dB·(1024,512) 1dB+ 개선, SC를 0.4~0.5dB 능가; 45nm 합성 시 area overhead CRC 1.8%+PP 2.5%, throughput/latency 불변.
10. 한계: polar code·BP factor graph에 특화(frozen set 조작), throughput/area는 좋으나 부호종류가 LDPC 아님.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar BP 디코딩 전체 | 대응 없음 | 프로파일 명시 "turbo/polar = 대응 없음" |
| CRC 조기종료 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 중 - Prime ECC는 이미 Partial+Full CRC 조기종료 보유(중복) |
| frozen-set R메시지 섭동/강화 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 낮음 - polar frozen set 개념은 LDPC에 없음, 유형별 post-processing 아이디어만 유사 |
| soft decision 신뢰도 기반 bias | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 낮음 - 적응형 LLR 유사하나 polar 전용 알고리즘 |
```

적용 가치: **낮음** - polar code + BP factor graph 전용(frozen set R메시지 조작)이고 CRC 조기종료는 Prime ECC가 이미 보유. 오류유형 분류·섭동/강화 아이디어의 개념적 참고 가치만 존재.
