> 총평: 대칭 지수행렬로 girth-8 regular QC-LDPC를 기존보다 짧은 block length(같은 rate)로 구성하는 부호설계 논문. compact 표현으로 HW 효율 가능성은 있으나 디코더/정정성능 개선은 없고 성능은 기존 부호와 동등.

### B. 알고리즘 요약

1. 대상: CPM 기반 regular QC-LDPC, base graph는 fully-connected m×n(dv=m, dc=n), block-length `L=nN`, rate `R≥1-m/n`, m=3~6, BPSK/AWGN.
2. 문제: girth-8 달성 최소 lifting degree `N`(=최단 block length)을 exhaustive search로 찾는 건 m,n,N 커지면 폭발 — 특히 고rate(큰 n) 불가.
3. 핵심 기법: n짝수일 때 지수행렬을 `P^e = (A | -A)` 대칭구조로 강제(A는 m×n/2, 첫 행 0, `-A`는 modulo-N 가법역원). n홀수는 앞에 all-zero 열 추가.
4. 식: girth 조건은 `Σ(p_{m_i n_i} - p_{m_i n_{i+1}}) ≡ 0 mod N` (1) — 이를 위배하지 않도록 4/6-cycle 부등식 그룹 (I),(II)로 전개. 대칭 덕에 자유변수 절반으로 감소.
5. 의미: 대칭으로 탐색 변수 수를 줄여 exhaustive search 도달범위를 큰 n(고rate)까지 확장, 동시에 지수행렬 compact 표현.
6. 확장(Theorem 1/2): 짧은 girth-8 부호를 building block으로 degree-`mN` cyclic lifting → m×2n base graph의 girth-8 부호로 결정론적 확장(더 긴/고rate).
7. 구현 디테일: search는 column-wise greedy + backtracking, MAGMA 구현, dmin은 MAGMA "MinimumDistance"로 확인.
8. 성질: `dmin ≤ 2m` 상한 증명(Proposition 2). 예) (3,50) rate-0.94 L=24000 (기존 [13] 62550 대비 대폭 단축).
9. 결과: 대부분 케이스에서 기존 girth-8 부호보다 짧거나 동일 block length. layered min-sum(최대 150 iter) 시뮬에서 비대칭 [20] 부호와 거의 동등한 BER/BLER(약 10^-7까지).
10. 한계: 순수 부호구성(디코더/HW 미설계), 성능은 기존 대비 "동등" 수준, 정정능력·error-floor 개선 주장 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 대칭 지수행렬 girth-8 QC-LDPC 구성 | `ecc_top.cpp` `Load_PCM()` + H-matrix | 새 H-matrix를 로드하는 형태로만 이식 가능(부호 고정, 재검증 부담 큼) |
| CPM base+shift 구조 | `ecc_data.h` `PCM_b`(base+shift) | Prime ECC도 base+circulant shift 구조라 표현 정합성은 있음 |
| compact 지수행렬 표현 | 대응 없음 | HW 라우팅/메모리 이득은 논문이 정량화 안 함 |

적용 가치: **중간** — Prime ECC와 동일한 binary QC-LDPC(base+shift) 계열이라 원리상 호환되나, 이 논문은 girth-8/최단길이 "구성법"일 뿐 Prime ECC의 고정 고rate H-matrix를 교체하려면 전면 재검증이 필요하고 디코더/정정성능 이득이 없다.
