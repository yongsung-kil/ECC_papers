### B. 알고리즘 요약

1. 대상: `(n,k)` QC-LDPC, prototype matrix `HP`의 각 원소가 `p×p` CPM 또는 zero. LBP(layered BP) 디코딩 가정, 메시지는 LLR `L[·]`.
2. 문제: 표준 PEG는 variable node를 순차 추가 → CN degree `dci`가 들쭉날쭉해 CNP(=CFU) 입력포트 최대치가 커지고, VN belief 갱신이 느려 iteration이 많다.
3. 핵심 전환: OBP-PEG는 Tanner graph를 **check node를 순차 추가**하며 build → 각 CN에 붙는 VN 수(=`dci`)를 고정값으로 만들어 CNP 입력포트를 잘 제어(최대 25% 감소).
4. BPC(belief propagation condition): Gaussian approx로 오류확률 `Pe = ½erfc(√(E(L[Qj])/2))` 유도 → `Pe`는 `E(L[Qj])`에 반비례. `E(L[Qj])` 낮은 VN을 골라 belief 강화, 수렴 가속.
5. `E(L[qji])`, `E(L[rij])`는 φ 함수(식4, tanh 기반 sum-product 규칙)로 iterative 계산 → BPC의 근거식.
6. RCC(relaxant constraint): VN degree ratio `γj = d̃vj/dvj`에 대해 `γĵ < α·min(γj)`(α=1.5) 만족하는 VN만 후보로 선택 → degree 분포 균형.
7. WCC(weight constraint): QC 확장 시 path weight `wp(i,j)=Σ(−1)^s[HP]is,js`에 대해 `(wp+[HP]i,j) mod p = 0`으로 circulant 값 결정 → girth≥6 (4-cycle 제거).
8. 절차: VN 먼저 배치→`dvj`를 DE/EXIT로 초기화→CN `i`마다 depth `l` 서브그래프 확장→RCC로 `Sc1`, BPC로 `Sc2` 후보→WCC 만족 circulant로 edge 연결.
9. 결과: `(960,480)` 등에서 PEG 대비 `Tqef` 최대 11.7%↓, fast-convergence 부호 구성 성공률 최대 10배↑, CNP 입력포트 최대 25%↓ (SNR 3.0dB, rate 0.5). BER은 WiMAX/DVB-S2 표준부호와 유사(약간 우위).
10. 한계: 순수 부호구성+시뮬만, HW 미설계. sum-product/GA 기반 분석이며 정정능력(BER) 자체는 표준부호와 동등 수준 - 이득은 iteration/CNP 복잡도.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| OBP-PEG로 QC-LDPC HP 구성 (RCC/BPC/WCC) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` + H-matrix | 부호설계 레이어 - 우리 QC-LDPC 고정, 교체 시 재검증 부담 높음 |
| CN degree 균일화 → CNP 입력포트 축소 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()` | 우리는 이미 HCU로 비대칭 degree col 처리 - degree 균일 부호면 HCU 단순화 여지, 참고가치 있음 |
| layered BP 수렴 가속 (avg iteration↓) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | avg iteration 감소는 관심 항목이나, 부호 자체를 바꿔야 얻어지는 이득이라 이식 난이도 있음 |
| WCC girth≥6 (4-cycle 제거) | 대응 없음 | 우리 PCM은 고정 girth - 교과서 수준 조건 |

적용 가치: **중간** - avg iteration·CNP 복잡도 감소라는 관심 항목을 건드리나, 그 이득이 QC-LDPC 부호 재설계에 종속돼 우리 고정 PCM 대비 재검증 부담이 크다. sum-product 기반이라 min-sum 환경 이식 시 추가 검증 필요.
