### B. 알고리즘 요약

1. 문제: PEG는 우수한 large-girth LDPC를 만들지만 단일 rate만 생성 → 실용 통신은 rate-compatible(동일 encoder/decoder 공유)이 필요.
2. 배경: 다중rate 구성법(puncturing/extending/row-split)이 있으나 row-splitting은 HW 공유엔 유리하나 최적 degree 분포 맞추려 '1' 추가/삭제 필요.
3. 핵심 아이디어: PEG 구성 과정에 index matrix `P`를 도입해 사이클 형성 edge 위치를 기록, 이를 근거로 high-rate mother code를 row-splitting해 low-rate 부호 유도(MR-PEG).
4. rate 관계: `R = 1 - L(1 - R_o)`, `L`=대체 행 수, `R_o`=mother rate.
5. 인덱스 규칙: mother matrix `H_M`의 각 행을 `L`개 행으로 분할, 새 행 index `t=(n+k) mod L`, `λ=m+t×i`, `p(i,j)=λ`로 P 채움.
6. girth 이점: depth `l`의 edge(ci,sj)와 depth `l-1`의 ci-인접 edge를 분리 배치 → girth가 기존 PEG의 `2(l+1)`보다 큼, 복잡도 증가 없음.
7. 채널/복호: BPSK+AWGN, sum-product 복호, 최대 iteration 50.
8. 구성 예: mother matrix(6×12) PEG로 생성 후 L=3로 분할해 target matrix H 도출.
9. 결과: rate 0.5/0.625/0.75/0.875, length 1024/2048에서 random row-splitting 대비 BER 0.1~0.25dB 이득(예: rate 0.5, N=1024, BER=1e-3에서 0.25dB).
10. 한계: HW 미설계, 시뮬만, 이득 소폭, 부호구조가 QC가 아닌 일반 PEG 기반.
