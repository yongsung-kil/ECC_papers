> 총평: dual-diagonal(ALT) QC-LDPC 인코더의 역행렬 회피·다중rate·저메모리 FPGA 구현 기법으로, Prime ECC가 이미 dual-diagonal 인코딩을 보유해 신규성은 낮으나 병렬 MVM·저장압축 아이디어는 인코더 최적화 참고용으로 중간 가치.

### B. 알고리즘 요약

1. 대상은 IEEE 802.16e(WIMAX) QC-LDPC 인코더, 6개 code rate(1/2~5/6), length 576~2304, base matrix rate별 상이(Table 1).
2. 문제: 기존 인코더는 단일 rate 고정이거나 rate 변경 시 재합성 필요 → rate/length 무관 단일 유연 인코더가 목표.
3. 기반: RU(Richardson-Urbanke) 인코딩에서 패리티행렬을 A,B,C,D,E,T로 분할, WIMAX는 `φ=D+ET^-1B=I`(3/4B 제외) 특성.
4. 핵심: T의 dual-diagonal 구조로 `N=T^-1M`을 역행렬 없이 back-substitution 반복식 `N_p = M_p + N_{p-1}`로 계산.
5. 파생: `E=(O,...,O,I)`라서 `P0 = CS + N_j = CS + Σ A_r S`로 단순화, `P1 = T^-1(AS+BP0)`도 동일 누산형으로 유도.
6. 복잡도: 전체 logic complexity `2(gt+kt)+t^2+(4g-1)t-g`로 code length의 선형 함수임을 표로 분석(rate 고정 시 m,g,t 상수).
7. HW: 병렬 MVM(matrix-vector multiplication) unit + 양방향(bidirectional) 연산 + storage compression으로 속도↑·메모리 bit↓, 각 path z-bit 폭.
8. 아키텍처: control module(rate별 초기조건·ROM 주소·shift값)과 5-stage encoding module, 총 latency ≈ `9i+2j+5` clock.
9. 결과: Altera Cyclone II FPGA, 최대 117~195MHz, throughput 최대 1.28Gbps(rate 1/2), 선행 serial 인코더(71.5/127.5Mbps) 대비 3~10배, Ref.[12] 대비 LE·메모리 절감.
10. 한계: 무선 WIMAX 표준 전용 인코더로 부호성능/복호와 무관, NAND용 고rate(~0.9)·4KB 구조와 rate/length 상이, BER 결과 없음.
