### B. 알고리즘 요약

1. 대상은 CDTMB(중국 지상파 DTV) 표준 QC-LDPC, code length 7493, rate 0.4/0.6/0.8, PCM은 `b=127` circulant 블록 `A_i,j`(right cyclic shift) 구조.
2. 문제: LDPC 복호기마다 BER 성능 vs 메모리/복잡도 trade-off가 달라, CDTMB 3개 rate에 어느 알고리즘이 적합한지 비교 필요.
3. 모델: AWGN 채널 + BPSK, message-passing(초기화-수평스캔-수직스캔-decision) 공통 4단계.
4. 핵심 기법: LDSPA(log-domain sum-product, `φ(x)=-log tanh(x/2)`), MSD(min-sum, CN 갱신을 `min` 근사로 대체)를 비교.
5. 식 의미: min-sum은 SPA의 `tanh`/`log` 연산을 `min`으로 근사해 HW 복잡도·연산시간을 줄이나 BER 약 0.3dB 손실.
6. 확장: SSMSD(single-scan)는 수평·수직 스캔을 `L(qij)=L(Qi)-L(rji)`로 단일 스캔에 통합해 메모리 절감, NSSMSD는 `L(rji)=λk·L(rji)`(λk<1) 정규화 계수 추가.
7. 구현 디테일: single-scan은 CN/VN 갱신 블록을 하나로 병합, 메모리를 29972B→22479B(23% 절감).
8. 최적화: 별도 학습 없음, max iteration 10~20.
9. 결과: NSSMSD가 LDSPA 대비 연산시간 최소(1464s→665s, ~35%+ 절감)이며 BER는 유사; LDSPA는 rate 0.4/0.6에서 0.3dB, 0.8에서 0.1dB 우위.
10. 한계: Matlab 소프트웨어 비교뿐, HW 미설계·gatecount/throughput 없음, 모든 알고리즘이 기존 표준 기법.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CN 갱신(MSD) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 낮음 (Prime ECC 이미 min-sum 보유) |
| normalized 계수(NSSMSD) | `decoder.cpp` (min-sum scaling) | 낮음 (normalized/scaled min-sum은 기존 기법) |
| single-scan 메모리 병합 | `decoder.cpp` `LDPC_Decoder()` | 낮음 (Prime ECC pipeline과 무관, 교과서 최적화) |
| QC-LDPC circulant PCM 로드 | `ecc_top.cpp` `Load_PCM()` | 낮음 (구조 공유하나 CDTMB 부호 특화) |
| LLR/soft-info | `channel.cpp` `Set_LLR_Th()` | 낮음 |

적용 가치: **낮음** — min-sum·single-scan·normalized min-sum은 모두 표준 교과서 기법이며 Prime ECC가 이미 보유(min-sum, 적응형 LLR)해 신규 이식 가치가 없다. HW 미설계로 아키텍처 참고도 어렵다.
