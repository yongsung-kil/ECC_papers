### B. 알고리즘 요약

1. 대상 부호: WiMAX 802.16e rate 1/2 (288,576) LDPC, model matrix `Hbm`(nb=24, mb=12) 확장(q=24~96, N=q·nb), BPSK/AWGN.
2. 문제: `Lmax`(최대 iteration)가 실제 평균 `L`보다 훨씬 커 저SNR에서 낭비, 기존 조기종료는 고SNR에서 너무 일찍 멈춰 BER 손실 발생.
3. 채널모델: `Λ_n^{[l]}=Lc·r_n+Σλ_{m,n}`, `Lc=2√(EbR)/σ²`, extrinsic `λ_{m,n}=2tanh⁻¹(∏ tanh(Λ/2))` (Sum-Product/tanh 규칙).
4. 핵심 기법(SNSC): 단일 M번째 check node만 이용한 LLR `Λ = loge[(1−P(s_{M−1}=0|r))/P(s_{M−1}=0|r)]`을 iteration마다 관찰.
5. 핵심 식: `P(s_m=0|r)=½+½∏(1−2P(c_n=1|r))` — sparse 분리된 bit node 독립가정으로 단일 check node syndrome 확률 근사.
6. 판정1: `Λ^{[l]} ≥ Tstop`(예 5)이면 성공 복호 가능성이 높다 판단해 stopping criterion을 즉시 disable → 고SNR BER 손실 회피(alert-flag 세팅).
7. 판정2: 최소 `Lmin` 이후 `Λ` 감소분이 step-threshold `T`보다 작으면 정지, `T`보다 크면(부호 무관) 계속 iterate.
8. 비교대상: AWRM([6], 임계 T1=M/26·T2·T3, CNMM/SYN 모니터링)보다 단일 check node만 써서 훨씬 저복잡도.
9. 결과: (288,576)에서 T=0.1·Tstop=5 곡선이 3dB에서 No-SC와 BER 사실상 동일, avg L은 0dB에서 <10, 3dB에서 ~5로 감소, AWRM보다 우수.
10. 한계: HW 미설계·단일 짧은 코드 시뮬만, tanh Sum-Product 전제, 다른 LDPC 코드 일반화는 미검증(future work).
