### B. 알고리즘 요약

1. 시나리오: 고이동성 위성통신(항공기·미사일 단말) - 큰 Doppler offset·동기 실패로 프레임 내 burst 데이터 손실 및 전체 packet 손실 발생.
2. 문제: 위성 지연이 커 ARQ 부적합, 손실 데이터를 수신단에서 복구해야 함 - erasure/error 혼재.
3. Burst 손실 복구: 손실 비트의 채널 초기정보 `L(P_i)`를 실제값 대신 `0`으로 세팅(0/1 등확률 가정) 후 반복 BP 복호로 사후확률 갱신 (Eq.2).
4. 핵심 근거: 손실 데이터의 soft-info는 신뢰불가하므로 중립(LLR=0)으로 두면 나머지 신뢰 비트로 반복복호가 복원 - burst 길이 100 이하까지 복구 가능(regular LDPC N=1008, dc=6, dv=3).
5. Packet 복구: memoryless→memory 채널 전환을 위한 group coding - `n`블록 precoding(N1,R1) 후 `m`개 sub-block으로 partition·재배열, 2차 channel coding(N2,R2), 전체 rate `R1·R2`.
6. sub-block 수 제약: LDPC erasure threshold 0.5 근거로 최소 3개, `m`은 3~N2 범위.
7. BEC 복호: IT(반복 message-passing, XOR·선형시간, stopping set 취약) vs ML(GF(2) 선형방정식을 pivoting/approx lower-triangulation+GE로 해결, stopping set 무관).
8. threshold: IT `≈0.42`, ML `≈0.48` erasure 확률 - ML이 stopping set 영향 없어 우수.
9. 결과: 최대복구 패킷 `Nmax` 공식화(Eq.5), 블록수 60이면 erasure 0.35에서도 복구확률 0.95↑, ML이 큰 `m`에서 IT보다 많은 패킷 복구.
10. 한계: HW 미설계(시뮬만), rate 미명시, 위성 erasure 복구 시스템 전용 - NAND 저장 ECC와 무관.

| BEC IT 반복복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | Prime ECC는 min-sum(AWGN/RBER)이고 BEC erasure 전용 IT는 대상 다름 |
| ML(GF(2) 소거) 복호 | 대응 없음 | Prime ECC는 iterative min-sum HW 디코더, GE/pivoting ML 미보유 |
| group coding / packet partition | 대응 없음 | 시스템 레벨 패킷 복구, ECC 부호설계 계층 밖 |
