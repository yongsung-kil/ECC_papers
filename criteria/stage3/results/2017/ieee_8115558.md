### B. 알고리즘 요약

1. 대상: regular (wc=3, wr=6) rate-1/2 mother LDPC(`n=2000`)를 펑처링해 rate-2/3까지 올리는 rate-compatible 부호. BPSK/AWGN, BP(SPA) soft decision.
2. 문제: 채널 상태별 rate 적응을 위해 encoder/decoder 쌍을 여러 개 두는 대신, 단일 mother code 펑처링으로 저비용 rate 유연성 확보 필요.
3. 모델: 펑처링 = Tanner graph의 VN 제거. degree-`k` 비트 펑처링 시 `k`개 parity 방정식 중 하나를 나머지에 더해 소거 → 등가 parity check matrix 도출([9]).
4. 핵심 기법 1단: 펑처링된 irregular 부호의 message density를 Gaussian 근사(DE)로 근사, mean만으로 추적. VN 출력 `m_v,i = m_u0 + (i-1)m_u(l-1)`, CN 업데이트는 `φ`/`φ^-1` 사용.
5. 식 의미: 큰 `t`에서 `Δt = f(s,t)-t = (i1-2)t + qΓ`. 기울기 `i1-2`(첫 non-zero `λ` 인덱스)와 상수 `qΓ`(threshold `s` 포함)가 성능 지표.
6. 핵심 기법 2단: threshold `s(Γ)`는 quadratic degeneracy로 `f(s,t)=t, f_t(s,t)=1` 연립을 풀어 산출([11]).
7. 구현 디테일: 증분 알고리즘 — 매 iteration마다 한 비트 위치만 추가 펑처링하여 `Δt` 최대화, 한번 선택 시 유지 → 자동으로 rate-compatible 계층 생성. `φ`의 elementary 근사 `φ~(x)=e^{αx^γ+β}` (α=-0.4527, β=0.0218, γ=0.86).
8. 학습/최적화: 없음(greedy 증분 최적화). exhaustive 탐색은 짧은 length만 가능.
9. 결과: rate-2/3 (n=2000, p=500 펑처) 제안 패턴이 random 펑처링 대비 우수, BLER는 same-rate native (1500) 부호에 근접.
10. 한계: 순수 설계 이론, HW 미설계, BER/BLER 곡선 수치 없음(그림만), regular mother code 한정 예제.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 펑처링 패턴/등가 H 설계 | ecc_top.cpp Load_PCM() | 우리는 rate 고정·비펑처 구조, 대응 약함 |
| 밀도진화 threshold 평가 | 대응 없음 (코드는 bit-exact 시뮬, DE 미탑재) | 부호설계 오프라인 도구 성격 |
| BP/SPA soft decoding | decoder.cpp LDPC_Decoder() (우리는 min-sum) | 알고리즘 계열 다름 |
```

적용 가치: **낮음** — rate-compatible 펑처링 설계 이론으로, rate가 고정된 NAND용 고정 QC-LDPC에는 펑처링 유즈케이스가 없고 DE 도구도 우리 bit-exact 시뮬과 무관하다.
