### B. 알고리즘 요약

1. 대상: binary LDPC의 ADMM-LP(penalized) 복호, AWGN·BPSK, regular C1 `(2640,dc=6)`·irregular C2 `(576,dc=6/7)`·rate 3/4 C3 `(1000,dc=12)`, rate 0.5~0.75.
2. 문제: ADMM-LP 복호에서 check polytope로의 Euclidean projection `PPd(v)`이 계산 병목 - 이를 LUT로 대체.
3. 모델: LP 완화 후 ADMM 반복 `x/z/λ` 업데이트(식4-6), `z_j` 갱신이 projection `PPd`.
4. 핵심기법1: projection과 self-map(permutation `σ`, componentwise flip `φV`)의 교환성 증명(Prop.1,2) - `PPd[σ(v)]=σ(PPd(v))`, `PPd[φV(v)]=φV(PPd(v))`.
5. 핵심식: 이 교환성으로 LUT 항목을 permutation은 오름차순 대표만, `|V|` 짝수 flip은 대칭 항목 제거 - 테이블 크기 `L^d`에서 지수적 축소(하한 `L^d/2^(d-1)`).
6. 핵심기법2: 입력 양자화 범위 `[a,b]`가 작아도(96%+ 입력 포함) projection 불변 증명(Prop.3), 대칭 양자화 `a+b=1`로 all-zero 가정 유지(Prop.4).
7. 구현디테일: 비선형 주소 LUT 검색을 위한 보조 LUT 기반 Algorithm 2, 8-bit unsigned 정수 저장으로 메모리 1/4.
8. CN분해: 고차수 CN(dc=12)을 degree-3/6/7 저차 projection들로 분해해 소수 d의 LUT만으로 구현(메모리 대폭 절감).
9. 결과: [15] 대비 복호시간 24~39% 절감, FER은 원 ADMM과 동등(적정 범위시), NMS 대비 0.1dB+ 이득.
10. 한계: HW 미설계(C++ 3.2GHz desktop 시뮬만), LUT 메모리 폭증(dc=12시 1.16억 실수), 수렴 느림·양자화시 error floor 발생 가능.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP projection LUT 복호 | 대응 없음 | Prime ECC는 min-sum message passing 복호로 LP/ADMM projection 연산 자체가 없음 |
| CN 분해(고차수→저차 projection) | 대응 없음 | min-sum CN 연산(`CNU_Update_New_Mag()`)과 개념 상이 |
| 입력 양자화·대칭 LUT | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Get_VNU_Table_Idx()` (개념만) | Prime ECC도 LLR 양자화 테이블 있으나 projection이 아닌 min-sum magnitude용, 직접 이식 불가 |

적용 가치: **낮음** — ADMM-LP 복호는 Prime ECC의 min-sum message passing과 근본적으로 다른 프레임워크로, projection LUT 기법이 적용될 지점이 없음.
