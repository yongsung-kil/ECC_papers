### B. 알고리즘 요약

1. 대상: m-sequence(primitive 다항식 `p(x)`, 주기 `N=2^r-1`) 검출·복호. 예시 `p(x)=1+x^2+x^5+x^8+x^11`, N=2047(M=1023까지 절단 실험).
2. 문제: 잡음만 입력 시 디코더가 codeword로 수렴하면 false alarm(PFA) 발생 → serial-search 획득시간 급증. K개 reference parity-check 등식 선택이 PFA를 0.1~10^-6까지 좌우.
3. 모델: 수신 `R(i)=(-1)^y(i)+w(i)`(H1) 또는 `w(i)`(H0), MS로 복호해 valid codeword 발견 시 H1 판정. dual code는 Hamming code, weight `t=3` trinomial만 사용.
4. 핵심 관찰: absorbing set(LDPC error-floor 원인)이 잡음 입력 시 디코더를 blocking해 false alarm을 방지하는 유익한 역할.
5. absorbing set 구성: 순환 trinomial `g_a=1+x^{i_a}+x^{r_a}`의 길이-6 cycle이 (3,3) fully absorbing set(조건 `r_a≠3i_a, r_a≠3i_a/2`), K 증가 시 Thm1/2로 크기 지수증가 `a_{2Q}=2^{Q-1}3^{Q+1}`.
6. hybrid cycle: 서로 다른 elementary 행렬의 CN을 잇는 cycle이 absorbing set을 파괴 → false alarm 증가. 이를 최소화하는 것이 목표.
7. 카운팅: Halford-Chugg 방법으로 `tr(A^3)`, `tr(A^4)` 기반 길이-6/8 cycle 수 I6(식12)·I8(식13) 해석식 유도, 하한 `I6,min=NK`, `I8,min=9NK(K-1)/2`.
8. 선택 알고리즘(Alg.1): 길이-4 cycle 없는 trinomial K개 중 `I6=I6,min` 만족하며 `I8-I8,min` 최소화하는 조합 탐색(Ns 반복 제한).
9. 결과: K=5, m-sequence에서 PFA를 0.77->10^-6로 감소, wrong detection도 유사 감소. 단 correct detection SNR은 약 2.5dB 악화(config a->g).
10. 한계: HW 미설계, 시뮬만, 검출용 목적(정정능력을 일부러 낮춤), full absorbing set 증명 미완, t>3 확장 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| absorbing set / hybrid cycle 이론 | 대응 없음 | Prime ECC는 고정 QC-LDPC H-matrix 사용, absorbing set을 보존하는 방향(정정능력↓)은 ECC 목표와 반대 |
| 짧은 사이클(I6/I8) 카운팅 해석식 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 설계 지표로만 간접 참고 가능(부호구조 로드 단계) |
| MS 디코더 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 동일 min-sum이나 검출용이라 신규성 없음 |

적용 가치: 낮음 — 그래프 사이클 카운팅 이론은 부호설계 참고 여지가 있으나, 본 논문 목표(absorbing set 보존으로 false alarm 억제, 정정능력 저하)가 NAND ECC와 상반되어 직접 이식 가치는 낮음.
