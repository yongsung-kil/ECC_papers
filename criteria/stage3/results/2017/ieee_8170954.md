### B. 알고리즘 요약

1. 대상은 AWGN 채널 BICM 시스템, IEEE 802.11n binary irregular QC-LDPC(`N=1944`, rate 1/2·2/3·3/4)와 Gray-labeled 256-QAM 고차변조.
2. 문제: 기존 interleaver 설계는 DE/EXIT가 VN-CN 연결관계를 무시해 특정 코드의 threshold를 부정확하게 예측, 시뮬레이션과 mismatch 발생.
3. 모델: coded bit를 신뢰도 레벨 `l`과 VN degree `g`로 기술하는 mapping distribution `m(x)=Σ ml,g x^(gl)`(`Eq.1`), 제약 `Σ ml,g Λg = 1/t`.
4. 핵심기법: demodulator EXIT와 QC-LDPC PEXIT를 결합한 수정 PEXIT 분석으로 base matrix 연결관계를 반영해 threshold `(Eb/N0)*`를 예측.
5. 의미: base graph를 1-copy protograph로 보고 VN↔CN MI 전달(`Eq.4~6`)을 정확히 추적해 DE보다 저복잡도로 정밀 threshold 산출.
6. 확장: mapping distribution 제약을 `t+e` 방정식으로 변환, 자유도 `NP=t×e−T` 벡터를 차분진화(DE algorithm, factor `F∈[0,2]`)로 최소 threshold mapping 탐색.
7. 구현: 비용함수가 수정 PEXIT, 이분법으로 threshold 정밀화, local optimum은 v 교란/고차 DE로 회피.
8. 학습/최적화: differential evolution 반복으로 mapping vector 갱신(더 낮은 threshold면 채택).
9. 결과: R=1/2 256QAM에서 even mapping 대비 약 0.5dB gain, 수치예측이 시뮬과 일치, R=2/3·3/4는 0.1~0.2dB로 이득 감소(고rate일수록 VN degree 균일화로 이득 소멸).
10. 한계: HW 미설계·시뮬만, error-floor는 최적화 대상 아님, 고rate/고차변조에서 interleaver 이득이 미미하고 delay·memory 비용 부담.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC base matrix / PEXIT 구조 | `ecc_top.cpp` `Load_PCM()` | base matrix 구조 반영 사상만 개념 공유, 성상 mapping은 무관 |
| BICM interleaver / 성상 labeling mapping | 대응 없음 | NAND는 고차 성상 변조·bit-to-symbol mapping이 없어 대응 모듈 부재 |
| soft-demapper LLR / MI 전달 | `channel.cpp` `Set_LLR_Th()` | LLR 산출 위치는 있으나 QAM demapper 전용이라 직접 대응 아님 |
| SPA(sum-product) 복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 채택, full-SPA는 대응 없음(§6 표) |
| degree distribution/threshold 최적화 | 대응 없음 | 오프라인 설계 도구, 코드베이스 직접 대응 없음 |

적용 가치: **낮음** — 고차 QAM BICM interleaver mapping 최적화로 NAND binary LDPC ECC에는 성상 labeling 개념 자체가 없어 이식 대상이 없고, PEXIT threshold 도구만 부수적 참고 가치.
