### B. 알고리즘 요약

1. 시스템: coherent 광통신용 rate-adaptive LDPC-coded modulation을 FPGA(Xilinx xc6vsx475t)로 에뮬레이션, BPSK~64-QAM 지원.
2. 문제: 400Gb/s+ 광채널의 시변 조건에 맞춰 code rate와 변조차수를 재구성 가능한 통합 FEC 엔진이 필요.
3. 부호: mother LDPC `(3,15)`, `n=34635, k=27710`(rate~0.8), circulant permutation matrix 기반 QC-LDPC.
4. 핵심기법: layered scaled min-sum, scaling factor `sf=0.75`, 식(4)(5)로 VN 갱신, 식(6)으로 CN(min1/min2) 갱신.
5. rate-adaptation: mother code에서 일부 block의 초기 LLR을 최대정수로 세팅(shortening) 또는 puncturing으로 rate `{0.8,0.786,...,0.7}` 생성.
6. CNU 구조: ABS→sign XOR→binary tree로 min1 탐색→survivor traceback으로 min2, 재구성; 메모리 사용 대폭 절감 트릭.
7. 구현: 8-bit uniform 양자화(에러플로어가 유한정밀도 아닌 부호설계 자체 기인함을 보장), max*→max 근사.
8. UEP: 16/64-QAM의 비트별 보호 불균형 이용, 비트쌍마다 rate 0.7/0.8 차등 배정(평균 0.75 유지).
9. 결과: `Imax=45`(고SNR ~24회 수렴), NCG 11.83~12.25 dB, UEP로 추가 0.5 dB, BER 1e-15까지 error floor 없음, throughput 2.11~5.94 Gbit/s.
10. 한계: HW=FPGA 에뮬레이션(ASIC 아님), gate count 미제시, 광채널·고차변조 특화라 NAND hard/soft-read 시나리오와 거리.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered scaled min-sum CNU(min1/min2, sign XOR) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리 min-sum과 중복, scaling factor만 참고 |
| VNU / 8bit 양자화 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | soft 8bit이라 우리 hard/2~3bit와 정합 필요 |
| shortening/puncturing rate-adaptation | ecc_top.cpp Load_PCM() | 고정 H-matrix 기반이라 rate 가변화는 재설계 부담 |
| QC-LDPC permutation matrix 구조 로드 | ecc_top.cpp Load_PCM() | 부호구조 참고 가능하나 특정 부호 고정 |
| M-QAM symbol/bit LLR 계산 | 대응 없음 | 변조 매핑, NAND에 무관 |
| early termination unit(ETU) | partial_CRC.cpp / full_CRC.cpp | 조기종료 이미 보유(CRC 방식 상이) |
```

적용 가치: **낮음** — layered scaled min-sum·8bit 양자화는 이미 보유 기법과 중복이고, 광통신 coherent 채널·고차변조·NCG 지표에 특화되어 NAND binary QC-LDPC로의 직접 이식 가치가 낮다.
