### B. 알고리즘 요약

1. 대상은 `(3,6)`-regular SC-LDPC(=LDPC convolutional) 부호, 결합길이 `L`, AWGN 채널, BP(sum-product) 복호.
2. 문제: 큰 `L`에서 block 복호는 latency/complexity가 폭증 → 지연 독립적인 slot 크기 `W`의 슬라이딩 윈도우 복호를 쓰지만, 윈도우 내 uniform serial 스케줄은 block 대비 기대한 2배 복잡도 절감을 못 준다.
3. 관찰: 윈도우 좌측(저차수 CN 연결) 노드는 빨리 수렴하고 우측 노드는 iteration이 지나도 `Pb`가 거의 안 변함(Fig.4) → 불필요 갱신 다수.
4. 핵심기법1(추정 BER 기반 비균일): 각 위치 `w`의 soft-BER `P̂b(w)=E[Z]`, `Z(k)=1/(1+e^{|Lout(k)|})`를 매 iteration 계산.
5. 갱신 규칙: `P̂b^{I+1}(w) ≤ θ·P̂b^{I}(w)`(분수 개선 `θ`)를 만족하는 위치만 다음 iteration에 갱신, 나머지는 `updateList[w]=false`로 꺼서 갱신 생략.
6. 확장(force update): 한 번 꺼진 노드는 영구 제외되어 재시작이 필요할 수 있어, `FU` iteration 동안 미갱신 위치를 강제 갱신 → parallel 스케줄 iteration 약 50% 감소.
7. 최적 파라미터: `θ=0.99`, `FU=W` (DE로 탐색; Fig.7).
8. Pragmatic 스케줄: soft-BER 계산 없이, 주기 `T=W`로 `wmax(i)=W-i+1`로 갱신 범위를 매 iteration 1씩 줄였다가 재초기화하는 고정 시간불변 스케줄.
9. 결과: DE 및 유한길이(PEG lifting `N=500`, `L=100`, `W=8`) 시뮬에서 BER 손실 없이 복잡도(`Uavg`) 절감 — non-uniform serial+FU가 uniform parallel 대비 약 50%, pragmatic serial(`Uavg=130`)은 uniform parallel(`Uavg=300`) 대비 55% 감소.
10. 한계: HW 미설계, 순수 복잡도(노드 갱신 수) 절감으로 정정능력/error-floor 개선은 없음, SC-LDPC/윈도우 복호 전용 가정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 이터레이션/노드 갱신 스케줄 | decoder.cpp LDPC_Decoder() | 낮음: Prime은 block QC-LDPC라 윈도우/위치 개념 없음, 스케줄 구조 상이 |
| soft-BER 기반 조기중단·갱신 취사선택 | partial_CRC.cpp partial_crc_HW_T4() | 낮음: Prime은 CRC 기반 조기종료를 이미 보유, soft-BER 지표는 별개 |
| SC-LDPC 윈도우 부호구조 | 대응 없음 | Prime은 고정 block QC-LDPC(dual-diagonal), SC 구조 미지원 |
```

적용 가치: 낮음 — SC-LDPC 윈도우 복호 전용 스케줄로 block QC-LDPC 기반 Prime ECC와 부호/복호 구조가 근본적으로 다르고, 얻는 것도 성능이 아닌 복잡도 절감뿐이라 이식 가치가 낮다.
