### B. 알고리즘 요약

1. 시스템: TDD 무선 채널 상호성을 이용한 물리계층 키 생성, Slepian-Wolf syndrome 방식 정합. rate 1/2, key length 16384, 최대 iteration 100.
2. 문제: 채널 특성이 Alice/Bob 간 완전 일치하지 않아 키에 오류 발생 → LLR-BP 복호는 check node 갱신 복잡도 높고 짧은 loop에서 수렴 실패, 정합 효율 저하.
3. 가정: 정보/parity 비트가 표준 AWGN 채널(분산 `σ`)을 통과, Bob은 가우시안 pdf만 고려. Alice/Bob의 분산 `σ_A=σ_B` 근사.
4. 핵심 기법 1단: intrinsic LLR을 물리계층 키 정합용으로 보정 — 세 분산 `σ, σ_A, σ_B`를 고려해 pdf 재정의, 이중적분 형태 intrinsic LLR `(10)` 사용.
5. 식 의미: 무선 채널로 얻은 `L(q)`가 매우 작아 표준 `tanh` 기반 `L(r)` 계산 시 오류가 커짐 → 보정 필요.
6. 핵심 기법 2단: check node 갱신을 offset-base 공식 `L(r)=∏sgn·min(|L(q)|)−β`로 대체, 본 논문은 `β=0` 채택 → syndrome 항 포함 `(15)`로 단순화.
7. 구현 디테일: RMP(순차 스케줄) 채택 — 갱신된 메시지를 즉시 재사용해 SMP 대비 빠른 수렴, 동일 복잡도.
8. 학습/최적화: 없음(β는 density evolution으로 계산 가능하나 0으로 고정).
9. 결과: SNR>13dB에서 BER=0, 초기 error rate 0.065에서 성공률 ~100%, error rate 0.06에서 iteration 약 3회로 감소.
10. 한계: MATLAB 시뮬만, HW 미설계, 물리계층 키 채널 모델 특화, offset β=0으로 실질 표준 min-sum과 동일.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset-base min-sum check node 갱신 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum은 이미 보유, offset(β) 도입만 차이이나 β=0이라 기여 미미 |
| RMP 순차 스케줄 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | Dual-Update 유사 스케줄 이미 보유 |
| intrinsic LLR 채널 보정 | channel.cpp Set_R_Offset(), Set_LLR_Th() | 무선 키정합 채널 특화라 NAND soft-read와 부정합 |
```

적용 가치: **낮음** — offset min-sum(β=0)·RMP 모두 Prime ECC가 이미 보유, 채널 모델이 물리계층 키 정합 특화라 NAND binary LDPC에 이식 이점 없음.
