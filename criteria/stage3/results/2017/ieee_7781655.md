### B. 알고리즘 요약

1. 대상은 IEEE 1901 표준의 time-varying periodic **LDPC-CC**(convolutional), 4개 rate, 주기 `T=3`, 최대 rate 4/5, encoding-memory depth 최대 `M=226`.
2. 문제: 기존 memory-based LDPC-CC 디코더는 unit processor마다 별도 memory bank 다수를 써 전력 소모가 큼(동일 용량 통합 메모리 대비).
3. 채널/디코딩 모델: layered decoding + on-demand variable-node activation, **normalized min-sum(NMS)** 기반, APP·C2V 메시지를 SRAM에 저장.
4. 핵심 기법: `I`개 unit processor의 memory-access 패턴을 수식화(read address `AR`, write address `AW`)하여 뱅크 병합 조건 도출.
5. Theorem 1: sliding-window 크기 `S`가 3의 배수이면 모든 processor의 메모리 접근이 동기화됨(주소 일치).
6. Theorem 2: `S > M + (td+2)/3`이면 data hazard(다른 processor가 갱신 전 LLR read) 없음 → 여기서 `S=228`(3의 배수, M=226) 채택.
7. 구현 디테일: unit processor 3-pipeline, 5-parallel(rate 4/5 지원), 7-input tree로 first/second minima 탐색(presorting stage를 1단으로 이동해 동작주파수↑), C2V는 30-bit vector(두 최소 magnitude+index+sign)로 압축.
8. 통합 결과: `I×b` APP 뱅크를 `b`개(폭 I배)로, `I-1`개 C2V 뱅크를 단일 single-port SRAM으로 병합.
9. 결과: 65nm CMOS, 87.5k gates + 152.8k bit, 375.9MHz, 501.2Mb/s, 기존 분리 뱅크 대비 전력 **최대 40% 절감**, 에너지효율 우위.
10. 한계: IEEE 1901 LDPC-CC 전용 도출(다른 코드엔 다른 window 제약 필요), 정정능력 개선 없음(전력만), BER 곡선 본문 미제시.

| first/second minima tree 탐색 | `decoder.cpp` `C2V_Cal()` | min1/min2 탐색은 이미 구현 |
