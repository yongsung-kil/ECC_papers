### B. 알고리즘 요약

1. 시스템: AWGN + BPSK 채널, (16,8) 부호를 "LDPC"로 칭하고 Hamming(7,4)·BCH(15,7)과 BER 비교.
2. 문제: LDPC의 message passing 반복 복호가 복잡하고 error-floor가 있다는 점을 배경으로 언급.
3. 핵심 알고리즘 MWD(Minimum Weight Decoding): 수신어 `R(x)`를 생성다항식 `G(x)`로 나눠 syndrome 산출.
4. syndrome 무게 `W`를 부호의 정정능력 `t = (d−1)/2`와 비교.
5. `W ≤ t`이면 syndrome 자체를 error pattern으로 확정, 아니면 `G(x)`를 순환 시프트 후 재나눗셈 반복.
6. 종료: `W ≤ t`가 될 때까지 순환 시프트/나눗셈 반복 → error pattern을 수신어와 XOR해 정정.
7. 이는 사실상 cyclic block code의 error-trapping 방식(참고문헌 4에서 modified error trapping으로 명시)이며 LDPC의 VN/CN BP와 무관.
8. 예시: T=(1111...), R=(1100110111111111), G=(1011)로 순환 나눗셈 진행해 W=3까지 감소, E(x) 복원.
9. 결과: Fig 5/6에서 (16,8) 부호가 Hamming/BCH보다 BER 우수하다고 정성 서술 (곡선 수치는 그림뿐, 본문에 없음 → 미상).
10. 한계: HW 없음, MATLAB BER 곡선뿐, (16,8) 초소형 부호, high-rate QC-LDPC·soft-read·반복복호 전무.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MWD syndrome 복호 루프 | 대응 없음 | Prime ECC는 min-sum message passing(`LDPC_Decoder()`), error-trapping 방식 미사용 |
| 정정능력 `t=(d−1)/2` / syndrome | `decoder.cpp` `LDPC_Decoder()` (개념상 syndrome 체크만 유사) | Prime ECC는 CRC 조기종료 사용, MWD와 알고리즘 무관 |
| BER 비교 (Hamming/BCH) | 대응 없음 | 부호 클래스 자체가 상이 |

적용 가치: **낮음** — MWD는 LDPC BP가 아닌 cyclic block code error-trapping이며 (16,8) 장난감 부호·시뮬뿐이라 Prime ECC 이식 근거가 없다.
