### B. 알고리즘 요약

1. 시스템: circular SC-LDPC 앙상블 `CC(l,r,L,w)`, 예제 `(3,6,30,3)`, 채널=BEC, 복호=BP, 유한길이 `M=1000`/position.
2. 문제: SC-LDPC는 reliable position의 shortened(known) 변수노드로 wave-like 신뢰전파를 얻지만 그만큼 rate-loss 발생.
3. 기존 한계: shortened 대신 reliable variable node를 추가해 DE로 degree분포 최적화하면 BP threshold는 유지되나 유한길이 성능이 열화됨.
4. 핵심기법1: reliable position에 irregular degree분포 `λr(x)=αx^{d1-1}+(1-α)x^{d2-1}` 변수노드 `Mr`개 배치, edge제약 `Mr/∫λr = Ml`로 평균차수 `lavg` 최소화가 rate-loss 최소화(식 2).
5. 핵심식: `RSCR = 1 - l/r - (l/r)(w-1)/(L + (w-1)(1-l/lavg))` — `lavg`↓이면 rate-loss↓.
6. 핵심기법2: DE 대신 expected graph evolution(식 3,4)로 유한길이 성능 예측 — degree-one check node 비율 `r̂1(τ)` 궤적 분석.
7. 설계규칙: `r̂1(τ)`가 critical phase 값보다 낮은 local minimum을 갖지 않도록 `κ`/`λr(x)`를 정하는 것이 유한길이 성능 유지 조건(2nd design rule).
8. 최적화: nearly-regular 제약 하 brute-force로 `(d1,d2,α)` 탐색, `λr,opt2(x)=0.24x^2+0.76x^5` 도출.
9. 결과: `CSCR(3,6,30,3,λr,opt2)` 설계율 0.4878로 rate-loss 60% 저감하면서 block error rate 성능은 원본 SC-LDPC와 동등.
10. 한계: BEC 가정, BP 시뮬만, HW/실채널 미검증, QC 구조나 AWGN/NAND 채널 미확장.

### C. Prime ECC 관련 모듈 핀포인트

대상이 `code-design`이라 C섹션 성능 항목은 전부 `N/A`. 구성 요소 대응만 참고로 기재:

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 앙상블/degree분포 구성 | 대응 없음 | 우리는 고정 binary QC-LDPC(base+circulant), SC-LDPC 앙상블 구조 부재 |
| expected graph evolution 유한길이 분석 | 대응 없음 | 코드 설계 분석 도구, 시뮬레이터 내 대응 함수 없음 |
| H-matrix/부호구조 로드 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 만약 부호 도입한다면 로드 지점이나 SC 구조 자체가 이질적 |

적용 가치: **낮음** — SC-LDPC 앙상블의 rate-loss를 줄이는 이론적 구성/분석 논문으로, 고정 QC-LDPC를 쓰는 Prime ECC의 부호설계(높은 재검증 부담)에도 직접 매핑되지 않고 디코더 성능 개선과도 무관.
