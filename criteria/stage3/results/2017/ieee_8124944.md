> 총평: min-sum 메시지를 저정밀 도메인으로 압축(NS-FAID)해 layered unrolled 디코더 메모리/라우팅 비용을 줄이는 HW 기법 — soft 메시지 양자화 절감 관점에서 Prime ECC 디코더에 참고가치 있으나 array code·FPGA 한정.

### B. 알고리즘 요약

1. 대상: array QC-LDPC, (3,12) regular, rate 3/4, code size 1296 bits, z=108, 3 layer unrolled — 목표 수십 Gbps.
2. 문제: unrolled layered 디코더에서 다중 codeword 동시처리로 throughput은 오르나 β/α 메시지 메모리·라우팅 오버헤드가 커짐.
3. 모델: layered Min-Sum, 메시지 `γ`(채널 LLR), `β`(C2V), `α`(V2C), `γ̃`(AP-LLR); `β,α`는 q비트, `γ̃`는 p≥q비트 양자화.
4. 핵심 기법: NS-FAID — VN/AP-LLR은 고정밀(q), CN 처리는 저정밀(w<q) 두 도메인 운영, framing(압축)·de-framing(복원) LUT로 도메인 교차.
5. 식 의미: `β` 메모리·라우팅을 낮은 w에 맞춰 크기 축소 → 저장/배선 비용 감소, framing/de-framing으로 정정능력 손실 최소화.
6. 확장: framing 함수 `F`(예 w=3 `[1,1,1,3,3,4,4,7]`)와 gain factor `µ`를 DE로 최적화, w=3 best는 MS 대비 DE threshold가 낮아(4.24 vs 4.326) 성능 개선.
7. HW 디테일: barrel-shifter 라우팅 제거(hardwired interconnect), β는 sign+min1+min2+idx 압축저장(SRL depth=nl), de-framing과 SM→C2 변환을 단일 LUT로 통합해 critical path 단축, AP-LLR 메모리 제거.
8. 최적화: density evolution으로 best framing/de-framing 함수와 gain factor 도출.
9. 결과: Virtex-7 합성서 MS 대비 logic 최대 30% 절감, TAR 25~125% 개선, throughput 최대 48.4 Gbps(4 iter, 3 codeword, FER<10^-4), w=3은 0.1dB 개선/w=2는 0.5dB 손실.
10. 한계: FPGA 합성 추정만(ASIC/실측 없음), 특정 (3,12) array code, NAND 직접 실증 없음, w=2는 성능 열화.
