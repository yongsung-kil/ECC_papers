### B. 알고리즘 요약

1. 대상: Mackay `(96,48)`, `(408,204)` binary LDPC, AWGN/BSC 채널, min-sum 복호(식 3~5).
2. 문제: LDPC는 SNR↑ 시 trapping set이 유발하는 error floor에 갇힘. 기존은 trapping set을 직접 열거(NP-hard)해 CN 제거/데이터 shortening으로 대응.
3. 아이디어: trapping set을 직접 찾지 않고 Monte Carlo로 각 비트의 오류빈도 리스트 `E=(E1..EN)`를 만들어 간접 위치추정.
4. Algorithm 1(bit-search): Eb/N0=4dB에서 10만~50만회 복호, 비트 j 오복호 시 `Ej++` 누적해 오류리스트 생성.
5. Algorithm 2(CN-addition): `E` 내림차순 정렬, 상위 오류 VN `Ai`와 하위(최고신뢰) VN들 `Bi`를 새 CN `Cadd`로 연결. regular 부호는 오류 VN당 `(dc-1)`개 신뢰 VN 연결.
6. 핵심 근거: 추가 CN은 vr의 새 degree-one CN이 되어 computation tree separation `k`를 키움 → Lemma 1에 의해 큰 separation이면 오류 VN이 복원됨(식 6~9).
7. 효과: `(a,b)` trapping set이 덜 유해한 `(a-1, b+dv)` 또는 `(a-1, b+dv-2)`로 변환.
8. 트레이드오프: CN 추가로 rate loss (1개 추가 시 0.5→0.4896, 2개 시 0.4792). CN 개수는 성능/rate 균형으로 결정.
9. 결과: (96,48) AWGN 7dB에서 BER `1.5e-8 → 8.9e-9(1CN) → 3e-9(2CN)`, error floor가 1e-10에서도 미출현. BSC crossover 6e-4에서 `1.2e-8 → 3.8e-9 → 8.6e-10`.
10. 한계: 저rate 비구조 Mackay 부호 시뮬만, HW/복잡도/throughput 미제시, QC-LDPC 구조·rate loss 정합성 불명.
