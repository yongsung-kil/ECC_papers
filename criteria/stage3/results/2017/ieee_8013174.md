### B. 알고리즘 요약

1. 시스템: NAND SSD 컨트롤러(FTL·ECC·data-path protection) 아키텍처와 SLC/MLC/TLC/QLC 셀 구조, page/block/superpage 조직 개관.
2. 문제: 공정 미세화·다치화로 Vth 마진 축소 → RBER 상승, P/E cycle에 따라 ECC 정정한계 초과 시 데이터 손실.
3. 채널/모델: 플래시 셀을 통신 채널로 모델링, P/E cycling 노이즈를 AWGN(state별 Gaussian, `μ`,`σ`)으로 근사.
4. ECC 흐름: BCH(hard, 단일 read)와 LDPC(hard→soft multi-level→superpage parity) 3단계 복호 흐름 (Algorithm 1) 제시.
5. LLR: soft read는 여러 Vref로 셀을 Vth 구간(R0~Rx)에 binning 후 구간별 LLR 부여. `LLR=log(P(x=0|Vth)/P(x=1|Vth))`, AWGN 근사식 (11) `LLR(y)=(μ1^2-μ0^2)/2σ^2 + y(μ0-μ1)/σ^2`.
6. Vref 최적화: read-retry로 분포 shift 보상, `Vopt` 학습(주기적), disparity 기반 `Vinitial` 근사로 최적 read reference voltage 추정.
7. LLR 정확도 개선안 소개: nonlinear 변환, scaling/rounding, 실패 read의 ECC 정보 활용, online/offline training으로 LLR 테이블 empirical 결정.
8. 소프트 레벨 tradeoff: level 추가마다 정정력↑이나 read 추가로 latency↑ (5-level ≈ 480μs, codeword failure 5자릿수↓ vs superpage parity ≈ 10ms).
9. 결과(실측 기반): rate≈0.936 soft LDPC가 RBER 5.0e-3까지 UBER 1e-16 유지(BCH rate 0.935는 1.0e-3), 약 0.28% 추가 read로 BCH 대비 정정력 2배.
10. 한계: 서베이(신규 알고리즘·HW 미제시), 수치는 인용/실측 그림 기반이라 구조·gatecount 등 없음, soft read는 정정력↑ 대가로 read latency 악화.

### C. Prime ECC 관련 모듈 핀포인트

| 구간별 LLR 값(AWGN 모델, 테이블) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | R0~Rx binning→LLR 매핑이 iteration/구간별 LLR 테이블 개념과 대응 |
| LDPC hard/soft 복호 흐름 | `decoder.cpp` `LDPC_Decoder()`, `VN_Cal_HD()` 등(HD/2SD/3SD) | HD→soft 단계별 복호 흐름이 Prime ECC HD/2SD/3SD 구조와 대응(개념적) |
