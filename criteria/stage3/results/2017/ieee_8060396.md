### B. 알고리즘 요약

1. 시스템: MLC NAND SSD, LDPC는 progressive read-retry(hard→soft sensing)로 복호, retention 길수록 RBER↑ → soft read 증가로 read latency 누적.
2. 문제: retention 오래된 old data는 soft-sensing read-retry가 많아 flash read 성능이 크게 저하(특히 read-intensive late-lifetime).
3. 관찰: read-hot 데이터는 소량이나 자주 접근됨 → 이들만 refresh하면 soft read를 hard read로 리셋해 큰 성능 이득.
4. 핵심기법: LDR(Lightweight Data Refresh) — decision tree로 soft-sensing + LRU hotness>T 페이지를 read-hot으로 식별.
5. 판정: 3개 decision node(sensing level soft? / LRU 존재? / hotness>T?)로 refresh 후보 선정, package별 refresh queue에 삽입.
6. refresh 동작: page-level로 error-free 데이터를 새 페이지에 reprogram → 이후 read는 hard sensing(85us)만으로 성공, block migration 없이 경량화.
7. 아키텍처: LDR은 flash controller 내 위치, LDPC encoder/decoder를 블랙박스로 호출(sensing level 획득 + re-encode), FTL·GC·WL과 협조.
8. 오버헤드: LRU 1000 item + 8 package refresh queue(각 100) ≈ 7KB RAM(256MB 중 무시가능).
9. 결과: DiskSim SSD ext + 17 workload, read 성능 29%↑(baseline 대비), extra P/E 0.2%(전통 refresh 107% 대비), ideal 대비 4% 손해.
10. 한계: 순수 시스템/펌웨어 정책, 부호설계·디코더 알고리즘·HW 미접촉, LDPC 내부 파라미터(rate/length/코드종류) 미명시.
