### B. 알고리즘 요약

1. 대상은 IEEE 802.11n/ac/ax(WiFi) QC-LDPC(base matrix + z×z circulant shift), N=648/1296/1944, rate 1/2~5/6; 합성은 N=648, rate 1/2, z=27로 수행.
2. 문제: layered 디코더의 파이프라인화 시 이전 layer 갱신 대기로 idle-cycle이 생기거나 수렴속도가 저하 - 멀티프레임 처리로 idle을 없애면 메모리가 프레임수만큼 증가.
3. 핵심 기법 1 - RMP(row message passing) layered 스케줄: check node 계산 후 연결된 variable node를 즉시 갱신, SMP 대비 수렴 약 2배 빠름.
4. 핵심 기법 2 - single-frame idle-cycle-free 3단 파이프라인: 멀티프레임 없이 한 프레임만 처리해 메모리 오버헤드 제거, write 후 read 순서로 depth 3이 depth 4 효과.
5. 핵심 식: offset min-sum `Lij = (min|Pji|-beta)·sign(...)` (식4)로 check node 근사, beta=0.5.
6. 파이프라인 깊이 영향: depth 0~3 시뮬로 수렴속도-깊이 trade-off 분석, depth 3이 적정.
7. HW 디테일: LLR Mem/CN Mem을 register 기반 메모리로 구성해 한 layer 전체 메시지를 1클록에 read/write; barrel shifter 6개 병렬(0-shift 2개 제외), z=27개 CNU 병렬.
8. 양자화: word length 8bit, fraction length 2bit, Imax=10, fixed-point가 floating-point에 근접.
9. 결과: 40nm TSMC 합성, 780MHz, 4.2Gbps, 0.38mm2, 218mW, 1.21V. throughput은 기존 최고([20] 1.71Gbps) 상회.
10. 한계: WiFi rate1/2 표준부호 전용 최적화, 정정능력/error-floor 개선 아님(속도·면적만), NAND/soft-read 비용 논의 없음, gatecount 미제시.
