### B. 알고리즘 요약

1. 시스템: 2D/3D TLC NAND(3-bit/cell) SSD, LDPC ECC code rate `9/10`, Upper/Middle/Lower 페이지 구조.
2. 문제: 연속 셀의 1-state 에러가 burst error를 형성하면 LDPC가 30 iteration으로도 정정 실패, Upper 페이지가 최악 BER로 블록 전체를 bad 판정.
3. 관찰: data-retention 시 1-state 에러가 2-state 에러보다 2D/3D에서 각각 7.7×10²/5.4×10⁴배 많아 conventional 데이터 순서에서 burst가 빈발.
4. 핵심 기법(EDC): 저장 데이터 순서를 바꿔 연속 데이터를 Lower/Middle/Upper 페이지에 순차 분산 기록(예: 1→Lower, 2→Middle, 3→Upper) 하여 burst 발생 조건을 1-state에서 2+state 조합으로 전환.
5. 효과 원리: 2-state 이상 에러가 드물게 발생하므로 burst error 빈도가 급감하고 동시에 세 페이지 BER이 평균화되어 worst BER 감소.
6. 확장: AEP-LDPC(w/o upper/lower cells) [7]와 결합, write 시 LDPC 인코딩 후 EDC 재배치, read 시 inverse EDC 후 LDPC 디코딩.
7. 구현 디테일: 단순 로직 회로로 SSD 컨트롤러에 구현, 면적 penalty나 access time overhead 없음, 별도 테이블 불필요(BEM 대비 장점).
8. 결과(실측): burst length=2에서 burst error 87% 감소, length 3+는 완전 제거. worst BER 2D/3D에서 각각 23%/40% 감소.
9. 결과2: 수용 가능 data-retention time을 BEM 대비 2D/3D에서 1.8/2.6배 연장, LDPC 디코딩 iteration도 감소.
10. 한계: LDPC 디코더/부호 자체 개선 아니고 데이터 재배치 전처리, TLC 페이지 매핑·data-retention 에러 통계에 종속.
