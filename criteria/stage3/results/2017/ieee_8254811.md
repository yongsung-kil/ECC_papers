### B. 알고리즘 요약

1. 대상: 폴라 부호 `N=128`(예제), AWGN+BPSK 채널, 5G control channel/IoT 지향 short block. LDPC 아님.
2. 문제: NN 디코더(NND)는 codebook 크기 `2^k`에 지수적 훈련 복잡도(curse of dimensionality)로 짧은 길이만 학습 가능.
3. 핵심 아이디어: 폴라 encoding graph를 독립 sub-graph로 partition(`M`개), 각 sub-block(`N_i,k_i,frozen set A_i`)을 개별 NN으로 학습해 sub-block별 MAP 성능 근사.
4. 결합: sub-block NN들을 남은 폴라 BP stage(식(1) L/R 메시지 업데이트, `f(a,b)=ln((1+e^{a+b})/(e^a+e^b))`)로 연결.
5. 복호 절차: 채널 LLR로 우측 stage 초기화 → BP로 전파 → 상위 sub-block NN 복호 → re-encode해 frozen처럼 고정 → 반복. block-wise sequential + one-shot(non-iterative).
6. 훈련: feed-forward DNN(ReLU 은닉 + sigmoid 출력), encoder+channel로 무한 라벨 데이터 생성, sub-block당 훈련 30분~1일(GPU). `k_max=12`로 partition 크기 제한.
7. 손실 분해: 전체 손실 `λ = λ_part`(partition으로 인한) + `λ_NN`(NN 준최적). PSCL(SCL로 sub-block 대체)로 두 손실 분리 관측 → 주 손실원은 partitioning.
8. 지표: normalized error `NE = (1/D)Σ BER/BER_SCL`. partition 많을수록 NE↑.
9. 결과: `N=128` BER가 SC/BP와 유사, SCL(L=32) 대비 손실 존재. 동기화 step `S_PNN`으로 SCL(`N logN`)/BP(`2I logN`) 대비 latency 대폭 감소.
10. 한계: HW 미설계(시뮬만), sub-block 크기 훈련 복잡도로 제약, 폴라 부호 전용, 짧은 길이 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sub-block NN 디코더 | 대응 없음 | NN/딥러닝 디코더는 min-sum 이터레이션 루프와 이질, 이식 불가 |
| 폴라 BP 결합 stage | 대응 없음 | 폴라 부호 factor graph, binary QC-LDPC PCM(`PCM_b`)과 구조 다름 |
| one-shot non-iterative 저지연 | `decoder.cpp` `LDPC_Decoder()` (개념적 latency 목표만) | 저지연 지향은 공유하나 알고리즘 이식 불가, 개념 참고 수준 |

적용 가치: **낮음** — 폴라 부호 + NN 전면 대체 디코더로 §6 표상 "대응 없음"에 해당하며, binary QC-LDPC min-sum 디코더에 떼어낼 알고리즘 요소가 없음.
