### B. 알고리즘 요약

1. 테스트 부호는 regular QC-LDPC `dv=3, dc=6, R=0.5, Z=54, N=1296`, BSC 채널(crossover `α`) 대상.
2. GDBF는 hard-decision BF 중 최고 성능이나 MS 대비 열세; PGDBF는 flip을 확률 `p<1`로 적용해 MS에 근접시키지만 Perturbation Block(PB) HW 비용이 큼(레지스터 8배, LUT 1.64배).
3. CN은 이웃 VN의 XOR로 parity 계산(`Equ.1`), VN 에너지 `En(k) = vn⊕yn + Σcm`로 채널일치도+불만족 CN 수 합산(`Equ.2`).
4. Maximum Finder(MF)가 최대 에너지 `Emax`를 찾고, 에너지가 최대이면서 랜덤비트 `Rn(k)=1`인 VN만 flip(`Equ.3`).
5. 핵심 통찰: 랜덤 시퀀스는 "강할" 필요 없음 — 랜덤 셔플(`Fr`)과 단순 cyclic-shift(`Fc`)가 성능 동등, 가중치 `ω(R)`가 800~1250 범위면 충분.
6. 짧은 시퀀스 `R'`(길이 `S<N`)를 concat+cyclic-shift로 N비트로 확장; `S=4Z=216`이면 이론 PGDBF와 동일, `S=Z=54`도 약간의 손실로 수용 가능.
7. 초기화법 2종: LFSR(32-bit, threshold 비교로 1비트 확률 제어)와 IVRG(첫 iteration CN 값의 complement를 재사용 — 실제 RG 불필요).
8. VN당 1비트 레지스터 2개, 조합회로 에너지 계산으로 1 iteration = 1 clock; MF는 Leading-Zero Counting Topology(LCT)로 critical path 단축.
9. 65nm ASIC 합성: IVRG-PGDBF는 GDBF 대비 +5.5% 면적으로 87.4Gbps(α=0.01), MS 대비 2~3배 throughput; iteration은 초기 GDBF→후기 PGDBF 스케줄로 `kave=4.68`까지 감소.
10. 한계: BSC hard-decision 전용, MS보다 error correction 약함, soft-read/일반 채널 미고려.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PGDBF 디코딩 루프(에너지/flip) | `decoder.cpp` `LDPC_Decoder()` | 알고리즘 자체는 min-sum과 상이한 BF 패러다임 — 루프 골격만 유사 |
| CN parity XOR 연산 | `decoder.cpp` `C2V_Cal_New_Sgn()` | BF의 CN은 순수 XOR, min-sum sign 계산과 개념 인접 |
| VN 에너지/flip 처리 | `decoder.cpp` `VN_Cal_HD()` | HD 경로와 접점, 단 min-sum magnitude 대신 에너지 sum 사용 |
| Perturbation Block(랜덤 시퀀스) | 대응 없음 | Prime ECC에 확률적 flip 랜덤화 모듈 없음 |
| IVRG(CN 값 재사용 난수) | `random.cpp` (XOR25) | Prime ECC는 별도 난수원 보유, IVRG식 재활용 아이디어만 참고 |

적용 가치: 낮음 — Prime ECC는 soft-capable min-sum 기반이라 hard-decision BF 계열 PB 최적화는 패러다임 불일치로 직접 이식 가치 낮으나, 저전력 hard-only fallback 참고용.
