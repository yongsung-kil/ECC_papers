### B. 알고리즘 요약

1. IEEE 802.16e WiMAX 전 code rate를 지원하는 multi-rate QC-LDPC 디코더 ASIC 아키텍처 (expansion factor `Z=24~96`).
2. 풀려는 문제: 다양한 rate/length·degree 분포를 낮은 복잡도로 지원하면서 메모리 접근 충돌·데이터 상관을 줄이는 것.
3. 부호: 일반 QC-LDPC 대신 overlapping matrix 기반 enhanced QC-LDPC(참조 [11])를 써서 병렬도·수렴속도 유연성 확보.
4. 복호 알고리즘: improved normalized min-sum + multi-layer row-merge — sparseness 이용해 여러 row를 한 row로 병합, layered 스케줄로 odd/even 층 분리.
5. row-merge의 의미: 라우팅 congestion을 줄이고 한 클록에 두 층(Odd/Even)을 MCSC에서 처리하기 위함.
6. MCSC(Minimum Comparator and Selection Circuit): C2V를 LLR에서 빼 high-precision soft info 계산, compression으로 min1/min2+magnitude+index+sign만 저장(출력 정보 6bit로 축소).
7. 구현 디테일: bit precision 6bit, eDRAM 사용(SRAM/DRAM 대비 4배 대역폭·저전력·독립 counter)으로 메모리 대역폭·interconnect 축소.
8. 파이프라인: single-stage hybrid pipelining으로 clock 없이 wave 자체 전파 + reset wave로 residual state 갱신, 다중 bit 동시 복호.
9. 결과: TSMC 65nm, core area `2.97mm2`, throughput `1528.6Mb/s`, power `389mW`@414MHz/10 iter, gate `448.2K`, TAR `514Gbps/mm2` — 비교군 대비 우수.
10. 한계: BER/정정능력 곡선 없음, 부호 rate/length·min-sum scaling factor 수치 미제시, 그림/표 위주 HW 서술로 알고리즘 재현 정보 부족.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| improved normalized min-sum + row-merge | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 계열이나 우리 min1/min2 압축과 이미 유사 — 신규성 낮음 |
| MCSC compression (min1/min2+index+sign 저장) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 우리도 min1/min2 압축 보유, 중복 |
| layered odd/even 스케줄 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 유사) | 우리 Dual-Update(even/odd 교대)와 개념 중복 |
| eDRAM 메모리 뱅크/라우팅 | 대응 없음 | Prime ECC는 bit-exact C++ 시뮬로 물리 메모리 소자 모델 없음 |
| enhanced QC-LDPC overlapping matrix 부호 | `ecc_top.cpp` `Load_PCM()` | 우리는 특정 H-matrix 고정 — 부호 교체 부담 큼 |

**적용 가치**: 낮음 — min-sum/row-merge/layered 스케줄은 이미 보유 기법과 중복이고 핵심 기여인 eDRAM·hybrid pipelining은 물리 HW 소자 최적화라 bit-exact 시뮬 코드에 이식 대상이 아니다.
