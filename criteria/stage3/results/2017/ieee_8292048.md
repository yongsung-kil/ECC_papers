### B. 알고리즘 요약

1. 대상 부호: `N=1296`, `dv=4`(and dv=3), `dc=8`, rate `0.5` regular QC-LDPC, BSC(hard-decision) 채널.
2. 문제: hard-decision GaB는 저복잡·고속이지만 dominant trapping set에 갇혀 error floor가 높아 soft-decision(MS/OMS) 대비 정정능력이 떨어짐.
3. 핵심 기법(PGaB): VNU가 CNU로 보내는 메시지 `v2c` 계산에만 랜덤 비트 `p_n^(i)`를 XOR로 주입 (Eq.3), Bernoulli `B(p0)`로 생성.
4. Eq.1(결정형 GaB)에 확률함수를 더해 `p_n^(i) ⊕ (r_n + Σ c2v)` 형태로 임계 `b(i)=dv/2` 판정을 교란 - 국소최소/trapping set 탈출이 목적.
5. 최적 `p=0.8`(0~1 중), 즉 VNU의 20%만 교란해도 trapping set 탈출에 충분.
6. HW 트릭: 고품질 RNG 대신 32-bit LFSR 기반 경량 RNG로 클록당 VNU에 1-bit 랜덤 공급 (N-크기 shift register).
7. iteration 최적화: `1≤i≤15`는 결정형 GaB, `15<i≤imax`에서만 PGaB 적용(heuristic switch)해 평균 iteration 대폭 감소.
8. VNU 구조: majority voter가 2bit 출력(00/11=다수, 01/10=tie), tie 시 채널값 `r` 포워딩.
9. 결과: FER error-floor(α=0.01)에서 GaB 대비 최대 4자릿수 개선, 처리량 42.6Gbps(dv3)로 MS(600Mbps)/OMS(593Mbps) 대비 ~84배, MS/OMS 대비 register 440%↓·LUT 대폭↓.
10. 한계: BSC hard-decision 전용, rate 0.5 regular 코드만 검증, 게이트카운트 절대값·공정 미제시(상대비율만).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GaB/PGaB 반복 복호 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | hard-decision majority 복호 - min-sum 루프와 알고리즘 상이, 구조만 대응 |
| VNU 계산 + 랜덤 주입 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` | HD 경로 VNU에 랜덤 XOR 추가 지점으로 실험 가능 |
| trapping-set 탈출 post-processing | 대응 없음 | Prime ECC는 CRC 조기종료·적응형 LLR 보유, 확률적 교란은 미보유 |
| 채널(BSC hard) | [channel.cpp](../Prime_ECC_3.1_Claude/) | Prime ECC는 HD 지원하나 BSC 1bit 전용 아님 |

적용 가치: **낮음** — hard-decision GaB 전용 트릭이라 min-sum 기반 고rate QC-LDPC Prime ECC 파이프라인에 직접 붙지 않고, error-floor 교란 아이디어만 참고 수준.
