> 총평: NMS(정규화 min-sum) QC-LDPC(rate 8/9, dv=4/dc=36) NAND 직접 타깃으로 LSB 복호결과와 retention 좌향편향을 MSB 초기 LLR에 결합해 latency를 크게 줄이는 채널/초기화 레이어 기법 — soft-read 비용 절감 관점에서 참고가치 있으나 MLC 페이지 페어링 전제라 이식엔 채널모듈 재구성 필요.

### B. 알고리즘 요약

1. 시스템: MLC NAND(2bit/cell, 4상태 S0~S3)의 LSB/MSB 페이지를 QC-LDPC(rate 8/9, col weight 4, row weight 36, info 2KB/4KB)로 보호, AWGN+4-PAM으로 채널 모사, NMS 복호.
2. 문제: retention 오류로 MSB RBER가 LSB의 약 2배 → MSB 복호 iteration이 max(50)에 도달, LSB/MSB latency 심하게 불균형해 read 성능 저하.
3. 모델: retention은 전자 누설로 threshold voltage가 좌향 이동(‘01’→‘00’ 46%, ‘00’→‘10’ 44%, ‘10’→‘11’ 5%), 인접상태 편향이 통계적으로 강함.
4. 핵심기법1: LSB를 먼저 복호(1라운드)해 결과 `R` 획득, MSB는 2라운드에서 `R`+retention 편향으로 초기 LLR `LLRmsb`를 최적화(`LLRmᴼsb`).
5. 핵심식: NMS CN 갱신 `CI=β·∏sgn·min(|VI|)`; 초기값 규칙 — `Rj≥0 & LLRlsbj≥0 → LLRmᴼsb=-1`, `Rj<0 & LLRlsbj<0 → +1`, `Rj·LLRlsbj≤0 → -1` (MSB 정답을 빠르게 0/1로 수렴 유도).
6. 확장: 4가지 error transition 결론에서 3개 초기화 수정 규칙 도출, 시뮬로 ±1 값 유효성 검증.
7. 구현: LSB 복호결과 저장용 page 크기 버퍼 1개 추가(2KB면 2KB), 초기정보 최적화 연산은 단순해 시간 오버헤드 무시가능.
8. 절차: Algorithm 1 — 감지레벨로 LSB 초기정보 획득→LSB 복호→성공 시 R 기록(실패 시 감지레벨 증가)→R+편향으로 MSB 초기 LLR 최적화→MSB 복호.
9. 결과: RBER 8.0e-3에서 복호 latency 2KB 87%·4KB 84% 감소, iteration 최대 84% 감소; CooECC+3bit 양자화 성능이 미적용+4bit에 근접(read latency 절감).
10. 한계: HW 미설계·시뮬만, MLC LSB/MSB 페어링·순차읽기 전제, 초기 LLR ±1 하드세팅, gatecount/throughput 미보고.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS CN min-sum 갱신 (β·min1) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 우리 min-sum과 동일 계열, 알고리즘 뼈대 정합 |
| MSB 초기 LLR 최적화 / soft-sensing 양자화 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | retention 편향·LSB결과를 초기 LLR에 반영하는 채널/LLR 레이어 개조 필요 |
| iteration별 초기정보 세팅 | `decoder.cpp` `Get_VNU_Table_Idx()`, `Get_VNU_*` | 초기 LLR 세팅 지점이나 페이지-페어 종속 |
| LSB/MSB 페이지 페어링·순차읽기 결합 | 대응 없음 | Prime ECC는 단일부호 복호기, MLC 페이지 상관 개념 부재 |

적용 가치: **중간** — NMS 기반이라 디코더 뼈대는 정합하고 soft-read 비용 절감(3bit로 4bit급 성능) 아이디어가 매력적이나, LSB 복호결과+retention 편향을 MSB 초기 LLR에 넣는 구조가 MLC 페이지 페어링에 강하게 종속돼 우리 채널/LLR 모듈을 재설계해야 이식 가능.
