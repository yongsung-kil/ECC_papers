### B. 알고리즘 요약

1. 시스템: array-based QC-LDPC block code `H(γ,p)` (블록길이 `p^2`, `σ`=순환치환), 예제 `γ=3`, AWGN 채널, min-sum/BP 복호.
2. 문제: iterative 복호의 error-floor를 유발하는 small absorbing set(AS)과 저 min-distance codeword(특히 weight-6 `(6,0)` AS) 다중도를 함께 줄여야 함.
3. 모델: `(a,b)` AS = 각 변수노드가 홀수차수 이웃 check(`O(D)`)보다 짝수차수 이웃을 더 많이 갖는 조합적 부분구조; `γ=3`의 최소 AS는 `(3,3)`(6-cycle과 등가), 지배적 FAS는 `(4,2)`.
4. 핵심기법: `(m+1)`진 `γ×p` spreading matrix `B`로 block code를 `H0..Hm` 성분행렬로 unwrapping해 SC-LDPCC(식 3) 구성 — 일반 edge spreading + 큰 memory `m`.
5. 핵심원리: unwrapping이 그래프의 짧은 cycle을 '끊어' `(3,3)`/`(4,2)` AS 다중도를 block code 대비 크게 감소(정리 2: `Σ(L-i)K_{i+1}`).
6. 확장: 기존 diagonal cut(`m=1`, 단조 이진 B)은 특수경우이고, 일반 B·`m=2`가 더 자유도 큰 설계 제공.
7. min-distance: 정리 3으로 `d(γ,p,m,L,B) ≥ d(C(γ,p))` 하한 보장, diagonal cut 완화 시 소~중 `p`에서 더 큰 dmin 달성(표 II).
8. 최적화: branch-and-bound 완전탐색(작은 `m,p`) + `10^9` guided random search로 AS 다중도/ dmin 균형 spreading vector `b` 탐색; AS와 dmin 사이 trade-off 존재(표 III).
9. 결과: `p=31, m=2` 코드가 rate `R=0.9053`, BER `10^-7`쯤에서야 floor 조짐, 대응 block code 대비 `10^-6`에서 약 1dB 이득.
10. 한계: HW 미설계, 시뮬(BER곡선 지면상 생략), `γ=3` 한정, SC-convolutional+windowed 디코더 전제.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| array-based QC 구조 / spreading matrix로 H 생성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 우리도 QC-LDPC지만 block base+circulant 고정, SC edge-spreading 구조 미대응 |
| absorbing set / error-floor 회피 설계 | 대응 없음 | 우리는 고정 H, AS-aware 부호설계 파이프라인 부재(재검증 부담 큼) |
| min-sum 복호에서의 AS/weight-6 codeword 영향 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 우리 min-sum이 겪는 error-floor와 개념 연결되나 부호구조 자체가 다름 |
| sliding-window 복호 | 대응 없음 | 우리는 block 단위 복호, windowed convolutional 스케줄 미지원 |

적용 가치: **낮음** — error-floor를 absorbing-set/min-distance 관점에서 낮추는 방향성은 고rate QC-LDPC를 쓰는 우리와 문제의식이 맞닿지만, SC-convolutional+windowed 구조의 edge-spreading 구성법 자체는 고정 block QC-LDPC인 Prime ECC에 직접 이식 불가(부호 재설계·재검증 부담).
