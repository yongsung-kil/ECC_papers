### B. 알고리즘 요약

1. 시스템: 압축 데이터(LZW 언어 rate 2.94 bpc / run-length+Huffman 이미지 0.27 bpp)를 systematic LDPC(rate `R`)로 보호, BEC 채널 가정.
2. 문제: 압축이 완벽하지 않아 데이터에 자연 잉여(NR)가 남으며, 이를 ECC와 결합해 정정능력을 높이려 함.
3. NR-Decoder(언어): 가변길이 sliding-window(`nmin`~`nmax`)로 후보해를 좁혀, 유효 단어/구·co-location으로 erasure를 복원.
4. NR-Decoder(이미지): 압축해제 성공 후보를 CNN(clean/noisy 분류)+연결성분 필터로 선별, 후보가 일치하는 erased bit만 복원.
5. 결합 방식: NR-decoder가 먼저 부분정정+soft info 갱신 → LDPC BP가 이어받는 concatenated(Fig.3d) / 반복(Fig.3e) 스킴.
6. 핵심 알고리즘: peeling(BEC)+Gallager-B(BSC)를 일반화한 파라미터(`π`,`τ`) LDPC 반복복호(Algorithm 3).
7. DE 분석: 정규 `(dv,dc)` 부호에 대해 error율 `αt`, erasure율 `βt`의 재귀식 유도(Theorem 4), CN 메시지 정오확률 `At`,`Bt` 정의.
8. 압축언어 모델: 텍스트를 `l`-bit segment로 나눠 `lθ` 이하 erasure면 확률 `p`로 정확복호하는 lucky/unlucky 모델로 반복 DE 유도.
9. 결과: rate 0.95 정규부호 erasure threshold `0.036`→이미지 `≥0.065`(+80.5%), 언어 `≥0.3`(+733%). NR 단독 노이즈 75%↑ 제거.
10. 한계: HW 미설계, 시뮬 없이 이론 DE만, BEC/erasure·압축소스 NR 의존, 일반 ECC용 sphere-packing 상한 추가 제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |

| 일반화 LDPC 반복복호(Algorithm 3) | decoder.cpp LDPC_Decoder() | erasure 전용·비 min-sum이라 직접 대응 안 됨 |
| NR-decoder(언어/이미지 CNN) | 대응 없음 | NN·소스 잉여 기반, Prime ECC에 없음 |

```

적용 가치: 낮음 - BEC/erasure·압축소스 NR 의존의 순수 이론 복호로, NAND binary QC-LDPC min-sum 파이프라인에 떼어 쓸 구성요소가 사실상 없음.
