> 총평: 내결함 메모리(Taylor-Kuznetsov)의 Gallager-B majority-logic 디코더가 timing error에 점근적으로 강건함을 computation-tree로 증명한 순수 이론 논문으로, min-sum 기반 Prime ECC에 이식할 알고리즘 요소는 없다.

### B. 알고리즘 요약

1. 시스템: 신뢰불가 HW로 만든 신뢰 메모리(Taylor-Kuznetsov, TK) - 정보를 regular LDPC 부호어로 저장, 주기적으로 correction circuit 통과. 실험 부호는 QC-LDPC `dv=4, n=1296`, rate 1/2(`dc=8`)와 3/4(`dc=16`).
2. 문제: 저전압/저클럭으로 메모리셀 transient error + correction circuit의 timing error 동시 발생 시 메모리 안정성이 유지되는가.
3. 모델: 셀 열화는 `BSC(α)`, timing error는 gate 출력이 클럭 전 스위칭 못하면 확률 `ε`로 이전 iteration 값 `w(t-1)` 유지(메모리 있는 오류모델, 식(3)).
4. 핵심기법: correction circuit = majority-logic 1-iteration = 수정된 faulty **Gallager-B 디코더** (CN=XOR 식(1), VN=MAJ≥b 식(2)). timing/transient error 하 성능을 등가 디코더로 분석.
5. 핵심식: computation tree 포함관계 `Ne(t+1) ⊆ Ñe(3t) ⊆ Ne(3t)` (Theorem 1) - timing error 디코더의 트리가 error-free 디코더 트리 사이에 bound됨.
6. 핵심 결과식: iteration `t→∞`이면 `Pe(+∞) = P̃e(+∞)` (Theorem 2) - concentration theorem으로 동일 computation tree는 동일 error prob 산출, 따라서 timing error가 점근 성능에 영향 없음.
7. 구현/근사: transient error는 트리 구조를 안 바꾸고 error prob만 바꿈; ε는 CN/VN 동일 가정(일반화 가능).
8. 학습/최적화: 없음 (순수 이론 + faulty density evolution 참조).
9. 결과: Monte Carlo로 rate 1/2, 3/4 부호, 100 iteration, ε=0 vs 0.2에서 BER 곡선이 겹침 - 중간 길이에서도 timing error 무영향 확인.
10. 한계: HW 미설계, Gallager-B 경판정 전용, BSC 채널·이론뿐. BP/Min-Sum 확장은 언급만(오류모델 현실성 낮아질 수 있음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager-B majority-logic 디코딩 (식(1),(2)) | `decoder.cpp` `LDPC_Decoder()` | 낮음 - Prime ECC는 min-sum soft 디코더, majority-logic 경판정과 알고리즘 상이 |
| CN XOR 연산 | `decoder.cpp` `C2V_Cal_New_Sgn()` | 낮음 - sign 연산만 개념적 유사, magnitude 없음 |
| timing/transient error 강건성 이론 | 대응 없음 | Prime ECC는 fault-free bit-exact 시뮬, faulty-HW 오류모델 대상 밖 |
| computation-tree 점근 분석 | 대응 없음 | 이론 도구일 뿐 코드 요소 아님 |

적용 가치: **낮음** — TK 메모리의 Gallager-B majority-logic 디코더가 timing error에 점근 강건함을 증명한 이론 논문으로, min-sum soft-decision 기반 Prime ECC에 이식할 부호설계·디코더·HW 요소가 없다. NAND 데이터저장 세션이라 관련성은 간접 수준.
