### B. 알고리즘 요약

1. 대상은 regular LDPC (16x32 H, wc=4, wr=8, N=32), AWGN+BPSK, 수신 soft `y`와 hard decision `z`.
2. 문제: BF의 multi-bit flipping은 threshold `∆` 설정이 어렵고, 고정 threshold는 message-passing의 동적 변화를 반영 못함.
3. 모델: flipping function `En = -yn(1-zn) - Σ rmn(1-sm)`, weight `rmn = min|yn|` (외재정보).
4. 핵심 기법: flipping intensity `Fn` (식19)을 도입 - 미충족 constraint 연결수와 최대 FF 위치를 결합, `θ0>θ1>θ2` 및 Kronecker delta로 노드 상태 반영.
5. 식 의미: `Fn>0`인 노드 = FF 최대이면서 미충족 constraint에 최다 연결 → flipping 우선 대상.
6. 확장: 적응형 threshold - `∆`=최대와 2위 FF의 차이, `∆1`=그 차이의 2배; Method2 실패 시 Method3 사용해 50~70% 정확 복호.
7. 구현 디테일: 매 iteration마다 FF 정렬(descending), weight `rmn`을 `Ω(-En)` (η=0 clip)으로 갱신하여 동적 반영.
8. 학습/최적화: 없음 (threshold를 trial-and-error 3방법으로 경험적 결정).
9. 결과: ATMBF가 FI 대비 0.75dB, M-DWBF 대비 1dB coding gain (16x32 부호); 단 FI/ATMBF는 타 multi-bit 대비 복잡도 증가.
10. 한계: HW 미설계, 초소형 부호 시뮬만, 적응 threshold로 iteration/복잡도 증가, error-floor 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| flipping intensity / multi-bit BF | 대응 없음 | Prime ECC는 min-sum message-passing, BF 계열 미보유 - 패러다임 상이 |
| 적응형 threshold | 대응 없음 | iteration별 조정 개념은 `PARAM_LLR` 테이블과 유사하나 BF threshold 로직은 무관 |
| syndrome / weight `min|yn|` | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | syndrome/min 연산 개념만 공유, flipping 결정은 무관 |
| dual-diagonal 유사 back-substitution 인코딩 | [encoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_encoder_4KB()` | 논문의 approximate lower-triangular 인코딩과 Prime ECC dual-diagonal은 별개 기법 |

적용 가치: **낮음** - BF 계열은 min-sum 기반 Prime ECC와 디코딩 패러다임이 다르고, 검증이 16x32 초소형 부호 시뮬에 그치며 복잡도 증가·HW 미설계라 이식 가치 없음.
