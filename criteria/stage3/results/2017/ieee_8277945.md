### B. 알고리즘 요약

1. 대상: 프로토그래프 `B=[3 3]` 기반 `(3,6)` regular SC-LDPC(convolutional), `L=50`, coupling width `w=2`, lifting `M=100/500`, `N=10000/50000`, actual rate 0.48, AWGN+BPSK.
2. 문제: SWD는 부분 Tanner graph로 target 심볼을 추정하고 평균오류확률 `Pe,t(u)`가 임계 아래로 떨어지거나 max iter 도달 시 종료하는데, `Pe`가 iteration에 대해 단조감소하지 않음.
3. 결과적으로 max iter로 종료 시의 최종 LLR이 최적이 아닐 수 있어 판정 신뢰도 저하.
4. 핵심 기법(ISWD-I): window가 심볼집합 `Sτ`를 포함하는 동안 `Pe,τ(u)`를 모니터링, 최소값 `Pe*,τ`와 그때의 LLR을 저장(식 12~13).
5. 종료 시 target 심볼 `St`를 최종 LLR이 아니라 저장된 최소-Pe LLR로 판정 → t≥W-1이면 한 심볼집합이 W개 window에 걸쳐 sampling됨.
6. Pe,j(u)=min{Pj,0,Pj,1}, target 평균 `Pe,t(u)=(1/Mβ)Σ Pe,j`(식 8~9)로 계산, BP는 full sum-product tanh 규칙(식 5).
7. 복잡도 절감 변형(ISWD-II): target이 되기 전 이미 `Pe*,t<Ξ`이면 해당 window의 BP 연산을 건너뜀 → 평균 iteration 감소 가능하나 후속 window 수렴에 악영향, W가 커야 유효.
8. 비용: ISWD-I는 추가 iteration은 없으나 `W·Mβ` 심볼 LLR 저장 메모리가 추가로 필요.
9. 결과: ISWD-I이 SWD보다 waterfall 영역에서 유의미하게 우수, SNR↑ 시 SWD로 점근 수렴. ISWD-II는 waterfall 개선하나 error floor 악화. iter 통계(Table I): W=3, SNR 5.0dB에서 45→46 등 W=3에서는 iter 절감 미미.
10. 한계: SC-LDPC convolutional 전용, HW/throughput/gatecount 미제시, 저장된 LLR 대부분이 첫/마지막 window에서 생성됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 반복 루프 / 종료 판정 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 반복 루프는 대응되나 SWD는 block QC-LDPC에 없는 스케줄 |
| 조기종료(Pe 임계) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | Prime ECC는 CRC 기반 조기종료, Pe 모니터링과 개념 상이 |
| 최소-Pe 시점 LLR 저장 | 대응 없음 | window별 LLR memorize 버퍼는 슬라이딩 윈도우 전제, block 복호에 부재 |
| full sum-product tanh BP | 대응 없음 | Prime ECC는 min-sum, full tanh Sum-Product 미사용 |

적용 가치: **낮음** — SC-LDPC convolutional + 슬라이딩 윈도우 + full-tanh BP라는 전제가 Prime ECC의 block binary QC-LDPC/min-sum 구조와 근본적으로 달라 떼어 쓸 요소가 없다.
