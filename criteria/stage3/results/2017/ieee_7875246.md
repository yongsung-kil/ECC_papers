### B. 알고리즘 요약

1. 대상: binary rate-compatible LDPC(RC-LDPC), mother code rate `r0=0.5`, length 1008(irregular C1·regular C2, PEG 구성), RC rate 0.6/0.7/0.8/0.9, AWGN+BPSK, 표준 SPA(`Imax=50`).
2. 문제: 퍼처링으로 고rate 부호를 만들 때 punctured VN의 recovery error가 성능을 좌우 — 기존 greedy(K-SR)·ACE 기법은 recovering iteration/degree 최소화에만 치중.
3. 핵심 관찰: recovery tree 안 unpunctured VN 수 `SM(v)`가 클수록 recovery error 확률 `Pe(v)` 증가(Gaussian Approximation `Pe≈Q(√(m(K)(c)/2))`, `m(K)(c)=φ^{-1}(1-(1-φ(m0/SM))·…)`).
4. 재정의: 기존 minimum recovery tree(단일 branch) 대신, root `v*`의 모든 이웃 `c∈C(v*)`로 완전 전개한 expanded recovery tree 도입 → SPA iterative decoding 본질 반영, 메트릭 `SE(v)=Σ_{c∈C(v)} W(c→v)`.
5. 알고리즘: (Alg.1) prior candidate 탐색 — `W(c)`·`d(c)`·`dp(c)=0`·`dr(v)`·`d(v)`·`SE(v)` 우선순위로 punctured VN·reserved CN 선정. (Alg.2) 모든 target rate에 대해 nested 집합 `V1⊂V2⊂…`로 punctured VN 확정.
6. dead check node: 이웃에 recover 불가 punctured VN이 있는 CN. (Alg.3) 후보 puncturing이 dead CN을 만드는지 반복 검증(모든 punctured VN이 recover되면 success).
7. 핵심 원리: `SE(v)`를 최소화하되 dead CN 비형성을 유지 → global SPA 관점에서 여러 branch가 root recovery를 도와 신뢰성 향상.
8. 퍼처링 수: `Np=⌈n(r-r0)/r0⌉`, 각 punctured VN에 reserved CN 1개 배정.
9. 결과: irregular C1에서 [1][2][7] 대비 rate 0.6/0.7/0.8에서 최고 BER, rate 0.7·0.8에서 BER=10^-6 기준 약 0.4dB coding gain; rate 0.9는 [7]과 유사. Table 5에서 SE(v)≤10 punctured node 수가 가장 많음.
10. 한계: 시뮬만·HW 미설계, node degree 분포는 [7]보다 열세지만 SE(v) 최소화로 BER 우위; rate-compatible(HARQ/rate-adaptive) 전용 기법.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| puncturing 패턴/reserved CN 선택(부호 구성) | ecc_top.cpp Load_PCM() | 우리는 고정 QC-LDPC + punct_col_num=2 고정 — 퍼처링 위치 선택 로직 없음 |
| expanded recovery tree·SE(v) 메트릭 | 대응 없음 | short-cycle/recovery 관점 부호설계 참고용, 직접 대응 함수 없음 |
| SPA(sum-product) 복호로 recovery error 평가 | decoder.cpp LDPC_Decoder() | 우리는 min-sum — SPA/GA 전제와 다름 |
| dead check node 검증 | 대응 없음 | 우리 코드에 대응 개념 없음 |
```

적용 가치: 낮음 — 이 기법은 rate-compatible 퍼처링 패턴 최적화가 목적이고, Prime ECC는 고정rate·고정 QC-LDPC(punct_col_num=2)라 rate-compatibility 수요와 코드 구조가 모두 불일치한다.
