> 총평: IEEE802.16e QC-LDPC용 fast encoder를 linked-list 동적메모리로 다중 rate 대응 FPGA 구현 — 인코더 전용·복호 성능 무관, Prime ECC의 고정 dual-diagonal 인코더와 정합성 낮아 이식성 하.

### B. 알고리즘 요약

1. 대상은 IEEE802.16e 표준 QC-LDPC 인코더, base matrix `Hb=[Hb1 Hb2]`를 expansion factor `z`(24~96)로 리프팅한 `H`(m×n).
2. 문제: 기존 인코더(Gaussian elimination은 O(n^2), RU는 저throughput·비적응)는 복잡도·자원 소모 큼 → code length에 선형인 fast encoding 필요.
3. 부호구조: `Hb2` 첫 열의 `h(1)=h(mb)` 특수열(dual-diagonal류)을 이용, `H2·p^T = H1·s^T`에서 parity를 순차 계산.
4. 핵심기법1(fast encoding): `p1 = (Z^h(1)+Z^h(r)+Z^h(mb))^-1 Σ Z^Hb1(i,j)·sj` (식5)로 첫 parity 계산 후, `pi = pi-1 + Σ Z^Hb1(i-1,j)·sj` (식8~9) 재귀로 나머지 parity 생성.
5. 식 의미: PCM 희소성·순환시프트 구조를 직접 활용해 생성행렬 없이 부호화 → 복잡도가 code length에 선형.
6. 핵심기법2: 다양한 code rate의 서로 다른 열 저장길이를 수용하기 위해 linked-list queue 동적 메모리(dual-port memory, head/tail pointer, length counter)로 노드 삽입/삭제, rate 전환 시 노드 정보만 갱신.
7. 트릭: 행렬-벡터 곱을 설정가능 barrel shifter로 구현(단위행렬 순환시프트=Hb 원소값), rate별 상이한 시프트량을 1클록에 처리, -1 원소 저장 낭비 회피(레지스터 2세트만 사용).
8. 구현: Verilog+ISE 14.7, XC7VX485T 합성/라우팅, 입출력변환·코어코딩·제어 모듈 구성, 직병렬 변환으로 속도 개선.
9. 결과: r=1/2에서 CLB 815·register 356·최대클록 214.5MHz(ref[14] 929/410/168MHz 대비 자원↓ 속도↑), 자원소모의 60%+가 base PCM 저장. length 576~1152, rate 1/2·2/3 검증. fast vs traditional 인코딩 실행시간 4.2s vs 25.2s.
10. 한계: 인코더 전용(복호 성능 무관), IEEE802.16e 표준부호 특화, ASIC gatecount·Gbps throughput 미제시(FPGA CLB/register만), rate/length 적응은 향후 과제.

| 부호용 PCM/열 재배치 | [encoder.cpp](../Prime_ECC_3.1_Claude/) `Column_Reordering_Dual_Update()` | dual-diagonal 열 처리 개념은 유사, 그러나 linked-list 동적메모리는 무관 |
| barrel shifter 순환시프트 곱셈 | 대응 없음 | Prime ECC는 z=32 circulant 고정, 설정가능 barrel shifter 구조 미대응 |
| linked-list queue 동적 메모리 / 다중 rate 저장 | 대응 없음 | Prime ECC는 단일 고정 QC-LDPC, 다중 rate 동적 저장 불필요 |

적용 가치: **낮음** — 인코더 전용이며 IEEE802.16e 표준부호·linked-list 동적메모리 구조가 Prime ECC의 고정 z=32 dual-diagonal 인코더와 정합되지 않는다.
