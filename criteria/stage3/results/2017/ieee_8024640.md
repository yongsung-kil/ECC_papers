### B. 알고리즘 요약

1. 대상 부호: LDPC-CC(convolutional), semi-infinite sparse `H`, rate R=b/c, syndrome former memory `ms`, constraint length `vs=c·(ms+1)`; 예제 rate 1/2 (2,3,6) LDPC-CC.
2. 문제: BP(SPA)는 tanh 연산 비용·수렴 느림, block-wise 디코딩은 초기 latency·메모리 과다 → 저비용·고throughput 디코더 설계가 난제.
3. 디코딩 접근: block-wise(BW, 전체 L block 수신 후 시작) vs windowed(WD, 크기 W window가 Tanner graph를 슬라이딩, 메모리 L/W배 절감).
4. 메시지 규칙: SPA(BP) 최적이나 tanh 고비용 → Min-Sum 근사 및 변형 NMS(normalized)·OMS(offset)로 error-ratio 손실 복구.
5. 핵심 latency 식: `W_BW=L·(Tr+I·Tp)`, `W_WD=W·Tr+I·Tp` — WD는 초기 latency가 L에 독립(ms에만 의존).
6. 스케줄1(flooding FS): CN/VN 전체 동시 갱신, 병렬화 용이하나 복잡도 `C=Imax·(M·O(CN)+N·O(VN))`, 메모리 `N+2m`.
7. 스케줄2(OVA/OVA-CC): CN을 layer로 보고 관련 VN만 활성화, 수렴 2배 빠름(iteration 절반), OVA-CC는 channel LLR 은닉으로 ~20% 메모리 절감.
8. 스케줄3(Horizontal Layered HL): CNA/VNA 커널 병합, VN 메시지 on-the-fly 누적(메모리 미보관) → FS 대비 절반 iteration·메모리 `N+m`.
9. 비균일 스케줄: `Pe(t)<Pemax` 또는 개선비 `<θ`인 노드 갱신 제외로 window+serial 조합 시 복잡도 최대 50% 절감.
10. HW 아키텍처: isomorphic processor(VNU/CNU/routing), Q processor pipeline vs single serial, register vs RAM(circular buffer) 저장, 메모리 뱅크 충돌(graph coloring) 이슈 정리; Table II에 ASIC/FPGA 2~3.66Gbps 사례 요약.
