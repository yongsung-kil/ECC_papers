### B. 알고리즘 요약

1. 대상은 장블록 LDPC의 인코딩/디코딩 복잡도를 낮추려는 Parallel Concatenated Gallager Code(PCGC), 성분부호는 PEG 구성 200x400, MCW 3, girth 8, rate 0.5, dedicated LDPC frame length 600.
2. 문제: 기존 non-zero syndrome PCGC는 성분부호 간 extrinsic 메시지가 상관되어(공통 systematic bit) iterative 성능이 dedicated LDPC보다 나빴다.
3. non-zero syndrome LDPC: codeword가 주어진 non-zero syndrome S에 대한 `HC=S` 해공간에 존재, 생성행렬은 augmented `[H S]`의 RREF에서 얻음(메시지에 '1' 1비트 append).
4. 핵심기법: 두 번째 성분 인코더 입력으로 **reversed message block**을 사용해 parity-check 식을 완전히 다르게 만들어 bit node가 받는 메시지를 독립화.
5. 식 의미: 예 `b6 = b7 ⊕ ...`처럼 두 성분에 공통 systematic bit(b7)가 오류나면 양쪽 extrinsic가 동시에 오염 → reverse로 이 공통성 제거.
6. 확장: reverse 대신 random interleaver를 써도 독립화 가능(성능 유사, 단 delay·복잡도 증가).
7. 구현 디테일: 디코더는 성분부호용 단순 LDPC 디코더 + bit node당 Reg1(zero)/Reg2(non-zero) 2조 레지스터, super-iteration으로 두 성분 교대 local iteration.
8. 복호 알고리즘은 modified Bit Flipping Algorithm(BFA): extrinsic는 1의 개수 - 0의 개수, parity/systematic bit 별도 갱신, non-zero syndrome bit sj를 check node 계산에 포함.
9. 결과: 제안 PCGC가 기존 non-zero syndrome PCGC 및 동일 rate/frame length dedicated LDPC보다 BSC에서 BER 우수(정량 gain은 그림상, 본문 수치 미상).
10. 한계: BSC + hard-decision BFA만 시뮬, AWGN/fading·soft-decision은 future work, HW 미설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| non-zero syndrome 성분부호 생성(`[H S]` RREF 생성행렬) | 대응 없음 | Prime ECC는 dual-diagonal QC-LDPC 인코딩, non-zero syndrome/PCGC 구성 없음 |
| reversed/interleaved 메시지 병렬 concatenation 부호구조 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 고정 QC-LDPC라 concatenation 구조 이식은 재설계·재검증 부담 큼(§4 부호설계 난이도 높음) |
| modified Bit Flipping Algorithm(BFA) 반복복호 | 대응 없음 | Prime ECC는 min-sum message passing, BFA 미사용 |
| super-iteration 교대 스케줄 + Reg1/Reg2 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 이터레이션 루프 개념만 겹칠 뿐, 2-성분 교대 구조는 상이 |
| syndrome 조건 종료 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 조기종료 개념 유사하나 syndrome-based vs CRC-based로 상이 |

적용 가치: 낮음 — BSC 전용 hard-decision BFA PCGC 부호구성법으로, Prime ECC의 고rate binary QC-LDPC + min-sum + dual-diagonal 파이프라인과 부호구조·복호알고리즘 모두 이질적이다. HW·soft-decision 검증도 없어 떼어 쓸 요소가 사실상 없다.
