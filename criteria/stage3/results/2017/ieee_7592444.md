### B. 알고리즘 요약

1. 시스템: Bit Patterned Media(BPM, 자기 하드디스크 차세대), 2D product code — LDPC1`(155,135)` 행부호(inner) + LDPC2`(1184,1024)` 열부호(outer), 둘 다 SPA 복호.
2. 문제: BPM은 고밀도라 media defect로 인한 long burst error가 심각; 기존 simple LDPC-LDPC product code는 EI를 1회만 전달해 부족.
3. 핵심 가정: 2D BPM 채널 `h(x,y)=hx(x)hy(y)`(along/across track 분리), AWGN, ITI/ISI + write-in burst error 모델.
4. 핵심 기법 1(iterative product): LDPC1이 못 고친 burst 행을 erasure 선언(LLR=0)해 위치 정보를 LDPC2에 전달, EI를 두 디코더 간 반복 교환.
5. 핵심 근거: burst 행 erasure로 outer(열)부호가 위치정보 활용해 정정; EI 반복 교환이 단발 전달보다 정정능력 향상.
6. 핵심 기법 2: 각 LDPC 내부 iteration은 줄이고(100→10) 두 디코더 간 외부 iteration(=8)을 늘리는 것이 성능 우세.
7. 구현 디테일: SOVA로 채널 detection 후 행/열 LLR을 각 LDPC에 공급; erasure 정보는 첫 스텝만, EI는 반복.
8. 학습/최적화: 없음(외부 iteration=8 등 실험적 선택).
9. 결과: random error BER 1e-6에서 simple 대비 약 1dB 개선; TMR 10%/30%에서도 약 1dB 개선; rate 0.84에서 타 product code 대비 우세.
10. 한계: HW 미설계, 시뮬만; BPM burst error·2D 채널 전제, product code 이중구조로 NAND 단일부호 파이프라인과 상이.

|---|---|---|

| erasure(LLR=0) burst 처리 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()`, `Set_R_Offset()` | erasure LLR 주입은 닿으나 burst-row 단위 erasure는 BPM 특화 |
| EI 반복 교환(외부 iteration) | 대응 없음 | Prime은 단일부호 내부 iteration만; product code 간 EI 교환 구조 없음 |
| product code(2D 행·열 부호) | 대응 없음 | Prime은 단일 QC-LDPC로 product code 미지원 |
