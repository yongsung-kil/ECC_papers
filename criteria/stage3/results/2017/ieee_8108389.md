> 총평: watermark inner decoder(forward-backward)와 LDPC outer decoder 사이 extrinsic 정보를 turbo형으로 교환해 non-binary insertion/deletion 동기오류를 잡는 통신용 concatenation. LDPC는 표준 (3,6)-regular BP를 블랙박스로 쓸 뿐이라 NAND QC-LDPC 코덱 이식가치 없음.

### B. 알고리즘 요약

1. 시스템: LDPC(outer) + watermark(inner) concatenation, 8-PSK 변조, `m=2`, symbol당 watermark 1bit + LDPC 2bit, non-binary insertion/deletion + AWGN 채널.
2. 문제: 기존 [10] non-iterative는 watermark decoder가 전송 LDPC codeword를 모른 채 uniform prior(1/M)로 forward/backward를 계산 → LLR 부정확, residual error 다수.
3. 채널모델: symbol 삽입확률 `pi`, 삭제 `pd`, 전송 `pt=1-pi-pd`, drift `ti∈[-tmax,tmax]`로 동기 오프셋 추적.
4. 핵심기법: LDPC decoder의 extrinsic soft info를 watermark decoder로 피드백해 forward `Fi`, backward `Bi` quantity를 갱신(turbo 원리).
5. 식 의미: Eq.(5)의 prior `P(d2i d2i+1)`를 uniform이 아닌 LDPC 피드백 확률(Eq.7,8)로 대체 → `βi` 정밀화 → 삽입/삭제 위치추정 정확도↑.
6. 이중 iteration: internal(LDPC BP) + external(watermark↔LDPC 정보교환) 두 루프, external `δ` 증가 시 codeword 추정이 정답으로 수렴.
7. 구현 디테일: symbol conditional prob `P(y|xi,w)`를 bit-by-bit LLR로 변환해 binary LDPC에 입력, LLR↔probability 변환 필요.
8. 복잡도: forward pass 연산이 non-iterative의 약 `δ`배(Table I/II), multiplication `(54M+34)NT δ` 등, δ 작을 때 추가비용 작음.
9. 결과: (3,6)-regular LDPC, length 4000, rate 1/2, PEG 구성, N=2000 symbol, I=5, SNR 20dB. `itermax=10`이 최적, FER는 non-iterative 대비 크게 개선(단조증가 아님), avg iteration 작음.
10. 한계: 순수 시뮬(HW 미설계), insertion/deletion 동기오류 채널·watermark 결합 전용, LDPC는 표준 log-BP라 디코더 자체 신규 기여 없음.

| extrinsic LLR ↔ watermark 피드백 | 대응 없음 | watermark/insertion-deletion inner decoder는 Prime ECC에 없음 |

| turbo형 외부 iteration 스케줄 | 대응 없음 | Prime ECC는 단일 LDPC 루프, concatenation 스케줄 없음 |
