> 총평: BSC hard-decision bit-flipping(GaB) 계열에 무작위 교란으로 trapping set을 탈출시키는 저비용 기법. NAND용 soft min-sum 기반 Prime ECC와 축이 달라 직접 이식가치는 낮으나, "확률적 교란으로 error-floor 탈출" 아이디어는 참고 가치.

### B. 알고리즘 요약

1. 대상: QC-LDPC, `N=1296`, `dv=4`, `dc=8/16`, rate `0.5`/`0.75`, BSC 채널, hard-decision(1-bit) 반복 message-passing.
2. 문제: 초기 hard-decision 알고리즘 Gallager B(GaB)는 HW가 매우 가볍고 클록이 빠르지만(FPGA 최고 Fmax) 복호성능(FER)이 나빠 실사용이 제한됨.
3. 관찰: GaB는 CN-VN 간 n차 의존성으로 oscillation이 생겨 trapping set에 갇히면 오류 정정 실패, cyclic state에 머무름.
4. 핵심기법: VNU 갱신식(Eq.1)에 확률함수 `p`를 곱해(Eq.3) 각 VNU 상태를 무작위로 교란 → trapping set 탈출 (PGaB).
5. 식 의미: `mi(e)`가 임계 `bi=⌈dv/2⌉` 초과/미달 판정 시 채널값 `r(v)`에 확률 `p`를 적용, 결정론적 진동을 확률적으로 깨뜨림.
6. 적용 시점: GaB가 `k` iteration 내 수렴 실패 시에만 확률 전략 발동. 무작위성은 VNU→CNU 방향 메시지 계산에만 주입(Noisy GaB 대비 단방향).
7. HW 구현: 각 VNU에 1-bit 채널값 + 1-bit LFSR 기반 RNG 입력 추가, majority(AND/OR) 게이트로 VNU 구성, 별도 adder 불필요.
8. 파라미터: `p`를 0.1~0.9 스윕, `p=0.8`이 최적. Xilinx Virtex-6 FPGA(vc6vlx240t) post P&R 평가.
9. 결과: FER 최대 4자릿수 개선, GDBF보다도 우수. R=0.5에서 register/Slice LUT +17%/+24%, Fmax 거의 동일(147.5→145.2MHz), user throughput ~18.9Gbps.
10. 한계: BSC hard-decision 전용, N=1296 소형 단일 코드, 1296-bit RNG register footprint 큼(future work로 VNU 클러스터 register 공유 제안).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VNU 확률적 갱신(PGaB) | `decoder.cpp` `VN_Cal_HD()` | HD 경로 VNU 처리와 위치 대응, 단 bit-flipping 스케줄이라 min-sum 구조와 상이 |
| 반복 message-passing 루프 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 대응, trapping set 탈출 훅 삽입 지점 |
| CN syndrome/XOR 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | GaB의 CNU는 순수 XOR로 min-sum과 다름(부분 대응) |
| H-matrix / QC 구조 로드 | `ecc_top.cpp` `Load_PCM()` | QC-LDPC 로드 공통 |
| LFSR RNG / 확률 주입 | 대응 없음 | Prime ECC에 확률 교란 모듈 없음 |
