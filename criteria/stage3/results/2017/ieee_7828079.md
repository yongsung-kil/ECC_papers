### B. 알고리즘 요약

1. 대상은 4kB user message, rate 0.9의 regular QC-LDPC (추가로 1kB, 2kB 코드도 검증), AWGN 채널, hard-decision `z` 입력만 사용.
2. 문제: 기존 WBF/MWBF/GDBF 성능개선은 soft-info를 요구하는데, NAND는 soft-info 생성에 다중 read가 필요해 지연이 커서 부적합.
3. 원본 Gallager BF는 iteration당 가장 의심스러운 1비트만 flip → 정정능력 부족.
4. 핵심: 한 iteration에 여러 비트를 flip하되 flip 최대 수를 `Fx`로 제한 (과정정 방지). 평가값 `En = Σ(2sm-1)` (식 4)이 임계 `TH` 초과인 비트를 후보로.
5. 후보 수 ≤ `Fx`면 전부 flip, 초과 시 `En` 기준 정렬 후 상위 x개만 flip → syndrome 갱신 후 반복.
6. `En`은 `dv`개 이산값만 가져 범위가 좁음 → 초기 iteration에 후보가 많아 과도 flip 시 non-error 비트까지 뒤집혀 복호 실패. F32/F64는 F4보다 오히려 나쁨.
7. 관측 기반으로 최대 flip 수를 4로 고정(`F4`에서 성능 포화), `TH=3`(dv 이하 양의 정수)로 결정.
8. HW 근사: N비트를 g개 그룹으로 분할, 각 그룹 local max에서 top-4 선택 → 정확한 global 4-max 근사. 8그룹이면 성능 열화 거의 없음.
9. 결과: 512-to-4 sorter에서 8그룹 근사 시 CASU 48.3% 절감(성능 유지), 기존 hard-info BF(GDBF/WBF/MWBF/TBF) 대비 WER 우수, latency는 soft LDPC보다 훨씬 짧고 대부분 BCH보다도 짧음.
10. 한계: BCH 및 soft-decision LDPC 대비 정정능력은 낮음(양호 채널 구간 fast path 용도), gate count/throughput 등 HW 수치는 표(Table II)에만 있어 본문 미기재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MBF 디코딩 루프(syndrome/En 계산, flip, 반복) | `decoder.cpp` `LDPC_Decoder()` | hard-info 전용 대체 디코더/폴백 경로로 이터레이션 루프에 삽입 가능 |
| syndrome 기반 CN 평가값 En | `decoder.cpp` `C2V_Cal()` | Min-Sum CN 대신 syndrome 합산형 평가로 대체(hard 모드) |
| hard 판정 VN flip 처리 | `decoder.cpp` `VN_Cal_HD()` | HD 경로의 다중비트 flip 갱신에 대응 |
| 조기종료(syndrome all-zero) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | syndrome 종료조건은 기존 조기종료와 병존 가능 |
| 512-to-4 근사 정렬 HW(그룹별 local-max) | 대응 없음 | Prime ECC에 정렬기 없음 - 신규 HW 블록 필요 |

적용 가치: **높음** - hard-1bit만으로 동작하는 저지연 fast 디코더라 NAND good-채널 구간에서 Min-Sum 앞단 폴백으로 곧장 붙일 수 있고, 우리 고rate QC-LDPC와 코드 스펙(rate 0.9, 4kB)이 정합한다.
