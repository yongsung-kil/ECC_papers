### B. 알고리즘 요약

1. 대상은 protograph 기반 SC-LDPC(=terminated LDPC convolutional) 부호, 예제는 asymptotic rate `R∞=3/6`, syndrome former memory `mh∈{38,73,108}`.
2. 문제: SC-LDPC는 block length가 매우 크지만 sliding window(SW) 디코더로 near-optimal 복호 가능. 복잡도 제약 시 윈도우 크기 `W`와 iteration `Imax` 최적 배분이 필요한데 LDPCC용 분석이 부재.
3. 모델: AWGN 채널, BPSK, SW 디코더가 `SPA-LLR`(sum-product LLR) CN/VN 갱신 규칙으로 동작.
4. 핵심 기법: 복잡도 metric을 `C = a'·α(mh+1)·Imax` (윈도우 `W=α(mh+1)`, `α∈{5..10}`)로 정의하고, 고정 `C` 하에서 W와 Imax를 정수 최적화.
5. 식 의미: 복잡도가 윈도우 크기·iteration에 선형 비례하므로, 둘의 곱을 고정하면 임계 `(Eb/N0)*`를 최소화하는 유일 최적점이 존재.
6. 확장: PEXIT(protograph EXIT) 분석으로 SW 디코딩의 asymptotic threshold `(Eb/N0)*`를 목표 mutual information 0.99 기준으로 산출.
7. 구현 디테일: base matrix에 대해 lifting factor `M=25`(circulant) 적용, 실제 부호는 복잡도가 M배 상향됨을 명시.
8. 최적화: PEXIT threshold를 W의 함수로 그리면 convex 곡선이 되어 정수 W·Imax 최소화로 최적점 도출.
9. 결과: 엄격 복잡도 제약(C=6000,24000)에서 최적 α는 3/2≤α*≤4로 통상 5~10보다 작음. Monte Carlo BER 곡선이 PEXIT 예측과 일치.
10. 한계: 새 부호 설계가 아닌 디코더 파라미터 트레이드오프 도출에 국한, HW 미설계, SC-LDPCC 특화라 block 코드에 직접 미적용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA-LLR CN/VN 갱신 | `decoder.cpp` `LDPC_Decoder()` | 낮음 (Prime ECC는 min-sum, sum-product 아님) |
| SW 윈도우/iteration 트레이드오프 | 대응 없음 | SC-LDPCC 전용, block QC-LDPC에 미대응 |
| PEXIT threshold 설계 | 대응 없음 | 부호설계 이론, 코드 이식 대상 아님 |
| lifting(circulant M) | `ecc_top.cpp` `Load_PCM()` | 낮음 (QC 개념만 공유, 구조 상이) |

적용 가치: **낮음** — SC-LDPC convolutional 슬라이딩윈도우 복잡도 최적화라 NAND block binary QC-LDPC(min-sum, 고정 H-matrix) 기반 Prime ECC에 떼어 쓸 요소가 없음.
