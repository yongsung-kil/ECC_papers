### B. 알고리즘 요약

1. 대상은 SC-LDPC(spatially-coupled) 부호, 특히 array-based(AB) column-weight 3 코드 `H(3,p)` (p=17, block length `Lp^2=289L`)의 유한길이 설계다.
2. 문제: SC-LDPC의 threshold saturation은 점근 이론이며, 유한길이에서는 absorbing set(ABS)이 error-floor를 지배하므로 이를 제거해야 한다.
3. 핵심 관찰: edge-spreading(`H=H0+...+Hm`)과 terminal lift를 하나의 대수적 graph lift로 통합, 허용 permutation은 `τ_L^k ⊗ λ` (Kronecker product, Theorem III.3) 형태다.
4. 핵심 기법: base graph의 6-cycle을 이루는 edge에 `τ_L^k`(shift permutation)를 전략적으로 배정하면 lift에서 cycle이 끊겨 (3,3)-ABS와 그 상위 (4,2)-ABS가 대수적으로 제거된다.
5. 핵심 식 `c2 - c1 = np` (Lemma V.2): identity 블록행에 속한 두 열의 차가 `p`의 배수라는 조건으로 6-cycle을 `(c1,c2)` 평면의 선분 위 정수점으로 환원한다.
6. 확장: `τ_L^k ⊗ τ_J` (J≥2)로 lift하면 결과 PCM이 quasi-cyclic이 됨(Lemma III.6), cutting vector·generalized cutting vector가 이 프레임워크의 진부분집합임을 증명(`C ⊂ Cg ⊂ E ⊂ Q ⊂ A`).
7. 부수: (3,3)-ABS 개수를 polygon 정수점 대신 line-counting(식11)으로 세는 저복잡도 열거법, `H(3,p,L)`은 piecewise line counting으로 확장.
8. 최적화: permutation 배정 행렬 `Bm`을 제한적 exhaustive search + iterative backtracking으로 수치 최적화하여 ABS 수를 최소화.
9. 결과: L=10~50에서 Code 3(m=2 최적 lift)이 cutting-vector Code 1 및 [11][12]보다 (3,3)-ABS 수가 훨씬 적고(예 L=10: 442 vs 19108), windowed 4p^2 window에서는 ABS가 0.
10. 한계: BER/FER 복호 시뮬 없이 ABS 개수 이론값만 제시, HW 미설계, column-weight 3 array-based SC-LDPC에 특화.
