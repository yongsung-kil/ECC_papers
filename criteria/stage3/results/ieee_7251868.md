### B. 알고리즘 요약

1. 대상 코드는 IEEE 802.3an(10GBASE-T)의 (2048,1723) regular LDPC, CN degree 32, VN degree 6.
2. Stochastic LDPC 디코더는 SPA를 확률 도메인에서 계산하며 메시지를 Bernoulli sequence의 비트열로 표현, 배선 혼잡을 줄이지만 Tanner graph의 girth로 인해 출력이 고정 상태에 머무르는 latching 문제가 있음.
3. 기존 해법인 edge memory(EM, `[4]`)는 각 edge마다 shift register+MUX를 두어 unhold state 확률을 추적하나, VN 면적의 약 80%(디코더 전체의 약 60%)를 차지해 면적/동적전력 낭비가 큼.
4. 본 논문은 EM을 saturated up/down counter로 대체: unhold 입력이 1이면 counter+1, 0이면 -1, hold state에서는 counter 값과 LFSR 난수를 비교해 출력 결정.
5. 64-bit EM(레지스터 64개) 대비 counter는 saturation value 128에서 7-bit(레지스터 7개)만 필요 — 면적 절감의 핵심 근거.
6. 추가로 VN 내 6개 sub-unit이 원래 개별 EM(probability tracer)을 가졌으나, 6개의 확률이 유사함을 이용해 counter 1개를 공유(1-counter VN)하도록 재설계.
7. 6-input sub-unit 내부에서 12개의 3-input 그룹 중 실제로는 7종류만 서로 다름을 확인, 동일 그룹 로직을 재사용해 sub-unit 구조를 축소(Table I).
8. 채널 LLR→확률 변환은 `Pi = e^Li/(e^Li+1)` 식과 LUT, NDS(noise dependent scaling)로 처리, 채널 입력은 6-bit(부호1+정수2+소수3) 양자화.
9. 결과: SMIC 65nm 1.0V 500MHz 합성 시 전체 디코더 면적 2.2mm²(EM 기반 14.4mm² 대비 15.3%), throughput 10.2Gb/s, SNR 5.5dB에서 평균 100.4 DC(termination 700 DC 상한)로 수렴, BER은 counter 기반이 64-bit EM보다 우수.
10. 한계: stochastic computing 고유 아키텍처(확률 비트스트림, LFSR 난수 비교)에 특화된 개선이라 결정론적 min-sum 계열 디코더에는 개념 이식이 어렵고, VN 면적 비교는 시뮬레이션이 아닌 합성 결과에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic 디코딩(확률 비트스트림, VN counter/EM) | 대응 없음 | Prime ECC는 min-sum 기반 결정론적 디코더로 확률 비트스트림 방식과 아키텍처 자체가 다름 |
| CN XOR 연산(패리티 체크) | `decoder.cpp` `C2V_Cal()` | 개념적으로만 유사(패리티 체크 자체는 공통), 구현 방식은 완전히 다름 |
| 조기종료(termination, DC count) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료라는 목적은 유사하나 본 논문은 CRC가 아닌 자체 Check block 기반 |
| 채널 LLR→확률 변환, 양자화 | `channel.cpp` `Set_LLR_Th()` | 소프트 양자화라는 점만 유사, 확률 표현 방식이 달라 직접 이식 불가 |

적용 가치: **낮음** — stochastic computing 패러다임 자체가 Prime ECC의 min-sum HW 구조와 근본적으로 다르고, 다룬 개선(EM→counter)도 stochastic 디코더에만 유효한 면적 최적화라 이식 불가.
