### B. 알고리즘 요약

1. 대상 부호는 5G-NR CRC-Aided Polar(CA-Polar) `(N,K)=(128,105)`, rate `0.82`, CRC 11비트 — LDPC가 아닌 Polar code.
2. 디코딩 알고리즘은 GRAND 계열의 ORBGRAND(Order Reliability Bits GRAND): 인코딩 구조와 무관하게 채널 LLR 순서에 기반해 오류패턴을 추측하고 codebook 멤버십(`H·y = H·E`)을 검사하는 방식.
3. ORBGRAND 성능의 핵심은 LLR 값을 최소~최대로 정렬하는 sorter이며, 이 sorter가 하드웨어 비용(면적/전력)의 대부분을 차지.
4. 정렬 네트워크로 bitonic sorter(Compare & Swap 유닛 기반)를 사용, 16-input 16-output 구조를 기본으로 함.
5. Half-Output(HO) 기법: 전체 출력 대신 필요한 상위 일부(예: 8개 중 4개)만 남겨 C&S 유닛과 연결선을 축소.
6. Segmentation(S) 기법: 입력을 `S`개 그룹으로 나눠 각각 작은 bitonic sorter로 정렬 후 최솟값들만 결합, 완전 정확도 대신 근사 정렬 허용.
7. 두 기법을 결합한 HO+S 방식은 C&S 유닛 수와 파이프라인 레지스터 수를 동시에 축소.
8. 별도 학습/최적화 절차 없음(S=2, S=4 실험적 비교로 S=2 채택, S=4는 FER 손실 0.5dB로 과도).
9. 결과(TSMC 40nm, 500MHz 합성): HO+S=2 조합 시 원본 대비 면적 38% 감소(445,893→277,438 um²), 전력 39% 감소, 대신 레이턴시는 8→5 cycle로 변화(파이프라인 단축이나 FER 0.1dB 손실 수반).
10. 한계: LDPC가 아닌 Polar/GRAND 전용 설계, sorter 모듈 단위 최적화로 전체 디코더(H·Y, decoding core) 설계는 타 논문([11][12]) 인용에 의존, throughput 수치 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ORBGRAND 전체 디코딩 알고리즘 | 대응 없음 | GRAND는 error-pattern guessing 기반으로 min-sum message-passing과 패러다임이 다름 |
| bitonic sorter HW (LLR 정렬) | 대응 없음 | Prime ECC 디코더는 LLR 정렬 연산이 필요 없는 min-sum CNU/VNU 구조(`decoder.cpp`)이며 표에 대응 모듈 없음 |
| soft LLR 양자화(5bit) | `channel.cpp` `Set_LLR_Th()` | 개념적으로만 유사(양자화 LLR), 실제 sorter 로직은 이식 불가 |

> 적용 가치: 낮음 — Polar code 전용 GRAND 디코더의 sorter HW 최적화로, NAND binary QC-LDPC min-sum 디코더 구조에 대응되는 요소가 없음.
