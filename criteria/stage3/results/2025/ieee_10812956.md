### B. 알고리즘 요약

1. 시스템은 BICM-ID(bit-interleaved LDPC coded modulation with iterative decoding): AWGN 채널 + 8PSK/16QAM/32QAM 고차변조 + demapper-decoder 간 반복(외부 iteration)이며, 코드 예시는 CCSDS `H_{1/2}` (length=240, rate=1/2, 4서브블록 순열행렬 구조).
2. 기존 LDPC 조기종료 기준은 variable node의 hard-decision(syndrome) 기반이 대부분이며, BICM-ID처럼 demapper-decoder가 확률영역에서 반복하는 구조의 조기종료 연구는 부족함.
3. 채널/모델 가정: AWGN + Gray mapping 고차변조, LDPC 디코더는 표준 BP(Sum-Product), 반복은 inner(디코더 내부, 최대 30회)와 external(demapper-decoder 간, 최대 3회) 2단 구조.
4. 핵심 기법(NBEP): 각 check node에 대응하는 "detection node" `g_i`를 도입해 `p(g_i=0)`(신드롬 만족 확률)을 정규화 합산한 지표 `p(G_sum) = (1/m)Σ p_{h_i→g_i}(g_i=0)` (식8)로 반복 상태를 하나의 스칼라로 표현.
5. 식의 의미: 기존 hard-decision syndrome 판정과 달리 확률영역 신뢰도를 연속값으로 추적해 수렴/진동/오수렴 3가지 패턴을 조기에 구분 가능케 함.
6. 확장(ISC): NBEP의 전후 차분을 이용한 "front-back numerical differential threshold detection"으로 진폭변동 임계값 `μ`, 연속수렴횟수 임계값 `λ`(inner 판정용), 신뢰도 임계값 `L`(external 판정용) 3개를 조합해 조기종료(Algorithm 1).
7. 부수 트릭: 최적 파라미터는 `L=0.83, μ=0.01, λ=2`로 실험적으로 결정되며, external iteration 종료 후 단 한 번의 hard-decision만 수행해 `H·x̂^T` 반복 곱셈을 줄임.
8. 학습/최적화 절차 없음(파라미터는 Monte Carlo 시뮬로 후보군 탐색 후 이론 BER과 비교해 확정하는 경험적 튜닝).
9. 결과: 기존 hard-decision 방식 대비 저-중 SNR에서 평균 반복복호 지연(ANDI) 최대 88% 감소, BER 성능 저하는 무시할 수준(BILCM-ID는 BILCM 대비 0.3dB 이득).
10. 한계: 변조·demapper가 필수인 BICM 특화 구조(inner/external 2단 iteration)이며 HW 구현·NAND soft-read 채널·순수 binary LDPC 단일 디코더 적용은 다루지 않음, 코드 길이도 240 하나만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 확률영역 NBEP 기반 조기종료(ISC) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 이미 CRC 기반 조기종료(Partial 4 termination point + Full CRC)를 보유하고 있어 개념적으로 중복 기여이며, 본 논문 기법은 확률(soft belief) 누적 지표라 CRC 대비 구현 복잡도가 더 큼 |
| LDPC BP 디코딩(표준 Sum-Product) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이고 본 논문은 표준 tanh 기반 SPA를 그대로 사용, 알고리즘 자체 개선 아님 |
| demapper-decoder external iteration (BICM 반복) | 대응 없음 | Prime ECC는 변조/demapper가 없는 NAND storage ECC라 external iteration 개념 자체가 대응 없음 |
| 고차변조(8PSK/16QAM/32QAM) 채널 모델 | `channel.cpp` | Prime ECC는 AWGN/RBER/fixed-error 채널만 지원, 변조 심볼 채널은 대응 없음 |

적용 가치: 낮음 — BICM 고차변조·demapper 반복 구조에 종속된 조기종료 기법이며, Prime ECC는 이미 CRC 기반 조기종료를 보유하고 변조 단계가 없어 핵심 아이디어(NBEP)를 이식해도 기존 기능과 중복되고 구조적 전제(external iteration)가 맞지 않음.
