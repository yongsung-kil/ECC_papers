### B. 알고리즘 요약

1. GF(q)·임의 순환크기 `e`의 QC-LDPC PCM `H(s,t)`(각 블록은 `e×e` 순환행렬)의 rank를 해석하는 문제. rank는 code dimension = length − rank로 직결.
2. 기존 Fourier transform 기반 rank 해석은 `gcd(e,q)=1`에서만 유효 — 많은 대수적 구성이 이 조건을 못 맞춤.
3. 핵심 도구: 순환행렬을 첫 행의 `associated polynomial`로 표현, PCM 전체를 `GF(q)[x]` 위 associated polynomial matrix `C`로 치환(곱·합이 `GF(q)[x]/(x^e−1)`와 동형).
4. 1 row-block: `rank(H(1,t)) = e − deg(gcd(x^e−1, A_1(x),…,A_t(x)))` (Theorem 1), CPM 특수형은 `e − gcd(e,a_1,…,a_t)` (Corollary 1).
5. 2·3 row-block 정확공식(Theorem 2,3): `ω_i(x)` 교차항 gcd로 rank 산출; 4-cycle 없고 `1+x+…+x^{e-1}` 기약이면 `rank(H(3,t))=3e−2` (Corollary 3).
6. 임의 row-block: minor의 기약인수 분해로 하한(Theorem 4)·상한(Theorem 5), `rank(C|x=1)` 평가로 간편 상한 `se−s+r_c` (Theorem 6), 상하한 결합 시 정확 rank(Theorem 7).
7. 순환크기 `e`를 키울 때 rank 변화, `gcd(e,q)=1`이면 Fourier transform 결과와 동치(Corollary 4)임을 보임.
8. Masking: masked matrix rank가 많은 경우 base와 무관하게 masking matrix만으로 결정 → 원하는 rank의 masking matrix 설계로 code 설계 단순화.
9. 응용: [6],[8],[10],[11],[12],[13] 등 대수적 구성과 그 masked 버전에 적용해 다수의 새 rank 공식 도출, 주어진 length/rate/순환크기에서 PCM 존재 가능 여부 판정(Theorem 26).
10. 한계: 순수 대수 이론(rank 해석)뿐 복호 알고리즘·HW·BER 성능 기여 없음(Fig.4 BER 곡선은 인용 구성의 예시일 뿐 본 기여 아님).
