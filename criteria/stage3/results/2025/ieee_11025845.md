### B. 알고리즘 요약

1. 대상은 6G eMBB+용 rate-compatible SC-LDPC 코드로 rate 0.34~0.87, 프레임 최대 `N=253110`, AWGN 채널 가정.
2. 기존 SC-LDPC 구성법은 언커플드 protomatrix(예: PBRL) 설계 후 edge-spreading을 적용해 커플드 `B_SC`의 최적성이 보장되지 않는 문제를 해결.
3. 채널은 AWGN, 코드는 `B = [BHRC 0; BIRC BSSC]` 구조(HRC/IRC/SSC 3요소)로 우상단을 비워 rate-compatibility 보장.
4. 핵심 기법은 unified graph `G_U`로, edge-spreading 행렬 `T`와 protomatrix `B`의 의존관계를 하나의 그래프로 표현해 사이클(`l-cycle`)을 message-passing으로 계수.
5. 누적 경로값 `φ = ΔT0+...+ΔT_{l/2-1}`가 0이면 사이클이 `G_SC`에서도 유지된다는 조건(식 9)을 이용, greedy하게 edge 재배치(`T*_{c,v}`, 식 17)로 짧은 사이클 최소화.
6. 확장 기법으로 modified coupled reciprocal channel approximation(RCA)을 도입해 유한 coupling length `L`에서도 `L→∞` 문턱값에 근접한 디코딩 threshold를 추정.
7. HW 친화 트릭으로 tail matrix `Q`를 tri-diagonal로 설계해 termination error floor를 억제하고, one-step-recoverable puncturing으로 threshold를 0.12dB 개선.
8. 최적화 절차는 PEG+ACE로 HRC 설계 → `G_U` 최적화 → modified coupled RCA로 threshold 평가를 반복(Algorithm 2/3)하는 결정론적 탐색이며 별도 학습은 없음.
9. 결과는 Imax=5에서 5G-NR 대비 R=0.8에서 1.15dB, R=0.34에서 0.65dB 이득, throughput 209.4Gbps(5G-NR 195.4Gbps) 및 50.1% 개선.
10. 한계는 6G 무선/AWGN 기준 시뮬레이션뿐이며 HW는 기존 5G-NR 디코더 아키텍처([27])를 그대로 차용한 수치이고 실합성/실측은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ESRL protomatrix/H-matrix 구성 (HRC/IRC/SSC, PEG+ACE) | `ecc_top.cpp` `Load_PCM()` | 부호구조 전면 교체 필요, 고정 QC-LDPC 재검증 부담 큼 |
| unified graph 기반 사이클 제거 최적화 | 대응 없음 | 부호설계 단계 도구, HW 디코더 코드와 무관 |
| modified coupled RCA(threshold 분석) | 대응 없음 | 이론 분석 툴, 실행 코드 대응 없음 |
| SLME(semi-layered multi-engine) 병렬 디코딩 | `decoder.cpp` `LDPC_Decoder()` | 다중 엔진 병렬화 아이디어는 z=32/HCU 병렬구조와 개념적으로 유사하나 SC-LDPC 전제 |
| L-NMS(layered normalized min-sum) 디코딩 | `decoder.cpp` `CNU_Update_New_Mag()` | 이미 보유한 min-sum 계열, 신규성 낮음 |
| IR-HARQ / puncturing 스킴 | 대응 없음 | NAND ECC는 HARQ 미사용 |

> 적용 가치: 낮음 — SC-LDPC 부호구성·6G HARQ 중심 논문으로 Prime ECC의 고정 binary QC-LDPC 구조와 정합성이 낮고, 디코더 알고리즘(L-NMS)도 이미 보유한 기법의 재사용 수준.
