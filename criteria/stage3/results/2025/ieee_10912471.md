### B. 알고리즘 요약

1. 고율 regular LDPC 코드(row weight 큼, `(dc,dv)`로 표기, 예: (60,6)/(81,8)/(127,6))를 AWGN+BPSK 채널에서 복호하는 문제를 다룸.
2. 기존 NMS 디코더는 고정밀 내부 연산·비교 연산이 많아 초고속 복호에 부담 → 이를 3-상태 메시지({+1,0,-1})로 대체하는 TMP 디코더로 복잡도 절감을 시도.
3. VN→CN 메시지는 임계함수 `F(x)=sign(x) if |x|>η else 0`로 ternary화, 신뢰도 가중치 `w^t`를 곱해 소실된 소프트 정보를 보상.
4. 핵심 갱신식: `l_j^{t→i} = F(l_j^{t-1} - l_i^{t-1→j}·w^{t-1})` (VN→CN), CN→VN은 부호곱만(식 3), VN 최종 갱신은 `l_j^t = l_j^0 + w^t·Σ l_i^{t→j}`(식 4).
5. 이 구조에서 모든 엣지가 동일 `w^t`를 공유하도록 하여(regular 코드 특성 활용) 곱셈수를 `2n·dv`에서 `(n+1)dv`로 감소, 룩업테이블화 시 곱셈 완전 제거 가능.
6. Density Evolution(DE)으로 η와 신뢰도벡터 w를 iteration마다 최적화(Algorithm 1, decoding threshold γ* 최소화).
7. Hard-decision(1-bit) 입력용으로 별도의 quantized DE(이산 확률분포 기반, 식 10~15)를 제안해 τ-bit 양자화된 TMP 성능을 예측·최적화.
8. 최적화 절차는 η 탐색 범위·step(floating: 0.2~3.0/0.1, quantized: 0~5/1)에서 γ* 최소화하는 grid search.
9. 결과: floating TMP는 BP 대비 FER=1e-5에서 약 0.15dB 차이, Q(4) TMP는 NMS 대비 0.1dB 이내 격차, Q(1)(hard-decision) TMP는 최적화 후 NMS를 능가.
10. 한계: 알고리즘·시뮬 수준 검증만 존재하며 HW 구현/gatecount/throughput 결과 없음, 코드 예시는 소규모(n≤127)이며 NAND 채널(RBER, retention) 실험 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TMP 노드 갱신(ternary message, 곱셈 제거) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | min-sum 대체 후보 알고리즘, HD 모드와 직접 비교 가능 |
| hard-decision quantized DE 최적화(η, w) | `decoder.cpp` `VN_Cal_HD()`, `ecc_data.h` `PARAM_LLR` | HD 파라미터 최적화 절차로 이식 시 LLR 테이블 재설계에 참고 가능 |
| iteration별 신뢰도 파라미터(η,w) 저장 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 파라미터 테이블 구조가 기존 적응형 LLR과 유사, 중복 가능성 |
| 채널 모델(AWGN/BPSK) | `channel.cpp` `Set_R_Offset()` | Prime ECC는 AWGN/RBER/fixed 채널 이미 지원, 대응 있음 |

> 적용 가치: 중간 — hard-decision 모드에서 곱셈 없는 저복잡도 디코딩이라는 아이디어는 Prime ECC의 HD 경로에 참고 가치가 있으나, min-sum과의 정량적 HW 비교가 없어 실이득 불확실.
