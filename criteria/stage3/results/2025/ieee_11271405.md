### B. 알고리즘 요약

1. 시스템: `M×N` 희소 parity-check matrix `H`(rate `R=K/N`)를 갖는 regular LDPC 코드, BPSK 변조 후 AWGN 채널(`y_n=x_n+w_n`)에서 채널 LLR `Ly(n)=4R(Eb/N0)y_n`을 계산해 반복 복호.
2. 문제: flooding BP는 15-50회 반복이 필요해 latency가 크고, 매 반복 `O(dc)` product-of-tanh 연산이 전체 edge에 걸쳐 발생해 계산 비용이 큼 — 반복횟수·복잡도·latency를 동시에 줄이는 것이 목표.
3. 채널/모델 가정: AWGN + BPSK, sum-product(log-domain, tanh 기반) C2V 갱신 `m_ci→vj = 2 tanh^-1(Π tanh(m_vk→ci/2))`을 기반 프레임워크로 사용.
4. 핵심 기법(AR-CID, 2단계): (1) syndrome 기반 메시지 불신뢰도 `Rv = Σ_{m∈N(v)} s_m` 산출 후 상위 `λN`개(λ=0.2) 노드를 후보로 선정, (2) 반복 간 LLR 변화량(contextual transition) `y = |φ(Ly)-φ(L̃y)|`(`φ(L)=1/(1+e^-L)`)를 결합한 가중 지표 `M(v,y)=α·Rv+β·y` (α=0.65, β=0.35)로 임계값 `γ`(0.1~0.15) 초과 노드만 능동 갱신.
5. 핵심 식 의미: `Rv`는 얼마나 많은 unsatisfied parity check에 관여하는지(오류 가능성), `y`는 반복 간 신뢰도 변화(수렴 진행도)를 나타내며, 두 지표를 결합해 CI(Conditional Innovation) 기법의 단일 기준보다 정교하게 갱신 대상을 선별.
6. 잔차(residual) 계산도 사전 산출된 신뢰도 반영 메시지 `m_ci→vj^pre`와 이전 메시지 차이로 수정(`r=m^pre-m^old`)해 modified RBP 스케줄링에 통합.
7. 부수 구현: 정렬/top-λN 선택 `O(N log N)`이 지배항이 될 수 있어(N>4096) 병렬 계산·근사 신뢰도 추정·계층적 노드 선택(`O(N+k log k)`)으로 확장성 확보 언급.
8. 별도 학습 절차 없음; γ∈[0.05,0.3], λ∈[0.1,0.5], α∈[0.5,0.8] 범위를 사전 스윕해 경험적으로 파라미터 결정(α=0.65, β=0.35, λ=0.2, γ=0.1~0.15).
9. 결과: (2048,1024) rate-1/2 코드에서 BP 대비 반복횟수 4-5회(BP는 200-300회) 소요, RBP 대비 40-45% 반복 감소, BER 1.18×10⁻⁷@Eb/N0=4.5dB(RBP 1.60×10⁻⁶ 대비 우수), 메모리 오버헤드 BP 대비 약 67%(≈95KB @(2048,1024)).
10. 한계: 순수 알고리즘/소프트웨어 시뮬레이션(ARM Cortex-A53 클록 기반 추정 latency)에 그치며 HW 합성·FPGA/ASIC 검증 없음, 정렬 기반 노드선택의 HW 파이프라인 친화성 미검증, 코드가 랜덤 생성 regular LDPC(N≤2048)로 QC 구조 특유의 z-병렬 처리와의 정합성은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| AR-CID 2단계 노드 선택적 반복 스케줄링 | decoder.cpp LDPC_Decoder(), C2V_Cal() | 이터레이션 루프에 syndrome 기반 Rv + LLR 변화량 y 신뢰도 지표를 추가해 갱신 대상 노드를 선별하는 방식으로 반복 스케줄에 이식 검토 가능 |
| Message unreliability index Rv (syndrome 기반) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 조기종료 CRC와 별개로 노드별 unsatisfied check 카운트를 활용한 부분갱신 스케줄과 연계 가능성 |
| Contextual information transition y (LLR 변화량) | decoder.h Get_VNU_* , ecc_data.h PARAM_LLR | VN LLR 갱신량을 추적하는 부가 지표로 기존 iteration별 LLR 테이블 구조에 결합 검토 가능 |
| top-λN 노드 정렬/선택 (O(N log N)) | 대응 없음 | Prime ECC HW는 고정 파이프라인(z=32 병렬)이며 런타임 정렬 기반 선택적 갱신 구조 없음 — HW 이식 시 재설계 필요 |
```

적용 가치: **중간** — min-sum이 아닌 sum-product 프레임워크 및 소프트웨어 시뮬 수준이라 직접 이식은 제한적이나, "신뢰도 기반 선택적 노드 갱신으로 반복횟수 감소"라는 아이디어 자체는 Prime ECC의 min-sum 이터레이션 루프에 조기 수렴/부분 갱신 최적화로 검토할 가치가 있음.
