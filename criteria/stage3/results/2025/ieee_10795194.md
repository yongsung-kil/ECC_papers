### B. 알고리즘 요약

1. 시스템: 방송 시스템(DTTB)용 QC-SC-LDPC 코드. L개의 uncoupled code(base matrix `Bunc`)를 smoothing parameter `w`로 결합해 spatially-coupled 구조 `HSC` 구성.
2. 문제: 기존 QC-SC-LDPC는 rate-compatibility·length-scalability가 부족해 다양한 code rate/length를 지원하려면 HW 인코더/디코더를 개별 설계해야 함(구현 복잡도 증가).
3. 모델: AWGN 및 Rayleigh fading 채널(`y = hx + v`), QPSK/64NUC 변조, 방송 시나리오(ATSC 3.0 등) 가정.
4. 핵심 기법 1단: rate-compatibility는 information node extension(정보 컬럼 `Kq` 삭제로 낮은 rate 유도), length-scalability는 modulo lifting(`Pα1 ≡ Pα2 mod Zα1`)으로 구현 — `(α, q, λ)` 3차원 인덱스(lifting factor, 정보열수, smoothing parameter)의 nested 관계로 코드 패밀리 구성.
5. 핵심 식: 3D-grid 상 각 노드 `(α,q,λ)`의 exponent matrix가 모든 부모 노드에 nested되도록 유지(`Pα1,q1,λ1` nested in `Pα2,q2,λ2`) — 이 nested 구조 덕에 HW 인코더/디코더를 최대 파라미터 기준으로 설계 후 부분 활성화만으로 여러 rate/length 지원.
6. 핵심 기법 2단(3D-G-PEGL): 3D-grid의 각 노드를 진행경로(growing path) `χ`를 따라 progressive lifting/extension으로 순차 구성, 매 단계 cycle/diversity/BLER pruning으로 후보 exponent matrix 축소.
7. 부수 트릭: 개선된 cycle 분류 기준으로 cycle length `g`뿐 아니라 uncoupled code interval `η`도 고려(`(g,η)-cycle`), rate loss 완화를 위한 information node shortening(우측 종단), 오류전파 완화를 위한 periodic information node shortening.
8. 최적화 절차: cycle pruning(가중 cycle 수 `n=Σn_{g,η}w_{g,η}` 임계값 기반) → diversity pruning → BLER pruning(실제 시뮬레이션 기반) 3단계로 그리드 노드별 후보 축소, 최종 선택.
9. 결과: 구성된 QC-SC-LDPC(Rdesign=2/3, n=92160)가 기존 2-step lifting 코드([13]) 및 ATSC 3.0 LDPC와 유사한 BLER 성능 달성. Uncoupled code 길이가 충분히 크면 5G-NR LDPC 대비 성능 우위, 최초 오류 uncoupled code 위치가 memoryless(`BLERl ≈ 1-e^{-εl}`)임을 실측 확인.
10. 한계: 디코더는 기존 sliding window decoder([16])를 그대로 사용(알고리즘 개선 없음), HW 미설계, 순수 부호 구성법·시뮬레이션 논문. 구성 복잡도가 매우 커 실제 구성에 최대 10일 소요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호 구조(uncoupled code 결합, sliding window) | 대응 없음 | Prime ECC는 SC-LDPC가 아닌 고정 QC-LDPC(base+circulant shift) 구조라 spatial coupling 개념 자체가 없음 |
| rate-compatible/length-scalable H-matrix 구성(3D-G-PEGL) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 대상이지만 Prime ECC는 단일 고정 QC-LDPC 구조라 구성법 재검증 부담 큼(§4 "부호설계" 항목: 이식 난이도 높음) |
| sliding window decoder | 대응 없음 | Prime ECC 디코더는 전체 블록 단위 반복복호(HCU/pipeline 구조)이며 window 단위 스트리밍 복호 개념 없음 |

적용 가치: **낮음** — SC-LDPC 특유의 spatial coupling·sliding window 구조가 Prime ECC의 고정 QC-LDPC 구조와 근본적으로 다르고, 디코더 알고리즘 개선도 없는 순수 부호 구성법 논문이라 이식 요소가 거의 없음.
