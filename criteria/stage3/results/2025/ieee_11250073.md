### B. 알고리즘 요약

1. 양자 QC-LDPC CSS 코드: 서로 직교하는 두 패리티검사행렬 `(HX, HZ)`, 각각 크기 `P`의 circulant permutation matrix(CPM) `J × L`개로 구성, 코드 길이 `n = PL`, 양자 코딩rate `R = 1 − (rank(HX)+rank(HZ))/n`.
2. 기존에는 유한rate 양자 코드에서 steep(threshold-like) 오류율 전이가 non-binary BP 디코더(계산복잡도가 유한체 크기에 quadratic) 외에는 거의 보고되지 않음 — 이 논문은 binary joint BP만으로도 steep 전이가 가능한지를 조사.
3. 채널 모델은 depolarizing channel, 물리적 오류율 `pD`로 파라미터화.
4. 핵심 기법: joint BP 디코딩으로 신드롬 `(s,t)` 입력 시 `x̂, ẑ`를 동시 추정하여 사후확률 `p(x,z|s,t)`를 근사 최대화; `(HZx̂, HXẑ)=(s,t)` 만족 또는 최대 iteration 도달 시 종료.
5. 성공 판정: `x̂+x ∈ C_X` 그리고 `ẑ+z ∈ C_Z` (coset 조건). 실패 시 잔여 비트오류 수를 코드 길이 `n`으로 나눈 값이 BER.
6. column weight `J=3`, `L=8` 예시로 구성법 제시 — girth 6이 되도록 sub-block(circulant shift 지수) 선택, 기존 column weight 2 구성법의 자연스러운 확장.
7. 코드 길이가 커질수록 FER/BER 곡선이 steep해지고, coding rate가 낮을수록 hashing bound와의 gap이 커짐(고전 binary LDPC에서도 알려진 현상).
8. 별도 학습/최적화 절차 없음 — 시뮬레이션 기반 관찰 연구.
9. 결과: (4,12,1452) 코드에서 error floor 영역 관측 오류의 97%가 `3L` 비트 이하, (4,12,4500) 코드에서는 98%가 `2L` 비트 이하로 국한 — 소수 비트로 구성된 trapping set이 error floor 원인임을 시사.
10. 한계: 순수 시뮬레이션 관찰 연구로 HW 설계 없음, post-processing(trapping set 회피) 기법은 후속연구([6])로 미룸, column weight `J≥3` 및 큰 `L`에 대한 일반화는 future work.
