> 총평: coarse quantization(2~3bit) LDPC 복호에서 이전 iteration CN 메시지를 side info로 재사용해 information loss를 보상하는 기법으로, NAND read 비용에 직결되는 soft-quant 설계 개선이라 Prime ECC 이식 가치가 높음.

### B. 알고리즘 요약 (10줄 내외)

1. 대상은 5G NR QC-LDPC(K=8448, base graph 1, rate 1/3~8/9), BPSK+AWGN 채널, VN/CN 메시지는 `w`-bit sign-magnitude 양자화.
2. 문제: 매 iteration마다 VN/CN 메시지를 새 값으로 덮어쓰는 기존 방식은 coarse quantization(`w`<4)에서 폐기되는 이전 메시지의 mutual information 손실이 커 성능 저하.
3. 핵심 가정: CN 메시지 `t^c_n`(현재 iter)과 `s^c_n`(이전 iter로 보존)의 차 `Δℓ^c_n = ℓ^c_n - ℓ̂^c_n`이 평균적으로 작아, `s`가 `ℓ^c_n` 복원에 보조정보로 유효.
4. 핵심 기법: VN 갱신식 `ℓ^v_n = φ(t^c_chol(n)) + Σ_m φ_α(m)(t^c_m, s^c_m)`처럼 재구성함수 `φ`가 현재 메시지 `t`와 메모리에 저장된 이전 메시지 `s`를 함께 사용(식 (3)).
5. 이 재구성으로 2-bit 양자화 손실을 보상 - 서로 다른 iteration의 서로 다른 quantizer 경계(alternating boundary)를 결합해 실질적으로 3-bit 수준 해상도에 근접.
6. 확장: 기존 sequential IB 알고리즘(Lewandowsky-Bauch)을 side-information `S`를 인식하도록 수정한 Algorithm 1(side-information-aware IB)로 대칭 threshold quantizer `Q_a`를 설계, `I(X;T|S)` 최대화.
7. 부수 트릭: 복잡도 절감을 위해 IB로 설계된 LUT을 관찰해 `Υ(s,t)=sgn(t)(2|t|+s')`, `Ψ(t_Υ)=sgn(t_Υ)⌊|t_Υ|/2^(w-1)⌋`라는 논리게이트 2개(XOR,AND)로 근사 가능한 규칙을 도출(식 (13)(14)).
8. 학습 절차: design-Eb/N0에서 10000개 코드워드로 오프라인 학습(FER 최소화), local optima 회피 위해 500개 초기화 병렬 실행, 실행 중 온라인 적응 없음.
9. 결과: rate 1/3에서 (2,2,4)-bit 메모리보조 디코더가 기존 (2,0,2) 대비 0.23dB, (3,3,6)이 (3,0,3) 대비 0.08dB 개선. CN-aware quantization과 결합 시 최대 0.68dB. row-layered HW 아키텍처(Zp=32 부분병렬)에서 고rate(8/9~11/12)에 한해 최대 32% area efficiency 향상.
10. 한계: 저rate(1/3)에서는 SRAM(side-info 저장)이 전체 면적의 61%를 차지해 이득 감소, HW는 ASIC 게이트면적 추정치 수준(실측 아님), 5G NR 코드 고정 설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| coarse quantization(2~3bit) VN/CN 메시지 설계, LLR 양자화 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | Prime ECC의 HD/2SD/3SD soft-read LLR 테이블 설계에 직접 대응, NAND read 비용 절감과 직결 |
| CN min-sum 업데이트 (식 (2), sign-magnitude min approx) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일한 min-sum 근사 구조이므로 side-info 재구성 로직을 CNU 이후 삽입 가능 |
| 이전 iteration 메시지(side information) 메모리 저장·재구성 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, `decoder.h` `Get_VNU_*` | VN 재구성 시점에 이전 CN 메시지 버퍼 추가 필요 - 기존 파이프라인 6-stage 구조에 메모리 추가 삽입 지점 |
| row-layered 부분병렬 아키텍처(Zp=32), permutation network | 디코더 HW 모델(z_sb=32, GT.cpp `Make_GT_HW()`) | Prime ECC의 z_sb=32 row 병렬 단위와 유사한 부분병렬 구조로 아키텍처 정합성 높음 |
| iteration별 quantizer boundary 최적화(IB 알고리즘) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 LLR threshold 테이블 설계 방법론이 기존 적응형 LLR 테이블과 유사한 목적 |

> 적용 가치: 높음. QC-LDPC coarse quantization(2~3bit) 디코더 성능 개선으로 Prime ECC의 soft-2~3bit(2SD/3SD) LLR 테이블 설계와 직접 맞닿아 있고, NAND read 비용(soft-read 횟수) 절감에 기여할 수 있는 실용적 기법.
