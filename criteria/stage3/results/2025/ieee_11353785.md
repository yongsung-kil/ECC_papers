### B. 알고리즘 요약 (10줄 내외)

1. 표준 CMOS(28nm)에서 coupled relaxation-oscillator(RXO) 스핀으로 COP를 푸는 physics-inspired 컴퓨팅 플랫폼, LDPC 복호를 데모 중 하나로 제시.
2. 기존 디지털 BP는 반복적이고 LLR 정밀도 요구가 커 CMOS 스케일링 이점에도 한계가 있다는 점을 문제로 지목.
3. 채널/코드는 LDPC(5G·위성·케이블 응용 언급)이며 sparse parity-check `H`의 각 row가 parity check 식을 정의.
4. LDPC 복호를 COP로 변환: 시스템 에너지함수 `E`를 스핀·클로즈 상호연결로 매핑, 에너지 최소값들이 각각 valid codeword에 대응(Fig. 8).
5. 수신 신호(오류 포함)로 스핀 초기조건 설정 후 방출하면 시스템이 가장 가까운 valid code로 수렴 — 반복 대신 CT 동역학으로 수렴.
6. 아키텍처: 메시지 변수를 RXO 스핀에 직접 매핑, twin crossbar가 오실레이터 출력을 XOR 게이트(parity check)에 연결하고 XOR 출력을 피드백.
7. hard 정보는 RXO 상태에, soft 정보는 capacitor 전압 초기조건에 인가 — soft init이 BER/throughput을 크게 개선.
8. 각 RXO는 SDAC(위상/커패시터 전압 초기화)+FDAC(relaxation threshold 변조)로 CT 오실레이터 피드백 구현, 96 compute node·XOR0~47.
9. 결과(실측, 28nm, core 0.21mm², 5-35MHz): 10^8 복호 사이클 측정, 2-5dB SNR에서 BER가 BP 대비 약 3자릿수 우수, 정규화 에너지도 SOTA 디지털 LDPC 디코더보다 우수.
10. 한계: 특정 아날로그 회로(오실레이터·크로스바)에 종속, 코드 rate/length·게이트카운트 본문 미기재, 디지털 Min-Sum 알고리즘 개선 요소 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| RXO 오실레이터 기반 아날로그 복호(에너지 최소화) | 대응 없음 | CT 아날로그 동역학 — 디지털 Min-Sum/message-passing 루프에 매핑 불가 |
| XOR crossbar parity check | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `C2V_Cal()` | 개념상 parity check지만 아날로그 XOR-피드백이라 알고리즘 이식 아님 |
| soft-info(capacitor 전압) 초기화 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | soft init 발상만 유사, LLR 양자화 테이블과 무관 |
