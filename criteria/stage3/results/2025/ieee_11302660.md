> 총평: 복호기 자체는 교과서 계층형 Min-Sum(QC-LDPC)이고 신규성은 UDP/Ethernet 통합·Smart-NIC 배포에 있어 NAND binary QC-LDPC 이식 가치가 낮다(이식성 하).

### B. 알고리즘 요약

1. 시스템: 3GPP 5G NR QC-LDPC(BG1/BG2), lifting `Z=2~384`(예제 `Z=384`, rate `1/5`), 프레임당 `19200` LLR, 4-bit signed LLR(`[-8,+7]`). NetFPGA-SUME(Virtex-7 690T) 실장.
2. 문제: 기존 FPGA LDPC 복호기는 throughput/자원만 강조하고 네트워크 통합·자율운용을 무시 → PCIe(≤1m) 한계로 원거리 오프로딩 불가.
3. 핵심 아이디어: 복호기 + full UDP/IP/Ethernet 스택을 한 FPGA에 통합, dual-clock(네트워크 156MHz / LDPC 75MHz) + async FIFO로 도메인 크로싱.
4. 복호 알고리즘: `layered Min-Sum`. CTV 메시지 `σcv = α·(∏sign(qv'c))·min|qv'c|`, offset 계수 `α`. 각 layer는 `Z` 연속행, 컬럼당 layer마다 1개 non-zero로 intra-layer 병렬.
5. VN 갱신: `qvc = yv + Σσc'v`(현 CN 제외), `zv = yv + Σσcv`(전체). `sign(z)·Hᵀ=0` 또는 `i=Imax`(=10) 시 종료.
6. 부수 트릭: base graph/lifting/rate를 parameter-driven synthesis로 런타임 구성, early termination으로 latency-정정 trade-off.
7. HW 구조: 네트워크 도메인(Ethernet MAC/IP/UDP)과 LDPC 도메인 분리, distributed RAM(VN 병렬) + Block RAM(FIFO 버퍼).
8. 검증: 실측(NetFPGA-SUME), MATLAB로 Nakagami-m(m=2)+AWGN 채널 LLR 생성해 하드웨어 BER 대조.
9. 결과: throughput 181.51Mbps(시스템)/1.19Gbps(LDPC 코어), end-to-end latency 423.1µs, 자원 15.1% LUT·69.1% BRAM, 5.731W, `Eb/N0=6dB`서 BER `4.688e-5`, 6dB까지 error-floor 없음.
10. 한계: 5G NR 전용, 복호 알고리즘 신규성 없음(표준 계층형 Min-Sum), 정정능력·iteration 개선 없음(기여는 시스템 통합).
