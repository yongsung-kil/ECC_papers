### B. 알고리즘 요약

1. 대상은 LDPC 부호의 Tanner graph를 포함한 일반 bipartite graph `G=(U∪W,E)`, girth `g`(최단 cycle 길이) 기준 `g`부터 `2·lmax`까지 길이의 short cycle 개수 `η_2l`을 계산하는 문제.
2. 풀려는 문제: short cycle(특히 trapping set을 구성하는 요소)은 LDPC error floor를 지배하는데, 일반 그래프의 cycle counting은 NP-hard이며 기존 IMPA는 계산복잡도 `O(g|E|²)`로 g가 클 때 비효율적.
3. 핵심 모델: 그래프 노드 간 message-passing으로 root `u`에서 시작하는 길이-`t` 경로 개수 `f_i(e,t)`를 매 시간 슬롯마다 갱신, 이를 이용해 root를 지나는 cycle 개수 계산.
4. 핵심 기법(IMPA 재구현): 활성 노드 집합 `A_t`(t가 짝수면 W, 홀수면 U 전체)에 대해 메시지 갱신식 `f(e,t) = Σ_{e'∈Ne(v)} f(e',t-1) - f(e,t-1)`을 적용해 경로 수 `p_i(v,l)`, `q_i(e,l)`을 계산.
5. 핵심 식 의미: `η2l_{u,v} = Σ_i Σ_e f_i(e,l)·q_i(e,l)` — 같은 노드 v에서 서로 다른 시작 edge를 가진 두 길이-l 경로 쌍이 곧 길이-2l cycle을 이룬다는 조합적 관계(Lemma 1)를 메시지 곱으로 계산.
6. 확장(FMPA): 기존 IMPA의 활성 집합 `A_t = W or U`(과대추정)를 실제 레벨 집합 `A_t = L_{t-1}`(root에서 길이 t-1 경로가 존재하는 노드만)로 축소하는 level-wise activation 전략을 도입.
7. 부수 구현 디테일: `A_t`를 명시적으로 미리 알 수 없으므로 Algorithm 2에서 `f_i(e,t)>0`일 때만 다음 활성집합 `A_{t+1}`에 노드 추가하는 방식으로 온라인 구성.
8. 복잡도 분석(수학적 유도, 학습·최적화 절차 없음): FMPA storage `O(δ|E|)`(δ=두 파티션 중 작은 최대차수), 계산량 `O((lmax-g/2+1)|E|²)`로 IMPA(`O(g|E|²)`) 대비 g배 절감(lmax-g/2가 상수일 때).
9. 결과: C1~C5(Tanner code, PEG 계열, 802.3an 코드, 20000×10000 코드 등) 5개 LDPC 코드에서 FMPA가 IMPA 대비 항상 유의미하게 빠름(예: C5에서 lmax=g/2일 때 0.63초 vs 267.40초), MBFSA와는 lmax=g/2+2에서 FMPA가 수배 빠름.
10. 한계: 순수 그래프 알고리즘/실행시간 비교 논문으로 실제 LDPC 디코딩 성능(BER/FER)이나 코드 설계에 적용한 결과는 제시하지 않으며, NAND/SSD 등 스토리지 응용 언급 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Tanner graph short cycle counting (FMPA) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 후 오프라인 girth/absorbing-set 분석 툴로만 잠재 활용 가능, 디코더/인코더 로직과 무관 |
| trapping set 관련 언급(cycle이 trapping set 구성요소) | 대응 없음 | Prime ECC는 trapping set 대응 코드가 없고, 이 논문도 trapping set 자체를 다루지 않음(cycle 개수만 카운트) |
| 계산복잡도 분석(그래프 이론) | 대응 없음 | 디코더/인코더/HW 어느 모듈과도 직접 대응 없음 |

> 적용 가치: **낮음** — LDPC 디코더·인코더·HW·부호설계 어느 이식 레이어에도 해당하지 않는 순수 그래프 알고리즘(오프라인 cycle counting 도구)으로, Prime ECC 코드베이스에 이식할 실질적 대상이 없음.
