### B. 알고리즘 요약

1. 대상은 MET-QC-LDPC(base matrix 원소 `Bi,j`가 1보다 크면 multi-edge) 코드의 pipelined layered BP(LBP) 디코더.
2. 문제: 같은 layer 내 check node들이 공통 variable node(common degree)를 공유하면 파이프라인 read/write 충돌이 발생, idle cycle 삽입으로 latency 증가·throughput 저하.
3. 가정: `z`-fold lifting, `z→∞`인 protograph 기반 QC-LDPC, 각 layer는 QC block 2개로 구성.
4. 핵심 기법: layer-by-layer 대신 두 layer `si,1`, `si,2`를 번갈아(alternating) 갱신하는 cross-layer scheduling(`Scl = {s1,1,s1,2; ...}`).
5. 핵심 식: 같은 layer 내 기대 공통차수 `Ec=(a1-1)a1p/z+(a2-1)a2p/z`, 교차 layer 갱신 시 `Ec'=b1a1p/z+b2a2p/z`. multiplicity 차이(a1,a2 vs b1,b2)가 클수록 `Ec' < Ec`가 되어 충돌 감소.
6. 확장: layer 간 "common degree" 정의(각 variable node의 최소 multiplicity 합)를 도입해 짝짓기 기준으로 사용.
7. 구현 디테일: Algorithm 1(Paired Algorithm)로 최고차수 layer부터 탐욕적으로 common degree가 가장 작은 layer와 짝지어 스케줄 시퀀스 `Scl` 생성. layer 내부는 memory conflict 최소화하는 greedy row 선택 병행.
8. 최적화 절차: 탐욕(greedy) 알고리즘 기반 pairing, 별도 학습/DE 없음(비교군 TSP-layer-by-layer만 differential evolution 사용).
9. 결과: rate-compatible LDPC(`k=1032`, rate 8/9~1/3, base matrix는 [25])와 3GPP MET-QC-LDPC(`k=384`)에서 중~저rate 구간 memory conflict 50~75% 감소, BLER 성능은 layer-by-layer/TSP 방식과 동일(성능 저하 없음).
10. 한계: HW 미설계(시뮬레이션 수준), 고rate(layer 수 적음) 구간에서는 pairing 풀이 작아 효과 제한적, 이 방식은 layer 간 충돌만 다루고 layer 내부 multi-edge 충돌은 여전히 미해결.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered BP 디코딩 이터레이션/스케줄 순서 | `decoder.cpp` `LDPC_Decoder()` | cross-layer 스케줄은 layer 처리 순서 변경이라 이터레이션 루프 구조에 닿음 |
| pipeline read/write memory conflict(idle cycle) | 디코더 HW 모델 (z=32, HCU, pipeline 6-stage) | Prime ECC도 pipeline 기반이나 single-edge라 layer 내 공통 VN 충돌 자체가 이 논문이 다루는 형태로 발생하지 않음 |
| MET-QC-LDPC base matrix(multi-edge) | `ecc_top.cpp` `Load_PCM()` | Prime ECC PCM이 multi-edge 구조(Bi,j>1)인지 확인 필요, 확인 전까지 대응 없음에 가까움 |
| HCU(비대칭 degree column 2-cycle 처리) | `decoder.cpp` HCU 로직 | 본 논문의 layer 쌍 처리와 유사한 "특수 구조 페어링" 아이디어이나 대상(column vs layer)이 다름 |

> 적용 가치: `중간` — 아이디어(스케줄 순서로 충돌 완화)는 일반적이나 Prime ECC가 single-edge QC-LDPC 전제라 논문의 핵심 문제(MET 특유 layer 내 공통 VN)가 그대로 이식되지 않음. Prime ECC가 향후 multi-edge 구조를 채택할 경우 재검토 가치 있음.
