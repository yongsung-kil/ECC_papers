### B. 알고리즘 요약

1. 대상은 짧은 BCH 코드(block length 63, 127, rate 0.51~0.78)로, BP/NMS 디코딩에 불리한 고밀도(HDPC) PCM을 갖는 대수적 순환 코드.
2. 표준 LDPC용 BP는 Tanner graph에 짧은 사이클이 많은 HDPC 코드(BCH/RM)에서는 성능이 크게 열화되는 문제를 해결하려 함.
3. 채널은 AWGN + BPSK, 표준 normalized min-sum(NMS) 갱신식 `x_cj→vi = α·sgn(·)·min|x_vk→cj|`을 기반 모델로 가정.
4. 핵심 기법 1단: 표준 H를 row echelon화 후 이진 덧셈으로 밀도를 최소화하고, 순환 시프트 탐색과 heuristic(simulated annealing 유사)로 length-4 cycle과 column weight 분산을 최소화한 최적화 PCM `H_o`를 구성(redundancy factor `β`로 행 수 조절).
5. `β` 증가는 FER을 개선하다 특정 지점(예: BCH(63,36)에서 β=50) 이후 복잡도만 늘고 성능은 오히려 악화되는 트레이드오프 관계.
6. 핵심 기법 2단: 3종 automorphism(interleaving `π_I`, Frobenius mapping `π_F`, cyclic shift `π_Cs`)을 수신 시퀀스 y에 조합 적용해 크기 9(`|Sn||Sp|=9`)의 입력 블록 `B(y)`를 만들고, 각 이터레이션 말미에 역치환·평균해 메시지를 병합(단일 루프, 다중 automorphism).
7. 부수 트릭: 코드의 rank deficiency(최소중량 dual codeword로 full-rank Ho를 구성 불가한 성질)가 FER에 영향을 미침을 분석해 rate·최소거리에 따라 (63,45)/(127,78) 같은 코드가 더 유리함을 규명.
8. normalization factor `α`는 신경망 형태로 학습해 특정 SNR에서 고정(NMS를 뉴럴넷으로 취급해 학습, TensorFlow 구현 공개).
9. BER 10⁻³ 기준 기존 parallelizable 디코더(BP-RNN, RNN-SS, GPT) 대비 1.5dB 이상 이득, mRRD 계열 대비 최대 100배 빠른 수렴; hybrid NMS+order-2 OSD로 near-ML 성능 달성.
10. HW 구현·합성 결과 없음(시뮬레이션 전용), 고rate 짧은 코드(63,45)에서는 undetected BER floor로 개선 제한, hybrid 구성 시 OSD는 순차처리라 완전병렬 이점이 희석됨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| normalized min-sum 갱신식 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 알고리즘 형태는 동일 계열이나 이미 보유(§3), BCH 특화 automorphism 병합은 QC-LDPC에 대응 없음 |
| PCM 최적화(밀도/중복도/cycle 절감) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 H-matrix, BCH 대수구조 기반 재구성 기법은 이식 난이도 높음 |
| automorphism 앙상블(3종 순열) | 대응 없음 | QC-LDPC circulant 구조에는 BCH 고유의 Frobenius/cyclic automorphism 개념이 성립하지 않음 |
| hybrid NMS+OSD | 대응 없음 | Prime ECC 디코더 구조에 OSD 후처리 단계 없음 |

> 적용 가치: 낮음 — 핵심 아이디어(NMS)는 이미 보유한 기법이며, 논문의 실질 기여(PCM 재구성·automorphism)는 BCH 고유의 대수적 순환 구조에 종속되어 QC-LDPC 기반 Prime ECC로 일반화하기 어려움.
