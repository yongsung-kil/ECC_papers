### B. 알고리즘 요약

1. 시스템은 짧은 이진 선형블록코드 `C(n,k)` (n=32~256, rate 1/2~0.71)를 AWGN+BPSK 채널로 전송, URLLC(짧은 코드) 대상.
2. 문제: 짧은 LDPC는 패리티체크행렬 sparsity 부족으로 BP 디코더 성능이 MLD 대비 크게 열화되고, 인코더·디코더가 별도 최적화되어 전역 최적이 아님.
3. 채널 모델은 표준 AWGN(BPSK), LLR `λ_i = 2y_i/σ_n^2`; 별도 NAND 특화 채널 모델 없음.
4. 1단(코드 설계): DRL(DDPG)이 패리티체크행렬을 lattice graph로 보고 MPMNN(actor/critic)으로 각 원소를 플립하는 액션 `A_t∈R^{m×n}`을 반복 생성하는 full graph-based MDP.
5. reward는 유효성(full-rank) + decoding BER reward `r_d=|ln(ε_b(S_{t+1}))|` + code structure reward(최소해밍거리 `d_min`, 4-cycle 개수 `c_λ`) 조합으로 girth/BER/MHD를 동시에 최적화.
6. 2단(디코더): EW-GNN이 Tanner graph 위에서 BP 유사 message-passing을 수행하되, C2V 메시지에 학습된 가중치 `w_{u→v}^{(t)}`(작은 사이클發 비신뢰 메시지를 억제)를 곱함.
7. 구현 디테일: BP의 `tanh^{-1}`을 클립 함수로 대체해 비수렴 문제 회피, 멀티로스 BCE로 반복별 손실 학습.
8. 학습 절차: DRL 코드설계자와 EW-GNN 디코더를 τ회 번갈아 학습하는 iterative joint training(직접 공동학습은 파라미터공간이 너무 커서 불가).
9. 결과: (32,16) 코드에서 MLD 대비 0.83dB, BP 대비 0.65dB coding gain; (63,51) BCH 대비 EW-GNN이 BP 대비 1.2dB, NBP 대비 0.62dB 개선; auto-encoder는 LDPC+MLD 대비 0.63dB 개선.
10. 한계: 순수 시뮬레이션(TensorFlow)만 수행, HW 설계·gatecount·throughput 전혀 없음. 코드 길이가 최대 256으로 NAND ECC(수천~수만 bit) 규모와 무관하며, 오프라인 DRL 학습 후 고정 코드/디코더 배포 구조.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EW-GNN의 C2V 가중치 메시지 | `decoder.cpp` `C2V_Cal()`, `CNU_Update_New_Mag()` | min-sum 대신 학습형 가중치를 곱하는 구조라 발상은 유사하나 NN 추론이 필요해 그대로 이식 불가 |
| DRL 기반 패리티체크행렬 설계 | `ecc_top.cpp` `Load_PCM()` | 짧은 코드(n≤256) 전용 구성법으로 QC-LDPC 구조(base+circulant, z=32) 미고려, 재검증 부담 큼 |
| 클립 함수 기반 tanh^-1 근사 | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC는 이미 min-sum(테이블 기반 양자화)을 사용 중이라 중복 |
| soft LLR(연속값) 입력 | `channel.cpp` `Set_LLR_Th()` | 논문은 soft-4bit+(사실상 float LLR) 가정, NAND read 비용(hard/2~3bit) 관점과 불일치 |
| GNN 추론 자체(NN 가중치) | 대응 없음 | Prime ECC는 순수 min-sum/HW 파이프라인, NN 추론 엔진 없음 |

적용 가치: **낮음** — 짧은 코드(URLLC) 전용 DRL+GNN auto-encoder로, NAND용 고rate·대용량 QC-LDPC 규모·구조와 맞지 않고 NN 추론 HW 비용도 검증되지 않음.
