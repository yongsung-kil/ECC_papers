### B. 알고리즘 요약

1. AWGN(BPSK) 채널에서 선형블록코드(LDPC/BCH/Polar) 공통 프레임워크로 동작하는 model-free transformer 디코더(ECCT) 개선안, LDPC 예시는 `(576,432)`, `(512,256)`, `(256,128)`, `(1056,528)`, `(155,64)` 등 rate 0.41~0.75.
2. 기존 ECCT는 중간~대형 code length에서 성능 유지를 위해 깊은(층이 많은) 구조가 필요해 모델 크기와 시간복잡도가 급증하는 확장성 문제가 있음.
3. syndrome 기반 reliability decoding을 가정: 채널출력 `y`의 magnitude `|y|`와 syndrome `s(y)=Hy_b`를 결합한 `ỹ`를 입력으로, 곱셈성 잡음 `z̃_b`를 추정하는 이진 크로스엔트로피(BCE) 손실로 학습.
4. U-ECCT: U-Net처럼 main flow(4단계 hidden dim `d1<d2<d_model<d3<d4`)와 shortcut connection을 element-wise 곱으로 결합, `d1=½d_model, d2=¾d_model, d3=5/4 d_model, d4=3/2 d_model`.
5. shortcut을 concat 대신 곱셈으로 결합하는 것이 embedding 공간에서 오류 비트 영역을 좁히는 위치 가중치 역할을 해 성능 향상에 기여.
6. VU-ECCT: shortcut 결합을 variational autoencoder(VAE) 구조로 대체, 재매개변수화 `s̃=μ+σ⊙ε`로 잠재공간을 근사하고 손실에 KL-divergence 항(`D_KL(N(μ,σ²I)‖N(0,σ_y²I))`)을 추가해 저rate(긴 syndrome) 코드 성능을 강화.
7. mirror-sharing: U자형 구조의 후반부가 전반부 transformer layer 파라미터를 재사용(rescale layer는 제외), M=10층에서 실질 M/2세트만 학습해 모델 크기를 최대 73%까지 압축.
8. 학습은 Adam optimizer, cosine annealing 스케줄, all-zero codeword 학습(BP/MS 대비 syndrome-based 접근이 codeword 시뮬레이션 필요 없음을 이용), Eb/N0 2~7dB 구간에서 훈련.
9. 결과: 길이 256~1024 폴라/LDPC 코드에서 baseline ECCT 대비 평균 0.3~0.6dB 성능 이득(Eb/N0 기준), VU-ECCT는 저rate 코드에서 최대 0.7dB 이득, mirror-sharing Type II 모델은 파라미터 27%(73% 감소)로도 baseline보다 우세.
10. GPU(RTX 4090) 기반 PyTorch 시뮬레이션만 수행, HW(RTL/ASIC/FPGA) 설계 없음. 대형 code length에서는 하드웨어 메모리 제약으로 배치 크기를 줄여 학습해 성능이 제한될 수 있음을 한계로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 U-ECCT/VU-ECCT 디코딩 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()` | min-sum 이터레이션 루프를 transformer 신경망으로 전면 대체하는 구조라 알고리즘 층위에서 대응 안 됨 |
| positional reliability encoding / soft magnitude 입력 | `channel.cpp` `Set_LLR_Th()` | LLR 양자화 개념과 유사점은 있으나 신경망 임베딩 방식으로 HW LLR 테이블과 직접 대응 불가 |
| mirror-sharing 가중치 압축 | 대응 없음 | 신경망 파라미터 공유 기법으로 Prime ECC의 min-sum/HCU HW 구조에 대응 개념 없음 |

> 적용 가치: 낮음 — NN(transformer) 전면 대체 디코더는 프로파일 §5의 "대응 없음" 판정 대상이며 Prime ECC의 min-sum 기반 bit-exact HW 디코더 구조와 근본적으로 이질적.
