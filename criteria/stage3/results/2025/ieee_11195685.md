### B. 알고리즘 요약

1. 대상은 length-63 BCH 코드 및 extended Golay(24,12) 등 짧은 대수적 블록코드, AWGN 채널에서 BPSK 전송 후 BP 디코딩.
2. 문제: 짧은 코드는 PCM이 조밀(dense)하여 4-cycle이 많아 BP 디코딩 성능이 ML 대비 크게 열화됨.
3. 채널/노이즈 모델은 binary AWGN, 표준 정규화 min-sum(NMS) BP 사용, damping factor `α`(0.5 또는 0.7)로 근사.
4. 핵심 기법: 최소가중치 parity-check row 중 겹치는 컬럼수 `t=w0/2`인 행들을 모아 auxiliary VN(AVN) `v'` 하나와 새 CN `c'`를 도입해 parity-check row block(PCRB)을 구성 (`c′: v1⊕v2⊕⋯⊕vt⊕v′=0`).
5. Lemma 2/3: 단일 PCRB 내부는 cycle-free이며, 서로 다른 PCRB의 AVN들은 같은 CN을 공유하지 않아 AVN들만으로 이루어진 cycle이 존재하지 않음을 증명.
6. 여러 PCRB를 결합해 rank(N−K)를 만족시키면 `ssPCM-I`, 성능 향상을 위해 PCRB를 더 추가하면 `ssPCM-II`로 확장.
7. PCRB 선택 시 3가지 규칙(rank 증가, ssPCM 내 4-cycle 최소화, coded-bit 컬럼 가중치 분산 최소화)으로 진행형(progressive) 탐색.
8. 별도 학습 절차는 없고, 낮은 가중치 parity-check row 탐색 공간을 [10]의 Algorithm 2 기반으로 구성 후 그리디하게 PCRB 조합.
9. 결과: extended Golay(24,12)에서 ssPCM-II가 ML 대비 0.35dB 이내 근접, 기존 원본 PCM 대비 edge수 10%만 사용. BCH(63,16/18/30)에서 [10] 대비 최대 1dB 성능 우위(edge는 1.2~1.4배).
10. 한계: 짧은 대수적 코드(N≤63) 전용 설계이며, HW 구현/합성 결과 없음(순수 알고리즘·시뮬레이션 단계), 이 방법을 긴 QC-LDPC 코드로 확장하는 것은 향후 과제로 언급될 뿐 실증되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 짧은 BCH/Golay용 ssPCM 구성(PCRB 기반) | 대응 없음 | Prime ECC는 고정된 QC-LDPC H-matrix(`Load_PCM()`) 사용, 짧은 대수적 코드용 PCM 재설계 기법과 직접 대응 없음 |
| NMS BP 디코딩 (α damping) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 min-sum 계열과 유사하나 이 논문은 코드 구조 개선이 핵심이라 디코더 알고리즘 자체의 개선은 없음 |
| AVN(auxiliary VN) 도입 방식 | 대응 없음 | Prime ECC PCM(`PCM_b`)은 punctured column 2개(HCU)만 존재, AVN 개념과 목적이 다름 |

적용 가치: **낮음** — 짧은 대수적 코드(N≤63) 전용 PCM 재설계 기법으로 NAND용 고rate QC-LDPC(N이 수천~수만) 이식에 스케일·구조 면에서 부적합.
