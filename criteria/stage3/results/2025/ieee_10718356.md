### B. 알고리즘 요약

1. 시스템: BPSK+AWGN, 짧고 고rate인 선형블록부호(polar `(128,105+11)`, `(256,240)`) 대상 code-agnostic ML 복호.
2. 문제: ORBGRAND의 noise sequence 생성(LWO/iLWO)이 query 복잡도(특히 worst-case)가 커 HW throughput/면적 효율 병목.
3. 모델: 정렬된 `|LLR|`를 기울기 `β`의 선형근사로 보고 `Rel(z̃)≈Σ i·z̃_i`(logistic weight)를 최소화하는 순서로 noise 후보 열거.
4. 핵심기법1: `sLWO`(shift LWO) - `sHW+LW=W`로 shift factor `s`를 도입, `s`로 FER와 query 복잡도를 경험적으로 절충(`s=0`이면 LWO).
5. 핵심식 의미: `s`가 클수록 고SNR에서 절대 LLR 분포를 더 정확히 근사해 FER↑, worst-case query 대폭↓(`s=6`이면 worst-case query 98.6%↓).
6. 핵심기법2: noise 생성을 distinct integer partition 문제로 재정식화, 인접 partition 차분벡터 `d`로 다음 partition을 online 유도.
7. 구현 트릭: base partition seed만 다중 독립 메모리에 저장하는 LUT-aided 방식, 저장 sequence/bit를 [22] 대비 각각 81~84%·95%↓.
8. HW: memory/sorter(folded odd-even)/APCU/`P`개 PGU·NGU·PCU 병렬 파이프라인, `P`,`Q`로 병렬도 조정.
9. 결과: TSMC 65nm 합성, 평균 53.6Gbps(1.26×), worst-case throughput 4.2Mbps(8.24×), worst-case 면적효율 12×, `|LLR|` 5-bit 양자화.
10. 한계: polar code에서만 합성·검증, gate count 등 절대 복잡도 본문 서술 없음, LDPC 이식성·확장성 미논의.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GRAND guess-and-check 복호 패러다임 | 대응 없음 | Prime ECC는 iterative min-sum BP, GRAND과 근본적으로 다른 디코딩 방식 |
| `\|LLR\|` 5-bit 양자화·정렬 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | soft-read LLR 처리 개념만 느슨히 연관, GRAND은 소트+플립 |
| 병렬 partition/noise 생성 HW(PGU/NGU) | 대응 없음 | min-sum CN/VN 병렬화(z=32, GT/HCU)와 구조 상이 |
| parity-check 검증(PCU) | 대응 없음 | Prime ECC는 CRC 조기종료+syndrome, guess-and-check 아님 |

적용 가치: **낮음** - GRAND/polar 대상 code-agnostic 디코더로 NAND binary QC-LDPC min-sum 파이프라인과 이질적이라 재사용 요소가 없음.
