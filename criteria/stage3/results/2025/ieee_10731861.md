### B. 알고리즘 요약 (10줄 내외)

1. 대상은 product code(PC)/staircase code(SC) 등 product-like code이며 성분부호는 BCH, 채널은 Bi-AWGN.
2. 기존 소프트보조 HDD인 iBDD-SR은 BDD 출력과 채널 LLR을 가중합한 soft reliability로 다음 반복 입력을 만드는데, 이 값의 절댓값이 작을 때(신뢰도 충돌 시) 오류에 취약해 성능 정체가 발생.
3. 채널 모델은 BPSK + Bi-AWGN, `Li = 2yi/σ²`로 채널 LLR 계산.
4. 핵심 기법: soft reliability `u_i(l) = w(l)·ū_i(l) + L_i`가 임계값 `T` 이하이면 확률 `Pf(l)`로 부호를 반전시키는 iBDD-RF(iterative BDD with Random Flipping) 제안.
5. 반전 규칙 `û_i(l) = -u_i(l) (prob Pf(l))`은 국소최적점에 갇힌 디코더가 탈출하도록 노이즈를 주입하는 역할.
6. 확장으로 PC와 SC 각각에 대해 density evolution(DE) 방정식을 유도하여 스케일링 인자 `w(l)`와 플립확률 `Pf(l)`, 임계값 `T`를 최적화.
7. 구현 디테일: 마지막 2회 반복은 순수 iBDD로 마무리, 무작위수는 LCG로 생성(O(1) 복잡도).
8. 최적화 절차: DE 기반 파라미터 근사 후 T만 수치탐색(step 0.1)하며, optuna 하이퍼파라미터 탐색과 비교해 DE가 20~100배 빠른 것을 확인(0.5시간 vs 20~100시간).
9. 결과: (255,239,2) BCH 기반 PC에서 iBDD-SR 대비 0.25dB, (254,230,3) BCH 기반 SC에서 0.28dB 개선. 복잡도는 iBDD-SR보다 약간 높고 DRSD보다 낮음(O(3.648n³)+O(2.96n) vs O(4.2n³)+O(2n²logn) 예시).
10. 한계: HW 구현 미설계, 순수 시뮬레이션(Bi-AWGN)뿐이며 BCH 기반 product-like code 프레임워크 전용으로 QC-LDPC 반복 메시지패싱(min-sum)에는 직접 적용되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| iBDD-RF 랜덤 플립(soft reliability 낮을 때 확률적 비트반전) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | min-sum LLR 갱신에 유사한 확률적 플립 규칙을 적용해 stuck pattern 탈출 시도 가능(개념 참고용) |
| BDD(Berlekamp-Massey) 성분 디코더 | 대응 없음 | Prime ECC는 QC-LDPC min-sum이며 BCH BDD 서브루틴 자체는 대응 없음 |
| DE 기반 파라미터(w(l), T, Pf(l)) 최적화 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 LLR 테이블 최적화 방법론은 개념적으로 유사하나 product code 전용 수식 |
| 무작위수 생성(LCG) | 대응 없음 | Prime ECC는 XOR25 난수(`random.cpp`) 사용, 직접 대응 없음 |

> 적용 가치: 낮음. product-like code(BCH) 전용 iBDD 프레임워크로 QC-LDPC 부호/디코더 구조와 근본적으로 다르며, NAND 직접 타깃도 아님. 다만 "낮은 신뢰도 영역 확률적 플립으로 local optimum 탈출" 아이디어는 min-sum error-floor 개선 참고용으로만 가치 있음.
