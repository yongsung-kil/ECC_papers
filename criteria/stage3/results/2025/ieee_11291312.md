### B. 알고리즘 요약

1. qLDPC(quantum LDPC) 부호, symmetric depolarising 채널(p, pX=pY=pZ=p/3), B1[882,24,≤24] generalised bicycle 코드로 실험.
2. MP(min-sum, MS) 단독 디코딩은 degeneracy·짧은 cycle로 인해 수렴 실패 시 error floor가 높음; OSD-0 등 PPT는 성능은 좋으나 Gaussian elimination 등으로 latency가 큼.
3. 채널은 depolarising 모델, 신드롬 `S = eH^T` 기반 복호, 소프트 정보는 LLR `Lj = log((1-2p/3)/(2p/3))`.
4. 1단 MS(`σi,j`, `Ri,j` 메시지, 정규화 factor α) 후 수렴 실패 시 2단 bit-flipping(GA/WBF/RBF) 적용, Algorithm 1(MP-BF)로 기술.
5. WBF는 GA의 hard reliability `Ej`를 MS의 soft 메시지 합 `Wi = Σ Ri,j`("lost-in-between" reliability)로 대체한 `Ej = -Σ(1-2Wi(⊕...))`을 사용.
6. RBF는 loop detector(iteration L1부터 L2개 신드롬 저장)로 GA가 cycle에 갇히면 최저 reliability 큐빗 중 랜덤 부분집합(크기 q)을 flip.
7. Early stopping: 현재/이전 신드롬 간 Hamming distance가 variable node degree dv 이하이면 MS를 중단하고 해당 시점 오류벡터·소프트정보를 BF 단계로 전달.
8. 별도 학습/최적화 절차 없음(α는 실험적으로 알고리즘별 고정값 선택: GA/WBF 0.75, OSD 0.625, SI 0.9, RBF 0.875).
9. p=0.05에서 WBF는 MS-GA 대비 약 2 orders of magnitude, RBF는 약 3 orders of magnitude logical error rate 개선; worst-case 복잡도는 MS-BF `O(2Tn)`으로 SI/GD의 `O(Tn²)`, OSD-0의 `O(n³)`보다 낮음.
10. HW 미설계(순수 알고리즘/시뮬), qLDPC 특화 채널·신드롬 모델 가정이며 classical binary LDPC로의 직접 이식은 재해석이 필요함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MS(min-sum) 1단 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유한 min-sum과 동일 계열 연산 구조 |
| BF(GA/WBF/RBF) post-processing | 대응 없음 | Prime ECC는 min-sum 기반이며 별도 bit-flipping post-processor 미보유; 이식 시 신규 모듈 필요 |
| Early stopping(신드롬 Hamming distance 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 개념은 유사하나 판정 기준(CRC vs 신드롬 거리)이 다름 |
| 소프트 정보 기반 reliability(lost-in-between) | `decoder.cpp` `VN_Cal_HD()` (LLR 참조) | soft 메시지 활용 관점에서만 참고 가능, 직접 대응 함수 없음 |

> 적용 가치: 중간 — qLDPC 특화 기법이지만 "min-sum 실패 후 저지연 bit-flip post-processing으로 error floor 절감"이라는 구조적 아이디어는 classical binary LDPC 디코더에도 재구성 가능성이 있음.
