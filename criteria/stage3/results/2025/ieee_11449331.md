### B. 알고리즘 요약

1. (5,50)-regular (36800, 33120) LDPC 코드(rate≈0.9), SLC NAND 채널, Fixed Number Noise(FNN) 모델(AWGN 대신 hard-decision 오차개수 t로 모델링).
2. GDBF(gradient descent bit-flipping)는 강력하지만 매 iteration syndrome 재계산과 GD(gradient descent) 메모리 접근으로 전력 소모가 큼 — SLC는 4KB당 error bit ≤8인 저오류 환경이라 이를 활용해 전력을 더 줄이는 것이 목표.
3. 핵심 가정: dynamic SLC 블록은 RBER이 매우 낮아(`P(t≥7)`≈6.13E-10) 단순화된 디코더로도 충분히 정정 가능.
4. 제안 ULPBF는 에너지함수를 `E_ULP[j]=Σ_{i∈M(j)} s_i`로 단순화(GD 비트 항 제거), threshold `T_ULP=4` 고정, 단일 iteration(`Lmax=1`)만 수행.
5. syndrome 기반 정정 성공판정 대신 CRC-32(32bit parity, LDPC syndrome 3680bit 대비 초경량)로 코드워드 유효성 검증 — undetected error `P≤2.97e-6 × 2^-32 ≈ 6.9e-16`.
6. 확장(2단계 전략): 1단계 ULPBF 실패 시 동일 HW 아키텍처를 공유하는 GDBF로 폴백(초기 syndrome 재계산 없이 즉시 전환).
7. 부수 트릭: 에러패턴 분석(`cxf` 표기)으로 T_ULP 임계값을 경험적으로 선정, good bit과 error bit의 unsatisfied-check 패턴 분포 차이를 근거로 함.
8. 학습/최적화 절차 없음 — 통계적 패턴 관찰(107 chunk 시뮬레이션)로 threshold 결정.
9. 결과: TSMC 12nm SpyGlass 합성 기준 decoding 상태에서 GDBF 대비 전력 42% 절감, output 상태 전력 0%(CRC 이미 계산됨), 전체 전력은 GDBF의 67% 수준. BLER은 worst-case(PE=100K, 10yr retention)에서 5.08e-9.
10. 한계: 저오류 SLC 전용(t≥7에서는 사실상 실패 가정), TLC/QLC 등 고오류 채널에는 부적합하여 GDBF 병행 필요, throughput/gatecount 수치는 본문에 없음(미상).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping 저전력 디코더(ULPBF), 단순화된 에너지함수 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 기반이라 BF 계열 디코더 자체는 대응 없음, 다만 SLC 저오류 조기종료/저전력 경로 아이디어는 이터레이션 루프에 참고 가능 |
| CRC 기반 조기종료(syndrome 대체) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 이미 Partial/Full CRC 조기종료 보유 — syndrome 재계산 생략 아이디어는 기존 기법과 중복 |
| syndrome 계산/메모리 절감(HW 전력) | `ecc_top.cpp` (통계), 디코더 HW 모델 | 저전력 목적의 메모리 접근 최소화 컨셉은 참고 가능하나 Prime ECC는 soft-info 기반이라 직접 대응 모듈 없음 |
| SLC/dynamic SLC 채널 인지 적응형 디코딩 전환 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 오류율 기반 디코더 모드 전환 개념은 채널 적응형 LLR threshold 구조와 유사 |

> 적용 가치: 중간 — bit-flipping 알고리즘 자체는 Prime ECC(min-sum soft-decision 기반)와 이질적이나, "저오류 조건에서 syndrome 재계산·중간 상태 저장을 생략해 전력을 줄인다"는 설계 철학과 CRC 기반 조기종료 결합 아이디어는 Prime ECC의 조기종료/저전력 모드 설계에 참고 가치가 있음.
