### B. 알고리즘 요약

1. CCSDS 표준 QC-LDPC 코드(rate 1/2, 2/3, 4/5; 실제 rate 0.4/4≈0.571/8≈0.727)를 BPSK 변조 후 AWGN 채널에서 MATLAB으로 시뮬레이션.
2. 목적은 단문장(320~1280bit, CCSDS 표준 통상 3000bit 이상 대비 짧음) QC-LDPC에서 BP와 NMSA의 정량적 성능차가 충분히 연구되지 않았다는 공백을 채우는 것.
3. 채널모델은 표준 AWGN, LLR 초기값 `λ_i = 2y_i/σ²`.
4. BP는 표준 check-node 갱신식 `μ_{j,i} = 2·tanh⁻¹(Π tanh(γ_{i',j}/2))` (식2) 사용.
5. NMSA는 min-sum 근사에 정규화 계수 β를 곱한 `μ_{j,i} = β·(Π sgn(γ))·min|γ|` (식3, β=0.75 고정)로 BP의 overestimation bias를 보정.
6. 두 알고리즘 공통 variable-node 갱신 `γ_{i,j} = λ_i + Σμ_{j',i}` (식4), 최종 판정 LLR `λ_i^{post}` (식5).
7. 부수 트릭: BP에 동적 조기종료 임계값(SNR별 τ≈1.05~1.5) 적용, 시뮬레이션 효율화를 위한 적응형 Monte Carlo(고SNR에서 반복횟수 지수적 증가) 및 신뢰구간 기반 조기 종료.
8. 별도 학습/최적화 절차 없음(고정 β=0.75, 최대 반복 15회).
9. 결과: code length 2배 증가마다 이득 체감(320→640: 0.7dB, 640→1280: 0.5dB); rate 1/2가 4/5 대비 BER=1e-5에서 1.7~2.5dB 우위; 저SNR(<1dB)에서 BP가 NMSA 대비 0.2~0.3dB 우세, 고SNR(>3dB)에서는 BP의 수치 불안정성(tanh/atanh overflow)으로 NMSA가 오히려 근소 우세.
10. 한계: 신규 디코딩 기법 제안 없이 기존 BP/NMSA 비교에 그침, 연산량(덧셈/곱셈/LUT)·전력·HW 구현 분석 전무(저자 스스로 명시한 한계), 시뮬뿐이며 실측/FPGA 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMSA(정규화 min-sum, β=0.75) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC의 min-sum과 동일 계열이나 정규화 계수 β는 이미 magnitude 양자화 테이블 기반으로 처리, 신규 기여 없음 |
| BP(sum-product, tanh 기반) | 대응 없음 | Prime ECC는 min-sum만 지원, full-tanh Sum-Product 비대상 |
| 동적 조기종료(LLR 비율 임계값 τ) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 목적은 유사하나 판정 기준(LLR 비율 vs CRC)이 상이해 직접 대응 아님 |
| QC-LDPC H-matrix(CCSDS 구조) | `ecc_top.cpp` `Load_PCM()` | 부호구조 참고용이나 CCSDS 규격 자체는 Prime ECC와 무관한 별도 표준 |
| BER/rate/length 비교 실험설계 | 대응 없음 | 순수 성능비교 방법론, 코드 모듈 대응 없음 |

적용 가치: **낮음** — 신규 알고리즘/HW 기여 없이 기존 BP·NMSA를 표준 CCSDS 코드에서 비교한 벤치마크 논문이며, Prime ECC는 이미 min-sum(NMSA 계열)을 보유하고 있어 이식할 신규 요소가 없음.
