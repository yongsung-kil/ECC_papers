### B. 알고리즘 요약

1. 대상은 binary LDPC 코드 3종: irregular `(576,384)` rate `2/3`(C1), irregular `(576,432)` rate `3/4`(C2, IEEE 802.16e), regular `(2640,1320)` rate `1/2` Margulis 코드(C3), AWGN+BPSK 채널.
2. 문제: ADMM-LP 디코더는 고iteration에서는 BP보다 우수하나 저iteration에서는 BP보다 성능이 크게 떨어져, throughput 제약이 있는 저iteration 영역에서의 수렴 가속이 필요.
3. 핵심 가정/모델: ML 디코딩을 LP relaxation(`x∈C` → `Pjx∈PPdj`, parity polytope)으로 근사하고, check node별 auxiliary variable `zj`를 도입해 ADMM으로 푸는 프레임워크(식 2~5).
4. 핵심 기법 1단: ADMM 반복 업데이트 `x^(k+1)`(식 6), `z^(k+1)`(식 7, parity polytope Euclidean projection), `λ^(k+1)`(식 8, dual variable update).
5. 핵심 식: 기존 CSA(Cut Search Algorithm)는 `u`가 polytope 밖이면 cutting set `V`를 찾아 facet `θ_V^T z=|V|-1` 위로 projection(Algorithm 1의 차분계수 `η*`); 이 projection이 정확히 facet 위에 위치해 수렴이 느림.
6. 핵심 기법 2단(제안, ICSA): projection 거리를 `ξ=β·η*`로 스케일링(Algorithm 2 line 18)해 projection 결과가 facet이 아닌 polytope 내부 쪽으로 더 들어가게 조정 — `β>1`이면 polytope 안쪽/facet에 근접해 수렴 가속.
7. 부수 트릭: over-relaxation 기법 적용(파라미터 1.9), 조기종료 기준은 `Hx^T=0`, unified flooding schedule 사용.
8. 학습/최적화 절차: 코드별로 penalty parameter `µ`, `α`(l1 penalty `g(x)=-α|x-0.5|`)와 제안 파라미터 `β`를 exhaustive search로 최적화(C1: β=1.07, C2: β=1.08, C3: β=1.05).
9. 결과: `Itersmax=20`에서 CSA 대비 FER 1.1dB 개선(C1, FER=10^-5), 평균 iteration 13%(C2 SNR=4.0dB 기준, LSA 대비 19%) 감소, 평균 디코딩 시간도 CSA 대비 8% 감소.
10. 한계: 시뮬레이션(FER/평균iteration/평균시간)뿐이며 HW 설계·gatecount·throughput 수치 없음. ADMM-LP는 실수(soft) 기반 convex optimization 프레임워크로 min-sum 기반 HW 디코더 구조와는 근본적으로 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP 반복 업데이트(x/z/λ, 식 6~8) | 대응 없음 | Prime ECC는 min-sum 기반 message-passing 디코더이며, LP-relaxation/ADMM convex optimization 구조 자체가 없음 |
| CSA/ICSA parity polytope projection | 대응 없음 | check node 처리를 polytope projection으로 수행하는 방식은 Prime ECC의 `CNU_Update_New_Mag()` min-sum 연산과 전혀 다른 수학적 프레임워크 |
| 조기종료(`Hx^T=0`) | `partial_CRC.cpp` `partial_crc_HW_T4()` (개념적 유사) | syndrome 기반 조기종료 목적은 공통이나 구현 메커니즘 상이 |
| iteration 감소를 통한 저iteration 성능 개선 | `decoder.cpp` `LDPC_Decoder()` (개념적 유사) | 저iteration 영역 성능 개선이라는 목표는 Prime ECC의 관심사와 일치하나 알고리즘 자체는 이식 불가 |

적용 가치: **낮음** — ADMM-LP는 min-sum 반복 메시지패싱과 다른 convex-optimization 기반 디코딩 패러다임으로, Prime ECC의 HW 파이프라인(CNU/VNU, min-sum 연산)에 알고리즘 수준에서 이식할 경로가 없으며 디코더 전체를 재설계해야 함.
