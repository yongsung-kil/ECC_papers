> 총평: SC-LDPC shuffled SWD의 CN update 연산을 MDQ 자료구조로 경량화해 ASIC 면적 63.7%↓·주파수 107.6%↑를 달성한 디코더 HW 최적화 논문으로, min-sum CN 연산 최적화 관점에서 Prime ECC 디코더에 참고 가치가 있으나 부호 구조(SC-LDPC vs QC-LDPC) 차이로 이식 난이도는 중간 수준.

### B. 알고리즘 요약

1. SC-LDPC 부호(protograph coupled, coupling length `L`, coupling width `ms`, lifting factor `Z`)를 대상으로 sliding window decoding(SWD, window size `W`)을 수행, 시뮬레이션 코드는 rate 0.79(길이32128)·0.82(길이18000).
2. 기존 SNMS(shuffled normalized min-sum)-SWD는 CN update에서 `|R_ij|=min_{n∈N(i)\j}|L_in|` 계산 시 매 반복마다 전체 V2C 메시지를 비교해 복잡도가 CN degree `dc`에 대해 O(dc²)로 증가하는 것이 문제.
3. 핵심 가정: target symbol(window 이동 시 제거되는 VN)의 V2C 메시지 중 최솟값만 CN update에 필요하다는 관찰(불필요한 반복 비교 제거 가능).
4. 핵심 기법: size-λ monotone double-ended queue(`λMDQ`)로 후보 magnitude를 오름차순 정렬 보관, `Q_i` 크기 λ(예: λ=2,3)로 고정. λ=dc이면 SNMS-SWD와 동일 성능.
5. 핵심 식: `|R_ij| = min(Q_i,1^(1), Q_i^min)` (식 9) — 큐의 최솟값과 target symbol 최소 magnitude(`Q_i^min`) 중 더 작은 값을 사용, 매 CN당 O(dc) 복잡도로 감소.
6. Q-UPD 함수(Algorithm 1)로 새 V2C 메시지 도착 시 큐 tail부터 조건부 삭제 후 삽입(element deletion/ED, element insertion/EI)해 정렬 유지.
7. HW 구현 디테일: magnitude 4bit, column index 5bit 양자화, λ=2 구조는 ED에 5bit CMP 1개+MUX 4개, EI에 4bit CMP 2개+MUX 6개로 구성(1 clock cycle 처리).
8. 별도 학습/최적화 절차 없음 — λ 값(2,3)을 파라미터로 시뮬레이션 비교.
9. 결과: λ=2 λMDQ-SWD가 SNMS-SWD 대비 거의 동일한 BER/FER 성능 유지하며 ASIC(TSMC 90nm)에서 CN update 면적 63.7%↓(3729→1353 µm²), 최대주파수 107.6%↑(1235→2564MHz), FoM 0.069→0.476.
10. 한계: SC-LDPC 전용(코드 구조가 QC-LDPC block code와 다름), avg iteration 수 자체는 감소하지 않고 CN update 연산량/HW만 경량화, 전체 디코더 throughput(Gbps) 수치 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN update min-sum 연산 (`|R_ij|` 계산) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | shuffled 스케줄 하의 CN 최소값 탐색 최적화 아이디어로, Prime ECC의 min1/min2 기반 CN 연산과 목적이 유사하나 λMDQ는 shuffled(column-serial) 갱신 전용 구조라 flooding형 Dual-Update 스케줄에 직접 대입은 어려움 |
| shuffled schedule (column-by-column 갱신) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 Dual-Update(even/odd 교대) 스케줄을 사용하며 shuffled schedule 자체는 별도 스케줄링 방식이라 이식 시 이터레이션 루프 구조 변경 필요 |
| sliding window decoding (SC-LDPC 전용) | 대응 없음 | Prime ECC는 QC-LDPC block code(SC-LDPC 아님)이므로 SWD 개념 자체가 대응 없음 |
| CN update 4bit magnitude/5bit index 양자화 HW | `decoder.h` `Get_VNU_*`, `ecc_data.h` `PARAM_LLR` | 저비트 양자화 큐 구조는 Prime ECC의 LLR 양자화 테이블 설계에 참고 가능 |

> 적용 가치: 중간 — CN update 연산의 min-sum 값 탐색을 MDQ 큐로 경량화하는 아이디어는 Prime ECC의 min1/min2 CNU 연산에 참고할 만하나, SC-LDPC/shuffled schedule 전용 구조라 QC-LDPC/Dual-Update 기반 Prime ECC에 직접 이식하려면 스케줄링 로직 재설계가 필요.
