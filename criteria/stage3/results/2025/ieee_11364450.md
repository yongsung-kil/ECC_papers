### B. 알고리즘 요약

1. Raptor 코드(LT code + QC-LDPC pre-code)를 이용해 Construction-D′ 기반 3-level 격자(lattice)를 구성, unconstrained AWGN 채널(Poltyrev 모델) 대상.
2. 기존 LT/Raptor 코드는 부호길이 증가 시 평균 출력차수가 로그로 증가해 선형시간 인코딩/디코딩이 어렵다는 문제를 QC-LDPC pre-code 결합으로 해결.
3. 채널은 unconstrained AWGN(전력 제약 없는 무한 격자), 노이즈는 분산 `σ²`의 i.i.d. 가우시안으로 가정.
4. Raptor 생성행렬 `G_Raptor = G_out · G_in` (G_out: QC-LDPC, G_in: LT) 구조, Construction-D′ 체크행렬 `H_Λ = D·H`로 격자점 `Λ = {x | H_Λ·x = b}` 정의.
5. QC-LDPC pre-code는 girth `g=8`, 열가중치 `wc=2~3`, 행가중치 `wr=12`, circulant 크기 `ϑ`로 설계해 코드길이 `N=ϑ·wr` 결정; girth-6 QC-LDPC와 LT 결합 시 발생하는 girth-4 Tanner graph를 추가 수정으로 제거해 짧은 사이클 문제 해결.
6. 디코딩은 modified Tanner graph 기반 반복(iterative) soft decoding(LLR)과, 레벨별 successive cancellation(재부호화 포함) 결합 구조.
7. LLR 계산식 `LLR(j) = (1-2y'_{i,j})/2σ²` 사용, 각 레벨 디코딩 결과로 다음 레벨 입력을 재구성(`y_{i+1} = (y_i - x̂_i)/2`).
8. 파라미터(D=12, α=0.27 등)는 heuristic 탐색으로 결정, LP 최적화 없이 [7]의 방법 사용.
9. 결과: 길이 995 Raptor lattice가 FER 기준으로 LDLC/LDPC lattice 대비 0.07dB/0.32dB 개선(Poltyrev limit 대비), 단 SER 기준으로는 LDPC/LDLC가 0.03~0.1dB 우세.
10. 시뮬레이션(SER/FER)만 수행, HW 구현·복잡도 수치 없음. binary NAND channel이 아닌 unconstrained AWGN lattice 전용 설계.
