### B. 알고리즘 요약

1. 위성 지상국 모뎀용으로 LDPC/RS/BCH 등 다양한 가변 블록코드를 병렬 처리하는 범용 초고속 인코딩/디코딩 구조를 FPGA로 구현.
2. 문제: 코드별로 별도 코덱을 하나씩 연구·구현하면 복잡·고가이므로, 여러 블록코드를 공통 구조로 병렬화해 throughput을 올리려 한다.
3. 6개 모듈 구성: indicator 생성, decorrelation, 병렬 캐시 루프, 병렬 가속(코덱 멀티플렉싱), 프레임 후처리(zero padding), 직렬 출력.
4. 핵심 아이디어 - decorrelation: 프레임 길이 가변·헤더 위치 임의라 인접 프레임이 한 Q바이트 클록에 섞여 들어오는데(상관), 버퍼 지시자 `Switch=i%M+1`과 헤더주소 `hr`로 이를 M채널 독립 프레임으로 분리.
5. enable/주소 분리는 순환행렬 `VSM`/`DSM`(cyclic matrix) 형태로 수식화(식 1~9).
6. 병렬 가속: M채널 encoder/decoder 멀티플렉싱으로 각 채널이 독립 코덱을 수행 — "encoder 내부 알고리즘 구조는 바꾸지 않고" 속도만 M배 향상.
7. 후처리: 코드 프레임 길이 `L_code=M*Q+N`에서 N!=0이면 끝에 N바이트 zero padding해 헤더가 항상 Q바이트 첫 비트에 오도록 정렬, 이후 parallel-to-serial 변환(`Cr`)으로 직렬 출력.
8. 소프트웨어가 Data Rate/RS/LDPC rate로부터 병렬 모듈 수 M을 계산해 해당 FPGA 비트스트림을 자동 로드·재구성(리소스 최적 스케줄).
9. 결과(xc7vx690t, 200MHz): M=2에서 throughput 4배, M=3에서 6배(0.5→3Gbps), zero BER로 RS(255,223/239), LDPC 1/2~7/8 등 9종 호환; M=3에서 LUT 66.95%/BRAM 55.75%.
10. 한계: 복호 알고리즘·부호설계 자체는 무손질(black-box)이며 자원 제약으로 실측 3Gbps에 머묾, 위성 모뎀 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| M채널 코덱 멀티플렉싱(병렬 가속) | 대응 없음 | 코덱 내부를 건드리지 않는 상위 데이터 분배 계층, 우리 디코더 알고리즘과 무관 |
| decorrelation / 프레임 캐시 스케줄 | 대응 없음 | 프레임 정렬·버퍼링 프레이밍 로직, ECC 코드베이스 범위 밖 |
| zero padding / 프레임 정렬 | 대응 없음 | 데이터 정렬 후처리, 알고리즘 무관 |
| LDPC 코덱 자체 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` (black-box로만 사용) | 논문이 내부를 다루지 않아 이식할 알고리즘 기여 없음 |
| 소프트웨어 M 계산 + 비트스트림 재구성 | 대응 없음 | FPGA reconfiguration/스케줄러, 시뮬레이터 범위 밖 |

적용 가치: **낮음** — 코덱을 black-box로 취급하는 위성 모뎀용 프레임 병렬화/스케줄 프레임워크로, binary QC-LDPC 복호 알고리즘이나 부호설계에 대한 기여가 전혀 없다.
