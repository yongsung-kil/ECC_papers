### B. 알고리즘 요약

1. NAND flash용 QC-LDPC 디코더로, 최대 codeword length 4kB(32768×1536 parity-check matrix, 원소당 16비트)를 지원하며 codeword length/code rate/row weight/column weight를 모두 재구성 가능하게 설계.
2. 문제의식: NAND flash는 벤더별 page size/error rate가 달라 통일된 ECC 스펙이 없어 디코더 재설계가 빈번히 필요함(기존 [1][2][3][4]는 재구성성 또는 codeword length 지원이 제한적).
3. base-matrix(Hb)의 M(행)×N(열)×Z(서브행렬 크기) 파라미터를 offset memory 모듈이 저장하고, 이로부터 확장 parity-check matrix의 row/column weight를 계산해 VNU/CNU에 전달.
4. VNU는 base-matrix 열마다 1개 유닛 할당(열 개수 재구성), 각 유닛은 column weight(1~6)에 따라 1~6 데이터 포인트 처리; CNU는 행마다 1개 유닛 할당, row weight(1~128)에 따라 1~128 데이터 포인트 처리. 입력 포트 앞단 MUX로 미사용 포트를 0으로 세팅해 weight 재구성 구현.
5. H-RAM은 parity-check matrix의 비영(non-zero) 원소만 저장(base-matrix 위치 매핑)해 직접구현 대비 저장량을 128×1536×16bit로 축소(원본 대비 1/256) — QC-LDPC folding 특성 활용.
6. 비영 원소만 저장 시 주소 접근이 non-trivial해지는 문제는 base-matrix shift값 + CNU별 bias network 조합으로 read/write 주소를 정확히 계산해 해결(permutation network 모듈).
7. 부수 트릭: Z(서브행렬 크기) 조절로 codeword length 대비 활성 VNU 유닛 수·throughput·전력 간 trade-off 확보(Z 증가 시 N 감소 → throughput 낮아지고 전력도 감소).
8. 학습/최적화 절차 없음(HW 설계 자체가 기여).
9. 결과: 22nm CMOS, core area 1.79mm², 최대 클럭 317MHz, 최대 throughput 20.3Gb/s(정규화, iteration=1 기준); 선행 [2](45Gb/s@90nm), [3](33.2Gb/s@28nm) 대비 throughput은 낮으나 재구성 파라미터 범위(codeword length/rate/row·col weight 전부)는 가장 넓음.
10. 한계: min-sum 알고리즘 자체의 정정성능 개선(waterfall/error-floor)은 다루지 않고 순수 재구성형 HW 아키텍처 기여이며, permutation/bias network의 세부 회로는 도식(Fig.3) 손실로 본문만으로 완전 파악 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VNU/CNU를 MUX로 row/column weight 가변 처리(재구성형 min-sum) | `decoder.cpp` `CNU_Update_New_Mag()`, `decoder.h` `Get_VNU_*` | Prime ECC는 z=32 고정 병렬 구조; MUX 기반 가변 weight 처리 아이디어는 GT(Graph Thinning)의 MUX edge 선택과 유사한 개념으로 참고 가능 |
| H-RAM 비영 원소만 저장 + base-matrix shift 기반 주소 매핑 | `ecc_data.h` `PCM_b`(base+shift 구조) | Prime ECC도 동일하게 base+circulant shift로 PCM 저장하므로 개념적으로 일치, 다만 Prime ECC는 codeword length 재구성이 없음 |
| Z(서브행렬 크기) 가변에 따른 throughput/전력 trade-off | 디코더 HW 모델(`z_sb=32` 고정) | Prime ECC의 고정 lifting factor(z_sb=32) 대비 가변 Z 지원은 아키텍처 확장 아이디어로 관련성 있으나 재검증 부담 큼 |
| 조기종료/CRC, iteration 감소 기법 | 대응 없음 | 본문에 조기종료 관련 기술 언급 없음 |

> 적용 가치: 중간 — NAND 직접 타깃 min-sum QC-LDPC HW이며 base+shift PCM 구조는 Prime ECC와 개념적으로 일치하나, 핵심 기여인 codeword length/weight 재구성성은 Prime ECC의 고정 z=32/N_b=129 구조와 이질적이라 구조 전체 이식은 난이도 높음(부분 아이디어 참고는 가능).
