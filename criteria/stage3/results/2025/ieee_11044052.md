### B. 알고리즘 요약

1. 양자 네트워크(QKD) 환경에서 QBER 0.01~0.11 범위의 변동 채널에 대응하는 LDPC 기반 오류정정 가속기.
2. 기존 FPGA LDPC 구현은 단일 code rate 또는 비적응적 다중 rate만 지원해 QBER 변화에 유연하게 대응하지 못함.
3. 채널 모델은 큐비트 오류율(QBER)이 0.01~0.11로 변동한다고 가정, 입력데이터 `x`와 QBER로 외부메시지 `Pi(b) = abs(xi - qber)`를 계산.
4. 하드웨어는 5개 처리 유닛(매트릭스 선택, VN, CN, 주소 제어, 결정)으로 구성되며 SPA와 NMSA를 동일 아키텍처로 지원.
5. SPA는 확률영역 곱셈 기반(`Q(ijl)(b)`), NMSA는 로그영역 min-sum(`Rj(li) = α · min|q| · sgn(q)`)으로 변환해 곱셈을 덧셈/비교로 대체.
6. 내부메시지를 CN→VN은 row-form, VN→CN은 column-form BRAM으로 저장해 파이프라인 중 누적곱/합 연산을 수행.
7. 부수 트릭: 희소 행렬의 0-원소 위치는 저장하지 않고 1-원소 위치만 전처리 저장해 자원 절감.
8. 학습/최적화 절차 없음(고정 매트릭스 8종을 사전 저장, [13] 문헌 기반 QBER-rate 매핑 사용).
9. 결과: CPI<20 cycle/iteration 달성, throughput SPA 180Mbps·NMSA 728Mbps, 기존 대비 최대 9.4배 속도 향상.
10. 한계: code length 100의 소규모 예시만 검증, error-floor/waterfall 등 정정성능 개선 논의 없음, 순수 QKD 채널 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA/NMSA 비트단위 파이프라인 아키텍처 | `decoder.cpp` `LDPC_Decoder()` | NMSA(=정규화 min-sum)는 이미 Prime ECC의 Min-Sum과 개념 중복, 신규성 낮음 |
| QBER 적응형 매트릭스 선택 | `ecc_top.cpp` `Load_PCM()` | 여러 code rate 중 런타임 선택이라는 아이디어만 유사, NAND RBER 적응과는 목적 상이 |
| 희소 행렬 1-원소 위치 저장 최적화 | `ecc_data.h` (PCM 구조체) | 희소 행렬 압축 저장은 이미 QC-LDPC base+shift 구조로 대체됨 |

> 적용 가치: 낮음 — QKD 채널·짧은 부호 특화이며 핵심 기법(NMSA/min-sum, 매트릭스 스위칭)이 Prime ECC에 이미 존재하거나 개념적으로 다른 목적(양자채널)에 국한됨.
