### B. 알고리즘 요약

1. 대상: 채널 메모리(연속 오류=버스트)를 정정하는 선형부호 문제를, LDPC의 block-circulant 구성(BCC) 기반으로 다룸.
2. 풀려는 문제: BCC는 최대 정정 버스트 길이가 블록크기로 제한(`bmax <= m-1`)돼 Reiger bound `bmax <= floor(r/2)`에서 멀고, 블록 수를 늘리면 rate·redundancy가 급격히 나빠짐.
3. 모델/정의: 길이 `b` 단일 버스트 = 비영 원소가 연속 `b` 위치 내에 모인 오류벡터, 정정능력은 모든 버스트가 서로 다른 coset(신드롬)에 있도록 하는 최대 `bmax`.
4. 핵심기법: 마스킹 - `H = HM ⊗ HB` (Kronecker), `HM`은 zero-block 위치를 지정하는 `(γ×ρ)` 이진 마스킹행렬로, 블록열별 서로 다른 zero 조합을 넣어 all-ones 신드롬 중복을 깨뜨림.
5. 핵심식 의미: 마스킹 없으면 블록열 컬럼합이 all-ones로 겹쳐 길이 `m` 버스트 2개가 같은 coset에 빠짐 - 각 `HM` 컬럼을 유일하게 만들면 이 겹침이 사라져 `bmax > m-1` 달성.
6. 확장(핵심정리): 일반화하면 `2κ <= γ`일 때 `bmax >= κm-1`, 조건은 `HM` 컬럼 부분집합들의 rank 조건(식10) - 이는 곧 `HM` 자체의 버스트 정정능력 `b(M)max = κ-1`을 최대화하는 문제로 환원됨.
7. 구현/휴리스틱: (1) Reiger 근처(`b(M)max = floor(γ/2)`)인 마스킹행렬 `HM`을 컴퓨터 탐색, (2) 그에 대해 `bmax` 큰 base matrix `HB`(랜덤 순환치환 시프트)를 탐색. `m`은 보통 소수 선택.
8. 부수사항: Tanner graph 짧은 사이클 회피나 일반 치환블록 사용은 버스트 정정능력을 오히려 나쁘게 하거나 좋은 부호 확률을 낮춰 도움 안 됨.
9. 결과: `m=11,γ=3,ρ=4`에서 마스킹 후 100개 중 75개가 `bmax>10`, 최대 15(Reiger 16). Table II: `(176,44)` BCCM `bmax=63`(Reiger 66), `(165,55)` `bmax=52` 등으로 dense random 부호와 대등하면서 탐색시간은 더 짧고 QC 구조 유지.
10. 한계: 순수 조합적 구성법+시뮬, HW 미설계, 초소형 부호(`n<=396`), 반복복호(min-sum) 성능·랜덤오류 정정과 무관하며 버스트 채널 가정에 특화.
