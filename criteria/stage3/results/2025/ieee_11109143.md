### B. 알고리즘 요약

1. 100Gb/s 대역폭 제한 flexible-rate IM/DD PON 시스템(NRZ/PAM4, 50GBaud)에서 25G-EPON LDPC mother code (17664,14592) 기반 adaptive-rate(0.7333~0.8889) FEC를 사용, 수신단 LLR을 soft-decision LDPC 디코더(SPA, 10회 반복) 입력으로 공급.
2. 기존 LLR 계산은 채널 잡음 파라미터 추정 후 다중 덧셈/곱셈/지수 연산(5차 Taylor 전개, bit당 21회 덧셈+56회 곱셈)이 필요해 실시간 HW 구현에 병목이 되며, 선행연구의 Taylor 근사식([13])은 고정 구간분할로 수동 파라미터 조정이 필요해 실시간 적용이 제한적이었음.
3. 채널 모델은 등화(FFE 31-tap + DFE 5-tap) 후 신호가 near-Gaussian 분포를 따른다고 가정(히스토그램/PDF로 확인).
4. 핵심 기법(PPF, piecewise polynomial fitting): 등화 후 신호값 `r`에 대해 극값 검출(extremum detection)로 LLR 곡선을 2~4개 구간으로 자동 분할, 각 구간을 `L'(r) = a_i*r^2 + b_i*r + c_i` (r ∈ [t_i, t_i+1]) 형태의 1~2차 다항식으로 근사.
5. 2차 다항식 근사가 정확도-복잡도 최적 균형점으로 실험적으로 확인됨 - 저차 다항식임에도 LDPC 정정능력 범위 내 오차로 디코딩 성능에 영향 없음.
6. 확장 없음(단일 PPF 기법, 세그먼트 수만 신호 형태에 따라 2~4개로 적응).
7. 구현 디테일: 구간 경계를 수동 튜닝 없이 신호 자체의 극값에서 자동 결정하는 것이 핵심 트릭 - 기존 Taylor 근사 대비 정적 채널 가정을 제거.
8. 별도 학습 절차 없음(다항식 계수는 각 구간에서 curve fitting으로 산출, 실시간 파라미터 조정 불요).
9. 결과: ROP=11.6dBm, code rate 0.8667에서 conventional BER 3.53e-2 vs PPF BER 3.55e-2로 차이 무시할 수준, link budget 저하 0.1dB 이내, LLR 연산량 덧셈 90.47%·곱셈 94.64% 감소(총 90% 이상 절감).
10. 한계: HW 구현 검증 없이 오프라인 MATLAB 처리 결과이며, 채널이 near-Gaussian임을 전제로 하고 VOA 분해능(0.1dB) 한계로 정밀 성능차 미확인, 저자도 "HW 배치 시 효과는 향후 연구 필요"라 명시.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|
| PPF 기반 저복잡도 LLR 근사 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read LLR 양자화/threshold 설계 개념과 직결 - NAND read 채널의 조각별 근사 LLR 테이블 설계에 참고 가능 |

| iteration별 LLR 테이블 | `ecc_data.h` `PARAM_LLR` | 다항식 근사 계수를 LLR 테이블 형태로 저장하는 방식과 유사한 구조 |

> 적용 가치: 중간 - LDPC 코드/디코더 자체의 개선은 없으나, 극값 기반 자동 구간분할+저차 다항식 LLR 근사라는 복잡도 절감 아이디어는 NAND soft-read 채널의 LLR 테이블 산출·양자화 설계에 개념적으로 참고할 가치가 있음(단, 광채널의 near-Gaussian 가정이 NAND RBER 분포와 다르므로 재검증 필요).
