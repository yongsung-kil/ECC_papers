> 총평: row-layered BP 디코더의 동적 스케줄링(EO→EC→ECC→CO 상태전이)으로 평균 iteration을 최대 76% 감소시키고 throughput을 최대 2.6배 향상시킨 실용적 기법으로, min-sum 이터레이션 루프에 상태전이 로직만 추가하면 되어 Prime ECC 디코더에 이식 난이도가 낮고 관심도가 높은 영역(iteration/latency 감소)에 정확히 부합.

### B. 알고리즘 요약

1. 시스템: 5G NR LDPC 코드(3GPP quasi-cyclic irregular, Base Graph 1/2, lifting factor `Z`)를 대상으로 row-layered BP(SPA/min-sum) 디코더의 체크노드(CN) 갱신 순서(스케줄) 최적화를 다룸. `H_BG`의 앞 4행은 고차수 "core rows", 나머지는 저차수 "extension rows".
2. 문제의식: 표준 top-to-bottom 순서는 임의의 스케줄(`M!`가지) 중 하나일 뿐이며, 코드워드/노이즈 실현별로 최적 스케줄이 달라 고정 스케줄은 평균 iteration·오류율 면에서 최적이 아님.
3. 핵심 가정/모델: 파티체크(parity-check) 오류 개수·LLR 크기 같은 간단한 내부 메트릭을 조기종료 트리거처럼 상태전이 조건으로 사용해 코드워드별 동적 스케줄을 실시간 결정.
4. 핵심 기법 1단(동적 스케줄링): 4개 상태 `EO(extension-only, bottom-to-top)→EC(extension+core)→ECC(core 2회 반복, waterfall 개선)→CO(core-only)`를 순환하며, 임계값 `T1(LLR magnitude=6.0)`, `T2=T4=T5(parity error 15%)`, `T3(35%)`로 전이.
5. 핵심 식/의미: `Ec==0`(모든 core check 만족)을 공격적 조기종료 조건으로, "SlowConv"(느린 수렴) 판정 시 별도 임계값으로 조기 포기해 처리량을 확보 — core row는 고rate 부분코드로 저오류 구간에서 효율적이고, extension row는 저internal-SNR 구간에서 mutual information 기여가 큼.
6. 핵심 기법 2단(앙상블 디코딩/universal decoding): 동일 BP 디코더를 CN 갱신 순서만 다르게(top-to-bottom, bottom-to-top, 랜덤 순열 `π`) `L`개 병렬 실행 후, 유효 코드워드(`x̂·H^T=0`) 중 수신 신호와의 유클리드 거리 최소(`ML-in-the-list`)를 최종 출력으로 선택.
7. 부수 트릭/구현: 스케줄 변경은 H-matrix 행을 순열 `π`로 재배열(`H'=H(π,:)`)해 기존 top-to-bottom 디코더 구조를 그대로 재사용, HW 구현 친화적. 앙상블은 polar/RM 코드에도 sparse graph 표현만 있으면 동일 디코더 하드웨어로 적용 가능("one-silicon fits all").
8. 실험/튜닝: 임계값은 실험적으로 설정(GPU 타이밍 테스트에서 core row가 R=1/3 기준 실행시간의 20% 차지함을 활용해 ANI 계산).
9. 결과(수치): R=1/2(Z=384,N=8448)에서 평균iteration(ANI) 최대 66% 감소, 2dB에서 throughput 1.8배 향상; R=1/3에서 ANI 최대 76% 감소, 2dB에서 throughput 2.6배 향상; 앙상블(L=8)로 AWGN 0.7dB, Rayleigh fading 1.6dB coding gain(BLER 10^-3~10^-4).
10. 한계: 모든 실험이 floating-point SPA 시뮬레이션(GPU 타이밍은 참고용)이며 ASIC/FPGA 합성·gatecount·정확한 throughput(Gbps) 수치는 제시되지 않음. 앙상블 디코딩은 latency는 유지되나 L배 병렬 디코더 인스턴스가 필요해 면적 비용이 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EO/EC/ECC/CO 동적 스케줄링(상태전이 기반 row 순서 변경) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 코어/확장 row 구분과 상태전이 로직을 이터레이션 루프에 추가하는 형태로 이식 가능(이식 난이도 낮음, 프로파일 §4). |
| 파티체크 오류율/LLR 크기 기반 상태전이 트리거 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `ecc_data.h` `PARAM_LLR` | 기존 Partial CRC 조기종료·적응형 LLR 테이블과 유사한 메커니즘으로 임계값 로직만 추가하면 결합 가능. |
| CN 갱신 순서 순열(H-matrix row 재배열) | `decoder.cpp` `C2V_Cal()`, `ecc_top.cpp` `Load_PCM()` | Dual-Update(홀짝 교대 스케줄)와 유사한 스케줄링 확장 지점. |
| 앙상블 디코딩(L개 병렬, 서로 다른 스케줄) | 대응 없음 | Prime ECC는 단일 디코더 인스턴스 구조로 다중 병렬 디코더 앙상블 아키텍처 미구현. |

> 적용 가치: 높음 — 기존 min-sum 이터레이션 루프에 core/extension row 구분과 파티체크 기반 상태전이만 추가하면 되는 저난이도 개선으로, Prime ECC가 이미 보유한 Partial CRC 조기종료·적응형 LLR과 결합 시 iteration/latency 감소 효과를 직접 노릴 수 있음(추천도↑ 영역: avg iteration·latency 감소).
