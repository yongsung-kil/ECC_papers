### B. 알고리즘 요약

1. 5G NR RL-QC-LDPC(BG1: 46x68, BG2: 42x52, lifting Z)를 x86 서버(AVX-512, Intel Xeon Gold 5218)에서 소프트웨어로 구현, 코드rate 0.33~0.89 범위 7개 code scheme 평가.
2. 기존 SD-LDPC(software-defined LDPC) 구현은 특정 코드 스펙에 한정되거나 message-passing 자체의 내재적 처리량 한계를 못 넘음.
3. 채널은 AWGN + BPSK, 입력 LLR 6비트/내부 메시지 8비트 양자화로 부동소수점 대비 성능 손실 최소화.
4. 인코더: 기존 efficient encoding[17]을 4가지 base-graph/Z 조합으로 일반화하고, AVX-512 byte-level shift/LUT 기반 rotate, twin-aligned 데이터 배치로 8.06배 가속.
5. 디코더: 층별(layered) min-sum(LMS) 기반, `βq,p = sgn × min|α|` CNP를 O(dc²)에서 O(dc) 선형시간(Algorithm 2)으로 변경, 언롤링(outer=CN degree dc, inner=SIMD 벡터 수)으로 분기 제거.
6. IPMS(iteration-based pseudo second-min min-sum): 초반 iteration은 second-min을 근사(명령어 33.4% 감소)하고 후반 iteration에서 정확 계산으로 전환해 error floor를 1E-4에서 1E-8 이하로 복구.
7. shuffle network를 partial-shuffling(누적 shift 차이만 적용)으로 최적화해 기존 대비 29% 레이턴시.
8. SIMD early stopping: exponent matrix-vector XOR 곱으로 syndrome check을 레이어 단위로 조기 종료, 특정 코드에서 69.1% 가속.
9. 결과: 디코더 391Mbps(단일코어)~5.1Gbps(16코어 선형 스케일), 상태최고기법[13] 대비 디코딩 50.8%, 인코딩 8.06배 개선. IPMS 적용 시 BER 0.3dB 손실 감내하고 59.3% 추가 가속.
10. 한계: x86 범용 프로세서(SIMD) 소프트웨어 전용이며 HW(RTL/ASIC) 설계는 없음; 5G NR 특화 코드 구조(BG1/BG2, Z)에 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IPMS(초반 근사→후반 정확 min-sum) | `decoder.cpp` `CNU_Update_New_Mag()` | iteration별 근사/정확 전환 아이디어는 적응형 LLR 테이블과 유사 맥락이나 x86 SIMD 명령어 최적화라 직접 이식 불가 |
| 선형시간 CNP(2-minima 탐색) | `decoder.cpp` `C2V_Cal_New_Sgn()` | 우리는 이미 HW search module로 2-minima를 구조적으로 구현 중이라 중복 |
| SIMD early stopping(XOR syndrome) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 목적은 동일하나 소프트웨어 XOR 벡터 연산 기법으로 HW CRC와 대응 안 됨 |
| shuffle/de-shuffle 최적화 | 대응 없음 | x86 벡터 레지스터 재배치 기법으로 HW 라우팅 구조와 무관 |
| 5G LDPC 인코더(AVX-512) | `encoder.cpp` `LDPC_encoder_4KB()` | 코드 자체가 5G NR 표준 PCM 기반이라 우리 dual-diagonal 인코더와 구조 상이, 대응 낮음 |

> 적용 가치: 낮음. x86 CPU 소프트웨어 가속(SIMD ISA 최적화) 논문으로 NAND HW ECC(Prime ECC 3.1)의 RTL/ASIC 설계와는 레이어가 완전히 다르며, 5G 표준 코드 특화라 부호설계 이식도 어려움.
