### B. 알고리즘 요약

1. 시스템: 심우주 통신 uplink, CCSDS 231.0-B-3 정의 `(128,64)` rate-1/2 systematic irregular QC-LDPC, AWGN·BPSK·S-band, 4-bit soft decision 입력.
2. 문제: soft-decision 복호(BP)는 정정력 좋으나 삼각함수 연산으로 HW 복잡, 최대 throughput이 필요한 uplink엔 fully-parallel min-sum이 적합하나 로직 사용량이 큼 — 효율적 CNU/VNU 필요.
3. 모델: base matrix를 cyclic-right shift + mod-2로 확장한 QC-LDPC H-matrix, `M=k/or n/48` circulant, offset-binary → 2's complement LLR 전처리.
4. 핵심기법: fully-parallel min-sum, 64 CNU + 128 VNU bank(각 16노드), CLCR 라우터가 H-matrix 따라 채널 LLR을 CN에 분배, VTCMEM이 CN↔VN 메시지 교환, FSM(decoder controller)이 syndrome=0까지 반복.
5. 핵심식: CNU가 min1만 정렬로 찾고 min2는 근사 `γ=⌊((2^{q-1}-1)-m1)·σ/2⌋`, `em2=m1+γ` (식 2,3)로 도출 — 두 최소값 정렬 회피로 로직 축소.
6. 확장: dynamic factor β(ref [12]) 대신 `σ=0.5`(code rate 비례)·`q=4` 고정으로 곱셈·상수연산 배제, HW 사용량 절감.
7. 구현: CNU는 2's comp→signed-magnitude 변환(TSM/SMT) 후 majority-vote sign + min1/min2, VNU는 연결 CNU 메시지 합 ± 자기 메시지 + 채널 LLR, HD 합만 syndrome 계산에 사용.
8. 학습/최적화: 없음(고정 파라미터 σ, q, 최대 반복).
9. 결과: Xilinx Kintex Ultrascale XCKU040 구현, LUT 9%·FF 5%·BRAM 3% 사용, S-band soft-decision 수신기 실측으로 BER 10⁻⁶에서 coding gain 5.2 dB.
10. 한계: 부호길이 128로 매우 짧음, gate count·throughput 절대수치 미제공(LUT % only), 근사 CNU는 기존 문헌([11],[12]) 기법 채택이라 신규성 제한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| fully-parallel min-sum 복호 전체 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | min-sum 복호로 우리와 동일 계열, 아키텍처 참고 가능 |
| CNU min1/min2 (2nd-min 근사) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 우리도 min1/min2+sign XOR 보유 — 근사식은 이미 보유 기법과 중복 |
| VNU 합/양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | VN 처리 대응 |
| offset-binary→2's comp LLR 전처리, 4-bit soft | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read 양자화 대응(우리는 2/3SD, 4-bit는 미지원) |
| CLCR/VTCMEM 라우팅 (edge/MUX) | `GT.cpp` `Make_GT_HW()` | 메시지 라우팅 대응(우리 graph thinning MUX와 유사) |
| syndrome check / 조기종료 | `partial_CRC.cpp`, `full_CRC.cpp` | 우리는 CRC 조기종료, 여긴 syndrome=0 종료 — 개념 대응 |
| G-matrix 행렬곱 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()` | 우리는 dual-diagonal, 여긴 G-matrix LUT 곱 — 다른 방식 |
| H-matrix/부호구조 로드 | `ecc_top.cpp` `Load_PCM()` | CCSDS 특정 부호, 우리 고정 QC-LDPC와 다름 |

마지막에 한 문장으로 **적용 가치**: `중간` — min-sum QC-LDPC fully-parallel HW 아키텍처와 4-bit soft LLR 전처리는 우리와 계열이 같아 참고 가치가 있으나, 부호가 저rate·짧은 CCSDS 전용이고 CNU 2nd-min 근사가 이미 보유 기법과 중복되어 직접 이식보다는 아키텍처 설계 참고용.
