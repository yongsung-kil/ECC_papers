### B. 알고리즘 요약

1. 시스템은 QC-LDPC(protograph lifting, lifting factor `z`) 표준 부호로 WiMAX(n=576,R=3/4), WiFi(n=648,R=5/6), 5G NR(n=352~2112, R=0.33~0.83)를 대상, 채널은 AWGN+BPSK, 양자화는 5-bit uniform(step 0.5, max 7.5)이 기본.
2. 풀려는 문제: LDPC의 error floor(낮은 FER 영역에서 완만해지는 구간) 때문에 6G xURLLC 요구(FER<10⁻⁹) 달성이 어렵고, waterfall 개선(WMS 등 weighting)은 오히려 error floor를 악화시키며 양자화도 error floor를 유발.
3. 핵심 가정/모델: NMS(neural min-sum) decoder — 채널 weight `w_v^(ℓ)`, CN weight `w_{c→v}^(ℓ)`를 학습 가능한 파라미터로 min-sum 메시지에 곱하는 구조(식1,2), 가중치 전부 1이면 표준 MS와 동일.
4. 핵심 기법 1단(boosting learning): 디코딩을 base(iteration 1~ℓ1, 가중치 `w_B`)와 post(ℓ1+1~ℓ, 가중치 `w_P`) 2단으로 나누고, base가 정정 실패한 UC(uncorrected) 벡터만 모아 post 단계를 그 벡터들로 재학습(사람이 만든 boosting 알고리즘 차용).
5. 핵심 식의 의미: error-floor 영역에서 표준 학습은 손실이 거의 0(대부분 정정 성공)이라 가중치 갱신이 안 되는데, UC 벡터만 선별 학습하면 error floor를 유발하는 소수 오류패턴(작은 trapping set)에 학습이 집중됨.
6. 핵심 기법 2단(보조): block-wise 학습 스케줄(Δ1 iteration씩 순차 학습 + Δ2 구간 재학습으로 vanishing gradient 회피), dynamic weight sharing(unsatisfied CN과 satisfied CN에 다른 weight `ŵ^(ℓ)` 부여, iteration별 3개 가중치로 축소, 가중치수 97% 감소).
7. 부수 트릭: transfer learning(다른 코드에서 학습된 가중치를 초기값으로 사용, weight sharing 구조 동일 코드 간 직접 전이 가능, 샘플수 50배 절감), data augmentation(오류비트 위치에 노이즈 β만큼 강화 주입해 UC 벡터 361배 빠르게 수집), FER loss function `(1/2)[1-sgn(min m_v^o)]` 사용.
8. 학습/최적화: Adam optimizer(lr 0.001, 20epoch마다 half decay), TensorFlow+GPU, Mersenne Twister 난수, Monte Carlo로 UC 벡터 최소 100개 확보까지 반복.
9. 결과: (512,256) 5G LDPC에서 boosted NMS가 5dB에서 FER 10⁻⁹(WMS는 동일 SNR서 10⁻⁵, 10⁴배 이득) 달성; WiMAX(576,432)에서 SOTA post-processing 대비 1/3 iteration만으로 comparable 성능; 5-bit 양자화된 boosted NMS가 floating-point 성능에 근접.
10. 한계: 학습·평가는 GPU 시뮬로 진행되며 실제 FPGA/ASIC 합성이나 하드웨어 실측 검증은 없고(gatecount 등 HW 수치 미제시), boosting으로 총 iteration 수(최대 50~250)가 늘어나는 latency trade-off는 병렬 배치로 완화한다고만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS 디코더(min-sum + 학습형 channel/CN weight) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC는 이미 min-sum 기반이며 magnitude 양자화 테이블을 쓰므로, 본 논문의 학습된 weight를 CN/VN 연산에 곱하는 형태로 이식 가능 — 알고리즘 자체 변경 없이 가중치 테이블 추가/치환만으로 적용 가능(이식 난이도 낮음) |
| dynamic weight sharing (SCN/UCN 별도 가중치) | `decoder.cpp` `Get_VNU_*`, `ecc_data.h` `PARAM_LLR` | 현재 iteration별 LLR threshold 테이블 구조와 유사한 형태로 SCN/UCN 구분 weight 테이블 추가 가능, 기존 적응형 LLR 구조와 정합성 높음 |
| boosting learning (base+post 2단 디코딩, UC 벡터 재학습) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | base 실패분만 별도 weight set으로 이어서 복호하는 구조는 기존 iteration 루프에 두 번째 weight 테이블만 추가하면 구현 가능, 조기종료(Partial CRC)와 결합해 base 단계서 CRC pass하면 post 생략 가능 |
| 5-bit 양자화 손실 완화(학습으로 quantization 부작용 상쇄) | `channel.cpp` `Set_LLR_Th()`, `decoder.h` `Get_VNU_*`(HD/2SD/3SD) | Prime ECC는 hard-1bit/soft-2~3bit 지원, soft-4bit+ 미지원이므로 본 논문의 5-bit 결과를 그대로 쓰려면 양자화 레벨 확장 또는 2~3bit 재학습 필요 |
| 부호(WiMAX/WiFi/5G QC-LDPC) 자체 | `ecc_top.cpp` `Load_PCM()` | 부호 구조 자체는 이식 대상 아님(Prime ECC 고유 H-matrix 유지, weight만 재학습) |

적용 가치: 높음 — Prime ECC가 이미 보유한 min-sum·적응형 LLR·조기종료 구조와 정합성이 높고, 부호/HW 아키텍처 변경 없이 가중치 학습·테이블 교체만으로 waterfall과 error-floor를 동시 개선할 수 있어 이식 난이도가 낮고 기대 효과가 큼. 단, soft-4bit+ 양자화 결과이므로 Prime ECC의 2~3bit 환경에서 재검증 필요.
