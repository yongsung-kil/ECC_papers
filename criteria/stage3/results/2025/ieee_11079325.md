### B. 알고리즘 요약

1. 대상은 짧은 블록코드(길이 32~48)로, 이진 내부코드(n1,k1,d1)(Code 1), GF(2^k1) 위의 Reed-Solomon 외부코드(n2,k2,d2)(Code 2), GF(2^(n1-k1)) 위의 오류국소화코드(n2,k3,d3)(Code 3) 3종을 캐스케이드로 결합하는 새 부호 구성.
2. 기존 캐스케이드/일반화 캐스케이드 코드를 재해석해 오류 위치를 특정(error localization)하는 근본적으로 새로운 부호 계열과 복호 알고리즘을 제안하려는 문제의식.
3. 채널 모델은 표준 AWGN(BER vs SNR) 시뮬레이션이며 별도 NAND/저장매체 특화 모델은 없음.
4. 핵심 기법은 인코딩: k2개 GF(2^k1) 심볼을 Code 2로 부호화 후 각 심볼을 이진 Code 1로 부호화해 `n1×n2` 행렬 구성, k3개 GF(2^(n1-k1)) 심볼은 Code 3로 부호화 후 의사난수 원소를 곱해 행렬 하단에 비트별 가산.
5. 복호는 Code 3가 지시하는 오류 위치(coset) 후보들에 대해 Code 1→Code 2 순으로 복호를 반복 수행하는 coset 기반 방식.
6. 확장 예시로 outer RS 코드를 길이 4~8로 다양화한 3차원 캐스케이드 구성(예: (32,16), (40,20), (48,24))을 제시, RS 외부코드에 erasure correction까지 추가하면 정정력 추가 개선 가능.
7. 구현 디테일: (32,16) 코드는 32,768개 코드워드 전수탐색 대신 56개 erasure 패턴만 검토해 ML급 성능에 근접(연판정/양자화 개념은 등장하지 않음).
8. 학습/최적화 절차 없음(수학적 구성법 및 복호 알고리즘 설계).
9. 결과: (32,16) 코드가 (32,15) ML 복호와 유사한 BER 성능, 5G NR 폴라코드보다 동일/일부 더 긴 블록길이에서도 우수한 블록오류율(폴라코드가 ELC 대비 3배 긴 블록길이일 때만 역전).
10. 한계: HW 설계 없이 이론/시뮬레이션 수준에 그침, 복잡도는 비관적 상한 추정치(예: 53,760회 이진 덧셈)뿐이고 Hamming(8,4) ML 반복 복호 횟수 축소 알고리즘은 "개발 중"으로 미완성.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 캐스케이드 ELC 부호 구성(Code1+Code2+Code3) | 대응 없음 | QC-LDPC가 아닌 Reed-Solomon/이진 블록코드 캐스케이드 구조로 Prime ECC 부호 체계와 무관 |
| coset 기반 준-ML 복호 알고리즘 | 대응 없음 | LDPC min-sum/BP 계열이 아닌 대수적 복호로 `decoder.cpp`와 대응 없음 |
| H-matrix/PCM 로드 | `ecc_top.cpp` `Load_PCM()` | 논문은 H-matrix 기반 LDPC가 아니므로 개념적 대응만 있고 실질 관련성 없음 |

> 적용 가치: 낮음 - LDPC 자체를 다루지 않는 캐스케이드/Reed-Solomon 계열 부호 구성법이라 Prime ECC의 QC-LDPC 디코더·부호설계 어느 레이어에도 이식할 지점이 없음.
