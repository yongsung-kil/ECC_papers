### B. 알고리즘 요약

1. Courtois-Finiasz-Sendrier(CFS) 코드 기반 전자서명 스킴을 QC-LDPC 부호(길이 16384, rate 0.75)로 대체해 개선.
2. 기존 CFS는 Goppa 코드 기반이라 공개키 크기가 매우 크고(6283256 bytes), 서명 시도 횟수가 평균 `t!`로 비효율적이라는 문제를 해결하려 함.
3. 채널/노이즈 모델 없음 — 암호학적 syndrome decoding 문제(SD)와 Goppa Code Distinguishing(GCD) 문제의 계산적 난해성에 기반한 보안 모델.
4. 핵심 기법: QC-LDPC의 parity-check 행렬 `H`에서 최소 해밍웨이트 codeword의 support를 기준으로 열을 puncturing(`GD`)한 뒤, 무작위 열을 재삽입(`GDI`)해 공개키 `H_pub = S·H_DI·P`를 생성.
5. Puncturing은 최소 weight codeword 구조를 이용해 오류정정능력은 유지하면서 행렬 규칙성을 깨뜨려 구조적 공격(Stern's attack, OTD attack)에 대한 저항성을 높임.
6. 서명 생성 시 BP(Belief Propagation) 디코딩으로 syndrome을 codeword로 복호, 준순환 구조 덕분에 반복 시도 횟수가 `t!`에서 `t`의 상수배로 감소.
7. 부수 트릭: random insertion으로 삽입된 마지막 p개 원소는 서명 시 별도 재계산식(`e_p = s*_p + R·e_{n-p}`)으로 복원.
8. 학습/최적화 절차 없음 — 파라미터(p, l 등)는 Stern's algorithm 공격 비용 분석으로 이론적 산출.
9. 결과: 키 크기 6144 bytes(원 CFS의 6283256 bytes 대비 대폭 감소), 서명 시도 횟수 `t!` → `O(t)`, Stern's attack에 대한 work factor `W=2^98.39`로 충분한 보안성(≥2^80) 확보.
10. 한계: 순수 이론적 보안 분석·키크기/시도횟수 비교뿐이며 HW 구현·실측 없음. BER/error-floor 등 통신 오류정정 성능 지표는 다루지 않음(서명 정확성만 문제).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 기반 puncturing/random insertion | `ecc_top.cpp` `Load_PCM()` | 부호구조 변형이지만 목적이 정정능력이 아닌 암호 공격 저항성이라 직접 대응 없음 |
| BP 디코딩(syndrome decoding) | `decoder.cpp` `LDPC_Decoder()` | 알고리즘 명칭만 유사(BP), 목적(서명 syndrome 복호 vs NAND 데이터 정정)이 달라 실질적 대응 없음 |
```

> 적용 가치: 낮음 — 논문은 전자서명(암호학) 응용이며 NAND의 데이터 오류정정 목적과 무관, QC-LDPC 구조를 다루지만 puncturing/random insertion은 보안 목적의 구조 교란이라 이식 대상이 아님.
