### B. 알고리즘 요약 (10줄 내외)

1. 압축센싱(CS)은 `y = Ax` (m<n)인 언더디터민드 선형계에서 sparse 신호 `x`를 복원하는 문제이며, LDPC 측정행렬 `A`의 Tanner 그래프에서 반복적 메시지 패싱(interval passing algorithm, IPA)으로 복원한다.
2. 기존 IPA(FIPA, flooding)는 매 반복마다 모든 CN을 동시에 갱신 후 모든 VN을 갱신하는 방식으로, 메시지 패싱 복잡도가 크고 termatiko set 구조에 취약하다.
3. 채널/신호 모델은 binary/non-negative sparse 신호 `x`, LDPC 측정행렬 `A`(QC 또는 랜덤 regular/irregular), 측정값 `y=Ax`.
4. 제안 기법(SIPA)은 채널코딩의 순차(sequential) BP 복호에서 영감을 받아, 각 반복 내에서 CN들을 무작위 순서로 하나씩 순차 스케줄링하며, 먼저 갱신된 CN 메시지를 같은 반복 내 다른 VN 갱신에 즉시 반영한다.
5. VN→CN 하한 메시지는 `μ(vi,tj)→cj = max(...)` 형태로 이미 스케줄된 이웃 CN의 최신(현재 반복) 메시지와 아직 스케줄되지 않은 CN의 이전 반복 메시지를 함께 사용 — 이 갱신 시점(`tj`) 구분이 SIPA의 핵심.
6. 이론적으로 SIPA는 FIPA보다 항상 같거나 더 정확하며(Theorem 1: SIPA가 실패하면 FIPA도 실패), (2,3) 정규 LDPC 측정행렬에서 FER(SIPA) ≤ FER(FIPA) 증명(Theorem 3).
7. termatiko set(IPA 실패를 유발하는 부구조)은 SIPA에도 동일하게 실패를 유발함(Lemma 3) — 채널코딩의 trapping set과 달리 순차 스케줄링으로 극복되지 않음.
8. 복잡도는 CMP/SUB/ADD 매크로 연산 수로 계산하며, VN을 한 이웃에 대해서만 계산 후 재사용하는 최적화(Remark 1,2)를 FIPA/SIPA 모두에 적용.
9. 결과: A3 행렬에서 sparsity 0.06일 때 SIPA가 FIPA 대비 평균 반복수 42.4% 감소, A5 행렬 sparsity 0.05에서 연산량 약 36% 감소; PCR(정확복원확률) 성능은 저하되지 않음.
10. 한계: SIPA는 순차 특성상 완전 병렬화 불가(FIPA는 완전 병렬화 가능), HW 구현 미제시(시뮬만 수행), BEC/이진 CS 환경에 한정된 분석.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 순차(sequential) CN 스케줄링 아이디어 | 대응 없음 | Prime ECC는 flooding 기반 min-sum(C2V_Cal 등) 구조이며, IPA 자체는 CS 복원용으로 decoder.cpp의 min-sum 알고리즘과 목적함수가 다름 |
| 측정행렬(LDPC) Tanner 그래프 구조 | 대응 없음 | 부호 PCM이 아닌 CS 측정행렬(non-negative, max/min 연산)이라 ecc_top.cpp Load_PCM()과 무관 |
| termatiko set 분석 | 대응 없음 | trapping set/ABS와 유사 개념이나 CS 전용 실패구조, decoder.cpp 이터레이션 루프와 직접 대응 없음 |
```

적용 가치: **낮음** — 압축센싱 신호복원 문제로 binary NAND LDPC ECC 복호(min-sum, error-floor)와 도메인이 달라 이식 가치가 낮음.
