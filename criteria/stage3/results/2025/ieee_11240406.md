### B. 알고리즘 요약

1. 대상은 이진 선형블록코드 `C(n,k,d_min)`, BSC 채널(`y=(1-2c)+z`, cross-over 확률 `ρ`), 검증 코드는 Tanner code (155,64,20).
2. 문제: 유한길이 코드에서 trapping set/pseudocodeword로 인한 error floor를 반복 디코더(BF/BP)가 근본적으로 해소하지 못함, RL로 디코딩을 순차 의사결정 문제(MDP)로 재구성.
3. 기본 MDP는 상태=신드롬(길이 `m`), 행동=비트 `i` 플립(`A={1,...,n}`), 보상은 성공 시 `+1`, 스텝마다 `-1/L` 페널티; 상태공간이 `2^{n-k}`로 저rate·긴코드에선 불가.
4. 핵심 기법: 기존 디코더 `Φ`(예: BF)를 그대로 두고, `Φ`가 실패하는 신드롬 집합 `S_f^Φ`만 새 MDP의 상태공간으로 축소하는 "feedback decoder" 구조(식 6~9).
5. 핵심 식: 실패영역 `R_f^Φ = {x | Φ(x)∉R_c^Φ∪R_m^Φ}` — 상태공간을 `2^{n-k}`에서 `|S_f^Φ|+1`로 줄여 Q-learning/DQN을 실용화.
6. 확장(2단): Hamming ball `B(c,w)`로 상태공간을 반경 `w` 이내로 추가로 자르는 truncated MDP(식 13~21)로 더 축소.
7. 구현: Q-learning(테이블 기반, `w=2,3`)과 Deep Q-Network(신경망 기반, `w=2,3,4`)를 각각 학습해 상태공간 크기에 따라 선택.
8. 학습/최적화: Q-learning 업데이트(`Q(s,a)←(1-α)Q(s,a)+α[R+γ max Q(s',a')]`), DQN은 replay buffer + target network로 안정화.
9. 결과: Tanner code (155,64,20) 기준, BF 단독 대비 feedback(Q-learning w=2,3; DQN w=2,3,4)가 FER/BER 개선, DQN(w=3,4)는 학습 시 못 본 error pattern까지 일반화해 교정.
10. 한계: BSC + bit-flipping(hard-decision) 조합만 실험, soft-decision min-sum/QC-LDPC 미검증, 상태공간 축소에도 `w` 커지면 Q-table이 여전히 비대(DQN 필수), HW 구현·gatecount·throughput 전혀 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 기존 디코더 실패 영역만 보정하는 feedback 구조 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 후단 | min-sum 디코딩이 실패(non-convergence)한 경우에 한해 추가 교정을 붙이는 post-processing 관점에서 개념적으로 대응 가능 |
| 조기종료 후 실패 시 추가 처리 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | CRC 실패(미수렴) 시점에 RL 피드백을 넣는 위치로 대응 가능하나 RL 정책 자체는 미구현 |
| bit-flipping 알고리즘 자체 | `decoder.cpp` | Prime ECC는 min-sum 기반이라 BF 알고리즘과 직접 대응 없음 |
| Q-table/DQN 추론 엔진 | 대응 없음 | 학습된 정책을 HW에서 추론하는 구조가 Prime ECC에 없음 |

적용 가치: **중간** — 실패한 error pattern만 다루는 feedback 개념은 error-floor 개선 post-processing으로서 응용 가능성이 있으나, hard-decision BF 소규모 검증뿐이라 Prime ECC의 soft min-sum·QC-LDPC 구조로의 이식은 추가 연구 필요.
