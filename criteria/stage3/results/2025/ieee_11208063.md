### B. 알고리즘 요약 (10줄 내외)

1. STT-MRAM 채널(BAC로 write/read disturb 오류 모델링 후 Gaussian mixture channel으로 R0/R1 분포 모델링)에서 protograph LDPC(P-LDPC) 코드, rate `0.625`, lifting size `Z=85`, 정보길이 `850`bit 사용.
2. 공정변이·열변동으로 인한 write/read 오류에 더해, 온도 상승 시 채널 디코더가 모르는 resistance offset `ΔR1`이 발생해 표준 MS/NMS/OMS 디코딩 성능이 저하되는 문제를 해결.
3. STT-MRAM은 본질적으로 비대칭 채널이라 i.i.d. channel adapter로 대칭화한 후 NN 기반 디코딩을 적용한다는 전제.
4. 핵심 기법 1단: base graph의 병렬 edge(같은 VN-CN 쌍에 다중 edge)를 단일 edge `ẽb`로 병합한 축소 base graph `G̃`를 만들고, 이를 기반으로 I=25개 hidden layer(각 iteration마다 VN/CN 업데이트 sublayer)를 갖는 neural MS 디코더를 구성.
5. 핵심 식: `L_{ẽc}^i = S × ReLU(α_{ẽ}^i × min|L_{ẽv}^i| − β_{ẽ}^i)`(min-sum에 학습 가능한 weight `α`, bias `β`를 도입한 형태, sign은 `S`로 별도 곱함) — 정확도를 높이면서도 min-sum 구조를 유지해 HW 친화성을 확보하려는 의도.
6. 핵심 기법 2단: 병렬 edge의 |eb|×Z개 LLR을 Z개 PE에 재배정할 때 Scheme A(cluster 내 최소 절대값 선택)와 Scheme B(cluster 내 평균) 두 가지 방식을 제안.
7. 부수 트릭: transfer learning으로 resistance offset 문제를 source domain(offset 없음)에서 사전학습 후 target domain(offset 있음)에서 파인튜닝, 학습데이터 90% 절감.
8. 학습 절차: Adam optimizer, learning rate `2×10⁻⁵`, 20,000 batch(batch당 codeword 10개)로 α/β 학습.
9. 결과: Scheme A/B 모두 표준 MS/NMS(α=0.8)/OMS(β=0.15)보다 FER 우수, offset 있는 채널에서 TL+10% 데이터로도 TL 없는 경우와 근접한 성능이며 BP보다도 우수.
10. 한계: STT-MRAM 특화 채널·단일 code 설정에서만 시뮬레이션, HW 설계·gatecount·throughput 정보 없음, iteration마다 별도 α/β 테이블을 학습·저장해야 하는 부담.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Neural MS의 weight/bias 학습 min-sum (식 1) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산 구조 자체를 유지한 채 iteration별 α/β를 학습된 상수 테이블로 대체 가능성 있음, HW 테이블 lookup 형태로 이식 시도 가능 |
| iteration별 학습 파라미터(α^i, β^i) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 iteration별 LLR threshold 테이블과 유사한 위치에 학습된 스케일/오프셋을 추가 적용 가능 |
| 병렬 edge 병합(Scheme A/B) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정 구조로 병렬 edge 병합 이슈가 다르게 나타남, 부호구조 종속이라 이식 난이도 높음 |
| Transfer learning 기반 offset 적응 | `channel.cpp` `Set_R_Offset()` | resistance offset 적응 개념은 채널 offset 보정 로직과 유사하나 TL 학습 파이프라인 자체는 대응 없음 |
```

적용 가치: `중간` — min-sum 골격을 유지한 채 학습된 weight/bias만 추가하는 구조라 decoder.cpp의 min-sum 연산에 상수 테이블 형태로 이식 검토 가치가 있으나, STT-MRAM 특화 채널 검증이라 NAND 채널 재검증이 필요함.
