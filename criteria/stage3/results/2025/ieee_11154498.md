### B. 알고리즘 요약

1. 대상은 (96,48) QC-LDPC 짧은 부호, BI-AWGN 채널, BPSK 변조, BP 디코딩 기반.
2. 짧은 길이 LDPC는 BP가 MLD 대비 열위이며, 소수의 문제 VN이 수렴 실패의 원인이라는 관찰에서 출발.
3. Multi-Round BP(MRBP)는 신뢰도 낮은 VN을 찾아 채널 LLR을 `Lich,t = -∞·sign(Lich)` 형태로 saturate·flip해 병렬로 `T`개 perturbation을 시도, 각 재디코딩 결과 중 `argmin||y-(-1)^c||`로 최종 선택.
4. 기존 expert rule(채널 LLR 크기, APP LLR 크기, nSMEA 부호반전 카운트)을 신경망으로 대체하는 것이 핵심 아이디어.
5. 핵심 개선: 입력을 채널값 `y` 대신 syndrome-based neural decoder(SBND)에서 검증된 충분통계량 `(|Lch|, sch)`로 바꾸고, 구조도 MLP에서 stacked GRU(5-layer, hidden size `6(2n-k)`)로 교체.
6. 학습은 binary cross-entropy loss `LBCE = -(1/n)Σ[bi·log(b̂i)+(1-bi)·log(1-b̂i)]`로 각 VN의 perturb 필요 여부를 이진분류.
7. 데이터셋은 3dB SNR에서 BP 실패 샘플 60M개, `l0=l1=20` iteration으로 라벨(perturb 성공여부) 생성, 단일 SNR 학습으로 넓은 SNR 범위에 일반화됨을 관찰.
8. 학습 절차: PyTorch, Adam(`lr=1e-4`), Reduce-on-Plateau, 250 epoch.
9. 결과: GRU-D2(20.6M 파라미터)가 nSMEA expert rule 대비 `T=1` perturbation만으로 우수, 기존 참고 디코더[9]는 절반의 perturbation 횟수로 동등 성능 달성. worst-case latency는 `l0+l1=40` BP iteration (모델 추론시간 제외).
10. 한계: 파라미터 수가 20.6M으로 비현실적으로 크며, HW 구현 없음, 검증은 단일 짧은 부호(96,48)에 한정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BP 실패시 VN perturbation을 통한 재디코딩 (MRBP) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | min-sum 실패 시 특정 VN LLR을 강제 조정 후 재시도하는 post-processing으로 이식 가능성 있음, 단 조기종료(CRC) 구조와 결합 설계 필요 |
| perturb 대상 VN 예측용 신경망(SBND, GRU) | 대응 없음 | NN 전면 도입은 Prime ECC의 bit-exact HW 시뮬레이터 구조와 이질적 |
| syndrome+LLR 크기 입력 표현 | `channel.cpp` `Set_LLR_Th()` | 개념적으로 LLR threshold/신뢰도 판정과 유사하나 실제 대응은 없음 |
```

> 적용 가치: 중간 — MRBP의 "실패 VN 식별 후 재시도" 개념 자체는 error-floor/waterfall 개선에 유의미하나, 제안된 구현체(대형 GRU)는 그대로 이식 불가하며 경량 휴리스틱화가 필요.
