### B. 알고리즘 요약

1. QC-LDPC 부호의 PCM은 J×L exponent matrix `E`와 circulant size `P`로 결정되며, block length는 `L×P`, rate는 `R ⩾ 1 - J/L`이다.
2. Girth-8(4-cycle, 6-cycle 없음) 코드를 탐색할 때 J,L,P가 커지면 완전탐색 복잡도가 급증하는 문제를 해결하고자 한다.
3. Tanner graph의 2k-cycle 존재 조건은 순환합 `Σ(p_{J_i L_i} - p_{J_{i+1} L_i}) = 0 mod P` 식 (3)으로 정의된다.
4. 핵심 기법 1: exponent matrix를 상하 절반이 부호 대칭(`A`, `-A`)이 되는 vertical symmetric 구조 (식 4)로 가정, J가 홀수면 첫 행을 0벡터로 패딩(식 5).
5. 대칭 구조 하에서 4-cycle/6-cycle을 배제하는 조건을 재유도한 것이 Proposition 1의 부등식 그룹 (6)(7)이며, 상반부만 만족시키면 자동으로 하반부도 만족되어 탐색량이 절반으로 줄어든다.
6. 핵심 기법 2: reverse 구조(식 9) - 상반부와 하반부가 행 순서가 뒤집힌 대칭 관계로, 마찬가지로 상반부만 탐색.
7. 탐색은 greedy row-by-row: 각 원소에 {0,...,P-1}을 순차 대입하며 4/6-cycle 위반 시 backtrack, P-1까지 실패하면 이전 원소로 되돌아감.
8. 최적화 절차는 별도 학습 없이 P를 1씩 줄여가며 최소 P(=Pmin)를 찾는 exhaustive greedy search.
9. 결과: 여러 (J,L) 조합에서 기존 horizontal symmetric[7], explicit vertical[13], RDS[14], IRS[15] 대비 비슷하거나 일부 더 작은/경쟁력있는 P값 및 비슷한 BER 성능(AWGN, BPSK, 50 iteration) 확보.
10. 한계: 순수 이론/시뮬(AWGN) 검증뿐이며 디코더 알고리즘·HW 구현은 다루지 않고, RDS는 J=4 한정, explicit 구성법들은 code length 면에서 불리함이 남음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-8 exponent matrix 구성법 (symmetric/reverse) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 대체하는 새 구성법이나, Prime ECC는 특정 QC-LDPC 고정이라 재검증 부담 큼 |
| 4/6-cycle 배제 탐색 알고리즘 | 대응 없음 | 부호 생성 단계(오프라인) 도구로, 런타임 디코더/인코더 모듈과 무관 |
| BER 시뮬레이션(AWGN, min-sum 아님) | 대응 없음 | 논문은 자체 이상적 디코더로 검증, Prime ECC의 min-sum HW 모델과 직접 연결점 없음 |

> 적용 가치: 낮음 — 순수 부호 구성법(exponent matrix 탐색)이며 디코더 알고리즘·HW·연판정과 무관해 이식성이 `하`이고, Prime ECC는 이미 고정된 QC-LDPC PCM을 사용하므로 재설계 부담이 큼.
