### B. 알고리즘 요약

1. 대상은 spatially coupled turbo-like codes(SC-TCs) 계열: half spatially coupled braided convolutional codes(HSC-BCCs)와 half spatially coupled parallel concatenated codes(HSC-PCCs). rate-2/3 duo-binary RSC 성분부호, AWGN/BEC 채널 가정.
2. 기존 SC-BCC(type-1/type-2)는 window decoding 시 decoding threshold 손실이 발생하고, threshold saturation을 위해 큰 coupling memory(m≥5)가 필요해 구현이 복잡함.
3. 핵심 가정: coupling 비율 `λi`가 완전 랜덤인 기존 SC-PCC 구조를 재해석해, 결정론적(deterministic) 결합 구조로 대체 가능함을 보임.
4. 핵심 기법: `λi=0 (i=-m,...,-1)`로 강제해 forward 방향으로만 결합 정보를 전달하는 single-sided SC-PCC를 정의하고, 특히 `m=1, λ0=λ1=0.5`인 half SC-PCC(HSC-PCC)를 제안.
5. HSC-PCC는 `λ0=...=λm=1/(m+1)`일 때 완전 BP 복호 threshold가 uncoupled ensemble의 MAP threshold에 도달함을 증명(Appx. A, scalar admissible system 이용).
6. HSC-BCC(지연계수 `δ≥2`)는 rate-1/2 RSC 대신 rate-2/3 duo-binary RSC를 사용, 이전 시점의 parity 시퀀스를 같은 side의 성분부호로 재인코딩해 짧은 decoding cycle을 회피.
7. 구현상 이점: 별도 segmentation 없이 delay factor `δ`만 증가시켜 coupling memory를 확장, (de-)multiplexing 연산과 디코더 복잡도를 크게 절감.
8. 별도 학습/최적화 절차 없음(density evolution 기반 이론 해석 및 시뮬레이션 검증).
9. 결과: rate-1/3 HSC-BCC(δ=2)의 window decoding threshold(ϵBP,wd=0.6661)가 uncoupled BCC의 MAP threshold에 근접, type-1/type-2 BCC 대비 BER 개선 및 평균 iteration 감소, (3,6)-regular protograph SC-LDPC의 full decoding보다도 우수한 window decoding 성능.
10. 한계: turbo-like code(BCC/PCC) 프레임워크로 QC-LDPC/binary LDPC 부호와 직접 대응되지 않으며, HW 구현·합성 결과 없이 시뮬레이션(AWGN/BEC, linear-log-MAP)에 국한됨.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| HSC-BCC/HSC-PCC 결합 부호구조 | 대응 없음 | turbo/BCC 계열 부호로 Prime ECC의 QC-LDPC binary 구조와 무관 |
| duo-binary RSC 성분부호, log-MAP 디코딩 | 대응 없음 | min-sum 기반 LDPC 디코더(decoder.cpp)와 알고리즘 계열이 다름(turbo-style vs LDPC BP/min-sum) |
| window decoding 스케줄링(FF/RT) | 대응 없음 | Prime ECC는 QC-LDPC 고정 스케줄(C2V/V2C, Dual-Update)이며 SC 구조 window decoding 개념 없음 |
```

적용 가치: **낮음** — turbo-like code(BCC/PCC) 계열 부호설계 논문으로 Prime ECC의 binary QC-LDPC 아키텍처(디코더/인코더/HW)와 알고리즘 계열 자체가 달라 이식 지점이 없음.
