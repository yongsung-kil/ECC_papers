### B. 알고리즘 요약 (10줄 내외)

1. CV-QKD(continuous-variable quantum key distribution) reverse reconciliation에서 Bob이 syndrome `s = uH^T`를 계산해 Alice에 전송, Alice가 Layered BP 디코딩으로 raw key를 복원하는 시스템, 코드 rate 0.1/0.2, 길이 N=96000/80000.
2. 고속 CV-QKD의 병목은 error-correction 처리율이며, FPGA 자원 제약상 fixed-point(제한된 비트폭) 연산이 필요하지만 이는 SNR<0dB인 CV-QKD 환경에서 정정 정확도를 떨어뜨려 residual bit error와 높은 FER을 유발.
3. 채널/신호 모델은 CV-QKD의 낮은 SNR(약 0dB 이하) 가정, LLR 기반 hard decision `û_n = 0 (LLR≤0) or 1 (LLR>0)`.
4. 핵심 기법은 Layered BP 디코딩 후 잔여 오류를 syndrome 기반으로 소거하는 residual bit error correction: `|LLR_n|<Δ`인 비트를 오류후보 집합 `e`, `|LLR_n|≥Δ`인 비트를 신뢰집합 `ē`로 분리.
5. 신뢰집합으로 보정 syndrome `s_c = s ⊕ s_ē` (`s_ē = û_ē H_ē^T`)를 계산하고, `H_e`의 특정 행(row-weight=1)을 이용해 해당 오류비트를 `s_c` 값으로 직접 확정, 확정된 비트를 `e`에서 `ē`로 이동시키며 반복.
6. 확장 기법 없음(단일 기법 + HW 파이프라이닝 구조).
7. HW 구현 디테일: `sub_decoder`(Layered BP) 9개 + `error_bits_erase`(잔여오류 소거) 3개 모듈을 파이프라인으로 연결(반복 디코딩이 오류소거보다 느려 3:1 비율로 자원 절약), Xilinx XCVU13P FPGA, throughput `= f·N/(D·t_max)` × 9(sub_decoder 개수).
8. 학습/최적화 절차 없음(threshold `Δ` 등은 고정 설계 파라미터로 서술, 최적화 알고리즘 언급 없음).
9. 결과: fixed-point만 사용 시 FER이 floating-point 대비 크게 열화되나, residual bit error correction 결합 시 FER이 floating-point 수준 또는 그 이상으로 개선; throughput 1.59Gbps(rate0.2, FER 0.3632→개선), 1.17Gbps(rate0.1, FER 0.1685→개선); 25km/50km 전송거리에서 각각 95.55Mbps/16.84Mbps 실시간 SKR 지원.
10. 한계: FER<0.1 수준(NAND 요구 BER 10⁻¹² 대비 매우 높음)이며 CV-QKD 특유의 초저rate·초장길이·초저SNR 조건에 특화되어 NAND 고rate 채널로 정량적 이식이 어려움; gatecount 등 세부 HW 자원 수치 본문 미기재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| syndrome 기반 residual bit error correction(erasure) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 후처리 | 조기종료 후 잔여오류를 syndrome/row-weight=1 조건으로 소거하는 아이디어는 error-floor 대책과 유사한 후처리 개념이나, Prime ECC의 조기종료(Partial/Full CRC)와는 메커니즘이 다름(참고용) |
| Layered BP 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 iterative message passing과 개념 중복, min-sum 대비 BP(합-곱) 계열이라 직접 대응 낮음 |
| fixed-point 정밀도 제한에 따른 LLR 설계 | `channel.cpp` + `ecc_data.h` `PARAM_LLR` | 비트폭 제약과 FER 관계를 다루나 CV-QKD 특유의 초저SNR 환경 전제라 NAND soft-read LLR 설계와 직접 대응 낮음 |
| 초저rate/초장길이 LDPC 부호 구조 | `ecc_top.cpp` `Load_PCM()` | NAND 고rate QC-LDPC와 부호 스펙이 근본적으로 다름, 대응 없음에 가까움 |

> 적용 가치: **낮음** — syndrome 기반 residual error erasure라는 후처리 아이디어는 참고할 만하나, 부호 스펙(초저rate·초장길이)과 채널 가정(CV-QKD 초저SNR)이 NAND 고rate binary QC-LDPC와 근본적으로 달라 직접 이식 가치는 낮음.
