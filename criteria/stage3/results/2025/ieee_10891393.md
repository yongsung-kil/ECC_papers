### B. 알고리즘 요약 (10줄 내외)

1. Hadamard-LDPC 부호: CN을 SPC 대신 order `r`의 Hadamard 부호 제약(HCN)으로 대체, 하나의 HCN에 `2r`개 VN 연결, VN은 NVN(정보/패리티 담당, `r+2`개)과 HVN(degree-1, 나머지)로 구분.
2. 문제: 고SNR에서 HCN의 EXIT 곡선이 concave→convex로 바뀌어 고rate Hadamard-LDPC 설계가 어려움; 기존은 저rate 위주.
3. 채널/모델: AWGN 채널, EXIT chart 기반 decoding threshold 분석.
4. 1단계: NVN/HVN 각각의 puncturing 분포 `P`(NVN 대상), `Q`(HCN 타입 대상)를 정의하고 NVND/HCND EXIT 함수 `I_E,VND`, `I_E,HCND`를 유도, differential evolution으로 최소 decoding threshold `(Es/N0)_th` 탐색.
5. 핵심 관찰: HVN을 puncture하면 HCN의 extrinsic info 곡선이 고SNR에서 convex→concave로 바뀌어 고rate 설계 가능해짐 (Fig.2).
6. 2단계: HCN의 i번째 edge 신뢰도를 최소 유클리드거리 `dis_i(η)`로 정의(식 6); `dis_i(η)=0`이면 해당 NVN은 첫 iteration에 정보 획득 불가.
7. Greedy 알고리즘: 신뢰 가능한 punctured NVN 수 `φ`를 최대화하도록 unreliable NVN을 다른 unpunctured NVN과 교환, HVN puncture 위치도 재탐색(최대 T=10 라운드).
8. 부호 스펙: rate-0.09 mother QC-Hadamard-LDPC (r=4, base matrix 9×19, lifting factor 104), rate 1/2 이상은 기존 LDPC 사용.
9. 결과: 5G NR LDPC(BG2) 대비 약 0.1dB 개선, 기존 puncturing 전략(largest/lowest-NVN, only-HVN) 대비 최대 0.4~0.7dB 개선.
10. 한계: HW 미설계, 시뮬레이션(BER/threshold)뿐이며 Hadamard 특유 구조 의존적이라 일반 binary LDPC로 직접 이식 어려움.
