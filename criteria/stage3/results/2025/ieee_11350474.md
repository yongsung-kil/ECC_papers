### B. 알고리즘 요약

1. 시스템: 5G NR 3GPP TS 38.212 표준 QC-LDPC 인코더, Base Graph BG1(Kb=8488)/BG2(Kb=3840) 지원, lifting size Z 기반 구조.
2. 문제: QC-LDPC 인코딩 HW 구현이 다양한 code rate/lifting size 적응성과 낮은 지연·높은 처리량을 동시에 요구해 어려움.
3. 모델: H 행렬을 `[[A B][C D][O I]]` 커널 구조로 분해, 코드워드 `C=[S|P1|P2]`.
4. 핵심 기법: `H·C^T=0` 제약에서 `P1^T = B^-1·A·S^T`, `P2^T = C·S^T + D·P1^T` 로 두 단계 parity 계산.
5. 의미: B 블록의 이중대각(double-diagonal) 구조 덕분에 P1 계산이 단순 행렬 연산으로 환원되며, 이후 P2도 C/D 블록으로 순차 계산 가능.
6. 확장: 람다(`λ_i = Σ a_i,j·S_j`) 사전계산으로 barrel-shift + XOR 누적 방식으로 P1 계산 단순화.
7. 구현 디테일: C/D 행렬은 FPGA BRAM에 저장, cyclic shift는 barrel shifter, CRC16-CCITT(`g(x)=x^16+x^12+x^5+1`)를 모듈로 별도 파이프라인화, FSM 기반 top module로 P1/P2/CRC 통합.
8. 학습/최적화 절차: 없음(표준 알고리즘 직접 구현, DE/GA/NN 없음).
9. 결과: Nexys4 DDR에서 TB length 1664bit 기준 26 clock cycle, 33.33MHz, throughput 0.410Gbps, LUT 6589개(10%), on-chip power 0.142W.
10. 한계: 인코더만 구현(디코더 없음), CRC24 미지원, 클록 300~400MHz급 최신 설계 대비 처리량 낮음, 시뮬/FPGA 검증만이고 ASIC 합성 없음.

