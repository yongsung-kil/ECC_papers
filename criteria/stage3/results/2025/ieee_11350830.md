> 총평: NMS 기반 binary QC-LDPC HW 실시간 구현(50Gbps 실측)으로 라우팅·파이프라인 스케줄 아이디어는 참고가치 있으나 표준 부호 고정 구현이라 알고리즘 이식성은 중.

### B. 알고리즘 요약

1. 50G-PON(ITU-T G.9804)용 표준 FEC인 binary QC-LDPC(17280,14592), rate≈0.844, H=3072×17664(12×69 array of 256×256 submatrix, CPM shift 또는 -1 zero)를 FPGA OLT/ONU에 구현.
2. 문제: LDPC H-matrix가 커서 chip area/power가 커지고 실시간 50Gbps 처리가 어렵다 — 알고리즘 선택과 HW 아키텍처가 관건.
3. 인코더는 14592 정보비트에 2688 parity 부가 후 384비트 puncturing해 17280 코드워드 생성.
4. 디코더 알고리즘은 normalized min-sum(NMS) 기반 iterative message passing (VNU↔CNU).
5. 병렬 구조: 병렬화 factor Z=3, 동일 디코딩 서브모듈 3개가 round-robin 입력, 405 클록 내 코드워드 처리로 throughput 극대화.
6. 서브모듈은 5단 파이프라인 — Blockwise Cyclic Shift Registers(CCVM), VNU(3단), CNU(2단)를 fixed wiring으로 연결, 3072×552 공간을 5클록에 처리.
7. VNU가 채널 LLR을 내부 계산해 메모리 접근을 줄이고, fixed interconnect로 복잡한 라우팅 로직 제거.
8. H-matrix를 32 phase로 분할, 각 phase가 수직 슬라이스에 매핑되어 conflict-free로 동시 실행(T=552 VNU, 3072 CNU, 69열이라 1 iteration에 36클록).
9. 결과: 388.8MHz에서 50Gbps, 실시간 optical gain ≈8.7dB(target BER 1e-2), 40Gbps error-free; OLT LUT 67%/FF 29%/8.47W, ONU LUT 74%/FF 35%/29.92W; ZTE·Fudan 대비 40배 throughput.
10. 한계: 표준 부호에 고정된 아키텍처 구현으로 알고리즘 신규성 없음; ASIC gate count 미제시(FPGA 자원만), soft-decision 세부 양자화 비트폭 본문 미기술.
