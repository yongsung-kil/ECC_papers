### B. 알고리즘 요약

1. (576, 432) irregular LDPC 코드(IEEE 802.16e, rate 3/4), AWGN+BPSK 채널을 대상으로 Tanner graph를 unroll한 신경망 기반 BP 디코더 설계.
2. 기존 NBP는 매 iteration·매 edge마다 별도 가중치를 학습해 code length가 길어질수록 파라미터 수와 메모리 소요가 급증하는 문제가 있음.
3. 신경망은 BP의 메시지 전달 구조(Tanner graph unrolling)를 그대로 계층화하고, 각 계층(CN/VN layer)에 학습 가능한 가중치를 부여하는 모델 기반 방식.
4. 핵심 기법: 같은 iteration 내 모든 edge가 동일 가중치를 공유하는 weight-sharing으로 CN→VN 메시지를 `u_c,v^t = 2·arctanh(ω^(t-1)·Π tanh(...))` 형태로 계산(식 11), 파라미터 수를 `3T`(T=iteration 수)로 축소.
5. 핵심 식(13)은 정규화 loss `L = cross-entropy + Σ_t η^(T-t)(...)` 형태로, iteration별 출력 기여도를 학습 초반 균등(η=0)에서 후반 마지막 iteration 위주(η=1)로 점진 이동시켜 학습 안정화.
6. 확장 없음(단일 SNBP-R 구조), ablation으로 weight-sharing 단독 효과와 loss 최적화 단독 효과를 분리 검증(Fig.6, 7).
7. 부수 트릭: 채널 가중치 `w_ch,v^t`와 edge 가중치 `ω^t`를 분리해 정규화 인자처럼 취급, BP와 구조적으로 근접시킴(모든 가중치=1이면 순수 BP로 환원).
8. 학습/최적화: mini-batch(size 100) gradient descent, Adam optimizer(lr=0.001), SNR 1~5dB 균등 분포 데이터로 학습.
9. 결과: 42-layer(20 iteration) 구성에서 SNR=5dB 기준 BER 10^-8 도달, NBP 대비 약 0.6dB 성능 이득, 파라미터 수는 NBP의 `Σ(m_j)·T`에서 `3T`로 대폭 감소.
10. 한계: FPGA/HW 구현 없이 시뮬레이션(BER curve)만 제시, 학습된 H-matrix(Basic Graph A)와 다른 구조(Basic Graph B)로 테스트 시 일반화 성능이 크게 저하되는 model-driven 방식의 근본 한계 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-Product(tanh) 기반 CN 메시지 계산 | 대응 없음 | Prime ECC는 min-sum(magnitude+sign) 기반으로 tanh/arctanh 연산 자체가 없어 회로 대응 불가 |
| iteration별 weight-sharing(정규화 인자 ω^t) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration별 스케일링 테이블이라는 개념은 Prime ECC의 적응형 LLR threshold 테이블과 유사하나, 학습된 실수 가중치 vs 사전계산 정수 테이블로 구현 방식이 상이 |
| 신경망 학습(gradient descent) 기반 파라미터 최적화 | `local_opt.cpp` | 목적(파라미터 자동 탐색)은 유사하나 방법론(NN 역전파 vs 기존 LLR 자동최적화)이 상이하고 신경망 학습 인프라 자체가 코드베이스에 없음 |

> 적용 가치: 낮음 — tanh 기반 Sum-Product 계열과 신경망 학습 구조가 Prime ECC의 min-sum 고정소수점 HW와 근본적으로 다르고, HW 미구현·낮은 일반화(H-matrix 변경 시 재학습 필요)로 실이식 부담이 큼.
