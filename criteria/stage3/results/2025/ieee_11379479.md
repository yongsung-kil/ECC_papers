### B. 알고리즘 요약

1. 시스템: LDPC(`H·c^T=0`) 및 Polar(`x=u·G_N`) 코드 모두에 적용 가능한 codebook-agnostic soft-decision 디코더, AWGN/Rayleigh 채널, LLR `= 2y_i/σ²` 기반.
2. 문제: 기존 soft-decision 디코더(LDPC belief propagation, Polar SCL)는 계산 복잡도·메모리 사용량이 크고 HW 처리량이 낮아 multi-Gbps 5G/6G 요구를 못 맞춤; 기존 ORBGRAND HW도 완전 병렬화 부재로 순차적 노이즈패턴 생성·정렬이 병목.
3. 가정: 수신 LLR 벡터의 절댓값 크기로 비트 신뢰도를 정렬하고, 신뢰도가 낮은 비트부터 우선적으로 flip하는 에러패턴을 논리적 무게(logistic weight, LW) 순으로 탐색.
4. 핵심 기법: ORBGRAND - LLR 절댓값 정렬로 permutation vector `ind`를 만들고, 각 `LW ≤ LW_max`에 대해 정수 분할(integer partition) `λ=(λ1,...,λP)`로 에러패턴 `e`를 생성.
5. 핵심 식: 후보 코드워드는 `y_test = ŷ ⊕ e`, 유효성은 `H·y_test^T=0`(식 11)으로 검사하고 통과 시 `u = y_test·G^-1`로 메시지 복원 - 파티션 순회로 최소 무게 에러패턴부터 탐색해 ML에 근접.
6. 확장: 파라미터 `LW_max`와 파티션 최대원소수 `P`로 탐색 복잡도-FER 트레이드오프 조절(예: 128-length polar에서 `LW_max=64, P≤6`).
7. HW 구현 디테일: 병렬 노이즈추측 엔진 E개로 클록당 여러 에러패턴을 동시 검사(`Throughput ∝ E·f_clk`), `O(log n)` 지연의 하드웨어 정렬기(병렬 비교기 네트워크)로 소프트웨어 정렬(`O(n log n)`) 대체, 엔진 간 파이프라이닝(한 엔진이 `LW_i` 생성 중 다른 엔진은 `LW_i-1` 검증).
8. 학습/최적화 절차 없음(알고리즘·파라미터 선택 + HDL 아키텍처 설계).
9. 결과: FER < 10^-7(단·중간 길이 LDPC/Polar, 중간 SNR), Hard-decision GRAND 대비 최소 2dB FER 이득, 기존 GRAND 기반/전통 soft-decision 디코더 대비 복잡도 최소 한 자릿수 감소(구체 수치는 그림에만 존재).
10. 한계: FPGA/ASIC 합성을 언급하나 본문 서술에 gatecount·정확 throughput·clock 수치가 없음(그림 4,5에만 존재해 `미상` 처리), 짧고 중간 길이 코드 대상이며 긴 블록길이·MIMO 페이딩은 future work로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ORBGRAND 에러패턴 탐색(정수분할, 신뢰도 정렬) | `decoder.cpp` `LDPC_Decoder()` | Min-Sum message-passing과 전혀 다른 codebook-agnostic guessing 방식이라 이터레이션 루프 대체 불가, 대응 없음에 가까움 |
| 병렬 노이즈추측 엔진 / 파이프라이닝 구조 | 디코더 HW 모델 (z=32, pipeline 6-stage) | 병렬화·파이프라인 설계 철학은 Prime ECC의 pipeline 구조와 유사하나 알고리즘 단위가 달라 직접 대응 불가 |
| 경량 하드웨어 신뢰도 정렬기 (`O(log n)`) | `channel.cpp` `Set_LLR_Th()` | Prime ECC는 LLR threshold 테이블 기반, 정렬 자체를 HW로 구현하지 않아 대응 없음 |
| 조기종료(패리티체크 통과 시 stop) | `partial_CRC.cpp` `full_CRC.cpp` | 개념적으로 유사(유효 코드워드 발견 시 종료)하나 CRC가 아닌 parity-check 직접 검사 방식 |

> 적용 가치: 낮음 - ORBGRAND는 min-sum message-passing과 근본이 다른 codebook-agnostic 알고리즘이라 Prime ECC 디코더 루프에 직접 이식 불가하며, 참고 가치는 병렬 엔진·경량 정렬 등 HW 설계 철학 수준에 그침.
