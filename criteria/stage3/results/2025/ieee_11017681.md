### B. 알고리즘 요약

1. 대상은 LDPC ECC를 쓰는 3-D TLC NAND flash(Micron B16A)로, code length `18432`, code rate `8/9`, min-sum 복호, 최대 iteration 40(page당 7 frame이라 page 최대 280).
2. 문제: P/E·retention 증가로 RBER이 오르면 LDPC iteration·read-retry가 늘어 복호 latency가 급증한다.
3. 관찰 1: 실측상 복호 성공 시 iteration은 항상 max 미만, 실패 시 max에 도달 → iteration 수를 복호 정답확률 지표로 사용 가능(Fig.3).
4. 핵심 기법 IACPO: MSB page가 저RBER임을 이용해 MSB 복호결과로 CSB/LSB를 단방향 확정하는 error model 구축, MSB의 `IterMSBi < IterMAX`이면 해당 page의 LLR을 `LLR' = LLR + β·LLR`로 증폭(식 3).
5. 의미: MSB가 신뢰(조기수렴)될 때만 LSB/CSB LLR을 키워 iterative 수렴을 가속, 잘못된 MSB로 인한 오확정을 회피.
6. 파라미터: `β=1.5`가 평균 iteration 최소(Fig.8)로 실험 전반에 채택.
7. 관찰 2/기법 ULDCO: RBER이 LDPC 정정범위 내면 성공하는 read voltage 구간 SDA가 존재하고, 서로 다른 retention의 SDA가 겹치는 `SDAoverlap` 중앙값을 read offset으로 쓰면 hard-decision read만으로 저RBER 확보.
8. 구현: offline에서 P/E·retention별 성공 offset을 측정해 Read Index Table(21bit) 구축, online은 host의 P/E·retention으로 테이블 lookup 후 MSB 복호→IACPO 실행(Algorithm 2).
9. 결과: 실측(ZYNQ-7000 self-developed platform)에서 iteration latency IACPO 73.43%, ULDCO 81.51% 감소, EMAL 대비 약 20% 개선, 1400 P/E·270일에서 복호성공률 38.17%p 향상, 저장/연산 overhead 무시가능.
10. 한계: HW 디코더 미설계(알고리즘·테이블만), min-sum 자체는 미개선, TLC page 구조·특정 칩 특성 의존.
