### B. 알고리즘 요약

1. 시스템: WLAN/WiMAX/5G NR 다중표준 호환 QC-LDPC 디코더를 ASIP(SIMD) 방식으로 설계, 병렬도는 세 표준 lifting factor `Z`의 최대공약수인 8로 설정하고 `P=Z/8` SISO로 확장.
2. 문제: 기존 SOTA는 single-port 주파수배증 메모리로 최대 클록이 제약되고, layered 스케줄의 posterior 접근 충돌(RAW)로 throughput 저하.
3. 복호 알고리즘: Min-Sum 기반 `f(x,y)=sign(x)sign(y)min{|x|,|y|}`에 곱셈형 보정계수 `λ`(scaling factor=0.75)를 도입해 CN 갱신 진폭 과대 문제 완화.
4. 하이브리드 스케줄: 검사행렬을 `Z×N_Z` 레이어 M개로 분할, forward(위→아래) + backward(아래→위) 반복을 파이프라인 병렬 실행하며 VN/CN 갱신을 병합(열갱신을 행갱신에 포함)해 수렴 가속.
5. 핵심식: `LLR'_n = LLR_n - R_{m→n} + R'_{m→n} = LLR_n + Δ_{m→n}` — backward에서 forward 갱신값을 재사용해 1 iteration 내 다중 보정.
6. 충돌 제거: RowSwapping + SequenceReordering(공통원소를 읽기순서 우측에 배치)로 write-read 거리 `d_{j,θ}` 최대화, chip 동작 전 시뮬로 재정렬 테이블을 만들어 원 base matrix LUT를 대체(HW 수정 불필요).
7. 잔여 충돌은 하이브리드 스케줄+FIFO(8bit)로 이전 LLR을 잔차 계산해 해소.
8. 메모리: single-port 배증/dual-port 대신 RAMA/RAMB ping-pong 구조로 한 클록 내 동시 read/write 실현, 주파수/면적비 약 3~3.6배, 주파수/전력비 약 1.6~8배 개선.
9. 결과: 28nm SMIC 합성, core area 0.86mm², logic gate 1716K, memory 2420K, throughput WLAN 9.96Gbps / WiMAX 7.69Gbps / 5G NR 33Gbps, dynamic power <103mW, SOTA 대비 최대 4.5배 throughput.
10. 한계: 무선표준 특화 ASIP 아키텍처, 8bit 고정소수점 error floor ~1e-9 수준으로 정정능력 자체 개선은 아님(scaling MS 수준).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| forward-backward 하이브리드 스케줄 / 이터레이션 루프 | decoder.cpp LDPC_Decoder() | Prime의 Dual-Update(even/odd 교대) 스케줄과 유사 개념 — 파이프라인 병합 아이디어 참고 |
| Min-Sum + 보정계수 λ(=0.75) CN 갱신 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime도 min-sum 보유, scaling factor 명시 여부 확인용 참고(중복 가능) |
| 매트릭스 재정렬(RAW 충돌 제거) LUT 대체 | GT.cpp Make_GT_HW() / ecc_top.cpp Load_PCM() | HW edge/LUT 재배치 관점에서 Graph Thinning·PCM 로드에 간접 대응 |
| ping-pong 메모리 서브시스템 (RAMA/RAMB) | 대응 없음 | Prime 프로파일 모듈 지도에 메모리 뱅크 아키텍처 대응 항목 없음 |
| 8bit LLR 양자화 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | LLR 정밀도/테이블 관점 참고 |
```

적용 가치: **중간** — RAW 충돌 제거용 스케줄/메모리 기법은 Prime의 pipeline·스케줄러에 참고가치가 있으나, 정정능력 개선이 아니고 무선 다중표준 특화라 z=32 고정 NAND QC-LDPC로의 직접 이식은 재해석 부담이 있다.
