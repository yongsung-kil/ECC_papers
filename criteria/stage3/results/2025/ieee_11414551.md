### B. 알고리즘 요약 (10줄 내외)

1. 802.11n QC-LDPC(12×24 base matrix, mother rate 1/2, 확장·puncturing으로 길이 1024, rate 1/2·2/3·3/4·5/6)를 evaporation duct 해상 OTH 채널에서 복호.
2. 문제: 표준 BP의 flooding(동기·완전병렬) 스케줄이 시변 채널·radar 임펄스 잡음에서 "과신 오류(over-confident error)"를 Tanner graph 전체로 전파해 오수렴.
3. 채널 모델: log-normal(slow) + Rayleigh(fast) 복합 페이딩 + S-band radar 임펄스 잡음, 남중국해 실측으로 검증(KL divergence < 0.05).
4. 핵심 기법: Informed Dynamic Scheduling — residual `r_ij = |e_new - e_current|`(C2V 메시지 갱신 예상 변화량)이 가장 큰 edge를 우선 갱신하는 RBP.
5. residual이 클수록 새 정보가 많다는 원리로, argmax residual edge `(i*,j*)`만 갱신 후 인접 V2C 갱신·반복해 수렴 가속.
6. RBP의 greedy 문제(특정 trapping set 국소 반복 → 국소최적 고착, error-floor 악화)를 완화하기 위해 node-weighted/node-wise RBP(NWRBP) 도입.
7. NW-RBP: edge가 아닌 node 단위 스케줄 — residual 최대 check node를 찾아 그 CN의 모든 C2V를 한 번에 갱신, 메시지 전파 다양성↑로 trapping set 교란.
8. 정규화 가중(normalised weighting)으로 node 메시지 기여 균형 + dynamic residual adjustment로 채널 변동 추종.
9. 결과(시뮬): 강감쇠 Scenario 2에서 저SNR 1.8dB·고SNR 2.3dB 이득(고rate 5/6에서 특히 큼), BP 대비 avg iteration 35% 감소, Polar(SCL L=8) 대비 0.8dB·RS 대비 3dB 이득, 최저 error floor.
10. 한계: HW 미설계(시뮬만), iteration당 연산량 증가, 무선 evaporation duct 채널 특화, node 가중치 계산식 본문 부재.
