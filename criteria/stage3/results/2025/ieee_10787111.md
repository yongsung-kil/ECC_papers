### B. 알고리즘 요약

1. 5G NR LDPC 부호(BG1/BG2, base matrix + Z×Z lifting, Hcore/Hex/I 구조), rate 1/5·1/3·1/2·2/3, K=400, BPSK+AWGN 채널, irregular 부호(degree-1 VN 다수).
2. 문제: LLR-BPA는 비선형 연산이 많아 HW 부적합, MSA/NMSA/OMSA는 저복잡도지만 irregular 5G LDPC에서 LLR-BPA 대비 성능 격차(0.25~0.3dB@BER 1e-4)가 큼; 기존 2D-SCMSA도 잔여 격차(~0.2dB) 존재.
3. 채널모델: AWGN, VN 초기 LLR `L(pn)=log(Pr(yn|cn=0)/Pr(yn|cn=1))`.
4. 핵심 기법: 기존 2D-SCMSA(PSC: Hcore VN의 부호반전 메시지를 0으로 삭제 + Hex 메시지 진폭 β배 증폭)에 더해, CN 메시지 진폭 기반 보정값 `Lcv = λ·L(rlm)_n`을 도입해 erasure 판정 기준을 `sign(L̂(qln)m − s(l-1)·Lcv) = s(l-1)`로 동적화(식 11~12).
5. 핵심 식의 의미: 기존엔 부호반전 여부만으로 unreliable 메시지를 삭제했지만, 진폭이 0 근접(부호는 동일)한 메시지도 unreliable로 간주해 erasure 범위를 CN 메시지 크기에 비례해 넓히거나 좁힘.
6. 확장: 반복 전체를 두 구간(`l<Iset` vs 이후)으로 나눠 서로 다른 보정계수 λ1(큼)/λ2(작음, 거의 0)를 적용 — 초반엔 적극 삭제, 후반엔 신뢰 메시지 보존.
7. 부수 트릭: NMSA의 정규화 factor α를 함께 사용, 비선형 연산 없이(LUT 불필요) CN 업데이트는 기존 MSA/NMSA와 동일해 HW 친화성 유지.
8. 파라미터(Iset=15, α, β=1.1, λ1, λ2)는 code rate별로 시뮬레이션을 통해 경험적으로 결정(Rate 1/5: α=0.7125,λ1=0.18,λ2=0.02 등).
9. 결과: 2D-SCMSA 대비 약 0.1dB 성능 이득(BER 1e-4), LLR-BPA와의 격차 0.1dB 미만; Rate=1/5, Eb/N0=1.2dB에서 2D-SCMSA 대비 약 8 iteration 감소(decoding delay 약 640 clock 감소, `2×Z×I` clock 기준); 연산량은 2D-SCMSA 대비 비교연산 +16%, 덧셈 +26% 증가하나 전체 VNPU 중 Hcore용 26/14개만 해당(Ncore 대비 극소수).
10. 한계: 순수 시뮬레이션(floating-point) 기반이며 HW 합성/gatecount·실측 없음; 5G NR BG 구조(Hcore/Hex 분리, degree-1 VN 다수) 전제로 설계되어 다른 H-matrix 구조로는 파라미터 재조정이 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN 메시지 min-sum 연산(식 5, NMSA 기반) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 계열 min-sum CN 업데이트 구조로 직접 대응 |
| VN 메시지 self-correction/erasure(식 6~13) | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD) | VN 처리 단계에 erasure 로직 삽입 가능한 지점 |
| iteration 구간별 파라미터 전환(Iset 기준 λ1/λ2) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 iteration별 LLR threshold 테이블 구조에 유사하게 확장 가능 |
| Hcore/Hex 구조별 differential 처리(VNPU 일부만 보정) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 기반 col 그룹 구분 개념은 HCU(비대칭 degree col 처리)와 유사하나 5G BG 구조 전제라 재검증 필요 |

> 적용 가치: 높음 — min-sum 계열 VN erasure 보정 기법으로 코드 구조·연산 형태가 Prime ECC의 min-sum 디코더 루프와 거의 동일 계열이며, 무LUT·저복잡도 증가로 waterfall 개선과 iteration 감소를 동시에 달성해 이식 난이도가 낮음.
