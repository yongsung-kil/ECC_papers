### B. 알고리즘 요약

1. 시스템: LTE-A Pro/5G-NR 대상 Turbo 코드(BPSK+AWGN), 프레임 `K=128~256`, window `WS=16/32`, radix-4 trellis, 완전 pipeline iteration-unrolled(UXMAP) 디코더.
2. 문제: 초고속·저지연 Turbo 복호 시 BCJR/MLM의 재귀 특성이 병렬화를 방해하고, shuffled decoding은 불완전 외재정보로 성능저하·추가 반복 필요.
3. 모델: 두 component 디코더가 인터리버 π로 연결되어 외재정보 `e`를 교환(HI=half iteration, FI=full iteration), AWGN 채널.
4. 핵심기법1: max-Log-MAP 대체로 Local-SOVA(LSOVA) — trellis path `P={M,u,L}`를 merge 연산 `Mc=max(Ma,Mb)`, 신뢰도 `Lc`는 Hagenauer/Battail rule로 갱신, SOU 단계 merge가 memory length ν 트리로 축소.
5. 핵심식 의미: merge의 교환·결합성으로 임의 순서 병합 가능 → forward state metric 암묵계산, SOU 복잡도가 radix와 무관하게 ν층으로 감소.
6. 핵심기법2: Dual-Sided LSOVA — SOU의 동일 비교트리를 forward/backward path에 재사용해 두 radix-2^r step을 병렬 처리, SOU 인스턴스 수 절반.
7. 부수트릭: iteration-overlap ARP 인터리버 `ARP(i)=P·i+S(i mod Q) mod K`를 layer 수 `Q=WS`로 설계해 precedence 제약 `0≤C(i)-G(i)` 충족 → HI 간 즉시 외재정보 교환·완전 overlap, delay register 제거.
8. 최적화: 별도 학습 없음. 4 full iteration 고정, protograph 기반 인터리버 설계(puncture 제약 반영).
9. 결과: 22nm 합성, DS-LSOVA+overlap이 MLM+non-overlap 대비 약 22% 면적효율(Gb/s/mm2) 개선, FER은 MLM과 동등(WS32/K128에서만 약 0.25dB 저하).
10. 한계: Turbo(convolutional) 코드 전용, LDPC와 무관. throughput/gatecount 수치는 본문 서술에 없음(표에만), soft 비트폭 명시 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DS-LSOVA trellis 복호 알고리즘 | 대응 없음 | Prime ECC는 QC-LDPC min-sum 메시지패싱, Turbo BCJR/SOVA와 계열 상이 |
| iteration-overlap ARP 인터리버 | 대응 없음 | Turbo 전용 인터리버 스케줄, LDPC lifting/QC 구조와 무관 |
| UXMAP fully-pipelined 아키텍처 | 대응 없음 (개념적으로 pipeline은 우리 6-stage와 유사하나 trellis 전제) | LDPC 디코더 pipeline과 직접 대응 없음 |
```

적용 가치: **낮음** — Turbo 코드 trellis/SOVA 복호와 인터리버 설계로, NAND binary QC-LDPC min-sum 기반 Prime ECC에 이식할 요소가 없다.
