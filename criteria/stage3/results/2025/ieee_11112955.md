### B. 알고리즘 요약 (10줄 내외)

1. NAND SSD의 대역폭/발열(Dark NAND) 한계를 데이터 이동 축소로 풀기 위해 Processing-in-NAND(PiN) 아키텍처를 제안하는 vision 논문.
2. 풀려는 문제: PiN은 칩 내 로직/전력 예산이 극히 제한되어 HW ECC(LDPC) 탑재가 비현실적 → bit-flip을 소프트웨어가 감당해야 함.
3. 채널/에러 모델: 페이지 내 비트가 위치·타 비트와 독립적으로 flip. 비대칭 오류 `p0→1`, `p1→0`, 전체 `p↑↓`로 표기.
4. 핵심 기법: HW ECC 대신 확률적 자료구조(Bloom filter, binary sketch)를 PiN 위에서 오류내성 있게 사용(OTS-PiN의 search/AND/xor+popcnt로 평가).
5. 핵심 식: Bloom filter 최적 해시함수 수 `kopt = ln(tFP)/(ln(0.5)+ln(1−p1→0+p0→1))` — 양방향 flip이 상쇄되면 FPR 영향 없음.
6. 확장: false-negative 상한 `kmax = ln(1−tFN)/ln(1−p1→0)`, SLC의 단방향(1→0) 오류 특성을 이용해 정확도/용량 트레이드오프.
7. 구현 디테일: 4KiB 페이지=512×64bit word, SiM의 SEARCH로 512bit 결과만 전송해 채널 전력 절감. binary sketch는 popcnt+thresholding 회로 필요.
8. 최적화: 페이지별 실제 RBER에 맞춘 error-adaptive 파라미터화(worst-case 회피)로 용량 손실 최소화.
9. 결과: `p0→1`가 약 0.01~0.125 이하면 Bloom filter 저장비용에 사실상 영향 없음(이론/수치 분석, Fig.4/5). HW ECC 대비 소프트웨어 오류내성의 유연성 주장.
10. 한계: LDPC/디코더 알고리즘 기여 전무, 실측·HW 검증 없음, 확률적 자료구조 한정, popcnt 회로는 미완.
