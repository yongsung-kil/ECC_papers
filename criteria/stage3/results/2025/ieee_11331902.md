### B. 알고리즘 요약

1. DVB-S2 표준 기반 QC-LDPC, 코드율 1/4~5/6, 정보비트 길이 `Kldpc`=3240~13320, 시스템형 부호 `c=[u p]`.
2. 기존 FPGA 인코더는 전체 생성행렬 G 또는 대형 LUT를 직접 저장해 BRAM 자원 부담과 병렬성-유연성 충돌 문제가 있음.
3. 생성행렬 `G=[I | P]`를 `P = P_L(low-rank) + P_S(sparse residual)`로 분해한다는 것이 핵심 가정.
4. strictly sparse부는 block 단위 병렬 shift/XOR 네트워크로 직접 처리, residual부는 재구성 가능 small-core 모듈로 처리.
5. 핵심 식 `y = G·x = (G_strict + G_residual)·x` (식 8) — 두 부분을 분리 계산 후 합산해 residual 처리 자원만 최소화.
6. residual 누적은 저정밀 approximate adder로 근사(`y_residual ≈ approx_adder(Σ G_residual_i·x_i)`, 식 9)해 전력/면적 절감.
7. parity 주소 계산을 `addr_n = b_m + xQ_m + mQ_m mod M` (식 11)로 분해해 곱셈·모듈로 연산을 사전계산 ROM 테이블 lookup과 조건부 뺄셈으로 대체.
8. 별도의 DE/GA/NN 학습 절차는 없음 — 구조적(algebraic) 분해와 파라미터(shift_index, enable_mask) 설정이 전부.
9. 결과: 제안 설계 2108 LUT/1219 FF/7 BRAM (0.96%/0.28%/1.28% 활용률) vs 참조설계 3665 LUT/2630 FF/4.5 BRAM(2.72%/0.98%/1.23%); Eb/N0 4.5dB에서 BER 약 1×10⁻²로 약 2.7배 개선(단, iteration 10회·1bit LLR 근사 vs 참조 50회·고정밀).
10. 한계: BER 비교가 반복횟수·정밀도가 다른 두 설계 간 비대칭 비교이며, throughput(Gbps) 수치 미제시, approximate adder의 오차 한계·low-rank 랭크 선택 기준이 본문에 구체적으로 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 생성행렬 G의 sparse+residual 분해 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 고정 인코딩 구조라 low-rank/residual 분해 방식 자체를 이식하려면 부호 PCM 재설계 필요 |
| parity 주소 재구성 가능 매핑(식 10~12) | `encoder.cpp` `Column_Reordering_Dual_Update()` | 주소 lookup 최적화 아이디어는 참고 가능하나 코드 구조가 상이해 직접 이식 어려움 |
| approximate adder 저정밀 누적 | 대응 없음 | Prime ECC는 encoder를 bit-exact로 시뮬레이션하며 approximate 연산 개념 없음 |
| 다중 code rate 재구성(1/4~5/6) 지원 | 대응 없음 | Prime ECC는 고정 QC-LDPC 구조(matrix_sel 종속)로 런타임 rate 전환 개념 없음 |

> 적용 가치: 낮음 — encoder 전용 FPGA 자원 절감 기법이고 DVB-S2 표준 부호 구조에 특화되어 있어, Prime ECC의 dual-diagonal 인코더·binary QC-LDPC 디코더 이식 관점에서 직접 재사용 가치가 낮음.
