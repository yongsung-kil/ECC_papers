> 총평: 암호용 (v,w)-regular 부호의 2-iteration hard-decision bit-flipping decoder에 대해 syndrome weight 분포를 이용한 이론적 최적 threshold 산출 기법으로, NAND QC-LDPC의 min-sum soft 디코더에는 직접 적용이 어렵지만 hard-decision 조기 threshold 설계 아이디어로는 참고 가치가 있음.

### B. 알고리즘 요약

1. 대상은 `(v,w)-regular` 이진 부호(LDPC/MDPC 포함)의 병렬 hard-decision bit-flipping decoder(Algorithm 1, Gallager 방식)이며, 부호율 `k/n = 1 - v/w`.
2. 기존 threshold 선택은 iteration 번호에만 의존(고정값)했으나, syndrome 상태를 반영하지 않아 DFR(decoding failure rate) 추정이 부정확했던 문제를 해결하고자 함.
3. 채널/에러 모델은 초기 오류벡터 `e`(weight `t`)로부터 syndrome `s = He^T`를 계산하는 표준 이진 syndrome decoding 모델.
4. 1단계: 1차 반복 후 syndrome weight `wt(s^(1))`의 분포를 Markov process로 모델링하여 `Pr(W_{ϵ01,ϵ11}^(1) = z)`의 closed-form p.m.f.를 유도.
5. 이 분포로부터 각 parity-check 카테고리(`J_{a,b}`)별 flip 확률(`pflip|ab`)을 구하고, 2차 반복 후 DFR을 `y(초기 syndrome weight), ϵ01, ϵ11, z0, z1`의 함수로 closed-form 유도.
6. 2단계: threshold `th^(1)(y)`(1차, syndrome weight 함수)와 `th^(2)(y,z0,z1)`(2차, 1차 후 satisfied→unsatisfied/unsatisfied 유지 카운트 함수)를 DFR 최소화 방향으로 순차 최적화(joint 최적화는 계산량 `r^3nt`로 과다하여 분리).
7. 부수 트릭: threshold 최적화를 1차/2차로 분리해 암호급 대형 파라미터(n~10^4)에서도 워크스테이션에서 계산 가능하게 함.
8. 최적화 절차: 코드 파라미터 기반 수치적 arg min 탐색(그리드 형태로 각 y, (z0,z1) 조합별 최적 threshold 테이블 산출).
9. 결과: 제안 threshold가 majority-threshold 대비 DFR을 약 `10^3배` 개선, BIKE 대비 leftover discrepancy를 약 `3×10^2배`, majority 대비 `>5×10^3배` 감소.
10. 한계: hard-decision 2-iteration 병렬 디코더 및 암호용 (v,w)-regular 부호(BIKE/LEDAcrypt)에 특화된 이론 모델이며 HW 구현·시뮬레이션 검증 없음, NAND용 다중 iteration soft-decision LDPC로의 직접 확장은 논문 범위 밖.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping threshold 선택(syndrome weight 기반) | `channel.cpp` `Set_LLR_Th()` | hard-decision threshold 개념이 LLR threshold 테이블과 유사하나 Prime ECC는 soft min-sum 기반이라 직접 대응 어려움 |
| 2-iteration 병렬 디코더 구조 | `decoder.cpp` `LDPC_Decoder()` | 병렬 반복 구조 자체는 대응되나 hard-1bit 전용이라 이식 난이도 높음 |
| DFR 이론 모델(closed-form) | 대응 없음 | Prime ECC는 시뮬레이션 기반 통계 산출이며 이런 해석적 DFR 모델 사용 안 함 |

> 적용 가치: 낮음 — 암호용 (v,w)-regular MDPC/LDPC의 hard-decision 2-iteration decoder 전용 이론이며, Prime ECC의 soft min-sum QC-LDPC 구조와 목적(암호 vs NAND 저장)이 달라 직접 이식 여지가 작음.
