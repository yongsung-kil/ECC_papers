### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 위성-지상 uplink SCMA(Sparse Code Multiple Access) 시스템, J=6 사용자가 K=4 resource element를 공유, 사용자 간 시간지연(delay) τj로 인해 비동기(asynchronous) 수신 발생.
2. 문제: 기존 SCMA 검출/복호 연구는 대부분 사용자 간 완전 동기화를 가정하며, 비동기 SCMA 전용 검출 알고리즘은 성능이 동기식보다 열등하고 soft-output decoder와 결합되지 않음.
3. 채널 모델: Rayleigh fading 채널, 사용자별 시간지연 τj를 갖는 비동기 수신 신호를 확장 factor graph로 모델링.
4. 핵심 기법 1: 병렬 EPA 검출기(P-EPA) - 확장 factor graph 상에서 VN/FN 메시지를 Gaussian 분포(평균 `μ`, 분산 `ν`)로 근사하여 병렬 갱신.
5. 핵심 식: 사전정보를 활용한 새로운 초기화 방법(`μ_j,→k^(0)`, Eq.7~8)으로 기존의 상수 초기화 대비 성능 개선.
6. 핵심 기법 2: OLD(Ordered Likelihood Decoder) 기반 soft-output 디코더 S-OLD - TEP(test error pattern) 기반 weighted Hamming distance로 extrinsic LLR(Eq.19) 근사 계산.
7. 부수 기법: AIDD(Asynchronization-aided Iterative Detection and Decoding) 스킴에서 P-EPA와 S-OLD를 외부 루프로 반복 교환, 각 사용자 복호결과 수렴 또는 최대 반복수 도달 시 종료.
8. 학습/최적화 절차 없음(순수 알고리즘/시뮬레이션 논문).
9. 결과: AIDD가 개별 스킴/동기 IDD/JDD-MPA/JDD-EPA 대비 BER=10⁻⁴에서 각각 1.6/3.4/3.6/6.2 dB 이득, P-EPA는 기존 EPA 대비 검출시간 약 40~65배 감소(0.0854s→0.0019s).
10. 한계: HW 미설계, MATLAB 시뮬레이션만 존재, LDPC C(128,64)는 실험에 쓰인 기존 표준 short code일 뿐 논문이 개선한 대상이 아님.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| P-EPA (SCMA 다중접속 검출기) | 대응 없음 | SCMA/다중접속 검출은 Prime ECC의 LDPC 복호와 무관한 상위 계층 신호처리 |
| S-OLD (OLD 기반 soft-output 디코더) | 대응 없음 | LDPC min-sum이 아닌 OSD/TEP 기반 short-block code 디코더로 구조가 상이 |
| LDPC C(128,64) 사용 | 대응 없음 | 실험용 기존 표준 코드로만 언급, 부호설계나 디코더 개선 내용 없음 |
```

적용 가치: `낮음` — SCMA 다중접속 무선통신 논문으로 LDPC ECC 부호설계·디코더 자체에 기여하는 내용이 없어 Prime ECC 이식 관점과 무관.
