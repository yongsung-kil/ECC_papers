### B. 알고리즘 요약

1. 대상은 GC-LDPC(globally coupled LDPC) 코드로, local LDPC 코드 t개가 대각으로 배치되고 global parity check 행렬로 연결된 구조(`H_GC`, `M=mt+s`, `N=nt`).
2. 기존 NMS는 고정 정규화 인자 λ를 사용해 min-sum의 CN 메시지 과대추정을 보정하지만, 고정값이라 성능 개선에 한계가 있음.
3. 채널은 AWGN과 flash memory 채널(Nanocycler 테스트 플랫폼, P/E cycle과 retention time(T)로 RBER 결정)을 모두 사용.
4. 핵심 기법: 정규화 인자 ρ를 반복마다 동적 조정. 초기값 `ρ=0.7`, 2번째 iteration에서 `ρ = 0.9·LLR0/LLR1`, 이후 `ρ = 0.8·LLRr-1/LLRr`.
5. `LLRr = E{LLRj(r) | LLRj(r) > 0}`는 양의 LLR 값만의 평균으로, LLR 극성화(polarization) 정도를 반영해 반복이 진행될수록 정규화 강도를 조정.
6. Layered schedule 기반으로 VN/CN/사후확률을 (7)-(9)식으로 갱신하며, 병렬 local code 처리 시 비갱신/갱신 LLR 정보 중 무엇을 쓰느냐에 따라 ANMS-1, ANMS-2 두 구현으로 분기.
7. 두 GC-LDPC 코드([15], [16] 참조 코드)에 대해 AWGN·flash memory 채널에서 NMS, 개선판 IN-MS와 비교.
8. 별도 학습/최적화 절차 없음 (수동 상수 0.7/0.9/0.8 설정).
9. AWGN에서 Eb/N0=4.6dB 시 BER 1.5×10⁻⁷ 도달에 평균 iteration 11(NMS 14, IN-MS 15); flash 채널에서 NMS·IN-MS 대비 iteration 1.2~10%(1레벨), 22~50%(2레벨) 감소.
10. HW 설계 없이 순수 알고리즘/시뮬레이션 논문이며, GC-LDPC라는 특정 코드 구조와 layered schedule에 의존적이라 범용 QC-LDPC 이식 시 검증 필요.
