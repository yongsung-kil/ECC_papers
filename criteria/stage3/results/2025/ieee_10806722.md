### B. 알고리즘 요약 (10줄 내외)

1. LDPC 복호를 XOR-SAT 문제로 재정의: 패리티 클로즈 `Ci = ⊕ uj`를 만족시키는 코드워드 탐색 문제로 봄.
2. 기존 GDBF(Gradient Descent Bit Flipping) 계열은 SPA 대비 성능 격차가 크고, MP 기반 기존 연구는 differential 회로 복잡도가 커 확장성이 부족함을 문제로 지적.
3. Boolean 변수 `uj`를 연속변수 `xj∈[-1,1]`로 매핑, `qj=log|xj|`를 도입해 log-sum-exponential 함수 `Hlog(q,H)`로 XOR-SAT를 최적화 문제화.
4. 채널 상관항(extrinsic factor) `Σ rj dj qj`를 더한 `HLDPC(q,r,d,H)=Hlog(q,H)+Σrjdjqj`를 gradient ascent로 최대화(식 11~13).
5. gradient 내 multinomial logistic 확률항을 Margin Propagation(MP) 확률점수 `Pi=τ[yi-ζ]+`(ReLU+뺄셈만, 곱셈/지수연산 불필요)로 근사(식 15~20).
6. 스케일링 항 `Aj`(양쪽 clause 집합에서 유의미한 횟수)를 도입해 extrinsic factor 가중치를 휴리스틱하게 조정.
7. 매 iteration마다 `qj<θ`(threshold)이면 비트 `dj`를 플립, multi-bit flip이 single-bit flip보다 빠르게 수렴함을 실험으로 확인.
8. 하이퍼파라미터(τ=클로즈 수 M, θ≈0, η=0.01~0.001)는 grid search로 튜닝.
9. BCH(63,45)/(127,106)/(255,215) 등에서 SPA 대비 0.1dB 이내, GDBF 계열 대비 10배 적은 iteration; 복잡도는 O(MlogM)로 SPA(O(ρ²M+γ²N))보다 낮음.
10. HW는 analog/mixed-signal(2-트랜지스터 MP 블록) 구현을 미래 과제로만 언급, 디지털 HW/FPGA/ASIC 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MP-XOR-SAT 비트플리핑 디코딩 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()` | min-sum 대체 알고리즘으로 이터레이션 루프 구조에 대응, 곱셈기 제거 아이디어는 참고 가능 |
| CN 만족/불만족 분류(P+/P-), gradient 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 유사 위치이나 min-sum 연산과 근본적으로 다른 비트플리핑 갱신식 |
| 초기 soft 정보 qj 산출, 양자화(fixed-point 실험) | `channel.cpp` + `PARAM_LLR` | soft-read LLR 양자화 아이디어와 부분 대응 |
| 곱셈기 없는 analog/MP 회로 HW | 대응 없음 | Prime ECC는 디지털 bit-exact HW 시뮬레이터로 analog 회로와 직접 대응 없음 |
```

> 적용 가치: 낮음 - 곱셈기 제거라는 복잡도 절감 아이디어는 흥미로우나 HW 미설계·analog 지향이며 이미 보유한 min-sum/HCU 구조와 이질적이라 디지털 NAND ECC HW로의 직접 이식은 어렵다.
