### B. 알고리즘 요약

1. (127,106) BCH(HDPC), MDPC(127,106), MDPC(127,92) 3개 코드, BPSK 변조 후 AWGN 채널에서 LLR 기반 반복복호(rate 21/127≈0.83, 35/127≈0.28 등 코드별 상이).
2. 문제의식: HDPC(BCH)는 Tanner 그래프의 short cycle이 많아 표준 sum-product(SPA/BP) 성능이 낮고, LDPC/MDPC는 상대적으로 양호하나 개선 여지가 있음.
3. 채널 모델: `l_v = 2y_v/σ²` (AWGN LLR), 메시지는 variable-to-check `μ_{v,c}^i = l_v + Σ μ_{c',v}^{i-1}`, check-to-variable는 표준 tanh 곱(식 4).
4. 핵심 기법(neural BP): Tanner 그래프를 트렐리스로 펼쳐 각 반복을 2개 신경망 레이어(check layer, variable layer)로 매핑, 메시지에 학습 가중치 `w_{v,in}^i, w_{v,c,c'}^i, w_{c',v,out}^i`를 곱하는 neural SPA(식 6~8).
5. 식의 의미: 가중치가 모두 1이면 표준 SPA와 동일(성능 하한 보장), 학습을 통해 short cycle의 부정확한 상관 메시지 영향을 감쇠시켜 trapping set 회피.
6. 확장(2단): min-sum 근사에도 동일 아이디어 적용 — neural NMS(식 11, 가중치 `w_{c,v}^i`로 min-sum 크기 스케일)와 neural OMS(식 13, 학습된 offset `β_{c,v}^i`로 min-sum 값 차감, `max(·-β,0)`).
7. 구현 디테일: 전 반복 대칭성(symmetry condition) 이용해 학습·테스트 모두 all-zero 코드워드 1개만으로 충분, sigmoid 출력 + binary cross-entropy loss(식 9), multiloss(매 반복마다 손실 계산)로 하위 레이어 학습 촉진.
8. 학습/최적화: TensorFlow + Adam, mini-batch=120(Eb/N0 1~8dB 균등 분포), incremental learning(10000 mini-batch씩 7 epoch로 분할, overfitting 전 조기 중단).
9. 결과: neural SPA(5 iter)가 BCH(127,106)에서 standard SPA(50 iter) 대비 우수(복잡도 감소 10배 이상 추정), MDPC(127,106)는 이미 standard SPA 성능이 좋아 neural 이득 적음; MDPC(127,92) 실험에서 30000 mini-batch가 최적, 이후 overfitting.
10. 한계: 코드 길이 127로 극히 짧고 3종뿐, HW 설계·복잡도/throughput 정량화 없음(순수 알고리즘 시뮬), neural 학습을 실제 미보정 에러(uncorrectable) 데이터셋으로 하는 것은 future work로 남김.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Neural offset min-sum(학습된 offset `β_{c,v}`로 CNU 메시지 차감) | `decoder.cpp` `CNU_Update_New_Mag()` | Prime ECC의 min-sum CNU 크기 연산과 구조적으로 가장 유사 — offset을 iteration/edge별 학습된 상수 테이블로 대체하는 방식은 기존 `PARAM_LLR` 테이블 튜닝과 개념적으로 연결 가능 |
| Neural normalized min-sum(스케일 가중치) | `decoder.cpp` `CNU_Update_New_Mag()`, `ecc_data.h` `PARAM_LLR` | scaling factor 개념은 Prime ECC의 magnitude 양자화 테이블과 유사하나, per-edge/per-iteration 학습 가중치는 코드 고정 테이블 구조 확장 필요 |
| Neural SPA(전체 메시지에 가중치, full-tanh 기반) | 대응 없음 | Prime ECC는 min-sum 기반이며 full-tanh Sum-Product 자체는 프로파일상 "대응 없음" 명시 항목 |
| 트렐리스 신경망 학습(오프라인, TensorFlow) | `local_opt.cpp` | 목적(파라미터 자동 튜닝)은 유사하나 신경망 학습 프레임워크 자체는 대응 없음, LLR 자동최적화 모듈에 개념적 참고만 가능 |

적용 가치: 중간 — neural OMS의 "학습된 offset/scale로 min-sum CNU 메시지 보정" 아이디어는 Prime ECC의 기존 min-sum + LLR 테이블 구조와 접점이 있어 참고 가치가 있으나, 검증이 극단적으로 짧은 코드(n=127)·순수 시뮬레이션에 국한되어 NAND급 QC-LDPC(수천~수만 길이) 및 HW 이식 가능성은 별도 검증 필요.
