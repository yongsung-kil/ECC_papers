### B. 알고리즘 요약

1. 3D TLC NAND(62-layer, page 16kB, 코드 rate 8/9, codeword length 18364bit, min-sum LDPC)를 대상으로, 장기 데이터 보존(retention) 중 발생하는 Vth 분포 좌측 drift를 온라인으로 추정.
2. 기존 Vopt 트래커/soft-read 설계는 배치(batch) 단위 P/E-사이클 매핑 테이블에 의존해, 블록 간 공정편차(IBPV)에 취약하고 최적 파라미터 추정이 부정확.
3. 핵심 가정(Characteristic 1,2): 동일 wordline·동일 state의 셀들은 retention 중 Vth drift가 균일하며, 분포 평균 `μ`는 retention 시간에 로그적으로 비례(`μ = k·log(t) + b`).
4. probe wordline 설계: 각 층 중앙의 한 wordline만 선택해 state window별 셀 카운트 증가량 `N_i(t)`로부터 순차적으로 각 state의 평균 drift `μ_i`를 역산(식 19, 21, 23) — 단 1회 read로 전체 Vth 분포 drift 산출.
5. 식 (27): 임의의 두 layer m,n에서 평균 drift 비율 `μ_n^j(t3,t2) / μ_i^m(t3,t2) = k_i^m / k_j^n`이 retention 길이에 무관하게 상수 — 이를 이용해 probe wordline 측정값으로 다른 모든 층의 drift를 추정(cross-layer estimation).
6. 노이즈 억제를 위해 7개 state의 추정치를 가중평균(식 31,32)해 RTN 등 랜덤 오차를 완화.
7. 부수 트릭: state S0-S1 경계는 read margin이 넓어 측정 생략, 메타데이터(진폭/평균/분산/drift/probe wordline index)는 spare SLC 영역에 저장.
8. 별도 학습절차 없음(폐형식 기반 추정, DE/GA/NN 미사용) — Algorithm 1(메타데이터 획득)/Algorithm 2(플래시 read 적용)로 구성.
9. 결과: 실제 Micron B16A 칩 실측 기준, 샘플링 오버헤드는 layer read 0.08회(기존 RFBC 0.17, DRRM 0.5 대비 50%+ 감소); LDPC hard decision FER 19%(DRRM 43%, RFBC 61% 대비 대폭 개선); soft decision 적용 시(PreLDPC/EBDN) 총 read latency 최대 45% 감소.
10. 한계: LDPC 디코딩 알고리즘/부호 자체는 전혀 수정하지 않는 순수 memory-sensing 전략(정정능력 자체 향상 아님), retention loss(좌측 drift) 중심 모델이며 우측 drift·기타 노이즈원에는 메타데이터 주기적 갱신이 별도 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Vth 분포 기반 read level/offset/LLR 산출 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read LLR 양자화 threshold 설계 개선에 직접 연결 가능한 memory-sensing 레이어 |
| 초기 read level 예측(진행적 디코딩 시도 감소) | `decoder.cpp` `LDPC_Decoder()` (반복 트리거 조건) | 초기 read level 정확도 향상 시 불필요한 재시도(추가 read retry) 감소 효과, 디코더 로직 자체는 무관 |
| 층별(layer) Vopt 캘리브레이션 | 대응 없음 | Prime ECC는 HW 클록 레벨 디코더 시뮬레이터로 NAND 셀 물리/층 구조 모델을 포함하지 않음 |
| H-matrix/부호구조 | `ecc_top.cpp` `Load_PCM()` | 논문은 부호구조 자체를 다루지 않음(대상=other), 참고용 명시일 뿐 관련성 낮음 |

적용 가치: `중간` — 디코더 알고리즘이 아닌 NAND soft-read/LLR 산출 파라미터 결정 레이어의 기법으로, Prime ECC의 channel.cpp LLR threshold 설계에 아이디어 차용 가능하나 직접적인 코드 이식보다는 개념적 참고(온라인 Vth drift 추정) 수준.
