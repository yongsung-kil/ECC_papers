### B. 알고리즘 요약

1. 대상 채널은 레이저(자유공간 광) 통신이며, 부호는 범용 QC-LDPC(정확한 rate/length 본문 미기재).
2. 기존 sum-product/belief propagation 디코더는 연산량이 커 고throughput 요구를 만족 못한다는 문제의식.
3. 채널은 대기 감쇠·자유공간 손실·굴절률 난류에 의한 페이딩으로 가정.
4. 핵심 디코딩은 표준 min-sum 근사(`(2)`,`(3)`): C2V 메시지 `n_{ap→sq} = λ·min|K|·sign product`, V2C 메시지 `K_{sq→ap} = Σn + D_sq`.
5. 두 프레임(1st/2nd frame)을 하나의 RAM 블록에 동시 저장·처리하는 Inter-Frame Pipeline(IFPP)으로 VNU/CNU idle time을 줄여 리소스 효율을 높임.
6. UAMP(Unitary AMP, 식 `e=Ψa+δ`)를 메시지 패싱 관리에 적용해 프레임 간 정보 이동을 매끄럽게 근사.
7. VPPSOA(속도 정지 PSO, 식 `(7)`,`(8)`)로 디코더의 시스템 레벨 파라미터(스케줄, 리소스 배분)를 최적화 — 표준 PSO에 "velocity pausing"(일정 확률로 속도 유지) 항 추가.
8. 부수 HW 트릭: 메시지 패킹(파라미터 P)으로 RAMM 한 워드에 여러 메시지 압축, 데이터 정렬 유닛(오프셋 사전계산)으로 메모리 충돌 회피, RAMC 뱅크 수 n→1로 축소.
9. Xilinx XCKU060 FPGA 구현 결과 기존 대비 decoding throughput 12~19%, hardware complexity 16~27%, BER 18~27% 개선(반복횟수 100/300/500 조건별).
10. 한계: gatecount/throughput 절대수치·FPGA 주파수(260MHz) 외 상세 HW 자원표는 본문에 없고 그림에만 존재, 정정능력(에러플로어/waterfall) 자체 개선이 아닌 리소스·속도 최적화에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum C2V/V2C 반복 디코딩 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이미 보유한 표준 min-sum과 동일 알고리즘, 신규성 없음 |
| Inter-Frame Pipeline(2-frame 인터리브) | `decoder.cpp` pipeline (col_M1/Z/P1~P4) | 우리 6-stage pipeline과 별개의 프레임 인터리빙 방식, 구조 상이해 이식 난이도 중 |
| VPPSOA(메타휴리스틱 파라미터 최적화) | `local_opt.cpp` (LLR 자동최적화) | 목적(파라미터 튜닝)만 유사, 기법(PSO) 자체는 대응 없음 |
| 메시지 패킹/데이터 정렬 유닛 | `ecc_data.h` PARAM_DEC (메모리 폭 정의) | 개념적 유사도만 있고 구체 대응 없음 |
| UAMP 메시지 패싱 | 대응 없음 | non-min-sum 근사 기법, Prime ECC는 min-sum 고정 |

적용 가치: **낮음** — 레이저 통신 채널 타깃 논문으로 NAND 관련성이 낮고, 디코더 알고리즘은 표준 min-sum 그대로이며 개선은 PSO 기반 HW 스케줄링/리소스 최적화에 그쳐 정정성능(waterfall/error-floor) 이득이 없음.
