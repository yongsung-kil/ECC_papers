### B. 알고리즘 요약

1. 5G/위성/심우주 통신에서 쓰이는 LDPC 코드의 전통적 BP 계열 디코딩(BP, LLR-BP, min-sum, normalized/bias min-sum)을 배경으로, 딥러닝 기반 디코딩을 개관.
2. 풀려는 문제: 전통 BP 디코더는 긴 코드 길이·고잡음 환경에서 복잡도가 높고 수렴이 느림.
3. 채널 모델은 AWGN, 코드 길이 16/648 및 H(8,16), H(648,108) parity check matrix 예제 사용.
4. 데이터 기반(data-driven) 딥러닝 디코딩: LLR 수신값을 입력, 원본 정보비트를 라벨로 하여 CNN/ResNet/KAN 네트워크가 코드워드-원문 매핑을 직접 학습, BP의 CN-VN 메시지 전달 구조를 명시적으로 사용하지 않음.
5. KAN 네트워크는 노드에 비선형 활성함수 없이 엣지(가중치)에 학습 가능한 단변수 B-spline 함수를 배치하는 구조; Table II에 KAN-ResNet-1~5, KAN-6 등 네트워크 구조 비교.
6. 모델 기반(model-driven) 디코딩: SNNMS는 전통 NMS(normalized min-sum) 알고리즘의 반복 구조를 신경망 레이어로 1:1 전개(unfold)해 보정계수 등 파라미터에 물리적 의미를 부여, 해석가능성 확보.
7. 학습 절차(Table I): 배치 단위로 은닉층 LLR 메시지(`L^l_{i→j}`, `L^l_{v→j}`)와 특징 `f_a=F(r_a)`를 계산 후 MSE 손실 `L_MSE`로 역전파, 파라미터 갱신 `θ←θ-η∇L_MSE`.
8. 손실함수는 cross-entropy 또는 MSE, 하이퍼파라미터(학습률/배치크기)는 그리드/랜덤/베이지안 탐색으로 결정.
9. 결과: Eb/N0 1~10dB(1dB step) AWGN에서 KAN 계열 네트워크 및 SNNMS와 전통 디코더의 BER/FER 비교를 Fig.5, Fig.6으로 제시하나 구체 수치는 본문에 기재되지 않고 그림에만 존재.
10. 한계: 모델 기반 방법은 메모리 제약으로 매우 긴 LDPC 코드에 적용 불가(저자 명시), HW 설계 없음, 순수 시뮬레이션·서베이 성격.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNN/ResNet/KAN 데이터 기반 디코더 | 대응 없음 | NN 전면 대체 계열은 프로파일상 대응 없음으로 분류 |
| SNNMS(모델 기반, NMS 언폴딩) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | NMS 반복 구조 자체는 min-sum과 유사하나 신경망 학습형 보정계수는 이식 난이도 높음 |
| 코드 구조/PCM 예제 | 대응 없음 | 임의 예제 행렬로 부호설계 관점의 기여 없음 |

적용 가치: 낮음 — 신규 알고리즘 제안 없는 서베이이며 NN 기반 디코더는 Prime ECC의 bit-exact HW 시뮬레이터 구조와 접점이 없음.
