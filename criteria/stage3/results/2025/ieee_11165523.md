### B. 알고리즘 요약

1. 5G NR eMBB용 QC-LDPC(BG1/BG2, rate-compatible/length-scalable)를 기반으로, HRLLC(초신뢰·저지연, latency<1ms, reliability>1-10^-5) 요구를 만족하는 "5G-NR-Like" 새 부호를 설계.
2. 기존 5G-NR LDPC는 BLER<10^-5에서 error-floor가 나타나 HRLLC 요건을 못 채우는 문제; base matrix의 최소거리 상한이 제한적인 것이 원인.
3. 채널모델은 BI-AWGN, layered BP 디코딩(최대 iteration `I=25`), MET-DE로 점근 임계값 분석.
4. 핵심 기법 1: base matrix `B`를 `BH`(최고rate부)+`BU`(incremental redundancy부)로 구성하고, progressive row extension으로 각 `(K,M)`마다 점근 디코딩 임계값 `dSNR_{K,M} < δ_K`를 유지하며 최소거리 상한 `d_{K,M} ≥ d_{K,G}+ΔK(M-G)`를 점증적으로 키움(δK=0.65/0.60/0.55dB, ΔK=4.86/3.66/3.5 for K=6,8,10).
5. 이 제약으로 새 base matrix는 원 5G-NR BG2 대비 SNR loss 최대 0.15dB만 감수하고 최소거리 상한을 1.5배 이상 개선 — waterfall 성능은 거의 유지하며 error-floor를 크게 낮추는 효과.
6. 핵심 기법 2: lifting 설계는 3D-G-PEGL(3D-grid 기반 progressive edge growth+lifting), cycle/최소거리/BLER 기준으로 후보를 순차 pruning(CyclePruning, MinDistPruning, DiversityPruning, BLERPruning)해 exponent matrix `P`를 구성.
7. 부수 기법: improved saturated BP 디코딩 — 미천공(not-punctured) `n1`개 비트 중 최대 `p`개 조합에 LLR을 `±LLRmax`로 saturate시키는 임펄스 패턴을 추가해 BP 반복을 `C=Σ2^q·C(n1,q)`회 수행, LLR ordering 단계를 생략해 SMS(saturated min-sum) 대비 연산 절감.
8. 최적화 절차: base matrix 설계는 Algorithm 1(그리드 서치+제약 조정), lifting 설계는 Algorithm 2(3D-grid 성장+가지치기)로, 실전 구성에 최대 10일 소요.
9. 결과: k=384에서 hybrid list decoding이 near-ML 성능 근접, k=960에서 BP 대비 크게 개선; BLER<10^-5에서 error-floor 소거(원 5G-NR은 truncated union bound에 막힘).
10. 한계: BI-AWGN 시뮬레이션뿐이며 HW 구현/합성/실측 없음. lifting 설계에 최대 10일 소요되는 고비용 오프라인 구성 과정이며 특정 (K,M,Z) 조합군에 한정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| improved saturated BP hybrid list decoding (LLR impulse post-processing) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 조기 수렴 실패 시 저비용 LLR saturation 재시도로 error-floor 개선 가능 — 이식 난이도 낮음 |
| base matrix 최소거리 최적화(progressive row extension) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체 재설계라 Prime ECC 고정 QC-LDPC 구조와 재검증 부담 큰 "높음" 난이도 |
| 3D-G-PEGL lifting 설계 | `ecc_top.cpp` `Load_PCM()` | 특정 lifting factor 집합(Z=3×2^(γ-1))에 종속된 부호 구성법으로 Prime ECC z_sb=32 고정 구조와 직접 대응 어려움 |
| BI-AWGN 채널/soft LLR 가정 | `channel.cpp` `Set_LLR_Th()` | soft-4bit+ 가정이라 Prime ECC의 2SD/3SD 대비 정밀도 높음 — NAND read 비용 고려 필요 |
```

> 적용 가치: 중간 — 부호설계(base matrix/lifting) 자체는 이식 난이도가 높아 현실적이지 않지만, improved saturated BP의 LLR impulse 기반 후처리 아이디어는 decoder.cpp의 이터레이션 루프에 상대적으로 낮은 비용으로 추가해 error-floor 개선을 시도할 가치가 있음.
