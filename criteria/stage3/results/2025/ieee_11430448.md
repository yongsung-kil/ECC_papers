### B. 알고리즘 요약

1. (N,K)=(2048,1024)/(8192,4096)/(32768,16384) CCSDS 표준 LDPC, rate=1/2, AWGN+BPSK 채널.
2. 기존 BP는 고복잡도, MSA/NMS/OMS는 경험적 보정계수로 성능 손실 있어 check node update의 정확도-복잡도 트레이드오프 개선이 목표.
3. Box-Plus 연산 `L1⊞L2`으로 check node 갱신식의 비선형 항을 `min` + 오차함수 `Δ=ln[(1+exp(-|x+y|))/(1+exp(-|x-y|))]`로 분해.
4. Δ를 `Z=-min(|x|,|y|)/(1+max(|x|,|y|))`로 근사(핵심 변수: `x,y`=incoming LLR magnitude), 지수/로그 연산 제거.
5. 스케일 계수 `α`를 도입한 `Z'=-α·min(|x|,|y|)/(1+max(|x|,|y|))`로 MSE 최소화(α=1일 때 MSE=0.0018).
6. 최종 check node update식: `rnm=∏sgn(qm'n)·[μ·min|qm'n| + min(|x|,|y|)/(1+max(|x|,|y|))]`, μ=0.75(경험값).
7. μ와 α는 Monte Carlo 시뮬레이션과 density evolution으로 탐색, 코드rate/length별로 값이 달라질 수 있음.
8. 결과: BER 1E-05 기준 NMS/OMS 대비 약 0.4dB 이득, BP 대비 0.1dB 이내로 근접.
9. 연산수는 BP 18회 대비 ABP 11회(NMS 8회, OMS 9회)로, NMS/OMS보다 연산은 소폭 많지만 성능 이득이 더 큼.
10. 한계: HW 설계·합성 없이 알고리즘/시뮬레이션 수준에 그침, NAND 채널이 아닌 AWGN 우주통신 표준 코드로만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check node update (Box-Plus 근사, ABP) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산 정확도 개선 함수로 직접 대체 가능한 위치 |
| 전체 이터레이션 루프 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | ABP 알고리즘을 통합할 최상위 루프 |
| μ 스케일 계수 최적화 | `local_opt.cpp` (LLR 자동최적화) | μ/α 탐색 절차가 기존 LLR 최적화 모듈과 유사한 역할 |

> 적용 가치: 중간 — check node update의 min-sum 근사 정확도를 개선하는 기법으로 `decoder.cpp`의 CN 연산부에 이식 가능성은 있으나, 연산수가 NMS 대비 오히려 증가(11 vs 8)하여 HW 면적/전력 관점에서 실이득이 불확실하고 NAND 채널 검증이 없음.
