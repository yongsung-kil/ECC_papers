### B. 알고리즘 요약

1. AWGN/BPSK 채널, 4-bit 양자화 메시지의 regular LDPC 코드(`dv=3, dc=6, Nv=8000, Nc=4000, R=0.5`)를 대상으로 함.
2. 기존 Information Bottleneck(IB) LDPC 디코더는 check node 연산을 LUT로 구현하나, HW 합성 시 LUT를 elementary gate로 풀어내야 해 게이트 수가 큼(예: 첫 check node 연산에 612 gate 소요).
3. 채널은 AWGN+BPSK+양자화 수신, IB의 목표는 관련변수 X와 압축변수 T 간 상호정보 `I(X;T)`를 최대화하는 매핑 `pT|Y(t|y)` 설계.
4. 핵심 기법: LUT 대신 check node 연산을 GF(2) 이진 행렬곱 `t = A·y` (XOR 게이트 회로)로 직접 표현하고, 이 행렬 `A`와 selective inversion mask `p`를 유전 알고리즘(GA)으로 탐색해 mutual information을 최대화.
5. 핵심 식 `t = A·y` (식 5): 입력 메시지 비트 벡터를 이진 행렬로 곱해 압축 출력 비트를 얻음으로써 LUT 탐색 없이 곱셈-비교 없는 순수 XOR/AND/OR 조합회로로 구현.
6. 확장: 3개의 병렬 행렬 `A(m,0), A(m,1), A(m,2)`과 각각의 inversion mask를 사용해 출력을 OR로 결합하는 구조(Fig.7)로 정밀도-복잡도 트레이드오프 조절(matrix 개수로 조절).
7. GA는 population/genotype(행렬+mask의 이진 표현)에 대해 crossover, mutation, elitism을 적용해 fitness function(= density evolution 기반 mutual information)을 최대화하는 방식으로 세대 진화.
8. 학습 절차: density evolution으로 먼저 asymptotic 성능 확인 후 특정 `Eb/N0`(예: 1.575dB) 설계점에서 GA로 회로 최적화, variable node는 그대로 기존 IB LUT 사용(check node만 대체).
9. 결과: 3-matrix 구성이 min-sum decoder를 근소하게 상회하는 BER 성능을 보이며 LUT 기반 IB 디코더 대비 0.4dB 열화 있으나 게이트 수는 첫 연산 기준 612→42(93% 절감).
10. 한계: HW 합성/실리콘 구현 없이 논리 게이트 수 정성적 비교에 그침, variable node 최적화는 future work로 남김, 4-bit soft 정수 메시지의 AWGN 채널 특화로 NAND 채널 특성 미반영.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| check node 연산을 XOR 회로로 대체(GA 학습) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산을 대체하는 저복잡도 대안 아이디어로 참고 가능, 다만 Prime ECC는 이미 min-sum(하드웨어 검증)이라 교체 리스크 큼 |
| iteration별 mutual-information 최적화 절차(GA + density evolution) | `local_opt.cpp` | LLR 파라미터 자동최적화 모듈과 목적(파라미터 탐색) 유사하나 최적화 대상(회로 구조 자체)이 다름 |
| 4-bit 양자화 메시지 처리 | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | soft-4bit 계열 양자화와 개념적으로 유사하나 IB 압축 매핑 방식은 Prime ECC의 LLR 테이블 방식과 상이 |

> 적용 가치: 중간 — check node를 저복잡도 XOR 회로로 대체하는 아이디어는 min-sum 대비 게이트 절감 잠재력이 있어 참고 가치가 있으나, HW 미구현·검증 부족과 variable node 미해결로 직접 이식보다는 개념 참고 수준.
