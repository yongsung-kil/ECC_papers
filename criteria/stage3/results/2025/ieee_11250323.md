### B. 알고리즘 요약 (10줄 내외)

1. 대상은 bivariate bicycle (BB) QLDPC 코드 `[[90,8,10]]`, `[[144,12,12]]` (Gross code), (3,6)-regular 구조에서 회로레벨 stabilizer 측정 노이즈를 다룸.
2. 기존 circuit-level Tanner graph는 노이즈원마다 변수노드를 만들어 불규칙 degree와 짧은 사이클이 많아 message-passing 성능이 낮고, 이를 보완하려면 O(N³)의 OSD가 필요해 비현실적.
3. 채널 모델: ancilla qubit에서 발생한 X error가 동일 stabilizer 측정 내 다수 data qubit으로 전파되는 hook error를 시간축 상관노이즈(FSM)로 모델링.
4. 핵심 기법: hook fault를 하나의 ancilla당 "equalizer node"로 묶고 BCJR SISO 추정기로 로컬 디코딩, 원래 Tanner graph와 결합한 joint Tanner graph `HJ=[[HZ, In],[0, HXT]]` 위에서 min-sum과 turbo equalization.
5. 핵심 식: BCJR forward/backward/branch metric(`log α_t`, `log β_t`, `log γ_t`)로 각 equalizer node가 사후 LLR을 계산, extrinsic info `σ_j = L_j - μ_i,j`를 constraint node에 전달.
6. 확장: check-to-variable update는 표준 min-sum(정규화 factor β=0.875), variable node update는 MS-PI(Min-Sum with Past Influence, BB코드 전용) 규칙 적용.
7. 구현 디테일: 3개 디코더(layered ×2 + flooding ×1)를 병렬 실행하는 diversity decoding으로 trapping set/수렴 실패 완화.
8. 별도 학습/최적화 절차 없음 (파라미터는 heuristic tuning, 저자도 향후 최적화 필요성 언급).
9. 결과: `[[90,8,10]]` 코드에서 표준 MS 대비 성능 크게 개선, BPOSD0에 근접(logical error rate 10⁻²~10⁻⁶ 구간). `[[144,12,12]]`는 개선폭이 더 작음. 복잡도는 O(n) (n=원 코드 blocklength).
10. 한계: 시뮬레이션만 수행(HW 미설계), X error만 고려(Z 대칭 가정), idling/measurement noise 미포함, 코드 크기에 따라 성능 이득 편차 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Turbo equalization / BCJR SISO 추정기 | 대응 없음 | Prime ECC는 non-binary trellis/BCJR 미보유, quantum-specific 상관노이즈 모델 |
| min-sum check update (정규화 β=0.875) | `decoder.cpp` `CNU_Update_New_Mag()` | 정규화 min-sum 자체는 이미 보유 기법과 유사하나 quantum 특화 확장 |
| MS-PI (Min-Sum with Past Influence) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 이전 iteration 메시지 반영이라는 아이디어는 이론상 대응 가능하나 BB code 전용 설계 |
| diversity decoding (병렬 3디코더) | 대응 없음 | Prime ECC는 단일 디코더 구조, 병렬 다중 디코더 미보유 |
| QLDPC (bivariate bicycle) 부호 구조 | `ecc_top.cpp` `Load_PCM()` | non-binary/quantum 부호이므로 판정 필터상 이식성 하 |

> 적용 가치: 낮음. 양자 QLDPC 특화 회로노이즈(hook error) 디코딩이며 non-binary/quantum 도메인이라 NAND binary LDPC 이식 근거가 거의 없음.
