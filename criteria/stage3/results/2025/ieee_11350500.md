### B. 알고리즘 요약

1. 광통신 백본용 FEC로 28%-오버헤드 HC-GLDPC-CC(rate≈0.78)를 설계, 내부코드로 사용하고 외부 BCH(8000,7896)와 연접.
2. 기존 SPC-only LDPC는 waterfall 성능이 부족해, SPC 제약 노드 일부를 BCH 기반 generalized-check(GC) 노드로 교체하고 convolutional 구조로 추가 이득을 얻는다.
3. 채널은 광통신 노이즈, LLR 입력 기반 SISO 복호를 가정.
4. 부호구성: regular (3,15) QC-LDPC 블록코드를 unwrapping해 LDPC-CC로 만든 뒤, 매 32행 중 앞 `k`행(k=1~5, 즉 GC비율 k/32)을 BCH parity 열로 치환. z(CPM 크기)=1024/1536/2304, BCH는 t=1의 BCH(15,11)~BCH(75,67).
5. 복호는 layered 알고리즘: shifter → VN update `Q_nm = L_n - R_mn` → CN update → posterior `L'_n = Q_nm + R'_mn`.
6. CN 갱신은 노드별 상이: SPC 노드는 normalized min-sum(NMS, 스케일 α∈(0,1]), GC 노드는 Chase 알고리즘 적용.
7. Chase 디코더는 최소 |LLR| p비트를 LRB로 골라 test pattern 생성 → BCH hard decode → syndrome flip → 내적 최소 후보 2개 선택 → extrinsic 정보 계산.
8. error floor를 10^-15까지 낮추기 위해 32-lane convolutional interleaver + 외부 BCH(8000,7896, t=8) 연접.
9. 결과: 최적 파라미터 4/32, z=1024, 24 iter에서 input BER 4.51e-2일 때 GLDPC 출력 BER 3×10^-12, 외부 BCH 연접 후 10^-15 이하, NCG 12.28dB, Shannon 갭 0.93dB.
10. 한계: GC 비율↑이 waterfall만 개선하고 error floor는 개선 못함; FPGA emulation뿐 gate count·ASIC 미제시, 광통신 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPC 노드 NMS 갱신 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` | normalized min-sum은 우리 min-sum과 유사하나 α 스케일링은 이미 유사 기능 보유, 중복 |
| GC(BCH) 노드 Chase 복호 | 대응 없음 | Chase/BCH SISO는 binary QC-LDPC min-sum 엔진에 없음, 이식 부적합 |
| layered VN/CN/posterior 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 이터레이션 루프 골격은 유사하나 GLDPC-CC 특화 |
| BCH parity 열 치환 부호구성 | 대응 없음 | Prime ECC는 고정 binary QC-LDPC H-matrix, GLDPC 치환 미지원 |
| 외부 BCH 연접 + interleaver | 대응 없음 | 연접부호/인터리버는 코드베이스 범위 밖 |

적용 가치: **낮음** — non-binary BCH component + Chase(SISO) 기반 GLDPC-CC로 NAND용 binary QC-LDPC min-sum 아키텍처와 구조적으로 이질적이며 광통신 특화이다.
