> 총평: NAND flash(rate 0.8, sector 1kB)용 연접부호 비교 논문. 내부 LDPC는 LDGM(degree-1 노드 error-floor)뿐이고 결론이 대수적 BCH/GC 우위. Prime ECC의 min-sum binary QC-LDPC와 부호군이 이질적이며 이식 가치가 낮아 이식성 하.

### B. 알고리즘 요약

1. 시스템: MLC NAND flash ECC, 전체 rate ~0.8, sector 1kB(payload 1036B, `ko=8288`), 채널 BSC로 모델링.
2. 문제: NAND 오율 증가로 BCH 한계, LDPC/turbo는 error-floor·복잡도로 미채택; 연접이 standalone BCH 대안인지 검증.
3. 구성1(직렬연접): outer 이진 BCH `no=8624`(GF(2^14), t=24) + inner LDPC/turbo, inner rate R≈7/8, 최대길이 10200bit.
4. inner LDPC = LDGM: 체계적 생성행렬 `G=[I|P]`, `H=[P^T|I]`, 파라미터 `(X,Y)`(P의 행/열 1의 수), degree-1 parity 노드가 error-floor 유발.
5. 핵심 식: error-floor BER 근사 `BER≈Σ_{l>X/2} C(X,l)p^l(1-p)^{X-l}` (X odd) — degree X 노드가 절반 이상 flip 시 오류.
6. 선택: (6,34) LDGM, n=10160 선정(X=6이 고오율 성능·낮은 floor 균형).
7. 문제점: 잔류오류가 통계적 독립이 아니어서 outer BCH super-channel을 BSC 이항분포로 못 봄 → 초저 block error rate 보증 불가; sum-product는 채널오율 추정 필요, min-sum은 turbo보다 열등.
8. 구성2(GC): outer RS(GF(2^10)) 2레벨 + inner BCH, multistage 디코딩, union bound `Pe≤ΣPa,l`로 오류확률 설계·최적화.
9. 결과: WER상 LDGM+BCH(sum-product)가 소폭 최우수이나, Pe=10^-16 보증은 BCH(pmax=6e-3)·GC(pmax=3.3e-3)만 가능; GC 디코더 크기는 BCH 대비 약 1/6.2.
10. 한계: HW 미설계(복잡도는 O(m^2) 곱셈기·Chien search 정성 추정), 시뮬뿐; 최종 결론은 대수적 BCH가 여전히 최선.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDGM 내부부호 / degree-1 error-floor | 대응 없음 (Prime ECC는 정규 QC-LDPC, LDGM 아님) | 부호군 상이, 이식 대상 아님 |
| outer BCH/RS 연접·multistage 디코딩 | 대응 없음 | Prime ECC는 CRC 조기종료(partial_CRC.cpp)만, 대수적 outer code 없음 |
| sum-product / min-sum 디코딩 | decoder.cpp LDPC_Decoder() (min-sum) | 논문의 LDGM min-sum은 성능 열등 결론, 이식가치 없음 |
| BSC 채널오율 추정 이슈 | channel.cpp Set_LLR_Th() | Prime ECC도 read 조건별 LLR 필요하나 논문은 BSC hard만 다룸 |
```

적용 가치: 낮음 - NAND 직접 타깃이나 부호군(LDGM/연접 BCH/RS)이 Prime ECC의 binary QC-LDPC와 이질적이고, 결론이 iterative LDPC 비우호적이라 떼어 쓸 기법이 없음.
