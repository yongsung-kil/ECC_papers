### B. 알고리즘 요약

1. 대상은 time-invariant SC-LDPC(convolutional) 부호로, `H_SC(γ,c,m,L)` 밴드 구조, constraint length `νs=(m+1)c`, memory `m`, coupling length `L`.
2. 유한 블록길이에서 error floor를 유발하는 지배적 구조는 변수정규 부호의 LETS(leafless ETS)이며, 이를 작은 `m`으로 제거하는 것이 목표.
3. 채널/복호 모델: binary-input AWGN + box-plus(pair-wise) BP sliding window decoder, 최대 50 iteration, floating-point.
4. 핵심 관찰: LETS는 짧은 사이클(6/8/10-cycle)의 dot_m 확장으로 생성되므로, 특정 사이클을 LSS-non-expandable로 만들면 하위 LETS 전체가 사라진다.
5. 사이클 존재조건은 exponent matrix `P`의 차분 파라미터 합 `ΔP=0`(식 4)으로 판정 — HW/그래프 탐색 없이 `P`만으로 검사.
6. 8-cycle을 Type-A/B/C로 분류(관여 행 개수 2/3/4), dv=3,4,5 및 girth 6,8 각각에 대해 "Type-A(with chord)/Type-B 회피 + 특정 사이클 LSS-nE" 형태의 충분·필요조건(Prop.1~12) 제시.
7. 각 조건 충족에 필요한 memory 하한 유도(예: dv=3,g=8이면 `m≥c(c-1)/4`; dv=4,g=8이면 `m≥3c(c-1)/4`).
8. 부호 구성은 greedy column-by-column backtracking 탐색(Algorithm 1, MATLAB), girth 하한 `m0`에서 시작해 조건 만족 exponent matrix를 탐색.
9. 결과: 동일 degree분포에서 기존 [3][5][6] 및 CB-SC-LDPC[37][38] 대비 더 작은 `m`/`νs`와 우수한 LETS 분포·낮은 error floor(예: g=6이 g=8 대비 우수), rate 0.16~0.88 예제.
10. 한계: HW 미설계, floating-point 시뮬만, fixed-point 복호에서의 error floor는 향후 과제(각주 9), 일부 memory 하한은 loose.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC exponent matrix `P` 구성/사이클 제약 | 대응 없음 | Prime ECC는 고정 binary QC-LDPC(block), SC convolutional 구성법과 무관 |
| H-matrix/부호구조 로드 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 부호구조 로드 지점은 있으나 SC 밴드행렬 구조 자체가 비호환 |
| LETS 제거로 error-floor 개선 (설계 철학) | 대응 없음 | Prime ECC는 min-sum HW 디코더, 부호설계 층 재검증 부담 높음(프로파일 §4) |
| sliding window BP 복호 | 대응 없음 | Prime ECC는 block QC-LDPC의 pipeline min-sum, window 복호 아님 |

적용 가치: **낮음** — SC-LDPC convolutional 전용 부호설계 구성법으로, Prime ECC 3.1의 고정 block QC-LDPC 구조·min-sum HW 디코더와 정합되지 않아 이식 불가.
