### B. 알고리즘 요약

1. 대상: RAID 디스크 어레이의 이중 디스크(소거) 장애 허용을 위한 MDS array code. IF2 위의 [p+1, p-1] 코드(p=m+1은 소수), 코드율 p-1(정보열 p-1개, 패리티열 2개).
2. 문제: 기존 RS 코드는 유한체 연산이 필요해 HW 변경 부담이 크고, EVENODD 코드는 XOR 기반이지만 파리티체크 행렬이 상대적으로 조밀해 인코딩/복호 연산량이 큼.
3. 모델: 디스크 장애를 소거(erasure)로 모델링, 채널은 랜덤 비트 오류가 아니라 특정 열(디스크) 전체의 소실/오류.
4. 핵심 기법: 순환치환행렬 `Ep`와 이를 변형한 `Em(l)`로 구성한 파리티체크 행렬 `H`(식 (1))를 정의, p가 소수일 때 최소 해밍거리 3(=MDS, 두 소거 정정 가능)임을 증명.
5. 핵심 식: 수평 패리티 `a[i,p-1]=⊕a[i,j]` (식 (3))와 대각 패리티 `a[i,p]=⊕a[⟨i-j⟩p,j]` (식 (4))로 구성 — EVENODD와 유사하나 대각 패리티 계산에서 제외되는 대각선이 하나뿐이라 행렬이 더 희소.
6. 확장: erasure decoding algorithm(그림 3)으로 임의의 두 소거 열을 수평/대각 syndrome을 번갈아 사용하는 루프로 복구, error decoding algorithm(그림 4)으로 단일 열 오류도 syndrome 비교로 위치 검출·정정.
7. 부수 트릭: 인코딩 복잡도 척도 `β`=(XOR 연산수)/(데이터 블록수)로 EVENODD 대비 비교, 소프트웨어 구현에선 두 패리티 계산의 병렬 불가가 문제되지 않음.
8. 학습/최적화 절차: 없음(순수 조합론적 구성·증명).
9. 결과: 인코딩·소거복호 XOR 연산량 β가 EVENODD보다 작음(Table 1, 2, 수치는 표에만 존재해 본문 서술 없음 → 미상).
10. 한계: LDPC/반복복호가 아닌 RAID 소거정정 array code 이론 논문. HW 미설계, 시뮬/실측 없음, NAND·soft-decision 개념과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MDS array code 파리티체크 행렬 구성 | 대응 없음 | RAID 소거코드(EVENODD 계열)로 QC-LDPC 부호구조와 무관 |
| 소거/오류 복호 알고리즘 | 대응 없음 (Prime ECC는 min-sum message-passing) | 소거(disk 단위) 기반 대수적 복호로 패러다임 상이 |
| XOR 기반 저복잡도 인코딩 | 대응 없음 | RAID 패리티 계산으로 NAND binary LDPC 인코더(dual-diagonal)와 무관 |

적용 가치: **낮음** — RAID 디스크 어레이용 MDS array code(EVENODD 변형) 이론 논문으로, LDPC 반복복호·soft-decision·NAND 채널과 전혀 무관해 Prime ECC에 이식할 요소가 없음.
