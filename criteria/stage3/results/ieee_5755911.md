> 총평: 고rate(0.95+) self-orthogonal QC-LDPC의 parity-check 행렬을 4-cycle뿐 아니라 generalized Pasch(6-cycle 관련) 구조까지 배제하도록 설계해 dmin을 확장하고 error-floor·undetected error를 개선한 부호구성 논문으로, Prime ECC의 H-matrix 설계 레이어에 이식성 있으나 재검증 부담이 있음.

### B. 알고리즘 요약

1. 대상은 self-orthogonal(SO, 4-cycle 없는) quasi-cyclic(QC) 이진 LDPC 코드로, parity-check 행렬 `H(p)`는 `p`개의 순환치환행렬(circulant, `m×m`)로 구성되며 코드율은 `1-(1/p)` 근방의 고rate(0.95 이상)이다.
2. 풀려는 문제는 SO 코드의 최소 해밍거리 `d_min`이 column weight `w`에 대해 보통 `w+1`에 그치는데, 이를 `w+2`(짝수) 또는 `w+3`(홀수)로 확장해 error-floor를 낮추는 것 - 기존 finite-geometry 기반 구성(PG/EG 코드)은 이 확장을 만족하지 못한다.
3. 핵심 가정/모델은 AWGN 채널이며, circulant의 첫 열 원소 위치 집합인 base address set(`BAS`, 식(5)(6))으로 parity-check 행렬을 표현한다.
4. 핵심 기법: BAS가 difference set 조건(식(7)) 및 서로 다른 BAS 간 disjoint 조건(식(8))을 만족하도록 하여 4-cycle-free를 보장하고, 나아가 generalized anti-Pasch(GAP) 조건(식(9))을 추가로 만족하도록 해 길이-6 cycle 기반의 generalized Pasch 구조를 제거한다.
5. 핵심 식 `by-1,0 - bz-1,0 + b0,z - by-1,z + bz,y - b0,y ≠ 0` (식(9))는 임의의 (w+1)-튜플 컬럼 조합에서 generalized Pasch 구성이 발생하지 않도록 보장하며, 이 조건이 성립하면 `d_min ≥ w+2`(짝수 w) 혹은 `w+3`(홀수 w)가 증명된다.
6. 확장 기법: 식(13)을 추가로 만족하는 BAS를 선택하면 길이-6 cycle의 개수를 최소화(성능 추가 개선)할 수 있으나 p(circulant 개수, 즉 code rate)를 다소 희생한다.
7. 부수 트릭/구현: 2단계 컴퓨터 탐색(Step1: 조건(5)(6)(7)(13) 만족 BAS 리스트 생성, Step2: 조건(8)(9) 만족하는 p개 BAS를 tree+random local search로 선택)으로 m>250, p>20의 큰 코드도 설계 가능.
8. 학습/최적화 절차는 트리서치+랜덤서치 기반 local search이며 별도 DE/GA/NN 없음.
9. 결과: 기존 PG/EG 참조코드(rate 0.9524/0.9526) 대비 신규 코드(rate 0.9550/0.9563, dmin=6 이상)는 Eb/N0 5dB 이상에서 참조코드에 나타나던 error-floor와 undetected word error 급증이 관측되지 않음(BER 시뮬, sum-product 최대 20 iteration).
10. 한계: HW 인코더/디코더 회로는 다루지 않으며(순수 부호구성 논문), 시뮬레이션은 word error 100개 수집 기준으로 매우 낮은 BER 영역(<1e-6)의 통계적 신뢰도는 제한적일 수 있다.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GAP configuration 기반 QC-LDPC H-matrix 설계 | `ecc_top.cpp` `Load_PCM()` | 특정 QC-LDPC H-matrix 구조/설계법이므로 이식 시 코드 자체 재검증 필요(난이도 높음) |
| 4-cycle/6-cycle free BAS 탐색(local search) | 대응 없음 | 부호 구성 알고리즘으로 디코더/인코더 실행 경로와 무관, 오프라인 코드설계 도구 |
| Error-floor 개선(고rate SO-QC 특유 구조) | `decoder.cpp` `LDPC_Decoder()` (간접) | 디코더 자체 변경 없이 H-matrix 구조 개선으로 error-floor를 낮추는 접근이라 decoder 코드 수정 없이도 PCM 교체만으로 효과 가능성 |

> 적용 가치: **중간** — 표준 min-sum/sum-product 디코더는 그대로 두고 H-matrix(BAS) 설계만으로 고rate QC-LDPC의 error-floor를 개선하는 기법으로, Prime ECC의 `Load_PCM()` 레벨에 이식 가능성은 있으나 특정 QC-LDPC 구조 고정에 따른 재검증 부담이 큼(§4 "부호설계는 이식 난이도 높음"에 해당).
