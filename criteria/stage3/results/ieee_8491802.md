### B. 알고리즘 요약

1. 대상: `(3,12)`-정규 QC-LDPC, `z=108`, 부호길이 1296, rate 3/4, Xilinx Virtex-7 FPGA, 플러딩 스케줄.
2. 문제: LDPC 디코더에서 메시지 저장 메모리·인터커넥트가 면적/전력 주 병목 — 메모리 폭을 줄이고 싶다.
3. 기법: NS-FAID(Non-Surjective FAID) — 두 개의 양자화 도메인 사용. CN 처리·메모리 저장은 저양자화 `w`, VN·AP-LLR 갱신은 고양자화 `q`.
4. 크로싱: `framing`(q→w bit 압축) / `de-framing`(w→q bit 복원) 함수를 LUT로 구현. 메모리엔 압축 메시지만 저장.
5. `q=4`, `w=2,3` 사용. 프레이밍 함수 예: `w=3` 최적 `F=[0,1,1,3,3,3,7,7]`(4bit MS 대비 +0.12dB), `w=2` 최적 `F=[1,1,1,1,1,6,6,6]`(-0.315dB).
6. framing/deframing 함수와 gain factor `μ`(입력 LLR에 곱)는 Density Evolution으로 SNR threshold 기준 선정.
7. HW: 메모리-센트릭 플러딩 아키텍처 — VNU=B열수, CNU=B행수, 메모리 블록이 메시지 전달(인터커넥트 라우팅 제거). 한 iteration=`z` 클록.
8. VNU 변경: de-frame + SM→C2 변환을 단일 LUT로 병합(critical path 영향 최소화). CNU는 비트폭만 줄고 로직 변경 없음.
9. 결과: MS(q=4) 대비 TAR 개선 20~110%, 비용 최대 45% 절감. `w=2`는 CN 비교기 트리·메모리 폭 축소로 주파수까지 상승.
10. 한계: BER 실측 곡선 없이 DE threshold·FPGA 합성 수치만 제시, 저rate 단일 부호·플러딩만 검증.

적용 가치: **중간** — NS-FAID 압축 테이블 아이디어(CN 저양자화 + VN 복원 LUT)는 Prime ECC 양자화 레이어에 이식성 높으나, 아키텍처(플러딩 vs pipeline)와 부호 rate가 달라 프레이밍 함수 재도출 필요.
