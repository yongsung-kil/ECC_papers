> 총평: base graph/matrix 설계 시 girth·trapping set을 함께 최적화하는 구성법과, BP 조기종료 정보를 활용해 OSD 호출 횟수를 1~2회로 줄이는 후처리 디코딩이 핵심이나, 5G 소형 TBS·soft-4bit+ LLR 전제라 NAND binary LDPC로의 직접 이식은 제한적.

### B. 알고리즘 요약

1. 5G NR의 QC-LDPC(base graph+base matrix, lifting size `Z`)를 URLLC/mMTC용 소형 TBS(40~256bit)에 최적화하는 문제를 다룸.
2. 기존 5G LDPC는 eMBB 위주로 설계돼 소형 TBS에서 polar 코드 대비 성능이 열세.
3. 채널은 AWGN 가정, flooding BP 디코더(50 iteration)로 평가.
4. 트래핑셋을 `(a,b)` 파라미터(변수노드 수 a, 홀수차 연결 검사노드 수 b)로 정의하고, cycle에서 시작해 변수노드를 점진 추가하는 PVG(progressive variable node growth) 기법으로 poor trapping set을 식별.
5. base graph/matrix 설계 시 BGPP/PCM priority principle(큰 girth, 적은 최단사이클 수, 큰 a·b)을 기준으로 후보 중 최적 패턴 선택.
6. lifting size 계열(`i×2^j`, i∈{2,3,5,7,9,11,13,15})별로 base matrix를 별도 설계, 작은 Z의 순환이동값이 큰 Z 후보를 제한(`{s, s+Z/2}`)하는 원리 활용.
7. 디코딩 측: 개선 BP+OSD는 모든 BP iteration 종료 후 `H·c'`의 non-zero bit 수가 가장 적은 1~2개 LLR 시퀀스만 OSD(order=1)에 입력해 OSD 호출을 10회→1~2회로 축소.
8. 학습/최적화는 없고 결정적 탐색(girth 알고리즘 [11] 활용)으로 base matrix 구성.
9. 결과: 새 LDPC가 5G LDPC 대비 BLER=1E-4에서 0.3~0.4dB 이득, 개선 BP+OSD는 popular BP 대비 ~1dB, polar+SCL(list=32) 대비 ~0.3dB 이득이며 OSD 연산량은 1/10~1/5로 감소.
10. 한계: HW 설계 없음(시뮬만), soft-decision(연속 LLR) 전제로 NAND의 저비트 soft-read 환경과 괴리, TBS 40~256bit의 소형 코드만 검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| base graph/base matrix 설계(girth·trapping set 최적화) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체가 다른 고정 QC-LDPC라 재검증 부담 큰 code-design 레이어 |
| PVG 트래핑셋 탐색 방법론 | 대응 없음 | Prime ECC는 고정 PCM 로드만 하며 코드 설계용 탐색 알고리즘 모듈 없음 |
| BP+OSD 결합 디코딩(조기종료 기반 OSD 선별) | `decoder.cpp` `LDPC_Decoder()`, `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 판정 로직과 유사한 개념이나 OSD 자체는 min-sum 기반 Prime ECC에 없는 별도 후처리 단계 |
| soft LLR(OSD 입력용 연속값) | `channel.cpp` `Set_LLR_Th()` | OSD는 연속 LLR 정렬이 필요해 hard-1bit/soft-2~3bit 위주인 Prime ECC와 정합성 낮음 |

적용 가치: `낮음` — 5G eMBB/URLLC용 소형 TBS QC-LDPC 부호설계와 soft-decision OSD 후처리가 핵심이라, NAND 고rate·hard/저비트 soft 환경의 Prime ECC와 스펙·전제가 크게 달라 직접 이식 여지가 적음.
