### B. 알고리즘 요약

1. 시스템: BEC/AWGN 채널, 길이 n=1000, rate 1/2의 irregular (λ,ρ)-LDPC 앙상블 (BP 반복복호, 100회 iteration 시뮬 가정).
2. 문제: 짧은/중간 길이 irregular LDPC의 error-floor는 stopping set(작은 크기) 구조에 의해 결정되나, 최소 stopping set 크기를 최적화하는 설계법은 NP-hard로 알려져 미해결.
3. 핵심 가정: 모든 stopping set은 사이클을 포함하므로, 사이클의 "연결성"을 메트릭화(ACE)하여 사이클을 간접 제어하면 stopping set 크기를 키울 수 있다는 휴리스틱.
4. 핵심 개념: 사이클 `C`의 ACE 값 `ACE(C) = Σ_{v∈VC}(d(v)-2)` (식 3) — 사이클에 포함된 변수노드들의 (차수-2) 합. 값이 작을수록 stopping set이 되기 쉬움.
5. 확장(신규): 단일 threshold(flat spectrum) 대신, 사이클 길이 `2i`별로 최소 ACE값 `η2i`를 모은 "ACE spectrum" `η(G(H))=(η2,η4,...,η2dmax)` (식 6)을 정의해 그래프 구조를 세밀히 프로파일링.
6. 확장 설계: Generalized ACE-constrained 알고리즘은 사이클 길이별로 다른 `ηACE` threshold를 부여(비-flat), Viterbi류 shortest-ACE-path tree 탐색으로 노드 추가 시 위반 여부 검사.
7. 부수 개념: expanded ACE spectrum(히스토그램 형태)으로 그래프의 사이클 분포 시각화, "extremal ACE spectrum" 부분집합 정의(주어진 앙상블 내 최대 ACE 컴포넌트를 갖는 코드).
8. 최적화 절차 없음(학습X) — 그래프 성장 시 무작위 이웃 샘플링 + ACE 제약 검사(원 알고리즘과 동일한 랜덤 성장 방식, 제약만 일반화).
9. 결과: (5,3) flat ACE 코드 대비 generalized 설계(예: η=(∞,∞,16,9,5))가 error-floor를 크게 낮춤; ACE-spectrum-optimized PEG가 시뮬 범위 내 error-floor 없이 최상 성능.
10. 한계: BEC 기반 설계 원칙을 AWGN에 그대로 적용(저자도 "BEC에 좋으면 AWGN도 좋다"는 가정에 의존), HW/복잡도 분석 없음, near-codeword/trapping set 구조로 인한 예외(좋은 ACE에도 성능 나쁜 사례) 존재 인정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ACE-constrained 그래프 성장 설계 (H-matrix 구성) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 특정 QC-LDPC H-matrix 고정 사용, 이 논문의 랜덤 irregular 그래프 성장 방식을 적용하려면 H-matrix 전면 재설계 필요 (이식 난이도 높음) |
| stopping set/error-floor 개선 메트릭 (ACE spectrum) | `ecc_top.cpp` `Load_PCM()` | error-floor 개선이라는 목표는 겹치나 QC-LDPC(base+circulant shift) 구조와 무작위 irregular 그래프 성장 알고리즘 간 정합성 낮음 |
| BP 반복복호 검증 (시뮬만) | 대응 없음 | 본 논문은 표준 BP 시뮬 검증만 하며 min-sum/HW 디코더 요소 없음 |

> 적용 가치: 낮음 — Prime ECC는 특정 QC-LDPC(base matrix+circulant) 구조가 고정되어 있고, 본 논문은 무작위 irregular 앙상블에서의 그래프 성장형 구성법(ACE spectrum 기반)이라 H-matrix 재설계 비용이 크며, 디코더/HW 요소가 전혀 없어 이식 접점이 부호설계 레이어(난이도 최고) 하나뿐임.
