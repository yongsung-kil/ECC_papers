### B. 알고리즘 요약

1. 802.16e rate-1/2 2304-bit QC-LDPC(96×96 서브행렬, 12×24 base matrix, irregular)를 ASIC(TSMC 90nm)으로 구현한 사례 연구.
2. 기존 순차적 layered decoding(LDA)은 레이어를 하나씩 처리해 iteration당 지연이 크고, 병렬화를 시도한 기존안은 crossbar 상호연결로 면적이 커지는 문제가 있었음.
3. Min-sum 알고리즘(`α=0.75` 정규화)과 loosely coupled 알고리즘(VTC/CTV 대신 변수합 `Λn`, 검사합만 저장)을 기반 채널·연산 모델로 채택.
4. 핵심 기법 1단: Parallel Layered Decoding Architecture(PLDA) — 모든 레이어(CNU)가 동시에 `qj,nj = Λnj − rj,nj`(6), `rj,nj`(7, min-sum), `Λnj = qj,nj + rj,nj`(8)를 계산하고, 레이어 간 전용 메시지 경로로 crossbar 없이 통신.
5. 핵심 식은 base matrix의 순열값(permutation value) 크기 순서로 각 레이어가 다음 레이어로 `Λnj`를 전달하는 라우팅 규칙(각 컬럼별 내림차순 연결) — crossbar 대신 고정 배선으로 상호연결 복잡도를 제거.
6. 핵심 기법 2단: base matrix에 레이어별 오프셋(`+0~+88`)을 추가해 인접 레이어 간 처리 시점 간격을 5 클록 이상으로 벌림 (critical path splitting의 전제 조건).
7. 부수 트릭: CNU를 4단 파이프라인으로 분할(레지스터 3세트 삽입), 메시지는 6비트 고정소수점(정수부 3비트+소수부 2비트+부호비트)으로 양자화, early termination(이전/현재 iteration 하드결정 비교) 채택.
8. 학습/최적화 절차 없음 — 오프셋 값은 수작업/구조적으로 선정(레이어 간 간격 ≥5 조건 충족).
9. 결과: 10 iteration에서 950MHz 동작, 2.2Gbps throughput, PLDA로 iteration당 지연 약 75% 감소, critical path splitting으로 최대 주파수 950MHz vs 305MHz(3배 개선), 코어면적 2.9mm², 전력 870mW.
10. 한계: 특정 802.16e 고정 코드(2304-bit, rate-1/2)로만 검증, 정정성능(BER) 개선은 목표가 아니며 기존 min-sum과 동일(오히려 순수 처리량/지연 개선에 집중), 전력소모가 기존 대비 큼.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 레이어 병렬 처리(PLDA) 및 레이어 간 메시지 라우팅 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Dual-Update(홀짝 교대 스케줄)와 유사 목적이나, PLDA는 전 레이어 동시처리로 더 공격적인 병렬화 구조 |
| CNU min-sum 연산(6/7-input comparator) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum 보유, 비교기 트리 구조는 참고 가능 |
| critical path splitting(CNU 4단 파이프라인) | 디코더 HW 모델(z=32, HCU, GT) - pipeline 6-stage | 기존 6-stage 파이프라인과 통합 시 재설계 필요, 오프셋 기반 레이어 간격 확보 개념은 GT/HCU와 별개 아이디어 |
| loosely coupled algorithm(VNU 제거) | `decoder.cpp` `VN_Cal_HD()` | Prime ECC도 이미 유사 구조(별도 VNU 없이 CNU가 요약 갱신) 보유 가능성 - 대응 불명확 |
| 6비트 고정소수점 메시지 양자화 | `ecc_data.h` `PARAM_LLR` | 이미 자체 양자화 테이블 보유, 참고 수준 |
| early termination(하드결정 비교) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료를 이미 사용, 이 논문은 단순 비트 비교 방식이라 대응 약함 |

> 적용 가치: 낮음 | PLDA의 레이어 간 완전병렬 라우팅과 critical path splitting은 흥미로우나, min-sum/loosely-coupled/layered decoding 자체가 Prime ECC에 이미 구현되어 있고 부호설계·정정성능 기여가 전혀 없어 순수 HW 스케줄링 최적화 아이디어 참고용에 그침.
