### B. 알고리즘 요약

1. 시스템: 분산소스코딩(Wyner-Ziv 비디오)용 LDPCA 파생 LDPC, 단일 5940 변수노드 코드에서 rate 0.83/0.49/0.1 세 행렬(edges 32635 고정), BSC 모델, `MAX`=100.
2. 문제: DISCOVER 코덱은 QCIF 1프레임 side-info 복호에 4~8초 소요 — BP 복호를 실시간화해야 함.
3. 채널/모델: source-side info 상관을 BSC crossover로 모델, syndrome `s=Hc` 전송, side info를 오류판 신호로 간주해 복원.
4. 핵심기법 1단: 세 BP 알고리즘 비교 — Sum-Product `2tanh⁻¹(∏tanh)` (식1), Min-Sum `∏sgn·min` (식3), Algorithm E (메시지를 -1/0/+1 3치 양자화).
5. 식 의미: Min-Sum·Alg-E는 SP 근사로 tanh 제거 → 벡터명령 친화적(모든 check node 동일 연산).
6. 핵심기법 2단(병렬화): SPMD로 syndrome block을 코어별 분배(block-level, 데이터 의존 없음), iteration-level은 병렬화 불가, check node loop를 코어 내 병렬화(변수노드보다 시간비중 큼).
7. 부수 트릭: check-variable 메시지를 transpose+reorder로 128-bit 블록에 정렬(32-bit 메시지 4개 동시), neutral 메시지 X를 add=0/mult=1로 삽입; SSE로 Min-Sum, 논리연산으로 Alg-E 16 노드 동시.
8. 최적화: SP는 tanh LUT화해도 벡터화 부적합, Fast Alg-E는 조건부 갱신이라 벡터화 부적합 → Vector Min-Sum이 최선.
9. 결과(실측, Core2 Quad 3GHz): Vector Min-Sum이 Alg-E 수준 속도에 SP 근접 성능; H(rate0.83) crossover 0.05서 8.8ms/decoding, 300 bitplane/frame → 2.64s/frame(QCIF 6.06 fps).
10. 한계: HW(RTL) 미설계 순수 CPU SW 병렬화, 분산비디오 side-info 복호 타깃, 알고리즘 자체 신규성 없음(기존 SP/MS/Alg-E 재사용).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum check node 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 min-sum(min1/min2+sign XOR) 보유 — 중복 |
| check node 병렬 처리(transpose/remap) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 z=32 row 병렬 HW로 이미 병렬; CPU SIMD 방식과 상이 |
| Sum-Product / Algorithm E | 대응 없음 | full-tanh SP·3치 Alg-E는 Prime ECC 대응 방향 아님 |

적용 가치: **낮음** — Min-Sum·SPMD 병렬화는 Prime ECC가 이미 보유(§3)하고, CPU SIMD 소프트웨어 최적화라 HW 디코더 이식과 정합성이 없다.
