### B. 알고리즘 요약

1. 시스템: LDPC 기반 SSD, rate-8/9·length-1kB QC-LDPC(column weight 4), 20nm MLC NAND, ONFI 2.2(정상 166MB/s).
2. 문제: worst-case 지향 ECC redundancy가 수명 대부분 미사용 → 잉여 정정력으로 I/O link를 오버클록해 속도 향상, 단 유발되는 link 오류를 복호가 잘 처리해야 함.
3. 채널 모델: cell 저장은 BSC, I/O link는 plane·8-bit 라인별로 다른 `parallel binary asymmetric channel(BAC)` (die당 2~4 plane, 16 신호경로, 1→0/0→1 비대칭).
4. 기법 1단(overclocking-aware LLR): `LLR(xk)=log p(xk=0|yk)/p(xk=1|yk)`를 전확률 전개해 link 조건확률 `pm(l)(yk|vk=i)`(라인 l, plane m)와 P/E-종속 cell 확률 `p(vk=i|xk=j,c)`를 명시적으로 반영.
5. 식 의미: link 오류(라인/plane별 비대칭)와 cell 오류(P/E cycle c 종속)를 분리 결합해 정확한 LLR 산출 → 복호 성능 향상.
6. 기법 2단(interplane codeword interleaving): 각 codeword를 n plane의 n physical page에 균등 분산 → plane별 상이한 link BER를 codeword 간 균일화(통신 인터리빙 유사).
7. 구현 디테일: on-chip randomizer로 cell의 1→0/0→1 등화 가정, BAC 파라미터는 측정으로 설정.
8. (학습절차 없음 — 측정 기반 파라미터화 + 시뮬)
9. 결과: 275MB/s 오버클록(I/O throughput +60%)에서 link BER>10^-3; 두 기법 병용 시 복호 power 60% 절감, decoding failure(retransfer) rate 2 orders(초기수명 최대 114.7배) 감소. 후기수명엔 이득 5.8배로 축소.
10. 한계: HW 미설계(FPGA는 측정 플랫폼), 시뮬 평가, 이득이 I/O overclocking 특수 오류원에 종속(late-life엔 미미).
