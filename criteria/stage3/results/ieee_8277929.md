> 총평: NAND 3D flash용 고rate QC-LDPC error-floor 개선 핵심 논문 — harmful ETS를 cycle-6 연결로 환원해 부호구성 단계에서 회피, Prime ECC H-matrix 선별에 직접 이식가치 높음.

### B. 알고리즘 요약

1. 타깃은 NAND flash용 고rate QC-LDPC(rate `R≈0.89`, message 1KByte, regular `(dv,dc)` with `dv=4,5`), RS기반 CPM-dispersion으로 구성.
2. 풀려는 문제: 1KB/2KB급 부호는 CPM(lifting) 크기가 작아 small cycle 제거가 불가 → error-floor가 storage에 치명적.
3. 채널 모델: NAND용 HRE(High Reliability Error) `r=(-1)^{u⊕e}+n` = BSC(플립 e)⊕AWGN(n) 연접(SLC→TLC 복사 플립, 셀 결함).
4. 핵심 기법: base matrix `P=[p_ij]` 위에서 length-2k cycle 존재조건 `Σ(p_{i,j}-p_{i,j+1})=0 mod(q-1)`(식4)로 cycle-6를 식별.
5. cycle-6 두 개의 연결관계를 5종(공통노드/공통에지/연속2에지/대향2에지/연속3에지)으로 분류, 특히 공통 수평에지 CHE와 conjugate vertical edge CVE가 harmful.
6. 확장: `(a,b)` trapping set을 `(a,a',e1,e2,b)`로 확장 표기, `e1+e2≥b`면 absorbing set. dv=4 최악은 cycle-6 4개가 연결된 `(4,4,6,0,4)` (dv=4 최소 absorbing set), dv=5는 `(4,4,6,0,8)`.
7. 구현: min-sum 디코더 HRE 시뮬로 max iteration 실패 시 ETS 수집·빈도 프로파일(Table I~III).
8. 학습/최적화 절차: RS-LDPC 랜덤 생성 → cycle-6 연결 프로파일 → `(4,4,6,0,4)/(4,4,6,0,8)` ETS 존재여부로 good/bad 라벨, good 중 CHE 적은 코드 추가 선택.
9. 결과: AWGN에서 good 코드가 bad 대비 FER/UBER error-floor 뚜렷이 개선(Fig.6), dv=4는 dominant ETS 구조가 단순해 error-floor 더 심함.
10. 한계: HW 미설계, 직접 설계가 아닌 랜덤 생성 후 선별, dv=4/5·girth-6 RS-LDPC 특정 케이스 결과.
