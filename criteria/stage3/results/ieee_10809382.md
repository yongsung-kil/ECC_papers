### B. 알고리즘 요약 (10줄 내외)

1. CCSDS 표준 QC-LDPC 부호 C2 (rate `0.875`, `(8176,7154)`, 511×511 circulant submatrix)를 코히런트 광통신(BPSK, inter-satellite) 채널에 적용.
2. 문제: 기존 NMSA 디코더가 constant-cycle weight를 쓰고 memory conflict를 무시해 Gbps급 throughput 달성이 어려움.
3. 채널 모델: BPSK 변조 heterodyne coherent receiver, 광섬유/위성 채널의 신호감쇠·SNR 저하 환경.
4. 핵심 기법: log-domain BP를 Normalized Min-Sum Algorithm(NMSA)으로 근사, 정규화 계수 `α`(0.6~0.9)로 정확도 손실 보정.
5. 디코딩 사이클 수식 `N_cyc = N_iter × N_1 + N_2 + 2`(N_iter=반복횟수, N_1/N_2는 처리시간)로 throughput을 iteration 수와 직접 연결.
6. 인코더는 정보열을 14개 세그먼트로 분할, 각 세그먼트를 511-bit 순환 shift register와 AND-XOR 연산으로 순환 부분행렬과 곱해 인코딩 복잡도·자원 절감.
7. 디코더는 3개 RAM 블록(Initial LLR RAM, 2개 iteration RAM)과 VNU/CNU/LLR초기화 유닛으로 구성된 파이프라인 구조로 반복 message-passing 수행.
8. 별도 학습/최적화 절차 없음 — MATLAB으로 iteration 수(8/10/12/16/20/24)별 성능-throughput 트레이드오프를 사전 스윕해 8 iteration을 채택.
9. 결과: 285MHz 클록에서 최대 2229.38Mbps throughput, 2Gbps BPSK에서 BER<1E-7·수신감도 -56dBm·코딩이득 8dB, 인코더 LUT -83.3%/레지스터 -84.9%(참고문헌[12] 대비), 디코더 LUT -62%/레지스터 -70%(참고문헌[13] 대비, throughput은 -15.8%).
10. 한계: 부호 자체는 CCSDS 표준 고정 사용(부호설계 기여 없음), min-sum 알고리즘도 표준 NMSA 그대로이며 신규 디코딩 기법 없음, ASIC 합성 아닌 FPGA(Xilinx 7VX690T) 구현에 한정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMSA(Normalized Min-Sum, α 정규화계수) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이라 정규화 계수 α 적용 방식은 참고 가치 있으나 이미 유사 기법 보유(중복) |
| iteration별 throughput-성능 트레이드오프(8 iter 채택) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 반복횟수-throughput 관계는 이미 Prime ECC의 적응형 LLR/조기종료 설계와 유사한 목적, 신규성 낮음 |
| 3-RAM 파이프라인(Init LLR RAM + 2 iteration RAM), VNU/CNU 분리 구조 | `decoder.cpp` `LDPC_Decoder()` pipeline 6-stage | HW 파이프라인 구조는 Prime ECC의 col_M1/Z/P1~P4 파이프라인과 유사한 개념이나 세부 구조 다름, 참고용 |
| 14-segment 인코더 순환 shift register 구조 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | CCSDS 표준 인코딩 방식(순환 shift+AND-XOR)으로 Prime ECC의 dual-diagonal 인코딩과 방식이 달라 직접 재사용은 어려움 |
| CCSDS C2 H-matrix(511×511 circulant) | `ecc_top.cpp` `Load_PCM()` | 특정 표준 고정 부호 로드이며 Prime ECC는 자체 QC-LDPC 구조 보유, 재검증 부담 큼 |
```

> 적용 가치: **중간** — 부호설계·알고리즘 자체의 신규성은 없으나(표준 CCSDS + 표준 NMSA), FPGA 자원 절감과 iteration-throughput 트레이드오프 실측 데이터는 Prime ECC의 HW 최적화 방향성 참고 자료로 활용 가능.
