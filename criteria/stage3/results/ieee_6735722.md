### B. 알고리즘 요약

1. NASA 심우주 AR4JA(Accumulate Repeat-4 Jagged-Accumulate) protograph LDPC 4종을 FPGA(VHDL)로 구현: rate 1/2 (2048, 16384bit), rate 7/8 (4096, 16384bit).
2. AR4JA는 protograph를 lifting하여 단순 circulant(단일 순환 identity)만으로 구성 -> DVB-S2의 superimposed circulant보다 layered 디코더가 훨씬 단순.
3. circulant 한 행 = 한 "layer", layer 내 c-node는 서로 다른 v-node를 갱신하므로 메모리 충돌 없이 병렬 처리.
4. layered 디코딩: `LQi=LLR(xi)` 초기화 -> Step1 `Lqij=LQi-Lrji` -> Step2 c2v `Lrji` -> Step3 `LQi=Lqij+Lrji` 갱신, layer/iteration 반복.
5. Step2 c2v는 `λ-min` 알고리즘(`λ=3`): Vj\i 중 최소 3개만 사용, `f(x)=-log tan(x/2)`와 min* 덧셈규칙(식8)을 소형 LUT로 근사.
6. circulant 크기 M(128/1024/64/256)이 코드마다 달라, 4종이 단일 디코더 아키텍처 공유하도록 `L=M/z` 부분병렬(L=64 최대) 채택.
7. sub-row 재배열: 원 circulant rotation `RM`을 `t1=RM/z`, `t2=RM%z`로 분해해 sub-row 정렬 rotation `Rs`·처리순서 `As` 유도(식9~12).
8. HW: LQ/Lr BRAM(64-wide), barrel shifter, min-sort 후 min* 연산 CNU, layer 재정렬로 pipeline 충돌 회피(불가시 wait 삽입), double-buffer 입출력.
9. 결과: Virtex-6 LX130에서 175MHz, 30 iteration 기준 uncoded throughput 29~74Mbps, Slice reg 18627/LUT 25035/BRAM 36.
10. 한계: BER/throughput 곡선은 그림에만 있어 수치 미상, 부호가 방위통신용 저rate·장블록이라 NAND 고rate ECC와 정합 부족.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered 디코딩 이터레이션 루프 | decoder.cpp LDPC_Decoder() | Prime는 dual-update 스케줄, 논문은 layered - 스케줄 개념 참고 |
| λ-min(λ=3) c2v 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime는 min1/min2 min-sum, λ-min은 상위 근사 - 대체 검토 여지 |
| circulant rotation / L 부분병렬 | ecc_top.cpp Load_PCM(), GT.cpp Make_GT_HW() | Prime z_sb=32 병렬, 논문 sub-row 정렬식은 다른 병렬화 방식 |
| LLR/Lr metric width 양자화 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 5/8/6bit vs Prime 2~3bit soft, 양자화 폭 비교용 |
```

적용 가치: **중간** — layered 부분병렬 스케줄링·메모리 라우팅과 λ-min CNU는 참고할 만하나, 부호가 저rate·장블록이고 정정능력/error-floor 개선 기여가 없어 Prime 고rate QC-LDPC로의 직접 이식 이득은 제한적.
