### B. 알고리즘 요약 (10줄 내외)

1. 시스템/코드 스펙: IEEE 802.11n(2012) QC-LDPC, `mb×nb = 12×24`, `z ∈ {27,54,81}` → `n = 24z = 648/1296/1944` bit, `rate R = 1/2`.
2. 풀려는 문제: 5G mm-wave 시스템은 처리 시간 예산이 더 줄어들어 Gb/s급 저지연 LDPC 디코더가 필요하나 기존 구현([4]의 baseline)은 420Mb/s에 그침.
3. 기반 알고리즘: [4]의 Min-Sum Algorithm(MSA) 기반 row-layered decoding([11] 기법)을 그대로 채택, CN을 `NPU`로 처리하며 `LNPU`(Local)/`GNPU`(Global)로 분리.
4. 핵심 기법 1단(소프트웨어 파이프라이닝): 레이어 내 블록 간 메시지 교환이 없다는 성질(`βI(u,w)=βI(u,w')`)을 이용해 GNPU/LNPU를 파이프라인 처리(2x 버전)하여 유휴 시간 제거.
5. 핵심 기법 2단(멀티코어 병렬화): 독립된 디코더 코어를 다수(최대 6개) 배치하고 프레임을 round-robin으로 분배, 고정 지연(fixed latency)으로 복잡한 handshake/buffering 없이 동작.
6. 부수 트릭/구현: 블록을 valid/invalid로 나눠 무효 블록(all-zero submatrix) 처리를 생략하는 `βI`(block index)/`βS`(block shift) 행렬 표현 사용.
7. 개발 도구: National Instruments LabVIEW CSDS의 알고리즘 컴파일러로 그래픽 고수준 기술을 VHDL로 자동 변환, rapid-prototyping.
8. 학습/최적화 절차: 없음(설계 파라미터 스윕만, DE/GA/NN 없음).
9. 결과: 코어 1개(420Mb/s)→6개(2476Mb/s, 200MHz)로 선형에 가깝게 확장, Slice 사용률 28%→97%; Pipelined(2x)는 Baseline 대비 자원 거의 동일하면서 처리율 1.5배(290→420Mb/s).
10. 한계: 정정능력/error-floor 등 디코딩 성능 개선은 다루지 않음(BER 그래프만 제시, 수치 서술 없음), Min-Sum 알고리즘 자체는 신규성 없음, 4x/8x/16x 버전은 미완성("currently being developed").

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum 기반 row-layered decoding, NPU(GNPU/LNPU) 처리 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC도 min-sum 기반이나 이미 보유 기법과 중복 |
| 2-layer 소프트웨어 파이프라이닝(레이어 내 독립 블록 병렬 처리) | 디코더 HW 모델(z=32, HCU, GT) | 병렬화 스케줄링 아이디어는 Prime ECC의 row 단위 병렬 구조와 유사하나 구현 방식(FPGA VHDL 컴파일)이 상이 |
| 다중 디코더 코어 병렬화(round-robin 프레임 분배) | `ecc_top.cpp` 시뮬 오케스트레이션 | 단일 코드베이스 관점에서 코어 복제/스케줄링은 시스템 레벨 이슈로 직접 대응 모듈 없음 |
| QC-LDPC H-matrix (802.11n base matrix) | `ecc_top.cpp` `Load_PCM()` | 부호구조 자체는 표준 규격 코드로 Prime ECC와 무관한 고정 매트릭스 |
| 무효 블록(all-zero submatrix) skip 표현 (βI/βS) | `GT.cpp` `Make_GT_HW()` | Graph Thinning과 유사한 edge selection 아이디어이나 이미 Prime ECC가 보유한 기법과 개념적으로 중복 |

> 적용 가치: 낮음~중간 — 정정성능 개선이 아닌 FPGA 다중 코어 병렬화·rapid-prototyping 툴체인 논문이며, 핵심 디코딩 알고리즘(min-sum, layered decoding)은 Prime ECC가 이미 보유. 멀티코어 처리량 확장 아이디어만 참고 가치.
