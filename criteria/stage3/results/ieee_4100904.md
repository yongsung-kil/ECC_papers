### B. 알고리즘 요약

1. AWGN 채널 상의 고rate(약 0.72~0.94) LDPC 부호, 길이 `n≈759~6890`, girth 6을 갖는 STS(Steiner Triple System)/OC(One-Configuration) 기반 부호.
2. 임의 코드에서 stopping/trapping set과 cycle 구조를 완전히 특징짓기는 매우 어렵다는 문제를 idempotent·symmetric Latin square라는 제한된 구조에서 해소.
3. 핵심 가정: parity-check 행렬 `H`가 Latin square의 순열행렬(permutation matrix) 블록으로 구성되며, Cayley Latin square는 `L(i,j) = (i+j) mod m` 형태로 정의.
4. 핵심 기법 1단: Latin square 내 `partial subrectangle`(각 행/열에 최소 2개 원소, 각 심볼 최소 2회 등장)이 코드의 stopping set과 1:1 대응함을 증명 - subrectangle 크기 `t`가 곧 stopping set 크기.
5. 핵심 식: `Theorem 3.4`로 소수 order Cayley Latin square의 stopping number가 최소 `m-1`(order `m`)임을 유도, subrectangle 부재를 통한 하한 보장.
6. 핵심 기법 2단: elementary trapping set(check node degree 1,2만 갖는 부분그래프)을 subrectangle의 심볼 반복 패턴별로 `(3,5)`, `(4,2)`, `(4,4)`, `(4,6)`, `(4,8)`, `(6,2)`, `(6,4)`, `(6,6)` 등으로 완전 분류.
7. 부수 기법: block-cycle의 `Fan sum`(순열 offset의 합)이 0이면 길이 `2k` cycle 존재함을 이용해 6-cycle을 class (a)~(d) 4종으로 분류·계수, order-modification(블록 크기 3→5 확장)과 구조적 column 제거(shortening)로 class (d) 6-cycle을 제거.
8. 최적화 절차는 별도 없음 - 폐형(closed-form) 조합론적 카운팅과 결정론적 구조 수정(order 확장, 특정 block-column 삭제)만 사용.
9. 결과: order-modification으로 동일 길이에서 6-cycle 수를 약 12배 감소시켰고(`dim5` vs `dim3`), 구조적 shortening된 (1026,897) rate 0.87 코드가 MacKay random 코드와 거의 동일 성능, error-floor 영역의 오류가 대부분 (3,5)/(4,8) trapping set에 기인함을 실측 확인.
10. 한계: HW 설계 전무, 결정론적 대수 구조(Cayley Latin square, 소수 order)에 의존해 일반 QC-LDPC 설계 파이프라인에 이식 어려움, 시뮬레이션은 AWGN·고rate·중간 길이(~1000~6900)에 국한.
