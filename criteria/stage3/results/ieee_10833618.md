### B. 알고리즘 요약

1. CCSDS 표준 TC(Telecommand) 통신에서 CLTU(Communications Link Transmission Unit)는 start sequence + N개 LDPC 코드워드 + optional tail sequence(TS)로 구성, 대상 코드는 (128,64) rate=0.5 QC-유사 LDPC(부록 A의 shifted-identity 기반 H).
2. 문제: BCH 코드는 bounded-distance 디코더로 TS(비correctable 패턴)를 이용한 종료 검출이 잘 되지만, LDPC의 확률적 반복 디코더(BP 계열)는 TS를 코드워드로 오인해 종료를 놓칠 위험(not-acknowledged termination, Pnat)이 큼.
3. 채널모델은 AWGN + BPSK, TS는 최대우도(ML) 디코딩 관점에서 모든 코드워드와의 최소 Hamming distance를 최대화하도록 설계.
4. TC rejection probability `PTCrej = Pmd + (1-Pmd)[PLDPC + (1-PLDPC)Pnat]`로 분해, 세 항(시작시퀀스 미검출 Pmd, 코드워드 디코딩 실패 PLDPC, TS 미인식 Pnat)을 각각 이론식·몬테카를로로 분석.
5. `Pnat`은 노이즈 섞인 TS가 디코더에서 우연히 코드워드로 수렴하는 확률로 정의되며, 표준 TS(랜덤화 안 됨, Hamming distance=15)는 특정 3개 코드워드로 자주 수렴해 error floor를 유발.
6. 핵심 개선안: 인포워드와 동일한 randomizer(LFSR)로 TS도 랜덤화해 새 TS `t'`를 만들어 최근접 코드워드와의 Hamming distance를 15→18로 증가, Pnat을 전 SNR 구간에서 크게 감소시킴.
7. 부수 분석: LLR-SPA, MSA(min-sum), NMSA(normalized min-sum, factor=0.8) 3개 디코딩 알고리즘 비교, 반복횟수(Nit=20 vs 100)가 CER과 Pnat에 상반된 영향(iteration 늘리면 CER↓이지만 Pnat↑)을 미침을 실증.
8. 최적화 절차 없음(학습 기반 아님); 코드워드 검색은 Stern's information set decoding으로 저중량 코드워드 확인.
9. 결과: 제안 TS 적용 시 Pnat이 전 SNR에서 대폭 감소, N=1(코드워드 1개)에서 MSA가 TC rejection probability 최저; N=10, 40으로 늘면 PLDPC가 지배적이 되어 성능 저하.
10. 한계: 순수 이론+시뮬레이션(HW 미설계), (128,64) 초단문 코드 1종 전용 검증, LDPC 디코딩 알고리즘/부호 자체는 기존 표준(LLR-SPA/MSA/NMSA) 그대로 사용해 정정성능이나 iteration 개선을 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TS(tail sequence) 최대 Hamming distance 설계 | 대응 없음 | CLTU 프레이밍/종료 검출 프로토콜로 Prime ECC의 디코더/부호설계 범위 밖 |
| TC rejection probability 이론 모델 | 대응 없음 | 통신 시스템 레벨 확률 분석으로 HW ECC 모듈과 무관 |
| LLR-SPA/MSA/NMSA 성능 비교 | `decoder.cpp` `CNU_Update_New_Mag()` | MSA/NMSA는 이미 보유한 min-sum 계열과 개념적으로 동일하나 본 논문은 알고리즘 개선이 아닌 기존 알고리즘 활용 비교일 뿐 |
| 반복횟수(Nit) 조정이 디코더 수렴에 미치는 영향 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 조기종료/최대반복 설계에 참고 가능하나 본 논문 목적(TS 오검출 방지)과 우리 목적(성능/latency)이 달라 직접 대응 낮음 |

> 적용 가치: 낮음. LDPC 코드/디코더 알고리즘 자체 개선이 아닌 CCSDS 위성 통신 프로토콜의 프레임 종료 검출 확률 분석이라 NAND HW ECC(Prime ECC 3.1)와 레이어가 근본적으로 다름.
