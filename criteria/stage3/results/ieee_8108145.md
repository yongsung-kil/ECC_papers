### B. 알고리즘 요약

1. 시스템: `N=4096, R=0.5` polar code, Arıkan Bhattacharyya 기반 frozen-bit 구성, BP(belief propagation) 반복 복호, GPU 32-bit 시뮬.
2. 문제: polar은 이론상 error floor가 없다고 증명됐으나 실측 BP 곡선에서 error floor가 관측됨 — 원인 규명 필요.
3. 모델/원인: LLR을 수치 안정성·HW bit-width 때문에 `LLRmax`로 clipping하는데, 이 saturation이 error floor의 주원인.
4. 핵심 기법: PE(processing element) 단위 L/R 메시지 갱신에 `boxplus` `f(L1,L2)=L1⊞L2`(식 4) 사용, 신호처리로 min-approximation 채택.
5. 핵심 식: 상대 error floor 지표 `NE(LLRmax)=평균(BER_LLRmax/BER_ref)` (ref=LLRmax=100) — 단일값으로 floor 정량화, LLRmax↑ 또는 N↓일수록 NE→1.
6. 관찰: 소수 iteration 만에 trapping set에 빠져 clipping에 도달하므로 max-iteration 증가로는 해결 안 됨 → 알고리즘 수정 필요.
7. 완화기법 4종: (A) oscillating 비트 sign guessing + ±LLRmax 할당, (B) 수렴 실패 시 virtual noise `nv~N(0,σv²)` 추가, (C) boxplus 출력 α(=0.9375)<1 scaling, (D) multi-trellis(다른 factor-graph realization) BP.
8. 검증 절차: Eb/N0=5dB test set(고 LLRmax 성공·저 LLRmax 실패 케이스)에서 각 수정의 success rate 측정.
9. 결과: 성공률 A(1~3비트) 0.9585~0.9911, B 0.9925, C 0.8889, D 1.0; LLRmax↑ 시 error floor 소멸(BER 10^-9 이하), floor는 code property 아닌 decoder property로 확인.
10. 한계: HW 미설계(GPU/CPU 시뮬만), throughput/gatecount 없음, polar factor-graph 전용 알고리즘, 부적절 frozen-bit 선택도 floor 유발 가능.
