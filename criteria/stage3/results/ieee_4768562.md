### B. 알고리즘 요약

1. 코드는 Gallager의 regular LDPC 구성을 변형: 여러 single-parity-check row를 묶어 "super code"로 구성, 예제는 rate `R=1/2`, length `n=3990`(`ν=6, ω=3`, `n=3326` 실험 코드는 `z=665` 파라미터로 구성).
2. 표준 BP는 각 check node가 개별 sub-decoder이며 긴 cycle에서 메시지 교환 비효율로 수백~수천 iteration이 필요한 문제를 해결하려 함.
3. super code의 parity-check submatrix `H0`을 특정 인덱스 집합 `I={I0,...,Iz}`로 구성하면, 각 super code가 사이클이 없는 2-state trellis로 표현 가능하다는 것이 핵심 가정.
4. 각 super code를 BCJR 알고리즘으로 복호하는 iterative decoder를 구성: ω개의 동일한 2-state trellis sub-decoder가 extrinsic LLR `γe^(i)`를 교대로 주고받음.
5. 입력 LLR은 `λ(i) = γ + (Σ_{j≠i} γe^(j,p) Πj) Π_i^{-1}`로 계산되며, 이전 iteration의 다른 sub-decoder들의 extrinsic 정보를 a priori로 사용.
6. trellis 두 종류(type I/II, type II는 crossover branch 제거된 pruned 버전)로 recursion을 단순화: `αk = αk−1 ⊞ λk`, `βk = βk+1 ⊞ λk+1`, `[γe]k = αk−1 ⊞ βk` (⊞은 log-domain 연산자).
7. type II trellis에서는 ⊞ 연산이 단순 덧셈(`αk = αk−1 + λk`)으로 더 축소되어 저장량·연산량을 기존 BCJR 대비 최소 2배 절감.
8. 별도 학습/최적화 절차 없음(구성 파라미터는 수동 설계).
9. 결과: 제안 구성+trellis 디코딩이 Gallager 코드+표준 BP 대비 수렴속도 약 2배, 80회 iteration 시 BER 소폭 우수, replica shuffled BP와 유사 성능이나 complexity는 더 낮음(Table I, 예: 덧셈 36 vs standard BP 27).
10. 한계: 순수 시뮬레이션/이론 분석뿐이며 HW 구현 없음, 코드 구성이 표준 QC-LDPC와 다른 특수 구조를 요구해 기존 부호와의 호환성 부족.
