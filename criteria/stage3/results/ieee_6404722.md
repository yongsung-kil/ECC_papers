### B. 알고리즘 요약

1. 대상: BEC(이진소거채널)의 spatially-coupled LDPC = terminated LDPC convolutional(LDPCC) 앙상블 `(l,r,L,M)`(예 (3,6),(5,10)), 길이 n=M·L.
2. 문제: SC-LDPC는 BP threshold가 MAP에 근접하지만 유한길이에서 rate loss ∝1/L 때문에 큰 L·M 필요 → 코드길이·복호 latency 과다.
3. 채널모델: BEC, TEP는 Tanner graph 상의 peeling-type 알고리즘(degree-1/2 CN 순차 처리).
4. 기법1(TEP): BP의 peeling에 더해 degree-2 check node 제거 단계(Alg.1 Step13-16) 추가 — pairwise marginal로 두 변수의 equality/inequality를 추론해 변수 `V_r`이 `V_o`의 연결을 상속.
5. 의미: degree-2 CN 제거가 결국 degree-1 CN을 생성해 BP가 막히는 stopping-set(SS)을 깸 → BP 대비 WER 개선.
6. 구조: LDPCC protograph `C(l,r,L)=T⊗b_l,r`(식(1)), copy-and-permute로 코드 샘플 생성, half-terminated 변형 `D`로 rate loss 절반 감소(경계 한쪽만 저차 CN).
7. 복잡도: degree-2 CN 제거 시 행렬이 sparse 유지되어 O(n), 평균 변수차수 `Λavg`가 감소함을 실측(평균 <10 binary sum/제거).
8. 윈도우(wTEP): LDPCC의 공간제약을 이용, 윈도우 크기 W 슬라이딩으로 연속복호 → latency와 성능 trade-off(Table I~III).
9. 결과: (5,10,20,M)에서 TEP가 M=256/512를 BP의 512/1024 수준으로 → 약 50% 길이 감소; wTEP가 wBP WER을 큰 폭 개선(예 ε=0.45, W=20에서 0.0025 vs 0.0233).
10. 한계: BEC 전용(비트 error·soft 정보 없음), SC/convolutional 구조 전제, HW 미설계, block QC-LDPC 아님.
