### B. 알고리즘 요약

1. 시스템: binary LDPC(BPSK+AWGN), flooding schedule, regular 코드 2종(R=0.841 N=2048 dc32/dv6, R=0.5 N=8000 dc6/dv3), 채널·메시지 3/4bit, 내부해상도 wφ=8.
2. 문제: MIM(상호정보 최대) 디코더의 비균일 임계 양자화는 고해상도 비교기·임계값 메모리를 요구해 HW 복잡도가 기존 min-sum보다 큼.
3. 접근: computational domain 방식(translation table→가감산→단일 양자화)을 유지하되 비균일 양자화를 대칭 균일 양자화로 교체.
4. 핵심기법: 균일양자화 `Q(y)=sgn(y)·min(|y|/2^r+1, 2^(w-1))` `(6)` — r-bit right shift + clipping만으로 구현, 비교기·임계 메모리 제거.
5. 스케일링: translation table을 `φ∆(t)≈(1/∆)φ(t)`로 정수화 `(3)`, (∆,r) grid search로 `max I(X;T)`, 경계간격 ∆2^r 조정.
6. VN 대칭화: 두 종류 VN(음/양 LLR 합산)으로 `p(xv,y)` 대칭 강제 `(4)` → magnitude만 양자화·translation 테이블 메모리 절반.
7. CN 옵션: computational-domain `(1)` 또는 최소근사 `(7)` `tc=∏sgn·min|tv|`(translation table 불필요, 최저 복잡도).
8. 분석: 이산 density evolution으로 균일 vs 비균일 MI 비교 — CN 손실 0.0443→0.0441(0.0003), min-approx는 0.0407로 더 큰 손실.
9. 결과: 균일이 비균일 대비 FER 손실 <0.01dB, 3-bit 균일이 4-bit OMSQ와 유사 성능, min-approx CN은 comp-domain 대비 0.015~0.025dB 손실.
10. 한계: 정정능력 개선이 아니라 동일 성능에서 HW 복잡도 절감이 목적, HW는 비교/게이트 카운트 수준(FPGA/ASIC 미검증), <10^-5에서 cycle 효과는 DE 미반영.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 균일양자화 Q(y)=bit-shift+clipping, VN 양자화 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | VN magnitude 양자화를 shift 기반으로 단순화 |
| translation table φ 정수 스케일링, iteration별 LLR | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | LLR 테이블 정수화·대칭화 방식 참조 |
| CN 최소근사 (min1/min2) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유 min-sum과 동일 — 균일양자화만 신규 |
| 채널 양자화 소프트리드 | channel.cpp Set_LLR_Th(), Set_R_Offset() | 3/4bit soft-read 양자화 경계 설계 |
```

적용 가치: **중간** — 성능 개선이 아니라 동일 성능에서 양자화 HW 복잡도를 절반 줄이는 기법이라, Prime ECC가 이미 magnitude 양자화 테이블·min-sum을 보유한 점을 고려하면 순수 신규 이득은 제한적이나 3/4bit soft LLR 테이블 단순화(면적·라우팅) 참조로는 유용하다.
