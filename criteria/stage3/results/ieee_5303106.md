### B. 알고리즘 요약

1. DMB-TH(중국 지상파 DMB) 표준의 QC-LDPC(127×127 순환행렬, rate 0.4/0.6/0.8)를 배경으로 하되, 실제 시뮬레이션은 (3048,7493) irregular LDPC(λ(x)=0.1273x^6+0.8727x^7, ρ(x)=0.3382x^2+0.2168x^3+0.4x^10)에서 수행.
2. 풀려는 문제: 기존 uniform quantizer(q:f 비트, f=소수부)는 좋은 성능을 위해 정밀도(f)를 늘려야 하고 이는 메모리 증가로 이어짐; non-uniform quantizer는 메모리는 줄지만 lookup table 저장 비용이 큼 - 두 방식의 절충으로 all-integer(소수부 없이 정수부만 사용) 양자화를 제안.
3. 채널모델: BPSK+AWGN, `LLR = (2/σ²)·r_v`, normalized/offset min-sum 두 알고리즘 모두에 적용.
4. 핵심 기법 1단: 채널 초기 메시지에 스케일링 팩터 `γ`를 곱해 `Lqv(0)' = 2·r_v/σ²·γ`로 값 범위를 정수 양자화에 유리하게 확대(offset min-sum의 β도 `β' = β·γ`로 동일 스케일).
5. 핵심 식 의미: min-sum은 채널정보를 스케일링해도 성능이 불변(정규화 특성)함을 실험으로 확인(Fig.1) → 스케일 자유도를 양자화 최적화에 활용 가능.
6. 핵심 기법 2단: 일정 iteration 수 이후 variable/check node 메시지 분포가 좁은 범위로 수렴하는 통계(Fig.2, 6회 반복 후 대부분 [-10,+10])를 이용해 특정 iteration `T_shift`에서 메시지를 `θ`배 축소(스케일다운) - `Lqv(i)=Lqv(i)·θ (t=T_shift)`.
7. 부수 트릭/HW: 스케일업/다운을 단순 shift 연산으로 구현(곱셈기 불필요), γ·T_shift 값은 Eb/N0별로 lookup table(Table 1)에서 참조; variable node 6-bit, check node 4-bit 정수로 양자화(부호비트 포함).
8. 별도 학습/최적화 절차 없음(파라미터는 사전 시뮬레이션 기반 lookup table로 고정).
9. 결과: (3048,7493) 코드에서 normalized min-sum 기준 improved Q(6,4)가 부동소수점보다 우수, offset min-sum 기준 improved Q(6,4)가 부동소수점에 근접(Eb/N0<1.5dB에서 Q(6,3)/Q(6,4)도 근접).
10. 한계: max iteration 20회 고정 시뮬레이션 결과만 제시, 실제 gatecount/throughput 등 합성 결과 없음(HW 아키텍처는 블록도 수준 제안), γ/T_shift 최적화 절차의 구체적 도출 방법(자동화 여부) 불명확.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LLR 정수 양자화(6-bit VN, 4-bit CN, all-integer) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | Prime ECC도 iteration별 LLR threshold 테이블 기반 적응형 양자화를 사용 중이므로 직접 비교 가능 |
| iteration별 채널메시지 스케일다운(shift 기반) | `decoder.cpp` `Get_VNU_Table_Idx()`, `ecc_data.h` `PARAM_LLR` | Prime ECC의 "iteration 구간별 LLR threshold 테이블"과 개념적으로 유사한 적응형 스케일링 |
| normalized/offset min-sum CN 업데이트 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 min-sum 변형을 보유(§3), 논문은 표준적 normalized/offset min-sum이라 중복 기여 |
| 채널 초기 LLR 스케일업(γ) | `channel.cpp` `Set_R_Offset()` | 초기 soft-read LLR 스케일 설계와 관련, 양자화 비트폭 절감 관점에서 참고 가능 |

> 적용 가치: 중간 - min-sum 알고리즘 자체는 Prime ECC가 이미 보유해 중복이나, all-integer 양자화 + iteration별 shift 기반 스케일다운 아이디어는 LLR 테이블/soft-read 비용 절감 관점에서 참고할 만함.
