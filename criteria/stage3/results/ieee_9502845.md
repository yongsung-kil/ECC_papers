> 총평: ECC 여유를 활용한 controller단 endurance 데이터 셰이핑 기법으로 LDPC 부호·디코더 자체는 손대지 않아(오히려 iteration↑) binary LDPC ECC 이식 관점에서는 이식성 하.

### B. 알고리즘 요약

1. 시스템: SLC/MLC/TLC/QLC NAND, logical page 단위 별도 인코딩, soft decoding ECC(LDPC) 전제(그림1·2 시스템 모델).
2. 문제: P/E cycling wear-out을 줄이려는 data shaping은 보통 LUT/가변장부호로 오버헤드·복잡도가 큼(AEC, reverse compression, direct shaping 등).
3. 핵심 아이디어: 초기 수명엔 RBER이 낮아 ECC 여유가 있으므로, parity를 오류정정과 endurance 복조에 **공동 사용** → 별도 rate loss·LUT 불필요.
4. 셰이핑(변조): 각 logical page에서 `i ≡ 0 (mod ns)`인 i번째 '0' 비트를 뒤집어 프로그램 셀('0') 비율을 낮춤 (`ns` = shaping parameter).
5. 복조 위치추정: hard estimate `ŝ`의 zero 개수 `C_k(ŝ)`로 뒤집혔을 수 있는 인덱스 집합 `I_r = {l_r+1,…,l_r+m_r-1}` 추정.
6. LLR 부여: DMC 가정하 max-log 근사로 `j∈I`는 `L(j) ≈ -log(m_r-2)`, `j∉I`는 `L(j) ≈ Lch(j) + (-1)^{ŝ_j} log(|B_ŝ|/|B_š|)` (유효 셰이핑코드워드 개수비).
7. 실무: 오류전파 제한 위해 codeword를 chunk(=128bit) 단위 독립 적용, circular-shift interleaver로 wear-leveling, RBER threshold 초과 시 controller가 셰이핑 disable.
8. 검증: TLC balanced Gray, density evolution + EXIT chart로 이론 분석(regular LDPC `rate 8/9`, CN degree 3 / VN degree 27), 채널 왜곡(parasitic coupling·프로그램 부정확·RTN·retention) 모델링.
9. 결과: page별 최적 `ns*`(예 lower page 1K P/E에서 `ns*=16`) 도출, endurance gain 산출; 단 평균 복호 iteration은 셰이핑 없을 때보다 증가(Table II), `Lmax=5`·max iteration 20·2KB codeword.
10. 한계: HW 미설계, 이론/시뮬 위주, LDPC를 블랙박스로 사용해 부호·디코더 개선 아님, endurance 이득이 ECC 정정능력에 종속(강한 코드일수록 이득↑).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 데이터 셰이핑(비트 flip) 변조 | 대응 없음 | controller단 endurance 셰이핑은 LDPC ECC 파이프라인 밖 계층 |
| 뒤집힌 비트 max-log LLR 부여 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()`, [ecc_data.h](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `PARAM_LLR` | soft 입력 LLR 생성·양자화 지점에 개념적으로 닿으나 부호별 셰이핑 규칙이 추가로 필요 |
| soft decoder(LDPC)로 정정+복조 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 디코더는 그대로 재사용(변경 없음), 입력 LLR만 셰이핑 인지형으로 바뀜 |
| RBER threshold로 CRC 성공 판정·모드전환 | [full_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) | 셰이핑 on/off 판정에 조기종료 CRC 성공정보 재사용 가능 |

적용 가치: **낮음** — LDPC 부호설계/디코더/HW를 개선하지 않고 이를 블랙박스로 활용하는 상위 endurance 셰이핑 기법이라, Prime ECC에 떼어 넣을 알고리즘적 요소가 거의 없다.
