### B. 알고리즘 요약

1. 시스템/코드: AWGN+BPSK 채널, (128,64)·(256,128) polar code (rate 0.5). polar BP를 대상.
2. 문제: polar BP factor graph는 stage 수가 code length에 종속(`n=log2 N` stage) → latency 큼. 기존 dense NN 디코더[5,6]는 복잡도가 지수적, edge-weight NND[7]는 depth가 iteration·length에 종속되어 latency 김.
3. 모델: polar generator graph를 `H`(식 1)로부터 LDPC-like bipartite Tanner graph로 변환 후 pruning[9]으로 sparse화, 이를 unfold하여 feed-forward DNN 구성(`2T` hidden layer, hidden node=edge).
4. 핵심기법1: SPA를 min-sum 근사로 대체 - odd layer는 CN 연산 `x=sign·min|x|`(식 3, 무파라미터), even layer는 VN에 학습 가중치 `x=Lv+Σ w·x`(식 4), 출력은 sigmoid(식 5).
5. 핵심식: 단일 가중치화(식 6) - 모든 CN→VN edge에 통합 스칼라 `w` 하나만 사용, 학습 파라미터를 대폭 축소(RNN weight-sharing보다 더 축소).
6. 핵심기법2: `w* = argmin L(x,o)` - cross-entropy loss(식 8) 최소화하는 단일 스칼라 학습. (128,64)에서 `w*≈0.83`, (256,128)에서 `≈0.82`로 수렴.
7. 부수트릭: zero-message가 CN min을 0으로 만드는 문제 회피 위해 sign(0):=1 정의, 파라미터 초기화 `N(μ=1,σ=0.1)`.
8. 학습/최적화: PyTorch, mini-batch SGD+Adam, Lr=0.001, all-zero codeword 학습셋, 10 iter unfold(20-layer), 600 epoch.
9. 결과: SNND-FF가 SPA와 동등 BER, MS 대비 약 0.1~0.5dB 이득. 단일가중치 SNND도 거의 동등. iteration당 latency=2(code length 무관, `log2 N` 감소), NND[7] 대비 약 60% 연산 감소.
10. 한계: polar code 전용, HW·gatecount·throughput 없음(시뮬만), SPA 대비 고SNR 0.1dB 열화, NN 학습 의존.
