### B. 알고리즘 요약

1. 대상은 protograph 기반 spatially-coupled(SC) 및 multi-dimensional(MD) LDPC 부호로, base matrix를 partitioning(component matrix로 분할)한 뒤 circulant power로 lifting하는 2단계 구성 절차를 다룸.
2. 기존 finite-length 최적화기(FL-AO)는 SC/MD 부호의 partitioning·lifting·relocation 자유도가 커질수록 연산량이 급증해 비효율적이라는 문제를 해결하고자 함.
3. 목적함수 `C(x) = w4*C4(x) + w6*C6(x) + w8*C8(x)`는 partitioning/lifting 배치 `x` 하에서 활성 상태로 남는 cycle-4/6/8 후보 개수의 가중합.
4. 핵심 기법은 Gibbs sampling 기반 MCMC(MC2)로, `P(x) ∝ exp(-βC(x))`인 분포에서 표본을 추출하며 d-tuple 단위로 조건부확률 `T(x';x|ν)`을 갱신해 상태를 전이.
5. `β`는 목표 accept rate `fT`에 맞춰 PID 컨트롤러로 적응 조정되며(simulated annealing 형태), 이는 탐색 초반 광역 탐색→후반 탐욕적 최적화를 가능케 함.
6. Detailed balance(식 14)로 `P(x)`의 불변성과 비주기성을 증명(Lemma 2), 목적함수 최소값과 정상분포 기대값의 차이를 상한(Theorem 1)으로 도출.
7. 부수적으로 목적함수 분포를 Gaussian으로 피팅(`μ_A(β)`, `σ_A(β)`)해 최소 cycle 수·필요 반복횟수·Markov chain recurrent class 크기를 사전 추정.
8. GD(gradient-descent, 기존 논문[19]) 기반 초기화와 결합하면 더 나은 recurrent class로 수렴; ordered overrelaxation으로 Gibbs sampling 수렴 10~20배 가속.
9. 결과: cycle-6 최대 48.13%, cycle-8 최대 22.37% 감소, 동일 카운트 도달 시 연산량 최대 3.32 orders of magnitude 절감, FER/FER 성능 최대 1.61 orders of magnitude 개선(SC Code 3에서 literature 코드의 error floor를 size-10 stopping set 제거로 해소).
10. 한계: 순수 소프트웨어 오프라인 부호설계 알고리즘(MATLAB)이며 HW 구현·양자화·디코더 구조 변경은 다루지 않고, 검증은 시뮬레이션(BEC/AWGNC)에 한정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC/MD-LDPC partitioning·lifting 최적화 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 QC-LDPC(비-SC) H-matrix를 로드만 하며, 이 논문의 MCMC 재설계 절차와는 구조적으로 무관 |
| cycle-6/8, UAS/UTS 최소화 목적함수 | 대응 없음 | Prime ECC 코드베이스는 부호설계 알고리즘 자체를 포함하지 않음(고정 H-matrix 사용) |
| 디코딩 성능 검증(peeling/sum-product decoder) | `decoder.cpp` `LDPC_Decoder()` | 검증용 디코더 알고리즘은 다르지만(min-sum) 부호 품질이 디코더 성능에 영향을 준다는 관점만 공통 |

> 적용 가치: 낮음 — SC-LDPC 전용 부호설계 알고리즘으로 Prime ECC 3.1의 고정 QC-LDPC(non-SC) 구조 이식 난이도가 매우 높고(§4 "부호설계" 행 참조), HW/디코더 레이어 기여가 전무함.
