### B. 알고리즘 요약

1. 대상: 2D/3D NAND (MLC 4-state, TLC 8-state) SSD 신뢰성 실험용 소프트웨어 에뮬레이터, FEMU(오픈소스 SSD emulator) 확장 구현.
2. 문제: 기존 emulator는 셀을 비트로 관리하고 random bit-flip으로 오류 주입 → gradual threshold-voltage shift(retention/disturbance)와 read-retry(reference voltage 조정) 실험 불가.
3. 모델: 셀 상태를 임계전압 범위(analog)로 관리, `Vth = Vmean + Cloc·α` (식1), `Cloc`=분포 내 상대 위치, state별 Gaussian random으로 strong/weak cell 표현.
4. Noise 3종: endurance `Vnoise=MBE·Wend+Cloc·β`(식2), disturbance `MBR·Wdis+Cloc·γ`(식3), retention `MPW·Wret+Cloc·ω`(식4). metadata=블록당 erase수 `MBE`/read수 `MBR`, 페이지당 last-write `MPW`.
5. 식의 의미: 첫 항=평균 전압 shift(weight×metadata), 둘째 항(Cloc×β/γ/ω)=noise 시 분포 표준편차 확대 반영.
6. 파라미터 추정: 8개 파라미터(Vmean,Wend,Wdis,Wret,α,β,γ,ω)를 실측 MLC/TLC voltage-shift 데이터에 polynomial regression(ML)으로 state별 도출.
7. 저오버헤드 알고리즘: 주기적 전 셀 업데이트(초기판 0.46 ms/period) 대신 read 시점에만 noise·shift 계산; 셀은 절대전압 대신 `Cloc`(상대위치)만 저장.
8. 메모리: `Cloc` 2 bytes/셀 → MLC 8배·TLC 5.3배 메모리 증가 (Huffman식 인코딩은 future work).
9. 결과: 실측 대비 voltage-shift 차이 최대 4.3%(3K/4K P/E, 100K~1000K reads, 최대 1년 retention); 사용성으로 BCH ECC(7 parity/512B)+read-retry의 UBER 저감을 reference voltage 조정하며 시연(기존 emulator로 불가).
10. 한계: ECC/디코더 알고리즘 자체 기여 아님, FPGA 검증·QLC는 future work, ECC 예시는 BCH뿐(LDPC 부호/디코더 미포함).

| read-retry reference voltage 조정 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()` | soft-read/HD read offset 개념과 유사, 알고리즘 이식보다 검증 시나리오 참고 |
| ECC(BCH/LDPC) 성능 평가 프레임 | 대응 없음 | ACE는 외부 검증 툴, Prime ECC 디코더/부호설계 내부에 넣을 알고리즘 요소 없음 |
| LDPC 디코더/부호설계 기여 | 대응 없음 | 본 논문은 LDPC 알고리즘 기여 없음 (BCH 예시만) |

적용 가치: **낮음** — NAND에 직접 관련되나 ECC 알고리즘 논문이 아닌 신뢰성 실험용 emulator라 Prime ECC 부호/디코더/HW에 이식할 알고리즘 없음. 다만 Vth/noise 모델은 우리 채널·soft-read 검증 환경 강화용 참고 가치.
