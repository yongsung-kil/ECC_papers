### B. 알고리즘 요약

1. 대상: 임의 length/rate/circulant size `N`/VN-degree 분포의 binary QC-LDPC 부호 구성, PCM은 `J×K`개 `N×N` circulant 배열 (single- 또는 multi-weighted).
2. 문제: 기존 QC-PEGA는 CES(cyclic edge set) 배치 시 undetectable CMVC(cyclic-edge-set-minimum-virtual cycle)를 놓쳐 불필요한 4/8-cycle(multi/single-weighted) 생성 -> girth·error 성능 저하.
3. 핵심개념: CMVC를 "아직 안 놓은 CES가 만들 수 있는 최소 cycle"로 정의, 이를 정확 계산하려면 a posteriori test(CES 추가->girth 계산->제거, BFS `O(D)`) 필요.
4. 핵심기법 1 (MM-QC-PEGA): 각 stage에서 multi-edge local girth `g_vc^(r,G)`를 최대화하는 CN을 선택(edge-trial `r`), a posteriori test로 모든 CMVC metric 정확 계산 -> short cycle 회피, 복잡도 `O(m^r·D^2/N)`.
5. 핵심기법 2 (GCD-approximation): CMVC를 δ(t)=`f_{cπ,vj}+1` 기반 케이스 (7)~(10)식으로 근사, `gcd(N,t)`로 3개↑ virtual edge 통과 cycle 처리 -> a posteriori test 없이 short CMVC 탐지, `O(N)`.
6. G-MM-QC-PEGA: GCD근사로 a posteriori test를 대체, 복잡도 `O(m^r·D + m^{r-1}·D^2/N)`로 감소 - QC-PEGA와 MM-QC-PEGA 사이 tradeoff. undetectable CMVC 최소길이 multi/single = 8/10.
7. 핵심기법 3 (PEG-masking): base PCM `H`를 fully-connected로 보고 MM-QC-PEGA로 masked matrix `M`을 직접 구성 (edge를 H의 부분집합으로 제한) -> 기존 masking [3]-[10] 대비 유연·저복잡.
8. 최적화 지표: girth, ACE spectrum, VNLGD(VN-local-girth distribution)로 코드 품질 사전 평가.
9. 결과: irregular (576,288) N=24 및 VN-degree-3 (1024,512) N=27, masked (1752,876) 예제에서 QC-PEGA 대비 ACE spectrum·VNLGD·FER/BER 개선(BPSK/AWGN, SPA 복호); trapping-set 유발 cycle-structure((12,2)(10,4) 등) 회피가 error-floor에 유리.
10. 한계: HW·throughput·복잡도(gate) 미제시, 시뮬(SPA)만, offline 구성 알고리즘이라 디코더 자체 개선 아님, rate 예제가 ~0.5로 NAND high-rate(~0.9)와 상이.
