### B. 알고리즘 요약

1. 대상은 IEEE 802.11n irregular QC-LDPC (rate 1/2·2/3·3/4·5/6, length 648/1296/1944, sub-block Z=27/54/81)용 fully-pipelined 디코더.
2. 문제: 고병렬 LDPC 디코더의 배선(interconnect)·메모리 비트·동적전력이 과다 -> 라우팅 오버헤드 최소화가 목표.
3. 복호는 layered (row-layered) min-sum: flooding 대비 약 2배 빠른 수렴을 이용.
4. CN 처리는 3-stage pipeline: marginalization(자기메시지 제외) -> Min1/Min2 검출 + sign XOR -> `α_ij = sign × min|β_ij|` 계산(식 7).
5. VN은 CN 입력 soft bit + 현재 CN 출력을 더하고 이전 CN 출력을 빼서 exclusion 처리(RAM 내부 저장), 별도 VN 라우팅 최소화.
6. 핵심 HW 트릭: 정방향/역방향에 **하나의 logarithmic barrel shifter permutation network 재사용** — CN 출력이 이미 업데이트된 메시지이므로 입력에 직접 append, layer 간 회전차만큼만 재회전.
7. small check-node granularity: CN이 클록당 입력 1개만 처리 -> block RAM 1개/CN(총 81개), FIFO depth = Ncols/Ninputs로 라우팅·복잡도 축소.
8. CN parallelism P=81(최대 sub-block), CN을 27개씩 3그룹으로 나눠 작은 codeword에서 idle 그룹 clock-gating으로 전력 절감.
9. 결과: 6-bit(s4.1) 고정소수점, N=648 6 iteration에서 부동소수 대비 손실 <0.2dB. 2-network 대비 slice/LUT 5%↓, 동적전력 19%↓, throughput 12%↑(최대 281Mbps), path delay 1.3ns↓.
10. 한계: 802.11n 부호·min-sum 고정으로 정정능력/error-floor 개선 없음, FPGA(Virtex-6)만 검증, gatecount·ASIC 수치 없음.
