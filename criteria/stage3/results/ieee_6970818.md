### B. 알고리즘 요약

1. 대상은 고속링크 per-flow 측정용 counter braid(CB) - 여러 layer의 counter가 flow size를 "on-the-fly"로 압축하는 sparse-graph 구조로, ECC 부호가 아니다.
2. 문제: 기존 CB의 반복(message passing) 복호 threshold가 낮아 압축률/복원 성능이 제한됨.
3. 모델: 각 layer는 left-regular bipartite graph `Gl`, flow node degree `k`, counter node degree는 Poisson(평균 `γl`), overflow 시 상위 layer counter를 modulo 증가.
4. 핵심 기법: spatial coupling을 CB에 적용(SC-CB) - `N`개 flow-node 그룹을 위치 `{1..N}`에 배치하고 window `w`로 band 구조 결합, 경계 socket 제거로 termination.
5. 핵심 식: single-layer density evolution `x(t+1)=ε[1-ρ(1-(1-ρ(1-x(t)))^{k-1})]^{k-1}` (식1)에 coupling matrix `A`를 넣어 SC recursion(식2)로 확장.
6. odd/even iteration에서 flow update 규칙이 달라(min/max 교대) SC recursion이 4중 합 구조가 되고, 이는 표준 coupled scalar recursion framework에 안 맞음.
7. EBP EXIT curve와 MAP EXIT curve로 threshold saturation을 해석, Maxwell curve와 비교.
8. 최적화: `γ` 최적화로 entropy lower bound 대비 rate gap을 `N`의 함수로 계산.
9. 결과: SC-CB가 uncoupled 대비 반복 복호 threshold 향상, `N,w` 증가 시 fixed 값으로 수렴(예: (N,w)=(32,5)와 (128,10) 거의 동일). single-layer SC-CB는 CS로 볼 때 random Gaussian + ℓ1 대비 sparse 영역에서 우수한 undersampling-sparsity trajectory.
10. 한계: 순수 이론/수치 해석, HW 미설계, NAND/LDPC ECC와 무관한 counting/CS 응용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Counter Braid / spatial coupling 구조 | 대응 없음 | ECC 부호·디코더가 아니라 per-flow counter 구조 |
| Density evolution / EXIT 해석 | 대응 없음 | 이론 해석 도구, Prime ECC 코드 모듈과 무관 |
| message passing (min/max 교대) | 대응 없음 | CB 전용 update 규칙, binary LDPC min-sum과 다름 |
```

적용 가치: **낮음** - NAND binary LDPC ECC가 아닌 네트워크 flow counting/compressed sensing 구조로, 어느 모듈에도 이식 대상이 되지 않는다.
