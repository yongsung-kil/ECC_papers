### B. 알고리즘 요약

1. 대상은 GF(2^m) RS `(N,K)` 부호(예 (31,25),(63,55),(127,107),(255,239))를 이진표현 PCM `Hb`((n-k)×n, n=N·m)로 변환한 것; BPSK/AWGN 가정.
2. 문제: `Hb`는 밀도 ~0.5로 4-cycle 다수라 직접 BP 부진; 기존 ABP(행렬적응)는 LRVN 업데이트 부족·MRVN 오류 정정난.
3. 핵심모델: VN을 신뢰도로 정렬해 저신뢰(LRVN)는 systematic(weight-1), 고신뢰(MRVN)는 고밀도 부분에 배치.
4. 핵심기법1 NP-RBP: flooding 대신 edge-wise IDS, 잔차 `rC2V(i,j)=|m_pre - m|`가 큰 edge를 우선 갱신.
5. 의미: 잔차가 큰 저신뢰 LRVN이 한 iteration 내 다중 갱신(중요 메시지 우선)으로 신뢰도 급상승, LRVN↔MRVN swap 촉진(0.26→15.70개).
6. 핵심기법2 nested-polling: VN 순차 방문 후 각 VN의 이웃 CN을 라운드별로 폴링, MRVN 상관 누적 억제.
7. 부수트릭 reset: iteration간 LLR을 0으로 리셋해 연속 Tanner graph간 메시지 상관 누적 차단.
8. 부수트릭 bit-flipping(BF): 매 iteration 시작 시 MRVN 대상 전처리 flip, 만족 CN 수 급증 여부로 flip 위치 판정(최대 λ=3).
9. 결과: (255,239)에서 NP-RBP-BF가 ABP 대비 ~0.4dB 이득, ML bound와 격차 ~0.25dB@FER 2×10^-3; 짧은 부호는 near-ML.
10. 한계: HW 미설계, 시뮬만; 매 iteration 행렬적응+대수적 HDD+최대 4000 iteration으로 복잡도·latency 큼.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| edge-wise IDS(잔차 기반 스케줄) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 스케줄 아이디어는 닿으나 sparse min-sum 구조와 상이, 이식가치 낮음 |
| C2V 메시지 계산(tanh/BP) | decoder.cpp C2V_Cal() | 우리 min-sum과 달리 full-tanh BP, 대응 부분적 |
| 조기종료(syndrome/valid codeword) | partial_CRC.cpp / full_CRC.cpp | 우리는 CRC 조기종료 보유, 논문은 syndrome+HDD |
| 행렬적응(PCM 재구성) | ecc_top.cpp Load_PCM() | 우리 QC-LDPC PCM 고정, 매 iteration 적응 개념 부재 |
| bit-flipping post/pre-proc | 대응 없음 | RS 이진영상 특화, 대응 없음 |
| RS/non-binary 부호구조 | 대응 없음 | Prime ECC는 binary QC-LDPC, RS GF(2^m) 대응 없음 |
```

적용 가치: **낮음** — RS 이진영상 고밀도 PCM 전용 기법이라 sparse binary QC-LDPC Prime ECC와 부호·디코더 구조가 근본적으로 달라 이식 실익이 적다.
