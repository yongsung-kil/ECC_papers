### B. 알고리즘 요약

1. 대상은 5G URLLC용 QC-LDPC 디코더로 NR base graph 2(BG2, 42 layers × 52 columns) PC matrix를 single-layer로 복호.
2. 문제: 전통적 single-layer 구조는 VN↔CN 라우팅용 cyclic shift module이 항상 pair(정방향+역방향 2개)로 존재해 LUT 자원을 크게 소모하고 data dependency를 유발.
3. 채널/복호: QPSK·AWGN에서 LLR `Ln=2yn/σ²` 초기화, normalized min-sum(`α=0.625`)으로 CNU=`α·∏sign·min(|·|)`, VNU=합산.
4. 핵심 기법: [15] 개선하여 shift module 2(역변환)를 제거하고 FIFO 크기를 축소, LLR 저장 위치 차이를 보상하는 새 shift value를 신규 shift matrix로 생성.
5. 핵심 식: 첫 iteration용 `s_{n,m}=(s_{n,m}−s_{n-1,m}) mod Z`, 잔여 iteration용 `r_{n,m}`(마지막 shift `g_m` 반영), 최종 hard decision용 `h_m=(−g_m) mod Z`.
6. shift matrix 3종(첫 iteration / 잔여 iteration / 최종 판정)을 PC matrix로부터 구성, [15] 대비 추가 저장 행렬은 1개뿐.
7. 부수 디테일: shift module은 좌/우 시프트 후 XOR merger로 다양한 block size `Z` 처리, PC matrix는 single-port ROM(BRAM)에 저장.
8. 추가 메모리 비용: `(2r)·Nc·logZ` (r=code rate, Z=block size, Nc=PC matrix 열 수)만큼 증가.
9. 결과: Xilinx Virtex 7 VC707 구현에서 [4] 대비 LUT 38296→30605(≈20.1%↓), decoding cycle 5766→5440(≈6.8%↓, throughput ∝ 1/cycle), FER 저하 없음, BRAM 22개 동일.
10. 한계: 5G BG2 layered 구조 전용, throughput 절대값·gatecount·공정 미기재, 정정능력(FER) 개선은 없고 자원·cycle 절감에 한정.
