### B. 알고리즘 요약

1. 대상 코드: QC-LDPC(CCSDS C2), 표준 AMSA(attenuated min-sum, α=0.75) 기반 반복 MP 디코딩.
2. 문제: 기존 partially-parallel 디코더는 VNU/CNU가 교대 동작해 절반 시간 유휴 상태 -> throughput 병목.
3. 핵심 기법 1: FOMP(Full-Overlap Message Passing) - 2개 프레임을 동시에 적재해, 한 프레임의 V2C 업데이트와 다른 프레임의 C2V 업데이트를 같은 사이클에 수행하여 VNU/CNU 완전 병렬화.
4. 핵심 기법 2: DAA(Dynamic Address Access) 알고리즘 - `Addr_j^(i)new = mod(Addr_j^(i-1)old + lv or lh, L)`로 read/write 주소를 동적으로 갱신해 true dual-port RAM 한 블록에서 두 프레임의 메시지를 충돌 없이 저장.
5. 이 두 기법으로 `Tpro/Tconv ≈ 2`가 되어 이론상 throughput 2배 달성(식 4~6).
6. 확장: 메시지 패킹(P개 메시지를 한 워드에 packing)으로 P배 추가 병렬화, low-complexity data aligner(2×Q-bit 레지스터, 1클록 지연)로 정렬.
7. 구현 디테일: hard decision과 extrinsic message를 함께 RAMM에 패킹(1bit HD + Q-bit magnitude), PCU가 CNU와 같은 주소정보 공유해 별도 주소생성기 불필요; CNU는 CTSM/SM 변환 + FTMFM(첫 2개 최소값 트리) + AMM(α 곱셈).
8. 학습/최적화 절차: 없음(고정 아키텍처 설계).
9. 결과: Q=6bit 양자화, P=7 메시지 패킹, 10 iteration/350MHz에서 단일 디코더 2.67Gbps, 5디코더 병렬 시 13.3Gbps(SOTA 대비 1.3배, 자원 50% 절감).
10. 한계: BER 개선은 목표가 아니며 기존 알고리즘 대비 동등 성능 유지가 전제; FPGA(XCKU060) 검증만, ASIC 미검증; 특정 QC-LDPC(CCSDS C2)에 특화 설계.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FOMP(2프레임 동시 VNU/CNU 완전병렬) | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Dual-Update(parity even/odd 교대)와 유사한 스케줄링 최적화 아이디어, 다만 단일 프레임이 아닌 2프레임 동시처리라 구조 상이 |
| DAA(동적 주소 접근) | `decoder.cpp` (RAM 접근 스케줄) | 메모리 주소 관리 기법으로 HW arch 레이어에 해당, 이식 난이도 중 |
| 메시지 패킹/data aligner | 대응 없음 | Prime ECC z_sb=32 row 병렬 구조와 다른 패킹 전략, 직접 대응 모듈 없음 |
| CNU(FTMFM+AMM, α=0.75 attenuated min-sum) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum 계열 기법과 동일 범주(attenuated factor), 신규성 낮음 |
| 조기종료(syndrome 체크) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 논문은 단순 syndrome-zero 종료만 언급, Prime ECC의 partial-CRC 대비 기초적 |

> 적용 가치: 낮음 | 정정성능 개선이 아닌 순수 throughput/HW 자원 최적화(2프레임 동시처리+동적 주소)이며, 이미 Prime ECC가 보유한 min-sum/dual-update 계열과 겹치는 부분이 많고 부호(CCSDS C2)도 NAND 타깃과 무관.
