### B. 알고리즘 요약

1. 시스템: BI-AWGN 극저noise(σ²→∞, SNR>ln2=-1.59dB) 채널, IoT 저전력용 극저rate 코드; 대표 부호는 `[8208,400]`(w=3), `[8192,450]`(w=4), rate ~0.05.
2. 문제: 극저rate에서 biorthogonal/RM/polar-repetition/기존 LDGM은 길이·복잡도·latency가 과도하게 커짐.
3. 기본 부호: 생성행렬 `G=[I_k|J_{k,d,w}]`의 systematic LDGM, `J`는 순환행렬 `P_k^s`로 구성, 각 열 weight `w`(≤4), 각 행 weight `d`(큼), 길이 `n=k+kd/w`.
4. BP 1단: 정보심볼만 재계산하는 belief propagation. offset `u_i=tanh(z_i)`, parity check (5)로 `u`, l.l.h. `h` 갱신, `µ~2log2(k)` iteration.
5. 핵심식: 각 정보비트가 `d`개 parity check `a_i=a_j·∏a_{i'}` (4)를 만족 → 정보비트 재계산만으로 복잡도 `Φ~2nw(w+4)log2 k`.
6. 반복 강화: 정보비트를 `s`(≤5)회 복제 `G=[I_{k,1}|..|I_{k,s}|J]` → 수신 신뢰도↑, BP BER 2~3배 감소.
7. 2단 결합: BP 출력 l.l.h.를 짧은 polar code `P[k,kr]`(k=512)의 SCL(L=32) 디코더에 입력, 후보를 다시 길이 n으로 인코딩해 ML 선택.
8. 알고리즘 A~D: A=단일 BP+SCL, B=BP 2회, C=CRC-11 추가, D=W(-)/W(+) 분할 + CRC-q1/q2 + half-way 결합.
9. 결과: `[8208,400]` 코드가 SNR=1dB에서 WER~10⁻⁴, CA-polar A5[8192,400]와 동등하되 latency 대폭 감소; [16] 대비 0.75dB 이득(단 rate는 더 높음).
10. 한계: 순수 시뮬레이션만, HW 미설계, 극저rate·polar 결합 특화. CRC/half-way(C,D)가 A,B 대비 개선 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 정보비트만 재계산하는 BP | `decoder.cpp` `LDPC_Decoder()` | tanh 기반 full BP + LDGM 정보비트 한정 방식이라 min-sum 루프와 이질적 |
| BP 출력 l.l.h. + polar SCL 결합 | 대응 없음 | polar/SCL은 Prime ECC 범위 밖 |
| offset `u=tanh(z)` 채널 변환 | `channel.cpp` `Set_LLR_Th()` | full-tanh soft 처리로 hard/2~3bit 양자화와 불일치 |
| LDGM 순환행렬 부호 구성 | `ecc_top.cpp` `Load_PCM()` | LDGM(weight w) 구조가 고rate QC-LDPC PCM과 다름 |

적용 가치: **낮음** — non-QC LDGM + polar 결합, full-tanh BP, 극저rate/soft-4bit+ 채널 특화로 고rate binary QC-LDPC NAND ECC와 부호·디코더·채널 모든 층위가 어긋남.
