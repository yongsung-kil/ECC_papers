### B. 알고리즘 요약

1. 시스템: 랜덤구성 regular LDPC (96,48)/(204,102)/(2000,1000), rate 1/2, girth 6, AWGN+BPSK, 6bit(1 sign+5 quant) approximate-min* 디코더, 최대 200 iter.
2. 문제: BP 복호는 ML 대비 성능손실 — 짧은 부호는 waterfall, 긴 부호는 error-floor에서 큰 gap이 발생.
3. 모델: 디코더 실패(detected error)는 bipartite graph의 short cycle 조합인 `trapping set`(x,y) (x=오류비트수, y=unsatisfied check수)에서 기인.
4. 핵심기법1(식별): 그래프탐색으로 shortest-cycle 목록화 후, importance sampling(IS)으로 각 행 cycle bit에 bias(합=2)를 걸어 실제 디코더 실패를 유발하는 dominant trapping set 확정.
5. 의미: IS로 "실제 실패를 일으키는" trapping set만 골라 오탐을 줄이고 관심 BER구간의 지배적 set을 보장.
6. 핵심기법2(제거): 식별된 (trapping parity checks → trap bits) 쌍을 LUT로 구성. 최대 iter 후 unsatisfied check 패턴이 LUT와 match되면 해당 trap bit의 hard-decision을 flip → 모든 parity 만족, 정상 codeword 수렴.
7. 디테일: 부호 최소거리보다 작은 trapping set만 구현(오정정 방지). 추가복잡도는 LUT 로직 + 비트반전뿐.
8. 최적화: 별도 학습 없음, 오프라인 IS 특성화.
9. 결과: (204,102)/(96,48) 각 0.4/0.5 dB 개선; (2000,1000) WER~10^-4 error-floor 제거, Eb/No 2.75 dB에서 2자릿수 개선.
10. 한계: WER<10^-3에서만 유효(고WER는 긴 cycle의 대형 trapping set이라 LUT 비대), HW 미설계·시뮬만, trapping set은 부호별 재식별 필요.
