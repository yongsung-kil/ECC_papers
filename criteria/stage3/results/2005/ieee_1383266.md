### B. 알고리즘 요약

1. 대상: rate 3/4, 1024-bit block LDPC (regular/irregular 모두 지원), AWGN 채널(SNR 0.5dB 예시).
2. 문제: LDPC 디코더 HW의 실병목은 노드별 높은 메모리 요구와 복잡한 인터커넥트 → 지원 코드 종류나 아키텍처가 제약됨.
3. 핵심: bit node/check node를 PE로 보고 25노드(16 bit + 9 check) 5x5 2D mesh NoC에 매핑, XY 라우팅 + on/off backpressure(패킷 드롭 방지).
4. check node는 로그영역 bilinear transform `B(llr)=ln(1+... )`을 ROM LUT로 계산(Eq.1), sign-magnitude 16bit 고정소수점.
5. bit node는 `stored_llr + Σllr - llri_new` 누적(Eq.2)으로 수렴 판정, 반복 횟수 제어.
6. 매핑 최적화: bipartite graph를 물리 노드에 지능적 배치해 평균 홉 거리를 랜덤 대비 절반으로 축소, 2블록 lockstep 병렬 복호.
7. 패킷: 48bit 단일 phit(16bit 헤더 + 16bit 데이터×2), 라우터 비파이프라인 1홉/클록.
8. 전력최적화 1: 값이 0/±∞(전체의 25~40%)일 때 데이터 라인 불변, S1/S2 2bit만 갱신 → 스위칭 감소(12.75% 절감).
9. 전력최적화 2: early termination — 5연속 llr이 0.25 미만/0.75 초과면 즉시 확정, 파이프라인 우회. 두 기법 합쳐 30% 전력 절감(24.32W), HW 오버헤드 <5%, BER 영향 <1%.
10. 한계: 160nm 500MHz 합성, ~110mm2, 34.8W로 전력·면적 큼. 정정능력 개선 없음, NoC 통신구조가 특수함.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit/check node 반복 복호 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `C2V_Cal()` | 개념 대응하나 NoC 통신구조라 알고리즘 이식 불가 |
| check node bilinear transform LUT | 대응 없음 (full-tanh/Sum-Product 계열, 우리는 Min-Sum) | 낮음 |
| early termination(수렴 임계 판정) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | 조기종료 아이디어만 유사, 방식 다름(CRC 기반 vs llr 임계) |
| NoC 라우팅/메모리 라우팅 | 대응 없음 (우리는 z=32 row 병렬 pipeline) | 낮음 |

적용 가치: **낮음** — 범용 NoC 인터커넥트 기반 디코더로 부호설계·정정능력 기여가 없고, full-tanh LUT와 NoC 통신구조가 Prime ECC의 Min-Sum·z=32 pipeline과 이질적이다.
