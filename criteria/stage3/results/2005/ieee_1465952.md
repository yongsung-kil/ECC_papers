### B. 알고리즘 요약

1. 대상: BSC(inter/intra-chip bus, SoC 배선 crosstalk) FEC용 (n,3,6)-regular LDPC(rate 1/2), LLR 기반 message passing, fully-parallel FPGA(Xilinx Virtex) 구현.
2. 문제: 저지연 요구로 재전송 불가 → fully-parallel 디코더 필요하나 VN/CN·버스 폭이 LLR 정밀도에 비례해 HW 자원이 과다 → 성능 유지하며 정밀도 최소화가 목표.
3. 모델: BSC(bit-flip), LLR `F = log P(x=+1|r)/P(x=-1|r)`, VNU는 입력 LLR 합, CNU는 sign 곱 + magnitude 최소연산(min-sum류 근사) 형태.
4. 핵심기법1 (Uniform quantization): VN/CN의 LLR을 각각 스텝 `ε`, `ε_h`의 정수배로 매핑하고 정수 인덱스만 메시지로 전달 → `ε` 클수록 메시지 종류↓·HW↓; 최대 `ε`, `ε_h` 탐색.
5. 결과 관찰: 채널 BER 조건에서 `ε`을 2까지 키워도 성능저하 미미(그 이상은 손실), `ε_h` 상한도 존재 → "near floating-point" 성능 유지하는 최대 스텝 선택.
6. 핵심기법2 (Variable quantization): CN LLR 정수값 `k`의 히스토그램이 iteration 진행에 따라 좁아짐(1st iter 넓고 후반 협소)을 관찰 → 반복별로 CN 입출력 워드폭을 다르게 설계(수학적 정밀도는 uniform과 동일 → 무손실).
7. 구현 디테일: CN을 반복별 워드폭 (Input 1/2/4bit, Output 1/6/8bit)으로 축소, HW 제약 시 최고정밀도 기준 단일 VN+CN loop-back 구성 가능.
8. 학습절차: 없음(경험적 히스토그램·BER 시뮬 기반 파라미터 선택).
9. 결과: uniform 양자화로 CN LUT 767→601 수준, variable 양자화로 CN을 iter1 24 LUT / iter2 193 / iter3 388 LUT로 대폭 축소; floating-point 대비 FER 근사 유지(최대 LLR 7 가정).
10. 한계: 텍스트 추출본에서 수식·수치 다수 손상, code length·throughput 미보고, BSC(hard bit-flip) 특화라 NAND soft-read 분포로 재검증 필요.
