> 총평: LDPC 컨볼루션 부호(LDPC-CC)의 레지스터 기반 processor 타일링 ASIC 인코더/디코더 최초 구현. min-sum(sign-min) 노드 연산은 Prime ECC와 겹치나, 부호 클래스(convolutional vs block QC-LDPC)와 아키텍처(반복=processor 공간 전개)가 근본적으로 달라 이식성 낮음.

### B. 알고리즘 요약

1. 대상은 regular (3,6) rate-1/2 LDPC 컨볼루션 부호(LDPC-CC), memory `M=128`, 하삼각 패리티행렬로 인코딩 지연·복잡도 `O(M)`.
2. 문제: LDPC-CC가 스트리밍/패킷망에 block LDPC보다 유리하나 실제 ASIC/FPGA 구현 이점이 미검증.
3. 모델: 코드비트 `v(nM+φ)`를 과거 정보/코드비트의 GF(2) XOR로 생성(식1), `δ` 항이 패리티행렬 결정, 채널은 근사 AWGN.
4. 핵심기법1(인코더): FIFO 지연라인(코드/정보비트 레지스터)+φ 구동 MUX로 XOR 입력 선택, block 제어 불필요한 스트리밍 인코딩.
5. 핵심기법2(디코더): "processor" 개념 — 지연소자+단일 CN+단일 VN 단위를 직렬 타일링, processor `j−1` 출력이 `j` 입력(LLR), block LDPC의 iteration에 대응.
6. 핵심식: 패리티노드는 sign-minimum(min-sum) `yk = min|xn|·Πsgn(xn)` (식2, 6입력), 이후 VN에서 LLR 합산.
7. 트릭: LLR word-length 6bit(프로세서당 48bit I/O), processor latency `N=140` cycle(파이프라인용 여분 레지스터), 결함 프로세서 우회 라우팅.
8. 최적화: processor 수로 throughput↔BER trade-off(30 proc = 3×10 decoder 또는 1×30 decoder), 인코더 임계경로(XOR) 파이프라이닝.
9. 결과: 0.18µm 합성 — 디코더 164Mbps(10 proc, 10.09M µm², 3.9W), 인코더 430Mbps(파이프라인, 52.6k µm²), latency 4.65ns, 125MHz 목표.
10. 한계: processor가 인코더 대비 ~20배 면적, 10~20개 필요, block 부호 아님, 정정능력 개선 아닌 구현 연구.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 패리티노드 sign-minimum(min-sum) 연산 (식2) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 낮음: min-sum은 이미 보유(중복), CN 연산 아이디어만 개념 일치 |
| VN LLR 합산 처리 | decoder.cpp VN_Cal_HD() 등 | 낮음: 표준 VN 합산, 컨볼루션 processor 문맥이라 이식 대상 아님 |
| processor 직렬 타일링 = iteration 공간전개 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 낮음: Prime ECC는 block iteration 시간전개, LDPC-CC processor 공간전개와 구조 상이 |
| LDPC-CC 하삼각 인코딩 | encoder.cpp LDPC_encoder_4KB() | 낮음: Prime ECC는 dual-diagonal block 인코딩, convolutional 부호와 다름 |
```

적용 가치: **낮음** — min-sum(sign-min) CN 연산은 Prime ECC가 이미 보유해 중복이고, 부호 클래스가 convolutional(LDPC-CC)이라 block QC-LDPC인 Prime ECC의 부호설계·디코더 아키텍처(processor 공간 타일링 vs iteration 시간 루프)와 근본적으로 달라 떼어 쓸 실익이 없다.
