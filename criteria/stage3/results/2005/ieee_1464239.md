### B. 알고리즘 요약

1. 자기 테이프 기록의 VR2 RLL(variable rate randomized run-length-limited) 제약을 LDPC 부호와 결합하는 방식을 제안.
2. 문제: 기존 VR2 RLL은 bit 삽입 방식이라 가변 블록길이·오류전파 단점이 있고 LDPC의 soft-decision 복호와 부적합.
3. 관찰: LDPC 디코더는 RLL 제약을 위해 codeword에 의도적으로 삽입한 소수 에러도 복원 가능 [ref2].
4. 인코딩: user block → LDPC codeword → LFSR 의사난수열과 XOR → VR2 RLL 위반 지점에서 bit 삽입 대신 `bit flipping`으로 제약 충족.
5. LFSR 초기값을 바꿔가며 XOR 시도, 사전정의된 시도 횟수 내에서 flip 비트 수가 최소인 시퀀스를 기록.
6. 복호: 채널 검출기는 symbol-by-symbol APP 검출기(BCJR 또는 SOVA), 출력 LLR을 hard-decision부와 reliability부로 분리.
7. 데이터 헤더의 LFSR 초기값으로 의사난수열 복제 → hard-decision부와 XOR → LDPC 디코더가 XOR된 시퀀스+reliability로 복호.
8. 복호 실패 시 LDPC extrinsic 정보를 다시 hard/reliability로 분리, XOR 후 채널 검출기로 피드백하는 iterative detection-decoding(turbo) 구성.
9. 결과: QC-LDPC(length 4608, rate 8/9, PEG 기반 [ref3]) EPR4 채널에서 VR2+1 XOR시 BER 10^-6에서 0.6dB 손실, 8 XOR시 0.1dB 미만; 실제 테이프 testbed 220~250kbpi에서 수천 블록 무오류.
10. 한계: 부호/디코더 자체 개선 아님, 자기 테이프 RLL 변조제약 전용, HW 미설계, NAND(RLL 제약 없음)와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| VR2 RLL bit-flipping 제약 결합 | 대응 없음 | RLL 변조제약은 NAND ECC에 없는 개념 |
| LFSR 스크램블 + XOR | 대응 없음 | 기록 채널 변조용, Prime ECC 데이터경로에 대응 없음 |
| BCJR/SOVA turbo detection-decoding | 대응 없음 | 자기기록 채널 검출 전용, NAND(AWGN/RBER)와 무관 |
| QC-LDPC(rate 8/9, PEG) | `ecc_top.cpp` `Load_PCM()` | 고rate QC-LDPC라는 점만 유사, 부호구성법은 별도 논문[ref3]에 있음 |

적용 가치: **낮음** — 자기 테이프 RLL 변조제약을 LDPC와 결합하는 constraint-coding 기법으로, NAND binary QC-LDPC ECC의 부호설계/디코더/HW에 떼어 쓸 요소가 없음.
