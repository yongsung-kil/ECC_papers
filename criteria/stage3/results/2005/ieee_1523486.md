### B. 알고리즘 요약

1. 대상은 moderate-length LDPC로, 예제는 irregular `(N,K)=(800,300)`, `(800,500)` (`R=3/8`, `5/8`), BEC/AWGN 채널.
2. 풀 문제: 고SNR에서 error floor를 유발하는 소형 stopping set을 부호 설계 단계에서 명시적으로 제거하고자 함(ACE 등 암묵적 conditioning은 실제 검출 미보장).
3. 핵심 아이디어: Tanner graph에서 두 종류 메시지를 전파하는 BP 유사 알고리즘 — CD(cycle detection) 메시지와 SD(stopping set detection) 메시지.
4. CD 메시지는 이진값으로 유효 경로 여부를 OR/XOR 갱신(식 `(1)`), 시작 변수노드 v*의 모든 간선에서 '1'로 출발.
5. SD 메시지는 (변수노드, 제약노드, flag) 3-tuple 집합으로 경로상 노드 정보를 운반, 제약노드/변수노드에서 식 `(3)`,`(5)`로 갱신.
6. hitting 노드(모든 간선으로 SD 수신 변수노드 or 2개 이상 수신 제약노드)에서 aggregate 메시지 `ξ̄`를 모으고 cardinality condition으로 stopping set 후보 판정.
7. Procedure 1의 removal 절차로 'open' 간선을 가진 노드를 제거해 실제 stopping set `Mi` 추출(반복당 1회).
8. 부호 설계: 변수노드를 하나씩 추가하며 `Ntry`회 랜덤 간선배치 시도, cycle 길이 `Lcycle` 조건 통과분에 stopping set 검출→변수노드 수·radius `rS` 가중합 metric으로 최적 배치 선택(Fig.2).
9. 결과: 동일 girth/ACE 부호 대비 BEC·AWGN 모두에서, 특히 error floor 영역 성능 향상(sum-product 50 iter, Fig.3~4). 저SNR은 유사.
10. 한계: HW 미설계, 시뮬만, 검출집합이 최소 stopping set 보장 안 됨(다른 시작노드 반복으로 근사).
