### B. 알고리즘 요약

1. 대상: binary LDPC, rate 1/2 regular `(3,6)`/`(4,8)` 코드 `(8000,4000)`,`(1008,504)` 등을 AWGN에서 LLR-BP 복호.
2. 문제: 정확 LLR-BP(tanh rule)는 `tanh`/`atanh` 연산으로 HW 부담 크고, 순수 min-sum(BP-based)은 0.5~0.6dB 성능 손실.
3. check-node 정확표현 3종 정리: tanh rule, Gallager involution transform `f(x)=-ln tanh(x/2)`, Jacobian(Max-Log) 로그.
4. 핵심 근사 - BP-based(min-sum): `|Λ| ≈ min(|λ_i|)`, 부호는 XOR. 두 최소값과 부호만 저장.
5. Normalized BP-based: min 결과를 `1/α`(α>1)로 스케일 `Λ = min/α`. 레지스터 shift(2,4,8)로 HW 구현 가능.
6. Offset BP-based: min 결과에서 양의 상수 β를 빼고 0에서 clip `Λ = max(min-β, 0)`. HW에 더 유리(정규화보다 편리).
7. APP-based symbol-node 간소화: extrinsic 대신 a posteriori LLR만 계산해 저장/연산량 축소, 단 단축 사이클 많은 코드(DSC)에서만 유효.
8. DE(density evolution)로 α, β 최적값과 양자화 step/bit 수를 결정, 유한정밀 threshold 도출.
9. 결과: normalized/offset BP-based가 BP 대비 0.05~0.1dB 손실로 근접; offset은 4-bit 양자화로 error-floor 회피(Gallager 방식은 7-bit 필요).
10. 한계: HW 미설계(알고리즘·시뮬만), rate1/2 regular AWGN 위주, 고rate/NAND·short-cycle 코드 일반화 제한.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum check-node 근사 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 정확히 동일 min1/min2+sign 구조, 이미 보유 |
| normalization/offset 보정 | decoder.cpp CNU_Update_New_Mag() 내 magnitude 처리 | 스케일/오프셋을 magnitude 단계에 삽입, 이식 쉬움 |
| iteration별 파라미터/LLR 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | α/β를 iteration 구간별로 반영 가능(논문은 고정 권장) |
| 양자화 step/bit, threshold | channel.cpp Set_LLR_Th(), Set_R_Offset() | 4-bit 양자화 threshold 설계 근거로 참조 |
| APP-based symbol-node 간소화 | decoder.cpp VN_Cal_HD() 등 | VN 처리 간소화 옵션(단축사이클 코드 한정, NAND 고rate엔 리스크) |
```

적용 가치: **중간** — normalized/offset min-sum·4-bit 양자화 근거는 Prime ECC의 min-sum 코어(§3 이미 보유)와 정합성 높아 파라미터 튜닝 참고서로 유용하나, 핵심 기법 다수를 이미 구현 중이라 신규 기여는 제한적.
