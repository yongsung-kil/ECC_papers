> 총평: SPA의 small-loop 문제를 Pearl clustering(CFG)으로 완화하고 binary 코드에 even/odd 메시지로 복잡도를 억제한 이론적 복호 개선안이나, min-sum QC-LDPC 기반 Prime ECC와 방식 불일치 + 연산·메모리 증가로 이식가치 낮음.

### B. 알고리즘 요약

1. 대상: MacKay regular LDPC (`length=1008`, col-weight 3, row-weight 6, `rate=0.5`, length-4 loop 없음), 채널 AWGN/BEC, 복호는 factor graph(FG) 위 sum-product(SPA).
2. 문제: SPA는 FG의 loop(특히 length-4/6)에서 정확 사후확률을 못 구해 성능 저하, BEC에서는 stopping set에 걸리면 복호 실패.
3. 기법 1단: Pearl clustering으로 loop에 포함된 변수노드들을 하나의 `cluster node`로 묶은 Cluster Factor Graph(CFG) 구성 → 해당 loop 소거.
4. 핵심 아이디어: cluster node는 묶인 변수들의 국소 결합확률을 계산 → BEC의 stopping set을 연립방정식 풀이로 해소, loop로 인한 오류 감소.
5. 복잡도 문제: cluster node의 메시지 원소 수가 alphabet 크기(2^|cluster|)로 커져 일반적으로 SPA보다 연산 급증.
6. 기법 2단(핵심): binary parity-check 코드에서는 제약이 "합이 mod2로 0/1"뿐 → cluster-check 메시지를 `even-message`/`odd-message` 2개로 축약해 Horizontal step은 기존 SPA와 동일 연산량 유지.
7. 구현 디테일: Vertical step에서만 각 원소 계산 + marginalization(`e()`/`o()` 함수로 even/odd weight 분리) 수행, 정규화 상수 `α` 사용.
8. 실험 설정: length-6 loop 165개 중 조건 만족 98개를 clustering(결과 CFG의 length-6 loop는 오히려 191개), max iteration 100, SPA vs CSP 비교.
9. 결과: BEC에서 SPA보다 뚜렷이 우수(stopping set 오류 감소), AWGN(SNR 2.5~3.5)에서는 "약간" 우수. 1 iteration 연산량 SPA 65520 vs CSP 101628.
10. 한계: HW 미설계, 연산·메모리 증가, 기존 코드 사용으로 clustering 최적화 부족, loop 수와 성능이 단순 비례 아님.

| Horizontal(check) step | [decoder.cpp](../../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN 위치는 대응하나 even/odd 메시지 구조는 미보유 |
| Vertical step / marginalization | [decoder.cpp](../../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` | VN 처리 위치 대응, cluster 원소별 marginalization은 미대응 |
