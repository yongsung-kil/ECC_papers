> 총평: circulant 블록(block-cyclic=QC) 구성으로 shift-register 저복잡도 인코딩을 갖는 고rate 단블록 LDPC를, 부호밀도·weight spectrum 조정으로 error-floor/미검출오류율을 개선하는 code-design 논문. QC 구조·저복잡도 인코딩·weight-spectrum 지역탐색은 참고가치 있으나 블록길이·최적화 목표가 NAND와 어긋남.

### B. 알고리즘 요약

1. 시스템: TWACS 전력선 통신(자동검침), 매우 짧은 패킷(100~150bit), 고rate 요구. 기존은 (255,247) shortened Hamming(1bit 정정)+CRC-16(24bit overhead, rate≤0.85).
2. 문제: 고rate·단블록 LDPC는 미검출오류(undetected error)가 발생하는데, 별도 CRC 추가는 대역폭 낭비 → LDPC 자체가 정정+검출을 모두 만족해야 함.
3. 핵심 모델: 부호밀도(column weight)를 올리면 Tanner graph에 4-cycle이 추가되어 전체 오류율은 나빠지지만 min-distance가 커져 미검출오류율은 개선 → 밀도로 조정 가능한 트레이드오프.
4. 핵심 기법(부호구성): block-cyclic 구조 `H=[C1 C2 ... CL]` (각 Cj는 square circulant) = QC-LDPC. cyclic difference family/Steiner triple로 4-cycle 회피, weight 3·M redundancy시 4-cycle-free 최대길이 `N=M(M-1)/6`.
5. 저복잡도 인코딩: systematic `G^T=[IK; G1;...;GL-1]`, `Gj=CL^-1 Cj`도 circulant → shift-register 연산 + 블록당 벡터 1개만 저장(메모리 최소). 단 CL은 GF(2)에서 invertible 필요(홀수 weight 필요조건).
6. min-distance 트릭: 모든 블록이 홀수 column weight면 codeword가 all-even → min-dist 6 부호에서 weight-6 codeword 제거 시 min-dist 8로 상승.
7. 부수: H에 circular permutation matrix를 더하고 빼서 weight spectrum(저weight codeword 수)을 조정.
8. 최적화: fitness = `Σ ni/C(N,i)` (weight≤v codeword 비율), Hu et al.[8]로 저weight codeword 추정, 지역탐색(distance-1 이웃에서 최적 부호로 점진 이동).
9. 결과: (155,124) 최적화 부호들 — code 4(weight7)는 BCH/CRC 대비 미검출오류율 개선+overall 약 1.4 dB 이득; code 3(weight5)은 Hamming/CRC 대비 약 3 dB 이득(단 rate 소폭 감소). soft-decision 입력이 이득 일부 기여.
10. 한계: 순수 시뮬(HW 미설계), 블록 100~155bit로 이론한계 접근 불가, 최적화 목표가 미검출오류율(통신 검출용)이라 NAND 정정 목표와 상이.
