### B. 알고리즘 요약

1. 대상: (2048, 3, 6) rate 1/2 regular LDPC, AWGN, 최대 20 iteration, MPSP/BP(Sum-Product) 디코딩.
2. 문제: fully-parallel 디코더는 bipartite graph를 직접 매핑하나, random H의 조밀·불규칙 배선이 code length↑ 시 라우팅 congestion을 야기(wire 길이·면적 ∝ n^0.5).
3. 접근: H의 1 위치가 CNU-VNU 물리 연결을 결정하므로, 1을 국소화하면 배선도 국소화된다는 관찰 → layout-지향 code 구성.
4. 1단(graph partitioning): 행을 C1/C2, 열을 Va/Vb/Vc로 나눠 1을 subgroup에 국한. Vb만 양 그룹에 연결. worst-case 라우팅 거리 절반.
5. 지표 `Overlap Ratio (OR)=OL/n`: 두 파티션에 걸친 열 비율. 시뮬 결과 OR을 0.2까지 낮춰도 BER 무열화(양 CNU 연결 VNU 20% 감소).
6. 2단(일반화): 파티션을 다분할하면 1이 대각선 주변에 몰림 → DLDPC. 폭 W의 두 대각선 사이로 1을 제한, span L(check)·H(variable) 축소.
7. 구성법: (a) random(대각 경계 내 무작위) 또는 (b) 구조화 sub-matrix(260×16, 16열마다 반복, 인접 8행 shift, 4-cycle 제거) 반복.
8. HW: cell-based array — 각 cell = 1 CNU + 2 VNU(rate 1/2), cyclic 성질로 ring 배치해 규칙 배선. CNU는 ψ변환 조합논리, VNU는 majority-vote+LLR 합.
9. 결과: BER 2.56e-6 @2.3dB로 random 부호와 동등. TSMC 0.25um die ~59mm2(0.16um 환산 ~24mm2), [7](1024-b, 52.5mm2) 대비 면적 ~75% 감소.
10. 한계: 정정능력·iteration 개선 없음(HW/배선만), throughput/gate count 수치 미제시, fully-parallel 직접매핑 전제.
