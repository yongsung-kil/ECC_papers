### B. 알고리즘 요약

1. 대상: N=2040, rate 1/2, (3,6) regular LDPC. j=3, k=6. 20 iteration MP 디코딩.
2. 문제: LDPC 디코더는 H matrix를 메모리(ROM)에 저장하고 bit↔check 라우팅 switch logic이 커서 HW 비용·클록 병목.
3. 기존 병렬(scalable) 아키텍처[2]는 M개 bit node를 병렬 처리하되 memory 충돌 방지 제약(연속열 non-overlap 등)이 필요.
4. 핵심 기법(Tiling): scalable 제약을 만족하는 M×M 크기의 소수 패턴(sub-matrix)을 만들어 H 전체를 이 패턴들의 반복 배치로 구성 → 전체 H 대신 패턴만 저장.
5. 패턴 배치 제약: (i) 인접 패턴쌍의 재현 금지로 길이-4 cycle 제거, (ii) 공정성 위해 각 패턴 최소 `⌈m/l⌉`회 반복, (iii) 최대한 random 배치로 BER 유지.
6. 예시: M=340(=N/6)이면 18 tiles를 5개 패턴으로, M=170(=N/12)이면 36 tiles를 10~12 패턴으로 채움.
7. HW: 저장된 H ROM을 제거하고 패턴 선택을 check node당 8-to-1(→6-to-1) MUX로 구현, 340개 MUX의 select line을 공유해 switch logic 단순화.
8. 결과(HW): M=340에서 클록 5배 향상(≈99MHz), gate count 1.791M→1.424M; M=170은 0.947M로 감소.
9. 결과(BER): M=N/6, N/12에서 패턴 tiling 대비 random 대비 BER 열화 <0.2dB(무열화). M=N/24는 열화 큼.
10. 한계: 순수 HW/구성 관점(정정능력·iteration 개선 없음), soft 채널 가정, gate count 공정 미기재.
