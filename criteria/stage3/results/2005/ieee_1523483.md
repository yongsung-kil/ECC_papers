### B. 알고리즘 요약

1. 배경: BEC 상 선형부호의 반복복호 성능은 Tanner graph의 가장 작은 stopping set 크기가 결정 — 이를 stopping distance `s(H)`라 부름.
2. 핵심 구분: 최소거리 `d(C)`는 부호 `C`의 성질이지만 `s(H)`는 패리티검사행렬 `H`(Tanner graph) 선택에 의존. 항상 `s(H) ≤ d(C)`.
3. 정의(신규): stopping redundancy `ρ(C)` = `s(H)=d(C)`를 만족하는 `H`의 최소 행수. `ρ(C), r(C), d(C)` 사이 trade-off 최초 연구.
4. 잘 정의됨(Thm 2): `C⊥`의 모든 비영 codeword로 `H*`를 구성하면 `s(H*)=d(C)` → `ρ(C) ≤ 2^{r(C)}-1`.
5. 일반 상한(Thm 4): `d(C)≥3`이면 `ρ(C) ≤ Σ_{i=1}^{d-2} C(r,i)` — `d`가 상수면 stopping redundancy는 redundancy(길이)의 다항식.
6. 부호 결합의 영향: direct-sum `(u,v)` → `ρ(C3) ≤ ρ(C1)+ρ(C2)` (Thm 5), `(u,u)` → `ρ ≤ ρ(C1)+n` (Thm 6), overall parity 확장 → `ρ(Ĉ) ≤ 2ρ(C)` (Thm 7, d=3).
7. Reed-Muller(핵심결과): `(u,u+v)` 재귀 구성 A로 `H(r,m)` 설계 → `s(H(r,m))=2^{r+1}`, `ρ(R(r,m)) ≤ d(C)·r(C)/2` (Thm 11) — 상수 `d`면 redundancy 증가는 상수배뿐.
8. Golay(사례): `(24,12,8)` `G24`의 기존 double-circulant `H24`는 `s=4`(최대치의 절반). greedy(lexicographic) 컴퓨터 탐색으로 `s=8`인 `34×24` 행렬 구성 → `ρ(G24) ≤ 34` (ternary Golay ≤ 22).
9. 성능 입증: `G24`에 대해 ML / `H24`(s=4) / `H̄24`(s=8) 세 복호기의 BEC 복호실패확률을 정확 해석식으로 계산(Table II, Fig 1) — stopping distance 증가가 error-floor를 개선함을 확인.
10. 한계: BEC 한정, RM/Golay/MDS 등 대수부호 중심 순수 이론, HW/시뮬 없음, `ρ`가 `r`보다 지수적으로 커질 수 있음(RM은 `2^r` 배).
