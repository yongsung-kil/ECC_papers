> 총평: bit-flipping 기반 (0,k) run-length constrained coding에서 의도적으로 뒤집힌 비트를 message-passing 정보로 되돌리는 flip decoder — NAND에는 run-length 제약이 없어 이식 대상이 아니며 error-floor 개선도 제약 부호 특유 문제 해결일 뿐.

### B. 알고리즘 요약

1. 시스템: LDPC 부호어를 (0,k) run-length 제약 AWGN 채널로 전송하는 Vasic의 bit-flipping 기반 constrained coding 시스템.
2. 부호화 2단계: (1) error control code로 메시지 부호화, (2) 연속 `k+1`개 zero run이 나오면 `k+1`번째 zero를 flip해 (0,k) 제약 충족(예: k=3에서 6번째 비트 flip).
3. 풀려는 문제: flip 비트 수가 많으면 원래 방식에서 message-passing 디코더가 모든 에러를 못 잡아 고SNR에서도 error floor 발생.
4. 핵심 기법(flip decoder): 각 iteration이 (1) message-passing 1회, (2) `unflip`으로 의도적 flip 비트 정정, (3) `reflip`으로 unflip이 잘못 바꾼 비트 검출의 3단계.
5. 핵심 변수: `M_j`=비트 j가 자신 포함 패리티들로 보내는 메시지 합, `UR_j`=비트 j가 받는 메시지 합, `contradict`=`M_j`와 `UR_j`의 부호가 다른 비트 수.
6. unflip 조건: 비트 `c_i`가 `k`개 연속 zero run 뒤에 오고(`M_j<0` for j=i-1..i-k), 자신은 1일 확률이 더 높고(`M_i>0`), 포함 패리티 다수가 음수(`UR_i<0`)이면 unflip 후보.
7. 판정: 여러 iteration 후 `contradict`>threshold `T1`이면 후보를 unflip; reflip은 unflip된 비트 `d_i`의 `UR_i`가 이후 커져 `T2` 초과 시 다시 flip.
8. 용량 계산: stochastic expectation-maximization으로 (0,k) constrained AWGN 채널 용량 경계 산출(입력은 `k+1` 상태 Markov, 길이 n=10^4); k>4면 제약/무제약 용량 gap이 작음.
9. 결과: (0,7) constrained LDPC(column weight 3)가 무제약 LDPC와 비슷한 rate 달성; (0,9) constrained rate 0.95 bps 부호에서 제안 디코더가 error floor 제거(제약 부호 floor가 무제약 부호 floor로 제한됨).
10. 한계: 1페이지 digest, HW 미설계 시뮬만, bit-flipping constrained coding·(0,k) run-length 제약 전용, 추가 unflip/reflip 단계로 iteration당 연산·latency 증가.
