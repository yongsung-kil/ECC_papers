### B. 알고리즘 요약

1. 시스템: BPSK+AWGN, (λ=3, ρ=6) regular LDPC. 예제부호 rate-1/2, N=3072, K=1536, 엣지수 M=9216, girth 6.
2. 문제: 랜덤 LDPC는 PCM 저장 필요(다중 rate/length 시 메모리 폭증), 완전병렬은 HW 과다·직렬은 latency 과다.
3. 핵심 구성: PCM 연결을 MLLCS 재귀 `A_{n+1}=(a·A_n+b) mod M`으로 생성 → 수신단이 (a,b,A0) 3값만으로 그래프 재생성, 저장 불필요.
4. 조건 D1~D8: maximal-length 시퀀스 보장(D1~D4) + length-4 cycle 회피(D5~D8, λ=3 전용).
5. 정리1(핵심): X를 나누고 ρ로 나눠지는 M1에 대해 `C_n=(C_{n-1}a+b) mod M1`도 MLLCS → 원 시퀀스 mod M1이 길이 M1 주기로 반복 → check connection을 M1개 subgroup으로 분할 가능.
6. 아키텍처: extrinsic 메모리를 M2행 x M1열 블록 배열로, 열=subgroup. M1개 메시지 병렬 read → M1/ρ개 CNU, M1/3개 BNU 동시 처리.
7. 라우터: subgroup 순서가 일정하게 반복 → 시간불변 고정 라우팅(단일 부호는 hardwire 가능). [5]의 매 클록 스위칭 대비 이점.
8. 복호: SPA(`ψ(x)=log tanh(|x|/2)`), CN update식(9)(10), BN update식(11), 채널 LLR `Lch=-2r/σ²`.
9. 결과: iteration당 `M/(M1·M2)+N/(M1/3)` 클록. 동일 PCM으로 M1/M2 조절해 병렬도 스케일 → BER 불변. MLLCS 부호가 랜덤(no len-4) 대비 성능 손실 없음(Fig4).
10. 한계: HW 아키텍처 제안이나 gatecount/throughput 수치 없음, 시뮬만, MLLCS 부호구조에 종속.
