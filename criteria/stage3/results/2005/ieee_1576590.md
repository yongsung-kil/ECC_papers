### B. 알고리즘 요약

1. 대상은 BCH/RS 등 sparse graph 표현이 어려운 binary linear block code `C(N,K)`, BPSK/AWGN. 예: eBCH(128,64,22), RS(31,25,7).
2. 문제: BP는 이런 dense code의 short cycle 때문에 수렴 실패하고, 저차 OSD/BMA는 sub-optimum, 고차 OSD/BMA는 order에 따라 복잡도가 지수적으로 증가.
3. ABP(adaptive belief propagation): 매 iteration `H`를 `Hm`으로 축소해 최소 신뢰도 bit들의 column weight를 1로 만들어(오류 bit를 graph leaf로 이동) short cycle 영향 완화.
4. 2단계 구조: (a) `Lm-1` LLR로 새 MRB 생성 후 order-1 reprocessing(OSD(1) 또는 IT-BMA(1)), (b) sufficient condition으로 optimum codeword 판정, 미충족 시 ABP 1회 수행해 bit reliability `Lm` 갱신.
5. 핵심 아이디어: reliability가 optimal codeword 방향으로 수렴하면 MRB 내 오류 수가 줄어, 낮은 order OSD/BMA로도 정정 가능해짐(ABP가 MRB 갱신 도구 역할).
6. BMA는 OSD 대비 계산량을 제곱근으로 줄이되 `2^(S-K)` 메모리 사용; IISR(IT-BMA)로 information set을 반복 축소.
7. sum-product update는 참고문헌 [2] 방식 채택([1]보다 성능 우위), phase-0/phase-1 reprocessing으로 iteration당 O(K) candidate만 처리.
8. 학습/최적화 절차 없음 (파라미터: ABP iteration 수 M1, IISR iteration 수 M2 등 수동 설정).
9. 결과: eBCH(128,64,22)에서 WER 10^-4 기준 MLD 대비 0.3dB 이내; RS(31,25,7)는 near-MLD. 복잡도는 iteration 수에 선형 비례(order에 지수 비례하는 고차 reprocessing 대체).
10. 한계: HW 미설계, 시뮬만; iteration마다 Gaussian elimination과 OSD/BMA reprocessing 필요로 latency/복잡도 큼; NAND용 sparse QC-LDPC가 아닌 dense code 대상.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ABP: 매 iteration adaptive H 축소 + sum-product | 대응 없음 | Prime ECC는 고정 `PCM_b` min-sum, iteration마다 H 재구성/Gaussian elimination 없음 |
| OSD/BMA order-statistics reprocessing | 대응 없음 | order-statistics list decoding은 min-sum message passing과 계열 다름 |
| soft reliability(LLR) 입력 | [channel.cpp](../../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 소프트 LLR 입력이라는 점만 느슨히 대응, 정정 알고리즘은 무관 |

적용 가치: **낮음** — dense BCH/RS 대상 OSD/BMA+ABP 반복복호로, NAND용 binary QC-LDPC min-sum 디코더에 이식할 재사용 요소가 사실상 없음.
