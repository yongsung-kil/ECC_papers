### B. 알고리즘 요약

1. LDPC BP 복호를 min-sum으로 근사; 비선형 함수 `Ψ(x)=log((1+e^-x)/(1-e^-x))`가 최소 입력 `qmin`에 지배됨.
2. 문제: 순수 min-sum은 저SNR에서 큰 성능손실; 고정 정규화[8]는 부호별 단일 α로만 보정.
3. 정규화 min-sum: `r_mn ≈ (1/α)·sign·min|q|`, `α = E[min|q|]/E[Ψ(ΣΨ|q|)]` (부호 고정, 모든 SNR 동일).
4. 핵심: 동적 정규화 `r_mn ≈ β·sign·min|q|`, β를 최소입력 `qmin` 조건부로 선택 (식 10,11).
5. 분석: β는 check node degree가 클수록 작아지고, `qmin≥T`에서 qmin이 클수록 β 증가 (Gaussian 입력 가정).
6. β 직접 계산 곤란(분포가 SNR·iteration 의존) → `β∈[βmin,1]`, `βmin=Ψ(dc·Ψ(qmin))/qmin`.
7. 실용화: 2단 선택 `β=B1 if qmin≤K else B2`; `B1=0.5, B2=0.75`(2^-x 형태 → shifter+adder만 필요).
8. 임계값 `K`는 B1,B2 고정 후 시뮬로 결정 (rate-3/5: `K=1.125`, rate-9/10: `K=1`).
9. 결과: DVB-S2 64800-bit LDPC, fixed-point, 최대 50 iter; rate-3/5에서 순수 min-sum 대비 ~1.0dB SNR 개선, rate-9/10은 (고SNR 영역이라) 개선 미미.
10. 한계: HW 합성/gate count 없음, 이득이 저SNR/저rate 집중, β 선택이 degree·qmin 통계 가정에 의존.
