### B. 알고리즘 요약

1. 표준 sum-product(SPA)는 매 iteration마다 모든 check/bit node를 갱신 — LDPC 복호의 연산·전력 병목.
2. 문제: BER 성능 손실 없이 노드 갱신 연산량을 줄이는 것.
3. 핵심(SNU): syndrome check에서 unsatisfied인 check node를 `unreliable`로 태깅하고, 그 이웃 bit node도 `unreliable`로 표시 — 나머지 reliable 노드는 passive.
4. 갱신 규칙: `if(unreliable_checks[j]) do CN op`, `if(unreliable_bits[i]) do VN op`만 수행, boolean 벡터 `unreliable_checks[M]`/`unreliable_bits[N]`로 추적.
5. 중요 precaution: 한 번 unreliable로 태깅된 노드는 이후 satisfied가 되어도 복호 끝까지 계속 갱신(TRUE→FALSE 불가) — 교정된 비트가 오류로 되돌아가는 것 방지.
6. 초기화는 SPA와 동일(`Li = 2yi/σ²`), CN 연산은 표준 `Rji` (sign×min 형태) 그대로 사용.
7. 구현: 표준 SPA에 boolean 마스킹만 추가 — 추가 HW 복잡도 negligible.
8. 결과: WER 10^-5에서 (96,48) 코드 CN 80%/BN 91.8%, (504,252) 코드 CN 41%/BN 72.7% 연산 절감, BER 손실 ≤0.1dB.
9. 결과 2: 절감량은 SNR(저WER)에서 커지며, 직렬 구현에서 메모리 접근 감소로 복호 속도↑, 전력↓.
10. 한계: 평균 iteration은 대체로 증가(수렴 느려짐), 고WER(>10^-2)에선 bit node 연산 23% 증가, HW 미설계·시뮬(AWGN/BPSK)뿐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SNU 노드 선택 갱신 로직 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 반복 루프에 활성노드 마스킹 추가 가능하나 병렬 파이프라인과 정합 필요 |
| unsatisfied check 판정(syndrome) | `partial_CRC.cpp` `partial_crc_HW_T4()` / `full_CRC.cpp` | syndrome 기반 활성화는 조기종료 인프라와 개념 유사, 재사용 여지 |
| CN/VN 선택 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 갱신 대상 선택 마스크만 추가 |

적용가치: 낮음 — 직렬 메모리접근 절감 위주라 z=32 병렬 파이프라인 구조엔 이득이 불명확하고, 평균 iteration 증가 리스크가 있다.
