### B. 알고리즘 요약

1. array LDPC(ALDPC) 부호를 Xilinx VirtexE FPGA에 구현: `(3,6)` regular, `p=173`, `N=1038`, `K=519`, rate 1/2.
2. 문제: 랜덤 LDPC는 VN-CN 배선이 무작위라 라우팅이 칩면적 지배(문헌[3]은 라우팅에 50% 면적) — 병렬화 활용이 어려움.
3. 부호구조: `H`가 단위행렬 `I`와 순환치환행렬 `E`의 `j×k` 블록으로 구성(quasi-cyclic)되어 배선이 국소화됨.
4. 디코더: belief-propagation(sum-product), 메시지는 log 도메인 LLR, 정밀도 `4-bit`.
5. 핵심 아이디어: circular shift로 bit→check edge 패턴을 생성해 라우팅 문제 회피; edge를 한 클록에 다 처리하지 않고 **직렬** 처리.
6. 구성 유닛: BNU(bit node), CNU(check node), CTU(temporary check node) — H를 row 단위로 처리.
7. 처리 흐름: 각 `p` row 처리=6클록 + CNU→CTU 복사 1클록 = row당 7, row수 3 → iteration당 21클록(복사 생략 시 18).
8. 검증: Verilog RTL을 HDL 시뮬로 검증, BER이 C 시뮬과 근사 일치, PC 병렬포트로 FPGA 동작 확인.
9. 결과: 자원 slices 10883(56%), 합성 43MHz→P&R 후 26.3MHz(라우팅이 임계경로의 81%), 18 iteration에서 throughput 72Mbps(문헌[4] 54Mbps 대비 ~130% 빠름, 전력 ~50%↓ 추정).
10. 한계: rate-1/2 저rate, 구식 FPGA, 26MHz·72Mbps로 throughput 낮음, BER 개선 없이 순수 HW 구현.
