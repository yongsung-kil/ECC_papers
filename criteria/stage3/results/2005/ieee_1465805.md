### B. 알고리즘 요약

1. 대상: 길이-2048 (2048,1723) RS-based Gallager (6,32)-regular LDPC(rate≈0.84), hard-decision message passing, fully-parallel 디코더 (0.18µm CMOS).
2. 문제: LDPC의 랜덤 H-matrix가 fully-parallel 디코더에서 복잡한 배선/라우팅 혼잡을 유발해 면적·속도를 악화([7]에선 칩면적 50%가 라우팅 때문에 낭비).
3. 핵심기법1 (Broadcasting): inverse-check 함수(`⊟`)를 check 노드에서 variable 노드 내부로 이동시켜 각 check 노드가 모든 인접 variable에 **하나의 공통 메시지만 broadcast**하도록 변경.
4. 효과: 개별 메시지 대신 공통 메시지 → 배선 공유 가능 → top-level check-to-variable 배선 총량 40%+ 절감, 추가 연산 HW 비용 없음, 정정성능 무손실.
5. 핵심기법2 (Column reordering): floorplan 좌표와 H로 Manhattan 거리 cost map `D` 구성, 임계 `α·Lmax` 초과 위치를 금지영역 `K`로 표시.
6. 배치: `K`의 가장 제약 큰 열부터 금지영역에 '1'이 없는 H의 열을 매칭해 `Ĥ`(열 재배열) 생성, `α`를 점진 축소하며 반복 → 최대 배선길이 단축.
7. 성질: `Ĥ`는 H의 열 재배열일 뿐이라 코드 구조 불변, 임의 regular/irregular H·임의 floorplan에 적용 가능, generic 배치툴보다 1 order 이상 빠름.
8. Full-broadcasting 확장: variable 출력도 공통 메시지화 가능하나 추가 inverse 함수·저장소가 필요해 배선이득이 상쇄될 수 있어 미채택.
9. 결과: 4.2mm×4.2mm die, 32 iteration/block, 2 clock/iter, >100MHz, 3.2Gbps; 배선 40% + 최대 배선길이 37% 감소.
10. 한계: hard-decision·fully-parallel 특화(정정성능 개선 없음), gatecount 미보고, soft-info 미지원.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| Broadcasting (inverse-check를 VN으로 이동) | 대응 없음 (fully-parallel 배선 전용; `decoder.cpp` C2V/VN 로직과 목적 상이) | fully-parallel 배선 절감 기법이라 z=32 부분병렬 pipeline 구조엔 부적합 |
| Column reordering 배치 | 대응 없음 (물리배치/floorplan은 시뮬 범위 밖) | H 열 재배열은 코드 불변 최적화지만 HW P&R 단계 이슈 |
| 메시지 패싱 이터레이션 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `C2V_Cal()` | 알고리즘 자체는 표준 MP, 신규 기여 아님 |

적용 가치: **낮음** — fully-parallel 디코더의 배선/물리배치 최적화라 부분병렬(z=32) QC-LDPC 시뮬레이터에는 문제설정·이득이 이식되지 않음.
