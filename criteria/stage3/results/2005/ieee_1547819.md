### B. 알고리즘 요약

1. 시스템: binary regular LDPC(column weight 3, rate 1/2), 랜덤 구성 `(1008,504)`·`(504,252)`, BPSK/AWGN, 최대 iteration 10/50/80.
2. 문제: 표준 SPA의 CN update `2·tanh^{-1}(∏ tanh(·))`가 저BER에서 error floor 발생 — 고SNR에서 채널 신뢰도가 초기 iteration의 extrinsic보다 커짐.
3. 핵심 기법: tanh 함수를 정규화하는 modified tanh — `|x|>x0`이면 `sign(x)`, 아니면 `tanh(x)` (clipping). tanh의 무한대 인자 근사 겸용.
4. 핵심 파라미터: `x0` 최적값 탐색 → `x0=7`(tanh(7)=0.999998) 선택; x0=3/10은 오히려 저하, 7/8/9가 최적.
5. 근사1: tanh를 7구간 piecewise-linear로 근사(Table I, RMSE≈0.02), 역tanh에도 동일 적용.
6. 근사2: 3-bit coarse quantization LUT(8값, Table II, RMSE≈0.07); x0=3/7이면 LUT를 8→6, 구간 7→5로 축소 가능.
7. 구현 디테일: piecewise-linear가 수치 overflow를 회피해 modified tanh 대비 +0.13dB, quantization은 -0.25dB 저하.
8. 학습/최적화: 없음(시뮬로 x0 스윕).
9. 결과: modified tanh로 error floor 제거 + 저BER(1e-6/1e-7)에서 약 0.5dB coding gain, 결과가 확률영역 SPA와 일치.
10. 한계: HW 미설계(복잡도 정량 없음), 소형 랜덤 rate-1/2 부호·시뮬만, full-tanh SPA 대상(min-sum 아님).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA CN update tanh 연산 / modified tanh | 대응 없음 | Prime은 min-sum(`CNU_Update_New_Mag()`)으로 tanh 미사용, full-tanh Sum-Product는 대응 없음 |
| LLR clipping/정규화(error floor 대책) | [ecc_data.h](Prime_ECC_3.1_Claude) `PARAM_LLR`, [decoder.cpp](Prime_ECC_3.1_Claude) `Get_VNU_Table_Idx()` | 적응형 LLR·magnitude 양자화로 개념 중복 |
| tanh piecewise-linear / 3bit LUT | 대응 없음 | min-sum엔 tanh LUT 불필요 |

적용 가치: **낮음** — 핵심 기여(tanh clipping·tanh LUT)가 min-sum 기반 Prime에 무의미하고, error-floor용 LLR 정규화는 이미 magnitude 양자화·적응형 LLR로 보유.
