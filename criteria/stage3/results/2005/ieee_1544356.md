### B. 알고리즘 요약

1. 대상: regular quasi-cyclic LDPC(circulant 순환이동 단위행렬로 PCM 구성), `(3,5)`-regular, block 305/1655/3005 및 FPGA 구현은 12288-bit rate1/2.
2. 문제: LDPC 부호 품질이 coding gain과 복호기 HW 복잡도를 함께 좌우 → 부호설계와 HW 아키텍처의 밀결합 필요.
3. 핵심 기법: GA 기반 부호 탐색 엔진 - algebraic 구성법 + BER 시뮬 + 아키텍처 제약을 동시 스크리닝 기준으로 사용.
4. GA 파라미터: 50 generations, population 30, crossover 0.8, mutation 0.2, decoding iteration 100; fitness에 average girth 반영.
5. QC 구조로 복호 시 메모리 주소 생성이 단순해지고 block 단위 처리로 오버랩 파이프라인·멀티스레드 복호가 쉬워짐.
6. 복호기 아키텍처: CNFU/VNFU 2-phase 오버랩 파이프라인, semi-parallel(다중 CNFU/VNFU 동시).
7. 핵심 스케줄: Jump-Reset 알고리즘으로 CNFU/VNFU가 dual-port 메모리를 충돌 없이 접근, 버블 없는 스케줄로 HUE(하드웨어 이용효율) 극대화.
8. 스케줄 제약이 PCM 생성에 역으로 부과됨: time offset `w < m/2`(m=circulant 서브행렬 크기)를 GA 평가함수로 사용, 낮은 w는 latency도 감소.
9. 결과: GA 탐색 부호가 random/algebraic 부호와 동등 이상 성능(short-medium); FPGA(Altera EPXA10) 12288-bit rate1/2 구현, 3064 LCs, 32.64MHz, 19.11Mbps.
10. 한계: 극단적으로 짧은 논문으로 BER 곡선·gate count·연판정 비트폭 등 수치 부재, 성능 "동등" 수준(정정능력 향상 아님), 저rate·통신용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| regular QC-LDPC PCM 구성(circulant) | ecc_top.cpp Load_PCM(), H-matrix | Prime ECC도 QC-LDPC(base+circulant shift)라 구조 정합, 단 부호 고정으로 재검증 부담 큼 |
| HW 제약 반영 GA 부호탐색 | 대응 없음 (부호탐색 도구 미보유) | fitness에 아키텍처 제약 넣는 방법론 참고 가능 |
| Jump-Reset 메모리 스케줄/오버랩 파이프라인 | decoder.cpp LDPC_Decoder() pipeline (z=32), GT.cpp | 우리 6-stage pipeline·GT와 목적 유사(충돌 없는 메모리 접근), 구조 정합 필요 |
| offset 제약 w<m/2 (latency) | decoder.cpp 이터레이션 루프 스케줄 | lifting size 종속 스케줄 제약, z=32와 매핑 검토 |
```

적용 가치: **중간** — QC-LDPC 구조와 오버랩 파이프라인·메모리 충돌 회피 스케줄은 Prime ECC와 정합되나, 부호 성능은 기존 대비 향상이 없고 수치가 빈약하며 부호탐색은 특정 QC 고정된 우리 코드에 재검증 부담이 커 신규 이식 가치는 제한적.
