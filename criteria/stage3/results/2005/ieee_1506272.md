### B. 알고리즘 요약

1. 시스템: (3,6)-regular rate-1/2 LDPC, frame 2640 symbol을 Rayleigh 블록페이딩 채널로 전송(블록당 `W`심볼, 블록수 `U`).
2. 문제: 채널상태 `s_i`(페이딩 진폭)가 미지·연속이라 SPA가 연속변수 함수 메시지를 전달해야 해 복잡도 폭증, 상태노드 고차수로 short cycle 발생.
3. 모델: 각 블록에 하나의 상태 `s_i`가 부여되며 `y_ij = s_i x_ij + n_ij`(AWGN), 상태는 Rayleigh 사전분포.
4. 핵심기법: EM 알고리즘을 factor graph 메시지패싱으로 재정식화한 FGEM — hidden `x`(코드심볼)는 E-step, 파라미터 `s`(채널상태)는 M-step에 배치.
5. E-step: `s=ŝ` 가정 하 SPA로 `f(x_ij|y)` 계산(=기존 LDPC 복호), soft decision `x̂_ij` 스칼라만 추출.
6. M-step: 상태노드+부속 factor로 구성된 M-step 그래프에서 max-product SPA로 `s`의 최대화값 산출(Rayleigh prior 반영, `ŝ_i = Σ x̂_ij·y_ij / (W ...)`).
7. 핵심 성질: E-step/M-step 그래프가 각각 tree이면 원 그래프에 cycle이 있어도 EM이 정확 구현 — LDPC의 short cycle 문제를 저복잡도로 해소.
8. Slepian-Wolf 예: 동일 프레임으로 상관 이진소스 압축, 상태 `s_i`=반전확률, `Q_ij`를 log 형태로 M-step 처리.
9. 결과: 블록페이딩에서 one-stage/무기억가정 검출기 대비 유의미한 SNR 이득(정성적, 곡선만 — 수치 미제시).
10. 한계: HW 미설계, 시뮬 곡선만, 미지 연속 채널상태 추정에 특화 — NAND의 고정 read 채널에는 부적합.
