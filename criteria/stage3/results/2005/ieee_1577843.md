### B. 알고리즘 요약

1. 대상: AWGN/BPSK, LDPC BP 복호의 스케줄 최적화. 분석용 `(3,6)` regular(rate 0.5), 실증용 `(16200,7200)` irregular(DVB-S2 초기판) 사용.
2. 문제: 표준 2-step BP는 iteration `i`에서 직전 iteration 값 `V^(i-1)`만 사용 → 수렴 느림(latency↑).
3. 핵심기법1(shuffled BP): 수평/수직 step을 섞어 같은 iteration 내 먼저 갱신된 `V^(i)`(`n'<n`)를 즉시 활용, 나머지는 `V^(i-1)`(`n'>n`) 사용 (식 3).
4. 의미: 더 독립적·최신 정보로 메시지를 갱신할수록 신뢰도↑ → 수렴 가속. 표준 BP 대비 약 2배 빠른 수렴.
5. 핵심기법2(replica shuffled BP): 서로 다른 갱신 순서(오름/내림차순)의 여러 subdecoder를 동시 협력, iteration 후 서로 더 신뢰도 높은 메시지 교환(synchronous/non-synchronous).
6. 결과 관찰: subdecoder 수↑일수록 수렴↑, synchronous가 non-synchronous보다 빠름(메모리는 적게 쓰나 접근 빈번).
7. 분석1(DE): all-zero·cycle-free 가정하 bit-index별 pdf 추적(식 10,13), check 노드 재귀연산 `Ψ` (식 7~9)로 효율 계산.
8. 분석2(EXIT): VN/CN EXIT 곡선 폐형식 유도(식 15~24), shuffled의 이상화 모델(PEG 그래프 근사)로 CN EXIT 도출.
9. 핵심 정리(Theorem 1): shuffled/replica shuffled BP는 스케줄만 바꾸므로 tree 기반 **threshold를 개선하지 못함**(속도만 개선).
10. 결과 수치: replica shuffled BP `Imax=10,G=32`가 표준 BP `Imax=70`과 동등 성능(≈7x iteration 감소); group 버전은 병렬성 유지. HW 미설계.
