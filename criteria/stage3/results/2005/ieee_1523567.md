### B. 알고리즘 요약

1. 대상: rate `R=1/2` (J,2J)-regular LDPC convolutional 부호 ensemble `CP(J,2J,M)`. syndrome former `H^T`가 M×M 랜덤 순열행렬 블록으로 구성, syndrome former memory `ms=J-1`. binary-input output-symmetric memoryless 채널(AWGN/BEC).
2. 문제: (J,2J) 블록 LDPC의 BP 임계값이 용량과 큰 격차 — convolutional 구조로 이를 좁힐 수 있는가.
3. 핵심 관찰: 부호를 종단(termination)하면 그래프 양끝의 심볼이 0으로 알려져 check node 차수가 2/4/6으로 낮아지는 **구조적 불규칙성** 발생 → 강한(저차수) check가 그래프 전체로 신뢰 전파.
4. 종단 rate: `R=0.5/(1+J/L)`, L>>J이면 rate 손실 무시. 예: rate 손실 2% 위해 R=0.49.
5. 디코딩: Tanner graph 위 BP(sum-product), check node `β(j)=2arctanh(∏tanh(z/2))`, symbol node `z(j)=α+Σβ`. 위치별로 message pdf가 달라 시간레벨마다 별도 pdf 추적.
6. 독립반복 보장: 정리1 — 임의 L에 대해 독립 디코딩 반복수 `l0 > 2logM/[log(2J-1)(J-1)] - c1` 존재.
7. 수렴 분석: 각 edge의 Bhattacharyya parameter `B^(l)_(t,t+k)` 재귀부등식(Lemma1)으로 추적, breakout value `Bbr` 이하로 떨어지면 오류확률이 **이중지수적**으로 0 수렴.
8. 임계값 계산: 표준 병렬 스케줄 대신 **슬라이딩 윈도우**(윈도우 W 내 심볼 반복 갱신 후 시프트) 도입 — 연산량 대폭 축소, 비윈도우 스케줄과 동등함을 Proposition으로 보증.
9. 결과(Table I): AWGN 임계값 (3,6) 0.55dB, (4,8) 0.35dB, (5,10) 0.30dB — 블록 (3,6) 1.11dB 대비 크게 개선되고 J↑에서 rate-1/2 용량 한계로 접근.
10. 한계: 순수 density evolution 임계값 해석(HW·유한길이 BER 곡선·error-floor 미제시), 중간 노드 수렴 반복수가 L에 선형 증가(latency 부담).
