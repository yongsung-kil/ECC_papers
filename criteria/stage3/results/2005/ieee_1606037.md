### B. 알고리즘 요약

1. LDPC 부분병렬 디코더의 실병목은 "무작위" 배치된 bit/check node 간 통신(라우팅)과 메모리 접근 — 다수의 작은 메모리 블록은 VLSI 면적 비효율.
2. 문제: 여러 작은 메모리 대신 단일 area-efficient 메모리 블록으로 collision-free 병렬 접근하기.
3. 핵심 제약: M개 bit node 그룹에서, 첫 노드의 인접 check가 `a1..adv`면 i번째 노드의 인접 check는 `{ak+(i-1)p} mod(N-K)`, `p=(N-K)/M` — 주기적 shift 구조.
4. 자유상수 `a1..adv`는 무작위 탐색으로 cycle-4 free·cycle-6 최소화하도록 선택(성능 확보).
5. 추가로 `H=[A | B]`에서 B를 staircase lower-triangular로 고정 → 재귀적 parity 계산으로 간단 인코딩, degree-2 노드 간 short cycle 부재 보장.
6. Edge RAM을 top(degree>2 edge)/bottom(degree≤2 edge)로 개념 분할하되 물리적으로 단일 블록, 열 수 = M(병렬엔진 수).
7. check node 처리: top RAM `q=dc-2`행 + bottom 2행을 연속 접근(ROM 불필요), bit node 처리: 행 인덱스와 첫 노드 열 오프셋 `j`만 ROM 저장(barrel-shift로 나머지 유도).
8. M 선택식 `M=(b·i·e·f)/c` (b=코드비트율, i=최대iter, c=클록, e=평균 bit degree, f=4); 예: b=150Mbps,i=50,e=3.5,c=292MHz → M=360.
9. 결과: N=64800, 11개 rate(1/4~9/10)를 M=360으로 설계, DVB-S2 표준 채택, DVB-S1(conv+RS) 대비 >35% 용량개선, Shannon 한계로부터 0.7~0.8dB, ROM 면적 <0.1mm².
10. 한계: 복호 알고리즘 자체는 [13]에 위임(본 논문 무관), 성능검증은 AWGN 시뮬, 위성/DVB 특화 긴 블록.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 구조화 PCM(주기적 shift) 부호 설계 | `ecc_top.cpp` `Load_PCM()` (+ H-matrix) | QC-LDPC 로드와 대응, 단 특정 IRA 구조 재설계 부담 |
| staircase(dual-diagonal) 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | dual-diagonal 인코딩 이미 보유 — 중복 |
| 단일 메모리 collision-free 접근 / edge RAM 배치 | 디코더 HW 모델(z=32 row 병렬) | QC circulant 병렬로 이미 collision-free 달성 — 개념 중복 |
| M 병렬엔진 sizing 공식 | 대응 없음 | z=32 고정이라 직접 대응 함수 없음 |
