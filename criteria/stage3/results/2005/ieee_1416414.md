### B. 알고리즘 요약

1. 시스템: LDPC 외부부호 + ISI 채널(rate-1 내부부호)의 turbo equalization, 자기기록 채널 등 고rate 코드 대상.
2. 문제: BCJR/SOVA/DAE 채널 SISO는 순차적, LDPC는 병렬 선호 → 중간 soft-info 저장 메모리·접근이 고속 turbo-eq의 병목.
3. 복호 알고리즘: sum-product의 LLRXOR를 `LLRXOR(y,z)≈ -sign(y)sign(z)min(|y|,|z|)`로 근사한 signed sum product(SSP, 즉 min-sum)로 CN 연산.
4. 관찰: SSP에서 equation의 extrinsic 크기는 llrQ 중 최소 두 값(`min1`,`min2`)만으로 결정, 나머지는 부호에만 기여.
5. 핵심기법: edge당 `llrRml` 저장(`ML` 워드) 대신 equation당 `min1im`,`min2im`, min1 위치 `Iim`, 전체부호 `sim`, 각 llrQ 부호만 저장.
6. 식 (6): `llrRml = -sim·sign(llrQlm)·min1` (l≠I), `min2`(otherwise)로 llrR를 on-the-fly 재계산.
7. 메모리: llrP용 `L`, llrQ 부호용 `ML` bit, equation 통계용 `(L-k)` 워드 → 고rate에서 `(L-k)` 작아 llrP가 지배.
8. 지연: llrQ 계산과 llrR 계산을 하나의 채널 SISO 스윕에 흡수 → iteration당 `2L`→`L` 클록, LDPC 지연이 채널 지연에 흡수.
9. 조기종료: `sim`가 만족 시 -1이므로 별도 메모리 없이 parity check 가능, 성공 시 hard decision 출력.
10. 한계: (L,k) product code(girth>4, column weight 2, dmin 4) 예시뿐, gatecount/throughput/BER 수치 없음, 순수 아키텍처 제안(이론 단계).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SSP(min-sum) CN 연산 (min1/min2/sign) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리도 min1/min2 min-sum 사용 - 기법 중복 |
| equation당 min1/min2/I/s 저장(llrR 메모리 제거) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 메시지 저장방식 대안, HW 메모리 최적화 관점 |
| llrR on-the-fly 재계산 | decoder.cpp C2V_Cal() | C2V 재계산 방식과 대응, 정합성 검토 필요 |
| parity 만족 조기종료(sim) | partial_CRC.cpp / full_CRC.cpp | 우리는 CRC 조기종료 - 방식 상이 |
```

적용 가치: **중간** — min1/min2/sign만 저장하는 min-sum 메모리 절감 아이디어는 우리 min-sum HW 메모리 설계에 참고되나, min-sum 자체는 이미 보유이고 ISI turbo-eq·product code 전제라 QC-LDPC(z=32, pipeline)로의 직접 이식가치는 제한적이다.
