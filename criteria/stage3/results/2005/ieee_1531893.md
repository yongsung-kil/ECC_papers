### B. 알고리즘 요약

1. 시스템: `P`개 프로세서 병렬 반복복호기(Turbo=2 constituent decoder, LDPC=VN/CN), 총 `L`개 LLR을 `M=L/P` 시간구간에 걸쳐 메모리 접근.
2. 문제: 여러 프로세서가 같은 시각 같은 메모리 뱅크 접근 시 collision → 직렬화로 병렬성 훼손, 고데이터율에서 반복복호 latency 병목.
3. 모델: LLR을 `M(ℓ)`-번째 메모리뱅크에 `S(ℓ)`-번째 시각에 접근하도록 (mapping function `M`, scheduling function `S`) 쌍 정의, 좌/우 spatial·temporal clustering 사이 인터페이스 설계.
4. 기법 1단: `[3]`의 mapping function을 crossbar 복잡도 관점에서 재해석 - `P×M` mapping matrix에서 tiling 제약(같은 열·타일에 같은 값 금지)으로 충돌 회피, crossbar 복잡도 하한식 유도.
5. 핵심 식: crossbar 복잡도 하한 `Σ(Λ*1+Λ*2)` (각 행이 서로 다른 값을 최소화할수록 crossbar 단순화) → mapping function 품질을 하한과의 거리로 평가.
6. 기법 2단(dual): mapping을 left spatial clustering으로 고정하고 scheduling function을 annealing으로 설계하는 dual problem 제시(그래프 채색 구조 동일), `P` 클 때 crossbar 한쪽 제거 가능해 LDPC에 유리.
7. 구현 디테일: greedy 초기화(행을 좌→우로 충돌없이 채움) vs zero 초기화(전부 blank) 후 annealing, scheduling은 좌/우 인터리버뱅크 + 중앙 P×P crossbar 3층 구조로 실현.
8. 최적화 절차: annealing이 blank를 채우며 이웃에서 최소 반복되는 label 선택, N-branch 일반화는 K-valued mapping function = multihypergraph의 edge-coloring(chromatic index) 문제로 정식화, `K_N ≤ min(L, N(P-1))`.
9. 결과: 1000개 random permutation 앙상블에서 fractional performance 평균이 greedy 64.5~70% vs zero 57~64%로 greedy 초기화가 crossbar 복잡도 우수(특정 M,P에서 격차 큼).
10. 한계: annealing이 `K≥K*(P)`서 항상 성공한다는 증명 없음, 모든 중간 mapping 저장 필요(L 크면 부담), 실HW 합성/gate count·throughput 없음.
