> 총평: sum-product 디코더를 블랙박스로 두 번 EI 섭동시켜 LDPC 최소거리·distance spectrum을 추정하는 오프라인 분석 알고리즘으로, ECC 엔진에 이식할 기법은 아니나 부호 error-floor 특성화 툴로서 부수적 참고가치만 있다.

### B. 알고리즘 요약

1. 대상: systematic (n,k) LDPC, BPSK·AWGN, rate 0.5 예시 다수, 길이 114~4000, 3-bit LLR 양자화 sum-product 복호.
2. 문제: parity-check `H`가 커서 LDPC 최소거리 `dm` 직접 계산은 NP(다항시간 불가), 저BER 성능 예측에 필요.
3. 배경: Error Impulse(EI) 기법[2,3]은 all-zero 송신 가정 하에 디코더를 발산시키는 최소 impulse 강도가 `dm`과 같다는 유클리드 기하 논리.
4. 핵심기법: EI를 **2단(two-level)** 적용 — 1단은 위치 `i`에 강도 `A`를 키워가며 디코더를 all-zero에서 발산시켜 근접 codeword `ci` 획득.
5. 2단: `ci≠0`이면 모든 `j≠i`에 두 번째 impulse(강도 `Am`)를 추가로 넣어 `c` 근방의 서로 다른 codeword를 열거.
6. 산출: 단순 `dm`뿐 아니라 codeword **다중도**와 입력 error event의 Hamming weight까지 enumerate해 distance spectrum 구성, `dm = min{dm,i,j}`.
7. 상계 검증: `Hpu`(double-diagonal)를 identity로 환원한 systematic 형태에서 `Hu` 최소 열가중치+1로 `dm,b` 상계를 구해 알고리즘 결과 타당성 확인.
8. 구현 팁: `dm`만 필요하면 while 종료 시점의 최소 `A`로 `Am`을 낮춰 반복수 절감, 유효 codeword를 조기 발견해도 `Nit`까지 계속 iterate하면 결과가 더 좋음.
9. 결과: 6개 LDPC(L1~L6, dm 6~20)에서 FER 시뮬과 dmin 항만 쓴 상계(U.B.)가 잘 일치, 소형 H는 육안 검증으로 정확한 dm 확인; 기존 EI는 때때로 `dm,b`를 초과하는 느슨한 추정.
10. 한계: ML 디코더 가정 기반이라 sum-product suboptimality·짧은 cycle 존재 시 이론적 정확성 미보장, HW 미설계 오프라인 특성화 도구.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sum-product 디코더를 EI로 섭동 실행 | decoder.cpp LDPC_Decoder() | 디코더를 분석대상 블랙박스로 호출할 뿐, 알고리즘 변경 없음 |
| all-zero + error impulse 주입 | channel.cpp Set_R_Offset() (fixed-error 주입) | 특정 위치 오류 패턴 주입과 개념적 유사 - 분석용 |
| 최소거리·distance spectrum 추정 | 대응 없음 | 오프라인 부호 특성화, 코드베이스 기능 아님 |
| dm,b 상계(systematic 환원) | ecc_top.cpp Load_PCM() (H-matrix 참조) | H 구조 분석 지점, 직접 대응 아님 |
```

적용 가치: **낮음** — Prime ECC에 떼어 넣을 디코더/부호설계 기법이 아니라 LDPC 부호의 최소거리·error-floor를 오프라인으로 추정하는 검증 도구다. 특정 QC-LDPC의 error-floor 특성화에 부수적으로 쓸 수 있으나 엔진 이식 대상은 아니다.
