### B. 알고리즘 요약

1. 대상: RA/IRA/ARA 계열을 확장한 loop-free 모듈 기반 LDPC, AWGN/BPSK, rate 1/2~8/9(펑처링), 입력 블록 k=1024/4096.
2. 문제: 일반 irregular LDPC나 IRA는 매우 낮은 threshold를 위해 일부 노드의 최대 반복(=max VN degree)을 매우 크게 요구 - HW 부담.
3. 핵심 기법: accumulator, repetition, differentiator, punctured single-parity-check(SPC) 같은 단순 loop-free 모듈을 직렬/병렬 연접해 부호 구성(ARA=Accumulate-Repeat-Accumulate 등).
4. protograph 표현: 작은 Tanner graph(protograph)를 "copy-and-permute"로 T배 lifting - 동일 rate/degree 분포 유지하며 고속 인/디코딩 가능.
5. ARA: 반복부호 앞에 punctured accumulator를 precoder로 두어 high extrinsic-SNR 영역 개선(Gaussian DE의 EXIT 곡선 근거), rate-1/2에서 capacity 0.08dB 이내 threshold를 낮은 max degree로 달성.
6. 확장: 내부 accumulator 앞에 SPC 추가=ARCA(threshold 더 낮춤), accumulator를 하나 더=ARAA(RAA의 낮은 error-floor + ARA의 낮은 threshold 결합).
7. 부수: parity 비트/accumulator 펑처링만으로 rate-compatible 부호군 생성 - AR4A/ARCA/ARAA가 rate 1/2~8/9 전구간에서 capacity에 균일 근접.
8. 설계 도구: threshold는 실제 LDPC density evolution으로, EXIT/extrinsic-SNR 분석은 Gaussian DE로 산출.
9. 결과: FPGA로 구현된 JPL LDPC 디코더로 시뮬, AR4A rate-1/2 k=4096이 Flarion low-error-floor 코드와 비교; ARAA가 AR4A보다 error-floor 낮음(threshold는 소폭 높음).
10. 한계: turbo-like/RA 계보 protograph 부호로 gatecount/throughput 미제시, 저~중rate 중심(고rate는 펑처링 파생), 디코더 병렬 아키텍처는 protograph "one page/clock" 수준 개념만.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph 부호 구성/lifting | ecc_top.cpp Load_PCM() + H-matrix | 부호구조 설계, 우리 QC(base+circulant)와 다른 계보(copy-and-permute) |
| accumulator/dual-diagonal 인코딩 | encoder.cpp LDPC_encoder_4KB() | ARA의 accumulator와 우리 dual-diagonal 개념적 유사, 구조 상이 |
| protograph 병렬 디코더(one page/clock) | decoder.cpp LDPC_Decoder() | 병렬화 개념 참고, 우리 z=32 row-병렬과 정합 필요 |
| punctured-SPC / accumulator precoder | 대응 없음 | turbo-like 모듈, Prime ECC 미보유 |
| Gaussian DE / EXIT 기반 threshold 설계 | local_opt.cpp (오프라인 설계) | 설계 도구, 코드 직접 대응 미약 |
```

적용 가치: **낮음** - ARA/ARAA/ARCA는 turbo-like RA 계보의 protograph 부호로 error-floor/threshold가 우수하나, Prime ECC의 고rate binary QC-LDPC(circulant) 구조·NAND soft-read 요구와 계보가 달라 부호설계 자체를 이식하기 어렵고 재검증 부담이 크다.
