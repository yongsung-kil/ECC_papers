### B. 알고리즘 요약

1. 시스템: 4092-bit, rate 0.75, row weight 32 binary LDPC를 finite-field geometry(`GF(2^5)`)로 구성, comm/storage용 고처리량 디코더.
2. 문제: fully-parallel 디코더는 4092-bit에 130nm 기준 130mm² 이상 필요(routing congestion); serial은 면적 작으나 memory 크고(16384 edge×5-bit≈10kB) throughput 낮음.
3. 부호구성: `1023×1023` cyclic finite-geometry 행렬을 1:4 column splitting해 `1023×4092`로 확장, 각 check node가 연속 VN 4개 중 최대 1개에만 연결.
4. 아키텍처: V2C 메시지를 4개 shift-register chain에 분산, 32개 위치를 MUX로 단일 check-node 계산블록에 입력, 32개 VN 계산블록.
5. staggered(layered) 스케줄(`(1)(2)`): 매 step 부분집합 `S(k)` check node만 갱신, VN당 메시지 `Q'n` 1개만 저장 → memory 87% 절감 + 수렴속도 향상.
6. check-to-variable 계산: `Θ(x,y)=Φ⁻¹[Φ(x)+Φ(y)]≈min{x,y}+f(x,y)`, correction `f`는 2D LUT(값 0~0.75), binary tree 구현 — 비선형 `Φ`의 양자화 오차 회피용 reduced-complexity(offset-min-sum류).
7. 5-bit fixed-point LLR 메시지, floating-point 대비 BER 열화 <0.1dB.
8. 22-stage pipelined check-to-variable 계산(그림에선 3-stage 예시).
9. 결과: 130nm 6M CMOS test chip, 256k gate/1.8M Tr, core 4.0mm², 1.16GHz@1.2V, 1.5W, 1.16Gb/s/iteration.
10. 한계: throughput이 iteration 수에 반비례; 아키텍처가 finite-geometry column-split 부호에 특화; 추가 병렬화는 future work.
