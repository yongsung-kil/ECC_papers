### B. 알고리즘 요약

1. 목적: LDPC의 error floor를 매우 낮은 BER(`10^-10`)까지 관측하기 위한 고throughput FPGA 시뮬레이터 개발(PR/EPR4 채널).
2. 문제: 순차적 C 시뮬은 `10^12` bit에 최적화 코드·2GHz로도 수개월 소요 — 저BER error-floor 분석 불가.
3. 아키텍처: write/read processor + RNG + LDPC ENC + interleaver + 이상 PR 채널 + AWGN gen + soft 채널검출기(DET) + LDPC decoder + error counter, PCI로 PC(MATLAB/C) 연동.
4. RNG는 64-bit LFSR m-sequence(주기 `2^64-1`)로 비반복 난수 생성, error counter가 동일 시퀀스와 복호 출력을 비교해 BER/BLER 산출.
5. AWGN gen은 Box–Muller를 LUT+다항식·구간근사로 구현, PR 모듈은 feedforward shift register로 target 계수 가변.
6. 채널검출기 비교: MAP=Log-Map, Max-Log-Map=Bi-directional SOVA, two-step SOVA는 `10^-?`에서 약 0.2 dB 열화 최저복잡도 — FPGA엔 Bi-directional SOVA와 two-step SOVA 구현.
7. 핵심 난제: 일반화(generalized) LDPC 디코더 — H가 small regular submatrix로 구성된 broad class를 config 파일만으로 재구성해 부호 변경 없이 평가.
8. QC-LDPC 인코더는 simple shift register로 구현, 구조적(easily-encodable) 부호만 실용 대상으로 한정.
9. 결과: Xilinx Vertex II XCV6000, VHDL, synth clock 110 MHz, throughput 약 `185 Mb/s`; DDS-LDPC(`j=3,5`, rate 8/9)·array-LDPC(`j=3`) error floor를 `10^-10`까지 1.5시간에 관측(7-bit SOVA, 6-bit LDPC), PR-AWGN 약 2 dB차, 구현손실 약 0.2 dB.
10. 한계: 자기기록 PR 채널·SOVA 전제, 부호·복호 알고리즘 개선이 아닌 평가 플랫폼, logic consumption은 표(Table I)에만.
