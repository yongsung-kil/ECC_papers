### B. 알고리즘 요약

1. 시스템: IEEE 802.16 광대역 무선 인터넷용, OFDM 결합, 정보 블록 500~2000bit, rate 1/2, AWGN 채널.
2. 문제: 일반 LDPC는 인코딩 복잡도가 크고, 짧은 블록(500~2000bit)에서 최대 성능 이점을 못 낸다.
3. 제안: parity-check matrix에 트리 구조 제약을 준 tree-LDPC. 반복부호(repetition)+tree code의 직렬연접부호(SCC)로 표현.
4. 구조: tree code `H_t`는 recursive info bit `d_r`, non-recursive info bit `d_nr`, punctured parity `p_p`, non-punctured parity `p_np`로 구성.
5. 규칙: 한 정보비트가 최소 2개 recursive check sum에 속하도록(`J_r >= 2`) 설계해 성능 확보.
6. 인코딩: punctured parity는 recursive bit들의 SPC+accumulator, non-punctured parity는 punctured parity와 non-recursive bit들의 SPC로 얻어 선형시간 인코딩.
7. RA-like 관계: 각 행이 SPC이고 dual-diagonal part가 accumulator 역할을 해 RA 부호와 유사 구조로 해석됨.
8. 분석: Gaussian approximation 기반 density evolution으로 threshold 유도, 특정 interleaver에 대한 upper bound 제시.
9. 결과: tree-LDPC가 RA 대비 약 0.3dB, (3,6) 정규 LDPC 대비 약 0.4dB gain (BER 10^-7, 20/50 iteration), threshold도 우수.
10. 한계: HW 미설계, 시뮬만, 무선(802.16) 특화 짧은 블록 가정. NAND/SSD 무관.
