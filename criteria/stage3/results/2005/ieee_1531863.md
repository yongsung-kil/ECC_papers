### B. 알고리즘 요약

1. 대상: 이진 regular LDPC 부호의 하드판정 iterative 복호, 확장 parity-check matrix `H`(행 수 > n−k)와 syndrome `s=Hy'` 사용, 예제는 `(73,45,10)` PG/difference-set 부호.
2. 문제: 기존 bit-flipping(BF)은 coset leader를 임의 선택 — decodable error pattern과 부호 파라미터의 관계가 불명확.
3. 접근: syndrome weight `u`를 줄이는 위치를 flip하되 후보가 여럿이면 모두 분기하는 tree-structured search → 결과는 수신어에서 같은 거리의 codeword "list".
4. 핵심량: 위치별 parity failure 수 `f(i)`(그 열을 checking하는 행 중 syndrome이 1인 수), `f(i)>­/2`인 위치를 flip 후보로 선택.
5. 관계식: single-error syndrome weight `­`, error weight `w`의 syndrome weight `w­ − w(w−1) ≤ u ≤ w­`, min distance 하한 `­+1`, 유일 복호 조건 `w<(­+1)/2`.
6. 이론: Theorem 1(list BF 복호 필요충분조건: 모든 부분패턴에 `f(i)>­/2` 위치 존재), Lemma 2(`u>w(­−1)/2`면 최소 한 에러위치 flip 보장).
7. 목표: MLLD(Maximum-Likelihood List Decoding) — coset 내 최소 weight error pattern 전체를 출력.
8. 복잡도 제어: 복호 tree의 node 수로 측정, 해 없는 branch·중복 pattern 가지치기(`w` 상한, 기존 해와 overlap 과다 시 종료), 남은 에러 ≤2면 단순화.
9. 결과(`(73,45,10)`): min distance 10, 대부분 decodable pattern은 weight 7~8; 4-parity-failure인 6-arc coset(list size 21)까지 확장 flip으로 처리, coset을 `u`별 분류(Table I)해 거의 모든 case가 Lemma 2 만족 → MLLD 근접.
10. 한계: 하드판정·BF 계열, 시뮬/HW 없음(순수 조합 분석), 특정 PG/difference-set 부호 중심, 일반 부호에서 list size·복잡도 폭증 우려.
