### B. 알고리즘 요약

1. 대상: 반복 message-passing 디코더가 매 iteration마다 여러 복호 규칙 중 하나를 선택(gear-shift)해 target error rate를 최소 복호시간으로 달성하는 문제.
2. 문제: 고복잡도(sum-product)는 강하나 느리고, 저복잡도(min-sum/Alg.E/Alg.B)는 threshold가 나빠 수렴에 더 많은 iteration 필요 → 단일 알고리즘으로는 총 연산시간 최적 아님.
3. 예시(도입): (4,8) regular LDPC, AWGN 2.50dB에서 sum-product 9iter=46.8µs/b, min-sum 20iter=56.0µs/b이지만 SP 4 + Alg.E 5 + Alg.B 4 조합은 26.86µs/b(SP의 58% 미만).
4. 핵심기법1: 각 알고리즘의 EXIT chart `p_out=f_i(p_in,p_0)`(extrinsic message error rate 추적)와 1-iter 소요시간 `t_i`를 입력으로, 시퀀스 `S=(a_1,...,a_l)`의 총시간 `T_S=Σ t_{a_i}`를 최소화.
5. 핵심식: 출력 오류율 `P_S(p_0)=f_{a_l}(...f_{a_1}(p_0,p_0)...)`; `P_S≤p_t`면 admissible, 최소 `T_S`가 optimum.
6. 핵심기법2(동복잡도): `t_i` 동일 시 매 iteration에서 `p_out` 최소 알고리즘을 고르는 greedy가 최적임을 증명(Theorem 1).
7. 핵심기법3(이복잡도): `f_i`를 격자점 `P={p_0>...>p_n=p_t}`로 양자화한 `f̂_i`(비관적 근사)로 "decoding performance trellis" 구성 — 각 branch=알고리즘 i, weight=`t_i` → 최소 weight 경로를 DP로 탐색(수천 state도 처리 가능).
8. 이론보장: gear-shift 디코더의 수렴 threshold는 가용 알고리즘 최고 threshold 이상(Theorem 2) — latency는 줄이되 threshold는 손해 없음.
9. 결과: (3,6) LDPC, 1.75dB에서 optimum=min-sum2+SP5+min-sum10=40.9µs/b(단독 SP 61.5µs/b 대비 >50%↓, min-sum 52.7µs/b 대비 >28%↓); 채널 SNR 0.05dB 과추정 시 min-sum은 발산하나 gear-shift는 SP로 전환해 robust.
10. 한계: HW 미설계·SW 소요시간(µs/b, Pentium IV) 기준, cycle-free soft 복호에선 threshold 개선 불가(latency만), 알고리즘 간 message alphabet 호환 매핑 필요, 고최적화 irregular 부호는 EXIT가 tight/closed라 적용 제한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| iteration별 복호 규칙 전환 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | iteration 루프에 알고리즘/스케줄 전환을 삽입하는 관점 — 다만 Prime ECC는 min-sum 단일이라 gear 후보 부재 |
| iteration별 파라미터 테이블 기반 적응 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | Prime ECC의 iteration 구간별 LLR threshold 테이블과 개념 유사(이미 적응형 보유 → 중복성) |
| 조기종료로 iteration 수 최소화 | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | latency 최소화 목표가 겹침 — Prime ECC는 CRC 조기종료로 이미 iteration 절감 |
| EXIT chart + DP 스케줄 최적화(오프라인) | 대응 없음 | 설계 시 스케줄 결정 도구로, 런타임 HW 모듈과 무관 |

적용 가치: **중간** — iteration별 강/약 규칙 전환으로 latency를 줄이는 발상은 디코더 루프에 붙일 여지가 있으나, Prime ECC는 min-sum 단일+적응형 LLR+CRC 조기종료를 이미 보유해 gear 후보와 이득이 제한적이고, 논문 자체가 SW-latency·HW 미설계·threshold 무개선.
