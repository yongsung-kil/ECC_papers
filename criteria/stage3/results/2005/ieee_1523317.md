> 총평: 순환치환 QC H1 + dual-diagonal H2 구조의 구조화 eIRA로 병렬 디코더 친화성과 낮은 error-floor(BER 1e-8~9)를 동시 달성 — 구조가 Prime ECC와 일치하고 error-floor 설계지침이 유의미하나 부호 재설계 부담이 큼.

### B. 알고리즘 요약

1. 대상 부호: 고rate(0.5~0.9) 효율인코딩 eIRA LDPC. `H=[H1 H2]`, `H2`는 마지막 열 제외 weight-2인 dual-diagonal(accumulator).
2. 문제: 기존 eIRA의 `H1`이 랜덤이라 병렬 인코더/디코더 구현이 어려움 — 구조화된 H가 필요.
3. 핵심 구성: `H1=Π^T·P`로 두고 `P`를 `L×J` 순환치환(circulant permutation) 부분행렬 base matrix(지수 `b_{l,j}∈{∞,0..Q-1}`)로 대체, `A^T` 자리에 P를 넣어 인접열 합에 의한 저중량 accumulator 출력 회피.
4. 결정적 인터리버 Π: 행 `qL+l`을 P의 행 `lQ+q`로 매핑해 weight-2 입력의 p1에서 '1'을 분산 → 최소 codeword weight 증대.
5. error-floor 설계 핵심: weight-1/2 입력이 최저중량 codeword를 지배하므로 H1 열의 '1'과 열쌍 합의 '1'을 최대분리 → parity weight↑ → floor↓.
6. 앙상블 분석: serial-turbo IOWE `A_w(C,h)`와 accumulator IOWE로 union bound(식 4~8) 계산, `P_w^{(Co)}`(IOWTP)가 성능 지배. wc=5가 wc=3보다 우수(floor 없음, Pb 1e-9까지).
7. cycle 설계조건: Lemma1(length-4), Lemma2(length-6) — `b_{l,j}` mod Q 조건으로 girth 보장, masking(cyclic masking)으로 열중량 조정.
8. 등가 변형: `H'=Π^{-T}H=[P ΠH2]`가 더 구조적이라 저복잡도 인코더/디코더 유도(성능은 H와 동일).
9. 결과: Ex1(n≈1050,R≈0.77) L=5가 최적·FER 1e-6까지 floor 없음, MacKay 대비 우수. Ex2 rate-1/2(2048,1024) HW/SW, BER 1e-9 근처 near-zero-slope(trapping set 기인). Ex3 rate0.8/0.9(k≈4096) floor 없음(BER≈1e-8), HW로 3.67dB에서 7.3억 codeword 중 1개 오류.
10. 한계: 부호설계 논문(디코더 HW/알고리즘 미설계), 채널 AWGN(NAND 미검증), gatecount/throughput 미제시, floor가 trapping set 종속.
