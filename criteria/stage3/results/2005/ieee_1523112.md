> 총평: 2005년 affine-permutation QC-LDPC 구성과 H 무저장 modulo-N 주소법 기반 가변rate FPGA 디코더로, 복호 성능·min-sum 개선이 없고 이식 가치 낮음.

### B. 알고리즘 요약

1. 목표: 저rate~고rate 가변 LDPC 부호를 낮은 복잡도로 구성하고 FPGA에 효율 구현 (interconnect/메모리/병렬화 이슈 완화).
2. 문제: 완전병렬 디코더는 노드 간 배선/면적 폭증, 직렬 디코더는 H 저장 메모리 폭증 — 대형 H에서 실장 곤란.
3. 부호 구성: H를 `R×S` 개의 `N×N` 부분행렬 블록으로 구성, 각 블록 `A_rs`는 순환행렬(shift `b`) 또는 affine 치환행렬 `J_{a,b}` (`n ≡ am+b mod N`)의 합.
4. 노드 이중 인덱싱: 행노드 `(r,m)`, 열노드 `(s,n)`, 엣지는 `n ≡ m+b_rs mod N`로 파라미터만으로 표현 → H 전체를 shift값 집합으로 기술.
5. 핵심 트릭: `N = 2^k±1` 선택 시 `mod (2^k+1)` 을 `C = C0·2^{2k}+C_H·2^k+C_L` 분해로 덧셈/비교만으로 고속 계산 (식 2~3).
6. 이점: H 행렬을 저장하지 않고 `am`과 `n`의 modulus 비교로 '1' 위치를 즉석 생성 → 메모리 최소화 (1542×771 행렬 저장 회피).
7. 아키텍처: 직렬(면적 최소, 배선 단순) / 완전병렬(속도↑ 면적↑) / 그 중간의 versatile 직렬(R·S 유닛이 N회 직렬) 3종 비교, Xilinx 전용 승산기 활용.
8. 메시지: bit/check 노드 간 메시지 버스 4비트 (soft message passing).
9. 결과: length 255 rate 0.8 (3,15)-regular 구현 — 직렬 1461 slices/26.67MHz, 병렬 15231 slices/19.11MHz. length 1950 rate 0.9 및 rate 1/2 코드가 기존 구성 대비 BER 양호.
10. 한계: min-sum 등 복호 알고리즘 개선 없음, 직렬 latency ∝ n^2, affine/modulo 구성에 종속, ASIC gatecount·throughput 미제시.
