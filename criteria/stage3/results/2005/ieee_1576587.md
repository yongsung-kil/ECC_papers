### B. 알고리즘 요약

1. 대상: 이진 LDPC 컨볼루션 부호(LDPC-CC)의 time-varying periodic parity-check 행렬 구성법 — memory `M`, period `T`, rate `R=b/c`.
2. 배경: LDPC-CC는 임의 길이 데이터를 처리하고 LDPC-BC와 성능이 유사하나, 기존 구성법(Jimenez-Zigangirov, Lentmaier, algebraic QC)이 파라미터에 제한적.
3. 핵심 아이디어: PEG(progressive edge-growth) 알고리즘을 LDPC-CC용으로 수정해 large girth Tanner graph를 생성(BP 복호가 short cycle에 민감하므로 girth 최대화가 핵심).
4. 구조 변환: PEG로 rectangular `H_r`(`T×T` submatrix) 생성 후 cut-and-move로 parallelogram `H_p`(한 주기)로 변환, semi-infinite `H_[0,∞]` 구성.
5. Theorem 1: `T>M`이면 `H_p`를 `T×T` rectangular `H_r`로 변환 가능하고 LDPC-CC의 girth ≥ `H_r`의 girth임을 증명(주기성·modulo T 사상 이용).
6. 제약: (i) `H_t^(0)`가 full rank(identity 포함)여야 인코딩 가능, (ii) `T(T-M-1)` zero submatrix에 1을 두지 않아 실제 memory가 설계 `M`을 넘지 않게 함.
7. 수정 PEG 절차: 각 variable node `s_j`에서 subgraph를 depth `l`까지 확장해 최저 degree check node를 골라 local girth 최대화, rate-1/2(2×1 submatrix)에서 일반 rate-b/c(c×(c-b))로 확장.
8. 설계 유연성: 같은 memory `M`에서 다른 period `T` 생성 가능, `T`를 `M`보다 약간 크게 하면 HW processor 동기화에 유리(ref memory-based decoder architecture).
9. 결과: (3,6) rate-1/2, `M=128~512`에서 BER 시뮬; `T` 증가보다 `M` 증가가 free distance/성능에 크게 기여, 복호 복잡도도 `M`이 결정; Jimenez-Zigangirov보다 우수하나 [2]보다 0.1-0.2 dB 낮음.
10. 한계: LDPC-CC(컨볼루션) 전용이라 block QC-LDPC와 구조가 상이, HW 미설계, 성능 개선폭이 작음.
