### B. 알고리즘 요약

1. 시스템: DVB-S2 위성 방송용 binary LDPC, 블록 `N=64800`, `K=32400`, rate `1/2`, `dc=7`, `dv=13`, 표준 권고 iteration `~70`.
2. 문제: 대형 블록 + 다수 iteration → 연산량·전력·복호지연 과다, 성능저하 없이 iteration/연산 감축 필요.
3. 기본 복호: LLR 도메인 BP. CN 갱신 `g(a,b)=sign(a)sign(b)·min(|a|,|b|)+LUTg(a,b)`, `LUTg=log(1+e^-|a+b|)-log(1+e^-|a-b|)` (min-sum + LUT 보정 = 정확 sum-product).
4. 기법1 순차복호: 체크노드를 `Sm(m=1..p)` 그룹으로 분할(cycle-4 최소화하도록 랜덤), 그룹별 serial 복호. 앞 그룹의 bitnode 확률을 다음 그룹이 이어받음(직렬 concat turbo와 유사).
5. 의미: 그룹 갱신을 순차화하면 정보 전파가 빨라져 동일 성능을 더 적은 iteration으로 달성 (`p=2`, `Nr=35`이 `p=1`, `Nr=70`과 동등).
6. 기법2 조기검출: 신뢰도 높은(LLR 큰) bit/check 노드·에지를 조기 확정. `w_{k→ni}≥Tc` 또는 `v_{n→ki}≥Tb`면 EarlyDetect 플래그=1.
7. 조기검출된 에지는 이후 iteration에서 재계산 없이 이전 값 전달 → 후속 연산 생략.
8. threshold 실험: `Tc=10` 고정, `Tb≈18`에서 conventional과 동일 BER.
9. 결과: 조기검출로 check node update 연산 약 50%↓(일부 99%↓ 표기), 순차복호로 iteration 약 절반(70→35), BER 성능 저하 없음.
10. 한계: HW 미구현, 순수 BER 시뮬, DVB-S2 특정 대형 블록 가정, gatecount/throughput/양자화 비트폭 미제시.
