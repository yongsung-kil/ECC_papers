### B. 알고리즘 요약

1. 대상은 bit가 아닌 패킷(32bit 이상 심볼) 단위 LDPC로, 심볼이 균일랜덤 오류값을 갖는 `q-ary symmetric channel(qSC)`을 다룬다(대알파벳 `q`, 예 `2^32`).
2. 풀려는 문제: q-bit checksum+erasure 정정은 rate 오버헤드가 크고, RS 등 MDS의 ML 복호는 복잡도가 과다 - 선형시간으로 심볼 오류를 국소검사로 잡고 싶다.
3. 핵심 아이디어 verification: 체크노드 이웃 심볼합이 constraint(=0 mod q, `2^b`면 bitwise XOR)를 만족하면 높은 확률로 모든 이웃이 correct라고 "검증(verified)"하여 값 고정.
4. 규칙 2개 - (R1) 이웃 심볼합이 constraint면 미검증 이웃 전부 verified, (R2) 한 이웃만 미검증이면 합으로부터 그 값을 복원·검증.
5. Lemma 1: false verification 확률은 스텝당 `≈ 1/q` 이하 - 패킷당 `Ω(log n)` bit면 (예 32~64bit) 실패확률이 다항적으로 작음.
6. message-passing 버전으로 tree/branching 근사 하에 라운드별 오류전파 확률(`p_j, q_j`)의 밀도진화 재귀식(2)~(6)을 유도, `q_l→0`이면 복호 성공.
7. Theorem 2: erasure 부호(참고문헌 [2]) 차수분포를 재사용해 regular rate 부호가 약 17% 오류를 보정 - RS worst-case 경계와 비교 가능.
8. 개선 (Additional Verification): 이웃들이 제안한 proposed value 2개가 일치하면 verified - regular 부호에서 보정률이 ~17%→~21%로 상승(단 4-cycle 금지, false verification 확률↑).
9. RS 결합: 체크노드에 `(2t,t)` RS 심볼 저장 시 이웃 중 t개까지 오류 정정, erasure 부호와 동일 재귀(12)로 RS worst-case에 근접.
10. Code scrambling(공유 난수로 심볼 곱·치환)으로 적대적 채널을 qSC와 등가로 환원(Theorem 4). 한계: 점근·확률적 결과뿐, HW·BER 실측 전무, non-binary 패킷 전용.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|
| verification 복호(심볼 검증/정정) | 대응 없음 | Prime ECC는 binary min-sum LLR 복호, 심볼 검증 스킴 아님 |
| q-ary/패킷 심볼 채널 | 대응 없음 | `channel.cpp`는 binary AWGN/RBER; 대알파벳 qSC 모델 없음 |
| 밀도진화 기반 차수분포 설계 | 대응 없음 | Prime ECC는 고정 QC-LDPC, 차수분포 최적화 파이프라인 없음 |

적용 가치: **낮음** - 패킷 단위 non-binary qSC용 verification 복호로 Prime ECC의 binary QC-LDPC·soft LLR 복호와 채널·심볼·알고리즘이 모두 상이, 떼어낼 요소 없음.
