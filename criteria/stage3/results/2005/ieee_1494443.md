### B. 알고리즘 요약

1. 시스템: ISI 채널(자기기록, 선형 등화기로 PR4 타깃 정형)에서 등화 후 발생하는 **상관 잡음** `wn`을 다루는 채널 검출기; 외부 LDPC 부호와 extrinsic 정보 교환.
2. 문제: PR 타깃 등화기가 잡음을 상관시켜 AWGN 가정 BP/MAP 검출기가 최적이 아님 — 잡음 whitening 필요.
3. 모델: `rn = Σ hi·xn-i + wn` (채널길이 ν+1), `wn`은 상관 additive noise; PR4 factor graph는 cycle-free라 BP≈MAP.
4. 핵심 기법(NPBP): BP iteration 종료 후 hard decision `x̂n`으로 이상적 ISI 출력 `r̂n`을 계산, FIR 예측필터 `P(D)=Σ pk·D^k`로 예측오차 `en = wn - Σ pk(rn-k - r̂n-k)`를 구해 whitened 출력 `r̃n = rn - w̃n`을 이후 BP edge 확률계산에 사용.
5. 의미: 반복적으로 잡음을 white화 → 잡음 분산·상관 감소로 검출 성능 향상; NPML과 달리 **global soft decision feedback**.
6. 확장: 단극 IIR 예측필터 `E(D)=1/(1+αD)`, `w̃n = α·en-1`로 짧은 필터로 긴 FIR 효과.
7. 필터 최적화(신규): decision feedback의 비트오류(`zn`, 값 {-2,0,2})를 반영해 `Rw = (E{WW^T}+E{CC^T})·P`로 계수 산출 — 저SNR에서 유리(9dB에서 BER 9.6e-3→7.9e-3).
8. 구현 이슈: BP 수렴(≈10 iter) 후 NP 시작, 5~10 iter마다 NP, 3~4 NP step이면 충분; 매 NP step은 원본 `rn`을 기준으로.
9. 결과: BER 1e-5에서 무부호 시 MAP/BP 대비 약 0.8dB(N=1), N=2로 1.2dB 개선; LDPC 부호결합 시 code2(rate 0.89)에서 추가 0.4~0.5dB — 고rate 부호에서 이득 큼.
10. 한계: 순수 시뮬레이션(HW 미설계), 자기기록 PR 채널·상관잡음 가정에 특화, 복잡도/throughput 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NPBP 채널검출기(ISI+상관잡음) | 대응 없음 | Prime ECC는 memoryless RBER/AWGN 채널로 ISI 검출기 없음 |
| 잡음 예측필터(FIR/IIR whitening) | 대응 없음 | soft-read LLR 양자화와 층위가 다름 (채널단 whitening) |
| soft channel output → 외부 LDPC | [channel.cpp](../Prime_ECC_3.1_Claude/) `Set_LLR_Th()` (간접) | Prime ECC의 LLR 입력단이 유사 위치이나 상관잡음 처리는 부재 |
| 외부 LDPC 복호 자체 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 논문은 LDPC 복호를 개선하지 않고 검출기만 개선 |

적용 가치: **낮음** — 기여가 ISI 상관잡음 채널검출기(자기기록)에 국한되며, NAND의 memoryless read 채널·binary LDPC 복호에는 상관잡음 whitening 전제가 성립하지 않아 이식 여지가 거의 없다.
