### B. 알고리즘 요약

1. 대상: 알려진 ISI 채널(AWGN 포함) 상 전송심볼의 joint APP를 factor graph로 표현, turbo equalization용 SISO 검출기 도출.
2. 풀려는 문제: 최적 BCJR는 채널 메모리 `L`에 복잡도 의존이고 직렬구조라 병렬화 불가 → sparse 채널(메모리 크나 간섭자 적음)에서 비효율.
3. 핵심 모델: `r_k = sum f_i x_{k-i} + n_k` (whitened matched filter 출력, AWGN), 채널 임펄스응답 `f` 기지.
4. 핵심 기법: joint APP를 나타내는 FG에 flooding schedule로 SP(sum-product) 알고리즘 적용 → 병렬 반복 검출. 복잡도는 nonzero 간섭자 수에만 의존.
5. 핵심 관찰: FG의 girth가 최소 6이면 min/max-phase 채널에서 몇 iteration 내 BCJR 성능에 실질적 수렴. girth 4면 발산.
6. 확장(stretching): girth-4 그래프의 edge를 경로상 변수노드로 stretch해 등가 girth-6 그래프로 변환, 복잡도 동차 유지하며 수렴 확보.
7. 구현 디테일: log-domain 갱신은 곱셈 없이 덧셈+Jacobian logarithm(`max*`) LUT로 비선형함수 평가; 복잡도 축소는 강한 간섭자만 summary(위치 무관 truncation)로 BCJR 대비 유연.
8. 분석: AMI(average mutual information)/EXIT chart로 수렴/threshold 예측, CND EXIT와 ISID+VND 곡선 조합.
9. 결과: sparse 채널서 5 iteration 내 BCJR 수렴, 축소복잡도 시 BCJR 대비 ~1dB 우위; LDPC(rate 0.5) 결합 시 E2PR4/EPR4 자기채널에서 원 그래프 0.3~0.5dB, 수정 그래프 0.05~0.15dB 손실.
10. 한계: HW 미설계(시뮬만), 자기기록 partial-response(E2PR4/EPR4) 채널 특화, LDPC는 외부코드로만 사용(부호/복호기 자체 개선 아님).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ISI 채널 SP 기반 SISO 검출기(turbo equalization) | 대응 없음 | Prime ECC는 채널 등화기 없음; NAND 채널은 `channel.cpp`의 RBER/AWGN 주입만 |
| joint detection-decoding 통합 그래프 | 대응 없음 | Prime ECC 디코더는 코드 그래프만 처리, 채널 검출 결합 구조 없음 |
| flooding schedule 병렬 message passing | `decoder.cpp` `LDPC_Decoder()` | 개념적 유사(반복 MP)일 뿐, 논문은 채널 검출 그래프 대상 |
| girth 기반 그래프 stretching | `ecc_top.cpp` `Load_PCM()` | 부호 그래프가 아닌 채널 그래프 변형 기법이라 QC-LDPC 부호설계와 무관 |

적용 가치: **낮음** — 자기기록 ISI 채널용 SP 등화기/turbo equalization 논문으로, NAND용 binary QC-LDPC의 부호설계·min-sum 디코더·HW 어느 레이어에도 이식 대상이 없다. LDPC는 외부코드로만 등장.
