### B. 알고리즘 요약

1. protograph 기반 QC-LDPC 부호족(rate 1/2~16/17)을 AWGN 채널에서 공통 HW 디코더로 복호하도록 구성.
2. 문제: 기존 rate-compat 방식(puncturing/shortening)은 rate별 HW 활용도가 불균일하고 고rate에서 check-node degree가 커짐.
3. 앵커는 Kou-Lin-Fossorier의 regular `(4,32)` EG 부호 `C2*Z` (rate 7/8, `n=16Z=8176`, circulant `Z=511`).
4. Expurgation: protograph의 check node를 `2->4->8->16`으로 분할해 저rate 부호를 만들되 길이 `n` 고정.
5. Node-splitting 규칙을 `(4,2)` 거리-3 4진 부호로 정의 → `C16`에서 임의 두 변수노드의 공유 체크 <=1 → 길이-4 loop 제거.
6. Lengthening: 각 check에 dedicated accumulator(degree-2 ancillary 변수, 2 edge)를 부착 → rate 16/17~16/32의 R4A 구조로 threshold 개선.
7. Pre-coding: 일부 fixed 변수노드를 ARA 스타일로 puncture+degree-3 check 부착 → threshold 추가 개선(light pre-coding 채택).
8. 공통 HW: 64 EG circulant edge는 전 rate 공통이고, auxiliary/ancillary edge의 activation schedule로 rate 전환(모든 rate에서 항상 32개 auxiliary+ancillary edge 활성).
9. 결과: 전 rate threshold가 capacity 대비 0.4~0.5 dB 이내, WER `10^-6`까지 error floor 미검출.
10. 한계: light pre-coding은 0.1~0.2 dB 성능 개선에 복잡도 `Mb`가 ~1 dB 증가; HW는 개념적 아키텍처(합성·gatecount 없음), 우주응용 AWGN 기준.
