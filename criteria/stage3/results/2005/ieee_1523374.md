> 총평: min-sum(normalized/offset)의 irregular 부호 error-floor 원인을 고립 서브그래프로 규명하고 완화책 제시. Prime ECC의 min-sum 코어와 직접 맞닿아 error-floor 관점 참고가치 있으나 부호구조 의존.

### B. 알고리즘 요약

1. 대상: AWGN/BPSK 상 irregular LDPC, `(N,K)=(4608,2304)` rate 0.5, `λ(x)`(degree 2 비중 큼)·`ρ(x)=0.25x^7+0.75x^8`, ACE 구성, max iter 200.
2. 문제: BP를 단순화한 min-sum은 CN 출력을 과대추정(0.6~1.1dB 손실)하고, 개선판(normalized/offset)이 irregular 부호에선 regular와 다르게 동작(특히 block error 악화).
3. 모델: CN 출력 크기 `|L_MS|=min|Z_i|`가 `|L_BP|`보다 항상 큼 → 과대추정. 개선: `|L_NORM|=min|Z_i|/α`(α>1), `|L_OFF|=max(min|Z_i|−β,0)`.
4. 핵심관찰: normalized min-sum은 irregular 부호에서 block error 매우 나쁨; 오류블록의 94.5%가 degree-2 bit에서 발생, 메시지가 정상상태(steady state)로 고착.
5. 핵심식: degree-2 노드로 구성된 길이-6 loop 서브그래프에서 `Ax=y`(식7, normalized)는 α>1일 때 full-rank 고유해 → 외부 메시지가 커도 특정 조건 시 bit-1 오정정 확정.
6. 원인: 입력 메시지가 매우 크면 min-sum≈BP가 되어 normalization/offset이 오히려 과소추정(underestimation) → 고립 서브그래프가 외부와 단절되어 저중량 오류블록(trapping set류) 발생.
7. 개선책1(offset): 200 iter 실패 시 offset을 0으로 두고 50 iter 추가 → 과소추정 메시지 크기를 키워 정상상태 탈출, error floor 하강.
8. 개선책2(normalized): 차수별 정규화(degree-2는 1.0=무정규화, degree-20은 2.0, 나머지 1.3) → block error 대폭 개선.
9. 결과: offset min-sum은 waterfall에서 BP 대비 0.15dB; 개선 offset은 error-floor 영역에서 BP보다 오히려 소폭 우수(유한길이 BP 비최적).
10. 한계: 시뮬만(HW·양자화 상세·gatecount 없음), degree-2 많은 irregular 부호 특화, α/β는 앙상블별 density evolution으로 설정.
