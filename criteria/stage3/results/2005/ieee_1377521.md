### B. 알고리즘 요약

1. 대상은 sparse `H`로 정의되는 binary LDPC의 Tanner 그래프로, short/moderate block에서 랜덤 구성 시 "나쁜 그래프(짧은 cycle)"를 뽑을 확률이 높아 성능이 저하되는 문제를 푼다.
2. 핵심 기법 PEG: symbol node `sj`에 edge를 하나씩 추가할 때, `sj`에서 BFS로 subgraph를 depth `l`까지 펼쳐 도달 불가/가장 먼 check node를 골라 연결 - 이렇게 생기는 cycle이 최장 `2(l+2)`가 되도록 local girth를 국소 최대화한다.
3. 다중 후보 시 현재 최저 degree의 check node를 선택 - 결과 그래프의 check-node degree를 최대한 균일(concentrated)하게 만든다.
4. girth 하한(Theorem 1): `g ≥ 2(⌊t_low^irr⌋+2)`, `t_low^irr = ⌊log(...)/log[(d_s^max-1)(d_c^max-1)]⌋-1`; degree가 균일할수록(`d_s^max, d_c^max`↓) 하한이 좋아지며 Gallager 명시 구성보다 항상 우수.
5. 이 하한은 Erdös–Sachs bound(랜덤 정규그래프 girth의 비구성적 하한, 부호론의 Varshamov–Gilbert 대응)에 부합.
6. 최소거리 하한((16)(17)): `(d_s,d_c)`-regular(`d_s≥3`)에서 `d_min ≥ 1 + d_s[(d_s-1)^⌊(g-2)/4⌋-1]/(d_s-2)`, girth와 직접 연동되어 high-SNR(error-floor) 성능을 보증.
7. 변형: nongreedy(depth를 `l_max`로 제한), look-ahead-enhanced(한 스텝 앞을 보고 최대 depth 후보 선택), ACE 결합([40][41]) - error-floor 추가 개선.
8. 선형시간 인코딩: `H^p`를 하삼각 형태로 강제 구성(21)하여 Gaussian elimination 없이 `p_i`를 재귀적으로 계산(22), 성능 저하 거의 없이 linear-time encodable.
9. 결과: rate-1/2 (504,252)에서 PEG는 랜덤/MacKay 대비 avg local girth 8.01 vs 6.42/6.74, block-error 5e-5에서 MacKay 대비 0.2dB, DE-최적 irregular(max degree 15) 조합 시 (1008,504)에서 0.5dB 개선.
10. GF(q) 확장: 동일 PEG로 non-binary LDPC 구성, GF(2^6)에서 binary MacKay 대비 ~0.75dB 개선(당시 short-block SOTA). 한계: 시뮬만, HW/gatecount 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG Tanner 그래프 / H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 QC-LDPC H를 로드; PEG는 랜덤 구조라 circulant 제약과 직접 호환 안됨 |
| girth/최소거리 기반 부호설계 | 대응 없음 | Prime ECC에 부호구성/girth 최적화 파이프라인 없음(H 고정) |
| 선형시간(triangular) 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 인코딩 고정 - triangular 방식과 개념 인접하나 이미 보유 |
| GF(q) non-binary 확장 | 대응 없음 | Prime ECC는 binary 전용, non-binary 미지원 |

적용 가치: **중간** - PEG는 binary LDPC의 girth/최소거리를 높여 error-floor를 개선하는 검증된 부호구성법이나, Prime ECC의 고정 QC-LDPC circulant 구조와 직접 호환되지 않아(QC 제약 하 PEG 변형 필요) 재설계·재검증 부담이 있다.
