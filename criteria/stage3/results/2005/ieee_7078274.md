### B. 알고리즘 요약

1. 대상은 무선 FEC용 binary LDPC로, (3,6) regular code `N=4000`(MacKay 4000.2000.3.243), max 40 iter, AWGN 채널에서 평가.
2. 풀려는 문제: Forced Convergence(FC)는 복잡도를 줄이지만 threshold를 낮게 잡으면 error floor가 발생한다.
3. 배경 기법 FC: 이미 수렴한 node(`|Bi|>tv` 또는 `|Cj|<tc`)를 이후 iteration에서 비활성화하고, parity 불만족 node는 재활성화.
4. 핵심 기법: FC 디코딩 뒤 Gallager식 hard-decision bit-flipping(BF) post-processor 추가 — 가장 많은 unsatisfied check에 연결된 VN을 flip하고 syndrome 갱신을 반복(`NBF=30`).
5. 게이팅: BF는 unsatisfied check 수 `≤ 2·NBF`일 때만 활성화(FC 잔여오류는 <10개라 저비용, 일반 block error는 수백개라 무익).
6. min-sum 계열 통합: MSA·NMSA(`α=13/16=0.8125`)에도 FC+BF 적용, min-sum에서 `tc`는 lower bound로 동작.
7. 복잡도 모델: node별 MAC ops로 `ΘD` 유도, BF 오버헤드는 정보비트당 최대 6 MAC로 무시가능.
8. threshold `tv,tc`는 수동 설정(학습 절차 없음).
9. 결과: FC로 복잡도 20~25% 감소, BF가 error floor 대폭 하락; NMSA는 원 성능 0.2dB 이내, MSA는 high threshold+BF로 원 성능 능가 가능.
10. 한계: 평균 iteration 증가로 이득 일부 상쇄, threshold 선택 민감, HW 미설계, 저rate code.

| forced convergence 노드 비활성화(복잡도 감소) | `GT.cpp` `Make_GT_HW()`, `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 edge 씨닝(GT)으로 유사 목적, node 비활성화는 미보유 |

| 조기종료(syndrome/parity 만족) | `partial_CRC.cpp`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료 보유 |
