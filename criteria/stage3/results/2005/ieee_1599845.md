> 총평: 확률 메시지를 Bernoulli 스트림으로 다루는 스토캐스틱 디코딩 — HW는 극단적으로 단순(soft-XOR=XOR)하나 min-sum bit-exact 파이프라인과 패러다임이 달라 Prime ECC 이식 부적합.

### B. 알고리즘 요약

1. 대상: rate-1/2 length-16 (16,8) LDPC (equality 16 + parity-check 8 노드), 통신표준(10GBASE-T/DVB-S2/WiMAX) 배경.
2. 문제: 반복 디코더의 다비트 디지털 메시지가 라우팅 혼잡·면적을 유발 → 아날로그 디코더의 단순성을 디지털로 취하려는 시도.
3. 핵심표현: sum-product의 확률 메시지 P를 길이 N Bernoulli 시퀀스의 1의 빈도 `P=m/N`로 인코딩(정밀도는 N에 비례, 표현 비유일).
4. 핵심기법 — 확률 게이트의 스토캐스틱화: parity-check "soft-XOR" `Pc=Pa(1-Pb)+(1-Pa)Pb`가 스트림 입력에선 단순 XOR로, equality 게이트는 AND(곱)+JK플립플롭(나눗셈/정규화)로 축약.
5. 알고리즘: 매 클록 각 에지로 1비트만 교환("cycle"; SP iteration과 불일치), 완전병렬 팩터그래프를 실리콘에 직접 사상, 에지당 2선(양방향)만 필요.
6. 부수 트릭 — supernode: JK-FF equality 노드가 입력 상관에 lock되는 문제를, counter+digital-to-stochastic 변환기(3bit 폭 경험적 최적)로 스트림을 재생성해 상관 제거.
7. 입력 변환: BPSK noisy → LLR→확률 LUT → 랜덤비트로 스토캐스틱 스트림 생성(m=4, N4를 확률1/2 랜덤비트로 두어 표현범위 1/32~31/32).
8. 랜덤성: LFSR은 인접비트 상관이 커 부적합, LHCA(Linear Hybrid Cellular Automata)로 다수 랜덤비트 생성.
9. 출력: up/down counter의 부호비트로 hard decision, TINIT(비갱신)+TCHECK(갱신) 2단계 후 유효 codeword까지 진행(가변 사이클).
10. 결과: Altera Cyclone(12,060 LE)에 1491 LE(12%), 100MHz→5.1Mbps, Stratix 165MHz→8.4Mbps; 고SNR시 정보비트당 ~19.5 clock; SW시뮬 대비 0.4dB 손실(4bit 양자화).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 스토캐스틱 확률 게이트(soft-XOR/equality) | 대응 없음 | Prime ECC는 min-sum magnitude 연산, Bernoulli 스트림 패러다임 아님 |
| fully-parallel 팩터그래프 사상 | `decoder.cpp` `LDPC_Decoder()` | 개념상 디코더 루프 위치이나 계산모델 전혀 다름 |
| 입력 LLR→확률 변환 LUT | `channel.cpp` `Set_LLR_Th()` | soft-read/LLR 변환 위치만 유사, 방식 상이 |
| 에지당 2선 라우팅 절감 | `GT.cpp` `Make_GT_HW()` | 라우팅 축소라는 목표만 공통, 기법 무관 |
| supernode/LHCA 랜덤비트 생성 | 대응 없음 | Prime ECC에 상응 개념 없음 |

적용 가치: **낮음** — 스토캐스틱 컴퓨팅은 Prime ECC의 min-sum bit-exact 파이프라인과 패러다임이 달라 이식이 사실상 불가하고, 5~8Mbps 저throughput·(16,8) 소부호로 NAND 고rate 장부호 적용 근거가 부족.
