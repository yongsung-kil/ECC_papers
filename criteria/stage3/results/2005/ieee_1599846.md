### B. 알고리즘 요약

1. 대상: multirate LDPC 디코딩용 재구성 VLSI fabric(PE 정사각 매트릭스 + link/switch 인터커넥트), 무선 채널 용량 변동 대응.
2. 문제: 완전병렬 LDPC 디코더는 불규칙 인터커넥트가 면적·전력을 지배(예 Blanksby[4] 레이아웃이 배선 지배) → 배선을 설계 초기부터 고려 필요.
3. 핵심 관찰: `φ(x)=ln((eˣ+1)/(eˣ-1))` 비선형(LUT)로 code node가 `φ(λ)`를 보내도 알고리즘 불변 → PE 하나가 code node·check node를 겸용 가능(설계시간↓, 규칙적 레이아웃).
4. Fabric: block = PE + 인접 crosspoint; routing crosspoint(6 latch, Fig2 구성)와 pin crosspoint(4 latch, 3구성)로 임의 LDPC 연결을 프로그래밍, PE간 값은 직렬(7bit) 전송.
5. 재구성 복잡도: 총 config 비트 `N·(6·nh·nv + 4(nh+nv)(pinscv+pinspv))`; 1024bit blk(b=32,nh=nv=32,code pin4/chk pin8)→ 1.5MB config, 7680 cycle(400MHz서 19.2µs)로 재구성.
6. 핵심기법 — 라우팅 인지 부호 생성(Algorithm 2): check/code 노드 고정 후 pass 단위로 최소차수 노드에 에지 추가, 기존 girth·degree 제약에 **routability 제약**을 추가.
7. 라우팅: block을 5노드(4방향+중심)로 표현한 routing graph에서 edge weight=사용 트랙수(혼잡도)로 두고 Dijkstra 최단경로로 least-congestion 경로 선택, 트랙 균일 소진 유도.
8. 디코딩: 표준 soft-decision message passing(Algorithm 1, log영역 φ LUT), 7bit 정밀도(소수 4bit).
9. VLSI: code node degree≤8, check≤16; PE에 8bit adder 1개로 15 add+15 sub → message node 31 cycle·code node 15 cycle; 0.13µm서 ~0.5×0.5mm, 1024bit·8iter·46cycle/iter → 360MHz면 1Gbps 목표 달성.
10. 결과: 생성한 rate-1/2 1024bit 부호가 Blanksby[4] 대비 코딩손실 <0.3dB(구현효율 부호를 코딩손실 거의 없이 생성); 실측/합성 없이 면적·throughput은 추정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft MP 디코딩(φ=-log tanh LUT) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum, SPA φ LUT 방식과 상이 → 대응 약함 |
| interconnect-aware 부호 생성 CAD | `ecc_top.cpp` `Load_PCM()` | 부호설계 레이어(이식 난이도 높음), QC 구조와 무관한 랜덤부호 |
| 재구성 fabric/crosspoint 라우팅 | `GT.cpp` `Make_GT_HW()` | 라우팅 축소 목표만 공통, QC z=32 규칙 라우팅과 방식 상이 |
| multirate(가변 rate) 지원 | 대응 없음 | Prime ECC는 고정 고rate 부호 |
| code/check 겸용 PE | 대응 없음 | Prime ECC는 전용 CNU/VNU 파이프라인 |

적용 가치: **낮음** — 무선 multirate용 FPGA류 재구성 fabric과 라우팅 제약 랜덤부호 CAD 구성은, 고정 고rate QC-LDPC(z=32) ASIC 파이프라인인 Prime ECC와 방향이 달라 직접 이식 가치가 낮다.
