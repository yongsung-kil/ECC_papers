### B. 알고리즘 요약

1. 부호·채널: `(3,p)`-regular **QC-LDPC**, BIAWGN/BPSK, SPA(belief propagation) 복호. 목표는 짧은 블록·고rate에서 4-cycle 제거.
2. 문제: 랜덤 구성은 짧은 블록·고rate에서 4-cycle 제거가 어렵고 저복잡도 인코딩이 힘듦 → 대수적 구조 부호 필요.
3. 구조: PCM `H=[H1;H2;H3]`, 각 `Hi`는 `n x n` permutation(및 identity) 블록 배열 → `3n x n^2` 크기, column weight `j=3`, row weight `k=p`.
4. 핵심 기법: 합성정수 `n=p1^e1...pm^em` 분해 후 `φ(n)`개 Latin square를 GF(p) 원소 `γ`로 채움(cell `(j,k)=γ_j+γ_i·γ_k`), 3개 레벨의 `n x n` Latin square로 PCM 구성.
5. 매핑: Latin square 인덱스에 permutation/interleaving을 적용해 `H(x,y)=(n-1)·l(x)+l(y)`로 부호행렬과 연결 ([5]의 linear mapping 대신 permutation 매핑으로 더 랜덤하게 구성).
6. 성질: column weight 3만 사용(대형 weight column은 short cycle 유발) → 설계상 모든 length-4 cycle 제거, girth>=6 보장.
7. rate: `Rd = 1-(j/k) = 1-(3/n)` → `n` 클수록 고rate(Rd>3/4).
8. 최소거리: `(j+1)!` 한계까지 증가한다고 **가정**(미증명).
9. 결과: 예시 부호 (400,rate0.855), (784,0.8954), (1936,0.9318); 짧은 블록(n=20 rate0.85, n=28 rate0.8928)에서 [5]의 MOLS 구성보다 BER 우수(SPA 최대 50 iter 추정).
10. 한계: 순수 부호구성(HW/디코더 아키텍처·양자화·soft-read 논의 없음), column weight 3 고정, 최소거리 주장 미증명, error-floor 정량 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Latin square/GF(p) permutation 매핑으로 QC-LDPC PCM 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 지점 — QC 형식은 동일하나 permutation 블록 표현이 circulant-shift(base+shift)와 상이 |
| circulant/permutation 블록 배열 구조 | `ecc_data.h` `PCM` (`PCM_b`=base+shift) | QC-LDPC base matrix 표현과 개념적 대응, 단 z 단위(=32)와 정합 필요 |
| girth>=6 / 4-cycle 제거(부호설계) | 대응 없음 | 부호설계 단계 성질, 디코더 코드 모듈에 직접 대응 없음 |
| SPA 반복 복호 | `decoder.cpp` `LDPC_Decoder()` (우리는 min-sum) | 논문은 full SPA, 우리는 min-sum 근사 — 알고리즘 이식 아님 |

적용 가치: **낮음** — QC-LDPC라는 형식은 Prime ECC와 일치하지만, column weight 3 고정·Latin square 기반 permutation 배열은 우리 고VN-degree(17) 고rate·circulant-shift 구조와 정합되지 않고 HW·soft-read 논의가 없어 실이식 가치는 낮다.
