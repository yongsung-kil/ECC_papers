### B. 알고리즘 요약

1. 대상 부호: `(1024, 3, 6)` regular LDPC(column weight 3, row weight 6, rate-1/2), factor graph 기반 message passing 복호.
2. 문제: 완전병렬 LDPC 디코더는 sparse H의 랜덤 1 위치 때문에 PU 간 배선·레지스터가 폭증(Blanksby: net 평균 3mm, die 52.5mm²), 직렬화는 반대로 수천 입력 MUX·수만 cycle 필요.
3. 채널/알고리즘: 수신 LLR로 초기화 후 식 (7),(8)의 `tanh`/`tanh⁻¹` 기반 sum-product를 `LT=-log(tanh(x/2))`, `AE=2tanh⁻¹(exp(-x))` LUT로 근사.
4. 핵심 기법1(loosely-coupled): 모든 메시지 대신 행 합산값 `Λi`·열 합산값 `Δj`만 이웃 PU에 전달, 각 PU가 중간값 `δij,λji`로 개별 메시지를 복원(중복연산 감수) → 배선 대폭 축소.
5. 식 의미: `Δji=Δj−δij`, `Λij=Λi−log(tanh(Δji/2))` 처럼 합산값에서 자기 기여분을 빼 개별 메시지 재구성 — 메시지 전체 교환을 합산값 2종 교환으로 대체.
6. 핵심 기법2(부분병렬): 한 PU가 여러 행/열 담당, 중간값 `{δij},{λji}`를 FF 대신 PU 전용 local memory에 저장 → 면적 절감, MUX 공유로 배선 추가 축소.
7. 핵심 기법3(오버랩 스케줄): CTV가 대응 행 합산값이 준비되면 VTC 진행 중에도 시작하도록 겹쳐 처리, cycle 감소.
8. 학습/최적화: matrix permutation 기반 스케줄링 알고리즘 — H를 재배열해 좌하/우상 코너에 빈공간을 만들어 겹침 가능 순서 도출, 평균 25%+ cycle 절감.
9. 결과: 0.18μm 4-metal CMOS, 32 check PU+64 variable PU, 543K gate·10.08mm²·~985Mbps@200MHz — Blanksby(1750K gate, 52.5mm²) 대비 대폭 면적감, 8 iteration으로 충분한 BER.
10. 한계: 정정성능 개선 없음(동일 SP), 랜덤 LDPC 대상, 중복연산으로 PU 로직 증가, 실측 아님(레이아웃/합성 수준).
