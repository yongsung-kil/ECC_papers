> 총평: QC-LDPC 순환구조로 셔플 네트워크·메모리 주소제어를 단순화한 초기(2005) 반병렬 FPGA 디코더로, Prime ECC의 z=32 병렬 QC 구조와 개념은 겹치나 이식할 신규 기법은 낮음.

### B. 알고리즘 요약

1. 대상은 (3,6) regular **QC-LDPC**, H를 순환 부분행렬 `Hic = x^a_ic + x^b_ci`(binomial)로 구성한 특정 패밀리(girth≥8 보장 조건).
2. 문제: 완전병렬 LDPC 디코더는 Tanner graph 에지 라우팅이 면적의 최대 80%를 차지 → 순환구조로 인터커넥트/메모리 제어를 줄이려 함.
3. 복호 알고리즘은 belief propagation(sum-product), 메시지 폭 4bit로 양자화.
4. 핵심 기법: **α2 아키텍처** — check node 모듈 α개, bit node 모듈 2α개를 인스턴스화, 각 순환행렬 `Hij`당 메시지 위치를 파라미터 `{e(1), s}`로 표현.
5. 순환행렬의 두 '1' 위치를 `{e(1), s}`(첫 위치 + 간격)로 두면 n번째 check가 필요한 메시지 위치 관계가 모든 `Hij`에서 고정 → 다음 메시지 주소가 결정적.
6. 이 덕분에 전체 메모리 주소 제어를 **0~m-1 단일 카운터** 하나로 축소, 셔플 네트워크 Π1/Π2를 2α개 소형 네트워크로 분할.
7. q_mem 블록은 3개 m×4 메모리, r_mem 블록은 6개 m×4 메모리; write/read 페이즈를 m 클록마다 교대.
8. 구현 디테일: Xilinx VirtexE Block SelectRAM+의 1~16bit 가변 폭·듀얼포트 활용, m<512면 한 포트에 2메모리 매핑.
9. 결과: α2가 994 slices/1689 LUTs/24 BRAM, 37.04ns 주기, 15Mbps — 완전병렬(16843 slices, 186Mbps)보다 면적↓·throughput↓, semi-parallel(11.7Mbps)보다 우수.
10. 한계: BER/정정능력 개선 없음, 실측 아닌 합성 수준, throughput 매우 낮고 코드 파라미터 규모 작음.
