### B. 알고리즘 요약

1. AWGN/BPSK 상의 block-structured QC-LDPC (`p×p` block: zero 또는 identity의 right cyclic shift), rate 1/2 및 7/8 예제. 목표는 code-encoder-decoder 공동설계.
2. 문제: "코드 먼저, HW 나중" 방식은 fully-parallel의 라우팅 폭주/부분병렬 매핑 곤란을 초래 — 코드구성과 HW를 반드시 함께 설계해야 함.
3. 두 제약 부류: (a) implementation-oriented = decoder 제약(전체 PCM을 `p×p` block-cyclic 배열로) + encoder 제약(lower macro-block triangular, 각 submatrix col weight ≤1).
4. 핵심 기법: performance-oriented 제약으로 degree distribution(density evolution), girth(4-cycle 회피), 그리고 "degree of a cycle"(cycle 경로상 VN degree 합)을 크게 유지.
5. 핵심 개념: `degree of a cycle` — 저degree VN을 지나는 짧은 cycle이 더 해로우므로 이를 회피하면 유한반복 복호 성능이 개선(ACE 유사).
6. 코드구성: RBFS(random block flipping/shifting) — 회색(macro-identity) 영역 고정, 나머지 zero-block을 girth·cycle-degree 제약 하 랜덤 flip; 실패 시 제약 완화. 여러 seed로 코드군 생성.
7. 코드선택: cycle effect(loopiness) `Σ` 지표 최소인 코드 선택.
8. 인코더: Richardson-Urbanke approx lower-triangular 인코딩을 back-substitution + block-cyclic sparse matrix-vector 곱(cyclic shift = subvector 회전 → vector XOR)으로 구현, graph-coloring으로 task 스케줄링해 single-port register file 재사용.
9. 디코더: CNU/VNU를 block row/col 단위로 TDM 공유하는 partial-parallel 아키텍처, DMMB/CMMB/HDMB 메모리, counter로 cyclic-shift 주소 생성. 노드 유닛 수를 `p`배 축소.
10. 결과/한계: 제약 미도입 코드 대비 BER 열화 미미(Fig.4)하나 성능 "향상"은 아님; gate/throughput은 파라미터 수식(표3/4)만 제시, 절대 gatecount·Gbps·공정 미상, NAND 실증 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| RBFS 코드구성 + cycle degree/girth 제약 | 대응 없음 (H-matrix 고정) | 부호설계 레이어라 이식 난이도 높음, 재검증 부담 |
| approx lower-triangular / block-cyclic 인코딩 | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | Prime ECC는 dual-diagonal 고정; 유사 sparse 인코딩 계열 |
| partial-parallel CNU/VNU TDM 디코더 | decoder.cpp LDPC_Decoder(), C2V_Cal(), VN_Cal_HD() | z=32 row-병렬 구조와 개념적 동일, 세부 min-sum은 본 논문 미기술 |
| cyclic-shift 주소용 counter 라우팅 | decoder.cpp (메모리 주소 생성) | 우리 z-block 주소 로직과 대응 |
```

적용 가치: **중간** — Prime ECC의 block-cyclic QC-LDPC + partial-parallel 아키텍처와 동일 계열의 기초 설계라 새로 얻을 기법은 적으나, "degree of a cycle / loopiness 기반 코드선택 지표"는 우리 H-matrix 후보 선별에 참고 가능.
