### B. 알고리즘 요약

1. 대상 시스템은 40Gb/s WDM 장거리 광전송, code rate 0.809(23.6% redundancy) 등, GLDPC로 FEC 구성.
2. 풀려는 문제: 기존 LDPC의 message-passing은 sub-optimal이라 Shannon 한계까지 여유가 있음 → GLDPC로 minimum distance·코딩게인 개선.
3. 핵심 모델: global LDPC의 각 single parity-check(check node)를 로컬 선형 블록코드(`Hamming`/`BCH`/`RM`)의 parity-check matrix로 대체 = GLDPC.
4. 핵심 기법: B-construction(block-diagonal `H0` + 랜덤 column permutation `pi_i`로 super-code 교집합), LZ-construction(row의 1들을 로컬코드 컬럼으로 치환).
5. 핵심 식(1): minimum distance 하한 `d_min ~ (d_local)^(g/2)` 형태 — girth `g`가 클수록 지수적 증가, column weight가 base 증가.
6. 확장: 구조적(structured) regular/irregular GLDPC를 primitive element 기반 정수집합(식4)·Latin-square로 설계, girth ≥ 8~20 보장. Theorem 1/2로 girth 하한 증명.
7. 구현 디테일: 로컬 디코딩은 BCJR/Max-Log-MAP 또는 modified Walsh-Hadamard MAP; irregular화는 permutation block 삭제로 rate 상향.
8. 최적화: irregular degree 분포는 EXIT chart로 탐색; cycle-4 제거 알고리즘으로 BCH도 MPA로 복호 가능.
9. 결과: GLDPC(6048,4896,0.809)가 23.53% overhead로 최대 ~12dB 코딩게인(BER<1e-9), 기존 turbo(24.6%)/LDPC 대비 1dB+ 우위, 로컬 iteration <10.
10. 한계: HW 미설계(시뮬만), 광채널(ASE/Kerr/chromatic dispersion/ISI) 특화 모델, error-floor 존재 여부는 open issue.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC 구조 (로컬 블록코드로 check node 대체) | 대응 없음 | Prime ECC는 binary QC-LDPC 단일계층; GLDPC/로컬 SISO 계층 구조 없음 |
| 로컬 BCJR/Max-Log-MAP SISO 디코딩 | 대응 없음 | Prime ECC 디코더는 min-sum(`decoder.cpp` `CNU_Update_New_Mag()`)만; BCJR/trellis MAP 미보유 |
| 구조적 global LDPC girth 설계(Latin-square/circulant) | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 지점이나, QC circulant 설계 방식이 달라 재검증 부담 큼 |
| soft-input 초기화(LLR) | `channel.cpp` `Set_LLR_Th()` | 개념적 대응만; 광채널 LLR과 NAND read LLR 상이 |

적용 가치: **낮음** — GLDPC(로컬 블록코드+BCJR/MAP) 구조와 광통신 채널 특화 설계로, binary QC-LDPC min-sum 엔진(Prime ECC)에 떼어 쓸 계층이 없다. girth 설계 이론만 원거리 참고 가능.
