### B. 알고리즘 요약

1. AWGN 채널, 평가부호 2종: (3,6) regular LDPC(N=4000, MacKay), 802.11n TGnSync 유래 irregular LDPC(N=5312, rate 1/2). min-sum은 MPA 대비 약 0.4dB 손실.
2. 문제: LDPC 복호 평균 복잡도 = (노드당 연산수) x (평균 iteration수) x (iteration당 active node수). 기존 기법은 첫 인자만 공략.
3. 관찰: 다수 variable node가 소수 iteration 만에 강한 belief로 수렴 → 이후 갱신을 생략 가능. Forced Convergence(FC)는 유일하게 세 번째 인자(active node 수)를 줄임.
4. FC 핵심: 각 노드의 aggregate message `B_i`(VN), `Θ_j`(CN)를 임계값 `t_v`, `t_c`와 비교해 수렴 노드를 "deactivate", 이후 iteration에서 메시지 갱신 중단.
5. tentative decoding에서 parity 불만족 CN과 연결 VN은 재활성화하여 오수렴 노드 동결 방지; active node 인덱스 벡터로 관리.
6. 확장: min-sum에도 적용, `t_c`를 `Φ(Θ_j)=min|Q_i,j|`에 대한 하한으로 사용. CN 복잡도 축소기법(piecewise-linear `Φ` 근사, 3-segment)과 결합.
7. 구현 트릭: `Φ` 근사 선분 기울기를 2의 거듭제곱으로 잡아 bit-shift 구현, 대칭조건 `Φ_i(Φ_i(x))=x` 만족; 3-segment로 CN 복잡도 약 50% 절감.
8. 분석: EXIT chart로 노드 비활성화가 transfer characteristic을 어떻게 바꾸는지(높은 extrinsic 영역에서 tunnel 닫힘→error-floor) 정량화.
9. 결과: FER 10^-2 목표에서 성능 0.1dB 이내 유지하며 복잡도 ~20% 절감(MPA·MSA 모두). `Φ` 근사와 결합 시 최대 36% 절감. 단, 평균 iteration 수는 사실상 불변(active node 수만 감소).
10. 한계: HW 미설계·시뮬만, 임계값이 부호/채널(fading시 재조정) 의존, 임계값 과소시 error-floor 유발, 무선(802.11n) 지향.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Forced Convergence 노드 비활성화 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 수렴 노드 skip을 루프에 삽입 가능; 단 z=32 row-parallel 구조에 불규칙 skip 정합 필요 |
| aggregate message 임계 판정(tentative decode) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 기반 재활성화 판정과 유사 개념(이미 Partial/Full CRC 보유) |
| min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유(min1/min2+sign) — 중복 기여 |
| piecewise-linear `Φ` 근사 (CN) | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum 이미 채택, magnitude 양자화 테이블 존재 — 신규성 낮음 |
| aggregate message 갱신(VN) | `decoder.cpp` `VN_Cal_HD()` 등 | active-node skip 도입 지점 |

적용 가치: **중간** — active-node 비활성화 아이디어 자체는 `decoder.cpp` 루프에 이식 가능하나, min-sum/piecewise-CN 근사는 이미 보유해 신규성은 FC 스케줄에 국한되고, 불규칙 노드 skip이 고정 z=32 파이프라인 HW에서 실이득이 될지 검증 필요.
