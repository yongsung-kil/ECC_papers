> 총평: 구조화 binary QC-LDPC를 modified min-sum(α-β 이중재귀)으로 복호하는 0.18µm 합성 디코더(375K게이트, ~0.94Gbps) — HW 면적/throughput/양자화 정량 레퍼런스로 유용하나 min-sum·양자화는 기 보유이고 아키텍처가 WLAN 특화.

### B. 알고리즘 요약

1. 시스템: IEEE 802.11n WWiSE WLAN용 구조화 QC-LDPC, `N=1944` 고정, sub-block `P=27`(identity/회전-identity), IRA-like, rate `1/2·2/3·3/4·5/6`, 16/64-QAM.
2. 문제: LDPC HW의 라우팅 혼잡·메모리 단편화·멀티rate 유연성 부족 → 저복잡도 고throughput 디코더 설계가 핵심 난제.
3. 복호: log-MAP(sum-product) 반복 message-passing, CN은 modified Min-Sum `M-min*(|µa|,|µb|)=min(|µa|,|µb|)-LUT(|D|)`, `D=||µa|-|µb||`.
4. CN 이중재귀: forward `α_{i+1}=M-min*(α_i,|µ_i|)`, backward `β_i=M-min*(β_{i+1},|µ_{i+1}|)`, 출력 `M-min*(α_i,β_i)` → `dC(dC-2)`회를 `3(dC-2)`회로 축소.
5. 의미: trellis형 α-β 이중재귀로 CN 출력의 공통 중간결과를 재사용, 직렬 CNP로 연속 스트림 처리(재정렬로 latency 최소화).
6. 멀티rate 확장: master CNP를 rate1/2 prototype `dC=7`로 설계, 고rate는 prototype 행을 mod-2 합(구조 보존)하고 2개 slave aux-CNP가 µ*를 병렬 산출 → dC=21까지 무오버헤드·무추가latency.
7. 고정소수점: 입력 LLR/extrinsic 메시지 clip=12, 5-bit(`Nλ=Nm=5`) → floating 대비 BER=10^-8까지 0.2dB 손실.
8. 아키텍처: Vectorised + memory paging(paging `G=6`, `G²=36` 페이지, 병렬도 `G·P=162`), P=27 회전은 크기 L=64 Banyan switch로 구현.
9. 결과: 0.18µm CMOS 합성 375.14 Kgates + RAM 88452b/ROM 10872b, 180MHz, latency 5.73µs(<6µs, rate 무관), useful throughput 565Mbps(1/2)~941.8Mbps(5/6), 채널 1.13Gbps.
10. 한계: RTL/합성까지(실측 없음), WLAN 짧은 블록 특화, `Nit=12` 고정(iteration 감축 아님), 정정능력 개선 아닌 HW 효율.
