> 총평: LDPC-CC 디코더의 register-intensive delay line을 순환버퍼로 대체한 FPGA HW 기법, block QC-LDPC(Prime ECC)엔 구조가 이질적이라 이식가치 낮음.

### B. 알고리즘 요약

1. 시스템: LDPC-CC(convolutional LDPC), regular (3,6) rate 1/2, memory `M=128`, LLR wordlength W=6~8bit, AWGN/BPSK, streaming/packet 통신 타깃.
2. 문제: LDPC-CC 디코더는 processor당 `M+1` 길이 delay line이 필요해 register-intensive → FPGA에서 LE 낭비(단일 processor 13246 LE, 16%).
3. 부호식: code-bit `v(nM+φ)`가 이전 정보/코드 비트의 GF(2) parity-check로 생성, parity-check matrix가 lower-diagonal → 인코딩 단순·저지연.
4. 복호: message-passing, iteration 대신 물리적으로 독립된 processor(delay element+CN 1개+VN 1개)를 타일링, processor i-1 출력이 i 입력(LLR 전달).
5. 핵심 기법: FIFO delay line을 순환버퍼(circular buffer)로 대체, on-chip dual-port RAM + 포인터 `p(t)` 추적으로 구현.
6. 메모리 분할: 순환버퍼를 8개 columnar RAM(높이 P, 폭 W)으로 분할, 정보 LLR 4 + 코드 LLR 4 저장.
7. 주소생성: ROM이 phase φ별 offset `δ` 6개 제공, `µ=(p(t)-δ) mod P`로 순환버퍼 주소 계산, read 주소는 t+Δp에 write 주소로 재사용.
8. 오버클럭: 한 baud period에 모든 메모리 연산 수행 위해 RAM을 base clock의 x3(최대 128MHz)로 구동, MUX A~E로 클록도메인 전환.
9. 결과: 메모리 기반이 register 기반 대비 5배 이상 효율(processor당 2388 LE, 3%), 최대 33 processor 수용, throughput 120Mb/s, BER 2x10^-6 @ Eb/N0=3dB(FPGA 실측, Matlab과 일치).
10. 한계: 저자 지식상 최초 LDPC-CC HW 디코더, block LDPC 대비 BER는 열세, sign-min 근사·유한정밀도 열화. NAND/SSD 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| message-passing 복호 (tanh/sign-min CN) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 낮음: Prime ECC는 block QC-LDPC min-sum, convolutional processor 파이프라인과 구조 이질 |
| LLR 순환버퍼/메모리 라우팅 (dual-port RAM, x3 clock) | 디코더 HW 모델 (z=32, HCU, GT) | 낮음: LDPC-CC delay-line 전용 최적화, QC-LDPC block 병렬구조에 부적합 |
| soft LLR 양자화 (W=6~8bit) | `channel.cpp` LLR 테이블 (`PARAM_LLR`) | 낮음: 일반 LLR 양자화 수준, 이미 보유 |

적용 가치: **낮음** — LDPC-CC(convolutional) 디코더의 delay-line을 순환버퍼로 바꾼 FPGA HW 기법으로, Prime ECC의 block QC-LDPC(z=32 병렬, min-sum)와는 부호 구조·디코더 아키텍처가 근본적으로 다르다.
