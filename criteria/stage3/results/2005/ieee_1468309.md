### B. 알고리즘 요약

1. 시스템: circulant permutation matrix(`P`, L×L)와 zero matrix로 구성된 binary QC-LDPC, base(mother) matrix `M(H)` 기반. 시뮬은 rate 1/2, length 1000(25×25 블록, 20×40 base).
2. 문제: 일반 LDPC는 인코딩 복잡도 `O(N)+O(g^2)`와 PCM 저장 메모리가 커서 상용화 걸림돌.
3. 핵심분석1(girth): base matrix의 block-cycle(길이 2l)을 exponent chain으로 대수적으로 표현, 두 block-cycle이 p개 overlap하면 girth 상한 `2(2l+2k-p)` (Theorem 4). regular QC-LDPC(zero 없음)는 girth ≤ 12 (Corollary 5).
4. 핵심분석2(array code): array code `H(q,j)`의 girth 6, 길이-6 cycle 개수 `2q(q-1)(j choose 3)/3` 폐형식 도출.
5. 핵심기법(B-LDPC): parity part `H_P`를 almost lower triangular로 제약 + Richardson-Urbanke 분해 `[A B T; C D E]`에서 `Φ=ET^{-1}B+D`를 identity로 만들도록 `P_x,P_y` exponent 선택(Theorem 9, 식(7)(8)).
6. 핵심식: `Φ=I`이면 `p1 = ET^{-1}As^T + Cs^T` 계산이 circulant 크기 L 무관하게 `O(N)`으로 선형화(비-sparse `Φ^{-1}` 회피).
7. 인코딩 절차: Step1 `p1^T=E[P... ]As^T` 등 4-step, 총 XOR 연산 `(c̄-1+R)N-2L`, 일반 almost-lower-triangular 대비 복잡도비 `θ<1` (L>2, L 클수록 유리).
8. 부호설계(Principle A): 저degree 노드 간 cycle 최소길이를 먼저 최대화 후 고degree 노드 girth 최대화, degree분포는 density evolution으로 결정(정보비트 error prob만 최적화).
9. 결과: B-LDPCC1(Principle A 적용)은 random LDPC와 저SNR 동등·고SNR 우수, B-LDPCC2(미적용)는 error floor 발생 → Principle A가 error-floor 억제에 기여. 메모리는 1/L로 축소.
10. 한계: HW/throughput/gatecount 없음(시뮬만), 소형(rate1/2 len1000) 예제, 인코더 중심으로 디코더 정정능력 개선은 girth 통제의 간접효과.
