### B. 알고리즘 요약

1. 대상: 고rate QC-LDPC(순환 permutation 행렬 `t×t`, `J×K` 블록) + (modified) Min-Sum, 예제 `(4608,4096)` `(4,36)` regular, 6bit 양자화.
2. 문제: LDPC codec은 대부분 메모리가 HW를 지배 → 부분병렬 구조에서 extrinsic 메시지 저장 메모리를 줄이는 것이 핵심.
3. 관찰: 식 (1) 한 행(check node)의 C2V 크기값은 두 값(최소 min1, 두번째 min2)만 존재 → 압축 저장 가능.
4. 재구성 절차: `L_cn = S_n - R_cn + I_n` (3), `R_cv = D·sgn(L_cn)·min|L_cn|` (4), `S_v = Σ R_mv` (5)로 재배열해 V2C를 개별 저장하지 않고 합산 `S_v`만 저장.
5. 핵심 식 의미: (3)(5)로 V2C(`L_cv`)를 즉석 재계산해 CPU로 전달 → 개별 V2C 메모리 제거, C2V만 압축 형태로 유지.
6. 압축 형식: R-memory 한 엔트리 = {min1, min2, min2의 index, 전체 sign}; data distributor가 이를 개별 `R_cv`로 복원(sign-magnitude→2's complement 변환 SM-2'S 포함).
7. CPU 최적화: `2×2` Pseudo Rank Order Filter(PROF)로 presorted 두 열에서 최소 2개를 1-stage compare-and-swap로 추출, index를 stage별 aggregate → 연산 지연 최소화.
8. 부분병렬화: `p` 인접 행/열을 한 메모리 엔트리에 저장, barrel-shifter 기반 data scheduling/reverse scheduling으로 행·열 순서 충돌 해소; QC 구조상 shift 패턴이 적어(블록열당 ~4) `W:1` MUX 스위칭으로 단순화.
9. 결과: 관례 설계 258,048bit → 제안 163,840bit로 약 37% 메모리 절감(6bit 양자화, 128 circulant 예); throughput은 `p`에 선형 증가(작은 HW 오버헤드).
10. 한계: BER/합성 수치 없음(해석적 메모리 표만), min1/min2/sign 압축·min-sum은 표준, 부호 예제 단일.
