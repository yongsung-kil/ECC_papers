> 총평: 조기종료로 절약된 iteration을 입력 버퍼로 재분배해 디코더 회로 복잡도를 낮추는 Markov chain 기반 버퍼 최적화 letter로, HW throughput/전력 관점에서 참고가치는 있으나 부호·복호 알고리즘 자체 개선은 아님.

### B. 알고리즘 요약

1. 시스템: LDPC BP 디코더 입력에 길이 `NB ≥ 2N` 버퍼를 두고, 디코더는 `N` LLR 윈도우 단위로 처리(rate 1/3·5/6·9/10, length 16384).
2. 풀려는 문제: 디코더 처리능력은 최대 iteration에 맞춰 설계되나 조기종료(valid codeword) 시 다음 블록 시작 전까지 유휴 처리능력이 낭비됨.
3. 핵심 아이디어: 입력 버퍼가 조기종료로 남은 처리능력을 흡수해 필요한 블록에 on-demand 추가 iteration을 공급 → 디코더 회로 속도(복잡도)를 낮춤.
4. 핵심 모델: 스텝 `n`의 가용 iteration 수 `nB`(버퍼 비움 정도 종속)를 상태로, `nmin = N·T/TD`, `nmax = (NB−N)·T/TD` 범위의 이산시간 **Markov chain**으로 버퍼 상태 `σn` 진화 표현.
5. 핵심 식 의미: `σn`이 `σ(n-1)`에만 의존(마르코프성) → irreducible·aperiodic 체인의 정상상태 확률벡터 `π=π(0)·P^n`로 수렴, 정상상태 FER `PF(e)=Σ P(σn=Si)·PF(e|Si)`.
6. 반해석적 기법: (1) 단일 시뮬로 `PF(e|Si)`와 iteration 수 `nit` 분포 획득, (2) 식으로 여러 `nmax/nmin`(=버퍼길이·회로속도)에 대한 FER를 재계산 → 반복 시뮬 없이 최적 버퍼 길이 도출.
7. 결과: 목표 FER `10^-5`에서 nmin 임계값 rate 1/3·5/6·9/10 = 32·12·10(평균 iteration 22.5·9.5·8.4), 필요 버퍼길이 각각 2.5N·4N·5N.
8. 관찰: rate가 낮을수록 필요한 디코더 회로 속도가 높고, 버퍼를 키우면 짧은 버퍼로도 큰 nmax(예: 100) 효과를 얻어 회로 부담 완화.
9. 확장성: 동일 기법을 turbo code iterative decoder에도 적용 가능하다고 언급.
10. 한계: 부호·복호 알고리즘 자체는 손대지 않는 스케줄링/버퍼 최적화이며, 특정 코드 SW 시뮬뿐이고 실제 RTL/gatecount/throughput 수치·NAND 채널 없음.
