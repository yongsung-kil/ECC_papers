### B. 알고리즘 요약

1. 대상: AWGN, `(3,6)`-regular LDPC, 부호길이 `N=1536` (`b=256`, 3x6 sub-block), rate 1/2, 8bit 양자화(sign1/int4/frac3).
2. 문제: partially-parallel 디코더는 row/column 연산을 독립 수행해 sub-block별 shift 차이로 column 연산이 최신 α를 못 쓰는 타이밍 손실 → 제한 iteration 내 수렴·BER 저하.
3. 구조: PCM을 wc x wr sub-block(각 `b x b` shifted identity)으로 구성, wc CFU + wr BFU로 부분병렬.
4. 핵심 기법: column 연산을 row 연산과 **동시(in conjunction)** 수행 - wc row 연산 직후 각 check node에 연결된 모든 bit node에 대해 `wc x wr` column 연산을 최신 α로 즉시 실행.
5. 효과: column 연산 횟수가 기존 대비 wc배 증가하고 최신 α 반영 → message-passing 효율↑, 제한 iteration 내 수렴 확률↑.
6. CN 연산: Gallager `f(x)` LUT+선형보상, 또는 Min-Sum 근사(식5) `min|β|`·sign곱 사용.
7. HW: pipeline BFU가 3 column 연산 6 cycle, row 연산 10 cycle(min 1st/2nd 7 cycle) - row가 병목이라 throughput 미저하.
8. FPGA 구현: Virtex-II xc2v6000, >100MHz, column module slice +32%(P&R 기준 전체 +13%), block RAM 동수.
9. 결과: avg iteration 최대 45% 감소, 디코딩 지연 45%↓, throughput 23→38 Mbps(1.7x), BER 개선(제한 iteration서 refs 미달 구간을 0 BER 달성).
10. 한계: AWGN 시뮬만, ASIC 게이트카운트 없음, 이득이 부분병렬 스케줄 구조에 종속.
