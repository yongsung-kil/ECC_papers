### B. 알고리즘 요약

1. 데이터 스토리지용 고rate(8/9) LDPC의 error-floor(BER<10^-11)를 AWGN 채널에서 FPGA로 10^-12까지 실측 평가.
2. 문제: SW 시뮬은 10^-6~10^-8까지만 가능해 스토리지가 요구하는 초저 BER의 error-floor를 관측 불가.
3. 두 부호군 비교: `(v,j,t)-DDS` 기반과 `array code` 기반 QC-LDPC, column weight `j=3,4,5`.
4. DDS 부호: 순환행렬 `H=[H1..Ht]`, column weight `j`, row weight `k=jt`, length `N=vt`, rate>`(1-1/t)`; `j>3`이면 girth=6.
5. Array 부호: `p×p` 순환치환행렬 `σ`의 `j×p` 블록행렬, length `N=p^2`, 4-cycle free, `dmin≥6`이나 6-cycle 다수.
6. 관측: column weight 증가 시 error-floor는 낮아지나 waterfall(cliff) 성능은 저하 — array j=5는 floor<10^-11로 스토리지에 유망.
7. Girth 최적화: `H`를 정수격자 위 slope별 평행선 집합으로 해석, 삼각형(6-cycle)/사각형(8-cycle)을 만드는 slope를 회피.
8. 6-cycle 회피 조건 `2sb - sa - sab ≠ 0 mod p` (j=3) — 이를 만족하는 slope 집합 테이블(p=5~103)로 6/8-cycle-free 부호 구성.
9. 결과: p=173 j=3 slope 최적화로 rate 8/9, N=4671 부호 구성 시 기존 array 대비 약 0.5dB gain.
10. 한계: 순수 부호설계·평가 중심, 디코더 알고리즘·HW 복잡도/throughput 수치는 별도 문헌 참조, 채널은 AWGN(NAND 아님).
