### B. 알고리즘 요약

1. 대상: 임의 **binary linear code**(factor graph/parity-check 표현), 실험은 rate `1/2·1/4` Gallager LDPC(length 40~200, 좌차수 3, 우차수 4/6)와 Reed-Muller.
2. 문제: min-sum/BP는 성능은 좋으나 factor graph에 cycle이 있으면 수렴·성공조건 해석이 어렵고 ML 인증이 없음.
3. 모델: 임의 discrete memoryless symmetric channel(BSC/AWGN/BEC), cost `γ_i` = negative LLR을 code bit에 부여.
4. 핵심 기법 1: 각 체크노드의 local codeword polytope 교집합으로 **LCLP**(Linear Code Linear Program) polytope `Q`를 만들고, `min Σγ_i f_i` LP로 최적점을 찾음 — 정수해면 ML codeword(= **ML certificate property**).
5. 핵심 식: `f_i ∈ [0,1]` + 각 체크 `Σ_{S} w_{j,S}=1`, `f_i = Σ_{S∋i} w_{j,S}`(짝수 부분집합만) → codeword 아닌 fractional vertex 발생이 실패 원인.
6. 핵심 기법 2: **pseudocodeword** = LCLP 해의 정수배; 실패 iff 비용≤0인 nonzero pseudocodeword 존재. stopping set/deviation set/graph cover/trellis cycle을 통합.
7. 구현 디테일: check degree가 큰 일반부호는 제약이 지수적 → Yannakakis parity polytope로 `O(d³)` 변수·제약의 compact 표현, redundant parity check 추가로 relaxation tightening.
8. 최적화: LP는 simplex/ellipsoid로 풂; lift-and-project 계층으로 성능↑ 복잡도↑ (마지막이 ML).
9. 결과: **fractional distance** `d_frac`이 고전 distance의 하한이고 LP는 `⌈d_frac/2⌉-1` 오류까지 정정; girth `g`에 대해 max-fractional distance ≥ 지수적(logarithmic girth면 `n^{const}`). BSC에서 min-sum보다 우수, sum-product보단 다소 열세.
10. 한계: **HW 미설계**, LP solver(simplex/CPLEX) 기반이라 계산모델이 min-sum 반복복호와 이질적, 시뮬 block length 40~200에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP 완화 복호(LCLP) | 대응 없음 | simplex/ellipsoid LP는 HW min-sum 반복복호와 계산모델 상이, 이식 불가 |
| pseudocodeword / fractional distance | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` (진단 참고만) | 간접 — error-floor/수렴실패 원인 분석 개념으로만 참고 가능, 알고리즘 대체는 아님 |
| stopping set / min-sum 연결 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `C2V_Cal()` | 간접(이론적) — min-sum 실패조건과의 관계 서술뿐 |

적용 가치: **낮음** — LP 복호는 이론·ML 인증 관점의 창시적 기여이나, Prime ECC의 min-sum 기반 HW 디코더와 계산모델이 근본적으로 달라 떼어 쓸 수 없다(pseudocodeword 개념을 error-floor 진단 렌즈로만 참고 가능).
