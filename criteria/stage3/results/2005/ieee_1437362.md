### B. 알고리즘 요약

1. 대상: IPP(Irregular Partitioned Permutation) LDPC — global matrix M의 각 1을 p×p cyclic permutation, 0을 zero matrix로 치환해 jp×kp PCM 구성(부분병렬 BP 디코더 HW 친화).
2. 문제: 기존 SFT 기반 IPP는 생성쌍 (a,b) 존재 제약으로 파라미터 자유도 낮고 error floor 높음.
3. 사이클 분석: Theorem 1 — H의 모든 사이클은 M의 fold 없는 closed path(CP)에 대응. Theorem 2 — 길이-2n CP가 만드는 H 사이클 길이 `L = 2n·p/(p,S)`.
4. LEF: length expansion factor `p/(p,S)`, total cyclic shift `S≠0`이면 LEF>1(p가 소수면 LEF=p). CP 사이클을 더 긴 사이클로 확장.
5. ACE 연계: CP의 `ACE=a`, `LEF=f`이면 H 사이클 `ACE=f·a`. 즉 작은 ACE의 CP는 LEF>1(S≠0)로 강제하면 harmful small-ACE cycle 회피.
6. 핵심 방법(SSC, Stopping Set Conditioning): degree distribution 최적 M 생성 → ACE<threshold인 illegal CP 검출 → 각 CP에 부등식 `S≠0` 부과 → 모든 부등식 만족하는 cyclic shift 집합을 computer search.
7. 검출 가속: 인접행렬 `A`의 거듭제곱 `A^n`으로 length-n path 존재 판정, M이 작아 검출 빠름. 고려 CP는 길이 ≤18로 제한.
8. 파라미터: 16×32 global matrix, degree distribution [5]로 최적(max bit-node degree 11), p=97(len 3104), p=257(len 8224), rate-1/2.
9. 결과: AWGN/BP 최대 200 iteration. Waterfall은 SFT-IPP와 거의 동일, 저-BER 영역에서 SSC-IPP가 확연히 우수, 두 SSC-IPP 모두 BER floor <10^-7.
10. 한계: 순수 부호설계(HW 미설계), illegal CP 과다 시 shift search 실패 가능, ACE threshold 수동 best-effort 조정.
