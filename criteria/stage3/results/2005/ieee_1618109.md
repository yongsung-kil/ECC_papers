### B. 알고리즘 요약

1. 대상: moderate-length(10^3<N<10^4) rate-compatible LDPC, PEG로 만든 regular 모체부호, girth>=8 강제, AWGN.
2. 문제: 시변 채널의 AMC/ARQ에서 단일 코덱으로 다양한 rate를 지원해야 함. 기존 puncturing은 저rate에서 성능 나쁨.
3. 접근: 고rate 모체부호에서 저rate 부호를 만드는 두 방법 제안 - extending, information-nulling (puncturing과 반대 방향).
4. Extending: 모체 H(M0xN0)에 M_i 행·열을 padding, upper-right는 0, bottom-right는 column weight 3(sparse 보장) + 행마다 identity 2개로 dependence 확보. 결과 rate `R_l = K/(N0 + sum M_j)`.
5. 핵심 관찰: extended H의 첫 M0행 평균 row weight `S0 = t/(1-R0)` (t=column weight). R0=16/20일 때 S0=15인데 padded 행은 row weight 5뿐 -> row weight 편차가 커 성능 저하 원인.
6. Rmin: extending으로 도달 가능한 최소 rate 임계 `Rmin`을 최초 정의(K행 추가·총행수=N 상황). 모체 rate가 낮을수록 Rmin 감소.
7. Information-nulling: parity는 모두 유지하고 정보비트 일부를 null로 채워 유효 정보비트를 줄임 -> rate `K_eff/(K_eff+M)`. rate 임계 없음, 넓은 rate 범위.
8. 복잡도(Table1): information-nulling이 인코딩 복잡도 최저, puncturing 최고. extending 복잡도는 채널이 좋아질수록 감소.
9. 결과: 모체(2560,2048,16/20) 기준 rate 16/23~16/20 및 4/8~4/13 실험. 저rate(0.4~0.6)에서 extending이 puncturing보다 우수, 고rate에서는 열등. information-nulling은 넓은 rate·임계없이 양호.
10. 한계: 순수 부호구성법(rate 유연화)이며 error-floor/정정능력 직접 개선 주장 없음, HW 미설계, 시뮬만.
