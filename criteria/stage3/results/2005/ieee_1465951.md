### B. 알고리즘 요약

1. 대상: (1024, 3, 6)-regular LDPC(rate 1/2), partially-parallel 디코더, sum-product(belief propagation) message passing, 0.18µm CMOS.
2. 문제: fully-parallel(Blanksby)은 배선복잡·대면적, fully-serial(Yeo)은 9612 cycle/iter·wide-input MUX 필요 → 부분병렬이 절충안이나 PU 공유 시 행/열 처리순서·의존성 스케줄이 성능 좌우.
3. 배경: 전통적으로 CTV는 VTC 전체 완료 후에만 시작 → overlap 스케줄로 row 합산값이 준비되면 CTV를 VTC 도중 시작해 iter당 사이클 축소.
4. 핵심기법: 스케줄링을 **H matrix의 행·열 permutation**으로 정식화 — permuted 행렬의 좌하단·우상단 코너에 빈 슬롯을 최대화.
5. 원리: 열에 빈 슬롯 k개면 그 열은 `⌊k/m⌋` 사이클 일찍 시작 가능; m×n 좌하단 블록이 모두 비면 모든 variable PU가 한 사이클 일찍 시작(제어 단순화 위해 연속 처리 가정).
6. 알고리즘: (Step2) 임의 행 선택, (Step3) 미선택 행 중 CTC(완료될 열 수) 최대 행 선택, 동률이면 CC(공통 열 수) 최대, (Step4) 완료 열을 앞으로 이동, (Step6-7) permuted H를 m행/n열씩 각 사이클에 배정.
7. 구현 디테일: check PU가 row별 계산한 Λij를 local memory에 그룹 저장해 variable PU가 후속 접근(면적 절감).
8. 학습절차: 없음 (휴리스틱 그리디 permutation, QC 전용 [9]와 달리 general LDPC 적용).
9. 결과: (1024,3,6)에서 다양한 병렬도(4,8)~(64,32)에 대해 iter당 사이클 평균 25% 절감; (32,64) 디코더 10.08mm²·200MHz·985Mbps(8 iteration), Blanksby fully-parallel(52.5mm²) 대비 대면적 절감.
10. 한계: 정정성능·부호설계 무개선, hard/soft 양자화 비트폭 미명시, gatecount 미보고, QC 구조 미활용.

| Λij local memory 그룹 저장 | 대응 없음 (부분병렬 PU 공유 메모리 배치; 우리 z=32 row-병렬 구조와 상이) | HW 메모리 구성 이슈, 알고리즘 무관 |
| 메시지 패싱 (sum-product) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `C2V_Cal()`, `CNU_Update_New_Mag()` (우리는 min-sum 근사) | 표준 BP, 신규 기여 아님 |

적용 가치: **낮음** — 부호 스펙 고정·pipeline 스케줄(Dual-Update)을 이미 가진 QC-LDPC Prime ECC 입장에서, general LDPC용 permutation 스케줄은 이득 중복이며 정정성능 개선도 없음.
