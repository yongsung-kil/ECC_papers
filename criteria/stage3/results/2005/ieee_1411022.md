### B. 알고리즘 요약

1. AWGN·BPSK 채널에서 (N,j,k) 정규 Gallager(=regular LDPC) 코드, 예제 `(1008,3,6)` (rate 0.5)를 SPA로 복호.
2. 문제: 기존 flooding schedule(FS)은 수렴이 느려 latency·저장 부담이 큼 — turbo-decoding schedule(TDS)의 빠른 수렴을 이론적으로 설명하려 함.
3. 모델: Gallager 코드를 `j`개 super code `Ci`(각 `N/k`개 SPCC `(k,k-1,2)`의 direct sum)의 교집합으로 봄 = 병렬연접부호.
4. TDS: 각 super code에 SISO 디코더를 두고 extrinsic 정보 `Eni`를 교환. LLR 도메인 반복: `Ll(m)=Zl(m)+Al(m)`, `tanh(Eli/2)=∏ tanh(Ll(n)/2)`.
5. FS: 2-phase(체크노드↔변수노드) 스케줄, LDPC를 반복부호+SPCC 직렬연접으로 해석.
6. 핵심 도구: EXIT chart로 각 구성 디코더의 입출력 상호정보 `IA=I(X;A)`, `IE=I(X;E)`를 추적, Gaussian·consistency 가정 `A=µA·X+nA`, `µA=σA²/2`.
7. `IE`는 시간평균 `IE≈1-(1/N)Σ log2(1+e^{-xn·En})`로 근사.
8. FS는 (j,1,j) 반복부호 + (k,k-1,2) SPCC EXIT 곡선으로, TDS는 3D turbo EXIT(`IE1=T(IE2,IE3)`)로 궤적 시각화.
9. 결과: `(1008,3,6)` 코드에서 TDS가 FS보다 훨씬 빠르게 수렴(Fig.3b iteration 수 감소), 임계값은 두 스케줄 모두 1.15dB로 동일.
10. 한계: HW 미설계·시뮬/이론뿐, SPA(tanh) 전제, min-sum/양자화/NAND 미고려, 정정능력(임계값) 자체 개선 없음.
