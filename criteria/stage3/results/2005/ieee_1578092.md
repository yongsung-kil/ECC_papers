> 총평: QC-LDPC partly-parallel 디코더의 CN/VN 오버랩 스케줄링을 최적화해 throughput을 약 2배로 끌어올림. 부호 성능은 무변화이며 특정 HW 아키텍처에 의존적.

### B. 알고리즘 요약

1. 유한체 곱셈구조 기반 regular QC-LDPC(순환shift identity `Ixs,t`, `xs,t=b^s·a^t mod m`, VN degree `j`·CN degree `k`)의 partly-parallel 디코더가 대상.
2. 문제: 데이터 의존성 때문에 CNFU와 VNFU가 교대로만 동작 → 하드웨어 이용효율(HUE)·throughput 저하.
3. OMP: 체크노드/변수노드 갱신을 부분 오버랩. intra 대기 `wcv`, inter 대기 `wvc`를 정의하고 throughput gain `G = 2mn/(n·wcv+(n-1)·wvc)`로 표현.
4. HW 제약 `wcv+wvc ≥ m` (연속 iteration 시작 간격 ≥ m). 등호에서 `G≈2`가 최대이며 wcv를 더 줄여도 이득 미미.
5. intra 최소화: CNFU 시작 행주소 `{c0..cj-1}`를 조정해 컬럼주소 차 `d(l,ts)=(cl+xl,t)-(cs+xs,t) mod m`를 최소화, `wcv*=w+1`. exhaustive search 복잡도 `m^(j-1)` (사전계산, HW 오버헤드 없음).
6. inter 최소화: Lemma1로 `wvc ≥ wcv*`가 충분필요조건, `wvc*=max(wcv*, m-wcv*)`.
7. 핵심 개선점: 각 VNFU가 자기만의 reference block row를 선택([8]은 모든 VNFU가 공통 row 사용) → 대기시간 추가 단축.
8. 주소 생성: 매 iteration마다 시작주소를 `+w mod m` 시프트 → 대기시간이 iteration-invariant.
9. 결과: n=30에서 [8] 대비 wcv가 모두 감소, `wcv* ≤ m/2` 만족으로 HUE 100%, G가 [8]의 1.40~1.91에서 1.97~1.98로 향상 (Table I).
10. 한계: 특정 HW 구조·유한체 QC 구성 가정, throughput은 이론적 계산값뿐, 실측·합성 없음.
