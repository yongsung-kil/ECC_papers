### B. 알고리즘 요약

1. 목표: 짧고 단순한 constituent matrix들을 structured base matrix에 겹쳐(superpose) girth≥6인 structured LDPC/QC-LDPC 부호를 대수적으로 구성.
2. 문제: pseudorandom LDPC는 waterfall은 좋으나 인코딩이 복잡; QC-LDPC는 shift-register 인코딩(선형복잡도) 가능하나 좋은 구조적 구성법 필요.
3. 핵심 제약: RC(row-column) constraint - 두 행/열이 1-성분을 1개 초과 공유 금지 → length-4 cycle 제거 → girth≥6.
4. 핵심 기법: base matrix `H_b`의 각 1-entry를 constituent matrix set `Q`의 멤버로, 0-entry를 zero matrix로 치환(replacement constraint: 한 행/열의 nonzero submatrix 모두 distinct).
5. base가 circulant면 cyclic replacement로 QC-LDPC가 되고, constituent가 circulant array면 결과도 circulant array.
6. Class-I: EG(Euclidean geometry) 라인 incidence vector로 circulant 구성, column/row decomposition으로 permutation matrix 생성.
7. Class-II: 정보심볼 2개 shortened RS code의 coset을 symbol-location matrix로 변환하여 constituent 구성.
8. Class-III: cyclic Hamming code의 minimum-weight codeword로 circulant 구성.
9. 결과(시뮬, SPA max 100 iter): (8176,4088) rate 1/2는 Shannon limit 1.45dB, (16128,14127) rate 0.876은 0.8dB이며 BER 10^-x까지 error floor 없음; MacKay 랜덤부호와 waterfall 성능 동등.
10. 한계: HW/디코더 미설계, 이론+SPA 시뮬만, EG/RS/Hamming 대수구조에 rate/length가 종속.

```
| circulant permutation / base matrix | ecc_data.h PCM_b (base+shift) | 우리 부호도 base+circulant shift 표현이라 개념 대응하나 특정 H-matrix 고정 |
| SPA 시뮬 성능평가 | decoder.cpp LDPC_Decoder() | 논문은 SPA, 우리는 min-sum - 디코딩 알고리즘 기여는 없음(대응 없음) |
```

적용 가치: **중간** - QC-LDPC 구조 구성법으로 개념은 정합하나, Prime ECC의 z=32 고rate H-matrix를 교체/재검증하는 부담이 크고 디코더/HW 기여가 없어 실이식 가치는 제한적.
