### B. 알고리즘 요약

1. 대상: irregular LDPC 앙상블 `{λ_i}`(variable-degree)/`{ρ_j}`(check-degree), BSC(crossover `p0=0.05`), 목표오율 `p_t=10^-6`.
2. 문제: threshold 최대화로 설계한 부호는 threshold 근처에서 iteration이 무한대로 발산 → rate제약下 복잡도(iteration) 직접 최소화가 필요.
3. 디코더: Gallager Algorithm B — message alphabet `{0,1}`(soft 미사용), CN은 modulo-2 합, VN은 extrinsic 중 `b`개 이상 불일치 시만 뒤집는 majority-류 규칙(식 3, 최적 `b`는 식 4).
4. 핵심 도구: error-rate 기반 EXIT chart `f(p)`가 exact인 알고리즘(Algorithm B)에서, irregular EXIT는 regular elementary EXIT의 선형결합 `f(p)=Σ λ_i f_i(p)`로 분해됨(식 5).
5. 복잡도 정의: iteration당 연산 `5E-C≈5E`(edge 수 비례), 정보비트당 복잡도 `K=N/(Σλ_i/i - Σρ_i/i)` (`N`=iteration 수, 식 6).
6. iteration 추정: EXIT 형상으로 `N=∫_{p_t}^{p0} dp/[p·log(p/f(p))]` (식 9) — 실제 iteration과 매우 근접(Table I: 192 vs 193.41 등).
7. 핵심 정리(Theorem 1): `N` 측정치가 `f(p)≥e^{-2p}` 영역에서 `λ_i`의 convex 함수 → integrand 2차미분 양수로 증명.
8. 최적화: `minimize K s.t. R≥R0` (식 11), rate 제약은 `λ_i`에 선형(식 12) → convex programming(interior-point)으로 최적 `λ_i` 도출(식 13).
9. 결과: check-degree 14 고정·variable degree≤20, 기존(약간 나쁜 채널로 설계) 대비 복잡도 30~40% 절감. Algorithm B는 rate 0.575까지 복호 가능, Table II에 K=1639/530/232 별 최적 분포.
10. 한계: HW 미설계·이론뿐, hard-decision Gallager B + BSC 전용(exact EXIT 성립 알고리즘에 국한), check-degree 고정 가정.
