### B. 알고리즘 요약

1. 대상: iterative(min-sum) 복호에 강한 구조적 LDPC 부호 구성. BiAWGN 채널, min-sum 복호로 random LDPC와 비교.
2. 문제: 다수 구조적 LDPC가 random 부호에 밀리고, 짧은~중간 길이에서 iterative와 ML 복호 성능차가 pseudocodeword 때문에 발생.
3. 핵심 관점: 최소 pseudocodeword 무게 `wmin`이 iterative 복호 성능(특히 error floor)의 주 예측자, `wmin<dmin`인 저중량 pseudocodeword가 특히 유해.
4. 핵심 기법: root 변수노드에서 `d`-regular 트리를 정해진 layer만큼 전개 후, mutually orthogonal Latin squares(MOLS) 기반 연결 알고리즘으로 트리를 닫아 그래프 완성.
5. 설계 목표: Tanner 트리 하한을 만족시키며 `wmin = dmin`이 되도록 구성 → 유해한 저중량 pseudocodeword 제거.
6. 구성 종류: Type I-A/I-B(girth 6, `N=p^{2s}+1`, d=2^s에서 rate>0.5), Type II(=3,=4; girth 6/8, 유한기하 PG/EG-LDPC 및 FGQ-LDPC를 특수사례로 포함).
7. 차수: `d=p^s` 및 `d=p^s+1`(p 소수) 지원, 잘 알려진 finite-geometry LDPC를 일반화.
8. 결과: Type I-A/II girth-6이 comparable random 부호보다 우수, Type I-B는 random과 달리 뚜렷한 error floor 없음. 대다수 부호가 detected error(수렴실패)를 거의 안 냄 → 좋은 wmin 덕.
9. 한계-1: 큰 블록길이 waterfall 영역에선 random 부호가 더 좋고(상대 최소거리 우위), Type II girth-8도 장블록에서 밀림.
10. 한계-2: HW 미설계·시뮬만, 전체 dmin이 작음, dmin=wmin 유지하며 dmin 키우기는 open problem. 부호가 QC 형태 아님.
