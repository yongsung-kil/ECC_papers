### B. 알고리즘 요약

1. WiFi/WiMax/DVB-S 등에 쓰이는 (dv,dc)-regular LDPC(테스트 케이스 576×288, rate 0.5)를 대상으로 sum-product(SPA)/min-sum(MS) 대비 ADMM 기반 LP 디코딩을 분석.
2. 기존 LP 디코딩은 ML 인증(certificate) 특성과 error-floor 미관측 장점이 있지만 범용 LP solver가 계산량·메모리 부담이 커 실용성이 낮다는 문제를 다룸.
3. LP 완화 문제(식 1)에 penalty parameter `μ`, scaled dual variable `λ`를 도입한 augmented Lagrangian(식 2)을 message-passing 형태로 재구성(Barman et al. ADMM-l2).
4. 핵심 알고리즘(Algorithm 1): VN 커널이 `t_i^(k) = Σ(L_ji^(k-1)) - γ_i/μ`를 계산 후 `Π[0,1]^dv` 사영, CN 커널이 parity polytope `P_dcj`(짝수 개 1을 갖는 이진벡터의 convex hull)로 `z_j`를 Euclidean 사영하고 `λ_j`를 갱신.
5. Euclidean projection `Π_Pdc`가 ADMM 복잡도의 핵심 병목(정렬 연산 2회 포함, 비원자적 연산)이며, over-relaxation 파라미터 `ρ`로 수렴성을 개선.
6. VN 커널은 SPA보다 곱셈/나눗셈이 많아 복잡, CN 커널은 SPA의 arctanh(평균 40 클록) 대비 projection(평균 504 클록)이 더 무거움(Core-i7 RDTSC 실측); 메모리 풋프린트는 SPA `N+2m` 대비 ADMM `2(N+m)`로 더 큼.
7. 병렬성 3단계: (1) CN/VN 커널 내부 dv/dc개 값의 병렬 축약, (2) 한 iteration 내 전체 VN→전체 CN 순차 병렬(flooding schedule), (3) 여러 코드워드를 동시 디코딩(SIMD 8-way + Core-i7 4코어/8스레드 OpenMP).
8. 별도 학습/최적화(DE, GA, NN) 없음 - 병렬성 분석 및 소프트웨어 가속(x8) 기반 실험.
9. 결과: 576×288 코드에서 8코어 구현 시 throughput 30Mbps(WiMax 요구사항 충족) 도달; 평균 iteration 수는 SPA 대비 약 1/3, MS 대비 약 1/2로 감소; 단일 코어에서도 2dB일 때 1.7Mbps, 3dB일 때 5Mbps.
10. 한계: 실제 하드웨어(RTL/ASIC/FPGA) 설계·합성 없이 x86 멀티코어 부동소수점 소프트웨어 구현에 그치며, projection 연산의 고정소수점 양자화 및 HW 친화적 구현은 향후 과제로 남김.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP 디코딩 알고리즘(VN/CN 커널, projection) | `decoder.cpp` `LDPC_Decoder()` | Min-Sum과 다른 메시지패싱 체계(사영 기반)라 대체 시 CN/VN 연산 전면 재설계 필요, 이식 난이도 높음 |
| VN/CN flooding 스케줄, 병렬 처리 | `decoder.cpp` `C2V_Cal()`, `CNU_Update_New_Mag()` | flooding 방식 자체는 유사하나 min-sum 대신 사영 연산이라 함수 재사용 어려움 |
| iteration 수 감소(수렴 가속) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` 이터레이션 루프 | 조기 수렴 아이디어는 참고 가치 있으나 알고리즘 기반 자체가 다름 |
| 멀티코어 SIMD 소프트웨어 병렬화 | 대응 없음 | Prime ECC는 HW(RTL) bit-exact 시뮬레이터로 x86 SIMD/OpenMP 최적화와 무관 |
```

적용 가치: 중간 — ADMM-LP는 Min-Sum과 근본적으로 다른 디코딩 패러다임(사영 기반)이라 직접 이식은 어렵지만, iteration 감소·error-floor 회피 특성은 참고할 만하며 아직 HW 미설계 상태라 추가 검증 없이는 적용 판단이 이름.
