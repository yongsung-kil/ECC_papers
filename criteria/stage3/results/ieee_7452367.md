### B. 알고리즘 요약

1. 대상: AWGN/Rayleigh 채널용 multiple-rate QC-LDPC (base matrix + b×b circulant, `n=61440`, `b=512`, target rate {0.4,0.6,0.8}).
2. 문제: rate-compatible 코드 생성 시 puncturing은 blocklength 감소로 성능↓, row-combining(RCEV)은 강결합이라 전 rate 동시 최적화 어렵고 지원 rate 제한.
3. 핵심기법1 (row-splitting): 고rate mother code의 각 행을 여러 행으로 분할(역으로 combine 가능), 열 degree 불변 → VPU 공유 가능. cycle-free라 girth 유지.
4. 핵심기법2 (unequal '1' 분배): 한 행을 `p+1`개 행으로 쪼개되 '1'을 불균등 분배 후 작은 degree 행들을 재결합 → 정수배 제약 제거, 넓은 rate 범위 지원.
5. 목표 행 degree: `dc_k = (1-r1)/(1-rk) · dc1` (식5), row-split/combine이 '1' 개수 불변임을 이용.
6. edge variation [7]: split 후 열 degree 분포를 최적값에 맞추려 '1'을 추가/삭제.
7. 최적화 절차: EXIT charts로 rate별 열 degree 분포 최적화 → 최고rate 코드는 RS-PEG로 큰 girth 확보 → PEXIT로 decoding threshold 계산(gap<0.25dB 제약).
8. 계층 파라미터: level of fragmentation `α = dcK / gcd(dc1..dcK)` (식7), α≤2 권장(HPU 파편화·복잡도 억제).
9. 결과: BER 1e-5에서 DVB-T2(blocklength 64800) 대비 rate 0.6에서 약 0.15dB(AWGN)/0.32dB(Rayleigh), rate 0.8에서 0.09dB 이득(더 짧은 blocklength로).
10. 한계: 순수 부호구성법(HW는 HPU/VPU 공유 개념도만 제시), soft/양자화·NAND 특화 없음, error-floor는 girth 근거의 정성적 주장.
