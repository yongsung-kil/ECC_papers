### B. 알고리즘 요약

1. 대상: 짧은/중간 길이 LDPC 부호를 PEG(progressive edge-growth)로 구성, `n=1000`, `R=0.5`, AWGN 채널.
2. 문제: 기존 LPEG(linear-encoding PEG)는 상삼각 H로 선형 인코딩은 되나 인코딩 시간 `TL=m`으로 code length에 비례해 김.
3. 핵심 아이디어: H를 상삼각 패턴 + M개 submatrix `H1..HM`로 분할, 각 submatrix 내 행들이 같은 열에 '1'을 갖지 않게 설계(조건 B) → submatrix 단위로 parity bit 병렬 생성.
4. FPEG 알고리즘: check node를 M개 그룹 `G1..GM`으로 나누고, 각 심볼노드 edge 생성 시 이미 사용한 그룹 `Gi`를 후보집합 `SC`에서 제거하여 조건 B 유지.
5. 인코딩 시간: `TF = M ≥ dsmax`로 code length 무관. `M=dsmax` 선택 시 최소.
6. 유효조건: 심볼노드 degree 분포가 `Σλj ≥ Σrj/n (누적)`을 만족해야 FPEG 구성 가능(식 8).
7. Girth 하한: 식 (9)로 유도, `J=rmin(dsmax−1)+1`, PEG 원 코드의 girth 하한과 유사함을 증명.
8. MFPEG: FPEG는 `r1`개 degree-1 심볼노드를 만들어 iterative decoding에서 문제(EXIT 불일치 등) → line 14 수정으로 degree-1 노드를 1개로 축소, `TM=(dsmax−1)+r1`.
9. 결과: `m=500,n=1000`에서 `TL=500` 대비 `TF=4~6`, `TM=88~128`. FPEG는 waterfall 개선/error-floor 악화, MFPEG는 LPEG와 유사 성능.
10. 한계: 순수 구성법(HW 미설계, gate/throughput 없음), 정정능력 개선 아님, rate 0.5 예시뿐.
