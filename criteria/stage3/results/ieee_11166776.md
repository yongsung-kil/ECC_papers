### B. 알고리즘 요약

1. 5G NR BG1 QC-LDPC 인코더 대상, `kb=22`, lifting factor `Z=3~384`, 입력 44~8448bit.
2. Gaussian elimination 기반 직접 인코딩은 code length에 대해 저장/연산량이 quadratic으로 증가해 대형 블록에 부적합.
3. protograph 구성으로 만든 base graph의 shift 값 `P[r,c]`으로 각 서브행렬(circulant permutation matrix, Z×Z)을 생성한다고 가정.
4. forward substitution 알고리즘 5단계로 코드워드 생성: 입력 저장 → 서브행렬 A/B/C/D 확장 → `λ = Σ aij·ibj^T` 계산(식 1) → core parity `PC,1~PC,4` 계산(식 2) → other parity 계산 후 정보비트+core parity+other parity 연접.
5. 식(1),(2)는 core parity 4비트를 서브행렬 A와 정보비트만으로 먼저 구하고 이를 재사용해 other parity(42비트)까지 순차적으로 유도하는 구조.
6. cyclic shifter가 base graph 엔트리마다 서브행렬 A/B/C/D를 동적으로 채우는 configurable 회로로 다양한 base graph에 대응.
7. 모든 덧셈을 XOR 논리로 처리, lambda 계산 모듈을 core/extra parity 계산에 재사용해 자원 절감(파이프라인 방식).
8. 별도 학습/최적화 절차 없음 (고정 알고리즘의 HW 구현/시뮬레이션).
9. Z=192에서 6526 clock에 codeword 생성, 50MHz 기준 throughput 32.36Mbps; Z=12에서는 2.03Mbps.
10. SystemVerilog 시뮬레이션(Questa sim)만 수행, FPGA/ASIC 합성·gatecount·전력 등 실제 HW 자원 수치 없음. 인코더 전용이며 디코더/정정성능은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| forward substitution 인코딩 알고리즘 | `encoder.cpp` `LDPC_encoder_4KB()` | 5G 표준 전용 forward substitution 구조로 dual-diagonal 인코더와 설계 원리가 다름 |
| base graph 서브행렬 permutation 저장 최적화 | `encoder.cpp` `Generate_PCM_encoding()` | 메모리 절감 아이디어는 참고 가능하나 5G BG1 고정 구조라 이식 난이도 높음 |
| cyclic shifter / configurable 회로 | 대응 없음 | Prime ECC는 특정 QC-LDPC 고정 구조라 configurable base graph 지원 개념 자체가 없음 |

> 적용 가치: 낮음 — 5G NR 표준 인코더의 HW 시뮬레이션이며 binary NAND LDPC의 디코더/부호설계/정정성능과 무관, 인코딩 메모리 절감 아이디어만 부분 참고 가능.
