### B. 알고리즘 요약

1. 대상은 rate-1/2 irregular random LDPC (2000,1000) PCM(Neal software 생성), AWGN 채널, offset min-sum 복호.
2. 문제: flooding schedule은 CN 전부 갱신 후 VN 전부 갱신(2 phase)이라 한쪽 프로세서가 항상 idle → HW 이용률 50%로 throughput 제한.
3. 기존 overlapped decoding[5]은 row/column permutation으로 부분 overlap하나 실제 PCM에서 완전 이용률 불가, shuffled/layered는 sub-iteration으로 throughput 제한.
4. 핵심 기법 `dual-rail decoding(DRD)`: 노드 간 연결(interconnection)과 무관하게 매 cycle에 한 row(CN)와 두 column(VN)을 동시 처리 → 1 iteration을 3 cycle에 완료, idle 자원 없음.
5. shuffled decoding처럼 각 노드는 partial update(일부는 현재 iteration, 일부는 직전 iteration 메시지 사용)하되, CN·VN을 재스케줄 없이 serial 진행.
6. 데이터플로우를 intra-phase(같은 phase 내 계산·사용, 수렴 촉진)와 inter-phase(직전 phase 생성분 사용)로 개념 구분하나 HW에선 구분 불필요(가용 메시지 접근만).
7. 일반화: NC개 CN유닛·NV개 VN유닛 병렬 시 1 iteration = `max(M/NC, N/NV)` cycle, M/NC=N/NV일 때 이용률 최대(100%).
8. 성능 향상 트릭: column permutation으로 대각 방향이 비도록 재배열해 inter-phase dataflow를 줄여 error 성능 개선(802.16e rate-1/2 base PCM에 알고리즘 적용, greedy candidate 선택).
9. 결과: MAI=15 AWGN에서 DRD(NC=1)는 flooding 대비 BER/FER 열화 무시할 수준, NC 증가 시 소폭 열화, 특정 FER 도달까지 총 cycle 수는 flooding/overlapped/shuffled/TSP보다 크게 적음(Fig.5).
10. 한계: iteration 수는 오히려 소폭 증가(partial update), HW 합성/gate·throughput 수치 없음, offset min-sum·irregular random 부호 가정.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-rail overlapped 스케줄(CN/VN 동시) | decoder.cpp LDPC_Decoder() 이터레이션 루프 | Prime ECC의 Dual-Update(even/odd 교대)와 유사 계열, 스케줄 변형 이식 지점 |
| offset min-sum CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 계열 동일, offset은 Prime ECC 미명시(중복 가능) |
| partial update / 메시지 스케줄링 | decoder.cpp C2V_Cal(), VN_Cal_HD() | 메시지 갱신 순서 변경 = 디코더 루프 내 이식 |
| column permutation(대각 비움) | ecc_top.cpp Load_PCM() (H-matrix) | PCM 재배열은 부호구조 로드 단계, QC 고정 구조라 재검증 부담 |
| NC/NV 병렬 유닛 매핑 | 대응 없음 (Prime ECC는 z_sb=32 row 병렬 고정) | 임의 NC/NV 가정과 고정 z=32 병렬이 불일치 |
```

적용 가치: **낮음~중간** — 완전 overlap 스케줄로 latency/throughput을 높이는 아이디어는 디코더 루프에 이식 가능하나, Prime ECC는 이미 Dual-Update 교대 스케줄을 보유하고 z=32 고정 row 병렬 구조와 임의 NC/NV·column permutation 가정이 정합되지 않아 이득 불확실.
