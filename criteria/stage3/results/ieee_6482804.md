### B. 알고리즘 요약

1. 시스템: IEEE 802.11n H-matrix, 부호길이 `648`bit, sub-matrix `z=27`, code rate `0.5`, BPSK/AWGN, `5 iteration`.
2. 문제: MS 디코더의 CNU(row)·VNU(column) 연산이 교대 실행되어 병렬화 불가, idle cycle 낭비.
3. 복호: MS는 CNU `L(rji)=∏sign·min|L(qij)|`, VNU `L(qij)=L(ci)+Σ L(rj'i)`.
4. MMS: VNU에 scaling factor `γ` 적용 `L(qij)=(L(ci)+ΣL(rj'i))·γ`, `γ=0.75`로 shift-add(`3a/4=a/2+a/4`) 구현.
5. 핵심 기법: parity-check matrix reordering(row·column 동시 이동, 오류정정 성능 불변)으로 CNU/VNU overlap 가능케 함.
6. overlap 패턴 0~4 비교, 제안 pattern 4는 pattern 3 대비 VNU 75%만 사용하며 유사 overlapped cycle 유지.
7. HW: CNU는 20개 2-to-1 magnitude comparator로 min 탐색+XOR sign tree, VNU는 11 adder + shift-add scaling.
8. 아키텍처: permutation/de-permutation 버퍼, CNU/VNU 뱅크, 중간결과 메모리(27 addr/block), control unit.
9. 결과: XC2VP30-7, 194MHz, 76.6Mbps(pipeline 없이), pipeline 삽입 시 127.36Mbps, MS 10450 slices/16184 FF/19332 LUT.
10. 한계: null 서브행렬 많은 648bit 부호에만 overlap 이득 국한, MMS는 FPGA서 면적↑ 이득 미미(ASIC 권장), soft 양자화 비트수 미명시.
