### B. 알고리즘 요약

1. 시스템: 3D 스택 메모리(TSV wide-I/O)에 전용 EC 다이를 얹어 QC-LDPC 인코딩/디코딩; 코드워드 (512,64)/(1024,128)/(2048,256)/(4096,512), extended Hamming SECDED 대비.
2. 문제: 미세공정에서 MBU(multi-bit upset)가 급증해 SEC ECC로 정정 불가, 저비용 다중오류정정 필요.
3. 모델: 메모리 오류를 BSC(cross-over α)로 모델링, 목표 WER=10^-6에서 α 내성 비교.
4. 핵심기법1: 온라인 메모리 스크러빙 정책 — 스크러빙 간격/시간에 SR/SW로 전체 메모리를 스캔·정정, 정상 MR/MW보다 낮은 우선순위로 투명 동작(Fig.2,3).
5. 핵심식: QC-LDPC 부호 dv=4, dc=36; write invalidation / on-the-fly 인코딩·디코딩으로 check bit 무효화 대응.
6. 핵심기법2: OS 제어 적응 — 고오류 환경에서 스크러빙이 못 따라가면 write/read on-the-fly로 전환해 무결성 보존(접근시간 증가 대가).
7. 디코더: Layered/Flooded Min-Sum(LMS/FMS)과 Self-Corrected 변형(LSCMS/FSCMS)을 3-bit/4-bit soft 입력·내부메시지로 운용.
8. HW: 40nm LP CMOS로 (2048,256)/(4096,512) codec 물리합성(Cadence Encounter), Z(sub-matrix) 16~128별 area/latency/energy per-bit(Table I).
9. 결과: WER=10^-6에서 SECDED는 α=1e-5, LDPC는 α=8e-2 내성 → 4710배 fault tolerance; soft-info·긴 부호일수록 이득 큼; 인코딩 energy/latency/area는 디코딩보다 3~4자릿수 작고 Z 독립.
10. 한계: gatecount·throughput 미보고, 디코더 알고리즘은 기성 min-sum 계열 재사용, dv=4/dc=36 부호 스펙과 스크러빙 구조가 NAND 컨트롤러와 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered/Flooded Min-Sum, Self-Corrected MS | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 우리 min-sum과 동계열; Self-Corrected 변형은 CN 갱신에 얹을 후보 |
| 3-bit/4-bit soft 입력 양자화 | `channel.cpp` `Set_LLR_Th()`, `VN_Cal_HD()` 등 | 우리는 soft-2~3bit까지; 4-bit 입력은 미지원 영역 |
| QC-LDPC H-matrix(dv=4/dc=36) 로드 | `ecc_top.cpp` `Load_PCM()` | 부호 스펙 상이(우리 고rate VN degree 17), 직접 재사용 불가 |
| 스크러빙/TSV wide-I/O 아키텍처 | 대응 없음 | 메모리 시스템 레벨 정책, NAND 컨트롤러 파이프라인과 다름 |

적용 가치: **중간** — Self-Corrected Min-Sum 변형과 soft-info(3/4bit) 이득 비교는 디코더 개선 참고가 되지만, 논문 본질은 3D 메모리 스크러빙 시스템 아키텍처이고 부호 스펙이 NAND 고rate QC-LDPC와 달라 알고리즘만 발췌 참조 수준.
