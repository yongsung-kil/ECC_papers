> 총평: min-sum 디코더에 successive relaxation(고정 β=0.5 damping)을 LLR 도메인에서 적용하면 표준 MS 대비 최대 0.6dB 이득을 얻는다는 것을 시뮬레이션으로 재확인한 정정 논문으로, 이식 난이도가 낮은 VN/CN 업데이트 변형.

### B. 알고리즘 요약

1. 배경: Hemati et al.이 제안한 successive relaxation(SR)을 LDPC 반복복호의 fixed-point 문제에 적용하는 기법을 LR(likelihood ratio)/LLR(log-likelihood ratio) 두 도메인에서 BP/MS 조합으로 재검증.
2. 문제: 선행 연구의 SR-BP-LR 시뮬레이션이 all-zero codeword 가정에 기반했는데 SR-LR 알고리즘은 비대칭(non-symmetric)이라 결과가 부정확함 — 이를 정정.
3. 채널 모델: 이진 입력 AWGN(BIAWGN) 채널, `Y = X + N`, `N~N(0, σ²=N0/2)`.
4. 핵심 기법: SR은 매 반복에서 메시지를 `이전값`과 `표준 업데이트값`을 relaxation factor `β`로 가중 결합(`m_new = β·m_std + (1-β)·m_old` 형태)해 fixed-point 수렴을 가속.
5. MS in LR domain 연산: VAR `f(λ0,λ1,λ2)=λ0λ1λ2`, CHK `g(λ1,λ2)=[min(λ1^sgn,λ2^sgn)]^sgn·sgn` — LLR domain min-sum과 수학적으로 동치이나 LR 도메인에서 곱셈 기반 연산.
6. BP in LLR domain 연산: VAR `f=Λ0+Λ1+Λ2`, CHK `g=2tanh⁻¹(tanh(Λ1/2)tanh(Λ2/2))` (표준 sum-product).
7. 4가지 변형(SR-BP-LR, SR-BP-LLR, SR-MS-LR, SR-MS-LLR) 모두 relaxation factor `β`만 추가 적용하는 구조적으로 단순한 변형.
8. 최적화 절차: 코드/Eb·N0/최대반복수 M과 거의 무관하게 알고리즘별 최적 β를 실험적으로 탐색(SR-BP-LR≈0.8, SR-BP-LLR≈0.5, SR-MS-LR≈0.98, SR-MS-LLR≈0.5).
9. 결과: (504,252) regular 코드, (1268,456) irregular 코드 모두에서 SR-MS-LLR이 4변형 중 최고 성능, 표준 MS 대비 최대 약 0.6dB, 표준 BP 대비 최대 약 0.3dB 이득(M=200/10000 두 조건 모두).
10. 한계: 짧은 코드 길이(≤1268bit) BIAWGN 채널 시뮬레이션(waterfall 영역)만 다루며, error floor·HW 구현·양자화·복잡도 분석은 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SR-MS-LLR (min-sum + relaxation factor β) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN/VN 업데이트 시 이전 메시지와 β 가중 결합만 추가하면 되는 경량 변형, min-sum 코어 위에 이식 난이도 낮음 |
| iteration별 메시지 relaxation 적용 | `decoder.cpp` `LDPC_Decoder()`, `ecc_data.h` `PARAM_LLR` | 반복별 β 스케줄을 LLR 테이블처럼 파라미터화 가능 |
| SR-BP-LR / SR-MS-LR (LR domain, 곱셈 기반) | 대응 없음 | Prime ECC는 min-sum/LLR 기반이라 LR domain 곱셈 연산은 구조 불일치, 이식 대상 아님 |

> 적용 가치: 중간 — SR-MS-LLR은 min-sum 위에 고정 relaxation factor(β=0.5)만 추가하는 저비용 변형으로 waterfall 영역 성능 개선 가능성이 있으나, 짧은 코드·이론 시뮬 검증뿐이라 NAND 고rate·긴 코드·HW 조건에서 이득 재검증 필요.
