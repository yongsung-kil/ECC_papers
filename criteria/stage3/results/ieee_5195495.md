### B. 알고리즘 요약

1. DVB-S2 표준 대상, code length L=16200 bit 고정, 8종 rate(1/2~9/10)를 지원하는 QC-LDPC 코드 설계.
2. 문제의식: 기존 rate-compatible LDPC(펑처링 기반)와 달리, 서로 다른 rate의 H-matrix가 서로 독립적으로 도출되는 가변 rate 구성법이 필요.
3. Vandermonde-like 순환치환행렬(permutation matrix `A`, `A^x` cyclic shift)로 base matrix `Hb`(M×N, regular, full rank)를 먼저 생성, girth≥6 보장.
4. 핵심 기법: `Hb`의 각 원소를 `p×q` 작은 행렬(`Iq`, `0`, 또는 zero-row-inserted 순환치환 `a_{p×q}`)로 치환·확장하는 "replacement and expansion" 방식으로 확장 행렬 `H(Hb,p,q)` 생성.
5. 핵심 식: `Rate = 1 - (p·M)/(q·N)` (식3) — p=q이면 base rate와 동일, p를 증가시키면 zero-row 삽입으로 parity 제약이 늘어 낮은 rate 코드가 파생됨.
6. `Hb`가 length-4 cycle이 없으면 확장된 `H(Hb,p,q)`도 length-4 cycle이 보존됨(구조적 girth 보장).
7. Tanner graph 관점에서 `H(Hb,p,q)`의 bipartite graph는 `Hb`의 각 branch를 `a_{p×q}` 서브그래프로 치환한 형태이며, 인접 서브섹션은 check node 연결을 한 칸씩 shift한 관계(QC 구조 특성).
8. 별도 학습/최적화 절차 없음(대수적 폐형 구성).
9. 결과: L=16200, column weight=3 고정, row weight 6~24로 8개 rate 구현, BER=10^-5 기준 Shannon limit 대비 평균 약 0.72~0.75 dB 이내.
10. 한계: AWGN/BPSK/sum-product(50회 반복) 시뮬레이션만 수행, HW 구현·복잡도·throughput 분석 없음, NAND 채널 특성 반영 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Vandermonde 기반 QC-LDPC base matrix + expansion 구성 | `ecc_top.cpp` `Load_PCM()` | 부호설계 자체(QC-LDPC 생성)이나 Prime ECC는 특정 QC-LDPC 구조 고정이라 이식 난이도 높음 |
| 가변 rate(1/2~9/10) 지원 구조 | 대응 없음 | Prime ECC는 고rate(~0.9) 단일 부호 시뮬레이터로 다중 rate 스위칭 아키텍처 자체가 범위 밖 |
| sum-product(BP) 디코딩 | 대응 없음 | Prime ECC는 min-sum 기반, tanh 기반 BP와 다름 |

적용 가치: **낮음** — DVB-S2(위성/무선 통신) 대상 가변 rate 부호 구성법이며 부호설계 영역은 프로파일상 이식 난이도가 가장 높은 항목이고, 디코더/HW 관련 기여가 전혀 없음.
