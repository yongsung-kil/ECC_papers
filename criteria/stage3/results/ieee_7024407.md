### B. 알고리즘 요약

1. DVB-S2 위성 방송용 FEC로 LDPC를 택했으나, 실제 구현은 rate-1/2 `6x12` parity check matrix의 초소형 예제 코드다.
2. 인코딩은 Richardson-Urbanke의 approximate lower-triangulation(식 2.1~2.6)으로 parity `p1,p2`를 계산한다.
3. 디코딩은 **bit-flipping** 알고리즘: initialization → parity update(check node가 parity 식 만족 여부 판정) → bit update(수신 메시지 다수결 불일치 시 비트 flip), 반복.
4. 복호는 hard-decision(무 LLR)이며, 시뮬 결과 **1비트 오류만 정정** 가능하다고 명시.
5. 구조는 serial-to-parallel → codeword → parity check(6개 f0~f5 XOR 식) → bit-flipping → parallel-to-serial의 VHDL 블록으로 나뉜다.
6. 4개 설계 변형(6x12, 15x20, 4x7 등, message-passing 포함)을 period/frequency/memory 관점에서 비교.
7. 결과: Virtex-4 100MHz에서 bitrate 21.4Mbps, 시스템 동작 frequency 3.57MHz, 6비트당 280ns(28 clock), 구현 시 4.98us.
8. 자원: slice 102/10752, FF 107, LUT 186, memory 약 209936 KB (표·수치 신뢰도 낮음).
9. LA(Logic Analyzer)로 VHDL 시뮬과 실제 구현 결과 일치를 검증.
10. 한계: toy 사이즈 코드, hard-decision bit-flipping의 1비트 정정 한계, error-floor/waterfall·soft-read·고rate 논의 전무.
