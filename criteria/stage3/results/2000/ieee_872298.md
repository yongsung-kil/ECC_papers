### B. 알고리즘 요약

1. 시스템: 자기기록 ME2PR4(수정 확장 EPR4) 채널, 사용자 데이터를 rate 16/17 quasi-MTR(RLL) 부호로 먼저 부호화 후 고rate LDPC로 추가 부호화.
2. 문제: 기존 자기기록용 turbo(2개 성분부호 또는 conv+PR)는 약한 conv 부호에 의존 - 대신 강력한 블록부호(LDPC)로 대체.
3. 모델: PR 채널을 사실상 두 번째 부호로 보고, quasi-MTR이 지배적 오류패턴 `[1,-1,1]`을 줄이며 RLL 제약 제공.
4. 핵심 기법: 읽기단에서 PR 채널 디코더의 soft-output을 LDPC 디코더에 입력, LDPC의 extrinsic 정보를 다시 PR 채널 디코더의 a priori로 되먹여 turbo 복호 성립.
5. 알고리즘 구성: PR 채널 디코더 = Log-MAP, LDPC 디코더 = log-domain belief propagation(BP).
6. 부호: LDPC1 = rate 0.9358 / len 4376 / info 4095(MacKay 웹), LDPC2(자체설계) = rate 0.9402 / len 4629 / info 4352, 필요 가중치로 random parity-check 생성 후 length-4 cycle 제거.
7. 조기종료: 유효 LDPC codeword 발견 시 전체 복호 중단(자연스러운 stopping + 오류블록 flag) - conv 기반 대비 장점.
8. 결과(overall rate): LDPC2 시스템 overall rate 0.8674(user block 4096b), LDPC1 시스템 0.8622(user block 1854b).
9. 결과(이득): user density 2.505에서 LDPC2가 uncoded PR4 대비 6.7dB, EPR4 대비 5.3dB(BER 1e-5); density 2.8에서 각각 7.5dB, 5.9dB 이득.
10. 한계: 시뮬만(Kavcic 비선형 채널모델 결과는 미표기), HW 미설계, 자기기록 PR/turbo-equalization 전제라 NAND read 채널과 이질적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| log-domain BP LDPC 복호 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum 근사, 본 논문은 full log-BP(sum-product) - 알고리즘 계열 유사하나 근사 방식 다름 |
| 고rate LDPC 부호(random, 4-cycle 제거) | `ecc_top.cpp` `Load_PCM()` | 고rate(~0.94) 부호라는 점만 접점, QC 구조 아닌 random 구성이라 직접 이식 부적합 |
| valid-codeword 조기종료 | `full_CRC.cpp` / `partial_CRC.cpp` | Prime ECC는 CRC 기반 조기종료 보유 - 본 논문의 parity-check 만족 종료와 목적 유사(중복 기여) |
| PR 채널 Log-MAP + turbo equalization | 대응 없음 | Prime ECC 채널은 AWGN/RBER, ISI/turbo-equalization 채널 디코더 없음 |

적용 가치: **낮음** - 고rate·soft LDPC라는 점은 NAND ECC와 접점이 있으나, 자기기록 PR채널·Log-MAP turbo-equalization 구조와 random(비QC) 부호 구성이라 Prime ECC에 떼어 쓸 신규 요소가 없음(BP·조기종료는 이미 보유).
