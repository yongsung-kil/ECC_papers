> 총평: 2000년 무선통신용 LDPC 디코더의 초기 재구성형 FPGA(2-node 순차) 구현 논문 — 확률영역 곱셈·나눗셈 정규화 기반이라 Prime ECC의 로그영역 min-sum HW와 정합 낮아 이식성 하.

### B. 알고리즘 요약

1. 시스템은 무선통신용 near-Shannon ECC(turbo vs LDPC)를 재구성형 HW(FPGA)에 구현하는 비교 연구로, LDPC는 rate 1/4, block 1200bit 예제.
2. 문제: turbo(MAP/log-MAP)는 상태 metric의 직렬 의존성으로 병렬화가 제한 — LDPC는 병렬성·타일성이 커 HW에 유리함을 논증.
3. 부호/채널: regular Gallager LDPC, `t=G^T u mod2` 인코딩, AWGN(±1 signaling) 가정.
4. 핵심 기법: sum-product 메시지패싱을 bipartite graph의 check node/bit node 네트워크로 구현 — check는 `r_ij` (parity 만족 확률), bit는 `q_ij` 계산.
5. 노드 기능은 포트 수(행/열의 1 개수)에만 의존하고 위치 무관 → regular 부호는 두 종류 노드만으로 타일링(복제·연결) 가능.
6. bit node는 확률 정규화(`q0+q1=1`, `e0+e1=1`) 필요 — 나눗셈은 digit-recurrence(Divcell) 8bit 정밀도 파이프라인 회로로 구현.
7. HW 병목: 포트 수 증가 시 곱셈기·정규화기가 급증(16-port bit node면 32개 15-operand 곱셈기+17 정규화기) → 완전병렬 구현은 당시 FPGA에 안 맞음.
8. 실제 설계: bit node 1개 + check node 1개만 얹고 q/r/p/e 값을 메모리 통해 순차 처리(2-node system), check node는 address ROM으로 그래프 구조에 맞게 재정렬 접근.
9. 결과: rate 1/4·block 1200 코드가 Xilinx Virtex에서 3039 LUT + 9600B RAM, 50MHz, 10 iteration 기준 4Mb/s(coded)/1Mb/s(user); 메모리 뱅크 2배화로 2배 개선 가능.
10. 한계: 확률영역 곱셈·나눗셈 기반이라 로그영역 전환(곱→덧셈) 미적용, 정규화의 로그영역 구현 난제 미해결, 무선통신 타깃이며 정정성능 개선 아님.
