### B. 알고리즘 요약

1. 목적: distributive law를 일반화해 "product 함수의 marginalize(MPF)" 문제를 message passing으로 푸는 통합 알고리즘 GDL 제시(semitutorial).
2. 문제: BP/Viterbi/BCJR/GTW/FFT/Hadamard 등 여러 fast 알고리즘이 개별 발견됐으나 상호관계가 불명확 -> 하나의 틀로 통합.
3. 모델: commutative semiring `(S, +, ·)`(가법 역원 불필요)상에서 local domain/local kernel의 곱을 global kernel로 두고 특정 변수를 marginalize.
4. 핵심 기법: local domain을 junction tree로 조직하면 각 vertex가 이웃으로부터 받은 메시지 곱을 local kernel과 결합해 전송(message 식(3.1), state 식(3.2))하는 message passing으로 MPF 해결.
5. 핵심 식: `min-sum` semiring(min=+, +=·)에서 marginalize하면 최단경로(Viterbi), `sum-product`에서는 BCJR/BP가 됨 — semiring 선택이 알고리즘을 결정.
6. junction tree 존재 판정: local domain graph의 maximal-weight spanning tree 무게가 상한과 같으면 존재(Theorem 4.1), 없으면 moral graph triangulation으로 clique 기반 tree 구성.
7. 스케줄링: single-vertex(serial, leaf->target)와 all-vertices(fully parallel/serial/hybrid) 지원, Scheduling Theorem 3.1로 message trellis 경로 존재 시 정확성 증명.
8. 복잡도: junction tree 복잡도를 edge complexity 합 `(5.3)/(5.6)`으로 정의, all-vertices는 single-vertex의 최대 4배 이내(최적 triangulation은 NP-complete).
9. 결과: Gallager-Tanner-Wiberg LDPC 복호, turbo, Viterbi, BCJR, Pearl BP, Hadamard/FFT, 행렬연쇄곱을 GDL 특수사례로 유도.
10. 한계: junction tree(무cycle)에서만 정확성 보증, cycle 있는 LDPC/turbo는 실험적으로만 우수(single-cycle만 이론 정립), HW·부호설계·수치 성능 전무.
