### B. 알고리즘 요약

1. LDPC를 `t=3`, information length `30000`, code rate `1/2`, iteration `40`으로 Sum-Product(SPA) 복호하는 구현 이슈를 다룸.
2. 문제: SPA는 두 확률차의 subtraction을 포함해 양자화 오차에 민감 -> 실용 구현 시 심각한 성능 저하.
3. 해결: subtraction을 회피하는 parity likelihood ratio(PLR) 함수 기반 PLRA 제안, 완전 산술에서는 SPA와 등가.
4. PLR 함수 `f(·)`는 양수 집합에 대해 재귀적으로 정의되며(식(1)(2)), 주어진 집합에 대해 유일하게 결정됨(Appendix 증명).
5. 복호는 Horizontal step(check node, 식(3))과 Vertical step(variable node, 식(4))을 반복하며 subtraction 미포함.
6. two-way(forward-backward) 재귀로 horizontal/vertical step을 계산해 SPA와 동일 연산량(곱셈/덧셈 수 유사) 유지.
7. 양자화: 비율값이 `[0,1]` 범위이므로 log 도메인 midtread uniform으로 `s`-레벨 양자화, f-함수 입력을 clip해 LR=1(정보없음) 오류 방지.
8. 빠른 구현: 로그 테이블 + 64x64 2D look-up table로 horizontal step을 table search로 처리(SPA는 4096x4096 곱셈 테이블 필요).
9. 결과: PLRA는 6bit로 이상적 성능에 근접, SPA는 동등 성능에 12bit 필요(2배) + 양자화 유발 error-floor가 PLRA에서 훨씬 낮음.
10. 한계: HW 미설계(시뮬만), gate/throughput 수치 없음, adaptive 양자화는 향후 과제로만 언급.
