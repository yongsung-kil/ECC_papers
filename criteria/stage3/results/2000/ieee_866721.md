### B. 알고리즘 요약

1. 문제: turbo/threshold/LDPC message-passing 등 인기 suboptimal 비트복호기가 "common sense"로 개발됐을 뿐, 최적 복호를 어떻게/얼마나 잘 근사하는지 미규명.
2. 핵심 가정: memoryless 채널, 등확률 codeword, binary block code `C`를 parity-check matrix `H=(h_ij)` 즉 dual code `C'`의 생성자 `h_i`로 정의.
3. 핵심 기법: dual code 기반(Hartmann-Rudolph식) 전체 부호 `C`의 최적 복호식을 유도한 뒤 `h_i`만 명시적으로 포함하도록 재작성.
4. 핵심 식: 비트 `m`의 log-likelihood `L_m = log[P(c_m=0|r)/P(c_m=1|r)]`를 dual generator 항으로 전개(식 (1))가 bit-optimal 기준.
5. turbo 표현: 두 성분부호 `C1,C2`(dual 생성자집합 `I1,I2`, `I1∩I2=∅`, `I1∪I2=I`)로 분해, iteration `(α+1)`에서의 turbo LLR을 동일 형식으로 표현.
6. 근사 구조: `Δ_j = [p(r_j|0)-p(r_j|1)]/[p(r_j|0)+p(r_j|1)]` 및 `A_i∘A_j` 곱연산 항으로 iterative 갱신을 대수적으로 기술.
7. 통합 해석: turbo 알고리즘을 블록도로 표현하고 이를 최적해 (1)과 직접 비교, Gallager LDPC message-passing과 threshold decoding도 동일 방식으로 분석 가능하다고 제시.
8. 학습/최적화: 없음(대수적 유도만).
9. 결과: suboptimal 알고리즘들이 bit-optimal의 어떤 근사인지, 특수한 경우 근사가 얼마나 좋은지에 대한 새 통찰(정량 성능수치·부호실험 없음).
10. 한계: 순수 이론 프레임워크, HW·복잡도·BER 결과 없음, 새로운 복호기나 성능개선 제시 없음.
