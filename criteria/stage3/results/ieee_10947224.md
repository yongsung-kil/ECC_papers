### B. 알고리즘 요약

1. 시스템: 3D TLC NAND flash SSD, 기본 ECC로 QC-LDPC(다양한 code rate) 사용, FTL이 read/write/erase 및 read-retry를 관리.
2. 문제: 고밀도 NAND는 신뢰성이 낮아 단일 code rate LDPC로는 read retry가 빈번해 성능 저하가 심각하고, 낮은 code rate ECC를 전체 데이터에 적용하면 parity 저장비용이 과도해짐.
3. 핵심 가정: flash 수명을 P/E 사이클 기준 Young/Middle/Old 3개 신뢰성 구간으로 나누고, 데이터를 hot read(자주 읽힘)/other read로 분류.
4. 핵심 기법 1단(READECC table): 신뢰성 구간별로 strong ECC(ECCLS/ECCMS/ECCHS, code rate 0.86/0.8/0.66)와 normal ECC(ECCN, rate 0.9) 쌍을 두고, hot read 데이터만 strong ECC로 재인코딩해 read retry를 감소.
5. 핵심 기법 2단(access-characteristic-aware recoding): read counter+threshold(TR)로 hot read를 식별, retry가 실제 발생한 hot read 데이터만 재인코딩(host read 시 또는 GC/wear-leveling 시점에)해 recoding 비용 최소화.
6. 부수 트릭(storage cost optimization): strong ECC용 parity 저장공간 확보를 위해 논리페이지 크기를 가변화(strong 12kB / normal 16kB, 모두 4kB 정렬)해 매핑테이블 오버헤드 없이 저장비용 절감.
7. 학습/최적화 절차 없음(휴리스틱 정책 설계, degree distribution/디코더 알고리즘 변경 없음).
8. 결과: FEMU 시뮬레이션에서 old 신뢰성 구간 IOPS 최대 46% 개선(합성 워크로드), 실제 워크로드에서 DECC 대비 IOPS 최대 21.7% 추가 개선, 저장비용 33.3% 절감(adaptive logical page), recoding 비용 최대 2.2%.
9. 정정능력/waterfall/error-floor/iteration/latency 등 디코더 성능 지표는 전혀 다루지 않음 (LDPC 자체는 기존 것을 그대로 사용).
10. 한계: 순수 시스템/FTL 시뮬레이션(FEMU)이며 LDPC 부호구성·디코딩 알고리즘·HW 설계는 전혀 다루지 않음. "LDPC 디코딩 과정을 수정하지 않으며 READECC와 직교적"이라고 저자 스스로 명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC code rate 선택/전환 정책 (READECC table) | 대응 없음 | Prime ECC는 시뮬레이터로 단일 H-matrix 기반 bit-exact 디코더이며 FTL 레벨 code rate 스위칭 정책과 무관 |
| hot read 식별 및 재인코딩(recoding) | 대응 없음 | SSD 컨트롤러/FTL 레벨 로직으로 ECC 엔진 범위 밖 |
| 가변크기 논리페이지(storage cost optimization) | 대응 없음 | 매핑테이블/저장구조 정책으로 디코더/인코더 코드와 무관 |

> 적용 가치: 낮음 - LDPC 디코딩/부호설계를 전혀 다루지 않는 SSD 컨트롤러 FTL 레벨 정책(어떤 code rate의 기존 LDPC를 언제 쓸지) 논문으로, Prime ECC 코드베이스(디코더/인코더/HW)와 접점이 없음.
