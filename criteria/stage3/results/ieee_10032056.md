### B. 알고리즘 요약

1. 대상: 고밀도(TLC/QLC) NAND, 8kB page, LDPC `code rate=0.94`, BP 디코딩, soft read-retry로 read i-level을 점증.
2. 문제: RBER를 사전에 모르므로 read level을 낮은 값부터 점증(read-retry)해 불필요한 sensing/전송/decoding latency 누적.
3. 관찰: 처음 두 read voltage 사이 "error-prone area"에 놓인 셀 비율이 최종 성공 read i-level과 비례.
4. 기법1 (예측): error-prone 셀 수를 세어 페이지 크기로 나눈 비율을 corr-table(7 entry, ~7~10 byte, DRAM 상주)로 조회해 최종 read i-level을 예측 후 바로 그 level로 read.
5. corr-table은 flash chip type별 특성화(7000 page read)로 구축, 예측이 실제 level과 정확 일치 확률 50~70%, ±1 level 포함 시 90%+.
6. 기법2 (fine-grained): 정수 i-level을 소수 d-level(예 1.25/1.5/1.75)로 분할, 각 retry에서 read voltage를 1개만 추가(TLC 0.25, MLC 0.5, QLC 0.125 단위).
7. d-level 추가 시 asymmetric interstate error 고려해 RBER 높은 P6-P7 영역에 우선 배치.
8. PreLDPC: hard read(i-level1) → 다음 d-level → 실패 시 예측 i-level 점프 → 그 후 d-level 단위 fine-grained retry (Algorithm 1).
9. 결과: Baseline(progressive 7 i-level) 대비 read latency 평균 30% 감소(P/E 4000·1개월 retention), 세분화 단독 7%, 예측 단독 25%.
10. 한계: HW 미설계, 디코더/부호 알고리즘 변경 없음, MATLAB(LDPC)+Disksim(SSD) 시뮬만, low P/E·짧은 retention에선 예측 이득 음수.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| soft read voltage/threshold 배치, d-level | channel.cpp Set_R_Offset(), Set_LLR_Th() | read voltage 개수·위치 개념은 닿으나 논문은 컨트롤러 retry 정책이라 직접 대응 약함 |
| read-retry 정책 / 최종 read level 예측 | 대응 없음 | 디코더 이터레이션 외부의 sensing 스케줄이라 코드베이스 대응 모듈 없음 |
| LDPC 디코딩(BP) | decoder.cpp LDPC_Decoder() | 논문은 디코더 내부를 건드리지 않음(블랙박스로 사용) |
```

마지막으로 적용 가치: **낮음** — read voltage/soft-read 비용 관점에서 채널 threshold 설정과 개념적으로만 겹치고, 논문 기여(예측·retry 세분화)는 디코더/부호가 아닌 컨트롤러 read 정책이라 Prime ECC 이식 대상이 아니다.
