### B. 알고리즘 요약

1. TD(k,n) transversal design의 incidence matrix를 PCM으로 쓰는 LDPC 부호 계열; column weight `k`, row weight `n`, block length `N=n^2`, rate `R≥(n-k)/n`.
2. 문제: 기존 TD LDPC(주로 column weight 3, Pasch-free)는 error-floor를 좌우하는 작은 stopping set을 체계적으로 회피하지 못함. column weight 4+로 확장 필요.
3. 채널 가정: binary erasure channel(BEC). BEC에서 복호성능은 stopping set이 완전히 결정하므로 작은 stopping set 회피가 목표.
4. 핵심기법: TD를 MOLS로 환원하고, MOLS를 Galois field 기반 `L_q^{(alpha,beta)}[x,y]=alpha·x+beta·y` 단순구조로 정의. stopping set을 Latin square의 "full-correlating subrectangle"로 재해석.
5. 핵심식: `-polygon` 조건 `sum (-1)^{t-1} s_t = 0` (over F_q) — 작은 subrectangle(=stopping set) 존재 여부를 scale factor 대수조건으로 판정.
6. 확장: `L_q^2`(column weight 4)에서 크기 8 stopping set 두 유형을 회피하는 scale factor 제약 C1-C3 유도, 나아가 크기 10까지 억제하는 C4-C7 유도.
7. 구현 디테일: 특정 대표계 `(alpha_i,beta_i)` 선택으로 PCM을 quasi-cyclic 구조(circulant q×q 블록)로 만들어 shift-register 기반 선형복잡도 인코딩 가능.
8. 최적화: 대수적 유도(C1-C4) + 크기 10 stopping set 대규모 컴퓨터 탐색으로 C5-C7 추가 도출.
9. 결과: `L_q^2` 부호에서 stopping distance 8→10 상향; L2^13(N=169,R=0.71) 등에서 C1-C3 만족 시 크기 8·9 stopping set 0개, C4로 크기 10 개수 대폭 감소, BER ~3.7e-8 수준.
10. 한계: BEC·peeling decoder 시뮬만; AWGN/soft-read 미검증, HW 미설계, 정정능력은 stopping set 관점에 한정.
