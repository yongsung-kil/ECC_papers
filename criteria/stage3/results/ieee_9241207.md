### B. 알고리즘 요약

1. 대상은 regular/irregular QC-LDPC(protograph 기반, base matrix `B`, circulant lifting size `p`) - 일반 통신표준(802.16e/802.11n/5G)급 스펙, 특정 rate/length 예제 없음.
2. 풀려는 문제: minimum distance `d_min`이 작은 저weight 코드워드는 error-floor와 undetectable error의 주 원인인데, 기존 연구는 trapping-set 확장기법으로 코드워드를 탐색해 계산비용이 크다는 한계.
3. 핵심 가정: column-regular(weight γ=2) 코드는 코드워드-사이클 대응이 명확하나 γ>2/irregular인 경우 불명확 - 이를 SC-LDPC 이론([1])을 일반화해 해결.
4. 핵심 기법: `Theorem 1` - cycle-coverable 행렬 `H̃`의 최소 코드워드 `T(x)`는 component codeword `t_j(x)`들의 합 `T(x) = Σ t_j(x)/s · 2`(`s`=사이클 참여횟수)로 분해 가능함을 증명.
5. `Corollary 3`의 change-making 방정식(`Σ 2W(t_j) = s·Σ w_i·(i/p, i/p)`)을 만족하는 사이클 조합을 `Algorithm 1`로 열거 - 사이클 길이가 "동전 액면", 코드워드 weight가 "총액"인 change-making 문제로 환원.
6. 부수 트릭: 탐색 복잡도 `∆_total`(식 9)은 girth `g̃`와 사이클 개수 `N_i`에 종속 - 실제 구현에서 MacKay의 기존 최소거리 탐색 툴([10])로 저weight 코드워드를 먼저 찾은 뒤 사이클 분해.
7. `Algorithm 2`(설계 절차): ensemble 상한 `d_{Ep(B)}`와 개별 코드 상한 `d_{H(x)}` 계산 → 코드가 상한을 못 만족하면 저weight 코드워드 `T`를 찾아 그 최단 사이클(`L^T_min`-cycle)들을 모두 제거하도록 `H(x)` 재구성 → 반복.
8. 학습/최적화 절차 없음 - 결정론적 사이클 제거 반복(경험적 가정: 최단 사이클이 다른 저weight 코드워드에도 공통 관여).
9. 결과(예시): `p=83`, `m=3` 코드에서 8-cycle 전부 제거 시 `d_min`이 22→24로 개선, 동일 girth/d_min 조건에서 기존 문헌([8]) 코드보다 BER 성능 우수(Fig.1, BPSK+AWGN, BP 100 iteration).
10. 한계: 매우 작은 lifting size(p=83) 단일 사례만 검증, HW 미설계, NAND/flash 채널 미고려, 고차원 대형 코드에서 탐색 복잡도 확장성 미검증.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 사이클 제거 기반 QC-LDPC H-matrix 구성(Algorithm 2) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체 재설계 방법론이므로 새로운 PCM 생성 로직에 해당, 이식난이도 "높음" 레이어 |
| 최소거리(d_min) 저하 원인인 저weight 코드워드/짧은 사이클 억제 | 대응 없음 | Prime ECC는 오프라인에서 이미 고정된 H-matrix(`PCM_b`)를 사용, 사이클 탐색·제거 알고리즘 자체는 코드에 없음 |
| error-floor 개선(trapping-set 유발 short cycle 억제) | `decoder.cpp` `LDPC_Decoder()` | 개선 대상은 디코더가 아닌 부호구조이나, 결과적으로 min-sum 반복복호의 error-floor 완화에 기여하는 방향은 Prime ECC의 관심사(§5 "관심 높음: error-floor")와 일치 |
| BP/min-sum 디코더(성능 검증용) | `decoder.cpp` `C2V_Cal()` | 논문은 이상적 BP(100회 iteration) 사용, Prime ECC의 min-sum과는 알고리즘 다르나 반복복호 프레임워크는 동일 |

> 적용 가치: 중간 - H-matrix 재설계는 이식난이도가 "높음"(프로파일 §4)이라 코드 직접 이식은 어려우나, error-floor 완화를 위한 "짧은 사이클을 코드워드 단위로 선택 제거"하는 설계 방법론은 Prime ECC의 고rate QC-LDPC 부호설계 시 참고할 가치가 있음(단, 소규모 사례만 검증되어 확장성 확인 필요).
