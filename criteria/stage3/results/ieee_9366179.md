### B. 알고리즘 요약

1. Binary/non-binary irregular LDPC 코드(rate 0.41065 및 0.5, code length 336~5000)의 저복잡도 인코딩 알고리즘을 다룸.
2. Generator matrix `G` 기반 인코딩은 code length에 대해 복잡도가 이차(quadratic)로 증가해 병목이 되는 문제를 다룸.
3. Parity-check 행렬 `H`를 행/열 치환만으로 `H_P`(parity part), `H_I`(message part)로 분리하고 `H_P p^T = -H_I u^T`를 푸는 프레임워크를 가정.
4. 핵심 기법: `H_P`를 대각 서브행렬 `L_i`(가능한 한 diagonal 또는 cycle matrix, 아니면 approximate triangular matrix, ATM)를 갖는 block triangular 행렬 `H`로 반복적 전처리 변환.
5. 핵심 식 `L_i p_i^T = -Σ_{j>i}(K_{i,j} p_j^T) - H_{I,i} u^T`: 대각 서브행렬이 diagonal/cycle이면 역행렬 계산 없이 선형시간으로 `p_i` 산출 가능해 복잡도 절감.
6. 확장 기법으로 Huang-Zhu(HZ) non-binary cycle code 인코딩 알고리즘의 일반화임을 이론적으로 증명(Theorem 1), non-binary cycle code에서는 HZ와 동일 결과로 수렴.
7. 부수 구현 디테일: cycle matrix 풀이는 `ω_i` 재귀식(3k-1 곱셈, 2(k-1) 덧셈)으로 O(k) 처리, ATM 서브행렬은 RU의 back-substitution 방식 재사용.
8. 최적화 절차: greedy 알고리즘으로 ATM 근사 변환, SB(Singly Bordered) block-diagonalization으로 서브블록 분할 후 행/열 permutation 반복 수행.
9. 결과: F8 비이진 코드(길이 336~1680)와 F2 이진 코드(길이 1000~5000) 각 100개 parity-check 행렬 평균 실험에서 제안 알고리즘이 RU 알고리즘 대비 곱셈/덧셈 연산수 모두 항상 더 낮음(예: F2 길이 5000에서 α/µ 17473/23467 vs RU 20892/26847).
10. 한계: HW 미설계, 순수 이론적 연산 카운트(곱셈/덧셈) 비교로 실제 HW 면적/throughput 검증 없음, Prime ECC처럼 특정 QC-LDPC(circulant) 구조에 특화되지 않은 범용 이진/비이진 프레임워크.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 인코딩 복잡도 절감(H_P 전처리) | `encoder.cpp` `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal로 고정된 부호 PCM(`PCM_b_enc`) 생성 방식이라, 본 논문의 block-triangular 저가중치 대각화 전처리를 대체 적용할 여지가 이론적으로 있음 |
| 열 재배치(column reordering)로 서브행렬 구조화 | `encoder.cpp` `Column_Reordering_Dual_Update()` | 열/행 permutation 기반 구조화라는 점에서 개념적 유사성 있으나 목적(dual-update 스케줄 vs 대각화)이 다름 |
| RU(ATM) 대비 저복잡도 back-substitution | `encoder.cpp` `LDPC_encoder_4KB()` | 인코딩 연산 자체를 다루는 최상위 함수이나, Prime ECC 인코더는 이미 dual-diagonal 고정 구조라 이 알고리즘 대체 이식은 난이도 높음(§4 "인코더: 중~높음") |

> 적용 가치: 중간 — encoder 복잡도 절감이라는 방향성은 Prime ECC와 관련 있으나, dual-diagonal로 고정된 기존 인코더 구조를 대체해야 하므로 이식 난이도가 높고(프로파일 §4 "인코더: 중~높음"), 검증도 이론적 연산 카운트에 그쳐 HW 이득 실증이 없음.
