### B. 알고리즘 요약

1. 대상은 5G-NR-Like raptor-like QC-LDPC로, nested base matrix `B1..BL`와 nested lifting(`Z=A·2^j`, 작은 lifting이 큰 lifting에 modulo로 내포)로 rate-compatible·length-scalable을 달성한다.
2. 문제: nested base matrix와 lifting size가 2차원 grid로 얽혀 error floor 성능 절충이 어렵고, 기존 cPEG는 최대 base matrix 하나에만 적용돼 nested 구조에 부적합하다.
3. 핵심 관찰: 큰 lifting 성능이 작은 lifting 성능과 강하게 상관(Tanner graph cycle이 연관)되므로, nested lifting 설계를 lifting size별 layer로 분해한다(`layer j` = `Z=A·2^j`).
4. 핵심 기법(tree growth): 각 layer에서 base matrix 전체 edge를 `e1..eτL`로 nested 순서 인덱싱하고, 각 edge의 lifting value 후보를 tree의 level로 확장(layer0은 A개 자식, layer j>1은 nested 관계로 최소 2배 자식).
5. tree는 diversified 중간 결과를 보존해 여러 base matrix 간 error floor 절충을 가능케 한다(핵심 이유).
6. 핵심 기법(tree pruning): tree size `σj`가 `σ1` 초과 시 세 기준을 조합해 `σth`까지 가지치기.
7. 세 기준: (1) improved cycle classification(저복잡도, 임의 level), (2) minimum distance test(ANC 알고리즘), (3) BLER performance evaluation(고정밀·고비용). `σ1~10^5, σ2~10^3, σ3~10^2`.
8. improved cycle classification: cycle을 길이 `g`와 1-CN 개수 `λ`로 분류. BP의 CN 식 `tanh` 근거로 1-CN이 적은 cycle이 trapping set에 더 해로움 → 이를 우선 제거. nested 구조에서 ACE보다 안정적(λ 불변).
9. 결과: 5G-NR BG2 K=6, `Φ13={13,26}`에 적용해 rate 0.375~0.75, layered BP 50 iter, AWGN에서 Z=13 전체·Z=26(R≤0.55) error floor와 minimum distance, undetected error 개선.
10. 한계: 순수 lifting 구성법(HW·디코더 없음), 5G-NR nested/raptor-like 구조 전제, base matrix 자체는 미최적화(향후 과제).
