> 총평: LDPC convolutional 부호의 weight spectrum 계산 알고리즘. 복호/부호설계 이식 대상 아님, NAND block QC-LDPC와 무관.

### B. 알고리즘 요약

1. 시스템: rate `R=k/n`, memory `ms`인 LDPC convolutional 부호(QC block 부호의 circulant를 다항식 `H(D)` monomial로 치환해 구성), 예시 rate 2/5, `ms=4`.
2. 문제: error-floor 예측에 필요한 minimum free distance와 low-part weight spectrum 계산에서 기존 BEAST/syndrome BEAST는 memory가 큰 부호에 RAM 소모 과다로 부적합.
3. 핵심 기법: 부호트리에서 syndrome 상태 `s(t)`와 누적 weight `w(t)`로 노드를 정의, forward 트리와 reversed 부호 `C̃`의 backward 트리를 각각 탐색해 상태 매칭으로 codeword를 발견(식 8,9,15,16).
4. forward-backward 결합: backward에서 `w̃≥w̃min` leaf 상태를 저장, forward 상태 `s(t)`와 대응시켜 weight `w(t)+w̃(r-t)` codeword 계수 → free distance 갱신.
5. 가지치기: `s0(t)≠0` 또는 `w(t)>dfree+N`이면 branch cut으로 탐색 축소.
6. 복잡도 저감(핵심): PCM `H(D)` 각 행/열의 공통인수 `D^q`로 나누고 특정 열에 `D` 곱하는 memory reducing operation으로 distance profile을 보존한 채 memory `ms`를 대폭 축소(예 `ms=4→2`).
7. 열 shift가 sequence weight를 불변으로 유지하므로 축소 PCM으로 원부호 weight spectrum을 그대로 계산 가능.
8. 학습/최적화: 없음(결정론적 트리탐색+대수적 memory 축소).
9. 결과: memory 21/57/126/134 부호에 적용, `dfree=24` 등 산출. `H2(D)` weight-24 계산 시 확장노드 16,888(forward)로 BEAST(3.5M)·syndrome BEAST(6.5M) 대비 크게 감소, 1.8초 vs 8.1/7.4초.
10. 한계: 복호기/HW 아님, 거리분포 분석 도구, 대상이 LDPC convolutional 부호(block QC-LDPC 아님).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| weight spectrum / free distance 계산 | 대응 없음 | Prime ECC는 거리분포 분석기 없음, 오프라인 부호평가 도구 |
| LDPC convolutional PCM(H(D)) memory 축소 | 대응 없음 | Prime ECC는 block QC-LDPC(PCM_b), convolutional 구조 미지원 |
| QC circulant 기반 부호 구조 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | circulant/QC 개념만 공유, 부호 클래스(convolutional vs block) 상이 |

적용 가치: **낮음** — 복호·부호설계·HW 어느 계층도 아닌 LDPC convolutional 거리분포 계산 도구로 NAND block QC-LDPC에 이식할 요소 없음.
