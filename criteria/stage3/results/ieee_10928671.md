### B. 알고리즘 요약

1. 시스템: ReRAM crossbar 어레이(m×n) 이진 셀, LDPC(CCSDS 표준, N=5120, K=2048, R=1/2, M=1024) 인코딩 후 코드워드를 T개 독립 메모리 어레이로 분산(across-array coding).
2. 문제: selector 다중 고장 시 target cell에 2개 이상 병렬 sneak path(SP)가 동시 발생하는데, 기존 연구는 단일 SP 모델만 가정해 부정확.
3. 핵심 가정/모델: selector 고장은 i.i.d. 확률 `p_f`, SP 조합 타입 `λ=(L;k_r,k_c)`(L=최대 3)에 따라 병렬저항 `a(λ)`이 달라지며, 채널은 `Pr(R(0)|0)=1-λ, Pr(r(λ')|0)=p_λ(λ'), Pr(R(1)|1)=1`의 비대칭 이산+가우시안 결합 모델(식 2).
4. 핵심 기법1: cross-array coding - 길이 N 코드워드를 T등분해 서로 다른 물리 어레이에 배치, 각 서브워드가 독립 ReRAM 채널을 거치게 해 SPI 상관도를 낮추고 diversity gain 획득.
5. 핵심 식: elementary signal estimator(ESE)가 각 array의 SP 발생확률 추정치 `λ̂_t≈n_r(λ)/(n_r(λ)+n_R0)`(식 3)로 각 비트의 LLR을 계산(식 4), 이 LLR을 LDPC soft BP 디코더 입력으로 사용.
6. 확장: SP 타입별 발생확률 `p_λ(λ')`는 위치별 이웃 셀 분포(식 2 하단, [9]의 Table I 참조)로 계산.
7. 부수 디테일: T 증가 시 각 array가 저장 가능한 코드워드 수는 `mnT/N`, array 수 증가에 따라 read/write latency 증가(시뮬레이션 시간 T=1: 21381 → T=16: 30961).
8. 학습/최적화: 별도 학습 없음 - closed-form 채널모델과 ESE 추정만 사용.
9. 결과: array 수 T가 커질수록(2→16) BER이 점진 개선(특히 저잡음에서 뚜렷), 단 디코딩(시뮬레이션) 시간은 소폭 증가.
10. 한계: LDPC는 표준 코드를 그대로 사용(부호설계/디코더 알고리즘 무개선), HW 구현 없음(시뮬뿐), ReRAM 특유의 sneak-path 채널모델에 강하게 종속되어 NAND 채널과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ReRAM sneak-path 채널모델 | `channel.cpp` | NAND과 무관한 ReRAM 특유 채널(비대칭 SP 간섭) - 채널 모듈 위치만 유사, 모델 자체는 대응 없음 |
| ESE soft LLR 추정 | `channel.cpp` `Set_LLR_Th()` | 목적(soft LLR 계산)은 유사하나 SP 확률 추정 로직은 ReRAM 전용이라 직접 이식 불가 |
| LDPC(CCSDS 표준) soft BP 디코딩 | `decoder.cpp` `LDPC_Decoder()` | 외부 표준 코드를 블랙박스로 사용, 논문이 개선하지 않음 |
| cross-array 코드워드 분산 배치 | 대응 없음 | Prime ECC는 단일 다이/플레인 대상 시뮬레이터로 멀티 array 분산 배치 개념 없음 |

> 적용 가치: **낮음** — 논문 핵심은 ReRAM sneak-path 채널모델링과 cross-array 분산 저장 전략이며 LDPC 자체(부호설계·디코더)는 CCSDS 표준을 그대로 사용하는 블랙박스라 NAND binary QC-LDPC ECC 이식 요소가 거의 없음.
