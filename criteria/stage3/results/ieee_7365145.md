### B. 알고리즘 요약

1. 시스템: WDM coherent 광통신, differential QPSK 변조 + LDPC(RS 2% overhead로 error floor 제거) 직렬 결합, block length 19200, rate 0.5/0.85.
2. 문제: differential modulation은 위상/사이클슬립 모호성을 해소하지만 BER을 두 배로 악화시키는 페널티가 있음.
3. 채널 모델: random walk 위상잡음(laser linewidth) + XPM 유래 colored 위상잡음 + AWGN을 합성한 비선형 광채널 모델(`σ_lw`, `σ_corr`).
4. 핵심 기법: MAP SISO differential decoder와 LDPC decoder 사이의 double-iterative(turbo) 루프. LDPC decoder가 `N_inner`회 반복 후 extrinsic 정보를 differential decoder로 전달, 전체 루프를 `N_outer`회 반복.
5. 핵심 식: 표준 BP 메시지 `λ_v→c = λ_chan + Σλ_c'→v`(VN→CN), `λ_c→v = 2·arctanh(Πtanh(λ_v'→c/2))`(CN→VN); min-sum 근사 `λ_c→v ≈ min|λ|·Πsign(λ)`도 병기.
6. 확장: LDPC 파트에 layered decoding(Mansour&Shanbhag 2003) 적용 — CN을 순차 업데이트하며 영향받는 VN을 즉시 갱신(`λ_v→c = θ_v − λ_c→v^prev`), 수렴 속도 향상.
7. 부수 트릭: rate 0.85 QC-LDPC 구조로 부분 병렬화하여 100Gbit/s throughput 요구를 layered decoding과 병행 충족한다고 언급(구체 HW 미제시).
8. 학습/최적화 절차: 없음(고정 코드, 시뮬레이션 기반 성능 비교만).
9. 결과: rate 0.5에서 `N_outer` 20→10(50% 감소), rate 0.85에서 15→10로 동일 BER 성능 유지; turbo 이득 약 1dB(BER 10⁻⁸ 기준); BER 10⁻¹²까지 error floor 미관측(GPGPU 5-bit 양자화 시뮬).
10. 한계: 알고리즘/시뮬레이션 수준(HW 미설계), 광통신 특화 채널·변조 가정, layered decoding 자체는 기존 문헌 기법의 turbo 시스템 적용일 뿐 신규 디코더 알고리즘 기여는 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Layered(순차 CN 업데이트) LDPC 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 이미 layered 유사 구조(min-sum iterative)를 보유해 중복 기여 |
| Min-sum 근사(식 3) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 표준 min-sum, 이미 구현됨과 동일 |
| Turbo loop (differential decoder ↔ LDPC decoder 반복) | 대응 없음 | Prime ECC는 differential 복조/turbo 외부 루프 대상이 아닌 순수 LDPC ECC 엔진 |
| 5-bit GPGPU soft 양자화 시뮬 | `channel.cpp` `Set_LLR_Th()` | 양자화 레벨 설계 참고 가능하나 채널 모델 자체는 광통신 특화라 이식성 낮음 |

적용 가치: **낮음** — NAND와 무관한 광통신 turbo differential 시스템이며, LDPC 디코더 개선 요소(layered scheduling)도 Prime ECC가 이미 보유한 반복 복호 구조와 중복.
