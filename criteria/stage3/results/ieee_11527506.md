### B. 알고리즘 요약

1. 대상: 1KB NAND flash, 확장 BCH(576,545,t=3) 코드, rate 0.892, 전체 길이 9216bit(정보 8224bit).
2. 문제: LDPC는 soft information 필요 → NAND read 지연/HW 복잡도 증가, BCH는 정정능력 한계. oFEC는 soft decoding이라 NAND에 부적합.
3. 채널/모델: 별도 채널 모델 없음, SNR 기반 BLER 시뮬레이션(Fig.5, Fig.12)로 성능 비교.
4. 핵심 기법: OpenROADM oFEC의 tile 구조(B×B 블록, `(R,C,r,c)` 좌표계)를 차용하되, 전 tile을 HIHO(Hard-Input Hard-Output)로 통일해 soft decoding 요구를 제거.
5. 핵심 식: 인터리빙 좌표 변환식 (원문 식 1~3), front/back bit를 `k<N`/`k≥N` 기준으로 나눠 확장 BCH(576,545)로 인코딩.
6. 확장: "Smashed Structure"로 원래 20개 row block 중 빈 블록을 제거해 10개 row block으로 압축, ECC 성능 저하 없이 메모리 절감.
7. 구현 디테일: KES 단계는 PGZ(Peterson-Gorenstein-Zierler) 알고리즘의 inversionless 변형식(식 7) 사용, division 제거로 HW 친화. Chien search는 병렬factor P=115의 Straight-Forward 구조로 5 cycle 내 575근 탐색.
8. 최적화: 32-decoder vs 16-decoder 구성 비교, Area×Cycle 기준 16-decoder가 최적(28.12Gbps).
9. 결과: SNR=5.4dB에서 BLER 4×10^-5(28.12Gbps), LDPC MATE-GDBF(6.39Gbps)/UP-GDBF(4.9Gbps) 대비 처리율 우위. TSMC 40nm, 0.16pJ/bit, 0.027mm2.
10. 한계: BCH 기반 hard-decision 전용이라 soft-info를 활용하는 LDPC 대비 근본적 정정능력 상한이 다르며, HW 결과는 post-layout 시뮬(ASIC 합성)에 한정, 실측 데이터 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 확장 BCH(576,545) 부호구성 (oFEC tile 구조) | 대응 없음 | Prime ECC는 binary QC-LDPC 전용이며 BCH 부호구조 자체를 다루지 않음 |
| HIHO(hard-decision) 전용 디코딩 | `decoder.cpp` `VN_Cal_HD()` | Prime ECC도 HD 지원하나, BCH의 PGZ/Chien search 알고리즘은 min-sum과 무관해 알고리즘 이식 불가 |
| Smashed Structure (메모리 절감 row block 압축) | 대응 없음 | LDPC의 H-matrix/PCM 구조와 무관한 BCH tile 전용 최적화 |
| Chien Search 병렬 아키텍처 (P=115) | 대응 없음 | LDPC CN/VN 처리 구조와 근본적으로 다른 BCH 근탐색 회로 |

> 적용 가치: 낮음 — BCH 기반 부호설계·디코딩 알고리즘 전반이 Prime ECC의 QC-LDPC min-sum 디코더와 계층이 달라 이식 가능한 모듈이 사실상 없음.
