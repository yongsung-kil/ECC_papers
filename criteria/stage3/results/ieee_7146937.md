### B. 알고리즘 요약

1. 대상은 임의 위치 non-zero를 갖는 irregular random parity-check matrix(`M`행 `N`열, 행당 최대 `nmax` non-zero)의 LDPC 부호이며, 예시는 16k열·4k행(rate≈0.75).
2. 문제: irregular random 행렬은 Shannon 한계 근접(0.0045dB loss) 성능을 주지만 read/write·process 단계에서 collision 회피가 필요해 HW가 복잡·저throughput이 됨.
3. 모델: 표준 Min-Sum message passing — CN 메시지 `α_ij = ∏sign(β)·min(|β|)`, VN 메시지 `β_ij = Σα + Z_start`, 최종 `Z_j = Σα_i + Z_start`.
4. 핵심 기법: `β_ij = Z_j − α_ij` 관계로 β를 저장하지 않고 즉석 계산, `Z_temp_j = Z_temp_j + α_ij`로 vertical step을 부분 실행하여 메모리 절감.
5. 핵심 식 의미: (4)식으로 β 메모리를 제거하고 `Z_j`만 저장하면 되므로 index memory + α memory만으로 재구성형 디코더 구성 가능.
6. 재구성: 행당 실제 원소 수 `n_i < nmax`면 특수 index값(`2^bits − 1`)으로 마스킹하고 `var_processing` 신호로 불필요 원소 스킵, FPGA 재프로그래밍 없이 index 값만 로드해 행렬 교체.
7. 구현 디테일: CN 처리를 병렬 대신 shift-register 기반 sequential unit(고정 지연)으로 구성해 자원·복잡도 절감; index memory에 따라 `Z_j`/`Z_temp` 접근은 collision 회피 위해 순차 접근.
8. 병렬화: two-port block-RAM을 2배 클록으로 quad-port로 모사, 4개 decoding unit 연결 시 동일 index 미충돌 조건(row permutation 전처리)으로 throughput 4배.
9. 결과: Altera Stratix IV에서 combinational ALUT 10869, M9K 388블록 사용; 16k×4k, 200MHz, 1 iteration에서 45Mbit/s(10 iter 4.5Mbit/s), 기존 AHA4704(14.5Mbit/s) 대비 우수.
10. 한계: 알고리즘은 표준 Min-Sum이고 error correction은 Eb/N0>4.5dB에서 충분 수준의 정성 결과뿐, BER 곡선·gate count·NAND 실험 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum CN/VN 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 이미 보유한 min-sum과 동일, 중복 |
| index memory 기반 행렬 재구성 | 대응 없음 | Prime ECC는 고정 QC-LDPC(base+circulant)라 임의 위치 index 재구성 불필요 |
| quad-port 메모리 병렬 4-unit | 대응 없음 | Prime ECC 병렬은 z=32 row 단위·GT/HCU 구조, FPGA quad-port 트릭과 무관 |
| soft LLR 양자화 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | 일반 LLR 양자화 언급뿐, NAND soft-read 특화 아님 |

적용 가치: **낮음** — 알고리즘이 교과서 Min-Sum이고 대상 부호가 irregular random(QC-LDPC 아님)이며 FPGA-specific 메모리 트릭이 Prime ECC의 QC-LDPC 병렬 구조와 맞지 않는다.
