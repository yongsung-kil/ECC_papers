### B. 알고리즘 요약 (10줄 내외)

1. LDPC 복호를 `min_x sum_n f_n(x_n) s.t. Hx^T=0` 형태의 combinatorial optimization 문제로 정식화(`f_n(x_n) = -ln p(x_n/y_n)`, LLR과 연결).
2. 기존에는 normalized min-sum이 sum-product의 근사로만 이해되었으나, 그 수학적 원리(왜 최적 코드워드를 찾는가)가 불분명하다는 문제의식.
3. Cooperative optimization: 다변수 목적함수 `E(x)`를 하위문제로 분해해 병렬로 풀고 하위해를 선형결합으로 협력시키는 일반 최적화 기법(로컬 미니마 없음, 지수수렴).
4. Lower bound function `Ψ(x)=Σ Ψ_i(x_i)`를 반복적으로 조여가며(difference equation, 식(1)) `min Ψ(x) = min E(x)`에 도달하면 전역최적.
5. Fragmented unary lower bound(식(3),(4))로 일반화하고, LDPC 분해 `E_nm(x) = f_Xm(X_m) + Σ f_n'(x_n')`를 대입하면 check-node/variable-node update 식(10),(11),(13)이 유도됨.
6. Binary 특수화(식(12),(13))로 정확히 normalized min-sum의 check node 업데이트 규칙(`sgn·min|Z|`)이 도출됨을 증명.
7. Consensus solution(모든 하위문제가 각 변수에 대해 동일한 최적값 도출) = 전역최적이라는 종료조건 정리(Theorem 2.1) 제시.
8. 목적함수 분석: normalized min-sum의 실제 목적함수는 `Σ A_n f_n(x_n)`이며 `A_n = Σ_{m∈M(n)}(d_c(m)-1)`로 variable degree에 비례해 편향(regular LDPC일 때만 무편향).
9. 결과는 수식 유도뿐이며 BER, iteration, gate count 등 수치 결과 없음(시뮬레이션 미수행).
10. 한계: 알고리즘 자체는 새로 제안하지 않고 기존 normalized min-sum의 재해석/일반화 프레임워크(GF(q), 임의 제약으로 확장 가능성)만 제시, 실험적 검증 전무.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Normalized min-sum check node 업데이트 재유도 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 구현된 min-sum과 수학적으로 동일한 결과이며 새 기법 아님 |
| Cooperative optimization 일반 프레임워크(GF(q), 임의 제약) | 대응 없음 | Prime ECC는 binary LDPC 전용, non-binary/일반화 프레임워크는 범위 밖 |
| Variable degree 기반 목적함수 편향 분석 | `decoder.cpp` `LDPC_Decoder()` | 이론적 관찰일 뿐 코드 변경 여지 없음(규칙 vs 불규칙 코드 특성 설명) |

적용 가치: **낮음** — 이미 보유한 normalized min-sum을 다른 수학적 틀로 재유도만 할 뿐, 새로운 알고리즘·수치 개선·HW 관점이 전혀 없어 이식할 요소가 없음.
