### B. 알고리즘 요약

1. 시스템: outer LDPC(또는 convolutional) 부호 + inner marker code로 구성된 concatenated code, IDS(삽입pI/삭제pD/치환pS) 채널을 통해 다중 사본(M개) 전송 (DNA 저장 응용).
2. 문제: inner code는 통상 BCJR(MAP)로 복호하는데, M개 사본을 joint decoding하면 복잡도가 `O(nin(κδ)^M)`로 지수적으로 증가해 M>4에서 사실상 불가능.
3. 채널 모델: drift 변수 `Di`(누적 삽입-삭제 차)로 IDS 채널의 Markov 성질을 유지, `F(ξ,ri)`로 substitution 확률 반영.
4. 핵심 기법 1단: BCJRFormer — decoder-only transformer, BCJR의 forward-backward 확률식(`αi`,`βi`)에서 유도한 입력 토큰 `Yi,j,ξ = P(x=ξ)F(ξ,ri+dj)`를 사용해 inner code를 신경망으로 복호.
5. 핵심 확장: M개 사본 joint decoding은 토큰을 시퀀스 위치별로 concat하고 두 번째 positional embedding으로 처리, 최종 출력은 M개 예측의 평균(식 7)으로 계산 — 복잡도는 M에 대해 이차(`O(M²nin²Ndecdk)`)로 감소.
6. 핵심 기법 2단: outer 복호기로 기존 BP 대신 ECCT(Error Correcting Code Transformer)를 사용, 입력은 LLR 크기 `|xϕ|`와 신드롬 `syn(bin(xϕ))`의 결합.
7. 부수 디테일: outer decoding 시 noise가 codeword-dependent이므로(BISO 채널과 달리) all-zero codeword만으로 학습 불가, pseudorandom codeword로 학습.
8. 학습 절차: BCJRFormer/ECCT 모두 BCE loss로 학습(Adam, cosine decay), BCJRFormer는 hidden dim 96/6-layer/~700K 파라미터, ECCT는 hidden dim 128/8-layer/~1.6M 파라미터.
9. 결과: BCJRFormer는 단일 전송에서 BCJR과 유사 BER 달성, M>3에서도 BER 지속 감소(BCJR은 계산 불가); BCJRFormer+ECCT 조합이 BCJR+BP 대비 저노이즈 영역에서 더 낮은 BER.
10. 한계: 순수 시뮬레이션 결과뿐 HW 미설계, IDS 채널(동기화 오류) 전용 구조이며 전체 파이프라인이 신경망 기반이라 기존 HW 디코더 아키텍처로의 직접 이식 불가.
