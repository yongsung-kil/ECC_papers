### B. 알고리즘 요약

1. 시스템은 blockchain Coded Merkle Tree(CMT): 각 layer를 LDPC로 부호화, light node가 random sampling으로 data availability(DA) 공격 탐지. rate-1/2, `dv=4` regular LDPC, `n=100` 예시.
2. 문제: 악의적 노드가 LDPC의 최소 stopping set을 숨기면 hash-aware peeling decoder가 복호 실패 → light node 탐지 실패. 실패확률은 stopping ratio(최소 stopping set 크기/`n`)에 의존.
3. 모델: light node가 `s` 샘플 요청, 악의노드가 크기 `µ` stopping set을 랜덤 은닉 시 실패확률 `pf(s,µ)`. 기존[1]은 random ensemble+random sampling `(1-µ/n)^s`.
4. 핵심 아이디어: stopping set을 없애기보다 소수 VN 집합에 "집중"시켜, light node가 그 소집합만 greedy 샘플링하면 다수 stopping set을 커버.
5. 이론 근거: `dv`-reg LDPC의 stopping set은 cycle로 구성. `Σ(dv,g)`(길이≥g cycle만인 최소 stopping set 무게), 예 `Σ(4,10)≥17`. g-sample set 샘플 시 `µ ≤ Σ(dv,g)-1`이면 `Pf=0`(Lemma 1).
6. 핵심 기법: EC-PEG 알고리즘 — PEG로 edge 추가 시 cycle 생성되면(`F=1`, `g'<gc`) 후보 CN 중 joint normalized cycle count 분포 `α_gc`의 entropy `H(α_gc)`를 최소화하는 CN 선택.
7. 구현 디테일: `λ_g'`(VN-g'-cycle count), `α_g'`(정규화), `α_gc`(joint) 유지. `gc=10` 복잡도 제약. considered cycle(`g'`)만 카운트하고 큰 cycle은 무시(Remark 2).
8. 샘플링 전략: greedy sample set `Sgreedy`를 `Sa_gc`에서 s개 greedy 선택, 확률 1로 고정 샘플링. `pf(s,µ)=1-τ(Sgreedy,µ)`.
9. 결과(`n=100`): EC-PEG가 원 PEG보다 `ζ6/ζ8` 분포를 같은 VN에 집중, greedy 10-sample set `|Sa10|` 감소. random ensemble(β*=0.064)+random sampling 대비 `pf(s,µ)` 대폭 감소(deterministic 부호+greedy 샘플링+집중 3중 이득).
10. 한계: 시뮬레이션만(HW 미설계), erasure peeling/보안 목적, soft-read 무관, 강한(전략 인지) adversary 미대응(future work).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG 기반 LDPC 부호설계 (EC-PEG) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 설계 계층 접점이나 QC-LDPC 고정 구조와 무관, stopping set 집중 목적이 상반 |
| stopping set / cycle 구조 | 대응 없음 | Prime ECC는 min-sum BP 복호이며 stopping set 집중은 error-floor 개선과 반대 방향 |
| peeling decoder (erasure) | 대응 없음 | Prime ECC는 soft/hard min-sum 디코더, erasure peeling 미사용 |

적용 가치: 낮음 — blockchain DA 공격 방어라는 보안 목적의 erasure 부호설계로, stopping set을 집중시키는 방향이 NAND ECC의 error-floor 개선과 상반되고 non-QC deterministic PEG 구성이라 Prime ECC와 정합하지 않는다.
