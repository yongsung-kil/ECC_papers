### B. 알고리즘 요약

1. 5G NR QC-LDPC 복호를 NVIDIA GPU에서 가속. 부호는 base matrix `mb×nb`(BG1 46×68, BG2 42×52) + lifting factor `Z`(2~384) 구조.
2. 문제: 5G LDPC는 rate/block/use-case 구성이 방대(51개 Z, 2 BG)해 FPGA/ASIC로 전부 지원하기 어려움 → SW(GPU)의 유연성으로 해결.
3. 복호 알고리즘은 scaled min-sum(MSA), scale factor `γ`=0.75. CN 메시지 `Rmn = γ·(∏sign)·min|Qmn|` (식 2), VN 갱신 `Ln = Ln + Σ(Rmn 변화)` (식 3).
4. GPU 매핑: CN 커널·VN 커널 2개. 각 thread가 1개 CN 또는 VN 담당, block은 base graph의 row/col 처리.
5. 병렬화: `α`개 CW를 macro-codeword(MCW)로 묶어 같은 block, `β`개 MCW를 batch 전송 → `α·β` CW 동시 복호. 제약 `αZ≤1024`.
6. two-min 알고리즘으로 CN에서 min1/min2 추적, sign product 계산(Kernel 1).
7. 최적화: word length를 8-bit char로 축소(전송·global memory 접근 4x 절감), 최종 hard-decision bit-packing 커널, CUDA stream `η`개로 전송/연산 오버랩, H를 row/col-major compact로 constant memory 캐싱.
8. latency/throughput은 `α,β,η` 조정으로 트레이드오프(URLLC vs eMBB 타깃).
9. 결과: TITAN RTX에서 char 기준 3964Mbps throughput, 최저 87us latency(전송 포함), iteration당 13.7us. 8-bit는 BER 미미하게 열화.
10. 한계: 순수 GPU SW 구현, 알고리즘 신규성 없음, NAND용 HW 디코더 설계 아님, error-floor/정정능력 개선 없음.
