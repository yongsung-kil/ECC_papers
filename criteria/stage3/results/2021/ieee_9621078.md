### B. 알고리즘 요약 (10줄 내외)

1. 시스템: AWGN 채널, 3GPP NR 기반 `(128,40)` polar code, rate `R=0.31`, generator `G_N=F^{⊗n}` (`F=[[1,0],[1,1]]`).
2. 문제: polar code의 BP 디코딩 latency는 iteration 수에 비례 — 목표 error rate를 더 적은 iteration으로 달성해 수렴 속도를 높이려 함.
3. 모델: factor graph 상 left/right LLR 메시지 전파, 초기 채널 LLR `L_{n+1,i}=2y_i/σ²`, frozen bit는 `R_{1,i}=∞`.
4. 핵심 관찰: processing unit에서 `L_out,2`는 `L_in,2`가 box-plus(`g`) 없이 직접 전달되어 `L_in,1`보다 기여가 큼 → 마지막 bit `(n+1,N)`의 신뢰도가 LLR 전파에 가장 영향.
5. 핵심 식: `g(x,y)=ln(cosh((x+y)/2)/cosh((x-y)/2))` — box-plus 연산으로 약한 방향 메시지 희석을 설명.
6. 핵심 기법: `n_F`개 최저 index coded bit을 펑처링(LLR=0)하고, 최고 index bit `N`을 `n_F`번 반복 확장(LLR이 `n_F`배 증가).
7. 구현 디테일: 확장은 단순 반복이라 추가 복잡도 없음 (펑처+repetition만 사용).
8. 학습/최적화: 별도 학습 없음 — 분석 기반 정적 설계 규칙.
9. 결과: `Imax=2`, Eb/N0 7dB에서 `n_F=5`가 error rate 50% 감소; iteration 증가 시 최적 `n_F`는 작아짐(5→1).
10. 한계: 시뮬만 (HW 미설계), polar code 전용, early-termination BP 결합은 미래 과제로만 언급.
