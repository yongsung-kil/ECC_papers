### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 5G NR QC-LDPC (base graph 1, lift factor `128`), AWGN 채널, BPSK, `Imax=10`, N-MSA(normalized min-sum) layered 복호.
2. 문제: 기존 조기종료(SDC 전체 syndrome / HDA sign-check)는 전 layer 연산으로 비용 크거나 정확도 미보장으로 성능 저하.
3. 모델: base graph는 core check(`mcore=4`) + extension check로 구성, column은 systematic + core parity + extension parity; extension check는 1의 분포가 core보다 희소.
4. 핵심 기법: layer 복호 순서를 extension check 먼저 → core check 나중으로 두고, 조기종료 판정은 **core check 파트에만** 국한.
5. 핵심 판정: 각 core layer에서 `HDcmp=(HDold != HDnew)` 누적(`HDacc`) + partial syndrome(`Syndromacc`) 계산.
6. 종료 조건: `PartialHDcmp==0` && `Partialsyndrom==0`이면 `ETcond=0` → iterative decoding 종료 (식 7~8).
7. N-MSA CN 업데이트: `R_cv = a · ∏sign(Q_v'c) · min|Q_v'c|` (normalized factor `a`).
8. 학습/최적화: 별도 학습 없음 — 구조 기반 정적 규칙.
9. 결과: 기존 syndrome check 대비 평균 iteration `18.7%` 감소(-3.4dB Es/No), BLER `0.2dB` 개선.
10. 한계: 시뮬만(HW 미설계), gate count/throughput 미제시, 5G NR base graph core/extension 구조에 강하게 의존.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| core-check 국한 조기종료 판정 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | Prime ECC의 partial/full CRC 조기종료 대비 대안 판정 로직 |
| partial syndrome + sign-change 누적 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 종료 판정을 루프 내 부분집합 layer로 삽입 |
| layered N-MSA CN 업데이트 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 연산은 이미 보유(중복) |
| layer 복호 순서 재배치 | ecc_top.cpp Load_PCM() | base graph의 core/extension 구분을 PCM에서 정의해야 함 |
```

대상 decoder이므로 C섹션 채움. **적용 가치: 중간** — Prime ECC는 이미 partial-CRC 조기종료를 보유(중복 소지)하나, syndrome+sign 기반 부분 layer 판정은 대안 축으로 검토 가치가 있고, 5G core/extension 구조 의존이 이식 부담.
