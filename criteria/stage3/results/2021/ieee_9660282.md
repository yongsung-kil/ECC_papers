> 총평: NAND SSD read latency를 refresh 정책으로 줄이는 시스템 논문으로 LDPC 부호/디코더 자체는 손대지 않아 이식성 하.

### B. 알고리즘 요약

1. 시스템: NAND(MLC/TLC/3D) SSD, LDPC ECC, read level 1~7 soft decoding, DiskSim+MS SSD 시뮬(8ch×8chip×4plane×2048block, block당 64×8KB page).
2. 문제: 신뢰도 저하 시 LDPC가 여러 번 재읽기(soft decoding)로 read latency 급증 → 응답시간 악화.
3. 모델: retention/program/read error로 BER 상승, read level↑ → 복호정확도↑·지연↑ (Table1: first read 50us + extra read 14us 등).
4. 핵심기법1: "high LDPC delay block(HLDB)" 정의 — 블록 내 read level 3(또는 5) 이상 page 수 `N1`/`N2`가 임계(=page/block의 절반) 초과 시 고지연 블록.
5. 식 의미: level 3/5에서 응답지연이 계단식 급증(실험상 waterfall 아닌 지연 곡선)하므로 그 지점을 refresh 트리거로 사용.
6. 핵심기법2: 자가적응 — P/E cycle < 수명/2(초기)면 `N1`(level 3) 기준, 이후(후기, P/E≥10⁴)면 `N2`(level 5) 기준으로 판정해 refresh 빈도·수명손상 절감.
7. 부수: hash table(key=level≥3/5 page 수)로 블록 정보 저장 → 주기적 refresh 시 조회·계산 오버헤드 감소.
8. 학습/최적화: 없음(휴리스틱 임계 N=page/block/2).
9. 결과: 평균 응답시간 baseline 대비 6~40% 감소(read비율 높을수록 큼, Websearch3 47%); 자가적응이 refresh 횟수 평균 41%·peak 70% 감소로 수명 보호.
10. 한계: LDPC 디코더/부호 알고리즘 자체는 미개입(refresh 정책만), HW 미설계, 시뮬만, read level을 지연 대용지표로 가정.
