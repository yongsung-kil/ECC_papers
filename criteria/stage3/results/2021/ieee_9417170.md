### B. 알고리즘 요약

1. 대상은 LEDAcrypt(NIST PQC) 코드기반 암호의 복호단; 비밀키=희소 QC 패리티검사행렬 `H`, `Q` (`L=HQ`), Niederreiter 프레임.
2. 부호는 QC-LDPC/QC-MDPC, circulant 블록(size `p`, 소수, 수천~1.5만), `n=p·n0`; rate/length는 보안파라미터 종속(본문 표는 이미지, 서술값 없음→미상).
3. 풀려는 문제: 암호 복호는 syndrome decoding problem(SDP) 해 e(무게 t) 찾기 = NP-hard, 저지연·저면적 HW 필요(에러율 2⁻¹²⁸급).
4. 핵심 기법 1단: `Q-decoder` = bit-flipping 변형. 매 iter `σ=sH`(정수), `ρ=σQ`(정수 UPC 카운터), 임계 `b=f(s)`(LUT) 초과 위치 플립.
5. `L=HQ` 인수분해를 이용해 카운터 계산을 2단계로 나눠 일반 MDPC 대비 약 4배 빠름(`m⁻¹+d_H⁻¹` 절감).
6. 핵심 기법 2단: 곱셈이 지배적 → Vector-by-Sparse-Circulant(`VbSC`)로 Schoolbook을 희소화(복잡도 `p²→p·d`), 입력도 희소면 `SVbSC`(`d_v·d_A`).
7. 구현 디테일: 벡터를 `nb` 워드로 분할 `Mv/Mr`, 회전을 `log2(nb)+1`단 2입력 MUX cascade(collapse unit)로 → 면적이 병렬도에 선형(기존 지수/초선형 대비).
8. 스케줄: Data Path(Syndrome/Correlation/Syndrome Update/Error Search/Threshold) + Control + Memory, 병렬도 8~256 설계시 선택; v1은 SVbSC, v2는 VbSC/SVbSC 혼용으로 고병렬 가속.
9. 결과: Artix-7 총 0.6ms(LUT15%/FF3%/BRAM30%), STM FDSOI 28nm 0.15ms·0.09mm², UMC 65nm P&R 0.3ms·0.42mm²·≤10.5mW; Latency×LUT 지표 최소.
10. 한계: 저전력 미최적화, 채널 ECC 성능(BER/waterfall) 무관, 파라미터는 NIST 2라운드용(LEDAcrypt는 weak-key로 3라운드 탈락).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Q-decoder bit-flipping 루프 | 대응 없음 | Prime는 soft min-sum message-passing, F2 bit-flipping+UPC와 알고리즘 상이 |
| 임계 LUT f(s) 기반 플립결정 | 대응 없음 | Prime의 적응형 LLR 테이블/CRC 조기종료와 성격 다름 |
| VbSC/SVbSC 희소 순환행렬 곱셈기 | encoder.cpp Generate_PCM_encoding(), ecc_top.cpp Load_PCM() | QC circulant 곱셈 HW 기법으로 인코딩/syndrome 계산에 부분 참고 여지 |
| syndrome 계산/갱신 HW | 대응 없음 | Prime는 디코더 내부 parity check로 처리, 별도 SDP 곱셈 구조 아님 |
```

적용 가치: **낮음** — 코드기반 PQC 복호(SDP)라 채널 LDPC 디코더와 목적·알고리즘(hard bit-flipping vs soft min-sum)이 달라 이식 대상 아님. VbSC 희소 순환곱셈기 HW만 인코더/syndrome 면적최적화 참고용 여지.
