### B. 알고리즘 요약

1. 시스템: 5G NR LDPC, `pyldpc` 생성행렬 `wr=8`/`wc=2`, rate `0.75`, message `k=433`, codeword `n=576`, BPSK 변조.
2. 문제: 실제 채널의 상관잡음(oversampling/filtering/multi-user)을 고정 모델 기반 soft-decision BP가 잘 처리 못 함.
3. 채널 모델: correlation parameter `η=0.8`인 Gaussian correlated noise, `y = s + n`.
4. 핵심 기법 1단: 표준 BP(Sum-Product/Min-Sum)와 CNN denoise 블록을 concatenate한 Iterative BP-CNN. CNN이 잔차잡음을 추정해 BP LLR을 보정.
5. 핵심 식: CNN loss `Loss = r²/N + λ(S² + (1/4)(C-3)²)` — Jarque-Bera 정규성 검정(왜도 `S`, 첨도 `C`)으로 잔차를 Gaussian화.
6. 핵심 기법 2단: CNN 입력단에 Batch Normalization 층 추가 → 입력 스케일 표준화 + vanishing gradient 완화.
7. 구현 디테일: CNN 구조 `{4; 9,3,3,15; 64,32,16,1}`, Adam optimizer, offset min-sum(learnable `β`)도 언급.
8. 학습: mini-batch SGD/Adam으로 CNN 가중치 학습, 채널잡음 심볼을 label로 사용.
9. 결과: 학습 iteration 30k→60k 증가 시 BP-CNN+BN이 Sum-Product/Min-Sum 대비 BER vs SNR 성능 향상(수치 곡선만, 정량값 본문 미기재).
10. 한계: 딥러닝 기반·상관잡음 채널 특화, HW 미설계, gate/throughput 없음, BER 개선 정량치 없음.
