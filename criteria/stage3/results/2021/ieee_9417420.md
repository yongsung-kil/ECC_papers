### B. 알고리즘 요약

1. 대상: 3GPP LTE turbo 및 NR 5G LDPC 디코더, MMSE 등화 후 반복 복호(`IMAX`=12 turbo, 15 LDPC), fading 채널(EPA-5Hz, TDL-A).
2. 문제: 반복 복호의 전력·지연을 줄이는 조기종료(ET)에서 기존 PCE/IHDA는 성능-복잡도 tradeoff 한계, ML 기반 ET는 미탐색.
3. 모델: 각 반복에서 비트 LLR을 `[-M,M]`(M=1000)로 클리핑 후 `K`=20 bin 히스토그램을 만들고 codeword 길이 `N`으로 정규화 → 길이 무관 입력.
4. 핵심 기법: fully-connected DNN (입력 K, hidden `L1,L2,L3`, LReLU α=0.1, 출력 3-class softmax)이 정규화 히스토그램을 받아 CRC 상태를 예측.
5. 라벨 정의: `IMAX/2`와 `IMAX` 시점 CRC 조합으로 3 class(`z0` 성공-성공, `z1` 실패-실패, `z2` 불확실) 구성.
6. 출력 해석: class0=CRC-success(하드비트 변환 후 CRC 실검사, 통과 시 종료), class1=CRC-failure(단 `IMAX/2` 이후에만 종료), class2=계속 진행(Algorithm 1).
7. 학습: 50% BLER 조건 4×10^4 codeword, QPSK/16-QAM/64-QAM, `IMAX/2` 반복 데이터로 학습, Nadam(lr=0.001), epoch 2000, batch 1000.
8. 파라미터: turbo (K,L1,L2,L3)=(20,40,20,10), LDPC=(20,40,30,15).
9. 결과: LDPC ANI가 PCE 대비 30~36%, IHDA 대비 23~32% 감소(10% BLER), BLER 손실 거의 없음. 추가 비용은 LDPC 1반복의 24.27%.
10. 한계: HW 미설계, 무선 fading·고차 QAM 전제, LDPC 부호 자체 정정능력은 개선하지 않음(순수 ET).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| DNN 기반 조기종료 판정 | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 조기종료 지점 판단을 대체/보완하는 자리이나 우리는 CRC 규칙 기반(ML 아님) |
| 반복별 LLR 히스토그램 입력 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | LLR 상태 수집 위치는 동일하나 히스토그램·DNN 추론 신설 필요 |
| soft LLR 클리핑/양자화 | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | LLR 범위 제한·정규화 아이디어가 LLR 테이블과 접점 |
| DNN 추론 자체 | 대응 없음 | NN 디코더/추론 경로 없음 |
```

적용 가치: 낮음 — avg iteration 감소 목표는 부합하나 조기종료는 이미 Partial/Full CRC로 보유하고, DNN 추론·무선 soft-LLR 전제가 NAND HW 파이프라인과 정합성이 떨어져 재학습·검증 부담 대비 이득이 불확실하다.
