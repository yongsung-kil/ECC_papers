### B. 알고리즘 요약

1. 시스템은 TLC 3D NAND(블록당 32 wordline, page 16KB), ECC는 1KB 프레임을 rate 0.9 LDPC로 부호화·min-sum 복호(HD 최대 iter 50, SD 250).
2. 문제: 컨트롤러가 retention time `t`와 block 공정편차 `s`를 알 수 없어 복호모드를 잘못 골라 불필요한 추가센싱·지연이 발생.
3. 채널모델: threshold 전압을 상태별 Gaussian으로, block RBER를 `e_b=min(a·10^{s·N},1)`, retention을 `log(1+t)` 스케일로 모델링(공정편차 `s`는 bounded Gaussian).
4. 핵심 착안: 접근불가한 `t,s`를 대체할 관측량으로 on-cell ratio `r_i`(기준전압 `V_i`에서 켜진 셀 비율)를 사용, 특히 `r_6,r_7`이 retention·편차에 민감.
5. 라벨링: 각 메모리 상태에서 모드 `d`(1=HD,2=2SD,3=3SD)로 시작할 때 평균 read latency `E[lat_d]`를 PER 연쇄식(식 8~10)으로 계산.
6. 최적모드 `y=argmin_d E[lat_d]`로 각 feature vector를 라벨링(입력 feature: `N,w,p` + 대체정보 on-cell ratio).
7. 분류기: decision tree(Gini impurity), Bayesian optimization 100회로 하이퍼파라미터 튜닝(max depth 12, min leaf 42), 5-fold CV, 80/20 split.
8. 추론 지연은 거의 0으로 가정, 1 sensing=100/7 µs, 복호 0.5 µs/iter로 latency 산정.
9. 결과: 최적모드 예측 정확도 평균 94% 이상(class2/3는 ~100%, LSB 100%), Static 대비 평균 read latency 약 53~68%(전체 약 61%) 개선, OPT(genie) 근접.
10. 한계: LDPC 디코더 자체 미개선(모드 선택기), HW 미설계·시뮬만, 특정 3D TLC NAND 통계모델에 의존.
