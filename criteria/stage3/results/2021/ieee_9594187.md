### B. 알고리즘 요약

1. 대상: AWGN/BPSK 채널, 양자화 offset min-sum(OMS) 디코더 (`β` offset, LLR을 `[-Q,Q]`로 saturation, 예제에서 C2V 5bit·AP 8bit).
2. 풀려는 문제: LDPC 디코더 에너지를 회로 구현 전(합성/레이아웃 전)에 정확히 예측할 분석적 모델 부재 — 기존은 post-layout 시뮬 의존.
3. 핵심 관찰: 디코더 에너지의 대부분이 flip-flop(FF) 기반 register/memory이며, FF 에너지는 입력 비트의 **스위칭 활동**에 지배된다.
4. 핵심 기법 1단: clock-gated FF 에너지 모델 `E_FF = EC + Esc·Ps`, 스위칭확률 `Ps = P+(1-P-)+(1-P+)P-`, CGU/enable을 포함해 register `E_W`·SCM memory `E_M`로 확장 (가정 A1~A3, 특히 A3=클록 전후 비트 독립).
5. 신호 비트확률은 PMF 벡터 `P_S`에 변환행렬 `T`(sign-magnitude)/`C`(2's complement)를 곱해 유도.
6. 핵심 기법 2단: 신호 PMF를 DE로 예측하되, OMS의 "2-min search tree"(min1/min2) 신호는 표준 DE로 안 되므로 **order-statistic CDF**(식 9,10)로 max1/max2 분포를 유도.
7. 부수 트릭: all-zero codeword 가정이 스위칭 활동엔 부적합 → parity 제약 하 랜덤비트(`⊕bj'=0`) 조건부 PMF로 재정의 `P|⊕(k)=½(P(k)+P(-k))` (식 11,12).
8. 파라미터: TSMC 65nm GP에서 Cadence Genus로 `Ece,EC,Esc` 등 추출 (Table I, 예: `EC=5fJ`, `Esc=5.24fJ`).
9. 결과: `∆-update` OMS CNPE(`DCMAX=26`)에서 모델 vs post-synth 오차 register 2~8%(glitch 제거 시 1% 이내), C2V memory 4% 이내, pipeline depth 4→8 시 register 에너지 최대 32% 증가.
10. 한계: FF/SCM(순차소자)만 모델링(조합논리·glitch 제외), CNPE 단위 검증뿐 전체 디코더·다중 부호 미검증, 에너지 예측이지 성능/throughput 개선 아님.
