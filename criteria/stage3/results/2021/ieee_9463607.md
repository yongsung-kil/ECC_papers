### B. 알고리즘 요약

1. 대상: 5G 3GPP TS38.212 QC-LDPC. base matrix `HBG`를 sub-block 크기 `N`(384/352/320 등) 순환치환행렬로 확장, code block 최대 25344 bit.
2. 문제: 긴 sub-block의 순환 시프트를 FPGA shift register로 하면 `O(N)`이라 시프트량이 클수록 느리고, 다중 code length 시 재구성이 어렵다.
3. 인코딩 원리: `Hc·c = Hw·w`를 sub-block 순환시프트+row 누적으로 분해, `Hw`가 이중대각(행당 순환행렬 1~2개)이라 순차적으로 parity subblock `w` 산출.
4. 핵심 기법 — BP-CCS: 출력 각 바이트는 입력의 인접 2바이트에만 의존. 시프트값 `S`를 고6비트 `SH=⌊S/8⌋`(어느 두 바이트 접합)와 저3비트 `SL=S mod 8`(접합어 내 시프트량)로 분리.
5. 바이트 주소: `PL(i)=(SH+i) mod Nbyte`, `PH(i)=(PL(i)+1) mod Nbyte`. IND 명령어로 두 바이트 추출.
6. Splicing shift: `r(i)=(BH(i)‖BL(i))≪SL`의 하위 8비트. 시간복잡도 `O(N)→O(1)`.
7. HW: 자체 DSP UCP(512bit SIMD, CGRA 파이프라인, crossbar). Mod Add/IND 기존 명령어 활용 + 전용 Splicing Shift 명령어 신설.
8. 부하 균형: 명령어를 IMA0~3/BIU0~3/SHU0~3 유닛에 분배(cascade diagram)해 유닛 간 동기 대기 최소화.
9. 결과: TSMC 16nm 테이프아웃 칩(1.2GHz) 실측. 4시나리오 인코딩 0.0125~0.0143ms로 5G 규정(0.5ms)의 약 1/40~1/50. sub-block 전환 오버헤드 0.29%.
10. 한계: 인코더 전용, 5G BG 특화, UCP 명령어셋 의존(타 SIMD 이식 가능하다고만 주장). 복호는 future work.

| 부호 PCM(HBG) 로드/저장 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | sparse base matrix 저장 개념은 유사하나 5G BG 특화 |
