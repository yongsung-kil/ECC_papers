### B. 알고리즘 요약

1. 대상 부호는 CCSDS 근지구용 `(8176,7154)` QC-LDPC, rate≈0.875, PCM은 2×16 블록이고 각 블록 `Ai,j`는 두 개의 511×511 cyclic-shift identity의 중첩(row weight 32, col weight 4).
2. 복호는 attenuated Min-Sum(AMSA), 감쇠계수 `α=0.75`, LLR 양자화 `Q=6bit`(1 sign/3 int/2 frac).
3. 풀려는 문제: partially-parallel 디코더에서 인접 행을 동시 계산하면 BRAM 다중 위치 접근으로 memory access conflict가 생겨 병목이 된다.
4. 핵심 기법 1: 고 circulant-weight 부분행렬(두 shift identity 중첩)을 각각 분리 저장하여 병렬도를 높이고, `P=7` 인접 행(열)을 한 메모리 엔트리에 packing해 한 클록에 P개 LLR 접근.
5. 핵심 기법 2: 저복잡도 data aligner — 각 RAM offset을 `mod(bi,j, P)`로 사전계산해 LUT에 저장, 이전 클록 데이터와 splice 후 output window로 인접 P LLR 정렬(대형 switching network 대비 자원 절감).
6. 처리유닛: V2C는 `D+1` 입력 cascaded adder, C2V는 TCSM→FTMFM(recursive tree 최소2값)→AMM(α 곱, shift/sub)→SU(sign XOR)→SMTC.
7. 메모리 구성: CMB 16 RAM, IMB 64 RAM(중첩 identity 분리로 2배), HDMB 16 RAM, depth 511/7≈73, 총 253kbit.
8. throughput 식 `T = fclk·k / (tinit + (tc2v+tv2c)·Imax)`, `tinit=75, tc2v=90, tv2c=78` 클록.
9. 결과: Virtex-5 XC5VLX330, `fclk=250MHz`, 10 iteration에서 `1.02Gb/s`, LUT 46294/FF 39103/BRAM 88.
10. 한계: 특정 CCSDS 부호·511 circulant에 특화된 HW 최적화이며, 정정능력/error-floor 개선은 없음(순수 architecture-level).
