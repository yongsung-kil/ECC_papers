### B. 알고리즘 요약

1. 시스템: CCSDS Telecommand LDPC(QC 구성, systematic, irregular, half-rate `R=1/2`, transparent), `k=64/256`, `n=128/512`, BPSK/AWGN, 4-bit 양자화.
2. 문제: CCSDS 정의 LDPC 복호 알고리즘 분석 문헌이 희소 → 기존 알고리즘들의 coding gain을 정리.
3. 모델: base matrix를 cyclic-right-shift + XOR로 확장해 H 구성, `G=(P⁻¹Q)ᵀ`로 생성행렬 도출, `S=G·Hᵀ=0` 검증.
4. 핵심기법: soft(BP, ABP/Min*BP, MSA, NMSA)와 hard(BF, ABTBF, TABTBF) 복호기를 동일 부호에 구현·BER 비교.
5. 핵심식: CN은 `Ch_k=(∏sgn(α))·Φ(ΣΦ(|α|))` (식2), min-sum은 max-log 근사 `Φ(ΣΦ(v))≈min|v|` (식4~5), NMSA는 상수 c 곱셈 보정.
6. 관찰1: 양자화 입력에서 ABP가 CN 입력이 모두 0일 때 NaN/최대치 폭주 → storage 변수를 `2^(n-1)-1`로 초기화 권고(그래도 비수렴 위험).
7. 관찰2: CCSDS H는 한 컬럼이 두 CN 부분집합과 교차 → **layered scheduling 불가**, informed dynamic은 HW 부담 → flooded scheduling 채택.
8. 최적화: 최대 iteration 10/50/100 변화 시 Eb/N0 이득(식별) — BP 0.38dB, ABP 0.32dB, MSA 0.7dB, NMSA 0.2dB.
9. 결과: floating soft coding gain 7~7.7dB(NMSA 7.7), MSA 4-bit 양자화 6dB, hard(ABTBF/TABTBF) 4.1dB.
10. 한계: 신규 알고리즘 없음(기존 비교), MATLAB 시뮬만·HW 미설계, CCSDS 단소블록 특화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum / Normalized Min-Sum CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유한 min-sum과 중복 |
| ABP 양자화 비수렴(NaN) 및 storage 초기화 | decoder.cpp VN_Cal_HD() 등 양자화 처리 | 양자화 코너케이스 주의점 참고 수준 |
| flooded vs layered 스케줄 제약 | decoder.cpp LDPC_Decoder() 이터레이션 스케줄 | 부호구조-스케줄 정합 참고, CCSDS 특화 |
| H-matrix/QC 구조 로드 | ecc_top.cpp Load_PCM() | 부호 구조 상이(CCSDS 단소블록) |
```

적용 가치: **낮음** — min-sum/NMSA/BF는 이미 보유 기법과 중복이고, 신규 알고리즘 없이 CCSDS 단소블록 부호에 대한 기존 복호기 비교·스케줄 제약 분석이라 NAND 고rate QC-LDPC 이식 가치가 낮다. ABP 양자화 비수렴 회피용 초기값 팁 정도만 참고.
