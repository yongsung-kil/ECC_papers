### B. 알고리즘 요약

1. 대상은 **nested** array-based(AB) SC-LDPC 부호: 여러 sub-code `C_i`가 global code `C`에 중첩(`C_i ⊂ C`), 다중단말 무선 통신용, sub-code별 열무게 `ω_i=3,4,5`.
2. 문제: nested 구조는 sub-code PCM이 부분 중첩되어 한 sub-code 최적화가 다른 sub-code에 제약을 걸며, 유한길이에서 지배적 absorbing set(ABS)이 error floor를 유발.
3. 핵심 관찰(Remark 2): `γ=3,4,5`의 지배적 ABS((3,3),(4,4),(5,4),(6,4) 등)는 모두 6-cycle을 포함 → 6-cycle 최소화가 곧 ABS 최소화.
4. 핵심 기법: **Adapted Line-Counting(ALC)** — 임의 열무게-3 부분행렬에서 6-cycle을 `(c1,c2)` 평면의 정수점 카운팅으로 다항시간 열거(Lemma 1 식 14, Prop.1 식 15).
5. 6-cycle 존재조건을 exponent/circulant 관계식 `j3=(q2 j2 - q3 j1 - λp)/(q2-q3)`으로 유도, 열거 복잡도를 기존 `O(gγ²p³)`에서 라인계산 `O(p²)` 수준으로 축소.
6. 최적화 전략: edge-spreading 행렬 `B_{i,m}`을 순차/제약 최적화(Method 1: global 먼저, Method 2: nested 먼저), 최적화 순서 `t`가 harmful ABS 수를 좌우.
7. guided search: forward-search + back-tracking으로 6-cycle 수 `ρ`를 최소화(max iter `10^4`).
8. **Terminal lift**(lifting factor `J`)로 QC-SC-LDPC화하여 잔여 6-cycle을 추가 제거(circulant 소진탐색).
9. 결과: 공간결합만으로 6-cycle/ABS 대폭 감소, terminal lift(`J=5`)로 특정 파라미터(`p=5,7,m=1,2` 등)에서 6-cycle 완전 제거, AWGN sliding window BP(윈도 `4ν`, 50 iter)에서 waterfall 이득·error floor 없음 확인.
10. 한계: HW 미설계, floating-point 시뮬만, rate/length는 nested·`p`·`J`에 종속(단일 확정 불가), non-nested 단일부호 대비 이득 불명확.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| nested AB-SC-LDPC edge-spreading 구성 | 대응 없음 | Prime ECC는 단일 block QC-LDPC, nested SC convolutional 구조와 무관 |
| ALC 6-cycle 열거로 ABS 최소화 (부호설계) | 대응 없음 | Prime ECC는 고정 H-matrix, 부호설계 층 재검증 부담 높음(프로파일 §4) |
| H-matrix/부호구조 로드 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 로드 지점은 있으나 SC 밴드/nested 구조 자체가 비호환 |
| sliding window BP 복호 | 대응 없음 | Prime ECC는 block QC-LDPC pipeline min-sum, window 복호 아님 |

적용 가치: **낮음** — 다중단말용 nested SC-LDPC 부호설계 최적화로, Prime ECC 3.1의 단일 block binary QC-LDPC·min-sum HW 디코더와 정합되지 않아 이식 불가.
