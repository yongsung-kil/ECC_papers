### B. 알고리즘 요약

1. 시스템: protograph 기반 generalized LDPC-Hadamard 부호(PLDPC-HC), ultimate-Shannon-limit 접근(−1.59 dB 근처), AWGN+BPSK, `r=4`, rate `R=0.0494`, 이중 lifting `z1=32`,`z2=512`, code length `l=1,327,104`.
2. 문제: PLDPC-HC의 표준 BP 복호는 수렴이 느리고(최대 300 iter) 메모리/연산자원 큼; generalized LDPC용 layered 복호가 부재.
3. 모델: SPC를 Hadamard 부호로 대체한 HCN + degree-1 Hadamard VN(D1H-VN) 구조, 각 HCN에 `2^r−r−2`개 D1H-VN 부착(r=4→10개).
4. 핵심기법: base matrix `B_{m×n}`를 두번 lift한 `H_{M×N}`을 `m·z1` layer로 분할(각 layer=`z2×nz1z2` block-row), layer 단위로 APP LLR `L^app_PVN`를 즉시 갱신하는 layered 스케줄.
5. 핵심식: `L^app_PVN(β)=L^app_Hpp(α,β)` (식8) — layer 처리 직후 갱신된 APP를 다음 layer가 바로 사용해 within-iteration 정보전파 가속(수렴 2배).
6. 핵심기법2: HCN 처리는 symbol-by-symbol MAP Hadamard decoder(FHT/DFHT 변환 `T`)로 `d=r+2` extrinsic + `2^r−r−2` D1H-VN 입력 받아 출력.
7. 구현 디테일: layered는 `{L^app_PVN}`,`{L^ex_Hx}`만 저장하면 되고 중간변수(`L^app_Hpp`,`L^ex_PxVN`) 무저장; 표준은 `{L^ch_PVN}`을 iter 내내 유지해야 해 메모리 큼(N=180224 LLR).
8. 학습/최적화: 없음(스케줄 변경만), degree distribution은 기존 [6] PEXIT 최적화 부호 재사용.
9. 결과: BER `2.0×10^-5`에서 layered 20 iter가 표준 40 iter 대비 0.03 dB 손실뿐, layered 21 iter = 표준 41 iter 성능; 평균 iteration은 표준의 약 절반, 메모리·연산로직 감소.
10. 한계: HW 미구현(HW 적합성만 정성 언급), 초저rate LDPC-Hadamard 특화, BER 시뮬만.
