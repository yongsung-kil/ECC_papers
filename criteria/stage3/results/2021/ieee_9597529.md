### B. 알고리즘 요약

1. 대상: 19nm planar MLC NAND(2,096 blocks×512 pages, 페이지당 8 섹터), 섹터=2kB user + 234B spare, ECC는 100-bit BCH(code rate `0.92`, parity 188B) 가정.
2. 풀려는 문제: 섹터별 endurance(P-E cycle 한계) 편차가 커서 최악값 기준 보수적 스펙 → 섹터별 true endurance를 초기수명에 예측해 더 단순·저가 ECC 사용과 수명연장을 노림.
3. 가정/모델: 일부 셀은 process variation으로 charge trap이 심한 `weak cell`이고, 섹터 EOL 에러는 weak cell 비율에 상관, 이 특성은 초기수명 측정으로 식별 가능하다는 가설.
4. 데이터: 45개 device·6,675 blocks·2,700만+ 섹터를 Raspberry Pi 기반 커스텀 테스터로 가속시험(81°C, 500h cycling + 18h retention bake), 6k~15k cycle 10단계.
5. 초기수명 예측 피처: 섹터 에러수, page program time, block erase time + page number + cycling level (Table 2).
6. 핵심 발견: CWER은 mean RBER이 아니라 RBER 분포 tail이 결정(15k가 mean 최고이나 8k tail이 더 높음); 실패 섹터의 57~79%가 소수 page(page 1,7 등 LSB/MSB even 첫 워드라인)에 집중.
7. ML 기법: gradient boosting(R `gbm`, n.trees=400, depth=3, shrinkage=0.09)으로 pre/post-retention 이진분류(pass/fail), 심한 클래스 불균형(>99.9% pass)은 undersampling으로 완화.
8. 최적화: SMOTE 변형 synthetic oversampling이 duplication보다 우수(specificity 98.95%, sensitivity 99.61%, AUC 0.9993).
9. 튜닝(핵심 신규): 학습은 80-bit decision boundary로, 테스트는 100-bit로 두고 pass/fail 확률 임계값을 0.45로 낮춰 false-negative 0(100% sensitivity, guardband 포함) 달성 → 데이터 손실 없이 수명연장/단순 ECC 허용.
10. 한계: 디코더·부호설계·HW를 다루지 않는 시스템/ML 레이어 연구이며 planar MLC 특정 device·gatecount/throughput 등 ECC HW 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

대상이 `other`(섹터 수명예측 ML 시스템)로, LDPC 디코더/부호설계/HW 어느 것도 개선하지 않아 코드 모듈 대응이 사실상 없다.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| gradient boosting 섹터 endurance 예측 | 대응 없음 | 컨트롤러/펌웨어 레이어 ML로 디코더 시뮬과 무관 |
| RBER/CWER·weak cell·retention 에러 모델 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()` (RBER 주입) | 채널 에러 통계 관점에서만 개념적 연관, 기법 이식 아님 |
| ECC threshold(100-bit)·code rate 0.92 가정 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` (고rate 코드 스펙) | 고rate NAND ECC 문맥만 공유, 부호 자체 기여 없음 |

적용 가치: **낮음** — NAND 신뢰성에 직접 관련되나 기여가 섹터 수명예측 ML 시스템 레이어라, binary QC-LDPC 디코더/부호설계/HW를 다루는 Prime ECC에 떼어 쓸 알고리즘 요소가 없다.
