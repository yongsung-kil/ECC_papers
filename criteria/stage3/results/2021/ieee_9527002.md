### B. 알고리즘 요약

1. 대상은 5G/통신용 LDPC로, 테스트셋 C1[1040,5408]/C2[560,1120]/C3[1040,2288], regular·irregular 모두 지원(rate·length 임의 조정 가능).
2. 문제: GPP에서 LDPC 소프트 디코더의 throughput 부족 — AVX-512 기반 범용 디코더 설계가 공백이었음.
3. 알고리즘은 `TDMP`(layered/horizontal) 스케줄링 기반 `OMS`(offset min-sum), flooding 대비 수렴 반복수 1/2, 메모리 40% 절감.
4. 핵심 기법: AVX-512 512bit 레지스터로 32 float / 64 char 동시 처리, 프레임 레벨 병렬(`q`개 프레임 SIMD).
5. CN 갱신은 sign은 `∏sign(Q)`, magnitude는 `min|Q|`; `Q = Ln - R`, `Ln = Q + R`로 실시간 갱신(Q 별도 저장 생략).
6. 3단계 병렬성(프레임/CN/CN내부) 중 CN·CN내부는 데이터의존성·저차수로 비효율 → 프레임병렬만 채택.
7. 구현 최적화: 입력 8-byte 고정소수점화(값 <2^6), 행가중치 통계 후 H 재정렬, sign·prod를 `vpandd`/`vpxord`로, min을 `vpcmb`+`vpblendmb`+`vpxord`로 대체.
8. 학습/최적화 절차 없음(순수 SIMD 구현 최적화).
9. 결과: i9-10920X, 10 iter, 10000회 평균 throughput 53~131Mbps; AVX-512가 SSE 대비 최대 +60%, AVX2 대비 +20%.
10. 한계: HW(RTL) 미설계·CPU SW 전용, error correction 성능 개선 없음(순수 속도), 지연은 프레임병렬로 오히려 증가.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| OMS(offset min-sum) CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 알고리즘은 min-sum 계열로 유사하나 Prime은 이미 보유 |
| TDMP layered 스케줄링 | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 스케줄링 개념 대응하나 Prime은 dual-update 스케줄 이미 보유 |
| 고정소수점/양자화 | decoder.cpp VN_Cal_HD() 등, ecc_data.h PARAM_LLR | 양자화 개념만 유사, SIMD 구현은 무관 |
| AVX-512 SIMD 명령어 최적화 | 대응 없음 | Prime은 HW(RTL) bit-exact 모사, CPU 명령어 최적화와 무관 |
```

적용 가치: **낮음** — 기여가 x86 AVX-512 SIMD 구현 최적화에 국한되고 알고리즘(TDMP OMS)은 표준이며 Prime이 이미 보유, HW ECC 이식 접점 없음.
