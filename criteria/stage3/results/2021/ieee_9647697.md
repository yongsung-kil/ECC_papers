### B. 알고리즘 요약

1. 시스템: OpenAirInterface(OAI) 기반 소프트웨어 5G NR 물리계층 처리에 FPGA LDPC 가속기를 PCIe(Gen2 8-lane)로 연결한 오프로딩 프로토타입.
2. 문제: 범용 CPU 소프트웨어 처리에서 LDPC 디코딩이 PUSCH 수신 처리 시간의 최대 63%를 차지하는 병목이라 5G NR 실시간(30kHz SCS, slot 500us) 요건 미충족.
3. 코드 스펙: 3GPP 5G NR LDPC(BG1/BG2), code block `K=8448`, code rate `R=1/3`~`8/9`, LLR 8-bit 표현, 2/5/8 iteration.
4. 핵심 기법: LDPC Encoder/Decoder Wrapper를 독립 VHDL 모듈로 구현하고 Xilinx LDPC IP Core를 인스턴스화, RIFFA 프레임워크로 PCIe 인터페이스 구성.
5. 스트리밍 구조: code block별 설정 레지스터를 두지 않고 128-bit 헤더에 config/status를 실어 파이프라이닝, 인코더/디코더 채널 독립·병렬 접근 지원.
6. 통합 디테일: OAI LLR 포맷의 saturation·sign inversion, 인코더 출력 포맷 변환 등 데이터 포맷 정합 작업 수행.
7. FPGA 구현: VC707 보드, 250MHz 클록, Intel Xeon E5-1620 v3 호스트, nr_ulsim/nr_dlsim 테스트벤치로 기능·성능 검증.
8. 결과1: 디코더 오프로딩은 큰 code block·낮은 rate·많은 iteration에서 유리, 최대 5.23배 속도 향상(BG1, K=8448, R=1/3, 8It).
9. 결과2: rate 8/9 2It에서는 오버헤드가 오프로딩 시간의 90.2%로 속도 향상 거의 없음; PCIe 송신 20.24us가 최대 기여(8-bit LLR로 송신 크기 24배).
10. 한계: 알고리즘/부호 개선 없음, 5G NR IP Core 블랙박스 사용, 오프로딩이 항상 유리하지 않음을 밝힌 시스템 레벨 분석에 그침.
