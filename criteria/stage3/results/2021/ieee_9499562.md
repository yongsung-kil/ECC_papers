### B. 알고리즘 요약

1. 시스템: QC-LDPC 인코더, 생성행렬 `G=[I|P]`(P는 순환치환 부분행렬), lifting size `m`, `K=k·m`, `N=n·m`, rate `R=k/n`; 검증부호 CCSDS (7154, 8176), rate≈0.875.
2. 문제: 전통 LDPC 인코딩은 블록길이 제곱 복잡도; SRAA(shift register adder accumulator) 방식도 직렬은 throughput 제약, 병렬은 레지스터 과다.
3. 모델: 부호화 `C=uG`, 정보열 `u`를 `k` 세그먼트로 분할 후 각 `u_i`에 서브행렬 `B_ij` 곱해 parity `V_j` 산출.
4. 핵심기법: Bit Selection(BS) — `B_ij` 첫 열의 비영 위치집합 `λ_ij=find(b_ij==1)`만 골라 `u_i(λ_ij)` 유효비트만 XOR, `0` 무효비트 연산 회피 (`hij(s)=mod(sum(u_i'),2)` 후 `circshift`).
5. 핵심식: `V_j = Σ_i u_i·B_ij` 를 `Σ_i h_ij` 로 치환 — 정보비트-생성행렬 곱을 비영 위치 선택 누산으로 대체(무효 비트 곱 제거가 핵심).
6. HW 구조: data-preprocessing + `k`개 cyclic shift network + `k(n−k)`개 bit selection unit + `(n−k)`개 bit merge unit.
7. 구현 디테일: shift는 `m` 클록; 생성행렬 밀도 높아 유효비트 길이 약 `k·m/2`, 누산은 `log(k·m/2)` 스테이지 트리; AND 게이트 0, XOR `k·m`.
8. 최적화: [8](Wei Yan) 대비 AND 완전 제거, XOR 약 50% 감소, 인코딩 지연 고정(유효비트 수 무관하게 결정적).
9. 결과: Xilinx Virtex-4 VC4VLX80 구현 시 Slice 10% / 4-input LUT 11% 절감, 최대 clock 425MHz, 최대 throughput 5.84 Gbps.
10. 한계: 인코더 전용(복호 성능 무관), CCSDS 위성통신 부호 특화, BER/정정능력 미평가.
