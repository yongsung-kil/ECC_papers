### B. 알고리즘 요약

1. 대상은 유리(rational) 패리티검사행렬을 갖는 LDPC 컨볼루션 부호(LDPC-CC-RPCM), AWGN+BPSK 채널, block length 10000/20000.
2. 문제: RPCM을 직접 Laurent 급수로 전개하면 memory order `ms=∞`, length-4 cycle 다수 발생해 BP 성능 붕괴.
3. Procedure 1/2로 RPCM을 등가 다항식 PCM `H̄(D)`로 변환 → 유한 memory order + girth 확대(length-4/6 cycle 제거). 이때 원부호는 mother code에서 punctured child code로 해석됨.
4. 핵심 문제: 변환 그래프의 일부 punctured variable node가 flooding schedule 초기 iteration에서 복원 안 됨 → 이웃 check node 갱신 마비 + 오류 전파.
5. 해법: 각 punctured node에 supporting node(가상 check)를 부여, 다른 unpunctured node로부터 virtual channel output(LLR)을 생성해 공급.
6. 초기화 LLR: `2·tanh⁻¹(Π tanh(y_ij/σ²))` (식 8), 반복 중 추가 메시지 `2·tanh⁻¹(Π tanh(L(v_ij)/2))` (식 9).
7. supporting node 4조건: 등가 패리티 제약 / 이웃은 unpunctured만 / 이웃 최소화 / short cycle·trapping set 미도입.
8. BP는 원래 `G(H̄(D))`에서만 수행하고 supporting node는 메시지 생성용으로만 사용 → 추가 HW/연산 제한적.
9. 결과: punctured bit당 support node 1개로 flooding 대비 BER 10⁻⁵에서 0.6~0.8dB SNR 이득, dynamic scheduling 대비 0.3dB 우위, LDPC-CC-PPCM보다 우수. iteration 50 고정.
10. 한계: HW 미설계·시뮬만, LDPC-CC(convolutional)+RPCM 전용, memory order 200~400 대형, latency는 flooding 병렬성 유지로만 개선.
