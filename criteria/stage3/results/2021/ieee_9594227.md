### B. 알고리즘 요약

1. 대상 부호: (16200,7200) DVB-S2 표준 LDPC(rate 0.444), (3096,1032) PBRL protograph 부호(rate 0.333), BPSK+AWGN 채널.
2. 문제: Neural NMS(N-NMS)는 edge마다 곱셈 가중치를 학습해 성능↑이지만, 긴 부호에서 edge 수가 급증해 Pytorch/Tensorflow 메모리 요구가 비현실적.
3. 핵심 관찰: 학습된 N-NMS 가중치가 check node degree `d_c`, variable node degree `d_v`, iteration `t`와 강한 상관 → degree 큰 노드일수록 가중치 `B(t,d_c)` 작음.
4. 핵심 기법: N-2D-NMS — 같은 `(d_c, d_v)`를 갖는 edge에 동일 가중치 `beta_c`, `alpha_v` 공유. Type 0~8로 공유 정도(무공유/degree/protomatrix layer/iteration-only) 구분.
5. 갱신식: `u_ci->vj = beta(t) * (prod sgn) * min|...|`, `u_vj->ci = I_ch + alpha(t) * sum u_ci'->vj` (식 16-17).
6. 효율 구현: forward에서 `min1,min2,pos1,pos2,sign`만 저장, backprop 그래디언트를 iteration별 전체가 아닌 직전 iteration만 저장 → Tensorflow 메모리 병목 해소.
7. Hybrid 구조: 앞 `I'` iteration은 feedforward(고유 파라미터), 나머지는 recurrent(파라미터 재사용). DVB-S2에서 `I'=20`이면 full feedforward와 동등.
8. 학습: multi-loss cross-entropy로 flooding/layered schedule 하에 파라미터 학습(최대 iter 10 또는 50).
9. 결과: Type 1/2/5가 N-NMS와 동일 FER이면서 파라미터 99.7%/99.9%/99.3% 감소. N-2D-NMS가 DVB-S2에서 BP보다 낮은 error floor. N-NMS는 NMS 대비 0.5dB+ 개선.
10. 한계: HW 미설계, 통신부호·AWGN 시뮬만, non-degree Type은 0.05~0.2dB 열화.
