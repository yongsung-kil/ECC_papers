### B. 알고리즘 요약

1. 대상은 `(3,6)`-regular SC-LDPC(coupling width `ω=2`, base matrix `B=[3 3]`, lifting `M`), rate 1/2 를 슬라이딩윈도 디코딩(SWD, window `W`)으로 복호.
2. 문제: 저지연 위해 `W < 6v`로 작으면 드물지만 심각한 decoder error propagation(연속 블록오류 버스트)이 발생해 BLER가 급락.
3. 채널은 BI-AWGN, BP flooding 스케줄로 최대 `I` iteration 후 첫 윈도의 target block을 LLR 부호로 결정하고 윈도를 1 블록 이동.
4. 핵심: 디코더 상태를 랜덤오류 `S0`, 중간상태 `S1..S_{J-1}`, 버스트 상태 `SJ`로 두는 다상태 마르코프 모델을 세움. 상태 `Si`의 BLER를 `qi`로 정의.
5. `qJ-1≥...≥q1≥q0`: 윈도에 남은 오류 블록이 다음 블록 복호를 악화시켜 상태가 오를수록 오류율 증가. `qJ→1`이면 무한 에러전파.
6. 무한길이(`L→∞`) 정상상태 확률 `pi`로부터 평균 BLER `PB=rJ/(1-qJ+rJ)` 폐형식 유도 (`rJ`는 `q0..qJ-1`의 재귀식 (12)).
7. 유한 `L` 확장: dwell time `d0=1/q0`, `d1=1/(1-q1)`, cycle time `x`, cycle 수 `y=L/x`로 각 상태 블록오류수 `n̄i`를 세어 `PB^(L)=Σn̄i/L` (식 (30)).
8. 파라미터 추정: 목표 SNR에서 SC-LDPC를 시뮬해 finite-length/EOF 버스트 길이 분포 파일을 얻고, `Tj,Ej`로부터 `qj=Ej/Tj` 계산 (식 (31)-(38)).
9. 결과(Ex.7): `M=1000,W=12,Eb/N0=0.9dB`에서 시뮬 BLER=0.467을 모델이 정확히 재현, `L=5000→2500`에서 재시뮬 없이 doping 이득(BLER 약 50% 감소) 예측.
10. 한계: HW 미설계, 순수 성능예측 분석 모델이며 디코더 정정능력 자체를 개선하지 않고 스트리밍 SC-LDPC에 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SWD 슬라이딩 윈도 복호 루프 | 대응 없음 | Prime ECC는 block QC-LDPC 전체 이터레이션(`decoder.cpp` `LDPC_Decoder()`)이라 윈도 스트리밍 개념 없음 |
| BP flooding + LLR 부호 결정 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `VN_Cal_HD()` | 일반적 BP만 공유, min-sum과 무관 |
| 다상태 BLER 예측 모델 | 대응 없음 | 통계 예측 모델일 뿐, 코드/디코더 로직 아님 |
| code doping (CN/VN 삽입) | 대응 없음 | 부호구조 변형이나 SC-LDPC 인코더 특화 |
