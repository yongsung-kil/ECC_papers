### B. 알고리즘 요약

1. 대상: WiMAX(802.16e)/WiFi(802.11n) QC-LDPC(base matrix + circulant shift), 코드길이 N=576~2304, circulant size z=24~96.
2. 문제: 기존 병렬 LDPC FPGA 디코더는 단일 표준/단일 코드길이 종속 or PCM 저장 방식 비효율.
3. 알고리즘: 표준 min-sum(MSA) - VTC `qmn=pn+Σrm'n`(식2), APP `Λn=pn+Σrmn`(식3), CTV `rmn=(∏sign)·α·min|qmn|`(식4)에 scaling factor α만 적용.
4. 아키텍처: 입력 직렬→병렬 변환 후 block memory(K=N/z blocks)에 저장, VTC계산·min/submin탐색·CNU(CTV+APP)·sign삽입 블록으로 구성.
5. min/submin 블록: 8입력 signed, ABS로 절대값·sign분리, sign XOR, 4단 comparator cascade로 min1/min2 탐색(row weight 최대 7).
6. 병렬화: PCM 행의 모든 APP를 동시 갱신 - core 수 = circulant size z, z개 memory instance로 CTV 저장.
7. PCM 저장 트릭: base matrix Hb만 저장하고 3벡터(`vals`=−1 아닌 원소, `pos`=열인덱스, `elementSize`=행별 유효원소수)로 압축해 메모리 절감.
8. APP memory: dual-port block RAM `(z×input width)×(N/z)`, 디코딩 완료 후 전 블록 출력.
9. 결과: Zynq-7000(XC7Z100)에서 fmax 최대 240MHz, throughput 최대 1.2Gbps(N=2304, 1 iteration 기준), LUT 80k~188k.
10. 한계: 정정능력/BER 개선 없음(순수 HW), 양자화 bit수·code rate·iteration당 cycle 미보고, 특정 표준 PCM 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum CTV(min1/min2 + sign XOR) 계산 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 min1/min2+sign 연산 - 이미 보유 기법, 중복 |
| VTC/APP 갱신, iteration 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()`, `VN_Cal_HD()` | 표준 VN/APP 갱신과 대응 |
| z-core 병렬(core수=circulant z) | 디코더 HW 모델(z=32) | Prime ECC z_sb=32 row 병렬과 유사 개념, 구조 정합 필요 |
| 3벡터 PCM 압축 저장(vals/pos/elementSize) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix/PCM 로드·저장 방식 참고 가능 |
| APP dual-port BRAM/메모리 라우팅 | 디코더 HW 모델(HCU, GT) | 메모리 구조 참고이나 HCU/GT와 직접 대응 없음 |

적용 가치: **낮음** - 표준 min-sum(scaling α) 병렬 FPGA 구현으로 알고리즘 기여가 이미 Prime ECC 보유 범위와 중복이며 정정성능 개선이 없음. 3벡터 PCM 저장·z-core 병렬 방식만 HW 참고 수준.
