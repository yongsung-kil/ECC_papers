### B. 알고리즘 요약

1. 대상은 5G NR QC-LDPC(base graph 2, rate `R=1/2`, `K=154` short packet, QPSK)로 URLLC용 저지연 short-code 복호가 목표.
2. 문제: layer min-sum 계열(LNMS 정규화 α, LOMS 오프셋 β)의 성능이 인자값에 크게 좌우되나, 기존 DE·trapping-set 기반 방법이 layer 알고리즘의 최적 인자를 못 준다.
3. 5G NR base graph는 degree-1 VN(extension VN)을 가져 전통 2D EXIT가 `(1,1)`에 도달 못 함 → core/extension VN·CN을 분리한 `3D EXIT chart`(문헌 [12]) 사용.
4. 핵심: SPA용 3D EXIT를 layer NMS/OMS로 확장. min-sum은 Gaussian 근사(J함수)가 안 맞으므로, 상호정보를 통계적으로 계산 `I(L;X)≈1-(1/N)Σ log2(1+e^{-x_j L_j})` (식15).
5. 이 식으로 core VN 출력 `Ivc,E`(16)와 core CN 출력 `Icc,E`(17)를 α 또는 β를 넣은 (6)/(7) 갱신으로 실측하듯 산출.
6. Algorithm 1: layer 순서로 `Lj→i`(8)와 `Li→j`(6/7)를 갱신하며 `Ivc,E(Ivc,A)` 곡선을 그림; 출력정보 곡선이 높을수록 좋은 인자.
7. 부수: 6-bit 균일 양자화(부호1/정수3/소수2)에서 α=0.25/0.5/0.625/0.75를 shift+add로 근사(고정소수점 HW 친화).
8. 학습/최적화: NN 없음. EXIT 곡선 비교로 인자 스윕 최적화.
9. 결과: LNMS는 α=0.6~0.7이 최적(경험값 α=0.5 대비 BLER=1e-3에서 약 0.3dB 이득); LOMS는 β∈[0.6,1.0]에서 구별 없음(고유 최적 없음); 고정소수점 α=0.625도 α=0.5 대비 0.3dB 이득이나 error floor 존재.
10. 한계: HW 미설계, 시뮬만, EXIT 모델이 5G NR base graph 구조에 특화, LOMS는 명확한 최적 β 도출 실패.

| 정규화 인자 shift+add 고정소수점 근사 | `decoder.cpp` `VN_Cal_HD()` 등 / `ecc_data.h` `PARAM_LLR` | 양자화 magnitude 테이블 튜닝에 참고 |
