### B. 알고리즘 요약

1. 시스템: side blockchain의 data-availability oracle, N개 oracle 노드에 βN(β<0.5)까지 악의적, LDPC로 부호화한 Coded Interleaving Tree(CIT) 청크를 분산; 파라미터 `M=256`, `R=0.5`, `l=4` layer(`n=32,64,128,256`).
2. 문제: [6]의 dispersal+LDPC 분리 설계는 최소 stopping-set 크기에 통신비용이 반비례해 큰 코드가 필요하고 N 확장에 한계.
3. 모델: peeling decoder가 stopping set 미존재 시 각 CIT layer 복호 보장(가용성). stopping set = 연결 CN이 모두 2회 이상 연결된 VN 집합.
4. 핵심기법 1단(dispersal): `k*-secure dispersal` — secure phase(작은 stopping set들을 f+1 노드에 배치) + valid phase(`k*`-dispersal). 통신비용 `Cs`가 base layer `|V_l^gr|`에 강하게 의존.
5. 핵심식: 각 stopping set S가 `|neigh(S)|≥f+1`이면 정직 노드 1개 이상이 해당 청크 보유 → peeling 실패 방지(securely dispersed).
6. 핵심기법 2단(부호설계): `DE-PEG` — PEG로 VN 순차 연결하되, `g≤gmax` cycle 생성 시 각 후보 CN에 대해 greedy-size로 예상 secure 집합 크기 `s̄`를 계산해 최소인 CN 선택 → 작은 `V_l^gr`.
7. 부수 트릭: EMD(extrinsic message degree) ≤ `Tth`인 cycle을 bad cycle로 관리(stopping set 형성 가능성 높음), `dv=4`, `gmax=6~8`, `Tth=5`.
8. 최적화: greedy 방식으로 최다 stopping set 포함 VN 반복 선택해 `V_j^gr` 구성, CIT 컬럼 순열로 layer 간 POM 정합.
9. 결과: `N=9000,β=0.49`에서 base layer Mmin PEG=17/DE-PEG=18, `µ=20`일 때 총비용 CT가 baseline(5.125GB) 대비 DE-PEG 0.528GB 감소; N=15000에서 9.3% 절감.
10. 한계: 순수 시뮬, HW 미설계, erasure/peeling·통신비용 목적으로 soft-decision·BER 개선과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PEG/DE-PEG H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base+shift) 고정, PEG 랜덤 TG 구성과 다른 부호계열 |
| stopping set / EMD cycle 관리 | 대응 없음 | erasure peeling stopping set 최적화는 Prime ECC 범위(min-sum BER) 밖 |
| peeling(erasure) decoder | 대응 없음 | Prime ECC는 iterative min-sum(soft), peeling 미지원 |
| dispersal 통신비용 최적화 | 대응 없음 | 분산 저장/통신 목적, NAND on-chip ECC와 무관 |

적용 가치: **낮음** — non-QC PEG 부호설계 + erasure/peeling + 분산 dispersal 통신비용 목적으로, 고rate binary QC-LDPC의 min-sum BER/error-floor 개선과 목표·부호·복호 모두 어긋남.
