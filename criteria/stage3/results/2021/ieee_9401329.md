> 총평: row-based layered NMS 파이프라인의 Lpost/C2V 데이터 의존 stall을 명령·데이터 레벨 재정렬로 제거하는 스케줄링 기법. 5G NR 특화지만 layered NMS 자체는 우리 min-sum 디코더와 동종이라 파이프라인 스케줄 아이디어는 조건부 이식 가능(부호는 비이식).

### B. 알고리즘 요약

1. 시스템: 5G NR LDPC 전체 구성(2 base graph, 8 set index, 51 lifting size = 102 PCM) 지원 reconfigurable 디코더, 5-bit 양자화.
2. 문제: irregular QC-LDPC(BG1 row degree 3~19, col degree 1~30) + 파이프라인의 데이터 의존 해저드가 latency를 늘려 고throughput 설계를 저해.
3. 기반: row-based layered NMS, 1 iteration을 BG1 46 / BG2 42 sub-iteration(layer)으로 분할, block-parallel 아키텍처로 라우팅 복잡도 완화.
4. 핵심 해저드: `Lpost` 갱신이 해당 layer의 GMS(global min & sign)에 의존 → CNU가 GMS 얻는 데 `dc`(row degree) cycle, Lpost update가 파이프라인을 `dc` cycle stall.
5. 기법 1 (instruction-level reordering): Lpost를 GMS 확보 즉시 갱신하지 않고 "다음 접근 시점"에 복원(update를 이전 stage로 이동) → GMS 의존 `dc` cycle 제거.
6. 잔여 문제: 파이프라인 깊이·C2V↔Lpost 해저드로 여전히 3 cycle stall.
7. 기법 2 (data-level rescheduling): (a) 각 layer에 non-overlap CPM ≥3 확보(check node rescheduling), (b) 각 layer 첫 3 cycle에 non-overlap CPM만 처리(variable node rescheduling)로 잔여 stall 제거, 복호성능 열화 없음.
8. HW: VNU를 PipeStage1(Lpost update)/PipeStage2(V2C)로 분할, CNU를 PipeStage3(accumulated sorter+sign), SyndromeUpdate/EarlyTermination 별도.
9. 결과: FPGA(VCU1525) 6.7 Gbps/iter @111MHz; TSMC 28nm post-layout 33.2 Gbps/iter @556MHz, core 1.97mm², 2.83M gate, power 232mW, 조기종료 시 BG1 avg iter 14.44/9.12/5.
10. 한계: 5G NR 저rate(1/3~1/2)·특정 PCM 전용, 정정성능·iteration 자체 개선 없음(스케줄링으로 stall latency만 개선), rescheduling이 "layer당 non-overlap CPM ≥3" 구조 조건에 의존.
