### B. 알고리즘 요약

1. 대상은 protograph LDPC(5G NR BG2), copy-and-permute lifting factor `Z`로 `(N,K)=(Nb·Z, ...)` 파생. 예제 `(156,30)` BG2(`Z=3`), 추가 `Z=16`. AWGN·Rayleigh, BPSK.
2. 문제: 기존 neural SP/NMS/OMS 디코더는 NN 크기가 codelength·iteration에 급증해 short~moderate만 지원, 학습 복잡도 과다.
3. 핵심: layered MS(TDMP)의 CN 순차 스케줄을 그대로 unfold해 커스텀 sparse NN 구성. iteration=1 layer, 각 layer를 VN/CN/decision sublayer로 분할.
4. 핵심 변수: CN sublayer에 학습 weight `α`(normalize)·bias `β`(offset)를 edge·iteration별 부여. `μc→v = sgn(...)·ReLU(α·min|μv'→c| - β)` (식5의 NMS/OMS를 학습형으로).
5. lifting 제약: base graph 동일 edge에서 파생된 모든 lifted edge는 동일 `α,β` 공유(식10). 파라미터쌍 수 = base graph edge 수로 고정 -> 임의 codelength 지원.
6. layered 스케줄로 같은 iteration 내 갱신 즉시 반영, 수렴 iteration 절반 단축(SP 20 iter ≈ layered 10 iter).
7. 손실: 각 decision sublayer 출력 `o=σ(μ)`와 송신 codeword의 MSE. Adam 최적화.
8. 학습: vanishing gradient 회피 위해 iteration-wise greedy 학습 - 한 layer 학습 후 고정하고 다음 layer 성장. 조기종료·iteration 재사용과 정합.
9. 결과: 제안 neural layered MS가 plain layered MS 대비 큰 이득, 고SNR서 layered SP도 상회(waterfall). AWGN 학습 파라미터를 Rayleigh서 재학습 없이 사용해도 강건. multi-loss 학습 대비 조기 iteration서 우수.
10. 한계: HW 미설계, gate/throughput 미보고, 저rate 5G NR 부호 시뮬만. 이득 원천이 학습 α/β라 오프라인 학습 인프라 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 학습형 min-sum α(normalize)/β(offset) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum에 scaling/offset factor 이식점; 학습값을 정적 magnitude 테이블로 증류 필요 |
| iteration별 α/β 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration-구간별 파라미터 개념이 적응형 LLR 테이블과 정합 |
| layered(TDMP) CN 순차 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 dual-update(even/odd 교대) 스케줄 보유, layered는 유사 계열 |
| protograph lifting(base+permute) 부호구조 | `ecc_top.cpp` `Load_PCM()` | QC base matrix+circulant과 개념 대응(둘 다 lifting), 단 부호 재설계 부담 |

적용 가치: **중간** — min-sum·layered·lifting 구조가 Prime ECC와 정합적이고 학습된 iteration별 α/β를 정적 파라미터로 증류하면 waterfall/수렴 이득 이식 여지가 있으나, HW 미설계·저rate 5G 부호·NN 학습 인프라 의존이 리스크.
