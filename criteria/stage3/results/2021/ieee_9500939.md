### B. 알고리즘 요약

1. 시스템: 유한 블록길이 `n`, rate `r`, 최대 변수노드 degree `dv`, 최대 체크노드 degree `dc`가 고정된 이진 LDPC 앙상블 (예제 `n=100, r=1/2, dv=4, dc=5`).
2. 문제: 기존 LDPC 최적화는 블록길이 무한 가정(DE/EXIT)이라 짧은 코드에서 유효 degree distribution 이산성을 무시함.
3. 모델: Tanner 그래프의 edge 비율 벡터 `λ`(변수노드), `ρ`(체크노드) 쌍을 degree distribution pair로 정의, 제약식 (1)~(3)로 유효성 규정.
4. 핵심기법 1단: 고정 edge수 `ne`와 한 유효 pair `(λ,ρ)`에서, 차이 벡터 `h`를 null-space 제약(식 (4)~(8))으로 표현해 나머지 모든 유효 `λ̂=λ+h`를 생성.
5. 핵심 식: `h`는 `Ch=0`(식 6)의 null space에 있어야 하고 `hi`는 `i/ne`의 정수배여야 함 - 유효 pair의 이산 격자를 규정.
6. 핵심기법 2단: edge수 하한 `ne_min=2n`, 상한 `ne_max=min{ndv, mdc}` (식 Step2)로 탐색 범위를 확정.
7. 구현 디테일: Algorithm 1로 최소 edge pair 생성 후, Algorithm 2가 `k`를 `ne_min`~`ne_max`로 증가시키며 각 edge수마다 유효 집합 `Ĥλ, Ĥρ`(식 12)를 구성.
8. 최적화 절차: 열거된 전체 pair 집합을 이산 최적화 루틴 입력으로 넘겨, AWGN에서 시뮬 BER 등 임의 목적함수를 최소화.
9. 결과: `n=100, ne=202` 예제에서 유효 변수노드 분포는 단 2개, 최적 pair는 `λ=(0,196/202,6/202,0)`, `ρ=(0,0,0,192/202,10/202)` (Eb/N0=5dB, AWGN).
10. 한계: FLOP 복잡도가 `dv,dc`에 지수적이라 소형 코드에만 실용, 디코더/HW 미설계, degree distribution 열거만으로 실제 H-matrix 구성·성능은 별도 절차.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| degree distribution 열거 (앙상블 설계) | 대응 없음 | Prime ECC는 특정 고rate QC-LDPC H-matrix 고정, 열거 대상 아님 |
| 부호 pair 최적화 목적함수 (BER 시뮬) | ecc_top.cpp Load_PCM() (간접) | 구조 로드만 대응, 열거·최적화 로직 부재 |
```

적용 가치: **낮음** - 소형 irregular LDPC 앙상블 설계 이론이며, 고rate 고정 QC-LDPC binary NAND ECC(Prime ECC)에는 열거 알고리즘·목적함수 모두 직접 대응 모듈이 없고 이식 부담만 큼.
