### B. 알고리즘 요약

1. 대상은 DVB-S2 위성표준 LDPC(short mode `N=16200`, rate 3/5~8/9), QPSK·16APSK 변조에 BICM-ID 적용.
2. 문제: DVB-S2는 rate마다 개별 PCM이라 단일 enc/dec 구조가 어렵고 기존 부호의 SNR 임계가 Shannon과 격차 큼.
3. 채널 모델은 AWGN + APSK 매핑, 복조기 LLR을 디코더와 반복 교환(BICM-ID).
4. 핵심 기법: 작은 기저행렬 `B`(D×F)를 PEXIT로 분석·설계 후 `T`회 copy-permute lifting으로 protograph LDPC 생성.
5. PEXIT 4단계(`J(σ)` 초기화 → VN→CN `I_Ev` → CN→VN `I_Ec` → `I_App`)로 `I_App=1` 수렴 SNR 임계를 예측, 이 임계 최소화가 설계 목표.
6. 확장: rate 3/5~8/9별 기저행렬(예 3/5는 4×10 `H3/5`)을 PEXIT 유도 exhaustive search로 확정.
7. 설계규칙: degree-2 VN이 waterfall에 유리하나 error-floor 유발 → degree-2 노드 수를 check수-1로 제한(cycle 방지), punctured 노드 회피.
8. 학습절차 없음(PEXIT 기반 exhaustive search + 경험규칙); PPV bound로 유한블록 한계도 비교.
9. 결과: 설계 부호의 복호 임계가 DVB-S2 대비 전 rate에서 개선(예 8/9: 3.73→3.37dB), BER 10^-6에서 최대 0.2dB 이득, 16APSK BICM-ID도 유효.
10. 한계: 순수 시뮬(HW·복잡도·throughput 미제시), 위성 APSK 채널 전용, BP max iter 50 고정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph 기저행렬 설계·lifting | ecc_top.cpp Load_PCM() | Prime은 특정 QC-LDPC 고정, 부호구조 재설계 부담 큼 |
| PEXIT 임계·degree-2 VN 규칙 | 대응 없음 | 부호설계 툴(오프라인), Prime 런타임 모듈에 대응 없음 |
| BP 복호(반복 message passing) | decoder.cpp LDPC_Decoder() | Prime은 min-sum 기반으로 이미 보유, BP full-tanh는 대응 없음 |
| BICM-ID(복조기-디코더 반복) | 대응 없음 | Prime은 NAND read 소프트정보 사용, APSK BICM-ID 대응 없음 |
```

적용 가치: **낮음** — degree-2 VN 제어로 error-floor를 관리하는 PEXIT 기반 부호설계 원칙 자체는 NAND QC-LDPC 부호설계에 개념 참고 가치가 있으나, 부호구조(비-QC protograph, length 16200 고정)·위성 BICM-ID 채널이 Prime과 상이하고 부호설계 이식은 재검증 부담이 커 실효성 낮음.
