### B. 알고리즘 요약

1. 대상은 BCH 성분코드로 구성된 2D product code(PC)이며, 광통신 400G+/1Tbps 초고속·저데이터플로 FEC가 배경 (rate 0.82/0.872/0.897).
2. 문제: 기존 iBDD(hard-in hard-out)는 데이터플로가 작아 고속에 유리하나 SD LDPC 대비 성능 손실이 크다.
3. 모델: PC를 regular GLDPC 앙상블(`n^2` degree-2 VN, `2n` degree-n CN)의 멤버로 보고 bi-AWGN/BICM 채널에서 분석.
4. 핵심기법: iBDD-CR — 각 성분 BDD의 3진 출력 `μ̄∈{±1,0}`을 채널 LLR `l`과 결합해 소프트값 `l̃ = μ̃ + l`을 만들고 `B(l̃)`로 hard 결정(binary message passing 유지).
5. 핵심식 `l̃ = μ̃ + l` (식 8): `μ̃`는 `p(μ̄|l̂,c)` 로그우도비로, BDD 출력 신뢰도를 LLR 도메인으로 환산한 값.
6. 확장: BICM(M²-QAM, max-log LLR, 채널 믹싱)까지 DE 방정식 유도.
7. 구현 트릭: `μ̃`를 작은 LUT(Table I/II, iteration당 3값)로 저장, Lloyd-Max 3~4bit 채널 LLR 양자화에도 손실 0.045~0.07dB로 둔감.
8. 최적화: DE로 message error probability `x^(l)`를 반복 계산해 LUT 성분을 사전 산출(성분코드별 offline).
9. 결과: iBDD 대비 0.29~0.36dB(bi-AWGN), BICM 256-QAM에서 최대 0.51dB 이득, ideal iBDD보다도 우수. 12 iteration(10 iBDD-CR + 2 iBDD).
10. 한계: HW 미설계(복잡도는 정성 평가만), product/BDD 전용이라 LDPC 디코더와 알고리즘 이질적, error-floor보다 waterfall 개선.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BDD 출력×채널 LLR 결합 LUT | 대응 없음 | Prime ECC는 min-sum CN/VN 기반, product-code BDD 결합 규칙 대응부 없음 |
| 채널 LLR 양자화(Lloyd-Max, 3~4bit) | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | 개념적 유사(양자화)뿐, product-code 특화라 직접 이식 불가 |
| iteration별 LUT 성분 | `ecc_data.h` `PARAM_LLR` | 형식상 iteration별 테이블 유사하나 의미(BDD reliability)가 달라 재사용 불가 |

적용 가치: **낮음** — product code + BDD 전용 소프트 결합 기법으로, NAND용 binary QC-LDPC min-sum 디코더인 Prime ECC와 알고리즘 계열이 달라 이식 경로가 없다.
