### B. 알고리즘 요약

1. 시스템: BIAWGN 채널 `Y=X+Z`, `Z~N(0,σ²)` 출력을 M-level 대칭 양자화기(`M=2^q-1`)로 저정밀 메시지화 후 LDPC 복호(flooding, 50 iter).
2. 문제: SP의 체크노드 `tanh/arctanh` 부동소수 연산이 복호시간 지배 → 메시지를 이산집합으로 제한해 가속.
3. 양자화기: 결과 DMC가 weakly symmetric 되도록 임계값 `η_i`를 정리(식 5, `Q(·)` 사용)로 설정, binary search로 계산 가능하고 노이즈 σ에만 의존.
4. 핵심 기법: 이산 boxplus `â b̂ = sgn·min{|a|,|b|}` 단 `|a|+|b|>l`, 아니면 0. 여기서 `l`=sparsity index, `l=1`이면 순수 min-sum과 동치.
5. 의미: sparsity index로 boxplus의 감쇠(작은 신뢰도 합을 0으로)를 흉내내 SP 대비 저정밀 손실을 보상.
6. 확장: `l`을 정수 벡터 `[l0,...,l_{n-1}]`(description length n)로 일반화(식 22)해 체크노드 매핑족 확장, 부분순서로 우열 비교.
7. 구현: 체크노드 매핑을 LUT로 구현하면 description length가 커져도 복호속도 불변, VN은 결합법칙형(전체 입력합에 clip `g`)이 비결합형보다 성능 우수하나 중간합 저장 메모리 필요.
8. 최적화: 매핑 후보수가 적어(Table II) 전수탐색 가능, density evolution으로 목표 BER 10⁻⁴ SNR threshold 최소화하는 `l` 선택(Table IV).
9. 결과: rate-1/2·2/3 DVB-S2(len 64800) 4bit에서 FP SP 대비 waterfall 0.4/0.1dB 이내, 단블록(MacKay 204)에서는 고 SNR서 FP SP를 상회.
10. 한계: HW 미설계·시뮬만, BIAWGN 전제, error-floor는 정성 언급 수준.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 이산 boxplus 체크노드 연산 (min-sum 일반화) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min1/min2 기반 CN 연산을 sparsity index/LUT형으로 대체·보완 가능 |
| VN 정수합 clip 연산 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | VN 양자화·clip 로직과 직접 대응 |
| 비균일 채널 양자화 임계값 | channel.cpp Set_LLR_Th(), Set_R_Offset() | soft-read LLR 양자화·threshold 설계와 대응 |
| iteration별/양자화 LLR 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 양자화 레벨·매핑 테이블 관리 접점 |
| 체크노드 LUT 매핑 | 대응 없음 | Prime ECC는 min-sum 산술 기반, 명시적 CN LUT 매핑은 없음 |
```

적용 가치: 중간 — 저정밀 min-sum 변형·비균일 양자화는 우리 CNU/VNU/LLR 테이블에 닿는 관심 영역이나, 4bit+ soft 전제가 NAND 1~3bit read 예산과 상충해 재양자화·재최적화 검증이 선행되어야 한다.
