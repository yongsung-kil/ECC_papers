### B. 알고리즘 요약

1. 대상: 위성통신용 binary QC-LDPC 코드족, 고정 code length에서 가변 rate(예제 rate `2/9~11/12`, length `960/5616/16200`), AWGN+BPSK, sum-product float 복호(max iter 50).
2. 문제: RC-LDPC(가변길이)는 위성 ARQ 불가 환경에 부적합, 기존 MR-LDPC(RCEV/row-splitting/RCSEV)는 전rate 균일성능·PCM 구조(구현성) 고려 부족.
3. 모델: protograph 기반 base matrix `B=[b_{m,n}]` + exponent matrix `C`, lifting factor `L`로 CPM 확장.
4. 핵심기법1: 고rate mother 코드에서 shortening(정보열 제거)+extending(행/열 추가)을 결합해 length 고정·rate 하강, doubly-diagonal 유지로 저복잡도 인코딩.
5. 의미: nested 구조(`B_j`⊂`B_{j+1}`)로 코드족 전체가 단일 encoder/decoder 공유 가능.
6. 핵심기법2: Algorithm 1(progressive row elimination/addition)이 P-EXIT threshold 최소화와 cycle-4 수 최소화를 결합해 추가행 원소 탐색.
7. 구현 디테일: double-lifting(1차 `L1`로 parallel-edge 제거, 2차 `L2`로 CPM)으로 girth·min-distance 개선, base matrix sparsification(masking 일반화)으로 저rate 성능 향상(단 nested 성질 일부 훼손).
8. 최적화: exponent matrix는 GF(q) 순환계수 행렬 `T`(4-cycle free 보장)에서 Algorithm 2로 6-cycle 최소 행 선택.
9. 결과: DVB-S2X 대비 전rate에서 `0.2~0.8 dB` 우수, BLER `1e-5`까지 error-floor 없음, PPV bound와 `0.5~0.8 dB` 격차.
10. 한계: float sum-product 시뮬만, HW/RTL·gate count·throughput·soft 양자화 없음, 위성(고정길이 다중rate) 시나리오 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| base/exponent 행렬 기반 QC-LDPC 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 로드 위치에 대응하나 특정 고정 QC-LDPC라 다중rate 구성 이식은 재검증 부담 큼(§4 "높음") |
| doubly-diagonal 인코딩 | [encoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_encoder_4KB()` | Prime ECC도 dual-diagonal 인코딩 보유, 신규 이득 없음 |
| shortening+extending 다중rate | 대응 없음 | Prime ECC는 단일 고정 고rate 코드, 다중rate nested 요구 없음 |
| cycle-4/6 최소화·P-EXIT 부호설계 | 대응 없음 | 부호설계 방법론, 실행 디코더 모듈과 무관 |

적용 가치: **낮음** — binary QC-LDPC 부호설계라는 점만 겹치고, 다중rate·고정길이라는 위성 특화 목적이 단일 고정 고rate 코드를 쓰는 NAND Prime ECC와 맞지 않아 이식 가치 낮음.
