### B. 알고리즘 요약

1. 대상: 802.11n/802.16e/CCSDS QC-LDPC. PCM은 `Z×Z` 순환시프트 non-zero submatrix(NZS)들로 구성, base matrix `Hb`(`Mb×Nb`)가 위치 저장. 예: `(648,324)Z=27`~`(8176,7154)Z=511`.
2. 문제: FPGA LDPC 복호기의 RTL 개발이 오래 걸리고, 기존 HLS 설계는 성능/자원 균형이 나쁘거나 특정 부호 전용이라 범용성 부족.
3. 채널/부호: min-sum(MSA) 반복복호. 6bit 양자화(부호1/정수3/소수2). AWGN 등 채널 가정은 명시 없음(표준 부호 사용).
4. 핵심 기법 — MSA를 min/sub-min 탐색으로 변환: C2V `mci→vj = min|Lvb→ci|·∏sign`, comparison tree(4의 배수 입력, `log2X` 클록)로 pipeline 친화 구현.
5. 병렬화: C2V/V2C 병렬도 각각 `Mb`/`Nb`, 한 iteration을 `2×Z` 클록에 완료. pipeline directive로 II를 1클록까지 단축.
6. 압축저장: 반복 메시지를 `Mes[T×Z]`(NZS별 1행)로 압축, `Mlab`/`Pos`로 PCM 구조정보 저장하여 BRAM 절감.
7. 대각별 분산저장: 다중 대각 submatrix는 같은 배열에 두면 병렬접근 충돌 → 원 PCM을 `Q`개 단일대각 sub-PCM 중첩으로 보고 대각별 배열 `Mes(q)` 분리 저장.
8. 최적화: array partitioning(L/Out/Mes 1차원 완전분할), ping-pong으로 초기화·복호·출력 파이프라인화.
9. 결과: Kintex-7 330MHz, throughput 153~356 Mbps. [10] 대비 2~5배, [13] 대비 22배(자원 5배) throughput. Table II 3모드로 유연성 시연.
10. 한계: HLS라 RTL 대비 logic 사용량 많음; 다중 모드(802.11n 12모드 등) 지원 시 대형 배열 완전분할로 HLS 스케줄링에 수시간~하루 소요.
