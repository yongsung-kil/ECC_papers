### B. 알고리즘 요약

1. 대상: terminated SC-LDPC, 길이 `N=51328`, sub-block `c=640`, coupling width `ms=1`, rate `R=0.8`, H0/H1은 공개 DB에서 취득.
2. 풀려는 문제: unrolled full-parallel block 디코더는 Tbit/s급 throughput은 되지만 routing congestion으로 block size가 ~2000bit로 제한 → error-correction 성능 상한.
3. 핵심 아이디어: spatial coupling으로 큰 부호를 작은 block 부호에서 구성, sliding-window 복호로 window `W` sub-block만 저장(초기 latency·메모리 절감).
4. 기저 아키텍처: Full-Parallel Window Decoder(FPWD) - 각 processor가 sub-window(`W≥2ms+1`) 처리, target symbol의 VN만 갱신, pipeline 배치.
5. 핵심 기법1 Check Node Splitting: degree `dc` CN을 좌/우 부분 CN(`dc/2`)으로 분할해 매 iteration sign+min magnitude 교환 → 처리행렬이 `HMFPWD=[I H0 0; 0 H1 I]`로 축소, 교환메시지 `c·dv/2 → c-b`(R=0.8, dv=4에서 90% 감소), 최대 CN degree ≈ 절반.
6. 핵심 기법2 Iteration Slicing(CN-CN): iteration을 CN 최소값·sign 계산 후 시작하도록 재배치, 교환메시지를 CN 사이에서 주고받아 수렴 유지(Algorithm 1: exch_msg로 min0/min1/sign 갱신).
7. 프로세서 5-stage: UMU→CFUo→VFU→CFUi→exchange(min0+sign concat), R2L는 자기입력 피드백/L2R는 2단 앞 전파, first/last sub-block은 neutral/termination LLR로 초기화.
8. 양자화: channel·extrinsic 4bit, RLPD의 APP 6bit, NMS `γ=0.75`, floating point 대비 손실 <0.2dB(BER 1e-6).
9. 결과: 22nm FD-SOI, modified FPWD 7-iter, `525MHz`, `336Gbps`, area `1.33mm2`, 9.4pJ/bit, orig FPWD 대비 area util +33%·energy eff +40%, RLPD 대비 latency -15%.
10. 한계: 6G 무선 AWGN 타깃, 정정능력 개선 아닌 동일 성능 하 throughput/latency 최적화이며 SC-LDPC 전용 구조.
