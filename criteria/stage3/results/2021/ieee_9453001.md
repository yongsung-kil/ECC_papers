### B. 알고리즘 요약

1. 대상은 5G NR QC-LDPC(BG1 46x68, BG2 42x52), base matrix를 `z×z` circulant로 확장, 예시 (N,K,Z)=(576,288,24), rate 1/2·2/3·3/4·5/6, BPSK+AWGN, 20 iteration.
2. 풀려는 문제: 5G 고throughput 요구(2~200Gbps, <1ms)를 만족하는 인코더/디코더를 HLS로 빠르게 설계하고 자원/latency를 최적화.
3. 채널/모델: BPSK `2a-1`, AWGN, LLR `Li = 2ri/σ²`, Q함수로 BER 산정.
4. 인코딩 핵심: 생성행렬 대신 PCM 사용, base graph의 double-diagonal 부행렬 D 구조를 이용해 parity bit를 낮은 복잡도로 순차 계산.
5. 디코딩 핵심: soft-decision message-passing min-sum 근사, CN 연산은 `tanh(Iz/2)=∏tanh(Ix/2)`를 `sgn(Iz)=∏sgn(Ix)`와 min1/min2로 근사(식 7-9).
6. 핵심기법 - Layered Decoding: PCM을 layer(block row, z rows 단위)로 그룹핑, 한 layer 처리 후 즉시 column(APP) 갱신해 수렴 가속·iteration 감소. 이전 iteration의 layer 기여를 빼는 subtraction 단계가 안정성에 필수.
7. Offset 도입: min-sum에 scaling 대신 min에서 offset을 빼는 offset min-sum(OMS)으로 정확도 향상.
8. 구현: Vivado HLS pragma(Unroll/Pipeline/Interface) + array partitioning으로 partial-parallel·full-pipeline 구조, 한 클록에 row·column 동시 갱신.
9. 결과: xc7k160t에서 (576,288,24), 20 iter, SNR 1.5 기준 BRAM 104/LUT 52670/FF 36640, latency 4790 cycle(19.88us), peak throughput 2.113GB/s(기존 serial/layered 606MB/s 대비 향상), SNR>2에서 BER 10^-4.
10. 한계: 알고리즘 신규성 없음(layered OMS·double-diagonal은 표준 기법), gate count/공정·정정능력 개선 없음, HLS 자동합성 RTL 효율 검증 부족.
