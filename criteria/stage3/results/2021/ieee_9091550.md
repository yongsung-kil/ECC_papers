### B. 알고리즘 요약

1. 시스템: 3-D MLC/TLC NAND SSD, 4KB 주소매핑 단위, QC-LDPC로 4KB 데이터 보호(`q=1024`, column weight 4).
2. 문제: 오류 유무와 무관하게 모든 read 데이터가 LDPC 디코더로 몰려 디코딩 큐 병목 → read latency/전력 급증.
3. 관찰: 3-D NAND 초기 수명에는 error-free 페이지 비율이 매우 높음(4KB-subpage R-SWBE가 P/E<4001, RT<4주에서 90% 이상).
4. 핵심 기법: RBER과 error-free subpage 비율(R-SWBE)의 관계를 모델링(`R-SWBE-RBER model`)하여 RBER 임계값 `RBERth` 도출.
5. 의미: RBER은 저비용 계산 가능, R-SWBE는 실시간 측정 불가 → 모델로 RBER→R-SWBE 변환해 프리체킹 이득 구간 판정.
6. 동작: read 시 `RBER<RBERth` & flag=0이면 CRC-32 프리체킹 실행, 오류 없으면 LDPC 우회, 있으면 flag=1 세팅 후 LDPC 디코딩.
7. 서브페이지 스킴: 4KB를 2KB/1KB로 논리 분할하여 부분 오류 페이지에서 error-free subpage를 더 방출(84% 4KB→95% 1KB error-free).
8. PC 관리: CRC-32(HD=6)를 전용 PC-Block에 tree 구조 주소매핑으로 저장(OOB 미사용), 채널 분산으로 병렬화, hot PC를 RAM 캐시(LRU).
9. 결과: 실제 trace(SSDsim) 기준 3-D MLC read 성능 평균 34~55% 향상(R-SWBE 50~95%), WA/RA 오버헤드 미미(RA 14.2‰ 수준).
10. 한계: LDPC 디코더/부호 자체는 개선하지 않음. 수명 후반 RBER 상승 시 이득 급감(초기 수명 1-iteration 가정). HW 미설계, 시뮬만.
