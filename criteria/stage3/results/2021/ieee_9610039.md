### B. 알고리즘 요약

1. 시스템: 5G NR PUSCH/PDSCH용 LDPC를 FPGA로 가속하고 OpenAirInterface(OAI) 소프트웨어 5G 스택에 통합. base graph BG1/BG2, code rate `R=1/5~8/9`, code block `K=1280~8448`.
2. 문제: 소프트웨어 기반 Open RAN에서 LDPC 복호가 실시간 병목 → FPGA 오프로딩 필요하나 기존 가속기는 완전한 SW 5G 시스템에 통합·평가된 사례가 없음.
3. 핵심 HW: FPGA에 Xilinx LDPC IP Core + RIFFA(Gen2 x8 PCIe) 프레임워크를 배치하고, 커스텀 VHDL Encoder/Decoder Wrapper로 감싸 스트리밍 파이프라인 구성 (버퍼링 최소화, 외부 메모리 불사용).
4. 신규 모듈: `Pack/Un-Pack`으로 여러 code block을 단일 PCIe 전송에 묶어 대역폭·파이프라인 효율↑; 128-bit 헤더로 code block별 설정을 캐리해 서로 다른 config를 파이프라인.
5. CRC 오프로딩: Decoder/Encoder CRC 모듈이 CRC24A/24B/16을 8-bit 병렬로 계산·비교, SW CRC 대비 최대 76.8us 절감.
6. 데이터 포맷: OAI 8-bit LLR을 Xilinx 요구대로 6-bit saturation + sign invert (SIMD 최적화); 인코더 출력 bit-per-byte 재배열을 FPGA에서 수행.
7. 복호 알고리즘: 사용된 Xilinx IP Core는 normalized/offset min-sum을 구현하며 normalization factor `0.75`가 최적 (알고리즘 자체는 IP Core 내부, 논문이 설계·변경한 것 아님).
8. 검증: Virtex-7(VC707) 250MHz, on-chip power 5.897W. nr_dlsim/nr_ulsim 및 RF full-stack으로 기능 검증.
9. 결과: 단일코어 OAI 대비 최대 13.5x 스피드업, 최대 decoded throughput 1.65Gbps (34 CB, R=8/9, K=8448, 2 iter). R=8/9에서 min-sum OAI 대비 약 1.5dB BLER 이득 (IP Core 알고리즘 덕).
10. 한계: 디코더 IP는 블랙박스라 내부 알고리즘·부호설계 개선 없음; 인코딩은 계산량이 낮아 PCIe 오버헤드에 묻혀 가속 이득 미미. 5G NR 전용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Xilinx IP Core의 normalized/offset min-sum 복호 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 개념적으로만 대응 (우리도 min-sum 보유, 논문은 IP 블랙박스라 이식할 코드 없음) |
| CRC 기반 검증(코드블록 CRC 체크) | full_CRC.cpp | 조기종료가 아닌 5G 코딩체인 CRC라 목적 상이 |
| LLR 8→6bit saturation/sign invert | channel.cpp Set_LLR_Th() | soft-read 양자화 개념만 유사, NAND read 비용과 무관 |
| PCIe/RIFFA 스트리밍·code block packing | 대응 없음 | HW 통합·인터페이스 계층, Prime ECC 시뮬레이터와 무관 |
| 5G NR base graph 부호구조 로드 | ecc_top.cpp Load_PCM() | 부호구조 고정, 5G BG1/BG2는 우리 QC-LDPC와 상이 |
```

적용 가치: **낮음** — 디코더 알고리즘이 상용 블랙박스 IP이고 기여의 본질이 5G NR PCIe 오프로딩·시스템 통합이라, NAND binary QC-LDPC ECC에 떼어 쓸 요소가 없다.
