> 총평: OMS를 BP 언롤 신경망으로 학습하는 model-driven 디코더로 edge별 가산 offset·가중치 공유·codebook 양자화가 핵심, min-sum 계열이라 개념은 근접하나 오프라인 학습 인프라·short code 시뮬만이라 고rate NAND QC-LDPC 이식성은 낮음.

### B. 알고리즘 요약

1. 시스템: binary LDPC(576,432), rate `3/4`, AWGN/Rayleigh, BPSK/QPSK/QAM, CN=144·VN=576, `T` iteration을 `2T+2` layer로 언롤.
2. 문제: 데이터구동 DL 디코더는 블랙박스·긴 코드에서 복잡도 폭증. model-driven으로 Tanner 그래프를 layer로 펼쳐 성능·복잡도 개선.
3. 모델: OMS BP를 기반 — CN update `u_{c,v}=max(min|u_{v',c}|−β, 0)·∏sgn(...)`(식 3).
4. 핵심기법1(NOMS): 전역 단일 offset `β`를 edge·iteration별 학습 파라미터 `β_{c,v}^t`로 대체(식 6), ReLU를 활성화로 사용(식 7). NNMS[17]의 CN단 곱셈 `Σa_j`를 가산으로 바꿔 곱셈 절반 감소.
5. 식 의미: 학습 가산 offset이 min-sum 근사·H의 short-cycle 오차를 보정 → 동일 조건서 NNMS보다 BER 우수.
6. 핵심기법2(SNOMS): iteration 간 `w_{v,c}, w_v, β_{c,v}` 공유(식 12~14)로 파라미터 반감, `w=1`이면 OMS로 환원. NOMS보다 약간 성능 낮으나 복잡도 대폭↓.
7. 부수트릭: codebook 기반 weight 양자화 — float32/64를 q bit로, 빈도순 index 정렬해 c bit codebook(예 c=3)으로 압축(Fig.4).
8. 학습절차: cross-entropy 손실(식 10) minibatch SGD/Adam(lr=0.001), mini-batch=120.
9. 결과: 동일조건서 NOMS가 NNMS보다 0.65dB↑(5 iter), NNMS의 15 iter 성능을 NOMS 5 iter로 달성. 5 iter/12 layer로 BER 1e-7, 15 iter/32 layer로 ~1e-9. codebook(q=5,c=4)은 무양자화보다도 BER 우수하면서 저장공간 대폭↓.
10. 한계: HW 미설계·시뮬만, short code(576,432) 한정, gate/throughput 수치 없음, 오프라인 학습 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset Min-Sum CN 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 계열이라 offset 항 삽입 지점은 대응 가능 |
| VN 처리 / 가중치 | [decoder.cpp](../Prime_ECC_3.1_Claude) `VN_Cal_HD()` 등, `decoder.h` `Get_VNU_*` | VN scaling weight `w_v` 대응 위치 |
| iteration별 offset/weight 테이블 | [ecc_data.h](../Prime_ECC_3.1_Claude) `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 학습된 β/w를 iteration별 테이블에 매핑 가능 |
| codebook weight 양자화(메모리) | 대응 없음 | Prime은 magnitude 양자화 테이블 기반, codebook 압축 인프라 없음 |
| BP 언롤 model-driven NN 학습 | 대응 없음 | Prime ECC은 NN 학습 파이프라인 미보유(`local_opt.cpp` LLR 자동최적화와 성격 상이) |

적용 가치: **중간** — offset min-sum 코어와 iteration별 offset/weight 학습은 Prime의 min-sum·적응형 LLR 테이블과 접목 여지가 있으나, edge별 학습 offset과 codebook 양자화는 오프라인 학습·재검증 인프라가 필요하고 short code 시뮬뿐이라 고rate NAND QC-LDPC 즉시 이식은 어렵다.
