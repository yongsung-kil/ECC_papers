> 총평: SC-LDPC window decoder의 diversity 손실을 backward sweep으로 보완하는 연구로, block binary QC-LDPC 기반 Prime ECC에는 부호구조·스케줄링이 달라 이식성 낮음.

### B. 알고리즘 요약

1. 대상은 protograph 기반 SC-LDPC 부호(spatially coupled, terminated LDPC convolutional). 예시 앙상블 `(5,10,ms)` rate-1/2, lifting `M=300`, coupling length `L=100`, girth-6.
2. 문제: window decoder(WD)는 memory/latency를 blocksize와 무관하게 줄이지만, flood BP 대비 성능 손실이 있다.
3. 무한 길이: WD threshold `ρ(W,Imin)`가 flood threshold에 항상 수렴하지 않음. 비대칭 앙상블에서 `ρL > ρR`이면 `lim ρ(W) ≥ ρL > ρf`로 gap 잔존.
4. 원인 진단: flood BP는 forward+backward 두 decoding wave, WD는 forward wave 하나뿐 → 유한 길이에서 waterfall slope 감소(diversity gain 상실, "avalanche effect").
5. 간단 대책: 앙상블을 "revert"(`Bi = B_{ms-i}`)하면 `ρL`, `ρR` 교환으로 threshold 수렴 개선.
6. 핵심 기법: forward-backward WD(FB-WD) — WD forward run 실패 시(parity check로 판정) window를 backward로 재처리(second sweep).
7. 3변형: Full-FB(forward edge message 재사용, 메모리 flood급), Turbo-FB(channel+양쪽 extrinsic LLR 합산, channel/output LLR만 저장), Channel-FB(channel LLR만으로 backward, 최종결정 backward만).
8. 복잡도: `Z_FB ≈ (1+PW)·ZWD` (PW=WD의 WER), 평균 latency `≈ W + PW·L` — backward는 실패 시에만 활성화되어 평균 부담 작음.
9. 결과: Full-FB/Turbo-FB의 waterfall slope가 flood BP와 유사하게 회복, Channel-FB도 WD보다 개선. 기존 WSD[13] 대비 반대방향 wave 활용으로 slope 우위.
10. 한계: 시뮬레이션만(HW 미설계), SC-LDPC convolutional 부호 전용, gatecount/throughput 없음, Full-FB는 flood급 메모리 필요.
