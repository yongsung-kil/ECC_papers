### B. 알고리즘 요약

1. 시스템은 5G URLLC 같은 저지연 스트리밍이며, sub-frame `[n,k]=[64,32]`, coupling length `L=32`, BPSK/AWGN 채널.
2. 문제: 기존 LDPC-CC/SC-LDPC는 좋은 성능에 큰 window size(`d≥6(m+1)`)가 필요해 복호 지연이 크다.
3. 부호구조: parity-check가 banded block matrix로, sparse `H0=[P,I]`(PEG로 `wc=wr=2`)와 **dense** 서브행렬 `R`(Bernoulli(1/2)로 무작위 고정)의 결합 — 이 dense 블록이 conventional LDPC-CC와의 핵심 차이.
4. 인코딩: `v(t)=u(t)Pᵀ + c(t-1)Rᵀ`, register `D`로 이전 sub-frame 결합 (encoding memory `m=1`).
5. 복호 핵심: sliding-window list(SWL) 알고리즘. window size 고정 `d=2`(또는 3)로 지연을 `dn` bit로 억제.
6. 절차: sparse 부분 partial Tanner graph에서 BP(SPA) 수행 후 list decoding(REL 또는 OREL) → 후보 codeword 리스트 생성 후 soft metric `ΓSWL`으로 선택.
7. list 순서화: flipping pattern tree(FPT)로 flipping pattern을 soft weight 순 정렬 → FPT-REL / FPT-OREL(OSD 유사).
8. 성능분석: first-event error 기반 upper bound `≤2·fER0`, underlying LDPC-BC의 GA(genie-aided) lower bound 제시.
9. 복잡도 절감: two-round serial 구현 + 조기종료(soft metric 최대·차순위 차이 `δ`에 threshold `T`) → 평균 list size 10배 이상 감소.
10. 결과: 동일 latency에서 (3,6)-regular SC-LDPC 대비 최대 1.4dB coding gain(`fER=10⁻⁴`), max iteration은 5(SC-LDPC 50 대비 소수). HW 미설계, 이론+시뮬만.
