### B. 알고리즘 요약

1. 시스템: M×N ReRAM crossbar array에 LDPC 부호어(`n=495, k=433, rate=0.8747`, Mackay 부호, col weight 3)를 `T`개 어레이에 분산 저장.
2. 문제: crossbar의 sneak-path interference(SPI)는 data-dependent이고 array 내에서 상관됨 → 인접 부호비트 채널이 상관되어 BP 복호 성능 저하.
3. 채널모델: SPI(이산 비대칭 채널)와 저항변동(lognormal mixture)을 직렬 연결한 cascaded 채널 모델 제안.
4. 핵심기법1: cell별 sneak-path 발생확률 `π_{i,j}`를 같은 행·열의 hard-decision 통계 `n_{R0'}/(n_{R0}+n_{R0'})`로 추정 (기존 [6] ESE는 array 전체 평균).
5. 핵심식: LLR `L(x_{i,j}) = log[ π f(y,μ0,σ0) + (1-π) f(y,μ0',σ0') ] / f(y,μ1,σ1)` - 셀별 SPI 확률을 반영해 더 정확한 연판정 정보 생성.
6. 핵심기법2: 2단 어크로스-어레이 비트할당 - Step1은 Tanner graph neighborhood overlapping `M(v_p, A_t)` 최소화로 한 CN에 연결된 VN들을 다른 array에 배치.
7. 구현디테일: Step2는 dispersion matrix `π_l`(cyclic shift)로 array 내에서 같은 부호어 비트를 다른 행·열로 재배치.
8. 최적화: 비트할당은 offline 설계 후 online에서는 단순 인터리버로 1회 구현 (추가 latency 미미).
9. 결과: SPA/min-sum 복호에서 어크로스-어레이 미적용 및 [6] 대비 유의한 BER 개선, SPI 추정 복잡도는 [6] 대비 약 5% 증가에 그침.
10. 한계: HW 미설계·시뮬만, ReRAM crossbar 전용 채널/할당 기법이라 NAND flash 채널·QC-LDPC 부호설계·디코더 코어와 무관.
