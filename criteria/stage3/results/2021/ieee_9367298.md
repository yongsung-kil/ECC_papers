### B. 알고리즘 요약

1. 시스템: AWGN(및 Rayleigh) 채널에서 rate-1/2, length-4800 protograph LDPC(PLDPC) coded BICM-ID, 8-PSK/16-QAM/64-QAM 고차변조, 복호는 Max-log-Map demapper + BP decoder, outer iter `Tout=8`/inner `Tin=25`.
2. 문제: Gray 매핑은 BICM엔 최적이나 BICM-ID의 반복이득이 작음; IM 스킴은 두 성상을 따로 설계해 매칭이 불완전.
3. 핵심 기법1(AIM): 3-step 전략으로 초기성상에서 label pair `{x_i, x_(M+1-i)}`를 만들고 target labeling bit `m'`를 유지, 나머지 bit를 Hamming distance 최대/인접 최소 규칙으로 채워 adaptive 성상 생성(비feedback HMMSED 최대 + feedback HMMSED 큰).
4. 핵심 식: HMMSED `Dh2(χ)`(비feedback)와 `Dh2~(χ)`(feedback), offset gain `Gh=-10log10(Dh2/Dh2~)`로 성상의 ID/non-ID 성능을 정량화.
5. 핵심 기법2(AIM-PEXIT): demapper-decoder 간 MI 진화를 추적하는 수정 PEXIT로 decoding threshold(min Eb/N0) 산출; RM과 달리 nv개 `IE,Dem(j)`/`IA,Dem(j)`를 동시 갱신, `J(·)`/`J^-1(·)` 함수와 VN/CN MI 갱신식 (7)(8) 사용.
6. 핵심 기법3(부호설계): base matrix `Binit`의 앞 nv/2 VN을 degree-2로 고정, 나머지 원소를 `b_ij∈{0,1,2,3}`로 systematic enumeration하되 총 edge 수를 원본과 동일하게 유지(동일 복잡도) → threshold 최소 base matrix를 enhanced code로 선택.
7. 부수: degree-2 VN은 초기성상, 나머지 VN은 adaptive 성상으로 매핑(UEP 성격).
8. 최적화 절차: AIM-PEXIT를 목적함수로 하는 system-search로 enhanced (3,6)/RJA base matrix 도출 (예: Ben_(3,6)=[2 4]).
9. 결과: AIM(Gray)가 Gray 대비 ~0.1~0.2dB, AIM(MSEW)/AIM(anti-Gray)는 ~1~1.4dB@BER=1e-5 이득; enhanced code는 원본 대비 ~0.3~0.4dB 추가 이득; Rayleigh에서도 유사.
10. 한계: HW 미설계, 시뮬만, 무선 고차변조(BICM-ID) 전제라 NAND hard/soft-read binary LDPC와 채널·매핑 구조가 상이.
