### B. 알고리즘 요약

1. 서베이 대상은 HDPC(BCH(63,36)), protograph LDPC((800,160), Z=16, rate 0.2), polar((256,128)/(8,4)) 등 무선/5G 선형부호.
2. 문제: 표준 BP/MS는 Tanner graph의 short cycle로 성능 저하, data-driven NN은 codebook `2^k` 학습이라 초단부호만 가능.
3. DL 디코딩을 data-driven(full-connected NN)과 model-driven(뉴럴 BP/MS/SC)으로 분류.
4. 핵심 기법: model-driven은 factor graph edge마다 학습가능 weight `w_i,e`/offset `β_i,e`를 부여해 표준 BP/MS를 확장(NBP/NNMS/NOMS).
5. NNMS `x = w·min|x|·∏sign`, NOMS `x = ReLU(min|x|-β)·∏sign` — weight/offset이 min-sum 근사오차·short cycle 영향을 보상.
6. protograph LDPC는 lifting 구조로 base graph edge 단위 파라미터 공유(`E=Z·E_b`) → 파라미터·학습복잡도 대폭 감소.
7. 학습은 all-zero codeword만으로 가능(sparse·structural), cross-entropy loss + SGD.
8. polar은 ResNet-like BP(damping factor `α`)와 뉴럴 SC(`g,f`에 weight `w`) — long BP의 gradient vanishing 완화.
9. 결과: 뉴럴 MS/BP가 표준 대비 BER/BLER 개선(BCH·(800,160)), ResNet BP T=10이 표준 BP T=40 수준.
10. 한계: 순수 시뮬(HW 미설계), NN 학습·파라미터 오버헤드, 무선 AWGN 대상이라 NAND read/양자화 비용 미고려.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 뉴럴 MS/BP 디코더(학습 weight/offset) | decoder.cpp LDPC_Decoder(), CNU_Update_New_Mag() | Prime은 min-sum 이미 보유, 학습 weight 층은 대응 없음 |
| iteration별 학습 offset/normalize | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 적응형 LLR 테이블과 개념 유사, 학습 파라미터화는 무관 |
| NN 학습기반 디코더(NBP/NSC) | 대응 없음 | Prime은 bit-exact HW 모사, 딥러닝 디코더 대응 없음 |
| protograph lifting 파라미터공유 | ecc_top.cpp Load_PCM() | QC lifting과 유사하나 부호구조 재설계로 이식 부담 큼 |
```

적용 가치: **낮음** — model-driven 뉴럴 MS의 학습 weight/offset을 고정 스케일로 근사한다면 개념은 참고 가능하나, NN 학습 인프라 의존·HW 미설계·무선 대상이라 Prime의 HW binary QC-LDPC ECC에 직접 이식 실익 없음.
