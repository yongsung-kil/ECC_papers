### B. 알고리즘 요약

1. 시스템: FSO 링크에 regular PEG LDPC `(1008,504)` (rate 0.5) + OOK 변조, 수신 `r(t)=y(t)ηI+n(t)`, AWGN 배경잡음.
2. 문제: FSO 대기 난류(scintillation)로 성능 저하 — FEC로 완화, 기존 연구는 고복잡도 soft-decision BP만 사용.
3. 채널 모델: 약~중 난류=log-normal PDF(식 3), 중~강 난류=Gamma-Gamma PDF(Meijer-G, 식 4), 여기에 weather attenuation·pointing error 결합.
4. 핵심 기법 1(WBF): 각 CN에서 최소 soft magnitude VN `|y_nmin|` 식별, error term `En = Σ_m (2sm-1)|y_nmin|`(식 13)로 flip 대상 결정 (hard-decision).
5. 식 의미: `En`은 신뢰도 가중 checksum — 큰 값일수록 해당 비트가 오류일 가능성 높아 flip.
6. 핵심 기법 2(IERRWBF): reliability ratio `Rmn=Υ|yn|/|ynmax|`(식 15) 기반 RRWBF를 단순화, `Tm=Σ|yn|`, `En=Σ_m (1/|yn|)(2sm-1)Tm`(식 18~19)로 처리시간 절감형.
7. 인코딩: H를 근사 하삼각(approximate lower-triangular)으로 변환(Richardson-Urbanke), back-substitution으로 parity 계산, `~n²` XOR.
8. 최적화 절차: 별도 학습 없음, 파라미터는 문헌 인용값 사용, target BER 1e-6, λ=1550nm, L=1000m.
9. 결과: uncoded 대비 강난류에서 IERRWBF가 ~10dB Eb/No 이득, 약/중 난류에서 WBF보다 반복 수 적고 수렴 빠름. 단 BP가 BER는 여전히 최고(고복잡).
10. 한계: FSO 채널 특화, HW 미설계·Matlab 시뮬만, IERRWBF는 WBF보다 decoding time↑(latency 악화), throughput 이점 미미.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| WBF/IERRWBF 반복 flip 루프 | decoder.cpp LDPC_Decoder() | 이론상 이터레이션 루프 자리이나 Prime은 min-sum 보유 — WBF는 성능 열세라 대체 이유 없음 |
| CN error term / reliability ratio | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum과 다른 계열, 이식 시 성능 하락 위험 |
| hard-decision VN 처리 | decoder.cpp VN_Cal_HD() | HD 지원은 이미 있음(중복 기여) |
| 인코딩(approximate lower-triangular) | encoder.cpp (Prime은 dual-diagonal 고정) | 대응 방식 상이, 이식 이점 없음 |
```

적용 가치: **낮음** — WBF/IERRWBF는 교과서 hard-decision bit-flipping으로 Prime ECC가 이미 보유한 min-sum보다 BER 열세이고, 채널도 FSO 특화라 NAND binary QC-LDPC에 떼어 쓸 이유가 없음.
