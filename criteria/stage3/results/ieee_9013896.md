### B. 알고리즘 요약

1. 시스템: MLC NAND, LSB/MSB 별도 페이지, rate 8/9 binary LDPC(14400->16200), protograph 4x36 기반 CPEG+ACE 2단 lifting(x3, x150).
2. 문제: 기존 enhanced-precision 연구[2,3]는 모든 임계값을 항상 읽는다 가정, 실제 progressive(실패 시에만 추가 read) 특성 미반영.
3. 채널 모델: input-dependent time-varying Gaussian `Y=X+N(t,X)`, erased state 분산이 크고 P/E 증가 시 분산↑·retention 전압↓.
4. 핵심 기법: 각 progressive read 임계값 `T_left/T_right`를 조건부 MI `I(X;Y_prog|Y_initial)` 최대화로 결정, Newton법으로 최적화.
5. 핵심 식: chain rule로 MI를 initial read 항 + progressive read 항으로 분해 (식 6-9), 각 조건부 항이 독립 최적화로 decouple됨.
6. 확장: MSB 페이지는 4개 progressive read를 3개 독립 최적화 문제로 분리(식 12-14), TLC/QLC로 확장해도 동시 최적화는 최대 2개 임계값뿐.
7. 순서화: 각 read의 MI 기여도(I1>I2>...)로 progressive read 수행 순서 결정 (Fig.3).
8. 학습/최적화: Newton법 기반 임계값 1D/2D 최적화 (NN·GA 없음).
9. 결과: FER가 MI 예측과 일치(Fig.6-8); joint encoding의 MI 이점은 hard에서 작고 progressive read 시 무시가능(APL Indep 1618 vs Joint 1609).
10. 한계: 디코더/HW 미설계, BP 20 iter 표준 사용, 단순 Gaussian 채널·순수 시뮬, 저지연 요구 FER까지 미도달.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| progressive read threshold 최적화 | channel.cpp Set_R_Offset(), Set_LLR_Th() | soft-read threshold 배치 로직에 대응, 순서화 개념 이식 여지 |
| soft-info 양자화(2/3 read) | channel.cpp + ecc_data.h PARAM_LLR | 2SD/3SD LLR 테이블과 연결 가능 |
| LDPC 복호(BP 20 iter) | decoder.cpp LDPC_Decoder() | 논문은 표준 BP 사용, 디코더 개선 기여 없음 |
| 부호 구조(protograph 8/9) | ecc_top.cpp Load_PCM() | 특정 외부 부호, Prime QC-LDPC와 상이 |
```

적용 가치: 중간 — NAND soft-read threshold 순서/배치 최적화 개념은 `channel.cpp`의 LLR threshold 설정에 접목 여지가 있으나, 디코더·부호·HW에는 직접 기여가 없고 채널 전처리 레이어에 한정됨.
