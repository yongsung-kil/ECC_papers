### B. 알고리즘 요약

1. 대상은 protograph 기반 terminated LDPC Convolutional Code(LDPCCC), rate `RL=1-(1+ms/L)(1-R)`, 예제 rate 14/32(A,C)·15/32(B), M=512 lifting, L=16/50 종단.
2. 문제: 큰 blocklength LDPC는 복호 latency가 커 packet-level coding에 부적합 - convolutional 구조의 sliding window 복호로 latency-성능 trade-off 확보.
3. 모델: BEC 소거채널, syndrome former memory `ms`로 (ms+1) 이상 떨어진 VN은 동일 parity check에 무관 - window로 국소 복호 가능.
4. 핵심기법: `Jw=WJ'M` 행 × `Kw=WK'M` 열 슬라이딩 윈도우가 targeted symbol(첫 c 심볼)만 복호 후 (c-b)행·c열 이동, latency `δw=WcTs`로 전체 BP 대비 `w` 비율 축소.
5. 핵심식: windowed-threshold `εW`를 targeted symbol 복호성공 상한 소거율로 정의 - 전체 codeword 아닌 targeted symbol 기준.
6. 확장: Protograph-EXIT로 CN 갱신 `Iout=∏Iin,i` (식4), VN 갱신 `Iout=1-I0∏(1-Iin,i)` (식5)로 targeted VN의 사후 상호정보 수렴(→1) 추적.
7. 트릭: Lemma 1,2 단조성으로 fixed-point I 상·하한(Prop 1,2) 유도, 유한 computation tree만으로 zero-threshold 증명.
8. 핵심 발견: multiedge 없는 protograph에서 한 CN이 degree-1과 degree-2 VN 둘에 연결되면 항상 zero windowed-threshold - 3-6 regular LDPCCC(ms=2)가 이 bad structure로 실패.
9. 결과: 이를 회피한 Ensemble B(ms=1, Bo=[2 2],B1=[1 1]), C(4-8, ms=2)는 작은 window에서도 BP threshold(0.488/0.496) 근접, W≥5면 SER이 BP와 거의 동일(시뮬 일치).
10. 한계: BEC 전용 점근 threshold 해석·SER 시뮬뿐, HW 미설계, LDPC convolutional code로 NAND block QC-LDPC와 구조 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 슬라이딩 윈도우 소거복호 (BEC) | 대응 없음 | Prime ECC는 block QC-LDPC min-sum, convolutional windowed 복호 미사용 |
| protograph LDPCCC 부호설계 (bad structure 회피) | `ecc_top.cpp` `Load_PCM()` (원리만 간접) | H-matrix 로드 위치이나 convolutional 구조·BEC 대상이라 직접 이식 불가 |
| Protograph-EXIT threshold 해석 | 대응 없음 | AWGN/RBER 채널·min-sum 대상 아님 |

적용 가치: **낮음** - BEC 소거채널 + LDPC convolutional code라는 이중 mismatch로, NAND block binary QC-LDPC의 min-sum 디코더·부호설계에 떼어 쓸 요소가 없다.
