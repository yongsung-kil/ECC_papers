> 총평: NAND 스케일링·3D·하이브리드 SSD 서베이 안에 EP-LDPC ECC가 포함되어 있으며, 3 Vref만으로 read latency를 7배 줄이는 soft-read 비용 절감 아이디어가 NAND ECC에 직접 관련되나 부호/디코더 세부가 없어 개념 참조 수준.

### B. 알고리즘 요약

1. 대상은 2D/3D NAND(1Xnm, MLC/TLC/QLC) 신뢰성으로, ECC 절은 FG-FG capacitive coupling에 의한 셀 열화 대응.
2. 문제: design rule 축소로 inter-cell coupling이 심해지고, 기존 soft-decoding LDPC는 정확한 VTH 정보를 위해 21개 Vref가 필요 → 21×50µs = 1050µs read(3b/4b/cell은 2~5배)로 read latency 과대.
3. 핵심 기법(EP-LDPC): 단 3개의 Vref만 읽고, VTH 정보에 더해 inter-cell coupling, write/erase cycle `Nw/e`, retention time `Tretention`을 함께 이용해 셀 상태를 예측·보정.
4. 즉, 부족한 soft-read 해상도(3 Vref)를 error-prediction 모델(EP table)로 보완해 실질 LLR 정확도를 확보하는 방식.
5. HW: NAND 컨트롤러 내 error prediction sequencer + EP/Nw/e/Tretention table + error mode detector + error recovery sequencer를 단순 로직으로 구현, 추가 면적은 무시할 수준.
6. 인코더 측: initial data의 "1" 개수 `N"1"initial` 카운터 값을 user data에 부가(에러 모드 검출용).
7. read flow: LDPC decode 실패 시 error mode detect → error recovery pulse 적용(ER scheme).
8. 결과: 기존 LDPC 대비 read 7배 빠름, over 10배 수명 연장.
9. (논문 나머지: bulk/SOI NAND 스케일 한계 13nm/8nm, 3D BiCS 18층 40nm pitch=15nm planar, ReRAM/NAND 하이브리드 SSD가 성능 11배·write energy 93%↓·endurance 6.9배 — ECC와 무관한 device/system 절.)
10. 한계: invited 서베이라 LDPC 부호종류·rate·length·복호 알고리즘·검증 방식이 본문에 없고 EP 세부는 참조문헌 [4,5]에 있음.
