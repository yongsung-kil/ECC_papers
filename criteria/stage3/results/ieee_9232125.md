> 총평: 4-cycle-free 제약 구성 + 6-cycle 가중치 최소화(OCCE)로 QC-LDPC를 설계해 동일 code length 대비 반복 횟수를 최대 76% 줄이고, RS 외부코드 결합으로 error-floor를 억제한 부호설계 논문.

### B. 알고리즘 요약

1. NAND flash MLC 채널(AWGN 근사, 4-PAM 변조)에서 RS 외부코드 + QC-LDPC 내부코드의 concatenated 구조를 제안.
2. 기존 QC-LDPC-PEG는 BER은 좋으나 반복횟수가 많아(최대 50회) flash 지연에 불리 - 반복수 감소가 목표.
3. QC-LDPC 검사행렬 `Hqc`를 인덱스행렬 `P`로 표현(`p_i,j = a^(j-1) b^(i-1)`)하고 소수 L에서 `(bf-1)(ad-1) ≠ 0 mod L` 조건으로 4-cycle 배제(기존 [9][10] 기법).
4. 행렬을 `H = [H1 | Hp]`로 분리, `Hp`는 quasi-double-diagonal 구조(인덱스 `Px`, shift 값 `x`)로 설계해 특이행렬(singular) 문제와 row correlation을 동시 해결.
5. `H1`은 4/6-cycle 상관수 `C4(i,j)`, `C6(i,j)`의 가중합을 계산해 최대 가중치 노드를 우선 shift하는 OCCE(optimized computation cycle elimination) 알고리즘으로 구성 - 임계값 τ 도달 시 반복 조기종료.
6. 기존 CE 알고리즘([11]) 대비 "요소별 순차 수정" 대신 "최대 가중치 우선 수정"으로 신규 4-cycle 재발생을 방지하고 계산량을 낮춤.
7. RS-QC concatenation 설계 원칙 3가지 제시: RS는 고code rate 사용, RS와 QC-LDPC의 rate를 근접시킴, RS와 QC-LDPC의 code length를 정합(불일치 시 zero-padding으로 노이즈 에너지 증가).
8. 별도 학습/최적화(DE/GA/NN) 없음 - 결정론적 파라미터 서치(a, b, x, τ) 기반 구성.
9. QC-LDPC(8610,8190)이 QC-PEG 대비 BER 10^-8에서 반복수 12 vs 50(76% 감소); RS(273,269)-QC는 SNR 3.9dB에서 BER 10^-10 달성, error-floor 미관측.
10. HW 미설계(RTL/게이트 없음), MATLAB 시뮬 및 AWGN 채널 가정뿐, 실제 flash 실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 4-cycle 배제 조건 기반 QC-LDPC H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체 재설계이므로 신규 PCM 생성/검증 로직 필요, 기존 로더는 로드만 담당 |
| H1/Hp 분리 설계(quasi-double-diagonal) | `encoder.cpp` `Generate_PCM_encoding()` | 인코딩용 dual-diagonal 구조와 개념적으로 유사(특이행렬 회피 목적) |
| OCCE 6-cycle 최소화 알고리즘 | 대응 없음 | 오프라인 부호설계 단계 도구로 실시간 디코더/인코더 모듈과 무관 |
| RS 외부코드 concatenation | 대응 없음 | Prime ECC는 binary QC-LDPC 단일 계층, RS 외부코드 계층 미보유 |
| min-sum 디코더(반복횟수 측정용) | `decoder.cpp` `LDPC_Decoder()` | 논문도 min-sum 사용 - 알고리즘 자체는 동일 계열이나 개선점 없음(단순 벤치마크 용도) |

> 적용 가치: 중간 - 부호설계(H-matrix) 재구성은 이식난이도가 "높음"(프로파일 §4)이라 코드 이식보다는 설계 방법론 참고용. 다만 4/6-cycle 억제로 반복수 76% 감소는 고rate NAND QC-LDPC에 실질적 가치가 있어 부호설계팀 참고자료로는 유효.
