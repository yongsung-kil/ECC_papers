### B. 알고리즘 요약

1. 시스템: 3GPP 5G NR LDPC 코드(TS 38.212), base graph1(46×68, mother rate 1/3)과 base graph2(42×52, mother rate 1/5), lifting size Z=128, layered(serial) 스케줄링 사용.
2. 문제: MSA(min-sum)는 BPA(`φ(x)=-log(tanh(x/2))`) 연산을 min 탐색으로 근사해 복잡도를 낮추지만 근사오차로 BER/FER 성능이 열화됨.
3. 가정: AWGN 채널, early-stop(syndrome check), 최대 40회 iteration.
4. 핵심 기법: CN 업데이트 시 인접 VN 중 최소 3개 값 `v1≤v2≤v3`과 인덱스(`vidx1,2,3`)를 추출해, 자기 자신을 제외한 나머지 두 값으로 `R_cv`를 계산(식 10~13).
5. 핵심 식: `f(x,y)=(x*y)/(x+y)` (조화평균의 절반, 식 9) - 기존 MSA가 `min(v1,v2)` 한 값만 쓰는 것과 달리 두 최소값을 모두 반영해 근사 오차를 축소.
6. 확장: 고SNR 영역 성능 보완을 위해 제안 CN 업데이트 결과에 정규화 계수 `α`를 추가로 곱함(식 16, NMSA와 결합).
7. 구현 디테일: 부호별 최적 `α`가 다름(BG1: α=1.1, NMSA 비교군 α=0.65) - 정규화 없이는 BG1/BG2 모두 고SNR에서 FER 열화 관찰.
8. 최적화 절차: 정규화 계수는 시뮬레이션으로 탐색한 고정값 사용(적응형/온라인 최적화 없음).
9. 결과: MSA 대비 FER 약 1.6dB 개선(BG1), 최적 정규화 NMSA 대비 평균 iteration 수 약 10% 감소.
10. 한계: HW 구현/gatecount/throughput 분석 없음(순수 알고리즘 시뮬레이션), 3rd-min 탐색과 나눗셈 연산의 HW 비용 미평가, BG2(저rate)는 정규화 없이는 성능 열화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum 근사오차 개선(3-min 기반 f(x,y)) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min1/min2 기반 CNU 로직에 3rd min 탐색과 조화평균형 근사식 추가하는 형태로 대응 가능 |
| 정규화 계수(α) 적용 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 (LLR 스케일링) | 기존 magnitude 양자화 테이블에 정규화 반영 여지 |
| iteration별 LLR 테이블/정규화 값 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 부호(BG1/BG2 상당)별 최적 α가 다르므로 테이블화 필요 |
| 5G NR base graph / H-matrix 구조 | `ecc_top.cpp` `Load_PCM()` | 논문은 5G NR 특정 부호이며 Prime ECC는 자체 QC-LDPC 고정 - 알고리즘만 분리 이식 대상 |
```

적용 가치: 중간 - CN 업데이트 근사식 자체는 `decoder.cpp`의 min-sum 루프에 이식 낮은 난이도로 적용 가능하나(§4 이식표 "디코더 알고리즘"=낮음 난이도), 3rd-min 탐색기 추가에 따른 HW 비용 검증이 없어 실이식 전 별도 HW 영향 분석이 필요함.
