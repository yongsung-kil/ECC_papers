> 총평: 표준 BP의 parity-check 단계만 활용한 저비용 조기종료+노드 비활성화 결합 기법으로 iteration/노드연산 절감 효과가 크지만, HW 검증이 없고 Prime ECC의 기존 조기종료·스케줄링 기법과 개념적으로 겹칠 가능성이 있음.

### B. 알고리즘 요약

1. IEEE 802.16e (2304, 1152) 코드(rate 1/2), AWGN 채널에서 표준 BP(belief propagation) 디코더를 대상으로 한다.
2. 기존 조기종료 기법은 평균 iteration 수만 줄이고 iteration 내 모든 노드 연산은 그대로 수행한다는 한계, 그리고 고정 threshold는 SNR별 성능 열화를 유발한다는 문제를 다룬다.
3. 채널/노이즈 가정은 표준 AWGN, 별도 특수 모델 없음.
4. 핵심 기법 1단(적응형 threshold 조기종료): 매 iteration의 만족된 parity check 수 `Nspc = M - Σsi`를 측정, `Nspc`가 `Tmin` 미만이면 `T=M`(항상 계속), `Tmax` 초과면 `T=0`(중단 비활성화), 그 사이면 `T = M - (Nspc-Tmin)×d`로 threshold를 적응 조정.
5. `Nspc`가 클수록(코드워드가 덜 손상됨) 낮은 threshold를 적용해 과도한 지속 디코딩을 방지 - 채널조건 의존성을 제거하는 것이 핵심 의도. (2304,1152) 코드에서 `Tmin=625, Tmax=675, d=6` 사용.
6. 핵심 기법 2단(노드 비활성화, dual-syndrome): `u` iteration마다 전체 노드를 reliable로 재초기화(Step1), 최근 2개 iteration(`i`,`i-1`) 모두에서 unsatisfied인 parity check에 연결된 bit/check node를 unreliable로 표시(Step2, dual-syndrome), unreliable bit node에 연결된 check node는 reliable이어도 unreliable로 correction(Step3) - 이후 iteration에서 reliable 노드는 비활성화되어 연산 생략.
7. 부수 구현 디테일: 두 기법 모두 표준 BP의 내재적 parity-check 단계만 활용해 counter/comparator만으로 구현 가능하다고 서술(HW 합성은 없음).
8. 별도 학습 절차 없음 - `Tmin, Tmax, d, u` 등은 시뮬레이션으로 결정되는 고정 파라미터.
9. 결과: 표준 BP 대비 최대 80.06% 노드 연산 절감(SNR 0~2.5dB), adaptive threshold 단독 대비 node deactivation 추가로 최대 37.85% 추가 절감; dual-syndrome 방식이 single-syndrome 대비 BER 열화 없이 유사한 연산 절감 달성.
10. 한계: 실제 HW 합성/gatecount/throughput 결과 없음(시뮬레이션뿐), 단일 코드·단일 채널 모델만 검증, 저SNR에서 우수하나 고SNR에서는 타 기법([10])이 더 나음(향후 결합 필요성 언급).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 적응형 threshold 조기종료(Nspc 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 아이디어 자체는 Partial/Full CRC와 유사 목적이나, syndrome count 기반 adaptive threshold는 CRC 방식과 다른 별도 메커니즘 |
| Dual-syndrome 노드 비활성화(선택적 연산 스킵) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 노드별 선택적 업데이트로 연산량 감소 - min-sum 이터레이션 루프에 조건부 스킵 로직 추가하는 형태로 참고 가능 |
| iteration별 적응형 threshold 파라미터(Tmin/Tmax/d) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR threshold 테이블 구조와 유사한 파라미터 테이블 개념 |

> 적용 가치: 중간 — 조기종료·선택적 노드 업데이트라는 목표는 Prime ECC의 Partial CRC/적응형 LLR과 개념적으로 겹치나, syndrome-count 기반 adaptive threshold와 dual-syndrome deactivation은 저비용 대안으로 참고할 가치가 있음. HW 실증 부재가 리스크.
