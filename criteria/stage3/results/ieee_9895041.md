### B. 알고리즘 요약

1. 시스템: AR4JA(CCSDS) protograph LDPC, rate `R=0.5`, data length 1024, lifting factor `z=256`, AWGN 채널, 10 iteration.
2. 문제: BP의 CN 연산(`tanh`/`atanh`)이 복잡도·메모리를 크게 요구 → LUT 기반 finite-alphabet 디코더로 대체.
3. 모델: 채널 출력 LLR `L_ch`를 q-bit 정수 `L_ch^q ∈ {0,...,2^q-1}`로 양자화하는 quantizer `Ψ(·)` 정의.
4. 핵심 기법: CN/VN 연산을 2-input LUT `ϕ(L1,L2)`의 순차 합성 `Φ`로 표현 (식 3~7). CN은 `f_cn=-2atanh(tanh(L1/2)tanh(L2/2))`, VN은 `f_vn=L1+L2`.
5. 핵심 식: 각 노드 연산 = 2-input 연산 `U=f(L1,L2)` 후 양자화 `L=Ψ(U)`를 합쳐 `ξ(L1,L2)`로 표현(식 12,13) — 정수 in/out 매핑을 LUT로 굳힘.
6. 확장: protograph 특유의 parallel edge, degree-1 VN, punctured VN을 LUT 설계에 반영(기존 degree-distribution 변환 없이 protograph 직접 처리).
7. 구현 트릭: parallel edge는 동일 LUT 재사용; degree-1 VN은 채널 LLR을 LUT 없이 CN에 직접 전달; punctured VN은 관련 CN/VN LUT 자체를 제거(항상 0 메시지).
8. 최적화: protograph density evolution으로 error prob `p_e`(식 19)를 cost로, differential evolution으로 채널·CN·VN quantizer boundary `{r_ch,r_CN,r_VN}` 동시 최적화(식 20).
9. 결과: BER=10^-4에서 BP 대비 coding loss가 2/3/4-bit LUT에서 각각 1.6/0.5/0.1 dB → 4-bit면 BP 근접.
10. 한계: HW 미설계, gate/throughput/iteration 수치 없음, 저rate(0.5) CCSDS 코드만 검증, error-floor 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LUT 기반 CN 연산 (tanh 대체) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime은 min-sum. LUT는 별개 근사이나 CN 연산 대체 위치는 동일 |
| VN 양자화 / finite LLR 처리 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | q-bit 정수 LLR 처리 지점과 대응 |
| 채널 quantizer / LLR 양자화 | channel.cpp Set_LLR_Th(), Set_R_Offset() | 채널 출력→정수 LLR 경계 최적화가 soft-read threshold와 직결 |
| iteration별 LLR 테이블 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | LUT를 iteration별 테이블로 굳히는 구조와 유사 |
| protograph/parallel-edge 부호구조 | ecc_top.cpp Load_PCM() | Prime은 특정 QC-LDPC 고정 — protograph 부호설계는 이식 부담 |
```

적용 가치: **중간** — LUT/양자화 quantizer 최적화(soft-2~3bit read 비용 절감) 아이디어는 channel.cpp/LLR 테이블에 참고할 만하나, 저rate protograph·HW 미설계라 Prime의 고rate min-sum 엔진에 직접 이득으로 옮기기엔 검증 부담이 크다.
