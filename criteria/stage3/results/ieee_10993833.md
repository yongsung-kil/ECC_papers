### B. 알고리즘 요약

1. 시스템: 위성간 optical line-of-sight 통신, SPAD(Single-Photon Avalanche Diode) 기반 광자 카운팅 수신, laser burst on/off로 비트 표현.
2. 문제: SPAD의 dark-count(비존재 광자 오검출)로 인해 채널 오류가 비대칭적(1→0 오류율이 0→1보다 높음)이며, 우주환경 방사선 노출 시 dark-count 증가.
3. 채널모델: 광자 도착을 지수분포로 모델링, dark-count rate/SPDE(Single Photon Detection Efficiency)/dead-time을 파라미터로 하는 SPAD 물리모델(Table I).
4. 핵심기법1: On-Off coding에서 cautious(2개 이상 검출 시 1, 0개면 0, 1개면 erasure) vs aggressive 디코딩 모드 비교.
5. 핵심기법2: Manchester coding — 검출구간을 반으로 나눠 n0,n1(전반/후반 광자수) 비교로 디코딩, 오류율을 소거율로 전환.
6. 확장기법: nibble(4비트, `u1~u4`)을 2x2 행렬로 배치 후 행/열/전역 parity(`p1~p5`, odd parity) 추가해 3x3 부호어 구성, Tanner graph(그림4)로 표현하되 사이클이 많아 표준 BP 적용 곤란.
7. 부수기법: 3가지 디코딩 비교 — Probability Propagation(PP, ad-hoc 행/열 순차 ML 갱신), 전역 ML(전체 유효 메시지 확률 계산 후 최댓값 선택), sum-product 형태의 belief propagation.
8. 최적화 절차: 별도 학습 없음, 라운드 수(5~10)와 최소확률 컷오프를 시뮬레이션으로 탐색(그림5,6).
9. 결과: ML이 PP와 BP를 능가, 검출간격 140nsec·도착간격 50nsec에서 오류율 10^-3 이하 달성(비트당 광자 6.3개 필요, overhead 125%).
10. 한계: 상위 계층 EECC(LDPC-BP 언급뿐) 미설계, 4비트 소형 부호 수준의 물리계층 실험(시뮬)뿐이며 NAND 저장매체나 이진 QC-LDPC 부호구조와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPAD 채널 비대칭 오류 모델링 | `channel.cpp` `Set_R_Offset()` | NAND read 채널이 아닌 광자계수 채널로 모델 자체가 상이, 직접 재사용 불가 |
| nibble parity code (PP/ML/BP 비교) | 대응 없음 | 4비트 초소형 부호로 QC-LDPC 스케일과 무관 |
| 상위계층 LDPC-BP (언급만, 미설계) | 대응 없음 | 본문에서 실제 설계·구현되지 않음 |
