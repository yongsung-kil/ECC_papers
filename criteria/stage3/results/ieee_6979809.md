### B. 알고리즘 요약

1. 대상: LDPC convolutional code(LDPC-CC), semi-infinite Tanner graph의 주기 구조를 pipeline decoder(연속 iteration message-passing)로 복호.
2. 문제: 기존 (k,n)-regular LDPC-CC(QC-LDPC 파생)는 free distance 상한이 `(k+1)!`로 제한되고 하한은 미규명 → 근접 상·하한을 갖는 구성 필요.
3. 핵심구조 TCDPSM: 짧은 constraint length convolutional code에 delay processor(`Gi(D)`)와 signal mapper(`Gm(D)`)를 연접해 큰 constraint length convolutional code로 만듦.
4. 부호구성: 생성행렬 `G(D)=Gc(D)Gi(D)Gm(D)`, 이로부터 polynomial parity-check `H'(D)=Hc(D)(Gi(D)Gm(D))^-T Dτ1` (식14) 유도.
5. free distance 경계: `d_LB(λ) ≤ dfree(λ) ≤ d_UB(λ)` (식2·3), `∆j`(하한용)·`δj`(상한용) Hamming distance 척도로 정의, λ 충분히 크면 근접.
6. 핵심트릭: monomial 형태 `Hc(D)`를 써서 `H'(D)` 항 수를 억제.
7. 4-cycle 제거: 열 간 공통항으로 발생하는 4-cycle을 auxiliary polynomial bit node `A1,A2`와 auxiliary check node 추가로 제거(식19-25), pipeline decoder에 노드 2+2 추가(복잡도 소폭 증가).
8. 최적화: FAST algorithm으로 `Gc(D)` free distance 계산 후 상·하한 산출. 예: (2,3)-code에서 `10≤dfree≤12` (λ≥11).
9. 결과(AWGN, BPSK 시뮬): (2,3) R=1/3, order M≈146에서 기존 (2,3)-regular(상한 6) 대비 우수. (2,4) R=1/2는 IEEE 802.11n/ac LDPC-BC(N=648, 1296)와 유사 length에서 더 나은 error 성능.
10. 한계: HW 미설계, AWGN 시뮬만, NAND 무관. LDPC-CC(convolutional) 전용 구성이라 block 코드 구조와 다름.
