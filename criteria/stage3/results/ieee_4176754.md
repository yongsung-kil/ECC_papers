### B. 알고리즘 요약

1. 대상은 array LDPC(regular QC-LDPC, `H`는 p×p 순환치환행렬 블록)와 rate-compatible/Block LDPC(irregular QC-LDPC, WiMax 802.16e·802.11n 표준), rate 0.5~0.909, length 1266~3000.
2. 기존 semi-parallel LDPC 디코더는 message-passing 메모리와 라우터(인터커넥트) 요구량이 구현 복잡도를 지배하는 문제; 기존 TDMP(layered decoding)도 여전히 메모리 재계산·중복 저장이 많음.
3. 채널은 BPSK+AWGN, soft LLR 기반 offset min-sum(OMS) belief propagation, 5-bit 양자화로 floating-point SP 대비 0.1~0.2dB SNR 손실.
4. 핵심 기법 1단: **value-reuse 기반 serial CNU** — 각 check node의 R 메시지는 입력 크기 중 최소 두 값(M1, M2)과 부호만으로 결정되므로, 전체 k개 엣지 메시지 대신 M1/M2/M1위치의 3값(partial state→final state)만 저장해 메모리를 대폭 절감 (`R_mn = sign · max(K-β, 0)`, K는 M1 또는 M2).
5. `Q`, `R`은 각각 변수→검사, 검사→변수 메시지이며, offset `β`는 density evolution으로 결정(예: (3,6) rate 0.5 array 코드에서 `β=0.15`) — offset 보정을 3값에만 적용해 2's complement 변환 연산도 k회에서 2회로 축소.
6. 핵심 기법 2단(TDMP/layered decoding): array LDPC의 j개 block row를 j개 layer로 보고, 한 block row의 check-node 처리가 끝나는 즉시 variable node를 갱신해 다음 block row 처리에 바로 사용(on-the-fly) — 이터레이션당 필요 반복 횟수를 TPMP 대비 50% 절감, throughput 2배.
7. 부수 트릭: cyclic shifter를 log2(p) 단계의 스위치로 구성한 logarithmic shifter로 구현, irregular Block LDPC의 null block에는 check message=0/variable message=+∞를 대입해 처리 통일; WiMax 114종 PCM을 런타임 지원하기 위해 control memory 없이 Master-slave Benes network cyclic shifter 사용.
8. 별도 학습 절차는 없음(density evolution으로 offset β를 사전 결정하는 정도).
9. 결과: 메시지 메모리 80%(TPMP)/55%(layered) 절감, 라우터 요구 50%+ 절감; ASIC(130nm) 구현 시 802.11n 디코더 1.37Gbps@500MHz, 429mW, area efficiency 기존 대비 5.5배·energy efficiency 2.6배.
10. 한계: array/Block LDPC(순환치환 기반 QC) 구조에 특화된 스케줄링·라우팅 설계이며, 정정능력(BER) 자체의 개선은 목표가 아니라 기존 OMS와 동등 성능을 유지하며 메모리/면적/처리량만 최적화한 순수 HW 아키텍처 논문.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum(OMS) CNU 연산, value-reuse(M1/M2/부호만 저장) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min1/min2+sign 기반 min-sum이라 직접 대응, value-reuse 최적화로 메모리 절감 아이디어 이식 가능성 높음 |
| TDMP(layered decoding), on-the-fly block-serial 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update(even/odd 교대)와 유사한 스케줄링 최적화 계열, layer 단위 즉시갱신 방식과의 비교·이식 검토 대상 |
| cyclic shifter(logarithmic shifter, Master-slave Benes network) | HW arch(병렬화, 메모리, 라우팅, pipeline) — decoder HW 모델(z=32, HCU, GT) | QC-LDPC circulant shift 처리라는 점에서 GT(Graph Thinning) MUX 기반 edge 선택과 목적(라우팅 절감) 유사, 직접 대응 모듈은 `GT.cpp` `Make_GT_HW()` |
| irregular Block LDPC null-block 처리(out-of-order R selection) | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD), HCU 유사 비대칭 처리 | Prime ECC의 HCU(비대칭 degree column 2-cycle 처리)와 개념적으로 유사한 비정형 구조 처리 기법 |
| 5-bit quantization LLR | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft 양자화 비트폭 설계 참고 가능(Prime ECC는 2SD/3SD, 본 논문은 5-bit 고정) |
```

적용 가치: 높음 — Prime ECC가 이미 보유한 min-sum/HCU/dual-update와 겹치는 부분은 있으나, value-reuse 기반 메모리 절감과 cyclic shifter/라우팅 최적화는 Prime ECC의 HW arch(GT, pipeline) 레이어에 구체적으로 이식 검토할 가치가 있는 실측(ASIC) 검증 완료 기법.
