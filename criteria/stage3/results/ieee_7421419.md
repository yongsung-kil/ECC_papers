### B. 알고리즘 요약

1. 대상은 (dv,dc)-regular LDPC 코드, 실험은 (6,32)-regular, `N=2048`, `R=13/16` (10GBASE-T Ethernet 표준 코드)로 검증.
2. 문제의식: 실제 HW는 4~7bit 유한정밀도 메시지를 쓰는데, 저해상도일수록 특히 error floor 영역에서 성능이 심하게 열화됨.
3. 채널모델은 BI-AWGN + 대칭 quantizer `Q_L`; 대칭 채널·대칭 quantizer 조건이면 어떤 채널에도 적용 가능(BICM 등도 언급).
4. 핵심 기법: Min-Sum의 CN 업데이트(`sign(μ)·min|μ|`)는 그대로 두고, VN 업데이트 `Φ_v`만 LUT로 대체하는 "min-LUT" - LUT는 `Φ_v^(i) = argmax_Φ I_i(m;x)` (식15)로 반복별 mutual information을 최대화하도록 density evolution(DE) 기반 설계.
5. 식(15)는 VN 출력 메시지와 코드비트 간 상호정보량을 최대화해 반복마다 최적 양자화 매핑을 찾는다는 의미로, 대칭조건(식11)을 만족하는 매핑 중에서 탐색.
6. LUT 크기가 VN degree에 지수적(`|L||M|^(dv-1)`)으로 커지는 문제를 해결하기 위해 nested LUT tree 구조(예: dv=6일 때 이진트리 형태 중첩) 사용, cumulative depth `λ(T)`로 트리 우수성 평가.
7. 부수 트릭: LUT reuse(반복 구간별 동일 LUT 재사용, 예: I={1,5})로 오히려 성능 개선 관찰, message alphabet downsizing(3→2→1bit로 점진 축소)으로 근소한 성능 손실만 발생.
8. 학습/최적화 절차: Algorithm 1(이분탐색 기반 DE)로 noise threshold `σ*` 계산 및 반복별 LUT 시퀀스 설계, design SNR `γ`가 waterfall vs error-floor 트레이드오프를 결정.
9. 결과: 3bit min-LUT가 4bit conventional MS는 물론 floating-point MS보다도 우수한 FER 달성(Fig.4), LUT reuse 시 이득 더 증가.
10. 한계: 순수 알고리즘/시뮬 논문(HW 구현은 별도 [3] 참조), (6,32)-regular 특정 코드로만 검증되어 QC-LDPC/고rate 코드로의 일반화는 별도 검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-LUT VN 업데이트(LUT 기반, min-sum 대체) | `decoder.cpp` `VN_Cal_HD()` 등 (HD/2SD/3SD), `decoder.h` `Get_VNU_*` | VN 처리부를 LUT 매핑으로 교체하는 아이디어가 직접 대응, 저비트 soft(2~3bit) 환경에 적합 |
| Min-Sum CN 업데이트(변경 없음, sign·min 유지) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 논문도 CN은 그대로 min-sum 사용해 구조상 동일 |
| iteration별 LUT 시퀀스 / reuse pattern | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 반복별 LLR 테이블 인덱싱 구조와 개념적으로 유사(적응형 LLR) |
| DE 기반 LUT 설계(Algorithm 1) | 대응 없음 | 오프라인 설계 절차로, 코드 내 실행 모듈은 없음(설계 시점 도구) |

적용 가치: `높음` — Prime ECC가 이미 iteration별 LLR 테이블(PARAM_LLR)을 쓰는 구조라 VN 업데이트를 min-LUT식 정보이론 최적화 LUT로 교체하는 이식이 자연스럽고, 저비트(soft-2~3bit) 환경에서 성능·복잡도 동시 개선 가능성.
