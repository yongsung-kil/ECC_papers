### B. 알고리즘 요약

1. 대상은 일반 irregular LDPC block code(예: MacKay 저장소의 `120.64.3.111`, `PEGirReg504`, `96.33.968`, rate `0.4667~0.5`), 채널은 주로 BEC(stopping set 기준)로 분석·검증.
2. 문제의식: 최적화된 irregular LDPC는 waterfall 성능은 좋지만 error floor가 높아 저-BER 응용에 부적합, 기존 방법(디코더 수정/코드 재생성)은 복잡도 부담이 크거나 효과가 제한적.
3. 핵심 가정: error floor는 그래프의 작은 stopping set(BEC) 혹은 trapping set(일반 채널)에 의해 지배되며, 이는 코드를 바꾸지 않고도 parity-check matrix에 행(row)을 추가해 그래프 표현만 바꿔 제거 가능.
4. 핵심 기법 1단(LWRRS, Low Weight Redundant Row Search): 코드의 저가중치 codeword를 열거해 low-density generator matrix `G`를 만들고, 쌍대 코드의 저가중치 codeword를 열거해 그 중 가장 많은 stopping set을 제거하는 저가중치 redundant row(선형종속, rate 불변)를 탐색.
5. 핵심 식 의미: `Theorem 1`—stopping set `S`를 제거하는 redundant row는 그래프 girth `g`에 대해 최소 `g/2`개의 원래 row 선형결합이어야 함(저가중치 row 탐색의 이론적 하한 근거).
6. 핵심 기법 2단(BSSE/MSSE, 선형독립 row 추가): stopping set이 곧 codeword인 경우(=최소거리 자체가 낮은 경우)엔 redundant row로 제거 불가하므로, 최소 개수의 선형독립 parity-check를 그리디(`ApproximatedBSSE`)로 추가해 최소거리를 늘림(약간의 rate penalty).
7. 부수 트릭: 두 기법을 결합(negligible rate penalty의 독립 row 소수 + rate 불변 redundant row 다수)해 stopping set이 codeword인 경우까지 포함해 더 낮은 rate 손실로 error floor 개선; PEG 코드 생성 시 일부 행을 비워두고 사후에 BSSE/LWRRS로 채우는 통합 절차도 제안.
8. 이론 결과: MSSE/BSSE 문제가 1-in-3 SAT로 환원되어 NP-hard임을 증명(Theorem 2), 필요한 parity-check 개수의 상한을 union bound로 유도(Theorem 3).
9. 결과 수치: `120.64.3.111` 코드에 redundant row 3개(가중치 16,24,25) 추가로 error floor 2자릿수 개선; `PEGirReg504`에 독립 row 1개(가중치 19)로 최소거리 13→17, error floor 2자릿수 개선(추정); `96.33.968`에서 독립 1개+redundant 4개 조합이 최소 rate 손실로 최적 성능.
10. 한계: HW 구현·복잡도 분석 전혀 없음(순수 알고리즘/시뮬), 검증이 대부분 BEC(block error rate vs erasure probability)에 국한되고 AWGN/NAND-read soft-decision 채널 실험은 없음, redundant row 추가는 H-matrix 밀도를 높여 반복복호 효율에 부정적 영향 가능성 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Parity-check matrix에 redundant/independent row 추가 (H-matrix 확장) | `ecc_top.cpp` `Load_PCM()` | H-matrix/부호구조 로드 단계에서 다룰 부호설계 기법이나, Prime ECC는 특정 QC-LDPC(base+circulant, z_sb=32) 고정 구조라 임의 row 추가 시 QC 구조·인코더(`encoder.cpp`)와의 정합성 재검증 필요, 이식 난이도 높음 |
| stopping/trapping set 열거 및 error-floor 예측 | `decoder.cpp` `LDPC_Decoder()` (참고) | 이 코드의 min-sum 반복복호와 직접 연동되는 부분은 아니며, 순수 오프라인 코드 분석·설계 도구에 해당 |
| MSSE/BSSE 탐욕 알고리즘(H-matrix 후처리) | 대응 없음 | Prime ECC 코드베이스에 코드 생성/후처리 최적화 모듈 자체가 없음 |

적용 가치: `중간` — H-matrix를 사후 확장해 error floor를 낮추는 아이디어는 부호설계 관점에서 흥미롭지만, Prime ECC의 고정 QC-LDPC(H-matrix, 인코더 dual-diagonal 구조)에 row를 추가하려면 circulant/QC 정합성과 인코딩 재설계가 필요해 이식 난이도가 높고, 검증도 BEC 중심이라 NAND 채널 재검증이 필요함.
