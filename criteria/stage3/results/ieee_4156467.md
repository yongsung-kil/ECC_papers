### B. 알고리즘 요약

1. HARQ+FEC 통신시스템에서 채널상태에 따라 rate를 바꿔 쓰는 rate-compatible LDPC 코드를 생성하는 것이 목표, AWGN 채널, BP(sum-product) 복호 가정.
2. 기존 puncturing/extending 기반 rate-compatible 코드 생성법([1]-[3])은 encoder/decoder 구조를 크게 변경해야 하는 문제가 있음.
3. 핵심 가정: mother parity-check matrix `H`의 한 행(check node)을 여러 행으로 분해(decompose)하면 check node 수만 늘고 variable node degree distribution은 불변.
4. 핵심 기법: linear row decomposition - mother matrix 행 `h_i`를 P개 행으로 나누어 확장 matrix `H'`를 생성, complete decomposition(전체 행 분해) 또는 partial row decomposition(일부만 분해, 원하는 rate에 맞춰 분해 행 수 조절)으로 rate를 세밀 제어.
5. 핵심 식: Gaussian Approximation 기반 threshold 계산 `m^(l) = φ^-1(1 - Σλ_i φ(...))` (식4) - row decomposition으로 check degree가 절반이 되면 `m`이 유의미하게 증가(식5)해 threshold가 커짐을 이론적으로 보임.
6. 확장: column combination(열 결합) 방식도 대안으로 제시했으나 blocklength가 줄어들어 성능 저하, partial row decomposition을 권장.
7. 부수 트릭: row decomposition은 mother matrix에 short cycle이 없으면 확장 matrix도 short cycle을 유지해 저error-floor 보장, decoder 기본 구조(variable degree distribution 기반)는 거의 변경 불필요.
8. 최적화 절차: Gaussian Approximation으로 density evolution의 message 평균 `m`을 추적해 threshold 계산, 근사식 `φ(x)≈e^(-αx^β+γ)` 사용.
9. 결과: (433,495) irregular mother code(rate 0.87)에서 rate 0.37까지 확장 시 threshold가 5.82dB→-0.35dB로 개선(표1), partial row decomposition이 동일 rate에서 column combination 대비 SNR 2.5dB 이득.
10. 한계: HW 구현·게이트수·throughput 전무, 순수 이론(GA)+시뮬(BP, 최대 100 iteration)뿐이며 NAND read soft-info나 min-sum 복호 환경 언급 없음.
