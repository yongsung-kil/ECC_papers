### B. 알고리즘 요약

1. 목표: 하나의 Viterbi ACS(add-compare-select) 블록으로 convolutional 코드와 LDPC를 모두 디코딩하는 통합 디코더 아키텍처.
2. LDPC(sparse H를 가진 선형블록부호)를 BCJR trellis로 표현: partial syndrome `sr+1=sr+hr*xr`를 state로 두고, xr=0(수평)/xr=1(대각) 전이, 종단 `sn+1=0`만 유효 codeword.
3. trellis 상태수는 최대 `2^(n-k)`; 예시는 (7,4) 소형 블록부호로 시연.
4. path metric 갱신 `pk+1 = Mk ⊗ pk`, 여기서 `⊗`는 곱셈/합을 덧셈/min으로 치환한 유사 행렬-벡터곱: `pk+1[j]=min_i(mk[i,j]+pk[j])`.
5. survivor 추적 위해 `min^-1`로 원 상태 `î` 저장; branch metric은 hard(Hamming)/soft(Euclidean) 선택.
6. strongly-connected trellis: 여러 stage를 하나로 병합(압축비 `rmax=floor(logb N)`)해 PE 활용률·throughput 향상.
7. ACS를 3가지 locally-connected(systolic) PE 토폴로지로 구현 — Linear(N-cell, 면적小), Square(N×N, 속도大), Division-Fusion(M그룹×N, 절충).
8. survivor path 저장: truncation length `δ`(δ≥5K, strongly-connected시 δ≥5K/n) 블록 단위, N개 버퍼 뱅크에 이전 state 인덱스 포인터 저장.
9. 결과: Altera Cyclone II EP2C70, K=5(16-state)/K=6(32-state)에서 90~111MHz, Linear가 최소면적, Square가 최고속·최대면적, DF가 절충(latency `δ(N/M+1)+2(δ-1)+1`).
10. 한계: state 수 지수증가로 대형 LDPC 비현실, 실제 LDPC 크기 성능·BER 미제시, fixed-point/truncation로 성능 열화 언급.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BCJR trellis + Viterbi LDPC 디코딩 | 대응 없음 | Prime는 iterative min-sum BP, trellis/ACS 패러다임 없음 |
| ACS systolic PE 병렬 아키텍처 | 대응 없음 | Prime는 z=32 row 병렬 min-sum, systolic ACS와 무관 |
| soft branch metric(Euclidean) | channel.cpp Set_LLR_Th() | soft-decision 입력만 개념적 연결, 알고리즘은 상이 |
```

적용 가치: **낮음** — LDPC를 trellis/Viterbi로 디코딩하는 접근은 Prime ECC의 iterative min-sum QC-LDPC 구조와 근본적으로 다르고, 상태수 2^(n-k) 지수증가로 NAND 고rate 장블록에 비현실적이라 이식 가치 없음.
