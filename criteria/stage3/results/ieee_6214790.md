### B. 알고리즘 요약 (10줄 내외)

1. 대상 부호: regular (1024, 512) LDPC 코드(`rate=0.5`), BPSK+AWGN 채널, factor graph 상 variable node/check node 반복 메시지패싱.
2. 문제의식: 기존 동기식(synchronous) LDPC 디코더(stochastic 포함)는 인터리버 배선 지연의 wire-to-wire 편차 때문에 최악 지연(worst-case delay)에 클럭 주파수가 제약되어 throughput 상한이 낮음.
3. 핵심 가정/모델: 확률 메시지를 Bernoulli 확률스트림(0/1 비트열의 1 비율 = 확률값)으로 표현하는 stochastic decoding 방식을 채택, variable node는 `p(C)=p(A)p(B)/(p(A)p(B)+(1-p(A))(1-p(B)))`(식 1, AND게이트+edge memory), check node는 `p(C)=p(A)(1-p(B))+(1-p(A))p(B)`(식 2, XOR게이트)로 구현.
4. 핵심 기법: 전역 클럭 없이 request/acknowledge 로컬 핸드셰이킹으로 variable-check 노드 간 데이터 전송을 비동기화(Fig. 3) — 각 노드는 최신 도착 메시지만으로 동작해 느린 배선(tardy wire)을 기다리지 않음.
5. 타이밍 모델: `VAR: X(t)=f(Y(t-T_X),R)`, `CHK: Y(t)=g(X(t-T_Y))`, 데이터전송 `DT: Z=h(Z')` 4개 블록이 각자 로컬 타이밍으로 독립 동작(식 3~8)하는 비동기 거동을 BER 시뮬레이터로 모델링.
6. Stochastic decoding 고유 리스크: `lock-up` 현상(수렴 전 조기 정지로 coding gain 손실)을 edge memory(EM, 48-bit) 기반 랜덤화로 완화 — VAR 계산지연 `T_Xi`가 데이터전송지연보다 너무 작으면 EM이 동일 입력으로 중복 갱신되어 lock-up 유발.
7. 부수 구현 디테일: 90nm CMOS(ASPLA, Synopsys Design Compiler) 합성 결과로 `T_Xi=280ps`, `T_Yi=100ps`(모두 최소값), `T_XY=300~2400ps`(배선 길이 편차) 타이밍 파라미터를 실측 기반으로 설정.
8. 평가: `T_Xi`를 100~1000ps로 스윕한 시뮬레이션에서 `T_Xi=400ps`일 때 최소 latency 118ns(Eb/N0=3dB) 확인, `T_Xi`가 300ps 미만이면 lock-up으로 비수렴.
9. 결과: 제안 비동기 stochastic 디코더는 BER 성능은 sum-product/동기 stochastic과 거의 동일하게 유지하면서, 평균 latency가 동기식 대비 2.52배, 동기 파이프라인식 대비 7.37배 개선(843ns→295ns→118ns, Table II).
10. 한계: HW 시제작(chip) 없이 합성 결과 기반 타이밍 모델 시뮬레이션에 그치며, 핸드셰이킹 컨트롤러의 면적 오버헤드는 정량화되지 않음. 코드 정정능력 자체의 개선은 없음(기존 stochastic decoder와 BER 동등 수준 유지가 목표).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Stochastic decoding (VN/CN Bernoulli 확률연산) | 대응 없음 | Prime ECC는 min-sum LLR magnitude 기반 디코더로, 확률스트림 연산 패러다임 자체가 다름 |
| 비동기 request-acknowledge 스케줄링/클럭 제거 | `decoder.cpp` `LDPC_Decoder()` 파이프라인, `decoder.cpp` HW 6-stage pipeline | Prime ECC는 동기식 clocked pipeline(col_M1/Z/P1~P4) 구조라 clockless 아키텍처와 정합 어려움 (이식 난이도 높음) |
| Edge memory 기반 lock-up 방지 | 대응 없음 | Prime ECC는 min-sum이라 lock-up 문제 자체가 존재하지 않음(대응 없음) |
| 배선지연 기반 throughput 모델링 | `GT.cpp` `Make_GT_HW()` | Graph Thinning의 MUX 기반 edge 처리와 개념적으로 유사한 배선/라우팅 최적화 관심사이나 구현 방식은 무관 |

> 적용 가치: 낮음 — stochastic computation 기반 비동기 클럭리스 디코딩은 Prime ECC의 동기식 min-sum HW 파이프라인과 근본적으로 다른 패러다임이라 직접 이식이 불가능하며, 배선지연 완화라는 문제의식만 참고할 여지가 있음.
