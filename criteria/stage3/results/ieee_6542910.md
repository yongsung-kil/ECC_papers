### B. 알고리즘 요약

1. 대상: B-GLDPC(constraint node를 BCH 구성부호 `C0(n,k)`로 대체), `J=2/3` super-code, 예제 rate `0.429~0.686`, length `3780/7650`, sub-code `BCH(63,51)t=2`·`BCH(255,215)t=6`.
2. 문제: 광/초고속 무선은 soft LLR용 초고속 ADC 확보가 어려워 hard-decision 복호가 필수; 기존 WBFV는 성능은 좋으나 수천 vote 중 max-vote 탐색이 HW에 부적합(직렬=수천 cycle latency, 병렬=`n·log n` 비교기 자원).
3. 모델: 각 심볼이 구성부호 HDD로부터 vote(`V`=유효, `e`/`E`=오류 표시)를 받고, `J`개 vote를 pair(예 J=2: `VV,eV,ee,EV,Ee,EE`)로 묶어 신뢰도 순서화.
4. 핵심 기법: max-vote 탐색을 제거하고 vote pair를 3개 group으로 분류(1st=최저신뢰 `EE,Ee` 등), 신뢰도 낮은 group부터 일괄 flip.
5. 의미: weight 계산·정렬·max 탐색이 불필요 → 비교기·정렬망 제거로 HW 단순화 및 latency 단축.
6. 스케줄: 1st group bit가 남아있는 동안 1st만 flip, 없어지면 2nd group flip, 새 1st 발생 시 다시 1st 우선; 모두 3rd(신뢰) group이면 조기종료.
7. 구현 디테일: full-parallel J=2 decoder(register array + BCH HDD 2계열 + permutation network `π`), vote type 결정 블록(syndrome valid→`V`, 아니면 입출력 비교로 `E`/`e`)과 group voting 모듈을 AND/OR gate로 구성.
8. 학습/최적화: 시뮬레이션으로 group 3개 분할과 J=2/3별 최적 group 규칙(Table I) 도출.
9. 결과: BER 곡선이 WBFV와 거의 중첩(무손실). iteration은 J=3에서 min iteration `25~30 → 5~8`로 대폭 감소, J=2는 `10 → 5~7` 수준.
10. 한계: BER 개선 없음(수렴 속도만), gate count/throughput 수치 없음, GLDPC+BCH 구성부호에 특화되어 고전 QC-LDPC와 계통 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC WBFV bit-flip 복호 | 대응 없음 | Prime ECC는 min-sum message passing QC-LDPC - bit-flip voting 계통과 무관 |
| BCH 구성부호 HDD | 대응 없음 | GLDPC 전용 구성부호, Prime ECC의 단일 parity-check QC-LDPC와 다름 |
| iteration 조기종료(group=3rd) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()` | 개념상 조기종료지만 Prime ECC는 CRC 기반이라 이미 보유·기법 상이 |
| full-parallel HW 구조 | 대응 없음 | GLDPC 전용 아키텍처, z=32 QC-LDPC pipeline과 정합 안 됨 |

적용 가치: **낮음** - GLDPC hard-decision bit-flip voting 전용 기법으로 Prime ECC의 min-sum QC-LDPC와 부호·복호 계통이 근본적으로 달라 직접 이식 불가.
