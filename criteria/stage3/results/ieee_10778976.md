### B. 알고리즘 요약

1. 대상은 polar-like 코드(5G polar, dynamic/static RM 코드)와 이를 component code로 쓰는 product code, GLDPC(RM check node)이며 biAWGN 채널을 가정.
2. 기존 SCL 디코딩은 완비적(complete)이라 오검출(undetected error)을 줄이기 어렵고, CRC 결합 시 code-rate 손실이 발생하는 문제를 다룸.
3. 채널은 이진입력 AWGN(biAWGN)이며 SC 디코딩 트리의 3분할(visited leaves / unvisited valid subtrees / invalid subtrees) 구조를 이용.
4. 핵심 기법은 codebook probability `Q_U(y^N)`을 SC 디코딩 중 미방문 서브트리 확률을 균등분포 가정으로 근사(`Q*_U`)하는 것 (`식 12`).
5. `Q*_U`를 이용해 blockwise SO `Γ*(y^N, û^N) = Q_{U^N|Y^N}(û^N|y^N) / Q*_U(y^N)`를 정의, 이는 디코딩 결과가 정답일 확률의 정확한 근사치임.
6. 확장으로 bitwise SO `ℓ*_APP,i`를 리스트 관측과 채널 관측 가중합(가중치 `φ`, 리스트 밖 codeword 확률 근사)으로 산출(SO-SCL, `식 16`).
7. 부수 트릭: threshold test `Γ*(y^N,û^N) > 1-ϵ`로 generalized decoding 구현, MDR(오검출률 상한)을 ϵ로 직접 제어.
8. 별도 학습/최적화 절차는 없음(파라미터 최적화는 1-GMI 최대화를 통한 post-processing 스케일링 δ, γ 튜닝 정도).
9. 결과: (64,42) dynamic RM 코드가 동급 CRC-polar 대비 BLER/UER 모두 개선, GLDPC(1024,640) 앙상블의 반복복호 문턱값(Eb/N0)*이 기존 list-max/list-sum 대비 낮아짐(향상).
10. 한계: polar/RM 계열 SC 기반 디코더 전용 기법이며 HW 설계·복잡도 분석 없음, min-sum 기반 QC-LDPC 디코더에는 알고리즘 구조상 직접 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SO-SCL 디코딩 알고리즘 전체 | 대응 없음 | SC 트리탐색 기반으로 min-sum QC-LDPC 이터레이션 구조와 무관 |
| GLDPC(RM check node) turbo decoding | 대응 없음 | non-binary/구조화된 RM check node이며 `CNU_Update_New_Mag()`의 min-sum CN과 상이 |
| bitwise SO(soft LLR) 산출 방식 | `channel.cpp` `Set_LLR_Th()` | LLR 기반 soft-info라는 점만 개념적으로 유사, 산출식은 무관 |

> 적용 가치: 낮음 — polar/RM/product/GLDPC 코드의 SC 기반 soft-output 근사 기법으로, binary QC-LDPC min-sum 디코더와 알고리즘적 접점이 거의 없음.
