### B. 알고리즘 요약

1. 대상은 IEEE 802.16e WiMAX rate-1/2 LDPC(base matrix + z-lifting 순환천이 → QC-LDPC), length 576~960, BPSK/AWGN(및 block Rayleigh fading).
2. 문제: 표준 BP는 suboptimal이라 짧은 cycle에서 특정 오류패턴 복호 실패 → 성능 한계.
3. 기법1 MBBP(multiple-bases BP): 서로 다른 redundant PCM `H1..Hl`로 표준 BP 인스턴스 `l`개를 완전 병렬 실행, 유효 codeword로 수렴한 후보 중 `argmin Σ|y_v - s_v|^2`로 최종 선택("decoder diversity").
4. 핵심식 (1) `x̂ = argmin_{s∈S} Σ|y_v - s_v|^2` — 수렴한 디코더들 중 수신신호에 최근접 codeword를 고름.
5. 기법2 leaking 알고리즘: iteration별 신뢰도 threshold를 두어 채널정보를 점진적으로 "leak"시켜 short cycle의 unreliable feedback 억제(L-MBBP).
6. 핵심 기여: base matrix `H_b`(entry=순환천이량, -1=zero)에서 양의 원소가 열에서 겹치지 않는 두 행의 선형결합으로 redundant check 생성 → lifting으로 z개 binary check 확보, QC 구조 보존.
7. 구현 디테일: 원 PCM에서 10~16개 행을 교체해 full-rank의 서로 다른 PCM 생성, 무게 상승/차별성 tradeoff.
8. 비교: PEG 최적화 rate-1/2 부호(length 500~1000) 및 Gallager/sphere-packing bound와 대조.
9. 결과: `l`=15에서 표준 BP 대비 약 0.15dB 이득(BER 10^-5), Gallager bound 갭을 약 20% 축소, 최대 30 디코더 병렬(각 최대 200 iteration).
10. 한계: HW 미설계(시뮬만), 다수 병렬 디코더 자원 요구, WiMAX 저rate 부호 대상이라 고rate NAND ECC 정합성 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 병렬 BP(MBBP) 디코딩/후보 선택 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | 우리는 단일 min-sum 디코더, 다중 병렬 인스턴스 구조로의 확장은 면적·전력상 부담 커 이식성 낮음 |
| leaking(iteration별 신뢰도 threshold) | [decoder.cpp](../Prime_ECC_3.1_Claude) `Get_VNU_Table_Idx()`, [ecc_data.h](../Prime_ECC_3.1_Claude) `PARAM_LLR` | 우리의 적응형 LLR threshold 테이블과 개념 유사(이미 보유) → 중복성 높음 |
| base matrix에서 redundant PCM 생성/lifting | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | H-matrix 로드는 대응되나, redundant PCM 다중 생성은 우리 고정 부호구조와 상충 |

적용 가치: **낮음** — leaking은 우리 적응형 LLR과 중복되고, MBBP는 다중 병렬 디코더 자원을 요구해 고rate NAND QC-LDPC ECC에 비용 대비 이득이 작다. base matrix 기반 redundant check 생성 아이디어만 개념 참고 여지.
