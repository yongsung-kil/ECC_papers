### B. 알고리즘 요약

1. 시스템: QC-LDPC, protograph 16×136 block matrix, circulant Z=241, N=32776, M=3856, Rcode≃0.882; 채널은 AWGN(TLC-channel gray code 모델)에서 시뮬.
2. 문제: 기존 belief-propagation/min-sum 디코더는 LLR 갱신에 고정밀 산술연산(덧셈/비교/스케일링)이 필요해 FPGA에서 routing congestion과 자원 소모가 큼.
3. 핵심 가정/모델: LLR 메시지를 저정밀 정수 알파벳으로 매핑하고, 그 매핑 자체를 밀도진화(DE)로 사전 계산한 두 테이블(R, Q)로 대체할 수 있다는 RCQ(Reconstruction-Computation-Quantization) 패러다임.
4. 핵심 기법 1단: 레이어드 스케줄 디코더. VN 갱신 `l_{m,r} = γ_r + Σβ_{n,r}` 후 `α_{m,r} = sign(l)·Q(layer, min(|l|,VNmax))`로 양자화, CN 갱신은 `β_{m,r} = min(R(layer,|α|),CNmax)·sign(product)`로 재구성(reconstruction) 테이블을 사용.
5. 핵심 식/의미: `τ_i = log(p_i/p_{-i})`로 이산 확률분포를 부호 있는 LLR로 변환 — 표준 min-sum의 `tanh` 근사 대신, DE로 얻은 분포를 그대로 인덱싱 테이블화해 곱/log 연산을 룩업으로 치환.
6. 핵심 기법 2단: R/Q 테이블은 iteration과 protograph row(layer)별로 별도 구성되어 메모리가 전체 PCM이 아닌 layer 수(M·Imax)에 선형 비례 — fine-tuning은 (a) mutual information 기반 최적화(Monte Carlo), (b) importance sampling 기반 error-floor 미세조정 2단계.
7. 부수 디테일: R/Q 값의 대칭성(symmetry)을 이용해 음수 없이 절대값만 저장, BRAM 절반 절감; VNmax=31, Smax=3, Imax=24, (3,6)-bit quantization; HLS `#pragma HLS PIPELINE II=1`, VNU/CNU 모두 initiation interval 2.
8. 최적화 절차: 이진입력 AWGN 채널 기반 density evolution으로 확률분포를 iteration마다 컨볼루션으로 전파시켜 R/Q 초기 테이블 생성, 이후 importance sampling으로 rare error event(trapping set 유발) 대응 error-floor 파인튜닝.
9. 결과: 4×20 protograph 기반 코드에서 error-floor를 비최적화 대비 최대 3자리(10^-11~10^-13→개선) 낮춤; 평균 4 iteration으로 raw throughput ~7Gbps(정보 throughput 5.7Gbps@rate 0.882); Codelucida 대비 raw BER 3.3% 개선; 자원 19% LUT, 4% BRAM, DSP 0% 사용, 280MHz.
10. 한계: DSP-free 설계라 저비용 FPGA에 적합하지만 코드(특정 protograph/Z)에 종속적인 테이블 최적화가 필요하며, ASIC 합성/게이트카운트는 미보고, 실측(real flash chip) 검증 없이 시뮬레이션(Monte Carlo AWGN) 기반.

| CN Min 연산 + Reconstruction 테이블 R(·,·) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 기존 min-sum 크기 계산을 R 테이블 룩업으로 치환하는 형태로 이식 가능 |
| VN 양자화 + Quantization 테이블 Q(·,·) | `decoder.cpp` `VN_Cal_HD()`, `decoder.h` `Get_VNU_*` | (3,6)-bit RCQ 양자화를 기존 HD/2SD/3SD 양자화 대신/추가로 적용 가능 |
| iteration/layer별 LLR·R/Q 테이블 인덱싱 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | layer-dependent table 구조가 기존 iteration별 LLR 테이블 구조와 유사 |
| Importance-sampling 기반 error-floor 최적화 절차 | 대응 없음 | Prime ECC는 오프라인 DE/IS 최적화 도구를 코드 프로파일 상 보유하지 않음(local_opt.cpp는 LLR 자동최적화 stub) |

적용 가치: `높음` — DSP-free LUT 기반 RCQ는 기존 min-sum 대비 연산을 테이블 룩업으로 대체하는 구체적 대안이며, error-floor 개선(정정능력 향상)과 HW 자원/throughput 실증까지 갖춰 Prime ECC 관심 우선순위(§5)에 정확히 부합.
