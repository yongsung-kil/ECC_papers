### B. 알고리즘 요약

1. 대상: IEEE 802.11ad AA-QC-LDPC, 부호길이 `N=672`, sub-matrix `z=42`, rate 1/2·5/8·3/4·13/16 지원, base matrix `Mb×Nb` (13/16일 때 `Nb=16`).
2. 문제: multi-gigabit LDPC 디코더는 row-paralleled로 고throughput을 얻으나 면적·전력이 큼, 모바일용으로 부적합.
3. 복호: partial-paralleled SPA/normalized Min-Sum layer decoding, posterior `Qn`(size z)와 extrinsic `rmn`을 permutation network(shift `P(m,n)`)로 라우팅, 5bit LLR 양자화 시 floating 대비 BER 손실 무의미.
4. 핵심 기법1 (half-row): bit-node 병렬도를 code length 절반(672→336)으로 줄여 VFN/APP RAM/permutation 블록을 `Nb/2=8`로 반감, CN 병렬도는 z=42 유지.
5. CN 2-phase: 첫 8입력으로 `m1_p1,m2_p1,sg_p1`(식 1-3), 둘째 8입력으로 phase2 계산 후 최종 `m1_fnl,m2_fnl,sg_fnl`(식 4-6)로 min1/min2 병합.
6. 핵심 기법2 (single permutation network): 두 개의 barrel-shifter permutation을 하나로 축소, APP RAM을 비순차 저장하고 연속 두 layer의 shift 차분(식 7)만큼만 회전, inverse shift `τ=0`.
7. 파이프라인: CN 처리 3-stage, 각 half-layer 4 cycle(VC/RP/CT/CF/CF2), 두 half-layer 간 데이터 의존성이 없어 half-layer level 파이프라인 적용.
8. cycle 수: row-based `N_row=3·Nlayers·Niter`, half-row `N_half=4·Nlayers·Niter+1` (13/16에서 iteration당 12 cycle).
9. 결과: 40nm G, 500MHz, 5 iteration, rate 13/16에서 5.6Gbps, 99mW, 3.53 pJ/bit/iter, 35 Gbps/sqmm, area 0.16mm2, CFU가 전력 68% 차지.
10. 한계: 802.11ad 짧은부호·저rate 전용, gate count 미제공(면적만), half-layer 파이프라인으로 iteration당 cycle 증가(12 vs 9)라 latency 관점 불리.
