> 총평: 자기기록(BPM) 버스트에러용 2D LDPC product code로 NAND 단일 binary QC-LDPC와 목적·구조가 달라 이식 가치 낮음.

### B. 알고리즘 요약

1. BPM(bit patterned media) 자기기록 채널, PR 타깃 + SOVA 검출기. 100 페이지, 페이지당 1184×155 2D 데이터.
2. 풀려는 문제: BPM은 media defect/write failure로 긴 버스트에러 발생, 기존 LDPC-LDPC product code로 부족.
3. 채널모델: along/across-track PR target `((0,0.2,0),(±0.1,1,0.1),(0,0.2,0))` (SUL 유무), SNR=`10log10(1/σ²)`.
4. 핵심 기법: 2D product code - inner(row) LDPC1이 랜덤·단burst 정정, 미정정 long burst는 erasure 선언해 해당 라인 LLR을 0으로.
5. Erasure 전달: inner decoder가 검출한 버스트 위치를 outer(column) decoder에 알려 erasure 보상.
6. 핵심 확장(iterative): LDPC1↔LDPC2가 extrinsic information(EI)을 반복 교환. erasure는 첫 스텝만, EI는 반복적으로 교환.
7. 스케줄: LDPC1 10 iter, LDPC1↔LDPC2 간 EI 교환 8회 (기존 방식은 LDPC 100 iter + EI 1회).
8. 구현 디테일: SOVA→LDPC1(row LLR/parity)→LDPC2(column LLR/parity + EI + erasure)→다시 LDPC1로 EI 피드백.
9. 결과: BER 10^-6에서 기존 LDPC-LDPC product code 대비 약 1dB 개선(SUL 유무 무관), 버스트 3100~12400bit 구간에서도 우수.
10. 한계: 시뮬만, HW/throughput 미보고, BPM/SOVA/PR 채널 특화라 NAND 무관, LDPC1/LDPC2 세부 부호구조 미명시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC1↔LDPC2 EI 반복교환 (product code 디코딩) | 대응 없음 | Prime ECC는 단일 QC-LDPC 디코더, 2D product/turbo형 교환 스케줄 없음 |
| erasure 선언 → LLR=0 처리 | channel.cpp Set_LLR_Th() | soft-read LLR 설정과 개념적 근접, 단 버스트 erasure 메커니즘은 미대응 |
| iterative message passing (LDPC 복호) | decoder.cpp LDPC_Decoder() | 일반 반복복호 대응, 단 product 구조 외곽루프는 없음 |
```

적용 가치: 낮음 - 자기기록 BPM 채널의 2D product code + SOVA 특화 기법이라 NAND용 단일 binary QC-LDPC 코드베이스(Prime ECC)에 직접 이식할 요소가 없다.
