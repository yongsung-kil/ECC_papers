### B. 알고리즘 요약 (10줄 내외)

1. 대상은 유한길이 임의 LDPC 부호이며 채널은 주로 binary erasure channel(BEC), 확장하여 일반 memoryless symmetric channel(BiAWGNC 포함)의 trapping set까지 다룸.
2. 풀려는 문제: 유한길이 부호의 error-floor는 최소 크기의 stopping set(BEC)/trapping set(일반 채널)이 지배하는데, 이를 완전탐색하는 것은 NP-complete임을 증명하면서도 실용적 짧은 길이(n≲500)에 대해 계산 가능한 완전탐색 알고리즘을 제시.
3. 핵심 가정: iterative(belief propagation) 디코딩을 Boolean 함수(`AND`=variable node, `OR`=check node)로 표현하는 "iterative decoding function" 프레임워크를 정의.
4. 핵심 기법: decoding tree 기반 알고리즘(`Algorithm 2`)으로 BER의 타이트한 상한 `UB(y)`를 반복적으로 좁혀가며(narrowing search), 이 upper bound가 asymptotic order 상 error-floor와 정확히 일치할 때까지 sharpen.
5. 핵심 식은 `UB(y) = E[f(y)]`이며 Rule 0(독립 가정 시 density-evolution 식 평가), Rule 1(완화), Rule 2(pivoting, 등식 보존)로 dependency를 처리해 tree를 구성.
6. 확장 기법: `k-out trapping set`(check node 중 정확히 k개가 degree-1) 정의로 stopping set 개념을 일반 채널(비erasure)로 확장, minimum stopping set exhaustive search(SSES)로 환원해 풀이(Algorithm 3).
7. 구현 디테일: tree pruning(Rule 3, 4)으로 계산량 억제, branch-and-bound(분할정복)로 탐색공간을 partition해 효율 향상, 자동사상(automorphism) 구조 활용 시 lifting factor만큼 복잡도 절감.
8. 최적화 절차: 완전탐색으로 얻은 최소 stopping/trapping set 리스트를 objective function으로 삼는 "code annealing"(별도 companion paper) 부호 최적화에 활용.
9. 결과: Golay code(23,12)에서 130개 최소 stopping set을 2초에 완전탐색, rate-1/2 최적화 (512,256) 비정규 부호에서 최소 stopping distance 13(104개) 확보 및 FER 개선(전형적 부호 대비 개선 factor 관찰), n≤500 부호에서 stopping set(size≤13), trapping set(size≤11) 완전탐색 가능.
10. 한계: n이 커지면(수천 이상) 탐색가능 범위(searchable range)가 최소거리 성장 속도를 따라가지 못해 결과가 느슨해짐, HW 미설계·이론+시뮬만, BEC 위주 분석이며 AWGN/soft-decision 채널 확장은 근사적(near-codeword 개념 의존).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 최소 stopping/trapping set 완전탐색 및 error-floor 최적화 목적함수 | `ecc_top.cpp` `Load_PCM()` (H-matrix/부호구조 로드 단계) | 부호구성 시 trapping-set 최소화 objective로 H-matrix를 재설계하려면 이 로드 단계와 연계되나, 코드베이스에 자체 부호 최적화 루틴은 없어 이식은 오프라인 도구 형태로만 가능 |
| decoding tree / iterative decoding function 프레임워크 (Boolean AND/OR) | 대응 없음 | min-sum 실수 연산 기반 디코더(`decoder.cpp`)와 다른 BEC 전용 Boolean 해석 프레임이라 직접 대응 없음 |
| k-out trapping set (일반 채널 error-floor 분석) | 대응 없음 | 정성적으로 error-floor 원인 분석에 참고 가능하나 코드 모듈로 실존 대응 없음 |
```

적용 가치: `중간` — decoder/HW 이식 요소는 없으나, error-floor를 지배하는 trapping set을 정밀 식별해 부호 최적화 objective로 삼는 아이디어는 Prime ECC의 H-matrix 설계(고rate QC-LDPC error-floor 개선) 단계에 간접적으로 참고할 가치가 있음. 단, 완전탐색은 짧은 길이 한정이라 실제 NAND급 부호(n≫500)에는 직접 적용 곤란.
