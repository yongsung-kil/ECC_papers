### B. 알고리즘 요약

1. 대상: layered QC-LDPC MS 디코더, WiMAX(rate 1/2, 5/6), WPAN(rate 1/2, 3/4, 7/8), 비표준 regular 행렬(rate 1/2, 3/4), 코드길이 1296, `(β,γ)` 양자화 `(4,6)`bit.
2. 문제: layered 구조에서 조기종료는 HW 복잡도·처리율 저하를 유발할 수 있어, 종료회로 자체의 면적/전력 오버헤드가 절감되는 에너지를 상쇄하지 않아야 함.
3. 채널/모델: AWGN 가정, layer마다 AP-LLR(`γv`)이 반복 갱신되며 코드워드 도달 후에도 값이 흔들린다는 관찰(fluctuation) 기반.
4. 핵심 기법(Imprecise PC): layer L 처리 후 해당 layer의 parity check `Π γv(m) = 1, c∈M_L`를 확인, `n_L`개 layer 전부 만족 시 카운터 증가, 이 partial syndrome check가 3회 연속 성립하면 종료.
5. 핵심 식(3): `layer_pc()`가 layer 내 모든 parity check 만족 시 1 반환 — 코드워드 검출의 근사적(imprecise) 판정.
6. 비교 대상 기법: SignStability(부호 안정성), OTF Syndrome, SDA(임계값 기반), Sun2008(부호+임계값), SSD/SSiA(Ferrari의 safe 조건).
7. 구현 디테일: layer parity check는 circulant 크기만큼의 XOR 게이트, `⌈log2 n_L⌉`bit 카운터(partial syndrome용) + 2bit 카운터(연속 iteration용)만 필요, 파이프라인 단수·양자화·circulant 크기에 파라미터화됨.
8. 별도 학습 절차 없음(고정 규칙 기반 회로).
9. 결과: 제안 기법은 baseline 대비 FER 열화 0.2dB 미만, 평균 iteration 증가는 코드 무관하게 약 1.6회로 일정; Virtex-7 합성에서 저비용(파이프라인 단수 늘어도 비용 거의 불변, SignStability 등은 파이프라인 3단만으로 10배 비용 증가).
10. 한계: 조기종료 회로 자체 개선이라 정정능력(BER/error-floor)은 개선하지 않으며, WPAN rate 1/2 등 일부 코드에서 SignStability/SSiA는 종료조건 도달 실패(=최대 iteration까지 수행).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layer 단위 partial parity check 3연속 조기종료 | `partial_CRC.cpp` `partial_crc_HW_T4()` | 우리는 Partial CRC 4 termination point 방식을 이미 보유 — layer 단위 parity check 누적 방식과 비교해 회로비용/평균iteration 트레이드오프 참고 가능 |
| Full syndrome check 비교 baseline | `full_CRC.cpp` | 논문의 conventional syndrome check baseline과 대응 |
| layered scheduling / iteration 루프 제어 | `decoder.cpp` `LDPC_Decoder()` | 조기종료 신호를 컨트롤러에 삽입하는 지점과 대응 |
| SignStability/Sun2008 등 파이프라인 지연버퍼 기반 기법 | 대응 없음 | 우리 조기종료(Partial CRC)는 이미 파이프라인 독립적 구조로 별도 이식 불요 |

> 적용 가치: 낮음 — 조기종료 아이디어 자체는 이미 Partial CRC로 유사 기능을 보유하고 있어 중복 기여이며, 논문의 layer 단위 parity check 근사 방식이 우리 대비 뚜렷한 우위를 보이는지 불명확함.
