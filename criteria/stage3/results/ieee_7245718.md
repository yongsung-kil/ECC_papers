### B. 알고리즘 요약

1. 대상 코드는 finite geometry(FG) LDPC 코드 `FG(273,191)` (rate≈0.70, length 273bit), 고차수 variable/check node 구조에 적합한 MDD-BMP(modified differential decoding using binary message passing) 알고리즘을 아날로그로 구현.
2. 풀려는 문제: fully-parallel LDPC HW는 처리·배선 복잡도가 커서, offset min-sum 대비 저복잡도인 binary message-passing(MDD-BMP/IDB)을 아날로그 회로로 구현해 면적·에너지를 추가로 절감.
3. 핵심 모델: variable node는 메모리 `M_v`를 갖고 부호(sign)만 외부로 broadcast(`b_{v→c}=sgnr(M_v)`), check node는 XOR(부호곱) 연산만 수행하는 1-bit 메시지 패싱.
4. 핵심 기법: variable node의 discrete-time 누산(`M_v^(k) = M_v^(k-1) + floor(s·Σb_{c'→v})`)을 커패시터 기반 continuous-time 전류적분으로 대체 - inverter 출력전류를 합산해 커패시터 `C`를 충방전.
5. `M_v`의 부호는 커패시터 전압을 inverter 문턱값과 비교해 결정 - 디지털 가산기/누산기(adder/accumulator) 회로가 불필요해 면적 감소.
6. 확장: absorbing set 대응을 위한 degeneration 기법(`M_v^(k)`에서 `d·sgnr(M_v^(k-1))` 차감)은 FG코드가 아닌 RS-LDPC(2048,1723)용으로 원 논문[5]에서 제안된 것으로 본 논문 자체는 다루지 않음.
7. 구현 디테일: PMOS/NMOS 전류 mismatch(계통오차 최대 20%, random mismatch 표준편차 5%), inverter 문턱값 mismatch(표준편차 13mV, LLR오차 0.26)를 4차 다항식 모델로 behavioural 시뮬레이션에 반영.
8. 최적화 절차 없음(학습/DE 없음) - transistor-level(TSMC 65nm) 검증으로 대체.
9. 결과: 면적 0.27mm²→0.11mm²(FG(273,191) 기준), throughput 137Gb/s, 에너지효율 0.56pJ/bit(디지털 버전 4.88pJ/bit 대비 대폭 개선); 4.5dB SNR에서 예상 mismatch 수준의 BER 열화는 미미.
10. 한계: transistor-level BER 테스트는 시간이 너무 오래 걸려 16개 codeword만 검증(실측 아님, 시뮬 한정), FG(273,191) 단일 코드에만 적용, 고rate QC-LDPC로의 일반화는 미검증.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MDD-BMP binary message-passing 디코딩 | 대응 없음 | Prime ECC는 min-sum(magnitude+sign) LLR 기반 디코더로 sign-only 1-bit 메시지 패싱 알고리즘 자체가 없음 |
| CN sign-product(XOR) 연산 | `decoder.cpp` `C2V_Cal_New_Sgn()` | check node 부호곱 연산이라는 개념만 부분 대응, 아날로그 전류적분 구현은 대응 없음 |
| VN 누산을 아날로그 전류적분으로 대체 | 대응 없음 | Prime ECC는 디지털 bit-exact HW 시뮬레이터로 아날로그 회로 모델 자체가 없음 |
| 조기종료(absorbing set 대응 degeneration) | `decoder.cpp` LDPC_Decoder() 이터레이션 루프 | error-floor 대책이라는 주제만 공유, 알고리즘 구현은 상이 |
```

> 적용 가치: 낮음 - 아날로그 mixed-signal 회로 구현이며 디코딩 알고리즘도 min-sum이 아닌 binary(sign-only) message-passing이라 디지털 bit-exact HW 시뮬레이터인 Prime ECC와 근본적으로 다른 설계 패러다임.
