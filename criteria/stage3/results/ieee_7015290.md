> 총평: IRA-like(QC+zigzag) 부호 구성법으로 선형시간 인코딩과 error-floor 개선이 핵심. 순수 부호설계 논문이며 HW·디코더 개선 없음. NAND용 고정 QC-LDPC 교체 부담 크고 rate=0.5 저rate 위주라 이식가치 낮음.

### B. 알고리즘 요약

1. IRA-like 부호: `H=[H1 H2]`, H1은 정보노드(IN) 대응 QC sparse 행렬, H2는 패리티노드(PN) 대응 zigzag(dual-diagonal) 행렬. rate 0.5, length 504~9216 시뮬.
2. 문제: 일반 LDPC는 인코딩 복잡도가 블록길이 `n`의 제곱으로 커짐 → zigzag로 H를 준삼각화해 선형시간 인코딩 확보하려 함.
3. H1은 base matrix B를 PEG로 설계 후 circulant permutation `I(Si,j)`로 lifting(사이클 연장), degree-2 IN 금지로 성능 유지.
4. degree-2 VN 최대개수 제약 `Nv,max(2)=m-1` (식2): degree-2 VN을 전부 zigzag H2에 몰아넣어 저중량 코드워드 억제.
5. 직접 zigzag 연결 시 degree-3 VN이 길이-8 사이클과 weight-5 코드워드를 (k/p)·C(p,2)개까지 유발 → error-floor 상승.
6. 개선된 zigzag 연결: 이웃 PN을 서로 다른 CN그룹(CNG)에 연결해 같은 ING 내 degree-3 VN 간 거리를 확대, 짧은 사이클·저중량 코드워드 제거.
7. cycle optimization-constraint: CN 간 link length를 크게, IN 짝수 degree는 H2-link에 비중복 연결 → 최소거리 증가·weight 분포 최적화·error-floor 저하.
8. 구성 알고리즘: Hx 초기화 → `Hx(ip+1,i)=1` 설정 → upper-shift로 H2j 생성 → row order 조정으로 H2를 표준 zigzag 형태로(QC 구조·패리티 제약 불변).
9. 결과: SPA(max iter 100, AWGN/BPSK)에서 improved가 direct 대비 error-floor 소멸, CMMB-LDPC(dv=3) 대비 (9216,4608)에서 BER=1e-5에서 약 0.18dB 우수, H의 1 개수도 더 적음.
10. 한계: 순수 부호 구성법(시뮬만), HW/디코더 설계 없음, rate=0.5 위주, degree 분포 `λ(x)=0.5x²+0.5x⁴` 등 특정 설계 종속.
