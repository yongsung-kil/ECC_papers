### B. 알고리즘 요약

1. 시스템: BSC(binary symmetric channel), hard-decision 반복 복호, Hamming constituent를 갖는 H-LDPC 부호, rate `R ≈ 0.1~0.9`.
2. 문제: H-LDPC의 보증정정 오류비율(guaranteed corrected errors fraction) 하한을 더 높게(더 낙관적으로) 도출하려 함.
3. 모델: block-diagonal `H_b`(constituent Hamming PCM `H0`)에 랜덤 열치환 `π_l`을 여러 layer로 쌓아 sparse PCM `H`(길이 `n=b·n0`) 구성.
4. 핵심 기법 1단(알고리즘 A): Hamming constituent 성질 이용, 각 iteration마다 "inversion" 기준으로 unsatisfied constituent code 수를 줄이는 심볼 반전.
5. 핵심 식(알고리즘 B): H-LDPC를 SPC constituent의 irregular LDPC로 보고, 변수노드 degree `l` 중 `e_i > l_i/2`(과반 unsatisfied) 심볼을 반전 → syndrome weight 감소.
6. 확장: Lemma 1로 한 iteration에 반전될 심볼 존재 충분조건 `Σe_ij > (1/2)Σl_ij` 제시.
7. 구현 디테일: 복호 복잡도 `O(n log n)`, degree profile `λ̃(x)=(1/(x·n0))((1+x)^m0 −1)`, `ρ̃(x)=x^((n0−1)/2)`.
8. 최적화: Theorem 1(점근 열거법 기반)으로 root `ω0` 존재 시 `ω0·n/2` 미만 오류패턴 정정 가능한 부호가 확률 1로 존재함을 증명.
9. 결과: Table I/II - 알고리즘 B 하한(`ω0/2`)이 알고리즘 A(`α0ωγ`) 대비 유의하게 큼(예: R=0.5, n0=127에서 1.36e-3 vs 0.95e-3).
10. 한계: 순수 점근 하한 증명(시뮬/HW 없음), BSC hard-decision, Hamming-constituent 생성부호로 실제 부호길이·gate/throughput 미제시.
