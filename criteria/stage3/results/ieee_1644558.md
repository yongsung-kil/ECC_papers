### B. 알고리즘 요약

1. QC-LDPC 부호: H = [H_S | H_P], q×q 순환치환행렬(circulant permutation matrix) 블록으로 구성된 block-row `J`개 × block-column `L`개 구조, rate `R=1-J/L=2/3`, length `n=q×L`(1536~1728).
2. 문제: irregular LDPC는 capacity-approaching 성능이 좋지만 저차수(low-degree) column 때문에 BER 10^-7 부근에서 error floor가 발생, 이를 낮추면서 waterfall 성능도 유지하려 함.
3. 핵심 가정: error floor의 원인은 BP 복호 시 stopping set/trapping set이라는 그래프 구조적 결함(짧은 사이클) 때문.
4. 핵심 기법: block-column을 최소 degree(오른쪽)부터 최대 degree(왼쪽)로 순서대로 구성. 저차수 BC 사이에는 block 4-cycle을, degree-4 BC 사이에는 double block 4-cycle을 회피하도록 순환행렬 위치를 배치(`J>10` 권장).
5. 핵심 식: 2i-cycle 회피 조건 `Σ(C_jk,lk - C_jk+1,lk) ≠ 0 mod q` (식 2) — shift 계수 `C_j,l`을 이 조건을 만족하도록 랜덤 선택해 짧은 사이클을 차단.
6. 확장: degree 2·3 BC는 길이 4,6,8,10 사이클 회피, degree 4 BC는 길이 4,6 사이클 회피로 차등 적용, 길이-4 사이클은 고차수 BC로만 구성되도록 유도.
7. 부수 관찰: waterfall 성능은 최대 column degree `Wc`를 늘리거나 `Wc`를 갖는 열의 비율 `λ_Wc`를 줄이면 개선됨(Code 3→4: Wc 6→15).
8. 학습/최적화 절차: 수치 시뮬레이션 기반 trial(GA/DE 없음, degree distribution·cycle 구조를 직접 설계).
9. 결과: error floor가 TGn 802.11n 코드(10^-6대) 대비 code 1은 10^-7 이하로, code 2/3/4(J=16)는 10^-9까지 floor 없음. 인코더 FPGA(Xilinx Virtex-II) 구현으로 100MHz, t=32에서 4.8Gbit/s throughput, 수백 XOR 게이트.
10. 한계: 디코더 HW는 미구현(시뮬만), FPGA 결과는 인코더에 한정, NAND/저장장치급 BER 10^-15 요구사항은 추가 최적화(block-row 수 증가, 저차수 열 비율 감소)가 필요하다고 저자가 인정.
