### B. 알고리즘 요약

1. 대상: AWGN+BPSK, 4bit 채널 양자화 인덱스로 복호하는 `(dv,dc)=(3,6)` regular binary LDPC(code a=Margulis 2640, code b=8000, 둘 다 rate 1/2).
2. 문제: BP는 고정밀 LLR·비선형 함수로 복잡하고, min-sum/저정밀 양자화는 BER 손실 → BP급 성능의 저복잡 디코더 필요.
3. 핵심 아이디어: 변수·검사노드의 box-plus/합 연산을 실수 LLR 없이 **부호없는 정수 인덱스만 처리하는 LUT**로 대체.
4. 설계: Tishby Information Bottleneck으로 입력 `y`를 압축변수 `T`(|T|=`2^q`, `q=4`)로 매핑하되 관련정보 `I(T;X)` 최대화; 2-input LUT `t0=f(y0,y1)`가 box-plus의 정보최적 등가.
5. 핵심식: `w = LUT[ptr(i,m)+idx(u,v)]`, `ptr(i,m)=iM·2^{2q}+m·2^{2q}` — iteration `i`·LUT종류 `m`별 서브벡터 오프셋 주소계산만으로 노드연산 수행(정수산술).
6. LUT 구성: 고차 노드는 2-input LUT 분해(concatenation)로 구현, `dc=6`에서 서로 다른 LUT 3종(f0/f1/f2)만 필요.
7. HW 트릭: 중간결과 재사용으로 `dc=6` 전 출력 12연산(재사용 없으면 24연산); LUT reduction(4bit 2개를 uint8에 packing→메모리 절반) vs LUT expansion(주소 일부 사전계산→접근당 정수덧셈 1회, 메모리 4배).
8. 설계절차: discretized density evolution으로 iteration별 `p(x,y)` 추적 → IB로 최적 매핑 도출 → design-Eb/N0별 정적 LUT 세트 생성(별도 preliminary work [8]).
9. 결과: DSP(TI C6474) 실측 — IB가 Jacobian log 대비 BER 손실 <0.1dB이면서 throughput 26~52% 향상, min-sum 대비 BER ~0.4dB 우수; imax=50, parity-check 조기종료.
10. 한계: HW(RTL)·ASIC 미설계(DSP 소프트 구현), design-Eb/N0 고정 정적 LUT, q=4bit라 우리 ≤3bit 정책과 불일치, error-floor 미분석.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN/VN LUT 노드연산(box-plus 대체) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `VN_Cal_HD()` 등 | 높음 — min-sum CN·VN 연산 자리를 IB LUT로 치환하는 구조적 대체 대상 |
| iteration별 LUT 세트 | decoder.cpp `Get_VNU_Table_Idx()`, ecc_data.h `PARAM_LLR` | 높음 — 이미 iteration 구간별 LLR 테이블 인프라 존재, LUT 세트로 확장 가능 |
| 채널 4bit 양자화/soft-read | channel.cpp `Set_LLR_Th()`, `Set_R_Offset()` | 중 — 우리는 ≤3bit(2SD/3SD)라 q=4 그대로는 불가, 양자화 인덱스 매핑 재설계 필요 |
| DE 기반 LUT 사전설계 | 대응 없음 | Prime ECC엔 discretized DE 설계 파이프라인 없음(테이블은 local_opt.cpp 자동최적화 별개) |

적용 가치: **중간** — min-sum 노드연산을 정보최적 LUT로 바꾸는 접근은 decoder.cpp에 직접 대응되고 BER/throughput 이득이 실측되어 매력적이나, q=4bit·design-Eb/N0별 정적 LUT 세트 저장이 NAND의 저bit soft-read·다양한 RBER 조건과 충돌해 3bit 재설계·오프라인 DE 파이프라인 구축이 선행돼야 함.
