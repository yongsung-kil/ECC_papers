### B. 알고리즘 요약

1. (3,6) regular LDPC, length 8000, q=4bit 양자화 메시지, AWGN+BPSK 채널에서 information bottleneck(IB) 디코딩을 다룸.
2. 기존 IB 디코더는 check/variable 노드 연산을 LUT로 구현하는데, 입력 개수 N에 대해 테이블 크기가 `(2^q)^N`으로 지수적으로 커져 비실용적임.
3. 이를 노드 연산을 2-입력 단위로 분해해 순차 LUT로 처리하는 기존 방식은 성능 손실과 지연을 유발.
4. 제안: 매핑 `t(i) = f(i)(y(i))`를 신경망 `f_θ(y(i))`로 근사, 은닉층 1개(ReLU)만으로 전체 입력을 한 번에 처리.
5. 출력식 `t(i) = softmax(W(L) max(0, W(1)y(i)))`; 학습 시 softmax, 디코딩 시 arg max로 정수 클러스터 인덱스 산출.
6. 확장: 한 노드의 모든 출력 메시지를 한 번에 내는 multi-in-multi-out 구조, 그리고 iteration 인덱스를 추가 입력으로 받는 단일 iteration-aware 신경망(모든 반복에서 재사용 가능).
7. 입력/출력은 one-hot 또는 binary 인코딩 두 방식 비교, 학습은 cross-entropy loss로 기존 IB LUT가 생성한 샘플을 이용.
8. 학습 데이터는 discrete density evolution 기반 기존 IB 알고리즘([7])이 설계한 LUT에서 샘플링해 생성(별도 train/test 셋 없음, 지속 샘플링).
9. 결과: 신경망 기반 IB 디코더가 훨씬 적은 파라미터로 non-decomposed LUT 벤치마크 성능을 거의 재현, 기존 decomposed(seq.) IB 디코더 대비 BER 개선(double-precision BP 대비 최대 0.05dB 이내), min-sum보다 우수.
10. 한계: HW 구현/게이트카운트 없음, 시뮬레이션 검증만 수행, 가중치 양자화 영향은 향후 연구로 미룸.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| information bottleneck 노드연산(신경망 대체) | 대응 없음 | Prime ECC는 min-sum 기반이며 IB/신경망 노드연산 구조 자체가 없음 |
| 노드 LUT 양자화(q=4bit) | `channel.cpp` `Set_LLR_Th()` | soft 양자화 threshold 설계 개념만 유사, 구현 방식은 전혀 다름 |
| min-sum 대비 성능 비교 | `decoder.cpp` `CNU_Update_New_Mag()` | 벤치마크 대상으로만 언급, 이식 요소 아님 |

적용 가치: 낮음 — non-min-sum/신경망 기반 노드연산이며 HW 미설계·시뮬뿐이라 binary QC-LDPC HW 디코더에 직접 이식할 요소가 없음.
