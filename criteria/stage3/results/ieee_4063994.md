> 총평: 여러 rate에 걸쳐 concentrated check-node degree를 유지하는 QC-LDPC 구성법과 연산유닛 재사용형 universal 디코더 아이디어는 흥미롭지만, Prime ECC가 단일 고rate 부호 고정이라 실질 이식가치는 제한적.

### B. 알고리즘 요약

1. 마더코드(mother code)는 QC-LDPC, rate 11/15(0.733), 길이 2715; 여기서 row combination으로 rate 12/15, 13/15, 14/15(0.80~0.93)의 고rate 코드를 파생 생성.
2. 문제의식: 기존 row combination 방식은 파생 코드의 check-node degree 분포가 불균일(non-concentrated)해져 density evolution 관점에서 반복복호 성능이 저하됨.
3. 채널모델: AWGN, BPSK, SPA(log-likelihood 버전) 반복복호, 최대 반복 30회.
4. 핵심 기법: fractional row combination — 마더코드의 특정 행(row-block)을 여러 sub-row로 분할해 서로 다른 행과 결합함으로써, 파생된 각 row의 weight(=check-node degree)를 균일하게 맞춤 (예: `row weight=4,4,5`로 분할해 R(x)=2/3 x^17+1/3 x^18).
5. 핵심 식: check-node update를 sub-row 단위 중간값 `S_{m,i}=Σφ(β_mn')`, `t_mn=Σφ(β_mn')`로 분해(식 7~9) — 이 분해 덕분에 서로 다른 rate 코드 간에도 동일 연산유닛(중간값 계산기)을 재사용할 수 있음.
6. 2단 확장: QC 구조 유지를 위해 row-block 단위로만 결합/분할하도록 제약(circulant 경계에서만 분할) — 이를 통해 마더코드를 BIBD(Balanced Incomplete Block Design, Bose Design 기반) 로 설계하면 파생 코드 전부가 4-cycle-free(girth≥6)임을 보장.
7. 부수 트릭: 마더코드 자체가 girth-6이어도 row combination 후 새 코드에 4-cycle이 생길 수 있는데(반례 제시), λ=1 BIBD 기반 마더코드(circulant 분해 `S_{k,i}`)를 쓰면 이 문제가 원천 차단됨.
8. 최적화 절차: 마스크 행렬(4×15, 0/1)로 정규코드를 비정규화하고, degree 분포 쌍은 density evolution 결과에서 선택(`L(x)=12/15x²+63/15x⁶+84/15x⁸`, `R(x)=x^13`).
9. 결과: 4개 rate(11/15~14/15) 모두 BER 10^-6에서 Shannon한계 대비 1.40~1.56dB, error floor 없음(BER 10^-7까지); concentrated 분포 코드가 비concentrated 대비 약 0.1dB 우수.
10. 한계: 순수 시뮬레이션(AWGN)만 검증, HW는 "연산유닛 재사용 가능" 수준의 개념적 서술뿐 실제 gatecount/throughput/파이프라인 설계 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| fractional row combination (multi-rate 파생) | ecc_top.cpp Load_PCM() | Prime ECC는 단일 rate H-matrix 고정, multi-rate 파생 구조 자체는 대응 없음 |
| BIBD 기반 girth-6 QC 마더코드 구성 | ecc_top.cpp Load_PCM() | 신규 H-matrix 구성법이라 재검증 부담 큼(이식 난이도 높음) |
| universal decoder 연산유닛 재사용(중간값 S_m,i, t_mn) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal() | check-node 갱신 구조 아이디어는 CNU 연산부와 개념적으로 유사하나 multi-rate 전제이므로 직접 대응 없음 |
```

> 적용 가치: 낮음 — multi-rate 지원이 Prime ECC(단일 고정 rate QC-LDPC)의 설계 목표와 어긋나며, HW 검증도 없어 실질 이식 실익이 작음.
