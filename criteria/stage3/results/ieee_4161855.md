### B. 알고리즘 요약

1. (3,6)-regular QC-LDPC (1536,768), rate 1/2 코드를 Xilinx Virtex-II FPGA 상에서 partly-parallel 디코더로 구현, SNR 4.5dB에서 61.20Mbps 추정 throughput.
2. 기존 direct SP 구현은 CNU-VNU 데이터 의존성 때문에 메모리 2배 소요 또는 낮은 throughput/HW 이용효율(HUE)이 문제; overlapped SP(OSP)가 이를 개선하나 본 논문은 partly-parallel parity check(PPPC)와 파이프라인 메모리 액세스를 추가 결합.
3. CNU는 `R_cv = Π sign(L_c)·φ(Σφ(L_c))`(식1, SP) 또는 min-sum 근사 `R_cv=Π sign·min|L_c|`(식2)로 갱신, VNU는 `L_v=2r/σ²+ΣR_mv`.
4. j개 CNFU, k개 VNFU가 각각 block row/column을 동시 처리하고, j개 PCFU가 CNU와 동시에(concurrent) parity check를 수행해 hard-decision 메모리와 PCFU 개수를 절감(fully-parallel 대비).
5. dual-port synchronous 메모리의 1-cycle read latency를 활용해 intra-unit(같은 유닛 내 2-step 파이프라인, gain 1.5배) vs inter-unit(CNFU/VNFU 교대, single-port 대체 가능하나 처리량은 intra의 절반) 파이프라인 방식을 비교.
6. operation folding으로 φ(x) LUT와 q-bit adder 개수를 인자 p로 축소(`⌈k/p⌉·j + ⌈j/p⌉·k`), folding과 intra-unit pipeline을 결합한 "combined access"로 ROM을 약 절반 절감.
7. 8-bit uniform(843bu/834bu)·nonuniform(833bnu/842bnu) quantization 5종을 비교, MS 알고리즘이 uniform quantization에서 유한정밀도에도 SP보다 우수한 BER을 보임.
8. 학습/최적화 절차 없음(고정 스케줄링/아키텍처 설계, 시뮬레이션으로 quantization 선택).
9. 결과: MS 기반 OSP 아키텍처가 기존 [10]의 MS/HEMS 아키텍처 대비 flip-flop 39~51% 절감, BRAM 65% 절감, throughput은 MS 대비 2배 이상, HEMS 대비 61% 향상(iteration은 HEMS가 더 적음에도).
10. NOSP는 wcv=1 고정으로 HUE 100% 달성해 (5,13)-regular 코드에서 throughput 약 10% 개선(effective iteration 6.32→5.76)하지만 wcv>m/2 조건 코드에만 유효, 하드웨어 파이프라인/합성(ASIC) 검증은 없고 FPGA·시뮬레이션에 국한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CNU/VNU overlap 스케줄링 (OSP/NOSP) | `decoder.cpp` `LDPC_Decoder()` Dual-Update 스케줄 | Prime ECC의 parity even/odd 교대(Dual-Update)와 유사한 목적의 스케줄 최적화, 세부 방식은 다름 |
| CN min-sum 연산 (식2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 동일 min-sum 근사, 이미 보유 |
| 파이프라인 메모리 액세스(intra/inter-unit) | `decoder.cpp` pipeline 6-stage(col_M1/Z/P1~P4) | Prime ECC는 이미 bit-exact 6-stage pipeline 보유, 유사 목적 |
| operation folding (LUT/adder 개수 축소) | 대응 없음 | Prime ECC는 min-sum 기반이라 φ(x) LUT 자체가 없어 folding 대상 불일치 |
| message quantization (uniform/nonuniform 8bit) | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 관련; Prime ECC는 이미 iteration별 LLR threshold 테이블 보유, 균일/비균일 비교 결과는 참고용 |
| partly-parallel parity check(PPPC) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 목적 유사(조기 parity 확인)하나 본 논문은 각 iteration 내 병렬 parity check, Prime ECC는 CRC 기반 조기종료로 메커니즘 다름 |

적용 가치: **낮음** — 정정성능 개선이 없는 순수 스케줄링/메모리 효율 최적화이며, Prime ECC가 이미 dual-update 스케줄링과 pipeline을 보유해 중복. BRAM 65% 절감이라는 메모리 효율 수치는 참고할 만하나 직접 이식 가치는 제한적.
