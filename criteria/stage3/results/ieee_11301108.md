### B. 알고리즘 요약

1. QC-LDPC base matrix(420,196, rate≈0.47) 기반 GLDPC 코드를 구성, 각 check node를 BCH(15,11) component code로 대체.
2. 표준 LDPC는 short blocklength·저지연 환경에서 정정능력이 부족해 GLDPC로 확장.
3. AWGN 채널, BPSK 변조 가정, component code decoder로 BCJR(trellis MAP) 사용.
4. Algorithm 1로 base LDPC의 각 parity row를 component code(Hcomp)의 열에 랜덤 매핑해 `HGLDPC` 생성.
5. Algorithm 3의 check node update는 해석적 재계산 대신 `llrout = component_decoder(llrin, σ2)`로 BCJR을 직접 호출.
6. Algorithm 2로 각 generalized check와 codeword index 간 매핑 구조 `D`를 사전 구성해 디코딩 루프에서 재사용.
7. base 코드로 Gallager 랜덤 LDPC와 quasi-cyclic LDPC 두 종류를 비교 구성.
8. 별도 학습/최적화 절차 없음(구성은 랜덤 매핑 기반).
9. 결과: QC 기반 GLDPC가 Sum-Product 대비 1.2~1.8dB 이득(iteration≥4), PEG-LDPC(420,193) 대비는 SNR 구간별로 우열이 갈림(차이 0.4dB 이내).
10. 한계: 이론/시뮬레이션뿐 HW 설계 전무, BCJR trellis decoder의 실제 복잡도·latency 미평가, non-QC(Gallager) 기반은 성능이 더 나쁨.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GLDPC check node를 BCJR component decoder로 대체 | 대응 없음 | Prime ECC는 min-sum 기반 단일 parity check이며 trellis MAP component decoder 구조 자체가 없음 |
| QC-LDPC base matrix 구성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 개념만 유사, 실제 구성법(랜덤 매핑)은 이식 난이도 높음 |
| generalized belief propagation 반복 루프 | `decoder.cpp` `LDPC_Decoder()` | 반복 구조는 유사하나 CN 업데이트가 component decoder 호출로 완전히 다름 |

> 적용 가치: 낮음 — GLDPC+BCJR은 binary QC-LDPC의 단일 parity-check 구조를 벗어난 이질적 아키텍처이며 HW 미설계·rate 0.47(NAND 대비 저rate)로 Prime ECC와의 접점이 거의 없음.
