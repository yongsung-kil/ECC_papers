### B. 알고리즘 요약

1. 대상은 binary linear cyclic code(예: n=15,k=7 BCH; n=23,k=12 Golay; n=31,k=16 BCH), LDPC가 아닌 일반 cyclic code, rate는 예제별로 상이(약 0.47~0.52).
2. 문제: Feldman의 LP(linear programming) decoder는 LDPC에는 성능이 준수하나 일반(non-LDPC) 코드에 적용 시 성능 저하가 큼.
3. 채널모델은 AWGN 및 BSC(이진대칭채널) 가정, LP decoder는 fundamental polytope의 정수해를 ML codeword로 인식(ML certification property).
4. 핵심 기법: parity check matrix H에 dual code의 codeword(및 그 cyclic shift 전부)를 추가해 redundant parity check matrix를 구성 - `H`가 cyclic이면 대응 polytope도 cyclic(Theorem 1).
5. Theorem 2: H가 cyclic이면 BSC에서 오류벡터의 cyclic shift가 LP decoder 성패에 영향 없음 - 즉 오류 패턴만이 성패를 결정.
6. 확장 알고리즘(3단계): (1) H에 없는 dual code의 최소 가중치 codeword를 찾음 (2) 그 codeword와 모든 cyclic shift를 H에 추가 (3) 목표 성능 도달 시 종료, 아니면 반복.
7. 무작위로 dual codeword를 추가하는 방식(RRHLP)과 비교해 제안 방식(LRHLP)이 short cycle을 줄여 성능 개선폭이 훨씬 큼(정성적 conjecture 기반).
8. 최적화/학습 절차는 별도 없고 반복적으로 저가중치 dual codeword를 탐색·추가하는 그리디 방식.
9. 결과: n=31 BCH code(t=3)에서 완전 redundant(2^15-1행) 대신 62행만으로 LP decoder가 3비트 오류 정정 능력 100% 달성; 반복 횟수 증가시 WER 개선되나 계산복잡도 증가.
10. 한계: LDPC/HW 디코더가 아닌 GLPK 소프트웨어 LP solver 기반 시뮬레이션뿐이며 매우 짧은 코드(n≤31)에서만 검증, 대규모 QC-LDPC로의 확장성은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LP(linear programming) decoder 전체 | 대응 없음 | Prime ECC는 min-sum 기반 message-passing 디코더만 보유, LP solver 구조 없음 |
| dual code 기반 redundant parity check matrix 확장 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 구성법 관점에서만 개념적 참고 가능, 이식 난이도 높음(§4) |
| cyclic code 구조(BCH/Golay) | 대응 없음 | Prime ECC는 QC-LDPC 전용, cyclic BCH/Golay 코드 미지원 |

> 적용 가치: 낮음 - LP decoder는 Prime ECC의 min-sum HW 디코더와 알고리즘 계열이 전혀 다르고, 대상 코드도 짧은 cyclic BCH/Golay code로 QC-LDPC와 무관하다.
