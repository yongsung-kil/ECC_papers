### B. 알고리즘 요약

1. 대상은 두-단계(two-phase) Turbo/LDPC 병렬 디코더의 contention-free 인터리버 주소 생성 회로. 부호/복호 알고리즘 자체가 아니라 메모리 매핑 HW.
2. 풀려는 문제: 병렬 프로세서 `m`개가 동시에 같은 메모리에 접근하는 memory contention. QPP 인터리버 `f(i)=(ki+li^2)mod N`는 window `n | N`이면 maximum contention-free.
3. 핵심 가정: 길이 `N=nm`을 `i=s+jn`(`j`=프로세서 인덱스, `s`=메모리 위치)로 분할, 각 프로세서가 독립 extrinsic RAM 뱅크 접근.
4. 핵심 기법: 인터리브 주소를 저장하지 않고 세로 방향 재귀로 on-the-fly 계산. `f((s+1)+jn)=(f(s+jn)+g((s+1)+jn))mod N`, `g`도 `g((s+1)+jn)=(g(s+jn)+2l)mod N`로 재귀.
5. 핵심 식 의미: partition `q`와 position `r`을 각각 `mod m`/`mod n` 재귀로 갱신 → 곱셈·모듈러 없이 add-compare-select(ACS)만으로 주소쌍 계산.
6. 구현: 4개 서브블록(각 ACS), block1/3은 `log2(m)=4bit`, block2/4는 `log2(n)=8bit` 데이터패스. 회로 복제로 병렬화.
7. HW 디테일: 모듈러/나머지 연산 제거하도록 pseudo-code 단순화(플래그 `I1~I4`로 carry 보정).
8. (학습/최적화 절차 없음 — 결정론적 다항식 인터리버.)
9. 결과: Toshiba 90nm, 250MHz. 12병렬 인터리버 0.05mm2/12mW로 3Gbps 주소 생성. 메모리 기반 대비 면적 40배·전력 12.8배 절감. [13] QPP 대비 13.3% 작음.
10. 한계: Turbo 코드 QPP 인터리버(802.11n/16, LTE) 대상. LDPC는 "edge results 저장"으로 한 줄 언급뿐. 부호설계/BER/iteration 성능 무관, 순수 인터리버 회로.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QPP 인터리버 재귀 주소 계산 회로 | 대응 없음 | Prime ECC QC-LDPC는 circulant shift로 permutation 내장, 별도 QPP 인터리버 없음 |
| 병렬 프로세서 contention-free 메모리 매핑 | 대응 없음 | z=32 row 병렬 구조는 base matrix shift로 라우팅 결정, QPP 매핑 무관 |
| 인터리버 메모리 → 온더플라이 계산 | 대응 없음 | 시뮬레이터 범위 밖(물리 메모리/RTL 영역) |

적용 가치: **낮음** — Turbo 코드 QPP 인터리버 주소 생성 HW 최적화로, QC-LDPC(circulant shift 기반)에는 인터리버 개념이 없어 Prime ECC에 이식할 요소가 없음.
