### B. 알고리즘 요약

1. 시스템: AWGN + BPSK, LLR `L=2y/σ²` 입력. binary LDPC. CCSDS (8176,7154) regular QC-LDPC(row deg 32, col deg 4) 및 PEG 구성 irregular (8176,7154) rate 7/8.
2. 문제: NMS는 모든 check node·iteration에 동일 `α`(경험적 ~0.8) 사용 → BP 대비 잔여 격차. BP의 `tanh`/`tanh⁻¹`은 HW 부적합, 순수 NN 디코더는 길이 8000+에서 메모리 폭발(추정 160GB).
3. 모델: min-sum이 산출하는 `|Lij|`가 BP보다 항상 크다는 성질을 CN별 정규화로 보정.
4. 핵심 기법: check node `i`의 입력 `Z(i)={Zij}`를 사전학습 NN에 넣어 CN별 factor `αNN(i)` 추론, `Lij = αNN(i)·min|Zij'|·∏sign` (식9). VNP/CNP는 MS와 동일 유지.
5. 핵심 식: 학습 라벨 `αL(i)=L̄(i)/Zmin(i)` — BP가 낸 평균 CN메시지를 min값으로 나눈 이상적 정규화 계수.
6. 확장: irregular 대응 — row weight `w(i)<wmax`일 때 부족분을 큰 값(50/100)으로 채워 tanh≈1이 되게 하여 단일 NN(입력 `wmax=32`) 재사용.
7. 구현 디테일: NN 12층(입력 + dense 128×8 + dense64, ReLU 9개 + Softplus 1개). ReLU=선택기+비교기, Softplus=LUT로 FPGA 구현 주장. 오프라인 사전학습, 복호 중 재학습 없음.
8. 학습/최적화: TensorFlow 2.4, Adam lr=0.001, MSE loss, max 1000 epoch, early-stopping patience 20. 데이터 102200 CN × Eb/N0 0~6dB.
9. 결과: NN-aided NMS가 MS 대비 0.3dB 우수, BP와 0.02dB 격차(best NMS α=0.7의 0.04dB 격차를 50% 축소). PEG 부호가 CCSDS 대비 0.08dB 우수(둘 다 max iter 20).
10. 한계: HW 미설계(시뮬만, FPGA 미구현), gate count/throughput/iteration 감소 수치 없음, float LLR·NN 추론 오버헤드, waterfall만 평가(error-floor 미검증).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN별 normalization factor αNN(i) 적용 | decoder.cpp CNU_Update_New_Mag() | 우리 min-sum(min1/min2) magnitude에 CN별 scaling 곱 삽입 지점 |
| min-sum CN 연산(min·sign) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 논문 CNP와 동일 min·sign 구조 |
| factor를 iteration별로 조정 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | iteration 구간별 스칼라로 근사 시 기존 적응형 LLR 테이블에 흡수 가능 |
| NN 추론 모듈 자체 | 대응 없음 | NN forward-pass는 우리 HW 모델에 없음, LUT/테이블로 대체 필요 |
```

적용 가치: 중간 — CN별 normalization이라는 아이디어는 우리 min-sum scaling에 이식 여지가 있으나, NN 자체는 대응 없음이고 이득(0.02dB)을 iteration별 스칼라 테이블 근사로 재현 가능한지가 관건.
