### B. 알고리즘 요약

1. 시스템은 BICM-ID(bit-interleaved coded modulation with iterative decoding) 구조로, protograph/tail-biting spatially coupled protograph(TB-SC-P) LDPC 부호를 M-ary PSK/QAM(8-PSK, 16-QAM, 64-QAM) 변조와 결합, AWGN 채널을 가정한다.
2. 풀려는 문제: 기존 Gray mapper는 BICM에는 최적이나 반복복호(ID) 이득이 작고, TB-SC-P 부호는 rate-loss 없이 종단화되지만 기존 SC 부호 대비 threshold 개선(decoding-wave)이 없다.
3. 채널 모델은 복소 AWGN, `y_j = x_j + n_j`이며 demapper(Max-Log-Map)와 decoder(BP) 간 반복적 extrinsic LLR 교환을 가정한다.
4. 핵심 기법 1: LBPM(labeling-bit-partial-match) constellation mapper — 2단계 설계로 (1) 보호도가 높은 labeling bit 수를 최대화하고 (2) 나머지 bit는 인접 label 간 해밍거리를 최대화하도록 배치.
5. 핵심 식: 평균상호정보량 `C_BICM = m - E[log2(Σ_z p(y|z)/Σ_{z∈χ(i,b)} p(y|z))]` — mapper별 BICM capacity를 비교해 LBPM의 용량 우위를 정량화.
6. 핵심 기법 2: VNMM(variable node matched mapping) interleaving — labeling bit의 불균등 보호도를 이용해 코드워드 양끝단 bit를 고보호 위치에 배치, TB-SC-P 부호에 decoding-wave 현상을 인위적으로 유발.
7. 부수 트릭: 64-QAM 이상 고차 변조에서는 보호도를 2단계가 아닌 `⌈m/2⌉`단계로 세분화하는 "optimized VNMM"으로 확장.
8. 최적화/분석 절차: hierarchical EXIT 알고리즘(demapper-decoder 간 outer iteration + decoder 내부 inner iteration의 2단 MI 추적)으로 decoding threshold를 계산 (DE 계열 반자동 분석, NN 없음).
9. 결과: (3,6) TB-SC-P 부호 + LBPM + VNMM이 8-PSK에서 BER 10^-5 기준 SP/anti-Gray/MSEW 대비 0.7~2.0dB 이득, 16-QAM에서 RJA protograph 부호 대비 0.4dB 추가 이득.
10. 한계: 순수 시뮬레이션/EXIT 분석이며 HW 미설계. 코드율 0.5, 규칙적 (3,6) protograph 한 예시 중심이고 부호 자체의 정정능력이 아닌 변조 매핑·인터리빙 최적화가 핵심이라 부호설계/디코더 개선과 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LBPM constellation mapper (M-PSK/QAM 매핑) | 대응 없음 | Prime ECC는 NAND read/write 채널 대상이며 고차 변조·성상 매핑 개념 자체가 없음 |
| VNMM interleaving (decoding-wave 유발) | 대응 없음 | demapper-decoder 간 bit interleaving 최적화로, binary NAND LDPC의 단일 채널 구조와 무관 |
| Hierarchical EXIT 알고리즘 (decoding threshold 추정) | 대응 없음 | 점근적 threshold 분석 도구로 Prime ECC의 bit-exact HW 시뮬레이터 목적과 다름 |
| TB-SC-P protograph 부호 구조 자체 | `ecc_top.cpp` `Load_PCM()` | 개념적으로만 protograph 기반 H-matrix 구조와 연결, 논문의 초점은 부호가 아닌 변조/매핑이라 실질 관련성 낮음 |
