### B. 알고리즘 요약

1. 시스템: rate 2/3, length 61440의 QC-LDPC(column weight 13/3/2, row weight 9/10/11), 최대 30 iteration, 64QAM/Rayleigh(시뮬)·AWGN(HW) 채널.
2. 문제: SPA 수평처리의 `tanh`/`atanh`는 HW 구현이 어렵고, Min-Sum은 간단하나 성능 손실이 있음.
3. 핵심 가정: 다입력 수평처리를 2입력 재귀 `y=2*atanh(tanh(x1/2)*tanh(x2/2))`로 분해, `tanh`가 홀함수이므로 x1,x2>=0만 다루면 됨.
4. 핵심 기법: 2입력 결과를 `y = min(x1,x2) + fc(x1,x2)`로 표현, 보정항 `fc`(식(2)와 식(4)의 차이)를 2D LUT로 구현.
5. 핵심 식: `fc`는 항상 음수·대부분 작은 값 -> 절댓값만 적은 비트로 양자화 가능, `x1=x2` 대칭성 존재.
6. 확장: 좌표변환 `x1'=min(x1,x2)`, `x2'=|x2-x1|`로 LUT를 재배치 -> 큰 x1'에서 fc≈0, 큰 x2'에서 값 거의 동일해 redundancy로 LUT 축소.
7. 구현 디테일: 2입력 HPU = 좌표변환부 + LUT 검색부 + post-processing(작은 입력에 fc 가산), 입력은 [0,16] 범위 8bit(1bit 부호) 양자화, 다입력은 2입력 HPU 재귀.
8. 최적화: LUT 크기 3bit(8x8) vs 4bit(16x16) 비교로 양자화-성능 트레이드오프 결정.
9. 결과: BER=10^-5에서 4bit(16x16) LUT는 SPA와 0.01 dB 차, normalized Min-Sum 대비 약 0.1 dB 이득; HW 실측(AWGN, 8x8 LUT)은 normalized Min-Sum 대비 약 0.05 dB 이득.
10. 한계: 이득이 소폭, gatecount/throughput 미제시, Rayleigh/AWGN 통신채널 기준(NAND 직접 검증 없음), error-floor 미검증.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| 보정항 2D LUT / 좌표변환 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 기존 LLR/magnitude 테이블 파이프라인에 유사 LUT 추가 방식 유사 |
