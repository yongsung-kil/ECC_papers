> 총평: NAND read latency를 줄이려 무손실 압축으로 강한 LDPC를 기회적으로 쓰고 soft-sensing 정밀도를 낮추는 시스템 레벨 트레이드오프 전략 — Prime ECC 디코더/부호설계 자체는 손대지 않아 직접 이식 요소는 없음.

### B. 알고리즘 요약

1. 시스템: 2bits/cell MLC NAND, 4KB 페이지, 컨트롤러+다수 NAND 칩. rate-19/20 QC-LDPC `(34520, 32794)`, 검사행렬 2×40 circulant array, col weight 4, min-sum 복호.
2. 문제: 미래 SSD에서 advanced ECC(LDPC)는 soft-decision 입력이 필요→ n-bit soft sensing이 on-chip sensing latency `(2^n-1)/(2^2-1)`배, bus transfer latency `n/2`배로 증가 (6-bit시 read 77μs→346μs).
3. 모델: MLC threshold voltage 분포 = 중앙 균일분포(폭 ΔVpp)+양측 가우시안 꼬리(σ), erase state는 가우시안. 이 모델로 PER 시뮬.
4. 핵심 기법: 페이지별 run-time 무손실 압축으로 확보한 여유공간에 더 많은 ECC redundancy를 넣어 기회적으로 더 강한(저rate) LDPC 사용. firmware/OS에 투명(intra-page 독립).
5. 식 의미: `Rn→Rn/rc` (압축비 rc만큼 rate 하향). 강한 ECC의 여분 정정능력을 coarser sensing 허용에 트레이드→ `Lsens=α·(2^qf-2^qr)/(2^qf-1)`, `Ltrans=α·(qf-qr)/qf` latency 절감.
6. 확장: 구현 단순화 위해 코드rate 2단(Rn 정상, Rs 강함)만 허용, 압축비 threshold rt로 스위칭.
7. 구현 디테일: 무손실 압축은 LZ77 기반, 해제는 table lookup으로 on-the-fly(추가 latency 무시가능). 압축비 1.11:1이면 rate 19/20→6/7로 낮춰도 됨.
8. 최적화 절차: 각 감축 sensing 정밀도(Q=2~6)에 대해 성능손실 없는 LDPC rate를 시뮬로 탐색 (Q=5→16/17, 4→12/13, 3→7/8, 2→6/7).
9. 결과: 실제 파일 압축시 α=0.858~0.999. 최대 on-chip sensing latency 95.24%, flash-to-controller transfer latency 66.67% 절감. 평균 read latency 346μs→106μs(qr=2).
10. 한계: HW 미설계, 시뮬만. 복호기/부호구조 개선이 아니라 압축+가변rate 시스템 전략. 압축 실패 페이지·write 관리 오버헤드 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC min-sum 복호(예시 부호) | `decoder.cpp` `LDPC_Decoder()` | 논문은 min-sum을 그대로 쓸 뿐 개선 없음 — 이미 보유(§3) |
| soft-sensing 정밀도별 LLR 양자화 | `channel.cpp` `Set_LLR_Th()`, `Set_R_Offset()` | soft 2~3bit read 비용 개념은 겹치나 압축 연계 로직은 대응 없음 |
| 페이지 단위 가변 코드rate 스위칭 | 대응 없음 | Prime ECC는 고정 QC-LDPC(H-matrix), 가변rate/압축 프론트엔드 없음 |
| 무손실 압축(LZ77) | 대응 없음 | ECC 파이프라인 밖의 시스템 레이어 |

적용 가치: **낮음** — NAND read latency 관점은 관련 있으나, 기여가 압축+가변rate 시스템 전략이라 Prime ECC의 디코더/부호설계/HW 어디에도 떼어 붙일 알고리즘이 없음.
