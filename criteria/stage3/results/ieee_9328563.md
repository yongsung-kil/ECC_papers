> 총평: 5G NR용 GPU LDPC 디코더 가속(warp-shuffle parallel-reduce) 연구로 부호정정 개선 없이 GPU 아키텍처 최적화에 국한, 우리 HW binary QC-LDPC 엔진 이식 가치 없음.

### B. 알고리즘 요약

1. 시스템: 5G NR uplink cloud RAN, 3GPP QC-irregular LDPC(BG1/BG2), lifting factor `Z`, `Nc=26112bit`, rate `R=1/3~1/2`, 목표 지연 500μS@10 iteration.
2. 문제: layered 디코더는 순차적이라 코드워드 1개 내 병렬성이 부족해 GPU SM 코어를 저활용(Turing 최소 75%).
3. 배경: min-sum 근사 `Lci→v ≈ β·min|Lvj→c|·∏sgn`, VN LLR `Lv=Lb+ΣLck→v`, `Lvi→c=Lvi−Lic→v` (교과서 layered BP).
4. 핵심기법 1단(high-throughput): row당 1 thread, `Nthread=ρ⁻¹Z` (ρ=2 fp16 / ρ=4 int8 SIMD packing). 그러나 shared memory/대역폭 한계로 활용도 못 올림.
5. 핵심 식(5)(6): 저지연 전략 `Nthread=(α/ρ)Z`, α는 cooperative group 확장인자(2의 배수), thread당 `wr/α` 컬럼 처리.
6. 핵심기법 2단(parallel-reduce): CN의 min/sub-min/sign 계산을 warp-shuffle로 `log2(n)` 단계 축약, 코드워드 내 추가 병렬성 확보.
7. 구현 트릭: 8bit LLR용 `ordVec8`(vcmpltu4+LOP3, 6 asm), 16bit용 `ordVec16`(set.le.f16x2+prmt.b32, 10 asm) intrinsic 자체 설계.
8. 최적화: α를 Z에 맞춰 조정해 활용도 유지, Lv는 shared memory·Lc→v는 main memory 배치.
9. 결과: 24 SM GPU에서 flooding GPU 대비 5×, FPGA(747K gate) 대비 3× throughput. 단 전력효율은 FPGA의 1/3. int8은 32bit 대비 0.25dB, 5bit는 1.2dB 손실.
10. 한계: 부호정정 성능 개선 아님(순수 가속), CUDA warp-shuffle/SIMD 아키텍처 특화, 5G NR 무선 타깃(NAND 아님).
