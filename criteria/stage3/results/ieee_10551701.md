### B. 알고리즘 요약

1. 대상은 5G NR(3GPP TS 38.212) 기반 QC-LDPC, BG1의 (440,200)·(3000,1000) 등 rate 0.33~0.45 코드, layered BP(LBP) 디코딩을 baseline으로 개선.
2. 문제: 기존 LBP는 check-node를 parity-check 행렬의 행 순서(고정)로 갱신할 뿐 순서 자체를 최적화하지 않아 개선 여지가 있음.
3. 복잡도 지표 `NMP`(number of messages passed, 엣지 단위 메시지 전달 횟수)와 성능 지표 `AE`(average entropy, density evolution 기반 변수노드 불확실성)·`GAP`(Bethe free energy 기반 MAP과의 거리)로 스케줄링 정책을 정량화.
4. 핵심 기법: 스케줄 시퀀스 `β`에 대해 `τ_AE(c,β) = Σ_t AE_β(c,t)`(디코딩 초반 감소를 더 중시하는 누적합)를 정의, 이를 최소화하는 `β`를 탐색 목표로 설정(`τ_GAP`도 동일 구조).
5. `τ_AE`가 작을수록 초반에 빠르게 불확실성이 줄어 조기종료(early stop)에 유리하다는 것이 식의 의미.
6. SSBP(Successive-Searching BP): local search 기반 알고리즘으로, 초기 시퀀스에서 무작위로 `h`개(≈0.5%×M) check-node 쌍의 순서를 바꾼 `b`개(=100) 후보를 생성해 `τ_AE`/`τ_GAP`가 개선되면 갱신, `S`(=50)회 연속 실패 시 종료.
7. 탐색 공간 축소를 위해 (protograph) CN-centric 스케줄링(각 스텝에서 check node 하나 선택, 연결된 모든 엣지 동시 갱신)을 채택 — QC-SSBP.
8. 최적화 절차: 초기 시퀀스(layered/nested/LD 스케줄 등)에서 시작해 반복적 local search로 개선, offline이므로 실제 디코딩에는 추가 연산 부담 없음.
9. 결과: (440,200) BG1에서 SSBP(AE)가 LBP 대비 평균 NMP 약 30% 감소, 전체적으로 20% 이상 복잡도 절감, BER/BLER도 중간 단계에서 최저 수준으로 조기종료 유리.
10. 한계: 최대 반복수 5로 제한된 실험, gatecount/throughput 등 HW 검증 없음, 스케줄은 code-specific(사전 탐색 필요)이며 5G NR 저rate 코드에서만 검증되어 NAND ECC 고rate 영역 이식 시 재검증 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered BP check-node 스케줄링 순서 최적화(SSBP) | decoder.cpp LDPC_Decoder() 이터레이션 루프, Dual-Update 스케줄 | Prime ECC의 even/odd 교대 Dual-Update 스케줄을 대체/보강할 수 있는 오프라인 순서 최적화 아이디어로 이식 난이도 낮음~중 |
| NMP(복잡도)·AE/GAP(성능) 평가 모델 | decoder.cpp 이터레이션 카운터 (부분 유사) | 스케줄 설계용 오프라인 시뮬레이션 지표이며 HW 파이프라인 자체에는 직접 대응 없음 |
| (protograph) CN-centric 시퀀스 | ecc_top.cpp Load_PCM() / decoder.cpp HCU 처리 순서 | col-block 단위 처리(z_sb=32, HCU)와 개념적으로 유사하나 실제 갱신 순서 재배치는 GT.cpp Make_GT_HW()의 edge 선택 테이블과 연계 검토 필요 |
```

적용 가치: **중간** — offline 스케줄 순서 최적화는 디코더 알고리즘 레벨 개선으로 Prime ECC의 기존 Dual-Update 스케줄과 결합·비교 검토 가치가 있으나, 5G NR 저rate 코드 기준 검증이라 NAND 고rate QC-LDPC 재검증이 필요하고 HW 구현 정보가 없음.
