### B. 알고리즘 요약

1. Spatially coupled LDPC 코드 앙상블 `(dv, dc, L)`을 대상으로, 기존 [6]의 recursive encoder는 `dc/dv`가 정수인 경우에만 적용 가능했음.
2. 문제: check node degree와 variable node degree 비율이 정수가 아니면 parity bit가 이전 위치 인코딩 결과만으로 순차 결정되지 않아 shift-register 인코딩 불가.
3. 채널 모델은 이진 소거 채널(BEC)이며, 성능은 density evolution의 erasure probability로 분석.
4. 핵심 기법: protograph의 edge를 재배열해 parity-check matrix의 각 서브행렬 내 permutation 위치를 이동, `vi,1`, `vi,2` 등 parity bit가 이전 위치의 encoded bit와 현재 정보 bit만으로 순차 결정되도록 변형.
5. 핵심 식 `viHiT = [si, qi]` (partial syndrome 기반 recursive 갱신)로, 정보비트·인코딩된 parity·partial syndrome만으로 다음 parity를 계산해 shift-register 구현이 가능해짐.
6. 확장 없음 (단일 구조 변형).
7. 구현 디테일: shift-register는 `(dc + dv·M − 1)M` 저장 유닛 필요, 비트당 계산복잡도는 `dc`에 비례 - 기존 [6] 대비 유사 수준 유지.
8. 별도 학습/최적화 절차 없음 (구조적 설계 + density evolution 해석적 분석).
9. 결과: 수정 구조가 기존 구조 대비 BP threshold 개선 (예: `(6,10)` 앙상블에서 `εBP` 0.5292→0.5926), design rate 손실은 L이 클수록 무시 가능.
10. 한계: BEC 채널·유한장 시뮬(BER, M=500, L=50)에 한정, HW 구현·양자화·AWGN/NAND 채널 검증 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 임의 rate recursive encoder 구조 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC는 QC-LDPC(block code) dual-diagonal 고정 구조라 SC-LDPC convolutional 인코딩 자체는 대응 없음, 유사 위치만 표기 |
| Density evolution BP threshold 분석 | 대응 없음 | 부호설계 이론 분석 도구로 HW 시뮬레이터와 무관 |
| Parity-check matrix H-matrix 구성/edge 재배열 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 구조와 개념적으로만 연결, SC-LDPC는 별도 부호족 |

> 적용 가치: 낮음 - SC-LDPC(convolutional) 인코더 구조 논문으로 Prime ECC의 QC-LDPC block code encoder(dual-diagonal 고정)와 부호족 자체가 다르고, BEC 채널·이론 분석 중심이라 NAND binary QC-LDPC로 이식할 요소가 거의 없음.
