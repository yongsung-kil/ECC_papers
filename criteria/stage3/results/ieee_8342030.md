### B. 알고리즘 요약

1. 대상: TLC NAND에서 read disturb로 인한 read refresh(rrGC) 폭증 문제를 FTL 레벨에서 완화 (부호/디코더 아님).
2. 문제: read-intensive 워크로드에서 rrGC가 전체 block erase의 최대 99%를 차지, worst-case read latency를 블록 erase의 66배까지 악화.
3. 핵심 아이디어: TLC 8레벨 중 P2/P4/P6를 `guard level`로 비워 tolerance margin을 넓혀 read-count limit을 10K→100K(10배)로 확장 (셀당 3bit→2bit 저밀도).
4. 핵심 기법 1 (IPR): 셀 threshold voltage는 상향 재프로그램만 가능하다는 물성 이용, MSB만 덮어써 고밀도 블록을 in-place로 저밀도화 → 데이터 복사량 감소(read refresh당 새 블록 3/2 → 평균 1/3).
5. 핵심 기법 2 (read-hot 식별): regular→monitor→read-hot 3-state 블록 전이로, 블록 read-refresh 간격/페이지 read count 비교(Bavg, Pavg)를 통해 저비용 read-hot 페이지 분류(모니터 블록 ≤10%).
6. 마모 관리: P/IPR/E cycle의 정규화 마모가 IPR rate≥0.2일 때 1.2로 수렴 → IPR을 IPR pool(상위 20% 주소)에 집중, effective P/E count로 wear leveling.
7. 오버헤드: 128GB flash 기준 블록/페이지 메타데이터 약 1.37MB embedded RAM.
8. 실험: DiskSim 기반 SSD 시뮬레이터, SNIA IOTTA/UMass read-intensive 트레이스 9종, Baseline/RedFTL/RL/HotLD/IPR 비교.
9. 결과: Baseline 대비 rrGC erase count 최대 85% 감소(SRC), 총 erase 감소로 SSD 수명 57% 연장, 평균 read response time 22% 개선(261→206us).
10. 한계: LDPC 언급은 "read retry가 disturb를 증폭"하는 배경뿐, ECC 부호/디코더/HW 설계는 전혀 없음. 순수 FTL 시뮬레이션.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| read disturb / read refresh 관리 | 대응 없음 | FTL/블록관리 계층으로 Prime ECC(부호·디코더·HW) 범위 밖 |
| guard level / cell bit-density | 대응 없음 | NAND 셀·threshold 설계 이슈, 부호설계와 무관 |
| in-place reprogramming (IPR) | 대응 없음 | flash 프로그래밍 물성 활용, ECC 코드베이스에 접점 없음 |

적용 가치: **낮음** — LDPC 부호설계/디코더/HW 어느 것도 다루지 않는 FTL read-disturb 관리 논문이라 Prime ECC에 이식할 알고리즘이 없다.
