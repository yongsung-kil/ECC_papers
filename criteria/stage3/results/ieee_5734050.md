### B. 알고리즘 요약

1. 대상: systematic `(N,K)` RS 부호(`GF(2^m)`, `N=2^m-1`)의 bit-level soft BP 디코딩. RS(15,11),(31,25),(255,239) 예제, BPSK/AWGN.
2. 문제: RS PCM의 binary image는 밀도 ~50%로 short cycle 다수 → 표준 BP가 message 상관·error propagation으로 실패. ADP[3]는 매 iteration Gaussian Elimination이 너무 복잡.
3. 저밀도 PCM: dual code의 저 hamming weight codeword를 support set `R`로 탐색해 밀도 ~30%로 낮춤.
4. 확장 PCM: 겹침 최대인 두 check를 골라 새 row/col 추가로 4-cycle 제거, 밀도 ~15%까지 축소(`r_e=(N-K)m·1.5`, `n_e=Nm+(N-K)m/2`). 추가 변수는 erasure(zero LLR) 처리.
5. 핵심: `BP_it-max`회 표준 BP 후 미수렴 시 information correction — 평균 LLR로 최소신뢰비트 `c_p` 선택, 채널정보를 `±inf`로 강제하고 `BP_add`회 재수렴.
6. 트리 확장: step `j`에서 이전 `2^(j-1)`비트 각각 2 test → `2^j`개 test로 후보 비트 확장(`IC_max`단계).
7. 기하학적 해석: BP는 gradient descent, potential `J(H_b,T)=-Σ Π T_p`. pseudo-equilibrium(local min)에 갇힐 때 신뢰도 변경으로 `J` 형태를 바꿔 탈출, codeword 도달 시 `J_min=-(n-k)`.
8. 부수: 고신뢰비트(`|ρ|>τ`)는 hard-decision으로 고정해 저신뢰비트 오염 방지. 매 BP iteration마다 BM 경판정 병행, 성공 codeword 저장 후 최소 Euclidean 선택. `d≤r` extra stopping으로 iteration 절감.
9. 결과: RS(15,11) CER 1e-3에서 BM 대비 2.5dB, KV 대비 1.5dB, ML과 0.25dB 이내. 평균 BP iteration은 SNR↑ 시 급감(RS(255,239) 6.5dB에서 2회).
10. 한계: HW 미설계, 시뮬만. 최대 360 iteration+BM 병행+트리탐색으로 복잡, RS(non-binary) 전용, full-tanh BP.
