### B. 알고리즘 요약

1. 대상은 두 regular LDPC: Tanner `[155,64,20]` (3,5) `R≈0.413`, girth 8, dmin 20, 그리고 IEEE 802.3an `[2048,1723]` (6,32) `R≈0.841`, girth 6 (NAND 아님, 일반 binary LDPC).
2. 문제: O(n) 반복복호는 고SNR에서 error-floor가 생기며 이는 code의 absorbing set(trapping set)에 error 패턴이 lock-up되기 때문. 왜/언제 dominant해지는지 규명.
3. 모델: AWGN에서 dominant trapping set = `absorbing set (a,b)` (a=크기, b=홀수연결 unsatisfied check 수), 이 집합의 message dynamics를 선형대수로 모델링.
4. 핵심 기법: absorbing set 내부 edge message를 `x_I = Σ(VC)^i λ + (VC)^{I-i} λ_ex`로 전개, `V`=변수노드 가산행렬, `C`=reflection permutation, 최대고유값 `µ_max`가 오류 LLR 증폭속도를 지배.
5. 식 의미: `µ_max`(예: Tanner (8,2) set에서 1.7870)가 클수록 set 내부 오류 신호가 빠르게 자라 floor 기여↑ — 낮은 무게 codeword가 error rate를 지배하는 것과 유사.
6. 확장: 오류확률 `P_AS = Q(...)`를 유도, 외부 extrinsic 평균 `m_λ(ex)`이 `µ_max` 성장을 앞지르면 floor 사라짐 → 핵심 제어변수는 LLR clipping threshold.
7. 구현 디테일: `tanh` 기반 대신 correcting-term 포함 corrected min-sum `λ_j→i = min|λ| + CT · Πsign` 사용(식 39), `CT`는 check degree 의존.
8. 최적화/검증: importance sampling(IS, mean-shift bias)으로 ultra-low BER(10^-20 이하) 영역까지 공식 검증, C++·FPGA 결과와 대조.
9. 결과: LLR clipping을 10→100→1000으로 키우면 floor가 수 자리수 저하; 대신 양자화 비트폭 필요량 6→10→14bit로 증가(clipping 10/100/1000 각각).
10. 한계: HW RTL 미설계, short code(Tanner)는 clipping>100에서 LLR 상관으로 추가이득 없음(DE 예측 이탈).
