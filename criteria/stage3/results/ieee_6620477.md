### B. 알고리즘 요약

1. MBIOS 채널의 binary LDPC를 대상, LP 완화 ML 복호 `min γᵀu s.t. Tju∈Pdj`를 ADMM으로 풂. 실험은 rate-0.77 (3,13)-regular n=1057, rate-1/3 irregular n=1920.
2. 문제: 범용 LP solver(MALP+GLPK)는 제약 sparsity를 못 살려 느리다 → 병렬 가능한 ADMM 반복이 대안.
3. 모델: check polytope `Pd` = 짝수 무게 binary 벡터의 convex hull, 각 check 제약 `Tju∈Pdj`.
4. 핵심 기법(ADMM): scaled augmented Lagrangian `Lρ`를 x-update `(5)`, z-update(projection) `(6)`, dual y-update `(7)`의 message-passing 반복으로 표현(Algorithm 1), 모든 check/variable 동시 갱신 가능.
5. 병목: 복잡도는 실벡터를 check polytope로 Euclidean 투영하는 `ΠPdj`가 지배 → projection 효율이 전체 속도를 결정.
6. 핵심 기여(projection): 기존 "two-slice"는 좌표를 내림차순 정렬해야 함. 제안법은 CSA(cut search, Algorithm 2)로 정렬 없이 각 좌표 1회 방문해 polytope 내부 여부·cutting set V를 판정.
7. 이론(Theorem 3): 외부 벡터의 hypercube 투영 z의 cutting set V가 있으면 `ΠPd(u)`는 facet `FV: θVᵀx=|V|-1` 위에 있음 → facet 투영 `x*=Π[0,1]d(u-ν*θV)`로 귀결(Algorithm 3에서 ν* 계산).
8. 부수 트릭: modified ADMM은 early termination + over-relaxation으로 추가 2배 가속.
9. 결과: 제안 projection이 [3]의 two-slice 대비 최소 3배 빠름, 모든 ADMM-LP가 MALP보다 빠르고 저rate 부호에서 복호시간 3자릿수 감소, FER는 Tmax=200에서 ALP와 동등하고 LP류 곡선이 SPA보다 가파름.
10. 한계: HW 미설계·C++ SW 실측뿐, LP/ADMM 패러다임이라 정정성능 자체 향상은 아니고 ML-LP 근사 속도 개선.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM-LP 복호 전체 | 대응 없음 | Prime ECC는 min-sum message-passing, LP/ADMM solver 없음 |
| check-polytope projection(CSA) | 대응 없음 | min-sum CN 연산과 수학적 성격 상이 |
| message-passing 병렬 갱신 구조 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `C2V_Cal()` | 반복 C2V/V2C 구조만 형식적 유사, 연산 내용 다름 |
| 조기종료(early termination) | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념만 유사, ADMM 수렴판정과 CRC는 다름 |

적용 가치: **낮음** - ADMM-LP는 min-sum과 별개 복호기라 Prime ECC HW 엔진에 이식 불가, 병렬 message-passing/조기종료 발상 정도만 형식적 참고.
