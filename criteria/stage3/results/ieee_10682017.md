### B. 알고리즘 요약

1. 대상 부호는 Bernoulli parity-check(BPC) 코드: 정보벡터 `u∈F_2^k`에 대해 `H=G^T`(G는 k×m Bernoulli(ρ) 랜덤 행렬, ρ≪1/2)로 정의되는 systematic LDGM 계열, code rate `R≥(k-m)/k`; 채널은 BPSK+AWGN, 디코딩은 sum-product(BP), 최대 iteration 50.
2. 기존 BGM 코드는 낮은 weight codeword로 인해 error floor가 높은데, BPC 코드(parity bit를 디코더가 이미 아는 systematic 변형)로 낮췄으나 추가 성능 개선 여지가 있음.
3. 핵심 가정: normal graph(variable+check 이분 그래프)를 하나의 simple network `G`(인접행렬 `A=[[0,H^T],[H,0]]`)로 변환해 복잡계 네트워크 이론의 assortativity(degree correlation) 개념을 그대로 적용.
4. 핵심 기법: assortativity coefficient `r = (1/σ_q^2) Σ_ij ij(e_ij - q_i q_j)`(식 4)로 variable-check 양단 degree 간 상관을 정량화하고, disassortative(r<0, 서로 다른 degree끼리 연결)할수록 성능 향상 가설을 세움.
5. 식의 의미: `e_ij`는 무작위 선택 edge가 degree i 노드와 degree j 노드를 연결할 결합확률, `r`이 음수면 고차수 variable node가 저차수 check node에 주로 연결되는 구조.
6. 핵심 기법 2단: NGCM(Normal Graph Configuration Model, Algorithm 1) — 목표 `r*`에 대해 연결확률 `P_ij(a)=1/Z·|(i-d_v)-(j-d_c)|^a`(식 5)를 정의하고 파라미터 `a`를 이진탐색해 configuration-model 방식으로 stub을 매칭, 목표 assortativity를 갖는 H를 생성.
7. 부수 트릭: `a`와 평균 `r̃` 관계를 사전 계산(Fig.4)해 이진탐색 범위 축소로 수렴 가속.
8. 최적화 절차: `r*`를 -0.45(원래 코드)에서 -0.68까지 스윕하며 BER/FER 최소화 지점(최적 disassortativity)을 실험적으로 탐색.
9. 결과: BPC 코드(k=2048,m=1024,ρ=0.005) 기준 r*=-0.68에서 BER 10^-4 기준 코딩이득 0.6dB, FER 10^-3 기준 0.4dB, 평균 iteration 수는 SNR 1.6dB에서 45→24로 감소(waterfall 영역).
10. 한계: 소규모(k=1024~2048) AWGN/BPSK 시뮬레이션뿐이며 HW 설계·양자화·QC 구조 고려 전혀 없음, 과도한 disassortativity는 오히려 성능 저하(최적점 존재).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BPC/BGM Bernoulli 랜덤 H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC(base+circulant) 고정 구조라 완전 랜덤 Bernoulli 생성 방식은 대응 없음에 가까움 |
| Normal graph configuration model(assortativity 제어) | 대응 없음 | H-matrix 설계 시 degree correlation을 제어하는 아이디어 자체는 QC-LDPC의 base matrix 설계에 개념적으로만 참고 가능(코드 모듈 대응 없음) |
| BP(sum-product) 디코딩, iteration 수 감소 | `decoder.cpp` `LDPC_Decoder()` | 이미 Min-Sum(SPA 근사) 보유, 논문은 SPA 자체가 아닌 그래프 구조 최적화로 iteration 감소를 달성해 상보적일 수 있으나 기반 부호구조가 다름 |

> 적용 가치: **낮음** — 대상 부호(Bernoulli 랜덤 LDGM/BPC)가 Prime ECC의 QC-LDPC 구조와 근본적으로 다르고 코드 모듈 대응이 거의 없음. 다만 "variable-check degree correlation을 조절해 BP 복호효율(iteration/error-floor)을 개선"한다는 설계 원리는 향후 QC-LDPC base matrix 설계 시 아이디어 차원에서 참고 가능.
