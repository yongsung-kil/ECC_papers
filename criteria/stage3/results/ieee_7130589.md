### B. 알고리즘 요약

1. 대상은 3D 지향 charge-trapping(CT) MLC NAND 소자의 신뢰성이며, LDPC 부호/디코더가 아니라 program 알고리즘·read-retry·SSD QoS를 다룬다.
2. 문제: CT-NAND는 shallow trap 전하손실로 BER·retention·endurance가 나빠 ECC/RR 개입이 늘고 SSD QoS가 저하된다.
3. 모델/셋업: 4Mbit 2D CT-NAND(p-Si/SiO2/Si3N4/SiO2/Si3N4/Al2O3 스택) 실측, MLC full-sequence, LP는 V2·UP는 (V1,V3)로 read.
4. 핵심기법: 표준 ISPP 대비 REC(Recovery) program — 매 program pulse 후 −10V/100μs soft-erase로 shallow trap 전하 제거, deep trap 전하로 VT 안정화.
5. read-retry(RR): BER>ECCth일 때 read reference 전압을 상·하로 시프트해 재읽기(페이지당 100 error를 ECCth 기준으로 사용).
6. 트레이드오프: REC는 program time 증가(L3 기준 ISPP 1.23ms → REC 6.97ms).
7. 시스템 평가: SSDExplorer co-simulation으로 multi-thread BCH ECC 엔진·PCIe Gen2 x8(유효 1150MB/s QoS) SSD 아키텍처의 throughput/uncorrectable page 평가.
8. (LDPC는 서론에서 BCH와 함께 ECC 옵션으로 언급될 뿐, 실제 모델·시뮬은 BCH 엔진 사용.)
9. 결과: RR 적용 시 REC의 LP-BER가 20k cycle까지 <1e-3 유지, uncorrectable page 0% 유지(ISPP는 5k에서 전멸), enterprise endurance 약 4배(3k→12k P/E) 향상.
10. 한계: ECC 부호설계·디코더·BER 곡선(부호 관점) 없음 — 소자/시스템 신뢰성 연구로 LDPC ECC 이식 대상 아님.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CT-NAND program 알고리즘(REC)·read-retry | 대응 없음 | NAND 소자/컨트롤러 read 정책으로 LDPC 디코더·부호 로직과 무관 |
| RR read reference 전압 시프트 | channel.cpp Set_R_Offset(), Set_LLR_Th() | 개념적으로 read offset과 닿으나 논문은 LDPC soft-read LLR을 다루지 않음 |
| 시스템 ECC 엔진(BCH multi-thread) | 대응 없음 | Prime ECC는 QC-LDPC이며 본 논문은 BCH 엔진 사용 |
```

적용 가치: **낮음** — CT-NAND 소자 신뢰성·SSD QoS 논문으로, Prime ECC의 binary QC-LDPC 디코더/부호/HW에 떼어 쓸 알고리즘 기법이 없다(REC로 개선된 RBER은 채널 조건 참고자료 수준).
