> 총평: OMS의 단일 β를 edge·iteration별 학습 offset으로 일반화해 곱셈 없이 min-sum 성능을 향상하는 HW친화적 기법으로, Prime ECC min-sum 디코더에 파라미터만 교체하는 형태로 이식 가능하나 LDPC·고rate 재검증 필요.

### B. 알고리즘 요약

1. 대상은 BPSK/AWGN 채널의 binary linear code로, LLR `lv=2yv/σ²` 초기화 후 Tanner graph 위 iterative message passing(BP) 복호를 수행한다.
2. 문제: min-sum은 SPA 대비 BER 열화가 있고, 곱셈 기반 neural BP(Nachmani et al. [4])는 성능은 좋으나 다수 곱셈으로 HW 비용이 커 실시간 구현이 어렵다.
3. 핵심 기법(NOMS): offset min-sum `μ_c,v = ReLU(min|μ_v,c| - β)·∏sign`에서 전역 β를 edge·iteration별 학습 파라미터 `β_{c,v}^t`로 대체한다(식15).
4. 이 오프셋은 min-sum 근사오차 보정과 Tanner graph의 짧은 cycle 감쇠라는 이중 역할을 하며, 곱셈 대신 뺄셈만 사용해 OMS 대비 추가 산술연산이 없다.
5. 학습: 디코더를 T iter unroll한 신경망으로 보고, cross-entropy loss `J=E[H(X,S)]`를 minibatch SGD(Adam, lr=0.1)로 최소화, all-zeros codeword+AWGN으로 학습데이터 생성.
6. 비미분점(ReLU/min/abs/sign)은 subgradient로 backprop(TensorFlow), sign 미분이 0이라 gradient는 사실상 ReLU 분기로만 흐른다.
7. 파라미터 수: `(n,k)` 코드, edge E, iter T 기준 neural SPA는 `nT+ET(dc-1)+n+nE`개·다수 곱셈이 필요하나 NOMS는 `ET`개 오프셋만 필요(곱셈 대비 절반 미만).
8. 오프셋 제약: 여러 edge가 오프셋을 공유하도록 제약 가능하며, 단일 오프셋으로 제약하면 NOMS는 OMS로 환원(OMS β를 학습으로 정하는 새 방법).
9. 결과: BCH(63,36)/(63,45)/(127,106) 5-iter unroll, SPA 대비 최대 1 dB, neural SPA와 0.1 dB 이내. 오프셋 히스토그램에서 학습 후 iter별 두 모드(β~0대와 ~7) 형성 → 근사보정+cycle 감쇠 학습 관찰.
10. 한계: HW 미설계·시뮬만, 검증은 BCH(HDPC) 5 iter뿐이고 LDPC는 미검증(future work), edge별 오프셋 테이블 저장 부담.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 연산 (β 적용) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum(min1/min2+sign)에 학습 offset β_{c,v}^t를 감산항으로 삽입 가능 |
| iteration별 offset 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iter 구간별 LLR threshold 테이블 구조와 유사 - offset 테이블로 확장 가능 |
| 복호 이터레이션 루프 | `decoder.cpp` `LDPC_Decoder()` | unroll·offset 적용 지점, 알고리즘 골격 그대로 재사용 |
| 오프라인 학습(SGD) | 대응 없음 | Prime ECC는 학습 파이프라인 없음, offset은 오프라인 학습 후 테이블만 이식 |

적용 가치: **중간** - min-sum offset을 edge·iter별로 학습해 곱셈 없이 waterfall 성능을 올리는 방향이 Prime ECC min-sum 디코더 철학과 정합하고 `CNU_Update` 감산항 삽입으로 이식 용이하나, 검증이 BCH 5-iter뿐이라 고rate QC-LDPC·다iter에서 이득·오프셋 저장비용 재검증이 필요하다.
