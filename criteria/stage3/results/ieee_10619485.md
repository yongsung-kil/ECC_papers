### B. 알고리즘 요약

1. 시스템은 이진 선형블록부호 `C[n,k]`를 BIOS(binary-input output-symmetric) 채널로 전송, BPSK-AWGN에서 시뮬레이션(예: `n=128, k=64`).
2. 기존 OSD는 재부호화 복잡도 `O(k^t)`와 순차(serial) Gaussian Elimination(GE) 복잡도 `O(k^3)`가 지연의 주 원인이며, 특히 순차 GE는 병렬화가 어려움.
3. 제안 코드(SGMC, staircase generator matrix code)는 생성행렬을 계단형(`gℓ = (hℓ, iℓ, jℓ)`, `iℓ`은 폭 `wℓ`의 all-one 시퀀스)으로 구성하여 GE를 열 단위로 병렬 수행 가능하게 함(Algorithm 1, `RowReduction(ℓ)`).
4. `hℓ`(계단 아래 원소)을 성공확률 1/2 베르누이 과정으로 랜덤 샘플링한 것이 random SGMC이며, 무한장 영역에서 BIOS 채널에 대해 capacity-achieving임을 부분 오차지수(`E(wℓ,Rℓ)`)로 증명(Theorem 1, 2).
5. 유한장 영역 복호는 LC-ROSD(Representative OSD with local constraints): `k+δ` 비트로 확장 MRB를 구성하고, 부분 연판정 가중치 `Γ(e)=Σ eᵢ|λᵢ|`가 비감소하는 순서로 TEP(test error pattern)을 탐색.
6. 성능 분석용으로 앙상블 weight spectrum `B(X)=1+Σ 2^ℓ X^{wℓ}(1/2+1/2X)^{nℓ-wℓ}`과, 기존 RCU보다 타이트한 partially RCU bound(식 12-14)를 유도해 프로파일(w0 등 폭 배분) 설계 기준으로 사용.
7. 열 배치는 각 계단마다 가장 신뢰도 높은 비트를 선택(§III-A i,ii)하여 permutation `Π`를 구성, 순차 GE 대신 병렬 GE 적용.
8. 프로파일 최적화는 이론적 NU(nearly uniform) 프로파일(`w=(28,2,2,...,1,1,...)`)을 RM 프로파일과 비교하는 형태로 진행(별도 학습 알고리즘 없음).
9. 결과: `n=128,k=64`에서 5G LDPC 대비 최대 1.5dB, 5G polar 대비 0.7dB, RM 대비 0.6dB 개선(FER=1e-5); LC-ROSD는 원 OSD 대비 복호시간 1000배, LC-OSD 대비 고SNR에서 최대 75% 추가 지연 감소.
10. 한계: 순수 이론/시뮬레이션(FPGA/ASIC 미검증), short block(n=128) 한정, LDPC가 아닌 별도 부호 클래스라 기존 LDPC 인프라에 직접 적용 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SGMC 부호 구성(계단형 생성행렬) | 대응 없음 | Prime ECC는 QC-LDPC(base+circulant) 전용, staircase generator matrix 구조 자체가 이질적 |
| LC-ROSD / 병렬 GE | 대응 없음 | min-sum 메시지패싱이 아닌 OSD/GE 기반 복호로 `decoder.cpp` 구조와 무관 |
| weight spectrum / partially RCU bound | 대응 없음 | 순수 이론적 성능 분석 도구, HW 이식 대상 아님 |
| soft LLR 기반 TEP 정렬(`Γ(e)`) | `channel.cpp` `Set_LLR_Th()` | 연판정 LLR 활용 개념만 유사, 알고리즘 자체는 이식 불가 |

적용 가치: **낮음** — LDPC가 아닌 별도 부호 클래스와 OSD 계열 복호로, Prime ECC의 QC-LDPC min-sum 디코더 구조와 근본적으로 다른 접근이라 이식 여지가 거의 없음.
