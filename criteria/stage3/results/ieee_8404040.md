### B. 알고리즘 요약

1. 시스템: 스트리밍 소스가 매 시각 `K`-bit 메시지를 내보내고, anytime encoder가 과거 전체를 누적해 `M`-bit `xt`를 생성(rate `Rc=K/M`), BEC/BIAWGN 채널로 전송.
2. 문제: 블록/convolutional 부호는 무한 delay에서만 신뢰성 확보→불안정 제어 루프. anytime 부호는 오류확률이 delay에 지수감쇠(`Pe(i,d)≤β e^{-αd}`)해야 함.
3. 모델: (Q,A)-regular RA protograph를 `L`개 chain으로 spatially couple, lifting factor `M`으로 lifting한 SC-RA 부호. 대표 예제 `(Q,A)=(4,4)`, rate `Rd=1/2`.
4. 핵심 기법: message bit node 위치 `i`의 `Q` 연결을 `[i, i+γ-1]` 범위에서 **지수분포**(`Pr(k)=e^{-kλ}(1-e^{-λ})`)로 배치 → 최근 메시지 빠른 복구(anytime 특성).
5. 식 의미: 지수 rate `λ`가 낮고 coupling `γ`가 클수록 error-floor 낮으나, 실용상 유한 메모리/높은 delay-exponent가 필요해 floor가 재등장.
6. 확장: floor 완화를 위한 **피드백** 도입. BEC는 delay-threshold 기반 재전송(가장 이른 erased bit index 요청), BIAWGN은 재전송 대신 **적응형 PCM**(tagged 불신 bit에 `E`개 edge 추가).
7. 구현 디테일: BEC는 peeling decoder로 window 크기를 erased bit 수에만 종속시켜 복잡도 절감; 피드백으로 window 크기를 상수로 유지.
8. 최적화: BEC/BIAWGN density evolution recursion 유도(정보/parity bit 별 erasure확률·LLR평균), max iteration 200으로 finite-length(M=16) 시뮬과 비교.
9. 결과: 제안 SC-RA anytime 부호가 SC-LDPC anytime 부호[6]보다 우수, 피드백으로 floor 제거+decoder 복잡도 감소, 항공기 추적(Kalman)에서 블록부호 대비 오차 근제로.
10. 한계: HW 미설계·순수 시뮬/이론, NAND 무관(스트리밍+무손실 피드백 링크 전제), rate 예제 0.5 고정 저rate.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-RA anytime 부호구성(지수분포 coupling) | 대응 없음 | Prime ECC는 고정 QC-LDPC block 부호, SC/streaming 구조 아님 |
| BP LLR message passing | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 알고리즘은 표준 SPA `tanh`, min-sum과 별개·비HW |
| 피드백 기반 적응형 PCM | 대응 없음 | NAND는 피드백 재전송/PCM 변경 불가 |
| BEC peeling decoder | 대응 없음 | Prime ECC는 AWGN/RBER 대상, BEC peeling 미사용 |

적용 가치: 낮음 — anytime/streaming + 피드백이라는 통신제어 전용 프레임워크로, 블록 단위·피드백 불가한 NAND binary QC-LDPC ECC와 근본 부정합이며 HW 설계도 없음.
