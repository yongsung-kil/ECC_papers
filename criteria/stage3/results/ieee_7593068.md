> 총평: BEC 전용 near-ML 소거복호(wrap-around sliding-window)라 채널·복호모델이 NAND(BSC/AWGN soft)와 근본적으로 달라 Prime ECC min-sum 파이프라인에 이식 부적합.

### B. 알고리즘 요약

1. 이진 QC-LDPC 블록부호를 BEC상에서 low-complexity near-ML로 복호하는 알고리즘 제안 (rate 1/2, 길이 1026~12000).
2. 문제: BEC ML 복호는 소거 `v`개에 대한 선형방정식 풀이(`O(v^2)`~`O(v^3)`)로 수천 bit 코드에서 비현실적.
3. 핵심 가정: QC-LDPC 블록부호를 다항식 PCM `H(D)` 기반 tail-biting 부모 convolutional code로 간주 (`h_ij(D)∈{0,D^wij}`, 신드롬 메모리 `μ`).
4. 핵심 기법: BP 복호 후, 짧은 sliding window에 zero-tail terminated LDPC convolutional code의 ML 복호를 `quasi-cyclically` 적용 (PCM 식 3, window `W≥2μ+1`).
5. wrap-around: 수신열을 따라 window를 순환 시프트(`S=e` bit)하며 소거를 반복 정정, 새 정정이 없으면 종료 (Algorithm 2). 복잡도는 window 길이엔 다항, 코드길이엔 선형.
6. tail-biting convolutional의 wrap-around 준최적 복호와 유사, sparsity로 계산량 절감.
7. 이론: row-regular LDPC 앙상블 `E(n,r,K)`의 평균 ML 소거복호 오류확률 상한 유도 (Theorem 1, 식 6~10), 소거정정 실패=식 (2) 해 비유일 확률.
8. 부수: window shift `S` 크면 속도↑ 성능↓, `S=e`가 최저 FER.
9. 결과: class A/B/C(irregular R=1/2, (3,6), (4,8)-regular) QC-LDPC에서 WML이 BP 능가, (4,8) class C에서 ML-BP threshold차·최소거리 최대라 이득 최대 (FER 10^-3 기준 소거확률 개선).
10. 한계: HW 미설계, 채널이 BEC 소거 전용, 메모리 제약으로 코드길이≫window면 앙상블 ML 성능 미달.
