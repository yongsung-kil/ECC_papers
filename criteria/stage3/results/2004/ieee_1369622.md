### B. 알고리즘 요약

1. 대상: 일반 binary LDPC(regular 1200/600·8000/4000, irregular 1268/456·3072/1024), BPSK, AWGN/Rayleigh/BSC 채널.
2. 문제: 짧은 블록 LDPC의 TG에 작은 cycle이 많아 flooding 스케줄이 최적(cycle-free) 대비 조기에 suboptimal해지고 error-floor 발생.
3. 핵심 아이디어: 각 노드/엣지의 outgoing 메시지가 처음으로 최적성을 잃는 iteration을 girth `g`와 최단 nontrivial closed-walk 길이 `p`로 규명 (Lemma 1/2, Theorem 1/2).
4. 결정 스케줄: 노드/엣지에 `초기값 = ceil((g+2)/4)`(또는 `p` 기반) 카운터 부여, 0 되면 갱신 중단, 전체 0 되면 리셋·반복.
5. 확률 스케줄: 노드/엣지가 자기 girth에 비례한 확률 `g/g_max` 등으로 갱신 → girth 큰 노드가 더 자주 갱신.
6. 분류 축: node-based vs edge-based, unidirectional vs bidirectional, deterministic vs probabilistic 조합.
7. 핵심 관찰: edge girth 분포가 node girth와 달라, edge-based가 특정 코드(GA/BSC)에서 node-based보다 우수(>0.3dB@BER).
8. 적용 알고리즘: BP(sum-product), min-sum, Gallager-A 모두에 스케줄 적용, 채널별로 최적 스케줄이 달라짐.
9. 결과: Code II(8000,4000) BP에서 양방향 edge 스케줄이 flooding의 early error-floor 제거, BER 2자릿수 이상 개선.
10. 대가: 스케줄일수록 평균 iteration 증가(양방향 ~50%↑, 22→33회), edge/양방향은 구현 복잡도↑.
11. 한계: HW 미설계·순수 시뮬, QC 구조 아님, 스케줄이 규칙적 병렬 처리와 상충.
