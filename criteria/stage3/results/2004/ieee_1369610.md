### B. 알고리즘 요약

1. 대상: binary LDPC, BPSK/AWGN, 짧은 부호 (C1 irregular (1000,500) BP, C2 regular (273,191) GA, C3 PG-LDPC (273,191) SS). 최대 200 iteration.
2. 문제: TG와 복호 연산이 고정된 상태에서 message-passing '스케줄'만 바꿔 짧은 부호(짧은 cycle 다수로 BP 준최적)의 성능/속도를 개선.
3. 아이디어: 신뢰도 낮은 비트의 메시지가 초기 iteration에 그래프로 전파되는 것을 막는 reliability-based schedule(RBS). 기존 graph-based(girth 기반) 스케줄과 달리 TG에 직접 의존하지 않음.
4. 핵심 정의: iteration `k`, 비트 `i`의 신뢰도 추정 `r`(예: LLR magnitude 또는 수신값 `|y_i|`)이 임계값 미만이면 unreliable로 라벨; check node는 연결된 다른 모든 비트가 reliable일 때만 reliable.
5. 동작: reliable 비트만 메시지 송신, reliable check node만 메시지 생성/회신 (bit/check 노드 내부 연산은 flooding과 동일).
6. 단순화: 스케줄을 첫 iteration에만 적용(2번째부터 전부 reliable=flooding), 임계값 `T`를 모든 비트 공통 상수로 → 최적화 차원 1로 축소. 정규화 임계값은 `E_b/N_0`에 둔감.
7. 구현 비용: hard-decision 알고리즘에서 초기 메시지에 reliable/unreliable 표시 1비트만 추가(첫 iter만 사용); 이후 교환 메시지는 여전히 binary.
8. 적용 알고리즘: BP, Gallager A(GA), Sipser-Spielman(SS, 병렬 BF). 최적 정규화 T ≈ 0.3(C1,BP)/0.7(C2,GA)/0.4(C3,SS).
9. 결과: BER 1e-4에서 RBS-GA가 flooding GA 대비 약 1 dB 개선, BF-WBF 성능 격차 대부분 해소(WBF 대비 0.1~0.25 dB 이내), 수렴 iteration 2배 이상 감소(SS 예). BP엔 개선 미미(단 iteration 표준편차는 크게 감소).
10. 한계: HW 미설계, hard-decision/BF 위주 효과, 짧은 부호 AWGN 시뮬뿐, soft BP(min-sum류)엔 이득 작음.
