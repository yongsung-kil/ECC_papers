### B. 알고리즘 요약

1. 대상은 array LDPC 계열 binary LDPC, DSL 채널(BI-AWGN), rate 0.8~0.909, 길이 999~1675.
2. 문제: DSL은 채널 가변성으로 다수의 rate/length를 지원해야 하나 랜덤 puncturing 코드는 패밀리마다 PCM 저장이 필요해 구현 비용이 큼.
3. rate-compatibility 조건: puncturing 패턴 `v(A)`가 `vi(A0)=1 ⇒ vi(A)=1 ∀A≥A0` (고rate 코드 비트가 저rate 코드 비트의 부분집합).
4. array LDPC PCM `HA`는 소수 `p`, 정수 `k,j`로 지정, 블록은 `p×p` 항등/순환 permutation `α` 거듭제곱, 설계상 4-cycle 없음.
5. 효율적 선형 인코딩 위해 `HA`를 상삼각형 `H`(식7)로 변환; `N=kp, M=jp, K=(k−j)p`, 삼각화로 약한 irregularity 유발(식8,9의 degree 분포).
6. puncturing은 단순 erasure 시 성능급감하므로 심볼 puncturing에 parity-check 방정식 pruning을 결합(`ΔM≥(M/N)ΔN`, 식3).
7. nested PCM: 좌측 `p`컬럼·상단 `p`행을 순차 제거해 rate 0.800→0.833→0.869→0.909 패밀리 생성(예: j=5,p=67,k=25).
8. 코드는 단일 파라미터 `q`(0≤q≤3p)로 유일 지정, 중간 rate도 미세 granularity로 달성.
9. 결과: (999,888) punctured array 코드가 동급 랜덤 코드와 유사 성능(고SNR 소폭 손실), rate-1340/1675는 j=5 threshold에서 0.8dB, 고puncturing(1340/1474)은 손실 큼.
10. 한계: 디코더는 [15]의 flexible switch-unit 아키텍처 언급뿐 본 논문서 HW 미설계, puncturing이 심할수록 성능 저하.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| array LDPC 상삼각 PCM 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | QC계열 부호구조 로드, 그러나 rate-compatible 삼각 구조는 고정 QC와 상이 |
| puncturing 기반 rate 조절 | 대응 없음 | Prime ECC는 punct_col_num=2 고정, 가변 rate 패밀리 미지원 |
| 선형 인코딩 상삼각 구조 | [encoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코더로 삼각 puncturing과 구조 불일치 |

적용 가치: **낮음** — DSL의 다중 rate 지원을 위한 puncturing/pruning 구성법으로, 단일 고정 rate·고정 QC 구조인 Prime ECC에는 목적·구조 모두 부합하지 않음.
