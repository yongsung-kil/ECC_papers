### B. 알고리즘 요약

1. rate 1/2 protograph 코드를 대상: `(n,u,r)` protograph(n=전송 VN, u=비전송 VN, r=CN)에서 rate `k/n ≥ (n+u-r)/n`.
2. 문제: 구조적 규칙성을 유지하면서(HW 간단·저복잡 인코딩) 우수한 threshold를 얻는 LDPC 설계.
3. base graph(protograph)를 T copy 복제 후 각 edge `e`를 길이 T의 permutation `π_e`로 연결 → Tanner graph 생성.
4. 핵심1: base graph를 simulated annealing으로 탐색(edge insert/delete/interchange)해 density evolution decoding threshold를 최소화.
5. threshold는 reciprocal channel approximation[3]으로 근사 계산 → DE 전체 반복 없이 빠른 평가.
6. 핵심2: 확장 시 각 `π_e`를 circulant permutation(순환 shift identity) 집합에서 선택 → QC 구조 확보(빠른 인코더).
7. circulant 선택은 PEG 변형으로 순차 진행, 각 단계에서 minimum loop length를 최대화(작은 loop·저중량 codeword 억제).
8. minimum loop length는 그래프상 Viterbi-like 탐색으로 판정.
9. 결과: `(4,1,3)` protograph threshold 0.5 dB(ARA 코드와 유사), `(6,0,3)` 0.65 dB(확장/복호/채널 변화에 robust). k≈4000 두 코드 시뮬로 WER/SER 제시.
10. 한계: HW 미설계(참조[6] 언급만), 2쪽 요약이라 세부 파라미터 부족, 통신 rate 1/2 AWGN 가정.

### C. Prime ECC 관련 모듈 핀포인트

| protograph+circulant 부호구조 | `ecc_top.cpp` `Load_PCM()` | Prime ECC의 base matrix+circulant shift QC-LDPC와 구성 원리 동일, H-matrix 설계 방법론 참고 |
| circulant permutation(순환 shift) 선택 | `ecc_data.h` `PCM_b`(base+shift) | shift 값 결정 방식(loop length 최대화)을 우리 PCM 설계에 참고 가능 |
| PEG 변형 loop length 최대화 | 대응 없음 | 부호설계 오프라인 도구 영역, 런타임 디코더 모듈과 무관 |
| density evolution threshold 최적화 | 대응 없음 | 부호설계 단계 최적화, 코드베이스에 미보유 |

적용 가치: **중간** — Prime ECC와 동일한 QC-LDPC(base+circulant) 계열이라 설계 방법론(loop length 최대화 shift 선택)은 참고 가치가 있으나, rate 1/2 통신용이며 HW·세부 파라미터가 없어 직접 이식은 제한적.
