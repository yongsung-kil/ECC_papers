### B. 알고리즘 요약

1. rate-1/2 regular LDPC(예시 (192,96), row weight 6, column weight 3)를 QPSK/AWGN 채널에서 다룬다.
2. 문제: LDPC 복호의 입력 양자화 비트수는 HW 복잡도와 BER 사이 trade-off로 정해야 하고, 채널신뢰도 `Lc=2/σ²`를 Log-MAP 복호에 넣는 방식이 복잡도·성능에 큰 영향을 준다.
3. 복호는 log-domain sum-product: 초기화 `Lq_{n→m}=Lc·r_n`, check node update는 `Φ=Σ ln tanh(Lq/2)`를 LUT로 계산, bit node update는 `Lq=Lin+ΣLr`.
4. 핵심 트릭1: `Lc`를 부동소수 대신 {1,2,4,...} 이산값으로 근사하면 곱셈을 좌측 shift(`<<L*`)로 대체 가능 → HW 곱셈기 제거(bit-shifting).
5. Table 1: 추정 표준편차 σ*의 범위별로 Lc*와 shift량 L*를 non-uniform quantization으로 매핑.
6. 핵심 트릭2/분석: noise variance 추정오차를 `σ*=X·σ`로 두고 X를 스캔, BER 열화가 무시되는 범위가 X∈[0.8, 1.15]임을 Monte-Carlo로 확인(Eb/N0=0,1,2dB).
7. 양자화 파라미터 튜닝: 입력 n비트, bit-shift 후 m비트, edge metric r/q, LLR L 비트를 스윕.
8. 결과: `(n=6, m=8, r=8, q=8, L=8)` 조합이 float 시뮬과 거의 동일한 BER; 최적 파라미터 표(입력 6bit, 나머지 8bit) 제시.
9. Lc를 이산화해도 σ* 오차가 허용범위 내면 성능 열화 없음.
10. 한계: HW 미제작(향후 FPGA 목표), 소형 regular 부호·AWGN·QPSK 가정, 그림/표 OCR 손상.
