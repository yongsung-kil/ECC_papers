### B. 알고리즘 요약

1. 시스템: IRA(irregular repeat-accumulate) 부호, systematic, rate `R=1/3`, blocksize 1000/5000, BPSK/AWGN, 무선(UMTS Turbo 비교) 타깃.
2. 문제: IRA/LDPC 복호 병목은 Tanner graph 연결의 randomness → 완전병렬은 wire routing 불가, 임의 permutation은 동시 RAM 접근 충돌 유발.
3. 부호구조: N variable = K info node(IN) + M parity node(PN), IN은 degree i로 CN 연결(`fi`), CN↔PN은 고정 zigzag(accumulator = memory-1 RSC, 2-state trellis) → 선형시간 인코딩.
4. 핵심기법 1단: joint graph-decoder 설계 — 주어진 permutation network 제약 하에서 통신성능 좋은 IRA 부호를 탐색.
5. 병렬 네트워크: 단순 shuffling network(P 입력→P 개별 target 메모리)로 Π 구현, 하지만 규칙성 때문에 length≤4 짧은 사이클 다발.
6. 핵심 트릭: target RAM의 정교한 addressing scheme으로 girth≥6(length≥6 cycle) 보장 → 충돌 없이 성능 유지, 이 shuffling+addressing이 인코딩용 인터리버 Π를 결정.
7. 복호: SP 알고리즘 + accumulator에 대한 MAP(2-state trellis, windowing으로 병렬화), 식 (2)(3)의 `log(tanh(λ/2))` CN 연산.
8. 보정: Max-Log-MAP의 낙관적 편향을 extrinsic scaling factor `ESF=0.75`로 보정해 Log-MAP에 근접.
9. 아키텍처: P=4 예시 — IN/CNin/CNout 유닛 + shuffling network Π/Π⁻¹ + parallel MAP + 뱅크 메모리, degree j 노드는 `2·j` cycle 소요, 최대 degree/blocksize만 제약.
10. 결과: blocksize 5000, FER 10⁻³에서 IRA(P=10) Turbo(UMTS) 대비 0.2dB 우위, IRA 40 iter vs TC 10 iter; P=20 설계는 최소거리 저하로 error-floor 발생.
11. 한계: throughput/gate 수치 없음, 무선 rate 1/3 타깃, IRA(repeat-accumulate) 구조라 고rate NAND QC-LDPC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IRA 부호구조 (repeat-accumulate + zigzag PN) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | Prime ECC는 QC-LDPC base+shift 구조, IRA와 부호계열 상이 |
| CN의 log(tanh) SP + accumulator MAP | 대응 없음 | Prime ECC는 min-sum(비 full-tanh), MAP/trellis 복호 없음 |
| conflict-free RAM 접근용 shuffling+addressing | [GT.cpp](../Prime_ECC_3.1_Claude/) `Make_GT_HW()` (edge/MUX) | 개념적 유사(메모리 충돌 회피)일 뿐, QC circulant 구조 기반이라 방법론 이식 부적합 |
| ESF=0.75 extrinsic scaling | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | normalized/offset factor로 흡수 가능하나 MAP 전용이라 직접 이식 아님 |

적용 가치: **낮음** — IRA(SP+MAP accumulator) 부호/복호 계열이 Prime ECC의 binary QC-LDPC min-sum과 근본적으로 다르고, conflict-free 병렬화 방법론도 QC circulant 구조에 직접 대응되지 않음.
