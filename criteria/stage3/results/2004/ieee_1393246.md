### B. 알고리즘 요약

1. 목표: 코드마다 별도 디코더를 설계하지 않고 다양한 structured LDPC를 재합성으로 평가하는 범용 partially-parallel FPGA 디코더 제공.
2. 문제: full-parallel은 라우팅이 칩면적 지배, full-serial은 throughput/latency 열악 → 둘의 절충인 부분병렬이 필요.
3. 대상 부호: DDS(disjoint difference set) 기반과 array-based 등 quasi-cyclic 고rate LDPC(순환 서브행렬 구조로 라우팅 단순화).
4. 아키텍처: 공유메모리(Fig.4b) 기반 partially-parallel — IMEM, BCU(bit node), CSU, CCU(check node), EMEM(edge memory), 재구성 인터커넥트, 주소 컨트롤러(AC).
5. 디코딩: bit node 합산 + check node에서 sign/amplitude 계산(Eq.4, min 연산과 XOR sign)의 반복 message passing.
6. 메모리 관리 핵심: edge memory를 서브행렬별 `en = Σ d_i` 블록으로 나눔, `d_i`는 이분그래프의 edge chromatic number(엣지컬러링 문제) — 이것이 partially-parallel 라우팅의 핵심 트릭.
7. 주소 컨트롤러: QC코드(초기주소만), 산술계산형, 저장위치형 3가지 아키텍처를 코드 구조에 맞춰 선택.
8. LLR 폭 `dw`는 6bit면 충분, 블록크기 `N`(데이터저장용 ~4000)은 요구에 따라 변경 가능.
9. 결과: Xilinx Virtex-II XC2V6000, array-based(N=4617, rate 8/9) 200MHz에서 2.7Gbps/iteration + 118k bit 메모리, DDS(N=4923) 0.9Gbps/iteration. C 시뮬 대비 대폭 가속.
10. 한계: 정정성능/알고리즘 개선 없는 순수 평가용 인프라, 특정 채널·부호 최적화 아님, gate/전력 정량치 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 반복 message passing 디코딩 루프 | `decoder.cpp` `LDPC_Decoder()` | 개념적 대응이나 우리는 고정 QC-LDPC pipeline이라 범용 재구성 구조는 불필요 |
| check node sign/amplitude(min) 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum류 CN 연산으로 이미 보유 |
| edge memory / 메모리 라우팅 관리 | 대응 없음 | 우리는 z=32 고정 구조·GT(`GT.cpp`)로 처리, 엣지컬러링식 범용 메모리 관리는 미사용 |
| H-matrix 파라미터 로드/재구성 | `ecc_top.cpp` `Load_PCM()` | 우리도 PCM 로드하나 범용 재합성이 목적 아님 |

적용 가치: **낮음** — 신규 알고리즘·정정성능 기여 없이 다중 코드 평가용 재구성 FPGA 인프라라 고정 QC-LDPC HW인 Prime ECC에 떼어 쓸 요소가 없다.
