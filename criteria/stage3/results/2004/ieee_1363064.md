### B. 알고리즘 요약

1. 대상: IRA(Irregular Repeat-Accumulate) 부호, LDPC의 하위클래스, 선형시간 인코딩(zigzag accumulator) 가능. rate `R=0.33~0.9`, 최대 `I=5000` 정보비트.
2. 문제: LDPC/IRA 디코더의 permutation network(Tanner 그래프 연결)가 고병렬시 라우팅·메모리 병목이라 유연·고throughput 구현이 어렵다.
3. 부호 구조: 정보노드(IN, 가변 degree `fi`) + 패리티노드(PN, zigzag) + 체크노드(CN, `a`개 IN 연결), 연결 `Π`가 error-floor 결정.
4. 복호: LLR message passing, VN update `λk=λch+Σλl`, CN update `tanh(λk/2)=Πtanh(λl/2)` (sum-product, tanh LUT).
5. 아키텍처: 부분병렬 P (동시처리 edge 수), IN/PN branch 분리, CN유닛이 IN/PN 두 입력 병렬 처리, permutation network + zigzag network + 메시지/주소/permutation RAM.
6. joint graph-decoder 설계: permutation network(shuffling) 제약 하에서 IRA ensemble 중 성능저하 없는 부호 선택, girth>=6 위해 addressing scheme 적용.
7. 양자화: 6bit 메시지시 무한정밀 대비 손실 <=0.1dB, 5bit시 0.1~0.2dB.
8. throughput 모델: `T=I/(#cycles)·fcycle`, `#cycles≈It·2·EΠ/P`, `fcycle=334MHz`.
9. 결과: 0.13µm 합성, P=40~80에서 면적 7.09~9.35mm2, 30 iteration시 throughput 53~106Mbit/s(~100Mbit/s), IRA(30 iter)가 CDMA2000 Turbo-Code(10 iter)와 동등, 40 iter시 상회.
10. 한계: 통신채널(AWGN/BPSK, FER-vs-Eb/N0) 대상, NAND 무관, 정정능력 개선 아닌 구현 트레이드오프 분석, tanh sum-product(우리 min-sum 아님).

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| LLR message passing / CN·VN update | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()`, `C2V_Cal()` | 개념적 대응이나 tanh sum-product라 우리 min-sum과 알고리즘 상이 |
| 6bit 메시지 양자화 | [channel.cpp](../Prime_ECC_3.1_Claude/) LLR 테이블(`PARAM_LLR`) | 양자화 손실 데이터는 참고 가능하나 soft-4bit+라 우리(2SD/3SD)와 불일치 |
| IRA 부호구조(zigzag/accumulate) | 대응 없음 | QC-LDPC 고정 코드베이스, IRA는 이질적 부호클래스 |
| permutation network / zigzag network | 대응 없음 | 우리는 circulant shift 기반, permutation RAM 구조 미보유 |

적용 가치: **낮음** — IRA는 우리 binary QC-LDPC와 다른 부호클래스이고, permutation-network·tanh sum-product 아키텍처가 z=32 circulant/min-sum 구조와 이질적. 양자화 손실 참고 정도만 유효.
