### B. 알고리즘 요약

1. 대상: turbo-like 부호(PCCC/SCCC/LDPC)의 고속 병렬 반복 디코더에서 메모리 뱅크 접근 충돌 문제. 부호 rate/length는 일반화(구체값 미상).
2. 문제: 병렬 프로세서가 동시에 같은 메모리 뱅크에 접근하면 collision → 직렬화로 병렬성 저하. 기존엔 collision-free를 위해 전용 인터리버/PCM 설계에 의존.
3. 주장: 어떤 부호·어떤 read/write 스케줄이라도 변수의 메모리 매핑을 적절히 고르면 collision-free 가능 → 전용 부호설계 불필요(구성적 증명).
4. 형식화: 원소 집합에 두 분할(`P1`, `P2`, 각 서브셋 크기 M)이 주어질 때, 같은 서브셋 원소를 서로 다른 bin에 보내는 mapping function 탐색. graph coloring / Latin square 문제로 환원.
5. 알고리즘 1단계: (1),(2) 제약을 위반하지 않되 일부 미정(blank)을 남긴 예비 매핑 생성.
6. 알고리즘 2단계(annealing): blank에서 시작해 collision을 의도적으로 주입하고 각 스텝에서 다른 collision으로 넘기며 해소, cycle 종료 조건 충족까지 반복.
7. 정당성: Proposition 1으로 단일 cycle이 루프에 빠지지 않음을 증명 → Corollary 2로 임의 분할쌍에 대해 valid mapping 존재 보장.
8. 표현: mapping matrix(열=P1 서브셋, 타일=P2 서브셋)로 매핑을 시각화, turbo/LDPC 예제(6 VN, 5 CN)로 절차 예시.
9. 실구현: 3계층(입력 crossbar - 인터리버 뱅크 - 출력 crossbar) 시변 permutation으로 read/write 라우팅 구현(Fig.9). 뱅크 균형·최소 뱅크수 최적성.
10. 한계: 순수 이론(HW 합성·throughput·gatecount 미제시), BER/정정능력과 무관, 시변 채널로 재구성 시 매핑 전면 재계산.
