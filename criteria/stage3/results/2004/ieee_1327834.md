### B. 알고리즘 요약

1. rate-1/2 LDPC (regular (3,6) 및 irregular, `N`=2000~16000) 대상, AWGN/BPSK 채널.
2. 문제: message-passing 복호는 그래프 edge 수에 비례해 fixed-point 메모리가 선형 증가 → 수신단 메모리 병목.
3. 핵심 기법 1단: check node를 degree 균형 맞춰 Half-1/Half-2 두 반쪽으로 semirandom 분할, variable node의 edge를 두 반쪽에 균등 배분.
4. 각 반쪽에 대해 독립적으로 sum-product를 `l1` subiteration 수행, extrinsic을 다른 반쪽에 a priori로 전달 → `l2` superiteration 반복 (turbo식).
5. 새 subiteration 시작 시 edge LLR을 0으로 초기화 → 한 시점에 활성 edge가 절반뿐이라 메모리 절반.
6. 핵심 식: 반쪽 간 전달 extrinsic `(5)`와 최종 LLR `(6)`으로 hard decision.
7. 메모리 절감: 기존 `E`(edge수)+`n` 대비 제안식은 `E/2`+extrinsic/채널 LLR → 식 (7)~(9)로 정량화. address mapping 절반도 추가 절감.
8. 분석/최적화: Gaussian approximation 기반 density evolution으로 modified degree profile 유도, fmincon으로 rate별 최적 profile 설계. threshold 0.4dB로 기존 SP와 동일.
9. 결과: BER은 기존 SP와 거의 동일(75 iter서 ~10^-5), 메모리 절감 irregular ≥23%, regular ≥12.5%.
10. 한계: 수렴이 느려 총 iteration 동일시 성능 저하, HW 미설계, 계산량 절감 아님(메모리만).
