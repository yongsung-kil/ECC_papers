### B. 알고리즘 요약

1. 대상 부호는 `1/2` rate, `(3,6)` regular LDPC, codeword `n=2048`, `m≈1024` (2048 예제).
2. 풀려는 문제: LDPC 디코더의 VLSI 구현이 하드웨어 자원(라우팅/메모리)이 커서 실제 구현이 난제, 이를 아키텍처별로 정량 비교.
3. 채널 모델: AWGN 가정, LLR 소프트 입력을 log 도메인에서 처리, 메시지는 3-bit 워드로 양자화.
4. 복호 알고리즘: Pearl belief propagation = sum-product, flooding schedule (모든 C2V 후 모든 V2C).
5. 핵심 식: throughput `f_D = (n·r·f_c)/(N_it·L)` — 클록 `f_c`, iteration 수 `N_it`, iteration latency `L`(클록 사이클)에 의존.
6. Fully-parallel: Tanner graph 노드를 HW에 직접 매핑, `L=1` 최소 latency지만 라우팅 congestion이 주 병목.
7. Turbo Gallager Code 사례: node splitting으로 variable node 2048→3072, 64 cluster로 그룹핑해 라우팅 완화, 48 iteration.
8. Partially-parallel: `n_p<n`, `m_p<m` 프로세서만 구현, 메모리(엣지당 1셀)로 통신, 재사용도 `k=n/n_p`로 latency `L∝k` 증가.
9. 결과: fully-parallel 2.1M gates@0.18um, 42mm² core, 367.5mW@50MHz, 1Gbps; 250Mbps WLAN에서 partially-parallel은 면적 87~93%·전력 84~82% 절감.
10. 한계: 알고리즘 개선 없이 아키텍처 복잡도 분석에 국한, 정정능력/error-floor/iteration 개선 없음, NAND soft-read 무관.
