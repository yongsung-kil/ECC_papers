### B. 알고리즘 요약

1. 대상: 블록 순환(circulant shift) 구조 QC-LDPC(rate 0.4/0.5/0.9, length 1055~2070), (r,c)-regular. 예제는 (3,5) length 1055.
2. 문제: LDPC 디코더 VLSI의 병목은 복잡한 배선과 큰 메모리(R/Q 두 메시지 클래스 각각 저장).
3. 핵심 관찰(Block Message Independence): H를 `p×p` 블록으로 분할하면, R 메시지의 j번째 블록행은 Q의 j번째 블록열에만 의존, Q의 k번째 블록행은 R의 k번째 블록열에만 의존.
4. 기법: R/Q 메시지 계산 파이프라인을 동기화해 한 클래스가 생성되면 즉시 다른 클래스 계산유닛이 소비 → 별도 메시지 메모리 불필요.
5. 식(9)~(16): `R_{j,k}=ψ(A_j - B_{k,j})·G`, `Q_{k,j}=C_k - D_{j,k} - λ_k`, Gallager `ψ(x)=log(tanh(x/2))` LUT로 구현.
6. 구현: CBU(Check-to-Bit) p개 직렬유닛이 partial sum(ps1~ps3)을 multiplex, BCU가 ps4를 순차 계산. B FIFO(길이 `c(r-1)+1`), D FIFO(길이 r)로 중간값 저장.
7. p×p 인터리버는 `log2(p)`단 스위치 네트워크 + Switching Sequence 메모리로 shift 구현. 데이터패스 5bit.
8. 조기종료: dH^T=0 만족 시 반복 중단, 최대 20 iteration, iteration당 18 클록.
9. 결과: gate 2.4e5@0.18um, 150MHz, 440Mbps. 메모리 75% 절감, 메모리 액세스 66% 절감(vs [3,4]), [5,8] 대비 액세스 50% 감소.
10. 한계: 성능(BER) 개선 없음(메모리/전력 최적화만), 특정 구조화 코드 가정, HW 스케줄 복잡, 5bit 데이터패스로 정밀도 제한.
