### B. 알고리즘 요약

1. 대상은 PCGC(Parallel Concatenated Gallager Code) — 인터리버 없이 두 component LDPC 인코더를 병렬 연접(LDPC의 내재 인터리빙 성질로 인터리버 불필요), 블록 1920, 각 component 부호길이 1280·rate 0.5 → PCGC rate 0.333.
2. 문제: 기존 PCGC 복호는 component LDPC마다 modified BP(PCGC-BP)를 쓰는데 곱셈이 많아 HW 구현에 부적합.
3. 복호 구조: turbo 복호처럼 두 component decoder가 "a priori" 정보를 교환(super iteration), 각 super iteration 내에서 여러 local iteration 수행. 수렴 또는 max super iteration까지 반복, 미수렴 시 MCW(Mean Column Weight) 높은 쪽 출력 채택.
4. 1단 개선: modified BP를 log 도메인으로 옮긴 PCGC-Log-BP. 관건은 "a priori" 정보를 log 도메인 LLR로 계산·주입하는 것(초기화 단계에서 `L=ln(...)` 형태로 유도).
5. 핵심 식: CN 갱신 `L_mn = ln((1-T_mn)/(1+T_mn))`를 `-∏sign(z_mn')·min{|z_mn'|}`로 근사 (식 15) → 곱셈 제거.
6. 2단(최종): 위 min 근사가 부정확을 유발하므로 normalized factor `β`를 도입한 Normalized PCGC-Min-Sum. 곱셈 없이 add/compare만 사용.
7. 부수 이점: 초기화 LLR 값 비교만 하므로 SNR 추정 없이 복호 가능 → HW 친화.
8. 최적화: factor `β`가 성능을 좌우하며, 시뮬로 스윕해 β=0.9가 최적.
9. 결과: AWGN, max super iteration 15·local iteration 25. PCGC-BP 대비 PCGC-Min-Sum의 성능차 <0.2dB(적절한 β), BER 곡선상 거의 동일.
10. 한계: HW 미설계(적합성만 주장)·시뮬만, gate count/throughput 없음, 단일 LDPC가 아닌 PCGC 전용 구조.
