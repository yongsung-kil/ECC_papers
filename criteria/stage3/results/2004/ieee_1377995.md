### B. 알고리즘 요약

1. 대상: 짧은/중간 블록 irregular LDPC, rate 0.5, (504,252)·(1008,504), density-evolution 최적화 degree분포(max symbol degree 15), BPSK/AWGN/BP(max 80 iter).
2. 문제: DE로 최적화된 irregular 코드는 waterfall는 우수하나 error-floor가 조기에 나타나 고SNR 성능 저하.
3. 핵심: 표준 PEG는 edge-by-edge로 local girth를 greedy 최대화하는데, 동률 후보 CN이 다수일 때 index최소/랜덤 선택 → 성능 좌우 못함.
4. 수정: 후보집합 `Ω_sj^k`에서 새로 생기는 cycle(길이 `2(l+2)`)의 그래프 연결도가 최대인 CN 선택.
5. 연결척도 ACE: cycle 내 symbol node에 대해 `Σ(d_i − 2)` (extrinsic message degree), [14][15]에서 차용 - min ACE를 최대화하는 CN 채택.
6. 효과 원리: 연결도 높은 cycle은 나머지 그래프에서 extrinsic 정보를 많이 받아 harmful trapping set 형성을 억제.
7. 적용범위: irregular 전용 - regular 코드는 모든 신규 cycle의 ACE가 동일해 무효. 시뮬 코드에서 절반 이상 edge가 `|Ω|>1` 상황.
8. 복잡도: PEG 대비 미미한 증가(1008,504 기준 1.43→1.55초, 1GHz PIII).
9. 결과: 고SNR에서 modified PEG가 표준 PEG 대비 상당한 BER/MER 개선, 저SNR은 동등. (1008,504) 2.8dB에서 undetected error 36%→10%, 최소코드워드무게 12→15, trapping set (6,1)→오직 (15,1).
10. 한계: HW 미설계·순수 시뮬, 짧은 랜덤 irregular 코드 대상, QC/lifting 구조 없음.
