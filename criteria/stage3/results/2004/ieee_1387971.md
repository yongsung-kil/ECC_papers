### B. 알고리즘 요약

1. 시스템: regular/irregular LDPC, BPSK+AWGN, Gallager permutation-matrix 구성 (예: (3,4)-LDPC length 16, 그림 예제 length 9); H를 j개 sub-matrix(각 열 단일 1) 연접으로 정의.
2. 문제: 기존 two-phase(TPMP) MP 스케줄은 message 통신의 구조적 불규칙성 → 복잡한 interconnect(병렬화 제약) + 큰 메모리(serial 구현) 요구.
3. 채널 모델: 수신 채널정보 `β`와 각 constituent code의 extrinsic reliability `λ_i`로 표현, dual extrinsic principle 적용.
4. 핵심 기법: LDPC를 여러 super-code(single parity-check 집합)로 보고 trellis-decoding(SISO+interleaver)로 순차 디코딩 (TDMP); 한 super-code 갱신=sub-iteration, 전 super-code 1회전=1 iteration.
5. 식 의미: BCJR forward/backward state metric recursion(식1,2)과 soft-output LLR(식3)을 metric difference(Δα,Δβ,Λ) 형태로 변환해 연산량 절감.
6. 확장: Jacobi logarithm(MAX*)로 forward/backward metric 차이 계산, correction function `f(x)=ln(1+exp(-x))`를 lookup table로 근사.
7. 구현 디테일: 8-bit 2's complement(1 sign+4 int+3 frac), f(x)를 ACS(comparator+MUX+LUT) 유닛으로; programmable 6-컴포넌트 arch(SISO MPU, λ/γ/H/Z-Memory, router+reverse router).
8. 스케줄 이점: sub-iteration마다 갱신값을 즉시 다음 super-code에 사용 → two-phase보다 빠른 수렴, (C-1) super-code의 partial bit message만 저장(메모리↓).
9. 결과: 동일 SNR에서 수렴 iteration 20~50%↓, 소수 iteration·중~고 SNR에서 coding gain 최대 한 자릿수(order of magnitude)↑; 코드 특성(regular/irregular, girth 등) 무관하게 two-phase보다 우수.
10. 한계: HW arch 제시하나 gatecount/throughput 수치 부재(시뮬 검증), BCJR SISO 커널이라 min-sum과 상이, Gallager permutation 구성으로 QC-LDPC 아님.
