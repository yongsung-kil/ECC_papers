### B. 알고리즘 요약

1. 시스템: 자기기록 PR 채널(Lorentzian → MEEPR4 등화, 밀도 D)에서 BCJR + LDPC + 신규 MTR enforcer 3개 SISO 디코더를 turbo equalization으로 반복 결합.
2. 문제: 기존 BCJR/LDPC 디코더가 RLL/MTR 제약이 주는 추가 redundancy를 활용하지 못함 → 이를 쓰는 별도 디코더 모듈을 추가하려 함.
3. 대상 부호: [2]의 regular rate `0.90` LDPC + [5]의 rate `16/17` MTR (3;11) code, precoder, 섹터당 채널 입력 4721 bits.
4. 핵심 기법: MTR enforcer는 각 비트 주변 window(길이 `2W`) 내에서 One Constraint(연속 1 ≤3), Zero Constraint(연속 0 ≤11)를 만족하는 단어 집합에 대한 사후확률로 APP LLR을 계산하는 Bayesian 알고리즘.
5. 핵심 식: 사후확률 `P(bit|constraints)`를 window 내 허용 패턴 합으로 계산(식 5~9), extrinsic = 출력LLR − 입력LLR.
6. 확장: window 파라미터 `W`가 실행시간(∝ 상태 수)과 제약 강도를 동시에 좌우, 너무 작으면 제약 미적용.
7. 구현 트릭: 확률 도메인 원판 외에 log 도메인(log-sum-exp 연산 (17)) 및 max-log 근사(식 20) 변형 파생 — log-MAP/max-log-MAP LDPC와 유사.
8. 학습/최적화: 없음(닫힌형 확률 계산).
9. 결과: MTR enforcer 1회 반복(`W=8`)이 약 `0.2 dB` 이득, 반복↑이나 특정 window↑은 오히려 성능 저하; log/max-log 3변형 성능 동일 → 최소 연산량 max-log 권장.
10. 한계: 시뮬레이션만, HW 미설계, 자기기록 MTR/RLL 제약 및 PR채널 특화로 NAND flash와 무관.
