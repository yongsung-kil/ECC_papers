### B. 알고리즘 요약

1. 임의의 binary parity-check matrix `H`가 주어지면 완전병렬 log-domain LDPC 디코더를 빠르게 설계·구현하는 방법론을 제시한다.
2. 기존 girth/최소거리 기반 코드 분석은 실제 HW BER를 예측 못 하는 문제를 지적하고, HW 거동을 모사하는 C 시뮬레이터로 대체한다.
3. 채널은 binary field 기준 log-domain belief propagation(sum-product)으로 복호한다.
4. 핵심 기법: bit node/check node를 "virtual node"로 만들고 interconnect map으로 연결해 임의 `H`에 대한 가상 디코더를 구성한다.
5. log-domain으로 곱셈을 제거하나 대신 `Φ`(및 `Φ^-1`) 함수가 필요하며, 곱셈은 부호(sign) XOR로만 남는다.
6. 세 파라미터로 노드를 일반화: message resolution `N`, I/O 연결수 `i/j`, `Φ` 근사 비트해상도 `L`.
7. `Φ(x)`는 x가 커지면 급격히 0으로 수렴하므로 큰 x에서 여분 비트는 낭비 -> `2^L-1` 초과 시 0으로 강제해 면적 절감.
8. VHDL generic으로 bit/check 노드를 파라미터화하고, 시뮬 소프트웨어가 그대로 VHDL 글루로직·인스턴스화까지 자동생성한다.
9. 결과: 282-bit 코드, N=L=4bit 최적, 0.35um CMOS ASIC, core+IO 25mm^2, 100MHz, 총 245k 게이트(제어·wrapper 포함).
10. 한계: BER-면적 트레이드오프만 다루고 정정능력 개선·iteration 감소 없음, 완전병렬이라 코드길이 확장 시 라우팅 폭발.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| log-domain 디코딩 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime은 min-sum, 논문은 full Φ Sum-Product -> 알고리즘 계열 다름 |
| Φ 근사 / message 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 양자화 개념은 유사하나 Prime은 magnitude 테이블, Φ근사는 대응 낮음 |
| 완전병렬 노드 interconnect / VHDL 자동생성 | 대응 없음 | Prime은 z=32 row-병렬 + HCU/GT 고정 구조, 완전병렬 자동생성과 무관 |

적용 가치: **낮음** — 표준 log-domain SPA + Φ근사에 임의 H행렬 완전병렬 자동생성으로, Prime의 min-sum·QC-LDPC·row병렬 구조에 떼어 쓸 요소가 없다.
