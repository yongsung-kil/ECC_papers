### B. 알고리즘 요약

1. 시스템: AWGN + BPSK, regular/PG-LDPC·Gallager 부호(예: (273,191) col-weight 3/17, (1057,813) PG-LDPC).
2. 문제: 표준 WBF는 flip 판정에 check 제약 메시지 `E_n`(식(1)의 최소 신뢰도 합)만 사용 → bit 자체 신뢰도(`|y_n|`)를 무시.
3. 표준 WBF: syndrome `s`로 각 bit의 `E_n = Σ (2s_m-1)|y_n|_min` 계산 후 최대 `E_n` bit를 flip, parity 만족까지 반복.
4. 핵심 아이디어: `E_n`(check-based 메시지)과 `|y_n|`(bit-based intrinsic 메시지)를 함께 고려 → 식(3)에서 `E_n - |y_n|` 형태로 판정.
5. 의미: 첫 항은 check 정보 기반 flip 압력, 둘째 항 `|y_n|`은 초기값 유지 압력 → 두 정보를 결합해 flip 여부 결정.
6. 확장: 가중치 `α`(real, 0<α)를 bit 메시지에 곱해 식(4) `E_n - α|y_n|`; α=0이면 표준 WBF로 환원.
7. α 최적화: SNR·column-weight마다 최적 α 상이 (표1: column weight↑ → 최적 α↑), 시뮬로 결정.
8. 실용화: 한 SNR(BER=10^-5)에서 정한 α를 고정해도 성능 저하 미미 → code-dependent, SNR-independent로 사용 가능.
9. 결과: bit당 addition 1회만 추가(곱셈은 사전처리). 긴 부호(1057)에서 표준 WBF 대비 약 1/2 iteration; BER은 BWBF 수준 개선.
10. 한계: WBF 계열이라 magnitude(soft) 필요, BP/min-sum 대비 성능 열위, HW 미설계, α tuning 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| WBF flip 판정 (decoding loop) | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum message-passing 채택 — WBF flip 스케줄과 알고리즘 계열이 달라 직접 이식 아님 |
| bit intrinsic 메시지 `α|y_n|` 결합 | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | VN에서 채널 LLR 반영은 이미 존재, WBF식 가중치는 대응 개념 없음 |
| 채널 magnitude / 신뢰도 | `channel.cpp` `Set_LLR_Th()` | soft magnitude 사용 필요, WBF는 우리 min-sum 대비 성능 낮음 |

적용 가치: **낮음** — min-sum(BP급) 디코더를 이미 보유한 Prime ECC에 WBF 계열 저복잡도 알고리즘은 성능·신규성 모두 부족.
