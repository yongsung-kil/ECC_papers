> 총평: 임의구성 regular LDPC의 저복잡도 인코더(PABR+LU)와 Costas array PCM을 FPGA로 구현한 인코딩 논문 — Prime ECC의 QC/dual-diag 구조와 안 맞아 이식성 낮음.

### B. 알고리즘 요약

1. 대상: `(3,6)`-regular binary LDPC, rate `1/2`, 부호길이 `n=900, 1800`, sum-product(log-domain) 복호, iteration 10.
2. 문제: irregular보다 인코딩 복잡도가 낮은 regular LDPC라도 straightforward 인코딩은 복잡도가 높고 LDPC 인코더 VLSI 구현 고려가 부족함.
3. 부호구성: `H=[A|B]`, systematic 형태로 check bit `c=A^{-1}Bs`를 계산하되 A의 non-singularity가 필요.
4. 핵심 기법 1(PABR): Costas array PCM에서 A가 non-singular가 되도록 column/row를 pivoting 재배열, 실패 시 bit-reverse(대각원소 0↔1 반전)로 non-singular화 — 동시에 4-cycle 제거·high-girth 유지.
5. 핵심 식: `LUc=z (z=Bs)`를 `Ly=z`(forward subst.)와 `Uc=y`(backward subst.)로 분해해 매 s마다 재분해 없이 parity bit 계산.
6. 핵심 기법 2(Costas array): 소수 p의 원시원소 a로 Welch 방식 n=p−1 Costas array를 만들어 row-cyclic shift 서브행렬로 PCM 구성 → high-girth·최소거리 확보로 BER 개선.
7. 구현 디테일: Crout 알고리즘 기반 GF(2) LU분해를 XOR/AND 비트연산으로 수행(코드 리스팅 제시).
8. 결과: AWGN/BPSK에서 (900,3,6)·(1800,3,6) BER이 참조문헌 이상, regular임에도 irregular 못지않은 성능.
9. HW: 3-stage pipeline(z→y→c) 인코더 + min-sum 디코더 FPGA 구현, `DataRate=clock×Par/Wr=30.72MHz×3/6=15Mbps`, 약 100k gate(turbo 디코더급).
10. 한계: 4세대 이동통신 플랫폼 대상, throughput 15Mbps로 낮고 min-sum 디코더는 성능 열화 언급, 부호길이 소규모.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LU분해/PABR 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 인코딩 담당 지점이나 Prime ECC는 dual-diagonal 고정 — LU/A^{-1}B 방식과 상이 |
| Costas array high-girth PCM 구성 | `ecc_top.cpp` `Load_PCM()` | 부호구조 로드 — Prime ECC는 고정 QC-LDPC라 random-like Costas 구성 부적합 |
| non-singular A / column 재배열 | `encoder.cpp` `Column_Reordering_Dual_Update()` | column 재배치 개념은 유사하나 목적(dual-update vs pivoting) 다름 |
| min-sum 디코더 FPGA | `decoder.cpp` `LDPC_Decoder()` | min-sum은 이미 보유 |

적용 가치: **낮음** — PABR/LU 인코딩과 Costas array 구성은 임의구성 regular LDPC 전용으로 Prime ECC의 고정 QC-LDPC·dual-diagonal 인코더 구조와 정합되지 않고, NAND가 아닌 이동통신 타깃이다.
