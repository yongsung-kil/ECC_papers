### B. 알고리즘 요약

1. 대상: 큰 블록길이(예 rate-1/3, N=48K)의 Repeat-Accumulate(RA)와 LDPC를 regular/irregular 모두 통합 처리하는 고성능 디코더 설계.
2. 문제: Tanner 그래프의 랜덤 permuter(RA는 명시적, LDPC는 암묵적)가 병렬 디코더의 라우팅/메모리 병목 → 코드 랜덤성과 디코더 복잡도를 분리(decouple)해야 함.
3. 모델: sum-product LLR message passing, check node 갱신은 `ψ(z)=(sgn(z), -log tanh|z|/2)` 매핑 사용.
4. 핵심 기법 1단: architecture-aware(AA) 그래프 — 변수/체크 노드를 크기 `S` 클러스터로 묶고 클러스터 쌍을 순열 `π`로 연결(Fig.3), H를 `S×S` 순열행렬 블록으로 표현(Iπ, Fig.4).
5. 부호구성: AA 구조는 Ramanujan/quasi-cyclic/cyclotomic coset 등 대수적 explicit 구성으로 얻으며 큰 minimum distance와 iterative 성능 양립이 목표.
6. 핵심 기법 2단: staggered(super-code) 디코딩 — H의 블록행을 1차원 super-code로 보고 직렬 디코딩, 이웃 super-code 간 message 전달(sub-iteration).
7. 스케줄 이점: 단일 message 타입(extrinsic) + 전체합 저장 후 자기 extrinsic 감산으로 저장량을 edge 수에 비례(기존 2배 대비)로 절감.
8. permuter 구현: 블록 순열행렬 분해로 이웃을 `log S` 공간에 계산, Benes 네트워크(`S(2 log S -1)` 스테이지 2x2 스위치)로 실현.
9. 부수 트릭: 비선형 check 갱신을 max-quartet `Q(x,y)=max(x,y)+max(5/8-|x-y|/4,0)-...` 근사로 단순화.
10. 결과/한계: S=256에서 100MHz에 1Gbps 초과 throughput, 병렬/직렬 대비 메모리·permuter 대폭 절감(Fig.8-10); 그러나 복잡도 추정·throughput뿐이고 합성/BER 실측 없음, 연판정 비트폭 미명시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| sum-product ψ / max-quartet Q 근사 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | CN 갱신 대응이나 우리는 min-sum 채택이라 tanh/Q근사는 불필요 |
| staggered(super-code) 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update(parity 교대) 스케줄을 이미 보유, 개념 중복 |
| Benes permuter 네트워크 / 라우팅 | 대응 없음 | 우리는 QC-LDPC circulant shift 고정, 범용 Benes 라우팅 미사용 |
| AA 클러스터/순열행렬 H 구성 | `ecc_top.cpp` `Load_PCM()` | QC-LDPC와 유사(순환) 개념이나 RA 대상이라 이식 부담 큼 |

적용 가치: **낮음** — RA(turbo-like) 통합과 랜덤 permuter/Benes 네트워크가 핵심이라 min-sum 고정·circulant-shift QC-LDPC인 Prime ECC에 떼어낼 신규 요소가 없다.
