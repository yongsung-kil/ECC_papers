> 총평: girth 극대화로 error-floor를 낮추는 well-structured QC-LDPC 구성법 — 개념은 이식가능하나 column weight 3 고정·낮은 waterfall 이득으로 Prime ECC 고rate H-matrix에 직접 대체는 재검증 부담이 큼.

### B. 알고리즘 요약

1. 코드 스펙: column weight `j=3` regular (n,3,k) QC-LDPC, girth 8 및 10, flexible rate/length (통신·데이터 저장용).
2. 문제: 짧은 cycle이 sum-product 수렴을 방해 → large-girth 부호 필요; j=3(j>2)이라야 최소거리가 n에 선형 증가(j=2는 log).
3. 모델: H를 v개 점의 structure graph로 표현, 각 column을 3점으로 이루어진 column triangle로 매핑 → cycle을 triangle/loop로 시각 식별.
4. 핵심 기법: edge의 slope `s=j-i`와 admissible slope pair(ASP) `s'=-sgn(s)·mod(p-|s|)` 정의, 3개 section S0/S1/S2에 ASP 집합 A_i 배치.
5. 핵심 식: 6-cycle-free 조건 `mod(s0+s1+s2, p)=0`인 triple이 p개 column triangle을 생성(Fact 1,2) → block length `n=Ns·p`, rate `r=(Ns-3)/Ns`.
6. 확장: Construction 1(Type-I, g=8) → Type-II(Form-II 8-cycle 일부 제거, rate 희생) → Construction 2 수정으로 g=10 (모든 8-cycle 제거).
7. 구현 디테일: H가 정수집합(parity check 번호+slope)만으로 완전 표현되는 well-structured/QC 구조 → 저장·구현 용이.
8. 최적화: Ns를 admissible slope set 탐색으로 최대화해 고rate 달성(예 v=231,n=1155,r=0.8).
9. 결과: AWGN, sum-product 복호에서 random 대비 고SNR에서 Type-II가 BER=10^-5에 0.15dB 우수, girth 8/10 부호가 error floor 더 낮음(waterfall 이득은 0.02~0.15dB로 작음).
10. 한계: HW 미설계, j=3 column weight 고정, waterfall 개선폭 작고 주 이득은 error-floor.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth 최적 H-matrix / QC 부호구조 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` + H-matrix | H-matrix 교체 지점이나 Prime ECC는 고정 QC-LDPC라 재검증 부담 높음 |
| slope/ASP 기반 base matrix 생성 | 대응 없음 | Prime ECC의 base+circulant shift 생성 로직과 직접 대응 함수 없음 |
| error-floor 개선(girth 확대) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` (복호 성능 관점) | 부호설계가 error-floor를 낮추므로 복호 성능에 간접 기여 |

적용 가치: **중간** — girth 기반 error-floor 저감이라는 부호설계 방향은 유용하나, column weight 3·낮은 rate 예시가 Prime ECC의 고rate(VN degree 17) QC-LDPC와 정합되지 않아 직접 이식보다 설계 참고 수준.
