### B. 알고리즘 요약

1. 목표: irregular LDPC의 부호구성과 인코더/디코더 VLSI 구현을 **공동 설계**하여 성능과 구현효율을 동시 확보.
2. 부호 제약: (1) 근사 상삼각(Richardson-Urbanke형, `g`를 최소화) + (2) `p×p` 블록구조(zero 또는 identity의 우향 순환시프트) = QC-LDPC 형태.
3. 문제의식: 성능 최적 부호를 먼저 만든 뒤 HW를 붙이는 전통방식은 LDPC 구현에 부적합, 특히 인코더 복잡도·체계적 구성법 부재.
4. 핵심 아이디어 - cycle degree: 한 사이클 위 모든 variable node의 degree 합을 `cycle degree`로 정의. 저차수 노드를 많이 지나는 small cycle이 더 해로우므로 이를 회피.
5. HBP(Heuristic Block Padding): 블록열마다 `a_j`개 NULL 블록을 순환시프트 identity로 채우되, 최소 cycle degree `f(H)<d` 이거나 4-cycle 발생 시 reject; 수렴 안되면 `d`를 1씩 완화.
6. 인코딩: `H=[A B T; C D E]` 분해, 상삼각 `T`에서 `T=T^-1` 성질을 이용해 back-substitution 제거; `Φ^-1`(크기 `g^2`) 곱 외에는 전부 sparse matrix-vector 곱.
7. 인코더 HW: 6-stage 파이프라인, 6개 sparse mat-vec 블록을 inter-vector-parallel/intra-vector-serial(t벡터 병렬, 벡터당 1bit씩 `p` 사이클)로 처리, address generator(binary counter)로 순환시프트 구현.
8. 디코더 HW: `m` CNU + `n` VNU, TDM 방식, 메모리 어드레싱+하드와이어 인터커넥트로 message passing(순환시프트는 카운터로 주소생성).
9. 결과: rate-1/2, 8K 부호(m=64,n=128,p=64,γ=3, g=192, dmin=8) BPSK/AWGN 시뮬에서 기존 결과와 동등 이상 BER/FER; 4bit 양자화·20 iter 가정 시 인코더 38K/디코더 205K gate.
10. 한계: 2쪽 개념 논문으로 실제 디코딩 알고리즘(SP/min-sum) 미명시, gatecount는 추정식, 실HW 미구현, NAND 무관.
