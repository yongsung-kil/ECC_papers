### B. 알고리즘 요약

1. 시스템: AWGN/BPSK, `(n,k)` binary linear block code (rate `~1/2`), 대상 예제 eBCH `[128,64,22]`·`[192,96,28]`·`[256,187,20]`·`[512,475,10]`. LLR 기반 soft-decision(reliability = LLR).
2. 문제: MLD/near-MLD는 부호길이에 지수적 시간복잡도 → 길이 큰 rate-1/2 부호 복호 불가. 메모리 자원은 남는데 활용 안 됨.
3. 기반: order-`i` reprocessing(OSD [6]) — most reliable basis(MRB)를 greedy Gaussian elimination으로 구하고, MRB 상 Hamming weight ≤`i` 코셋원소를 생성해 최소 ellipsoidal weight(EW=LLR 가중 metric)를 선택.
4. 핵심 기법: BMT(box and match) — MRB에 control band(CB)를 더한 집합 `S` 상에서 low-weight 코셋원소를 찾되, weight-`i` 코셋원소를 두 개의 weight-≤`⌈i/2⌉` 패턴 합으로 분해하고 "박스"(해시 버킷)로 짝을 matching → 후보 수를 `2^p`로 나눔.
5. 핵심 보조정리(Lemma 3.2): 두 MRB 패턴 `u,v`의 CB-complement가 `(u⊕v)`의 CB-complement와 XOR 관계 → 박스 인덱스로 짝을 O(1) 검색.
6. 복잡도: 시간은 OSD 대비 코드길이 선형 배수만큼 감소(정렬 제거), 메모리는 `O(2^p)`(상수~지수, 본 알고리즘은 상수 배). 파라미터 `(i,s)`로 시간/메모리/WER 조절, 최적 `s`는 시간복잡도 최소화 값.
7. 구현 디테일: conditions for optimality(C.O.) — 필요/충분 조건으로 평균복잡도 대폭 절감(고SNR에서 사실상 diagonalization 비용); 32bit 정수로 MRB 패턴/complement 표현, 박스는 초기화 1회.
8. 확장: iterative information set reduction(IISR) 결합 → 반복적으로 신뢰위치 error-free 가정, near-MLD와 practical-MLD 사이 성능 정밀조정.
9. 결과: `[128,64,22]` eBCH practical-MLD 달성, worst-case 후보수 order-3 대비 10배↓, 평균 100배↓(2dB); rate-1/2 length 192를 최초로 near-MLD; 500 word error 시뮬 검증.
10. 한계: 반복복호(LDPC) 아님, HW 미설계, 박스 메모리·후보수가 부호길이에 취약; deterministic C.O.는 probabilistic보다 느림.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| OSD + box/match near-MLD 복호 | 대응 없음 (Prime ECC는 min-sum 반복복호) | 알고리즘 계열이 다름(OSD-MLD vs iterative message passing) |
| soft reliability(LLR) 처리 | channel.cpp Set_LLR_Th() (개념적) | 둘 다 LLR 사용하나 BMA는 full-precision reliability로 4bit+ soft 필요, Prime ECC는 2~3bit |
| matching용 박스(해시) 메모리 | 대응 없음 | Prime ECC는 GT MUX/LLR 테이블 기반, 해시 매칭 구조 없음 |
| 코셋원소/EW metric 최소화 | 대응 없음 | 반복복호의 min-sum extrinsic 계산과 무관 |
```

적용 가치: **낮음** — OSD 기반 near-MLD 소프트결정 복호는 Prime ECC의 min-sum 반복복호와 알고리즘 계열이 다르고, 짧은 블록부호(eBCH)·full-precision LLR·지수적 후보/박스 메모리에 의존하며 HW 설계가 없어 NAND QC-LDPC ECC에 떼어 쓸 수 없다.
