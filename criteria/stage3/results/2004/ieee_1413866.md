> 총평: 순환행렬 원시다항식 기반 reversible LDPC 구성법으로 부호구성이 error-floor에 영향을 주나, rate 0.5·HW 미설계·비고rate라 NAND용 Prime ECC 이식가치 낮음.

### B. 알고리즘 요약 (10줄 내외)

1. 대상은 GF(2) 원시다항식으로 만든 순환행렬 기반 (3,6)-regular binary LDPC, block length `(288,144)`~`(4061,2013)`, rate≈1/2, AWGN+BPSK+sum-product 복호.
2. 풀려는 문제: reversible LDPC(반복 인코딩 가능)의 저BER(<10^-5) 성능과 block size 자유도 부족을 개선.
3. 핵심 가정: 패리티행렬을 `H=[Hp|Hu]`로 분할, `Hu` invertible이면 `Ax=B` 형태로 Jacobi 반복 인코딩 가능.
4. 반복 인코딩: `x_{m+1}=(A⊕I)x_m + B`, `(A⊕I)^m=0` 즉 `A^m=I`이면 m회 반복으로 수렴 → 디코더 회로(노드+라우팅) 재사용해 O(n) 인코딩.
5. `Hu`는 첫 행을 다항식 `p(x)`로 정한 순환행렬로 구성, m=4 반복·4-loop free 조건 `(P⊕I)^4=0` 만족.
6. 핵심 확장: `Hu`의 1 위치를 GF(2) 원시다항식 지수로 지정하면 4-loop이 자동 제거됨(참고 [5]) → 다양한 block size 확보.
7. 여러 원시다항식(degree 4/5/6 등)을 Table I에서 골라 `(P⊕I)^4=0` 만족하는 (3,3) 순환행렬만 채택, Hp는 4-loop 없이 랜덤 생성.
8. 결과: `1+x^3+x^5`(등) 다항식이 `1+x+x^3`보다 저BER 영역에서 우수, block size 4061에서 coding gain이 두드러짐.
9. 결과: 일부 행렬은 SNR>1.8dB에서 error floor 관측, 원인은 6-loop 수·최소거리로 추정.
10. 한계: HW 미설계, 시뮬만, rate 0.5 저rate, error-floor 정량 분석 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 원시다항식 순환행렬 H 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 고rate QC-LDPC, 저rate reversible 구성법 이식 부담 큼 |
| reversible 반복(Jacobi) 인코딩·디코더 회로 재사용 | `encoder.cpp` `LDPC_encoder_4KB()` | Prime ECC는 dual-diagonal 인코딩 별도, Jacobi 재사용 방식과 상이 |
| sum-product 복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum 근사 사용, full sum-product는 대응 없음 |
| error-floor / 6-loop·최소거리 | 대응 없음 | 논문이 정량 분석 안 함 |

적용 가치: **낮음** — reversible 인코딩(디코더 재사용)은 Prime ECC의 dual-diagonal과 다르고, 저rate·시뮬만·HW 미설계라 NAND 고rate QC-LDPC 이식 이득이 없다.
