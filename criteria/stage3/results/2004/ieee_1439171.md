> 총평: edge-coloring으로 LDPC 디코더의 통신 충돌을 회피하는 프로그래머블 semi-parallel 아키텍처 프레임워크로, 알고리즘/정정능력 개선 없이 임의 그래프 유연성을 노려 규칙적 QC 구조를 이미 쓰는 Prime ECC엔 이식 가치가 낮다.

### B. 알고리즘 요약

1. 대상: 일반 (j,k) LDPC, sum-product 복호, AWGN 채널; 부호 rate/length 특정값 없음(프레임워크 논문).
2. 풀려는 문제: LDPC의 불규칙 그래프 통신은 fully-parallel(라우팅 폭발)·serial(저throughput) 모두 한계, semi-parallel은 메모리 접근 충돌이 병목.
3. 관찰: 병렬성은 계산이 아니라 many-to-many 통신에 의해 제약, 메시지가 소스/목적지를 공유해 메모리 충돌 다발.
4. 핵심 기법: bipartite graph를 edge-coloring — 같은 색 엣지는 정점을 공유하지 않음(König 정리, k=최대 정점 degree 색이면 충돌 없음).
5. 핵심 원리: 엣지를 k 색 그룹으로 분할 → 각 노드가 그룹당 최대 1개 엣지 → 동일 메모리블록 동시 접근 불가(충돌-free) 보장.
6. 구현 옵션 2가지: (a) 색별 multi-port 메모리블록, (b) 노드마다 k개 MUX(색 그룹별 1개)로 임의 노드 메시지 읽기.
7. 매핑: code size를 물리노드 수로 나눠 그룹화, 그룹번호=읽기 타이밍, 색 그룹 내 엣지번호=MUX 셀렉터/주소(예: 32노드/8물리노드 → 4그룹).
8. Throughput: iteration당 2 cycle(check 1+bit 1), 병렬화 32·10ns·20iter → 80Mbps, 병렬화 512 → 1.28Gbps(HW 비용 증가).
9. HW 비용: 총 MUX 수 `Sk+Tk`(S bit/T check 물리노드), 구조화 부호로 MUX 입력 제한 시 비용↓·유연성↓ 트레이드오프.
10. 한계: behavioral VHDL 기능검증뿐(합성/gatecount 없음), 통신 HW 비용 큼, 정정능력/iteration 개선 없음, 저장장치는 예시 언급뿐 NAND 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| edge-coloring 그룹핑 / MUX 라우팅 | [GT.cpp](../Prime_ECC_3.1_Claude) `Make_GT_HW()` | Graph Thinning(MUX 기반 edge 선택)과 개념 유사하나 Prime ECC는 고정 QC 구조 전용, 임의그래프 프로그래머블성 불필요 |
| semi-parallel 메모리 충돌 회피 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z=32 규칙적 QC로 이미 충돌-free, 신규 이점 없음 |
| sum-product 복호 스텝 | `decoder.cpp` `LDPC_Decoder()` | 교과서 BP, Prime ECC는 min-sum — 직접 대응 없음 |

적용 가치: **낮음** — 임의 LDPC 그래프의 프로그래머블 통신 충돌 회피가 목표라, 규칙적 QC-LDPC(z=32)와 Graph Thinning을 이미 보유한 Prime ECC엔 알고리즘·정정능력·이식 가치 모두 낮다.
