### B. 알고리즘 요약

1. 대상: 대수적으로 구조화된 regular QC-LDPC block 부호와 그 convolutional 대응부호, BPSK-AWGN, 반복 BP 복호(최대 50~100 iteration).
2. 문제: 랜덤 LDPC는 큰 H 저장·접근·인코딩·거리분석이 어렵다 → 구조(대수성)를 부여해 이를 극복.
3. 구성 핵심: 소수/합성수 `m`의 곱셈군에서 order `j`,`k`를 갖는 원소 `a`,`b`로 `j×k` circulant(shift된 단위행렬) 배열을 배치해 `(j,k)`-regular QC-LDPC의 PCM 생성 (Example 1의 `[155,64,20]` 부호가 대표).
4. QC 성질: 각 circulant 블록 내 1위치 순환이동이 또 다른 codeword → shift-register 인코딩·고속 VLSI에 유리.
5. Girth/거리 하한: 소수 `m`이면 girth가 특정 tree-bound 이하로 보장되고, 곱셈군 구조로 짧은 cycle 존재를 효율적으로 검사 가능; MacKay-Davey 상한으로 min-distance ≤ 특정 값.
6. 확장: 마지막 circulant들을 all-zero로 치환해 irregular QC-LDPC 생성 → threshold(waterfall) 개선(예: regular 2.65dB → irregular 0.87dB)하나 거리 저하로 error-floor 악화.
7. Convolutional 파생: circulant 다항식 표현을 delay 연산자로 재해석해 QC block 부호를 "unwrap"한 LDPC convolutional 부호 유도, tail-biting 관계(Theorem 3.1) 규명.
8. 인코딩: parity-check 구조 기반 shift-register 인코딩, erasure-decoding 기반 선형복잡도 인코딩(Lemma 3.1), transform theory로 누락 generator row 보완.
9. 결과: 대수적 QC 부호가 짧은~중간 블록길이(~10^5)에서 랜덤 부호와 대등하거나 우수; convolutional 대응부호가 waterfall에서 QC block 대비 유의한 이득(circulant 크기 무한 극한).
10. 한계: min-distance가 `3(k-1)` 상한이라 error-floor 존재, HW/디코더 미구현(순수 구성·이론), 긴 블록에서 랜덤 부호에 열세.
