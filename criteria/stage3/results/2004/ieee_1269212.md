### B. 알고리즘 요약

1. 대상: (n,k) random(비구조) LDPC, 예 rate `R=0.9`(K=CN degree 30), codeword length `n=4000~10000`(구현 block 9300), storage(광/자기 디스크) 응용, throughput 300Mbps.
2. 풀려는 문제: full-parallel "one arc = one wire" 구현은 random 그래프의 배선 혼잡(routing congestion)으로 대형 부호에서 비현실적.
3. 핵심 가정: storage는 head repositioning(수십 ms)이 커서 수십~수백 codeword의 디코딩 지연이 허용됨 → serial(sequential) 디코딩 머신 다수 사용 가능.
4. 핵심 기법 1단(UMP): Fossorier의 min-sum 계열 UMP 채택 — CN에서 `tanh` 등 복잡 함수 불필요, 부호는 입력 sign들의 XOR, magnitude는 자기 제외 K-1개 최솟값.
5. 핵심 관찰: CN의 K개 "j 제외 최솟값"은 3개 값으로 압축 가능 — 전체 min, one-but-min, min이 나온 index `j`.
6. 핵심 기법 2단(check-node-centric 재정의): 좌↔우 half-iteration 교대를 없애고 CN당 old/new 3값 running state로 통합 → 저장을 전부 CN으로 이동, 메모리 ~10배(150k→43k flip-flop, 약 3.5배 순감) 절감.
7. 구현 디테일: min 4bit, one-but-min 4bit, min 주소 5bit, sign 30bit. bit node가 iteration당 1회 처리되며 연결된 3 CN의 running min/one-but-min 갱신.
8. 아키텍처(time-folded): prefetcher + computation kernel + memory bank(4 single-port RAM) + µRom + state machine. connection table 정렬·prefetch·delayed writeback으로 read/write 충돌 해소, throughput = ClockFreq / NumIterations.
9. 결과: 0.13µm CMOS, 147MHz@1.12mm²(area) / 211MHz@1.14mm²(delay). 300Mbps 위해 57 tiles(~36mm²), full-parallel(>94mm²) 대비 소면적. µRom 하나로 전 tile 제어. block>4k에서 [2] 대비 배선 우위.
10. 한계: 정정능력 개선 없음(UMP은 full-BP 대비 수십분의 dB 손실). serial 처리로 latency 증가(iteration당 bit node 수 사이클). random LDPC 전제, storage 지연 요건 특화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| UMP(min-sum) CN 연산 (min/one-but-min/sign XOR) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 우리 min-sum과 사실상 동일(min1/min2 + sign XOR) — 중복 |
| CN당 3값 압축 저장(min/one-but-min/index) | decoder.cpp C2V_Cal() 주변 CN 데이터 저장 | 메모리 절감 아이디어, 우리 CN 결과 저장 방식과 비교 가능 |
| time-folded serial + prefetcher/memory bank | 디코더 HW 모델(z=32, HCU, GT) | 우리는 z=32 row-병렬 QC 구조 — serial random 아키텍처와 정합 낮음 |
| 반복 루프/스케줄 | decoder.cpp LDPC_Decoder() | 전체 이터레이션 루프 대응 |
```

적용 가치: **중간** — CN당 3값 압축 저장을 통한 메모리 절감 개념은 참고가치가 있으나, UMP 자체는 우리 min-sum과 중복이고 random-LDPC serial/time-folded 아키텍처는 QC-LDPC z=32 병렬 구조와 맞지 않아 직접 이식은 제한적.
