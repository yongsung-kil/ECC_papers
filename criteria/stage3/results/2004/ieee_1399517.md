### B. 알고리즘 요약

1. 목표: 하나의 디코더 HW로 여러 code rate를 지원하되 blocklength를 고정(CBMR)해 puncturing/shortening의 성능저하를 회피.
2. 문제: 기존 방식(puncturing로 고rate화, shortening으로 저rate화)은 blocklength를 줄여 성능이 크게 열화.
3. 핵심 기법: 저rate "mother" 부호의 parity-check 행렬 행(check node)들을 선형 결합(row-combining)해 행 수를 줄임 → 동일 blocklength로 고rate "effective" 부호 생성.
4. 결합 조건: 결합할 행들이 같은 열에 1을 공유하지 않도록(공통 neighbor 없게) mother를 설계, 결합 시 check degree 집중분포 유지 가능.
5. 핵심 관계: rate-3/4 행 i = rate-1/2의 행 i + 행 i+M/2; rate-5/6은 i, i+M/3, i+2M/3 결합 (M=mother check 수).
6. 그래프 컨디셔닝: ACE 알고리즘([4])과 βc/βp 제약([5])을 mother와 모든 effective 부호에 동시 적용(simultaneous graph conditioning)해 stopping set을 키워 error floor 억제.
7. HW 친화 구조: [8]의 p×p 순환시프트 sub-matrix 블록구조로 병렬처리, back-substitution 인코딩용 H2 하삼각(staircase) 구조를 모든 rate에서 유지.
8. Information nulling(shortening): 열 삭제로 저rate·저blocklength 부호 파생, bit padding에 활용(마지막 frame은 iteration 절감).
9. 결과: blocklength 1944, sub-matrix 27, 최대 200 iter, JPL HW 디코더 시뮬로 모든 rate에서 낮은 error floor; rate-5/7 shortening 시 절반 iteration으로 동등 성능.
10. 한계: HW 미구현(부호구성법), rate-2/3은 check degree 불균일로 상대적 성능저하; NAND read/soft-decision 무관.
