### B. 알고리즘 요약

1. 시스템: near-Shannon turbo-like 부호(PCCC/SCCC/RA/IRA/ARA 및 LDPC)의 고속 iterative 복호. 예제 rate `1/2`(ARA4 repetition 4), FPGA 구현.
2. 문제: 병렬 turbo/LDPC 복호기의 최대 병목은 memory access(interleaver 충돌) → 특수구조 interleaver로 해결 필요.
3. 병렬 turbo 복호: 수신 데이터를 겹치지 않는 window로 분할, 각 window를 BCJR(SISO)로 병렬 복호, 인접 window의 alpha/beta 초기값을 이전 iteration에서 공급.
4. 핵심개념: 그래프를 M개 동일 sub-graph(page)로 분할 후 한 page에 사영한 `projected graph` 도입 — 각 edge에 message 벡터 대응.
5. 핵심 아이디어: projected graph가 부호구조뿐 아니라 interleaver 구조까지 표현. edge interleaver = 벡터 내부 원소 순열(page 간 순열).
6. 확장: 이 projected-graph 클래스에 turbo/SCCC/RA/IRA/ARA/LDPC 모두 포함됨을 보임. LDPC는 SPC 성분그래프+repetition 성분그래프 두 개로 표현(local/border message 없음).
7. HW: window processor 어레이 + programmable permuter + 단일 scheduling controller. extrinsic 벡터를 별도 RAM에 저장하고 주소 순열로 interleaving 처리.
8. 설계법: 기존부호 그래프 분할(1안) 또는 성분그래프 설계 후 결합(2안), 성능은 density evolution으로 성분그래프 단위 분석.
9. 결과: Xilinx FPGA에 ARA 부호 복호기 구현, `800 Mbps/iteration` 복호속도 달성. reverse interleaver로 latency 감소.
10. 한계: turbo/BCJR 중심 프레임워크, gate count 미기재, 순수 NAND QC-LDPC와 부호클래스·복호(BCJR) 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| edge interleaver / extrinsic RAM 주소순열 | 대응 없음 | turbo interleaver 메모리구조로 Prime ECC(QC circulant shift 라우팅)와 개념 상이 |
| window processor 병렬 복호 아키텍처 | `decoder.cpp` `LDPC_Decoder()` (z=32 병렬) | 병렬화 개념은 유사하나 BCJR/turbo 기반, Prime ECC min-sum row 병렬과 정합 어려움 |
| projected graph 부호설계 | `ecc_top.cpp` `Load_PCM()` | turbo-like 통합설계로 고정 QC-LDPC와 무관 |
| BCJR/SISO message passing | 대응 없음 | Prime ECC는 turbo/BCJR 미사용(min-sum LDPC 전용) |

적용 가치: **낮음** — turbo-like 부호의 projected-graph 고속복호·interleaver 프레임워크로, NAND binary QC-LDPC의 부호설계·min-sum 디코더에 직접 떼어낼 요소가 거의 없음(메모리 라우팅 관점만 참고 가치).
