### B. 알고리즘 요약

1. 대상: recording/wireless용 구조적 LDPC, regular(dv/dc), rate 1/3~0.95, block 1024~8900, 5-bit 양자화 메시지.
2. 문제: random-like LDPC의 그래프 배선 복잡도(routing congestion)·대형 칩이 완전병렬 VLSI의 병목 → code-design과 VLSI를 decouple한 기존 방식 비효율.
3. 핵심: VLSI 배치·라우팅 제약과 코드 girth/min-distance를 **공동 최적화**, 코드를 칩 배선에 embed(PCM 저장 불필요).
4. HW 핵심: check/variable 노드를 medium-size PLA 네트워크(NPLA)로 분해 - pre-charged NOR-NOR PLA로 crosstalk 면역·2-level 저지연.
5. VN: channel+타 edge 정보 합산, `log(dv+1)+1` 단 2입력 Manchester 가산기 스테이지, 부호가 현재 추정치.
6. CN: log-likelihood 도메인 reliability update, logtanh/logarctanh를 5입력5출력 32행 PLA LUT로 구현(dc=3 예).
7. 배치: check 1개+variable 다수 = cluster, 동심원(concentric) S1~S4 bank 배치, Hilbert-Peano 공간충전곡선으로 근접 배치.
8. 코드제약: V를 4분할하고 대각 bank끼리 교차연결 억제(`|Si∩Sj|<d`)해 중앙 관통 배선 제거 - array code 변형 H*로 생성.
9. 결과(0.1µm 추정): n=8900 rate3/4에서 최대 357Gbps@칩 6.198mm, rate1/2 n=1024는 2.956mm/0.723W(기존 [1] 7.5x7mm 대비 대폭 축소).
10. 성능대가: H* 변형이 diameter 증가로 원 array code 대비 BER 열화(단 rate3/4 16iter, 10^-8까지 error-floor 없음).
11. 한계: 코드=칩 하드와이어(재구성 불가), 라우팅·parity update·배선오버헤드 추정 제외, tanh 기반 full sum-product LUT.
