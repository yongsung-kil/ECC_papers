> 총평: QC-LDPC partly-parallel 디코더에서 CNFU/VNFU를 하나의 HW로 시분할 재사용해 21% 면적감소를 얻는 아키텍처 기법으로, NAND QC-LDPC HW 면적절감에 개념적으로 유용하나 LUT(SP) 전제·min-sum 대비 이득 재검증 필요.

### B. 알고리즘 요약

1. 시스템: regular QC-LDPC(`(3,5)`, 예시 `(155,64,31)`, rate `R≥1−j/k=0.4` 이상, 실제 155/64로 ~0.587), AWGN, partly-parallel 디코더.
2. 문제: partly-parallel BP 디코더는 CNFU/VNFU를 별도 HW로 두는데, 한 iteration에서 CN update와 VN update가 순차라 각 유닛이 절반은 idle → HUE 50%.
3. 부호구성: base all-one `j×k` 행렬의 각 원소를 `m×m` circulant(shift = `b^{s-1}·a^{t-1} mod m`)로 치환, `jm×km` PCM → 메모리 주소생성·배선 단순.
4. BP 원식: CNFU는 `Scv=∏sign(Lcn)`, `Rcv=−Scv·Ψ(Σ Ψ(Lcn))`, `Ψ(x)=log(tanh|x/2|)`는 LUT로 구현(식1,2); VNFU는 `Lv=Σ Rmv − 2rv/σ²`, `Lcv=Lv−Rmv`(식3,4).
5. 핵심 관찰: CNFU/VNFU가 sign-magnitude↔2's complement 변환과 Ψ LUT를 공유하는 구조적 유사성을 가짐 → 동일 HW로 통합 가능.
6. 핵심기법 1단(재정식화): 계산부하 재분배로 CNFU/VNFU 각각 critical path에 LUT 1개만 두도록 균형화(식5~8) — 원본은 CNFU에 LUT 2개, VNFU에 0개였음.
7. 핵심기법 2단(re-map): CNFU/VNFU가 clock당 처리하는 입력수가 동일(=block H의 non-zero 수, 예시 15)임을 이용, control 신호(0=CN, 1=VN)로 같은 HW를 시분할 → 매 iteration busy, HUE 100%.
8. 메모리 스케줄: `z`(intrinsic)·`ms,t`(extrinsic) 메모리, CN update는 block-row(같은 s의 k 메모리), VN update는 block-column(같은 t의 j 메모리), 각 update `m` clock cycle.
9. 결과: VHDL·Synopsys 합성, 원본 7634 gate → re-map 6041 gate로 **약 21% 면적감소**, 성능손실 없음, 부호길이(m) 무관하게 절감폭 유지.
10. 확장성: `(j,k)`에서 `k`가 `j`의 배수면 re-map이 더 단순, CNFU/VNFU 수 증가 시 로직 공유 비율↑로 절감폭 증가.
11. 한계: throughput/공정노드 명시 없음, LUT 기반 full-Ψ(SP) 전제라 min-sum 디코더에는 이득 직접 적용 불확실, error-floor/정정능력 논의 없음.
