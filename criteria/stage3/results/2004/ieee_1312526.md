### B. 알고리즘 요약

1. 대상: `(dv,dc)`-regular LDPC를 BSC에서 이진 메시지전달(majority-based, MB) 알고리즘으로 복호, 예제 `C(8,9)`·`C(4,5)`, finite length 9000.
2. 문제: 개별 MB 알고리즘(`MB0..MBdv-1`)이나 Gallager-B류 switch-type(HST)은 임계값/수렴이 채널파라미터 지식에 민감.
3. 아이디어: 여러 MB 알고리즘을 각 iteration에서 확률벡터 `α`로 노드를 무작위 분할해 혼합, 혼합비를 iteration 불변으로 고정한 "time-invariant hybrid"(HTI).
4. 메시지맵: `Ψv=Σ αi Ψi,v`, `Ψc=Σ βi Ψi,c` (MB는 CN맵 동일해 `β` 생략).
5. 수렴조건: DE 곡선 `hα(x)=Σ αω fω(x)`가 `hα(x)<x, ∀x∈(0,ε]`이면 오류확률 0 수렴 → 개별 MB가 실패해도 혼합이 성공 가능(Example1, ε=0.0720).
6. 최적 `α`는 exhaustive search로 threshold 최대화; Table I에서 HTI/HST 모두 구성요소 대비 threshold 최대 +39.7%(HTI)/+53.8%(HST) 향상.
7. HST(=Gallager-B) 최적화엔 정확한 채널값 필요, HTI는 채널 미상에도 max-threshold 세팅으로 광범위 채널에서 준최적(universal).
8. 강건성: 채널 과소추정 시 HST는 수렴 실패, HTI는 iteration <15~% 증가로 거의 영향 없음(Ex5,6).
9. 결과: length-9000 (8,9)코드에서 HTI가 HST를 약간 상회; BER=1e-5 평균 iteration HTI≈15.7 vs HST≈17.5, MB2 외 early error-floor 미관측(max iter=500).
10. 한계: BSC hard-decision·이진 알파벳 `{-1,1}` 메시지 한정, HW 미설계, soft/LLR·NAND 채널 미고려.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MB 이진 메시지전달 디코딩 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 이터레이션 루프는 대응하나 Prime은 min-sum(magnitude 양자화)이라 다수결 이진맵과 알고리즘 상이 |
| HTI 혼합(노드 그룹별 다른 규칙) | 대응 없음 | Prime에 노드별 알고리즘 혼합 스케줄 개념 부재 |
| CN/VN 메시지맵 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `VN_Cal_HD()` | HD 경로와 개념 접점 있으나 메시지 표현(부호+크기 vs ±1)이 다름 |
| threshold/DE 기반 설계 | 대응 없음 | Prime은 고정 QC-LDPC, DE 기반 알고리즘 설계 미포함 |

적용 가치: **낮음** — BSC hard-decision 다수결(MB/Gallager-B) 계열 디코더 개선으로, min-sum soft 기반 Prime ECC 3.1에 떼어 쓰기 어렵고 이미 보유한 HD/adaptive 스킴과 목적이 겹치지 않으면서도 계열이 이질적.
