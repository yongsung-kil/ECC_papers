### B. 알고리즘 요약

1. 문제: 랜덤 long LDPC 대신 소수 파라미터로 완전결정되는 구조적(well-structured) regular LDPC를 구성해 고속·저복잡도 구현(자기recording/광통신 >1Gb/s) 가능케 함.
2. 핵심틀: BIBD `2-(v,c,1)`의 point-block 결합행렬을 PCM으로 사용 -> `lambda=1`이 girth-4 부재(cycle length 4 없음) 보장, girth>=6.
3. 구성1(차집합족 DF/CDF): Abelian군 `Z_v`의 base block을 orbit로 전개해 quasi-cyclic PCM 생성(circulant 부분행렬). Bose/Netto/Buratti 구성으로 `2-(v,3,1)`,`c=4,5` STS 생성, 최적 rate-length tradeoff.
4. 핵심식: 코드 rate `R = 1 - rank_GF2(H)/n`, 하한 `1 - b/v` (loose). 열무게 `c`, 행무게 조정으로 rate 제어.
5. 최소거리 보증: Pasch configuration(quadrilateral) 존재 여부로 `d_min` 결정 - Pasch 포함시 4개 선형종속열 -> `d_min<=4`(error-floor), Pasch-free STS(Netto)면 `d_min=6`. 상한 `d_min<=c+1`.
6. 구성2(CIDS, 신규): `n-fold cycle-invariant difference set`을 정의(순환shift해도 차집합 유지), circulant permutation block으로 block-circulant PCM 생성 -> girth>=6, `d_min>=6`.
7. 구성3(정수격자): `q x q` 격자의 서로 다른 기울기 line을 block으로 -> 1-configuration, 큰 코드족 생성(rate 손실 대가), 격자에서 girth-8 subdesign 알고리즘(등차수열 없는 slope 집합 이용).
8. 인코딩: block-circulant PCM은 generator도 block-circulant permutation -> 매우 빠른 인코딩.
9. 결과(시뮬): 15~100 iter message-passing, rate 0.5~0.89, BER~10^-6. 차집합 코드는 5.75dB error-floor(예상), girth-8 열무게4 코드는 3.25dB에서 급락. 구조적 코드가 유사길이 랜덤(MacKay) 코드와 대등.
10. 한계: HW 미설계, 시뮬만, latency/gate/throughput 수치 없음. 5~6 iter 제약시 5회부터 겨우 이득, 고rate·단길이 코드 다양성 부족.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BIBD/CDF/CIDS 기반 QC-LDPC PCM 구성 | `ecc_top.cpp` `Load_PCM()` + H-matrix | 부호구조(circulant shift 배치) 설계에 개념 이식 - 그러나 특정 QC-LDPC 고정, 재검증 부담 |
| block-circulant permutation PCM (circulant shift) | `ecc_data.h` `PCM_b`(base+shift) | Prime ECC의 base+circulant shift 구조와 동형 - shift 배치 방법론 참조 가능 |
| block-circulant generator 고속 인코딩 | `encoder.cpp` `Generate_PCM_encoding()` | dual-diagonal과 다른 방식이나 인코딩 구조 대안 참고 |
| girth/Pasch로 최소거리·error-floor 하한 | 대응 없음 | Prime ECC엔 부호구조 error-floor 보증 로직 별도 없음 - 설계 단계 참고용 |
| iterative message-passing 복호 | `decoder.cpp` `LDPC_Decoder()` | 표준 SPA/MP 사용(min-sum 아님), 복호기 기여는 없음 |

적용 가치: **중간** — binary QC-LDPC 부호설계(circulant shift 배치, girth/최소거리 보증)라는 Prime ECC와 같은 부호 클래스를 다루나, HW 미검증·고rate 단길이 코드 부족·특정 length 종속으로 부호설계 계층 이식은 재검증 부담이 크다.
