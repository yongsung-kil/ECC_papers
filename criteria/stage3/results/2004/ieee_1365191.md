### B. 알고리즘 요약

1. 대상: array LDPC 부호를 rate-compatible 계열(RCA-LDPC)로 확장, 효율적 encodability와 decoder 구현성 확보.
2. 문제: rate-compatible convolutional/turbo가 가진 인코딩·디코딩 구조상 이점을 LDPC에서도 확보하고자 함.
3. 기본 구조: array LDPC PCM `H`는 prime `p`와 정수 `k,j`(`k,j≤p`)로 결정, block이 `p×p` 단위행렬 `I`, 영행렬 `O`, 순열행렬 `α`(cyclic shift of `I`)의 거듭제곱으로 구성 → 4-cycle free, 선형시간 인코딩([1]).
4. 핵심 기법 1(puncturing): `H`의 최상단 행·최좌측 열을 삭제하되 상삼각 구조 유지 → 고rate 부호 획득([2]).
5. 핵심 기법 2(extending): 4-cycle을 피하며 매우 sparse한 sub-matrix를 덧붙여 저rate로 확장([3]), 상삼각 유지로 선형시간 인코딩 보존.
6. building block: `α`와 그 거듭제곱만 사용 → 유연한 decoder 아키텍처([4]) 가능(QC-LDPC류 특성).
7. 예시 계열: `j=5, p=67, k=25` → mother code 길이 `N=1675`, rate `r=0.8`; 계열 길이 `1474~2680`, rate `0.5~0.909`.
8. 성능 평가: regular LDPC(다양한 column weight) threshold와 4개 RCA-LDPC의 `Eb/N0 @BER=10^-5` 비교(Fig.1).
9. 결과: puncturing이 고rate 부호 획득에 효과적, DSL 시스템에 매력적; hybrid-ARQ throughput 평가(Fig.2)에서는 RCA-LDPC가 ARQ에 부적합하다고 결론.
10. 한계: 1페이지 ISIT 요약, HW 미설계(디코더 arch는 [4] 참조), 통신(DSL/ARQ) 타깃, gatecount/throughput 수치 없음.
