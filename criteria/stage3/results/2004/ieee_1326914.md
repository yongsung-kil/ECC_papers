### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 채널, regular LDPC(`dv=3, dc=6`, rate=0.5, `N=10228`)를 TI TMS320C64xx fixed-point DSP에 구현.
2. 문제: LDPC BP의 check node `tanh/atanh`(식 3) 연산이 fixed-point DSP에서 비효율 → 실시간 처리 throughput 부족.
3. 채널/메시지 모델: 메시지는 LLR (부호=비트, 크기=신뢰도), 16bit 양자화로 성능손실 무시 가능 수준.
4. 핵심기법 1단: check node 연산을 Max-Log-MAP 유사 근사 `L ≈ min|Z|·∏sgn(Z)`(식 4)로 대체(min-sum).
5. 보정: offset BP `L ← sgn(L)·max(|L|−β, 0)`(식 5), β는 밀도진화로 코드설계 시 최적화(참조 [4]).
6. min 연산 최적화: 6입력 min을 pairwise(`x12,x34,x56`→조합)로 분할(식 7,8), 분기 없이 12 min/6 add/24 logic로 계산.
7. 병렬화 트릭: 32bit 워드에 odd/even 노드 메시지를 상/하위 16bit로 배치, intrinsic으로 2노드 동시 처리 → throughput 2배.
8. edge interleaver를 check node 연산에 통합(랜덤 메모리 접근을 연산과 오버랩).
9. 조기종료: 모든 check node의 sign 곱이 양수면 valid codeword로 판정해 반복 중단(이미 계산된 sign 재사용), 실패 블록 마킹 가능.
10. 결과: 11.1 cycles/iter/bit, 10 iter 가정 시 600MHz에서 5.4Mbps(동종 Turbo 대비 ~2.6배), 고 Eb/N0에서 cycle 수 Turbo보다 낮음.
11. 한계: HW(ASIC/FPGA) 미설계 순수 DSP SW, rate=0.5 regular, gate/전력 수치 없음, error-floor 논의 없음.
