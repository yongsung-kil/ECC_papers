> 총평: LDPC convolutional codes의 파이프라인 디코더 저장량·지연을 줄이는 스케줄링 기법으로, block QC-LDPC 기반 Prime ECC와 구조가 달라 이식성 낮음.

### B. 알고리즘 요약

1. AWGN 채널에서 `(ms, J, K)` LDPC convolutional codes(`J=3`, `K=6`, 부호길이 128 및 2048)의 continuous pipeline 디코딩을 대상으로 한다.
2. 문제: convolutional 파이프라인 디코더는 성능은 우수하나 저장량이 크고 초기 디코딩 지연(delay)이 길다.
3. 기법1 — 정지규칙(stopping rule): 무한 크기 parity-check 행렬 때문에 block용 정지규칙을 못 쓰므로, `P`개 연속 만족된 parity-check가 관측되면 프로세서가 sleep 모드로 진입(다음 위반까지 정지), `P`는 syndrome former memory `ms` 규모.
4. 기법2 — on-demand variable node activation: 표준 스케줄은 VN을 영역 이탈 직전 일괄 활성화하나, 대신 active constraint node가 메시지를 요청할 때마다 해당 VN을 활성화한다.
5. 효과: constraint node가 VN의 최신 정보에 접근 가능해져, 같은 iteration 내 후속 constraint node 활성화에 갱신 메시지 재사용.
6. 기법3 — compact pipeline decoder: on-demand 스케줄 하에서 인접 operating region을 overlap 시켜 저장량 감소(overlap이 처리영역 절반 이하면 성능 저하 무시 가능).
7. 결과: (128,3,6) 코드에서 Eb/N0=1.00dB일 때 비트당 평균 iteration 72회, 1.80dB에서 7회로 감소.
8. on-demand 스케줄이 고정 iteration에서 BER를 개선하고, 고정 BER에서는 필요 iteration을 줄여 초기지연·저장량 절감.
9. 계산 복잡도 희생 없이 지연·저장 요구를 줄임(최대 I=100 iteration 허용).
10. 한계: LDPC convolutional 전용 continuous pipeline 구조, 시뮬레이션만 수행, HW 미설계.
