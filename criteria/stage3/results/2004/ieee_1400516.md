### B. 알고리즘 요약

1. 대상: 고rate(R≥1/2) irregular extended-IRA LDPC, AWGN/BPSK, length 2000~10688, rate 1/2~8/9.
2. 문제: 고rate에서 최적 degree distribution은 degree-2 노드가 `n-k`보다 많아 degree-2 노드끼리 stopping set(사이클)을 형성해 error floor가 높음.
3. 모델: 패리티검사행렬 `H=[H1|H2]`, H2는 bi-diagonal(가중치-2 열 + 가중치-1 열 1개)로 accumulator 인코딩(`G=[I|H1·H2^-T]`) → 저복잡도 인코딩.
4. 핵심기법1: ACE(approximate cycle extrinsic message degree) = 길이 2d 사이클의 `Σ(di-2)`. 코드가 `(dACE, η)` 성질 = 길이 ≤2dACE 사이클이 모두 ACE≥η.
5. 식 의미: 열을 저차수부터 하나씩 무작위 생성하며 `(dACE,η)` 미충족 시 재생성 → harmful short cycle 회피(Viterbi류 선형복잡도).
6. 핵심기법2: 제약 최적화 — degree distribution을 `Nv(2)<n-k` 제약 하에 density evolution으로 재최적화(Scheme-B). 이로써 그래프 컨디셔닝이 성공적으로 수행됨.
7. 트릭: rate-compatible은 rate-8/9 mother code에서 information-nulling(정보블록에 0 삽입)으로 저rate 파생, degree-2/3/10만 허용해 degree 일관성 유지.
8. 최적화: density evolution으로 threshold 최소화, dACE 고정 후 η를 최대까지 증가시켜 후보 부호 시뮬 비교.
9. 결과: rate-3/4 length-10688 (B,4,6) 코드가 BER=10^-5에서 Shannon 한계 0.67dB, error floor <10^-7, semi-regular 대비 0.38dB 개선. threshold 손실은 <0.1dB.
10. 한계: AWGN 시뮬만, HW/디코더 미설계, QC 구조 아님, degree-2 제약이 소폭 threshold 손실 유발.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ACE 기반 그래프 컨디셔닝 / H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | 부호구조 로드 지점, 다만 Prime ECC는 고정 QC-LDPC이라 재설계 부담 큼 |
| extended-IRA bi-diagonal 인코딩 (`G=[I|H1·H2^-T]`) | `encoder.cpp` `Generate_PCM_encoding()` | Prime ECC는 dual-diagonal 인코딩, 유사 개념이나 부호 다름 |
| error-floor 개선(정정능력) | `decoder.cpp` `LDPC_Decoder()` | 부호설계로 error-floor를 낮춰 디코더 수렴 개선 (간접) |
| stopping set / 사이클 제거 | 대응 없음 | Prime ECC에 ACE류 사이클 탐색 모듈 없음 |

적용 가치: **낮음** — error-floor 개선 아이디어는 유효하나 QC-LDPC가 아닌 임의 irregular 구성이고 Prime ECC의 고정 H-matrix 재설계 부담이 커 부호설계 레이어에만 참고 가치.
