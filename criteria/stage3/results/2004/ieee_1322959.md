### B. 알고리즘 요약

1. 대상은 systematic irregular LDPC 부호로, 부호길이 `N = Nb·L = 48·127 = 6096`, rate=1/2, VN degree 분포 `λ(x)=0.0025+0.6366x+0.2406x^2+0.1203x^3`, CN 분포 `ρ(x)=0.4694x^5+0.5306x^6`.
2. 문제: LDPC 부호는 보통 random·nonsystematic이라 인코더 생성행렬이 dense해져 HW 구현(특히 인코딩)이 어렵다.
3. 구성법: `Mb×Nb` base parity-check matrix를 random으로 만든 뒤, 각 원소를 `L×L`(`L=2^p−1`, p 소수) sub-matrix로 확장. degree-1/degree-2 VN 수를 제약해 systematic 부호를 보장.
4. sub-matrix는 identity를 GF(2^p) 필드 곱셈 파라미터로 permute하여 생성하며, 확장 시 길이-4 짧은 cycle이 생기지 않도록 4가지 case로 row/col index를 배치.
5. 디코더: `Nb`개 VNCU + `Mb`개 CNCU와 3개 메모리 배열(MEMIO/MEMV/MEMC)로 구성된 partial-parallel 구조. message passing은 메모리 주소 생성(counter 또는 GF(2^p) field multiplier)과 메모리 블록 간 연결로 구현.
6. 디코딩 절차: 초기화(L clock) → iterative decoding(2·L clock, 전반 L은 VN 계산·후반 L은 CN 계산) → 출력(time-division multiplex).
7. 인코더: `c = X·G`를 sparse 부분행렬 곱과 upper-triangular `(E^-1)` 부분을 binary accumulator로 처리. MEMI + Mb개 PCBGU(parity-check bit 생성 유닛)로 구성, matrix inversion/permutation 제거.
8. 인코더는 디코더와 동일 구조를 가져 단일 FPGA에 통합 구현 가능.
9. 결과: Xilinx Virtex-E XCV3200E에 구현, 최대 클록 64MHz, 24 iteration 시 encoding/decoding throughput 약 64Mb/s. slice 사용률 낮아 routing 혼잡 회피.
10. 한계: 정정능력은 random 생성 대비 "동일" 유지만 주장(개선 없음), gate count·BER 곡선·복호 알고리즘(min-sum/SPA) 세부는 본문에 없음.
