### B. 알고리즘 요약 (10줄 내외)

1. 대상: array LDPC `C(q,m)` (홀수 소수 `q`, `m≤q`), 길이 `q^2`, `(m,q)`-regular QC-LDPC; 다룬 범위 `m≤7`, `q≤79`.
2. 풀려는 문제: 최소거리 `d(q,m)`·stopping distance `h(q,m)`의 q-독립 상계가 `m=6`에서 기존 `≤32`로 느슨하고 `m=7`은 미확립.
3. 핵심 개념: template support matrix — `m×w` 행렬로, 엔트리가 `q`의 함수이며 `q≥q0`인 모든 `q`에서 어떤 codeword/stopping set의 support matrix(열치환 허용)가 됨. codeword 열은 `(x, x+y, ..., x+(m-1)y) mod q` 형태(정수쌍 `(x,y)`로 표현).
4. 1단(그래프 cycle 구조): 서로 다른 `q1<q2`의 minimal codeword `c1,c2`에서 행쌍 `(i,j)`별 support matrix graph `G(i,j)`를 만들고, 특정 edge를 포함하는 proper cycle 길이 분포가 같으면 "same graphical cycle structure".
5. 왜 필요: 같은 cycle 구조를 가진 codeword 쌍이 같은 template의 instance일 가능성이 높음(공통 구조 추론의 근거).
6. 2단(template 추론, Algorithm 1): 두 cycle을 매칭해 열별로 2×2 방정식·합동식(CRT, extended Euclid의 `κ`)을 풀어 `x,y`의 "가장 단순한 해"를 채움 → 후보 template.
7. 부수 트릭: `a·2^{-1} mod q = (q+a)/2`, `I=m-1`로 성공확률↑; 개선판은 cycle 구조 완전일치 대신 "최소 cycle 길이 일치"만 요구해 큰 `m` 대응.
8. 3단(형식 증명): instance가 (i) 각 행 엔트리 짝수번 등장, (ii) 유효 열, (iii) column-reduced 후 비어있지 않음을 만족 → codeword support임을 증명. stopping set은 "행당 2회 이상" 조건으로 변형.
9. 결과: `d(q,6)≤20` (`q>11`, 기존 32 대폭 개선), 신규 `d(q,7)≤24` (`q>7`); 확률적 탐색상 두 상계 모두 tight로 추정, `q≤79` 신규 수치표 제공.
10. 한계: HW 미설계, 디코더 무관, 순수 최소/정지거리 상계; 큰 `m`은 low-weight codeword 탐색 실패로 확장 곤란.
