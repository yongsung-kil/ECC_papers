### B. 알고리즘 요약

1. QC-LDPC PCM을 B×D seed matrix에서 P×P 순환 shift identity sub-matrix로 확장(expansion factor `P`); rate 1/4~9/10, 코드길이 360~4200bit를 12종 seed matrix로 지원.
2. 문제: 무선 패킷 통신은 프레임 크기·QoS별로 다양한 block size/rate가 필요하나 기존 디코더(예: [1][2])는 고정 길이/rate만 지원, 다중 PCM을 온칩 저장하는 것은 비현실적.
3. 채널 모델: BPSK on AWGN, 채널 LLR 초기값 `2rj/σ²`.
4. 핵심 기법: 단일 seed matrix + shift값 `m(P0,i,j)`만 저장하고, 임의 `Px<P0`에 대해 `m(Px,i,j) = floor(m(P0,i,j)·Px/P0)`로 shift값을 재계산해 다양한 block size 생성.
5. 핵심 식: layered(row-layer) 갱신 `L(qij)=L(qj)-Rij`, offset min-sum `Rij ≈ sign(L(qij))·max(min|L(qij)|-β, 0)`; layered 갱신으로 수렴속도 2배(iteration 절반).
6. 확장: 6-bit 고정소수점 양자화로 unquantized BP 대비 약 0.1dB 열화만 발생(인용 근거).
7. 구현 디테일: permuter는 O(P log2 P) 복잡도의 barrel shifter로 구현, cyclic shift 특성 이용해 de-permuter 불필요(shift 증분 `Δ=(shiftₙ-shiftₙ₋₁) mod P`만 적용).
8. 파이프라이닝: 인접 두 layer의 read/min-sum과 write-back을 오버랩, layer 내부 column weight ≤1로 무의존이나 인접 layer 간 데이터 의존성으로 stall 발생 → horizontal rescheduling으로 read 순서 재배열해 stall 감소.
9. 결과: TSMC 0.13µm 합성, core area 4.5mm², 350MHz, 740mW, 2.2dB SNR에서 평균 1.0Gbps throughput; 기존 [1](52.5mm², 고정 rate/길이) 대비 면적 대폭 축소.
10. 한계: 통신(802.11n/802.16e류) FER 성능 비교 위주이며 NAND 채널(RBER, retention 등) 실험 없음, error-floor 분석 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Offset min-sum (sign+magnitude, offset β) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min-sum(min1/min2+sign XOR)과 동일 계열, offset 파라미터 비교 가능 |
| Layered(row-layer) 스케줄 | `decoder.cpp` `LDPC_Decoder()`, Dual-Update(odd/even 교대) | layered 방식은 Prime ECC의 Dual-Update 스케줄과 유사 목적(수렴 가속) |
| 2-layer 파이프라인 + horizontal rescheduling(stall 감소) | `decoder.cpp` pipeline 6-stage (col_M1/Z/P1~P4) | 파이프라인 데이터 해저드 회피 기법으로 기존 pipeline 구조와 직접 비교 가능 |
| Barrel shifter permuter (cyclic shift, no de-permuter) | `decoder.cpp` (VNU/CNU 간 라우팅), `GT.cpp` `Make_GT_HW()` | 순환 shift 기반 라우팅은 QC-LDPC 공통 구조이나 Prime ECC는 z=32 고정, 구체 대응 함수는 불명 |
| seed matrix + expansion factor 가변 block/rate 지원 | `ecc_top.cpp` `Load_PCM()` | 부호구조 설계 영역으로 이식 난이도 높음(§4 기준) |
| 6-bit 고정소수점 양자화 | `channel.cpp` LLR 양자화 테이블(`PARAM_LLR`) | 양자화 비트수/성능 열화 비교 참조 가능 |

> 적용 가치: 중간 — offset min-sum과 파이프라인 stall 회피(horizontal rescheduling)는 decoder.cpp의 기존 6-stage 파이프라인과 직접 비교할 만한 구체적 HW 디테일이나, 가변 block/rate 지원 자체는 Prime ECC의 고정 rate 단일 부호 구조와 목적이 달라 이식 난이도가 높음.
