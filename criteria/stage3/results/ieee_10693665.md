### B. 알고리즘 요약 (10줄 내외)

1. 대상 시스템은 3D NAND flash 기반 SSD의 FTL이며, LDPC 코드의 read retry(sensing level 1~7)로 인한 read latency 증가가 배경 문제.
2. 기존 방식(active-rewrite, interval-based rewrite)은 read latency가 길어지면 데이터를 재기록해 error bit을 제거하지만, 과도한 rewrite는 SSD 수명을 조기에 소모.
3. 핵심 가정은 read latency가 LDPC sensing level(오류 비트 수 구간)에 비례해 증가한다는 채널/시스템 모델(Table I, 실측값 인용).
4. 핵심 기법은 Q-learning: 상태 `s`=(inter-request interval 32단계, access type 2단계, idle time 4단계), 행동 `a`={rewrite, no rewrite, expand interval, shrink interval}.
5. 핵심 식은 `New Q(s,a) = Q(s,a) + α(r + γ maxQ(s',a') - Q(s,a))`로, 학습률 α와 할인율 γ를 통해 미래 보상을 반영한 재기록 정책 최적화.
6. 보상 함수는 dynamic-interval 방식의 행동과 비교해 일치 시 +100, 불일치(rewrite vs interval조정) 시 -50~+50으로 설계.
7. 구현은 FTL 내 Q-learning 컴포넌트와 dynamic interval 컴포넌트를 read 완료 후 트리거하는 방식으로, read 경로 자체에는 오버헤드 없음.
8. 학습/최적화는 온라인 Q-table 업데이트(256 entry, 4.25KB)이며 별도 학습 단계 없이 런타임에 갱신.
9. 결과는 active-rewrite 대비 rewrite 데이터량 최대 98.59% 감소, 평균 실행시간 4.55% 증가(트레이스 기반 시뮬레이션).
10. 한계는 FTL 정책 시뮬레이션에 국한되며 ECC 부호설계·디코더·HW 구현은 전혀 다루지 않음(기존 문헌의 LDPC latency 수치만 인용).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC sensing level별 read latency(배경 인용) | 대응 없음 | Prime ECC의 디코더/부호 알고리즘과 무관, 상위 FTL 정책 문제 |
| Q-learning 기반 rewrite 트리거 | 대응 없음 | ECC 코드베이스가 아닌 FTL 계층 로직 |
```

적용 가치: **낮음** — LDPC ECC는 배경 인용(read latency 테이블)에 그치고, 논문의 실제 기여(FTL rewrite 정책)는 Prime ECC 3.1의 부호설계/디코더/HW 어느 레이어와도 대응되지 않음.
