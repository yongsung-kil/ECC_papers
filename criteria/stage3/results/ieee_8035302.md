### B. 알고리즘 요약

1. 시스템: AWGN+BPSK, regular binary LDPC `(N,K,dv,dc)`, hard decision `zj`, 채널 magnitude `rj`를 신뢰도로 사용하는 WBF 계열.
2. 문제: IMWBF에서 큰 magnitude를 가지나 부호가 반대인 "outlier"는 신뢰도가 높게 취급되어 flip 기회를 못 얻어 오류 잔존.
3. 모델: outlier = magnitude 큰데 hard decision이 틀린 수신신호(deconstructive large noise). Monte Carlo로 다수 outlier가 고SNR에서도 flip 안 됨을 확인.
4. 핵심기법: 비트 `j`가 참여한 불만족 parity-check 수 `fj > θ`이면 수신 magnitude를 `rj = ρ·rj` (`0<ρ≤1`)로 동적 축소.
5. 식의 의미: magnitude가 줄면 check-based reliability `wj,i`(식1)와 flipping function `γj`(식2)가 커져 outlier의 flip 확률 상승.
6. 확장: magnitude update는 `Q` iteration마다 1회만 수행 — 갱신효과 대기 + 복잡도 절감 목적.
7. 구현 디테일: 파라미터 `α`(bit reliability weight), `θ`(불만족 임계), `ρ`(correction factor), `Q`(갱신 주기); 추가 곱셈복잡도 `Na/Q`로 무시가능 수준.
8. 최적화: 파라미터 스윕으로 `ρ=0.8, θ=1, Q=7~10`이 Code1에서 최적, 저column-degree일수록 이득 큼.
9. 결과: Code1 `(1008,504,3,6)`에서 IMWBF 대비 BER=1e-5에서 ~0.5dB 이득, DWBF 대비는 0.25dB 열위이나 복잡도 낮음.
10. 한계: HW 미설계·시뮬만, column degree 증가 시 이득 급감(Code2에서 0.1dB), min-sum BP 대비 근본 성능 열위.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| IMWBF flipping function / 반복 루프 | decoder.cpp LDPC_Decoder() | 우리는 min-sum BP라 BF 루프로 대체 불가, 개념만 참고 |
| outlier 수신 magnitude ρ 축소 | decoder.cpp VN_Cal_HD() 등, decoder.h Get_VNU_* | soft magnitude 조정 아이디어를 VN LLR 스케일링에 응용 여지 |
| 불만족 check수 기반 신뢰도 갱신 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | syndrome/check 정보 활용, min-sum과 연산 상이 |
| 채널 magnitude/양자화 | channel.cpp Set_R_Offset(), Set_LLR_Th() | AWGN magnitude를 soft-read LLR로 매핑 시 참고 |
| 조기종료(syndrome=0) | partial_CRC.cpp / full_CRC.cpp | 종료조건 개념 유사, 구현 상이 |
```

적용 가치: **낮음** — Prime ECC는 min-sum BP 기반인데 본 논문은 BF/WBF 계열의 소폭(0.1~0.5dB) 개량이고 HW 미설계라, outlier magnitude 축소 발상 정도만 참고 가치가 있다.
