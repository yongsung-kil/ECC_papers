> 총평: 5G PBRL LDPC용 4비트 information bottleneck 디코더로 min-sum 대비 우수한 waterfall 성능을 보이나, HW 설계·gatecount 검증이 없고 protograph 특화 구조라 QC-LDPC 이식은 개념(4비트 lookup table 기반 check/variable node 연산) 수준에 그침.

### B. 알고리즘 요약 (10줄 내외)

1. 대상은 5G 표준의 PBRL(protograph-based raptor-like) LDPC 코드(BI-AWGN, BPSK), K=1032 정보비트, rate `Rc=1/3~2/3` 범위, 4비트 정수 메시지로 완전 이산화된 디코더 설계.
2. 문제의식: finite-precision 메시지 패싱(min-sum 등)은 비트폭이 좁아질수록 성능 저하가 크고, 기존 IB 디코더는 특정 rate 전용이라 5G의 rate-compatible 요구를 못 채움.
3. Information bottleneck(IB) 방법으로 check/variable node 연산을 근사 산술이 아닌, mutual information을 보존하는 lookup table(`p(t|y)` 매핑)로 대체 — LLR을 전혀 계산하지 않고 정수 cluster index `t`만 교환.
4. Puncturing 처리: 펑처링된 노드는 랜덤변수 `P`(punctured 여부)를 IB에 결합해 `p(x,[t_ch^in,t1_in]|P)`를 재정의하고, `message alignment`([5] 기법)로 `(T_out,P)`를 압축변수 `Z`로 매핑해 `I(X;T_out,P) ≈ I(X;Z)`를 유지.
5. Message alignment 핵심 식: 서로 다른 degree `d`의 메시지를 정렬하는 `p(z|t_out,d)`로 `I(X;T_out,D) ≈ I(X;Z)`를 만족시켜 동일 cluster index가 유사한 신뢰도를 갖도록 보정.
6. Rate-compatible 확장: degree-1 variable node의 puncturing이 effective degree distribution `λ_eff(z), ρ_eff(z)`를 바꾼다는 점을 이용, 최저 rate(최대 lookup tree 깊이)용 테이블 하나로 모든 rate를 커버(더 높은 rate는 depth만 줄여 재사용).
7. 오프라인 설계: 고정 design-`Eb/N0`에서 lookup table을 1회 생성 후 전 SNR 구간에 재사용(density evolution 기반, iteration별 joint distribution 필요).
8. 참조 대비 구성: 64bit BP, 6bit layered NMSA, 4bit offset-min-sum(check 4bit/var 6bit) vs 제안 4bit IB(채널양자화 4bit, check/var 모두 4bit).
9. 결과: rate 1/2에서 제안 IB 디코더가 64bit BP 대비 0.2dB 이내 성능, 4bit offset-min-sum 대비 0.75dB 이득; 모든 검증 rate(1/3, 1/2, 2/3)에서 4bit offset-min-sum과 6bit NMSA를 모두 능가.
10. 한계: HW 구현(gatecount, throughput, 면적) 없이 시뮬레이션(FER)만 제시. 표를 만드는 계산(density evolution) 자체의 오프라인 복잡도는 논문에서 다루지 않음. Iteration 수/latency 개선 수치도 없음(max 100 iteration 종료 조건만 언급).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Check/variable node 연산을 4bit lookup table로 대체 (IB 방법) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()`, `VN_Cal_HD()` | Min-sum 대신 정보이론적 lookup table 대체 아이디어는 CN/VN 연산부에 대응되나, Prime ECC는 이미 min-sum+LLR 테이블(`PARAM_LLR`) 구조라 완전 대체는 재검증 부담 큼 |
| 채널 quantizer `p(t_ch\|y)` (soft-read LLR 양자화) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 개념적으로 유사(soft-read 비용 절감 목적)하나 IB 기반 클러스터 방식은 기존 threshold 테이블 방식과 설계 원리가 다름 |
| Message alignment (다른 degree 노드 메시지 정렬) | 대응 없음 | Prime ECC는 QC-LDPC 고정 degree(17, HCU 대상 2개 col만 예외) 구조라 이 논문의 다양한 degree 정렬 문제 자체가 크게 부각되지 않음 |
| Puncturing 지원 (rate-compatible PBRL) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 QC-LDPC H-matrix이며 PBRL식 puncturing 구조 없음 → 대응 미미 |

적용 가치: 중간 — IB 기반 4비트 lookup table로 CN/VN 연산을 대체하는 아이디어는 soft-read 비용 절감과 결이 맞아 `decoder.cpp`/`channel.cpp`에 참고할 여지가 있으나, protograph/PBRL 특화 구조(puncturing, message alignment)와 HW 미검증이 이식 난이도를 높임.
