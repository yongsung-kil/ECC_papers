### B. 알고리즘 요약

1. 코드 스펙: IEEE 802.11n LDPC 3종(Code1 R=1/2,N=1944 / Code2 R=1/2,N=648 / Code3 R=3/4,N=648), AWGN 채널, QPSK 변조, layered offset min-sum(OMS) 디코더 기반.
2. 문제: OMS/BP 반복복호의 높은 연산·메모리접근 비용을 줄이되 FER 성능 저하를 허용범위 내로 제한하는 것이 기존 forced convergence(FC)의 과제.
3. 채널/노드 모델: 반복이 진행될수록 신뢰도 높은 variable node의 LLR `Qv`가 `±∞`로 발산한다는 관찰에 기반, `|Qv|>θ`인 노드를 "frozen"으로 고정.
4. 핵심 기법: 기존 FC는 frozen node도 `Rcv`를 매 iteration 재계산했으나, 본 논문은 frozen node의 `Qtemp,vc`를 `Qv,frozen`으로 근사(extrinsic message simplification)해 layered OMS 계산에서 완전히 제외 (`Q_temp,vc = Qv,frozen if frozen else Qv-Rcv`, 식(1)).
5. 이 근사로 frozen node는 min1/min2 탐색·`Rcv` 계산·`Qv` 갱신 3단계 모두 skip되어, 활성 노드 수 `ñ_a`에 비례하던 layer당 복잡도 `3ñ_a + ñ_x + log2(ñ_x) + 2`가 감소.
6. 확장: θ를 채널조건 `ρ`(SNR)에 맞춰 자동 결정하는 제약최적화 `min C(θ,ρ) s.t. FER(θ,ρ)≤FERc`를 stochastic approximation(SA, Kiefer-Wolfowitz 변형)로 풀이 - `θ_{k+1}=Π_Θ[θ_k - a_k·∂Ĉ/∂θ]`.
7. 구현 디테일: 복잡도/FER는 폐형식이 아니라 window W=1000 블록의 표본평균으로 노이즈 추정, feasible set projection(θ≥0, FER(θ)≤φc)으로 발산 방지.
8. 학습/최적화 절차: SA 알고리즘(Algorithm II)이 온라인으로 θ*를 탐색, 수렴에 약 5만~10만 블록(100Mbps 가정 시 0.16~0.32초) 소요.
9. 결과: 기존 FC(θ grid search) 대비 동일 코드(802.11n R=1/2,N=648)에서 복잡도 절감 35%→53%(수정 규칙만), adaptive θ까지 적용 시 최대 55% 연산 절감·75% 메모리 접근(노드 활성) 절감.
10. 한계: HW 구현/합성 없음, AWGN·QPSK·802.11n irregular 코드 시뮬로만 검증, FER 개선이 아닌 동일 FER 유지 하 복잡도 절감이 목적(정정능력 향상 아님).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Forced convergence (노드별 조기 고정) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 Partial/Full CRC 기반 코드워드 단위 조기종료(§3)만 보유, per-bit(per-VN) 단위 조기고정은 미보유 - 개념적으로 유사하나 세부 구현 다름 |
| min-sum CN 연산 (Qmin1/Qmin2, offset ω) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min1/min2 기반 min-sum과 동일 계열 연산 구조 |
| 복잡도-성능 최적화(θ 자동탐색, SA) | 대응 없음 | Prime ECC는 iteration별 LLR threshold 테이블을 사전 설계(`PARAM_LLR`)하며 온라인 SA 최적화 루틴은 없음 |
| 메모리 접근(활성 노드 수) 절감 | `decoder.cpp` VN 처리 파이프라인 (`VN_Cal_HD()` 등) | frozen node skip에 의한 메모리 read/write 절감 아이디어는 HW pipeline 최적화에 참고 가능 |

> 적용 가치: 중간 - layered OMS의 노드별 조기고정 아이디어는 Prime ECC의 Partial CRC 조기종료(§3)와 개념적으로 겹치며 완전히 새로운 기법은 아니지만, "frozen 노드의 extrinsic message를 상수로 근사해 연산·메모리 접근을 절감"하는 세부 트릭은 decoder.cpp 파이프라인 최적화에 참고 가치가 있음. 단 θ 자동탐색(SA)은 NAND 실시간 요구와 맞지 않아 이식 우선순위는 낮음.
