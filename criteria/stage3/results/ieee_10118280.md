### B. 알고리즘 요약

1. 대상은 3D NAND Flash 스토리지 시스템 전반(SLC~QLC~PLC, plane/block 조직) 리뷰로 특정 ECC 부호/알고리즘이 아니다.
2. 문제: layer stacking으로 block size가 커지면 blocks/plane 감소 → spare block penalty·write amplification(WA)·zone size 증가.
3. Block by Deck(BBD): 물리 block을 deck 단위로 분리해 logical block 수를 늘려 block stripe 2배 확보, spare block 절감; 단 deck 간 disturb(aggressor/victim)와 refresh 증가 부작용.
4. 초고내구 SLC: performance buffer·system table 용도, cyclic buffer(감시/robo-taxi L5 black box) 등 endurance-limited 신규 유즈케이스, `TBW = write speed × active hours/day × useful life`.
5. TLC→QLC 전환: QLC는 cost/GB 우위, 성능·energy/bit·endurance·QoS 열위; SLC/MLC cache와 ZNS로 완화.
6. Retention: 세그먼트별 BOL/EOL retention 요구(Table I), power-off는 소자·power-on은 시스템 data integrity scan/refresh로 관리.
7. Trigger rate 문맥에서만 LDPC 언급: LDPC ECC가 hard decode limit(hard bit)·soft decode limit(hard+soft bit) 2단계, RBER이 hard limit 초과 시 error handling(EH) 트리거 → 성능/QoS 저하.
8. RBER 요인: 시간·온도·read disturb·cross-temperature, 인접 page coupling, read voltage miscalibration; QLC는 Vt state가 많아 time-0 RBER↑.
9. 결론적 방향: post-stress RBER 저감 기법·smart program/read 알고리즘으로 ECC spare byte 의존을 줄이는 것이 QLC/PLC cost 차별화 유지에 중요.
10. 한계: 정성적 시스템 리뷰(invited), 구체 수치·부호·디코더·HW 설계 없음, ECC는 요구조건 배경으로만 다뤄짐.
