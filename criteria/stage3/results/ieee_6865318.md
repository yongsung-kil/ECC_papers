### B. 알고리즘 요약

1. 시스템: NAND flash + Storage Class Memory(ReRAM 등) 하이브리드 SSD의 신뢰성·성능·전력 신호처리 기술 오버뷰(빅데이터 엔터프라이즈 스토리지 대상).
2. 문제: 미세공정에서 FG-FG capacitive coupling이 셀 신뢰성을 크게 저하시키고, 기존 soft LDPC는 정확한 VTH를 위해 다수(예 21개)의 Vref를 요구해 read time이 과도(21 Vref × 50µs = 1050µs)함.
3. 핵심 기법 1(EP-LDPC): Vref 3개만 사용하되 VTH·inter-cell coupling에 더해 write/erase cycle(`NW/E`)·retention time(`TRetention`)을 pre-recorded table로 참조해 BER을 예측·보정(error prediction unit).
4. 프로그램 시 lower page write-data의 `1`의 개수(`N"1"initial`)를 세어 user data에 부가 → read 시 retention time을 역추정.
5. 추정 BER을 LDPC decoder로 전달, 정정 실패 시 error recovery(ER) 펄스로 에러를 줄인 뒤 재정정.
6. 효과: read가 기존 soft LDPC 대비 7배 빠르고, 수명은 10배 이상 연장. NAND 컨트롤러 내 단순 로직 게이트로 구현(면적 오버헤드 무시가능).
7. 핵심 기법 2(Adaptive Codeword ECC): SCM은 32KByte, NAND는 2KByte codeword를 적응적으로 적용. codeword가 클수록 acceptable RBER↑(단 면적·전력↑).
8. NAND는 16채널 10Gbps read 위해 16개 ECC 회로 필요→codeword 2KB; SCM은 2.6Gbps 단일 채널이라 16배 면적 허용→codeword 32KB로 확장.
9. 결과: SCM의 acceptable RBER 2.6배(2K→32KB), NAND/SCM 합산 3.6배·1.8배 향상. 동일 ECC decoder 공유로 면적 오버헤드 무시가능.
10. 한계: 오버뷰/리뷰 논문으로 실제 LDPC 부호·디코더 알고리즘·gatecount·throughput 세부 없음. 하이브리드 SSD(ReRAM+NAND) 데이터 관리(11배 성능, 93% 에너지 절감)는 ECC와 무관.
