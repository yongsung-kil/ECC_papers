### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 채널, LDPC 부호 (1008,504) PEG(dv=3,dc=6)와 (1023,781) FG-LDPC(dv=32,dc=32) 두 종.
2. 문제: GDBF는 저복잡도지만 MS 대비 성능이 크게 열세 - 성능 격차를 줄이려 함.
3. 모델: 수신 `y_n = x_n + v_n`, 경판정 `z`, 신드롬 `s = u_l H^T`로 반복 복호.
4. 핵심기법: GDBF 목적함수 `f(x)=Σx_k y_k + Σs_i`, flipping function `E_n = u_n y_n + Σ(1-2s_m)` 최소값 비트를 flip.
5. 핵심 아이디어: 대부분 노드 신뢰도는 신뢰 가능하나 flip 대상 노드의 신뢰도는 불신 - 실시간 정보로 갱신 필요.
6. 확장(HGDBF): flip된 노드의 신뢰도를 `y_n = y_n + u_n·η·(N0/Eb)·E_n^k`로 반복 끝마다 갱신 (Step 4 추가).
7. 트릭: `E_n`이 `y_n`보다 커서 진동 유발 가능 → 가중치 `α = η·N0/Eb` (η≈0.5)로 안정화.
8. 최적화: η 최적값은 복호 전 시뮬레이션으로 사전 결정.
9. 결과: GDBF 대비 약 0.4dB 이득 (BER 10^-4), 반복당 곱셈 2회만 추가 증가.
10. 한계: MS 대비 (1023,781)에서 0.8dB, (1008,504)에서 2.8dB 손실, HW 미설계, 순수 시뮬.
