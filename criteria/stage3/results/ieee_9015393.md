> 총평: NAND용 긴 QC-LDPC 디코더의 라우팅 혼잡을 block interleaver data packing으로 푸는 HW 아키텍처 - 우리 메모리/라우팅 레이어에 참고가치 있으나 정정성능 기여는 없음.

### B. 알고리즘 요약

1. 대상은 NAND flash 컨트롤러용 QC-LDPC 디코더, 예제 code length `10240`, (40,4) QC-LDPC, parity length `1024`, rate `0.9`, 3bit 양자화.
2. 문제: 긴 code length QC-LDPC는 message passing의 랜덤 액세스로 라우팅 혼잡(routing congestion)과 floor planning 복잡도가 커 ECC 엔진이 HW 자원을 과다 점유.
3. 핵심 가정: circulant cyclic shift로 랜덤화된 메시지 순서가 BRAM 워드 액세스와 불일치해 중복 대기(redundant address access)를 유발.
4. 핵심 기법: 수신 LLR을 RAM 저장 전 block interleaver `π(i)=(p/l)*(i mod l)+(i/l)`로 재배열해 같은 check node로 갈 메시지를 동일 BRAM 주소에 packing.
5. 이 재배열식은 depth `l`, 묶음 `p/l`(단 `l`은 `p`의 인수)로 한 주소에 한 bundle을 담아 CNP가 circular shift만으로 정렬 메시지에 접근하게 함.
6. 아키텍처: VNP+CNP를 결합한 5개 process unit; 2개 `Zi,j` 블록을 동시 처리하는 partial-parallel 구조; flip-flop 재타이밍으로 최단경로 확보.
7. 구현 디테일: BSRAM wordlength 96bit(3bit 양자화 × 32), block RAM은 8개 `Zi,j` 블록을 4 section으로 나눠 액세스, CNP 후 최소값 선택(min-sum류).
8. 학습/최적화 절차 없음(고정 HW 데이터패스 설계).
9. 결과: TSMC 0.18um 합성 636.74K gate, decoding latency 6144 clock, clock 320MHz에서 throughput 최대 633Mbps; FPGA floor planning으로 라우팅 개선 확인.
10. 한계: 정정성능(BER/error-floor) 개선 아님, throughput 낮음(0.633Gbps), 구공정(0.18um), 아키텍처 상세는 그림(Fig.1~3) 의존으로 본문 서술 제한적.
