### B. 알고리즘 요약

1. 대상은 이진 classical block code(BCH, 확장 Golay 등)로, 패리티체크 행렬이 조밀(dense)하고 short cycle이 많아 표준 반복복호 성능이 낮음.
2. 기존 RRD(Random Redundant Decoding)는 코드의 순열군(permutation group) `Per(C)`에서 무작위 원소를 뽑아 매 반복마다 LLR을 순열시키고 damping coefficient `α`를 점진적으로 증가시킴.
3. 기존 MBBP(Multiple-Bases Belief-Propagation)는 dual code의 최소weight 코드워드(CGG)로부터 만든 `n×n` 패리티체크 행렬 여러 개를 병렬 디코더에 각각 적용.
4. 제안 기법 mRRD는 두 접근을 결합: 여러 병렬 디코더가 동일한 `(n−k)×n` 패리티체크 행렬을 쓰되, 각 디코더 내부에서 외부루프(`I2`회)마다 무작위 순열 `θ`를 적용하고 damping coefficient는 고정값으로 둠.
5. 각 디코더는 외부루프 `I2`회 × 내부루프(SP 반복) `I1`회의 이중 루프 구조이며, 유효 코드워드(`ĉ·H^T=0`) 도달 시 누적 역순열 `Θ^{-1}`을 적용해 종료.
6. 여러 병렬 디코더(`l`개) 결과 중 least metric selector(LMS)로 최종 코드워드 선택; 아무도 수렴 못하면 전체 후보 중 LMS 선택.
7. 복잡도는 그래프 edge 수로 정규화해 비교(Table I: Golay/BCH 코드별 relative complexity 1.4~3.23).
8. 학습/최적화 절차 없음 (파라미터 `I1, I2, l`은 실험적으로 설정).
9. 결과: mRRD(3 병렬)가 RRD와 동일 성능을 1/10(3dB) ~ 1/3(7dB) 복잡도로 달성, mRRD(20)은 RRD 대비 0.25dB 이득, MBBP 대비 평균 반복수 20 vs 66(약 10배 연산량 차이).
10. 한계: HW 구현 없이 순수 시뮬레이션(AWGN, BPSK)뿐이며, 코드길이 24~63의 짧은 classical block code에 국한, LDPC의 sparse 구조를 전제하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| mRRD 반복 복호 알고리즘 전체 | 대응 없음 | dense parity-check 기반 classical code용 SP 디코딩으로 sparse QC-LDPC min-sum 구조(`decoder.cpp` `LDPC_Decoder()`)와 전제가 다름 |
| 순열군 기반 패리티체크 행렬 재구성 | 대응 없음 | QC-LDPC의 고정 base+circulant 구조(`ecc_top.cpp` `Load_PCM()`)와 무관한 classical code 자동사상군 개념 |
| 병렬 디코더 + LMS 선택 | 대응 없음 | Prime ECC는 단일 PCM 기반 파이프라인 구조로 병렬 다중-PCM 방식 미보유 |

> 적용 가치: 낮음 — 대상 코드가 BCH/Golay 등 dense HDPC이며 binary QC-LDPC의 sparse 구조·min-sum 디코더 이식 대상과 근본적으로 다름.
