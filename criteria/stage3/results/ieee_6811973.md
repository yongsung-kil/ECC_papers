### B. 알고리즘 요약

1. 타깃: WiMAX 5/6 code rate LDPC, 광통신 100Gb/s급 초고속 처리량 목표(완전병렬 디코더).
2. 문제: 완전병렬 LDPC 디코더는 병렬도가 커질수록 배선혼잡(wiring congestion)이 급증해 면적·속도가 병목.
3. 알고리즘: Normalized Min-Sum(식 (1)) 사용, 스케일링 팩터 `D`로 `min` 근사 보정, SPA의 `tanh` 대체.
4. 갱신식: `R` = CN→VN(부호곱 × min), `L` = VN→CN(식 (2)), 사후정보 `S`(식 (3)) → 경판정 식 (4), `x·H^T=0`(식 (5))이면 조기종료.
5. 핵심 트릭 clock multiplexing: 메모리를 MEM0(positive edge)/MEM1(negative edge)로 2배화, 조합블록(I2RLS, RLS_RENEW)은 공유, `ACTIVE` 신호로 두 코드워드를 클록 양엣지에 교대 처리.
6. 효과: 한 디코딩 사이클에 코드 2개 처리 → 종래 완전병렬 대비 처리량 2배, 조합블록 공유로 2칩 대비 면적 절감.
7. 메모리 최적화: R_MEM 16bit 단위(min 5bit, 2nd min 5bit, min 위치 5bit, sign곱), LS_MEM은 sign bit만 저장 → layered 대비 LLR 메모리 5/6 절감.
8. 양자화: 26666 스킴(INTR/L/R/S 6bit, 2bit 소수부), max iteration 15, error floor 없음.
9. 결과: SMIC 0.13um 200MHz에서 54.2Gb/s, 면적 57.6mm2, ACE(throughput/area/clock) 1.881로 비교군 최고. 65nm·400MHz 시 108.4Gb/s 추정.
10. 한계: 전력 매우 높음(추가개선 필요), gatecount 미제시, 광통신/WiMAX 특화, NAND soft-read·고rate QC-LDPC와 무관.
