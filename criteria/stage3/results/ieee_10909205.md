### B. 알고리즘 요약 (10줄 내외)

1. 시스템: URLLC(5G 초저지연) 응용을 겨냥한 짧은 블록부호(길이 40~4920, rate 1/2) 구성법. AWGN+BPSK 채널.
2. 문제: 기존 FTP 코드(길이 2n, 차원 n, `d_min≥2√n`)는 order 2/3/5 원소로 구성 시 MDS이지만, 이진 weight spectrum이 랜덤부호보다 나쁨(저중량 코드워드 多) → 성능 개선 필요.
3. 핵심 가정: 유한체 `F_q`(q=p^m)에서 order n 원소 β로 FTP 코드 생성행렬 `G_FTP=[I,P_FTP]`(Vandermonde 행렬 `P_FTP`) 구성, 대각 랜덤행렬 `S=diag(α^j0,...,α^j(2n-1))`를 곱해 GFTP로 일반화(`G_GFTP=G_FTP·S`).
4. 핵심 기법 1단: GFTP 부호는 FTP와 동일한 symbol weight spectrum `A_w`(MDS 케이스 n=2,3,5는 닫힌형태 `(3)(4)`)를 갖지만, 이진 이미지 변환 시 원소별 이진 벡터의 이항분포 특성으로 평균 이진 weight spectrum `A(b)(x)`가 랜덤부호에 더 근접.
5. 핵심 식: 평균 이진 weight enumerator `A^(b)(x) = 1 + Σ A_w (W_s(x))^w`(식 6) — 심볼당 이진 가중치 분포(이항분포, 식 5)를 심볼 weight spectrum과 결합해 이진 이미지의 저중량 코드워드 수를 줄임.
6. 확장: 특성 2인 체(q=2^m)로 제한해 이진 이미지 `C_GFTP^(b)`를 정의, parity-check 행렬 `H^(b)`는 companion matrix `A_i`로 원소를 치환해 구성.
7. 부수 트릭: GFTP 인코딩은 FTP 대비 2n번의 추가 유한체 곱셈만 필요(랜덤 대각행렬 S로 저중량 코드워드 억제).
8. 디코딩: LC-OSD(local-constraint ordered statistic decoding) 알고리즘 사용 — LLR 정렬 후 Gaussian elimination으로 reduced echelon form, k+δ 확장 MRB에 대해 serial list Viterbi(SLVA)로 최소 soft weight TEP 탐색.
9. 결과: `C_GFTP[10,5]` (F_2^12, m=12)의 이진 이미지가 eBCH[128,64], LDPC[128,64], CA-polar[128,64]와 유사한 성능으로 RCU bound에 근접(길이가 늘수록 성능 개선).
10. 한계: 순수 시뮬레이션(BER curve)뿐 HW/실측 없음, LDPC는 비교 baseline일 뿐 본 논문의 기여 대상이 아니며 min-sum/BP 디코더가 아닌 OSD 계열 알고리즘 사용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GFTP 부호 구성(유한체 Fourier transform) | 대응 없음 | QC-LDPC와 무관한 대수적 부호(Vandermonde 기반), Prime ECC의 base matrix+circulant 구조와 이질적 |
| LC-OSD 디코딩(SLVA) | 대응 없음 | min-sum 계열이 아닌 ordered statistic decoding, Prime ECC min-sum 파이프라인(`decoder.cpp`)과 알고리즘 구조 상이 |
| LDPC(128,64) 비교 baseline | 대응 없음 | 논문 기여 대상이 아닌 성능비교용 참조부호일 뿐 |

> 적용 가치: **낮음** — 논문의 핵심 기여(GFTP 부호 구성, LC-OSD 디코딩)가 QC-LDPC 및 min-sum 디코더 프레임워크 밖에 있고, LDPC는 단순 비교 baseline으로만 언급되어 Prime ECC 모듈 어디에도 대응되지 않음.
