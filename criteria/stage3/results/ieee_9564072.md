### B. 알고리즘 요약

1. 시스템: 고rate product code(PC)/staircase code(SC), 성분코드 = (n,k,t) BCH 또는 even-weight subcode, 광통신 FEC(rate>0.8, 예 (511,484,3) PC rate≈0.897).
2. 문제: HDD는 채널 hard-decision으로 SDD 대비 1~2dB 손실, 그러나 SDD는 고속·저전력에서 부담 -> 저비용 절충 필요.
3. 모델: BI-AWGN 출력을 3-level(ternary) ADC로 양자화 `{0,?,1}`, `|r̃|<T`면 erasure "?", 채널 `(Es/N0, T)`의 전이확률 `δc, Δc` (식1).
4. 핵심기법: 성분코드를 error-and-erasure decoder(EaED, BDD 2회+판정로직)로 복호, ternary message passing으로 CN/VN 갱신(EMP/IMP).
5. 핵심식: `2D+E < d_des(t)` (식2) - D오류 E erasure를 확실히 정정하는 조건; even-weight subcode는 `d_des=2t+2`로 erasure 1개 추가 정정.
6. 확장: EaED+ (Hamming sphere `S_t^3` 결정론적 버전) 도입, 두 복호기 성능이 구성에 따라 교차.
7. 구현/분석 트릭: GLDPC/SC-GLDPC 앙상블로 근사, weight/biweight 분포와 decoder transition prob `T_{α→β}(D,E)`로 miscorrection 포함 DE 유도.
8. 최적화: noise threshold `(Es/N0)*(T)` 최소화하는 `T_opt` 탐색 - capacity 최대 T와 다름(문턱 이득 < capacity 이득).
9. 결과: EaED가 HDD 대비 최대 0.6dB(예 (63,39,4) SC 0.65dB) 이득, ternary 추가복잡도 소량, Monte-Carlo로 DE 검증(early error floor 없음).
10. 한계: HW 미설계(이론+시뮬만), IMP는 DE와 불일치, 성분코드가 BCH 기반 대수복호(BDD)로 binary LDPC min-sum과 무관, 광통신 타깃.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| product/staircase(GLDPC) 부호구조 | 대응 없음 (binary QC-LDPC 전용) | 부호 클래스 상이, Load_PCM 구조와 무관 |
| BCH 성분 BDD / EaED 대수복호 | 대응 없음 | min-sum 반복복호와 계열 다름 (turbo/algebraic류) |
| ternary(3-level) 채널·erasure 양자화 threshold T | channel.cpp Set_R_Offset(), Set_LLR_Th() | soft-read threshold 최적화 개념만 정성적 참고 가능 |
| DE 기반 quantizer 최적화 | 대응 없음 (local_opt.cpp는 LLR 자동최적화 별개) | 이론 분석틀, 우리 코드 직접 대응 없음 |
```

적용 가치: **낮음** — 부호(product/staircase+BCH)와 복호(대수 BDD/EaED)가 Prime ECC의 binary QC-LDPC min-sum과 근본적으로 다른 계열이며 HW도 없다. ternary erasure threshold 최적화 아이디어만 soft-read LLR threshold 설정에 정성적 참고 여지가 있을 뿐이다.
