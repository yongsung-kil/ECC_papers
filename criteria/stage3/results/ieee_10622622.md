### B. 알고리즘 요약

1. 시스템: Quantum LDPC(QLDPC) 코드 `[[n,k]]`, n개 물리 큐비트로 k개 논리 큐비트 인코딩, i.i.d. symmetric depolarizing noise model(`pX=pY=pZ=p/3`)에서 bit-flip(X) 에러 정정.
2. 문제: Tanner graph 내 short cycle로 인해 BP 기반 SB-MS(syndrome-based min-sum) 디코더가 특정 QLDPC 부호에서 error floor를 보임(물리 에러율 개선이 논리 에러율 개선으로 이어지지 않음).
3. 핵심 가정: 신드롬 `s=eH^T`만 관측(신드롬 측정은 오류 없다고 가정), 디코더는 신드롬에 매칭되는 에러 패턴 `ê`를 추정.
4. 핵심 기법 1단(SB-LP): Vontobel-Koetter의 classical LP decoding을 QLDPC용으로 수정 — 각 엣지 `u_i,j`를 `u_i,j = (α1/2)(T_i,j^(0) - T_i,j^(1) - S_i,j)`로 병렬 갱신(식, Algorithm 2), `S_i,j = λj + Σu_i',j`.
5. 핵심 식의 의미: `T_i,j^(0/1)`는 신드롬 비트 `si`에 따라 even/odd parity 집합(`Bi`/`Ci`) 위에서의 max 연산으로, 기존 VN-CN 교대 업데이트 대신 엣지별 완전 병렬 갱신을 가능케 함(디코딩 속도 개선).
6. 핵심 기법 2단(결합): SB-MS로 먼저 빠르게 수렴 시도 후, 신드롬 해밍거리 `dH(ŝ_pre, ŝ) ≤ dv`(VN degree)를 early stopping 기준으로 삼아 SB-MS 종료 시점을 자동 검출, 그 결과(사후 LLR)를 SB-LP 초기값으로 넘겨 post-processing 수행(Algorithm 3).
7. 부수 트릭: SB-LP는 Vontobel-Koetter 원 알고리즘의 "soft minimum"을 단순 minimum 연산으로 대체(성능 손실 미미), dual 변수 제거 등 단순화.
8. 최적화 절차: 별도 학습 없음, 스케일링 계수 `α=0.75`(SB-MS), `α1=0.9`(SB-LP)는 실험적으로 고정.
9. 결과: `[[882,24]]` QLDPC 코드 B1에서 SB-MS 단독은 p=0.04에서 logical error 10^-2 근처 error floor 발생, 결합(early stopping 적용) 디코더는 동일 총 iteration 대비 2~3 자릿수 error floor 개선, p=0.04에서 3자릿수 이상 개선.
10. 한계: 이론+시뮬레이션뿐 HW 미설계, quantum stabilizer code(신드롬 기반, depolarizing 채널) 전용 구조로 classical binary LDPC 채널/부호 모델과 직접 대응되지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SB-MS min-sum 연산 (CN sgn·min) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | classical min-sum과 CN 연산 구조는 유사하나 신드롬 기반 갱신(`si` 의존 항)이라 그대로 대응 안 됨 |
| Early stopping criterion (신드롬 해밍거리 기반) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 조기종료 개념은 유사하나 CRC 기반이 아닌 syndrome oscillation 검출이라 판정 로직 자체는 다름 |
| SB-LP post-processing (LP relaxation) | 대응 없음 | Prime ECC에 LP 기반 후처리 디코더 없음, non-binary/quantum 특화 |
| Trapping-set 기반 error floor 완화 | 대응 없음 | QLDPC 신드롬 구조 특화 trapping set 처리로 코드 직접 대응 없음 |

> 적용 가치: **낮음** — quantum stabilizer 부호의 신드롬 디코딩 특화 기법으로 채널/부호 구조가 NAND binary LDPC와 근본적으로 다름. 다만 "신드롬 오실레이션 감지 기반 조기종료"라는 아이디어 자체는 개념적으로 참고할 여지가 있음.
