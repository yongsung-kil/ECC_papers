### B. 알고리즘 요약

1. 대상 시스템은 sub-22nm MLC NAND SSD (8채널×8칩), I/O 링크 스펙 166MBps(ONFI 2.2), 페이지 4/8/16kB.
2. 문제: 최악조건 지향 ECC가 대부분의 시간·페이지에서 과잉 정정능력(under-utilization)을 가짐 → 낭비되는 정정여유를 활용하고자 함.
3. 핵심 아이디어: 무오류 I/O 전송을 오류허용 전송으로 전환하고, 잔여 ECC 정정능력으로 오버클로킹 유발 I/O 링크 에러를 흡수 (`τ_xfer` 감소).
4. 실측: FPGA(Altera Stratix IV) 플랫폼으로 read/write 경로별 오버클로킹 시 I/O 링크 raw BER 특성화; 237.5MBps 이하 거의 무오류, 275MBps에서 BER>1e-3.
5. read 경로 오버클로킹이 write보다 유리 — read 에러는 재전송으로 복구 가능하나 write 에러는 영구 저장오류.
6. 관찰: 동일 die 내 서로 다른 plane이 서로 다른 I/O 에러율 → plane별 이종(heterogeneous) 오버클로킹 여지.
7. 적응 기법 1: aggressive over-clocking + read-retry(ECC 실패 시 저속 재전송, 페이지 레지스터 잔류 시 재센싱 불필요).
8. 적응 기법 2: multi-transfer-rate(plane별 다른 전송률), P/E 사이클 열화에 따라 동적 전환.
9. 결과: DiskSim 트레이스 구동 시뮬로 read 응답시간 최대 ~20%(16kB) 감소, prj 트레이스 26.1% 감소.
10. 한계: ECC(LDPC rate-8/9 4kB, layered min-sum)는 성능 벤치마크 입력으로만 인용 — 부호/디코더 자체 기여 없음, 코드베이스 이식 대상 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC rate-8/9 4kB layered min-sum (벤치 인용) | 대응 없음 | 디코더 성능은 배경 비교용, 신규 알고리즘 아님 |
| I/O 링크 오버클로킹 / 에러 예산 관리 | 대응 없음 | 컨트롤러 I/O 아키텍처 영역, ECC 코어 밖 |
| plane별 raw BER 변동/적응 | 대응 없음 | 채널 통계는 [channel.cpp](../Prime_ECC_3.1_Claude) 성격이나 논문은 I/O 링크 에러로 ECC 코어와 무관 |

적용 가치: **낮음** — LDPC를 블랙박스 정정능력으로만 소비하는 SSD I/O 시스템 논문으로, 부호설계·디코더·HW 어느 레이어에도 떼어 쓸 신규 기법이 없다.
