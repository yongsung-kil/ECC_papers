### B. 알고리즘 요약

1. 대상 시스템은 2X/1X/mid-1X MLC NAND 기반 512GB SSD, LDPC ECC는 HD(hard decision) 실패 시 SD(soft decision)로 전환하는 2단 구조.
2. 표준 SD는 soft level마다 NAND 페이지를 2회 read하고 그 결과를 2회 별도로 ECC 엔진에 전송(data move)하여, N레벨 SD 시 N회 페이지 전송이 필요해 전력·지연이 증가.
3. 핵심 가정은 NAND 플래시 내부에서 2회 read 결과를 조합할 수 있다는 것 - 이 조합을 메모리 내부에서 수행하면 ECC로 보내는 데이터량을 줄일 수 있음.
4. 제안 기법 NA-SD(NAND-Assisted Soft Decision)는 각 soft level마다 NAND 내부에서 2회 read 후 결과를 결합해 ECC로는 1회만 전송(`#n page read, #n/2 page move`), 표준 SD 대비 데이터 전송 횟수를 절반으로 감소(Tab. 2).
5. LDPC 디코딩 알고리즘 자체(min-sum, 반복 구조 등)는 표준 방식과 동일하며 변경 사항 없음 - 오직 SD 단계의 I/O 데이터 전송 경로만 최적화.
6. HD 단계는 4320 Bytes 코드워드당 최대 100 errors 정정 능력의 BCH급 강도로 설정, SD 단계는 2 level로 고정(2-level SD 후 항상 정정 성공 확인).
7. 검증은 실제 FPGA 기반 NAND 특성화 보드 + LDPC HW 보드로 실측 RBER/지연 데이터를 수집하고, SSDExplorer co-simulation 프레임워크로 SSD 전체 성능(대역폭/지연/전력)을 산출.
8. 별도의 학습/최적화 절차는 없음(실측 기반 시뮬레이션).
9. 결과: mid-1X 노드에서 read bandwidth 19%, 평균 read latency 23%, NAND I/O 전력 91% 개선(9k P/E 시점); 2X/1X는 성능 개선은 미미하나 전력은 각 32%/8% 개선.
10. 한계: LDPC 부호/디코더 알고리즘에 대한 기여가 아니라 시스템(SSD controller-NAND 인터페이스) 레벨 최적화이며, mid-1X처럼 uncorrectable page 비율이 높은 경우에만 효과가 크고 2X/1X에서는 효과가 제한적.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NA-SD (NAND 내부 read 결합, ECC 전송량 절반화) | 대응 없음 | Prime ECC는 NAND 컨트롤러 내부 I/O 전송 구조를 다루지 않는 bit-exact 디코더 시뮬레이터로, NAND 칩 자체의 내부 read-combine 로직은 범위 밖 |
| HD+SD 2단 디코딩 구조 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold 개념은 유사하나 논문은 그 자체를 개선하지 않고 전송 경로만 변경 |
| 조기종료(2-level SD 확정) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 개념적으로 유사하나 논문에서 상세 메커니즘 언급 없음 |

> 적용 가치: 낮음 - 논문은 LDPC 부호/디코더 알고리즘이 아닌 NAND-ECC 간 데이터 전송 아키텍처를 다루므로 Prime ECC(디코더 시뮬레이터)에 이식할 알고리즘적 요소가 없음.
