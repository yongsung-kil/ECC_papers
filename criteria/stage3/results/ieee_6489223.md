> 총평: MIMO 반복 검출/복호용 knowledge-aided reweighted BP(full tanh) 논문. 오프라인 그래프 지식 기반 reweight 아이디어는 흥미로우나 MIMO·SP-BP 특화로 NAND min-sum ECC 이식성 하.

### B. 알고리즘 요약

1. 시스템: `NT×NR` 공간다중화 MIMO(예 4×4), Rayleigh flat fading, LDPC(PEG 설계, `N=1000`, rate `R=0.5`, VN degree 3/5, girth 6)-coded, IDD(반복검출복호) 구조.
2. 문제: 표준 BP는 그래프에 짧은 사이클이 있으면 수렴 미보장·저SNR에서 반복 많음·delay 증가, MIMO IDD에서 성능 저하.
3. 검출: SISO PIC(병렬간섭제거)+선형 MMSE 수신필터가 스트림별 간섭 제거 후 사후 LLR `Λ`를 계산, LDPC 디코더와 외부 반복 교환.
4. 핵심: reweighted BP 메시지 규칙 — VN→CN `Ψ_ji = Λ_ln,j + Σρ_i'Λ_i'j -(1-ρ_i)Λ_ij` (식 7), CN은 `2tanh⁻¹(Πtanh(...))` (식 8, full SP-BP)로 reweighting vector `ρ` 사용.
5. 의미: FAP(factor appearance probability) `ρ`로 사이클로 인한 과신뢰 메시지를 감쇠, `ρ=1`이면 표준 BP(온라인 추가복잡도 없음).
6. CKAR-BP: 짧은 사이클 카운팅([19] 행렬곱)으로 check node별 length-g 사이클 수 `g_ci`를 구해, `g_ci<μ_g`면 `ρ_i=1` 아니면 `ρ_v=2/n_D`로 오프라인 결정(식 10).
7. EKAR-BP: 원 하이퍼그래프를 modified PEG로 `T` 서브그래프로 확장 후 서브그래프별 `ρ_t`를 conditional gradient로 국소 최적화(식 12~14), `T=1`이면 TRW-BP.
8. 최적화: 서브그래프별 상호정보 `I_t` 기반 `min -ρ_t^T I_t`를 선형화·재귀 갱신, 모든 최적화는 오프라인(온라인 복잡도 불변).
9. 결과: EXIT 차트에서 EKAR-BP+PIC가 가장 넓은 tunnel, BER 곡선에서 CKAR/EKAR-BP가 표준 BP·URW-BP 능가(30 inner, 최대 3 outer iter), BER 1e-6 부근 error floor 존재.
10. 한계: HW 미설계, 시뮬만, MIMO IDD 특화, full-tanh SP-BP 기반, reweight가 오프라인 그래프 지식에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| reweighted BP 메시지 규칙(VN/CN) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | 개념상 같은 위치이나 Prime ECC는 min-sum, 논문은 full-tanh SP-BP |
| reweighting factor ρ (오프라인) | [decoder.cpp](../Prime_ECC_3.1_Claude) `Get_VNU_Table_Idx()` (`PARAM_LLR`) | LLR 테이블로 흡수 가능성은 있으나 사이클/하이퍼그래프 유도라 직접 대응 아님 |
| PIC-MMSE MIMO 검출기 | 대응 없음 | Prime ECC는 NAND 채널만, MIMO 검출 계층 없음 |
| 사이클 카운팅/하이퍼그래프 확장 | 대응 없음 | Prime ECC에 그래프 사이클 분석·PEG 확장 모듈 없음 |

적용 가치: **낮음** — MIMO IDD와 full-tanh SP-BP에 묶인 reweighting 기법으로, min-sum HW 기반 NAND Prime ECC에 떼어 쓰기 어렵고 HW 수치도 없음.
