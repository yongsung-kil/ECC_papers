### B. 알고리즘 요약 (10줄 내외)

1. 대상: 일반 binary LDPC 부호설계, BPSK/AWGN, rate-1/2 예제 중심(길이 502~20118), 복호는 표준 LLR belief propagation(max iter 80).
2. 문제: PEG 알고리즘은 girth 최대화엔 좋으나 복잡도가 `O(nm)`로 코드길이 `n`에 비례해 장부호 구성이 느림.
3. 핵심 아이디어: 소형 base matrix `Hb`(크기 `mb×nb`)만 PEG로 만들고, 이를 CRT로 큰 `H`(크기 `m×n`, `m=mb·L`, `n=nb·L`)로 확장.
4. 확장 규칙: `Hb`의 각 `bij=1`을 서로 다른 `L`개 행·열에 걸친 `L`개의 '1'로 매핑, 행·열 인덱스 `It(i)`,`Jt(j)`를 CRT 공식으로 계산.
5. CRT 공식: `It(i)=iBL+Σ rtp·Apmb·L̄p mod m` (식7), 열도 유사(식10). 서로소 moduli `L1..Ls`와 `mb,nb` 사용.
6. 핵심 정리(Thm 5.1): 확장 후 girth `gc ≥ gb` — CRT 확장이 girth를 줄이지 않음(모순 증명).
7. 최소거리(Thm 5.3): regular 코드에서 `dm(H) = dm(Hb)`로 base의 최소거리 보존.
8. 복잡도: 전체 구성이 `O(nb·mb)`로 base 크기에만 의존, CRT 계산비용은 무시가능 → 코드길이 `n`과 무관.
9. 결과: PEG-CRT 코드가 base PEG 코드(짧은 길이)보다 BER 우수, 동일길이 PEG 코드와 유사하되 훨씬 저복잡도; 저장비율 η=1/L로 최대 0.2~1%까지 절감; NNCS로 min-distance(49→880 등) 확인.
10. 한계: HW 미설계, 순수 코드구성법·이론 증명 중심, QC 구조 보장 없음(random-like PEG base), soft-read/NAND 무관.
