### B. 알고리즘 요약

1. URLLC(초저지연·초고신뢰) 요구에 맞춰 짧은 블록길이 코드를 대상으로, MacKay-Neal(MN) LDPC 구성에서 파생된 sparse parity random linear(SPRL) 코드를 (32,16) 예제로 제안.
2. 기존 MN 코드 구성은 parity check matrix가 full-rank가 아닐 수 있어 열 삭제로 rate가 저하되는 문제가 있었음; SPRL 구성은 행을 하나씩 rank가 증가하도록 추가해 열 삭제 없이 목표 rate를 유지.
3. 채널 모델: bad(B)/good(G) 2-state Markov 채널(버스트 오류 모델)로, State B에서는 매 심볼이 오류, State G에서는 오류 없음(전이확률 g, r로 정의).
4. 핵심 기법 1: GRAND-MO(noise guessing decoder) — 채널 통계(전이확률)로 노이즈 시퀀스를 확률 순으로 정렬해 순차적으로 codeword에 더하고 syndrome이 0이 될 때까지 탐색하는 hard-decision 디코더.
5. 핵심 기법 2: binary Matching Pursuit(MP) list decoder — syndrome 잔차(residual) `S = H·r̂`를 최소화하는 atom을 반복적으로 식별하는 sparse recovery 알고리즘에, dot product 동점(tie) 처리를 위한 list decoding을 추가.
6. 부수 기법: SPRL parity matrix에서 Tanner graph의 4-cycle을 열 재배치로 제거하는 수정법(단, rank는 유지) 제시; 4-cycle 유무가 GRAND/MP 성능에 영향 없음을 실험적으로 확인(sum-product와 달리 cycle에 둔감).
7. 복잡도: GRAND-MO는 O(n²) parity check를 기대 노이즈 시퀀스 수(Hamming distance의 절반 수준)만큼 반복, MP list decoder는 MP가 식별한 atom 수(≤ GRAND 대비 적음)로 한정되어 GRAND보다 계산량이 항상 작거나 같음(수식 (1)~(3)으로 상한 증명).
8. 별도 학습/최적화 절차 없음(코드 구성은 결정론적 랜덤 생성, 디코더는 non-learning 알고리즘).
9. 결과: 버스트가 심한 채널(g=0.4)에서는 GRAND-MO가 MP list decoder보다 BER/BLER 우수, 버스트가 적은 채널(g=0.9)에서는 두 디코더 성능이 근접(약 0.5dB 차이 이내 BLER 10⁻⁵ 달성).
10. 한계: 매우 짧은 (32,16) 코드 예제 1~2종만 검증, HW 구현 없이 시뮬레이션(BER/BLER)만 제시, AWGN 단독 채널이 아닌 2-state Markov 버스트 채널 전용 가정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPRL 부호 구성(MacKay-Neal 변형, full-rank 보장) | `ecc_top.cpp` `Load_PCM()` | 짧은 랜덤 sparse 부호 구성법으로 QC-LDPC 구조(base+circulant)와 무관, H-matrix 로드 단계에만 형식적으로 대응 |
| GRAND-MO / MP list decoder (syndrome 기반 노이즈 추정) | 대응 없음 | min-sum 기반 iterative C2V/V2C message passing이 아닌 syndrome 잔차 탐색 방식으로 `decoder.cpp` 구조와 이질적 |
| Tanner graph 4-cycle 제거(열 재배치) | `ecc_top.cpp` `Load_PCM()` | H-matrix 구조 조정이라는 점에서만 형식적 대응, 실질적 girth 최적화 기법은 아님 |

> 적용 가치: 낮음 — 버스트 채널·초단문 URLLC 특화 sparse random 부호와 syndrome 기반 code-agnostic 디코더로, NAND용 고rate binary QC-LDPC min-sum 디코더 구조와 부호설계·디코딩 알고리즘 모두 이질적.
