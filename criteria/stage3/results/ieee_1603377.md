### B. 알고리즘 요약

1. 시스템/코드 스펙: binary irregular (16200,7200) LDPC 코드(rate≈0.444), BPSK+AWGN 채널, check degree dist `ρ(x)=0.00006x²+0.14917x³+0.29851x⁴+0.44777x⁵+0.10449x⁶`, bit degree dist `λ(x)=0.00002+0.38803x+0.31344x²+0.29851x⁷`.
2. 풀려는 문제: 표준 min-sum(MS)은 BP 대비 하드웨어 친화적이나 irregular LDPC에서는 기존 1-D(check node만) 정규화/오프셋으로도 BP 대비 성능 열화가 큼.
3. 핵심 가정: check node뿐 아니라 bit node로 나가는 메시지도 노드 degree에 따라 다른 가중치를 받아야 한다는 직관(irregular 구조에서 degree 차이를 무시하면 손실).
4. 핵심 기법: 2-D normalized MS — horizontal step에서 `U=α_{dc(m)}·sgn(·)·min|V|`(식6), vertical step에서 `V=U_ch+β_{dv(n)}·ΣU`(식5/6)로 check·bit 양쪽에 degree별 정규화 계수 α_j(j=1..dc_max), β_j(j=1..dv_max) 적용.
5. 핵심 식 의미: 기존 1-D는 check node degree만 반영해 정규화하지만, 2-D는 bit node degree까지 반영해 irregular 코드 특유의 불균형 신뢰도 전파를 보정.
6. 확장: 2-D offset MS도 동일 컨셉으로 `U=sgn(·)·max(min|V|-σ_{dc(m)},0)`(check), `V=U_ch+sgn(SU)·max(|SU|-υ_{dv(n)},0)`(bit) 형태로 오프셋 계수 σ_j, υ_j 적용.
7. 부수 트릭: 최적 계수쌍 탐색은 brute force가 `dvmax·dcmax` 차원에서 intractable하므로, parallel differential optimization(진화 알고리즘)으로 density evolution 기반 threshold를 최소화(population `L=T·dvmax·dcmax`, mutation/test/stopping 절차).
8. 학습/최적화 절차: density evolution + parallel differential evolution(진화 세대 반복, 최적 계수 pair 갱신, threshold 수렴 시 종료) — 오프라인 1회 수행 후 계수 고정 사용.
9. 결과(16200,7200 코드, Imax=200): density evolution threshold(dB) — BP 0.77, MS 0.93, 1-D norm-MS 0.90, 2-D norm-MS/offset-MS 0.85(둘 다). 시뮬 WER에서 2-D 방식이 1-D 대비 약 0.3dB 개선, BP와 comparable하면서 error floor는 BP보다 낮음. 최적 α=(1.00,0.94,0.92,0.88,0.86), β=(1.00,1.00,0.91,0.83); σ=(0.00,0.26,0.31,0.35,0.37), υ=(0.00,0.00,0.20,0.31).
10. 한계: HW 구현/복잡도(게이트수·throughput)는 다루지 않는 순수 알고리즘+시뮬레이션 논문이며, 계수는 특정 degree distribution에 종속되어 코드 변경 시 재최적화 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Check node 2-D normalized/offset MS (degree별 α_j, σ_j) | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum CN 연산에 degree별 정규화·오프셋 계수를 추가 적용하는 형태로 직접 이식 가능 |
| Bit node(VN) degree별 정규화 β_j, 오프셋 υ_j | `decoder.cpp` `VN_Cal_HD()` 등 VN 처리부 | VN 메시지 갱신 시 degree별 계수 반영, 현재 min-sum 대비 추가 보정 레이어로 이식 가능 |
| Iteration별 LLR 테이블(관련 참조) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | degree별 계수를 iteration별 LLR threshold 테이블과 유사한 방식으로 저장·참조 가능 |
| Parallel differential optimization(오프라인 계수 탐색) | 대응 없음 | 코드 실행부가 아닌 설계 시 오프라인 최적화 절차로 코드베이스 내 대응 모듈 없음 |

> 적용 가치: 높음 — CN/VN 양쪽에 degree 종속 정규화·오프셋 계수를 추가하는 것은 기존 min-sum 구조를 유지하면서 성능(waterfall+error-floor)을 함께 개선하는 저비용 변형으로, `CNU_Update_New_Mag()`/VN 처리부에 계수 테이블만 추가하면 되는 낮은 이식 난이도.
