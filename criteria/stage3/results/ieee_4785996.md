### B. 알고리즘 요약

1. 시스템: 랜덤 생성 LDPC 부호 (150x250 parity check matrix, row weight 6, 4-cycle 제거), rate 대략 0.4.
2. 문제: 기존 BP(belief propagation) 디코더는 성능은 좋으나 iteration 수가 많고 parity check density 증가시 성능 붕괴, row/column interleaver 표현의 HW 복잡도 문제.
3. 핵심 가정: 각 비트의 확률 `p_i`가 독립이라는 가정 하에 Gallager's lemma로 짝수패리티 확률 `P_E = (1+Π(1-2p_i))/2`를 O(K) 연산으로 계산.
4. 핵심기법 1단(Parity Augmentation, PA): 매 반복에서 각 row의 비트 확률을 "projected probability"로 대체해 `P_E`가 항상 증가하도록 조정 - row 정보만 사용, column interleaver 불필요.
5. 핵심 성질: PA는 한 iteration당 최대 1개 비트만 "flip"하며 `P_E<0.5`일 때만 flip 발생(수렴 안정성 근거).
6. 핵심기법 2단(Parity Maximization, PM): `P_E`를 직접 최적화 목표로 gradient ascent 적용 - `J_m = P_{E,m} - Ω*Σδ_i^2`(정규화항 포함), channel LLR을 perturbation `δ_i`로 조정.
7. 채널 posterior 계산은 BPSK ±a 진폭 가정, `p_i = 1/(1+e^{-ρ(y_i+δ_i)})` (ρ=2a/σ²) 형태의 sigmoid.
8. 최적화 절차: gradient ascent(또는 line search, Newton's method)로 `δ_i` 갱신, PM은 본 논문에서 성능평가는 후속 논문으로 유보.
9. 결과: fixed iteration 수 제한 시 소수 iteration에서는 PA가 BP보다 우수하나 iteration 늘어나면 BP가 우세(단 큰 차이 아님); PA 계산시간은 iteration 수 증가에 따라 BP 대비 감소폭 커짐(속도 이점).
10. 한계: HW 미설계, 검증은 소규모 랜덤 매트릭스 시뮬뿐, row weight 증가(heavy H)시 PA는 BP보다 더 열화, QC-LDPC/protograph 등 구조화된 부호 미고려.
