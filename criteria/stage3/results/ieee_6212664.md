### B. 알고리즘 요약

1. 대상은 DVB-T2 systematic IRA-LDPC(정보-검사 HA + staircase parity HB), 주기성 `M=360`, code rate 3/5~5/6.
2. 층별(layered/TDMP) 복호를 IRA-LDPC에 직접 적용하면 QC 변환 시 다중 대각선을 가진 MSI 블록에서 message-updating conflict가 발생(동일 APP를 동시 갱신).
3. 기존 해법(충돌층 TPMP 복호 / PCM 수정 / 선택적 재계산)은 성능손실 또는 비HW친화적.
4. 핵심기법1 `PCM Rearrange`: 정수 인자 `H`(360의 약수)로 재정렬해 seed matrix `HAi*`(블록 `P=M/H`)로 변환, 식(4)(5)로 shift값 계산.
5. 재정렬 목적은 층간 data dependency 제거 + `H`개 identity 중 1개만 저장해 PCM 정보를 `1/H`로 축소(H=9 기준 600kbit→축소).
6. 핵심기법2 `Layer Division`: 남은 충돌층을 두 sub-layer로 분할(dark→white 순차 갱신)해 동일 APP 동시 갱신 회피, 병렬도 유지.
7. 분할 가능성은 블록크기 `P`와 두 대각선 거리 `d`로 판정(Fig.4 알고리즘), H=9·P≥40에서 3개 분할패턴이 전 충돌블록 커버.
8. 파이프라인: FIFO-delay+Bypass-delay 2사이클 지연을 bypass로 겹쳐 충돌층당 2 clock만 추가(rate 4/5 최악 1.2% redundancy).
9. 결과: rate 5/6 15iter에서 [8] 대비 BER 개선(conflict-free), SMIC 90nm 합성 gate 193k, 319MHz, air throughput 116Mbps@25iter.
10. 한계: DVB-T2 통신표준 특화, HD/soft 정보 명시 없음(무관), NAND/SSD 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 층별 복호 스케줄 / conflict 해결 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 base+circulant QC-LDPC라 MSI 다중대각 충돌 부재 - 개념적 참조뿐 |
| PCM 재정렬 / H-matrix 로드 | `ecc_top.cpp` `Load_PCM()` | DVB-T2 address table 기반 재정렬은 QC 고정 구조와 무관 |
| Min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()` | 논문은 Normalized Min-sum 사용 언급뿐, 신규 기여 없음 |

적용 가치: **낮음** — DVB-T2 IRA-LDPC의 층별 충돌은 Prime ECC의 QC-LDPC 구조엔 존재하지 않는 문제라 이식 대상 자체가 없음.
