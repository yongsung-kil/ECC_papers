### B. 알고리즘 요약 (10줄 내외)

1. 목표: 코드워드 내 비트를 소스 데이터 민감도별로 여러 "sensitivity class"로 나누고, 각 클래스가 다른 오류보호 수준(UEP)을 갖도록 하는 bit-regular / check-irregular LDPC 코드를 설계.
2. 배경 문제: 기존 LDPC 최적화는 수렴속도 최대화를 위해 check-degree를 2개 연속 degree로 제한(concentrated)하는 것이 표준이나, 이는 코드워드 전체에 균일한 보호만 제공.
3. 채널/모델: AWGN 채널, Gaussian approximation 기반 detailed density evolution(Kasai et al.의 joint degree distribution `π(b,d)` 사용).
4. 핵심 기법: check node degree 분포를 클래스별로 다르게 설계 — 특정 클래스 `Ck`의 평균 check 연결도 `ρ(Ck)`를 낮출수록 해당 클래스의 EXIT 수렴이 빨라짐(식 (3)~(6)).
5. 핵심 식 의미: `ρ(Ck)` (클래스 Ck의 평균 check-degree)를 비용함수로 삼아, 보호를 강화하고 싶은 클래스의 `ρ(Ck)`를 최소화하도록 check-degree를 재배치.
6. 확장/구현 기법: 이론적 degree 분포를 실제 부호에 구현하기 위해 "code pruning"(모 코드에서 일부 열 삭제)을 사용 — 하나의 mother code/decoder로 다양한 UEP 부호를 생성.
7. 부수 트릭: pruning 시 girth 감소 방지, stability/convergence/code-rate 제약을 만족하는 열만 선택하는 순차적(sequential) 프루닝 절차 제시.
8. 최적화 절차: 클래스별로 가장 보호할 비트노드를 선정하고 연결된 check-degree를 목표치 근방으로 낮추는 반복적 탐색(GA/DE 아님, 규칙기반 탐색).
9. 결과: (3,6) regular mother code (`N0=2000, R0=1/2`)를 pruning해 `N1=1500, R1=1/3` 서브코드 생성, 3개 클래스로 최적화 — 30회 반복 후 BER 비교 시 non-concentrated 코드가 최우선 보호 클래스에서 더 빠른 수렴을 보이나 최하위 클래스는 오히려 열화.
10. 한계: check-irregularity 확대는 전체 코드의 convergence threshold(성능)를 희생시키는 trade-off이며, 시뮬레이션만 수행(HW 미설계), 목적 자체가 NAND의 균일 데이터 보호와 무관(멀티미디어 스트림의 헤더/압축데이터 차등보호용).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Check-irregular degree 분포 설계 | `ecc_top.cpp` `Load_PCM()` | 부호구조 변경이나 UEP 목적이 NAND ECC와 무관 |
| Code pruning(mother code → subcode) | 대응 없음 | Prime ECC는 고정 QC-LDPC H-matrix 사용, 코드워드 내 비트별 차등보호 개념 없음 |

> 적용 가치: 낮음 — UEP는 멀티미디어 계층적 데이터 보호가 목적이며 NAND의 균일 오류정정 요구와 부합하지 않고, 전체 코드 성능(threshold)을 희생하는 구조라 Prime ECC 목표(waterfall/error-floor 개선)와 상충.
