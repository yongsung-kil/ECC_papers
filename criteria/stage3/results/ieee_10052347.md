### B. 알고리즘 요약

1. 대상: `(N,K)(dv,dc)` regular binary LDPC, 실험 코드 `dv3R050N1296` = (dv=3, dc=6, N=1296, R=1/2), BSC(crossover `α`) 채널, hard-decision만 사용.
2. 문제: 기존 BF(GDBF/PGDBF/TRGDBF)는 flipping threshold를 위해 global maximum 탐색이 필요해 critical path가 길고, ISBF/LISBF는 critical path는 줄였으나 수렴·trapping set 극복이 부족.
3. 기반: ISBF/LISBF는 이전 iteration 에너지값으로 threshold를 예측(`Emax_t = max E_{k,t-1}`, `E_k = dv+1-E_k` 업데이트)해 threshold와 energy를 병렬 계산; LISBF는 column-layered + double-maximum.
4. 핵심 기법(MLISBF, 3종 전략): (1) normal flipping - `E_k>Th`인 고에너지 비트를 높은 확률 `p0`로 플립(기존은 놓침), `S_k>=Th`는 `p1`, `E_k>=Th`는 `p2`(p1>=p2).
5. 근거: `R_{Th+1}≈1`(에너지 Th+1 비트는 거의 오류)라 고확률 플립 필요; `R_{Th-1}`은 sw 감소 시 증가.
6. 전략 (2) strict flipping - syndrome weight `sw < SW`일 때 트리거, `Th`를 `Th-1`(단 >=2)로 낮춰 더 많은 비트 플립해 수렴 가속.
7. 전략 (3) perturbation flipping - 특정 iteration `t∈Tper`에서 낮은 threshold `Th_per`와 낮은 확률 `p3`로 섭동을 주어 trapping set 탈출.
8. HW: layered(컬럼별) 파이프라인 - iteration당 클록수 `Nc=L`, layer 간 조합논리/레지스터 다중화, VNU/CNU 단순, RPG는 CRC 회로로 난수 생성, forward-backward로 layer threshold 계산.
9. 결과: `dv3R050N1296`에서 GDBF/PGDBF/TRGDBF/ISBF/LISBF 대비 최고 FER, 낮은 평균 iteration; TSMC 90nm 합성 area 0.141mm2(ISBF의 42.2%), throughput 3.13Gbps, 면적효율 552.5Gbps/mm2(ISBF의 101.1%).
10. 한계: BSC hard-decision 전용(soft-info/LLR 배제가 설계 전제), 단일 rate=0.5 regular 코드만 검증, 통신/스토리지 일반 대상으로 NAND 실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flipping 디코딩 알고리즘 전체 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 디코더 이터레이션 루프 위치는 같으나 BF 계열은 min-sum과 별개 알고리즘 |
| perturbation/strict flipping (trapping set/error-floor) | 대응 없음 | Prime ECC는 min-sum 기반, BF post-processing 대응 함수 없음 |
| syndrome weight 기반 조기종료 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | 조기종료 개념은 유사하나 Prime ECC는 CRC 기반, syndrome-weight 방식 미사용 |

적용 가치: 낮음 - hard-decision BF 계열 디코더로 Prime ECC의 min-sum/soft-read 구조와 알고리즘 계열이 달라 직접 이식 불가, perturbation 아이디어만 개념 참고 여지.
