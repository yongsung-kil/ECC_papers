### B. 알고리즘 요약

1. 대상 부호는 regular `(1024,512) (3,6)` LDPC, BPSK/AWGN, 완전병렬 stochastic 디코더(SPA 대신 Bernoulli 비트스트림으로 확률 표현).
2. 문제: 비동기 stochastic 디코딩은 throughput/전력 이점이 크나 "lock-up"으로 수렴 전 정지 → error floor 발생.
3. 채널/노드 모델: VAR는 AND 기반 식`(1)`+TFM 재랜덤화, CHK는 XOR 식`(2)`, 각 노드/interleaver에 배선지연 `TDT` 부여.
4. 핵심 기법: WDD(wire-delay dependent) 비동기 업데이트 - 각 노드 업데이트 주기를 연결된 배선지연들의 평균으로 결정 `TXi = α·avg(TDTi,0..dv-1)`.
5. 식 의미: clockless 디코딩(입력변화 시 동작)을 흉내내 노드별 업데이트 타이밍에 변동을 주어 스위칭 활동↑ → lock-up 확률↓.
6. 확장 기법: WDI↔WDD 결합 스케줄 - 총 디코딩시간을 `γ` 비율로 나눠 두 방식 교대 적용, 스위칭 활동을 더 늘림.
7. 구현 디테일: TFM 메모리 5bit, noise-dependent scaling 1.5, 데이터전송은 request-ack 핸드셰이킹(min~max TDT).
8. 최적화: `α`, `γ`를 Eb/N0별 시뮬로 튜닝(Eb/N0=3.4dB에서 α=2/3, γ=1/10 최적).
9. 결과: BER=10⁻⁶에서 conventional 대비 WDI 0.2dB, WDI→WDD 0.25dB 이득, clockless 대비 동등 BER(최대 디코딩시간 25%↑), 비동기가 동기 대비 6.4~9.8x 낮은 latency.
10. 한계: HW 미설계(90nm timing 모델 시뮬만), stochastic 전용 패러다임, gatecount/throughput 수치 본문 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stochastic VAR/CHK 노드 연산 | 대응 없음 | Bernoulli 비트스트림 기반, min-sum CN/VN과 무관 |
| 비동기 업데이트 스케줄링 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 개념상 스케줄 레이어이나 stochastic 타이밍 모델이라 이식 불가 |
| error floor 저감 | 대응 없음 | lock-up은 stochastic 고유 현상, 우리 min-sum 디코더에 없음 |
| 배선지연/interleaver 라우팅 | 대응 없음 | HW 라우팅 timing 모델로 z=32 QC 구조와 무관 |

적용 가치: **낮음** — stochastic 디코딩 전용 lock-up/error-floor 기법으로 Prime ECC의 binary min-sum QC-LDPC 디코더와 알고리즘 패러다임이 근본적으로 다르다.
