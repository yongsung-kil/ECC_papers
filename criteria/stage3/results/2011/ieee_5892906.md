### B. 알고리즘 요약

1. 시스템: 위성/항공 원격탐사 데이터를 error-prone 무선링크로 지상국 전송, DVB-S2 등의 LDPC를 GPU로 복호.
2. 풀려는 문제: ASIC/FPGA는 코드 rate/length/설계파라미터 유연성이 부족 → 지상국에 유연·저비용 고성능 복호 플랫폼으로 GPU 활용.
3. 부호: regular LDPC(row weight `dc`, col weight `dv` 고정), 실험 3개 행렬 중 대표는 4000×8000 (rate ~0.5).
4. 디코더 알고리즘: log-SPA(log sum-product) 5단계 - LLR 초기화, horizontal(CN, `sign×amplitude`), vertical(BN), iteration, hard decision.
5. 핵심 식: CN 갱신은 `sign(L)`의 곱과 진폭 함수 합, BN 갱신은 인접 CN 메시지 합 + 채널 LLR (표준 message passing).
6. GPU 매핑: H행렬의 1(edge)을 GPU thread에 대응, block당 thread 수는 row/col edge 수의 LCM 배수 `n_t`로 결정, block 수 = edge수/n_t.
7. 핵심 트릭: 비동기 데이터전송(ADT) - CPU↔GPU 데이터전송(DS/DR)과 kernel 실행(GKE)을 오버랩, 동기전송(SDT) 대비 최대 2배 이득(codeword 다수 시).
8. 정밀도: 32bit single-precision float(참조 [18]의 8bit min-sum보다 정정능력↑ 주장), CN당 BN들은 같은 block에 배치(동기화).
9. 결과(수치): Tesla C1060(240 core)에서 4000×8000·10 iteration 복호 1.808ms vs CPU(Xeon, double) 490.504ms → **271배** 가속, 행렬 클수록 가속 배율↑.
10. 한계: 알고리즘 신규성 없음(표준 log-SPA), HW(ASIC/FPGA) 설계 아님, throughput(Gbps)·gate 수치 없음, GPU float 구현이라 임베디드 NAND 컨트롤러와 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| log-SPA 디코딩 루프 | decoder.cpp LDPC_Decoder() | 반복 복호 루프는 대응하나 우리는 min-sum(HW), 논문은 full-SPA(GPU) |
| CN horizontal 갱신 | decoder.cpp C2V_Cal(), CNU_Update_New_Mag() | sign×amplitude는 개념 대응하나 tanh 기반 SPA는 우리 min-sum과 이질 |
| BN vertical 갱신 | decoder.cpp VN_Cal_HD() 등 | message passing 대응, 그러나 32bit float |
| GPU thread/block 매핑, ADT | 대응 없음 | GPU 병렬화·데이터전송 최적화로 HW(z=32, HCU) 구조와 무관 |
```

적용 가치: **낮음** — 표준 log-SPA를 GPU로 가속한 소프트웨어 구현 논문으로, 알고리즘 신규성이 없고 full-SPA/float/GPU 전제가 NAND 컨트롤러의 min-sum·고정소수·HW 병렬 구조와 맞지 않음.
