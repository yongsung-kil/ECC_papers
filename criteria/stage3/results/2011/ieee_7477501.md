> 총평: 10GBase-T PHY 트랜시버 소개 슬라이드로, LDPC error-floor를 온칩 trapping-set LUT로 우회하는 아이디어는 흥미로우나 상세·수치가 없고 NAND 이식성은 낮음.

### B. 알고리즘 요약

1. Aquantia 90nm/40nm 10GBase-T Ethernet PHY 트랜시버로, FEC로 LDPC 인코더/디코더를 내장(802.3an급, twisted-pair 채널).
2. 채널: 45dB+ insertion loss twisted-pair, ISI·echo·NEXT/FEXT·alien crosstalk·RFI 등 다중 잡음을 필터·canceller로 제거 후 LDPC로 코딩 이득 확보.
3. LDPC는 iterative 디코더 아키텍처로 ~10dB 코딩 이득 제공(BER 1e-3 → 1e-12 목표).
4. 디코더는 Max ~10 iteration으로 설계되나 평균 ~3 iteration이며, 99.9%+ 코드워드는 3 iteration 내 수렴.
5. `LDPC Time Extension`: 10 iteration용 고클럭 대신 half-rate(5 iteration)로 돌리고 초과 2 iteration은 FIFO로 흡수 → 낮은 Vdd로 타이밍 충족(전력 절감).
6. 메시지 해상도 7~8bit·iteration 8~10에서 코딩 이득 최대이며, 저해상도/저iteration은 이득 제한.
7. Error-floor 원인은 `Trapping Set`(LDPC가 정정 못하는 codeword 집합), BER 1e-12 이하에서 발현.
8. 핵심 대책: 약 100개 TS codeword를 사전 식별·온칩 저장하고, 수신 시 LDPC와 분리된 LUT로 검출·정정 → error-floor 제거.
9. 복잡도: 2048 Equality + 384 Parity 노드, Equality degree 6 / Parity degree 32, 양방향 8bit 메시지 → ~200K 배선(시리얼/차동 전송으로 12K로 축소).
10. 한계: 발표 슬라이드라 부호 스펙·gate count·throughput 수치가 없고, 채널·구조가 twisted-pair Ethernet 특화라 NAND 이식 근거 부족.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC iterative 디코더 | decoder.cpp LDPC_Decoder() | Min-Sum 이터레이션 루프와 대응하나 알고리즘 상세 없음 |
| Trapping-set LUT 정정 | 대응 없음 | error-floor 후처리 개념이나 우리 구조엔 별도 LUT 없음 (corner.cpp 수렴실패 추적과 유사) |
| iteration FIFO/클럭 완충 | 대응 없음 | HW 타이밍 최적화(전력)로 알고리즘 무관 |
| 메시지 해상도(양자화) | decoder.cpp VN_Cal_HD() 등, ecc_data.h PARAM_LLR | 7~8bit 해상도 언급, 우리 양자화 테이블과 개념 대응 |
```

적용 가치: **낮음** — 슬라이드형 개요라 이식할 구체 알고리즘·수치가 없고, 유일한 신규 아이디어(TS codeword 온칩 LUT 정정)도 상세 미공개이며 Ethernet 특화다.
