### B. 알고리즘 요약

1. 시스템: convolutional(CC)/turbo(CTC)/LDPC 3종을 하나의 VLIW-SIMD ASIP로 복호. LDPC는 Wimax rate `3/4`, 576 info-bit 부호.
2. 문제: LTE/Wimax 등은 여러 채널부호를 요구 → 전용 IP block 대신 multi-mode 디코더 통합이 면적·전력상 유리한지 정량 비교 필요.
3. 통합 기법 3계층: memory sharing(고수준) / functional-unit(FU) merging(중간) / algorithmic-operation merging(ACS 같은 커널 공유, 저수준).
4. 아키텍처: core당 4 PE(SIMD), 각 PE는 Gamma(branch metric/extrinsic 감산)·Alphabeta(state metric/min-search)·Lambda(LLR)·DALU(extrinsic 재결합) FU 구성.
5. LDPC 복호: layered offset-min-sum(`[8]` Chen et al.). interconnect는 4x4 crossbar + barrel shifter로 QC 부분행렬 circular permutation 정렬.
6. 파생: 하나의 multi-mode "mother"에서 CC/turbo/LDPC 전용 single-mode 디코더를 유도해 동일 합성조건으로 공정 비교.
7. FU 역할 재사용: 같은 Gamma/Alphabeta FU가 CC/CTC에선 trellis metric, LDPC에선 extrinsic 감산·min-sum에 쓰임(표 1).
8. 합성: Synopsys DC, TSMC 65nm worst-case, 200MHz. throughput 고정이라 면적·전력만으로 통합 타당성 판정.
9. 결과: turbo+viterbi 통합 data path 오버헤드 10%, 3종 통합 시 단독 합 대비 data path 26% 절감. 그러나 전력은 Viterbi +116%, LDPC +43%, turbo +19% 증가(유휴 FU·복잡 FU 원인).
10. 한계: 전력 penalty 커서 저전력 환경엔 부적합(기지국급만 타당). LDPC 정정성능·부호설계 기여 없음, soft-decision 무선 세팅.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC layered offset-min-sum 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC가 이미 보유한 min-sum과 중복, 신규 기여 없음 |
| QC 부분행렬 barrel-shift permutation | 디코더 HW 모델(z=32 row 병렬) | QC circulant 정렬 개념만 공유, ASIP crossbar 구조는 무관 |
| multi-mode(turbo/Viterbi 통합) | 대응 없음 | Prime ECC는 단일 binary QC-LDPC 전용, turbo/Viterbi 미사용 |

적용 가치: **낮음** — LDPC 부분이 표준 offset-min-sum layered 복호 재사용에 불과하고 핵심 기여는 이종 디코더(turbo/Viterbi) 통합의 면적·전력 tradeoff라, 단일-부호 NAND QC-LDPC min-sum 엔진(Prime ECC)에 떼어 쓸 요소가 없음.
