### B. 알고리즘 요약

1. 대상: WiMax(802.16e)/Wi-Fi(802.11n) QC-LDPC, 부호길이 2304-bit, rate 1/2~5/6, 총 126개 모드 지원 재구성형 디코더.
2. 문제: BW 시스템에서 LDPC 디코더가 큰 칩 면적을 차지 → 프로그래머블·저전력·소면적 요구.
3. 복호 알고리즘: original BP 대신 Modified Min-Sum(`MS`[9]) + layered scheduling으로 수렴속도 2배 가속.
4. 아키텍처: SPI 제어인터페이스, base matrix 저장 CMM, 채널 LLR/hard-decision I/O 버퍼, PMM(posterior)·EMM(extrinsic) 분할 메모리, overlapping PU, message exchange network.
5. PU 구성: check node용 min-operation(`MOP`)과 bit node용 sum-operation(`SOP`)을 인접 두 block layer 간 완전 overlap.
6. 면적 트릭: min-operation을 register feedback(serial scanning)으로 folding, [9]의 2-cycle pipeline stall을 process 재정렬로 제거해 MOP/SOP cycle 일치 → seamless overlap.
7. 메모리 최적화 1: layered decoding에서 인접 두 layer의 non-zero entry 위치 중복이 많아, PMM write-back 없이 다음 layer로 직접 전달해 접근 횟수 축소(최적 layer concatenation 순서 탐색).
8. 메모리 최적화 2: 짧은 부호길이/고rate 모드에서 미사용 메모리를 standby로. PMM을 2 group×8 bank, I/O 12 bank, EMM 34 bank로 medium-grain 분할(면적 5% overhead로 에너지 절감).
9. 결과: SMIC 0.13μm, 4.32mm2(core 2.97), 145MHz/1.2V에서 170mW, 302Mb/s, 52 pJ/bit/iter, max.iter=10. 기존[6] 대비 core area 30%↓, throughput 2~3배, EE 50~75%↑.
10. 한계: WiMax/Wi-Fi 표준 특화 부호에 고정, 연판정 bit 수·Modified MS scaling factor 등 알고리즘 세부 미기재, error-floor 언급 없음.
