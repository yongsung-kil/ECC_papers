### B. 알고리즘 요약

1. 시스템: 위성영상 다운링크용 재구성형 LDPC 복호 플랫폼, regular LDPC H행렬 A/B/C(408×816, 2000×4000, 4000×8000, rate≈`1/2`), NVIDIA Tesla C1060(240 core) + CUDA.
2. 문제: 통신표준마다 다른 부호를 빠르고 유연하게 복호할 개발 플랫폼 필요(ASIC/FPGA는 사전정의 부호에만 효율적, 일반 부호 지원 부족).
3. 모델: AWGN 채널 가정, log-SPA(log-sum-product) iterative message-passing 표준 복호.
4. 핵심기법 1단(알고리즘): 교과서 log-SPA — 초기 LLR(식1), horizontal(CN) 갱신 `L(v_mn)`(식2, sign·amplitude 분리 tanh), vertical(BN) 갱신 `L(q_nm)`(식3), posterior LLR(식4), hard decision(식5).
5. 핵심 식: CN 연산에서 `2 tanh^-1`/`log tanh` 형태의 정확 SPA를 32-bit single-precision float로 계산(근사 아님).
6. 핵심기법 2단(GPU 매핑): 한 CN에 연결된 모든 BN을 같은 block에 배치(block간 비동기 문제 회피), thread=`L(v_mn)`/`L(q_nm)` 원소 단위 병렬.
7. 구현 디테일: global memory→shared memory 버퍼링, coalesced memory access 정렬(Table1의 block/thread↔H행렬 매핑), asynchronous data transfer(ADT)로 host-device 전송 은닉.
8. 학습/최적화: 없음(부호설계·복호알고리즘 개선 없음, 순수 구현 가속).
9. 결과: 4000×8000 행렬·10 iteration에서 GPU 1.808 ms vs CPU(double) 490.504 ms → 271x 가속, H행렬 클수록 speedup↑.
10. 한계: HW(RTL) 설계 아님(GPU SW), 32-bit float라 HW 양자화·면적 무관, throughput(Mbps/Gbps) 수치는 본문에 명시 없음(선행연구 [7] 109.1Mbps 인용뿐).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| log-SPA CN/BN 갱신 (식2,3) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `C2V_Cal()` | 알고리즘 자체는 대응되나 full-tanh SPA라 Prime ECC의 min-sum과 다름(대응 없음에 가까움) |
| GPU block/thread 병렬 매핑·shared memory | 대응 없음 | Prime ECC는 z=32 row 병렬 HW 모델이라 GPU SW 병렬화와 구조 상이 |
| coalesced memory access / ADT 버퍼링 | 대응 없음 | HW 메모리/라우팅(GT, HCU)과 개념 계층이 다름 |

적용 가치: **낮음** — GPU/CUDA 소프트웨어 가속 플랫폼으로, NAND 컨트롤러 binary QC-LDPC의 RTL ECC(Prime ECC) 이식과 계층·목적이 다르며 부호설계·복호알고리즘 신규 기여도 없음.
