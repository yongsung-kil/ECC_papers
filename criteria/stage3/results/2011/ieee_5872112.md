### B. 알고리즘 요약

1. 대상 시스템은 LTE/WiMAX로, WiMAX는 rate 1/2 memory-6 tail-biting convolutional code(FCH payload 12/24bit), LTE는 rate 1/3 turbo code를 사용한다.
2. 문제: convolutional/turbo와 LDPC가 나란히 쓰이나 Viterbi/MAP과 BP는 서로 다른 HW 구조를 요구 → 통합 디코더 부재.
3. 핵심 아이디어: convolutional·turbo 코드의 generator 행렬 `G`로부터 parity-check 행렬 `H`를 유도하여 Tanner graph로 표현.
4. tail-biting convolutional code의 `H`는 impulse response `g(j)`로 구성된 `Pi` 블록의 end-round-shift 형태 (식 7,8).
5. 얻어진 `H`는 행/열당 non-zero 개수가 일정치 않은 **irregular LDPC와 유사** → degree distribution `λ(x),ρ(x)`로 기술.
6. turbo code의 `H`는 두 RSC 부호 `H1,H2`와 interleaver 행렬 `M`으로 `Hturbo=[H2 H1 0; H2M^T 0 H1]` (식 13) 형태로 구성.
7. 이렇게 구한 `H`에 LDPC용 표준 BP(belief propagation)를 직접 적용하여 복호 (flooding schedule).
8. 최대 iteration 30, AWGN 채널, ML-Viterbi/MAP/SOVA와 BER/FER 비교.
9. 결과: convolutional은 ML-Viterbi 대비 FER 1.85dB 이내 손실, turbo는 MAP/SOVA 대비 BER 10^-2에서 약 1.7dB 열화. 대신 BP의 구현 복잡도가 낮음(edge 수에 선형).
10. 한계: HW 미설계, non-zero 분포가 충분히 random하지 않고 short cycle 다수로 성능 열화, 통합 아키텍처는 향후 과제로만 제시.
