> 총평: multi-mode QC-LDPC에서 유연 permuter network를 없애기 위해 DFU 수를 확장인자들의 GCD(=4)로 잡고 stride 스케줄링으로 메모리를 그룹 단위로 배치한 면적효율 HW 논문. 메모리 배치/스케줄링 아이디어는 참고 가치 있으나 부호는 NAND용 아님.

### B. 알고리즘 요약

1. 시스템: IEEE 802.16e WiMAX QC-LDPC, 가변 rate(1/2~5/6)·length(576~2304), 확장인자 24~96(step 4). PCM은 cyclic-shift identity/zero submatrix 조합.
2. 문제: multi-mode 지원을 위해 기존 partially-parallel 디코더는 DFU 수를 최대 확장인자(96)만큼 두고 유연 permuter(Benes/barrel shifter, 입력 128이면 1664 MUX)를 써서 면적·전력이 큼.
3. 핵심 관찰: WiMAX throughput 요구가 30Mbps로 낮아 극단적 병렬화 불필요 → 병렬도(DFU 수)를 낮춰 permuter 크기까지 함께 축소 가능.
4. 핵심기법 1(DFU=GCD): DFU 수 `p`를 가능한 확장인자들의 GCD인 `4`로 설정 → HW 이용률 극대화, permuter 입출력 수 축소. min-sum 기반 layered decoding, 4-stage 파이프라인(read/pre-DFU/DFU/WB).
5. 핵심기법 2(stride scheduling): 한 layer의 check node를 순차 대신 `p`개 subset으로 분할, DFUi가 subset i를 담당 → 동시 처리되는 두 check node index 간격이 항상 `n/p`(=stride).
6. 메모리 배치: APP 메모리 i번째 entry에 row `i, i+n/p, i+2n/p, i+3n/p`를 저장(그룹=modular). cyclic shift 양과 무관하게 동시 접근 데이터가 한 entry에 모임 → 1 cycle 접근.
7. permuter 축소: 그룹 크기 4에 맞춰 permuter를 2-stage barrel shifter(32 MUX)로 구현 — 순차 스케줄(입력 8, 3-stage) 대비 복잡도 1/3.
8. single-port 메모리 트릭: pipeline이 read/write 2회 접근하나, entry를 2배(8 data)로 키우고 read/write buffer로 interleave해 2-port 대신 single-port 메모리 사용.
9. 결과: 0.18µm CMOS 합성, 312.5MHz(3.2ns), 10 iter, gate 49K, 메모리 54,144bit, 최대 throughput 116Mbps(rate 5/6, 최대길이). APP 7bit / C2V·V2C 5bit 양자화. 최소 면적 아키텍처(Table I).
10. 한계: WiMAX multi-mode 가변부호 전용, throughput 낮음(통신 표준 요구 충족 목적). soft-read/error-floor 무관, HW는 합성 단계까지(실측 아님).
