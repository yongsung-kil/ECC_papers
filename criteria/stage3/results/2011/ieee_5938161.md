### B. 알고리즘 요약

1. 시스템: CMMB(중국 디지털 모바일 TV)용 `(9216,4608)` `(3,6)` regular LDPC, code rate 1/2·3/4, partially parallel layered 디코더(병렬도 18).
2. 문제: layered 디코더는 수렴은 빠르나 정확한 HC(parity) checking이 어려워 불필요한 반복이 발생, throughput·전력 손실.
3. 기존 조기종료([5][6])는 sub-iteration HC로 일부 VN/서브반복을 정지하나, LLR hard-decision이 완전 수렴 전 변동하여 오판→BER error-floor 발생.
4. 핵심 기법: 매 sub-iteration 후 갱신 LLR `Λ` 부호와 직전 LLR 부호를 전 VN에 대해 비교, 같으면 `flag_early_ter++`, 다르면 0으로 리셋.
5. 종료 조건: `flag_early_ter == φ`(=PCM row 수/병렬도, serial degree)일 때 반복 종료. 1비트 비교 연산 수 = 병렬도×row weight로 HW 부담 작음.
6. layered min-sum 갱신식: `L = Λ - R`(V2C), `R = α·∏sign·min|L|`(C2V, α=정규화), `Λ = L + R`(LLR 갱신) — 매 layer 갱신으로 빠른 수렴.
7. 비균일 양자화: 6비트, `|y|<1.75`는 step 2^-3 균일, 이상은 2^-2/2^-1로 확대 → (7:3) 균일과 근사 성능, 16.7% 메모리 절감.
8. EX 메모리: row 내 EX 메시지를 1개 mixed message로 압축(min/second-min/위치), 병렬도(18)만큼의 single-port 256x26 메모리 → 27.8% 절감.
9. 결과: 평균 반복 8.4021(1.4~2.2dB)로 16% 감소, effective throughput 267Mbps(1/2)/401Mbps(3/4), 메모리 171Kbit, SMIC 130nm 구현, [5] 대비 error-floor 제거.
10. 한계: 특정 CMMB regular 코드·layered 아키텍처 전제, HD-only(부호비교) 판정이라 soft magnitude 정보 미활용, 자체 gatecount 미제시.

| layered min-sum C2V/V2C 갱신 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 자체는 이미 보유, layered 교대 스케줄만 상이 |
