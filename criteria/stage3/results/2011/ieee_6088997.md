### B. 알고리즘 요약

1. 시스템: QC-LDPC(base matrix Hb + circulant shift factor `SF`, z-factor extension), mother code `(2016,1008)`, sub-matrix C 크기 252×252, rate 0.5~0.8.
2. 문제: rate-compatible FEC는 rate마다 별도 encoder/generator를 저장하면 면적·메모리 폭증. conventional puncturing은 punctured bit LLR=0으로 잡음 주입 → 고rate에서 trapping-set·성능 저하.
3. 핵심 가정: 복호기는 punctured 위치를 안다. AWGN+BPSK, 복호는 normalized Min-Sum(정규화계수 0.75, 최대 10 iter).
4. 핵심 기법 1: MacKay ALT(almost-lower-triangular) forward-substitution 인코딩을 QC-LDPC용으로 수정. circulant 대각이 all-ones가 아닌 문제를 식 `(6)~(8)`로 행 인덱스 `r`을 `SF` 기반 permute해 MacKay 제약 충족, 파생 parity를 z개 병렬 계산.
5. 핵심 식: `P = [P1 P2]`, `P1`은 sub-matrix A forward-substitution(식 6), `P2 = C^-1·B·P1^T`(식 5). G 행렬 대신 sparse H와 작은 `C^-1`만 저장 → 메모리·복잡도 절감.
6. 핵심 기법 2: matrix puncturing — bit 스킵 대신 H에서 l쌍의 행/열을 제거해 `CRl = (n-m)/(n-l)` rate 생성. 최대 rate `CRmax = (n-m)/(n-m+sc)` = 0.8.
7. 부호 구성: C(가역, 열당 ace≥2) + B(4-cycle 최소화, 최소거리 최대화) 구성 후 행/열을 순차 추가해 MacKay 구조·최소거리를 보존하며 rate 범위 확장. 이후 z-extension.
8. HW: encoder core 3-stage, 기본 연산은 row vector-matrix multiplication(rVMM). A/B는 monomial circulant → barrel shifter/shift register로 shift 구현, A·B ROM에 인덱스+shift factor 압축 저장. puncturing은 해당 행/열 주소 생성을 생략만 하면 됨.
9. 결과(시뮬): 제안 puncturing이 prior art `[15]`(Vellambi-Fekri) 대비 rate 0.6/0.7/0.75에서 각각 >0.8/2/3dB coding gain. rate 0.8에서 prior art는 error floor, 제안은 mother-rate와 0.8dB 이내.
10. 한계: HW는 아키텍처 제시뿐 gatecount/throughput 수치 없음, 복호 성능은 시뮬만. MacKay 구조 제약이 mother-rate 부호 성능을 비구조 부호보다 떨어뜨릴 수 있음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| rate-compatible H/부호 구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 고정 구조라 puncturing식 rate 가변 도입은 난이도 높음 |
| matrix puncturing(행/열 제거) | 대응 없음 | Prime ECC는 punctured column(punct_col_num=2) 개념은 있으나 rate 가변 puncturing 미지원 |
| normalized Min-Sum 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()` | 복호는 검증용으로만 사용, Prime ECC min-sum과 유사 |

적용 가치: **중간** — NAND에서 여러 rate를 하나의 부호로 지원하려는 경우 modified-MacKay 인코딩과 구조 보존 puncturing은 참고 가치가 있으나, Prime ECC의 dual-diagonal 인코더·고정 QC 부호를 교체·재검증해야 해 이식 난이도가 높고 rate-compatibility 요구가 없으면 실익이 제한적.
