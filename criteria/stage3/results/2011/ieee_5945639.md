> 총평: min-sum 행처리를 반으로 분할해 배선·병렬화를 개선하는 디코더 HW 지향 기법이나 0.3~0.5dB 성능 손실을 감수하며, Prime ECC의 기존 병렬화(z=32/GT/HCU)와 중복 가능성이 있어 이식성 중.

### B. 알고리즘 요약

1. 시스템: BPSK/AWGN 채널, irregular LDPC. 예시 부호 rate `R=0.75`(N=1536, Hs=(6,18)), `R=0.84`(N=1520 및 2014, Hs=(6,32)), `Imax=7~15`. Hp는 WiMax mobile 채택 구조, Hs는 QC 구조 예시.
2. 문제: 고 row-weight LDPC의 표준 SPA/min-sum 디코더는 check node↔variable node 간 배선 상호연결이 복잡하고 행처리기가 커짐.
3. 기법 배경: 표준 SPA는 row processing `α_ij = Πsign(β)·φ(Σφ(β))`, `φ(x)=-log(tanh(x/2))`; min-sum은 magnitude를 `min|β|`으로 근사하고 scale `s≤1` 정규화.
4. 핵심 기법(Split-Row): parity check matrix를 열방향으로 두 반쪽 `H^Split`으로 나눠 행처리를 두 개의 거의 독립적인 half-processor로 분리. 두 반쪽 사이엔 방향당 sign 1-wire만 연결.
5. 핵심 식: half 메시지만 사용해 `α_ij^Split = Πsign(β)·φ(Σ_{VSplit}φ(β))`. `φ`가 양·감소함수이므로 `α^Split ≥ α`(근사가 magnitude 과대추정).
6. 보정: scale factor `S<1`로 `α^Split`을 정규화(`α^Split=S·Πsign·φ(...)`)해 과대추정 완화. `S`는 rate·SNR별 실험 결정(예 S=0.35).
7. 확장: 동일 분할을 Min-Sum에도 적용 가능. 이 논문은 regular용 기존 Split-Row[5]를 irregular로 확장한 것.
8. 이점: 행처리 병렬성 2배, row processor당 메모리 접근↓, 배선 상호연결↓ → 디코더가 더 작고 빠르고 저전력(정성적 서술).
9. 결과: MS 대비 BER 성능 손실 `N=1536`에서 ~0.5dB, `N=1520`에서 ~0.35dB, `N=2014`에서 ~0.3dB. row weight·code length 높을수록 손실 폭 작음.
10. 한계: MATLAB 시뮬만(gatecount/throughput/합성 수치 없음), 정정능력 향상이 아니라 복잡도-성능 트레이드오프, scale S는 실험 튜닝 필요.
