### B. 알고리즘 요약

1. 문제: LDPC 성능은 Tanner graph의 girth(최단 cycle 길이)에 좌우되나, 많은 부호가 girth 제약 없이 설계됨 → 주어진 부호의 girth를 사후 증가시키는 효율적 방법 필요.
2. 접근: 부호 구성이 아니라 **기존 H-matrix에서 edge를 최소 삭제**해 target girth 달성(문제는 NP-hard, PFESP).
3. 핵심 기법 1(MBFS): 표준 BFS를 변형해 FIFO queue에 노드+정수+asterisk를 넣어 predecessor 저장 없이 support tree를 level `g/2`까지 탐색, cycle 검출 시 pending 노드 중복 출현으로 판정.
4. 핵심 식: cycle 탐색 공간복잡도 `C_ℓ = O(η^{ℓ/2})`, `η = E{A}·E{B}`(변수/체크 평균 차수 곱) — [15]의 `O(N·η^{ℓ/2})` 대비 factor N 절감.
5. 핵심 기법 2(edge deletion): cycle 표를 SCP(set covering)로 보고 greedy — 각 edge `e_{i,j}`가 포함된 cycle 수 `K_{i,j}`를 H 자체에 카운트, 최대 카운트 edge부터 삭제 후 갱신.
6. 트릭: cycle 길이 ℓ을 증분적으로(작은 것부터) 처리해 하위 cycle 삭제로 상위 cycle이 이미 깨지는 효과 활용, connectivity 유지(각 cycle당 1 edge만 삭제).
7. 반복성: 1회 적용 시 girth `g→g+2`, k회로 `g+2k`; 단 과도 삭제는 degree 분포를 왼쪽으로 이동시켜 구조 파괴→성능 저하.
8. 결과: [15] Trellis 기반 대비 동일 성능·저복잡도(QC(510,255)에서 동일 메모리로 girth 8→10, MNC(30000,2000)은 8→12 달성).
9. 성능: rate-1/3 (816,272) `d_v=4` 부호는 girth conditioning으로 최대 0.8dB 개선, `d_v=5` random regular는 6→10에서 유의 개선; 반면 최적화된 irregular·PEG 부호는 개선 없음.
10. 한계: HW 미설계·off-chip 전용, 잘 설계된(density-evolution 최적화) 부호엔 무효과, edge 삭제가 최소거리·구조 훼손 위험.
