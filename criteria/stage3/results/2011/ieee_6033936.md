### B. 알고리즘 요약

1. 대상: flash/PCM 등 비휘발성 메모리, 셀레벨(전압/저항) drift로 인한 비대칭 오류(1→0 우세), 경판정 read.
2. 문제: 고정 threshold `vf`는 시간에 따른 분포 이동으로 대량오류 유발, 순수 비대칭부호는 실제 혼합오류에 부적합.
3. 핵심 아이디어: 셀레벨 분포를 몰라도, block 내 상위 `k`개 셀만 '1'로 읽히도록 threshold `vd`를 동적조정(`|y(vd)|=k`).
4. Sub-optimality(Thm.1): `Ne(vd) ≤ 2·Ne(v*)` — 동적 threshold 오류는 최악에도 최적의 2배 이내, 고정 threshold(최악 n개 오류)보다 월등.
5. 통계적 관점: n 大에서 `(n-k)∫gt = k∫ht` 조건으로 vd 결정, Gaussian 모델(drift/variance 증가) 예제로 BER 감소 검증(Fig.3-4).
6. Scheme 1 (Berger): 정보워드 u의 1개수 v를 metadata로 저장, 고정 threshold로 v 읽고→v에 맞춘 동적 threshold로 u 읽음, C1/C2 두 BCH부호(C2가 더 강함). weight>n/2면 flip.
7. 효율 예: n=255, C1=BCH(t=8,k1=191), C2=BCH(t=18,k2=131) → 효율 0.705 (단일 고정 threshold 0.514 대비 향상).
8. Scheme 2 (balanced): 각 block의 1개수를 n/2 고정(Knuth balanced code)→metadata 불필요, partial-balanced+BCH(t=8) → 효율 0.7216, 인코딩/디코딩 더 빠름.
9. vd 계산(§V): sort O(n log n) 대신 half-interval search로 평균 O(n)(최악 O(ln)) 달성.
10. 한계: 이론/Gaussian 모델 분석뿐, HW·실측 없음, 부호는 BCH 기반이라 LDPC 무관, multi-level 확장은 개념적 언급만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 동적 read threshold(vd) 결정 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | 개념적 유사(read offset 조정)이나 Prime ECC는 LLR 기반 soft-read, 논문은 경판정 셀카운트 |
| Berger/balanced+BCH 부호 | 대응 없음 | Prime ECC는 binary QC-LDPC, BCH/balanced code 미보유 |
| block 1개수 기반 threshold | 대응 없음 | Prime ECC에 셀레벨 counting 로직 없음 |

적용 가치: **낮음** — read-threshold drift 대응 관점만 NAND와 접점이 있으나, 부호(Berger/balanced+BCH)·복호 모두 LDPC 디코더/HW와 무관하여 Prime ECC 이식 대상 아님.
