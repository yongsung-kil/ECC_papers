> 총평: 기존 802.16e QC-LDPC layered Min-Sum 복호를 GPU에 효율 매핑한 소프트웨어 병렬화 연구로, 알고리즘/부호 신규성이 없고 Prime ECC(고정 HW min-sum)에 떼어 쓸 요소가 없음.

### B. 알고리즘 요약

1. 시스템: IEEE 802.16e block-structured QC-LDPC(rate 1/2, n=576~2304, base 24열, expansion factor `zf`=24~96), AWGN+BPSK, GeForce 9800 GTX+ GPU.
2. 문제: layered decoding은 수렴 빠르나(TPMP 대비 ~2배) data-parallelism이 layer 크기 `zf`로 제한돼 GPU 코어가 유휴 → HW 이용률 저하.
3. 채널/복호: layered(TDMP) Min-Sum - `Q=L-R`, `R=sign·min|Q|`, `L=Q+R` 식(1)(2)(3)을 layer 순차 처리, 층내 갱신 estimate 재사용으로 5 iter가 TPMP 10 iter와 동등 BER.
4. 핵심1 (multi-codeword): block당 threads=`zf`로 한 코드워드 담당, SM당 최대 block 실행(식4~8 NBR/NBS/NBT)으로 여러 코드워드 동시복호해 global memory latency 은닉.
5. 핵심2 (H 표현): edge 저장 대신 base matrix 비영 submatrix의 열위치 P·shift S만 constant memory에 저장, `addr=P[j]*zf+(S[j]+tid)mod zf`로 tid별 1 위치 계산(레지스터급 접근).
6. 핵심3 (data packing): 서로 다른 4 코드워드의 8/6bit 메시지를 32bit 워드에 pack → global memory 접근 1/4, 동시복호 코드워드 4배(식10).
7. 핵심4 (async transfer): multi-stream으로 kernel 실행과 CPU-GPU 전송 overlap, 전송시간 비중 10%→1.38%(식11).
8. 핵심5 (coalesced access): 랜덤 permutation의 `Lj` 접근을 write는 permutation 수정(2nd shifter 제거 기법), read는 shared memory prefetch로 coalescing.
9. 결과: throughput 160Mbps(n=1536/2304), CPU 대비 2 자릿수·기존 GPU[9] 대비 4배 빠름, ASIC(60~71Mbps)/FPGA 대비 comparable, 6bit 고정소수점.
10. 한계: 순수 GPU/CUDA 소프트웨어 병렬화로 부호·복호 알고리즘 신규성 없음(layered MSA 그대로), ASIC/RTL 미설계라 NAND 컨트롤러 HW에 비이식.
