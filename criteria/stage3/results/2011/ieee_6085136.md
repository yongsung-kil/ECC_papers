> 총평: TDMP layered 복호에 행렬 reordering+two-layer 병렬을 결합해 iteration당 cycle을 크게 줄이고 register 메모리로 초저전력을 달성한 WiMAX 디코더 HW 논문으로, 알고리즘은 기존이나 병렬 스케줄링/저전력 아키텍처 관점에서 참고가치가 있음.

### B. 알고리즘 요약

1. 대상: WiMAX QC-LDPC, code length 576~2304, rate 1/2·2/3A·2/3B·3/4A·3/4B·5/6, TDMP(turbo-decoding message passing, layered)+normalized min-sum 복호.
2. 풀려는 문제: WiMAX 4G는 고throughput과 저전력을 동시에 요구 - 빠른 수렴 알고리즘+고병렬 아키텍처 필요.
3. 기존 [2]는 두 세트 PU와 reordering으로 iteration당 48~54 cycle까지 축소했으나 더 높은 병렬성 필요.
4. 핵심 기법 1단(two-layer 병렬): 한 layer의 24 block을 12-block씩 2 cluster로 나눠 처리, 96개 CBPE(clustered block processing engine)로 12 block 병렬 갱신, matrix reordering으로 iteration당 8~16 cycle 달성.
5. Matrix Reordering: WiMAX seed matrix의 non-overlapping 특성 이용, row reordering으로 8 layer를 데이터 의존성 없는 4 merged layer로 묶고, column reordering으로 두 layer의 block을 odd-even 교차 배치 → cross-bar switch(CBS)로 단순 분리.
6. 결과 cycle/iter: rate 1/2·2/3A·2/3B·3/4A·3/4B·5/6에 대해 12,16,8,12,12,8 (기존 [2] 대비 병렬성 3x~6.5x).
7. CBPE 파이프라인: layer당 4 cycle(앞 2 cycle VN 갱신+min1/min2 탐색, 뒤 2 cycle CN/APP 갱신)이나 pipeline으로 layer당 실효 2 cycle. 16-input min-sorter 사용.
8. Message Storing: APP/VN·CN 메시지를 모두 register에 저장(SRAM 미사용) - 고병렬 word width 확보·주변회로/라우팅 문제 회피, 총 메모리 47,232 bit(reuse 스킴). Permutation Network는 12개 96×96 barrel rotator(7-stage MUX)+CBSA.
9. 결과: SMIC 65nm, 2.89mm², 40MHz/1.08V에서 1152Mbps, 29.5mW. logic gate 919k(등가 1250k). 정규화 비교 시 [2] 대비 전력 76% 감소, energy/bit/iteration은 1/5.
10. 한계: 정정능력·알고리즘 자체는 기존(TDMP+normalized min-sum), WiMAX seed matrix 특성 의존, register-only로 combinational logic이 크게 증가(등가 gate 1.4x).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| normalized min-sum + min1/min2 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 보유, normalized factor는 코드상 미확인 |
| two-layer 병렬 / matrix reordering | 디코더 HW 모델 (z=32, HCU, GT), `ecc_top.cpp` `Load_PCM()` | 병렬 스케줄 개념 대응(구조 상이) |
| register 기반 메시지 저장/라우팅 | 디코더 HW 모델 (z=32), `GT.cpp` `Make_GT_HW()` | 메모리·라우팅 절감 관점 참고 |

적용 가치: **중간** — 알고리즘(TDMP+min-sum)은 Prime ECC가 이미 보유해 중복이나, matrix reordering으로 layer간 의존성을 없애 iteration당 cycle을 줄이는 병렬 스케줄링과 register-only 저전력 메모리 아이디어는 HW throughput/전력 개선 관점에서 참고 가능(단 WiMAX 부호·구조 종속).
