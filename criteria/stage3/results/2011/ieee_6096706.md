### B. 알고리즘 요약

1. 시스템은 half-duplex decode-and-forward(DF) relay(source/relay/destination, 2 time slot `T1=t`,`T2=1-t`), protograph-LDPC 코딩 BICM, Gray 16QAM(I/Q 각 4PAM), 정보비트 `4000`.
2. 문제: relay 세그먼트별로 SNR이 달라 → half-duplex relay BICM 설계를 rate-compatible BICM 설계 문제로 변환.
3. 채널모델: path-loss `h²_SD=1`, `h²_SR=(1/d)^β`, `h²_RD=(1/(1-d))^β`, AWGN 단위분산, DF 달성률 `R_DF=min{tC_SR, tC_SD+(1-t)C_RD}`.
4. 핵심 기법: rate-compatible protograph 구성 3가지 — 병렬 concatenation(`H2=[[A1 B 0],[A2 0 C]]`), check-splitting(check node C를 C1/C2로 분할하고 degree-2 새 변수노드 추가), lengthening(정보비트 제거로 저rate화).
5. 핵심: base matrix 원소는 병렬 edge 수, check-splitting `D=D1+D2`로 incremental redundancy 생성해 rate-compatible protograph 형성.
6. 매핑패턴 최적화: protograph EXIT로 변수노드-변조비트 매핑(예 `{2,3;1,4}`)의 수렴 threshold 평가, s1과 s를 동시 평가.
7. 구현: EXIT의 `J(μ)` 함수와 상호정보 갱신식(VN→CN, CN→VN, APP-LLR)으로 threshold(수렴 최저 SNR) 산출.
8. 최적화: 여러 d(0.2/0.3/0.5)에 대해 concatenated/check-split BICM 매핑을 동시 최적화, capacity bound와 비교.
9. 결과: log-MAP 복조 + 표준 sum-product(max 200 iter) 시뮬, 최적 BICM이 S-D link 대비 0.9~1.1dB, concatenated가 lengthen 대비 0.1dB·check-split 대비 0.16dB 이득.
10. 한계: HW 미설계, 시뮬/이론만, relay 채널·16QAM 변조에 강결합, rate/length가 예제 종속.
