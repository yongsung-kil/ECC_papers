### B. 알고리즘 요약

1. 대상은 802.16e(WiMAX) rate-1/2 QC-LDPC, PCM `1152x2304`, 서브행렬 `96x96`, 12 layer x 24 submatrix. offset-modified 버전 사용.
2. 풀려는 문제: 기존 LDA(layered)는 CNU-VNU 라우팅이 복잡하고 critical path가 길어 파이프라인 삽입 시 BER 악화. PLDA는 CNU/VNU를 단일 check node로 합쳐 파이프라인 가능하나 [3][4] 구현은 A를 여러 RAM에 중복저장해 면적↑.
3. 복호식: `q = new_A - r` (1), `new_r = prod(sign(q)) x min(|q|)xα` (2, scaled min-sum), `new_A = q + new_r` (3). α는 scaling factor.
4. PLDA 스케줄: 행렬을 96 part로 분할, iteration당 96 step, 각 step은 12행(check node 12개)을 처리하며 매 clock 1 step 진행.
5. 핵심 트릭(offset-modified 부호): 같은 열에서 인접 두 layer의 shift값 차를 최소 5 이상으로 설계 -> 한 A가 5 step 내 재사용 안 됨을 보장 -> check node에 파이프라인 4단 삽입 가능(BER 악화 없이 clock↑).
6. 핵심 기법(면적절감): A를 RAM에 중복저장하는 대신 shift register chain 2개(Ping-Pong)에 프레임당 2304개 A를 1회만 저장. chain을 24 subchain으로 분할, 매 clock 좌로 1 unit 순환 shift -> 읽기/쓰기 위치가 고정 -> check node-저장부 라우팅 고정·단순화.
7. 데이터 흐름: step N에 6~7 subchain에서 A 읽어 식 (1)(2)(3) 수행, N+4에 new_A를 좌로 4 unit shift된 (고정) 위치로 반환. Puncturing으로 mother code(rate-1/2)에서 rate 1/2~1 임의 생성(punctured parity를 0으로 patch 후 복호).
8. 정량화: A와 r을 6bit fixed point로 표현, floating 대비 0.1dB 손실. punctured code는 selected 802.16e 부호 대비 약 0.1dB 손실(BER 동등 수준).
9. 결과: TSMC 65nm, core area `1.14mm2`(puncturing 복구회로 0.25~0.3mm2 포함), 최대 1GHz, throughput `1.2Gbps`(10 iter, 전 rate), 전력 613mW. [3] 대비 면적 40%↓·전력 30%↓, throughput 동등. shift chain부(2 chain+12 check node)만 0.44mm2 vs [3] RAM방식 1.2mm2.
10. 한계: 정정성능 개선 없음(BER 동등), offset-modified 특정 H-matrix 전제, gate count 미기재, throughput/area(TAR) 지표 최적화 중심.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PLDA scaled min-sum 복호 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal()` | min-sum·layered 이미 보유, 알고리즘 신규성 낮음 |
| shift register chain(Ping-Pong) APP 저장 / 라우팅 고정 | 디코더 HW 모델(z=32 pipeline, GT, HCU) | HW arch 참고 가치 - 메시지 저장·라우팅 최적화는 우리 pipeline/GT와 정합 검토 필요 |
| check node 파이프라인 4단 삽입 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) pipeline(col_M1/Z/P1~P4) | 우리도 6-stage pipeline 보유 - offset 스케줄 아이디어 대조 가능 |
| offset-modified H-matrix (layer간 shift차 >=5) | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` + H-matrix | 부호구조 재설계 필요 - 이식 부담 큼 |
| puncturing (rate 1/2~1) | 대응 없음 | 우리는 punct_col_num=2 고정, 임의 rate puncturing 미보유 |

**적용 가치**: 중간 - min-sum·layered는 이미 보유해 알고리즘 신규성은 낮으나, APP를 RAM 중복 대신 shift register chain에 1회 저장해 라우팅 고정·면적/전력을 줄이는 HW 아이디어는 우리 pipeline/GT HW 모델에 참고할 만하다.
