> 총평: (3,6)-regular binary LDPC의 error-floor를 trapping set 개수로 예측하는 통계 연구로, error-floor 원인 진단·부호선별에 참고가치는 있으나 직접 이식할 알고리즘/HW는 없음.

### B. 알고리즘 요약

1. 대상은 length ~280, rate ~0.5의 `(3,6)`-regular, girth 8 binary LDPC 3계열: random ensemble, block-circulant(K3,6 cover, `p=47`), Fano plane cover(`20-cover`).
2. 문제: SPA(sum-product)의 error-floor는 시뮬로 추정 비용이 크므로, H행렬의 어떤 조합/대수 특성이 성능을 예측하는지 규명.
3. 예측변수: 8-cycle 개수, spectral gap 2종(adjacency의 `Tanner gap`, edge-flow 행렬의 `flow gap`), 그리고 trapping set 개수.
4. `(a,b)` trapping set = `a`개 bit node 중 `b`개 check가 odd degree인 부분그래프; girth 8·degree-3이므로 8-cycle은 `(4,4)`. `(5,3),(6,2),(6,4)` 및 codeword `(6,0),(8,0)`을 다룸.
5. 방법: AWGN, 최대 500 iteration, SNR 4.5(error-floor 영역)에서 `log(FER)`를 종속변수로 Pearson 상관·adjusted r² 회귀.
6. weight-6/8 codeword((6,0),(8,0)) 보유 부호는 극단적 outlier라 회귀에서 검열(censoring).
7. spectral gap 결과: Tanner/flow gap 자체는 예측력 약함. 단, K3,6 cover에서 flow 행렬 제3고유값 `e3`가 실수면 FER이 약 1자릿수 높음(p≈7e-11).
8. trapping set 결과: `(6,2)` 개수가 개별 계열별 최적 단일 예측변수(Fano r̄²=0.76, random 0.53); `(5,3),(6,4)`는 계열 간에는 예측력 있으나 계열 내부에서는 약함.
9. 다중회귀(8-cycle+(6,2)+(5,3)+(6,4))는 all/K3,6에서 개선, Tanner gap 추가는 유의미한 개선 없음.
10. 한계: HW 미설계, 부호 생성 지침이 아닌 사후 예측 연구, 대수 구성 부호에서만 예측력이 높고 random에서는 낮음.
