### B. 알고리즘 요약

1. 대상: rational parity-check matrix(RPCM)를 갖는 LDPC convolutional code(LDPC-CC-RPCM) 복호. rate 1/2, block length `10^6`, AWGN/BPSK, flooding SPA는 iter 100.
2. 문제: RPCM을 직접 Laurent 급수로 전개하면 memory order `m_s=∞`, length-4 cycle 대량 발생 → pipeline 복호 지연/복잡도·성능 파탄.
3. 그래프 변환: dummy bit를 도입해 RPCM을 등가 다항식 PPCM(finite `m_s`)으로 변환, 원 코드는 mother code에서 puncture된 child code로 간주.
4. 핵심 기법1(Procedure 1): 변환그래프에서 length-l cycle을 dummy bit(식(6) α,β 확장)로 순차 제거, girth를 `γ`까지 확대. Property 1~2로 짧은 cycle 미생성·length+2 이동 보장.
5. stopping set 성질: Property 3~4로 dummy 노드 집합 Ω는 stopping set 불가, `H̄` stopping set이 `H_eq`의 것과 동일 → waterfall/error-floor 개선 근거.
6. 핵심 기법2(EDS): window 기반 dynamic scheduling. residual `R(ν)=|Qa-Qb|/|Qa+Qb|`(식8) 큰 노드 우선 갱신.
7. EDS-MRF(식9): residual에 `|Qb(ν)|` 곱해 신뢰 높은 노드 우선 → 원 EDS의 error-floor 상승 결함 완화.
8. signal perturbation: window를 K 블록 분할, 평균 log-APP가 threshold 미만인 블록(trapping) 노드에 소분산 Gaussian 섭동 후 재복호.
9. error cancellation: 저weight codeword 집합을 오류패턴 후보로 빼서 ML likelihood 최대 후보 선택 → error-floor 대폭 개선.
10. 결과: 제안 기법이 flooding 대비 waterfall/error-floor 동시 개선, C3(RPCM)이 동일 rate·memory PPCM C4 대비 BER 10^-6에서 0.4dB(양측 개선 시 0.2dB) 이득. HW 미설계·시뮬만.
