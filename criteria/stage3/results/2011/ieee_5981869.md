### B. 알고리즘 요약

1. 대상: 산업용 43nm 2bit/cell MLC NAND, regular QC-PEG binary LDPC `(9180, 8364)`, rate≈0.911, circulant 204×204.
2. 문제: MLC raw-BER가 10^-2급으로 상승 → t=96 BCH도 parity 1344bit 필요해 비현실적, LDPC+soft-info 필요.
3. 모델: 각 region Vth를 Gaussian(`μi,σi`)으로 모델, deviation factor `α`로 disturbance 주입(LoPage/HiPage 독립 page 인코딩→binary LDPC 선호).
4. 핵심기법1(soft): 원래 reference voltage(ORV) 2^n-1개 외에 추가 reference voltage(ARV)로 Vth 위치 세분 → region 확률식 (5)로 LLR `log(P(bk=1)/P(bk=0))` 산출.
5. 핵심 식: ORV 부근에서 오류가 몰리므로 ORV 근처 ARV가 정보량이 큼 → 균일 배치는 낭비.
6. 핵심기법2(NURV): ARV를 ORV 거리별 집합 `Si`으로 분할, 거리 작은 집합부터 증분 추가하며 시뮬로 성능 만족 시 종료 → ARV 수 최소화(URV-28 대비 NURV-17).
7. 구현: LLR은 설계단계 precompute 후 lookup table 저장, α별 configurable(endurance 대응); QC 형태로 G도 cyclic → k-bit(=8) parallel encoder.
8. 디코더: TDMP(turbo-decoding message-passing) 알고리즘 + scaling factor β=0.75 Min-Sum SISO, γ-memory bank 분할로 k(=1~4) row 병렬, 최대 4 iteration.
9. 결과: NURV-17이 α<3.2에서 BER<10^-11, BCH59 대비 10^7배 개선; 2 MPU 디코더 16.32μs/313.49K gates(0.13um), ONFi 2.2 최속 요건 충족.
10. 한계: throughput 수치 미제시, soft-read가 다수 ARV(14개) read 요구, iteration 4 고정(iter 감소 논의 없음).
