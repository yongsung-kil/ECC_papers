### B. 알고리즘 요약

1. 대상: rate `R=1-j/k` regular `(ms,J,K)` time-invariant LDPC convolutional code(LDPCcc), QC LDPC block code에서 유도, 예제 GF(7)/a=2/b=6로 rate `1/3`.
2. 문제: SPA 복호에서 factor graph의 short cycle(작은 girth)이 error-floor·수렴속도를 악화 → 코드 설계 시 제거 필요.
3. 모델: QC block PCM `H_QC`(circulant identity 배치)를 monomial matrix `H(D)`로 변환(각 shift량을 `D^shift`로 표현), transpose가 polynomial syndrome former `H^T(D)`.
4. 핵심 개념: 같은 행/열 monomial은 delay `Δ(s_il,s_kl)=s_kl−s_il`로 연결, 닫힌 path의 delay 합=0이면 cycle 성립(식 12).
5. 핵심 식: `N_12-Cycles ≥ C(j,2)·C(k,3) + C(j,3)·C(k,2)` — (j,k) LDPCcc의 구조적 12-cycle 하한.
6. 핵심 결과 1(불가피 12-cycle): j≥2, k≥3인 monomial syndrome former는 monomial power 선택과 무관하게 2×3/3×2 submatrix 구조에서 12-cycle 필연 발생.
7. 핵심 결과 2(entry weight>1): Property1(weight>2→girth≤6), Property2(같은 행/열 두 entry weight>1→girth≤8), Property3(임의 entry weight>1→girth≤10).
8. 설계 규칙: girth를 크게 하려면 weight 큰 polynomial보다 monomial(weight=1) 선호, girth>12 위해 H^T(D)에 2×3/3×2 submatrix 배제.
9. 예: (3,5)-code m/ms=2311/2005는 하한(17) 만족하며 4,6,8,10-cycle 없어 cycle 관점 "perfect".
10. 한계: 순수 이론(girth 분석)뿐, HW·시뮬·BER·free distance 평가 없음, block code가 아닌 convolutional code 전용.
