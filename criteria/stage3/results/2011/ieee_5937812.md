### B. 알고리즘 요약

1. 시스템: IEEE 802.15.3c mmWave WPAN용 672-bit, rate-1/2 QC-LDPC (336×672 H, base matrix 16×32, sub-matrix `z=21`).
2. 문제: 기존 layered 복호는 shuffling/reshuffling용 양방향 crossbar SN이 필요해 HW·라우팅 복잡도가 크고, 행간 데이터 의존성으로 파이프라인·병렬화가 직접 적용 불가.
3. 핵심기법 1단: 4-parallel block layered decoding architecture(4-PBLDA) — 선택 부호의 4개 block row가 재정렬 없이 column weight ≤1이라 4개 CNBPs로 1 클록에 동시 처리, iteration당 클록 75% 감소·throughput 약 4배.
4. 핵심기법 2단(라우팅): 각 CNBPs가 담당하는 4개 block row가 동일 순서·동일 permutation 값을 공유 → SN의 sub-switch가 항상 같은 cyclic shift 수행 → 12개 SN 전부를 crossbar 대신 fixed wire network로 대체.
5. 핵심식(1)~(4): Min-Sum 기반 layered 갱신. `L[k]=y[k]-R[k-1]`(V2C), `R[k]=S·min(|L|)`(C2V), `S=∏sign`, 그리고 파이프라인용 latest variable message 재계산식(4).
6. 부수 트릭: approximate layered decoding으로 2단 파이프라인 — 최신 variable message를 즉시 사용해 파이프라인 지연으로 인한 BER 손실 최소화(파라미터 `p=2`).
7. 양자화: 내재 메시지 `(q,f)=(6,2)`, extrinsic 7bit, CNBP 내부 9bit. Min-Sum block은 비교 트리 구조.
8. 결과: 90nm CMOS, 285MHz, 정보 throughput 1.45Gbps(16 iter), gate 759K. 파이프라인 유무 비교시 throughput 13%↑ / gate 24%↑.
9. 성능: 근사 layered 16 iter가 표준 layered 8 iter와 유사 BER(Fig.4) — iter 늘지만 클록 상승으로 순 throughput 이득.
10. 한계: rate-1/2 짧은 부호 전용, NAND 고rate 아님. soft-read/양자화 세부는 통신 채널 기준, HW는 합성 단계까지만(실측 아님).
