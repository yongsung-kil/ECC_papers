### B. 알고리즘 요약

1. 대상: `rate=1/2` IRA(Irregular Repeat Accumulate) 부호, AWGN/BPSK, BP 복호(최대 iteration 50), 부호길이 150~6000 가변.
2. 문제: Hui Jin의 IRA protograph는 우수한 degree distribution을 갖지만 실제(practical) 부호가 설계되지 않았음 → 구현 가능한 부호 필요.
3. 모델: protograph = 6 base VN(degree 4/3/2), 3 base CN(degree 6/5), degree distribution `λ(x)=0.25x³+0.375x²+0.375x`, `ρ(x)=0.375x⁵+0.725x⁴`.
4. 핵심기법: 3×6 base check matrix `Hbase`를 π1(column weight 1, identity/shift)·π2(column weight 2, dual-diagonal)·0 블록으로 구성, base shift matrix `B`(3×6)로 각 서브행렬의 우측 순환 shift량 지정.
5. 서브행렬 크기 `z`를 가변 선택 → 부호길이 `N=z·n0` 유연 조정.
6. length-4 cycle 제거: identity 계열은 `B(i,j+n)-B(i,j) ≠ B(i+m,j+n)-B(i+m,j)`, dual-diagonal 계열은 `... ≠ ... ±1` 조건 만족까지 base matrix 재구성.
7. 사용 base matrix `B` = [[96,49,46,45,93,41],[24,90,2,62,74,94],[61,77,83,80,18,92]]로 length-4 cycle 없음 검증.
8. 결과: 부호길이 증가할수록 BER 개선, length-4 제거 전후로 error floor 크게 낮아짐(BLER 곡선 cliff화).
9. (600,300) IRA vs IEEE802.16e (576,288) LDPC: BLER=0.01에서 0.5dB 이내로 근접.
10. 한계: HW 미설계, 저rate·통신 응용(멀티미디어), 순수 시뮬 검증뿐, NAND soft-read/양자화 논의 없음.
