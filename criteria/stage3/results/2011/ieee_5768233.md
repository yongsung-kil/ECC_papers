### B. 알고리즘 요약

1. 목표: PEG로 base matrix `M(H)` 생성 후 각 nonzero block의 shift 지수 `a_ij`를 골라 girth `2g`의 QC-LDPC(`mL×nL`) 구성.
2. 문제: 전통적 방식은 basic matrix의 모든 short cycle을 탐색·제거(cycle-search)하는데, `n`이 커지면 `O(C_n^{g-1}(C_d^2)^{g-1} g/2)`로 계산량이 폭증.
3. block-cycle 조건: exponent chain `P^{a1}→...→P^{a2l}→P^{a1}`이 `Σ(-1)^{i-1} a_i ≡ 0 mod L` (식5)을 만족하면 H에 길이 `2l` cycle 발생 → 이 등식을 피하도록 `a_ij` 선택.
4. 핵심 개념: `k-order rows relation value` `C^k_{i,j}` — 두 행 i,j가 같은 열에서 nonzero면 1-order 값 `a_{i,r}-a_{j,r}`, 이를 연쇄하여 k-order까지 확장.
5. cycle-search 대신 `C^{l-1}_{i,j}` 집합을 유지: `(a_{i,r+1}-a_{j,r+1}) ∈ C^{l-1}_{i,j}`이면 길이 `2l` cycle이 생기므로 이 값을 회피.
6. 개선 절차: 열 순서로 `a_{i,r}`를 결정할 때 회피집합 `Sc={t | t=C^1..g_{i,j}+a_{j,r}}` (식7)을 만들고 `S_t \ Sc`에서 선택.
7. 값 확정 후 `C^k` 갱신(식8): 새 값 `a_{i,r}-a_{j,r}`를 `C^1`에 추가하고 상위 order로 전파.
8. 갱신 복잡도 `O(n C_d^2[1+2(2m+3m^2+...+(g-2)m^{g-3})])`로 cycle-search보다 낮음.
9. 결과: (690,345) girth=10 구성 시간 PEG-C 1.62s vs PEG-R 1.55s, PEG 원본 1241s; (5400,2520) PEG-C 1589s vs PEG-R 459s. BER/FER는 random PEG와 거의 동등.
10. 한계: 구성 시간만 개선하고 정정 성능은 동등, HW 미설계, AWGN 시뮬만, rate=1/2 예제 위주.
