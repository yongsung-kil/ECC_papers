> 총평: APP-based(=min-sum 계열) 복호에서 정규화 위치를 check node→bit node로 옮겨 곱셈수와 성능을 동시 개선한 저복잡도 알고리즘으로, Prime ECC의 min-sum과 개념 중복이 크지만 bit-node scaling 아이디어 자체는 낮은 난이도로 이식 검토 가능.

### B. 알고리즘 요약

1. 시스템: BPSK/AWGN, LLR 기반 반복복호. 검증 코드 `(504,252,6,3)`, `(1008,504,6,3)`, irregular `(1944,1485)`(IEEE802.11n, rate 3/4).
2. 문제: 표준 APP-based(Fossorier/Imai)는 BP보다 성능 열화; bit node 간 correlation·overestimation이 주 원인.
3. 배경: normalized APP-based[5]는 check node에 정규화 인자를 곱해 성능 개선하나 곱셈이 check node degree 수만큼 필요.
4. 핵심 기법: bit node 처리식에 suppression factor `γ` (0<γ<1) 도입 → `Qn = γ·(Pn + Σ_m Rmn)` (식6, 기존 식7 대체).
5. 식 의미: γ가 (i) Qn을 BP의 Qnm에 근접시켜 정확도↑, (ii) check→bit 메시지를 등가 축소해 정규화와 동일 효과 → bit node correlation 감소.
6. check node 연산은 min 기반: `Rmn = (-1)^(zm⊕zmn)·min|Qn|` (식4), 즉 min-sum류에 부호 XOR.
7. 복잡도: 곱셈을 check node(Σ degree)에서 bit node(`N=M/(1-r)`개)로 이동 → 항상 normalized APP 대비 적음; r=1/2에서 곱셈 2/3+ 감소.
8. 최적화: γ는 해석 부재로 시뮬로 결정, 최적 `γ≈0.40`이 SNR·rate·length에 둔감함을 실험으로 확인.
9. 결과: APP 대비 ~1dB(504/1008), >0.5dB(1944) 개선; 짧은 코드는 BP-based보다 우수; 평균 iteration도 상당히 감소.
10. 한계: 시뮬레이션(max 50 iter)뿐, HW 미설계, gate/throughput 수치 없음, error-floor 미검토.
