### B. 알고리즘 요약

1. 대상: regular LDPC code 앙상블 `E(l,r)`(column weight `l`, row weight `r`)의 error-floor 관련 구조인 absorbing set / fully absorbing set의 점근(asymptotic) 개수 분포를 이론적으로 유도.
2. 문제: 기존 [6]은 trapping set 점근 분포만 다뤘음. Absorbing set(iterative/bit-flipping 디코더에서 실제로 안정적으로 갇히는 부분집합)의 분포를 별도로 구해 trapping set과의 차이를 규명.
3. 정의: `(a,b)` absorbing set = 크기 `a`인 VN 부분집합 `D`가 unsatisfied check(`O(D)`) `b`개에 연결되되, `D`의 각 노드가 satisfied check에 더 많이 연결. Fully absorbing set은 `D` 밖 VN도 동일 조건.
4. 핵심 도구: 주어진 행/열 weight를 갖는 random binary 행렬 개수의 점근 enumerator(Theorem 2.3, Bender [14]) + 행렬을 서브행렬로 분할·재정렬하는 multinomial ordering factor(Lemma 2.5).
5. 핵심 식: PCM transpose를 `M1|M2 / M3`(fully는 `M4` 추가)로 분할해 각 서브행렬을 absorbing set 제약(even/odd column weight, `β_j=0 for j<l/2`)에 맞춰 세고 결합.
6. 결과식: absorbing set 점근 scaling `a(θ,λ)`(Theorem 3.1)와 fully absorbing(Theorem 3.2)을 `(a,b)=(θn,λn)` 기준으로 유도.
7. E(3,r) 특수화: elementary (fully) absorbing set의 닫힌형(Corollary 3.3/3.4). `λ→0`이면 trapping·absorbing·fully-absorbing이 동일해짐.
8. 수치 결과: 고정 `θ=1e-3`에서 λ 증가 시 trapping vs absorbing 카디널리티 차이 증가, 고rate `E(3,15)`에서 차이가 더 큼(Fig.2-4); `λ/θ≤1`에서만 absorbing set 존재 = `λ/θ>1` trapping set은 iterative decoding에 불안정.
9. 활용 함의: 작은 set·소수 unsatisfied check 영역에서는 trapping set이 (fully) absorbing set의 좋은 근사이므로 error-floor 성능분석의 proxy로 사용 가능.
10. 한계: 특정 부호/디코더/HW 설계·시뮬 없이 앙상블 점근 개수만 다루는 순수 이론, bit-flipping·regular 앙상블 가정에 한정.

### C. Prime ECC 관련 모듈 핀포인트

| H-matrix 구조/부호구성 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | error-floor 유발 구조를 간접 시사하나 QC-LDPC 특정 H 설계·검증 도구는 아님 |
| error-floor 관련 성능분석 | 대응 없음 (Prime ECC는 min-sum/CRC 조기종료) | bit-flipping 디코더 안정성 기반 분석이며 Prime ECC min-sum 디코더와 무관 |
