### B. 알고리즘 요약

1. 시스템: IEEE 802.16e WiMAX LDPC(coded block `N=576`, rate `1/2~5/6`), BPSK, AWGN, BP 복호(최대 200 iter).
2. 문제: 무선 시스템은 rate-matching이 필수인데, 표준은 rate마다 별도 부호를 지정 — 하나의 mother code로 임의 저rate를 만들고 싶다.
3. 모델: dummy bit 삽입(code shortening) — 알려진 `K_d`개 bit를 정보열에 넣고 인코딩 후 puncture, 수신단에서 `±∞` LLR로 완전 사전정보 제공, 유효 rate `R_eff = R / (1 + (1-R)·K_d/K_eff)`.
4. 핵심 기법: dummy bit를 단순 끝에 붙이지 않고, Tanner graph에서 dummy 정보노드들이 최대한 균일하게 퍼지도록 배치 — 노드 간 평균 최소 pairwise 거리 최대화.
5. 핵심 식: 첫 두 노드는 `(i_max,j_max)=argmax D_ij`(가장 먼 쌍, 식 4), 이후 `i*=argmax (1/|V|)Σ D_ij`(선택된 노드집합까지 평균거리 최대, 식 5).
6. 확장: 가변 `K_d` 지원 — `K_d=K`로 한 번 실행해 노드 추가 순서를 리스트로 저장, 요청 rate마다 앞 `K'_d`개만 사용.
7. 구현: BFS로 `K×K` 거리행렬 D(대칭, 대각 0) 계산, 상/하 삼각만 저장. dummy 노드/edge는 복호 시 `+∞`(boxplus 항등원)라 계산에서 제외 → 복잡도 증가 없음.
8. 최적화: 별도 학습 없음, greedy heuristic(Alg.1).
9. 결과: WiMAX `N_eff=576`, mother rate 5/6에서 최적 삽입(DBI B)이 단순 삽입(DBI A) 대비 error floor 개선, `R_eff=1/2`에서 원 rate-1/2 부호 대비 waterfall 손실 <0.2dB, 수렴도 원부호보다 빠름.
10. 한계: HW 미설계, 시뮬만, WiMAX 무선 표준·rate-matching 전용, AWGN·BP(full tanh) 기반.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dummy bit(shortening) 위치 최적화 | 대응 없음 | Prime ECC는 고정 rate ECC, code shortening/rate-matching 미사용 |
| Tanner graph 거리 기반 노드 배치 | ecc_top.cpp Load_PCM() | H-matrix 구조 관점만 접점, 부호설계 변경은 이식난이도 높음 |
| BP(tanh) 반복 복호 | 대응 없음 | Prime ECC는 min-sum, 논문은 full-tanh Sum-Product/boxplus |
```

마지막 적용 가치: **낮음** — 무선 rate-matching(code shortening)용 dummy bit 배치 최적화로, 고정 rate이며 shortening을 쓰지 않는 NAND binary QC-LDPC ECC와 사용 시나리오가 달라 이식 가치가 없다.
