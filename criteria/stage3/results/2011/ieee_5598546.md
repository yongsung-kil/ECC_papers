> 총평: WiMAX multimode QC-LDPC codec으로 offset-min-sum·layered decoding·조기종료·dual-diagonal 인코딩 등 Prime ECC와 겹치는 요소가 많고, 메모리 address generator/task 분할·hazard 처리 기법이 참고가치 있으나 다중모드 유연성 지향이라 단일코드 NAND엔 부분 적용.

### B. 알고리즘 요약

1. 대상: WiMAX irregular QC-LDPC (6 code class, rate `1/2~5/6`, expansion factor `z=24~96`, 총 114 코드, PCM 최대 `(2304,1152)`).
2. 문제: WiMAX는 rate별로 PCM이 개별 규정되어 full-mode 저복잡도 디코더 설계가 난제. 특히 저병렬 memory-based 구조에서 불규칙 shift index·복잡한 address 생성이 걸림돌.
3. 기법: LMPD-ICM(identical core matrix layered decoding, offset min-sum `OMSA`)의 core matrix 복호연산을 여러 개의 작고 유사한 `task`(2x2/4x4 array)로 분할·정렬하는 task-arrangement 알고리즘.
4. task 분할로 core matrix 전체를 한꺼번에 읽지 않고 task 단위로 read→permute→process, 출력 buffer와 처리 latency 축소.
5. collision pair(같은 sublayer 두 check node가 동일 APP를 병렬 갱신, 식9)를 인접 task에 배정해 data hazard를 최소 비용으로 처리.
6. address generator: task별 4~8-stage shift register + Index Calculator로 APP memory block 주소를 규칙적으로 생성, 같은 class 19개 코드가 동일 routing network 공유.
7. CNU는 2-stage sorter(CMP4+S-CMP4)로 최소 두 값 탐색, 4 CNU/task로 6개 class 재사용. pipeline 12-cycle latency, 3종 data hazard(type-I forwarding, type-II feedback, type-III update 교환)로 대응.
8. dual-diagonal prototype 구조를 이용한 task-based layered encoding: 인코더가 디코더의 memory/routing/early-termination 자원 재사용(추가 gate 0.87K).
9. 결과: 90nm, 0.679mm2, 400MHz, 인코딩 800Mbps/디코딩 200Mbps, ET 시 전력 57.8mW. length-2304에서 fixed(APP/V2C 7bit, C2V 5bit)-float BER 근접, 동종 대비 최소 (scaled) 면적.
10. 한계: 병렬도 4의 유연성 우선 설계라 절대 throughput 낮음, multimode 지원이 핵심 목적이라 단일 고정코드(NAND) 시나리오엔 task 분할 이득이 제한적.

적용 가치: **중간** — offset-min-sum·layered decoding·조기종료·dual-diagonal 인코딩·양자화가 Prime ECC와 계열이 같아 개별 기법(특히 memory address generator, hazard 처리, task 분할)은 참고가치가 있으나, 핵심 가치가 "multimode 유연성"이라 단일 고정 QC-LDPC를 쓰는 NAND ECC에는 이식 이득이 부분적.
