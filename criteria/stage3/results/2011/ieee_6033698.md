### B. 알고리즘 요약 (10줄 내외)

1. 대상: binary (γ,ρ)-regular LDPC, parity-check H가 RC제약(두 row/col이 1 이상 위치서 동시 nonzero 불가) 만족, AWGN·iterative decoding(SPA/MSA).
2. 문제: 대부분 LDPC의 error-floor는 Tanner graph의 작은 trapping set(작은 κ, 작은 τ/κ)에서 기인.
3. 정의: `(κ,τ)` trapping set = κ개 VN이 유도하는 서브그래프에 홀수차 CN이 정확히 τ개; small = κ≤√n, τ/κ≤4 (Def.2) 또는 τ/κ<1(통상).
4. 핵심 기법: RC제약으로 각 error bit이 γ개 orthogonal syndrome-sum으로 체크됨을 이용, 오류 패턴 t≤γ의 trapping set 구조를 조합적으로 분석.
5. 핵심 결과식: t개 오류를 쌍으로 균등분포시킨 최악 구성이 `(t, t(γ-(t-1)))` trapping set → 홀수차 CN 최소 개수의 하한.
6. Theorem 1: γ>1 RC-regular LDPC는 크기 γ+1 미만이며 홀수차 CN이 γ 미만인 trapping set이 없음.
7. Theorem 2: γ>3, γ≤√n 이면 Def.2 기준 크기 γ-3 미만 small trapping set 없음.
8. 적용 부호: EG/PG(finite geometry), finite-field QC-LDPC(CPM array), Latin-square QC-LDPC 등 알려진 RC제약 대수적 부호에 적용, 모두 큰 min-weight·큰 row redundancy.
9. 예시 수치: cyclic (4095,3367) EG-LDPC(min-weight 65) → 크기 65 미만 small trapping set 없음; (992,750) Latin-square QC-LDPC → 크기 36 미만 trapping set 컴퓨터 탐색서 미발견.
10. 한계: 순수 이론(HW/시뮬 없음), regular·large column-weight 코드 한정, min-weight가 크다는 전제; 디코더 알고리즘 자체는 개선 아님.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RC제약 H-matrix / 부호구조 (girth≥6, orthogonal check-sums) | `ecc_top.cpp` `Load_PCM()`, H-matrix | 부호설계 계층; 우리 QC-LDPC가 RC제약·column-weight γ 만족 여부 판단에 참조 |
| trapping set / error-floor 보증 분석 | 대응 없음 | 부호 특성 이론 분석 도구로, 디코더 실행 코드 대응 없음 |
| QC-LDPC (CPM array) 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 base+circulant QC-LDPC라 구조적으로 유사 |

적용 가치: **중간** — error-floor를 부호구조로 보증하는 이론적 근거를 제공해 Prime ECC H-matrix 설계·검증 관점 참고가치는 있으나, Prime ECC는 고rate·비대칭 낮은 col-weight(VN deg 17, punct col) 구조라 regular·large-γ 전제 정리를 직접 이식하긴 어려움.
