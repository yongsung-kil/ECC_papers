> 총평: CMMB용 LDPC 인코더의 RU ALT gap 최소화(backtracking) 전처리 기법으로, 우리는 dual-diagonal 고정 인코더를 쓰고 방송용 저throughput이라 이식 가치 낮음.

### B. 알고리즘 요약

1. 대상: CMMB 표준 QC-LDPC(High-Structured regular QC) 인코더, codeword length `9216`, rate `1/2`(K=4608)/`3/4`(K=6912).
2. 문제: LDPC의 generator 행렬은 dense → 직접 인코딩은 length에 quadratic 복잡도, HW 구현 부적합; RU의 greedy는 전역 최적 ALT를 못 찾음.
3. 모델: 패리티 방정식 `Hc^T=0`, IDX permutation으로 systematic form `H_IDX=H·T_IDX^T`를 `H_P|H_S`로 분리, `H_P p^T + H_S s^T = 0`.
4. 핵심 기법: `H_P`를 Gaussian elimination + row/column permutation으로 sparse LU 분해(`H_P C = LU`), 인코딩은 `x=-H_S s^T` -> forward-subst `Ly=x` -> back-subst `Up_C=y` -> `p=C p_C`.
5. 핵심 식: `p^T = -H_P^{-1} H_S s^T`의 dense 역행렬을 sparse LU 대체 -> linear-time 인코딩.
6. 핵심 확장: greedy 대신 **optimized backtracking algorithm**으로 ALT 삼각화 — degree-1 row를 node로, residual 행렬의 row/col degree 합 최소 node를 existence-node로 확장/역추적하며 최소 rank(gap) 경로 탐색.
7. 최적화 트릭: degree 합 계산을 row만 계산하도록 축소해 연산시간 단축, existence-node 수가 적어 backtracking 시간 감내 가능.
8. 전처리는 소프트웨어에서 1회만 수행(HW 인코딩 시간/자원 무소모), HW는 MVM/FS/BS/CWG 4-stage double-buffering 병렬 파이프라인.
9. 결과: ALT gap 축소(1/2: 302->171, 3/4: 60->38); Altera Stratix II에서 logic 4%/memory 62%, 200MHz에서 `36.18Mbps`(1/2)/`72.39Mbps`(3/4).
10. 한계: CMMB 특정 코드용, 정정능력/복호성능 논의 없음, throughput 방송용 수준으로 매우 낮음, gate count 등 정밀 복잡도 수치 미기재(참고문헌 오염).
