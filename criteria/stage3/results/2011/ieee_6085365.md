### B. 알고리즘 요약

1. 시스템은 4x4 spatial-multiplexing MIMO-OFDM, CP-OFDM으로 각 서브캐리어를 flat-fading 채널로 변환, IEEE 802.11n LDPC(`rate 1/2`, `length 1296`, bidiagonal PCM), QPSK.
2. 문제: turbo-equalization(outer loop)에서 LDPC 복호·MMSE-IC 등화가 반복되어 복잡도·지연이 커짐 → LDPC 반복 수를 줄이되 성능은 유지하는 스케줄링 필요.
3. 채널 모델은 `y = H·s + n` (AWGN), MMSE-IC 간섭제거기가 `p_k`,`q_k` 필터로 심볼 추정.
4. 핵심 기법: 정적 스케줄(off-line로 각 outer loop당 iteration 상한 산출) 대신 실시간 신뢰도 metric 기반 동적 정지.
5. 기준 metric은 mean reliability `MR = (1/N)Σ|LLR(b_i)|` — 복호 codeword의 평균 LLR 크기.
6. 확장: 코드 irregularity 반영 — `WMR`은 degree `d_i`로 가중(고degree 노드 강조), `WPMR`은 `1/d_i`로 가중(저degree 노드 강조), `MRi`는 정보비트만으로 계산.
7. 정지규칙: CMR(MR이 두 iteration 연속 불변 시 정지), FMMR(MR이 처음 감소하면 정지).
8. (학습/최적화 절차 없음 — 규칙 기반)
9. 결과: MR/MRi/WMR/WPMR은 syndrome criterion과 동일 PER 유지하며 avg iteration 감소, FMMR은 최악 27회(SC/MRi는 50회)이나 성능 열화; 반복수신 이득 0.5 dB.
10. 한계: HW 미설계, 시뮬만, MIMO-OFDM 특정 시나리오, 정지기준을 채널·outer loop별로 적응해야 함.
