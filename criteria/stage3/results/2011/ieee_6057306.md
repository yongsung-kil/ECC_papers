### B. 알고리즘 요약

1. QC-LDPC 부호(패리티행렬이 circulant permutation/zero 블록으로 구성)의 구성·저장 부담을 줄이는 recursive compression 알고리즘을 제안하고, VHDL로 FPGA 복호기를 구현.
2. 문제: 일반 QC-LDPC는 부호마다 검사행렬과 생성행렬을 함께 저장해야 해 저장부담·인코딩 복잡도가 크다.
3. 핵심 아이디어: girth가 큰 부호를 base 부호로 고르고, shift 값 `s''_ij = ceil(s_ij / q)` 형태로 압축비 `q`를 적용해 검사행렬을 축소, girth/최단환(shortest cycle) 특성을 비교하며 더 나은 쪽으로 갱신.
4. 한 base 부호와 압축 알고리즘만 저장하면 넓은 부호길이 범위에서 연속적으로 여러 QC-LDPC를 파생 가능 - 부호별 개별 저장 불필요.
5. 인코딩은 순환시프트 레지스터로 `c = s x G` 수행, 부호길이에 선형인 복잡도.
6. 검증 부호: column weight 3, row weight 5, rate 0.4, girth 10인 3D lattice 부호를 base로 사용.
7. 실험: 길이 6000 base에 압축비 1/2를 2단계 적용해 길이 3000·1500 부호(둘 다 girth 8) 생성, BPSK/AWGN·log-domain(min-sum류) 복호·최대 100 iteration.
8. 결과(성능): 길이 3000·SNR 2.2dB에서 BER 10^-7 (error floor 없음), 길이 1500·SNR 2.4dB에서 BER<10^-5; girth 10인 원 3D lattice 부호와 유사 성능.
9. FPGA 구현: (8192,4096) QC-LDPC 복호기를 Altera Stratix II EP2S60F1020C3에 VHDL/Quartus II 7.2로 설계, 8-bit 양자화, 165MHz, throughput 100Mbps, 부호이득(coding gain) 8.5dB.
10. 한계: 압축의 주목적이 부호 저장/구성 편의라 girth가 10->8로 낮아지고, 본문 대부분이 인코딩 손상된 중국어라 세부 검증 제한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC 부호구성(recursive compression) | `ecc_top.cpp` `Load_PCM()` + H-matrix | 부호 재구성 영역, 이식 난이도 높음(우리 QC-LDPC 고정) |
| base+shift 행렬 표현 | `ecc_data.h` `PCM_b`(base+shift) | 개념 유사하나 우리는 shift 압축 미사용 |
| log-domain(min-sum류) 복호 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | 이미 보유, 신규성 없음 |
| FPGA VHDL 복호기 구현 | 대응 없음 | 우리는 C++ bit-exact 시뮬, RTL 영역 밖 |

적용 가치: **낮음** - 핵심 기여인 재귀압축 부호구성은 부호가 이미 고정된 Prime ECC에 재검증 부담이 크고(§4 부호설계=높음), girth 감소로 정정이득도 불명확하다.
