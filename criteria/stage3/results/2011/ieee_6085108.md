> 총평: QC-LDPC partially-parallel 디코더의 extrinsic message 메모리를 병합해 면적을 크게 줄이는 HW 아키텍처 논문으로, Prime ECC와 병렬구조가 다르나 메모리 라우팅 절감 아이디어는 참고가치가 있음.

### B. 알고리즘 요약

1. 대상: IEEE 802.16e QC-LDPC (base matrix + circulant shift `S_j,k`, `Z×Z` 순환행렬), rate-1/2, length 576~2304, 19 modes, min-sum 복호.
2. 풀려는 문제: partially-parallel 구조는 병렬 접근을 위해 내부 메모리를 소형 블록 `J×K`개로 분할하는데, 소형 블록의 주변회로(peripheral) 오버헤드로 면적이 증가.
3. 핵심 기법 1단(Block Row Merging): 같은 block row의 `M_j,k`들을 한 memory group `G_i`로 병합 → K개 메시지 동시 접근으로 CNU 즉시 처리 가능, 대신 VN phase에서 정렬 buffer 필요.
4. 주소 모델: `RI(A)=A-RSI_j`, `CI(A)=(A+S_j,k-RSI_j) mod Z` (식3,4)로 row/column index 매핑.
5. Starting Address 결정: `SAv(G_j)=argmin[max((b-a) mod Z)]` (식6)로 그룹 내 블록 간 최대 거리(수명, life time)를 최소화 → 추가 buffer 수 최소화.
6. relative block delay `dR(M_j,k)=(SAv(M_j,k)-SAv(G_j)) mod Z` (식7), 그룹 최대 delay `dR(G_j)=max dR(M_j,k)` (식8)로 R_FIFO/W_FIFO 길이 산정(합은 `dR(G_j)` 상수).
7. HW 친화 변형: R_FIFO(읽기 정렬)+W_FIFO(쓰기 정렬)를 data distributor/combiner와 함께 배치.
8. MBS(Memory Block Selection) 알고리즘: `SAv` 정렬 후 FIFO 최대길이 `L` 제약 하 그룹 분할(Step4), 인접 그룹 간 블록 이동으로 비용 `N(G_i,G_{i+1})=dR(G_i)|G_i|+dR(G_{i+1})|G_{i+1}|` (식10) 최소화 refinement(Step5).
9. 결과: Z=96, L=4 설정에서 10개 memory group, extra delay element 148개, gate count 273k@0.13um(참조 [5] 420k 대비 ~35% 감소), 1.72mm² core, 86.55mW.
10. 한계: 정정능력·알고리즘 불변(순수 메모리 아키텍처), RSI/CSI 결정은 범위 밖, VNU가 (Z+2L) 사이클로 throughput ~4% 저하.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| extrinsic message 메모리 병합/라우팅 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC의 병렬 메모리 구조와 개념 대응(구조 상이) |
| min-sum CN/VN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 이미 보유, 메시지 소스일 뿐 |
| HW edge/버퍼 정렬(FIFO) | `GT.cpp` `Make_GT_HW()` | Graph Thinning MUX 선택과 유사한 HW edge 스케줄 관점 |
| H-matrix/QC 구조(circulant) | `ecc_top.cpp` `Load_PCM()` | QC-LDPC PCM 로드 대응 |

적용 가치: **낮음~중간** — Prime ECC는 z=32 고정 병렬·HCU·GT 등 자체 HW 구조를 이미 갖춰 802.16e 다중모드용 block-row merging을 직접 이식하긴 어렵고, 메모리 그룹 병합으로 면적을 줄이는 아이디어만 참고 가능.
