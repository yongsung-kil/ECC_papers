### B. 알고리즘 요약

1. 대상: 고정된 binary LDPC 부호의 **LP 복호**(Feldman LP)를 BSC에서 분석. LP는 ML-certificate·이산출력·해석 용이 장점.
2. 문제: error-floor를 지배하는 저확률 실패 noise 구성은 시뮬로 잡기 어려워 해석적 특성화 필요.
3. 정의: all-zero 전송 가정 하, LP가 all-zero 아닌 pseudo-codeword로 복호하게 만드는 최소 flip 집합을 **BSC-instanton**으로 정의(support 축소 시 all-zero로 복호).
4. 핵심결과 Lemma 7: LP 복호 실패 error vector의 support는 반드시 어떤 instanton의 support를 superset으로 포함 → 모든 실패를 instanton으로 완전 특성화.
5. cost/weight: pseudo-codeword의 `cost`(식5)와 BSC pseudo-codeword weight, median noise vector 개념 도입. `w_frac`(fractional weight) 하한 `d_frac/2 < w_min` (Lemma 1).
6. ISA: 충분한 flip을 가진 random 입력→LP→pseudo-codeword `Γ`→median `m` 계산→LP 재적용, weight 감소 반복. subset(1 flip↓) 모두 all-zero면 현재가 instanton.
7. 종결 보장 Theorem 1: cost/weight가 단조 감소, 입력 flip 수 `k`의 최대 `2k` step 내 종료(대개 `<k`).
8. FER 예측: log-log FER 곡선의 error-floor 기울기 = 최소 weight instanton 크기. instanton 통계 + Monte-Carlo 결합으로 FER 추정(식6,7).
9. 결과: (3,5)-regular Tanner[155,65,20]는 최소 instanton size 5(=Tanner trapping set), (3,6) MacKay[204]는 4. 10000회 초기화로 통계 수집, semi-analytical FER가 MC와 일치.
10. 한계: 순수 이론(정리+시뮬), HW 없음, LP 복호 전용, BSC hard만, 부호 자체 정정능력 개선 아닌 실패 진단 도구.

### C. Prime ECC 관련 모듈 핀포인트

|---|---|---|
| LP 복호기 자체 | 대응 없음 | Prime ECC는 min-sum 반복복호이지 LP 복호가 아님 |
| instanton/trapping-set 기반 error-floor 진단 | [corner.cpp](../Prime_ECC_3.1_Claude) (수렴실패 추적) | 개념적으로 수렴실패 분석과 닿으나 알고리즘 직접 이식 대상 아님 |
| BSC hard-decision 채널 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()` | 채널 모델 대응하나 논문은 부호/디코더 개선 아님 |
| 부호 최적화 metric(instanton-bar-graph) | 대응 없음 | 우리 QC-LDPC 부호는 고정, 재설계 대상 아님 |
