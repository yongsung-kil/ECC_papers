### B. 알고리즘 요약

1. 대상은 Tanner `[155,64,20]` (3,5) regular LDPC (`R≈0.4129`, girth `g=8`, `dmin=20`)와 IEEE 802.3an `[2048,1723,14]` (6,32) 코드, AWGN/BPSK 채널.
2. 문제: sub-optimal iterative decoding(SP/MP)에서 absorption set(trapping set의 지배적 부분집합)이 error-floor를 유발.
3. absorption set을 순서쌍 `(a,b)` (a=크기, b=extrinsic message degree)로 정의; Tanner 코드의 지배 집합은 (8,2)로 codeword 유사 구조.
4. 선형 모델: 변수노드 출력 `x`, 입력 `y`, 순열행렬 `C`, 변수노드 합산행렬 `V`로 iteration 전개 `xI = Σ(VC)^i λ + ...` (식7).
5. `VC`의 최대 고유값 `µmax≈1.7870`과 고유벡터 `vmax`가 실패 메커니즘을 지배; `β=xI^T·1 ≤ 0`이면 오류.
6. 오류확률 `PAS = Q(mβ/√σβ²)` (식28)을 A,B,C,D 위상 상수로 근사.
7. 정밀화(§III): `PAS=Pr(max_j xI,j ≤0)`로 조건 강화, 초기 iteration에서 spectral 근사 대신 행렬식(7) 직접 사용.
8. 핵심 기법(§IV-A): 초기 4 iteration 동안 모든 불만족 CN이 되돌리는 LLR을 factor 2배 boosting, 이후 표준 SP 복귀.
9. 결과: 6.5dB에서 error-floor를 약 1자릿수 개선, waterfall 영역 무영향; (8,2) 집합 오류 기여율 약 10%p 감소.
10. 부가(§IV-B): LLR clipping threshold를 ±10→±100→∞로 확장하면 absorption set 지배력이 감소하는 것을 확인.
11. 한계: HW 미설계, 짧은 regular 코드 시뮬만, boosting 정도/구간이 SNR·decoder 설정 종속 휴리스틱.

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 초기 iteration LLR boosting (불만족 CN) | decoder.cpp LDPC_Decoder(), C2V_Cal() | iteration 루프 내 CN 출력 스케일 삽입 지점 |
| iteration별 LLR 스케일/threshold | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 이미 iteration 구간별 LLR 테이블 보유, boosting factor 추가 여지 |
| CN min-sum 연산 (min-sum 확장 언급) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 논문이 min-sum 확장 가능 명시 |
| LLR clipping 범위 확장 | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | 양자화 magnitude 테이블 범위 조정과 대응 |
| absorption set 이론/열거 | 대응 없음 | 부호 고유 구조 분석, 코드베이스에 해당 툴 없음 |
```

적용 가치: **중간** — 초기 iteration LLR boosting은 이미 보유한 적응형 LLR 테이블에 factor를 얹는 방식으로 저비용 이식 가능하나, 효과 검증이 짧은 regular 코드 한정이라 고rate QC-LDPC 재튜닝·재검증 부담이 있다.
