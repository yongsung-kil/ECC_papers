### B. 알고리즘 요약

1. 대상은 partially-parallel LDPC 디코더의 `interleaver`(메모리 매핑) 설계 문제로, `P`개 PE가 `B=P`개 메모리 뱅크를 매 time instance마다 충돌 없이 read/write하도록 데이터를 배치하는 것이 목표.
2. 기존 방식(구조화 코드 설계, 추가 라우팅 HW, simulated-annealing 매핑, node/edge coloring)은 코드성능 저하·HW 증가·종료시점 예측불가·"데이터당 뱅크 1개" 제약 등 한계.
3. 핵심 아이디어: 데이터를 read한 뱅크와 다른 뱅크로 write하는 multiple read/write 전략을 허용하여 매핑 자유도를 높임.
4. 매핑 행렬(`P`행×`N`열, 각 열은 read데이터/read뱅크/write뱅크 3개 서브열)을 삼분 그래프 `G=(TR ∪ TW ∪ L, E)`로 모델링(`TR`=read시각, `TW`=write시각, `L`=데이터).
5. 제약: (1)각 time instance마다 모든 뱅크 정확히 1회 사용, (2)어떤 데이터의 마지막 write 뱅크 = 다음 첫 read 뱅크 → `related edges`는 같은 색(뱅크)이어야 함.
6. 병렬 디코딩 특성상 모든 time node degree가 `dt=P`로 동일 → 그래프는 `TR`,`TW`에서 semi-regular.
7. 2-step coloring: Step I 분할(partitioning) — process of addition/deletion을 교대 적용해 각 time node degree가 `dt'=2`(짝수 Lemma1) 또는 `dt'=1`(홀수 Lemma2)인 regular partition으로 분해.
8. Step II 채색 — 각 regular partition을 최대 2색으로 색칠(색=메모리 뱅크), related edge에 같은 색 부여해 제약 충족.
9. 결과: K=6,P=B=3 교육적 예제 하나로 충돌 없는 매핑 행렬 도출을 제시(정량 실험·합성·throughput 없음).
10. 한계: 실제 HW 미구현, gate count/throughput/코드 rate·length 부재, 상호연결 네트워크 복잡도는 future work.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 메모리 뱅크 매핑/interleaver 설계 | 대응 없음 | Prime ECC는 z=32 고정 QC구조로 매핑이 base matrix에 내장, 별도 자동매핑 불필요 |
| partially-parallel 데이터 라우팅 | 대응 없음 | HW 라우팅은 GT.cpp(edge/MUX)로 처리, 논문의 뱅크 충돌 문제와 층위 다름 |
| 디코더 알고리즘/부호 성능 | 대응 없음 | 논문은 알고리즘·부호와 무관한 매핑 방법론 |

적용 가치: **낮음** — 부호설계·디코더 성능·soft-read 어느 것도 개선하지 않고, 고정 QC-LDPC(z=32) 구조에선 매핑이 base matrix로 결정되어 방법론 이식 필요성이 거의 없음.
