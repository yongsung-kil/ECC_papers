### B. 알고리즘 요약

1. 대상은 finite-field 기반 binary structured (7,p) regular QC-LDPC (예: (961,721) rate≈0.75, RC-constraint로 girth≥6).
2. 문제: SPA는 성능 좋으나 복잡, WBF/RBI-MLGD 계열은 저복잡도지만 성능 열세 → 저복잡 유지하며 성능 개선 목표.
3. 모델: AWGN+BPSK, 수신 `y`를 `A,b` 파라미터로 `2^b-1` 구간 균일 양자화(예제 8-bit)해 정수 신뢰도 `q_j`로 변환.
4. 핵심1(turbo-like): 반복 중 갱신된 `R_j^(k)` 대신 **초기값 `R_j^(0)`를 고정 입력**으로 사용, VN 출력은 새 extrinsic만 반영 → `R_j`가 고정 구간 유지(turbo 디코더 유사).
5. 핵심2(scaling): 총 extrinsic `E_j`에 scaling factor `α`를 곱해 `E_j^(k)=⌊α·Σ(2q-1)⌋`로 수정, `α`는 discretized density evolution로 최적화(예제 α=3.5).
6. 갱신식: `R_j^(k+1)=R_j^(0)+E_j^(k)`, syndrome `s=zH^T`로 조기종료, 정수·논리연산만 사용.
7. 구현 디테일: scaling은 `n`개 lookup table(크기 2×27)로 대체 → 추가 복잡도 무시 가능.
8. 복잡도(표I): 반복당 `2δ+n+m` binary op + 6 integer add + n LT (SPA는 6δ log 필요), majority-logic 계열이라 곱셈·log 없음.
9. 결과: (961,721) 부호 8-bit 양자화에서 RBI-MLGD 대비 BER=1e-5 부근 ~0.3dB gain, SPA 대비 ~0.6dB 손실.
10. 한계: HW/RTL·gatecount 미제시 시뮬만, finite-field structured 부호 전용, α 부호별 재튜닝 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| MRBI-MLGD 반복 복호 루프 | decoder.cpp LDPC_Decoder() | majority-logic 계열이라 min-sum CN/VN 루프를 대체하는 별도 복호기 (직접 교체 아님) |
| CN extrinsic (syndrome 기반) | decoder.cpp C2V_Cal() | min-sum min1/min2 대신 syndrome XOR + 정수합, 연산 형태 다름 |
| 양자화/scaling lookup table | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | scaling factor α의 LT 구현은 iteration별 LLR 테이블 구조와 유사 (부분 대응) |
| syndrome 조기종료 | full_CRC.cpp / partial_CRC.cpp | 우리는 CRC 조기종료 보유, 논문은 syndrome=0 종료 (기능 유사) |
```

적용 가치: **중간** — 정수·논리연산 위주 저복잡 복호기라 HW 친화적이나, Prime ECC는 이미 min-sum을 보유하고 본 알고리즘은 SPA/min-sum 대비 성능 손실이 있어 전면 대체보다 저전력 대안 참고 수준.
