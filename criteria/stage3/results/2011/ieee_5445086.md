### B. 알고리즘 요약

1. 대상: regular `(N,K)` binary LDPC, 예제 대표 rate `1/2`(`N=2M`), AWGN + BPSK 채널. 여러 크기의 H matrix(matrix A~G) 테스트.
2. 문제: 실시간 LDPC 복호에 통상 VLSI가 쓰이나 비유연·비확장. 유연한 SW 기반 병렬 디코더를 멀티코어에서 실현하려는 것이 목표.
3. 알고리즘: 확률영역 Sum-Product(SPA/belief propagation). horizontal(CN→BN, 식1~2)·vertical(BN→CN, 식3~4) 두 kernel의 반복. `float` 연산으로 VLSI 대비 정밀도·BER 유리하다고 주장.
4. 핵심 데이터구조: Tanner graph를 `HBN`(row-major, kernel1용)·`HCN`(col-major, kernel2용) 두 컴팩트 스트림으로 분리 저장 → 병렬 접근에 적합.
5. Forward-Backward recursion으로 CN 업데이트 연산·메모리접근을 quadratic→linear로 축소, arithmetic intensity(연산/메모리접근)를 SPA 대비 개선(`FBA≈3`).
6. 멀티코드워드 병렬: Tanner graph가 모든 코드워드에 공통이므로 여러 codeword를 SIMD/SWAR(128-bit)로 동시 복호.
7. GPU(CUDA): thread-per-node, 128 threads/block, warp=32. global memory latency(400~600 cycle)를 coalesced access(half-warp 16 thread 1회 접근)로 완화, 주소 permutation(식13~20) 적용.
8. CELL/B.E.: PPE가 6 SPE에 오프로드, DMA double-buffering으로 연산/전송 overlap. small/large single-SPE 모델(LS 256KB 초과 시 Tanner graph 분할).
9. 결과(10 iter): CELL 최대 약 `69.5 Mbps`(중소형 코드), GPU 약 `10~40 Mbps`(대형 코드, 8-bit packing 시 최대 40.1 Mbps), x86 OpenMP는 `<2.6 Mbps`. 일부 VLSI에 근접.
10. 한계: HW 미설계(SW 디코더), throughput이 VLSI 대비 여전히 낮고 정밀도도 32/8-bit float, min-sum·NAND ECC와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| HBN/HCN 컴팩트 데이터구조 | 대응 없음 | Prime ECC는 QC-LDPC circulant 구조 고정, 임의 Tanner graph 스트림 인코딩 불필요 |
| coalesced memory / DMA 병렬화 | 대응 없음 | GPU/CELL 특화 SW 최적화로 z=32 HW pipeline과 무관 |

적용 가치: **낮음** — GPGPU/CELL용 float SPA SW 디코더 병렬화로, 정수 min-sum HW 기반 Prime ECC 3.1과 플랫폼·알고리즘이 달라 이식 대상이 사실상 없음.
