### B. 알고리즘 요약

1. 대상: WiMedia UWB QC-LDPC(802.16e 유사), `z=30` circulant, 부호길이 1200bit, 4가지 rate 0.5/0.625/0.75/0.8, irregular(CN degree 4~15, VN degree 2~8).
2. 문제: 완전병렬은 면적 과다, 무선 고속(수백Mbps~1Gbps)에서 저면적·multi-rate 동시 지원 필요.
3. 모델: BP(belief propagation) 소프트 메시지 교환, 입력 LLR soft bit width 6bit, 내부 메시지 9bit, 16-QAM 채널.
4. 핵심기법 1: layered BP로 sub-iteration당 CN subset 갱신 즉시 VN 전파 → iteration을 2-phase 대비 약 절반(15→6)으로 감소.
5. 핵심식 의미: 식(5)-(7) 레이어드 갱신 + CN은 min-sum 근사식(8)로 `tanh`/`tanh⁻¹` 제거해 LUT/승산기 회피.
6. 핵심기법 2: 3-parallel check node PE — Min4 블록이 min1/min2 추적, irregular degree는 미사용 입력에 최대값 할당해 대응, 3배 throughput.
7. 부수 트릭: multi-rate는 내부 execution-order RAM/shift RAM 내용만 재기록해 지원(H-matrix 재합성 불필요).
8. 최적화: check-bit(R RAM) 저장을 전체 메시지 대신 min1/min2+최소값 주소+sign만 저장 → rate 1/2·CN degree 6에서 48%로 축소.
9. 결과: FPGA(Virtex4) 최대 360Mbps, 90MHz, 총 메모리 34,090bit(압축 후), slices 15096/LUT 25634; iteration 15→6.
10. 한계: 무선 저rate(0.5~0.8)·단길이(1200) irregular QC-LDPC 전제, 시뮬+FPGA만, NAND 고rate·soft-read 비용 논의 없음.

| check-bit 압축(min1/min2+addr+sign) | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` 이터레이션 루프 | HW 메모리 절감 참고, Prime는 QC-LDPC 최적화 이미 보유 |
| irregular degree 대응 CN PE | [decoder.cpp](../Prime_ECC_3.1_Claude/) `VN_Cal_HD()` 등 | Prime HCU(비대칭 col) 유사 문제, 다른 접근 |
| multi-rate execution-order RAM | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | Prime는 단일 고정 QC-LDPC → 다중rate 유연성 실익 낮음 |

적용 가치: **낮음** — layered min-sum·min1/min2 압축은 Prime가 이미 보유, multi-rate 유연성은 단일 부호 고정 Prime에 불필요하며 무선 저rate·단길이 전제라 NAND 부호와 정합 낮음.
