### B. 알고리즘 요약

1. 대상: WiMAX QC-LDPC(6 rate=1/2~5/6, 19 length=576~2304, expansion factor z=24~96), layered normalized min-sum 복호, 6bit 양자화.
2. 문제: 기존 partial-parallel 디코더는 z-by-z sub-block 1개(또는 radix-2로 2개)씩 처리해 병렬도 한계·데이터 충돌.
3. 핵심 아키텍처: macro-block(MB)=수평 8개 연속 sub-block을 MBPE(macro-block processing engine)로 매 클록 동시 처리(96 MBPE array).
4. layered 알고리즘: 식(1) `Omn=Ln-Λmn[t-1]`, 식(2) `Λmn[t]=θ·min|Omn|`(normalized factor `θ=0.75`), 식(3) `Ln=Omn+Λmn[t]`.
5. VMU(VAR update)/AMU(APP update) 2 phase로 나눠 인접 layer의 VMU와 AMU를 overlap해 throughput 향상.
6. 전용 행렬 재정렬(3 step): 24열을 3 sub-matrix로 분할, column reordering으로 zero MB 수 `N` 최대화 → iteration당 cycle수 `Q` 감소(식4).
7. step2: 각 sub-matrix에 static column 1개 확보로 shifter 8→7 감소(16.8k gate 절감), step3: row reorder로 read-write 충돌 회피.
8. 재정렬 알고리즘: greedy division으로 상·하한 추정 후 integer cutting + pattern set 달성 가능성 검사로 max(n) 탐색.
9. 결과: iteration당 12~24 cycle(선행 [5] 대비 2x~4.3x 병렬), 메모리 48,480bit(72.5k 대비 감소), gate 562k@0.13um, 1142Mbps@62.5MHz, 144mW.
10. 한계: WiMAX 다중 rate/length 유연성 전용, 정정능력 개선 없음(순수 HW throughput/전력), Banyan 순열망 96x96 기반.

| macro-block 8 sub-block 병렬 처리 | 디코더 HW 모델(z=32 row 병렬, GT) | 병렬 처리 단위 개념 유사하나 z·구조 상이, 재정합 필요 |
| 전용 base matrix 재정렬(zero MB 최대화) | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 다중 rate 대응 재정렬이라 단일고정 PCM에는 부분 적용 |

적용 가치: **중간** — 고병렬 layered 디코더 HW 아키텍처와 메모리 절감·전력 개선 기법은 참고 가치가 있으나, WiMAX 다중 rate/length 유연성 위주 설계라 Prime ECC의 단일고정 QC-LDPC(z=32, HCU/GT)에 그대로 이식하기엔 구조 재정합 부담이 있음.
