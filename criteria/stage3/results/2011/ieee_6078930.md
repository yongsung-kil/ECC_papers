> 총평: DVB-T2 IRA-LDPC의 block-circulant 구조를 이용한 conflict-free 메모리/프로세서 아키텍처(flooding, M=360, 130nm 합성)로 알고리즘 무관 유연성이 장점이나, Prime ECC의 z=32 QC 구조·layered 스케줄과 정합 어렵고 NAND 무관, 이식성 하.

### B. 알고리즘 요약

1. 시스템: DVB-T2 IRA-LDPC, `N=64800/16200`, rate `1/4~5/6`, AWGN/QPSK, BP 복호(SPA/MSA 선택). lifting `M=360`.
2. 문제: DVB-S2/T2 반병렬 디코더가 메모리 접근 충돌을 겪어 dual-port RAM이나 ad-hoc 로직으로 해결 → 유연성·면적 손해.
3. 모델: `H=[H1 H2]`, `H2`는 dual-diagonal(IRA), `H1_q`는 block-circulant(열 `j`는 열0을 `jQ` 순환 shift), `Q=(N-K)/M`.
4. 핵심 기법: 기울기 `Q`로 감기는 대각선 에지집합을 **strand** `S_{q,d}`로 정의, 서로 교차하지 않고 `H1`의 모든 1을 덮음.
5. 핵심 식: `S_{q,d}={((r_d(q)+jQ) mod (N-K), j)}` — 한 strand=M 에지가 RAM 한 word에 정렬되어 병렬 read/write 시 충돌 없음.
6. 확장: node group(같은 순환그룹의 M 노드) 단위 병렬 처리, group은 직렬. cyclic shifter가 offset만큼 순환 정렬.
7. 구현 디테일: 단일포트 RAM(depth `Ns+3Q+Q̂`, 최대 972 word, b=4bit면 ~1.4Mbit), ROM에 code 파라미터(75Kbit), ACU가 주소/offset 계산·재사용, 파이프라인.
8. 알고리즘 독립: FU 내부 연산만 교체하면 SPA/Min-Sum/λ-Min 등 임의 CN 알고리즘 사용 가능(기본 아키텍처 불변).
9. 결과: 130nm 200MHz 합성, `(N=64800,R=3/5)`가 한계 케이스로 50 iter시 59Mbps(목표 60.4Mbps는 45 iter로 충족). SPA 5bit는 float 대비 0.2dB, MSA 4bit는 0.1dB 이내.
10. 한계: FU 게이트수·전체 면적 미보고(후속 저널 예정), flooding이라 수렴 iter 많음, DVB-T2 특정 구조 전용, NAND/soft-read 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| strand 기반 메모리 배치/충돌회피 HW arch | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z=32 row 병렬+GT/HCU 구조, M=360 strand 메모리맵과 정합 어려움 |
| cyclic shifter / node-group 병렬 처리 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | 순환 shift 개념은 QC 공통이나 라우팅/메모리 세부는 상이 |
| algorithm-agnostic FU (SPA/Min-Sum 교체) | `decoder.cpp` `CNU_Update_New_Mag()` 등 | Prime ECC는 min-sum 고정 구현, 유연성 아이디어 자체는 이미 min-sum 확정 |
| H-matrix(block-circulant/IRA) 로드 | `ecc_top.cpp` `Load_PCM()` | IRA dual-diagonal H2 + block-circulant H1, NAND 고rate QC와 구조 다름 |

적용 가치: **낮음** — conflict-free 메모리맵은 DVB-T2 IRA(M=360, block-circulant) 구조 전용 최적화라 Prime ECC의 z=32 QC HW 모델로 직접 이식 이득이 작고, 알고리즘·정정성능 개선이 없어 NAND ECC 관점 가치 낮음.
