### B. 알고리즘 요약

1. 대상: BPSK/AWGN 상 irregular LDPC 부호(예 PEGirReg252x504, (1944,972) IEEE802.11, (1000,499)). trapping set 탐색 문제.
2. 문제: brute-force는 크기 `t`·길이 `n`에서 `C(n,t)`로 폭발; 기존 탐색기는 길이 ~500·크기 11까지로 제한적.
3. 핵심 관찰: 임계값 최적화된 고rate irregular 부호는 degree-2 변수노드 비율이 크고(rate 8/9서 31%), 이들이 `2-chain`(길이 2k = `(k,2)` trapping set)을 형성해 dominant TS의 핵심을 이룸.
4. 정의: `(a,b)` trapping set = 변수노드 `a`개, 홀수차수(unsatisfied) 검사노드 `b`개. dominant = 주어진 `a`에 대해 최소 `b`.
5. Algorithm 1(기존 [7]): 짧은 cycle을 반복 확장. 각 변수노드가 satisfied 검사노드 2개에 연결되는 조건이라 2-chain을 못 찾음.
6. Algorithm 2(신규): `(a-1,b)` TS의 unsatisfied 검사노드에 degree-2 변수노드 `v`를 붙여 확장 — v가 다른 unsatisfied CN에도 연결되면 `(a,b-2)`로 b 감소.
7. Algorithm 3: 초기입력 = 짧은 cycle + 저차수 변수노드 + 저ACE(`ACE = Σ(d_i - 2)`) cycle → Alg.1 → Alg.2 순으로 결합.
8. Theorem 1: 특정 길이의 2-chain을 피하기 위한 `m` vs `n_v2` 필요조건 제시(부호설계 가이드).
9. 결과: PEGirReg252x504서 [8]의 7시간 대비 5~15분, (1944,972)서 [2]의 5일 대비 5~45분; [2]보다 많은 TS 발견. 3-bit min-sum 디코더 error-floor를 importance sampling으로 추정→Monte Carlo와 일치.
10. 한계: 순수 TS 탐색 도구로 디코더/HW 미설계; degree-2 비율 낮은 부호에선 Alg.2 단독으로 전부 못 찾을 수 있음.
