### B. 알고리즘 요약

1. 대상은 QC-LDPC(sub-matrix size z, 우측 순환천이 identity)로 rate 0.75~0.95, N=2016~16128, WiMAX 802.16e degree distribution 사용.
2. 문제: 가변 rate·블록길이·변조를 하나의 HW로 지원하면서 병렬도를 자유롭게 스케일하고 메모리 충돌을 없애는 flexible layered 디코더가 필요.
3. H를 `l×c` base matrix에서 반복적으로 확장: 각 unit을 z×z 순환천이 identity로 치환하고, Algorithm A가 짧은 cycle(size `e`,`e+2`)을 최소화하는 shift set을 선택.
4. 핵심: layered(row subset 순차) min-sum 디코딩. sub-iteration `L`개로 한 iteration 구성, 계산 직후 해당 VN에 즉시 전달해 수렴 가속.
5. 반복구성 덕분에 각 확장 sub-matrix의 열마다 '1'이 유일 → `z1·z2` 행을 데이터 충돌 없이 병렬 처리 가능.
6. CPU(check unit)는 min/2nd-min 레지스터(R_M/R_M2)와 flagging으로 min-sum 수행, v2c sign XOR로 수렴(early termination) 신호 계산.
7. 구현 트릭: distributed local memory(각 VPU에 RAM_Sv), 18-bit LLR, MUX_shortening으로 shortened bit를 0 고정, saturation.
8. FLEX_INPUT/MQAM_OUTPUT: 큐 기반 재그룹핑으로 가변 rate(R1~R2 step)·정보워드 길이(shortening)·QAM 차수(Q=7)를 동일 HW에서 on-the-fly 지원.
9. 결과: Virtex5 FPGA에서 15~1569Mbps, 병렬도 p=21시 throughput 20.6배 vs LUT 6.24배(효율 3배+), 고SNR 평균 iteration 2.8, WiMAX rate-3/4 대비 BER 동등.
10. 한계: 성능(정정능력) 개선 없이 동등, gatecount·ASIC 없음(FPGA만), soft-read 비트폭 최적화 미언급, WiMAX 유연성 위주.
