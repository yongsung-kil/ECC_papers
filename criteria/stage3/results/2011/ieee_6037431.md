### B. 알고리즘 요약

1. 대상은 WiMAX(802.16e)/WiFi(802.11n)의 구조적 QC-LDPC(base matrix HBASE + Z×Z circulant, rate 1/2~5/6, length 576~2304, Z=24~96)를 하나의 IP로 복호.
2. 문제: min-sum 직렬 check node는 read/write latency가 `dc` clock이라 layered 스케줄·고throughput 실현이 어려움. 기존 병렬 CN([8][9])은 dc=6/8 고정.
3. 복호는 layered(TDMP) min-sum: VN update를 `αij=αj-βij`로 합쳐 horizontal/vertical scan을 단일 scan으로, iteration 약 50%↓(throughput 2배).
4. 핵심기법: "Tree-way" 완전병렬 CN — VN(leaf)에서 tree 순회로 min/second-min 계산, DVC(Direct VN Comparison)→MSC(Multiple Shuffled Comparison, 순환shift ACS network)→EC(Extrinsic Calc, shift 1/2) 3종 스테이지로 dc≤32까지 일반화.
5. CN 복잡도 `dc/2` comparator(기존 [8] `dc(1+0.5(log2 dc-1))`, [9] `dc+log2 dc-2` 대비 소형), sign product도 tree-way(XOR) 구조.
6. Table II로 각 dc별 clock cycle 수 `Ncc`·permutation 규정, SM(switch matrix) memory에 중간결과 저장·binary label 주소로 재활용.
7. Block-level Channel Memory: 단일 bank를 Nb circulant 그룹 병렬 bank로 분할, 같은 sub-iteration에서 충돌 최소로 접근(데이터 독립 열 판별 절차), WiMAX rate 전체를 8 bank 고정으로 지원(3/4-B는 4 clock 페널티).
8. Sub-block level memory로 스케일러블성 확보: `Z mod P=0`인 P에 대해 두 인접 VN을 `Z/P` 거리로 배치, circulant shift와 무관하게 P개 VN을 1 clock에 접근(Z=32,P=4 예시 검증).
9. 결과(130nm ASIC 합성, 300MHz, 15 iter, 7-bit 양자화): throughput WiMAX 74~240Mbps / WiFi 141~249Mbps, area 2.76/2.74 mm2, TAR·DE 지표에서 다수 기존 대비 우수, P=96 시 1.7Gbps@4.5mm2.
10. 한계: error 정정성능 개선이 아니라 HW throughput/flexibility 최적화, 다표준 유연성 비용으로 CN이 area의 ~50%, 실측(silicon) 아닌 합성 수준.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Tree-way 완전병렬 CN(min1/min2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum CN 연산부에 대응; dc 가변 병렬화 아이디어 참고 가능 |
| layered(TDMP) min-sum 스케줄 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC의 Dual-Update(parity even/odd) 교대 스케줄과 유사 목적 |
| block/sub-block Channel Memory 조직 | 디코더 HW 모델(z=32, HCU, GT) | z=32 병렬단위·라우팅 정합 검토 대상(중 난이도) |
| ACS 순환 permutation network | `GT.cpp` `Make_GT_HW()` | edge/MUX 선택 그래프 씨닝과 개념적으로 유사 |
| soft LLR(7-bit) 양자화 | `channel.cpp` + `PARAM_LLR` | Prime ECC는 hard/2SD/3SD, 7-bit soft는 미지원(정합 낮음) |

적용 가치: **중간** — dc 가변 완전병렬 check node와 충돌회피 sub-block memory 조직화는 Prime ECC 디코더 병렬화/메모리 라우팅에 참고가치가 있으나, 다표준 유연성을 위한 오버헤드가 크고 error 성능 개선은 없다.
