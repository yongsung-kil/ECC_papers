### B. 알고리즘 요약

1. 대상은 QC-LDPC(802.11n/16e, `P≤96` CPM), 6/6/8-bit 양자화 NMSA 층별(layered) 디코더; flash/10GbE 등 `10^-12` 이하 저 error-floor 요구.
2. 문제: error floor는 trapping set 때문. 부호설계로 회피는 조합적으로 어려워 decoder-based로 해결하되 기존 post-processing은 dominant trapping set 사전지식 필요.
3. 핵심 기법: 원 디코딩 실패 시 미충족 CN 최소 집합 `S_min`에 이웃한 candidate VN을 하나씩 최대 LLR로 부호반전(flip)해 재디코딩하는 trial-and-error backtracking(1차) + hit-trial 재활용(2차).
4. 기여 1(복잡도): [11]의 correlation(다비트 다입력 덧셈) 기반 hit-trial 선택을 `hard-decision LLR와 trial 출력 간 Hamming distance` 최소화로 대체 → 단일비트 연산으로 면적/임계경로 대폭 절감.
5. `y` generator가 선택된 VN을 최대 크기·반대 부호 LLR로 세팅해 error-inducing 패턴을 교란.
6. 기여 2(latency): early trap detection(미충족 CN 수 반복/진동을 임계 `Th_rep`로 감지해 조기중단) + VN-degree flipping(irregular 부호에서 저차수 VN 우선 flip)로 iteration 최대 75%, trial 수 감소.
7. HW: 재구성 QC-LDPC 디코더에 self-contained backtracking 모듈을 인터페이스; y-generator, difference detector(XOR+96b OR), Hamming distance unit, TNSU(parallel one-bit search), MMU(페이지 할당/회수).
8. optimum state storage: current/previous/optimum 3버전 hard-decision + tag memory(2버전)로 최소 미충족-CN iteration 상태 유지; syndrome을 QSN으로 순환시프트해 tag 생성.
9. 결과: TSMC 0.18um 합성, 디코더 200MHz/backtracking 400MHz(클록 penalty 0), 게이트 7.5K/10.3K, 메모리 오버헤드 13%/46%; 1차 backtracking이 error floor 0.5~1.5 orders, double이 추가 0.5 order 저감; (2304,1152) 5-iter시 344Mbps.
10. 한계: backtracking invoke 시 worst-case latency 매우 큼(평균은 미미), error-floor 전용(waterfall 무개선), 별도 메모리 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 반복 디코딩 루프·재디코딩 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | backtracking은 이 루프를 반복 호출하는 래퍼로 이식 가능 |
| candidate VN flip / LLR 최대치 세팅 | `decoder.cpp` `VN_Cal_HD()`, `ecc_data.h` `PARAM_LLR` | VN 초기 LLR 강제 세팅 후크 필요, 양자화 테이블 재사용 |
| unsatisfied-CN(syndrome) 카운트 | `decoder.cpp` `CNU_Update_New_Mag()`, `full_CRC.cpp` | 미충족 CN 최소집합 추적은 신규 로직, syndrome 계산부에 연결 |
| syndrome 순환시프트(QSN)로 tag 생성 | `GT.cpp` `Make_GT_HW()` | edge/MUX 라우팅 유사, tag memory는 신규 |
| optimum state/tag memory 관리(MMU) | 대응 없음 | Prime에 backtracking용 상태저장·MMU 미존재, 신규 HW |

적용 가치: **높음** — NAND flash를 직접 겨냥한 code-agnostic error-floor 저감 HW 모듈로, Prime의 QC-LDPC 디코더 루프에 애드온으로 붙여 waterfall 손실 없이 floor를 낮출 수 있으나 상태저장/MMU 신규 HW와 worst-case latency 관리가 과제.
