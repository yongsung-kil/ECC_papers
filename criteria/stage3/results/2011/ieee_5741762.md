### B. 알고리즘 요약

1. 대상: BEC(binary erasure channel)에서 iterative decoding하는 유한길이 LDPC 부호, rate-`1/2`, length `500`, PEG로 생성.
2. 문제: 작은 크기 stopping set이 error floor를 유발하며 기존 PCM 확장법([3],[1])은 최소 크기 stopping set을 잘 제거하지 못함.
3. 모델: 디코딩 오류확률 `Pr(error|H)=Σ D_i·ε^i·(1-ε)^(n-i)`, 각 stopping set이 `(ε/(1-ε))^|S|` 만큼 오류에 기여.
4. 핵심기법: 새 parity-check 행에 1을 놓을 열 인덱스를 고를 때, 제거될 stopping set과 재활성될 stopping set의 개수 차 대신 `(ε/(1-ε))^|S|` 합의 차를 최대화하는 열 선택.
5. 식 의미: 작은 stopping set일수록 오류기여가 크므로 가중치를 부여해 우선 제거 → 최소 크기 set부터 없앰.
6. 확장: `w_max`(새 행 최대 weight)까지 반복해 한 행 생성, 반복 적용으로 여러 행 추가 가능(Algorithm 1).
7. 구현 디테일: 새 행을 추가(rate 손실)하거나, 원 PCM의 하위 행을 교체(rate 무손실)하는 두 방식 모두 실험.
8. 최적화: stopping set 열거는 [4] 기법 사용, 각 ε마다 scheme 정의.
9. 결과: 새 행 1(2)개 추가 시 최소거리 15(17)로 기존 method1/2의 8~10 대비 대폭 향상, WER이 타 방식 대비 최소 한 자릿수(order of magnitude) 개선.
10. 한계: BEC 전용, ε 사전지식 요구(단 10% 오추정에도 성능 유사), HW/일반 채널 미검증.
