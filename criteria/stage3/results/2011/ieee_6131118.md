### B. 알고리즘 요약

1. 대상 코드: IEEE 802.16e/802.11n 구조적 QC-LDPC (base matrix `Hb` 24열 + 순열행렬 확장), 길이 576~2304 / 648~1944, rate 1/2~5/6.
2. 문제: BP/min-sum HW 디코더는 메모리·면적 큼, 전용 HW는 유연성 부족 → 모바일 SW 디코더로 코드길이·rate 유연성 확보하되 속도가 관건.
3. 채널/판정 모델: 경판정 기반 비트플리핑, LLR 아닌 신뢰도비(reliability ratio) 사용.
4. 핵심 기법: IRRWBF(Implementation-Efficient Reliability Ratio Weighted Bit-Flipping)를 초기화/체크노드/변수노드/결정 4단계로 분해, 각 단계가 데이터 독립적이라 SIMD 병렬화 가능.
5. 핵심 식: `Oc ≈ N[3Wc + i]` — 총 연산량이 코드길이 `N`과 iteration `i`에 선형, 나눗셈을 곱셈으로 근사해 add/mul만 남김.
6. 자료구조 트릭: 행 packing `Hr` (N-K)×Wr, 열 packing `Hc` Wc×N 로 H를 압축, packed 벡터 `Tpa`, `Sipa`, `EPpic` 형태로 SIMD 명령 1회에 d개 데이터 처리.
7. 구현 디테일: 64bit 정수 SIMD(8개 병렬) / 128bit 부동소수 SIMD(16개 병렬), 정수 SIMD는 스케일링 후 반올림 근사.
8. 최적화: 최댓값 index 탐색(argmax)도 스트리밍 SIMD 명령으로 결정 단계에 병합.
9. 결과: 2304,1/2,i=50에서 2.6ms(64bit)/2.475ms(128bit), 46.55Mbps, 비-SIMD 대비 1800~2000%+ 속도향상, 처리메모리 50%+ 절감.
10. 한계: 순수 SW(모바일 SIMD)만 검증, BER는 SP/min-sum보다 나쁨, HW/RTL 미설계, 정정능력 개선 없음.
