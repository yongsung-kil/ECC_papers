> 총평: check node merging으로 punctured QC-LDPC의 avg iteration을 줄여 병렬도↓·throughput↑한 FPGA min-sum 디코더; NAND는 rate-compatible이 아니라 이득 제한적.

### B. 알고리즘 요약

1. 대상은 IEEE 802.16e/802.11n의 rate-compatible QC-LDPC (mother rate-1/2를 puncturing해 rate 0.5~0.85), min-sum 반복복호를 FPGA로 구현.
2. 풀려는 문제: punctured node의 LLR을 0으로 초기화하는 고정rate BP는 수렴이 느려 iteration이 많음.
3. 핵심 기법 `check node merging`: punctured degree-2 variable node에 연결된 purged check node들의 edge를 이웃 check node로 병합(Fig.1 3단계)해 메시지 전파 속도를 높이고 수렴 iteration을 줄임.
4. 병합 결과 parity-check 부분행렬의 column weight가 1보다 커질 수 있어, 제안 architecture는 bit-sum을 직접 계산하지 않고 `bit-sum 차분(difference)`을 누적해 old bit-sum에 더하는 방식으로 처리(overlap 대응).
5. Architecture 파라미터: 확장인자 Z, 병렬도 Q, 최대 check degree C, bit-sum 워드길이 `wq`, min-sum magnitude 워드길이 `wr`, edge index `we`.
6. CFU는 min-sum 알고리즘상 min1/min2 두 magnitude와 최소 index·parity·sign만 저장하고, old c2v 메시지를 빼 v2c 생성 후 새 min-sum과의 차분을 출력. programmable `offset`으로 min-sum 과대추정 보정.
7. cyclic barrel shifter로 bit-sum을 CFU로 라우팅하고, 여러 CFU가 같은 bit-sum을 갱신할 때 shift register를 reload해 처리(bit-sum update unit, Fig.7).
8. 메모리: bit-sum을 Q개 뱅크(wq bit), min-sum을 Q개 뱅크(we+1+2wr bit)로 조직.
9. 결과: check merging으로 avg iteration 감소→더 낮은 Q로 구현 가능. 802.16e ref 24/4730LUT → prop 12/2997LUT, 802.11n ref 81/16376LUT → prop 27/7727LUT (Cyclone II 100MHz), 고SNR에서 throughput 크게 증가.
10. 한계: rate-compatible puncturing·dual-diagonal parity 구조가 전제이며 특정 무선표준 코드 대상, BER는 오히려 약간만 변화(정정능력 이득 아님).
