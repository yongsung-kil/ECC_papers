### B. 알고리즘 요약

1. 목표: 터보 부호와 LDPC 부호를 **하나의 공통 디코더**로 처리하기 위해, 터보 부호를 블록 부호로 보고 sparse PCM으로 표현해 MP(message passing) 복호를 적용.
2. 문제: 관례상 터보는 trellis 기반 `BCJR`, LDPC는 Tanner graph 기반 `MP`로 복호 — 공통 아키텍처가 없음.
3. PCM 구성: 컨볼루션 부호 생성행렬 `G(D)`로부터 transform-domain PCM `H(D)`를 얻고, 시간영역 PCM의 부분행렬 `h_t`를 `H(D)`의 `D^t` 계수로 채우는 **GBT(Generator matrix based transformation)** 제안 (Smith Decomposition 대체, 더 단순).
4. 터보 PCM: RSC 성분부호 PCM `Hc`를 정보/잉여 열 `Hi`,`Hr`로 분리하고 인터리버 치환행렬 `Π_T`로 결합(식 5).
5. 핵심 명제: 4-cycle이 코딩 이득을 해치므로 이를 없애는 기준 도출.
6. Rule 2/3: 컨볼루션 부호에서 `N1<=3`(계수 1의 개수) 또는 (a)`H1`에 4-cycle 없음 + (b)모든 행차 `d_i`가 상이 → 4-cycle 없음.
7. Rule 4/5: 터보 부호는 (a)성분부호 무4-cycle + (b)`Hi` 각 행·열에 1이 하나 / 1/2 부호는 `S=max{d_i}`인 S-random 인터리버 사용 시 무4-cycle 보장.
8. 학습/최적화: 없음 (해석적 기준 + 인터리버 설계).
9. 결과: rate 1/2, length 3072, AWGN, 최대 10 iter에서 4-cycle 있는 TC1은 MP BER 크게 열화, 무4-cycle TC2/TC3은 MP가 BCJR에 근접한 성능 (수치 곡선은 그림만, 본문 수치 없음).
10. 한계: 터보/컨볼루션 부호 특화, HW 미설계, 시뮬만, QC-LDPC 부호설계와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GBT 기반 sparse PCM 구성 (터보/컨볼루션) | 대응 없음 | Prime ECC는 고정 binary QC-LDPC이며 터보 PCM 구성과 무관 |
| MP 복호 적용 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 개념적으로 MP지만, 터보 부호 대상이라 직접 대응 불가 |
| 4-cycle 회피(girth) 기준 | 대응 없음 | Prime ECC H-matrix는 이미 확정, 논문 기준은 터보 인터리버 설계용 |

적용 가치: **낮음** — 터보/컨볼루션 부호를 MP로 복호하기 위한 PCM 4-cycle 회피 구성법으로, NAND용 binary QC-LDPC의 부호설계·디코더·HW 어디에도 떼어 쓸 수 없다.
