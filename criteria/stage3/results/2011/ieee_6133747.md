### B. 알고리즘 요약

1. 대상은 packet-based LDPC 부호(각 변수노드=하나의 packet, `t∈{0,1}^l`)를 BSC 위에서 복호하는 EVA(Enhanced Verification-based decoding)이다.
2. 문제: 원래 VA는 이웃 1개 check로만 검증해 BSC에서 false verification 확률이 높다. EVA는 다중 check 제약으로 이를 줄인다.
3. EVA는 verification 단계(packet 상태 V/C/U 판정)와 correction 단계(다른 이웃이 모두 verified면 나머지 packet 값을 합으로 복원)의 2단 반복.
4. 검증제약 `d_{k,j}` = degree-j 변수노드를 k-th iteration에 verify하는 데 필요한 최소 검증 check 수(iteration별 동적 조정).
5. 기여: EVA를 message-passing 관점으로 재정식화하고 packet-level 재귀 통계모델로 FER를 예측(false verification 반영).
6. 모델은 packet→codeword 확률변환식 (2),(3)과 재귀식으로 `q̇_f, q̇_u, q_u`를 iteration마다 갱신해 (1)의 FER 누적식에 대입.
7. 핵심 가정: iteration마다 packet 값·상태가 통계적 독립, unverified packet의 bit error 확률 ε 불변.
8. 확률 근사는 β≤2 (한 제약당 총 오류 bit≤2+β)까지만 error pattern 조합으로 계산(Appendix B).
9. 결과: regular 4800-(3,6), irregular 1920 부호에서 packet size 32~256bit, 다양한 ε에 대해 모델과 시뮬 FER 곡선이 일치.
10. 한계: 순수 통계적 성능분석(HW 미설계, cycle 등 세부 구조 미반영으로 small ε에서 편차), 알고리즘 신규성 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Verification/correction 복호 | 대응 없음 | packet 단위 verification 복호는 min-sum bit-level 복호와 다른 패러다임 |
| packet-level FER 통계모델 | 대응 없음 | Prime ECC는 bit-exact 시뮬로 성능 측정, 통계 근사 모델 아님 |
| irregular LDPC degree 분포 | [ecc_top.cpp](../Prime_ECC_3.1_Claude) `Load_PCM()` | 부호 구조만 개념적 연결, packet-based 구조는 무관 |

적용 가치: **낮음** — packet-based verification 복호는 NAND용 binary bit-level QC-LDPC min-sum 파이프라인과 패러다임이 다르고, HW/알고리즘 신규 기여 없이 성능분석에 그친다.
