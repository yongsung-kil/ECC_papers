### B. 알고리즘 요약

1. 대상은 고rate `R≥0.875` (3,k) regular LDPC 인코더; 예시로 (3,24) regular, `N=9360`, `R=0.875` (표에는 `N=6000`도 등장).
2. 문제: (3,k) regular 고rate(R≥0.875) 인코더 구현 사례 부재 + 가변 rate/length 유연 인코더 연구 부족.
3. LDPC 인코딩 방식 정리: G 행렬(systematic/non-systematic), Richardson-Urbanke(근사 하삼각 + 부행렬 A~F, gap g), Neal(LU 분해) 비교.
4. 채택: 랭크 결손 없는 H를 code-construction [24]로 확보 후 Richardson-Urbanke 방식으로 식(8),(9)를 풀어 parity `p1,p2` 계산, 코드워드 `c=[s,p1,p2]`.
5. 핵심 식: `p1^T = -F^{-1}(ET^{-1}A + C)s^T` (8), `p2^T = -T^{-1}(As^T + Bp1^T)` (9); `F = -ET^{-1}B + D` (7).
6. 프레임워크: 소프트웨어부가 부행렬 A,B,C,D,T,E 준비, 하드웨어부가 실제 인코딩 수행 후 permutation table로 최종 코드워드 생성.
7. 핵심 아키텍처: 인코딩을 5-stage로 분할, 각 stage 워크로드를 **H 부행렬의 non-zero 원소 수** 기준으로 균형화 (Table I: A=15120, T=2100 등).
8. 병렬화: stage 간 double buffering으로 병렬 실행, 각 stage가 finish signal 생성, 완료 시 controller가 다음 실행 start signal 발행.
9. stage 배치: S1 정보비트 버퍼링, S2 행렬 A 연산, S3 T forward substitution, S4 p1/p2 계산, S5 permutation.
10. 결과: (3,24) regular `N=9360, R=0.875` 코드가 BER 10^-6에서 Shannon limit 대비 1.12dB 접근 (BER 시뮬만).
11. 한계: decoder 무관 인코더 전용, FPGA 합성/gate count/throughput 실측치 본문에 없음, RU·(3,k) regular 특정.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC 인코딩 (parity p1,p2 생성) | encoder.cpp LDPC_encoder_4KB(), Generate_PCM_encoding() | 인코딩 전체 대응하나 방식(RU) 상이 |
| Richardson-Urbanke 부행렬 분할 | 대응 없음 | Prime ECC는 dual-diagonal 고정, RU 근사 하삼각 미사용 |
| stage 워크로드 분할/double buffering 병렬 | 대응 없음 | HW 인코더 파이프라인 스케줄, 코드베이스에 별도 모듈 없음 |
| 부호 PCM 로드/구성 | ecc_top.cpp Load_PCM() | H-matrix 로드 지점 대응 |
| dual-update column 재배치 | encoder.cpp Column_Reordering_Dual_Update() | 인코더 col 재배치 개념적 유사(방식 다름) |
```

적용 가치: **낮음** — Prime ECC 인코더는 dual-diagonal 고정 방식이고 이 논문은 Richardson-Urbanke 부행렬 분할 기반 (3,k) regular 인코더라 인코딩 알고리즘 자체가 상이하다. 워크로드 분할·double buffering 병렬화는 일반적 HW 기법으로 참고 가치가 제한적이며, HW 실측치도 없다.
