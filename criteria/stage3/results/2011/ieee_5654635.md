### B. 알고리즘 요약

1. 대상은 regular column-weight-2 `(2,k)` QC-LDPC 부호, rate 1/3~1/2, 길이 63~228672(Table I 범위).
2. 문제: column-weight-2 부호는 인코딩/복호 복잡도가 낮고 partial-response 채널·비이진 부호 설계에 유리하나, 기존 구성법은 girth가 최대 24로 제한.
3. 핵심 아이디어: `(k,k)` QC-LDPC의 Tanner graph를 finite geometry(점=CN, 선=VN)로 간주하면 그 incidence matrix `H2`가 row-weight-k, column-weight-2이며 girth가 원 부호 `H1`의 2배가 됨(재귀 유도).
4. 즉 girth `2g`인 `(2,k)` 부호를 얻으려면 girth `g`인 `(k,k)` QC-LDPC를 먼저 설계해야 함.
5. `(k,k)` mother matrix가 complete bipartite graph면 girth<=12(→(2,k)는 girth<=24) 한계 → 12-cycle은 exponent로 제거 불가(식 `a4-a1+...=0`).
6. girth>24를 위해 BIBD `(v,k,2)`/`(v,k,1)` design의 incidence matrix를 mother matrix로 사용(각각 g<=14, g<=18).
7. Exponents Search Algorithm: circulant shift 지수를 row-by-row로 배정, tree expansion으로 각 depth의 potential cycle 열거, cycle 조건 `sum (-1)^{i-1} a_i ≡ 0 mod L`을 위배하는 guilty exponent/guilty block size를 배제.
8. (학습 절차 없음, 결정적 탐색.)
9. 결과: BIBD(7,3,1) 기반 (3,3) girth-18 부호에서 재귀 유도로 (2,3) girth-36 부호(길이 6489) 생성; girth↑일수록 길이 짧아 RandPEG 대비 우수, girth 클수록 BER 개선·랜덤부호 능가.
10. 한계: column-weight-2는 min-distance가 로그적 증가로 근본적으로 약함, HW 미설계, column weight>2/irregular 확장과 block size 최소화는 미해결.
