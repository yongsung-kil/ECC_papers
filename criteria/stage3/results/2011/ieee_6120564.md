### B. 알고리즘 요약

1. 대상: regular (200,100)·(504,252) LDPC (rate 0.5), AWGN 채널, sum-product(SPA) 복호.
2. 문제: 짧은 부호의 사이클·trapping set이 복호기를 나쁜 상태로 끌어당기고, SPA의 단조(monotonic) 거동으로 탈출 불가.
3. 아이디어: 선택 비트노드의 정규화 likelihood ratio를 `1/2`로 설정해 "erase"(엔트로피↑) → 나쁜 sub-structure를 부분/전체 해체.
4. 핵심 변수: `ns`(세트 수), `Ic`(초기 SPA 반복), `Ir`(세트별 반복), `δ`(참여 비트노드 비율, 본문 0.9~0.95), `n×ns` erasure 행렬.
5. 필요 비트노드 최소비율 `δ > R + ε`(식1): rate R보다 크면 성공 복호 가능(SNR↑ 시 ε↓).
6. 절차: 전 비트노드로 `Ic` 반복 SPA → 실패 시 각 세트가 서로 다른 erasure 패턴으로 `Ir` 반복 → 유효 codeword들 중 수신벡터에 Hamming 최근접 선택.
7. 비단조성: 세트마다 다른 복호 경로 → trapping set 인지·특성화 불필요(2번째 카테고리 기법과 차별점).
8. erasure 행렬은 LDPC 행렬 생성법과 유사하게 각 행·열의 0 개수 균등하게 랜덤 배치.
9. 결과: (200,100)에서 error floor 대신 waterfall 유지, Eb/No 4.8dB에서 약 1 decade 개선; (504,252) MacKay 코드는 recovery 적용 시 PEG+conventional을 능가; 평균 iteration은 오히려 소폭 감소(5.90 vs 6.01 @3dB).
10. 한계: HW 미설계·시뮬만, rate 0.5 규칙부호, δ·erasure 세트 최적화 미해결, 실패 시 다중 세트 순차 실행.
