### B. 알고리즘 요약

1. 대상은 QC-LDPC (802.11n WiFi Z=81, 802.16e WiMAX Z=96, length 1944~24576), log-SPA 및 Min-Sum 복호를 GPU에서 가속.
2. 문제: 기존 GPU LDPC 구현은 특정 GPU 아키텍처·특정 코드에만 최적화돼 확장성·유연성 부족.
3. 매핑: 복호를 horizontal CTV kernel과 vertical VTC/APP kernel 두 CUDA 커널로 분할, 행/열별로 M/N 스레드 spawn 후 `__syncthreads()`로 동기화.
4. 3D 다중 코드워드 스킴: `Z` 스레드(1 layer) → `NCW` 코드워드 묶은 macro-codeword → `NMCW` macro-codeword 병렬로 3차원 workload 확장.
5. 이유: GPU SM을 idle 없이 채우려면 workload(코드워드 수)를 충분히 키워 device memory latency를 은닉해야 함.
6. 적응형 튜닝: GPU 아키텍처 정보(SM수, 스레드/블록, 레지스터, shared mem)를 조회해 `Nactive_threads/SM` 최대화하는 `NCW`,`NMCW`를 제약식하에서 계산.
7. 구현 디테일: H matrix 컴팩트 표현, coalesced/constant memory 활용, early termination 적용.
8. 최적화: 4개 제약(threads/block, active threads/SM, 레지스터, shared mem) 하의 최대화 문제로 정식화.
9. 결과: GTX470에서 코드별 약 39~49 Mbps throughput, ~200-300 코드워드 이상에서 peak 도달, 구형 8600GT에서도 튜닝 동작 확인.
10. 한계: 시뮬 가속 목적, HW 미설계, 정정능력·부호설계 개선 없음, latency는 낮게 유지만 언급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| CTV/VTC 병렬 스케줄 | 대응 없음 | GPU SIMT 병렬화라 RTL pipeline(z=32)과 정합 안 됨 |
| 적응형 성능 튜닝 | 대응 없음 | GPU 하드웨어 자원 매핑 전용 |
