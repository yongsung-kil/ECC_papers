### B. 알고리즘 요약

1. 대상: binary QC-LDPC, 예제 `802.11n (1944, 972)` rate 1/2 (Z=81, 12x24 base) 및 `WiMAX (2304, 1152)`, GPU(NVIDIA GTX470) SW 디코더.
2. 문제: GPU의 대규모 병렬 코어를 LDPC 복호에 충분히 활용하지 못함(메모리 접근 지연·워크로드 불균형이 병목).
3. 복호 알고리즘: 표준 log-SPA — VTC `Qmn = Ln + sum Rm'n` (1), CTV `Rmn`는 box-plus 연산 `(4)(5)`(min + 보정항 S), APP 갱신 `Ln = Ln + sum Δmn` (6).
4. 대안으로 scaled min-sum: `Rmn = α·(sign 곱)·min|Qmn|`, `α=0.75` 고정 (7) — 성능 손실 최소로 복잡도 절감.
5. GPU 매핑: 복호를 horizontal processing(CTV 계산, M threads/row)과 APP update(N values, VN별 독립) 두 CUDA kernel로 분리, 각 sub-matrix layer는 동일 shift라 Z threads가 동일 연산.
6. multi-codeword: NCW codeword를 macro-codeword로 묶어 thread block당 처리(약 300 codeword 병렬) → read-after-write 의존·bank conflict를 활성 스레드로 은닉.
7. 압축 H 표현: QC 구조를 12x24 base로 보고 수평/수직 압축(`Hkernel1/Hkernel2`, 각 entry는 x/y/shift/valid 4byte) → 메모리·branch 명령 40% 감소, constant memory broadcast로 +8%.
8. 메모리 최적화: Rmn/Δmn을 압축 저장 후 column-wise 기록으로 coalesced access → +20% throughput.
9. 결과(시뮬): log-SPA와 min-sum throughput 유사(intrinsic logf/expf 사용, 메모리 지연이 지배), 802.11n 최대 100.3Mbps(ET 포함), 선행연구 [5] 대비 50배; SNR↑시 tag-based ET로 avg iteration 감소.
10. 한계: HW(ASIC/RTL) 미설계·GPU SW 시뮬 가속 전용, 복호 알고리즘은 표준이며 정정능력·부호설계 기여 없음.
