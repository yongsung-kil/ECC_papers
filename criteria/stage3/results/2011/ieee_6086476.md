### B. 알고리즘 요약

1. 대상은 중국 DTMB 방송표준의 QC-LDPC 인코더, 3개 rate(0.4/0.6/0.8), sub-block b=127, (k,c)=(24,35)/(36,23)/(48,11).
2. 문제: 기존 SRAA serial은 자원 소모 크고, 단일 rate 최적화/3-rate 조합은 속도가 느림 → 속도·자원 균형 필요.
3. 인코딩은 정보열과 generator matrix G의 곱 `C=M×G`, G가 quasi-cyclic이면 순환 shift+AND+XOR로 단순화.
4. 핵심 회로 `SRAA`(Shift Register Adder Accumulator): 패리티 `p_i=Σ a_l G_{l,i}`를 첫 행벡터 `g^(0)_{i,j}`를 피드백 SR B에 넣고 정보비트 shift-in마다 AND곱을 누산 레지스터 A에 축적.
5. 각 부분행렬마다 b회 반복, `g^(0)` 재적재 후 전체 정보열이 들어갈 때까지 반복해 A에 패리티 결과 확정.
6. 확장(부분병렬): c(=35/23/11)개 SRAA 대신 12 SRAA만 두고 rate별로 3/2/1회 순차 동작 = "partially-parallel"로 slice 자원과 속도 절충.
7. 구현 디테일: input buffer / generator matrix storage / output buffer / control module 구성, in_mode로 rate 선택.
8. (학습 절차 없음 — 순수 HW 구현.)
9. 결과: Virtex4 xc4vlx100 합성, 3909 Slices(7%)·5119 FF(5%)·24 RAMB16(10%), 최대 208.106MHz(요구 48.73MHz 초과), modelsim=matlab 결과 일치.
10. 한계: 인코더만 다루고 복호/정정성능 무관, DTMB 저속 방송 payload(32.5Mbps)·특정 부호 전용.
