### B. 알고리즘 요약

1. 시스템: DVB-S2 위성방송 FEC용 irregular IRA(Repeat-Accumulate) LDPC, systematic, `N` 최대 64800bit(short 16200), normal frame 11개 rate(B1~B11), row/col weight는 rate 종속.
2. 문제: LDPC 복호는 데이터 의존성·대량 메모리접근으로 병렬화가 어렵고, 특히 긴 irregular 코드가 실시간 처리에 부담 → 전용 VLSI 대신 범용 many-core GPU로 대체 시도.
3. 부호 구조: `H=[A|B]`, `A`는 sparse, `B`는 staircase 하삼각 → 주소가 규칙적이라 on-the-fly 계산 가능, periodicity로 저장 절감.
4. 핵심 기법(복호): Min-Sum(Sum-Product보다 저복잡), `Lrmn = ∏sign(Lqnm)·min|Lqnm|`(CN, kernel1), `Lqnm = Lpn + Σ Lrm'n`(BN, kernel2), `LQn`로 사후 LLR→hard decision.
5. 병렬화 핵심: thread-per-node(row당/col당 thread), flooding schedule로 BN↔CN 메시지 교환의 데이터 의존성 제거 → thread-level 완전 병렬.
6. 확장: multicodeword decoding — 16 codeword를 동시 복호해 GPU 점유율 극대화, 데이터는 8bit 표현(기존 VLSI 5~6bit 대비 여유).
7. 메모리 최적화: coalesced global memory 접근(연속 정렬)으로 느린 global memory 접근 최소화, 32-thread warp 단위 처리.
8. GPU-optimized 변형: staircase `B` 행렬의 Tanner edge 주소를 GPU에서 on-the-fly 생성 → global memory 접근 축소로 throughput 향상.
9. 결과: Fermi C2050(448코어)에서 전 rate 90Mbps+ 달성, GPU-optimized 시 반복10에서 B1 87Mbps~반복1 B1 284Mbps, 전용 VLSI와 throughput·BER 경쟁 가능.
10. 한계: 순수 GPU 소프트웨어 가속(HW 미설계), 부호는 irregular IRA(QC-LDPC 아님), kernel1 divergence로 고 row-weight 코드(B7~B11) 성능 저하.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum CN/BN 커널 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()`, `VN_Cal_HD()` | min-sum 자체는 이미 보유 — 중복 기여, GPU 병렬화 기법은 ASIC과 무관 |
| flooding schedule 병렬 복호 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 layered/dual-update 계열이라 스케줄 상이 |
| staircase B행렬 주소 on-the-fly | 대응 없음 (dual-diagonal 인코딩은 `encoder.cpp`이나 IRA staircase와 구조 다름) | 부호구조 상이(IRA vs QC dual-diag) |
| GPU thread/coalesced 메모리 | 대응 없음 (Prime ECC는 z=32 HW 모델) | 실행 플랫폼 자체가 다름 |

적용 가치: 낮음 — min-sum은 이미 보유하고 GPU 소프트웨어 병렬화·irregular IRA 부호·8bit 표현 모두 NAND ASIC binary QC-LDPC와 정합하지 않는다.
