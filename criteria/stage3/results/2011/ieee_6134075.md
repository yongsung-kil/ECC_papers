### B. 알고리즘 요약

1. 시스템: FEC+BPSK 수신단, 디코더 앞 버퍼메모리가 AVS(공격적 전압스케일링)로 균일 랜덤 비트플립 유발. 예제: WiMAX turbo `N=240 R=1/2`, DVB-S2 LDPC `N=64800 R=1/2`, `(8,5)` 고정소수점.
2. 문제: 기존 FEC는 Gaussian 채널노이즈만 가정, 메모리 전압스케일링이 만드는 비트플립 노이즈를 무시 → 고SNR에서 noise flooring.
3. 모델: 메모리 출력 분포 = 채널노이즈 분포와 메모리 k-bit flip 분포의 결합. 각 비트 flip 확률 `Pe` 동일(spatially uniform), sign/MSB 오류가 지배적(값 `2/2^r` 만큼 shift·scale된 sidelobe).
4. 핵심 기법(공통): sign-bit 단일 오류만 고려해 결합 채널확률을 mainlobe(원분포)+sidelobe 2개 3항 근사 (식 9).
5. Turbo 경로: MAX-Log-MAP의 branch metric·extrinsic 계산에 새 분포 반영 — MAX-Log 근사 1단을 기존 단 앞에 추가 (식 12~18), forward/backward metric 식은 불변.
6. LDPC 경로: BP/min-sum/TDMP에서 **LLR 초기화만** 수정 (식 24), CN/VN 업데이트 규칙은 완전히 불변.
7. 구현 디테일: 디코더 재설계 없이 front-end에 작은 전처리 블록 추가로 초기 LLR을 `Pe` 기반 보정. TDMP도 동일하게 식 24를 앞단에 적용.
8. 학습/최적화: 없음 — `Pe`는 수신 SNR slack에 따라 적응적으로 조절(고SNR일수록 큰 `Pe` 허용 → 전력↓).
9. 결과: 25% 전압강하로 메모리전력 ~40% 절감(`Pe`≈1e-6~5e-3), 수정 디코더는 全SNR에서 BER 개선·noise flooring 제거, 이상디코더 대비 BER 1e-6에서 0.5dB. 면적/전력 오버헤드 LDPC 0.013%/0.016%, turbo 0.82%/1.1% (65nm 합성).
10. 한계: LDPC는 DVB-S2 rate1/2만, 개선이 sign-bit 지배 가정 의존, throughput 미보고, NAND가 아닌 통신 버퍼메모리 타깃.
