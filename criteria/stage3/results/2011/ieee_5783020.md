### B. 알고리즘 요약

1. 대상 시스템: 12.5 Gb/s NRZ-OOK DWDM 광통신(60km SMF-28), FEC로 `LDPC(32000, 29759)` (7% overhead, rate ~0.93, var deg 4, check deg 57/58).
2. 문제: 광 LDPC는 soft-decision이 필요하나 기존 3-to-2 encoder 방식 all-electrical front-end가 14W@32GS/s로 전력·복잡도 과다.
3. 핵심 아이디어: 광수신기(photodetector) 앞에서 광신호 일부를 optical coupler로 tap하여 hard/soft 경로 분리, HD 임계값 위·아래 두 개 confidence threshold 배치.
4. soft-decision 회로: 2개 comparator(VTh1/VTh0 상·하 임계) 출력을 단일 XNOR로 결합 → `VConf = VTh1 XNOR VTh0` (식 1), 3-to-2 encoder를 게이트 1개로 단순화.
5. 물리 의미: 두 confidence 비트가 같으면 신뢰도 1, 다르면 0 → HD 근처(불확실 영역) 비트를 낮은 신뢰도로 마킹.
6. 초기 LLR: 수신 샘플의 조건부확률 P(x|y)로부터 `LLR = log[P(x=0|y)/P(x=1|y)]` (식 2) 정확 계산 후 Q값별 LUT로 n-bit 양자화.
7. HD/soft 두 경로 물리길이 차 211ps를 Rogers 기판 microstrip delay line(~4.6cm)으로 정합.
8. 임계 최적화: 광증폭기 잡음의 non-central chi-squared 분포 특성상 mark는 넓게, space는 좁게 spacing(비율 ~1.3), coupling ratio 20%가 최적.
9. 결과(실측): 80:20 coupling에서 HD 무부호 대비 coding gain 2.75dB@BER 1e-4, 6.73dB@1e-9; all-electrical 대비 Q penalty 0.35dB; soft 회로 ~5W, error floor는 1e-16까지 없음.
10. 한계: 복호는 표준 SPA(50 iter)를 그대로 사용, 기여는 순수 광/아날로그 front-end이며 디코더·부호설계·HW 아키텍처 개선 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| optical coupler + XNOR 기반 2-bit soft-decision 생성 | 대응 없음 | NAND는 전기적 read reference voltage로 soft-info 생성, 광 tap 방식은 물리계층 상이 |
| Q값별 LUT LLR 양자화 (식 2) | `channel.cpp` `Set_LLR_Th()`, `Set_R_Offset()` | 개념상 soft-read 양자화 대응은 있으나, 광 채널 chi-squared 모델 종속이라 이식 불가 |
| SPA(sum-product) 복호 | 대응 없음 | 우리는 min-sum 사용, full-tanh SPA는 미대응(모듈 지도상 "대응 없음") |
| delay line / comparator front-end HW | 대응 없음 | 순수 아날로그 광회로, HW 디코더 무관 |
