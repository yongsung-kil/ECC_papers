### B. 알고리즘 요약

1. 시스템: WiMax(802.16e) rate-1/2 mother code(length 2304, submatrix z=96) 기반 rate-compatible QC-LDPC 디코더, mother rate와 1 사이 임의 rate 지원.
2. 문제: 기존 PLDA는 layer간 고정 연결이라 단일 코드에만 맞고 rate 유연성 부족; 다중 rate 저장은 메모리·스위칭 네트워크 부담.
3. 모델: dual-diagonal parity 구조(weight-3 열 + zigzag) QC-LDPC, AWGN 채널, BP/min-sum 복호.
4. 핵심 기법: parity bit puncturing으로 rate 조정, `k-SR`(k-step recoverable) 변수노드 이론으로 낮은 index 그룹부터 puncture하여 성능 손실 최소화.
5. 핵심 식: 특수 시퀀스 `S`를 재귀식으로 생성해 블록 내 punctured bit를 분산, LUT 저장 후 rate로부터 즉시 index 조회.
6. 확장: weight-3 vertical submatrix를 puncturing 블록에 포함(scheme 3)하면 이웃 check가 많아 반복 중 정보 제공이 늘어 BER 우수.
7. 구현 디테일: PLDA에 puncturing 통합 — LLR initialization block(punctured LLR을 0으로 patch), 12x12 LUT, address generator(accumulator step 조정)로 rate 유연성 구현.
8. HW 아키텍처: permutation 값 내림차순 정렬 후 layer간 message passing, offset-modified permutation으로 layer 간 최소 5 clock 간격 확보 → CNU 4~5-stage pipeline(critical path split)으로 clock 향상.
9. 결과: dedicated WiMax 부호 대비 BER 손실 <0.2dB, TSMC 65nm 합성 1.1GHz, core area 1.96mm2, 입력 throughput 1.28Gb/s(10 iter), power 908mW.
10. 한계: WiMax 무선 특화, 소프트 양자화 bit수 본문 미명시, 정정능력 자체 향상 아닌 rate 유연성/throughput 목표.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum layered CNU 연산 | [decoder.cpp](../Prime_ECC_3.1_Claude/) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum 보유, layered 스케줄과 대응이나 중복 |
| dual-diagonal parity 구조 | [encoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_encoder_4KB()`(dual-diagonal) | Prime ECC도 dual-diagonal 인코딩 사용, 중복 |
| puncturing으로 rate-compat | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | Prime ECC는 punct_col_num=2 고정, 무선용 가변 puncturing과 목적 상이 |
| offset-permutation 5-cycle CNU pipeline | 디코더 HW 모델(z=32, pipeline 6-stage) | Prime ECC pipeline 구조와 정합 검토 여지 있으나 z 단위 상이 |

적용 가치: **중간** — 디코더 HW/스케줄링 아이디어(offset-permutation pipeline)는 참고 가치가 있으나 min-sum·dual-diagonal·layered는 Prime ECC가 이미 보유하고 puncturing은 무선 rate-compat 목적이라 NAND 이식 이득 제한적.
