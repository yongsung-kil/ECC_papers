### B. 알고리즘 요약

1. 시스템: AWGN 채널, 정보길이 `k=192`, 초기 블록 `256` (protograph를 32회 lifting), rate `6/8~6/18`(0.75~0.33) 범위의 rate-compatible LDPC.
2. 문제: 짧은 블록에서 rate-compatible 부호 필요. 기존 puncturing 방식은 puncturing 패턴 최적화가 어렵고 성능 저하 발생.
3. 핵심 기법: protograph-based Raptor-like(PBRL) - 왼쪽에 LDPC precode protograph, 오른쪽에 LT 부호 protograph(각 새 check node에 degree-1 변수노드 부착). puncturing 없이 precode 심볼의 XOR로 추가 parity 생성.
4. 인코딩: 전체 PCM `H = [Hp O; HLT I]`, `σ`는 32x32 순환(circulant) shift 행렬. degree-1 변수노드로 incremental redundancy를 효율 인코딩.
5. 복호: precode와 LT 부호를 동일 BP 디코더로 joint decoding(기존 Raptor의 2-stage 분리 디코딩 대비 복잡도 감소). 초기 전송부터 모든 저rate 코드워드에서 BP 성립 보장.
6. 확장: PN-PBRL - precode protograph의 첫 변수노드를 puncturing하여 threshold 개선(rate-6/7 precode). 병렬 edge 추가 시 threshold 더 개선되나 lifting 후 trapping/absorbing set 유발 위험.
7. 최적화: 고정 초기 rate에서 precode protograph를 greedy 탐색, LT part는 density evolution으로 새 check node의 최적 degree/연결 결정. cPEG로 순환 permutation 선택(length-4 cycle 제거, length-6 최소화).
8. 결과: rate 3/4에서 PN-PBRL이 3GPP RCPT와 유사, SNR>3dB에서 우세. FER `10^-7`까지 error floor 징후 없음. threshold-capacity gap은 rate 감소 시 증가(구조적 제약).
9. 한계: 짧은 블록에서 asymptotic(DE) 분석 정확도 저하. HW 설계·gatecount·throughput 없음, 순수 부호구성법.
