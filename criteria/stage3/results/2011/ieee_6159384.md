### B. 알고리즘 요약

1. 대상: SoC 메모리 soft error(SEU/MBU/SET) 완화용 EG-LDPC(Euclidean Geometry LDPC), 예제 (31,16)·(42,16), rate 약 0.38~0.52, GF(2^5)/GF(2^6) 기반.
2. 문제: Hamming은 MBU 부족, RM/Matrix code는 HW overhead 크거나 encoder/decoder SET 미대응; [4]의 EG-LDPC는 원시 코드길이만 지원해 메모리의 2n bit 데이터폭에 직접 못 씀.
3. 접근: G-matrix 열 삭제로 (63,37)->(42,16)처럼 2n bit 데이터폭 EG-LDPC 구성; parallel majority decoding + detector feedback loop로 storage cell·encoder·decoder 모든 transient fault 감내(fault-secure).
4. 부호구성: 최소다항식으로 생성다항식 `G(X)` 도출, vF를 순환시프트해 systematic G-matrix 생성; encoder는 `v=u*G` XOR 로직.
5. 복호: majority-logic decoding — parity check-sum `A`의 다수결값이 1이면 해당 bit 반전; correction matrix `W`가 J/2 정정·J-1 검출.
6. 핵심 개선: 원래 (31,16)은 dmin `J`=6으로 이론상 3-step majority decoding 필요하나, 제안 알고리즘이 orthogonal vector 재구성으로 **2-step**으로 축소.
7. HW: (31,16) 디코더는 7개 parallel majority logic 회로 + syndrome 회로, N번째 회로 입력은 (N-1)번째를 5bit 순환우시프트해 얻어 1 클록에 정정.
8. detector: syndrome matrix `S`로 d·S^T=0 여부 검사, 0 아니면 error 신호로 데이터 중단·재전송(fault-secure 보증).
9. 결과: Verilog+SMIC 0.18um 합성; 100만 fault injection 실험서 (42,16) EG-LDPC가 최고 보호; MTTF (42,16)이 Matrix code 대비 18.9%, RM 대비 4.5% 높음.
10. 한계: iterative message passing 아님(hard majority-logic one-shot), 코드길이 31~42로 극소, gatecount/throughput 수치 본문 부재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| majority-logic 복호 | 대응 없음 | Prime ECC는 iterative min-sum message passing, majority-logic one-shot과 무관 |
| EG-LDPC 부호구성(GF 생성다항식) | 대응 없음 | Prime ECC는 QC-LDPC(base+circulant), Euclidean geometry 구성과 이질적 |
| G-matrix XOR 인코더 | [encoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_encoder_4KB()` | 개념상 인코더지만 dual-diagonal QC 인코딩과 방식 상이, 이식 불가 |
| syndrome detector | 대응 없음 | fault-secure feedback/재전송 구조는 Prime ECC 범위 밖 |

적용 가치: **낮음** — hard majority-logic 복호 + Euclidean Geometry 부호구성이 NAND용 iterative soft-decision binary QC-LDPC 엔진과 알고리즘/부호구조 모두 이질적이라 떼어 쓸 요소가 없다.
