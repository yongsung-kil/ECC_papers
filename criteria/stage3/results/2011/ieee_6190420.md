### B. 알고리즘 요약

1. 대상은 802.15.3c/802.11ad의 (672,588) LDPC(rate 7/8, N=672, M=84), 6-bit 입력 양자화, partial-parallel 디코더.
2. 문제: partial-parallel 디코더에서 층(layer)마다 V-to-C/C-to-V 라우팅을 바꾸는 mux 망이 면적·전력·critical path 병목.
3. 복호는 horizontal layered belief propagation + normalized Min-Sum(`Sfactor_MS` 보정계수), Y=4 layer, U=32 column group.
4. 핵심 개념 `Permutational` H: 층 Lk의 column group을 Lk+1의 동일/영 submatrix로 보내는 one-to-one `valid mapping M_P`가 순환적으로 존재하는 행렬.
5. 이 구조면 라우팅 망을 최대 row-degree 층(Lmax) 기준으로 hard-wire 고정하고, 그 앞에 `Mapping^-1` 기반 상수 shift 망만 두면 층 전환이 배선 이동만으로 처리됨.
6. 각 sub-iteration에서 variable node 출력 Fj를 고정 shift(Sj)한 뒤 고정 연결행렬로 check node에 투입하면, 물리 배선 변화 없이 서로 다른 층이 순차 처리됨.
7. 트릭: Rij는 check node 내부 레지스터에 저장되어 shift 불필요, 마지막 sub-iteration 후 출력 순서가 원위치라 별도 재배열 불필요.
8. 단점: 층별 syndrome check 불가, 1 iteration 완료 시점에만 조기종료 판정 가능.
9. 결과: (672,588) 65nm P&R 구현에서 라우팅 게이트 1344개 4:1 mux → 126개 2:1 mux(96%↓), 면적 1.26배·throughput 1.3배 개선, 235MHz 7.9Gbps@5-iter, gate 125k.
10. 한계: H가 permutational 조건을 만족해야만 적용, BER/정정능력 변화 없음(라우팅 최적화 전용).
