> 총평: NAND retention/program-disturb 오류의 비대칭성을 이용한 데이터-쉐이핑(asymmetric coding)과 전력저감(SPEA) 시스템 논문으로 실측 완성도는 높으나 LDPC 부호설계·디코더와 무관해 Prime ECC 코어 이식성은 낮음.

### B. 알고리즘 요약

1. 대상: 4X/3X/2Xnm MLC NAND 기반 SSD(512GB, SATA), retention error 95%↓·전력 43%↓ 목표.
2. 문제: 미세화로 retention/program-disturb 오류 증가, 기존 SSD는 1KB codeword당 20~40b 정정 ECC(BCH)·LDPC로도 Shannon 한계 근접 → ECC 외 추가 신뢰성 기법 필요, bit-line 커패시턴스로 전력도 증가.
3. 핵심 관측: retention·program-disturb 모두 "0→1" 오류와 "1→0" 오류 수가 비대칭(non-equal). retention 오류는 SILC로 VTH 감소("10→00","00→01"), disturb 오류는 GIDL로 VTH 증가.
4. Asymmetric Coding: retention 오류가 disturb보다 100배 이상 많음을 이용, lower page "1"·upper page "0"을 최소 60%로 늘려 고VTH 상태("10","00") population을 25%→약 16/24%로 축소.
5. 효과 원리: "10","00" 감소 → (1) 낮은 program voltage로 cell 전압 stress 감소, (2) 낮은 VTH로 tunnel oxide 전계 감소·SILC 저감 → retention 오류↓.
6. 구현: code length 16-bit 단위, data unit 내 "0"이 8bit 이상이면 flip하고 flag=1 세팅(작은 length일수록 "1" 비율↑ but flag overhead↑; length 16이면 overhead 6.3%).
7. SPEA(Stripe Pattern Elimination): program 시 bit-line "1010…" column-stripe 패턴이 inter-bitline 커패시턴스 최대 충전→최대 전류. even/odd column의 "1" 개수 차가 threshold `NTH` 초과 시 odd 먼저·even 나중 재배치로 worst 패턴 회피.
8. 파이프라인: host 데이터 → SPEA Encoder(전력) → Asymmetric Coding Encoder(신뢰성) → ECC(BCH) → NAND program. 두 코딩 모두 access-time·면적 overhead 없음.
9. 결과(실측): retention error 91%(전압stress)+40%(패턴)→총 95%↓, program current 4Xnm 35%↓·3Xnm 43%↓(미세화일수록 효과↑), 16 NAND chip SSD 실측.
10. 한계: LDPC/BCH 부호·디코더 알고리즘 자체 개선 아님(전처리 data-shaping), ECC는 BCH 사용, LDPC는 "advanced ECC와 협업" 수준 언급뿐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Asymmetric coding (data-shaping 전처리) | 대응 없음 | ECC 인코딩 이전 단계 data 편향, Prime ECC 부호/디코더 밖 |
| SPEA (bit-line 전력 저감) | 대응 없음 | NAND program 물리계층 전력, ECC 무관 |
| 비대칭 0→1/1→0 채널 오류 통계 | [channel.cpp](../Prime_ECC_3.1_Claude) `Set_R_Offset()`, `Set_LLR_Th()` | 참고: 비대칭 error 통계는 soft-read LLR 설계 시 asymmetric threshold 반영 여지 |
| ECC(BCH) 블록 | 대응 없음 | 논문 ECC는 BCH, Prime ECC는 QC-LDPC로 대상 상이 |

적용 가치: **낮음** — LDPC 부호설계·디코더·HW와 직접 무관한 data-shaping·전력저감 시스템 기법. 다만 "0→1/1→0 오류 비대칭" 관측은 Prime ECC의 soft-read LLR/threshold를 비대칭으로 설계할 근거로만 간접 참고 가능.
