### B. 알고리즘 요약

1. 대상: IEEE802.11n WLAN QC-LDPC (4 rate x 3 codeword length = 12 모드, `Z`=27/54/81 확장, `N`=648, `R`=1/2 예시).
2. 문제: 멀티모드 지원 시 불규칙 배선/라우팅 혼잡/고전력/고복잡도가 HW 구현 난제.
3. 접근: `ASIP`(application-specific instruction-set processor) 구조로 디코딩 흐름을 마이크로명령어로 제어해 프로그래머블화.
4. 마이크로명령어 8종: `CONFIG`/`Decode_Row_SubH`/`Decode_Row_End`/`Parity_Chk`/`ORG_PC` 등, 포맷은 Opcode+Shift_value+Address (submatrix cyclic shift·주소 지정).
5. 복호 알고리즘: Layered BP + Normalized Min-Sum(`LBP-NMS`) 채택. Check node update 후 즉시 variable node에 반영하는 layered 스케줄.
6. 핵심 이점: LBP가 NMS 대비 수렴 iteration 50%↑ 감소, throughput 최대 2배, 메모리 절감.
7. HW: 6-stage 파이프라인(FETCH/DECODE/R&S/CPE/VPE/ISH&WB). APP값 8bit 포화, cyclic shift로 CN 매핑.
8. 검증: Xilinx Virtex5 FPGA + MicroBlaze 임베디드 시스템(PLB Bus wrapper, UART 스트리밍)에 IP 통합.
9. 결과: UMC 0.18um CMOS 합성 시 최대 203MHz, 면적 3.94mm2, 전력 326.49mW; 정규화 면적·전력 모두 기존 [6][7] 대비 우수.
10. 한계: 802.11n 표준 특화 멀티모드 유연성이 초점, BER/error-floor 개선 없음, gatecount·throughput 수치 본문 부재(그림/표만).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LBP-NMS 복호 루프 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` | min-sum 자체는 이미 보유, layered 스케줄은 Dual-Update와 유사 |
| CN min-search | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()` | 표준 min-sum, 신규성 없음 |
| 마이크로명령어 파이프라인 제어 | 대응 없음 | 멀티모드 프로그래머블 명령어 구조는 Prime ECC 단일부호 고정 HW와 무관 |
| cyclic shift/APP 라우팅 | 대응 없음 | 802.11n 특화 submatrix shift 제어 |

적용 가치: **낮음** — 802.11n 멀티모드 프로그래머빌리티가 핵심 기여이고 복호 알고리즘(LBP-NMS)은 표준 기법이라 NAND 단일 고rate QC-LDPC 엔진에 떼어 쓸 요소가 없다.
