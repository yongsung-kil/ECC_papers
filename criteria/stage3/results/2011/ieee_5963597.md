### B. 알고리즘 요약

1. 시스템: 2 user + 1 relay + 1 destination 반이중 MARC(다중접속 릴레이 채널), time-division 3 time-slot. user-relay 채널은 이상적 가정, relay-destination/user-destination은 Rayleigh fading.
2. 문제: relay가 두 user 데이터를 결합할 때 전통적 XOR 기반 network coding은 다이버시티 이득이 제한적 → 추가 redundancy 생성이 필요.
3. 핵심 기법: relay가 XOR 대신 LDPC **product code**로 두 user 패킷을 결합, 열(column) 방향 추가 redundancy만 forwarding. 목적지에서 user 패킷 2개 + relay redundancy가 완전한 product 부호 행렬을 구성.
4. 복호: 세 sparse check matrix `H1(U1), H2(U2), HR(Relay)`를 하나의 bipartite graph로 합쳐 JICD(joint iterative channel decoding) 수행. 차원 `m=m1+m2+mR`, `n=n1+n2+nR`.
5. 알고리즘: 표준 LLR-BP(sum-product). 함수 `Θ(x)=-log(tanh(x/2))` 사용, VN update(식4) → CN update(식5, sign 곱 × Θ 합) → decision(식6). `HR·v̂=0`이면 조기종료, 아니면 최대 `IT=50` 반복.
6. 부호: 각 user는 IEEE 802.16e LDPC(정보 480비트) 사용. rate 5/6는 relay를 2/3 열비율로, rate 1/2는 802.16e에 행렬이 하나뿐이라 relay는 802.11n 부호를 써서 randomness 확보.
7. 결과(AWGN, rate 5/6): PER=1E-3에서 LDPC-PC가 XOR 대비 약 0.75 dB 이득.
8. 결과(Rayleigh, rate 5/6): 동일 지점에서 약 1.0 dB 이득. rate 1/2 AWGN에서는 LDPC-PC 0.6 dB vs XOR 0.4 dB(PER 2E-4).
9. 분석: 이득의 원천은 product code가 XOR 대비 추가 redundancy를 생성하는 데 있음. 고rate일수록 relay가 더 많은 redundancy 전달.
10. 한계: 무선 MARC 릴레이 특화 응용, 표준 LDPC 부호 재사용, HW 설계·복잡도·throughput 전무, 시뮬레이션만.
