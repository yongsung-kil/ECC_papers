### B. 알고리즘 요약

1. 시스템은 DVB-T2 BICM-ID 수신기, rotated QAM(SSD, Q성분 d심볼 지연) + IRA-LDPC(frame `16200/64800`, rate `1/2~5/6`, 실험은 `R=4/5`, 64800bit), QPSK~256QAM.
2. 문제: rotated-QAM demapper와 LDPC 복호를 반복(BICM-ID)하면 성능은 오르나 interleaver/de-interleaver latency와 연산복잡도가 큼.
3. 채널은 15% erasure를 갖는 수정 Rayleigh fading, AWGN 분산 `σ²`.
4. 핵심 기법: turbo 분야의 shuffled decoding을 확장 — vertical shuffled schedule(VSS)로 병렬 성분 복호기가 extrinsic을 즉시 교환, frame 종료 전 데매퍼-복호기 정보 교환.
5. demapper는 2D Euclidean distance `Deuc(x_t)`와 Jacobian log(⊞)로 bit LLR `v̂_ti` 계산, iterative 시 sub-region 근사 대신 Max-log만 적용.
6. 복호는 normalized Min-Sum의 VSS판(VSSMS), 정밀도 위해 3rd minimum까지 저장하는 VSSMS3 채택(check node 정확도↑).
7. 구현 트릭: RAM 기반 block interleaver를 LUT로 대체해 버퍼 "가상화", QPSK에서 같은 심볼 유래 열을 짝/홀로 나눠 갱신 LLR 재사용.
8. 스케줄 A/B/C(병렬도 1 vs 90) 비교, 설계엔 Schedule C 채택, 병렬도 90(check/bit node processor 각 90 + 간이 demapper 90).
9. 결과: FPGA(Virtex5 XC5VLX330) 80MHz, 15 iter시 throughput 107Mbps; VSSMS3 반복수신이 비반복 대비 0.9dB, 비회전 QPSK 대비 10dB+ 이득.
10. 한계: rotated-QAM demapper 강결합, 방송 fading/erasure 채널 특화, HW는 QPSK만 우선 구현.

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| vertical shuffled(layered) 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | Prime ECC는 Dual-Update(even/odd) 스케줄 보유, 개념 중복 |
| rotated-QAM demapper/BICM-ID 결합 | 대응 없음 | Prime ECC는 NAND 채널, demapper·SSD 없음 |
| LUT 기반 interleaver, 병렬 90 HW | 대응 없음 | Prime ECC는 z=32/HCU/GT 고유 구조, DVB interleaver와 무관 |

적용 가치: **낮음** — min-sum 스케줄은 이미 보유(Dual-Update)하고, 기여의 핵심이 demapper 결합 BICM-ID 저지연 스케줄이라 NAND 단독 LDPC에 떼기 어렵다.
