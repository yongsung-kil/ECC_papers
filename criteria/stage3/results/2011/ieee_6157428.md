### B. 알고리즘 요약

1. 목표: 코드 길이를 고정한 채 여러 code rate를 지원하는 QC-LDPC 부호족을 구성 (동일 디코더 아키텍처 재사용).
2. 문제: puncturing/shortening은 길이를 줄여 성능 저하, 기존 row-combining[4]은 effective 행렬에 짧은 사이클이 증가.
3. 기반: `EG(d, p^s)` Euclidean geometry의 점(point)=블록열, μ-flat(초평면)=블록행으로 대응하는 base matrix `B` 구성.
4. Mother matrix: `B`의 각 '1'을 `q×q` cyclic-permutation, '0'을 zero matrix로 치환 → rate `rM = 1 - δ/p^{ds}` (식 7).
5. 핵심기법(row decomposing): μ-flat `F`를 (μ−j)-flat parallel bundle `P_l`의 incidence matrix `vPl`로 치환 → 행 weight `p^{μs}` 블록행이 `p^{js}`개의 weight `p^{(μ−j)s}` 블록행으로 분해 (Theorem 1).
6. 효과: 이는 check node를 분할하되 variable node 수 불변 → 코드 길이 상수 `L = qp^{ds}` (식 11). rate `r_j = 1 - δ/p^{(d−j)s}` (식 10).
7. row-combining과 반대로 row-decomposing은 mother 사이클을 깨뜨려 effective 행렬이 mother보다 사이클이 적음 → shift값 배정 시 mother만 고려하면 됨(Algorithm 1).
8. shift 배정: 조건식 (8)(길이 2i 사이클 존재조건)을 위배하지 않도록 `a_{m,n}`을 증가시키며 탐색.
9. 결과: EG(5,2)로 Example 4(L=2304, rate 7/8→3/4→1/2), Example 5(L=672)를 만들어 IEEE 802.16e/802.15 대비 BER·error-floor 우수(BPSK/AWGN, log-likelihood BP, max iter 50) 주장.
10. 한계: 순수 구성론+시뮬만, HW·복잡도·throughput 없음, 무선표준 rate/length 타깃, soft-read/양자화 무관.
