### B. 알고리즘 요약

1. 대상 코드: CMMB 표준 regular LDPC, 길이 `N=9216`, rate 1/2 `(9216,4608)` dc=6/dv=3, rate 3/4 `(9216,6912)` dc=12/dv=3. QC-LDPC 아닌 "high-structured(HS)" 코드.
2. 문제: 긴 코드워드로 반복 복호 latency 크고 메모리 요구량 큼 → high throughput + 저비용 HW 동시 달성 목표.
3. H 생성 모델: 기본 서브행렬(rate1/2는 18×6, rate3/4는 9×12)만 저장하고 나머지는 `Hm,n = H'(m mod 18),(n+36·qu(m/18)) mod N` AGU로 동적 생성 → 메모리 절감.
4. 핵심 기법1 (OMP): CTV(check→var)와 VTC(var→check) 연산을 메모리 충돌·데이터 종속 없는 범위에서 병렬 처리. H-matrix permutation으로 0 블록을 좌하/우상에 몰아 오버랩 가능 영역 확보.
5. permutation 제약: CMMB AGU 생성식 유지 위해 행은 18(또는 9)의 배수, 열은 36의 배수 단위로만 회전 → 1260 cycle 오버랩(약 10% latency 감소).
6. 핵심 기법2 (기능유닛 공유): VTC 연산(식3,4의 덧셈)과 min 연산을 adder로 통합, CTV용 adder 유휴 시 VTC에 재사용. 비교는 2's complement 뺄셈 `Y-X = Y+X̄+1`으로 adder가 수행.
7. 핵심 기법3 (메모리 그룹핑): LLR을 permuted matrix 열-major로 8개 메모리 그룹에 배치, 주소 하위 3bit로 그룹 선택, CTV read→VTC read→VTC write→CTV write 순서로 충돌 회피.
8. 아키텍처: 8세트 기능유닛의 partially-parallel 구조, 185MHz 동작.
9. 결과: rate1/2 17.37Mbps, rate3/4 31.34Mbps(20 iter), throughput 40%+ 향상(OMP 10% + 기능유닛공유 33%), 게이트 178k@0.18um, 메모리 163kbit로 [20](212k gate)/[21](455k) 대비 소형.
10. 한계: CMMB HS-LDPC 특정 구조 종속, 정정능력 개선 없음(스케줄링·면적 최적화만), 연판정 비트폭 명시 없음.
