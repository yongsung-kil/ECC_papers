### B. 알고리즘 요약

1. 대상: WiMax/WLAN/DVB-S2용 가변 rate(1/3~5/6)·length(576~64800bit) QC-LDPC, 부분병렬 디코더, Min-Sum 4bit 양자화, BPSK/AWGN, maxiter 10.
2. 문제: 순수 QC 행렬은 부분병렬 구현엔 유리하나 large girth 확보 한계로 unstructured(PEG) 대비 BER 열세. 유연한 다중 code length/rate 구성이 어려움.
3. 핵심 기법: 3-Level Hierarchical QC(3L-HQC) 행렬 + Layered Permutation(LP).
4. Level-1(Core): rate/regularity 결정 (예: rate-1/2 (3,6) → 3×6 core).
5. Level-2: core 원소를 크기 N의 순환천이 항등행렬 L로 확장, 다시 크기 R의 순환천이 Permuted 행렬 Rx 배열로 확장 → code length 결정, 층(layer)별로 다른 Permuted 조합 삽입해 virtual randomness 부여.
6. Level-3: Permuted 비영원소를 크기 P의 순환천이 항등행렬(Base matrix)로 확장 → P가 병렬화 계수(check/variable node 병렬 수) 결정.
7. N/R/P 파라미터 조정으로 멀티표준 다중 code length 지원(Table 1).
8. HW: Verilog, 부분병렬 아키텍처(DC+DP: VNPU/CNPU/VN/CN/IMB/PMMB), Modified Min-Sum(MMS, 4bit intrinsic/2bit extrinsic, Tm=3,W=5,w=1), Xilinx Virtex2, 64MHz.
9. 결과: 제안 행렬이 2L-HQC 대비 BER 0.3dB 우수(@10⁻⁵), PEG 대비 0.1dB 미만 열세(@10⁻⁶), 평균 iteration 소폭 감소. LUT/Register는 code length 무관 일정, BRAM 용량만 증가. Avg throughput 44~55Mbps.
10. 한계: throughput Mbps급으로 NAND ECC(Gbps)에 미달, 정정능력 개선이 아니라 유연성/메모리 절감이 목적, 단일 고rate 부호엔 오버엔지니어링.
