### B. 알고리즘 요약

1. 대상: separable circulant-based(SCB) 정규 QC-LDPC, PCM `H = σ^{f(i,j)}` (`f(i,j)=a(i)·b(j)` 분리형), array-based/Tanner 구성 포함, 예제 `r=5`(col weight 5), `p=67`, `N=4489`, rate `0.9263`.
2. 풀려는 문제: 유한길이 LDPC의 고SNR/저FER error-floor는 trapping set의 부분집합인 absorbing set이 주원인.
3. 핵심 모델: `(a,b)` absorbing set = 각 변수노드가 짝수차수 이웃 > 홀수차수 이웃인 서브그래프(bit-flip 안정), AWGN 채널.
4. 핵심 기법: 각 absorbing set에 Cycle Consistency Matrix(CCM) `M` 정의 — 사이클별 `Σ(i_{m-1}-i_m)u_m=0 mod p`의 계수행렬, `M·u=0`이 존재조건.
5. 핵심 식: absorbing set 존재의 필요조건은 `M`이 full column-rank가 아닐 것 → RSF/CSF로 `det M≠0 mod p` 강제하면 해당 set 제거.
6. 핵심 기법 2단: Corollary 1 — 작은 absorbing set의 VN그래프가 큰 것의 부분그래프면, 작은 것 제거 시 큰 것도 제거(계층적 소거).
7. 부수 디테일: row-selection function(RSF) `a(i)`로 (4,8)(5,9)(6,8) 동시 제거, 예 `[0,1,2,4,17]`(p=67), `[0,1,2,4,7]`(p>179).
8. 최적화 절차: VN그래프 크기 순으로 dominant absorbing set을 CCM full-rank화 → EAB 대비 `O(p^3)` 개수를 0으로.
9. 결과: FPGA 시뮬(200 iter, Q4.2 고정소수점, sum-product)에서 SR-SCB가 EAB 대비 error-floor 기울기 급해지고 저error율 최소 1자릿수 개선, (4,8)(5,9)(6,8) 완전 소거(Table I).
10. 한계: `r=5`·prime `p` 특정 계열 국한, row-only로는 (6,10)/(6,4) 잔여 → column selection 추가 필요, 대상 부호가 SCB separable 구조여야 적용.
