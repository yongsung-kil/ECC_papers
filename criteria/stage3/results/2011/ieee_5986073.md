### B. 알고리즘 요약

1. 대상: rate-compatible time-varying `(491,3,6)` LDPC 콘볼루션 코드(LDPC-CC), 5개 code-rate(1/2~5/6), 90nm CMOS 칩, input 6bit 양자화.
2. 문제: LDPC-CC는 near-Shannon·가변길이·간단 인코딩 장점에도 긴 latency·고전력·낮은 throughput으로 실용화 저조(기존 최대 600Mb/s).
3. 모델: sliding window(ms+2) trellis를 processor 사슬로 복호, processor 1개 = LDPC-BC의 1 iteration에 해당.
4. 핵심기법1(OVA): on-demand variable node activation 스케줄링으로 VN 활성 위치를 각 CN 입력 직전으로 이동(layered 유사) → 수렴 2배 가속.
5. 핵심 식: VN-to-CN 메시지가 공통항 공유 → `n2 = n1 - m2 + m1'`로 pre/post-SVNU 분할 계산, 채널값을 메시지에 은닉해 채널값 저장(메모리 17%) 제거.
6. 핵심기법2(folding): CNU/VNU를 folding factor ρ배 복제해 throughput `ρ×fclk`, 조합회로만 주로 복제, FIFO를 ρ분할.
7. 구현: pre/post-SVNU retiming으로 메시지 bit-width `(w+1)`bit화·critical path 1 adder 단축; 정규화 min-sum(scaling 0.875), 고정소수 (6,2).
8. 메모리: irregular LDPC-CC에 register FIFO(고전력)·memory FIFO(충돌) 모두 부적합 → hybrid-partitioned FIFO(연속 sector 병합, 초과분 register)로 50% 메시지를 3개 2-port memory(10.5Kb)에 저장, clock buffer 54% 감소.
9. 결과: 90nm 2.24mm2/479K gate, ρ=12로 198MHz에서 2.37Gb/s/284mW(1.2V), 0.8V로 90.2mW·0.0114nJ/bit/proc; processor 10→5로 동일 BER(latency 절반).
10. 한계: 부호가 LDPC-CC(콘볼루션)라 block LDPC와 구조 상이, error-floor/정정능력 개선은 아님, NAND soft-read 무관.
