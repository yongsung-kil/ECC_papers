### B. 알고리즘 요약

1. CMMB(len 9216)/DTMB(len 7493) 방송표준용 멀티스탠다드 LDPC 디코더로, perfectly structured(QC/SDSM)와 not-perfectly-structured(DDSM 포함) 부호를 한 HW로 지원.
2. TDMP(layered/turbo message-passing)는 TPMP 대비 절반 iteration으로 수렴하나, DDSM(double-diagonal sub matrix)이 있으면 한 변수노드에 posterior가 2개 생겨 memory access conflict(cutting edge) 발생.
3. NMS(normalized min-sum, `α`=0.75 CMMB / 0.625 DTMB) 채택으로 extrinsic 갱신 복잡도·메모리 축소.
4. 핵심 해법: PMP에서 추가 prior message `Lv→c0+c2 = Lv^{k-1} - R_{c0}^{k-1} - R_{c2}^{k-1}` (식16)을 하나 생성해 저장, 두 posterior를 별도 저장하지 않음.
5. VNP에서 `Lv(temp)=Lv→c0+c2 + R_{c2}` (식17), `Lv = Lv(temp) + R_{c0}` (식18)로 정확한 posterior 복원 - conflict 없이 정상 복호 경로 유지.
6. CMMB의 DDSM은 (1) 레이어당 최대 1개 (2) identity+QC 구성이라는 성질을 이용, layer 전환 시 삽입되는 대기클록(3개)을 재사용해 식16~18을 추가 HW 없이 처리.
7. extrinsic 저장 최적화: min/submin/index/sign을 클록당 7bit씩 분할 read/write하여 메모리를 193,536->96,768 bit(50%)로 축소.
8. recover와 accumulate를 한 프로세서로 병합 (2's complement 변환을 add/sub 전환으로 대체해 가산기 절감).
9. 결과: SMIC 0.13um, core 4.75mm2, 200MHz, throughput 230~366Mb/s@10iter, power 48.4mW(CMMB)/130.9mW(DTMB)@5iter; TPMP(25iter) 대비 TDMP+해법(15iter)이 BER 우위.
10. 한계: 방송표준(CMMB/DTMB) 전용, DDSM/IRA 구조 전제라 NAND용 순수 QC-LDPC에는 conflict 문제 자체가 비적용; soft-read/NAND 채널 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NMS(min-sum) CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유(min-sum), 신규성 없음 |
| layered/TDMP posterior 갱신 | `decoder.cpp` `LDPC_Decoder()` | 스케줄만 유사, DDSM conflict는 우리 QC-LDPC에 비적용 |
| extrinsic 분할저장/메모리 최적화 | 대응 없음 | 우리 z=32 고정구조와 무관 |
| DDSM conflict 해결(식16~18) | 대응 없음 | Prime ECC는 SDSM(QC) 전용, double-diagonal 없음 |

적용 가치: **낮음** - 핵심 기여인 DDSM conflict 해결은 double-diagonal(IRA류) 부호 전제라 QC-LDPC 기반 Prime ECC에는 문제 자체가 존재하지 않는다.
