### B. 알고리즘 요약

1. 대상은 LDPC 컨볼루셔널 부호(LDPC-CC), rate `R=b/c(b<c)`, 무선(WiMAX 대비) 채널, `νs=ms·c` 제약길이.
2. 문제: 기존 LDPC-BC는 부호길이가 커 지연이 크고, LDPC-CC는 iterator 50~100개 요구로 지연·복잡도 과다, encoder/decoder 실현성 무시.
3. 핵심: QC 구조(`p×p` circulant/zero)를 LDPC-CC의 `Hi(t)`에 유도, base form `Hb`로 표현(`-1≤Hbj,i<p`), 병렬도 `p`.
4. 2단계 구성: 1단계는 수정 PEG(Algorithm 1)로 `Hb`의 비음수 위치 결정(check node 관점, unique relation area·girth 고려).
5. 2단계는 greedy(Algorithm 2)로 permutation factor 값을 girth `gk` 최소화하도록 탐색.
6. Encoder: H1의 우측을 identity로 최적화하여 `o(p)` 복잡도(기존 RU `o(νs)` 대비 감소), interleave network + p encode unit.
7. Decoder: MMSA-CC(수정 min-sum), 시간축 iteration을 공간축 N개 iterator 파이프라인으로 변환, 각 iterator에 p개 MMSA-CC core.
8. 수정 인자 `α`로 BP→MSA 성능 손실 보정, row-by-row 메시지 갱신.
9. 결과: WiMAX QC-LDPC-BC 동일 부호길이 대비 0.5~1dB 이득, error floor 개선, N=20 iterator로 수렴, 지연 1/p로 감소.
10. 한계: GPU 시뮬 검증뿐(GTX260, 15Mbps), 실제 HW 게이트카운트·throughput 미제시, 무선 부호 타깃.
