### B. 알고리즘 요약

1. 대상은 DVB-S2 표준 LDPC 부호 `(64800,32400)`, `(16200,7200)` (rate 0.5), Offset Min-Sum 복호.
2. 풀려는 문제: 위성 채널은 재전송 비용이 커 강한 FEC가 필요하나 LDPC 복호 iteration 수가 많아 속도 저하.
3. HSS(Horizontal Shuffling Scheduling = layered) 스케줄은 갱신된 V2C 메시지를 같은 iteration 내에서 즉시 사용해 FS(flooding)보다 LLR 수렴이 빠름(요구 iteration 약 2배 감소).
4. STEP: 첫 check node에 대해 C2V(`M_cv`) offset min-sum 갱신(식3: `min(|·|,|·|)-offset`, sign XOR) → 연결된 V2C 갱신 → SO 계산 → 다음 iteration 반영.
5. `N-K` step을 모두 수행하면 한 iteration 완료(HSS 구조).
6. CAET(Channel-Adaptive Early Termination): 초기 iteration `Ite`와 SCR threshold `Thr`를 설정.
7. 매 iteration 후 각 비트 HD의 sign을 계산, 직전 iteration과 sign이 바뀐 비율 SCR = `Σ|S_k - S_{k-1}|/N` 을 구함(식8).
8. 정지조건: `k ≥ Ite` 이고 `SCR ≤ Thr` 이거나 `k ≥ Imax` 이면 종료. SCR→0 이면 decodable.
9. Risk Index `RI`(식7)로 분석해 두 부호 모두 최소 iteration 9 필요, `Ite=9`, `Thr`=18%(64800)/5%(16200) 선택.
10. 결과: EbNo 1.0~2.2dB에서 평균 iteration 효과적 감소, BER 10^-7(quasi-error-free) 수용 가능. HW 미설계, 시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| HSS(layered) 스케줄 | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 스케줄) | Prime ECC 이미 layered 계열 스케줄 보유 |
| Offset Min-Sum C2V 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 자체는 이미 보유, offset은 미확인 |
| CAET(SCR 기반 조기종료) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료 사용 - 대체 접근 |
| HD/sign 계산 | `decoder.cpp` `VN_Cal_HD()` | SCR 계산에 HD sign 필요 |

적용 가치: **낮음** — layered 스케줄과 조기종료를 모두 이미 보유하며, SCR 기반 CAET는 CRC 조기종료 대비 명확한 이점이 없고 부호도 DVB-S2 전용.
