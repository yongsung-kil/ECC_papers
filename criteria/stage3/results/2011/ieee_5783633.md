### B. 알고리즘 요약

1. 시스템: WiMAX QC-LDPC 114개(6 class × 19 코드, z별 확장) 전모드 codec을 90nm CMOS로 구현. PCM은 `z×z` circulant 배열, `Mb×Nb` prototype. 예: rate-5/6, z=24, (2304,1152) 비교 부호.
2. 문제: 서로 다른 rate·length WiMAX 부호들을 HW 재사용 극대화하며 저복잡도로 복호하기.
3. 복호: layered MPD(LMPD)에 offset min-sum 사용. C2V `R_ij = S_ij · max(min_{j'≠j}|Q_j'i| - β, 0)`, `β`는 offset 상수, sign은 XOR; V2C `Q_ji = Λ^{k,ℓ-1} - R_ij^{k-1}`, layer 끝에 APP `Λ` 갱신(식1).
4. 핵심: **LMPD-ICM** — H의 행을 z개 layer로 나누면 각 layer의 core matrix(non-zero 열)가 서로 column-permuted 버전이라 **모든 layer가 동일 라우팅망 공유**, 같은 class 19개 부호도 공유.
5. 의미: QC의 순환 성질로 layer별 주소가 `(s(j)+ℓ) mod z`로 결정 → 주소생성기·라우팅이 매우 단순, multi-length 내장 지원.
6. Unified task processor: 6 class 중 최소 layer 크기 4를 병렬도로 채택(4 CNU + 4 VNU, 4-input). 한 layer를 최대 16값/cycle 제약에 맞춰 5개 task로 분할(computer search로 배치), 24 APP 메모리 블록에서 16값씩 읽기.
7. 구현 디테일: CNU 4-stage(CMP4→S-CMP4로 최소2값 탐색→Sign→C2V Calculator), VNU 10-stage pipeline. CN/VN 동시 수행, layer ℓ의 APP 갱신 중 CN은 layer ℓ+1 처리.
8. 데이터 해저드: layer 간 의존성을 idle cycle 없이 **delta-based update**(Parallel-update Unit)로 흡수, computer search로 스케줄.
9. 결과(post-layout, (2304,1152)): 90nm, 400MHz, 7-bit 양자화, 8-12 iter, 처리량 133-240Mb/s, 면적 0.679mm², TAR 196-353 Mb/s/mm², 에너지 516-845 pJ/bit — 선행 [1][2] 대비 최소 면적·최고 에너지효율. BER=10^-5 @2.15dB.
10. 한계: 성능(BER) 개선이 아니라 면적/처리량/에너지 목표, WiMAX 다부호·z가변 표준 특화, gatecount·throughput은 Mb/s급(Gbps 아님).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 연산 (`max(min-β,0)`, sign XOR) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min-sum과 동종, offset β 적용은 참고 가능하나 이미 보유 계열 |
| layered decoding + delta-based APP 갱신 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | Dual-Update(even/odd 교대) 스케줄과 유사 개념, 수렴 가속 참고 |
| QC 순환 주소 `(s(j)+ℓ)mod z` 기반 라우팅/메모리 | [decoder.cpp](../Prime_ECC_3.1_Claude) HW 모델(z=32, HCU, GT) | Prime ECC 메모리/라우팅 병목과 직접 접점, 단 z 가변 다부호 전제라 z=32 고정 구조와 정합 필요 |
| unified task processor / CNU-VNU pipeline | [decoder.cpp](../Prime_ECC_3.1_Claude) pipeline 6-stage 모델 | pipeline 구조 참고 가능하나 병렬도·task 분할이 우리와 다름 |

적용 가치: **중간** — QC 순환성을 이용한 고정 라우팅망·주소생성·layered delta 갱신·offset-MS는 Prime ECC HW/스케줄 레이어와 접점이 있으나, WiMAX 다부호·z가변에 특화되어 단일 z=32 고정 부호인 Prime ECC로의 직접 이식은 부분적이고 성능 개선 기여는 아니다.
