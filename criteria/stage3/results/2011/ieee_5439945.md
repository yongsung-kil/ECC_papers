### B. 알고리즘 요약

1. 대상은 672-bit rate-1/2 irregular LDPC(IEEE 802.15 표준, base matrix 16×32, `z=21` 예시 및 구현은 `z=32`) layered 디코더.
2. 문제: 기존 layered 디코더는 shuffle/reshuffle을 위해 SN1, SN2 두 개의 switch network가 필요해 라우팅 HW 복잡도가 큼.
3. 핵심 관찰: 한 sub-iteration 내에서 reshuffling(SN2)과 다음 constituent code의 shuffling(SN1)이 시간적으로 중첩되므로 두 연산을 하나로 합칠 수 있음.
4. 핵심 기법: offset shifting value `s_{m,n}`를 사전계산해 SN1 하나만으로 처리 — `s_{m,n} = a_{m,n} ⊟ b_{l,n}`(⊟는 mod-z 뺄셈, null 처리 포함), Algorithm 1로 생성.
5. 의미: SN1의 누적 offset이 SN2의 역permutation을 흡수 → 중복 shift 연산 제거.
6. 출력 정렬 정보 `y = E·x`(Algorithm 2의 `d_n`)로 하드디시전을 추가 HW 없이 출력 포트에 매핑, 별도 unshuffle 불필요.
7. 구현 디테일: `(q,f)=(6,2)` 양자화, CNBP는 min-sun(Log-BP 근사) 4 pipeline stage, SN은 불필요 switch 제거한 Benes network(3 pipeline stage).
8. block parallel: C2V 메모리 뱅크 수를 row weight에 맞춰 constituent code의 모든 메시지를 1 clock에 처리.
9. 결과: Virtex-4 FPGA, 335MHz, 822Mb/s(8 iter), 표준 layered 대비 slice 9.3% 감소, SNR 3dB에서 약 3 iteration 수렴.
10. 한계: 정정능력/BER 개선 없음(동등 성능 유지), gate count는 오히려 [9]보다 높음, 특정 QC 구조·pipeline 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset permutation / SN1 단일화 | `GT.cpp` `Make_GT_HW()` | 라우팅 edge/MUX 선택 최적화 관점에서 유사, 다만 GT는 edge 축소 목적 |
| layered CNBP(min-sum 처리) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | layered 스케줄·CN 연산 대응 |
| (6,2) 양자화 | `decoder.cpp` `VN_Cal_HD()` 등, `ecc_data.h` `PARAM_LLR` | 양자화 테이블 대응 |
| 출력 정렬(y=E·x) 매핑 | 대응 없음 | Prime ECC는 별도 unshuffle-free 매핑 구조 미기재 |

적용 가치: 낮음 — 순수 shuffle network 라우팅 HW 최적화이며 min-sum/양자화는 이미 보유, Prime ECC의 z=32·HCU·GT 구조와 정합해도 성능 이득 없음.
