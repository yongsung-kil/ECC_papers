### B. 알고리즘 요약

1. 대상 시스템은 WiMAX(IEEE 802.16e) QC-LDPC, 대표 부호 `(2304, 1/2)`, expansion factor `z`(24~96 가변), rate 1/2~5/6.
2. 문제: partially-parallel 디코더에서 두 phase(CNU/VNU) 오버랩과 unfolding 시 sum memory 접근 충돌·data dependency가 throughput 병목.
3. 복호 알고리즘은 normalized min-sum의 CNU 함수 `(1)`을 TDMP(각 block row를 constituent code로 처리)의 SISO 엔진에 대입한 "improved TDMP".
4. CNU는 first/second min magnitude, 첫 min 위치, sign bit의 component set만 저장하고 compare-select + shift-add로 구현 (정규화 factor `beta` 적용).
5. 핵심 아키텍처: dual-path(A/B 두 nonzero sub-matrix 동시 처리) + fully-overlapped 두 phase, sum memory에 posterior sum과 prior message를 공유 저장(FIFO 제거, 11,520 bit 절감).
6. symmetrical six-stage pipelining: CNU/VNU 각 4단, write는 read보다 항상 3클록 후 발생 → read 충돌만 해결하면 write 충돌 자동 해소.
7. 메모리 충돌 해결: base matrix를 block column index mod 4로 4 quadrant partition, block column/row interleaving + nonzero sub-matrix reordering으로 각 block row가 4 bank에 고르게 분산.
8. read-write bypass: VNU write와 CNU read의 tag/addr 일치 시 메모리 우회 직접 전달 → sum memory 접근 24.3%~48.8% 감소(전력 지배 요인).
9. 결과(실측, SMIC 0.13um): 214MHz/10 iter에서 gross 847~955 Mb/s, 342~397 mW, 39~46 pJ/bit/iter; 6-bit 양자화는 7-bit 대비 미미한 손실, BER 10^-x 이상 error floor 없음.
10. 한계: WiMAX 특정 부호·행렬 구조 종속(interleaving/reordering이 행렬에 맞춰짐), 정정능력 개선은 없고 throughput/전력 최적화에 국한.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| improved TDMP + normalized min-sum CNU | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()`, `CNU_Update_New_Mag()` | min-sum 코어는 우리도 보유(min1/min2+sign); TDMP layered 스케줄은 우리 Dual-Update와 유사 개념 |
| component set(min1/min2/pos/sign) 저장·recover | `decoder.cpp` `C2V_Cal_New_Sgn()`, `C2V_Cal()` | 우리 min-sum 메시지 저장과 동일 계열, 신규성 낮음 |
| sum memory quad-partition / read-write bypass | 디코더 HW 모델 (z=32, pipeline) | 대응 개념 부분적; 우리 6-stage pipeline과 스케줄 다름, 메모리 뱅킹 아이디어만 참고 |
| block column/row interleaving·reordering | `ecc_top.cpp` `Load_PCM()` (H-matrix 로드) | 부호 전처리 단계 대응 가능하나 WiMAX 행렬 종속 |
| permutation network (96-way logarithmic shifter) | 대응 없음 | 우리는 z=32 고정 shift, WiMAX 가변 z용 network 불필요 |

적용 가치: **중간** — normalized min-sum+TDMP 코어는 이미 보유하여 신규성 낮으나, sum memory quad-partition·read-write bypass의 메모리 접근/전력 절감 아이디어는 HW 관점에서 참고할 만함.
