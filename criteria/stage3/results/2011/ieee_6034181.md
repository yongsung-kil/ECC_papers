### B. 알고리즘 요약

1. 시스템: `b×n` 2D 비트 배열(column=storage device), `n` 고정, `b→∞`. RAID류 저장장치 모델.
2. 채널: 혼합형 erasure 모델 — combinatorial로 실패 column 수 `≤r` 제한 + 각 실패 column 내부는 확률 `ε`의 BEC(binary erasure channel).
3. 문제: 기존 algebraic array code(MDS)는 full-column erasure만 가정, `r` 초과 시 급격한 실패. 부분 실패/큰 `r`에 취약.
4. 핵심 구성: 각 parity 제약이 column당 정확히 1비트만 포함하도록 제한한 확률적 LDPC ensemble(`(l,d)-regular` 또는 irregular).
5. 핵심 분석 도구: 비실패 column의 VN은 초기 메시지 0 → 그래프에서 제거. 실패 `r` column만 남긴 induced decoding graph `G_R`로 분석.
6. 핵심 결과 Thm1: `(l,n)-regular` array ensemble은 rate `1-l/n`, threshold는 `(l,r)-regular` 1D LDPC와 동일.
7. 확장 Thm2/3: check degree `d<n`이면 induced check-degree 분포가 `T_i(n,r,d)=[C(r,i)C(n-r,d-i)/C(n,d)]`로 irregular화, 길이 `n` 의존성 발생.
8. 최적화: bounded-degree 분포에 대한 linear-programming으로 λ(x)/ρ(x) 설계, 여러 `r`값에 threshold spectrum 조정.
9. 결과: `n=16, r=4`에서 λ(x) 설계로 rate `0.7553`, threshold `0.974`(≈1) 달성 — MDS array code에 근접하되 디코더가 단순.
10. 한계: erasure-only 이론, 실측/HW/BER-vs-SNR 없음. systematic code·부분 erasure 조합은 future work.
