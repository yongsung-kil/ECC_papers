### B. 알고리즘 요약

1. 시스템: AWGN 채널, LLR 기반 min-sum 반복복호. 검증 코드 regular `(1024,512)`, `(1296,864)`, `(648,324)`, max 10 iteration.
2. 문제: 표준 min-sum(MSA)은 SPA 대비 성능 열화; normalized/offset MSA의 scaling factor는 code rate·length마다 exhaustive search 필요.
3. 배경: check node update `Cmn = Π sign(V)·min|V|` (식2)가 magnitude를 과대추정 → 정규화로 보정하는 계열.
4. 핵심 기법: 정규화를 check node가 아닌 variable(vertical) 처리에서 수행하되, 연속 두 iteration의 부호 비교로 분기.
5. 분기식: 임시 `V_tmp = Rn - Cmn` (식4a) 계산 후, `sign(V_tmp)==sign(V^{i-1})`이면 `Vmn = sf1·V_tmp` (식7), 다르면 `Vmn = sf2·(V_tmp + V^{i-1})` (식8).
6. 핵심 아이디어: 부호가 바뀌면 magnitude 증가가 크므로 더 작은 `sf2`로 강하게 억제, 같으면 완만한 `sf1`로 보정.
7. 파라미터: `sf1=0.5`, `sf2=0.25` 사용 → 각각 divide-by-2, divide-by-4 = 우측 shift로 곱셈 불필요.
8. HW: shift register 2개 + sign comparator + MUX로 구현; 기존 MMSA[11] 대비 shift register 1개만 추가.
9. 결과: TABLE 1에서 New MSA가 MMSA·SPA보다 낮은 BER (예 `(1024,½,4.5dB)`: SPA 2.0e-3, MMSA 1.6e-3, New MSA 5.9e-4); 중·단문에서 SPA도 상회.
10. 한계: 시뮬레이션(10 iter)뿐, HW는 개념도만(합성·gatecount·throughput 없음), 이전 iteration 메시지 저장 필요, error-floor·장문 코드 미평가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| variable message 2-factor 정규화(식7/8) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `VN_Cal_HD()` 등, `LDPC_Decoder()` 이터레이션 루프 | VN 갱신에 sf1/sf2 shift 적용 형태로 이식 가능(난이도 낮음), 단 이전 V 메시지 저장 필요 |
| check node min-sum 연산(식2) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC min1/min2 min-sum과 동일, 신규성 없음 |
| shift-register 기반 scaling HW | 대응 없음 (Prime ECC는 magnitude 양자화 테이블 기반) | divide-2/4 shift는 개념 이식 가능하나 Prime ECC는 테이블 기반이라 구조 정합 필요 |
| 이전 iteration 메시지 저장 | 대응 없음 | z=32 병렬 구조에 iteration 간 V 버퍼 추가 시 메모리·라우팅 검토 필요 |

적용 가치: **중간** — VN 갱신부 2-factor 정규화 아이디어는 낮은 난이도로 min-sum에 얹을 수 있으나, Prime ECC의 min-sum과 개념 중복 + 이전 메시지 저장 메모리 오버헤드가 있어 실이득 검증 필요.
