### B. 알고리즘 요약

1. 시스템: 저복잡도 bit-flipping LDPC 복호, AWGN/BPSK, 예제 Code1 PEGReg252x504 (`M=252,N=504`, rate 1/2), Code2 HRGC 273.
2. 문제: BP는 성능 좋으나 복잡도 큼; bit-flip은 저복잡도지만 iteration 많아 수렴 느림.
3. 모델: syndrome `s_m=Σ x_n H_mn`, inversion function `E_n=Σ_{i∈M(n)} Π_{j∈N(i)} x_j` (실패 체크 수 기반, GDBF는 `+x_n y_n` 상관항 추가).
4. 핵심 기법(ATBF): 비트별 threshold `λ_n`, `E_n≥λ_n`이면 flip, 아니면 threshold를 `λ_n(k+1)=θλ_n(k)`로 낮춤 (gradient descent, min-search 불필요).
5. 핵심 식: `θ=Σ_{l=1}^{L} 2^{-l}` — scaling을 곱셈기 없이 shift+add L개로 구현 (예: 0.625 = 1/2 + 1/8, shift 1·3).
6. 확장(DSSA): coarse(θ1=0.5, 큰 step)와 fine(θ2=0.97, 작은 step)을 objective function `f(x,k)` 비교로 스위칭, 좋은 branch 채택 후 coarse decoder off.
7. 구현 디테일: 비트 프로세서 자율(비트간 독립), 조기종료 `f(x,k)=f(x,k-11 or -10)`이면 halt(진동 대비 2조건).
8. 최적화: θ는 code·SNR 종속으로 1000블록 평균 flipping threshold 관찰로 추정, 학습형 아님.
9. 결과: dual θ=0.625/0.97이 단일 θ=0.97과 BER 동등하나 avg iteration 6dB에서 58→8; early-stop은 4dB에서 43→15로 감소.
10. 한계: HW 미설계, 시뮬만, low column-weight(=3) 코드 대상, BER는 IRRWBF/BP 대비 열위, soft-read 미고려.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bit-flip / GDBF 복호 알고리즘 | 대응 없음 | Prime ECC는 min-sum message-passing, bit-flip 패밀리 미구현 |
| adaptive threshold(θ) 이터레이션별 갱신 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration 구간별 threshold 테이블 개념은 유사하나 대상량(LLR vs flip-threshold) 다름 |
| 조기종료(objective function 정체) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | Prime ECC는 CRC 기반 조기종료로 이미 보유, 방식 상이 |
| 저복잡도 shift-add scaling(2^-m) | `decoder.cpp` `CNU_Update_New_Mag()` | min-sum normalized/scaling factor 참고 여지 있으나 bit-flip 전용 |

적용 가치: **낮음** — dual step-size scaling으로 iteration을 줄이는 아이디어는 흥미롭지만 bit-flipping 디코더 전용이라 min-sum 기반 Prime ECC에 직접 이식 불가, BER도 BP 대비 열위이며 NAND soft-read 미고려.
