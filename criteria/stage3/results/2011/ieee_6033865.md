### B. 알고리즘 요약

1. 대상은 base matrix `B`를 edge spreading으로 시간축 결합한 종단 LDPC convolutional code 앙상블((3,6)-regular, rate `R=1/2`, memory `mcc`).
2. 문제: 종단 LDPC convolutional code는 threshold saturation으로 BP threshold가 우수하나, 표준 block BP를 쓰면 심볼당 복잡도가 결합 블록 수 `L`에 선형 증가.
3. 채널은 AWGN, 메시지는 LLR, DE(Chung의 discretized DE)로 위치별 `Pb(t)` 분석.
4. MPS-I(flooding): 모든 CN→VN을 매 iteration 갱신. `Pb(t)`가 양 끝에서 중앙으로 전파됨을 관찰.
5. MPS-II(target-based): 위치 `t`의 `Pb(t)<Pbmax`이면 그 위치 노드 갱신 제외 → 유효 iteration `Ieff=1/L·ΣIt` 감소.
6. MPS-III(improvement-based): 개선률 `ΔPb(t)=(Pold-Pnew)/Pold < θ`인 위치의 VN 갱신 추가 제외. 결과적으로 `Ieff`가 `L`에 무관해짐.
7. MPS-IV(window decoding): 폭 `W` 창을 위치 `p`마다 `Ipwin` iteration 갱신 후 shift. MPS-III의 자연스러운 근사이자 실장 용이한 형태.
8. window는 `Pbmax=10^-5` 달성 시 이동(adaptive `Ipwin`), 예제 최적 창 크기 `W=6`. Ensemble B(edge spreading B0=[2,2],B1=[1,1])가 더 작은 W로 목표 달성.
9. 결과: 시뮬(N=500,1000)에서 window 복호가 block code 대비 BER·복잡도 모두 우수, 복잡도 이득 ΔC=52~78%; 지연이 window 크기로 결정되어 `L`에 무관.
10. 한계: HW 미설계·시뮬 수준, AWGN·rate 1/2 regular 예제 중심, convolutional(공간결합) 구조 전용.

| target 확률/개선률 기반 조기 노드 비활성 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` (조기종료) | 조기종료 개념만 유사, 위치별 스케줄은 대응 없음 |
