### B. 알고리즘 요약

1. 시스템: binary QC-LDPC(부호 = CPM 배열), 예제 (8176,7156) 4/32-regular rate 0.8752, (3969,3213) irregular rate 0.81. BPSK/AWGN, 6~8bit 양자화.
2. 문제: LDPC 디코더 HW의 실병목은 메모리 대역폭. 기존 FPGA 구현은 block RAM 워드폭(36bit)에 메시지 1개(8bit)만 저장 → 대역폭 ~22%만 사용.
3. 모델: NMSA(normalized min-sum) - SPA의 감소복잡도 근사. CN update에 정규화 인자 곱, VN update로 reliability 갱신, syndrome=0이면 조기종료.
4. 핵심기법1 (vectorization): block RAM의 가변 폭을 활용해 한 워드에 K개 메시지 packing → CNU/VNU를 K배 복제해 throughput 최대 K배. double buffering으로 read alignment 제거.
5. 핵심 디테일: message packing + sequential alignment unit(1-cycle latency)으로 CN/VN 접근순서 충돌 회피, VOMP(vector overlapped message passing)로 CNU/VNU 오버랩(quad-port를 2배 클록 emulation).
6. 핵심기법2 (folding): block RAM 깊이를 활용해 여러 CPM을 한 물리 block RAM에 매핑(memory virtualization) → circulant 수 > block RAM 수인 대형부호(DVB-S2 등)도 구현. folding factor F.
7. 구현 디테일: irregular 부호는 block 열/행을 degree 내림차순 정렬 후 그룹화(VNU 입력 port 최소화), VRU/CRU reorder unit으로 메시지 라우팅.
8. 자동화: QCSyn(Python) - H matrix + FPGA 자원사양 입력 → vector/folded Verilog 자동생성(설계기간 개월→일), bit-accurate C model로 검증.
9. 결과(실측): Virtex-4에서 (3969,3213) 1.162 Gbps, (8176,7156) 713.8 Mbps; vectorization으로 throughput 8.4배(99.1→830.6 Mbps), 에너지효율 10.26→4.25 nJ/bit. (8176,7156)은 error floor 없음(BER 1e-12↓).
10. 한계: FPGA block RAM aspect-ratio 특화 HW 최적화라 ASIC 이식성 제한, NMSA 정정성능 자체 개선은 없음(순수 구현효율).

적용 가치: **중간** — NMSA·overlapped message passing은 우리 min-sum·병렬 디코더와 정합해 참고 가치가 있으나, 핵심 기여인 vectorization·folding이 FPGA block RAM aspect-ratio에 특화돼 우리 ASIC(RTL) 고정 메모리 구조에는 직접 이식되지 않음.
