### B. 알고리즘 요약

1. 대상 시스템은 RLL(0,3) 제약 4-level PAM 광기록 채널, AWGN only (jitter 무시), PEG-LDPC 사용.
2. 문제: 고rate RLL 부호는 rate loss가 크고, deliberate flipping으로 삽입한 오류를 고rate 부호가 정정하기 어렵다.
3. 모델: 오류벡터 `e = r ⊕ z`, AEWE `Δ2e(X) = (1/M)Σ||f(z)-f(z⊕e)||²`로 signal labeling(Gray vs natural)의 유클리드 가중치 정의.
4. 핵심기법: regular inter-leaver로 코드워드를 flipping-error 절반과 non-flipping 절반으로 분리, flipping 오류를 고차(high-degree) VN 강보호 섹션에 배치하는 UEP 할당.
5. 강보호 섹션의 고차 VN이 높은 오류정정능력을 가져 flipping bit를 iterative decoding 몇 iteration 내에 복구.
6. UEP scheme type I(short-distance flipping)과 type II(long-distance flipping, nearest-neighbor 적은 labeling) 두 변형 제안.
7. 부호 파라미터: `VND=[2,5]` `λ=[0.5,0.5]`, `CND`는 rate별 상이(rate0.65 [10,11], 0.75 [13], 0.8 [16,17,18]).
8. outer iteration `Uo=15`, inner iteration `Ui=1`로 iterative equalization+decoding 수행.
9. 결과: type II가 non-flipped 시스템 대비 rate0.75에서 0.1dB 이내, RLL code [7] 및 [6] 대비 BER 개선(단 [6] flipped는 error floor 발생).
10. 한계: HW 미설계, 시뮬만, gate/throughput 없음, 광기록·RLL·PAM 특화로 일반화 제약.
