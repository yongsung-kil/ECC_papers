### B. 알고리즘 요약

1. Regular binary LDPC(M×N PCM, column weight λ, row weight ρ) 대상, BPSK+AWGN 채널에서 hard-decision 수신열 `z`와 syndrome `s=zH^T` 사용.
2. 기존 BF는 저복잡도지만 성능이 낮고, WBF/I-WBF/RRWBF는 soft 정보(실수 덧셈)로 성능을 높이지만 복잡도가 크다는 문제를 해결하려 함.
3. 채널/노이즈 모델은 별도 정의 없이 표준 BPSK-AWGN 가정, 디코더 입력은 hard-decision bit뿐.
4. 핵심 기법: 각 code bit `z_i`의 unsatisfied check 수 `u_i`, 최댓값 `u_max`를 구해 `u_i=u_max`인 비트를 candidate bit로 지정. Check `m`의 candidate bit 개수 `c_m`으로 가중치 `w_m=c_m-1` 정의, `r_i = Σ_{m∈M(i)} w_m·s_m`(unsatisfied check만) 계산.
5. `r_i`가 작을수록(해당 unsatisfied check에 candidate bit가 적을수록) 오류 확률이 높다는 것이 핵심 식의 의미 — `r_i = Σ P_{i,j}`(컬럼 간 상관값 합)로 재해석되며, candidate column과의 상관이 낮을수록 오류 가능성 ↑.
6. `u_i=u_max`이면서 `r_i=min{r_i}`인 비트(들)을 한 iteration에 동시에 flip — WBF류가 매 iteration 1비트만 flip하는 것과 달리 다중 비트 동시 flip으로 수렴 가속.
7. 부수 트릭: column weight가 큰 코드(예: EG-LDPC, λ=32)에서는 threshold θ를 고정하지 않고 실패 시 적응적으로(θ=17까지) 낮춰 추가 디코딩 허용.
8. 학습/최적화 절차 없음(고정 규칙 기반), 정수 덧셈/비교 연산만 사용(step4~6 복잡도: 정수 덧셈 최대 `K_F(λ-1)` + `K_F-1`).
9. 결과: (1057,813) Gallager code 기준 평균 iteration 수(SNR 4~7dB)가 CBBF 8.41→1.09로 WBF(12.51→1.46)/I-WBF/RRWBF보다 적고, (1008,504) 코드에서는 WBF를 능가하고 I-WBF에 근접하는 BLER 달성.
10. 한계: HW 설계·게이트/throughput 수치 없이 시뮬레이션(BLER, 평균 iteration)만 제공, EG-LDPC(λ=32)에서는 CBBF가 BF 수준에 그치고 다른 soft 알고리즘보다 열세, high-SNR에서 단순 BF가 최선인 코드도 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| hard-decision bit-flipping 디코딩 알고리즘 전체 | `decoder.cpp` `LDPC_Decoder()` | HD 모드 디코더의 저복잡도 대안/보조 알고리즘으로 검토 가능, min-sum과는 다른 결정 규칙(정수 상관 기반)이라 통합 시 별도 경로 필요 |
| candidate bit 선정 및 다중 비트 동시 flip | `decoder.cpp` `VN_Cal_HD()` | HD VN 갱신 로직에 대응되나 CBBF는 unsatisfied check 카운트+상관합 기반이라 기존 min-sum VN 연산과 다른 별도 로직 추가 필요 |
| adaptive threshold(θ) 감소 전략 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | iteration/threshold 테이블 조정 개념과 유사하나 LLR이 아닌 HD unsatisfied-check count 기반이라 직접 재사용은 어려움 |
| 조기종료(all-zero syndrome / candidate bit 소진 시 stop) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | syndrome 기반 조기종료는 이미 CRC로 구현되어 있어 중복 기여, 추천도에 감점 요인 |

> 적용 가치: 중간 — soft 정보 없이 정수 연산만으로 WBF급 성능·더 빠른 수렴을 보이는 hard-decision 알고리즘이라 저복잡도 HD 보조 경로 후보이나, 이미 min-sum/HD 경로를 보유한 Prime ECC 대비 성능 우위가 코드 종류에 따라 일관되지 않아 재검증 부담 있음.
