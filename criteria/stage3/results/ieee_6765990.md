> 총평: normalized MSA를 min1/min2/pos/sign 압축저장 + 2-파이프라인 row-layered로 재정식화해 면적·메모리·수렴을 개선한 실제 ASIC 디코더로, min-sum HW 개선 관심사와 부합하나 core 기법(min1/min2, row-layered)이 이미 보유와 상당 부분 겹쳐 이식 이득은 제한적.

### B. 알고리즘 요약

1. 대상은 binary QC-LDPC(802.11n의 (4,24,54,85) 부호, `p`=54 circulant, length 1296, rate 5/6), AWGN+BPSK.
2. 문제: QC-LDPC 디코더의 큰 면적·메모리와, I/O 클록 사이클이 throughput을 크게 잠식하는 것.
3. 채널 모델: AWGN, LLR 8-bit 양자화(4-bit로 floor 회피 가능하나 성능 위해 8-bit 선택).
4. 핵심 기법: normalized MSA(`α`=0.5)를 재정식화해 check→variable 메시지 `Rmn`을 `1stminRAM`,`2ndminRAM`,`posRAM`,`SignRAM`,`SignFlagRAM`에서 복원, 전체 C2V 메시지 저장 제거.
5. 채널 intrinsic LLR을 별도 저장 없이 `SumRAMn`(posterior LLR)에 직접 누적 → 메모리 추가 부담 제거.
6. 확장: parallel pipelined row-layered 스케줄 — nonzero submatrix 1개를 1클록에 처리, layer `m`의 VN 처리와 layer `m'=(m+1)mod M`의 CN 처리를 인터리브(2-stage 파이프라인).
7. 구현 트릭: 두 번째 파이프라인은 서로 다른 code word를 교대 투입하는 dual-decoder 구조로 I/O 클록을 Decoding에 흡수, throughput`=w×R×f`로 코드길이·iteration 무관화(식 3, 4).
8. 학습/최적화 절차 없음.
9. 결과: 동일 5 iteration에서 [7] 대비 0.7dB 이득, 동일 BER에서 iteration 절반; TSMC 0.13um ASIC 230MHz에서 767Mb/s, core 3.12mm², 메모리 38,508bit, I/O 포트 36개(기존 <10%).
10. 한계: 특정 802.11n 부호 예시, `α`=0.5 고정, NAND soft-read 비용 미고려, ASIC 합성/AWGN 시뮬 수준.
