### B. 알고리즘 요약

1. 대상: regular protograph LDPC, GF(2), column weight `γ=3`(일부 `γ=4`), rate `R=0.4~0.8`, block length `N=376~8064`, AWGN/uncorrelated Rayleigh 채널, SPA 최대 `I=100` iteration.
2. 문제: pseudorandom PCM은 성능은 좋으나 compact description이 없어 LUT/배선 메모리가 block length에 선형 증가 → HW 구현 비효율.
3. 핵심 아이디어: base matrix `Hb`(Mb×Nb)를 `J`개 constituent matrix `Qj`로 분해하고, `J×J` adjacency matrix `PJ`가 `Qj`들의 배치를 결정하는 multilevel-structured(MLS) protograph 구성.
4. 필수 제약: Constraint 1(각 `Qj` girth≥6), Constraint 2(모든 `Qj` 비영원소가 base matrix 동일 위치, 서로소) → 전체 PCM girth `g>4` 보장, edge 저장량이 `|E(Hb)|=Nb·γ`로 감소.
5. 저장 복잡도: pseudorandom은 `Nγ`(N=NbJ) edge 필요하나 MLS는 `Nbγ`만 필요 → 레벨수 `J`에 비례해 description 복잡도 감소.
6. adjacency 클래스: Class-I는 HCC(homogeneous coherent configuration) 기반 circulant → shift register로 생성, Class-II는 Latin square 기반(더 적은 제약, 평균 girth↑, 표현 다양성↑).
7. 추가 제약(HW용): Constraint 3(각 `Qj` 행·열당 비영원소 1개, `J≥ρ`)로 J memory block이 `ρ` edge 병렬 처리, Constraint 4(비영원소를 크기 `q` circulant로 치환)로 전체를 QC화 → 저장 `1/qJ`로 감소, shift register 인코딩.
8. 부호탐색: Latin square isotopy/conjugate 동형성(Claim 7.1)을 이용해 order-6에서 검사 대상을 방대한 수에서 88개로 축소하는 효율적 large-girth 탐색.
9. 결과: MLS(8064,4032,6,28)은 144 edge로 기술(MacKay(8064,4030)은 24192 edge). BER/BLER는 pseudorandom 대비 손실 없음(10^-6까지), Class-II(1008,504,6)는 Class-I 대비 +0.15dB, MacKay 대비 +0.21dB.
10. 한계: 순수 부호구성법으로 정정능력 자체 향상은 목표 아님(동등 성능 유지), HW 미설계(시뮬만), soft 양자화·NAND 무관.
