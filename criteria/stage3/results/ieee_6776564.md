### B. 알고리즘 요약

1. 시스템: regular column-weight-3 QC-LDPC (예: (7807,7177) rate~0.92, (2388,1793)), BSC 채널, message-passing 디코딩.
2. 문제: BP/min-sum은 trapping set으로 error-floor 발생, 기존 대책은 다중 디코딩 시도·고비용 post-processing·복잡한 VN 처리 필요.
3. 핵심가정: 메시지를 매우 작은 유한 알파벳(7-level=3bit)으로 표현, sign=비트추정, magnitude=신뢰도.
4. 핵심기법1 (FAID): VNU를 사전설계된 Boolean map/LUT `Φv`로 정의, 채널값 가중치가 대칭함수 `Ω`로 계산 - min-sum과 달리 비선형 LUT로 trapping set 진입 방지.
5. 핵심식 의미: CNU는 min-sum과 동일(부호곱·min), VNU LUT의 비선형성(인접 진폭갭>1 등)이 특정 error 이벤트 topology(trapping set) 교정을 개선.
6. 핵심기법2 (FAID diversity): 서로 다른 LUT의 `Nd`개 FAID를 순차 적용, (7807,7177)에서 4-error 패턴 보증 정정 목표, error-floor 한 decade 추가 개선.
7. 구현 트릭: bit-serial CNU에서 min1·min2 모두 계산(근사로 min2=min1 쓰면 floor 상승), UZD(unique zero detector)로 두 경우 구분, sign-magnitude 포맷.
8. VNU 최적화: 대칭 Boolean map을 Karnaugh 축약, min-sum VNU의 adder·saturation·포맷변환 불필요 -> 소면적, VNU 시분할로 CNU 속도 매칭.
9. 결과: SMIC 180nm 합성, 7-level FAID가 표준 min-sum과 동일 throughput에 면적 52%, 6-bit offset min-sum 대비 45% 면적, error-floor에서 float BP 능가, 7-table diversity는 +23% 면적만.
10. 한계: column-weight-3 + BSC로 범위 제한, LUT·trapping set이 부호 종속, AWGN·irregular·고차수 일반화는 pseudo-codeword로 비자명(저자 명시).
