### B. 알고리즘 요약

1. 시스템은 차세대 지상파 방송(UHDTV)용, QPSK/OFDM/AWGN, BCH outer + QEF 기준 BER<1.0E-7, iteration 30회.
2. 부호는 SC-LDPC(LDPC convolutional 기반), code length 269280, rate 10/15·11/15·13/15, 연결수 `L=17`로 DVB-T2 대비 약 4배 긴 부호.
3. 문제: 매우 긴 SC-LDPC를 HW로 구현하려면 복호 연산 복잡도와 HW volume을 줄여야 한다.
4. 부호 생성: irregular LDPC component base matrix `H`를 short cycle 최소화로 만들고, parity 부분을 `HL`/`HU`로 랜덤 분해해 대각 근처에 coupling하여 SC-LDPC 행렬 `HC` 구성(인코딩은 우측 double-diagonal 부착).
5. 복호는 TDMP(Turbo Decoding Message Passing) + offset min-sum 조합 — TDMP가 TPMP(sum-product)보다 빠름.
6. 핵심 식: row operation에서 sum-product의 Gallager `f(x)` 대신 offset min-sum(min 근사 후 `beta` 감산, 0으로 clip)으로 CN 메시지 계산.
7. 스케줄: 각 iteration에서 posteriori `Sn`를 갱신하고 hard decision 후 parity check(`x̂ H^T = 0`) 만족 시 조기 종료, 아니면 jmax까지 반복.
8. 최적화: code rate별 offset `beta`를 고정 C/N에서 BER 최소가 되도록 스윕(각각 0.60/0.65/0.65).
9. 결과: LLR 양자화 4~8bit 스윕, 6bit까지 열화 0.1dB 미만이나 HW 편의상 8bit(1byte)로 결정. 프로토타입 modulator/demodulator 실측과 시뮬 BER 차이 0.1dB 미만.
10. 한계: gate count/throughput 등 HW 수치 본문 미기재, 방송(QPSK/OFDM/AWGN) 전용 부호·채널, 향후 고차 QAM 검증 예정.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 연산 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 중복: Prime ECC도 min-sum 기반(min1/min2), offset/normalized factor는 이미 magnitude 양자화 테이블로 처리 |
| LLR 양자화 비트 결정 | `channel.cpp` `Set_LLR_Th()`, `ecc_data.h` `PARAM_LLR` | 낮음: 8bit 방송 LLR은 Prime ECC의 hard-1bit/2~3bit soft-read와 비용 전제가 다름 |
| SC-LDPC 부호 구성(HL/HU coupling) | `ecc_top.cpp` `Load_PCM()` | 낮음: Prime ECC는 고정 block QC-LDPC 로드만, SC coupling 파이프라인 없음 |
| TDMP layered 스케줄 | 대응 없음 | Prime ECC는 dual-update(even/odd) 스케줄로 TDMP layered 별도 구현 없음 |

적용 가치: **낮음** — offset min-sum offset·LLR 양자화 최적화 절차는 참고 가능하나 min-sum/LLR 테이블은 Prime ECC가 이미 보유하며, 대상 부호가 260kbit급 방송 SC-LDPC로 NAND block QC-LDPC와 정합되지 않는다.
