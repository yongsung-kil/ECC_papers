### B. 알고리즘 요약 (10줄 내외)

1. 시스템: FPGA(Alveo U50) QC-LDPC 인코더, systematic 코드, code length `1536`, rate `1/2, 2/3, 3/4, 5/6` 평가.
2. 문제: 기존 FPGA 인코더는 특정 코드·플랫폼 전용이라 유연성이 낮고, VMM 인코더도 직렬 처리 또는 cycle당 접근 bit-width 제한으로 병렬성을 못 살림.
3. 접근: dual-diagonal/RU 등 특정 코드 의존 알고리즘 대신 일반적 direct generator matrix 곱(VMM) 채택 — `c = sG`, systematic이라 dataword 복제 + parity VMM 2단계.
4. 핵심 기법: fine-grained **partially-parallel VMM** — G를 z×z submatrix로 분할, super-row간 병렬·super-row내 직렬, GF(2) AND 배열 + XOR tree.
5. 병렬 구조: r×c PE, 각 Cell이 submatrix 1개를 shift register에 저장, z cycle 동안 row shift하며 벡터-행렬 곱, 같은 super-column 결과를 XOR tree로 누적.
6. Encode PE 파이프라인: Vector Splitter → Cell array 병렬 연산 → Vector Aggregator, 3단은 데이터 의존으로 비파이프라인, cell 내부는 파이프라인.
7. 구현 디테일: Xilinx Vitis HLS 2020.2, C/C++ + PIPELINE/UNROLL/DATAFLOW pragma로 RTL 생성, codeword buffer 두 부분(dataword 직결 + parity) 병렬 채움.
8. 유연성: cell array 크기(z,r,c)를 코드에 맞춰 HLS로 신속 조정, 다른 FPGA로 이식 용이.
9. 결과: peak throughput `98.4Gbps`, baseline(CCSDS column-parallel) 대비 최대 `17.62×`, SOTA RTL 대비 `14.75×`, HW 소비 증가는 최대 `19.31%`뿐. rate 1/2는 768 연산이 HLS pipeline 한계 초과로 throughput 낮음.
10. 한계: 인코더 전용, 정정능력·복호와 무관, gate count/공정 등은 그림에만 있고 본문 수치 없음.
