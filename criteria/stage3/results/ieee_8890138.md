### B. 알고리즘 요약

1. 시스템: 16nm TLC(3-bit/cell) NAND flash에 저장된 JPEG 파일, 기존 JPEG 뷰어와 호환되는 애플리케이션 레벨 보호/복구, FPGA(Zedboard)+heater 실측.
2. 문제: flash long-term retention error(수년 후 BER~1%)로 uncorrectable 오류 발생, JPEG는 단일 비트오류가 이미지 전체를 손상시킴.
3. 관찰: (a) JPEG 헤더가 데이터부보다 critical, (b) skewed reliability(LSB=strong, CSB=weak page), (c) Huffman 가변길이/DPCM으로 인한 bit·DC error propagation.
4. 기법1 `Strong-Page Header Protection`: 헤더를 강한 페이지(LSB)에 배치(vendor command 필요).
5. 기법2 `Bit Error Propagation Prevention`: 각 블록 bitstream 길이를 COM marker에 저장해 오류 블록 경계 복원.
6. 기법3 `DC Error Propagation Mitigation`: APP1 thumbnail(1/8 크기)을 각 8x8 블록의 DC 참조값으로 사용해 DC 편차 큰 값을 thumbnail로 대체.
7. 기법4 `Huffman-Assisted Error Correction`: COM에 블록별 non-zero 성분 수 저장 후, 개수 불일치 시 한 비트씩 brute-force flip하며 재복호(Fig.5).
8. 실측 방법: JESD47H.01 온도가속(85C 7h≈25C 1년), 8KB 페이지=2KB codeword 4개 interleave, BER 5e-3 초과 시 uncorrectable 가정.
9. 결과: 손상 JPEG를 최대 23.5dB PSNR 향상 복구(예 35.8dB/SSIM 0.96), 저장 오버헤드 9.9%, 복구 11.9초/파일.
10. 한계: 순수 애플리케이션 레벨(ECC 미변경), HW ECC 설계 없음, LDPC 등 하위 ECC는 고정 전제로만 취급.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 애플리케이션 레벨 JPEG 보호/복구 | 대응 없음 | ECC 계층 밖(JPEG 마커·Huffman) 처리로 부호/복호와 무관 |
| TLC 페이지별 skewed BER / Vth retention | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | RBER/LLR 모델 파라미터 참고 정도, 직접 이식 아님 |
| LDPC ECC (고정 전제) | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | 논문은 ECC를 블랙박스로만 가정, 신규 기여 없음 |

적용 가치: **낮음** — LDPC ECC를 고정 블랙박스로 두고 그 위 JPEG 애플리케이션 계층만 개선하므로 부호설계/복호/HW 이식 요소가 전무하다. 페이지별 BER 특성은 참고 데이터로만 소용.
