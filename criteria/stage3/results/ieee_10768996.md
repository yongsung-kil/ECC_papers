### B. 알고리즘 요약

1. n-bit binary linear codeword(rate r=k/n, 재구성 가능 범위 n=32~256, rate 0.8~1), AWGN + BPSK 채널, LLR 기반 soft 정보 사용.
2. ML 복호는 code length에 NP-hard라 기존 표준은 code-specific 복호기(LDPC-BP, Turbo, CA-Polar 등)에 의존; 본 논문은 code-agnostic ORBGRAND를 HW로 구현해 하나의 칩으로 임의 binary linear code를 근사 ML 복호.
3. 채널모델은 연속 AWGN 노이즈; 노이즈 자체가 아닌 "noise effect sequence" `e^n`(비트 flip 여부)를 추정하는 것이 GRAND의 핵심 문제 정의.
4. ORBGRAND는 reliability `|LLR(Yi)|` 오름차순 정렬 후 logistic weight `LW = Σ i·ẑi` 순서로 noise-effect 후보를 생성, `yH^T=0`이면 코드북 멤버십 확인.
5. `LW`는 정렬된 비트 위치의 flip 조합에 가중치를 매겨 신뢰도 낮은 비트를 우선 flip하도록 유도하는 순서 지표(식 3).
6. 하드웨어 최적화 1: "minimum reliability sorter"가 전체 정렬 없이 필요한 만큼(LWmax=104)만 순차 정렬 후 codeword 발견 시 즉시 중단(dynamic abandonment)해 bitonic sorter 대비 9~72배 에너지 절감.
7. 하드웨어 최적화 2: landslide 알고리즘(정수 분할 생성)을 LUT 없이 온더플라이로 생성, 4개 병렬 branch(고정 offset 6/5/13 cycle 활성화)로 파이프라인.
8. 별도 학습/최적화 절차 없음(HW 아키텍처·파라미터는 설계 시 고정).
9. 40nm CMOS 측정 결과: 256-bit/rate 0.94 코드에서 0.76pJ/bit, 4.9mW, throughput 6.5Gb/s, latency 40ns @ FER 10^-7, core area 0.4mm2.
10. LDPC/QC-LDPC 부호구조나 min-sum 반복복호와 무관한 GRAND 패러다임이며, 짧은 코드(≤256bit) 및 CRC/CA-Polar로만 실측 검증되어 NAND용 긴 QC-LDPC로의 확장성은 본문에 없음(미상).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ORBGRAND 디코딩 알고리즘 전체(noise-effect guessing) | 대응 없음 | min-sum 기반 Tanner-graph 반복복호와 근본적으로 다른 패러다임(code-agnostic) |
| soft reliability 정렬(minimum reliability sorter) | `channel.cpp` `Set_LLR_Th()` | LLR 활용이라는 점만 유사, 정렬 기반 구조는 대응 없음 |
| 동적 clock-gating/sleep mode(저전력) | 대응 없음 | Prime ECC는 bit-exact 시뮬레이터로 저전력 clock-gating HW 최적화 자체가 스코프 밖 |
| H-matrix 멤버십 체크(error checker) | `decoder.cpp` `C2V_Cal()` (syndrome 계산 관점만) | 신드롬 계산 자체는 공통 개념이나 GRAND의 반복 없는 guess-and-check 구조와는 무관 |

> 적용 가치: 낮음 — 부호·복호 패러다임이 QC-LDPC min-sum과 근본적으로 달라(code-agnostic guessing) 직접 이식 대상이 아니며, sorter/파이프라인 저전력 설계 기법만 참고 가치가 있음.
