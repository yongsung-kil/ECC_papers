### B. 알고리즘 요약

1. 대상은 5G NR protograph LDPC(BG2, base 42x52, R=1/5~0.3, lifting Z=3~30, N=150~1500)로, min-sum(MS)을 학습으로 SP에 근접시키는 것이 목표.
2. 문제: 기존 neural BP/OMS 디코더는 파라미터 수가 codelength에 비례해 짧은 코드(<200bit)만 지원.
3. 핵심 아이디어(parameter-sharing): protograph는 base graph의 edge-type만큼만 서로 다른 파라미터가 필요 - 같은 edge-type에서 lifting된 `Z`개 edge는 동일 α,β 공유 (식 15a/b).
4. 학습형 CN 갱신: `c→v = (∏sgn) × ReLU(α^(i)·min|v→c| − β^(i))` (식 11) - min-sum에 iteration별 normalizing α와 offset β를 학습으로 부여한 neural NOMS.
5. `α=1, β=0`이면 표준 MS로 환원되며, α/β가 short-cycle로 인한 message correlation을 보정.
6. 4개 기본 타입(Type I~IV: edge별/iter별 파라미터 정도차) + damping factor `γ^(i)` 추가한 Type V/VI(식 17)로 확장.
7. 학습: cross-entropy loss, Adam(lr=0.001), AWGN에서 최대 25 iter, iteration당 50000 batch x 50 sample.
8. iteration-by-iteration greedy 학습: 마지막 layer만 학습·이전 고정 - vanishing gradient 회피, 조기종료와 정합, I 재사용(recycle) 가능. codelength/rate compatible 학습으로 여러 코드 일반화.
9. 결과: Type-I이 OMS/NMS 대비 0.2~0.5dB gain, 짧은 코드(Z=3)는 고SNR에서 SP도 능가, tanh 없어 NMS/OMS와 유사 복잡도. 이론툴 T-EXIT chart 제안.
10. 한계: HW 미구현·시뮬만, iteration별 파라미터 저장 메모리 추가, 파라미터 양자화·HW 구현 영향은 future work.
