### B. 알고리즘 요약

1. 짧은 블록길이 LDPC 코드(HRLLC용)에서 BP(belief propagation) 디코딩이 Tanner graph의 short cycle 때문에 준최적인 문제를 다루며, 대상 코드는 (63,6) simplex, (132,66) 5G QC-LDPC(Z=11), (273,191) projective-geometry LDPC.
2. 핵심 아이디어: ensemble decoding — M개의 독립적인 constituent BP 디코더를 병렬로 구동해 각각 codeword 후보 `ĉ_j`를 생성하고, syndrome이 0인 유효 후보 중 ML-in-the-list 기준(`ĉ = argmin_j L_ch^T·(-1)^ĉ_j`)으로 최종 출력을 선택.
3. 기법1 MBBP(Multiple-Bases BP): 서로 다른 저가중치 parity-check로 구성한 복수의 H_j 행렬로 병렬 디코딩(대칭 codeword 집합이 다름).
4. 기법2 AED(Automorphism Ensemble Decoding): 코드의 automorphism 군(순환 permutation `π^d_{Q,C,Z}(i)`)을 이용해 관측벡터 y를 permute한 뒤 복호, 역permute하여 후보 생성(`ĉ_j = π_j^{-1}(Dec(π_j(L_ch)))`); QC-LDPC는 BP가 QC permutation에 불변이라 그대로 적용 불가해 row 삭제/추가/합산으로 대칭성을 깨야 함.
5. 기법3 SED(Scheduling Ensemble Decoding): layered BP의 check-node 서브셋 처리 순서를 다르게 하여 다양성 확보(`N/N_L`개 스케줄 조합), 병렬도는 `N_L`로 조절되며 QC-LDPC 최대 `N_L=Z`.
6. 기법4 NED(Noise-Aided ED): 첫 디코더 외 나머지에 채널 LLR에 가우시안 노이즈 `L_j,ch = L_ch + n_j, n~N(0,σ²_NED)`를 더해 다양성 유도(σ²는 튜닝 필요).
7. 기법5 SBP(Saturated BP): `log2(M)`개의 가장 신뢰도 낮은 LLR 위치를 ±∞로 saturate하는 모든 조합을 병렬 분기로 입력해 다양성 확보.
8. 학습/최적화 절차: NED의 노이즈 분산 σ²_NED를 M=4/8 각각 최적화(0.64/0.25); SED 스케줄은 무작위 선택(비최적화).
9. 결과: 모든 ensemble이 단일 BP 대비 오류율 개선, 코드 구조 활용 기법(MBBP, AED)이 가장 큰 이득(특히 대칭성 큰 코드에서 AED가 near-ML 도달); ensemble recovery probability `ρ = P[ĉ=c | Dec1(L_ch)≠c]`로 이득 정량화, SNR 증가에 따라 ρ 증가.
10. 한계: M개 병렬 디코더 인스턴스로 latency/면적 비용 증가(특히 SED는 순차 스케줄로 latency 추가 악화), 짧은 코드(N≤273)만 검증되어 NAND용 고rate 긴 QC-LDPC 확장성 불명확, HW 구현 없이 시뮬레이션(Monte-Carlo BLER)만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ensemble decoding 전체 구조(M개 병렬 BP + ML-in-list 선택) | `decoder.cpp` `LDPC_Decoder()` | 반복 message-passing 루프 자체는 기존 min-sum과 동일 구조를 M개 복제해야 함 - 구조적으로 이식 가능하나 자원 M배 필요 |
| AED(automorphism, QC permutation 활용) | `ecc_top.cpp` `Load_PCM()` | Prime ECC의 QC-LDPC(base+circulant, z_sb=32) 구조와 직접 연관되나, BP가 QC permutation에 불변이라는 문제(논문에서도 지적)를 동일하게 겪어 대칭성 파괴용 H-matrix 변형이 필요 |
| SED(layered 스케줄 다양화) | `decoder.cpp` `LDPC_Decoder()` (Dual-Update 스케줄과 유사 개념) | Prime ECC의 Dual-Update(parity even/odd 교대)와 유사한 "스케줄 다양성" 개념이나, 앙상블 병렬 실행이 아닌 단일 스케줄만 사용 중 |
| 조기종료(syndrome 검사로 유효 codeword만 채택) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 각 constituent 디코더의 syndrome=0 검사는 기존 CRC 조기종료와 유사한 역할 |
| NED/SBP(LLR 왜곡 기반 다양성) | `channel.cpp` `Set_LLR_Th()` | LLR 조작이라는 점에서 소프트 채널 모듈과 연관되나 목적(양자화 threshold vs 인위적 다양성 주입)이 다름 |

> 적용 가치: 중간 — QC-LDPC 대칭성을 활용하는 AED/SED는 Prime ECC의 QC 구조와 개념적으로 맞닿아 있어 error-floor/waterfall 개선 아이디어로 참고 가치가 있으나, M배 병렬 디코더 자원 소모가 커 그대로 이식보다는 개념 차용 수준.
