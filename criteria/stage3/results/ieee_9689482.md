### B. 알고리즘 요약

1. 시스템: (1024, 512) 계통형(systematic) polar code, BPSK/AWGN, 최대 반복 `Imax=30`, 스케일링 `s=0.9`.
2. 문제: SC 계열 대비 병렬화가 쉬운 BP polar 디코더는 많은 반복이 필요해 노드당 산술연산(덧셈/비교/곱셈) 비용이 큼.
3. 대상: factor graph 상의 BP 계산단위(CU)는 상하좌우 4 터미널(a,b,c,d)을 가지며 각 출력은 min-sum 근사식 `f(x,y)≈0.9·sign(x)·sign(y)·min(|x|,|y|)`로 계산.
4. 핵심기법: 각 CU의 4개 출력식(Rc, Rd, La, Lb)에 대해 입력 LLR이 특수값(0, ±Umax)인 경우를 exhaustive search로 열거해 연산이 생략/단순화되는 등가식들을 도출.
5. 의미: 특정 입력조합에서는 출력이 0이거나 sign/덧셈만으로 결정되어 min·곱셈 등 연산을 건너뜀.
6. 확장: 상·하 노드 식이 쌍(pair)을 이루는 패턴을 발견해 유효 조건 수를 제한, 구현 복잡도를 더 낮춤 (Table II/III의 valid pair).
7. 구현 디테일: 조건을 논문 제시 순서대로 검사, 조건 성립 시 해당 단순식 실행 후 종료(우선순위 분기).
8. 최적화 절차: 별도 학습 없음, 조합 분석 기반 규칙 도출.
9. 결과: Monte Carlo 시뮬(10000 벡터)에서 BER/FER는 교과서 BP와 동일, +0dB에서 약 23.6%, +4dB에서 약 27.2% 연산 절감. Cortex-R4 예비 추정 사이클 순이득 ~13%.
10. 한계: HW 미설계, polar code 전용, 오정정 성능 개선 없음(연산량만 절감), 시뮬 수준.
