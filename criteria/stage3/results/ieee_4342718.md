### B. 알고리즘 요약

1. IEEE 802.16e 표준의 19가지 모드(코드워드 576~2304 bit) QC-LDPC 부호를 지원하는 멀티모드 디코더 설계.
2. 기존 방식은 BNU(bit node unit)와 CNU(check node unit) 연산이 순차적이라 하드웨어 활용률이 50%에 그치는 문제.
3. base matrix를 재정렬(reorder)하여 `1`의 위치를 대각선 인접으로 모아 BNU/CNU 연산을 중첩 처리 가능하게 함(Park & Kang 스케줄링 알고리즘 기반).
4. Min-Sum Algorithm(MSA) 기반 디코더에서 오버랩 연산으로 idle time을 66% 감소, 하드웨어 활용률을 50%→75%로 향상, 디코딩 지연을 31.25% 감소.
5. 조기종료(Early Termination) 기법: 연속 두 iteration의 hard-decision 결과가 같으면 즉시 종료, 최대 iteration 도달 시에도 종료.
6. 분산 메모리 뱅크: 중앙 메모리 대신 24개 single-port + 76개 two-port register file 100개 뱅크로 데이터 접근 병렬성 확보, 저전력화.
7. 재구성 가능(reconfigurable) 아키텍처: Counter Unit 신호 기반 Address Generation Unit(AGU)이 저장소자·연산유닛 수정 없이 19개 모드의 파리티 체크 행렬 관계를 활용해 주소 생성.
8. 체커보드(checkerboard) 레이아웃으로 100개 메모리 뱅크를 10×10 2-D 배열 배치, 라우팅 복잡도와 칩 면적 절감.
9. 결과: TSMC 0.13um 공정, 코어 4.45mm², 다이 8.29mm², 420K 게이트, 83.3MHz@52mW(1.2V), 0.85V에서 41.67MHz@16mW; SNR 4.0dB에서 (2304,1152) 부호 디코딩 지연 45% 감소(최대 iteration 8일 때).
10. 한계: 성능은 표준 MSA와 "거의 동일" 수준 유지가 목표이며 정정능력 자체의 개선은 없음. Fig 1 회로 블록도가 텍스트 손상으로 상세 구조 파악 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| BNU/CNU 오버랩 스케줄링 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 이터레이션 루프 내 파이프라인 스케줄과 유사한 문제이나 우리는 이미 6-stage pipeline 보유 |
| 조기종료(연속 2 iteration 동일 hard-decision) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 우리는 CRC 기반 조기종료를 이미 보유, 본 논문은 hard-decision 비교 방식으로 방식이 다름 |
| 분산 메모리 뱅크 / 체커보드 레이아웃 | 디코더 HW 모델 (z=32, HCU, GT) | 메모리 라우팅 최적화 아이디어이나 우리 z=32 구조와 직접 대응 어려움 |
| 재구성 가능 멀티모드 AGU | `ecc_top.cpp` `Load_PCM()` | 다중 코드길이 지원용 주소생성 로직, 우리는 단일 H-matrix 고정이라 대응 없음 |
| Min-Sum 디코딩 알고리즘 | `decoder.cpp` `CNU_Update_New_Mag()` | 표준 MSA로 이미 보유 기법과 동일, 신규성 없음 |

> 적용 가치: 낮음 | 이유: 정정성능 개선이 아닌 멀티모드 지원용 HW 스케줄링/레이아웃 최적화이며, 우리는 이미 조기종료·pipeline을 보유하고 단일 고rate QC-LDPC 고정 구조라 멀티모드 재구성 아키텍처 자체의 이식 필요성이 낮음.
