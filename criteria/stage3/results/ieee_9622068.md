### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 홍채 인식 SoC-FPGA(Xilinx ZCU104), IEEE 802.16 QC-LDPC(rate `2/3A`, 예: `(1920,1280)`, `(960,640)` 등), layered min-sum(`Imax=50`), ARM(HPS)+PL(FPGA) AXI 연결.
2. 문제: 홍채 recognition의 참조/probe iris 간 변이·노이즈로 인식 정확도 저하 — 임베디드 저복잡도·저전력 요구.
3. 모델: enrolment에서 iris code `X0` 인코딩 후 parity `P0` 저장, verification에서 다중 샘플 평균 `X3`에 `P0` 결합해 QC-LDPC 복호로 오류정정 후 Hamming distance 매칭.
4. 핵심 기법: FPGA Dynamic Partial Reconfiguration(DPR)로 단일 QC-LDPC 디코더들을 시분할·재구성해 다중 모드 디코더처럼 동작(나머지 회로 무중단).
5. 소프트판정: iris code를 양자화 fix-point(2/4/8-bit)로 유지해 soft-decision 복호(hard 대비 1~2dB 코딩 이득), 복호 전 unsigned→signed(-128) 변환.
6. 디코더 아키텍처: 참조 [16]의 layered QC-LDPC 구조, CN/VN processor 수는 802.16 H-matrix에 맞춰 조정, 병렬/직렬 (960,640) 두 설계 비교.
7. 구현 디테일: 3 모드 정의 — fast((960,640) 병렬 4-bit), low-power((1920,1680) 4-bit, 13mW), accuracy((1920,1680) 8-bit); 3 디코더를 1 partition에 배치.
8. 학습/최적화: 없음(코드 선택은 iris code 크기 1280 bit 정합 기준 실험 선정).
9. 결과: (1920,1680)가 TAR/FAR 최고, code length 증가는 정확도 손실; Q4가 전력 절감 옵션; DPR bitstream 375KB, 회로 전환 1.75ms; (1920,1280) Q4 latency 726839ns, 13mW.
10. 한계: 부호/디코더 신규 알고리즘 없음(기존 802.16+layered min-sum 재사용), throughput/gate공정 미제시, 응용은 biometrics 특화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered min-sum QC-LDPC 복호 | decoder.cpp LDPC_Decoder(), CNU_Update_New_Mag() | min-sum·layered는 Prime ECC 이미 보유(중복) |
| 양자화 fix-point(2/4/8-bit) soft 입력 | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | 양자화 LLR은 이미 보유(2SD/3SD), 접근은 유사 |
| FPGA DPR 다중 모드 재구성 | 대응 없음 | Prime ECC는 고정 QC-LDPC bit-exact 시뮬, 런타임 회로 재구성 개념 없음 |
| iris recognition 통합/매칭 | 대응 없음 | NAND ECC와 무관한 응용 계층 |
```

대상 decoder이나 부호/복호 성능 개선 요소가 없어 C섹션 성능은 대부분 X/N/A. **적용 가치: 낮음** — 디코더·부호 자체는 기존 기법 재사용이고, 신규 기여인 DPR 다중 모드 재구성은 Prime ECC의 고정 시뮬 구조에 대응 모듈이 없다.
