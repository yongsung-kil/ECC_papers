### B. 알고리즘 요약

1. 대상: LDPC 부호 구성, strict-sense(모든 check=XOR)와 wide-sense(더 복잡한 subcode) LDPC를 구분, 목표는 WS-LDPC에도 irregularity 이득 적용.
2. 문제: irregular LDPC가 regular 대비 threshold가 크게 우수([1]), 이를 wide-sense(subcode 사용) 부호로 확장하고자 함.
3. 모델: 기저 그래프를 최대한 유지하며 factor graph 구성, 큰 girth를 위해 Ramanujan 그래프 `RG_LPS(q,p)`(서로 다른 홀수 소수 `q,p`) 사용.
4. 핵심기법 1단: regular bipartite 그래프의 left/right vertex와 edge를 subcode를 담은 factor-graph 조각으로 치환(12-regular 예시), check node를 단순 XOR 대신 subcode로 대체.
5. 식/설계 의미: high-degree bit node와 여러 subcode의 비율을 최적화하여 성능 이득 확보, small-weight codeword 회피가 배치 기준.
6. 핵심기법 2단: Code 3에서 left 50% type-1 `[12,8,3]`+50% type-2 degree-12, right는 `[12,8,3]/[12,9,2]/[12,10,2]/[12,11,2]`(단순 XOR) 비율 혼합.
7. 구현 디테일: subcode 배치는 small-weight codeword 회피 기준, Cayley 그래프면 generator에 따라 subcode 배치 가능, threshold 최적화 부호는 인코딩 복잡도도 낮은 경향.
8. 최적화: density evolution/threshold 계산으로 비율 튜닝.
9. 결과: BER 시뮬(binary-input AWGN, sum-product) rate-1/3 3종 비교 — Code1 (4,6)-regular SS(threshold 1.67 dB), Code2/3 WS 기반 `RG_LPS(13,11)`, Code 3은 Code1/2보다 length 46% 단축.
10. 한계: HW 미설계, 저rate(1/3) 시뮬뿐, full sum-product 복호 가정, wide-sense(generalized) 구조로 NAND 고rate binary QC-LDPC와 상이.
