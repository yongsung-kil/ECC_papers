### B. 알고리즘 요약

1. 대상 부호: 1024-b, rate-1/2, irregular LDPC (avg col-weight 3.25, row-weight 6.5, 3328 edges); 3G 무선 turbo와 동일 블록/rate로 비교 목적.
2. 문제: turbo/turbo-product 디코더는 block-serial 이라 iteration당 대량 state fetch/store → throughput·전력 손해.
3. 핵심 아이디어: LDPC의 message-passing은 block-parallel 가능 → 그래프를 HW에 직접 인스턴스화(VN·CN 각 1회 실체화, edge=물리 배선).
4. CN 연산: 로그 도메인에서 parity(sign XOR) + reliability update((5),(6)식)로 곱셈·나눗셈 제거, log-MAP 유사.
5. 근사: 입력 log는 combinatorial 10-gate, 출력 exp는 leading-zero count 11-gate로 근사 (3-b magnitude라 소면적).
6. VN 연산: 수신 LLR + 모든 check message 합산, 각 edge는 (group sum − 자기 edge)로 outgoing 형성, sign이 판정.
7. 양자화: 메시지 4-b(sign 1 + magnitude 3), 64 iteration에서 float 1000-iter 대비 손실 0.2dB.
8. 데이터패스: register를 VN에 배치 → CN은 순수 combinatorial, 3-block 파이프라인(load/decode/readout), shift register로 I/O.
9. 결과: 64MHz에서 1Gb/s 실측, 690mW@1.5V(SNR 3dB), 52.5mm2, 26624 top-level message net(평균 3mm), turbo 대비 throughput 10배+.
10. 한계: 완전병렬이라 area 큼·코드/rate/length 고정, 부호설계/error-floor 개선 없음, soft 4-b AWGN 전제.
