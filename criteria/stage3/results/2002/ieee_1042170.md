### B. 알고리즘 요약

1. 시스템: BPSK codeword를 이진입력 PR(ISI)+AWGN 채널로 전송, BCJR 등화기 + LDPC MP 디코더 결합(turbo 등화).
2. 문제: PR 채널 LDPC의 threshold를 DE로 계산하려면 BCJR 부분에 대량 Monte Carlo가 필요해 비용이 큼.
3. 모델: BCJR 등화기 출력 LLR을 BPSK-over-AWGN처럼 등가 SNR `s`의 가우시안으로 근사(식 (3)).
4. 핵심기법: Chung et al.의 Gaussian approximation을 ISI 채널로 확장, 메시지 pdf를 평균/등가 SNR 하나의 파라미터 `s_l` 갱신식으로 압축(식 (10)).
5. 핵심식: 채널부분 `F_ch`(SNR 변화)를 Monte Carlo로 평가하고 log-domain 스플라인 보간으로 시뮬 점 최소화, MP부분은 [3]의 갱신식 사용.
6. 확장: dicode·EPR4 채널의 regular/irregular 부호 threshold 산출, GA가 DE threshold에 근접(약간 높음).
7. 스케줄 트릭: 스케줄을 B(BCJR)/L(LDPC) 문자열로 표기, "BCJR once"(최저복잡도, ~1 dB 손실)와 "Resetting LDPC"(BCJR 시 CN→BN 메시지 0으로 리셋해 메모리 재사용) 두 가지 suboptimal 스케줄 비교.
8. 학습/최적화: DE/GA 기반 threshold를 스케줄 설계 지표로 사용, 최적 대비 손실 정량화.
9. 결과: GA가 DE 대비 0.1 dB 이내 정확·계산량 대폭 절감, resetting 스케줄이 optimal과 BCJR-once의 좋은 절충; (3,27)-regular 4K/16K/64K BER 곡선이 blocklength↑에 threshold로 수렴.
10. 한계: HW 미설계·이론(threshold/스케줄 분석)뿐, BCJR 등화기 결합 PR 채널 전제로 NAND엔 부합하지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|

| threshold 계산(GA/DE) | 대응 없음 | Prime ECC는 부호 고정, DE/threshold 설계도구 아님 |
| MP 디코더 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` 이터레이션 루프 | 스케줄 개념은 닿으나 BCJR-LDPC 교대 스케줄이라 min-sum 단독 루프와 상이 |
| Resetting LDPC 메모리 재사용 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) (C2V 메모리) | 메시지 리셋으로 메모리 재사용 아이디어만 개념적 참고 가능 |

적용 가치: **낮음** — threshold/스케줄 분석이 BCJR 등화기 결합 PR 채널 turbo 등화에 강결합되어, 등화기 없는 NAND binary QC-LDPC ECC의 디코더·부호설계·HW에 떼어 쓸 요소가 사실상 없음(메모리 재사용 개념만 미약한 참고).
