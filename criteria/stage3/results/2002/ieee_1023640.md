### B. 알고리즘 요약

1. 대상 부호: MDS array code 3종 — BR code, modified EVENODD code, X-Code. 모두 structured binary linear code.
2. 문제: random irregular LDPC는 capacity에 근접하나 encoding·메모리 부담 큼; algebraic 구조로 encoding/거리특성/메모리를 개선하려 함.
3. 관점: 이 array code들의 parity-check matrix를 column weight 주로 2인 **LDPC로 간주**하고 belief propagation으로 복호.
4. 핵심 성질: X-Code와 modified EVENODD의 그래프에는 4-cycle이 없음 → BP에 유리; BR code는 4-cycle 있으나 성능 저하 크지 않음.
5. modified EVENODD: PCM의 high-weight column을 제거하도록 EVENODD를 수정.
6. 채널: binary-input AWGN, BER vs SNR(`Es/N0`) 곡선으로 평가.
7. 부호 rate: BR=`1-2/(p+2)`, X-Code=`1-2/p` (p=37 소수), 공정비교 위해 shortening으로 rate 동일화(rate≈0.946).
8. 추가 이점: soft BP 복호 후 array code의 대칭(large-symbol error/erasure) 정정 알고리즘을 outer decoder로 사용 가능.
9. 결과: 세 array code 성능 유사, 유사 파라미터 regular LDPC(Gallager 구성)보다 약간 우수(Fig.1).
10. 한계: 2쪽 ISIT 요약이라 정량 수치·HW·양자화·error-floor 분석 없음, full version은 외부 링크.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| array code PCM을 LDPC로 로드 | `ecc_top.cpp` `Load_PCM()` | 부호구조가 QC circulant이 아니라 매핑 불가에 가까움 |
| BP(message-passing) 복호 | `decoder.cpp` `LDPC_Decoder()` | 알고리즘은 표준 BP, 우리는 이미 min-sum 보유(중복) |
| 부호설계(4-cycle 제거) | 대응 없음 | array code 특유 대각/EVENODD 구조는 QC base+shift와 상이 |

적용 가치: **낮음** — 부호 계열(MDS array code)이 NAND binary QC-LDPC와 달라 부호설계·복호 모두 그대로 떼어 쓸 수 없고, BP 복호도 이미 보유 기법과 중복.
