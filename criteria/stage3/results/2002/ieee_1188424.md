### B. 알고리즘 요약

1. 시스템: BPSK 코드워드 `c`를 `z=(-1)^c`로 매핑, AWGN 채널(분산 `σ²`) 전송, LLR로 초기화. regular (3,6)/(4,8) 앙상블, rate 1/2, 시뮬은 (8000,4000) 코드.
2. 문제: BP(sum-product)를 min 근사한 BP-based(=min-sum) 알고리즘은 구현이 간단하나 성능이 약 0.5dB 저하됨.
3. 개선1 normalized BP-based: check node 출력 `L_mn`을 상수 `α>1`로 나눠 과대추정된 extrinsic message를 보정. `α`는 모든 iteration·SNR에 대해 하나로 고정.
4. 개선2 offset BP-based: `L_mn ← sgn(L_mn)·max(|L_mn|-β, 0)` — reliability를 양의 상수 `β`만큼 감산하고 β 미만은 0으로 clip.
5. 핵심 식 의미: min-sum이 magnitude를 항상 과대추정(`|L2|>|L1|`)하므로 α로 나누거나 β로 빼서 BP magnitude에 정렬 — extrinsic 정확도 개선.
6. 파라미터 결정: density evolution으로 각 앙상블의 최적 `α`, `β` threshold를 수치 계산(min WER 목표, 최대 iteration 100).
7. 양자화: 메시지를 `q`비트(부호 1 + magnitude), step `Δ`로 표현. offset은 정수 β로 discretize 용이, normalized는 α가 shift로 구현가능한 특수값(2,4,8)만 discretize 용이.
8. 결과: (8000,4000) (3,6) 코드에서 BP 대비 min-sum gap 0.5dB를 α=1.25 / β=0.15로 약 0.05dB까지 축소.
9. 양자화 결과: 6비트 uniform 양자화 offset BP-based가 BP 대비 0.1dB 미만 저하.
10. 한계: HW 미설계, 시뮬레이션·이론만; 최적 α/β가 코드 앙상블·rate 종속이라 재튜닝 필요.
