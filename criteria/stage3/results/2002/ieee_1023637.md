### B. 알고리즘 요약

1. 데이터 저장 응용을 겨냥해, 보장 최소거리(정정능력)를 갖는 **cyclic Euclidean Geometry(EG) LDPC**의 hard-decision 복호 성능을 분석한다.
2. 대상 부호: `n = 2^{2s}-1`, `k = 2^{2s}-3^s`, `d_min ≥ 2^s+1`, 각 비트가 `2^s`개 parity check에 연결된 1-step majority-logic 복호 가능 부호.
3. Threshold decoder: 비트 `i`에 연결된 check 중 위반 수가 threshold `T`를 넘으면 flip. `N_0`(정수신 시 위반 check 수)·`N_1`(오수신 시) 분포로 오복호 확률 `p_1` (식 (1)) 정의.
4. BSC에서 첫 iteration의 `T`를 `p_1` 최소화하도록 최적화한 optimal threshold `T*`를 유도(`T=2^{s-1}`일 때 모든 `2^{s-1}` 가중치 오류 정정).
5. 2-iteration threshold decoder: 1차·2차 threshold를 각각 `T`, `2^{s-1}`로 설정. 1차 후 총 오류 수를 `X`로 두고 2차 블록오류 확률 `P_2 = Prob{X > 2^{s-1}}` (식 (2)).
6. 비트 간 복호 의존성 때문에 `P_2` 직접 계산은 복잡하나, 기대값 선형성으로 `E[X]=n·p_1`을 정확히 구해 정정 가능 오류 수를 heuristic하게 추정.
7. 사례(`s=9`, `t=500` 오류): `E[X]=83.5`, 시뮬로 표준편차 `σ=15.1`, Gaussian 근사로 정정 불가 분율 `Q≈1.6×10^{-31}` → 500비트 미만 거의 모든 오류패턴 정정 주장.
8. 실제로 threshold decoder는 `2^{s-1}`보다 70%가량 큰 가중치 오류패턴까지 거의 모두 정정.
9. 결과: `(262143, 242461, t≥256)` EG 부호((512,512)-regular LDPC)가 `t=580`비트 오류를 `1-1×10^{...}` 확률로 정정, threshold 최적화로 1-iteration 대비 정정능력이 크게 향상.
10. 한계: HW 미설계, majority-logic/threshold 복호와 cyclic EG 부호 전용 해석·시뮬 근거로, soft-decision·일반 QC-LDPC로의 직접 확장은 다루지 않음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| threshold/majority-logic 복호 루프 | `decoder.cpp` `LDPC_Decoder()` | 이터레이션 루프 자리이나 min-sum(`CNU_Update_New_Mag()`)과 복호 계열이 달라 직접 대체 불가 |
| EG 부호 H-matrix 구조 | `ecc_top.cpp` `Load_PCM()` | cyclic EG 부호는 우리 QC-LDPC와 구성 상이, 교체 시 전면 재검증 |
| hard-decision(1bit) 입력 처리 | `channel.cpp` `Set_R_Offset()` (HD 경로) | HD 지원은 겹치나 threshold 복호 특유 로직은 대응 없음 |

적용 가치: **낮음** — 보장 정정능력·error-floor 관점은 흥미로우나, cyclic EG 부호 + majority-logic/threshold 복호는 Prime ECC의 고rate QC-LDPC + min-sum 파이프라인과 계열이 달라 이식 부담이 크고 HW·실측 근거가 없음.
