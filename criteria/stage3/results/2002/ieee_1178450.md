### B. 알고리즘 요약

1. 시스템: LDPC(sparse parity check) 부호를 intersymbol interference(ISI)가 있는 memory 채널(HDD, 전화선, 방송 반사 등)로 전송.
2. 문제: Gallager 복호는 memoryless 채널 가정 → ISI 채널에서 부정확. "separation"(MLSE/DFE로 ISI 제거 후 복호)보다 통합 복호가 우수.
3. 모델: finite-memory ISI 채널, 인접 심볼 간 상관을 명시적으로 모델링.
4. 핵심기법1(basic): Gallager 복호에 BCJR를 1회 사용해 인접 digit 상관으로 prior 확률 추정 개선, iterative 부분은 그대로 유지.
5. 핵심식: BCJR로 얻은 인접 심볼 상관 → variable node의 prior LLR을 갱신(memoryless 가정 완화).
6. 핵심기법2(generalized): 인접 digit이 참여하는 parity check 정보까지 활용해 rightbound 메시지를 수정, 주어진 인접 digit 수 기준 BER 관점에서 asymptotically optimal.
7. 트릭: 병렬 독립 소형 디코더 구조 유지(복호 지연 최소화 가능)라고 언급.
8. 최적화: 없음(closed-form 유도 + DE).
9. 결과: 두 알고리즘 모두 separation 방식 대비 BER 2~3자릿수 개선, generalized > basic; DE 이론 threshold와 시뮬 일치.
10. 한계: HW 미설계, 시뮬만, ISI 채널 전용(memoryless NAND read와 채널 모델 불일치), generalized 알고리즘은 DE 시 all-zero codeword 가정 불가(arbitrary codeword 필요).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager iterative 복호 루프 | `decoder.cpp` `LDPC_Decoder()` | 개념 대응하나 본문은 full Sum-Product/BCJR, Prime ECC는 min-sum |
| BCJR 기반 prior LLR 갱신(ISI 제거) | 대응 없음 | Prime ECC는 memoryless read 채널, ISI equalizer/BCJR 모듈 없음 |
| 채널 상관 → soft prior 주입 | `channel.cpp` `Set_LLR_Th()` | soft-read LLR 설정과 형식만 유사, ISI 상관은 무관 |

적용 가치: **낮음** — ISI(메모리) 채널용 BCJR 결합 복호라 memoryless read를 쓰는 NAND ECC와 채널 모델이 달라 이식 이득 없음.
