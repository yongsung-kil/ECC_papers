### B. 알고리즘 요약

1. 시스템: PCPC(Parallelly Concatenated Parity Check) LDPC, rate 8/9, k=512, n=576, (6,2) 고정소수점, AWGN 채널.
2. 문제: 일반 LDPC는 H는 sparse지만 G가 dense라 encoder가 복잡; PCPC는 G·H 모두 sparse해 encoder/decoder 모두 단순화.
3. 부호구조: 16x512 블록 `P1`(각 행에 32개 연속 1, 행마다 32열 시프트)을 기본블록으로, `H`=[I_64 | P1,P2,P3,P4](P2~P4는 P1의 독립 열순열), `G`=[P1..P4 | I_512]. 열가중치 4 또는 1, 행가중치 33, edge 2112개.
4. 인코더: G를 그대로 HW 매핑 — 순열(라우팅) + Parity-Check(XOR) + P/S 변환 블록.
5. 디코더: full-parallel SPA. Cell-A(bit→check, weight 1/4용 A1/A2)와 Cell-B(check→bit), B2C/C2B 인터리버(순열 라우팅), 2단 pipeline register.
6. Cell-A2: 4개 입력+LLR prior 합산 후 MSB로 hard decision, 내부 (8,2)→(6,2) saturation.
7. Cell-B: saturating adder array + LUT-B1(비선형 `f(z)=log((e^z+1)/(e^z-1))` 조합논리) + Sign-Eval(MSB XOR) + LUT-B2, 내부 (12,2)→(6,2) saturation.
8. 메모리: pipeline register 당 6x2112=12672 flip-flop, 인터리버가 순열 라우팅 담당.
9. 결과: encoder 5952 NAND2/delay 10·T_NAND, decoder 약 2.4M NAND2(92%가 LUT 포함 Cell-B)/delay 158·T_NAND, max 20 iter, 6bit 양자화 손실 미미(부동소수점 근접).
10. 한계: 시뮬(BER)만·합성공정/throughput 미기재, full-parallel LUT-SPA라 면적 과대, 특정 PCPC 부호 전용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SPA check→bit LUT(f(z) tanh형) 연산 | decoder.cpp CNU_Update_New_Mag() | Prime은 min-sum 근사라 LUT full-SPA와 다름(Prime이 이미 HW친화) |
| bit→check + hard decision(Cell-A) | decoder.cpp VN_Cal_HD() | 기능 대응하나 완전병렬 vs z=32 semi-parallel 구조 상이 |
| (6,2) 고정소수점 양자화 | ecc_data.h PARAM_LLR, decoder.cpp Get_VNU_Table_Idx() | 양자화 개념 대응, 비트폭·테이블 방식 다름 |
| B2C/C2B 인터리버(순열 라우팅) | GT.cpp Make_GT_HW() | 라우팅/edge 매핑 개념 대응, 구조 상이 |
| PCPC sparse-G 인코딩 | encoder.cpp LDPC_encoder_4KB() | Prime은 dual-diagonal 고정, PCPC 구조 미채택 |
| H/G 부호구조 로드 | ecc_top.cpp Load_PCM() | Prime은 QC-LDPC 고정, PCPC 부호 아님 |
```

적용 가치: **낮음** — full-parallel LUT 기반 SPA와 PCPC 부호는 Prime ECC의 min-sum·z=32 semi-parallel·QC-LDPC 구조와 정합되지 않고, 이미 보유한 기법 대비 새 이득이 없다.
