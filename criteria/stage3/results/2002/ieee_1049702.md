### B. 알고리즘 요약

1. 대상은 regular/irregular binary LDPC. LDPC를 turbo code처럼 `c`개 super-code(`C^1..C^c`)의 concatenation으로 재해석(각 super-code = `n/r`개 SPC sub-code의 direct sum).
2. 기존 TPMP(two-phase message-passing)는 모든 check↔bit 메시지를 저장해야 하고 랜덤 구조로 인터커넥트가 복잡, 직렬 구현 시 latency·전력 병목.
3. 제안 TDMP(turbo-decoding message-passing): 각 super-code를 SISO 디코더로 순차 복호, 앞 super-code가 방금 만든 extrinsic을 다음 super-code에 즉시 a priori로 전달.
4. 한 iteration = `c`개 sub-iteration. 새 메시지가 이전 iteration의 낡은 값이 아닌 갱신값으로 계산되어 수렴이 빨라짐(TPMP 대비 20~50% 적은 iteration).
5. SISO 서브디코더는 SPC sub-code의 2-state syndrome trellis에 Gallager MP 또는 BCJR 알고리즘 적용(BCJR가 양자화 노이즈에 강함, GLD 확장 시 필수).
6. 메모리: TPMP는 모든 중간메시지 저장, TDMP는 super-code당 partial bit message만 저장 → `(c-1)` 분량만 필요, 절감 `(75+25n/node-degrees)% > 75%`.
7. HW 아키텍처: bit message를 `n`개 circular buffer(각 `cj-1` stage)에 저장, `L`개 SISO 디코더가 배열의 한 row를 병렬 처리(`L`은 throughput vs 인터커넥트 tradeoff VLSI 파라미터).
8. 부호설계: parity-check matrix를 permutation matrix들로 분해하면 인터커넥트·제어로직 단순화 → throughput↑. Ramanujan graph(LPS 구성, 큰 girth/expander)로 이 구조를 갖는 코드 생성.
9. Ramanujan graph→factor graph 변환(node splitting/edge splitting/edge merging 등)으로 다양한 rate/length의 regular/irregular/GLD 코드 구성.
10. 결과: CODE1~4(n=1440~15840, rate 0.5~0.75) AWGN/BPSK 시뮬. TDMP가 TPMP 대비 최대 50% iteration↓, BER 최대 2 orders↓(moderate-high SNR·소 iteration). HW 미합성, throughput은 `nf/(aN·I·rtot)`식만 제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| TDMP 순차 super-code 스케줄(layered decoding 원형) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | 스케줄 개념은 이식 검토 가능하나 SISO/SPC 기반이라 min-sum 루프와 상이 |
| SISO/BCJR CN-VN 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal()` | 대응 없음(우리는 min-sum, 논문은 BCJR/SPC trellis) |
| circular buffer 메모리 절감 아키텍처 | 디코더 HW 모델(z=32, HCU, GT) | 메모리 라우팅 개선 아이디어 참고 가능, 구조 정합 필요 |
| permutation-matrix 구조 부호설계 | `ecc_top.cpp` `Load_PCM()` + H-matrix | QC-LDPC circulant이 이미 permutation 구조 — 개념 중복 |
| Ramanujan graph 부호 구성 | `ecc_top.cpp` `Load_PCM()` | 대응 없음(우리는 QC circulant 고정) |

적용 가치: **중간** — layered/turbo 스케줄에 의한 수렴·메모리 개선 아이디어는 min-sum 디코더에도 통용되는 고전적 기여지만, 이 논문의 SISO/BCJR/SPC trellis 구현은 Prime ECC의 min-sum 파이프라인과 직접 호환되지 않아 개념 참조 수준.
