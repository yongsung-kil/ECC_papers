### B. 알고리즘 요약

1. 대상: `m`차원 Euclidean geometry `EG(m,p^s)` (점 `n=p^{ms}`개, 각 line은 `p^s`개 점) 기반 두 부류의 LDPC 부호.
2. 구조: line들은 `K=(p^{ms}-1)/(p^s-1)`개 parallel bundle로 분할되며, 각 bundle은 기하의 모든 점을 정확히 한 번씩 포함.
3. 핵심 아이디어: line `L`의 incidence vector(점 포함 여부 0/1)를 PCM의 행으로 사용 → 대수적으로 sparse H 생성.
4. 구성: bundle `P_1..P_K` 각각으로 `k×n` 행렬 `H_p(j)`(열 weight 1) 형성, `y`개 bundle을 쌓아 `H_GA` 구성.
5. 성질: 두 line은 최대 한 점만 공유 → PCM의 어떤 두 행/열도 공통 "1"이 하나 이하 → Tanner 그래프에 length-4 cycle 없음(girth≥6).
6. 부호: `H_GA^T`의 null space가 length `n`, 최소거리 `≥ y+1`인 EG-Gallager code `C_GA`.
7. 보완부호: `H_GA`의 null space는 length `y·p^{(m-1)s}`인 complementary EG-Gallager code, 최소거리 `≥ p^s+1`.
8. 예1: `EG(3,2^3)`(512점, 73 bundle×64 line), `y=6` → rate-1/2 `(512,256)` code, 최소거리 `≥7`; SPA(sum-product) 반복복호로 동일 길이 컴퓨터생성 LDPC·64-state convolutional code 대비 우수.
9. 예2: `EG(2,41)` 기반 `(1681,1520)`은 BER `10^{-6}`에서 Shannon 1.3dB, `(1681,1440)`은 1.7dB; complementary 예 `(3840,3467)` rate 0.9028은 Shannon 1.1dB(BER 1e-5).
10. 한계: HW 미설계, ISIT 확장초록 수준으로 gate/throughput 없음; BER 곡선은 그림에만 존재.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EG 대수 구성 H-matrix(girth≥6) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 EG 구성으로 교체 가능하나 Prime ECC는 QC circulant 고정 |
| SPA(sum-product) 반복복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 min-sum(§3 보유), SPA의 tanh는 미채용 → 알고리즘 상이 |
| 최소거리/girth 보장(waterfall) | 대응 없음 | 정정능력은 부호구조에서 오며 별도 post-processing 모듈 없음 |

적용 가치: **중간** — girth≥6·최소거리 하한을 보장하는 대수 구성은 고rate 고신뢰 부호설계 관점에서 참고 가치가 있으나, EG-LDPC는 QC circulant 구조가 아니어서 Prime ECC(고정 QC-LDPC)에 그대로 넣으려면 QC화 재배열과 재검증이 필요.
