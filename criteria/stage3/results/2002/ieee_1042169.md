### B. 알고리즘 요약

1. 시스템: 자기기록 PR 채널(Lorentzian+MR head), rate 16/17 MTR + systematic LDPC, 정보비트 4096, 내부 MaxLogMAP APP + 외부 sum-product LDPC.
2. 문제: LDPC의 5 dB 이득이 RS-ECC 결합 시 rate loss로 1 dB로 감소, 그러나 반복복호는 TA/MD로 인한 erasure를 정정 못함.
3. 부호: 랜덤 구성 regular 패리티검사행렬, column weight 3, 4-cycle 없음; run-length 조건 위해 guard bit 삽입.
4. 핵심기법: ECC-less LDPC + erasure 보상 — 임계값 기반 TA 검출기 + run-length 위반 기반 MD(defect) 검출기 두 개로 erasure flag 생성.
5. 핵심식: (4)식으로 erasure flag가 ON인 비트의 채널정보를 masking(unknown 처리)하여 브랜치메트릭 계산, LDPC가 패리티만으로 복원.
6. 부호율 선택: rate 0.833 이상은 48-byte TA에 부족, 0.814(407/500)를 채택.
7. 구현 디테일: 첫 반복에서 tentative decision → LLR thresholding으로 제약 위반 구간을 찾아 erasure flag, 위반 수가 임계 미만이면 flag 전부 clear.
8. 학습/최적화: 없음(설계·시뮬 위주).
9. 결과: ECC-less rate 0.814가 ECC 포함 rate 0.889 대비 0.8 dB, MTR+ECC 대비 1.8 dB 이득; 48-byte TA·24-byte MD 정정, erasure 보상 없으면 TA 정정 불가.
10. 한계: HW 미설계·시뮬만, TA/MD·MTR·MaxLogMAP 등 자기기록 특유 채널 결함 모델에 강결합.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| erasure flag masking(브랜치메트릭) | 대응 없음 | Prime ECC는 MaxLogMAP 내부복호 없음; erasure를 LLR로 표현하려면 [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()`에 근접 |
| TA/MD·run-length 결함 검출 | 대응 없음 | NAND에는 TA/MD·run-length 제약 부재 |
| regular column-weight-3 LDPC | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | Prime ECC는 고rate QC-LDPC 고정, regular random 구조와 상이 |
| sum-product 외부 LDPC 복호 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | Prime ECC는 min-sum 근사 사용, full sum-product 아님 |

적용 가치: **낮음** — erasure를 LLR masking으로 넣는 아이디어는 개념적으로만 흥미로우나, 검출 대상(TA/MD/run-length)과 부호·내부복호가 자기기록 채널에 강결합되어 NAND binary QC-LDPC ECC에 직접 이식할 요소가 없음.
