### B. 알고리즘 요약

1. 대상: Tanner의 `[155,64,20]` sparse graph 부호에서 일반화된 `(j,k)`-regular QC-LDPC 블록부호 클래스에서 출발.
2. QC-LDPC PCM은 circularly-shifted identity(순환) 블록들로 구성되며, 각 circulant은 다항식으로 등가 표현됨.
3. 핵심 아이디어: QC 블록부호의 다항식형 PCM에서 부정원(indeterminate)을 `D`로 치환 → 대응하는 LDPC convolutional code `H(D)` 획득(QC를 unwrap).
4. 구성: `H(D)`의 원소는 `D^{a^p b^q -1}` 형태, `a,b`는 `GF(m)`의 곱셈위수 `k,j`인 비영 원소, 지수는 mod m.
5. rate `R = 1 - j/k`; convolutional code는 constraint length가 커서 trellis/sequential 디코딩이 비현실적.
6. 디코딩: constraint graph(QC 부호와 동일하되 period를 무한대로 확장)로 표현 → BP 알고리즘에 동일하게 적합.
7. 장점: convolutional 표현 덕에 초기 지연 후 결과를 continuous하게 출력(연속 디코딩).
8. 실험: rate `R=2/5`(`j=3, k=5`), 블록부호는 소수 `m=211`, `m=421`로 구성한 (3,5) 블록 LDPC → 대응 convolutional code 생성.
9. 결과: convolutional code가 cut-off rate 한계를 넘어 좋은 성능, 대응 블록부호보다 우수(더 큰 free distance 추정). BER 곡선은 그림에만 존재(수치 본문 없음).
10. 한계: HW 미설계, ISIT 확장초록 수준으로 gate/throughput/BER 정량치 없음. sparse graph라 고속 VLSI에 적합하다는 정성 언급만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC 블록부호 다항식형 PCM | `ecc_top.cpp` `Load_PCM()` | QC circulant 구조는 Prime ECC와 유사하나 unwrap 후 convolutional로 변형 |
| LDPC convolutional code `H(D)` 구성 | 대응 없음 | Prime ECC는 유한 블록 QC-LDPC 고정, convolutional 구조 미지원 |
| BP(continuous) 디코딩 | 대응 없음 | Prime ECC는 min-sum 블록 디코더(`decoder.cpp`), 연속 디코딩 아님 |

적용 가치: **낮음** — 블록 QC-LDPC를 unwrap해 만드는 convolutional/spatially-coupled 계열은 Prime ECC의 유한블록 min-sum 디코더 구조와 정합되지 않아 이식 부담이 매우 크다.
