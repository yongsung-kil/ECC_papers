### B. 알고리즘 요약

1. 시스템: BPSK 전송 binary LDPC(예시 regular (3,6)), memoryless compound channel(디코더는 채널 클래스만 알고 구체 채널 파라미터는 모름).
2. 문제: 채널 파라미터가 미지일 때 LDPC 복호 성능을 Monte-Carlo가 아닌 numerical(DE)로 해석·설계하고자 함.
3. 모델: compound channel `p(y|x;θ)`, θ∈Θ는 미지. 인코더는 θ를 알고 디코더는 클래스 Θ만 앎.
4. 핵심기법: 기존 variable/check node에 더해 `estimator node`를 그래프에 추가(4중 그래프 (λ,ρ,η,e)), Sum-Product 기반으로 한 iteration을 left-left-right-right 4단 메시지 패싱으로 구성.
5. 핵심식: estimator-to-variable 메시지 `m = sign(Y)·f₂(...)`로 채널 관측 Y와 인접 variable 메시지를 결합 → 미지 채널의 LLR을 온라인 추정하는 역할.
6. 확장: 초기화 함수 `f₁(Y)=θY`(AWGN/BSC 시 선형)와 estimator 함수 `f₂`를 채널 타입에 맞춰 설계.
7. 트릭: estimator node 수 `e`는 code rate와 무관하게 자유롭게 선택 가능(check node 수와 달리 제약 없음).
8. 최적화: off-line로 각 채널 대표 θ에 대해 DE를 iteration ℓ까지 수행, 통계로 best-fit 채널을 골라 estimator 함수 f_f 설계(Krichevsky-Trofimov 확률 예측을 LLR로 변환한 형태 활용).
9. 결과: (3,6) 앙상블 + compound BSC에서 known-channel threshold에 4번째 자리까지 손실 없이 접근(ML estimator ≈ KT estimator).
10. 한계: 순수 이론·DE 해석, HW 미설계, 정정능력/error-floor 개선 없음, 미지 채널 추정 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-Product 기반 메시지 패싱 루프 | `decoder.cpp` `LDPC_Decoder()` | full-tanh Sum-Product이며 Prime ECC는 min-sum → 직접 이식 안됨 |
| estimator-to-variable 채널 추정 메시지 | 대응 없음 | Prime ECC는 채널 통계 기지 전제라 추정 노드 개념 없음 |
| 초기화 함수 f₁(Y)=θY (LLR 변환) | `channel.cpp` `Set_LLR_Th()` | 개념적으로만 유사, 미지 채널 추정 목적은 무관 |

적용 가치: **낮음** — 미지 채널 파라미터의 online 추정용 이론이라 채널 통계가 알려진 NAND ECC 문제와 무관하고, HW·성능 개선 기여도 없음.
