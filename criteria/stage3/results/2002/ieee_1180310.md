### B. 알고리즘 요약

1. 시스템: rate 1/3 unpunctured turbo code(구성 (1,5/7) RSC 2개 + interleaver), BPSK, AWGN. 정보길이 N, codeword 3N.
2. 문제: turbo 복호의 BCJR(MAP/SOVA)는 복잡 → turbo의 PCM도 sparse함에 착안, LDPC식 sum-product로 대체해 복잡도 절감.
3. 모델: turbo PCM `H`(RSC PCM은 대각 분포, interleaver로 일부 random성 추가)를 factor graph(bit node/check node)로 표현, `c·Hᵀ=0`.
4. 핵심기법: log-domain sum-product로 `L(qᵢⱼ)`, `L(rⱼᵢ)` 갱신, `φ(x)=-ln tanh(x/2)` 함수 사용, APP/LLR `P(cᵢ=1|y,Sᵢ)` 계산.
5. 핵심식: check-node 갱신 `L(rⱼᵢ)=∏sign·φ(Σφ(|L(qᵢ'ⱼ)|))` — sparse PCM 상에서 SISO 복호를 병렬·저복잡도로 수행.
6. 확장(min-sum): 합에서 최소 `|L(qᵢⱼ)|` 항이 지배 → `φ(Σφ)≈min`으로 근사, check 갱신을 min-sum으로 치환, LLR을 `2/σ²`로 정규화해 noise 파라미터 σ² 불필요.
7. 트릭: `φ(x)` lookup table 또는 min-sum 사용, bit node당 LLR 1개만 저장 → 저장량 감소; turbo PCM의 대각 규칙성 활용해 일반 LDPC 대비 메모리 크게 절감.
8. 최적화: 없음(고정 구조).
9. 결과: block size 2종, 10 iteration. factor-graph sum-product/min-sum의 BER은 기존 turbo 복호 대비 개선 없음(short cycle, block size 작음, random성 부족 탓).
10. 한계: HW 미설계, 시뮬만, turbo code 전용, BER 개선 실패(정성적 원인만 제시), min-sum·LLR 정규화는 표준 재유도.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| log-domain sum-product / min-sum check 갱신 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC가 이미 min-sum(min1/min2+sign XOR) 보유 → 신규성 없음 |
| φ(x) lookup table / LLR 정규화 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 양자화 테이블 기반 magnitude와 개념 동일, 기보유 |
| turbo PCM factor graph 구성 | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 QC-LDPC 고정, turbo 대각 PCM 구조는 무관 |

적용 가치: **낮음** — 도출한 min-sum/LLR 정규화는 Prime ECC가 이미 보유한 표준 기법이고, 대상이 turbo code라 NAND binary QC-LDPC 이식 이득이 없다.
