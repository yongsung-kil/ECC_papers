### B. 알고리즘 요약

1. 대상은 binary LDPC convolutional code(연속 전송용, block code처럼 프레임 분할 불필요).
2. 문제: 기존 [1] 시변 구성은 랜덤, [2] 시불변 구성은 대수적(algebraic)이라 유연성 부족 — 임의 크기 H(D)를 얻기 어려움.
3. 핵심: [2]의 QC-LDPC block code를 "unwrap"하여 얻은 LDPC convolutional code의 parity-check `H(D)`(j×k, monomial in D)에 랜덤하게 선택한 다항식 entry를 도입.
4. sparse constraint graph 유지 위해 sparse weight 다항식만 사용 — weight>2 다항식은 length-6 cycle을 만들므로 monomial/binomial로 제한.
5. [2]와 달리 임의 크기 `H(D)` 구성 가능 → 더 넓은 code rate 범위 확보.
6. binomial entry를 쓰면 monomial만 쓴 경우보다 mildly irregular graph가 됨.
7. 실험: rate 3/6 LDPC convolutional code(3×6 H) 세 종류 — (a)monomial only, (b)binomial 2개+나머지monomial, (c)binomial 3개+나머지monomial. 다항식 최대차수 D^204, span 200.
8. 비교군: rate 2/5 convolutional code(span=200, [2]식)를 puncturing한 rate 2/4 부호.
9. 결과: 세 구성 모두 유사 성능, [1]의 시변 (1025,3(2),5)/(2049,3(2),5) 부호와 comparable(AWGN, Eb/No 1~1.8dB 영역).
10. 한계: degree distribution 최적화 여지 언급, HW 미설계, gate/throughput/error-floor 데이터 없음(1페이지 요약).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC convolutional code H(D) 구성 | `ecc_top.cpp` `Load_PCM()` + H-matrix | 대응 없음(convolutional 부호, block QC-LDPC와 상이) |
| QC block code unwrapping | `ecc_top.cpp` `Load_PCM()` | 개념적 연관만, 우리는 block QC circulant 고정 |

적용 가치: **낮음** — LDPC convolutional code는 연속 전송용 부호 클래스로 Prime ECC의 block QC-LDPC(4KB 페이지) NAND ECC와 근본적으로 다르며, HW·정정성능 데이터도 없는 1페이지 워크숍 노트라 이식 가치 없음.
