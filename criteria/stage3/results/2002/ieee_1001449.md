### B. 알고리즘 요약

1. 두 개의 초록 중 LDPC 관련은 GP 05 "Regular Lattice LDPC Codes"(GP 04는 DDFE equalizer로 LDPC 무관).
2. 문제: 랜덤 LDPC는 성능 우수하나 반복 디코더 복잡도가 커 차세대 자기기록의 >1 Gbps 데이터율에 부적합.
3. 목표: 고rate·저HW복잡·구조화된 regular LDPC를 정수격자로 구성해 수직자기기록 PR 채널 성능 조사.
4. 기법: BIBD(V,B)의 incidence matrix `H`를 패리티체크로 사용 (점 i가 블록 j에 있으면 `h_ij=1`).
5. 격자구성: 직사각 정수격자 `L={(x,y):0≤x≤k-1, 0≤y≤m-1}`, `m`은 소수; slope `s`의 직선상 점들의 k-원소 집합이 BIBD(λ=1).
6. 의미: `m`이 소수라 각 격자점을 지나는 slope `s` 직선이 유일 → 임의 두 점을 지나는 직선 유일 → λ=1 BIBD 보장(girth·구조 규칙성).
7. 파라미터: 블록크기 `k`, 블록수 `b=m^2`, 각 점은 정확히 `m`개 블록에 출현 → 서로 다른 column weight `J`의 고rate LDPC.
8. 시뮬: `J=4,5`에 대해 여러 PR target, 사용자밀도 ND, precoder, noise 조건에서 BER 평가.
9. 결과: ND=2에서 길이 4489 격자부호를 uncoded·random LDPC와 비교, 부호길이·`J`·media jitter(%)에 따른 BER 곡선 제시(수치는 그림에만 있어 미상).
10. 한계: 순수 구성법·시뮬 초록, HW 미설계, 수직자기기록 PR 채널 전제(NAND memoryless와 다름).

| BIBD/lattice 기반 H-matrix 구성 | `ecc_top.cpp` `Load_PCM()` | 낮음 - Prime ECC는 circulant QC-LDPC 고정, lattice 구조로 대체 시 전면 재검증 필요 |
| regular LDPC column weight J 설계 | `ecc_data.h` `PCM_b` | 낮음 - 우리 부호 degree(17)와 설계철학 상이 |
