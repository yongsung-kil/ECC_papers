### B. 알고리즘 요약

1. AWGN 채널용 binary LDPC로, PCM `H`가 단일 permutation 벡터(3정수 key `[m,a,b]`)로 완전히 기술되는 π-rotation 부호를 irregular·다중 rate로 확장.
2. 기존 random/quasi-regular 구성 대비, 결정론적 구성 + 단순 인코딩 회로 유지 + irregular로 성능 개선을 노림.
3. key `[m,a,b]`로 permutation 벡터 `π_A`를 생성하는 알고리즘(중첩 루프)으로 `TA` permutation 행렬을 만든다.
4. classic π-rotation은 `H=[Hp|Hd]`, `Hp`=dual-diagonal, `Hd`=`TA`의 4개 회전(π_A,π_B,π_C,π_D) → 길이 `8m`, rate 1/2.
5. 인코딩은 `Hp·cP = y = Hd·cd`(projection vector `y`)를 R^d/R^π 레지스터에서 정보 서브벡터를 permutation 주위로 회전시켜 계산.
6. 확장 패턴: parity 벡터 `cP`를 projection cycle로 되먹임해 행렬을 4m×8m → 16m×20m로 키우고 irregular column weight 생성.
7. 부수 트릭: 우측에 permutation 스택을 수직으로 쌓고 압축행렬 `CM_a`(1/a 압축)로 수평 압축 → column weight 밀도↑ 성능↑.
8. rate 제어: 파라미터 `LVec`(정보 세그먼트 수 1~4), `P.Vec`(펑처 parity 세그먼트 수), Stack Height(0~12), Compression(1/a); rate `R = LVec/(16·LVec - P.Vec)`.
9. 결과: 정보길이 `k=15,000` 샘플 부호들이 AWGN에서 여러 rate(≈3/8, 1/2 등)에서 Shannon limit 1 dB 이내 동작(Fig.7).
10. 한계: 시뮬(BER 곡선)뿐, gate count/throughput/error-floor 수치 없음, 통신 채널 타깃(soft-decision decoding 전제).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-diagonal `Hp` 인코딩 회로 | `encoder.cpp` `LDPC_encoder_4KB()` | 우리도 dual-diagonal 인코딩 사용 — 개념 유사하나 permutation-rotation 방식이 달라 직접 이식 아님 |
| permutation key `[m,a,b]`로 H 생성 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 지점이나 QC circulant(base+shift)과 구조가 달라 재검증 부담 큼 |
| message-passing 반복 복호 | `decoder.cpp` `LDPC_Decoder()` | 표준 BP 복호 언급뿐, 알고리즘 개선 없음 |
| 펑처링 다중 rate | 대응 없음 | Prime ECC는 고정 고rate QC-LDPC, 펑처 다중 rate 미대응 |

적용 가치: **낮음** — 결정론적 permutation LDPC 구성·인코더는 개념적으로 dual-diagonal과 겹치나, QC-LDPC circulant 구조·NAND 채널과 정합되지 않고 복호/HW 개선 기여가 없음.
