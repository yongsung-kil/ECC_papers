### B. 알고리즘 요약

1. 대상: `(j,k)=(3,6)` regular binary LDPC 코드, rate 0.5, length 2048, AWGN 채널 message-passing 복호.
2. 문제: 랜덤 구성 그래프는 연결 구조를 메모리(lookup table)에 저장해야 해 HW/DSP에서 저장 비용이 크고, 길이·rate가 다른 여러 코드마다 별도 저장이 필요함.
3. 핵심 가정: 짧은 사이클(길이 2·4)이 message-passing 성능과 error-floor를 좌우하므로 이를 피해야 함.
4. 핵심 기법: 최대길이 선형합동수열(LCS) 재귀 `π(i+1) = (a·π(i) + c) mod (jN)`로 bit connection ↔ check connection 매핑을 생성.
5. 식 의미: 재귀 파라미터 `a`, `c`만 저장하면 그래프 edge 주소를 on-the-fly로 계산 가능해 lookup 저장을 제거.
6. 확장: 조건 C1~C4(최대길이 보장)에 더해 C5~C9를 추가하면 girth>4 (길이 2·4 사이클 없음)를 이론적으로 보장(Theorem 1).
7. 구현 디테일: 복호 시 check/bit node의 extrinsic 정보 메모리 주소를 재귀로 생성, `a`,`c`,`π0`만 저장.
8. 최적화: 여러 length·rate 코드를 동일 재귀 틀로 설계(Code 1~6 예시), 유한기하 코드와 파라미터 매칭.
9. 결과: LCS 생성 코드의 BER이 4-cycle을 명시 회피한 제약 랜덤 구성과 거의 동일, 유한기하(PG/EG) 코드와도 대등.
10. 한계: HW 미설계, 시뮬레이션만, rate-1/2 짧은 regular 코드에 국한, 최소거리는 유한기하 코드보다 낮을 수 있음.
