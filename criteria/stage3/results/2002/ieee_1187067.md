### B. 알고리즘 요약

1. 시스템: 일반 binary LDPC(bipartite graph, `MxN` PCM `H`), soft-decision message-passing 복호, rate-1/2~8/9 예제, 블록 1024~10^7 bit.
2. 문제: LDPC 복호는 병렬성 내재하나 sparse graph로 인해 완전병렬은 인터커넥트(배선) 폭증, 직렬은 메모리 폭증 -> 구현 병목 정리가 목표.
3. 핵심연산: VN->CN 메시지 `Q_{n,m}`(2), CN->VN 메시지 `R_{m,n}`(3), CN 항은 `Φ(x)=-log tanh(|x|/2)` lookup table 근사 + MSB XOR로 sign 처리, signed-magnitude 표현.
4. 완전병렬 아키텍처: 노드->PE, 엣지->배선 직접매핑. 1024b 디코더[5]는 avg VN degree 3.25, 3-bit 메시지에 9750 wire, logic density 50%로 저하; 불규칙 H로 서브블록 분할 곤란.
5. 직렬 아키텍처: 소수 PE에 시분할, `Q`/`R`을 메모리에 임시저장 -> 면적·라우팅↓ 그러나 메모리↑ (rate-8/9 4608b는 iteration당 37000 메모리 접근).
6. 메모리 병목: SRAM 접근 ~2ns가 datapath(5-bit 4개 덧셈 <400ps)의 critical path 지배; 멀티포트 메모리는 면적 quadratic 증가로 부적합.
7. 라우팅 완화: crossbar switch로 P PE ↔ P SRAM 연결[6](충돌 시 stall, ad-hoc 스케줄을 ROM에 저장), pipeline+time-share[9](상용 IP, 최대 병렬도 128).
8. staggered 스케줄[10]: VN->CN 메시지 marginalization 생략 -> 메모리를 VN 수에만 의존; iteration<=5 제한 시 메모리 >75% 절감 대가로 BER <0.5dB 손실.
9. 부호구성↔구현: density-evolution 불규칙부호(max VN degree 100, 10^7 bit, >1000 iter)는 병렬 부적합; finite-field 기반 (32,64) rate-1/2 8190b 부호는 cyclic 구조로 shift-register·column-splitting 병렬화 이점.
10. 한계: 튜토리얼 성격, 신규 알고리즘 없음, 완전병렬은 대블록에서 라우팅 congestion으로 chip area 초과가 근본 난제.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN Φ(x)=-log tanh lookup + MSB XOR | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime은 min-sum 근사(더 단순), 본 논문의 tanh-LUT는 상위 개념 참고 |
| VN/CN 메시지 계산·양자화 | decoder.cpp VN_Cal_HD(), Get_VNU_* | 메시지 wordlength/signed-magnitude 논의는 양자화 설계 참고 |
| 직렬 메모리 병목 / crossbar 라우팅 | 대응 없음 (Prime은 z=32 부분병렬 HW 모델, GT.cpp Make_GT_HW로 edge 축소) | HW arch 병목 관점만 참고, 구체 매핑은 상이 |
| finite-field cyclic 부호(shift-register 병렬) | ecc_top.cpp Load_PCM() | QC-LDPC circulant 구조와 유사 개념이나 부호 자체는 다름 |
```

적용 가치: **중간** — 신규 알고리즘·부호는 없으나 LDPC 복호기의 병렬/직렬 메모리·라우팅 병목과 부호구성↔HW 정합 논의가 Prime ECC의 z=32 부분병렬·Graph Thinning 설계 배경 이해에 참고.
