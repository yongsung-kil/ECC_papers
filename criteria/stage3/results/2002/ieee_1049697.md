### B. 알고리즘 요약

1. 대상은 `N=L·k²=256·36=9216`-bit, rate-1/2, `(3,6)`-regular LDPC를 Xilinx Virtex-E XCV2600E FPGA에 구현한 partly-parallel 디코더.
2. 문제: fully-parallel 디코더는 초고속(1Gbps)이나 code length 증가 시 gate(1K-bit에 1.7M gates)와 Tanner graph 랜덤성으로 인한 라우팅 오버헤드가 비현실적.
3. joint code&decoder 설계: `H=[H1^T,H2^T,H3^T]^T` 구조에서 `[H1^T,H2^T]^T`가 girth-12인 (2,k)-regular가 되도록 구성(H1=identity 블록, H2=cyclic-shift `P=T^u(I)`), 4-cycle-free semi-random (3,k) ensemble에서 좋은 부호 선택.
4. 아키텍처: `k²` VNU와 `3k` CNU, 각 반복을 `2L` clock에 완료(1st L=CNP, 2nd L=VNP 교대), intrinsic/C2V/V2C 정보 모두 5-bit 양자화.
5. PE Block(`PE_x,y`)마다 RAM bank(EXT_RAM 3개+INT_RAM+DEC_RAM)와 VNU 1개, 3개 bi-directional shuffle network `π1,π2,π3`와 3k CNU로 연결.
6. CNP 루프: read-shuffle-modify-unshuffle-write로 V2C를 C2V로 변환, 양방향을 별도 wire set으로 구현해 datapath pipeline 가능(gate 절감 대신 wire↑).
7. 핵심 기법 π3: concatenated configurable random shuffle - intra-row + intra-column 2단으로 분리, 각 단이 R/Id permutation을 1-bit 제어(ROM 저장)로 매 clock 전환.
8. 라우팅 국소화: FPGA 다층 metal로 두 shuffle array를 겹쳐 배치, 배선을 한 row/column 소영역에 국한 → 라우팅 오버헤드 대폭 감소.
9. AG(address generator)를 counter+load로 구현, `t_{z,y}` 랜덤 로드에 2개 제약을 걸어 H가 항상 4-cycle-free 보장; 3-frame 2단 pipeline I/O.
10. 결과: post-routing 56MHz, `s=18` iteration에서 54Mbps throughput, 2dB@AWGN BER; slices 11792(46%)·LUT 15933·BlockRAM 90(48%), CNU/VNU가 slice의 74% 차지.
