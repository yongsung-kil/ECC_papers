### B. 알고리즘 요약

1. Extrinsic Information Transfer(EXIT) 차트가 반복복호/검출의 수렴거동을 예측하는데, 이를 정밀화하는 일반 복호 모델을 도입하는 1페이지 ISIT 초록.
2. 문제: EXIT 분석이 시뮬 관찰에 의존해 왔으며 그 특성을 이론적으로 규명할 필요.
3. 모델: 이진대칭 소스가 k 정보비트를 rate `k/n` 인코더로 length-n 부호어 `x`에 매핑, 두 채널을 정의 — `x→y`(communication channel)와 extrinsic(a priori) channel.
4. EXIT 차트는 디코더 입력 a priori LLR `a_j`에서 출력 extrinsic LLR `e_j`로의 평균 정보 증폭을 `I_E` vs `I_A`로 표현.
5. 병렬연접·직렬연접·LDPC 부호에서 `e_j`가 `y`와 `a_[j]`의 함수이며 MAP 비트 디코더가 `I(V_j;E_j)=I(V_j;Y,A_[j])` 의미로 최적임을 보임.
6. 핵심(Area Property, Theorem 1): EXIT 함수 아래 면적 `A=∫₀¹ I_E(I_A)dI_A`가 extrinsic 채널이 BEC일 때 부호 rate와 직결.
7. 직렬연접 적용: BEC에서 outer 부호 면적 `A_out=1-R_out`, inner 부호 면적 `A_in=I(X;Y)/n/R_in`.
8. 성공적 복호 조건 `1-A_out<A_in`으로부터 전체 rate가 채널 용량 미만이어야 함을 유도, inner rate<1이면 outer가 회복 못하는 고유 용량손실 → 반복복호 시 rate-1 inner 부호 권장.
9. 결과: 면적 결과는 복호 복잡도 함의와 LDPC EXIT 차트로 확장(참고문헌 [3]). 구체 부호·BER·수치·시뮬 없음.
10. 한계: HW 미설계, BEC 가정의 정보이론 증명 중심, LDPC는 확장 언급뿐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| EXIT 차트 면적/부호 rate 이론 | 대응 없음 | 부호설계 분석이론으로 Prime ECC의 고정 QC-LDPC H-matrix 로드(`ecc_top.cpp` `Load_PCM()`)와는 별개, 직접 대응 없음 |
| MAP 비트 디코더 최적성 증명 | 대응 없음 | Prime ECC는 min-sum 기반이며 본 논문의 MAP/정보이론 분석은 구현 모듈에 대응 없음 |

적용 가치: **낮음** — EXIT 면적=부호 rate 관계를 증명하는 순수 정보이론 초록으로, LDPC degree 분포 설계의 배경이론일 뿐 NAND binary QC-LDPC의 부호설계/디코더/HW에 떼어 쓸 구체 요소가 없다.
