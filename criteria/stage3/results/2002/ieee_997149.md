### B. 알고리즘 요약

1. 시스템: DMT 기반 ADSL/VDSL, 각 subcarrier에 2^b-QAM 멀티레벨 변조. 내부 trellis code를 binary LDPC로 대체하는 것이 목표.
2. 문제: ADSL은 RS+trellis 연접이나 capacity 근접이 부족 -> 고rate·linear-time 인코딩 가능하고 error-floor 없는 LDPC 필요.
3. 핵심 기법1(부호설계): "array code"를 이진 LDPC로 해석. PCM은 소수 `p`와 정수 `j,k`로 결정, `H_A`는 `jp×kp`, block마다 순환치환행렬 `sigma`의 거듭제곱 배치 (식 1).
4. 구성상 `H_A`는 두 행이 2곳 이상 겹치지 않아 4-cycle-free. `j`=열가중치, `k`=행가중치.
5. 핵심 식/트릭: block 단위 순환 shift로 `H_S` 생성 후 하삼각 원소를 0으로 만들어 상삼각형 `H`(식 4) 획득 -> generator 행렬 없이 recursive 인코딩.
6. 인코딩 복잡도 `(N/2)[r(j+3)+(j-3)]` XOR = linear-time. `p,j,k,N'`만으로 부호 완전 규정(on-the-fly 구성, 표준화 친화).
7. min-distance: `j=3`이면 `d_min=6`, `j=4`이면 `d_min>=8`. 상삼각화가 `j=3`에서 거리 손실 없음(`j=4`는 exhaustive search 검증).
8. 멀티레벨 매핑: set partitioning + double Gray-code labeling으로 LSB(부호비트)에 최상 soft 정보, MSB(무부호비트)는 thresholding. Soft demapper가 APP(식 7) 계산, 가까운 2 심볼만 합산 근사.
9. 복호: full SPA (또는 LLR 도메인 간이 SPA [13]), iteration 20 제한. 결과: N=529/2209/4489에서 random LDPC 동등 이상, 10^-7까지 error-floor 없음. net coding gain 4.1~6.2 dB (latency 0.5~8ms).
10. 한계: HW 미설계(복잡도 비교만), DSL/QAM 채널 특화, 순수 시뮬레이션.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| array-code 상삼각 PCM 구성 (linear-time 인코딩) | encoder.cpp Generate_PCM_encoding(), ecc_top.cpp Load_PCM() | 부호구조가 다름(array vs circulant QC), 인코딩 아이디어만 유사 |
| recursive 인코딩 (상삼각 exploit) | encoder.cpp LDPC_encoder_4KB() | 우리는 dual-diagonal, 개념적 유사 |
| soft demapper APP/LLR 계산 | channel.cpp Set_LLR_Th() | DSL QAM용 소프트정보라 NAND soft-read와 매핑 불일치 |
| SPA (full sum-product) 복호 | decoder.cpp LDPC_Decoder() | 우리는 min-sum, full-tanh SPA는 대응 없음(면적 큼) |
| 4-cycle-free / error-floor 억제 부호 | ecc_top.cpp Load_PCM() (H-matrix) | 참고 가치 있으나 재검증 부담 큼 |
```

적용 가치: **낮음** - 부호설계 관점의 array-code 상삼각 구성과 error-floor 억제는 참고할 만하나, DSL/QAM 멀티레벨·full-SPA 특화이고 우리의 circulant QC-LDPC/min-sum/dual-diagonal과 정합이 낮아 직접 이식 가치는 제한적.
