### B. 알고리즘 요약

1. 시스템: BIAWGN(binary-input AWGN) 채널, irregular LDPC 부호(rate 1/2, `Dv`=20), BP(sum-product) 디코딩. SNR mismatch ratio `η=σ_n²/σ_es²`(실제/추정 잡음분산 비)로 SNR 오추정 정도를 정의.
2. 문제: 수신단이 채널 SNR(잡음분산)을 정확히 모르면 BP 초기 LLR이 부정확(`N(2/σ_es², 4σ_n²/σ_es⁴)`, non-consistent)해져 성능이 크게 저하됨. 기존 zero-mismatch 최적화 irregular 앙상블(C1)은 mismatch에 매우 취약.
3. 핵심 가정: mismatch 시 메시지의 variance-to-mean 비가 반복마다 변하므로(no longer consistent), 기존 EXIT chart 생성법(ten Brink 등, no-mismatch 가정)은 그대로 사용 불가.
4. 핵심 기법: 주어진 offset `η`와 SNR `γ`에서 density evolution으로 특정 (dv,dc) regular 앙상블의 iteration별 메시지 분포를 얻고 상호정보량 `I(X,Y)`(식 (2))를 계산해 mismatch 하의 variable/check node EXIT curve를 생성, 다항식(6차) 곡선적합으로 irregular 앙상블의 curve는 constituent degree curve의 평균으로 근사.
5. 핵심 식 의미: SNR threshold `α(η)`(BER→0이 되는 최소 SNR)를 정의하고, 관심 offset 구간 `[η1,η2]`에서 `max α(η)`를 최소화하는 min-max 최적화로 degree distribution `λ(x), ρ(x)`를 설계(식 (4)(5) rate 제약 하에서 curve-fitting).
6. 확장: Min-Sum(MS) 알고리즘은 BP의 `η=+∞`(무한 SNR 과대추정) 극한과 동일함을 이용, 동일 EXIT 기법으로 MS 최적화 앙상블도 설계 가능(density evolution 대비 저복잡도 대안).
7. 구현 디테일: 특별한 양자화/HW 트릭 없음. 순수 이론적 앙상블 설계(asymptotic).
8. 최적화 절차: EXIT chart 기반 curve-fitting(최대 6차 다항식)으로 관심 offset 구간의 worst-case threshold를 최소화하는 degree distribution 탐색(density evolution 직접 탐색 대비 저복잡도).
9. 결과: 설계된 앙상블 C2(offset 구간 [-2,2]dB 최적화)는 offset zero 성능 손실 거의 없이 C1 대비 offset 범위 전반에서 threshold 개선. C3(MS 최적화)는 offset→∞까지 threshold 증가가 0.2dB에 불과(C1은 1.2dB). 유한장(length 5000~50000) 시뮬에서도 BER 10⁻⁴ 기준 mismatch 허용 범위가 C1([-1.5,1.5]dB) 대비 C2([-1.7,2.0]dB)로 확장.
10. 한계: BIAWGN 채널·SNR(잡음분산) 오추정이라는 특정 문제 설정에 국한되며, HW 구현/복잡도 분석 없음(순수 asymptotic+시뮬 검증). 코드 length 5000 이상 장부호 위주로 NAND ECC의 짧은-중간 length QC-LDPC와는 다른 영역.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SNR mismatch 견고 irregular degree distribution 설계(EXIT chart min-max) | `ecc_top.cpp` `Load_PCM()` (H-matrix 설계 관점) | 이식 난이도 높음(부호설계 영역) + NAND는 SNR mismatch 개념 자체가 안 맞아 관련성 낮음 |
| Min-Sum이 BP의 무한 SNR 과대추정 극한이라는 관찰 및 MS 앙상블 최적화 | `decoder.cpp` `LDPC_Decoder()` (min-sum 이미 사용) | Prime ECC는 이미 min-sum 채택, 해당 관찰은 참고용 배경지식 수준 |
| iteration별 LLR 분포(density evolution) | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | LLR 테이블 설계와 개념적으로 유사하나 SNR mismatch 대응 목적이 아니라 직접 대응 없음 |

> 적용 가치: 낮음 — BIAWGN·SNR 오추정 문제에 특화된 부호설계 이론이며, NAND 컨트롤러는 채널 특성(RBER)을 자체 추정/캘리브레이션하는 구조라 이 논문의 문제 정의가 그대로 적용되지 않음.
