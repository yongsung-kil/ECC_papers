### B. 알고리즘 요약

1. 대상은 길이 `n=128`의 eBCH, 5G CA-polar `[128,64]`, random linear code `[128,106]` 등 short linear block code이며 BPSK-AWGN 채널을 가정.
2. 기존 GRAND(GND) 계열은 짧은 코드에서 ML/near-ML 성능을 내지만 저·중 rate에서 guess 수가 많아 비효율적이므로, guessing codeword decoding(GCD)이 GND보다 효율적임을 규명하고 실용 알고리즘화가 목표.
3. TEP(test error pattern) `e`는 `v=z+e`로 정의되며, `H`를 systematic form `[I,P]`로 변환 후 offline GE는 최대 1회만 수행(OSD는 매 수신마다 online GE 필요).
4. 정수화된 신뢰도 순위 `λi`로 logistic weight `Γ(e)=Σλi·ei`를 정의하고, 정보부 partial TEP `e_R`을 `Γ` 비내림차순으로 생성하는 ORB-GCD를 제안(식 2, 5).
5. 조기종료를 위해 `τ_DAI = Σ E[ei|ri]·λi` (식 10)를 통계적 기댓값으로 근사해 `Γ(eopt) ≤ τDAI + Γ(e_R^(j))`이면 탐색을 중단하는 DAI 기준을 도입(Algorithm 1).
6. 확장판 ORB-LC-OSD(Algorithm 2)는 국소 제약(local constraint, degree `δ`)을 두어 부분 GE만 수행하고 동일한 DAI 종료 기준을 적용.
7. 부수 트릭: 신뢰도를 실수 LLR 대신 정수 순위(rank)로 표현해 정수 연산만으로 logistic weight 계산이 가능(HW 친화적 정수화).
8. 별도 학습/최적화 절차는 없음(고정된 정수 파티션 생성기와 통계적 기댓값 기반 종료 기준만 사용).
9. 결과: eBCH `[128,k]` 계열에서 ORB-GCD가 ORBGRAND 대비 평균 TEP 수가 유의미하게 적고, ORB-LC-OSD가 RCU bound에 근접하는 최저 요구 SNR을 달성.
10. 한계: 순수 시뮬레이션(이론+수치)이며 HW 설계·gatecount·throughput 언급 없음. LDPC가 아닌 short block code(BCH/polar/random linear code) 전용이며 반복 메시지 패싱 디코더 구조와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ORB-GCD / ORB-LC-OSD (guessing codeword decoding) | 대응 없음 | Prime ECC는 min-sum 기반 반복 메시지 패싱 LDPC 디코더로, GE 기반 codeword-guessing 디코딩 구조 자체가 없음 |
| DAI 조기종료 기준 | `partial_CRC.cpp` `partial_crc_HW_T4()` (개념적 유사, 구현 불일치) | 조기종료라는 목적은 유사하나 CRC 기반과 통계적 logistic weight 기반은 메커니즘이 전혀 다름 |
| 정수 기반 신뢰도 순위(정수화 LLR) | `channel.cpp` `Set_LLR_Th()` (개념적 유사) | LLR 양자화라는 공통점은 있으나 순위 기반 재정렬은 Prime ECC 디코더 파이프라인과 무관 |

적용 가치: **낮음** — short block code(BCH/polar/random) 대상 GE 기반 OSD/GRAND 계열 디코딩으로 QC-LDPC 반복 min-sum 복호 구조와 근본적으로 다르며, NAND용 고rate 긴 코드(length 수천~수만 bit)에는 GE 연산 비용이 과도해 실용적 이식 경로가 없음.
