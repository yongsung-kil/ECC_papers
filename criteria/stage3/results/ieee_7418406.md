### B. 알고리즘 요약

1. IEEE 802.11ad 표준의 QC-LDPC 4종 rate(1/2, 5/8, 3/4, 13/16), Z=42 서브행렬, mZ x nZ parity-check matrix(m=16 고정, n=3/4/6/8) 대상 min-sum 디코더.
2. 기존 row-parallel 구조는 대용량 memory와 pipeline register로 전력 소모가 크고, eDRAM(선행연구 [2])이나 FDSOI(선행연구 [3])는 표준 저전력 CMOS 공정에 부적합해 아키텍처 혁신이 필요.
3. 핵심 모델: min-sum 기반 VN/CN 반복 처리, LLR 5비트 양자화, 최대 iteration 7회로 고정.
4. 핵심 기법 1단 - column-parallel 아키텍처: CN을 8Z개로 완전병렬, VN은 2Z개로 부분병렬 처리, parity-check matrix를 8개 chunk로 분할하고 각 chunk를 좌/우 half로 나눠 V2C/C2V shifter에 매핑.
5. 메모리 절감 핵심: posterior/V2C 메시지는 sign bit만 저장(soft bit 불필요), C2V 메모리는 전체 메시지 대신 min/2nd-min 값만 저장 → 총 메모리 12096bit로 기존(30240bit) 대비 60% 절감(Table I).
6. 핵심 기법 2단 - reduced-complexity 2-stage variable shifter: 기존 log2(Z) 단 barrel shifter 대신, 4종 rate의 12가지 shift case를 modulo-42 덧셈 두 조합으로 분해해 2단 MUX-cascade로 구현, 회로 42% 절감·critical path 33% 감소(Fig. 6).
7. 부수 트릭: coarse clock gating(조기종료 기반, 최대 55% 전력 절감) + fine-grained gating(비제로 서브행렬 수/row 수 기준 pipeline register 선택적 활성화), shortened code(PHY header/MCS0,1)는 chunk 수만 줄여 처리해 디코딩 지연 절반 감소.
8. 최적화 절차: shifter 분해 조합은 exhaustive search로 고정 cyclic shifter 개수를 최소화하도록 탐색(학습이 아닌 조합 탐색).
9. 결과: 40nm LP-CMOS 시제품 6.16Gb/s, 4.7pJ/bit/iteration(정규화 후에도 최저), LDPC 디코더 면적 0.80mm², BER<10^-8 @ Eb/N0=5dB, 전력 203mW(early termination 적용 시 106mW로 감소).
10. 한계: 알고리즘(min-sum) 자체의 정정성능 개선은 없음(순수 아키텍처/회로 최적화), 802.11ad 표준의 특정 Z=42/4-rate 구조에 특화되어 일반화에는 재설계 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| column-parallel 데이터패스(V2C sign-only, C2V min/2nd-min 저장) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 min-sum(min1/min2) 기반이라 C2V min/2nd-min 저장 개념은 유사하나, column-parallel 자체 구조(2Z VN, 8Z CN)는 Prime ECC의 row-parallel + HCU/GT 구조와 상이 |
| reduced-complexity 2-stage variable shifter | `GT.cpp` `Make_GT_HW()`(MUX 기반 edge 선택과 개념적 유사) | 목적(회로 절감)은 유사하나 802.11ad 특정 Z=42/4-rate 조합에 특화된 설계라 z_sb=32 구조로의 직접 이식은 재설계 필요 |
| coarse/fine clock gating (조기종료 기반 전력절감) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 자체는 Prime ECC에 이미 구현(Partial CRC), gating을 통한 전력 절감 아이디어는 참고 가능하나 본 프로파일에 전력 최적화 항목 명시 없음 |
| 메모리 절감(C2V/V2C bit 수 축소) | `ecc_data.h` `PARAM_DEC`, `decoder.cpp` `LDPC_Decoder()` | HW 메모리 사이즈 최적화 아이디어로 Prime ECC decoder HW 모델에 참고 가치 있음 |

> 적용 가치: 중간 — 정정성능 개선은 없지만 실측 검증된 column-parallel HW 아키텍처와 메모리/shifter 절감 기법은 Prime ECC의 HW arch 레이어(병렬화·메모리·라우팅) 최적화에 참고할 만함.
