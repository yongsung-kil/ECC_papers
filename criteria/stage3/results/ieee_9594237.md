### B. 알고리즘 요약

1. 대상: B5G용 초고throughput FEC 디코더. LDPC는 (1032,860) EPIC 부호(rate 0.833), (648,540) 802.11n 부호(rate 0.833), Min-Sum, Q=4bit, BPSK/AWGN.
2. 문제: throughput↑ 요구 대비 TDP(전력) 예산 고정 → 에너지효율 `E_eff=P/T_inf`이 병목. throughput `T_inf ~ N·R·(1/I)·π·f·(1-ω)` (식4).
3. 핵심 관찰: unrolled 아키텍처(`π=I`)는 iteration을 펼쳐 각 stage에 VFU/CFU + 2·I 배선망 인스턴스화 → 라우팅 면적/전력 지배, cell 이용률 저하.
4. 기법1 (LDPC layered unrolled): H-matrix를 `l=4` 수평 layer로 분할, 한 iteration stage를 l개 stage로 대체해 stage당 check node 병렬도·라우팅 복잡도↓. layered로 수렴 빨라져 iteration `I` 감소(TP 8 → layered 4).
5. 효과: EPIC 부호 FER 1e-5@5dB 고정 시 면적 10%↓, 에너지효율 26%↓, 전력 21%↓, 이용률 10%↑. extrinsic은 compressed beta format + circular buffer FIFO로 토글 절감.
6. 기법2 (Iteration Control, IC): unrolled TP에 PCHK(parity-check) 조기종료 도입 → clock gating으로 불필요 iteration stage off, HD를 채널값 파이프라인 LSB에 기록해 별도 메모리 불필요.
7. IC trade-off: 802.11n 부호에서 면적 29%↑·주파수 8%↓(라우팅/HD 오버헤드)이나 고SNR에서 전력·에너지효율 대폭 개선(8dB에서 8-iter가 conventional 2-iter 전력 수준).
8. 기법3 (Polar): SC 디코더 deeply pipelined, register/clock-tree가 전력 70% → `f`·`V_DD` 스케일링(1.0V→0.7V)+공간병렬(dual-core)로 에너지효율·전력밀도 최적화.
9. 결과: LDPC layered 474Gbps/4iter, unrolled IC 837Gbps. Polar dual-core 0.7V 800Gbps에서 에너지효율 1.59pJ/bit. 모두 22/28nm FD-SOI 합성·P&R.
10. 한계: 통신부호(EPIC/802.11n) 대상, HW 합성·layout 검증(실측 아님), unrolled 완전병렬 전제가 NAND 컨트롤러 구조와 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered 스케줄링(수렴 가속, iteration↓) | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` 이터레이션 루프 | 우리는 이미 Dual-Update(parity even/odd 교대) 보유, layer 분할 개념 유사 |
| PCHK 조기종료 + clock gating | [partial_CRC.cpp](../Prime_ECC_3.1_Claude) `partial_crc_HW_T4()`, `full_CRC.cpp` | 우리는 CRC 기반 조기종료 보유, parity-check 기반 종료는 대안 관점 |
| Min-Sum CN/VN 연산(Q=4bit) | [decoder.cpp](../Prime_ECC_3.1_Claude) `CNU_Update_New_Mag()`, `VN_Cal_HD()` | 동일 min-sum 계열, 양자화 폭 참고 |
| unrolled/라우팅 HW 아키텍처(π=I) | 대응 없음 | 우리 HW 모델은 z=32 부분병렬(HCU/GT), unrolled 완전병렬 미대응 |

적용 가치: **중간** — layered로 iteration을 줄이고 라우팅을 완화한다는 방향성은 우리 디코더 최적화에 참고되나, unrolled 완전병렬(π=I) 전제가 NAND 컨트롤러 부분병렬 구조와 달라 아키텍처 이식은 어렵고 알고리즘 관점(스케줄링·조기종료)만 부분 참고 가능.
