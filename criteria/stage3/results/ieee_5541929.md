### B. 알고리즘 요약

1. 대상: 이진 LDPC(구조적 코드 length 2016/info 1692, PEG 랜덤 코드 동일 밀도분포), BPSK+AWGN, 최대 iteration 50.
2. 문제: Min-Sum은 CN update에서 `f(x1,x2)=sign·min(|x1|,|x2|)+small_factor`의 small factor를 버려 BER 손실. 양자화 시 bit수 절감과 성능 유지 동시 달성이 목표.
3. 모델: BPSK+AWGN에서 `L(xi)`는 두 가우시안의 합(분산 `1/σ²`)이라는 채널출력 LLR 분포를 명시적으로 유도.
4. 핵심기법1: BP-on-LLR을 정밀화 — CN update를 `L(rji)=Σ⊕L(qi'j)`로 쓰고, `f`를 magnitude항 `min` + search table `F(x1,x2)`로 분해(식 22,23)해 min-sum이 버린 small factor를 복원.
5. 핵심식: 양자화 스케일 기준 `[-3/σ, 3/σ]`(식 20) — 채널출력 LLR의 각 가우시안 성분 99%(±3σ)만 커버. 스케일이 너무 크면 수렴 실패, 너무 작으면 정보 손실.
6. 핵심기법2: step length `Δs`(식 21) 도입 — 채널 분산·bit수로 결정. Δs가 2의 거듭제곱이면 정확 비트 양자화 의미.
7. 구현 디테일: CN update용 search table 크기 `2^q×2^q`를 채널분산 기반으로 iteration 전 미리 구축; VN update 결과는 `[-2^(q-1), 2^(q-1)-1]`로 clip(식 24).
8. 학습/최적화: 없음(해석적 기준). HW용으로 채널분산별 테이블 대신 정적 step length 1개 선택(4/5/6bit=0.75/0.375/0.1875, Table I).
9. 결과: 6bit 무시할 손실, 5bit <0.01dB, 4bit ~0.1dB(10^-5 BER 기준). 10^-7까지 error floor 없음; 정적 step length가 가변과 동등하거나 더 우수.
10. 한계: HW 미설계(gatecount/throughput 없음), search table 필요로 순수 min-sum보다 무겁고 4~6bit 고해상도 전제.
