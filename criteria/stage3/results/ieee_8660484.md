### B. 알고리즘 요약

1. 시스템: 상향 NOMA 랜덤액세스(IDMA), BPSK, Gaussian MAC. 코드는 rate-compatible RL-QC-LDPC, `L=32`, `k=320`, `RL=1/5~2/5`, bit-repetition(`Rrep`) 옵션 병용.
2. 문제: 5G-NR LDPC를 IDMA에 직접 쓰면 낮은 user load에선 되나 high user load/throughput에서 outage 성능이 급락.
3. 모델: MUD(ESE)와 LDPC 디코더 간 outer iteration + LDPC 내부 inner iteration을 함께 추적해야 임계값 계산 가능(turbo joint-decoding).
4. 핵심기법1: base matrix를 row-by-row로 확장(raptor-like 구조 유지)하며 nested base matrix 계열 `B(K,M)` 구성.
5. 핵심식: 각 edge를 `(p,v,c)`(punctured여부·VN degree·CN degree) 3-tuple로 분류한 EC-I spectrum과 `(p,v)` 2-tuple EC-II spectrum으로 후보를 그룹화.
6. 핵심기법2: EC-I로 임계값 계산 대상 후보 축소, EC-II로 seed 다양성 확보(beam search, `nseed=4` seed 유지).
7. 구현 디테일: `cmax/cmin`로 확장 row degree 제약, `Jtarget=18`(초기엔 `Jtarget=M+1`) target user 수로 임계값 gap 최적화.
8. 최적화 절차: multi-edge type DE-aided EXIT chart로 `SNRDE-EXIT` 임계값 예측, Shannon limit과의 gap 최소화.
9. 결과: M=22→23 확장 시 후보 65536→5880(degree 제약)→1457(EC-I)로 축소. 새 코드계열이 5G-NR 대비 최대 sum spectral efficiency를 0.9→2.1~3.17 bits/chip로 향상.
10. 한계: 순수 부호 구성법+시뮬(BLER)뿐, HW 미설계, IDMA/NOMA 다중접속 특화 저rate 코드로 NAND ECC와 무관.
