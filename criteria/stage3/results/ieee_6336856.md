### B. 알고리즘 요약

1. 시스템: irregular LDPC(edge degree 분포 `λ(t)`,`ρ(t)`)를 BSC(`p0`)에서 Gallager B 디코더로 복호, hard-decision message passing.
2. 문제: 나노공정 소자는 본질적으로 불량(transient error) - faulty HW 위 디코더의 최종 BER를 정량화하고 자원(프로세서 신뢰도)을 최적 배분해야 함.
3. 모델: von Neumann transient error 모델 - 각 VN/CN 메시지가 프로세서 오류율 `ql`로 확률적 반전(stochastic computing = noisy channel 해석), degree별로 다른 오류율 허용.
4. 핵심 기법(DE): 노이지 density evolution - iteration `i`의 평균 VN→CN 오류 `pi`의 점화식 `pi+1 = f(pi)` (Theorem 1), CN측 집계 `α(pi)` 정의.
5. 의미: 프로세서 오류 `ql`이 각 노드 갱신 확률에 `(1-2ql)Pr{...}+ql`로 편입되어 채널오류와 처리오류를 동시에 반영.
6. 임계값: 잡음하 VN 임계값 `b_j`가 오류 `ql`에 무관하게 닫힌형(Lemma 1)으로 유도, 잔여오류 `p`의 고정점 도출.
7. 잔여BER: 작은 `p0,ql` 근사에서 `p ≈ [θ(v)+(λ2+2p0λ3)θ(c)] / [1-...]`(Theorem 2), `θ(v)`,`θ(c)`는 VN/CN 평균 프로세서 오류.
8. 최적화(LP): 총 비용 `W` 제약하 `p`를 최소화하는 프로세서 배치를 선형계획(simplex/interior point)으로 해결.
9. 결과: MATLAB 시뮬(코드1 9972 VN / 코드2 1920 VN)에서 최적배치가 무작위 대비 동일 BER(~3e-4)를 약 4/5 비용으로 달성.
10. 한계: HW 미설계, 무한블록 이론+유한블록 시뮬, hard-decision Gallager B 전용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Gallager B 하드결정 message passing | `decoder.cpp` `LDPC_Decoder()` | Prime는 soft min-sum, hard BF 계열 아님 |
| noisy density evolution / 잔여BER 이론 | 대응 없음 | Prime은 bit-exact 시뮬레이터, faulty-gate 모델 없음 |
| degree별 프로세서 신뢰도 LP 배분 | 대응 없음 | Prime HW 모델(z=32/HCU)에 노드별 신뢰도 개념 없음 |

적용 가치: **낮음** - 불량 게이트 위 Gallager B(hard-decision) 디코더의 신뢰성 이론과 자원배분으로, Prime의 min-sum/soft-read 기반 정정성능·HW 최적화와 겹치는 부분이 없음.
