### B. 알고리즘 요약

1. 대상: (9728,1024) QC-LDPC, code rate=0.89, row weight=36, NAND Flash 채널, 레이어드 디코딩, 조기종료(early termination) 적용, max iteration=8.
2. 문제: 기존 probabilistic min-sum algorithm(PMSA, [2])은 그룹 트리 탐색으로 하드웨어 비교기 수를 줄이지만 확률적으로 구한 2nd minimum과 정확한 1st minimum 간 gap이 커져 오류정정 성능이 저하됨.
3. 채널가정: NAND Flash read 채널(σ로 표현되는 노이즈 파라미터, 상/하위 페이지 별도 곡선)에서 시뮬레이션.
4. 핵심 기법: 정확한 1st minimum(`min1st`)과 확률적 2nd minimum(`prob.min2nd`)을 가중치 `α`, `β`와 스케일링 인자 `γ`로 결합해 `newmin2nd = γ*(α*min1st + β*prob.min2nd)` 산출.
5. 식의 의미: 기존 [3]은 고정 음수 factor로 2nd minimum을 조정했으나 표준별 최적 factor를 찾기 어려웠고 overcorrection(reversal, new min2nd < min1st) 문제 발생; 본 논문은 `α+β=1` 제약(α=2^-x, β=1-2^-x, x는 양의 정수)으로 reversal 문제를 추가 비교기 없이 원천 차단.
6. 그룹 분할 수(1/2/4/8)에 따라 prob.min2nd 오류확률이 감소(Table I: 0%/50%/25%/12.5%)하며, 그룹 수가 늘수록 가중치 α를 감소시켜 min1st 영향을 줄이는 설계.
7. 부수 트릭: 가중치·스케일링 조합을 adder와 shifter만으로 구현해 하드웨어 친화적으로 설계(곱셈기 불필요).
8. 학습/최적화 절차: 없음(고정 가중치 설계, Table I 기반 경험적 설정).
9. 결과: 4그룹 기준 INPMSA는 [3]과 유사한 BER 성능을 달성하면서 critical path(period 2.7ns, 370.37MHz)가 [3](rExmin, 2.78ns) 대비 짧고, area는 14521um2로 NPMSA(13286um2) 대비 소폭 증가.
10. 한계: BER 수치는 그래프(Fig.2)로만 제시되어 본문에 구체적 개선폭 수치가 없음(구조상 waterfall 개선 언급, error-floor 별도 분석 없음); 공정 노드 미기재, 시뮬레이션 검증과 ASIC 합성 결과만 있고 FPGA/실측 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| probabilistic min-sum (그룹 트리 min1/min2 탐색) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min1/min2 기반 min-sum 연산과 직접 대응, 그룹 가중치 보정 로직 이식 검토 가능 |
| 가중치·스케일링 인자(α, β, γ) 조합 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 내 min-sum 후처리 | 신규 보정 로직으로 이식 시 low-cost(adder/shifter) 추가만 필요, 난이도 낮음 |
| reversal 문제 방지(α+β=1 제약) | `decoder.cpp` `CNU_Update_New_Mag()` | 기존 로직에 조건 없이 안전성 보장 방식으로 대체 가능성 검토 가치 있음 |
| NAND Flash 채널 시뮬레이션(σ 기반) | `channel.cpp` | Prime ECC 채널 모델(RBER/fixed-error)과 유사 목적, 직접 재사용은 불필요 |

적용 가치: 중간 — NAND Flash 채널에서 직접 검증되고 min-sum 후처리만 경량 추가하는 구조라 이식 난이도는 낮으나, 정량적 성능 개선폭이 본문에 명시되지 않아 실질 이득 확인이 우선 필요.
