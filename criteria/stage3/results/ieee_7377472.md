### B. 알고리즘 요약

1. 대상은 soft-decision/adaptive 디코딩 알고리즘(ABP, OSD, box-and-match, LP decoding 등)이 요구하는 full parity check matrix(H) 저장 문제.
2. 기존 방식은 H를 RAM에 통째로 저장하거나 LUT로 구현해 하드웨어 자원을 많이 소모.
3. 제안 방식은 H가 구조(BCH/RS의 Vandermonde 구조, QC-LDPC의 circulant 구조)를 가진다는 점을 이용해 "저장" 대신 "생성"하는 아키텍처를 설계.
4. BCH: 열 `c`의 첫 행 GF 지수는 `c`, 이후 행은 `c (mod 2^m-1)`씩 누적 가산하는 Vandermonde Generator + 지수→이진벡터 변환 LUT(Binary Transformation) 2단 구조.
5. RS: 비이진 Vandermonde 행렬의 binary image 표현에서 `A^i = [α^i α^(i+1) ... α^(i+m-1)]` 관계를 도출, GF 원소별 m×m 행렬 전체 대신 GF 원소의 이진벡터 열만 LUT로 저장하는 구조를 제안(Section IV-B).
6. LDPC(QC-LDPC, WiMax 코드): base matrix(각 z×z circulant의 위치·shift값)를 LUT로 저장하고, slice generator가 shift 벡터와 offset 주소로부터 해당 열을 계산하는 2단 아키텍처(Fig. 5, 6).
7. 구현/검증: Virtex-7 FPGA(Vivado P&R) 및 65nm ASIC(Synopsys DC) 합성, 클록 300MHz(FPGA)/650MHz(ASIC) 이상.
8. 별도 학습/최적화 절차 없음 (구조 기반 결정론적 설계).
9. 결과: RS(255,x)에서 최대 5배 면적 절감(RAM 404,800 → Generator 73,060 μm² 등, Table II); LDPC(576,288)에서 RAM 183,000 → Generator 9,120 μm²(Table IV). 단, 소형 행렬(BCH(63,39) 등)은 LUT가 더 효율적.
10. 한계: 디코딩 알고리즘 자체는 다루지 않고 H 저장/생성 HW만 다룸; LDPC 파트는 WiMax QC-LDPC 3개 코드만 예시로 다뤄 논문의 주 기여가 BCH/RS 쪽에 치우침; error-floor/waterfall 등 정정성능 논의 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| QC-LDPC base matrix + shift LUT 생성 아키텍처 | `ecc_top.cpp` `Load_PCM()` | Prime ECC도 base+shift(`PCM_b`) 구조를 이미 사용 중이라 유사 개념이나, 논문은 순수 저장 최적화 HW이며 부호설계/디코더 성능과 무관 |
| BCH/RS Vandermonde 생성기 | 대응 없음 | binary QC-LDPC 전용 코드베이스와 무관한 비이진/algebraic code 기법 |
| Slice Generator (circulant 열 생성) | `ecc_top.cpp` `Load_PCM()` | 유사 목적(circulant 압축 저장/생성)이나 이미 유사 구조 보유 추정, 개선 폭 제한적 |

> 적용 가치: 낮음 — Prime ECC는 이미 base+shift 압축 PCM 구조(`PCM_b`)를 사용 중이며, 본 논문은 min-sum/디코더/부호설계 성능 개선이 아닌 순수 H 저장 HW 최적화(그것도 주로 BCH/RS 대상)라 이식 실익이 적음.
