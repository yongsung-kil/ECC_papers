### B. 알고리즘 요약

1. 3D TLC NAND(8개 전압상태, LSB/CSB/MSB 3비트)에서 read latency는 data sensing(SRO 수)과 ECC(LDPC) 디코딩 시간에 좌우됨.
2. 기존 연구들은 데이터 배치(hot/cold page 분리)나 데이터 변조(Huffman coding)만 다뤄 read variation과 reliability를 동시에 개선하지 못함.
3. 채널모델: retention error(좌측 이동, 고전압 상태에 취약)와 program/interference error(우측 이동)로 인접 상태로만 전압천이가 발생한다고 가정.
4. Coupled Data Storage(CDS): 원본데이터(raw)와 매핑데이터(coupled)를 plane 0/1에 동시 프로그램. 고속모드는 8개 상태를 4개(S1~S4 또는 S5~S8)로 줄인 reduced mapping table을 사용해 MLC급 SRO 수로 감소.
5. 정정모드는 역매핑 테이블(FRM/CRM/HRM 3종, 각 4개 pairing)을 두고, 읽은 두 상태 `Li`, `Ri`로 조정식 `Si* = Li + Remap(Ri))/2` (경계 상태는 ±8 보정)를 적용해 전압천이 오류를 복원.
6. 부분천이(coupled data 중 한쪽만 shift)의 경우 상태별 비대칭 천이확률(고전압상태가 더 left-shift 경향)을 근거로 floor/ceiling 결정하는 state-aware 분석 적용.
7. HRM(상태 간 거리=4, 인접 매핑쌍 없음)이 FRM/CRM 대비 실패케이스가 적어 RBER 감소폭이 가장 큼(HRM 52.7% > CRM 46.4% > FRM 37.1%).
8. 별도 학습/최적화 절차 없음(고정 매핑테이블, 실측 기반 P/E cycle별 천이확률 분석만 수행).
9. 실측 결과: 고속모드는 read latency 50~66.6%, write latency 25.7~27.5% 감소; 정정모드는 RBER 37~52% 감소, decoding level(soft-read 단계 수) 10~38.14% 감소, 수명 1.6~3배 연장.
10. 저장공간 2배(coupled data) 오버헤드 필요, HW(RTL) 미설계, LDPC 부호/디코더 알고리즘 자체는 변경하지 않고 입력 RBER만 낮추는 시스템/펌웨어 레벨 기법.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Coupled data storage / reverse mapping table | 대응 없음 | NAND 컨트롤러 데이터 배치·매핑 계층 기법으로 ECC 코어 모듈과 무관 |
| RBER 감소를 통한 decoding level 축소 | `channel.cpp` `Set_LLR_Th()` | RBER 입력값이 낮아지면 LLR threshold 테이블의 low-RBER 구간 선택 빈도가 늘어나는 정도로만 간접 연관 |
| 전압 상태 조정식(adjustment equation) | 대응 없음 | 아날로그 Vth 레벨 보정으로 디코더/부호 로직과 접점 없음 |
| soft-decision 단계 수(decoding level) | `ecc_data.h` `PARAM_LLR`(iteration별 LLR 테이블) | 소프트 리드 단계 감소는 LLR 테이블 재조회 빈도를 줄이는 효과와 개념적으로 유사 |

> 적용 가치: 낮음 — LDPC 부호설계·디코더 알고리즘·HW 구조를 다루지 않는 상위 데이터 저장/매핑 기법으로 Prime ECC 코드 모듈과 직접적 접점이 없음.
