### B. 알고리즘 요약

1. 시스템: 분산저장(DSS)에서 데이터를 `n`개 노드에 분산, 노드당 저장 `α=1`, BEC 소거 채널 모델로 데이터 손실 기술, 코드 rate `R=1-d̄v/d̄c`.
2. 문제: RS/재생부호는 저장 최적이나 수리대역폭 크거나 복호복잡도 높음, LDPC는 저복잡도지만 수리대역폭-복원능력 trade-off가 비정형(irregular) check node에서 미특성화.
3. 모델: 소거 확률 `ε` 입력, density evolution `xl = λ(1-ρ(1-x_{l-1}))`로 소거정정능력 기술, 복호임계값 `ε*`로 복원능력 정의.
4. 핵심기법1(random access repair): 임의 가용노드에서 수리, 평균 수리대역폭 `γ̄ = Σ dc,i(dc,i-1)/E`, check node 정규일 때 최소.
5. 핵심식: 비정수 차수 `d`에 대해 fractional phantom distribution으로 `ρ(x)` 완화(식1), `ε*=ε*min`일 때 최소대역폭 `γ̄_min = f(R)-1` 폐형식(식5).
6. 핵심기법2(ideal repair): 최소 degree parity-check 식을 선택해 수리, `γ̄(ideal)(λ,ρ)` 를 multiset 조합확률로 표현(식6~7).
7. 구현 트릭: 정수 차수를 fractional phantom distribution으로 실수화해 차수분포 제약 유지하며 연속 최적화.
8. 최적화: Differential Evolution(DE) 변형, 두 mutation 전략(DE/rand/1, DE/rand-to-best/1) 무작위 교대, epsilon-constraint로 단일목적화(식11), bisection으로 임계값 계산(Algorithm 1).
9. 결과: R=2/3, 3/4에서 ideal이 random 대비 우수, R=3/4에서 ε* 0.02 이상 개선, blocklength 2000 유한길이 BP 시뮬로 확인.
10. 한계: HW 미설계, BEC 소거채널·분산저장 수리대역폭 특화, NAND read/soft-decision/정정능력과 무관, 시뮬레이션(이론+몬테카를로)만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 차수분포(λ,ρ) 최적화 부호설계 | 대응 없음 | Prime ECC는 특정 QC-LDPC H-matrix 고정, 앙상블 차수분포 설계 경로 없음 |
| BEC density evolution / 복호임계값 | 대응 없음 | Prime ECC는 AWGN/RBER 기반 min-sum 복호, 소거채널 미대상 |
| DE 기반 차수분포 탐색 | 대응 없음 | 부호구성 오프라인 도구 영역, 코드베이스 매핑 없음 |
| 수리대역폭 metric | 대응 없음 | 분산저장 전용 개념, NAND ECC 무관 |

적용 가치: 낮음 — 분산저장 수리대역폭 특화 BEC 부호구성 이론이라 NAND binary QC-LDPC ECC의 부호설계·디코더·HW 어느 레이어에도 이식할 요소가 없음.
