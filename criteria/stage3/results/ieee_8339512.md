### B. 알고리즘 요약

1. 대상은 CPM 기반 QC-LDPC 블록부호(H는 m×n 배열의 N×N 순환치환행렬 CPM)와 이를 unwrapping한 시불변 SC-LDPC-CC로, 예제 rate `R=4/7`, girth `g=10,12`.
2. 문제: iterative BP에서 짧은 사이클(작은 girth)이 성능을 해치므로 girth를 키우되, M2M 등 저지연 응용을 위해 length(N)와 SC-CC의 syndrome former memory `mh`를 최소화해야 함.
3. QC-LDPC 사이클 존재조건: 길이 2k 사이클은 `Σ(p_{m_i,n_i} - p_{m_i,n_{i+1}}) ≡ 0 mod N` (식 2)로 판정.
4. 핵심 기법 - SMC(sequentially multiplied columns) 가정: exponent matrix `P = [0, P1, γ2⊗P1, ..., γ_{n-1}⊗P1]`로 base 열 `P1`에 계수 `γ_j`를 mod-N 곱해 열 생성(식 7).
5. SMC는 탐색 자유도를 `mn`에서 `m+n-4`로 축소해 girth 12까지 exhaustive 탐색을 가능케 함(식의 의미).
6. Proposition 1: 부분행렬 `[0 P1]`이 길이 10까지 strictly avoidable cycle이 없으면, 충분히 큰 N과 적절한 `γ_j` 선택으로 전체 부호도 girth ≥ 10 보장.
7. 확장: 동일 exponent matrix 표현으로 SC-LDPC-CC의 `mh`를 정의(P 원소 간 최대 차이)하고, min-max 정수계획 최적화 모델로 최소 `mh` 탐색.
8. 최적화 절차: Algorithm 1 greedy search가 `γ_j ∈ {γ_{j-1}+1,...,N-1}`에서 girth 조건을 만족하는 최소값을 순차 선택.
9. 결과: 기존 대비 lifting degree 비 `Θ_N`을 0.47까지(디코딩 latency 50%+ 감소), `Θ_{mh}`를 0.23까지(latency·복잡도 75%+ 감소) 낮춤. BER은 100 iteration BP/SW 시뮬에서 고전 부호 대비 손실 미미.
10. 한계: 디코더/HW 미설계, hard/soft 무관, 예제 rate가 4/7로 NAND ECC 고rate와 거리, error-floor 직접 보장 아님.
