### B. 알고리즘 요약

1. 대상: 802.11n/802.16e 표준의 QC-LDPC 코드(codeword length 648/1296/1944, rate 1/2~5/6), 부호check matrix `H`는 `Z×Z` circulant sub-block으로 구성되고 parity 부분 `Hp`가 dual-diagonal 구조(단일 weight-3 column `hp` + 하위 대각 identity blocks)를 가짐.
2. 풀려는 문제: Richardson 인코딩은 `p0 = (-ET⁻¹A+C)s` 직접해를 구해 인코딩(연산량 `n²`~`n³`급)하지만, 이보다 더 낮은 복잡도의 인코딩을 원함.
3. 핵심 가정: parity 부분의 weight-3 column의 순환 shift값을 모두 동일(0)하게 만들도록 H matrix `Hp`를 소폭 수정(`Fig.2`)해도 성능 열화가 실질적으로 무시할 수준이라 가정.
4. 핵심 기법: 첫 sub-block의 parity bit `p0,...,p(Z-1)`을 임의값(예: 전부 0)으로 두고, dual-diagonal 구조를 이용해 `x=As` 벡터와 결합, `(x_i,...,x_{i+Z-1})^T + Q·(p_i,...,p_{i+2Z-1})^T=0` 재귀식으로 나머지 parity bit를 순차적으로 결정.
5. 핵심 식: 마지막 sub-block 검증에서 `f=(x_{(m-1)Z},...,x_{mZ-1})^T+(p0,...,p_{Z-1})^T+(p_{(m-1)Z},...,p_{mZ-1})^T`(식 11)로 오류(불일치) 벡터를 구함 — 임의로 정한 초기 parity bit 가정이 틀렸던 위치를 식별.
6. 검증 벡터 `f`를 region Q 길이만큼 반복 이어붙인 벡터 `v=(f^T,...,f^T)^T`를 만들어 초기 parity bit들에 XOR(가산)해 한 번에 정정 — 별도 반복탐색 없이 벡터 가산으로 correction 완료.
7. 부수 트릭: region Q(bit-flip 대상)와 region U(비대상)로 H matrix를 3분할(A/Q/U)해 정정 연산 범위를 최소화, HW용 shift register 구조에 적합하다고 언급(구체 설계는 없음).
8. 최적화 절차 없음(GA/DE/NN 미사용), 순수 대수적 유도.
9. 결과: 모듈로-2 덧셈 횟수 비교(Table I)에서 codeword 1944, R=1/2 기준 제안기법 6561회 vs Richardson 방식 10368회로 연산량 감소. BLER 성능은 표준 H 대비 대체로 comparable하나 고rate·짧은 codeword에서 BLER 10⁻⁴대 error-floor 열화 관찰.
10. 한계: HW 미설계(구조 언급뿐), H matrix를 표준과 다르게 수정해야 하므로 표준 정합성 이슈, error-floor 열화가 code rate 증가·길이 감소 시 뚜렷해짐.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-diagonal 인코딩(부호 PCM 생성, 임의 parity bit 후 재귀+정정) | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코딩을 사용하므로 알고리즘 대체 후보이나, weight-3 column을 zero cyclic shift로 요구하는 H matrix 제약이 기존 PCM_b_enc 구조와 정합하는지 재검증 필요 |
| H matrix weight-3 column 수정(부호구조 변경) | `ecc_top.cpp` `Load_PCM()` | H-matrix 자체를 건드리는 변경이라 이식 난이도 높음(프로파일 §4 기준) |
| Richardson 효율적 인코딩과의 복잡도 비교 | 대응 없음 | Prime ECC는 이미 dual-diagonal 고정 구조 사용, Richardson 방식 자체는 미적용 |

> 적용 가치: 중간 — encoder 복잡도 절감 아이디어는 Prime ECC의 dual-diagonal 인코더와 개념적으로 맞닿아 있으나, H matrix 구조 변경을 요구해 재검증 부담이 있고 error-floor 트레이드오프 존재.
