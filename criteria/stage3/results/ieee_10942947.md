> 총평: SC-LDPC(ESRL) 전용 재구성 가능 HW 디코더로 rate-compatible 고처리량 아키텍처와 양자화 최적화(sign-preserving)를 제시하나, 코드 구조(spatially-coupled)와 목표(6G 무선, 대형 프레임)가 NAND 고정 QC-LDPC와 달라 부호설계 레이어는 이식성이 낮고, 양자화/CN 연산 최적화 아이디어만 부분 이식 가능.

### B. 알고리즘 요약

1. ESRL(edge-spreading Raptor-like) 코드는 uncoupled protomatrix `B`, edge-spreading matrix `T`, tail matrix `Q`로 정의되는 rate-compatible SC-LDPC 코드 계열이며, `B`를 `ω+1`개 서브행렬로 분할해 결합행렬 `B_SC`를 구성한다.
2. 5G-NR 대비 동일 복잡도에서 약 1dB 우수한 정정성능과 높은 병렬성을 목표로 하며, 기존 unrolled SC-LDPC HW는 특정 코드에 고정돼 rate 유연성이 부족한 문제를 해결하려 함.
3. Layered normalized min-sum(L-NMS) 디코딩을 채택: `tv = qv - rc,v`, `rc,v = sgn(·)·min(|tv'|)·α`, `qv = tv + rc,v` (식 1a~1c), `α`는 정규화 계수.
4. 양자화 전략 `(qQ, qR)`로 채널 LLR `Λ(yv)=sgn(yv)·min(round(|yv|/δ+0.5), 2^(qQ-1)-1)`을 정의(식 2), CN(check node) 연산에 sign-preserving 재배치를 적용해 정규화 `α<1` 곱셈으로 인한 R-message quantization range 낭비를 해소.
5. 곱셈 위치를 R-message에서 T-message 계산으로 이동시켜 R-message의 양자화 범위를 완전히 활용, 추가 곱셈기 비용이 들지만 `α`가 2의 거듭제곱이면 shift-and-add로 저비용 구현 가능.
6. HW 아키텍처는 SLME(semi-layered multi-engine) 디코딩으로 `W=L+ω`개의 병렬 서브디코더로 분해, 각 서브디코더는 partial row-parallel 구조(폴딩도 `S`, `Z'=Z/S`)를 사용해 라우팅 복잡도를 낮춤.
7. Gathering Routing Unit(GRU)이 Q-message를 `dcmax` 그룹으로 재구성 → QC-LDPC Shift Network(QSN)로 순환 시프트 → Row Computation Unit(RCU)이 식 1a~1c 수행 → Distribution Routing Unit(DRU)이 재분배.
8. 별도 학습 절차 없음(코드 자체는 [8] ESRL 논문에서 설계); 본 논문은 양자화 스킴과 HW 구현에 집중.
9. 결과: TSMC 16nm, K=17160에서 peak throughput 103.9Gbps, 면적효율 91.2Gbps/mm²; K=85800(Z=390)에서 최대 437.6Gbps; (6,4) 양자화로 float 대비 0.05dB 손실(5G-NR은 (7,5) 필요); 5G-NR 디코더[16] 대비 최대 2.76배 처리량.
10. 한계: 낮은 code rate에서는 row-parallel 구조 특성상 HW 효율(처리량/면적)이 5G-NR block-parallel 대비 낮고, 코드 자체(ESRL)는 별도 논문[8] 설계이며 본 논문은 HW 구현·양자화만 다룸(post-layout 아닌 synthesis 결과).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sign-preserving CN 연산(정규화 곱셈 재배치로 양자화 범위 활용) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 정규화 factor를 다루는 CN 연산부와 직접 대응, 양자화 비트 절감 아이디어 이식 가능성 있음 |
| 채널 LLR 양자화(식 2, `Λ(yv)`) | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 양자화 threshold 설계 개념 유사 |
| Row-parallel 아키텍처, GRU/QSN/DRU 라우팅 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC는 z=32 고정 row-parallel + Graph Thinning 구조로 이미 보유, 재구성가능 라우팅 개념은 참고 수준 |
| SC-LDPC ESRL 부호설계(B/T/Q 행렬, coupling) | `ecc_top.cpp` `Load_PCM()` | 코드 구조 자체가 QC-LDPC 전용 PCM과 달라 대응 없음에 가까움 |

> 적용 가치: 중간. 부호구조(SC-LDPC)와 대형 6G 프레임 타깃은 이식 난이도가 높으나, sign-preserving 양자화 재배치로 R-message 비트폭을 절감하는 CN 연산 최적화는 Prime ECC의 min-sum 양자화 테이블에 참고 가능.
