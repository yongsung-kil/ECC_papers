### B. 알고리즘 요약

1. 대상은 regular `(1024, 512)` LDPC (rate 1/2, row weight 10, column weight 6), BPSK, AWGN 및 Rayleigh fading 채널.
2. 문제: min-sum이 check node 메시지를 과대추정(overestimation)해 SP 대비 성능 손실 발생, 기존 OMS/NMS는 고정 보정계수라 최적이 아님.
3. 채널 모델은 표준 AWGN(`y = x + n`, zero-mean gaussian) 및 Rayleigh fading, intrinsic LLR은 채널값 기반.
4. 핵심 기법: layered(수직) 스케줄링으로 sub-iteration마다 한 layer의 check/variable node를 순차 갱신하며, offset factor `α`로 check node 메시지 크기를 감산 보정(`|R| - α`, floor 0).
5. 핵심 식 (13): `R_mn = α * ∏ sgn(...) * (...)` 형태로 CN 메시지에 offset을 곱/감산 적용 — 과대추정된 amplitude를 줄여 BP에 근사.
6. 확장: 첫 layer에 intrinsic 정보 절대값이 낮은(신뢰도 낮은) variable node 집합을 배치해 그 비트의 결정 신뢰도를 높이는 그룹화 전략.
7. 구현 디테일: variable node를 intrinsic 절대값 오름차순으로 정렬 후 K개 subgroup으로 나눠 layer 구성, 모든 CN 메시지 0 초기화.
8. 최적화: adaptive offset factor는 시뮬레이션으로 normalization look-up table을 만들어 결정(구체 절차 본문 미기술).
9. 결과: AWGN에서 BER 1e-2일 때 OMS 대비 0.08dB 이득, LOMS가 MS·OMS보다 우수하고 SP에 근접하나 여전히 SP에 미달.
10. 한계: HW 미설계, 복잡도·throughput 정량 없음, 이득이 매우 작고 offset table 산출 절차가 불명확.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| offset min-sum CN 갱신 (`|R|-α`) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC의 min-sum magnitude 보정과 동일 계열, offset 상수만 대체 |
| layered(수직) 스케줄링 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | Dual-Update와 유사한 부분 스케줄 개념, 재구성 필요 |
| adaptive offset factor 테이블 | `ecc_data.h` `PARAM_LLR`, `decoder.cpp` `Get_VNU_Table_Idx()` | 적응형 LLR 테이블과 취지 중복(iteration 구간별 보정) |
| variable node 신뢰도 그룹화 | 대응 없음 | Prime ECC에 intrinsic 정렬 기반 layer 그룹화 없음 |

적용 가치: **낮음** — offset min-sum과 iteration별 적응형 보정은 Prime ECC가 이미 보유했고, 논문 이득(0.08dB)이 미미하며 HW·복잡도 근거가 전무.
