### B. 알고리즘 요약

1. 대상은 dual-diagonal(DD) QC-LDPC 부호: exponent matrix `Edd = [El, Er]`, 좌측 `El`은 GCD 프레임워크 기반 공식, 우측 `Er`은 dual-diagonal 구조로 단순 encoding을 보장.
2. 기존 GCD 기반 방법들은 우측 서브행렬의 특정 위치값을 컴퓨터 탐색에 의존하거나 exponent matrix 행 수(J′)가 제한되는 문제가 있어, 탐색 없이 임의 row weight `L`에 적용 가능한 명시적 공식이 필요.
3. 채널모델은 AWGN + BPSK, SPA(sum-product) 디코딩 최대 50 iteration으로 시뮬레이션.
4. 핵심 기법: GCD 제약을 만족하는 수열 `S2=(s0,...,sJ-1)`을 찾을 때 계수 `si = αi·L + βi`에서 기존과 달리 `βi`에 음수를 허용하는 GNC(GCD+Negative Coefficient) 탐색 알고리즘 제안.
5. GCD 제약식 `(sc−sa)/gcd(sc−sa, sb−sa) ≥ L`이 모든 triple에서 성립하면 girth 정확히 8을 보장(`exponent matrix E = S2^T·S1`).
6. column weight `J=5,6,7`에 대해 각각 명시적 exponent matrix 공식(`E1, E2, E3`) 도출, 기존 하한보다 개선된 circulant size 하한(예: `(3L−1)(L−1)+1`) 제공.
7. 우측 DD 서브행렬은 기존 `Er(0)`(컴퓨터 탐색 필요)를 대각 flip으로 변형한 `E(r1)`(전부 명시적, 탐색 불필요)로 대체, Theorem 4로 4-cycle/6-cycle 없음을 증명.
8. 별도 학습절차 없음, 순수 조합론적 탐색+공식 유도(Algorithm 1, 정리 증명 기반).
9. 결과: `(5,10)` 새 DD 코드가 동일 length 기존 DD 코드 대비 8-cycle 수 `274×71` vs `349×71`로 감소, BER/BLER 성능 개선; `(6,L=7)` 코드는 5G BG2 기반 코드 대비 고SNR 영역에서 error-floor 없이 우월.
10. HW 미설계, 순수 이론+시뮬 수준이며 특정 column weight(5~7)·circulant size 조건에 국한된 구성법.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth-8 exponent matrix 명시적 공식(GNC) | ecc_top.cpp `Load_PCM()` | H-matrix/PCM 구조 자체를 재설계하는 부분이라 이식 난이도 높음(§4 표 기준) |
| dual-diagonal 우측 서브행렬(E(r1)) 및 단순 encoding | encoder.cpp `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코딩 보유(§3) — 개념 중복, 신규 기여는 좌측 exponent matrix 공식 부분 |
| Column weight 5~7 특정 구성 | ecc_top.cpp `Load_PCM()` | Prime ECC의 VN degree 17, N_b=129 구조와 직접 대응 안 됨 |
| SPA(sum-product) 디코딩 시뮬 | decoder.cpp `LDPC_Decoder()` | Prime ECC는 이미 min-sum(§3) 사용, SPA 자체는 이식 대상 아님(비교용 벤치마크일 뿐) |
```

> 적용 가치: `중간` — 부호설계(H-matrix) 층위 개선이라 이식 난이도는 높지만, girth-8·낮은 encoding 복잡도라는 목표는 Prime ECC의 고rate QC-LDPC 부호설계 방향과 부합.
