### B. 알고리즘 요약

1. AWGN 채널 기반 type-II hybrid ARQ 시스템에서 LDPC 코드를 ECC+EDC로 겸용해 rate-compatible(RC) 코드 시퀀스를 구성.
2. 기존 puncturing 기반 RC-LDPC는 고rate에서 성능 열화, extending 기반은 항상 최저 rate용 대형 디코더 HW가 필요해 모바일에 부적합함이 문제.
3. 채널은 AWGN, LLR은 `Lch(cl|yl) = 2yl/σ²`로 정의되는 표준 Log-BP 복호를 가정.
4. 마더코드(H, rate R0=0.5, n=1536)에 새 행을 추가해 하위 rate 코드를 생성하는 프레임워크 - 새 H'는 기존 H를 부분행렬로 포함해 항상 "고rate 디코더"로 저rate 코드도 복호 가능.
5. `r'H'^T=0`이면 `rH^T=0`도 만족하므로, 마더코드용 소형 디코더 하나로 전체 RC 시퀀스를 처리할 수 있다는 것이 핵심 아이디어.
6. PEG(progressive edge growth) 알고리즘을 변수노드뿐 아니라 새로 추가되는 체크노드에도 zigzag 패턴으로 확장 적용해 선형시간 인코딩이 가능한 근사 하삼각 구조 유지.
7. 새 체크노드의 degree는 기존 행 weight의 평균을 반올림해 결정, PEG로 이 새 체크노드와 데이터 부분집합 간 엣지를 연결.
8. 최적화 절차 없음(구성 규칙 기반), 시뮬레이션으로 rate 8/13~8/19 시퀀스 성능만 확인.
9. 결과: R0=0.5, n=1536 마더코드 기반 RC 시퀀스(8/13~8/19)가 AWGN에서 양호한 BER 성능을 보임(구체 수치는 그림 2에만 존재).
10. 한계: HW 미설계, 시뮬뿐, ARQ 피드백 채널(무손실 가정) 전제로 저장 채널에는 부적합, 성능 수치는 그림에만 있어 본문 서술 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| RC-LDPC 행 추가 구성법 (H-matrix 확장) | `ecc_top.cpp` `Load_PCM()` | Prime ECC는 고정 QC-LDPC H-matrix 사용, 가변 rate 확장 구조와 무관 |
| PEG 기반 마더코드/체크노드 구성 | 대응 없음 | irregular PEG 구성법으로 QC-LDPC 구조와 불일치 |
| type-II hybrid ARQ 재전송·패킷결합 | 대응 없음 | NAND는 재전송 개념이 없는 저장 채널 |
| Log-BP 표준 디코딩 | `decoder.cpp` `LDPC_Decoder()` | 표준 sum-product 언급뿐, Prime ECC는 이미 min-sum 채택 |
