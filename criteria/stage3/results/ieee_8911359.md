### B. 알고리즘 요약

1. Time-invariant SC-LDPC 부호(syndrome former memory `m`, constraint length `νs=(m+1)c`, γ×c 블록 `Hi`)를 대상으로, 기존 문헌([5],[6])의 최소 constraint length 부호와 동일한 degree distribution·girth·m을 유지하며 error floor만 개선.
2. 기존 [5],[6]은 girth만 통해 간접적으로 error floor를 다뤘고 실제로는 작은 trapping set·최소거리 부족으로 error floor가 높음.
3. 채널은 binary-input AWGN, 디코더는 floating-point sliding window belief propagation(box-plus 근사, 창크기 `W`, 최대 50 iteration/window).
4. Dominant LETS(leafless elementary trapping set) 구조를 `dpl`(dot/path/lollipop) 확장 기반 forest로 조직화하고, 유해도 순으로 정렬해 각 구조의 multiplicity를 순차적으로 최소화하는 exponent matrix 탐색(Algorithm 1).
5. 후보 exponent matrix 집합 `Φ`를 girth 유지하며 열 일부를 랜덤 수정해 생성 후, 각 iteration마다 가장 유해한 미방문 구조의 multiplicity가 큰 후보를 일부 비율만큼 제거(trimming)하여 최종 후보로 수렴.
6. 부수 요소로 최소거리 `dmin`도 함께 고려(예: C1은 dmin=14로 [5] 대비 개선), 단 dmin 크기 자체보다 LETS distribution이 error-floor를 더 잘 설명함을 확인(C2는 dmin이 C1보다 작지만 error floor 더 우수).
7. 학습/최적화 절차는 랜덤 열 수정 + 순차적 후보 trimming(GA/DE 아님, greedy 반복).
8. 결과: γ=3,c=6,m=7,g=8 코드에서 [5] 대비 BER curve 상 error floor 영역 대폭 개선(예: C1은 10^-7 부근까지 개선, C2는 (6,2)/(8,2) LETS 추가 제거로 더 우수), c=17/γ=4, g=10 등 다양한 파라미터에서도 일관된 개선; MATLAB 구현, 초기 후보 탐색 0.5~20시간, 최종 선별 40~289분 소요.
9. 한계: 시뮬레이션(floating-point BP)뿐이며 HW 미설계, SC-LDPC(convolutional) 구조 전용이라 QC-LDPC block code와 부호 형식이 근본적으로 다름, rate/length가 예제별로 상이.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC exponent matrix(trapping set 최소화) 설계 | 대응 없음 | Prime ECC는 QC-LDPC(block code) 전용이며 SC-LDPC(convolutional, sliding window) 구조 자체를 지원하지 않음 |
| dpl 기반 trapping set forest 탐색 아이디어 | `ecc_top.cpp` `Load_PCM()` | 트래핑셋 최소화라는 설계 철학만 유사 참고 가능, 실제 QC 부호 재구성 시 별도 재작업 필요 |
| sliding window BP 디코더(box-plus) | 대응 없음 | Prime ECC는 min-sum 고정 iteration 기반이며 sliding window 방식 미보유 |

> 적용 가치: 낮음 | 부호 계열(SC-LDPC vs QC-LDPC)과 디코더 아키텍처(sliding window vs 고정 iteration min-sum)가 모두 상이해 직접 이식 불가, trapping-set 우선순위 기반 successive minimization 개념만 참고 가치.
