### B. 알고리즘 요약

1. 대상 채널은 BSC/AWGN, 코드는 (N=155, K=64, Dmin=20) Tanner LDPC 코드로 시뮬레이션.
2. 문제의식: 결정론적 GDBF(gradient descent bit-flipping)는 trapping set에 갇히면 고정점에서 진동하며 복호 실패.
3. 핵심 가정: 하드웨어/알고리즘에 인위적 랜덤 노이즈를 섞으면 오히려 trapping set attraction을 탈출할 수 있다는 stochastic-resonance류 관찰.
4. 핵심 기법 1: PGDBF - flipping set `F(l)`의 각 비트를 확률 `p0`로만 실제 flip (`ε_n` uniform random과 비교).
5. 핵심 기법 2: NGDBF - 에너지 함수 `E_vn`에 가우시안 노이즈 `ε(l)~N(0,σp²)`를 더해 threshold `θ` 기준으로 flip 여부 결정.
6. 두 알고리즘 모두 5-bit trapping set 상태공간에서 Markov 전이확률(`Λ(S,S')`)로 탈출확률을 해석적으로 계산.
7. 확장 기법: DDS-PGDBF - 현재 iteration 에너지 대신 이전 iteration의 최대 에너지 `b(l-1)`를 flipping 기준으로 사용해 스케줄을 변형, 더 큰 state-space jump 유도.
8. 최적화 절차 없음(파라미터 `p0`, `σp`는 TS 상태공간 분석 및 시뮬레이션으로 수동 탐색).
9. 결과: DDS-PGDBF는 300 iteration 제한에서도 원래 PGDBF/GDBF 대비 큰 FER 개선을 보이며, 매우 큰 L(최대 10^8) 허용 시 MLD 한계에 근접. 평균 iteration 수는 조기종료로 a=0.03에서 16.8, a=0.02에서 4.9.
10. 한계: HW 미설계, 시뮬레이션(FER 곡선)에 국한, worst-case iteration이 매우 커질 수 있어 latency 예측 불가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GDBF/NA-GDBF (bit-flipping 알고리즘 전체) | 대응 없음 | Prime ECC는 min-sum 계열이며 BF 계열 알고리즘 자체는 미보유 |
| 노이즈 주입을 통한 trapping set 탈출 아이디어 | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프 | error-floor 대책으로 min-sum 루프에 유사 개념(랜덤 섭동) 이식 검토 가능하나 알고리즘 구조가 달라 재설계 필요 |
| DDS-PGDBF 스케줄링(이전 iteration 임계값 재사용) | 대응 없음 | BF 전용 스케줄로 min-sum 구조와 직접 대응 안 됨 |
| 조기종료(평균 iteration 감소 관찰) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기종료 개념은 유사하나 본 논문은 CRC가 아닌 코드워드 수렴 판정 |

> 적용 가치: 낮음. hard-decision BF 계열 특유의 노이즈 주입 기법으로, Prime ECC의 min-sum 기반 구조와 알고리즘 계열이 근본적으로 달라 직접 이식보다는 개념적 참고 수준.
