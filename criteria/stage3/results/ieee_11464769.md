> 총평: 무선 고이동성 채널(OTFS)을 위한 블록 마르코프 중첩전송·EXIT 최적화 논문으로, LDPC는 기존 5G 규격을 그대로 쓰고 시스템/변조 레벨 이득만 다뤄 NAND binary LDPC ECC 이식 관점에서 관련성이 없음.

### B. 알고리즘 요약

1. 시스템: OTFS 변조 + 5G LDPC(BG2, rate 1/2, n∈{512,1024,2080}) 부호화, 고이동성(150km/h) doubly-selective 채널, P개의 resolvable path.
2. 문제: 부호길이를 늘려도 resolvable path 수가 적은 채널에서는 부호이득이 한계에 도달(marginal gain), 재전송은 오버헤드·지연 유발.
3. 채널모델: DD 도메인에서 P개 독립 경로의 doubly-selective 채널, SPA 검출기 출력 MI가 non-ergodic하게 요동(`I_E,Det(X;Y)`의 PDF).
4. 핵심 기법: BMST 기반 SC-OTFS - 이전 블록 부호어 `v(t-1)`의 일부(`α`, superposition fraction)를 선택행렬 `S`로 골라 현재 블록에 XOR 중첩: `c(t) = v(t) ⊕ v(t-1)S`.
5. 의미: 이전 시점의 부호어 일부를 다음 블록에 실어 보내 깊은 페이딩을 겪은 블록도 이후 양호한 채널 조건의 블록 정보로 복구 가능(부호이득+블록간 다이버시티).
6. 수신단: 윈도우 크기 `d`의 sliding-window decoding(SWD), normal graph 상 forward/backward 메시지 패싱(OTFS→M→+→=→LDPC→=→S).
7. 최적화: 비에르고딕 OTFS 특성 때문에 검출기 출력 MI의 PDF를 추정·샘플링하여 EXIT 차트 초기값으로 사용, decoding threshold 기반으로 최적 `α` 도출.
8. 학습/최적화 절차: EXIT 차트 알고리즘(Algorithm 2) - PDF 샘플링→초기화→d-layer 메시지 패싱 반복→LDPC 노드에서 수렴 확인.
9. 결과: `α=0.6`, `d=4`, `L=100`에서 기존 5G LDPC-coded OTFS 대비 FER 10^-3 기준 3dB 이상 이득, n=512·P=2 SC-OTFS가 n=2080·P=4 기존 방식보다 우수.
10. 한계: 시뮬레이션만 수행(HW 미설계), BPSK·균일 전력 프로파일 가정, LDPC 부호 자체는 5G 규격 그대로 사용(부호설계 기여 없음).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-OTFS 블록간 XOR 중첩(BMST) | 대응 없음 | NAND 정적 채널에는 블록간 시간축 중첩 개념 자체가 불필요 |
| Sliding-window decoding(SWD) | 대응 없음 | 다중 시간블록 윈도우 복호로 단일 코드워드 복호 구조인 decoder.cpp와 무관 |
| EXIT 차트 기반 α 최적화 | 대응 없음 | OTFS 검출기 MI 분포 특화 설계 도구, 정적 NAND 채널 LLR 설계와 무관 |
| 5G LDPC(BG2) 부호 자체 | 대응 없음(참조 규격일 뿐) | 논문이 직접 설계한 부호가 아니라 기존 5G 표준 재사용 |

적용 가치: `낮음` — OTFS 무선 고이동성 채널 특화 시스템 레벨 기법으로 NAND의 정적 채널·binary QC-LDPC 복호/부호 설계와 접점이 없음.
