### B. 알고리즘 요약

1. 대상: 시불변 tail-biting LDPC 컨볼루션 부호(TB-LDPC-CC), rate `R=b/c=1/2`, 3개 frame size(1248/2496/4992), 4x4 MIMO 방송 시나리오.
2. 문제: 기존 LDPC-CC 디코더[3]는 연속 스트림만 처리해 packet 기반(802.11ac/ad, LTE-A) 종단 불가. LDPC-BC는 다중 rate/frame 지원 비용 큼.
3. tail-biting으로 컨볼루션 부호를 QC 블록부호화하여 code-rate 손실 없이 codeword 종단. state-space 변수로 초기상태 `S0` 계산, `I+A` 가역이면 `tN=2^n`에서 tail-biting 가능함을 `I+A^{tN}=(I+A)^{2n}`로 증명.
4. 핵심 기법: layered normalized min-sum 디코딩[9]을 변형한 스케줄링으로 flooding 대비 수렴속도 2배, 4 iteration이 기존 5 iteration과 동등 성능.
5. 부호구성: 정보/parity 대각 성분을 1로(fast encoding), parity part를 lower-triangular(tail-biting 보장), monomial 성분으로 단주기 억제, 같은 열 인접 계수차 >1로 메모리 충돌 회피, 6x6 QC-expansion으로 병렬화.
6. 아키텍처: 4 processor + 27 memory bank, 각 processor에 7 SVNU array(6 SVNU/array) + 6 CNU(1 pipeline stage), inter-processor·inter-row permutation network.
7. adaptive channel value addressing: 채널 LLR을 접근마다 적응적 시프트 저장하여 storage 30.77% 절감, 4-stage pipelining(R/π/S/CNU/π/W).
8. QC-expansion으로 메모리 접근 충돌 없는 병렬 디코딩 확보, two-port SRAM 사용.
9. 결과: UMC 90nm, 2.18mm2, 0.8V/305MHz에서 275mW·max throughput 3.66Gb/s·18.8pJ/bit/proc, 4 iter 시 정보 throughput 1.83Gb/s. BER은 AWGN에서 mother code (48,24,21,1) LDPC-CC에 근접.
10. 한계: 무선 MIMO 타깃, AWGN 시뮬 + 칩 실측이나 정정능력 자체 향상은 없음, non-terminated 대비 성능은 iteration 수 trade-off.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| layered normalized min-sum 스케줄링 | decoder.cpp LDPC_Decoder(), CNU_Update_New_Mag() | min-sum 계열이라 근접하나 Prime ECC는 dual-update 교대 스케줄 이미 보유 |
| adaptive channel value addressing / 메모리 | decoder.cpp VN_Cal_HD() 주변 + 채널 LLR 저장 | 메모리 절감 아이디어는 참고 가능하나 우리 pipeline 구조와 정합 필요 |
| QC-expansion 병렬화(충돌회피) | 대응 없음 | z=32 lifting·HCU 기반 우리 병렬구조와 다른 6x6 QC 확장 |
| tail-biting 부호구성 | 대응 없음 | LDPC-CC 종단 기법, 블록 QC-LDPC 부호구조와 무관 |
```

마지막에 한 문장으로 **적용 가치**: `낮음` - min-sum·메모리 스케줄링 참고점은 있으나 tail-biting LDPC-CC라는 부호구조와 무선 MIMO 타깃이 NAND binary QC-LDPC 블록부호 파이프라인과 달라 이식 가치가 낮다.
