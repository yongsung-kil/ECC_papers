### B. 알고리즘 요약

1. 대상: LDPC Convolutional Codes(LDPC-CC), rate `R=b/c`, semi-infinite 전치 패리티검사행렬(syndrome former) `H[T0,∞]`로 정의, syndrome former memory `ms`, time-invariant(`T=1`) 코드를 주로 다룸.
2. 문제: LDPC 디코더의 belief propagation 메시지 교환에 필요한 메모리/라우팅 네트워크 복잡도는 코드의 그래프 구조에 좌우되며, semi-random 그래프는 복잡한 라우팅을 요구.
3. 핵심 가정: 코드 그래프는 완전 병렬 처리 가능(각 iteration 내 CN/VN 연산에 데이터 의존성 없음)하다고 가정, tail-biting LDPC-CC(순환 wrap)도 포함.
4. 핵심 기법 1단: CN/VN 연산을 direct sum(⊕) 표기로 벡터화하여 `수평 병렬화`(같은 타입 노드 NΓ개를 한 사이클에 처리)와 `수직 병렬화`(다른 타입 노드 NΛ개를 동시 처리)를 정의.
5. 핵심 식: `C[(tk:tk+NΓ-1),(j:j+NΛ-1)](l) = ⊕ⁿ⁼ʲ...ⁿ⁺ᴺᴧ⁻¹ ⊕ᵗ⁼ᵗᵏ...ᵗᵏ⁺ᴺᴦ⁻¹ Cn(Vt,n(l))` — 수평-수직 결합 병렬 CN/VN 연산을 벡터ALU가 동시에 처리할 수 있음을 정식화.
6. 핵심 기법 2단: `Proposition 1/2`(연속 노드 간 엣지는 연속 노드로 매핑, 연결 노드 간 시간 거리 ≤ ms)를 이용해 셔플 네트워크를 barrel shifter로, 크로스바 스위치를 ms 범위 내 연결로 단순화.
7. 부수 트릭: lifted code(circulant 확장, 확장계수 X)에서는 NΓ가 X의 배수가 되도록 하여 셔플을 circulant 기반 barrel shifter로 단순화.
8. 확장 아키텍처: `Block Processing`(여러 디코딩 코어에 데이터 블록 분산, 처리량 NB배 증가)과 `Iteration Pipelining`(circular pipeline decoder를 IP개 코어로 파이프라인화)을 Proposition 2 기반으로 제시.
9. 결과: 별도 신규 수치 실험 없음(이전 연구 [2],[7],[13],[14] 실현 사례를 인용해 "수백 Mbit/s급, 저전력/저면적" 언급).
10. 한계: 순수 대수적 프레임워크/아키텍처 템플릿 제시 논문으로 자체 HW 합성·시뮬레이션·정정성능 결과 없음. SC-LDPC(convolutional) 전용이라 QC-LDPC block code에는 구조 변환 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN/VN 병렬 연산 벡터화(수평/수직 병렬화) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | Prime ECC는 이미 z=32 row 단위 병렬(HCU 포함)을 구현하므로 개념적으로 유사하나 SC-LDPC 대수 정식화라 직접 이식 불가 |
| 셔플/크로스바 기반 라우팅 아키텍처 | 대응 없음 | GT.cpp(Graph Thinning)의 MUX edge 선택과 목적은 유사하나 SC-LDPC 전용 barrel shifter/crossbar 구조로 실존 대응 없음 |
| Block Processing / Iteration Pipelining | `decoder.cpp` pipeline 6-stage | 파이프라인 개념은 유사하나 convolutional 코드의 순환 메모리 섹션 구조 특유 기법 |

> 적용 가치: 낮음 — SC-LDPC(convolutional) 코드 고유의 대수적 병렬화 프레임워크로, QC-LDPC block code 기반 Prime ECC 3.1의 고정 PCM 구조와 근본적으로 다르며 실측/합성 검증도 없음.
