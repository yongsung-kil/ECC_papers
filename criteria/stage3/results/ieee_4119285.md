### B. 알고리즘 요약

1. GLD(Generalized Low-Density) 코드는 length-Nc eBCH/BCH 코드를 component code로 사용해 Gallager의 LDPC 구성을 일반화한 부호로, `(Nc², 2, Nc)_Kc` 형태(예: 길이 4096~16384, rate 0.6350~0.9613)를 다룸. AWGN+BPSK 채널 가정.
2. 풀려는 문제: 기존 GLD 복호는 trellis-MAP(APP) 알고리즘 기반으로 복잡도가 높아, 그래프 기반 저복잡도 알고리즘과 수렴성 분석이 부재했음.
3. 핵심 가정: 반복복호 중 심볼/제약 노드 메시지 밀도가 대칭 Gaussian 분포를 따른다고 근사(Gaussian Approximation).
4. 핵심 기법: Chase-II 알고리즘(최소신뢰위치 LRP `p`개 기반 리스트 디코딩)과 LDPC의 message-passing(MP)을 결합한 Chase-MP 알고리즘. Constraint-node는 `μ_{m,n}^(l) = Ψ_CHS(λ 벡터)`로 Chase-II 기반 extrinsic 정보 계산.
5. 핵심 식: Symbol-node 갱신은 `λ_{n,m}^(l) = λ_n^(0) + Σ_{m'∈Mn\m} μ_{m',n}^(l)`로 표준 LDPC MP와 동일 구조지만 constraint-node 연산만 Chase-II로 대체.
6. 확장: Gaussian mean의 반복별 동역학 `m_λ^(l) = 2d_s·Q^{-1}(P_b^(l)/2) - (d_s-1)·m_λ^(l-1)` 유도, decoding threshold `σ*`를 정의해 수렴성 판단.
7. 부수 트릭: Chase 디코더의 weight factor=1/2, reliability factor=1 고정, LRP 수 `p=3,4,5`로 실험.
8. 최적화 절차 없음(해석적 Gaussian approximation 기반 threshold 계산).
9. 결과: (64²,2,64)57 GLD(길이4096, rate0.7932)에서 Shannon limit 대비 약 1.1dB, (128²,2,128)120(길이16384, rate0.8789)에서 약 0.9dB 이내로 수렴, trellis-MAP과 유사 BER이나 복잡도는 더 낮음(정성적 서술).
10. 한계: HW 미설계, 시뮬레이션(이론적 threshold 분석 포함)뿐이며 QC-LDPC가 아닌 eBCH-component 대수 코드 기반 구조라 Prime ECC의 QC-LDPC min-sum 디코더에 직접 이식하기 어려움.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Chase-MP 전체 반복복호 구조 | `decoder.cpp` `LDPC_Decoder()` | symbol-node 갱신식은 표준 MP와 유사하나 constraint-node가 Chase-II 리스트디코딩 기반이라 min-sum CNU와 근본적으로 다름 |
| Constraint-node(Chase-II) 연산 | 대응 없음 | Prime ECC의 `CNU_Update_New_Mag()`는 min-sum이며 리스트 디코딩(Chase) 알고리즘은 미보유 |
| Gaussian Approximation 기반 threshold/수렴 분석 | 대응 없음 | 코드 자체가 아닌 이론적 분석 도구로 시뮬레이터에 대응 모듈 없음 |

> 적용 가치: 낮음 — eBCH-component GLD 코드 및 Chase-II 기반 constraint-node 복호는 Prime ECC의 QC-LDPC min-sum 구조와 부호·알고리즘이 모두 달라 직접 이식이 어려움.
