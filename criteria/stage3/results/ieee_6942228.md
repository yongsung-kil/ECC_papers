### B. 알고리즘 요약

1. 대상은 LTE/WiMAX 등 무선 통신 표준의 turbo 코드(SBTC/DBTC)와 LDPC 코드(WiMAX rate-1/2 QC-LDPC)를 하나의 하드웨어 커널로 번갈아 복호하는 VLSI SISO(soft-input soft-output) 설계.
2. 문제의식: 기존 멀티모드 SISO 커널은 LDPC를 지원하지 않거나, turbo/LDPC를 결합해도 radix-2 구조에 머물러 DBTC 지원이 제한적이었음. Sum-product 알고리즘(SPA)은 EML-MAP과 연산 형태가 달라 결합 불가능.
3. 핵심 가정: LDPC 복호는 표준 SPA/MSA 대신 radix-2 FBA(forward-backward algorithm, TPMP 기반)를 radix-4로 확장해 사용 — turbo의 forward/backward recursion(`α`, `β`)과 연산 유형(MAX ↔ MIN, 부호 XOR)이 유사해지도록 재정식화함.
4. 핵심 기법: check node update `Lr_{c→b}(u_b) = sgn(·) × MIN(·)`(식 7), forward/backward LLR `Lα`, `Lβ`(식 9~10)를 radix-4 단위로 재귀 계산 — turbo의 ACS(add-compare-select) 유닛을 LDPC의 min-sum 계열 연산에도 재사용.
5. 식(6)~(11)의 의미: bit node update `L_Q^{(i+1)}(u_b) = L_c(u_b) + Σ Lr_{m→b}`는 사실상 min-sum LDPC 복호의 메시지 갱신과 동일 구조이며, 이를 turbo의 α/β 재귀 연산 유닛(MFRP)과 하드웨어 상 공유.
6. 핵심 기법 2단: 3개의 재귀처리요소(RP; MFRP-α, MFRP-β, MFRP-βd)로 구성된 커널이 모드 신호(L/T, S/D)에 따라 turbo 또는 LDPC 연산 경로로 전환. LDPC 모드에서는 ACSU1/ACSU2와 sign block만 활성화.
7. 구현 디테일: word length는 고수준 시뮬레이션 기반 결정(Table III), saturation 유닛으로 오버플로 방지. WiMAX rate-1/2 QC-LDPC의 base matrix expansion factor `z` 단위로 파이프라인 스케줄링(row당 1z 사이클에 4개 check node update, 12 column bit node update).
8. 별도 학습/최적화 절차 없음(구조적 HW 설계, DE/GA 미사용).
9. 결과: 결합 커널이 미결합 대비 면적 21.8% 절감(0.25mm² vs 0.31mm², TSMC 90nm), LDPC 모드 BER 손실 0.1dB 미만(BER 10⁻⁵ 기준), throughput 800.2Mb/s(LDPC 모드, iteration 미포함), NAE(정규화 면적효율) LDPC모드 10.65 bits/mm².
10. 한계: WiMAX rate-1/2 QC-LDPC(soft 값 사용, radix-4 FBA/TPMP)로 한정된 통신 표준 검증. Turbo와의 자원 공유가 핵심 기여이므로 LDPC 단독 최적화가 아니며, NAND의 고rate binary LDPC·min-sum 계열과는 알고리즘·목적이 다름.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| radix-4 FBA(TPMP) 기반 LDPC 복호 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | Prime ECC는 min-sum 기반 message passing이며 FBA/TPMP 계열이 아니어서 알고리즘 자체는 직접 대응 안 됨(연산 형태만 유사) |
| check node MIN 연산 (radix-4 ACSU) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min 연산이라는 개념은 공통되나 turbo 겸용 구조라 HW 유닛 자체를 그대로 이식하기 어려움 |
| QC-LDPC base matrix expansion factor 단위 파이프라인 | 디코더 HW 모델 (z=32, HCU, GT) | Prime ECC도 z_sb=32 병렬단위를 쓰나, 본 논문은 turbo와의 자원공유가 핵심이라 구조 정합 난이도 중~높음 |

적용 가치: **낮음** — Turbo/LDPC 겸용 통신용 SISO 커널로, 복호 알고리즘 계열(TPMP/FBA)이 Prime ECC의 min-sum과 다르고 설계 목적(멀티표준 자원공유)이 NAND 단일 LDPC ECC와 부합하지 않음.
