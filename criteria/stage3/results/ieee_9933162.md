> 총평: NAND LDPC read의 read-retry 횟수를 줄이려 error-prone cell 비율로 최종 read level을 예측하고 서브레벨 fine-grained read를 도입(PreLDPC). Disksim 시뮬로 평균 30% read latency 감소. 디코더/부호가 아닌 read-level 스케줄링 정책이라 Prime ECC 코어 이식성은 낮음.

### B. 알고리즘 요약

1. 대상: high-density TLC/QLC NAND flash의 LDPC read. 부호 rate/length 명시 없음, 페이지 4KB, TLC 7-level(ER,P1~P7).
2. 문제: 셀 상태 증가로 read voltage가 많아져 read latency 증가; RBER 사전 미지라 read-retry로 read level을 점진 증가시키는데 불필요한 재시도가 많음.
3. 관찰: 인접 상태 사이 error-prone area에 놓인 셀 비율(`error-prone cell ratio`)이 성공 디코딩에 필요한 최종 read level과 비례.
4. 기법 1 (Read Level Prediction): Level 1 + 다음 서브레벨 2회 센싱으로 error-prone cell 수를 세고, `corr-table`(비율 구간↔read level)을 룩업해 최종 read level을 사전 예측 → 불필요한 read-retry 회피.
5. 대표 영역: P6~P7 사이 영역을 전체 error-prone area 대표로 선택해 정보 수집 가속.
6. 기법 2 (Fine-grained Reading): 한 read level을 서브레벨(Level 1.25/1.5/1.75)로 분할, 인접 레벨 정정능력 사이 RBER 데이터에 대해 read voltage를 1개씩만 추가하는 미세 스텝 read.
7. PreLDPC 통합: (1) Level 1+서브레벨 read → (2) 비율로 read level 예측 후 read → (3) fine-grained read 진행.
8. 학습/최적화: corr-table은 5000 upper page(4KB) 모의로 비율↔레벨 통계를 사전 구성.
9. 결과: Disksim+SSD extension(64GB 3D TLC), MSR 5개 workload. Baseline 대비 PreLDPC가 P/E 4000·retention 1개월에서 평균 read latency 30% 감소.
10. 한계: WIP 2쪽, decoder/부호설계 미개선, 시뮬(behavioral) 검증뿐, HW 미설계, corr-table 재보정·정확도 편차 미평가.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| read level 예측(error-prone ratio→level) | 대응 없음 | read-retry 스케줄링은 SSD 컨트롤러 read 정책, ECC 디코더 외부 |
| fine-grained 서브레벨 read | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | 간접: soft-read threshold 설정과 개념적 인접하나 Prime ECC는 read voltage 스케줄러를 갖지 않음 |
| corr-table(ratio↔level) | 대응 없음 | 논문 고유 룩업테이블, 부호/디코더 구조와 무관 |

적용 가치: **낮음** — read-retry 횟수·read level 스케줄링을 줄이는 flash read 정책 논문으로, 서브레벨 soft-read 아이디어가 `channel.cpp`의 read offset/LLR threshold와 개념적으로만 닿을 뿐 디코더·부호설계 개선이 없어 Prime ECC 3.1 이식 가치가 낮음.
