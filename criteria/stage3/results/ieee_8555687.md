### B. 알고리즘 요약

1. 대상은 3GPP NR QC-LDPC(base matrix 5×27, 46×68, `Z=384`, `L=8448`), layered BP 복호, min/max iteration 2/12.
2. 문제: 전통 CRC 조기종료는 iteration 끝에서 CRC를 계산하므로 CRC latency만큼 불필요한 복호가 발생.
3. 모델: 정보비트에 CRC 부착(`G(x)=x²⁴+x²³+x⁶+x⁵+x+1`), layered 복호에서 각 layer가 base matrix 한 행에 대응.
4. 핵심기법: CRC의 선형성으로 `A(x)`를 `L/Z`개 비중첩 segment로 분해, `C[A(x)]=ΣC[C[Aj(x)]C[x^...]]`로 부분 CRC를 누적(Lemma 1,2).
5. Lemma 3: `gD=1`이면 `C[B(x)]=0 ⇔ C[A(x)]=0` — segment 기반 `C[B(x)]`로 종료 판정 가능, LUT 메모리를 factor Z만큼 절감.
6. 핵심기법 2단: 이후 layer에서 갱신 안 되는 컬럼집합 `Λj=(Φj∩...∩Φ_{M-1})`이 비면 해당 segment `Pk` 계산을 iteration 종료 전 조기 착수(Lemma 4).
7. 구현 디테일: CRC 병렬도 R비트, `W=Z/R` 클록마다 새 Pk 처리, 마지막 갱신시각 정렬 후 스케줄(식17,18), base matrix 불변이라 오프라인 계산.
8. 최적화: 별도 학습 없음, controller가 Pk 계산 시점을 결정하는 결정론적 스케줄.
9. 결과: BLER는 전통 스킴과 거의 동일, avg iteration은 64QAM 5×27에서 PCE 대비 14.5%·전통 CRC 대비 4.5%, QPSK 46×68에서 PCE 대비 11.2%·전통 CRC 대비 1.4% 감소.
10. 한계: 정정능력 무개선(순수 종료 스킴), gatecount/throughput 절대수치 없음, controller·LUT 오버헤드 발생, NR/AWGN 검증(NAND 직접 아님).
