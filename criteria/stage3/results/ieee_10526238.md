### B. 알고리즘 요약 (10줄 내외)

1. 대상은 이진 LDPC 부호(H matrix, VN/CN Tanner graph), BPSK+AWGN(BIOS) 채널에서 ADMM 기반 penalized LP decoding 문제를 다룸.
2. 기존 ADMM decoding은 1차(first-order) 방법이라 BP 대비 수렴 속도가 느린 문제를 해결하고자 함.
3. LLR 벡터 `γ`와 quadratic penalty `-α/2‖x-0.5‖²`를 포함한 penalized LP 목적함수(식 1)를 최소화하는 문제로 디코딩을 정식화.
4. 핵심 기법 1단: 안정화 반복열 `e^k`, `u_j^k`(지수가중평균, 식 4a/4b)를 도입해 원 변수 `x`, `z_j`를 이 안정값 근처로 당기는 quadratic proximal term(`(ηx/2)‖x-e^k‖²` 등, 식 5)을 augmented Lagrangian에 추가한 Pro-ADMM 프레임워크(Algorithm 1) 제안.
5. 식(6)은 `x_i` 업데이트를 [0,1] 구간 projection 형태로 closed-form 유도한 것으로, proximal term이 반영되어 iterate 발산을 억제.
6. 핵심 기법 2단: Pro-ADMM을 CN 단위 message-passing(Li→j, Lj→i) 형태로 재해석하고 horizontal layered(HL) 스케줄링을 적용한 Pro-ADMM-HL(Algorithm 2) 제안, over-relaxation 계수 `ρ=1.9` 사용.
7. 조기종료: hard decision 후 syndrome check(`Hĉ=0`)로 매 CN 처리마다 유효 코드워드 판정 시 종료.
8. 최적화 절차 없음(grid search로 하이퍼파라미터 `µ=5, α=2, ηx=6, ηz=1, β=0.9`만 수동 선정).
9. 결과: Margulis(2640,1320), 802.16e(576,288), 802.11n(648,432) 3개 코드에서 FER과 평균 iteration 수 모두 기존 ADMM-QP/VNP/ℓp-box/HL/G-BADMM/BP 대비 개선(구체 수치는 그림에만 존재, 본문 서술 없음).
10. 한계: 하드웨어 미설계, 시뮬레이션 결과만 제시, complexity는 code length에 선형(O(n)/O(m) 추가)이라는 이론적 분석뿐 실측 gate/throughput 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| ADMM penalized/HL 디코딩 알고리즘 전체 | 대응 없음 | Prime ECC는 min-sum 기반이며 ADMM/LP decoding 계열 미보유 |
| 수렴 가속 proximal term (질적으로 iteration 감소 목적) | decoder.cpp LDPC_Decoder() | 목적(반복수 감소)만 유사, 구현 방식(min-sum CNU/VNU)과 무관 |
| HL(horizontal layered) 스케줄링 | decoder.cpp LDPC_Decoder(), Dual-Update 옵션 | 레이어드 스케줄이라는 개념만 유사, Prime ECC는 parity even/odd 교대 방식으로 별개 구현 |
| 조기종료(syndrome check) | partial_CRC.cpp partial_crc_HW_T4(), full_CRC.cpp | 조기종료 목적은 동일하나 논문은 단순 syndrome, Prime ECC는 CRC 기반으로 메커니즘 다름 |
```

적용 가치: `낮음` - ADMM 계열 디코더는 Prime ECC의 min-sum 기반 아키텍처와 근본적으로 다른 알고리즘 패러다임이라 모듈 단위 이식이 아닌 전면 재설계가 필요함.
