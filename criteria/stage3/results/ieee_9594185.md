### B. 알고리즘 요약

1. 시스템: spatially coupled serially concatenated codes(SC-SCC), RSC(1,5/7) 내·외부 인코더+인터리버, 전체 rate `R=1/3`(펑처링), block `K=32~256`, coupling memory `m`.
2. 문제: 고전 window decoding(WD)은 최소 block 크기 또는 큰 iteration을 강제해 완전 파이프라인 구현·병렬화를 제약(특히 B5G Tbps 무선).
3. 모델: SC-SCC를 compact graph로 표현, inner/outer trellis를 BCJR의 log 도메인 근사인 max-Log-MAP(MLM)로 복호, 채널 LLR 교환.
4. 핵심 기법(JWD): window를 1블록이 아니라 `∆=(W/Wref)·K`만큼 점프시켜 window당 iteration `IJWD=Ieff/Wref`를 K·W와 독립화.
5. 핵심 식 의미: `Ieff=W·IWD`(식4)로 복잡도 고정, JWD는 작은 K에서도 `IWD<1`로 붕괴하지 않고 동일 IJWD 유지 가능.
6. 확장: coupling memory `m=W/2-1`, latency `L=K·2(m+1)`로 block과 coupling을 고정 latency 내에서 trade-off.
7. HW 트릭: iteration-unrolled UXMAP 파이프라인에 inner(2K)·outer(K) 두 종 HI 스테이지, radix-4 MLM 커널(BMU/RU/SOU) 피드백 연결로 spatial coupling 실현.
8. 최적화: `nHI=2·Ieff` 스테이지 수를 Ieff 축소로 줄여 면적↓, 성능-면적 trade-off 조정.
9. 결과: JWD가 고전 WD 대비 0.5~0.9dB 개선·error floor를 10^-3→10^-6 이하로 저감, uncoupled SCC 대비 0.2~0.8dB 우위. 12nm PnR 면적(K=256, Ieff=16 시 26.66mm²), throughput 최대 204.8Gbps(K=256).
10. 한계: turbo-like SC-SCC 전용(LDPC 아님), 무선 B5G 타깃, 면적/throughput은 합성 추정(AFIFO 미포함), NAND 적용 근거 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| JWD 윈도 스케줄링(SC-SCC) | 대응 없음 | Prime ECC는 block QC-LDPC 반복복호이며 sliding-window·spatial coupling 구조 없음 |
| BCJR/max-Log-MAP 트렐리스 복호 | 대응 없음 | min-sum LDPC 복호와 알고리즘 계열 자체가 다름(turbo/trellis) |
| 완전 파이프라인 UXMAP HW 아키텍처 | [decoder.cpp](../Prime_ECC_3.1_Claude) `LDPC_Decoder()` (개념적, 간접) | pipeline 아이디어만 유사, Prime ECC는 6-stage QC-LDPC 전용 파이프라인이라 직접 대응 없음 |

적용 가치: **낮음** — turbo-like SC-SCC의 BCJR/window decoding 전용 HW 스케줄링으로, binary QC-LDPC min-sum 디코더에 이식할 알고리즘 요소가 없음. 무선 Tbps 타깃이라 NAND ECC와도 무관.
