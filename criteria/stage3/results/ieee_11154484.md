### B. 알고리즘 요약

1. 대상은 polar code `C(N,κ)` (Arikan 극화 변환 기반)로, 5G 표준 규격의 CRC-aided SCL(Successive Cancellation List) 디코딩이 기준.
2. 문제: SCL 디코딩은 list size L을 키우면 성능이 좋아지지만 HW 면적이 초선형으로 증가(L=8→16 시 면적 5배 이상), 고정 list size에서 성능을 높이는 것이 목표.
3. 채널은 BI-AWGN(binary-input AWGN)을 가정, 기존 Automorphism Ensemble Decoding(AED)은 특정하게 설계된 polar code에만 적용 가능하다는 제약이 있음.
4. 핵심 기법: pre-transformation(PT)을 encoding/decoding/decision-genie 참여 여부에 따라 PT-A/PT-B/PT-C 3종으로 분류하는 통합 프레임워크 제시, 이 중 디코딩에만 관여하는 신규 PT-C로 polar subcode를 생성해 LDPC용으로 제안된 바 있던 Subcode Ensemble Decoding(ScED)을 polar code에 확장.
5. PT는 아핀변환 `T(u_p) = u_p·A + b` (`A`는 상삼각행렬, `b`는 오프셋)로 정의되며, target bit들이 dynamic frozen bit가 되어 원 코드의 부분집합(subcode)을 형성; `M`개 subcode가 원 코드를 합집합으로 covering(`∪C_i = C`)하도록 앙상블 구성.
6. Theorem 1: 단일 SC 디코딩이 정답을 맞히면, 해당 코드워드를 포함하는 subcode의 SC 디코딩도 동일한 정답을 낸다는 것을 증명해 ScED의 평균 성능이 SC 이상임을 보장.
7. PT-C 선택 절차: URP(undecodable received pattern)를 다수 샘플링해 depth `d_p`별 디코딩 성공률을 분석(depth=2가 효율적), 후보 r개 중 URP를 최대로 커버하는 M개 부분집합을 그리디 방식으로 선택(최적화, DE 아님).
8. 최종 결정은 ML-in-the-list(각 subcode 디코딩 결과 중 코드에 속하고 likelihood가 최대인 것 선택), path metric 비교로 subcode 추정치가 stand-alone 추정치보다 우수할 때만 채택.
9. 결과: 5G polar code `C(64,32)`(CRC-6)에서 ScED-2-SCL-L이 SCL-L 대비 0.1~0.25dB 개선, list size를 2배로 늘린 SCL-2L과 동등 성능을 절반 list size로 달성.
10. 한계: HW 구현·게이트카운트 없음, 순수 알고리즘/시뮬레이션이며 SCL(리스트 디코딩) 대상 분석은 향후 과제로 남김(SC 대상만 정리 증명).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Polar SC/SCL 디코딩 및 subcode 앙상블 전체 | 대응 없음 | Prime ECC는 QC-LDPC min-sum 메시지패싱 구조이며 polar code의 순차(successive) 디코딩·list 구조 미보유 |
| 앙상블 병렬 경로를 통한 latency 개선 아이디어 | 대응 없음 | GC/QC-LDPC의 반복 디코딩과 polar SC의 순차 디코딩은 병렬화 개념 자체가 상이 |

> 적용 가치: 낮음 — polar code 전용 pre-transformation/subcode 프레임워크로 QC-LDPC 기반 Prime ECC와 부호·디코딩 알고리즘 계열이 근본적으로 달라 이식 불가.
