### B. 알고리즘 요약

1. `(N,K)` LDPC mother code를 shortening `s`bit + puncturing `p`bit 하여 `(N-s-p, K-s)` daughter code 생성, 수신은 mother code용 동일 BP 디코더 사용(shortening LLR=+inf, puncturing LLR=0).
2. 문제: 다양한 `(s,p)` daughter code 성능을 매번 시뮬하지 않고 mother 성능에서 예측.
3. 모델: 채널 품질 척도로 코드비트-LLR 간 평균 상호정보 `I_avg` 사용, 채널은 BEC와 BI-AWGNC.
4. 핵심가정: 동일 WER를 내는 서로 다른 채널은 `I_avg` 값이 매우 좁게 몰린다(기존 실험 관찰 [8][9]).
5. 핵심식: daughter가 `WER0`를 내는 MI `Id = (I0 - alpha_s)/(1 - alpha_s - alpha_p)` (`alpha_s=s/N`, `alpha_p=p/N`, `I0`=mother가 `WER0` 내는 MI).
6. 재표현: `Id = Rd + (I0-R0)/(1-alpha_d)` (`alpha_d=alpha_s+alpha_p`) - capacity gap `I0-R0`가 puncturing/shortening 많을수록 커짐.
7. 특수형: shortening만이면 `(Is,Rs)`가 `(1,1)` 지나는 직선, puncturing만이면 원점 지나는 직선 `Ip=(I0/R0)Rp` ([7] 결과 일반화).
8. 검증대상: G.hn·DOCSIS 표준 irregular LDPC (`N=1120~5940`), variable/fixed 패턴 모두, MC 시뮬로 `WER=1e-3` 지점 비교.
9. 결과: 예측-시뮬 오차 BI-AWGNC ~0.1dB, BEC ~0.007 erasure, MI 직선 이탈 <0.032bit로 waterfall 예측 정확.
10. 한계: error-floor는 예측 불가(예: `LDPC(1120,840)` `beta_p=0.7`에서 예측 floor 9e-2 vs 실측 3e-4로 큰 차이), HW·디코더·부호구성 변경 없음.
