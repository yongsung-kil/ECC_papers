### B. 알고리즘 요약

1. 시스템: DNA 저장, 데이터를 ~150 base(payload 84bp) oligo로 인코딩·합성·Illumina 시퀀싱, 채널은 Poisson 샘플링 erasure + substitution(~0.4%) + indel(del~0.85%, ins~0.05%).
2. 문제: 기존은 short inner code + erasure outer code 분리(RS/Fountain)라 짧은 블록 한계·consensus 의존으로 비최적.
3. 모델: 단순화 시 `BSC(ε)` + erasure, `cw`(write cost), `cr`(read cost), `λ=cr/cw`인 Poisson, 무카피 확률 `e^-λ`로 erasure capacity `1-e^-λ`.
4. 핵심 기법: 분리 프레임워크를 버리고 **단일 대블록(256K bit) regular LDPC** 하나로 erasure+error 동시 정정.
5. 핵심 식 의미: `cr ≥ cw·ln(cw/(cw-1))` 트레이드오프로 `cw`↑ 시 중복 read 감소 → `cr`↓, 대블록 부호가 이 bound에 근접.
6. 확장: index에 BCH(2bit 정정)+pseudorandom permutation, oligo 중앙에 sync marker(AGT)로 indel을 substitution/erasure로 변환.
7. 구현 트릭: MSA(Kalign2)로 index별 read 정렬 후 위치별 `(k0,k1)` 카운트를 `LLR=(k0-k1)ln((1-ε)/ε)`(ε=4%)로 변환해 LDPC 연판정 복호.
8. 학습/최적화: 없음(기성 MacKay/Neal LDPC-codes 라이브러리, redundancy 파라미터만 조절).
9. 결과: 실데이터 9회 실험에서 SOTA 대비 reading cost 30~40% 감소(예: 동등 write cost에서 40%↓), redundancy↓ 시 write↓·read↑ 확인.
10. 한계: HW 미설계(Python/C++ SW), NAND 무관, regular LDPC 기성품 그대로라 부호설계 신규성 없음, indel은 휴리스틱 처리.
