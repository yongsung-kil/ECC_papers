### B. 알고리즘 요약

1. 시스템은 PON/NG-EPON FEC용 long QC-LDPC (`(N,K)` base matrix `W`, `Z×Z` circulant, code rate `>0.8`), BPSK/AWGN.
2. 문제: girth만 키우거나 flat ACE spectrum만 추구하는 기존법은 long QC-LDPC에서 poor-connectivity 단주기(stopping set)를 남겨 error floor 유발.
3. 핵심지표: cycle의 ACE `η(a)=Σ(d(sj)-2)` (연결성), ACE spectrum `η(W)`(길이별 최소 ACE), number spectrum `τ(W)`.
4. 핵심기법(EEA): base matrix `W`의 non-zero shift값 `wr,b`를 하나씩 교환(edge exchange)하며 girth↑, ACE spectrum 원소↑, number spectrum 원소↓를 순차 최적화.
5. 핵심 아이디어: 모든 단주기가 똑같이 해롭지 않다 - low ACE(poor connectivity) 단주기만 선택적으로 제거해 error floor를 낮춘다.
6. 부수 트릭: QC 구조 활용해 `c_{(i-1)Z+1}` check node에서 확장된 partial cycle만 BFS로 탐색, `|A2l|=(|A2l'|×Z)/2l`로 전체 cycle 수 산출(탐색량 축소).
7. code length·rate·degree 분포는 zero 원소를 건드리지 않아 불변 유지.
8. 학습/최적화: Algorithm 1이 `metrics()`/`cmp()`로 현 `W`와 최적 `W` 비교, 개선 없을 때까지 반복.
9. 결과: 3개 코드에서 error floor 최대 2 order 개선, Code1은 3.9dB에서 error floor `8.16e-13` 이하, PMP 알고리즘 대비 우수, post-FEC BER `<10^-12`(PON 요구) 충족.
10. 한계: 순수 시뮬(BPSK/AWGN), HW 미설계, waterfall 개선은 소폭, 부호설계(오프라인) 도구로 디코더/HW 변경 아님.
