### B. 알고리즘 요약

1. 시스템: 802.11n 기반 QC-LDPC, rate 1/2·2/3·3/4·5/6, codeword 648/1296/1944 bit, encoder+AWGN 채널+decoder 3단, Simulink/System Generator로 FPGA(Virtex-6) 구현.
2. 문제: 큰 인코딩 비트 스트림 처리 시 throughput·복호 bit rate가 낮아 저복잡도 디코더 필요.
3. 모델: 인코딩은 `H*cT=0` 만족하는 parity bit 부가, 예제로 324 message bit → 648 encoded bit.
4. 기법: CNU/VNU를 min-sum algorithm으로 row/column processing 수행해 slice 자원 최소화.
5. 핵심식: `H*cT=0` (systematic + n-m parity), Navbits/NCBPS 기반 encoding step으로 available bit 계산.
6. 확장: 없음(단일 min-sum 구조).
7. 트릭: node message 저장에 multiplexed storage structure를 사용해 FPGA 자원 최소화, 채널은 4-bit magnitude frame 전송.
8. 최적화: 없음(학습/DE/GA 등 없음).
9. 결과: Virtex-6(6VLX760)에서 slice reg 5534, LUT 61351, Fmax 188MHz, throughput 12Gbps, dynamic power 4.448uW(수치 신뢰성 의심), 오류 없는 시뮬만 제시.
10. 한계: BER·정정능력·iteration 수렴 결과 없음, 서술 불명확·오탈자 다수, 표준 min-sum 교과서 재현 수준.
