### B. 알고리즘 요약

1. GC-LDPC(글로벌 CN이 여러 로컬 LDPC를 묶는 구조)의 글로벌부에 `tail-biting` 순환 구조를 도입해 글로벌 CN degree를 낮춘 TB-GC-LDPC를 정의.
2. 문제: 기존 GC-LDPC는 글로벌 CN의 높은 연결도로 고차 CN·짧은 cycle이 많고, field 크기가 CPM/블록길이/degree를 제약해 code length 유연성이 부족.
3. 방법1 REM(replicate, extract, mask): `GF(q)`상 2×2 SM-constrained 모mother matrix `BW`를 복제·추출 후 tail-biting 마스킹 `Bmask`로 글로벌부 RC-constraint를 보장, code length 유연성 확보.
4. 핵심 조건 `⟨t'n'-1, r n'⟩ ≥ (ms+1)n'-1` 등(식12)으로 `(q-1)`-fold dispersion 후 girth≥6 QC-LDPC를 확정 (field 재구축 없이 length 확장).
5. 방법2 packing: Bose-BIBD 유도 special-packing(S-packing, 식15)의 incidence matrix로 또 다른 TB-GC-QC-LDPC를 구성.
6. Theorem 5/Corollary 3로 파라미터 `nb,f0,f1,α,c` 선택 시 girth-8을 결정론적으로 달성 → error floor 저하.
7. 복호: 로컬/글로벌 2-phase TPD(로컬 디코더 실패 시에만 글로벌 디코더 가동)와 flooding FSD를 min-sum(`MSA`, max iter 100)로 시뮬.
8. BEC상 density evolution으로 FSD/TPD threshold 유도(식23~25), TB 구조가 GC 대비 threshold 개선을 정량 분석.
9. 결과: girth-8 예제(rate 0.85~0.95, length 최대 23184)에서 BER 10^-8까지 error-floor 무관측, waterfall이 GC 대비 우수, TPD가 FSD 대비 복잡도·latency·메모리 대폭 절감.
10. 한계: HW 미설계, GF(q) 대수 구성 의존, TPD 스케줄이 우리 flooding min-sum 파이프라인과 상이.
