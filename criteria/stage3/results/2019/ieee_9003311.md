### B. 알고리즘 요약

1. 대상은 이진/비이진 regular LDPC, codeword 길이 `n=2000`, rate `R=0.8`, column weight `wc=3,5,7,9`; 비이진은 GF(4)~GF(64), AWGN + BPSK.
2. 문제: 고throughput 광통신은 고rate·긴 codeword가 필요한데 soft BP 복호가 HW에 넣기엔 너무 복잡 -> 저복잡도 hard-decision 복호가 필요.
3. 채널 모델 `y=c+η`, `η~N(0,σ²I)`; q-ary 심볼은 `log2(q)` 비트로 BPSK 변조.
4. 핵심: syndrome `S=rH`의 weight를 매 iteration 줄이는 threshold decoding. 오류 확률 높고(불만족 check 다수) 결정 신뢰 높은 심볼부터 정정.
5. 단일 임계값(Single-Threshold) = 원조 majority decoding: 심볼별 비제로 메시지 최다 부분집합 크기 `a=|Lmax|`가 `a - z > θ`면 해당 심볼을 `m=argmax(L)`로 갱신.
6. 다중 임계값(Multi-Threshold): 임계값 리스트 `θ1>...>θn`을 큰 값부터 순차 적용, 각 단계에서 Single-Threshold 호출 후 syndrome weight=0이면 종료.
7. 큰 θ부터 시작해 확실한 오류만 먼저 정정하고 점차 임계값을 낮추는 순차 스케줄이 핵심 트릭.
8. 학습/최적화 절차 없음(임계값은 사전 설정, 개수 최적화는 후속연구로 남김).
9. 결과: 다중 임계값이 모든 q에서 단일 대비 정정성능 개선; wc>5에서 GF(16)이 최고, sparse(wc<7)에선 이진이 우위; 입력 BER 기준 이진 우세, SER 기준 비이진 우세.
10. 한계: 시뮬만(FER/BER/SER 곡선), HW 설계 없음, 광통신 타깃·비이진 GF(q) 위주, soft-info 미사용.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 다중 임계값 hard-decision 복호 iteration 루프 | decoder.cpp LDPC_Decoder() | 복호 루프에 닿지만 우리는 min-sum soft 기반, majority-threshold와 알고리즘 상이 |
| syndrome 기반 심볼 정정 | 대응 없음 | min-sum C2V/V2C 구조와 다른 syndrome-weight majority 방식 |
| 비이진 GF(q) LDPC | 대응 없음 | Prime ECC는 binary QC-LDPC 전용, non-binary 미지원 |
```

적용 가치: **낮음** - Prime ECC는 이미 soft min-sum + 적응형 LLR을 보유하며 binary QC-LDPC 전용인데, 본 논문은 비이진 위주 hard-decision majority(교과서 수준) 복호라 이식 이득이 없다.
