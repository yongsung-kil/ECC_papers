### B. 알고리즘 요약

1. 대상은 polar code `P(2048,1024)`, `P(1024,512)`, BPSK/AWGN, 1 dB SNR 기준으로 구성 (LDPC 아님).
2. 문제: polar BP(및 BPL)는 SC/SCL 대비 병렬성은 좋으나 false/oscillation/unconverged 오류로 BER이 나쁨.
3. 모델: 채널출력 `y`에 평균 0 가우시안 인공노이즈 `n_i`를 더한 `ỹ`를 각 브랜치 입력으로 사용.
4. 핵심 기법: `L`개의 동일 구조 병렬 BP 디코더 각각에 서로 다른 표준편차 `σ_i`(0부터 0.0125 step, 최대 0.3875)의 인공노이즈를 주입 — `n_i(x)=μ_i+σ_i×wgn(x)`.
5. 식 의미: 소량 노이즈가 stochastic resonance처럼 갇힌 수렴점을 흔들어 미수렴 오류를 탈출시킴.
6. 확장: 기존 BPL[6]의 permuted factor graph 대신 동일 구조 + 노이즈 차등으로 리스트 다양성 확보(리스트 크기 = 노이즈 레벨 수).
7. 구현 디테일: RT(round trip) 스케줄링으로 평균 iteration 감소, frozen 노드 고정용 좌측 check node 추가.
8. 조기종료: 매 iteration 후 G-matrix 기반 및 CRC-aided EDTC 체크, 어느 브랜치든 만족 시 성공 처리(별도 post-decision 불필요).
9. 결과: list 32, max iter 200에서 `P(2048,1024)` 10^-5 BER 기준 SCL+CRC-16 대비 0.3 dB 이내, BP/SC/SCL보다 우수.
10. 한계: HW 미설계·시뮬만, polar 전용, list 크기 탓 2.75 dB 이하 구간 평균 iteration이 SMS BP보다 많음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar BP list 디코더 전체 | 대응 없음 | polar factor graph 기반, binary QC-LDPC decoder(`decoder.cpp`)와 구조 상이 |
| 인공노이즈 주입/EDTC 체크 | 대응 없음 (개념적으로만 `channel.cpp` soft-read, `partial_CRC.cpp` 유사) | 채널 노이즈 모사와 CRC 조기종료는 이미 보유, 인공노이즈 아이디어는 미보유 |
| RT 스케줄링 | 대응 없음 | Prime ECC는 Dual-Update(even/odd) 스케줄, polar RT와 무관 |

적용 가치: 낮음 — polar 전용 non-LDPC 기법이고 인공노이즈 아이디어 정도만 개념적 참고 대상.
