> 총평: 오류허용 approximate data를 known bit(all-1)로 대체해 코드워드 유효 RBER를 낮추는 데이터 조직·할당·GC 기법으로, LDPC 디코더/부호는 손대지 않아 Prime ECC 디코더 코어 이식 대상이 아니다.

### B. 알고리즘 요약

1. 시스템: LDPC 기반 3D NAND SSD (SSDSim, 2ch/2chip/4plane, 256blk, 64×4KB page). ECC rate 예시 `8/9`.
2. 문제: 고 RBER 데이터는 LDPC soft decoding에 다중 sensing이 필요해 read latency 급증.
3. 관찰: 모바일/PC 워크로드에 오류허용 approximate data(멀티미디어 캐시 등, 1시간 미만 보존, 대부분 초 단위 삭제)가 다량 존재 → 신뢰성 요구 완화 가능.
4. 핵심 기법(Robust ECC): 한 페이지에 regular+approximate를 함께 담되, approximate `k/2`bit는 ECC에 미포함, `n-k` parity 전량을 `k/2`bit regular 보호에 사용.
5. 인코딩: regular `r_{k/2}`에 known vector(예 `1_{k/2}`)를 결합해 `k`bit 구성 → parity `p_{n-k}` 생성 → known 부분 제거 후 프로그램. read 시 all-1을 자동 재결합해 디코딩.
6. 핵심 식: `RBER2 = RBER1·(1 - k/2n)` — known bit엔 오류가 없어 코드워드 유효 RBER 감소(rate 8/9면 5/9로 감소).
7. 데이터 할당: 요청 큐의 regular/approximate 비율에 따라 RAB/RB/HRB/AB 4종 블록에 write buffer로 재조직, RAB 최대화가 목표. L2P 매핑은 RAB/HRB에서 half-page 단위로 2배.
8. GC: `IVmax - min(IVR,IVA) ≤ T`이면 RB+AB 쌍을 victim으로 골라 valid page를 RAB로 합침(GC효율 소폭 희생, T=2 최적).
9. 결과: Baseline 대비 read latency early 12%/late 30% 감소, HRB 사용 시 lifetime 15% 연장. RAB 비중 ~10%라 FTL 매핑 오버헤드 낮음.
10. 한계: 디코더/부호 자체 미변경(ECC를 블랙박스로 사용), HW 미설계, SSDSim trace 시뮬만, approximate data 식별은 파일타입 태그 의존.
