> 총평: 표준 binary IR-QC-LDPC(dual-diag+NMSA)의 FPGA 구현으로 부호·복호기법은 이미 보유하나 체크행렬 BRAM 병렬 주소변환 라우팅 기법은 HW 메모리병목 관점에서 참고가치 있어 이식성 중.

### B. 알고리즘 요약

1. 대상은 binary IR-QC-LDPC (dual-diagonal parity), z=81 circulant, `l=1944, z=81, n=24`, code rate 1/2·2/3·3/4·5/6, QKD key reconciliation용 BSC 채널.
2. 문제: 체크행렬을 BRAM에 관행적으로 저장하면 메시지패싱 시 read/write 효율이 낮아 throughput이 제한됨.
3. 모델: BSC 가정, 수신확률 `P0/P1`을 8-bit로 양자화하여 LLR `L0=lg(P0/P1)` 초기화.
4. 핵심 기법(인코딩): Yoon[35]의 recursive accumulated 알고리즘으로 dual-diagonal `HP` 구조에서 `λi=Σ H_{i,j}·s_j` 계산 후 `p0=Σλi`, 이후 forward recursion으로 parity 순차 산출(1 clock/parity).
5. 핵심 식 의미: dual-diagonal 구조가 parity를 재귀적으로 풀 수 있게 해 인코딩을 저장·연산 최소화.
6. 핵심 기법(디코딩): 8-bit 양자화 NMSA — CNU는 `δ=α·Πsign·min|ρ|`(정규화 α), VNU는 C2V 합산; 알고리즘상 α=0.75, 시뮬 최적 α=0.4.
7. 구현 디테일: VN/CN 메시지를 init-array·C2V-array·V2C-array 세 BRAM 배열에 분리 저장, circulant shift만큼 주소를 변환하는 병렬 cyclic-shift address decoder((18)(19))로 모든 BRAM 동시 read.
8. 학습/최적화: 정규화 인자 α를 Matlab로 스윕(1000 frame/회)해 FER<0.01 조건 최적값 결정.
9. 결과: Xilinx Kintex UltraScale에서 25 MHz, frame 1296 bit, 초기 BER 6%에서 FER<1%(효율 1.49), 인코더 183.36 Mbps·디코더 평균 27.85 Mbps, LUT 42.34%·BRAM 35% 사용.
10. 한계: 정정능력·iteration 개선 없는 순수 구현, BSC/QKD 타깃이라 NAND soft-read 미지원, dual-diag+NMSA는 신규성 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |

| 반복 message-passing 루프 | `decoder.cpp` `LDPC_Decoder()`, `C2V_Cal()` | 표준 이터레이션, 이미 보유 |
| dual-diagonal 재귀 인코딩 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | Prime ECC도 dual-diagonal 인코딩 보유, 중복 |
| BRAM 병렬 cyclic-shift 주소변환 | `GT.cpp` `Make_GT_HW()` (부분 유사) | circulant edge/메모리 라우팅 관점 참고, 구조는 상이 |

| 8-bit LLR 양자화(BSC) | `channel.cpp` `Set_LLR_Th()` | Prime ECC는 hard/2SD/3SD 테이블, BSC 8-bit는 부분 대응 |
