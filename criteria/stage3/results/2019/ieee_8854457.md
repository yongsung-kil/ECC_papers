### B. 알고리즘 요약

1. 대상 시스템: BPSK/AWGN 채널, IEEE 802.11 QC-LDPC `(648,324)` rate 0.5, `z=27` circulant, LLR은 5정수+2소수 fixed-point sign-magnitude.
2. 문제: SRAM FPGA는 config-memory SEU가 회로기능을 영구 변형 → ASIC와 달리 단일 SEU 하나로도 LDPC 디코더 성능이 크게 열화됨을 규명.
3. 모델: config bit flip을 ICAP fault injection으로 주입, 각 critical bit가 PE 출력에 미치는 효과(부호변화/magnitude 증감/control error)를 통계화.
4. 디코더 알고리즘: Layered Belief Propagation(LBP) + Modified Min-Sum(MMS), 정정오프셋 `β`를 빼는 근사; layer당 `Z`개 CN 병렬(`T=1`), 두 최소 magnitude만 계산.
5. HW: `Z`개 PE + cyclic shifter + H ROM(shift 차분 저장), bit memory `W*Z`; shift를 되돌리지 않고 저장해 지연 감소.
6. 핵심 지표 WPD: `WPD(c)=(CB(c)/ΣCB)*(BER(c)/BER_FF)` - 컴포넌트의 critical bit 비율 × fault 시 BER 열화배수.
7. 면적 반영 지표 WPDA `= WPD(c)/(LUTs(c)+FFs(c))` 로 컴포넌트를 랭킹, 높은 것부터 DMR/TMR 적용.
8. Mem 계열은 block RAM ECC로 이미 커버되므로 제외, PE(전체면적 87%)의 SUB/SUM/Smallest2 등 연산부만 대상.
9. 결과: 상위 3개 컴포넌트(SUM/SUB/Smallest2)만 커버해 성능열화 89% 감소를 면적 55% 커버로 달성; 최소 magnitude 처리 경로(SUB β0/Max0 0)가 더 임계적.
10. 한계: FPGA config-오류 전용(영구 fault), ASIC엔 부적합; 부호정정능력 자체 개선 아님, throughput/gatecount·수렴 iteration 수치 본문 미제시.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LBP+MMS CN 연산(두 최소 magnitude, β 오프셋) | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | min-sum 자체는 §3 이미 보유, 본 논문 기여(fault tolerance)와 무관 |
| PE VN 처리/양자화(fixed-point sign-mag) | decoder.cpp VN_Cal_HD() 등 | 소프트 LLR 처리 대응은 있으나 논문 초점은 아님 |
| cyclic shifter / H ROM 라우팅 | 대응 없음 (참고: ecc_top.cpp Load_PCM) | Prime ECC는 z=32 고정 HW, FPGA shifter/ROM 구조와 상이 |
| config-memory SEU DMR/TMR fault tolerance | 대응 없음 | ASIC bit-exact 시뮬레이터엔 FPGA config-fault 모델 부재 |
```

적용 가치: **낮음** — QC-LDPC LBP+Min-Sum 디코더를 쓰지만 기여의 본질이 SRAM-FPGA config-memory 고장 내성이라 ASIC 기반 NAND Prime ECC의 부호/복호성능·HW 최적화와 접점이 없다.
