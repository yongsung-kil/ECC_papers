### B. 알고리즘 요약

1. 시스템: MLC NAND(2bit/cell, SE/S1/S2/S3), QC-LDPC(`CR`=0.89, 0.80, 정보 2KB), AWGN+4-PAM 채널, min-sum 복호 기준.
2. 문제: LDPC soft-decision 반복복호로 read latency 증가, 특히 MSB 페이지 RBER가 LSB보다 훨씬 높아 MSB iteration이 급증.
3. 관찰: 한 셀의 LSB/MSB가 동시에 틀리는 `PBE`(pair-bit error) 특성 — FPGA 플랫폼에서 블록별 PBE 비율 최대 67% 측정.
4. 핵심 기법: LSB 페이지를 먼저 복호하여 soft 결과 `(L1..Ln)`를 버퍼에 저장, MSB 초기 LLR `(M1..Mn)` 최적화에 활용.
5. 판정식: LSB 복호값과 초기 LLR의 곱 `VNj·ISILSBj < 0`이면 LSB 오류로 간주 → 대응 MSB LLR을 `OSIMSBj = -0.85·ISIMSBj`로 flip(부호 반전).
6. 원리: PBE 상관상 LSB가 틀리면 MSB도 틀릴 확률이 높으므로, LSB 오류 위치의 MSB 초기 정보를 반전시켜 초기 정밀도를 높이고 수렴 가속.
7. 구현 디테일: min-sum CN 갱신 `CN = α·sgn·min(|VN|)`, `α=0.85`, 최대 iteration 50, LSB/MSB 두 페이즈로 분리 처리.
8. 학습/최적화: 별도 학습 없음, PBE 통계 관찰 기반 휴리스틱 전처리.
9. 결과: `CR`=0.89에서 RBER≈2.5e-2일 때 MSB 복호 iteration·time 최대 54% 감소, RBER≈1.3e-2에서 26.9% 감소. 오버헤드는 1페이지(2KB) 버퍼와 소량 시간.
10. 한계: HW 미설계(MATLAB 시뮬만), MLC LSB/MSB 페이지 쌍 구조 및 순차 복호에 의존, 순차 read 시나리오에 유리.
