### B. 알고리즘 요약

1. 시스템/채널: AWGN·BPSK, inner=SC-LDPC(`(nc,nv)=(3,6)`, `M=126`, `ω=3`, `L=60`, girth≥8), outer=RS `(63,59)`. 코드 rate: SC-LDPC 0.47, 연접 0.44.
2. 풀려는 문제: SC-LDPC의 sliding window decoding(SWD)은 타깃 심볼셋에서 버스트 에러가 남아 error-floor를 유발.
3. 핵심 가정: SWD 잔류 에러를 강한 버스트 정정 능력의 RS outer code로 제거하면 error-floor를 없앨 수 있다.
4. 핵심 기법 1단: RS-SC-LDPC 연접 - RS로 먼저 인코딩한 이진 시퀀스 `U[0,L-1]`를 SC-LDPC로 다시 인코딩, SWD 후 하드결정 심볼을 BM 알고리즘으로 RS 복호.
5. 핵심 식: inner SC-LDPC의 **지역-시스테매틱 인코딩**(식 24/25) - `H0(t)`의 마지막 `Nc`행을 항등부분행렬화하여 SWD 출력을 outer RS가 다시 복호 가능하게 함.
6. 핵심 기법 2단: inner 부호를 4-cycle 제거(대표블록 `BR`/구성블록 `BC`/제외패턴 `BE`) 2단계 설계로 girth≥6 확보 후, Fossorier 조건 M-fold lifting으로 girth≥8까지 확대.
7. 구현 디테일: RS 복호 성공 시 타깃 심볼 LLR을 `±∞`로 확정해 SWD에 재투입(BM feedback), 반복적으로 SWD 성능 향상.
8. 학습/최적화: 없음 (조합적 부호 설계).
9. 결과: 설계된 RS-SC-LDPC는 BER `10^-8`까지 error-floor 미출현. RS `(63,59)`가 `(63,61)`보다 잔류 에러 제거에 유리. 단 rate 손실로 waterfall 소폭 열화.
10. 한계: HW 미설계, 시뮬만. 윈도우 `W`가 작으면(W=8) error-floor 완전 제거 실패. 저rate·SC-LDPC 스트리밍 특화.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| SC-LDPC 부호구조 / protograph lifting | 대응 없음 | Prime ECC는 고정 binary QC-LDPC이고 SC-LDPC(대각밴드+coupling) 구조 미지원 |
| Sliding window BP 복호 | 대응 없음 | Prime ECC 디코더는 블록 min-sum(`decoder.cpp` `LDPC_Decoder()`)으로 SWD 흐름과 상이 |
| RS outer 연접 / BM 복호 | 대응 없음 | 외부 RS+BM 연접은 Prime ECC 코드베이스 범위 밖 |
| 지역-시스테매틱 인코딩 | 대응 없음 | Prime ECC 인코딩은 dual-diagonal(`encoder.cpp`)로 구조 상이 |
| 4-cycle 제거 / girth 설계 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | H-matrix 설계 레이어에 닿으나 SC-LDPC 특화로 재사용 난도 매우 높음 |

적용 가치: **낮음** - SC-LDPC + SWD + RS 연접이라는 스트리밍 특화 구조가 NAND용 고rate binary QC-LDPC(블록 min-sum, dual-diagonal 인코딩)와 부호설계·디코더·인코더 전 레이어에서 정합되지 않음.
