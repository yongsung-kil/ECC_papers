### B. 알고리즘 요약

1. 대상은 GLDPC 앙상블 `C_{J,K,ν}`: base DD의 SPC 노드 일부(비율 `ν`)를 blocklength-`K` 컴포넌트 부호(GC 노드)로 치환, BEC(소거확률 `ε`) 전송.
2. 문제: GC 노드 ML 복호의 점근 분석은 decodable erasure 패턴 나열을 위한 multi-edge-type(MET) DD가 필요해 파라미터가 폭증.
3. 핵심 아이디어: 각 GC 노드의 weight-`w` erasure 패턴 복호 성공을 확률 벡터 `p=(p1..pK)`로 파라미터화, `pw`는 무작위 weight-w 패턴이 복호되는 비율.
4. P-PD(Algorithm 2): peeling 중 GC 노드가 edge를 잃을 때 `Bernoulli(pw)`로 decodable 태깅 - MET DD 없이 beyond-BD/ML 복호를 포함. `pw=1(w<d), 0(w≥d)`이면 BD-PD로 환원.
5. rate 경계: `R(ν)=R0-ν(1-R0)(k_avg-1)`, Hamming(sphere-packing) 상계와 Varshamov 하계로 `(ν,d,K)` 함수로 GLDPC rate를 bound.
6. weight 분포 성장률 `G(α)`로 최소거리 선형성 확인 - `J>2`면 항상 good growth, `J=2`면 `ν>(K-2)/(K-1)` 필요.
7. 점근 분석: Wormald 정리로 residual graph DD 진화를 미분방정식계(식 24~28)로 기술, blocklength 증가 시 지수적으로 수렴.
8. 컴포넌트 부호는 길이 6~15의 Hamming/Cyclic/QC/Cordaro-Wagner 등 실제 선형블록부호를 `(d, p)`로 표로 정리해 대입.
9. 결과: (2,7) base + (7,4)-Hamming GC에서 BEC threshold ML=0.7025 vs BD=0.5135; capacity gap 최소화하는 임계 `ν*<1` 존재, 고rate 컴포넌트가 유리; P-PD BER이 실제 ML-PD와 정확히 일치(n=10000 시뮬).
10. 한계: BEC 전용 순수 이론, HW·throughput·soft-read 없음, min-sum 복호 무관, NAND 채널 미고려.
