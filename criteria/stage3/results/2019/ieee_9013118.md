### B. 알고리즘 요약

1. 시스템: QC-LDPC 블록부호(exponent matrix `P`, circulant size `N`, lifting)를 edge spreading으로 QC-SC-LDPCC로 변환, BPSK/AWGN에서 sliding-window(`W=5(ms+1)`, 100 iter) 복호.
2. 문제: Tanner graph의 harmful object(stopping/trapping/absorbing set, pseudocodeword)가 error-floor에서 성능을 크게 떨어뜨림, 기존 최적화는 특정 구조·작은 memory로 제한됨.
3. 모델: `(ms+1)`-ary spreading matrix `B`(정수벡터 `b`)로 블록부호를 memory `ms`의 convolutional exponent matrix `P[0,∞]`로 spread.
4. 핵심 조건: block-cycle이 SC부호에 살아남는지는 `Remark 1`의 조건(cycle 존재조건 식 `Σ(p_mi,ni − p_mi,ni+1)=0 mod N`을 `B` 부분행렬 `Bλ`가 정수상에서 만족하는지)으로 판정.
5. 의미: 한 exponent matrix의 block-cycle 개수만큼 방정식만 확인하면 spreading 후 남는 cycle 다중도를 결정 가능 → 탐색 비용을 대폭 축소.
6. 핵심 기법(MIHAO): all-zero `B`를 root로 하는 트리탐색, tier `l`은 non-zero 항목 `l`개로 harmful object 평균 다중도를 부모보다 줄이는 후보들, 개선 없으면 backtrack(greedy).
7. 지표: node당 평균 harmful object 수 `Eλ`(block-cycle 및 `(a,b)` AS/FAS) — 후보 goodness 측정.
8. 결과(다중도): `(3,5)`-regular Tanner code(`R=2/5`)에 적용해 길이 8 cycle 완전 제거(`b1`), memory `ms=3`까지 늘려 길이 10 cycle까지 제거(`b3`).
9. 결과(BER/속도): array `R=10/13` 및 Tanner 부호에서 최적화 spreading이 random 대비 BER 개선(Fig.1,2), random search 대비 평균 3.5~8.2배 속도향상(Table II).
10. 한계: HW 미설계·시뮬만, non-binary 아님이나 SC-convolutional 특화 구조, 복잡도/gatecount/throughput 미제시.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| harmful object 최소화 부호설계(MIHAO) | `ecc_top.cpp` `Load_PCM()` (H-matrix/부호구조) | 부호설계 층이나 SC-convolutional 대상이라 고정 QC-LDPC 블록 H와 직접 대응 안 됨 |
| edge spreading으로 QC-LDPC→QC-SC 변환 | 대응 없음 | Prime ECC는 spatially-coupled 미지원, 블록부호 전용 |
| trapping/absorbing set 제거 → error-floor 개선 | `corner.cpp` (수렴실패 추적) | 목적(수렴실패/error-floor 진단)은 유사, 설계 방법은 이식 부담 큼 |
| sliding-window 복호 | 대응 없음 | Prime ECC는 블록 min-sum, window decoding 없음 |

적용 가치: **낮음** — error-floor 개선용 부호설계 원리는 유의미하나, 대상이 QC-SC-LDPC convolutional 구조라 프로파일 §4의 "부호설계(H-matrix) 이식 난이도 높음"에 해당하고 Prime ECC의 고정 blocklength QC-LDPC 블록부호와 정합되지 않음.
