### B. 알고리즘 요약

1. 시스템: `(N,K)` polar code(`N=2^n`), 5G 제어채널용, BPSK+AWGN, 검증 `(64,32)`·`(1024,512)`(rate 0.5).
2. 문제: SC 디코더가 양자화 LLR에서 float 대비 성능 손실 큼(HW 구현 시 병목).
3. 모델: 채널 LLR `L_n=2y/σ^2`, uniform 양자화 `Q(x)`, 절단 `M=2/σ^2+3·2/σ`, 3σ 원칙.
4. 핵심기법: WSC — SC의 LLR 업데이트 f/g 출력에 가중치 곱: `f*(a,b)=w·sign(a)sign(b)min(|a|,|b|)`, `g*(a,b,ŝ)=w·(a+(1-2ŝ)b)`.
5. 의미: 양자화가 LLR 분포를 왜곡 → 가중치로 분포 스케일을 조정해 손실 보상(디코딩 규칙 자체는 SC 유지).
6. 확장: SC factor graph 구조 그대로 NN(입력N/은닉2(N-1)/출력K)으로 매핑, 가중치를 학습으로 자동선정.
7. 구현 디테일: XOR로 인한 gradient 소실을 softened sign `ssign(x)=ReLU` 조합으로 대체(부분합을 {1,-1} 곱셈으로), frozen/rate-1 노드 제거한 SNN으로 복잡도 감소.
8. 학습: all-zero 코드워드로만 학습(SC가 코드워드 독립), Adam+mini-batch, MMSE 손실. 은닉층당 가중치 공유(SNN*)로 학습 batch 절반 감소.
9. 결과: `(64,32)` SNN 3bit가 float SC 근접, `(1024,512)` SNN 5bit ≈ QSC 6bit 성능(1bit 절감), 동일 정밀도 QSC 대비 BLER 개선.
10. 한계: HW 미설계·시뮬만, polar/SC 전용이라 LDPC min-sum과 무관, 양자화 알고리즘 자체는 논외.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| polar SC/WSC 디코딩 | 대응 없음 | polar code + SC는 LDPC min-sum과 다른 계열 |
| NN 학습 가중치 | 대응 없음 | Prime ECC는 딥러닝 디코더 미보유 |
| f 함수 `sign·sign·min` | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 형태만 min-sum과 유사, 부호·스케줄 상이 |
| LLR 양자화 정밀도 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Get_VNU_Table_Idx()` | 3bit 양자화 성능회복이라는 목표만 개념 공유 |

적용 가치: **낮음** — 부호(polar)와 디코더(SC+NN)가 모두 NAND binary LDPC/min-sum과 이질적이라 직접 이식 불가. min-sum에 학습 가중치를 붙이는 아이디어는 참고문헌 [11][12]에서 별도 확인이 나음.
