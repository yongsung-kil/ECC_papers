> 총평: 인지무선용 병렬연접 Gallager(LDPC) 부호의 디코더 구조 개선 논문으로 NAND 고rate QC-LDPC에는 이식가치 낮음, 이식성 하.

### B. 알고리즘 요약

1. 대상은 인지무선(secondary user) 신뢰전송용 **PCGC**(병렬연접 Gallager=LDPC 부호), rate `1/3`, frame `N`=600, BPSK/AWGN·Rayleigh.
2. 문제: LDPC는 블록길이 증가 시 인코딩(2차)·디코딩 복잡도 급증, 기존 PCGC는 dedicated 부호 성능 미달·직렬복호로 지연 큼.
3. 모델: primary user 간섭+채널잡음을 AWGN으로 가정, spectral handoff로 인한 데이터 손실 시나리오.
4. 핵심기법: **단일 인코더 PCGC** - 하나의 mother code(MCW 기준 선정)로 systematic `x`+parity `p`를 만들고 `p`를 2회 전송, 동일 PCM의 두 component decoder가 동시 복호.
5. 핵심 아이디어: 두 component codeword가 동일하므로 super iteration마다 posterior LLR(채널정보+extrinsic)을 교환해 상호 신뢰도 boost, 빠른 수렴.
6. 구조 개선: **인터커넥트 체크노드 제거** → 기존 단일인코더 PCGC의 `3M` 체크노드를 `2M`으로 축소.
7. 복호식: MSA(min-sum) - `Extr = ∏sign · min|Intr|`, `Mr = LLR + ΣExtr`, 정지조건 `H·C^T=0` 또는 `SI=SImax`.
8. 학습/최적화: 없음(component code는 mean column weight로 선택).
9. 결과: 제안 PCGC가 AWGN 10^-5에서 기존 PCGC 대비 +0.5dB(fading +0.8dB) 코딩게인, dedicated LDPC 동급, decoding delay 0.098s(기존 0.269/0.666s)로 최소, error floor 낮음.
10. 한계: HW 미설계(노드/연산 count 정성비교뿐), gatecount·throughput 없음, 저rate·단프레임 무선 특화.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| min-sum(MSA) CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 표준 min-sum, 이미 보유 - 중복 |
| VN LLR 갱신 | decoder.cpp VN_Cal_HD() | 표준 VN 업데이트, 이미 보유 |
| 병렬연접(turbo류) 2-decoder 구조 | 대응 없음 | Prime ECC는 단일 QC-LDPC 복호로 연접구조 아님 |
| syndrome 정지조건 | partial_CRC.cpp / full_CRC.cpp | Prime ECC는 CRC 기반 조기종료 사용, syndrome-only와 상이 |
```

적용 가치: **낮음** — turbo류 병렬연접·저rate 무선 부호설계로 고rate NAND QC-LDPC 파이프라인과 정합 안 됨, 디코더 요소(min-sum/정지조건)는 이미 보유.
