### B. 알고리즘 요약

1. 대상: 이산 pairwise MRF 위의 Sum-Product BP(PGM 추론), 벤치마크는 Ising grid(100x100, 200x200)와 chain(100000), 실데이터는 protein-folding — LDPC/ECC 아님.
2. 문제: BP는 loopy 그래프에서 수렴 보장이 없고, GPU 다중코어에서 스케줄링 선택이 속도·수렴성을 크게 좌우.
3. 모델: 메시지 `m_{i→j}(x_j) ∝ Σ_{x_i} ψ_{i,j}ψ_i Π_{k∈∂i\j} m_{k→i}`, residual `r(m)=||f_BP(m)-m||`로 갱신 우선순위.
4. 핵심 기법: frontier 기반 병렬 프레임워크로 LBP/RBP/RS를 GPU 구현, top-k를 sort-and-select로 근사해 병렬성 파라미터 `k`(=`p·2|E|`) 조절.
5. 핵심 의미: 병렬성↓이면 수렴↑·속도↓, 병렬성↑이면 속도↑·수렴↓라는 트레이드오프를 실증.
6. 확장(RnBP): 정확한 top-k 대신 residual>ε 필터 후 무작위 비율 `p` 선택으로 저오버헤드 스케줄링.
7. 구현 트릭: `EdgeRatio=New/Old` 미수렴 edge비로 `p`를 동적 조절(0.9 초과 시 LowP), CUDA/CUB/cuRAND 사용.
8. 학습/최적화: 없음(휴리스틱 파라미터 `LowP`/`HighP` 수동 설정).
9. 결과: RnBP가 SRBP 대비 최대 ~2203x, GPU LBP 대비 평균 9x(Ising 200x200) 가속, 어려운 그래프에서 유일하게 전 그래프 수렴, 정확도는 SRBP와 동등(KL).
10. 한계: HW 미설계(GPU SW), sort-and-select 오버헤드(런타임 90~98%), 어려운 그래프에서 `p` 민감, ECC/NAND 무관.

### C. Prime ECC 관련 모듈 핀포인트

대상=other(일반 PGM BP)이므로 Prime ECC 디코더 모듈과 직접 대응 없음.

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Sum-Product BP 메시지 갱신 | 대응 없음 | full-tanh Sum-Product·비ECC 추론이라 min-sum decoder.cpp와 무관 |
| GPU frontier/무작위 스케줄링 | 대응 없음 | HW(RTL) bit-exact 시뮬 구조(z=32 파이프라인)와 실행 모델 상이 |
| residual 기반 메시지 선택 | 대응 없음 | 우리 스케줄은 layered/dual-update 고정, 무작위 GPU 스케줄 이식 불가 |

적용 가치: **낮음** — 일반 PGM 추론용 GPU BP 스케줄링 연구로 NAND binary QC-LDPC 디코더/HW와 대응 모듈이 없다.
