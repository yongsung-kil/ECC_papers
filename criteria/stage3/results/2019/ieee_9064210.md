### B. 알고리즘 요약

1. 5G LDPC(BG2, `Z=16`, rate 1/2)와 polar code(N=256, rate 1/2)를 하나의 BP SISO로 디코딩하는 재구성형 통합 디코더 아키텍처를 제안한다.
2. 문제: LDPC와 polar를 각각의 알고리즘(SC/SCL 등)으로 처리하면 HW 오버헤드가 크므로 동일 알고리즘(LDPC BP)으로 통일해 CNU/VNU 로직을 공유하려 함.
3. polar 인수그래프를 sparse bipartite graph로 pruning([12] 방식)하면 LDPC-like BP 적용이 가능하나, short cycle(길이 6,8)이 많아 성능이 나쁘다(TABLE I).
4. 핵심 기법: `n!`개의 stage permutation 중 `L`개를 골라 서로 다른 sparse PCM `H_s`를 만들고, 각 그래프에 LDPC BP를 돌리는 BP List(BPL) 디코딩(Algorithm 1)으로 short cycle 영향을 완화.
5. 첫 permutation은 원본 stage 순서(성공확률 높음)로 고정하고 나머지 `L-1`개는 무작위 permute.
6. CN 갱신 `L(r)`은 min-sum형 식(1), VN 갱신 `L(q)`는 식(2), 사후확률 `L(Q)` 식(3)로 계산 — LDPC BP와 동일 절차.
7. 조기종료: dense H 기반 criterion(sparse H_s 대신 작은 H로 syndrome 검사, HW 친화)과 CRC 기반 criterion(5G LFSR 재사용, 오종료 감소) 두 종류 제안.
8. LDPC측은 산업표준 Layered BP([14])를 그대로 채택(신규성 없음).
9. 결과: polar에서 제안 BPL(List 8, max iter 40)이 BP/SC를 상회, 고SNR에서 SCL에 근접하며 SC/SCL 대비 latency 1~2 orders 낮음; 복잡도는 원본 polar BP 대비 CMP/ADD/XOR 대폭 감소(TABLE III).
10. 한계: SCL/CA-SCL보다는 성능 열위, permutation이 무작위라 최적 아님, FPGA 미구현(future work).

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC BP 디코딩(CN/VN 갱신) | decoder.cpp LDPC_Decoder(), C2V_Cal() | polar 배제 시 표준 min-sum류라 신규성 없음 |
| CN min-sum 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | 이미 보유 기법과 중복 |
| CRC 기반 조기종료 | full_CRC.cpp, partial_CRC.cpp partial_crc_HW_T4() | 이미 보유(CRC 조기종료) |
| polar sparse bipartite graph / BPL | 대응 없음 | polar 전용, NAND binary QC-LDPC 무관 |
| QC-LDPC layered BP | decoder.cpp LDPC_Decoder() 이터레이션 루프 | 표준 재사용, 중복 |
```

적용 가치: 낮음 — 신규성은 polar용 BPL과 LDPC/polar HW 공유에 있고, NAND binary QC-LDPC 디코더에 떼어 쓸 고유 LDPC 기법은 표준 layered BP/min-sum/CRC 조기종료뿐이라 모두 이미 보유.
