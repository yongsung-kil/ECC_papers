### B. 알고리즘 요약

1. 불변(immutable) 분산 스토리지용 period-1 convolutional LDPC, 이진 소거채널(BEC), 노드(디바이스) 단위 완전 소거만 고려, systematic·고rate·저locality 목표.
2. 문제: 소거 부호는 데이터 도착 즉시 저장하려 길이가 짧아야 하고, 짧은 LDPC는 유한길이 성능 미확립 - RS 대비 열세.
3. `bSTEP(s,p)`: 각 데이터 객체를 `s` 소스 블록으로 나눠 이미 저장된 `t=s+p` 포인터 블록과 얽고 `p` 패리티 블록 생성 (`p×2t` 이진 H).
4. 얽힘 전략 `α=(α1,...,αt)`로 convolution 정의, 포인터 열은 데이터/패리티 열의 shift - 비종단 convolutional 부호.
5. 복호는 message passing과 일치하는 STEP-archive 재귀 재구성으로 소거 복구.
6. 핵심: minimal erasure(재구성 실패 최소 소거패턴) 분석 - intrinsic(H 유래)/incidental(α 유래) 구분, cycle 조건(Lemma 1~4)으로 size 4/6 회피.
7. Stopping Set Enumeration(SSE)로 최소 minimal erasure 크기 `σ` 최대·multiplicity `nσ` 최소인 `H,α` exhaustive search (`1≤s≤5`, `p=2`).
8. 최적 결과: bSTEP(1,2) `σ=12`, ... bSTEP(5,2) `σ=8, nσ=6` (Table I).
9. Markov 모델로 시스템 MTTDL 계산, `r(i)`(크기 i 복구가능 패턴 비율)를 minimal erasure 지배 근사로 추정 - 동일 overhead에서 RS/LRC 대비 수 자리수 신뢰도 우위.
10. 한계: HW 미설계, 이론/Markov 분석뿐, BEC 노드 소거 전제, tail 블록은 복제 보호 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| bSTEP H,α 구성 | 대응 없음 | 스토리지 얽힘 부호로 NAND QC-LDPC H-matrix 로드(`Load_PCM()`)와 성격 상이 |
| minimal erasure/SSE 분석 | 대응 없음 | BEC 소거 패턴 분석으로 AWGN/RBER soft 복호와 무관 |
| message passing(소거 복구) | 대응 없음 | 소거 채널 재구성이라 min-sum LLR 메시지패싱과 다름 |

적용 가치: **낮음** - BEC 노드 소거 기반 분산 스토리지 부호설계로, NAND 비트 단위 soft-read binary QC-LDPC ECC 코드베이스에 대응 모듈이 없고 이식 대상이 아님.
