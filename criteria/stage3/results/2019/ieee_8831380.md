### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 5G URLLC용 short-blocklength Polar 부호(`N=32~128`, `R=0.5/0.75`), BPSK+AWGN, 8bit CRC 결합.
2. 문제: SC/SCL/PBF 디코더는 비트 간 데이터 의존성으로 latency가 blocklength에 선형 증가, URLLC의 ms급 저지연 요구와 충돌.
3. 모델: memoryless AWGN, `yi=xi+ni`, `xi=1-2ci`, LLR `Li=2yi/sigma^2`.
4. 핵심기법1: OSD로 데이터 의존성 제거 - LLR 절댓값 정렬 permutation `lambda`로 MRI 컬럼 선정, 생성행렬을 systematic `Gs=[I,P]`화해 모든 비트를 병렬 one-shot 복원.
5. 핵심식: Proposition 1 - `M(c) = sum_{c_i != h(y_i)} |y_i|`를 최소화하는 코드워드가 복호오류 최소, 곱셈 없이 덧셈만으로 계산돼 HW 친화.
6. 핵심기법2: adaptive OSD - 후보 metric이 현재 최소값보다 크면 즉시 제거(`Mhat = D(e)+Rhat(e)`), Gaussian threshold의 error-floor 문제 회피.
7. 구현디테일: redundancy part `R(e)`를 `D(e)`로 근사 추정(식 18)해 재구성 없이 metric 산출, CRC로 조기종료.
8. 복잡도절감: Polar decomposition(Prop.2)으로 코드워드를 `2^k` outer + `2^(n-k)` inner 서브코드로 분해, SC 디코더가 soft input 생성 후 concatenated adaptive OSD가 복호.
9. 결과: 짧은 blocklength·고rate에서 CA-SCL(L=8/32) 대비 BLER 우위, 고SNR에서 clock cycle이 one-shot 수준(~1)으로 급감.
10. 한계: 완전 시뮬만(HW 미합성), Polar 부호·AWGN 특화, gate/throughput 수치 없음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Polar OSD 디코더 전체 | 대응 없음 (`decoder.cpp`는 LDPC min-sum) | Polar은 프로파일상 명시적 "대응 없음" |
| codeword searching metric (덧셈 기반) | 대응 없음 | Polar 코드워드 탐색 metric, LDPC iterative BP와 구조 무관 |
| Gaussian elimination / systematic Gs | 대응 없음 | OSD 고유 절차, QC-LDPC 순환행렬 처리와 무관 |
| CRC 조기종료 | `partial_CRC.cpp` / `full_CRC.cpp` | 개념만 유사하나 Polar OSD 종료조건이라 직접 이식 불가 |

적용 가치: 낮음 - LDPC가 아닌 Polar 부호 전용 병렬 OSD 기법으로, Prime ECC의 min-sum QC-LDPC 디코더/인코더/부호설계와 접점이 없음.
