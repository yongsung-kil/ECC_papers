### B. 알고리즘 요약

1. 시스템: AWGN+BPSK에서 binary linear block code `C(N,K)`, uRLLC용 short-block(eBCH `(128,64,22)`/`(128,22,48)`/`(64,16,24)`, length-32 LDPC·Polar도 실험).
2. 문제: OSD[7]는 order-`m` reprocessing에서 후보수 `Σ C(K,l)`로 복잡도 `O(K^m)`라 order↑ 시 재인코딩 폭증.
3. 기반: 신뢰도 `α_i=|r_i|` 정렬 후 π1·가우스소거·π2로 MRB(most reliable basis) `G=[I_K P]` 구성, TEP `e`를 MRB에 XOR 후 재인코딩해 WHD `D_e=Σα_i` 최소화.
4. 핵심1 세그먼테이션: phase-`l` TEP 집합 `S_l`을 `Q+1` 경계 `{β_0..β_Q}`로 `Q`개 세그먼트로 분할, 각 세그먼트는 sub-MRB 구간에 nonzero를 갖는 TEP만 포함.
5. 경계 적응결정: `α_{β_i}=(1/λ)E_{[1,β_{i-1}-1]}·f(α,D_min)`로 현재 `D_min` 기반 경계를 적응 이동, offset `f=D_min/E_{[1,N]}`.
6. 핵심2 discarding rule: 세그먼트 하한 `D^l_lower=L(1+τσ(α)E_{[K+1,N]}/E_{[1,K]})` 계산, `D_min<D^l_lower`면 남은 세그먼트 폐기.
7. stopping rule: `β_1≥K-l+1`이면 sub-MRB가 weight-`l` TEP 생성 불가로 판단해 즉시 종료·출력.
8. 복잡도: 세그먼트/하한 추가비용 `(K+1)mQ`는 무시 가능, 지배항은 정렬·가우스소거·`Na`회 재인코딩.
9. 결과: `(128,22,48)` eBCH에서 low-SNR 시 fast OSD[16] 대비 TEP 수 `1/3`, `(128,64,22)`는 절반 미만 `Na`로 동등 BLER; order-3에서 `Na` 640 vs fast OSD 1409(SNR=-5dB) 등.
10. 한계: HW 미설계, short-block 전용, LDPC는 length-32급만 시연(NAND급 long QC-LDPC 아님), 정정능력 향상이 아닌 복잡도 절감.
