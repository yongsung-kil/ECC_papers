> 총평: LDPC가 아니라 MLC NAND write voltage를 DE로 최적화해 채널용량을 올리는 채널측 연구 — LDPC(regular dv=3,dc=27, N=4096)는 용량 이득 확인용으로만 쓰이고 디코더 자체는 손대지 않아 Prime ECC 이식성 하.

### B. 알고리즘 요약

1. 시스템: MLC(2bit/cell) NAND flash, 4준위 write voltage μ=['11','10','00','01'], 비대칭 채널.
2. 문제: 기존 연구는 read voltage 최적화·ECC에 집중, write voltage 최적화가 미흡 — write voltage가 채널용량에 크게 영향.
3. 모델: 각 심볼 분포를 Gaussian/erf로 정의(`p11`~`p01`), noise `σ_n=0.00025(PE)^0.62`, RTN·cell-to-cell 간섭(`γ_y`,`γ_xy`) 반영, PE에 따라 분산 증가.
4. 핵심 기법: 목적함수 `f(μ)=max_μ I(X;Y)` (상호정보 최대화)를 differential evolution(DE)로 탐색, μ=[μ10,μ00] 최적화.
5. 식 의미: 채널용량 `C=max I(X;Y)`가 ECC 정정능력 상한이므로 용량↑이 곧 성능 여지↑.
6. 확장: DE의 mutation `m=μr1+F(μr2-μr3)`, crossover(CR), selection으로 세대별 수렴(약 40~50세대).
7. 구현 디테일: μ11=1.4, μ01=3.93 고정, Vpp=0.3, 개체수 90~100, F=1, CR=0.5.
8. 학습/최적화: DE 반복으로 exhaustive 대비 탐색공간 9×10^8→4500으로 축소(해상도 비의존).
9. 결과: 최적 write voltage가 fixed 대비 채널용량 우세, PE=50,000에서 gain 2.94%; regular LDPC(`dv=3, dc=27`, `N=4096`, BP 10 iteration)로 고PE에서 FER 개선 확인.
10. 한계: LDPC 디코더/HW 미설계·미변경, write voltage 최적화 시뮬만, 실측·컨트롤러 구현성 미확인.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NAND 채널 모델 / write voltage 분포 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | write voltage 최적화는 채널측(프로그램 전압)이라 Prime의 read offset/LLR threshold와 결은 다르나 채널 파라미터 관점 간접 연결 |
| 상호정보 기반 용량 최적화 | 대응 없음 | Prime ECC는 write voltage/DE 최적화 모듈 없음 (NAND 셀 프로그래밍은 상위 계층) |
| 용량 검증용 regular LDPC BP | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | Prime은 QC-LDPC min-sum 사용, 논문은 regular LDPC를 도구로만 사용(디코더 개선 아님) |

적용 가치: **낮음** — 기여가 LDPC 디코더/부호가 아니라 NAND write voltage 채널 최적화(DE)에 있어, Prime ECC 디코더·인코더 이식 요소가 없다. 채널 파라미터 튜닝 아이디어로만 참고.
