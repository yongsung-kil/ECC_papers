### B. 알고리즘 요약

1. 시스템: SMR/HDD, 4096 byte/sector, 128/130 RLL + LDPC 인코딩, PR1 채널, array head 3-reader + 2D-FIR(Nt=15) 등화 후 APP 디코딩.
2. 문제: 패턴 의존 매체잡음(예: "010","101") 때문에 APP→SP LDPC 반복 복호가 error floor에 빠짐.
3. 모델: 시스템잡음 AWGN, `SNRS=20log10(A/σS)`, 4 Tbit/in² 밀도, Voronoi 입자 매체 모델.
4. 핵심 기법: APP 디코더 LLR 시퀀스 `xk`를 3층 소형 신경망(`N1=4,N2=1,N3=2`)에 입력해 신뢰도를 재계산.
5. 식: 3층 출력 `yk(3)`은 softmax로 APP 판정 `tk`가 "1"/"0"일 확률 `p(tk|xk,w)`; 왜 필요한가 = 원 LLR보다 신뢰도 정밀 추정.
6. 확장: 최대우도로 가중치 `w` 학습, 오차함수 `E(w)=-lnL(w)`를 back-propagation으로 최소화(학습계수 α).
7. 구현 트릭: LLR 변조 규칙 - 신경망 출력 > `T1`이면 "1", < `T0`이면 "0", 그 외 보류; 하드판정과 일치하면 LLR에 `W` 곱함.
8. 파라미터: 실험으로 `T1=0.9, T0=0.1, W=1.6` 선정.
9. 결과: 제안 변조기가 정확 LLR 선택 확률 증가; error floor 회피, 무오류 SNR을 22.0→21.5 dB로 0.5 dB 개선 (`isp=15, iglobal=10`).
10. 한계: HW 미설계, 시뮬만, SMR PR1 채널·특정 패턴 잡음 가정에 강하게 종속, LDPC 부호/디코더 코어 미변경.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| APP-SP 사이 LLR 변조/재계산 | `channel.cpp` `Set_LLR_Th()` | soft-read LLR threshold 개념만 부분 대응, 신경망 재계산은 대응 없음 |
| 신경망 LLR 변조기 | 대응 없음 | Prime ECC는 NN 디코더/변조기 미보유 |
| SP(Sum-Product) 반복 복호 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Min-Sum 사용, 알고리즘 상이 |
| turbo 등화(APP↔SP) | 대응 없음 | Prime ECC는 채널 등화·turbo loop 미보유 |

적용 가치: **낮음** — SMR PR 채널·패턴잡음 특화 신경망 LLR 변조기로 NAND binary QC-LDPC 디코더/부호에 직접 이식할 요소가 없음.
