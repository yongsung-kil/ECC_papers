### B. 알고리즘 요약

1. 대상 시스템: 3D NAND flash controller의 FTL 레이어, LDPC ECC는 read 시 error 정정 블랙박스로만 사용(부호 스펙 명시 없음).
2. 풀려는 문제: RBER 상승으로 soft-decision sensing 단계가 늘어 read latency 증가, 기존 page-based refresh는 lifetime capacity를 크게 소모.
3. 핵심 가정: bit-alterable 3D NAND(p-channel/dual-channel)로 cell 단위 program/erase 가능, error 패턴은 sparse, 정규분포 error 모델.
4. 핵심 기법 1단 - `read speed tracking`: 각 page의 초기 read speed(target read speed)를 program 직후 verify latency로 기록.
5. 핵심 식/판단: `speedDiff = targetReadSpeed - currentReadSpeed`가 1 soft-sensing level latency보다 크면 rewrite-required 표기(current/target/average 3속도 비교, Algorithm 1).
6. 핵심 기법 2단 - `sensing latency reduction`: rewrite-required page를 idle time에 bit-level rewrite로 일부 error bit만 제거해 sensing level을 낮춤(예: level6->5).
7. 구현 디테일: subpage(64B, cache line) 단위 rewrite + refresh pivot을 circular 패턴으로 이동해 intra-page wear leveling.
8. bit comparator(HW-assisted, ref[4])로 subpage별 error bit 수 계산, correctable = bit error - refresh-failed(permanent) error.
9. 결과: read latency 최대 25.22% 개선(평균 12.64%), sensing latency 최대 35.13% 감소, rewritten bit 평균 40.39% 감소(page-based 대비).
10. 한계: LDPC 디코더/부호 자체는 손대지 않음, flash simulator + 실trace 시뮬만, HW 미설계, sensing level-error 관계는 선행연구 테이블 인용.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC sensing level / soft-read latency | 대응 없음 | FTL 상위 관점의 sensing 횟수만 다룸, [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) LLR threshold와 개념적 접점만 있고 알고리즘 기여 없음 |
| bit-level rewrite / refresh 스킴 | 대응 없음 | FTL/컨트롤러 정책 레이어로 ECC 엔진 밖 |
| bit comparator (error bit 식별) | 대응 없음 | re-encode 후 XOR 비교, 디코더 내부 아님 |

적용 가치: **낮음** - LDPC를 latency 소비원으로만 참조하는 FTL 스킴이라 Prime ECC의 부호설계/디코더/HW 어느 레이어에도 이식할 요소가 없음.
