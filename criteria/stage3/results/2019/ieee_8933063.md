### B. 알고리즘 요약

1. 대상은 STT-MRAM용 short binary protograph LDPC 부호, 정보길이 520bit, rate `0.91/0.83/0.72` 계열(rate-compatible).
2. 문제: 기존 고정rate BCH/EG-LDPC는 worst-case raw BER로 설계돼 redundancy 과다, adaptive ECC는 인코더/디코더 다중화로 면적·전력 증가.
3. 채널 모델: STT-MRAM cascaded 모델(BAC write/read-disturb + Gaussian mixture resistance), 비대칭이라 `σ0/µ0`로 threshold 측정.
4. 핵심 기법: i.i.d. channel adapter(`vi=ui·(1-2ti)`)로 비대칭 채널을 대칭화 후, 대칭화 채널용 modified P-EXIT로 decoding threshold 산출.
5. 핵심 식: 대칭화 채널 입력 LLR `v`의 PDF를 2-항 Gaussian-Mixture로 EM 피팅해 VN/CN MI를 `Ivn`(식(3)) 갱신 → 모든 VN의 MI→1 되는 최대 `σ0/µ0`를 threshold로 정의.
6. 확장: AWE로 normalized log AWE `r(δ)`와 TMDR(second-zero crossing) 계산, 양수면 선형 최소거리 성장→낮은 error floor 보증.
7. 구현 디테일: short code에선 P-EXIT/AWE 단독 신뢰도 낮아 L(=30)개 후보 생성 후 실제 시뮬로 waterfall+floor 최적 부호 선택(combined guideline).
8. 부호구성: rate-0.91 daughter code(base matrix `1×11`, VN degree≤5, degree-2 VN 1개로 4-cycle 제거) 후 VN/CN 동시 추가하는 code extension으로 rate 계열 생성(단일 인코더/디코더).
9. 결과: AR4JA·고정rate QC-LDPC 대비 `BER=1e-6`에서 rate0.91 `σ0/µ0` 2%+ 여유 개선, error floor(~1e-5) 극복, 평균 iteration 감소로 read latency 개선. 디코더는 RB-MS(정수/논리 연산).
10. 한계: HW 미설계, STT-MRAM 비대칭 채널 특화 설계, 짧은 정보길이(520bit) 시뮬만.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| protograph base matrix / code extension 부호설계 | `ecc_top.cpp` `Load_PCM()` | H-matrix/PCM 로드 지점이나 우리는 고정 QC-LDPC라 재설계·재검증 부담 큼 |
| RB-MS(min-sum) 복호 | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 계열로 이미 보유(중복 기여) |
| soft-read LLR / 3-bit 양자화, 채널 대칭화 | `channel.cpp` `Set_R_Offset()`, `Set_LLR_Th()` | 비대칭 메모리 채널 대칭화·LLR 생성, NAND read 비용 고려 필요 |

적용 가치: **중간** — binary protograph LDPC의 error-floor 보증(AWE/TMDR) + rate-compatible 설계 발상은 참고가치가 있으나, STT-MRAM 비대칭 채널 특화이고 우리 QC-LDPC 구조 고정·min-sum 이미 보유라 직접 이식 이득은 제한적.
