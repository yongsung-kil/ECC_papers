### B. 알고리즘 요약

1. 대상은 fully-connected `3×n`(dv=3) 또는 `4×n`(dv=4) base graph를 cyclic lifting(CPM 크기 N)한 protograph binary QC-LDPC, girth `g=8` 고정.
2. 풀려는 문제: exponent matrix 탐색공간(`N^{mn}`)이 방대하고 trapping set 분포와의 관계가 복잡해 error-floor를 직접 최적화하기 어렵다.
3. 핵심 가정/모델: BI-AWGN 채널에서 error-floor의 주범은 leafless elementary trapping set(LETS), LSS(layered superset) 특성으로 8-cycle에서 dot_m 확장으로 성장.
4. 핵심 기법(dv=3): 8-cycle을 check node 행블록 수로 Type-A(2행블록)/Type-B(3행블록)로 구분, Type-A를 금지하고 모든 Type-B를 LSS-nE로 만들면 `a≤8, b≤3` 범위 LETS가 제거됨(Theorem 1).
5. 핵심 의미: LSS-nE(cycle 밖 변수노드가 degree-1 check와 최대 1개만 연결)면 Lemma 1에 의해 그 cycle의 dot_m 자식 trapping set이 그래프에 존재 불가.
6. 핵심 기법(dv=4): 8-cycle의 4개 degree-2 check node를 4개 서로 다른 행블록에 두면 모든 8-cycle이 LSS-nE가 되어 `{a≤11,b≤2}∪{a≤10,b≤4}∪{a≤9,b≤6}∪{4<a≤7,b≤8}` LETS 제거(Lemma 2).
7. 구현 디테일: 각 8-cycle에 대해 (a) check가 2행블록뿐인지, (b) cycle 밖 공통 이웃 변수노드 존재 여부만 검사하면 되어 큰 n·N에도 적용 가능.
8. 하한 유도: pigeonhole로 lifting degree 하한을 도출 - dv=3은 `N≥n(n-1)+1`(기존[8]의 `2n(n-1)+1`보다 작음), dv=4는 `N≥3n(n-1)+1`.
9. 결과: 3×5,3×6,3×10에서 lifting degree 27/41/165로 기존[8]의 41/63/251보다 짧은 부호를 얻고, 5-bit min-sum(clip 7.5, iter 50) FER에서 error-floor 영역 우위.
10. 한계: HW 미설계, MATLAB search가 코드당 2127~24485초로 매우 느림, dv=3/4 저차수 protograph에 국한.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| exponent matrix(P) 설계 → H-matrix | ecc_top.cpp Load_PCM() | 우리 base+shift PCM 로드 지점, 논문식 8-cycle 제약을 shift 설계에 반영 가능하나 base graph·dv 상이 |
| girth-8 / trapping set 제거(error-floor) | 대응 없음 | 우리 코드에 부호설계 최적화 루틴 없음(H-matrix 외부 고정), 이식 시 재검증 부담 |
| 5-bit min-sum 디코딩(성능 평가) | decoder.cpp LDPC_Decoder(), CNU_Update_New_Mag() | 성능 검증에 사용한 min-sum은 우리와 동종이나 논문 기여 아님 |
```

적용 가치: 중간 — error-floor를 부호설계로 낮추는 원리는 우리 관심사(고rate QC-LDPC 부호설계)에 부합하나, dv=3/4·저rate protograph 한정이고 Prime ECC는 특정 QC 부호 고정이라 exponent matrix 재설계·재검증 비용이 큼.
