### B. 알고리즘 요약 (10줄 내외)

1. 대상 부호: M차원 SPC product code (constituent `Pk(n=k+1,k)`의 Kronecker product), 예제 `(5,4)^2`,`(7,6)^2`,`(3,2)^3`,`(5,4)^3`, `dmin_M=2^M`, BPSK/AWGN.
2. 문제: OSD가 short code에서 ML에 근접하나 복잡도(order I에서 후보 `Nc=Σ C(k,i)`)가 실시간 구현에 과대 - 특히 병렬화 불가한 Phase A(reliable basis+생성행렬).
3. 배경식: 최소 order `Imin = ⌈dmin/4⌉` (식2) - dmin≤4면 `Imin=1`로 후보 매우 적음.
4. 기법 1(Phase A 단순화): product code의 모든 행/열/축이 parity check이므로 병렬로 각 벡터의 least-reliable bit 삭제 → 남은 비트로 MRB 구성 (Algorithm A, 2D는 `n^2−α` 비트).
5. 기법 1b(SRB): 각 행의 least reliable 2비트로 Simplified Reliable Basis 직접 구성 (Algorithm B) - 열 swap·행렬 축약 불필요, 자동 information set.
6. 핵심 관찰: 1D SPC의 SISO OSD LLR 출력식(식9-10)이 LDPC bipartite 그래프의 min-sum CN update(식11)와 동일한 reprocessing.
7. 기법 2(3D): 3n^2 벡터 병렬로 least-reliable 삭제 후 order-2 OSD (Algorithm C), dmin=8 → Imin=2.
8. 기법 3(iterative): 3D를 2D subcode(dmin≤4, Imin=1)로 분해, SRB 기반 order-1 SISO OSD(Algorithm D) 다수 병렬 사본이 LLR을 반복 교환.
9. 결과: Algorithm A는 원 OSD=ML과 일치, B는 약간 sub-optimal이나 우수; iterative D는 ML 대비 gap 있으나 successive cancellation 대비 약 0.5dB 개선.
10. 한계: HW 미설계, 시뮬만, SPC product code 전용, short block 대상, 후보/BER 절대수치 등 대부분 그림에만 존재.
