### B. 알고리즘 요약

1. 시스템: NLM 비대칭 Flash 채널(sub-20nm MLC, 3 reads, sector 512B), OD 코드를 MD로 결합. rate `≈0.82~0.90`, length `4240~15162` bits.
2. 문제: OD 코드는 error floor에서 absorbing set(AS)이 성능을 지배하는데, 기존 MD 기법[9]은 짧은 사이클만 줄일 뿐 AS를 직접 겨냥하지 않음.
3. 모델: unlabeled 그래프(edge weight=1) 기준, `(a,d1)` unlabeled elementary absorbing set(UAS)을 detrimental object로 정의.
4. 핵심기법: `M-1`개 보조행렬 `X_ℓ`로 NZ entry를 relocate(매핑 `R`), 소수 `M`>2 사용. UAS의 cycle basis `B_c`(basic cycle `n_f`개)로 활성/비활성 판정.
5. 핵심식: 사이클 활성 조건 `Σ(-1)^w R(E_iw,jw) ≡ 0 (mod M)` — basic cycle 전부가 이 식을 만족해야 UAS가 살아남음(Theorem 1).
6. 확장: UAS를 비활성화하는 relocation 비율 `Fnou = 1 - 1/M^{n_f}` 유도, 사이클 기준 대비 절감 `S1` 산출(예: 48%).
7. 구현 디테일: 빠른 인코딩/디코딩 위해 entry 대신 circulant 단위로 relocate(Algorithm 1), 보조행렬 간 NZ circulant 균형(Step 16).
8. 최적화: Algorithm 1이 특정 `(a,d1)` UAS를 표적으로 vote-tally 방식 반복 relocation, Theorem 1로 활성 목록 갱신.
9. 결과: 약 7%(resp. 4.5%) circulant relocation만으로 (4,2)/(4,4) UAS 인스턴스를 4218/3392 -> 0으로 제거. NLM Flash에서 waterfall ~1200, error floor ~1800 P/E cycle 이득.
10. 한계: HW 미설계, 시뮬(FFT-QSPA)뿐, 실험 코드가 SC/MD·non-binary 중심이라 binary QC-LDPC 일반화·재검증 필요.
