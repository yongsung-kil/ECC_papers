### B. 알고리즘 요약

1. 시스템: 3-way(확장 M-way) 무선 relay, BPSK, Rayleigh block fading + AWGN, 노드 A/B/C가 relay R 경유 정보교환.
2. 문제: relay 채널이 이질적이라 모든 채널에 동일부호 쓰면 저품질 채널은 보호부족·고품질 채널은 spectral efficiency 손해 → 불균등보호(UEP) 필요.
3. 모델: `[n×(n+k1+k2)]` binary LDPC를 puncturing으로 rate 다른 두 부호 생성, 저품질 채널엔 full-length·고품질 채널엔 punctured 할당(`k1,k2`는 채널품질에 반비례).
4. 핵심기법 1단: relay에서 hard-decision joint 복호로 `ca⊕cb`, `cb⊕cc` 산출, 초기화-반복 message passing-finalization 3단계.
5. 핵심 식: 수신 4-constellation 심볼확률 `Pi=Pr{Yab|Sab=S(i)}·C`(식6)로부터 `Pr(ca|A)`,`Pr(cb|A)` 분해(식7).
6. 확장: node B 정보를 저rate 부호로 이중보호, `ca⊕cb`·`cb⊕cc` 두 디코더 외에 B용 세 번째 확장-LDPC 디코더를 추가해 결합 soft info로 추가 이득.
7. 구현 디테일: systematic 부호일 때 두 디코더의 B soft info를 `LLRy(cb|A)`로 결합(식8), `n+k1+k2` 차원 soft 벡터를 cb 디코더 입력으로.
8. PNC 매핑: finalization에서 `Cab=argmax_i Pi(j)` XOR 매핑 후 parity check, 전 체크 만족 또는 최대 iteration에서 종료.
9. 결과: 고rate 부호 대비 약 3dB coding gain(단 full-length 대비 3dB 열세), 5 iteration 내 수렴, code rate<0.33에서 20%↑ throughput 개선(상한 33.33%).
10. 한계: HW 미설계, 무선 relay/PNC 특화 시뮬만, 짧은~중간 부호(63×37/255×175)·quasi-static block fading 가정.
