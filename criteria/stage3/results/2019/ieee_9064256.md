### B. 알고리즘 요약

1. 시스템: time-invariant left-terminated SC-LDPC, sub-matrix `H_i` dim `mS×nS`, syndrome former memory `P`, weakly coupled(`P=1,2`) 집중; 예제 `m=6,n=30,S=400`, rate `R=4/5`, 길이 `N=Ln`.
2. 문제: SC-LDPC의 semi-infinite 구조라 block 인코딩/복호가 부적합하고 error-floor를 낮추려면 degree 분포와 sub-matrix girth를 함께 최적화해야 함.
3. 인코딩: `H_0`의 마지막 `mS` 열을 단위행렬로 두어 full-rank 보장, 재귀식(식3~5)으로 parity `v_t^1`를 shift register로 계산 — `(nS·P)+(nS-mS)` 메모리 셀 필요.
4. degree 분포 최적화: degree-3/degree-6 VN 비율 `a_v,3`,`a_v,6`을 조정(Tab.I: d3.9/d3.6/d3.3), waterfall threshold를 조절.
5. Split variable nodes 알고리즘: 먼저 sub-matrix와 같은 차원의 QC-LDPC `H_all`(shift값 `DW_i`) 구성 → girth profile `g_GP`를 열단위 정의(식7).
6. `DW_i`를 무작위 탐색해 `g_GP` 비교(식8 우열 기준)로 최적 행렬 선택 후, `DW_i`를 sub-matrix들에 균등·무작위 분배하되 `H_0`의 단일 대각 구조 유지, CN degree 균일화.
7. 결과 girth: 예제에서 `H_all`의 girth 8이 분할 후 SC-LDPC(H0,H1,H2)에서 girth 10으로 증가.
8. Sliding window 디코딩: window size `WmS×(W+u)nS`, LLR 레지스터 초기화 30, 갱신식 `r_t^i=(1-α)r_t^i+α s_t^i`, 유효 iteration `I_e=I_w·W`; layered 디코딩에 Log-BP/scaled-MS/offset-MS 선택.
9. 결과: d3.9 SC-LDPC가 regular d3 대비 ~0.2dB gain(Fig.2); d4 SC-LDPC(Iw=5)가 d3 LDPC-BC 대비 0.2dB gain, offset Min-sum(factor 0.4)+`α=1`이 Log-BP 대비 0.05dB 손실로 최적; d4 SC-LDPC(Iw=2)가 d3 LDPC-BC 대비 복잡도/latency 각 1.85/2.5배 절감.
10. 한계: HW 미설계(gatecount/throughput 없음), AWGN·QPSK 가정, weakly coupled SC-LDPC 한정, rate-compatible/HARQ는 future work.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| girth 최적화 위한 QC shift값(DW_i) 탐색 | ecc_top.cpp Load_PCM() + H-matrix | 개념만 참고 가능, SC-LDPC 구조라 직접 이식 부담 큼 |
| sub-matrix degree 분포 최적화 | ecc_top.cpp Load_PCM() (부호설계) | Prime ECC는 특정 QC-LDPC 고정, 재검증 부담 높음 |
| shift-register 기반 SC 인코딩 | encoder.cpp LDPC_encoder_4KB() | dual-diagonal block 인코더와 이질(컨볼루션 구조), 대응 약함 |
| Log-BP/scaled·offset Min-sum, layered | decoder.cpp LDPC_Decoder(), CNU_Update_New_Mag() | 이미 보유(min-sum), 중복 |
| sliding window 디코딩 | 대응 없음 | block QC-LDPC엔 window 개념 없음 |
```

적용 가치: 낮음 — SC-LDPC 컨볼루션 부호설계·windowed decoding이 핵심이라 block QC-LDPC 기반 Prime ECC에 직접 이식이 어렵고, min-sum·layered는 이미 보유. girth 최적화용 shift 탐색 착상만 부분 참고 가능.
