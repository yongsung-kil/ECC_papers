### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 위성 심우주 통신용 CCSDS 표준 QC-LDPC 인코더(AR4JA code group, 외부 BCH + 내부 QC-LDPC), Gbps급 고throughput 목표.
2. 문제: 기존 SRAA(shift-register-add-accumulator) 인코더는 b-bit 레지스터 2세트로 자원·전력 낭비, 단일 비트 입력만 처리.
3. 핵심 구조: generator matrix `G`의 quasi-cyclic성을 이용, block size(예 32)에 맞춰 `G`를 row vector로 압축해 `GF(2)`상 곱셈-누산으로 인코딩.
4. 핵심 기법: RCE(Recursive Convolutional Encoder) 유닛 - `GF(2)` FIR 필터와 유사하게 생성행렬 각 행을 순차 곱해 부호 계산, SRAA 대비 상단 shift register 제거하고 ROM 직결.
5. 병렬화 트릭: 1bit 대신 4bit씩 이동, `GF(2)` 곱셈-누산 유닛 4배로 4개 데이터 동시 인코딩(critical path 부담 적음).
6. 재구성: RCE 유닛을 cascade, N개 RCE 마지막 비트가 bus 형성, MUX로 각 RCE 첫 비트 할당해 다양한 주기길이 shift register 구성.
7. 구성 결정: RCE 구성은 code length `k`·rate `r`로 결정. AR4JA는 확장인자 7종(l=32~2048, 2배 관계), 단일 코어에 64 RCE 유닛으로 7개 code type 지원.
8. 부수 모듈: 주소생성 제어(ROM 주소·rate별 enable), ASM(frame sync marker), 출력제어(sync flag→system bit→parity), 입력 FIFO 상태머신.
9. 결과: Xilinx VC690T FPGA, 기존 대비 register 39.4%·LUT 24.4% 절감, throughput 약 3배 증가, 최대 약 20Gbps.
10. 한계: 인코더 전용(디코더·정정성능 무관), CCSDS AR4JA 특화, gate count·공정 미제시(FPGA 자원만).

### C. Prime ECC 관련 모듈 핀포인트

| generator matrix 압축·병렬(4bit) 처리 | `encoder.cpp` (dual-diagonal 고정) | 낮음 - 인코딩 방식 근본적으로 상이 |
| AR4JA 다중 rate 재구성(MUX cascade) | 대응 없음 | 우리 부호 QC-LDPC 고정, 다중 code type 미지원 |
| 디코더/정정성능 요소 | 대응 없음 | 인코더 논문, C섹션 성능 대상 아님 |

적용 가치: **낮음** - target이 encoder이고 generator-matrix 기반 RCE 구조가 Prime ECC의 dual-diagonal 인코더와 방식이 달라, 자원 절감 아이디어 외에 직접 이식할 요소가 없음.
