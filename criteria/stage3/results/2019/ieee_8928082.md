### B. 알고리즘 요약

1. 시스템/채널: BSC(hard-decision), BP 복호. 실코드는 length `16384`, rate `5/6` irregular LDPC 4종(code A~D)을 IPEG로 구성.
2. 풀려는 문제: 표준 LDPC는 노드 연결이 무작위라 모든 노드가 유사 신뢰도 → error-floor 발생, 부분 복호 불가.
3. 핵심 가정/관찰: all-0 송신 가정 하에 고차수 변수노드는 더 신뢰(edge 추가 시 `v_i→j`↑), 저차수 체크노드는 더 신뢰(`tanh(x)∈(-1,1)`이라 edge 제거 시 `u_j→i`↑).
4. 핵심 기법 1단: **연결 극화** - 고차수 변수노드↔저차수 체크노드, 저차수 변수노드↔고차수 체크노드를 연결해 polar code 유사 극화. 노드를 layer로 나누어 같은 layer 및 하위 layer에만 연결.
5. 핵심 식: sum-product `v_i→j = v_i,0 + Σu_j'→i`, `tanh`-rule(식5), `R(a,b)=2tanh^-1(tanh(a/2)tanh(b/2))` - polarization으로 layer별 신뢰도 차등화.
6. 핵심 기법 2단: **층별 복호** - 고층(고신뢰) 비트를 먼저 복호 후 공통 체크노드로 보내는 LLR을 `∞`로 설정, 하위 layer의 간섭을 제거(polar SIC 유사).
7. 구현 디테일: 차수별로 별도 degree polynomial(`ρ_i(x)`, `λ_j(x)`) 사용, 고층 수렴 후 상위 체크노드로의 edge를 잘라 간섭 방지.
8. 학습/최적화: **개선 Density Evolution**(Algorithm 1) - 차수별 LLR 분포를 개별 추적(i.i.d. 가정 폐기).
9. 결과: DVB-S.2 대비 저crossover 영역에서 error-floor 크게 낮춤(code A/B는 error-floor 미관측). layer 수↑ 시 성능↑(단 error propagation 위험). iteration당 복잡도 소폭 증가.
10. 한계: BSC hard-decision만, HW 미설계·시뮬만, gate/throughput 수치 없음. 저차수(degree-4) 변수노드가 프레임에러 대부분 유발.
