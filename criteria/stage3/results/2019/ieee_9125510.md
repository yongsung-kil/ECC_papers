### B. 알고리즘 요약

1. 시스템: 광섬유 통신용 26.3% overhead concatenated FEC, 내부 LDPC-CC(convolutional) + 외부 BCH, BPSK/AWGN 채널, 수신 6-bit LLR soft decision.
2. 문제: LDPC-CC의 복잡한 convolutional 구조 때문에 ultra-low(1e-15) BER의 error-floor를 이론/수치로 예측 불가 → 고속 에뮬레이션 플랫폼 필요.
3. 모델: 25% overhead LDPC-CC를 regular QC-LDPC block code(`dv=3`, `dc=15`)에서 matrix unwrapping으로 유도, 파생 LDPC-CC의 girth는 원 block code 이상.
4. 핵심기법: parity-check matrix를 base matrix `Hi,j`의 layer로 구성(주기 `T=dv`, 반semi-infinite), 각 `Hi,j`는 5개 `z×z` circulant permutation matrix로 구성, code length `L=dc·z`.
5. 디코더: sliding-window decoding buffer(`B` codeword), buffer를 top→bottom 스윕, 한 스윕당 iteration 수 = buffer size `B`, revised min-sum layered decoding 채택.
6. 확장: 다중 buffer sweep로 성능 개선 가능(대신 복잡도/latency 증가).
7. 구현 디테일: 32-lane 병렬 BCH 인코더, block interleaver(`m·2309-bit`) + convolutional interleaver, 48-bit noise sample, SNR 해상도 0.005dB.
8. 최적화 절차: FPGA 에뮬레이션으로 code length/girth/buffer size/interleaving/outer code 강도를 progressive하게 스윕 비교.
9. 결과: girth(6/8/10)는 waterfall·error-floor 개선 미미, code length 증가는 비효율(21배 증가에 threshold 4.19e-2→4.3e-2), buffer size는 waterfall만 개선; 최종 4.19e-2 pre-FEC threshold로 1e-15 post-FEC 달성.
10. 한계: LDPC-CC 특화(NAND block code 아님), 실측 아닌 FPGA 에뮬레이션, error-floor 억제를 outer BCH concatenation에 의존.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LDPC-CC decoding (revised min-sum, layered) | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()` | min-sum은 공유하나 convolutional sliding-window 스케줄이 block과 상이 |
| 6-bit LLR soft decision | `channel.cpp` `Set_LLR_Th()` | soft-4bit+로 Prime ECC(최대 3SD) 미지원 범위 |
| circulant permutation base matrix | `ecc_top.cpp` `Load_PCM()` | QC 구성은 유사하나 CC unwrapping 구조는 대응 없음 |
| sliding-window buffer / concatenation(BCH) | 대응 없음 | Prime ECC는 block 단일 LDPC, outer BCH concatenation 없음 |

적용 가치: **낮음** — LDPC-CC(convolutional) + 외부 BCH concatenation + 6-bit soft는 NAND block binary QC-LDPC 및 Prime ECC 지원범위(hard/2SD/3SD)와 구조가 달라 이식 실익이 없다.
