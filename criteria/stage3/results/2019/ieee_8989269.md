### B. 알고리즘 요약 (10줄 내외)

1. 시스템/코드: BI-AWGN 채널, rate-1/2 polar code `P(128,64)`, `P(256,128)`, `P(512,256)`를 대상으로 하고 5G 3GPP polar 신뢰도 순서를 참조 부호로 사용.
2. 문제: polar code를 off-the-shelf LDPC 디코더(SPA/flooding BP)로 복호하면 Arıkan BP 대비 성능 저하 발생 — 부호와 LDPC parity-check 행렬 설계가 LDPC-style 스케줄을 반영하지 않기 때문.
3. 핵심 해석: polar 편극행렬 `G_N`의 frozen index 열로 `Hdense` 구성 → Arıkan factor graph를 두 집합(VN `V`, CN `C`)의 bipartite로 재그룹화 → pruning(약 80% 축소)으로 `Hpruned`(sparse) 획득, 잔존 "hidden VN"을 punctured parity VN으로 해석.
4. 핵심 기법: Genetic Algorithm(GenAlg)으로 정보/frozen set `A`와 대응 pruned H-matrix를 LDPC-style BP 디코더 및 채널을 루프에 넣은(decoder-in-the-loop) 상태로 공동 최적화. 목적함수는 BER 또는 BLER.
5. 제약 반영: punctured VN 집합의 연결을 설계에 고려, 부호와 디코딩 그래프를 동시에 튜닝.
6. 부수 디테일: 저복잡도 polar 인코더로 부호화하되 복호는 conventional LDPC BP로 수행하여 기존 HW·툴 재사용, soft-output 제공.
7. 학습/최적화: GenAlg 세대별 population 진화(Fig. 5b/c에 BER 수렴 곡선), design SNR별(2.5/3 dB) 별도 최적화 부호 도출.
8. 분석: 최적화 부호는 girth-6로 동일하나 girth-6 cycle 수 감소(5G 2234 → 최적화 1825), H-matrix 크기 축소(503×633 → 365×493)로 punctured VN 최소화, `dmin`은 동일(8)하되 최소중량 codeword 수 `Admin` 감소(P(128,64): 304→170).
9. 결과: `P(128,64)`에서 BLER 10⁻² 기준 [14] 원부호 대비 0.25 dB, `P(256,128)` 약 0.2/0.1 dB, `P(512,256)` BER 10⁻³ 약 0.4 dB 이득. 고SNR에서 Arıkan BP 상회, 표준 LDPC(CCSDS/AR3A/ARJA) 및 plain SCL 접근/상회.
10. 한계: HW 미설계(시뮬만), 5G/AWGN 무선 특화, polar→LDPC-like 해석 특수 구조로 NAND용 고전 QC-LDPC와 무관.
