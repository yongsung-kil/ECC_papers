### B. 알고리즘 요약 (10줄 내외)

1. 대상 부호: ATSC 3.0 raptor-like LDPC, `length=16200`, `rate=3/15`, BI-AWGN/QPSK, `itermax=50`.
2. 문제: LDM core layer/over-engineered network처럼 SNR이 충분히 높은 저전력 IoT/handheld 수신기에서 디코더 복잡도(ASIC 면적 ~50%)를 낮추고 싶음.
3. 모델: PCM은 core matrix `Hcore`(A,B) + `Hc`(C) + identity `HI`(degree-one parity node) 구조.
4. 핵심 정리: degree-one VN에 연결된 CN이 core VN으로 보내는 메시지는 iteration 무한대에도 magnitude가 `|Lˆv1|`로 bounded (φ(x) 단조감소 이용, 식 9-10).
5. 따라서 초기 LLR이 크면(고SNR) 해당 CN 연산은 iteration→∞ 상황과 유사하므로 생략 가능.
6. 기법 1: skip 집합 `Γ`에 속한 CN의 update 연산(`Lvi→cj`, `Lcj→vi`)을 건너뛰는 Algorithm 1 (식 11-13).
7. 기법 2: Algorithm 2로 `Γ` 선택 - 최대 degree CN부터 skip 시도, `DE(PCM\Γ)==1`(수렴) 유지되는 한 추가.
8. 근거식 의미: threshold snr(density evolution로 산출)이 FER 10^-4 목표 SNR과 근접하므로 threshold 기반 skip이 타당.
9. 결과: SPA/NMSA 모두 적용, 복잡도(RA/LUT/XOR/CP) SNR에 따라 0.1%~80% 감소(고SNR일수록↑), BER은 10^-6~10^-7 유지.
10. 한계: HW 미설계, 시뮬만, raptor-like degree-one VN 구조 전제, 정정능력 개선은 아님(복잡도만 절감).
