### B. 알고리즘 요약

1. 시스템: SSD를 블록 캐시로 쓰는 분산 스토리지에서 오류허용(비디오/멀티미디어) 애플리케이션을 위한 근사 SSD 캐시 ASCache 제안, 8/9 rate LDPC 정정흐름 가정.
2. 문제: RBER 상승 시 LDPC 정정(hard→soft 6레벨→RAID) 오버헤드로 read 지연 급증, 정정실패 시 불필요한 cache miss 발생.
3. 모델: 블록 RBER를 `RBER=G(P/Es)+H(T_retention)`(G 지수·H 선형)로 근사, 블록 첫 페이지 RBER를 상한 `RBERU`로 pessimistic tracking.
4. 핵심기법1(Error-Aware Space Management): LDPC 정정흐름 임계(hard≈1e-3, soft 6레벨 1e-3~1.3e-2, 초과 시 RAID)에 따라 블록을 8개 reliability level 그룹으로 분류·매핑.
5. 식 의미: `RBERU`(+bias δ)로 블록단위 오류 특성화 → 앱의 허용 `AP_BER`와 대조해 정정 우회 가능여부 판단.
6. 핵심기법2(Approximate Caching): `AP_READ/AP_WRITE` 인터페이스로 `AP_BER≥RBERU`면 정정 없이 오류포함 데이터 반환, 정정실패를 cache miss가 아닌 hit로 흡수.
7. 구현 디테일: reference page로 GC/idle 시 `RBERU` 갱신, write-back/through/around별 백엔드 end-to-end 무결성 보장 전략, level 기반 소프트디코딩 종료점 예측.
8. 학습/최적화: 없음(휴리스틱 정책).
9. 결과: `AP_BER>0.011` 시 평균 read 지연 약 30% 감소, codeword failure rate 0.3에서 read-error 기인 cache miss 최대 52% 제거(qemu open-channel SSD 에뮬).
10. 한계: LDPC 디코더/부호 자체 미수정(블랙박스), HW 미설계, 에뮬레이터 시뮬만, 오류허용 앱에 한정.
