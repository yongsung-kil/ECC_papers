### B. 알고리즘 요약 (10줄 내외)

1. 시스템: 2GB MLC NAND Flash + FTL(wear-leveling/GC/address mapping). 페이지레벨 매핑 스키마의 전원손실 복구(PLR) 기법을 다룸.
2. 문제: 시스템 크래시·급작 전원차단(power outage) 시 RAM에 있던 매핑 정보 손실 → 기존 In-Page/In-Block/Hybrid Backup, A-PLR, HYFLUR, C-HYFLUR 등은 복구시간·매핑 관리비용에 단점.
3. 배경(ECC): NAND ECC로 BCH(512B당 13bit, 또는 1024B당 4bit)와 LDPC를 언급하나 이는 관련연구 서술일 뿐 본 논문의 기여 아님.
4. 핵심 기법: spare area를 `ECC` / `mapping information`(Block info + ASN=allocation sequence number) / `reserved`로 분할. 새 페이지 write 시 LPN을 spare에 기록.
5. 복구 절차: 전원손실 시 ASN 최상위 블록부터 내림차순으로 페이지 매핑 엔트리 복원. spare area의 write 흔적(LPN)으로 매핑 엔트리 복구, 흔적 없으면 free block.
6. 구현: 블록당 128 페이지 중 대표 5개 페이지만 스캔(전체 스캔 회피)해 맵테이블 복구 시간 단축.
7. 실험 가정: page 4224B(user 4096B + spare 128B), block 128 page, 4096 block, 2GB. BCH(1024B당 4bit ECC), ECC용 spare 63B. tD=60µs, tR=25ns.
8. 결과(이론 계산): In-Page 총 복구 31,510ms, In-Block 80ms, 제안기법 1,270ms. read time은 In-Page 수준, 총 시간은 경쟁력 있음.
9. 검증: 실측·시뮬 없이 datasheet 파라미터 기반 수식 계산(tsum)만 제시.
10. 한계: LDPC/ECC 부호설계·디코딩 기여 없음, HW 미설계, 순수 FTL 맵복구 시스템 논문.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| FTL PLR / 맵테이블 복구 | 대응 없음 | FTL 계층 기능으로 ECC 부호·디코더와 무관 |
| spare area 분할(ECC 저장) | 대응 없음 | ECC 저장공간 배치, 부호설계 아님 |
| BCH/LDPC 언급 | 대응 없음 | 배경 서술뿐, 알고리즘 기여 없음 |

적용 가치: 낮음 — FTL 전원손실 복구 시스템 논문으로 binary QC-LDPC 부호설계·디코더 어느 것도 건드리지 않아 Prime ECC 이식 접점이 전무하다.
