> 총평: 모바일 GPU 상 layered Min-Sum LDPC의 다중스트림 SW 구현(실측 최대 0.23Gbps)으로 알고리즘/정정능력 기여 없이 GPU 병렬화·프레임 스루풋이 핵심이라 NAND HW 디코더에 이식할 지점이 적음.

### B. 알고리즘 요약

1. 시스템: 모바일 SoC(NVIDIA Tegra K1: Cortex-A15 ARM 4코어 + Kepler GK20a GPU 192 CUDA core)에서 SW LDPC 디코딩. 부호 `(576,288)`~`(9972,4086)`, rate 대략 1/2 및 0.41.
2. 문제: 전용 HW(ASIC/FPGA)는 저지연이나 유연성·개발비 문제, x86 SIMD는 전력이 커 모바일 부적합, 순수 ARM 전용 디코더는 다른 앱과 자원 경쟁.
3. 모델: LDPC를 H-matrix/Tanner graph로 표현, 표준 TPMP 2-phase message passing과 그 변형인 Min-Sum(Algorithm 1)을 기반.
4. 핵심기법1: Min-Sum의 CN→VN, VN→CN 분리 갱신은 병렬성이 나빠, horizontal layered decoding(Algorithm 2)로 갱신 정보를 즉시 재사용해 수렴 가속(선행연구서 2배 수렴, 메모리 45~50% 절감).
5. 핵심식/구조: 병렬화 2단계 - (1) CN 내부 atomic 병렬은 계산량이 작아 비효율, (2) frame-level 병렬(같은 연산 시퀀스를 `F` frame에 SIMD 적용)을 채택.
6. 핵심기법2: SIMT로 `W`개 스트림을 동시 실행해 `W×F` frame 병렬 디코딩. 각 GPU 스트림은 ARM 상 pthread(worker)로 제어.
7. 구현 디테일: sparse H-matrix의 비연속 메모리 접근을 완화하려 디코딩 전후 data interleaving/deinterleaving(kernel 1/3)으로 `F` frame을 정렬.
8. 최적화 절차: 별도 학습 없음. 스트림 수 `W` 선택으로 성능-자원(CPU 부하) trade-off 조절.
9. 결과: 1스트림 25Mbps→3스트림 35Mbps(약 30%↑), ARM 24% 사용으로 [13] 100% 사용 대비 유사 스루풋. host-GPU 전송지연 20ms→4.8ms 개선, 고성능 GPU서 최대 230Mbps.
10. 한계: 정정능력/알고리즘 개선 없음(구현 논문), gatecount/공정·부호종류·연판정 미상, GPU/CUDA 특화라 NAND 컨트롤러 RTL과 무관.
