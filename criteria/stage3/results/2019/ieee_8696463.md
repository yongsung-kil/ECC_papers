### B. 알고리즘 요약

1. 시스템: 50G-PON FEC, `(γ,ρ)`-irregular QC-LDPC mother code `γ=12, ρ=69, b=256`, 부호길이 25669/17664 bit, 부호율 57/69≈`0.826`(참조 HD는 punctured/shortened로 0.849), AWGN 채널.
2. 문제: 802.3ca가 채택한 HD-LDPC(rate 0.849)로는 PR30(29dB) 링크버짓 부족 → 더 높은 coding gain 필요.
3. 핵심가정: 동일 mother code에 SD(soft-decision) 복호를 적용하면 rate를 약간 낮추는 대신 gain 확보.
4. 핵심기법 1단: `0.75 scaled min-sum` 복호 - VNP는 sum, CNP는 min1/min2 탐색, APP(`Lcv`)/LLR(`Lv`) 메모리 layered 갱신.
5. 핵심식: bit LLR을 채널에서 계산해 min-sum 반복 message passing, 최대 iteration 15로 제한(저지연).
6. 핵심기법 2단: min 연산은 `2^n` degree binary search tree로 min1/min2와 위치 탐색.
7. 부수트릭: irregular 구조에서 sum부는 전블록 LLR/APP를 쓰되 min부는 입력에서 compress·출력에서 expand → 32-degree tree로 69블록 병렬(최대 col weight 24), min-sum 처리 13 clock.
8. 최적화: 없음(구조 고정, 실측 검증 중심).
9. 결과: 동일 rate 0.826에서 SD가 HD 대비 NCG 1.1 dB↑, 참조 HD(0.849) 대비 ~1.3 dB gross coding gain, 실측 처리속도 ~0.1 Gbps, output BER 1e-12 목표.
10. 한계: gatecount·정확 throughput은 그림/표에만 있어 미상, PON 광통신 특화, iteration 감소 기여 없음.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 0.75 scaled min-sum CN 연산 | decoder.cpp CNU_Update_New_Mag(), C2V_Cal_New_Sgn() | Prime ECC도 min-sum(min1/min2) 보유, scaling factor는 중복 가능성 |
| VNP sum / APP 갱신 (layered) | decoder.cpp VN_Cal_HD() 등, LDPC_Decoder() | layered 갱신은 Prime ECC dual-update와 유사 계열 |
| SD LLR 계산/메모리 (Lv) | channel.cpp Set_LLR_Th(), ecc_data.h PARAM_LLR | soft-read LLR 계층에 대응, 다만 NAND read 비용 모델과 상이 |
| irregular min 블록 compress/expand 라우팅 | GT.cpp Make_GT_HW() | edge/MUX 축소 관점에서 graph thinning과 목적 유사 |
| QC-LDPC mother H-matrix 로드 | ecc_top.cpp Load_PCM() | 부호구조 참고 가능하나 PON 전용 (γ=12,ρ=69,b=256) |
```

적용 가치: **중간** — 고rate binary QC-LDPC + SD min-sum FPGA 실측이라 관점은 유사하나, scaled min-sum·layered·SD LLR·edge 축소가 Prime ECC 기보유 기법(min-sum/dual-update/적응형 LLR/graph thinning)과 대체로 중복되어 신규 이식 여지가 제한적.
