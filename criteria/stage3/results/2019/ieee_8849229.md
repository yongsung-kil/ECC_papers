### B. 알고리즘 요약

1. 대상: URLLC용 short 선형블록부호 `C(N,K)`(BPSK/AWGN), 실험은 eBCH `(128,64,22)`; OSD(ordered statistics decoding)를 근사 ML로 사용.
2. 문제: OSD 복잡도가 order-m에서 `O(K^m)`로 큼 → distance 기반 skipping/stopping 조건으로 효율화하려면 distance 분포의 정확한 통계 모델이 필요.
3. 모델: 수신 신호 정렬 후 MRB(most reliable basis) K비트 + redundancy로 분할, 신뢰도 `αi=|ri|`의 order statistics 사용.
4. 핵심기법: 0-reprocessing(TEP=0)의 Hamming distance `d0`를 두 랜덤변수 혼합으로 표현 — MRB 오류 없을 때 `E_{K+1}^N`(redundancy 오류수), 오류 있을 때 `W_cP`(부호 중량분포).
5. 핵심식: `pd0(j) = Pe·pE(j) + Pc·pW(j)` (식 (11)), 혼합가중 `Pe,Pc`는 채널조건(SNR)에 종속(식 (14),(15)).
6. 확장: order-i reprocessing으로 확장(식의 `j→j+i`, TEP weight 반영), 최소 Hamming distance `di` 분포도 두 분포 혼합.
7. 트릭: random linear code 가정 시 부호 중량분포를 truncated binomial `bi=(1/2^N)C(N,i)`로 근사(식 (20)).
8. 통찰: `d0` 분포의 극소점이 부호 최소거리 `dH` 근방 → correct/incorrect decoding 경계 지표로 활용.
9. 결과: 이론분포가 시뮬레이션(SNR 0~3dB, (128,64,22) eBCH)과 정밀 일치; NC/SC 조건 설계로 segmentation-discarding OSD 제안(복잡도↓, 성능저하 미미).
10. 한계: HW 미설계, 성능 gatecount/throughput 없음, LDPC 아닌 일반 블록부호 OSD 전용 통계, 반복 message-passing 복호와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| OSD 근사 ML 복호(MRB reprocessing) | 대응 없음 | Prime ECC는 min-sum 반복복호, OSD 재부호화 패러다임과 구조 상이 |
| distance 기반 조기종료(NC/stopping) | `partial_CRC.cpp` `partial_crc_HW_T4()`, `full_CRC.cpp` | 개념상 조기종료지만 CRC 기반이라 distance 통계와 직접 대응 안 됨 |
| 부호 중량분포/최소거리 분석 | 대응 없음 | 부호구조는 QC-LDPC 고정, weight enumerator 분석 미사용 |
| soft LLR 신뢰도(|ri|) order statistics | `channel.cpp` `Set_LLR_Th()` | soft-info 사용은 유사하나 정렬·MRB 개념은 미사용 |

적용 가치: **낮음** — OSD는 short block code용 근사 ML 복호로 NAND용 반복 min-sum QC-LDPC 디코더와 알고리즘 패러다임이 근본적으로 다르며, 본 논문은 그중에서도 순수 distance 분포 통계이므로 Prime ECC에 떼어 쓸 요소가 사실상 없음.
