### B. 알고리즘 요약

1. 시스템: 분산 저장(DSS) 노드 장애를 견디는 소거정정 코드. 이미지(Lena)를 9등분해 각각 (12,8) 소형 LDPC로 인코딩, LAN 서버들에 분산 저장.
2. 문제: 3-way replication은 200% 오버헤드, RS 코드는 Galois field 연산으로 복호 복잡·느림. 저비용 소거정정 필요.
3. 채널 모델: erasure 채널(노드/블록 소실). 비트 오류가 아니라 블록 전체 유실 가정.
4. 핵심 기법: sparse `H` matrix의 bipartite/Tanner graph에서 parity block = 연결된 data block들의 XOR. 복원도 알려진 block들의 XOR로 lost block 재구성.
5. 핵심 식: `є = (n+m) - fn` (erasure correction capability), `R = k/n`, stretch `S = 1/R`.
6. 부호 구성: Monte Carlo 랜덤 생성(확률 `p`=0.4~0.6)으로 overhead factor `f`(≥1) 좋은 그래프 선정.
7. 구현: 인코더/디코더 모두 MATLAB + Simulink. HW 없음.
8. (학습/최적화 없음 — 랜덤 그래프 생성만)
9. 결과: (12,8) 코드, `R=0.57`, `f=1.25`, erasure 정정능력 `є=2` (최대 2블록 소실까지 이미지 복원 성공).
10. 한계: soft-decision·BER 개념 없음, HW 미설계, 소거정정 도메인 한정, 부호 길이 극소(12).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| XOR 소거복원(erasure) | 대응 없음 | Prime ECC는 soft/hard-decision iterative min-sum 복호, erasure 복원 없음 |
| 랜덤 bipartite graph 부호구성 | 대응 없음 | Prime ECC는 고정 QC-LDPC(base+shift), 랜덤 그래프 아님 |
| (12,8) 소거정정 부호설계 | 대응 없음 | 부호길이·용도 모두 NAND ECC와 무관 |

적용 가치: **낮음** — 소거정정(node failure) 전용 XOR LDPC로 NAND soft-decision binary QC-LDPC ECC의 부호설계/디코더/HW 어디에도 이식 요소가 없다.
