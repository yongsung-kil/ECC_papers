### B. 알고리즘 요약

1. 대상은 BSC(hard-decision)에서 동작하는 regular QC-LDPC (dv4R050N1296, dv3R050N1296, dv4R050N4000, dv4R087N16383) 복호.
2. 문제: PGDBF 등 기존 hard-decision GDBF가 소형 trapping-set(TS)에 갇혀 error-floor가 발생하고 soft-decision(OMS)과 성능 격차가 큼.
3. 모델: all-zero codeword를 BSC(crossover α)로 전송, inversion(energy) 함수 `Ξk(c)=ck⊕rk + Σ_{j∈Nv(k)} sj`로 flip 대상 결정.
4. 핵심 기법1(RGDBF): inversion 함수에 random-penalty 항 `λ`(0/1, Pr(λ=1)=σ) 추가 `Ξ_k^R = ck⊕rk + Σsj + λ`로 규칙성을 깨 TS 탈출.
5. `λ`는 0/1 정수라 부동소수 없이 최소 복잡도로 steepest-descent가 아닌 bit도 flip 기회를 부여.
6. 핵심 기법2(tabu-list): 현재 iteration에서 flip된 bit를 tabu-list에 넣어 다음 iteration flip/최대값 탐색에서 제외, repeated-flip 문제 해소(GDBF 실패의 83%@α=0.01 원인).
7. HW 트릭: RNG 대신 penalty term을 사전생성·고정하고, QC 규칙성 이용해 codeword 벡터를 매 iteration 1-position cyclic shift → penalty 저장용 N 레지스터 제거, 성능손실 무시가능.
8. inversion indicator를 LCT 기반 modified one-hot(Eik=1 if Ei≥k)으로 재설계, ECU 입력 n=dv+1로 줄여 면적 57N→26N, delay 7→5.
9. 결과: TRGDBF는 PGDBF 대비 최대 2자릿수 FER 개선, avg iteration 약 15% 감소(8.5→7.3@α=0.03), OMS-layered에 근접; 90nm 0.299mm2(PGDBF의 93%, MS의 22%), 435MHz, avg throughput 142Gbps.
10. 한계: BSC(hard-1bit)만 다루고, ASIC은 특정 dv3R050N1296 코드에 고정 배선되어 다른 코드 run-time 미지원, soft-read 시나리오 미평가.
