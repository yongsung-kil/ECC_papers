> 총평: SSD 시스템 레벨(RAID/write-amp)의 여분 정보를 활용한 joint bilayer LDPC 복호로 1.8dB gain을 얻으나, 단일 부호 디코더 개선이 아니라 상위 stripe 구조 의존이라 Prime ECC 단일 QC-LDPC 엔진에 떼어 쓰기 어려움 → 이식성 하.

### B. 알고리즘 요약

1. 대상: NAND SSD, page당 [n,k] LDPC + stripe 단위 RAID(4/5/6, RDP) 또는 write-amp(WA) 복제 페이지. 시뮬은 (3,6) regular LDPC, length 800, AWGN, max iter 80.
2. 문제: ECC·RAID가 둘 다 실패하면 stripe 내 여분 페이지가 있어도 기존 방식은 복구 불가; 새 부호 설계 대신 기존 디코더로 여분 정보 활용.
3. 모델: 여러 noisy 페이지 `y_i^n`가 zero-parity(개별 LDPC) + RAID/WA joint parity 두 층 제약을 만족 → bilayer LDPC로 표현.
4. 핵심기법 1단(RAID 4): d개 hard-error 페이지를 lower(개별 LDPC)·upper(RAID XOR 제약) 두 layer로 묶어 message-passing을 교대 수행. 상·하 check node의 edge 연결 확률이 달라 LLR을 별도 추적.
5. 핵심식: V2C `v = u0 + (dv-1)u_lower + u_upper` (식2), C2V는 표준 `2 tanh⁻¹(∏ tanh(v/2))` — upper layer(RAID) LLR을 추가 주입해 상관 정보 이용.
6. 핵심기법 2단: RAID 6(RDP, 대각+행 parity)로 확장해 3 hard error를 middle/left/right bilayer part로 분해 복호; WA(같은 정보비트 공유)·coupling WA로도 일반화.
7. 구현 디테일: bilayer이라 upper/lower check node별로 density evolution threshold를 별도 계산(Table I, II).
8. 학습/최적화: 없음(부호 학습 아님). relay-channel(decode-and-forward) 관점의 용량 분석(Appendix A/B, RAID-relay·WA-relay 용량).
9. 결과: AWGN, BER 10⁻³에서 conventional 대비 gain — diff WA 0.6dB, coupling WA 0.8dB, RAID4(2 err) 1.4dB, RAID4(3 err) 1.6dB, RDP(3 err) 1.8dB.
10. 한계: HW 미설계(향후 과제로 명시), 복잡도·latency·RAM/엔진 자원 증가, 단일 LDPC가 아닌 RAID/WA 상위 구조 의존.
