### B. 알고리즘 요약

1. 시스템: raptor-like LDPC(ATSC3.0 방송), 부호길이 16200/64800, rate 3/15~5/15, BPSK/64QAM AWGN, SPA/OMSA/NMSA 복호 기준.
2. 문제: 전통 BP는 매 iteration 모든 edge LLR을 계산하지만 raptor-like에서는 일부 연산이 불필요.
3. 구조 가정: raptor-like는 다수의 degree-1 VN을 가지며 각 CN은 최대 1개 degree-1 VN에 연결, PCM은 core part(submatrix `A`,`B`=dual-diagonal)+`Z`(zero)+`D`(identity)로 구성.
4. 핵심 기법1: `two-type degree distribution` 프레임워크(edge를 E1/E2, VN을 `Va`/`Vb`, CN을 `C1`/`C2`로 분류)로 raptor-like를 표현.
5. 핵심 기법2: two-type GADE로 각 iteration의 메시지 평균 `m(u1(l))` 추적, 식(5)(6)(12)(13)으로 snr threshold 계산.
6. 핵심 관찰: 초기 iteration에서 core part(A,B)로 향하는 CN→VN LLR `m(u1(l))≈0` → 해당 행/열 연산 불필요.
7. 제안 복호(Alg.1): `iter ≤ numearlyiter`이면 submatrix A,B의 `L(vj→ci)`,`L(ci→vj)`를 0으로 대체하고 C,D만 연산, 이후엔 전체 PCM 연산.
8. early iteration 최적화(Alg.2): 식(16)으로 제안 방식의 snr threshold `snrth,2`를 예측, `|snrth,2 - snrth,1| ≤ ε` 만족하는 최대 `numearlyiter` 선택, SNR 구간별 lookup table 가능.
9. 결과: SPA/OMSA는 성능 동일, NMSA는 장부호에서 오히려 개선, 복잡도 감소율 `numcoreone·numearlyiter/(numone·nummaxiter)` = 5~17%, 평균 약 10.14% (성능 손실 없음).
10. 한계: HW 미설계·시뮬만, raptor-like(degree-1 VN·core part) 구조 전용, 정정능력·waterfall 개선 없음, IoT/handheld 전력 절감 목적.
