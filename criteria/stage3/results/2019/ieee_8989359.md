### B. 알고리즘 요약

1. 대상은 spatially-coupled turbo류 **PIC-TC**(부분정보결합 turbo, 성분부호 rate `1/3` PCCC), BEC 채널 전송.
2. 문제: 기존 PIC-TC[12]는 coupling memory `m=1`만 다뤘고 EXIT 기반 임계값이 상계라 실제와 큰 격차, 이론적 성능 미규명.
3. 모델: 정보열 `u`를 `L`개 CB로 분할, 각 CB의 정보 일부(coupled `u_{t,t+1}`)를 인접 CB와 공유, coupling ratio `λ=D/K∈[0,1/2]`가 전체 rate·임계값 결정.
4. 핵심기법: coupling memory `m≥1` 일반 구성 - coupled 정보열을 `m`개로 나눠 `(t+1)~(t+m)` CB에 분배, zero-padding 처리.
5. 핵심 식: 전체 rate `R_PIC → (R−λR)/(1−λR)` (L→∞), `λ`를 키우면 rate 하락(coupled 비트는 turbo 2회=convolutional 4회 부호화).
6. 그래프 모델: turbo compact graph(상/하 factor node `f^U/f^L`)로 표현, coupled 정보노드가 인접 시각 CB를 수평 연결.
7. 복호: FF-FB(feedforward-feedback) 직렬 스케줄, 각 CB는 BCJR 성분복호 turbo decoder, zero-pad 비트는 LLR=∞.
8. 학습/최적화: density evolution - erasure 확률 전달함수 `F_p^U/F_q^U` 등으로 iteration별 erasure 확률 추적해 BP threshold 산출.
9. 결과: rate 0.2~0.32 전 구간 BEC capacity 격차 약 0.02(최저 rate 0.2에서도 0.024), `m` 증가해도 threshold 불변, 유한길이(K=6144,L=100) BER이 DE threshold 0.005 이내.
10. 한계: BEC 전용 순수 이론, AWGN/NAND 미확장, HW·throughput·gatecount 논의 전무.
