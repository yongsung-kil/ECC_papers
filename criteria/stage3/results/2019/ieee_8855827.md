### B. 알고리즘 요약

1. 시스템: 무선 fast fading(Rayleigh) 채널, BPSK, WiMax 표준 LDPC(length 576, rate 0.75), BP 최대 25 iteration.
2. 문제: fading 채널에서 BP 성능은 채널이득 예측 정확도에 의존하고, 채널이득/잡음이 상관되면 BP 성능이 저하됨(식 3 `y=s⊙h+n`).
3. 모델: 채널이득 `h=gΘ_f^{1/2}`, 상관 잡음 `n=zΘ_n^{1/2}`(단일계수 지수 상관행렬 Θ, 계수 ξ_f/ξ_n).
4. 핵심 기법: 두 1D-CNN - CNN-1은 채널이득 `ĥ`, CNN-2는 잡음 `n̂`을 수신신호 y에서 예측(local connectivity로 code length 무관).
5. denoise: `y'=y-n̂=s⊙h+n'`로 잔차잡음 n' 얻고, 예측 이득 ĥ로 LLR 재계산.
6. 핵심 식: 잔차잡음 n'이 더 이상 가우시안이 아니므로 히스토그램 EPDF `F(·)`로 `LLR_i=ln[F(y'_i-ĥ_i)/F(y'_i+ĥ_i)]`(식 14) 유도.
7. 구현: 1D-Conv 5층(kernel 9/3/3/15, feature map 64/32/16/1, ReLU+Linear), same padding.
8. 학습: Keras/TF, Adam, MSE loss, lr 0.001, epoch 1000, batch 700, SNR {0,2,4,5,6,7,8}dB × 5만 프레임.
9. 결과: EG-BP 대비 성능이득 ξ_n=0.5에서 1dB, 0.8에서 4dB, 0.9에서 7dB(상관 강할수록 이득↑).
10. 한계: 무선 fading/상관 잡음 채널 전용, CNN 학습/추론 오버헤드(GPU), HW 미설계, NAND soft-read/양자화 무관.

```
|---|---|---|

| BP(SPA) 디코더 코어 | decoder.cpp LDPC_Decoder() | 낮음 - Prime은 min-sum, 논문은 full BP + 외부 LLR 보정 |

```

적용 가치: **낮음** - NN 전처리로 무선 상관 fading의 채널이득/잡음을 보정하는 기법으로, NAND에는 상관 채널이득 개념 자체가 없고 Prime ECC의 §5 필터상 NN 디코더는 이식성 `하`에 해당한다.
