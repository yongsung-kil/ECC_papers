### B. 알고리즘 요약

1. 대상은 layered 스케줄링 QC-LDPC 디코더 HW, 단일포트 SRAM `nbanks`개 뱅크 구조, 검증 부호는 WiMAX(N=2304, z=96, rate 1/2~5/6)와 DVB-S2X(N=64800, z=360).
2. 문제: 단일포트 메모리뱅크에서 동시접근 충돌과 pipeline 지연(RAW hazard)으로 stall이 발생해 throughput·HW효율(HUE) 저하.
3. 핵심 기법 1(오프라인): 3단계 — (a) check-node 순서 σ를 genetic TSP로, (b) 메모리뱅크 매핑을 hypergraph 컬러링(Modified Misra-Gries, MMG)으로, (c) in-layer 읽기접근 재정렬 ψ로 stall 최소화.
4. 매핑을 base graph G의 변수노드 컬러링(=뱅크 배정)으로 정식화, 뱅크당 최대 동색 이웃수 `TG(ξ,m)=max_c |Nξ(m,c)|`가 read cycle 수.
5. 핵심 식: HUE = `Σ dc(m) / (IG·nbanks)`로 유효 메모리접근 비율을 정의, 매핑충돌+pipeline stall을 통합 평가.
6. 핵심 기법 2(아키텍처): residue-based layered 스케줄링 — overlapping update 허용, β 갱신차이를 residue `δt = βnew - βold`로 register file에 누적했다 마지막 check-node에서 AP-LLR에 반영(=부분적 flooded).
7. 구현 디테일: CNU는 min-sum(min1/min2/sign), barrel shifter로 α/β(AP-LLR 아닌 저양자화 메시지) 라우팅, reverse write-back, opcode ROM로 residue 제어.
8. 최적화: 컬러링은 GA(fitness=1/TG), in-layer는 tabu search로 iteration duration `IG` 최소화.
9. 결과: 오프라인 알고리즘만으로 HUE 3~49% 향상, residue 병용 시 25~57%, FPGA에서 conventional layered 대비 TAR 최대 85% 개선; residue n=∞ FER는 미미한 저하(AWGN, Imax=30).
10. 한계: 정정능력 개선이 아니라 HW효율(HUE/TAR) 최적화이며, 오프라인 알고리즘이 부호별로 장시간(1~2h) 실행 필요.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| CN min-sum 연산(min1/min2/sign) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 우리도 이미 보유(§3 min-sum) → 중복, 이식가치 낮음 |
| layered 스케줄 iteration 루프/AP-LLR 갱신 | `decoder.cpp` `LDPC_Decoder()` | Prime ECC는 Dual-Update(even/odd) 스케줄이라 residue-based layered와 상이하나 스케줄링 아이디어 참고 가능 |
| 메모리뱅크 매핑/라우팅/pipeline | 디코더 HW 모델(z=32, HCU, GT) + `GT.cpp` `Make_GT_HW()` | HW 메모리라우팅·pipeline hazard 완화 관점서 관련, 단 z/HCU/GT 구조와 정합 검증 필요(난이도 중) |
| H-matrix/부호구조 로드 | `ecc_top.cpp` `Load_PCM()` | 오프라인 매핑이 base matrix B 종속이라 부호로드와 연계 |

적용 가치: 중간 — min-sum 자체는 중복이나, 단일포트 메모리뱅크 매핑+pipeline RAW 완화(residue-based layered)는 Prime ECC의 HW 메모리라우팅/pipeline 병목 관점에서 참고할 만하며, z=32·HCU·GT 구조 정합만 확인되면 부분 이식 가능.
