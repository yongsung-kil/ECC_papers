### B. 알고리즘 요약

1. 대상 시스템은 MLC NAND(cell당 2bit, "11">"10">"00">"01" Vth 순), 16KB 페이지, MATLAB 채널 시뮬(Table IV 파라미터).
2. 문제: retention error는 고Vth("01") 상태에서, program interference도 고Vth 프로그래밍에서 심화 - 기존 DPA/AC/NRC는 1s 비율만 올려 state "11"을 늘림.
3. 관찰: 두 오류가 Vth를 반대 방향으로 이동시켜 부분 상쇄되며, 기존 기법은 state "01"을 효과적으로 줄이지 못함.
4. 핵심 기법: 페이지를 N개 data segment로 분할, hotness와 1s비율(<50%/>50%)에 따라 4개 flip sub-scheme(H0/H1/C0/C1) 적용.
5. LSB/MSB 페이지 조합별로 flip 여부를 결정해 hot은 "11" 최대·"01" 최소, cold는 "10" 최대·"01" 최소로 remapping.
6. cold data는 state "10"을 늘려 program interference로 retention error를 상쇄(counteraction 활용)하는 것이 특징.
7. 구현 디테일: flag bit(H0=11,H1=10,C0=00,C1=01) OOB 저장, 공통 bit 압축으로 페이지당 N+1 bit만 소모.
8. HW 오버헤드: hot data MSB 기록 시 대응 LSB 캐싱(SSD 예시 31.5MB), READ PAGE CACHE로 read latency 약 7% 증가.
9. 결과: N=1(최악조건)에서 NRC 대비 hot RBER 최대 20.30%, cold 최대 67.24% 저감. randomizer 비활성에도 개선.
10. 한계: 실제 HW 미검증(future work), ECC 결합 미평가, 순수 write-side 전처리로 LDPC 복호/부호와 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| cell state remapping(데이터 flip) | 대응 없음 | write-path 데이터 전처리로 ECC 코어 밖 |
| RBER 저감(채널 입력 개선) | 대응 없음 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md)의 RBER는 주입 모델이며 여기선 매핑을 바꾸는 개념 - 직접 대응 아님 |
| 부호/디코더 요소 | 대응 없음 | LDPC 부호설계·복호 언급 자체 없음 |

적용 가치: 낮음 - LDPC ECC 부호설계/디코더/HW 어느 레이어도 건드리지 않는 순수 write-side 데이터 전처리라 Prime ECC 3.1에 이식할 요소가 없다.
