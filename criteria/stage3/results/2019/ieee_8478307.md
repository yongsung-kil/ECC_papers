### B. 알고리즘 요약

1. 대상: binary [n,k,d] 선형부호(Golay [24,12,8], QR [48,24,12], Tanner (3,5)-regular LDPC [155,64,20]), BEC 상 iterative message-passing 복호.
2. 문제: BEC에서 iterative 복호는 `stopping set` 때문에 ML 복호보다 성능이 열등 -> H 행렬에서 stopping set 제거 필요.
3. 핵심 가정: parity-check 행렬에 dual 부호어(C⊥) redundant row를 추가하면 stopping set이 `covered` 되어 실패율 감소.
4. 핵심 기법1: `coverable stopping set`(비영 부호어 support를 포함하지 않는 stopping set)을 정의, size가 `d` 이상인 영역까지 stopping redundancy hierarchy를 확장.
5. 핵심 상계: Theorem 1에서 first τ row를 결정론적으로 고른 뒤 확률적 분석(π(r,i,j) 단조성)으로 기존 [7][8] 상계 개선.
6. 확장: Theorem 2/3에서 coverable stopping set 개수 u_i로 ℓ-th stopping redundancy(ρ_ℓ) 상계를 ℓ≤r=n-k 전 구간에 유도.
7. 구현 디테일: u_i 계산이 어려운 장부호는 Edgeworth 기반 신뢰구간(Lemma 4)으로 통계적 추정.
8. 최적화: 초기 row weight를 w_opt∈[⌈(n+1)/ℓ⌉,⌊n/ℓ⌋]로 선택(Lemma 7), greedy(lexicographic) 탐색으로 Golay redundant 행렬 구성.
9. 결과: Golay에서 H(370) redundant 행렬로 BP 복호가 ML과 동일 성능 달성(FER 곡선 일치), SRE·Gallager 앙상블 상계와 수치 결과 일치.
10. 한계: BEC 전용·소~중 길이 부호, 상계가 지수적으로 증가(장부호 비실용), HW/throughput/soft-decision 논의 전무.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| stopping set 제거용 redundant row 추가 | 대응 없음 | BEC 이론 구성, Prime ECC의 고정 QC-LDPC PCM과 무관 |
| iterative(BP) 복호 성능-ML 격차 | 대응 없음 | Prime ECC는 AWGN/RBER상 min-sum, BEC 아님 |
| PCM/부호 구조 설계 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Load_PCM()` | 개념상 부호설계지만 BEC redundant-row 이론은 이식 불가 |

적용 가치: 낮음 - BEC 상 stopping redundancy의 정보이론 상계로 soft-decision AWGN/NAND용 binary QC-LDPC 디코더·HW와 접점이 없고, 상계도 지수적이라 Prime ECC 3.1에 떼어 쓸 요소가 없다.
