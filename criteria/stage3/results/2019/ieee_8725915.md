### B. 알고리즘 요약

1. 대상은 구조화 binary LDPC (`(2209,1978)` rate-0.89 array code, `(1944,1620)` rate-0.83 802.11n code), BPSK+AWGN, min-sum 복호.
2. 문제: iterative 디코더가 non-codeword elementary trapping set(ETS)에 갇혀 error-floor 발생, 기존 코드재설계/concatenation/스케줄링은 비용·구조훼손 큼.
3. 핵심 아이디어: simulated annealing 유추 - 국소최소(ETS)를 `heating`(메시지 reweight/soft flip)으로 교란 후 `cooling`(BP)으로 정답 codeword로 수렴.
4. Neighborhood set `N`: 불만족 degree-1 CN 부호로 VN이 자기 라벨링, 이 집합에만 heating 적용.
5. Heating 식 (6)/(7): 만족 CN→N VN 메시지 magnitude를 낮은 값 `A0`로 축소(reweight), `A0` 낮을수록 교란 강함.
6. Quenching(P=1): 1회 heating 후 즉시 cooling, Type I ETS(각 VN이 정확히 degree-1 CN 1개) 대응.
7. Extended heating(P>1, P=10): inner bit까지 온도 전파, neighborhood를 매 iter 갱신하여 Type II ETS 해결.
8. Focused heating(L=5, B0=1~3): plural bit(불만족 CN 2개 이상 연결)을 soft bit flip으로 약화해 neighborhood 축소, Type III ETS 대응. Extended+focused 결합(G=10).
9. 결과: array code ETS 236개 중 99% 해결·BER 2오더↓, 802.11n는 95% 성공·1~2오더↓, quenching 단독(23%)/bi-mode(59%) 대비 우수.
10. HW: VN에 1bit 라벨·MUX 추가만으로 fully-parallel/row-parallel 디코더에 통합, FPGA(Virtex-5)에서 slice reg/LUT 4.5~10% 오버헤드, 100MHz 유지로 throughput/latency 동등.
