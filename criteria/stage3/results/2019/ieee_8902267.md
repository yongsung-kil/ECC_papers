### B. 알고리즘 요약

1. 시스템: SMR/TDMR HDD, array head 3-reader로 대상 트랙+인접 2트랙 읽기, PR1 채널, 128/130 RLL + (3,30)-regular LDPC(rate≈0.88, 4KB 섹터).
2. 문제: APP 디코더 출력 LLR에 인접 트랙으로부터의 ITI(inter-track interference) 영향이 남아 반복복호 성능이 저하됨.
3. 채널/모델: Voronoi granular medium(4 Tbit/in²), write switching field Gaussian(2 kOe, 5%), 각 채널비트가 약 5개 자성 grain, system noise는 AWGN.
4. 핵심 기법: NN LLR modulator — 대상 트랙 3bit LLR + 인접 트랙 각 2bit LLR을 입력으로 3층 NN(`N1,N2,N3`)이 "0"/"1" 신뢰도 `yk(3)`(soft-max)를 재계산.
5. 변조 규칙: NN 출력이 임계 `H1=0.9` 초과→"1", `H0=0.1` 미만→"0", 사이면 hold. 하드결정과 일치하는 LLR만 변조 가중치 `W=1.6`로 증폭.
6. 학습: 오차함수 `E(w)=-Σ ln p(dk|xk,w)`를 BPA로 최소화하여 연결가중치 `wk(mj)` 갱신.
7. 확장: HGA(GA+BPA)로 인접트랙 입력 비트 선택·NN 구성 단순화, fitness `fni`에 제거된 연결 수 보너스 `Bni` 부여, 가중치<0.10 연결 제거.
8. 복호: APP 디코더 ↔ SP(sum-product) 디코더 간 turbo equalization, SP 내부 `isp=15`, 전역 `iglobal=10`.
9. 결과: 인접트랙 고려 시 correct LLR 증가·BER 개선, HGA 적용 후 `iglobal=10`에서 error-free SP 출력 달성(특정 섹터).
10. 한계: HW 미설계, 시뮬만, SMR 인접트랙 ITI·PR1·sum-product turbo 구조에 특화되어 일반화 제한.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| NN 기반 LLR modulator | 대응 없음 | NN/딥러닝 LLR 보정은 Prime ECC에 대응 모듈 없음 |
| APP↔SP turbo equalization | 대응 없음 | full sum-product + APP turbo 구조로, min-sum 기반 Prime ECC와 무관 |
| soft LLR 입력/양자화 | [channel.cpp](../Prime_ECC_3.1_Claude/) `Set_LLR_Th()`, [ecc_data.h](../Prime_ECC_3.1_Claude/) `PARAM_LLR` | 개념적 접점뿐 — NAND read가 아닌 자기채널 인접트랙 LLR이라 직접 대응 아님 |
| (3,30) regular LDPC 부호 | [ecc_top.cpp](../Prime_ECC_3.1_Claude/) `Load_PCM()` | 우리 QC-LDPC와 부호구조 상이, 이식성 없음 |

적용 가치: **낮음** — HDD SMR/TDMR의 인접트랙 ITI 완화용 NN LLR modulator로 채널 모델·복호기(sum-product turbo)·부호구조가 NAND binary QC-LDPC와 전혀 달라 Prime ECC에 이식 대응이 없다.
