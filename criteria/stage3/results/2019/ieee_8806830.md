> 총평: 3D NAND 층별 RBER 편차에 대응하는 read-retry 레벨증가 granularity 스케줄링(FTL/컨트롤러 정책)으로, 디코더 알고리즘·부호설계는 손대지 않아 Prime ECC 디코더 코어 이식 대상이 아니다.

### B. 알고리즘 요약

1. 시스템: 3D charge-trap NAND SSD, 64GB/16KB page, 7-level progressive LDPC read (레벨↑ = ECC 강도↑ & read latency↑, 예: L1 85us ~ L7 229us, 누적 1099us).
2. 문제: 3D 플래시는 제조 PV로 층간 RBER가 최대 6.6배 차이(중간층 최고). 기존 단일-granularity(+1레벨) progressive read는 중간/하위층에서 불필요한 중간 레벨을 다 거쳐 누적 latency 과다.
3. 관찰(motivation): retention 증가 시 상위층 RBER는 거의 불변(L1로 충분), 중간/하위층은 급증해 높은 read level 급상승.
4. 핵심 기법 MG-LDPC: 레벨 증가 granularity가 각각 `1,2,3,4,6`인 5개 LDPC 엔진 설계. 시작 L1, 최대 L7. 고granularity 엔진은 실패가 예상되는 중간 레벨을 건너뜀.
5. FixAdp: RBER 특성 분석으로 층 구간을 5분할해 고정 엔진 배정(예: 층14~18=granularity 6).
6. DynAdp: 층별 직전 read level `R`을 extra flash 공간에 1B로 저장, 다음 접근 시 granularity `R` 엔진 사용(패턴 변화에 적응).
7. Algorithm 1: 최초 접근은 granularity 1로 fine-grained 탐색해 `precision` 확정, 이후 `error_rate < ERR_RATE[r]`이면 확정, 층 read level 변화 시 `R` 갱신. 선택 복잡도 O(1).
8. 오버헤드: DynAdp는 30층 기준 30B 저장(무시가능), FixAdp는 추가 오버헤드 없음. 코드워드 유일성으로 read 정확도 불변.
9. 결과(Disksim SSD ext, 9 workloads, P/E 8000): flash read latency FixAdp/DynAdp가 PrLDPC 대비 각 20%/39% 감소, read response time 최대 47% 개선(DynAdp가 LaLDPC 대비 26% 추가 개선).
10. 한계: 디코더/부호 미변경(read-level을 정성적 강도로만 취급), HW 미설계, 실칩 아닌 Disksim 시뮬·multiplier 기반 RBER 추정, 짧은 retention에선 FixAdp 열위.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| progressive read-level 증가 정책(granularity 스케줄) | 대응 없음 (FTL/컨트롤러 read-retry 상위 정책) | Prime ECC는 디코더 bit-exact 시뮬로, read-level 재시도 정책은 코드베이스 범위 밖 |
| read-level↔ECC 강도(soft read 비용) | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_R_Offset()`, `Set_LLR_Th()` | soft-read threshold 관점에서만 느슨한 접점, 층별 정책 자체는 이식 대상 아님 |
| 조기종료/iteration 스케줄 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `partial_crc_HW_T4()` | 개념적 read-retry 유사성뿐, 논문은 디코더 내부 조기종료를 다루지 않음 |

적용 가치: **낮음** — 디코더 알고리즘·부호설계·HW를 전혀 바꾸지 않고 컨트롤러/FTL의 read-retry 레벨증가 정책만 다루므로, Prime ECC 디코더 코어에 떼어 넣을 기법이 없다.
