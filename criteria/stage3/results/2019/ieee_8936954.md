### B. 알고리즘 요약

1. 5G NR data channel용 binary QC-LDPC(BG1 mother rate 1/3, BG2 1/5), 예시 시뮬은 rate `R=0.9258`, `Q=8`, lifting `Z`.
2. 문제: rate matcher가 puncturing/repetition으로 code bit 일부만 선택하는데 기존 구현은 full-base matrix로 불필요 bit까지 인코딩/디코딩해 throughput 낭비.
3. 채널/구조: 인코딩은 `Hc^T=0`, 코드워드 `c=[s,p]`를 circular buffer에 쓰고 redundancy version `rv`별 시작점 `k`부터 rate-matching(식(7),(8)), AWGN 채널 검증.
4. 핵심 기법: full-base matrix `Pm`을 전송에 필요한 행·열만 남긴 sub-base matrix로 pruning(`Pm=Pm(µ,ν)`), leading sub-base(앞 i행 j열)로 제한하지 않고 filler/불필요 systematic·parity 열까지 제거.
5. 인코더/디코더 분리: 인코더는 선택 code bit만, 디코더는 HARQ 재전송 시 직전 전송 행·열까지 합집합(`Pm(µ∪µ', ν∪ν')`)해 sub-base 구성(디코더 sub-base가 인코더보다 큼).
6. 구현 디테일: sub-base 지시를 configuration bit로 인코더 13bit(`b†`), 디코더 42bit mask(`b‡`)로 표현, controller만 수정하면 기존 dual-diagonal 인코더/block-parallel 디코더에 이식 가능(critical path 밖이라 주파수 무영향).
7. 디코더: pipeline block-parallel + layered normalized min-sum(scaling 0.85, 최대 iteration 12), row별 min-sum unit → extrinsic/LLR buffer 갱신, parity check pass 또는 max iter까지.
8. throughput 모델: `dt=K·fd/(id·ud)`(ud=sub-base circulant block 수)로 sub-base가 작을수록 향상.
9. 결과: 재전송에서 제안 pruning이 leading sub-base 대비 최대 1.40배 디코딩 throughput 향상(초기전송 최대 1.05배), BLER은 traditional과 사실상 동일(정정능력 변화 없음).
10. 한계: 정정성능 이득 없음(오직 throughput/energy), gatecount·절대 Gbps 본문 미기재, 5G rate-matching/HARQ 전제에 강하게 종속.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| dual-diagonal 인코더 sub-base 구성 | `encoder.cpp` `LDPC_encoder_4KB()`, `Generate_PCM_encoding()` | 우리도 dual-diagonal 인코더이나 NAND는 rate-matching/puncturing 없어 pruning 적용지점 부재 |
| block-parallel layered normalized min-sum 복호 | `decoder.cpp` `LDPC_Decoder()`, `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | min-sum 계열로 이미 보유(중복), normalized scaling만 참고 |
| base matrix / sub-base 로드 | `ecc_top.cpp` `Load_PCM()` | H-matrix 로드 지점이나 우리는 단일 고정 부호라 pruning 무관 |

적용 가치: **낮음** — binary QC-LDPC + dual-diagonal + layered min-sum으로 부호/디코더 계열은 일치하나, 기여의 본질이 5G rate-matching·HARQ용 base-matrix pruning throughput 최적화라 puncturing/재전송이 없는 NAND 고정 고rate 부호에는 적용점이 없다.
