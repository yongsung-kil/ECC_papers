### B. 알고리즘 요약

1. 시스템: 2-D product code(PC), 동일 binary eBCH component code(행/열), AWGN 채널, 대표 예제는 `(256,239,6)` eBCH로 PC rate `R=239^2/256^2≈0.8716`, PC 길이 `256^2=65536`.
2. 풀려는 문제: iBDD(hard)는 SDD(soft) 대비 1dB+ 손실, TPD/iGMDD-SR(soft 교환)은 성능은 좋지만 data flow(soft 교환)가 iBDD보다 훨씬 커 Tbps급 광통신에 부적합.
3. 핵심 모델: component 복호를 GMDD(generalized minimum distance, error-erasure)로 수행 — `|L_i|` 신뢰도 순위로 최소신뢰 `m`비트 erase, `t+1` trial 벡터에 algebraic error-erasure 복호.
4. 핵심 기법 1단: GMDD 최종 판정을 generalized distance `d_GD`(식1, `α_i=|L_i|/max|L_j|`) 대신 **Hamming distance** `argmin d_H(r,c)`(식2)로 대체 → soft 신뢰도 교환 불필요.
5. 식 의미: `d_GD`는 정규화 신뢰도 `α_i`를 쓰므로 soft를 넘겨야 하지만, `d_H`로 바꾸면 hard 결정+짧은 리스트만 교환해도 되어 data flow 급감.
6. 핵심 기법 2단: 반복 교환 메시지 = hard decision `ψ`(식4, `B(·)`)와 `d_min-1`개 최소신뢰 비트의 정렬 리스트 `L`; 신뢰도 갱신 `μ = w·μ̄ + L`(식3, `w`는 최적화 스케일, `μ̄∈{±1,0}`).
7. 구현 디테일: 리스트는 비트당 `(⌈log2(n)⌉+⌈log2(d_min-1)⌉)(d_min-1)` 비트로 표현; error-floor 회피 위해 BMP 8 iteration 후 conventional iBDD 2 iteration 부가.
8. 최적화: 스케일 벡터 `w=(w^(1),...,w^(ℓmax))`를 고정 Eb/N0에서 Monte-Carlo BER로 공동 최적화(단조증가 제약).
9. 결과(시뮬): BER 10^-7에서 iBDD-TPD 갭의 약 50% 폐쇄, iGMDD-SR 대비 0.074dB만 열화하면서 data flow는 iBDD 대비 +21.48%(n=256, d_min=5), n 클수록 증가율 감소(8.5~34.3%).
10. 한계: HW 미설계·시뮬만, LDPC 아닌 BCH product code, 광통신/무선 고throughput 타깃, LLR 강신뢰·오부호 시 error-floor 위험(iBDD 후처리 의존).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| GMDD/BDD component 복호 (product code) | 대응 없음 | Prime ECC는 QC-LDPC 그래프 복호로 component algebraic 복호 개념 없음 |
| BMP hard-message + 최소신뢰 리스트 교환 | 대응 없음 | LDPC의 C2V/V2C min-sum 메시지 구조와 상이 |
| 신뢰도 스케일 `μ=w·μ̄+L` | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` 이터레이션 루프 | 개념적 유사(scaled reliability)뿐, 알고리즘 직접 이식 불가 |
| iterative message passing 스케줄 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `C2V_Cal()` | 반복복호라는 공통점뿐, product code 전용 |
| error-floor 회피용 iBDD 후처리 | 대응 없음 | LDPC엔 BDD 후처리 개념 없음 |
