### B. 알고리즘 요약

1. 시스템: BSC(교차확률 `α`) 상 binary `(dv,dc)`-regular LDPC, 검증 부호는 `(155,64)` Tanner code와 `(4096,3712)` column-weight-5 QC-LDPC(12×128 PEG 모행렬을 32로 lifting, rate≈0.906).
2. 풀려는 문제: MS 알고리즘의 2-bit precision 양자화 시 심한 성능 저하 — 기존 저-error-floor MS는 최소 4-bit 필요.
3. 핵심 가정: error floor는 소수 trapping/absorbing set 때문이며 그 영역 디코더 출력은 신드롬 무게 `ω`가 작다; scaled channel message `ρ=A/Bmax` 범위가 작을수록 absorbing set 오류를 잘 정정([11]).
4. 핵심 기법: 신드롬 무게 범위를 `K`개 세그먼트 `Ω_j=(ω_{j-1},ω_j]`로 나누고 각 세그먼트마다 별도 2-bit MS 디코더 `D_j=(S_j,T_j,Q_{j1},Q_{j2})`를 매 iteration 적응 선택.
5. 핵심 식: VN 업데이트 `q = Φ_{V,k}(Φ_{H,k}(y_v) + Σ r)`, 매핑 `Φ_{V,k}`는 임계 `T_k`로 4-level(`±Q1,±Q2`) 출력 — 2-bit로 표현 가능. CN은 표준 MS(sign·min).
6. 확장: 매 iteration `ω̂=w_H(Hx̂^t)` 계산 후 `ω̂∈Ω_j`인 `j`로 디코더 교체(`k=j`); `K=1`이면 일반 2-bit MS로 환원.
7. 구현 디테일: 디코더 파라미터는 `S_u≤S_v, T_u≤T_v, Q_{u1}≥Q_{v1}, Q_{u2}≥Q_{v2}`(u<v) 순서 제약 → `ρ_1≤...≤ρ_K` 보장; 저degree VN은 매핑 테이블화(FAID식) 가능.
8. 설계 절차: waterfall 좋은 `D_K`부터 선택 후 `D_{K-1},...`를 시뮬로 파생, 세그먼트 경계 `ω_j`도 시뮬로 결정(`ω_1`은 보통 ≤20).
9. 결과: `K=3`이 최적(그 이상 이득 미미). 두 부호 모두 BSC에서 SPA(floating-point) 근접, 기존 bi-mode erasure[12]·4-level FAID[9]보다 waterfall 우수, error floor는 FAID 유사.
10. 한계: BSC 전용(AWGN 미검증), HW 미설계·시뮬만, 추가 연산은 신드롬 무게 계산뿐이라 latency 거의 불변(감소는 아님).
