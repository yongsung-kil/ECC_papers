### B. 알고리즘 요약 (10줄 내외)

1. 시스템: NAND flash 컨트롤러 ECC로 QC-LDPC 사용, `k` message + `m` check bit로 codeword 구성, P/E 사이클 증가에 따라 error rate 상승을 다룸.
2. 문제: 고정 rate LDPC는 P/E 후기 에러 급증에 대응 못 하고, parity를 더 만들어도 spare region에 못 넣어 read/write overhead 발생.
3. 핵심 기법 1단(부호설계): 기본 pchk를 `A`(m×m 비특이 QC) + `B`(m×k QC)로 구성하고, `e` bit 추가 parity를 위해 `D`(e×e), `I`(e×e 단위), `O`(m×e 영), `E`(e×k)로 확장한 `pchke`(=`Ae`,`Be`) 생성.
4. 핵심 식: 생성행렬 `G = A^{-1} x B`, 확장은 `Ge = Ae^{-1} x B` — rate-compatible이라 상위 레벨 parity `Pe`가 하위 codeword에 그냥 append되어 기존 parity와 안 섞임.
5. 확장: 2~3개 P/E 구간(예: 2000, 4000에서 rate 전환)마다 다른 길이의 parity를 단계적으로 생성.
6. 구현 디테일(parity 관리): 1단 parity는 기존처럼 page spare에 저장, 2단 이상 parity는 별도 flash 영역에 수집 저장(4KB당 256B parity면 16 page분을 1 page에), DRAM 캐시로 관리, FTL 매핑테이블에 parity 정보 추적.
7. 디코딩 순서: 하위→상위 codeword 순으로 복호, 하위에서 성공하면 상위 parity 미인출로 retrieval overhead 절감.
8. 결과: rate 0.8/0.67/0.57 3단계 구성으로 시뮬, 확장 LDPC가 기본 대비 높은 frame error recovery rate(Figure 3)와 낮은 상대 read overhead(Figure 4)를 보임 — 수명 연장 주장.
9. 정량 수치: BER/overhead는 상대 곡선(그림)만 제시, 절대 gate count/throughput/iteration 수치 없음.
10. 한계: Work-in-Progress 4쪽 논문, 디코더 알고리즘·HW 설계·정정능력 정량 결과 부재, codeword length·연판정 방식 미기재.
