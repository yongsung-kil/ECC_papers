### B. 알고리즘 요약

1. 시스템: regular binary LDPC `(N,K)(dv,dc)`, 실험은 PEGReg `1008×504`(rate `0.5`, `dv=3`, girth 8), BPSK/AWGN.
2. 문제: BF는 저복잡도지만 BP(sum-product) 대비 성능 열위이고, SD BF는 일정 iteration 후 진동(loop)에 빠져 복호 실패.
3. 모델: syndrome `s = û·H^T (mod 2)`, flipping function(FF) `E_n` 최대값 VN을 뒤집는 반복 복호, AWGN 채널값 `y_n` 활용.
4. 핵심기법: 1단 SD BF(IMWBF/GDBF/MMWBF 등) 실패 시 loop detector가 감지하는 순간 2단 Gallager HD BF로 전환(잔여 소수 오류 정정), 3단은 RBF(random BF)로 2차 loop 탈출.
5. 핵심식: IMWBF FF `E_n = -α|y_n| - Σ w_mn(1-2s_m)` — scaling factor `α`가 잔여오류/성능을 좌우, 모든 단이 같은 BF 구조 공유해 복잡도 증가 최소.
6. 확장: loop detector는 `L1` iteration 후 활성화, 직전 `L2` iteration만 비교해 저장·연산 절감(SNR 무관 near-optimal SS timing).
7. 구현 디테일: Gallager FF는 다른 FF들의 특수형이라 파라미터 0/1 설정만으로 동일 디코더에서 구현, 채널값 의존 없는 HD BF가 loop breaker로 적합.
8. 최적화: 새 지표 `ρ(α)=P_B(α)·N/P_F(α)`(실패 프레임 조건부 평균 잔여오류)로 1단 `α`를 재튜닝(FER 최소 α≈0.2와 다름, ρ 최소는 α∈[0.5,1]).
9. 결과: PEGReg 504×1008에서 FER≈10^-5 기준 SD BF 대비 2SBF 약 1~1.5 dB, 3SBF는 추가 ~0.25 dB 이득, 적응 SS가 고정 임계보다 우수.
10. 한계: HW 미설계, 시뮬(FER)뿐, 일반 AWGN LDPC 대상(NAND 미실험), BF 계열이라 min-sum/BP 대비 절대성능은 낮음.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| 다단 BF 복호 이터레이션 루프/stage-switching | [decoder.cpp](../Prime_ECC_3.1_Claude/) `LDPC_Decoder()` | 이터레이션 루프에 loop-detection 기반 조기 알고리즘 전환을 접목 가능하나 min-sum 대체는 아님 |
| Gallager HD BF / SD BF FF(syndrome·채널값) | 대응 없음 | Prime ECC는 min-sum(`CNU_Update_New_Mag()`) 기반, BF flipping function 미대응 |
| loop 감지 기반 조기종료/전환 | [partial_CRC.cpp](../Prime_ECC_3.1_Claude/) `partial_crc_HW_T4()` | 조기종료 시점 판단이라는 목적은 유사(이미 partial/full CRC 보유), 중복성 높음 |

적용 가치: **중간** — binary LDPC 디코더 개선이라 관련성은 있으나, Prime ECC는 성능 우위인 min-sum + CRC 조기종료를 이미 보유해 BF 다단화 자체 이식은 downgrade이고 loop-detection 스위칭 아이디어만 참고 가치.
