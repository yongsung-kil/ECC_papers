> 총평: polar codes용 DNN BP 복호 지연 절감 기법으로 부호·복호 구조가 NAND binary LDPC와 달라 이식성 낮음.

### B. 알고리즘 요약

1. 시스템: 5G eMBB 제어채널용 (N,K) polar code, `Gp = F^⊗log2N`, BPSK/AWGN, NMS BP 복호, N=16/128/1024/2048 시뮬.
2. 문제: polar BP 복호 지연·복잡도 큼, 기존 early-stop(G-matrix, MinLLR)은 매 iteration마다 metric 검사해 파이프라인 중단 오버헤드 발생.
3. 모델: 정확한 복호함수를 수식화 어려워 DNN(3-hidden-layer, NH=128)으로 LLR→복호가능성/iteration수 매핑을 학습(model-free).
4. 핵심기법1 `decodability detection`: 깊은 페이딩(LLR≈0)으로 Nmax 반복해도 복호 불가한 'Type I' 블록을 사전 판별해 즉시 NACK, 불필요 복호 제거.
5. 핵심식: BE 기반 최적화 (식 8) — 라벨을 `1/NT·|u-û| ≥ γu`로 두고 γu(≈N/2)를 키워 예측정확도를 monotone 향상(Lemma 1).
6. 핵심기법2 `early-stopping prediction`: DNN `fTP`로 필요 최소 iteration수 `Tk^min`을 직접 예측(식 9), p=2일 때 정확도 최적.
7. 보정: 예측 부족분 대비 예측반복 후 CRC 검사 실패 시 나머지 (Nmax-fTP) 반복 수행하는 compensation으로 BLER 손실 방지.
8. 학습: Keras 3-hidden-layer DNN(NH, NH/2, NH/4), ReLU, γu·p 튜닝, LLR 데이터로 라벨링 학습.
9. 결과: N=1024, Eb/N0=5dB에서 BP 대비 복잡도 10배 감소, 복호지연 71% 절감, BLER는 BP/G-matrix/MinLLR과 동등 유지.
10. 한계: HW 미설계·시뮬만, polar code 전용(다른 부호 확장 미검증), 복잡도를 add=mul=comp로 단순 취급.
