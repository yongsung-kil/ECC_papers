### B. 알고리즘 요약

1. 대상: 60GHz WLAN용 IEEE 802.11ad·802.15.3c 두 표준의 LDPC(QC-LDPC, rate 1/2~7/8, 8개 PCM), layered decoding.
2. 문제: 두 표준이 PCM·code rate·expansion factor가 달라 각각 별도 디코더 HW가 필요 -> 구현 비용 증가.
3. 핵심 기법: `check node unit(CNU)`을 hardware-shared sorter로 구성해 두 표준을 하나의 재구성 HW로 공유.
4. block layer 기법: row weight가 interlaced인 인접 2 layer를 데이터 의존성 없이 동시 처리, 8개 행렬을 최대 4 block layer로 분할해 4 cycle에 완료.
5. sorter 재구성: `Sorter8`(8입력) 21세트 병렬, Sorter16=Sorter8x2+4to2 comparator, Sorter32=Sorter16x2로 8/16/32 스위칭.
6. min 탐색은 참조[5]의 sorter + NPMSA(normalized probabilistic min-sum) 기반, row weight 최대 32 지원하도록 재설계.
7. code rate별로 sorter 조합을 배정(예: 802.11ad rate-5/8은 block layer1,2를 Sorter16으로 병합).
8. 학습/최적화 절차 없음(순수 HW 아키텍처).
9. 결과: TSMC 40nm@200MHz, HW usage 802.15.3c 100%(7/8)·90%(1/2), 802.11ad 96%(13/16)·90%(1/2), sorter area 660~1263μm2·power 226~264μW.
10. 한계: BER/정정성능·전체 throughput·code length 미보고, sorter 모듈만 합성, NAND 무관 무선 표준 전용.
