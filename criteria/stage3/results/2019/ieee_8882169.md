### B. 알고리즘 요약

1. 시스템: `(204,102)` binary LDPC 코드, BPSK 변조, Middleton "Class A" impulse noise 채널(PLC 환경 상정), rate 0.5.
2. 문제: 표준 SPA는 채널 특성 지식(CSI, 잡음 종류·전력)을 요구하는데, impulse noise 채널에서 이 CSI 확보가 어렵고 부정확하다.
3. 채널모델: Class A는 서로 다른 분산 `σm²`의 zero-mean 가우시안 PDF들을 Poisson `Pm`으로 가중합한 잡음(식 1~3), 파라미터 `A=0.1`, `T=0.1`.
4. 핵심기법: 수신벡터 `r`에 가장 가까운 code word를 GA로 heuristic 탐색. 개체 `p`는 `[0,1]` 실수벡터, fitness 최소화로 최적 후보 `q` 선택 후 crossover/mutation.
5. 핵심식: fitness `ffitness=(Sw+1)(d+1)` — syndrome weight `Sw`(코드 소속)와 Euclidean distance `d`(수신벡터 근접도)를 동시 최소화, +1 offset으로 두 항 균형.
6. 확장: 수신 실수벡터를 sigmoid로 `[0,1]`에 압축(식 4)해 개체와 비교(식 5), 개체 성분이 evolving threshold로 동작.
7. 구현 디테일: population 400, elite 2, crossover 0.8, Gaussian mutation(variance 0.5, 0.75배 shrink), max generation 100.
8. 최적화: 한 수신벡터당 GA를 `M=15`회 독립 실행, 최저 syndrome(동률시 최소 distance) 후보 선택으로 local minima 회피.
9. 결과: 20 dB 이상 고 SNR에서 SPA보다 낮은 BER/FER, 10~18 dB에서는 SPA에 열세. Table I은 세대수(vertical)와 block수 M(horizontal)의 BER/FER trade-off.
10. 한계: 시뮬레이션(MATLAB)만, HW 미설계(FPGA는 향후계획), impulse noise 특정 채널 가정, GA 반복으로 latency 큼.

| syndrome 계산 | [decoder.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `LDPC_Decoder()` | syndrome 개념은 공유하나 GA fitness 용도는 무관 |
| sigmoid 압축 / soft 처리 | [channel.cpp](../Prime_ECC_3.1_Claude/paper_screening_profile.md) `Set_LLR_Th()` | soft-info 처리 접점이나 LLR 양자화 방식과 상이 |
| 병렬 genetic block 토폴로지 | 대응 없음 | z=32 row 병렬 min-sum HW와 구조 무관 |
