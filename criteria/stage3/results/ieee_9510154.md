### B. 알고리즘 요약

1. 시스템: DVB(ETSI) MF-TDMA 위성 MODEM 내부 LDPC 내부부호(외부부호 BCH 연접), rate `1/2`, 프레임 `16200` bit.
2. 문제: LDPC 복호의 반복 연산 비용이 크고 VLSI 전용 구현은 개발비가 높아, 유연·저비용 대안 플랫폼이 필요함.
3. 모델: 이진 LDPC를 `H·xᵀ=0`의 bipartite 그래프로 표현, PTB(parity→bit)·BTP(bit→parity)·parity-check 3단 반복.
4. 핵심기법: 복호를 CUDA 커널 4단으로 분할 - Kernel1 초기화, Kernel2 PTB(행/수평), Kernel3 BTP(열/수직), Kernel4 hard-decision parity check, 각 단이 독립 스레드에서 SIMT 병렬 실행.
5. 핵심 아이디어: PTB 각 계산이 상호 독립이라 스레드 간 병렬화 가능, 블록 내 GDDR 접근 경합은 `__syncthreads()`로 방지.
6. 부수 트릭: `cudaHostAlloc()` pinned memory로 CPU-GPU 전송 지연을 줄이고 GPU 스레드가 GDDR에 직접 접근하도록 우회.
7. 구현 디테일: hard-decision parity check로 오류 비트를 검출해 연산시간 절감, 전 과정을 병렬 처리.
8. 플랫폼: 4코어 1.6GHz CPU + NVIDIA GeForce MX150(2GB GDDR5, 384 CUDA cores@1469MHz), CUDA 8, Xilinx Virtex 보드와도 비교.
9. 결과: GPU가 25 iter에서 CPU 대비 1.26배 빠름, throughput rate½에서 25 iter시 190.3 Mbps(45 iter 79.2 Mbps로 감소), GPU가 CPU/FPGA보다 디코딩 시간 짧음.
10. 한계: BP 알고리즘 자체 신규성 없음(표준 복호의 GPU 이식), 복호 성능(BER/error-floor) 개선 없음, HW(RTL/ASIC) 미설계, GPU 특화 최적화라 NAND 컨트롤러 무관.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| PTB/BTP 반복 복호 (CN/VN 처리) | decoder.cpp LDPC_Decoder(), C2V_Cal() | 개념적 대응은 있으나 GPU SIMT 병렬화 방식은 HW pipeline(z=32) 구조와 상이 |
| hard-decision parity check 종료 | full_CRC.cpp / decoder.cpp (간접) | Prime ECC는 partial/full CRC 조기종료 사용, 단순 hard parity와 다름 |
| GPU pinned memory / GDDR 접근 최적화 | 대응 없음 | GPU 메모리 모델 특화, RTL 디코더에 이식 불가 |
```

적용 가치: **낮음** - 표준 DVB LDPC 복호의 GPU 병렬 구현 사례로, Prime ECC의 알고리즘/HW(min-sum, pipeline, HCU, GT)와 중복도 없고 대응 모듈도 GPU 최적화 부분은 전무하다.
