### B. 알고리즘 요약

1. AWGN 채널·BPSK, `(N,K)` FG-LDPC(PG-LDPC `(1057,813)`, EG-LDPC `(255,175)`/`(4095,3367)`) 대상 WBF 복호.
2. 기존 WBF는 iteration당 1비트만 flip해 수렴이 느리고, 같은 비트를 왕복 flip하는 무한루프로 복호 실패가 잦음.
3. flip 대상 비트를 error 함수 `En`(채널값 `|yn|`, check별 최소 `|y|min-m`, 최대 `|y|max-m`로 계산)으로 선택.
4. 관찰: M-WBF와 LC-WBF는 같은 iteration에서 서로 다른 비트를 flip할 확률이 높음(초기 ~85%, 안정화 ~60%, Fig.1).
5. 핵심 기법: 두 판정식을 동시 사용 - `En1 = Σ(2sm−1)|y|min-m − α·|yn|` (M-WBF), `En2 = Σ(|yn| − |y|min-m/2 − sm·|y|max-m)` (LC-WBF).
6. `p = argmax En1`, `q = argmin En2` 산출 후 `p≠q`면 두 비트 모두 flip, `p=q`면 한 비트만 flip.
7. iteration당 두 비트를 flip해 수렴 가속, 한쪽이 무한루프에 빠져도 다른 판정식이 다른 비트를 골라 루프 탈출.
8. 가중치 `α`는 코드별 튜닝(2.1 / 1.5 / 3).
9. 결과: iteration당 복잡도는 약 2배지만 동일 총복잡도 기준 CM-WBF(25 iter)가 타 WBF(100 iter)와 동등, WER 10^-3에서 LC-WBF 대비 약 0.2dB, RR/M-WBF 대비 약 0.1dB 개선, 평균 iteration 고SNR에서 절반.
10. 한계: HW 미설계·시뮬만, FG-LDPC·AWGN 한정, 연판정 magnitude 필요.

### C. Prime ECC 관련 모듈 핀포인트

```
| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| WBF 반복 복호 루프(En1/En2 기반 비트 선택) | decoder.cpp LDPC_Decoder(), C2V_Cal() | WBF는 min-sum과 다른 계열 - 루프 위치만 대응, 연산 상이 |
| 채널 magnitude |yn| 기반 신뢰도 | channel.cpp Set_R_Offset(), Set_LLR_Th() | WBF는 LLR 아닌 |y| magnitude 사용, 부분 대응 |
| 조기종료(패리티 만족 시 중단) | partial_CRC.cpp / full_CRC.cpp | 종료 조건은 CRC 보유, 개념만 유사 |
```

적용 가치: 낮음 — Prime ECC는 이미 min-sum + 적응형 LLR + partial/full CRC 조기종료를 보유했고, WBF 계열은 FG-LDPC·연판정 magnitude 의존이라 고rate binary QC-LDPC 파이프라인에 떼어 쓸 이득이 없다.
