### B. 알고리즘 요약

1. 5G NR QC-LDPC(BG1/BG2, code length 576~8424, 8단계 MCS 레벨, 코드율 0.153~0.875, spectral efficiency 0.153~2.8955), AWGN 채널, 레이어드(layered) 디코딩 구조.
2. 문제: 기존 layered NMS/OMS는 체크 라인 갱신 순서를 정적으로 고정하거나 동적 스케줄링(복잡도 급증)에 의존해 성능-복잡도 트레이드오프가 존재.
3. 채널/모델: AWGN, BPSK/QPSK/8PSK/16APSK 변조, 3GPP BG matrix 펑처링(첫 2열 puncture) 구조.
4. 핵심 기법: LNOMS(Layered Normalized Min-Sum)에 3-layer static scheduling 결합. 레이어1은 각 행의 펑처된 non-zero 원소 개수(`O0,O1,O2` 그룹)로 정렬해 먼저 업데이트, 레이어2는 동일 펑처 수 내에서 row weight 큰 행 우선, 레이어3은 나머지를 랜덤 순서.
5. 핵심 식: `m_cn→vm = α·sign(·)·(min1+min2, offset β 보정)` 정규화+오프셋 min-sum. 펑처 non-zero 2개인 행을 먼저 업데이트하면 min-sum이 0이 되어 정보량이 없으므로, 펑처 적은 행부터 처리해 파리티 정보를 조기 확보.
6. 3-layer 기여도: 레이어1/2/3이 BLER 개선의 각각 20%/70%/10% 기여(row weight 우선 반영한 레이어2가 지배적).
7. HW 구현 디테일: 정규화 factor α를 3/8~7/8로 근사해 시프트+덧셈으로 대체(곱셈기 제거), pre-processing 기반 routing unit(PRU)으로 ROM에 레이어별 초기 주소 저장해 barrel shifter/Banyan 없이 메모리 오버헤드 절감, 검증은 BG 상위 4행만 사용(improved check unit)해 검증시간 단축.
8. 최적화 절차: normalization factor는 사전 curve-fitting 시뮬레이션으로 8개 rate별 LUT 구성.
9. 결과: NMS(α=0.75)/OMS(β=0.5) 대비 평균 BLER 0.1dB 개선, 전통 LNMS 대비 0.5dB 개선, 평균 iteration 약 1회 감소(OMS/NMS 대비), throughput 625Mbps @200MHz(XC7VX-690T, LUT 41364/기존 대비 개선, BRAM·register 절감), 검증 사이클 시간 평균 25% 감소.
10. 한계: LEO 위성통신 특화 5G NR BG 구조 전제, BP 대비는 여전히 성능 열세, error-floor 영역 별도 분석 없음(주로 BLER-SNR 곡선 중심), gatecount는 미제공(LUT 수치만 제시).

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| LNOMS(Normalized+Offset Min-Sum) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | Prime ECC도 min-sum 기반이나 정규화/오프셋 factor 조합은 별도 검토 필요 |
| 3-layer static scheduling (펑처 수/row weight 기반 레이어 순서) | `decoder.cpp` `LDPC_Decoder()` 이터레이션 루프, Dual-Update 스케줄 | Prime ECC의 Dual-Update(even/odd 교대)와 다른 축의 스케줄링(펑처 기반 우선순위)이라 병행 적용 검토 가치 |
| 정규화factor 근사(3/8~7/8 시프트연산) | `ecc_data.h` `PARAM_LLR` | LLR 스케일링 테이블과 유사 계열, 근사 비트폭 정합 확인 필요 |
| Pre-processing 기반 Routing Unit(PRU, ROM 주소 사전계산) | 대응 없음 | Prime ECC는 z=32 고정 row-parallel 구조로 barrel shifter/PRU에 대응하는 별도 라우팅 모듈 표기 없음 |
| Improved check unit(상위 4행만 검증) | `partial_CRC.cpp` `partial_crc_HW_T4()` | 조기 검증/조기종료 취지는 유사하나 방식은 CRC가 아닌 부분 parity-check라 상이 |

적용 가치: `중간` — 기반 min-sum/layered 기법은 Prime ECC와 중복되지만, 펑처 정보 기반 3-layer static scheduling과 근사 정규화factor 시프트 연산은 이식 검토할 만한 독립적 개선 포인트.
