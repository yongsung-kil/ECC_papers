### B. 알고리즘 요약

1. 시스템: WCDMA용 rate R=1/3 Turbo code (component RSC `g=(1,(1+D+D3)/(1+D2+D3))`), interleaver 길이 512, AWGN 채널.
2. 문제: BCJR/MAP/Log-MAP/SOVA 등 정통 Turbo 복호는 연산복잡도·지연 커서 HW 부적합.
3. 가정: Turbo 부호의 parity-check matrix가 매우 sparse → LDPC의 특수 케이스로 간주(semi-infinite low-density H).
4. 핵심 기법1: RSC를 check matrix `H`로 표현해 Tanner graph 구성, LDPC Min-Sum(MPA)으로 복호.
5. 핵심 식: CNU `L(rji)=∏sign(L(qi'j))·min|L(qi'j)|`, VNU `L(qij)=L(ci)+Σ L(rj'i)` — tanh 대신 min으로 근사해 저복잡도.
6. 확장: CNU를 1st-min/2nd-min/position 기반으로 재구성(현재 위치가 min 위치면 2nd-min, 아니면 1st-min 사용).
7. 구현 디테일: 열단위 압축저장(`H_ccs`, 행수=column weight), C2V(`L(rji)`)와 V2C(`L(qij)`)를 한 MEMORY에 교대 저장해 메모리 대폭 절감.
8. HW: semi-parallel 구조(CNP/VNP, CNP_Router/VNP_Router/Switch), CNP=CAVS(cpr_min/cpr_min2)+CSS, VNP=누산·clipping·MSB decision.
9. 결과: ModelSim SE6.0 검증, 100MHz에서 throughput 60.7Mbps, Synopsys SMIC 0.18um 합성 시 24.5K gate + 75K memory bit.
10. 한계: short cycle·짧은 블록으로 BER가 정통 Turbo/SOVA보다 열화, iteration 20 필요, NAND 무관.

### C. Prime ECC 관련 모듈 핀포인트

| 논문 요소 | Prime ECC 모듈 | 관련성 |
|---|---|---|
| Min-Sum CNU (1st/2nd min + sign XOR) | `decoder.cpp` `CNU_Update_New_Mag()`, `C2V_Cal_New_Sgn()` | 이미 보유한 min-sum과 동일 개념, 신규성 없음 |
| VNU 누산·decision | `decoder.cpp` `VN_Cal_HD()` 등, `LDPC_Decoder()` | 표준 VN 처리, 중복 |
| C2V/V2C 교대 압축저장(H_ccs) | `decoder.cpp` `LDPC_Decoder()` 메모리 (z=32 구조) | 우리는 z=32 row-병렬 고정, Turbo용 CCS 저장은 정합 어려움 |
| Turbo(RSC) 부호 자체 | 대응 없음 | Prime ECC는 QC-LDPC 고정, Turbo 미지원 |

적용 가치: **낮음** — Min-Sum·min1/min2 CNU·압축저장은 이미 보유했고, 대상이 Turbo 부호이며 성능이 오히려 열화되어 NAND binary QC-LDPC 이식 이득이 없음.
