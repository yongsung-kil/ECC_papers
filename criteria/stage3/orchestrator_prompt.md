# Stage 3 오케스트레이터 프롬프트 (멀티 agent 병렬 실행)

> 위치: criteria/stage3/orchestrator_prompt.md
> 사용처: 메인 Claude(오케스트레이터)에게 이 파일 내용을 그대로 지시문으로 준다.
> 짝 문서: [analysis_prompt.md](analysis_prompt.md) - 각 subagent가 따라야 할 분석 절차.


---

## 작업

[analysis_prompt.md](analysis_prompt.md) 의 절차를 따라 `data/pdfs/` 아래 미처리 논문(**.txt 추출본**)을 분석한다.
- 결과 저장 위치: `criteria/stage3/results/{연도}/{id_underscore}.md` (한 편당 한 파일, **연도별 폴더**)
  - `id_underscore` = 논문 id의 `:`를 `_`로 치환(점·버전은 유지). 예: `ieee:11123581`→`ieee_11123581`, `arxiv:1012.3201v1`→`arxiv_1012.3201v1`.
  - `{연도}` = 처리한 `.txt`가 있던 `data/pdfs/{ieee|arxiv}/{연도}/**/*.txt` 경로의 그 **연도 그대로** 사용한다 (analysis 헤더에 적은 연도가 아님). `YEAR`가 범위(`2023-2025`)나 `all`이면 논문마다 자기 `.txt`가 실제로 위치한 개별 연도 폴더를 그대로 쓴다(예: `2023-2025` 처리 중이라도 각 논문은 자기 연도의 `results/2023/`, `results/2024/`, `results/2025/` 중 해당하는 곳에 저장).
    - 이 값은 `papers.db`의 `published` 연도와 항상 일치하도록 이미 구성돼 있다(파일 다운로드 시 그 기준으로 분류됨). 분석 중 논문 본문에서 다른 연도가 읽히더라도(예: 컨퍼런스 발표일 vs 저널 게재일) **폴더는 반드시 입력 경로 연도를 따른다** — 안 그러면 연도별 진행률 집계가 어긋난다.
  - 연도 폴더가 없으면 새로 생성한다.
- "미처리 논문" 판정: 논문이 있던 `data/pdfs/.../{연도}/`와 같은 이름의 `criteria/stage3/results/{연도}/`에 `{id_underscore}.md`가 없는 논문 (입력 경로 연도와 출력 폴더 연도가 항상 같으므로 그 논문의 연도 폴더만 보면 된다)

## 입력 파라미터 (사용자가 호출 시 지정)

사용자는 이 프롬프트를 호출할 때 **연도(`YEAR`) 하나만** 넘긴다.
오케스트레이터는 **사용자에게 되묻지 말고** 받은 값으로 자동 분배한다.

| 파라미터 | 의미 | 예시 |
|---|---|---|
| `YEAR` | 처리 대상 연도 (단일 또는 범위) | `2024` / `2023-2025` / `all` |

- 처리 대상 경로: `data/pdfs/{ieee|arxiv}/{YEAR}/**/*.txt` (`YEAR=all` 이면 모든 연도)
  - **PDF는 이 환경에서 Read 불가(poppler/`pdftoppm` 미설치) + 편당 수십 페이지라 토큰 과다 + git 미추적. 반드시 `.txt` 추출본을 읽는다.**
- `YEAR`가 없으면 그 때만 사용자에게 한 번 확인한다.

## 고정 정책 (변경 금지)

| 항목 | 값 |
|---|---|
| subagent 1명당 처리 편수 | **정확히 3편** (3편 끝나면 즉시 종료) |
| 한 라운드 동시 호출 subagent 수 | **10명** |
| 한 라운드 처리량 | **30편** |
| agent 재사용 | **금지** (`SendMessage` 사용 금지) |

처리량이 3편 미만으로 남는 경우(마지막 라운드)에만 묶음 크기가 1~2편이 될 수 있다.

---

## 병렬 처리 규칙 (엄수)

### 라운드 구성
- **한 라운드 = 10개의 subagent를 동시 호출**.
  반드시 **하나의 메시지 안에 10개의 `Agent` tool_use 블록**을 넣는다.
  (메시지를 쪼개서 순차 호출하면 안됨 - 병렬 효과가 사라짐)
- 각 subagent에게 **논문 3편을 한 묶음**으로 할당한다.
  (한 라운드에서 **30편** 처리)
- 마지막 라운드에서 남은 편수가 30미만이면 가능한 만큼만 할당
  (예: 17편 남으면: agent 5명 x 3편 + agent 1명 x 2편).

### Agent 재사용 금지 (가장 중요)
- subagent 한 명이 자기 할당분을 끝내면 **그 agent는 폐기**한다.
- **절대 `SendMessage`로 기존 agent를 재사용하지 말 것.**
- 다음 라운드는 반드시 **새 `Agent` 호출로 fresh context**의 agent 10명을 새로 만들어 시작한다.
  - 이유 1: context 누적시 토큰 비용 증가
  - 이유 2: context 누적 시 이전 논문의 판단이 다음 논문에 누설(leak)되어 품질 저하
  - 이유 3: agent 자체 context 한계로 압축이 일어나면 분석 일관성이 깨짐

### 라운드 반복
- 라운드 종료 시 결과 확인:
  - 성공한 묶음은 처리 완료 목록에 추가
  - 실패/누락한 묶음은 다음 라운드에 **새 agent로** 재투입
- **남은 미처리 논문이 0편이 될 때가지** 라운드 반복.

### subagent 타입
- `subagent_type`: `general-purpose` 또는 `claude` (도구 제약 없는 타입).
  `Explore`는 read-only라 결과 저장 불가라 사용금지.

## 각 agent에게 줄 프롬프트 템플릿

오케스트레이터는 매 라운드 각 subagent를 호출할 때 아래 템플릿을 채워 보낸다.
**프롬프트가 자기완결적이어야 한다** - agent는 이 대화의 맥락을 모른다.

```
너는 LDPC ECC 논문 분석 작업자다.

[필수 사전 읽기]
1. criteria/stage3/analysis_prompt.md - 분석 절차, 출력 형식
2. criteria/stage3/categories.md - enum 목록
3. criteria/stage3/Prime_ECC_3.1_Claude/paper_screening_profile.md - Prime ECC 3.1 코드 프로파일 (판정 기준, 자기완결 / 코드 접근 불필요)

[할당된 논문 (정확히 3편 - 마지막 batch만 1~2편 가능)]
(각 논문의 저장 연도는 그 `.txt` 경로 `data/pdfs/{ieee|arxiv}/{연도}/...`의 연도를 그대로 쓴다 - 아래 예시처럼 경로에 이미 포함돼 있음)
- {paper_path_1} (예: data/pdfs/ieee/2024/03/ieee_XXXXXXX.txt → 연도 2024)
- {paper_path_2}
- {paper_path_3}

이 3편을 다 처리하면 즉시 작업을 종료하고 결과를 반환한다.
**추가 작업을 자발적으로 가져오지 말 것.** (다음 작업은 새 agent가 받음)

[수행 절차 - 한 편씩 순차 처리]
할당된 논문을 **위에서부터 한 편씩** 처리한다.
**한 편을 완전히 끝내기 전까지 다음 편의 .txt를 읽지 말것.**
(여러 편을 동시에 Read 해서 컨텍스트에 쌓아두면 분석 품질이 떨어진다.)

각 논문 1편 처리 사이클:
1. 해당 논문의 .txt 본문을 Read 로 읽는다. (이 한 편만)
2. analysis_prompt.md의 "출력 형식 (A/B/C/D)" 그대로 생성한다.
3. criteria/stage3/results/{연도}/{id_underscore}.md 에 Write 로 저장한다. (`{연도}` = 이 논문의 `.txt` 경로에 있던 연도. 본문에서 다른 연도가 읽혀도 폴더는 항상 경로 연도 사용)
    - 연도 폴더가 없으면 새로 생성한다.
    - 이미 파일이 존재하면 건너뛰고 그 사실을 보고한다 (덮어쓰지 말 것).
4. 저장 완료를 확인한 뒤에야 **다음 편**으로 넘어간다.
5. enum 오타 금지. 본문 근거 없으면 `미상`. 추측 금지.

3편을 모두 위 사이클로 끝내면 [반환값] 형식으로 보고하고 종료한다.

[반환값]
다음 형식으로만 반환하라 (자유 텍스트 금지):

DONE:
- {paper_id_1} -> results/{연도_1}/{id_underscore_1}.md
- {paper_id_2} -> results/{연도_2}/{id_underscore_2}.md

SKIPPED:
- {paper_id} (이미 존재)

FAILED:
- {paper_id} (사유 한 줄)
```

## 진행 보고 포맷

매 라운드 종료 시 사용자에게 **한 줄** 로 보고:

```
Round K: 완료 X편 / 스킵 Y편 / 실패 Z편 / 남은 W편
```

3라운드 연속으로 신규 완료가 0편이면 자동 중단하고 원인 보고.

## 시작 절차
0. 대상 연도 폴더(들) `criteria/stage3/results/{연도}/`가 없으면 생성한다. (`YEAR`가 범위/`all`이면 포함된 각 연도마다)
1. 사용자가 넘긴 `YEAR`로 대상 `.txt` 경로 목록을 만든다.
   - `data/pdfs/{ieee|arxiv}/{YEAR}/**/*.txt`
2. 각 논문을 **그 논문이 있던 연도 폴더** `criteria/stage3/results/{그 논문의 연도}/`와 대조해 **미처리분**만 추린다.
3. 미처리 목록을 **3편씩** 묶어 batch로 분할 (마지막 batch는 1~2편 가능).
4. **10 batch = 1 라운드**. 라운드 단위로 처리.
5. 첫 라운드 시작 전 한 번만 보고:
    - 총 미처리 편수 /batch 수 / 예상 라운드 수
6. 사용자 승인 없이 바로 라운드 루프 시작 (`YEAR`가 명시적으로 주어졌으므로).
