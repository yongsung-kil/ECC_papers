# Phase 3 카테고리 / 출력 스키마 (enum 기반)

판정 대상: **NAND 고전 바이너리 LDPC ECC**에 이식 관점.
각 항목은 **정해진 선택지(enum) 중 하나**만 고른다. 나중에 필터링 가능.
숫자 항목(rate, length, gatecount, throughput)은 본문 수치 그대로(없으면 `미상`)
자유 텍스트는 핵심기여, 한계, 추가확인 3개 뿐.
`[추천]` = 사용자 요청 외 내가 추가 제안 (승인 후 확정).

---

## A. 공통
| 카테고리 | 선택지(enum) | 비고 |
|---|---|---|
| **이식성** | `상` \| `중` \| `하` | NAND 바이너리 LDPC에 떼어 쓸 수 있는 정도. **핵심** |
| NAND관련성| `직접` \| `간접` \| `낮음` | 직접=NAND/SSD 실험 or 타깃 / 간접=일반LDPC 이식가능 / 낮음=억지|
| 대상 | `decoder` \| `encoder` \| `both` \| `code-design` \| `other` | 부호설계가 정정능력/error-floor에 직접 기여하면 `both` (부호설계+복호성능). 순수 구성법·이론뿐이면 `code-design` |
| 부호종류 [추천] | `QC-LDPC` \| `SC-LDPC` \| `regular` \| `irregular` \| `protograph` \| `기타` \| `미상` | |
| 부호rate [추천] | 숫자 \| `미상` | 예: 0.9. 구성법 논문 등 예제 다수면 범위표기 가능(예: `0.51~0.97`) |
| 부호length [추천] | 숫자(bit/kB) \| `미상` | 예제 다수면 범위표기 가능(예: `63~66048`) |
|연판정 [추천] | `hard-1bit` \| `soft-2~3bit` \| `soft-4bit+` \| `무관` \| `미상` | NAND read 비용 직결 |
| 핵심기여 | (자유 텍스트 한 줄) | |

## 결과 파일 형식 ('results/{id}.md')
- 상단: **2열 표**(카테고리 | 값) - 값은 위 enum/숫자만.
- 하단: 동일 내용 **JSON 블록** (DB 적재, 기계 필터용).
