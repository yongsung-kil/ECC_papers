# 1차 선별 기준 (Selection Criteria)

> 이 문서는 **살아있는 기준서**입니다. 선별을 거듭하며 경계 사례·새 카테고리를 계속 추가합니다.
> 매 선별 세션 시작 시 이 문서를 먼저 읽고, 끝나면 새로 배운 판단을 아래 **변경 이력**에 append 합니다.

## 0. 대전제 (가장 중요)

**"NAND 플래시에 직접 쓰인 논문"만 통과가 아니다.**
다음 중 하나라도 해당하면 **NAND에 활용 가능**한 것으로 본다:

1. **디코더에 쓰이는 기법** — LDPC 복호 알고리즘(BP, min-sum, OSD, 신경망 디코더, 스케일링/가중치 적응 등)
2. **하드웨어로 구현 가능** — 디코더/인코더의 VLSI·FPGA·병렬/파이프라인 아키텍처, 메모리 접근 최적화, permutation network 등
3. **관련 알고리즘/설계** — LDPC 부호 구성(QC-LDPC, girth/짧은사이클 제거, error floor, 유한길이, 비이진), LLR/연판정/양자화/read-retry 등

즉 **"이 논문의 기법을 떼어내 NAND LDPC ECC 시스템(부호 설계 → 디코더 알고리즘 → HW 구현)에 이식할 수 있는가?"** 가 판단의 축이다.
응용 도메인(무선/통신/우주)이 NAND가 아니더라도, **이식 가능한 디코더·HW·코드설계 기법이 있으면 통과**.

## 1. 통과(INCLUDE) 카테고리

| 카테고리 | 설명 | 점수대 |
|---|---|---|
| **A. 직접 NAND/SSD/플래시** | NAND·SSD·플래시 메모리 LDPC ECC, read-retry, LLR 양자화, retention/wear, 3D TLC/QLC, 컨트롤러 | 0.90~1.00 |
| **B. 스토리지 ECC 일반** | 디스크/erasure 등 저장장치 ECC(비이진 LDPC for storage 등) — NAND로 전이 쉬움 | 0.60~0.85 |
| **C. 이식 가능 디코더 알고리즘** | 도메인 무관 LDPC 복호 알고리즘(min-sum 변형, BP 개선, OSD, 신경망/메시지패싱 디코더) | 0.40~0.65 |
| **D. 이식 가능 HW 아키텍처** | LDPC 디코더/인코더 HW 구현(FPGA/VLSI), 병렬화, permutation network, 메모리 충돌 회피 | 0.40~0.65 |
| **E. 이식 가능 코드 설계** | QC-LDPC·SC-LDPC 구성, girth/짧은사이클 제거, error floor 저감, 유한길이 스케일링, 비이진 LDPC | 0.40~0.60 |

## 2. 제외(EXCLUDE) 카테고리

| 카테고리 | 이유 | 점수대 |
|---|---|---|
| **양자 LDPC(qLDPC)/양자 EC** | 안정자 부호·degeneracy 등 NAND와 부호/채널 구조가 근본적으로 다름 | 0.00~0.15 |
| **QKD / Wiretap / 보안** | 키분배·정보이론 보안 응용, 스토리지 ECC와 무관 | 0.00~0.20 |
| **무선/통신 응용 특이적** | OFDM·5G·802.11 등에서 LDPC가 *부수적으로만* 언급, 떼어낼 이식 기법 없음 | 0.15~0.35 |
| **순수 이론 bound** | 디코더/HW/구성으로 이어지지 않는 점근적 bound·존재성 증명만 | 0.20~0.35 |

> 주의: 같은 무선/양자 논문이라도 **디코더 알고리즘·HW 아키텍처가 명확히 이식 가능**하면 C/D로 통과시킨다.
> (예: "양자 LDPC용 적응형 min-sum"이라도 min-sum 스케일링 기법 자체는 NAND 디코더에 적용 가능 → 약한 통과)

## 3. 점수 → 결정

- **decision = "in"** if score ≥ **0.40**, else **"out"**
- 점수 루브릭:
  - `0.90–1.00` A: NAND 직접
  - `0.60–0.89` B: 스토리지 ECC / NAND에 바로 적용
  - `0.40–0.59` C·D·E: NAND에 이식 가능한 알고리즘·HW·코드설계
  - `0.20–0.39` 약한 관련(이론적/응용특이적, 일부 기법만 추출 가능) → 보통 out
  - `0.00–0.19` 무관(양자/QKD/순수통신) → out

## 4. 판단 절차

1. 제목·초록에서 **떼어낼 수 있는 기법**을 먼저 찾는다 (디코더? HW? 코드설계?).
2. 있으면 도메인과 무관하게 C/D/E 후보 → 이식 난이도로 점수.
3. 없고 NAND/스토리지 직접이면 A/B.
4. 떼어낼 기법이 없고 NAND/스토리지도 아니면 제외.
5. 애매하면 **약한 통과(0.40~0.45)** 로 살려두고 `reason`에 근거를 남긴다 (Phase 3에서 재검토).

## 5. 변경 이력 (여기를 계속 키운다)

### 2026-06-15 — 최초 작성
- 대전제 확립: NAND 직접 여부가 아니라 **이식 가능한 디코더/HW/알고리즘 여부**로 판단.
- 초기 36건 선별에서 도출된 경계 사례:
  - 양자 LDPC·QKD가 arXiv 최신분의 대다수 → 기본 제외. 단 *디코더 알고리즘이 명확히 이식 가능하면* 약한 통과로 재고.
  - LDPC 디코더 HW 아키텍처(무선·우주용 포함)는 NAND 컨트롤러에 이식 가능 → 통과(0.45~0.55).
  - error floor/유한길이 스케일링 분석은 NAND 저BER 설계에 유용 → 약한 통과.
